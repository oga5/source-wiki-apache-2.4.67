1Install Lua 5.1
2    http://www.lua.org/download.html
3    
4    Lua does not use autoconf for compiling. This means that you do not use
5    ./configure. It has good build instructions, though, so hopefully things
6    will go smoothly.
7    
8    I like to change the directory Lua installs to. In order to do this you
9    need to set LUA_TOP in the configuration makefile for Lua. For these 
10    instructions I have set LUA_TOP to /Users/brianm/.opt/lua-5.1.2 -- you
11    will see this directory referred to later.
12    
13    
14Install Apache HTTPD 2.2
15    http://httpd.apache.org/download.cgi
16    
17    You can build apache pretty much any way you like, as long as you enable
18    dynamic module loading (--enable-so) so that mod_wombat can be loaded.
19    
20    You may user (and I encourage you to!) the threaded MPMs -- mod_wombat
21    plays nicely with them.
22
23    I build it with these flags:
24        
25    ./configure --prefix=/Users/brianm/.opt/httpd-2.2.4-worker-wombat \
26                --with-mpm=worker  \
27                --enable-so
28    
29    
30Install libapreq2
31    http://httpd.apache.org/apreq/download.cgi
32        The download link is in the page body, NOT under the "Download!" link
33        in the left hand column.
34    
35    Right now, mod_wombat requires libapreq2 for parsing entity bodies. This
36    dependency will probably be made optional in the near future, but for now
37    you need it.
38    
39    I build it with these flags:
40    
41    ./configure --prefix=/Users/brianm/.opt/libapreq2-2.0.8 \
42      --with-apache2-apxs=/Users/brianm/.opt/httpd-2.2.4-worker-wombat/bin/apxs
43    
44    
45Install mod_wombat from subversion
46    http://svn.apache.org/repos/asf/httpd/mod_wombat/trunk
47    
48    The first step, when building from subversion, is to bootstrap autoconf. 
49    To do this run the bootstrap script:
50    
51    ./bootstrap
52    
53    The bootstrap script may report an error that it cannot find
54    libtoolize or glibtoolize. That is fine as long as it 
55    doesn't report that it cannot find both of them. The script
56    just sets up the autoconf magic. 
57    
58    After that, it is a normal configure and build:
59    
60    ./configure  --with-lua=/Users/brianm/.opt/lua-5.1.2/ \
61      --with-apxs=/Users/brianm/.opt/httpd-2.2.4-worker-wombat/bin/apxs \
62      --with-apreq2=/Users/brianm/.opt/libapreq2-2.0.8/
63      
64    If compiling (make) reports an error that it cannot find the
65    libapreq2 header file, please tell me ( brianm@apache.org )
66    as this occurs under some configurations but we haven't 
67    hammered down the weird things libapreq2 does with its
68    install. If you build libapreq2 with a --prefix configuration
69    option, it always seems to work.
70  
71      
72That is it. To configure mod_wombat, look at the basic-configuration.txt document.
