1See sample_httpd.conf for examples
2
3The basic module loading directive is
4    LoadModule lua_module modules/mod_lua.so
5
6The handler name is "lua-script" so you can use the normal
7AddHandler directive, such as "AddHandler lua-script .lua" to
8set anything ending in .lua to use mod_lua to evaluate
9
10mod_lua exports several additional directives:
11
12    LuaRoot /path/to/a/directory
13        Specify the base path which will be used to evaluate all
14        relative paths within mod_lua. If not specified they
15        will be resolved relative to the current working directory,
16        which may not always work well for a server.
17
18    LuaScope once|request|conn|server [max|min max]
19        Specify the lifecycle scope of the Lua interpreter which will
20        be used by handlers in this "Directory." The default is "once"
21        
22        once: use the interpreter once and throw it away.
23        
24        request: use the interpreter to handle anything based on 
25                 the same file within this request, which is also 
26                 request scoped.
27                 
28        conn: Same as request but attached to the connection_rec
29        
30        server: This one is different than others because the
31                server scope is quite long lived, and multiple threads
32                will have the same server_rec. To accommodate this
33                server scoped interpreter are stored in an apr
34                resource list. The min and max arguments are intended
35                to specify the pool size, but are unused at this time.
36
37    LuaMapHandler uri-pattern /path/to/lua/script.lua [function-name]
38        This directive matches a uri pattern to invoke a specific
39        handler function in a specific file. It uses PCRE regular
40        expressions to match the uri, and supports interpolating
41        match groups into both the file path and the function name
42        be careful writing your regular expressions to avoid security
43        issues.
44        
45        Examples:
46            LuaMapHandler /(\w+)/(/w+) /scripts/$1.lua handle_$2
47                This would match uri's such as /photos/show?id=9
48                to the file /scripts/photos.lua and invoke the
49                handler function handle_show on the lua vm after
50                loading that file.
51                
52            LuaMapHandler /bingo /scripts/wombat.lua
53                This would invoke the "handle" function, which
54                is the default if no specific function name is
55                provided.
56    
57    LuaPackagePath /path/to/include/?.lua
58        Add a path to lua's module search path. Follows the same
59        conventions as lua. This just munges the package.path in the 
60        lua vms.
61        
62        Examples:
63            LuaPackagePath /scripts/lib/?.lua
64            LuaPackagePath /scripts/lib/?/init.lua
65
66    LuaPackageCPath /path/to/include/?.soa
67        Add a path to lua's shared library search path. Follows the same
68        conventions as lua. This just munges the package.cpath in the 
69        lua vms.
70        
71        Examples:
72            LuaPackagePath /scripts/lib/?.so
73
74    LuaCodeCache stat|forever|never
75        Specify the behavior of the in-memory code cache. The default
76        is stat, which stats the top level script (not any included
77        ones) each time that file is needed, and reloads it if the
78        modified time indicates it is newer than the one it has
79        already loaded. The other values cause it to keep the file
80        cached forever (don't stat and replace) or to never cache the 
81        file.
82        
83        In general stat or forever is good production and stat or never
84        for development.
85        
86        Examples:
87            LuaCodeCache stat
88            LuaCodeCache forever
89            LuaCodeCache never
90    
91    LuaHookTranslateName  /path/to/lua/script.lua  hook_function_name
92        Add a hook (at APR_HOOK_MIDDLE) to the translate name phase of
93        request processing. The hook function receives a single
94        argument, the request_rec, and should return a status code, 
95        which is either an HTTP error code, or the constants defined
96        in the apache2 module: apache2.OK, apache2.DECLINED, or
97        apache2.DONE. 
98
99        For those new to hooks, basically each hook will be invoked
100        until one of them returns apache2.OK. If your hook doesn't
101        want to do the translation it should just return
102        apache2.DECLINED. If the request should stop processing, then
103        return apache2.DONE.
104
105        Example:
106            LuaHookTranslateName /scripts/conf/hooks.lua silly_mapper
107
108            -- /scripts/conf/hooks.lua --
109            function silly_mapper(r)
110                if r.uri == "/" then
111                    r.file = "/var/www/home.lua"
112                    return apache2.OK
113                else
114                    return apache2.DECLINED
115                end
116            end
117
118    LuaHookFixups  /path/to/lua/script.lua  hook_function_name
119        Just like LuaHookTranslateName, but executed at the fixups phase
120
