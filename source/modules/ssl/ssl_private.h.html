638/* BIG FAT WARNING: SSLModConfigRec has unusual memory lifetime: it is
639 * allocated out of the "process" pool and only a single such
640 * structure is created and used for the lifetime of the process.
641 * (The process pool is s->process->pool and is stored in the .pPool
642 * field.)  Most members of this structure are likewise allocated out
643 * of the process pool, but notably sesscache and sesscache_context
644 * are not.
645 *
646 * The structure is treated as mostly immutable after a single config
647 * parse has completed; the post_config hook (ssl_init_Module) flips
648 * the bFixed flag to true and subsequent invocations of the config
649 * callbacks hence do nothing.
650 *
651 * This odd lifetime strategy is used so that encrypted private keys
652 * can be decrypted once at startup and continue to be used across
653 * subsequent server reloads where the interactive password prompt is
654 * not possible.
655
656 * It is really an ABI nightmare waiting to happen since DSOs are
657 * reloaded across restarts, and nothing prevents the struct type
658 * changing across such reloads, yet the cached structure will be
659 * assumed to match regardless.
660 *
661 * This should really be fixed using a smaller structure which only
662 * stores that which is absolutely necessary (the private keys, maybe
663 * the random seed), and have that structure be strictly ABI-versioned
664 * for safety.
665 */ view japanese
