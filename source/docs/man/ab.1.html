33.SH "SUMMARY"
34 
35.PP
36ab is a tool for benchmarking your Apache Hypertext Transfer Protocol (HTTP) server\&. It is designed to give you an impression of how your current Apache installation performs\&. This especially shows you how many requests per second your Apache installation is capable of serving\&.
37 
38
39.SH "OPTIONS"
40 
41 
42.TP
43\fB-A \fIauth-username\fR:\fIpassword\fR\fR
44Supply BASIC Authentication credentials to the server\&. The username and password are separated by a single \fB:\fR and sent on the wire base64 encoded\&. The string is sent regardless of whether the server needs it (\fIi\&.e\&.\fR, has sent an 401 authentication needed)\&.  
45.TP
46\fB-b \fIwindowsize\fR\fR
47Size of TCP send/receive buffer, in bytes\&.  
48.TP
49\fB-B \fIlocal-address\fR\fR
50Address to bind to when making outgoing connections\&.  
51.TP
52\fB-c \fIconcurrency\fR\fR
53Number of multiple requests to perform at a time\&. Default is one request at a time\&.  
54.TP
55\fB-C \fIcookie-name\fR=\fIvalue\fR\fR
56Add a \fBCookie:\fR line to the request\&. The argument is typically in the form of a \fB\fIname\fR=\fIvalue\fR\fR pair\&. This field is repeatable\&.  
57.TP
58\fB-d\fR
59Do not display the "percentage served within XX [ms] table"\&. (legacy support)\&.  
60.TP
61\fB-e \fIcsv-file\fR\fR
62Write a Comma separated value (CSV) file which contains for each percentage (from 1% to 100%) the time (in milliseconds) it took to serve that percentage of the requests\&. This is usually more useful than the 'gnuplot' file; as the results are already 'binned'\&.  
63.TP
64\fB-E \fIclient-certificate-file\fR\fR
65When connecting to an SSL website, use the provided client certificate in PEM format to authenticate with the server\&. The file is expected to contain the client certificate, followed by intermediate certificates, followed by the private key\&. Available in 2\&.4\&.36 and later\&.  
66.TP
67\fB-f \fIprotocol\fR\fR
68Specify SSL/TLS protocol (SSL2, SSL3, TLS1, TLS1\&.1, TLS1\&.2, or ALL)\&. TLS1\&.1 and TLS1\&.2 support available in 2\&.4\&.4 and later\&.  
69.TP
70\fB-g \fIgnuplot-file\fR\fR
71Write all measured values out as a 'gnuplot' or TSV (Tab separate values) file\&. This file can easily be imported into packages like Gnuplot, IDL, Mathematica, Igor or even Excel\&. The labels are on the first line of the file\&.  
72.TP
73\fB-h\fR
74Display usage information\&.  
75.TP
76\fB-H \fIcustom-header\fR\fR
77Append extra headers to the request\&. The argument is typically in the form of a valid header line, containing a colon-separated field-value pair (\fIi\&.e\&.\fR, \fB"Accept-Encoding: zip/zop;8bit"\fR)\&.  
78.TP
79\fB-i\fR
80Do \fBHEAD\fR requests instead of \fBGET\fR\&.  
81.TP
82\fB-k\fR
83Enable the HTTP KeepAlive feature, \fIi\&.e\&.\fR, perform multiple requests within one HTTP session\&. Default is no KeepAlive\&.  
84.TP
85\fB-l\fR
86Do not report errors if the length of the responses is not constant\&. This can be useful for dynamic pages\&. Available in 2\&.4\&.7 and later\&.  
87.TP
88\fB-m \fIHTTP-method\fR\fR
89Custom HTTP method for the requests\&. Available in 2\&.4\&.10 and later\&.  
90.TP
91\fB-n \fIrequests\fR\fR
92Number of requests to perform for the benchmarking session\&. The default is to just perform a single request which usually leads to non-representative benchmarking results\&.  
93.TP
94\fB-p \fIPOST-file\fR\fR
95File containing data to POST\&. Remember to also set \fB-T\fR\&.  
96.TP
97\fB-P \fIproxy-auth-username\fR:\fIpassword\fR\fR
98Supply BASIC Authentication credentials to a proxy en-route\&. The username and password are separated by a single \fB:\fR and sent on the wire base64 encoded\&. The string is sent regardless of whether the proxy needs it (\fIi\&.e\&.\fR, has sent an 407 proxy authentication needed)\&.  
99.TP
100\fB-q\fR
101When processing more than 150 requests, ab outputs a progress count on \fBstderr\fR every 10% or 100 requests or so\&. The \fB-q\fR flag will suppress these messages\&.  
102.TP
103\fB-r\fR
104Don't exit on socket receive errors\&.  
105.TP
106\fB-s \fItimeout\fR\fR
107Maximum number of seconds to wait before the socket times out\&. Default is 30 seconds\&. Available in 2\&.4\&.4 and later\&.  
108.TP
109\fB-S\fR
110Do not display the median and standard deviation values, nor display the warning/error messages when the average and median are more than one or two times the standard deviation apart\&. And default to the min/avg/max values\&. (legacy support)\&.  
111.TP
112\fB-t \fItimelimit\fR\fR
113Maximum number of seconds to spend for benchmarking\&. This implies a \fB-n 50000\fR internally\&. Use this to benchmark the server within a fixed total amount of time\&. Per default there is no timelimit\&.  
114.TP
115\fB-T \fIcontent-type\fR\fR
116Content-type header to use for POST/PUT data, eg\&. \fBapplication/x-www-form-urlencoded\fR\&. Default is \fBtext/plain\fR\&.  
117.TP
118\fB-u \fIPUT-file\fR\fR
119File containing data to PUT\&. Remember to also set \fB-T\fR\&.  
120.TP
121\fB-v \fIverbosity\fR\fR
122Set verbosity level - \fB4\fR and above prints information on headers, \fB3\fR and above prints response codes (404, 200, etc\&.), \fB2\fR and above prints warnings and info\&.  
123.TP
124\fB-V\fR
125Display version number and exit\&.  
126.TP
127\fB-w\fR
128Print out results in HTML tables\&. Default table is two columns wide, with a white background\&.  
129.TP
130\fB-x \fI<table>-attributes\fR\fR
131String to use as attributes for \fB<table>\fR\&. Attributes are inserted \fB<table \fIhere\fR >\fR\&.  
132.TP
133\fB-X \fIproxy\fR[:\fIport\fR]\fR
134Use a proxy server for the requests\&.  
135.TP
136\fB-y \fI<tr>-attributes\fR\fR
137String to use as attributes for \fB<tr>\fR\&.  
138.TP
139\fB-z \fI<td>-attributes\fR\fR
140String to use as attributes for \fB<td>\fR\&.  
141.TP
142\fB-Z \fIciphersuite\fR\fR
143Specify SSL/TLS cipher suite (See openssl ciphers)

145.SH "OUTPUT"
146 
147.PP
148The following list describes the values returned by ab:
149 
150 
151.TP
152Server Software
153The value, if any, returned in the \fIserver\fR HTTP header of the first successful response\&. This includes all characters in the header from beginning to the point a character with decimal value of 32 (most notably: a space or CR/LF) is detected\&.  
154.TP
155Server Hostname
156The DNS or IP address given on the command line  
157.TP
158Server Port
159The port to which ab is connecting\&. If no port is given on the command line, this will default to 80 for http and 443 for https\&.  
160.TP
161SSL/TLS Protocol
162The protocol parameters negotiated between the client and server\&. This will only be printed if SSL is used\&.  
163.TP
164Document Path
165The request URI parsed from the command line string\&.  
166.TP
167Document Length
168This is the size in bytes of the first successfully returned document\&. If the document length changes during testing, the response is considered an error\&.  
169.TP
170Concurrency Level
171The number of concurrent clients used during the test  
172.TP
173Time taken for tests
174This is the time taken from the moment the first socket connection is created to the moment the last response is received  
175.TP
176Complete requests
177The number of successful responses received  
178.TP
179Failed requests
180The number of requests that were considered a failure\&. If the number is greater than zero, another line will be printed showing the number of requests that failed due to connecting, reading, incorrect content length, or exceptions\&.  
181.TP
182Write errors
183The number of errors that failed during write (broken pipe)\&.  
184.TP
185Non-2xx responses
186The number of responses that were not in the 200 series of response codes\&. If all responses were 200, this field is not printed\&.  
187.TP
188Keep-Alive requests
189The number of connections that resulted in Keep-Alive requests  
190.TP
191Total body sent
192If configured to send data as part of the test, this is the total number of bytes sent during the tests\&. This field is omitted if the test did not include a body to send\&.  
193.TP
194Total transferred
195The total number of bytes received from the server\&. This number is essentially the number of bytes sent over the wire\&.  
196.TP
197HTML transferred
198The total number of document bytes received from the server\&. This number excludes bytes received in HTTP headers  
199.TP
200Requests per second
201This is the number of requests per second\&. This value is the result of dividing the number of requests by the total time taken  
202.TP
203Time per request
204The average time spent per request\&. The first value is calculated with the formula \fBconcurrency * timetaken * 1000 / done\fR while the second value is calculated with the formula \fBtimetaken * 1000 / done\fR  
205.TP
206Transfer rate
207The rate of transfer as calculated by the formula \fBtotalread / 1024 / timetaken\fR

209.SH "BUGS"
210 
211.PP
212There are various statically declared buffers of fixed length\&. Combined with the lazy parsing of the command line arguments, the response headers from the server and other external inputs, this might bite you\&.
213 
214.PP
215It does not implement HTTP/1\&.x fully; only accepts some 'expected' forms of responses\&. The rather heavy use of \fBstrstr(3)\fR shows up top in profile, which might indicate a performance problem; \fIi\&.e\&.\fR, you would measure the ab performance rather than the server's\&.
