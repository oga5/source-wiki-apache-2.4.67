35    <p>Although the Apache HTTP Server provides generic error responses
36    in the event of 4xx or 5xx HTTP status codes, these responses are
37    rather stark, uninformative, and can be intimidating to site users.
38    You may wish to provide custom error responses which are either
39    friendlier, or in some language other than English, or perhaps which
40    are styled more in line with your site layout.</p>
41
42    <p>Customized error responses can be defined for any HTTP status
43    code designated as an error condition - that is, any 4xx or 5xx
44    status.</p>
45
46    <p>Additionally, a set of values are provided, so
47    that the error document can be customized further based on the
48    values of these variables, using <a href="howto/ssi.html">Server
49    Side Includes</a>. Or, you can have error conditions handled by a
50    cgi program, or other dynamic handler (PHP, mod_perl, etc) which
51    makes use of these variables.</p>

61<h2><a name="configuration" id="configuration">Configuration</a></h2>
62
63    <p>Custom error documents are configured using the <code class="directive"><a href="./mod/core.html#errordocument">ErrorDocument</a></code> directive,
64    which may be used in global,
65    virtualhost, or directory context. It may be used in .htaccess files
66    if <code class="directive"><a href="./mod/core.html#allowoverride">AllowOverride</a></code> is set to
67    FileInfo.</p>
68
69    <pre class="prettyprint lang-config">ErrorDocument 500 "Sorry, our script crashed. Oh dear"
70ErrorDocument 500 /cgi-bin/crash-recover
71ErrorDocument 500 http://error.example.com/server_error.html
72ErrorDocument 404 /errors/not_found.html
73ErrorDocument 401 /subscription/how_to_subscribe.html</pre>
74
75
76    <p>The syntax of the <code>ErrorDocument</code> directive is:</p>
77
78    <pre class="prettyprint lang-config">ErrorDocument &lt;3-digit-code&gt; &lt;action&gt;</pre>
79
80
81    <p>where the action will be treated as:</p>
82
83    <ol>
84      <li>A local URL to redirect to (if the action begins with a "/").</li>
85      <li>An external URL to redirect to (if the action is a valid URL).</li>
86      <li>Text to be displayed (if none of the above). The text must be
87          wrapped in quotes (") if it consists of more than one word.</li>
88    </ol>
89
90    <p>When redirecting to a local URL, additional environment variables
91    are set so that the response can be further customized. They are not sent to
92    external URLs.</p>

96<h2><a name="variables" id="variables">Available Variables</a></h2>
97
98      <p>Redirecting to another URL can be useful, but only if some
99      information can be passed which can then be used to explain or log
100      the error condition more clearly.</p>
101
102      <p>To achieve this, when the error redirect is sent, additional
103      environment variables will be set, which will be generated from
104      the headers provided to the original request by prepending
105      'REDIRECT_' onto the original header name. This provides the error
106      document the context of the original request.</p>
107
108      <p>For example, you might receive, in addition to more usual
109      environment variables, the following.</p>
110
111      <div class="example"><pre class="prettyprint lang-config">REDIRECT_HTTP_ACCEPT=*/*, image/gif, image/jpeg, image/png
112REDIRECT_HTTP_USER_AGENT=Mozilla/5.0 Fedora/3.5.8-1.fc12 Firefox/3.5.8
113REDIRECT_PATH=.:/bin:/usr/local/bin:/sbin
114REDIRECT_QUERY_STRING=
115REDIRECT_REMOTE_ADDR=121.345.78.123
116REDIRECT_REMOTE_HOST=client.example.com
117REDIRECT_SERVER_NAME=www.example.edu
118REDIRECT_SERVER_PORT=80
119REDIRECT_SERVER_SOFTWARE=Apache/2.2.15
120REDIRECT_URL=/cgi-bin/buggy.pl</pre>
121</div>
122
123      <p><code>REDIRECT_</code> environment variables are created from
124      the environment variables which existed prior to the
125      redirect. They are renamed with a <code>REDIRECT_</code>
126      prefix, <em>i.e.</em>, <code>HTTP_USER_AGENT</code> becomes
127      <code>REDIRECT_HTTP_USER_AGENT</code>.</p>
128
129      <p><code>REDIRECT_URL</code>, <code>REDIRECT_STATUS</code>, and
130      <code>REDIRECT_QUERY_STRING</code> are guaranteed to be set, and
131      the other headers will be set only if they existed prior to the
132      error condition.</p>
133
134      <p><strong>None</strong> of these will be
135      set if the <code class="directive"><a href="./mod/core.html#errordocument">ErrorDocument</a></code> target is an
136      <em>external</em> redirect (anything starting with a
137      scheme name like <code>http:</code>, even if it refers to the same host
138      as the server).</p>

141<h2><a name="custom" id="custom">Customizing Error Responses</a></h2>
142
143      <p>If you point your <code>ErrorDocument</code> to some variety of
144      dynamic handler such as a server-side include document, CGI
145      script, or some variety of other handler, you may wish to use the
146      available custom environment variables to customize this
147      response.</p>
148
149      <p>If the ErrorDocument specifies a local redirect to a CGI
150      script, the script should include a "<code>Status:</code>"
151      header field in its output in order to ensure the propagation
152      all the way back to the client of the error condition that
153      caused it to be invoked. For instance, a Perl ErrorDocument
154      script might include the following:</p>
155
156       <pre class="prettyprint lang-perl">...
157print  "Content-type: text/html\n";
158printf "Status: %s Condition Intercepted\n", $ENV{"REDIRECT_STATUS"};
159...</pre>
160
161
162      <p>If the script is dedicated to handling a particular error
163      condition, such as <code>404&nbsp;Not&nbsp;Found</code>, it can
164      use the specific code and error text instead.</p>
165
166      <p>Note that if the response contains <code>Location:</code>
167      header (in order to issue a client-side redirect), the script
168      <em>must</em> emit an appropriate <code>Status:</code> header
169      (such as <code>302&nbsp;Found</code>). Otherwise the
170      <code>Location:</code> header may have no effect.</p>

174<h2><a name="multi-lang" id="multi-lang">Multi Language Custom Error Documents</a></h2>
175
176    <p>Provided with your installation of the Apache HTTP Server is a
177    directory of custom error documents translated into 16 different
178    languages. There's also a configuration file in the
179    <code>conf/extra</code> configuration directory that can be included
180    to enable this feature.</p>
181
182    <p>In your server configuration file, you'll see a line such as:</p>
183
184    <pre class="prettyprint lang-config"># Multi-language error messages
185#Include conf/extra/httpd-multilang-errordoc.conf</pre>
186
187
188    <p>Uncommenting this <code>Include</code> line will enable this
189    feature, and provide language-negotiated error messages, based on
190    the language preference set in the client browser.</p>
191
192    <p>Additionally, these documents contain various of the
193    <code>REDIRECT_</code> variables, so that additional information can
194    be provided to the end-user about what happened, and what they can
195    do now.</p>
196
197    <p>These documents can be customized to whatever degree you wish to
198    provide more useful information to users about your site, and what
199    they can expect to find there.</p>
200
201    <p><code class="module"><a href="./mod/mod_include.html">mod_include</a></code> and <code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code>
202    must be enabled to use this feature.</p>
