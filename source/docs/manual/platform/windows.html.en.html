31    <p>This document explains how to install, configure and run
32    Apache HTTP Server 2.4 under Microsoft Windows.  If you have questions after
33    reviewing the documentation (and any event and error logs), you
34    should consult the peer-supported

36    list</a>.</p>

38    <p>This document assumes that you are installing a binary
39    distribution of httpd. If you want to compile Apache httpd yourself
40    (possibly to help with development or tracking down bugs),

42    Windows</a>.</p>

55<h2><a name="req" id="req">Operating System Requirements</a></h2>
56    
57
58    <p>The primary Windows platform for running Apache httpd 2.4 is Windows
59    2000 or later. Always obtain and
60    install the current service pack to avoid operating system bugs.</p>
61
62    <div class="note">Apache HTTP Server versions later than 2.2 will not run on any
63    operating system earlier than Windows 2000.</div>

66<h2><a name="down" id="down">Downloading httpd for Windows</a></h2>
67    
68
69    <p>The Apache HTTP Server Project itself does not provide binary releases of
70    software, only source code. Individual committers <em>may</em> provide
71    binary packages as a convenience, but it is not a release deliverable.</p>
72    <p>If you cannot compile the Apache HTTP Server
73    yourself, you can obtain a binary package from numerous binary distributions
74    available on the Internet.</p>
75
76    <p>Popular options for deploying Apache httpd, and, optionally, PHP
77    and MySQL, on Microsoft Windows, include:</p>
78    <ul>
79    <li><a href="http://www.apachelounge.com/download/">Apache Lounge</a></li>
80    <li><a href="http://www.wampserver.com/">WampServer</a></li>
81    </ul>

84<h2><a name="cust" id="cust">Customizing httpd for Windows</a></h2>
85    
86
87    <p>Apache httpd is configured by the files in the <code>conf</code>
88    subdirectory. These are the same files used to configure the Unix
89    version, but there are a few different directives for httpd on
90    Windows. See the <a href="../mod/directives.html">directive index</a>
91    for all the available directives.</p>
92
93    <p>The default configuration of the source distribution expects the server
94    to be installed into \Apache24. This can be customized at compilation time,
95    but it's important that ultimately the <code class="directive">ServerRoot</code> in
96    httpd.conf match the actual installation root. Binary distributions may
97    customize either the default installation root or the overall layout of
98    the initial configuration.</p>
99
100    <p>The main differences in httpd for Windows are:</p>
101    <ul>
102      <li><p>Because httpd for Windows is multithreaded, it does not
103      use a separate process for each request, as httpd can on Unix.
104      Instead there are usually only two httpd processes running: a
105      parent process, and a child which handles the requests. Within
106      the child process each request is handled by a separate thread.
107      </p>
108
109      <p>The process management directives are also different:</p>
110
111      <p><code class="directive"><a href="../mod/mpm_common.html#maxconnectionsperchild">MaxConnectionsPerChild</a></code>:
112      Like the Unix directive, this controls how many connections a single
113      child process will serve before exiting.
114      However, unlike on Unix, a replacement process is not instantly
115      available.  Use the default <code>MaxConnectionsPerChild 0</code>,
116      unless instructed to change the behavior to overcome a memory leak
117      in third party modules or in-process applications.</p>
118
119      <div class="warning"><strong>Warning: The server configuration
120      file is reread when a new child process is started. If you have
121      modified <code>httpd.conf</code>, the new child may not start or
122      you may receive unexpected results.</strong></div>
123
124      <p><code class="directive"><a href="../mod/mpm_common.html#threadsperchild">ThreadsPerChild</a></code>:
125      This directive is new. It tells the server how many threads it
126      should use. This is the maximum number of connections the server
127      can handle at once, so be sure to set this number high enough for
128      your site if you get a lot of hits. The recommended default is
129      <code>ThreadsPerChild 150</code>, but this must be adjusted to
130      reflect the greatest anticipated number of simultaneous
131      connections to accept.</p></li>
132
133      <li><p>The directives that accept filenames as arguments must use
134      Windows filenames instead of Unix ones. However, because httpd
135      may interpret backslashes as an "escape character" sequence, you
136      should consistently use forward slashes in path names, not
137      backslashes.</p></li>
138
139      <li><p>While filenames are generally case-insensitive on
140      Windows, URLs are still treated internally as case-sensitive
141      before they are mapped to the filesystem.  For example, the
142      <code class="directive"><a href="../mod/core.html#location">&lt;Location&gt;</a></code>,
143      <code class="directive"><a href="../mod/mod_alias.html#alias">Alias</a></code>, and <code class="directive"><a href="../mod/mod_proxy.html#proxypass">ProxyPass</a></code> directives all use
144      case-sensitive arguments.  For this reason, it is particularly
145      important to use the <code class="directive"><a href="../mod/core.html#directory">&lt;Directory&gt;</a></code> directive when attempting
146      to limit access to content in the filesystem, since this
147      directive applies to any content in a directory, regardless of
148      how it is accessed.  If you wish to assure that only lowercase
149      is used in URLs, you can use something like:</p>
150
151      <pre class="prettyprint lang-config">RewriteEngine On
152RewriteMap lowercase int:tolower
153RewriteCond "%{REQUEST_URI}" "[A-Z]"
154RewriteRule "(.*)" "${lowercase:$1}" [R,L]</pre>
155</li>
156
157      <li><p>When running, httpd needs write access only to the logs
158      directory and any configured cache directory tree.  Due to the
159      issue of case insensitive and short 8.3 format names, httpd must
160      validate all path names given.  This means that each directory
161      which httpd evaluates, from the drive root up to the directory
162      leaf, must have read, list and traverse directory permissions.
163      If Apache2.4 is installed at C:\Program Files, then the root
164      directory, Program Files and Apache2.4 must all be visible
165      to httpd.</p></li>
166
167      <li><p>httpd for Windows contains the ability to load modules at
168      runtime, without recompiling the server. If httpd is compiled
169      normally, it will install a number of optional modules in the
170      <code>\Apache2.4\modules</code> directory. To activate these or
171      other modules, the <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code>
172      directive must be used. For example, to activate the status
173      module, use the following (in addition to the status-activating
174      directives in <code>access.conf</code>):</p>
175
176      <pre class="prettyprint lang-config">LoadModule status_module "modules/mod_status.so"</pre>
177
178
179      <p>Information on <a href="../mod/mod_so.html#creating">creating
180      loadable modules</a> is also available.</p></li>
181
182      <li><p>httpd can also load ISAPI (Internet Server Application
183      Programming Interface) extensions such as those used by Microsoft
184      IIS and other Windows servers. <a href="../mod/mod_isapi.html">More
185      information is available</a>. Note that httpd <strong>cannot</strong>
186      load ISAPI Filters, and ISAPI Handlers with some Microsoft feature
187      extensions will not work.</p></li>
188
189      <li><p>When running CGI scripts, the method httpd uses to find
190      the interpreter for the script is configurable using the
191      <code class="directive"><a href="../mod/core.html#scriptinterpretersource">ScriptInterpreterSource</a></code>
192      directive.</p></li>
193
194      <li><p>Since it is often difficult to manage files with names
195      like <code>.htaccess</code> in Windows, you may find it useful to
196      change the name of this per-directory configuration file using
197      the <code class="directive"><a href="../mod/core.html#accessfilename">AccessFilename</a></code>
198      directive.</p></li>
199
200      <li><p>Any errors during httpd startup are logged into the
201      Windows event log when running on Windows NT. This mechanism
202      acts as a backup for those situations where httpd is not yet
203      prepared to use the <code>error.log</code> file. You can
204      review the Windows Application Event Log by using the Event Viewer,
205      e.g. Start - Settings - Control Panel - Administrative Tools
206      - Event Viewer.</p></li>
207    </ul>

211<h2><a name="winsvc" id="winsvc">Running httpd as a Service</a></h2>
212    
213
214    <p>Apache httpd comes with a utility called the Apache Service Monitor.
215    With it you can see and manage the state of all installed httpd
216    services on any machine on your network. To be able to manage an
217    httpd service with the monitor, you have to first install the
218    service (either automatically via the installation or manually).
219    </p>
220
221    <p>You can install httpd as a Windows NT service as follows from
222    the command prompt at the httpd <code>bin</code> subdirectory:</p>
223
224    <div class="example"><p><code>
225      httpd.exe -k install
226    </code></p></div>
227
228    <p>If you need to specify the name of the service you want to
229    install, use the following command. You have to do this if you
230    have several different service installations of httpd on your
231    computer. If you specify a name during the install, you have to
232    also specify it during any other -k operation.</p>
233
234    <div class="example"><p><code>
235      httpd.exe -k install -n "MyServiceName"
236    </code></p></div>
237
238    <p>If you need to have specifically named configuration files for
239    different services, you must use this:</p>
240
241    <div class="example"><p><code>
242      httpd.exe -k install -n "MyServiceName" -f "c:\files\my.conf"
243    </code></p></div>
244
245    <p>If you use the first command without any special parameters except
246    <code>-k install</code>, the service will be called <code>Apache2.4</code>
247    and the configuration will be assumed to be <code>conf\httpd.conf</code>.
248    </p>
249
250    <p>Removing an httpd service is easy. Just use:</p>
251
252    <div class="example"><p><code>
253      httpd.exe -k uninstall
254    </code></p></div>
255
256    <p>The specific httpd service to be uninstalled can be specified by using:</p>
257
258    <div class="example"><p><code>
259      httpd.exe -k uninstall -n "MyServiceName"
260    </code></p></div>
261
262    <p>Normal starting, restarting and shutting down of an httpd
263    service is usually done via the Apache Service Monitor, by using
264    commands like <code>NET START Apache2.4</code> and <code>NET STOP
265    Apache2.4</code> or via normal Windows service management. Before
266    starting httpd as a service by any means, you should test the
267    service's configuration file by using:</p>
268
269    <div class="example"><p><code>
270      httpd.exe -n "MyServiceName" -t
271    </code></p></div>
272
273    <p>You can control an httpd service by its command line switches,
274    too. To start an installed httpd service you'll use this:</p>
275
276    <div class="example"><p><code>
277      httpd.exe -k start -n "MyServiceName"
278    </code></p></div>
279
280    <p>To stop an httpd service via the command line switches, use
281    this:</p>
282
283    <div class="example"><p><code>
284      httpd.exe -k stop -n "MyServiceName"
285    </code></p></div>
286
287    <p>or</p>
288
289    <div class="example"><p><code>
290      httpd.exe -k shutdown -n "MyServiceName"
291    </code></p></div>
292
293    <p>You can also restart a running service and force it to reread
294    its configuration file by using:</p>
295
296    <div class="example"><p><code>
297      httpd.exe -k restart -n "MyServiceName"
298    </code></p></div>
299
300    <p>By default, all httpd services are registered to run as the
301    system user (the <code>LocalSystem</code> account). The
302    <code>LocalSystem</code> account has no privileges to your network
303    via any Windows-secured mechanism, including the file system, named
304    pipes, DCOM, or secure RPC. It has, however, wide privileges locally.
305    </p>
306
307    <div class="warning"><strong>Never grant any network privileges to
308    the <code>LocalSystem</code> account! If you need httpd to be able
309    to access network resources, create a separate account for httpd as
310    noted below.</strong></div>
311
312    <p>It is recommended that users create a separate account for running
313    httpd service(s). If you have to access network resources via httpd,
314    this is required.</p>
315
316    <ol>
317      <li>Create a normal domain user account, and be sure to
318      memorize its password.</li>
319
320      <li>Grant the newly-created user a privilege of <code>Log on
321      as a service</code> and <code>Act as part of the operating
322      system</code>. On Windows NT 4.0 these privileges are granted via
323      User Manager for Domains, but on Windows 2000 and XP you probably
324      want to use Group Policy for propagating these settings. You can
325      also manually set these via the Local Security Policy MMC snap-in.
326      </li>
327
328      <li>Confirm that the created account is a member of the Users
329      group.</li>
330
331      <li>Grant the account read and execute (RX) rights to all document
332      and script folders (<code>htdocs</code> and <code>cgi-bin</code>
333      for example).</li>
334
335      <li>Grant the account change (RWXD) rights to the
336      httpd <code>logs</code> directory.</li>
337
338      <li>Grant the account read and execute (RX) rights to the
339      <code>httpd.exe</code> binary executable.</li>
340    </ol>
341
342    <div class="note">It is usually a good practice to grant the user the httpd
343    service runs as read and execute (RX) access to the whole Apache2.4
344    directory, except the <code>logs</code> subdirectory, where the
345    user has to have at least change (RWXD) rights.</div>
346
347    <p>If you allow the account to log in as a user and as a service,
348    then you can log on with that account and test that the account has
349    the privileges to execute the scripts, read the web pages, and that
350    you can start httpd in a console window. If this works, and you
351    have followed the steps above, httpd should execute as a service
352    with no problems.</p>
353
354    <div class="note"><strong>Error code 2186</strong> is a good indication that
355    you need to review the "Log On As" configuration for the service,
356    since httpd cannot access a required network resource. Also, pay
357    close attention to the privileges of the user httpd is
358    configured to run as.</div>
359
360    <p>When starting httpd as a service you may encounter an error
361    message from the Windows Service Control Manager. For example,
362    if you try to start httpd by using the Services applet in the
363    Windows Control Panel, you may get the following message:</p>
364
365    <div class="example"><p><code>
366      Could not start the Apache2.4 service on \\COMPUTER <br />
367      Error 1067; The process terminated unexpectedly.
368    </code></p></div>
369
370    <p>You will get this generic error if there is any problem with
371    starting the httpd service. In order to see what is really causing
372    the problem you should follow the instructions for Running httpd
373    for Windows from the Command Prompt.</p>
374
375    <p>If you are having problems with the service, it is suggested
376    you follow the instructions below to try starting httpd.exe from
377    a console window, and work out the errors before struggling to
378    start it as a service again.</p>
379  </div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

381<h2><a name="wincons" id="wincons">Running httpd as a Console Application</a></h2>
382    
383
384    <p>Running Apache httpd as a service is usually the recommended way to
385    use it, but it is sometimes easier to work from the command line,
386    especially during initial configuration and testing.</p>
387
388    <p>To run httpd from the command line as a console application,
389    use the following command:</p>
390
391    <div class="example"><p><code>
392      httpd.exe
393    </code></p></div>
394
395    <p>httpd will execute, and will remain running until it is stopped
396    by pressing Control-C.</p>
397
398    <p>You can also run httpd via the shortcut Start Apache in Console
399    placed to <code>Start Menu --&gt; Programs --&gt; Apache HTTP Server
400    2.4.xx --&gt; Control Apache Server</code> during the installation.
401    This will open a console window and start httpd inside it. If you
402    don't have httpd installed as a service, the window will remain
403    visible until you stop httpd by pressing Control-C in the console
404    window where httpd is running in. The server will exit in a few
405    seconds. However, if you do have httpd installed as a service, the
406    shortcut starts the service. If the httpd service is running
407    already, the shortcut doesn't do anything.</p>
408
409    <p>If httpd is running as a service, you can tell it to stop by opening another console
410    window and entering:</p>
411
412    <div class="example"><p><code>
413      httpd.exe -k shutdown
414    </code></p></div>
415
416    <p>Running as a service should be preferred over running in a
417    console window because this lets httpd end any current operations
418    and clean up gracefully.</p>
419
420    <p>But if the server is running in a console window, you can
421    only stop it by pressing Control-C in the same window.</p>
422
423    <p>You can also tell httpd to restart. This forces it to reread
424    the configuration file. Any operations in progress are allowed to
425    complete without interruption. To restart httpd, either press
426    Control-Break in the console window you used for starting httpd,
427    or enter</p>
428
429    <div class="example"><p><code>
430      httpd.exe -k restart
431    </code></p></div>
432
433    <p>if the server is running as a service.</p>
434
435    <div class="note">Note for people familiar with the Unix version of httpd:
436    these commands provide a Windows equivalent to <code>kill -TERM
437    <em>pid</em></code> and <code>kill -USR1 <em>pid</em></code>. The
438    command line option used, <code>-k</code>, was chosen as a reminder
439    of the <code>kill</code> command used on Unix.</div>
440
441    <p>If the httpd console window closes immediately or unexpectedly
442    after startup, open the Command Prompt from the Start Menu --&gt;
443    Programs. Change to the folder to which you installed httpd, type
444    the command <code>httpd.exe</code>, and read the error message. Then
445    change to the logs folder, and review the <code>error.log</code>
446    file for configuration mistakes. Assuming httpd was installed into
447    <code>C:\Program Files\Apache Software Foundation\Apache2.4\</code>,
448    you can do the following:</p>
449
450    <div class="example"><p><code>
451      c: <br />
452      cd "\Program Files\Apache Software Foundation\Apache2.4\bin" <br />
453      httpd.exe
454    </code></p></div>
455
456    <p>Then wait for httpd to stop, or press Control-C. Then enter the
457    following:</p>
458
459    <div class="example"><p><code>
460      cd ..\logs <br />
461      more &lt; error.log
462    </code></p></div>
463
464    <p>When working with httpd it is important to know how it will
465    find the configuration file. You can specify a configuration file
466    on the command line in two ways:</p>
467
468    <ul>
469      <li><p><code>-f</code> specifies an absolute or relative path to
470      a particular configuration file:</p>
471
472      <div class="example"><p><code>
473        httpd.exe -f "c:\my server files\anotherconfig.conf"
474      </code></p></div>
475
476      <p>or</p>
477
478      <div class="example"><p><code>
479        httpd.exe -f files\anotherconfig.conf
480      </code></p></div></li>
481
482      <li><p><code>-n</code> specifies the installed httpd service
483      whose configuration file is to be used:</p>
484
485      <div class="example"><p><code>
486        httpd.exe -n "MyServiceName"
487      </code></p></div>
488      </li>
489    </ul>
490
491    <p>In both of these cases, the proper
492    <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> should be set in
493    the configuration file.</p>
494
495    <p>If you don't specify a configuration file with <code>-f</code>
496    or <code>-n</code>, httpd will use the file name compiled into the
497    server, such as <code>conf\httpd.conf</code>. This built-in path
498    is relative to the installation directory. You can verify the compiled
499    file name from a value labelled as <code>SERVER_CONFIG_FILE</code> when
500    invoking httpd with the <code>-V</code> switch, like this:</p>
501
502    <div class="example"><p><code>
503      httpd.exe -V
504    </code></p></div>
505
506    <p>httpd will then try to determine its <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> by trying the following, in this order:</p>
507
508    <ol>
509      <li>A <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> directive
510      via the <code>-C</code> command line switch.</li>
511
512      <li>The <code>-d</code> switch on the command line.</li>
513
514      <li>Current working directory.</li>
515
516      <li>A registry entry which was created if you did a binary
517      installation.</li>
518
519      <li>The server root compiled into the server. This is <code>
520      /apache</code> by default, you can verify it by using <code>
521      httpd.exe -V</code> and looking for a value labelled as
522      <code>HTTPD_ROOT</code>.</li>
523    </ol>
524
525    <p>If you did not do a binary install, httpd will in some
526    scenarios complain about the missing registry key. This warning can
527    be ignored if the server was otherwise able to find its
528    configuration file.</p>
529
530    <p>The value of this key is the
531    <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> directory which
532    contains the <code>conf</code> subdirectory. When httpd starts it
533    reads the <code>httpd.conf</code> file from that directory. If
534    this file contains a <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>
535    directive which contains a different directory from the one
536    obtained from the registry key above, httpd will forget the
537    registry key and use the directory from the configuration file. If
538    you copy the httpd directory or configuration files to a new
539    location it is vital that you update the
540    <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> directive in the
541    <code>httpd.conf</code> file to reflect the new location.</p>
542  </div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>
543<div class="section">
544<h2><a name="test" id="test">Testing the Installation</a></h2>
545    
546
547    <p>After starting Apache httpd (either in a console window or as a
548    service) it will be listening on port 80 (unless you changed the
549    <code class="directive"><a href="../mod/mpm_common.html#listen">Listen</a></code> directive in the
550    configuration files or installed httpd only for the current user).
551    To connect to the server and access the default page, launch a
552    browser and enter this URL:</p>
553
554    <div class="example"><p><code>
555      http://localhost/
556    </code></p></div>
557
558    <p>httpd should respond with a welcome page and you should see
559    "It Works!". If nothing happens or you get an error, look in the
560    <code>error.log</code> file in the <code>logs</code> subdirectory.
561    If your host is not connected to the net, or if you have serious
562    problems with your DNS (Domain Name Service) configuration, you
563    may have to use this URL:</p>
564
565    <div class="example"><p><code>
566      http://127.0.0.1/
567    </code></p></div>
568
569    <p>If you happen to be running httpd on an alternate port, you
570    need to explicitly put that in the URL:</p>
571
572    <div class="example"><p><code>
573      http://127.0.0.1:8080/
574    </code></p></div>
575
576    <p>Once your basic installation is working, you should configure it
577    properly by editing the files in the <code>conf</code> subdirectory.
578    Again, if you change the configuration of the Windows NT service
579    for httpd, first attempt to start it from the command line to
580    make sure that the service starts with no errors.</p>
581
582    <p>Because httpd <strong>cannot</strong> share the same port with
583    another TCP/IP application, you may need to stop, uninstall or reconfigure
584    certain other services before running httpd. These conflicting
585    services include other WWW servers, some firewall implementations,
586    and even some client applications (such as Skype) which will use port
587    80 to attempt to bypass firewall issues.</p>
588  </div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

590<h2><a name="windrivemap" id="windrivemap">Configuring Access to Network Resources</a></h2>
591    
592
593  <p>Access to files over the network can be specified using two
594  mechanisms provided by Windows:</p>
595
596  <dl>
597    <dt>Mapped drive letters</dt>
598    <dd>e.g., <code>Alias "/images/" "Z:/"</code></dd>
599
600    <dt>UNC paths</dt>
601    <dd>e.g., <code>Alias "/images/" "//imagehost/www/images/"</code></dd>
602  </dl>
603
604  <p>Mapped drive letters allow the administrator to maintain the
605  mapping to a specific machine and path outside of the Apache httpd
606  configuration.  However, these mappings are associated only with
607  interactive sessions and are not directly available to Apache httpd
608  when it is started as a service.  <strong>Use only UNC paths for
609  network resources in httpd.conf</strong> so that the resources can
610  be accessed consistently regardless of how Apache httpd is started.
611  (Arcane and error prone procedures may work around the restriction
612  on mapped drive letters, but this is not recommended.)</p>
613
614  <div class="example"><h3>Example DocumentRoot with UNC path</h3><pre class="prettyprint lang-config">  UNCList dochost
615  DocumentRoot "//dochost/www/html/"</pre>
616</div>
617
618  <div class="example"><h3>Example DocumentRoot with IP address in UNC path</h3><pre class="prettyprint lang-config">  UNCList 192.168.1.50
619  DocumentRoot "//192.168.1.50/docs/"</pre>
620</div>
621
622  <div class="example"><h3>Example Alias and corresponding Directory with UNC path</h3><pre class="prettyprint lang-config">UNCList imagehost1 imagehost2
623Alias "/images/" "//imagehost/www/images/"
624Alias "/images2/" "//imagehost2/www/images/"
625
626&lt;Directory "//imagehost/www/images/"&gt;
627#...
628&lt;/Directory&gt;
629&lt;Directory "//imagehost2/www/images/"&gt;
630#...
631&lt;/Directory&gt;</pre>
632</div>
633
634  <p>When running Apache httpd as a service, you must create a
635  separate account in order to access network resources, as described
636  above.</p>
