34    <p>Ce document explique l'installation, la configuration et le
35    lancement d'Apache 2.0 sous Novell NetWare 6.0 et les versions
36    ultérieures. Si vous trouvez une bogue, ou voulez tout simplement
37    contribuer de quelque manière que ce soit, utilisez s'il vous plait
38    notre <a href="http://httpd.apache.org/bug_report.html">page des
39    rapports de bogues.</a></p>
40
41    <p>La page des rapports de bogues et la liste de diffusion dev-httpd
42    ne doivent pas être utilisées pour poser des questions à propos de
43    la configuration ou du lancement d'Apache. Avant de soumettre un
44    rapport de bogue ou une question, consultez ce document, la <a href="https://cwiki.apache.org/confluence/display/httpd/FAQ">FAQ</a> ou tout autre sujet de la
45    documentation en rapport avec votre problème. Si vous n'avez
46    toujours pas résolu votre problème, postez votre question dans le
47    newsgroup <a href="news://developer-forums.novell.com/novell.devsup.webserver">
48    novell.devsup.webserver</a>, où de nombreux utilisateurs d'Apache
49    sont prêts à répondre à toutes les nouvelles et obscures questions à
50    propos de l'utilisation d'Apache sous Netware.</p>
51
52    <p>Dans la majeure partie de ce document, vous êtes sensé avoir
53    installé Apache à partir d'une distribution binaire. Si vous voulez
54    compiler Apache vous-même (par exemple pour aider au développement,
55    ou pour rechercher des bogues), reportez-vous à la section traitant
56    de la <a href="#comp">Compilation d'Apache pour Netware</a>
57    ci-dessous.</p>

69<h2><a name="req" id="req">Prérequis</a></h2>
70
71    
72
73    <p>Apache 2.0 nécessite NetWare 6.0 service pack 3 et supérieurs
74    pour fonctionner. Si vous utilisez un service pack antérieur à SP3,
75    vous devez installer les dernières <a href="http://developer.novell.com/ndk/libc.htm">Bibliothèques
76    Netware pour C (LibC)</a>.</p>
77
78    <p>Vous trouverez les service packs Netware <a href="http://support.novell.com/misc/patlst.htm#nw">ici</a>.</p>
79
80    <p>Apache 2.0 pour NetWare peut aussi fonctionner dans un
81    environnement NetWare 5.1, à partir du moment où le dernier service
82    pack ou la dernière version des <a href="http://developer.novell.com/ndk/libc.htm">Bibliothèques
83    Netware pour C (LibC)</a> ont été installés. <strong>ATTENTION
84    :</strong> Apache 2.0 pour NetWare n'a pas été testé dans cet
85    environnement car il n'a pas été conçu pour ce dernier.</p>

102<h2><a name="inst" id="inst">Installation d'Apache pour NetWare</a></h2>
103
104    
105
106    <p>Il n'existe pas actuellement de programme d'installation d'Apache
107    pour Netware. Si vous installez Apache 2.0 pour NetWare à partir des
108    sources, vous devrez copier les fichiers sur le serveur
109    manuellement.</p>
110
111    <p>Suivez ces instructions pour installer Apache sous Netware à
112    partir de la distribution binaire (en supposant que vous effectuez
113    l'installation dans <code>sys:/apache2</code>) :</p>
114
115    <ul>
116      <li>Décompressez le fichier binaire téléchargé à la racine du
117      volume <code>SYS:</code> (vous pouvez cependant l'installer dans
118      tout volume)</li>
119
120      <li>Editez le fichier <code>httpd.conf</code> et définissez les
121      directives <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> et
122      <code class="directive"><a href="../mod/core.html#servername">ServerName</a></code> avec les valeurs
123      correctes des chemins de fichiers qui correspondent à la
124      configuration de votre serveur.</li>
125
126      <li>Ajoutez <code>SYS:/APACHE2</code> au chemin de recherche, par
127      une commande du style : <div class="example"><p><code>SEARCH ADD
128      SYS:\APACHE2</code></p></div></li>
129
130    </ul>
131
132    <p>Suivez ces instructions pour installer Apache pour Netware
133    manuellement à partir de votre propre répertoire de sources (en
134    supposant que vous effectuez l'installation dans
135    <code>sys:/apache2</code>) :</p>
136
137    <ul>
138      <li>Créez un répertoire que vous appellerez <code>Apache2</code>
139      dans un volume Netware.</li>
140
141      <li>Copiez <code>APACHE2.NLM</code>, <code>APRLIB.NLM</code> dans
142      <code>SYS:/APACHE2</code>.</li>
143
144      <li>Créez un répertoire que vous appellerez <code>BIN</code> dans
145      <code>SYS:/APACHE2</code>.</li>
146
147      <li>Copiez <code>HTDIGEST.NLM</code>, <code>HTPASSWD.NLM</code>,
148      <code>HTDBM.NLM</code>, <code>LOGRES.NLM</code>,
149      <code>ROTLOGS.NLM</code> dans <code>SYS:/APACHE2/BIN</code>.</li>
150
151      <li>Créez un répertoire que vous appellerez <code>CONF</code> dans
152      <code>SYS:/APACHE2</code>.</li>
153
154      <li>Copiez le fichier <code>HTTPD-STD.CONF</code> dans le
155      répertoire <code>SYS:/APACHE2/CONF</code> et renommez-le en
156      <code>HTTPD.CONF</code>.</li>
157
158      <li>Copiez les fichiers <code>MIME.TYPES</code>,
159      <code>CHARSET.CONV</code> et <code>MAGIC</code> dans le répertoire
160      <code>SYS:/APACHE2/CONF</code>.</li>
161
162      <li>Copiez tous les fichiers et sous-répertoires de
163      <code>\HTTPD-2.0\DOCS\ICONS</code> dans
164      <code>SYS:/APACHE2/ICONS</code>.</li>
165
166      <li>Copiez tous les fichiers et sous-répertoires de
167      <code>\HTTPD-2.0\DOCS\MANUAL</code> dans
168      <code>SYS:/APACHE2/MANUAL</code>.</li>
169
170      <li>Copiez tous les fichiers et sous-répertoires de
171      <code>\HTTPD-2.0\DOCS\ERROR</code> dans
172      <code>SYS:/APACHE2/ERROR</code>.</li>
173
174      <li>Copiez tous les fichiers et sous-répertoires de
175      <code>\HTTPD-2.0\DOCS\DOCROOT</code> dans
176      <code>SYS:/APACHE2/HTDOCS</code>.</li>
177
178      <li>Créez le répertoire <code>SYS:/APACHE2/LOGS</code> sur le
179      serveur.</li>
180
181      <li>Créez le répertoire <code>SYS:/APACHE2/CGI-BIN</code> sur le
182      serveur.</li>
183
184      <li>Créez le répertoire <code>SYS:/APACHE2/MODULES</code> et
185      copiez tous les modules nlm dans le répertoire
186      <code>modules</code>.</li>
187
188      <li>Editez le fichier <code>HTTPD.CONF</code>, et recherchez
189      toutes les marques <code>@@Value@@</code> afin de les remplacer
190      par les valeurs appropriées.</li>
191
192      <li>Ajoutez <code>SYS:/APACHE2</code> au chemin de recherche, par
193      une commande du style : <div class="example"><p><code>SEARCH ADD
194      SYS:\APACHE2</code></p></div>.</li>
195    </ul>
196
197    <p>Outre le volume par défaut <code>SYS</code>, Apache peut être
198    installé dans tout autre volume.</p>
199
200    <p>Au cours du processus d'installation, l'ajout du mot-clé
201    "install" à la ligne de commande du makefile va provoquer la
202    construction d'une distribution complète sous forme d'un paquetage
203    dans le sous-répertoire <code>DIST</code>. Vous pouvez simplement
204    installer Apache en copiant la distribution créée précédemment à la
205    racine d'un volume Netware (voir <a href="#comp">Compilation
206    d'Apache pour NetWare</a> ci-dessous).</p>

210<h2><a name="run" id="run">Exécuter Apache pour NetWare</a></h2>
211
212    
213
214    <p>Pour démarrer Apache, tapez simplement <code>apache</code> dans
215    la console. Ceci aura pour effet de charger Apache dans l'espace
216    d'adressage du système d'exploitation. Si vous préférez charger
217    Apache dans un espace d'adressage protégé, vous pouvez spécifier cet
218    espace d'adressage à l'aide de l'instruction de chargement suivante
219    :</p>
220
221    <div class="example"><p><code>
222      load address space = apache2 apache2
223    </code></p></div>
224
225    <p>Cette instruction va charger Apache dans un espace d'adressage
226    appelé apache2. Il est possible d'exécuter plusieurs instances
227    simultanées d'Apache sous Netware, en chargeant chacune d'entre
228    elles dans son propre espace d'adressage protégé.</p>
229
230    <p>Une fois démarré, Apache écoute le port 80 (à moins que vous
231    n'ayez modifié la directive <code class="directive"><a href="../mod/mpm_common.html#listen">Listen</a></code> dans les fichiers de
232    configuration). Pour vous connecter au serveur et afficher la page
233    par défaut, lancez un navigateur et entrez le nom du serveur ou son
234    adresse IP. Vous devriez voir une page de bienvenue, et un lien vers
235    le manuel Apache. Si rien ne se produit, ou si vous obtenez un
236    message d'erreur, consultez le fichier <code>error_log</code> dans
237    le répertoire <code>logs</code>.</p>
238
239    <p>Lorsque votre installation de base fonctionne, vous devez la
240    configurer correctement en éditant les fichiers du répertoire
241    <code>conf</code>.</p>
242
243    <p>Pour arrêter une instance d'Apache s'exécutant dans l'espace
244    d'adressage du système d'exploitation, entrez simplement dans la
245    console :</p>
246
247    <div class="example"><p><code>
248      unload apache2
249    </code></p></div>
250
251    <p>ou</p>
252
253    <div class="example"><p><code>
254      apache2 shutdown
255    </code></p></div>
256
257    <p>Si Apache s'exécute dans un espace d'adressage protégé, spécifiez
258    cet espace d'adressage dans l'instruction d'arrêt :</p>
259
260    <div class="example"><p><code>
261      unload address space = apache2 apache2
262    </code></p></div>
263
264    <p>Lorsqu'on travaille avec Apache, il est important de savoir
265    comment il trouve ses fichiers de configuration. Vous pouvez
266    spécifier un fichier de configuration sur la ligne de commande de
267    deux manières :</p>
268
269    <ul>
270      <li><code>-f</code> spécifie un chemin vers un fichier de
271      configuration particulier</li>
272    </ul>
273
274    <div class="example"><p><code>
275      apache2 -f "vol:/nom-serveur/conf/fich-conf.conf"
276    </code></p></div>
277
278    <div class="example"><p><code>
279      apache -f test/test.conf
280    </code></p></div>
281
282    <p>Dans ces cas, la directive <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> doit être correctement définie
283    dans le fichier de configuration.</p>
284
285    <p>Si vous ne spécifiez pas de nom de fichier de configuration avec
286    l'option <code>-f</code>, Apache utilisera le nom de fichier codé en
287    dur dans le serveur, en général <code>conf/httpd.conf</code>.
288    L'invocation d'Apache avec l'option <code>-V</code> indiquera ce nom
289    comme valeur de l'étiquette <code>SERVER_CONFIG_FILE</code>. Apache
290    va ensuite déterminer son <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> en effectuant les tests
291    suivants, dans cet ordre </p>
292
293    <ul>
294      <li>Une directive <code>ServerRoot</code> via une option
295      <code>-C</code> switch.</li>
296
297      <li>L'option de ligne de commande <code>-d</code>.</li>
298
299      <li>Le contenu du répertoire courant.</li>
300
301      <li>La racine du répertoire d'installation codée en dur dans le
302      serveur.</li>
303    </ul>
304
305    <p>La racine du répertoire d'installation codée en dur dans le
306      serveur est en général <code>sys:/apache2</code>. L'invocation
307      d'Apache avec l'option <code>-V</code> indiquera ce chemin comme
308      valeur de l'étiquette <code>HTTPD_ROOT</code>.</p>
309
310    <p>Apache 2.0 pour Netware comporte un jeu d'options de ligne de
311    commande permettant d'afficher ou de modifier certaines
312    caractéristiques de l'instance du serveur web en cours d'exécution.
313    Ces options ne sont disponibles que lorsqu'Apache est en cours
314    d'exécution. Chacune de ces options doit être précédée du mot-clé
315    <code>APACHE2</code>.</p>
316
317    <dl>
318      <dt>RESTART</dt>
319      <dd>Demande à Apache d'arrêter tout worker thread en cours
320      d'exécution lorsqu'il devient inactif, de recharger le fichier de
321      configuration, et de redémarrer chaque worker thread en fonction
322      de la nouvelle configuration.</dd>
323
324      <dt>VERSION</dt>
325      <dd>Affiche des informations à propos de la version de l'instance
326      d'Apache en cours d'exécution.</dd>
327
328      <dt>MODULES</dt>
329      <dd>Affiche la liste des modules chargés (intégrés et
330      externes). </dd>
331
332      <dt>DIRECTIVES</dt>
333      <dd>Affiche la liste des directives disponibles.</dd>
334
335      <dt>SETTINGS</dt>
336      <dd>Active ou désactive l'affichage du statut des threads sur la
337      console. En cas d'activation, l'état de chaque thread en cours
338      d'exécution s'affiche sur l'écran de la console Apache.</dd>
339
340      <dt>SHUTDOWN</dt>
341      <dd>Arrête l'instance du serveur web Apache en cours
342      d'exécution.</dd>
343
344      <dt>HELP</dt>
345      <dd>Décrit chacune des options disponibles au cours de l'exécution
346      d'Apache.</dd>
347    </dl>
348
349    <p>Par défaut, ces options sont passées à l'instance d'apache
350    s'exécutant dans l'espace d'adressage du système d'exploitation.
351    Pour passer une option à une instance d'Apache spécifique
352    s'exécutant dans un espace d'adressage protégé, ajouter le paramètre
353    -p suivi du nom de l'espace d'adressage. Pour plus d'informations,
354    tapez "apache2 Help" sur la ligne de commande.</p>

358<h2><a name="use" id="use">Configuration d'Apache pour NetWare</a></h2>
359
360    
361
362    <p>Apache lit en général ses fichiers de configuration dans le
363    répertoire <code>conf</code>. Ces fichiers sont les mêmes que ceux
364    de la version Unix, mais quelques directives sont différentes sous
365    Netware. Voir la <a href="../mod/">Documentation Apache</a> pour
366    l'ensemble des directives disponibles.</p>
367
368    <p>Les principales différences propres à Apache pour NetWare sont
369    : </p>
370
371    <ul>
372      <li>
373        <p>Comme Apache pour Netware est une application multithread,
374	elle n'utilise pas de processus séparé pour chaque requête,
375	comme c'est le cas pour certaines implémentations sous Unix. Il
376	n'y a que des threads en cours d'exécution : un thread parent,
377	et plusieurs threads enfants ou worker qui traitent les
378	requêtes.</p>
379
380        <p>En conséquence, les directives de gestion des "processus"
381	sont différentes :</p>
382
383        <p><code class="directive"><a href="../mod/mpm_common.html#maxconnectionsperchild">MaxConnectionsPerChild</a></code> - comme sous
384	Unix, cette directive contrôle le nombre maximum de connexions
385	qu'un worker thread peut traiter avant de s'arrêter. Avec la
386	valeur par défaut <code>MaxConnectionsPerChild 0</code>,
387	le thread va pouvoir traiter un nombre illimité de requêtes.
388	Cette valeur est recommandée sous Netware, à moins que vous
389	n'ayez des raisons particulières de la modifier.</p>
390
391        <p><code class="directive"><a href="../mod/mpm_common.html#startthreads">StartThreads</a></code> -
392	Cette directive indique au serveur le nombre de threads qu'il
393	doit lancer au démarrage. Il est recommandé de conserver la
394	valeur par défaut <code>StartThreads 50</code>.</p>
395
396        <p><code class="directive"><a href="../mod/mpm_common.html#minsparethreads">MinSpareThreads</a></code> -
397	Cette directive indique au serveur le nombre de worker threads
398	additionnels qu'il doit lancer si le nombre de threads inactifs
399	tombe en dessous de cette valeur. Il est recommandé de conserver la
400	valeur par défaut <code>MinSpareThreads 10</code>.</p>
401
402        <p><code class="directive"><a href="../mod/mpm_common.html#maxsparethreads">MaxSpareThreads</a></code> -
403	Cette directive indique au serveur qu'il doit commencer à
404	arrêter des worker threads si le nombre de threads inactifs
405	passe au dessus de cette valeur. Il est recommandé de conserver
406	la valeur par défaut <code>MaxSpareThreads 100</code>.</p>
407
408        <p><code class="directive"><a href="../mod/mpm_netware.html#maxthreads">MaxThreads</a></code> -
409	Cette directive impose un nombre maximum de worker threads. Il
410	est recommandé de conserver la valeur par défaut
411	<code>ThreadsPerChild 250</code>.</p>
412
413        <p><code class="directive"><a href="../mod/mpm_common.html#threadstacksize">ThreadStackSize</a></code> -
414	Cette directive indique au serveur la taille de la pile à
415	utiliser pour un worker thread individuel. Il est recommandé de
416	conserver la valeur par défaut <code>ThreadStackSize
417	65536</code>.</p>
418      </li>
419
420      <li>
421        <p>Les directives qui acceptent des noms de fichiers comme
422	arguments ne doivent pas utiliser des noms de fichiers Unix,
423	mais des noms de fichiers Netware. Cependant, comme Apache
424	utilise des noms de style Unix en interne, on doit utiliser des
425	slashes et non des antislashes. Il est recommandé de préfixer
426	tous les chemins de fichiers racines par un nom de volume. Si ce
427	dernier est omis, Apache supposera que le volume est
428	<code>SYS:</code>, ce qui n'est pas forcément correct.</p>
429      </li>
430
431      <li>
432        <p>Apache pour Netware a la possibilité de charger des modules
433	en cours d'exécution, sans avoir à recompiler le serveur. Si
434	Apache est compilé avec les options par défaut, il va installer
435	de nombreux modules optionnels dans le répertoire
436	<code>\Apache2\modules</code>. Pour les activer, ou en activer
437	d'autres, on doit utiliser la directive <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code>. Par exemple, pour
438	activer le module status, ajoutez la ligne suivante :</p>
439
440        <div class="example"><p><code>
441          LoadModule status_module modules/status.nlm
442        </code></p></div>
443
444        <p>Des informations à propos de la  <a href="../mod/mod_so.html#creating">création de modules
445	chargeables</a> sont aussi disponibles.</p>
446      </li>
447    </ul>

479<h2><a name="comp" id="comp">Compilation d'Apache pour NetWare</a></h2>
480
481    
482
483    <p>La compilation d'Apache nécessite MetroWerks CodeWarrior 6.x ou
484    supérieur. Une fois compilé, Apache peut être installé à la racine
485    de tout volume Netware. Le répertoire d'installation par défaut est
486    <code>sys:/Apache2</code>.</p>
487
488    <p>Avant de démarrer Apache, vous devez remplir le répertoire
489    <code>conf</code>. Copiez le fichier <code>HTTPD-STD.CONF</code>
490    depuis le répertoire <code>conf</code> de la distribution et
491    renommez-le en <code>HTTPD.CONF</code>. Editez le fichier
492    <code>HTTPD.CONF</code> en recherchant les repères
493    <code>@@Value@@</code>, et remplacez ces derniers par la valeur
494    appropriée. Copiez de même les fichiers <code>conf/magic</code> et
495    <code>conf/mime.types</code>. Vous pouvez aussi construire une
496    distribution complète en ajoutant le mot-clé <code>install</code>
497    lors de l'invocation des makefiles.</p>
498
499    <h3><a name="comp-req" id="comp-req">Prérequis :</a></h3>
500
501      
502
503      <p>Les outils de développement suivants sont nécessaires pour la
504      compilation d'Apache pour Netware :</p>
505
506      <ul>
507        <li>Metrowerks CodeWarrior 6.0 ou supérieur avec le <a href="http://developer.novell.com/ndk/cwpdk.htm">NetWare PDK
508	3.0</a> ou supérieur.</li>
509
510        <li>Les <a href="http://developer.novell.com/ndk/libc.htm">Bibliothèques
511	Netware pour C (LibC)</a></li>
512
513        <li>Les <a href="http://developer.novell.com/ndk/cldap.htm">Bibliothèques
514	LDAP pour C</a></li>
515
516        <li>Le <a href="http://www.gzip.org/zlib/">Code source de la
517	bibliothèque de compression ZLIB</a></li>
518
519        <li>L'utilitaire AWK (awk, gawk ou similaire). AWK peut être
520	téléchargé à <a href="http://developer.novell.com/ndk/apache.htm">http://developer.novell.com/ndk/apache.htm</a>.
521	L'utilitaire doit se trouver dans votre chemin windows et avoir
522	pour nom <code>awk.exe</code>.</li>
523
524        <li>Pour utiliser les makefiles, vous devez disposer de GNU make
525	version 3.78.1 (GMake) disponible à <a href="http://developer.novell.com/ndk/apache.htm">http://developer.novell.com/ndk/apache.htm</a>.</li>
526      </ul>
527
528    
529
530    <h3><a name="comp-make" id="comp-make">Compiler Apache en utilisant les makefiles Netware
531      :</a></h3>
532
533      
534
535      <ul>
536        <li>Définissez la variable d'environnement
537	<code>NOVELLLIBC</code> avec le chemin des bibliothèques Netware
538	pour C SDK ; par exemple : <div class="example"><p><code>Set
539	NOVELLLIBC=c:\novell\ndk\libc</code></p></div></li>
540
541        <li>Définissez la variable d'environnement
542	<code>METROWERKS</code> avec le chemin de votre compilateur
543	Metrowerks CodeWarrior ; par exemple : <div class="example"><p><code>Set
544	METROWERKS=C:\Program Files\Metrowerks\CodeWarrior</code></p></div>. Si
545	vous l'avez installé dans le répertoire par défaut
546	<code>C:\Program Files\Metrowerks\CodeWarrior</code>, vous
547	n'avez pas besoin de définir cette variable.</li>
548
549        <li>Définissez la variable d'environnement <code>LDAPSDK</code>
550	avec le chemin des bibliothèques LDAP pour C ; par exemple :
551	<div class="example"><p><code>Set
552	LDAPSDK=c:\Novell\NDK\cldapsdk\NetWare\libc</code></p></div></li>
553
554        <li>Définissez la variable d'environnement <code>ZLIBSDK</code>
555	avec le chemin du code source de la bibliothèque Zlib ; par
556	exemple : <div class="example"><p><code>Set ZLIBSDK=D:\NOVELL\zlib</code></p></div></li>
557
558	<li>Définissez la variable d'environnement <code>PCRESDK</code>
559	avec le chemin d'installation du code source de la bibliothèque
560	PCRE ; par exemple :
561	<div class="example"><p><code>Set PCRESDK=D:\NOVELL\pcre</code></p></div>
562	</li>
563
564        <li>Définissez la variable d'environnement <code>AP_WORK</code>
565	avec le chemin du code source de <code>httpd</code>.
566	<div class="example"><p><code>Set AP_WORK=D:\httpd-2.0.x</code></p></div></li>
567
568        <li>Définissez la variable d'environnement <code>APR_WORK</code>
569	avec le chemin du code source d'<code>apr</code> ; en général
570	<code>\httpd\srclib\apr</code>, mais le projet APR peut se
571	trouver en dehors de la structure des répertoires de httpd.
572	<div class="example"><p><code>Set APR_WORK=D:\apr-1.x.x</code></p></div></li>
573
574        <li>Définissez la variable d'environnement <code>APU_WORK</code>
575	avec le chemin du code source d'<code>apr-util</code> ; en
576	général <code>\httpd\srclib\apr-util</code>, mais le projet
577	APR-UTIL peut se trouver en dehors de la structure des
578	répertoires de httpd. <div class="example"><p><code>Set
579	APU_WORK=D:\apr-util-1.x.x</code></p></div></li>
580
581        <li>Vérifiez que les chemins des utilitaires AWK et GNU make
582	(<code>gmake.exe</code>) ont bien été inclus dans la variable
583	d'environnement système <code>PATH</code>.</li>
584
585        <li>Téléchargez le code source et décompressez-le dans un
586	répertoire de votre choix sur votre station de travail.</li>
587
588        <li>Positionnez-vous dans le répertoire <code>\httpd-2.0</code>
589	et compilez les utilitaires précompilés à l'aide de la commande
590	"<code>gmake -f nwgnumakefile prebuild</code>". Cette cible va
591	créer le répertoire <code>\httpd-2.0\nwprebuild</code>, et y
592	copier tous les utilitaires nécessaires au franchissement des
593	étapes suivantes de la compilation.</li>
594
595        <li>Copiez les fichiers
596	<code>\httpd-2.0\nwprebuild\GENCHARS.nlm</code> et
597	<code>\httpd-2.0\nwprebuild\DFTABLES.nlm</code> vers le volume
598	<code>SYS:</code> d'un serveur Netware et exécutez-les à l'aide
599	des commandes suivantes :
600          <div class="example"><p><code>
601            SYS:\genchars &gt; sys:\test_char.h<br />
602            SYS:\dftables sys:\chartables.c<br />
603          </code></p></div>
604        </li>
605
606        <li>Copiez les fichiers <code>test_char.h</code> et
607	<code>chartables.c</code> vers le répertoire
608	<code>\httpd-2.0\os\netware</code> de la machine où s'effectue
609	la compilation.</li>
610
611        <li>Positionnez-vous dans le répertoire <code>\httpd-2.0</code>
612	et compilez Apache à l'aide de la commande "<code>gmake -f
613	nwgnumakefile</code>". Vous pouvez créer un répertoire pour la
614	distribution en ajoutant le paramètre install à la commande ;
615	par exemple :
616        <div class="example"><p><code>gmake -f nwgnumakefile install</code></p></div>
617        </li>
618      </ul>

622    <h3><a name="comp-add" id="comp-add">Options de make supplémentaires</a></h3>
623
624      
625
626      <ul>
627        <li><code>gmake -f nwgnumakefile</code><p>Compile les versions
628	de distribution de tous les binaires et les copie dans un
629	répertoire <code>\release</code>.</p></li>
630
631        <li><code>gmake -f nwgnumakefile DEBUG=1</code><p>Compile les versions
632	de débogage de tous les binaires et les copie dans un
633	répertoire <code>\debug</code>.</p></li>
634
635        <li><code>gmake -f nwgnumakefile install</code><p>Crée une
636	distribution complète d'Apache avec les binaires, la
637	documentation et les fichiers support dans un répertoire
638	<code>\dist\Apache2</code>.</p></li>
639
640        <li><code>gmake -f nwgnumakefile prebuild</code><p>Compile tous
641	les utilitaires précompilés et les copie dans le répertoire
642	<code>\nwprebuild</code>.</p></li>
643
644        <li><code>gmake -f nwgnumakefile installdev</code><p>Même effet
645	que l'option install, mais en plus, les répertoires
646	<code>\lib</code> et <code>\include</code> sont créés dans le
647	répertoire de destination, et les en-têtes et fichiers d'import
648	y sont copiés.</p></li>
649
650        <li><code>gmake -f nwgnumakefile clean</code><p>Supprime tous
651	les fichiers objets et les binaires de la zone de compilation
652	<code>\release.o</code>, ou <code>\debug.o</code> si
653	<code>DEBUG</code> a été défini.</p></li>
654
655        <li><code>gmake -f nwgnumakefile clobber_all</code><p>Même effet
656	que clean, mais en plus, le répertoire de la distribution est
657	supprimé s'il existe.</p></li>
658      </ul>

682    <h3><a name="comp-mod_ssl" id="comp-mod_ssl">Compilation de mod_ssl pour la plate-forme Netware</a></h3>
683
684      
685
686      <p>Pour fournir les services SSL, Apache pour Netware utilise par
687      défaut le module intégré <code class="module"><a href="../mod/mod_nw_ssl.html">mod_nw_ssl</a></code>. Ce module ne
688      fournit que les services SSL implémentés par le système
689      d'exploitation Netware lui-même pour gérer tous les chiffrements
690      pour un port donné. Cependant, on peut aussi utiliser mod_ssl de
691      la même manière que sur les autres plate-formes.</p>
692
693      <p>Afin de pouvoir compiler mod_ssl pour la plate-forme Netware,
694      les bibliothèques OpenSSL doivent être disponibles. Elles peuvent
695      être installées de la manière suivante :</p>
696
697      <ul>
698          <li>Téléchargez la dernière distribution du code source
699	  d'OpenSSL 0.9.8 depuis la page <a href="http://www.openssl.org/source/">OpenSSL Source</a> (les
700	  versions 0.9.7 doivent être patchées, et ne sont donc pas
701	  recommandées).</li>
702
703          <li>Editez le fichier <code>NetWare/set_env.bat</code>, et
704	  effectuez toutes modifications nécessaires des chemins des
705	  outils et utilitaires en fonction de votre environnement de
706	  développement.</li>
707
708          <li>Exécutez les scripts suivants depuis la racine du
709	  répertoire des sources d'OpenSSL :
710          <div class="example"><p><code>
711              Netware\set_env netware-libc<br />
712              Netware\build netware-libc
713          </code></p></div>
714          Pour des raisons de performances, vous devez activer la
715	  compilation avec le code ASM. Télécharger NASM depuis le <a href="http://nasm.sourceforge.net/">site SF</a>. Configurez
716	  ensuite OpenSSL pour utiliser le code ASM :
717          <div class="example"><p><code>
718              Netware\build netware-libc nw-nasm enable-mdc2 enable-md5
719          </code></p></div>
720          Attention : n'utilisez pas l'Assembleur CodeWarrior - il
721	  produit un code de mauvaise qualité !</li>
722
723          <li>Avant de compiler Apache, définissez la variable
724	  d'environnement <code>OSSLSDK</code> avec le chemin absolu de
725	  la racine du répertoire du code source d'openssl, et
726	  définissez WITH_MOD_SSL à 1.
727          <div class="example"><p><code>
728              Set OSSLSDK=d:\openssl-0.9.8x<br />
729              Set WITH_MOD_SSL=1
730          </code></p></div></li>
731
732      </ul>
