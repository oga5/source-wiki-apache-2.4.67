33    <p>Ce document décrit l'installation, la configuration et
34    l'exécution du serveur HTTP Apache 2.4 sous Microsoft Windows. Si vous avez des
35    questions après avoir lu la documentation, ou si vous avez rencontré
36    des évènements particuliers ou des rapports d'erreur, vous pouvez
37    consultez la <a href="http://httpd.apache.org/userslist.html">liste
38    de diffusion de la communauté des utilisateurs</a>.</p>
39
40    <p>Dans ce document, nous supposons que vous installez une
41    distribution binaire d'httpd. Si vous voulez compiler httpd
42    vous-même (par exemple pour aider au développement ou pour
43    rechercher des bogues), référez-vous au document <a href="win_compiling.html">Compilation de httpd pour Microsoft
44    Windows</a>.</p>

58<h2><a name="req" id="req">Prérequis du système d'exploitation</a></h2>
59    
60
61    <p>La plate-forme Windows de base pour l'exécution de httpd 2.4 est
62    Windows 2000 ou supérieur. Veillez à toujours vous procurer et installer le
63    dernier service pack afin d'éviter les bogues du système
64    d'exploitation.</p>
65
66    <div class="note">Les versions du serveur HTTP Apache supérieures à 2.2 ne
67    fonctionneront sous aucun système d'exploitation d'une version
68    antérieure à Windows 2000.</div>

72<h2><a name="down" id="down">Téléchargement de httpd pour Windows</a></h2>
73    
74
75    <p>Le projet du serveur HTTP Apache proprement dit ne fournit pas de
76    distribution binaire mais seulement le code source. Certains membres
77    du projet <em>peuvent</em> mettre à disposition des paquets binaires
78    à titre individuel, mais ceux-ci n'ont pas vocation à être
79    distribués publiquement.</p>
80    
81    <p>Si vous n'êtes
82    pas en mesure de compiler le serveur HTTP Apache vous-même, vous
83    pouvez vous procurer un paquet binaire auprès des nombreuses
84    distributions disponibles sur Internet.</p>
85
86    <p>Quelques solutions populaires pour déployer Apache httpd, et
87    éventuellement PHP et MySQL sous Microsoft Windows :</p>
88    <ul>
89    <li><a href="http://www.apachelounge.com/download/">Apache Lounge</a></li>
90    <li><a href="http://www.wampserver.com/">WampServer</a></li>
91    </ul>

94<h2><a name="cust" id="cust">Personnaliser httpd pour Windows</a></h2>
95    
96
97    <p>La configuration de httpd est enregistrée dans les fichiers du
98    sous-répertoire <code>conf</code>. Ce sont les même fichiers que
99    ceux utilisés pour configurer la version Unix, mais il y a quelques
100    directives spécifiques à httpd pour Windows. Voir l'<a href="../mod/directives.html">index des directives</a> pour la liste
101    des directives disponibles.</p>
102
103    <p>La configuration par défaut de la distribution source installe
104    le serveur dans \Apache24. Cet emplacement peut être modifié à
105    la compilation, mais il est important qu'en fin de compte, la directive
106    <code class="directive">ServerRoot</code> dans httpd.conf corresponde effectivement
107    à la racine de l'installation. Les distributions binaires peuvent parfois
108    modifier la racine de l'installation par défaut, ou même la structure
109    complète de la configuration initiale.</p>
110
111    <p>Les principales spécificités de httpd pour Windows sont :</p>
112    <ul>
113      <li><p>Comme httpd pour Windows est un programme multithread, il
114      ne lance pas de processus séparé pour chaque requête, comme httpd
115      peut le faire sous Unix. En fait, il n'y a en général que deux
116      processus httpd en exécution : un processus parent, et un
117      processus enfant qui traite les requêtes. Chaque requête est
118      traitée par un thread séparé au sein du processus enfant.</p>
119
120      <p>Les directives de gestion de processus diffèrent également :</p>
121
122      <p><code class="directive"><a href="../mod/mpm_common.html#maxconnectionsperchild">MaxConnectionsPerChild</a></code>
123      : comme dans la version Unix, cette directive contrôle le nombre
124      de connexions qu'un
125      processus enfant particulier va traiter avant de s'arrêter.
126      Cependant, à la différence d'Unix, un processus de remplacement
127      n'est pas instantanément disponible. Utilisez la définition par
128      défaut <code>MaxConnectionsPerChild 0</code>, sauf si vous
129      risquez de manquer de mémoire dans des modules tiers ou dans des
130      applications in-process.</p>
131
132      <div class="warning"><strong>Attention : le fichier de
133      configuration du serveur est rechargé lorsqu'un nouveau processus
134      enfant est démarré. En conséquence, si vous avez modifié
135      <code>httpd.conf</code>, le nouveau processus enfant peut ne pas
136      démarrer, ou vous pouvez obtenir des résultats
137      inattendus.</strong></div>
138
139      <p><code class="directive"><a href="../mod/mpm_common.html#threadsperchild">ThreadsPerChild</a></code> : il
140      s'agit d'une nouvelle directive. Elle indique au serveur le nombre
141      de threads qu'il doit utiliser. Elle définit le nombre maximum de
142      connexions simultanées que le serveur peut gérer ; vous devez
143      donc vous assurer que ce nombre soit suffisamment grand pour les
144      besoins de votre site. La valeur par défaut <code>ThreadsPerChild
145      150</code> est recommandée, mais doit être ajustée à la valeur
146      maximale estimée de connexions simultanées à accepter.</p></li>
147
148      <li><p>Les directives qui acceptent des noms de fichiers comme
149      arguments doivent utiliser des noms de fichiers Windows et non
150      Unix. Cependant, comme httpd peut interpréter les anti-slashes
151      comme des séquences d'échappement de caractères, vous devez
152      absolument utiliser des slashes dans les noms de chemins à la
153      place des anti-slashes.</p></li>
154
155      <li><p>Alors que les noms de fichiers sont en général insensibles
156      à la casse sous Windows, les URLs sont encore sensibles à la casse
157      en interne avant d'être mises en correspondance avec le système de
158      fichiers. Par exemple, les directives <code class="directive"><a href="../mod/core.html#location">&lt;Location&gt;</a></code>, <code class="directive"><a href="../mod/mod_alias.html#alias">Alias</a></code>, et <code class="directive"><a href="../mod/mod_proxy.html#proxypass">ProxyPass</a></code> utilisent toutes des
159      arguments sensibles à la casse. Pour cette raison, il est
160      particulièrement recommandé d'utiliser la directive <code class="directive"><a href="../mod/core.html#directory">&lt;Directory&gt;</a></code> lorsqu'on
161      désire limiter l'accès à certains contenus du système de fichiers,
162      car cette directive s'applique à tout contenu d'un répertoire,
163      sans tenir compte de la manière dont on y accède. Pour vous
164      assurer que seules des minuscules sont utilisées dans les URLs,
165      vous pouvez utiliser ceci :</p>
166
167      <pre class="prettyprint lang-config">RewriteEngine On
168RewriteMap lowercase int:tolower
169RewriteCond "%{REQUEST_URI}" "[A-Z]"
170RewriteRule "(.*)" "${lowercase:$1}" [R,L]</pre>
171</li>
172
173      <li><p>Lors de son exécution, httpd n'a besoin d'un accès en
174      écriture qu'au répertoire des journaux et à toute arborescence de
175      répertoires de cache configurée. Suite au problème d'insensibilité
176      à la casse et au format de noms courts 8.3, httpd doit valider
177      tous les noms de chemins fournis. Cela signifie que chaque
178      répertoire que httpd évalue doit avoir les droits en lecture,
179      listage et parcours, et ceci depuis la racine jusqu'aux feuilles.
180      Si Apache2.4 est installé dans C:\Program Files, le répertoire
181      racine, Program Files et Apache2.4 doivent tous être visibles pour
182      httpd</p></li>
183
184      <li><p>httpd peut charger divers modules sans qu'il soit nécessaire
185      de recompiler le serveur. Si httpd est compilé
186      normalement, il va installer de nombreux modules optionnels dans
187      le répertoire <code>\Apache2.4\modules</code>. Pour activer ces
188      modules ou d'autres modules, on doit utiliser la
189      directive <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code>. Par
190      exemple, pour activer le module status, ajoutez la ligne suivante
191      (en plus des directives d'activation de status dans
192      <code>access.conf</code>) :</p>
193
194      <pre class="prettyprint lang-config">LoadModule status_module "modules/mod_status.so"</pre>
195
196
197      <p>Des informations sont aussi à votre disposition pour <a href="../mod/mod_so.html#creating">créer des modules
198      chargeables</a></p></li>
199
200      <li><p>httpd peut aussi charger des extensions ISAPI (Internet
201      Server Application Programming Interface), comme celles qu'utilise
202      Microsoft IIS et d'autres serveurs Windows. <a href="../mod/mod_isapi.html">Voir ici pour plus
203      d'informations</a>. Notez que httpd ne peut <strong>pas</strong>
204      charger de filtres ISAPI, et que les gestionnaires ISAPI contenant
205      des extensions de fonctionnalités Microsoft ne fonctionneront
206      pas.</p></li>
207
208      <li><p>Pour les scripts CGI, la méthode qu'utilise httpd pour
209      déterminer l'interpréteur du script est configurable grâce à la
210      directive <code class="directive"><a href="../mod/core.html#scriptinterpretersource">ScriptInterpreterSource</a></code></p></li>
211
212      <li><p>Comme il est souvent difficile de gérer des fichiers avec
213      des noms du style <code>.htaccess</code> sous Windows, vous avez
214      tout intérêt à changer le nom de ce fichier de configuration par
215      répertoire à l'aide de la directive <code class="directive"><a href="../mod/core.html#accessfilename">AccessFilename</a></code>.</p></li>
216
217      <li><p>Toute erreur survenant au cours du processus de démarrage
218      de httpd est enregistrée dans le journal des évènements de
219      Windows si l'on est sous Windows NT. Ce mécanisme fonctionne comme
220      une sauvegarde pour les situations où httpd n'est pas encore prêt
221      à utiliser le fichier <code>error.log</code>. Vous pouvez
222      consulter le journal des évènements applicatifs Windows en
223      utilisant l'observateur d'évènements : Démarrage - Paramètres -
224      Panneau de configuration - Outils d'administration - Observateur
225      d'évènements.</p></li>
226    </ul>

230<h2><a name="winsvc" id="winsvc">Exécuter httpd en tant que service</a></h2>
231    
232
233    <p>httpd fournit un utilitaire nommé Apache Service Monitor
234    (Moniteur du service httpd). Grâce à lui, vous pouvez voir et gérer
235    l'état de tous les services httpd installés sur toutes les machines
236    du réseau. Pour pouvoir gérer un service httpd avec le moniteur,
237    vous devez d'abord installer le service (soit automatiquement au
238    cours de l'installation, soit manuellement).</p>
239
240    <p>Vous pouvez installer httpd en tant que service Windows NT à
241    partir de la ligne de commandes et depuis le sous-répertoire httpd
242    <code>bin</code> comme suit :</p>
243
244    <div class="example"><p><code>
245      httpd.exe -k install
246    </code></p></div>
247
248    <p>Si vous avez installé plusieurs services httpd sur votre
249    ordinateur, vous devrez spécifier le nom du service que vous voulez
250    installer en utilisant la commande suivante (notez que si vous
251    spécifiez un nom durant l'installation, vous devrez aussi le
252    spécifier pour toute opération comportant l'option -k) :</p>
253
254    <div class="example"><p><code>
255      httpd.exe -k install -n "Nom-service"
256    </code></p></div>
257
258    <p>Si un service doit utiliser un fichier de configuration
259    spécifique, utilisez ceci :</p>
260
261    <div class="example"><p><code>
262      httpd.exe -k install -n "Nom-service" -f "c:\fichiers\Nom-service.conf"
263    </code></p></div>
264
265    <p>Si vous utilisez la première commande sans paramètre particulier,
266    excepté <code>-k install</code>, le service aura pour nom
267    <code>Apache2.4</code> et le fichier de configuration sera censé
268    être <code>conf\httpd.conf</code>.</p>
269
270    <p>Supprimer un service httpd est très simple. Utilisez
271    simplement :</p>
272
273    <div class="example"><p><code>
274      httpd.exe -k uninstall
275    </code></p></div>
276
277    <p>On peut spécifier un service httpd particulier en utilisant
278    :</p>
279
280    <div class="example"><p><code>
281      httpd.exe -k uninstall -n "Nom service"
282    </code></p></div>
283
284    <p>Normalement, le démarrage, le redémarrage et l'arrêt d'un
285    service httpd s'effectuent via le Moniteur de Service httpd, ou en
286    utilisant des commandes telles que <code>NET START Apache2.4</code> et
287    <code>NET STOP Apache2.4</code>, ou encore via le gestionnaire de
288    services standard de Windows. Avant de démarrer httpd en tant que
289    service dans quelque but que ce soit, vous devez tester le fichier
290    de configuration du service en utilisant :</p>
291
292    <div class="example"><p><code>
293      httpd.exe -n "Nom-service" -t
294    </code></p></div>
295
296    <p>Vous pouvez aussi contrôler un service httpd à l'aide de ses
297    options de ligne de commande. Avec cette méthode, pour démarrer un
298    service httpd installé, vous utiliserez :</p>
299
300    <div class="example"><p><code>
301      httpd.exe -k start -n "Nom-Service"
302    </code></p></div>
303
304    <p>Pour arrêter un service httpd via les options de lignes de
305    commande, utilisez ceci :</p>
306
307    <div class="example"><p><code>
308      httpd.exe -k stop -n "Nom-Service"
309    </code></p></div>
310
311    <p>ou</p>
312
313    <div class="example"><p><code>
314      httpd.exe -k shutdown -n "Nom-Service"
315    </code></p></div>
316
317    <p>Vous pouvez aussi redémarrer un service en exécution et le forcer
318    à relire son fichier de configuration en utilisant :</p>
319
320    <div class="example"><p><code>
321      httpd.exe -k restart -n "Nom-Service"
322    </code></p></div>
323
324    <p>Par défaut, tous les services httpd sont configurés pour
325    s'exécuter sous l'utilisateur system (le compte
326    <code>LocalSystem</code>). Le compte <code>LocalSystem</code> n'a
327    pas de privilèges sur votre réseau, que ce soit via un mécanisme
328    sécurisé de Windows, y compris le système de fichiers, des tubes
329    nommés, DCOM ou des RPC sécurisés. Il a cependant des privilèges
330    élevés en local.</p>
331
332    <div class="warning"><strong>N'accordez jamais de privilèges réseau
333    au compte <code>LocalSystem</code> ! Si httpd doit pouvoir accéder
334    à des ressources réseau, créez un compte séparé pour httpd comme
335    indiqué ci-dessous.</strong></div>
336
337    <p>Il est fortement fortement conseillé aux utilisateurs de créer un
338    compte séparé pour exécuter le(s) service(s) httpd, et même
339    obligatoire si vous devez accéder à des ressources réseau via
340    httpd.</p>
341
342    <ol>
343      <li>Créez un compte d'utilisateur du domaine normal, et
344      assurez-vous de retenir son mot de passe.</li>
345
346      <li>Accordez à l'utilisateur nouvellement créé les privilèges
347      <code>Log on as a service</code> et <code>Act as part of the
348      operating system</code>. Sous Windows NT 4.0, ces privilèges sont
349      accordés via le Gestionnaire des utilisateurs du Domaine, mais
350      sous Windows 2000 et XP, vous aurez plutôt intérêt à utiliser une
351      GPO pour propager ces configurations. Vous pouvez aussi effectuer
352      ces réglages via la Politique de Sécurité Locale intégrée à la
353      MMC.</li>
354
355      <li>Vérifiez que le compte nouvellement créé est membre du groupe
356      Utilisateurs</li>
357
358      <li>Accordez à ce compte les droits Lecture et Exécution (RX) sur
359      tous les documents et répertoires de scripts (<code>htdocs</code>
360      et <code>cgi-bin</code> par exemple), et aussi sur l'exécutable
361      binaire <code>httpd.exe</code>.</li>
362
363      <li>Accordez aussi à ce compte les droits de modification sur le
364      répertoire <code>logs</code>.</li>
365
366    </ol>
367
368    <div class="note">Il est en général de bonne pratique d'accorder à l'utilisateur
369    sous lequel le service httpd s'exécute les droits en lecture et
370    exécution (RX) sur l'ensemble du répertoire Apache2.4, sauf pour le
371    sous-répertoire <code>logs</code>, sur lequel l'utilisateur doit
372    avoir au moins les droits de modification (RWXD).</div>
373
374    <p>Si vous permettez à ce compte de se connecter en tant
375    qu'utilisateur et service, vous pouvez ouvrir une session sous ce
376    compte et vérifier s'il a bien le droit d'exécuter les scripts, de
377    lire les pages web, et si vous pouvez démarrer httpd à partir d'une
378    console Windows. Si tout fonctionne, et si vous avez suivi les
379    étapes ci-dessus, httpd devrait s'exécuter en tant que service sans
380    problème.</p>
381
382    <div class="note"><strong>Le code d'erreur 2186</strong> indique probablement
383    que httpd ne peut pas accéder à une ressource réseau nécessaire, et
384    que vous devez revoir la configuration "Log On As" (Se connecter en
385    tant que ...) du service.</div>
386
387    <p>Lorsqu'httpd démarre en tant que service, il se peut que vous
388    obteniez un message d'erreur du Gestionnaire de Services Windows.
389    Par exemple, si vous essayez de démarrer httpd en utilisant
390    l'applet Services du Panneau de configuration de Windows, vous
391    pouvez obtenir le message suivant :</p>
392
393    <div class="example"><p><code>
394      Could not start the Apache2.4 service on \\COMPUTER <br />
395      Error 1067; The process terminated unexpectedly.
396    </code></p></div>
397
398    <p>Vous obtiendrez cette erreur à caractère général pour tout
399    problème survenant au cours du démarrage du service httpd. Afin de
400    déterminer exactement la cause du problème, vous devez suivre les
401    instructions permettant d'exécuter httpd pour Windows depuis la
402    ligne de commande.</p>
403
404    <p>Si vous rencontrez des problèmes avec le service, il est
405    conseillé de suivre les instructions ci-dessous afin d'essayer de
406    démarrer httpd.exe depuis une console, et d'analyser les erreurs
407    plutôt que vous démener à essayer de démarrer le service.</p>

411<h2><a name="wincons" id="wincons">Exécuter httpd depuis la console</a></h2>
412    
413
414    <p>Il est en général recommandé d'exécuter httpd en tant que
415    service, mais il est parfois plus simple d'utiliser la ligne de
416    commande, en particulier au cours de la configuration initiale et
417    les tests.</p>
418
419    <p>Pour exécuter httpd depuis la ligne de commande et en tant
420    qu'application de console, utilisez la commande suivante :</p>
421
422    <div class="example"><p><code>
423      httpd.exe
424    </code></p></div>
425
426    <p>httpd va démarrer, et continuera son exécution jusqu'à ce qu'on
427    l'arrête en tapant Ctrl-C.</p>
428
429    <p>Vous pouvez également démarrer httpd via le raccourci "Démarrer
430    httpd dans une console" placé dans <code>Démarrer --&gt;
431    Programmes --&gt; Apache HTTP Server 2.4.xx --&gt; Control Apache
432    Server</code> au cours de l'installation. Ceci va
433    ouvrir une console Windows, et y démarrer httpd.
434    Si vous n'avez pas installé httpd en tant que service, la
435    fenêtre Windows restera ouverte jusqu'à ce que vous arrêtiez httpd
436    en tapant Ctrl-C dans cette fenêtre. Le serveur va alors s'arrêter
437    au bout de quelques secondes. Cependant, si vous avez installé
438    httpd en tant que service, c'est ce dernier que le raccourci
439    ci-dessus va lancer. Si le service httpd est déjà en cours
440    d'exécution, le raccourci va rester sans effet.</p>
441
442    <p>Si httpd s'exécute en tant que service, vous pouvez l'arrêter en
443    ouvrant une autre console et en entrant :</p>
444
445    <div class="example"><p><code>
446      httpd.exe -k shutdown
447    </code></p></div>
448
449    <p>Plutôt que de lancer httpd à partir d'une console, il est
450    préférable de l'exécuter en tant que service car dans ce cas, il
451    termine proprement les opérations en cours avant de s'éteindre.</p>
452
453    <p>Si le serveur a été lancé depuis une console, vous ne pouvez
454    l'arrêter qu'en pressant la combinaison de touches Ctrl-C dans la
455    même fenêtre.</p>
456
457    <p>Vous pouvez aussi redémarrer httpd. Ceci le force à recharger
458    son fichier de configuration. Toute opération en cours peut être
459    achevée sans interruption. Pour redémarrer httpd, vous pouvez soit
460    taper Control-Break dans la fenêtre de console que vous avez
461    utilisée pour le démarrer, soit entrer :</p>
462
463    <div class="example"><p><code>
464      httpd.exe -k restart
465    </code></p></div>
466
467    <p>si le serveur s'exécute en tant que service.</p>
468
469    <div class="note">Note pour les utilisateurs familiers de la version Unix
470    de httpd : les commandes ci-dessus représentent pour Windows
471    l'équivalent des commandes <code>kill -TERM <em>pid</em></code> et
472    <code>kill -USR1 <em>pid</em></code>. L'option de ligne de commande
473    <code>-k</code> a été choisie à titre de rapprochement avec la
474    commande <code>kill</code> utilisée sous Unix.</div>
475
476    <p>Si la fenêtre de la console httpd se ferme immédiatement ou
477    inopinément après le démarrage de httpd, ouvrez une console Windows
478    depuis le menu Démarrer --&gt; Programmes. Placez-vous dans le
479    répertoire d'installation de httpd, tapez la commande
480    <code>httpd.exe</code>, et observez le message d'erreur. Allez
481    ensuite dans le répertoire des journaux, et visualisez le fichier
482    <code>error.log</code> pour détecter d'éventuelles erreurs de
483    configuration. Si httpd a été installé dans <code>C:\Program
484    Files\Apache Software Foundation\Apache2.4\</code>, vous
485    pouvez entrer ce qui suit :</p>
486
487    <div class="example"><p><code>
488      c: <br />
489      cd "\Program Files\Apache Software Foundation\Apache2.4\bin" <br />
490      httpd.exe
491    </code></p></div>
492
493    <p>Attendez ensuite que httpd s'arrête ou tapez Ctrl-C. Entrez alors
494    la commande suivante :</p>
495
496    <div class="example"><p><code>
497      cd ..\logs <br />
498      more &lt; error.log
499    </code></p></div>
500
501    <p>Lorsqu'on travaille avec httpd, il est important de comprendre
502    comment ce dernier trouve son fichier de configuration. Vous pouvez
503    spécifier un fichier de configuration à partir de la ligne de
504    commande de deux façons :</p>
505
506    <ul>
507      <li><p>L'option <code>-f</code> permet de spécifier un chemin
508      absolu ou relatif vers un fichier de configuration particulier
509      :</p>(sous Windows 9x, il est recommandé d'utiliser la ligne de
510    commande à cause du manque de fiabilité du support des services
511    fourni par ce système).
512
513      <div class="example"><p><code>
514        httpd.exe -f "c:\fichiers-de-mon-serveur\autre-config.conf"
515      </code></p></div>
516
517      <p>ou</p>
518
519      <div class="example"><p><code>
520        httpd.exe -f fichiers-de-mon-serveur\autre-config.conf
521      </code></p></div></li>
522
523      <li><p>L'option <code>-n</code> permet de spécifier le service
524      httpd installé dont le fichier de configuration doit être utilisé
525      :</p>
526
527      <div class="example"><p><code>
528        httpd.exe -n "Nom-service"
529      </code></p></div>
530      </li>
531    </ul>
532
533    <p>Dans les deux cas, la directive <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> doit être correctement définie
534    dans le fichier de configuration.</p>
535
536    <p>Si vous ne spécifiez aucun fichier de configuration à l'aide des
537    options <code>-f</code> ou <code>-n</code>, httpd utilisera le nom
538    du fichier de configuration compilé dans le serveur, en général
539    <code>conf\httpd.conf</code>. Ce chemin codé en dur est relatif au
540    répertoire d'installation. Vous pouvez vérifier ce chemin à partir
541    de la valeur de l'étiquette <code>SERVER_CONFIG_FILE</code> en
542    invoquant httpd avec l'option <code>-V</code>, comme ceci :</p>
543
544    <div class="example"><p><code>
545      httpd.exe -V
546    </code></p></div>
547
548    <p>httpd va ensuite essayer de déterminer la valeur de son
549    <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> en effectuant les
550    recherches suivantes, dans cet ordre :</p>
551
552    <ol>
553      <li>Une directive <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>
554      via l'option de ligne de commande <code>-C</code>.</li>
555
556      <li>L'option de ligne de commande <code>-d</code>.</li>
557
558      <li>Le répertoire de travail courant.</li>
559
560      <li>Une entrée de la base de registre créée dans le cas d'une
561      installation binaire.</li>
562
563      <li>La racine des documents (<code>DocumentRoot</code>) codée en dur
564      dans le serveur. Elle
565      correspond par défaut à <code>/apache</code>, et vous pouvez le
566      vérifier en tapant <code>httpd.exe -V</code> et en recherchant
567      l'étiquette <code>HTTPD_ROOT</code>.</li>
568    </ol>
569
570    <p>Si vous n'avez pas effectué d'installation binaire, dans certains
571    scénarios, httpd va signaler l'absence de cette clé de registre.
572    On peut passer outre cet avertissement si le serveur a été en mesure
573    de trouver son fichier de configuration d'une autre manière.</p>
574
575    <p>La valeur de cette clé correspond au répertoire <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> qui contient lui-même le
576    sous-répertoire <code>conf</code>. Lors de son démarrage, httpd lit
577    le fichier <code>httpd.conf</code> à partir de ce répertoire. Si ce
578    fichier contient une directive <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> qui spécifie un répertoire
579    différent de celui que contient la clé de registre ci-dessus, httpd
580    oubliera la clé de registre, et utilisera le répertoire spécifié par
581    le fichier de configuration. Si vous déplacez le répertoire httpd
582    ou ses fichiers de configuration, il est vital de mettre à jour la
583    directive <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> dans
584    <code>httpd.conf</code> afin de refléter la nouvelle
585    localisation.</p>

589<h2><a name="test" id="test">Vérification de l'installation</a></h2>
590    
591
592    <p>Une fois httpd démarré (soit à partir d'une console Windows,
593    soit en tant que service), ce dernier va se mettre à l'écoute sur
594    le port 80 (à moins que vous ayiez modifié la directive <code class="directive"><a href="../mod/mpm_common.html#listen">Listen</a></code> dans les fichiers de
595    configuration ou que vous ayiez installé httpd pour l'utilisateur
596    courant seulement). Pour vous connecter au serveur et accéder à la
597    page par défaut, lancez un navigateur et entrez cette URL :</p>
598
599    <div class="example"><p><code>
600      http://localhost/
601    </code></p></div>
602
603    <p>httpd devrait renvoyer une page de bienvenue et vous devriez
604    voir s'afficher "It Works!". Si rien ne se passe ou si vous obtenez
605    une erreur, consultez le fichier <code>error.log</code> dans le
606    sous-répertoire <code>logs</code>. Si votre serveur n'est pas
607    connecté au réseau, ou si vous avez de sérieux problèmes avec la
608    configuration de votre DNS (Domain Name Service), vous devez
609    utiliser cette URL :</p>
610
611    <div class="example"><p><code>
612      http://127.0.0.1/
613    </code></p></div>
614
615    <p>Si httpd écoute un port non standard, vous devez le préciser
616    explicitement dans l'URL :</p>
617
618    <div class="example"><p><code>
619      http://127.0.0.1:8080/
620    </code></p></div>
621
622    <p>Après que votre installation de base fonctionne, vous devez la
623    configurer correctement en éditant les fichiers du sous-répertoire
624    <code>conf</code>. Encore une fois, si vous modifiez la
625    configuration du service httpd sous Windows NT, essayez d'abord de
626    redémarrer le service depuis la ligne de commande afin de vous
627    assurer de l'absence d'erreur.</p>
628
629    <p>Comme httpd <strong>ne peut pas</strong> partager le même port
630    avec d'autres applications TCP/IP, il se peut que vous soyez amené à
631    arrêter, désinstaller ou reconfigurer certains services avant de
632    démarrer httpd. Ces services entrant en conflit avec httpd
633    comprennent les autres serveurs WWW, certaines implémentations de
634    pare-feu, et même certaines applications client (comme Skype) qui
635    utilisent le port 80 afin de contourner les pare-feu.</p>

639<h2><a name="windrivemap" id="windrivemap">Configuration de l'accès aux ressources réseau</a></h2>
640    
641
642  <p>L'accès à des fichiers par le réseau peut être spécifié via deux
643  mécanismes fournis par Windows :</p>
644
645  <dl>
646    <dt>Association de lettres de lecteur</dt>
647    <dd>Par exemple, <code>Alias "/images/" Z:/</code></dd>
648
649    <dt>chemins UNC</dt>
650    <dd>Par exemple, <code>Alias "/images/" "//imagehost/www/images/"</code></dd>
651  </dl>
652
653  <p>L'association de lettres de lecteur permet à l'administrateur de
654  maintenir une correspondance avec une certaine machine et un certain
655  chemin en dehors de la configuration d'Apache httpd. Cependant, ces
656  associations ne sont possibles que dans le cadre des sessions
657  interactives, et ne sont pas directement disponibles pour Apache httpd
658  lorsqu'il est démarré en tant que service. <strong>N'utilisez par
659  conséquent que des
660  chemins UNC pour les ressources réseau dans httpd.conf,</strong> de
661  façon à ce que les ressources soient accessibles quelle que soit la
662  manière dont Apache httpd a été démarré (des procédures exotiques et
663  probablement sujettes aux erreurs peuvent permettre de contourner la
664  restriction due aux associations de lettres de lecteur, mais leur
665  utilisation est déconseillée).</p>
666
667  <div class="example"><h3>Exemple de DocumentRoot avec chemin UNC</h3><pre class="prettyprint lang-config">  UNCList dochost
668  DocumentRoot "//dochost/www/html/"</pre>
669</div>
670
671  <div class="example"><h3>Exemple de DocumentRoot avec adresse IP dans le chemin UNC</h3><pre class="prettyprint lang-config">  UNCList 192.168.1.50
672  DocumentRoot "//192.168.1.50/docs/"</pre>
673</div>
674
675  <div class="example"><h3>Exemple d'Alias et répertoire correspondant avec
676  chemin UNC</h3><pre class="prettyprint lang-config">UNCList imagehost1 imagehost2  
677Alias "/images/" "//imagehost/www/images/"
678Alias "/images2/" "//imagehost2/www/images/"
679
680&lt;Directory "//imagehost/www/images/"&gt;
681#...
682&lt;/Directory&gt;
683&lt;Directory "//imagehost2/www/images/"&gt;
684#...
685&lt;/Directory&gt;</pre>
686</div>
687
688  <p>Lorsque Apache s'exécute en tant que service, vous devez créer un
689  compte spécifique afin de pouvoir accéder aux ressources réseau, comme
690  décrit ci-dessus.</p>
