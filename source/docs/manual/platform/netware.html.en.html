52    Apache on NetWare.</p>

70<h2><a name="req" id="req">Requirements</a></h2>
71
72    
73
74    <p>Apache 2.0 is designed to run on NetWare 6.0 service pack 3
75    and above. If you are running a service pack less
76    than SP3, you must install the latest
77    NetWare Libraries
78    for C (LibC) (formerly at <code>http://developer.novell.com/ndk/libc.htm</code>).</p>
79
80    <p>NetWare service packs are available here (formerly at <code>http://support.novell.com/misc/patlst.htm#nw</code>).</p>
81
82    <p>Apache 2.0 for NetWare can also be run in a NetWare 5.1 environment
83    as long as the latest service pack or the latest version
84    of the NetWare Libraries
85    for C (LibC) (formerly at <code>http://developer.novell.com/ndk/libc.htm</code>) has been installed .  <strong>WARNING:</strong> Apache 2.0
86    for NetWare has not been targeted for or tested in this environment.</p>

104<h2><a name="inst" id="inst">Installing Apache for NetWare</a></h2>
105
106    
107
108    <p>There is no Apache install program for NetWare currently. If you
109    are building Apache 2.0 for NetWare from source, you will need to
110    copy the files over to the server manually.</p>
111
112    <p>Follow these steps to install Apache on NetWare from the
113    binary download (assuming you will install to
114    <code>sys:/apache2</code>):</p>
115
116    <ul>
117      <li>Unzip the binary download file to the root of the <code>SYS:</code>
118      volume (may be installed to any volume)</li>
119
120      <li>Edit the <code>httpd.conf</code> file setting <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> and <code class="directive"><a href="../mod/core.html#servername">ServerName</a></code> along with any file path values
121      to reflect your correct server settings</li>
122
123      <li>Add <code>SYS:/APACHE2</code> to the search path, for example:
124        <div class="example"><p><code>SEARCH ADD SYS:\APACHE2</code></p></div>
125      </li>
126
127    </ul>
128
129    <p>Follow these steps to install Apache on NetWare manually
130    from your own build source (assuming you will install to
131    <code>sys:/apache2</code>):</p>
132
133    <ul>
134      <li>Create a directory called <code>Apache2</code> on a
135      NetWare volume</li>
136
137      <li>Copy <code>APACHE2.NLM</code>, <code>APRLIB.NLM</code>
138      to <code>SYS:/APACHE2</code></li>
139
140      <li>Create a directory under <code>SYS:/APACHE2</code>
141      called <code>BIN</code></li>
142
143      <li>Copy <code>HTDIGEST.NLM</code>, <code>HTPASSWD.NLM</code>,
144      <code>HTDBM.NLM</code>, <code>LOGRES.NLM</code>, <code>ROTLOGS.NLM</code>
145      to <code>SYS:/APACHE2/BIN</code></li>
146
147      <li>Create a directory under <code>SYS:/APACHE2</code>
148      called <code>CONF</code></li>
149
150      <li>Copy the <code>HTTPD-STD.CONF</code> file to the
151      <code>SYS:/APACHE2/CONF</code> directory and rename to
152      <code>HTTPD.CONF</code></li>
153
154      <li>Copy the <code>MIME.TYPES</code>, <code>CHARSET.CONV</code> and
155      <code>MAGIC</code> files to <code>SYS:/APACHE2/CONF</code> directory</li>
156
157      <li>Copy all files and subdirectories in <code>\HTTPD-2.0\DOCS\ICONS</code>
158      to <code>SYS:/APACHE2/ICONS</code></li>
159
160      <li>Copy all files and subdirectories in <code>\HTTPD-2.0\DOCS\MANUAL</code>
161      to <code>SYS:/APACHE2/MANUAL</code></li>
162
163      <li>Copy all files and subdirectories in <code>\HTTPD-2.0\DOCS\ERROR</code>
164      to <code>SYS:/APACHE2/ERROR</code></li>
165
166      <li>Copy all files and subdirectories in <code>\HTTPD-2.0\DOCS\DOCROOT</code>
167      to <code>SYS:/APACHE2/HTDOCS</code></li>
168
169      <li>Create the directory <code>SYS:/APACHE2/LOGS</code>
170      on the server</li>
171
172      <li>Create the directory <code>SYS:/APACHE2/CGI-BIN</code>
173      on the server</li>
174
175      <li>Create the directory <code>SYS:/APACHE2/MODULES</code>
176      and copy all nlm modules into the <code>modules</code> directory</li>
177
178      <li>Edit the <code>HTTPD.CONF</code> file searching for all
179      <code>@@Value@@</code> markers and replacing them with the
180      appropriate setting</li>
181
182      <li>Add <code>SYS:/APACHE2</code> to the search path, for example:
183        <div class="example"><p><code>SEARCH ADD SYS:\APACHE2</code></p></div>
184      </li>
185    </ul>
186
187    <p>Apache may be installed to other volumes besides the default <code>SYS</code> volume.</p>
188
189    <p>During the build process, adding the keyword "install" to the makefile command line
190    will automatically produce a complete distribution package under the subdirectory
191    <code>DIST</code>. Install Apache by simply copying the distribution that was produced
192    by the makfiles to the root of a NetWare volume (see: <a href="#comp">Compiling Apache for
193    NetWare</a> below).</p>

197<h2><a name="run" id="run">Running Apache for NetWare</a></h2>
198
199    
200
201    <p>To start Apache just type <code>apache</code> at the
202    console. This will load apache in the OS address space. If you
203    prefer to load Apache in a protected address space you may
204    specify the address space with the load statement as follows:</p>
205
206    <div class="example"><p><code>
207      load address space = apache2 apache2
208    </code></p></div>
209
210    <p>This will load Apache into an address space called apache2.
211    Running multiple instances of Apache concurrently on NetWare is
212    possible by loading each instance into its own protected
213    address space.</p>
214
215    <p>After starting Apache, it will be listening to port 80
216    (unless you changed the <code class="directive"><a href="../mod/mpm_common.html#listen">Listen</a></code>
217    directive in the configuration files).
218    To connect to the server and access the default page,
219    launch a browser and enter the server's name or address. This
220    should respond with a welcome page, and a link to the Apache
221    manual. If nothing happens or you get an error, look in the
222    <code>error_log</code> file in the <code>logs</code>
223    directory.</p>
224
225    <p>Once your basic installation is working, you should
226    configure it properly by editing the files in the
227    <code>conf</code> directory.</p>
228
229    <p>To unload Apache running in the OS address space just type
230    the following at the console:</p>
231
232    <div class="example"><p><code>
233      unload apache2
234    </code></p></div>
235
236    <p>or</p>
237
238    <div class="example"><p><code>
239      apache2 shutdown
240    </code></p></div>
241
242    <p>If apache is running in a protected address space specify the
243    address space in the unload statement:</p>
244
245    <div class="example"><p><code>
246      unload address space = apache2 apache2
247    </code></p></div>
248
249    <p>When working with Apache it is important to know how it will
250    find the configuration files. You can specify a configuration
251    file on the command line in two ways:</p>
252
253    <ul>
254      <li><code>-f</code> specifies a path to a particular
255      configuration file</li>
256    </ul>
257
258    <div class="example"><p><code>
259      apache2 -f "vol:/my server/conf/my.conf"
260    </code></p></div>
261
262    <div class="example"><p><code>
263      apache -f test/test.conf
264    </code></p></div>
265
266    <p>In these cases, the proper <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>
267    should be set in the configuration file.</p>
268
269    <p>If you don't specify a configuration file name with <code>-f</code>,
270    Apache will use the file name compiled into the server, usually
271    <code>conf/httpd.conf</code>. Invoking Apache with the <code>-V</code>
272    switch will display this value labeled as <code>SERVER_CONFIG_FILE</code>.
273    Apache will then determine its <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>
274    by trying the following, in this order:</p>
275
276    <ul>
277      <li>A <code>ServerRoot</code> directive via a
278      <code>-C</code> switch.</li>
279
280      <li>The <code>-d</code> switch on the command line.</li>
281
282      <li>Current working directory</li>
283
284      <li>The server root compiled into the server.</li>
285    </ul>
286
287    <p>The server root compiled into the server is usually <code>sys:/apache2</code>.
288    invoking apache with the <code>-V</code> switch will display this value labeled as
289    <code>HTTPD_ROOT</code>.</p>
290
291    <p>Apache 2.0 for NetWare includes a set of command line directives that can
292    be used to modify or display information about the running instance of the
293    web server. These directives are only available while Apache is running. Each
294    of these directives must be preceded by the keyword <code>APACHE2</code>.</p>
295
296    <dl>
297      <dt>RESTART</dt>
298      <dd>Instructs Apache to terminate all running worker
299      threads as they become idle, reread the configuration file and restart each
300      worker thread based on the new configuration.</dd>
301
302      <dt>VERSION</dt>
303      <dd>Displays version information about the currently
304      running instance of Apache.</dd>
305
306      <dt>MODULES</dt>
307      <dd>Displays a list of loaded modules both built-in
308      and external.</dd>
309
310      <dt>DIRECTIVES</dt>
311      <dd>Displays a list of all available directives.</dd>
312
313      <dt>SETTINGS</dt>
314      <dd>Enables or disables the thread status display
315      on the console. When enabled, the state of each running threads is displayed
316      on the Apache console screen.</dd>
317
318      <dt>SHUTDOWN</dt>
319      <dd>Terminates the running instance of the Apache
320      web server.</dd>
321
322      <dt>HELP</dt>
323      <dd>Describes each of the runtime directives.</dd>
324    </dl>
325
326    <p>By default these directives are issued against the instance of Apache running
327    in the OS address space. To issue a directive against a specific instance running
328    in a protected address space, include the -p parameter along with the name of the
329    address space. For more information type "apache2 Help" on the command line.</p>

333<h2><a name="use" id="use">Configuring Apache for NetWare</a></h2>
334
335    
336
337    <p>Apache is configured by reading configuration files usually stored
338    in the <code>conf</code> directory. These are the same as files used
339    to configure the Unix version, but there are a few different directives for
340    Apache on NetWare. See the <a href="../mod/">Apache module
341    documentation</a> for all the available directives.</p>
342
343    <p>The main differences in Apache for NetWare are:</p>
344
345    <ul>
346      <li>
347        <p>Because Apache for NetWare is multithreaded, it does not
348        use a separate process for each request, as Apache does on some Unix
349        implementations. Instead there are only threads running: a parent
350        thread, and multiple child or worker threads which handle the requests.</p>
351
352        <p>Therefore the "process"-management directives are different:</p>
353
354        <p><code class="directive"><a href="../mod/mpm_common.html#maxconnectionsperchild">MaxConnectionsPerChild</a></code> -
355        Like the Unix directive, this controls how many connections
356        a worker thread will serve before exiting. The recommended default,
357        <code>MaxConnectionsPerChild 0</code>, causes the thread to continue servicing
358        request indefinitely. It is recommended on NetWare, unless there is some
359        specific reason, that this directive always remain set to <code>0</code>.</p>
360
361        <p><code class="directive"><a href="../mod/mpm_common.html#startthreads">StartThreads</a></code> -
362        This directive tells the server how many threads it should start initially.
363        The recommended default is <code>StartThreads 50</code>.</p>
364
365        <p><code class="directive"><a href="../mod/mpm_common.html#minsparethreads">MinSpareThreads</a></code> -
366        This directive instructs the server to spawn additional worker threads
367        if the number of idle threads ever falls below this value. The recommended
368        default is <code>MinSpareThreads 10</code>.</p>
369
370        <p><code class="directive"><a href="../mod/mpm_common.html#maxsparethreads">MaxSpareThreads</a></code> -
371        This directive instructs the server to begin terminating worker threads
372        if the number of idle threads ever exceeds this value. The recommended
373        default is <code>MaxSpareThreads 100</code>.</p>
374
375        <p><code class="directive"><a href="../mod/mpm_netware.html#maxthreads">MaxThreads</a></code> -
376        This directive limits the total number of work threads to a maximum
377        value. The recommended default is <code>ThreadsPerChild 250</code>.</p>
378
379        <p><code class="directive"><a href="../mod/mpm_common.html#threadstacksize">ThreadStackSize</a></code> -
380        This directive tells the server what size of stack to use
381        for the individual worker thread. The recommended default
382        is <code>ThreadStackSize 65536</code>.</p>
383      </li>
384
385      <li>
386        <p>The directives that accept filenames as arguments must use
387        NetWare filenames instead of Unix names. However, because Apache
388        uses Unix-style names internally, forward slashes must be used
389        rather than backslashes. It is recommended that all rooted file paths
390        begin with a volume name. If omitted, Apache will assume the
391        <code>SYS:</code> volume which may not be correct.</p>
392      </li>
393
394      <li>
395        <p>Apache for NetWare has the ability to load modules at
396        runtime, without recompiling the server. If Apache is
397        compiled normally, it will install a number of optional
398        modules in the <code>\Apache2\modules</code> directory.
399        To activate these, or other modules, the <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code> directive
400        must be used. For example, to active the status module, use
401        the following:</p>
402
403        <div class="example"><p><code>
404          LoadModule status_module modules/status.nlm
405        </code></p></div>
406
407        <p>Information on <a href="../mod/mod_so.html#creating">creating loadable
408        modules</a> is also available.</p>
409      </li>
410    </ul>
411
412    <h3><a name="use-add" id="use-add">Additional NetWare specific directives:</a></h3>
413
414      
415
416      <ul>
417        <li><code class="directive"><a href="../mod/core.html#cgimapextension">CGIMapExtension</a></code> -
418        This directive maps a CGI file extension to a script interpreter.</li>
419      </ul>
420      <ul>
421        <li><code class="directive"><a href="../mod/mod_nw_ssl.html#securelisten">SecureListen</a></code> -
422        Enables SSL encryption for a specified port.</li>
423      </ul>
424      <ul>
425        <li><code class="directive"><a href="../mod/mod_nw_ssl.html#nwssltrustedcerts">NWSSLTrustedCerts</a></code> -
426        Adds trusted certificates that are used to create secure connections to proxied servers.</li>
427      </ul>
428      <ul>
429        <li><code class="directive"><a href="../mod/mod_nw_ssl.html#nwsslupgradeable">NWSSLUpgradeable</a></code> -
430        Allow a connection created on the specified address/port to be upgraded to an SSL connection.</li>
431      </ul>

437<h2><a name="comp" id="comp">Compiling Apache for NetWare</a></h2>
438
439    
440
441    <p>Compiling Apache requires MetroWerks CodeWarrior 6.x or higher. Once
442    Apache has been built, it can be installed to the root of any NetWare
443    volume. The default is the <code>sys:/Apache2</code> directory.</p>
444
445    <p>Before running the server you must fill out the <code>conf</code>
446    directory. Copy the file <code>HTTPD-STD.CONF</code> from the distribution
447    <code>conf</code> directory and rename it to <code>HTTPD.CONF</code>.
448    Edit the <code>HTTPD.CONF</code> file searching for all <code>@@Value@@</code>
449    markers and replacing them with the appropriate setting. Copy over
450    the <code>conf/magic</code> and <code>conf/mime.types</code> files as well.
451    Alternatively, a complete distribution can be built by including the keyword
452    <code>install</code> when invoking the makefiles.</p>
453
454    <h3><a name="comp-req" id="comp-req">Requirements:</a></h3>
455
456      
457
458      <p>The following development tools are required to build
459      Apache 2.0 for NetWare:</p>
460
461      <ul>
462        <li>Metrowerks CodeWarrior 6.0 or higher with the
463        NetWare PDK 3.0 (formerly at <code>http://developer.novell.com/ndk/cwpdk.htm</code>)
464        or higher.</li>
465
466        <li>NetWare Libraries
467        for C (LibC) (formerly at <code>http://developer.novell.com/ndk/libc.htm</code>)</li>
468
469        <li>LDAP Libraries
470        for C (formerly at <code>http://developer.novell.com/ndk/cldap.htm</code>)</li>
471
472        <li><a href="http://www.gzip.org/zlib/">ZLIB Compression Library source code</a></li>
473
474        <li>AWK utility (awk, gawk or similar). AWK can be downloaded from
475        <code>http://developer.novell.com/ndk/apache.htm</code>.
476        The utility must be found in your windows path and must be named <code>awk.exe</code>.</li>
477
478        <li>To build using the makefiles, you will need GNU make version 3.78.1 (GMake) available at
479        <code>http://developer.novell.com/ndk/apache.htm</code>.</li>
480      </ul>
481
482    
483
484    <h3><a name="comp-make" id="comp-make">Building Apache using the NetWare makefiles:</a></h3>
485
486      
487
488      <ul>
489        <li>Set the environment variable <code>NOVELLLIBC</code> to the
490        location of the NetWare Libraries for C SDK, for example:
491        <div class="example"><p><code>Set NOVELLLIBC=c:\novell\ndk\libc</code></p></div>
492        </li>
493
494        <li>Set the environment variable <code>METROWERKS</code> to the
495        location where you installed the Metrowerks CodeWarrior compiler,
496        for example:
497        <div class="example"><p><code>Set METROWERKS=C:\Program Files\Metrowerks\CodeWarrior</code></p></div>
498        If you installed to the default location <code>C:\Program
499        Files\Metrowerks\CodeWarrior</code>, you don't need to set this.</li>
500
501        <li>Set the environment variable <code>LDAPSDK</code> to the
502        location where you installed the LDAP Libraries for C, for example:
503        <div class="example"><p><code>Set LDAPSDK=c:\Novell\NDK\cldapsdk\NetWare\libc</code></p></div>
504        </li>
505
506        <li>Set the environment variable <code>ZLIBSDK</code> to the
507        location where you installed the source code for the ZLib Library,
508        for example:
509        <div class="example"><p><code>Set ZLIBSDK=D:\NOVELL\zlib</code></p></div>
510        </li>
511
512        <li>Set the environment variable <code>PCRESDK</code> to the location
513        where you installed the source code for the PCRE Library, for example:
514        <div class="example"><p><code>Set PCRESDK=D:\NOVELL\pcre</code></p></div>
515        </li>
516
517        <li>Set the environment variable <code>AP_WORK</code> to the full path of
518        the <code>httpd</code> source code directory.
519        <div class="example"><p><code>Set AP_WORK=D:\httpd-2.0.x</code></p></div>
520        </li>
521
522        <li>Set the environment variable <code>APR_WORK</code> to the full path of
523        the <code>apr</code> source code directory. Typically <code>\httpd\srclib\apr</code>
524        but the APR project can be outside of the httpd directory structure.
525        <div class="example"><p><code>Set APR_WORK=D:\apr-1.x.x</code></p></div>
526        </li>
527
528        <li>Set the environment variable <code>APU_WORK</code> to the full path of
529        the <code>apr-util</code> source code directory. Typically <code>\httpd\srclib\apr-util</code>
530        but the APR-UTIL project can be outside of the httpd directory structure.
531        <div class="example"><p><code>Set APU_WORK=D:\apr-util-1.x.x</code></p></div>
532        </li>
533
534        <li>Make sure that the path to the AWK utility and the GNU make utility
535        (<code>gmake.exe</code>) have been included in the system's
536        <code>PATH</code> environment variable.</li>
537
538        <li>Download the source code and unzip to an appropriate directory on
539        your workstation.</li>
540
541        <li>Change directory to <code>\httpd-2.0</code> and build the prebuild utilities
542        by running "<code>gmake -f nwgnumakefile prebuild</code>". This target will create
543        the directory <code>\httpd-2.0\nwprebuild</code> and copy each of the utilities
544        to this location that are necessary to complete the following build steps.
545        </li>
546
547        <li>Copy the files <code>\httpd-2.0\nwprebuild\GENCHARS.nlm</code> and
548        <code>\httpd-2.0\nwprebuild\DFTABLES.nlm</code> to the <code>SYS:</code> volume of a
549        NetWare server and run them using the following commands:
550          <div class="example"><p><code>
551            SYS:\genchars &gt; sys:\test_char.h<br />
552            SYS:\dftables sys:\chartables.c<br />
553          </code></p></div>
554        </li>
555
556        <li>Copy the files <code>test_char.h</code> and <code>chartables.c</code>
557        to the directory <code>\httpd-2.0\os\netware</code> on the build machine.</li>
558
559        <li>Change directory to <code>\httpd-2.0</code> and build Apache by running
560        "<code>gmake -f nwgnumakefile</code>". You can create a distribution directory by
561        adding an install parameter to the command, for example:
562        <div class="example"><p><code>gmake -f nwgnumakefile install</code></p></div>
563        </li>
564      </ul>

568    <h3><a name="comp-add" id="comp-add">Additional make options</a></h3>
569
570      
571
572      <ul>
573        <li><code>gmake -f nwgnumakefile</code><p>Builds release versions of all of the
574        binaries and copies them to a <code>\release</code> destination directory.</p></li>
575
576        <li><code>gmake -f nwgnumakefile DEBUG=1</code><p>Builds debug versions of all of the
577        binaries and copies them to a <code>\debug</code> destination directory.</p></li>
578
579        <li><code>gmake -f nwgnumakefile install</code><p>Creates a complete Apache
580        distribution with binaries, docs and additional support files in a
581        <code>\dist\Apache2</code> directory.</p></li>
582
583        <li><code>gmake -f nwgnumakefile prebuild</code><p>Builds all of the prebuild utilities
584        and copies them to the <code>\nwprebuild</code> directory.</p></li>
585
586        <li><code>gmake -f nwgnumakefile installdev</code><p>Same as install but also creates a
587        <code>\lib</code> and <code>\include</code> directory in the destination directory
588        and copies headers and import files.</p></li>
589
590        <li><code>gmake -f nwgnumakefile clean</code><p>Cleans all object files and binaries
591        from the <code>\release.o</code> or <code>\debug.o</code> build areas depending on whether
592        <code>DEBUG</code> has been defined.</p></li>
593
594        <li><code>gmake -f nwgnumakefile clobber_all</code><p>Same as clean and also deletes
595        the distribution directory if it exists.</p></li>
596      </ul>

619    <h3><a name="comp-mod_ssl" id="comp-mod_ssl">Building mod_ssl for the NetWare platform</a></h3>
620
621      
622
623      <p>By default Apache for NetWare uses the built-in module
624      <code class="module"><a href="../mod/mod_nw_ssl.html">mod_nw_ssl</a></code> to provide SSL services.  This module
625      simply enables the native SSL services implemented in NetWare OS
626      to handle all encryption for a given port.  Alternatively, mod_ssl
627      can also be used in the same manner as on other platforms.</p>
628
629      <p>Before mod_ssl can be built for the NetWare platform, the OpenSSL
630      libraries must be provided.  This can be done through the following
631      steps:</p>
632
633      <ul>
634          <li>Download the recent OpenSSL 0.9.8 release source code from the
635          <a href="http://www.openssl.org/source/">OpenSSL Source</a>
636          page (older 0.9.7 versions need to be patched and are therefore not
637          recommended).</li>
638
639          <li>Edit the file <code>NetWare/set_env.bat</code> and modify any
640          tools and utilities paths so that they correspond to your build
641          environment.</li>
642
643          <li>From the root of the OpenSSL source directory, run the following
644          scripts:
645          <div class="example"><p><code>
646              Netware\set_env netware-libc<br />
647              Netware\build netware-libc
648          </code></p></div>
649          For performance reasons you should enable to build with ASM code.
650          Download NASM from the <a href="http://nasm.sourceforge.net/">SF site</a>.
651          Then configure OpenSSL to use ASM code:
652          <div class="example"><p><code>
653              Netware\build netware-libc nw-nasm enable-mdc2 enable-md5
654          </code></p></div>
655          Warning: don't use the CodeWarrior Assembler - it produces broken code!
656          </li>
657
658          <li>Before building Apache, set the environment variable
659          <code>OSSLSDK</code> to the full path to the root of the openssl
660          source code directory, and set WITH_MOD_SSL to 1.
661          <div class="example"><p><code>
662              Set OSSLSDK=d:\openssl-0.9.8x<br />
663              Set WITH_MOD_SSL=1
664          </code></p></div></li>
665
666      </ul>
