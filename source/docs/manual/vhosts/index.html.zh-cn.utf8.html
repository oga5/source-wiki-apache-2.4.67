37    <p>术语<cite>虚拟主机</cite>指的是在单一机器上运行多个网站
38    (例如 <code>company1.example.com</code> 和
39    <code>company2.example.com</code>) 。
40    虚拟主机可以“<a href="ip-based.html">基于 IP</a>”，即每个 IP 一个站点；
41    或者“<a href="name-based.html">基于名称</a>”，
42    即每个 IP 多个站点。这些站点运行在同一物理服务器上的事实不会明显的透漏给最终用户。</p>
43
44    <p>Apache 是第一个支持基于 IP 的虚拟主机的服务器。
45    Apache 版本 1.1 和更新的版本同时支持基于 IP 和基于名称的虚拟主机。
46    基于名称的虚拟主机有时候称为<em>基于主机</em>或<em>非 IP</em> 的虚拟主机.</p>
47
48    <p>以下解释是在 Apache 中支持虚拟主机的所有详细信息的文档页面列表。</p>

78    <p>如果你要调试虚拟主机配置，你会发现 Apache 的命令行参数 <code>-S</code>
79    非常有用。即输入以下命令:</p>
80
81    <div class="example"><p><code>
82    /usr/local/apache2/bin/httpd -S
83    </code></p></div>
84
85    <p>这个命令将会显示 Apache 是如何解析配置文件的。仔细检查 IP
86    地址与服务器名称可能会帮助你发现配置错误
87    (参见 <code class="program"><a href="../programs/httpd.html">httpd</a></code> 程序文档，以便了解其它命令行选项)。</p>
