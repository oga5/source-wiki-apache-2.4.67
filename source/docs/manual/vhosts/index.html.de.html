38    <p>Der Begriff <cite>virtueller Host</cite> <span class="transnote">(<em>Anm.d.&#220;.:</em> engl. 'virtual
39    host')</span> bezieht sich auf die Praxis, mehr als ein Webangebot
40    (z.B. <code>www.company1.com</code> und <code>www.company2.com</code>)
41    auf einer einzigen Maschine zu betreiben. Virtuelle Hosts k&#246;nnen
42    "<a href="ip-based.html">IP-basiert</a>" sein, was bedeutet, dass jedes
43    Webangebot eine andere IP besitzt, oder  "<a href="name-based.html">Namens-basiert</a>", was bedeutet, dass
44    unter jeder IP-Adresse mehrere Namen laufen. Die Tatsache, dass sie
45    auf dem gleichen physischen Server laufen, ist f&#252;r den Endbenutzer
46    nicht offensichtlich.</p>
47
48    <p>Der Apache war einer der ersten Server, der IP-basierte
49    virtuelle Hosts von Haus aus direkt unterst&#252;tzt hat. Seit Version 1.1
50    unterst&#252;tzt der Apache sowohl IP-basierte als auch namensbasierte
51    virtuelle Hosts (vhosts). Letzteres wird zuweilen auch
52    <em>Host-basiert</em> oder <em>non-IP-Virtual-Host</em> genannt.</p>
53
54    <p>Nachfolgend finden Sie eine Liste von Dokumenten, die alle Details
55    der Unterst&#252;tzung von virtuellen Hosts ab Apache Version 1.3
56    beschreiben.</p>

94    <p>Bei der Suche von Fehlern in Ihrer Virtual-Host-Konfiguration ist
95    die Apache-Befehlszeilenoption <code>-S</code> m&#246;glicherweise
96    hilfreich. Geben Sie dazu den folgenden Befehl ein:</p>
97
98    <div class="example"><p><code>
99    /usr/local/apache2/bin/httpd -S
100    </code></p></div>
101
102    <p>Diese Anweisung gibt eine Beschreibung aus, wie der Apache die
103    Konfigurationsdatei analysiert hat. Eine sorgf&#228;ltige
104    &#220;berpr&#252;fung der IP-Adressen und Servernamen kann helfen,
105    Konfigurationsfehler aufzudecken. (Lesen Sie die Dokumentation zum
106    <code class="program"><a href="../programs/httpd.html">httpd</a></code>-Programm f&#252;r weitere
107    Befehlszeilenoptionen.)</p>
