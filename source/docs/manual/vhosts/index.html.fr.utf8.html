36    <p>Le principe des <cite>Serveurs Virtuels</cite> consiste à 
37    faire fonctionner un ou plusieurs serveurs Web (comme 
38    <code>www.company1.example.com</code> et <code>www.company2.example.com</code>) 
39    sur une même machine. Les serveurs virtuels peuvent être soit 
40    "<a href="ip-based.html">par-IP</a>" où une adresse IP est 
41    attribuée pour chaque serveur Web, soit "<a href="name-based.html">par-nom</a>" où plusieurs noms de domaine se côtoient sur 
42    des mêmes adresses IP. L'utilisateur final ne perçoit pas 
43    qu'en fait il s'agit d'un même serveur physique.</p>
44
45    <p>Apache a été le précurseur des serveurs proposant cette 
46    méthode de serveurs virtuels basés sur les adresses IP. Ses 
47    versions 1.1 et suivantes proposent les deux 
48    méthodes de serveurs virtuels : par-IP et par-nom. Cette 
49    deuxième méthode est parfois également appelée <em>host-based</em> 
50    ou <em>serveur virtuel non-IP</em>.</p>
51
52    <p>Vous trouverez ci-dessous une liste documentaire qui vous 
53    expliquera en détails le fonctionnement du support des serveurs
54    virtuels par le serveur HTTP Apache.</p>

89
90    <p>Pour vérifier et analyser la configuration de vos serveurs 
91    virtuels, vous pouvez utiliser l'argument <code>-S</code> sur 
92    la ligne de commande.</p>
93
94    <div class="example"><h3>Exemple Unix</h3><p><code>
95    
96    apachectl -S
97    </code></p></div>
98
99    <div class="example"><h3>Exemple Windows</h3><p><code>
100    
101    httpd.exe -S
102    </code></p></div>
103
104    <p>Cette commande affichera dans le détail comment Apache a 
105    traité son fichier de configuration. Les erreurs de configuration 
106    peuvent être corrigées par l'examen attentif des adresses IP et 
107    des noms de serveurs. (Consultez la documentation du programme 
108    <code class="program"><a href="../programs/httpd.html">httpd</a></code> pour les autres arguments de la ligne de 
109    commande)</p>
