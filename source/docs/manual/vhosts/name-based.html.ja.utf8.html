47<h2><a name="namevip" id="namevip">名前ベースと IP ベースのバーチャルホストの比較</a></h2>
48
49    <p>IP ベースのバーチャルホストでは、応答する
50    バーチャルホストへのコネクションを決定するために IP
51    アドレスを使用します。ですから、それぞれのホストに個々に IP
52    アドレスが必要になります。これに対して名前ベースのバーチャルホストでは、
53    クライアントが HTTP ヘッダの一部としてホスト名を告げる、
54    ということに依存します。この技術で同一 IP 
55    アドレスを異なる多数のホストで共有しています。</p>
56
57    <p>名前ベースのバーチャルホストは通常単純で、それぞれのホスト名と
58    それに対応する正確な IP アドレスを DNS で設定し、異なる
59    ホスト名を区別するように Apache HTTP サーバを設定するだけです。
60    さらに、名前ベースのバーチャルホストは不足する IP
61    アドレスの需要を緩和します。したがって、IP ベースのバーチャルホストを
62    選択すべき特定の理由がなければ名前ベースのバーチャルホストを使うべきです。
63    IP ベースのバーチャルホストを使用することを考慮する理由として、</p>
64
65    <ul> 
66      <li>名前ベースのバーチャルホストに対応していない古いクライアントがある
67      名前ベースのバーチャルホストが働くためには、クライアントは
68      HTTP ホストヘッダを送ってこなければなりません。
69      これは HTTP/1.1 の仕様で要求されていて、すべての現代的な
70      HTTP/1.0 ブラウザでも拡張として実装されています。
71      とても古いクライアントをサポートしつつ、名前ベースの
72      バーチャルホストを行いたい場合は、この文書の最後の方に
73      書かれている解決策になるかもしれない方法を見てください。</li>
74
75      <li>名前ベースのバーチャルホストは SSL プロトコルの特徴により、
76      SSL セキュアサーバには使えません。</li>
77
78      <li>オペレーティングシステムやネットワーク装置のなかには、
79      別の IP アドレス上でない場合、複数のホストを別扱いできないような
80      帯域管理の方法を実装しているものがあります。</li>
81    </ul>

85<h2><a name="using" id="using">名前ベースのバーチャルホストを利用する</a></h2>
86
87<table class="related"><tr><th>関連モジュール</th><th>関連ディレクティブ</th></tr><tr><td><ul><li><code class="module"><a href="../mod/core.html">core</a></code></li></ul></td><td><ul><li><code class="directive"><a href="../mod/core.html#documentroot">DocumentRoot</a></code></li><li><code class="directive"><a href="../mod/core.html#namevirtualhost">NameVirtualHost</a></code></li><li><code class="directive"><a href="../mod/core.html#serveralias">ServerAlias</a></code></li><li><code class="directive"><a href="../mod/core.html#servername">ServerName</a></code></li><li><code class="directive"><a href="../mod/core.html#serverpath">ServerPath</a></code></li><li><code class="directive"><a href="../mod/core.html#virtualhost">VirtualHost</a></code></li><li><code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code></li></ul></td></tr></table>
88
89    <p>名前ベースのバーチャルホストを使うには、そのホストへの
90    リクエストを受け付けるサーバの IP アドレス (もしかしたらポートも)
91    を指定する必要があります。
92    これは <code class="directive"><a href="../mod/core.html#namevirtualhost">NameVirtualHost</a></code>
93    ディレクティブで設定します。通常、<code class="directive"><a href="../mod/core.html#namevirtualhost">NameVirtualHost</a></code> で
94    <code>*</code> の属性を使ってサーバの全ての IP アドレスを使います。
95    (例えば SSL の使用などで) 複数のポートを使うことを計画しているのであれば、
96    引数に <code>*:80</code> のようにポートも含めるようにしてください。
97    <code class="directive"><a href="../mod/core.html#namevirtualhost">NameVirtualHost</a></code> ディレクティブで
98    IP アドレスを書いても、
99    自動的にサーバがその IP アドレスをリッスンするということはないことに
100    注意してください。詳細は「<a href="../bind.html">Apache の使うアドレスと
101    ポートを設定する</a>」を読んでください。さらに、ここで指定された
102    IP アドレスは全てサーバのネットワークインターフェースと関連付けられて
103    いなければなりません。</p>
104
105    <p>次は、扱うホストそれぞれに対して <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code> ブロックを
106    作成してください。<code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
107    ディレクティブの引数は <code class="directive"><a href="../mod/core.html#namevirtualhost">NameVirtualHost</a></code>
108    ディレクティブの引数と同じにしてください (すなわち、IP アドレスか、全てのアドレスを意味する
109    <code>*</code>)。それぞれの <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
110    ディレクティブの中には、最低限、どのホストが扱われるかを示す <code class="directive"><a href="../mod/core.html#servername">ServerName</a></code> ディレクティブと、
111    そのホスト用のコンテンツがファイルシステム上のどこにあるかを示す
112    <code class="directive"><a href="../mod/core.html#documentroot">DocumentRoot</a></code> ディレクティブを
113    書く必要があります。</p>
114
115    <div class="note"><h3>メインホストはなくなります</h3>
116        <p>既にあるウェブサーバにバーチャルホストを追加する場合、
117        既存のウェブサーバに対しても <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
118        ブロックを作らなければなりません。このバーチャルホストの
119        <code class="directive"><a href="../mod/core.html#servername">ServerName</a></code> と
120        <code class="directive"><a href="../mod/core.html#documentroot">DocumentRoot</a></code>
121        は、グローバルな <code class="directive"><a href="../mod/core.html#servername">ServerName</a></code> と
122        <code class="directive"><a href="../mod/core.html#documentroot">DocumentRoot</a></code>
123        と同じものにします。また、このバーチャルホストを設定ファイルの中で
124        先頭に置いて、デフォルトホストとして動作するようにします。</p>
125    </div>
126
127    <p>たとえば、<code>www.domain.tld</code> を動かしていて、
128    さらにバーチャルホスト <code>www.otherdomain.tld</code>
129    を追加するとしましょう。このバーチャルホストは同一 IP を指しているとします。
130    そのような場合は、<code>httpd.conf</code>
131    に以下のようなコードを追加するだけです</p>
132
133    <div class="example"><p><code>
134        NameVirtualHost *:80<br />
135        <br />
136        &lt;VirtualHost *:80&gt;<br />
137        <span class="indent">
138            ServerName www.domain.tld<br />
139            ServerAlias domain.tld *.domain.tld<br />
140            DocumentRoot /www/domain<br />
141        </span>
142        &lt;/VirtualHost&gt;<br />
143        <br />
144        &lt;VirtualHost *:80&gt;<br />
145        <span class="indent">ServerName www.otherdomain.tld<br />
146            DocumentRoot /www/otherdomain<br />
147        </span>
148        &lt;/VirtualHost&gt;<br />
149    </code></p></div>
150
151    <p><code class="directive"><a href="../mod/core.html#namevirtualhost">NameVirtualHost</a></code> 及び
152    <code class="directive"><a href="../mod/core.html#virtualhost">VirtualHost</a></code> のどちらの場合も、
153    * の部分には明示的に IP アドレスを指定することができます。
154    例えば、ある IP アドレスでは名前ベースのバーチャルホストを使いたい一方で、
155    別の IP アドレスでは、他の IP ベースのバーチャルホストや
156    別組の名前ベースのバーチャルホストを使いたい場合、
157    そう設定することになるでしょう。</p>
158
159    <p>複数の名前でサーバアクセスができるようにしたいことも多いでしょう。
160    このようなことは、<code class="directive"><a href="../mod/core.html#serveralias">ServerAlias</a></code> ディレクティブを <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
161    セクションに記述することで実現できます。
162    例えば上記の <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code> の例であれば、
163    次のように一覧に挙げられた名前が、
164    ユーザが同一のウェブサイトとして目にして使用できるサーバ名である、
165    と <code class="directive"><a href="../mod/core.html#serveralias">ServerAlias</a></code>
166    ディレクティブで指定できます。</p> 
167
168    <div class="example"><p><code>
169        ServerAlias domain.tld *.domain.tld
170    </code></p></div>
171
172    <p><code>domain.tld</code> ドメインへの全てのホストへのリクエストは
173    <code>www.domain.tld</code> のバーチャルホストが処理します。
174    名前をマッチさせるために、ワイルドカード文字 * や ? 
175    を使用することもできます。もちろん思いつきの名前を作って、
176    <code class="directive"><a href="../mod/core.html#servername">ServerName</a></code> や 
177    <code class="directive"><a href="../mod/core.html#serveralias">ServerAlias</a></code>
178    にその名前を書くといったことはできません。まずは、
179    これらの名前が サーバに付けられた IP アドレスにマップされるように
180    DNS サーバを適切に設定しなければなりません。</p>
181
182    <p>最後に、<code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code> コンテナの中に
183    他のディレクティブを書くことで、バーチャルホストの設定を細かく調整
184    することができます。
185    ほとんどのディレクティブはこれらのコンテナに設置することができて、
186    変更点はそのバーチャルホストに対してのみ有効になります。
187    どのディレクティブを書くことができるかは、ディレクティブの <a href="../mod/directive-dist.html#context">コンテキスト</a> を
188    調べてください。<em>主サーバコンテキスト</em>
189    (<code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
190    コンテナの外) の設定用ディレクティブはバーチャルホストでの設定で
191    上書きされない場合のみ使用されます。</p>
192
193    <p>リクエストが来ると、サーバはまず最初に <code class="directive"><a href="../mod/core.html#namevirtualhost">&lt;NameVirtualHost&gt;</a></code>
194    にマッチする IP アドレスかどうかをチェックします。マッチすれば
195    マッチした IP アドレスの <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
196    のそれぞれのセクションの中から 
197    <code class="directive"><a href="../mod/core.html#servername">ServerName</a></code> か
198    <code class="directive"><a href="../mod/core.html#serveralias">ServerAlias</a></code> 
199    に要求されたホスト名があるか探します。
200    見つかればそのサーバ用の設定を使います。マッチするバーチャルホスト
201    が見つからなければ、マッチした IP アドレスの 
202    <strong>リストの最初にあるバーチャルホスト</strong> が使われます。</p>
203
204    <p>結果として、リストの最初のバーチャルホストが <em>デフォルト</em> の
205    バーチャルホストになります。IP アドレスが <code class="directive"><a href="../mod/core.html#namevirtualhost">NameVirtualHost</a></code>
206    ディレクティブにマッチした場合は、<em>メインのサーバ</em> の
207    <code class="directive"><a href="../mod/core.html#documentroot">DocumentRoot</a></code> 
208    は<strong>決して使われません</strong>
209    どのバーチャルホストにもマッチしないリクエストに対して、
210    特別な設定をしたいのであれば、設定ファイル中の最初の
211    <code>&lt;VirtualHost&gt;</code> コンテナにそれを記述してください。</p>

215<h2><a name="compat" id="compat">古いブラウザとの互換性</a></h2>
216
217    <p>以前述べたように、名前ベースのバーチャルホストが正しく動作する
218    ために必要な情報を送ってこないクライアントが依然として存在しています。
219    そのようなクライアントに対しては、該当する IP アドレスについて、
220    一番最初に設定されているバーチャルホスト
221    (<cite>プライマリ</cite>の名前ベースのバーチャルホスト)
222    からページが送り返されます。</p>
223
224    <div class="note"><h3>どのぐらい古いの ?</h3>
225    <p>「古い」と表現している場合、本当に古いことを意味して使っています。
226    不幸にして今現在でもこのような古いブラウザに遭遇することがあります。
227    現在のブラウザは全て、名前ベースのバーチャルホストに必要な
228    <code>Host</code> ヘッダを送ります。</p>
229    </div>
230
231    <p><a href="../mod/core.html#serverpath"><code>ServerPath</code></a>
232    ディレクティブで対処が可能です。ちょっと不格好ですけれども。</p>
233
234    <p>設定例</p>
235
236    <div class="example"><p><code>
237        NameVirtualHost 111.22.33.44<br />
238        <br />
239        &lt;VirtualHost 111.22.33.44&gt;<br />
240        <span class="indent">
241            ServerName www.domain.tld<br />
242            ServerPath /domain<br />
243            DocumentRoot /web/domain<br />
244        </span>
245        &lt;/VirtualHost&gt;<br />
246    </code></p></div>
247
248    <p>この例にはどういう意味があるでしょうか? これは
249    "<code>/domain</code>" で始まる URI へのリクエストはすべて、
250    バーチャルホスト <code>www.domain.tld</code> で処理される、
251    という意味です。つまり、すべてのクライアントで
252    <code>http://www.domain.tld/domain/</code> でアクセスできるページが、
253    <code>Host:</code> ヘッダを送ってくるクライアントであれば
254    <code>http://www.domain.tld/</code> としてもアクセスできる、
255    という意味です。</p>
256
257    <p>これが動作するようにするには、
258    プライマリのバーチャルホストのページに
259    <code>http://www.domain.tld/domain/</code> へのリンクを設置します。
260    そして、バーチャルホストのページでは、純粋な相対リンク (<em>例:</em>
261    "<code>file.html</code>" や "<code>../icons/image.gif</code>")、
262    あるいは <code>/domain/</code> で始まるリンク (<em>例:</em>
263    "<code>http://www.domain.tld/domain/misc/file.html</code>" や
264    "<code>/domain/misc/file.html</code>") だけを設置します。</p>
265
266    <p>これには、幾分かの規律が必要となりますが、
267    このようなガイドラインを忠実に守ることにより、たいていの場合、
268    すべてのブラウザで ― 新しいブラウザでも古いものでも ―
269    作成したページが見えるということを保証します。</p>
