43<h2><a name="requirements" id="requirements">Système requis</a></h2>
44
45    <p>Comme l'indique le terme <cite>par IP</cite>, le serveur
46    <strong>doit disposer de différentes paires adresses IP/port pour chaque
47    serveur virtuel par IP</strong>. La machine peut posséder
48    plusieurs connexions physiques au réseau, ou utiliser des
49    interfaces virtuelles qui sont supportées par la plupart des
50    systèmes d'exploitation modernes (Consultez la documentation des
51    systèmes d'exploitation pour plus de détails, notamment les "alias
52    IP" et la commande "ifconfig" pour les activer), et/ou utiliser
53    plusieurs numéros de port.</p>
54
55    <p>Selon la terminologie du serveur HTTP Apache, l'utilisation d'une
56    seule adresse IP avec plusieurs ports TCP s'apparente aussi à de
57    l'hébergement virtuel basé sur IP.</p>

60<h2><a name="howto" id="howto">Comment configurer Apache</a></h2>
61
62    <p>Il y a deux manières de configurer Apache pour le support de
63    multiples serveurs virtuels. Il suffit soit de faire tourner un
64    processus résident <code class="program"><a href="../programs/httpd.html">httpd</a></code> pour chaque nom de
65    domaine, soit de faire tourner un unique processus résident qui
66    gère tous les serveurs virtuels.</p>
67
68    <p>Utilisez des processus résidents multiples lorsque&nbsp;:</p>
69
70    <ul>
71      <li>il y a des problèmes de répartition de sécurité, tels
72      qu'une entreprise1 ne souhaite que personne d'une entreprise2
73      ne puisse lire ses données excepté via le Web. Dans ce cas,
74      vous aurez besoin de deux processus résidents, chacun fonctionnant
75      avec des paramètres <code class="directive"><a href="../mod/mod_unixd.html#user">User</a></code>,
76      <code class="directive"><a href="../mod/mod_unixd.html#group">Group</a></code>,
77      <code class="directive"><a href="../mod/mpm_common.html#listen">Listen</a></code>, et
78      <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> différents.</li>
79
80      <li>vous disposez suffisamment de mémoire et de
81      <a href="../misc/descriptors.html">descripteurs de fichiers</a>
82      pour l'écoute de chaque alias IP de la machine. Il est seulement
83      possible d'appliquer la directive
84      <code class="directive"><a href="../mod/mpm_common.html#listen">Listen</a></code>, soit sur toutes
85      les adresses avec le joker "*", soit uniquement sur des adresses
86      spécifiques. Donc, si vous avez besoin d'écouter une adresse
87      en particulier, vous devrez le faire pour l'ensemble des
88      autres adresses (Bien qu'il soit plus simple de lancer un
89      processus <code class="program"><a href="../programs/httpd.html">httpd</a></code> pour écouter N-1 adresses,
90      et un autre pour l'adresse restante).</li>
91    </ul>
92
93    <p>Utilisez un unique processus résident lorsque&nbsp;:</p>
94
95    <ul>
96      <li>le partage de la configuration httpd entre les serveurs
97      virtuels est acceptable.</li>
98
99      <li>la machine assume déjà une grande quantité de requêtes, et
100      que l'ajout de processus résidents supplémentaires en affecterait
101      les performances.</li>
102    </ul>

106<h2><a name="multiple" id="multiple">Configuration de processus multiples</a></h2>
107
108    <p>Créez une installation indépendante du programme
109    <code class="program"><a href="../programs/httpd.html">httpd</a></code> pour chaque serveur virtuel. Pour
110    chacune d'elle, utilisez la directive
111    <code class="directive"><a href="../mod/mpm_common.html#listen">Listen</a></code> dans le fichier
112    de configuration pour définir l'adresse IP (ou serveur virtuel)
113    que le processus résident doit gérer. Par exemple&nbsp;:</p>
114
115    <pre class="prettyprint lang-config">Listen 192.0.2.100:80</pre>
116
117
118    <p>Il est recommandé d'utiliser une adresse IP plutôt qu'un nom
119    de domaine (consultez <a href="../dns-caveats.html">Problèmes DNS
120    avec Apache</a>).</p>
121

124<h2><a name="single" id="single">Configuration d'un unique processus
125résident pour des serveurs virtuels</a></h2>
126
127    <p>Dans ce cas, un unique processus httpd va gérer les requêtes
128    pour le serveur principal et tous les serveurs virtuels. Dans le
129    fichier de configuration, la directive
130    <code class="directive"><a href="../mod/core.html#virtualhost">VirtualHost</a></code> va servir à
131    définir les autres directives
132    <code class="directive"><a href="../mod/core.html#serveradmin">ServerAdmin</a></code>,
133    <code class="directive"><a href="../mod/core.html#servername">ServerName</a></code>,
134    <code class="directive"><a href="../mod/core.html#documentroot">DocumentRoot</a></code>,
135    <code class="directive"><a href="../mod/core.html#errorlog">ErrorLog</a></code> et
136    <code class="directive"><a href="../mod/mod_log_config.html#transferlog">TransferLog</a></code> ou
137    <code class="directive"><a href="../mod/mod_log_config.html#customlog">CustomLog</a></code> avec des
138    valeurs différentes pour chaque serveur virtuel. Par exemple&nbsp;:</p>
139
140    <pre class="prettyprint lang-config">&lt;VirtualHost 172.20.30.40:80&gt;
141    ServerAdmin webmaster@www1.example.com
142    DocumentRoot "/www/vhosts/www1"
143    ServerName www1.example.com
144    ErrorLog "/www/logs/www1/error_log"
145    CustomLog "/www/logs/www1/access_log" combined
146&lt;/VirtualHost&gt;
147
148&lt;VirtualHost 172.20.30.50:80&gt;
149    ServerAdmin webmaster@www2.example.org
150    DocumentRoot "/www/vhosts/www2"
151    ServerName www2.example.org
152    ErrorLog "/www/logs/www2/error_log"
153    CustomLog "/www/logs/www2/access_log" combined
154&lt;/VirtualHost&gt;</pre>
155
156
157    <p>Il est recommandé d'utiliser une adresse IP plutôt qu'un nom
158    de domaine comme argument à la directive &lt;VirtualHost&gt;
159     (consultez <a href="../dns-caveats.html">Problèmes DNS
160    avec Apache</a>).</p>
161
162    <p>Presque <strong>toutes</strong> les directives de configuration
163    peuvent être employées dans une directive VirtualHost, à l'exception
164    des directives qui contrôlent la création du processus et de
165    quelques autres. Pour connaître celles utilisables dans une
166    directive VirtualHost, vérifiez leur
167    <a href="../mod/directive-dict.html#Context">Contexte</a> en utilisant
168    l'<a href="../mod/quickreference.html">directive index</a>.</p>
169
170
171    <p><code class="directive"><a href="../mod/mod_suexec.html#suexecusergroup">SuexecUserGroup</a></code> peut être
172    utilisées à l'intérieur d'une directive VirtualHost si l'exécution se fait
173    sous suEXEC. (Voir <a href="../suexec.html">suEXEC</a>).</p>
174
175    <p><em>SÉCURITÉ&nbsp;:</em> lorsque vous spécifiez où écrire les
176    fichiers journaux, soyez attentif aux risques si quelqu'un d'autre
177    que celui qui a démarré Apache dispose des droits d'écriture
178    sur l'emplacement de ces fichiers. Consultez les
179    <a href="../misc/security_tips.html">Conseils sur la sécurité</a>
180    pour plus de détails.</p>
