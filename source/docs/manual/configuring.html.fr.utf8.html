45<h2><a name="main" id="main">Fichiers de configuration principaux</a></h2>
46    
47    <table class="related"><tr><th>Modules Apparentés</th><th>Directives Apparentées</th></tr><tr><td><ul><li><code class="module"><a href="./mod/mod_mime.html">mod_mime</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/core.html#ifdefine">&lt;IfDefine&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#include">Include</a></code></li><li><code class="directive"><a href="./mod/mod_mime.html#typesconfig">TypesConfig</a></code></li></ul></td></tr></table>
48
49    <p>La configuration du serveur HTTP Apache est effectuée en plaçant des <a href="mod/directives.html">directives</a> dans des fichiers de
50    configuration au format texte. Le fichier de configuration principal se nomme
51    en général
52    <code>httpd.conf</code>. La localisation de ce fichier est définie
53    à la compilation, mais peut être redéfinie à l'aide de l'option
54    de ligne de commande <code>-f</code>. En outre, d'autres fichiers de
55    configuration peuvent être ajoutés à l'aide de la directive
56    <code class="directive"><a href="./mod/core.html#include">Include</a></code>, et des caractères de
57    remplacement
58    peuvent être utilisés pour inclure de nombreux fichiers de configuration.
59    Des directives de tous types peuvent être placées dans chacun de ces fichiers
60    de configuration. Les modifications dans les fichiers de configuration
61    principaux ne sont prises en compte par httpd que lorsque le serveur
62    est démarré ou redémarré.</p>
63
64    <p>Le serveur lit aussi un fichier contenant les types de document mime&nbsp;;
65    ce fichier est défini par la directive <code class="directive"><a href="./mod/mod_mime.html#typesconfig">TypesConfig</a></code>,
66    et se nomme <code>mime.types</code> par défaut.</p>

69<h2><a name="syntax" id="syntax">Syntaxe des fichiers de configuration</a></h2>
70    
71
72    <p>Les fichiers de configuration de httpd contiennent une directive par
73    ligne.  On peut utiliser l'anti-slash "\" comme dernier caractère d'une
74    ligne pour indiquer que la directive continue à la ligne suivante ; pour que
75    cette continuation soit prise en compte, il ne doit y avoir aucun caractère
76    ni blanc entre l'anti-slash et la fin de la ligne.</p>
77
78    <p>Les lignes qui commencent par un croisillon «&nbsp;#&nbsp;» sont interprétées comme
79    des commentaires et sont ignorées. Les commentaires <strong>ne doivent
80    pas</strong> se trouver sur la même ligne qu’une directive de configuration,
81    y compris au début de la continuation d’une ligne comme décrit dans le
82    paragraphe précédent. Les commentaires peuvent aussi continuer sur la ligne
83    suivante, ce qui signifie que la prochaine ligne physique est aussi
84    interprétée comme un commentaire car la ligne logique commence par un
85    croisillon.
86    </p>
87
88    <p>Les blancs apparaissant avant une directive sont ignorés ; vous pouvez
89    donc indenter ces dernières pour améliorer la lisibilité. Les lignes vides
90    sont aussi ignorées. les arguments de directive sont séparés par des blancs.
91    Si un argument contient des espaces, il doit être entouré de guillemets.</p>
92
93    <p>Les directives dans les fichiers de configuration ne sont pas
94    sensibles à la casse, mais leurs arguments le sont souvent.</p>
95
96    <p>Les valeurs des variables d'environnement ou des variables définies via
97    la directive <code class="directive"><a href="./mod/core.html#define">Define</a></code> peuvent être
98    utilisées dans le fichier de configuration en utilisant la syntaxe
99    <code>${VAR}</code>. Si "VAR" est le nom d'une variable valable, la valeur
100    de la variable est alors substituée à la chaîne <code>${VAR}</code>, et le
101    processus de lecture du fichier de configuration continue comme si la
102    chaîne correspondant à la valeur de la variable s'y était trouvée
103    littéralement. Les variables définies via la directive <code class="directive"><a href="./mod/core.html#define">Define</a></code> l'emportent sur les autres variables
104    d'environnement de l'interpréteur de commande. Si la variable "VAR" n'est
105    pas trouvée, la chaîne <code>${VAR}</code> n'est pas modifiée, et un
106    avertissement est enregistré dans le journal. Le caractère ":" est interdit
107    dans les noms de variables afin d'éviter tout conflit avec la syntaxe de la
108    directive <code class="directive"><a href="./mod/mod_rewrite.html#rewritemap">RewriteMap</a></code>.</p>
109
110    <p>Seules les variables d'environnement de l'interpréteur de commande
111    définies avant le démarrage du serveur peuvent être utilisées en extension.
112    Les variables d'environnement définies dans le fichier de configuration
113    lui-même, par exemple avec <code class="directive"><a href="./mod/mod_env.html#setenv">SetEnv</a></code>,
114    prennent effet trop tard pour pouvoir être utilisées en extension au sein
115    du fichier de configuration.</p>
116
117    <p>La longueur maximale d'une ligne dans un fichier de configuration
118    normal, après substitution des variables et fusion des lignes
119    interrompues, est approximativement de 16 Mo. Dans les <a href="configuring.html#htaccess">fichiers .htaccess</a>, la longueur
120    maximale est de 8190 caractères.</p>
121
122    <p>Vous pouvez vérifier l'absence d'erreurs de syntaxe dans vos fichiers
123    de configuration sans démarrer le serveur à l'aide de la commande
124    <code>apachectl configtest</code> ou de l'option de ligne de commande
125    <code>-t</code>.</p>
126
127    <p>Vous pouvez utiliser la définition <code>-DDUMP_CONFIG</code> de
128    <code class="module"><a href="./mod/mod_info.html">mod_info</a></code> pour afficher la configuration avec tous
129    les fichiers inclus et les variables d'environnement évaluées, tous
130    les commentaires et les sections <code class="directive"><a href="./mod/core.html#ifdefine">&lt;IfDefine&gt;</a></code> et <code class="directive"><a href="./mod/core.html#ifmodule">&lt;IfModule&gt;</a></code> non actives ayant
131    été supprimés. Cependant, la sortie ne reflète
132    pas les fusions ou écrasements pouvant intervenir en cas de
133    définitions multiples de directives.</p>

136<h2><a name="modules" id="modules">Modules</a></h2>
137    
138
139    <table class="related"><tr><th>Modules Apparentés</th><th>Directives Apparentées</th></tr><tr><td><ul><li><code class="module"><a href="./mod/mod_so.html">mod_so</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/core.html#ifmodule">&lt;IfModule&gt;</a></code></li><li><code class="directive"><a href="./mod/mod_so.html#loadmodule">LoadModule</a></code></li></ul></td></tr></table>
140
141    <p>httpd est un serveur modulaire. Ceci implique que seules les
142    fonctionnalités les plus courantes sont incluses dans le serveur de base.
143    Les fonctionnalités étendues sont fournies à l'aide de <a href="mod/">modules</a> qui peuvent être chargés dans httpd.
144    Par défaut, un jeu de <a href="mod/module-dict.html#Status">modules de base</a> est inclus dans le
145    serveur à la compilation. Si le serveur est compilé de façon à utiliser
146    les <a href="dso.html">modules chargés dynamiquement</a>,
147    alors les modules peuvent être compilés séparément et chargés à
148    n'importe quel moment à l'aide de la directive
149    <code class="directive"><a href="./mod/mod_so.html#loadmodule">LoadModule</a></code>.
150    Dans le cas contraire, httpd doit être recompilé pour ajouter ou
151    supprimer des modules.
152    Les directives de configuration peuvent être incluses de manière
153    conditionnelle selon la présence ou l'absence d'un module particulier
154    en les plaçant dans un bloc <code class="directive"><a href="./mod/core.html#ifmodule">&lt;IfModule&gt;</a></code>.</p>
155
156    <p>Pour voir quels modules ont été compilés avec le serveur,
157    vous pouvez utiliser l'option de ligne de commande <code>-l</code>.</p>

160<h2><a name="scope" id="scope">Portée des directives</a></h2>
161    
162
163    <table class="related"><tr><th>Modules Apparentés</th><th>Directives Apparentées</th></tr><tr><td /><td><ul><li><code class="directive"><a href="./mod/core.html#directory">&lt;Directory&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#directorymatch">&lt;DirectoryMatch&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#files">&lt;Files&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#filesmatch">&lt;FilesMatch&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#location">&lt;Location&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#locationmatch">&lt;LocationMatch&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code></li></ul></td></tr></table>
164
165    <p>Les directives placées dans les fichiers de configuration principaux
166    s'appliquent au serveur dans son ensemble. Si vous souhaitez modifier la
167    configuration d'une partie du serveur seulement, vous pouvez limiter la
168    portée de vos directives en les plaçant dans une section
169    <code class="directive"><a href="./mod/core.html#directory">&lt;Directory&gt;</a></code>, <code class="directive"><a href="./mod/core.html#directorymatch">&lt;DirectoryMatch&gt;</a></code>, <code class="directive"><a href="./mod/core.html#files">&lt;Files&gt;</a></code>, <code class="directive"><a href="./mod/core.html#filesmatch">&lt;FilesMatch&gt;</a></code>, <code class="directive"><a href="./mod/core.html#location">&lt;Location&gt;</a></code> ou <code class="directive"><a href="./mod/core.html#locationmatch">&lt;LocationMatch&gt;</a></code>.
170    Ces sections limitent le champ d'application des directives qu'elles
171    contiennent à des URls ou des portions du système de fichiers particulières.
172    Elles peuvent aussi être imbriquées, ce qui permet
173    une configuration très fine.</p>
174
175    <p>httpd peut servir simultanément de nombreux sites web au travers des <a href="vhosts/">Serveurs Virtuels</a>. La portée des directives peut ainsi
176    être limitée en les plaçant dans des sections <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>, afin qu'elles ne s'appliquent
177    qu'aux requêtes pour un site web particulier.</p>
178
179    <p>Bien que la plupart des directives puissent être placées dans
180    chacune de ces sections, certaines d'entre elles n'ont aucun sens
181    dans certains contextes.
182    Par exemple, les directives qui contrôlent la création des processus
183    n'ont de sens que dans le contexte du serveur global. Pour déterminer
184    quelles directives peuvent être placées dans quelles sections, consultez
185    le <a href="mod/directive-dict.html#Context">Contexte</a> de la
186    directive. Pour plus d'informations, nous fournissons des détails dans
187    <a href="sections.html">Comment fonctionnent les sections Directory,
188    Location et Files</a>.</p>

191<h2><a name="htaccess" id="htaccess">Fichiers .htaccess</a></h2>
192    
193
194    <table class="related"><tr><th>Modules Apparentés</th><th>Directives Apparentées</th></tr><tr><td /><td><ul><li><code class="directive"><a href="./mod/core.html#accessfilename">AccessFileName</a></code></li><li><code class="directive"><a href="./mod/core.html#allowoverride">AllowOverride</a></code></li></ul></td></tr></table>
195
196    <p>httpd permet la gestion décentralisée de la configuration
197    à l'aide de fichiers spéciaux placés dans l'arborescence du site web.
198    Ces fichiers spéciaux se nomment en général <code>.htaccess</code>,
199    mais tout autre nom peut être spécifié à l'aide de la directive
200    <code class="directive"><a href="./mod/core.html#accessfilename">AccessFileName</a></code>.
201    Les directives placées dans les fichiers <code>.htaccess</code>
202    s'appliquent au répertoire dans lequel vous avez placé le fichier,
203    ainsi qu'à tous ses sous-répertoires.
204    La syntaxe des fichiers <code>.htaccess</code> est la même que celle
205    des fichiers de configuration principaux. Comme les fichiers
206    <code>.htaccess</code> sont lus à chaque requête, les modifications de
207    ces fichiers prennent effet immédiatement.</p>
208
209    <p>Pour déterminer quelles directives peuvent être placées
210    dans les fichiers <code>.htaccess</code>, consultez le
211    <a href="mod/directive-dict.html#Context">Contexte</a> de la
212    directive. L'administrateur du serveur peut contrôler quelles
213    directives peuvent être placées dans les fichiers
214    <code>.htaccess</code> en définissant la directive
215    <code class="directive"><a href="./mod/core.html#allowoverride">AllowOverride</a></code>
216    dans les fichiers de configuration principaux.</p>
217
218    <p>Pour plus d'informations sur les fichiers <code>.htaccess</code>,
219    se référer au <a href="howto/htaccess.html">tutoriel .htaccess</a>.</p>
