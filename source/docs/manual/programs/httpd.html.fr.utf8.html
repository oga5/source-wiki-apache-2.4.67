32     <p><code class="program"><a href="../programs/httpd.html">httpd</a></code> est le programme du serveur HTTP d'Apache. Il
33     a été conçu pour fonctionner sous forme de processus démon
34     indépendant. Lorsqu'il est utilisé ainsi, il va créer un jeu de
35     processus enfants ou de threads qui traiteront les requêtes.</p>
36
37     <p>En général, <code class="program"><a href="../programs/httpd.html">httpd</a></code> n'est pas invoqué directement,
38     mais plutôt via <code class="program"><a href="../programs/apachectl.html">apachectl</a></code> sur les systèmes de
39     style Unix ou <a href="../platform/windows.html#winsvc">en tant que service sous
40     Windows NT, 2000 et XP</a> et <a href="../platform/windows.html#wincons">comme application de
41     console sous Windows 9x et ME</a>.</p>

71<h2><a name="options" id="options">Options</a></h2>
72
73<dl>
74<dt><code>-d <var>racine-serveur</var></code></dt>
75
76<dd>Définit la valeur initiale de la directive <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> à <var>racine-serveur</var>. Cette
77valeur peut être écrasée par la directive ServerRoot du fichier de
78configuration. La valeur par défaut est
79<code>/usr/local/apache2</code>.</dd>
80
81<dt><code>-f <var>config</var></code></dt>
82
83<dd>Utilise les directives du fichier <var>config</var> au démarrage. Si
84<var>config</var> ne commence pas par un '/', il est considéré comme
85relatif au chemin défini par la directive <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>. La valeur par défaut est
86<code>conf/httpd.conf</code>.</dd>
87
88<dt><code>-k <code>start|restart|graceful|stop|graceful-stop</code></code></dt>
89
90<dd>Permet de démarrer, redémarrer ou arrêter <code class="program"><a href="../programs/httpd.html">httpd</a></code>. Voir <a href="../stopping.html">Arrêter Apache httpd</a> pour plus d'informations.</dd>
91
92<dt><code>-C <var>directive</var></code></dt>
93
94<dd>Exécute la directive de configuration <var>directive</var> avant de
95lire les fichiers de configurations.</dd>
96
97<dt><code>-c <var>directive</var></code></dt>
98
99<dd>Exécute la directive de configuration <var>directive</var> après
100avoir lu les fichiers de configurations.</dd>
101
102
103<dt><code>-D <var>paramètre</var></code></dt>
104
105<dd>Définit un <var>paramètre</var> de configuration à utiliser dans les
106sections <code class="directive"><a href="../mod/core.html#ifdefine">&lt;IfDefine&gt;</a></code>
107des fichiers de configuration, ces dernières permettant d'exécuter ou
108non des
109commandes au démarrage ou au redémarrage du serveur. Sert aussi à
110définir certains paramètres de démarrage moins courants comme
111<code>-DNO_DETACH</code> (empêche le processus parent de lancer des
112processus enfants) et <code>-DFOREGROUND</code> (empêche le processus
113parent d'appeler <code>setsid()</code> et autres).</dd>
114
115<dt><code>-e <var>niveau</var></code></dt>
116
117<dd>Définit la directive <code class="directive"><a href="../mod/core.html#loglevel">LogLevel</a></code> à
118<var>niveau</var> pendant le démarrage du serveur. Ceci permet
119d'augmenter temporairement la verbosité des messages d'erreur afin de
120déterminer les problèmes de démarrage.</dd>
121
122<dt><code>-E <var>fichier</var></code></dt>
123
124<dd>Envoie les messages d'erreur de démarrage vers le fichier
125<var>fichier</var>.</dd>
126
127<dt><code>-h</code></dt>
128
129<dd>Affiche un bref résumé des options de ligne de commande
130disponibles.</dd>
131
132<dt><code>-l</code></dt>
133
134<dd>Affiche la liste des modules compilés dans le le serveur. Ce
135paramètre n'affiche <strong>pas</strong> les modules chargés
136dynamiquement via la directive <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code>.</dd>
137
138<dt><code>-L</code></dt>
139
140<dd>Affiche une liste des directives fournies par les modules statiques
141avec les arguments associés, ainsi que les contextes dans lesquels elles
142sont valides. Les directives fournies par les modules partagés
143(dynamiques) ne sont pas affichées).</dd>
144
145<dt><code>-M</code></dt>
146
147<dd>Affiche une liste des modules statiques et des modules chargés
148dynamiquement.</dd>
149
150<dt><code>-S</code></dt>
151
152<dd>Affiche la configuration telle qu'elle est issue de l'interprétation
153du fichier de configuration (actuellement, seule la configuration des
154serveurs virtuels est affichée).</dd>
155
156<dt><code>-T</code> (disponible depuis la version 2.3.8)</dt>
157
158<dd>Empêche la vérification de la racine des documents (DocumentRoot) au
159démarrage/redémarrage.</dd>
160
161<dt><code>-t</code></dt>
162
163<dd>Exécute une vérification de syntaxe pour les fichiers de
164configuration seulement. Le programme se termine immédiatement après ces
165tests de vérification de syntaxe avec soit un code de retour de 0
166(syntaxe OK), soit un code de retour différent de 0 (erreur de
167syntaxe). Si -D <var>DUMP</var>_<var>VHOSTS </var> est défini, les
168détails de la configuration des serveurs virtuels seront affichés. Si -D
169<var>DUMP</var>_<var>MODULES </var> est défini, tous les modules chargés
170seront affichés.</dd>
171
172<dt><code>-v</code></dt>
173
174<dd>Print the version of <code class="program"><a href="../programs/httpd.html">httpd</a></code>, and then exit.</dd>
175
176<dt><code>-V</code></dt>
177
178<dd>Le programme affiche la version et les paramètres de compilation de
179<code class="program"><a href="../programs/httpd.html">httpd</a></code>, puis se termine.</dd>
180
181<dt><code>-X</code></dt>
182
183<dd>Exécute httpd en mode debug. Un seul processus sera démarré, et le
184serveur ne rendra pas la main à la console.</dd>
185
186</dl>
187
188<p>Les arguments suivants ne sont disponibles que sur la <a href="../platform/windows.html">plate-forme Windows</a> :</p>
189
190<dl>
191
192<dt><code>-k install|config|uninstall</code></dt>
193
194<dd>Respectivement : installe Apache httpd en tant que service Windows NT ;
195modifie les options de démarrage du service Apache httpd ; désinstalle le
196service Apache httpd.</dd>
197
198<dt><code>-n <var>nom</var></code></dt>
199
200<dd>Le <var>nom</var> du service Apache httpd à actionner.</dd>
201
202<dt><code>-w</code></dt>
203
204<dd>Garde la console Windows ouverte en cas de problème de façon à ce
205que le message d'erreur puisse être lu.</dd>
206
207</dl>
