32    <p><code class="program"><a href="../programs/htcacheclean.html">htcacheclean</a></code> is used to keep the size of
33    <code class="module"><a href="../mod/mod_cache_disk.html">mod_cache_disk</a></code>'s storage within a given size limit, or
34    limit on inodes in use. This tool can run either manually or in daemon mode.
35    When running in daemon mode, it sleeps in the background and checks the cache
36    directory at regular intervals for cached content to be removed. You can stop
37    the daemon cleanly by sending it a TERM or INT signal. When run manually, a
38    once off check of the cache directory is made for cached content to be
39    removed. If one or more URLs are specified, each URL will be deleted from
40    the cache, if present.</p>

89<h2><a name="options" id="options">Options</a></h2>
90    <dl>
91    <dt><code>-d<var>interval</var></code></dt>
92    <dd>Daemonize and repeat cache cleaning every <var>interval</var> minutes.
93    This option is mutually exclusive with the <code>-D</code>, <code>-v</code>
94    and <code>-r</code> options. To shutdown the daemon cleanly, just send it
95    a <code>SIGTERM</code> or <code>SIGINT</code>.</dd>
96
97    <dt><code>-D</code></dt>
98    <dd>Do a dry run and don't delete anything. This option is mutually
99        exclusive with the <code>-d</code> option. When doing a dry run and
100        deleting directories with <code>-t</code>, the inodes reported deleted
101        in the stats cannot take into account the directories deleted, and will
102        be marked as an estimate.</dd>
103
104    <dt><code>-v</code></dt>
105    <dd>Be verbose and print statistics. This option is mutually exclusive
106    with the <code>-d</code> option.</dd>
107
108    <dt><code>-r</code></dt>
109    <dd>Clean thoroughly. This assumes that the Apache web server is
110    not running (otherwise you may get garbage in the cache). This option
111    is mutually exclusive with the <code>-d</code> option and implies
112    the <code>-t</code> option.</dd>
113
114    <dt><code>-n</code></dt>
115    <dd>Be nice. This causes slower processing in favor of other
116    processes. <code class="program"><a href="../programs/htcacheclean.html">htcacheclean</a></code> will sleep from time to time
117    so that (a) the disk IO will be delayed and (b) the kernel can schedule
118    other processes in the meantime.</dd>
119
120    <dt><code>-t</code></dt>
121    <dd>Delete all empty directories. By default only cache files are
122    removed, however with some configurations the large number of
123    directories created may require attention. If your configuration
124    requires a very large number of directories, to the point that
125    inode or file allocation table exhaustion may become an issue, use
126    of this option is advised.</dd>
127
128    <dt><code>-p<var>path</var></code></dt>
129    <dd>Specify <var>path</var> as the root directory of the disk cache. This
130    should be the same value as specified with the <code class="directive"><a href="../mod/mod_cache_disk.html#cacheroot">CacheRoot</a></code> directive.</dd>
131
132    <dt><code>-P<var>pidfile</var></code></dt>
133    <dd>Specify <var>pidfile</var> as the name of the file to write the
134    process ID to when daemonized.</dd>
135
136    <dt><code>-R<var>round</var></code></dt>
137    <dd>Specify <var>round</var> as the amount to round sizes up to, to
138    compensate for disk block sizes. Set to the block size of the cache
139    partition.</dd>
140
141    <dt><code>-l<var>limit</var></code></dt>
142    <dd>Specify <var>limit</var> as the total disk cache size limit. The value
143    is expressed in bytes by default (or attaching <code>B</code> to the
144    number). Attach <code>K</code> for Kbytes, <code>M</code> for
145    MBytes or <code>G</code> for Gbytes.</dd>
146
147    <dt><code>-L<var>limit</var></code></dt>
148    <dd>Specify <var>limit</var> as the total disk cache inode limit.
149    <code>K</code>, <code>M</code> or <code>G</code> suffix can also be
150    used.</dd>
151
152    <dt><code>-i</code></dt>
153    <dd>Be intelligent and run only when there was a modification of the disk
154    cache. This option is only possible together with the <code>-d</code>
155    option.</dd>
156
157    <dt><code>-a</code></dt>
158    <dd>List the URLs currently stored in the cache. Variants of the same URL
159    will be listed once for each variant.</dd>
160
161    <dt><code>-A</code></dt>
162    <dd>List the URLs currently stored in the cache, along with their
163    attributes in the following order: url, header size, body size, status,
164    entity version, date, expiry, request time, response time, body present,
165    head request.</dd>
166    </dl>

170<h2><a name="delete" id="delete">Deleting a specific URL</a></h2>
171    <p>If <code class="program"><a href="../programs/htcacheclean.html">htcacheclean</a></code> is passed one or more URLs, each URL will
172    be deleted from the cache. If multiple variants of an URL exists, all
173    variants would be deleted.</p>
174
175    <p>When a reverse proxied URL is to be deleted, the effective URL is
176    constructed from the <strong>Host</strong> header, the
177    <strong>port</strong>, the <strong>path</strong> and the
178    <strong>query</strong>. Note the '?' in the URL must always be specified
179    explicitly, whether a query string is present or not. For example, an
180    attempt to delete the path <strong>/</strong> from the server
181    <strong>localhost</strong>, the URL to delete would be
182    <strong>http://localhost:80/?</strong>.</p>
183

185<div class="section">
186<h2><a name="list" id="list">Listing URLs in the Cache</a></h2>
187    <p>By passing the <code>-a</code> or <code>-A</code> options to
188    <code class="program"><a href="../programs/htcacheclean.html">htcacheclean</a></code>, the URLs within the cache will be listed
189    as they are found, one URL per line. The <code>-A</code> option
190    dumps the full cache entry after the URL, with fields in the
191    following order:</p>
192
193    <dl>
194        <dt>url</dt><dd>The URL of the entry.</dd>
195        <dt>header size</dt><dd>The size of the header in bytes.</dd>
196        <dt>body size</dt><dd>The size of the body in bytes.</dd>
197        <dt>status</dt><dd>Status of the cached response.</dd>
198        <dt>entity version</dt><dd>The number of times this entry has been
199        revalidated without being deleted.</dd>
200        <dt>date</dt><dd>Date of the response.</dd>
201        <dt>expiry</dt><dd>Expiry date of the response.</dd>
202        <dt>request time</dt><dd>Time of the start of the request.</dd>
203        <dt>response time</dt><dd>Time of the end of the request.</dd>
204        <dt>body present</dt><dd>If 0, no body is stored with this request,
205        1 otherwise.</dd>
206        <dt>head request</dt><dd>If 1, the entry contains a cached HEAD
207        request with no body, 0 otherwise.</dd>
208    </dl>

212<h2><a name="exit" id="exit">Exit Status</a></h2>
213    <p><code class="program"><a href="../programs/htcacheclean.html">htcacheclean</a></code> returns a zero status ("true") if all
214    operations were successful, <code>1</code> otherwise. If an URL is
215    specified, and the URL was cached and successfully removed,
216    <code>0</code> is returned, <code>2</code> otherwise. If an error
217    occurred during URL removal, <code>1</code> is returned.</p>
