32
33    <p><code class="program"><a href="../programs/htdbm.html">htdbm</a></code> permet de manipuler des fichiers au format DBM
34    ou sont stockés des nom d'utilisateurs et mots de passe à des fins
35    d'authentification de base des utilisateurs HTTP via le module
36    <code class="module"><a href="../mod/mod_authn_dbm.html">mod_authn_dbm</a></code>. Voir la documentation de
37    <code class="program"><a href="../programs/dbmmanage.html">dbmmanage</a></code> pour plus de détails à propos de ces
38    fichiers DBM.</p>

139<h2><a name="options" id="options">Options</a></h2>
140    <dl>
141    <dt><code>-b</code></dt>
142    <dd>Utilise le mode batch ; en d'autres termes, le mot de passe est
143    extrait de la ligne de commande au lieu d'être demandé à
144    l'opérateur. Cette option doit être utilisée avec la plus grande
145    prudence, car <strong>le mot de passe est visible en clair</strong>
146    dans la ligne de commande. Pour utiliser un script, voir l'option
147    <code>-i</code>.</dd>
148
149    <dt><code>-i</code></dt>
150    <dd>Lit le mot de passe depuis stdin sans vérification (à utiliser
151    dans le cadre d'un script).</dd>
152
153    <dt><code>-c</code></dt>
154    <dd>Crée le <var>fichier-mots-de-passe</var>. Si
155    <var>fichier-mots-de-passe</var> existe déjà, il est réécrit et
156    tronqué. Cette option ne peut pas être combinée avec l'option
157    <code>-n</code>.</dd>
158
159    <dt><code>-n</code></dt>
160    <dd>Affiche les résultats sur la sortie standard et ne met pas à
161    jour la base de données. Cette option modifie la syntaxe de la ligne
162    de commande, car l'argument <var>fichier-mots-de-passe</var> (en
163    général le premier) est omis. Elle ne peut pas être combinée avec
164    l'option <code>-c</code>.</dd>
165
166    <dt><code>-m</code></dt>
167    <dd>Utilise un hachage MD5 pour les mots de passe. Sous Windows
168    et Netware, c'est l'option par défaut..</dd>
169
170    <dt><code>-B</code></dt>
171    <dd>Utilise l'algorythme de hachage bcrypt pour les mots de
172    passe. C'est un algorythme actuellement considéré comme sûr.</dd>
173
174    <dt><code>-C</code></dt>
175    <dd>Ce drapeau n'est autorisé qu'en conjonction avec le drapeau
176    <code>-B</code> (hachage bcrypt). Il permet de définir la durée
177    de traitement pour l'algorythme de chiffrement bcrypt (plus elle est
178    longue, plus la sécurité est élevée, mais la rapidité est diminuée
179    d'autant) ; la valeur par défaut est 5, les valeurs valides vont de
180    4 à 31.</dd>
181
182
183    <dt><code>-d</code></dt>
184    <dd>Utilise un hachage <code>crypt()</code> pour les mots de
185    passe. C'est l'option par défaut sur toutes les plates-formes, sauf
186    Windows et Netware. Bien que <code class="program"><a href="../programs/htdbm.html">htdbm</a></code> supporte ce
187    chiffrement sur toutes les plates-formes, il n'est pas supporté par
188    le serveur <code class="program"><a href="../programs/httpd.html">httpd</a></code> sous Windows et Netware. Cet
189    algorythme est considéré comme <strong>non sûr</strong> selon les
190    standards actuels.</dd>
191
192    <dt><code>-s</code></dt>
193    <dd>Utilise le hachage SHA pour les mots de passe. Facilite la
194    migration vers/depuis les serveurs Netscape qui utilisent le format
195    LDAP Directory Interchange (ldif). Cet
196    algorythme est considéré comme <strong>non sûr</strong> selon les
197    standards actuels.</dd>
198
199    <dt><code>-p</code></dt>
200    <dd>Utilise des mots de passe au format texte en clair. Bien que
201    <code class="program"><a href="../programs/htdbm.html">htdbm</a></code> supporte ce format sur toutes les plates-formes,
202    le démon <code class="program"><a href="../programs/httpd.html">httpd</a></code> n'accepte les mots de passe au
203    format texte en clair que sous Windows et Netware.</dd>
204
205    <dt><code>-l</code></dt>
206    <dd>Affiche chaque nom d'utilisateur de la base de données
207    accompagné de son commentaire sur la sortie standard.</dd>
208
209    <dt><code>-v</code></dt>
210    <dd>Vérifie une association nom d'utilisateur/mot de passe. Le
211    programme affichera un message indiquant si le mot de passe fourni
212    est valide. Si le mot de passe n'est pas valide, le programme
213    s'arrête et renvoie un code d'erreur 3.</dd>
214
215    <dt><code>-x</code></dt>
216    <dd>Supprime l'utilisateur. Si le nom d'utilisateur existe dans le
217    fichier DBM spécifié, il sera supprimé.</dd>
218
219    <dt><code>-t</code></dt>
220    <dd>Interprète le dernier paramètre en tant que commentaire. Avec
221    cette option, il est possible d'ajouter une chaîne supplémentaire à
222    la fin de la ligne de commande ; le contenu de cette chaîne sera
223    stocké dans la base de données dans le champ "Comment" associé au
224    nom d'utilisateur spécifié.</dd>
225
226    <dt><code><var>nom-fichier</var></code></dt>
227    <dd>Le nom du fichier au format DBM en général sans l'extension
228    <code>.db</code>, <code>.pag</code>, ou <code>.dir</code>. Avec
229    l'option <code>-c</code>, le fichier DBM est mis à jour s'il existe
230    ou créé dans le cas contraire.</dd>
231
232    <dt><code><var>nom-utilisateur</var></code></dt>
233    <dd>Le nom d'utilisateur à créer ou mettre à jour dans le
234    <var>fichier-mots-de-passe</var>. Si <var>nom-utilisateur</var>
235    n'existe pas dans ce fichier, une entrée est ajoutée. S'il existe,
236    son mot de passe est modifié.</dd>
237
238    <dt><code><var>mot-de-passe</var></code></dt>
239    <dd>Le mot de passe en clair destiné à être haché et stocké dans
240    le fichier DBM. Ne s'utilise qu'avec l'option <code>-b</code>.</dd>
241
242    <dt><code>-T<var>DBTYPE</var></code></dt>
243    <dd>Type de fichier DBM (SDBM, GDBM, DB, ou "default").</dd>
244    </dl>

247<h2><a name="bugs" id="bugs">Bugs</a></h2>
248    <p>Vous devez garder à l'esprit qu'il existe de nombreux formats de
249    fichiers DBM différents, et que selon toute vraisemblance, des
250    bibliothèques pour plus d'un format sont présentes sur votre
251    système. Les trois exemples de base sont SDBM, NDBM, le projet GNU
252    GDBM, et Berkeley/Sleepycat DB 2/3/4. Malheureusement, toutes ces
253    bibliothèques
254    utilisent des formats de fichiers différents, et vous devez vous
255    assurer que le format de fichier utilisé par <var>nom-fichier</var>
256    correspond au format attendu par <code class="program"><a href="../programs/htdbm.html">htdbm</a></code>.
257    Actuellement, <code class="program"><a href="../programs/htdbm.html">htdbm</a></code> n'a aucun moyen de savoir à
258    quel type de fichier DBM il a à faire. S'il est utilisé avec un
259    format inapproprié, il ne renverra rien, ou pourra créer un fichier
260    DBM différent avec un nom différent, ou au pire, va corrompre le
261    fichier DBM si vous avez tenté de le modifier.</p>
262
263    <p>Vous pouvez utiliser le programme <code>file</code> fourni par la
264    plupart des systèmes Unix pour déterminer le format d'un fichier
265    DBM.</p>

268<h2><a name="exit" id="exit">Valeur renvoyée</a></h2>
269    <p><code class="program"><a href="../programs/htdbm.html">htdbm</a></code> renvoie 0 ("true") si les nom d'utilisateur et
270    mot de passe ont été créés ou mis à jour avec succès dans le fichier
271    DBM. <code class="program"><a href="../programs/htdbm.html">htdbm</a></code> renvoie <code>1</code> s'il a rencontré un
272    problème d'accès aux fichiers, <code>2</code> si la ligne de
273    commande comportait une erreur de syntaxe, <code>3</code> si le mot
274    de passe a été fourni interactivement et s'il est invalide pour
275    l'entrée considérée, <code>4</code> si l'opération a été
276    interrompue, <code>5</code> si une valeur est trop longue (nom
277    utilisateur, nom fichier, mot de passe, ou l'enregistrement après
278    son élaboration), <code>6</code> si le nom d'utilisateur contient
279    des caractères illégaux (voir la section <a href="#restrictions">Restrictions</a>), et <code>7</code> si le
280    fichier n'est pas un fichier de mots de passe DBM valide.</p>

283<h2><a name="examples" id="examples">Exemples</a></h2>
284    <div class="example"><p><code>
285      htdbm /usr/local/etc/apache/.utilisateurs-htdbm jsmith
286    </code></p></div>
287
288    <p>Ajoute ou modifie le mot de passe de l'utilisateur
289    <code>jsmith</code>. Le mot de passe est demandé à l'opérateur. Sous
290    Windows, le mot de passe sera haché en utilisant l'algorithme MD5
291    Apache modifié ; dans les autres cas, c'est la routine
292    <code>crypt()</code> du système qui sera utilisée. Si le fichier
293    n'existe pas, <code class="program"><a href="../programs/htdbm.html">htdbm</a></code> s'arrêtera et renverra une
294    erreur.</p>
295
296    <div class="example"><p><code>
297      htdbm -c /home/doe/public_html/.htdbm jane
298    </code></p></div>
299
300    <p>Crée un nouveau fichier et y enregistre une entrée pour
301    l'utilisateur <code>jane</code>. Le mot de passe est demandé à
302    l'opérateur. Si le fichier existe et ne peut pas être lu, ou ne peut
303    pas être écrit, il ne sera pas modifié et
304    <code class="program"><a href="../programs/htdbm.html">htdbm</a></code> affichera un message et renverra un code
305    d'erreur.</p>
306
307    <div class="example"><p><code>
308      htdbm -mb /usr/web/.htdbm-tous jones Pwd4Steve
309    </code></p></div>
310
311    <p>Chiffre le mot de passe entré avec la ligne de commande
312    (<code>Pwd4Steve</code>) à l'aide de l'algorithme MD5, et
313    l'enregistre dans le fichier spécifié.</p>

316<h2><a name="security" id="security">Considérations à propos de sécurité</a></h2>
317    <p>Les fichiers de mots de passe Web tels que ceux que gère
318    <code class="program"><a href="../programs/htdbm.html">htdbm</a></code> ne doivent <em>pas</em> être stockés dans
319    l'espace d'URI du serveur Web -- en d'autres termes, il ne doit pas
320    être possible d'y accéder à l'aide d'un navigateur.</p>
321
322    <p>L'utilisation de l'option <code>-b</code> est déconseillée, car
323    lorsqu'il est utilisé, le mot de passe apparaît en clair sur la
324    ligne de commande.</p>
325
326    <p>Notez que lorsque vous utilisez l'algorythme
327    <code>crypt()</code>, seuls les 8 premiers caractères du mot de
328    passe sont pris en compte. Si le mot de passe fourni est plus long,
329    les caractères supplémentaires seront ignorés sans avertissement.</p>
330
331    <p>L'algorythme de hachage SHA ne permet pas de spécifier une valeur
332    d'initialisation pour la génération de nombres aléatoires (salting)
333    : un mot de passe donné ne possède ainsi qu'une seule représentation
334    hachée. Les algorythmes <code>crypt()</code> et MD5 permettent quant à
335    eux des représentations hachées multiples en acceptant comme
336    paramètre une chaîne d'initialisation (salt), rendant les attaques à
337    base de dictionnaires contre les mots de passe plus difficiles.</p>
338
339    <p>Les algorythmes SHA et <code>crypt()</code> sont considérés comme
340    non sûrs selon les standards actuels.</p>

342<div class="section">
343<h2><a name="restrictions" id="restrictions">Restrictions</a></h2>
344    <p>Sur la plate-forme Windows, les mots de passe hachés avec
345    <code class="program"><a href="../programs/htdbm.html">htdbm</a></code> ont une taille limitée à <code>255</code>
346    caractères. Si le mot de passe fourni est plus long, il sera tronqué
347    à 255 caractères.</p>
348
349    <p>L'algorithme MD5 utilisé par <code class="program"><a href="../programs/htdbm.html">htdbm</a></code> est spécifique à
350    Apache ; les mots de passe hachés en utilisant cet algorithme
351    seront inutilisables sur d'autres serveurs Web.</p>
352
353    <p>Les noms d'utilisateurs ont une taille limitée à <code>255</code>
354    octets et ne doivent pas contenir de caractère <code>:</code>.</p>
