34     <p><code class="program"><a href="../programs/apachectl.html">apachectl</a></code> est un frontal pour le serveur HTTP
35     Apache. Il a été conçu pour aider l'administrateur à contrôler le
36     fonctionnement du démon Apache <code class="program"><a href="../programs/httpd.html">httpd</a></code>.</p>
37
38     <p>Le script <code class="program"><a href="../programs/apachectl.html">apachectl</a></code> possède deux modes de
39     fonctionnement. Il peut fonctionner en tant que simple frontal
40     de la commande <code class="program"><a href="../programs/httpd.html">httpd</a></code> et ne fait alors que
41     définir toute variable d'environnement nécessaire, puis invoque
42     <code class="program"><a href="../programs/httpd.html">httpd</a></code> en lui passant tout argument de ligne de
43     commande souhaité. Il peut aussi fonctionner en tant que script
44     d'initialisation SysV n'acceptant qu'un seul argument tel que
45     <code>start</code>, <code>restart</code> et <code>stop</code>, et
46     traduisant ce dernier en signaux appropriés pour le démon
47     <code class="program"><a href="../programs/httpd.html">httpd</a></code>.</p>
48
49     <p>Si votre installation d'Apache utilise des chemins non
50     standards, vous devrez éditer le script <code class="program"><a href="../programs/apachectl.html">apachectl</a></code> afin
51     de définir les chemins appropriés pour le binaire
52     <code class="program"><a href="../programs/httpd.html">httpd</a></code>. Vous pouvez aussi spécifier tout argument
53     de ligne de commande de <code class="program"><a href="../programs/httpd.html">httpd</a></code> nécessaire. Voir
54     les commentaires dans le script pour plus de détails.</p>
55
56     <p>Le script <code class="program"><a href="../programs/apachectl.html">apachectl</a></code> renvoie une valeur égale à 0 en
57     cas de succès, et une valeur supérieure à 0 en cas de problème.
58     Voir les commentaires dans le script pour plus de détails.</p>

67<h2><a name="synopsis" id="synopsis">Résumé</a></h2>
68
69<p>En mode frontal (pass-through), <code class="program"><a href="../programs/apachectl.html">apachectl</a></code> peut spécifier
70tous les arguments qu'accepte le binaire <code class="program"><a href="../programs/httpd.html">httpd</a></code>.</p>
71
72<p><code><strong>apachectl</strong> [ <var>argument-httpd</var> ]</code></p>
73
74<p>En mode script d'initialisation SysV, <code class="program"><a href="../programs/apachectl.html">apachectl</a></code>
75n'accepte qu'un seul des arguments définis ci-dessous.</p>
76
77<p><code><strong>apachectl</strong> <var>commande</var></code></p>
78

81<h2><a name="options" id="options">Options</a></h2>
82
83<p>Seules les options du style initialisation SysV sont décrites ici.
84Les autres arguments sont décrits dans la page de manuel de
85<code class="program"><a href="../programs/httpd.html">httpd</a></code>.</p>
86
87<dl>
88
89<dt><code>start</code></dt>
90
91<dd>Démarre le démon Apache <code class="program"><a href="../programs/httpd.html">httpd</a></code>. Renvoie une erreur
92s'il est déjà en cours d'exécution. Équivalent à <code>apachectl -k
93start</code>.</dd>
94
95<dt><code>stop</code></dt>
96
97<dd>Arrête le démon Apache <code class="program"><a href="../programs/httpd.html">httpd</a></code>. Équivalent à
98<code>apachectl -k stop</code>.</dd>
99
100<dt><code>restart</code></dt>
101
102<dd>Redémarre le démon Apache <code class="program"><a href="../programs/httpd.html">httpd</a></code>. Si le démon
103n'est pas en cours d'exécution, il est démarré. Cette option vérifie
104automatiquement les fichiers de configuration (de la même manière que
105l'option <code>configtest</code> ) avant de lancer le redémarrage, afin
106d'être sûr que le fonctionnement du démon ne sera pas compromis.
107Equivalent à <code>apachectl -k restart</code>.</dd>
108
109<dt><code>fullstatus</code></dt>
110
111<dd>Affiche le rapport d'état complet du module
112<code class="module"><a href="../mod/mod_status.html">mod_status</a></code>. Pour que ceci fonctionne,
113<code class="module"><a href="../mod/mod_status.html">mod_status</a></code> doit être activé dans votre serveur et vous
114devez disposer d'un navigateur en mode texte tel que <code>lynx</code>
115sur votre système. L'URL utilisée pour accéder au rapport d'état peut
116être modifiée en définissant la variable <code>STATUSURL</code> dans le
117script.</dd>
118
119<dt><code>status</code></dt>
120
121<dd>Affiche un rapport d'état succinct. Similaire à l'option
122<code>fullstatus</code>, excepté que la liste des requêtes en cours de
123traitement est omise.</dd>
124
125<dt><code>graceful</code></dt>
126
127<dd>Redémarre le démon Apache <code class="program"><a href="../programs/httpd.html">httpd</a></code> en douceur. Si le
128démon n'est pas en cours d'exécution, il est démarré. À la différence
129d'un redémarrage normal, les connexions en cours ne sont pas fermées.
130Comme effet de bord, les anciens fichiers journaux ne seront pas fermés
131immédiatement. Cela signifie que si l'on utilise un script de rotation
132des journaux, un délai suffisant sera nécessaire afin d'être sûr que les
133fichiers journaux seront bien fermés avant leur traitement par le script
134de rotation. Cette option vérifie
135automatiquement les fichiers de configuration (de la même manière que
136l'option <code>configtest</code> ) avant de lancer le redémarrage, afin
137d'être sûr que le fonctionnement du démon ne sera pas compromis.
138Équivalent à <code>apachectl -k graceful</code>.</dd>
139
140<dt><code>graceful-stop</code></dt>
141
142<dd>Arrête le démon Apache <code class="program"><a href="../programs/httpd.html">httpd</a></code> en douceur. À la
143différence d'un arrêt normal, les connexions en cours ne sont pas
144fermées. Comme effet de bord, les anciens fichiers journaux ne seront
145pas fermés immédiatement. Équivalent à <code>apachectl -k
146graceful-stop</code>.</dd>
147
148<dt><code>configtest</code></dt>
149
150<dd>Effectue une vérification de la syntaxe du fichier de configuration.
151Avec cette option, le script parcourt le fichier de configuration et
152renvoie soit <code>Syntax Ok</code>, soit des informations détaillées à
153propos des éventuelles erreurs de syntaxe. Equivalent à <code>apachectl
154-t</code>.</dd>
155
156</dl>
157
158<p>Les options suivantes étaient disponibles dans les anciennes versions
159et ont été supprimées.</p>
160
161<dl>
162
163<dt><code>startssl</code></dt>
164
165<dd>Pour démarrer <code class="program"><a href="../programs/httpd.html">httpd</a></code> avec le support SSL, vous
166devez éditer votre fichier de configuration et y inclure les
167directives appropriées, puis utiliser la commande de démarrage normale
168<code>apachectl start</code>.</dd>
169
170</dl>
