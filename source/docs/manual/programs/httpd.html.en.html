32     <p><code class="program"><a href="../programs/httpd.html">httpd</a></code> is the Apache HyperText Transfer Protocol
33     (HTTP) server program.  It is designed to be run as a standalone
34     daemon process. When used like this it will create a pool of
35     child processes or threads to handle requests.</p>
36
37     <p>In general, <code class="program"><a href="../programs/httpd.html">httpd</a></code> should not be invoked directly,
38     but rather should be invoked via <code class="program"><a href="../programs/apachectl.html">apachectl</a></code> on Unix-based systems or <a href="../platform/windows.html#winsvc">as a service on Windows NT,
39     2000 and XP</a> and <a href="../platform/windows.html#wincons">as
40     a console application on Windows 9x and ME</a>.</p>

69<h2><a name="options" id="options">Options</a></h2>
70
71<dl>
72<dt><code>-d <var>serverroot</var></code></dt>
73
74<dd>Set the initial value for the <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> directive to
75<var>serverroot</var>.  This can be overridden by the ServerRoot
76directive in the configuration file. The default is
77<code>/usr/local/apache2</code>.</dd>
78
79<dt><code>-f <var>config</var></code></dt>
80
81<dd>Uses the directives in the file <var>config</var> on startup. If
82<var>config</var> does not begin with a /, then it is taken to be a
83path relative to the <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>. The default is
84<code>conf/httpd.conf</code>.</dd>
85
86<dt><code>-k <code>start|restart|graceful|stop|graceful-stop</code></code></dt>
87
88<dd>Signals <code class="program"><a href="../programs/httpd.html">httpd</a></code> to start, restart, or stop.  See <a href="../stopping.html">Stopping Apache httpd</a> for more information.</dd>
89
90<dt><code>-C <var>directive</var></code></dt>
91
92<dd>Process the configuration <var>directive</var> before reading
93config files.</dd>
94
95<dt><code>-c <var>directive</var></code></dt>
96
97<dd>Process the configuration <var>directive</var> after reading config
98files.</dd>
99
100
101<dt><code>-D <var>parameter</var></code></dt>
102
103<dd>Sets a configuration <var>parameter </var>which can be used with
104<code class="directive"><a href="../mod/core.html#ifdefine">&lt;IfDefine&gt;</a></code> sections
105in the configuration files to conditionally skip or process commands
106at server startup and restart. Also can be used to set certain
107less-common startup parameters including <code>-DNO_DETACH</code>
108(prevent the parent from forking) and <code>-DFOREGROUND</code>
109(prevent the parent from calling <code>setsid()</code> et al).</dd>
110
111<dt><code>-e <var>level</var></code></dt>
112
113<dd>Sets the <code class="directive"><a href="../mod/core.html#loglevel">LogLevel</a></code> to
114<var>level</var> during server startup.  This is useful for
115temporarily increasing the verbosity of the error messages to find
116problems during startup.</dd>
117
118<dt><code>-E <var>file</var></code></dt>
119
120<dd>Send error messages during server startup to <var>file</var>.</dd>
121
122<dt><code>-h</code></dt>
123
124<dd>Output a short summary of available command line options.</dd>
125
126<dt><code>-l</code></dt>
127
128<dd>Output a list of modules compiled into the server.  This will
129<strong>not</strong> list dynamically loaded modules included using
130the <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code> directive.</dd>
131
132<dt><code>-L</code></dt>
133
134<dd>Output a list of directives provided by static modules, together with expected arguments and
135places where the directive is valid. Directives provided by shared modules are not listed.</dd>
136
137<dt><code>-M</code></dt>
138
139<dd>Dump a list of loaded Static and Shared Modules.</dd>
140
141<dt><code>-S</code></dt>
142
143<dd>Show the settings as parsed from the config file (currently only
144shows the virtualhost settings).</dd>
145
146<dt><code>-T</code> (Available in 2.3.8 and later)</dt>
147
148<dd>Skip document root check at startup/restart.</dd>
149
150<dt><code>-t</code></dt>
151
152<dd>Run syntax tests for configuration files only.  The program
153immediately exits after these syntax parsing tests with either a return code
154of 0 (Syntax OK) or return code not equal to 0 (Syntax Error).  If -D
155<var>DUMP</var>_<var>VHOSTS </var>is also set, details of the virtual host
156configuration will be printed. If -D <var>DUMP</var>_<var>MODULES </var> is
157set, all loaded modules will be printed.</dd>
158
159<dt><code>-v</code></dt>
160
161<dd>Print the version of <code class="program"><a href="../programs/httpd.html">httpd</a></code>, and then exit.</dd>
162
163<dt><code>-V</code></dt>
164
165<dd>Print the version and build parameters of <code class="program"><a href="../programs/httpd.html">httpd</a></code>, and
166then exit.</dd>
167
168<dt><code>-X</code></dt>
169
170<dd>Run httpd in debug mode.  Only one worker will be started and the
171server will not detach from the console.</dd>
172
173</dl>
174
175<p>The following arguments are available only on the <a href="../platform/windows.html">Windows platform</a>:</p>
176
177<dl>
178
179<dt><code>-k install|config|uninstall</code></dt>
180
181<dd>Install Apache httpd as a Windows NT service; change startup options for
182the Apache httpd service; and uninstall the Apache httpd service.</dd>
183
184<dt><code>-n <var>name</var></code></dt>
185
186<dd>The <var>name</var> of the Apache httpd service to signal.</dd>
187
188<dt><code>-w</code></dt>
189
190<dd>Keep the console window open on error so that the error message can
191be read.</dd>
192
193</dl>
