34    <p><code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> permet de créer et de maintenir les
35    fichiers textes où sont stockés les noms d'utilisateurs et mots de
36    passe pour l'authentification de base des utilisateurs HTTP. Si
37    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> rencontre un problème d'accès à un fichier,
38    que ce soit pour écrire dans le fichier de sortie, ou pour lire le
39    fichier d'entrée dans le but de le mettre à jour, il renvoie un code
40    d'erreur et n'effectue aucune modification.</p>
41
42    <p>Il est possible de limiter l'accès aux ressources du serveur HTTP
43    Apache aux seuls utilisateurs présents dans les fichiers créés par
44    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code>. Ce programme ne sait gérer les noms
45    d'utilisateurs et mots de passe que s'ils sont stockés dans des
46    fichiers textes. Il peut cependant hacher et afficher les mots de
47    passe à des fins d'utilisation dans d'autres types de bases de
48    données. Pour utiliser une base de données DBM, voir le programme
49    <code class="program"><a href="../programs/dbmmanage.html">dbmmanage</a></code> ou <code class="program"><a href="../programs/htdbm.html">htdbm</a></code>.</p>
50
51    <p><code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> hache les mots de passe en utilisant bcrypt, une
52    version de MD5 modifiée pour Apache, SHA-1 ou la routine système
53    <code>crypt()</code>. Les hachages de type SHA-2 (SHA-256 and SHA-512) sont
54    pris en charge pour <code>crypt()</code>. Les fichiers gérés par
55    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> peuvent contenir un mélange de différents types de
56    codage des mots de passe&nbsp;; par exemple, certaines entrées utilisateur
57    pourront comporter des mots de passe hachés avec bcrypt ou MD5, alors que
58    d’autres dans le même fichier pourront comporter des mots de passe hachés
59    avec <code>crypt()</code>.</p>    
60
61    <p>Cette page de manuel ne décrit que les arguments de la ligne de
62    commande. Pour plus de détails à propos des directives nécessaires à
63    la configuration de l'authentification des utilisateurs dans
64    <code class="program"><a href="../programs/httpd.html">httpd</a></code>, voir le manuel Apache qui est fourni avec
65    la distribution ou peut être consulté à <a href="http://httpd.apache.org">http://httpd.apache.org/</a>.</p>

133<h2><a name="options" id="options">Options</a></h2>
134    <dl>
135    <dt><code>-b</code></dt>
136    <dd>Utilise le mode batch ; c'est à dire, extrait le mot de passe de
137    la ligne de commande au lieu de le demander à l'opérateur. Cette
138    option doit être utilisée avec la plus grande prudence, car
139    <strong>le mot de passe est visible en clair</strong> dans la ligne
140    de commande. Pour utiliser un script, voir l'option
141    <code>-i</code>.
142    Disponible à partir de la version 2.4.4 du serveur HTTP Apache.</dd>
143
144    <dt><code>-i</code></dt>
145    <dd>Lit le mot de passe depuis stdin sans vérification (à utiliser
146    dans les scripts).</dd>
147
148    <dt><code>-c</code></dt>
149    <dd>Crée le <var>fichier-mots-de-passe</var>. Si
150    <var>fichier-mots-de-passe</var> existe déjà, il est réécrit et
151    tronqué. Cette option ne peut pas être combinée avec l'option
152    <code>-n</code>.</dd>
153
154    <dt><code>-n</code></dt>
155    <dd>Affiche le résultat du traitement sur la sortie standard au lieu
156    de mettre à jour le fichier. Ceci peut s'avérer utile pour générer
157    des enregistrements de mots de passe qu'Apache pourra utiliser à des
158    fins d'inclusion dans des fichiers de données au format autre que
159    texte. Cette option modifie la syntaxe de la ligne de commande, car
160    l'argument <var>fichier-mots-de-passe</var> (en général le premier)
161    est omis. Elle ne peut pas être combinée avec l'option
162    <code>-c</code> option.</dd>
163
164    <dt><code>-m</code></dt>
165    <dd>Utilise le hachage MD5 pour les mots de passe. C'est le
166    comportement par défaut (depuis la version 2.2.18).</dd>
167
168    <dt><code>-2</code></dt>
169    <dd>Utilise les hachages SHA-256 basés sur <code>crypt()</code> pour les
170    mots de passe. Pris en charge sur la plupart des plateformes de style Unix.</dd>
171
172    <dt><code>-5</code></dt>
173    <dd>Utilise les hachages SHA-512 basés sur <code>crypt()</code> pour les
174    mots de passe. Pris en charge sur la plupart des plateformes de style Unix.</dd>
175
176    <dt><code>-B</code></dt>
177    <dd>Utilise bcrypt pour hacher les mots de passe. c'est un
178    algorythme de chiffrement actuellement considéré comme sûr.</dd>
179
180    <dt><code>-C</code></dt>
181    <dd>Ce drapeau n'est autorisé qu'en conjonction avec le drapeau
182    <code>-B</code> (hachage bcrypt). Il permet de définir la durée
183    de traitement pour l'algorytme bcrypt (plus elle est longue,
184    meilleure sera la sécurité, mais inférieure la rapidité). La valeur
185    par défaut est 5 et les valeurs autorisées vont de 4 à 17.</dd>
186
187    <dt><code>-r</code></dt>
188    <dd>Ce drapeau n’est autorisé qu’en combinaison avec les drapeaux
189    <code>-2</code> ou <code>-5</code>. Il permet de définir le nombre de passes
190    de hachage utilisé pour les algorithmes SHA-2 (un nombre élevé améliore la
191    sécurité mais le traitement est plus lent&nbsp;; la valeur par défaut est 5000.</dd>
192
193    <dt><code>-d</code></dt>
194    <dd>Utilise le hachage <code>crypt()</code> pour les mots de
195    passe. Cette option n'est pas supportée par le
196    serveur <code class="program"><a href="../programs/httpd.html">httpd</a></code> sous Windows ou Netware. Cet
197    algorithme limite la longueur des mots de passe à 8 caractères ; il
198    est considéré comme <strong>non sur</strong> du point de vue des
199    standards actuels. C'était l'algorithme par défaut jusqu'à la
200    version 2.2.17.</dd>
201
202    <dt><code>-s</code></dt>
203    <dd>Utilise le hachage SHA-1 (160-bit) pour les mots de passe. Facilite la
204    migration vers/depuis les serveurs Netscape qui utilisent le format
205    LDAP Directory Interchange (ldif). Cet algorithme
206    est considéré comme <strong>non sûr</strong> du point de vue des
207    normes actuelles.</dd>
208
209    <dt><code>-p</code></dt>
210    <dd>Enregistre les mots de passe en clair. Bien que
211    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> supporte la création des mots de passe en
212    clair sur toutes les plates-formes, le démon
213    <code class="program"><a href="../programs/httpd.html">httpd</a></code> n'accepte les mots de passe en clair que
214    sous Windows et Netware.</dd>
215
216    <dt><code>-D</code></dt>
217    <dd>Supprime un utilisateur, sous réserve qu'il existe dans le
218    fichier spécifié par htpasswd.</dd>
219
220    <dt><code>-v</code></dt>
221    <dd>Vérifie si le mot de passe fourni correspond au mot de passe de
222    l'utilisateur enregistré dans le fichier de mots de passe spécifié.
223    Disponible à partir de la version 2.4.5 du serveur HTTP Apache.</dd>
224
225    <dt><code><var>fichier-mots-de-passe</var></code></dt>
226    <dd>Le nom du fichier contenant les noms d'utilisateurs et mots de
227    passe. Avec l'option <code>-c</code>, le fichier est créé s'il
228    n'existe pas, ou réécrit et tronqué s'il existe déjà.</dd>
229
230    <dt><code><var>nom-utilisateur</var></code></dt>
231    <dd>Le nom d'utilisateur à créer ou mettre à jour dans le
232    <var>fichier-mots-de-passe</var>. Si <var>nom-utilisateur</var>
233    n'existe pas, une nouvelle entrée est ajoutée. Dans le cas
234    contraire, le mot de passe est modifié.</dd>
235
236    <dt><code><var>mot-de-passe</var></code></dt>
237    <dd>Le mot de passe en clair et destiné à être haché puis stocké
238    dans le fichier. Cet argument ne s'utilise qu'avec l'option
239    <code>-b</code>.</dd>
240    </dl>

243<h2><a name="exit" id="exit">Valeur renvoyée</a></h2>
244    <p><code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> renvoie 0 ("true") si le nom d'utilisateur
245    et le mot de passe ont été enregistrés ou mis à jour avec succès
246    dans le <var>fichier-mots-de-passe</var>. <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code>
247    renvoie <code>1</code> s'il a rencontré un problème d'accès aux
248    fichiers, <code>2</code> si la ligne de commande comportait une
249    erreur de syntaxe, <code>3</code> si le mot de passe entré
250    interactivement ne correspondait pas au nom d'utilisateur,
251    <code>4</code> si l'opération a été interrompue, <code>5</code> si
252    une valeur était trop longue (nom-utilisateur, nom-fichier,
253    mot-de-passe, ou l'enregistrement résultant), <code>6</code> si le
254    nom d'utilisateur contenait des caractères illégaux (voir la section
255    <a href="#restrictions">Restrictions</a>), et <code>7</code> si le
256    fichier spécifié n'était pas un fichier de mots de passe
257    valide.</p>

260<h2><a name="examples" id="examples">Exemples</a></h2>
261    <div class="example"><p><code>
262      htpasswd /usr/local/etc/apache/.utilisateurs-htpasswd jsmith
263    </code></p></div>
264
265    <p>Ajoute ou modifie le mot de passe de l'utilisateur
266    <code>jsmith</code>. Le mot de passe est demandé à l'opérateur. Le
267    mot de passe sera haché en utilisant l'algorithme MD5
268    modifié pour Apache. Si le fichier spécifié
269    n'existe pas, <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> renverra un code d'erreur.</p>
270
271    <div class="example"><p><code>
272      htpasswd -c /home/doe/public_html/.htpasswd jane
273    </code></p></div>
274
275    <p>Crée un nouveau fichier de mots de passe et y enregistre une
276    entrée pour l'utilisateur <code>jane</code>. Le mot de passe est
277    demandé à l'opérateur. Si le fichier existe et ne peut être ni lu ni
278    écrit, il n'est pas modifié et <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> affichera un
279    message et renverra un code d'erreur.</p>
280
281    <div class="example"><p><code>
282      htpasswd -db /usr/web/.htpasswd-tous jones Pwd4Steve
283    </code></p></div>
284
285    <p>Chiffre le mot de passe spécifié dans la ligne de commande
286    (<code>Pwd4Steve</code>) en utilisant l'algorithme
287    <code>crypt()</code>, et le stocke dans le fichier spécifié.</p>

290<h2><a name="security" id="security">Considérations à propos de sécurité</a></h2>
291    <p>Les fichiers de mots de passe Web comme ceux que gère
292    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> ne doivent <em>pas</em> être situés dans
293    l'espace d'URI du serveur Web -- en d'autres termes, il ne doit pas
294    être possible d'y accéder à partir d'un navigateur.</p>
295
296    <p>En tant qu'exécutable setuid, ce programme n'est pas sûr, et il
297    ne faut par conséquent <em>pas</em> lui attribuer de permissions
298    setuid.</p>
299
300    <p>L'utilisation de l'option <code>-b</code> est déconseillée, car
301    avec elle, les mots de passe apparaissent en clair sur la ligne de
302    commande.</p>
303
304    <p>Notez qu'avec l'algorithme <code>crypt()</code>, seuls les huit
305    premiers caractères du mot de passe spécifié sont pris en compte. Si
306    le mot de passe spécifié est plus long, les caractères
307    supplémentaires sont ignorés.</p>
308
309    <p>Le format de hachage SHA-1 n'utilise pas d'amorçage aléatoire
310    (salting) : à un mot de passe donné correspond une seule
311    représentation hachée. Les formats <code>crypt()</code> et MD5
312    permutent la représentation en la préfixant par une chaîne d'amorce
313    aléatoire, afin de rendre les attaques de mots de passe à base de
314    dictionnaires plus difficiles.</p>
315
316    <p>Les algorithmes de chiffrement SHA-1 et <code>crypt()</code> 
317    sont considérés comme <strong>non surs</strong> du point de vue des
318    standards actuels.</p>
319
320    <p>Les formats de <code>crypt()</code> basés sur SHA-2 (SHA-256 et SHA-512)
321    sont pris en charge sur la plupart des systèmes de style Unix récents et
322    respectent la spécification de <a href="https://www.akkadia.org/drepper/SHA-crypt.txt">https://www.akkadia.org/drepper/SHA-crypt.txt</a>.</p>

325<h2><a name="restrictions" id="restrictions">Restrictions</a></h2>
326    <p>Sur les plates-formes Windows, la taille des mots de passe
327    hachés avec <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> est limitée à <code>255</code>
328    caractères. Les mots de passe dont la taille est supérieure seront
329    tronqués.</p>
330
331    <p>L'algorithme MD5 utilisé par <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> est spécifique
332    à Apache ; les mots de passe hachés en utilisant cet algorithme
333    seront inutilisables sur d'autres serveurs Web.</p>
334
335    <p>La taille des noms d'utilisateurs est limitée à <code>255</code>
336    octets et ceux-ci ne doivent pas contenir de caractère
337    <code>:</code>.</p>
338
339    <p>Le coût en performances de la génération de la valeur de hashage d'un mot
340    de passe bcrypt augmente avec le nombre de passes spécifié par l'option
341    <code>-C</code>. A partir de sa version <code>1.6.0</code>, la bibliothèque
342    <code>apr-util</code> limite le nombre de passes à 17.</p>
