32    <p><code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> is used to create and update the flat-files used to
33    store usernames and password for basic authentication of HTTP users. If
34    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> cannot access a file, such as not being able to write
35    to the output file or not being able to read the file in order to update it,
36    it returns an error status and makes no changes.</p>
37
38    <p>Resources available from the Apache HTTP server can be restricted to
39    just the users listed in the files created by <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code>. This
40    program can only manage usernames and passwords stored in a flat-file. It
41    can hash and display password information for use in other types of data
42    stores, though. To use a DBM database see <code class="program"><a href="../programs/dbmmanage.html">dbmmanage</a></code> or
43    <code class="program"><a href="../programs/htdbm.html">htdbm</a></code>.</p>
44
45    <p><code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> hashes passwords using either bcrypt, a
46    version of MD5 modified for Apache, SHA-1, or the system's
47    <code>crypt()</code> routine. SHA-2-based hashes (SHA-256 and
48    SHA-512) are supported for <code>crypt()</code>.  Files managed by
49    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> may contain a mixture of different encoding
50    types of passwords; some user records may have bcrypt or
51    MD5-hashed passwords while others in the same file may have
52    passwords hashed with <code>crypt()</code>.</p>
53
54    <p>This manual page only lists the command line arguments. For details of
55    the directives necessary to configure user authentication in
56    <code class="program"><a href="../programs/httpd.html">httpd</a></code> see the Apache manual, which is part of the
57    Apache distribution or can be found at <a href="http://httpd.apache.org">http://httpd.apache.org/</a>.</p>

125<h2><a name="options" id="options">Options</a></h2>
126    <dl>
127    <dt><code>-b</code></dt>
128    <dd>Use batch mode; <em>i.e.</em>, get the password from the command line
129    rather than prompting for it. This option should be used with extreme care,
130    since <strong>the password is clearly visible</strong> on the command
131    line. For script use see the <code>-i</code> option.
132    Available in 2.4.4 and later.</dd>
133
134    <dt><code>-i</code></dt>
135    <dd>Read the password from stdin without verification (for script usage).</dd>
136
137    <dt><code>-c</code></dt>
138    <dd>Create the <var>passwdfile</var>. If <var>passwdfile</var> already
139    exists, it is rewritten and truncated. This option cannot be combined with
140    the <code>-n</code> option.</dd>
141
142    <dt><code>-n</code></dt>
143    <dd>Display the results on standard output rather than updating a file.
144    This is useful for generating password records acceptable to Apache for
145    inclusion in non-text data stores. This option changes the syntax of the
146    command line, since the <var>passwdfile</var> argument (usually the first
147    one) is omitted. It cannot be combined with the <code>-c</code> option.</dd>
148
149    <dt><code>-m</code></dt>
150    <dd>Use MD5 hashing for passwords. This is the default (since version
151    2.2.18).</dd>
152
153    <dt><code>-2</code></dt>
154    <dd>Use SHA-256 <code>crypt()</code> based hashes for passwords.  This is
155    supported on most Unix platforms.</dd>
156
157    <dt><code>-5</code></dt>
158    <dd>Use SHA-512 <code>crypt()</code> based hashes for passwords.  This is
159    supported on most Unix platforms.</dd>
160
161    <dt><code>-B</code></dt>
162    <dd>Use bcrypt hashing for passwords. This is currently considered to
163    be very secure.</dd>
164
165    <dt><code>-C</code></dt>
166    <dd>This flag is only allowed in combination with <code>-B</code> (bcrypt
167    hashing). It sets the computing time used for the bcrypt algorithm
168    (higher is more secure but slower, default: 5, valid: 4 to 17).</dd>
169
170    <dt><code>-r</code></dt>
171    <dd>This flag is only allowed in combination with <code>-2</code>
172    or <code>-5</code>. It sets the number of hash rounds used for the
173    SHA-2 algorithms (higher is more secure but slower; the default is
174    5,000).</dd>
175
176    <dt><code>-d</code></dt>
177    <dd>Use <code>crypt()</code> hashing for passwords. This is not
178    supported by the <code class="program"><a href="../programs/httpd.html">httpd</a></code> server on Windows and
179    Netware. This algorithm limits the password length to 8 characters.
180    This algorithm is <strong>insecure</strong> by today's standards.
181    It used to be the default algorithm until version 2.2.17.</dd>
182
183    <dt><code>-s</code></dt>
184    <dd>Use SHA-1 (160-bit) hashing for passwords. Facilitates migration
185    from/to Netscape servers using the LDAP Directory Interchange
186    Format (ldif).  This algorithm is <strong>insecure</strong> by
187    today's standards.</dd>
188
189    <dt><code>-p</code></dt>
190    <dd>Use plaintext passwords. Though <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> will support
191    creation on all platforms, the <code class="program"><a href="../programs/httpd.html">httpd</a></code> daemon will
192    only accept plain text passwords on Windows and Netware.</dd>
193
194    <dt><code>-D</code></dt>
195    <dd>Delete user. If the username exists in the specified htpasswd file, it
196    will be deleted.</dd>
197
198    <dt><code>-v</code></dt>
199    <dd>Verify password. Verify that the given password matches the password
200    of the user stored in the specified htpasswd file.
201    Available in 2.4.5 and later.</dd>
202
203    <dt><code><var>passwdfile</var></code></dt>
204    <dd>Name of the file to contain the user name and password. If
205    <code>-c</code> is given, this file is created if it does not already exist,
206    or rewritten and truncated if it does exist.</dd>
207
208    <dt><code><var>username</var></code></dt>
209    <dd>The username to create or update in <var>passwdfile</var>. If
210    <var>username</var> does not exist in this file, an entry is added. If it
211    does exist, the password is changed.</dd>
212
213    <dt><code><var>password</var></code></dt>
214    <dd>The plaintext password to be hashed and stored  in the file. Only
215    used with the <code>-b</code> flag.</dd>
216    </dl>

219<h2><a name="exit" id="exit">Exit Status</a></h2>
220    <p><code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> returns a zero status ("true") if the username and
221    password have been successfully added or updated in the
222    <var>passwdfile</var>. <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> returns <code>1</code> if it
223    encounters some problem accessing files, <code>2</code> if there was a
224    syntax problem with the command line, <code>3</code> if the password was
225    entered interactively and the verification entry didn't match,
226    <code>4</code> if its operation was interrupted, <code>5</code> if a value
227    is too long (username, filename, password, or final computed record),
228    <code>6</code> if the username contains illegal characters (see the
229    <a href="#restrictions">Restrictions section</a>), and <code>7</code>
230    if the file is not a valid password file.</p>

233<h2><a name="examples" id="examples">Examples</a></h2>
234    <div class="example"><p><code>
235      htpasswd /usr/local/etc/apache/.htpasswd-users jsmith
236    </code></p></div>
237
238    <p>Adds or modifies the password for user <code>jsmith</code>. The user
239    is prompted for the password. The password will be hashed using the
240    modified Apache MD5 algorithm. If the file does not exist,
241    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> will do nothing except return an error.</p>
242
243    <div class="example"><p><code>
244      htpasswd -c /home/doe/public_html/.htpasswd jane
245    </code></p></div>
246
247    <p>Creates a new file and stores a record in it for user <code>jane</code>.
248    The user is prompted for the password. If the file exists and cannot be
249    read, or cannot be written, it is not altered and <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code>
250    will display a message and return an error status.</p>
251
252    <div class="example"><p><code>
253      htpasswd -db /usr/web/.htpasswd-all jones Pwd4Steve
254    </code></p></div>
255
256    <p>Encrypts the password from the command line (<code>Pwd4Steve</code>)
257    using the <code>crypt()</code> algorithm, and stores it in the specified
258    file.</p>

261<h2><a name="security" id="security">Security Considerations</a></h2>
262    <p>Web password files such as those managed by <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> should
263    <em>not</em> be within the Web server's URI space -- that is, they should
264    not be fetchable with a browser.</p>
265
266    <p>This program is not safe as a setuid executable. Do <em>not</em> make it
267    setuid.</p>
268
269    <p>The use of the <code>-b</code> option is discouraged, since when it is
270    used the plaintext password appears on the command line.</p>
271
272    <p>When using the <code>crypt()</code> algorithm, note that only the first
273    8 characters of the password are used  to form the password. If the supplied
274    password is longer, the extra characters will be silently discarded.</p>
275
276    <p>The SHA-1 hashing format does not use salting: for a given
277    password, there is only one hashed representation. The
278    <code>crypt()</code> and MD5 formats permute the representation by
279    prepending a random salt string, to make dictionary attacks
280    against the passwords more difficult.</p>
281
282    <p>The SHA-1 and <code>crypt()</code> formats are insecure by
283    today's standards.</p>
284
285    <p>The SHA-2-based <code>crypt()</code> formats (SHA-256 and
286    SHA-512) are supported on most modern Unix systems, and follow the
287    specification at <a href="https://www.akkadia.org/drepper/SHA-crypt.txt">https://www.akkadia.org/drepper/SHA-crypt.txt</a>.</p>

291<h2><a name="restrictions" id="restrictions">Restrictions</a></h2>
292    <p>On the Windows platform, passwords hashed with
293    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> are limited to no more than <code>255</code>
294    characters in length. Longer passwords will be truncated to 255
295    characters.</p>
296
297    <p>The MD5 algorithm used by <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> is specific to the Apache
298    software; passwords hashed using it will not be usable with other Web
299    servers.</p>
300
301    <p>Usernames are limited to <code>255</code> bytes and may not include the
302    character <code>:</code>.</p>
303
304    <p>The cost of computing a bcrypt password hash value increases
305    with the number of rounds specified by the <code>-C</code> option.
306    The <code>apr-util</code> library enforces a maximum number of
307    rounds of 17 in version <code>1.6.0</code> and later.</p>
