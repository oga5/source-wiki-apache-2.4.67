33    <p><code class="program"><a href="../programs/httxt2dbm.html">httxt2dbm</a></code> permet, à partir d'une entrée au format
34    texte, de générer des fichiers dbm à utiliser dans les directives
35    <code class="directive"><a href="../mod/mod_rewrite.html#rewritemap">RewriteMap</a></code> avec le type
36    de table <code>dbm</code>.
37    </p>
38
39    <p>Si le fichier de sortie existe déjà, il ne sera pas tronqué. Les
40    nouvelles clés seront ajoutées et les clés préexistantes mises à
41    jour.</p>

58<h2><a name="options" id="options">Options</a></h2>
59    <dl>
60    <dt><code>-v</code></dt>
61    <dd>Sortie plus verbeuse</dd>
62
63    <dt><code>-f <var>TYPE_DBM</var></code></dt>
64    <dd>Spécifie le type DBM à utiliser pour le fichier de sortie.
65    S'il n'est pas spécifié, c'est la valeur par défaut de
66    l'<a class="glossarylink" href="../glossary.html#apr" title="voir glossaire">APR</a> qui sera utilisée. Les types disponibles
67    sont :
68    <code>GDBM</code> pour les fichiers GDBM,
69    <code>SDBM</code> pour les fichiers SDBM,
70    <code>DB</code> pour les fichiers DB,
71    <code>NDBM</code> pour les fichiers NDBM,
72    <code>default</code> pour le type DBM par défaut
73    </dd>
74
75    <dt><code>-i <var>TEXTE_SOURCE</var></code></dt>
76    <dd>Le fichier d'entrée à partir duquel le fichier dbm sera créé. Le
77    fichier doit être formaté de façon à ne contenir qu'un seul
78    enregistrement par ligne, de la forme : <code>clé valeur</code>.
79    Voir la documentation de la directive <code class="directive"><a href="../mod/mod_rewrite.html#rewritemap">RewriteMap</a></code> pour plus de détails à
80    propos du format de ce fichier et de sa signification.
81    </dd>
82
83    <dt><code>-o <var>SORTIE_DBM</var></code></dt>
84    <dd>Nom du fichier dbm de sortie.</dd>
85    </dl>
