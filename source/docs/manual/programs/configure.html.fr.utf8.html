31
32    <p>Le script <code class="program"><a href="../programs/configure.html">configure</a></code> permet de configurer
33    l'arborescence des sources afin de compiler et installer le serveur
34    HTTP Apache sur votre plate-forme spécifique. De nombreuses options
35    vous permettent de compiler un serveur correspondant à vos propres
36    besoins.</p>
37
38    <p>Ce script, situé dans le répertoire racine de la distribution des
39    sources, ne concerne que la compilation sur les systèmes Unix et
40    apparentés. Pour les autres plates-formes, voir la <a href="../platform/">documentation spécifique</a> de ces
41    dernières.</p>

49<h2><a name="synopsis" id="synopsis">Résumé</a></h2>
50    <p>Vous devez appeler le script <code class="program"><a href="../programs/configure.html">configure</a></code> depuis le
51    répertoire racine de la distribution.</p>
52
53    <p><code><strong>./configure</strong> [<var>OPTION</var>]...
54    [<var>VARIABLE</var>=<var>VALEUR</var>]...</code></p>
55
56    <p>Pour définir des variables d'environnement (par exemple
57    <code>CC</code>,<code>CFLAGS</code>, etc...), utilisez la clause
58    <code><var>VARIABLE</var>=<var>VALEUR</var></code>. Voir <a href="#env">ci-dessous</a> pour la description de quelques variables
59    usuelles.</p>

71  <h3><a name="configurationoptions" id="configurationoptions">Options de Configuration</a></h3>
72
73    <p>Les options suivantes affectent le comportement du script
74    <code class="program"><a href="../programs/configure.html">configure</a></code>.</p>
75
76    <dl>
77      <dt><code>-C</code></dt>
78      <dt><code>--config-cache</code></dt>
79      <dd>C'est un alias pour <code>--cache-file=config.cache</code></dd>
80
81      <dt><code>--cache-file=<var>FICHIER</var></code></dt>
82      <dd>Les résultats des tests seront mis en cache dans le fichier
83      <var>FICHIER</var>. Cette option est désactivée par défaut.</dd>
84
85      <dt><code>-h</code></dt>
86      <dt><code>--help [short|recursive]</code></dt>
87      <dd>Affichage de l'aide et sortie du script. Avec l'argument
88      <code>short</code>, seules les options spécifiques à ce paquet
89      seront affichées. L'argument <code>recursive</code> permet
90      d'afficher l'aide de tous les paquets inclus.</dd>
91
92      <dt><code>-n</code></dt>
93      <dt><code>--no-create</code></dt>
94      <dd>Le script <code class="program"><a href="../programs/configure.html">configure</a></code> s'exécute normalement, mais
95      ne crée pas les fichiers résultants. Ceci permet de vérifier les
96      résultats des tests avant de générer les fichiers makefile pour la
97      compilation.</dd>
98
99      <dt><code>-q</code></dt>
100      <dt><code>--quiet</code></dt>
101      <dd>Les messages <code>checking ...</code> ne sont pas affichés au
102      cours du processus de configuration.</dd>
103
104      <dt><code>--srcdir=<var>DIR</var></code></dt>
105      <dd>Définit le répertoire <var>DIR</var> comme répertoire des
106      fichiers sources. Par défaut, c'est le répertoire où se situe le
107      script <code class="program"><a href="../programs/configure.html">configure</a></code>, ou le répertoire parent.</dd>
108
109      <dt><code>--silent</code></dt>
110      <dd>Identique à <code>--quiet</code></dd>
111
112      <dt>-V</dt>
113      <dt>--version</dt>
114      <dd>Affichage des informations de copyright et sortie du
115      script.</dd>
116    </dl>

119  <h3><a name="installationdirectories" id="installationdirectories">Répertoires
120  d'installation</a></h3>
121
122    <p>Ces options permettent de spécifier le répertoire d'installation.
123    L'arborescence de l'installation dépend de l'organisation (layout)
124    sélectionnée.</p>
125
126    <dl>
127      <dt><code>--prefix=<var>PREFIX</var></code></dt>
128      <dd>Installe les fichiers indépendants de l'architecture dans
129      <var>PREFIX</var>. Par défaut, le répertoire d'installation est
130      <code>/usr/local/apache2</code>.</dd>
131
132      <dt><code>--exec-prefix=<var>EPREFIX</var></code></dt>
133      <dd>Installe les fichiers dépendants de l'architecture dans
134      <var>EPREFIX</var>. La valeur par défaut de cette option
135      correspond à la valeur de la variable
136      <var>PREFIX</var>.</dd>
137    </dl>
138
139    <p>Par défaut, <code>make install</code> va installer tous les
140    fichiers dans <code>/usr/local/apache2/bin</code>,
141    <code>/usr/local/apache2/lib</code>, etc... Vous pouvez cependant
142    spécifier un préfixe d'installation autre que
143    <code>/usr/local/apache2</code> en utilisant l'option
144    <code>--prefix</code> (par exemple <code>--prefix=$HOME</code>).</p>
145
146    <h4><a name="layout" id="layout">Spécifier une organisation (layout) des
147    répertoires</a></h4>
148      <dl>
149        <dt><code>--enable-layout=<var>LAYOUT</var></code></dt>
150        <dd>Configure le code source et les scripts de compilation de
151	façon à ce que l'arborescence d'installation adopte
152	l'organisation <var>LAYOUT</var>. Ceci vous permet de spécifier
153	des chemins séparés pour chaque type de fichier de
154	l'installation du serveur HTTP Apache. Le fichier
155	<code>config.layout</code> contient de nombreux exemples de
156	configurations, et vous pouvez créer vos propres configurations
157	personnalisées en vous basant sur ces exemples. Les différentes
158	organisations contenues dans ce fichier sont enregistrées sous
159	forme de sections <code>&lt;Layout
160	FOO&gt;...&lt;/Layout&gt;</code> et référencées dans ce cas par
161	le nom <code>FOO</code>. L'organisation par défaut
162	est <code>Apache</code>.</dd>
163      </dl>

166    <h4><a name="directoryfinetuning" id="directoryfinetuning">Configuration avancée des
167    répertoires d'installation</a></h4>
168
169      <p>Pour une définition plus précise des répertoires
170      d'installation, utilisez les options ci-dessous. Notez que les
171      répertoires par défaut sont définis par <code>autoconf</code>, et
172      que leurs valeurs sont écrasées par les valeurs correspondantes
173      définies lors du choix de l'organisation des répertoires
174      (layout).</p>
175
176      <dl>
177        <dt><code>--bindir=<var>DIR</var></code></dt>
178        <dd>Installe les exécutables utilisateur dans <var>DIR</var>.
179	Les exécutables utilisateur sont des programmes support comme
180	<code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code>, <code class="program"><a href="../programs/dbmmanage.html">dbmmanage</a></code>,
181	etc..., et destinés aux administrateurs du site. Par défaut,
182	<var>DIR</var> est défini à
183	<code><var>EPREFIX</var>/bin</code>.</dd>
184
185        <dt><code>--datadir=<var>DIR</var></code></dt>
186        <dd>Installe les données non modifiables indépendantes de
187	l'architecture dans <var>DIR</var>. Par défaut,
188	<code>datadir</code> est défini à
189	<code><var>PREFIX</var>/share</code>. Cette option est fournie
190	par autoconf et actuellement inutilisée.</dd>
191
192        <dt><code>--includedir=<var>DIR</var></code></dt>
193        <dd>Installe les fichiers d'en-têtes C dans <var>DIR</var>. Par
194	défaut, <code>includedir</code> est défini à
195	<code><var>EPREFIX</var>/include</code>.</dd>
196
197        <dt><code>--infodir=<var>DIR</var></code></dt>
198        <dd>Installe la documentation info dans <var>DIR</var>. Par
199	défaut, <code>infodir</code> est défini à
200	<code><var>PREFIX</var>/info</code>. Cette option est
201	actuellement inutilisée.</dd>
202
203        <dt><code>--libdir=<var>DIR</var></code></dt>
204        <dd>Installe les fichiers objet des bibliothèques dans
205	<var>DIR</var>. Par défaut, <code>libdir</code> est défini à
206	<code><var>EPREFIX</var>/lib</code>.</dd>
207
208        <dt><code>--libexecdir=<var>DIR</var></code></dt>
209        <dd>Installe les exécutables du programme (autrement dit les
210	modules partagés) dans <var>DIR</var>. Par défaut,
211	<code>libexecdir</code> est défini à
212	<code><var>EPREFIX</var>/modules</code>.</dd>
213
214        <dt><code>--localstatedir=<var>DIR</var></code></dt>
215        <dd>Installe les données temporaires modifiables spécifiques à
216	la machine dans
217	<var>DIR</var>. Par défaut, <code>localstatedir</code> est
218	défini à <code><var>PREFIX</var>/var</code>. Cette option est
219	fournie par <code>autoconf</code> et est actuellement
220	inutilisée.</dd>
221
222        <dt><code>--mandir=<var>DIR</var></code></dt>
223        <dd>Installe les pages de manuel dans  <var>DIR</var>. Par
224	défaut, <code>mandir</code> est défini à
225	<code><var>EPREFIX</var>/man</code>.</dd>
226
227        <dt><code>--oldincludedir=<var>DIR</var></code></dt>
228        <dd>Installe les fichiers d'en-têtes C pour les programmes
229	autres que gcc dans <var>DIR</var>. Par défaut,
230	<code>oldincludedir</code> est défini à
231	<code>/usr/include</code>. Cette option est fournie par
232	<code>autoconf</code> et est actuellement inutilisée.</dd>
233
234        <dt><code>--sbindir=<var>DIR</var></code></dt>
235        <dd>Installe les exécutables de l'administrateur système dans
236	<var>DIR</var>. Ce sont les programmes du serveur comme
237	<code class="program"><a href="../programs/httpd.html">httpd</a></code>, <code class="program"><a href="../programs/apachectl.html">apachectl</a></code>,
238	<code class="program"><a href="../programs/suexec.html">suexec</a></code>, etc..., qui sont nécessaires à
239	l'exécution du serveur HTTP Apache. Par défaut,
240	<code>sbindir</code> est défini à
241	<code><var>EPREFIX</var>/sbin</code>.</dd>
242
243        <dt><code>--sharedstatedir=<var>DIR</var></code></dt>
244        <dd>Installe les données modifiables indépendantes de
245	l'architecture dans <var>DIR</var>. Par défaut,
246	<code>sharedstatedir</code> est défini à
247	<code><var>PREFIX</var>/com</code>. Cette option est fournie par
248	<code>autoconf</code> et est actuellement inutilisée.</dd>
249
250        <dt><code>--sysconfdir=<var>DIR</var></code></dt>
251        <dd>Installe les données non modifiables spécifiques à la
252	machine comme les fichiers de configuration du serveur
253	<code>httpd.conf</code>, <code>mime.types</code>, etc... dans
254	<var>DIR</var>. Par défaut, <code>sysconfdir</code> est défini à
255	<code><var>PREFIX</var>/conf</code>.</dd>
256      </dl>

260  <h3><a name="systemtypes" id="systemtypes">Types de systèmes</a></h3>
261
262    <p>Ces options sont utilisées pour la cross-compilation du serveur
263    HTTP Apache afin de pouvoir l'utiliser sur un autre système. Dans le
264    cas général où la compilation et l'exécution du serveur ont lieu sur
265    le même système, ces options ne sont pas utilisées.</p>
266
267    <dl>
268      <dt><code>--build=<var>BUILD</var></code></dt>
269      <dd>Définit le type du système sur lequel les outils sont
270      compilés. Par défaut, il s'agit de la chaîne renvoyée par le
271      script <code>config.guess</code>.</dd>
272
273      <dt><code>--host=<var>HOST</var></code></dt>
274      <dd>Définit le type du système sur lequel le serveur s'exécutera.
275      Par défaut, <var>HOST</var> est identique à <var>BUILD</var>.</dd>
276
277      <dt><code>--target=<var>TARGET</var></code></dt>
278      <dd>Configure pour construire des compilateurs pour le type de
279      système <var>TARGET</var>. Par défaut, <var>TARGET</var> est
280      identique à <var>HOST</var>. Cette option est fournie par
281	<code>autoconf</code> et n'est pas requise par le serveur HTTP
282	Apache.</dd>
283    </dl>

286  <h3><a name="optionalfeatures" id="optionalfeatures">Fonctionnalités
287  optionnelles</a></h3>
288
289    <p>Ces options vous permettent de configurer avec précision les
290    fonctionnalités de votre futur serveur HTTP.</p>
291
292    <h4><a name="generaloptfeat" id="generaloptfeat">Syntaxe générale</a></h4>
293      <p>D'une manière générale, vous pouvez utiliser la syntaxe
294      suivante pour activer ou désactiver une fonctionnalité :</p>
295
296      <dl>
297        <dt><code>--disable-<var>FONCTIONNALITE</var></code></dt>
298        <dd>Désactive la fonctionnalité <var>FONCTIONNALITE</var>.
299	Identique à
300	<code>--enable-<var>FONCTIONNALITE</var>=no</code>.</dd>
301
302        <dt><code>--enable-<var>FONCTIONNALITE</var>[=<var>ARG</var>]</code></dt>
303        <dd>Active la fonctionnalité <var>FONCTIONNALITE</var>. La
304	valeur par défaut de <var>ARG</var> est <code>yes</code>.</dd>
305
306        <dt><code>--enable-<var>MODULE</var>=shared</code></dt>
307        <dd>Le module spécifié sera compilé en tant que module DSO. Par
308	défaut, les modules activés sont liés dynamiquement.</dd>
309
310        <dt><code>--enable-<var>MODULE</var>=static</code></dt>
311        <dd>Le module correspondant sera lié statiquement.</dd>
312      </dl>
313
314      <div class="note"><h3>Note</h3>
315        Si vous spécifiez <code>--enable-<var>foo</var></code>, et si
316	<var>foo</var> n'existe pas, <code class="program"><a href="../programs/configure.html">configure</a></code> ne le
317	signalera pas ; vous devez donc prendre soin de taper les
318	options correctement.
319      </div>

322    <h4><a name="choosemodules" id="choosemodules">Choix des modules à compiler</a></h4>
323      <p>La plupart des modules sont compilés par défaut et ils doivent être
324      désactivés de manière explicite ou via le mots-clé <code>few</code> (voir
325      ci-dessous <code>--enable-modules</code>,
326      <code>--enable-mods-shared</code> et <code>--enable-mods-static</code>
327      pour une explication plus détaillée), ou
328      <code>--enable-modules=none</code> pour les désactiver tous.</p>
329
330      <p>Par défaut, les autres modules ne sont pas compilés et doivent
331      être activés explicitement, ou en utilisant les mots-clés
332      <code>all</code> ou <code>reallyall</code> pour être disponibles.</p>
333
334      <p>Pour déterminer quels modules sont compilés par défaut,
335      exécutez la commande <code>./configure -h</code> ou
336      <code>./configure --help</code>, et consultez les <code>Optional
337      Features</code>. Par exemple, supposons que vous soyez intéressé
338      par les modules <code>mod_example1</code> et
339      <code>mod_example2</code>, et que vous voyiez ceci :</p>
340
341        <div class="example"><pre>Optional Features:
342  ...
343  --disable-example1     example module 1
344  --enable-example2      example module 2
345  ...</pre></div>
346
347        <p>Le module <code>mod_example1</code> est ici activé par
348	défaut, et vous devez spécifier <code>--disable-example1</code>
349	si vous ne voulez pas le compiler. Par contre, le module
350	<code>mod_example2</code> est désactivé par défaut, et vous
351	devez spécifier <code>--enable-example2</code> si vous voulez le
352	compiler.</p>

356    <h4><a name="mpms" id="mpms">Modules Multi-Processus</a></h4>
357      <p>Les <a href="../mpm.html">Modules Multi-Processus</a>, ou MPMs,
358      constituent le coeur du serveur. Un seul MPM doit être actif pour
359      que le serveur puisse fonctionner. Vous trouverez la liste des
360      MPMs disponibles à <a href="../mod/">module index page</a>.</p>
361
362      <p>Les MPMs peuvent être compilés en tant que modules DSO pour un
363      chargement dynamique, ou liés statiquement avec le serveur, et
364      sont activés via les options suivantes :</p>
365
366      <dl>
367        <dt><code>--with-mpm=MPM</code></dt>
368        <dd>
369          <p>Sélectionne le MPM par défaut pour votre serveur. Si les
370	  MPMs sont compilés en tant que modules DSO (voir
371	  <code>--enable-mpms-shared</code>), cette option spécifie le
372	  MPM qui sera chargé par défaut selon le fichier de
373	  configuration. Dans le cas contraire, cette option spécifie le
374	  seul MPM disponible qui sera lié statiquement avec le
375	  serveur.</p>
376          <p>Si cette option est omise, c'est le <a href="../mpm.html#defaults">MPM par défaut</a> pour votre
377	  système d'exploitation qui sera utilisé.</p>
378        </dd>
379
380        <dt><code>--enable-mpms-shared=<var>Liste de MPM</var></code></dt>
381        <dd>
382          <p>Définit une liste de MPMs à compiler en tant que modules
383	  dynamiquement partagés (DSO). Un de ces modules doit être
384	  chargé dynamiquement via la directive <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code>.</p>
385          <p><var>Liste de MPM</var> est une liste, entourée
386	  d'apostrophes,  de noms de MPM séparés par des espaces. Par
387	  exemple :</p>
388          <div class="example"><p><code>
389            --enable-mpms-shared='prefork worker'
390          </code></p></div>
391          <p>Vous pouvez aussi utiliser le mot-clé <code>all</code>, ce
392	  qui aura pour effet de spécifier tous les MPMs qui supportent
393	  le chargement dynamique sur la plate-forme considérée, et de
394	  les compiler en tant que modules DSO. Par exemple :</p>
395          <div class="example"><p><code>
396            --enable-mpms-shared=all
397          </code></p></div>
398	</dd>
399      </dl>

402    <h4><a name="modules" id="modules">Modules tiers</a></h4>
403      <p>Pour ajouter des modules tiers, utilisez les options suivantes
404      :</p>
405
406      <dl>
407        <dt><code>--with-module=<var>type-module</var>:<var>fichier-module</var>[,
408          <var>type-module</var>:<var>fichier-module</var>]</code></dt>
409        <dd><p>Ajoute un ou plusieurs modules tiers à la liste des
410	modules liés statiquement. Le fichier source du module
411	<var>fichier-module</var> sera recherché dans le sous-répertoire
412	<var>type-module</var> de l'arborescence des sources de votre
413	serveur HTTP Apache. S'il ne l'y trouve pas,
414	<code class="program"><a href="../programs/configure.html">configure</a></code> considèrera <var>fichier-module</var>
415	comme un chemin de fichier absolu et essaiera de copier le
416	fichier source dans le sous-répertoire <var>type-module</var>.
417	Si ce sous-répertoire n'existe pas, il sera créé et un fichier
418	<code>Makefile.in</code> standard y sera enregistré.</p>
419          <p>Cette option est conçue pour ajouter de petits modules
420	  externes ne comportant qu'un seul fichier source. Pour des
421	  modules plus complexes, vous devrez lire la documentation du
422	  fournisseur du module.</p>
423          <div class="note"><h3>Note</h3>
424            Si vous voulez compiler un module DSO (lié de manière
425	    dynamique au lieu de statique), utilisez le programme
426	    <code class="program"><a href="../programs/apxs.html">apxs</a></code>.</div>
427        </dd>
428
429      </dl>

432    <h4><a name="otheroptfeat" id="otheroptfeat">Options cumulatives et autres
433    options</a></h4>
434      <dl>
435        <dt><code>--enable-maintainer-mode</code></dt>
436        <dd>Active les avertissements de débogage et de compilation et
437	charge tous les modules compilés.</dd>
438
439        <dt><code>--enable-mods-shared=<var>LISTE-MODULES</var></code></dt>
440        <dd>
441          <p>Définit une liste de modules à activer et à compiler en
442	  tant que modules dynamiques partagés. Cela signifie que ces
443	  modules doivent être chargés dynamiquement en utilisant la
444	  directive <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code>.</p>
445          <p><var>LISTE-MODULES</var> est une liste, entourée
446	  d'apostrophes, de noms de modules
447	  séparés par des espaces. Les noms
448	  des modules sont spécifiés sans le préfixe <code>mod_</code>.
449	  Par exemple :</p>
450          <div class="example"><p><code>
451            --enable-mods-shared='headers rewrite dav'
452          </code></p></div>
453          <p>Vous pouvez aussi utiliser les mots-clés <code>reallyall</code>,
454	  <code>all</code>, <code>most</code> et <code>few</code>. Par
455	  exemple,</p>
456          <div class="example"><p><code>
457            --enable-mods-shared=most
458          </code></p></div>
459          <p>va compiler la plupart des modules en tant que modules DSO,</p>
460	  <div class="example"><p><code>
461            --enable-mods-shared=few
462          </code></p></div>
463          <p>ne compilera qu'un jeu de modules de base.</p>
464          <p>Le jeu par défaut correspond au mot-clé <code>most</code>.</p>
465            
466	  <p>Les directives <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code> correspondant aux
467	  différents modules choisis sont automatiquement générées dans
468	  le fichier de configuration principal. Par défaut, toutes ces
469	  directives sont mises en commentaire, sauf pour les modules
470	  requis ou ceux explicitement sélectionnés par un argument
471	  <code>--enable-nom-module</code> du script configure. Vous
472	  pouvez modifier le jeu de modules chargé en activant ou
473	  désactivant les directives <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code> dans le fichier
474	  <code>httpd.conf</code>. En outre, les directives <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code> peuvent être activées
475	  pour tous les modules compilés via l'option
476	  <code>--enable-load-all-modules</code> du script configure.</p>
477          
478        </dd>
479
480        <dt><code>--enable-mods-static=<var>MODULE-LIST</var></code></dt>
481        <dd>Cette option produit le même effet que l'option
482	<code>--enable-mods-shared</code>, à l'exception que les modules
483	seront liés statiquement. Cela signifie que les modules
484	spécifiés seront toujours disponibles au cours du fonctionnement
485	de <code class="program"><a href="../programs/httpd.html">httpd</a></code>. Ils n'ont pas besoin d'être chargés
486	via la directive <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code>.</dd>
487
488	<dt><code>--enable-modules=<var>MODULE-LIST</var></code></dt>
489        <dd>Cette option se comporte comme
490	<code>--enable-mods-shared</code>, et va aussi lier les modules
491	concernés dynamiquement. Le mot-clé spécial <code>none</code>
492	désactive la compilation de tous les modules.</dd>
493
494        <dt><code>--enable-v4-mapped</code></dt>
495        <dd>Permet aux sockets IPv6 de traiter les connexions IPv4.</dd>
496
497        <dt><code>--with-port=<var>PORT</var></code></dt>
498        <dd>Permet de définir le port que le programme
499	<code class="program"><a href="../programs/httpd.html">httpd</a></code> va écouter. Ce numéro de port est
500	utilisé lors de la génération du fichier de configuration
501	<code>httpd.conf</code>. Sa valeur par défaut est 80.</dd>
502
503        <dt><code>--with-program-name</code></dt>
504        <dd>Permet de définir un nom d'exécutable alternatif. Le nom par
505	défaut est <code>httpd</code>.</dd>
506      </dl>

510  <h3><a name="packages" id="packages">Paquets optionnels</a></h3>
511    <p>Ces options permettent de définir des paquets optionnels.</p>
512
513    <h4><a name="generalpackages" id="generalpackages">Syntaxe générale</a></h4>
514      <p>D'une manière générale, vous pouvez utiliser la syntaxe
515      suivante pour définir un paquet optionnel :</p>
516
517      <dl>
518        <dt><code>--with-<var>PAQUET</var>[=<var>ARG</var>]</code></dt>
519        <dd>Utilise le paquet <var>PAQUET</var>. La valeur par défaut de
520	<var>ARG</var> est <code>yes</code>.</dd>
521
522        <dt><code>--without-<var>PAQUET</var></code></dt>
523        <dd>N'utilise pas le paquet <var>PAQUET</var>. Cette option est
524	identique à <code>--with-<var>PAQUET</var>=no</code>. Elle est
525	fournie par <code>autoconf</code> mais n'est pas très utile pour
526	le serveur HTTP Apache.</dd>
527      </dl>
528    
529
530    
531
532    <h4><a name="packageopt" id="packageopt">Paquets spécifiques</a></h4>
533      <dl>
534        <dt><code>--with-apr=<var>REP</var>|<var>FICHIER</var></code></dt>
535        <dd>La <a class="glossarylink" href="../glossary.html#apr" title="voir glossaire">Bibliothèque pour la portabilité
536	d'Apache ou
537	Apache Portable Runtime</a> (APR) fait partie de la
538	distribution des sources de httpd et est compilée
539	automatiquement avec le serveur HTTP. Si vous voulez utiliser
540	une APR déjà installée à la place, vous devez indiquer à
541	<code class="program"><a href="../programs/configure.html">configure</a></code> le chemin du script
542	<code>apr-config</code>. Vous pouvez spécifier le chemin absolu
543	et le nom ou le répertoire d'installation de l'APR.
544	<code>apr-config</code> doit se trouver dans ce répertoire ou
545	dans le sous-repertoire <code>bin</code>.</dd>
546
547        <dt><code>--with-apr-util=<var>REP</var>|<var>FICHIER</var></code></dt>
548        <dd>Les utilitaires pour la Bibliothèque pour la portabilité
549	d'Apache ou Apache Portable Runtime Utilities (APU) font partie de la
550	distribution des sources de httpd et sont compilés
551	automatiquement avec le serveur HTTP. Si vous voulez utiliser
552	des APU déjà installés à la place, vous devez indiquer à
553	<code class="program"><a href="../programs/configure.html">configure</a></code> le chemin du script
554	<code>apu-config</code>. Vous pouvez spécifier le chemin absolu
555	et le nom ou le répertoire d'installation des APU.
556	<code>apu-config</code> doit se trouver dans ce répertoire ou
557	dans le sous-repertoire <code>bin</code>.</dd>
558
559        <dt><code>--with-ssl=<var>REP</var></code></dt>
560        <dd>Si <code class="module"><a href="../mod/mod_ssl.html">mod_ssl</a></code> a été activé,
561	<code class="program"><a href="../programs/configure.html">configure</a></code> recherche une installation d'OpenSSL.
562	Vous pouvez définir le répertoire de la boîte à outils SSL/TLS à
563	la place.</dd>
564
565        <dt><code>--with-z=<var>REP</var></code></dt>
566        <dd><code class="program"><a href="../programs/configure.html">configure</a></code> recherche automatiquement une
567	bibliothèque <code>zlib</code> installée si la configuration de
568	vos sources en nécessite une (par exemple lorsque
569	<code class="module"><a href="../mod/mod_deflate.html">mod_deflate</a></code> est activé). Vous pouvez définir le
570	répertoire de la bibliothèque de compression à la place.</dd>
571      </dl>
572
573      <p>De nombreuses fonctionnalités du serveur HTTP Apache, y compris
574      les directives <code class="directive"><a href="../mod/mod_rewrite.html#rewritemap">RewriteMap</a></code> DBM de
575      <code class="module"><a href="../mod/mod_rewrite.html">mod_rewrite</a></code> et <code class="module"><a href="../mod/mod_authn_dbm.html">mod_authn_dbm</a></code>
576      utilisent une base de données simple
577      de la forme clé/valeur pour une recherche rapide d'informations.
578      SDBM, inclus dans les APU, est donc toujours disponible. Si vous
579      souhaitez utiliser d'autres types de bases de données, utilisez
580      les options suivantes afin de les activer :</p>
581
582      <dl>
583        <dt><code>--with-gdbm[=<var>chemin</var>]</code></dt>
584        <dd>Si aucun <var>chemin</var> n'est spécifié,
585	<code class="program"><a href="../programs/configure.html">configure</a></code> va rechercher les fichiers d'en-têtes et
586	les bibliothèques d'une installation DBM GNU dans les chemins
587	standards. Avec un <var>chemin</var> explicite,
588	<code class="program"><a href="../programs/configure.html">configure</a></code> recherchera les fichiers concernés dans
589	<code><var>chemin</var>/lib</code> et
590	<code><var>chemin</var>/include</code>. En fait,
591	<var>chemin</var> permet de spécifier plusieurs chemins
592	d'en-têtes et bibliothèques spécifiques en les séparant par des
593	caractères ':'.</dd>
594
595        <dt><code>--with-ndbm[=<var>chemin</var>]</code></dt>
596        <dd>Identique à <code>--with-gdbm</code>, mais recherche une
597	installation de New DBM.</dd>
598
599        <dt><code>--with-berkeley-db[=<var>chemin</var>]</code></dt>
600        <dd>Identique à <code>--with-gdbm</code>, mais recherche une
601	installation de Berkeley DB.</dd>
602      </dl>
603
604      <div class="note"><h3>Note</h3>
605        <p>Les options DBM sont fournies par les APU et passées en
606	paramètres à son script de configuration. Elles sont inutiles
607	lorsqu'on utilise des APU déjà installés définis par
608	<code>--with-apr-util</code>.</p>
609        <p>Vous pouvez utiliser plusieurs implémentations DBM avec votre
610	serveur HTTP. Le type DBM approprié sera choisi au cours de la
611	configuration de l'exécution à chaque démarrage.</p>
612      </div>

616  <h3><a name="supportopt" id="supportopt">Options pour les programmes de
617  support</a></h3>
618    <dl>
619      <dt><code>--enable-static-support</code></dt>
620      <dd>Permet de compiler une version des binaires de support liés
621      statiquement. En d'autres termes, la compilation produira un
622      exécutable indépendant comportant toutes les bibliothèques
623      nécessaires. Sans cette option, les binaires de supports sont liés
624      dynamiquement.</dd>
625
626      <dt><code>--enable-suexec</code></dt>
627      <dd>Utilisez cette option pour activer la programme
628      <code class="program"><a href="../programs/suexec.html">suexec</a></code>, qui vous permet de définir un uid et un
629      gid pour les processus lancés. <strong>N'utilisez cette option que
630      si vous maîtrisez toutes les implications en matière de sécurité
631      de l'exécution d'un binaire suid sur votre serveur.</strong>
632      D'autres options permettent de configurer
633      <code class="program"><a href="../programs/suexec.html">suexec</a></code> comme décrit <a href="#suexec">ci-dessous</a>.</dd>
634    </dl>
635
636    <p>Il est possible de lier statiquement le binaire d'un programme
637    support particulier en utilisant les options suivantes :</p>
638
639    <dl>
640      <dt><code>--enable-static-ab</code></dt>
641      <dd>Compile une version liée statiquement du programme
642      <code class="program"><a href="../programs/ab.html">ab</a></code>.</dd>
643
644      
645      <dt><code>--enable-static-checkgid</code></dt>
646      <dd>&gt;Compile une version liée statiquement du programme
647      <code>checkgid</code>.</dd>
648
649      <dt><code>--enable-static-htdbm</code></dt>
650      <dd>Compile une version liée statiquement du programme <code class="program"><a href="../programs/htdbm.html">htdbm</a></code>.</dd>
651
652      <dt><code>--enable-static-htdigest</code></dt>
653      <dd>Compile une version liée statiquement du programme <code class="program"><a href="../programs/htdigest.html">htdigest</a></code>.</dd>
654
655      <dt><code>--enable-static-htpasswd</code></dt>
656      <dd>Compile une version liée statiquement du programme <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code>.</dd>
657
658      <dt><code>--enable-static-logresolve</code></dt>
659      <dd>Compile une version liée statiquement du programme <code class="program"><a href="../programs/logresolve.html">logresolve</a></code>.</dd>
660
661      <dt><code>--enable-static-rotatelogs</code></dt>
662      <dd>Compile une version liée statiquement du programme <code class="program"><a href="../programs/rotatelogs.html">rotatelogs</a></code>.</dd>
663    </dl>
664
665    <h4><a name="suexec" id="suexec">Options de configuration de <code class="program"><a href="../programs/suexec.html">suexec</a></code><code class="program"><a href="../programs/suexec.html">suexec</a></code></a></h4>
666      
667      <p>Les options suivantes permettent de définir avec précision le
668      comportement du programme <code class="program"><a href="../programs/suexec.html">suexec</a></code>. Voir <a href="suexec.html#install">Configurer et installer suEXEC</a>
669	pour plus de détails.</p>
670
671      <dl>
672        <dt><code>--with-suexec-bin</code></dt>
673        <dd>Définit le chemin du binaire <code class="program"><a href="../programs/suexec.html">suexec</a></code>. La
674	valeur par défaut est <code>--sbindir</code> (voir <a href="#directoryfinetuning">Définition précise des répertoires
675	d'installation</a>).</dd>
676
677        <dt><code>--with-suexec-caller</code></dt>
678        <dd>Définit l'utilisateur qui a l'autorisation d'appeler
679	<code class="program"><a href="../programs/suexec.html">suexec</a></code>. Il est en général souhaitable que ce
680	soit le même que celui sous lequel <code class="program"><a href="../programs/httpd.html">httpd</a></code>
681	s'exécute.</dd>
682
683        <dt><code>--with-suexec-docroot</code></dt>
684        <dd>Définit l'arborescence des répertoires dans laquelle le
685	lancement des exécutables via <code class="program"><a href="../programs/suexec.html">suexec</a></code> est
686	autorisé. La valeur par défaut est
687	<code>--datadir/htdocs</code>.</dd>
688
689        <dt><code>--with-suexec-gidmin</code></dt>
690        <dd>Définit la valeur de GID la plus basse autorisée comme
691	valeur cible pour <code class="program"><a href="../programs/suexec.html">suexec</a></code>. La valeur par
692	défaut est 100.</dd>
693
694        <dt><code>--with-suexec-logfile</code></dt>
695        <dd>Définit le nom du fichier journal de
696	<code class="program"><a href="../programs/suexec.html">suexec</a></code>. La valeur par défaut est
697	<code>--logfiledir/suexec_log</code>.</dd>
698
699        <dt><code>--with-suexec-safepath</code></dt>
700        <dd>Définit la valeur de la variable d'environnement
701	<code>PATH</code> pour les processus lancés par
702	<code class="program"><a href="../programs/suexec.html">suexec</a></code>. La valeur par défaut est
703	<code>/usr/local/bin:/usr/bin:/bin</code>.</dd>
704
705        <dt><code>--with-suexec-userdir</code></dt>
706        <dd>Définit le sous-répertoire du répertoire utilisateur qui
707	contient tous les exécutables pouvant être lancés par
708	<code class="program"><a href="../programs/suexec.html">suexec</a></code>. Cette option est nécessaire si vous
709	souhaitez utiliser <code class="program"><a href="../programs/suexec.html">suexec</a></code> avec des
710	répertoires utilisateurs (définis via
711	<code class="module"><a href="../mod/mod_userdir.html">mod_userdir</a></code>). La valeur par défaut est
712	<code>public_html</code>.</dd>
713
714        <dt><code>--with-suexec-uidmin</code></dt>
715        <dd>Définit la valeur d'UID la plus basse autorisée comme
716	valeur cible pour <code class="program"><a href="../programs/suexec.html">suexec</a></code>. La valeur par
717	défaut est 100.</dd>
718
719        <dt><code>--with-suexec-umask</code></dt>
720        <dd>Définit le masque de permissions <code>umask</code> pour les
721	processus lancés par <code class="program"><a href="../programs/suexec.html">suexec</a></code>. Il correspond
722	par défaut au masque défini par la configuration de votre
723	système.</dd>
724      </dl>

729<h2><a name="env" id="env">Variables d'environnement</a></h2>
730  <p>Certaines variables d'environnement permettent de modifier les
731  choix effectués par <code class="program"><a href="../programs/configure.html">configure</a></code>, ou d'aider ce dernier à
732  trouver les bibliothèques et programmes possédant des noms et chemins
733  non standards.</p>
734
735  
736  <dl>
737    <dt><code>CC</code></dt>
738    <dd>Définit la commande du compilateur C à utiliser pour la
739    compilation.</dd>
740
741    <dt><code>CFLAGS</code></dt>
742    <dd>Définit les paramètres du compilateur C que vous voulez utiliser
743    pour la compilation.</dd>
744
745    <dt><code>CPP</code></dt>
746    <dd>Définit la commande du préprocesseur C à utiliser.</dd>
747
748    <dt><code>CPPFLAGS</code></dt>
749    <dd>Définit les paramètres du préprocesseur C/C++, par exemple
750    <code>-I<var>répertoire-include</var></code>, si certains de vos
751    fichiers d'en-têtes se trouvent dans le répertoire non standard
752    <var>répertoire-include</var>.</dd>
753
754    <dt><code>LDFLAGS</code></dt>
755    <dd>Définit les paramètres de l'éditeur de liens, par exemple
756    <code>-L<var>répertoire-lib</var></code>, si certaines de vos
757    bibliothèques se trouvent dans le répertoire non standard
758    <var>répertoire-lib</var>.</dd>
759  </dl>
