33    <p><code class="module"><a href="../mod/mod_rewrite.html">mod_rewrite</a></code> provides a way to modify incoming
34    URL requests, dynamically, based on <a href="intro.html#regex">regular
35    expression</a> rules. This allows you to map arbitrary URLs onto
36    your internal URL structure in any way you like.</p>
37
38      <p>It supports an unlimited number of rules and an
39      unlimited number of attached rule conditions for each rule to
40      provide a really flexible and powerful URL manipulation
41      mechanism. The URL manipulations can depend on various tests:
42      server variables, environment variables, HTTP
43      headers, time stamps, external database lookups, and various other
44      external programs or handlers, can be used to achieve granular URL
45      matching.</p>
46
47      <p>Rewrite rules can operate on the full URLs, including the path-info
48      and query string portions, and may be used in per-server context
49      (<code>httpd.conf</code>), per-virtualhost context (<code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code> blocks), or
50      per-directory context (<code>.htaccess</code> files and <code class="directive"><a href="../mod/core.html#directory">&lt;Directory&gt;</a></code> blocks). The
51      rewritten result can lead to further rules, internal
52      sub-processing, external request redirection, or proxy
53      passthrough, depending on what <a href="flags.html">flags</a> you
54      attach to the rules.</p>
55
56      <p>Since <code class="module"><a href="../mod/mod_rewrite.html">mod_rewrite</a></code> is so powerful, it can indeed be rather
57      complex. This document supplements the <a href="../mod/mod_rewrite.html">reference documentation</a>, and
58      attempts to allay some of that complexity, and provide highly
59      annotated examples of common scenarios that you may handle with
60      <code class="module"><a href="../mod/mod_rewrite.html">mod_rewrite</a></code>. But we also attempt to show you when you should not
61      use <code class="module"><a href="../mod/mod_rewrite.html">mod_rewrite</a></code>, and use other standard Apache features instead,
62      thus avoiding this unnecessary complexity.</p>
