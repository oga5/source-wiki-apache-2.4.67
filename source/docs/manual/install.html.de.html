38    <p>Dieses Dokument umfa&#223;t nur die Kompilierung und Installation des
39    Apache auf Unix und Unix-&#228;hnlichen Systemen. F&#252;r die
40    Kompilierung und Installation unter Windows lesen Sie bitte <a href="platform/windows.html">Den Apache unter Microsoft Windows
41    betreiben</a>. F&#252;r andere Plattformen lesen Sie bitte die
42    Dokumentation <a href="platform/">Plattformen</a>.</p>
43
44    <p>Die Konfigurations- und Installationsumgebung des Apache 2.0 hat sich
45    seit dem Apache 1.3 komplett ver&#228;ndert. Der Apache 1.3 benutzt einen
46    speziellen Satz von Skripten, um eine einfache Installation zu
47    erm&#246;glichen. Der Apache 2.0 dagegen verwendet nun
48    <code>libtool</code> und <code>autoconf</code>, um eine Umgebung zu
49    schaffen, die der vieler anderer Open Source Projekte &#228;hnlich
50    sieht.</p>
51
52    <p>Wenn Sie von einer Unterversion auf die n&#228;chste aktualisieren (z.B.
53    von 2.0.50 auf 2.0.51), springen Sie bitte zum Abschnitt <a href="#upgrading">Upgrade</a>.</p>

131<h2><a name="requirements" id="requirements">Anforderungen</a></h2>
132
133    <p>Folgende Anforderungen gelten f&#252;r die Erstellung des
134    Apache:</p>
135
136    <dl>
137      <dt>Plattenplatz</dt>
138      <dd>Stellen Sie sicher, dass Sie kurzzeitig wenigstens 50 MB freien
139      Festplattenplatz zur Verf&#252;gung haben. Nach der Installation
140      belegt der Apache ungef&#228;hr 10 MB Plattenplatz. Der
141      tats&#228;chliche Platzbedarf variiert in Abh&#228;ngigkeit von den
142      gew&#228;hlten Konfigurationseinstellungen und
143      Modulen von Drittanbietern.</dd>
144
145      <dt>ANSI-C-Compiler und Generierungswerkzeuge</dt>
146      <dd>Stellen Sie sicher, dass Sie einen ANSI-C Compiler installiert
147      haben. Der <a href="http://www.gnu.org/software/gcc/gcc.html">GNU C
148      Compiler (GCC)</a> der <a href="http://www.gnu.org/">Free Software
149      Foundation (FSF)</a> ist empfehlenswert (Version 2.7.2 ist gut). Wenn
150      Sie den GCC nicht besitzen, stellen Sie zumindest sicher, dass der
151      Compiler Ihres Anbieters ANSI-kompatibel ist. Au&#223;erdem muss Ihr
152      <code>PATH</code> wesentliche Generierungswerkzeuge wie
153      <code>make</code> enthalten.</dd>
154
155      <dt>Zeitgenauigkeit bewahren</dt>
156      <dd>Elemente des HTTP-Protokolls werden in Form einer Tageszeit
157      ausgedr&#252;ckt. Darum sollten Sie jetzt pr&#252;fen, ob Ihr System
158      die F&#228;higkeit zur Zeitsynchronisation besitzt, und diese
159      gegebenenfalls installieren. &#220;blicherweise werden hierf&#252;r
160      die Programme <code>ntpdate</code> oder <code>xntpd</code> verwendet,
161      die auf dem Network Time Protocol (NTP) basieren. N&#228;here
162      Informationen &#252;ber NTP Software und &#246;ffentliche Zeitserver
163      finden Sie in der Usenet Newsgroup <a href="news:comp.protocols.time.ntp">comp.protocols.time.ntp</a>
164      und auf der <a href="http://www.ntp.org">NTP
165      Homepage</a>.</dd>
166
167      <dt><a href="http://www.perl.org/">Perl 5</a>
168      [OPTIONAL]</dt>
169      <dd>F&#252;r einige Hilfsskripte wie <code class="program"><a href="./programs/apxs.html">apxs</a></code>
170      oder <code class="program"><a href="./programs/dbmmanage.html">dbmmanage</a></code> (die in Perl
171      geschrieben sind) wird der Perl 5 Interpreter ben&#246;tigt (die
172      Versionen ab 5.003 sind ausreichend). Wenn Sie mehrere Perl
173      Interpreter haben (beispielsweise eine systemweite Installation von
174      Perl 4 und Ihre eigene Perl 5-Installation), dann sollten Sie die
175      <code>--with-perl</code>-Option (siehe unten) verwenden, um
176      sicherzustellen, dass der richtige Interpreter von
177      <code class="program"><a href="./programs/configure.html">configure</a></code> ausgew&#228;hlt wird.
178      Wenn kein Perl 5-Interpreter vom <code class="program"><a href="./programs/configure.html">configure</a></code>-Skript
179      gefunden werden kann, k&#246;nnen Sie die betroffenen Hilfsskripte nicht
180      verwenden, sind jedoch selbstverst&#228;ndlich nach wie vor in der Lage,
181      den Apache 2.0 zu bauen und zu installieren.</dd>
182    </dl>

185<h2><a name="download" id="download">Download</a></h2>
186
187    <p>Der Apache kann von der <a href="http://httpd.apache.org/download.cgi">Apache HTTP Server
188    Downloadseite</a> heruntergeladen werden, auf der verschiedene Spiegelserver
189    angegeben sind. F&#252;r die meisten Benutzer des Apache ist es auf
190    Unix-&#228;hnlichen Systemen am Besten, die Quellcodeversion herunterzuladen
191    und zu kompilieren. Der Erstellungsprozess (weiter unten beschrieben) ist
192    einfach und erlaubt es Ihnen, den Server Ihren Bed&#252;rfnissen anzupassen.
193    Dazu kommt, dass Bin&#228;rdistributionen gegen&#252;ber der aktuellen
194    Quellcodeversion oft veraltet sind. Wenn Sie tats&#228;chlich ein
195    Bin&#228;rpaket herunterladen, folgen Sie bitte den Anweisungen in der Datei
196    <code>INSTALL.bindist</code>, die der Distribution beiliegt.</p>
197
198    <p>Es ist wichtig, dass Sie nach dem Herunterladen &#252;berpr&#252;fen,
199    dass es sich um einer vollst&#228;ndige und unver&#228;nderte Version des
200    Apache HTTP Servers handelt. Das k&#246;nnen Sie erreichen, indem Sie das
201    heruntergeladene Paket gegen die PGP-Signatur pr&#252;fen. Einzelheiten dazu
202    erfahren Sie auf der <a href="http://httpd.apache.org/download.cgi#verify">Download-Seite</a>. Es
203    ist auch ein erweitertes Beispiel verf&#252;gbar, dass die <a href="http://httpd.apache.org/dev/verification.html">Anwendung von PGP</a>
204    beschreibt.</p>

208<h2><a name="extract" id="extract">Auspacken</a></h2>
209
210    <p>Das Auspacken des Quellcodes aus dem Apache HTTPD Tarball besteht
211    aus einem simplen Dekomprimieren und danach "Ent-tarren":</p>
212
213  <div class="example"><p><code>
214      $ gzip -d httpd-2_1_<em>NN</em>.tar.gz<br />
215      $ tar xvf httpd-2_1_<em>NN</em>.tar
216  </code></p></div>
217
218    <p>Dies erstellt unterhalb des aktuellen Verzeichnisses ein neues
219    Verzeichnis, das den Quellcode f&#252;r die Distribution enth&#228;lt.
220    Sie sollten mit <code>cd</code> in dieses Verzeichnis wechseln,
221    bevor Sie mit der Kompilierung des Servers weitermachen. </p>

225<h2><a name="configure" id="configure">Den Codebaum konfigurieren</a></h2>
226
227    <p>Der n&#228;chste Schritt ist die Konfiguration des
228    Apache-Codebaumes f&#252;r Ihre spezielle Plattform und Ihre
229    pers&#246;nlichen Bed&#252;rfnisse. Dies wird mit dem Skript
230    <code class="program"><a href="./programs/configure.html">configure</a></code> durchgef&#252;hrt, das im Wurzelverzeichnis
231    der Distribution enthalten ist. (Entwickler, welche die CVS Version
232    des Apache-Codebaumes herunterladen, m&#252;ssen <code>autoconf</code>
233    und <code>libtool</code> installiert haben und m&#252;ssen
234    <code>buildconf</code> ausf&#252;hren, bevor sie mit den
235    n&#228;chsten Schritten fortfahren k&#246;nnen. Dies wird bei
236    offiziellen Releases nicht notwendig sein.)</p>
237
238    <p>Um den Codebaum mit den Standardeinstellungen zu konfigurieren,
239    geben Sie einfach <code>./configure</code> ein. Zur &#196;nderung
240    dieser Voreinstellungen akzeptiert <code class="program"><a href="./programs/configure.html">configure</a></code> eine
241    Reihe von Variablen und Kommandozeilenoptionen.</p>
242
243    <p>Die wichtigste Option ist <code>--prefix</code>, der Ablageort, an dem
244    der Apache sp&#228;ter installiert wird, da er f&#252;r diesen Ort
245    konfiguriert werden muss, um korrekt zu arbeiten. Eine feinere Einstellung
246    der Dateiablagen ist mit weiteren <a href="programs/configure.html#installationdirectories">configure-Optionen</a>
247    m&#246;glich.</p>
248
249    <p>Weiterhin k&#246;nnen Sie zu diesem Zeitpunkt festlegen, welche <a href="programs/configure.html#optionalfeatures">Funktionalit&#228;t</a> Sie
250    in den Apache aufnehmen m&#246;chten, indem Sie <a href="mod/">Module</a>
251    aktivieren oder deaktivieren. Der Apache bindet standardm&#228;&#223;ig
252    einen Satz von <a href="mod/module-dict.html#Status">Basismodulen</a> ein.
253    Andere Module werden mit Hilfe der Option
254    <code>--enable-<var>module</var></code> aktiviert, wobei <var>module</var>
255    den Namen des Moduls ohne das Pr&#228;fix <code>mod_</code> darstellt.
256    Ausserdem sind alle Unterstriche durch Bindestriche zu ersetzen. Sie
257    k&#246;nnen sich auch entscheiden, Module als "<a href="dso.html">Shared
258    Objects (DSOs)</a>" zu kompilieren, welche zur Laufzeit ge- und entladen
259    werden k&#246;nnen. Dazu verwenden Sie die Option
260    <code>--enable-<var>module</var>=shared</code>. Entsprechend k&#246;nnen Sie
261    Basismodule mit der Option <code>--disable-<var>module</var></code>
262    deaktivieren. Lassen Sie Vorsicht walten. wenn Sie diese Optionen verwenden,
263    da <code class="program"><a href="./programs/configure.html">configure</a></code> Sie nicht warnen kann, wenn die von Ihnen
264    angegebenen Module nicht existieren; die Option wird dann einfach
265    ignoriert.</p>
266
267    <p>Zus&#228;tzlich ist es zuweilen notwendig, das
268    <code class="program"><a href="./programs/configure.html">configure</a></code>-Skript mit Extrainformationen zum Ablageort
269    Ihres Compilers, Ihrer Bibliotheken oder Header-Dateien zu versorgen. Das
270    tun Sie, indem Sie entweder Umgebungsvariablen oder Kommandozeilenoptionen
271    an <code class="program"><a href="./programs/configure.html">configure</a></code> &#252;bergeben. F&#252;r mehr Informationen
272    lesen Sie bitte die Hilfeseite zu <code class="program"><a href="./programs/configure.html">configure</a></code>.</p>
273
274    <p>Um einen kurzen Eindruck zu gewinnen, welche M&#246;glichkeiten Sie
275    haben, folgt hier ein typisches Beispiel, das den Apache mit einem
276    speziellen Compiler und Compilerflags f&#252;r das
277    Installationsverzeichnis <code>/sk/pkg/apache</code> kompiliert, sowie
278    die beiden zus&#228;tzlichen Module <code class="module"><a href="./mod/mod_rewrite.html">mod_rewrite</a></code> und
279    <code class="module"><a href="./mod/mod_speling.html">mod_speling</a></code> f&#252;r sp&#228;teres Laden durch den
280    DSO-Mechanismus:</p>
281
282  <div class="example"><p><code>
283      $ CC="pgcc" CFLAGS="-O2" \<br />
284       ./configure --prefix=/sw/pkg/apache \<br />
285       --enable-rewrite=shared \<br />
286       --enable-speling=shared
287  </code></p></div>
288
289    <p>Wenn <code class="program"><a href="./programs/configure.html">configure</a></code> startet, ben&#246;tigt es mehrere
290    Minuten, um die Verf&#252;gbarkeit von Features auf Ihrem System zu
291    pr&#252;fen und ein Makefile zu generieren, das sp&#228;ter zur
292    Kompilierung des Servers verwendet wird.</p>
293
294    <p>Einzelheiten zu den vielen verschiedenen <code class="program"><a href="./programs/configure.html">configure</a></code>-Optionen finden Sie auf der Hilfeseite zu
295    <code class="program"><a href="./programs/configure.html">configure</a></code>.</p>

337<h2><a name="test" id="test">Testen</a></h2>
338
339    <p>Sie k&#246;nnen nun Ihren Apache HTTP Server <a href="invoking.html">starten</a>, indem Sie einfach</p>

342

344
345    <p>Danach sollten Sie Ihr erstes Dokument unter dem URL
346    <code>http://localhost/</code> anfordern k&#246;nnen. Die Webseite,
347    die Sie sehen, ist im <code class="directive"><a href="./mod/core.html#documentroot">DocumentRoot</a></code>
348    abgelegt, welches &#252;blicherweise <code><em>PREFIX</em>/htdocs/</code>
349    ist. Den Server <a href="stopping.html">stoppen</a> Sie wieder durch
350    Ausf&#252;hren von:</p>

352  <div class="example"><p><code>$ <em>PREFIX</em>/bin/apachectl stop</code></p></div>

355<h2><a name="upgrading" id="upgrading">Upgrade</a></h2>
356
357    <p>Der erste Schritt beim Aktualisieren besteht darin, die
358    Versionsank&#252;ndigung sowie die <code>CHANGES</code>-Datei in der
359    Quelltextdistribution zu lesen, um &#196;nderungen zu finden, die Ihr
360    System m&#246;glicherweise betreffen. Wenn Sie einen gr&#246;&#223;eren
361    Versionssprung durchf&#252;hren (z.B. vom 1.3 auf 2.0 oder von 2.0 auf
362    2.2), wird es wahrscheinlich auch gr&#246;&#223;ere Unterschiede in der
363    Kompilier- und Laufzeitkonfiguration geben, die manuelle Nacharbeiten
364    erfordern. Au&#223;erdem m&#252;ssen alle Module aktualisiert
365    werden, um den &#196;nderungen der Modul-API gerecht zu werden.</p>
366
367    <p>Die Aktualisierung einer Unterversion auf eine andere (z.B. von 2.0.55
368    auf 2.0.57) ist einfacher. <code>make install</code> &#252;berschreibt
369    keine der bereits existierenden Dokumente, Log- und Konfigurationsdateien.
370    Ausserdem bem&#252;hen sich die Entwickler, inkompatible &#196;nderungen
371    der <code class="program"><a href="./programs/configure.html">configure</a></code>-Optionen, der Laufzeitkonfiguration sowie
372    der Modul-API zu vermeiden. In den meisten F&#228;llen sollten Sie in der
373    Lage sein, den gleichen <code class="program"><a href="./programs/configure.html">configure</a></code>-Befehl, die gleiche
374    Konfiguration und die gleichen Module wieder zu verwenden. (Das gilt erst
375    seit Version 2.0.41 -- fr&#252;here Versionen enthielten noch inkompatible
376    &#196;nderungen).</p>
377
378    <p>Um auf eine neue Unterversion zu aktualisieren, suchen Sie zun&#228;chst
379    die Datei <code>config.nice</code> im <code>build</code>-Verzeichnis
380    Ihrer Serverinstallation oder im Wurzelverzeichnis des Quelltextbaums
381    der alten Installation. Die Datei enth&#228;lt den genauen
382    <code class="program"><a href="./programs/configure.html">configure</a></code>-Befehl, der verwendet wurde, um den
383    Quellcode zu konfigurieren. Um jetzt von einer Version auf die
384    n&#228;chste zu aktualisieren, kopieren Sie einfach die
385    <code>config.nice</code> in das Verzeichnis der neuen Version,
386    passen sie bei Bedarf an, und f&#252;hren Sie sie aus:</p>
387
388    <div class="example"><p><code>
389      $ ./config.nice<br />
390      $ make<br />
391      $ make install<br />
392      $ <var>PREFIX</var>/bin/apachectl stop<br />
393      $ <var>PREFIX</var>/bin/apachectl start<br />
394    </code></p></div>
395
396    <div class="warning">Sie sollten jede neue Version immer in Ihrer Umgebung
397    testen, bevor Sie sie produktiv schalten. Beispielsweise k&#246;nnen Sie
398    die neue Version neben der alten installieren, indem Sie ein anderes
399    <code>--prefix</code> und einen anderen Port w&#228;hlen (durch Anpassen der
400    <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>-Direktive). So
401    k&#246;nnen Sie auf eventuelle Inkompatibilit&#228;ten testen, bevor Sie
402    endg&#252;ltig die neue Version verwenden.</div>
