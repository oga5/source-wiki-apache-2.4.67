43<h2><a name="definition" id="definition">What is a Handler</a></h2>
44    
45    <table class="related"><tr><th>Related Modules</th><th>Related Directives</th></tr><tr><td><ul><li><code class="module"><a href="./mod/mod_actions.html">mod_actions</a></code></li><li><code class="module"><a href="./mod/mod_asis.html">mod_asis</a></code></li><li><code class="module"><a href="./mod/mod_cgi.html">mod_cgi</a></code></li><li><code class="module"><a href="./mod/mod_imagemap.html">mod_imagemap</a></code></li><li><code class="module"><a href="./mod/mod_info.html">mod_info</a></code></li><li><code class="module"><a href="./mod/mod_mime.html">mod_mime</a></code></li><li><code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code></li><li><code class="module"><a href="./mod/mod_status.html">mod_status</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/mod_actions.html#action">Action</a></code></li><li><code class="directive"><a href="./mod/mod_mime.html#addhandler">AddHandler</a></code></li><li><code class="directive"><a href="./mod/mod_mime.html#removehandler">RemoveHandler</a></code></li><li><code class="directive"><a href="./mod/core.html#sethandler">SetHandler</a></code></li></ul></td></tr></table>

47
48    <p>A "handler" is an internal Apache representation of the
49    action to be performed when a file is called. Generally, files
50    have implicit handlers, based on the file type. Normally, all
51    files are simply served by the server, but certain file types
52    are "handled" separately.</p>
53
54    <p>Handlers may also be configured explicitly,
55    based on either filename extensions or on location,
56    without relation to file type. This is
57    advantageous both because it is a more elegant solution, and
58    because it also allows for both a type <strong>and</strong> a
59    handler to be associated with a file. (See also <a href="mod/mod_mime.html#multipleext">Files with Multiple
60    Extensions</a>.)</p>
61
62    <p>Handlers can either be built into the server or included in
63    a module, or they can be added with the <code class="directive"><a href="./mod/mod_actions.html#action">Action</a></code> directive. The
64    built-in handlers in the standard distribution are as
65    follows:</p>
66
67    <ul>
68      <li><strong>default-handler</strong>: Send the file using the
69      <code>default_handler()</code>, which is the handler used by
70      default to handle static content. (core)</li>
71
72      <li><strong>send-as-is</strong>: Send file with HTTP headers
73      as is. (<code class="module"><a href="./mod/mod_asis.html">mod_asis</a></code>)</li>
74
75      <li><strong>cgi-script</strong>: Treat the file as a CGI
76      script. (<code class="module"><a href="./mod/mod_cgi.html">mod_cgi</a></code>)</li>
77
78      <li><strong>imap-file</strong>: Parse as an imagemap rule
79      file. (<code class="module"><a href="./mod/mod_imagemap.html">mod_imagemap</a></code>)</li>
80
81      <li><strong>server-info</strong>: Get the server's
82      configuration information. (<code class="module"><a href="./mod/mod_info.html">mod_info</a></code>)</li>
83
84      <li><strong>server-status</strong>: Get the server's status
85      report. (<code class="module"><a href="./mod/mod_status.html">mod_status</a></code>)</li>
86
87      <li><strong>type-map</strong>: Parse as a type map file for
88      content negotiation. (<code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code>)</li>
89    </ul>

92<h2><a name="examples" id="examples">Examples</a></h2>
93    
94
95    <h3><a name="example1" id="example1">Modifying static content using a CGI script</a></h3>
96      
97
98      <p>The following directives will cause requests for files with
99      the <code>html</code> extension to trigger the launch of the
100      <code>footer.pl</code> CGI script.</p>
101
102      <pre class="prettyprint lang-config">Action add-footer /cgi-bin/footer.pl
103AddHandler add-footer .html</pre>
104
105
106      <p>Then the CGI script is responsible for sending the
107      originally requested document (pointed to by the
108      <code>PATH_TRANSLATED</code> environment variable) and making
109      whatever modifications or additions are desired.</p>
110
111    
112    <h3><a name="example2" id="example2">Files with HTTP headers</a></h3>
113      
114
115      <p>The following directives will enable the
116      <code>send-as-is</code> handler, which is used for files which
117      contain their own HTTP headers. All files in the
118      <code>/web/htdocs/asis/</code> directory will be processed by
119      the <code>send-as-is</code> handler, regardless of their
120      filename extensions.</p>
121
122      <pre class="prettyprint lang-config">&lt;Directory "/web/htdocs/asis"&gt;
123    SetHandler send-as-is
124&lt;/Directory&gt;</pre>

130<h2><a name="programmer" id="programmer">Programmer's Note</a></h2>
131    
132
133    <p>In order to implement the handler features, an addition has
134    been made to the <a href="developer/API.html">Apache API</a> that
135    you may wish to make use of. Specifically, a new record has
136    been added to the <code>request_rec</code> structure:</p>
137
138    <pre class="prettyprint lang-c">char *handler</pre>
139
140
141    <p>If you wish to have your module engage a handler, you need
142    only to set <code>r-&gt;handler</code> to the name of the
143    handler at any time prior to the <code>invoke_handler</code>
144    stage of the request. Handlers are implemented as they were
145    before, albeit using the handler name instead of a content
146    type. While it is not necessary, the naming convention for
147    handlers is to use a dash-separated word, with no slashes, so
148    as to not invade the media type name-space.</p>
