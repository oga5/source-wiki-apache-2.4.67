34    <p>この文書は <code>mod_mmap_static</code> モジュールを Apache 2.0 用に移植した時に
35    学んだ経験をもとに書いた、最初の手引き書です。まだまだ完全じゃないし、

37    取っ掛りにはなるでしょう。</p>

44<h2><a name="easy" id="easy">簡単な変更点</a></h2>
45
46    <h3><a name="cleanup" id="cleanup">クリーンナップ ルーチン</a></h3>
47      <p>クリーンナップルーチンは <code>apr_status_t</code> 型である必要があります。
48      そして、apr_status_t 型の値を返さなくてはなりません。
49      クリーンナップ中のエラーを通知する必要がなければ、返り値は普通、
50      <code>ARP_SUCCESS</code> です。たとえエラーを通知したとしても、
51      すべてのコードがその通知をチェックしたり、
52      エラーに応じた動作をするわけではないことに気をつけてください。</p>
53    
54
55
56    <h3><a name="init" id="init">初期化ルーチン</a></h3>
57
58    <p>初期化ルーチンは処理全体から見てしっくりくるような意味を表すように、
59    名前が変更されました。ですから、<code>mmap_init</code> から <code>mmap_post_config</code>
60    のようにちょっと変更されました。
61    渡される引数は大幅に変更され、次のようになりました。</p>
62
63      <ul>
64        <li><code>apr_pool_t *p</code></li>
65        <li><code>apr_pool_t *plog</code></li>
66        <li><code>apr_pool_t *ptemp</code></li>
67        <li><code>server_rec *s</code></li>
68      </ul>
69    
70
71    <h3><a name="datatypes" id="datatypes">データ型</a></h3>
72      <p>データ型のほとんどは <a href="http://apr.apache.org/">APR</a> に移されました。つまり、
73      いくつかの名前が前述のように変更されています。
74      施すべき変更点の簡単な一覧を以下に示します。</p>
75
76      <ul>
77        <li><code>pool</code> becomes <code>apr_pool_t</code></li>
78        <li><code>table</code> becomes <code>apr_table_t</code></li>
79      </ul>

83<h2><a name="messy" id="messy">もっと厄介な変更点…</a></h2>
84
85    <h3><a name="register-hooks" id="register-hooks">フックの登録</a></h3>
86      <p>新しいアーキテクチャでは作成した関数を呼び出すのに
87      一連のフックを使用します。このフックは、新しい関数
88      <code>static void register_hooks(void)</code> を使って登録するよう、
89      モジュールに書き足さなくてはなりません。
90      この関数は、なにをすべきか一旦理解してしまえば、
91      十分にわかりやすいものです。
92      リクエストの処理のあるステージで呼び出さなくてはならない
93      関数は登録する必要があります。ハンドラは登録する必要はありません。
94      関数を登録できるフェーズはたくさんあります。
95      それぞれのフェーズで、関数を呼び出す相対的な順番は、
96      かなりの程度制御できます。</p>
97
98    <p>以下は、<code>mod_mmap_static</code> に追加したコードです:</p>
99
100      <div class="example"><pre>static void register_hooks(void)
101{
102    static const char * const aszPre[]={ "http_core.c",NULL };
103    ap_hook_post_config(mmap_post_config,NULL,NULL,HOOK_MIDDLE);
104    ap_hook_translate_name(mmap_static_xlat,aszPre,NULL,HOOK_LAST);
105};</pre></div>
106
107    <p>ここでは呼びだすべき二つの関数を登録しています。一つは
108    <code>post_config</code> ステージ用 (ほとんどすべてのモジュール
109    はこれが必要です) で、もう一つは <code>translate_name</code> フェーズ用です。
110    それぞれの関数は名前は違うけれども形式は同じであることに注意してください。
111    それでは、形式はどのようになっているでしょうか?</p>
112
113      <div class="example"><p><code>
114        ap_hook_<var>phase_name</var>(<var>function_name</var>,
115        <var>predecessors</var>, <var>successors</var>, <var>position</var>);
116      </code></p></div>
117
118    <p>三つの位置が定義されています…</p>
119
120      <ul>
121        <li><code>HOOK_FIRST</code></li>
122        <li><code>HOOK_MIDDLE</code></li>
123        <li><code>HOOK_LAST</code></li>
124      </ul>
125
126    <p>位置を定義するには、上記の「位置」を指定し、
127    修飾子である「先行」と「後行」で手を加えます。
128   「先行」「後行」は、呼ばれるべき関数のリストです。
129   「先行」は関数の実行前に呼ばれるもので、
130   「後行」は実行後に呼ばれるものです。</p>
131
132    <p><code>mod_mmap_static</code> の場合、<code>post_config</code>
133    ステージでは必要ありませんが、
134    <code>mmap_static_xlat</code> が core モジュールが名前の変換を実行した後に
135    <strong>呼ばれなければなりません</strong>。
136    そこで aszPre を使って <code>HOOK_LAST</code> の修飾子を定義しています。</p>
137  
138
139    <h3><a name="moddef" id="moddef">モジュールの定義</a></h3>
140      <p>モジュールの定義を作成する際に注意しなければならない
141      ステージの数は激減しています。古い定義は次のようになっていました。</p>
142
143      <div class="example"><pre>module MODULE_VAR_EXPORT <var>module_name</var>_module =
144{
145    STANDARD_MODULE_STUFF,
146    /* initializer */
147    /* dir config creater */
148    /* dir merger --- default is to override */
149    /* server config */
150    /* merge server config */
151    /* command handlers */
152    /* handlers */
153    /* filename translation */
154    /* check_user_id */
155    /* check auth */
156    /* check access */
157    /* type_checker */
158    /* fixups */
159    /* logger */
160    /* header parser */
161    /* child_init */
162    /* child_exit */
163    /* post read-request */
164};</pre></div>
165
166    <p>新しい構造体はとってもシンプルです…</p>
167      <div class="example"><pre>module MODULE_VAR_EXPORT <var>module_name</var>_module =
168{
169    STANDARD20_MODULE_STUFF,
170    /* create per-directory config structures */
171    /* merge per-directory config structures  */
172    /* create per-server config structures    */
173    /* merge per-server config structures     */
174    /* command handlers */
175    /* handlers */
176    /* register hooks */
177};</pre></div>
178
179    <p>このうちのいくつかは古いものから新しいものに直接読み替えられるもので、
180    いくつかはそうではありません。どうすればいいのかを要約してみます。</p>
181
182    <p>直接読み替えられるステージ:</p>
183
184    <dl>
185      <dt><code>/* ディレクトリ設定作成関数 */</code></dt>
186      <dd><code>/* ディレクトリ毎設定構造体作成 */</code></dd>
187
188      <dt><code>/* サーバ設定作成関数 */</code></dt>
189      <dd><code>/* サーバ毎設定構造体作成 */</code></dd>
190
191      <dt><code>/* ディレクトリ設定マージ関数 */</code></dt>
192      <dd><code>/* ディレクトリ毎設定構造体マージ */</code></dd>
193
194      <dt><code>/* サーバ設定マージ関数 */</code></dt>
195      <dd><code>/* サーバ毎設定構造体作成マージ */</code></dd>
196
197      <dt><code>/* コマンド・テーブル */</code></dt>
198      <dd><code>/* コマンド apr_table_t */</code></dd>
199
200      <dt><code>/* ハンドラ */</code></dt>
201      <dd><code>/* ハンドラ */</code></dd>
202    </dl>
203
204    <p>古い関数の残りのものはフックとして登録されるべきです。
205    現時点で次のようなフック・ステージが定義されています…</p>
206
207    <dl>
208      <dt><code>ap_hook_post_config</code></dt>
209      <dd>(以前の <code>_init</code> ルーチンが登録されるべき場所です)</dd>
210
211      <dt><code>ap_hook_http_method</code></dt>
212      <dd>(リクエストから HTTP メソッドを取得します (互換用))</dd>
213
214      <dt><code>ap_hook_open_logs</code></dt>
215      <dd>(特定のログのオープン)</dd>
216
217      <dt><code>ap_hook_auth_checker</code></dt>
218      <dd>(リソースが権限を必要とするかどうかの確認)</dd>
219
220      <dt><code>ap_hook_access_checker</code></dt>
221      <dd>(モジュール固有の制約の確認)</dd>
222
223      <dt><code>ap_hook_check_user_id</code></dt>
224      <dd>(ユーザ ID とパスワードの確認)</dd>
225
226      <dt><code>ap_hook_default_port</code></dt>
227      <dd>(サーバのデフォルト・ポートの取得)</dd>
228
229      <dt><code>ap_hook_pre_connection</code></dt>
230      <dd>(処理の直前に必要なことを実行。ただし accept 直後に呼ばれる)</dd>
231
232      <dt><code>ap_hook_process_connection</code></dt>
233      <dd>(プロトコルの処理)</dd>
234
235      <dt><code>ap_hook_child_init</code></dt>
236      <dd>(子プロセル起動直後)</dd>
237
238      <dt><code>ap_hook_create_request</code></dt>
239      <dd>(??)</dd>
240
241      <dt><code>ap_hook_fixups</code></dt>
242      <dd>(応答内容の生成を変更するラスト・チャンス)</dd>
243
244      <dt><code>ap_hook_handler</code></dt>
245      <dd>(応答内容の生成)</dd>
246
247      <dt><code>ap_hook_header_parser</code></dt>
248      <dd>(モジュールにヘッダの照会をさせる。ほとんどのモジュールでは使われません。post_read_request を使います)</dd>
249
250      <dt><code>ap_hook_insert_filter</code></dt>
251      <dd>(フィルタ・チェインにフィルタを挿入)</dd>
252
253      <dt><code>ap_hook_log_transaction</code></dt>
254      <dd>(リクエストについての情報を記録する)</dd>
255
256      <dt><code>ap_hook_optional_fn_retrieve</code></dt>
257      <dd>(オプションとして登録された関数の取得)</dd>
258
259      <dt><code>ap_hook_post_read_request</code></dt>
260      <dd>(リクエストを読みこんだ後、他のフェーズの前に呼ばれる)</dd>
261
262      <dt><code>ap_hook_quick_handler</code></dt>
263      <dd>リクエストの処理が始まる前に呼ばれる。キャッシュモジュールが
264      使用している</dd>
265
266      <dt><code>ap_hook_translate_name</code></dt>
267      <dd>(URI をファイル名に変換する)</dd>
268
269      <dt><code>ap_hook_type_checker</code></dt>
270      <dd>(文書型の決定と設定。あるいはその片方)</dd>
271    </dl>
