34    <p>In general, a hook function is one that the Apache HTTP Server
35    will call at some point during the processing of a request.
36    Modules can provide functions that are called, and specify when
37    they get called in comparison to other modules.</p>

45<h2><a name="corehooks" id="corehooks">Core Hooks</a></h2>
46    <p>The httpd's core modules offer a predefinined list of hooks
47    used during the standard <a href="./request.html">request processing</a>
48    phase. Creating a new hook will expose a function that 
49    implements it (see sections below) but it is essential to understand that you will not 
50    extend the httpd's core hooks. Their presence and order in the request processing is in fact 
51    a consequence of how they are called in <code>server/request.c</code> 
52    (check <a href="./modguide.html#hooking">this section</a> 
53    for an overview). The core hooks are listed in the 
54    <a href="https://ci.apache.org/projects/httpd/trunk/doxygen/group__hooks.html">doxygen documentation</a>.</p>
55
56    <p>Reading <a href="./modguide.html">guide for developing modules</a> and 
57    <a href="./request.html">request processing</a> before proceeding is 
58    highly recommended.
59    </p>

62<h2><a name="create" id="create">Creating a hook function</a></h2>
63    <p>In order to create a new hook, four things need to be
64    done:</p>
65
66    <h3><a name="create-declare" id="create-declare">Declare the hook function</a></h3>
67      <p>Use the <code>AP_DECLARE_HOOK</code> macro, which needs to be given
68      the return type of the hook function, the name of the hook, and the
69      arguments. For example, if the hook returns an <code>int</code> and
70      takes a <code>request_rec *</code> and an <code>int</code> and is
71      called <code>do_something</code>, then declare it like this:</p>
72      <pre class="prettyprint lang-c">AP_DECLARE_HOOK(int, do_something, (request_rec *r, int n))</pre>
73
74
75      <p>This should go in a header which modules will include if
76      they want to use the hook.</p>
77    
78
79    <h3><a name="create-create" id="create-create">Create the hook structure</a></h3>
80      <p>Each source file that exports a hook has a private structure
81      which is used to record the module functions that use the hook.
82      This is declared as follows:</p>
83
84      <pre class="prettyprint lang-c">APR_HOOK_STRUCT(
85  APR_HOOK_LINK(do_something)
86  ...
87)</pre>
88
89    
90
91    <h3><a name="create-implement" id="create-implement">Implement the hook caller</a></h3>
92      <p>The source file that exports the hook has to implement a
93      function that will call the hook. There are currently three
94      possible ways to do this. In all cases, the calling function is
95      called <code>ap_run_<var>hookname</var>()</code>.</p>
96
97      <h4>Void hooks</h4>
98        <p>If the return value of a hook is <code>void</code>, then all the
99        hooks are called, and the caller is implemented like this:</p>
100
101        <pre class="prettyprint lang-c">AP_IMPLEMENT_HOOK_VOID(do_something, (request_rec *r, int n), (r, n))</pre>
102
103
104        <p>The second and third arguments are the dummy argument
105        declaration and the dummy arguments as they will be used when
106        calling the hook. In other words, this macro expands to
107        something like this:</p>
108
109        <pre class="prettyprint lang-c">void ap_run_do_something(request_rec *r, int n)
110{
111    ...
112    do_something(r, n);
113}</pre>
114
115      
116
117      <h4>Hooks that return a value</h4>
118        <p>If the hook returns a value, then it can either be run until
119        the first hook that does something interesting, like so:</p>
120
121        <pre class="prettyprint lang-c">AP_IMPLEMENT_HOOK_RUN_FIRST(int, do_something, (request_rec *r, int n), (r, n), DECLINED)</pre>
122
123
124        <p>The first hook that does <em>not</em> return <code>DECLINED</code>
125        stops the loop and its return value is returned from the hook
126        caller. Note that <code>DECLINED</code> is the traditional
127        hook return value meaning "I didn't do anything", but it can be
128        whatever suits you.</p>
129
130        <p>Alternatively, all hooks can be run until an error occurs.
131        This boils down to permitting <em>two</em> return values, one of
132        which means "I did something, and it was OK" and the other
133        meaning "I did nothing". The first function that returns a
134        value other than one of those two stops the loop, and its
135        return is the return value. Declare these like so:</p>
136
137        <pre class="prettyprint lang-c">AP_IMPLEMENT_HOOK_RUN_ALL(int, do_something, (request_rec *r, int n), (r, n), OK, DECLINED)</pre>
138
139
140        <p>Again, <code>OK</code> and <code>DECLINED</code> are the traditional
141        values. You can use what you want.</p>
142      
143    
144
145    <h3><a name="create-call" id="create-call">Call the hook callers</a></h3>
146      <p>At appropriate moments in the code, call the hook caller,
147      like so:</p>
148
149      <pre class="prettyprint lang-c">int n, ret;
150request_rec *r;
151
152ret=ap_run_do_something(r, n);</pre>
153

157<h2><a name="hooking" id="hooking">Hooking the hook</a></h2>
158    <p>A module that wants a hook to be called needs to do two
159    things.</p>
160
161    <h3><a name="hooking-implement" id="hooking-implement">Implement the hook function</a></h3>
162      <p>Include the appropriate header, and define a static function
163      of the correct type:</p>
164
165      <pre class="prettyprint lang-c">static int my_something_doer(request_rec *r, int n)
166{
167    ...
168    return OK;
169}</pre>
170
171    
172
173    <h3><a name="hooking-add" id="hooking-add">Add a hook registering function</a></h3>
174      <p>During initialisation, the server will call each modules hook
175      registering function, which is included in the module
176      structure:</p>
177
178      <pre class="prettyprint lang-c">static void my_register_hooks()
179{
180    ap_hook_do_something(my_something_doer, NULL, NULL, APR_HOOK_MIDDLE);
181}
182
183mode MODULE_VAR_EXPORT my_module =
184{
185    ...
186    my_register_hooks       /* register hooks */
187};</pre>
188
189    
190
191    <h3><a name="hooking-order" id="hooking-order">Controlling hook calling order</a></h3>
192      <p>In the example above, we didn't use the three arguments in
193      the hook registration function that control calling order of 
194      all the functions registered within the hook.
195      There are two mechanisms for doing this. The first, rather
196      crude, method, allows us to specify roughly where the hook is
197      run relative to other modules. The final argument control this.
198      There are three possible values: <code>APR_HOOK_FIRST</code>,
199      <code>APR_HOOK_MIDDLE</code> and <code>APR_HOOK_LAST</code>.</p>
200
201      <p>All modules using any particular value may be run in any
202      order relative to each other, but, of course, all modules using
203      <code>APR_HOOK_FIRST</code> will be run before <code>APR_HOOK_MIDDLE</code>
204      which are before <code>APR_HOOK_LAST</code>. Modules that don't care
205      when they are run should use <code>APR_HOOK_MIDDLE</code>. <em>These
206      values are spaced out, so that positions like <code>APR_HOOK_FIRST-2</code>
207      are possible to hook slightly earlier than other functions.</em></p>
208
209      <p>Note that there are two more values,
210      <code>APR_HOOK_REALLY_FIRST</code> and <code>APR_HOOK_REALLY_LAST</code>. These
211      should only be used by the hook exporter.</p>
212
213      <p>The other method allows finer control. When a module knows
214      that it must be run before (or after) some other modules, it
215      can specify them by name. The second (third) argument is a
216      NULL-terminated array of strings consisting of the names of
217      modules that must be run before (after) the current module. For
218      example, suppose we want "mod_xyz.c" and "mod_abc.c" to run
219      before we do, then we'd hook as follows:</p>
220
221      <pre class="prettyprint lang-c">static void register_hooks()
222{
223    static const char * const aszPre[] = { "mod_xyz.c", "mod_abc.c", NULL };
224
225    ap_hook_do_something(my_something_doer, aszPre, NULL, APR_HOOK_MIDDLE);
226}</pre>
227
228
229      <p>Note that the sort used to achieve this is stable, so
230      ordering set by <code>APR_HOOK_<var>ORDER</var></code> is preserved, as far
231      as is possible.</p>
