35    <p>On Windows, Apache is normally run as a service.
36    For details, see <a href="platform/windows.html#winsvc">Running Apache as a Service</a>.
37    </p>
38
39    <p>On Unix, the <code class="program"><a href="./programs/httpd.html">httpd</a></code> program
40    is run as a daemon that executes continuously in the
41    background to handle requests.  This document describes how
42    to invoke <code class="program"><a href="./programs/httpd.html">httpd</a></code>.</p>

51<h2><a name="startup" id="startup">How Apache Starts</a></h2>
52
53    <p>If the <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
54    specified in the configuration file is default of 80 (or any other
55    port below 1024), then it is necessary to have root privileges in
56    order to start apache, so that it can bind to this privileged
57    port. Once the server has started and performed a few preliminary
58    activities such as opening its log files, it will launch several
59    <em>child</em> processes which do the work of listening for and
60    answering requests from clients. The main <code>httpd</code>
61    process continues to run as the root user, but the child processes
62    run as a less privileged user. This is controlled by the selected
63    <a href="mpm.html">Multi-Processing Module</a>.</p>
64
65    <p>The recommended method of invoking the <code class="program"><a href="./programs/httpd.html">httpd</a></code>
66    executable is to use the <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> control script. This
67    script sets certain environment variables that are necessary for
68    <code class="program"><a href="./programs/httpd.html">httpd</a></code> to function correctly under some operating
69    systems, and then invokes the <code class="program"><a href="./programs/httpd.html">httpd</a></code> binary.
70    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> will pass through any command line
71    arguments, so any <code class="program"><a href="./programs/httpd.html">httpd</a></code> options may also be used with
72    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code>.  You may also directly edit the
73    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> script by changing the <code>HTTPD</code>
74    variable near the top to specify the correct location of the
75    <code class="program"><a href="./programs/httpd.html">httpd</a></code> binary and any command-line arguments that you
76    wish to be <em>always</em> present.</p>
77
78    <p>The first thing that <code class="program"><a href="./programs/httpd.html">httpd</a></code> does when it is
79    invoked is to locate and read the <a href="configuring.html">configuration file</a>
80    <code>httpd.conf</code>. The location of this file is set at
81    compile-time, but it is possible to specify its location at run
82    time using the <code>-f</code> command-line option as in</p>
83
84<div class="example"><p><code>/usr/local/apache2/bin/apachectl -f
85      /usr/local/apache2/conf/httpd.conf</code></p></div>
86
87    <p>If all goes well during startup, the server will detach from
88    the terminal and the command prompt will return almost
89    immediately. This indicates that the server is up and running.
90    You can then use your browser to connect to the server and view
91    the test page in the <code class="directive"><a href="./mod/core.html#documentroot">DocumentRoot</a></code> directory.</p>

94<h2><a name="errors" id="errors">Errors During Start-up</a></h2>
95
96    <p>If Apache suffers a fatal problem during startup, it will
97    write a message describing the problem either to the console or
98    to the <code class="directive"><a href="./mod/core.html#errorlog">ErrorLog</a></code> before
99    exiting. One of the most common error messages is "<code>Unable
100    to bind to Port ...</code>". This message is usually caused by
101    either:</p>
102
103    <ul>
104      <li>Trying to start the server on a privileged port when not
105      logged in as the root user; or</li>
106
107      <li>Trying to start the server when there is another instance
108      of Apache or some other web server already bound to the same
109      Port.</li>
110    </ul>
111
112    <p>For further trouble-shooting instructions, consult the
113    Apache <a href="https://cwiki.apache.org/confluence/display/httpd/FAQ">FAQ</a>.</p>

116<h2><a name="boot" id="boot">Starting at Boot-Time</a></h2>
117
118    <p>If you want your server to continue running after a system
119    reboot, you should add a call to <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> to your
120    system startup files (typically <code>rc.local</code> or a file in
121    an <code>rc.N</code> directory). This will start Apache as
122    root. Before doing this ensure that your server is properly
123    configured for security and access restrictions.</p>
124
125    <p>The <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> script is designed to act like a
126    standard SysV init script; it can take the arguments
127    <code>start</code>, <code>restart</code>, and <code>stop</code>
128    and translate them into the appropriate signals to
129    <code class="program"><a href="./programs/httpd.html">httpd</a></code>.  So you can often simply link
130    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> into the appropriate init directory. But be
131    sure to check the exact requirements of your system.</p>
