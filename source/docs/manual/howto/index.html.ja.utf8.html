40<h2><a name="howto" id="howto">How-To / チュートリアル</a></h2>
41
42    
43
44    <dl>
45      <dt>認証と承認</dt>
46      <dd>
47        <p>認証とは、誰かが自分は誰であるかを名乗っているものを検証する
48        処理のことです。承認とは、誰かが望みの場所に辿り着けたり、
49        望みの情報を手に入れたりすることを許可する処理のことです。</p>
50
51        <p>参照: <a href="auth.html">認証と承認</a></p>
52      </dd>
53    </dl>
54
55    <dl>
56      <dt>アクセス制御</dt>
57      <dd>
58        <p>アクセス制御は、さまざまな条件でリソースに対するアクセスを
59        許可したり制限したりすることを指します。
60        実現方法には様々な異なる手法があります。</p>
61
62        <p>参照: <a href="access.html">アクセス制御</a></p>
63      </dd>
64    </dl>
65
66   <dl>
67      <dt>CGI による動的コンテンツ</dt>
68      <dd>
69        <p>CGI (Common Gateway Interface) はウェブサーバが外部のコンテンツ
70        生成プログラムとどのように相互動作をするかを定義します。
71        その外部プログラムは通常 CGI プログラムや CGI スクリプトと呼ばれます。
72        CGI はウェブサイトに動的なコンテンツを追加するための、
73        単純な方法です。この文書は Apache ウェブサーバに
74        CGI を設定し、CGI プログラムを書き始めるためのイントロダクションです。</p>
75
76        <p>参照: <a href="cgi.html">CGI: 動的コンテンツ</a></p>
77      </dd>
78    </dl>
79
80    <dl>
81      <dt><code>.htaccess</code> ファイル</dt>
82      <dd>
83        <p><code>.htaccess</code> ファイルはディレクトリ毎に設定を変更するための
84        方法を提供します。設定ディレクティブが書かれたファイルが、あるドキュメント
85        ディレクトリに置かれると、ディレクティブはそのディレクトリと
86        すべてのサブディレクトリに適用されます。</p>
87
88        <p>参照: <a href="htaccess.html"><code>.htaccess</code> ファイル</a></p>
89      </dd>
90    </dl>
91
92    <dl>
93      <dt>Server Side Includes イントロダクション</dt>
94      <dd>
95        <p>SSI (Server Side Includes) は HTML ページ中に書かれるディレクティブで、
96        ページが送られる時にサーバにより評価されます。これにより、ページ全体を
97        CGI プログラムで生成したり、他の動的な技術を使うことなく、既存の HTML
98        ページに動的に生成された内容を付加することができます。</p>
99
100        <p>参照: <a href="ssi.html">Server Side Includes (SSI)</a></p>
101      </dd>
102    </dl>
103
104    <dl>
105      <dt>ユーザ毎のウェブディレクトリ</dt>
106      <dd>
107        <p>複数ユーザの存在するシステムでは、それぞれのユーザは <code class="directive"><a href="../mod/mod_userdir.html#userdir">UserDir</a></code> ディレクティブを使うことによって
108        ホームディレクトリ上にウェブサイトを作成することができます。
109        URL <code>http://example.com/~username/</code> を訪れた人は
110        ユーザ "<code>username</code>" のホームディレクトリの、<code class="directive"><a href="../mod/mod_userdir.html#userdir">UserDir</a></code> ディレクティブで指定された
111       サブディレクトリからコンテンツを得ることになります。</p>
112
113        <p>参照: <a href="public_html.html">ユーザウェブディレクトリ (<code>public_html</code>)</a></p>
114      </dd>
115    </dl>
