46<h2><a name="introduction" id="introduction">介绍</a></h2>
47
48    <p>Apache HTTP 服务器被设计为一个功能强大，并且灵活的 web 服务器，
49    可以在很多平台与环境中工作。不同平台和不同的环境往往需要不同
50    的特性，或可能以不同的方式实现相同的特性最有效率。Apache httpd
51    通过模块化的设计来适应各种环境。这种设计允许网站管理员通过在
52    编译时或运行时，选择哪些模块将会加载在服务器中，来选择服务器特性。</p>
53
54    <p>Apache HTTP 服务器 2.0 扩展此模块化设计到最基本的 web 服务器功能。
55    它提供了可以选择的多处理模块(MPM)，用来绑定到网络端口上，接受请求，
56    以及调度子进程处理请求。</p>
57
58    <p>扩展到这一级别的服务器模块化设计，带来两个重要的好处:</p>
59
60    <ul>
61      <li>Apache httpd 能更优雅，更高效率的支持不同的平台。尤其是
62      Apache httpd 的 Windows 版本现在更有效率了，因为
63      <code class="module"><a href="./mod/mpm_winnt.html">mpm_winnt</a></code> 能使用原生网络特性取代在
64      Apache httpd 1.3 中使用的 POSIX 层。它也可以扩展到其它平台
65      来使用专用的 MPM。</li>
66
67      <li>Apache httpd 能更好的为有特殊要求的站点定制。例如，要求
68      更高伸缩性的站点可以选择使用线程的 MPM，即
69      <code class="module"><a href="./mod/worker.html">worker</a></code> 或 <code class="module"><a href="./mod/event.html">event</a></code>；
70      需要可靠性或者与旧软件兼容的站点可以使用
71      <code class="module"><a href="./mod/prefork.html">prefork</a></code>。</li>
72    </ul>
73
74    <p>在用户看来，MPM 很像其它 Apache httpd 模块。主要是区别是，在任何时间，
75    必须有一个，而且只有一个 MPM 加载到服务器中。可用的 MPM 列表位于
76    <a href="mod/">模块索引页面</a>。</p>

92<h2><a name="static" id="static">构建 MPM 为静态模块</a></h2>
93
94    <p>在全部平台中，MPM 都可以构建为静态模块。在构建时选择一种
95    MPM，链接到服务器中。如果要改变 MPM，必须重新构建。</p>
96
97    <p>为了使用指定的 MPM，请在执行 <code class="program"><a href="./programs/configure.html">configure</a></code> 脚本
98      时，使用参数 <code>--with-mpm=<em>NAME</em></code>。<em>NAME</em>
99      是指定的 MPM 名称。</p>
100
101    <p>编译完成后，可以使用 <code>./httpd -l</code> 来确定选择的 MPM。
102    此命令会列出编译到服务器程序中的所有模块，包括 MPM。</p>
103
104</div><div class="top"><a href="#page-header"><img alt="top" src="./images/up.gif" /></a></div>
105<div class="section">
106<h2><a name="dynamic" id="dynamic">构建 MPM 为动态模块</a></h2>
107
108    <p>在 Unix 或类似平台中，MPM 可以构建为动态模块，与其它动态模块一样在运行时加载。
109    构建 MPM 为动态模块允许通过修改 <code class="directive"><a href="./mod/mod_so.html#loadmodule">LoadModule</a></code>
110    指令内容来改变 MPM，而不用重新构建服务器程序。</p>
111
112    <p>在执行 <code class="program"><a href="./programs/configure.html">configure</a></code> 脚本时，使用
113    <code>--enable-mpms-shared</code> 选项可以启用此特性。
114    当给出的参数为 <code><em>all</em></code> 时，所有此平台支持的 MPM
115    模块都会被安装。还可以在参数中给出模块列表。</p>
116
117    <p>默认 MPM，可以自动选择或者在执行 <code class="program"><a href="./programs/configure.html">configure</a></code>
118    脚本时通过 <code>--with-mpm</code> 选项来指定，然后出现在生成的服务器配置文件中。
119    编辑 <code class="directive"><a href="./mod/mod_so.html#loadmodule">LoadModule</a></code> 指令内容可以选择不同的 MPM。</p>
