48<h2><a name="introduction" id="introduction">Einf&#252;hrung</a></h2>
49
50  <p>Der Apache HTTP Server wurde als leistungsf&#228;higer und flexibler Webserver
51    konzipiert, der auf einer Vielzahl von Plattformen in einer
52    Reihe unterschiedlicher Umgebungen arbeiten kann. Unterschiedliche
53    Plattformen und unterschiedliche Umgebungen verlangen oftmals verschiedene
54    F&#228;higkeiten oder kennen verschiedene Wege, die gleiche
55    Funktionalt&#228;t sehr effizient zu implementieren. Der Apache hat durch
56    seinen modularen Aufbau schon immer eine breite Auswahl von Umgebungen
57    unterst&#252;tzt. Dieses Design erlaubt es dem Webmaster, durch Auswahl der
58    Module, die zur Kompilierungszeit oder zur Laufzeit geladen werden, die
59    Features auszuw&#228;hlen, die in den Server intregiert werden.</p>
60
61  <p>Der Apache 2.0 erweitert dieses modulare Design auf die grundlegenden
62    Funktionen eines Webservers. Der Server wird mit einer Auswahl von
63    Multi-Processing-Modulen (MPMs) ausgeliefert, die f&#252;r die Bindung an
64    Netzwerkports der Maschine, die Annahme von Anfragen und die Abfertigung von
65    Kindprozessen zur Behandlung der Anfragen zust&#228;ndig sind.</p>
66
67  <p>Die Erweiterung des modularen Aufbaus auf diese Ebene des Servers
68    bringt zwei wesentliche Vorteile:</p>
69
70  <ul>
71    <li>Der Apache kann nun eine Vielfalt von Betriebssystemen sauberer und
72      effizienter unterst&#252;tzen. Insbesondere die Windows-Version des Apache
73      ist jetzt deutlich effizienter, da <code class="module"><a href="./mod/mpm_winnt.html">mpm_winnt</a></code> native
74      Netzwerkf&#228;higkeiten anstelle der im Apache 1.3 verwendeten
75      POSIX-Schicht benutzen kann. Dieser Vorteil gilt auch f&#252;r andere
76      Betriebssysteme, f&#252;r die spezielle MPMs implementiert sind.</li>
77    
78    <li>Der Server l&#228;&#223;t sich besser auf die Bed&#252;rfnisse der
79      jeweiligen Website anpassen. Sites beispielsweise, die eine hohe
80      Skalierbarkeit ben&#246;tigen, k&#246;nnen ein Threaded-MPM wie
81      <code class="module"><a href="./mod/worker.html">worker</a></code> oder <code class="module"><a href="./mod/event.html">event</a></code> w&#228;hlen,
82      w&#228;hrend Sites, die Stabilit&#228;t oder Kompatibilit&#228;t mit
83      &#228;lterer Software erfordern, <code class="module"><a href="./mod/prefork.html">prefork</a></code> w&#228;hlen
84      k&#246;nnen.</li>
85  </ul>
86
87  <p>Auf Anwenderebene erscheinen MPMs fast wie andere Apache-Module. Der
88    Hauptunterschied ist, dass jeweils nur ein einziges MPM in den Server
89    geladen werden kann. Die Liste der verf&#252;gbaren MPMs finden Sie im <a href="mod/">Modul-Index</a>.</p>

93<h2><a name="choosing" id="choosing">Auswahl eines MPMs</a></h2>
94
95  <p>MPMs m&#252;ssen w&#228;hrend der
96    <span class="transnote">(<em>Anm.d.&#220;.:</em> Quelltext-)</span>Konfiguration ausgew&#228;hlt und in den
97    Server einkompiliert werden. Compiler sind in der Lage eine Reihe von
98    Funktionen zu optimieren, wenn Threads verwendet werden. Sie k&#246;nnen
99    dies allerdings nur, wenn sie wissen, dass Threads benutzt werden.</p>
100
101  <p>Um das gew&#252;nschte MPM tats&#228;chlich auszuw&#228;hlen, verwenden Sie
102    beim <code class="program"><a href="./programs/configure.html">configure</a></code>-Skript das Argument
103    <code>--with-mpm=<em>NAME</em></code>. <em>NAME</em> ist der Name des
104    gew&#252;nschten MPMs.</p>
105
106  <p>Ist der Server kompiliert, so ist es mittels <code>./httpd -l</code>
107    m&#246;glich, das ausgew&#228;hlte MPM zu ermitteln. Dieser Befehl listet
108    alle in den Server einkompilierten Module auf, einschlie&#223;lich des
109    MPM.</p>
