45<h2><a name="core" id="core">Améliorations du Système de Base</a></h2>
46        
47
48        <dl>
49            <dt>Threading Unix</dt>
50
51            <dd>Sur les systèmes Unix qui supportent les threads
52	    POSIX, Apache httpd
53            peut à présent tourner en mode hybride multi-processus et 
54            multi-threadé, ce qui augmente l'extensibilité et la performance 
55            du serveur pour la plupart des configurations.</dd>
56
57            <dt>Nouveau Système de Compilation</dt>
58
59            <dd>Le processus de compilation a été refait de A à Z; 
60            il utilise à présent <code>autoconf</code> et <code>libtool</code>, 
61            ce qui rend la compilation d'Apache httpd plus familière aux utilisateurs 
62            d'autre logiciels de mème type.</dd>
63
64            <dt>Support Multiprotocole</dt>
65
66            <dd>Le serveur HTTP Apache dispose désormais de
67	    l'infrastructure nécessaire pour supporter
68            d'autres protocoles. Le module <code class="module"><a href="./mod/mod_echo.html">mod_echo</a></code> illustre ces 
69            possibilités.</dd>
70
71            <dt>Support amélioré des Plate-formes non-Unix</dt>
72        
73            <dd>Le serveur HTTP Apache 2.0 se montre plus rapide et plus stable sur les plate-formes 
74            non Unix, telles BeOS, OS/2, NetWare et Windows. L'apparition des 
75            <a href="mpm.html">Modules Multi-Processus</a> (MPMs), ainsi que de 
76            la bibliothèque "Apache Portable Runtime" (APR) permet à Apache de 
77            tirer parti des API natives de ces plate-formes, sans s'appuyer sur leurs 
78            couches POSIX souvent boguées et peu optimisées.</dd>
79
80            <dt>Nouvelle API d'Apache httpd</dt>
81
82            <dd>L'Interface de Programmation (API) des modules a beaucoup changé 
83            avec le passage à la version 2.0.
84            Les problèmes d'ordre et de priorité des modules, rencontrés 
85            avec la version 1.3, devraient maintenant être résolus. Apache 2.0 
86            gère ces problèmes de façon automatique. L'ordre des modules 
87            est géré au moyen de "crochets" (hooks), ce qui rend la gestion 
88            flexible. De nouveaux appels ont été également créés 
89            afin de permettre l'implémentation d'autres fonctions dans les modules, 
90            sans devoir corriger le noyau du serveur HTTP Apache.</dd>
91
92            <dt>Support IPv6</dt>
93
94            <dd>Sur les systèmes où la bibliothèque Apache Portable Runtime 
95            supporte IPv6, Apache peut par défaut écouter sur des interfaces 
96            de connexions IPv6. Les directives <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>, 
97            <code class="directive"><a href="./mod/core.html#namevirtualhost">NameVirtualHost</a></code> et 
98            <code class="directive"><a href="./mod/core.html#virtualhost">VirtualHost</a></code> supportent également 
99            les adresses IPv6 (comme par exemple, dans "<code>Listen[2001:db8::1]:8080</code>").</dd>
100
101            <dt>Filtering</dt>
102
103            <dd>Il est maintenant possible d'écrire des modules
104	    pour Apache  httpd pour filtrer
105            les flux de données entrant ou sortant du serveur. A titre d'exemple, 
106            il est possible de filtrer des directives Server Side Include de la sortie 
107            standard d'un script CGI, au moyen du filtre <code>INCLUDES</code> fourni
108            par le module <code class="module"><a href="./mod/mod_include.html">mod_include</a></code>. Le module
109            <code class="module"><a href="./mod/mod_ext_filter.html">mod_ext_filter</a></code> permet quant à lui l'utilisation comme
110            filtres de programmes externes à Apache, de la même manière 
111            qu'on peut utiliser des programmes CGI comme Handlers.</dd>
112
113            <dt>Réponses d'Erreurs Multilangues</dt>
114
115            <dd>Les messages d'erreur envoyés au navigateur existent à présent en
116            plusieurs langues avec des documents SSI. Ces messages peuvent être
117            personnalisés par l'administrateur afin de s'intégrer avec le site web.</dd>
118
119            <dt>Simplification de la Configuration</dt>
120
121            <dd>Beaucoup de directives, auparavant peu claires, ont été simplifiées.
122            Les directives <code>Port</code> et <code>BindAddress</code>, souvent
123            sources d'incompréhension, ont disparus. Désormais seule la directive 
124            <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code> sert de liaison pour les 
125            adresses IP; la directive <code class="directive"><a href="./mod/core.html#servername">ServerName</a></code> ne 
126            précise le nom du serveur et son port que pour les redirections et la
127            gestion des hôtes virtuels.</dd>
128
129            <dt>Support natif de l'Unicode sous Windows NT</dt>
130
131            <dd>Apache httpd 2.0 sur Windows NT utilise à présent l'utf-8 pour tous les 
132            noms de fichiers. Ces noms de fichiers sont directement traduits vers
133            l'encodage Unicode du système de fichiers, ce qui permet le support
134            multilangue pour toutes les installations sur la famille NT de Windows, y
135            compris Windows 2000 et Windows XP.<em>Ce support n'est pas fonctionnel
136            pour Windows 95, 98 ni ME, qui utilisent les pages de code locales pour
137            les accès au système de fichiers, comme auparavant.</em></dd>
138
139            <dt>Mise à jour de la Bibliothèque d'Expressions Rationnelles</dt>
140
141            <dd>Apache httpd 2.0 contient la <a href="http://www.pcre.org/">bibliothèque 
142            d'expressions rationnelles compatible Perl </a>(Perl Compatible Regular 
143            Expression Library - PCRE). Toutes les expressions rationnelles sont dont
144            gérées avec la syntaxe de Perl 5, plus puissante.</dd>
145
146        </dl>

149<h2><a name="module" id="module">Amélioration des Modules</a></h2>
150        
151
152        <dl>
153            <dt><code class="module"><a href="./mod/mod_ssl.html">mod_ssl</a></code></dt>
154
155            <dd>Apparu dans Apache httpd 2.0, ce module est une interface aux protocoles de 
156            chiffrement SSL/TLS fournis par OpenSSL.</dd>
157
158            <dt><code class="module"><a href="./mod/mod_dav.html">mod_dav</a></code></dt>
159
160            <dd>Apparu dans Apache httpd 2.0, ce module implémente les spécifications HTTP de 
161            gestion distribuée de versions et de rédaction (Distributed Authoring and 
162            Versioning - DAV), destinées à la mise en ligne et à la maintenance des 
163            contenus Web.</dd>
164
165            <dt><code class="module"><a href="./mod/mod_deflate.html">mod_deflate</a></code></dt>
166
167            <dd>Module apparu dans Apache httpd 2.0, mod_deflate permet aux navigateurs qui 
168            le supportent de demander la compression des contenus envoyés par le serveur. 
169            Cela a l'avantage de réduite l'occupation de la bande passante.</dd>
170
171            <dt><code class="module">mod_auth_ldap</code></dt>
172
173            <dd>Apparu dans Apache httpd 2.0.41, ce module permet aux administrateurs
174            d'utiliser un arbre LDAP pour gérer la base d'utilisateurs pour les 
175            Authentifications Basiques HTTP. Un module voisin,
176            <code class="module"><a href="./mod/mod_ldap.html">mod_ldap</a></code>, permet de globaliser les connexions à l'arbre LDAP
177            et de garder en mémoire cache ces accès.</dd>
178    
179            <dt><code class="module"><a href="./mod/mod_auth_digest.html">mod_auth_digest</a></code></dt>
180
181            <dd>Améliore les fonctions de cache sur une session entre les différents
182            processus, en utilisant de la mémoire partagée.</dd>
183
184            <dt><code class="module"><a href="./mod/mod_charset_lite.html">mod_charset_lite</a></code></dt>
185
186            <dd>Apparu dans Apache httpd 2.0, ce module expérimental permet la conversion 
187            et l'enregistrement entre jeux de caractères.</dd>
188
189            <dt><code class="module"><a href="./mod/mod_file_cache.html">mod_file_cache</a></code></dt>
190
191            <dd>Apparu dans Apache httpd 2.0, ce module implémente les fonctionnalités du 
192            module <code>mod_mmap_static</code> présent du serveur
193	    HTTP Apache 1.3, et offre des 
194            fonctions plus avancées pour la gestion du cache.</dd>
195
196            <dt><code class="module"><a href="./mod/mod_headers.html">mod_headers</a></code></dt>
197
198            <dd>Ce module gagne beaucoup de flexibilité avec Apache
199	    httpd 2.0 : on peut 
200            désormais l'utiliser pour modifier les en-têtes des requêtes 
201            utilisés par <code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code>, et pour positionner les 
202            en-têtes des réponses de manière conditionnelle.</dd>
203
204            <dt><code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code></dt>
205
206            <dd>Le module proxy a été réécrit de A à Z. Il tire 
207            maintenant avantage de la nouvelle infrastructure de filtrage, et implémente 
208            un mandataire plus fiable, et conforme aux normes HTTP/1.1. De nouvelles 
209            sections de configuration ajoutées à 
210            <code class="directive"><a href="./mod/mod_proxy.html#proxy">&lt;Proxy&gt;</a></code> 
211            donnent un contrôle plus lisible et un traitement plus rapide des requêtes 
212            mandatées ; les configurations surchargées <code>&lt;Directory 
213            "proxy:..."&gt;</code> ne sont pas supportées. Le module a aussi été 
214            fragmenté en plusieurs modules qui gèrent chacun leur protocole : 
215            <code>proxy_connect</code>, <code>proxy_ftp</code> et 
216            <code>proxy_http</code>.</dd>
217
218            <dt><code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code></dt>
219
220            <dd>Une nouvelle directive, <code class="directive"><a href="./mod/mod_negotiation.html#forcelanguagepriority">ForceLanguagePriority</a></code> a été ajoutée, 
221            elle permet de garantir que le client reçoit un seul document dans tous les 
222            cas, au lieu de réponses NOT ACCEPTABLE ou MULTIPLE CHOICES. Les 
223            algorithmes gérant la négociation et les vues multiples (MultiViews) ont 
224            été nettoyés et donnent des réponses plus logiques. Un nouveau format de 
225            carte de types (map type) qui peut gérer le contenu de documents a 
226            aussi été ajouté.</dd>
227    
228            <dt><code class="module"><a href="./mod/mod_autoindex.html">mod_autoindex</a></code></dt>
229
230            <dd>Les listes auto-générées par Autoindex sont à présent 
231            configurables, et peuvent utiliser des tables HTML pour une mise en forme plus propre. 
232            L'ordre d'affichage des fichiers est également finement paramètrable, 
233            comme pour le tri par version, et le filtrage par caractères jokers du 
234            listage du répertoire.</dd>
235
236            <dt><code class="module"><a href="./mod/mod_include.html">mod_include</a></code></dt>
237
238            <dd>De nouvelles directives permettent de modifier la valeur par défaut 
239            des drapeaux <em>start</em> et <em>end</em> des éléments SSI. Ces directives 
240            permettent à la configuration d'affichage de dates et heures d'être 
241            effectuée dans le fichier de configuration principal, plutôt que dans le 
242            document SSI. Les réponses données par des recherches par expressions 
243            rationnelles (qui gèrent à présent les regex Perl) sont 
244            recupérées au moyen des variables <code>$0</code> à <code>$9</code>.</dd>
245
246            <dt><code class="module">mod_auth_dbm</code></dt>
247
248            <dd>Plusieurs bases de données DBM sont supportées, et sélectionnables 
249            via la directive <code class="directive">AuthDBMType</code>.</dd>
250        </dl>
