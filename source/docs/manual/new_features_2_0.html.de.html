43<h2><a name="core" id="core">Core-Erweiterungen</a></h2>
44    
45
46    <dl>
47      <dt><strong>Unix-Threading</strong></dt>
48
49      <dd>Auf Unix-Systemen mit Unterst&#252;tzung f&#252;r
50      POSIX-Threads, kann der Apache httpd jetzt in einem Multi-Process,
51      Multi-Threaded Hybrid-Mode gestartet werden. Dies verbessert die
52      Skalierf&#228;higkeit f&#252;r viele, jedoch nicht unbedingt alle
53      Konfigurationen.</dd>
54
55      <dt><strong>Neues Build-System</strong></dt>
56
57      <dd>Das Build-System wurde komplett auf der Basis von
58      <code>autoconf</code> und <code>libtool</code> neu geschrieben.
59      Dadurch wird das Konfigurationssystem des Apache httpd dem vieler
60      anderer Packages &#228;hnlicher.</dd>
61
62      <dt><strong>Multi-Protokoll-Unterst&#252;tzung</strong></dt>
63
64      <dd>Der Apache HTTP Server stellt jetzt die notwendigen
65      Grundfunktionalit&#228;ten bereit, um mehrere Protokolle
66      unterst&#252;tzen und verarbeiten zu k&#246;nnen.
67      <code class="module"><a href="./mod/mod_echo.html">mod_echo</a></code> wurde hierf&#252;r als Beispiel
68      geschrieben.</dd>
69
70      <dt><strong>Bessere Unterst&#252;tzung von
71      Nicht-Unix-Plattformen</strong></dt>
72
73      <dd>Der Apache HTTP Server 2.0 ist schneller und stabiler auf
74      Nicht-Unix-Plattformen wie BeOS, OS/2 und Windows. Mit der
75      Einf&#252;hrung von Plattform-spezifischen <a href="mpm.html">Multi-Processing Modulen</a> (MPMs) und der Apache
76      Portable Runtime (APR), sind diese Plattformen jetzt in ihrem
77      nativen API implementiert, wodurch die Verwendung der h&#228;ufig
78      fehlerbehafteten und schlecht funktionierenden
79      POSIX-Emulation-Layer vermieden wird.</dd>
80
81      <dt><strong>Neues Apache-httpd API</strong></dt>
82
83      <dd>Das API f&#252;r Module hat sich in 2.0 stark ver&#228;ndert.
84      Die meisten der Sortierungs-/Priorit&#228;tsprobleme von Modulen bei
85      1.3 sollten nun verschwunden sein. In 2.0 wird hiervon vieles
86      automatisch durchgef&#252;hrt. Die Modulsortierung wird jetzt
87      &#252;ber einen pre-hook vorgenommen, um mehr Flexibilit&#228;t
88      zu bieten. Au&#223;erdem wurden neue API-Calls hinzugef&#252;gt,
89      die zus&#228;tzliche Modulf&#228;higkeiten zur Verf&#252;gung stellen,
90      ohne den Kern des Apache HTTP Servers anpassen zu m&#252;ssen.</dd>
91
92      <dt><strong>IPv6-Unterst&#252;tzung</strong></dt>
93
94      <dd>Auf Systemen, bei denen die zugrundeliegende Apache Portable
95      Runtime-Bibliothek IPv6 unterst&#252;tzt, bekommt der Apache httpd
96      standarm&#228;&#223;ig IPv6 Listening Sockets. Zus&#228;tzlich
97      unterst&#252;tzen die Konfigurationsanweisungen <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>, <code class="directive"><a href="./mod/core.html#namevirtualhost">NameVirtualHost</a></code> und <code class="directive"><a href="./mod/core.html#virtualhost">VirtualHost</a></code> numerische IPv6-Adressangaben
98      (z.B., "<code>Listen [2001:db8::1]:8080</code>").</dd>
99
100      <dt><strong>Filterung</strong></dt>
101
102      <dd>Apache-httpd-Module k&#246;nnen jetzt als Filter entwickelt
103      und zur Filterung des rein- und rausgehenden Datenstroms des
104      Servers eingesetzt werden. Hierdurch kann beispielsweise die
105      Ausgabe von CGI-Skripten durch den <code>INCLUDES</code>-Filter
106      von <code class="module"><a href="./mod/mod_include.html">mod_include</a></code> bearbeitet werden und so
107      Server-Side Include-Anweisungen ausgef&#252;hrt werden. Das Modul
108      <code class="module"><a href="./mod/mod_ext_filter.html">mod_ext_filter</a></code> erlaubt externen Programmen als
109      Filter zu agieren, in der gleichen Weise wie CGI-Programme als
110      Eingabe dienen k&#246;nnen.</dd>
111
112      <dt><strong>Mehrsprachige Fehlermeldungen</strong></dt>
113
114      <dd>Fehlermeldungen die an den Browser rausgehen, stehen jetzt als
115      SSI-Dokumente in verschiedenen Sprachen zur Verf&#252;gung. Sie
116      k&#246;nnen bei Bedarf durch den Administrator angepasst werden,
117      um ein einheitliches Design zu erreichen.</dd>
118
119      <dt><strong>Vereinfachte Konfiguration</strong></dt>
120
121      <dd>Viele der verwirrenden Konfigurationsanweisungen wurden vereinfacht.
122      Die oft f&#252;r Verwirrung sorgenden <code>Port</code>- und
123      <code>BindAddress</code>-Anweisungen wurden entfernt.
124      Ausschlie&#223;lich die <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>-Anweisung wird nun zum
125      Setzen von IP-Addressen und Portnummern benutzt.
126      Der Servername und die Portnummer, die f&#252;r Weiterleitungen und
127      zur Erkennung virtueller Server verwendet werden, werden &#252;ber
128      die <code class="directive"><a href="./mod/core.html#servername">ServerName</a></code>-Anweisung
129      konfiguriert.</dd>
130
131      <dt><strong>Native Windows NT Unicode-Unterst&#252;tzung</strong></dt>
132
133      <dd>Der Apache httpd 2.0 auf Windows NT benutzt jetzt utf-8
134      f&#252;r alle Dateinamen-Kodierungen. Diese werden direkt auf das
135      zugrundeliegende Unicode-Dateisystem abgebildet, wodurch
136      Mehrsprach-Unterst&#252;tzung f&#252;r alle Windows NT-basierten
137      Installationen, inklusive Windows 2000 und Windows XP, zur
138      Verf&#252;gung gestellt wird.  <em>Diese Unterst&#252;tzung ist
139      nicht auf Windows 95, 98 oder ME verf&#252;gbar. Hier wird
140      weiterhin die jeweils lokale Codepage des Rechners f&#252;r den
141      Zugriff auf das Dateisystem verwendet.</em></dd>
142
143      <dt>Bibliothek f&#252;r regul&#228;re Ausdr&#252;cke aktualisiert</dt>
144
145      <dd>Der Apache httpd 2.0 enth&#228;lt die <a href="http://www.pcre.org/">"Perl Compatible
146      Regular Expression Library"</a> (PCRE).
147      Bei der Auswertung aller regul&#228;ren Ausdr&#252;cke wird nun
148      die leistungsf&#228;higere Syntax von Perl 5 verwendet.</dd>
149
150    </dl>

153<h2><a name="module" id="module">Modul-Erweiterungen</a></h2>
154    
155
156    <dl>
157      <dt><code class="module"><a href="./mod/mod_ssl.html">mod_ssl</a></code></dt>
158
159      <dd>Neues Modul in Apache httpd 2.0. Dieses Modul ist ein
160      Interface zu den von OpenSSL bereitgestellten SSL/TLS
161      Verschl&#252;sselungs-Protokollen.</dd>
162
163      <dt><code class="module"><a href="./mod/mod_dav.html">mod_dav</a></code></dt>
164
165      <dd>Neues Modul in Apache httpd 2.0. Dieses Modul implementiert
166      die HTTP Distributed Authoring and Versioning (DAV) Spezifikation
167      zur Erzeugung und Pflege von Web-Inhalten.</dd>
168
169      <dt><code class="module"><a href="./mod/mod_deflate.html">mod_deflate</a></code></dt>
170
171      <dd>Neues Modul in Apache httpd 2.0. Dieses Modul erlaubt es
172      Browsern, die dies unterst&#252;tzen, eine Komprimierung des
173      Inhaltes vor der Auslieferung anzufordern, um so
174      Netzwerk-Bandbreite zu sparen.</dd>
175
176      <dt><code class="module">mod_auth_ldap</code></dt>
177
178      <dd>Neues Modul in Apache httpd 2.0.41. Diese Modul
179      erm&#246;glicht die Verwendung einer LDAP-Datenbank zur
180      Speicherung von Berechtigungsdaten f&#252;r die
181      HTTP-Basic-Authentication.  Ein Begleitmodul,
182      <code class="module"><a href="./mod/mod_ldap.html">mod_ldap</a></code>, stellt einen Verbindungs-Pool und die
183      Pufferung von Abfrageergebnissen zur Verf&#252;gung.</dd>
184
185      <dt><code class="module"><a href="./mod/mod_auth_digest.html">mod_auth_digest</a></code></dt>
186
187      <dd>Zus&#228;tzliche Unterst&#252;tzung f&#252;r
188      prozess&#252;bergreifendes Session-Caching mittels Shared-Memory.
189      </dd>
190
191      <dt><code class="module"><a href="./mod/mod_charset_lite.html">mod_charset_lite</a></code></dt>
192
193      <dd>Neues Modul in Apache httpd 2.0.
194      Dieses experimentelle Modul erlaubt Zeichensatz-&#220;bersetzungen oder
195      -Umschl&#252;sselung.</dd>
196
197      <dt><code class="module"><a href="./mod/mod_file_cache.html">mod_file_cache</a></code></dt>
198
199      <dd>Neues Modul in Apache httpd 2.0. Dieses Modul beinhaltet die
200      Funktionalit&#228;t von <code>mod_mmap_static</code> aus Version
201      1.3 des Apache HTTP Server zuz&#252;glich einiger weiterer
202      Caching-Funktionen.</dd>
203
204      <dt><code class="module"><a href="./mod/mod_headers.html">mod_headers</a></code></dt>
205
206      <dd>Dieses Modul ist in Apache httpd 2.0 deutlich flexibler
207      geworden. Es kann jetzt die von <code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code>
208      genutzten Request-Header manipulieren und es ist m&#246;glich
209      Response-Header auf Basis von definierten Bedingungen zu
210      ver&#228;ndern.</dd>
211
212      <dt><code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code></dt>
213
214      <dd>Das Proxy Modul wurde komplett neu geschrieben um die
215      M&#246;glichkeiten der neuen Filter-Funktionalit&#228;t
216      auszusch&#246;pfen und um einen zuverl&#228;ssigen Proxy zu haben, der
217      den HTTP/1.1-Spezifikationen entspricht. Neue <code class="directive"><a href="./mod/mod_proxy.html#proxy">&lt;Proxy&gt;</a></code>
218      -Konfigurationsabschnitte bieten eine besser lesbare (und intern
219      schnellere) Kontrolle der vermittelten Seiten.
220      Die &#252;berladenen <code>&lt;Directory
221      "proxy:..."&gt;</code>-Konfigurationen werden nicht
222      mehr unterst&#252;tzt. Das Modul ist nun in mehrere Module
223      unterteilt, die jeweils ein bestimmtes &#220;bertragungsprotokoll
224      unterst&#252;tzen, wie <code>proxy_connect</code>,
225      <code>proxy_ftp</code> und <code>proxy_http</code>.</dd>
226
227      <dt><code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code></dt>
228
229      <dd>Die neue Konfigurationsanweisung <code class="directive"><a href="./mod/mod_negotiation.html#forcelanguagepriority">ForceLanguagePriority</a></code>
230      kann benutzt werden, um sicherzustellen, dass ein Client auf jeden
231      Fall ein einzelnes Dokument, anstatt einer NOT ACCEPTABLE- oder
232      MULTIPLE CHOICES-Antwort, bekommt. Zus&#228;tzlich wurden die
233      Negotiation- und Multiview-Algorithmen angepasst um einheitlichere
234      Ergebnisse zu liefern. Au&#223;erdem wird ein neues
235      Type-Map-Format bereitgestellt, das Dokumenteninhalte direkt
236      enthalten kann.</dd>
237
238      <dt><code class="module"><a href="./mod/mod_autoindex.html">mod_autoindex</a></code></dt>
239
240      <dd>Automatisch erzeugte Verzeichnisindizes k&#246;nnen zur besseren
241      &#220;bersichtlichkeit durch HTML-Tabellen dargestellt werden.
242      Genauere Sortierungen, wie Sortierung nach Versionsnummer und
243      Wildcard-Filterung des Verzeichnisindizes werden unterst&#252;tzt.</dd>
244
245      <dt><code class="module"><a href="./mod/mod_include.html">mod_include</a></code></dt>
246
247      <dd>Neue Anweisungen erlauben es, die Standard Start- und Endtags von
248      SSI-Elementen zu &#228;ndern. Zudem k&#246;nnen die Default-Formate
249      f&#252;r Fehlermeldungen und Zeitangaben nun ebenfalls in der
250      Serverkonfiguration vorgenommen werden. Auf die Ergebnisse der
251      Auswertung und Gruppierung von regul&#228;ren Ausdr&#252;cken (jetzt
252      auf Basis der Perl-Syntax f&#252;r regul&#228;re Ausdr&#252;cke) kann
253      &#252;ber die <code class="module"><a href="./mod/mod_include.html">mod_include</a></code> Variablen <code>$0</code>
254      bis <code>$9</code> zugegriffen werden.</dd>
255
256      <dt><code class="module">mod_auth_dbm</code></dt>
257
258      <dd>DBM-&#228;hnliche Datenbanken werden jetzt durch die
259      Konfigurationsaweisung <code>AuthDBMType</code>
260      unterst&#252;tzt.</dd>
261    </dl>
