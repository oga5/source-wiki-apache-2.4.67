45<h2><a name="core" id="core">Principais Melhorias</a></h2>
46    
47
48    <dl>
49      <dt>Threading Unix</dt>
50
51      <dd>Em sistemas Unix com suporte a threads POSIX, o Apache pode
52      funcionar em modo h&#237;brido multiprocesso e multithread. N&#227;o funciona
53      em todas configura&#231;&#245;es, mas melhora a escalabilidade em muitas.</dd>
54
55      <dt>Novo Sistema de Compila&#231;&#227;o</dt>
56
57      <dd>O sistema de compila&#231;&#227;o foi reescrito do zero para utilizar o
58      <code>autoconf</code> e o <code>libtool</code>, tornando a
59      configura&#231;&#227;o do sistema Apache mais similar a de outros
60      pacotes.</dd>
61
62      <dt>Suporte Multi-protocolo</dt>
63
64      <dd>O Apache possui agora uma infraestrutura feita para suportar
65      m&#250;ltiplos protocolos. O m&#243;dulo <code class="module"><a href="./mod/mod_echo.html">mod_echo</a></code> &#233;  um
66      exemplo ilustrativo de sua utiliza&#231;&#227;o.</dd>
67
68      <dt>Suporte Aperfei&#231;oado para Plataformas N&#227;o-Unix</dt>
69
70      <dd>O Apache 2.0 est&#225;  mais r&#225;pido e mais est&#225;vel em plataformas
71      N&#227;o-Unix como BeOS, OS/2 e Windows. Com a introdu&#231;&#227;o de m&#243;dulos
72      <a href="mpm.html">multi-processamento</a> (MPMs) espec&#237;ficos e a
73      Apache Portable Runtime (APR), essas plataformas est&#227;o implementando
74      as suas APIs nativas, evitando as camadas de emula&#231;&#227;o POSIX que se
75      mostravam lentas e defeituosas.</dd>
76
77      <dt>Nova API Apache</dt>
78
79      <dd>A API para m&#243;dulos mudou significativamente na vers&#227;o 2.0.
80      Muitos dos problemas de ordenamento/prioridade da vers&#227;o
81      1.3 foram resolvidos. A vers&#227;o 2.0 faz o ordenamento autom&#225;tico
82      "per-hook" para permitir mais flexibilidade. Novas chamadas foram
83      adicionadas para fornecer capacidades adicionais sem a necessidade
84      de se aplicar nenhum patch ao servidor Apache principal.</dd>
85
86      <dt>Suporte IPv6</dt>
87
88      <dd>Em sistemas onde o IPv6 &#233; suportado pela biblioteca de base
89      Apache Portable Runtime, o Apache monitora por padr&#227;o
90      as interfaces IPv6. Em adi&#231;&#227;o as diretrizes  <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>, <code class="directive"><a href="./mod/core.html#namevirtualhost">NameVirtualHost</a></code> e <code class="directive"><a href="./mod/core.html#virtualhost">VirtualHost</a></code>, suportam correntes (strings) de
91      endere&#231;os num&#233;ricos do tipo IPv6. (ex. "<code>Listen
92      [2001:db8::1]:8080</code>").</dd>
93
94      <dt>Filtrando</dt>
95
96      <dd>Os m&#243;dulos do Apache agora s&#227;o feito filtros que
97      agem na corrente do conte&#250;do na medida que este &#233; entregue, tanto
98      na entrada quando na sa&#237;da de dados do servidor. &#201; poss&#237;vel ent&#227;o,
99      por exemplo, que o retorno de dados de scripts CGI sejam analisados
100      pelas diretrizes do "Server Side Include" usando o filtro <code>INCLUDES</code> do <code class="module"><a href="./mod/mod_include.html">mod_include</a></code>. O m&#243;dulo <code class="module"><a href="./mod/mod_ext_filter.html">mod_ext_filter</a></code>, permite que programas externos trabalhem
101      como filtros do mesmo modo que aplica&#231;&#245;es CGI funcionam como
102      manipuladores.</dd>
103
104      <dt>Respostas de Erro Multi-linguais</dt>
105
106      <dd>Mensagens de erro para o navegador agora s&#227;o fornecidas em
107      diversas l&#237;nguas, usando documentos SSI. Podem ser personalizadas
108      pelo administrador que desejar definir seus pr&#243;prios
109      padr&#245;es.</dd>
110
111      <dt>Configura&#231;&#227;o Simplificada</dt>
112
113      <dd>Muitas diretrizes confusas foram simplificadas. Entre elas,
114      <code>Port</code> e <code>BindAddress</code> n&#227;o existem
115      mais; apenas a diretriz <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
116      &#233; usada para direcionar endere&#231;os IP; a diretriz <code class="directive"><a href="./mod/core.html#servername">ServerName</a></code> especifica o nome do servidor
117      e o n&#250;mero da porta apenas para redirecionamento e reconhecimento
118      de hospedeiros virtuais.</dd>
119
120      <dt>Suporte Nativo ao Unicode do Windows NT</dt>
121
122      <dd>O Apache 2.0 para Windows NT agora usa utf-8 para codifica&#231;&#227;o
123      de todos os nomes de arquivos. A tradu&#231;&#227;o para o sistema
124      base Unicode, torna poss&#237;vel o suporte multi-lingual para todas
125      as instala&#231;&#245;es da fam&#237;lia NT, incluindo o Windows 2000 e Windows XP.
126      <em>Esse suporte n&#227;o se estende ao Windows 95, 98 ou ME, que
127      continuam usando o c&#243;digo de p&#225;ginas da m&#225;quina local para o
128      acesso ao sistema de arquivos.</em></dd>
129
130      <dt>Biblioteca de Express&#245;es Regulares Atualizada</dt>
131
132      <dd>O Apache 2.0 inclui a <a href="http://www.pcre.org/">Biblioteca
133      de Express&#245;es Regulares Compat&#237;veis Perl</a> (PCRE).  Todas as
134      avalia&#231;&#245;es de express&#245;es regulares usam a mais poderosa sintaxe
135      do Perl 5.</dd>
136
137    </dl>

140<h2><a name="module" id="module">Melhorias nos M&#243;dulos</a></h2>
141    
142
143    <dl>
144      <dt><code class="module"><a href="./mod/mod_ssl.html">mod_ssl</a></code></dt>
145
146      <dd>Novo m&#243;dulo no Apache 2.0. Esse m&#243;dulo &#233; uma interface
147      para os protocolos de codifica&#231;&#227;o SSL/TLS fornecidos pela
148      OpenSSL.</dd>
149
150      <dt><code class="module"><a href="./mod/mod_dav.html">mod_dav</a></code></dt>
151
152      <dd>Novo m&#243;dulo no Apache 2.0. Este m&#243;dulo implementa as
153      especifica&#231;&#245;es de Autoria Distribu&#237;da e Vers&#245;es (Distributed
154      Authoring and Versioning - DAV) para HTTP, para a publica&#231;&#227;o
155      e a manuten&#231;&#227;o de conte&#250;do da web.</dd>
156
157      <dt><code class="module"><a href="./mod/mod_deflate.html">mod_deflate</a></code></dt>
158
159      <dd>Novo m&#243;dulo no Apache 2.0.  Esse m&#243;dulo permite o suporte
160      a navegadores que solicitam que o conte&#250;do seja comprimido antes
161      da entrega, economizando banda da rede.</dd>
162
163      <dt><code class="module"><a href="./mod/mod_auth_ldap.html">mod_auth_ldap</a></code></dt>
164
165      <dd>Novo m&#243;dulo no Apache 2.0.41. Este m&#243;dulo permite que 
166      bancos de dados LDAP sejam usados para armazenar credenciais
167      para Autentica&#231;&#227;o B&#225;sica HTTP. Um m&#243;dulo que o acompanha <code class="module"><a href="./mod/mod_ldap.html">mod_ldap</a></code>, fornece a concilia&#231;&#227;o de conex&#245;es e armazenamento
168      de resultados.</dd>
169
170      <dt><code class="module"><a href="./mod/mod_auth_digest.html">mod_auth_digest</a></code></dt>
171
172      <dd>Inclui suporte adicional para armazenamento de sess&#245;es
173      atrav&#233;s de processos que usam mem&#243;ria compartilhada.</dd>
174
175      <dt><code class="module"><a href="./mod/mod_charset_lite.html">mod_charset_lite</a></code></dt>
176
177      <dd>Novo m&#243;dulo no Apache 2.0. Este modo experimental permite a
178      tradu&#231;&#227;o de tabelas de caracteres ou re-codifica&#231;&#227;o.</dd>
179
180      <dt><code class="module"><a href="./mod/mod_file_cache.html">mod_file_cache</a></code></dt>
181
182      <dd>Novo m&#243;dulo no Apache 2.0. Esse m&#243;dulo inclui a funcionalidade
183      do <code>mod_mmap_static</code> do Apache 1.3, al&#233;m de disponibilizar
184      outras possibilidades de armazenamento.</dd>
185
186      <dt><code class="module"><a href="./mod/mod_headers.html">mod_headers</a></code></dt>
187
188      <dd>Este m&#243;dulo est&#225; muito mais flex&#237;vel no Apache 2.0. Pode
189      modificar pedidos de cabe&#231;alhos usados pelo <code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code>, e incondicionalmente pode ajustar cabe&#231;alhos de respostas.</dd>
190
191      <dt><code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code></dt>
192
193      <dd>O m&#243;dulo proxy foi totalmente reescrito para levar vantagem
194      da nova infraestrutura de filtro e implementar um proxy mais fiel e 
195      de acordo com o padr&#227;o HTTP/1.1. Al&#233;m disso, uma nova se&#231;&#227;o
196      de configura&#231;&#227;o <code class="directive"><a href="./mod/mod_proxy.html#proxy">&lt;Proxy&gt;</a></code> fornece controles mais leg&#237;veis (e internamente
197      mais r&#225;pidos) para sites com proxies; configura&#231;&#245;es
198      sobrecarregadas <code>&lt;Directory "proxy:..."&gt;</code>, n&#227;o
199      s&#227;o suportadas. O m&#243;dulo agora &#233; dividido em suporte
200      de protocolos espec&#237;ficos incluindo <code>proxy_connect</code>,
201      <code>proxy_ftp</code> e <code>proxy_http</code>.</dd>
202
203      <dt><code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code></dt>
204
205      <dd>A nova diretriz <code class="directive"><a href="./mod/mod_negotiation.html#forcelanguagepriority">ForceLanguagePriority</a></code> pode ser usada para assegurar que
206      o cliente receba um &#250;nico documento em todos os casos, ao inv&#233;s de
207      respostas "NOT ACCEPTABLE" ou "MULTIPLE CHOICES". Novos algoritmos
208      de negocia&#231;&#227;o e vis&#245;es m&#250;ltiplas (MultiViews) foram organizados para
209      obter resultados mais consistentes e uma nova forma de tipo de mapa
210      (map type) que podem incluir o conte&#250;do de documentos &#233; fornecido.</dd>
211
212      <dt><code class="module"><a href="./mod/mod_autoindex.html">mod_autoindex</a></code></dt>
213
214      <dd>As listagens de diret&#243;rios autom&#225;ticas podem ser
215      configuradas para usar tabelas HTML para formata&#231;&#245;es mais limpas
216      e permitir controles mais acurados de classifica&#231;&#227;o, incluindo
217      ordena&#231;&#227;o por vers&#227;o e filtro da lista de
218      diret&#243;rios atrav&#233;s de caracteres-coringa.</dd>
219
220      <dt><code class="module"><a href="./mod/mod_include.html">mod_include</a></code></dt>
221
222      <dd>Novas diretrizes permitem que as tags padr&#245;es <em>start</em> e
223      <em>end</em> para elementos SSI, possam ser alteradas e permitir que
224      as configura&#231;&#245;es de formatos de erro e hora sejam inclu&#237;dos no
225      arquivo de configura&#231;&#227;o principal, ao inv&#233;s de serem adicionadas
226      ao documento SSI. Resultados de an&#225;lises de express&#245;es regulares
227      e agrupamento (baseadas na sintaxe de express&#245;es regulares do Perl)
228      podem ser obtidas usando as vari&#225;veis do m&#243;dulo <code class="module"><a href="./mod/mod_include.html">mod_include</a></code>, de <code>$0</code> a <code>$9</code>.</dd>
229
230      <dt><code class="module"><a href="./mod/mod_auth_dbm.html">mod_auth_dbm</a></code></dt>
231
232      <dd>Agora suporta m&#250;ltiplos tipos de banco de dados similares ao DBM,
233      usando a diretriz <a href="../2.0/mod/mod_auth_dbm.html#AuthDBMType">
234	  <code>AuthDBMType</code></a>
235      .</dd>
236
237    </dl>
