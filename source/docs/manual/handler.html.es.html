43<h2><a name="definition" id="definition">&#191;Qu&#233; es un Handler?</a></h2>
44    
45    <table class="related"><tr><th>M&#243;dulos Relacionados</th><th>Directivas Relacionadas</th></tr><tr><td><ul><li><code class="module"><a href="./mod/mod_actions.html">mod_actions</a></code></li><li><code class="module"><a href="./mod/mod_asis.html">mod_asis</a></code></li><li><code class="module"><a href="./mod/mod_cgi.html">mod_cgi</a></code></li><li><code class="module"><a href="./mod/mod_imagemap.html">mod_imagemap</a></code></li><li><code class="module"><a href="./mod/mod_info.html">mod_info</a></code></li><li><code class="module"><a href="./mod/mod_mime.html">mod_mime</a></code></li><li><code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code></li><li><code class="module"><a href="./mod/mod_status.html">mod_status</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/mod_actions.html#action">Action</a></code></li><li><code class="directive"><a href="./mod/mod_mime.html#addhandler">AddHandler</a></code></li><li><code class="directive"><a href="./mod/mod_mime.html#removehandler">RemoveHandler</a></code></li><li><code class="directive"><a href="./mod/core.html#sethandler">SetHandler</a></code></li></ul></td></tr></table>
46
47
48    <p>Un "handler" es una representaci&#243;n interna de Apache de
49    una acci&#243;n que se va a ejecutar cuando hay una llamada a un
50    fichero. Generalmente, los ficheros tienen handlers
51    impl&#237;citos, basados en el tipo de fichero de que se
52    trata. Normalmente, todos los ficheros son simplemente servidos
53    por el servidor, pero algunos tipos de ficheros se tratan de forma
54    diferente.</p>
55
56    <p>Handlers pueden ser usados de manera explicita,
57     bas&#225;ndose en la extensi&#243;n del fichero o en
58    la ubicaci&#243;n en la que est&#233;, se pueden especificar handlers
59    sin tener en cuenta el tipo de fichero que se trate. Esto es
60    una ventaja por dos razones. Primero, es una soluci&#243;n
61    m&#225;s elegante. Segundo, porque a un fichero se le pueden
62    asignar tanto un tipo <strong>como</strong> un handler. (Consulte
63    tambi&#233;n la secci&#243;n <a href="mod/mod_mime.html#multipleext">Ficheros y extensiones
64    m&#250;ltiples</a>.)</p>
65
66    <p>Los Handlers pueden tanto ser compilados con el servidor
67    como incluidos en un m&#243;dulo, o a&#241;adidos con la
68    directiva <code class="directive"><a href="./mod/mod_actions.html#action">Action</a></code>. Los
69    handlers que vienen incluidos en el core con el servidor de la distribuci&#243;n
70    est&#225;ndar de Apache son:</p>
71
72    <ul>
73      <li><strong>default-handler</strong>: Env&#237;a el fichero
74      usando el <code>default_handler()</code>, que es el handler
75      usado por defecto para tratar contenido
76      est&#225;tico. (core)</li>
77
78      <li><strong>send-as-is</strong>: Env&#237;a el fichero con
79      cabeceras HTTP tal y como es. (<code class="module"><a href="./mod/mod_asis.html">mod_asis</a></code>)</li>
80
81      <li><strong>cgi-script</strong>: Trata el fichero como un sript
82      CGI. (<code class="module"><a href="./mod/mod_cgi.html">mod_cgi</a></code>)</li>
83
84      <li><strong>imap-file</strong>: Trata el fichero como un mapa de
85      im&#225;genes. (<code class="module"><a href="./mod/mod_imagemap.html">mod_imagemap</a></code>)</li>
86
87      <li><strong>server-info</strong>: Extrae la informaci&#243;n de
88      configuraci&#243;n del
89      servidor. (<code class="module"><a href="./mod/mod_info.html">mod_info</a></code>)</li>
90
91      <li><strong>server-status</strong>: Extrae el informe del estado
92      del servidor. (<code class="module"><a href="./mod/mod_status.html">mod_status</a></code>)</li>
93
94      <li><strong>type-map</strong>: Trata el fichero como una
95      correspondencia de tipos para la negociaci&#243;n de contenidos.
96      (<code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code>)</li> 
97    </ul>

100<h2><a name="examples" id="examples">Ejemplos</a></h2> 
101      
102
103      <h3><a name="example1" id="example1">Modificar contenido est&#225;tico usando un script
104      CGI</a></h3>
105      
106
107      <p>Las siguientes directivas hacen que cuando haya una
108      petici&#243;n de ficheros con la extensi&#243;n
109      <code>html</code> se lance el script CGI
110      <code>footer.pl</code>.</p>
111
112      <div class="example"><p><code>
113        Action add-footer /cgi-bin/footer.pl<br />
114        AddHandler add-footer .html
115      </code></p></div>
116
117      <p>En este caso, el script CGI es el responsable de enviar el
118      documento originalmente solicitado (contenido en la variable de
119      entorno <code>PATH_TRANSLATED</code>) y de hacer cualquier
120      modificaci&#243;n o a&#241;adido deseado.</p>
121
122    
123    <h3><a name="example2" id="example2">Archivos con cabeceras HTTP</a></h3>
124      
125
126      <p>Las siguientes directivas activan el handler
127      <code>send-as-is</code>, que se usa para ficheros que contienen
128      sus propias cabeceras HTTP. Todos los archivos en el directorio
129      <code>/web/htdocs/asis/</code> ser&#225;n procesados por el
130      handler <code>send-as-is</code>, sin tener en cuenta su
131      extension.</p>
132
133      <pre class="prettyprint lang-config">&lt;Directory "/web/htdocs/asis"&gt;
134    SetHandler send-as-is
135&lt;/Directory&gt;</pre>

141<h2><a name="programmer" id="programmer">Nota para programadores</a></h2>
142    
143
144    <p>Para implementar las funcionalidades de los handlers, se ha
145    hecho un a&#241;adido a la <a href="developer/API.html">API de
146    Apache</a> que puede que quiera usar. Para ser m&#225;s
147    espec&#237;ficos, se ha a&#241;adido un nuevo registro a la
148    estructura <code>request_rec</code>:</p>
149
150    <pre class="prettyprint lang-c">char *handler</pre>
151
152
153    <p>Si quiere que su m&#243;dulo llame a un handler , solo tiene
154    que a&#241;adir <code>r-&gt;handler</code> al nombre del handler
155    en cualquier momento antes de la fase <code>invoke_handler</code>
156    de la petici&#243;n. Los handlers se implementan siempre como se
157    hac&#237;a antes, aunque usando el nombre del handler en vez de un
158    tipo de contenido. Aunque no es de obligado cumplimiento, la
159    convenci&#243;n de nombres para los handlers es que se usen
160    palabras separadas por guiones, sin barras, de manera que no se
161    invada el media type name-space.</p>
