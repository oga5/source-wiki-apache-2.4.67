43<h2><a name="core" id="core">Core Enhancements</a></h2>
44    
45
46    <dl>
47      <dt>Unix Threading</dt>
48
49      <dd>On Unix systems with POSIX threads support, Apache httpd can
50      now run in a hybrid multiprocess, multithreaded mode. This
51      improves scalability for many, but not all configurations.</dd>
52
53      <dt>New Build System</dt>
54
55      <dd>The build system has been rewritten from scratch to be
56      based on <code>autoconf</code> and <code>libtool</code>.
57      This makes Apache httpd's configuration system more similar to
58      that of other packages.</dd>
59
60      <dt>Multiprotocol Support</dt>
61
62      <dd>Apache HTTP Server now has some of the infrastructure in place to
63      support serving multiple protocols. <code class="module"><a href="./mod/mod_echo.html">mod_echo</a></code> has
64      been written as an example.</dd>
65
66      <dt>Better support for non-Unix
67      platforms</dt>
68
69      <dd>Apache HTTP Server 2.0 is faster and more stable on non-Unix
70      platforms such as BeOS, OS/2, and Windows. With the
71      introduction of platform-specific <a href="mpm.html">multi-processing modules</a> (MPMs) and the
72      Apache Portable Runtime (APR), these platforms are now
73      implemented in their native API, avoiding the often buggy and
74      poorly performing POSIX-emulation layers.</dd>
75
76      <dt>New Apache httpd API</dt>
77
78      <dd>The API for modules has changed significantly for 2.0.
79      Many of the module-ordering/-priority problems from 1.3 should
80      be gone. 2.0 does much of this automatically, and module ordering
81      is now done per-hook to allow more flexibility. Also, new calls
82      have been added that provide additional module capabilities
83      without patching the core Apache HTTP Server.</dd>
84
85      <dt>IPv6 Support</dt>
86
87      <dd>On systems where IPv6 is supported by the underlying
88      Apache Portable Runtime library, Apache httpd gets IPv6 listening
89      sockets by default. Additionally, the <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>, <code class="directive"><a href="./mod/core.html#namevirtualhost">NameVirtualHost</a></code>, and <code class="directive"><a href="./mod/core.html#virtualhost">VirtualHost</a></code> directives support
90      IPv6 numeric address strings (e.g., "<code>Listen
91      [2001:db8::1]:8080</code>").</dd>
92
93      <dt>Filtering</dt>
94
95      <dd>Apache httpd modules may now be written as filters which act on
96      the stream of content as it is delivered to or from the
97      server. This allows, for example, the output of CGI scripts to
98      be parsed for Server Side Include directives using the
99      <code>INCLUDES</code> filter in <code class="module"><a href="./mod/mod_include.html">mod_include</a></code>. The
100      module <code class="module"><a href="./mod/mod_ext_filter.html">mod_ext_filter</a></code> allows external programs to
101      act as filters in much the same way that CGI programs can act as
102      handlers.</dd>
103
104      <dt>Multilanguage Error Responses</dt>
105
106      <dd>Error response messages to the browser are now provided in
107      several languages, using SSI documents. They may be customized
108      by the administrator to achieve a consistent look and feel.</dd>
109
110      <dt>Simplified configuration</dt>
111
112      <dd>Many confusing directives have been simplified. The often
113      confusing <code>Port</code> and <code>BindAddress</code> directives
114      are gone; only the <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
115      directive is used for IP address binding; the <code class="directive"><a href="./mod/core.html#servername">ServerName</a></code> directive specifies the
116      server name and port number only for redirection and vhost
117      recognition.</dd>
118
119      <dt>Native Windows NT Unicode Support</dt>
120
121      <dd>Apache httpd 2.0 on Windows NT now uses utf-8 for all filename
122      encodings. These directly translate to the underlying Unicode
123      file system, providing multilanguage support for all Windows
124      NT-based installations, including Windows 2000 and Windows XP.
125      <em>This support does not extend to Windows 95, 98 or ME, which
126      continue to use the machine's local codepage for filesystem
127      access.</em></dd>
128
129      <dt>Regular Expression Library Updated</dt>
130
131      <dd>Apache httpd 2.0 includes the <a href="http://www.pcre.org/">Perl
132      Compatible Regular Expression Library</a> (PCRE).  All regular
133      expression evaluation now uses the more powerful Perl 5
134      syntax.</dd>
135
136    </dl>

139<h2><a name="module" id="module">Module Enhancements</a></h2>
140    
141
142    <dl>
143      <dt><code class="module"><a href="./mod/mod_ssl.html">mod_ssl</a></code></dt>
144
145      <dd>New module in Apache httpd 2.0. This module is an interface
146      to the SSL/TLS encryption protocols provided by
147      OpenSSL.</dd>
148
149      <dt><code class="module"><a href="./mod/mod_dav.html">mod_dav</a></code></dt>
150
151      <dd>New module in Apache httpd 2.0. This module implements the HTTP
152      Distributed Authoring and Versioning (DAV) specification for
153      posting and maintaining web content.</dd>
154
155      <dt><code class="module"><a href="./mod/mod_deflate.html">mod_deflate</a></code></dt>
156
157      <dd>New module in Apache httpd 2.0.  This module allows supporting
158      browsers to request that content be compressed before delivery,
159      saving network bandwidth.</dd>
160
161      <dt><code class="module">mod_auth_ldap</code></dt>
162
163      <dd>New module in Apache httpd 2.0.41.  This module allows an LDAP
164      database to be used to store credentials for HTTP Basic
165      Authentication.  A companion module, <code class="module"><a href="./mod/mod_ldap.html">mod_ldap</a></code>
166      provides connection pooling and results caching.</dd>
167
168      <dt><code class="module"><a href="./mod/mod_auth_digest.html">mod_auth_digest</a></code></dt>
169
170      <dd>Includes additional support for session caching across
171      processes using shared memory.</dd>
172
173      <dt><code class="module"><a href="./mod/mod_charset_lite.html">mod_charset_lite</a></code></dt>
174
175      <dd>New module in Apache httpd 2.0. This experimental module allows
176      for character set translation or recoding.</dd>
177
178      <dt><code class="module"><a href="./mod/mod_file_cache.html">mod_file_cache</a></code></dt>
179
180      <dd>New module in Apache httpd 2.0. This module includes the
181      functionality of <code>mod_mmap_static</code> in Apache HTTP
182      Server version 1.3, plus adds further caching abilities.</dd>
183
184      <dt><code class="module"><a href="./mod/mod_headers.html">mod_headers</a></code></dt>
185
186      <dd>This module is much more flexible in Apache httpd 2.0. It can now
187      modify request headers used by <code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code>, and
188      it can conditionally set response headers.</dd>
189
190      <dt><code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code></dt>
191
192      <dd>The proxy module has been completely rewritten to take
193      advantage of the new filter infrastructure and to implement a
194      more reliable, HTTP/1.1 compliant proxy. In addition, new
195      <code class="directive"><a href="./mod/mod_proxy.html#proxy">&lt;Proxy&gt;</a></code>
196      configuration sections provide more readable (and internally
197      faster) control of proxied sites; overloaded <code>&lt;Directory
198      "proxy:..."&gt;</code> configuration are not supported. The module
199      is now divided into specific protocol support modules including
200      <code>proxy_connect</code>, <code>proxy_ftp</code> and
201      <code>proxy_http</code>.</dd>
202
203      <dt><code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code></dt>
204
205      <dd>A new <code class="directive"><a href="./mod/mod_negotiation.html#forcelanguagepriority">ForceLanguagePriority</a></code> directive can be used to assure that
206      the client receives a single document in all cases, rather than
207      NOT ACCEPTABLE or MULTIPLE CHOICES responses. In addition, the
208      negotiation and MultiViews algorithms have been cleaned up to
209      provide more consistent results and a new form of type map that
210      can include document content is provided.</dd>
211
212      <dt><code class="module"><a href="./mod/mod_autoindex.html">mod_autoindex</a></code></dt>
213
214      <dd>Autoindex'ed directory listings can now be configured to
215      use HTML tables for cleaner formatting, and allow finer-grained
216      control of sorting, including version-sorting, and wildcard
217      filtering of the directory listing.</dd>
218
219      <dt><code class="module"><a href="./mod/mod_include.html">mod_include</a></code></dt>
220
221      <dd>New directives allow the default start and end tags for SSI elements
222      to be changed and allow for error and time format configuration
223      to take place in the main configuration file rather than in the
224      SSI document. Results from regular expression parsing and grouping
225      (now based on Perl's regular expression syntax) can be retrieved
226      using <code class="module"><a href="./mod/mod_include.html">mod_include</a></code>'s variables <code>$0</code>
227      .. <code>$9</code>.</dd>
228
229      <dt><code class="module">mod_auth_dbm</code></dt>
230
231      <dd>Now supports multiple types of DBM-like databases using the
232      <code class="directive">AuthDBMType</code> directive.</dd>
233
234    </dl>
