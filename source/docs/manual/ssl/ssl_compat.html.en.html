30<p>
31This page covers backwards compatibility between mod_ssl and other
32SSL solutions.  mod_ssl is not the only SSL solution for Apache; four
33additional products are (or were) also available: Ben Laurie's freely
34available <a href="http://www.apache-ssl.org/">Apache-SSL</a> (from
35where mod_ssl were originally derived in 1998), Red Hat's commercial
36Secure Web Server (which was based on mod_ssl), Covalent's commercial
37Raven SSL Module (also based on
38mod_ssl) and finally C2Net's (now Red Hat's) commercial product <a href="http://www.redhat.com/explore/stronghold/">Stronghold</a> (based
39on a different evolution branch, named Sioux up to Stronghold 2.x, and
40based on mod_ssl since Stronghold 3.x).</p>
41
42<p>
43mod_ssl mostly provides a superset of the functionality of all the other
44solutions, so it's simple to migrate from one of the older modules to
45mod_ssl. The configuration directives and environment variable names
46used by the older SSL solutions vary from those used in mod_ssl;
47mapping tables are included here to give the equivalents used by mod_ssl.</p>

55<h2><a name="configuration" id="configuration">Configuration Directives</a></h2>
56<p>The mapping between configuration directives used by Apache-SSL
571.x and mod_ssl 2.0.x is given in <a href="#table1">Table
581</a>. The mapping from Sioux 1.x and Stronghold 2.x is only partial
59because of special functionality in these interfaces which mod_ssl
60doesn't provide.</p>

194<h2><a name="customlog" id="customlog">Custom Log Functions</a></h2>
195<p>
196When mod_ssl is enabled, additional functions exist for the <a href="../mod/mod_log_config.html#formats">Custom Log Format</a> of
197<code class="module"><a href="../mod/mod_log_config.html">mod_log_config</a></code> as documented in the Reference
198Chapter. Beside the ``<code>%{</code><em>varname</em><code>}x</code>''
199eXtension format function which can be used to expand any variables provided
200by any module, an additional Cryptography
201``<code>%{</code><em>name</em><code>}c</code>'' cryptography format function
202exists for backward compatibility. The currently implemented function calls
203are listed in <a href="#table3">Table 3</a>.</p>
204
205<h3><a name="table3" id="table3">Table 3: Custom Log Cryptography Function</a></h3>
206
207<table>
208
209<tr><th>Function Call</th><th>Description</th></tr>
210
211<tr><td><code>%...{version}c</code></td>   <td>SSL protocol version</td></tr>
212<tr><td><code>%...{cipher}c</code></td>    <td>SSL cipher</td></tr>
213<tr><td><code>%...{subjectdn}c</code></td> <td>Client Certificate Subject Distinguished Name</td></tr>
214<tr><td><code>%...{issuerdn}c</code></td>  <td>Client Certificate Issuer Distinguished Name</td></tr>
215<tr><td><code>%...{errcode}c</code></td>   <td>Certificate Verification Error (numerical)</td></tr>
216
217<tr><td><code>%...{errstr}c</code></td>    <td>Certificate Verification Error (string)</td></tr>
218</table>
