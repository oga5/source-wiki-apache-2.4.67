49<h2><a name="main" id="main">メインの設定ファイル</a></h2>
50    
51    <table class="related"><tr><th>関連モジュール</th><th>関連ディレクティブ</th></tr><tr><td><ul><li><code class="module"><a href="./mod/mod_mime.html">mod_mime</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/core.html#ifdefine">&lt;IfDefine&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#include">Include</a></code></li><li><code class="directive"><a href="./mod/mod_mime.html#typesconfig">TypesConfig</a></code></li></ul></td></tr></table>
52
53    <p>Apache は <a href="mod/directives.html">ディレクティブ</a> を設定ファイルに平文で書くことにより設定します。
54    メインの設定ファイルは普通は <code>httpd.conf</code> という名前です。
55    このファイルの位置はコンパイル時に設定されますが、コマンドラインの
56    <code>-f</code> フラグにより上書きできます。
57    また、他の設定ファイルを <code class="directive"><a href="./mod/core.html#include">Include</a></code>
58    ディレクティブによって追加でき、ワイルドカードを使用して多数の
59    設定ファイルを追加することができます。
60    どんなディレクティブも、これらの設定ファイルどれにでも入れることができます。
61    Apache は起動時か再起動時のみメイン設定ファイルの変更を認識します。</p>
62
63    <p>サーバは MIME
64    ドキュメントタイプを含んでいるファイルも読み込みます。ファイル名は
65    <code class="directive"><a href="./mod/mod_mime.html#typesconfig">TypesConfig</a></code>
66    で設定され、デフォルトでは <code>mime.types</code>
67    になっています。</p>

70<h2><a name="syntax" id="syntax">設定ファイルの構文</a></h2>
71    
72
73    <p>Apache の設定ファイルは 1 行に 1 つのディレクティブからなります。
74    バックスラッシュ "\" はディレクティブが次の行に継続していることを
75    示すために行の最後の文字として使われているかもしれません。
76    行の最後とバックスラッシュの間に他の文字や空白があってはいけません。
77    </p>
78
79    <p>設定ファイルのディレクティブは大文字小文字を区別しませんが、
80    引数にはしばしば区別するものがあります。ハッシュ文字 "#"
81    で始まる行はコメントと見なされて無視されます。
82    設定ディレクティブと同一行の末尾にコメントが含まれていては<strong>いけません</strong>。ディレクティブの前の空行と空白は無視されますので、
83    わかりやすくするためにディレクティブをインデントしても構いません。
84    </p>
85
86    <p>設定ファイルの構文エラーは、
87    <code>apachectl configtest</code>
88    かコマンドラインオプション
89    <code>-t</code> を使って調べられます。</p>

92<h2><a name="modules" id="modules">モジュール</a></h2>
93    
94
95    <table class="related"><tr><th>関連モジュール</th><th>関連ディレクティブ</th></tr><tr><td><ul><li><code class="module"><a href="./mod/mod_so.html">mod_so</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/core.html#ifmodule">&lt;IfModule&gt;</a></code></li><li><code class="directive"><a href="./mod/mod_so.html#loadmodule">LoadModule</a></code></li></ul></td></tr></table>
96
97    <p>Apache はモジュール化されたサーバです。
98    コアサーバには最も基本的な機能だけが含まれています。拡張機能は
99    Apache にロードされる<a href="mod/">モジュール</a>として利用可能です。デフォルトでは、コンパイル時にモジュールの
100    <a href="mod/module-dict.html#Status">Base</a> セット (基本セット) が
101    サーバに含まれます。サーバが<a href="dso.html">動的ロード</a>モジュールを使うようにコンパイルされている場合は、
102    モジュールを別にコンパイルして、いつでも
103    <code class="directive"><a href="./mod/mod_so.html#loadmodule">LoadModule</a></code>
104    ディレクティブを使って追加できます。
105    そうでない場合は、モジュールの追加や削除をするためには Apache
106    を再コンパイルする必要があります。設定ディレクティブは <code class="directive"><a href="./mod/core.html#ifmodule">&lt;IfModule&gt;</a></code>
107    ブロックに入れることで特定のモジュールが存在するときだけ
108    設定ファイルに含まれるようにすることができます。</p>
109
110    <p>コマンドラインオプション <code>-l</code> を使って現時点で
111    どのモジュールがサーバにコンパイルされているかを知ることができます。</p>

114<h2><a name="scope" id="scope">ディレクティブの適用範囲</a></h2>
115    
116
117    <table class="related"><tr><th>関連モジュール</th><th>関連ディレクティブ</th></tr><tr><td /><td><ul><li><code class="directive"><a href="./mod/core.html#directory">&lt;Directory&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#directorymatch">&lt;DirectoryMatch&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#files">&lt;Files&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#filesmatch">&lt;FilesMatch&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#location">&lt;Location&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#locationmatch">&lt;LocationMatch&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code></li></ul></td></tr></table>
118
119    <p>メイン設定ファイルにあるディレクティブはサーバ全体に適用されます。
120    サーバの一部分の設定だけを変更したい場合は <code class="directive"><a href="./mod/core.html#directory">&lt;Directory&gt;</a></code>, <code class="directive"><a href="./mod/core.html#directorymatch">&lt;DirectoryMatch&gt;</a></code>, <code class="directive"><a href="./mod/core.html#files">&lt;Files&gt;</a></code>, <code class="directive"><a href="./mod/core.html#filesmatch">&lt;FilesMatch&gt;</a></code>, <code class="directive"><a href="./mod/core.html#location">&lt;Location&gt;</a></code>, <code class="directive"><a href="./mod/core.html#locationmatch">&lt;LocationMatch&gt;</a></code>
121    セクションの中に置くことで適用範囲を決められます。
122    これらのセクションはその中にあるディレクティブの適用範囲を
123    特定のファイルシステムの位置や URL に限定します。
124    非常に細粒度の設定を可能にするために、
125    セクションを入れ子にすることもできます。</p>
126
127    <p>Apache は同時に多くの違うウェブサイトを扱う能力があります。
128    これは <a href="vhosts/">バーチャルホスト</a> と呼ばれています。
129    特定のウェブサイトにのみ適用されるようにするために、
130    ディレクティブは
131    <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
132    セクションの中に置くことでも適用範囲を変えることができます。</p>
133
134    <p>ほとんどのディレクティブはどのセクションにでも書けますが、
135    中にはコンテキストによっては意味をなさないものもあります。
136    例えば、プロセスの作成を制御しているディレクティブはメインサーバの
137    コンテキストにのみ書くことができます。
138    どのディレクティブをどのセクションに書くことができるかを知るためには
139    ディレクティブの <a href="mod/directive-dict.html#Context">コンテキスト</a> を調べてください。詳しい情報は、
140    <a href="sections.html">Directory, Location, Files
141    セクションの動作法</a>にあります。</p>

144<h2><a name="htaccess" id="htaccess">.htaccess ファイル</a></h2>
145    
146
147    <table class="related"><tr><th>関連モジュール</th><th>関連ディレクティブ</th></tr><tr><td /><td><ul><li><code class="directive"><a href="./mod/core.html#accessfilename">AccessFileName</a></code></li><li><code class="directive"><a href="./mod/core.html#allowoverride">AllowOverride</a></code></li></ul></td></tr></table>
148
149    <p>Apache ではウェブツリーの中に置かれた特別なファイルを使って
150    非中央集権的な設定管理をできます。その特別なファイルは普通は
151    <code>.htaccess</code> という名前で、
152    <code class="directive"><a href="./mod/core.html#accessfilename">AccessFileName</a></code>
153    ディレクティブでどんな名前にでも指定できます。
154    <code>.htaccess</code>
155    ファイルに書かれたディレクティブはファイルを置いた
156    ディレクトリとその全てのサブディレクトリに適用されます。
157    <code>.htaccess</code> ファイルは、メインの設定ファイルと同じ
158    構文を使います。
159    <code>.htaccess</code>
160    ファイルはすべてのリクエストで読み込まれるため、
161    変更はすぐに反映されます。</p>
162
163    <p>どのディレクティブが <code>.htaccess</code>
164    ファイルに書けるかを調べるには、ディレクティブの<a href="mod/directive-dict.html#Context">コンテキスト</a>
165    を調べてください。サーバ管理者はさらにメイン設定ファイルの
166    <code class="directive"><a href="./mod/core.html#allowoverride">AllowOverride</a></code>
167    を設定することでどのディレクティブを <code>.htaccess</code>
168    ファイルに書けるようにするかを制御することができます。</p>
169
170    <p><code>.htaccess</code> ファイルに関する詳しい情報は
171    <a href="howto/htaccess.html">.htaccess チュートリアル</a>
172    を参照してください。</p>
