36    <p>This document covers compilation and installation of the Apache HTTP Server
37    on Unix and Unix-like systems only. For compiling and
38    installation on Windows, see <a href="platform/windows.html">Using Apache HTTP Server with Microsoft
39    Windows</a> and <a href="platform/win_compiling.html">Compiling Apache for Microsoft Windows</a>.
40    For other platforms, see the <a href="platform/">platform</a> documentation.</p>
41
42    <p>Apache httpd uses <code>libtool</code> and <code>autoconf</code>
43    to create a build environment that looks like many other Open Source
44    projects.</p>
45
46    <p>If you are upgrading from one minor version to the next (for
47    example, 2.4.8 to 2.4.9), please skip down to the <a href="#upgrading">upgrading</a> section.</p>

161<h2><a name="requirements" id="requirements">Requirements</a></h2>
162
163    <p>The following requirements exist for building Apache httpd:</p>
164
165    <dl>
166      <dt>APR and APR-Util</dt>
167      <dd>Make sure you have APR and APR-Util already installed on
168      your system. If you don't, or prefer to not use the system-provided
169      versions, download the latest versions of both APR and APR-Util
170      from <a href="http://apr.apache.org/">Apache APR</a>, unpack
171      them into <code>/httpd_source_tree_root/srclib/apr</code> and <code>/httpd_source_tree_root/srclib/apr-util</code>
172      (be sure the directory names do not have version numbers; for example,
173      the APR distribution must be under /httpd_source_tree_root/srclib/apr/) and use
174      <code>./configure</code>'s <code>--with-included-apr</code>
175      option.  On some platforms, you may have to install the
176      corresponding <code>-dev</code> packages to allow httpd to build
177      against your installed copy of APR and APR-Util.</dd>
178
179      <dt>Perl-Compatible Regular Expressions Library (PCRE)</dt>
180      <dd>This library is required but no longer bundled with httpd.
181      Download the source code from <a href="http://www.pcre.org/">http://www.pcre.org</a>,
182      or install a Port or Package.  If your build system can't find
183      the pcre-config script installed by the PCRE build, point to it
184      using the <code>--with-pcre</code> parameter.  On some platforms,
185      you may have to install the corresponding <code>-dev</code>
186      package to allow httpd to build against your installed copy
187      of PCRE.</dd>
188
189      <dt>Disk Space</dt>
190      <dd>Make sure you have at least 50 MB of temporary free disk
191      space available. After installation the server occupies
192      approximately 10 MB of disk space. The actual disk space
193      requirements will vary considerably based on your chosen
194      configuration options, any third-party modules, and, of course,
195      the size of the web site or sites that you have on the server.</dd>
196
197      <dt>ANSI-C Compiler and Build System</dt>
198      <dd>Make sure you have an ANSI-C compiler installed. The <a href="http://gcc.gnu.org/">GNU C
199      compiler (GCC)</a> from the <a href="http://www.gnu.org/">Free Software Foundation (FSF)</a>
200      is recommended. If you don't have GCC
201      then at least make sure your vendor's compiler is ANSI
202      compliant. In addition, your <code>PATH</code> must contain
203      basic build tools such as <code>make</code>.</dd>
204
205      <dt>Accurate time keeping</dt>
206      <dd>Elements of the HTTP protocol are expressed as the time of
207      day. So, it's time to investigate setting some time
208      synchronization facility on your system. Usually the
209      <code>ntpdate</code> or <code>xntpd</code> programs are used for
210      this purpose which are based on the Network Time Protocol (NTP).
211      See the <a href="http://www.ntp.org">NTP
212      homepage</a> for more details about NTP software and public
213      time servers.</dd>
214
215      <dt><a href="http://www.perl.org/">Perl 5</a>
216      [OPTIONAL]</dt>
217      <dd>For some of the support scripts like <code class="program"><a href="./programs/apxs.html">apxs</a></code> or <code class="program"><a href="./programs/dbmmanage.html">dbmmanage</a></code> (which are
218      written in Perl) the Perl 5 interpreter is required (versions
219      5.003 or newer are sufficient).  If no Perl 5 interpreter is found by the
220      <code class="program"><a href="./programs/configure.html">configure</a></code> script, you will not be able to use
221      the affected support scripts. Of course, you will still be able to
222      build and use Apache httpd.</dd>
223    </dl>

226<h2><a name="download" id="download">Download</a></h2>
227
228    <p>The Apache HTTP Server can be downloaded from the <a href="http://httpd.apache.org/download.cgi">Apache HTTP Server
229    download site</a>, which lists several mirrors.  Most users of
230    Apache on unix-like systems will be better off downloading and
231    compiling a source version.  The build process (described below) is
232    easy, and it allows you to customize your server to suit your needs.
233    In addition, binary releases are often not up to date with the latest
234    source releases.  If you do download a binary, follow the instructions
235    in the <code>INSTALL.bindist</code> file inside the distribution.</p>
236
237    <p>After downloading, it is important to verify that you have a
238    complete and unmodified version of the Apache HTTP Server. This
239    can be accomplished by testing the downloaded tarball against the
240    PGP signature.  Details on how to do this are available on the <a href="http://httpd.apache.org/download.cgi#verify">download
241    page</a> and an extended example is available describing the <a href="http://httpd.apache.org/dev/verification.html">use of
242    PGP</a>.</p>
243

246<h2><a name="extract" id="extract">Extract</a></h2>
247
248    <p>Extracting the source from the Apache HTTP Server tarball is a
249    simple matter of uncompressing, and then untarring:</p>
250
251<div class="example"><pre class="prettyprint lang-sh">$ gzip -d httpd-<em>NN</em>.tar.gz
252$ tar xvf httpd-<em>NN</em>.tar</pre>
253</div>
254
255    <p>This will create a new directory under the current directory
256    containing the source code for the distribution. You should
257    <code>cd</code> into that directory before proceeding with
258    compiling the server.</p>
259</div><div class="top"><a href="#page-header"><img alt="top" src="./images/up.gif" /></a></div>
260<div class="section">
261<h2><a name="configure" id="configure">Configuring the source tree</a></h2>
262
263    <p>The next step is to configure the Apache source tree for your
264    particular platform and personal requirements. This is done using
265    the script <code class="program"><a href="./programs/configure.html">configure</a></code> included in
266    the root directory of the distribution. (Developers downloading
267    an unreleased version of the Apache source tree will need to have
268    <code>autoconf</code> and <code>libtool</code> installed and will
269    need to run <code>buildconf</code> before proceeding with the next
270    steps. This is not necessary for official releases.)</p>
271
272    <p>To configure the source tree using all the default options,
273    simply type <code>./configure</code>. To change the default
274    options, <code class="program"><a href="./programs/configure.html">configure</a></code> accepts a variety of variables
275    and command line options.</p>
276
277    <p>The most important option is the location <code>--prefix</code>
278    where Apache is to be installed later, because Apache has to be
279    configured for this location to work correctly.  More fine-tuned
280    control of the location of files is possible with additional <a href="programs/configure.html#installationdirectories">configure
281    options</a>.</p>
282
283    <p>Also at this point, you can specify which <a href="programs/configure.html#optionalfeatures">features</a> you
284    want included in Apache by enabling and disabling <a href="mod/">modules</a>.  Apache comes with a wide range of modules
285    included by default.  They will be compiled as
286    <a href="dso.html">shared objects (DSOs)</a> which can be loaded
287    or unloaded at runtime.
288    You can also choose to compile modules statically by using the option
289    <code>--enable-<var>module</var>=static</code>.</p>
290
291    <p>Additional modules are enabled using the
292    <code>--enable-<var>module</var></code> option, where
293    <var>module</var> is the name of the module with the
294    <code>mod_</code> string removed and with any underscore converted
295    to a dash.  Similarly, you can disable modules with the
296    <code>--disable-<var>module</var></code> option.  Be careful when
297    using these options, since <code class="program"><a href="./programs/configure.html">configure</a></code> cannot warn you
298    if the module you specify does not exist; it will simply ignore the
299    option.</p>
300
301    <p>In addition, it is sometimes necessary to provide the
302    <code class="program"><a href="./programs/configure.html">configure</a></code> script with extra information about the
303    location of your compiler, libraries, or header files.  This is
304    done by passing either environment variables or command line
305    options to <code class="program"><a href="./programs/configure.html">configure</a></code>.  For more information, see the
306    <code class="program"><a href="./programs/configure.html">configure</a></code> manual page. Or invoke
307    <code class="program"><a href="./programs/configure.html">configure</a></code> using the <code>--help</code> option.</p>
308
309    <p>For a short impression of what possibilities you have, here
310    is a typical example which compiles Apache for the installation
311    tree <code>/sw/pkg/apache</code> with a particular compiler and flags
312    plus the two additional modules <code class="module"><a href="./mod/mod_ldap.html">mod_ldap</a></code> and
313    <code class="module"><a href="./mod/mod_lua.html">mod_lua</a></code>:</p>
314
315<div class="example"><pre class="prettyprint lang-sh">$ CC="pgcc" CFLAGS="-O2" \
316  ./configure --prefix=/sw/pkg/apache \
317  --enable-ldap=shared \
318  --enable-lua=shared</pre>
319</div>
320
321    <p>When <code class="program"><a href="./programs/configure.html">configure</a></code> is run it will take several minutes to
322    test for the availability of features on your system and build
323    Makefiles which will later be used to compile the server.</p>
324
325    <p>Details on all the different <code class="program"><a href="./programs/configure.html">configure</a></code> options are
326    available on the <code class="program"><a href="./programs/configure.html">configure</a></code> manual page.</p>
327</div><div class="top"><a href="#page-header"><img alt="top" src="./images/up.gif" /></a></div>

342<h2><a name="install" id="install">Install</a></h2>
343
344    <p>Now it's time to install the package under the configured
345    installation <em>PREFIX</em> (see <code>--prefix</code> option

359
360    <p>Next, you can customize your Apache HTTP server by editing

367    <code><em>PREFIX</em>/docs/manual/</code> or consult <a href="http://httpd.apache.org/docs/2.4/">http://httpd.apache.org/docs/2.4/</a> for the most recent
368    version of this manual and a complete reference of available <a href="mod/directives.html">configuration directives</a>.</p>

371<h2><a name="test" id="test">Test</a></h2>
372
373    <p>Now you can <a href="invoking.html">start</a> your Apache
374    HTTP server by immediately running:</p>
375
376<div class="example"><p><code>$ <em>PREFIX</em>/bin/apachectl -k start</code></p></div>
377
378    <p>You should then be able to request your first document
379    via the URL <code>http://localhost/</code>. The web page you see is located
380    under the <code class="directive"><a href="./mod/core.html#documentroot">DocumentRoot</a></code>,
381    which will usually be <code><em>PREFIX</em>/htdocs/</code>.
382    Then <a href="stopping.html">stop</a> the server again by
383    running:</p>
384
385<div class="example"><p><code>$ <em>PREFIX</em>/bin/apachectl -k stop</code></p></div>

388<h2><a name="upgrading" id="upgrading">Upgrading</a></h2>
389
390    <p>The first step in upgrading is to read the release announcement
391    and the file <code>CHANGES</code> in the source distribution to
392    find any changes that may affect your site.  When changing between
393    major releases (for example, from 2.0 to 2.2 or from 2.2 to 2.4),
394    there will likely be major differences in the compile-time and
395    run-time configuration that will require manual adjustments.  All
396    modules will also need to be upgraded to accommodate changes in the
397    module API.</p>
398
399    <p>Upgrading from one minor version to the next (for example, from
400    2.2.55 to 2.2.57) is easier.  The <code>make install</code>
401    process will not overwrite any of your existing documents, log
402    files, or configuration files.  In addition, the developers make
403    every effort to avoid incompatible changes in the
404    <code class="program"><a href="./programs/configure.html">configure</a></code> options, run-time configuration, or the
405    module API between minor versions.  In most cases you should be able to
406    use an identical <code class="program"><a href="./programs/configure.html">configure</a></code> command line, an identical
407    configuration file, and all of your modules should continue to
408    work.</p>
409
410    <p>To upgrade across minor versions, start by finding the file
411    <code>config.nice</code> in the <code>build</code> directory of
412    your installed server or at the root of the source tree for your
413    old install.  This will contain the exact
414    <code class="program"><a href="./programs/configure.html">configure</a></code> command line that you used to
415    configure the source tree.  Then to upgrade from one version to
416    the next, you need only copy the <code>config.nice</code> file to
417    the source tree of the new version, edit it to make any desired
418    changes, and then run:</p>
419
420    <div class="example"><pre class="prettyprint lang-sh">$ ./config.nice
421$ make
422$ make install
423$ <em>PREFIX</em>/bin/apachectl -k graceful-stop
424$ <em>PREFIX</em>/bin/apachectl -k start</pre>
425</div>
426
427    <div class="warning">You should always test any new version in your
428    environment before putting it into production.  For example, you
429    can install and run the new version along side the old one by
430    using a different <code>--prefix</code> and a
431    different port (by adjusting the <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code> directive) to test for any
432    incompatibilities before doing the final upgrade.</div>
433
434    <p>You can pass additional arguments to <code>config.nice</code>,
435    which will be appended to your original <code class="program"><a href="./programs/configure.html">configure</a></code>
436    options:</p>
437
438    <div class="example"><p><code>
439    $ ./config.nice --prefix=/home/test/apache --with-port=90
440    </code></p></div>

443<h2><a name="thirdp" id="thirdp">Third-party packages</a></h2>
444
445    <p>A large number of third parties provide their own packaged
446    distributions of the Apache HTTP Server for installation on
447    particular platforms. This includes the various Linux distributions,
448    various third-party Windows packages, Mac OS X, Solaris, and many
449    more.</p>
450
451    <p>Our software license not only permits, but encourages, this kind
452    of redistribution. However, it does result in a situation where the
453    configuration layout and defaults on your installation of the server
454    may differ from what is stated in the documentation. While
455    unfortunate, this situation is not likely to change any time
456    soon.</p>
457
458    <p>A <a href="https://cwiki.apache.org/confluence/display/httpd/DistrosDefaultLayout">description
459    of these third-party distributions</a> is maintained in the HTTP
460    Server wiki, and should reflect the current state of these
461    third-party distributions. However, you will need to familiarize
462    yourself with your particular platform's package management and
463    installation procedures.</p>
