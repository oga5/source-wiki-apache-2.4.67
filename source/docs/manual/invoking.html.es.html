56<h2><a name="startup" id="startup">C&#243;mo iniciar Apache</a></h2>
57
58    <p>Si el puerto especificado en la directiva <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code> del fichero de
59    configuraci&#243;n es el que viene por defecto, es decir, el
60    puerto 80 (o cualquier otro puerto por debajo del 1024), entonces
61    es necesario tener privilegios de usuario root (superusuario) para
62    iniciar Apache, de modo que pueda establecerse una conexi&#243;n a
63    trav&#233;s de esos puertos privilegiados. Una vez que el servidor
64    Apache se ha iniciado y ha completado algunas tareas preliminares,
65    tales como abrir sus ficheros log, lanzar&#225; varios procesos,
66    procesos <em>hijo</em>, que hacen el trabajo de escuchar y atender
67    las peticiones de los clientes.  El proceso principal,
68    <code>httpd</code> contin&#250;a ejecut&#225;ndose con el usuario root, pero los
69    procesos hijo se ejecutan con menores privilegios de usuario.
70    Esto lo controla el <a href="mpm.html">M&#243;dulo de
71    MultiProcesamiento (MPM)</a> seleccionado.</p>
72
73    <p>La forma recomendada para invocar el ejecutable
74    <code class="program"><a href="./programs/httpd.html">httpd</a></code> es usando el script de control 
75    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code>.  Este script fija
76    determinadas variables de entorno que son necesarias para que
77    <code class="program"><a href="./programs/httpd.html">httpd</a></code> funcione correctamente en el sistema operativo,
78    y despu&#233;s invoca el binario <code class="program"><a href="./programs/httpd.html">httpd</a></code>.
79    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> pasa a <code class="program"><a href="./programs/httpd.html">httpd</a></code>
80    cualquier argumento que se le pase a trav&#233;s de la l&#237;nea de comandos, 
81    de forma que cualquier opci&#243;n de <code>httpd</code> puede ser usada
82    tambi&#233;n con <code>apachectl</code>.  Puede editar
83    directamente el script <code>apachectl</code> y cambiar la
84    variable <code>HTTPD</code> variable que est&#225; al principio y
85    que especifica la ubicaci&#243;n exacta en la que est&#225; el
86    binario <code class="program"><a href="./programs/httpd.html">httpd</a></code> y cualquier argumento de l&#237;nea de
87    comandos que quiera que est&#233; <em>siempre</em> presente.</p>
88
89    <p>La primera cosa que hace <code class="program"><a href="./programs/httpd.html">httpd</a></code> cuando es invocado
90    es localizar y leer el <a href="configuring.html">fichero de
91    configuraci&#243;n</a> <code>httpd.conf</code>. El lugar en el que
92    est&#225; ese fichero se determina al compilar, pero tambi&#233;n
93    es posible especificar la ubicaci&#243;n en la que se encuentra al
94    iniciar el servidor Apache usando la opci&#243;n de l&#237;nea de
95    comandos <code>-f</code></p>
96
97<div class="example"><p><code>/usr/local/apache2/bin/apachectl -f
98      /usr/local/apache2/conf/httpd.conf</code></p></div>
99
100    <p>Si todo va bien durante el arranque, la sesi&#243;n de terminal
101    se suspender&#225; un momento y volver&#225; a estar activa casi
102    inmediatamente. Esto quiere decir que el servidor est&#225; activo
103    y funcionando. Puede usar su navegador para conectarse al
104    servidor y ver la p&#225;gina de prueba que hay en el directorio de
105    la directiva
106    <code class="directive"><a href="./mod/core.html#documentroot">DocumentRoot</a></code>.</p>

109<h2><a name="errors" id="errors">Errores Durante el Arranque</a></h2>
110
111    <p>Si Apache encuentra una error irrecuperable durante el
112    arranque, escribir&#225; un mensaje describiendo el problema en la
113    consola o en el archivo <code class="directive"><a href="./mod/core.html#errorlog">ErrorLog</a></code> antes de abortar la
114    ejecuci&#243;n. Uno de los mensajes de error m&#225;s comunes es
115    "<code>Unable to bind to Port ...</code>". Cuando se recibe este
116    mensaje es normalmente por alguna de las siguientes razones:</p>
117
118    <ul>
119      <li>Est&#225; intentando iniciar el servidor Apache en un puerto
120      privilegiado (del 0 al 1024) sin haber hecho login como usuario
121      root; &#243; bien</li>
122
123      <li>Est&#225; intentando iniciar el servidor Apache mientras
124      est&#225; ya ejecutando Apache o alg&#250;n otro servidor web en
125      el mismo puerto.</li>
126    </ul>
127
128    <p>Puede encontrar m&#225;s informaci&#243;n sobre c&#243;mo
129    solucionar problemas, en la secci&#243;n de <a href="https://cwiki.apache.org/confluence/display/httpd/FAQ">Preguntas Frecuentes</a> de Apache.</p>

132<h2><a name="boot" id="boot">Iniciar Apache al Iniciar el Sistema</a></h2>
133
134    <p>Si quiere que el servidor Apache contin&#250;e su ejecuci&#243;n
135    despu&#233;s de reiniciar el sistema, debe a&#241;adir una llamada
136    a <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> en sus archivos de arranque (normalmente
137    <code>rc.local</code> o un fichero en ese directorio del tipo
138    <code>rc.N</code>). Esto iniciar&#225; Apache como usuario
139    root. Antes de hacer esto, aseg&#250;rese de que la
140    configuraci&#243;n de seguridad y las restricciones de acceso de
141    su servidor Apache est&#225;n correctamente configuradas.</p>
142
143    <p>El script <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> est&#225; dise&#241;ado para
144    actuar como un script est&#225;ndar de tipo <code>SysV init</code>; puede tomar los
145    argumentos <code>start</code>, <code>restart</code>, y
146    <code>stop</code> y traducirlos en las se&#241;ales apropiadas
147    para <code class="program"><a href="./programs/httpd.html">httpd</a></code>.  De esta manera, casi siempre puede
148    simplemente enlazar <code class="program"><a href="./programs/apachectl.html">apachectl</a></code>con el directorio init
149    adecuado. Pero aseg&#250;rese de comprobar los requisitos exactos
150    de su sistema.</p>
