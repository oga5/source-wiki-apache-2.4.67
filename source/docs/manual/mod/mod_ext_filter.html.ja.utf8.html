42<h3>概要</h3>
43
44    <p><code class="module"><a href="../mod/mod_ext_filter.html">mod_ext_filter</a></code> では <a href="../filter.html">フィルタ</a>
45    の慣れ親しんだ単純なプログラミングモデルが提供されます。このモジュールを
46    使えば、標準入力から読み込んで、標準出力に書き出すプログラム
47    (すなわち Unix 形式のフィルタコマンド) を Apache のフィルタにすることが
48    できます。このフィルタの機構は、Apache API 向けに書かれた Apache
49    サーバプロセス内で実行される専用のフィルタよりもずっと遅いですが、
50    以下のような利点もあります。</p>
51
52    <ul>
53      <li>ずっとシンプルなプログラミングモデル</li>
54
55      <li>プログラムが標準入力から読んで標準出力に書くものである限り、
56      どんなプログラム言語やスクリプト言語でも使うことができる</li>
57
58      <li>既存のプログラムを変更することなく Apache のフィルタとして
59      使うことができる</li>
60    </ul>
61
62    <p>性能の問題により実運用に適さないとしても、フィルタのプロトタイプ用の
63    環境としては <code class="module"><a href="../mod/mod_ext_filter.html">mod_ext_filter</a></code> は使えます。</p>

82    <h3>他のタイプのレスポンスから HTML を生成する</h3>
83      <div class="example"><p><code>
84        # mod_ext_filter directive to define a filter<br />
85        # to HTML-ize text/c files using the external<br />
86        # program /usr/bin/enscript, with the type of<br />
87        # the result set to text/html<br />
88        ExtFilterDefine c-to-html mode=output \<br />
89        <span class="indent">
90          intype=text/c outtype=text/html \<br />
91          cmd="/usr/bin/enscript --color -W html -Ec -o - -"<br />
92        </span>
93        <br />
94        &lt;Directory "/export/home/trawick/apacheinst/htdocs/c"&gt;<br />
95        <span class="indent">
96          # core directive to cause the new filter to<br />
97          # be run on output<br />
98          SetOutputFilter c-to-html<br />
99          <br />
100          # mod_mime directive to set the type of .c<br />
101          # files to text/c<br />
102          AddType text/c .c<br />
103          <br />
104          # mod_ext_filter directive to set the debug<br />
105          # level just high enough to see a log message<br />
106          # per request showing the configuration in force<br />
107          ExtFilterOptions DebugLevel=1<br />
108        </span>
109        &lt;/Directory&gt;
110      </code></p></div>

113    <h3>コンテントエンコーディングのフィルタを実装する</h3>
114      <p>注: この gzip の例はデモ用です。実用的な実装は
115      <code class="module"><a href="../mod/mod_deflate.html">mod_deflate</a></code> を参照してください。</p>
116
117      <div class="example"><p><code>
118        # mod_ext_filter directive to define the external filter<br />
119        ExtFilterDefine gzip mode=output cmd=/bin/gzip<br />
120        <br />
121        &lt;Location /gzipped&gt;<br />
122        <span class="indent">
123          # core directive to cause the gzip filter to be<br />
124          # run on output<br />
125          SetOutputFilter gzip<br />
126          <br />
127          # mod_header directive to add<br />
128          # "Content-Encoding: gzip" header field<br />
129          Header set Content-Encoding gzip<br />
130        </span>
131        &lt;/Location&gt;
132      </code></p></div>

135    <h3>サーバを遅くする</h3>
136      <div class="example"><p><code>
137        # mod_ext_filter directive to define a filter<br />
138        # which runs everything through cat; cat doesn't<br />
139        # modify anything; it just introduces extra pathlength<br />
140        # and consumes more resources<br />
141        ExtFilterDefine slowdown mode=output cmd=/bin/cat \<br />
142        <span class="indent">
143          preservescontentlength<br />
144        </span>

146        &lt;Location /&gt;<br />
147        <span class="indent">
148          # core directive to cause the slowdown filter to<br />
149          # be run several times on output<br />
150          #<br />
151          SetOutputFilter slowdown;slowdown;slowdown<br />
152        </span>
153        &lt;/Location&gt;
154      </code></p></div>

177    <h3>別のフィルタのトレース</h3>
178      <div class="example"><p><code>
179        # Trace the data read and written by mod_deflate<br />
180        # for a particular client (IP 192.168.1.31)<br />
181        # experiencing compression problems.<br />
182        # This filter will trace what goes into mod_deflate.<br />
183        ExtFilterDefine tracebefore \<br />
184        <span class="indent">
185          cmd="/bin/tracefilter.pl /tmp/tracebefore" \<br />
186          EnableEnv=trace_this_client<br />
187        </span>
188        <br />
189        # This filter will trace what goes after mod_deflate.<br />
190        # Note that without the ftype parameter, the default<br />
191        # filter type of AP_FTYPE_RESOURCE would cause the<br />
192        # filter to be placed *before* mod_deflate in the filter<br />
193        # chain.  Giving it a numeric value slightly higher than<br />
194        # AP_FTYPE_CONTENT_SET will ensure that it is placed<br />
195        # after mod_deflate.<br />
196        ExtFilterDefine traceafter \<br />
197        <span class="indent">
198          cmd="/bin/tracefilter.pl /tmp/traceafter" \<br />
199          EnableEnv=trace_this_client ftype=21<br />
200        </span>
201        <br />
202        &lt;Directory /usr/local/docs&gt;<br />
203        <span class="indent">
204          SetEnvIf Remote_Addr 192.168.1.31 trace_this_client<br />
205          SetOutputFilter tracebefore;deflate;traceafter<br />
206        </span>
207        &lt;/Directory&gt;
208      </code></p></div>

231<div class="directive-section"><h2><a name="ExtFilterDefine" id="ExtFilterDefine">ExtFilterDefine</a> <a name="extfilterdefine" id="extfilterdefine">ディレクティブ</a></h2>
232<table class="directive">
233<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>外部フィルタを定義</td></tr>
234<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>ExtFilterDefine <var>filtername</var> <var>parameters</var></code></td></tr>
235<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル</td></tr>
236<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
237<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_ext_filter</td></tr>
238</table>
239    <p><code class="directive">ExtFilterDefine</code> は、実行するプログラムや
240    引数など、外部フィルタの特性を定義します。</p>
241
242    <p><var>filtername</var> は定義するフィルタの名前を指定します。
243    この名前は後で <code class="directive"><a href="../mod/core.html#setoutputfilter">SetOutputFilter</a></code>
244    ディレクティブで指定できます。名前は登録されるすべてのフィルタで
245    一意でなくてはなりません。<em>現時点では、フィルタの登録 API からは
246    エラーは報告されません。ですから、重複する名前を使ってしまったときでも
247    ユーザにはそのことは報告されません。</em></p>
248
249    <p>続くパラメータの順番は関係無く、それらは実行する外部コマンドと、
250    他の特性を定義します。<code>cmd=</code> だけが必須のパラメータです。
251    指定可能なパラメータは:</p>
252
253    <dl>
254      <dt><code>cmd=<var>cmdline</var></code></dt>
255
256      <dd><code>cmd=</code> キーワードは実行する外部コマンドを指定します。
257      プログラム名の後に引数がある場合は、コマンド行は引用符で囲む
258      必要があります (<em>例えば</em>、<code>cmd="<var>/bin/mypgm</var>
259      <var>arg1</var> <var>arg2</var>"</code> のように)。プログラムは
260      シェル経由でなく、直接実行されますので、通常のシェル用の
261      エスケープは必要ありません。プログラムの引数は空白で区切られます。
262      プログラムの引数の一部となる必要のある空白はバックスペースでエスケープ
263      できます。引数の一部になるバックスラッシュはバックスラッシュで
264      エスケープする必要があります。標準の CGI 環境変数に加えて、
265      環境変数 DOCUMENT_URI, DOCUMENT_PATH_INFO, and 
266      QUERY_STRING_UNESCAPED がプログラムのために設定されます。</dd>
267
268      <dt><code>mode=<var>mode</var></code></dt>
269
270      <dd>応答を処理するフィルタには <code>mode=output</code> (デフォルト)
271      を使います。リクエストを処理するフィルタには <code>mode=input</code>
272      を使います。<code>mode=input</code> は Apache 2.1 以降で利用可能です。</dd>
273
274      <dt><code>intype=<var>imt</var></code></dt>
275
276      <dd>このパラメータはフィルタされるべきドキュメントの
277      インターネットメディアタイプ (<em>すなわち</em>、MIME タイプ) を
278      指定します。デフォルトではすべてのドキュメントがフィルタされます。
279      <code>intype=</code> が指定されていれば、フィルタは指定されていない
280      ドキュメントには適用されなくなります。</dd>
281
282      <dt><code>outtype=<var>imt</var></code></dt>
283
284      <dd>このパラメータはフィルタされたドキュメントの
285      インターネットメディアタイプ (<em>すなわち</em>、MIME タイプ) を
286      指定します。フィルタ動作にともなってインターネットメディアタイプが
287      変わる場合に有用です。デフォルトではインターネットメディアタイプは
288      変更されません。</dd>
289
290      <dt><code>PreservesContentLength</code></dt>
291
292      <dd><code>PreservesContentLength</code> キーワードはフィルタが
293      content length <span class="transnote">(<em>訳注:</em> コンテントの長さ)</span>
294      を変更しないということを指定します。ほとんどのフィルタは
295      content length を変更するため、これはデフォルトではありません。
296      フィルタが長さを変えないときは、このキーワードを指定すると
297      よいでしょう。</dd>
298
299      <dt><code>ftype=<var>filtertype</var></code></dt>
300
301      <dd>このパラメータはフィルタが登録されるべきフィルタタイプの
302      数値を指定します。ほとんどの場合は、デフォルトの AP_FTYPE_RESOURCE で
303      十分です。フィルタがフィルタチェーンの別の場所で動作する必要がある
304      場合は、このパラメータを指定する必要があります。指定可能な値は
305      util_filter.h の AP_FTYPE_foo 定義を参照してください。</dd>
306
307      <dt><code>disableenv=<var>env</var></code></dt>
308
309      <dd>設定されていた場合にフィルタを無効にするための環境変数を
310      指定します。</dd>
311
312      <dt><code>enableenv=<var>env</var></code></dt>
313
314      <dd>このパラメータはフィルタが有効になるために設定されていなければ
315      ならない環境変数を指定します。</dd>
316    </dl>

320<div class="directive-section"><h2><a name="ExtFilterOptions" id="ExtFilterOptions">ExtFilterOptions</a> <a name="extfilteroptions" id="extfilteroptions">ディレクティブ</a></h2>
321<table class="directive">
322<tr><th><a href="directive-dict.html#Description">説明:</a></th><td><code class="module"><a href="../mod/mod_ext_filter.html">mod_ext_filter</a></code> のオプションを設定</td></tr>
323<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>ExtFilterOptions <var>option</var> [<var>option</var>] ...</code></td></tr>
324<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>ExtFilterOptions DebugLevel=0 NoLogStderr</code></td></tr>
325<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>ディレクトリ</td></tr>
326<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
327<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_ext_filter</td></tr>
328</table>
329    <p><code class="directive">ExtFilterOptions</code> ディレクティブは
330    <code class="module"><a href="../mod/mod_ext_filter.html">mod_ext_filter</a></code> の特別な処理用のオプションを
331    指定します。<var>Option</var> には以下のどれかを指定します。</p>
332
333    <dl>
334      <dt><code>DebugLevel=<var>n</var></code></dt>
335
336      <dd>
337        <code>DebugLevel</code> で <code class="module"><a href="../mod/mod_ext_filter.html">mod_ext_filter</a></code> 
338        の生成するデバッグメッセージのレベルを設定できます。
339        デフォルトでは、デバッグメッセージは生成されません。
340        これは <code>DebugLevel=0</code> と設定するのと同じです。
341        数字が大きくなればなるほど、より多くのデバッグメッセージが
342        生成され、サーバの性能は落ちます。数値の実際の意味は
343        <code>mod_ext_filter.c</code> の先頭近くの DBGLVL_ 定数の
344        定義で説明されています。
345
346        <p>注: デバッグメッセージを Apache のエラーログに
347        保存するようにするためには、core のディレクティブ 
348        <code class="directive"><a href="../mod/core.html#loglevel">LogLevel</a></code> 
349        を使う必要があります。</p>
350      </dd>
351
352      <dt><code>LogStderr | NoLogStderr</code></dt>
353
354      <dd><code>LogStderr</code> キーワードは外部フィルタプログラムにより
355      標準エラー <span class="transnote">(<em>訳注:</em> stderr)</span> に書かれたメッセージを
356      Apache のエラーログに保存するようにします。<code>NoLogStderr</code> は
357      逆に保存しないようにします。</dd>
358    </dl>
359
360    <div class="example"><h3>例</h3><p><code>
361      ExtFilterOptions LogStderr DebugLevel=0
362    </code></p></div>
363
364    <p>この例では、フィルタの標準出力に書かれたメッセージは
365    Apache のエラーログに保存されます。<code class="module"><a href="../mod/mod_ext_filter.html">mod_ext_filter</a></code> からは
366    デバッグメッセージは生成されません。</p>
