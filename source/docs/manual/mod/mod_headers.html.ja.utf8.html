44<h3>概要</h3>
45

47    変更するためのディレクティブを提供します。ヘッダを追加したり、
48    置き換えたり、削除したりすることができます。</p>

65<h2><a name="order" id="order">処理の順番</a></h2>
66
67    <p><code class="module"><a href="../mod/mod_headers.html">mod_headers</a></code> のディレクティブはサーバ設定のほぼどこにでも
68    書くことができ、影響する範囲を<a href="../sections.html">設定用セクション</a>で囲むことで限定する
69    ことができます。</p>
70
71    <p>処理の順番は重要で、設定ファイル中の順番と、<a href="../sections.html">設定用セクション</a>内の位置との両方に
72    影響されます。以下の二つのヘッダは順番が逆になると
73    違う結果になります:</p>
74
75    <div class="example"><p><code>
76      RequestHeader append MirrorID "mirror 12"<br />
77      RequestHeader unset MirrorID
78    </code></p></div>
79
80    <p>この順番の場合は、<code>MirrorID</code> ヘッダは設定されません。
81    逆になっていると、MirrorID ヘッダは "mirror 12" に設定されます。</p>
82</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

84<h2><a name="early" id="early">早期処理、後期処理</a></h2>
85    <p><code class="module"><a href="../mod/mod_headers.html">mod_headers</a></code> では、リクエストの早期か後期かの
86    どちらで適用するかを選べます。通常は後期モードで、
87    コンテンツ生成が実行される直前にリクエストヘッダがセットされ、
88    レスポンスとして送出される直前にレスポンスヘッダがセットされます。
89    運用中のサーバでは必ず後期モードを使ってください。</p>
90
91    <p>早期モードは開発者向けのテスト/デバッグ用に設計されています。
92    <code>early</code> キーワード指定されたディレクティブによって、
93    リクエスト処理の開始地点になります。
94    つまり、異なるリクエストを試したりテストケースをセットアップするのに
95    活用できる一方で、レスポンスを生成する前に他のモジュールによって
96    ヘッダが書き換えられてしまうかもしれないということを意味します。</p>
97
98    <p>early ディレクティブではリクエストパスの設定が解決される前に
99    処理されるので、メインサーバかバーチャルホストコンテキストでのみ、
100    早期ヘッダをセットできます。early ディレクティブはリクエストパスに
101    依存することはできませんので、<code>&lt;Directory&gt;</code> や
102    <code>&lt;Location&gt;</code> といったコンテキスト内では使用
103    できません。</p>

106<h2><a name="examples" id="examples">例</a></h2>
107
108    <ol>
109      <li>リクエストヘッダ中の "TS" で始まるフィールドをすべて応答ヘッダに
110        コピーします:
111        <div class="example"><p><code>
112          Header echo ^TS
113        </code></p></div>
114      </li>
115
116      <li>
117        リクエストを受け付けた時刻とリクエストを処理した時間を入れたヘッダ、
118        <code>MyHeader</code> を応答に追加します。このヘッダはクライアントが
119        サーバの負荷を直観的に知るためや、クライアント-サーバ間の
120        ボトルネックを調べるために使うことができます。
121
122        <div class="example"><p><code>
123           Header add MyHeader "%D %t"
124        </code></p></div>
125
126        <p>上記の設定では、以下のようなヘッダが応答に追加されることになります:</p>
127
128        <div class="example"><p><code>
129           MyHeader: D=3775428 t=991424704447256
130        </code></p></div>
131      </li>
132
133      <li>
134        Joe にあいさつをします:
135
136        <div class="example"><p><code>
137           Header add MyHeader "Hello Joe. It took %D microseconds for Apache to serve this request."
138        </code></p></div>
139
140        <p>以下のようなヘッダが応答に追加されることになります</p>
141
142        <div class="example"><p><code>
143           MyHeader: Hello Joe. It took D=3775428 microseconds for Apache to serve this request.
144        </code></p></div>
145      </li>
146
147      <li>リクエストに "MyRequestHeader" があるときに限り <code>MyHeader</code> を応答に
148      付けます。これは、クライアントの要求に応えてヘッダを作成するときに
149      役に立ちます。この例では <code class="module"><a href="../mod/mod_setenvif.html">mod_setenvif</a></code> モジュールが必要なことに
150      注意してください。
151
152        <div class="example"><p><code>
153           SetEnvIf MyRequestHeader value HAVE_MyRequestHeader<br />
154           Header add MyHeader "%D %t mytext" env=HAVE_MyRequestHeader
155        </code></p></div>
156
157        <p>もし HTTP リクエストに <code>MyRequestHeader: value</code> ヘッダが
158        あると、応答には以下のようなヘッダが付加されます。</p>
159
160        <div class="example"><p><code>
161           MyHeader: D=3775428 t=991424704447256 mytext
162        </code></p></div>
163      </li>
164    </ol>

167<div class="directive-section"><h2><a name="Header" id="Header">Header</a> <a name="header" id="header">ディレクティブ</a></h2>
168<table class="directive">
169<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>HTTP 応答ヘッダの設定</td></tr>
170<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>Header [<var>condition</var>] set|append|add|unset|echo
171<var>header</var> [<var>value</var>] [early|env=[!]<var>variable</var>]</code></td></tr>
172<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト, ディレクトリ, .htaccess</td></tr>
173<tr><th><a href="directive-dict.html#Override">上書き:</a></th><td>FileInfo</td></tr>
174<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
175<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_headers</td></tr>
176</table>
177    <p>このディレクティブは HTTP 応答ヘッダを置換、追加、削除できます。
178    ヘッダはコンテントハンドラや出力フィルタが実行された直後に実行され、
179    出て行くヘッダを変更できるようになっています。</p>
180
181    <p>オプションの <var>condition</var> は <code>onsuccess</code> か
182    <code>always</code> のどちらかを指定できます。これは内部ヘッダテーブルのどれを
183    操作するかを決定します。<code>onsuccess</code> は <code>2<var>xx</var></code>
184    ステータスコードの、<code>always</code> は全てのステータスコード
185    (<code>2<var>xx</var></code> を含む) の意味になります。
186    あるモジュールでセットされるヘッダをアンセットしたい場合は特に、
187    どのテーブルが影響を受けるかを実際に試したほうがよいでしょう。</p>
188
189    <p>行なう処理は二番目のの引数で決まります。
190    この引数には次の値を指定できます:</p>
191
192    <dl>
193    <dt><code>set</code></dt>
194    <dd>応答ヘッダを設定します。同じ名前のヘッダが存在する場合はそれを
195    置き換えます。<em>value</em> にはフォーマット文字列を
196    指定することもできます。</dd>
197
198    <dt><code>append</code></dt>
199    <dd>応答ヘッダを既に存在する同じ名前のヘッダに追加します。
200    新しい値が既存のヘッダに追加されるときには、既存のヘッダの
201    後にコンマで区切られて追加されます。これはヘッダに複数の値を
202    指定するときの HTTP の標準の方法です。</dd>
203
204    <dt><code>add</code></dt>
205    <dd>ヘッダが既に存在しているときでさえも、応答ヘッダを
206    既存のヘッダに追加します。これにより、二つ (かそれ以上) の
207    ヘッダの名前が同じになることがあります。その結果、想定できない
208    ことが起こる可能性がありますので、一般的には <code>append</code> の方を
209    使う方が良いでしょう。</dd>
210
211    <dt><code>unset</code></dt>
212    <dd>もし指定された名前の応答ヘッダが存在していれば、削除されます。
213    同じ名前のヘッダが複数あるときは、すべて削除されます。
214    <var>value</var> をつけてはいけません。</dd>
215
216    <dt><code>echo</code></dt>
217    <dd>指定されたものと同じ名前のリクエストヘッダを応答ヘッダで
218    そのまま返します。<var>header</var> には正規表現も指定できます。
219    <var>value</var> をつけてはいけません。</dd>
220    </dl>
221
222    <p>この引数の後にはヘッダ名 (<var>header</var>) が続きます。
223    ヘッダ名には最後にコロンを含めることもできますが、無くても構いません。
224    <code>set</code>, <code>append</code>, <code>add</code>,
225    <code>unset</code> では大文字小文字は
226    区別されません。echo の <em>header</em> 名は大文字小文字を区別し、
227    正規表現を指定することもできます。</p>
228
229    <p><code>add</code>, <code>append</code>,
230    <code>set</code> では <var>value</var> を三つ目の
231    引数として指定します。<var>value</var> に空白がある場合は二重引用符で
232    囲む必要があります。<var>value</var> は文字のみからなる文字列、
233    フォーマット指示子を含む文字列、もしくは両方からなる文字列を指定できます。
234    <var>value</var> は以下のフォーマット指示子をサポートします:</p>
235
236    <table class="bordered"><tr class="header"><th>フォーマット</th><th>解説</th></tr>
237<tr><td><code>%%</code></td>
238        <td>パーセント記号</td></tr>
239<tr class="odd"><td><code>%t</code></td>
240        <td>リクエストを受け取った時刻を、
241        Universal Coordinated Time での始まりの時刻 (Jan. 1, 1970) から経過した
242        時間をマイクロ秒として現したもの。値の最初には 
243        <code>t=</code> が付加されます。</td></tr>
244<tr><td><code>%D</code></td>
245        <td>リクエストを受け取った時刻と、ヘッダを送り出した
246        時間との差。これは、リクエストが存在していた期間を現します。
247        値の最初には <code>D=</code> が付加されます。</td></tr>
248<tr class="odd"><td><code>%{FOOBAR}e</code></td>
249        <td><a href="../env.html">環境変数</a>
250        <code>FOOBAR</code> の値です。</td></tr>
251<tr><td><code>%{FOOBAR}s</code></td>
252        <td><code class="module"><a href="../mod/mod_ssl.html">mod_ssl</a></code> が有効な場合、
253        <a href="mod_ssl.html#envvars">SSL 環境変数</a> <code>FOOBAR</code>
254        の内容</td></tr>
255</table>
256
257    <div class="note"><h3>注</h3>
258      <p><code>%s</code> フォーマット指定子は 2.1 以降でのみ利用できます。
259      <code>SSLOptions +StdEnvVars</code> を有効にすることによるオーバーヘッドを
260      避けるため、<code>%e</code> の代わりとして使えます。
261      他の理由などがあって、どうしても <code>SSLOptions +StdEnvVars</code> 
262      を有効にしなければならない場合は、<code>%e</code> のほうが
263      <code>%s</code> よりも処理効率は良いです。</p>
264    </div> 
265
266    <p><code class="directive">Header</code> ディレクティブには追加の引数を持たせることが
267    できて、どういったアクションが行われたかの条件を指定したり、
268    <a href="#early">早期処理</a> を指定する <code>early</code> キーワードを
269    指定できます。
270    <code>env=...</code> 引数で指定された <a href="../env.html">環境変数</a> が存在する (もしくは <code>env=!...</code>
271    が指定されていて環境変数が存在しない) 場合は、<code class="directive">Header</code>
272    ディレクティブで指定された動作が行なわれます。そうでない場合は、
273    ディレクティブはそのリクエストには何もしません。</p>
274
275    <p><a href="#early">早期処理</a>モードの場合以外では、
276    <code class="directive">Header</code>
277    ディレクティブは応答がネットワークに送られる直前に
278    処理されます。これは、ヘッダフィルタにより追加されるヘッダを
279    除き、ほとんどのヘッダを設定したり上書きしたりすることが
280    可能、ということです。</p>

284<div class="directive-section"><h2><a name="RequestHeader" id="RequestHeader">RequestHeader</a> <a name="requestheader" id="requestheader">ディレクティブ</a></h2>
285<table class="directive">
286<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>HTTP リクエストヘッダの設定</td></tr>
287<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>RequestHeader set|append|add|unset <var>header</var>
288[<var>value</var>] [early|env=[!]<var>variable</var>]</code></td></tr>

290<tr><th><a href="directive-dict.html#Override">上書き:</a></th><td>FileInfo</td></tr>
291<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>

294    <p>このディレクティブは HTTP リクエストヘッダを置換、追加、削除できます。
295    ヘッダはコンテントハンドラが実行される直前に実行され、
296    入って来るヘッダを変更することが可能になっています。
297    行なう処理は第 1 引数により決まります。これには以下の値を指定
298    することができます:</p>
299
300    <dl>
301    <dt><code>set</code></dt>
302    <dd>リクエストヘッダを設定します。同じ名前のヘッダが存在していると、
303    それを置き換えます。</dd>
304
305    <dt><code>append</code></dt>
306    <dd>リクエストヘッダは、既に存在する同じ名前のヘッダに追加されます。
307    新しい値が既存のヘッダに追加されるときには、既存のヘッダの
308    後にコンマで区切られて追加されます。これはヘッダに複数の値を
309    指定するときの HTTP の標準の方法です。</dd>
310
311    <dt><code>add</code></dt>
312    <dd>ヘッダが既に存在しているときでさえも、リクエストヘッダを
313    既存のヘッダに追加します。これにより、二つ (かそれ以上) の
314    ヘッダの名前が同じになることがあります。その結果、想定できない
315    ことが起こる可能性がありますので、一般的には <code>append</code> の方を
316    使う方が良いでしょう。</dd>
317
318    <dt><code>unset</code></dt>
319    <dd>もし指定された名前のリクエストヘッダが存在していれば、削除されます。
320    同じ名前の複数のヘッダがあるときは、すべて削除されます。
321    <var>value</var> をつけてはいけません。</dd>
322    </dl>
323
324    <p>この引数の後にはヘッダ名 (<em>header</em>) が続きます。
325    ヘッダ名には最後にコロンを含めることもできますが、無くても構いません。
326    大文字小文字は区別されません。<code>add</code>,
327    <code>append</code>, <code>set</code> の場合は、<em>value</em> が三つ目の
328    引数として指定されます。<em>value</em> に空白がある場合は二重引用符で
329    囲む必要があります。unset の場合は、<em>value</em> は指定しません。
330    <var>value</var> は文字列、フォーマット指定子、あるいは、その混合です。
331    使うことのできるフォーマット指定子は、<code class="directive"><a href="#header">Header</a></code> と同じですので、
332    詳細はそちらをご覧ください。</p>
333
334    <p><code class="directive">RequestHeader</code> ディレクティブは、
335    どういった条件下でアクションを行うかを指定する追加引数
336    あるいは、<a href="#early">早期処理</a> を指定する <code>early</code> 
337    キーワードを設定することができます。
338    <code>env=<var>...</var></code> の引数で設定されている
339    <a href="../env.html">環境変数</a> が存在している
340    (あるいは <code>env=!<var>...</var></code> で指定された環境変数が
341    存在しない) 場合、<code class="directive">RequestHeader</code> ディレクティブは
342    有効になります。それ以外の場合、ディレクティブは効力を持ちません。</p>
343
344    <p><a href="#early">early</a> モードでない場合に限り、
345    <code class="directive">RequestHeader</code> ディレクティブは
346    fixup フェーズでリクエストがハンドラに扱われる直前に
347    処理されます。これにより、ブラウザや Apache の入力フィルタにより
348    生成されたヘッダを上書きしたり修正したりできるようになっています。</p>
