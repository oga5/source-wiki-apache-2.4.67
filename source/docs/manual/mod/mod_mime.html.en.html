40<h3>Summary</h3>
41
42    <p>This module is used to assign content metadata to the content
43    selected for an HTTP response by mapping patterns in the
44    URI or filenames to the metadata values.  For example, the filename
45    extensions of content files often define the content's Internet
46    media type, language, character set, and content-encoding. This
47    information is sent in HTTP messages containing that content and
48    used in content negotiation when selecting alternatives, such that
49    the user's preferences are respected when choosing one of several
50    possible contents to serve. See
51    <code class="module"><a href="../mod/mod_negotiation.html">mod_negotiation</a></code> for more information
52    about <a href="../content-negotiation.html">content negotiation</a>.</p>
53
54    <p>The directives <code class="directive"><a href="#addcharset">AddCharset</a></code>, <code class="directive"><a href="#addencoding">AddEncoding</a></code>, <code class="directive"><a href="#addlanguage">AddLanguage</a></code> and <code class="directive"><a href="#addtype">AddType</a></code> are all used to map file
55    extensions onto the metadata for that file. Respectively
56    they set the character set, content-encoding, content-language,
57    and <a class="glossarylink" href="../glossary.html#media-type" title="see glossary">media-type</a> (content-type) of documents.  The directive <code class="directive"><a href="#typesconfig">TypesConfig</a></code> is used to specify a
58    file which also maps extensions onto media types. </p>
59
60    <p>In addition, <code class="module"><a href="../mod/mod_mime.html">mod_mime</a></code> may define the <a href="../handler.html">handler</a> and <a href="../filter.html">filters</a> that originate and process
61    content.  The directives <code class="directive"><a href="#addhandler">AddHandler</a></code>, <code class="directive"><a href="#addoutputfilter">AddOutputFilter</a></code>, and <code class="directive"><a href="#addinputfilter">AddInputFilter</a></code> control the modules
62    or scripts that serve the document.  The <code class="directive"><a href="#multiviewsmatch">MultiviewsMatch</a></code> directive allows
63    <code class="module"><a href="../mod/mod_negotiation.html">mod_negotiation</a></code> to consider these file extensions
64    to be included when testing Multiviews matches.</p>
65
66    <p>While <code class="module"><a href="../mod/mod_mime.html">mod_mime</a></code> associates metadata
67    with filename extensions, the <code class="module"><a href="../mod/core.html">core</a></code> server
68    provides directives that are used to associate all the files in a
69    given container (<em>e.g.</em>, <code class="directive"><a href="../mod/core.html#location">&lt;Location&gt;</a></code>, <code class="directive"><a href="../mod/core.html#directory">&lt;Directory&gt;</a></code>, or <code class="directive"><a href="../mod/core.html#files">&lt;Files&gt;</a></code>) with particular
70    metadata. These directives include <code class="directive"><a href="../mod/core.html#forcetype">ForceType</a></code>, <code class="directive"><a href="../mod/core.html#sethandler">SetHandler</a></code>, <code class="directive"><a href="../mod/core.html#setinputfilter">SetInputFilter</a></code>, and <code class="directive"><a href="../mod/core.html#setoutputfilter">SetOutputFilter</a></code>.  The core directives
71    override any filename extension mappings defined in
72    <code class="module"><a href="../mod/mod_mime.html">mod_mime</a></code>.</p>
73
74    <p>Note that changing the metadata for a file does not
75    change the value of the <code>Last-Modified</code> header.
76    Thus, previously cached copies may still be used by a client or
77    proxy, with the previous headers. If you change the
78    metadata (language, content type, character set or
79    encoding) you may need to 'touch' affected files (updating
80    their last modified date) to ensure that all visitors are
81    receive the corrected content headers.</p>

120<h2><a name="multipleext" id="multipleext">Files with Multiple Extensions</a></h2>
121    <p>Files can have more than one extension; the order of the
122    extensions is <em>normally</em> irrelevant. For example, if the
123    file <code>welcome.html.fr</code> maps onto content type
124    <code>text/html</code> and language French then the file
125    <code>welcome.fr.html</code> will map onto exactly the same
126    information.  If more than one extension is given that maps onto
127    the same type of metadata, then the one to the right will
128    be used, except for languages and content encodings. For example,
129    if <code>.gif</code> maps to the <a class="glossarylink" href="../glossary.html#media-type" title="see glossary">media-type</a>
130    <code>image/gif</code> and <code>.html</code> maps to the
131    media-type <code>text/html</code>, then the file
132    <code>welcome.gif.html</code> will be associated with the
133    media-type <code>text/html</code>.</p>
134
135    <p><a href="#charset-lang">Languages</a> and <a href="#contentencoding">content encodings</a> are treated accumulative, because one can assign
136    more than one language or encoding to a particular resource. For example,
137    the file <code>welcome.html.en.de</code> will be delivered with
138    <code>Content-Language: en, de</code> and <code>Content-Type:
139    text/html</code>.</p>
140
141    <p>Care should be taken when a file with multiple extensions
142    gets associated with both a <a class="glossarylink" href="../glossary.html#media-type" title="see glossary">media-type</a>
143    and a handler. This will
144    usually result in the request being handled by the module associated
145    with the handler. For example, if the <code>.imap</code>
146    extension is mapped to the handler <code>imap-file</code> (from
147    <code class="module"><a href="../mod/mod_imagemap.html">mod_imagemap</a></code>) and the <code>.html</code> extension is
148    mapped to the media-type <code>text/html</code>, then the file
149    <code>world.imap.html</code> will be associated with both the
150    <code>imap-file</code> handler and <code>text/html</code> media-type.
151    When it is processed, the <code>imap-file</code> handler will be used,
152    and so it will be treated as a <code class="module"><a href="../mod/mod_imagemap.html">mod_imagemap</a></code> imagemap
153    file.</p>
154
155    <p>If you would prefer only the last dot-separated part of the
156    filename to be mapped to a particular piece of meta-data, then do
157    not use the <code>Add*</code> directives. For example, if you wish
158    to have the file <code>foo.html.asis</code> served by the <code>send-as-is</code>
159    handler provided by <code class="module"><a href="../mod/mod_asis.html">mod_asis</a></code>
160    script, but not the file <code>bar.asis.html</code>, then instead
161    of using <code>AddHandler send-asis .asis</code>, use</p>
162
163    <div class="example"><h3>Configure handler based on final extension only</h3><pre class="prettyprint lang-config">&lt;FilesMatch "[^.]+\.asis$"&gt;
164  SetHandler send-asis
165&lt;/FilesMatch&gt;</pre>

170<h2><a name="contentencoding" id="contentencoding">Content encoding</a></h2>
171    <p>A file of a particular <a class="glossarylink" href="../glossary.html#media-type" title="see glossary">media-type</a> can additionally be encoded a
172    particular way to simplify transmission over the Internet.
173    While this usually will refer to compression, such as
174    <code>gzip</code>, it can also refer to encryption, such a
175    <code>pgp</code> or to an encoding such as UUencoding, which is
176    designed for transmitting a binary file in an ASCII (text)
177    format.</p>
178
179    <p>The <a href="http://www.ietf.org/rfc/rfc2616.txt">HTTP/1.1
180    RFC</a>, section 14.11 puts it this way:</p>
181
182    <blockquote cite="http://www.ietf.org/rfc/rfc2616.txt">
183      <p>The Content-Encoding entity-header field is used as a modifier to
184      the media-type. When present, its value indicates what additional
185      content codings have been applied to the entity-body, and thus what
186      decoding mechanisms must be applied in order to obtain the media-type
187      referenced by the Content-Type header field. Content-Encoding is
188      primarily used to allow a document to be compressed without losing
189      the identity of its underlying media type.</p>
190    </blockquote>
191
192    <p>By using more than one file extension (see <a href="#multipleext">section above about multiple file
193    extensions</a>), you can indicate that a file is of a
194    particular <em>type</em>, and also has a particular
195    <em>encoding</em>. </p>
196
197    <p>For example, you may have a file which is a Microsoft Word
198    document, which is pkzipped to reduce its size. If the
199    <code>.doc</code> extension is associated with the Microsoft
200    Word file type, and the <code>.zip</code> extension is
201    associated with the pkzip file encoding, then the file
202    <code>Resume.doc.zip</code> would be known to be a pkzip'ed Word
203    document.</p>
204
205    <p>Apache sends a <code>Content-encoding</code> header with the
206    resource, in order to tell the client browser about the
207    encoding method.</p>
208
209    <pre class="prettyprint lang-config">Content-encoding: pkzip</pre>

213<h2><a name="charset-lang" id="charset-lang">Character sets and languages</a></h2>
214    <p>In addition to file type and the file encoding,
215    another important piece of information is what language a
216    particular document is in, and in what character set the file
217    should be displayed. For example, the document might be written
218    in the Vietnamese alphabet, or in Cyrillic, and should be
219    displayed as such. This information, also, is transmitted in
220    HTTP headers.</p>
221
222    <p>The character set, language, encoding and mime type are all
223    used in the process of content negotiation (See
224    <code class="module"><a href="../mod/mod_negotiation.html">mod_negotiation</a></code>) to determine
225    which document to give to the client, when there are
226    alternative documents in more than one character set, language,
227    encoding or mime type. All filename extensions associations
228    created with <code class="directive"><a href="#addcharset">AddCharset</a></code>,
229    <code class="directive"><a href="#addencoding">AddEncoding</a></code>, <code class="directive"><a href="#addlanguage">AddLanguage</a></code> and <code class="directive"><a href="#addtype">AddType</a></code> directives
230    (and extensions listed in the <code class="directive"><a href="../mod/mod_mime_magic.html#mimemagicfile">MimeMagicFile</a></code>) participate in this select process.
231    Filename extensions that are only associated using the <code class="directive"><a href="#addhandler">AddHandler</a></code>, <code class="directive"><a href="#addinputfilter">AddInputFilter</a></code> or <code class="directive"><a href="#addoutputfilter">AddOutputFilter</a></code> directives may be included or excluded
232    from matching by using the <code class="directive"><a href="#multiviewsmatch">MultiviewsMatch</a></code> directive.</p>
233
234    <h3><a name="charset" id="charset">Charset</a></h3>
235      <p>To convey this further information, Apache optionally sends
236      a <code>Content-Language</code> header, to specify the language
237      that the document is in, and can append additional information
238      onto the <code>Content-Type</code> header to indicate the
239      particular character set that should be used to correctly
240      render the information.</p>
241
242      <div class="example"><p><code>
243Content-Language: en, fr
244Content-Type: text/plain; charset=ISO-8859-1
245      </code></p></div>
246
247      <p>The language specification is the two-letter abbreviation
248      for the language. The <code>charset</code> is the name of the
249      particular character set which should be used.</p>

255<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Maps the given filename extensions to the specified content
256charset</td></tr>
257<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>AddCharset <var>charset</var> <var>extension</var>

264    <p>The <code class="directive">AddCharset</code> directive maps the given
265    filename extensions to the specified content charset (the Internet
266    registered name for a given character encoding). <var>charset</var>
267    is the <a href="http://www.iana.org/assignments/character-sets">media
268    type's charset parameter</a> for resources with filenames containing
269    <var>extension</var>. This mapping is added to any already in force,
270    overriding any mappings that already exist for the same
271    <var>extension</var>.</p>
272
273    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">AddLanguage ja .ja
274AddCharset EUC-JP .euc
275AddCharset ISO-2022-JP .jis
276AddCharset SHIFT_JIS .sjis</pre>
277</div>
278
279    <p>Then the document <code>xxxx.ja.jis</code> will be treated
280    as being a Japanese document whose charset is <code>ISO-2022-JP</code>
281    (as will the document <code>xxxx.jis.ja</code>). The
282    <code class="directive">AddCharset</code> directive is useful for both to
283    inform the client about the character encoding of the document so that
284    the document can be interpreted and displayed appropriately, and for <a href="../content-negotiation.html">content negotiation</a>,
285    where the server returns one from several documents based on
286    the client's charset preference.</p>
287
288    <p>The <var>extension</var> argument is case-insensitive and can
289    be specified with or without a leading dot. Filenames may have <a href="#multipleext">multiple extensions</a> and the
290    <var>extension</var> argument will be compared against each of
291    them.</p>

301<div class="directive-section"><h2><a name="AddEncoding" id="AddEncoding">AddEncoding</a> <a name="addencoding" id="addencoding">Directive</a></h2>
302<table class="directive">
303<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Maps the given filename extensions to the specified encoding
304type</td></tr>
305<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>AddEncoding <var>encoding</var> <var>extension</var>
306[<var>extension</var>] ...</code></td></tr>
307<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
308<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>FileInfo</td></tr>
309<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>

312    <p>The <code class="directive">AddEncoding</code> directive maps the given
313    filename extensions to the specified HTTP content-encoding.
314    <var>encoding</var> is the HTTP content coding to append to the
315    value of the Content-Encoding header field for documents named with the
316    <var>extension</var>. This mapping is added to any already in force,
317    overriding any mappings that already exist for the same
318    <var>extension</var>.</p>
319
320    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">AddEncoding x-gzip .gz
321AddEncoding x-compress .Z</pre>
322</div>
323
324    <p>This will cause filenames containing the <code>.gz</code> extension
325    to be marked as encoded using the <code>x-gzip</code> encoding, and
326    filenames containing the <code>.Z</code> extension to be marked as
327    encoded with <code>x-compress</code>.</p>
328
329    <p>Old clients expect <code>x-gzip</code> and <code>x-compress</code>,
330    however the standard dictates that they're equivalent to
331    <code>gzip</code> and <code>compress</code> respectively. Apache does
332    content encoding comparisons by ignoring any leading <code>x-</code>.
333    When responding with an encoding Apache will use whatever form
334    (<em>i.e.</em>, <code>x-foo</code> or <code>foo</code>) the
335    client requested. If the client didn't specifically request a
336    particular form Apache will use the form given by the
337    <code>AddEncoding</code> directive. To make this long story
338    short, you should always use <code>x-gzip</code> and
339    <code>x-compress</code> for these two specific encodings. More
340    recent encodings, such as <code>deflate</code>, should be
341    specified without the <code>x-</code>.</p>
342
343    <p>The <var>extension</var> argument is case-insensitive and can
344    be specified with or without a leading dot. Filenames may have <a href="#multipleext">multiple extensions</a> and the
345    <var>extension</var> argument will be compared against each of
346    them.</p>

419<table class="directive">
420<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Maps the given filename extension to the specified content
421language</td></tr>
422<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>AddLanguage <var>language-tag</var> <var>extension</var>
423[<var>extension</var>] ...</code></td></tr>
424<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>

429    <p>The <code class="directive">AddLanguage</code> directive maps the given
430    filename extension to the specified content language.  Files with the
431    filename <var>extension</var> are assigned an HTTP Content-Language
432    value of <var>language-tag</var> corresponding to the language
433    identifiers defined by RFC 3066.
434    This directive overrides any mappings that already exist for the same
435    <var>extension</var>.</p>
436
437    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">AddEncoding x-compress .Z
438AddLanguage en .en
439AddLanguage fr .fr</pre>
440</div>
441
442    <p>Then the document <code>xxxx.en.Z</code> will be treated as
443    being a compressed English document (as will the document
444    <code>xxxx.Z.en</code>). Although the content language is
445    reported to the client, the browser is unlikely to use this
446    information. The <code class="directive">AddLanguage</code> directive is
447    more useful for <a href="../content-negotiation.html">content
448    negotiation</a>, where the server returns one from several documents
449    based on the client's language preference.</p>
450
451    <p>If multiple language assignments are made for the same
452    extension, the last one encountered is the one that is used.
453    That is, for the case of:</p>
454
455    <pre class="prettyprint lang-config">AddLanguage en .en
456AddLanguage en-gb .en
457AddLanguage en-us .en</pre>
458
459
460    <p>documents with the extension <code>.en</code> would be treated as
461    being <code>en-us</code>.</p>
462
463    <p>The <var>extension</var> argument is case-insensitive and can
464    be specified with or without a leading dot. Filenames may have <a href="#multipleext">multiple extensions</a> and the
465    <var>extension</var> argument will be compared against each of
466    them.</p>

699<div class="directive-section"><h2><a name="MultiviewsMatch" id="MultiviewsMatch">MultiviewsMatch</a> <a name="multiviewsmatch" id="multiviewsmatch">Directive</a></h2>
700<table class="directive">
701<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>The types of files that will be included when searching for
702a matching file with MultiViews</td></tr>
703<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>MultiviewsMatch Any|NegotiatedOnly|Filters|Handlers
704[Handlers|Filters]</code></td></tr>
705<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>MultiviewsMatch NegotiatedOnly</code></td></tr>
706<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
707<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>FileInfo</td></tr>
708<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
709<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_mime</td></tr>
710</table>
711    <p><code class="directive">MultiviewsMatch</code> permits three different
712    behaviors for <a href="mod_negotiation.html">mod_negotiation</a>'s
713    Multiviews feature.  Multiviews allows a request for a file,
714    <em>e.g.</em> <code>index.html</code>, to match any negotiated
715    extensions following the base request, <em>e.g.</em>
716    <code>index.html.en</code>, <code>index.html.fr</code>, or
717    <code>index.html.gz</code>.</p>
718
719    <p>The <code>NegotiatedOnly</code> option provides that every extension
720    following the base name must correlate to a recognized
721    <code class="module"><a href="../mod/mod_mime.html">mod_mime</a></code> extension for content negotiation, <em>e.g.</em>
722    Charset, Content-Type, Language, or Encoding.  This is the strictest
723    implementation with the fewest unexpected side effects, and is the
724    default behavior.</p>
725
726    <p>To include extensions associated with Handlers and/or Filters,
727    set the <code class="directive">MultiviewsMatch</code> directive to either
728    <code>Handlers</code>, <code>Filters</code>, or both option keywords.
729    If all other factors are equal, the smallest file will be served,
730    <em>e.g.</em> in deciding between <code>index.html.cgi</code> of 500
731    bytes and <code>index.html.pl</code> of 1000 bytes, the <code>.cgi</code>
732    file would win in this example. Users of <code>.asis</code> files
733    might prefer to use the Handler option, if <code>.asis</code> files are
734    associated with the <code>asis-handler</code>.</p>
735
736    <p>You may finally allow <code>Any</code> extensions to match, even if
737    <code class="module"><a href="../mod/mod_mime.html">mod_mime</a></code> doesn't recognize the extension. This can cause
738    unpredictable results, such as serving .old or .bak files the webmaster
739    never expected to be served.</p>
740
741    <p>For example, the following configuration will allow handlers
742    and filters to participate in Multviews, but will exclude unknown
743    files:</p>
744
745    <pre class="prettyprint lang-config">MultiviewsMatch Handlers Filters</pre>
746
747
748    <p><code class="directive">MultiviewsMatch</code> is not allowed in a

974<div class="directive-section"><h2><a name="TypesConfig" id="TypesConfig">TypesConfig</a> <a name="typesconfig" id="typesconfig">Directive</a></h2>
975<table class="directive">
976<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>The location of the <code>mime.types</code> file</td></tr>
977<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>TypesConfig <var>file-path</var></code></td></tr>
978<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>TypesConfig conf/mime.types</code></td></tr>
979<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config</td></tr>
980<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
981<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_mime</td></tr>
982</table>
983    <p>The <code class="directive">TypesConfig</code> directive sets the
984    location of the <a class="glossarylink" href="../glossary.html#media-type" title="see glossary">media types</a>
985    configuration file. <var>File-path</var> is relative to the
986    <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>. This file sets
987    the default list of mappings from filename extensions to content
988    types. Most administrators use the <code>mime.types</code> file
989    provided by their OS, which associates common filename
990    extensions with the official list of IANA registered media types
991    maintained at <a href="http://www.iana.org/assignments/media-types/index.html">http://www.iana.org/assignments/media-types/index.html</a>
992    as well as a large number of unofficial types.  This
993    simplifies the <code>httpd.conf</code> file by providing the
994    majority of media-type definitions, and may be overridden by
995    <code class="directive"><a href="#addtype">AddType</a></code> directives as
996    needed. You should not edit the <code>mime.types</code> file,
997    because it may be replaced when you upgrade your server.</p>
998
999    <p>The file contains lines in the format of the arguments to
1000    an <code class="directive"><a href="#addtype">AddType</a></code> directive:</p>
1001
1002    <div class="example"><p><code>
1003      <var>media-type</var> [<var>extension</var>] ...
1004    </code></p></div>
1005
1006    <p>The case of the extension does not matter. Blank lines, and lines
1007    beginning with a hash character (<code>#</code>) are ignored.
1008    Empty lines are there for completeness (of the mime.types file).
1009    Apache httpd can still determine these types with <code class="module"><a href="../mod/mod_mime_magic.html">mod_mime_magic</a></code>.
1010    </p>
1011
1012    <div class="note">
1013      Please do <strong>not</strong> send requests to the Apache HTTP
1014      Server Project to add any new entries in the distributed
1015      <code>mime.types</code> file unless (1) they are already
1016      registered with IANA, and (2) they use widely accepted,
1017      non-conflicting filename extensions across platforms.
1018      <code>category/x-subtype</code> requests will be automatically
1019      rejected, as will any new two-letter extensions as they will
1020      likely conflict later with the already crowded language and
1021      character set namespace.
1022    </div>
