41<h3>Sommaire</h3>
42
43    <p>Ce module fournit un support avancé de l'internationalisation
44    pour les modules de filtrage supportant les balises (markup-aware)
45    comme <code class="module"><a href="../mod/mod_proxy_html.html">mod_proxy_html</a></code>. Il est capable de détecter
46    automatiquement l'encodage des données en entrée et de s'assurer
47    qu'elle sont traitées correctement par l'interpréteur <a href="http://xmlsoft.org/">libxml2</a>, y compris la conversion en
48    Unicode (UTF-8) si nécessaire. Il peut aussi convertir les données
49    dans l'encodage de votre choix après le traitement des balises, et
50    s'assurera que le <var>jeu de caractères</var> approprié sera défini
51    dans l'en-tête HTTP <var>Content-Type</var>.</p>

71<h2><a name="usage" id="usage">Utilisation</a></h2>
72    <p>Il existe deux scénarios d'utilisation : le cas des modules
73    programmés pour travailler avec mod_xml2enc ; et les autres :</p>
74    <dl>
75    <dt>Modules de filtrages programmés pour mod_xml2enc</dt><dd>
76    <p>Les modules comme <code class="module"><a href="../mod/mod_proxy_html.html">mod_proxy_html</a></code> versions 3.1 et
77    supérieures utilisent la fonction optionnelle
78    <code>xml2enc_charset</code> pour déterminer la valeur de l'argument
79    "jeu de caractères" à transmettre à l'interpréteur libxml2, et
80    disposent de la fonction optionnelle <code>xml2enc_filter</code>
81    pour effectuer un encodage ultérieur éventuel. L'utilisation de
82    mod_xml2enc avec un module préprogrammé à cet effet ne nécessite
83    aucune configuration : ce dernier configurera mod_xml2enc pour vous
84    (sachant que vous pouvez tout de même le personnaliser via les
85    directives de configuration ci-dessous).</p>
86    </dd>
87    <dt>Modules de filtrages non programmés pour mod_xml2enc</dt><dd>
88    <p>Pour utiliser mod_xml2enc avec un module basé sur libxml2 qui n'a
89    pas été explicitement programmé pour mod_xml2enc, vous devrez
90    configurer la chaîne de filtrage vous-même. Ainsi, pour utiliser
91    mod_xml2enc avec un filtre <strong>foo</strong> fourni par un module
92    <strong>mod_foo</strong> et pour
93    améliorer le support i18n de ce dernier avec HTML et XML, vous
94    pouvez utiliser les directives suivantes :</p>
95    <pre><code>
96    FilterProvider iconv    xml2enc Content-Type $text/html
97    FilterProvider iconv    xml2enc Content-Type $xml
98    FilterProvider markup   foo Content-Type $text/html
99    FilterProvider markup   foo Content-Type $xml
100    FilterChain     iconv markup
101    </code></pre>
102    <p><strong>mod_foo</strong> supportera alors tout jeu de caractère supporté soit par
103    libxml2, soit par apr_xlate/iconv, soit par les deux.</p>
104    </dd></dl>

107<h2><a name="api" id="api">API de programmation</a></h2>
108    <p>Les programmeurs de modules de filtrage basés sur libxml2 sont
109    encouragés à les préprogrammer pour mod_xml2enc, afin de fournir un
110    support i18n solide aux utilisateurs sans avoir à réinventer la
111    roue. L'API de programmation est décrite dans
112    <var>mod_xml2enc.h</var>, et <code class="module"><a href="../mod/mod_proxy_html.html">mod_proxy_html</a></code> est un
113    exemple de son utilisation.</p>
114</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

116<h2><a name="sniffing" id="sniffing">Détection et encodage</a></h2>
117    <p>A la différence de <code class="module"><a href="../mod/mod_charset_lite.html">mod_charset_lite</a></code>, mod_xml2enc
118    est conçu pour travailler avec des données dont l'encodage ne peut
119    pas être connu, et donc configuré, à l'avance. Il utilise donc les
120    techniques de 'reniflage' suivantes pour détecter le type d'encodage
121    des données HTTP :</p>
122    <ol>
123        <li>Si l'en-tête HTTP <var>Content-Type</var> contient un
124	paramètre <var>charset</var>, c'est ce dernier qui sera utilisé.</li>
125        <li>Si les données commancent par une balise XML concernant
126	l'ordre des octets (BOM) ou par une déclaration d'encodage XML,
127	c'est celle-ci qui sera utilisée.</li>
128        <li>Si un type d'encodage est déclaré dans un élément HTML
129	<code>&lt;META&gt;</code>, c'est ce dernier qui sera utilisé.</li>
130        <li>Si aucun des éléments précédents n'est trouvé, c'est la
131	valeur par défaut définie par la directive
132	<code class="directive">xml2EncDefault</code> qui sera utilisée.</li>
133    </ol>
134    <p>Les conditions sont testées dans cet ordre . Dès qu'une règle
135    s'applique, elle est utilisée et la détection est terminée.</p>

138<h2><a name="output" id="output">Codage en sortie</a></h2>
139<p><a href="http://xmlsoft.org/">libxml2</a> utilise toujours UTF-8
140(Unicode) en interne, et les modules de filtrage basés sur libxml2
141utiliseront cet encodage en sortie par défaut. mod_xml2enc peut modifier
142l'encodage en sortie via l'API, mais il n'y a actuellement aucun moyen de le
143configurer directement.</p>
144<p>La modification de l'encodage en sortie ne devrait (du moins en théorie)
145jamais être nécessaire, et est même déconseillée à cause de la charge de
146traitement supplémentaire imposée au serveur par une conversion non
147nécessaire.</p>
148</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

150<h2><a name="alias" id="alias">Codages non supportés</a></h2>
151<p>Si vous travaillez avec des encodages non supportés par aucune des
152méthodes de conversion disponibles sur votre plateforme, vous pouvez
153tout de même leur associer un alias vers un code supporté via la
154directive <code class="directive">xml2EncAlias</code>.</p>
155</div>

157<div class="directive-section"><h2><a name="xml2encalias" id="xml2encalias">Directive</a> <a name="xml2EncAlias" id="xml2EncAlias">xml2EncAlias</a></h2>
158<table class="directive">
159<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Définit des alias pour les valeurs d'encodage</td></tr>
160<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>xml2EncAlias <var>jeu-de-caractères alias [alias ...]</var></code></td></tr>
161<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale</td></tr>
162<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
163<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_xml2enc</td></tr>
164</table>
165    <p>Cette directive de niveau serveur permet de définir un ou
166    plusieurs alias pour un encodage. Elle permet au support d'encodage de
167    libxml2 de traiter en interne des encodages non reconnus par libxml2
168    en utilisant la table de conversion pour un encodage reconnu. Elle
169    permet d'atteindre deux objectifs : supporter des jeux (ou noms) de
170    caractères non reconnus par libxml2 ou iconv, et éviter une
171    conversion pour un encodage lorsque cela n'est pas nécessaire.</p>

175<div class="directive-section"><h2><a name="xml2encdefault" id="xml2encdefault">Directive</a> <a name="xml2EncDefault" id="xml2EncDefault">xml2EncDefault</a></h2>
176<table class="directive">
177<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Définit un encodage par défaut à utiliser lorsqu'aucune
178information ne peut être <a href="#sniffing">automatiquement détectée</a></td></tr>
179<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>xml2EncDefault <var>nom</var></code></td></tr>
180<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale, serveur virtuel, répertoire, .htaccess</td></tr>
181<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
182<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_xml2enc</td></tr>
183</table>
184    <p>Si vous traitez des données dont l'encodage est connu, mais ne
185    contenant aucune information à propos de ce dernier, vous pouvez
186    définir une valeur par défaut afin d'aider mod_xml2enc à traiter
187    correctement les données. Par exemple, pour définir la valeur par
188    défaut Latin1 (<var>iso-8859-1</var>) specifiée dans HTTP/1.0,
189    utilisez :</p>
190    <pre class="prettyprint lang-config">xml2EncDefault iso-8859-1</pre>

195<div class="directive-section"><h2><a name="xml2startparse" id="xml2startparse">Directive</a> <a name="xml2StartParse" id="xml2StartParse">xml2StartParse</a></h2>
196<table class="directive">
197<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Indique à l'interpréteur à partir de quelle balise il doit
198commencer son traitement.</td></tr>
199<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>xml2StartParse <var>élément [élément ...]</var></code></td></tr>
200<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale, serveur virtuel, répertoire, .htaccess</td></tr>
201<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
202<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_xml2enc</td></tr>
203</table>
204    <p>Cette directive permet de spécifier à partir de quelle balise,
205    parmi les éléments spécifiés, l'interpréteur de balise doit
206    commencer son traitement. Ccei permet de contourner le problème des
207    serveurs d'arrière-plan qui insèrent des éléments non conformes en
208    début de données, ce qui a pour effet de perturber l'interpréteur (<a href="http://bahumbug.wordpress.com/2006/10/12/mod_proxy_html-revisited/">voir un exemple ici</a>).</p>
209    <p>Elle ne doit être utilisée ni pour les documents XML, ni pour les
210    documents HTML correctement formatés.</p>
