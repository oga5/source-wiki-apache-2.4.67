46<h3>概要</h3>
47
48
49    <p>いくつかのオペレーティングシステムでは、サーバの再コンパイルをする代わりに、
50    このモジュールを使用して
51    <a href="../dso.html">動的共有オブジェクト</a>
52    (DSO) 機構により、実行時に Apache HTTP Server にモジュールを読み込ませることが
53    できます。</p>
54
55    <p>Unix 上では、読み込まれるコードは通常は共有オブジェクトファイル
56    (普通 <code>.so</code> という拡張子が付いています) からです。
57    Windows 上ではこのモジュールの拡張子は <code>.so</code> か <code>.dll</code>
58    です。</p>
59
60    <div class="warning"><h3>警告</h3>
61    <p>Apache HTTP Server のあるメジャーバージョン向けにビルドされたモジュールは一般に
62    他のメジャーバージョンでは動きません。(例えば 1.3 と 2.0、 2.0 と 2.2)
63    またメジャーバージョン間ではAPIの変更がしばしば発生し、そのため新しい
64    メジャーバージョン向けにモジュールの修正が必要になることがあります。</p>
65    </div>

80<h2>Windows 用のロード可能なモジュールを作成する</h2>
81
82    <div class="note"><h3>注</h3>
83    <p>Windows において動的にロードされるモジュールの拡張子は普通 <code>.dll</code>
84    ですが、Apache httpd のモジュールは <code>mod_whatever.so</code>
85    といった名前を持ちます。これは、他のプラットフォームでの通常の形式に
86    あわせたものです。しかしながら、サードパーティ製モジュール、例えばPHPなど、
87    は今でも <code>.dll</code> の拡張子を使っています。</p>
88
89    <p>まだ <code>mod_so</code> で <code>ApacheModuleFoo.dll</code> という名前の
90    モジュールもロードされますが、新しい名前の付け方を使う方が好まれます。
91    モジュールを 2.0 用に移植しているのであれば、2.0 の習慣に合うように名前を
92    修正してください。</p></div>
93
94    <p>Apache httpd のモジュール API は UNIX と Windows 間では変更されていません。
95    多くのモジュールは全く変更なし、もしくは簡単な変更により Windows 
96    で実行できるようになります。ただし、それ以外の Windows には無い Unix
97    アーキテクチャーの機能に依存したモジュールは動作しません。</p>
98
99    <p>モジュールが実際に動作するときは、
100    二つの方法のどちらかでサーバに追加することができます。まず、Unix
101    と同様にサーバにコンパイルして組み込むことができます。Windows
102    用の Apache httpd は Unix 用の Apache にある <code>Configure</code>
103    プログラムがありませんので、モジュールのソースファイルを
104    ApacheCore プロジェクトファイルに追加し、シンボルを
105    <code>os\win32\modules.c</code> ファイルに追加する必要があります。</p>
106
107    <p>二つ目はモジュールを DLL としてコンパイルする方法です。
108    DLL は共有ライブラリで、実行時に
109    <code><code class="directive">LoadModule</code></code>
110    ディレクティブによりサーバに読み込むことができます。これらのモジュール
111    DLL はそのまま配布することが可能で、サーバを再コンパイルすることなく、Windows
112    用の Apache httpd のすべてのインストールで実行することができます。</p>
113
114    <p>モジュール DLL を作成するためには、
115    モジュールの作成に小さな変更を行なう必要があります。
116    つまり、モジュールのレコード (これは後で作成されます。
117    以下を参照してください) が DLL からエクスポートされなければなりません。
118    これを行なうには、<code>AP_MODULE_DECLARE_DATA</code> (Apache httpd
119    のヘッダファイルで定義されています) をモジュールのモジュールレコード
120    定義の部分に追加してください。たとえば、モジュールに</p>
121<div class="example"><p><code>
122    module foo_module;
123</code></p></div>
124
125    <p>があるとすると、それを次のもので置き換えてください。</p>
126<div class="example"><p><code>
127    module AP_MODULE_DECLARE_DATA foo_module;
128</code></p></div>
129
130    <p>Unix 上でもこのモジュールを
131    変更無しで使い続けられるように、このマクロは Windows
132    上でのみ効力を持ちます。<code>.DEF</code>
133    ファイルの方を良く知っているという場合は、
134    代わりにそれを使ってモジュールレコードを
135    エクスポートすることもできます。</p>
136    <p>さあ、あなたのモジュールの DLL を作成しましょう。これを、
137    libhttpd.lib 共有ライブラリがコンパイルされたときに作成された
138    ibhttpd.lib エクスポートライブラリとリンクしてください。この時に、
139    Apache httpd のヘッダファイルが正しい位置にあるように、
140    コンパイラの設定を変える必要があるかもしれません。
141    このライブラリはサーバルートの modules ディレクトリにあります。
142    ビルド環境が正しく設定されるように、既存のモジュール用の .dsp を
143    取ってくるのが一番良いでしょう。もしくは、あなたの .dsp と
144    コンパイラとリンクのオプションを比較する、というものでも良いです。</p>
145
146    <p>これで DLL 版のモジュールが作成されているはずです。
147    サーバルートの <code>modules</code>
148    ディレクトリにモジュールを置いて、
149    <code><code class="directive">LoadModule</code></code>
150    ディレクティブを使って読み込んでください。</p>

153<div class="directive-section"><h2><a name="LoadFile" id="LoadFile">LoadFile</a> <a name="loadfile" id="loadfile">ディレクティブ</a></h2>
154<table class="directive">
155<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>指定されたオブジェクトファイルやライブラリをリンクする</td></tr>
156<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>LoadFile <em>filename</em> [<em>filename</em>] ...</code></td></tr>
157<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>
158<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
159<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_so</td></tr>

161
162    <p>LoadFile ディレクティブは、サーバが起動されたときや再起動されたときに、
163    指定されたオブジェクトファイルやライブラリをリンクします。
164    これはモジュールが動作するために必要になるかもしれない追加の
165    コードを読み込むために使用されます。<em>Filename</em> は絶対パスか、<a href="core.html#serverroot">ServerRoot</a> からの相対パスです。</p>
166
167    <p>例:</p>
168
169    <pre class="prettyprint lang-config">LoadFile libexec/libxmlparse.so</pre>
170

175<div class="directive-section"><h2><a name="LoadModule" id="LoadModule">LoadModule</a> <a name="loadmodule" id="loadmodule">ディレクティブ</a></h2>
176<table class="directive">
177<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>オブジェクトファイルやライブラリをリンクし、使用モジュールの
178リストに追加する</td></tr>
179<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>LoadModule <em>module filename</em></code></td></tr>
180<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>
181<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
182<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_so</td></tr>
183</table>
184
185    <p>LoadModule ディレクティブは <em>filename</em>
186    というオブジェクトファイルおよびライブラリをリンクし、<em>module</em>
187    という名前のモジュールの構造をアクティブなモジュールのリストに追加します。
188    <em>Module</em> はファイル中の <code>module</code>
189    型の外部変数の名前で、モジュールのドキュメントに
190    <a href="module-dict.html#moduleidentifier">モジュール識別子</a>として書かれているものです。例 :</p>
191
192   <pre class="prettyprint lang-config">LoadModule status_module modules/mod_status.so</pre>
193
194
195    <p>これは ServerRoot の modules サブディレクトリから指定された名前の
196    モジュールをロードします。</p>
