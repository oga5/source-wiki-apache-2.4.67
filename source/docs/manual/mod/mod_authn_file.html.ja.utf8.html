43<h3>概要</h3>
44
45    <p>本モジュールは <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code> や
46    <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> といった認証フロントエンドに対して、
47    プレインテキストのパスワードファイル内からユーザを検索することで、
48    ユーザ認証機能を提供します。似たような機能は <code class="module"><a href="../mod/mod_authn_dbm.html">mod_authn_dbm</a></code>
49    でも提供されています。</p>
50
51    <p><code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> や <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>
52    を使用する際には、
53    <code class="directive"><a href="../mod/mod_auth_basic.html#authbasicprovider">AuthBasicProvider</a></code> や
54    <code class="directive"><a href="../mod/mod_auth_digest.html#authdigestprivider">AuthDigestPrivider</a></code>
55    で <code>file</code> と指定することでこのモジュールは起動されます。</p>

74<div class="directive-section"><h2><a name="AuthUserFile" id="AuthUserFile">AuthUserFile</a> <a name="authuserfile" id="authuserfile">ディレクティブ</a></h2>
75<table class="directive">
76<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>認証に使用するユーザとパスワードの一覧が格納されている、
77テキストファイルの名前を設定する</td></tr>
78<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>AuthUserFile <var>file-path</var></code></td></tr>
79<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>ディレクトリ, .htaccess</td></tr>
80<tr><th><a href="directive-dict.html#Override">上書き:</a></th><td>AuthConfig</td></tr>
81<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Base</td></tr>
82<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_authn_file</td></tr>
83</table>
84    <p><code class="directive">AuthUserFile</code> ディレクティブは、
85    ユーザ認証のためのユーザとパスワードの一覧を格納した
86    テキストファイルの名前を設定します。<var>file-path</var>
87    はユーザファイルへのパスです。
88    もし絶対パスでなければ、
89    <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>
90    からの相対パスとして扱われます。</p>
91
92    <p>ユーザファイルの各行には、ユーザ名、コロン、
93    暗号化したパスワードを記述します。
94    同一ユーザ ID が複数回登録された時は、
95    <code class="module"><a href="../mod/mod_authn_file.html">mod_authn_file</a></code>
96    は最初に見つかったパスワードを使用して認証します。</p>
97
98    <p>バイナリ配布の一部としてインストールされるか、
99    あるいは <code>src/support</code> にある
100    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code>
101    ユーティリティで、この <em>HTTP 基本認証</em>
102    用パスワードファイルをメインテナンスします。
103    詳細は <a href="../programs/htpasswd.html">man
104    ページ</a>をご覧頂くとして、簡単には:</p>
105
106    <p>初期 ID <code>username</code> で、<code>Filename</code>
107    というパスワードファイルを生成します。
108    次のコマンドを発行するとパスワードが要求されます:</p>
109
110    <div class="example"><p><code>
111      htpasswd -c Filename username
112    </code></p></div>
113
114    <p>パスワードファイル <code>Filename</code> に、<code>username2</code>
115    を追加したり修正したりします:</p>
116
117    <div class="example"><p><code>
118      htpasswd Filename username2
119    </code></p></div>
120
121    <p>(訳注: 非常に多くのユーザを登録すると大きなファイルになりますが)
122    大きなテキストファイルを検索するのは<em>非常に効率が悪い</em>
123    ということに注意してください。そのような必要のある時は、
124    <code class="directive"><a href="../mod/mod_authn_dbm.html#authdbmuserfile">AuthDBMUserFile</a></code>
125    を代わりに使ってください。</p>
126
127    <p><em>HTTP ダイジェスト認証</em>を使用する場合は、
128    <a href="../programs/htpasswd.html">htpasswd</a>
129    プログラムでは不十分です。その代わりに
130    <code class="program"><a href="../programs/htdigest.html">htdigest</a></code>
131    を使用してください。ダイジェスト認証用のデータと
132    基本認証用のデータを同一ファイルに混ぜて保存できない、
133    ということに注意してください。</p>
134
135    <div class="warning"><h3>セキュリティ</h3>
136    <p><code class="directive">AuthUserFile </code>
137        は、ウェブサーバのドキュメントツリーの外側に保管するようにしてください。
138        保護しようとしているディレクトリ以下には、<strong>置かないで下さい</strong>。
139        そうしないと <code class="directive">AuthUserFile</code> は
140        ダウンロードできてしまいます。</p>
141    </div>
