37<h3>Sommaire</h3>
38
39    <p>Ce module fournit le coeur des fonctionnalités d'authentification
40    permettant d'accorder ou de refuser l'accès à certaines zones du
41    site web. Les directives fournies par le module
42    <code class="module"><a href="../mod/mod_authn_core.html">mod_authn_core</a></code> sont communes à tous les
43    fournisseurs d'authentification.</p>

60<h2><a name="authnalias" id="authnalias">Création d'alias de fournisseurs
61d'authentification</a></h2>
62
63    <p>Il est possible de créer des fournisseurs d'authentification
64    étendus dans le fichier de configuration et de leur assigner un
65    alias. Le fournisseur ainsi nommé peut alors être référencé à l'aide
66    des directives <code class="directive"><a href="../mod/mod_auth_basic.html#authbasicprovider">AuthBasicProvider</a></code> ou <code class="directive"><a href="../mod/mod_auth_digest.html#authdigestprovider">AuthDigestProvider</a></code> tout comme
67    un fournisseur d'authentification de base. Outre la possibilité de
68    créer et attribuer un alias à un fournisseur étendu, le même
69    fournisseur d'authentification peut aussi être référencé par
70    plusieurs sections relatives à une zone du site web.</p>
71
72    <h3><a name="example" id="example">Exemples</a></h3>
73
74	<p>Cet exemple vérifie les mots de passe dans deux fichiers
75	textes différents.</p>
76
77	<div class="example"><h3>Vérification dans plusieurs fichiers de mots de
78	passe au format texte</h3><pre class="prettyprint lang-config"># Première vérification
79&lt;AuthnProviderAlias file file1&gt;
80    AuthUserFile "/www/conf/passwords1"
81&lt;/AuthnProviderAlias&gt;
82
83# Vérification suivante
84&lt;AuthnProviderAlias file file2&gt;   
85    AuthUserFile "/www/conf/passwords2"
86&lt;/AuthnProviderAlias&gt;
87
88&lt;Directory "/var/web/pages/secure"&gt;
89    AuthBasicProvider file1 file2
90    
91    AuthType Basic
92    AuthName "Protected Area"
93    Require valid-user
94&lt;/Directory&gt;</pre>
95</div>
96
97
98
99        <p>Dans l'exemple ci-dessous, deux fournisseurs
100	d'authentification ldap sont créés à partir du fournisseur ldap
101	de base, et se voient attribuer un alias. L'authentification
102	d'une même zone peut alors être traitée par plusieurs serveurs
103	ldap :</p>
104
105        <div class="example"><h3>Vérification auprès de plusieurs serveurs
106	LDAP</h3><pre class="prettyprint lang-config">&lt;AuthnProviderAlias ldap ldap-alias1&gt;
107    AuthLDAPBindDN cn=youruser,o=ctx
108    AuthLDAPBindPassword yourpassword
109    AuthLDAPURL ldap://ldap.host/o=ctx
110    &lt;/AuthnProviderAlias&gt;
111    &lt;AuthnProviderAlias ldap ldap-other-alias&gt;
112    AuthLDAPBindDN cn=yourotheruser,o=dev
113    AuthLDAPBindPassword yourotherpassword
114    AuthLDAPURL ldap://other.ldap.host/o=dev?cn
115&lt;/AuthnProviderAlias&gt;
116
117Alias "/secure" "/webpages/secure"
118&lt;Directory "/webpages/secure"&gt;
119    
120    AuthBasicProvider ldap-other-alias  ldap-alias1
121    
122    AuthType Basic
123    AuthName LDAP_Protected Place
124    Require valid-user
125    # Notez que Require ldap-* ne fonctionnerait pas ici, car
126    # AuthnProviderAlias ne fournit pas de configuration pour les
127    # fournisseurs d'autorisation implémentés dans le même module que le
128    # fournisseur d'authentification.
129&lt;/Directory&gt;</pre>
130</div>

135<div class="directive-section"><h2><a name="authname" id="authname">Directive</a> <a name="AuthName" id="AuthName">AuthName</a></h2>
136<table class="directive">
137<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>L'identifiant de l'autorisation à utiliser avec
138l'authentification HTTP</td></tr>
139<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>AuthName <var>domaine d'autorisation</var></code></td></tr>
140<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>répertoire, .htaccess</td></tr>
141<tr><th><a href="directive-dict.html#Override">Surcharges autorisées:</a></th><td>AuthConfig</td></tr>
142<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
143<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authn_core</td></tr>
144</table>
145    <p>Cette directive permet de définir l'identifiant d'autorisation
146    pour un répertoire. Cet identifiant est fourni au client de façon à
147    ce qu'il sache quels nom d'utilisateur et mot de passe envoyer.
148    <code class="directive">AuthName</code> accepte un seul argument ; s'il
149    contient des espaces, il doit être entouré de guillemets. Pour
150    pouvoir fonctionner, la directive <code class="directive">AuthName</code>
151    doit être utilisée en combinaison avec les directives <code class="directive"><a href="#authtype">AuthType</a></code> et <code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code>, ainsi que des
152    directives comme <code class="directive"><a href="../mod/mod_authn_file.html#authuserfile">AuthUserFile</a></code> et <code class="directive"><a href="../mod/mod_authz_groupfile.html#authgroupfile">AuthGroupFile</a></code>.</p>
153
154   <p>Par exemple :</p>
155
156   <pre class="prettyprint lang-config">AuthName "Top Secret"</pre>
157
158
159    <p>La chaîne fournie comme argument à <code>AuthName</code>
160    apparaîtra dans la boîte de dialogue d'authentification pour la
161    plupart des navigateurs.</p>
162
163     <p>A partir de la version 2.4.55 du serveur HTTP Apache, il est possible de
164    définir cette directive en utilisant la <a href="../expr.html">syntaxe des
165    expressions</a> pour spécifier l'identifiant d'autorisation de manière
166    dynamique.</p>
167
168   <p>Exemple :</p>
169
170   <pre class="prettyprint lang-config">AuthName "%{HTTP_HOST}"</pre>

182<div class="directive-section"><h2><a name="authnprovideralias" id="authnprovideralias">Directive</a> <a name="AuthnProviderAlias" id="AuthnProviderAlias">&lt;AuthnProviderAlias&gt;</a></h2>
183<table class="directive">
184<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Regroupe un ensemble de directives qui constituent une
185extension d'un fournisseur d'authentification de base et lui attribue
186l'alias spécifié</td></tr>
187<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>&lt;AuthnProviderAlias <var>alias-fournisseur</var>&gt;
188... &lt;/AuthnProviderAlias&gt;</code></td></tr>
189<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale</td></tr>
190<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
191<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authn_core</td></tr>
192</table>
193    <p>Les balises <code>&lt;AuthnProviderAlias&gt;</code> et
194    <code>&lt;/AuthnProviderAlias&gt;</code> permettent de regrouper un
195    ensemble de directives d'authentification qui seront référencées par
196    l'alias spécifié à l'aide des directives <code class="directive"><a href="../mod/mod_auth_basic.html#authbasicprovider">AuthBasicProvider</a></code> ou <code class="directive"><a href="../mod/mod_auth_digest.html#authdigestprovider">AuthDigestProvider</a></code>.</p>
197
198    <div class="note">Cette directive n'a aucune influence sur le processus
199    d'autorisation, même pour les modules qui fournissent à la fois
200    l'authentification et l'autorisation.</div>

205<div class="directive-section"><h2><a name="authtype" id="authtype">Directive</a> <a name="AuthType" id="AuthType">AuthType</a></h2>
206<table class="directive">
207<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Type d'authentification utilisateur</td></tr>
208<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>AuthType None|Basic|Digest|Form</code></td></tr>
209<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>répertoire, .htaccess</td></tr>
210<tr><th><a href="directive-dict.html#Override">Surcharges autorisées:</a></th><td>AuthConfig</td></tr>
211<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
212<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authn_core</td></tr>
213</table>
214    <p>Cette directive permet de définir le type d'authentification
215    utilisateur pour un répertoire. Les types d'authentification
216    disponibles sont <code>None</code>, <code>Basic</code> (implémenté
217    par <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code>), <code>Digest</code>
218    (implémenté par <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>), et
219    <code>Form</code> (implémenté par
220    <code class="module"><a href="../mod/mod_auth_form.html">mod_auth_form</a></code>).</p>
221
222    <p>Pour mettre en oeuvre l'authentification, vous devez aussi
223    utiliser les directives <code class="directive"><a href="#authname">AuthName</a></code> et <code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code>. De plus, le serveur
224    doit pouvoir disposer d'un module fournisseur d'authentification
225    comme <code class="module"><a href="../mod/mod_authn_file.html">mod_authn_file</a></code> et d'un module d'autorisation
226    comme <code class="module"><a href="../mod/mod_authz_user.html">mod_authz_user</a></code>.</p>
227
228    <p>Le type d'authentification <code>None</code> désactive
229    l'authentification. Lorsqu'une authentification est définie, elle
230    est en général héritée par chacune des <a href="../sections.html#mergin">sections de configuration</a> qui
231    suivent, à moins qu'un autre type d'authentification ne soit
232    spécifié. Si l'on ne souhaite pas mettre en oeuvre
233    d'authentification pour une sous-section d'une section authentifiée,
234    on doit utiliser le type d'authentification <code>None</code> ; dans
235    l'exemple suivant, les clients peuvent accéder au répertoire
236    <code>/www/docs/public</code> sans devoir s'authentifier :</p>
237
238    <pre class="prettyprint lang-config">&lt;Directory "/www/docs"&gt;
239    AuthType Basic
240    AuthName Documents
241    AuthBasicProvider file
242    AuthUserFile "/usr/local/apache/passwd/passwords"
243    Require valid-user
244&lt;/Directory&gt;
245
246&lt;Directory "/www/docs/public"&gt;
247    AuthType None
248    Require all granted
249&lt;/Directory&gt;</pre>
250
251
252    <p>A partir de la version 2.4.55, il est possible de définir cette
253    directive en utilisant la <a href="../expr.html">syntaxe des expressions</a> pour
254    spécifier le type d'authentification de manière dynamique.</p>
255
256    <div class="note">Veuillez noter que, lorsque l'authentification n'est pas
257    activée, les clients qui se sont déjà authentifiés pour une autre
258    zone de l'arborescence du site continueront en général à envoyer des
259    en-tête d'authentification HTTP ou des cookies avec chaque requête,
260    sans se préoccuper de savoir si le serveur nécessite vraiment une
261    authentification pour chaque ressource.</div>
