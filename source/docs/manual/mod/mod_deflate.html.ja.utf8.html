42<h3>概要</h3>
43
44    <p><code class="module"><a href="../mod/mod_deflate.html">mod_deflate</a></code> モジュールは <code>DEFLATE</code>
45    出力フィルタを提供します。これはサーバからの出力を、ネットワークを
46    通してクライアントに送る前に圧縮することを可能にします。</p>

71<h2><a name="recommended" id="recommended">サンプル設定</a></h2>
72    <p>下にせっかちな人向けの簡単な設定例を示します。</p>
73
74    <div class="example"><h3>数タイプのみ圧縮する</h3><p><code>
75      AddOutputFilterByType DEFLATE text/html text/plain text/xml
76    </code></p></div>
77
78    <p>以下の設定はコンテンツをより圧縮しますが、ずっと複雑な設定になります。
79    設定の隅々までよく理解しないで使わないでください。</p>
80
81    <div class="example"><h3>画像以外全て圧縮する</h3><p><code>
82      &lt;Location /&gt;<br />
83      <span class="indent">
84        # Insert filter<br />
85        SetOutputFilter DEFLATE<br />
86        <br />
87        # Netscape 4.x has some problems...<br />
88        BrowserMatch ^Mozilla/4         gzip-only-text/html<br />
89        <br />
90        # Netscape 4.06-4.08 have some more problems<br />
91        BrowserMatch ^Mozilla/4\.0[678] no-gzip<br />
92        <br />
93        # MSIE masquerades as Netscape, but it is fine<br />
94        # BrowserMatch \bMSIE             !no-gzip !gzip-only-text/html<br />
95        <br />
96        # Don't compress images<br />
97        SetEnvIfNoCase Request_URI \<br />
98        <span class="indent">
99          \.(?:gif|jpe?g|png)$ no-gzip dont-vary<br />
100        </span>
101        <br />
102        # Make sure proxies don't deliver the wrong content<br />
103        Header append Vary User-Agent env=!dont-vary<br />
104      </span>
105      &lt;/Location&gt;
106    </code></p></div>

110<h2><a name="enable" id="enable">圧縮を有効にする</a></h2>
111
112    <h3><a name="output" id="output">Output Compression</a></h3>
113      <p>圧縮機能は <code>DEFLATE</code> <a href="../filter.html">フィルタ</a>
114      により実装されています。以下のディレクティブはそのディレクティブのある
115      コンテナ中のドキュメントを圧縮するようにします:</p>
116      
117      <div class="example"><p><code>
118        SetOutputFilter DEFLATE
119      </code></p></div>
120
121      <p>よく使われているブラウザでは、すべてのコンテンツに対する
122      圧縮を扱えるわけではありません。ですから、<code>gzip-only-text/html</code> 
123      ノートを <code>1</code> にして、html ファイルに対してのみ
124      圧縮が働くようにした方がよいかもしれません (以下参照)
125      この値を <em><code>1</code> 以外の値</em>に設定した場合は無視されます。</p>
126      
127      <p>通常、特定のMIMEタイプについてのみ圧縮したいのであれば、
128      <code class="directive"><a href="../mod/core.html#addoutputfilterbytype">AddOutputFilterByType</a></code>
129      ディレクティブを使用します。次に Apache のドキュメントの html
130      ファイルのみの圧縮を有効にする例を示します。</p>
131
132      <div class="example"><p><code>
133        &lt;Directory "/your-server-root/manual"&gt;<br />
134        <span class="indent">
135          AddOutputFilterByType DEFLATE text/html<br />
136        </span>
137        &lt;/Directory&gt;
138      </code></p></div>
139
140      <p>全てのファイルタイプでの圧縮に問題を抱えているブラウザに対しては、
141      <code class="directive"><a href="../mod/mod_setenvif.html#browsermatch">BrowserMatch</a></code>
142      ディレクティブを使用して、特定のブラウザに <code>no-gzip</code>
143      ノートをセットし、圧縮が行なわれないようにします。
144      <code>no-gzip</code> と <code>gzip-only-text/html</code>
145      を組み合わせることで上手く対処できます。
146      この場合、前者が後者をオーバーライドします。
147      上記の<a href="#recommended">設定例</a>の抜粋を
148      次に示しますのでご覧下さい。</p>
149
150      <div class="example"><p><code>
151        BrowserMatch ^Mozilla/4         gzip-only-text/html<br />
152        BrowserMatch ^Mozilla/4\.0[678] no-gzip<br />
153        BrowserMatch \bMSIE             !no-gzip !gzip-only-text/html
154      </code></p></div>
155
156      <p>まず始めに <code>User-Agent</code> 文字列から Netscape Navigator
157      4.x であるかどうかを調べます。これらのバージョンでは、
158      <code>text/html</code> 以外のタイプの圧縮を扱うことができません。
159      4.06, 4.07, 4.08 は html ファイルの伸張にも問題を抱えています。
160      ですからこれらに対しては、完全に deflate フィルタをオフにします。</p>
161
162      <p>3 番目の <code class="directive"><a href="../mod/mod_setenvif.html#browsermatch">BrowserMatch</a></code>
163      ディレクティブで、推測したユーザーエージェントを修正します。
164      なぜなら Microsoft Internet Explorer も "Mozilla/4" と特定されますが、
165      これらは実際には圧縮を扱うことができるからです。
166      <code>User-Agent</code> ヘッダを "MSIE"
167      (<code>\b</code> は「単語の境界」を意味します) の追加文字で検査して、
168      これ以前に設定した制限を再び解除します。</p>
169
170      <div class="note"><h3>注</h3>
171        <code>DEFLATE</code> フィルタは必ず、PHP や SSI といった RESOURCE
172        フィルタの後になります。
173        DEFLATE フィルタは内部的なサブリクエストを関知しません。
174      </div>
175      <div class="note"><h3>注</h3>
176        <code class="directive"><a href="../mod/core.html#setenv">SetEnv</a></code> で設定される
177        <code>force-gzip</code> 環境変数がありますが、これは
178        ブラウザの accept-encoding 設定を無視し、圧縮した出力をします。
179      </div>

182    <h3><a name="inflate" id="inflate">出力の伸長</a></h3>
183      <p><code class="module"><a href="../mod/mod_deflate.html">mod_deflate</a></code> モジュールは、gzip 圧縮されたレスポンス
184      本文を inflate/uncompress するフィルタも提供しています。
185      この機能を有効にするには、<code class="directive"><a href="../mod/core.html#setoutputfilter">SetOutputFilter</a></code>
186      や <code class="directive"><a href="../mod/mod_mime.html#addoutputfilter">AddOutputFilter</a></code> を使って、
187      <code>INFLATE</code> フィルタを出力フィルタチェインに挿入します。
188      例えば次のようにします。</p>
189
190      <div class="example"><p><code>
191        &lt;Location /dav-area&gt;<br />
192        <span class="indent">
193          ProxyPass http://example.com/<br />
194          SetOutputFilter INFLATE<br />
195        </span>
196        &lt;/Location&gt;
197      </code></p></div>
198
199      <p>この例では、example.com からの gzip 圧縮された出力を伸長し、
200      その他のフィルタがさらにその出力を処理できるようにします。
201      </p>
202      
203    
204    <h3><a name="input" id="input">入力の伸張</a></h3>
205      <p><code class="module"><a href="../mod/mod_deflate.html">mod_deflate</a></code> モジュールは、gzip
206      で圧縮されたリクエスト本体を伸張するフィルタも提供しています。
207      この機能を有効にするには、<code class="directive"><a href="../mod/core.html#setinputfilter">SetInputFilter</a></code>
208      か <code class="directive"><a href="../mod/mod_mime.html#addinputfilter">AddInputFilter</a></code> を使用して、
209      <code>DEFLATE</code> フィルタを入力フィルタチェインに組み込みます。
210      例えば次のようになります。</p>
211
212      <div class="example"><p><code>
213        &lt;Location /dav-area&gt;<br />
214        <span class="indent">
215          SetInputFilter DEFLATE<br />
216        </span>
217        &lt;/Location&gt;
218      </code></p></div>
219      
220      <p>この設定であれば、<code>Content-Encoding: gzip</code>
221      ヘッダを含むリクエストが来ると、本体は自動的に伸張されます。
222      gzip リクエスト本体を送信するブラウザはあまりありません。
223      しかし、例えば <a href="http://www.webdav.org">WebDAV</a>
224      クライアントの幾つかなど、特別なアプリケーションでリクエストの
225      圧縮を実際にサポートしているものもあります。</p>
226
227      <div class="warning"><h3>Content-Length に関する注意</h3>
228        <p>リクエスト本体それ自体を評価する場合は、<em><code>Content-Length</code>
229        ヘッダを信用しないでください</em>。Content-Length ヘッダは、
230        クライアントから送信されるデータの長さを反映しているのであって、
231        伸張されたデータストリームの<em>バイトカウントではありません</em>。</p>
232      </div>

236<h2><a name="proxies" id="proxies">Proxy サーバでの扱い</a></h2>
237
238    <p><code class="module"><a href="../mod/mod_deflate.html">mod_deflate</a></code> モジュールは <code>Vary: Accept-Encoding</code>
239    HTTP 応答ヘッダを送信して、適切な <code>Accept-Encoding</code>
240    リクエストヘッダを送信するクライアントに対してのみ、
241    プロクシサーバがキャッシュした応答を送信するように注意を喚起します。
242    このようにして、圧縮を扱うことのできないクライアントに
243    圧縮された内容が送られることのないようにします。</p>
244
245    <p>もし特別に何かに依存して除外したい場合、例えば <code>User-Agent</code>
246    ヘッダなどに依存している場合、手動で <code>Vary</code> ヘッダを設定して、
247    追加の制限についてプロクシサーバに注意を行なう必要があります。
248    例えば <code>User-Agent</code> に依存して <code>DEFLATE</code>
249    を追加する典型的な設定では、次のように追加することになります。</p>
250
251    <div class="example"><p><code>
252      Header append Vary User-Agent
253    </code></p></div>
254    
255    <p>リクエストヘッダ以外の情報 (<em>例えば</em> HTTP バージョン)
256    に依存して圧縮するかどうか決める場合、
257    <code>Vary</code> ヘッダを <code>*</code> に設定する必要があります。
258    このようにすると、仕様に準拠したプロクシはキャッシュを全く行なわなくなります。</p>
259
260    <div class="example"><h3>例</h3><p><code>
261      Header set Vary *
262    </code></p></div>

309<div class="directive-section"><h2><a name="DeflateFilterNote" id="DeflateFilterNote">DeflateFilterNote</a> <a name="deflatefilternote" id="deflatefilternote">ディレクティブ</a></h2>
310<table class="directive">
311<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>ロギング用に圧縮比をメモに追加</td></tr>
312<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>DeflateFilterNote [<var>type</var>] <var>notename</var></code></td></tr>
313<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>
314<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
315<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_deflate</td></tr>
316<tr><th><a href="directive-dict.html#Compatibility">互換性:</a></th><td><var>type</var> is available since Apache 2.0.45</td></tr>
317</table>
318    <p><code class="directive">DeflateFilterNote</code> ディレクティブは
319    圧縮比に関するメモがリクエストに付加されることを指定します。
320    メモ (note) の名前はディレクティブに指定された値です。
321    メモは<a href="../logs.html#accesslog">アクセスログ</a>に
322    値を記録し、統計を取る目的にも使えます。</p>
323
324    <div class="example"><h3>例</h3><p><code>
325      DeflateFilterNote ratio<br />
326      <br />
327      LogFormat '"%r" %b (%{ratio}n) "%{User-agent}i"' deflate<br />
328      CustomLog logs/deflate_log deflate
329    </code></p></div>
330
331    <p>ログからもっと精密な値を抽出したい場合は、<var>type</var>
332    引数を使用して、データタイプをログのメモとして残すように指定できます。
333    <var>type</var> は次のうちの一つです。</p>
334
335    <dl>
336      <dt><code>Input</code></dt>
337      <dd>フィルタの入力ストリームのバイトカウントをメモに保存する。</dd>
338
339      <dt><code>Output</code></dt>
340      <dd>フィルタの出力ストリームのバイトカウントをメモに保存する。</dd>
341
342      <dt><code>Ratio</code></dt>
343      <dd>圧縮率 (<code>出力 / 入力 * 100</code>) をメモに保存する。
344      <var>type</var> 引数を省略した場合は、これがデフォルトとなります。</dd>
345    </dl>
346
347    <p>まとめると、次のようにログを取ることになるでしょう。</p>
348
349    <div class="example"><h3>精密なログ採取</h3><p><code>
350      DeflateFilterNote Input instream<br />
351      DeflateFilterNote Output outstream<br />
352      DeflateFilterNote Ratio ratio<br />
353      <br />
354      LogFormat '"%r" %{outstream}n/%{instream}n (%{ratio}n%%)' deflate<br />
355      CustomLog logs/deflate_log deflate
356    </code></p></div>
357

420<div class="directive-section"><h2><a name="DeflateWindowSize" id="DeflateWindowSize">DeflateWindowSize</a> <a name="deflatewindowsize" id="deflatewindowsize">ディレクティブ</a></h2>
421<table class="directive">
422<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>Zlib の圧縮用ウィンドウの大きさ</td></tr>
423<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>DeflateWindowSize <var>value</var></code></td></tr>
424<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>DeflateWindowSize 15</code></td></tr>
425<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>
426<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
427<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_deflate</td></tr>
428</table>
429    <p><code class="directive">DeflateWindowSize</code> ディレクティブは
430    zlib の圧縮用ウィンドウ (訳注: zlib で使用される履歴バッファ) 
431    の大きさを指定します (1 から 15 の間の値)。
432    一般的に大きなウィンドウサイズを使用すると圧縮率が向上します。
433    (訳注: 2 を底とする対数の値になります。
434    8 から 15 にするのが良いでしょう。)</p>
