38<h3>Summary</h3>
39
40    <p>This module provides a convenient way to set timeouts and minimum data
41    rates for receiving requests. Should a timeout occur or a data rate be
42    to low, the corresponding connection will be closed by the server.</p>
43
44    <p>This is logged at <code class="directive"><a href="../mod/core.html#loglevel">LogLevel</a></code>
45    <code>info</code>.</p>
46    
47    <p>If needed, the <code class="directive"><a href="../mod/core.html#loglevel">LogLevel</a></code> directive
48    can be tweaked to explicitly log it:</p>
49    <pre class="prettyprint lang-config">LogLevel reqtimeout:info</pre>

64<h2><a name="examples" id="examples">Examples</a></h2>
65
66    <ol>
67      <li>
68        Allow for 5 seconds to complete the TLS handshake, 10 seconds to
69        receive the request headers and 30 seconds for receiving the
70        request body:
71
72        <pre class="prettyprint lang-config">RequestReadTimeout handshake=5 header=10 body=30</pre>
73
74      </li>
75
76      <li>
77        Allow at least 10 seconds to receive the request body.
78        If the client sends data, increase the timeout by 1 second for every
79        1000 bytes received, with no upper limit for the timeout (except for
80        the limit given indirectly by
81        <code class="directive"><a href="../mod/core.html#limitrequestbody">LimitRequestBody</a></code>):
82
83        <pre class="prettyprint lang-config">RequestReadTimeout body=10,MinRate=1000</pre>
84
85      </li>
86
87      <li>
88        Allow at least 10 seconds to receive the request headers.
89        If the client sends data, increase the timeout by 1 second for every
90        500 bytes received. But do not allow more than 30 seconds for the
91        request headers:
92
93        <pre class="prettyprint lang-config">RequestReadTimeout header=10-30,MinRate=500</pre>
94
95      </li>
96
97      <li>
98        Usually, a server should have both header and body timeouts configured.
99        If a common configuration is used for http and https virtual hosts, the
100        timeouts should not be set too low:
101
102        <pre class="prettyprint lang-config">RequestReadTimeout header=20-40,MinRate=500 body=20,MinRate=500</pre>
103
104      </li>
105
106    </ol>

110<table class="directive">
111<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Set timeout values for completing the TLS handshake, receiving
112the request headers and/or body from client.
113</td></tr>
114<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>RequestReadTimeout
115[handshake=<var>timeout</var>[-<var>maxtimeout</var>][,MinRate=<var>rate</var>]
116[header=<var>timeout</var>[-<var>maxtimeout</var>][,MinRate=<var>rate</var>]
117[body=<var>timeout</var>[-<var>maxtimeout</var>][,MinRate=<var>rate</var>]
118</code></td></tr>
119<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>RequestReadTimeout handshake=0 header=20-40,MinRate=500 body=20,MinRate=500</code></td></tr>
120<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host</td></tr>
121<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
122<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_reqtimeout</td></tr>
123<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Available in version 2.2.15 and later; defaulted to disabled in
124version 2.3.14 and earlier. The <code>handshake</code> stage is available since
125version 2.4.39.
126</td></tr>
127</table>
128    <p>This directive can set various timeouts for completing the TLS handshake,
129    receiving the request headers and/or the request body from the client.
130    If the client fails to complete each of these stages within the configured
131    time, a <code>408 REQUEST TIME OUT</code> error is sent.</p>
132
133    <p>For SSL virtual hosts, the <code>handshake</code> timeout values is the time
134    needed to do the initial SSL handshake.  If the user's browser is configured to
135    query certificate revocation lists and the CRL server is not reachable, the
136    initial SSL handshake may take a significant time until the browser gives up
137    waiting for the CRL.  Therefore the <code>handshake</code> timeout should take
138    this possible overhead into consideration for SSL virtual hosts (if necessary).
139    The body timeout values include the time needed for SSL renegotiation
140    (if necessary).</p>
141
142    <p>When an <code class="directive"><a href="../mod/core.html#acceptfilter">AcceptFilter</a></code> is in use
143    (usually the case on Linux and FreeBSD), the socket is not sent to the
144    server process before at least one byte (or the whole request for
145    <code>httpready</code>) is received. The handshake and header timeouts
146    configured with <code class="directive">RequestReadTimeout</code> are only effective
147    after the server process has received the socket.</p>
148
149    <p>For each of the three timeout stages (handshake, header or body), there are
150    three ways to specify the timeout:
151    </p>
152
153    <ul>
154
155    <li><strong>Fixed timeout value</strong>:<br />
156
157    <div class="example"><p><code><var>stage</var>=<var>timeout</var></code></p></div>
158
159    <p>The time in seconds allowed for completing the whole stage (handshaking,
160    reading all of the request headers or body). A value of 0 means no limit.</p>
161    </li>
162
163    <li><strong>Disable module for a vhost</strong>:<br />
164
165    <div class="example"><p><code>handshake=0 header=0 body=0</code></p></div>
166
167    <p>This disables <code class="module"><a href="../mod/mod_reqtimeout.html">mod_reqtimeout</a></code> completely (note that
168    <code>handshake=0</code> is the default already and could be omitted).</p>
169    </li>
170
171    <li><strong>Timeout value that is increased when data is
172    received</strong>:<br />
173    <div class="example"><p><code>
174    <var>stage</var>=<var>timeout</var>,MinRate=<var>data_rate</var>
175    </code></p></div>
176
177    <p>Same as above, but whenever data is received, the timeout value is
178    increased according to the specified minimum data rate (in bytes per
179    second).</p>
180    </li>
181
182    <li><strong>Timeout value that is increased when data is received, with an
183    upper bound</strong>:<br />
184    <div class="example"><p><code>
185    <var>stage</var>=<var>timeout</var>-<var>maxtimeout</var>,MinRate=<var>data_rate</var>
186    </code></p></div>
187
188    <p>Same as above, but the timeout will not be increased above the second
189    value of the specified timeout range.</p>
190    </li>
191
192    </ul>
