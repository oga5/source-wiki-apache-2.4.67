38<h3>Summary</h3>
39
40<p>This module provides an output filter to rewrite HTML links in a
41proxy situation, to ensure that links work for users outside the proxy.
42It serves the same purpose as Apache's <code class="directive"><a href="../mod/mod_proxy.html#proxypassreverse">ProxyPassReverse</a></code> directive does
43for HTTP headers, and is an essential component of a reverse proxy.</p>
44
45<p>For example, if a company has an application server at
46<code>appserver.example.com</code> that is only visible from within
47the company's internal network, and a public webserver
48<code>www.example.com</code>, they may wish to provide a gateway to the
49application server at <code>http://www.example.com/appserver/</code>.
50When the application server links to itself, those links need to be
51rewritten to work through the gateway. <code class="module"><a href="../mod/mod_proxy_html.html">mod_proxy_html</a></code> serves to rewrite
52<code>&lt;a href="http://appserver.example.com/foo/bar.html"&gt;foobar&lt;/a&gt;</code> to
53<code>&lt;a href="http://www.example.com/appserver/foo/bar.html"&gt;foobar&lt;/a&gt;</code>
54making it accessible from outside.</p>
55
56<p>mod_proxy_html was originally developed at WebÞing, whose
57extensive <a href="http://apache.webthing.com/mod_proxy_html/">documentation</a> may be useful to users.</p>

79<div class="directive-section"><h2><a name="ProxyHTMLBufSize" id="ProxyHTMLBufSize">ProxyHTMLBufSize</a> <a name="proxyhtmlbufsize" id="proxyhtmlbufsize">Directive</a></h2>
80<table class="directive">
81<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Sets the buffer size increment for buffering inline scripts and
82stylesheets.</td></tr>
83<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ProxyHTMLBufSize <var>bytes</var></code></td></tr>
84<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>ProxyHTMLBufSize 8192</code></td></tr>
85<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
86<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
87<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_proxy_html</td></tr>
88<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Version 2.4 and later; available as a third-party
89for earlier 2.x versions</td></tr>
90</table>
91<p>In order to parse non-HTML content (stylesheets and scripts) embedded
92in HTML documents, <code class="module"><a href="../mod/mod_proxy_html.html">mod_proxy_html</a></code>
93has to read the entire script or stylesheet into a buffer.  This buffer will
94be expanded as necessary to hold the largest script or stylesheet in a page,
95in increments of <var>bytes</var> as set by this directive.</p>
96<p>The default is 8192, and will work well for almost all pages.  However,
97if you know you're proxying pages containing stylesheets and/or
98scripts bigger than 8K (that is, for a single script or stylesheet,
99NOT in total), it will be more efficient to set a larger buffer
100size and avoid the need to resize the buffer dynamically during a request.
101</p>

125<div class="directive-section"><h2><a name="ProxyHTMLDocType" id="ProxyHTMLDocType">ProxyHTMLDocType</a> <a name="proxyhtmldoctype" id="proxyhtmldoctype">Directive</a></h2>
126<table class="directive">
127<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Sets an HTML or XHTML document type declaration.</td></tr>
128<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ProxyHTMLDocType HTML|XHTML [Legacy]<br /><strong>OR</strong>
129<br />ProxyHTMLDocType <var>fpi</var> [SGML|XML]</code></td></tr>
130<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
131<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
132<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_proxy_html</td></tr>
133<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Version 2.4 and later; available as a third-party
134for earlier 2.x versions</td></tr>
135</table>
136<p>In the first form, documents will be declared as HTML 4.01 or XHTML 1.0
137according to the option selected.  This option also determines whether
138HTML or XHTML syntax is used for output.   Note that the format of the
139documents coming from the backend server is immaterial: the parser will
140deal with it automatically.  If the optional second argument is set to
141<code>Legacy</code>, documents will be declared "Transitional", an option that may
142be necessary if you are proxying pre-1998 content or working with defective
143authoring/publishing tools.</p>
144<p>In the second form, it will insert your own FPI.  The optional second
145argument determines whether SGML/HTML or XML/XHTML syntax will be used.</p>
146<p>The default is changed to omitting any FPI,
147on the grounds that no FPI is better than a bogus one.  If your backend
148generates decent HTML or XHTML, set it accordingly.</p>
149<p>If the first form is used, mod_proxy_html
150will also clean up the HTML to the specified standard.  It cannot
151fix every error, but it will strip out bogus elements and attributes.
152It will also optionally log other errors at <code class="directive"><a href="../mod/core.html#loglevel">LogLevel</a></code> Debug.</p>

177<div class="directive-section"><h2><a name="ProxyHTMLEvents" id="ProxyHTMLEvents">ProxyHTMLEvents</a> <a name="proxyhtmlevents" id="proxyhtmlevents">Directive</a></h2>

179<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Specify attributes to treat as scripting events.</td></tr>

187<p>Specifies one or more attributes to treat as scripting events and
188apply <code class="directive"><a href="#proxyhtmlurlmap">ProxyHTMLURLMap</a></code>s to where enabled.
189You can specify any number of attributes in one or more
190<code class="directive">ProxyHTMLEvents</code> directives.</p>
191<div class="note"><h3>Requires ProxyHTMLExtended</h3>
192<p>Scripting events are only processed when
193<code class="directive"><a href="#proxyhtmlextended">ProxyHTMLExtended</a></code> is set to
194<var>On</var>.  Without it, event attributes are passed through unmodified
195even if listed in <code class="directive">ProxyHTMLEvents</code>.</p>
196</div>
197<p>Normally you'll set this globally.  If you set <code class="directive">ProxyHTMLEvents</code> in more than
198one scope so that one overrides the other, you'll need to specify a complete
199set in each of those scopes.</p>
200<p>A default configuration is supplied in <var>proxy-html.conf</var>
201and defines the events in standard HTML 4 and XHTML 1.</p>
202
203</div>

208and scripting events.</td></tr>
209<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ProxyHTMLExtended On|Off</code></td></tr>
210<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>ProxyHTMLExtended Off</code></td></tr>

217<p>Set to <code>Off</code>, HTML links are rewritten according to the
218<code class="directive"><a href="#proxyhtmlurlmap">ProxyHTMLURLMap</a></code> directives, but links appearing
219in Javascript and CSS are ignored.</p>
220<p>Set to <code>On</code>, all scripting events (as determined by
221<code class="directive"><a href="#proxyhtmlevents">ProxyHTMLEvents</a></code>) and embedded scripts or
222stylesheets are also processed by the <code class="directive"><a href="#proxyhtmlurlmap">ProxyHTMLURLMap</a></code>
223rules, according to the flags set for each rule.  Since this requires more
224parsing, performance will be best if you only enable it when strictly necessary.
225</p><p>
226You'll also need to take care over patterns matched, since the parser has no
227knowledge of what is a URL within an embedded script or stylesheet.
228In particular, extended matching of <code>/</code> is likely to lead to
229false matches.
230</p>

278<div class="directive-section"><h2><a name="ProxyHTMLLinks" id="ProxyHTMLLinks">ProxyHTMLLinks</a> <a name="proxyhtmllinks" id="proxyhtmllinks">Directive</a></h2>
279<table class="directive">
280<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Specify HTML elements that have URL attributes to be rewritten.</td></tr>
281<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ProxyHTMLLinks <var>element attribute [attribute2 ...]</var></code></td></tr>
282<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
283<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>

288<p>Specifies elements that have URL attributes that should be rewritten
289using standard <code class="directive"><a href="#proxyhtmlurlmap">ProxyHTMLURLMap</a></code>s.
290You will need one <code class="directive">ProxyHTMLLinks</code> directive per element,
291but it can have any number of attributes.</p>
292<p>Normally you'll set this globally.  If you set <code class="directive">ProxyHTMLLinks</code> in more than
293one scope so that one overrides the other, you'll need to specify a complete
294set in each of those scopes.</p>
295<p>A default configuration is supplied in <var>proxy-html.conf</var>
296and defines the HTML links for standard HTML 4 and XHTML 1.</p>
297<div class="example"><h3>Examples from proxy-html.conf</h3><pre class="prettyprint lang-config">ProxyHTMLLinks  a          href
298ProxyHTMLLinks  area       href
299ProxyHTMLLinks  link       href
300ProxyHTMLLinks  img        src longdesc usemap
301ProxyHTMLLinks  object     classid codebase data usemap
302ProxyHTMLLinks  q          cite
303ProxyHTMLLinks  blockquote cite
304ProxyHTMLLinks  ins        cite
305ProxyHTMLLinks  del        cite

315<div class="directive-section"><h2><a name="ProxyHTMLMeta" id="ProxyHTMLMeta">ProxyHTMLMeta</a> <a name="proxyhtmlmeta" id="proxyhtmlmeta">Directive</a></h2>
316<table class="directive">
317<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Turns on or off extra pre-parsing of metadata in HTML
318<code>&lt;head&gt;</code> sections.</td></tr>
319<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ProxyHTMLMeta On|Off</code></td></tr>
320<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>ProxyHTMLMeta Off</code></td></tr>
321<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
322<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
323<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_proxy_html</td></tr>
324<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Version 2.4 and later; available as a third-party
325module for earlier 2.x versions.</td></tr>
326</table>
327    <p>This turns on or off pre-parsing of metadata in HTML
328    <code>&lt;head&gt;</code> sections.</p>
329    <p>If not required, turning ProxyHTMLMeta Off will give a small
330    performance boost by skipping this parse step.  However, it
331    is sometimes necessary for internationalisation to work correctly.</p>
332    <p><code class="directive">ProxyHTMLMeta</code> has two effects.  Firstly and most importantly
333    it enables detection of character encodings declared in the form</p>
334    <pre>&lt;meta http-equiv="Content-Type" content="text/html;charset=<var>foo</var>"&gt;</pre>
335    <p>or, in the case of an XHTML document, an XML declaration.
336    It is NOT required if the charset is declared in a real HTTP header
337    (which is always preferable) from the backend server, nor if the
338    document is <var>utf-8</var> (unicode) or a subset such as ASCII.
339    You may also be able to dispense with it where documents use a
340    default declared using <code class="directive"><a href="../mod/mod_xml2enc.html#xml2encdefault">xml2EncDefault</a></code>, but that risks propagating an
341    incorrect declaration.  A <code class="directive"><a href="#proxyhtmlcharsetout">ProxyHTMLCharsetOut</a></code>
342    can remove that risk, but is likely to be a bigger processing
343    overhead than enabling ProxyHTMLMeta.</p>
344    <p>The other effect of enabling <code class="directive">ProxyHTMLMeta</code> is to parse all
345    <code>&lt;meta http-equiv=...&gt;</code> declarations and convert
346    them to real HTTP headers, in keeping with the original purpose
347    of this form of the HTML &lt;meta&gt; element.</p>
348
349    <div class="warning"><h3>Warning</h3>
350      Because ProxyHTMLMeta promotes <strong>all</strong>
351      <code>http-equiv</code> elements to HTTP headers, it is important that you
352      only enable it in cases where you trust the HTML content as much as you
353      trust the upstream server. If the HTML is controlled by bad actors, it
354      will be possible for them to inject arbitrary, possibly malicious, HTTP
355      headers into your server's responses.
356    </div>
357
358</div>

360<div class="directive-section"><h2><a name="ProxyHTMLStripComments" id="ProxyHTMLStripComments">ProxyHTMLStripComments</a> <a name="proxyhtmlstripcomments" id="proxyhtmlstripcomments">Directive</a></h2>
361<table class="directive">
362<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Determines whether to strip HTML comments.</td></tr>
363<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ProxyHTMLStripComments On|Off</code></td></tr>
364<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>ProxyHTMLStripComments Off</code></td></tr>
365<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
366<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
367<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_proxy_html</td></tr>
368<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Version 2.4 and later; available as a third-party
369for earlier 2.x versions</td></tr>
370</table>
371<p>This directive will cause mod_proxy_html to strip HTML comments.
372Note that this will also kill off any scripts or styles embedded in
373comments (a bogosity introduced in 1995/6 with Netscape 2 for the
374benefit of then-older browsers, but still in use today).
375It may also interfere with comment-based processors such as SSI or ESI:
376be sure to run any of those <em>before</em> mod_proxy_html in the
377filter chain if stripping comments!</p>
378
379</div>

381<div class="directive-section"><h2><a name="ProxyHTMLURLMap" id="ProxyHTMLURLMap">ProxyHTMLURLMap</a> <a name="proxyhtmlurlmap" id="proxyhtmlurlmap">Directive</a></h2>
382<table class="directive">
383<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Defines a rule to rewrite HTML links</td></tr>
384<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ProxyHTMLURLMap <var>from-pattern to-pattern [flags] [cond]</var></code></td></tr>
385<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
386<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
387<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_proxy_html</td></tr>
388<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Version 2.4 and later; available as a third-party
389module for earlier 2.x versions.</td></tr>
390</table>
391<p>This is the key directive for rewriting HTML links. When parsing a document,
392whenever a link target matches <var>from-pattern</var>, the matching
393portion will be rewritten to <var>to-pattern</var>, as modified by any
394flags supplied and by the 
395<code class="directive"><a href="#proxyhtmlextended">ProxyHTMLExtended</a></code> directive.
396Only the elements specified using
397the <code class="directive"><a href="#proxyhtmllinks">ProxyHTMLLinks</a></code> directive
398will be considered as HTML links.</p>
399
400<p>The optional third argument may define any of the following
401<strong>Flags</strong>.  Flags are case-sensitive.</p>
402<dl>
403<dt>h</dt>
404<dd><p>Ignore HTML links (pass through unchanged)</p></dd>
405<dt>e</dt>
406<dd><p>Ignore scripting events (pass through unchanged)</p></dd>
407<dt>c</dt>
408<dd><p>Pass embedded script and style sections through untouched.</p></dd>
409
410<dt>L</dt>
411<dd><p>Last-match.  If this rule matches, no more rules are applied
412(note that this happens automatically for HTML links).</p></dd>
413<dt>l</dt>
414<dd><p>Opposite to L.  Overrides the one-change-only default
415behavior with HTML links.</p></dd>
416<dt>R</dt>
417<dd><p>Use Regular Expression matching-and-replace.  <code>from-pattern</code>
418is a regexp, and <code>to-pattern</code> a replacement string that may be
419based on the regexp.  Regexp memory is supported: you can use brackets ()
420in the <code>from-pattern</code> and retrieve the matches with $1 to $9
421in the <code>to-pattern</code>.</p>
422
423<p>If R is not set, it will use string-literal search-and-replace.
424The logic is <em>starts-with</em> in HTML links, but
425<em>contains</em> in scripting events and embedded script and style sections.
426</p>
427</dd>
428<dt>x</dt>
429<dd><p>Use POSIX extended Regular Expressions.  Only applicable with R.</p></dd>
430<dt>i</dt>
431<dd><p>Case-insensitive matching.  Only applicable with R.</p></dd>
432
433<dt>n</dt>
434<dd><p>Disable regexp memory (for speed).  Only applicable with R.</p></dd>
435<dt>s</dt>
436<dd><p>Line-based regexp matching.  Only applicable with R.</p></dd>
437<dt>^</dt>
438<dd><p>Match at start only.  This applies only to string matching
439(not regexps) and is irrelevant to HTML links.</p></dd>
440<dt>$</dt>
441<dd><p>Match at end only.  This applies only to string matching
442(not regexps) and is irrelevant to HTML links.</p></dd>
443<dt>V</dt>
444<dd><p>Interpolate environment variables in <code>to-pattern</code>.
445A string of the form <code>${varname|default}</code> will be replaced by the
446value of environment variable <code>varname</code>.  If that is unset, it
447is replaced by <code>default</code>.  The <code>|default</code> is optional.</p>
448<p>NOTE: interpolation will only be enabled if
449<code class="directive"><a href="#proxyhtmlinterp">ProxyHTMLInterp</a></code> is <var>On</var>.</p>
450</dd>
451
452<dt>v</dt>
453<dd><p>Interpolate environment variables in <code>from-pattern</code>.
454Patterns supported are as above.</p>
455<p>NOTE: interpolation will only be enabled if
456<code class="directive"><a href="#proxyhtmlinterp">ProxyHTMLInterp</a></code> is <var>On</var>.</p>
457</dd>
458</dl>
459
460<p>The optional fourth <strong>cond</strong> argument defines a condition
461that will be evaluated per Request, provided
462<code class="directive"><a href="#proxyhtmlinterp">ProxyHTMLInterp</a></code> is <var>On</var>.
463If the condition evaluates FALSE the map will not be applied in this request.
464If TRUE, or if no condition is defined, the map is applied.</p>
465<p>A <strong>cond</strong> is evaluated by the <a href="../expr.html">Expression Parser</a>.  In addition, the simpler syntax of conditions
466in mod_proxy_html 3.x for HTTPD 2.0 and 2.2 is also supported.</p>
