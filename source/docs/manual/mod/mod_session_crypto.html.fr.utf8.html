37<h3>Sommaire</h3>
38
39    <div class="warning"><h3>Avertissement</h3>
40      <p>Les modules de session font usage des cookies HTTP, et peuvent
41      à ce titre être victimes d'attaques de type Cross Site Scripting,
42      ou divulguer des informations à caractère privé aux clients.
43      Veuillez vous assurer que les risques ainsi encourus ont été pris
44      en compte avant d'activer le support des sessions sur votre
45      serveur.</p>
46    </div>
47
48    <p>Ce sous-module du module <code class="module"><a href="../mod/mod_session.html">mod_session</a></code> fournit le
49    support du chiffrement des sessions utilisateur avant de les
50    enregistrer dans une base de données locale, ou dans un cookie HTTP
51    au niveau du navigateur distant.</p>
52
53    <p>Il peut contribuer à préserver la confidentialité des sessions
54    lorsque leur contenu doit rester privé pour
55    l'utilisateur, ou lorsqu'une protection contre les attaques de type
56    cross site scripting est nécessaire.</p>
57
58    <p>Pour plus de détails à propos de l'interface des sessions, voir
59    la documentation du module <code class="module"><a href="../mod/mod_session.html">mod_session</a></code>.</p>

80<h2><a name="basicusage" id="basicusage">Utilisation de base</a></h2>
81
82      <p>Pour créer une session chiffrée et la stocker dans un cookie
83      nommé <var>session</var>, configurez la comme suit :</p>
84
85      <div class="example"><h3>Session chiffrée stockée au niveau du
86      serveur</h3><pre class="prettyprint lang-config">Session On
87SessionCookieName session path=/
88SessionCryptoPassphrase secret</pre>
89</div>
90
91      <p>La session sera chiffrée avec la clé spécifiée. Il est possible
92      de configurer plusieurs serveurs pour qu'ils puissent partager des
93      sessions, en s'assurant que la même clé de chiffrement est
94      utilisée sur chaque serveur.</p>
95
96      <p>Si la clé de chiffrement est modifiée, les sessions seront
97      automatiquement invalidées.</p>
98
99      <p>Pour des détails sur la manière dont une session peut être
100      utilisée pour stocker des informations de type nom
101      d'utilisateur/mot de passe, voir la documentation du module
102      <code class="module"><a href="../mod/mod_auth_form.html">mod_auth_form</a></code>.</p>
103
104    </div>

106<div class="directive-section"><h2><a name="sessioncryptocipher" id="sessioncryptocipher">Directive</a> <a name="SessionCryptoCipher" id="SessionCryptoCipher">SessionCryptoCipher</a></h2>
107<table class="directive">
108<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>L'algorithme à utiliser pour le chiffrement de la session</td></tr>
109<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>SessionCryptoCipher <var>algorithme</var></code></td></tr>
110<tr><th><a href="directive-dict.html#Default">Défaut:</a></th><td><code>SessionCryptoCipher aes256</code></td></tr>
111<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale, serveur virtuel, répertoire, .htaccess</td></tr>
112<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Expérimental</td></tr>
113<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_session_crypto</td></tr>
114<tr><th><a href="directive-dict.html#Compatibility">Compatibilité:</a></th><td>Disponible depuis la version 2.3.0 du serveur HTTP Apache</td></tr>
115</table>
116    <p>La directive <code class="directive">SessionCryptoCipher</code> permet de
117    spécifier l'algorithme à utiliser pour le chiffrement. En l'absence
118    de spécification, l'algorithme par défaut est <code>aes256</code>.</p>
119
120    <p>L'algorithme peut être choisi, en fonction du moteur de chiffrement
121    utilisé, parmi les valeurs suivantes :</p>
122
123    <ul><li>3des192</li><li>aes128</li><li>aes192</li><li>aes256</li></ul>

128<div class="directive-section"><h2><a name="sessioncryptodriver" id="sessioncryptodriver">Directive</a> <a name="SessionCryptoDriver" id="SessionCryptoDriver">SessionCryptoDriver</a></h2>
129<table class="directive">
130<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Le pilote de chiffrement à utiliser pour chiffrer les
131sessions</td></tr>
132<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>SessionCryptoDriver <var>nom</var> <var>[param[=valeur]]</var></code></td></tr>
133<tr><th><a href="directive-dict.html#Default">Défaut:</a></th><td><code>aucun</code></td></tr>
134<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale</td></tr>
135<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Expérimental</td></tr>
136<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_session_crypto</td></tr>
137<tr><th><a href="directive-dict.html#Compatibility">Compatibilité:</a></th><td>Disponible depuis la version 2.3.0
138d'Apache</td></tr>
139</table>
140    <p>La directive <code class="directive">SessionCryptoDriver</code> permet de
141    spécifier le nom du pilote à utiliser pour le chiffrement. Si aucun
142    pilote n'est spécifié, le pilote utilisé par défaut sera le pilote
143    recommandé compilé avec APR-util.</p>
144
145    <p>Le pilote de chiffrement <var>NSS</var> nécessite certains
146    paramètres de configuration, qui seront spécifiés comme arguments de
147    la directive avec des valeurs optionnelles après le nom du
148    pilote.</p>
149
150    <div class="example"><h3>NSS sans base de données de certificats</h3><pre class="prettyprint lang-config">SessionCryptoDriver nss</pre>
151</div>
152
153    <div class="example"><h3>NSS avec base de données de certificats</h3><pre class="prettyprint lang-config">SessionCryptoDriver nss dir=certs</pre>
154</div>
155
156    <div class="example"><h3>NSS avec base de données de certificats et
157    paramètres</h3><pre class="prettyprint lang-config">SessionCryptoDriver nss dir=certs clé3=clé3.db cert7=cert7.db secmod=secmod</pre>
158</div>
159
160    <div class="example"><h3>NSS avec chemins contenant des espaces</h3><pre class="prettyprint lang-config">SessionCryptoDriver nss "dir=My Certs" key3=key3.db cert7=cert7.db secmod=secmod</pre>
161</div>
162
163    <p>Le pilote de chiffrement <var>NSS</var> peut avoir été configuré
164    au préalable dans une autre partie du serveur, par exemple depuis
165    <code class="module"><a href="../mod/mod_nss.html">mod_nss</a></code> ou <code class="module"><a href="../mod/mod_ldap.html">mod_ldap</a></code>. Si c'est le
166    cas, un avertissement sera enregistré dans le journal, et la
167    configuration existante s'en trouvera affectée. Pour éviter cet
168    avertissement, utilisez le paramètre <code>noinit</code> comme suit :</p>
169
170    <div class="example"><h3>NSS avec base de données de certificats</h3><pre class="prettyprint lang-config">SessionCryptoDriver nss noinit</pre>
171</div>
172
173    <p>Pour éviter la confusion, assurez-vous que tous les modules
174    utilisant NSS soient configurés avec des paramètres identiques.</p>
175
176    <p>Le pilote de chiffrement <var>openssl</var> accepte un paramètre
177    optionnel permettant de spécifier le moteur de chiffrement à
178    utiliser.</p>
179
180    <div class="example"><h3>OpenSSL avec spécification du moteur de chiffrement</h3><pre class="prettyprint lang-config">SessionCryptoDriver openssl engine=nom-moteur</pre>
181</div>

186<div class="directive-section"><h2><a name="sessioncryptopassphrase" id="sessioncryptopassphrase">Directive</a> <a name="SessionCryptoPassphrase" id="SessionCryptoPassphrase">SessionCryptoPassphrase</a></h2>
187<table class="directive">
188<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>La clé utilisée pour chiffrer la session</td></tr>
189<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>SessionCryptoPassphrase <var>secret</var> [ <var>secret</var> ... ] </code></td></tr>
190<tr><th><a href="directive-dict.html#Default">Défaut:</a></th><td><code>none</code></td></tr>
191<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale, serveur virtuel, répertoire, .htaccess</td></tr>
192<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Expérimental</td></tr>
193<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_session_crypto</td></tr>
194<tr><th><a href="directive-dict.html#Compatibility">Compatibilité:</a></th><td>Disponible depuis la version 2.3.0
195d'Apache</td></tr>
196</table>
197    <p>La directive <code class="directive">SessionCryptoPassphrase</code>
198    permet de spécifier les clés à utiliser pour chiffrer de manière
199    symétrique le contenu de la session avant de l'enregistrer, ou pour
200    déchiffrer le contenu de la session après sa lecture.</p>
201
202    <p>L'utilisation de clés longues et composées de caractères vraiment
203    aléatoires est plus performant en matière de sécurité. Modifier une
204    clé sur un serveur a pour effet d'invalider toutes les sessions
205    existantes.</p>
206
207    <p>Il est possible de spécifier plusieurs clés afin de mettre en
208    oeuvre la rotation de clés. La première clé spécifiée sera utilisée
209    pour le chiffrement, alors que l'ensemble des clés spécifiées le
210    sera pour le déchiffrement. Pour effectuer une rotation périodique
211    des clés sur plusieurs serveurs, ajoutez une nouvelle clé en fin de
212    liste, puis, une fois la rotation complète effectuée, supprimez la
213    première clé de la liste.</p>
214
215    <p>Depuis la version 2.4.7, si la valeur de l'argument commence par <var>exec:</var> , la commande
216    spécifiée sera exécutée, et la première ligne que cette dernière
217    renverra sur la sortie standard sera utilisée comme clé.</p>
218<div class="example"><pre class="prettyprint lang-config"># clé spécifiée et utilisée en tant que tel
219SessionCryptoPassphrase secret
220
221# exécution de /path/to/program pour générer la clé
222SessionCryptoPassphrase exec:/path/to/program
223
224# exécution de /path/to/program avec un argument pour générer la clé
225SessionCryptoPassphrase "exec:/path/to/otherProgram argument1"</pre>
226</div>

231<div class="directive-section"><h2><a name="sessioncryptopassphrasefile" id="sessioncryptopassphrasefile">Directive</a> <a name="SessionCryptoPassphraseFile" id="SessionCryptoPassphraseFile">SessionCryptoPassphraseFile</a></h2>
232<table class="directive">
233<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Le fichier contenant les clés utilisées pour chiffrer la
234session</td></tr>
235<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>SessionCryptoPassphraseFile <var>nom-fichier</var></code></td></tr>
236<tr><th><a href="directive-dict.html#Default">Défaut:</a></th><td><code>none</code></td></tr>
237<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale, serveur virtuel, répertoire</td></tr>
238<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Expérimental</td></tr>
239<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_session_crypto</td></tr>
240<tr><th><a href="directive-dict.html#Compatibility">Compatibilité:</a></th><td>Disponible depuis la version 2.3.0 du serveur HTTP Apache</td></tr>
241</table>
242    <p>La directive <code class="directive">SessionCryptoPassphraseFile</code>
243    permet de spécifier le nom d'un fichier de configuration contenant
244    les clés à utiliser pour le chiffrement et le déchiffrement de la
245    session (une clé par ligne). Le fichier est lu au démarrage du
246    serveur, et un redémarrage graceful est nécessaire pour prendre en
247    compte un éventuel changement de clés.</p>
248
249    <p>À la différence de la directive <code class="directive"><a href="#sessioncryptopassphrase">SessionCryptoPassphrase</a></code>, les
250    clés ne sont pas présentes dans le fichier de configuration de
251    httpd et peuvent être cachées via une protection
252    appropriée du fichier de clés.</p>
253
254    <p>Il est possible de spécifier plusieurs clés afin de mettre
255    en oeuvre la rotation de clés. La première clé
256    spécifiée sera utilisée pour le chiffrement, alors que
257    l'ensemble des clés spécifiées le sera pour le
258    déchiffrement. Pour effectuer une rotation périodique des
259    clés sur plusieurs serveurs, ajoutez une nouvelle clé en fin
260    de liste, puis, une fois la rotation complète effectuée,
261    supprimez la première clé de la liste.</p>
