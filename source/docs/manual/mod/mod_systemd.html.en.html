37<h3>Summary</h3>
38
39    <p>This module provides support for systemd integration. It allows
40    httpd to be used in a service with the systemd
41    <code>Type=notify</code> (see <a href="https://www.freedesktop.org/software/systemd/man/systemd.service.html">systemd.service(5)</a>
42    for more information).  The module is activated if loaded.</p>
43
44    <div class="example"><h3>Example of systemd service unit (more settings are probably needed for production systems)</h3><pre>[Unit]
45Description=The Apache HTTP Server
46After=network.target
47
48[Service]
49Type=notify
50ExecStart=/usr/local/apache2/bin/httpd -D FOREGROUND -k start
51ExecReload=/usr/local/apache2/bin/httpd -k graceful
52KillMode=mixed
53
54[Install]
55WantedBy=multi-user.target</pre></div>
56
57   <p>Special attention should be given to how <code>ExecStop</code>
58   and/or <code>KillMode</code> are configured for the service.  If
59   configured, an <code>ExecStop</code> command should be a
60   <em>synchronous operation</em> which itself exits when the daemon
61   has terminated.  Running <code>httpd -k stop</code>
62   <em>asynchronously</em> initiates daemon termination, so does not
63   satisfy this condition.  The example above uses
64   <code>KillMode=mixed</code> so that systemd sends
65   <code>SIGTERM</code> to signal the parent process (and only the
66   parent) to shut down.  The entire process group is then sent
67   <code>SIGKILL</code> after <code>TimeoutStopSec</code> elapses, if
68   any processes are still running.  See <a href="https://www.freedesktop.org/software/systemd/man/systemd.kill.html">systemd.kill(5)</a>
69   for more information.</p>
70    
71   <p>This module does not provide support for Systemd socket activation.</p>
72
73    <p><code class="directive"><a href="../mod/core.html#extendedstatus">ExtendedStatus</a></code> is
74    enabled by default if the module is loaded.  If <code class="directive"><a href="../mod/core.html#extendedstatus">ExtendedStatus</a></code> is not disabled in
75    the configuration, run-time load and request statistics are made
76    available in the <code>systemctl status</code> output.</p>
