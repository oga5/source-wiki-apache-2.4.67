82    <p>Dies beeinflusst das Verzeichnis, in welches der Apache zu wechseln
83    versucht, bevor er einen Hauptspeicherauszug <span class="transnote">(<em>Anm.d.&#220;.:</em> einen
84    so genannten Core-Dump)</span> erstellt. Die Voreinstellung ist das
85    <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>-Verzeichnis. Da dieses
86    jedoch nicht f&#252;r den Benutzer beschreibbar sein soll, unter dem
87    der Server l&#228;uft, werden normalerweise keine
88    Hauptspeicherausz&#252;ge geschrieben. Wenn Sie zum Debuggen
89    einen Hauptspeicherauszug haben m&#246;chten, k&#246;nnen Sie
90    ihn mit dieser Direktive an einem anderen Ort ablegen lassen.</p>
91
92    <div class="note"><h3>Hauptspeicherausz&#252;ge unter Linux</h3>
93      <p>Wenn Apache als <code>root</code> startet und zu einem anderen Benutzer 
94      wechselt, <em>deaktiviert</em> der Linux-Kernel Hauptspeicherausz&#252;ge
95      auch dann, wenn der Prozess in dem Verzeichnis schreiben darf. Ab Linux
96      2.4 reaktiviert Apache (ab 2.0.46) Hauptspeicherausz&#252;ge wieder,
97      jedoch nur dann, wenn Sie explizit
98      <code class="directive">CoreDumpDirectory</code> konfigurieren.</p>
99    </div>

140<div class="directive-section"><h2><a name="Listen" id="Listen">Listen</a>-<a name="listen" id="listen">Direktive</a></h2>
141<table class="directive">
142<tr><th><a href="directive-dict.html#Description">Beschreibung:</a></th><td>IP-Adressen und Ports, an denen der Server lauscht</td></tr>
143<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>Listen [<var>IP-Addresse</var>:]<var>Port</var></code></td></tr>
144<tr><th><a href="directive-dict.html#Context">Kontext:</a></th><td>Serverkonfiguration</td></tr>
145<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>MPM</td></tr>
146<tr><th><a href="directive-dict.html#Module">Modul:</a></th><td><code class="module"><a href="../mod/beos.html">beos</a></code>, <code class="module"><a href="../mod/leader.html">leader</a></code>, <code class="module"><a href="../mod/mpm_netware.html">mpm_netware</a></code>, <code class="module"><a href="../mod/mpm_winnt.html">mpm_winnt</a></code>, <code class="module"><a href="../mod/mpmt_os2.html">mpmt_os2</a></code>, <code class="module"><a href="../mod/perchild.html">perchild</a></code>, <code class="module"><a href="../mod/prefork.html">prefork</a></code>, <code class="module"><a href="../mod/threadpool.html">threadpool</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code></td></tr>
147<tr><th><a href="directive-dict.html#Compatibility">Kompatibilit&#228;t:</a></th><td>Seit Apache 2.0 vorgeschrieben</td></tr>
148</table>
149    <p>Die Direktive <code class="directive">Listen</code> weist den Apache an,
150    nur an den angegebenen IP-Adressen oder Ports zu lauschen.
151    Standardm&#228;&#223;ig antwortet er auf alle Anfragen an allen
152    IP-Interfaces. <code class="directive">Listen</code> ist nun eine notwendige
153    Anweisung. Wenn sie nicht in der Konfigurationsdatei enthalten ist,
154    wird der Server-Start fehlschlagen. Dies ist eine &#196;nderung
155    gegen&#252;ber fr&#252;heren Versionen des Apache.</p>
156
157    <p>Die Direktive <code class="directive">Listen</code> weist den Server an,
158    ankommende Anfragen am angegebenen Port oder der
159    Kombination aus Adresse und Port entgegenzunehmen. Wenn nur eine Portnummer
160    angegeben ist, dann lauscht der Server am angegebenen Port an allen
161    Interfaces. Wenn sowohl eine IP-Adresse als auch ein Port angegeben
162    sind, dann lauscht der Server am angegeben Port und Interface.</p>
163
164    <p>Es k&#246;nnen mehrere <code class="directive">Listen</code>-Anweisungen
165    verwendet werden, um eine Reihe von Adressen und Port anzugeben, an
166    denen gelauscht werden soll. Der Server antwortet auf Anfragen von
167    jedem der aufgef&#252;hrten Adressen und Ports.</p>
168
169    <p>Um beispielsweise den Server Verbindungen an den beiden Ports 80 und
170    8000 annehmen zu lassen, verwenden Sie:</p>
171
172    <div class="example"><p><code>
173      Listen 80<br />
174      Listen 8000
175    </code></p></div>
176
177    <p>Um den Server Verbindungen an zwei angegebenen Interfaces und Ports
178    annehmen zu lassen, verwenden Sie:</p>
179
180    <div class="example"><p><code>
181      Listen 192.170.2.1:80<br />
182      Listen 192.170.2.5:8000
183    </code></p></div>
184
185    <p>IPv6-Adressen m&#252;ssen wie in dem folgenden Beispiel in eckige
186    Klammern eingeschlossen werden:</p>
187
188    <div class="example"><p><code>
189      Listen [2001:db8::a00:20ff:fea7:ccea]:80
190    </code></p></div>
191
192    <div class="note"><h3>Fehlerm&#246;glichkeit</h3>
193      Mehrere <code class="directive">Listen</code>-Direktiven f&#252;r gleiche
194      IP-Adresse und Port f&#252;hren zur Fehlermeldung
195      <code>Address already in use</code> <span class="transnote">(<em>Anm.d.&#220;.:</em> Adresse schon in
196      Benutzung)</span>.
197    </div>

529<div class="directive-section"><h2><a name="ServerLimit" id="ServerLimit">ServerLimit</a>-<a name="serverlimit" id="serverlimit">Direktive</a></h2>
530<table class="directive">
531<tr><th><a href="directive-dict.html#Description">Beschreibung:</a></th><td>Obergrenze f&#252;r die konfigurierbare Anzahl von
532  Prozessen</td></tr>
533<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ServerLimit <var>Anzahl</var></code></td></tr>
534<tr><th><a href="directive-dict.html#Default">Voreinstellung:</a></th><td><code>F&#252;r Details siehe Beschreibung</code></td></tr>
535<tr><th><a href="directive-dict.html#Context">Kontext:</a></th><td>Serverkonfiguration</td></tr>
536<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>MPM</td></tr>
537<tr><th><a href="directive-dict.html#Module">Modul:</a></th><td><code class="module"><a href="../mod/leader.html">leader</a></code>, <code class="module"><a href="../mod/perchild.html">perchild</a></code>, <code class="module"><a href="../mod/prefork.html">prefork</a></code>, <code class="module"><a href="../mod/threadpool.html">threadpool</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code></td></tr>
538</table>
539    <p>Bei dem MPM <code class="module"><a href="../mod/prefork.html">prefork</a></code> bestimmt die Direktive
540    den w&#228;hrend der Lebensdauer des Apache-Prozesses maximal
541    einstellbaren Wert f&#252;r <code class="directive"><a href="#maxclients">MaxClients</a></code>. Beim MPM
542    <code class="module"><a href="../mod/worker.html">worker</a></code> bestimmt die Direktive in Verbindung mit
543    <code class="directive"><a href="#threadlimit">ThreadLimit</a></code> den Maximalwert
544    f&#252;r <code class="directive"><a href="#maxclients">MaxClients</a></code>
545    f&#252;r die Lebensdauer des Apache-Prozesses. Jeder Versuch, diese
546    Anweisung w&#228;hrend eines Neustarts zu &#228;ndern, wird ignoriert.
547    <code class="directive"><a href="#maxclients">MaxClients</a></code> kann jedoch
548    w&#228;hrend eines Neustarts ge&#228;ndert werden.</p>
549
550    <p>Lassen Sie besondere Vorsicht bei der Verwendung dieser Direktive
551    walten. Wenn <code class="directive">ServerLimit</code> auf einen Wert deutlich
552    h&#246;her als notwendig gesetzt wird, wird zus&#228;tzliches,
553    unbenutztes Shared-Memory belegt. Wenn sowohl
554    <code class="directive">ServerLimit</code> als auch <code class="directive"><a href="#maxclients">MaxClients</a></code> auf Werte gesetzt werden, die
555    gr&#246;&#223;er sind, als das System sie handhaben kann, dann kann
556    der Apache m&#246;glicherweise nicht starten, oder das System kann
557    instabil werden.</p>
558
559    <p>Verwenden Sie die Direktive bei dem MPM <code class="module"><a href="../mod/prefork.html">prefork</a></code>
560    nur, wenn Sie <code class="directive"><a href="#maxclients">MaxClients</a></code>
561    auf mehr als 256 (Voreinstellung) setzen m&#252;ssen. Setzen Sie den
562    Wert nicht h&#246;her als den Wert, den Sie f&#252;r <code class="directive"><a href="#maxclients">MaxClients</a></code> angeben m&#246;chten.</p>
563
564    <p>Verwenden Sie die Direktive bei <code class="module"><a href="../mod/worker.html">worker</a></code>,
565    <code class="module"><a href="../mod/leader.html">leader</a></code> und <code class="module"><a href="../mod/threadpool.html">threadpool</a></code> nur, wenn Ihre
566    <code class="directive"><a href="#maxclients">MaxClients</a></code>- und
567    <code class="directive"><a href="#threadsperchild">ThreadsPerChild</a></code>-Einstellungen
568    mehr als 16 Serverprozesse (Voreinstellung) erfordern. Setzen Sie den
569    Wert dieser Direktive nicht h&#246;her, als die Anzahl der Serverprozesse,
570    die daf&#252;r erforderlich ist, was Sie bei <code class="directive"><a href="#maxclients">MaxClients</a></code> und
571    <code class="directive"><a href="#threadsperchild">ThreadsPerChild</a></code> angeben
572    m&#246;chten.</p>
573
574    <p>Verwenden Sie die Direktive beim MPM <code class="module"><a href="../mod/perchild.html">perchild</a></code> nur,
575    wenn Sie <code class="directive"><a href="../mod/perchild.html#numservers">NumServers</a></code> auf einen
576    Wert gr&#246;&#223;er als 8 (Voreinstellung) setzen m&#252;ssen.</p>
577
578    <div class="note"><h3>Anmerkung</h3>
579      <p>Eine feste Begrenzung von <code>ServerLimit 20000</code> ist in den
580      Server einkompiliert (bei dem MPM <code class="module"><a href="../mod/prefork.html">prefork</a></code> 200000).
581      Dies soll unangenehme Effekte durch Tippfehler verhindern.</p>

717<div class="directive-section"><h2><a name="ThreadStackSize" id="ThreadStackSize">ThreadStackSize</a>-<a name="threadstacksize" id="threadstacksize">Direktive</a></h2>
718<table class="directive">
719<tr><th><a href="directive-dict.html#Description">Beschreibung:</a></th><td>Die Gr&#246;&#223;e des Stacks in Bytes, der von Threads
720verwendet wird, die Client-Verbindungen bearbeiten.</td></tr>
721<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ThreadStackSize <var>size</var></code></td></tr>
722<tr><th><a href="directive-dict.html#Default">Voreinstellung:</a></th><td><code>65536 unter NetWare; variiert bei anderen Betriebssystemen.</code></td></tr>
723<tr><th><a href="directive-dict.html#Context">Kontext:</a></th><td>Serverkonfiguration</td></tr>
724<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>MPM</td></tr>
725<tr><th><a href="directive-dict.html#Module">Modul:</a></th><td><code class="module"><a href="../mod/leader.html">leader</a></code>, <code class="module"><a href="../mod/mpm_netware.html">mpm_netware</a></code>, <code class="module"><a href="../mod/mpm_winnt.html">mpm_winnt</a></code>, <code class="module"><a href="../mod/perchild.html">perchild</a></code>, <code class="module"><a href="../mod/threadpool.html">threadpool</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code></td></tr>
726<tr><th><a href="directive-dict.html#Compatibility">Kompatibilit&#228;t:</a></th><td>Verf&#252;gbar seit Version 2.1</td></tr>
727</table>
728    <p>Die Direktive <code class="directive">ThreadStackSize</code> legt die
729    Gr&#246;&#223;e des Stacks (f&#252;r Autodaten) der Threads fest, die
730    Client-Verbindungen bearbeiten und Module aufrufen, welche bei der
731    Verarbeitung dieser Verbindungen helfen. In den meisten F&#228;llen ist die
732    Voreinstellung des Betriebssystems angemessen, doch unter bestimmten
733    Umst&#228;nden kann es sinnvoll sein, den Wert anzupassen:</p>
734
735    <ul>
736      <li>Auf Plattformen mit einer relativ kleinen Voreingestellung f&#252;r
737      die Gr&#246;&#223;e des Thread-Stacks (z.B. HP-UX) kann der Apache bei
738      der Verwendung einiger Drittanbietermodule, die einen relativ hohen Bedarf
739      an Speicherplatz haben, abst&#252;rzen. Ebendiese Module arbeiten
740      m&#246;glicherweise problemlos auf anderen Plattformen, wo der
741      voreingestellte Thread-Stack gr&#246;&#223;er ist. Derartige
742      Abst&#252;rze k&#246;nnen Sie vermeiden, indem Sie
743      <code class="directive">ThreadStackSize</code> auf einen h&#246;heren Wert als die
744      Betriebssystemvoreinstellung setzen.  Eine solche Anpassung ist nur
745      notwendig, wenn es vom Anbieter des Moduls so spezifiziert wurde oder die
746      Diagnose eines Apache-Absturzes ergeben hat, das die
747      Thread-Stackgr&#246;&#223;e zu klein war.</li>
748
749      <li>Auf Plattformen, wo die voreingestellte Thread-Stackgr&#246;&#223;e
750      f&#252;r die Webserverkonfiguration deutlich gr&#246;&#223;er als
751      notwendig ist, kann eine gr&#246;&#223;ere Anzahl von Threads pro
752      Kindprozess erreicht werden, wenn <code class="directive">ThreadStackSize</code>
753      auf einen Wert kleiner als die Betriebssystemvoreinstellung gesetzt wird.
754      Da es einzelne Anfragen geben kann, die mehr Stack zur Verarbeitung
755      ben&#246;tigen, sollte eine derartige Korrektur ausschlie&#223;lich in
756      Testumgebungen zum Einsatz kommen, auf denen die gesamte
757      Webserververarbeitung ausprobiert werden kann. Eine &#196;nderung der
758      Webserverkonfiguration kann den aktuellen Wert der
759      <code class="directive">ThreadStackSize</code> ung&#252;ltig machen.</li>
760    </ul>
