27<div id="page-content"><div id="preamble"><h1>Override Class Index for .htaccess</h1>
28<div class="toplang">
29<p><span>Available Languages: </span><a href="../en/mod/overrides.html" title="English">&nbsp;en&nbsp;</a> |
30<a href="../fr/mod/overrides.html" hreflang="fr" rel="alternate" title="Français">&nbsp;fr&nbsp;</a></p>
31</div>
32
33    <p>
34      This is an index of the directives that are allowed in .htaccess files for
35      various <code class="directive"><a href="../mod/core.html#allowoverride">AllowOverride</a></code> settings,
36      organized by class. Its intended purpose is to help server administrators
37      verify the privileges they're granting to .htaccess users. For an overview
38      of how .htaccess works, see the
39      <a href="../howto/htaccess.html">.htaccess tutorial</a>.
40    </p>
41
42    <p>
43      To determine the set of directives that your server configuration allows
44      .htaccess users to use:
45    </p>
46
47    <ol>
48      <li>Start with the set of directives in the <code>AllowOverrideList</code>
49      for the directory in question. (By default, this is set to
50      <code>None</code>.)</li>
51      <li>Find the <code>AllowOverride</code> setting for the directory in
52      question. (By default, it is set to <code>None</code>.) There are two
53      special cases:
54        <ol>
55          <li>If your <code>AllowOverride</code> setting is <code>All</code>,
56          add every directive listed on this page to the list.</li>
57          <li>If your <code>AllowOverride</code> setting is <code>None</code>,
58          you're done. Only the directives in the <code>AllowOverrideList</code>
59          (if any) will be allowed.</li>
60        </ol>
61      </li>
62      <li>For each override class listed in <code>AllowOverride</code>, look up
63      the corresponding set of directives below and add them to the list.</li>
64      <li>Finally, add the set of directives that is always allowed in
65      .htaccess (these are listed in the
66      <a href="#override-all">All section</a>, below).</li>
67    </ol>
68
69    <p>
70      Several of the override classes are quite powerful and give .htaccess
71      users a large amount of control over the server. For a stricter approach,
72      set <code>AllowOverride None</code> and use
73      <code class="directive"><a href="../mod/core.html#allowoverridelist">AllowOverrideList</a></code> to specify the
74      exact list of directives that .htaccess users are allowed to use.
75    </p>
