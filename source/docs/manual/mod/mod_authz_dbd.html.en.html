37<h3>Summary</h3>
38
39    <p>This module provides authorization capabilities so that
40       authenticated users can be allowed or denied access to portions
41       of the web site by group membership.  Similar functionality is
42       provided by <code class="module"><a href="../mod/mod_authz_groupfile.html">mod_authz_groupfile</a></code> and
43       <code class="module"><a href="../mod/mod_authz_dbm.html">mod_authz_dbm</a></code>, with the exception that
44       this module queries a SQL database to determine whether a
45       user is a member of a group.</p>
46    <p>This module can also provide database-backed user login/logout
47       capabilities.  These are likely to be of most value when used
48       in conjunction with <code class="module"><a href="../mod/mod_authn_dbd.html">mod_authn_dbd</a></code>.</p>
49    <p>This module relies on <code class="module"><a href="../mod/mod_dbd.html">mod_dbd</a></code> to specify
50       the backend database driver and connection parameters, and
51       manage the database connections.</p>

76<h2><a name="requiredirectives" id="requiredirectives">The Require Directives</a></h2>
77
78    <p>Apache's <code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code>
79    directives are used during the authorization phase to ensure that
80    a user is allowed to access a resource.  mod_authz_dbd extends the
81    authorization types with <code>dbd-group</code>, <code>dbd-login</code> and
82    <code>dbd-logout</code>.</p>
83
84    <p>Since v2.4.8, <a href="../expr.html">expressions</a> are supported
85    within the DBD require directives.</p>
86
87<h3><a name="reqgroup" id="reqgroup">Require dbd-group</a></h3>
88
89    <p>This directive specifies group membership that is required for the
90    user to gain access.</p>
91
92    <pre class="prettyprint lang-config">Require dbd-group team
93AuthzDBDQuery "SELECT user_group FROM authz WHERE user = %s"</pre>
94
95
96
97
98<h3><a name="reqlogin" id="reqlogin">Require dbd-login</a></h3>
99
100    <p>This directive specifies a query to be run indicating the user
101    has logged in.</p>
102
103    <pre class="prettyprint lang-config">Require dbd-login
104AuthzDBDQuery "UPDATE authn SET login = 'true' WHERE user = %s"</pre>
105
106
107
108
109<h3><a name="reqlogout" id="reqlogout">Require dbd-logout</a></h3>
110
111    <p>This directive specifies a query to be run indicating the user
112    has logged out.</p>
113
114    <pre class="prettyprint lang-config">Require dbd-logout
115AuthzDBDQuery "UPDATE authn SET login = 'false' WHERE user = %s"</pre>

122<h2><a name="login" id="login">Database Login</a></h2>
123
124<p>
125In addition to the standard authorization function of checking group
126membership, this module can also provide server-side user session
127management via database-backed login/logout capabilities.
128Specifically, it can update a user's session status in the database
129whenever the user visits designated URLs (subject of course to users
130supplying the necessary credentials).</p>
131<p>This works by defining two special
132<code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code> types:
133<code>Require dbd-login</code> and <code>Require dbd-logout</code>.
134For usage details, see the configuration example below.</p>
135</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>
136<div class="section">
137<h2><a name="client" id="client">Client Login integration</a></h2>
138
139<p>Some administrators may wish to implement client-side session
140management that works in concert with the server-side login/logout
141capabilities offered by this module, for example, by setting or unsetting
142an HTTP cookie or other such token when a user logs in or out.</p>
143<p>To support such integration, <code class="module"><a href="../mod/mod_authz_dbd.html">mod_authz_dbd</a></code> exports an
144optional hook that will be run whenever a user's status is updated in
145the database.  Other session management modules can then use the hook
146to implement functions that start and end client-side sessions.</p>
147</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

149<h2><a name="example" id="example">Configuration example</a></h2>
150
151<pre class="prettyprint lang-config"># mod_dbd configuration
152DBDriver pgsql
153DBDParams "dbname=apacheauth user=apache pass=xxxxxx"
154
155DBDMin  4
156DBDKeep 8
157DBDMax  20
158DBDExptime 300
159
160&lt;Directory "/usr/www/my.site/team-private/"&gt;
161  # mod_authn_core and mod_auth_basic configuration
162  # for mod_authn_dbd
163  AuthType Basic
164  AuthName Team
165  AuthBasicProvider dbd
166
167  # mod_authn_dbd SQL query to authenticate a logged-in user
168  AuthDBDUserPWQuery \
169    "SELECT password FROM authn WHERE user = %s AND login = 'true'"
170
171  # mod_authz_core configuration for mod_authz_dbd
172  Require dbd-group team
173
174  # mod_authz_dbd configuration
175  AuthzDBDQuery "SELECT group FROM authz WHERE user = %s"
176
177  # when a user fails to be authenticated or authorized,
178  # invite them to login; this page should provide a link
179  # to /team-private/login.html
180  ErrorDocument 401 "/login-info.html"
181
182  &lt;Files "login.html"&gt;
183    # don't require user to already be logged in!
184    AuthDBDUserPWQuery "SELECT password FROM authn WHERE user = %s"
185
186    # dbd-login action executes a statement to log user in
187    Require dbd-login
188    AuthzDBDQuery "UPDATE authn SET login = 'true' WHERE user = %s"
189
190    # return user to referring page (if any) after
191    # successful login
192    AuthzDBDLoginToReferer On
193  &lt;/Files&gt;
194
195  &lt;Files "logout.html"&gt;
196    # dbd-logout action executes a statement to log user out
197    Require dbd-logout
198    AuthzDBDQuery "UPDATE authn SET login = 'false' WHERE user = %s"
199  &lt;/Files&gt;
200&lt;/Directory&gt;</pre>

204<div class="directive-section"><h2><a name="AuthzDBDLoginToReferer" id="AuthzDBDLoginToReferer">AuthzDBDLoginToReferer</a> <a name="authzdbdlogintoreferer" id="authzdbdlogintoreferer">Directive</a></h2>
205<table class="directive">
206<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Determines whether to redirect the Client to the Referring
207page on successful login or logout if a <code>Referer</code> request
208header is present</td></tr>
209<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>AuthzDBDLoginToReferer On|Off</code></td></tr>
210<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>AuthzDBDLoginToReferer Off</code></td></tr>
211<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>directory</td></tr>
212<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
213<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authz_dbd</td></tr>
214</table>
215    <p>In conjunction with <code>Require dbd-login</code> or
216    <code>Require dbd-logout</code>, this provides the option to
217    redirect the client back to the Referring page (the URL in
218    the <code>Referer</code> HTTP request header, if present).
219    When there is no <code>Referer</code> header,
220    <code>AuthzDBDLoginToReferer On</code> will be ignored.</p>

224<div class="directive-section"><h2><a name="AuthzDBDQuery" id="AuthzDBDQuery">AuthzDBDQuery</a> <a name="authzdbdquery" id="authzdbdquery">Directive</a></h2>
225<table class="directive">
226<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Specify the SQL Query for the required operation</td></tr>
227<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>AuthzDBDQuery <var>query</var></code></td></tr>
228<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>directory</td></tr>
229<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
230<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authz_dbd</td></tr>
231</table>
232    <p>The <code class="directive">AuthzDBDQuery</code> specifies an SQL
233    query to run.  The purpose of the query depends on the
234    <code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code> directive in
235    effect.</p>
236    <ul>
237    <li>When used with a <code>Require dbd-group</code> directive,
238    it specifies a query to look up groups for the current user.  This is
239    the standard functionality of other authorization modules such as
240    <code class="module"><a href="../mod/mod_authz_groupfile.html">mod_authz_groupfile</a></code> and <code class="module"><a href="../mod/mod_authz_dbm.html">mod_authz_dbm</a></code>.
241    The first column value of each row returned by the query statement
242    should be a string containing a group name.  Zero, one, or more rows
243    may be returned.
244    <pre class="prettyprint lang-config">Require dbd-group
245AuthzDBDQuery "SELECT group FROM groups WHERE user = %s"</pre>
246
247    </li>
248    <li>When used with a <code>Require dbd-login</code> or
249    <code>Require dbd-logout</code> directive, it will never deny access,
250    but will instead execute a SQL statement designed to log the user
251    in or out.  The user must already be authenticated with
252    <code class="module"><a href="../mod/mod_authn_dbd.html">mod_authn_dbd</a></code>.
253    <pre class="prettyprint lang-config">Require dbd-login
254AuthzDBDQuery "UPDATE authn SET login = 'true' WHERE user = %s"</pre>
255
256    </li>
257    </ul>
258    <p>In all cases, the user's ID will be passed as a single string
259    parameter when the SQL query is executed.  It may be referenced within
260    the query statement using a <code>%s</code> format specifier.</p>
261

264<div class="directive-section"><h2><a name="AuthzDBDRedirectQuery" id="AuthzDBDRedirectQuery">AuthzDBDRedirectQuery</a> <a name="authzdbdredirectquery" id="authzdbdredirectquery">Directive</a></h2>
265<table class="directive">
266<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Specify a query to look up a login page for the user</td></tr>
267<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>AuthzDBDRedirectQuery <var>query</var></code></td></tr>
268<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>directory</td></tr>
269<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
270<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authz_dbd</td></tr>
271</table>
272    <p>Specifies an optional SQL query to use after successful login
273    (or logout) to redirect the user to a URL, which may be
274    specific to the user.  The user's ID will be passed as a single string
275    parameter when the SQL query is executed.  It may be referenced within
276    the query statement using a <code>%s</code> format specifier.</p>
277    <pre class="prettyprint lang-config">AuthzDBDRedirectQuery "SELECT userpage FROM userpages WHERE user = %s"</pre>
278
279    <p>The first column value of the first row returned by the query
280    statement should be a string containing a URL to which to redirect
281    the client.  Subsequent rows will be ignored.  If no rows are returned,
282    the client will not be redirected.</p>
283    <p>Note that <code class="directive">AuthzDBDLoginToReferer</code> takes
284    precedence if both are set.</p>
