41<h3>Sommaire</h3>
42
43    <p>Ce module permet aux frontaux d'authentification comme
44    <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code> et <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code>
45    d'authentifier les utilisateurs en les recherchant dans des fichiers
46    de mots de passe au format texte. <code class="module"><a href="../mod/mod_authn_dbm.html">mod_authn_dbm</a></code>
47    fournit une fonctionnalité similaire.</p>
48
49    <p>Lorsqu'on utilise <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> ou
50    <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>, ce module peut être invoqué en
51    affectant la valeur <code>file</code> à la directive <code class="directive"><a href="../mod/mod_auth_basic.html#authbasicprovider">AuthBasicProvider</a></code> ou <code class="directive"><a href="../mod/mod_auth_digest.html#authdigestprovider">AuthDigestProvider</a></code>.</p>

72<div class="directive-section"><h2><a name="authuserfile" id="authuserfile">Directive</a> <a name="AuthUserFile" id="AuthUserFile">AuthUserFile</a></h2>

74<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Définit le nom d'un fichier texte pour l'authentification
75contenant la liste des utilisateurs et de leurs mots de
76passe</td></tr>
77<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>AuthUserFile <var>chemin-fichier</var></code></td></tr>
78<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>répertoire, .htaccess</td></tr>
79<tr><th><a href="directive-dict.html#Override">Surcharges autorisées:</a></th><td>AuthConfig</td></tr>
80<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
81<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authn_file</td></tr>
82</table>
83    <p>La directive <code class="directive">AuthUserFile</code> permet de
84    définir le nom d'un fichier texte pour l'authentification contenant
85    la liste des utilisateurs et de leurs mots de passe.
86    <var>chemin-fichier</var> est le chemin vers le fichier
87    des utilisateurs. S'il n'est pas absolu, il est considéré comme
88    relatif au répertoire défini par la directive <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>.</p>
89
90    <p>Chaque ligne du fichier des utilisateurs se compose du nom de
91    l'utilisateur, du caractère ':' et du mot de passe chiffré. Si le
92    même identifiant utilisateur est référencé plusieurs fois,
93    <code class="module"><a href="../mod/mod_authn_file.html">mod_authn_file</a></code> utilisera la première occurrence pour
94    vérifier le mot de passe.</p>
95
96     <p>Le format du mot de passe chiffré dépend du frontal
97     d'authentification utilisé (par exemple
98     <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> ou
99     <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>). Voir la documentation sur les
100     <a href="../misc/password_encryptions.html">Formats de mots de
101     passe</a> pour plus de détails.</p>
102
103    <p>Pour <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code>, utilisez le programme
104    <code class="program"><a href="../programs/htpasswd.html">htpasswd</a></code> fourni avec la distribution binaire,
105    mais que vous trouverez aussi dans le répertoire
106    <code>src/support</code> de l'arborescence des sources. Voir sa <a href="../programs/htpasswd.html">page de manuel</a> pour plus de
107    détails. En bref :</p>
108
109    <p>On crée un fichier de mots de passe <code>nom-fichier</code> avec
110    <code>nom-utilisateur</code> comme identifiant initial. Le mot de
111    passe correspondant sera alors demandé :</p>
112
113    <div class="example"><p><code>
114      htpasswd -c nom-fichier nom-utilisateur
115    </code></p></div>
116
117    <p>Pour ajouter ou modifier <code>nom-utilisateur2</code> dans le
118    fichier de mots de passe <code>nom-fichier</code> :</p>
119
120    <div class="example"><p><code>
121      htpasswd nom-fichier nom-utilisateur2
122    </code></p></div>
123
124    <p>Noter qu'une recherche dans de grands fichiers texte peut être
125    très longue ; dans ce cas, il vaut mieux utiliser les fichiers DBM
126    avec la directive <code class="directive"><a href="../mod/mod_authn_dbm.html#authdbmuserfile">AuthDBMUserFile</a></code>.</p>
127
128    <p>Pour <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>, vous devez utiliser
129    le programme <code class="program"><a href="../programs/htdigest.html">htdigest</a></code>.
130    Notez que vous ne pouvez pas mélanger des données utilisateur pour
131    l'Authentification HTTP à base de condensé et des données pour
132    l'Authentification de Base dans le même fichier.</p>
133
134    <div class="warning"><h3>Sécurité</h3>
135      <p>Assurez-vous que le fichier <code class="directive">AuthUserFile</code>
136      soit bien stocké en dehors de l'arborescence des documents du
137      serveur web. Ne placez <strong>pas</strong> ce fichier dans le
138      répertoire qu'il protège. Dans le cas contraire, les clients
139      seraient en mesure de télécharger le fichier des mots de passe.</p>
140    </div>
