44<h3>概要</h3>
45
46
47    <p>リクエストの綴りが間違っていたり、
48    大文字小文字が違っていたりするために、Apache のコアサーバが
49    ドキュメントへのリクエストへの応答を正しく提供できないことがあります。
50    このモジュールは、他のすべてのモジュールがあきらめた後であったとしても、
51    リクエストに合うドキュメントを見つけようとすることによりこの問題の
52    解決を試みます。このモジュールはリクエストされたディレクトリにある
53    それぞれのドキュメントの名前と、リクエストされたドキュメントの名前とを
54    <strong>大文字小文字の区別を無視し</strong>、<strong>一文字までの
55    綴りの間違い</strong> (文字の挿入/省略/隣合う文字の置換、間違った文字)
56    を許可して比較することにより、目的を達成しようとします。
57    この方法でリクエストに合うドキュメントの一覧が作成されます。</p>
58
59    <p>ディレクトリをスキャンした後に、</p>
60
61    <ul>
62      <li>適切なドキュメントが見つからなかった場合、
63      Apache はいつもと同じように処理をし、
64      「ドキュメントが見つからない」というエラーを返します。</li>
65
66      <li>リクエストに「ほとんど」合うドキュメントが一つだけ見つかった場合、
67       それがリダイレクト応答として返されます。</li>
68
69      <li>よく似たドキュメントが複数見つかった場合、
70      そのリストがクライアントに返され、
71      クライアントが正しい候補を選択できるようにします。</li>
72    </ul>

116<div class="directive-section"><h2><a name="CheckSpelling" id="CheckSpelling">CheckSpelling</a> <a name="checkspelling" id="checkspelling">ディレクティブ</a></h2>
117<table class="directive">
118<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>spelling モジュールを使用するようにする</td></tr>
119<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>CheckSpelling on|off</code></td></tr>
120<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>CheckSpelling Off</code></td></tr>
121<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト, ディレクトリ, .htaccess</td></tr>
122<tr><th><a href="directive-dict.html#Override">上書き:</a></th><td>Options</td></tr>
123<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
124<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_speling</td></tr>
125<tr><th><a href="directive-dict.html#Compatibility">互換性:</a></th><td>CheckSpelling は Apache 1.1 では別配布のモジュールで、
126大文字小文字の間違いのみの機能でした。Apache 1.3 で Apache の配布に
127含まれるようになりました。Apache 1.3.2 より前では <code>CheckSpelling</code>
128ディレクティブは「サーバ」と「バーチャルホスト」コンテキストでのみ
129使用可能でした</td></tr>
130</table>
131    <p>このディレクティブは綴り用のモジュールを使用するかどうかを
132    決めます。使用時には、以下のことを覚えておいてください</p>
133
134    <ul>
135      <li>同時にたくさんの綴りの訂正を行なわなければならないときは、
136      そのために行なわれるディレクトリのスキャンがサーバの性能に
137      影響を与えます。</li>
138
139      <li>ドキュメントの中に綴りの「訂正」により
140      意図せず合ってしまうような重要なファイルがないようにしてください。
141      </li>
142
143      <li>モジュールはユーザ名の綴りの間違い
144      (<code>http://my.host/~apahce/</code> のように)
145      を訂正することはできません。
146      訂正できるのはファイル名とディレクトリ名だけです。</li>
147
148      <li>綴りの訂正は存在するファイルに厳密に適用されますので、
149      <code>&lt;Location /status&gt;</code>
150      はネゴシエーションの結果のファイル "<code>/stats.html</code>"
151      として間違って扱われるかもしれません。</li>
152    </ul>
153
154    
155    <p><a href="mod_dav.html">DAV</a> が有効なディレクトリでは
156    <code class="module"><a href="../mod/mod_speling.html">mod_speling</a></code> は有効にしないでください。
157    新しく作成したリソース名を既に存在するファイル名に「修正」しようとする、
158    例えば、新規ドキュメント <code>doc43.html</code> が既に存在する
159    <code>doc34.html</code> にリダイレクトされて、
160    期待とは違う挙動になるからです。</p>
