43<h3>概要</h3>
44
45    <p>本モジュールは <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code> や
46    <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> といった認証フロントエンドに対して、
47    <dfn>dbm</dfn> パスワードファイル内からのユーザ検索による
48    ユーザ認証機能を提供します。似たような機能は <code class="module"><a href="../mod/mod_authn_file.html">mod_authn_file</a></code>
49    でも提供されています。</p>
50
51    <p><code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> や <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>
52    を使用する際には、このモジュールは
53    <code class="directive"><a href="../mod/mod_auth_basic.html#authbasicprovider">AuthBasicProvider</a></code> や
54    <code class="directive"><a href="../mod/mod_auth_digest.html#authdigestprivider">AuthDigestPrivider</a></code>
55    で <code>dbm</code> と指定することで起動されます。</p>

77<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>パスワードを保存するために必要なデータベースファイルの種類を

86    <p>パスワードを保存するために使用するデータベースファイルの種類を
87    設定します。デフォルトのデータベースの種類はコンパイル時に決まります。
88    他の種類のデータベースが使用可能かどうかも <a href="../install.html#dbm">コンパイル時の設定</a>に依存します。</p>
89
90    <p>パスワードファイルを作成するのに使用するプログラムが同じ種類のデータベースを
91    使用するように設定することは非常に重要です。</p>

95<div class="directive-section"><h2><a name="AuthDBMUserFile" id="AuthDBMUserFile">AuthDBMUserFile</a> <a name="authdbmuserfile" id="authdbmuserfile">ディレクティブ</a></h2>
96<table class="directive">
97<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>認証用のユーザとパスワードのリストを保持している
98データベースファイル名を設定する</td></tr>
99<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>AuthDBMUserFile <var>file-path</var></code></td></tr>
100<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>ディレクトリ, .htaccess</td></tr>
101<tr><th><a href="directive-dict.html#Override">上書き:</a></th><td>AuthConfig</td></tr>
102<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
103<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_authn_dbm</td></tr>
104</table>
105    <p><code class="directive">AuthDBMUserFile</code> ディレクティブは
106    認証用のユーザとパスワードのリストを保持している DBM ファイルの
107    名前を設定します。<var>File-path</var> はユーザファイルへの
108    絶対パスです。</p>
109
110    <p>ユーザファイルのキーはユーザ名です。ユーザに対して返される値は
111    暗号化されたパスワードで、その後に、コロンに続いて任意のデータが
112    続いていることもあります。コロンとその後のデータはサーバは
113    無視します。</p>
114
115    <div class="warning"><h3>セキュリティ</h3>
116      <p><code class="directive">AuthDBMUserFile </code> は、
117      ウェブサーバのドキュメントツリーの外側に保管するようにしてください。
118      保護しようとしているディレクトリ以下には
119      <strong>置かないで下さい</strong>。
120      そうしないとクライアントが <code class="directive">AuthUserFile</code> を
121      ダウンロードできてしまいます。</p>
122    </div>
123
124    <p>重要な互換性に関する注意: apache module の <code>dbmopen</code> の実装は
125    文字列が NULL で終わっていることに依存するのではなく、DBM データストラクチャ
126    のハッシュ値の文字列の長さを読み取ります。Netscape ウェブサーバなど、
127    アプリケーションの中には文字列が NULL で終わっていることに依存している
128    ものがあります。ですから、異なるアプリケーション間での DBM ファイルの
129    使用に問題がある場合は、これが原因になっている可能性があります。</p>
130
131    <p>Apache には <code class="program"><a href="../programs/dbmmanage.html">dbmmanage</a></code> という
132    perl スクリプトが含まれています。このプログラムを使ってこの
133    モジュールが使用する DBM フォーマットのパスワードファイルを作成したり
134    更新したりすることができます。</p>
