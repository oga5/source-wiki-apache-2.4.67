40<h3>Summary</h3>
41
42
43    <p>Requests to documents sometimes cannot be served by the core
44    apache server because the request was misspelled or
45    miscapitalized. This module addresses this problem by trying to
46    find a matching document, even after all other modules gave up.
47    It does its work by comparing each document name in the
48    requested directory against the requested document name
49    <strong>without regard to case</strong>, and allowing
50    <strong>up to one misspelling</strong> (character insertion /
51    omission / transposition or wrong character). A list is built
52    with all document names which were matched using this
53    strategy. <strong>Erroneous extension</strong> can also be fixed
54    by this module.</p>
55
56    <p>If, after scanning the directory,</p>
57
58    <ul>
59      <li>no matching document was found, Apache will proceed as
60      usual and return an error (<code>404 - document not found</code>).</li>
61
62      <li>only one document is found that "almost" matches the
63      request, then it is returned in the form of a redirection
64      response (<code>301 - Moved Permanently</code>).</li>
65
66      <li>more than one document with a close match was found, then
67      the list of the matches is returned to the client, and the
68      client can select the correct candidate (<code>300 - Multiple
69      Choices</code>).</li>
70    </ul>

84<div class="directive-section"><h2><a name="CheckBasenameMatch" id="CheckBasenameMatch">CheckBasenameMatch</a> <a name="checkbasenamematch" id="checkbasenamematch">Directive</a></h2>

87<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>CheckBasenameMatch on|off</code></td></tr>

95    <p>When set, this directive extends the action of the spelling correction
96    to the file name extension. For example a file <code>foo.gif</code> will
97    match a request for <code>foo</code> or <code>foo.jpg</code>. This can be
98    particularly useful in conjunction with
99    <a href="../content-negotiation.html">MultiViews</a>.</p>

115    except when <code class="directive"><a href="#checkbasenamematch">CheckBasenameMatch</a></code> is also set.</p>

119<div class="directive-section"><h2><a name="CheckSpelling" id="CheckSpelling">CheckSpelling</a> <a name="checkspelling" id="checkspelling">Directive</a></h2>
120<table class="directive">
121<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Enables the spelling
122module</td></tr>
123<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>CheckSpelling on|off</code></td></tr>
124<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>CheckSpelling Off</code></td></tr>
125<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
126<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>Options</td></tr>
127<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
128<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_speling</td></tr>
129</table>
130    <p>This directive enables or disables the spelling module. When
131    enabled, keep in mind that</p>
132
133    <ul>
134      <li>the directory scan which is necessary for the spelling
135      correction will have an impact on the server's performance
136      when many spelling corrections have to be performed at the
137      same time.</li>
138
139      <li>the document trees should not contain sensitive files
140      which could be matched inadvertently by a spelling
141      "correction".</li>
142
143      <li>the module is unable to correct misspelled user names (as
144      in <code>http://my.host/~apahce/</code>), just file names or
145      directory names.</li>
146
147      <li>spelling corrections apply strictly to existing files, so
148      a request for the <code>&lt;Location /status&gt;</code> may
149      get incorrectly treated as the negotiated file
150      "<code>/stats.html</code>".</li>
151    </ul>
152
153    
154    <p><code class="module"><a href="../mod/mod_speling.html">mod_speling</a></code> should not be enabled in <a href="mod_dav.html">DAV</a>
155    enabled directories, because it will try to "spell fix" newly created
156    resource names against existing filenames, e.g., when trying to upload
157    a new document <code>doc43.html</code> it might redirect to an existing
158    document <code>doc34.html</code>, which is not what was intended.
159    </p>
