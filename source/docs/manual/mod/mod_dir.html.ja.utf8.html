44<h3>概要</h3>
45
46    <p>ディレクトリインデックスは、次の二つのうちどちらかが利用されます:</p>
47
48    <ul>
49      <li>一つ目は、ユーザが作成したファイルを用いるもので、通常
50      <code>index.html</code> というファイル名を使います。このファイル名は、
51      <code class="directive"><a href="#directoryindex">DirectoryIndex</a></code> ディレクティブで
52      指定することができます。この機能は <code class="module"><a href="../mod/mod_dir.html">mod_dir</a></code>
53      モジュールで提供されます。</li>
54
55      <li>もう一つの方法は、
56      サーバによって自動的に生成されるディレクトリリストを用いる場合です。
57      この機能は、<code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code>
58      モジュールにより提供されます。</li>
59    </ul>
60
61    <p>自動的なインデックス生成機能を削除 (もしくは交換)
62    できるように、この二つの機能は分離されています。</p>
63
64    <p>なお <code>http://servername/foo/dirname</code> という URL
65    へのリクエストがあった際に、<code>dirname</code>
66    というディレクトリがあれば、「最後にスラッシュをつけた形」の URL
67    へのリダイレクトを送出します。
68    ディレクトリへのアクセスはスラッシュで終わっている必要があり、
69    <code>mod_dir</code> は、<code>http://servername/foo/dirname/</code>
70    へのリダイレクトを送出することになります。</p>

100<div class="directive-section"><h2><a name="DirectoryIndex" id="DirectoryIndex">DirectoryIndex</a> <a name="directoryindex" id="directoryindex">ディレクティブ</a></h2>
101<table class="directive">
102<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>クライアントがディレクトリをリクエストしたときに調べる
103リソースのリスト</td></tr>
104<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>DirectoryIndex
105    <var>local-url</var> [<var>local-url</var>] ...</code></td></tr>
106<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>DirectoryIndex index.html</code></td></tr>
107<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト, ディレクトリ, .htaccess</td></tr>
108<tr><th><a href="directive-dict.html#Override">上書き:</a></th><td>Indexes</td></tr>
109<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Base</td></tr>
110<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_dir</td></tr>
111</table>
112    <p>
113    クライアントが、ディレクトリ名の最後に「/」
114    を指定してディレクトリインデックスを要求する場合に探すリソースのリストを
115    <code class="directive">DirectoryIndex</code> ディレクティブで設定します。
116    <var>Local-url</var>
117    は、リクエストされたディレクトリに対応する、サーバ上のドキュメントの
118    (% エンコードされた) URL で、普通はディレクトリ中のファイルの名前です。
119    複数の URL が設定された場合には、最初に見つかったものを返します。
120    それらが見つからず、<code>Indexes</code>
121    オプションがセットされている場合、ディレクトリのリストを生成します。
122    </p>
123
124    <div class="example"><h3>例</h3><p><code>
125      DirectoryIndex index.html
126    </code></p></div>
127
128    <p><code>http://myserver/docs/</code> へのアクセスがあり、
129    <code>http://myserver/docs/index.html</code>
130    が存在すれば、この URL が返されます。
131    もし存在しなければ、ディレクトリのリストが返されます。</p>
132
133    <p>注: ドキュメントが同じディレクトリ内に存在するは必要ありません。
134    </p>
135
136    <div class="example"><p><code>
137      DirectoryIndex index.html index.txt  /cgi-bin/index.pl
138    </code></p></div>
139
140    <p>とした場合、<code>index.html</code> と <code>index.txt</code>
141    のどちらもディレクトリ内に存在しない場合、CGI スクリプト
142    <code>/cgi-bin/index.pl</code> が実行されます。</p>
143
144</div>

163<div class="directive-section"><h2><a name="DirectorySlash" id="DirectorySlash">DirectorySlash</a> <a name="directoryslash" id="directoryslash">ディレクティブ</a></h2>
164<table class="directive">
165<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>パス末尾のスラッシュでリダイレクトするかどうかのオンオフをトグルさせる</td></tr>
166<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>DirectorySlash On|Off</code></td></tr>
167<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>DirectorySlash On</code></td></tr>
168<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト, ディレクトリ, .htaccess</td></tr>
169<tr><th><a href="directive-dict.html#Override">上書き:</a></th><td>Indexes</td></tr>
170<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Base</td></tr>
171<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_dir</td></tr>
172<tr><th><a href="directive-dict.html#Compatibility">互換性:</a></th><td>2.0.51 以降</td></tr>
173</table>
174    <p>要求のあった URL がディレクトリを指すかどうかを、
175    <code class="module"><a href="../mod/mod_dir.html">mod_dir</a></code> が調整するべきかどうかを
176    <code class="directive">DirectorySlash</code>
177    ディレクティブで設定します。</p>
178
179    <p>典型的には、ユーザが末尾のスラッシュ無しでリソースへのリクエストを発行し、
180    そして、そのリソースがディレクトリを指していた場合、<code class="module"><a href="../mod/mod_dir.html">mod_dir</a></code>
181    は、末尾にスラッシュを<em>付加</em>した上で同じリソースにリダイレクトさせます。
182    この挙動には幾つか理由があります:</p>
183
184    <ul>
185    <li>ユーザは、最終的にはリソースの別名 URL をリクエストすることになる。</li>
186    <li><code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code> が期待通りに動く。<code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code> 
187    の生成するリンクはパスを出力しませんので、スラッシュがない場合は間違ったパスを
188    指してしまうことになります。</li>
189    <li><code class="directive"><a href="#directoryindex">DirectoryIndex</a></code> は、
190    末尾にスラッシュがついているリクエストについて<em>のみ</em>評価される。</li>
191    <li>HTML ページの相対 URL 参照が正しく動作する。</li>
192    </ul>
193
194    <p>とはいえ、もしこういった効果を望まない、かつ、
195    上記のような理由が当てはまらない場合は、リダイレクトを次のようにしてオフにできます:</p>
196
197    <div class="example"><p><code>
198        # see security warning below!<br />
199        &lt;Location /some/path&gt;<br />
200        <span class="indent">
201            DirectorySlash Off<br />
202            SetHandler some-handler<br />
203        </span>
204        &lt;/Location&gt;
205    </code></p></div>
206
207    <div class="warning"><h3>セキュリティ警告</h3>
208    <p>末尾のスラッシュでのリダイレクトをオフにすると、結果的に情報漏洩を
209    招くことになるかもしれません。
210    <code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code> が有効 (<code>Options +Indexes</code>) で、
211    <code class="directive"><a href="#directoryindex">DirectoryIndex</a></code> が有効なリソース (例えば
212    <code>index.html</code>) を指していて、また、要求のあった URL に特別な
213    ハンドラが設定されていない場合を考えてみてください。
214    この場合末尾にスラッシュのついているリクエストに対しては <code>index.html</code>
215    ファイルが返されます。<strong>しかしスラッシュのないリクエストに対しては、
216    ディレクトリの内容一覧を返してしまいます。</strong></p>
217    </div>
