39<h3>Summary</h3>
40
41    <p>This module provides enhanced internationalisation support for
42    markup-aware filter modules such as <code class="module"><a href="../mod/mod_proxy_html.html">mod_proxy_html</a></code>.
43    It can automatically detect the encoding of input data and ensure
44    they are correctly processed by the <a href="http://xmlsoft.org/">libxml2</a> parser, including converting to Unicode (UTF-8) where
45    necessary.  It can also convert data to an encoding of choice
46    after markup processing, and will ensure the correct <var>charset</var>
47    value is set in the HTTP <var>Content-Type</var> header.</p>
48</div>

67<h2><a name="usage" id="usage">Usage</a></h2>
68    <p>There are two usage scenarios: with modules programmed to work
69    with mod_xml2enc, and with those that are not aware of it:</p>
70    <dl>
71    <dt>Filter modules enabled for mod_xml2enc</dt><dd>
72    <p>Modules such as <code class="module"><a href="../mod/mod_proxy_html.html">mod_proxy_html</a></code> version 3.1
73    and up use the <code>xml2enc_charset</code> optional function to retrieve
74    the charset argument to pass to the libxml2 parser, and may use the
75    <code>xml2enc_filter</code> optional function to postprocess to another
76    encoding.  Using mod_xml2enc with an enabled module, no configuration
77    is necessary: the other module will configure mod_xml2enc for you
78    (though you may still want to customize it using the configuration
79    directives below).</p>
80    </dd>
81    <dt>Non-enabled modules</dt><dd>
82    <p>To use it with a libxml2-based module that isn't explicitly enabled for
83    mod_xml2enc, you will have to configure the filter chain yourself.  So to
84    use it with a filter <strong>foo</strong> provided by a module
85    <strong>mod_foo</strong> to improve the latter's i18n support with HTML and
86    XML, you could use</p>
87    <pre><code>
88    FilterProvider iconv    xml2enc Content-Type $text/html
89    FilterProvider iconv    xml2enc Content-Type $xml
90    FilterProvider markup   foo Content-Type $text/html
91    FilterProvider markup   foo Content-Type $xml
92    FilterChain     iconv markup
93    </code></pre>
94    <p><strong>mod_foo</strong> will now support any character set supported by either
95    (or both) of libxml2 or apr_xlate/iconv.</p>
96    </dd></dl>

99<h2><a name="api" id="api">Programming API</a></h2>
100    <p>Programmers writing libxml2-based filter modules are encouraged to
101    enable them for mod_xml2enc, to provide strong i18n support for your
102    users without reinventing the wheel.  The programming API is exposed in
103    <var>mod_xml2enc.h</var>, and a usage example is
104    <code class="module"><a href="../mod/mod_proxy_html.html">mod_proxy_html</a></code>.</p>

107<h2><a name="sniffing" id="sniffing">Detecting an Encoding</a></h2>
108    <p>Unlike <code class="module"><a href="../mod/mod_charset_lite.html">mod_charset_lite</a></code>, mod_xml2enc is designed
109    to work with data whose encoding cannot be known in advance and thus
110    configured.  It therefore uses 'sniffing' techniques to detect the
111    encoding of HTTP data as follows:</p>
112    <ol>
113        <li>If the HTTP <var>Content-Type</var> header includes a
114        <var>charset</var> parameter, that is used.</li>
115        <li>If the data start with an XML Byte Order Mark (BOM) or an
116        XML encoding declaration, that is used.</li>
117        <li>If an encoding is declared in an HTML <code>&lt;META&gt;</code>
118        element, that is used.</li>
119        <li>If none of the above match, the default value set by
120        <code class="directive">xml2EncDefault</code> is used.</li>
121    </ol>
122    <p>The rules are applied in order.  As soon as a match is found,
123    it is used and detection is stopped.</p>
124</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

126<h2><a name="output" id="output">Output Encoding</a></h2>
127<p><a href="http://xmlsoft.org/">libxml2</a> always uses UTF-8 (Unicode)
128internally, and libxml2-based filter modules will output that by default.
129mod_xml2enc can change the output encoding through the API, but there
130is currently no way to configure that directly.</p>
131<p>Changing the output encoding should (in theory, at least) never be
132necessary, and is not recommended due to the extra processing load on
133the server of an unnecessary conversion.</p>
134</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

136<h2><a name="alias" id="alias">Unsupported Encodings</a></h2>
137<p>If you are working with encodings that are not supported by any of
138the conversion methods available on your platform, you can still alias
139them to a supported encoding using <code class="directive">xml2EncAlias</code>.</p>
140</div>

142<div class="directive-section"><h2><a name="xml2EncAlias" id="xml2EncAlias">xml2EncAlias</a> <a name="xml2encalias" id="xml2encalias">Directive</a></h2>
143<table class="directive">
144<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Recognise Aliases for encoding values</td></tr>
145<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>xml2EncAlias <var>charset alias [alias ...]</var></code></td></tr>
146<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config</td></tr>
147<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
148<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_xml2enc</td></tr>
149</table>
150    <p>This server-wide directive aliases one or more encoding to another
151    encoding.  This enables encodings not recognized by libxml2 to be handled
152    internally by libxml2's encoding support using the translation table for
153    a recognized encoding.  This serves two purposes: to support character sets
154    (or names) not recognized either by libxml2 or iconv, and to skip
155    conversion for an encoding where it is known to be unnecessary.</p>
