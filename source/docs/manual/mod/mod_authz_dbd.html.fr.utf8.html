39<h3>Sommaire</h3>
40
41    <p>Ce module fournit des fonctionnalités d'autorisation permettant
42    d'accorder ou de refuser aux utilisateurs authentifiés l'accès à
43    certaines zones du site web en fonction de leur appartenance à tel
44    ou tel groupe. Les modules <code class="module"><a href="../mod/mod_authz_groupfile.html">mod_authz_groupfile</a></code> et
45    <code class="module"><a href="../mod/mod_authz_dbm.html">mod_authz_dbm</a></code> fournissent une fonctionnalité
46    similaire, mais ici le module interroge une base de données SQL pour
47    déterminer si un utilisateur appartient ou non à tel ou tel groupe.</p>
48    <p>Ce module propose également des fonctionnalités de connexion
49    utilisateur s'appuyant sur une base de données, ce qui peut se révéler
50    particulièrement utile lorsque le module est utilisé conjointement avec
51    <code class="module"><a href="../mod/mod_authn_dbd.html">mod_authn_dbd</a></code>.</p>
52    <p>Ce module s'appuie sur <code class="module"><a href="../mod/mod_dbd.html">mod_dbd</a></code> pour spécifier le
53    pilote de la base de données sous-jacente et les paramètres de
54    connexion, et gérer les connexions à la base de données.</p>

79<h2><a name="requiredirectives" id="requiredirectives">Les directives Require</a></h2>
80
81    <p>Les directives <code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code> d'Apache permettent,
82    au cours de la phase d'autorisation, de s'assurer qu'un utilisateur
83    est bien autorisé à accéder à une ressource. mod_authz_dbd ajoute
84    les types d'autorisation <code>dbd-group</code>,
85    <code>dbd-login</code> et <code>dbd-logout</code>.</p>
86
87    <p>A partir de la version 2.4.8, les directives require DBD
88    supportent les <a href="../expr.html">expressions</a>.</p>
89
90<h3><a name="reqgroup" id="reqgroup">Require dbd-group</a></h3>
91
92    <p>Cette directive permet de spécifier à quel groupe un utilisateur
93    doit appartenir pour obtenir l'autorisation d'accès.</p>
94
95    <pre class="prettyprint lang-config">Require dbd-group team
96AuthzDBDQuery "SELECT user_group FROM authz WHERE user = %s"</pre>
97
98
99
100
101<h3><a name="reqlogin" id="reqlogin">Require dbd-login</a></h3>
102
103    <p>Cette directive permet de spécifier une requête à exécuter pour
104    indiquer que l'utilisateur s'est authentifié.</p>
105
106    <pre class="prettyprint lang-config">Require dbd-login
107AuthzDBDQuery "UPDATE authn SET login = 'true' WHERE user = %s"</pre>
108
109
110
111
112<h3><a name="reqlogout" id="reqlogout">Require dbd-logout</a></h3>
113
114    <p>Cette directive permet de spécifier une requête à exécuter pour
115    indiquer que l'utilisateur s'est déconnecté.</p>
116
117    <pre class="prettyprint lang-config">Require dbd-logout
118AuthzDBDQuery "UPDATE authn SET login = 'false' WHERE user = %s"</pre>

125<h2><a name="login" id="login">Reconnaissance d'identité s'appuyant sur une base de données</a></h2>
126
127<p>
128Outre sa fonction d'autorisation standard consistant à vérifier
129l'appartenance à des groupes, ce module permet aussi de gérer des
130sessions utilisateur côté serveur grâce à sa fonctionnalité de connexion utilisateur
131en s'appuyant sur une base de données. En particulier, il peut mettre à
132jour le statut de session de l'utilisateur dans la base de données
133chaque fois que celui-ci visite certaines URLs (sous réserve bien
134entendu que l'utilisateur fournissent les informations de connexion
135nécessaires).</p>
136<p>Pour cela, il faut definir deux directives <code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code> spéciales : <code>Require
137dbd-login</code> et <code>Require dbd-logout</code>. Pour les détails de
138leur utilisation, voir l'exemple de configuration ci-dessous.</p>
139</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>
140<div class="section">
141<h2><a name="client" id="client">Reconnaissance d'identité côté client</a></h2>
142
143<p>Certains administrateurs peuvent vouloir implémenter une gestion de
144session côté client fonctionnant de concert avec les fonctionnalités de
145connexion/déconnexion des utilisateurs côté serveur offertes par ce module, en
146définissant ou en annulant par exemple un cookie HTTP ou un jeton
147similaire lorsqu'un utilisateur se connecte ou se déconnecte.</p>
148
149<p>Pour supporter une telle intégration, <code class="module"><a href="../mod/mod_authz_dbd.html">mod_authz_dbd</a></code> exporte
150un programme à déclenchement optionnel (hook) qui sera lancé chaque fois
151que le statut d'un utilisateur sera mis à jour dans la base de données.
152D'autres modules de gestion de session pourront alors utiliser ce
153programme pour implémenter des fonctions permettant d'ouvrir et de
154fermer des sessions côté client.</p>
155</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

157<h2><a name="example" id="example">Exemple de configuration</a></h2>
158
159<pre class="prettyprint lang-config"># configuration de mod_dbd
160DBDriver pgsql
161DBDParams "dbname=apacheauth user=apache pass=xxxxxx"
162
163DBDMin  4
164DBDKeep 8
165DBDMax  20
166DBDExptime 300
167
168&lt;Directory "/usr/www/mon.site/team-private/"&gt;
169  # configuration de mod_authn_core et mod_auth_basic
170  # pour mod_authn_dbd
171  AuthType Basic
172  AuthName Team
173  AuthBasicProvider dbd
174
175  # requête SQL de mod_authn_dbd pour authentifier un utilisateur qui se
176  # connecte
177  AuthDBDUserPWQuery \
178    "SELECT password FROM authn WHERE user = %s AND login = 'true'"
179
180  # configuration de mod_authz_core pour mod_authz_dbd
181  Require dbd-group team
182
183  # configuration de mod_authz_dbd
184  AuthzDBDQuery "SELECT group FROM authz WHERE user = %s"
185
186  # lorsqu'un utilisateur échoue dans sa tentative d'authentification ou
187  # d'autorisation, on l'invite à se connecter ; cette page doit
188  # contenir un lien vers /team-private/login.html
189  ErrorDocument 401 "/login-info.html"
190
191  &lt;Files "login.html"&gt;
192    # il n'est pas nécessaire que l'utilisateur soit déjà connecté !
193    AuthDBDUserPWQuery "SELECT password FROM authn WHERE user = %s"
194
195    # le processus de connexion dbd exécute une requête pour enregistrer
196    # la connexion de l'utilisateur
197    Require dbd-login
198    AuthzDBDQuery "UPDATE authn SET login = 'true' WHERE user = %s"
199
200    # redirige l'utilisateur vers la page d'origine (si elle existe)
201    # après une connexion réussie
202    AuthzDBDLoginToReferer On
203  &lt;/Files&gt;
204
205  &lt;Files "logout.html"&gt;
206    # le processus de déconnexion dbd exécute une requête pour
207    # enregistrer la déconnexion de l'utilisateur
208    Require dbd-logout
209    AuthzDBDQuery "UPDATE authn SET login = 'false' WHERE user = %s"
210  &lt;/Files&gt;
211&lt;/Directory&gt;</pre>

215<div class="directive-section"><h2><a name="authzdbdlogintoreferer" id="authzdbdlogintoreferer">Directive</a> <a name="AuthzDBDLoginToReferer" id="AuthzDBDLoginToReferer">AuthzDBDLoginToReferer</a></h2>
216<table class="directive">
217<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Définit si le client doit être redirigé vers la page
218d'origine en cas de connexion ou de déconnexion réussie si un en-tête
219de requête <code>Referer</code> est présent</td></tr>
220<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>AuthzDBDLoginToReferer On|Off</code></td></tr>
221<tr><th><a href="directive-dict.html#Default">Défaut:</a></th><td><code>AuthzDBDLoginToReferer Off</code></td></tr>
222<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>répertoire</td></tr>
223<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Extension</td></tr>
224<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authz_dbd</td></tr>
225</table>
226    <p>Utilisée en conjonction avec <code>Require dbd-login</code> ou
227    <code>Require dbd-logout</code>, cette directive permet de rediriger
228    le client vers la page d'origine (l'URL contenue dans l'en-tête
229    de requête HTTP <code>Referer</code>, s'il est présent). En
230    l'absence d'en-tête <code>Referer</code>, la définition
231    <code>AuthzDBDLoginToReferer On</code> sera ignorée.</p>
232

235<div class="directive-section"><h2><a name="authzdbdquery" id="authzdbdquery">Directive</a> <a name="AuthzDBDQuery" id="AuthzDBDQuery">AuthzDBDQuery</a></h2>
236<table class="directive">
237<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Définit la requête SQL pour l'opération requise</td></tr>
238<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>AuthzDBDQuery <var>requête</var></code></td></tr>
239<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>répertoire</td></tr>
240<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Extension</td></tr>
241<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authz_dbd</td></tr>
242</table>
243    <p>La directive <code class="directive">AuthzDBDQuery</code> permet de
244    spécifier une requête SQL à exécuter. Le but de cette requête dépend
245    de la directive <code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code> en cours de
246    traitement.</p>
247    <ul>
248    <li>Avec la directive <code>Require dbd-group</code>, elle spécifie
249    une requête permettant de rechercher les groupes d'appartenance de
250    l'utilisateur courant. Ceci correspond à la fonctionnalité standard
251    d'autres modules d'autorisation comme
252    <code class="module"><a href="../mod/mod_authz_groupfile.html">mod_authz_groupfile</a></code> et
253    <code class="module"><a href="../mod/mod_authz_dbm.html">mod_authz_dbm</a></code>.
254    La première colonne de chaque enregistrement renvoyé par la requête
255    doit contenir une chaîne de caractères correspondant à un nom de
256    groupe. La requête peut renvoyer zéro, un ou plusieurs
257    enregistrements.
258    <pre class="prettyprint lang-config">Require dbd-group
259AuthzDBDQuery "SELECT group FROM groups WHERE user = %s"</pre>
260
261    </li>
262    <li>Avec la directive <code>Require dbd-login</code> ou
263    <code>Require dbd-logout</code>, elle ne refusera jamais l'accès,
264    mais au contraire exécutera une requête SQL permettant d'enregistrer
265    la connexion ou la déconnexion de l'utilisateur. Ce dernier doit
266    être déjà authentifié avec <code class="module"><a href="../mod/mod_authn_dbd.html">mod_authn_dbd</a></code>.
267    <pre class="prettyprint lang-config">Require dbd-login
268AuthzDBDQuery "UPDATE authn SET login = 'true' WHERE user = %s"</pre>
269
270    </li>
271    </ul>
272    <p>Dans tous les cas, l'identifiant utilisateur sera transmis comme
273    paramètre sous la forme d'une simple chaîne lorsque la requête SQL
274    sera exécutée. Il y sera fait référence dans la requête en utilisant
275    le spécificateur de format <code>%s</code>.</p>

279<div class="directive-section"><h2><a name="authzdbdredirectquery" id="authzdbdredirectquery">Directive</a> <a name="AuthzDBDRedirectQuery" id="AuthzDBDRedirectQuery">AuthzDBDRedirectQuery</a></h2>
280<table class="directive">
281<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Définit une requête pour rechercher une page vers laquelle
282rediriger l'utilisateur après une connexion réussie</td></tr>
283<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>AuthzDBDRedirectQuery <var>requête</var></code></td></tr>
284<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>répertoire</td></tr>
285<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Extension</td></tr>
286<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authz_dbd</td></tr>
287</table>
288    <p>Spécifie une requête SQL optionnelle à utiliser après une
289    connexion (ou une déconnexion) réussie pour rediriger l'utilisateur
290    vers une URL, qui peut être spécifique à l'utilisateur.
291    L'identifiant utilisateur sera transmis comme paramètre sous la
292    forme d'une simple chaîne lorsque la requête SQL sera exécutée. Il y
293    sera fait référence dans la requête en utilisant le spécificateur de
294    format <code>%s</code>.</p>
295    <pre class="prettyprint lang-config">AuthzDBDRedirectQuery "SELECT userpage FROM userpages WHERE user = %s"</pre>
296
297    <p>La première colonne du premier enregistrement renvoyé par la
298    requête doit contenir une chaîne de caractères correspondant à une
299    URL vers laquelle rediriger le client. Les enregistrements suivants
300    sont ignorés. Si aucun enregistrement n'est renvoyé, le client ne
301    sera pas redirigé.</p>
302    <p>Notez que <code class="directive">AuthzDBDLoginToReferer</code> l'emporte
303    sur cette directive si les deux sont définies.</p>
