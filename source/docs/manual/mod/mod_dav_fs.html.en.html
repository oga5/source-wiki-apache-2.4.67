38<h3>Summary</h3>
39
40    <p>This module <em>requires</em> the service of <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code>. It acts as a support module for <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code> and provides access to resources located in the
41    server's file system. The formal name of this provider is
42    <code>filesystem</code>. <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code> backend providers
43    will be invoked by using the <code class="directive"><a href="../mod/mod_dav.html#dav">Dav</a></code>
44    directive:</p>
45
46    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">Dav filesystem</pre>
47</div>
48
49    <p>Since <code>filesystem</code> is the default provider for
50    <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code>, you may simply use the value
51    <code>On</code> instead.</p>

64<div class="directive-section"><h2><a name="DavLockDB" id="DavLockDB">DavLockDB</a> <a name="davlockdb" id="davlockdb">Directive</a></h2>
65<table class="directive">
66<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Location of the DAV lock database</td></tr>
67<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>DavLockDB <var>file-path</var></code></td></tr>
68<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host</td></tr>
69<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
70<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_dav_fs</td></tr>
71</table>
72    <p>Use the <code class="directive">DavLockDB</code> directive to specify
73    the full path to the lock database, excluding an extension. If
74    the path is not absolute, it will be taken relative to <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>. The implementation of
75    <code class="module"><a href="../mod/mod_dav_fs.html">mod_dav_fs</a></code> uses a SDBM database to track user
76    locks.</p>
77
78    
79
80    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">DavLockDB "var/DavLock"</pre>
81</div>
82
83    <p>The directory containing the lock database file must be
84    writable by the <code class="directive"><a href="../mod/mod_unixd.html#user">User</a></code>
85    and <code class="directive"><a href="../mod/mod_unixd.html#group">Group</a></code> under which
86    Apache is running.  For security reasons, you should create a
87    directory for this purpose rather than changing the permissions on
88    an existing directory.  In the above example, Apache will create
89    files in the <code>var/</code> directory under the <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> with the base filename
90    <code>DavLock</code> and extension name chosen by the server.</p>
91

95<div class="directive-section"><h2><a name="DavLockDiscovery" id="DavLockDiscovery">DavLockDiscovery</a> <a name="davlockdiscovery" id="davlockdiscovery">Directive</a></h2>
96<table class="directive">
97<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Enable lock discovery</td></tr>
98<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>DavLockDiscovery on|off</code></td></tr>
99<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>DavLockDiscovery on</code></td></tr>
100<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
101<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
102<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_dav_fs</td></tr>
103<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Available from Apache 2.4.55 and later.</td></tr>
104</table>
105    <p><code class="directive">DavLockDiscovery</code> controls if the lock
106    discovery feature is enabled for <code>PROPFIND</code> method.
107    When disabled, <code>PROPFIND</code> always returns an empty
108    <code>lockdiscovery</code> section. This improves performance
109    if clients use <code>PROPFIND</code> a lot.</p>
110    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">DavLockDiscovery off</pre>
111</div>
