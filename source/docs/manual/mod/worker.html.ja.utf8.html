44<h3>概要</h3>
45
46    <p>このマルチプロセッシングモジュール (MPM) 
47    は、マルチスレッドとマルチプロセスのハイブリッド型サーバを
48    実装しています。リクエストの応答にスレッドを使うと、
49    プロセスベースのサーバよりも少ないシステム資源で、
50    多くのリクエストに応答することができます。
51    それにもかかわらず、多くのスレッドを持った複数のプロセスを
52    維持することで、
53    プロセスベースのサーバの持つ安定性も保持しています。</p>
54
55    <p>この MPM を制御するのに使われる最も重要なディレクティブは、
56    <code class="directive"><a href="../mod/mpm_common.html#threadsperchild">ThreadsPerChild</a></code> と
57    <code class="directive"><a href="../mod/mpm_common.html#maxclients">MaxClients</a></code> です。
58    <code class="directive"><a href="../mod/mpm_common.html#threadsperchild">ThreadsPerChild</a></code> は
59    各子プロセスで用意されるスレッド数を制御して、
60    <code class="directive"><a href="../mod/mpm_common.html#maxclients">MaxClients</a></code> は
61    起動されるスレッドの総数の最大値を制限します。</p>

96<h2><a name="how-it-works" id="how-it-works">動作方法</a></h2>
97    <p>一つの制御用プロセス (親) が子プロセスを起動します。
98    子プロセスは
99    <code class="directive"><a href="../mod/mpm_common.html#threadsperchild">ThreadsPerChild</a></code>
100    ディレクティブで指定された一定数のサーバスレッドと接続を
101    listen するスレッドを一つ作ります。
102    Listener スレッドは接続が来たときにサーバプロセスに渡します。</p>
103
104    <p>Apache は<dfn>スペアの</dfn>、つまりアイドルなサーバスレッドの
105    プールを常に維持していて、それらは入ってくるリクエストに
106    答えられるように待機しています。
107    このようにして、クライアントはリクエストの応答が得られるようになるために
108    新しいスレッドやプロセスが生成されるのを
109    待たなくてもよいようになっています。
110    起動初期時のプロセス総数は、
111    <code class="directive"><a href="../mod/mpm_common.html#startservers">StartServers</a></code>
112    ディレクティブで設定されます。稼働中に、
113    Apache は全プロセスのアイドルスレッドの合計数を見積もって、
114    <code class="directive"><a href="../mod/mpm_common.html#minsparethreads">MinSpareThreads</a></code> と
115    <code class="directive"><a href="../mod/mpm_common.html#maxsparethreads">MaxSpareThreads</a></code>
116    で指定された範囲の中にこの数が収まるように fork したり
117    kill したりします。この操作は非常に自律的なので、
118    これらのディレクティブをデフォルト値から変更する必要は
119    めったにないでしょう。
120    同時に応答することのできるクライアント数の最大数
121    (つまり全プロセス中の総スレッド数の最大値) は
122    <code class="directive"><a href="../mod/mpm_common.html#maxclients">MaxClients</a></code>
123    ディレクティブで決定されます。
124    活動中の子プロセス数の最大値は
125    <code class="directive"><a href="../mod/mpm_common.html#maxclients">MaxClients</a></code> を
126    <code class="directive"><a href="../mod/mpm_common.html#threadsperchild">ThreadsPerChild</a></code> で割った
127    ものになります。</p>
128    
129    <p>活動中の子プロセスの数と子プロセス中のサーバスレッドの数の越えられない
130    上限を設定するディレクティブが二つあります。これらはサーバを
131    完全に停止して、再起動することでしか変更することはできません。
132    <code class="directive"><a href="../mod/mpm_common.html#serverlimit">ServerLimit
133    </a></code> は活動中の子プロセスの越えられない上限を設定し、
134    <code class="directive"><a href="../mod/mpm_common.html#maxclients">MaxClients</a></code> ディレクティブ
135    の値を <code class="directive"><a href="../mod/mpm_common.html#threadsperchild">
136    ThreadsPerChild</a></code> の値で割った値以上である
137    必要があります。<code class="directive"><a href="../mod/mpm_common.html#threadlimit">ThreadLimit</a></code> は
138    サーバスレッドの越えられない上限で、<code class="directive"><a href="../mod/mpm_common.html#threadsperchild">ThreadsPerChild</a></code> ディレクティブの
139    値以上である必要があります。</p>
140
141    <p>活動中の子プロセス群に加えて、少なくとも一つのサーバスレッドが
142    既存のクライアントからの接続を扱っている終了しようとしている
143    子プロセスがある可能性があります。終了中のプロセスは <code class="directive"><a href="../mod/mpm_common.html#maxclients">MaxClients</a></code> で指定された数まで
144    存在できますが、実際に期待される数はずっと少なくなります。この
145    振舞いは各子プロセスを終了させないようにすることで回避できます。
146    これは以下の様にして実現できます。</p>
147     
148    <ul>
149      <li><code class="directive"><a href="../mod/mpm_common.html#maxrequestsperchild">
150      MaxRequestsPerChild</a></code> の値を 0 に設定する</li>
151
152      <li><code class="directive"><a href="../mod/mpm_common.html#maxsparethreads">
153      MaxSpareThreads</a></code> の値を
154      <code class="directive"><a href="../mod/mpm_common.html#maxclients">MaxClients</a></code>
155      と同じ値にする</li>
156    </ul>
157
158    <p><code class="module"><a href="../mod/worker.html">worker</a></code> MPM の典型的なプロセス・スレッド制御の
159    設定では、次のようになります。</p>
160
161    <div class="example"><p><code>
162      ServerLimit         16<br />
163      StartServers         2<br />
164      MaxClients         150<br />
165      MinSpareThreads     25<br />
166      MaxSpareThreads     75<br />
167      ThreadsPerChild     25
168    </code></p></div>
169
170    <p>通常 Unix では親プロセスは 80 番ポートにバインドするために
171    <code>root</code> で起動されますが、子プロセスやスレッドは
172    もっと低い権限のユーザで Apache によって起動されます。
173    <code class="directive"><a href="../mod/mpm_common.html#user">User</a></code> と
174    <code class="directive"><a href="../mod/mpm_common.html#group">Group</a></code> ディレクティブは
175    Apache の子プロセスの権限を設定するのに用いられます。
176    子プロセスはクライアントに送るコンテンツ全てを読めないといけませんが、
177    可能な限り必要最小限の権限のみを持っているようにするべきです。
178    さらに、<code class="program"><a href="../programs/suexec.html">suexec</a></code>
179    が使用されていない限り、これらのディレクティブは
180    CGI スクリプトで継承される権限も設定します。</p>
181
182    <p><code class="directive"><a href="../mod/mpm_common.html#maxrequestsperchild">MaxRequestsPerChild</a></code>
183    は、古いプロセスを停止して新しいプロセスを起動することによって、
184    どの程度の頻度でサーバがプロセスをリサイクルするかを制御します。</p>
