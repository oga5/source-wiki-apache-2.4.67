78<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>デフォルトの設定は説明文を読んでください</code></td></tr>

83    <p>Apache がコアダンプする前に移動を試みるディレクトリを制御します。
84    デフォルト値は <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>
85    ディレクトリですが、このディレクトリはサーバの実行されているユーザ権限で
86    書き込み可能であるべきではないので、通常はコアダンプは書き込まれません。
87    デバッグのためにコアダンプが必要であれば、
88    このディレクティブを使って他の位置にコアダンプを書き出すようにできます。</p>
89
90    <div class="note"><h3>Linux でのコアダンプ</h3>
91      <p>Apache が root として起動されて、別のユーザの権限に以降した場合は
92      Linux のカーネルはディレクトリがプロセスの権限で書き込み可能な場合でさえも
93      コアダンプを<em>無効</em>にします。Apache (2.0.46 以降) は
94      Linux 2.4 以降ではコアダンプを行なうように再指定しますが、それは
95      <code class="directive">CoreDumpDirectory</code> を明示的に設定したときに
96      限ります。</p>

142<div class="directive-section"><h2><a name="Listen" id="Listen">Listen</a> <a name="listen" id="listen">ディレクティブ</a></h2>
143<table class="directive">
144<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>サーバが listen するIP アドレスとポート番号</td></tr>
145<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>Listen [<var>IP-address</var>:]<var>portnumber</var> [<var>protocol</var>]</code></td></tr>
146<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル</td></tr>
147<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>MPM</td></tr>
148<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td><code class="module"><a href="../mod/beos.html">beos</a></code>, <code class="module"><a href="../mod/leader.html">leader</a></code>, <code class="module"><a href="../mod/mpm_netware.html">mpm_netware</a></code>, <code class="module"><a href="../mod/mpm_winnt.html">mpm_winnt</a></code>, <code class="module"><a href="../mod/mpmt_os2.html">mpmt_os2</a></code>, <code class="module"><a href="../mod/perchild.html">perchild</a></code>, <code class="module"><a href="../mod/prefork.html">prefork</a></code>, <code class="module"><a href="../mod/threadpool.html">threadpool</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code>, <code class="module"><a href="../mod/event.html">event</a></code></td></tr>
149<tr><th><a href="directive-dict.html#Compatibility">互換性:</a></th><td>Apache 2.0 から必須ディレクティブ。<var>protocol</var>
150引数は 2.1.5 で追加。</td></tr>
151</table>
152    <p><code class="directive">Listen</code> ディレクティブは Apache
153    が特定の IP アドレスやポート番号だけを listen するように指定します。
154    デフォルトでは全ての IP インターフェースのリクエストに応答します。
155    <code class="directive">Listen</code> ディレクティブは
156    現在は必須のディレクティブとなりました。
157    もし設定ファイルになければ、サーバは起動に失敗します。
158    これは以前のバージョンの Apache から変更のあった部分です。</p>
159
160    <p><code class="directive">Listen</code> ディレクティブでは、特定のポートあるいは
161    アドレスとポートの組み合わせから入ってくるリクエストに対して
162    応答するように指定します。
163    もしポート番号だけが指定された場合は、サーバは全インターフェースの
164    指定されたポート番号に対して listen します。
165    IP アドレスがポートとともに指定された場合は、
166    サーバは指定されたポートとインターフェースに対して listen 
167    します。</p>
168
169    <p>複数のアドレスとポートに対して listen するように、
170    複数の <code class="directive">Listen</code> ディレクティブを使うこともできます。
171    サーバは列挙されたアドレスとポート全てからのリクエストに対して
172    応答します。</p>
173
174    <p>例えば、サーバが 80 番ポートと 8000 番ポートの両方の
175    コネクションを受け入れる場合は、次のようにします。</p>
176
177    <div class="example"><p><code>
178       Listen 80<br />
179       Listen 8000
180    </code></p></div>
181
182    <p>二つの特定のインターフェースとポート番号からのコネクションを
183    受け入れるようにするには、次のようにします。</p>
184
185    <div class="example"><p><code>
186       Listen 192.170.2.1:80<br />
187       Listen 192.170.2.5:8000
188    </code></p></div>
189
190    <p>IPv6 アドレスは角括弧で囲まなければなりません。
191    例えば次の例のようにです。</p>
192
193    <div class="example"><p><code>
194       Listen [2001:db8::a00:20ff:fea7:ccea]:80
195    </code></p></div>
196
197    <p><var>protocol</var> オプション引数は通常の設定では必要ありません。
198       無指定の場合、443 番ポートには <code>https</code> が、他のポートには
199       <code>http</code> がデフォルト値として使用されます。
200       protocol 指定は、どのモジュールがリクエストを処理するかを決定し、
201       <code class="directive"><a href="../mod/core.html#acceptfilter">AcceptFilter</a></code> 
202       によるプロトコル特有の最適化を行うようにします。</p>
203
204    <p>非標準なポートで運用している際にのみ protocol 指定が必要になります。
205       たとえば <code>https</code> なサイトを 8443 番ポートで運用している場合 :</p>
206
207    <div class="example"><p><code>
208      Listen 192.170.2.1:8443 https
209    </code></p></div>
210
211    <div class="note"><h3>エラー条件</h3>
212      同一 IP アドレスとポートの組に、複数の <code class="directive">Listen</code> 
213      ディレクティブを指定してしまうと、<code>Address already in use</code> 
214      というエラーメッセージを受けることになります。
215    </div>

503<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td><code class="module"><a href="../mod/beos.html">beos</a></code>, <code class="module"><a href="../mod/leader.html">leader</a></code>, <code class="module"><a href="../mod/mpm_winnt.html">mpm_winnt</a></code>, <code class="module"><a href="../mod/perchild.html">perchild</a></code>, <code class="module"><a href="../mod/prefork.html">prefork</a></code>, <code class="module"><a href="../mod/threadpool.html">threadpool</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code></td></tr>

505    <p>Apache は親プロセスと子プロセス間の通信にスコアボードを用います。
506    この通信機能にファイルを必要とするアーキテクチャもあります。
507    ファイルが指定されていなければ、Apache はまずメモリ上
508    (匿名共有メモリ) にスコアボードを作ろうとし、それが失敗すると
509    ディスク上にファイル (ファイルベースの共有メモリ) を作ろうとします。
510    このディレクティブを指定すると、Apache
511    は必ずディスクにファイルを生成します。</p>
512
513    <div class="example"><h3>例</h3><p><code>
514      ScoreBoardFile /var/run/apache_status
515    </code></p></div>
516
517    <p>ファイルベースの共有メモリは、サードパーティー製のアプリケーションで
518    スコアボードに直接アクセスする必要がある場合に役に立ちます。</p>
519
520    <p><code class="directive">ScoreBoardFile</code> を使う場合、
521    RAM ディスク上に置くとスピードが向上するでしょう。
522    しかし、ログファイルの設置位置や
523    <a href="../misc/security_tips.html">セキュリティ</a>
524    と同様の注意点があるので、注意してください。</p>

559    <p><code class="module"><a href="../mod/prefork.html">prefork</a></code> MPM の場合は、このディレクティブは
560    Apache プロセス稼働中における
561    <code class="directive"><a href="#maxclients">MaxClients</a></code>
562    に設定可能な上限値を設定することになります
563    <span class="transnote">(<em>訳注:</em> prefork の場合は同時クライアント数 = サーバプロセス数なので)</span>。
564    <code class="module"><a href="../mod/worker.html">worker</a></code> MPM の場合には、このディレクティブは
565    <code class="directive"><a href="#threadlimit">ThreadLimit</a></code>
566    ディレクティブと組み合わせて、
567    Apache プロセス稼働中における
568    <code class="directive"><a href="#maxclients">MaxClients</a></code>

573    は再起動で変更することができます。
574    </p>
575
576    <p>このディレクティブを使用する際は特に注意してください。

578    設定された場合は、余計な未使用共有メモリが割り当てられます。

582    Apache は起動しないか、起動しても不安定になるでしょう。</p>
583
584    <p><code class="module"><a href="../mod/prefork.html">prefork</a></code> MPM では、
585    <code class="directive"><a href="#maxclients">MaxClients</a></code>
586    を 256 (デフォルト) よりも大きな値に設定する必要がある時にだけ使用してください。
587    希望の <code class="directive"><a href="#maxclients">MaxClients</a></code>
588    数とくらべて、必要以上に大きな値を指定することは避けてください。</p>
589
590    <p><code class="module"><a href="../mod/worker.html">worker</a></code>, <code class="module"><a href="../mod/leader.html">leader</a></code>, 
591    <code class="module"><a href="../mod/threadpool.html">threadpool</a></code> MPM では、
592    <code class="directive"><a href="#maxclients">MaxClients</a></code> と
593    <code class="directive"><a href="#threadsperchild">ThreadsPerChild</a></code>
594    の設定で 16 サーバプロセス (デフォルト) 
595    以上必要になる場合にのみ使用してください。希望の
596    <code class="directive"><a href="#maxclients">MaxClients </a></code> と
597    <code class="directive"><a href="#threadsperchild">ThreadsPerChild</a></code>
598    とくらべて、必要となるサーバプロセス数以上に大きな値を
599    設定することは避けてください。</p>
600
601    <p><code class="module"><a href="../mod/perchild.html">perchild</a></code> MPM では、
602    <code class="directive"><a href="../mod/perchild.html#numservers">NumServers</a></code> を 8 (デフォルト)
603    よろいも大きな値に設定する必要があるときにのみ使用してください。</p>
604
605    <div class="note"><h3>注意</h3>
606      <p><code>ServerLimit 20000</code> という制限付きでコンパイルされています
607      (<code class="module"><a href="../mod/prefork.html">prefork</a></code> MPM では <code>200000</code>) 。
608      これはスペルミスによって誤って酷い状況になるのを、
609      回避するための処置です。</p>

753    <p>クライアントコネクションを受け持ち、コネクション処理に必要なモジュールの
754    呼び出しを行なっているスレッドの、(自動変数用の) スタックサイズは
755    <code class="directive">ThreadStackSize</code> ディレクティブで指定します。
756    大抵の場合 OS の指定しているスタックサイズのデフォルト値は
757    適切なものですが、調整が必要になる場合もあります:</p>
758
759    <ul>
760      <li>スレッドスタックサイズのデフォルト値が比較的小さく設定されている
761      プラットホーム (例えば HP-UX) では、自動変数用の領域で大きな容量を
762      使用するサードパーティ製モジュールのために Apache がクラッシュする
763      場合もあります。そのモジュールは他のプラットホームでは
764      スタックサイズが大きいために、快調に動作するかもしれません。
765      このタイプのクラッシュは、<code class="directive">ThreadStackSize</code>
766      で OS のデフォルト値より大きな値を指定することで解決します。
767      サードパーティ製モジュールでこの処置が必要であると記載されている
768      場合か、Apache の出力するメッセージでスレッドスタックサイズが
769      小さすぎると指摘されている場合にのみ、この調整をしてください。</li>
770
771      <li>デフォルトスレッドスタックサイズが、Web サーバ用途に必要な量よりも
772      明らかに大きすぎる場合、<code class="directive">ThreadStackSize</code> 
773      を OS のデフォルト値よりも小さな値にすることで、子プロセスあたりの
774      スレッド数をより多く持たせられるようになります。
775      このタイプの調整は、テスト環境でウェブサーバを完全に
776      テストできる場合に限って行なうべきです。
777      まれに多数のスタックが要求されるリクエストを受けることがあるかも
778      しれないからです。
779      Web サーバの設定を変更すると、現在の <code class="directive">ThreadStackSize</code> 
780      の設定が取り消される場合があります。</li>
781    </ul>
