37<h3>Summary</h3>
38
39    <p><code class="module"><a href="../mod/mod_cache_socache.html">mod_cache_socache</a></code> implements a shared object cache
40    (socache) based storage manager for <code class="module"><a href="../mod/mod_cache.html">mod_cache</a></code>.</p>
41
42    <p>The headers and bodies of cached responses are combined, and stored
43    underneath a single key in the shared object cache. A
44    <a href="../socache.html">number of implementations</a> of shared object
45    caches are available to choose from.</p>
46
47    <p>Multiple content negotiated responses can be stored concurrently,
48    however the caching of partial content is not yet supported by this
49    module.</p>
50
51    <pre class="prettyprint lang-config"># Turn on caching
52CacheSocache shmcb
53CacheSocacheMaxSize 102400
54&lt;Location "/foo"&gt;
55    CacheEnable socache
56&lt;/Location&gt;
57
58# Fall back to the disk cache
59CacheSocache shmcb
60CacheSocacheMaxSize 102400
61&lt;Location "/foo"&gt;
62    CacheEnable socache
63    CacheEnable disk
64&lt;/Location&gt;</pre>
65
66
67    <div class="note"><h3>Note:</h3>
68      <p><code class="module"><a href="../mod/mod_cache_socache.html">mod_cache_socache</a></code> requires the services of
69      <code class="module"><a href="../mod/mod_cache.html">mod_cache</a></code>, which must be loaded before
70      <code class="module"><a href="../mod/mod_cache_socache.html">mod_cache_socache</a></code>.</p>

110<div class="directive-section"><h2><a name="CacheSocacheMaxSize" id="CacheSocacheMaxSize">CacheSocacheMaxSize</a> <a name="cachesocachemaxsize" id="cachesocachemaxsize">Directive</a></h2>
111<table class="directive">
112<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>The maximum size (in bytes) of an entry to be placed in the
113cache</td></tr>
114<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>CacheSocacheMaxSize <var>bytes</var></code></td></tr>
115<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>CacheSocacheMaxSize 102400</code></td></tr>
116<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
117<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
118<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_cache_socache</td></tr>
119<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Available in Apache 2.4.5 and later</td></tr>
120</table>
121    <p>The <code class="directive">CacheSocacheMaxSize</code> directive sets the
122    maximum size, in bytes, for the combined headers and body of a document
123    to be considered for storage in the cache. The larger the headers that
124    are stored alongside the body, the smaller the body may be.</p>
125
126    <p>The <code class="module"><a href="../mod/mod_cache_socache.html">mod_cache_socache</a></code> module will only attempt to
127    cache responses that have an explicit content length, or that are small
128    enough to be written in one pass. This is done to allow the
129    <code class="module"><a href="../mod/mod_cache_disk.html">mod_cache_disk</a></code> module to have an opportunity to cache
130    responses larger than those cacheable within
131    <code class="module"><a href="../mod/mod_cache_socache.html">mod_cache_socache</a></code>.</p>
132
133    <pre class="prettyprint lang-config">CacheSocacheMaxSize 102400</pre>

170    <p>The <code class="directive">CacheSocacheMinTime</code> directive sets the
171    amount of seconds beyond the freshness lifetime of the response that the
172    response should be cached for in the shared object cache. If a response is
173    only stored for its freshness lifetime, there will be no opportunity to
174    revalidate the response to make it fresh again.</p>

181<div class="directive-section"><h2><a name="CacheSocacheReadSize" id="CacheSocacheReadSize">CacheSocacheReadSize</a> <a name="cachesocachereadsize" id="cachesocachereadsize">Directive</a></h2>
182<table class="directive">
183<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>The minimum size (in bytes) of the document to read and be cached
184  before sending the data downstream</td></tr>
185<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>CacheSocacheReadSize <var>bytes</var></code></td></tr>
186<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>CacheSocacheReadSize 0</code></td></tr>
187<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
188<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
189<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_cache_socache</td></tr>
190<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Available in Apache 2.4.5 and later</td></tr>
191</table>
192    <p>The <code class="directive">CacheSocacheReadSize</code> directive sets the
193    minimum amount of data, in bytes, to be read from the backend before the
194    data is sent to the client. The default of zero causes all data read of
195    any size to be passed downstream to the client immediately as it arrives.
196    Setting this to a higher value causes the disk cache to buffer at least
197    this amount before sending the result to the client. This can improve
198    performance when caching content from a slow reverse proxy.</p>
199
200    <p>This directive only takes effect when the data is being saved to the
201    cache, as opposed to data being served from the cache.</p>
202
203    <pre class="prettyprint lang-config">CacheSocacheReadSize 102400</pre>
204

208<div class="directive-section"><h2><a name="CacheSocacheReadTime" id="CacheSocacheReadTime">CacheSocacheReadTime</a> <a name="cachesocachereadtime" id="cachesocachereadtime">Directive</a></h2>
209<table class="directive">
210<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>The minimum time (in milliseconds) that should elapse while reading
211  before data is sent downstream</td></tr>
212<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>CacheSocacheReadTime <var>milliseconds</var></code></td></tr>
213<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>CacheSocacheReadTime 0</code></td></tr>
214<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
215<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
216<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_cache_socache</td></tr>
217<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Available in Apache 2.4.5 and later</td></tr>
218</table>
219    <p>The <code class="directive">CacheSocacheReadTime</code> directive sets the minimum amount
220    of elapsed time that should pass before making an attempt to send data
221    downstream to the client. During the time period, data will be buffered
222    before sending the result to the client. This can improve performance when
223    caching content from a reverse proxy.</p>
224
225    <p>The default of zero disables this option.</p>
226
227    <p>This directive only takes effect when the data is being saved to the
228    cache, as opposed to data being served from the cache. It is recommended
229    that this option be used alongside the
230    <code class="directive"><a href="#cachesocachereadsize">CacheSocacheReadSize</a></code> directive
231    to ensure that the server does not buffer excessively should data arrive faster
232    than expected.</p>
233
234    <pre class="prettyprint lang-config">CacheSocacheReadTime 1000</pre>
