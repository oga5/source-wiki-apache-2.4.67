49<h2><a name="Status" id="Status">ステータス</a></h2>
50
51    <p>これは、そのモジュールが Apache
52    ウェブサーバにどれくらい密接に組み込まれているかを示します。
53    言い換えれば、モジュールを組み込み、その機能を利用するために、
54    サーバを再コンパイルする必要があるかもしれないということを示します。
55    この属性が取り得る値は以下のものです:</p>
56    <dl>
57      <dt>MPM</dt>
58
59      <dd>ステータスが "MPM" のモジュールは<a href="../mpm.html">マルチプロセッシングモジュール</a>です。
60      他の種類のモジュールとは違って、Apache は常に MPM を一つだけ
61      使用し続けます。この種類のモジュールは基本的なリクエストの扱いと
62      ディスパッチを行ないます。</dd>
63
64      <dt>Base</dt>
65
66      <dd>ステータスが "Base"
67      のモジュールは、デフォルトでコンパイルされてわざわざ設定から
68      モジュールを削除していない限り、通常は利用可能です。
69      </dd>
70
71      <dt>Extension</dt>
72
73      <dd>ステータスが "Extension" のモジュールは、
74      デフォルトではコンパイルされず、サーバにも読み込まれません。
75      そのモジュールとその機能を有効にするには、
76      サーバをビルドするための設定を変更して、Apache
77      を再コンパイルする必要があります。</dd>
78      <dt>Experimental</dt>
79
80      <dd>ステータスが "Experimental" のモジュールは、
81      Apache 配布物に同梱されていますが、
82      使用する場合は自己責任で行なう必要があります。
83      そのモジュールは、ドキュメントも完成に向けて作成中ですし、
84      サポートされるているとは限りません。</dd>
85      <dt>External</dt>
86
87      <dd>ステータスが "External" のモジュールは、基本 Apache
88      配布に同梱されません ("サードパーティーモジュール")。
89      そのため、我々に責任はありませんし、
90      そのモジュールのサポートもしていません。</dd>
91    </dl>

94<h2><a name="SourceFile" id="SourceFile">ソースファイル</a></h2>
95
96    <p>これは単純に、
97    そのモジュールに必要なコードを含むソースファイルの名前を列挙したものです。
98    これは、<code class="directive"><a href="../mod/core.html#ifmodule">&lt;IfModule&gt;</a></code>
99    ディレクティブで使用される名前でもあります。
100    </p>
101</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>
102<div class="section">
103<h2><a name="ModuleIdentifier" id="ModuleIdentifier">モジュール識別子</a></h2>
104
105    <p>この文字列は、モジュールの動的読み込みを行なうときに使用する <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code>
106    ディレクティブにおいて使用されるモジュールの識別子です。
107    詳しく書くと、ソースファイル内の module タイプの外部変数の名前です。
108    </p>

112
