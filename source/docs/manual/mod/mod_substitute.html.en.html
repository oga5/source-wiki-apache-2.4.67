54<div class="directive-section"><h2><a name="Substitute" id="Substitute">Substitute</a> <a name="substitute" id="substitute">Directive</a></h2>
55<table class="directive">
56<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Pattern to filter the response content</td></tr>
57<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>Substitute <var>s/pattern/substitution/[infq]</var></code></td></tr>
58<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>directory, .htaccess</td></tr>
59<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>FileInfo</td></tr>
60<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
61<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_substitute</td></tr>
62</table>
63    <p>The <code class="directive">Substitute</code> directive specifies a
64    search and replace pattern to apply to the response body.</p>
65
66    <p>The meaning of the pattern can be modified by using any
67    combination of these flags:</p>
68
69    <dl>
70        <dt><code>i</code></dt>
71        <dd>Perform a case-insensitive match.</dd>
72        <dt><code>n</code></dt>
73        <dd>By default the pattern is treated as a regular expression.
74        Using the <code>n</code> flag forces the pattern to be treated
75        as a fixed string.</dd>
76        <dt><code>f</code></dt>
77        <dd>The <code>f</code> flag causes <code>mod_substitute</code> to flatten the
78        result of a substitution allowing for later substitutions to
79        take place on the boundary of this one. This is the default.</dd>
80        <dt><code>q</code></dt>
81        <dd>The <code>q</code> flag causes <code>mod_substitute</code> to not
82        flatten the buckets after each substitution. This can
83        result in much faster response and a decrease in memory
84        utilization, but should only be used if there is no possibility
85        that the result of one substitution will ever match a pattern
86        or regex of a subsequent one.</dd>
87    </dl>
88
89    <p>The <var>substitution</var> may contain literal text and regular
90    expression backreferences</p>
91
92    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">&lt;Location "/"&gt;
93    AddOutputFilterByType SUBSTITUTE text/html
94    Substitute "s/foo/bar/ni"
95&lt;/Location&gt;</pre>
96</div>
97
98    <p>The character which is used to separate (or "delimit") the
99    various parts of the substitution string is referred to as the
100    "delimiter", and it is most common to use a slash for this
101    purpose.</p>
102
103    <p>If either the pattern or the substitution contain a slash
104    character then an alternative delimiter may be used to make the
105    directive more readable:</p>
106
107    <div class="example"><h3>Example of using an alternate delimiter</h3><pre class="prettyprint lang-config">&lt;Location "/"&gt;
108    AddOutputFilterByType SUBSTITUTE text/html
109    Substitute "s|&lt;BR */?&gt;|&lt;br /&gt;|i"
110&lt;/Location&gt;</pre>
111</div>
112
113    <p>Backreferences can be used in the comparison and in the substitution,
114    when regular expressions are used, as illustrated in the following example: </p>
115    <div class="example"><h3>Example of using backreferences and captures</h3><pre class="prettyprint lang-config">&lt;Location "/"&gt;
116    AddOutputFilterByType SUBSTITUTE text/html
117    # "foo=k,bar=k" -&gt; "foo/bar=k"
118    Substitute "s|foo=(\w+),bar=\1|foo/bar=$1|"
119&lt;/Location&gt;</pre>
120</div>
121
122    <p>A common use scenario for <code>mod_substitute</code> is the
123    situation in which a front-end server proxies requests to a back-end
124    server which returns HTML with hard-coded embedded URLs that refer
125    to the back-end server. These URLs don't work for the end-user,
126    since the back-end server is unreachable.</p>
127
128    <p>In this case, <code>mod_substitute</code> can be used to rewrite
129    those URLs into something that will work from the front end:</p>
130
131    <div class="example"><h3>Rewriting URLs embedded in proxied content</h3><pre class="prettyprint lang-config">ProxyPass        "/blog/" "http://internal.blog.example.com/"
132ProxyPassReverse "/blog/" "http://internal.blog.example.com/"
133
134Substitute "s|http://internal.blog.example.com/|http://www.example.com/blog/|i"</pre>
135</div>
136
137    <p><code class="directive"><a href="../mod/mod_proxy.html#proxypassreverse">ProxyPassReverse</a></code>
138    modifies any <code>Location</code> (redirect) headers that are sent
139    by the back-end server, and, in this example,
140    <code class="directive">Substitute</code> takes care of the rest of the problem by
141    fixing up the HTML response as well.</p>
142

157    <p>Whether to apply the inherited <code class="directive"><a href="#substitute">Substitute</a></code>
158    patterns first (<code>on</code>), or after the ones of the current
159    context (<code>off</code>).
160    <code class="directive">SubstituteInheritBefore</code> is itself inherited,
161    hence contexts that inherit it (those that don't specify their own
162    <code class="directive">SubstituteInheritBefore</code> value) will apply the
163    closest defined merge order.</p>
164

167<div class="directive-section"><h2><a name="SubstituteMaxLineLength" id="SubstituteMaxLineLength">SubstituteMaxLineLength</a> <a name="substitutemaxlinelength" id="substitutemaxlinelength">Directive</a></h2>
168<table class="directive">
169<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Set the maximum line size</td></tr>
170<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>SubstituteMaxLineLength <var>bytes</var>(b|B|k|K|m|M|g|G)</code></td></tr>
171<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>SubstituteMaxLineLength 1m</code></td></tr>
172<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>directory, .htaccess</td></tr>
173<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>FileInfo</td></tr>
174<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
175<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_substitute</td></tr>
176<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Available in httpd 2.4.11 and later</td></tr>
177</table>
178    <p>The maximum line size handled by <code class="module"><a href="../mod/mod_substitute.html">mod_substitute</a></code>
179    is limited to restrict memory use. The limit can be configured
180    using <code class="directive">SubstituteMaxLineLength</code>.
181    The value can be given as the number of bytes and can be suffixed
182    with a single letter <code>b</code>, <code>B</code>, <code>k</code>,
183    <code>K</code>, <code>m</code>, <code>M</code>, <code>g</code>,
184    <code>G</code> to provide the size in bytes, kilobytes, megabytes
185    or gigabytes respectively.</p>
186
187    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">&lt;Location "/"&gt;
188    AddOutputFilterByType SUBSTITUTE text/html
189    SubstituteMaxLineLength 10m
190    Substitute "s/foo/bar/ni"
191&lt;/Location&gt;</pre>
192</div>
