40<h3>Sommaire</h3>
41
42    <p>Ce module possède deux directives. La directive <code class="directive"><a href="#action">Action</a></code> vous permet de lancer
43    l'exécution de scripts CGI chaque fois qu'un fichier possédant un
44    certain <a class="glossarylink" href="../glossary.html#mime-type" title="voir glossaire">type de contenu MIME</a>
45    fait l'objet d'une requête. La directive <code class="directive"><a href="#script">Script</a></code> vous permet de lancer
46    l'exécution de scripts CGI chaque fois que la requête utilise une
47    méthode particulière. Ceci facilite grandement l'exécution de
48    scripts qui traitent des fichiers.</p> </div>

63<div class="directive-section"><h2><a name="action" id="action">Directive</a> <a name="Action" id="Action">Action</a></h2>
64<table class="directive">
65<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Active un script CGI pour un gestionnaire ou un type de
66contenu particulier</td></tr>
67<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>Action <var>type d'action</var> <var>script cgi</var>
68[virtual]</code></td></tr>
69<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale, serveur virtuel, répertoire, .htaccess</td></tr>
70<tr><th><a href="directive-dict.html#Override">Surcharges autorisées:</a></th><td>FileInfo</td></tr>
71<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
72<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_actions</td></tr>
73<tr><th><a href="directive-dict.html#Compatibility">Compatibilité:</a></th><td>Le modificateur <code>virtual</code> et le passage de
74gestionnaire ont été introduits dans Apache 2.1</td></tr>
75</table>
76    <p>Cette directive ajoute une action qui va activer <var>script
77    cgi</var> lorsque <var>type d'action</var> est déclenché par la
78    requête. <var>script cgi</var> est un chemin URL vers une ressource
79    qui a été désignée comme script CGI à l'aide des directives
80    <code class="directive"><a href="../mod/mod_alias.html#scriptalias">ScriptAlias</a></code> ou <code class="directive"><a href="../mod/mod_mime.html#addhandler">AddHandler</a></code>. <var>type d'action</var>
81    peut être soit un <a href="../handler.html">gestionnaire</a>, soit
82    un <a class="glossarylink" href="../glossary.html#mime-type" title="voir glossaire">type de contenu MIME</a>. L'URL
83    et le chemin du document correspondant sont envoyés en utilisant
84    les variables d'environnement CGI standards <code>PATH_INFO</code>
85    et <code>PATH_TRANSLATED</code>. Le gestionnaire utilisé pour cette
86    requête particulière est transmis à l'aide de la variable
87    <code>REDIRECT_HANDLER</code>.</p>
88
89    <div class="example"><h3>Exemple : type MIME</h3><pre class="prettyprint lang-config"># Requests for files of a particular MIME content type:
90Action image/gif /cgi-bin/images.cgi</pre>
91</div>
92
93    <p>Dans cet exemple, les requêtes pour des fichiers possédant
94    le type de contenu MIME <code>image/gif</code> seront traitées par
95    le script CGI <code>/cgi-bin/images.cgi</code>.</p>
96
97    <div class="example"><h3>Example: File extension</h3><pre class="prettyprint lang-config"># Files of a particular file extension
98AddHandler my-file-type .xyz
99Action my-file-type "/cgi-bin/program.cgi"</pre>
100</div>
101    <p>Dans cet exemple, les requêtes pour des fichiers possédant
102    l'extension <code>.xyz</code> seront traitées par
103    le script CGI <code>/cgi-bin/programme.cgi</code>.</p>
104
105    <p>Le modificateur optionnel <code>virtual</code> permet de
106    désactiver la vérification de l'existence du fichier demandé. Ceci
107    peut s'avérer utile, par exemple, si vous voulez utiliser la
108    directive <code class="directive">Action</code> pour des localisations
109    virtuelles.</p>
110
111    <pre class="prettyprint lang-config">&lt;Location "/news"&gt;
112    SetHandler news-handler
113    Action news-handler "/cgi-bin/news.cgi" virtual
114&lt;/Location&gt;</pre>

123<div class="directive-section"><h2><a name="script" id="script">Directive</a> <a name="Script" id="Script">Script</a></h2>
124<table class="directive">
125<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Active un script CGI dans le cas d'une méthode de requête
126particulière.</td></tr>
127<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>Script <var>méthode</var> <var>script cgi</var></code></td></tr>
128<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale, serveur virtuel, répertoire</td></tr>
129<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
130<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_actions</td></tr>
131</table>
132    <p>Cette directive ajoute une action qui va activer <var>script
133    cgi</var> lorsqu'un fichier est demandé en utilisant la méthode
134    <var>méthode</var>. <var>script cgi</var> est le chemin URL d'une
135    ressource qui a été désignée comme script CGI en utilisant les
136    directives <code class="directive"><a href="../mod/mod_alias.html#scriptalias">ScriptAlias</a></code> ou <code class="directive"><a href="../mod/mod_mime.html#addhandler">AddHandler</a></code>. L'URL et le chemin du
137    document demandé sont envoyés en utilisant les variables
138    d'environnement CGI standards <code>PATH_INFO</code> et
139    <code>PATH_TRANSLATED</code>.</p>
140
141    <div class="note">
142      Tous les noms de méthode peuvent être utilisés. <strong>Les noms
143      de méthode sont sensibles à la casse</strong>, si bien que
144      <code>Script PUT</code> et <code>Script put</code> ont des effets
145      totalement différents.
146    </div>
147
148    <p>Notez que la commande <code class="directive">Script</code> ne définit
149    que des actions par défaut. Si un script CGI est appelé, ou toute
150    autre ressource capable de gérer la méthode de la requête en
151    interne, il agira en conséquence. Notez aussi que
152    <code class="directive">Script</code> avec une méthode <code>GET</code> ne
153    sera appelé que si la requête possède des arguments (par exemple
154    foo.html?hi). Dans le cas contraire, la requête sera traitée
155    normalement.</p>
156
157    <pre class="prettyprint lang-config"># All GET requests go here
158Script GET "/cgi-bin/search"
159
160# A CGI PUT handler
161Script PUT "/~bob/put.cgi"</pre>
