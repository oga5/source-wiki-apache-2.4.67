39<h3>Summary</h3>
40
41    <p>This module provides authorization capabilities so that
42    authenticated users can be allowed or denied access to portions
43    of the web site by group membership. Similar functionality is
44    provided by <code class="module"><a href="../mod/mod_authz_dbm.html">mod_authz_dbm</a></code>.</p>

59<h2><a name="requiredirectives" id="requiredirectives">The Require Directives</a></h2>
60
61    <p>Apache's <code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code>
62    directives are used during the authorization phase to ensure that
63    a user is allowed to access a resource.  mod_authz_groupfile extends the
64    authorization types with <code>group</code> and <code>group-file</code>.
65    </p>
66
67    <p>Since v2.4.8, <a href="../expr.html">expressions</a> are supported
68    within the groupfile require directives.</p>
69
70<h3><a name="reqgroup" id="reqgroup">Require group</a></h3>
71
72    <p>This directive specifies group membership that is required for the
73    user to gain access.</p>
74
75    <pre class="prettyprint lang-config">Require group admin</pre>
76
77
78
79
80<h3><a name="reqfilegroup" id="reqfilegroup">Require file-group</a></h3>
81
82    <p>When this directive is specified, the filesystem permissions on
83    the file being accessed are consulted. The user must be a member of
84    a group with the same name as the group that owns the file.
85    See <code class="module"><a href="../mod/mod_authz_owner.html">mod_authz_owner</a></code> for more
86    details.</p>
87
88    <pre class="prettyprint lang-config">Require file-group</pre>
89

95<div class="directive-section"><h2><a name="AuthGroupFile" id="AuthGroupFile">AuthGroupFile</a> <a name="authgroupfile" id="authgroupfile">Directive</a></h2>
96<table class="directive">
97<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Sets the name of a text file containing the list
98of user groups for authorization</td></tr>
99<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>AuthGroupFile <var>file-path</var></code></td></tr>
100<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>directory, .htaccess</td></tr>
101<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>AuthConfig</td></tr>
102<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
103<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authz_groupfile</td></tr>
104</table>
105    <p>The <code class="directive">AuthGroupFile</code> directive sets the
106    name of a textual file containing the list of user groups for user
107    authorization. <var>File-path</var> is the path to the group
108    file. If it is not absolute, it is treated as relative to the <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>.</p>
109
110    <p>Each line of the group file contains a groupname followed by a
111    colon, followed by the member usernames separated by spaces.</p>
112
113    <div class="example"><h3>Example:</h3><p><code>
114      mygroup: bob joe anne
115    </code></p></div>
116
117    <p>Note that searching large text files is <em>very</em>
118    inefficient; <code class="directive"><a href="../mod/mod_authz_dbm.html#authdbmgroupfile">AuthDBMGroupFile</a></code> provides a much better performance.</p>
119
120    <div class="warning"><h3>Security</h3>
121      <p>Make sure that the <code class="directive">AuthGroupFile</code> is
122      stored outside the document tree of the web-server; do <em>not</em>
123      put it in the directory that it protects. Otherwise, clients may
124      be able to download the <code class="directive">AuthGroupFile</code>.</p>
125    </div>
