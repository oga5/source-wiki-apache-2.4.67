38<h3>Summary</h3>
39
40    <p>Any file that has the handler
41    <code>cgi-script</code> will be treated
42    as a CGI script, and run by the server, with its output being
43    returned to the client. Files acquire this handler either by
44    having a name containing an extension defined by the
45    <code class="directive"><a href="../mod/mod_mime.html#addhandler">AddHandler</a></code> directive, or by being
46    in a <code class="directive"><a href="../mod/mod_alias.html#scriptalias">ScriptAlias</a></code>
47    directory.</p>
48
49    <p>For an introduction to using CGI scripts with Apache, see
50    our tutorial on <a href="../howto/cgi.html">Dynamic Content
51    With CGI</a>.</p>
52
53    <p>When using a multi-threaded MPM under unix, the module
54    <code class="module"><a href="../mod/mod_cgid.html">mod_cgid</a></code> should be used in place of
55    this module. At the user level, the two modules are essentially
56    identical.</p>
57
58    <p>For backward-compatibility, the cgi-script handler will also be activated
59    for any file with the mime-type <code>application/x-httpd-cgi</code>. The
60    use of the magic mime-type is deprecated.</p>

85<h2><a name="env" id="env">CGI Environment variables</a></h2>
86    <p>The server will set the CGI environment variables as described
87    in the <a href="http://www.ietf.org/rfc/rfc3875">CGI specification</a>,
88    with the following provisions:</p>
89
90    <dl>
91      <dt>PATH_INFO</dt>
92
93      <dd>This will not be available if the <code class="directive"><a href="../mod/core.html#acceptpathinfo">AcceptPathInfo</a></code> directive is explicitly set to
94      <code>off</code>.  The default behavior, if <code class="directive">AcceptPathInfo</code> is not given, is that <code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code> will accept path info (trailing <code>
95      /more/path/info</code> following the script filename in the URI),
96      while the core server will return a 404 NOT FOUND error for requests
97      with additional path info. Omitting the <code class="directive">AcceptPathInfo</code> directive has the same effect as setting
98      it <code>On</code> for <code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code> requests.</dd>
99
100      <dt>REMOTE_HOST</dt>
101
102      <dd>This will only be set if <code class="directive"><a href="../mod/core.html#hostnamelookups">HostnameLookups</a></code> is set to <code>on</code> (it
103      is off by default), and if a reverse DNS lookup of the accessing
104      host's address indeed finds a host name.</dd>
105
106      <dt>REMOTE_IDENT</dt>
107
108      <dd>This will only be set if <code class="directive"><a href="../mod/mod_ident.html#identitycheck">IdentityCheck</a></code> is set to
109      <code>on</code> and the accessing host supports the ident
110      protocol. Note that the contents of this variable cannot be
111      relied upon because it can easily be faked, and if there is a
112      proxy between the client and the server, it is usually
113      totally useless.</dd>
114
115      <dt>REMOTE_USER</dt>
116
117      <dd>This will only be set if the CGI script is subject to
118      authentication.</dd>
119
120    </dl>
121    <p>This module also leverages the core functions 
122       <a href="https://ci.apache.org/projects/httpd/trunk/doxygen/group__APACHE__CORE__SCRIPT.html#ga0e81f9571a8a73f5da0e89e1f46d34b1">ap_add_common_vars</a> and 
123       <a href="https://ci.apache.org/projects/httpd/trunk/doxygen/group__APACHE__CORE__SCRIPT.html#ga6b975cd7ff27a338cb8752381a4cc14f">ap_add_cgi_vars</a> 
124       to add environment variables like:</p> 
125       <dl>
126           <dt>DOCUMENT_ROOT</dt>
127
128           <dd>Set with the content of the related <code class="directive"><a href="../mod/core.html#documentroot">DocumentRoot</a></code> directive.</dd>
129
130           <dt>SERVER_NAME</dt>
131
132           <dd>The fully qualified domain name related to the request.</dd>
133
134           <dt>SERVER_ADDR</dt>
135
136           <dd>The IP address of the Virtual Host serving the request.</dd>
137
138           <dt>SERVER_ADMIN</dt>
139
140           <dd>Set with the content of the related <code class="directive"><a href="../mod/core.html#serveradmin">ServerAdmin</a></code> directive.</dd>
141      </dl>
142    <p>For an exhaustive list it is suggested to write a basic CGI script 
143       that dumps all the environment variables passed by Apache in a convenient format.
144    </p>

147<h2><a name="cgi-debug" id="cgi-debug">CGI Debugging</a></h2>
148    <p>Debugging CGI scripts has traditionally been difficult, mainly
149    because it has not been possible to study the output (standard
150    output and error) for scripts which are failing to run
151    properly. These directives provide more detailed logging of errors
152    when they occur.</p>
153
154    <h3>CGI Logfile Format</h3>
155      <p>When configured, the CGI error log logs any CGI which does not
156      execute properly. Each CGI script which fails to operate causes
157      several lines of information to be logged. The first two lines
158      are always of the format:</p>
159
160      <div class="example"><p><code>
161        %% [<var>time</var>] <var>request-line</var><br />
162        %% <var>HTTP-status</var> <var>CGI-script-filename</var>
163      </code></p></div>
164
165      <p>If the error is that CGI script cannot be run, the log file
166      will contain an extra two lines:</p>
167
168      <div class="example"><p><code>
169        %%error<br />
170        <var>error-message</var>
171      </code></p></div>
172
173      <p>Alternatively, if the error is the result of the script
174      returning incorrect header information (often due to a bug in
175      the script), the following information is logged:</p>
176
177      <div class="example"><p><code>
178        %request<br />
179        <var>All HTTP request headers received</var><br />
180        <var>POST or PUT entity (if any)</var><br />
181        %response<br />
182        <var>All headers output by the CGI script</var><br />
183        %stdout<br />
184        <var>CGI standard output</var><br />
185        %stderr<br />
186        <var>CGI standard error</var><br />
187      </code></p></div>
188
189      <p>(The %stdout and %stderr parts may be missing if the script did
190      not output anything on standard output or standard error).</p>

194<div class="directive-section"><h2><a name="CGIScriptTimeout" id="CGIScriptTimeout">CGIScriptTimeout</a> <a name="cgiscripttimeout" id="cgiscripttimeout">Directive</a></h2>
195<table class="directive">
196<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>The length of time to wait for more output from the
197CGI program</td></tr>
198<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>CGIScriptTimeout <var>time</var>[s|ms]</code></td></tr>
199<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>value of <code class="directive"><a href="../mod/core.html#timeout">Timeout</a></code> directive when
200unset</code></td></tr>

206    <p>This directive limits the length of time to wait for more output from
207    the CGI program.  If the time is exceeded, the request and CGI are
208    terminated.</p>

216<div class="directive-section"><h2><a name="ScriptLog" id="ScriptLog">ScriptLog</a> <a name="scriptlog" id="scriptlog">Directive</a></h2>
217<table class="directive">
218<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Location of the CGI script error logfile</td></tr>
219<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ScriptLog <var>file-path</var></code></td></tr>
220<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host</td></tr>
221<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
222<tr><th><a href="directive-dict.html#Module">Module:</a></th><td><code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code>, <code class="module"><a href="../mod/mod_cgid.html">mod_cgid</a></code></td></tr>
223</table>
224    <p>The <code class="directive">ScriptLog</code> directive sets the CGI
225    script error logfile. If no <code class="directive">ScriptLog</code> is given,
226    no error log is created. If given, any CGI errors are logged into the
227    filename given as argument. If this is a relative file or path it is
228    taken relative to the <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>.
229    </p>
230
231    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">ScriptLog logs/cgi_log</pre>
232</div>
233
234    <p>This log will be opened as the user the child processes run
235    as, <em>i.e.</em> the user specified in the main <code class="directive"><a href="../mod/mod_unixd.html#user">User</a></code> directive. This means that
236    either the directory the script log is in needs to be writable
237    by that user or the file needs to be manually created and set
238    to be writable by that user. If you place the script log in
239    your main logs directory, do <strong>NOT</strong> change the
240    directory permissions to make it writable by the user the child
241    processes run as.</p>
242
243    <p>Note that script logging is meant to be a debugging feature
244    when writing CGI scripts, and is not meant to be activated
245    continuously on running servers. It is not optimized for speed
246    or efficiency, and may have security problems if used in a
247    manner other than that for which it was designed.</p>

251<div class="directive-section"><h2><a name="ScriptLogBuffer" id="ScriptLogBuffer">ScriptLogBuffer</a> <a name="scriptlogbuffer" id="scriptlogbuffer">Directive</a></h2>
252<table class="directive">
253<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Maximum amount of PUT or POST requests that will be recorded
254in the scriptlog</td></tr>
255<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ScriptLogBuffer <var>bytes</var></code></td></tr>
256<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>ScriptLogBuffer 1024</code></td></tr>
257<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host</td></tr>
258<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
259<tr><th><a href="directive-dict.html#Module">Module:</a></th><td><code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code>, <code class="module"><a href="../mod/mod_cgid.html">mod_cgid</a></code></td></tr>
260</table>
261    <p>The size of any PUT or POST entity body that is logged to
262    the file is limited, to prevent the log file growing too big
263    too quickly if large bodies are being received. By default, up
264    to 1024 bytes are logged, but this can be changed with this
265    directive.</p>

269<div class="directive-section"><h2><a name="ScriptLogLength" id="ScriptLogLength">ScriptLogLength</a> <a name="scriptloglength" id="scriptloglength">Directive</a></h2>
270<table class="directive">
271<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Size limit of the CGI script logfile</td></tr>
272<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ScriptLogLength <var>bytes</var></code></td></tr>
273<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>ScriptLogLength 10385760</code></td></tr>
274<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host</td></tr>
275<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
276<tr><th><a href="directive-dict.html#Module">Module:</a></th><td><code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code>, <code class="module"><a href="../mod/mod_cgid.html">mod_cgid</a></code></td></tr>
277</table>
278    <p><code class="directive">ScriptLogLength</code> can be used to limit the
279    size of the CGI script logfile. Since the logfile logs a lot of
280    information per CGI error (all request headers, all script output)
281    it can grow to be a big file. To prevent problems due to unbounded
282    growth, this directive can be used to set an maximum file-size for
283    the CGI logfile. If the file exceeds this size, no more
284    information will be written to it.</p>
