44<h3>概要</h3>
45
46    <p>このモジュールはクライアントリクエストの forensic ロギングを
47    行ないます。ログ収集はリクエストの処理の前と後に行なわれますので、
48    forensic ログは各リクエストに対して二行ログ収集します。
49    Forensic ロガーは非常に厳密です。これは以下のことを意味します:</p>
50
51    <ul>
52    <li>フォーマットは固定です。実行時にロギングフォーマットを変更することは
53    できません。</li>
54    <li>データを書けない場合は子プロセスはその場で終了し、さらにコアを
55    ダンプするかもしれません (<code class="directive"><a href="../mod/mpm_common.html#coredumpdirectory">CoreDumpDirectory</a></code> ディレクティブの設定に依ります)。</li>
56    </ul>
57
58    <p>Forensic ログの出力を検査するためには、
59    配布物の support ディレクトリにある <code>check_forensic</code> 
60    スクリプトが役に立つでしょう。</p>

77<h2><a name="formats" id="formats">Forensic ログフォーマット</a></h2>
78    <p>各リクエストは2回ログ収集されます。最初はリクエストが処理される
79    <em>前</em> (つまり、ヘッダを受け取った後) です。2度目のログは
80    リクエストが処理された<em>後</em>、通常のログ収集と同じときに
81    行なわれます。</p>
82
83    <p>各リクエストを識別するために、リクエストには
84    一意なリクエスト ID が割り当てられます。この forensic ID は
85    フォーマット文字列 <code>%{forensic-id}n</code> を使うことで
86    通常の transfer ログにログ収集することもできます。
87    <code class="module"><a href="../mod/mod_unique_id.html">mod_unique_id</a></code> を使っている場合は、それが生成する
88    ID が使われます。</p>
89
90    <p>最初の行は forensic ID、リクエスト行と受け取ったすべてのヘッダを
91    パイプ文字 (<code>|</code>) で分離してログ収集します。
92    例えば以下のようになります (実際はすべて同じ行になります):</p>
93
94    <div class="example"><p><code>
95        +yQtJf8CoAB4AAFNXBIEAAAAA|GET /manual/de/images/down.gif
96        HTTP/1.1|Host:localhost%3a8080|User-Agent:Mozilla/5.0 (X11;
97        U; Linux i686; en-US; rv%3a1.6) Gecko/20040216
98        Firefox/0.8|Accept:image/png, <var>etc...</var>
99    </code></p></div>
100
101    <p>最初のプラス文字がこのログは最初のログであることを示します。
102    二番目の行はマイナス文字と ID のみです:</p>
103
104    <div class="example"><p><code>
105      -yQtJf8CoAB4AAFNXBIEAAAAA
106    </code></p></div>
107
108    <p><code>check_forensic</code> スクリプトは引数としてログファイルの名前を
109    取ります。<code>+</code>/<code>-</code> の ID の組を調べ、完了していない
110    リクエストがある場合は警告を発します。</p>

113<h2><a name="security" id="security">セキュリティの問題</a></h2>
114    <p>ログファイルが保存されるディレクトリがサーバを起動したユーザ
115    以外で書き込み可能になっているときにセキュリティが破られる可能性が
116    あることについての詳細は<a href="../misc/security_tips.html#serverroot">セキュリティのこつ</a>を
117    参照してください。</p>

120<div class="directive-section"><h2><a name="ForensicLog" id="ForensicLog">ForensicLog</a> <a name="forensiclog" id="forensiclog">ディレクティブ</a></h2>
121<table class="directive">
122<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>Forensic ログのファイル名を設定する</td></tr>
123<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>ForensicLog <var>filename</var>|<var>pipe</var></code></td></tr>
124<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>
125<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
126<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_log_forensic</td></tr>
127</table>
128    <p><code class="directive">ForensicLog</code> ディレクティブは forensic 解析のための
129    サーバへのリクエストをログ収集に使います。
130    各ログエントリには、普通の <code class="directive"><a href="../mod/mod_log_config.html#customlog">CustomLog</a></code> ディレクティブを使ってリクエストと関連付けることの
131    できる
132    一意な ID が割り当てられます。<code class="module"><a href="../mod/mod_log_forensic.html">mod_log_forensic</a></code> は
133    <code>forensic-id</code> というトークンを作成し、フォーマット文字列
134    <code>%{forensic-id}n</code> を使うことでそのトークンを transfer ログに
135    追加することができます。</p>
136
137    <p>引数はログが書き出される位置を指定し、以下の 2種類の値のどちらかを
138    取ることができます:</p>
139
140    <dl>
141      <dt><var>filename</var></dt>
142      <dd><code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> からの
143      相対ファイル名</dd>
144
145      <dt><var>pipe</var></dt>
146      <dd>パイプ文字 "<code>|</code>" と、その後にログ情報を標準入力から
147      受け取るプログラム。プログラム名は <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> からの相対パスとしても
148      指定できます。
149
150      <div class="warning"><h3>セキュリティ:</h3>
151      <p>プログラムを使う場合、そのプログラムは <code class="program"><a href="../programs/httpd.html">httpd</a></code> を起動したユーザで
152      実行されます。つまり、サーバが root で実行された場合は root で
153      実行されるということです。プログラムが安全であるか、より権限の少ない
154      ユーザに切り替えるようになっていることを確かめてください。</p>
155      </div>
156
157      <div class="note"><h3>注</h3>
158        <p>Unix 以外のプラットフォームでファイル名を入力するときは、
159        プラットフォームがバックスラッシュの使用を許可している場合でも、
160        スラッシュのみが使われるように気をつけてください。
161        普通は設定ファイルすべてにおいて、スラッシュの方を使用するように
162        してください。</p>
163      </div></dd>
164    </dl>
