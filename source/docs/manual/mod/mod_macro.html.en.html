37<h3>Summary</h3>
38
39
40    <p>Provides macros within Apache httpd runtime configuration files,
41    to ease the process of creating numerous similar configuration
42    blocks. When the server starts up, the macros are expanded using the
43    provided parameters, and the result is processed as along with the
44    rest of the configuration file.</p>
45

63<h2><a name="usage" id="usage">Usage</a></h2>
64
65<p>Macros are defined using <code class="directive"><a href="#macro">&lt;Macro&gt;</a></code> blocks, which contain the portion of
66your configuration that needs to be repeated, complete with variables
67for those parts that will need to be substituted.</p>
68
69<p>For example, you might use a macro to define a <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code> block, in order to define
70multiple similar virtual hosts:</p>
71
72<pre class="prettyprint lang-config">&lt;Macro VHost $name $domain&gt;
73&lt;VirtualHost *:80&gt;
74    ServerName $domain
75    ServerAlias www.$domain
76
77    DocumentRoot "/var/www/vhosts/$name"
78    ErrorLog "/var/log/httpd/$name.error_log"
79    CustomLog "/var/log/httpd/$name.access_log" combined
80&lt;/VirtualHost&gt;
81&lt;/Macro&gt;</pre>
82
83
84<p>Macro names are case-insensitive, like httpd configuration
85directives. However, variable names are case sensitive.</p>
86
87<p>You would then invoke this macro several times to create virtual
88hosts:</p>
89
90<pre class="prettyprint lang-config">Use VHost example example.com
91Use VHost myhost hostname.org
92Use VHost apache apache.org
93
94UndefMacro VHost</pre>
95
96
97<p>At server startup time, each of these <code class="directive"><a href="#use">Use</a></code>
98invocations would be expanded into a full virtualhost, as
99described by the <code class="directive"><a href="#macro">&lt;Macro&gt;</a></code>
100definition.</p>
101
102<p>The <code class="directive"><a href="#undefmacro">UndefMacro</a></code> directive is
103used so that later macros using the same variable names don't result in
104conflicting definitions.</p>
105
106<p>A more elaborate version of this example may be seen below in the
107Examples section.</p>

111<h2><a name="tips" id="tips">Tips</a></h2>
112
113<p>Parameter names should begin with a sigil such as <code>$</code>,
114<code>%</code>, or <code>@</code>, so that they are clearly
115identifiable, and also in order to help deal with interactions with
116other directives, such as the core <code class="directive"><a href="../mod/core.html#define">Define</a></code> directive. Failure to do so will
117result in a warning. Nevertheless, you are encouraged to have a good
118knowledge of your entire server configuration in order to avoid reusing
119the same variables in different scopes, which can cause confusion.</p>
120
121<p>Parameters prefixed with either <code>$</code> or <code>%</code> are
122not escaped. Parameters prefixes with <code>@</code> are escaped in
123quotes.</p>
124
125<p>Avoid using a parameter which contains another parameter as a prefix,
126(For example, <code>$win</code> and <code>$winter</code>) as this may
127cause confusion at expression evaluation time. In the event of such
128confusion, the longest possible parameter name is used.</p>
129
130<p>If you want to use a value within another string, it is useful to
131surround the parameter in braces, to avoid confusion:</p>
132
133<pre class="prettyprint lang-config">&lt;Macro DocRoot ${docroot}&gt;
134    DocumentRoot "/var/www/${docroot}/htdocs"
135&lt;/Macro&gt;</pre>

143<h3>Virtual Host Definition</h3>
144
145
146<p>A common usage of <code class="module"><a href="../mod/mod_macro.html">mod_macro</a></code> is for the creation of
147dynamically-generated virtual hosts.</p>
148
149<pre class="prettyprint lang-config">## Define a VHost Macro for repetitive configurations
150
151&lt;Macro VHost $host $port $dir&gt;
152  Listen $port
153  &lt;VirtualHost *:$port&gt;
154
155    ServerName $host
156    DocumentRoot "$dir"
157
158    # Public document root
159    &lt;Directory "$dir"&gt;
160        Require all granted
161    &lt;/Directory&gt;
162
163    # limit access to intranet subdir.
164    &lt;Directory "$dir/intranet"&gt;
165      Require ip 10.0.0.0/8
166    &lt;/Directory&gt;
167  &lt;/VirtualHost&gt;
168&lt;/Macro&gt;
169
170## Use of VHost with different arguments.
171
172Use VHost www.apache.org 80 /vhosts/apache/htdocs
173Use VHost example.org 8080 /vhosts/example/htdocs
174Use VHost www.example.fr 1234 /vhosts/example.fr/htdocs</pre>
175
176 
177
178<h3>Removal of a macro definition</h3>
179
180
181<p>It's recommended that you undefine a macro once you've used it. This
182avoids confusion in a complex configuration file where there may be
183conflicts in variable names.</p>
184
185<pre class="prettyprint lang-config">&lt;Macro DirGroup $dir $group&gt;
186  &lt;Directory "$dir"&gt;
187    Require group $group
188  &lt;/Directory&gt;
189&lt;/Macro&gt;
190
191Use DirGroup /www/apache/private private
192Use DirGroup /www/apache/server  admin
193
194UndefMacro DirGroup</pre>

201<div class="directive-section"><h2><a name="Macro" id="Macro">&lt;Macro&gt;</a> <a name="macro" id="macro">Directive</a></h2>
202<table class="directive">
203<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Define a configuration file macro</td></tr>
204<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>
205&lt;Macro <var>name</var> [<var>par1</var> .. <var>parN</var>]&gt;
206... &lt;/Macro&gt;</code></td></tr>
207<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
208<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
209<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_macro</td></tr>
210</table>
211    <p>The <code class="directive">&lt;Macro&gt;</code> directive controls the
212    definition of a macro within the server runtime configuration files.
213    The first argument is the name of the macro.
214    Other arguments are parameters to the macro. It is good practice to prefix
215    parameter names with any of '<code>$%@</code>', and not macro names
216    with such characters.
217    </p>
218
219    <pre class="prettyprint lang-config">&lt;Macro LocalAccessPolicy&gt;
220    Require ip 10.2.16.0/24
221&lt;/Macro&gt;
222
223&lt;Macro RestrictedAccessPolicy $ipnumbers&gt;
224    Require ip $ipnumbers
225&lt;/Macro&gt;</pre>

247<div class="directive-section"><h2><a name="Use" id="Use">Use</a> <a name="use" id="use">Directive</a></h2>
248<table class="directive">
249<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Use a macro</td></tr>
250<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>Use <var>name</var> [<var>value1</var> ... <var>valueN</var>]
251</code></td></tr>
252<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
253<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
254<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_macro</td></tr>
255</table>
256    <p>The <code class="directive">Use</code> directive controls the use of a macro.
257    The specified macro is expanded. It must be given the same number of
258    arguments as in the  macro definition. The provided values are
259    associated to their corresponding initial parameters and are substituted
260    before processing.</p>
261
262    <pre class="prettyprint lang-config">Use LocalAccessPolicy
263...
264Use RestrictedAccessPolicy "192.54.172.0/24 192.54.148.0/24"</pre>
265
266
267    <p>is equivalent, with the macros defined above, to:</p>
268
269    <pre class="prettyprint lang-config">Require ip 10.2.16.0/24
270...
271Require ip 192.54.172.0/24 192.54.148.0/24</pre>
