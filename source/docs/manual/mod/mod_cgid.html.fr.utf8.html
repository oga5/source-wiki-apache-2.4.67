41<h3>Sommaire</h3>
42
43    <p>Exceptées les optimisations et la directive additionnelle
44    <code class="directive"><a href="#scriptsock">ScriptSock</a></code> décrite
45    ci-dessous, <code class="module"><a href="../mod/mod_cgid.html">mod_cgid</a></code> a un comportement similaire à
46    celui de <code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code>. <strong>Voir le résumé de
47    <code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code> pour plus de détails à propos d'Apache et
48    CGI.</strong></p>
49
50    <p>Sur certains systèmes d'exploitation de type unix, le lancement
51    (forking) d'un processus depuis un serveur multi-threadé est une
52    opération très lourde car le nouveau processus va répliquer tous les
53    threads du processus parent. Pour éviter cette dépense de ressouces
54    pour chaque invocation d'un programme CGI, <code class="module"><a href="../mod/mod_cgid.html">mod_cgid</a></code>
55    crée un démon externe qui est responsable du branchement de
56    processus enfants destinés au lancement de scripts CGI. Le serveur
57    principal communique avec ce démon par l'intermédiaire d'une socket
58    de domaine unix.</p>
59
60    <p>Si un MPM multi-threadé a été sélectionné lors du processus de
61    compilation, c'est ce module qui est utilisé par défaut à la place
62    de <code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code>. Du point de vue de l'utilisateur, ce
63    module est identique à <code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code> quant à sa
64    configuration et son utilisation. La seule différence est la
65    directive additionnelle <code>ScriptSock</code> qui permet de
66    définir le nom du socket à utiliser pour la communication avec le
67    démon CGI.</p>

85<div class="directive-section"><h2><a name="cgidscripttimeout" id="cgidscripttimeout">Directive</a> <a name="CGIDScriptTimeout" id="CGIDScriptTimeout">CGIDScriptTimeout</a></h2>
86<table class="directive">
87<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Durée maximale d'attente de la prochaine sortie du
88programme CGI</td></tr>
89<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>CGIDScriptTimeout <var>time</var>[s|ms]</code></td></tr>
90<tr><th><a href="directive-dict.html#Default">Défaut:</a></th><td><code>Si non définie ou définie à 0, valeur de la directive <code class="directive"><a href="../mod/core.html#timeout">Timeout</a></code></code></td></tr>
91<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale, serveur virtuel, répertoire, .htaccess</td></tr>
92<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
93<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_cgid</td></tr>
94<tr><th><a href="directive-dict.html#Compatibility">Compatibilité:</a></th><td>Disponible à partir de la version 2.4.10 du serveur HTTP Apache ;
95dans les versions précédentes, aucune durée d'attente n'était définie</td></tr>
96</table>
97    <p>Cette directive permet de limiter la durée d'attente avant les prochaines données
98    reçues en sortie du programme CGI. Si ce temps est dépassé, la requête et le
99    programme CGI se terminent.</p>
100
101    <div class="example"><h3>Exemple</h3><pre class="prettyprint lang-config">CGIDScriptTimeout 20</pre>
102</div>

107<div class="directive-section"><h2><a name="scriptsock" id="scriptsock">Directive</a> <a name="ScriptSock" id="ScriptSock">ScriptSock</a></h2>
108<table class="directive">
109<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Le préfixe du nom de fichier du socket à utiliser pour
110communiquer avec le démon CGI</td></tr>
111<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>ScriptSock <var>chemin fichier</var></code></td></tr>
112<tr><th><a href="directive-dict.html#Default">Défaut:</a></th><td><code>ScriptSock cgisock</code></td></tr>
113<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>configuration globale</td></tr>
114<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Base</td></tr>
115<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_cgid</td></tr>
116</table>
117    <p>Cette directive permet de définir le préfixe du nom de fichier de la
118    socket à utiliser pour communiquer avec le démon CGI, préfixe auquel
119    sera ajouté une extension correspondant à l'identifiant processus du
120    serveur. La socket sera ouverte avec les permissions de l'utilisateur
121    qui a démarré Apache (en général root). Afin de préserver la
122    sécurité des communications avec les scripts CGI, il est impératif
123    de n'accorder à aucun autre utilisateur la permission d'écrire dans
124    le répertoire où se trouve la socket.</p>
125
126    <p>Si <var>chemin fichier</var> n'est pas un chemin absolu, il est
127    relatif au chemin défini par la directive <code class="directive"><a href="../mod/core.html#defaultruntimedir">DefaultRuntimeDir</a></code>.</p>
128
129    <div class="example"><h3>Exemple</h3><pre class="prettyprint lang-config">ScriptSock /var/run/cgid.sock</pre>
130</div>
