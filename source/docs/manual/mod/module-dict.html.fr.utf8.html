48<h2><a name="Status" id="Status">Statut</a></h2>
49
50    <p>Ce terme indique le degré de rapprochement du module par rapport
51    au coeur du serveur web Apache ; en d'autres termes, vous pouvez
52    être amené à recompiler le serveur pour pouvoir accéder au module et
53    à ses fonctionnalités. Les valeurs possibles de cet attribut sont
54    :</p>
55
56    <dl>
57      <dt>MPM</dt>
58
59      <dd>Un module dont le statut est "MPM" est un <a href="../mpm.html">module Multi-Processus</a>. A la différence des
60      autres modules, un seul module MPM peut et doit être utilisé par Apache à
61      la fois. Ce type de module est responsable de la répartition et du
62      traitement de base des requêtes.</dd>
63
64      <dt>Base</dt>
65
66      <dd>Un module dont le statut est "Base" est compilé dans le
67      serveur et chargé avec ce dernier par défaut ; il est donc
68      toujours disponible à moins que vous n'ayez fait en sorte de
69      supprimer le module de votre configuration.</dd>
70
71      <dt>Extension</dt>
72
73      <dd>Un module dont le statut est "Extension" n'est pas compilé et
74      chargé dans le serveur par défaut. Pour activer le module et
75      accéder à ses fonctionnalités, vous devez modifier la
76      configuration de la compilation du serveur et recompiler
77      Apache.</dd>
78
79      <dt>Expérimental</dt>
80
81      <dd>Le statut "Experimental" indique que le module fait partie du
82      kit Apache, mais que vous devez l'utiliser à vos risques et
83      périls. Le module est documenté à des fins d'exhaustivité, et
84      n'est pas obligatoirement supporté.</dd>
85
86      <dt>Externe</dt>
87
88      <dd>Ce statut indique que le module ("module tiers") ne fait pas
89      partie de la distribution de base d'Apache. Nous ne sommes pas
90      responsables de ces modules et n'en assurons pas le support.</dd>
91    </dl>

94<h2><a name="SourceFile" id="SourceFile">Fichier source</a></h2>
95
96    <p>Il s'agit tout simplement de la liste des noms des fichiers
97    source qui contiennent le code du module. C'est aussi le nom utilisé
98    par la directive <code class="directive"><a href="../mod/core.html#ifmodule">&lt;IfModule&gt;</a></code>.</p>
99</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>
100<div class="section">
101<h2><a name="ModuleIdentifier" id="ModuleIdentifier">Identificateur de module</a></h2>
102
103    <p>C'est une chaîne permettant d'identifier le module à utiliser
104    dans la directive <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code>
105    pour le chargement dynamique des modules. En particulier, c'est le
106    nom de la variable externe de type module dans le fichier
107    source.</p>

110<h2><a name="Compatibility" id="Compatibility">Compatibilité</a></h2>
111
112    <p>Si le module ne faisait pas partie de la distribution originale
113    d'Apache version 2, la version à partir de laquelle il est
114    disponible est indiquée ici. En outre, si le module n'est disponible
115    que sur certaines plates-formes, cela sera mentionné ici.</p>
