72switch before dumping core</td></tr>
73<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>CoreDumpDirectory <var>directory</var></code></td></tr>
74<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>See usage for the default setting</code></td></tr>

78</table>
79    <p>This controls the directory to which Apache httpd attempts to
80    switch before dumping core. If your operating system is configured to
81    create core files in the working directory of the crashing process,
82    <code class="directive">CoreDumpDirectory</code> is necessary to change working
83    directory from the default <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>
84    directory, which should not be writable by the user the server runs as.</p>
85
86    <p>If you want a core dump for debugging, you can use this directive to
87    place it in a different location. This directive has no effect if your
88    operating system is not configured to write core files to the working directory
89    of the crashing processes.</p>
90
91    <div class="warning">
92      <h3>Security note for Linux systems</h3>
93
94      <p>Using this directive on Linux may allow other processes on
95      the system (if running with similar privileges, such as CGI
96      scripts) to attach to httpd children via the <code>ptrace</code>
97      system call.  This may make weaken the protection from certain
98      security attacks.  It is not recommended to use this directive
99      on production systems.</p>
100    </div>
101    
102    <div class="note"><h3>Core Dumps on Linux</h3>
103      <p>If Apache httpd starts as root and switches to another user, the
104      Linux kernel <em>disables</em> core dumps even if the directory is
105      writable for the process. Apache httpd (2.0.46 and later) reenables core dumps
106      on Linux 2.4 and beyond, but only if you explicitly configure a <code class="directive">CoreDumpDirectory</code>.</p>
107    </div>
108
109    <div class="note">
110    <h3>Core Dumps on BSD</h3>
111    <p>To enable core-dumping of suid-executables on BSD-systems (such
112    as FreeBSD), set <code>kern.sugid_coredump</code> to 1.
113    </p>
114    </div>
115
116    <div class="note"><h3>Specific signals</h3>
117      <p><code class="directive">CoreDumpDirectory</code> processing only occurs for
118      a select set of fatal signals: SIGFPE, SIGILL, SIGABORT,
119      SIGSEGV, and SIGBUS.</p>
120      <p>On some operating systems, SIGQUIT also results in a core dump but
121      does not go through <code class="directive">CoreDumpDirectory</code> or
122      <code class="directive">EnableExceptionHook</code> processing, so the core
123      location is dictated entirely by the operating system.</p>
124    </div>

170<div class="directive-section"><h2><a name="Listen" id="Listen">Listen</a> <a name="listen" id="listen">Directive</a></h2>
171<table class="directive">
172<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>IP addresses and ports that the server
173listens to</td></tr>
174<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>Listen [<var>IP-address</var>:]<var>portnumber</var> [<var>protocol</var>]</code></td></tr>
175<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config</td></tr>
176<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>MPM</td></tr>
177<tr><th><a href="directive-dict.html#Module">Module:</a></th><td><code class="module"><a href="../mod/event.html">event</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code>, <code class="module"><a href="../mod/prefork.html">prefork</a></code>, <code class="module"><a href="../mod/mpm_winnt.html">mpm_winnt</a></code>, <code class="module"><a href="../mod/mpm_netware.html">mpm_netware</a></code>, <code class="module"><a href="../mod/mpmt_os2.html">mpmt_os2</a></code></td></tr>
178<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>The <var>protocol</var> argument was added in 2.1.5</td></tr>
179</table>
180    <p>The <code class="directive">Listen</code> directive instructs Apache httpd to
181    listen to only specific IP addresses or ports; by default it
182    responds to requests on all IP interfaces. <code class="directive">Listen</code>
183    is now a required directive. If it is not in the config file, the
184    server will fail to start. This is a change from previous versions
185    of Apache httpd.</p>
186
187    <p>The <code class="directive">Listen</code> directive tells the server to
188    accept incoming requests on the specified port or address-and-port
189    combination. If only a port number is specified, the server listens to
190    the given port on all interfaces. If an IP address is given as well
191    as a port, the server will listen on the given port and
192    interface.</p>
193
194    <p>Multiple <code class="directive">Listen</code> directives may be used to
195    specify a number of addresses and ports to listen to. The server will
196    respond to requests from any of the listed addresses and ports.</p>
197
198    <p>For example, to make the server accept connections on both
199    port 80 and port 8000, use:</p>
200
201    <pre class="prettyprint lang-config">Listen 80
202Listen 8000</pre>
203
204
205    <p>To make the server accept connections on two specified
206    interfaces and port numbers, use </p>
207
208    <pre class="prettyprint lang-config">Listen 192.170.2.1:80
209Listen 192.170.2.5:8000</pre>
210
211
212    <p>IPv6 addresses must be surrounded in square brackets, as in the
213    following example:</p>
214
215    <pre class="prettyprint lang-config">Listen [2001:db8::a00:20ff:fea7:ccea]:80</pre>
216
217
218    <p>The optional <var>protocol</var> argument is not required for most
219       configurations. If not specified, <code>https</code> is the default for
220       port 443 and <code>http</code> the default for all other ports.  The
221       protocol is used to determine which module should handle a request, and
222       to apply protocol specific optimizations with the
223       <code class="directive"><a href="../mod/core.html#acceptfilter">AcceptFilter</a></code> directive.</p>
224
225    <p>You only need to set the protocol if you are running on non-standard
226       ports.  For example, running an <code>https</code> site on port 8443:</p>
227
228    <pre class="prettyprint lang-config">Listen 192.170.2.1:8443 https</pre>
229
230
231    <div class="note"><h3>Error condition</h3>
232      Multiple <code class="directive">Listen</code> directives for the same ip
233      address and port will result in an <code>Address already in use</code>
234      error message.
235    </div>

272<div class="directive-section"><h2><a name="ListenCoresBucketsRatio" id="ListenCoresBucketsRatio">ListenCoresBucketsRatio</a> <a name="listencoresbucketsratio" id="listencoresbucketsratio">Directive</a></h2>
273<table class="directive">
274<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Ratio between the number of CPU cores (online) and the number of
275listeners' buckets</td></tr>
276<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ListenCoresBucketsRatio <var>ratio</var></code></td></tr>
277<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>ListenCoresBucketsRatio 0 (disabled)</code></td></tr>
278<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config</td></tr>
279<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>MPM</td></tr>
280<tr><th><a href="directive-dict.html#Module">Module:</a></th><td><code class="module"><a href="../mod/event.html">event</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code>, <code class="module"><a href="../mod/prefork.html">prefork</a></code></td></tr>
281<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Available in Apache HTTP Server 2.4.17, with a kernel supporting
282the socket option <code>SO_REUSEPORT</code> and distributing new connections
283evenly across listening processes' (or threads') sockets using it (eg. Linux
2843.9 and later, but not the current implementations of <code>SO_REUSEPORT</code>
285in *BSDs.</td></tr>
286</table>
287    <p>A <var>ratio</var> between the number of (online) CPU cores and the
288    number of listeners' buckets can be used to make Apache HTTP Server create
289    <code>num_cpu_cores / ratio</code> listening buckets, each containing its
290    own <code class="directive"><a href="#listen">Listen</a></code>-ing socket(s) on the same port(s), and
291    then make each child handle a single bucket (with round-robin distribution
292    of the buckets at children creation time).</p>
293
294    <div class="note"><h3>Meaning of "online" CPU core</h3>
295    <p>On Linux (and also BSD) a CPU core can be turned on/off if 
296    <a href="https://www.kernel.org/doc/Documentation/cpu-hotplug.txt">Hotplug</a> 
297    is configured, therefore <code class="directive">ListenCoresBucketsRatio</code> needs to
298    take this parameter into account while calculating the number of buckets to create.</p>
299    </div>
300
301    <p><code class="directive">ListenCoresBucketsRatio</code> can improve the
302    scalability when accepting new connections is/becomes the bottleneck.
303    On systems with a large number of CPU cores, enabling this feature has
304    been tested to show significant performances improvement and shorter
305    responses time.</p>
306
307    <p>There must be at least twice the number of CPU cores than the
308    configured <var>ratio</var> for this to be active. The recommended
309    <var>ratio</var> is <code>8</code>, hence at least <code>16</code>
310    cores should be available at runtime when this value is used.
311    The right <var>ratio</var> to obtain maximum performance needs to be calculated
312    for each target system, testing multiple values and observing the variations in your 
313    key performance metrics.</p>
314
315    <p>This directive influences the calculation of the 
316    <code class="directive"><a href="#minsparethreads">MinSpareThreads</a></code> and 
317    <code class="directive"><a href="#maxsparethreads">MaxSpareThreads</a></code> lower bound values.
318    The number of children processes needs to be a multiple of the number 
319    of buckets to optimally accept connections.</p>
320
321    <div class="note">
322    <h3>Multiple <code class="directive">Listen</code>ers or Apache HTTP servers on
323    the same IP address and port</h3>
324    <p>Setting the <code>SO_REUSEPORT</code> option on the listening socket(s)
325    consequently allows multiple processes (sharing the same <code>EUID</code>,
326    e.g. <code>root</code>) to bind to the the same IP address and port,
327    without the binding error raised by the system in the usual case.</p>
328    <p>This also means that multiple instances of Apache httpd configured on a
329    same <code>IP:port</code> and with a positive <code class="directive">ListenCoresBucketsRatio</code>
330    would start without an error too, and then run with incoming connections
331    evenly distributed across both instances (this is NOT a recommendation or
332    a sensible usage in any case, but just a notice that it would prevent such
333    possible issues to be detected).</p>
334    <p>Within the same instance, Apache httpd will check and fail to start if
335    multiple <code class="directive">Listen</code> directives on the exact same IP (or
336    hostname) and port are configured, thus avoiding the creation of some

631<tr><th><a href="directive-dict.html#Module">Module:</a></th><td><code class="module"><a href="../mod/event.html">event</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code>, <code class="module"><a href="../mod/prefork.html">prefork</a></code>, <code class="module"><a href="../mod/mpm_winnt.html">mpm_winnt</a></code>, <code class="module"><a href="../mod/mpm_netware.html">mpm_netware</a></code>, <code class="module"><a href="../mod/mpmt_os2.html">mpmt_os2</a></code></td></tr>
632</table>
633    <p>Sets the server's TCP send buffer size to the number of bytes
634    specified. It is often useful to set this past the OS's standard
635    default value on high speed, high latency connections
636    (<em>i.e.</em>, 100ms or so, such as transcontinental fast pipes).</p>
637
638    <p>If set to the value of <code>0</code>, the server will use the
639    default value provided by your OS. If set to a non-zero value,
640    it must be at least <code>512</code>.</p>
641
642    <p>Further configuration of your operating system may be required to elicit
643    better performance on high speed, high latency connections.</p>
644
645    <div class="note"> <p> On some operating systems, changes in TCP behavior resulting
646    from a larger <code class="directive">SendBufferSize</code> may not be seen unless
647    <code class="directive"><a href="../mod/core.html#enablesendfile">EnableSendfile</a></code> is set to OFF.  This
648    interaction applies only to static files.</p> </div>

654<table class="directive">
655<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Upper limit on configurable number of processes</td></tr>
656<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ServerLimit <var>number</var></code></td></tr>
657<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>See usage for details</code></td></tr>
658<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config</td></tr>
659<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>MPM</td></tr>
660<tr><th><a href="directive-dict.html#Module">Module:</a></th><td><code class="module"><a href="../mod/event.html">event</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code>, <code class="module"><a href="../mod/prefork.html">prefork</a></code></td></tr>
661</table>
662    <p>For the <code class="module"><a href="../mod/prefork.html">prefork</a></code> MPM, this directive sets the
663    maximum configured value for <code class="directive"><a href="#maxrequestworkers">MaxRequestWorkers</a></code> for the lifetime of the
664    Apache httpd process. For the <code class="module"><a href="../mod/worker.html">worker</a></code> and <code class="module"><a href="../mod/event.html">event
665    </a></code> MPMs, this directive in combination with <code class="directive"><a href="#threadlimit">ThreadLimit</a></code> sets
666    the maximum configured value for <code class="directive"><a href="#maxrequestworkers">MaxRequestWorkers</a></code> for the lifetime of the
667    Apache httpd process. For the <code class="module"><a href="../mod/event.html">event</a></code> MPM, this directive
668    also defines how many old server processes may keep running and finish processing
669    open connections.
670    Any attempts to change this directive during a restart will be ignored, but
671    <code class="directive"><a href="#maxrequestworkers">MaxRequestWorkers</a></code> can be modified
672    during a restart.</p>
673
674    <p>Special care must be taken when using this directive.  If
675    <code class="directive">ServerLimit</code> is set to a value much higher
676    than necessary, extra, unused shared memory will be allocated.  If
677    both <code class="directive">ServerLimit</code> and <code class="directive"><a href="#maxrequestworkers">MaxRequestWorkers</a></code> are set to values
678    higher than the system can handle, Apache httpd may not start or the
679    system may become unstable.</p>
680
681    <p>With the <code class="module"><a href="../mod/prefork.html">prefork</a></code> MPM, use this directive only
682    if you need to set <code class="directive"><a href="#maxrequestworkers">MaxRequestWorkers</a></code> higher than 256 (default).
683    Do not set the value of this directive any higher than what you
684    might want to set <code class="directive"><a href="#maxrequestworkers">MaxRequestWorkers</a></code> to.</p>
685
686    <p>With <code class="module"><a href="../mod/worker.html">worker</a></code>, use this directive only if your
687    <code class="directive"><a href="#maxrequestworkers">MaxRequestWorkers
688    </a></code> and <code class="directive"><a href="#threadsperchild">ThreadsPerChild</a></code>
689    settings require more than 16 server processes (default). Do not set
690    the value of this directive any higher than the number of server
691    processes required by what you may want for <code class="directive"><a href="#maxrequestworkers">MaxRequestWorkers </a></code> and <code class="directive"><a href="#threadsperchild">ThreadsPerChild</a></code>.</p>
692
693    <p>With <code class="module"><a href="../mod/event.html">event</a></code>, increase this directive if the process
694    number defined by your <code class="directive"><a href="#maxrequestworkers">MaxRequestWorkers</a></code> and <code class="directive"><a href="#threadsperchild">ThreadsPerChild</a></code> settings, plus the
695    number of gracefully shutting down processes, is more than 16 server
696    processes (default).</p>
697
698    <div class="note"><h3>Note</h3>
699      <p>There is a hard limit of <code>ServerLimit 20000</code> compiled
700      into the server (for the <code class="module"><a href="../mod/prefork.html">prefork</a></code> MPM 200000). This is
701      intended to avoid nasty effects caused by typos. To increase it
702      even further past this limit, you will need to modify the value of
703      MAX_SERVER_LIMIT in the mpm source file and rebuild the server.</p>
704    </div>
705
706    <p>The value of <code class="directive">ServerLimit</code> must be at least
707    1. Values less than 1 will be automatically increased to 1 and a
708    warning will be logged.</p>

849<div class="directive-section"><h2><a name="ThreadStackSize" id="ThreadStackSize">ThreadStackSize</a> <a name="threadstacksize" id="threadstacksize">Directive</a></h2>
850<table class="directive">
851<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>The size in bytes of the stack used by threads handling
852client connections</td></tr>
853<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ThreadStackSize <var>size</var></code></td></tr>
854<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>65536 on NetWare; varies on other operating systems</code></td></tr>
855<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config</td></tr>
856<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>MPM</td></tr>
857<tr><th><a href="directive-dict.html#Module">Module:</a></th><td><code class="module"><a href="../mod/event.html">event</a></code>, <code class="module"><a href="../mod/worker.html">worker</a></code>, <code class="module"><a href="../mod/mpm_winnt.html">mpm_winnt</a></code>, <code class="module"><a href="../mod/mpm_netware.html">mpm_netware</a></code>, <code class="module"><a href="../mod/mpmt_os2.html">mpmt_os2</a></code></td></tr>
858<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Available in Apache HTTP Server 2.1 and later</td></tr>
859</table>
860    <p>The <code class="directive">ThreadStackSize</code> directive sets the
861    size of the stack (for autodata) of threads which handle client
862    connections and call modules to help process those connections.
863    In most cases the operating system default for stack size is
864    reasonable, but there are some conditions where it may need to be
865    adjusted:</p>
866
867    <ul>
868      <li>On platforms with a relatively small default thread stack size
869      (e.g., HP-UX), Apache httpd may crash when using some third-party modules
870      which use a relatively large amount of autodata storage.  Those
871      same modules may have worked fine on other platforms where the
872      default thread stack size is larger.  This type of crash is
873      resolved by setting <code class="directive">ThreadStackSize</code> to a
874      value higher than the operating system default.  This type of
875      adjustment is necessary only if the provider of the third-party
876      module specifies that it is required, or if diagnosis of an Apache httpd
877      crash indicates that the thread stack size was too small.</li>
878
879      <li>On platforms where the default thread stack size is
880      significantly larger than necessary for the web server
881      configuration, a higher number of threads per child process
882      will be achievable if <code class="directive">ThreadStackSize</code> is
883      set to a value lower than the operating system default.  This type
884      of adjustment should only be made in a test environment which allows
885      the full set of web server processing to be exercised, as there
886      may be infrequent requests which require more stack to process.
887      The minimum required stack size strongly depends on the modules
888      used, but any change in the web server configuration can invalidate
889      the current <code class="directive">ThreadStackSize</code> setting.</li>
890
891      <li>On Linux, this directive can only be used to increase the default
892      stack size, as the underlying system call uses the value as a
893      <em>minimum</em> stack size.  The (often large) soft limit for
894      <code>ulimit -s</code> (8MB if unlimited) is used as the default stack
895      size.</li>
896    </ul>
897
898    <div class="note">It is recommended to not reduce <code class="directive">ThreadStackSize</code>
899    unless a high number of threads per child process is needed. On some
900    platforms (including Linux), a setting of 128000 is already too low and
901    causes crashes with some common modules.</div>
