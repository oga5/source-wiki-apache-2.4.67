37<h3>Summary</h3>
38
39    <p>This module determines the <a class="glossarylink" href="../glossary.html#mime-type" title="see glossary">MIME
40    type</a> of files in the same way the Unix
41    <code>file(1)</code> command works: it looks at the first few
42    bytes of the file. It is intended as a "second line of defense"
43    for cases that <code class="module"><a href="../mod/mod_mime.html">mod_mime</a></code> can't resolve.</p>
44
45    <p>This module is derived from a free version of the
46    <code>file(1)</code> command for Unix, which uses "magic
47    numbers" and other hints from a file's contents to figure out
48    what the contents are. This module is active only if the magic
49    file is specified by the <code class="directive"><a href="#mimemagicfile">MimeMagicFile</a></code> directive.</p>

65<h2><a name="format" id="format">Format of the Magic File</a></h2>
66
67    <p>The contents of the file are plain ASCII text in 4-5
68    columns. Blank lines are allowed but ignored. Commented lines
69    use a hash mark (<code>#</code>). The remaining lines are parsed for
70    the following columns:</p>
71
72    <table class="bordered"><tr class="header"><th>Column</th><th>Description</th></tr>
73<tr><td>1</td>
74        <td>byte number to begin checking from<br />
75         "<code>&gt;</code>" indicates a dependency upon the previous
76         non-"<code>&gt;</code>" line</td></tr>
77<tr class="odd"><td>2</td>
78        <td><p>type of data to match</p>
79        <table class="bordered">
80        
81        <tr><td><code>byte</code></td>
82            <td>single character</td></tr>
83        <tr><td><code>short</code></td>
84            <td>machine-order 16-bit integer</td></tr>
85        <tr><td><code>long</code></td>
86            <td>machine-order 32-bit integer</td></tr>
87        <tr><td><code>string</code></td>
88            <td>arbitrary-length string</td></tr>
89        <tr><td><code>date</code></td>
90            <td>long integer date (seconds since Unix epoch/1970)</td></tr>
91        <tr><td><code>beshort</code></td>
92            <td>big-endian 16-bit integer</td></tr>
93        <tr><td><code>belong</code></td>
94            <td>big-endian 32-bit integer</td></tr>
95        <tr><td><code>bedate</code></td>
96            <td>big-endian 32-bit integer date</td></tr>
97        <tr><td><code>leshort</code></td>
98            <td>little-endian 16-bit integer</td></tr>
99        <tr><td><code>lelong</code></td>
100            <td>little-endian 32-bit integer</td></tr>
101        <tr><td><code>ledate</code></td>
102            <td>little-endian 32-bit integer date</td></tr>
103        </table></td></tr>
104<tr><td>3</td>
105        <td>contents of data to match</td></tr>
106<tr class="odd"><td>4</td>
107        <td>MIME type if matched</td></tr>
108<tr><td>5</td>
109        <td>MIME encoding if matched (optional)</td></tr>
110</table>
111
112    <p>For example, the following magic file lines would recognize
113    some audio formats:</p>
114
115    <div class="example"><pre># Sun/NeXT audio data
1160      string      .snd
117&gt;12    belong      1       audio/basic
118&gt;12    belong      2       audio/basic
119&gt;12    belong      3       audio/basic
120&gt;12    belong      4       audio/basic
121&gt;12    belong      5       audio/basic
122&gt;12    belong      6       audio/basic
123&gt;12    belong      7       audio/basic
124&gt;12    belong     23       audio/x-adpcm</pre></div>
125
126    <p>Or these would recognize the difference between <code>*.doc</code>
127    files containing Microsoft Word or FrameMaker documents. (These are
128    incompatible file formats which use the same file suffix.)</p>
129
130    <div class="example"><pre># Frame
1310  string  \&lt;MakerFile        application/x-frame
1320  string  \&lt;MIFFile          application/x-frame
1330  string  \&lt;MakerDictionary  application/x-frame
1340  string  \&lt;MakerScreenFon   application/x-frame
1350  string  \&lt;MML              application/x-frame
1360  string  \&lt;Book             application/x-frame
1370  string  \&lt;Maker            application/x-frame
138
139# MS-Word
1400  string  \376\067\0\043            application/msword
1410  string  \320\317\021\340\241\261  application/msword
1420  string  \333\245-\0\0\0           application/msword</pre></div>
143
144    <p>An optional MIME encoding can be included as a fifth column.
145    For example, this can recognize gzipped files and set the
146    encoding for them.</p>
147
148    <div class="example"><pre># gzip (GNU zip, not to be confused with
149#       [Info-ZIP/PKWARE] zip archiver)
150
1510  string  \037\213  application/octet-stream  x-gzip</pre></div>

154<h2><a name="performance" id="performance">Performance Issues</a></h2>
155    <p>This module is not for every system. If your system is barely
156    keeping up with its load or if you're performing a web server
157    benchmark, you may not want to enable this because the
158    processing is not free.</p>
159
160    <p>However, an effort was made to improve the performance of
161    the original <code>file(1)</code> code to make it fit in a busy web
162    server. It was designed for a server where there are thousands of users
163    who publish their own documents. This is probably very common
164    on intranets. Many times, it's helpful if the server can make
165    more intelligent decisions about a file's contents than the
166    file name allows ...even if just to reduce the "why doesn't my
167    page work" calls when users improperly name their own files.
168    You have to decide if the extra work suits your
169    environment.</p>

256<table class="directive">
257<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Enable MIME-type determination based on file contents
258using the specified magic file</td></tr>
259<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>MimeMagicFile <var>file-path</var></code></td></tr>

264    <p>The <code class="directive">MimeMagicFile</code> directive can be used to
265    enable this module, the default file is distributed at
266    <code>conf/magic</code>. Non-rooted paths are relative to the
267    <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>. Virtual hosts will use
268    the same file as the main server unless a more specific setting is
269    used, in which case the more specific setting overrides the main
270    server's file.</p>
