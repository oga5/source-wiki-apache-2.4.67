42<h3>概要</h3>
43
44    <div class="warning">このモジュールは <code class="directive"><a href="../mod/mod_authz_host.html#allow">Allow</a></code> や <code class="directive"><a href="../mod/mod_authz_host.html#deny">Deny</a></code> ディレクティブを無視しますので、
45    注意して使って下さい。クライアントのホスト名、アドレスや環境変数を使って
46    アクセスを制限したいコンテンツに対してはキャッシュ機能を有効にするべきではありません。
47    </div>
48
49    <p><code class="module"><a href="../mod/mod_cache.html">mod_cache</a></code> はローカルのコンテンツやプロキシされた
50    コンテンツをキャッシュするために使われる <a href="http://www.ietf.org/rfc/rfc2616.txt">RFC 2616</a> 準拠の
51    HTTP コンテンツキャッシュを実装しています。<code class="module"><a href="../mod/mod_cache.html">mod_cache</a></code>
52    の動作にはストレージを管理するモジュールが必要です。標準
53    Apache 配布には二つストレージ管理モジュールが含まれています:</p>
54    
55    <dl>
56    <dt><code class="module"><a href="../mod/mod_cache_disk.html">mod_cache_disk</a></code></dt>
57    <dd>ディスクを使用したストレージ管理機構を実装しています。</dd>
58
59    <dt><code class="module"><a href="../mod/mod_mem_cache.html">mod_mem_cache</a></code></dt>
60    <dd>メモリを使用したストレージ管理機構を実装しています。
61    <code class="module"><a href="../mod/mod_mem_cache.html">mod_mem_cache</a></code> は次の二つのモードのどちらかで動作する
62    ように設定できます: オープンされているファイル記述子をキャッシュするモードか、
63    ヒープ上でのオブジェクトの自体をキャッシュをするモードです。
64    <code class="module"><a href="../mod/mod_mem_cache.html">mod_mem_cache</a></code> はローカルで生成されるコンテンツや、
65    <code class="module"><a href="../mod/mod_proxy.html">mod_proxy</a></code> が
66    <code class="directive"><a href="../mod/mod_proxy.html#proxypass">ProxyPass</a></code> を使って設定されている
67    ときの (つまり<dfn>リバースプロキシ</dfn> での) バックエンドサーバの
68    コンテンツをキャッシュするのに使えます。</dd>
69    </dl>
70
71    <p>コンテンツのキャッシュへの保存と取得は URI に基づいたキーが使われます。
72    アクセス保護のかけられているコンテンツはキャッシュされません。</p>
73    <p>より詳細な解説や例については<a href="../caching.html">キャッシュ機能</a>
74    を参照してください。</p>

115<h2><a name="sampleconf" id="sampleconf">サンプル設定</a></h2>
116    <div class="example"><h3>Sample httpd.conf</h3><p><code>
117      #<br />
118      # Sample Cache Configuration<br />
119      #<br />
120      LoadModule cache_module modules/mod_cache.so<br />
121      <br />
122      &lt;IfModule mod_cache.c&gt;<br />
123      <span class="indent">
124        #LoadModule cache_disk_module modules/mod_cache_disk.so<br />
125        # If you want to use mod_cache_disk instead of mod_mem_cache,<br />
126        # uncomment the line above and comment out the LoadModule line below.<br />
127        &lt;IfModule mod_cache_disk.c&gt;<br />
128        <span class="indent">
129          CacheRoot c:/cacheroot<br />
130          CacheEnable disk  /<br />
131          CacheDirLevels 5<br />
132          CacheDirLength 3<br />
133        </span>
134        &lt;/IfModule&gt; <br />
135        <br />
136        LoadModule mem_cache_module modules/mod_mem_cache.so<br />
137        &lt;IfModule mod_mem_cache.c&gt;<br />
138        <span class="indent">
139          CacheEnable mem  /<br />
140          MCacheSize 4096<br />
141          MCacheMaxObjectCount 100<br />
142          MCacheMinObjectSize 1<br />
143          MCacheMaxObjectSize 2048<br />
144        </span>
145        &lt;/IfModule&gt;<br />
146        <br />
147        # When acting as a proxy, don't cache the list of security updates<br />
148        CacheDisable http://security.update.server/update-list/<br />
149      </span>
150      &lt;/IfModule&gt;
151    </code></p></div>

205<div class="directive-section"><h2><a name="CacheEnable" id="CacheEnable">CacheEnable</a> <a name="cacheenable" id="cacheenable">ディレクティブ</a></h2>
206<table class="directive">
207<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>指定したストレージ管理方式を使ってのキャッシュを有効にする</td></tr>
208<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>CacheEnable <var>cache_type</var> <var>url-string</var></code></td></tr>
209<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>
210<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
211<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_cache</td></tr>
212</table>
213    <p><code class="directive">CacheEnable</code> ディレクティブで <code class="module"><a href="../mod/mod_cache.html">mod_cache</a></code> 
214    モジュールが <var>url-string</var> 以下の URL をキャッシュするようにします。
215    キャッシュストレージ管理方式は <var>cache_type</var> 引数で指定します。
216    <var>cache_type</var> <code> mem</code> で、
217    <code class="module"><a href="../mod/mod_mem_cache.html">mod_mem_cache</a></code> で実装されているメモリを使ったストレージ
218    管理方式を使うように <code class="module"><a href="../mod/mod_cache.html">mod_cache</a></code> に指示します。
219    <var>cache_type</var> <code>disk</code> で、
220    <code class="module"><a href="../mod/mod_cache_disk.html">mod_cache_disk</a></code> で実装されているディスクを使ったストレージ
221    管理を使うように <code class="module"><a href="../mod/mod_cache.html">mod_cache</a></code> に指示します。
222    <var>cache_type</var> <code>fd</code> は <code class="module"><a href="../mod/mod_cache.html">mod_cache</a></code> に
223    <code class="module"><a href="../mod/mod_mem_cache.html">mod_mem_cache</a></code> により実装されているファイル記述子の
224    キャッシュを使うように指示します。</p>
225
226    <p>(下の例のように) <code class="directive">CacheEnable</code> ディレクティブの
227    URL 空間が重複しているときは、該当するストレージ方式を順に試して、
228    実際にリクエストの処理ができると、その方式で処理します。
229    ストレージ管理方式が実行される順番は設定ファイル中の
230    <code class="directive">CacheEnable</code> の順番により決定されます。</p>
231
232    <div class="example"><p><code>
233      CacheEnable  mem   /manual<br />
234      CacheEnable  fd    /images<br />
235      CacheEnable  disk  /<br />
236    </code></p></div>
237
238    <p>フォワードプロクシサーバとして動作する場合、
239    <var>url-string</var> を使って、キャッシュを有効にするリモートサイトや
240    プロクシプロトコルを指定することもできます。</p>
241
242    <div class="example"><p><code>
243      # Cache proxied url's<br />
244      CacheEnable  disk  /<br /><br />
245      # Cache FTP-proxied url's<br />
246      CacheEnable  disk  ftp://<br /><br />
247      # Cache content from www.apache.org<br />
248      CacheEnable  disk  http://www.apache.org/<br />
249    </code></p></div>

313    <p>RFC 2616 によると、hop-by-hop HTTP ヘッダはキャッシュには保管されません。
314    以下のヘッダは hop-by-hop ヘッダに該当しますので、
315    <code class="directive">CacheIgnoreHeaders</code>

319      <li><code>Keep-Alive</code></li>
320      <li><code>Proxy-Authenticate</code></li>
321      <li><code>Proxy-Authorization</code></li>
322      <li><code>TE</code></li>
323      <li><code>Trailers</code></li>
324      <li><code>Transfer-Encoding</code></li>
325      <li><code>Upgrade</code></li>
326    </ul>
327

329    キャッシュに保管しない追加の HTTP ヘッダを指定します。
330    例えば、クッキーをキャッシュに保管しないようにした方がよい場合も
331    あるでしょう。</p>
332
333    <p><code class="directive">CacheIgnoreHeaders</code> の引数は、
334    キャッシュに保管しない HTTP ヘッダを空白区切りにしたリスト形式です。
335    キャッシュに保管しないヘッダが hop-by-hop ヘッダだけの場合
336    (RFC 2616 準拠の動作のとき) は、
337    <code class="directive">CacheIgnoreHeaders</code> を <code>None</code>
338    に設定できます。</p>
339
340    <div class="example"><h3>例 1</h3><p><code>
341      CacheIgnoreHeaders Set-Cookie
342    </code></p></div>
343
344    <div class="example"><h3>例 2</h3><p><code>
345      CacheIgnoreHeaders None
346    </code></p></div>
347
348    <div class="warning"><h3>警告:</h3>
349      <code>Expires</code> のような適切のキャッシュ管理のために必要な
350      ヘッダが <code class="directive">CacheIgnoreHeaders</code> の設定により
351      保管されていないときは、mod_cache の動作は定義されていません。
352    </div>

431<div class="directive-section"><h2><a name="CacheLastModifiedFactor" id="CacheLastModifiedFactor">CacheLastModifiedFactor</a> <a name="cachelastmodifiedfactor" id="cachelastmodifiedfactor">ディレクティブ</a></h2>
432<table class="directive">
433<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>LastModified の日付に基づいて有効期限 (expiry) 
434を計算するための重みを指定する
435</td></tr>
436<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>CacheLastModifiedFactor <var>float</var></code></td></tr>
437<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>CacheLastModifiedFactor 0.1</code></td></tr>
438<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>
439<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
440<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_cache</td></tr>
441</table>
442    <p>ドキュメントに Last-Modified の日付が無いけれども有効期限 (expiry) 
443    の日付があるというときに、有効期限を最終修正時刻からの経過時間として
444    計算するようにできます。有効期限を次の計算式に従って生成するのですが、
445    そのときに使われる <var>factor</var> を
446    <code class="directive">CacheLastModifiedFactor</code> ディレクティブで指定します。
447    </p>
448
449    <p><code>expiry-period = time-since-last-modified-date * <var>factor</var>
450    expiry-date = current-date + expiry-period</code></p>
451
452    <p>例えば、ドキュメントが 10 時間前に最後に修正されていて、
453    <var>factor</var> が 0.1 であれば、期日は 10*0.1 = 1 時間に
454    設定されます。現在時刻が 3:00pm であれば、計算された期日は
455    3:00pm + 1hour = 4:00pm になります。</p>
456
457    <p>期日が <code class="directive">CacheMaxExpire</code> で設定されている値
458    より大きくなってしまっている場合は、<code class="directive">CacheMaxExpire</code>
459    の設定値が優先されます。</p>
460
461    <div class="example"><p><code>
462      CacheLastModifiedFactor 0.5
463    </code></p></div>

583<div class="directive-section"><h2><a name="CacheStoreNoStore" id="CacheStoreNoStore">CacheStoreNoStore</a> <a name="cachestorenostore" id="cachestorenostore">ディレクティブ</a></h2>
584<table class="directive">
585<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>no-store と指定されているレスポンスのキャッシュを試みる。</td></tr>
586<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>CacheStoreNoStore On|Off</code></td></tr>
587<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>CacheStoreNoStore Off</code></td></tr>
588<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>
589<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Extension</td></tr>
590<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_cache</td></tr>
591</table>
592    <p>通常 Cache-Control: no-store ヘッダのついているレスポンスは
593       キャッシュされません。<code class="directive">CacheStoreNoCache</code> 
594       ディレクティブでこの挙動を上書きできます。
595       <code class="directive">CacheStoreNoCache</code> On で no-store ヘッダのついている
596       リソースに対してもキャッシュを試みるようになります。
597       ただし認証の求められるリソースは <em>決して</em> キャッシュされません。</p>
598
599    <div class="example"><p><code>
600      CacheStoreNoStore On
601    </code></p></div>
602
603    <div class="warning"><h3>警告:</h3>
604       RFC 2616 に記載されているように no-store ディレクティブは、
605       "不注意による機密情報の漏洩や残留 (バックアップテープ等) を防ぐ"
606       目的で使われますが、このオプションを有効にすると、
607       機密情報を保持することになってしまいます。
608       ですので、ここで警告しておきます。
609    </div>

618<div class="directive-section"><h2><a name="CacheStorePrivate" id="CacheStorePrivate">CacheStorePrivate</a> <a name="cachestoreprivate" id="cachestoreprivate">ディレクティブ</a></h2>
619<table class="directive">
620<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>private と指定されているレスポンスのキャッシュを試みる。</td></tr>
621<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>CacheStorePrivate On|Off</code></td></tr>
622<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>CacheStorePrivate Off</code></td></tr>
623<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>

627    <p>通常 Cache-Control: private ヘッダのついているレスポンスは
628       キャッシュされません。<code class="directive">CacheStorePrivate</code> 
629       ディレクティブでこの挙動を上書きできます。
630       <code class="directive">CacheStorePrivate</code> On で private ヘッダのついている
