42<h3>概要</h3>
43
44    <p>このモジュールはファイルがクライアントに送られる前に処理するフィルタを
45    提供します。処理の内容は<dfn>要素</dfn>と呼ばれる特別な形式の SGML コメントにより

48    できます。</p>

79<h2><a name="enabling" id="enabling">Server-Side Includes を有効にする</a></h2>
80    
81
82    <p>Server Side Includes は <code>INCLUDES</code>
83    <a href="../filter.html">フィルタ</a> により実装されています。
84    Server-side include のディレクティブを含むドキュメントの拡張子が
85    .shtml の場合、以下のディレクティブでは Apache がそれらを
86    パースして、その結果できるドキュメントに <code>text/html</code> の
87    MIME タイプを割り当てます:</p>
88
89    <div class="example"><p><code>
90      AddType text/html .shtml<br />
91      AddOutputFilter INCLUDES .shtml
92    </code></p></div>
93
94    <p>以下のディレクティブは shtml ファイルのあるディレクトリで指定されている
95    必要があります (通常は <code class="directive"><a href="../mod/core.html#directory">&lt;Directory&gt;</a></code> セクションで指定しますが、
96    <code class="directive"><a href="../mod/core.html#allowoverride">AllowOverride</a></code> <code>Options</code>
97    が設定されていると、<code>.htaccess</code> ファイルに書くこともできます):</p>
98
99    <div class="example"><p><code>
100      Options +Includes
101    </code></p></div>
102
103    <p>互換性を保つために、<code>server-parsed</code>
104    <a href="../handler.html">ハンドラ</a> も INCLUDES フィルタを
105    有効にします。MIME タイプ <code>text/x-server-parsed-html</code> や
106    <code>text/x-server-parsed-html3</code> のドキュメントに対しても
107    Apache は INCLUDES フィルタを有効にします (出力されるものは
108    MIME タイプ <code>text/html</code> になります)。</p>
109
110    <p>詳しい情報は <a href="../howto/ssi.html">Tutorial on Server Side Includes</a>.</p>
111</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

113<h2><a name="pathinfo" id="pathinfo">サーバサイドインクルード (SSI) での PATH_INFO</a></h2>
114    
115
116    <p>SSI で処理されるファイルはデフォルトでは <code>PATH_INFO</code>
117    (後続のパス名情報)
118    付きのリクエストを受け入れなくなりました。<code class="directive"><a href="../mod/core.html#acceptpathinfo">AcceptPathInfo</a></code> ディレクティブで
119    <code>PATH_INFO</code> 付きのリクエストを受け入れるようにサーバを
120    設定できます。</p>
121</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>
122<div class="section">
123<h2><a name="elements" id="elements">基本要素</a></h2>
124    <p>ドキュメントは、SGML のコメントとして特別なコマンドが埋め込まれた
125    HTML ドキュメントとしてパースされます。コマンドの構文は次のように
126    なっています:</p>
127
128    <div class="example"><p><code>
129      &lt;!--#<var>element</var> <var>attribute</var>=<var>value</var>
130      <var>attribute</var>=<var>value</var> ... --&gt;
131    </code></p></div>
132
133    <p>値 <span class="transnote">(<em>訳注:</em> value)</span> は二重引用符で囲むのが一般的ですが、
134    シングルクオート (') とバッククオート (`) も使用できます。
135    多くのコマンドは属性-値 <span class="transnote">(<em>訳注:</em> attribute-value)</span> の組を一つだけ指定できます。
136    コメントの終わり (<code>--&gt;</code>)
137    の前には、SSI の句の一部だと解釈されないようにするために空白を
138    入れてください。最初の <code>&lt;!--#</code> はまとめて<em>一つ</em>の
139    句で、空白をふくんではいけないこと注意してください。</p>
140
141    <p>要素 <span class="transnote">(<em>訳注:</em> element)</span> を以下の表に示します。</p>
142
143    <table class="bordered">
144    <tr><th>要素</th><th>説明</th></tr>
145    <tr><td><code><a href="#element.config">config</a></code></td>
146        <td>configure output formats</td></tr>
147    <tr><td><code><a href="#element.echo">echo</a></code></td>
148        <td>print variables</td></tr>
149    <tr><td><code><a href="#element.exec">exec</a></code></td>
150        <td>execute external programs</td></tr>
151    <tr><td><code><a href="#element.fsize">fsize</a></code></td>
152        <td>print size of a file</td></tr>
153    <tr><td><code><a href="#element.flastmod">flastmod</a></code></td>
154        <td>print last modification time of a file</td></tr>
155    <tr><td><code><a href="#element.include">include</a></code></td>
156        <td>include a file</td></tr>
157    <tr><td><code><a href="#element.printenv">printenv</a></code></td>
158        <td>print all available variables</td></tr>
159    <tr><td><code><a href="#element.set">set</a></code></td>
160        <td>set a value of a variable</td></tr>
161    </table>
162
163    <p>SSI 要素は <code class="module"><a href="../mod/mod_include.html">mod_include</a></code> 以外のモジュールで
164    定義されることもあります。実際、
165    <code><a href="#element.exec">exec</a></code> 要素は
166    <code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code> で提供されていて、このモジュールが
167    ロードされる場合にのみ利用可能となります。</p>
168
169    <h3><a name="element.config" id="element.config">config 要素</a></h3>
170      <p>次のコマンドは解析の様々な側面を制御します。属性は次の通りです。</p>
171
172      <dl>
173      <dt><code>echomsg</code> (<em>Apache 2.1 以降</em>)</dt>
174      <dd>指定される値は、<code><a href="#element.echo">echo</a></code>
175      要素が未定義の変数をエコーしようとした際に、
176      クライアントに送られるメッセージになります。
177      <code class="directive"><a href="#ssiundefinedecho">SSIUndefinedEcho</a></code> 
178      ディレクティブを上書きします。</dd>
179
180      <dt><code>errmsg</code></dt>
181      <dd>この値が、ドキュメントの解析中にエラーが発生した時に
182      クライアントに送信されるメッセージになります。
183      <code class="directive"><a href="#ssierrormsg">SSIErrorMsg</a></code>
184      ディレクティブを上書きします。</dd>
185
186      <dt><code>sizefmt</code></dt>
187      <dd>この値は、ファイルのサイズを表示する際に使用する
188      フォーマットを設定します。値は バイトカウントの
189      <code>bytes</code>か、Kb や Mb を優先的に使用する
190      <code>abbrec</code> (例えば 1024 バイトは "1K" と表示されます)
191      です。</dd>
192
193      <dt><code>timefmt</code></dt>
194      <dd>この値は <code>strftime(3)</code> ライブラリルーチンが
195      日時をプリントする際に用いられます。</dd>
196      </dl>

199    <h3><a name="element.echo" id="element.echo">echo 要素</a></h3>
200      <p>このコマンドは以下で定義されている <a href="#includevars">include
201      変数</a> を表示します。変数が設定されていない場合は <code class="directive"><a href="#ssiundefinedecho">SSIUndefinedEcho</a></code> ディレクティブで
202      決定される結果となります。日付はその時点での <code>timefmt</code> に従って
203      表示されます。属性は次の通りです。</p>
204
205      <dl>
206      <dt><code>var</code></dt>
207      <dd>値は表示する変数の名前です。</dd>
208
209      <dt><code>encoding</code></dt>
210      <dd><p>変数を出力する前に、変数中の特別文字をどのようにエンコードするかを
211      指定します。<code>none</code> に設定されていると、エンコードは行なわれません。
212      <code>url</code> に設定されていると、URL エンコード (%-エンコードとも
213      呼ばれています。これはリンク等の URL の使用に適切です) が
214      行なわれます。<code>echo</code> 要素の開始時は、デフォルトは
215      <code>entity</code> に設定されています。これはエンティティエンコード
216      (段落やテキストなどのブロックレベルの HTML エレメントのコンテキストに
217      適しています) を行ないます。これは <code>encoding</code> 属性
218      を加えることで変更できます。変更は次の <code>encoding</code> 属性か、
219      要素の終了まで効力を持ちます。</p>
220
221      <p><code>encoding</code> 属性はエンコードの変更をしたい <code>var</code> 
222      の<em>前に</em> ある必要があることに注意してください。
223      また、ISO-8859-1 エンコーディングで
224      定義されている特別な文字だけがエンコードされます。
225      別の文字のエンコーディングの場合は、このエンコーディングは
226      望みの結果にならないかもしれません。</p>
227
228      <div class="warning">
229        クロスサイトスクリプティングの問題を避けるために、
230        <em>常に</em>ユーザからのデータをエンコードすべきです。
231      </div>
232      </dd>
233      </dl>

236    <h3><a name="element.exec" id="element.exec">exec 要素</a></h3>
237      <p><code>exec</code> コマンドは指定されたシェルコマンドや CGI スクリプトを
238      実行します。<code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code> がサーバに組み込まれているいなければ
239      なりません。<code class="directive"><a href="../mod/core.html#option">Option</a></code> 
240      <code>IncludesNOEXEC</code> はこのコマンドを無効にします。
241      使用可能な属性は次の通りです。</p>
242
243      <dl>
244      <dt><code>cgi</code></dt>
245      <dd><p>値は (%-エンコードされた) URL を指定します。パスが
246      スラッシュ (/) で始まらないときは、ドキュメントからの
247      相対パスとして扱われます。このパスで参照されているドキュメントは
248      サーバが CGI スクリプトとして扱っていなくても CGI スクリプトとして
249      起動されます。ただし、スクリプトのあるディレクトリでは
250      (<code class="directive"><a href="../mod/mod_alias.html#scriptalias">ScriptAlias</a></code>
251      や <code class="directive"><a href="../mod/core.html#option">Option</a></code> <code>ExecCGI</code>
252      によって) CGI スクリプトの使用が許可されている必要があります。</p>
253
254      <p>CGI スクリプトには、クライアントからの元々のリクエストの
255      <code>PATH_INFO</code> とクエリー文字列 (<code>QUERY_STRING</code>) が渡されます。
256      これらは URL パスとして特定<em>できない</em>ものです。
257      スクリプトは標準 <a href="mod_cgi.html">CGI</a> 環境に加えて、include 変数を
258      使用することができます。</p>
259
260      <div class="example"><h3>例</h3><p><code>
261        &lt;!--#exec cgi="/cgi-bin/example.cgi" --&gt;
262      </code></p></div>
263
264      <p>スクリプトが、出力の代わりに <code>Location:</code> ヘッダを返すと、
265      HTML のアンカー <span class="transnote">(<em>訳注:</em> リンク)</span> に変換されます。</p>
266
267      <p><code>exec cgi</code> よりも、
268      <code><a href="#includevirtual">include virtual</a></code>
269      の方を使うようにしてください。特に、CGI への追加の引数を
270      クエリー文字列を使って渡すことは <code>exec cgi</code> は
271      できませんが、<code>include virtual</code> は以下のようにして
272      可能です。</p>
273
274      <div class="example"><p><code>
275        &lt;!--#include virtual="/cgi-bin/example.cgi?argument=value" --&gt;
276      </code></p></div>
277      </dd>
278
279      <dt><code>cmd</code></dt>
280      <dd><p>サーバは指定された文字列を <code>/bin/sh</code> を使って
281      実行します。コマンドは通常の CGI 変数に加えて <a href="#includevars">include 変数</a>も使うことができます。</p>
282
283      <p>ほとんどの場合、<code><a href="#includevirtual">#include 
284      virtual</a></code> を使う方が <code>#exec cgi</code> や <code>#exec
285      cmd</code> を使うよりも良いです。前者 (<code>#include virtual</code>)
286      は標準の Apache のサブリクエスト機構を使ってファイルやスクリプトの
287      出力を取り込みます。
288      こちらの方がよくテストされメンテナンスされた方法です。</p>
289
290      <p>さらに、Win32 のようないくつかのプラットフォームや、<a href="../suexec.html">suexec</a> を使っている unix では、
291      <code>exec</code> ディレクティブのコマンドに
292      引数を渡したり、コマンドに空白を入れることはできません。
293      ですから、以下のものは unix の suexec でない設定では動作しますが、
294      Win32 や suexec を使っている unix では期待した結果にはなりません:</p>
295
296      <div class="example"><p><code>
297        &lt;!--#exec cmd="perl /path/to/perlscript arg1 arg2" --&gt;
298      </code></p></div>
299      </dd>
300      </dl>

303    <h3><a name="element.fsize" id="element.fsize">fsize 要素</a></h3>
304      <p>このコマンドは指定されたファイルの大きさを <code>sizefmt</code> の
305      書式指定に基づいて出力します。属性は次の通りです。</p>
306
307      <dl>
308      <dt><code>file</code></dt>
309      <dd>値は解析されているドキュメントの存在するディレクトリからの
310      相対パスです。</dd>
311
312      <dt><code>virtual</code></dt>
313      <dd> 値は (% エンコードされた) URL-path です。スラッシュ (/) で
314      始まらないときはドキュメントからの相対パスとして扱われます。
315      CGI の出力のサイズはプリント<em>されません</em>。CGI
316      スクリプト自体のサイズがプリントされることに注意してください。</dd>
317      </dl>

326    <h3><a name="element.include" id="element.include">include 要素</a></h3>
327      <p>このコマンドは別の文書やファイルのテキストを解析しているファイルに
328      挿入します。挿入されるファイルはアクセス制御の管理下にあります。
329      解析しているファイルの存在するディレクトリに
330      <a href="core.html#options">Option</a> <code>IncludesNOEXEC</code>
331      が設定されている場合、text <a class="glossarylink" href="../glossary.html#mime-type" title="用語集を参照">MIME タイプ</a>
332      (<code>text/plain</code>, <code>text/html</code> 等) 
333      のドキュメントのみインクルードが行なわれます。
334      その他の場合は、クエリー文字列も含め、コマンドで指定された
335      完全な URL を使って普通に CGI スクリプトが呼び出されます。</p>
336
337      <p>属性が文書の位置を指定します。include コマンドに与えられたそれぞれの
338      属性に対して挿入作業が行なわれます。有効な属性は次の通りです。</p>
339
340      <dl>
341      <dt><code>file</code></dt>
342      <dd>値は解析されているドキュメントの存在するディレクトリからの
343      相対パスです。
344      <code>../</code> を含んでいたり、絶対パスを指定したりはできません。
345      ですから、ドキュメントルートの外にあるファイルや、ディレクトリ構造で
346      上位にあるファイルを挿入することはできません。
347      常にこの属性よりは、<code>virtual</code> 属性を使うようにしてください。
348      </dd>
349
350      <dt><code><a id="includevirtual" name="includevirtual">virtual</a></code></dt>
351      <dd><p>値は解析されているドキュメントからの (% エンコードされた) URL
352      です。URL にはスキームやホスト名を含めることはできません。パスと、
353      もしあればクエリー文字列を指定できるだけです。スラッシュ (/) から
354      始まらない場合は、ドキュメントからの相対パスとして扱われます。</p>
355
356      <p>URL は属性から作られ、その URL をクライアントがアクセスしたときに
357      出力される内容が解析後の出力に含められます。ですから、挿入される
358      ファイルは入れ子構造にすることができます。</p>
359
360      <p>指定された URL が CGI プログラムであった場合は、
361      プログラムが実行され、その出力が解析しているファイル中の
362      ディレクティブがあった位置に挿入されます。CGI の url に
363      クエリー URL を入れることもできます。</p>
364
365      <div class="example"><p><code>
366        &lt;!--#include virtual="/cgi-bin/example.cgi?argument=value" --&gt;
367      </code></p></div>
368
369      <p>HTML ドキュメントに CGI プログラムの出力を含める方法としては、
370      <code>include virtual</code> の方が <code>exec cgi</code> よりも
371      好ましい方法です。</p>
372
373      <p><code class="directive"><a href="../mod/mod_request.html#keptbodysize">KeptBodySize</a></code>
374      ディレクティブが設定されていて、かつ、この対象ファイルが
375      <span class="transnote">(<em>訳注:</em> POST リクエストを)</span>受け入れできるなら、
376      POST リクエストを受け取ってサブリクエストを発行する際にも
377      POST リクエストが渡されます。
378      このディレクティブが設定されていない場合は、
379      サブリクエストは GET リクエストとして処理されます。</p>
380
381      </dd>
382      </dl>

445<h2><a name="substitution" id="substitution">変数置換</a></h2>
446
447    <p>変数置換はたいていの場合 SSI ディレクティブの引数として妥当な場所にある
448    引用符で囲まれた文字列中で行なわれます。これに該当するものには、
449    <code>config</code>,
450    <code>exec</code>, <code>flastmod</code>, <code>fsize</code>,
451    <code>include</code>, <code>echo</code>, <code>set</code> の
452    各ディレクティブと、条件分岐用のオペレータへの引数があります。
453    ドル記号はバックスラッシュを使うことで使うことができます:</p>
454
455    <div class="example"><p><code>
456      &lt;!--#if expr="$a = \$test" --&gt;
457    </code></p></div>
458
459    <p>変数名としてみなされる文字列の中で変数への参照を置換する必要があるときは、
460    シェルでの変数置換のように、中括弧で括ることで区別することができます:</p>
461
462    <div class="example"><p><code>
463      &lt;!--#set var="Zed" value="${REMOTE_HOST}_${REQUEST_METHOD}" --&gt;
464    </code></p></div>
465
466    <p>この例では、<code>REMOTE_HOST</code> が
467    "<code>X</code>" で <code>REQUEST_METHOD</code> が
468    "<code>Y</code>" のときに変数 <code>Zed</code> を "<code>X_Y</code>"
469    に設定します。</p>
470
471    <p>以下の例では、<code>DOCUMENT_URI</code> が <code>/foo/file.html</code>
472    のときに "in foo" を、<code>/bar/file.html</code> のときに "in bar" を、
473    どちらでもないときには "in neither" を表示します。</p>
474
475    <div class="example"><p><code>
476      &lt;!--#if expr='"$DOCUMENT_URI" = "/foo/file.html"' --&gt;<br />
477      <span class="indent">
478        in foo<br />
479      </span>
480      &lt;!--#elif expr='"$DOCUMENT_URI" = "/bar/file.html"' --&gt;<br />
481      <span class="indent">
482        in bar<br />
483      </span>
484      &lt;!--#else --&gt;<br />
485      <span class="indent">
486        in neither<br />
487      </span>
488      &lt;!--#endif --&gt;
489    </code></p></div>

492<h2><a name="flowctrl" id="flowctrl">フロー制御要素</a></h2>
493    
494
495    <p>基本的なフローコントロール要素は次の通りです。</p>
496
497    <div class="example"><p><code>
498      &lt;!--#if expr="<var>test_condition</var>" --&gt;<br />
499      &lt;!--#elif expr="<var>test_condition</var>" --&gt;<br />
500      &lt;!--#else --&gt;<br />
501      &lt;!--#endif --&gt;
502    </code></p></div>
503
504    <p><code>if</code> 要素はプログラミング言語の
505    if 文と同じように動作します。条件が評価され、結果が真であれば次の
506    <code>elif</code> か <code>else</code> か <code>endif</code>
507    要素までの文字列が出力に挿入されます。</p>
508
509    <p><code>elif</code> や <code>else</code> 文は <var>test_condition</var> 
510    が偽のときにテキストを出力に挿入するために使われます。
511    これらの要素はあってもなくても構いません。</p>
512
513    <p><code>endif</code> 要素は <code>if</code>
514    要素を終了させます。この要素は必須です。</p>
515
516    <p><var>test_condition</var> は以下のどれかです:</p>
517
518    <dl>
519      <dt><code><var>string</var></code></dt>
520      <dd><var>string</var> が空でない場合に真です</dd>
521
522      <dt><code><var>-A string</var></code></dt>
523      <dd><p><span class="transnote">(<em>訳注:</em> httpd の)</span>設定を検査して、
524      文字列で指定した URL にアクセスできる場合 true で、
525      そうでなければ false になります。
526      <code class="directive">SSIAccessEnable</code> が有効のときにのみ
527      この検査は行われます。
528      承認されていないユーザからは隠しておきたい URL についての情報、
529      たとえば URL へのリンクなどがある場合に、便利です。
530      検査では URL へアクセスできるかの権限のみが行われ、URL
531      が存在するかどうかについては検査されないことに注意してください。</p>
532
533      <div class="example"><h3>Example</h3><p><code>
534        &lt;!--#if expr="-A /private" --&gt;<br />
535        <span class="indent">
536          Click &lt;a href="/private"&gt;here&lt;/a&gt; to access private
537          information.<br />
538        </span>
539        &lt;!--#endif --&gt;
540      </code></p></div>
541      </dd>
542
543      <dt><code><var>string1</var> = <var>string2</var><br />
544      <var>string1</var> == <var>string2</var><br />
545      <var>string1</var> != <var>string2</var></code></dt>
546
547      <dd><p><var>string1</var> と <var>string2</var> を比較します。
548      <var>string2</var> が <code>/<var>string</var>/</code>
549      という形式であれば、正規表現として比較されます。正規表現は
550      <a href="http://www.pcre.org">PCRE</a> エンジンで実装されていて、
551      <a href="http://www.perl.com">perl 5</a> と同じ構文を使用します。
552      <code>==</code> は単に <code>=</code> の別名で、まったく同じ動作を
553      します。</p>
554
555      <p>正のマッチング (<code>=</code> または <code>==</code>) の場合は、
556      正規表現でグループ分けされたパーツをキャプチャすることができます。
557      キャプチャされた部分は特殊変数 <code>$1</code> .. <code>$9</code>
558      に格納されます。</p>
559
560      <div class="example"><h3>例</h3><p><code>
561        &lt;!--#if expr="$QUERY_STRING = /^sid=([a-zA-Z0-9]+)/" --&gt;<br />
562        <span class="indent">
563          &lt;!--#set var="session" value="$1" --&gt;<br />
564        </span>
565        &lt;!--#endif --&gt;
566      </code></p></div>
567      </dd>
568
569      <dt><code><var>string1</var> &lt; <var>string2</var><br />
570       <var>string1</var> &lt;= <var>string2</var><br />
571       <var>string1</var> &gt; <var>string2</var><br />
572       <var>string1</var> &gt;= <var>string2</var></code></dt>
573
574      <dd><var>string1</var> と <var>string2</var> を比較します。
575      <em>文字列として</em>比較される (<code>strcmp(3)</code> を使用)
576      ことに注意してください。ですから、文字列 "100" は "20" 
577      よりも小さいことになります。</dd>
578
579      <dt><code>( <var>test_condition</var> )</code></dt>
580      <dd><var>test_condition</var> が真のとき、真</dd>
581
582      <dt><code>! <var>test_condition</var></code></dt>
583      <dd><var>test_condition</var> が偽のとき、真</dd>
584
585      <dt><code><var>test_condition1</var> &amp;&amp;
586        <var>test_condition2</var></code></dt>
587      <dd><var>test_condition1</var> かつ
588      <var>test_condition2</var> が真のとき、真</dd>
589
590      <dt><code><var>test_condition1</var> ||
591        <var>test_condition2</var></code></dt>
592      <dd><var>test_condition1</var> または
593      <var>test_condition2</var> が真のとき、真</dd>
594    </dl>
595
596    <p>"<em>=</em>" と "<em>!=</em>" の方が "<em>&amp;&amp;</em>" より
597    きつく束縛します。"<em>!</em>" の束縛が一番きつくなっています。
598    ですから以下の二つは等価です:</p>
599
600    <div class="example"><p><code>
601      &lt;!--#if expr="$a = test1 &amp;&amp; $b = test2" --&gt;<br />
602      &lt;!--#if expr="($a = test1) &amp;&amp; ($b = test2)" --&gt;
603    </code></p></div>
604
605    <p>真偽値オペレータ <code>&amp;&amp;</code> と <code>||</code>
606    は同じ優先度です。
607    これらのオペレータで一方により強い優先度をつけたい場合には、
608    括弧を使う必要があります。</p>
609
610    <p>変数やオペレータとして認識されないものはすべて文字列として
611    扱われます。文字列は引用符で囲むこともできます: <code>'string'</code>
612    のように。引用符で囲まれていない文字列には空白 (スペースとタブ)
613    を含めることはできません。それらは変数などの句を分離するために
614    使われているからです。複数の文字列が続いているときは、
615    空白を間に入れて一つにくっつけられます。ですから、</p>
616
617    <div class="example"><p><code><var>string1</var>&nbsp;&nbsp;&nbsp;&nbsp;<var>string2</var></code> は <code><var>string1</var>&nbsp;<var>string2</var></code> になります。<br />
618      <br />
619      また、<br />
620      <br />
621      <code>'<var>string1</var>&nbsp;&nbsp;&nbsp;&nbsp;<var>string2</var>'</code> は <code><var>string1</var>&nbsp;&nbsp;&nbsp;&nbsp;<var>string2</var></code>
622      になります。</p></div>
623
624    <div class="note"><h3>真偽値表現の最適化</h3>
625      <p>式がもっと複雑になり、処理の速度低下が顕著になった場合は、
626      評価ルールに従って最適化してみると良いでしょう。</p>
627      <ul>
628      <li>評価は左から右に向かって行われます。</li>
629      <li>二値真偽値オペレータ (<code>&amp;&amp;</code> と <code>||</code>)
630          は、出来る限り短絡評価されます。つまり結果として上記のルールは、
631          <code class="module"><a href="../mod/mod_include.html">mod_include</a></code> が左の評価式を評価します。
632          左側で結果を十分決定できる場合は、評価はそこで停止します。
633          そうでない場合は右側を評価して、左と右の両方から結果を計算します。</li>
634      <li>短絡評価は評価の対象に正規表現が含まれる場合、オフになります。
635          後方参照する変数 (<code>$1</code> .. <code>$9</code>) 
636          を埋めるために、実際に評価する必要があるからです。</li>
637      </ul>
638      <p>特定の式がどのように扱われるかを知りたい場合は、
639      <code>-DDEBUG_INCLUDE</code> コンパイラオプションを付けて
640      <code class="module"><a href="../mod/mod_include.html">mod_include</a></code> をリコンパイルすると良いでしょう。
641      これにより、全てのパースされた式に対して、字句解析情報、
642      パースツリーと、
643      それがどのようにクライアントに送られた出力まで評価されたかを
644      挿入します。</p>
645    </div>
646 
647    <div class="note"><h3>正規表現内での / のエスケープ</h3>
648    <p>正規表現内でデリミタとして扱いたくない / があれば、それらは全て
649    エスケープしなければなりません。
650    正規表現の意味がどうであろうとエスケープは必要です。</p>
651    </div>

745<div class="directive-section"><h2><a name="SSIStartTag" id="SSIStartTag">SSIStartTag</a> <a name="ssistarttag" id="ssistarttag">ディレクティブ</a></h2>
746<table class="directive">
747<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>include 要素を開始する文字列</td></tr>
748<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>SSIStartTag <var>tag</var></code></td></tr>
749<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>SSIStartTag "&lt;!--#"</code></td></tr>
750<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト</td></tr>
751<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Base</td></tr>
752<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_include</td></tr>
753<tr><th><a href="directive-dict.html#Compatibility">互換性:</a></th><td>バージョン 2.0.30 以降で使用可能</td></tr>
754</table>
755
756    <p>このディレクティブは <code class="module"><a href="../mod/mod_include.html">mod_include</a></code> が探す、include
757    要素の開始を示す文字列を変更します。</p>
758
759    <p>二つのサーバで (もしかすると別々の段階で) ファイルの出力を解析していて、
760    それぞれに違うコマンドを処理させたい、
761    というようなときにこのオプションを使います。</p>
762
763    <div class="example"><h3>例</h3><p><code>
764      SSIStartTag "&lt;%"<br />
765      SSIEndTag   "%&gt;"
766    </code></p></div>
767
768    <p>上の例のように対応する
769    <code class="directive"><a href="#ssiendtag">SSIEndTag</a></code> を併せて使うと、
770    下に示す例のように SSI ディレクティブを使えます:</p>
771
772    <div class="example"><h3>違う開始と終了のタグを使った SSI ディレクティブ</h3><p><code>
773      &lt;%printenv %&gt;
774    </code></p></div>
775

778<li><code class="directive"><a href="#ssiendtag">SSIEndTag</a></code></li>
779</ul>

782<div class="directive-section"><h2><a name="SSITimeFormat" id="SSITimeFormat">SSITimeFormat</a> <a name="ssitimeformat" id="ssitimeformat">ディレクティブ</a></h2>
783<table class="directive">
784<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>日付けを現す文字列の書式を設定する</td></tr>
785<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>SSITimeFormat <var>formatstring</var></code></td></tr>
786<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>SSITimeFormat "%A, %d-%b-%Y %H:%M:%S %Z"</code></td></tr>
787<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト, ディレクトリ, .htaccess</td></tr>
788<tr><th><a href="directive-dict.html#Override">上書き:</a></th><td>All</td></tr>
789<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Base</td></tr>
790<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_include</td></tr>
791<tr><th><a href="directive-dict.html#Compatibility">互換性:</a></th><td>2.0.30 以降で使用可能</td></tr>
792</table>
793<p>このディレクティブは <code>DATE</code> 環境変数を echo して日付を現す文字列が
794    表示されるときの書式を変更します。<var>formatstring</var> は
795    C 標準ライブラリの <code>strftime(3)</code> と同じ形式です。</p>
796
797    <p>このディレクティブは <code>&lt;!--#config
798    timefmt=<var>formatstring</var> --&gt;</code> 要素と同じ効果になります。</p>
799
800    <div class="example"><h3>例</h3><p><code>
801      SSITimeFormat "%R, %B %d, %Y"
802    </code></p></div>
803
804    <p>上のディレクティブでは、日付は "22:26, June 14, 2002" という
805    形式で表示されます。</p>
806

830<div class="directive-section"><h2><a name="XBitHack" id="XBitHack">XBitHack</a> <a name="xbithack" id="xbithack">ディレクティブ</a></h2>
831<table class="directive">
832<tr><th><a href="directive-dict.html#Description">説明:</a></th><td>実行ビットが設定されたファイルの SSI ディレクティブを
833解析する</td></tr>
834<tr><th><a href="directive-dict.html#Syntax">構文:</a></th><td><code>XBitHack on|off|full</code></td></tr>
835<tr><th><a href="directive-dict.html#Default">デフォルト:</a></th><td><code>XBitHack off</code></td></tr>
836<tr><th><a href="directive-dict.html#Context">コンテキスト:</a></th><td>サーバ設定ファイル, バーチャルホスト, ディレクトリ, .htaccess</td></tr>
837<tr><th><a href="directive-dict.html#Override">上書き:</a></th><td>Options</td></tr>
838<tr><th><a href="directive-dict.html#Status">ステータス:</a></th><td>Base</td></tr>
839<tr><th><a href="directive-dict.html#Module">モジュール:</a></th><td>mod_include</td></tr>
840</table>
841    <p><code class="directive">XBitHack</code> ディレクティブは通常の HTML
842    ドキュメントの解析を制御します。このディレクティブは <a class="glossarylink" href="../glossary.html#mime-type" title="用語集を参照">MIME タイプ</a>
843    <code>text/html</code> と関連付けられているファイルにのみ影響します。
844    <code class="directive">XBitHack</code> は以下の値をとることができます。</p>
845
846    <dl>
847      <dt><code>off</code></dt>
848      <dd>実行可能ファイルに対して特別な扱いをしません。</dd>
849
850      <dt><code>on</code></dt>
851      <dd>ユーザの実行ビットが設定されている <code>text/html</code>
852      ファイルは全てサーバで解析する html ドキュメントとして扱われます。</dd>
853
854      <dt><code>full</code></dt>
855      <dd><code>on</code> と同様ですが、グループ実行ビットもテストします。
856      もしそれが設定されていれば、返されるファイルの <code>Last-modified</code> の
857      日付をファイルの最終修正時刻にします。それが設定されていないときは、
858      last-modified の日付は送られません。このビットを設定すると、
859      クライアントやプロキシがリクエストをキャッシュできるようになります。
860
861      <div class="note"><strong>注意</strong> 他の CGI を <code>#include</code>
862      するかもしれないものや、各アクセスに対して違う出力を生成する
863      (もしくは後のリクエストで変わるかもしれないもの)
864      すべての SSI スクリプトに対してグループ実行ビットが
865      設定されていないことを確認できない場合は、full は使わない方が良い
866      でしょう。</div>
867      </dd>
868    </dl>
869
