37<h3>Summary</h3>
38
39    <p>This module <em>requires</em> the service of <code class="module"><a href="../mod/mod_proxy.html">mod_proxy</a></code>. It provides support for the
40    <a href="http://python.ca/scgi/protocol.txt">SCGI protocol, version
41    1</a>.</p>
42
43    <p>Thus, in order to get the ability of handling the SCGI protocol,
44    <code class="module"><a href="../mod/mod_proxy.html">mod_proxy</a></code> and <code class="module"><a href="../mod/mod_proxy_scgi.html">mod_proxy_scgi</a></code> have to
45    be present in the server.</p>
46
47    <div class="warning"><h3>Warning</h3>
48      <p>Do not enable proxying until you have <a href="mod_proxy.html#access">secured your server</a>. Open proxy
49      servers are dangerous both to your network and to the Internet at
50      large.</p>
51    </div>

69<h2><a name="examples" id="examples">Examples</a></h2>
70    <p>Remember, in order to make the following examples work, you have to
71    enable <code class="module"><a href="../mod/mod_proxy.html">mod_proxy</a></code> and <code class="module"><a href="../mod/mod_proxy_scgi.html">mod_proxy_scgi</a></code>.</p>
72
73    <div class="example"><h3>Simple gateway</h3><pre class="prettyprint lang-config">ProxyPass "/scgi-bin/" "scgi://localhost:4000/"</pre>
74</div>
75
76    <p>The balanced gateway needs <code class="module"><a href="../mod/mod_proxy_balancer.html">mod_proxy_balancer</a></code> and
77    at least one load balancer algorithm module, such as
78    <code class="module"><a href="../mod/mod_lbmethod_byrequests.html">mod_lbmethod_byrequests</a></code>, in addition to the proxy
79    modules listed above.  <code class="module"><a href="../mod/mod_lbmethod_byrequests.html">mod_lbmethod_byrequests</a></code> is the
80    default, and will be used for this example configuration.</p>
81
82    <div class="example"><h3>Balanced gateway</h3><pre class="prettyprint lang-config">ProxyPass "/scgi-bin/" "balancer://somecluster/"
83&lt;Proxy "balancer://somecluster"&gt;
84    BalancerMember "scgi://localhost:4000"
85    BalancerMember "scgi://localhost:4001"
86&lt;/Proxy&gt;</pre>
87</div>

90<h2><a name="env" id="env">Environment Variables</a></h2>
91    <p>In addition to the configuration directives that control the
92    behavior of <code class="module"><a href="../mod/mod_proxy.html">mod_proxy</a></code>, an <dfn>environment
93    variable</dfn> may also control the SCGI protocol
94    provider:</p>
95    <dl>
96        <dt>proxy-scgi-pathinfo</dt>
97        <dd>By default <code class="module"><a href="../mod/mod_proxy_scgi.html">mod_proxy_scgi</a></code> will neither create
98        nor export the <var>PATH_INFO</var> environment variable. This allows
99        the backend SCGI server to correctly determine <var>SCRIPT_NAME</var>
100        and <var>Script-URI</var> and be compliant with RFC 3875 section 3.3.
101        If instead you need <code class="module"><a href="../mod/mod_proxy_scgi.html">mod_proxy_scgi</a></code> to generate
102        a "best guess" for <var>PATH_INFO</var>, set this env-var.  The
103        variable must be set before <code class="directive"><a href="../mod/mod_env.html#setenv">SetEnv</a></code>
104        is effective.  <code class="directive"><a href="../mod/mod_setenvif.html#setenvif">SetEnvIf</a></code> can be
105        used instead: <code>SetEnvIf Request_URI . proxy-scgi-pathinfo</code>
106        </dd>
107    </dl>

110<div class="directive-section"><h2><a name="ProxySCGIInternalRedirect" id="ProxySCGIInternalRedirect">ProxySCGIInternalRedirect</a> <a name="proxyscgiinternalredirect" id="proxyscgiinternalredirect">Directive</a></h2>
111<table class="directive">
112<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Enable or disable internal redirect responses from the
113backend</td></tr>
114<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ProxySCGIInternalRedirect On|Off|<var>Headername</var></code></td></tr>
115<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>ProxySCGIInternalRedirect On</code></td></tr>
116<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
117<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
118<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_proxy_scgi</td></tr>
119<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>The <var>Headername</var> feature is available in version
1202.4.13 and later</td></tr>
121</table>
122    <p>The <code class="directive">ProxySCGIInternalRedirect</code> enables the backend
123    to internally redirect the gateway to a different URL. This feature
124    originates in <code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code>, which internally redirects the
125    response if the response status is <code>OK</code> (<code>200</code>) and
126    the response contains a <code>Location</code> (or configured alternate
127    header) and its value starts with a slash (<code>/</code>). This value is 
128    interpreted as a new local URL that Apache httpd internally redirects to.</p>
129
130    <p><code class="module"><a href="../mod/mod_proxy_scgi.html">mod_proxy_scgi</a></code> does the same as
131    <code class="module"><a href="../mod/mod_cgi.html">mod_cgi</a></code> in this regard, except that you can turn off the
132    feature or specify the use of a header other than <code>Location</code>.</p>
133
134    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">    ProxySCGIInternalRedirect Off
135
136# Django and some other frameworks will fully qualify "local URLs"
137# set by the application, so an alternate header must be used.
138&lt;Location /django-app/&gt;
139    ProxySCGIInternalRedirect X-Location
140&lt;/Location&gt;</pre>
141</div>

145<div class="directive-section"><h2><a name="ProxySCGISendfile" id="ProxySCGISendfile">ProxySCGISendfile</a> <a name="proxyscgisendfile" id="proxyscgisendfile">Directive</a></h2>
146<table class="directive">
147<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Enable evaluation of <var>X-Sendfile</var> pseudo response
148header</td></tr>
149<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>ProxySCGISendfile On|Off|<var>Headername</var></code></td></tr>
150<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>ProxySCGISendfile Off</code></td></tr>
151<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
152<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
153<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_proxy_scgi</td></tr>
154</table>
155    <p>The <code class="directive">ProxySCGISendfile</code> directive enables the
156    SCGI backend to let files be served directly by the gateway. This is useful
157    for performance purposes — httpd can use <code>sendfile</code> or other
158    optimizations, which are not possible if the file comes over the backend
159    socket.  Additionally, the file contents are not transmitted twice.</p>
160    <p>The <code class="directive">ProxySCGISendfile</code> argument determines the
161    gateway behavior:</p>
162    <dl>
163    <dt><code>Off</code></dt>
164    <dd>No special handling takes place.</dd>
165
166    <dt><code>On</code></dt>
167    <dd>The gateway looks for a backend response header called
168    <code>X-Sendfile</code> and interprets the value as the filename to serve.
169    The  header is removed from the final response headers. This is equivalent to
170    <code>ProxySCGISendfile X-Sendfile</code>.</dd>
171
172    <dt>anything else</dt>
173    <dd>Similar to <code>On</code>, but instead of the hardcoded header name
174    <code>X-Sendfile</code>, the argument is used as the header name.</dd>
175    </dl>
176
177    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config"># Use the default header (X-Sendfile)
178ProxySCGISendfile On
179
180# Use a different header
181ProxySCGISendfile X-Send-Static</pre>
