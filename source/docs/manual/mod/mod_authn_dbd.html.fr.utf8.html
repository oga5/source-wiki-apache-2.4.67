38<h3>Sommaire</h3>
39
40    <p>Ce module permet aux frontaux d'authentification comme
41    <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code> et <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code>
42    d'authentifier les utilisateurs en les recherchant dans une base de
43    données SQL. <code class="module"><a href="../mod/mod_authn_file.html">mod_authn_file</a></code>, par exemple, fournit
44    une fonctionnalité similaire.</p>
45    <p>Ce module s'appuie sur <code class="module"><a href="../mod/mod_dbd.html">mod_dbd</a></code> pour spécifier le
46    pilote de la base de données sous-jacente et les paramètres de
47    connexion, mais aussi pour gérer les connexions à la base de
48    données.</p>
49
50    <p>Si l'on utilise <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> ou
51    <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>, on peut invoquer ce module en
52    affectant la valeur <code>dbd</code> à la directive <code class="directive"><a href="../mod/mod_auth_basic.html#authbasicprovider">AuthBasicProvider</a></code> ou <code class="directive"><a href="../mod/mod_auth_digest.html#authdigestprovider">AuthDigestProvider</a></code>.</p>

81<h2><a name="socache" id="socache">Performances et mise en cache</a></h2>
82
83<p>Certains utilisateurs de l'authentification DBD sous HTTPD 2.2/2.4 ont
84signalé une charge problématique au niveau de la base de données. Cela
85se produit en général lorsqu'une page HTML contient des centaines d'objets
86(comme des images, des scripts, etc...), chacun d'entre eux nécessitant
87une authentification. Les utilisateurs qui rencontrent ce genre de
88problème peuvent utiliser le module <code class="module"><a href="../mod/mod_authn_socache.html">mod_authn_socache</a></code>
89qui permet de mettre les données d'authentification en cache, et
90soulager ainsi la base de données de la plus grande partie de la charge.</p>
91</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>

93<h2><a name="example" id="example">Exemple de configuration</a></h2>
94
95<p>Voici un exemple simple d'utilisation de ce module dans un contexte
96d'authentification et de bases de données.</p>
97<pre class="prettyprint lang-config"># configuration de mod_dbd
98# MISE À JOUR pour inclure la mise en cache de l'authentification
99DBDriver pgsql
100DBDParams "dbname=apacheauth user=apache password=xxxxxx"
101
102DBDMin  4
103DBDKeep 8
104DBDMax  20
105DBDExptime 300
106
107&lt;Directory "/usr/www/mon-serveur/private"&gt;
108  # configuration de mod_authn_core et mod_auth_basic
109  # pour mod_authn_dbd
110  AuthType Basic
111  AuthName "Mon serveur"
112
113  # Pour mettre en cache les données d'authentification, placez socache
114  # avant dbd
115  AuthBasicProvider socache dbd
116
117  # Aussi nécessaire à la mise en cache : dire au cache de mettre en
118  # cache les recherches dbd !
119  AuthnCacheProvideFor dbd
120  AuthnCacheContext mon-serveur
121
122  # configuration de mod_authz_core
123  Require valid-user
124
125  # la requête SQL de mod_authn_dbd pour authentifier un utilisateur
126  AuthDBDUserPWQuery "SELECT password FROM authn WHERE user = %s"
127&lt;/Directory&gt;</pre>

131<h2><a name="exposed" id="exposed">Mise à disposition des informations de connexion</a></h2>
132
133<p>
134Pour chaque requête envoyée au serveur de
135base de données, toutes les valeurs de colonnes du premier
136enregistrement renvoyé par la requête sont affectées à des variables
137d'environnement avec le préfixe "AUTHENTICATE_".
138</p>
139<p>Par exemple, si une requête renvoie un nom d'utilisateur, un nom
140complet et un numéro de téléphone, un programme CGI pourra accéder à ces
141informations sans avoir besoin d'effectuer une deuxième requête vers la
142base de données.</p>
143<p>Ceci va entraîner une simplification considérable du code et de la
144configuration nécessaire de certaines applications web.
145</p>

148<div class="directive-section"><h2><a name="authdbduserpwquery" id="authdbduserpwquery">Directive</a> <a name="AuthDBDUserPWQuery" id="AuthDBDUserPWQuery">AuthDBDUserPWQuery</a></h2>
149<table class="directive">
150<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Requête SQL servant à vérifier le mot de passe d'un
151utilisateur</td></tr>
152<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>AuthDBDUserPWQuery <var>requête</var></code></td></tr>
153<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>répertoire</td></tr>
154<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Extension</td></tr>
155<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authn_dbd</td></tr>
156</table>
157    <p>La directive <code class="directive">AuthDBDUserPWQuery</code> permet de
158    spécifier une requête servant à vérifier le mot de passe d'un
159    utilisateur donné. L'identifiant utilisateur sera transmis comme
160    paramètre sous forme d'une seule chaîne de caractères lorsque la
161    requête sera exécutée. Cet identifiant est référencé dans la requête
162    en utilisant le spécificateur de format <code>%s</code>.</p>
163    <pre class="prettyprint lang-config">AuthDBDUserPWQuery "SELECT password FROM authn WHERE user = %s"</pre>
164
165    <p>La première colonne du premier enregistrement renvoyé par la
166    requête se présentera sous la forme d'une chaîne de caractères
167    contenant le mot de passe chiffré. Les enregistrements suivants sont
168    ignorés. Si aucun enregistrement n'est renvoyé, l'utilisateur ne
169    sera pas authentifié par <code class="module"><a href="../mod/mod_authn_dbd.html">mod_authn_dbd</a></code>.</p>
170    <p>Toute valeur de colonne supplémentaire
171    du premier enregistrement renvoyé par la requête sera stockée dans
172    une variable d'environnement dont le nom aura la forme
173    <code>AUTHENTICATE_<var>valeur-colonne</var></code>.
174    </p>
175    <p>Le format du mot de passe chiffré dépend du frontal
176    d'authentification utilisé (par exemple
177    <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> ou
178    <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>). Voir la documentation sur les <a href="../misc/password_encryptions.html">Formats de mots de passe</a> pour
179    plus de détails.</p>

183<div class="directive-section"><h2><a name="authdbduserrealmquery" id="authdbduserrealmquery">Directive</a> <a name="AuthDBDUserRealmQuery" id="AuthDBDUserRealmQuery">AuthDBDUserRealmQuery</a></h2>
184<table class="directive">
185<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Requête SQL servant à vérifier une empreinte de mot de
186passe pour un utilisateur et un identifiant d'authentification.
187</td></tr>
188<tr><th><a href="directive-dict.html#Syntax">Syntaxe:</a></th><td><code>AuthDBDUserRealmQuery <var>requête</var></code></td></tr>
189<tr><th><a href="directive-dict.html#Context">Contexte:</a></th><td>répertoire</td></tr>
190<tr><th><a href="directive-dict.html#Status">Statut:</a></th><td>Extension</td></tr>
191<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_authn_dbd</td></tr>
192</table>
193    <p>La directive <code class="directive">AuthDBDUserRealmQuery</code> permet
194    de spécifier une requête SQL servant à vérifier une empreinte de mot
195    de passe pour un utilisateur et un identifiant d'authentification
196    donnés au cours d'un processus d'authentification digest. Les
197    identifiants de l'utilisateur et de l'authentification
198    sont passés dans cet ordre comme paramètres à l'exécution de la
199    requête. Ils sont référencés dans la chaîne de la requête en
200    utilisant des spécificateurs de format <code>%s</code>.</p>
201    <pre class="prettyprint lang-config">AuthDBDUserRealmQuery "SELECT password FROM authn WHERE user = %s AND realm = %s"</pre>
202
203    <p>La première colonne du premier enregistrement renvoyé par la
204    requête se présentera sous la forme d'une chaîne de caractères
205    contenant le mot de passe chiffré. Les enregistrements suivants
206    seront ignorés. Si aucun enregistrement n'est renvoyé, l'utilisateur
207    ne sera pas authentifié par <code class="module"><a href="../mod/mod_authn_dbd.html">mod_authn_dbd</a></code>.</p>
208    <p>Toute valeur de colonne supplémentaire
209    du premier enregistrement renvoyé par la requête sera stockée dans
210    une variable d'environnement avec un nom de la forme
211    <code>AUTHENTICATE_<var>COLONNE</var></code>.
212    </p>
213    <p>Le format du mot de passe chiffré dépend du frontal
214    d'authentification utilisé (par exemple
215    <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> ou
216    <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>). Voir la documentation sur les <a href="../misc/password_encryptions.html">Formats de mots de passe</a> pour
217    plus de détails.</p>
