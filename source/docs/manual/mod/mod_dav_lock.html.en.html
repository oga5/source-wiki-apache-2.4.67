38<h3>Summary</h3>
39
40    <p>This module implements a generic locking API which can be used by any
41    backend provider of <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code>. It <em>requires</em> at least
42    the service of <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code>. But without a backend provider
43    which makes use of it, it's useless and should not be loaded into the
44    server. A sample backend module which actually utilizes
45    <code class="module"><a href="../mod/mod_dav_lock.html">mod_dav_lock</a></code> is <a href="http://subversion.apache.org/">mod_dav_svn</a>, the subversion provider module.</p>
46
47    <p>Note that <code class="module"><a href="../mod/mod_dav_fs.html">mod_dav_fs</a></code> does <em>not</em> need this
48    generic locking module, because it uses its own more specialized
49    version.</p>
50
51    <p>In order to make <code class="module"><a href="../mod/mod_dav_lock.html">mod_dav_lock</a></code> functional, you just have
52    to specify the location of the lock database using the <code class="directive"><a href="#davgenericlockdb">DavGenericLockDB</a></code> directive described
53    below.</p>
54
55    <div class="note"><h3>Developer's Note</h3>
56      <p>In order to retrieve the pointer to the locking provider function, you
57      have to use the <code>ap_lookup_provider</code> API with the arguments
58      <code>dav-lock</code>, <code>generic</code>, and <code>0</code>.</p>
59    </div>

71<div class="directive-section"><h2><a name="DavGenericLockDB" id="DavGenericLockDB">DavGenericLockDB</a> <a name="davgenericlockdb" id="davgenericlockdb">Directive</a></h2>
72<table class="directive">
73<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Location of the DAV lock database</td></tr>
74<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>DavGenericLockDB <var>file-path</var></code></td></tr>
75<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
76<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
77<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_dav_lock</td></tr>
78</table>
79    <p>Use the <code class="directive">DavGenericLockDB</code> directive to specify
80    the full path to the lock database, excluding an extension. If
81    the path is not absolute, it will be interpreted relative to <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code>. The implementation of
82    <code class="module"><a href="../mod/mod_dav_lock.html">mod_dav_lock</a></code> uses a SDBM database to track user
83    locks.</p>
84
85    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">DavGenericLockDB var/DavLock</pre>
86</div>
87
88    <p>The directory containing the lock database file must be
89    writable by the <code class="directive"><a href="../mod/mod_unixd.html#user">User</a></code>
90    and <code class="directive"><a href="../mod/mod_unixd.html#group">Group</a></code> under which
91    Apache is running.  For security reasons, you should create a
92    directory for this purpose rather than changing the permissions on
93    an existing directory.  In the above example, Apache will create
94    files in the <code>var/</code> directory under the <code class="directive"><a href="../mod/core.html#serverroot">ServerRoot</a></code> with the base filename
95    <code>DavLock</code> and an extension added by the server.</p>
