39<h3>Summary</h3>
40
41    <p>This module authorizes access to files by comparing the userid used
42    for HTTP authentication (the web userid) with the file-system owner or
43    group of the requested file. The supplied username and password
44    must be already properly verified by an authentication module,
45    such as <code class="module"><a href="../mod/mod_auth_basic.html">mod_auth_basic</a></code> or
46    <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code>. <code class="module"><a href="../mod/mod_authz_owner.html">mod_authz_owner</a></code>
47    recognizes two arguments for the <code class="directive"><a href="../mod/mod_authz_core.html#require">Require</a></code> directive, <code>file-owner</code> and
48    <code>file-group</code>, as follows:</p>
49
50    <dl>
51      <dt><code>file-owner</code></dt>
52      <dd>The supplied web-username must match the system's name for the
53      owner of the file being requested. That is, if the operating system
54      says the requested file is owned by <code>jones</code>, then the
55      username used to access it through the web must be <code>jones</code>
56      as well.</dd>
57
58      <dt><code>file-group</code></dt>
59      <dd>The name of the system group that owns the file must be present
60      in a group database, which is provided, for example, by <code class="module"><a href="../mod/mod_authz_groupfile.html">mod_authz_groupfile</a></code> or <code class="module"><a href="../mod/mod_authz_dbm.html">mod_authz_dbm</a></code>,
61      and the web-username must be a member of that group. For example, if
62      the operating system says the requested file is owned by (system)
63      group <code>accounts</code>, the group <code>accounts</code> must
64      appear in the group database and the web-username used in the request
65      must be a member of that group.</dd>
66    </dl>
67
68    <div class="note"><h3>Note</h3>
69      <p>If <code class="module"><a href="../mod/mod_authz_owner.html">mod_authz_owner</a></code> is used in order to authorize
70      a resource that is not actually present in the filesystem
71      (<em>i.e.</em> a virtual resource), it will deny the access.</p>
72
73      <p>Particularly it will never authorize <a href="../content-negotiation.html#multiviews">content negotiated
74      "MultiViews"</a> resources.</p>
75    </div>

89<h2><a name="examples" id="examples">Configuration Examples</a></h2>
90
91    <h3><a name="examples.file-owner" id="examples.file-owner">Require file-owner</a></h3>
92      <p>Consider a multi-user system running the Apache Web server, with
93      each user having his or her own files in <code>~/public_html/private</code>. Assuming that there is a single
94      <code class="directive"><a href="../mod/mod_authn_dbm.html#authdbmuserfile">AuthDBMUserFile</a></code> database
95      that lists all of their web-usernames, and that these usernames match
96      the system's usernames that actually own the files on the server, then
97      the following stanza would allow only the user himself access to his
98      own files. User <code>jones</code> would not be allowed to access
99      files in <code>/home/smith/public_html/private</code> unless they
100      were owned by <code>jones</code> instead of <code>smith</code>.</p>
101
102      <pre class="prettyprint lang-config">&lt;Directory "/home/*/public_html/private"&gt;
103    AuthType Basic
104    AuthName MyPrivateFiles
105    AuthBasicProvider dbm
106    AuthDBMUserFile "/usr/local/apache2/etc/.htdbm-all"
107    Require file-owner
108&lt;/Directory&gt;</pre>
109
110    
111
112    <h3><a name="examples.file-group" id="examples.file-group">Require file-group</a></h3>
113      <p>Consider a system similar to the one described above, but with
114      some users that share their project files in
115      <code>~/public_html/project-foo</code>. The files are owned by the
116      system group <code>foo</code> and there is a single <code class="directive"><a href="../mod/mod_authz_dbm.html#authdbmgroupfile">AuthDBMGroupFile</a></code> database that
117      contains all of the web-usernames and their group membership,
118      <em>i.e.</em> they must be at least member of a group named
119      <code>foo</code>. So if <code>jones</code> and <code>smith</code>
120      are both member of the group <code>foo</code>, then both will be
121      authorized to access the <code>project-foo</code> directories of
122      each other.</p>
123
124      <pre class="prettyprint lang-config">&lt;Directory "/home/*/public_html/project-foo"&gt;
125    AuthType Basic
126    AuthName "Project Foo Files"
127    AuthBasicProvider dbm
128    
129    # combined user/group database
130    AuthDBMUserFile  "/usr/local/apache2/etc/.htdbm-all"
131    AuthDBMGroupFile "/usr/local/apache2/etc/.htdbm-all"
132    
133    Satisfy All
134    Require file-group
135&lt;/Directory&gt;</pre>
