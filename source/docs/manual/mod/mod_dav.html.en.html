39<h3>Summary</h3>
40
41    <p>This module provides class 1 and class 2 <a href="http://www.webdav.org">WebDAV</a> ('Web-based Distributed
42    Authoring and Versioning') functionality for Apache. This
43    extension to the HTTP protocol allows creating, moving,
44    copying, and deleting resources and collections on a remote web
45    server.</p>
46</div>

67<h2><a name="example" id="example">Enabling WebDAV</a></h2>
68    <p>To enable <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code>, add the following to a
69    container in your <code>httpd.conf</code> file:</p>
70
71    <pre class="prettyprint lang-config">Dav On</pre>
72
73
74    <p>This enables the DAV file system provider, which is implemented
75    by the <code class="module"><a href="../mod/mod_dav_fs.html">mod_dav_fs</a></code> module. Therefore, that module
76    must be compiled into the server or loaded at runtime using the
77    <code class="directive"><a href="../mod/mod_so.html#loadmodule">LoadModule</a></code> directive.</p>
78
79    <p>In addition, a location for the DAV lock database must be
80    specified in the global section of your <code>httpd.conf</code>
81    file using the <code class="directive"><a href="../mod/mod_dav_fs.html#davlockdb">DavLockDB</a></code>
82    directive:</p>
83
84    <pre class="prettyprint lang-config">DavLockDB /usr/local/apache2/var/DavLock</pre>
85
86
87    <p>The directory containing the lock database file must be
88    writable by the <code class="directive"><a href="../mod/mod_unixd.html#user">User</a></code>
89    and <code class="directive"><a href="../mod/mod_unixd.html#group">Group</a></code> under which
90    Apache is running.</p>
91
92    <p>In the example below, <code class="directive"><a href="../mod/mod_authz_core.html#requireany">&lt;RequireAny&gt;</a></code> is used to ensure that
93    DAV-specific methods (i.e., methods other than <code>GET</code>,
94    <code>POST</code> and <code>OPTIONS</code>) are limited to the
95    authenticated user.</p>
96
97    <p>If you want to set the maximum amount of
98    bytes that a DAV client can send at one request, you have to use
99    the <code class="directive"><a href="../mod/core.html#limitxmlrequestbody">LimitXMLRequestBody</a></code>
100    directive. The "normal" <code class="directive"><a href="../mod/core.html#limitrequestbody">LimitRequestBody</a></code> directive has no effect on DAV
101    requests.</p>
102
103    <div class="example"><h3>Full Example</h3><pre class="prettyprint lang-config">DavLockDB "/usr/local/apache2/var/DavLock"
104
105&lt;Directory "/usr/local/apache2/htdocs/foo"&gt;
106    Dav On
107
108    AuthType Basic
109    AuthName DAV
110    AuthUserFile "user.passwd"
111
112    &lt;RequireAny&gt;
113        Require method GET POST OPTIONS
114        Require user admin
115    &lt;/RequireAny&gt;
116&lt;/Directory&gt;</pre>

121<h2><a name="security" id="security">Security Issues</a></h2>
122
123    <p>Since DAV access methods allow remote clients to manipulate
124    files on the server, you must take particular care to assure that
125    your server is secure before enabling <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code>.</p>
126
127    <p>Any location on the server where DAV is enabled should be
128    protected by authentication.  The use of HTTP Basic Authentication
129    is not recommended. You should use at least HTTP Digest
130    Authentication, which is provided by the
131    <code class="module"><a href="../mod/mod_auth_digest.html">mod_auth_digest</a></code> module. Nearly all WebDAV clients
132    support this authentication method. An alternative is Basic
133    Authentication over an <a href="../ssl/">SSL</a> enabled
134    connection.</p>
135
136    <p>In order for <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code> to manage files, it must
137    be able to write to the directories and files under its control
138    using the <code class="directive"><a href="../mod/mod_unixd.html#user">User</a></code> and
139    <code class="directive"><a href="../mod/mod_unixd.html#group">Group</a></code> under which
140    Apache is running.  New files created will also be owned by this
141    <code class="directive"><a href="../mod/mod_unixd.html#user">User</a></code> and <code class="directive"><a href="../mod/mod_unixd.html#group">Group</a></code>.  For this reason, it is
142    important to control access to this account.  The DAV repository
143    is considered private to Apache; modifying files outside of Apache
144    (for example using FTP or filesystem-level tools) should not be
145    allowed.</p>
146
147    <p><code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code> may be subject to various kinds of
148    denial-of-service attacks.  The <code class="directive"><a href="../mod/core.html#limitxmlrequestbody">LimitXMLRequestBody</a></code> directive can be
149    used to limit the amount of memory consumed in parsing large DAV
150    requests.  The <code class="directive"><a href="#davdepthinfinity">DavDepthInfinity</a></code> directive can be
151    used to prevent <code>PROPFIND</code> requests on a very large
152    repository from consuming large amounts of memory.  Another
153    possible denial-of-service attack involves a client simply filling
154    up all available disk space with many large files.  There is no
155    direct way to prevent this in Apache, so you should avoid giving
156    DAV access to untrusted users.</p>

159<h2><a name="complex" id="complex">Complex Configurations</a></h2>
160
161    <p>One common request is to use <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code> to
162    manipulate dynamic files (PHP scripts, CGI scripts, etc).  This is
163    difficult because a <code>GET</code> request will always run the
164    script, rather than downloading its contents.  One way to avoid
165    this is to map two different URLs to the content, one of which
166    will run the script, and one of which will allow it to be
167    downloaded and manipulated with DAV.</p>
168
169<pre class="prettyprint lang-config">Alias "/phparea" "/home/gstein/php_files"
170Alias "/php-source" "/home/gstein/php_files"
171&lt;Location "/php-source"&gt;
172    Dav On
173    ForceType text/plain
174&lt;/Location&gt;</pre>
175
176
177    <p>With this setup, <code>http://example.com/phparea</code> can be
178    used to access the output of the PHP scripts, and
179    <code>http://example.com/php-source</code> can be used with a DAV
180    client to manipulate them.</p>
181</div>

183<div class="directive-section"><h2><a name="Dav" id="Dav">Dav</a> <a name="dav" id="dav">Directive</a></h2>
184<table class="directive">
185<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Enable WebDAV HTTP methods</td></tr>
186<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>Dav On|Off|<var>provider-name</var></code></td></tr>
187<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>Dav Off</code></td></tr>
188<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>directory</td></tr>
189<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
190<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_dav</td></tr>
191</table>
192    <p>Use the <code class="directive">Dav</code> directive to enable the
193    WebDAV HTTP methods for the given container:</p>
194
195    <pre class="prettyprint lang-config">&lt;Location "/foo"&gt;
196    Dav On
197&lt;/Location&gt;</pre>
198
199
200    <p>The value <code>On</code> is actually an alias for the default
201    provider <code>filesystem</code> which is served by the <code class="module"><a href="../mod/mod_dav_fs.html">mod_dav_fs</a></code> module. Note, that once you have DAV enabled
202    for some location, it <em>cannot</em> be disabled for sublocations.
203    For a complete configuration example have a look at the <a href="#example">section above</a>.</p>
204
205    <div class="warning">
206      Do not enable WebDAV until you have secured your server. Otherwise
207      everyone will be able to distribute files on your system.
208    </div>

212<div class="directive-section"><h2><a name="DavBasePath" id="DavBasePath">DavBasePath</a> <a name="davbasepath" id="davbasepath">Directive</a></h2>
213<table class="directive">
214<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Configure repository root path</td></tr>
215<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>DavBasePath <var>root-path</var></code></td></tr>
216<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>None</code></td></tr>
217<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>directory</td></tr>
218<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
219<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_dav</td></tr>
220<tr><th><a href="directive-dict.html#Compatibility">Compatibility:</a></th><td>Available in version 2.4.58 and later</td></tr>
221</table>
222  <p>If a DAV repository is configured using a regular expression
223  match (such as <code class="directive"><a href="../mod/core.html#locationmatch">LocationMatch</a></code>)
224  then <code class="module"><a href="../mod/mod_dav.html">mod_dav</a></code> will not be able to find the root of
225  the repository from the pathname alone. Third-party providers (for
226  example, Subversion's <a href="https://svnbook.red-bean.com/en/1.7/svn.ref.mod_dav_svn.conf.html">mod_dav_svn</a>)
227  may fail to handle requests without the correct repository root.</p>
228
229  <p>To allow providers to work correctly in such a configuration,
230  <code class="directive">DavBasePath</code> must be used.</p>
231
232    <pre class="prettyprint lang-config">&lt;LocationMatch "^/repos/"&gt;
233    Dav svn
234    DavBasePath /repos
235    SVNParentPath /var/svn
236&lt;/LocationMatch&gt;</pre>

244<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>DavDepthInfinity on|off</code></td></tr>

250    <p>Use the <code class="directive">DavDepthInfinity</code> directive to
251    allow the processing of <code>PROPFIND</code> requests containing the
252    header 'Depth: Infinity'. Because this type of request could constitute
253    a denial-of-service attack, by default it is not allowed.</p>

257<div class="directive-section"><h2><a name="DavMinTimeout" id="DavMinTimeout">DavMinTimeout</a> <a name="davmintimeout" id="davmintimeout">Directive</a></h2>
258<table class="directive">
259<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Minimum amount of time the server holds a lock on
260a DAV resource</td></tr>
261<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>DavMinTimeout <var>seconds</var></code></td></tr>
262<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>DavMinTimeout 0</code></td></tr>
263<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory</td></tr>
264<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Extension</td></tr>
265<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_dav</td></tr>
266</table>
267    <p>When a client requests a DAV resource lock, it can also
268    specify a time when the lock will be automatically removed by
269    the server. This value is only a request, and the server can
270    ignore it or inform the client of an arbitrary value.</p>
271
272    <p>Use the <code class="directive">DavMinTimeout</code> directive to specify, in
273    seconds, the minimum lock timeout to return to a client.
274    Microsoft Web Folders defaults to a timeout of 120 seconds; the
275    <code class="directive">DavMinTimeout</code> can override this to a higher value
276    (like 600 seconds) to reduce the chance of the client losing
277    the lock due to network latency.</p>
278
279    <div class="example"><h3>Example</h3><pre class="prettyprint lang-config">&lt;Location "/MSWord"&gt;
280    DavMinTimeout 600
281&lt;/Location&gt;</pre>
