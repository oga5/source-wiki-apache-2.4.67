40<h3>Summary</h3>
41
42
43    <p>The <code class="module"><a href="../mod/mod_setenvif.html">mod_setenvif</a></code> module allows you to set
44    internal environment variables according to whether different aspects of
45    the request match regular expressions you specify. These
46    environment variables can be used by other parts of the server
47    to make decisions about actions to be taken, as well as becoming
48    available to CGI scripts and SSI pages.</p>
49
50    <p>The directives are considered in the order they appear in
51    the configuration files. So more complex sequences can be used,
52    such as this example, which sets <code>netscape</code> if the
53    browser is mozilla but not MSIE.</p>
54
55<pre class="prettyprint lang-config">BrowserMatch ^Mozilla netscape
56BrowserMatch MSIE !netscape</pre>
57
58
59   <p>When the server looks up a path via an internal 
60   <a class="glossarylink" href="../glossary.html#subrequest" title="see glossary">subrequest</a> such as looking 
61   for a <code class="directive"><a href="../mod/mod_dir.html#directoryindex">DirectoryIndex</a></code> 
62   or generating a directory listing with <code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code>,
63   per-request environment variables are <em>not</em> inherited in the 
64   subrequest. Additionally, 
65   <code class="directive"><a href="#setenvif">SetEnvIf</a></code> directives
66   are not separately evaluated in the subrequest due to the API phases
67   <code class="module"><a href="../mod/mod_setenvif.html">mod_setenvif</a></code> takes action in.</p>
68
69   <p>A regular expression only needs quoting when it contains space, 
70   in which case single and double quotes are equivalent. Unlike 
71   <code class="module"><a href="../mod/mod_rewrite.html">mod_rewrite</a></code>, <code class="module"><a href="../mod/mod_setenvif.html">mod_setenvif</a></code> strips every 
72   other backslash when parsing the expression; thus <code>\\</code> 
73   requires <code>\\\</code>, and <code>\\\</code> requires 
74   <code>\\\\\</code>.</p>

91<div class="directive-section"><h2><a name="BrowserMatch" id="BrowserMatch">BrowserMatch</a> <a name="browsermatch" id="browsermatch">Directive</a></h2>
92<table class="directive">
93<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Sets environment variables conditional on HTTP User-Agent
94</td></tr>
95<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>BrowserMatch <em>regex [!]env-variable</em>[=<em>value</em>]
96[[!]<em>env-variable</em>[=<em>value</em>]] ...</code></td></tr>
97<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
98<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>FileInfo</td></tr>
99<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
100<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_setenvif</td></tr>
101</table>
102  <p>The <code class="directive">BrowserMatch</code> is a special cases of the
103  <code class="directive"><a href="#setenvif">SetEnvIf</a></code> directive that
104  sets environment variables conditional on the
105  <code>User-Agent</code> HTTP request header.  The following two
106  lines have the same effect:</p>
107<pre class="prettyprint lang-config">BrowserMatch Robot is_a_robot
108SetEnvIf User-Agent Robot is_a_robot</pre>
109
110
111    <p>Some additional examples:</p>
112<pre class="prettyprint lang-config">BrowserMatch ^Mozilla forms jpeg=yes browser=netscape
113BrowserMatch "^Mozilla/[2-3]" tables agif frames javascript
114BrowserMatch MSIE !javascript</pre>

119<div class="directive-section"><h2><a name="BrowserMatchNoCase" id="BrowserMatchNoCase">BrowserMatchNoCase</a> <a name="browsermatchnocase" id="browsermatchnocase">Directive</a></h2>
120<table class="directive">
121<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Sets environment variables conditional on User-Agent without
122respect to case</td></tr>
123<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>BrowserMatchNoCase  <em>regex [!]env-variable</em>[=<em>value</em>]
124    [[!]<em>env-variable</em>[=<em>value</em>]] ...</code></td></tr>
125<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
126<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>FileInfo</td></tr>
127<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
128<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_setenvif</td></tr>
129</table>
130
131    <p>The <code class="directive">BrowserMatchNoCase</code> directive is
132    semantically identical to the <code class="directive"><a href="#browsermatch">BrowserMatch</a></code> directive.
133    However, it provides for case-insensitive matching. For
134    example:</p>
135<pre class="prettyprint lang-config">BrowserMatchNoCase mac platform=macintosh
136BrowserMatchNoCase win platform=windows</pre>
137
138
139    <p>The <code class="directive">BrowserMatch</code> and
140    <code class="directive">BrowserMatchNoCase</code> directives are special cases of
141    the <code class="directive"><a href="#setenvif">SetEnvIf</a></code> and <code class="directive"><a href="#setenvifnocase">SetEnvIfNoCase</a></code>
142    directives. The following two lines have the same effect:</p>
143<pre class="prettyprint lang-config">BrowserMatchNoCase Robot is_a_robot
144SetEnvIfNoCase User-Agent Robot is_a_robot</pre>
145

149<div class="directive-section"><h2><a name="SetEnvIf" id="SetEnvIf">SetEnvIf</a> <a name="setenvif" id="setenvif">Directive</a></h2>
150<table class="directive">
151<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Sets environment variables based on attributes of the request
152</td></tr>
153<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>SetEnvIf <em>attribute
154    regex [!]env-variable</em>[=<em>value</em>]
155    [[!]<em>env-variable</em>[=<em>value</em>]] ...</code></td></tr>
156<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
157<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>FileInfo</td></tr>
158<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
159<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_setenvif</td></tr>
160</table>
161    <p>The <code class="directive">SetEnvIf</code> directive defines
162    environment variables based on attributes of the request. The
163    <em>attribute</em> specified in the first argument can be one of four
164    things:</p>
165
166<ol>
167<li>An HTTP request header field (see <a href="http://www.rfc-editor.org/rfc/rfc2616.txt">RFC2616</a>
168    for more information about these); for example: <code>Host</code>,
169    <code>User-Agent</code>, <code>Referer</code>, and
170    <code>Accept-Language</code>.  A regular expression may be
171    used to specify a set of request headers.</li>
172
173<li>One of the following aspects of the request:
174    <ul>
175      <li><code>Remote_Host</code> - the hostname (if available) of
176      the client making the request</li>
177
178      <li><code>Remote_Addr</code> - the IP address of the client
179      making the request</li>
180
181      <li><code>Server_Addr</code> - the IP address of the server
182      on which the request was received (only with versions later
183      than 2.0.43)</li>
184
185      <li><code>Request_Method</code> - the name of the method
186      being used (<code>GET</code>, <code>POST</code>, <em>et
187      cetera</em>)</li>
188
189      <li><code>Request_Protocol</code> - the name and version of
190      the protocol with which the request was made (<em>e.g.</em>,
191      "HTTP/0.9", "HTTP/1.1", <em>etc.</em>)</li>
192
193      <li><code>Request_URI</code> - the resource requested on the HTTP
194       request line -- generally the portion of the URL
195      following the scheme and host portion without the query string. See
196      the <code class="directive"><a href="../mod/mod_rewrite.html#rewritecond">RewriteCond</a></code>
197      directive of <code class="module"><a href="../mod/mod_rewrite.html">mod_rewrite</a></code> for extra information on
198      how to match your query string.</li>
199    </ul>
200</li>
201
202<li>The name of an environment variable in the list of those
203associated with the request. This allows
204<code class="directive">SetEnvIf</code> directives to test against the result
205of prior matches. Only those environment variables defined by earlier
206<code>SetEnvIf[NoCase]</code> directives are available for testing in
207this manner. 'Earlier' means that they were defined at a broader scope
208(such as server-wide) or previously in the current directive's scope.
209Environment variables will be considered only if there was no match
210among request characteristics and a regular expression was not
211used for the <em>attribute</em>.</li>
212
213</ol>
214
215<p>The second argument (<em>regex</em>) is a <a class="glossarylink" href="../glossary.html#regex" title="see glossary">regular expression</a>.  If the <em>regex</em>
216matches against the <em>attribute</em>, then the remainder of the
217arguments are evaluated.</p>
218
219<p>The rest of the arguments give the names of variables to set, and
220optionally values to which they should be set. These take the form
221of</p>
222
223    <ol>
224      <li><code><em>varname</em></code>, or</li>
225
226      <li><code>!<em>varname</em></code>, or</li>
227
228      <li><code><em>varname</em>=<em>value</em></code></li>
229    </ol>
230
231    <p>In the first form, the value will be set to "1". The second
232    will remove the given variable if already defined, and the
233    third will set the variable to the literal value given by
234    <code><em>value</em></code>. Since version 2.0.51, Apache httpd will
235    recognize occurrences of <code>$1</code>..<code>$9</code> within
236    <var>value</var> and replace them by parenthesized subexpressions
237    of <var>regex</var>. <code>$0</code> provides access to the whole
238    string matched by that pattern.</p>
239
240<pre class="prettyprint lang-config">SetEnvIf Request_URI "\.gif$" object_is_image=gif
241SetEnvIf Request_URI "\.jpg$" object_is_image=jpg
242SetEnvIf Request_URI "\.xbm$" object_is_image=xbm
243    
244SetEnvIf Referer www\.mydomain\.example\.com intra_site_referral
245    
246SetEnvIf object_is_image xbm XBIT_PROCESSING=1
247    
248SetEnvIf Request_URI "\.(.*)$" EXTENSION=$1
249
250SetEnvIf ^TS  ^[a-z]  HAVE_TS</pre>
251
252
253    <p>The first three will set the environment variable
254    <code>object_is_image</code> if the request was for an image
255    file, and the fourth sets <code>intra_site_referral</code> if
256    the referring page was somewhere on the
257    <code>www.mydomain.example.com</code> Web site.</p>
258
259    <p>The last example will set environment variable
260    <code>HAVE_TS</code> if the request contains any headers that
261    begin with "TS" whose values begins with any character in the
262    set [a-z].</p>

272<div class="directive-section"><h2><a name="SetEnvIfExpr" id="SetEnvIfExpr">SetEnvIfExpr</a> <a name="setenvifexpr" id="setenvifexpr">Directive</a></h2>
273<table class="directive">
274<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Sets environment variables based on an ap_expr expression</td></tr>
275<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>SetEnvIfExpr <em>expr
276    [!]env-variable</em>[=<em>value</em>]
277    [[!]<em>env-variable</em>[=<em>value</em>]] ...</code></td></tr>
278<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
279<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>FileInfo</td></tr>
280<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
281<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_setenvif</td></tr>
282</table>
283    <p>The <code class="directive">SetEnvIfExpr</code> directive defines
284    environment variables based on an <a href="../expr.html">expression</a>.
285    These expressions will be evaluated at runtime,
286    and applied <em>env-variable</em> in the same fashion as <code class="directive">SetEnvIf</code>, including backreferences.</p>
287
288<pre class="prettyprint lang-config">    SetEnvIfExpr "tolower(req('X-Sendfile')) == 'd:\images\very_big.iso')" iso_delivered
289    SetEnvIfExpr "tolower(req('X-Sendfile')) =~ /(.*\.iso$)/" iso-path=$1</pre>
290
291
292    <p>This would set the environment variable <code>iso_delivered</code>
293    every time our application attempts to send it via <code>X-Sendfile</code></p>
294
295    <p>A more useful example would be to set the variable rfc1918 if the
296    remote IP address is a private address according to RFC 1918:</p>
297
298<pre class="prettyprint lang-config">SetEnvIfExpr "-R '10.0.0.0/8' || -R '172.16.0.0/12' || -R '192.168.0.0/16'" rfc1918</pre>
299

311<div class="directive-section"><h2><a name="SetEnvIfNoCase" id="SetEnvIfNoCase">SetEnvIfNoCase</a> <a name="setenvifnocase" id="setenvifnocase">Directive</a></h2>
312<table class="directive">
313<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Sets environment variables based on attributes of the request
314without respect to case</td></tr>
315<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>SetEnvIfNoCase <em>attribute regex
316        [!]env-variable</em>[=<em>value</em>]
317    [[!]<em>env-variable</em>[=<em>value</em>]] ...</code></td></tr>
318<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
319<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>FileInfo</td></tr>
320<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
321<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_setenvif</td></tr>
322</table>
323
324    <p>The <code class="directive">SetEnvIfNoCase</code> is semantically identical to
325    the <code class="directive"><a href="#setenvif">SetEnvIf</a></code> directive,
326    and differs only in that the regular expression matching is
327    performed in a case-insensitive manner. For example:</p>
328<pre class="prettyprint lang-config">SetEnvIfNoCase Host Example\.Org site=example</pre>
329
330
331    <p>This will cause the <code>site</code> environment variable
332    to be set to "<code>example</code>" if the HTTP request header
333    field <code>Host:</code> was included and contained
334    <code>Example.Org</code>, <code>example.org</code>, or any other
335    combination.</p>
