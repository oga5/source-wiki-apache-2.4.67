52<h2><a name="Syntax" id="Syntax">Syntaxe</a></h2>
53
54    <p>Ce terme introduit le format sous lequel la directive doit
55    apparaître dans le fichier de configuration. Cette syntaxe est très
56    spécifique à la directive et est décrite en détail dans la
57    définition de cette dernière. En général, le nom de la directive est
58    suivi d'un ou plusieurs arguments séparés par des espaces. Si un
59    argument contient un espace, il doit être entouré de guillemets. Les
60    arguments optionnels sont entourés de crochets. Lorsqu'un argument
61    accepte une valeur parmi une liste de valeurs possibles, cette liste
62    est spécifiée en séparant les valeurs par une barre verticale "|".
63    Les textes littéraux sont présentés dans la fonte par défaut, alors
64    que les types d'argument pour lesquels une substitution est
65    nécessaire sont en <em>gras</em>. La syntaxe des directives
66    acceptant un nombre variable d'arguments se termine par "...", ce
67    qui indique que le dernier argument peut être répété.</p>
68
69    <p>Les directives utilisent un grand nombre de types d'arguments
70    différents. Les plus courants sont définis ci-dessous.</p>
71
72    <dl>
73      <dt><em>URL</em></dt>
74
75      <dd>Un Localisateur de Ressource Uniforme (Uniform Resource
76      Locator) complet comportant un protocole, un nom d'hôte et un nom
77      de chemin optionnel comme dans
78      <code>http://www.example.com/chemin/vers/fichier.html</code></dd>
79
80      <dt><em>chemin-URL</em></dt>
81
82      <dd>La partie de l'<em>url</em> qui suit le protocole et le nom
83      d'hôte comme dans <code>/chemin/vers/fichier.html</code>. Le
84      <em>chemin-URL</em> représente la ressource vue du web, et est
85      différente de la représentation de cette même ressource vue du
86      système de fichiers.</dd>
87
88      <dt><em>chemin-fichier</em></dt>
89
90      <dd>Le chemin d'un fichier dans le système de fichiers local
91      commençant par le répertoire racine comme dans
92      <code>/usr/local/apache/htdocs/chemin/vers/fichier.html</code>.
93      Sauf mention contraire, un <em>chemin-fichier</em> qui ne commence
94      pas par un slash sera considéré comme relatif au répertoire défini
95      par la directive <a href="core.html#serverroot">ServerRoot</a>.</dd>
96
97      <dt><em>chemin-répertoire</em></dt>
98
99      <dd>Le chemin d'un répertoire dans le système de fichiers local
100      commençant par le répertoire racine comme dans
101      <code>/usr/local/apache/htdocs/chemin/vers/</code>.</dd>
102
103      <dt><em>nom-fichier</em></dt>
104
105      <dd>Le nom d'un fichier sans son chemin comme dans
106      <code>fichier.html</code>.</dd>
107
108      <dt><em>regex</em></dt>
109
110      <dd>Une <a class="glossarylink" href="../glossary.html#regex" title="voir glossaire">expression rationnelle</a>
111      compatible Perl. La définition de la directive spécifiera à quoi
112      <em>regex</em> sera comparée.</dd>
113
114      <dt><em>extension</em></dt>
115
116      <dd>En général, c'est la partie du <em>nom de fichier</em> qui
117      suit le dernier point. Cependant, Apache reconnaît plusieurs
118      extensions de noms de fichiers ; ainsi, si un <em>nom de
119      fichier</em>
120      contient plusieurs points, chacune des parties du nom de fichier
121      séparées par des points et situées après le premier point est une
122      <em>extension</em>. Par exemple, le <em>nom de fichier</em>
123      <code>fichier.html.en</code> comporte deux extensions :
124      <code>.html</code> et <code>.en</code>. Pour les directives
125      Apache, vous pouvez spécifier les <em>extension</em>s avec ou sans
126      le point initial. Enfin, les <em>extension</em>s ne sont pas
127      sensibles à la casse.</dd>
128
129      <dt><em>MIME-type</em></dt>
130
131      <dd>Une méthode de description du format d'un fichier consistant
132      en un type de format majeur et un type de format mineur séparés
133      par un slash comme dans <code>text/html</code>.</dd>
134
135      <dt><em>env-variable</em></dt>
136
137      <dd>Le nom d'une <a href="../env.html">variable
138      d'environnement</a> définie au cours du processus de configuration
139      d'Apache. Notez qu'elle peut être différente d'une variable
140      d'environnement du système d'exploitation. Voir la <a href="../env.html">documentation sur les variables d'environnement</a>
141      pour plus de détails.</dd>
142    </dl>

145<h2><a name="Default" id="Default">Défaut</a></h2>
146
147    <p>Si la directive possède une valeur par défaut (en d'autres
148    termes, si le serveur Web Apache se comporte comme si vous l'aviez
149    définie à une valeur particulière, alors que vous l'avez omise dans
150    votre configuration), elle est spécifiée ici. Si la directive ne
151    possède pas de valeur par défaut, cette section doit spécifier
152    "<em>Aucune</em>". Notez que la valeur par défaut dont il est
153    question n'est pas nécessairement la même que la valeur attribuée à
154    la directive dans le fichier httpd.conf par défaut distribué avec le
155    serveur.</p>
156</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>
157<div class="section">
158<h2><a name="Context" id="Context">Contexte</a></h2>
159
160    <p>Indique les parties des fichiers de configuration du serveur
161    où cette directive est valide. Il s'agit d'une liste d'une ou
162    plusieurs des valeurs suivantes séparées par des virgules :</p>
163
164    <dl>
165      <dt>configuration globale</dt>
166
167      <dd>Signifie que la directive peut être utilisée dans les fichiers
168      de configuration globale (par exemple <code>httpd.conf</code>),
169      mais <strong>pas</strong> à l'intérieur d'un conteneur <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code> ou <code class="directive"><a href="../mod/core.html#directory">&lt;Directory&gt;</a></code>. De même, elle
170      n'est pas valide dans les fichiers <code>.htaccess</code>.</dd>
171
172      <dt>serveur virtuel</dt>
173
174      <dd>Signifie que la directive peut apparaître à l'intérieur d'un
175      conteneur <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code> dans les fichiers de
176      configuration du serveur.</dd>
177
178      <dt>répertoire</dt>
179
180      <dd>Une directive spécifiée comme valide dans ce contexte peut
181      être utilisée à l'intérieur de conteneurs <code class="directive"><a href="../mod/core.html#directory">&lt;Directory&gt;</a></code>, <code class="directive"><a href="../mod/core.html#location">&lt;Location&gt;</a></code>, <code class="directive"><a href="../mod/core.html#files">&lt;Files&gt;</a></code>, <code class="directive"><a href="../mod/core.html#if">&lt;If&gt;</a></code>, et <code class="directive"><a href="../mod/mod_proxy.html#proxy">&lt;Proxy&gt;</a></code> dans les
182      fichiers de configuration du serveur, en tenant compte des
183      restrictions précisées dans la documentation sur les <a href="../sections.html">Sections de configuration</a>.</dd>
184
185      <dt>.htaccess</dt>
186
187      <dd>Si une directive est valide dans ce contexte, cela signifie
188      qu'elle peut apparaître à l'intérieur de fichiers de configuration
189      de <em>niveau</em> répertoire <code>.htaccess</code>. Elle sera ou
190      ne sera pas traitée, selon la définition de l'option <a href="#Override">overrides</a> pour le contexte courant.</dd>
191    </dl>
192
193    <p>La directive n'est autorisée <em>que</em> dans le contexte
194    désigné ; si vous essayez de l'utiliser ailleurs, vous générerez une
195    erreur de configuration qui va soit empêcher le serveur de traiter
196    les requêtes correctement dans ce contexte, soit tout simplement
197    empêcher le serveur de fonctionner -- en d'autres termes, le serveur
198    refusera de démarrer.</p>
199
200    <p>Les lieux de définition valides pour une directive résultent en
201    fait d'un
202    OU logique de tous les contextes spécifiés. En d'autres termes, une
203    directive spécifiée comme valide dans "<code>configuration globale,
204    .htaccess</code>" peut être utilisée dans le fichier
205    <code>httpd.conf</code> et dans les fichiers <code>.htaccess</code>,
206    mais pas dans un conteneur <code class="directive"><a href="../mod/core.html#directory">&lt;Directory&gt;</a></code> ou <code class="directive"><a href="../mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>.</p>

209<h2><a name="Override" id="Override">Surcharge/Écrasement</a></h2>
210
211    <p>Ce terme indique quelle autorisation de surcharge ("override") doit être
212    active pour que la directive puisse être traitée lorsqu'elle
213    apparaît dans un fichier <code>.htaccess</code>. Si le <a href="#Context">context</a> de la directive ne lui permet pas
214    d'apparaître dans un fichier <code>.htaccess</code>, aucun contexte
215    ne sera spécifié.</p>
216
217    <p>Les autorisations de surcharge sont activées via la directive
218    <code class="directive"><a href="../mod/core.html#allowoverride">AllowOverride</a></code>, et possèdent une
219    portée particulière, comme un répertoire et tous ses
220    sous-répertoires, sauf si une autre directive <code class="directive"><a href="../mod/core.html#allowoverride">AllowOverride</a></code> apparaît à un niveau
221    inférieur. La documentation pour cette directive spécifie aussi les
222    noms d'autorisations de surcharge disponibles.</p>
223</div><div class="top"><a href="#page-header"><img alt="top" src="../images/up.gif" /></a></div>
224<div class="section">
225<h2><a name="Status" id="Status">Statut</a></h2>
226
227    <p>Cet attribut indique le degré de rapprochement de la directive du
228    coeur d'Apache ; en d'autres termes, vous pouvez être amené à
229    recompiler le serveur avec un jeu de modules supplémentaires pour
230    pouvoir utiliser la directive, et ainsi accéder à ses
231    fonctionnalités. Les valeurs possible pour cet attribut sont :</p>
232
233    <dl>
234      <dt>Core</dt>
235
236      <dd>Lorsqu'une directive a pour statut "Core", cela signifie
237      qu'elle fait partie du coeur du serveur web Apache, et est de ce
238      fait toujours disponible.</dd>
239
240      <dt>MPM</dt>
241
242      <dd>Une directive dont le statut est "MPM" est fournie par un <a href="../mpm.html">module Multi-Processus</a>. Ce type de
243      directive sera disponible si et seulement si vous utilisez un des
244      MPMs spécifiés dans la ligne <a href="#Module">Module</a> de la
245      définition de la directive.</dd>
246
247      <dt>Base</dt>
248
249      <dd>Une directive dont le statut est "Base" est fournie par un des
250      modules Apache standards qui sont compilés dans le serveur par
251      défaut, et sont de ce fait toujours disponibles, sauf si vous avez
252      fait en sorte de les supprimer de votre configuration.</dd>
253
254      <dt>Extension</dt>
255
256      <dd>Une directive dont le statut est "Extension" est fournie par
257      un des modules inclus dans le kit du serveur Apache, mais qui ne
258      sont pas compilés dans le serveur par défaut. Pour activer la
259      directive et accéder à ses fonctionnalités, vous devez modifier
260      les fichiers de configuration de la compilation du serveur, et
261      recompiler Apache.</dd>
262
263      <dt>Expérimental</dt>
264
265      <dd>Le statut "Expérimental" indique que la directive fait partie
266      du kit Apache, mais que vous l'utilisez à vos risques et périls.
267      La directive est documentée à titre d'exhaustivité, et n'est pas
268      obligatoirement supportée. Le module qui fournit la directive peut
269      être compilé par défaut dans le serveur ou non ; consultez le haut
270      de la page qui décrit la directive et son module pour vérifier sa
271      disponibilité.</dd>
272    </dl>
