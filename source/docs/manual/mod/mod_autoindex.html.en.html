41<h3>Summary</h3>
42
43      <p>The index of a directory can come from one of two
44      sources:</p>
45
46    <ul>
47      <li>A file located in that directory, typically called
48      <code>index.html</code>. The <code class="directive"><a href="../mod/mod_dir.html#directoryindex">DirectoryIndex</a></code> directive sets the
49      name of the file or files to be used. This is controlled by
50      <code class="module"><a href="../mod/mod_dir.html">mod_dir</a></code>.</li>
51
52      <li>Otherwise, a listing generated by the server. The other
53      directives control the format of this listing. The <code class="directive"><a href="#addicon">AddIcon</a></code>, <code class="directive"><a href="#addiconbyencoding">AddIconByEncoding</a></code> and
54      <code class="directive"><a href="#addiconbytype">AddIconByType</a></code> are
55      used to set a list of icons to display for various file types;
56      for each file listed, the first icon listed that matches the
57      file is displayed. These are controlled by
58      <code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code>.</li>
59    </ul>
60    <p>The two functions are separated so that you can completely
61    remove (or replace) automatic index generation should you want
62    to.</p>
63
64    <p>Automatic index generation is enabled with using
65    <code>Options +Indexes</code>. See the
66    <code class="directive"><a href="../mod/core.html#options">Options</a></code> directive for
67    more details.</p>
68
69    <p>If the <code><a href="#indexoptions.fancyindexing">FancyIndexing</a></code> option is given with the <code class="directive"><a href="#indexoptions">IndexOptions</a></code> directive,
70    the column headers are links that control the order of the
71    display. If you select a header link, the listing will be
72    regenerated, sorted by the values in that column. Selecting the
73    same header repeatedly toggles between ascending and descending
74    order. These column header links are suppressed with the
75    <code class="directive"><a href="#indexoptions">IndexOptions</a></code> directive's
76    <code><a href="#indexoptions.suppresscolumnsorting">SuppressColumnSorting</a></code>
77    option.</p>
78
79    <p>Note that when the display is sorted by "Size", it's the
80    <em>actual</em> size of the files that's used, not the
81    displayed value - so a 1010-byte file will always be displayed
82    before a 1011-byte file (if in ascending order) even though
83    they both are shown as "1K".</p>

112<h2><a name="query" id="query">Autoindex Request Query Arguments</a></h2>
113    
114
115    <p>Various query string arguments are available to give the client
116    some control over the ordering of the directory listing, as well as
117    what files are listed. If you do not wish to give the client this
118    control, the <code><a href="#indexoptions.ignoreclient">IndexOptions
119    IgnoreClient</a></code> option disables that functionality.</p>
120
121    <p>The column sorting headers themselves are self-referencing
122    hyperlinks that add the sort query options shown below. Any
123    option below may be added to any request for the directory
124    resource.</p>
125
126    <ul>
127      <li><code>C=N</code> sorts the directory by file name</li>
128
129      <li><code>C=M</code> sorts the directory by last-modified
130      date, then file name</li>
131
132      <li><code>C=S</code> sorts the directory by size, then file
133      name</li>
134
135      <li class="separate"><code>C=D</code> sorts the directory by description, then
136      file name</li>
137
138      <li><code>O=A</code> sorts the listing in Ascending
139      Order</li>
140
141      <li class="separate"><code>O=D</code> sorts the listing in Descending
142      Order</li>
143
144      <li><code>F=0</code> formats the listing as a simple list
145      (not FancyIndexed)</li>
146
147      <li><code>F=1</code> formats the listing as a FancyIndexed
148      list</li>
149
150      <li class="separate"><code>F=2</code> formats the listing as an
151      HTMLTable FancyIndexed list</li>
152
153      <li><code>V=0</code> disables version sorting</li>
154
155      <li class="separate"><code>V=1</code> enables version sorting</li>
156
157      <li><code>P=<var>pattern</var></code> lists only files matching
158      the given <var>pattern</var></li>
159    </ul>
160
161    <p>Note that the 'P'attern query argument is tested
162    <em>after</em> the usual <code class="directive"><a href="#indexignore">IndexIgnore</a></code> directives are processed,
163    and all file names are still subjected to the same criteria as
164    any other autoindex listing. The Query Arguments parser in
165    <code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code> will stop abruptly when an unrecognized
166    option is encountered. The Query Arguments must be well formed,
167    according to the table above.</p>
168
169    <p>The simple example below, which can be clipped and saved in
170    a header.html file, illustrates these query options. Note that
171    the unknown "X" argument, for the submit button, is listed last
172    to assure the arguments are all parsed before <code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code>
173    encounters the X=Go input.</p>
174
175    <div class="example"><h3>Example</h3><pre class="prettyprint lang-html">&lt;form action="" method="get"&gt;
176    Show me a &lt;select name="F"&gt;
177        &lt;option value="0"&gt; Plain list&lt;/option&gt;
178        &lt;option value="1" selected="selected"&gt; Fancy list&lt;/option&gt;
179        &lt;option value="2"&gt; Table list&lt;/option&gt;
180    &lt;/select&gt;
181    Sorted by &lt;select name="C"&gt;
182        &lt;option value="N" selected="selected"&gt; Name&lt;/option&gt;
183        &lt;option value="M"&gt; Date Modified&lt;/option&gt;
184        &lt;option value="S"&gt; Size&lt;/option&gt;
185        &lt;option value="D"&gt; Description&lt;/option&gt;
186    &lt;/select&gt;
187    &lt;select name="O"&gt;
188        &lt;option value="A" selected="selected"&gt; Ascending&lt;/option&gt;
189        &lt;option value="D"&gt; Descending&lt;/option&gt;
190    &lt;/select&gt;
191    &lt;select name="V"&gt;
192        &lt;option value="0" selected="selected"&gt; in Normal order&lt;/option&gt;
193        &lt;option value="1"&gt; in Version order&lt;/option&gt;
194    &lt;/select&gt;
195    Matching &lt;input type="text" name="P" value="*" /&gt;
196    &lt;input type="submit" name="X" value="Go" /&gt;
197&lt;/form&gt;</pre>

284    <p>This sets the description to display for a file, for
285    <code><a href="#indexoptions.fancyindexing">FancyIndexing</a></code>.
286    <var>File</var> is a file extension, partial filename, wild-card
287    expression or full filename for files to describe.
288    <var>String</var> is enclosed in double quotes (<code>"</code>).</p>
289
290    <pre class="prettyprint lang-config">AddDescription "The planet Mars" mars.gif
291AddDescription "My friend Marshall" friends/mars.gif</pre>
292
293
294    <p>The typical, default description field is 23 bytes wide. 6
295    more bytes are added by the <code><a href="#indexoptions.suppressicon">IndexOptions SuppressIcon</a></code> option, 7 bytes are
296    added by the <code><a href="#indexoptions.suppresssize">IndexOptions SuppressSize</a></code> option, and 19 bytes are
297    added by the <code><a href="#indexoptions.suppresslastmodified">IndexOptions SuppressLastModified</a></code> option.
298    Therefore, the widest default the description column is ever
299    assigned is 55 bytes.</p>
300
301    <p>Since the <var>File</var> argument may be a partial file name,
302    please remember that a too-short partial filename may match
303    unintended files. For example, <code>le.html</code> will match the
304    file <code>le.html</code> but will also match the file
305    <code>example.html</code>. In the event that there may be ambiguity,
306    use as complete a filename as you can, but keep in mind that the
307    first match encountered will be used, and order your list of
308    <code>AddDescription</code> directives accordingly.</p>
309
310    <p>See the <a href="#indexoptions.descriptionwidth">DescriptionWidth</a> <code class="directive"><a href="#indexoptions">IndexOptions</a></code> keyword for details on overriding the size
311    of this column, or allowing descriptions of unlimited length.</p>
312
313    <div class="note"><h3>Caution</h3>
314      <p>Descriptive text defined with <code class="directive">AddDescription</code>
315      may contain HTML markup, such as tags and character entities. If the
316      width of the description column should happen to truncate a tagged
317      element (such as cutting off the end of a bolded phrase), the
318      results may affect the rest of the directory listing.</p>
319    </div>
320
321    <div class="note"><h3>Arguments with path information</h3>
322      <p>Absolute paths are not currently supported and do not match
323      anything at runtime. Arguments with relative path information,
324      which would normally only be used in htaccess context, are implicitly
325      prefixed with '*/' to avoid matching partial directory names.</p>
326    </div>

342    <var>name</var> for <code><a href="#indexoptions.fancyindexing">FancyIndexing</a></code>. <var>Icon</var> is either a (%-escaped)
343    relative URL to the icon, a fully qualified remote URL, or of the format <code>

353    is unnecessary if you're using <code>IndexOptions
354    HTMLTable</code>.</p>

516<div class="directive-section"><h2><a name="IndexIgnore" id="IndexIgnore">IndexIgnore</a> <a name="indexignore" id="indexignore">Directive</a></h2>
517<table class="directive">
518<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Adds to the list of files to hide when listing
519a directory</td></tr>
520<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>IndexIgnore <var>file</var> [<var>file</var>] ...</code></td></tr>
521<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>IndexIgnore "."</code></td></tr>
522<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
523<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>Indexes</td></tr>
524<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
525<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_autoindex</td></tr>
526</table>
527    <p>The <code class="directive">IndexIgnore</code> directive adds to the
528    list of files to hide when listing a directory. <var>File</var> is a
529    filename or pattern which may contain the <code>?</code> and
530    <code>*</code> wildcard characters. <code>?</code> matches any
531    single character, and <code>*</code> matches any sequence of
532    characters, including an empty sequence. Multiple IndexIgnore
533    directives add to the list, rather than replacing the list of
534    ignored files. By default, the list contains <code>.</code> (the
535    current directory).</p>
536
537    <pre class="prettyprint lang-config">IndexIgnore .??* *~ *# HEADER* README* RCS CVS *,v *,t</pre>
538
539
540    <p>If <var>File</var> contains a <code>/</code> character, only
541    the portion after the last <code>/</code> is used for matching
542    against filenames. For example, <code>*/.??*</code> is equivalent
543    to <code>.??*</code>.</p>
544
545    <div class="note"><h3>Differences from shell wildcard patterns</h3>
546      <p>The wildcard matching used here is <em>not</em> the same as
547      shell-style (glob) filename pattern matching. In particular:</p>
548      <ul>
549        <li>Bracket expressions such as <code>[abc]</code> or
550        <code>[!.]</code> are not supported; they are treated as
551        literal characters.</li>
552        <li>A leading <code>.</code> in a filename is not treated
553        specially. For example, <code>*~</code> will match
554        <code>.example~</code>, whereas a shell glob would not.</li>
555      </ul>
556    </div>
557
558    <div class="note"><h3>Regular Expressions</h3>
559      <p>This directive does not currently work in configuration sections
560      that have regular expression arguments, such as  <code class="directive"><a href="../mod/core.html#directorymatch">&lt;DirectoryMatch&gt;</a></code>
561      </p>
562    </div>

597<table class="directive">
598<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Various configuration settings for directory
599indexing</td></tr>
600<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>IndexOptions  [+|-]<var>option</var> [[+|-]<var>option</var>]

602<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>By default, no options are enabled.</code></td></tr>
603<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
604<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>Indexes</td></tr>
605<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
606<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_autoindex</td></tr>
607</table>
608    <p>The <code class="directive">IndexOptions</code> directive specifies the
609    behavior of the directory indexing. <var>Option</var> can be one
610    of</p>
611
612    <dl>
613      <dt><a name="indexoptions.addaltclass" id="indexoptions.addaltclass">AddAltClass</a></dt>
614      <dd>Adds an additional CSS class declaration to each row of the
615      directory listing table when <code>IndexOptions HTMLTable</code>
616      is in effect and an <code>IndexStyleSheet</code> is defined.
617      Rather than the standard <code>even</code> and <code>odd</code>
618      classes that would otherwise be applied to each row of the table,
619      a class of <code>even-<em>ALT</em></code> or
620      <code>odd-<em>ALT</em></code> where <em>ALT</em> is either the
621      standard alt text associated with the file style (eg. <em>snd</em>,
622      <em>txt</em>, <em>img</em>, etc) or the alt text defined by one of
623      the various <code>AddAlt*</code> directives.
624      </dd>
625
626      <dt><a name="indexoptions.charset" id="indexoptions.charset">Charset=<var>character-set</var></a></dt>
627
628      <dd>The <code>Charset</code> keyword allows you to
629      specify the character set of the generated page. The
630      default is <code>UTF-8</code> on Windows and Mac OS X,
631      and <code>ISO-8859-1</code> elsewhere.
632      (It depends on whether the underlying file system
633      uses Unicode filenames or not.)
634
635      <pre class="prettyprint lang-config">IndexOptions Charset=UTF-8</pre>
636
637      </dd>
638
639      <dt><a name="indexoptions.descriptionwidth" id="indexoptions.descriptionwidth">DescriptionWidth=[<var>n</var> | *]</a></dt>
640
641      <dd>The <code>DescriptionWidth</code> keyword allows you to
642      specify the width of the description column in
643      characters.</dd>
644
645      <dt><code>-DescriptionWidth</code> (or unset) allows
646      <code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code> to calculate the best width.</dt>
647
648      <dd><code>DescriptionWidth=<var>n</var></code> fixes the column width to
649      <var>n</var> bytes wide.</dd>
650
651      <dd><code>DescriptionWidth=*</code> grows the column to the
652      width necessary to accommodate the longest description
653      string.
654
655      <strong>See the section on <code class="directive"><a href="#adddescription">AddDescription</a></code> for dangers
656      inherent in truncating descriptions.</strong></dd>
657
658      <dt><a name="indexoptions.fancyindexing" id="indexoptions.fancyindexing">FancyIndexing</a></dt>
659
660      <dd>This turns on fancy indexing of directories.</dd>
661
662      <dt><a name="indexoptions.foldersfirst" id="indexoptions.foldersfirst">FoldersFirst</a></dt>
663
664      <dd>If this option is enabled, subdirectory listings will
665      <em>always</em> appear first, followed by normal files in the
666      directory. The listing is basically broken into two
667      components, the files and the subdirectories, and each is
668      sorted separately and then displayed subdirectories-first.
669      For instance, if the sort order is descending by name, and
670      <code>FoldersFirst</code> is enabled, subdirectory
671      <code>Zed</code> will be listed before subdirectory
672      <code>Beta</code>, which will be listed before normal files
673      <code>Gamma</code> and <code>Alpha</code>.
674      <strong>This option only has an effect if <a href="#indexoptions.fancyindexing"><code>FancyIndexing</code></a>
675      is also enabled.</strong>
676      </dd>
677
678      <dt><a name="indexoptions.htmltable" id="indexoptions.htmltable">HTMLTable</a></dt>
679
680      <dd>This option with <code>FancyIndexing</code> constructs
681      a simple table for the fancy directory listing.
682      It is necessary for utf-8 enabled platforms or if file
683      names or description text will alternate between
684      left-to-right and right-to-left reading order.</dd>
685
686      <dt><a name="indexoptions.iconsarelinks" id="indexoptions.iconsarelinks">IconsAreLinks</a></dt>
687
688      <dd>This makes the icons part of the anchor for the filename, for
689      fancy indexing.</dd>
690
691      <dt><a name="indexoptions.iconheight" id="indexoptions.iconheight">IconHeight[=<var>pixels</var>]</a></dt>
692
693      <dd>Presence of this option, when used with <code>IconWidth</code>,
694      will cause the server to include <code>height</code> and
695      <code>width</code> attributes in the <code>img</code> tag for the file
696      icon. This allows browser to precalculate the page layout without having
697      to wait until all the images have been loaded. If no value is given for
698      the option, it defaults to the standard height of the icons supplied
699      with the Apache httpd software.
700
701      <strong>This option
702      only has an effect if <a href="#indexoptions.fancyindexing"><code>FancyIndexing</code></a> is also enabled.</strong>
703
704      </dd>
705
706      <dt><a name="indexoptions.iconwidth" id="indexoptions.iconwidth">IconWidth[=<var>pixels</var>]</a></dt>
707
708      <dd>Presence of this option, when used with <code>IconHeight</code>,
709      will cause the server to include <code>height</code> and
710      <code>width</code> attributes in the <code>img</code> tag for
711      the file icon. This allows browser to precalculate the page
712      layout without having to wait until all the images have been
713      loaded. If no value is given for the option, it defaults to
714      the standard width of the icons supplied with the Apache httpd
715      software.</dd>
716
717      <dt><a name="indexoptions.ignorecase" id="indexoptions.ignorecase">IgnoreCase</a></dt>
718
719      <dd>If this option is enabled, names are sorted in a case-insensitive
720      manner.  For instance, if the sort order is ascending by name, and
721      <code>IgnoreCase</code> is enabled, file Zeta  will be listed after
722      file alfa (Note: file GAMMA will always be listed before file gamma).
723      </dd>
724
725      <dt><a name="indexoptions.ignoreclient" id="indexoptions.ignoreclient">IgnoreClient</a></dt>
726
727      <dd>This option causes <code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code> to ignore all
728      query variables from the client, including sort order (implies
729      <code><a href="#indexoptions.suppresscolumnsorting">SuppressColumnSorting</a></code>.)</dd>
730
731      <dt><a name="indexoptions.namewidth" id="indexoptions.namewidth">NameWidth=[<var>n</var>
732               | *]</a></dt>
733
734      <dd>The <code>NameWidth</code> keyword allows you to specify the width
735      of the filename column in bytes.</dd>
736
737      <dd><code>-NameWidth</code> (or unset) allows <code class="module"><a href="../mod/mod_autoindex.html">mod_autoindex</a></code> to calculate the best width, but only up
738      to 20 bytes wide.</dd>
739
740      <dd><code>NameWidth=<var>n</var></code> fixes the column width to
741      <var>n</var> bytes wide.</dd>
742
743      <dd><code>NameWidth=*</code> grows the column to the necessary
744      width.</dd>
745
746      <dt><a name="indexoptions.scanhtmltitles" id="indexoptions.scanhtmltitles">ScanHTMLTitles</a></dt>
747
748      <dd>This enables the extraction of the title from HTML documents
749      for fancy indexing. If the file does not have a description
750      given by <code class="directive"><a href="#adddescription">AddDescription</a></code>
751      then httpd will read the document for the value of the
752      <code>title</code> element. This is CPU and disk intensive.</dd>
753
754      <dt><a name="indexoptions.showforbidden" id="indexoptions.showforbidden">ShowForbidden</a></dt>
755
756      <dd>If specified, Apache httpd will show files normally hidden because
757      the subrequest returned <code>HTTP_UNAUTHORIZED</code> or
758      <code>HTTP_FORBIDDEN</code></dd>
759
760      <dt><a name="indexoptions.suppresscolumnsorting" id="indexoptions.suppresscolumnsorting">SuppressColumnSorting</a></dt>
761
762      <dd>If specified, Apache httpd will not make the column headings in a
763      FancyIndexed directory listing into links for sorting. The
764      default behavior is for them to be links; selecting the
765      column heading will sort the directory listing by the values
766      in that column. However, query string arguments which are appended
767      to the URL will still be honored. That behavior is controlled by <a href="#indexoptions.ignoreclient"><code>IndexOptions
768      IgnoreClient</code></a>.</dd>
769
770      <dt><a name="indexoptions.suppressdescription" id="indexoptions.suppressdescription">SuppressDescription</a></dt>
771
772      <dd>This will suppress the file description in fancy indexing
773      listings. By default, no file descriptions are defined, and
774      so the use of this option will regain 23 characters of screen

784
785      <dd>If the directory actually contains a file specified by the
786      <code class="directive"><a href="#headername">HeaderName</a></code>
787      directive, the module usually includes the contents of the file
788      after a standard HTML preamble (<code>&lt;html&gt;</code>,
789      <code>&lt;head&gt;</code>, <em>et cetera</em>). The

796      omitted from the output, under the assumption that you'll likely
797      put those closing tags in that file.</dd>

841
842      <dd>This returns the <code>Last-Modified</code> and <code>ETag</code>
843      values for the listed directory in the HTTP header. It is only valid
844      if the operating system and file system return appropriate stat()
845      results. Some Unix systems do so, as do OS2's JFS and Win32's
846      NTFS volumes. OS2 and Win32 FAT volumes, for example, do not.
847      Once this feature is enabled, the client or proxy can track
848      changes to the list of files when they perform a <code>HEAD</code>
849      request. Note some operating systems correctly track new and
850      removed files, but do not track changes for sizes or dates of
851      the files within the directory. <strong>Changes to the size
852      or date stamp of an existing file will not update the

875
876      <dd>The <code>VersionSort</code> keyword causes files containing
877      version numbers to sort in a natural way. Strings are sorted as
878      usual, except that substrings of digits in the name and
879      description are compared according to their numeric value.
880
881      <div class="example"><h3>Example:</h3><p><code>
882        foo-1.7<br />
883        foo-1.7.2<br />
884        foo-1.7.12<br />
885        foo-1.8.2<br />
886        foo-1.8.2a<br />
887        foo-1.12
888      </code></p></div>
889
890      <p>If the number starts with a zero, then it is considered to
891      be a fraction:</p>
892
893      <div class="example"><p><code>
894        foo-1.001<br />
895        foo-1.002<br />
896        foo-1.030<br />
897        foo-1.04
898      </code></p></div>
899      </dd>

911      <dd>The <code>None</code> keyword disables all indexing options
912      for the directory. No fancy indexing features will be enabled.

917      </dd>

923   <dl><dt>Incremental IndexOptions</dt>
924   <dd>
925     <p>Be aware of how multiple <code class="directive">IndexOptions</code> are
926     handled.</p>
927
928     <ul>
929     <li>Multiple <code class="directive">IndexOptions</code> directives for a
930     single directory are now merged together. The result of:
931
932     <pre class="prettyprint lang-config">&lt;Directory "/foo"&gt;
933    IndexOptions HTMLTable
934    IndexOptions SuppressColumnsorting
935&lt;/Directory&gt;</pre>
936
937
938     <p>will be the equivalent of</p>
939
940     <pre class="prettyprint lang-config">IndexOptions HTMLTable SuppressColumnsorting</pre>
941
942     </li>
943
944     <li>The addition of the incremental syntax (<em>i.e.</em>, prefixing
945     keywords with <code>+</code> or <code>-</code>).</li>
946     </ul>
947
948     <p>Whenever a '+' or '-' prefixed keyword is encountered, it
949     is applied to the current <code class="directive">IndexOptions</code>
950     settings (which may have been inherited from an upper-level
951     directory). However, whenever an unprefixed keyword is processed, it
952     clears all inherited options and any incremental settings encountered
953     so far. Consider the following example:</p>
954
955     <pre class="prettyprint lang-config">IndexOptions +ScanHTMLTitles -IconsAreLinks FancyIndexing
956IndexOptions +SuppressSize</pre>
957
958
959     <p>The net effect is equivalent to <code>IndexOptions FancyIndexing
960     +SuppressSize</code>, because the unprefixed <code>FancyIndexing</code>
961     discarded the incremental keywords before it, but allowed them to
962     start accumulating again afterward.</p>
963
964     <p>To unconditionally set the <code class="directive">IndexOptions</code> for
965     a particular directory, clearing the inherited settings, specify
966     keywords without any <code>+</code> or <code>-</code> prefixes.</p>
967   </dd>
968   </dl>

972<div class="directive-section"><h2><a name="IndexOrderDefault" id="IndexOrderDefault">IndexOrderDefault</a> <a name="indexorderdefault" id="indexorderdefault">Directive</a></h2>
973<table class="directive">
974<tr><th><a href="directive-dict.html#Description">Description:</a></th><td>Sets the default ordering of the directory index</td></tr>
975<tr><th><a href="directive-dict.html#Syntax">Syntax:</a></th><td><code>IndexOrderDefault Ascending|Descending
976Name|Date|Size|Description</code></td></tr>
977<tr><th><a href="directive-dict.html#Default">Default:</a></th><td><code>IndexOrderDefault Ascending Name</code></td></tr>
978<tr><th><a href="directive-dict.html#Context">Context:</a></th><td>server config, virtual host, directory, .htaccess</td></tr>
979<tr><th><a href="directive-dict.html#Override">Override:</a></th><td>Indexes</td></tr>
980<tr><th><a href="directive-dict.html#Status">Status:</a></th><td>Base</td></tr>
981<tr><th><a href="directive-dict.html#Module">Module:</a></th><td>mod_autoindex</td></tr>
982</table>
983    <p>The <code class="directive">IndexOrderDefault</code> directive is used
984    in combination with the <code><a href="#indexoptions.fancyindexing">FancyIndexing</a></code> index option. By default, fancyindexed
985    directory listings are displayed in ascending order by filename; the
986    <code class="directive">IndexOrderDefault</code> allows you to change this
987    initial display order.</p>
988
989    <p><code class="directive">IndexOrderDefault</code> takes two
990    arguments. The first must be either <code>Ascending</code> or
991    <code>Descending</code>, indicating the direction of the sort.
992    The second argument must be one of the keywords <code>Name</code>,
993    <code>Date</code>, <code>Size</code>, or <code>Description</code>,
994    and identifies the primary key. The secondary key is
995    <em>always</em> the ascending filename.</p>
996
997    <p>You can, if desired, prevent the client from reordering the list
998    by also adding the <code><a href="#indexoptions.suppresscolumnsorting">SuppressColumnSorting</a></code>
999    index option to remove the sort link from the top of the column,
1000    along with the <code><a href="#indexoptions.ignoreclient">IgnoreClient</a></code> index
1001    option to prevent them from manually adding sort options to the
1002    query string in order to override your ordering preferences.</p>
