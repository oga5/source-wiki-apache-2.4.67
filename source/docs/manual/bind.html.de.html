49<h2><a name="overview" id="overview">&#220;berblick</a></h2>
50    
51
52    <table class="related"><tr><th>Referenzierte Module</th><th>Referenzierte Direktiven</th></tr><tr><td><ul><li><code class="module"><a href="./mod/core.html">core</a></code></li><li><code class="module"><a href="./mod/mpm_common.html">mpm_common</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code></li><li><code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code></li></ul></td></tr></table>
53
54    <p>Beim Start bindet sich der httpd an bestimmte Adressen und Ports
55    der lokalen Maschine und wartet auf eingehende Anfragen.
56    Standardm&#228;&#223;ig lauscht er an allen Adressen des Systems.
57    Es kann jeodch notwendig sein, ihm mit zuteilen, nur an bestimmten
58    Ports zu lauschen oder nur an ausgew&#228;hlten Adressen, bzw. einer
59    Kombination aus beidem. Dies wird oft mit der Funktionalit&#228;t <a href="vhosts/">virtueller Hosts</a> kombiniert, die bestimmt, wie
60    der <code>httpd</code> auf verschiedene IP-Adressen, Hostnamen und
61    Ports reagiert.</p>
62
63    <p>Die Direktive <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
64    weist den Server an, eingehende Anfragen nur an bestimmten Port(s)
65    oder Adress/Port-Kombinationen zu akzeptieren. Wenn bei der
66    <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>-Direktive nur eine
67    Portnummer angegeben wird, dann lauscht der Server auf allen
68    Netzwerkinterfaces an dem angegebenen Port. Ist auch eine IP-Adresse
69    angegeben, dann lauscht der Server an der angegebenen Schnittstelle
70    auf dem angegebenen Port.  Es k&#246;nnen mehrere <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>-Anweisungen verwendet werden,
71    um eine Reihe von Adressen und Ports anzugeben, an denen gelauscht
72    werden soll. Der Server wird dann auf Anfragen an jeder der
73    abgeh&#246;rten Adressen und Ports antworten.</p>
74
75    <p>Um beispielsweise den Server zu veranlassen, auf allen
76    Netzwerkinterfaces sowohl an Port 80, als auch an Port 8000
77    Verbindungen zu akzeptieren, geben Sie an:</p>
78
79    <div class="example"><pre class="prettyprint lang-config">Listen 80
80Listen 8000</pre>
81</div>
82
83    <p>Um den Server Verbindungen an Port 80 auf einem Netzwerkinterface
84    akzeptieren zu lassen und an Port 8080 auf einem anderen Interface, geben
85    Sie an:</p>
86      
87    <div class="example"><pre class="prettyprint lang-config">Listen 192.0.2.1:80
88Listen 192.0.2.5:8000</pre>
89</div>
90
91    <p>IPv6-Adressen m&#252;ssen wie im folgenden Beispiel in eckigen
92      Klammern angegeben werden:</p>
93
94    <div class="example"><pre class="prettyprint lang-config">Listen [2001:db8::a00:20ff:fea7:ccea]:80</pre>
95</div>
96
97    <div class="warning"><p>Sich &#252;berlappende <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>-Direktiven generieren einen
98    fatalen Fehler, der verhindert, dass der Server hochf&#228;hrt.</p>
99
100    <div class="example"><p><code>
101      (48)Address already in use: make_sock: could not bind to address [::]:80
102    </code></p></div>
103
104    <p><a href="https://cwiki.apache.org/confluence/display/httpd/CouldNotBindToAddress">Diese
105    Diskussion im Wiki</a> gibt weitere Tipps zur Fehlerbehebung.</p>
106    </div>

110<h2><a name="reload" id="reload">&#196;nderung der Listen-Konfiguration beim Neustart</a></h2>
111    
112
113    <p>Wenn der httpd neu gestartet wird, m&#252;ssen &#196;nderungen an
114    <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>-Direktiven besonders
115    beachtet werden. W&#228;hrend eines Neustarts bleiben die Ports an den httpd
116    gebunden (wie in der urspr&#252;nglichen Konfiguration), um
117    "Connection refused"-Fehler f&#252;r neue Verbindungsversuche zum
118    Server zu vermeiden. Wenn &#196;nderungen an den verwendeten
119    <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>-Direktiven vorgenommen
120    werden, die mit der alten Konfiguration in Konflikt stehen, schl&#228;gt
121    die Konfiguration fehl und der Server wird beendet.</p>
122
123    <p>Zum Beispiel kann die &#196;nderung der Konfiguration von:</p>
124    
125    <div class="example"><pre class="prettyprint lang-config">Listen 127.0.0.1:80</pre>
126</div>
127
128    <p>zur folgenden Konfiguration fehlschlagen, da das Binden an
129    Port 80 auf allen Adressen mit dem Binden an Port 80 auf nur
130    127.0.0.1 in Konflikt steht.</p>
131    
132    <div class="example"><pre class="prettyprint lang-config">Listen 80</pre>
133</div>
134
135    <p>Damit solche Konfigurations&#228;nderungen wirksam werden, muss
136    der Server gestoppt und dann neu gestartet werden.</p>

140<h2><a name="ipv6" id="ipv6">Betrachtung von IPv6-Besonderheiten</a></h2>
141    
142
143    <p>Eine wachsende Anzahl von Plattformen implementiert IPv6. Die
144    <a class="glossarylink" href="./glossary.html#apr" title="siehe Glossar">APR</a> unterst&#252;tzt IPv6 auf den meisten
145    dieser Plattformen und erm&#246;glicht dem httpd, IPv6-Sockets zu
146    verwenden und &#252;ber IPv6 gesendete Anfragen zu behandeln.</p>
147
148    <p>F&#252;r httpd-Administratoren kommt erschwerend die Frage hinzu,
149    ob IPv6-Sockets sowohl IPv4- als auch IPv6-Verbindungen handhaben
150    k&#246;nnen. Zum Betrieb von IPv4-Verbindungen an IPv6-Sockets
151    werden auf IPv6 abgebildete IPv4-Adressen <span class="transnote">(<em>Anm.d.&#220;.:</em> so genannete
152    IPv4-gemappte IPv6-Adressen)</span> verwendet, welche
153    standardm&#228;&#223;ig auf den meisten Plattformen erlaubt sind.
154    Unter FreeBSD, NetBSD und OpenBSD jedoch sind sie
155    standardm&#228;&#223;ig deaktiviert, um den Systemgrunds&#228;tzen
156    dieser Plattformen zu entsprechen. Auf Systemen, wo dies
157    standardm&#228;&#223;ig dekativiert ist, kann dieses Verhalten mit
158    einem speziellen <code class="program"><a href="./programs/configure.html">configure</a></code>-Parameter f&#252;r den
159    httpd ge&#228;ndert werden.</p>
160
161    <p>Auf der anderen Seite ist die Verwendung von gemappten Adressen
162    bei einigen Plattformen wie Linux und True64 der
163    <strong>einzige</strong> Weg, sowohl IPv4 wie auch IPv6 zu
164    verwenden. Wenn Sie m&#246;chten, dass der <code>httpd</code> IPv4-
165    und IPv6-Verbindungen mit einem Minimum an Sockets behandelt, was
166    die Verwendung von IPv4-gemappten IPv6-Adressen erfordert, dann
167    m&#252;ssen Sie die <code class="program"><a href="./programs/configure.html">configure</a></code>-Option
168    <code>--enable-v4-mapped</code> angeben.</p>
169
170    <p><code>--enable-v4-mapped</code> ist die Voreinstellung auf allen
171    Plattformen au&#223;er FreeBSD, NetBSD und OpenBSD, so dass Ihr
172    httpd wahrscheinlich so &#252;bersetzt wurde.</p>
173
174    <p>Geben Sie wie in dem folgenden Beispiel bei allen <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>-Anweisungen eine IPv4-Adresse
175    an, wenn Sie m&#246;chten, dass Ihr httpd lediglich IPv4-Adressen
176    behandelt, unabh&#228;ngig davon, was Ihre Plattform und die APR
177    unterst&#252;tzen:</p>
178
179    <div class="example"><pre class="prettyprint lang-config">Listen 0.0.0.0:80
180Listen 192.0.2.1:80</pre>
181</div>
182
183    <p>Wenn Sie m&#246;chten, dass der httpd IPv4- und IPv6-Verbindungen
184    an separaten Sockets behandelt (d.h. IPv4-gemappte Adressen
185    deaktiviert werden sollen) und Ihre Plattform es unterst&#252;tzt,
186    dann m&#252;ssen Sie die <code class="program"><a href="./programs/configure.html">configure</a></code>-Option
187    <code>--disable-v4-mapped</code> angeben.  Unter FreeBSD, NetBSD und
188    OpenBSD ist <code>--disable-v4-mapped</code> voreingestellt.</p>

191<h2><a name="protocol" id="protocol">Angabe des Protokolls bei Listen</a></h2>
192    
193    <p>Das optionale zweite <var>Protokoll</var>-Argument von <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code> ist f&#252;r die meisten
194    Konfigurationen gar nicht erforderlich. Wenn nicht angegeben, sind
195    <code>https</code> f&#252;r Port 443 und <code>http</code> f&#252;r
196    alle anderen Ports die Voreinstellungen. Die Protokollangabe wird
197    sowohl dazu verwendet, herauszufinden, welches Modul Anfragen
198    verarbeiten soll, als auch, um protokollspezifische Optimierungen
199    bei der <code class="directive"><a href="./mod/core.html#acceptfilter">AcceptFilter</a></code>-Direktive
200    zu aktivieren.</p>
201
202    <p>Sie m&#252;ssen das Protokoll nur angeben, wenn Sie
203    ungew&#246;hnliche Ports benutzen, beispielsweise <code>https</code>
204    auf Port 8443:</p>
205
206    <div class="example"><pre class="prettyprint lang-config">Listen 192.170.2.1:8443 https</pre>
207</div>

210<h2><a name="virtualhost" id="virtualhost">Das Zusammenspiel mit virtuellen Hosts</a></h2>
211    
212
213    <p>Die Direktive <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
214    implementiert keine virtuellen Hosts - sie teilt dem Hauptserver
215    lediglich mit, an welchen Adressen und Ports er zu lauschen hat.
216    Werden keine <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>-Container verwendet, dann
217    verh&#228;lt sich der Server bei allen angenommenen Anfragen gleich.
218    <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>-Abschnitte k&#246;nnen jedoch
219    dazu verwendet werden, ein unterschiedliches Verhalten f&#252;r eine
220    oder mehrere Adressen und Ports festzulegen. Um einen virtuellen
221    Host einzurichten, muss dem Server zun&#228;chst mitgeteilt werden,
222    an den betreffenden Adressen oder Ports zu lauschen. Dann sollte ein
223    <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>-Abschnitt f&#252;r die
224    angebene Adresse und den angegebenen Port erstellt werden, um das
225    Verhalten dieses virtuellen Hosts festzulegen. Beachten Sie bitte,
226    dass auf einen <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code> nicht zugegriffen werden
227    kann, wenn er f&#252;r eine Adresse und einen Port eingerichtet
228    wurde, an dem der Server nicht lauscht.</p>
