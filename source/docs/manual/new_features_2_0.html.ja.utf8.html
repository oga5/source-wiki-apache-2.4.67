47<h2><a name="core" id="core">コア機能の拡張</a></h2>
48    
49
50    <dl>
51      <dt>Unix のスレッド</dt>
52
53      <dd>POSIX スレッドをサポートしている Unix システム上では、
54      Apache はマルチプロセス、マルチスレッドのハイブリッドモードで
55      実行できるようになりました。これにより
56      多くの設定においてスケーラビリティが向上します。</dd>
57
58      <dt>新しいビルドシステム</dt>
59
60      <dd>ビルドシステムは <code>autoconf</code> と <code>libtool</code>
61      に基づいたものになるように、
62      新しく書き直されました。これにより、Apache の configure のシステムは
63      他のパッケージと似たものになりました。</dd>
64
65      <dt>マルチプロトコルサポート</dt>
66
67      <dd>Apache に複数のプロトコルを扱うための機構が備わりました。
68      例として <code class="module"><a href="./mod/mod_echo.html">mod_echo</a></code> が書かれています。</dd>
69
70      <dt>Unix 以外のプラットフォームのサポートの改善</dt>
71
72      <dd>Apache 2.0 は BeOS、OS/2、Windows などの Unix 以外の
73      プラットフォームで、より速く、より安定して動作するようになりました。
74      プラットフォーム特有の <a href="mpm.html">マルチプロセッシングモジュール</a> (MPM) と
75      Apache Portable Runtime (APR) の導入により、
76      ネイティヴの API で実装されるようになり、
77      バグが多く、性能の悪いことが多い POSIX エミュレーションレイヤの使用を
78      回避することができました。</dd>
79
80      <dt>新しい Apache API</dt>
81
82      <dd>2.0 ではモジュールの API が大きく変わりました。
83      1.3 にあったモジュールの順番/優先度の問題の多くは
84      なくなっているはずです。2.0 は優先度の選択をほとんどを自動的に行ない、
85      モジュールの順番はより柔軟性を高めるためにフック毎に行なわれるように
86      なりました。また、コア Apache サーバにパッチをあてることなく
87      追加のモジュール機能を提供することができるように新しい関数が
88      追加されました。</dd>
89
90      <dt>IPv6 サポート</dt>
91
92      <dd>Apache が使用している Apache Portable Runtime library が
93      IPv6 をサポートしているシステムでは Apache は デフォルトで
94      IPv6 のソケットを listen します。さらに、
95      <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>,
96      <code class="directive"><a href="./mod/core.html#namevirtualhost">NameVirtualHost</a></code>,
97      <code class="directive"><a href="./mod/core.html#virtualhost">VirtualHost</a></code>
98      の各ディレクティブが IPv6 のアドレスを
99      サポートするようになりました (例えば、
100      "<code>Listen [2001:db8::1]:8080</code>")。</dd>
101
102      <dt>フィルタ</dt>
103
104      <dd>Apache のモジュールはサーバから送られてきたり、サーバへ
105      送るストリームに対して動作するフィルタとして書くことができるように
106      なりました。これにより、例えば CGI スクリプトの出力を
107      <code class="module"><a href="./mod/mod_include.html">mod_include</a></code> の <code>INCLUDES</code> フィルタを使って
108      Server Side Include のディレクティブを解析する、
109      というようなことが可能になりました。<code class="module"><a href="./mod/mod_ext_filter.html">mod_ext_filter</a></code>
110      で外部プログラムをフィルタとして動作させることができます。
111      これは CGI プログラムをハンドラとして動作させるのと
112      よく似た方法でできます。</dd>
113
114      <dt>多言語エラー応答</dt>
115
116      <dd>ブラウザへのエラー応答のメッセージが、SSI の文書を使って
117      複数の言語で提供されるようになりました。見ための一貫性を保つために
118      管理者がカスタマイズすることもできます。</dd>
119
120      <dt>設定の簡素化</dt>
121
122      <dd>多くの混乱を招きがちなディレクティブが簡素化されました。
123      よく混乱を引き起こしていた <code>Port</code> ディレクティブと 
124      <code>Bind</code> ディレクティブは
125      なくなりました。<code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
126      ディレクティブのみが IP アドレスのバインドに使われます。
127      <code class="directive"><a href="./mod/core.html#servername">ServerName</a></code> ディレクティブでは
128      リダイレクトと vhost の認識のためだけにサーバの名前とポート番号を
129      指定します。</dd>
130
131      <dt>Windows NT のネイティヴ Unicode サポート</dt>
132
133      <dd>Windows NT 上の Apache 2.0 はファイル名の文字エンコード全てに
134      utf-8 を使うようになりました。これらは Unicode ファイルシステムに
135      直接変換されるので、Windows 2000 と Windows XP を含む、全ての
136      Windows NT 系で多言語サポートが提供されます。
137      <em>このサポートは、ファイルシステムのアクセス時にローカルの
138      コードページを使う Windows 95, 98, ME には適用されません。</em></dd>
139
140      <dt>正規表現ライブラリのアップデート</dt>
141
142      <dd>Apache 2.0 は <a href="http://www.pcre.org/">Perl
143      互換正規表現ライブラリ (PCRE) </a>を含んでいます。
144      正規表現の評価には、より強力になった Perl 5
145      構文を使用します。</dd>
146
147    </dl>

150<h2><a name="module" id="module">モジュールの拡張</a></h2>
151    
152
153    <dl>
154      <dt><code class="module"><a href="./mod/mod_ssl.html">mod_ssl</a></code></dt>
155
156      <dd>Apache 2.0 の新モジュール。このモジュールは OpenSSL が
157      提供する SSL/TLS 暗号プロトコルへのインタフェースです。</dd>
158
159      <dt><code class="module"><a href="./mod/mod_dav.html">mod_dav</a></code></dt>
160
161      <dd>Apache 2.0 の新モジュール。このモジュールはウェブコンテンツを
162      送り、維持するための規格
163      HTTP Distributed Authoring and Versioning (DAV) を実装しています。</dd>
164
165      <dt><code class="module"><a href="./mod/mod_deflate.html">mod_deflate</a></code></dt>
166
167      <dd>Apache 2.0 の新モジュール。送信前に送信内容を圧縮して
168      ネットワーク帯域を節約する、というリクエストをブラウザが
169      要求できるようにします。</dd>
170
171      <dt><code class="module"><a href="./mod/mod_auth_ldap.html">mod_auth_ldap</a></code></dt>
172
173      <dd>Apache 2.0.41 の新モジュール。HTTP 基本認証の証明書を保存するのに、
174      LDAP データベースを使用できるようになります。
175      関連モジュールの <code class="module"><a href="./mod/mod_ldap.html">mod_ldap</a></code> で、
176      コネクションのプール機能と結果のキャッシュ機能が提供されます。</dd>
177
178      <dt><code class="module"><a href="./mod/mod_auth_digest.html">mod_auth_digest</a></code></dt>
179
180      <dd>このモジュールは共有メモリを使うことにより、プロセスをまたいだ
181      セッションのキャッシュをサポートするようになりました。</dd>
182
183      <dt><code class="module"><a href="./mod/mod_charset_lite.html">mod_charset_lite</a></code></dt>
184
185      <dd>Apache 2.0 の新モジュール。この実験的なモジュールは
186      キャラクタセットの変換や再符号化を可能にします。</dd>
187
188      <dt><code class="module"><a href="./mod/mod_file_cache.html">mod_file_cache</a></code></dt>
189
190      <dd>Apache 2.0 の新モジュール。このモジュールには、
191      Apache 1.3 における <code>mod_mmap_static</code> 機能が含まれ、
192      また、追加のキャッシュ機能が加わっています。</dd>
193
194      <dt><code class="module"><a href="./mod/mod_headers.html">mod_headers</a></code></dt>
195
196      <dd>このモジュールは Apache 2.0 で非常に柔軟性が
197      高くなりました。<code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code>
198      で使われるリクエストのヘッダを変更できるようになりましたし、
199      応答ヘッダを条件に応じて設定できるようになりました。</dd>
200
201      <dt><code class="module"><a href="./mod/mod_proxy.html">mod_proxy</a></code></dt>
202
203      <dd>proxy モジュールは新しいフィルタの機構を利用するためと、
204      より信頼できる、HTTP/1.1 に準拠した proxy を実装するために
205      完全に書き直されました。さらに、新しい 
206      <code class="directive"><a href="./mod/mod_proxy.html#proxy">&lt;Proxy&gt;</a></code>
207      設定セクションがproxy されるサイトのより読みやすく (内部的にもより速い)
208      設定を提供します。オーバーロードされた
209      <code>&lt;Directory "proxy:... &gt;</code>
210      設定はサポートされていません。このモジュールは <code>proxy_connect</code>,
211      <code>proxy_ftp</code>, <code>proxy_http</code> 
212      といった、特定のプロトコルをサポートする
213      モジュールに分割されるようになりました。</dd>
214
215      <dt><code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code></dt>
216
217      <dd>クライアントが NOT ACCEPTABLE や MULTIPLE CHOICES 応答の
218      代わりに常に単独の文書を受けとるようにするために、新しいディレクティブ
219      <code class="directive"><a href="./mod/mod_negotiation.html#forcelanguagepriority">ForceLanguagePriority</a></code>
220      を使うことができるようになりました。
221      さらに、より一貫性のある結果を提供するために
222      ネゴシエーションと MultiViews のアルゴリズムが改善され、
223      文書の内容を含めることのできる、新しい形式のタイプマップが
224      提供されるようになりました。</dd>
225
226      <dt><code class="module"><a href="./mod/mod_autoindex.html">mod_autoindex</a></code></dt>
227
228      <dd>Autoindex されるディレクトリの内容一覧が、
229      きれいに表示されるために HTML のテーブルを使うように
230      設定できるようになりました。また、バージョンによるソーティングなど、
231      より細かいソーティングの制御ができるようになり、ディレクトリ
232      の内容一覧をワイルドカードにより選別することができるようにもなりました。</dd>
233
234      <dt><code class="module"><a href="./mod/mod_include.html">mod_include</a></code></dt>
235
236      <dd>新しいディレクティブにより、SSI のデフォルトの開始タグと終了タグを
237      変更できるようになりました。また、エラーと時刻の形式の設定が SSI の
238      文書中ではなく、主設定ファイル中で行なえるようになりました。
239      正規表現の解析とグループ化の結果 (Perl の正規表現の構文に
240      基づいたものになりました) を <code class="module"><a href="./mod/mod_include.html">mod_include</a></code>
241      の変数 $0 .. $9 により取得できるようになりました。</dd>
242
243      <dt><code class="module"><a href="./mod/mod_auth_dbm.html">mod_auth_dbm</a></code></dt>
244
245      <dd><code class="directive"><a href="./mod/mod_auth_dbm.html#authdbmtype">AuthDBMType</a></code>
246      ディレクティブにより、複数の DBM 型のデータベースをサポートする
247      ようになりました。</dd>
248
249    </dl>
