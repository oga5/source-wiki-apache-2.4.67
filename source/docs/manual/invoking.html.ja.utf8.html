55<h2><a name="startup" id="startup">Apache の起動方法</a></h2>
56
57    <p>もし、設定ファイル中で指定されている
58    <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
59    がデフォルトの 80 (もしくは 1024 以下の他のポート)
60    である場合は、Apache を起動するためには root
61    権限が必要になりますが、
62    これはこの特権ポートにバインドするためです。
63    起動して、一度ログファイルを開くといった準備のための
64    動作を幾つか実行した後は、クライアントからのリクエストに対する
65    listen と応答を実際に行う<em>子</em>プロセスを起動します。
66    メインの <code class="program"><a href="./programs/httpd.html">httpd</a></code> プロセスは root 権限で走り続けますが、
67    子プロセスはもっと低い権限で走ります。
68    これは選択した<a href="mpm.html">マルチプロセッシングモジュール</a>で制御されます。</p>
69
70    <p>推奨の <code class="program"><a href="./programs/httpd.html">httpd</a></code> 実行プログラムの起動方法は、
71    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code>
72    制御スクリプトを使用する方法です。このスクリプトは、<code class="program"><a href="./programs/httpd.html">httpd</a></code> 
73    がオペレーティングシステム上で正常に動作するように必要な環境変数を
74    適切に設定して、<code class="program"><a href="./programs/httpd.html">httpd</a></code> バイナリを起動します。
75    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> はどんなコマンドライン引数も通過させますので、
76    <code class="program"><a href="./programs/httpd.html">httpd</a></code> のどのコマンドラインオプションも
77    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> のオプションとして使用できます。
78    また、<code class="program"><a href="./programs/apachectl.html">apachectl</a></code> スクリプトを直接編集し、
79    スクリプト先頭付近の <code>HTTPD</code> 変数を変更することで、
80    <code class="program"><a href="./programs/httpd.html">httpd</a></code> バイナリの正しい位置を指定したり、<em>常に</em>
81    付加させるコマンドライン引数を指定したりすることができます。</p>
82
83    <p><code class="program"><a href="./programs/httpd.html">httpd</a></code> が起動されてまず最初にすることは、
84    <a href="configuring.html">設定ファイル</a>
85    <code>httpd.conf</code> の位置を特定して読み込むことです。
86    このファイルの位置はコンパイル時に設定されますが、実行時に
87    <code>-f</code> コマンドラインオプションを使って
88    位置を指定することもできます。例えば次のようにです。</p>
89
90<div class="example"><p><code>/usr/local/apache2/bin/apachectl -f
91      /usr/local/apache2/conf/httpd.conf</code></p></div>
92
93    <p>スタートアップが万事上手くいったら、サーバはターミナルから
94    切り離されて、コマンドプロンプトが即座に戻ってくるでしょう。
95    これはサーバが起動している状態を示しています。
96    その後はブラウザでサーバに接続して、
97    <code class="directive"><a href="./mod/core.html#documentroot">DocumentRoot</a></code>
98    ディレクトリのテストページを見ることができるでしょう。</p>

101<h2><a name="errors" id="errors">起動時のエラー</a></h2>
102
103    <p>Apache は、起動時に致命的な問題に遭遇すると、
104    終了する前に、コンソールか
105    <code class="directive"><a href="./mod/core.html#errorlog">ErrorLog</a></code>
106    のどちらかに問題を記述したメッセージを出力します。
107    最もよくあるエラーメッセージは
108    「<code>Unable to bind to Port ...</code>」
109    です。このメッセージは普通は次のどちらかが原因です。</p>
110
111    <ul>
112      <li>root でログインしていない時に、
113      特権ポートでサーバを起動しようとした。</li>
114
115      <li>同じポートに既にバインドされている Apache
116      がもう一つあるときや他のウェブサーバが存在している時に、
117      サーバを開始しようとした。</li>
118    </ul>
119
120    <p>より多くの問題解決の方策の説明は、
121	Apache <a href="https://cwiki.apache.org/confluence/display/httpd/FAQ">FAQ</a> をご覧下さい。</p>

124<h2><a name="boot" id="boot">ブート時の起動</a></h2>
125
126    <p>システムがリブートした後でも
127    サーバが実行され続けるようにしたい場合は、
128    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code>
129    を呼び出すものをシステムスタートアップファイル
130    (通常 <code>rc.local</code> や <code>rc.N</code>
131    内のファイル) に追加しなければなりません。
132    この方法では Apache を root 権限で起動します。
133    これをする前に、セキュリティやアクセス制限が
134    適切に設定されていていることを確認してください。</p>
135
136    <p><code class="program"><a href="./programs/apachectl.html">apachectl</a></code> スクリプトは通常は、標準的な SysV init 
137    スクリプトとして動作するように設計されています。
138    <code>start</code>, <code>restart</code>, <code>stop</code>
139    といった引数をとって、<code class="program"><a href="./programs/httpd.html">httpd</a></code> 
140    への適切なシグナルに変換します。
141    ですから、通常は単に適切な init ディレクトリ内から
142    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> へリンクすることができます。しかし、
143    念のためシステムの要求に合致していることを確認してください。</p>
