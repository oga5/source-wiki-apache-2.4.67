30    <p>Le cache des objets partagés est un concept de partage de données
31    de base entre tous les processus d'un serveur, sans se préoccuper du
32    <a href="mpm.html">modèle de threads et de processus</a>. On
33    l'utilise lorsque les avantages apportés par le partage de données
34    entre processus contrebalance la perte de performances consécutive à
35    la communication interprocessus.</p>

39<h2><a name="providers" id="providers">Fournisseurs du cache d'objets partagés</a></h2>
40    
41    <p>Le cache d'objets partagés en tant que tel est une abstraction.
42    Il est implémenté par cinq modules différents. Pour pouvoir
43    utiliser le cache, un ou plusieurs de ces modules doivent être
44    présents et configurés.</p>
45    <p>Le seul élément de configuration consiste à définir le
46    fournisseur de cache à utiliser. Ceci est de la responsabilité des
47    modules qui utilisent le cache, et pour cela, ils activent la
48    sélection via des directives telles que <code class="directive"><a href="./mod/mod_cache_socache.html#cachesocache">CacheSocache</a></code>, <code class="directive"><a href="./mod/mod_authn_socache.html#authncachesocache">AuthnCacheSOCache</a></code>, <code class="directive"><a href="./mod/mod_ssl.html#sslsessioncache">SSLSessionCache</a></code>, et <code class="directive"><a href="./mod/mod_ssl.html#sslstaplingcache">SSLStaplingCache</a></code>.</p>
49    <p>Les fournisseurs actuellement disponibles sont :</p>
50    <dl>
51    <dt>"dbm" (<code class="module"><a href="./mod/mod_socache_dbm.html">mod_socache_dbm</a></code>)</dt>
52    <dd>Celui-ci utilise un fichier de hashage DBM. Le choix de la
53    DBM sous-jacente peut être configurable si la version
54    d'APR installée supporte de multiples implémentations de DBM.</dd>
55    <dt>"dc" (<code class="module"><a href="./mod/mod_socache_dc.html">mod_socache_dc</a></code>)</dt>
56    <dd>Celui-ci utilise les bibliothèques de mise en cache de sessions
57    distribuées <a href="http://distcache.sourceforge.net/">distcache</a>.</dd>
58    <dt>"memcache" (<code class="module"><a href="./mod/mod_socache_memcache.html">mod_socache_memcache</a></code>)</dt>
59    <dd>Celui-ci utilise le système à hautes performances de mise en
60    cache d'objets de mémoire distribuée <a href="http://memcached.org/">memcached</a>.</dd>
61    <dt>"redis" (<code class="module"><a href="./mod/mod_socache_redis.html">mod_socache_redis</a></code>)</dt>
62    <dd>Celui-ci utilise le système de mise en cache d'objets de mémoire
63    distribuée à hautes performances <a href="http://redis.io/">Redis</a>.</dd>
64    <dt>"shmcb" (<code class="module"><a href="./mod/mod_socache_shmcb.html">mod_socache_shmcb</a></code>)</dt>
65    <dd>Celui-ci utilise un tampon cyclique à hautes performances au
66    sein d'un segment de mémoire partagée.</dd>
67    </dl>
68
69    <p>L'API fournit les fonctions suivantes :</p>
70
71    <dl>
72      <dt>const char *create(ap_socache_instance_t **instance, const char *arg,
73                          apr_pool_t *tmp, apr_pool_t *p);</dt>
74      <dd>Cette fonction permet de créer un cache de session basé sur
75      la chaîne de configuration spécifiée. Le pointeur d'instance
76      renvoyé dans le paramètre instance sera passé comme premier
77      argument des invocations subséquentes.</dd>
78
79      <dt>apr_status_t init(ap_socache_instance_t *instance, const char *cname,
80                         const struct ap_socache_hints *hints,
81                         server_rec *s, apr_pool_t *pool)</dt>
82      <dd>Cette fonction permet d'initialiser le cache. L'argument cname
83      doit avoir une longueur maximale de 16 caractères et permet
84      d'identifier de manière unique l'utilisateur du cache au sein du
85      serveur ; il est recommandé d'utiliser le nom du module, par
86      exemple "mod_ssl-sess". Comme cette chaîne peut être utilisée au
87      sein d'un système de fichiers, il est conseillé de n'utiliser que
88      des caractères alphanumériques [a-z0-9_-]. Si l'argument hints
89      n'est pas égal à NULL, il fournit un ensemble d'indications au
90      fournisseur. La valeur retournée est le code d'erreur APR.</dd>
91
92      <dt>void destroy(ap_socache_instance_t *instance, server_rec *s)</dt>
93      <dd>Cette fonction permet de détruire l'instance de cache
94      spécifiée.</dd>
95
96      <dt>apr_status_t store(ap_socache_instance_t *instance, server_rec *s,
97                          const unsigned char *id, unsigned int idlen,
98                          apr_time_t expiry,
99                          unsigned char *data, unsigned int datalen,
100                          apr_pool_t *pool)</dt>
101      <dd>Cette fonction permet de stocker un objet dans une instance de
102      cache.</dd>
103
104      <dt>apr_status_t retrieve(ap_socache_instance_t *instance, server_rec *s,
105                             const unsigned char *id, unsigned int idlen,
106                             unsigned char *data, unsigned int *datalen,
107                             apr_pool_t *pool)</dt>
108      <dd>Cette fonction permet d'extraire un objet du cache.</dd>
109
110      <dt>apr_status_t remove(ap_socache_instance_t *instance, server_rec *s,
111                           const unsigned char *id, unsigned int idlen,
112                           apr_pool_t *pool)</dt>
113      <dd>Supprime un objet du cache.</dd>
114
115      <dt>void status(ap_socache_instance_t *instance, request_rec *r, int flags)</dt>
116      <dd>Renvoie le statut d'une instance de cache à destination de mod_status.</dd>
117
118      <dt>apr_status_t iterate(ap_socache_instance_t *instance, server_rec *s,
119                            void *userctx, ap_socache_iterator_t *iterator,
120                            apr_pool_t *pool)</dt>
121      <dd>Envoie tous les objets gardés en cache à une fonction pour traitement itératif.</dd>
122    </dl>
