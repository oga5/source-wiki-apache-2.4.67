38    <p>Ce document couvre l'installation et la compilation du serveur
39    HTTP Apache
40    sur les systèmes Unix et similaires seulement. Pour la compilation et
41    l'installation sous Windows, voir <a href="platform/windows.html">Utiliser le serveur HTTP Apache avec Microsoft

168<h2><a name="requirements" id="requirements">Prérequis</a></h2>
169
170    <p>Les prérequis pour la construction d'Apache httpd sont les suivants:</p>
171
172    <dl>
173      <dt>APR et APR-Util</dt>
174      <dd>APR et APR-Util doivent être déjà installés sur votre système.
175      Si ce n'est pas le cas, ou si vous préférez ne pas utiliser les
176      versions fournies par le système, téléchargez les dernières
177      versions d'APR et APR-Util depuis <a href="http://apr.apache.org/">Apache APR</a>, décompressez-les
178      respectivement dans <code>/racine_sources_httpd/srclib/apr</code> et
179      <code>/racine_sources_httpd/srclib/apr-util</code> (les noms des répertoires ne
180      doivent pas comporter de numéros de versions ; par exemple, la
181      distribution d'APR doit se trouver dans /racine_sources_httpd/srclib/apr/), et
182      utilisez l'option <code>--with-included-apr</code> du script
183      <code>./configure</code>. Sur certaines plateformes, vous devrez
184      peut-être installer les paquets <code>-dev</code> correspondants
185      pour permettre la compilation de httpd avec les versions
186      installées d'APR et APR-Util.</dd>
187
188      <dt>Bibliothèque d'expressions rationnelles compatibles Perl
189      (PCRE)</dt>
190      <dd>Cette bibliothèque est nécessaire mais n'est plus fournie avec la
191      distribution de httpd. Téléchargez le code source depuis <a href="http://www.pcre.org/">http://www.pcre.org</a> ou installez
192      un portage du paquet. Si votre suite de compilation ne trouve pas
193      le script pcre-config installé au cours du processus de
194      construction de PCRE, indiquez son chemin via l'option
195      <code>--with-pcre</code> du script <code>./configure</code>. Sur
196      certaines plateformes, vous devrez
197      peut-être installer les paquets <code>-dev</code> correspondants
198      pour permettre la compilation de httpd avec la version
199      installée de PCRE.</dd>
200
201      <dt>Espace disque</dt>
202      <dd>Assurez-vous d'avoir au moins 50 Mo d'espace disque disponible
203      temporaire. Après l'installation le serveur occupe
204      approximativement 10 Mo d'espace disque. L'espace disque réellement
205      nécessaire va varier considérablement en fonction de vos options
206      de configuration, de la présence éventuelle de
207      modules tiers, et bien entendu de la taille de votre site web et
208      des sites que vous hébergez sur votre serveur.</dd>
209
210      <dt>Compilateur ANSI-C et système de construction</dt>
211      <dd>Vous devez disposer d'un compilateur ANSI-C. Le compilateur <a href="http://gcc.gnu.org/">GNU C (GCC)</a> de la <a href="http://www.gnu.org/">Free Software Foundation (FSF)</a>
212      est recommandé. Si vous ne possédez pas GCC,
213      assurez-vous au moins que votre compilateur soit compatible ANSI.
214      En outre, votre <code>PATH</code> doit contenir
215      les outils de construction de base tels que <code>make</code>.</dd>
216
217      <dt>Connaissance de l'heure exacte</dt>
218      <dd>Les éléments du protocole HTTP font référence à l'heure du jour.
219      Par conséquent, il est nécessaire d'équiper votre système d'un
220      dispositif de synchronisation du temps. Les programmes
221      <code>ntpdate</code> ou <code>xntpd</code>, basés sur le protocole NTP,
222      sont couramment utilisés à cet effet.
223      Voir la <a href="http://www.ntp.org">page d'accueil de NTP</a>
224      pour plus de détails à propos du logiciel NTP et des serveurs
225      de temps publics.</dd>
226
227      <dt><a href="http://www.perl.org/">Perl 5</a>
228      [OPTIONNEL]</dt>
229      <dd>L'interpréteur Perl 5 (les versions 5.003 ou supérieures conviennent)
230      est nécessaire pour l'exécution de certains scripts comme
231      <code class="program"><a href="./programs/apxs.html">apxs</a></code> ou <code class="program"><a href="./programs/dbmmanage.html">dbmmanage</a></code>
232      (qui sont écrits en Perl).
233      Si le script <code class="program"><a href="./programs/configure.html">configure</a></code> ne trouve pas d'interpréteur
234      Perl 5, vous ne pourrez pas utiliser les scripts qui en ont besoin.
235      Bien entendu, vous pourrez tout de même construire et utiliser
236      Apache httpd.</dd>
237
238    </dl>

241<h2><a name="download" id="download">Téléchargement</a></h2>
242
243    <p>Le serveur HTTP Apache peut être téléchargé à partir du
244    <a href="http://httpd.apache.org/download.cgi">site de téléchargement
245    du serveur HTTP Apache</a>, qui fournit la liste de nombreux miroirs.
246    Il sera plus commode à la plupart des utilisateurs d'Apache sur les
247    systèmes UNIX ou similaires de télécharger et de compiler
248    la version sources.  Le processus de construction (décrit ci-dessous) est
249    simple, et vous permet de personnaliser votre serveur selon vos besoins.
250    En outre, les versions binaires sont souvent plus anciennes que les
251    dernières versions sources. Si vous téléchargez une version binaire,
252    suivez les instructions décrites dans le fichier
253    <code>INSTALL.bindist</code> inclus dans la distribution.</p>
254
255    <p>Après le téléchargement, il est important de vérifier que vous
256    disposez d'une version complète et non modifiée du serveur HTTP Apache.
257    Vous pouvez le faire en testant l'archive téléchargée à l'aide de
258    la signature PGP. Vous trouverez les détails de cette opération sur la <a href="http://httpd.apache.org/download.cgi#verify">page de téléchargement</a> ainsi qu'un exemple précis décrivant <a href="http://httpd.apache.org/dev/verification.html">l'utilisation de
259    PGP</a>.</p>

263<h2><a name="extract" id="extract">Extraction</a></h2>
264
265    <p>L'extraction des sources depuis l'archive du serveur HTTP Apache consiste

275    dans ce répertoire avant de procéder à la compilation du serveur.</p>

278<h2><a name="configure" id="configure">Configuration de l'arborescence des sources</a></h2>
279
280    <p>L'étape suivante consiste à configurer l'arborescence des sources
281    d'Apache en fonction de votre plateforme et de vos besoins personnels.
282    Le script  <code class="program"><a href="./programs/configure.html">configure</a></code>, situé à la racine du
283    répertoire de la distribution, a été conçu à cet effet
284    (Les développeurs qui téléchargent
285    une version non officielle de l'arborescence des sources d'Apache
286    devront disposer de
287    <code>autoconf</code> et <code>libtool</code> et
288    exécuter <code>buildconf</code> avant de passer à l'étape suivante,
289    ce qui n'est pas nécessaire pour les versions officielles).</p>
290
291    <p>Pour configurer l'arborescence des sources avec les valeurs par défaut
292    pour toutes les options, entrez simplement <code>./configure</code>.
293    Pour modifier les valeurs des options, <code class="program"><a href="./programs/configure.html">configure</a></code>
294    accepte toute une variété de variables et
295    d'options de ligne de commande.</p>
296
297    <p>L'option la plus importante <code>--prefix</code> est le chemin
298    du répertoire d'installation d'Apache, car Apache doit être configuré
299    en fonction de ce chemin pour pouvoir fonctionner correctement.
300    Il est possible de définir plus finement le chemin d'installation des fichiers
301    à l'aide d'<a href="programs/configure.html#installationdirectories">options
302    supplémentaires de configure</a>.</p>
303
304    <p>À ce niveau, vous pouvez aussi spécifier de quelles <a href="programs/configure.html#optionalfeatures">fonctionnalités</a> vous
305    voulez disposer dans Apache en activant ou désactivant des <a href="mod/">modules</a>.  Apache est fourni avec un grand nombre de
306    modules inclus par défaut. Ils seront compilés en tant qu'<a href="dso.html">objets partagés (DSOs)</a> qui pourront être chargés
307    ou déchargés à l'exécution. Vous pouvez aussi choisir de compiler
308    les modules statiquement via l'option
309    <code>--enable-<var>module</var>=static</code>.</p>
310    <p>Des modules supplémentaires peuvent être activés à l'aide de l'option
311    <code>--enable-<var>module</var></code>, où
312    <var>module</var> est le nom du module sans la chaîne
313    <code>mod_</code> et où tout caractère de soulignement est converti
314    en tiret. D'une manière similaire,
315    vous pouvez désactiver des modules à l'aide de l'option
316    <code>--disable-<var>module</var></code>.  Faites très attention
317    en utilisant ces options, car <code class="program"><a href="./programs/configure.html">configure</a></code> n'est pas en
318    mesure de vous avertir si le module que vous avez spécifié n'existe pas;
319    il ignorera tout simplement l'option.</p>
320
321    <p>En outre, vous devrez peut-être fournir au script
322    <code class="program"><a href="./programs/configure.html">configure</a></code> des informations supplémentaires sur
323    le chemin de votre compilateur, de vos bibliothèques, ou de vos fichiers
324    d'en-têtes.  A cet effet, vous pouvez passer des options de ligne de
325    commande ou des variables d'environnement au script
326    <code class="program"><a href="./programs/configure.html">configure</a></code>. Pour plus d'informations, voir la
327    page de manuel de <code class="program"><a href="./programs/configure.html">configure</a></code>, ou lancez le script
328    <code class="program"><a href="./programs/configure.html">configure</a></code> avec l'option <code>--help</code>.
329    </p>
330
331    <p>Pour vous faire une idée des possibilités qui s'offrent à vous, voici
332    un exemple typique de compilation d'Apache avec le répertoire
333    d'installation <code>/sw/pkg/apache</code>, un compilateur et des drapeaux
334    particuliers et les deux modules additionnels <code class="module"><a href="./mod/mod_ldap.html">mod_ldap</a></code>
335    et <code class="module"><a href="./mod/mod_lua.html">mod_lua</a></code> :</p>
336
337<div class="example"><p><code>
338      $ CC="pgcc" CFLAGS="-O2" \<br />
339       ./configure --prefix=/sw/pkg/apache \<br />
340       --enable-ldap=shared \<br />
341       --enable-lua=shared
342</code></p></div>
343
344    <p>Plusieurs minutes peuvent être nécessaires à
345    <code class="program"><a href="./programs/configure.html">configure</a></code> pour tester la disponibilité des
346    fonctionnalités
347    au sein de votre système, et construire les Makefiles qui seront utilisés
348    par la suite pour compiler le serveur.</p>
349
350    <p>Vous trouverez une description détaillée des options de
351    <code class="program"><a href="./programs/configure.html">configure</a></code> dans sa page de manuel.</p>

354<h2><a name="compile" id="compile">Construction</a></h2>
355
356    <p>Vous pouvez maintenant construire les différents éléments qui

366<h2><a name="install" id="install">Installation</a></h2>
367
368    <p>Il est temps maintenant d'installer le paquet dans le répertoire
369    d'installation défini par <em>PREFIX</em> (voir plus haut l'option
370    <code>--prefix</code>) en lançant:</p>
371
372<div class="example"><p><code>$ make install</code></p></div>
373
374    <p>Cette étape nécessite habituellement les privilèges
375    de root, car <em>PREFIX</em> est en général un
376    répertoire possèdant des droits en écriture
377    restreints.</p>
378
379    <p>Si vous effectuez une mise à jour, l'installation n'écrasera pas
380    vos fichiers de configuration ou autres documents.</p>

383<h2><a name="customize" id="customize">Personnalisation</a></h2>
384

388
389<div class="example"><p><code>$ vi <em>PREFIX</em>/conf/httpd.conf</code></p></div>

392    <code><em>PREFIX</em>/docs/manual/</code> ou
393    <a href="http://httpd.apache.org/docs/2.4/">http://httpd.apache.org/docs/2.4/</a> pour la version la plus
394    récente de ce manuel et la liste complète des <a href="mod/directives.html">directives de configuration</a> disponibles.</p>

397<h2><a name="test" id="test">Test</a></h2>
398
399    <p>Vous pouvez maintenant <a href="invoking.html">démarrer</a> votre
400    serveur HTTP Apache en lançant:</p>
401
402<div class="example"><p><code>$ <em>PREFIX</em>/bin/apachectl -k start</code></p></div>
403
404    <p>Vous devriez alors pouvoir requérir votre premier document
405    à l'aide de l'URL <code>http://localhost/</code>. La page web que vous
406    voyez est située dans le répertoire défini par la directive
407    <code class="directive"><a href="./mod/core.html#documentroot">DocumentRoot</a></code>,
408    qui est généralement <code><em>PREFIX</em>/htdocs/</code>.
409    Pour <a href="stopping.html">arrêter</a> le serveur, lancez:</p>
410
411<div class="example"><p><code>$ <em>PREFIX</em>/bin/apachectl -k stop</code></p></div>

414<h2><a name="upgrading" id="upgrading">Mise à jour</a></h2>
415
416    <p>La première étape d'une mise à jour consiste à lire l'annonce de la
417    sortie de la nouvelle version et le fichier <code>CHANGES</code>
418    dans la distribution des sources afin de déceler toutes les modifications
419    qui pourraient affecter votre site. Lors d'un changement majeur de version
420    (par exemple de 2.0 à 2.2 ou de 2.2 à 2.4),
421    il y aura certainement des différences importantes quant à la
422    configuration de la compilation et de l'exécution qui nécessiteront des
423    ajustements manuels.  Tous les
424    modules devront aussi être mis à jour pour qu'ils s'adaptent aux
425    changements de l'API des modules.</p>
426
427    <p>La mise à jour d'une version mineure à la suivante (par exemple, de
428    2.2.55 à 2.2.57) est plus aisée.  Le processus <code>make install</code>
429    n'écrasera aucun de vos documents existants, fichiers de log,
430    ou fichiers de configuration.  De plus, les développeurs font tout
431    leur possible pour éviter les changements entraînant une
432    incompatibilité dans les options de
433    <code class="program"><a href="./programs/configure.html">configure</a></code>, la configuration de l'exécution, ou l'API
434    des modules d'une version mineure à l'autre.  Dans la plupart des cas,
435    vous pourrez utiliser une ligne de commande
436    <code class="program"><a href="./programs/configure.html">configure</a></code> identique, le même fichier de configuration,
437    et tous vos modules continueront de fonctionner.</p>
438
439    <p>Pour effectuer une mise à jour entre deux versions mineures,
440    commencez par trouver le fichier
441    <code>config.nice</code> dans le répertoire de <code>construction</code>
442    de votre serveur installé ou à la racine de l'arborescence des sources
443    de votre ancienne installation.  Il contient la reproduction exacte de la
444    ligne de commande <code class="program"><a href="./programs/configure.html">configure</a></code> que vous avez utilisée pour
445    configurer l'arborescence des sources.  Ensuite, pour mettre à jour
446    l'ancienne version vers la nouvelle,
447    il vous suffit de copier le fichier <code>config.nice</code> dans
448    l'arborescence des sources de la nouvelle version, de l'éditer pour
449    effectuer toute modification souhaitée, et de lancer :</p>
450
451    <div class="example"><p><code>
452    $ ./config.nice<br />
453    $ make<br />
454    $ make install<br />
455    $ <em>PREFIX</em>/bin/apachectl -k graceful-stop<br />
456    $ <em>PREFIX</em>/bin/apachectl -k start<br />
457    </code></p></div>
458
459    <div class="warning">Vous devez toujours effectuer un test de la nouvelle
460    version dans votre environnement avant de la mettre en production.
461    Par exemple, vous pouvez installer et exécuter la nouvelle version
462    en parallèle avec l'ancienne en utilisant une option
463    <code>--prefix</code> et un port différents (en ajustant la directive
464    <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>) afin de déceler toute
465    incompatibilité avant d'effectuer la mise à jour définitive.</div>
466
467    <p>Vous pouvez ajouter des arguments supplémentaires à
468    <code>config.nice</code> ; ils seront alors ajoutés aux options de
469    votre script <code class="program"><a href="./programs/configure.html">configure</a></code> original :</p>
470
471     <div class="example"><p><code>
472     $ ./config.nice --prefix=/home/test/apache --with-port=90
473     </code></p></div>

477<h2><a name="thirdp" id="thirdp">Paquets tiers</a></h2>
478
479    <p>De nombreux tiers fournissent leur propre distribution du
480    serveur HTTP Apache à installer sur une plate-forme particulière. On
481    peut citer les différentes distributions Linux, divers
482    paquets tiers Windows, Mac OS X, Solaris et de nombreux autres.</p>
483
484    <p>Notre license logicielle non seulement permet, mais aussi
485    encourage ce genre de redistribution. Cependant, ceci conduit à une
486    situation ou l'organisation de la configuration et les valeurs par
487    défaut de votre installation du serveur peuvent ne pas correspondre
488    à ce qui est écrit dans la documentation. Bien que fâcheuse, cette
489    situation n'est pas appelée à évoluer de sitôt.</p>
490
491    <p>Une  <a href="https://cwiki.apache.org/confluence/display/httpd/DistrosDefaultLayout">description
492    de ces distributions tierces</a> est maintenue dans le wiki du
493    serveur HTTP, et doit en refléter l'état actuel. Vous devrez
494    cependant vous familiariser par vous-même avec la gestion du paquet
495    de votre plate-forme particulière et les procédures d'installation.</p>
