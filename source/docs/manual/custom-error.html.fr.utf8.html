35
36    <p>Le serveur HTTP Apache fournit des messages d'erreur génériques
37    pour les codes de statut 4xx ou 5xx ; ces messages sont cependant
38    relativement austères, imprécis, et peuvent s'avérer intimidants
39    pour les visiteurs du site. Si vous le souhaitez, vous pouvez
40    afficher des messages d'erreur plus conviviaux, dans un langage
41    autre que l'anglais, ou même sous une forme plus en adéquation avec
42    le style de votre site.</p>
43
44    <p>Il est possible de définir des messages d'erreur personnalisés
45    pour chaque code de statut HTTP associé à une condition d'erreur -
46    c'est à dire tout code de statut 4xx ou 5xx.</p>
47
48    <p>De plus, il est possible de
49    personnaliser le message d'erreur en fonction d'un jeu de valeurs
50    fourni, en utilisant les <a href="howto/ssi.html">Inclusions Côté
51    Serveur (SSI)</a>. Un programme CGI ou un autre gestionnaire
52    dynamique (PHP, mod_perl, etc...) peut aussi utiliser ces variables
53    pour gérer les conditions d'erreur.</p>
54

65<h2><a name="configuration" id="configuration">Configuration</a></h2>
66
67    <p>Les messages d'erreur personnalisés sont configurés via la
68    directive <code class="directive"><a href="./mod/core.html#errordocument">ErrorDocument</a></code>, qui
69    peut être utilisée dans un contexte global, serveur virtuel ou
70    répertoire. On peut utiliser cette directive dans les fichiers
71    .htaccess si <code class="directive"><a href="./mod/core.html#allowoverride">AllowOverride</a></code> est
72    définie à FileInfo.</p>
73
74    <pre class="prettyprint lang-config">ErrorDocument 500 "Désolé, notre script s'est
75crashé ; comme c'est dommage !"
76ErrorDocument 500 /cgi-bin/crash-recover
77ErrorDocument 500 http://error.example.com/server_error.html
78ErrorDocument 404 /errors/not_found.html 
79ErrorDocument 401 /subscription/how_to_subscribe.html</pre>
80
81
82<p>La syntaxe de la directive <code>ErrorDocument</code> est :</p>
83      <pre class="prettyprint lang-config">ErrorDocument &lt;code_3_chiffres&gt; &lt;action&gt;</pre>
84
85      <p>où action peut être traitée comme :</p>
86        <ol>
87      <li>Une URL de redirection local (si l'action commence par un "/").</li>
88      <li>Une URL de redirection externe (si action est une URL valide).</li>
89      <li>Le texte à afficher (si l'action ne répond à aucune des
90      deux conditions précédentes). Entourez le texte de guillemets (")
91      s'il contient plusieurs mots.</li>
92        </ol>
93
94    <p>Dans le cas d'une redirection vers une URL locale, des variables
95    d'environnement supplémentaires sont définies de façon à ce que la
96    réponse puisse être personnalisée par la suite. Elles ne sont pas
97    envoyées aux URLs externes.</p>
98

101<h2><a name="variables" id="variables">Variables disponibles</a></h2>
102
103      <p>La redirection vers une autre URL peut être utile, mais
104      seulement s'il est possible de transmettre certaines informations
105      qui pourront être utilisées pour expliquer ou journaliser
106      la condition d'erreur ou le problème plus clairement.</p>
107
108	<p>Pour y parvenir, lorsque la redirection d'erreur est envoyée,
109	des variables d'environnement supplémentaires sont définies à
110	partir des en-têtes de la requête originale en préfixant le nom
111	d'origine de l'en-tête par 'REDIRECT_', ce qui permet de fournir au
112	message d'erreur le contexte de la requête originelle.</p>
113
114      <p>Par exemple, en plus des variables d'environnement habituelles,
115      vous pouvez recevoir ce qui suit :</p>
116
117
118      <div class="example"><p><code>
119        REDIRECT_HTTP_ACCEPT=*/*, image/gif, image/jpeg, image/png<br />
120        REDIRECT_HTTP_USER_AGENT=Mozilla/5.0 Fedora/3.5.8-1.fc12 Firefox/3.5.8<br />
121        REDIRECT_PATH=.:/bin:/usr/local/bin:/sbin<br />
122        REDIRECT_QUERY_STRING=<br />
123        REDIRECT_REMOTE_ADDR=121.345.78.123<br />
124        REDIRECT_REMOTE_HOST=client.example.com<br />
125        REDIRECT_SERVER_NAME=www.example.edu<br />
126        REDIRECT_SERVER_PORT=80<br />
127        REDIRECT_SERVER_SOFTWARE=Apache/2.2.15<br />
128        REDIRECT_URL=/cgi-bin/buggy.pl
129      </code></p></div>
130
131      <p>Les variables d'environnement <code>REDIRECT_</code> sont
132      créées à partir des variables d'environnement préexistantes à la
133      redirection qui sont préfixées par la chaîne <code>REDIRECT_</code> ;
134      par exemple, <code>HTTP_USER_AGENT</code> devient
135      <code>REDIRECT_HTTP_USER_AGENT</code>.</p>
136
137      <p><code>REDIRECT_URL</code>, <code>REDIRECT_STATUS</code>, et
138      <code>REDIRECT_QUERY_STRING</code> sont systématiquement définies,
139      les autres variables n'étant définies que si l'en-tête
140      correspondant existait avant la condition d'erreur.</p>
141
142      <p><strong>Aucune</strong> d'entre elles ne sera définie si votre
143      directive <code class="directive"><a href="./mod/core.html#errordocument">ErrorDocument</a></code>
144      spécifie une redirection <em>externe</em> (toute URL commençant
145      par un protocole du style <code>http:</code>, même si elle fait
146      référence au même hôte que le serveur).</p>

150<h2><a name="custom" id="custom">Personnalisation des messages d'erreur</a></h2>
151
152
153	<p>Si vous faites pointer votre directive
154	<code>ErrorDocument</code> vers certains gestionnaires
155	dynamiques comme les inclusions côté serveur, les scripts CGI ou
156	d'autres gestionnaires, vous pouvez utiliser les variables
157	d'environnement supplémentaires disponibles pour personnaliser
158	le message.</p>
159
160
161      <p>Si la directive ErrorDname-basedocument spécifie une redirection locale
162      vers un script CGI, ce dernier doit ajouter un en-tête
163      "<code>Status:</code>" dans sa sortie afin de s'assurer du bon
164      acheminement jusqu'au client de la condition d'erreur qui a
165      provoqué cette redirection. Par exemple, un script Perl spécifié
166      par une directive ErrorDocument pourrait contenir ce qui suit
167      :</p>
168
169      <pre class="prettyprint lang-perl">...
170print  "Content-type: text/html\n"; 
171printf "Status: %s Condition Intercepted\n", $ENV{"REDIRECT_STATUS"}; 
172...</pre>
173
174
175      <p>Si un script est dédié à la gestion d'une condition d'erreur
176      spécifique, telle que <code>404&nbsp;Not&nbsp;Found</code>, il
177      peut utiliser le code et le texte de l'erreur spécifiques à la
178      place.</p>
179
180      <p>Notez que si la réponse contient un en-tête
181      <code>Location:</code> (afin d'initier une redirection côté
182      client), le script <em>doit</em> émettre un en-tête approprié
183      (comme <code>302&nbsp;Found</code>). Dans le cas contraire,
184      l'en-tête <code>Location:</code> ne produira aucun effet.</p>

187<h2><a name="multi-lang" id="multi-lang">Messages d'erreur personnalisés
188  multilingues</a></h2>
189
190    <p>Vous trouverez dans la distribution du serveur HTTP Apache un
191    répertoire contenant des messages d'erreur personnalisés traduits en
192    16 langues différentes. Pour activer cette fonctionnalité, vous
193    pouvez aussi inclure un fichier de configuration qui se trouve dans
194    le répertoire de configuration <code>conf/extra</code>.</p>
195
196    <p>Dans le fichier de configuration de votre serveur, vous trouverez
197    un groupe de lignes du style :</p>
198
199    <pre class="prettyprint lang-config">    # Multi-language error messages
200    #Include conf/extra/httpd-multilang-errordoc.conf</pre>
201
202
203    <p>Décommentez la ligne <code>Include</code> pour activer cette
204    fonctionnalité, et présenter des messages d'erreur dont le langage
205    sera négocié en fonction du langage préféré défini au niveau du
206    navigateur du client.</p>
207
208    <p>De plus, ces documents contiennent diverses variables
209    <code>REDIRECT_</code>, de façon à ce que l'utilisateur final
210    dispose d'informations supplémentaires à propos de ce qui a pu se
211    produire, et de ce qu'il est susceptible de faire maintenant.</p>
212
213    <p>Ces documents peuvent être personnalisés en fournissant autant
214    d'informations utiles que vous le souhaitez aux utilisateurs à
215    propos de votre site, et de ce qu'ils sont susceptibles d'y trouver.</p>
216
217    <p>Pour pouvoir utiliser cette fonctionnalité, vous devez activer
218    <code class="module"><a href="./mod/mod_include.html">mod_include</a></code> et <code class="module"><a href="./mod/mod_negotiation.html">mod_negotiation</a></code>.</p>
