67<h2><a name="http_recommendations" id="http_recommendations">Recommandations HTTP</a></h2>
68
69    <p>Indépendamment des modules compilés et utilisés, Apache en
70    tant que serveur web de base respecte les recommandations IETF
71    suivantes :</p>
72
73    <dl>
74      <dt><a href="http://www.rfc-editor.org/rfc/rfc1945.txt">RFC 1945</a>
75      (Informations)</dt>
76
77      <dd>Le Protocole de Transfert Hypertexte (Hypertext Transfer
78      Protocol - HTTP) est un protocole de niveau application avec la
79      clarté et la vitesse nécessaires pour les systèmes d'informations
80      distribués, collaboratifs et hypermédia. Cette RFC documente le
81      protocole HTTP/1.0.</dd>
82
83      <dt><a href="http://www.rfc-editor.org/rfc/rfc2616.txt">RFC 2616</a>
84      (Série de standards)</dt>
85
86      <dd>Le Protocole de Transfert Hypertexte (Hypertext Transfer
87      Protocol - HTTP) est un protocole de niveau application pour les
88      systèmes d'informations distribués, collaboratifs et hypermédia.
89      Cette RFC documente le protocole HTTP/1.1.</dd>
90
91      <dt><a href="http://www.rfc-editor.org/rfc/rfc2396.txt">RFC 2396</a>
92      (Série de standards)</dt>
93
94      <dd>Un Identificateur de Ressource Uniforme (Uniform Resource
95      Identifier - URI) est une chaîne de caractères compacte permettant
96      d'identifier une ressource physique ou abstraite.</dd>
97
98      <dt><a href="http://www.rfc-editor.org/rfc/rfc4346.txt">RFC 4346</a>
99      (Série de standards)</dt>
100
101      <dd>Le protocole TLS permet l'utilisation de communications
102      sécurisées sur l'Internet. Il fournit le chiffrement, et a été
103      conçu pour se prémunir contre l'interception, la modification et
104      la falsification de messages.</dd>
105    </dl>

109<h2><a name="html_recommendations" id="html_recommendations">Recommandations HTML</a></h2>
110
111    <p>En ce qui concerne le langage HTML, Apache respecte les
112    recommandations IETF et W3C suivantes :</p>
113
114    <dl>
115      <dt><a href="http://www.rfc-editor.org/rfc/rfc2854.txt">RFC 2854</a>
116      (Informations)</dt>
117
118      <dd>Ce document résume l'historique du développement de HTML, et
119      définit le type MIME "text/html" en pointant les recommandations
120      W3C correspondantes.</dd>
121
122      <dt><a href="http://www.w3.org/TR/html401">Spécification HTML
123      4.01</a>
124      (<a href="http://www.w3.org/MarkUp/html4-updates/errata">Corrections
125      d'erreurs</a>)
126      </dt>
127
128      <dd>Cette spécification définit le Langage à Balises HyperTexte
129      (HyperText Markup Language - HTML), le langage de publication du
130      World Wide Web. Elle définit HTML 4.01, qui est une sous-version
131      de HTML 4.</dd>
132
133      <dt><a href="http://www.w3.org/TR/REC-html32">Référence HTML
134      3.2</a></dt>
135
136      <dd>Le langage à Balises HyperTexte (HyperText Markup Language -
137      HTML) est un langage à balises simple permettant de créer des
138      documents hypertextes portables. Les documents HTML sont aussi des
139      documents SGML.</dd>
140
141      <dt><a href="http://www.w3.org/TR/xhtml11/">XHTML 1.1 -
142      XHTML sous forme de modules</a>
143      (<a href="http://www.w3.org/MarkUp/2009/xhtml11-2nd-edition-errata.html">Corrections
144      d'erreurs</a>)
145      </dt>
146
147      <dd>Cette recommandation définit un nouveau type de document XHTML
148      basé sur le cadre de développement des modules et les modules
149      définis dans la modularisation de XHTML.</dd>
150
151      <dt><a href="http://www.w3.org/TR/xhtml1">XHTML 1.0, le Langage à
152      Balises Hypertexte Extensible (Extensible HyperText Markup
153      Language) - Seconde édition</a>
154      (<a href="http://www.w3.org/2002/08/REC-xhtml1-20020801-errata/">Corrections
155      d'erreurs</a>)
156      </dt>
157
158      <dd>Cette spécification définit la seconde édition de XHTML 1.0,
159      une reformulation de HTML 4 en tant qu'application XML 1.0, ainsi
160      que trois DTDs correspondant à celles définies par HTML 4.</dd>
161    </dl>

166
167    <p>En ce qui concerne les différentes méthodes d'authentification,
168    Apache respecte les recommandations IETF suivantes :</p>
169
170    <dl>
171      <dt><a href="http://www.rfc-editor.org/rfc/rfc2617.txt">RFC 2617</a>
172      (Le track des standards)</dt>
173
174      <dd>"HTTP/1.0", y compris la spécification d'un protocole basique
175      d'authentification et de contrôle d'accès.</dd>
176
177    </dl>

181<h2><a name="language_country_codes" id="language_country_codes">Codes de langues et de
182  pays</a></h2>
183
184    <p>Les liens suivants fournissent des informations à propos des
185    codes de langues et de pays aux normes ISO ou autres :</p>
186
187    <dl>
188      <dt><a href="http://www.loc.gov/standards/iso639-2/">ISO 639-2</a></dt>
189
190      <dd>ISO 639 fournit deux jeux de codes de langues permettant de
191      représenter les noms des langues ; le premier est
192      un jeu de codes sur deux lettres (639-1), le second (celui
193      présenté dans le lien ci-dessus), est un jeu de codes sur trois
194      lettres (639-2).</dd>
195
196      <dt><a href="http://www.iso.org/iso/country_codes">
197      ISO 3166-1</a></dt>
198
199      <dd>Ce document présente les noms de pays (les noms raccourcis
200      officiels en anglais) dans l'ordre alphabétique, tels qu'ils sont
201      présentés dans la norme ISO 3166-1 et les éléments de codes
202      correspondants de la norme ISO 3166-1-alpha-2.</dd>
203
204      <dt><a href="http://www.rfc-editor.org/rfc/bcp/bcp47.txt">BCP 47</a>
205      (Les meilleurs pratiques courantes),
206      <a href="http://www.rfc-editor.org/rfc/rfc3066.txt">RFC 3066</a></dt>
207
208      <dd>Ce document décrit une balise de langue permettant de
209      spécifier la langue utilisé dans un objet contenant des
210      informations, la manière d'enregistrer des valeurs à utiliser dans
211      cette balise de langage, et une méthode pour comparer les balises
212      de langue de ce style.</dd>
213
214      <dt><a href="http://www.rfc-editor.org/rfc/rfc3282.txt">RFC 3282</a>
215      (Série de standards)</dt>
216
217      <dd>Ce document définit un en-tête "Content-language:" permettant
218      de spécifier le langage d'un élément possédant des en-têtes du
219      style RFC 822, comme les portions de corps MIME ou les documents
220      Web, et un en-tête "Accept-Language:" permettant de spécifier des
221      préférences en matière de langue.</dd>
222    </dl>
