53<h2><a name="startup" id="startup">Comment Apache démarre</a></h2>
54
55    <p>Si la directive <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
56    spécifiée dans le fichier de configuration est à sa valeur par défaut
57    de 80 (ou tout autre port inférieur à 1024), il est nécessaire de
58    posséder les privilèges root pour pouvoir démarrer apache, et lui
59    permettre d'être associé à ce port privilégié. Lorsque le serveur est
60    démarré, il effectue quelques opérations préliminaires
61    comme ouvrir ses fichiers de log, puis il lance plusieurs processus
62    <em>enfants</em> qui ont pour rôle d'écouter et de répondre aux
63    requêtes des clients. Le processus <code>httpd</code> principal
64    continue à s'exécuter sous l'utilisateur root, tandis que les processus
65    enfants s'exécutent sous un utilisateur aux privilèges restreints.
66    Ceci s'effectue par la voie du
67    <a href="mpm.html">Module Multi-Processus (MPM)</a>.</p>
68
69    <p>Il est recommandé d'utiliser le script de contrôle
70    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> pour invoquer l'exécutable
71    <code class="program"><a href="./programs/httpd.html">httpd</a></code>. A cet effet, ce script définit certaines variables
72    d'environnement nécessaires pour permettre à
73    <code class="program"><a href="./programs/httpd.html">httpd</a></code> de fonctionner correctement sous certains systèmes
74    d'exploitation.
75    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> accepte des arguments de ligne de
76    commande ;
77    ainsi toute option de <code class="program"><a href="./programs/httpd.html">httpd</a></code> peut aussi être utilisée avec
78    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code>.  Vous pouvez aussi éditer directement le
79    script <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> en modifiant la variable
80    <code>HTTPD</code> située en début de script pour spécifier la
81    localisation du binaire <code class="program"><a href="./programs/httpd.html">httpd</a></code> et tout argument de ligne
82    de commande que vous souhaitez voir <em>systématiquement</em> présent.</p>
83
84    <p>La première chose qu'effectue <code class="program"><a href="./programs/httpd.html">httpd</a></code> quand il est
85    invoqué est de localiser et lire le <a href="configuring.html">fichier de configuration</a>
86    <code>httpd.conf</code>. La localisation de ce fichier est définie à la
87    compilation, mais il est possible d'en spécifier une autre à
88    l'exécution en utilisant l'option de ligne de commande <code>-f</code> comme suit:</p>
89
90<div class="example"><p><code>/usr/local/apache2/bin/apachectl -f
91      /usr/local/apache2/conf/httpd.conf</code></p></div>
92
93    <p>Si tout se passe bien pendant le démarrage, le serveur va se dissocier
94    du terminal et l'invite de commande réapparaîtra presque immédiatement.
95    Ceci indique que le serveur a démarré et est en cours d'exécution.
96    À partir de ce moment, vous pouvez utiliser votre navigateur pour vous connecter
97    au serveur et afficher la page de test située dans le répertoire défini
98    par la directive <code class="directive"><a href="./mod/core.html#documentroot">DocumentRoot</a></code></p>

101<h2><a name="errors" id="errors">Erreurs en cours de démarrage</a></h2>
102
103    <p>Si un problème fatal survient pendant le démarrage
104    d'Apache, ce dernier va
105    afficher un message décrivant le problème sur la console ou
106    enregistrer ces informations dans le fichier défini par la directive
107    <code class="directive"><a href="./mod/core.html#errorlog">ErrorLog</a></code> avant de quitter.
108    Un des messages d'erreur les plus courants est "<code>Unable
109    to bind to Port ...</code>". Ce message d'erreur est habituellement
110    provoqué par :</p>
111
112    <ul>
113      <li>Une tentative de démarrage du serveur avec un port privilégié sans
114      être connecté root</li>
115
116      <li>Une tentative de démarrage du serveur alors qu'une autre instance
117      d'Apache ou un autre serveur web est déjà associé au même port.</li>
118    </ul>
119
120    <p>Pour plus d'instructions de dépannage, consultez la
121    <a href="https://cwiki.apache.org/confluence/display/httpd/FAQ">FAQ</a> Apache.</p>

124<h2><a name="boot" id="boot">Lancement au démarrage du système</a></h2>
125
126    <p>Si vous souhaitez que votre serveur web soit automatiquement
127    disponible après
128    un redémarrage du système, vous devez ajouter un appel à
129    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> à vos
130    fichiers de démarrage système (en général <code>rc.local</code> ou un
131    fichier dans un répertoire <code>rc.N</code>), ce qui démarrera Apache sous
132    l'utilisateur root. Avant de faire ceci, assurez-vous que votre serveur
133    soit correctement configuré en ce qui concerne la sécurité et les
134    restrictions d'accès.</p>
135
136    <p>Le script <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> est conçu pour fonctionner
137    comme un script d'initialisation SysV standard ; il accepte les arguments
138    <code>start</code>, <code>restart</code>, et <code>stop</code>
139    et les traduit en signaux appropriés pour
140    <code class="program"><a href="./programs/httpd.html">httpd</a></code>, et il suffit en général d'installer
141    un lien vers
142    <code class="program"><a href="./programs/apachectl.html">apachectl</a></code> dans le répertoire d'initialisation approprié.
143    Mais prenez soin de vérifier les besoins exacts de votre système
144    en la matière.</p>
