44<h2><a name="introduction" id="introduction">Introducci&#243;n</a></h2>
45
46    <p>Apache est&#225; dise&#241;ado para ser un servidor web potente
47    y flexible que pueda funcionar en la m&#225;s amplia variedad de
48    plataformas y entornos. Las diferentes plataformas y los
49    diferentes entornos, hacen que a menudo sean necesarias diferentes
50    caracter&#237;sticas o funcionalidades, o que una misma
51    caracter&#237;stica o funcionalidad sea implementada de diferente
52    manera para obtener una mayor eficiencia. Apache se ha adaptado
53    siempre a una gran variedad de entornos a trav&#233;s de su
54    dise&#241;o modular. Este dise&#241;o permite a los
55    administradores de sitios web elegir que caracter&#237;sticas van
56    a ser incluidas en el servidor seleccionando que m&#243;dulos se
57    van a cargar, ya sea al compilar o en tiempo de ejecuci&#243;n.</p>
58
59    <p>Apache 2.0 extiende este dise&#241;o modular hasta las
60    funciones m&#225;s b&#225;sicas de un servidor web. El servidor
61    viene con una serie de M&#243;dulos de MultiProcesamiento que son
62    responsables de conectar con los puertos de red de la
63    m&#225;quina, aceptar las peticiones, y generar los procesos hijo
64    que se encargan de servirlas.</p>
65
66    <p>La extensi&#243;n del dise&#241;o modular a este nivel del
67    servidor ofrece dos beneficios importantes:</p>
68
69    <ul>
70      <li>Apache puede soportar de una forma m&#225;s f&#225;cil y
71      eficiente una amplia variedad de sistemas operativos. En
72      concreto, la versi&#243;n de Windows de Apache es mucho m&#225;s
73      eficiente, porque el m&#243;dulo <code class="module"><a href="./mod/mpm_winnt.html">mpm_winnt</a></code>
74      puede usar funcionalidades nativas de red en lugar de usar la
75      capa POSIX como hace Apache 1.3. Este beneficio se extiende
76      tambi&#233;n a otros sistemas operativos que implementan sus
77      respectivos MPMs.</li>
78
79      <li>El servidor puede personalizarse mejor para las necesidades
80      de cada sitio web. Por ejemplo, los sitios web que necesitan
81      m&#225;s que nada escalabilidad pueden usar un proceso MPM como
82      <code class="module"><a href="./mod/worker.html">worker</a></code>, mientras que los sitios web que
83      requieran por encima de otras cosas estabilidad o compatibilidad
84      con software antiguo pueden usar
85      <code class="module"><a href="./mod/prefork.html">prefork</a></code>.
86      </li>
87    </ul>
88
89    <p>A nivel de usuario, los MPMs son como cualquier otro
90    m&#243;dulo de Apache. La diferencia m&#225;s importante es que
91    solo un MPM puede estar cargado en el servidor en un determinado
92    momento. La lista de MPMs disponibles est&#225; en la <a href="mod/">secci&#243;n &#237;ndice de M&#243;dulos</a>.</p>

96<h2><a name="defaults" id="defaults">MPM por defecto</a></h2>
97
98<p>En la siguiente tabla se muestran los MPMs por defecto para varios
99sistemas operativos.  Estos ser&#225;n los MPM seleccionados si no se
100especifica lo contrario al compilar.</p>
101
102<table class="bordered"><tr><td>Netware</td><td><code class="module"><a href="./mod/mpm_netware.html">mpm_netware</a></code></td></tr>
103<tr class="odd"><td>OS/2</td><td><code class="module"><a href="./mod/mpmt_os2.html">mpmt_os2</a></code></td></tr>
104<tr><td>Unix</td><td><code class="module"><a href="./mod/prefork.html">prefork</a></code>, <code class="module"><a href="./mod/worker.html">worker</a></code>, or
105    <code class="module"><a href="./mod/event.html">event</a></code>, depending on platform capabilities</td></tr>
106<tr class="odd"><td>Windows</td><td><code class="module"><a href="./mod/mpm_winnt.html">mpm_winnt</a></code></td></tr>
107</table>
108
109<div class="note"><p>aqu&#237;, 'Unix' se usa para designar a los sistemas operativos "Unix-like", como
110Linux, BSD, Solaris, Mac OS X, etc.</p></div>
111
112<p>En el caso de los Unix, la decisi&#243;n de que MPM se va a instalar
113  depende de dos pregunas:</p>
114<p>1. &#191;Nos permite el Sistema Operativo hilos?</p>
115<p>2. -&#191;Nos permite el sistema operativo soporte a pila de hilos seguros 
116  (Especificamente, las funciones kqueue y epoll)?</p>
