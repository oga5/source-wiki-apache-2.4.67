46<h2><a name="overview" id="overview">概要</a></h2>
47    
48
49    <table class="related"><tr><th>関連モジュール</th><th>関連ディレクティブ</th></tr><tr><td><ul><li><code class="module"><a href="./mod/core.html">core</a></code></li><li><code class="module"><a href="./mod/mpm_common.html">mpm_common</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code></li><li><code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code></li></ul></td></tr></table>
50
51
52    <p>Apache は起動時に、ローカルマシンのあるポートおよびアドレス
53    に対して接続し、リクエストが来るのを待ちます。
54    デフォルトではマシンのすべてのアドレスに対して Listen します。
55    特定のポートか、特定のアドレスのみか、
56    またはそれらの組み合わせで Listen するように指定したい場合もあります。
57    異なる IP アドレス、ホスト名、ポートに対して Apache がどのように
58    応答するかを制御するバーチャルホスト機能と組み合わせてよく使われます。</p>
59
60    <p><code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
61    ディレクティブで、特定のポートやアドレス・ポートの組から入ってくる
62    リクエストのみを受け付けるようにできます。
63    もしポート番号だけが <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
64    ディレクティブで指定された場合は、
65    すべてのインターフェースの与えられたポート番号に対して
66    Listen します。 IP アドレスとポート番号とが同時に与えられた場合は、
67    サーバは与えられたインターフェースのポートを Listen します。
68    <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code> ディレクティブを複数使って
69    Listen するアドレスとポートをいくつも指定できます。
70    サーバは指定されたアドレスやポートからのリクエストすべてに
71    対して応答します。</p>
72
73    <p>たとえば、全てのインターフェースのポート 80 と 8000 の両方において
74    接続を受け付けるには</p>
75
76    <div class="example"><p><code>
77      Listen 80<br />
78      Listen 8000
79    </code></p></div>
80
81    <p>とします。
82    あるインターフェースでは 80 番で、また、同時に他のインターフェースの
83    8000 番ポートで接続を受け付けるには、
84    </p>
85
86    <div class="example"><p><code>
87      Listen 192.0.2.1:80<br />
88      Listen 192.0.2.5:8000
89    </code></p></div>
90
91    <p>とします。
92    IPv6 アドレスは、角括弧で次の例のように囲まなければいけません。</p>
93
94    <div class="example"><p><code>
95      Listen [2001:db8::a00:20ff:fea7:ccea]:80
96    </code></p></div>

99<h2><a name="ipv6" id="ipv6">IPv6 の特記事項</a></h2>
100    
101
102    <p>多くのプラットホームで IPv6 がサポートされてきていて、
103    <a class="glossarylink" href="./glossary.html#apr" title="用語集を参照">APR</a> はこれらのほとんどで IPv6 をサポートしているので、
104    Apache は IPv6 ソケットを割り当てて IPv6
105    経由で送られてきたリクエストを扱うことができます。</p>
106
107    <p>IPv6 ソケットが IPv4 と IPv6 コネクションの両方を扱うことができるか
108    どうかは、Apache 管理者にとって厄介な問題です。
109    IPv4 コネクションを IPv6 ソケットで扱う場合は、
110    IPv4 マップされた IPv6 アドレスを使用していて、
111    ほとんどのプラットホームではデフォルトで使用可能ですが、
112    FreeBSD, NetBSD, OpenBSD では、システム全体としてのポリシーとの整合性から、
113    デフォルトでは使用不可に設定されています。
114    これらのデフォルトで使用不可のプラットホームであっても、
115    特別な <code class="program"><a href="./programs/configure.html">configure</a></code> の
116    設定パラメータで Apache の挙動を変化させることができます。</p>
117
118    <p>一方で、Linux や Tru64 といったプラットホームで IPv4 と IPv6
119    の両方を扱うには、マップドアドレスを使用する<strong>以外の方法はありません</strong>。
120    IPv4 と IPv6 のコネクションを最小限のソケットで扱いたいのであれば、
121    IPv4 マップの IPv6 アドレスを使用する必要があり、
122    <code>--enable-v4-mapped</code> <code class="program"><a href="./programs/configure.html">configure</a></code>
123    オプションを指定します。</p>
124
125    <p><code>--enable-v4-mapped</code> は、
126    FreeBSD, NetBSD, OpenBSD 以外の全てのプラットホームでのデフォルトです。
127    ですから、おそらくお手元の Apache はこの設定でビルドされているでしょう。</p>
128
129    <p>プラットフォームや APR が何をサポートするかに関わらず、
130    IPv4 コネクションのみを扱うようにしたい場合は、
131    次の例のように全ての
132    <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code> ディレクティブで
133    IPv4 アドレスを指定してください。</p>
134
135    <div class="example"><p><code>
136      Listen 0.0.0.0:80<br />
137      Listen 192.0.2.1:80
138    </code></p></div>
139
140    <p>条件を満たすプラットホームで、Apache が
141    IPv4 と IPv6 のコネクションを別々のソケットで扱うようにしたい場合
142    (つまり IPv4 マップのアドレスを無効にしたい場合)
143    は、<code>--disable-v4-mapped</code>
144    <code class="program"><a href="./programs/configure.html">configure</a></code>
145    オプションを指定して、次のように個別指定の
146    <code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code>
147    ディレクティブを使用してください。
148    <code>--disable-v4-mapped</code> は、
149    FreeBSD, NetBSD, OpenBSD プラットホームでのデフォルトです。</p>

152<h2><a name="virtualhost" id="virtualhost">バーチャルホストに対してどう働くのか</a></h2>
153    
154
155    <p><code class="directive"><a href="./mod/mpm_common.html#listen">Listen</a></code> ディレクティブ
156    でバーチャルホストが実装されるわけではありません。
157    Listen は単にメインサーバにどのアドレスとポートを Listen すべきかを
158    教えるだけです。
159    <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
160    ディレクティブが使われない場合は、
161    受け入れたリクエストすべてに対して全く同じ挙動をします。
162    しかしながら
163    <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
164    を使って、
165    一つ以上のアドレスやポートに対して異なる挙動をするように
166    指定することができます。
167    VirtualHost を実装するには、まず初めに使用したいアドレスとポートに対して
168    サーバが Listen していなければなりません。
169    そして、その指定したアドレスとポートでの
170    このバーチャルホストの挙動を設定するために、
171    <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
172    セクションを作ります。もし
173    <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>
174    が Listen していないアドレスとポートに対して
175    設定されてしまうと、
176    それにはアクセスできないということに注意してください。</p>
