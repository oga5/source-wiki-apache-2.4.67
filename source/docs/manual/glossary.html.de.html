45<h2><a name="definitions" id="definitions">Definitionen</a></h2>
46    
47  <dl>
48    <dt><a name="algorithm" id="algorithm">Algorithmus</a></dt>
49    <dd>Eine eindeutige Formel oder ein Satz von Regeln zur L&#246;sung eines
50      Problems in einer endlichen Anzahl von Schritten. Algorithmen zur
51      Verschl&#252;sselung werden &#252;blicherweise <a class="glossarylink" href="./glossary.html#cipher" title="siehe Glossar"><dfn>Chiffre</dfn></a> genannt.
52    </dd>
53
54    <dt><a name="apacheextensiontool" id="apacheextensiontool">APache
55      eXtension Tool</a> <a name="apxs" id="apxs">(apxs)</a></dt>
56    <dd>Ein Perl-Skript zur Kompilierung von <a href="#module">Modul</a>-Quelltexten zu Dynamic-Shared-Objects
57      (<a class="glossarylink" href="./glossary.html#dso" title="siehe Glossar">DSOs</a>)  und zur Installation dieser zum
58      Apache-Webserver.<br />
59      Siehe: <code class="program"><a href="./programs/apxs.html">apxs</a></code>-Dokumentation
60    </dd>
61
62    <dt><a name="apacheportableruntime" id="apacheportableruntime">Apache Portable Runtime</a> <a name="apr" id="apr">(APR)</a></dt>
63    <dd>Eine Menge von Bibliotheken, die viele grundlegende Schnittstellen
64      zwischen dem Server und dem Betriebssystem anbietet. Die APR
65      wird parallel zum Apache HTTP Server in einem eigenst&#228;ndigen Projekt
66      entwickelt.<br />
67      Siehe auch: <a href="http://apr.apache.org/">Apache Portable Runtime
68      Project</a>
69    </dd>
70
71    <dt><a name="authentication" id="authentication">Authentifizierung</a></dt> 
72    <dd>Die positive Identifizierung einer Netzwerkeinheit, wie z.B.
73      eines Servers, eines Clients oder eines Benutzers.<br />
74      Siehe: <a href="howto/auth.html">Authentisierung, Autorisierung und
75      Zugriffskontrolle</a>
76    </dd>
77
78    <dt><a name="certificationauthority" id="certificationauthority">Certification Authority</a>
79      <span class="phonetic">[s&#601;&#712;tifi&#712;kei&#643;&#601;n
80      &#596;&#720;&#952;&#596;riti]</span>
81      <a name="ca" id="ca">(CA)</a></dt>
82    <dd><span class="transnote">(<em>Anm.d.&#220;.:</em> die Zertifizierungsstelle)</span> Eine
83      vertrauensw&#252;rdige dritte Partei, deren Zweck es ist,
84      Zertifikate f&#252;r Netzwerkeinheiten zu signieren. Andere
85      Netzwerkeinheiten k&#246;nnen die Signatur pr&#252;fen, um
86      sicherzustellen, dass eine CA den Inhaber eines Zertifikats
87      authentifiziert hat.<br />
88      Siehe: <a href="ssl/">SSL/TLS-Verschl&#252;sselung</a>
89    </dd>
90
91    <dt><a name="certificatsigningrequest" id="certificatsigningrequest">Certificate Signing Request</a>
92      <span class="phonetic">[s&#601;&#712;tifikit saini&#331;
93      ri&#712;kwest]</span>  <a name="csr" id="csr">(CSR)</a></dt> 
94    <dd><span class="transnote">(<em>Anm.d.&#220;.:</em> Zertifikats-Signierungsanfrage)</span> Ein unsigniertes
95      <a class="glossarylink" href="./glossary.html#certificate" title="siehe Glossar">Zertifikat</a> zur Einreichung bei
96      einer <a class="glossarylink" href="./glossary.html#certificationauthority" title="siehe Glossar">Zertifizierungsstelle</a>, welche
97      es mit dem <a class="glossarylink" href="./glossary.html#privatekey" title="siehe Glossar">privaten Schl&#252;ssel</a>
98      ihres CA-<em>Zertifikats</em> signiert. Durch die Signatur wird ein CSR
99      zum echten Zertifikat.<br />
100      Siehe: <a href="ssl/">SSL/TLS-Verschl&#252;sselung</a>
101    </dd>
102
103    <dt><a name="cipher" id="cipher">Chiffre</a></dt> 
104    <dd>Die <dfn>Chiffre</dfn> ist ein Algorithmus oder System zur
105      Datenverschl&#252;sselung. Beispiele sind DES, IDEA, RC4 usw. Im
106      Englischen spricht man von
107      <dfn>Cipher</dfn>&nbsp;<span class="phonetic">[&#712;saif&#601;]</span><br />
108      Siehe: <a href="ssl/">SSL/TLS-Verschl&#252;sselung</a>
109    </dd>
110
111    <dt><a name="ciphertext" id="ciphertext">Chiffretext</a></dt>
112    <dd>Das Ergebnis, nachdem ein <a class="glossarylink" href="./glossary.html#plaintext" title="siehe Glossar">Klartext</a>
113      eine <a class="glossarylink" href="./glossary.html#cipher" title="siehe Glossar">Chiffre</a> durchlaufen hat.<br />
114      Siehe: <a href="ssl/">SSL/TLS-Verschl&#252;sselung</a>
115    </dd>
116    
117    <dt><a name="commongatewayinterface" id="commongatewayinterface">Common
118      Gateway Interface</a> <span class="phonetic">[&#712;k&#596;m&#601;n geitwei
119      &#712;int&#601;&#720;feis]</span>
120      <a name="cgi" id="cgi">(CGI)</a></dt>
121    <dd>Eine einheitliche Definition einer Schnittstelle zwischen einem
122      Webserver und einem externen Programm, welcher dem externen Programm die
123      Behandlung von Anfragen erm&#246;glicht. Die Schnittstelle ist
124      urspr&#252;nglich von der <a href="http://hoohoo.ncsa.uiuc.edu/cgi/overview.html">NCSA</a>
125      definiert worden. Es exisitert jedoch auch ein <a href="http://cgi-spec.golux.com/">RFC-Projekt</a>.<br />
126      Siehe: <a href="howto/cgi.html">Dynamische Inhalte mit CGI</a>
127    </dd>
128
129    <dt><a name="connect" id="connect">CONNECT</a>
130      <span class="phonetic">[k&#601;nekt]</span></dt> 
131    <dd>Eine <a class="glossarylink" href="./glossary.html#method" title="siehe Glossar">HTTP-Methode</a> zur Weiterleitung
132      von Rohdaten &#252;ber HTTP. Sie kann dazu verwendet werden, andere
133      Protokolle wie zum Beispiel das SSL-Protokoll zu kapseln.
134    </dd>
135
136    <dt><a name="digitalsignature" id="digitalsignature">Digitale
137      Signatur</a></dt>
138    <dd>Ein chiffrierter Textblock, der die G&#252;ltigkeit eines Zertifikats
139      oder einer anderen Datei best&#228;tigt. Eine <a class="glossarylink" href="./glossary.html#certificationauthority" title="siehe Glossar">Zertifizierungsstelle</a> erstellt 
140      eine digitale Signatur durch Generierung eines <a class="glossarylink" href="./glossary.html#hash" title="siehe Glossar">Hashs</a> aus dem in einem <em>Zertifikat</em>
141      enthaltenen <em>&#246;ffentlichen Schl&#252;ssel</em> und
142      anschlie&#223;ender Codierung des Hashs mit dem <em>privaten
143      Schl&#252;ssel</em> des Zertifikats. Nur der &#246;ffentliche
144      Schl&#252;ssel der CA kann die Signatur decodieren. So wird
145      sichergestellt, dass die CA die Netwerkeinheit, welche das
146      <em>Zertifikat</em> besitzt, authentifiziert hat.<br />
147      Siehe: <a href="ssl/">SSL/TLS-Verschl&#252;sselung</a>
148    </dd>
149
150    <dt><a name="directive" id="directive">Direktive</a></dt> 
151    <dd>Eine Konfigurationsanweisung, die das Verhalten des Apache in einem
152      oder mehreren Punkten steuert. Direktiven werden in den <a class="glossarylink" href="./glossary.html#configurationfile" title="siehe Glossar">Konfigurationsdateien</a> gesetzt.<br />
153      Siehe: <a href="mod/directives.html">Verzeichnis der Direktiven</a>
154    </dd>
155
156    <dt><a name="dynamicsharedobject" id="dynamicsharedobject">Dynamic
157      Shared Object</a> 
158      <span class="phonetic">[dai&#712;n&#230;mik &#643;&#603;&#601;d
159      &#712;&#596;bd&#658;ikt]</span> <a name="dso" id="dso">(DSO)</a></dt>
160    <dd>Separat von der Apache-Bin&#228;rdatei <code class="program"><a href="./programs/httpd.html">httpd</a></code>
161      kompilierte <a class="glossarylink" href="./glossary.html#module" title="siehe Glossar">Module</a>, die bei Bedarf
162      geladen werden k&#246;nnen.<br />
163      Siehe: <a href="dso.html">Unterst&#252;tzung f&#252;r
164      Dynamic-Shared-Objects</a>
165    </dd>
166
167    <dt><a name="export-crippled" id="export-crippled">exportbeschr&#228;nkt</a></dt>
168    <dd>Verminderte kryptografische St&#228;rke (und Sicherheit), um den
169      Exportbesimmungen der Vereinigten Staaten <span class="transnote">(<em>Anm.d.&#220;.:</em> konkret: United
170      States' Export Administration Regulations (EAR))</span> zu
171      entsprechen. Exportbeschr&#228;nkte Verschl&#252;sselungssoftware ist
172      auf eine kurze Schl&#252;ssell&#228;nge begrenzt, was zu
173      <em>Chiffretexten</em> f&#252;hrt, die gew&#246;hnlich mittels
174      Brute-Force dekodiert werden k&#246;nnen.<br />
175      Siehe: <a href="ssl/">SSL/TLS-Verschl&#252;sselung</a>
176    </dd>
177
178    <dt><a name="filter" id="filter">Filter</a></dt>
179    <dd>Ein Verfahren, dass auf vom Server empfangene oder zu sendende Daten
180      angewendet wird. Eingabefilter verarbeiten vom Client an den Server
181      gesendetet Daten, w&#228;hrend Ausgabefilter vom Server an den Client zu
182      sendende Daten verarbeiten. Der Ausgabefilter <code>INCLUDES</code>
183      beispielsweise untersucht Dokumente nach <a class="glossarylink" href="./glossary.html#ssi" title="siehe Glossar">Server-Side-Includes</a> und f&#252;hrt sie aus.<br />
184      Siehe: <a href="filter.html">Filter</a>
185    </dd>
186
187    <dt><a name="handler" id="handler">Handler</a>
188      <span class="phonetic">[&#712;h&#230;ndl&#601;]</span></dt>
189    <dd>Eine Apache-interne Darstellung der Aktion, die beim Aufruf einer
190      Datei auszuf&#252;hren ist. Im Allgemeinen besitzen Dateien implizite,
191      auf dem Dateityp basierende Handler. Gew&#246;hnlich werden alle Dateien
192      vom Server bedient, einige Dateitypen werden jedoch separat "behandelt"
193      <span class="transnote">(<em>Anm.d.&#220;.:</em> besitzen einen separaten Handler)</span>. Der
194      <code>cgi-script</code>-Handler beispielsweise kennzeichnet Dateien, die
195      als <a class="glossarylink" href="./glossary.html#cgi" title="siehe Glossar">CGI-Programme</a> ausgef&#252;hrt werden
196      sollen.<br />
197      Siehe: <a href="handler.html">Verwendung von Apache-Handlern</a>
198    </dd>
199
200    <dt><a name="hash" id="hash">Hash</a>
201      <span class="phonetic">[h&#230;&#643;]</span></dt>
202    <dd>Ein mathematischer, unumkehrbarer Einweg-Algorithmus zur Generierung
203      einer Zeichenfolge fester L&#228;nge aus einer anderen Zeichenfolge
204      beliebiger L&#228;nge. Unterschiedliche Zeichenfolgen bei der Eingabe
205      ergeben &#252;blischerweise unterschiedliche Hashes (abh&#228;ngig von
206      der Hash-Funktion).
207    </dd>
208
209    <dt><a name="header" id="header">Header</a>
210      <span class="phonetic">[hed&#601;]</span></dt>
211    <dd>Der Teil der <a class="glossarylink" href="./glossary.html#http" title="siehe Glossar">HTTP</a>-Anfrage und -Antwort,
212      der vor den eigentlichen Daten &#252;bermittelt wird und den Inhalt
213      beschreibende Meta-Informationen enth&#228;lt.
214    </dd>
215
216    <dt><a name="htaccess" id="htaccess">.htaccess</a></dt>

222      Direktiven enthalten, nicht nur Direktiven zur Zugriffskontrolle.<br />
223      Siehe: <a href="configuring.html">Konfigurationsdateien</a>
224    </dd>
225
226    <dt><a name="httpd.conf" id="httpd.conf">httpd.conf</a></dt>

283    <dt><a name="mime-type" id="mime-type">MIME-Typ</a>
284      <span class="phonetic">[maim ty&#720;p]</span></dt>
285    <dd>Eine Art und Weise, den Typ des &#252;bermittelten Dokuments zu
286      beschreiben. Sein Name leitet sich davon ab, dass sein Format den
287      Multipurpose Internet Mail Extensions entlehnt wurde. Er besteht aus
288      einem Haupttyp und einem Untertyp, getrennt durch einen 
289      Schr&#228;gstrich. Einige Beispiele sind <code>text/html</code>,
290      <code>image/gif</code> und <code>application/octet-stream</code>.
291      Bei HTTP wird der MIME-Typ mit dem <a class="glossarylink" href="./glossary.html#header" title="siehe Glossar">Header</a> <code>Content-Type</code>
292      &#252;bermittelt.<br />
293      Siehe: <a href="mod/mod_mime.html">mod_mime</a>
294    </dd>
295
296    <dt><a name="module" id="module">Modul</a></dt>
297    <dd>Ein selbstst&#228;ndiger Teil eines Programms. Ein Gro&#223;teil der
298      Funktionalit&#228;t des Apache ist in Modulen enthalten, die Sie einbinden
299      oder entfernen k&#246;nnen. In die Apache-Bin&#228;rdatei <code class="program"><a href="./programs/httpd.html">httpd</a></code> einkompilierte Module werden <em>statische Module</em>
300      genannt, w&#228;hrend Module, die separat gespeichert sind und optional
301      zur Laufzeit geladen werden k&#246;nnen, <em>dynamische Module</em> oder
302      <a class="glossarylink" href="./glossary.html#dso" title="siehe Glossar">DSOs</a> genannt werden.
303      Standardm&#228;&#223;ig eingebundene Module werden <em>Basismodule</em>
304      genannt. F&#252;r den Apache sind viele Module verf&#252;gbar, die nicht
305      als Bestandteil des <a class="glossarylink" href="./glossary.html#tarball" title="siehe Glossar">Apache-HTTP-Server-Tarballs</a> ausgeliefert
306      werden. Diese werden als <em>Drittmodule</em> bezeichnet.<br />
307      Siehe: <a href="mod/">Modulverzeichnis</a>
308    </dd>
309
310    <dt><a name="modulemagicnumber" id="modulemagicnumber">Module-Magic-Number</a>
311      <span class="phonetic">[&#712;m&#596;ju&#720;l m&#230;d&#658;ik
312      &#712;n&#652;mb&#601;]</span>
313      (<a name="mmn" id="mmn">MMN</a>)</dt>
314    <dd>Die Module-Magic-Number ist eine Konstante, die im Apache-Quelltext
315      definiert ist und im Zusammenhang mit der Bin&#228;rkompatibilit&#228;t
316      von Modulen steht. Sie wird ge&#228;ndert, wenn sich interne
317      Apache-Strukturen, -Funktionen oder andere signifikante Teile der API
318      derart &#228;ndern, dass eine Bin&#228;rkompatibilit&#228;t nicht mehr
319      gew&#228;hrleistet werden kann. Bei einer MMN-&#196;nderung m&#252;ssen
320      alle Module von Drittanbietern zumindest neu kompiliert und zuweilen auch
321      geringf&#252;gig angepa&#223;t werden, um mit der neuen Apache-Version zu
322      funktionieren.
323    </dd>
324
325    <dt><a name="publickey" id="publickey">&#214;ffentlicher
326      Schl&#252;ssel</a></dt>
327    <dd>Der &#246;ffentlich verf&#252;gbare Schl&#252;ssel in einem <a class="glossarylink" href="./glossary.html#publickeycryptography" title="siehe Glossar">Public-Key-Kryptographie</a>-System,
328      mit dem f&#252;r seinen Eigent&#252;mer bestimmte Nachrichten
329      verschl&#252;sselt und Signaturen von seinem Eigent&#252;mer
330      entschl&#252;sselt werden.<br />
331      Siehe: <a href="ssl/">SSL/TLS-Verschl&#252;sselung</a>
332    </dd>

357    <dt><a name="proxy" id="proxy">Proxy</a></dt>
358    <dd>Ein zwischen dem Client und dem <em>urspr&#252;nglichen Server</em>
359      <span class="transnote">(<em>Anm.d.&#220;.:</em> der Server, den der Client tats&#228;chlich erreichen
360      m&#246;chte)</span> liegender Server. Er nimmt Anfragen von
361      Clients entgegen, &#252;bermittelt diese Anfragen dem
362      urspr&#252;nglichen Server und liefert die Antwort des
363      urspr&#252;nglichen Servers an den Client zur&#252;ck. Wenn mehrere
364      Clients den gleichen Inhalt abfragen, dann kann der Proxy diesen Inhalt
365      aus seinem Zwischenspeicher ausliefern, anstatt ihn jedesmal vom
366      urspr&#252;nglichen Server anzufordern, und dadurch die Antwortzeit
367      verringern.<br />
368      Siehe: <a href="mod/mod_proxy.html">mod_proxy</a>
369    </dd>

371    <dt><a name="publickeycryptography" id="publickeycryptography">Public-Key-Kryptographie</a>
372      <span class="phonetic">[&#712;p&#652;blik ki&#720;
373      &#712;kyptogra&#712;fi&#720;]</span></dt>
374    <dd>Theorie und Anwendung asymmetrischer Verschl&#252;sselungssysteme,
375      die einen Schl&#252;ssel zur Verschl&#252;sselung und einen anderen zur
376      Entschl&#252;sselung verwenden. Zwei derart zusammengeh&#246;rende
377      Schl&#252;ssel bilden Sch&#252;sselpaar. Man spricht auch von
378      "Asymetrischer Kryptographie".<br />
379      Siehe: <a href="ssl/">SSL/TLS-Verschl&#252;sselung</a>
380    </dd>

382    <dt><a name="regularexpresion" id="regularexpresion">Regul&#228;rer
383      Ausdruck</a> <a name="regex" id="regex">(Regex)</a></dt>
384    <dd>Eine Form, ein Muster im Text zu beschreiben - zum Beispiel: "alle
385      W&#246;rter, die mit dem Buchstaben A beginnen" oder "Jeder Satz mit
386      zwei Kommata und ohne gro&#223;es Q". Beim Apache sind regul&#228;re
387      Ausdr&#252;cke hilfreich, da sie auf sehr flexible Art und Weise die
388      Anwendung bestimmter Eigenschaften auf eine Auswahl von Dateien oder
389      Ressourcen erm&#246;glichen. - Zum Beispiel k&#246;nnen alle .gif- und
390      .jpg-Dateien eines Verzeichnis "images" mit
391      "<code>/images/.*(jpg|gif)$</code>" beschrieben werden. Der Apache
392      verwendet Perl-kompatible regul&#228;re Ausdr&#252;cke, wie sie die
393      <a href="http://www.pcre.org/">PCRE</a>-Bibliothek bereitstellt.
394    </dd>
