48<h2><a name="main" id="main">Hauptkonfigurationsdateien</a></h2>
49    
50    <table class="related"><tr><th>Referenzierte Module</th><th>Referenzierte Direktiven</th></tr><tr><td><ul><li><code class="module"><a href="./mod/mod_mime.html">mod_mime</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/core.html#ifdefine">&lt;IfDefine&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#include">Include</a></code></li><li><code class="directive"><a href="./mod/mod_mime.html#typesconfig">TypesConfig</a></code></li></ul></td></tr></table>
51
52    <p>Der Apache wird konfiguriert, indem <a href="mod/directives.html">Direktiven</a> in einfache Textdateien
53      eingetragen werden. Die Hauptkonfigurationsdatei hei&#223;t
54      &#252;blicherweise <code>httpd.conf</code>. Der Ablageort dieser Datei
55      wird bei der Kompilierung festgelegt, kann jedoch mit der
56      Befehlszeilenoption <code>-f</code> &#252;berschrieben werden. Durch
57      Verwendung der Direktive  <code class="directive"><a href="./mod/core.html#include">Include</a></code>
58      k&#246;nnen au&#223;erdem weitere Konfigurationsdateien hinzugef&#252;gt
59      werden. Zum Einf&#252;gen von mehreren Konfigurationsdateien k&#246;nnen
60      Platzhalter verwendet werden. Jede Direktive darf in jeder dieser
61      Konfigurationsdateien angegeben werden. &#196;nderungen in den
62      Hauptkonfigurationsdateien werden vom Apache nur beim Start oder Neustart
63      erkannt.</p>
64
65    <p>Der Server liest auch eine Datei mit MIME-Dokumenttypen ein. Der
66      Name dieser Datei wird durch die Direktive <code class="directive"><a href="./mod/mod_mime.html#typesconfig">TypesConfig</a></code> bestimmt. Die Voreinstellung
67      ist <code>mime.types</code>.</p>

70<h2><a name="syntax" id="syntax">Syntax der Konfigurationsdateien</a></h2>
71    
72
73    <p>Die Konfigurationsdateien des Apache enthalten eine Direktive pro Zeile.
74      Der Backslash "\" l&#228;&#223;t sich als letztes Zeichen in einer Zeile
75      dazu verwenden, die Fortsetzung der Direktive in der n&#228;chsten Zeile
76      anzuzeigen. Es darf kein weiteres Zeichen oder Whitespace zwischen dem
77      Backslash und dem Zeilenende folgen.</p>
78
79    <p>In den Konfigurationsdateien wird bei den Direktiven nicht zwischen
80      Gro&#223;- und Kleinschreibung unterschieden. Bei den Argumenten der
81      Direktiven wird dagegen oftmals zwischen Gro&#223;- und Kleinschreibung
82      differenziert. Zeilen, die mit dem Doppelkreuz "#" beginnen, werden als
83      Kommentare betrachtet und ignoriert. Kommentare d&#252;rfen
84      <strong>nicht</strong> am Ende einer Zeile nach der Direktive
85      eingef&#252;gt werden. Leerzeilen und Whitespaces vor einer Direktive
86      werden ignoriert. Dadurch lassen sich Direktiven zur besseren Lesbarbeit
87      einr&#252;cken.</p>
88
89    <p>Sie k&#246;nnen die Syntax Ihrer Konfigurationsdateien auf Fehler
90      pr&#252;fen, ohne den Server zu starten, indem Sie <code>apachectl
91        configtest</code> oder die Befehlszeilenoption <code>-t</code>
92      verwenden.</p>

95<h2><a name="modules" id="modules">Module</a></h2>
96    
97
98    <table class="related"><tr><th>Referenzierte Module</th><th>Referenzierte Direktiven</th></tr><tr><td><ul><li><code class="module"><a href="./mod/mod_so.html">mod_so</a></code></li></ul></td><td><ul><li><code class="directive"><a href="./mod/core.html#ifmodule">&lt;IfModule&gt;</a></code></li><li><code class="directive"><a href="./mod/mod_so.html#loadmodule">LoadModule</a></code></li></ul></td></tr></table>
99
100    <p>Der Apache ist ein modularer Server. Das bedeutet, dass nur die abolute
101      Grundfunktionalit&#228;t im Kernserver enthalten ist. Weitergehende
102      F&#228;higkeiten sind mittels <a href="mod/">Modulen</a> verf&#252;gbar,
103      die in den Apache geladen werden k&#246;nnen. Standardm&#228;&#223;ig
104      wird bei der Kompilierung ein Satz von Basismodulen <span class="transnote">(<em>Anm.d.&#220;.:</em> die so
105        genannten <a href="mod/module-dict.html#Status">Base</a>-Module)</span> in den Server eingebunden. Wenn der
106      Server f&#252;r die Verwendung von <a href="dso.html">dynamisch
107        ladbaren</a> Modulen kompiliert wurde, dann k&#246;nnen Module separat
108      kompiliert und jederzeit mittels der Direktive <code class="directive"><a href="./mod/mod_so.html#loadmodule">LoadModule</a></code> hinzugef&#252;gt werden.
109      Andernfalls muss der Apache neu kompiliert werden, um Module
110      hinzuzuf&#252;gen oder zu entfernen. Konfigurationsanweisungen k&#246;nnen
111      abh&#228;ngig vom Vorhandensein eines bestimmten Moduls eingesetzt werden,
112      indem sie in einen <code class="directive"><a href="./mod/core.html#ifmodule">&lt;IfModule&gt;</a></code>-Block eingeschlossen werden.</p>
113
114    <p>Um zu sehen, welche Module momentan in den Server einkompiliert sind,
115      kann die Befehlszeilenoption <code>-l</code> verwendet werden.</p>

118<h2><a name="scope" id="scope">Der G&#252;ltigkeitsbereich von Direktiven</a></h2>
119    
120
121    <table class="related"><tr><th>Referenzierte Module</th><th>Referenzierte Direktiven</th></tr><tr><td /><td><ul><li><code class="directive"><a href="./mod/core.html#directory">&lt;Directory&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#directorymatch">&lt;DirectoryMatch&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#files">&lt;Files&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#filesmatch">&lt;FilesMatch&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#location">&lt;Location&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#locationmatch">&lt;LocationMatch&gt;</a></code></li><li><code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code></li></ul></td></tr></table>
122
123    <p>Direktiven in den Hauptkonfigurationsdateien gelten f&#252;r den
124      gesamten Server. Wenn Sie die Konfiguration nur f&#252;r einen Teil des
125      Servers ver&#228;ndern m&#246;chten, k&#246;nnen Sie den
126      G&#252;ltigkeitsbereich der Direktiven beschr&#228;nken, indem Sie diese
127      in <code class="directive"><a href="./mod/core.html#directory">&lt;Directory&gt;</a></code>-,
128      <code class="directive"><a href="./mod/core.html#directorymatch">&lt;DirectoryMatch&gt;</a></code>-,
129      <code class="directive"><a href="./mod/core.html#files">&lt;Files&gt;</a></code>-,
130      <code class="directive"><a href="./mod/core.html#filesmatch">&lt;FilesMatch&gt;</a></code>-,
131      <code class="directive"><a href="./mod/core.html#location">&lt;Location&gt;</a></code>- oder
132      <code class="directive"><a href="./mod/core.html#locationmatch">&lt;LocationMatch&gt;</a></code>-Abschnitte eingef&#252;gen.
133      Diese Abschnitte begrenzen die Anwendung der umschlossenen Direktiven
134      auf bestimmte Pfade des Dateisystems oder auf
135      bestimmte URLs. Sie k&#246;nnen f&#252;r eine fein abgestimmte
136      Konfiguration auch ineinander verschachtelt werden.</p>
137    
138    
139    <p>Der Apache besitzt die F&#228;higkeit, mehrere verschiedene Websites
140      gleichzeitig zu bedienen. Dies wird <a href="vhosts/">virtuelles
141        Hosten</a> genannt. Direktiven k&#246;nnen auch in ihrem
142      G&#252;ltigkeitsgereich eingeschr&#228;nkt werden, indem sie innerhalb
143      eines <code class="directive"><a href="./mod/core.html#virtualhost">&lt;VirtualHost&gt;</a></code>-Abschnittes angegeben werden.
144      Sie werden dann nur auf Anfragen f&#252;r eine bestimmte Website
145      angewendet.</p>
146
147    <p>Obwohl die meisten Direktiven in jedem dieser Abschnitte platziert
148      werden k&#246;nnen, ergeben einige Direktiven in manchen Kontexten
149      keinen Sinn. Direktiven zur Prozesssteuerung beispielsweise
150      d&#252;rfen nur im Kontext des Hauptservers angegeben werden. Pr&#252;fen
151      Sie den <a href="mod/directive-dict.html#Context">Kontext</a> der
152      Direktive, um herauszufinden, welche Direktiven in welche Abschnitte
153      eingef&#252;gt werden k&#246;nnen.  Weitere Informationen finden Sie unter
154      "<a href="sections.html">Wie Directory-, Location- und Files-Abschnitte
155        arbeiten</a>".</p>

159<h2><a name="htaccess" id="htaccess">.htaccess-Dateien</a></h2>
160    
161
162    <table class="related"><tr><th>Referenzierte Module</th><th>Referenzierte Direktiven</th></tr><tr><td /><td><ul><li><code class="directive"><a href="./mod/core.html#accessfilename">AccessFileName</a></code></li><li><code class="directive"><a href="./mod/core.html#allowoverride">AllowOverride</a></code></li></ul></td></tr></table>
163
164    <p>Der Apache erm&#246;glicht die dezentrale Verwaltung der
165      Konfiguration mittes spezieller Dateien innerhalb des
166      Web-Verzeichnisbaums. Diese speziellen Dateien hei&#223;en
167      gew&#246;hnlich <code>.htaccess</code>, mit der Direktive <code class="directive"><a href="./mod/core.html#accessfilename">AccessFileName</a></code> kann jedoch auch ein anderer
168      Name festgelegt werden. In <code>.htaccess</code>-Dateien angegebene
169      Direktiven werden auf das Verzeichnis und dessen Unterverzeichnisse
170      angewendet, in dem die Datei abgelegt ist. <code>.htaccess</code>-Dateien
171      folgen der gleichen Syntax wie die Hauptkonfigurationsdateien. Da
172      <code>.htaccess</code>-Dateien bei jeder Anfrage eingelesen werden,
173      werden &#196;nderungen in diesen Dateien sofort wirksam.</p>
174
175    <p>Pr&#252;fen Sie den <a href="mod/directive-dict.html#Context">Kontext</a> der Direktive, um
176      herauszufinden, welche Direktiven in <code>.htaccess</code>-Dateien
177      angegeben werden k&#246;nnen. Dar&#252;ber hinaus steuert der
178      Serveradministrator mit der Einstellung der Direktive <code class="directive"><a href="./mod/core.html#allowoverride">AllowOverride</a></code> in den
179      Hauptkonfigurationsdateien welche Direktiven in
180      <code>.htaccess</code>-Dateien verwendet werden d&#252;rfen.</p>
181
182    <p>Weitere Informationen &#252;ber <code>.htaccess</code>-Dateien finden
183      Sie in der <a href="howto/htaccess.html">.htaccess-Einf&#252;hrung</a>.</p>
