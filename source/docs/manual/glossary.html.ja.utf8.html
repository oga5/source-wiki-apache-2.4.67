45<h2><a name="definitions" id="definitions">定義</a></h2>
46
47  <dl>
48    <dt><a name="accesscontrol" id="accesscontrol">アクセス制御</a></dt>
49    <dd>ネットワーク認可領域へのアクセスを制限します。Apache においては、
50    普通はアクセスの制限は <em>URL</em> に対するものとなります。<br />
51    参照: <a href="howto/auth.html">認証、承認、アクセス制御</a>
52    </dd>
53    
54    <dt><a name="algorithm" id="algorithm">アルゴリズム</a></dt>
55    <dd>有限回のステップで問題を解くためのあいまいでない式もしくは規則の
56    集合。暗号のためのアルゴリズムは通常 <dfn>Cipher</dfn> と呼ばれます。
57    </dd>
58    
59    <dt><a name="apacheextensiontool" id="apacheextensiontool">APache
60        eXtension Tool</a> <a name="apxs" id="apxs">(apxs)</a></dt>
61    <dd><a class="glossarylink" href="./glossary.html#module" title="用語集を参照">モジュール</a> ソースを
62    動的共有オブジェクト (<a class="glossarylink" href="./glossary.html#dso" title="用語集を参照">DSO</a>) にコンパイルし、
63    Apache Web サーバにインストールする手助けをする perl スクリプト。<br />
64      参照: マニュアルページ: <code class="program"><a href="./programs/apxs.html">apxs</a></code>
65    </dd>
66
67    <dt><a name="apacheportableruntime" id="apacheportableruntime">Apache Portable Runtime</a> <a name="apr" id="apr">(APR)</a></dt>
68    <dd>サーバ<span class="transnote">(<em>訳注:</em> Apache HTTP Server)</span>と OS の
69      間の基本的なインターフェースの多くを提供する<span class="transnote">(<em>訳注:</em> 
70      OS の差を吸収する)</span>ライブラリのセット。 APR は 独立した
71      プロジェクトとして Apache HTTP Server と平行して開発が行われています。<br />
72      参照: <a href="http://apr.apache.org/">Apache Portable Runtime
73      プロジェクト</a>
74    </dd>
75
76    <dt><a name="authentication" id="authentication">認証</a></dt>
77    <dd>サーバ、クライアント、ユーザといったネットワークエンティティの
78    身元の特定。<br />
79      参照: <a href="howto/auth.html">認証、承認、アクセス制御</a>
80    </dd>
81
82    <dt><a name="certificate" id="certificate">証明書</a></dt>
83    <dd>サーバやクライアントといったネットワークエンティティを認証するのに
84    使用されるデータレコード。証明書には (subject と呼ばれる) 所有者と、
85    (issuer と呼ばれる) <a class="glossarylink" href="./glossary.html#certificationauthority" title="用語集を参照">認証局</a> の署名、所有者の
86    <a class="glossarylink" href="./glossary.html#publickey" title="用語集を参照">公開鍵</a> と、CA による署名という
87    X.509 の情報が含まれます。ネットワークエンティティはそれらの署名を
88    CA 証明書を使って検証します。<br />
89      参照: <a href="ssl/">SSL/TLS 暗号化</a>
90    </dd>
91
92    <dt><a name="certificatsigningrequest" id="certificatsigningrequest">証明書署名リクエスト</a>
93      <span class="transnote">(<em>訳注:</em> Certificate Signing Request)</span>
94      <a name="csr" id="csr">(CSR)</a></dt>
95    <dd><a class="glossarylink" href="./glossary.html#certificationauthority" title="用語集を参照">認証局</a> に提出
96    する未署名の <a class="glossarylink" href="./glossary.html#certificate" title="用語集を参照">証明書</a>。
97    認証局は CA <em>証明書</em> の <a class="glossarylink" href="./glossary.html#privatekey" title="用語集を参照">秘密鍵</a> で署名します。
98    一旦 CSR に署名がなされると、それは本物の証明書になります。<br />
99      参照: <a href="ssl/">SSL/TLS 暗号化</a>
100    </dd>
101
102    <dt><a name="certificationauthority" id="certificationauthority">証明局</a>
103      <span class="transnote">(<em>訳注:</em> Certification Authority)</span>
104      <a name="ca" id="ca">(CA)</a></dt>
105    <dd>安全な方法で認証を行なったネットワークエンティティの証明書を
106    署名するための信頼できる第三者機関。他のネットワークエンティティは
107    証明書の保持者が CA に認証されたかを署名を検証することで調べることが
108    できます。<br />
109      参照: <a href="ssl/">SSL/TLS 暗号化</a>
110    </dd>
111
112    <dt><a name="cipher" id="cipher">Cipher</a></dt>
113    <dd>データ暗号化のためのアルゴリズム。例えば DES, IDEA, RC4 など。<br />
114      参照: <a href="ssl/">SSL/TLS 暗号化</a>
115    </dd>
116
117    <dt><a name="ciphertext" id="ciphertext">暗号文</a></dt>
118    <dd><a class="glossarylink" href="./glossary.html#plaintext" title="用語集を参照">平文</a> が
119    <a class="glossarylink" href="./glossary.html#cipher" title="用語集を参照">Cipher</a> をかけられた結果。<br />
120    参照: <a href="ssl/">SSL/TLS 暗号化</a>
121    </dd>
122
123    <dt><a name="commongatewayinterface" id="commongatewayinterface">Common
124        Gateway Interface</a> <a name="cgi" id="cgi">(CGI)</a></dt>
125    <dd>外部プログラムがリクエストを扱うことができるようにするための
126    ウェブサーバと外部プログラム間のインタフェースの標準仕様。
127    インタフェースは元々 <a href="http://hoohoo.ncsa.uiuc.edu/cgi/overview.html">NCSA</a> により定義
128    されていましたが
129    <a href="http://cgi-spec.golux.com/">RFC プロジェクト</a> も存在します。<br />
130      参照: <a href="howto/cgi.html">CGI による動的コンテンツ</a>
131    </dd>
132
133    <dt><a name="configurationdirective" id="configurationdirective">設定ディレクティブ</a></dt>
134    <dd>参照: <a class="glossarylink" href="./glossary.html#directive" title="用語集を参照">ディレクティブ</a></dd>
135
136    <dt><a name="configurationfile" id="configurationfile">設定ファイル</a></dt>
137    <dd>Apache の設定を制御する <a class="glossarylink" href="./glossary.html#directive" title="用語集を参照">ディレクティブ</a>
138    の書かれたテキストファイル。<br />
139      参照: <a href="configuring.html">設定ファイル</a>
140    </dd>
141
142    <dt><a name="connect" id="connect">CONNECT</a></dt>
143    <dd>データチャネルをそのまま HTTP 上でプロキシするための HTTP
144    <a class="glossarylink" href="./glossary.html#method" title="用語集を参照">メソッド</a>。SSL のような他の
145    プロトコルをくるむために使うことができます。
146    </dd>
147    
148    <dt><a name="context" id="context">コンテキスト</a></dt>
149    <dd><a class="glossarylink" href="./glossary.html#configurationfile" title="用語集を参照">設定ファイル</a> 中で、
150    特定の種類の <a class="glossarylink" href="./glossary.html#directive" title="用語集を参照">ディレクティブ</a> が許可されている場所。<br />
151      参照: <a href="mod/directive-dict.html#Context">Apache のディレクティブの
152    説明に使われている用語</a>
153    </dd>
154
155    <dt><a name="digitalsignature" id="digitalsignature">デジタル署名</a></dt>
156    <dd>証明書や他のファイルを検証するための暗号化されたテキストブロック。
157    <a class="glossarylink" href="./glossary.html#certificationauthority" title="用語集を参照">認証局</a> は
158    <em>証明書</em> に埋め込まれた <em>公開鍵</em> のハッシュを作成し、
159    それを自身の <em>秘密鍵</em> で暗号化することで署名を作成します。
160    CA の公開鍵でのみその署名を復号することができますので、それにより
161    <em>証明書</em> を保有するネットワークエンティティを CA が認証した
162    ことを検証できます。<br />
163    参照: <a href="ssl/">SSL/TLS 暗号化</a>
164    </dd>
165    
166    <dt><a name="directive" id="directive">ディレクティブ</a></dt>
167    <dd>Apache のいろいろな振る舞いを制御する設定コマンド。ディレクティブは
168    <a class="glossarylink" href="./glossary.html#configurationfile" title="用語集を参照">設定ファイル</a> に
169    書かれます。<br />
170    参照: <a href="mod/directives.html">ディレクティブ索引</a>
171    </dd>
172
173    <dt><a name="dynamicsharedobject" id="dynamicsharedobject">動的
174    共有オブジェクト</a> <span class="transnote">(<em>訳注:</em> Dynamic
175        Shared Object)</span> <a name="dso" id="dso">(DSO)</a></dt>
176    <dd>必要に応じて読み込むことが可能な、Apache <code class="program"><a href="./programs/httpd.html">httpd</a></code> とは
177      別にコンパイルされた <a class="glossarylink" href="./glossary.html#module" title="用語集を参照">モジュール</a><br />
178      参照: <a href="dso.html">動的共有オブジェクトサポート</a>
179    </dd>
180
181    <dt><a name="environmentvariable" id="environmentvariable">環境変数</a>
182    <a name="env-variable" id="env-variable">(env-variable)</a></dt>
183    <dd>情報を保管したり、プログラム間の通信をするために使われる、
184    オペレーティングシステムのシェルにより管理されている名前付きの変数。
185    Apache も環境変数と呼ばれる内部変数を持っていますが、こちらは
186    シェル環境ではなく、Apache の内部構造体に保持されています。<br />
187      参照: <a href="env.html">Apache の環境変数</a>
188    </dd>
189
190    <dt><a name="export-crippled" id="export-crippled">輸出強度削減</a>
191    <span class="transnote">(<em>訳注:</em> Export-Crippled)</span></dt>
192    <dd>アメリカの Export Administration Regulations (EAR)
193    <span class="transnote">(<em>訳注:</em> 輸出管理規則)</span> に従うために暗号の強度 (とセキュリティ)
194    を削減すること。輸出強度削減された暗号ソフトウェアは小さいキーに
195    制限され、通常総当たり攻撃で復号できてしまう <em>暗号文</em> を生成する
196    ことになります。<br />
197      参照: <a href="ssl/">SSL/TLS 暗号化</a>
198    </dd>
199
200    <dt><a name="filter" id="filter">フィルタ</a></dt>
201    <dd>サーバから送られるデータとサーバが受け取るデータに適用される処理。
202    入力フィルタはクライアントからサーバに送られたデータを処理し、
203    出力フィルタはサーバにある文書をクライアントに送る前に処理します。
204    例えば、<code>INCLUDES</code> 出力フィルタは
205    <a class="glossarylink" href="./glossary.html#ssi" title="用語集を参照">Server Side Includes</a> の文書を
206    処理します。<br />
207      参照: <a href="filter.html">フィルタ</a>
208    </dd>
209
210    <dt><a name="fully-qualifieddomain-name" id="fully-qualifieddomain-name">完全修飾ドメイン名</a>
211      <span class="transnote">(<em>訳注:</em> Fully-Qualified Domain-Name)</span>
212      <a name="fqdn" id="fqdn">(FQDN)</a></dt>
213    <dd>IP アドレスに解決できるホスト名と、ドメイン名からなるネットワーク
214    エンティティの一意な名前。例えば、<code>www</code> はホスト名で、
215    <code>example.com</code> はドメイン名なので、
216    <code>www.example.com</code> は完全修飾ドメイン名になります。</dd> 
217    
218    <dt><a name="handler" id="handler">ハンドラ</a></dt>
219    <dd>ファイルが呼ばれたときに行なわれる動作の Apache の内部での表現。
220    一般にファイルにはファイルの種類に応じて暗黙のハンドラが設定されて
221    います。普通はすべてのファイルがサーバにより送られますが、別に
222    扱われる <span class="transnote">(<em>訳注:</em> handle)</span> ファイルの種類も存在します。
223    例えば <code>cgi-script</code> はファイルが
224    <a class="glossarylink" href="./glossary.html#cgi" title="用語集を参照">CGI</a> として処理されるように指定します。<br />
225      参照: <a href="handler.html">Apache のハンドラの使用</a>
226    </dd>
227
228    <dt><a name="hash" id="hash">ハッシュ</a></dt>
229    <dd>任意の文字列から固定長の文字列を生成する、数学的な一方向で不可逆な
230    アルゴリズム。異なった入力文字列からは普通は違うハッシュが生成されます
231    (ハッシュ関数に依存します)。</dd>
232
233    <dt><a name="header" id="header">ヘッダ</a></dt>
234    <dd>実際のコンテンツの前に送られ、コンテンツを説明するメタ情報の
235    入った <a class="glossarylink" href="./glossary.html#http" title="用語集を参照">HTTP</a> リクエストと応答の一部分。</dd> 
236    
237    <dt><a name="htaccess" id="htaccess">.htaccess</a></dt> 
238    <dd>ウェブツリーに置かれて、そのディレクトリとサブディレクトリに
239    <a class="glossarylink" href="./glossary.html#directive" title="用語集を参照">ディレクティブ</a> を適用する
240    <a class="glossarylink" href="./glossary.html#configurationfile" title="用語集を参照">設定ファイル</a>。
241    名前とは裏腹に、このファイルにはアクセス制御ディレクティブだけでなく、
242    ほとんどどんな種類のディレクティブでも書くことができます。<br />
243      参照: <a href="configuring.html">設定ファイル</a>
244    </dd>
245
246    <dt><a name="httpd.conf" id="httpd.conf">httpd.conf</a></dt>
247    <dd>メインの Apache <a class="glossarylink" href="./glossary.html#configurationfile" title="用語集を参照">設定
248    ファイル</a>。デフォルトの場所は
249    <code>/usr/local/apache2/conf/httpd.conf</code>
250    ですが、実行時やコンパイル時の設定により違う場所に移動されて
251    いるかもしれません。<br />
252      参照: <a href="configuring.html">設定ファイル</a>
253    </dd>
254
255    <dt><a name="hypertexttransferprotocol" id="hypertexttransferprotocol">HyperText Transfer Protocol</a>
256      <a name="http" id="hhtp">(HTTP)</a></dt>
257    <dd>World Wide Web で使われる標準の転送プロトコル。Apache
258    は HTTP/1.1 と呼ばれ、<a href="http://ietf.org/rfc/rfc2616.txt">RFC 2616</a>
259    で定義されているプロトコルのバージョン 1.1 を実装しています。</dd>
260
261    <dt><a name="https" id="https">HTTPS</a></dt>
262    <dd>The HyperText Transfer Protocol (Secure),
263    World Wide Web での暗号化された標準の通信機構。これは実際は
264    単に <a class="glossarylink" href="./glossary.html#ssl" title="用語集を参照">SSL</a> 上での HTTP です。<br />
265      参照: <a href="ssl/">SSL/TLS 暗号化</a>
266    </dd> 
267
268    <dt><a name="method" id="method">メソッド</a></dt>
269    <dd><a class="glossarylink" href="./glossary.html#http" title="用語集を参照">HTTP</a> の文脈では、
270    クライアントから指定されたリクエスト行に対応するリソース
271    に対して行なう動作。HTTP では <code>GET</code>, <code>POST</code>,
272    <code>PUT</code> といったようなメソッドがあります。</dd> 
273    
274    <dt><a name="messagedigest" id="messagedigest">メッセージダイジェスト</a></dt>
275    <dd>メッセージのハッシュで、メッセージの内容が転送時に変更されていないことの検証に
276    使える。<br />
277      参照: <a href="ssl/">SSL/TLS 暗号化</a>
278    </dd>
279
280    <dt><a name="mime-type" id="mime-type">MIME タイプ</a></dt>
281    <dd>送信されているドキュメントの種類を表すための方法。
282    この名前はフォーマットが Multipurpose Internet Mail Extensions から
283    借りてこられたことによります。これはスラッシュで分離された、
284    主タイプと副タイプからなります。例えば、<code>text/html</code>,
285    <code>image/gif</code>,  <code>application/octet-stream</code> など
286    があります。HTTP では、MIME タイプは <code>Content-Type</code>
287    <a class="glossarylink" href="./glossary.html#header" title="用語集を参照">ヘッダ</a> で送信されます。<br />
288    参照: <a href="mod/mod_mime.html">mod_mime</a>
289    </dd>
290
291    <dt><a name="module" id="module">モジュール</a></dt>
292    <dd>プログラムの独立した一部分。Apache の機能の多くは使用するかしないかを
293    選択できるモジュールの中にあります。Apache <code class="program"><a href="./programs/httpd.html">httpd</a></code>
294    に組み込まれているモジュールは<dfn>静的モジュール</dfn>と呼ばれ、
295    別に保存され、実行時に読み込むことのできるモジュールは
296    <dfn>動的モジュール</dfn> もしくは <a class="glossarylink" href="./glossary.html#dso" title="用語集を参照">DSO</a> と
297    呼ばれます。デフォルトで含まれているモジュールは<dfn>base モジュール</dfn>
298    と呼ばれます。Apache HTTP サーバの <a class="glossarylink" href="./glossary.html#tarball" title="用語集を参照">tarball</a>
299    の一部としては配られていない Apache 用のモジュールがあります。
300    それらは <dfn>サードパーティモジュール</dfn> と呼ばれます。<br />
301      参照: <a href="mod/">モジュール索引</a>
302    </dd>
303
304    <dt><a name="modulemagicnumber" id="modulemagicnumber">Module Magic
305      Number</a> (<a name="mmn" id="mmn">MMN</a>)</dt>
306    <dd>Apache ソースコードで定義されている、モジュールのバイナリ互換性に
307    関する定数。バイナリ互換性が保てなくなるような Apache 内部の構造体や、
308    関数呼び出し、その他の API の重要な部分の変更があったときに変更されます。
309    MMN が変わると、すべてのサードパーティモジュールは少なくとも再コンパイルを
310    する必要があり、場合によっては新しいバージョンの Apache で動作するために
311    少し変更する必要さえあるかもしれません。</dd>
312
313    <dt><a name="openssl" id="openssl">OpenSSL</a></dt>
314    <dd>SSL/TLS 用のオープンソースツールキット<br />
315      参照 <a href="http://www.openssl.org/">http://www.openssl.org/</a>#
316    </dd>
317
318    <dt><a name="passphrase" id="passphrase">パスフレーズ</a></dt>
319    <dd>秘密鍵のファイルを保護するための語句。権限の無いユーザが
320    暗号化するのを防ぎます。通常は単に <a class="glossarylink" href="./glossary.html#cipher" title="用語集を参照">Cipher</a> の秘密の暗号用と復号用のキーです。<br />
321      参照: <a href="ssl/">SSL/TLS 暗号化</a>
322    </dd>
323
324    <dt><a name="plaintext" id="plaintext">平文</a></dt>
325    <dd>暗号化されていないテキスト。</dd>
326
327    <dt><a name="privatekey" id="privatekey">秘密鍵</a></dt>
328    <dd>受け取るメッセージの復号と送出するメッセージの署名に使われる、
329    <a class="glossarylink" href="./glossary.html#publickeycryptography" title="用語集を参照">公開鍵暗号</a> の
330    秘密鍵。<br />
331      参照: <a href="ssl/">SSL/TLS 暗号化</a>
332    </dd>
333    
334    <dt><a name="proxy" id="proxy">プロキシ</a></dt>
335    <dd>クライアントと <em>オリジンのサーバ</em> の間に存在する中間サーバ。
336    クライアントからのリクエストを受け取り、オリジンのサーバに送信して、オリジンの
337    サーバからの応答をクライアントに返します。複数のクライアントが同じ
338    コンテンツを要求する場合は、毎回元のサーバにリクエストを送る代わり
339    プロキシはキャッシュからコンテンツを送り、応答時間を短縮することが
340    できます。<br />
341      参照: <a href="mod/mod_proxy.html">mod_proxy</a>
342    </dd>
343
344    <dt><a name="publickey" id="publickey">公開鍵</a></dt>
345    <dd>所有者に向けられたメッセージの暗号化と所有者による署名の復号に使われる、
346    <a class="glossarylink" href="./glossary.html#publickeycryptography" title="用語集を参照">公開鍵暗号</a>システムに
347    おける公けにされている鍵。<br />
348      参照: <a href="ssl/">SSL/TLS 暗号化</a>
349    </dd>
350
351    <dt><a name="publickeycryptography" id="publickeycryptography">公開鍵暗号</a></dt>
352    <dd>ある鍵を暗号に使い、別の鍵を復号に使う非対称暗号システムについての研究や
353    その応用を指す。対応する鍵はキーペアと呼ばれます。非対称暗号とも呼ばれます。<br />
354      参照: <a href="ssl/">SSL/TLS 暗号化</a>
355    </dd> 
356
357    <dt><a name="regularexpresion" id="regularexpresion">正規表現</a>
358      <a name="regex" id="regex">(Regex)</a></dt>
359    <dd>テキストのパターンを表現する方式の一つ。例えば、
360    「A で始まるすべての単語」や「すべての 10 桁の電話番号」や、
361    「コンマが二つあり、大文字の Q がないすべての文」というのでさえ表現
362    できます。
363    正規表現は Apache においても便利なもので、ファイルやリソースの集まりに対して
364    何らかの属性を適用することがとても柔軟にできます。例えば、
365    すべての "images" ディレクトリの下の、すべての .gif と .jpg ファイル
366    は <code>/images/.*(jpg|gif)$</code> と書くことができます。
367    Apache では <a href="http://www.pcre.org/">PCRE</a> ライブラリが提供する
368    Perl 互換正規表現 <span class="transnote">(<em>訳注:</em> Perl Compatible Regular Expressions)</span>
369    を使います。</dd> 
370
371    <dt><a name="reverseproxy" id="reverseproxy">リバースプロキシ</a></dt>
372    <dd>クライアントには <em>オリジンのサーバ</em> のように見える
373    <a class="glossarylink" href="./glossary.html#proxy" title="用語集を参照">プロキシ</a> サーバ。セキュリティの
374    ためや、負荷分散のためにクライアントからオリジンのサーバを隠したいときに
375    便利です。</dd>
376  
377    <dt><a name="securesocketslayer" id="securesocketslayer">Secure Sockets
378      Layer</a> <a name="ssl" id="ssl">(SSL)</a></dt>
379    <dd>Netscape Communications Corporation により
380    TCP/IP ネットワーク上で一般の通信の認証と暗号用に作られたプロトコル。
381    最もよく使われているものは <em>HTTPS</em> つまり SSL 上での
382    HyperText Transfer Protocol (HTTP) です。<br />
383    参照: <a href="ssl/">SSL/TLS 暗号化</a>
384    </dd> 
385
386    <dt><a name="serversideincludes" id="serversideincludes">Server Side
387      Includes</a> <a name="ssi" id="ssi">(SSI)</a></dt>
388    <dd>HTML ファイル中に処理ディレクティブを埋め込む技術の一つ。<br />
389    参照: <a href="howto/ssi.html">Server Side Includes 入門</a>
390    </dd>
391  
392    <dt><a name="session" id="session">セッション</a></dt>
393    <dd>一般的な通信における文脈情報。</dd>
394
395    <dt><a name="ssleay" id="ssleay">SSLeay</a></dt>
396    <dd>Eric A. Young 氏による SSL/TLS を実装した元々のライブラリ。</dd>
397
398    <dt><a name="symmetriccryptophraphy" id="symmetriccryptophraphy">対称暗号</a>
399    </dt>
400    <dd>一つの秘密鍵を暗号化と復号の両方に使う <em>Cipher</em> の
401    研究や応用を指す。</dd>
402
403    <dt><a name="tarball" id="tarball">Tarball</a></dt>
404    <dd><code>tar</code> ユーティリティを使ってまとめられたファイルのパッケージ。
405      Apache 配布は圧縮された tar アーカイブか pkzip で保管されています。</dd> 
406
407    <dt><a name="transportlayersecurity" id="transportlayersecurity">Transport
408        Layer Security</a> <a name="tls" id="tls">(TLS)</a></dt>
409    <dd>TCP/IP ネットワーク上での一般通信の認証と暗号化用に
410      Internet Engineering Task Force (IETF) により作成された SSL の
411      後継プロトコル。TLS バージョン 1 は SSL バージョン 3 とほぼ同じです。<br />
412        参照: <a href="ssl/">SSL/TLS 暗号化</a>
413    </dd>
414
415    <dt><a name="uniformresourcelocator" id="uniformresourcelocator">Uniform
416        Resource Locator</a> <a name="url" id="url">(URL)</a></dt>
417    <dd>Internet のリソースの名前、もしくはアドレス。これは正式には
418      <a class="glossarylink" href="./glossary.html#uniformresourceidentifier" title="用語集を参照">Uniform Resource Identifier</a>
419      と呼ばれるもののよく使われる非公式な名前です。URL は普通は、
420      <code>http</code> や <code>https</code> といったスキームとホスト名、
421      パスからなります。このページの URL はおそらく
422      <code>http://httpd.apache.org/docs/2.4/glossary.html</code>
423      と思われます。
424    </dd> 
425
426    <dt><a name="uniformresourceidentifier" id="uniformresourceidentifier">Uniform Resource Identifier</a>
427      <a name="URI" id="URI">(URI)</a></dt>
428    <dd>抽象的なリソースや物理リソースを同定するためのコンパクトな文字列。
429      正式には <a href="http://www.ietf.org/rfc/rfc2396.txt">RFC 2396</a> で
430      定義されています。WWW で使われている URI  は通常
431      <a class="glossarylink" href="./glossary.html#url" title="用語集を参照">URL</a> と呼ばれます。
432    </dd> 
433
434    <dt><a name="virtualhosting" id="virtualhosting">バーチャルホスト</a></dt>
435    <dd>一つの Apache を使って複数のウェブサイトを扱うこと。
436      <em>IP バーチャルホスト</em> は IP アドレスを使ってウェブサイトを
437      区別します。また <em>名前ベースのバーチャルホスト</em> は
438      ホストの名前だけを使って区別するので、同じ IP アドレス上での多くのサイトを
439      ホストできます。<br />
440      参照: <a href="vhosts/">Apache バーチャルホストの文書</a>
441    </dd>
442
443    <dt><a name="x.509" id="x.509">X.509</a></dt>
444    <dd>SSL/TLS 認証に使われている International
445      Telecommunication Union (ITU-T) により推奨されている認証証明書の形式。<br />
446      参照: <a href="ssl/">SSL/TLS 暗号化</a>
447    </dd>
