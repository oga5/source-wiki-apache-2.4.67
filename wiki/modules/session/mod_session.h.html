modules/session/mod_session.h

概要

Apache HTTP Server のセッション管理機能における中心的なヘッダーファイルです。セッションデータの構造定義、設定用構造体、およびセッションのライフサイクルを制御するためのフックインターフェースを定義しています。

モジュール内での役割

mod_session サブシステムの基盤となるインターフェースを提供します。このヘッダーは、セッションの保存先（DBD、Cookie、暗号化など）を実装する各サブモジュールが共通して準拠すべき規約を規定しています。

責務

  • セッションデータ（session_rec）のデータ構造の定義。
  • ディレクトリ単位の設定（session_dir_conf）の管理。
  • セッションのロード、保存、エンコード、デコードを行うためのフック（APR_HOOK）の宣言。
  • 他モジュールからセッションを操作するためのオプション関数の公開。

重要なシンボル

  • session_rec: セッションのキー・値ペア、有効期限、UUID、および状態フラグ（dirty, cached等）を保持する主要構造体。
  • session_dir_conf: mod_session のディレクティブ（有効化、有効期限、パス制限など）を保持する設定構造体。
  • session_load / session_save: セッションの読み込みと永続化を担うフック。
  • session_encode / session_decode: セッションデータのシリアライズ・デシリアライズを担うフック。

制御フローとデータフロー

本ファイル自体は宣言のみですが、以下のフローを規定しています： 1. **ロード**: session_load フックにより、リクエスト開始時にセッションを復元。 2. **操作**: ap_session_get / set を通じて session_rec 内の entries（apr_table_t）を操作。 3. **保存**: session_save フックにより、dirty フラグが立っている場合にのみ永続化処理を実行。

直接依存

  • apr_hooks.h, apr_optional.h: フックおよびオプション関数の定義。
  • apr_tables.h: セッションデータの保持に使用。
  • httpd.h, http_config.h: Apache モジュールとしての基本構成。
  • apr_uuid.h, apr_pools.h, apr_time.h: セッションの識別とメモリ管理。
  • ap_config.h

関連コード

  • modules/session/mod_session.c: 本ヘッダーで定義されたフックの実装と、セッション管理のメインロジック。
  • modules/session/mod_session_cookie.c 等: 本ヘッダーのフックを実装し、具体的なストレージバックエンドを提供するモジュール群。

読者向けメモ

  • **フックの重要性**: session_encode と session_decode は RUN_ALL で実行されるため、複数のセッション関連モジュールが連鎖的にデータを処理できる設計になっています。
  • **dirty フラグ**: パフォーマンス最適化のため、セッション内容が変更された場合のみ dirty フラグを立て、保存処理をトリガーする運用が推奨されます。
  • **拡張性**: 新しいストレージバックエンドを追加する場合は、本ヘッダーのフックを実装するだけで既存の mod_session インフラと統合可能です。
