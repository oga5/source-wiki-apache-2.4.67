modules/session/mod_session_crypto.c

概要

mod_session_crypto は、Apache HTTP Server のセッション情報を暗号化・復号化するためのモジュールです。mod_session が管理するセッションデータを、設定されたパスフレーズと暗号化アルゴリズムを用いて保護します。APR (Apache Portable Runtime) の apr_crypto API を抽象化レイヤーとして利用し、セッションの機密性を確保します。

モジュール内での役割

mod_session ファミリーの一員として、セッションデータの永続化（Cookie や DBD など）が行われる直前・直後に介入します。session_encode フックで暗号化を、session_decode フックで復号化を担当し、セッションストレージのバックエンドが何であるかに関わらず、透過的な暗号化を提供します。

責務

  • **暗号化・復号化の実行**: apr_crypto を用いたセッション文字列の変換。
  • **認証 (MAC)**: SipHash-2-4 を用いたデータの改ざん検知（暗号化データに MAC を付与）。
  • **設定管理**: サーバー全体およびディレクトリ単位での暗号化設定（パスフレーズ、暗号化アルゴリズム、ドライバ）の保持とマージ。
  • **キー管理**: パスフレーズからの暗号鍵生成と、複数のパスフレーズによる復号試行（キーローテーション対応）。

重要なシンボル

設定構造体

  • session_crypto_dir_conf: ディレクトリ単位の設定。パスフレーズのリストと使用する暗号アルゴリズムを保持。
  • session_crypto_conf: サーバー全体の設定。使用する暗号ライブラリ（ドライバ）とパラメータを保持。

暗号化・復号化ロジック

  • encrypt_string: セッション文字列を暗号化し、Base64 エンコードして返す。UUID をソルトとして使用。
  • decrypt_string: Base64 デコード後、登録されたパスフレーズを順次試行して復号。
  • compute_auth: SipHash を用いてデータに対する認証タグを生成。

初期化・フック

  • session_crypto_init: post_config フックで apr_crypto ドライバをロードし、サーバープロセスにコンテキストを保持。
  • session_crypto_encode / decode: mod_session のフック関数。

制御フローとデータフロー

1. **初期化**: session_crypto_init が起動時に呼ばれ、apr_crypto ドライバを初期化。生成された apr_crypto_t コンテキストは apr_pool_userdata_set を介してサーバープロセスプールに格納される。 2. **暗号化フロー**:

  • session_crypto_encode が呼ばれる。
  • encrypt_string が apr_crypto_passphrase で鍵を生成し、apr_crypto_block_encrypt でデータを暗号化。
  • compute_auth で MAC を計算し、[MAC][Salt][IV][Ciphertext] の形式で連結。
  • apr_base64_encode で文字列化。

    • 3. **復号化フロー**:

  • session_crypto_decode が呼ばれる。
  • decrypt_string が Base64 デコード後、compute_auth で MAC を検証。
  • 一致した場合のみ apr_crypto_block_decrypt を実行。複数のパスフレーズが設定されている場合、成功するまでループする。

読者向けメモ

  • **メモリ管理**: すべてのメモリ割り当ては request_rec のプール (r->pool) を使用しており、リクエスト終了時に自動解放されます。
  • **安全性**: ap_memeq_timingsafe を使用して MAC の比較を行っており、タイミング攻撃への耐性を考慮しています。
  • **制約**: APU_HAVE_CRYPTO が有効である必要があります。また、SessionCryptoPassphrase は複数指定可能で、復号時には古いパスフレーズも順次試行されるため、キーローテーションが可能です。
  • **省略された詳細**: SIPROUND などの SipHash 実装の詳細なビット演算は、APR 1.6 未満の環境向けのフォールバック実装です。最新の APR 環境では apr_siphash.h が優先されます。
