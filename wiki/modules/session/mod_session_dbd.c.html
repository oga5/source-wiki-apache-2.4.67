modules/session/mod_session_dbd.c

概要

mod_session_dbd.c は、Apache HTTP Server のセッション管理機能において、セッションデータをデータベース（SQL）に永続化するためのバックエンドモジュールです。mod_dbd を介してデータベース接続を確立し、セッションの読み込み、保存、削除を行います。

モジュール内での役割

mod_session サブシステムにおけるデータストア層の一つです。mod_session_cookie がセッションをクライアント側に保持するのに対し、本モジュールはサーバー側のデータベースを信頼できる唯一の情報源（Source of Truth）として利用します。

責務

  • **データベース連携**: mod_dbd のオプション関数を利用した接続の取得と、プリペアドステートメントの実行。
  • **セッション永続化**: 匿名セッション（UUIDベース）およびユーザー紐付けセッションの読み書き。
  • **設定管理**: ディレクトリ単位での SQL クエリラベルやクッキー名の設定、および設定の継承・マージ。
  • **フック処理**: session_load および session_save フックを実装し、リクエスト処理パイプラインに統合。

重要なシンボル

設定構造体

  • session_dbd_dir_conf: ディレクトリ単位の設定を保持。SQL クエリラベル（select/insert/update/delete）やクッキー名、ユーザー紐付けフラグを格納。

データベース操作（内部ヘルパー）

  • dbd_init: mod_dbd から接続を取得し、指定されたクエリラベルに対応するプリペアドステートメントを検証。
  • dbd_load, dbd_save, dbd_remove: データベースに対する実際の CRUD 操作を実行。

セッションライフサイクル

  • session_dbd_load: リクエストからキー（クッキーまたはユーザー名）を特定し、データベースからセッションを復元。
  • session_dbd_save: セッションの変更を検知し、データベースへの書き込みおよびクッキーの更新を行う。

設定・フック

  • session_dbd_cmds: SessionDBD* 系ディレクティブの定義。
  • register_hooks: セッションフックおよび監視用フックの登録。

制御フローとデータフロー

1. **読み込み**: session_dbd_load が呼ばれると、設定に基づきキーを抽出。dbd_load が mod_dbd を介して SQL を実行し、結果を session_rec に格納。 2. **保存**: session_dbd_save が呼ばれると、セッションの dirty フラグや UUID の有無を確認。dbd_save が更新（UPDATE）を試み、失敗した場合は挿入（INSERT）を行う。 3. **キャッシュ**: パフォーマンス向上のため、一度読み込んだセッションは request_rec->notes にキャッシュされ、同一リクエスト内での再パースを回避する。

読者向けメモ

  • **メモリ管理**: すべてのデータは request_rec のプール（r->pool）で管理されます。セッションの生存期間はリクエストの生存期間に依存します。
  • **未実装機能**: dbd_clean は現在 APR_ENOTIMPL を返しており、期限切れセッションの自動削除機能は未実装です。
  • **制約**: mod_dbd がロードされていない場合、モジュールは初期化に失敗します。
  • **エッジケース**: peruser 設定時、ログインしていないユーザーに対するセッション保存は拒否されます。また、セッションが存在しない場合は空のセッションが生成されます。
