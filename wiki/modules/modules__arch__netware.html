modules/arch/netware

システム内での役割

modules/arch/netware は、Apache HTTP Server を NetWare OS 上で動作させるためのプラットフォーム固有の拡張機能群です。OS ネイティブな実行モデル（NLM）やネットワークスタック（WinSock）と Apache を統合し、NetWare 環境特有の機能を提供します。

責務

  • **NLM ライフサイクル管理**: NetWare 固有のロード/アンロード処理およびネットワークスタックの初期化。
  • **CGI 実行環境の最適化**: ファイル拡張子と NLM のマッピング、およびアドレス空間の分離制御。
  • **SSL/TLS 統合**: NetWare ネイティブの SSL 実装を利用したセキュアなリスニングポートの提供と TLS アップグレード（RFC 2817）のサポート。

このモジュールのファイル

  • libprews.c: NLM ライフサイクル管理および WinSock 初期化の補助ライブラリ。
  • mod_netware.c: NetWare MPM 向け CGI 実行環境の制御および拡張子マッピング。
  • mod_nw_ssl.c: NetWare ネイティブ SSL/TLS 実装の統合モジュール。

主要なデータ構造

  • netware_dir_config: ディレクトリ単位の CGI 拡張子マッピング設定。
  • NWSSLSrvConfigRec: サーバーごとの SSL 設定（リスナーやアップグレード可能ポート）。
  • seclisten_rec / seclistenup_rec: セキュアリスナーの定義を保持するリンクリスト。
  • secsocket_data: SSL 接続のコンテキスト情報。

主なフロー

1. **初期化**: libprews.c が NLM としての基盤を構築し、mod_nw_ssl.c が SecureListen ディレクティブに基づき WinSock を介して SSL ソケットを生成・登録します。 2. **リクエスト処理**: mod_netware.c が CGI リクエストの拡張子を解析し、対応する NLM を特定して実行コマンドを構築します。 3. **SSL アップグレード**: mod_nw_ssl.c が Upgrade ヘッダーを検知すると、ソケットを暗号化状態へ遷移させます。 4. **終了**: libprews.c が NLM アンロード時にネットワークリソースを解放します。

外部依存

  • **Apache API**: httpd.h, http_config.h, http_protocol.h 等。
  • **APR**: apr_portable.h, apr_strings.h 等。
  • **NetWare OS**: WinSock (WSAIoctl), unilib.h, novsock2.h。

内部構成

本モジュールはディレクトリ境界に基づき構成されており、NetWare 固有の機能を Apache のフックシステムに統合する役割を担います。子モジュールは存在しません。

推奨読書順

1. mod_nw_ssl.c: NetWare 固有のネットワーク制御の理解。 2. mod_netware.c: CGI 実行モデルと設定管理の理解。 3. libprews.c: OS レベルのライフサイクル管理の理解。
