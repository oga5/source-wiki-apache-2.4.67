modules/ssl/ssl_engine_vars.c

概要

ssl_engine_vars.c は、Apache HTTP Server の設定式（ap_expr）やログ設定から SSL/TLS 関連の情報を取得・抽出するための変数ルックアップ機能を提供します。証明書情報、セッション状態、暗号スイートなどの内部状態を文字列として公開するインターフェースです。

モジュール内での役割

mod_ssl の中核となる変数解決エンジンです。他のモジュール（mod_log_config など）が SSL 情報を必要とする際のゲートウェイとして機能し、OpenSSL の複雑なデータ構造を Apache のメモリ管理（apr_pool_t）に適した文字列形式に変換します。

責務

  • **変数解決**: SSL_ で始まる変数や SSL() 関数に対するリクエストを処理し、適切な値を返す。
  • **証明書解析**: X.509 証明書から DN（識別名）、SAN（サブジェクト代替名）、有効期限、シリアル番号などを抽出する。
  • **ログ統合**: mod_log_config 用のハンドラを登録し、ログフォーマット文字列（%c, %x）経由での SSL 情報出力をサポートする。
  • **OpenSSL 抽象化**: OpenSSL のバージョン差異を吸収し、BIO を介した安全な文字列変換を行う。

重要なシンボル

変数解決・登録

  • ssl_var_lookup: 公開 API。リクエスト、接続、サーバー情報に基づき変数を解決するメイン関数。
  • ssl_var_register: モジュール初期化時に ap_expr フックとオプション関数を登録する。
  • ssl_expr_lookup: ap_expr の構文解析時に SSL 関連の関数や変数を解決するためのフック関数。

証明書・セッション抽出

  • ssl_var_lookup_ssl: SSLConnRec から SSL 固有の情報を抽出するディスパッチャ。
  • ssl_var_lookup_ssl_cert: 証明書（X509）から特定の属性（DN, SAN, 有効期限等）を抽出する。
  • modssl_var_extract_dns / modssl_var_extract_san_entries: 証明書情報をテーブルに一括抽出するユーティリティ。

ログハンドラ

  • ssl_var_log_config_register: mod_log_config にログハンドラを登録する。
  • ssl_var_log_handler_c / ssl_var_log_handler_x: ログ出力用の変数解決ハンドラ。

制御フローとデータフロー

1. **リクエスト**: ap_expr またはログハンドラが ssl_var_lookup を呼び出す。 2. **解決**:

  • リクエスト依存（HTTP ヘッダ等）、接続依存（SSL 状態）、独立（時刻等）の順で検索。
  • SSL_ プレフィックスがある場合、ssl_var_lookup_ssl へ委譲。

    • 3. **抽出**: ssl_var_lookup_ssl は SSL オブジェクトから証明書を取得し、ssl_var_lookup_ssl_cert 等のヘルパーを介して BIO を使用して文字列化する。 4. **メモリ管理**: 全ての戻り値は apr_pool_t 上で確保され、呼び出し元が解放を管理する。

直接依存

  • ssl_private.h: SSLConnRec などの内部構造体定義。
  • mod_ssl.h: 公開インターフェース。
  • ap_expr.h: Apache 式評価エンジンとの統合。
  • apr_time.h: 時刻関連の計算。
  • mod_log_config.h: ログ出力機能との統合。
  • ../../modules/loggers/mod_log_config.h

関連コード

  • ssl_util_ssl.c: OpenSSL の低レベルなラッパー関数群。
  • ssl_engine_kernel.c: SSL 接続のライフサイクル管理。
  • mod_ssl.c: モジュールのメインエントリポイント。

読者向けメモ

  • **メモリ管理**: ssl_var_lookup は apr_pstrdup を多用します。大量の変数参照が発生する環境ではメモリ消費に注意してください。
  • **OpenSSL 依存**: OPENSSL_VERSION_NUMBER による条件分岐が随所にあり、OpenSSL 1.1.x 以降とそれ以前の API 差異を吸収しています。
  • **証明書ライフサイクル**: SSL_get_peer_certificate は参照カウントを増やすため、必ず X509_free が対になるよう設計されています。SSL_get_certificate は参照カウントを増やさないため、解放は不要です。
  • **未実装・浅い部分**: ssl_var_lookup_ssl_cert_dn_rec に含まれない特定の証明書拡張フィールドの抽出は、汎用的な ssl_ext_list を介して行う必要があります。
