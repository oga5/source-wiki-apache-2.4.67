modules/ssl/mod_ssl.c

概要

mod_ssl.c は Apache HTTP Server の SSL/TLS モジュールのエントリポイントです。Apache のモジュール構造を定義し、設定ディレクティブの登録、フックの初期化、および接続ライフサイクル管理の起点を提供します。

モジュール内での役割

mod_ssl サブシステムにおける「司令塔」として機能します。具体的な暗号化処理や I/O 操作は ssl_engine_*.c に委譲し、本ファイルはそれらを Apache のリクエスト処理パイプラインに統合する役割を担います。

責務

  • **設定管理**: command_rec を通じて、サーバー、ディレクトリ、プロキシごとの SSL 設定ディレクティブを定義・登録。
  • **ライフサイクル管理**: モジュールの初期化 (pre_config, post_config)、子プロセス初期化、および終了時のクリーンアップ。
  • **接続フック**: 接続確立時 (pre_connection) および処理時 (process_connection) の SSL コンテキストの初期化とハンドシェイクのトリガー。
  • **プロバイダ登録**: mod_proxy との連携や、認証プロバイダ (ssl, ssl-verify-client) の登録。

重要なシンボル

モジュール定義

  • ssl_module: Apache に登録される module 構造体。設定の作成・マージ関数、コマンドテーブル、フック登録関数を保持。

設定ディレクティブ (マクロ)

  • SSL_CMD_ALL, SSL_CMD_SRV, SSL_CMD_PXY, SSL_CMD_DIR: 設定ディレクティブを簡潔に定義するためのヘルパーマクロ。

接続管理

  • ssl_init_connection_ctx: conn_rec に紐づく SSLConnRec を生成・取得し、設定を適用する。
  • ssl_init_ssl_connection: OpenSSL の SSL オブジェクトを生成し、接続にバインドする。
  • ssl_hook_pre_connection: 接続開始時に SSL エンジンを有効化し、ハンドシェイクの準備を行う。

ライフサイクル・フック

  • ssl_register_hooks: Apache の各フェーズ（接続、設定、認証など）にコールバックを登録。
  • ssl_hook_pre_config: OpenSSL ライブラリの初期化、エラー文字列のロード、エンジン登録を実行。

制御フローとデータフロー

1. **初期化**: ssl_register_hooks が起動時に呼ばれ、各フックを登録。ssl_hook_pre_config で OpenSSL ライブラリを初期化。 2. **接続確立**: ssl_hook_pre_connection が呼ばれ、ssl_init_connection_ctx で設定を読み込み、ssl_init_ssl_connection で SSL オブジェクトを生成。 3. **ハンドシェイク**: ssl_hook_process_connection が AP_MODE_INIT モードで入力を読み込むことで、OpenSSL のハンドシェイクをトリガーする。 4. **リクエスト処理**: ssl_hook_http_scheme 等がリクエスト時に呼ばれ、現在の接続が TLS かどうかを判定する。

直接依存

  • ssl_private.h: モジュール内部の共有構造体と型定義。
  • http_config.h: Apache 設定 API。
  • mod_proxy.h: プロキシ機能との連携用。
  • util_mutex.h: セッションキャッシュ等の排他制御用。
  • ap_provider.h
  • assert.h
  • util_md5.h

関連コード

  • ssl_engine_init.c: モジュールの初期化ロジックの詳細。
  • ssl_engine_io.c: 実際の SSL フィルタ処理とデータ転送。
  • ssl_engine_config.c: 設定構造体の作成とマージロジック。
  • mod_ssl.h: 他モジュールから利用可能な公開関数（ssl_var_lookup 等）。

読者向けメモ

  • **メモリ管理**: conn_rec や request_rec のプール (pool) を使用してメモリを管理しています。ssl_init_connection_ctx で生成される SSLConnRec は接続の生存期間中保持されます。
  • **OpenSSL バージョン**: OpenSSL 1.1.0 以降とそれ以前のバージョンで初期化・終了処理が大きく異なるため、#if による条件分岐が多用されています。
  • **省略された詳細**: 本ファイルはインターフェース層であるため、具体的な暗号化アルゴリズムの選択や証明書の検証ロジックは ssl_engine_kernel.c 等に委譲されており、本ファイルには含まれていません。
  • **注意点**: modssl_running_statically は、モジュールが静的にリンクされているか動的にロードされているかを判定し、OpenSSL のクリーンアップ挙動を制御するために使用されます。
