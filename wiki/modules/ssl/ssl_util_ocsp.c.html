modules/ssl/ssl_util_ocsp.c

概要

本ファイルは、Apache HTTP Server の mod_ssl における OCSP (Online Certificate Status Protocol) クライアント機能を実装しています。外部の OCSP レスポンダと通信し、証明書の有効性を検証するためのリクエスト送信およびレスポンス受信を担います。

モジュール内での役割

mod_ssl の OCSP サブシステムの一部であり、主に ssl_engine_ocsp.c や ssl_util_stapling.c から呼び出される低レベルな通信ユーティリティを提供します。HTTP クライアントライブラリに依存しない、スタンドアロンな簡易 HTTP/1.0 実装を含みます。

責務

  • **OCSP リクエストのシリアライズ**: OCSP_REQUEST オブジェクトを HTTP POST リクエスト形式に変換する。
  • **ネットワーク通信**: 指定された URI に対して TCP ソケットを確立し、リクエストの送信とレスポンスの受信を行う。
  • **レスポンスの解析**: 受信した HTTP レスポンスから OCSP データを抽出し、OCSP_RESPONSE オブジェクトへデコードする。
  • **証明書管理**: OCSP 検証に必要な信頼済み証明書チェーンを PEM ファイルから読み込む。

重要なシンボル

  • modssl_dispatch_ocsp_request: OCSP 通信のメインエントリポイント。リクエストの送信からレスポンスの取得までを制御する。
  • serialize_request: OCSP リクエストを HTTP POST メッセージとして BIO に書き出す。
  • read_response: ソケットから HTTP レスポンスを読み取り、OCSP 応答をデコードする。
  • ssl_init_ocsp_certificates: 設定ファイルで指定された OCSP 信頼証明書を読み込み、検証フラグを更新する。

制御フローとデータフロー

1. **リクエスト生成**: modssl_dispatch_ocsp_request が serialize_request を呼び出し、メモリ BIO に HTTP POST リクエストを作成する。 2. **送信**: send_request がソケットをオープンし、BIO の内容をネットワークへ送出する。 3. **受信**: read_response がソケットから HTTP ヘッダーを解析し、ボディを BIO に蓄積する。 4. **デコード**: 蓄積された BIO から d2i_OCSP_RESPONSE_bio を用いて OpenSSL の OCSP オブジェクトを生成する。

直接依存

  • ssl_private.h: mod_ssl 内部の共通定義およびマクロ。
  • apr_buckets.h: HTTP レスポンスのストリーム処理に使用。
  • apr_uri.h: OCSP レスポンダの URI 解析に使用。

関連コード

  • ssl_engine_ocsp.c: OCSP 検証のロジック層。本ファイルの通信関数を利用する。
  • ssl_util_stapling.c: OCSP Stapling 実装。証明書検証の文脈で本ファイルと密接に関連する。

読者向けメモ

  • **HTTP 実装の限定性**: 本ファイル内の HTTP クライアントは「おもちゃ（toy）」と称される簡易実装です。将来的に Apache が標準的な HTTP クライアントライブラリを採用した際には、大部分が置き換えられる予定です。
  • **メモリ保護**: MAX_HEADERS および MAX_CONTENT 定数により、不正なレスポンスによるメモリ枯渇を防ぐための制限が設けられています。
  • **OpenSSL 依存**: OPENSSL_NO_OCSP が定義されている環境では、本ファイルの内容は無効化されます。
