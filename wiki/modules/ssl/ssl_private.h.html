modules/ssl/ssl_private.h

概要

ssl_private.h は、Apache HTTP Server の mod_ssl モジュールにおける内部インターフェースと共有データ構造を定義する中心的なヘッダーファイルです。OpenSSL のバージョン差異を吸収する抽象化層、モジュール固有の構成構造体、および各サブコンポーネント（I/O、セッションキャッシュ、OCSP、設定管理）間の関数プロトタイプを定義します。

モジュール内での役割

mod_ssl の各ソースファイル（ssl_engine_*.c など）が共通して参照する「内部的な契約」を定義します。公開 API である mod_ssl.h とは異なり、モジュール内部のプライベートな実装詳細や、OpenSSL のバージョン依存コードを隠蔽する役割を担います。

責務

  • **互換性レイヤー**: OpenSSL 1.0.x から 3.x までの API 差異をマクロやラッパー関数で吸収する。
  • **設定管理**: サーバー全体（SSLModConfigRec）、仮想ホスト（SSLSrvConfigRec）、ディレクトリ（SSLDirConfigRec）ごとの設定構造体を定義する。
  • **状態保持**: 接続ごとの SSL 状態（SSLConnRec）を管理する。
  • **機能定義**: OCSP、TLS セッションチケット、SRP、ALPN などの機能フラグと関連構造体の提供。
  • **ユーティリティ**: 文字列操作、パスチェック、ログ出力、およびモジュール内関数プロトタイプの宣言。

重要なシンボル

設定構造体 (Configuration)

  • SSLModConfigRec: プロセス生存期間中保持されるグローバル設定（鍵キャッシュ、セッションキャッシュ、ミューテックス）。
  • SSLSrvConfigRec: 仮想ホストごとの SSL 設定。
  • SSLDirConfigRec: ディレクトリごとの SSL 設定（認証モード、暗号スイートなど）。
  • SSLConnRec: 現在の接続における SSL 状態（証明書、検証結果、再ネゴシエーション状態）。

互換性マクロ

  • MODSSL_USE_OPENSSL_PRE_1_1_API: OpenSSL 1.1.0 未満の API を使用するかどうかを判定するフラグ。
  • HAVE_FIPS, HAVE_TLSEXT, HAVE_OCSP_STAPLING: コンパイル時に有効化された機能を識別するフラグ。

ユーティリティ・アクセサ

  • myConnConfig(c), mySrvConfig(srv): Apache の ap_get_module_config をラップし、型安全に設定を取得するマクロ群。

制御フローとデータフロー

  • **初期化フロー**: ssl_init_Module がサーバー起動時に呼ばれ、SSLModConfigRec を初期化します。その後、各仮想ホストの設定が ssl_init_ConfigureServer で構築されます。
  • **リクエストフロー**: ssl_hook_ReadReq や ssl_hook_Access などのフック関数が、SSLConnRec を通じて接続ごとの SSL コンテキストを操作します。
  • **データフロー**: ssl_asn1_table_* 関数群が、DER 形式の鍵データを SSLModConfigRec 内のハッシュテーブルで管理し、プロセス全体で共有します。

読者向けメモ

  • **メモリ生存期間**: SSLModConfigRec はプロセスプールに割り当てられ、サーバーのライフサイクル全体で生存します。これは、パスフレーズを一度入力すれば再起動まで保持するためですが、ABI 互換性の観点からは注意が必要です。
  • **OpenSSL 3.0 移行**: OPENSSL_VERSION_NUMBER による条件分岐が多用されています。新しい機能を追加する際は、バージョン番号ではなく、HAVE_* マクロや機能ベースの検出を優先してください。
  • **省略された詳細**: 本ファイルは非常に広範な定義を含むため、個々の OpenSSL コールバック関数の詳細な引数仕様については、各 ssl_engine_*.c の実装を参照してください。
