modules/ssl/ssl_engine_config.c

概要

Apache HTTP Server の mod_ssl における設定ディレクティブの定義と、それらを保持する設定構造体の管理を行うファイルです。サーバー全体（Global）、仮想ホスト（Server）、ディレクトリ（Per-dir）の各スコープにおける SSL 設定の初期化、マージ、および検証ロジックを網羅しています。

モジュール内での役割

mod_ssl の設定サブシステムの中核であり、httpd の設定解析フェーズで呼び出されるコマンドハンドラを提供します。ここで生成・管理された設定構造体は、後の ssl_engine_init.c での SSL コンテキスト構築や、リクエスト処理時の動作決定に使用されます。

責務

  • **設定構造体のライフサイクル管理**: SSLModConfigRec（グローバル）、SSLSrvConfigRec（サーバー）、SSLDirConfigRec（ディレクトリ）の生成と初期化。
  • **設定のマージ**: 継承関係にある設定（サーバーから仮想ホスト、親ディレクトリから子ディレクトリ）の統合。
  • **ディレクティブの検証**: ファイルパスやディレクトリパスの存在確認、引数の妥当性チェック。
  • **コマンドハンドラの実装**: SSLPassPhraseDialog, SSLEngine, SSLCipherSuite 等のディレクティブに対するコールバックの実装。

重要なシンボル

設定構造体管理

  • ssl_config_global_create: グローバル設定の初期化。サーバーの生存期間中保持される。
  • ssl_config_server_create / ssl_config_perdir_create: 各スコープの設定構造体を生成。
  • ssl_config_server_merge / ssl_config_perdir_merge: 親の設定をベースに子設定をマージする。

コンテキスト初期化・マージ

  • modssl_ctx_init: modssl_ctx_t（SSL 設定の本体）の初期化。
  • modssl_ctx_cfg_merge: 共通の SSL 設定項目をマージする。
  • modssl_ctx_cfg_merge_server / modssl_ctx_cfg_merge_proxy: サーバーおよびプロキシ固有のコンテキストマージ。

コマンドハンドラ（主要なもの）

  • ssl_cmd_SSLEngine: SSL の有効/無効を切り替える。
  • ssl_cmd_SSLCipherSuite: 暗号スイートを設定する。
  • ssl_cmd_SSLProtocol: 使用可能なプロトコルバージョンを制御する。
  • ssl_cmd_SSLOptions: SSL 関連のオプションフラグを操作する。

制御フローとデータフロー

1. **初期化**: httpd が設定ファイルを読み込む際、各ディレクティブに対応する ssl_cmd_* 関数が呼び出されます。 2. **検証**: ssl_cmd_check_file や ssl_cmd_check_dir を経由して、指定されたパスがサーバーのルートディレクトリに対して妥当か確認されます。 3. **マージ**: サーバー起動時や設定の再読み込み時に、*_merge 関数が呼び出され、base（親）と add（子）の設定が mrg（マージ結果）に統合されます。この際、cfgMerge マクロ群を使用して、未設定（UNSET）項目を親の値で埋める処理が行われます。

直接依存

  • ssl_private.h: mod_ssl 内部の構造体定義とマクロ。
  • util_mutex.h: 排他制御用。
  • ap_provider.h: セッションキャッシュ等のプロバイダ検索用。

関連コード

  • ssl_private.h: 設定構造体（SSLModConfigRec, SSLSrvConfigRec 等）の定義元。
  • ssl_engine_init.c: 本ファイルで作成された設定を読み込み、実際の SSL_CTX を構築する。
  • mod_ssl.c: モジュールのフック定義とコマンドテーブルの登録。

読者向けメモ

  • **メモリ管理**: 設定構造体は apr_pool_t を使用して割り当てられます。サーバーの生存期間中保持される設定は s->process->pool を、リクエスト単位の設定は cmd->pool を使用しています。
  • **マージの注意点**: cfgMerge マクロは、add が UNSET の場合に base を採用する単純な継承ロジックです。一部の複雑な設定（証明書配列など）は専用の _merge 関数で個別に処理されています。
  • **未実装・浅い部分**: XXX コメントがある通り、一部の per-dir CA 設定などは将来的な再実装待ちの状態です。また、OpenSSL のバージョン依存（HAVE_SSL_CONF_CMD 等）により、利用可能なディレクティブがコンパイル環境によって変化することに注意してください。
