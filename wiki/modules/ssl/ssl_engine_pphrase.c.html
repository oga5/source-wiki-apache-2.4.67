modules/ssl/ssl_engine_pphrase.c

概要

本ファイルは、Apache mod_ssl における暗号化された秘密鍵の復号用パスフレーズ入力処理を担います。サーバー起動時や再起動時に、ファイルシステム上の鍵ファイル、外部コマンド（パイプ）、または OpenSSL エンジン/ストア API を介してパスフレーズを収集するメカニズムを提供します。

モジュール内での役割

mod_ssl の初期化フェーズにおいて、暗号化された秘密鍵を読み込むための「対話型インターフェース」として機能します。ssl_engine_init.c から呼び出され、ユーザーや外部プロセスからパスフレーズを取得し、OpenSSL の鍵読み込み処理を完結させます。

責務

  • **パスフレーズ収集**: 組み込み（標準入力）、外部パイプ、または外部フィルタプログラムを通じたパスフレーズの取得。
  • **対話管理**: ユーザーへのプロンプト表示、入力の検証、およびリトライ回数の制御。
  • **鍵のキャッシュと再利用**: サーバー再起動時に、ファイル更新時刻（mtime）をチェックし、変更がない場合は既存の鍵を再利用してパスフレーズ入力を省略。
  • **API 抽象化**: OpenSSL の UI_METHOD を実装し、エンジン（ENGINE API）やストア（OSSL_STORE API）経由での鍵読み込みをサポート。

重要なシンボル

データ構造

  • pphrase_cb_arg_t: パスフレーズ収集時の状態（サーバー設定、メモリプール、試行回数、鍵IDなど）を保持するコンテキスト構造体。

コールバック・UI メソッド

  • ssl_pphrase_Handle_CB: レガシーな PEM パスフレーズコールバック。
  • get_passphrase_ui: OpenSSL の UI_METHOD を構築し、passphrase_ui_open/read/write/close を登録。

鍵読み込み・管理

  • ssl_load_encrypted_pkey: 暗号化された秘密鍵を読み込むメインのエントリポイント。
  • modssl_load_engine_keypair: ENGINE または OSSL_STORE API を使用して鍵と証明書をロードする高レベル関数。
  • asn1_table_vhost_key: 仮想ホストごとの鍵IDを管理し、メモリリークを防ぐためのハッシュテーブル操作。

制御フローとデータフロー

1. **初期化**: ssl_load_encrypted_pkey が呼ばれ、pphrase_cb_arg_t が初期化されます。 2. **キャッシュ確認**: exists_and_readable で鍵ファイルの mtime を確認し、キャッシュ済みであれば処理を終了します。 3. **収集**:

  • SSLPassPhraseDialog 設定に基づき、組み込み、パイプ、フィルタのいずれかの方式でパスフレーズを要求します。
  • 外部パイプの場合、ssl_pipe_child_create で子プロセスを生成し、readtty/writetty を介して通信します。

    • 4. **復号**: modssl_read_privatekey または ENGINE_load_private_key がコールバック経由でパスフレーズを受け取り、鍵を復号します。 5. **クリーンアップ**: 成功後、機密データ（パスフレーズ）を OPENSSL_cleanse でメモリから消去し、パイプを閉じます。

直接依存

  • ssl_private.h: mod_ssl 内部定義および設定構造体。
  • openssl/ui.h: OpenSSL の対話型 UI API。
  • openssl/store.h: URI ベースの鍵/証明書読み込み API。

関連コード

  • ssl_engine_init.c: 本ファイルの関数を呼び出し、SSL サーバーの初期化を行う。
  • ssl_util_ssl.c: modssl_read_privatekey 等の低レベルな OpenSSL ラッパーを提供。

読者向けメモ

  • **メモリ管理**: pphrase_cb_arg_t 内のパスフレーズは apr_pool_t に割り当てられます。機密データは OPENSSL_cleanse で適切に消去されますが、メモリレイアウトは OpenSSL の仕様に依存します。
  • **Win32 の制約**: Windows 環境では SSLPassPhraseDialog builtin がサポートされておらず、パイプ経由の入力が必須となる点に注意してください。
  • **設計の意図**: asn1_table_vhost_key で s->process->pool を使用しているのは、再起動時にも鍵情報を保持し、パスフレーズ入力を最小限にするための設計です。
  • **省略された詳細**: 本ファイルはパスフレーズの「収集」に特化しており、復号後の鍵の具体的な利用方法や証明書チェーンの検証ロジックは他のファイル（ssl_engine_init.c 等）に委譲されています。
