modules/ssl/ssl_engine_ocsp.c

概要

本ファイルは、Apache HTTP Server の mod_ssl における OCSP (Online Certificate Status Protocol) 検証ロジックを実装しています。証明書の失効状態を外部の OCSP レスポンダに問い合わせ、その結果に基づいて証明書の検証ステータスを決定します。

モジュール内での役割

mod_ssl の証明書検証プロセスの一部として機能します。OpenSSL の X509_STORE_CTX を介してフックされ、証明書のチェーン検証中に動的な失効確認を行います。

責務

  • 証明書内の AIA (Authority Information Access) 拡張からの OCSP レスポンダ URI の抽出。
  • OCSP リクエストの生成（Nonce の付与を含む）。
  • OCSP レスポンダとの通信（modssl_dispatch_ocsp_request を利用）。
  • レスポンスの検証（署名検証、Nonce チェック、有効期限確認）。
  • 検証結果を OpenSSL の検証コンテキストへ反映。

重要なシンボル

  • modssl_verify_ocsp: 外部から呼び出されるメインのエントリポイント。
  • verify_ocsp_status: OCSP 検証の全工程を統括する内部関数。
  • extract_responder_uri: 証明書から OCSP URI を取得するヘルパー。
  • create_request: OpenSSL の OCSP_REQUEST オブジェクトを構築する関数。

制御フローとデータフロー

1. **初期化**: modssl_verify_ocsp が呼び出され、一時的なメモリプールを作成。 2. **URI 決定**: determine_responder_uri が設定値または証明書内の AIA 拡張から接続先を特定。 3. **リクエスト生成**: create_request で証明書 ID を作成し、必要に応じて Nonce を追加。 4. **通信と検証**: verify_ocsp_status がレスポンダと通信し、署名検証および OCSP_check_validity による有効期限チェックを実行。 5. **結果反映**: 検証結果（Good/Revoked/Unknown）を X509_STORE_CTX_set_error を通じて OpenSSL の検証エンジンに通知。

直接依存

  • ssl_private.h: mod_ssl 内部の構造体定義や設定情報。
  • apr_base64.h: Base64 変換ユーティリティ。
  • OpenSSL libcrypto (OCSP 関連 API)。

関連コード

  • modules/ssl/ssl_util_ocsp.c: OCSP 通信の低レイヤー実装（modssl_dispatch_ocsp_request など）。
  • modules/ssl/ssl_util_stapling.c: OCSP Stapling（サーバー側でレスポンスをキャッシュする機能）の実装。

読者向けメモ

  • 本ファイルは OPENSSL_NO_OCSP が定義されていない場合にのみ有効です。
  • OCSP レスポンダへの通信は同期的に行われるため、ネットワーク遅延がリクエスト処理時間に直結します。
  • ocsp_noverify 設定が有効な場合、レスポンダの署名検証をスキップしますが、セキュリティリスクを伴うため注意が必要です。
  • メモリ管理のため、検証プロセスごとに一時的な apr_pool_t を使用しています。
