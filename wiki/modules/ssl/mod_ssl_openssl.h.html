modules/ssl/mod_ssl_openssl.h

概要

本ファイルは、Apache HTTP Server の mod_ssl モジュールが提供する OpenSSL 固有の API インターフェースを定義するヘッダーファイルです。外部モジュールが mod_ssl の SSL 処理プロセスに介入するためのフックポイントを公開しています。

モジュール内での役割

mod_ssl の拡張性を担保するブリッジの役割を果たします。コアとなる SSL 処理ロジックと、サードパーティ製モジュールや他の Apache モジュールとの間で、SSL コンテキストやハンドシェイク状態を共有するための契約（インターフェース）を定義します。

責務

  • **OpenSSL バージョン管理**: OpenSSL のバージョンに応じたヘッダーのインクルードと、内部構造体へのアクセス制限（OPENSSL_NO_SSL_INTERN）の制御。
  • **拡張フックの定義**: SSL ライフサイクル（初期化、ハンドシェイク、OCSP ステープリング等）における外部介入ポイントの提供。

重要なシンボル

  • init_server: SSL コンテキスト（SSL_CTX）の初期化時に呼び出されるフック。
  • pre_handshake: TLS ハンドシェイク直前に実行されるフック。
  • answer_challenge: バーチャルホスト設定外の接続に対し、証明書と秘密鍵を動的に提供するためのフック。
  • init_stapling_status / get_stapling_status: OCSP ステープリングのステータス取得と提供を行うためのフック群。

制御フローとデータフロー

本ファイル自体はインターフェース定義であるため、直接的なロジックは持ちません。 1. **初期化フロー**: init_server フックを通じて、各サーバー設定に対応する SSL_CTX が構築されます。 2. **接続フロー**: 接続確立時、pre_handshake を経て SSL オブジェクトが準備されます。 3. **拡張フロー**: OCSP ステープリング等の機能では、init_stapling_status で責任者が決定され、以降の接続ごとに get_stapling_status が呼び出されて DER 形式のレスポンスデータが供給されます。

直接依存

  • mod_ssl.h: 基本的なモジュール定義。
  • openssl/ssl.h, openssl/evp.h, openssl/x509.h: OpenSSL のコア API。
  • openssl/opensslv.h: バージョン判定用。
  • openssl/macros.h

関連コード

  • ssl_private.h: mod_ssl 内部で共有されるプライベートな構造体や定数が定義されており、フックの実装時に参照されます。
  • ssl_util_stapling.c: 本ヘッダーで定義された OCSP フックを利用する主要な実装箇所です。

読者向けメモ

  • **メモリ管理**: get_stapling_status フックで返される DER データは malloc() で確保する必要があり、所有権は mod_ssl に移譲されます。この点に注意しないとメモリリークや二重解放の原因となります。
  • **バージョン互換性**: OpenSSL 3.0 以降とそれ以前でインクルードするヘッダーやマクロが分岐しているため、新しい機能を実装する際は OPENSSL_VERSION_NUMBER によるガードを適切に行ってください。
  • **フックの性質**: ここで定義されているフックは APR_DECLARE_EXTERNAL_HOOK を使用しており、Apache のフック機構に基づいています。戻り値が DECLINED の場合は次のフックへ処理が移る点に留意してください。
