modules/ssl/ssl_engine_init.c

概要

ssl_engine_init.c は、Apache HTTP Server の起動および再起動時に、SSL/TLS サブシステム全体を初期化する中心的な役割を担います。OpenSSL ライブラリのコンテキスト（SSL_CTX）の構築、証明書と秘密鍵の読み込み、暗号スイートの設定、および SNI や OCSP ステープリングなどの拡張機能の有効化を管理します。

モジュール内での役割

modules/ssl モジュールにおける「設定から実行状態への変換」を担当します。ssl_engine_config.c で解析された設定情報を基に、OpenSSL の API を呼び出して実際のメモリ上の SSL 構造体を生成します。

責務

  • **SSL コンテキストの構築**: サーバーおよびプロキシ用の SSL_CTX を生成し、プロトコルバージョン、暗号スイート、セッションキャッシュを設定する。
  • **証明書と鍵の管理**: PEM ファイルからの証明書・鍵の読み込み、整合性チェック、および中間証明書チェーンの構築。
  • **拡張機能の初期化**: SNI コールバック、TLS-SRP、OCSP ステープリング、およびセッションチケットの初期化。
  • **ハードウェアアクセラレーション**: OpenSSL エンジン（Crypto Device）のロードと設定。
  • **ライフサイクル管理**: モジュール初期化時のリソース確保と、終了時のクリーンアップ（ssl_init_ModuleKill）。

重要なシンボル

初期化・ライフサイクル

  • ssl_init_Module: モジュール全体の初期化エントリーポイント。
  • ssl_init_ConfigureServer: 特定のバーチャルホストに対する SSL 設定の適用。
  • ssl_init_ModuleKill: 終了時のリソース解放。

コンテキスト構築（ssl_init_ctx_* ファミリー）

  • ssl_init_ctx: SSL_CTX 構築のメインパイプライン。
  • ssl_init_ctx_protocol: プロトコルバージョン（TLS 1.2/1.3 等）の制御。
  • ssl_init_ctx_verify: クライアント認証および CA 証明書パスの設定。
  • ssl_init_ctx_cipher_suite: 暗号スイートの適用。
  • ssl_init_server_certs: 証明書と秘密鍵の読み込みと検証。

ユーティリティ・ヘルパー

  • create_sni_policy_hash: SNI ポリシーの MD5 ハッシュ生成。
  • ssl_is_challenge: ACME チャレンジ（mod_md）対応のためのフック。
  • modssl_get_dh_params: DH パラメータの取得（OpenSSL 1.1 未満用）。

制御フローとデータフロー

1. **初期化開始**: ssl_init_Module が呼ばれ、グローバル設定の固定と PRNG（乱数生成器）のシードが行われます。 2. **サーバー設定**: ssl_init_ConfigureServer が各バーチャルホストをループし、ssl_init_server_ctx を呼び出します。 3. **コンテキスト構築**: ssl_init_ctx が一連の ssl_init_ctx_* 関数を呼び出し、SSL_CTX を段階的に構築します。 4. **証明書ロード**: ssl_init_server_certs がファイルシステムから証明書を読み込み、SSL_CTX に紐付けます。 5. **検証**: 最後に SSL_CTX_check_private_key で証明書と鍵のペアが一致するかを確認し、失敗した場合は ssl_die でプロセスを終了させます。

直接依存

  • ssl_private.h: モジュール内部のデータ構造（modssl_ctx_t, SSLSrvConfigRec 等）の定義。
  • apr_md5.h: SNI ポリシーハッシュ計算用。
  • mpm_common.h: プロセス管理用。
  • mod_md.h: ACME チャレンジ連携用。

関連コード

  • ssl_engine_config.c: 設定ディレクティブの解析とデータ構造の生成。
  • ssl_util_ssl.c: OpenSSL API のラッパーや補助関数。
  • ssl_engine_kernel.c: 接続時のハンドシェイク処理（ssl_callback_* 等）。

読者向けメモ

  • **メモリ所有権**: SSL_CTX は modssl_ctx_t が所有し、ssl_init_ctx_cleanup で解放されます。証明書や鍵の読み込み時に OpenSSL が内部コピーを持つ場合と、Apache 側で管理が必要な場合があります。
  • **OpenSSL バージョン依存**: MODSSL_USE_OPENSSL_PRE_1_1_API マクロにより、OpenSSL 1.1.0 前後の API 差異が吸収されています。特に DH パラメータや証明書ロード処理で分岐が発生します。
  • **エラーパス**: 設定ミス（証明書不一致など）は ssl_die() を呼び出し、サーバー起動を即座に停止させます。
  • **省略された詳細**: 本ファイルは初期化ロジックに特化しており、実際の暗号化通信（I/O）やセッションキャッシュの具体的な実装（ssl_engine_io.c, ssl_scache.c）については詳細を省略しています。
