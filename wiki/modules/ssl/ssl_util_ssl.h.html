modules/ssl/ssl_util_ssl.h

概要

ssl_util_ssl.h は、OpenSSL ライブラリの機能を mod_ssl のコンテキストで利用しやすくするための高レベルなユーティリティ関数群を定義するヘッダーファイルです。

モジュール内での役割

modules/ssl モジュールにおいて、OpenSSL の低レベルな API を直接呼び出すのではなく、mod_ssl の設計思想に合わせて抽象化されたインターフェースを提供します。主に証明書操作、秘密鍵の読み込み、セッション管理、および SSL シャットダウンの補助を行います。

責務

  • **OpenSSL 抽象化**: OpenSSL の複雑なデータ構造（X509, EVP_PKEY 等）を扱うための簡略化されたラッパー関数の提供。
  • **リソース管理**: BIO や証明書関連のメモリ確保・解放を apr_pool_t と統合し、メモリリークを防ぐ。
  • **フォーマット変換**: PEM/DER 形式の相互変換や、証明書情報の文字列化。

重要なシンボル

  • modssl_read_privatekey: PEM または DER 形式の秘密鍵を柔軟に読み込む関数。
  • modssl_smart_shutdown: SSL 接続を安全かつ確実に終了させるためのステートマシン制御関数。
  • modssl_init_app_data2_idx: OpenSSL の ex_data インデックスを初期化し、アプリケーション固有データを SSL オブジェクトに紐付けるための基盤。
  • MODSSL_SESSION_MAX_DER: DER エンコードされたセッションの最大サイズ定義。

制御フローとデータフロー

本ファイルはインターフェース定義のみですが、実装（ssl_util_ssl.c）は以下のフローを支えます。 1. **初期化**: modssl_init_app_data2_idx により、プロセス起動時に ex_data インデックスを確保。 2. **鍵読み込み**: modssl_read_privatekey がファイルシステムから BIO を介して鍵を読み込み、PEM/DER の自動判別を行う。 3. **証明書処理**: modssl_read_cert や modssl_cert_get_pem を通じて、証明書チェーンの構築や PEM 文字列へのシリアライズを行う。

直接依存

  • openssl/ssl.h (OpenSSL ライブラリ)
  • apr_pools.h (Apache Portable Runtime のメモリ管理)
  • httpd.h (Apache HTTP Server の基本定義)

関連コード

  • modules/ssl/ssl_util_ssl.c: 本ヘッダーで宣言された関数の実装。
  • modules/ssl/ssl_private.h: モジュール全体で共有される内部データ構造や型定義。
  • modules/ssl/ssl_engine_init.c: modssl_init_app_data2_idx が呼び出される初期化フェーズのコード。

読者向けメモ

  • **スレッドセーフ**: app_data2_idx の初期化は起動時に一度だけ行われるため、リクエスト処理中のスレッドセーフ性は確保されています。
  • **OpenSSL バージョン**: MODSSL_LIBRARY_DYNTEXT マクロは、OpenSSL 1.1 以前とそれ以降の API 差異を吸収するために使用されています。
  • **FIXME**: MODSSL_SESSION_MAX_DER の値は OpenSSL の内部実装に依存しており、将来的な変更の可能性がある点に注意してください。
