modules/ssl/ssl_util.c

概要

ssl_util.c は、mod_ssl モジュール全体で使用される汎用的なユーティリティ関数群を提供します。仮想ホストの識別、外部プロセスとのパイプ通信、ファイルパスの検証、ASN.1 データのキャッシュ管理、および OpenSSL 1.1 未満の環境におけるスレッドセーフなロック機構の実装が含まれます。

モジュール内での役割

mod_ssl の各エンジン（設定、IO、初期化）が共通して利用する「低レイヤーの補助機能」を提供します。特に、OpenSSL のレガシーなスレッド管理や、Apache のリクエスト処理と SSL 接続状態の橋渡しを行うためのユーティリティとして機能します。

責務

  • **仮想ホスト管理**: サーバー名とポートに基づく識別子の生成およびマッチング。
  • **プロセス間通信**: 外部コマンドの実行とパイプ経由のデータ読み取り。
  • **ファイルシステム検証**: パスが存在するか、ディレクトリか、ファイルか等の属性チェック。
  • **ASN.1 キャッシュ**: EVP_PKEY 構造体を DER 形式でハッシュテーブルに永続化・管理。
  • **スレッド同期**: OpenSSL の古い API 用のロックコールバックおよびスレッド ID 管理。

重要なシンボル

仮想ホスト・リクエスト識別

  • ssl_util_vhostid: サーバー名とポートから一意な ID 文字列を生成。
  • ssl_util_vhost_matches: ServerName および ServerAlias (ワイルドカード含む) との照合。
  • modssl_request_is_tls: 現在のリクエストが TLS 経由か判定し、接続コンテキストを返す。

プロセス・ファイル操作

  • ssl_util_ppopen / ssl_util_ppclose: 外部コマンドのパイプ開閉。
  • ssl_util_readfilter: 外部コマンドを実行し、その出力を1行読み取る。
  • ssl_util_path_check: apr_stat を用いたパスの属性検証。

ASN.1 キャッシュ管理

  • ssl_asn1_table_set / get / unset: EVP_PKEY を DER 形式でハッシュテーブルに格納・取得・削除。

スレッド同期 (OpenSSL 1.1 未満)

  • ssl_util_thread_setup: OpenSSL のロックコールバックを登録。
  • ssl_dyn_create_function 等: OpenSSL が内部で使用する動的ロックの管理。

制御フローとデータフロー

  • **ASN.1 キャッシュ**: ssl_asn1_table_set は i2d_PrivateKey を使用して EVP_PKEY を DER に変換し、プロセス生存期間中保持するために malloc でメモリを確保します。
  • **スレッドロック**: ssl_util_thread_setup は apr_pool_cleanup_register を使用して、モジュール終了時に OpenSSL のコールバックを解除するクリーンアップハンドラを登録します。
  • **外部コマンド**: ssl_util_readfilter は ssl_util_ppopen で生成されたパイプから1バイトずつ読み込み、改行コードまでをバッファに蓄積します。

直接依存

  • ssl_private.h: モジュール内部の構造体定義 (SSLSrvConfigRec, SSLConnRec 等)。
  • ap_mpm.h: MPM 固有の機能へのアクセス。
  • apr_thread_mutex.h: APR レベルのスレッド同期プリミティブ。

関連コード

  • ssl_engine_init.c: ssl_util_thread_setup を呼び出し、初期化時にスレッド環境を構築する。
  • ssl_engine_io.c: modssl_request_is_tls を利用して接続の SSL 状態を確認する。
  • ssl_util_ssl.c: OpenSSL の抽象化レイヤー。

読者向けメモ

  • **メモリ所有権**: ssl_asn1_table_set で確保される cpData は、プールではなく malloc で確保されます。これはモジュールのリロードを跨いでデータを保持するためです。ssl_asn1_table_unset で適切に free する必要があります。
  • **スレッドセーフ**: スレッド関連のコードは MODSSL_USE_OPENSSL_PRE_1_1_API が定義されている場合のみ有効です。OpenSSL 1.1 以降ではこれらのコールバックは不要であり、無視されます。
  • **制限事項**: ssl_util_readfilter は MAX_STRING_LEN を超える出力に対しては切り詰めが発生します。また、1バイトずつの読み込みはパフォーマンス上のボトルネックとなる可能性があります。
