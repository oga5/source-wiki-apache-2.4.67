modules/ssl/mod_ssl.h

概要

mod_ssl.h は、Apache HTTP Server の SSL/TLS モジュール（mod_ssl）が提供する公開インターフェースを定義するヘッダーファイルです。他のモジュールが mod_ssl の機能を利用するための API やフックを公開する役割を担います。

モジュール内での役割

mod_ssl サブシステムにおける「パブリック API」の窓口です。内部実装の詳細（ssl_private.h など）を隠蔽し、外部モジュール（mod_proxy など）が SSL 関連の情報を取得したり、接続の SSL 状態を制御したりするための標準的な手段を提供します。

責務

  • **オプション関数の公開**: APR_DECLARE_OPTIONAL_FN を使用し、実行時に他のモジュールから呼び出し可能な SSL ユーティリティ関数を定義する。
  • **外部フックの定義**: APR_DECLARE_EXTERNAL_HOOK を通じて、他のモジュールが証明書や鍵の追加といったイベントに介入するための拡張ポイントを提供する。
  • **プラットフォーム抽象化**: SSL_DECLARE マクロ群により、Windows 等のプラットフォーム間でのシンボルエクスポートの差異を吸収する。

重要なシンボル

  • ssl_var_lookup: SSL 環境変数（証明書情報や暗号スイート名など）を取得するための汎用関数。
  • ssl_is_https: 指定した接続が SSL/TLS を使用しているかを判定する。
  • ssl_ext_list: X.509 証明書の拡張フィールドを解析し、配列として取得する。
  • SSL_CERT_HOOKS: 証明書や鍵の動的な追加を可能にするフック群を有効化する定義。

制御フローとデータフロー

このファイル自体はロジックを含みません。外部モジュールが APR_RETRIEVE_OPTIONAL_FN を使用してこのヘッダーで宣言された関数を取得し、実行時に mod_ssl の内部関数へディスパッチする仕組みです。データフローとしては、conn_rec や request_rec を介して接続情報が渡され、結果として SSL 状態や証明書データが返されます。

直接依存

  • httpd.h: Apache のコア構造体（conn_rec, server_rec 等）の定義。
  • http_config.h: 設定関連の型定義。
  • apr_optional.h: オプション関数メカニズムの提供。
  • apr_tables.h: apr_array_header_t 等のデータ構造。

関連コード

  • modules/ssl/mod_ssl.c: このヘッダーで宣言された関数の実装およびモジュールのメインエントリポイント。
  • modules/ssl/ssl_private.h: mod_ssl 内部でのみ使用されるプライベートな構造体や関数定義。
  • modules/ssl/ssl_engine_vars.c: ssl_var_lookup の具体的な実装ロジック。

読者向けメモ

  • 本ファイルは「公開 API」であるため、ここに含まれる関数は安定性が高く、他のモジュール開発者が利用すべきインターフェースです。
  • SSL_DECLARE マクロは、静的リンクと動的リンクの切り替えをサポートするためのもので、モジュール開発時には意識する必要はありませんが、移植性を理解する上で重要です。
  • 外部モジュールから mod_ssl の機能を利用する場合は、まず APR_RETRIEVE_OPTIONAL_FN で関数ポインタを取得する手順が必要になる点に注意してください。
