modules/ssl/ssl_engine_io.c

概要

ssl_engine_io.c は、Apache HTTP Server のフィルタチェーンと OpenSSL の BIO (Basic I/O) 抽象化レイヤーを接続するブリッジです。HTTPD の apr_bucket_brigade を用いて暗号化・復号化されたデータを効率的に転送し、SSL/TLS のハンドシェイク、データ読み書き、および接続終了を管理します。

モジュール内での役割

mod_ssl の I/O サブシステムの中核であり、HTTPD の接続フィルタとして動作します。OpenSSL がネットワークソケットを直接操作する代わりに、このファイルで定義されたカスタム BIO を通じて HTTPD のフィルタスタックを読み書きします。

責務

  • **SSL フィルタ管理**: 入力 (ssl_io_filter_input) および出力 (ssl_io_filter_output) フィルタの提供。
  • **BIO 抽象化**: OpenSSL が期待する BIO_METHOD を実装し、HTTPD のバケット操作と統合。
  • **データバッファリング**: char_buffer_t による読み取りデータのキャッシュと、coalesce_ctx による出力データの結合（TLS レコード効率化）。
  • **ハンドシェイク制御**: ssl_io_filter_handshake による接続開始時のネゴシエーションと、プロキシ接続時の検証。
  • **エラーハンドリング**: HTTP-over-HTTPS 検出や SSL エラーのログ出力。

重要なシンボル

コンテキスト構造体

  • ssl_filter_ctx_t: SSL 接続のメインコンテキスト。SSL オブジェクト、BIO、フィルタへのポインタを保持。
  • bio_filter_in_ctx_t: 入力フィルタ用コンテキスト。復号化前のバッファと apr_bucket_brigade を管理。
  • bio_filter_out_ctx_t: 出力フィルタ用コンテキスト。暗号化前のデータを保持するブリゲードを管理。

BIO メソッド (OpenSSL 連携)

  • bio_filter_in_read / bio_filter_out_write: OpenSSL からの読み書き要求を HTTPD フィルタへ転送するコア関数。
  • bio_filter_create / bio_filter_destroy: BIO のライフサイクル管理。

フィルタ関数

  • ssl_io_filter_input: 入力フィルタ。SSL_read を呼び出し、復号化されたデータを HTTPD に渡す。
  • ssl_io_filter_output: 出力フィルタ。SSL_write を呼び出し、暗号化データを下位フィルタへ渡す。
  • ssl_io_filter_coalesce: 出力データを結合し、TLS レコードの断片化を抑制する最適化フィルタ。

制御フローとデータフロー

1. **入力**: ssl_io_filter_input が呼ばれると、SSL_read が実行される。SSL_read は bio_filter_in_read を呼び出し、HTTPD の ap_get_brigade でネットワークからデータを取得。復号化後、呼び出し元へ戻る。 2. **出力**: ssl_io_filter_output が呼ばれると、データは SSL_write を経由。SSL_write は bio_filter_out_write を呼び出し、暗号化されたデータを apr_bucket_brigade に格納して ap_pass_brigade で下位へ送る。 3. **バッファリング**: char_buffer_t は AP_MODE_GETLINE 等で発生する「読みすぎたデータ」を保持し、次回の読み取り時に再利用する。

直接依存

  • ssl_private.h: mod_ssl 内部の共有定義と構造体。
  • apr_date.h: 日付処理ユーティリティ。
  • OpenSSL BIO API: BIO_new, BIO_meth_new 等。

関連コード

  • ssl_engine_init.c: フィルタの初期化と設定の適用。
  • ssl_private.h: SSLConnRec 等の接続状態管理構造体の定義。
  • mod_ssl.c: モジュールのエントリポイントとフック登録。

読者向けメモ

  • **メモリ所有権**: apr_bucket_brigade は conn_rec のプールに関連付けられており、フィルタのライフサイクルと同期します。
  • **レイアウト**: bio_filter_in_ctx_t と bio_filter_out_ctx_t は、OpenSSL の BIO データ領域に BIO_set_data で格納されます。
  • **パフォーマンス**: ssl_io_filter_coalesce は、小さなバケットを結合して TLS レコード数を減らすための重要な最適化です。
  • **制約**: AP_IOBUFSIZE (8192バイト) が読み取りの最大単位としてハードコードされています。
  • **省略**: 本ページでは、デバッグ用のダンプ関数 (ssl_io_data_dump) の詳細な実装ロジックは省略しています。これらは主にトレースログ出力用であり、メインの I/O フローには影響しません。
