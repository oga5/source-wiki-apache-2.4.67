modules/ssl/ssl_engine_mutex.c

概要

ssl_engine_mutex.c は、Apache HTTP Server の mod_ssl におけるセッションキャッシュ保護のための相互排他（Mutex）制御を実装するソースファイルです。マルチプロセス/マルチスレッド環境下で、共有されるセッションキャッシュへの安全なアクセスを保証します。

モジュール内での役割

mod_ssl のセッションキャッシュサブシステム（ssl_scache.c 等）を補完するユーティリティ層です。キャッシュプロバイダがスレッドセーフまたはプロセスセーフでない場合に、排他制御を透過的に提供します。

責務

  • **Mutex の初期化**: サーバー起動時にグローバル Mutex を作成し、キャッシュの保護が必要か判定する。
  • **子プロセスでの再初期化**: フォーク後の子プロセスで Mutex を再接続し、ロックを有効化する。
  • **排他制御の実行**: セッションキャッシュへの読み書き時に、ロックの取得（on）と解放（off）を行う。

重要なシンボル

  • ssl_mutex_init: サーバー起動時に Mutex を作成する。キャッシュプロバイダのフラグを確認し、不要な場合は作成をスキップする。
  • ssl_mutex_reinit: 子プロセス生成後に Mutex を再初期化する。
  • ssl_mutex_on / ssl_mutex_off: セッションキャッシュ操作の前後で呼び出されるラッパー関数。

制御フローとデータフロー

1. **初期化**: ssl_mutex_init が SSLModConfigRec を参照し、キャッシュプロバイダの AP_SOCACHE_FLAG_NOTMPSAFE フラグをチェックします。必要であれば ap_global_mutex_create で Mutex を生成します。 2. **実行時**: セッションキャッシュへのアクセスが必要な箇所で ssl_mutex_on を呼び出し、apr_global_mutex_lock を実行します。処理終了後に ssl_mutex_off で apr_global_mutex_unlock を呼び出します。 3. **エラーハンドリング**: ロックの取得・解放に失敗した場合、ap_log_error を通じてログを出力し、呼び出し元に失敗を伝播させます。

直接依存

  • ssl_private.h: SSLModConfigRec 構造体や SSL_CACHE_MUTEX_TYPE 定義など、モジュール内の共通定義に依存しています。

関連コード

  • ssl_scache.c: このファイルで管理される Mutex を実際に利用してキャッシュ操作を行う主要なコンポーネントです。
  • apr_global_mutex: Apache Portable Runtime (APR) が提供するクロスプラットフォームな Mutex 実装です。

読者向けメモ

  • **条件付き実行**: すべてのキャッシュプロバイダで Mutex が使用されるわけではありません。プロバイダ自体がスレッドセーフである場合、このモジュールの関数は実質的に何もしない（あるいは初期化をスキップする）設計になっています。
  • **デバッグ**: ロックの取得失敗は、多くの場合、OS のリソース制限や権限設定に起因します。ログ出力される APLOGNO を確認してください。
