modules/ssl/ssl_util_ssl.c

概要

OpenSSL ライブラリに対する高レベルなラッパー関数群を提供するユーティリティモジュールです。証明書操作、秘密鍵の読み込み、セッション管理、および SSL 接続の制御に関連する定型処理を抽象化し、mod_ssl の他のコンポーネントが OpenSSL の複雑な API を直接扱う負担を軽減します。

モジュール内での役割

mod_ssl サブシステムにおいて、OpenSSL の低レイヤー API と Apache HTTP Server のデータ構造（apr_pool_t など）を橋渡しする役割を担います。ssl_private.h を通じて共有定義を参照し、エンジン層（ssl_engine_*.c）から呼び出される補助的なロジックを提供します。

責務

  • **SSL 接続管理**: アプリケーションデータ用インデックスの管理およびスマートシャットダウン処理。
  • **証明書・鍵の操作**: PEM/DER 形式の鍵読み込み、証明書情報の抽出（SAN, Basic Constraints）、および証明書と鍵のペアリング。
  • **名前照合**: RFC 6125 に基づく DNS-ID/CN-ID の抽出とワイルドカード対応のホスト名照合。
  • **ASN.1 変換**: OpenSSL の ASN.1 構造体から Apache のメモリ管理下にある文字列への変換。
  • **パラメータ読み込み**: DH パラメータや EC グループのファイルからのロード。

重要なシンボル

アプリケーションデータ管理

  • modssl_init_app_data2_idx: SSL オブジェクトに付与する拡張データ用インデックスを初期化。
  • modssl_get_app_data2 / modssl_set_app_data2: SSL オブジェクトに関連付けられたカスタムデータを取得・設定。

証明書・鍵ユーティリティ

  • modssl_read_privatekey: PEM/DER 形式を自動判別して秘密鍵をロード。
  • modssl_read_cert: メモリバッファから証明書と鍵を読み込み、apr_pool_t に紐づくリソースとして管理。
  • modssl_X509_getSAN: 証明書の Subject Alternative Name を抽出。
  • modssl_X509_match_name: ホスト名と証明書内の ID（DNS-ID/CN-ID）を照合。

セッション・低レイヤー制御

  • modssl_smart_shutdown: SSL シャットダウンシーケンスを安全に実行するための再試行ロジック。
  • modssl_SSL_SESSION_id2sz: セッション ID を 16 進数文字列に変換。

制御フローとデータフロー

  • **メモリ管理**: 多くの関数が apr_pool_t を引数に取り、OpenSSL の BIO を介して生成されたデータを Apache のプールメモリにコピーして返します。modssl_bio_free_read はこのパターンの中心的なヘルパーであり、BIO の内容を読み取った後に必ず BIO_free を実行してリークを防ぎます。
  • **シャットダウンフロー**: modssl_smart_shutdown は、SSL_shutdown を最大 4 回呼び出し、SSL_SENT_SHUTDOWN フラグを確認しながらソケットのフラッシュを試みることで、不完全なシャットダウンによるハングを防ぎます。
  • **証明書照合フロー**: modssl_X509_match_name は内部で getIDs を呼び出し、SAN と CN を配列に集約した後、ワイルドカード判定を含む比較ロジックを適用します。

直接依存

  • ssl_private.h: モジュール内の共通定義および SSLConnRec 構造体へのアクセス。
  • openssl/ssl.h, openssl/x509.h 等: OpenSSL のコア API。

関連コード

  • ssl_util_ssl.h: 本ファイルの公開インターフェース定義。
  • ssl_engine_io.c: SSL フィルタリング処理において、本ファイルのシャットダウンやデータ変換関数が利用される。
  • ssl_engine_init.c: 証明書や鍵の初期ロード時に本ファイルのユーティリティが利用される。

読者向けメモ

  • **メモリ所有権**: modssl_bio_free_read を含む多くの関数は、OpenSSL が確保したメモリを解放し、Apache のプールにコピーした結果を返します。呼び出し側は返された文字列の解放を気にする必要はありません。
  • **スレッド安全性**: app_data2_idx は起動時に一度だけ初期化されるため、マルチスレッド環境でも安全です。
  • **OpenSSL バージョン**: OPENSSL_VERSION_NUMBER による条件分岐が随所にあり、OpenSSL 1.1.x/3.0.x 間の API 差異を吸収しています。
  • **省略された詳細**: 本ファイルはユーティリティ集であるため、特定のビジネスロジックは含みません。複雑な証明書検証ロジックや OCSP 関連の詳細は、それぞれ ssl_engine_*.c や ssl_util_ocsp.c に委譲されています。
