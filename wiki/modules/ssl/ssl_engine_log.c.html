modules/ssl/ssl_engine_log.c

概要

ssl_engine_log.c は、mod_ssl における診断情報の出力とエラーハンドリングを抽象化するロギング施設です。OpenSSL のエラーキューを Apache のログシステムへ統合し、特に X.509 証明書に関連する複雑なエラーメッセージを整形して出力する機能を提供します。

モジュール内での役割

mod_ssl の他のコンポーネントが OpenSSL の低レベルなエラーを直接扱うのではなく、本ファイルの関数を介してログを出力することで、一貫したエラーフォーマットと、一般的なエラーに対するヒント（アノテーション）の付与を実現しています。

責務

  • **OpenSSL エラーの変換**: OpenSSL のエラーキューを読み取り、Apache のログレベルに合わせて整形・出力する。
  • **エラーアノテーション**: よくある SSL エラーパターンに対し、原因究明に役立つ補足説明を自動付与する。
  • **証明書情報の詳細出力**: X.509 証明書に関連するエラー発生時に、サブジェクト、発行者、シリアル番号、有効期限などのメタデータをログに埋め込む。
  • **ログ出力の抽象化**: server_rec, conn_rec, request_rec の各コンテキストに応じた適切なログ出力関数の提供。

重要なシンボル

エラーアノテーション

  • ssl_log_annotate: エラー文字列のパターンと、それに対応する説明文を保持する静的配列。
  • ssl_log_annotation(): エラー文字列を受け取り、一致するアノテーションを返すヘルパー。

OpenSSL エラー処理

  • modssl_ERR_peek_error_data(): OpenSSL バージョン差異（1.1.x vs 3.0+）を吸収し、エラー情報を取得するインライン関数。
  • ssl_log_ssl_error(): OpenSSL エラーキューを走査し、アノテーションを付与してログ出力するメイン関数。

証明書エラーロギング

  • ssl_log_cert_error(): 証明書情報を BIO を介して文字列化し、ログメッセージに結合する内部関数。
  • ssl_log_xerror, ssl_log_cxerror, ssl_log_rxerror: それぞれ server, connection, request コンテキストに対応した公開ラッパー。

制御フローとデータフロー

1. **SSL エラー出力**: ssl_log_ssl_error が modssl_ERR_peek_error_data をループで呼び出し、OpenSSL のエラーキューを空になるまで処理します。各エラーに対し ssl_log_annotation でヒントを検索し、ap_log_error で出力します。 2. **証明書エラー出力**: ssl_log_xerror 等のラッパーが va_list を作成し、ssl_log_cert_error を呼び出します。 3. **証明書情報の整形**: ssl_log_cert_error 内で BIO_s_mem() を使用し、X.509 証明書の各フィールドを一時的なメモリバッファに書き出します。その後、元のメッセージと結合して ap_log_*error に渡します。

直接依存

  • ssl_private.h: mod_ssl の内部構造体定義および modssl_X509_NAME_to_string などのユーティリティ関数へのアクセス。

関連コード

  • ssl_util_ssl.c: 証明書情報の文字列化（modssl_X509_NAME_to_string）など、証明書操作の基盤。
  • mod_ssl.c: モジュールの初期化と設定。

読者向けメモ

  • **メモリ管理**: ssl_log_cert_error 内での証明書情報の文字列化には、呼び出し元から渡された apr_pool_t を使用します。BIO は関数内で生成・解放されます。
  • **OpenSSL 3.0 対応**: modssl_ERR_peek_error_data は OpenSSL 3.0 での API 変更を隠蔽するための重要な互換レイヤーです。
  • **制限事項**: ssl_log_cert_error 内の maxdnlen 計算は、ログ行の最大長（HUGE_STRING_LEN）を考慮した安全な設計になっています。
  • **省略された詳細**: 本ファイルはロギングに特化しており、SSL ハンドシェイクや暗号化通信のロジックは含みません。また、ログ出力のパフォーマンスへの影響を最小限にするため、APLOG_IS_LEVEL による事前チェックが行われています。
