modules/database/mod_dbd.h

概要

mod_dbd.h は、Apache HTTP Server におけるデータベース接続管理モジュール（mod_dbd）のパブリックインターフェースを定義するヘッダーファイルです。他のモジュールがデータベース接続を取得・管理するための API を提供します。

モジュール内での役割

mod_dbd サブシステムのフロントエンドとして機能します。このヘッダーは、接続プール管理や SQL ステートメントの準備といった機能を、他の Apache モジュールから利用可能にするための契約（コントラクト）を定義しています。

責務

  • データベース接続設定構造体（dbd_cfg_t）の定義。
  • 接続ハンドル（ap_dbd_t）の定義。
  • 接続の取得（acquire）、解放（close）、および準備済みステートメントの登録を行う API の公開。
  • 他モジュールが動的に連携するためのオプション関数（Optional Functions）の宣言。

重要なシンボル

  • dbd_cfg_t: サーバー単位のデータベース設定（ドライバ名、接続パラメータ、プールサイズ等）を保持する構造体。
  • ap_dbd_t: 実際のデータベース接続ハンドルとドライバ、準備済みステートメントを保持する構造体。
  • ap_dbd_acquire(): リクエストのライフサイクルに紐付いた接続を取得する推奨関数。
  • ap_dbd_prepare(): サーバー起動時に SQL ステートメントを事前準備するための関数。

制御フローとデータフロー

本ファイルはインターフェース定義であるため、直接的なロジックは含みません。 1. **設定フェーズ**: dbd_cfg_t を通じて設定が読み込まれます。 2. **接続取得**: ap_dbd_acquire 等が呼ばれると、mod_dbd.c 側の接続プール管理ロジックが起動し、適切な接続が返されます。 3. **フック**: post_connect フックにより、接続確立直後の初期化処理を外部モジュールが介入できるようになっています。

直接依存

  • httpd.h: Apache コア構造体へのアクセス。
  • apr_optional.h: オプション関数機能の利用。
  • apr_hash.h: クエリ管理用ハッシュテーブル。
  • apr_hooks.h: フック定義用。

関連コード

  • modules/database/mod_dbd.c: 本ヘッダーで宣言された API の実装および接続プールの実体。
  • apr_dbd.h: APR が提供する低レベルデータベース抽象化レイヤー。mod_dbd はこの上に構築されています。

読者向けメモ

  • **メモリ管理**: ap_dbd_acquire で取得した接続はリクエストのライフサイクルに依存するため、明示的なクローズは不要です。一方、ap_dbd_open を使用した場合は呼び出し側で ap_dbd_close を呼ぶ責任があります。
  • **スレッドセーフ**: APR_HAS_THREADS が有効な環境では、接続プール（apr_reslist）が自動的にスレッドセーフな接続管理を行います。
  • **拡張性**: APR_DECLARE_OPTIONAL_FN を使用しているため、mod_dbd がロードされていない環境でも、他のモジュールはリンクエラーなしに安全に接続関数を呼び出すことが可能です。
