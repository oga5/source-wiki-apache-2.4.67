modules/filters/mod_reqtimeout.c

概要

mod_reqtimeout は、Apache HTTP Server におけるリクエスト処理の各段階（TLSハンドシェイク、ヘッダー読み取り、ボディ読み取り）に対して、動的なタイムアウト制限を適用するフィルタモジュールです。低速なクライアントによるリソース枯渇攻撃（Slowloris 等）を防止することを目的としています。

モジュール内での役割

本モジュールは AP_FTYPE_CONNECTION レベルで動作する入出力フィルタとして実装されています。接続のライフサイクルに介入し、設定されたタイムアウト値に基づいてソケットの読み取り時間を監視・制御します。

責務

  • **設定管理**: RequestReadTimeout ディレクティブによる各段階のタイムアウト値および最小データ転送レートの定義。
  • **接続監視**: 入力フィルタ reqtimeout_filter を通じて、データ受信の進捗を監視し、タイムアウト超過時に接続を強制終了する。
  • **動的調整**: 受信したデータ量に応じてタイムアウト期限を延長する（最小レートが設定されている場合）。
  • **ライフサイクル制御**: pre_read_request および post_read_request フックを利用し、リクエストの各フェーズに応じたタイムアウト設定の切り替え。

重要なシンボル

設定構造体

  • reqtimeout_srv_cfg: サーバーごとの設定を保持。ハンドシェイク、ヘッダー、ボディの各段階の閾値を管理。
  • reqtimeout_con_cfg: 接続ごとの状態を保持。現在のタイムアウト期限、ソケット、一時バッファ（tmpbb）を格納。

フィルタ・フック関数

  • reqtimeout_filter: メインの入力フィルタ。ソケットのタイムアウト値を動的に変更し、期限切れをチェック。
  • reqtimeout_eor: 出力フィルタ。EOR（End of Request）バケットを検知し、リクエスト間のタイムアウト状態をリセット。
  • reqtimeout_init: 接続初期化時にフィルタを挿入。

ユーティリティ

  • extend_timeout: 受信データ量に基づき、timeout_at を延長する。
  • check_time_left: 現在時刻と期限を比較し、残り時間を算出。

制御フローとデータフロー

1. **初期化**: reqtimeout_init が接続開始時に呼ばれ、入出力フィルタをスタックに挿入。 2. **フェーズ移行**: pre_read_request でヘッダー読み取り用、post_read_request でボディ読み取り用の設定を reqtimeout_con_cfg にロード。 3. **データ読み取り**: reqtimeout_filter が ap_get_brigade をラップ。

  • AP_MODE_GETLINE の場合、非ブロッキング読み取りと apr_poll を組み合わせて、タイムアウトを厳密に監視しながら行単位でデータを取得。
  • 読み取り成功時、extend_timeout がデータ量に応じて期限を延長。

    • 4. **タイムアウト発生**: check_time_left が APR_TIMEUP を返すと、接続を AP_CONN_CLOSE に設定し、short-lingering-close を有効化して接続を即時切断。

関連コード

  • mod_ssl.c: ハンドシェイク段階のタイムアウト制御において、本モジュールが先行して動作する必要がある（AP_FTYPE_CONNECTION + 8）。
  • http_protocol.c: ap_get_brigade の呼び出し元であり、本フィルタの動作対象。

読者向けメモ

  • **メモリ管理**: reqtimeout_con_cfg は接続プール（c->pool）に割り当てられ、接続終了まで生存します。tmpbb は行読み取り時のバッファとして使用されます。
  • **エッジケース**: AP_MODE_GETLINE 以外のモード（AP_MODE_SPECULATIVE 等）では、タイムアウト延長を行わない設計になっています。
  • **制約**: APR_MAJOR_VERSION < 2 の場合、apr_poll を使用してソケットを監視しますが、それ以降のバージョンでは apr_socket_wait を使用するよう抽象化されています。
  • **未実装/浅い部分**: 本モジュールは HTTP プロトコルに特化しており、他のプロトコル（FTP等）での挙動は reqtimeout_init 内の判定により無効化されるよう設計されています。
