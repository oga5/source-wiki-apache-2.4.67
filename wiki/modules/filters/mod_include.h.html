modules/filters/mod_include.h

概要

mod_include.h は、Apache HTTP Server の Server Side Include (SSI) 機能を拡張・制御するためのパブリックインターフェースを定義するヘッダーファイルです。SSI フィルタの動作をカスタマイズするための定数、フラグ、およびコンテキスト構造体を提供します。

モジュール内での役割

mod_include.c と対を成し、SSI 処理のコアロジックと外部モジュール間の橋渡しを行います。他のモジュールが SSI の解析やタグ処理に介入するための API を公開する役割を担います。

責務

  • SSI 処理の状態を保持する include_ctx_t 構造体の定義。
  • SSI タグの解析や文字列展開のための API（APR_DECLARE_OPTIONAL_FN）の提供。
  • SSI フィルタの挙動（出力制御、条件分岐、エラーハンドリング）を制御するフラグの定義。

重要なシンボル

  • include_ctx_t: SSI 処理の現在の状態（フラグ、プール、リクエスト情報）を保持する主要な構造体。
  • SSI_FLAG_*: フィルタの出力状態や条件分岐の評価結果を管理するビットフラグ。
  • ap_register_include_handler: 外部モジュールが独自の SSI タグハンドラを登録するための関数。
  • ap_ssi_parse_string: SSI 変数展開を行うためのユーティリティ関数。

制御フローとデータフロー

SSI フィルタは include_ctx_t を介してリクエストのライフサイクルを管理します。 1. **初期化**: include_ctx_t が生成され、リクエスト情報が格納されます。 2. **解析**: ap_ssi_get_tag_and_value 等を用いて入力ストリームからタグを抽出します。 3. **実行**: 登録されたハンドラが include_ctx_t を参照し、条件分岐や変数展開を行い、結果を apr_bucket_brigade に挿入します。 4. **エラー処理**: SSI_CREATE_ERROR_BUCKET マクロにより、解析エラー発生時にエラーメッセージをバケットとして出力ストリームへ注入します。

直接依存

  • apr_pools.h: メモリ管理（コンテキスト内のプール用）。
  • apr_optional.h: 外部モジュールから利用可能なオプション関数の宣言用。

関連コード

  • modules/filters/mod_include.c: 本ヘッダーで定義された構造体や関数の実装本体。
  • httpd.h: request_rec 等の Apache 基本構造体。
  • util_filter.h: フィルタチェーンの制御に関する定義。

読者向けメモ

  • include_ctx_t 内の intern メンバは非公開データであり、直接操作してはなりません。
  • SSI_FLAG_ 定数は、条件付き出力（<!--#if ... -->）のネスト状態や、実行可否を制御するために重要です。
  • 外部から SSI 機能を拡張する場合は、ap_register_include_handler を使用して独自のタグハンドラを登録するのが標準的なアプローチです。
