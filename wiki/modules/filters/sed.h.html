modules/filters/sed.h

概要

sed.h は、Apache HTTP Server のフィルタモジュール群で使用される sed エンジンの定数定義および内部コマンドコードを定義するヘッダーファイルです。主に sed0.c や sed1.c で実装されるストリームエディタのロジックにおいて、コマンドの識別やバッファサイズの制約を規定します。

モジュール内での役割

modules/filters における sed 実装の基盤層に位置します。このファイルは、libsed.h が提供する高レベルな API と、実際の処理を行う sed0.c/sed1.c との間で共有される「言語仕様（コマンドセットと定数）」を定義する役割を担います。

責務

  • **コマンド識別子の定義**: sed スクリプトの各命令（a, b, c, s 等）を 8 進数ベースの定数として定義し、パーサーと実行エンジン間の共通言語を提供します。
  • **メモリ制約の定義**: バッファサイズやリサイズ単位などの静的な制約を定義し、メモリ管理のベースラインを提供します。

重要なシンボル

コマンドコード (Command Codes)

sed の各命令を識別するための定数群です。

  • ACOM, BCOM, CCOM, DCOM, SCOM, WCOM 等: sed の標準的なコマンド（Append, Branch, Change, Delete, Substitute, Write など）に対応する識別子。
  • CDCOM, CNCOM, CGCOM 等: 拡張または特定の条件付きコマンドを識別します。

メモリ・バッファ制約

  • LBSIZE: 行バッファ（Line Buffer）のデフォルトサイズ。
  • RESIZE: メモリ再割り当て時の増分単位。
  • CEND, CLNUM: 内部的な状態管理やアドレス指定に使用される定数。

制御フローとデータフロー

本ファイル自体はデータ構造や関数を含まない定数定義ヘッダーです。

  • **データフロー**: ここで定義された *COM 定数は、sed1.c 内のコマンドディスパッチテーブルや switch 文のケースラベルとして使用されます。
  • **制御フロー**: パーサーがスクリプトを解析する際、これらの定数をコマンド構造体に格納し、実行エンジンがその値を参照して処理を分岐させます。

直接依存

  • stdlib.h: 標準ライブラリの基本定義。
  • limits.h: システムの制限値定義。

関連コード

  • modules/filters/libsed.h: sed エンジンの公開 API 定義。
  • modules/filters/sed0.c: コマンドの解析とコンパイルを担当。
  • modules/filters/sed1.c: コンパイルされたコマンドの実行を担当。

読者向けメモ

  • **設計の性質**: 本ファイルは AT&T sed の実装を継承しており、定数定義が 8 進数で記述されている点に注意してください。
  • **メモリ管理**: LBSIZE や RESIZE は静的なデフォルト値ですが、実際の実行時には libsed のメモリ管理機構によって動的に拡張される可能性があります。
  • **省略された詳細**: 本ファイルにはロジックが含まれていないため、具体的なコマンドの挙動については sed1.c の実行ループを参照してください。
