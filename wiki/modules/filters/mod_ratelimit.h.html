modules/filters/mod_ratelimit.h

概要

mod_ratelimit.h は、Apache HTTP Server の出力フィルタモジュール mod_ratelimit が提供するカスタムバケット型を定義するヘッダーファイルです。このモジュールは、クライアントへのレスポンス送信速度を制限するために使用されます。

モジュール内での役割

mod_ratelimit は、出力フィルタチェーンに挿入され、レスポンスデータの転送レートを制御します。本ヘッダーは、フィルタ処理の境界（開始・終了）を明示するための特殊なバケット型を外部に公開し、他のモジュールやフィルタがこれらのマーカーを識別できるようにします。

責務

  • プラットフォーム依存のシンボルエクスポート制御（AP_RL_DECLARE 等）。
  • レート制限処理の開始と終了を示すカスタムバケット型の定義。
  • バケットが特定のレート制限マーカーであるかを判定するマクロの提供。

重要なシンボル

  • ap_rl_bucket_type_end / ap_rl_bucket_type_start: レート制限フィルタの範囲を示す特殊なバケット型。
  • AP_RL_BUCKET_IS_END(e) / AP_RL_BUCKET_IS_START(e): 指定されたバケットが上記のマーカーであるかを判定するマクロ。
  • ap_rl_end_create / ap_rl_start_create: フィルタチェーンに挿入するためのマーカーバケットを生成する関数。

制御フローとデータフロー

本ファイル自体は定義のみを提供します。実際のデータフローは mod_ratelimit.c 内の rate_limit_filter 関数が担います。 1. フィルタは subprocess_env から設定を読み取り、コンテキストを初期化します。 2. 処理対象のバケットブリゲード内に ap_rl_bucket_type_start や end が含まれる場合、フィルタはそれらを検知してレート制限の適用状態を切り替えます。 3. 制限が必要な場合、バケットを一時的に holdingbb に保持し、計算されたレートに基づいて順次下流へ渡します。

直接依存

  • apr_buckets.h: Apache Portable Runtime (APR) のバケット操作機能に依存しています。

関連コード

  • modules/filters/mod_ratelimit.c: フィルタのロジック実装および本ヘッダーで定義されたバケット型の利用箇所。
  • util_filter.h: Apache のフィルタチェーンアーキテクチャの基盤。

読者向けメモ

  • **将来の展望**: ヘッダー内の TODO にある通り、現在は直接的な関数呼び出しに依存していますが、将来的には「オプション関数（Optional Functions）」へ移行し、モジュールのロード順序に依存しない設計を目指しています。
  • **利用上の注意**: このヘッダーで定義されているバケット型は、主に mod_ratelimit 内部での状態管理用です。他のモジュールからこれらを利用する場合は、フィルタチェーンの順序と、バケットが適切に処理されることを確認してください。
