modules/filters/mod_sed.c

概要

mod_sed.c は、Apache HTTP Server のリクエストおよびレスポンスストリームに対して、sed 形式の正規表現置換を適用するためのフィルタモジュールです。libsed ライブラリをバックエンドとして利用し、ストリーム処理中に動的なテキスト変換を実現します。

モジュール内での役割

HTTP フィルタチェーンの一部として動作し、OutputSed および InputSed ディレクティブを通じて設定された変換ルールを適用します。mod_substitute と類似していますが、より複雑な sed スクリプトの実行を可能にする点に特徴があります。

責務

  • **設定管理**: OutputSed / InputSed ディレクティブの解析と sed 式のコンパイル。
  • **フィルタコンテキスト管理**: リクエストごとの状態（sed_filter_ctxt）の初期化と破棄。
  • **ストリーム処理**: apr_bucket_brigade を介したデータの読み込み、libsed への引き渡し、および変換後のデータ再構築。
  • **エラーハンドリング**: libsed からのコンパイルエラーや実行時エラーのログ出力。

重要なシンボル

設定構造体

  • sed_config: ディレクトリ単位の設定を保持。input と output の sed_expr_config を含む。
  • sed_expr_config: 個別の sed コマンドセットと直近のエラーメッセージを保持。

フィルタコンテキスト

  • sed_filter_ctxt: フィルタ実行中の状態を保持。sed_eval_t（libsed の評価状態）、出力用バッファ、一時プール（tpool）、および処理中のブリゲードを管理。

コールバック関数

  • sed_write_output: libsed が変換結果を出力する際に呼び出される。バッファリングを行い、一定量に達すると apr_bucket を生成してブリゲードに追加する。
  • log_sed_errf / sed_compile_errf: libsed からのエラー通知を受け取り、Apache のログシステムへ転送する。

フィルタエントリポイント

  • sed_response_filter: 出力フィルタ。レスポンスボディを逐次処理し、変換結果を後続のフィルタへ渡す。
  • sed_request_filter: 入力フィルタ。リクエストボディを読み込み、変換後に呼び出し元へ返す。

制御フローとデータフロー

1. **初期化**: init_context がリクエストごとに呼ばれ、libsed の評価エンジンを初期化する。 2. **データ処理**:

  • sed_response_filter はブリゲード内のバケットを反復し、sed_eval_buffer を呼び出す。
  • sed_write_output は変換されたデータを ctx->outbuf に蓄積し、MODSED_OUTBUF_SIZE に達するか EOS/Flush バケットに遭遇した時点で append_bucket を通じてブリゲードへ書き出す。

    • 3. **終了処理**: sed_finalize_eval が EOS バケット受信時に呼ばれ、sed の内部バッファに残ったデータをフラッシュする。

直接依存

  • libsed.h: sed 評価エンジンのコア API。
  • util_filter.h, apr_buckets.h: Apache フィルタチェーンおよびデータブリゲードの操作。
  • http_config.h, http_log.h: モジュール設定およびログ出力。
  • apr_general.h
  • apr_strings.h

関連コード

  • sed0.c, sed1.c: libsed の実装本体。
  • mod_substitute.c: 類似の置換機能を提供するモジュール。

読者向けメモ

  • **メモリ管理**: sed_filter_ctxt 内の tpool は、一時的なバケット生成に使用されます。clear_ctxpool によって定期的にクリアされ、メモリリークを防いでいます。
  • **Solaris sed 互換性**: sed_finalize_eval 実行時に改行がない場合、自動的に改行を付与する挙動があります。これは libsed の仕様であり、POST データ処理時に意図しないデータ変更が発生する可能性があるため注意が必要です。
  • **制約**: sed_request_filter は AP_MODE_READBYTES モードのみをサポートしており、それ以外のモードは後続のフィルタへ透過的にパスします。
  • **省略箇所**: libsed 内部の正規表現エンジン（regexp.c 等）の詳細は本ファイルには含まれていません。これらは libsed の責務として抽象化されています。
