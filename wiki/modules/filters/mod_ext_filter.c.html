modules/filters/mod_ext_filter.c

概要

mod_ext_filter は、外部の Unix スタイルフィルタプログラム（外部プロセス）を Apache HTTP Server の入力/出力フィルタとして統合するためのモジュールです。HTTP リクエスト/レスポンスのボディをパイプ経由で外部プロセスに渡し、その出力を後続のフィルタやクライアントへ返します。

モジュール内での役割

Apache のフィルタチェーンの一部として動作し、ExtFilterDefine ディレクティブで定義された外部コマンドを、指定された MIME タイプや環境変数に基づいて動的に起動・管理します。

責務

  • **設定管理**: ExtFilterDefine および ExtFilterOptions によるフィルタの定義と、ディレクトリごとの動作設定（エラー時の挙動など）の保持。
  • **プロセス管理**: 外部フィルタプロセスの起動、環境変数の引き継ぎ、リソース制限の適用、および終了時のクリーンアップ。
  • **データストリーミング**: apr_bucket_brigade を介した Apache のバケットデータと外部プロセスの標準入出力間の橋渡し。
  • **フィルタ制御**: フィルタの適用条件（MIME タイプ、環境変数）の評価と、条件不一致時のバイパス（noop）。

重要なシンボル

設定構造体

  • ef_server_t: サーバー単位の設定。フィルタ定義を保持するハッシュテーブルを持つ。
  • ef_dir_t: ディレクトリ単位の設定。エラー時の挙動（onfail）やログ出力を制御。
  • ef_filter_t: フィルタの定義情報（コマンドパス、引数、入出力タイプ、モード）。
  • ef_ctx_t: リクエストごとの実行コンテキスト。プロセスハンドル、パイプ、バッファ状態を保持。

フィルタ処理関数

  • ef_output_filter: 出力フィルタのメインエントリ。
  • ef_input_filter: 入力フィルタのメインエントリ。
  • ef_unified_filter: 共通のフィルタ処理ロジック。バケットを読み込み、外部プロセスへ書き出し、結果を読み取る。

ライフサイクル・ユーティリティ

  • define_filter: ExtFilterDefine ディレクティブの解析とフィルタ登録。
  • init_ext_filter_process: 外部プロセスの起動とパイプの設定。
  • child_errfn: 外部プロセスで発生したエラーを Apache のエラーログへ転送するコールバック。

制御フローとデータフロー

1. **初期化**: init_filter_instance がリクエスト時に呼ばれ、フィルタ定義の検索と適用条件（noop フラグ）の判定を行う。 2. **プロセス起動**: 条件を満たす場合、init_ext_filter_process が apr_proc_create を呼び出し、標準入出力パイプを接続する。 3. **データ転送**: ef_unified_filter がバケットを順次読み込み、pass_data_to_filter を通じて外部プロセスの stdin に書き込む。 4. **結果取得**: drain_available_output または ef_unified_filter 内のループで、外部プロセスの stdout からデータを読み取り、新しいバケットとしてブリゲードに挿入する。 5. **終了**: EOS (End of Stream) バケットを検知すると、外部プロセスの stdin を閉じ、残りの出力を読み取って完了する。

読者向けメモ

  • **メモリ管理**: すべての動的メモリは request_rec のプール（r->pool）に紐付いており、リクエスト終了時に自動解放されます。
  • **プロセス生存期間**: 外部プロセスはリクエストの生存期間に依存します。apr_pool_note_subprocess により、リクエスト終了時にプロセスが確実に終了するよう設計されています。
  • **エッジケース**: APR_FILES_AS_SOCKETS が定義されている環境では apr_pollset を使用して非同期的な入出力を行いますが、そうでない場合は apr_sleep を用いたポーリングが行われるため、パフォーマンスに影響が出る可能性があります。
  • **制限事項**: 現在の設計では、フィルタの入出力タイプは単一の MIME タイプのみをサポートしています。また、外部プロセスがブロックした場合のハンドリングは、EAGAIN を検知してポーリングする実装になっています。
