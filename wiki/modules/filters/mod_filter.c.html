modules/filters/mod_filter.c

概要

mod_filter.c は、Apache HTTP Server における「スマートフィルタ」機能の中核を担うモジュールです。リクエストのコンテキスト（Content-Type や式評価）に基づいて、動的に適切なフィルタプロバイダを選択・適用する抽象化レイヤーを提供します。

モジュール内での役割

他のフィルタモジュール（mod_deflate など）が特定の処理を実装するのに対し、本モジュールはそれらのフィルタを「プロバイダ」として管理し、実行時に条件分岐を行う「ハーネス（Harness）」として機能します。

責務

  • **フィルタの抽象化**: 複数のプロバイダを単一のフィルタ名で管理し、実行時に条件判定を行う。
  • **条件付きディスパッチ**: ap_expr を用いた式評価、または Content-Type に基づくプロバイダの選択。
  • **プロトコル整合性の維持**: フィルタ適用による HTTP ヘッダ（ETag, Content-Length 等）の変更を自動的に追跡・調整する。
  • **設定管理**: FilterDeclare, FilterProvider, FilterChain などのディレクティブを通じたフィルタチェーンの構築とマージ。

重要なシンボル

データ構造

  • ap_filter_provider_t: プロバイダの連結リスト。条件式 (ap_expr_info_t) と対象フィルタ (ap_filter_rec_t) を保持。
  • harness_ctx: 実行時のコンテキスト。選択されたプロバイダの関数ポインタと、初期化時のコンテキストを保持。
  • mod_filter_cfg: モジュール設定。登録済みフィルタのハッシュと、適用順序を定義するチェーンを保持。

制御関数

  • filter_harness: フィルタチェーンの実行エントリポイント。条件判定 (filter_lookup) を行い、適切なプロバイダへ処理を委譲する。
  • filter_lookup: プロバイダの条件を評価し、マッチしたプロバイダの関数を特定する。
  • filter_init: プロバイダの初期化関数を呼び出し、コンテキストを構築する。

設定・フック

  • filter_insert: ap_hook_insert_filter により呼び出され、設定されたフィルタをリクエストのフィルタスタックに挿入する。
  • filter_declare / add_filter: フィルタの定義とプロバイダの登録を行う。

制御フローとデータフロー

1. **設定フェーズ**: filter_declare でフィルタを定義し、add_filter でプロバイダを紐付ける。filter_chain で適用順序を決定。 2. **リクエスト処理**: filter_insert がリクエストごとに呼び出され、設定されたフィルタをスタックへ追加。 3. **実行フェーズ**:

  • filter_harness が呼ばれると、filter_lookup で ap_expr または Content-Type を評価。
  • マッチしたプロバイダの filter_func を実行。
  • 必要に応じて proto_flags に基づき、Cache-Control や Content-Length などのヘッダを操作。

直接依存

  • util_filter.h: Apache フィルタフレームワークの基盤。
  • ap_expr.h: 条件式評価エンジン。
  • httpd.h, http_config.h, http_request.h: Apache コア API。
  • apr_hash.h, apr_strings.h: APR ユーティリティ。

関連コード

  • util_filter.c: フィルタスタックの管理ロジック。
  • mod_include.c, mod_deflate.c: 本モジュールをプロバイダとして利用する典型的なフィルタ実装例。

読者向けメモ

  • **メモリ管理**: 設定データは cmd->pool を使用し、リクエストごとのコンテキストは r->pool を使用してライフサイクルを管理しています。
  • **制限事項**: filter_init 内でのフィルタプロバイダの連鎖（Chaining）はサポートされていません。
  • **プロトコルハック**: filter_lookup 内のプロトコル調整ロジックは、フィルタ適用による副作用（Content-Length の不整合など）を補正するためのものであり、実装上は「ハック」であると明記されています。
  • **省略された詳細**: mod_filter は非常に汎用的であるため、個別のプロバイダがどのように ctx を操作するかについては、各プロバイダモジュールの実装に依存します。本ファイルはあくまで「ディスパッチャ」としての役割に特化しています。
