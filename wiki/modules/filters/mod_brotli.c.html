modules/filters/mod_brotli.c

概要

mod_brotli.c は、Apache HTTP Server 用の出力フィルタモジュールであり、Brotli アルゴリズムを用いた動的なコンテンツ圧縮を提供します。mod_deflate と同様のインターフェースと設定項目を持ち、透過的な圧縮によって転送データ量を削減します。

モジュール内での役割

modules/filters サブシステムにおいて、コンテンツエンコーディングを操作するフィルタの一つです。AP_FTYPE_CONTENT_SET フィルタとして登録され、レスポンスボディがクライアントへ送信される直前のストリームを処理します。

責務

  • **設定管理**: Brotli の圧縮品質、ウィンドウサイズ、ETag の扱い、および統計用ノートの設定。
  • **フィルタリング**: Accept-Encoding: br をサポートするクライアントに対し、レスポンスを Brotli 形式で圧縮。
  • **メタデータ管理**: 圧縮に伴う Content-Encoding ヘッダーの更新、Content-Length の削除、および ETag の調整。
  • **統計出力**: 圧縮率や入出力バイト数をリクエストノートとして記録。

重要なシンボル

設定構造体

  • brotli_server_config_t: サーバー単位の設定（品質、ウィンドウサイズ、ETag 処理モード等）を保持。
  • brotli_ctx_t: リクエスト単位のコンテキスト。Brotli エンコーダの状態、出力用バケットブリゲード、入出力バイト数を管理。

設定用ハンドラ

  • set_filter_note, set_compression_quality, set_compression_lgwin, set_compression_lgblock, set_etag_mode: httpd.conf からの設定値を解析し、brotli_server_config_t に適用する。

フィルタ処理

  • compress_filter: メインのフィルタ関数。リクエストの適合性判定、コンテキストの初期化、バケットの圧縮処理を行う。
  • process_chunk: バケットデータを Brotli エンコーダに渡し、圧縮済みデータをブリゲードへ挿入する。
  • flush: BROTLI_OPERATION_FLUSH または FINISH を用いてエンコーダのバッファをフラッシュする。

制御フローとデータフロー

1. **初期化**: compress_filter がリクエストを評価し、圧縮対象であれば create_ctx で BrotliEncoderState を生成。 2. **データ処理**:

  • バケットを読み込み process_chunk へ渡す。
  • BrotliEncoderCompressStream を呼び出し、出力がある場合は BrotliEncoderTakeOutput で取得。
  • 取得したデータは apr_bucket_transient_create でブリゲードに挿入し、ap_pass_brigade で下流へ送出。

    • 3. **終了処理**: EOS バケット受信時に flush を呼び出し、残りのデータをフラッシュ。その後、統計情報をリクエストノートに書き込み、cleanup_ctx でエンコーダインスタンスを破棄。

直接依存

  • httpd.h, http_core.h, http_log.h: Apache HTTP Server のコア機能。
  • apr_strings.h: 文字列操作ユーティリティ。
  • brotli/encode.h: Brotli 圧縮ライブラリの API。

関連コード

  • mod_deflate.c: 本モジュールの設計上の参照元。ETag 処理や設定項目が類似している。
  • util_filter.h: Apache のフィルタチェーンの仕組み。

読者向けメモ

  • **メモリ管理**: brotli_ctx_t はリクエストプールに紐付いており、cleanup_ctx がプールクリーンアップ時に BrotliEncoderDestroyInstance を呼び出すことでメモリリークを防いでいます。
  • **ETag の扱い**: 圧縮によりコンテンツが変化するため、デフォルトで ETag に -br サフィックスを付与します。これは BrotliAlterETag で変更可能です。
  • **制限事項**: サブクエストや Content-Range が存在するリクエストは圧縮対象外となります。
  • **省略箇所**: 本ファイルは Brotli ライブラリの低レベルな圧縮ロジック（enc/ 以下）には関与せず、Apache のバケットブリゲードと Brotli API の橋渡しに特化しています。
