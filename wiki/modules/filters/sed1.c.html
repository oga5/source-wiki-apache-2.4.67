modules/filters/sed1.c

概要

sed1.c は、Apache HTTP Server のフィルタモジュールで使用される libsed エンジンの実行（評価）フェーズを担う中核ファイルです。入力ストリームに対して sed コマンドを適用し、バッファ管理、パターンマッチング、置換処理、およびコマンドディスパッチを実行します。

モジュール内での役割

sed0.c がコンパイルおよびコマンド解析を担当するのに対し、本ファイルは解析済みのコマンド構造体（sed_commands_t）を読み込み、実際のデータ処理を行うランタイム環境を提供します。

責務

  • **バッファ管理**: 行バッファ、ホールドバッファ、汎用バッファの動的拡張とメモリ管理。
  • **実行エンジン**: sed の各コマンド（置換、削除、読み込み、書き込み等）のディスパッチと実行。
  • **ストリーム処理**: 入力バッファの行単位の切り出しと、出力先への書き込み制御。
  • **状態管理**: 評価中の行番号、フラグ（置換成功、終了要求、削除要求など）の保持。

重要なシンボル

ライフサイクル管理

  • sed_init_eval: 評価状態（sed_eval_t）の初期化。
  • sed_reset_eval: 既存の評価状態をリセットし、ファイルハンドルやバッファを再設定。
  • sed_destroy_eval: 評価に関連するファイルハンドル等のリソース解放。

実行・評価

  • sed_eval_file: ファイルストリーム全体を評価。
  • sed_eval_buffer: メモリ上のバッファを評価（フィルタ処理の主軸）。
  • sed_finalize_eval: ストリーム終了時の残存バッファ処理。
  • execute: 行ごとのコマンド実行ループ。

バッファ操作（内部ヘルパー）

  • grow_buffer: 汎用的なメモリ拡張関数（4KB境界へのアライメント）。
  • append_to_linebuf / copy_to_linebuf: 行バッファへのデータ追加・置換。
  • append_to_holdbuf / copy_to_holdbuf: ホールドバッファ操作。

制御フローとデータフロー

1. **入力処理**: sed_eval_buffer が入力データを \n で分割し、execute を呼び出します。 2. **コマンド実行**: execute は sed_commands_t 内のコマンドリストを順次走査します。 3. **マッチング**: match 関数が sed_step を呼び出し、現在の行バッファに対して正規表現マッチングを行います。 4. **置換処理**: substitute -> dosub -> place の順で、置換文字列の構築とバッファの更新が行われます。 5. **出力**: wline を通じて eval->writefn が呼び出され、最終的な出力が行われます。

直接依存

  • apr.h, apr_lib.h: APR（Apache Portable Runtime）によるメモリ管理とファイルI/O。
  • libsed.h, sed.h: 共通データ構造と型定義。
  • regexp.h: 正規表現エンジンとのインターフェース。
  • apr_strings.h

関連コード

  • sed0.c: コマンドのコンパイルと解析。
  • regexp.c: 正規表現のコンパイルと実行ロジック。
  • mod_sed.c: Apache モジュールとしての統合レイヤー。

読者向けメモ

  • **メモリ所有権**: sed_eval_t 内のバッファは apr_pool_t に紐付いており、明示的な free は不要です。ただし、sed_destroy_eval でファイルハンドルを閉じる必要があります。
  • **バッファ拡張**: grow_buffer は MAX_BUF_SIZE（8MB）を上限としており、これを超える行データは APR_ENOMEM を返します。
  • **ポインタの整合性**: append_to_linebuf 等でバッファが再確保されると、step_vars 内のポインタがずれるため、オフセット計算による再同期が行われています。
  • **省略された詳細**: sed_step（regexp.c 内）の内部アルゴリズムは本ファイルには含まれていません。これは正規表現エンジンの詳細に依存するためです。
