modules/filters/mod_include.c

概要

mod_include.c は、Apache HTTP Server における Server Side Includes (SSI) 機能を実装する出力フィルタモジュールです。HTML コンテンツ内の SSI ディレクティブ（<!--#...-->）を解析し、動的な置換や条件分岐、外部ファイルのインクルードを実行します。

モジュール内での役割

modules/filters ディレクトリ内で、コンテンツの動的生成を担う主要なフィルタです。mod_include.h で公開されている API を通じて、他のモジュールが SSI ハンドラを登録したり、文字列解析機能を利用したりすることを可能にしています。

責務

  • **SSI 解析エンジン**: ストリームベースのステートマシンによるディレクティブの抽出。
  • **ディレクティブ処理**: if, echo, include, set 等の標準的な SSI コマンドの実行。
  • **式評価**: レガシーな SSI 式パーサーおよび ap_expr を利用した条件評価。
  • **環境変数管理**: SSI 実行時の変数展開とサブプロセス環境の管理。
  • **設定管理**: XBitHack やタグ形式のカスタマイズなど、ディレクトリ/サーバー単位の設定保持。

重要なシンボル

コンテキスト管理

  • include_ctx_t: フィルタの実行状態を保持する公開構造体。
  • ssi_internal_ctx: 解析状態、バッファ、引数リストなどを保持する非公開の内部構造体。

解析・実行

  • send_parsed_content: フィルタのメインループ。バケットを走査し、ステートマシンを駆動する。
  • handle_...: handle_if, handle_echo 等、各 SSI ディレクティブに対応するハンドラ関数群。
  • ap_ssi_parse_string: 文字列内の変数（$VAR）を展開するユーティリティ。
  • parse_expr / parse_ap_expr: 条件式を解析・評価するパーサー。

設定構造体

  • include_dir_config: ディレクトリ単位の設定（xbithack, lastmodified 等）。
  • include_server_config: サーバー単位の設定（タグの開始/終了シーケンス）。

制御フローとデータフロー

1. **初期化**: includes_filter がリクエストごとにコンテキストを初期化し、INCLUDES フィルタを登録。 2. **解析**: send_parsed_content がバケットブリゲードを走査。find_start_sequence でタグ開始を検出し、ステートマシン（parse_state_t）を遷移させる。 3. **引数抽出**: find_argument でディレクティブの引数を解析し、arg_item_t リストに格納。 4. **実行**: include_handlers ハッシュからディレクティブに対応する関数を検索し実行。 5. **出力**: 処理結果を新しいバケットとしてブリゲードに挿入し、ap_pass_brigade で後続のフィルタへ渡す。

読者向けメモ

  • **メモリ管理**: include_ctx_t 内には pool（永続）と dpool（ディレクティブ実行ごとにクリア）の2種類が存在します。解析中のデータは dpool を使用し、メモリリークを防ぐ設計になっています。
  • **ステートマシン**: PARSE_PRE_HEAD から PARSE_EXECUTE までの状態遷移は非常に複雑です。特に find_partial_start_sequence は、タグがバッファ境界をまたぐ場合の処理を慎重に行っています。
  • **サブリクエスト**: handle_include 等でサブリクエストを生成しますが、これらは r->pool を共有または継承するため、メモリ寿命の管理には注意が必要です。
  • **未実装/浅い部分**: bndm アルゴリズムによる高速検索が実装されていますが、複雑な正規表現の評価は ap_expr に依存しており、このファイル自体は主にパーサーの構造とディレクティブのディスパッチに注力しています。
