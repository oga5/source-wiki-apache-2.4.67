modules/filters/mod_request.c

概要

mod_request.c は、HTTP リクエストボディを一時的にメモリ（request_rec 内）に保持し、後続のサブリクエストやフィルタで再利用可能にするための Apache HTTP Server モジュールです。主に KeptBodySize ディレクティブを通じて、ボディの保持サイズ制限と管理機能を提供します。

モジュール内での役割

modules/filters サブシステムの一部として、入力フィルタチェーンに介入し、リクエストボディの「キャプチャ」と「再挿入」を担います。他のフィルタが処理を行う前にボディを確保する役割を持ちます。

責務

  • **ボディの保持管理**: keep_body_filter を通じて、設定されたサイズ制限内でリクエストボディを r->kept_body にコピーする。
  • **サブリクエストへの再挿入**: kept_body_filter を通じて、保持されたボディをサブリクエストの入力ストリームとして提供する。
  • **設定管理**: KeptBodySize ディレクティブの解析、ディレクトリ単位の設定保持、および設定のマージ。
  • **フィルタのライフサイクル**: 入力フィルタの登録、挿入、および削除の制御。

重要なシンボル

フィルタ関数

  • keep_body_filter: リクエストボディを読み込み、r->kept_body に保存する入力フィルタ。
  • kept_body_filter: r->kept_body に保存されたデータをサブリクエストに提供する入力フィルタ。
  • kept_body_filter_init: サブリクエスト開始時に Transfer-Encoding を除去し Content-Length を設定する初期化関数。

設定構造体

  • request_dir_conf: ディレクトリごとの設定（keep_body: 保持サイズ制限）。
  • create_request_dir_config / merge_request_dir_config: 設定の生成と継承ロジック。

ユーティリティ

  • bail_out_on_error: エラー発生時にブリゲードをクリーンアップし、クライアントへエラーレスポンスを送信する。
  • ap_request_insert_filter / ap_request_remove_filter: フィルタチェーンへの介入用フック。

制御フローとデータフロー

1. **キャプチャ**: keep_body_filter がリクエストボディを ap_get_brigade で取得し、r->kept_body（apr_bucket_brigade）にコピーします。制限を超えた場合、bail_out_on_error で中断します。 2. **再利用**: サブリクエストが開始されると、kept_body_filter_init がヘッダーを調整し、kept_body_filter が r->kept_body からデータをパーティション分割して読み出し、後続のフィルタへ渡します。 3. **データ構造**: keep_body_ctx_t は現在の保持状況（残りサイズ）を管理し、kept_body_ctx_t は再挿入時のオフセットと残量を管理します。

読者向けメモ

  • **メモリ所有権**: r->kept_body は r->pool に紐付いており、リクエストの終了とともに解放されます。
  • **不変条件**: kept_body_filter は、r->kept_body が存在する場合にのみ動作します。
  • **エッジケース**: Content-Length が KeptBodySize を超える場合、即座に HTTP_REQUEST_ENTITY_TOO_LARGE を返します。
  • **省略箇所**: apr_bucket_copy の失敗時の詳細なエラーハンドリングは、バケットの性質上発生しにくいと想定されているため、最小限の記述に留まっています。
