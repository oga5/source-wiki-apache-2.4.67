modules/filters/libsed.h

概要

libsed.h は、Apache HTTP Server のフィルタモジュール内で使用される sed ストリームエディタエンジンのコア定義を提供します。このヘッダーは、sed スクリプトのコンパイル（解析）と、その実行（評価）という二つの主要フェーズを分離するためのデータ構造とインターフェースを定義しています。

モジュール内での役割

modules/filters における sed 関連機能（mod_sed.c 等）の基盤となるライブラリ定義です。sed0.c（コンパイル）および sed1.c（実行）で共有されるデータ構造を定義し、Apache のメモリ管理（apr_pool_t）と統合されたセーフな実行環境を提供します。

責務

  • **スクリプトのコンパイル**: sed コマンドを解析し、実行可能な中間表現（sed_reptr_t）へ変換する。
  • **実行状態の管理**: 入力バッファ、ホールドスペース、パターンマッチング状態を保持する。
  • **抽象化された入出力**: apr_file_t やコールバック関数を通じた、ストリーム指向のデータ処理。
  • **メモリ管理**: apr_pool_t を利用したリソースのライフサイクル管理。

重要なシンボル

コンパイル関連

  • sed_commands_t: コンパイルされた sed スクリプト全体を保持する構造体。ラベルテーブル、コマンドリスト、メモリプールを含む。
  • sed_reptr_t: 個々の sed コマンド（アドレス、正規表現、置換文字列等）を表現するノード。
  • sed_label_t: sed の分岐先ラベルを管理する構造体。

実行関連

  • sed_eval_t: 実行時の状態を保持する構造体。パターンバッファ、ホールドスペース、現在行番号、フラグ類を管理する。
  • sed_err_fn_t / sed_write_fn_t: エラーハンドリングおよび出力処理のためのコールバック関数型。

ライフサイクル管理

  • sed_init_commands / sed_destroy_commands: コンパイル環境の初期化と破棄。
  • sed_init_eval / sed_destroy_eval: 実行環境の初期化と破棄。
  • sed_compile_string / sed_compile_file: スクリプトの読み込みと解析。
  • sed_eval_buffer / sed_eval_file: 実際のストリーム処理の実行。

制御フローとデータフロー

1. **コンパイルフェーズ**: sed_commands_t を初期化し、sed_compile_string または sed_compile_file を呼び出すことで、スクリプトを sed_reptr_t の連結リストに変換します。 2. **実行フェーズ**: sed_eval_t を初期化し、sed_eval_buffer 等にデータを渡します。内部では sed_reptr_t リストを走査し、各行に対してマッチングとコマンド適用を行い、writefn を通じて結果を出力します。 3. **状態遷移**: sed_eval_t 内の linebuf（パターンバッファ）と holdbuf（ホールドスペース）の間でデータが交換され、sed 特有のストリーム処理が進行します。

直接依存

  • limits.h: システムの制限値定義。
  • apr_file_io.h: APR (Apache Portable Runtime) のファイル I/O 抽象化。

関連コード

  • modules/filters/sed0.c: コンパイルロジックの実装。
  • modules/filters/sed1.c: 実行ロジックの実装。
  • modules/filters/mod_sed.c: libsed を利用した Apache フィルタモジュールの実装。

読者向けメモ

  • **メモリ所有権**: すべての構造体は apr_pool_t に依存しています。sed_destroy_... 関数を呼び出すことで、プールに関連付けられたメモリが解放されます。
  • **制限事項**: SED_NLINES, SED_DEPTH, SED_LABSIZE などの定数により、スクリプトの複雑さやネストに上限が設けられています。
  • **設計上の注意**: 本ヘッダーは純粋な C インターフェースを提供しており、C++ から利用する場合は extern "C" ブロックが適用されます。
  • **省略箇所**: 正規表現エンジンの詳細な実装（regexp.h）については本ファイルでは定義されておらず、外部依存として扱われています。
