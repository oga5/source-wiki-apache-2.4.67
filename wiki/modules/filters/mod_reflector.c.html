modules/filters/mod_reflector.c

概要

mod_reflector.c は、受信したリクエストのボディをそのままレスポンスとしてクライアントに送り返す（リフレクションする）ための Apache HTTP Server モジュールです。主にテストやデバッグ、あるいは特定のプロトコル変換のプロトタイプ作成に使用されます。

モジュール内での役割

modules/filters ディレクトリに属する他のフィルタモジュールがコンテンツの変換（圧縮や置換など）を行うのに対し、本モジュールは「入力ストリームをそのまま出力ストリームへ転送する」という極めてシンプルなパイプラインを提供します。

責務

  • **リクエストの反射**: POST リクエストのボディを読み取り、そのままレスポンスボディとして書き出す。
  • **ヘッダーの転送**: 設定されたルールに基づき、リクエストヘッダーをレスポンスヘッダーにコピーする。
  • **メタデータの管理**: Content-Length や Content-Type などの主要なヘッダーをリクエストから引き継ぎ、レスポンスの整合性を保つ。

重要なシンボル

  • reflector_handler: モジュールのメインロジック。リクエストを処理し、バケットブリゲードを介してデータを転送する。
  • reflector_cfg: ディレクトリ単位の設定を保持する構造体。反射させるヘッダーのペアを管理する。
  • ReflectorHeader: 設定ディレクティブ。リクエストヘッダーをレスポンスヘッダーにマッピングするルールを定義する。

制御フローとデータフロー

1. **ハンドラ起動**: reflector_handler が reflector ハンドラを持つリクエストを捕捉。 2. **ヘッダー処理**: header_do を介して、設定されたリクエストヘッダーをレスポンスヘッダーへコピー。 3. **ストリーム処理**:

  • ap_get_brigade を使用して入力フィルタからデータを読み取る。
  • 読み取ったバケットを bbout ブリゲードに書き込む。
  • ap_pass_brigade を呼び出し、出力フィルタへデータを渡す。

    • 4. **終了**: EOS (End of Stream) バケットを検出するまでループし、完了後に OK を返す。

直接依存

  • httpd.h, http_config.h, http_protocol.h: Apache のコア機能およびリクエスト処理用。
  • apr_strings.h, apr_tables.h: APR (Apache Portable Runtime) の文字列操作およびテーブル管理用。

関連コード

  • modules/filters/mod_filter.c: Apache のフィルタチェーン管理の基盤。本モジュールがフィルタとしてどのように統合されるかを理解する上で重要。
  • modules/filters/mod_buffer.c: データをバッファリングする手法の参考として有用。

読者向けメモ

  • 本モジュールは POST メソッドのみをサポートしています。他のメソッドに対する挙動は HTTP_METHOD_NOT_ALLOWED を返します。
  • ap_get_brigade と ap_pass_brigade を使用したストリーミング処理の典型的な実装例です。バケットのメタデータ（length == 0）のスキップや EOS の判定など、フィルタ開発の定石が記述されています。
  • ReflectorHeader ディレクティブを使用する際は、リクエストヘッダー名とレスポンスヘッダー名を指定可能です。デバッグ用途で特定の情報をエコーバックさせる際に非常に強力です。
