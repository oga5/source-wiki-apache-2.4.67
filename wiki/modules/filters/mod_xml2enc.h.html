modules/filters/mod_xml2enc.h

概要

mod_xml2enc.h は、Apache HTTP Server における XML/HTML コンテンツの文字エンコーディング変換を支援する mod_xml2enc モジュールのパブリックインターフェースです。主に他のモジュールから利用されるオプション関数やフックの定義を提供します。

モジュール内での役割

本ファイルは mod_xml2enc.c のヘッダーとして機能し、外部モジュール（mod_proxy_html など）が文字エンコーディングの自動検出や変換機能を呼び出すための契約を定義します。

責務

  • **API の公開**: 他のモジュールが利用可能なオプション関数（xml2enc_charset, xml2enc_filter）を定義する。
  • **フックの提供**: プリプロセス用の外部フックを定義する。
  • **プラットフォーム抽象化**: Windows 環境での DLL エクスポート/インポート制御のためのマクロ定義。

重要なシンボル

  • ENCIO_INPUT / ENCIO_OUTPUT: フィルタの入出力モードを指定するフラグ。
  • xml2enc_charset: コンテンツの文字エンコーディングを特定するためのオプション関数。
  • xml2enc_filter: 指定されたエンコーディングに基づいて変換フィルタを挿入するオプション関数。
  • XML2ENC_DECLARE: モジュール関数をエクスポートするためのプラットフォーム依存マクロ。

制御フローとデータフロー

本ファイル自体は宣言のみを提供します。実際のフローは以下の通りです： 1. 呼び出し側モジュールが APR_RETRIEVE_OPTIONAL_FN を使用して xml2enc_charset や xml2enc_filter を取得する。 2. xml2enc_filter が呼び出されると、mod_xml2enc.c 内で apr_xlate_t を用いた変換コンテキストが初期化される。 3. データはバケットブリゲードを通じて渡され、設定されたエンコーディングに従って変換される。

直接依存

  • apr_optional.h: オプション関数の公開に使用。
  • httpd.h (間接的): request_rec や ap_filter_t などの型定義に依存。
  • libxml/encoding.h (実装側で利用): xmlCharEncoding 型の定義元。

関連コード

  • modules/filters/mod_xml2enc.c: 本ヘッダーで宣言された関数の実装が含まれる。
  • modules/filters/mod_proxy_html.c: mod_xml2enc の主要な利用者であり、本ヘッダーの API を活用して HTML の文字コード変換を行う。

読者向けメモ

  • **プラットフォーム依存性**: XML2ENC_DECLARE などのマクロは Windows 環境でのビルドを考慮したものです。Linux 等の環境では単なる型定義に展開されます。
  • **利用上の注意**: 本モジュールは libxml2 に依存しています。コンパイル時には libxml2 のヘッダーパスが正しく設定されている必要があります。
  • **設計方針**: フィルタの挙動を ENCIO フラグで制御する設計となっており、入力チェック（スニッフィング）と変換処理を明確に分離しています。
