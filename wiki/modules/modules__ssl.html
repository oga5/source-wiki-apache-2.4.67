modules/ssl

システム内での役割

modules/ssl は、Apache HTTP Server における SSL/TLS 通信を司る中核モジュールです。OpenSSL ライブラリをラップし、HTTPD のフィルタチェーンと統合することで、セキュアなデータ転送、証明書管理、セッションキャッシュ、および OCSP ステープリング等の高度な TLS 拡張機能を提供します。

責務

  • **SSL/TLS ライフサイクル管理**: サーバー起動時のコンテキスト構築から、接続ごとのハンドシェイク、終了処理までを管理。
  • **OpenSSL 抽象化**: OpenSSL の複雑な API を HTTPD のメモリ管理（apr_pool_t）やフィルタシステムと適合させる。
  • **設定管理**: サーバー、仮想ホスト、ディレクトリ単位での SSL 設定ディレクティブの解析と適用。
  • **セキュリティ機能の提供**: クライアント認証、OCSP 検証、セッションキャッシュ、SNI 等のポリシー適用。

このモジュールのファイル

  • **エントリポイント**: mod_ssl.c, mod_ssl.h
  • **エンジン層**: ssl_engine_init.c (初期化), ssl_engine_io.c (I/O フィルタ), ssl_engine_kernel.c (リクエスト処理), ssl_engine_config.c (設定), ssl_engine_vars.c (変数解決)
  • **ユーティリティ/補助**: ssl_util_ssl.c, ssl_util_ocsp.c, ssl_util_stapling.c, ssl_scache.c, ssl_engine_log.c, ssl_engine_mutex.c, ssl_engine_pphrase.c, ssl_engine_rand.c
  • **内部定義**: ssl_private.h, mod_ssl_openssl.h

主要なデータ構造

  • SSLModConfigRec / SSLSrvConfigRec / SSLDirConfigRec: サーバー・仮想ホスト・ディレクトリごとの設定構造体。
  • SSLConnRec: 接続ごとの SSL 状態を保持する構造体。
  • ssl_filter_ctx_t: SSL フィルタのコンテキスト。
  • X509: OpenSSL の証明書構造体。

主なフロー

1. **初期化**: ssl_init_Module が SSL_CTX を構築し、証明書と鍵を読み込む。 2. **接続**: ssl_hook_pre_connection が接続を検出し、SSL オブジェクトを生成してフィルタを挿入。 3. **ハンドシェイク**: ssl_io_filter_input が入力を読み込む際に OpenSSL のハンドシェイクをトリガー。 4. **リクエスト処理**: ssl_hook_Access 等がリクエストフェーズごとに SSL 状態を検証し、環境変数を設定。 5. **終了**: ssl_io_filter_output を通じて安全なシャットダウンを実行。

外部依存

  • **OpenSSL**: 暗号化、証明書処理、TLS プロトコル実装の基盤。
  • **APR (Apache Portable Runtime)**: メモリ管理、スレッド同期、ファイル I/O、ネットワーク通信。
  • **Apache Core**: httpd.h, http_config.h 等のサーバーコア API。

内部構成

本モジュールは、設定管理、I/O フィルタ、リクエスト処理の各層が密接に連携するモノリシックな構成です。ディレクトリ境界に基づき、機能ごとに ssl_engine_*.c および ssl_util_*.c に分割されています。

推奨読書順

1. modules/ssl/ssl_private.h (内部構造の理解) 2. modules/ssl/mod_ssl.h (公開 API の確認) 3. modules/ssl/mod_ssl.c (モジュールの全体像とフック) 4. modules/ssl/ssl_engine_init.c (初期化フロー) 5. modules/ssl/ssl_engine_io.c (I/O フィルタリング) 6. modules/ssl/ssl_engine_kernel.c (リクエスト処理のロジック)
