server/mpm/winnt

システム内での役割

server/mpm/winnt は、Windows 環境における Apache HTTP Server のマルチプロセッシングモジュール（MPM）です。Windows のプロセスモデル（親プロセスによる管理と子プロセスによるリクエスト処理）を実装し、Windows サービスとしての実行や、I/O Completion Port (IOCP) を活用した高効率な接続処理を提供します。

責務

  • **プロセス管理**: 親プロセスによる子プロセスの生成、監視、およびライフサイクル管理。
  • **サービス統合**: Windows サービスコントロールマネージャ (SCM) との対話、およびサービスとしてのインストール/制御。
  • **接続処理**: IOCP を用いた非同期 I/O によるスレッドベースのリクエスト処理。
  • **プラットフォーム抽象化**: Windows 特有のイベントログ出力や、プロセス間通信（IPC）の同期制御。

このモジュールのファイル

  • child.c: 子プロセスのメインループおよび IOCP を用いた接続管理。
  • mpm_default.h: スレッド数やプロセス制限などのデフォルト設定値。
  • mpm_winnt.c: MPM のメインロジック、プロセスオーケストレーション。
  • mpm_winnt.h: モジュール内共通の型定義、関数プロトタイプ、定数。
  • nt_eventlog.c: stderr を Windows イベントログへリダイレクトする機能。
  • service.c: Windows サービスとしてのライフサイクル管理と SCM インターフェース。

主要なデータ構造

  • winnt_conn_ctx_t: 接続状態と OVERLAPPED 構造体を保持する接続コンテキスト。
  • nt_service_ctx_t: サービス状態やスレッドハンドルを保持するサービスコンテキスト。
  • ap_signal_parent_e: 親プロセスへのシグナル（シャットダウン、再起動等）を定義する列挙型。
  • io_state_e, accept_filter_e: I/O 状態や接続フィルタの定義。

主なフロー

1. **起動**: mpm_winnt.c が起動引数を解析し、サービスモードであれば service.c を介して SCM と連携。 2. **プロセス生成**: 親プロセスが create_process() を呼び出し、ハンドルを継承した子プロセスを生成。 3. **接続受け入れ**: 子プロセス内のリスナー・スレッドが AcceptEx 等で接続を待ち受け、IOCP を介してワーカー・スレッドへ通知。 4. **リクエスト処理**: ワーカー・スレッドが GetQueuedCompletionStatus で接続を取得し、HTTP 処理を実行。 5. **終了**: シャットダウンイベントを検知すると、全スレッドを停止し、プロセスをクリーンアップ。

外部依存

  • **APR (Apache Portable Runtime)**: プロセス、スレッド、共有メモリ、ミューテックス等のプラットフォーム抽象化。
  • **Windows API**: windows.h, mswsock.h 等を通じた SCM 操作、IOCP、ソケット拡張機能。
  • **Apache Core**: httpd.h, ap_mpm.h 等のサーバーコアインターフェース。

内部構成

本モジュールは Windows 専用であり、他の MPM とは異なり単一プロセスモデル（HARD_SERVER_LIMIT = 1）を前提としています。子プロセス内のスレッドプールが並列処理の主軸となります。

推奨読書順

1. server/mpm/winnt/mpm_winnt.h (全体像と共通定義) 2. server/mpm/winnt/mpm_default.h (制限値と構成パラメータ) 3. server/mpm/winnt/mpm_winnt.c (プロセス管理のメインロジック) 4. server/mpm/winnt/child.c (接続処理とスレッド管理) 5. server/mpm/winnt/service.c (サービス制御) 6. server/mpm/winnt/nt_eventlog.c (ログ出力)

未解決の問い

  • 現在の設計では stderr のリダイレクトが他モジュールと競合する可能性があるため、より安全なログ統合手法の検討が必要。
  • winnt_conn_ctx_t の再利用ロジックにおけるエラーパスでのメモリリークの精査。
