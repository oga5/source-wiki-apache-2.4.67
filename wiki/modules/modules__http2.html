modules/http2

システム内での役割

modules/http2 は、Apache HTTP Server における HTTP/2 プロトコル（RFC 7540）の実装モジュールです。クライアントとの単一の TCP 接続上で複数のリクエストを多重化（Multiplexing）し、効率的なデータ転送とリソース管理を実現します。本モジュールはディレクトリ境界に基づき、プロトコルスタック、セッション管理、ストリーム処理、およびプロキシ連携機能を集約しています。

責務

  • **HTTP/2 プロトコル処理**: フレームの解析、ヘッダー圧縮（HPACK）、状態遷移の管理。
  • **ストリーム多重化**: 単一接続内でのリクエスト/レスポンスの並列処理と優先度制御。
  • **リソース管理**: ワーカースレッドプール（h2_workers）による接続処理の並列化と、メモリ効率を考慮したバケット転送（h2_bucket_beam）。
  • **プロキシ連携**: mod_proxy_http2 を介したバックエンドサーバーとの HTTP/2 通信の抽象化。

このモジュールのファイル

49個のソースファイルで構成され、主に以下のカテゴリに分類されます。

  • **コア・セッション**: h2_session.c, h2_stream.c, h2_c1.c, h2_c2.c
  • **多重化・スレッド**: h2_mplx.c, h2_workers.c, h2_bucket_beam.c
  • **設定・ユーティリティ**: h2_config.c, h2_util.c, h2_headers.c, h2_request.c
  • **プロキシ**: mod_proxy_http2.c, h2_proxy_session.c

主要なデータ構造

  • h2_session: クライアント接続のライフサイクルと状態を管理する司令塔。
  • h2_stream: 個別のリクエスト/レスポンスを表現するコンテキスト。
  • h2_mplx: メイン接続（c1）とワーカースレッド（c2）を仲介する多重化器。
  • h2_bucket_beam: スレッド間でバケットを安全に受け渡すための同期機構。
  • h2_request: HTTP/2 フレームから変換された Apache 内部用リクエスト表現。

主なフロー

1. **接続確立**: h2_c1_setup により接続が HTTP/2 セッションとして初期化される。 2. **フレーム処理**: h2_session_process が nghttp2 を介してフレームを読み込み、イベントをディスパッチする。 3. **多重化**: 受信したリクエストは h2_mplx を経由し、h2_workers のスレッドへ割り当てられる。 4. **レスポンス返送**: ワーカースレッドが生成したレスポンスは h2_bucket_beam を通じてメイン接続へ戻され、クライアントへ送出される。

外部依存

  • nghttp2: HTTP/2 プロトコルスタックのコアライブラリ。
  • APR (Apache Portable Runtime): メモリプール、スレッド同期、バケット管理。
  • Apache HTTP Server Core: conn_rec, request_rec, フィルタチェーン API。

内部構成

本モジュールは、クライアント接続（C1）と内部処理用接続（C2）という二層構造で設計されています。

  • **C1層**: プロトコル解析とセッション維持を担当。
  • **C2層**: 実際のビジネスロジック実行とフィルタ処理を担当。

    • これらは h2_conn_ctx を通じて接続ごとのメタデータを共有し、h2_mplx によって疎結合に連携します。

推奨読書順

1. h2.h (共通定義) 2. h2_session.h (セッション管理) 3. h2_stream.h (ストリーム管理) 4. h2_mplx.h (多重化の仕組み) 5. h2_bucket_beam.h (スレッド間データ転送) 6. h2_workers.h (ワーカー管理) 7. h2_c1.h / h2_c2.h (接続層の役割) 8. mod_http2.c (モジュールエントリポイント)
