modules/generators/mod_suexec.c

概要

mod_suexec は、Apache HTTP Server において CGI プログラムなどを実行する際、サーバーの実行ユーザーではなく、特定のユーザーおよびグループの権限でプロセスを起動するためのモジュールです。

モジュール内での役割

本モジュールは、Apache のコア機能と連携し、リクエスト処理時に実行ユーザーの切り替え（suEXEC）が必要かどうかを判定するインターフェースを提供します。主に mod_cgi や mod_cgid などの実行系モジュールから参照される「ID 解決プロバイダー」として機能します。

責務

  • **設定管理**: SuexecUserGroup ディレクティブを解析し、指定されたユーザー名・グループ名を UID/GID に変換して保持する。
  • **ID 提供**: リクエスト処理時に、そのディレクトリで設定された実行ユーザー情報を get_suexec_id_doer を通じて提供する。
  • **初期化通知**: サーバー起動時に suEXEC が有効であることをログに出力する。

重要なシンボル

  • suexec_module: Apache モジュール構造体。フックや設定コマンドを登録する。
  • set_suexec_ugid: SuexecUserGroup ディレクティブのハンドラ。
  • get_suexec_id_doer: 他モジュールが suEXEC の実行権限を確認するために呼び出すフック関数。
  • suexec_config_t: 各ディレクトリ/サーバー設定を保持する構造体（mod_suexec.h で定義）。

制御フローとデータフロー

1. **設定フェーズ**: 設定ファイルで SuexecUserGroup が検出されると set_suexec_ugid が呼ばれ、UID/GID が suexec_config_t に格納されます。 2. **フック登録**: suexec_hooks により、get_suexec_identity フックが登録されます。 3. **実行フェーズ**: CGI 実行時、実行系モジュールが ap_run_get_suexec_identity を呼び出すと、get_suexec_id_doer が起動し、設定された UID/GID を返します。 4. **初期化**: suexec_post_config がサーバー起動時に一度だけ実行され、suEXEC の有効状態をログに記録します。

直接依存

  • httpd.h, http_config.h, http_core.h: Apache モジュール基盤。
  • unixd.h: UNIX 系のユーザー/グループ ID 管理機能。
  • mod_suexec.h: 共有設定構造体 suexec_config_t の定義。
  • mpm_common.h: MPM 関連のユーティリティ。
  • apr_strings.h
  • http_log.h
  • http_request.h

関連コード

  • modules/generators/mod_cgi.c: 本モジュールが提供する ID 情報を利用してプロセスを起動する主要なコンシューマー。
  • server/util_unixd.c: ap_uname2id や ap_gname2id などの ID 変換関数の実装元。

読者向けメモ

  • **スレッド安全性**: set_suexec_ugid 内のコメントにある通り、ap_uname2id 等の関数はスレッドセーフではない可能性があるため、.htaccess での利用は制限されています。
  • **実行環境**: このモジュールはあくまで「設定と権限情報の提供」を行うものであり、実際のプロセス切り替え（setuid 等）は、外部の suexec バイナリ（ラッパー）が実行時に行います。
  • **設定の有効性**: ap_unixd_config.suexec_enabled フラグが立っていない場合、設定を行ってもエラーとなります。
