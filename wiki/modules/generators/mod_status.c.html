modules/generators/mod_status.c

概要

mod_status は、Apache HTTP Server の内部状態、プロセス/スレッドの稼働状況、およびパフォーマンス統計をリアルタイムで提供するモジュールです。/server-status ハンドラを通じて、HTML または機械可読なプレーンテキスト形式でレポートを生成します。

モジュール内での役割

modules/generators ディレクトリにおいて、サーバーの運用監視を担う中心的なコンポーネントです。scoreboard.h が保持する共有メモリ上の統計情報を読み取り、ユーザーのリクエストに応じて整形・出力します。

責務

  • **統計レポート生成**: サーバーの稼働時間、負荷平均、リクエスト処理数、トラフィック量、CPU 使用率の集計と表示。
  • **プロセス/スレッド監視**: スコアボードを走査し、各ワーカーの状態（読み込み中、書き込み中、待機中など）を可視化。
  • **拡張性**: status_hook を提供し、他のモジュールがレポートに独自の情報を追加できるようにする。
  • **設定管理**: ExtendedStatus のデフォルト有効化や、MPM（Multi-Processing Module）の制限値の取得。

重要なシンボル

レポート生成・整形

  • status_handler: メインハンドラ。リクエストを解析し、レポート生成フローを制御。
  • format_byte_out / format_kbyte_out: バイト単位の数値を人間が読みやすい形式（B, kB, MB, GB）に変換。
  • show_time: 秒単位の稼働時間を「日・時間・分・秒」形式に変換。

設定・状態管理

  • status_module: Apache モジュール構造体。フックの登録とモジュールメタデータを保持。
  • status_options: クエリパラメータ（refresh, notable, auto）を管理するテーブル。
  • status_flags: スコアボードの内部状態（SERVER_READY 等）を文字表現（_, R, W 等）にマッピングする配列。

フック

  • status_hook: 外部モジュールがレポート末尾にコンテンツを追加するためのオプションフック。

制御フローとデータフロー

1. **初期化**: register_hooks がハンドラと設定フックを登録。status_init で MPM からスレッド/プロセス制限値を取得。 2. **リクエスト処理**: status_handler が server-status ハンドラを検知。 3. **スコアボード走査**: server_limit と thread_limit に基づき、全ワーカーの worker_score を ap_copy_scoreboard_worker で取得。 4. **集計**: ap_extended_status が有効な場合、各ワーカーのアクセス数、バイト数、CPU 時間を累積。 5. **出力**: short_report フラグに応じて、HTML テーブルまたはプレーンテキストを ap_rprintf 等でストリーム出力。 6. **拡張**: 最後に ap_run_status_hook を呼び出し、登録された外部モジュールの出力を追加。

読者向けメモ

  • **メモリ管理**: レポート生成にはリクエストプール (r->pool) を使用しており、ハンドラ終了時に自動解放されます。
  • **スレッド安全性**: スコアボードの読み取りは ap_copy_scoreboard_worker を介して行われ、共有メモリの整合性が保たれる設計です。
  • **パフォーマンス**: ExtendedStatus が有効な場合、全スレッドのスコアボードを走査するため、大規模なサーバーではリクエスト時に一定のオーバーヘッドが発生します。
  • **制限事項**: HAVE_TIMES が定義されていない環境（一部の OS）では、CPU 使用率の統計は表示されません。また、inetd モードではスコアボードが利用できないため、エラーを返します。
