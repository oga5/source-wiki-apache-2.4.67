modules/generators/mod_cgi.c

概要

mod_cgi.c は、Apache HTTP Server における CGI (Common Gateway Interface) 1.1 仕様の実装です。外部スクリプトの実行、環境変数の設定、標準入出力のパイプ処理、およびデバッグ用のスクリプトログ出力を管理します。

モジュール内での役割

mod_cgi は modules/generators サブシステムの中核を成すジェネレータモジュールです。mod_include (SSI) と密接に連携し、cgi_common.h で定義された共通ロジックを再利用して、CGI プロセスのライフサイクルを制御します。

責務

  • **CGI 実行管理**: リクエストに応じた CGI プロセスの生成と、環境変数（PATH_INFO, QUERY_STRING 等）の構築。
  • **SSI 統合**: exec cmd 指令によるサブプロセス実行のサポート。
  • **設定管理**: ScriptLog 等のデバッグ設定およびタイムアウト制御の提供。
  • **プロセス間通信**: APR (Apache Portable Runtime) を介したパイプによるスクリプトとの入出力制御。

重要なシンボル

設定構造体

  • cgi_server_conf: サーバー全体の設定（ログパス、ログサイズ制限、バッファサイズ）。
  • cgi_dirconf: ディレクトリ単位の設定（スクリプト実行タイムアウト）。

プロセス制御・実行

  • run_cgi_child(): apr_proc_create を呼び出し、環境変数とパイプ属性を設定して CGI プロセスを起動する。
  • cgi_build_command(): オプション関数。OS 固有のコマンド構築ロジック（Windows の shebang 等）を解決する。
  • cgi_handler(): CGI リクエストを処理するメインハンドラ。

ユーティリティ・ログ

  • log_script(): CGI の入出力内容をデバッグログに書き出す。
  • add_ssi_vars(): SSI 実行時に必要な環境変数を subprocess_env に追加する。

制御フローとデータフロー

1. **ハンドラ起動**: cgi_handler がリクエストを受け取り、cgi_build_command で実行コマンドを確定する。 2. **プロセス生成**: run_cgi_child がパイプを作成し、apr_procattr_t を設定して子プロセスを起動する。 3. **データ転送**:

  • リクエストボディは cgi_handle_request を経由してスクリプトの標準入力へ送られる。
  • スクリプトの出力は apr_bucket_brigade を介してクライアントへストリームされる。

    • 4. **終了処理**: プロセス終了後、cgi_handle_response がレスポンスヘッダとボディを解析・送信する。

直接依存

  • mod_cgi.h: CGI 実行情報構造体 (cgi_exec_info_t) の定義。
  • cgi_common.h: 共通のバケット処理やレスポンスハンドリングロジック。
  • apr_thread_proc.h: プロセス管理およびリソース制限 (RLIMIT)。
  • httpd.h / http_config.h: Apache のコアモジュール API。

読者向けメモ

  • **メモリ管理**: すべてのメモリ割り当ては request_rec のプール (r->pool) に依存しており、リクエスト終了時に自動的に解放されます。
  • **プロセス生存期間**: apr_pool_note_subprocess により、親プロセスが子プロセスのライフサイクルを管理します。
  • **制限事項**: AP_CGI_USE_RLIMIT が定義されている場合、OS がサポートする範囲で CPU やメモリ使用量の制限が適用されます。
  • **未実装/浅い部分**: mod_cgid と比較して、mod_cgi はリクエストごとにプロセスをフォークするため、高負荷環境では mod_cgid (デーモン方式) が推奨されます。本ファイル内の log_script は非常に高コストな処理であるため、本番環境での有効化には注意が必要です。
