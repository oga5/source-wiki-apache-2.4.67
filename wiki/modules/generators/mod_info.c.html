modules/generators/mod_info.c

概要

mod_info.c は、Apache HTTP Server の実行時設定、読み込まれているモジュール、フックの登録状況、およびプロバイダー情報を HTML 形式で出力する診断用モジュールです。サーバーのデバッグや構成確認を目的としており、/server-info ハンドラを提供します。

モジュール内での役割

modules/generators サブシステムの一部として、サーバーの内部状態を可視化する役割を担います。他の生成系モジュール（mod_cgi 等）がリクエスト処理を行うのに対し、本モジュールはサーバー自身のメタデータを抽出・整形してレスポンスとして返します。

責務

  • **設定情報の可視化**: ap_conftree を走査し、現在適用されているディレクティブを HTML で表示する。
  • **モジュール診断**: 読み込まれている全モジュールのフック参加状況や設定フェーズへの関与を表示する。
  • **サーバー環境情報の提供**: MPM 設定、コンパイル時の定義、バージョン情報などを集約する。
  • **拡張情報の管理**: AddModuleInfo ディレクティブを介して、各モジュールに対する追加情報を保持・表示する。

重要なシンボル

設定管理

  • info_svr_conf: モジュール固有のサーバー設定構造体。追加情報を保持する more_info 配列を持つ。
  • create_info_config / merge_info_config: 設定の初期化およびマージを行うライフサイクル関数。
  • info_cmds: AddModuleInfo ディレクティブを定義するコマンドテーブル。

表示・整形ロジック

  • mod_info_module_cmds: 設定ツリーを再帰的に走査し、ディレクティブを整形出力するコア関数。
  • mod_info_indent / mod_info_show_cmd: 設定ファイルの階層構造を視覚化するためのインデントおよび出力ヘルパー。
  • put_int_flush_right: 行番号などを右寄せで出力するユーティリティ。

フック・モジュール解析

  • startup_hooks / request_hooks / other_hooks: サーバー内の全フックポイントを網羅したルックアップテーブル。
  • module_find_hook: 特定のモジュールが特定のフックに登録されているかを判定する。
  • get_sorted_modules: 読み込まれているモジュールリストを名前順にソートして取得する。

制御フローとデータフロー

1. **リクエスト処理**: display_info がハンドラとして呼び出され、クエリ文字列（?server, ?hooks 等）に応じて表示内容を分岐する。 2. **設定走査**: mod_info_module_cmds が ap_conftree を辿り、各ディレクティブのファイル名や行番号を追跡しながら HTML を生成する。 3. **フック解析**: show_active_hooks が定義済みの hook_lookup_t テーブルをループし、各フックポイントに登録されているモジュールを module_find_hook で検索・表示する。 4. **DUMP_CONFIG 対応**: check_config フックにおいて -DDUMP_CONFIG が指定されている場合、標準出力へ設定ツリーをダンプする。

読者向けメモ

  • **メモリ管理**: すべての出力およびデータ構造は request_rec のプール（r->pool）を使用しており、リクエスト終了時に自動的に解放されます。
  • **セキュリティ**: 本モジュールはサーバーの全設定を露出させるため、本番環境では Require ディレクティブ等でアクセス制限をかけることが強く推奨されます。
  • **制限事項**: TODO コメントにある通り、一部のコンパイル時定義（BIG_SECURITY_HOLE 等）の表示は、ヘッダーの依存関係により完全ではない場合があります。
  • **レイアウト**: mod_info_indent は r が NULL の場合（DUMP_CONFIG 時）にファイルハンドル out を使用する設計になっており、リクエスト処理とコマンドライン出力の両方に対応しています。
