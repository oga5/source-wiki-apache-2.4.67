modules/generators/mod_autoindex.c

概要

mod_autoindex.c は、Apache HTTP Server においてディレクトリへのリクエストに対して自動的に HTML 索引ページを生成するモジュールです。ディレクトリ内に DirectoryIndex が存在しない場合、ファイル一覧を整形して表示する機能を担います。

モジュール内での役割

modules/generators サブシステムの一部として、リクエスト処理の最終段階（ハンドラ）で動作します。ディレクトリのファイルシステム情報を走査し、設定に基づいた HTML を動的に構築してクライアントへ返却します。

責務

  • **設定管理**: IndexOptions や AddIcon などのディレクティブを解析し、ディレクトリごとの設定（autoindex_config_rec）を保持・継承する。
  • **索引生成**: ディレクトリ内のファイルを走査し、ソート、フィルタリング（IndexIgnore）、メタデータ（サイズ、更新日時）の取得を行う。
  • **HTML 出力**: テンプレート化された HTML 構造（Preamble, Header, Body, Tail）を生成し、必要に応じてサブリクエストを用いてヘッダー/フッターファイルを挿入する。

重要なシンボル

設定構造体

  • autoindex_config_rec: ディレクトリごとの設定を保持。アイコンリスト、説明文、ソート順、表示オプションなどを管理。
  • struct item: アイコンや代替テキストのパターンマッチング用データ構造。
  • struct ent: 索引生成時に各ファイル/ディレクトリのメタデータを保持する一時的な構造体。

設定・管理関数

  • create_autoindex_config: 設定構造体の初期化。
  • merge_autoindex_configs: 親ディレクトリの設定を継承し、子ディレクトリの設定とマージする。
  • add_opts / add_icon / add_desc: 設定ディレクティブの解析とデータ構造への登録。

索引生成・出力関数

  • index_directory: 索引生成のメインループ。ディレクトリを開き、エントリを収集・ソートし、出力関数を呼び出す。
  • emit_head / emit_tail: 索引ページのヘッダー（Preamble含む）とフッター（Signature含む）の生成。サブリクエストによる外部ファイル挿入を処理。
  • dsortf: ファイルエントリのソートロジック。名前、サイズ、更新日時、説明文でのソートをサポート。

制御フローとデータフロー

1. **ハンドラ起動**: handle_autoindex がリクエストをフックし、Options +Indexes が有効か確認。 2. **ディレクトリ走査**: index_directory が apr_dir_open でディレクトリを走査。 3. **メタデータ収集**: 各ファイルに対して ap_sub_req_lookup_dirent を発行し、ファイル情報（apr_finfo_t）を取得。 4. **フィルタリング**: ignore_entry で IndexIgnore に該当するファイルを排除。 5. **ソート**: 収集した struct ent の配列を dsortf でソート。 6. **HTML 生成**: output_directories がソート済み配列をループし、HTML テーブルまたはリスト形式で出力。

直接依存

  • apr_strings.h, apr_fnmatch.h: 文字列操作およびパターンマッチング。
  • httpd.h, http_config.h, http_protocol.h: Apache のリクエスト処理および設定フレームワーク。
  • util_script.h: スクリプト関連のユーティリティ。

関連コード

  • modules/generators/mod_cgi.c: サブリクエストの仕組みやフィルタリングの挙動において概念的に近い。
  • http_core.c: Options ディレクティブの定義など、本モジュールの動作前提となるコア機能。

読者向けメモ

  • **メモリ管理**: すべての動的メモリ割り当ては request_rec->pool または設定用の apr_pool_t を使用しており、リクエスト終了時に自動解放されます。
  • **サブリクエスト**: ヘッダー/フッターファイルの挿入には ap_sub_req_lookup_uri を使用しています。これにより、SSI や他のハンドラが適用された結果を索引ページに埋め込むことが可能です。
  • **パフォーマンス**: 大規模なディレクトリでは apr_dir_read と ap_sub_req_lookup_dirent がボトルネックになる可能性があります。
  • **未実装/浅い部分**: find_title 関数は HTML ファイルの <title> タグを単純な文字列スキャンで抽出しており、複雑な HTML 構造やエンコーディングには対応していません。
