modules/generators/mod_status.h

概要

mod_status.h は、Apache HTTP Server の稼働状況を監視・表示する mod_status モジュールのパブリックインターフェースを定義するヘッダーファイルです。サーバーの統計情報やプロセス状態を外部から取得するためのフラグや、拡張機能用のフックを定義しています。

モジュール内での役割

mod_status モジュールにおける「コントラクト」の役割を果たします。このヘッダーは、他のモジュールが mod_status の出力形式を制御したり、独自の統計情報を server-status ページに注入したりするための共通定義を提供します。

責務

  • **出力形式の定義**: クライアントからのリクエストに応じて、レポートの詳細度や形式を制御するフラグ（AP_STATUS_*）の提供。
  • **拡張ポイントの提供**: 他のモジュールが mod_status の出力に情報を追加するためのオプションフック（status_hook）の宣言。
  • **プラットフォーム抽象化**: Windows 等の環境におけるシンボルのエクスポート/インポート制御（STATUS_DECLARE マクロ群）の提供。

重要なシンボル

  • AP_STATUS_SHORT: 短縮形式のレポートを要求するフラグ。
  • AP_STATUS_NOTABLE: HTML テーブルを使用しない形式を要求するフラグ。
  • AP_STATUS_EXTENDED: プロセスごとの詳細な統計情報を含めるフラグ。
  • status_hook: 他のモジュールが mod_status の出力に独自の情報を追加するために使用するフック。

制御フローとデータフロー

本ファイル自体はデータ構造やロジックを含みませんが、APR_DECLARE_EXTERNAL_HOOK を通じて、mod_status がリクエスト処理中に登録されたフックを順次呼び出す仕組みを定義しています。 1. mod_status がリクエストを処理する際、status_hook を実行。 2. 登録された各モジュールが request_rec を通じて ap_rputs 等でコンテンツを書き込む。 3. フラグ（flags）により、呼び出し元が要求する出力形式（HTML/テキスト/詳細度）を各フックが判断する。

直接依存

  • ap_config.h: Apache の設定用マクロおよびプラットフォーム依存の定義。
  • httpd.h: request_rec 構造体や Apache の基本型定義。

関連コード

  • modules/generators/mod_status.c: 本ヘッダーで定義されたフックの実装および、実際の統計情報生成ロジックが含まれるメインファイル。
  • include/scoreboard.h: mod_status が表示するプロセスやスレッドの稼働状況を保持する共有メモリ構造体。

読者向けメモ

  • このファイルは主に「拡張性」のために存在します。もし mod_status の出力に独自の情報を追加したい場合は、status_hook を実装してください。
  • STATUS_DECLARE などのマクロは、Apache のモジュール開発において Windows DLL との互換性を保つための定型的な記述です。通常のロジック実装時には意識する必要はありません。
  • AP_STATUS_* フラグはビットマスクとして定義されているため、複数のフラグを組み合わせてリクエストを制御することが可能です。
