modules/generators/mod_asis.c

概要

mod_asis は、Apache HTTP Server において「そのまま送信（Send-As-Is）」機能を提供するモジュールです。ファイルの内容を HTTP レスポンスヘッダーを含めてそのままクライアントに転送します。

モジュール内での役割

modules/generators サブシステムの一部として、特定の MIME タイプ（httpd/send-as-is）を持つファイルに対するリクエスト処理を担当します。サーバーによるヘッダーの自動生成をバイパスし、ファイル内に記述されたヘッダーを優先させる特殊なハンドラを提供します。

責務

  • **HTTP ヘッダーの解析**: ファイルの先頭部分を読み込み、ap_scan_script_header_err_ex を用いてレスポンスヘッダーとして解釈する。
  • **内部リダイレクトの処理**: ファイル内で Location ヘッダーが指定されている場合、内部リダイレクトをトリガーする。
  • **コンテンツのストリーミング**: ヘッダー以降のファイル本体を、Apache のバケットブリゲード（Bucket Brigade）機構を使用してクライアントへ効率的に送信する。

重要なシンボル

  • ASIS_MAGIC_TYPE: このモジュールが処理対象とする MIME タイプ（httpd/send-as-is）。
  • asis_handler: リクエストを処理するメイン関数。ファイルを開き、ヘッダーを解析し、本体を送信する。
  • asis: モジュールの定義構造体。register_hooks をフックとして登録する。

制御フローとデータフロー

1. **ハンドラ起動**: asis_handler がリクエストを受け取り、ハンドラ名が ASIS_MAGIC_TYPE と一致するか確認する。 2. **ファイルオープン**: 対象ファイルを読み込み専用で開く。 3. **ヘッダー解析**: ap_scan_script_header_err_ex を呼び出し、ファイル先頭のヘッダーを解析して r->headers_out 等に反映させる。 4. **リダイレクト判定**: Location ヘッダーが存在する場合、ap_internal_redirect_handler を呼び出し、処理をリダイレクト先に委譲する。 5. **本体送信**: ヘッダーを除いた残りのファイル内容を apr_brigade_insert_file でバケットブリゲードに追加し、ap_pass_brigade で出力フィルターチェーンへ流し込む。

直接依存

  • httpd.h, http_protocol.h: Apache のリクエスト処理およびプロトコル制御の基本。
  • util_script.h: スクリプトヘッダー解析用関数群。
  • apr_file_io.h (間接的に apr_strings.h 等経由): ファイル操作用。
  • ap_config.h
  • http_config.h
  • http_log.h
  • http_main.h
  • http_request.h
  • mod_core.h

関連コード

  • modules/generators/mod_cgi.c: CGI スクリプトのヘッダー解析など、類似したヘッダー処理ロジックを持つ。
  • server/protocol.c: Apache の標準的なレスポンス生成フロー。

読者向けメモ

  • 本モジュールは、サーバーが自動生成するヘッダー（Date や Server など）を上書きしたい場合や、完全に制御されたレスポンスを返したい場合に利用されます。
  • ファイルの先頭が空行で終わっていない場合、ヘッダー解析が正しく行われない可能性があるため、対象ファイルは適切な形式（ヘッダー群 + 空行 + ボディ）である必要があります。
  • 内部リダイレクトが発生した場合、元のリクエストメソッドに関わらず GET メソッドとして再処理される点に注意してください。
