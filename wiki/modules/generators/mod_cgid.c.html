modules/generators/mod_cgid.c

概要

mod_cgid は、CGI スクリプトを外部の専用デーモンプロセス経由で実行するための Apache HTTP Server モジュールです。マルチスレッド MPM（Worker や Event など）環境において、CGI プロセスの生成と管理をメインのサーバープロセスから分離し、安定性とパフォーマンスを向上させることを目的としています。

モジュール内での役割

mod_cgi が各リクエストごとに直接 fork() を行うのに対し、mod_cgid は Unix ドメインソケットを使用してメインサーバーと CGI デーモン間で通信を行います。これにより、スレッドセーフではない CGI 実行環境を隔離し、プロセス管理を中央集権化します。

責務

  • **デーモン管理**: CGI デーモンプロセスの起動、監視、および異常終了時の再起動。
  • **リクエスト転送**: Unix ドメインソケットを介したリクエスト情報（環境変数、引数、ファイルパス等）のシリアライズと転送。
  • **プロセス実行**: デーモン側での CGI プロセスの fork/exec、およびリソース制限（RLIMIT）の適用。
  • **SSI 統合**: mod_include と連携し、SSI 内の exec コマンドを CGI デーモン経由で実行。
  • **デバッグログ**: ScriptLog ディレクティブに基づく CGI 入出力の記録。

重要なシンボル

デーモン管理・状態保持

  • cgid_module: モジュール定義構造体。
  • cgid_server_conf: サーバー単位の設定（ログパス、バッファサイズ等）。
  • cgid_dirconf: ディレクトリ単位の設定（タイムアウト等）。
  • daemon_pid: CGI デーモンプロセスの PID。

通信・リクエスト処理

  • cgid_req_t: デーモンへ送信されるリクエストヘッダー構造体。
  • connect_to_daemon(): デーモンへの Unix ドメインソケット接続を確立。
  • send_req() / get_req(): リクエスト情報のシリアライズとデシリアライズ。
  • cgid_handler(): HTTP リクエストを CGI リクエストに変換するメインハンドラ。

プロセス制御

  • cgid_start(): デーモンプロセスの起動。
  • cgid_server(): デーモンプロセスのメインループ。
  • cleanup_script(): CGI プロセスの終了処理（SIGTERM/SIGKILL）。

制御フローとデータフロー

1. **初期化**: cgid_init が cgid_start を呼び出し、デーモンプロセスを fork します。 2. **リクエスト受信**: cgid_handler がリクエストを受け取ると、connect_to_daemon でソケット接続を確立します。 3. **データ転送**: send_req が環境変数や引数をソケット経由でデーモンへ送信します。 4. **実行**: デーモンは get_req で情報を受け取り、apr_proc_create を用いて CGI プロセスを生成します。 5. **レスポンス**: CGI の出力はパイプ経由で読み取られ、cgi_handle_response を通じてクライアントへ返送されます。

読者向けメモ

  • **メモリ管理**: pcgi プールはデーモンプロセスの生存期間を管理し、リクエストごとのメモリは ptrans プールで管理されます。
  • **エッジケース**: ScriptSock のパス長制限に注意が必要です。また、デーモンが異常終了した場合、cgid_maint が再起動を試みます。
  • **制限**: mod_cgid はマルチスレッド環境での CGI 実行を前提としており、プラットフォームによっては（Windows 等）サポートされていません。
  • **省略**: 本ドキュメントでは、cgi_common.h 内の具体的なバケット操作の詳細や、プラットフォーム固有の fd パッシングの複雑な実装については詳細を省略しています。
