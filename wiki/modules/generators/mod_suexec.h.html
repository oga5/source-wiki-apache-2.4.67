modules/generators/mod_suexec.h

概要

mod_suexec.h は、Apache HTTP Server において CGI プログラム等を別ユーザー権限で実行するための「suEXEC」機能に関連するデータ構造を定義するヘッダーファイルです。

モジュール内での役割

mod_suexec モジュールの中核となる設定データ構造を定義し、他のモジュール（主に CGI 実行系）が suEXEC の設定状態を参照するためのインターフェースを提供します。

責務

  • suEXEC 実行に必要なユーザー ID (UID) およびグループ ID (GID) を保持する構造体の定義。
  • 特定のディレクトリやサーバー設定において suEXEC が有効か否かを示すフラグの管理。

重要なシンボル

  • suexec_config_t: suEXEC の設定を保持する構造体。ap_unix_identity_t 型の ugid と、設定の有効状態を示す active フラグを保持します。

制御フローとデータフロー

本ファイルはデータ定義のみを提供します。実際のフローは mod_suexec.c に依存します：

  • **設定時**: mod_suexec.c 内の set_suexec_ugid が suexec_config_t を更新します。
  • **実行時**: get_suexec_id_doer がリクエストコンテキストからこの構造体を取得し、CGI 実行プロセスに UID/GID を提供します。

直接依存

  • unixd.h: ap_unix_identity_t 型の定義元であり、Unix 系のユーザー/グループ識別情報を扱うために必須です。

関連コード

  • modules/generators/mod_suexec.c: 本ヘッダーで定義された構造体を利用し、フックの登録や設定ディレクティブの処理を行う実装ファイル。
  • include/unixd.h: Apache の Unix 固有の権限管理に関する基盤定義。

読者向けメモ

  • このファイルは非常に軽量であり、主に mod_suexec の設定を他モジュールと共有するためのコントラクトとして機能します。
  • suexec_config_t を直接操作する際は、必ず mod_suexec.c が提供するフックやアクセサ経由で行うことが推奨されます。
  • suEXEC はセキュリティに関わる機能であるため、設定の有効化には ap_unixd_config.suexec_enabled のチェックが必須となります。
