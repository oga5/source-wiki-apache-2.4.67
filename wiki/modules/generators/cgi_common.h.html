modules/generators/cgi_common.h

概要

本ファイルは、Apache HTTP Server の CGI 実行モジュール（mod_cgi, mod_cgid）間で共有される共通ロジックを定義しています。主に、CGI スクリプトの標準エラー出力のキャプチャ、SSI（Server Side Includes）の exec ディレクティブ処理、および CGI 実行時のバケット処理を担います。

モジュール内での役割

modules/generators サブシステムにおいて、CGI 関連の実行基盤を提供します。個別のモジュール（mod_cgi.c 等）が実装すべき共通のユーティリティ関数群を静的関数として提供し、コードの重複を排除しています。

責務

  • **エラーログ管理**: スクリプトの標準エラー出力を読み取り、Apache のエラーログへ転送する。
  • **SSI 統合**: mod_include と連携し、exec コマンド（cmd および cgi）の実行をハンドリングする。
  • **CGI バケット管理**: スクリプトの入出力パイプを監視するためのカスタムバケット型（CGI）の定義とライフサイクル管理。
  • **リクエスト/レスポンス処理**: リクエストボディのスクリプトへの転送、およびスクリプトからのレスポンスヘッダ解析と転送。

重要なシンボル

CGI バケット管理

  • struct cgi_bucket_data: バケットに関連付けられたポーリング用データ（pollset、request_rec、タイムアウト値）。
  • cgi_bucket_create(): スクリプトの stdout/stderr を監視する CGI 型バケットを生成。
  • cgi_bucket_read(): バケット読み取り時に poll() を実行し、stderr をログへ排出しつつ stdout を取得する。

SSI 実行ハンドラ

  • cgi_handle_exec(): mod_include から呼び出される exec ディレクティブのメインハンドラ。
  • cgi_optfns_retrieve(): mod_include が提供するオプション関数を検索し、exec ハンドラを登録する。

ログ・エラー処理

  • log_scripterror(): スクリプトの stderr 出力をフォーマットしてエラーログに書き出す。
  • log_script_err(): パイプから stderr を行単位で読み取り、ログへ転送する。

リクエスト/レスポンス制御

  • cgi_handle_request(): リクエストボディを読み取り、スクリプトの標準入力へ書き込む。
  • cgi_handle_response(): スクリプトの出力を解析し、ヘッダ処理とレスポンスのフィルタリングを行う。

制御フローとデータフロー

1. **リクエスト処理**: cgi_handle_request が入力フィルタからデータを読み取り、スクリプトのパイプへ書き込みます。 2. **バケット読み取り**: cgi_bucket_read が apr_pollset_poll を使用して、スクリプトの stdout と stderr を非同期的に監視します。 3. **エラー転送**: stderr にデータがある場合、log_script_err が即座に読み取り、log_scripterror を経由して Apache のエラーログへ出力します。 4. **レスポンス解析**: cgi_handle_response がスクリプトの出力を ap_scan_script_header_err_brigade_ex で解析し、適切な HTTP ステータスを決定します。

直接依存

  • apr_buckets.h, apr_poll.h: バケット操作と非同期 I/O 監視。
  • httpd.h, util_script.h: Apache のリクエスト処理と CGI ユーティリティ。
  • util_filter.h: フィルタチェーンを通じたデータ転送。
  • apr.h
  • apr_lib.h
  • apr_strings.h
  • apr_want.h

関連コード

  • mod_cgi.c / mod_cgid.c: 本ヘッダの関数を呼び出し、CGI 実行のメインエントリポイントとなる。
  • mod_include.c: cgi_handle_exec を呼び出す SSI 処理の本体。

読者向けメモ

  • **メモリ管理**: cgi_bucket_data は request_rec のプールを使用して割り当てられます。バケットの生存期間はリクエストの生存期間に依存します。
  • **注意点**: cgi_bucket_read 内での poll() は、スクリプトの出力が完了するまでブロックする可能性があるため、タイムアウト設定が重要です。
  • **未実装/省略**: WANT_CGI_BUCKET マクロが定義されていない場合、バケット関連のロジックはコンパイルされません。これは環境やモジュールの構成に依存します。
  • **パフォーマンス**: log_scripterror 内で apr_stat を呼び出している箇所は、高負荷時にコストとなる可能性があるため注意が必要です。
