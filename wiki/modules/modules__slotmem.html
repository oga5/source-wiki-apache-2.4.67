modules/slotmem

システム内での役割

modules/slotmem は、Apache HTTP Server における固定長データ領域（スロット）の管理サブシステムです。メモリをスロット単位に分割し、その利用状況の追跡やデータアクセスを抽象化するプロバイダーインターフェースを提供します。

責務

  • **スロット管理**: 固定長メモリブロックの確保、解放、および読み書きの提供。
  • **プロバイダーの実装**: 共有メモリ（SHM）版とプロセスローカル（Plain）版の2つの実装を提供し、透過的な利用を可能にする。
  • **ライフサイクル管理**: インスタンスの生成、破棄、およびサーバー再起動を跨ぐための永続化処理（SHM版）。

このモジュールのファイル

  • modules/slotmem/mod_slotmem_plain.c: プロセスローカルなメモリ管理実装。
  • modules/slotmem/mod_slotmem_shm.c: apr_shm を利用したプロセス間共有メモリ実装。

主要なデータ構造

  • ap_slotmem_instance_t: スロットメモリの管理構造体。ベースアドレス、スロットサイズ、個数、利用状況フラグを保持。
  • sharedslotdesc_t: スロットの構成情報（サイズ、個数、型）を保持するヘッダー。
  • ap_slotmem_provider_t: 外部モジュールから利用される関数ポインタ群（create, attach, grab, put, get 等）。

主なフロー

1. **初期化**: pre_config でグローバルプールを初期化し、プロバイダーを登録。 2. **インスタンス生成**: slotmem_create によりメモリ領域を確保し、メタデータ（sharedslotdesc_t 等）を配置。 3. **データアクセス**: slotmem_get / slotmem_put を通じて、オフセット計算に基づき memcpy でデータ転送。 4. **永続化（SHMのみ）**: store_slotmem / restore_slotmem により、MD5チェックサムを用いた整合性検証付きのファイル保存・復元を実行。

外部依存

  • ap_slotmem.h: 共通APIおよびプロバイダー定義。
  • httpd.h, http_main.h, ap_mpm.h: サーバー設定、ログ出力、MPM世代管理。
  • apr ライブラリ: メモリ管理（apr_pools）および共有メモリ操作（apr_shm）。

内部構成

本モジュールはディレクトリ境界により定義されており、以下の2つの独立したプロバイダー実装で構成されます。

  • **SHMプロバイダー**: プロセス間通信が必要な用途向け。
  • **Plainプロバイダー**: 単一プロセス内でのデータ管理向け。

推奨読書順

1. modules/slotmem/mod_slotmem_shm.c（共有メモリの複雑なライフサイクルと永続化を理解するため） 2. modules/slotmem/mod_slotmem_plain.c（基本的なスロット管理の仕組みを理解するため）
