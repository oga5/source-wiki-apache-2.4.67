test

システム内での役割

test モジュールは、Apache HTTP Server のコア機能およびシステム環境の検証を行うためのテストプログラム群です。ディレクトリ境界に基づき構成されており、C言語による単体テスト・ベンチマークツールと、Python（pytest）によるテスト実行基盤が混在しています。

責務

  • **機能検証**: ap_find_list_item や ap_get_list_item 等のコアユーティリティの動作確認。
  • **システム性能評価**: プロセス間同期（セマフォ）やネットワークI/O（writev）の効率性・挙動の検証。
  • **ストレステスト**: 異常なリクエストに対するサーバーの堅牢性（境界値）の確認。
  • **テスト基盤の提供**: Python テストスイートにおける環境設定および実行制御の抽象化。

このモジュールのファイル

  • **C言語テスト/ツール**: test/cls.c, test/test-writev.c, test/test_find.c, test/test_limits.c, test/test_parser.c, test/test_select.c, test/time-sem.c
  • **Pythonテスト基盤**: test/conftest.py

主要なデータ構造

  • struct iovec: writev テストで使用されるバッファ配列。
  • struct timeval: select テストにおけるタイムアウト制御用。
  • HttpdTestEnv: conftest.py で使用されるテスト環境情報保持クラス。

主なフロー

1. **ユーティリティ検証**: メモリプールを初期化し、標準入力から受け取った文字列を対象関数（ap_get_list_item 等）で解析・検証する。 2. **システム負荷/性能試験**: 共有メモリやソケットを確保し、ロック獲得やネットワーク送信を反復実行して結果を計測する。 3. **テストスイート実行**: pytest が conftest.py を読み込み、環境設定を適用した上で各テストケースをパラメータ化して実行する。

外部依存

  • **Apache/APR**: httpd.h, apr_general.h
  • **システムライブラリ**: sys/socket.h, sys/mman.h, sys/sem.h, pthread.h 等

内部構成

  • **機能テスト**: ユーティリティ関数の単体テスト（test_find.c, test_parser.c）。
  • **診断・ベンチマーク**: システムコールや同期機構の検証（time-sem.c, test-writev.c, test_select.c, test_limits.c）。
  • **ユーティリティ**: キャッシュファイル解析（cls.c）。
  • **テスト基盤**: Python によるテスト実行制御（conftest.py）。

推奨読書順

1. test/time-sem.c （同期機構の検証） 2. test/test_find.c （ユーティリティ検証） 3. test/test_parser.c （パースロジック検証） 4. test/test_limits.c （境界値テスト） 5. test/cls.c （キャッシュ解析ツール） 6. test/test-writev.c （ネットワークI/O検証） 7. test/test_select.c （タイマー検証） 8. test/conftest.py （テスト基盤）
