include

システム内での役割

include ディレクトリは、Apache HTTP Server (httpd) の全モジュールおよびコアコンポーネントが依存する、公開インターフェースの集約地です。サーバーのライフサイクル管理、HTTP プロトコル処理、設定システム、およびモジュール間連携のための基盤となるデータ構造と API 定義を提供します。

※本モジュールはディレクトリ境界に基づき定義されています。

責務

  • **API の標準化**: サーバー全体で一貫した型定義、定数、および関数インターフェースの提供。
  • **モジュール間連携の基盤**: フックシステムやプロバイダーモデルを通じた、疎結合なモジュール拡張のサポート。
  • **プラットフォーム抽象化**: OS やコンパイラ間の差異を吸収し、移植性の高い開発環境の提供。

このモジュールのファイル

本モジュールには、サーバーのコア機能からユーティリティまで 43 個のヘッダーファイルが含まれます。

  • **コア定義**: httpd.h, http_config.h, http_core.h, http_protocol.h
  • **MPM/プロセス管理**: ap_mpm.h, mpm_common.h, scoreboard.h
  • **ユーティリティ/フィルタ**: util_filter.h, util_fcgi.h, util_ldap.h, util_mutex.h, ap_expr.h, ap_regex.h
  • **認証/認可**: mod_auth.h
  • **その他**: ap_config.h, ap_release.h, ap_hooks.h, ap_slotmem.h, apache_noprobes.h 等

主要なデータ構造

  • **リクエスト/接続**: request_rec, conn_rec, server_rec
  • **設定/モジュール**: module, command_rec, ap_conf_vector_t
  • **式評価**: ap_expr_info_t, ap_expr_eval_ctx_t
  • **MPM/プロセス**: mpm_child_status, ap_pod_t, worker_score
  • **フィルタ**: ap_filter_t, ap_filter_rec_t, apr_bucket_brigade

主なフロー

本モジュールはインターフェース定義群であるため、特定の単一フローを持ちませんが、以下のライフサイクルを支えています。 1. **設定読み込み**: http_config.h を通じてディレクティブを解析し、設定ツリーを構築。 2. **リクエスト処理**: httpd.h の構造体を受け渡し、util_filter.h のフィルタチェーンを通じてデータを処理。 3. **MPM 制御**: ap_mpm.h と scoreboard.h を用いてプロセス/スレッドの状態を管理。

外部依存

  • **APR (Apache Portable Runtime)**: apr.h, apr_pools.h, apr_buckets.h, apr_shm.h 等、OS 抽象化層として必須。
  • **システムヘッダー**: sys/resource.h, syslog.h 等。

内部構成

大規模なヘッダー群を以下のカテゴリで分類可能です。

  • **コア・プロトコル**: サーバーの基本構造と HTTP 処理。
  • **MPM・プロセス管理**: 並列処理と状態共有。
  • **設定・拡張**: モジュール登録とディレクティブ解析。
  • **ユーティリティ**: 正規表現、LDAP、FastCGI、フィルタ、式評価エンジン。

推奨読書順

1. httpd.h (基本構造) 2. http_config.h (設定システム) 3. http_core.h (コア機能) 4. ap_hooks.h (連携基盤) 5. http_protocol.h (プロトコル処理) 6. util_filter.h (データ処理) 7. ap_mpm.h / mpm_common.h (プロセス管理) 8. scoreboard.h (状態監視)
