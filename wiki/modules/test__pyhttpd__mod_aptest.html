test/pyhttpd/mod_aptest

本モジュールはディレクトリ境界に基づき定義されています。

システム内での役割

Apache HTTP Server のテスト自動化を支援する診断用モジュールです。Python ベースのテストフレームワークから Apache 内部の動作を検証するための「観測点」として機能します。

責務

  • **リクエスト監視**: post_read_request フックを用い、リクエスト処理の初期段階で特定のテスト用ヘッダーを検査する。
  • **診断ログの生成**: AP-Test-Name ヘッダーを検出し、リクエスト内容をログ出力する。
  • **透過的な動作**: 自身の処理後に DECLINED を返し、後続の処理を阻害しない。

このモジュールのファイル

  • test/pyhttpd/mod_aptest/mod_aptest.c

主要なデータ構造

  • module aptest: Apache モジュール定義構造体。
  • request_rec: リクエスト情報を保持する Apache コア構造体。

主なフロー

1. **初期化**: aptest_hooks により aptest_post_read_request を post_read_request フックに登録。 2. **リクエスト受信**: リクエスト到着時に aptest_post_read_request が実行される。 3. **ヘッダー検査**: r->headers_in から AP-Test-Name を取得し、存在すればログ出力。 4. **制御返却**: DECLINED を返し、リクエスト処理を次のハンドラへ引き継ぐ。

外部依存

  • **Apache Core**: httpd.h, http_protocol.h, http_request.h 等。
  • **APR (Apache Portable Runtime)**: 文字列操作やユーティリティ関数。
