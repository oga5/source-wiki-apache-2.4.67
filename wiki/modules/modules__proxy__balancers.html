modules/proxy/balancers

システム内での役割

本モジュールは、Apache HTTP Server の mod_proxy におけるロードバランシング機能の拡張メソッド群を提供します。ディレクトリ境界に基づき、リクエストの振り分けアルゴリズムを実装する複数のプラグインモジュールを包含しています。

責務

  • 各種アルゴリズム（リクエスト数、トラフィック量、負荷状況、ハートビート）に基づいた最適なバックエンドワーカーの選定。
  • mod_proxy の PROXY_LBMETHOD プロバイダーとしてのインターフェース提供。
  • ワーカーの負荷状態（lbstatus や busy 数）の管理と統計情報の更新。

このモジュールのファイル

  • mod_lbmethod_bybusyness.c: 処理中のリクエスト数（busy数）に基づく負荷分散。
  • mod_lbmethod_byrequests.c: 設定された重み（lbfactor）とリクエスト数に基づく負荷分散。
  • mod_lbmethod_bytraffic.c: 転送バイト数に基づく負荷分散。
  • mod_lbmethod_heartbeat.c: 外部から提供されるハートビート情報に基づく負荷分散。

主要なデータ構造

  • proxy_balancer_method: 各アルゴリズムが実装する共通のメソッド構造体。
  • lb_hb_ctx_t / hb_server_t: ハートビート用コンテキストおよびサーバー状態。
  • proxy_worker / proxy_balancer: mod_proxy が定義する共有メモリ上のワーカーおよびバランサー構造体。

主なフロー

1. **初期化**: post_config フックにて mod_proxy のコア関数（ワーカー走査用など）を取得し、プロバイダーを登録。 2. **選定**: バランサーがリクエストを受け取ると、各モジュールの find_best 関数が呼び出される。 3. **走査**: ap_proxy_balancer_get_best_worker_fn を介して全ワーカーを評価し、アルゴリズム固有の基準で最適なワーカーを特定。 4. **更新**: 選定されたワーカーの統計情報（lbstatus やカウンタ）を更新し、次回の選定に反映。

外部依存

  • mod_proxy.h: プロキシ機能の基盤定義。
  • scoreboard.h: 共有メモリ上のワーカー状態管理。
  • ap_slotmem.h: 共有メモリへのアクセス。
  • ap_hooks.h / ap_mpm.h: Apache のフックおよび MPM 制御。
  • heartbeat.h: ハートビート関連の共通定義。

内部構成

各ファイルは独立した PROXY_LBMETHOD プロバイダーとして実装されており、共通の mod_proxy インフラを利用して並列に動作します。

推奨読書順

1. mod_lbmethod_heartbeat.c: 外部データ連携を含む複雑な実装例。 2. mod_lbmethod_bybusyness.c: 負荷状況に基づく標準的な選定ロジック。 3. mod_lbmethod_byrequests.c: 重み付けラウンドロビンに近い基本的な選定ロジック。 4. mod_lbmethod_bytraffic.c: 統計データを用いた選定ロジック。

未解決の問い

  • age 関数など、現在空実装となっているフックの将来的な活用方針。
