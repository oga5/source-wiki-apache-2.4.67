modules/examples

システム内での役割

modules/examples は、Apache HTTP Server の拡張開発における主要な概念を学習するための教育用モジュール群です。ディレクトリ境界に基づき、フック、フィルタ、IPC といった Apache のコア機能を個別のサンプルとして提供します。

責務

  • Apache モジュール開発のベストプラクティスと実装パターンの提示。
  • フックの実行順序やコンテキストの可視化。
  • バケットブリゲードを用いたストリーム処理（フィルタ）の実装例。
  • 共有メモリとミューテックスを用いたプロセス間通信（IPC）のデモンストレーション。

このモジュールのファイル

  • mod_case_filter.c: 出力フィルタのサンプル。
  • mod_case_filter_in.c: 入力フィルタのサンプル。
  • mod_example_hooks.c: 全フックの実行追跡と可視化。
  • mod_example_ipc.c: 共有メモリとグローバルミューテックスによる排他制御。

主要なデータ構造

  • **設定構造体**: x_cfg, CaseFilterConfig, CaseFilterInConfig（各モジュールの設定保持）。
  • **フィルタコンテキスト**: CaseFilterInContext（入力フィルタの処理状態）。
  • **IPC データ**: exipc_data（共有メモリ上のカウンタ構造体）。

主なフロー

1. **フック登録**: register_hooks 関数により、サーバー起動時やリクエスト処理時の各フェーズにコールバックを登録。 2. **設定解析**: ディレクティブ（Example, CaseFilter 等）を解析し、設定構造体を生成・マージ。 3. **リクエスト処理**:

  • **フィルタ**: ap_input_filter / ap_output_filter を通じてバケットブリゲードを走査・変換。
  • **IPC**: apr_global_mutex で排他制御を行い、共有メモリ上のデータを更新。
  • **トレース**: request_rec や conn_rec のプールを利用して実行履歴を蓄積し、HTML レポートとして出力。

外部依存

  • **Apache Core API**: httpd.h, http_config.h, http_request.h, http_protocol.h, util_filter.h
  • **APR (Apache Portable Runtime)**: apr_buckets.h, apr_shm.h, apr_pools.h, apr_strings.h, util_mutex.h

内部構成

本モジュールは独立したサンプルコードの集合体であり、モジュール間での直接的な依存関係はありません。各ファイルは個別の機能単位として完結しています。

推奨読書順

1. mod_example_hooks.c（Apache のライフサイクルとフックの基本理解） 2. mod_example_ipc.c（プロセスモデルと同期制御の理解） 3. mod_case_filter_in.c（入力フィルタとバケット操作の理解） 4. mod_case_filter.c（出力フィルタの理解）
