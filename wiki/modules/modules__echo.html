modules/echo

システム内での役割

modules/echo は、Apache HTTP Server におけるプロトコルハンドラのデモンストレーションおよびテスト用モジュールです。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • クライアントから受信したデータをそのまま送り返すエコーサーバー機能の提供。
  • Apache の process_connection フックを利用した、HTTP 以外の独自プロトコル実装のテンプレート提供。
  • ProtocolEcho ディレクティブによるモジュール有効化の管理。
  • scoreboard を介した接続状態の可視化。

このモジュールのファイル

  • modules/echo/mod_echo.c

主要なデータ構造

  • EchoConfig: モジュールの有効状態を保持する設定構造体。

主なフロー

1. **初期化**: register_hooks により process_connection フックを登録。 2. **接続処理**: process_echo_connection が呼び出され、設定を確認。 3. **ループ処理**: ap_get_brigade でデータを受信し、update_echo_child_status でスコアボードを更新後、ap_pass_brigade でデータをクライアントへ送り返す。 4. **終了**: 接続切断までループし、リソースを解放。

  • apr_buckets.h, apr_strings.h
  • util_filter.h
  • ap_config.h, ap_mmn.h
