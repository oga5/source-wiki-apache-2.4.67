os/unix

システム内での役割

os/unix は、Apache HTTP Server が Unix 系プラットフォーム上で動作するための基盤を提供するモジュールです。ディレクトリ境界に基づき構成されており、プラットフォーム固有の差異を吸収し、MPM（Multi-Processing Module）が共通して利用する低レイヤー機能を提供します。

責務

  • **プラットフォーム抽象化**: Unix 特有のシステムコールや機能（rlimit、シグナル処理、プロセス権限管理）のラップ。
  • **MPM 共通基盤**: MPM が再起動を跨いで状態を保持するためのデータ構造や、権限降格時のミューテックス制御の提供。
  • **特権管理**: suexec を介した安全なプロセス生成および実行ユーザー/グループの識別。

このモジュールのファイル

  • os/unix/os.h: プラットフォーム識別および OS 固有インターフェースの宣言。
  • os/unix/unixd.c: プロセス権限管理、シグナルハンドリング、リソース制限の実装。
  • os/unix/unixd.h: MPM 共通のデータ構造、定数、およびユーティリティ関数の定義。

主要なデータ構造

  • unixd_config_rec: 実行ユーザー、グループ、chroot 設定など、サーバーの Unix 実行環境を保持。
  • ap_unix_identity_t: suexec 等で使用されるユーザー/グループ ID の識別子。
  • ap_unixd_mpm_retained_data: 再起動時にも保持すべき MPM 固有の状態（シグナルフラグ等）を格納。

主なフロー

1. **起動時権限管理**: サーバー起動時に ap_unixd_set_proc_mutex_perms を呼び出し、root 権限から非特権ユーザーへ移行する際のミューテックス所有権を調整。 2. **シグナル処理**: ap_unixd_mpm_set_signals で登録されたハンドラが retained_data を更新し、メインループが停止・再起動の制御フローを決定。 3. **特権プロセス生成**: ap_os_create_privileged_process を通じて、suexec バイナリを介した安全なプロセス生成を実行。

外部依存

  • **APR**: apr_proc_mutex.h, apr_signal.h, apr_thread_proc.h 等のプラットフォーム抽象化レイヤー。
  • **システムヘッダー**: <pwd.h>, <grp.h>, <sys/resource.h> 等の Unix 標準 API。
  • **Apache Core**: httpd.h, http_config.h, mpm_common.h 等のサーバーコア定義。

内部構成

本モジュールはディレクトリ境界により定義されており、直接的な子モジュールは持ちません。

推奨読書順

1. os/unix/unixd.h（提供されるインターフェースとデータ構造の理解） 2. os/unix/os.h（プラットフォーム固有の定義と機能フラグの確認） 3. os/unix/unixd.c（各機能の具体的な実装と制御フローの把握）
