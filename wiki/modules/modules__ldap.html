modules/ldap

システム内での役割

modules/ldap は、Apache HTTP Server における LDAP 接続管理およびキャッシュ機構を提供するサブシステムです。LDAP サーバーへの負荷を軽減し、認証・認可処理のパフォーマンスを最適化するためのバックエンド機能を提供します。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • **接続管理**: LDAP サーバーへの接続ライフサイクル（初期化、バインド、再利用、切断）の管理。
  • **キャッシュ層の提供**: 検索、比較、DN 比較結果をメモリ上に保持し、バックエンドへのクエリを抑制。
  • **SDK 抽象化**: 異なる LDAP SDK 間の API 差異を吸収し、一貫したインターフェースを提供。
  • **設定管理**: httpd.conf で指定された LDAP 関連ディレクティブの解析と状態反映。

このモジュールのファイル

  • util_ldap.c: LDAP 操作のメインロジックと接続管理。
  • util_ldap_cache.c: キャッシュノードの具体的な振る舞い（ハッシュ、比較、コピー等）の実装。
  • util_ldap_cache.h: キャッシュ機構のデータ構造とインターフェース定義。
  • util_ldap_cache_mgr.c: キャッシュのライフサイクル管理、メモリ管理（SHM/RMM）、統計情報の集計。

主要なデータ構造

  • **キャッシュ管理**: util_ald_cache_t（メイン構造体）、util_cache_node_t（リンクリストノード）。
  • **ノード型**: util_url_node_t（URL別ルート）、util_search_node_t（検索結果）、util_compare_node_t（比較結果）、util_dn_compare_node_t（DN比較結果）。

主なフロー

1. **接続取得**: uldap_connection_find により、既存の接続を検索または新規作成。 2. **キャッシュ検索**: util_ald_cache_fetch を呼び出し、ヒットすれば即座に結果を返却。 3. **LDAP 操作**: キャッシュミス時、LDAP サーバーへリクエストを送信し、結果を util_ald_cache_insert でキャッシュに格納。 4. **パージ**: キャッシュが上限に達した場合、util_ald_cache_purge により TTL 期限切れノードを解放。

外部依存

  • **Apache Core**: httpd.h, http_config.h, http_request.h 等。
  • **APR**: apr_shm.h, apr_rmm.h（共有メモリ管理用）。
  • **LDAP SDK**: apr_ldap.h を介した抽象化レイヤー。

内部構成

本モジュールは、LDAP 操作のメインロジック（util_ldap.c）と、それを支えるキャッシュ管理層（util_ldap_cache.c, util_ldap_cache_mgr.c）の二層構造で構成されています。

推奨読書順

1. modules/ldap/util_ldap_cache.h（データ構造の理解） 2. modules/ldap/util_ldap_cache_mgr.c（メモリ管理とキャッシュ基盤） 3. modules/ldap/util_ldap_cache.c（ノード操作の実装） 4. modules/ldap/util_ldap.c（メインロジックと接続管理）
