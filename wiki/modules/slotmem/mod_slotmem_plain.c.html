modules/slotmem/mod_slotmem_plain.c

概要

mod_slotmem_plain.c は、Apache HTTP Server における共有メモリ管理の「プレーン（非共有）」実装です。プロセス間共有を目的とした mod_slotmem_shm とは異なり、単一プロセス内のメモリ領域を固定サイズの「スロット」に分割して管理します。

モジュール内での役割

ap_slotmem プロバイダーインターフェースを実装するモジュールの一つです。AP_SLOTMEM_PROVIDER_GROUP に "plain" として登録され、共有メモリを必要としない環境や、単一プロセス内でのデータ管理において、他のモジュールから透過的に利用されます。

責務

  • **メモリ管理**: 固定サイズのデータブロック（スロット）の確保と解放。
  • **ライフサイクル管理**: globallistmem を用いたスロットメモリインスタンスのリスト管理。
  • **状態追跡**: 各スロットの利用状況（inuse フラグ）の保持と更新。
  • **プロバイダー提供**: ap_slotmem_provider_t 構造体を通じた機能の公開。

重要なシンボル

データ構造

  • ap_slotmem_instance_t: スロットメモリの管理単位。ベースアドレス、スロットサイズ、個数、利用状況フラグ、および次インスタンスへのポインタを保持。

インスタンス管理

  • globallistmem: モジュール内で作成された全インスタンスを保持する静的リストの先頭。
  • gpool: インスタンスの割り当てに使用されるグローバルプール。

プロバイダー関数 (API)

  • **作成・接続**: slotmem_create, slotmem_attach
  • **データ操作**: slotmem_get (読み込み), slotmem_put (書き込み), slotmem_dptr (直接ポインタ取得)
  • **スロット制御**: slotmem_grab (空きスロット確保), slotmem_fgrab (強制確保), slotmem_release (解放)
  • **統計**: slotmem_num_slots, slotmem_num_free_slots, slotmem_slot_size

制御フローとデータフロー

1. **初期化**: ap_slotmem_plain_register_hook がプロバイダーを登録し、pre_config でグローバルプールを初期化します。 2. **作成**: slotmem_create は gpool からメモリを確保し、base アドレスと inuse フラグ領域を計算してリストに追加します。 3. **アクセス**: slotmem_get/put は id からオフセットを計算し、memcpy を介してデータを転送します。この際、AP_SLOTMEM_TYPE_PREGRAB フラグが立っている場合は inuse フラグのチェックが行われます。 4. **反復処理**: slotmem_do は全スロットを走査し、コールバック関数を適用します。

直接依存

  • ap_slotmem.h: プロバイダーインターフェース定義および ap_slotmem_provider_t 構造体。
  • apr_pools.h (間接的): メモリ管理。
  • apr_strings.h (間接的): 文字列操作。

関連コード

  • modules/slotmem/mod_slotmem_shm.c: 共有メモリ版の実装。API は共通ですが、こちらはプロセス間通信を前提としています。

読者向けメモ

  • **メモリ所有権**: インスタンスおよびデータ領域は gpool に紐付いており、モジュールのライフサイクル全体で生存します。
  • **境界チェック**: slotmem_get/put 内の memcpy には明示的なサイズチェックが含まれていません。呼び出し側で dest_len / src_len がスロットサイズを超えないことを保証する必要があります。
  • **スレッド安全性**: 本実装は単一プロセス内での利用を想定しており、内部的なロック機構（Mutex等）は存在しません。マルチスレッド環境で利用する場合は、呼び出し側での排他制御が必要です。
  • **省略された詳細**: AP_SLOTMEM_IS_PREGRAB の挙動に関する詳細な設計意図はソース内にコメントとして残されていますが、実装自体は単純なフラグチェックに留まっています。
