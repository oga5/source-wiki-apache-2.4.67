modules/slotmem/mod_slotmem_shm.c

概要

mod_slotmem_shm.c は、Apache HTTP Server における共有メモリベースの固定長スロット管理機能を提供します。複数のプロセス間で共有されるメモリ領域をスロット単位に分割し、各スロットの利用状況（in-use）を追跡・永続化する仕組みを実装しています。

モジュール内での役割

slotmem サブシステムにおける「共有メモリプロバイダ」です。mod_slotmem_plain がプロセスローカルなメモリを扱うのに対し、本モジュールは apr_shm を利用してプロセス間共有を実現します。

責務

  • **共有メモリ管理**: apr_shm を用いたセグメントの作成、アタッチ、破棄。
  • **スロット操作**: 固定長データ領域の読み書き、スロットの確保（grab）と解放（release）。
  • **永続化**: サーバー再起動を跨ぐためのファイルベースのデータ保存と復元（MD5チェックサムによる整合性検証を含む）。
  • **ライフサイクル管理**: プロセス終了時のクリーンアップと、MPM（Multi-Processing Module）環境下での子プロセスへの継承。

重要なシンボル

データ構造

  • ap_slotmem_instance_t: 共有メモリインスタンスの管理構造体。SHMへのポインタ、スロットのメタデータ、利用状況フラグ配列を保持。
  • sharedslotdesc_t: スロットの構成情報（サイズ、個数、型）を保持するヘッダー。

プロバイダインターフェース

  • storage: ap_slotmem_provider_t 構造体。外部モジュールから利用される関数ポインタ群（create, attach, grab, put, get 等）。

ライフサイクル・管理関数

  • slotmem_create / slotmem_attach: インスタンスの初期化および既存SHMへの接続。
  • slotmem_grab / slotmem_release: スロットの利用権の排他制御と状態更新。
  • store_slotmem / restore_slotmem: 永続化ファイルへの書き出しと読み込み。

制御フローとデータフロー

1. **初期化**: pre_config でグローバルプールを初期化し、post_config で cleanup_slotmem を登録。 2. **インスタンス作成**: slotmem_create が呼ばれると、slotmem_filenames でパスを決定し、apr_shm_create でメモリを確保。sharedslotdesc_t を先頭に配置し、後続にデータ領域と inuse フラグ配列をレイアウトする。 3. **データアクセス**: slotmem_get / slotmem_put は、slotmem_dptr を介してオフセット計算を行い、指定されたスロットのメモリ位置を特定して memcpy を実行。 4. **永続化フロー**: store_slotmem は、データ領域のMD5ハッシュを計算し、データ＋ハッシュ＋記述子をファイルに書き出す。restore_slotmem はこれを読み込み、ハッシュ照合により整合性を確認する。

直接依存

  • ap_slotmem.h: プロバイダ定義と型定義。
  • httpd.h, http_main.h: サーバー設定とログ出力。
  • ap_mpm.h: MPMの世代管理（AP_MPMQ_GENERATION）によるファイル名の一意性確保。

関連コード

  • modules/slotmem/mod_slotmem_plain.c: 同一インターフェースを持つメモリ内（非共有）実装。
  • include/ap_slotmem.h: 共通のプロバイダAPI定義。

読者向けメモ

  • **メモリレイアウト**: 共有メモリは [desc] [num_free] [data_slots] [inuse_array] の順で配置されます。AP_SLOTMEM_OFFSET 等の定数でアライメントが厳密に管理されています。
  • **メモリ所有権**: gpool（グローバルプール）がインスタンスの生存期間を管理します。cleanup_slotmem はプロセス終了時にSHMを破棄しますが、永続化が有効な場合は終了前に store_slotmem が呼ばれます。
  • **エッジケース**: is_child_process は WIN32 環境でのプロセス継承を考慮しています。また、restore_slotmem は古い永続化ファイルとの互換性を保つため、EOF判定によるMD5チェックのスキップ処理を含みます。
  • **省略箇所**: 本ドキュメントでは、MD5計算の詳細や、各OS固有のファイルシステムAPIの差異については詳細を省略しています。これらは apr ライブラリの抽象化に依存しています。
