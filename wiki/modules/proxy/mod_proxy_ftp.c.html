modules/proxy/mod_proxy_ftp.c

概要

mod_proxy_ftp.c は、Apache HTTP Server のプロキシサブシステムにおいて、FTP プロトコルを介したリソース取得およびディレクトリ一覧表示機能を提供するモジュールです。FTP サーバーへの制御接続とデータ接続を管理し、FTP のディレクトリリストを HTML に変換してクライアントへ提供する役割を担います。

モジュール内での役割

modules/proxy サブシステムの一部として、mod_proxy が提供する汎用的なプロキシフレームワークに FTP 特有の処理を統合します。proxy_hook_scheme_handler を通じて ftp:// スキームのリクエストを処理し、proxy_hook_canon_handler で URL の正規化を行います。

責務

  • **設定管理**: FTP 固有のディレクトリ設定（ワイルドカード処理、文字セット等）の保持とマージ。
  • **プロトコル制御**: FTP コマンド（USER, PASS, CWD, EPSV/PASV/PORT, RETR, LIST）の送受信とステータスコードの解釈。
  • **データ転送**: 制御接続とは別のデータ接続を確立し、バイナリまたは ASCII モードでのデータストリーム転送。
  • **ディレクトリ変換**: FTP サーバーから取得した raw なディレクトリリストを、PROXY_SEND_DIR フィルタを通じて HTML 形式へ変換。

重要なシンボル

設定構造体

  • proxy_ftp_dir_conf: ディレクトリ単位の設定（ProxyFtpListOnWildcard 等）を保持。
  • proxy_dir_ctx_t: ディレクトリ一覧表示フィルタの実行状態（バッファ、解析状態）を保持。

コアハンドラ・フック

  • proxy_ftp_handler: ftp:// スキームのメインリクエスト処理関数。
  • proxy_ftp_canon: FTP URL の正規化（パス、クエリ、ポートの解析）。
  • proxy_send_dir_filter: ディレクトリリストを HTML に変換する出力フィルタ。

プロトコルヘルパー

  • proxy_ftp_command: FTP コマンド送信とレスポンス取得の汎用ラッパー。
  • ftp_getrc_msg: FTP サーバーからのレスポンス行を解析し、ステータスコードとメッセージを抽出。
  • ftp_string_read: ネットワークバケットから 1 行ずつ読み込むためのヘルパー。

制御フローとデータフロー

1. **リクエスト開始**: proxy_ftp_handler が呼び出され、URL を解析して接続先を決定。 2. **接続確立**: ap_proxy_acquire_connection でバックエンド接続を取得し、制御接続を確立。 3. **認証と移動**: USER/PASS コマンドでログインし、CWD で対象ディレクトリへ移動。 4. **データ接続**: EPSV -> PASV -> PORT の順でデータ接続を試行。 5. **データ取得**: RETR（ファイル）または LIST（ディレクトリ）を発行。 6. **フィルタ処理**: LIST の場合、proxy_send_dir_filter がデータを逐次読み込み、HTML タグを付与してクライアントへ送信。 7. **終了**: QUIT を送信し、接続を解放。

直接依存

  • mod_proxy.h: プロキシフレームワークの基盤。
  • apr_version.h: APR バージョン情報。
  • time.h: MDTM コマンドによるタイムスタンプ処理用。

関連コード

  • modules/proxy/mod_proxy.c: プロキシモジュールのメインエントリポイント。

読者向けメモ

  • **メモリ管理**: r->pool を多用しており、リクエスト終了時に自動解放されます。ただし、接続の生存期間に依存するデータは r->connection->pool を使用しています。
  • **エッジケース**: EPSV や PASV が失敗した場合のフォールバックロジックが複雑です。ファイアウォール環境下での PORT モードの挙動には注意が必要です。
  • **省略箇所**: 本ファイルは FTP プロトコルの全仕様を網羅しているわけではなく、プロキシとして必要な最小限のコマンドセットに絞られています。また、ディレクトリリストの HTML 変換は非常に限定的な正規表現（ls -s1 形式など）に依存しており、多様な FTP サーバーの出力形式には完全対応していません。
  • **注意**: ftp_string_read は ap_rgetline と類似の機能を持っていますが、歴史的経緯により独自実装されています。
