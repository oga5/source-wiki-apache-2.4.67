modules/proxy/ajp_link.c

概要

ajp_link.c は、Apache HTTP Server の AJP (Apache JServ Protocol) プロキシモジュールにおいて、ネットワークソケットを介した低レベルなデータ送受信を担当するファイルです。

モジュール内での役割

mod_proxy_ajp がバックエンドのアプリケーションサーバー（Tomcat 等）と通信する際の「物理層」に近い役割を果たします。上位層が構築した ajp_msg_t 構造体を、実際のネットワークバイトストリームへ変換・送出します。

責務

  • **メッセージ送信**: ajp_msg_t に格納されたデータをソケット経由で確実に送信する。
  • **メッセージ受信**: AJP プロトコルのヘッダーとボディを分離して読み取り、完全なメッセージとして再構築する。
  • **ソケット制御**: apr_socket_recv を用いたブロッキング読み取りと、エラーハンドリング（EOFやタイムアウトの判定）。

重要なシンボル

  • ajp_ilink_send: メッセージをソケットへ書き出す公開関数。送信完了までループ処理を行う。
  • ajp_ilink_receive: ソケットからヘッダーを読み取り、サイズを検証した後にボディを読み取る一連の受信処理を行う。
  • ilink_read: 受信用の内部ヘルパー関数。指定されたバイト数が揃うまでソケットから読み取りを継続する。

制御フローとデータフロー

1. **送信時**: ajp_msg_end() でメッセージの長さを確定させた後、ajp_ilink_send がソケットに対して apr_socket_send を呼び出し、バッファが空になるまでループします。 2. **受信時**:

  • まず ilink_read で固定長のヘッダーを読み取ります。
  • ajp_msg_check_header でヘッダーを検証し、続くボディの長さを特定します。
  • 特定された長さ分だけ再度 ilink_read を呼び出し、メッセージ全体を ajp_msg_t に格納します。

直接依存

  • ajp.h: AJP プロトコルの定義、ajp_msg_t 構造体、およびメッセージ操作用ユーティリティ関数。

関連コード

  • ajp_msg.c: ajp_msg_t 構造体の初期化やメモリ管理を行う。
  • mod_proxy_ajp.c: 本ファイルの関数を呼び出し、プロキシリクエストのライフサイクルを管理するメインモジュール。

読者向けメモ

  • 本ファイルは「バイトストリームの送受信」に特化しており、AJP プロトコルの意味論（リクエスト/レスポンスの解釈）には関与しません。
  • ilink_read は APR_STATUS_IS_EAGAIN を適切に処理しており、ノンブロッキングソケット環境下でも読み取りが完了するまで待機する設計になっています。
  • エラー発生時は APLOGNO を付与したログ出力が行われるため、デバッグ時にはこれらの番号を追跡することで、通信のどの段階で失敗したかを特定可能です。
