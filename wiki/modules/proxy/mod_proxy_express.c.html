modules/proxy/mod_proxy_express.c

概要

mod_proxy_express は、DBM ファイルを使用してリクエストのホスト名からバックエンドのプロキシ先を動的に解決する Apache HTTP Server モジュールです。設定ファイルに静的な ProxyPass を記述することなく、DBM ファイル内のキー（ホスト名）と値（バックエンド URL）のペアに基づいてリクエストをルーティングします。

モジュール内での役割

mod_proxy サブシステムの一部として動作し、リクエストの翻訳フェーズ（translate_name）で介入します。他のプロキシモジュールが静的な設定に依存するのに対し、本モジュールは外部データソース（DBM）を利用して実行時にルーティング先を決定する「動的プロキシ」の役割を担います。

責務

  • **設定管理**: DBM ファイルのパス、タイプ、および機能の有効/無効をサーバー単位で管理。
  • **動的ルーティング**: リクエストのホスト名をキーとして DBM を検索し、バックエンド URL を取得して r->filename を書き換える。
  • **自動設定**: 解決されたバックエンドに対して、必要に応じて ProxyPassReverse エントリを動的に追加する。

重要なシンボル

設定構造体

  • express_server_conf: サーバーごとの設定を保持。dbmfile（パス）、dbmtype（DBM形式）、enabled（フラグ）を格納。

設定ハンドラ

  • set_dbmfile, set_dbmtype, set_enabled: httpd.conf でのディレクティブを処理し、express_server_conf を更新する。

ライフサイクル・フック

  • server_create / server_merge: 設定構造体の初期化およびサーバー間での設定継承。
  • post_config: mod_proxy がロードされているかを確認。
  • xlate_name: リクエスト処理のメインロジック。DBM 検索とリクエストの書き換えを実行。

制御フローとデータフロー

1. **初期化**: register_hooks により xlate_name が APR_HOOK_FIRST で登録される。 2. **リクエスト処理 (xlate_name)**:

  • sconf->enabled を確認し、無効なら DECLINED。
  • apr_dbm_open2 (または apr_dbm_open_ex) で DBM ファイルを開く。
  • ap_get_server_name(r) で取得したホスト名をキーに apr_dbm_fetch を実行。
  • 成功時、r->filename を proxy:<backend><uri> に書き換え、r->handler を proxy-server に設定。
  • dconf->raliases を走査し、該当バックエンドの ProxyPassReverse が存在しなければ動的に追加。

    • 3. **終了**: apr_dbm_close でリソースを解放。

直接依存

  • mod_proxy.h: プロキシ機能の共通定義（proxy_dir_conf, proxy_alias 等）。
  • apr_dbm.h: DBM ファイル操作のための APR 抽象化レイヤー。
  • apr_version.h / apu_version.h: DBM ドライバ API のバージョン差異を吸収するための条件分岐に使用。

関連コード

  • modules/proxy/mod_proxy.c: プロキシのコア機能。本モジュールが proxy-server ハンドラをセットすることで連携する。
  • modules/proxy/proxy_util.c: プロキシ関連のユーティリティ関数群。

読者向けメモ

  • **メモリ管理**: DBM 検索や文字列操作には r->pool を使用しており、リクエスト終了時に自動的に解放されます。
  • **スレッド安全性**: DBM ファイルは読み取り専用 (APR_DBM_READONLY) で開かれるため、並行アクセスに対して安全です。
  • **制限事項**: ProxyPassReverse の動的追加は、現在のプロセス/スレッドの dconf に対して行われます。このため、設定の反映範囲はプロセス境界に依存する可能性がある点に注意してください。
  • **省略箇所**: DBM の内部構造や、特定の DBM ドライバ（GDBM, SDBM 等）の挙動については、APR の仕様に依存するため本ファイルでは詳細を扱っていません。
