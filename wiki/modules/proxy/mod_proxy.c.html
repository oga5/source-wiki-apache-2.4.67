modules/proxy/mod_proxy.c

概要

mod_proxy.c は Apache HTTP Server のプロキシ機能の中核となるモジュールです。リクエストの転送、ロードバランシング、およびプロキシ設定の管理を統括します。このファイルは、プロキシのライフサイクル管理、設定の解析、およびリクエスト処理の各フェーズ（翻訳、修正、ハンドラ実行）を定義する巨大なエントリポイントです。

モジュール内での役割

mod_proxy サブシステムにおける「司令塔」です。個別のプロトコルハンドラ（mod_proxy_http, mod_proxy_ajp 等）やユーティリティ（proxy_util.c）を統合し、Apache のリクエスト処理パイプラインとプロキシロジックを接続します。

責務

  • **設定管理**: ProxyPass, ProxyRemote, BalancerMember などのディレクティブを解析し、サーバーおよびディレクトリごとの設定構造体を構築・マージする。
  • **リクエストルーティング**: translate_name フックを通じて、リクエストがプロキシ対象かどうかを判定し、適切なハンドラへ割り当てる。
  • **ライフサイクル管理**: プロキシ用ミューテックスの初期化、子プロセスでのワーカー初期化、およびシャットダウン時のリソース解放。
  • **プロキシハンドラ**: proxy-server ハンドラを実装し、バックエンドへの接続、ロードバランシング、エラー処理、およびレスポンスの転送を制御する。

重要なシンボル

設定構造体

  • proxy_server_conf: サーバー単位のプロキシ設定（ワーカー、バランサー、リモートプロキシリスト等）。
  • proxy_dir_conf: ディレクトリ単位の設定（ProxyPass エイリアス、エラーオーバーライド設定等）。

コマンド・設定関数

  • proxy_cmds: Apache の設定ディレクティブを処理するコマンドテーブル。
  • create_proxy_config / merge_proxy_config: サーバー設定の生成とマージ。
  • add_pass / add_member: ProxyPass および BalancerMember の解析とワーカー/バランサーの定義。

リクエスト処理フック

  • proxy_handler: プロキシリクエストのメインハンドラ。
  • proxy_translate_name: URL を proxy:<URL> 形式に変換し、プロキシ対象としてマークする。
  • proxy_walk: <Proxy> セクションの設定をリクエストに適用する。

外部連携

  • proxy_mutex: プロキシの共有リソースを保護するグローバルミューテックス。
  • proxy_hcmethods[]: ヘルスチェックメソッドの定義テーブル。

制御フローとデータフロー

1. **設定フェーズ**: proxy_cmds がディレクティブを読み込み、proxy_server_conf にワーカーやバランサーを登録。merge_proxy_config により親サーバーの設定が継承される。 2. **リクエスト翻訳**: proxy_translate_name が URL を解析し、マッチする ProxyPass があれば r->filename を proxy:... に書き換え、ハンドラを proxy-server に設定。 3. **ハンドラ実行**: proxy_handler が呼び出される。

  • ap_proxy_pre_request で適切なワーカー/バランサーを選択。
  • proxy_run_scheme_handler を呼び出し、プロトコル固有のハンドラ（HTTP, AJP等）に処理を委譲。
  • 失敗時は max_attempts に基づき再試行。

    • 4. **クリーンアップ**: ap_proxy_post_request を呼び出し、接続の解放や統計情報の更新を行う。

直接依存

  • mod_proxy.h: 公開インターフェースとデータ構造。
  • proxy_util.h: ユーティリティ関数（ワーカー管理、URL正規化など）。
  • http_config.h, http_request.h: Apache のリクエスト処理基盤。
  • apr_global_mutex.h: プロセス間同期。

関連コード

  • proxy_util.c: ワーカーの初期化、共有メモリ管理、URL解析などの低レベル実装。
  • mod_proxy_http.c, mod_proxy_ajp.c: 具体的なプロトコルハンドラの実装。
  • mod_proxy_balancer.c: ロードバランシングアルゴリズムの実装。

読者向けメモ

  • **メモリ管理**: 設定構造体は apr_pool_t を使用しており、リクエスト終了時やサーバー再起動時に適切に解放される設計です。
  • **共有メモリ**: proxy_worker や proxy_balancer の一部の状態は共有メモリ上に保持されるため、proxy_mutex による排他制御が不可欠です。
  • **設計の深さ**: このファイルは「プロキシの枠組み」を提供することに特化しており、具体的なプロトコル処理はフック（scheme_handler）を通じて外部モジュールへ委譲されています。
  • **省略箇所**: 本ファイル内の alias_match_servlet などの文字列マッチングロジックは、特定のパスパラメータ処理に特化しており、複雑な正規表現やパス操作を含みます。これらは mod_jk 等の挙動を模倣しています。
