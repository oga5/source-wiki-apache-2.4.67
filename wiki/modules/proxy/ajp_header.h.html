modules/proxy/ajp_header.h

概要

ajp_header.h は、Apache JServ Protocol (AJP) 通信において使用される定数および識別子を定義するヘッダーファイルです。AJP プロトコルは、Web サーバー（Apache HTTP Server）とバックエンドのアプリケーションサーバー（Tomcat 等）間のバイナリ通信を規定しており、本ファイルはそのプロトコル仕様の「辞書」として機能します。

モジュール内での役割

modules/proxy サブシステムにおいて、AJP プロトコルを扱う各ソースファイル（ajp_msg.c, ajp_link.c 等）が共通して参照する定数定義を提供します。プロトコル上のマジックナンバーや属性コードをシンボル化することで、実装の可読性と保守性を担保しています。

責務

  • **属性コードの定義**: リクエスト属性（コンテキスト、SSL情報、認証情報等）の識別子定義。
  • **メソッドコードの定義**: HTTP メソッド（GET, POST 等）を AJP プロトコル上で表現するための数値コード定義。
  • **ヘッダーコードの定義**: 頻出するリクエストおよびレスポンスヘッダーの数値コード（0xA001〜）の定義。

重要なシンボル

リクエスト属性識別子 (SC_A_*)

  • SC_A_CONTEXT, SC_A_SERVLET_PATH: アプリケーションのコンテキスト情報。
  • SC_A_SSL_*: SSL/TLS 関連情報（証明書、暗号スイート、プロトコル等）。
  • SC_A_ARE_DONE: 属性リストの終端を示すマーカー。

HTTP メソッドコード (SC_M_*)

  • SC_M_GET, SC_M_POST, SC_M_OPTIONS 等: RFC 2616 等で定義された標準的な HTTP メソッドを 1 バイトの数値にマッピングしたもの。

ヘッダー識別子 (SC_*, SC_RESP_*)

  • SC_ACCEPT, SC_HOST 等: リクエストヘッダーの数値表現。
  • SC_RESP_CONTENT_TYPE, SC_RESP_STATUS 等: レスポンスヘッダーの数値表現。これらは AJP 通信の効率化のために文字列ではなく数値としてエンコードされます。

制御フローとデータフロー

本ファイル自体は定数定義のみを行うヘッダーであるため、直接的なロジックフローは持ちません。しかし、ペアとなる ajp_header.c 内の sc_for_req_header 関数などが、ここで定義された SC_* 定数を使用して、文字列ヘッダーをバイナリプロトコル用の数値コードへ変換するフローを形成します。

直接依存

  • なし（純粋な定数定義ファイル）

関連コード

  • modules/proxy/ajp_header.c: 本ヘッダーで定義された定数を用いて、文字列と数値コードの相互変換を行う実装が含まれます。
  • modules/proxy/ajp.h: AJP プロトコルの構造体やメッセージ形式を定義する上位ヘッダー。
  • modules/proxy/ajp_msg.c: 本ヘッダーの定数を使用して、実際にネットワークへ送出するメッセージを構築するコード。

読者向けメモ

  • **メモリとレイアウト**: 本ファイルで定義される定数はコンパイル時に解決されるため、実行時のメモリ消費やレイアウトへの影響はありません。
  • **設計上の注意**: ここで定義されている数値コード（特に 0xA001 等）は AJP プロトコル仕様に厳密に準拠しています。Tomcat 側の実装と同期しているため、値を変更する場合はプロトコル互換性を損なうリスクがあります。
  • **省略された詳細**: 本ファイルはプロトコルの「定義」に特化しており、実際のパケット構築ロジックやエラーハンドリングについては ajp_msg.c 等の関連ファイルを参照してください。
