modules/proxy/mod_proxy_fdpass.c

概要

mod_proxy_fdpass.c は、Apache HTTP Server のプロキシサブシステムにおいて、クライアントとの接続（ソケット）を Unix ドメインソケット経由で外部プロセスへ引き渡す（FD Passing）ためのモジュールです。

モジュール内での役割

mod_proxy のスキームハンドラの一つとして機能します。通常の HTTP プロキシがリクエストを転送するのに対し、本モジュールは「接続そのもの」を別のプロセスへ移譲することで、接続のハンドオフを実現します。

責務

  • fd:// スキームの URL を解釈し、指定された Unix ドメインソケットへ接続する。
  • sendmsg を使用し、クライアントとの接続ソケット（ファイルディスクリプタ）を外部プロセスへ送信する。
  • 接続引き渡し後、Apache 側がクライアントソケットを誤ってクローズしないよう、ダミーソケットへの差し替えを行う。
  • 拡張可能なフラッシュ機構（proxy_fdpass_flusher）を提供し、引き渡し前にレスポンスをフラッシュする。

重要なシンボル

  • proxy_fdpass_handler: fd:// スキームを処理するメインハンドラ。
  • send_socket: SCM_RIGHTS を用いてファイルディスクリプタを Unix ドメインソケット経由で送信する低レイヤー関数。
  • proxy_fdpass_canon: URL の正規化を行い、fd:// スキームを識別する。
  • builtin_flush: デフォルトのレスポンスフラッシュプロバイダ。

制御フローとデータフロー

1. **正規化**: proxy_fdpass_canon が fd:// URL を検出し、内部パスへ変換。 2. **接続**: proxy_fdpass_handler が get_socket_from_path を呼び出し、ターゲットとなる Unix ドメインソケットへ接続。 3. **フラッシュ**: 設定された flusher プロバイダを呼び出し、バッファをフラッシュして接続状態を整える。 4. **移譲**: send_socket が sendmsg を実行し、クライアントソケットの FD を相手プロセスへ渡す。 5. **保護**: Apache コアが接続をクローズするのを防ぐため、ap_set_core_module_config で接続ソケットをダミーに置き換える。

読者向けメモ

  • **プラットフォーム制限**: 本モジュールは CMSG_DATA マクロが定義されている環境（主に Unix 系）でのみ動作します。
  • **FD Passing の仕組み**: SCM_RIGHTS を用いた FD の受け渡しは、プロセス間での接続の「所有権」を移転させる強力な手法ですが、相手プロセスが正しく FD を受け取り、適切に管理できることが前提となります。
  • **ダミーソケットの重要性**: 接続引き渡し後に Apache が元のソケットをクローズすると、クライアントとの通信が切断されます。ダミーへの差し替えは、Apache のクリーンアップ処理を無害化するための重要なハックです。
