modules/proxy/mod_proxy_connect.c

概要

HTTP CONNECT メソッドを実装し、プロキシ経由での SSL/TLS トンネリングを可能にする Apache HTTP Server のモジュールです。クライアントとリモートサーバー間の透過的なデータ転送を仲介します。

モジュール内での役割

mod_proxy サブシステムの一部として、特定のプロトコル（CONNECT）を処理するハンドラを提供します。他のプロキシモジュール（HTTP, AJP 等）がリクエストの変換を行うのに対し、本モジュールは接続確立後にバイナリレベルのトンネルを構築する点に特徴があります。

責務

  • **設定管理**: AllowCONNECT ディレクティブによる接続先ポートのホワイトリスト管理。
  • **接続確立**: リクエスト URL からの宛先解決、DNS ルックアップ、およびバックエンドへのソケット接続。
  • **トンネル構築**: クライアントとバックエンド間の双方向データ転送のセットアップと実行。
  • **プロトコルハンドリング**: HTTP/1.0 200 Connection Established の応答生成、または上位プロキシへの CONNECT リクエスト転送。

重要なシンボル

設定構造体

  • connect_conf: サーバーごとの設定を保持。allowed_connect_ports（許可ポート範囲のリスト）を管理。
  • port_range: 許可ポートの範囲（first, last）を定義。

ハンドラ関数

  • proxy_connect_handler: メインの処理関数。接続先決定、バックエンド接続、トンネル実行、終了処理を統括。
  • proxy_connect_canon: CONNECT メソッドのリクエストを正規化（検証）する。
  • ap_proxy_connect_register_hook: フックの登録（scheme_handler, canon_handler）。

設定コマンド

  • set_allowed_ports: AllowCONNECT ディレクティブのパースと connect_conf への格納。

制御フローとデータフロー

1. **初期化**: proxy_connect_handler がリクエストをインターセプト。 2. **宛先解決**: apr_uri_parse_hostinfo で URL を解析し、apr_sockaddr_info_get でバックエンドの IP を取得。 3. **検証**: allowed_port でポート番号をチェックし、ap_proxy_checkproxyblock2 でアクセス制限を確認。 4. **接続**: ap_proxy_connect_to_backend でバックエンドとソケット接続。 5. **応答**:

  • 上位プロキシ経由の場合: CONNECT リクエストを転送。
  • 直接接続の場合: クライアントへ 200 Connection Established を送信。

    • 6. **トンネル**: ap_proxy_tunnel_create と ap_proxy_tunnel_run を呼び出し、ソケット間のデータ転送をループ実行。 7. **終了**: ap_lingering_close により接続を適切にクローズ。

直接依存

  • mod_proxy.h: プロキシサブシステムの共通 API（proxy_worker, proxy_server_conf 等）。
  • apr_poll.h: ソケットのポーリングおよび非同期 I/O 関連の APR プリミティブ。

関連コード

  • proxy_util.c: ap_proxy_connect_to_backend や ap_proxy_tunnel_* などのトンネル関連の基盤関数が定義されている。
  • mod_proxy.c: プロキシモジュールのメインエントリポイント。

読者向けメモ

  • **メモリ管理**: r->pool を使用してリクエスト単位のメモリを管理。トンネル作成時に割り当てられるリソースは、リクエスト終了時に自動的に解放されます。
  • **制約**: 本モジュールは透過的なトンネルを目的としているため、HTTP ヘッダーの解析やログ出力は最小限です。
  • **未実装/FIXME**: ソース内のコメントにある通り、転送バイト数のログ記録や、一部の認証チェックは未実装です。
  • **レイアウト**: connect_conf はサーバー設定（server_rec）に紐付いており、merge_config により親サーバーの設定を継承可能です。
