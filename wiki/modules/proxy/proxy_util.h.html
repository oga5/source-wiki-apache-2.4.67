modules/proxy/proxy_util.h

概要

proxy_util.h は、Apache HTTP Server の mod_proxy モジュール内部で使用されるプライベートなユーティリティ関数およびデータ構造を定義するヘッダーファイルです。プロキシ機能の基盤となる共通処理を提供します。

モジュール内での役割

mod_proxy サブシステムにおける「共通基盤」の役割を担います。各プロキシスキーム（HTTP, AJP, FCGI 等）の実装が依存する、URLの正規化や設定値の補間、アクセス制御の判定といった共通ロジックを公開しています。

責務

  • **URL 正規化**: リクエストされた URL をプロキシ処理に適した形式に変換する。
  • **設定値の動的評価**: 環境変数を含む設定文字列の補間（Interpolation）。
  • **アクセス制御判定**: ProxyBlock や ProxyRemote 等の設定に基づくホスト名・ドメイン・IPアドレスの照合。
  • **モジュール間連携**: mod_proxy 内部のフック登録や、グローバルな状態管理（バランサーのワーカー数など）の提供。

重要なシンボル

  • ap_proxy_canon_url: リクエストされた URL を正規化する主要関数。
  • ap_proxy_interpolate: 設定文字列内の環境変数を現在のリクエストコンテキストに基づいて展開する。
  • ap_proxy_is_ipaddr / ap_proxy_is_domainname: 設定されたアクセス制御ルールがどのタイプ（IP、ドメイン等）かを判定する。
  • proxy_lb_workers: ロードバランサーが管理するワーカーのグローバルカウンタ。

制御フローとデータフロー

本ファイルはインターフェース定義のみを提供します。実際のデータフローは proxy_util.c に委譲されており、主に以下の流れで利用されます： 1. mod_proxy のリクエスト処理フェーズで ap_proxy_canon_url が呼び出され、URL が正規化される。 2. 設定値が必要な場面で ap_proxy_interpolate が呼ばれ、request_rec から環境変数が抽出・結合される。 3. 接続先判定時に ap_proxy_is_* 系関数が dirconn_entry を評価し、接続可否を決定する。

直接依存

  • mod_proxy.h: モジュールの基本構造体定義。
  • apr_pools.h (間接的): メモリ管理用。
  • httpd.h (間接的): request_rec 等の Apache HTTP Server の基本型。

関連コード

  • modules/proxy/proxy_util.c: 本ヘッダーで宣言された関数の実装本体。
  • modules/proxy/mod_proxy.c: プロキシのメイン処理フロー。
  • modules/proxy/mod_proxy_balancer.c: ロードバランサー機能の実装。

読者向けメモ

  • 本ファイルは mod_proxy の内部 API です。外部モジュールからの利用は推奨されません。
  • PROXY_DECLARE マクロは、Windows 等のプラットフォームでシンボルを適切にエクスポートするために使用されています。
  • 複雑なロジックを追う際は、まず ap_proxy_canon_url の実装（proxy_util.c 内）から読み始めるのが効率的です。
