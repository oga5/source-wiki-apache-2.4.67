modules/proxy/mod_proxy.h

概要

mod_proxy.h は、Apache HTTP Server のプロキシサブシステムにおける中心的なヘッダーファイルです。プロキシモジュール（mod_proxy）および各プロトコルハンドラ（mod_proxy_http, mod_proxy_ajp 等）が共有するデータ構造、定数、および API を定義します。

モジュール内での役割

プロキシサブシステムの「契約」を定義するインターフェース層です。設定構造体、ワーカー（バックエンド接続先）、バランサー（負荷分散）、および接続管理のための共通定義を提供し、各プロキシモジュールがこの定義に従うことで、一貫したプロキシ動作を保証します。

責務

  • **設定管理**: サーバー全体およびディレクトリ単位のプロキシ設定構造体（proxy_server_conf, proxy_dir_conf）の定義。
  • **バックエンド抽象化**: バックエンドサーバーを表す proxy_worker と、負荷分散グループを表す proxy_balancer の管理。
  • **接続ライフサイクル**: バックエンドへの接続（proxy_conn_rec）の生成、取得、解放、および再利用の管理。
  • **フック定義**: プロキシ処理の各ステージ（pre_request, scheme_handler 等）で利用される外部フックの宣言。
  • **ユーティリティ提供**: URL 正規化、ヘッダーマッピング、UDS（Unix Domain Socket）処理、およびデータ転送のための共通関数群の宣言。

重要なシンボル

設定構造体

  • proxy_server_conf: サーバー全体のプロキシ設定（バランサーリスト、ワーカー、タイムアウト等）。
  • proxy_dir_conf: ディレクトリ単位の設定（ProxyPassReverse 等のパス変換ルール）。

バックエンド管理

  • proxy_worker: バックエンドサーバーの定義。共有メモリ（proxy_worker_shared）とプロセスローカルな状態を保持。
  • proxy_balancer: 負荷分散グループ。複数のワーカーを束ね、proxy_balancer_method を介して分散アルゴリズムを適用。
  • proxy_conn_rec: バックエンドへの接続状態。ソケット、接続プール、SSL 状態を保持。

ライフサイクル・制御

  • proxy_hcmethods_t: ヘルスチェック手法の定義。
  • proxy_tunnel_rec: WebSocket 等のトンネル接続用構造体。

制御フローとデータフロー

1. **リクエスト処理**: pre_request フックにより、リクエストに適したワーカー/バランサーが選択されます。 2. **接続取得**: ap_proxy_acquire_connection を呼び出し、ワーカーの接続プール（apr_reslist_t）から既存の接続を再利用するか、新規作成します。 3. **バックエンド通信**: ap_proxy_connect_backend でソケットを確立し、ap_proxy_transfer_between_connections を用いてクライアントとバックエンド間でデータを転送します。 4. **終了処理**: ap_proxy_release_connection により、接続をプールへ戻すか、必要に応じて破棄します。

  • **Apache Core**: httpd.h, http_config.h, http_request.h 等、サーバーのコアリクエスト処理機能。

読者向けメモ

  • **メモリ所有権**: proxy_server_conf 等の構造体は、Apache の設定プール（pool）によってライフサイクルが管理されます。
  • **共有メモリ**: proxy_worker_shared や proxy_balancer_shared は ap_slotmem を介してプロセス間で共有されます。マルチプロセス環境での更新には proxy_mutex による排他制御が必須です。
  • **省略された詳細**: 本ファイルはインターフェース定義に特化しており、具体的な負荷分散アルゴリズムの実装や、各プロトコル固有のパケット解析ロジックは各モジュール側に委譲されています。
  • **注意点**: proxy_conn_rec の uds_path 等、バージョンアップに伴い構造体が変更されることがあるため、バイナリ互換性には注意が必要です。
