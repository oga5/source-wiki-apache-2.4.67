modules/proxy/mod_proxy_fdpass.h

概要

mod_proxy_fdpass.h は、Apache HTTP Server のプロキシサブシステムにおいて、Unixドメインソケットを介したファイルディスクリプタ（FD）の受け渡し機能を提供するためのインターフェース定義ファイルです。

モジュール内での役割

mod_proxy の拡張モジュールの一つである mod_proxy_fdpass の公開ヘッダーです。このモジュールは、クライアントとの接続（ソケット）を別のプロセスへ引き継ぐための仕組みを提供します。

責務

  • FD受け渡し処理における「フラッシャー（flusher）」機能のプラグイン構造を定義する。
  • 外部モジュールが proxy_fdpass_flusher プロバイダーを登録・利用するための共通型定義を提供する。

重要なシンボル

  • PROXY_FDPASS_FLUSHER: プロバイダー検索に使用される識別子（文字列定数）。
  • proxy_fdpass_flush: FD受け渡し時にバッファのフラッシュを行うためのコールバック関数を保持する構造体。

制御フローとデータフロー

本ヘッダー自体はデータ構造の定義のみを行います。実際のフローは mod_proxy_fdpass.c に依存します： 1. mod_proxy_fdpass がリクエストを処理する際、ap_lookup_provider を通じて PROXY_FDPASS_FLUSHER を検索します。 2. 登録された proxy_fdpass_flush 構造体の flusher 関数が呼び出され、接続の引き継ぎ前に必要なデータフラッシュ処理が実行されます。

直接依存

  • mod_proxy.h: プロキシモジュールの基本構造体および API 定義。

関連コード

  • modules/proxy/mod_proxy_fdpass.c: 本ヘッダーで定義された構造体を利用し、sendmsg を用いて実際に FD を転送する実装。
  • modules/proxy/mod_proxy.c: プロキシの基盤となるモジュール。

読者向けメモ

  • **プラットフォーム依存**: 本機能は Unix 系 OS の sendmsg および SCM_RIGHTS に強く依存しています。非 Unix 環境では動作しません。
  • **拡張性**: proxy_fdpass_flush 構造体を利用することで、特定のプロトコルやアプリケーション要件に応じたフラッシュ処理を動的に追加可能です。
  • **利用上の注意**: このヘッダーは内部 API です。Apache のモジュール開発において、FD の受け渡しをカスタマイズする必要がある場合のみ参照してください。
