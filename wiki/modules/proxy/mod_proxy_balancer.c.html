modules/proxy/mod_proxy_balancer.c

概要

mod_proxy_balancer は、Apache HTTP Server のプロキシサブシステムにおけるロードバランサ機能を提供するモジュールです。複数のバックエンドワーカー間での負荷分散、セッションスティッキー（スティッキーセッション）、および動的なメンバー管理（balancer-manager）を担います。

モジュール内での役割

mod_proxy の拡張機能として動作し、balancer:// スキームを処理します。proxy_util.c で定義された共有メモリ（slotmem）を利用して、マルチプロセス環境下でバランサの状態とワーカーの統計情報を同期・共有します。

責務

  • **リクエストルーティング**: proxy_balancer_pre_request を通じて、負荷分散アルゴリズムに基づき最適なワーカーを選択する。
  • **セッション管理**: クッキーやパスパラメータを用いたスティッキーセッションの維持。
  • **状態管理と同期**: 共有メモリを用いたワーカーのヘルス状態、負荷状況、および統計情報の永続化と同期。
  • **管理インターフェース**: balancer-manager ハンドラによる、実行時のワーカーパラメータ変更やメンバー追加機能の提供。
  • **ヘルスチェック連携**: mod_proxy_hcheck 等の外部モジュールと連携し、ワーカーの稼働状態を監視・制御する。

重要なシンボル

ライフサイクル・初期化

  • balancer_pre_config: ミューテックスの登録とオプション関数の取得。
  • balancer_post_config: 共有メモリ（slotmem）の確保と各バランサ・ワーカーの初期化。
  • balancer_child_init: 子プロセス起動時の共有メモリへのアタッチ。

ルーティング・選択ロジック

  • proxy_balancer_pre_request: リクエスト処理の入口。バランサの選択とワーカーの決定を行う。
  • find_best_worker: ロードバランサアルゴリズム（lbmethod）を呼び出し、最適なワーカーを特定する。
  • find_session_route: スティッキーセッション情報から既存のワーカーを検索する。
  • find_route_worker: 特定のルートを持つワーカーを検索し、必要に応じてリトライやリダイレクトを処理する。

管理・状態更新

  • balancer_handler: balancer-manager のリクエストを処理する。
  • balancer_process_balancer_worker: 管理画面からのパラメータ変更をワーカー/バランサに適用する。
  • force_recovery: エラー状態のワーカーを再試行し、必要に応じて復旧を強制する。

制御フローとデータフロー

1. **リクエスト処理**: proxy_balancer_pre_request が呼ばれると、まず find_session_route でスティッキーセッションを試行。失敗した場合は find_best_worker でアルゴリズムに従いワーカーを選択する。 2. **同期**: balancer_post_config で作成された slotmem を通じて、全プロセスが同一の proxy_balancer および proxy_worker 構造体を共有する。 3. **排他制御**: PROXY_THREAD_LOCK/UNLOCK マクロにより、共有メモリへのアクセスを保護する。 4. **管理フロー**: balancer_handler は POST データを解析し、balancer_process_balancer_worker を経由して共有メモリ上のパラメータを更新する。CSRF 対策として nonce の検証を行う。

読者向けメモ

  • **メモリ管理**: バランサとワーカーのデータは slotmem を介して共有メモリに配置されます。プロセス間でのデータ整合性は gmutex によって保証されます。
  • **安全性**: balancer-manager は CSRF 攻撃を防ぐため、Referer ヘッダのチェックと nonce の照合を厳格に行います。
  • **制約**: balancer_post_config は設定読み込み時に二度呼ばれる可能性があるため、初期化の重複防止処理が含まれています。
  • **省略範囲**: 本ファイルはバランサの「制御」に特化しており、具体的な負荷分散アルゴリズム（byrequests, bytraffic 等）の実装は各 lbmethod プロバイダモジュールに委譲されています。
