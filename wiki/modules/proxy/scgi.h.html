modules/proxy/scgi.h

概要

scgi.h は、Apache HTTP Server の mod_proxy_scgi モジュールで使用される SCGI (Simple Common Gateway Interface) プロトコル関連の共有定義を保持するヘッダーファイルです。

モジュール内での役割

modules/proxy サブシステムにおいて、SCGI プロトコルを扱うための最小限の定数定義を提供します。主に mod_proxy_scgi.c と、プロキシユーティリティ間で一貫した設定値を共有するために存在します。

責務

  • SCGI プロトコル通信におけるデフォルトポート番号の定義。
  • プロキシモジュール間での設定値の整合性維持。

重要なシンボル

  • SCGI_DEF_PORT: SCGI プロトコルのデフォルトポート番号（4000）を定義するマクロ。プロトコル仕様ではなく、mod_proxy_scgi の慣習として定義されています。

制御フローとデータフロー

本ファイルは定数定義のみを提供するため、直接的な制御フローやデータフローは存在しません。このヘッダーをインクルードする各モジュールが、接続先ポートのデフォルト値として SCGI_DEF_PORT を参照します。

直接依存

  • なし（純粋なヘッダーファイルであり、他のヘッダーへの依存はありません）。

関連コード

  • modules/proxy/mod_proxy_scgi.c: 本ヘッダーの定義を実際に利用して SCGI 接続を処理するメインモジュール。
  • modules/proxy/mod_proxy.h: プロキシモジュール全体の基盤となるヘッダー。

読者向けメモ

  • 本ファイルは非常に小規模であり、主に「デフォルトポートの統一」という目的のために存在します。
  • SCGI_DEF_PORT はプロトコル仕様で規定されたものではないため、他の SCGI 実装と連携する際は、この値が環境設定と一致しているか確認してください。
