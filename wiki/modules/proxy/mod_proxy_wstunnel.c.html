modules/proxy/mod_proxy_wstunnel.c

概要

mod_proxy_wstunnel は、Apache HTTP Server における WebSocket プロトコル（ws: / wss:）のトンネリングを専門に扱うプロキシモジュールです。HTTP アップグレードリクエストをバックエンドへ転送し、双方向のデータストリームを確立します。

モジュール内での役割

mod_proxy サブシステムの一部として、特定のスキーム（ws, wss）に対するハンドラを提供します。mod_proxy_http と共存し、設定に応じてフォールバックを行う柔軟性を持ちます。

責務

  • **リクエスト処理**: WebSocket アップグレードリクエストの検証とバックエンドへの転送。
  • **トンネリング**: クライアントとバックエンド間の双方向通信を apr_pollset を用いて非同期的に中継。
  • **設定管理**: ProxyWebsocketFallbackToProxyHttp ディレクティブによる mod_proxy_http へのフォールバック制御。
  • **URL 正規化**: WebSocket 特有のスキームを処理し、バックエンド接続用の正規化された URL を生成。

重要なシンボル

設定とライフサイクル

  • proxyws_dir_conf: ディレクトリ単位の設定構造体。フォールバックの可否を保持。
  • create_proxyws_dir_config / merge_proxyws_dir_config: 設定の初期化と継承ロジック。
  • ws_proxy_hooks: proxy_hook_scheme_handler 等を登録し、mod_proxy_http より優先的に処理を行うためのフック設定。

プロキシ処理

  • proxy_wstunnel_handler: メインのエントリポイント。接続の確立からトンネリングの開始までを統括。
  • proxy_wstunnel_request: 実際のトンネリングループ。apr_pollset を使用してクライアントとバックエンドのソケットを監視し、データを転送。
  • proxy_wstunnel_canon: URL の解析と正規化。ws/wss スキームを処理し、バックエンド接続用の r->filename を構築。

制御フローとデータフロー

1. **フック**: proxy_wstunnel_check_trans がリクエストをインターセプトし、Upgrade ヘッダーの有無を確認。 2. **正規化**: proxy_wstunnel_canon が URL を解析し、proxy: スキームを付与して内部転送用形式に変換。 3. **ハンドラ**: proxy_wstunnel_handler がバックエンド接続を確保し、ap_proxy_connect_backend で接続を確立。 4. **トンネリング**: proxy_wstunnel_request が apr_pollset を作成し、クライアントとバックエンドのソケットを登録。ループ内で ap_proxy_transfer_between_connections を呼び出し、双方向のデータ転送を実行。 5. **終了**: どちらかの接続が切断されるかエラーが発生するまでループし、最後に接続を解放。

直接依存

  • mod_proxy.h: プロキシ共通のデータ構造（proxy_conn_rec, proxy_worker 等）と API。
  • http_config.h: モジュール設定およびディレクティブ定義用。

関連コード

  • modules/proxy/mod_proxy_http.c: 本モジュールがフォールバック先として依存する、標準的な HTTP プロキシモジュール。
  • modules/proxy/proxy_util.c: 接続の管理や転送処理の共通関数群。

読者向けメモ

  • **メモリ管理**: r->pool を使用してリクエスト単位のライフサイクルを管理しています。apr_brigade はバックエンドとの通信バッファとして使用されます。
  • **トンネリングの性質**: proxy_wstunnel_request 内のループは、トンネルが確立されるとリクエスト処理全体を占有します。そのため、このハンドラが実行されると他の処理は行われません。
  • **フォールバック**: mod_proxy_http がロードされている場合、設定によりそちらへ処理を委譲可能です。ws_proxy_hooks で mod_proxy_http を成功時の後続として指定している点に注意してください。
  • **制限事項**: 本ソースは WebSocket のトンネリングに特化しており、HTTP/2 などの高度なプロトコル変換ロジックは含まれていません。
