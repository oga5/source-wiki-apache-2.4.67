modules/core

システム内での役割

modules/core は、Apache HTTP Server の実行基盤を拡張・制御するためのコア機能群を提供します。このモジュールはディレクトリ境界に基づいて定義されており、サーバーの動的拡張、設定の抽象化、およびバックグラウンドタスク管理という、サーバー運用の根幹を支える機能を集約しています。

責務

  • **動的モジュール管理**: サーバー再起動を伴わない共有オブジェクト（DSO）のロードと統合。
  • **設定の再利用**: マクロ機能による設定ファイルの冗長性排除とテンプレート化。
  • **バックグラウンド処理**: プロセスライフサイクルに連動した定期的なタスク実行と監視（ウォッチドッグ）の提供。

このモジュールのファイル

  • modules/core/mod_macro.c: 設定ファイル内のマクロ定義・展開機能。
  • modules/core/mod_so.c / mod_so.h: 動的共有オブジェクト（DSO）のロード機能。
  • modules/core/mod_watchdog.c / mod_watchdog.h: バックグラウンドタスク監視フレームワーク。

主要なデータ構造

  • **設定・管理**: so_server_conf (DSO管理用), wd_server_conf_t (ウォッチドッグ設定)
  • **マクロ**: ap_macro_t (マクロ定義), array_contents_t (展開後の設定ストリーム)
  • **ウォッチドッグ**: ap_watchdog_t (監視インスタンス), watchdog_list_t (コールバックリスト)
  • **共通**: apr_pool_t (メモリ管理)

主なフロー

1. **DSO ロード**: LoadModule ディレクティブにより mod_so が共有ライブラリをロードし、Apache のモジュールリストへ動的に統合する。 2. **マクロ展開**: 設定解析フェーズで mod_macro が Use ディレクティブを検出し、引数を置換した内容をパーサーへ注入する。 3. **タスク監視**: mod_watchdog が起動時に専用スレッドを生成し、登録されたコールバックを STARTING / RUNNING / STOPPING の状態遷移に合わせて定期実行する。

外部依存

  • **Apache Core API**: httpd.h, http_config.h, http_log.h
  • **APR (Apache Portable Runtime)**: apr.h, apr_dso.h, apr_hash.h, apr_strings.h, apr_optional.h
  • **その他**: ap_provider.h (プロバイダ機能), util_mutex.h (プロセス間排他制御)

内部構成

本モジュールは、独立した機能を持つ3つのサブコンポーネント（mod_so, mod_macro, mod_watchdog）で構成されています。これらは互いに直接依存するのではなく、Apache のフック機構やオプション関数を通じて疎結合に連携します。

推奨読書順

1. modules/core/mod_watchdog.h（非同期処理の基盤理解） 2. modules/core/mod_so.h（動的拡張のインターフェース理解） 3. modules/core/mod_macro.c（設定解析の拡張ロジック） 4. modules/core/mod_watchdog.c（スレッド管理とコールバック実装） 5. modules/core/mod_so.c（DSO ロードとシンボル解決の実装）

未解決の問い

  • 各コンポーネントが将来的に独立したモジュールディレクトリへ分離されるべきか、あるいは現在の core への集約が最適か。
  • mod_macro の再帰的解析や複雑なネスト構造への対応における制限事項の精緻化。
