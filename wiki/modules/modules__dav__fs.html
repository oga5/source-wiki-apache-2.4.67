modules/dav/fs

システム内での役割

modules/dav/fs は、Apache HTTP Server の mod_dav モジュールに対し、ローカルファイルシステムを WebDAV リポジトリとして公開するためのバックエンド実装です。このモジュールはディレクトリ境界に基づいて構成されており、ファイルシステム上の実体と WebDAV リソースの橋渡しを行います。

責務

  • **リポジトリ提供**: ファイルシステム上のパスを WebDAV リソースとして抽象化し、読み書きやディレクトリ走査を可能にする。
  • **メタデータ管理**: ファイルやディレクトリに付随する WebDAV プロパティ（Dead Properties）を DBM 形式で永続化する。
  • **ロック管理**: DAVLockDB を介したリソースのロック状態（直接ロックおよび間接ロック）の管理と永続化。
  • **モジュール統合**: mod_dav が要求するフックインターフェースを実装し、Apache の設定と連携させる。

このモジュールのファイル

  • modules/dav/fs/dbm.c: プロパティ管理のための DBM データベース操作層。
  • modules/dav/fs/lock.c: ロック情報の永続化と解決ロジック。
  • modules/dav/fs/mod_dav_fs.c: モジュールのエントリポイントおよび設定管理。
  • modules/dav/fs/repos.c: リポジトリ操作のメイン実装。
  • modules/dav/fs/repos.h: 公開 API およびインターフェース定義。

主要なデータ構造

  • **リポジトリ/リソース**: dav_resource_private, dav_fs_walker_context
  • **ロック関連**: dav_lockdb_combined, dav_lock_discovery, dav_lock_indirect, dav_locktoken
  • **DBM/プロパティ**: dav_db, dav_propdb_metadata

主なフロー

1. **初期化**: mod_dav_fs.c が DAVLockDB 設定を読み込み、dav_fs_register を通じて mod_dav にプロバイダーを登録する。 2. **リソース解決**: repos.c がリクエストパスを物理パスへ変換し、dav_resource を構築する。 3. **プロパティ/ロック操作**: dbm.c および lock.c が .DAV ディレクトリ内の DBM ファイルを操作し、メタデータやロック状態の読み書きを行う。 4. **ディレクトリ走査**: repos.c が再帰的にファイルシステムを探索し、各リソースに対してコールバックを実行する。

外部依存

  • **APR**: apr_dbm.h, apr_file_io.h, apr_strings.h, apr_uuid.h 等
  • **Apache HTTPD**: httpd.h, http_config.h, http_log.h
  • **DAV**: mod_dav.h

内部構成

本モジュールは mod_dav のプロバイダーとして機能する単一のサブシステムであり、内部にサブモジュールは持ちません。

推奨読書順

1. modules/dav/fs/repos.h (インターフェースの理解) 2. modules/dav/fs/repos.c (リポジトリ操作の核心) 3. modules/dav/fs/lock.c (ロック管理の詳細) 4. modules/dav/fs/dbm.c (メタデータ永続化の仕組み) 5. modules/dav/fs/mod_dav_fs.c (モジュール構成と設定)

未解決の問い

  • COPY/MOVE 操作におけるファイルとプロパティの不整合リスクに対する堅牢性の検証。
