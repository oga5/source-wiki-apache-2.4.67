modules/loggers/mod_log_debug.c

概要

mod_log_debug.c は、Apache HTTP Server のリクエスト処理の各フェーズにおいて、設定された条件に基づきデバッグメッセージをエラーログへ出力するためのモジュールです。LogMessage ディレクティブを通じて、特定のフックポイントで任意の式（ap_expr）を評価し、その結果をログ出力する機能を提供します。

モジュール内での役割

modules/loggers サブシステムにおいて、本モジュールは「診断・デバッグ支援」の役割を担います。他のロギングモジュール（mod_log_config 等）がアクセスログの記録に特化しているのに対し、本モジュールはリクエスト処理の内部状態を開発者が追跡するための柔軟なフックを提供します。

責務

  • **設定管理**: LogMessage ディレクティブによる設定のパースと、ディレクトリ単位のコンフィグ（log_debug_dirconf）の保持・マージ。
  • **式評価**: ap_expr を利用した条件判定およびメッセージ文字列の動的生成。
  • **フック実行**: リクエスト処理の主要な 13 個のフェーズ（quick_handler から log_transaction まで）に割り込み、設定されたメッセージをログ出力する。

重要なシンボル

設定構造体

  • log_debug_dirconf: ディレクトリ単位の設定を保持。apr_array_header_t を用いて複数の msg_entry を格納する。
  • msg_entry: 個別の LogMessage 設定。評価対象の式（msg_expr）、条件式（condition）、および対象フック名を保持。

設定・ライフサイクル

  • log_debug_create_dconf: コンフィグ領域の初期化。
  • log_debug_merge_dconf: 親ディレクトリと子ディレクトリの設定をマージ（apr_array_append を使用）。
  • cmd_log_message: LogMessage ディレクティブのハンドラ。式をパースし、引数を検証して msg_entry を生成。

実行エンジン

  • do_debug_log: 全てのフックから呼び出される共通のロギング関数。条件判定とメッセージ生成、ログ出力を行う。

制御フローとデータフロー

1. **初期化**: register_hooks により、Apache の各フェーズ（translate_name, fixups 等）に log_debug_* 関数が APR_HOOK_FIRST で登録される。 2. **リクエスト処理**: 各フックが呼ばれると、do_debug_log が現在のリクエストコンテキスト（request_rec）とフック名を引数に実行される。 3. **評価**: do_debug_log は dconf からエントリを走査し、ap_expr_exec で条件を判定。条件が真であれば ap_expr_str_exec でメッセージを生成する。 4. **出力**: 生成されたメッセージは ap_log_rerror を介してエラーログに出力される。

直接依存

  • httpd.h, http_config.h, http_log.h: Apache コア機能およびロギング API。
  • ap_expr.h: 式評価エンジン（ap_expr_parse_cmd, ap_expr_exec 等）。
  • apr_strings.h: 文字列操作およびメモリ管理。
  • http_protocol.h
  • http_request.h

関連コード

  • modules/loggers/mod_log_config.c: ログフォーマットの定義と出力の標準的な実装。
  • server/util_expr.c: ap_expr の実装詳細。本モジュールの式評価の基盤。

読者向けメモ

  • **メモリ管理**: 設定データは cmd->pool（設定用プール）に割り当てられ、リクエスト処理中のメモリは r->pool を使用します。apr_array_append は新しい配列を生成するため、マージ時のメモリ消費に注意してください。
  • **パフォーマンス**: 各フックで ap_expr を評価するため、複雑な式を多用するとリクエスト処理のレイテンシに影響を与える可能性があります。
  • **制約**: LogMessage は RSRC_CONF および ACCESS_CONF でのみ有効です。
  • **省略された詳細**: ap_expr の構文解析の詳細や、Apache のフックシステムの内部実装については本ファイルでは省略されています。これらは server/ 以下のコアコードに依存しています。
