modules/loggers/mod_log_config.c

概要

mod_log_config.c は、Apache HTTP Server における柔軟なログ記録機能（TransferLog, LogFormat, CustomLog）を実装するコアモジュールです。リクエスト単位の情報を収集し、設定されたフォーマットに従ってファイルやパイプへ書き出す責務を持ちます。

モジュール内での役割

modules/loggers サブシステムにおけるメインエンジンです。他のログモジュール（mod_logio 等）が提供する拡張機能と連携し、リクエスト処理の最終段階（log_transaction フック）で実行されます。

責務

  • **設定管理**: サーバーおよびバーチャルホストごとのログ設定（フォーマット、出力先、条件）の保持とマージ。
  • **フォーマット解析**: LogFormat 文字列を解析し、実行時に評価可能な log_format_item 配列へ変換。
  • **リクエスト処理**: リクエストの各フェーズで収集された情報をフォーマットに従って文字列化し、出力先へディスパッチ。
  • **出力制御**: 標準的なファイル書き込みおよびバッファリングされた書き込みの制御。

重要なシンボル

設定構造体

  • multi_log_state: サーバー単位のログ設定（フォーマット定義、ログリスト）。
  • config_log_state: 個別のログファイル設定（ファイルパス、フォーマット、条件式）。
  • log_format_item: 解析済みのログ項目（ハンドラ関数、引数、条件）。

ログ処理・ディスパッチ

  • log_writer: ログ書き込みを行う関数ポインタ（デフォルトまたはバッファリング版）。
  • ap_default_log_writer: 非バッファリングの直接書き込み実装。
  • ap_buffered_log_writer: バッファリングされた書き込み実装。

登録・フック

  • log_config_module: モジュール定義構造体。
  • register_hooks: Apache フック（pre_config, check_config, child_init, open_logs, log_transaction）の登録。

制御フローとデータフロー

1. **初期化**: log_pre_config でログハンドラ（%h, %u 等）を log_hash に登録。 2. **設定解析**: add_custom_log 等により config_log_state が構築され、parse_log_string でフォーマットが中間形式に変換される。 3. **ログオープン**: open_multi_logs が open_logs フックで呼ばれ、ファイルディスクリプタを確保。 4. **トランザクション**: multi_log_transaction がリクエスト終了時に呼ばれる。

  • config_log_transaction が各ログ設定を評価。
  • process_item が各フォーマット項目を順次実行し、文字列を生成。
  • log_writer を介して出力。

読者向けメモ

  • **メモリ管理**: ほとんどの動的メモリは request_rec のプール（r->pool）から割り当てられ、リクエスト終了時に自動解放されます。
  • **バッファリング**: BufferedLogs が有効な場合、buffered_log 構造体が使用されます。マルチスレッド環境では apr_thread_mutex_t による排他制御が行われます。
  • **パフォーマンス**: log_request_time 等の関数は、頻繁に呼ばれるため、時刻のキャッシュやスタック利用の最適化が行われています。
  • **制約**: log_writer は open_logs フック実行前に設定する必要があります。
  • **省略箇所**: 本ドキュメントでは、各ログハンドラ関数（log_remote_host 等）の詳細な実装ロジックは省略しています。これらは log_hash に登録されたコールバックとして独立しており、必要に応じて個別に参照可能です。
