modules/loggers/mod_log_config.h

概要

mod_log_config.h は、Apache HTTP Server のログ記録機能を拡張するためのパブリックインターフェースを定義するヘッダーファイルです。主に、外部モジュールがカスタムログフォーマットや独自のログ出力先（ライター）を登録するための API を提供します。

モジュール内での役割

mod_log_config モジュールの中核となるヘッダーであり、他のモジュールがログ記録プロセスに介入するための「フック」を提供します。このファイルで定義された関数ポインタやオプション関数を通じて、Apache の標準的なログ記録メカニズムを外部から拡張可能にしています。

責務

  • ログフォーマットの解析用ハンドラ（ap_log_handler_fn_t）の型定義。
  • ログ出力先を初期化・書き込みするためのコールバックインターフェースの提供。
  • 外部モジュールがログ機能を拡張するためのオプション関数（APR_DECLARE_OPTIONAL_FN）の公開。

重要なシンボル

  • ap_log_handler_fn_t: ログフォーマット文字列内のタグ（例: %h）に対応する値を抽出する関数の型。
  • ap_log_writer: 実際にログをファイルやネットワークへ書き出すための関数型。
  • ap_register_log_handler: 外部モジュールが独自のログフォーマットタグを登録するための関数。
  • ap_log_set_writer: ログの書き込み処理をカスタム実装に置き換えるための関数。

制御フローとデータフロー

このヘッダー自体はデータ構造とインターフェース定義のみを提供します。 1. **登録フェーズ**: 外部モジュールが ap_register_log_handler を呼び出し、特定のタグに対する処理関数を登録します。 2. **初期化フェーズ**: ap_log_set_writer_init を通じて、ログ出力先の初期化ロジックが設定されます。 3. **実行フェーズ**: リクエスト処理の終了時、mod_log_config は登録されたハンドラを順次呼び出し、ログ行を構築して ap_log_writer を通じて出力します。

直接依存

  • apr_optional.h: オプション関数（APR_DECLARE_OPTIONAL_FN）の利用に必須。
  • httpd.h: request_rec や server_rec などの Apache コア構造体定義。
  • scoreboard.h: サーバーの稼働状況やプロセス情報へのアクセス。

関連コード

  • modules/loggers/mod_log_config.c: 本ヘッダーで定義されたインターフェースの実装本体。
  • modules/loggers/mod_logio.c: 本ヘッダーを利用して、入出力バイト数をログ記録する拡張モジュールの例。

読者向けメモ

  • **タイミングの重要性**: ap_log_set_writer_init や ap_log_set_writer を使用する場合、Apache の open_logs フックが実行される前に設定を完了させる必要があります。
  • **拡張性**: このヘッダーは、標準のログフォーマットでは不足している情報をログに含めたい場合（例: 特定のモジュールが保持する内部状態）の主要なエントリーポイントとなります。
