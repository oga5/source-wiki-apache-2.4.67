modules/test

システム内での役割

modules/test は、Apache HTTP Server の拡張メカニズム（オプションフック、オプション関数）および非同期 I/O 制御の検証・デモンストレーションを目的とした実験的なモジュール群です。本モジュールはディレクトリ境界に基づいて定義されています。

責務

  • **モジュール間連携の検証**: オプションフックおよびオプション関数を用いた、疎結合なモジュール間通信パターンの提供。
  • **非同期処理の実験**: SUSPENDED ステータスとタイマーコールバックを用いた、帯域制限等の非同期リクエスト処理の実証。
  • **開発者向けサンプル**: Apache の拡張 API を利用する際の最小構成例の提示。

このモジュールのファイル

  • mod_dialup.c: 低速回線シミュレーション（非同期 I/O）。
  • mod_optional_fn_export.c / .h: オプション関数エクスポートの実装と定義。
  • mod_optional_fn_import.c: オプション関数インポートの実装。
  • mod_optional_hook_export.c / .h: オプションフックエクスポートの実装と定義。
  • mod_optional_hook_import.c: オプションフックインポートの実装。

主要なデータ構造

  • dialup_dcfg_t: ディレクトリごとの設定（帯域制限値など）。
  • dialup_baton_t: 非同期処理のコンテキスト（バケットブリゲード、リクエスト情報）。
  • modem_speed_t: モデム規格とバイトレートの対応定義。

主なフロー

1. **オプション連携**: エクスポート側が AP_DECLARE_HOOK や APR_DECLARE_OPTIONAL_FN でインターフェースを公開し、インポート側が実行時に動的にバインドする。 2. **非同期ハンドリング**: mod_dialup がリクエストを SUSPENDED 状態へ移行させ、ap_mpm_register_timed_callback を用いて断続的にデータを送信する。

外部依存

  • httpd.h, http_config.h, http_log.h, http_protocol.h, http_core.h: Apache コア API。
  • apr_optional.h: オプション関数機能の基盤。
  • util_filter.h: バケットブリゲード操作。
  • ap_mpm.h: 非同期タイマー制御。

内部構成

本ディレクトリは独立したテスト用モジュールの集合体であり、階層的な子モジュールは持ちません。

推奨読書順

1. mod_optional_hook_export.h (フック定義の基礎) 2. mod_optional_fn_export.h (関数定義の基礎) 3. mod_optional_hook_export.c / mod_optional_hook_import.c (フックの連携) 4. mod_optional_fn_export.c / mod_optional_fn_import.c (関数の連携) 5. mod_dialup.c (非同期 I/O の応用)

未解決の問い

  • 実験的なモジュールであるため、本番環境への適用可能性や、将来的な MPM 変更に伴う SUSPENDED 状態の挙動の安定性について精緻化が必要。
