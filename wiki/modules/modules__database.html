modules/database

システム内での役割

modules/database は、Apache HTTP Server におけるデータベース接続管理の基盤を提供するモジュール群です。主に mod_dbd を含み、APR (Apache Portable Runtime) の apr_dbd フレームワークをラップすることで、他のモジュールがデータベースへ安全かつ効率的にアクセスするための共通インターフェースを提供します。

責務

  • **接続プーリング**: スレッド環境下での apr_reslist を活用した接続の再利用と管理。
  • **ライフサイクル管理**: リクエストまたはコネクションのライフサイクルに紐付いたデータベース接続の自動取得・解放。
  • **設定管理**: DBDriver や DBDParams 等のディレクティブを通じたデータベース接続設定の解析と保持。
  • **SQL 準備**: サーバー起動時の SQL ステートメントの事前準備（プリペア）とキャッシュ。

このモジュールのファイル

  • modules/database/mod_dbd.c: 接続管理、プーリング、設定解析の実装。
  • modules/database/mod_dbd.h: 他モジュール向けのパブリック API 定義。

主要なデータ構造

  • dbd_cfg_t: サーバー単位のデータベース設定（ドライバ名、接続パラメータ等）。
  • ap_dbd_t: データベース接続ハンドル。ドライバ、接続状態、プリペア済みステートメントを保持。
  • dbd_group_t: 接続プール（apr_reslist）または単一接続を管理するグループ構造体。
  • svr_cfg: サーバーごとの設定コンテナ。

主なフロー

1. **設定フェーズ**: dbd_cmds がディレクティブを読み込み、dbd_post_config で接続グループを初期化。 2. **接続取得**: ap_dbd_acquire 等が呼ばれると、リクエストのプールから既存の接続を再利用するか、新規に接続を確立（dbd_construct）する。 3. **SQL 実行**: 事前準備されたステートメントを用いてクエリを実行。 4. **解放**: リクエスト終了時に dbd_release が自動的に呼び出され、接続がプールへ返却される。

外部依存

  • **APR**: apr_dbd.h (DB抽象化), apr_reslist.h (プーリング), apr_hash.h, apr_tables.h 等。
  • **Apache Core**: http_config.h, http_protocol.h, http_log.h 等。

内部構成

本モジュールは単一の機能単位（mod_dbd）で構成されており、サブモジュールは持ちません。

推奨読書順

1. modules/database/mod_dbd.h: 公開 API と提供機能の概要を把握する。 2. modules/database/mod_dbd.c: 接続プーリングやライフサイクル管理の実装詳細を確認する。

未解決の問い

  • 現在の設計は apr_dbd に強く依存しているが、将来的なドライバ拡張や非同期 I/O 対応における制約の精査。
  • LLM ベースのモジュール合成を追加した後で、責務とフローをさらに精緻化する。
