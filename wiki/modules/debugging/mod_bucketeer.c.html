modules/debugging/mod_bucketeer.c

概要

mod_bucketeer は、Apache HTTP Server の出力フィルタとして動作するデバッグ用モジュールです。データストリーム内の特定の制御文字を検出し、バケット（Bucket）の分割やフラッシュ、送信を強制的に実行することで、バケット処理の挙動をテスト・検証するために使用されます。

モジュール内での役割

modules/debugging ディレクトリに属し、フィルタチェーンの挙動を意図的に操作する「インジェクション・ツール」としての役割を担います。主に開発者がバケットの断片化やストリーム制御をシミュレートする際に利用します。

責務

  • **バケットの分割**: 設定されたデリミタ（^B, ^P, ^F）をデータ内で検出し、バケットを再構成する。
  • **ストリーム制御**: 制御文字に応じて、バケットのフラッシュや下流フィルタへのブリゲード送信をトリガーする。
  • **コンテンツフィルタリング**: text/ で始まるコンテンツタイプのみを対象とし、それ以外は透過的に処理する。

重要なシンボル

  • bucketeer_out_filter: モジュールの核となる出力フィルタ関数。バケットの走査と分割ロジックを実装。
  • bucketeer_filter_config_t: デリミタ設定を保持する構造体。
  • bucketeer_ctx_t: フィルタ実行中の状態（ブリゲード）を保持するコンテキスト。

制御フローとデータフロー

1. **初期化**: create_bucketeer_server_config でデフォルトのデリミタ（0x02, 0x10, 0x06）を定義。 2. **フィルタ起動**: bucketeer_out_filter が呼び出されると、対象が text/ か判定し、非対象ならフィルタを解除してパススルーする。 3. **バケット走査**: ブリゲード内の各バケットを読み込み、データ内にデリミタが存在するか確認する。 4. **分割と処理**:

  • bucketdelimiter: バケットを分割し、新しいバケットを作成。
  • flushdelimiter: フラッシュバケットを挿入。
  • passdelimiter: 現在のブリゲードを下流へ送信（ap_pass_brigade）。

    • 5. **終了**: EOS（End of Stream）バケットを検知すると、残りのブリゲードを送信して終了する。

直接依存

  • httpd.h, http_config.h, util_filter.h: Apache フィルタフレームワークの基本。
  • apr_buckets.h: バケットおよびブリゲード操作の API。
  • http_protocol.h, http_request.h: リクエストおよびプロトコル制御。
  • apr_general.h
  • apr_strings.h
  • http_log.h

関連コード

  • modules/debugging/mod_dumpio.c: 同ディレクトリに存在し、I/O データのダンプを行うデバッグモジュール。併せて参照することで、フィルタチェーンの可視化と操作の両面を理解できます。

読者向けメモ

  • **副作用**: 本モジュールは Content-Length ヘッダーを削除します。バケットの分割・再構成によりデータ長が変化するため、この挙動は必須です。
  • **注意点**: bucketeer_out_filter 内の XXX コメントにある通り、現在の実装では残りのデータ処理に改善の余地があります。実運用環境ではなく、あくまでデバッグ環境での使用を前提としてください。
  • **デリミタ**: デフォルトのデリミタは ASCII 制御文字です。バイナリデータを扱う場合は、意図しない分割が発生する可能性があるため注意してください。
