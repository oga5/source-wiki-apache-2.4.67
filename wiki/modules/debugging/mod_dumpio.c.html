modules/debugging/mod_dumpio.c

概要

mod_dumpio.c は、Apache HTTP Server の入出力フィルタチェーンを流れるデータを傍受し、エラーログ（APLOG_TRACE7 レベル）へダンプするデバッグ用モジュールです。SSL 復号後および暗号化前の生のデータストリームを可視化するために設計されています。

モジュール内での役割

modules/debugging ディレクトリにおいて、ネットワーク I/O の低レイヤーにおけるデータフローを監視する「スニファー」の役割を果たします。フィルタチェーンの AP_FTYPE_CONNECTION + 3 に位置し、SSL フィルタよりも内側で動作します。

責務

  • **フィルタ登録**: 入力用 (DUMPIO_IN) および出力用 (DUMPIO_OUT) フィルタを接続フェーズで挿入。
  • **データダンプ**: バケット内のデータを読み取り、ログ出力可能な形式に変換して記録。
  • **設定管理**: DumpIOInput および DumpIOOutput ディレクティブによる機能の有効/無効化。

重要なシンボル

設定管理

  • dumpio_conf_t: 入出力の有効状態を保持するサーバー設定構造体。
  • dumpio_create_sconfig: 設定構造体の初期化（メモリ確保）。
  • dumpio_cmds: DumpIOInput / DumpIOOutput ディレクティブの定義。

フィルタ処理

  • dumpio_input_filter: 入力ストリームを傍受し、ap_get_brigade を呼び出した後にデータをダンプ。
  • dumpio_output_filter: 出力ストリームを傍受し、データをダンプした後に ap_pass_brigade で後続へ渡す。
  • dumpit: 共通のダンプ処理関数。バケットの内容を読み取り、ログ出力を行う。

フックと登録

  • dumpio_register_hooks: フィルタの登録と pre_connection フックの設定。
  • dumpio_pre_conn: 接続開始時に設定に基づきフィルタをチェーンに追加。

制御フローとデータフロー

1. **初期化**: dumpio_pre_conn が接続ごとに呼び出され、設定が有効であればフィルタをチェーンに挿入します。 2. **入力フロー**: dumpio_input_filter が ap_get_brigade を実行し、取得したバケット群を dumpit に渡してログ出力します。 3. **出力フロー**: dumpio_output_filter がバケット群を走査し、dumpit でログ出力した後、ap_pass_brigade で後続のフィルタへデータを渡します。 4. **ダンプ処理**: dumpit は apr_bucket_read を使用してデータを取得します。EBCDIC 環境では文字コード変換を行い、それ以外ではバイナリデータをそのままログ出力します。

直接依存

  • httpd.h, http_connection.h, http_config.h: Apache コアおよび接続管理。
  • http_core.h, http_log.h: ログ出力およびコア機能。
  • apr_strings.h: 文字列操作。

関連コード

  • util_filter.h: フィルタチェーンの仕組みを理解するために参照。
  • apr_buckets.h: バケットおよびブリゲードの操作（apr_bucket_read 等）の理解に必須。

読者向けメモ

  • **パフォーマンス**: 非常に大量のログを出力するため、本番環境での有効化は推奨されません。APLOG_TRACE7 レベルで動作するため、ログレベル設定に注意してください。
  • **メモリ**: dumpit 内で apr_bucket_read を使用しており、バケットの読み取り時にメモリが確保される場合があります。
  • **制限事項**: ソース内のコメントにある通り、バイナリデータ内の \0（ヌル文字）の扱いには課題があり、ログ出力時にエスケープ処理が不完全な場合があります。
  • **省略箇所**: dumpit 内の EBCDIC 変換ロジックは、特定のプラットフォームでのみ有効なため、一般的な環境では無視されます。また、フィルタの挿入位置（AP_FTYPE_CONNECTION + 3）は、SSL フィルタとの順序関係を意図的に制御しています。
