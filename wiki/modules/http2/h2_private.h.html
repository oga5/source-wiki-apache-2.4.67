modules/http2/h2_private.h

概要

h2_private.h は、Apache HTTP Server の HTTP/2 モジュール (mod_http2) における内部共有ヘッダーファイルです。モジュール全体で共通して利用される外部依存関係の取り込みと、モジュール定義の参照を目的としています。

モジュール内での役割

本ファイルは、mod_http2 の各ソースファイルが共通して必要とする「モジュールとしての識別子」や「ロギング設定」を定義する基盤です。他のヘッダーファイルが機能単位（ストリーム、セッション等）で分割されているのに対し、本ファイルはモジュール全体にまたがるグローバルな設定を束ねる役割を担います。

責務

  • http2_module 構造体への外部参照を提供し、モジュール設定やフック登録を可能にする。
  • APLOG_USE_MODULE(http2) を宣言し、モジュール固有のログ出力設定を有効化する。
  • 必須となる外部ライブラリ（nghttp2）および Apache Portable Runtime (APR) の時間関連ヘッダーをインクルードし、依存関係を統一する。

重要なシンボル

  • http2_module: Apache モジュールシステムが認識する mod_http2 のメイン構造体。
  • APLOG_USE_MODULE(http2): このモジュール内で発生するログメッセージに http2 というタグを付与するためのマクロ。

制御フローとデータフロー

本ファイルは実行コードを含まないヘッダーファイルであるため、直接的な制御フローは存在しません。各ソースファイルが本ファイルをインクルードすることで、コンパイル時にモジュール定義とロギング機能が静的に結合されます。

直接依存

  • apr_time.h: APR の時間操作機能。
  • nghttp2/nghttp2.h: HTTP/2 プロトコル処理の核となる外部ライブラリ。

関連コード

  • mod_http2.c: http2_module 構造体が実際に定義されているメインのエントリポイント。
  • h2.h: HTTP/2 モジュール全体で共有されるデータ構造や定数が定義された、より広範なヘッダーファイル。

読者向けメモ

  • 本ファイルは非常に軽量であり、主にビルド時の依存関係解決のために存在します。
  • 新しくモジュール内のソースファイルを追加する際は、ログ出力機能を利用するために本ファイルをインクルードすることが推奨されます。
  • nghttp2 ライブラリへの依存が明示されているため、ビルド環境には libnghttp2 の開発ヘッダーが必須となります。
