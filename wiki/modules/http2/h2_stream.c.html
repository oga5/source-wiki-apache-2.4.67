modules/http2/h2_stream.c

概要

HTTP/2 ストリームのライフサイクル管理、状態遷移、およびデータ入出力の仲介を行うコアモジュールです。HTTP/2 のストリーム状態（IDLE, OPEN, CLOSED 等）を厳密に管理し、nghttp2 ライブラリのコールバックと Apache HTTP Server のバケット処理を橋渡しします。

モジュール内での役割

h2_session が管理する個々のストリームを表現します。h2_mplx（多重化）と連携し、クライアントからのリクエスト受信と、バックエンド（c2）からのレスポンス送信をストリーム単位で制御します。

責務

  • **状態管理**: HTTP/2 仕様に基づくストリーム状態遷移の検証と実行。
  • **データ仲介**: h2_bucket_beam を介した入出力データのバッファリングとフロー制御。
  • **プロトコル変換**: nghttp2 フレームと Apache の apr_bucket_brigade 間の変換。
  • **リクエスト/レスポンス構築**: ヘッダーの検証、トレーラーの処理、およびプッシュリソースの収集。

重要なシンボル

状態管理

  • h2_stream_state_t: ストリームの状態定義（IDLE, OPEN, CLOSED 等）。
  • h2_stream_dispatch(): イベントに基づき状態遷移をトリガーするメイン関数。
  • transit(): 状態遷移の実行と、遷移に伴うクリーンアップ処理。

データ入出力

  • h2_stream_recv_DATA(): クライアントから受信した DATA フレームの処理。
  • h2_stream_read_to(): c1（クライアント接続）へ送信するデータをバッファから読み出す。
  • stream_data_cb(): nghttp2 がデータを要求した際に呼び出されるコールバック。

ライフサイクル

  • h2_stream_create(): ストリームの初期化とメモリプールの割り当て。
  • h2_stream_destroy(): ストリームの破棄とリソース解放。
  • h2_stream_rst(): ストリームの強制終了（RST_STREAM）。

制御フローとデータフロー

1. **入力フロー**: h2_stream_recv_frame が nghttp2 からのフレームを受け取り、transit で状態を更新。DATA フレームは input_append_data を経由して h2_bucket_beam に送られる。 2. **出力フロー**: h2_stream_on_output_change がトリガーとなり、stream_do_responses がレスポンスヘッダーを処理。その後、stream_data_cb が nghttp2 の要求に応じてバッファからデータを供給する。 3. **エラーフロー**: h2_stream_rst が呼ばれると、関連する c2 接続が中断され、状態が CLOSED または CLEANUP へ遷移する。

読者向けメモ

  • **メモリ管理**: ストリームは専用の apr_pool_t を持ち、破棄時に自動的にリソースが解放されます。
  • **不変条件**: AP_DEBUG が有効な場合、magic フィールドによりストリームの生存期間と整合性が検証されます。
  • **制約**: stream_data_cb は nghttp2 のコールバックであるため、非同期的なブロックを避ける設計になっています。
  • **省略**: 本ファイルは状態遷移テーブルの定義が非常に長いため、個別の遷移ロジックの詳細は trans_on_... 配列の構造を参照してください。
