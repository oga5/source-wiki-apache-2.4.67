modules/http2/h2_proxy_session.c

概要

h2_proxy_session.c は、Apache HTTP Server がバックエンドサーバーに対して HTTP/2 プロキシとして振る舞う際のセッション管理を担います。nghttp2 ライブラリをラップし、バックエンドとの接続状態、ストリームの多重化、およびプロキシ特有のヘッダー変換やフロー制御を管理する中心的なファイルです。

モジュール内での役割

mod_proxy_http2 の中核であり、h2_proxy_session.h で定義されたセッション構造体と、nghttp2 のコールバックを仲介します。フロントエンド（クライアント）からのリクエストをバックエンドへ転送し、バックエンドからのレスポンスをフロントエンドへ戻すためのブリッジとして機能します。

責務

  • **セッションライフサイクル管理**: 接続の初期化、PING による生存確認、GOAWAY による正常終了、エラー時の強制終了。
  • **ストリーム管理**: h2_proxy_stream を用いたリクエスト/レスポンスの追跡、ヘッダーの変換（Via ヘッダーや Location 等の書き換え）。
  • **I/O 制御**: nghttp2 と Apache のバケットブリゲード間のデータ転送、非同期的なストリームのサスペンド/レジューム制御。
  • **イベント駆動状態遷移**: h2_proxys_state に基づく有限オートマトンによるセッション状態の遷移管理。

重要なシンボル

セッション管理

  • h2_proxy_session_setup: セッションの初期化と nghttp2 コールバックの設定。
  • h2_proxy_session_process: メインループ。I/O の読み書きと状態遷移を駆動。
  • dispatch_event: セッションイベント（H2_PROXYS_EV_*）に基づき状態を遷移させる。

ストリーム操作

  • h2_proxy_stream: ストリームごとの状態、入出力ブリゲード、リクエスト情報を保持する内部構造体。
  • submit_stream: リクエストをバックエンドへ送信する。
  • stream_response_data: バックエンドからの DATA フレームを受信し、フロントエンドへ転送するコールバック。
  • stream_request_data: フロントエンドからのデータを読み取り、バックエンドへ送信するコールバック。

PING 制御

  • enter_ping_state: 接続の健全性確認（PING）状態への遷移とタイムアウト設定。
  • ping_ev_frame_received: PING フレーム受信時の状態更新。

制御フローとデータフロー

1. **初期化**: h2_proxy_session_setup で nghttp2 セッションを作成し、コールバックを登録。 2. **処理ループ**: h2_proxy_session_process が呼ばれるたびに、send_loop で送信待ちのフレームを処理し、h2_proxy_session_read でバックエンドからの入力を読み込む。 3. **データ転送**:

  • リクエスト時: stream_request_data が ap_get_brigade を通じてフロントエンドからデータを読み取り、nghttp2 に渡す。
  • レスポンス時: stream_response_data が nghttp2 からのデータを ap_pass_brigade でフロントエンドへ流す。

    • 4. **状態遷移**: dispatch_event がイベントを受け取り、transit 関数で session->state を更新する。

直接依存

  • nghttp2/nghttp2.h: HTTP/2 プロトコル処理。
  • mod_proxy.h: プロキシ機能の基盤。
  • h2_proxy_util.h: ヘッダー変換やユーティリティ関数。
  • httpd.h, http_protocol.h: Apache のリクエスト処理基盤。

関連コード

  • h2_proxy_session.h: セッション構造体定義。
  • h2_proxy_util.c: ヘッダー変換ロジックやブリゲード操作の補助。
  • mod_proxy_http2.c: このセッションを呼び出すプロキシモジュールのエントリポイント。

読者向けメモ

  • **メモリ管理**: セッションおよびストリームは apr_pool_t で管理されており、セッション終了時に proxy_session_pre_close が呼ばれ、リソースが解放されます。
  • **非同期性**: check_suspended 関数は、バックエンドからの応答待ち（100-continue や PING）でサスペンドされたストリームを管理します。
  • **注意点**: h2_proxy_session_process は非ブロッキングで設計されていますが、WAIT 状態ではタイムアウト付きのブロッキング読み込みが発生する場合があります。
  • **省略箇所**: win_update_ctx 構造体以降のコードは、ウィンドウ更新処理の一部であり、本ファイル内では断片的な定義に留まっています。
