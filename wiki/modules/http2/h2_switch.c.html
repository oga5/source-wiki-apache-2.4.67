modules/http2/h2_switch.c

概要

HTTP/1.1 から HTTP/2 へのプロトコル切り替え（アップグレードおよび ALPN ネゴシエーション）を管理する Apache HTTP Server モジュールのエントリーポイントです。

モジュール内での役割

mod_http2 の中核となる接続制御層です。Apache の protocol フックを実装し、HTTP/1.1 接続を HTTP/2 セッションへ昇格させるための判断と初期化を担当します。

責務

  • **プロトコル提案**: ALPN や Upgrade ヘッダーに基づき、HTTP/2 が利用可能かを判定し、サーバーがサポートするプロトコルを提案する。
  • **プロトコル切り替え**: 選択されたプロトコルに基づき、既存の HTTP/1.1 フィルターを解除し、HTTP/2 接続コンテキストを生成・起動する。
  • **状態管理**: 現在の接続が HTTP/2 であるかを問い合わせるインターフェースの提供。

重要なシンボル

  • h2_protocol_propose: サーバーがサポートするプロトコルを提案するフック関数。
  • h2_protocol_switch: 実際にプロトコルを切り替え、h2_c1（接続制御）を起動するフック関数。
  • h2_switch_register_hooks: Apache のフックシステムに上記関数を登録する。

制御フローとデータフロー

1. **提案フェーズ**: h2_protocol_propose が呼ばれ、SSL/TLS の有無や HTTP2-Settings ヘッダーの存在を確認し、条件を満たせば HTTP/2 を提案リストに追加する。 2. **切り替えフェーズ**: h2_protocol_switch が呼び出されると、接続コンテキスト (h2_conn_ctx) を作成する。 3. **フィルターの再構成**: 既存の HTTP/1.1 フィルター（http_in や HTTP_HEADER 等）を削除し、HTTP/2 用のスタックへ置き換える。 4. **セッション開始**: h2_c1_setup および h2_c1_run を呼び出し、HTTP/2 セッションのメインループへ制御を移す。

読者向けメモ

  • 本ファイルは「プロトコルの切り替え」という境界条件のみを扱います。HTTP/2 の詳細なフレーム処理やストリーム管理は h2_session.c や h2_stream.c に委譲されています。
  • remove_output_filters_below 等の関数は、HTTP/1.1 のフィルターチェーンを破壊して HTTP/2 に差し替えるという、Apache のフィルターシステムにおける非常に強力かつ繊細な操作を行っています。デバッグ時はフィルターの順序と型（ap_filter_type）に注意してください。
