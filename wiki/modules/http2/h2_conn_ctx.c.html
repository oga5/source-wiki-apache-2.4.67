modules/http2/h2_conn_ctx.c

概要

h2_conn_ctx.c は、Apache HTTP Server の接続（conn_rec）に対して、HTTP/2 固有のメタデータや状態を保持するコンテキスト（h2_conn_ctx_t）を管理するファイルです。

モジュール内での役割

HTTP/2 モジュールにおいて、クライアントとの直接通信を行う接続（C1）と、ストリーム処理のために内部生成される接続（C2）の両方で共通の管理基盤を提供します。mod_http2 が接続を識別し、適切なセッションやストリーム情報を紐付けるためのハブとして機能します。

責務

  • **コンテキストのライフサイクル管理**: 接続に対するコンテキストの生成、初期化、および破棄（デタッチ）。
  • **C1/C2 の識別と紐付け**: クライアント接続（C1）と、それによって派生したストリーム処理用接続（C2）のコンテキストを適切に構成する。
  • **リソースの集約**: セッション、マルチプレクサ（h2_mplx）、ストリーム情報、およびデータ転送用のバケットビーム（h2_bucket_beam）の参照を保持する。

重要なシンボル

  • h2_conn_ctx_t: 接続ごとの状態を保持する主要な構造体。
  • h2_conn_ctx_create_for_c1: クライアント接続用のコンテキストを作成し、ソケットのポーリング設定を行う。
  • h2_conn_ctx_init_for_c2: ストリーム処理用接続（C2）のコンテキストを初期化し、親接続（C1）の情報を継承させる。
  • h2_conn_ctx_detach: 接続からコンテキストを切り離し、設定をクリアする。

制御フローとデータフロー

1. **生成**: ctx_create が内部的に呼び出され、ap_set_module_config を通じて conn_rec にコンテキストをアタッチします。 2. **C1 初期化**: h2_conn_ctx_create_for_c1 が呼ばれると、ソケットをノンブロッキングモードに設定し、ポーリング用の apr_pollfd_t を準備します。 3. **C2 初期化**: h2_conn_ctx_init_for_c2 は、親接続（c2->master）から既存のコンテキストを取得し、ストリーム固有の req_pool や h2_mplx を割り当てます。 4. **タイムアウト管理**: h2_conn_ctx_set_timeout が、保持しているバケットビームやパイプに対して一括でタイムアウト値を適用します。

読者向けメモ

  • 本ファイルは conn_rec の conn_config を利用して情報を保持しています。h2_conn_ctx_get(c) マクロを通じてどこからでもアクセス可能です。
  • C1（クライアント接続）と C2（ストリーム処理用接続）でコンテキストの役割が微妙に異なる点に注意してください。C2 は c2->master を通じて親のコンテキストを参照する設計になっています。
  • apr_atomic を使用しているフィールド（started, done）は、マルチスレッド環境での状態追跡に使用されます。
