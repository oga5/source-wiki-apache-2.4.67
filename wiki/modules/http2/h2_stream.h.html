modules/http2/h2_stream.h

概要

h2_stream.h は、HTTP/2 プロトコルにおける「ストリーム」を定義するヘッダーファイルです。HTTP/2 のストリームは、単一の接続（h2_session）内で多重化される個別のリクエストとレスポンスのペアを指します。

モジュール内での役割

modules/http2 サブシステムにおいて、ストリームのライフサイクル管理と状態遷移を司る中心的なデータ構造を提供します。h2_session が接続全体を管理するのに対し、本ファイルは個別のリクエスト処理単位のコンテキストを保持します。

責務

  • **ストリーム状態の管理**: RFC 7540 に準拠したストリーム状態（IDLE, OPEN, CLOSED 等）の保持と遷移の制御。
  • **データバッファリング**: 入出力用のバケットブリゲード（in_buffer, out_buffer）およびデータ転送用ビーム（h2_bucket_beam）の管理。
  • **リクエスト/レスポンスの保持**: リクエストメタデータ、トレーラー、およびレスポンスヘッダーの格納。
  • **イベント通知**: h2_stream_monitor を介した状態遷移やイベントの外部通知。

重要なシンボル

  • struct h2_stream: ストリームの全情報を保持するメイン構造体。
  • h2_stream_state_t: ストリームの現在の状態を示す列挙型（h2_stream.c で定義）。
  • h2_stream_dispatch(): ストリームに対するイベント（フレーム受信等）を処理し、状態遷移をトリガーする関数。
  • h2_stream_monitor: 状態遷移を監視するためのコールバック構造体。
  • H2_STRM_LOG: ストリーム固有の情報を付与したログ出力用マクロ。

制御フローとデータフロー

1. **生成**: h2_stream_create() により初期化され、h2_session に紐付けられます。 2. **データ入力**: h2_stream_add_header() や h2_stream_recv_DATA() を通じて、クライアントからのフレームがストリームのバッファに蓄積されます。 3. **処理**: ストリームが準備完了（h2_stream_prepare_processing）になると、ワーカーがリクエストを処理し、レスポンスを生成します。 4. **データ出力**: h2_stream_read_to() を通じて、処理済みのレスポンスデータがクライアントへ送出されます。 5. **終了**: h2_stream_destroy() により、関連するメモリプールと共に破棄されます。

直接依存

  • http_protocol.h: Apache HTTP Server のプロトコル定義。
  • h2.h: HTTP/2 共通定義。
  • h2_headers.h: HTTP/2 ヘッダー処理用定義。

関連コード

  • modules/http2/h2_stream.c: 本ヘッダーで宣言された関数の実装および状態遷移テーブル。
  • modules/http2/h2_session.h: ストリームを所有・管理するセッション層。
  • modules/http2/h2_mplx.h: 複数のストリームをワーカーに多重化して割り当てるための仕組み。

読者向けメモ

  • 本ファイルは AP_DEBUG が有効な場合、マジックナンバーによるメモリ整合性チェックを行います。
  • ストリームの状態遷移は非常に厳格です。h2_stream_dispatch を呼び出す際は、現在の状態が期待通りであることを確認してください。
  • h2_bucket_beam は、ストリームの入出力データをスレッド間で安全に受け渡すための重要なコンポーネントです。
