modules/http2/h2_headers.c

概要

HTTP/2 通信におけるレスポンスヘッダーを管理するためのカスタムバケット型（H2HEADERS）および関連するデータ構造の定義と操作関数を提供します。Apache HTTP Server のバケットチェーン内で HTTP/2 特有のメタデータを保持・伝搬させる役割を担います。

モジュール内での役割

mod_http2 サブシステムにおいて、レスポンスヘッダーをストリーム処理のパイプラインに流し込むためのブリッジとして機能します。AP_HAS_RESPONSE_BUCKETS が定義されていない環境（古い Apache バージョン等）において、ヘッダー情報をバケットとしてカプセル化し、他のフィルタやモジュールと連携します。

責務

  • **カスタムバケット管理**: H2HEADERS 型バケットの生成、破棄、クローン、および読み取りインターフェースの提供。
  • **ヘッダーデータ構造の抽象化**: HTTP ステータス、ヘッダーテーブル、ノート（メモ）を保持する h2_headers 構造体のライフサイクル管理。
  • **プロトコル変換と検証**: request_rec からのヘッダー生成時の HTTP/2 特有の制約（TLS 再ネゴシエーション禁止など）の適用。
  • **統計・計測**: ヘッダー文字列の合計バイト数計算。

重要なシンボル

データ構造

  • h2_headers: ヘッダー情報（ステータス、テーブル、ノート）を保持するメイン構造体。
  • h2_bucket_headers: apr_bucket のデータ領域に格納される、参照カウント付きのヘッダーラッパー。

バケット操作

  • h2_bucket_headers_create/make: H2HEADERS 型バケットの初期化と生成。
  • h2_bucket_headers_get: バケットから h2_headers 構造体を取り出すアクセサ。
  • h2_bucket_headers_clone: バケットのディープコピー。

ヘッダー管理・生成

  • h2_headers_create/rcreate: 汎用的な生成関数および request_rec を基にした生成関数。
  • h2_headers_clone/copy: メモリプールを跨いだヘッダー情報の複製。
  • h2_headers_die: エラー発生時のレスポンスヘッダー生成。

制御フローとデータフロー

1. **生成**: h2_headers_rcreate が request_rec から情報を抽出し、h2_headers を作成します。この際、is_unsafe チェックや TLS 再ネゴシエーションの判定を行い、必要に応じてステータスコードを H2_ERR_HTTP_1_1_REQUIRED に書き換えます。 2. **バケット化**: 生成された h2_headers は h2_bucket_headers_create を通じて apr_bucket にラップされ、出力フィルタチェーンに投入されます。 3. **消費**: h2_bucket_headers_headers_length 等の関数がバケット内の h2_headers を参照し、ヘッダーサイズを計算します。 4. **破棄**: bucket_destroy が参照カウントを管理し、ゼロになった時点で h2_headers とバケットメモリを解放します。

読者向けメモ

  • **メモリ所有権**: h2_headers 構造体およびその内部の apr_table_t は、生成時に指定された apr_pool_t に依存します。h2_headers_clone を使用する際は、適切なプールを指定してメモリリークや生存期間の不一致を防いでください。
  • **制約**: AP_HAS_RESPONSE_BUCKETS が定義されている環境では、このファイル内のロジックは無効化されます。
  • **パフォーマンス**: h2_headers_length はテーブル内の全ヘッダーを走査するため、ヘッダー数が多い場合は注意が必要です。
  • **未実装**: bucket_read は意図的に空（str = NULL）として実装されています。これは H2HEADERS バケットがメタデータ専用であり、直接的なデータ読み取りを想定していないためです。
