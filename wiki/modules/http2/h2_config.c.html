modules/http2/h2_config.c

概要

h2_config.c は、Apache HTTP Server の mod_http2 モジュールにおける設定管理の基盤です。サーバーレベル（server_rec）およびディレクトリレベル（request_rec）の設定構造体の定義、初期化、マージ、および設定値の取得・更新ロジックを担います。

モジュール内での役割

mod_http2 の設定システムにおける中心的な役割を果たします。他のコンポーネント（h2_session.c や h2_stream.c など）が実行時に必要なパラメータを安全に取得するためのインターフェースを提供し、Apache の設定ディレクティブと内部パラメータを橋渡しします。

責務

  • **設定構造体の管理**: h2_config（サーバー用）および h2_dir_config（ディレクトリ用）の定義とライフサイクル管理。
  • **設定の継承**: merge 関数による、親コンテキストから子コンテキストへの設定値の階層的なマージ。
  • **ディレクティブのハンドリング**: command_rec を通じた Apache 設定ディレクティブのパースとバリデーション。
  • **アクセスインターフェース**: request_rec や conn_rec を引数に取り、適切なスコープから設定値を解決するゲッター関数の提供。

重要なシンボル

構造体とデフォルト値

  • h2_config: サーバー全体の設定を保持する構造体。
  • h2_dir_config: ディレクトリ単位の設定を保持する構造体。
  • defconf, defdconf: 未設定時に使用されるデフォルト値の静的インスタンス。

設定の解決（ゲッター）

  • h2_config_sgeti, h2_config_sgeti64: サーバーレベルの設定値を取得。
  • h2_config_rgeti, h2_config_rgeti64: リクエストコンテキストから設定値を取得（ディレクトリ設定を優先し、なければサーバー設定へフォールバック）。
  • h2_config_cgeti, h2_config_cgeti64: コネクションコンテキストから設定値を取得。

設定の更新（セッター）

  • h2_srv_config_seti, h2_srv_config_seti64: サーバー設定構造体への値の書き込み。
  • h2_config_seti, h2_config_seti64: ディレクトリ設定とサーバー設定を適切に振り分けて更新。

ディレクティブハンドラ

  • h2_cmds: Apache 設定ディレクティブの定義配列。
  • h2_conf_set_*: 各ディレクティブに対応するパース関数群。

制御フローとデータフロー

1. **初期化**: h2_config_create_svr / h2_config_create_dir が Apache の設定フェーズで呼ばれ、メモリプール上に構造体が確保されます。 2. **マージ**: h2_config_merge / h2_config_merge_dir が呼ばれ、H2_CONFIG_GET マクロを用いて「子設定が未設定（DEF_VAL）であれば親設定を使用する」というロジックで階層的な設定が構築されます。 3. **解決**: h2_config_rgeti 等の関数が呼ばれると、ap_get_module_config を介して現在のコンテキストの設定を取得し、デフォルト値との比較を経て最終的な値を返します。

読者向けメモ

  • **メモリ管理**: すべての設定構造体は Apache のリクエストまたはサーバーのメモリプール（apr_pool_t）上で確保されます。ライフサイクルは Apache の設定フェーズに依存します。
  • **マージの不変性**: マージ処理は既存の構造体を変更せず、常に新しい構造体を生成して返します。
  • **未設定値の扱い**: DEF_VAL (-1) は「未設定」を意味するマジックナンバーです。この値が残っている場合、マージ時に親の設定が適用されます。
  • **省略された詳細**: h2_priority のハッシュマップ操作や、push_list の配列操作は、APR のユーティリティ関数に依存しており、本ファイルでは主にそのラッパーとして機能しています。
