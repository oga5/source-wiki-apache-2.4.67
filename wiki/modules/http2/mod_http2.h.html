modules/http2/mod_http2.h

概要

本ファイルは、mod_http2 モジュールが他の Apache モジュールに対して公開する「オプション関数（Optional Functions）」のインターフェース定義です。HTTP/2 接続の状態確認や、環境変数の取得、および特定の条件下での接続監視機能を提供します。

モジュール内での役割

mod_http2 のパブリック API として機能します。他のモジュールが mod_http2 の内部実装に直接依存することなく、HTTP/2 関連の機能を利用できるようにするためのブリッジ層です。

責務

  • **HTTP/2 状態の公開**: 接続が HTTP/2 であるかの判定や、ワーカー数の取得。
  • **環境変数連携**: http2_var_lookup を通じた HTTP/2 固有の変数提供。
  • **接続監視の補助**: WebSocket 実装などで利用される apr_pollfd_t の取得支援。
  • **後方互換性の維持**: 非推奨となったリクエストエンジン関連 API の定義保持。

重要なシンボル

  • http2_is_h2: 接続が HTTP/2 かを判定する標準的な手段。
  • http2_get_pollfd_from_conn: HTTP/2 ストリーム接続の入力を監視するための apr_pollfd_t を取得する関数。
  • AP_HTTP2_HAS_GET_POLLFD: 上記関数が利用可能であることを示すマクロ。
  • h2_req_engine (非推奨): かつて実験的に導入されたリクエストエンジン機能の構造体。

制御フローとデータフロー

本ファイルはインターフェース定義のみであり、ロジックを含みません。

  • 他のモジュールが APR_RETRIEVE_OPTIONAL_FN を使用してこれらの関数を取得し、実行時に mod_http2 がロードされていれば呼び出しが可能になります。
  • http2_get_pollfd_from_conn は、呼び出し元が提供した apr_pollfd_t 構造体を mod_http2 内部の接続情報で埋めることで、非同期 I/O 監視を可能にします。

直接依存

  • apr_optional.h: オプション関数メカニズムを提供。
  • httpd.h / conn_rec / request_rec: Apache のコアデータ構造への依存。

関連コード

  • modules/http2/mod_http2.c: 本ヘッダーで宣言された関数の実装（APR_REGISTER_OPTIONAL_FN による登録）が含まれます。
  • modules/http2/h2_session.h: 実際の HTTP/2 セッション管理ロジック。
  • modules/http2/h2_conn_ctx.h: 接続コンテキストの管理。

読者向けメモ

  • **非推奨 API**: h2_req_engine 関連の関数は現在非推奨であり、将来的に削除される予定です。新規開発では使用しないでください。
  • **関数利用時の注意**: オプション関数は mod_http2 がロードされていない場合に NULL を返す可能性があるため、呼び出し時には必ずチェックを行ってください。
  • **WebSocket 連携**: http2_get_pollfd_from_conn は、Apache の標準的な ap_get_pollfd_from_conn が利用できない環境でのフォールバックとして設計されています。可能な限り標準 API を優先してください。
