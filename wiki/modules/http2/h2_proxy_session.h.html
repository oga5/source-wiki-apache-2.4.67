modules/http2/h2_proxy_session.h

概要

本ファイルは、Apache HTTP Server のプロキシモジュール（mod_proxy_http2）において、バックエンドサーバーとの HTTP/2 セッションを管理するためのインターフェースを定義しています。

モジュール内での役割

modules/http2 サブシステムの一部として、フロントエンドからのリクエストをバックエンドの HTTP/2 接続へ橋渡しする際の「セッション層」を担います。h2_session.h がクライアントとの接続を扱うのに対し、本ファイルはプロキシとしてのバックエンド接続を抽象化します。

責務

  • **セッション状態管理**: バックエンドとの接続状態（初期化、ビジー、シャットダウン等）の追跡。
  • **PING 制御**: 接続の生存確認とタイムアウト管理（h2_ping_state_t）。
  • **ストリーム管理**: バックエンドへ送信される個別のリクエスト（ストリーム）のライフサイクル管理。
  • **I/O 抽象化**: nghttp2 ライブラリと Apache のバケットブリゲード間のデータ転送の仲介。

重要なシンボル

  • h2_proxy_session: セッション全体を保持するメイン構造体。nghttp2_session インスタンスと接続状態を管理します。
  • h2_proxys_state: セッションの現在のフェーズを示す列挙型。
  • h2_ping_state_t: 接続の健全性を確認するための PING 状態管理用。
  • h2_proxy_session_process(): セッションのメインループ関数。読み書きのサイクルを 1 ステップ実行します。
  • h2_proxy_session_submit(): 新規リクエストをバックエンドへ送信するために使用します。

制御フローとデータフロー

1. **初期化**: h2_proxy_session_setup でセッションを生成し、バックエンド接続を確立します。 2. **リクエスト投入**: h2_proxy_session_submit を通じてリクエストがセッションのストリームキューに追加されます。 3. **処理ループ**: h2_proxy_session_process が呼び出されるたびに、nghttp2 のイベントループと Apache のバケットブリゲード（input/output）の間でデータの読み書きが行われます。 4. **状態遷移**: PING 応答や GOAWAY フレームの受信、またはタイムアウトイベントに応じて h2_proxys_state が遷移します。

直接依存

  • nghttp2/nghttp2.h: HTTP/2 プロトコル処理のコアライブラリ。
  • apr_strings.h, httpd.h, mod_proxy.h: Apache APR およびプロキシフレームワークの基本定義。

関連コード

  • h2_proxy_session.c: 本ヘッダーで宣言された関数の実装。
  • h2_proxy_util.h: プロキシ処理で共通して利用されるユーティリティ関数群。
  • h2_session.h: クライアント側の HTTP/2 セッション管理（対比用）。

読者向けメモ

  • 本ファイルは mod_proxy_http2 の心臓部です。特に h2_proxy_session_process の戻り値（APR_EAGAIN, APR_SUCCESS, APR_EOF）は、呼び出し元のイベントループが次に何をすべきかを決定する重要な指標となります。
  • PING 状態管理は、再利用可能な接続の健全性を保証するために重要です。バックエンドが PING に応答しない場合、セッションは適切にクローズされる設計になっています。
