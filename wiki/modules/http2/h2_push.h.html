modules/http2/h2_push.h

概要

HTTP/2 Server Push 機能を管理するためのヘッダーファイルです。サーバーがクライアントに対してリソースを先読み（Push）させる際の重複排除と、Push 候補の収集・管理を担います。

モジュール内での役割

modules/http2 サブシステムにおいて、リクエスト/レスポンス処理中に発生する「Push 可能なリソース」の特定と、同一コネクション内での重複送信を防ぐためのフィルタリング層として機能します。

責務

  • **Push 候補の収集**: レスポンスに含まれる Link ヘッダー等を解析し、Push すべきリソースを特定する。
  • **Push Diary（履歴管理）**: 既に Push 済みのリソースをハッシュ値で記録し、重複送信を抑制する。
  • **キャッシュダイジェストの生成**: 過去の Push 履歴を基に、クライアントのキャッシュ状態を推定するためのデータ構造を管理する。

重要なシンボル

  • h2_push: Push 対象のリクエスト情報と優先度を保持する構造体。
  • h2_push_diary: 過去の Push 履歴を保持する構造体。最大エントリ数やハッシュ計算アルゴリズムを管理。
  • h2_push_collect: レスポンスから Push 候補を抽出するメイン関数。
  • h2_push_diary_update: 候補リストを Diary と照合し、未送信のリソースのみを抽出するフィルタリング関数。

制御フローとデータフロー

1. **収集**: h2_push_collect がレスポンスヘッダーを走査し、h2_push オブジェクトの配列を生成します。 2. **フィルタリング**: h2_push_diary_update がセッション内の h2_push_diary を参照し、既に送信済みのハッシュ値を持つリソースを除外します。 3. **更新**: 新たに送信されるリソースのハッシュ値が h2_push_diary に追加され、古いエントリは破棄されます。

直接依存

  • http_protocol.h: Apache HTTP Server のプロトコル処理基盤。
  • h2.h: HTTP/2 モジュール共通の定数や型定義。
  • h2_headers.h: HTTP/2 レスポンスヘッダーの抽象化。

関連コード

  • modules/http2/h2_push.c: 上記で宣言された関数の実装。Link ヘッダーのパースロジックが含まれます。
  • modules/http2/h2_session.c: セッション単位で h2_push_diary を保持・管理する上位レイヤー。
  • modules/http2/h2_stream.c: ストリーム単位での Push 実行を制御。

読者向けメモ

  • **歴史的背景**: 本ファイルで言及されている「Cache-Digest」は、かつての IETF ドラフトに基づいています。現在は標準化から外れていますが、mod_h2 では重複排除のための効率的なデータ構造として引き続き利用されています。
  • **ハッシュ計算**: OpenSSL が利用可能な環境では SHA256 が使用され、それ以外では apr_hashfunc_default にフォールバックする設計になっています。
  • **メモリ消費**: h2_push_diary は固定サイズのメモリを使用するように設計されており、大規模なトラフィック下でも安定した動作を維持するよう配慮されています。
