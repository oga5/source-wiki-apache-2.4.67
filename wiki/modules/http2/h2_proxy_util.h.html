modules/http2/h2_proxy_util.h

概要

h2_proxy_util.h は、mod_proxy_http2 モジュールが HTTP/2 プロキシ処理を行う際に必要となる、汎用的なデータ構造とユーティリティ関数を定義するヘッダーファイルです。

モジュール内での役割

modules/http2 サブシステムにおいて、プロキシ固有のロジックを支える基盤ライブラリとして機能します。主に h2_proxy_session.c などの上位コンポーネントが、複雑なデータ管理を抽象化するために利用します。

責務

  • **データ構造の提供**: 整数IDベースのハッシュテーブル (ihash)、ソート可能な整数キュー (iqueue)、およびスレッドセーフな FIFO キュー (fifo) の管理。
  • **HTTP/2 変換支援**: nghttp2 形式へのヘッダー変換や、プロキシリクエスト構造体 (h2_proxy_request) の構築。
  • **ユーティリティ**: ログ出力用のフォーマット変換や、ヘッダーの正規化（キャメルケース変換など）といった補助的な処理。

重要なシンボル

  • h2_proxy_ihash_t: 構造体内の整数メンバーをキーとして高速にアクセスするためのハッシュテーブル。
  • h2_proxy_iqueue: ストリームIDなどを管理するための、ソート可能な優先度付きキュー。
  • h2_proxy_fifo: プロキシセッション間でのデータ受け渡しに用いる、スレッドセーフな FIFO キュー。
  • h2_proxy_request: プロキシ転送に必要なリクエスト情報を保持する構造体。

制御フローとデータフロー

本ファイルは主にデータ構造の定義とインターフェースを提供します。 1. **データ管理**: ihash や iqueue を通じて、アクティブなストリームやタスクを追跡します。 2. **変換処理**: h2_proxy_util_nghd_make 等を呼び出し、Apache の apr_table_t 形式のヘッダーを nghttp2 が解釈可能な nghttp2_nv 配列へ変換します。 3. **キューイング**: fifo を介して、ワーカースレッドとセッションスレッド間でリクエストやイベントの非同期的な受け渡しを行います。

直接依存

  • apr_lib.h, apr_thread_mutex.h 等の APR (Apache Portable Runtime) ライブラリ。
  • httpd.h, mod_proxy.h: Apache HTTP Server のコアおよびプロキシ API。
  • nghttp2/nghttp2.h: HTTP/2 プロトコル処理のための外部ライブラリ。

関連コード

  • h2_proxy_util.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • h2_proxy_session.h: 本ユーティリティを利用してプロキシセッションを管理する主要コンポーネント。
  • h2_util.h: プロキシに限定されない、より広範な HTTP/2 共通ユーティリティ。

読者向けメモ

  • 本ファイルで定義されている fifo は、標準の apr_queue よりも特定のプロキシ要件（重複排除など）に最適化されています。
  • ihash は構造体のオフセットを指定して初期化するため、利用時には offsetof() マクロを適切に使用する必要があります。
  • h2_proxy_request は、バックエンドへの転送前にリクエスト情報を正規化するための重要な中間表現です。
