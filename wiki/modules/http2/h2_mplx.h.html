modules/http2/h2_mplx.h

概要

h2_mplx.h は、HTTP/2 セッションにおける「ストリーム多重化（Multiplexer）」のコア定義を提供します。クライアントとのメイン接続（c1）と、個々のリクエストを処理するセカンダリ接続（c2）の間で、データ、イベント、および処理リソースを仲介するハブの役割を果たします。

モジュール内での役割

mod_http2 において、h2_session が HTTP/2 プロトコル層を管理するのに対し、h2_mplx はそのセッション内のリクエスト処理を非同期に管理します。h2_workers と連携し、スレッドプール上で実行されるリクエスト処理と、メイン接続の I/O を橋渡しします。

責務

  • **リクエストのスケジューリング**: 優先度に基づいたストリーム処理の順序付け。
  • **接続仲介**: c1（メイン）と c2（ワーカースレッド）間のデータフローの同期と管理。
  • **ライフサイクル管理**: ストリームの生成から破棄、およびワーカースレッドの割り当て管理。
  • **リソース制限**: 同時実行ストリーム数やメモリ使用量の制御。

重要なシンボル

  • h2_mplx: 多重化器のメイン構造体。ストリームの状態管理、ロック、キュー、ワーカースレッドへの参照を保持。
  • h2_mplx_c1_*: メイン接続（c1）から呼び出される API。ストリームの開始、ポーリング、シャットダウンを担当。
  • h2_mplx_c2_*: セカンダリ接続（c2）から呼び出される API。I/O 完了通知やストリーム取得を担当。
  • h2_mplx_worker_*: ワーカースレッドがリクエストを取得するために使用する API。

制御フローとデータフロー

1. **入力**: h2_mplx_c1_process により、新しいストリームが処理キューに追加されます。 2. **実行**: h2_mplx_worker_pop_c2 を通じてワーカースレッドがリクエストを取得し、c2 コンテキストで処理を開始します。 3. **同期**: h2_mplx_c1_poll を使用して、c1 は c2 からの出力データやイベントを監視します。 4. **終了**: 処理が完了したストリームは、h2_mplx_c1_stream_cleanup を経て破棄キュー（spurge）へ移動し、最終的にクリーンアップされます。

直接依存

  • apr_queue.h: 内部キュー管理。
  • h2_workers.h: ワーカースレッド管理。
  • h2_private.h, h2.h: HTTP/2 共通定義。

関連コード

  • h2_mplx.c: 上記 API の実装。
  • h2_stream.h: 多重化される個々のストリームの定義。
  • h2_c2.h: セカンダリ接続の処理詳細。
  • h2_bucket_beam.h: c1-c2 間のデータ転送（バケットの受け渡し）に使用される機構。

読者向けメモ

  • **命名規則**: 関数名のプレフィックス（c1_, c2_, worker_）は、その関数を呼び出すべきコンテキストを厳密に示しています。これらを混同するとデッドロックや競合状態を引き起こすため注意してください。
  • **スレッド安全性**: h2_mplx は内部で apr_thread_mutex_t を使用して保護されています。複雑な状態遷移を伴うため、ロックの取得範囲を最小限に抑える設計になっています。
  • **デバッグ**: H2_MPLX_MSG マクロを使用すると、ログ出力時に child_num と id を含めた一貫性のあるログが生成可能です。
