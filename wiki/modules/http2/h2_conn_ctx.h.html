modules/http2/h2_conn_ctx.h

概要

h2_conn_ctx.h は、Apache HTTP/2 モジュールにおいて、各接続（conn_rec）に付随する状態管理コンテキストを定義するヘッダーファイルです。HTTP/2 のセッション管理（C1）と、ストリーム処理のために生成されるサブ接続（C2）の両方で共通のメタデータ構造を提供します。

モジュール内での役割

mod_http2 は、クライアントとの直接接続（C1）と、内部的なストリーム処理用接続（C2）を区別して扱います。本ファイルは、これら異なる接続タイプに共通の「HTTP/2 関連のコンテキスト」を conn_config に格納・取得するためのインターフェースを提供し、モジュール全体の状態追跡を可能にします。

責務

  • **接続コンテキストの定義**: h2_conn_ctx_t 構造体による、接続ごとの状態（セッション情報、ストリームID、タイムアウト、入出力パイプ等）の保持。
  • **ライフサイクル管理**: 接続の初期化、セッションの割り当て、および接続終了時のデタッチ処理。
  • **C1/C2 識別**: クライアント接続と内部処理用接続のメタデータを統合的に管理し、必要に応じて適切なリソース（h2_mplx や h2_bucket_beam）を紐付ける。

重要なシンボル

  • h2_conn_ctx_t: 接続ごとの状態を保持するメイン構造体。
  • h2_conn_ctx_get(c): conn_rec からコンテキストを取得するマクロ。
  • h2_conn_ctx_create_for_c1: クライアント接続（C1）用のコンテキスト生成関数。
  • h2_conn_ctx_init_for_c2: ストリーム処理用接続（C2）用のコンテキスト初期化関数。
  • H2_PIPE_IN / H2_PIPE_OUT: C2 接続におけるデータ通知パイプのインデックス。

制御フローとデータフロー

1. **初期化**: 接続確立時に h2_conn_ctx_create_for_c1 が呼ばれ、ap_set_module_config を通じて conn_rec にコンテキストがアタッチされます。 2. **C2 への展開**: ストリーム処理が開始されると、h2_conn_ctx_init_for_c2 が呼ばれ、親接続（C1）の情報を引き継ぎつつ、h2_mplx や h2_bucket_beam がコンテキストに設定されます。 3. **状態更新**: 処理中、h2_conn_ctx_set_timeout 等を通じて、入出力ビームやパイプのタイムアウト設定が同期的に更新されます。 4. **破棄**: 接続終了時に h2_conn_ctx_detach が呼ばれ、コンテキストの参照が解除されます。

直接依存

  • h2.h: モジュール共通の定義。
  • apr_pools.h / apr_network_io.h: メモリ管理およびソケット操作用。
  • httpd.h: Apache HTTP Server のコア構造体（conn_rec, server_rec）。

関連コード

  • h2_conn_ctx.c: 本ヘッダーで宣言された関数の実装。
  • h2_session.h: C1 接続のセッション管理。
  • h2_mplx.h: C2 接続におけるストリーム多重化管理。
  • h2_bucket_beam.h: ストリーム間でのデータ転送メカニズム。

読者向けメモ

  • 本構造体は C1（クライアント接続）と C2（内部処理用接続）の両方で再利用されるため、各フィールドが「C1 用」「C2 用」のどちらで利用されるかというコメントを注視してください。
  • h2_conn_ctx_get マクロは、ap_get_module_config をラップしたものであり、HTTP/2 モジュールが有効な接続であれば常に有効なコンテキストが返ることを前提としています。
  • C2 接続のライフサイクルは h2_mplx と密接に関連しているため、デバッグ時は h2_mplx.c と併せて参照することをお勧めします。
