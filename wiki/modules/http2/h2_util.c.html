modules/http2/h2_util.c

概要

h2_util.c は、HTTP/2 モジュール全体で使用される汎用的なユーティリティ関数、データ構造、および変換ロジックを提供する基盤ファイルです。デバッグ支援、メモリ管理、データ構造の操作、および HTTP/1 と HTTP/2 間のヘッダー変換など、多岐にわたる補助機能を集約しています。

モジュール内での役割

modules/http2 サブシステムにおける「共通ライブラリ」として機能します。特定のセッションやストリームの状態管理に依存しない、純粋なデータ処理やデータ構造の管理を担当します。

責務

  • **デバッグ・フォーマット**: フレームのダンプ、ヘッダーのキャメルケース変換、ログ出力用フォーマット。
  • **データ構造**: 整数 ID をキーとするハッシュテーブル (h2_ihash)、ソート済み整数キュー (h2_iqueue)、スレッドセーフな FIFO キュー (h2_fifo, h2_ififo) の提供。
  • **HTTP 変換**: HTTP/1 の apr_table_t と HTTP/2 の nghttp2_nv 間の変換、ヘッダーのバリデーション、不要ヘッダーのフィルタリング。
  • **バケット操作**: apr_bucket_brigade の連結、コピー、サイズ計算、およびデバッグ出力。
  • **Base64URL 変換**: HTTP/2 仕様で利用される Base64URL エンコード/デコード。

重要なシンボル

データ構造

  • h2_ihash_t: 構造体内の整数メンバーをキーとするハッシュテーブル。h2_ihash_create で作成し、h2_ihash_add/get/remove で操作。
  • h2_iqueue: 整数 ID のソート済みキュー。優先度管理などに利用。
  • h2_fifo / h2_ififo: スレッドセーフな FIFO キュー。apr_thread_mutex_t と apr_thread_cond_t を使用し、ブロッキング/非ブロッキング操作をサポート。

ヘッダー変換

  • h2_req_add_header: HTTP/2 ヘッダーを HTTP/1 apr_table_t に変換して追加。
  • h2_res_create_ngheader: HTTP/1 レスポンスを HTTP/2 フレーム用ヘッダーに変換。
  • h2_util_ignore_resp_header: HTTP/2 転送時に無視すべきヘッダーの判定。

ユーティリティ

  • h2_util_hex_dump: バイナリデータのデバッグ用 16 進数ダンプ。
  • h2_log2: 2 のべき乗に対する対数計算。

制御フローとデータフロー

  • **キュー操作**: h2_fifo は apr_thread_mutex_lock で保護されたクリティカルセクション内で fifo_push_int / pull_head を実行します。not_empty / not_full 条件変数を用いてスレッド間の同期を行います。
  • **ヘッダー変換**: ngheader_create が中心となり、apr_table_do を介してテーブル内の全ヘッダーを走査し、add_header でバリデーションと nghttp2_nv 構造体への詰め込みを行います。
  • **バケット操作**: h2_brigade_concat_length は、バケットを src から dest へ移動させつつ、指定された length に達するまでバケットの分割 (apr_bucket_split) を行います。

関連コード

  • h2_headers.c: ヘッダーのより詳細な管理。
  • h2_bucket_beam.c: バケットのストリーム間転送。
  • h2_session.c: 本ユーティリティを利用してセッションを制御。

読者向けメモ

  • **メモリ管理**: 多くの関数が apr_pool_t を引数に取り、メモリのライフサイクルをプールに依存させています。
  • **スレッド安全性**: h2_fifo 系はスレッドセーフですが、h2_ihash や h2_iqueue はスレッドセーフではありません。呼び出し側で適切なロックを確保する必要があります。
  • **制約**: h2_ihash は構造体内のオフセット (offsetof) を利用してキーを抽出するため、対象構造体のメモリレイアウトに依存します。
  • **省略**: 本ファイルには h2_util.h で宣言されている一部の関数（h2_util_base64url_encode 等）の実装が含まれていますが、複雑なロジックを持つ h2_bucket_beam 関連の高度なバケット操作は別ファイルに分離されています。
