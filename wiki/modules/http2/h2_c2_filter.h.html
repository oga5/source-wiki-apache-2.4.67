modules/http2/h2_c2_filter.h

概要

HTTP/2 モジュールにおける「セカンダリ接続（C2）」用のフィルタ群を定義するヘッダーファイルです。Apache HTTP Server のフィルタチェーンに介入し、HTTP/2 ストリーム固有の処理や、HTTP/1.1 形式のレスポンスから HTTP/2 への変換を担います。

モジュール内での役割

modules/http2 内で、HTTP/2 のストリーム処理（C2）と Apache の標準的なフィルタパイプラインを橋渡しする役割を持ちます。特に、HTTP/2 のレスポンスバケット機能が有効か否かに応じて、処理パスを切り替える設計になっています。

責務

  • **入力フィルタ**: セカンダリ接続において、リクエスト情報を注入し、一度実行されたら自身を削除する「使い捨て」フィルタの提供。
  • **出力フィルタ**: レスポンスヘッダーやノート（notes）を検査し、HTTP/2 固有の制御（PUSH の無効化やエラーコードの調整）を行う。
  • **互換性レイヤー**: AP_HAS_RESPONSE_BUCKETS が無効な環境において、HTTP/1.1 形式のレスポンスを解析し、HTTP/2 ストリームへ変換するための代替フィルタを提供。

重要なシンボル

  • h2_c2_filter_request_in: ストリーム開始時にリクエスト情報を注入する入力フィルタ。
  • h2_c2_filter_notes_out: レスポンスのメタデータを検査し、HTTP/2 制御ノートを調整する出力フィルタ。
  • h2_c2_filter_catch_h1_out / h2_c2_filter_response_out: レスポンスバケット機能が利用できない環境での、HTTP/1.1 レスポンス解析用フィルタ。

制御フローとデータフロー

1. **入力**: h2_c2_filter_request_in が接続開始時に呼び出され、h2_conn_ctx からリクエスト情報を取得してパイプラインに挿入後、自身を ap_remove_input_filter で削除します。 2. **出力**: h2_c2_filter_notes_out がレスポンスバケットを走査し、内部リダイレクト時のノート継承や、設定に基づく PUSH モードの制御を行います。 3. **フォールバック**: レスポンスバケット非対応環境では、h2_c2_filter_catch_h1_out 等が HTTP/1.1 形式のストリームを解析し、HTTP/2 用のヘッダー構造へ変換します。

直接依存

  • h2.h: HTTP/2 の基本定義および定数。

関連コード

  • h2_c2_filter.c: 本ヘッダーで宣言された関数の実装。
  • h2_c2.c: セカンダリ接続のライフサイクル管理。
  • h2_conn_ctx.h: 接続コンテキストの管理（リクエスト情報の取得元）。
  • h2_headers.h: レスポンスヘッダーの解析・構築用ユーティリティ。

読者向けメモ

  • 本ファイルは AP_HAS_RESPONSE_BUCKETS マクロの有無により、提供される API が大きく異なります。現在の Apache HTTP Server のビルド環境がどちらをサポートしているかを確認してください。
  • h2_c2_filter_request_in は一度実行されると自身を削除する設計であるため、フィルタチェーンの管理において副作用を考慮する必要があります。
