modules/http2/h2_c1.c

概要

h2_c1.c は、Apache HTTP Server における HTTP/2 プロトコル処理の「プライマリ接続（C1）」を管理する中心的なモジュールです。クライアントから直接接続されたコネクションを監視し、HTTP/2 へのアップグレードまたは直接検出（H2Direct）を行い、セッションのライフサイクルを制御します。

モジュール内での役割

HTTP/2 サブシステムにおける「フロントエンド」として機能します。h2_session.c がプロトコルロジックを担うのに対し、本ファイルは Apache の conn_rec（接続レコード）と HTTP/2 セッションを橋渡しし、MPM（Multi-Processing Module）の接続状態管理と統合します。

責務

  • **接続の初期化と検出**: HTTP/1.1 接続からのプロトコル切り替え（Upgrade）や、マジックトークンによる直接検出（H2Direct）の実行。
  • **ライフサイクル管理**: h2_session の生成、実行、および終了処理。
  • **MPM 統合**: 非同期 MPM（Event MPM 等）と同期 MPM の両方に対応した接続状態（CONN_STATE）の遷移管理。
  • **リソース管理**: ワーカープール（h2_workers）の初期化とシャットダウン。

重要なシンボル

ライフサイクル管理

  • h2_c1_child_init: 子プロセス起動時にワーカープールと設定を初期化。
  • h2_c1_child_stopping: プロセス終了時のワーカーリソース解放。
  • h2_c1_setup: h2_session を生成し、接続コンテキストに割り当て。reqtimeout フィルタの除去を行う。

接続処理・フック

  • h2_c1_hook_process_connection: process_connection フックの本体。HTTP/2 への切り替え判定とセッション実行のトリガー。
  • h2_c1_run: セッションのメインループ。MPM の状態に応じて読み取り待ちや書き込み完了状態へ遷移させる。
  • h2_c1_pre_close: 接続終了直前の GOAWAY 送信等のクリーンアップ。

判定・設定

  • h2_c1_allows_direct: H2Direct 設定とプロトコル適合性の確認。
  • h2_c1_can_upgrade: Upgrade ヘッダーによるプロトコル切り替えの可否判定。

制御フローとデータフロー

1. **接続フック**: h2_c1_hook_process_connection が呼ばれ、AP_MODE_SPECULATIVE で 24 バイトを先読みし、HTTP/2 マジックトークンを検証。 2. **セッション生成**: 検出成功時、h2_c1_setup が呼ばれ、h2_session オブジェクトが生成される。 3. **メインループ**: h2_c1_run が h2_session_process を呼び出し、HTTP/2 フレームを処理。 4. **状態遷移**: h2_session の状態（H2_SESSION_ST_*）に基づき、c->cs->state を CONN_STATE_ASYNC_WAITIO や CONN_STATE_WRITE_COMPLETION に設定し、MPM に制御を戻す。

読者向けメモ

  • **メモリ所有権**: h2_session は接続プール（c->pool）に関連付けられ、接続の終了とともに解放されます。
  • **制約**: reqtimeout フィルタは HTTP/1.1 専用であるため、HTTP/2 確立時に明示的に除去されます。
  • **非同期 MPM**: async_mpm フラグにより、Event MPM などの非同期 I/O をサポートする MPM と、従来の Prefork/Worker MPM での挙動を切り替えています。
  • **注意点**: h2_c1_run 内の CONN_STATE_ASYNC_WAITIO への遷移は、PR 63534 に関連する複雑な互換性処理を含んでおり、MPM の動作に強く依存します。
