modules/http2/h2_switch.h

概要

h2_switch.h は、Apache HTTP Server における HTTP/1.1 から HTTP/2 へのプロトコル切り替え（アップグレード）を管理するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

modules/http2 サブシステムにおいて、本ファイルはプロトコルネゴシエーションの入り口を制御します。主に mod_http2 がサーバー起動時および接続確立時に、HTTP/2 を利用可能にするためのフック登録と初期化を担当します。

責務

  • **初期化の提供**: サーバー起動時のモジュール初期化処理 (h2_switch_init) の宣言。
  • **フックの登録**: Apache のプロトコル切り替えメカニズムに mod_http2 を統合するためのフック登録関数 (h2_switch_register_hooks) の提供。

重要なシンボル

  • h2_switch_init: モジュールのライフサイクル初期化関数。
  • h2_switch_register_hooks: Apache の ap_hook_register を介して、プロトコル提案や切り替えに関連するコールバックを登録する関数。

制御フローとデータフロー

本ファイル自体は宣言のみですが、対になる h2_switch.c では以下のフローが実行されます。 1. **初期化**: h2_switch_init が呼ばれ、モジュールの準備が行われます。 2. **提案 (Propose)**: クライアントからの接続に対し、HTTP/2 が利用可能か（TLS かクリアテキストか、MPM が対応しているか等）を判定します。 3. **検証**: Upgrade ヘッダーや HTTP2-Settings の存在を確認し、切り替え条件を満たしているか検証します。

直接依存

  • apr_pools.h: メモリ管理用。
  • httpd.h: Apache サーバーのコア構造体（server_rec 等）の定義。

関連コード

  • h2_switch.c: 本ヘッダーで宣言された関数の実装。
  • h2_protocol.c: プロトコル識別子やネゴシエーションのロジックが含まれます。
  • mod_http2.c: モジュールのメインエントリポイントであり、ここで h2_switch_register_hooks が呼び出されます。

読者向けメモ

  • このファイルは、HTTP/2 への切り替えロジックの「ゲートキーパー」です。
  • プロトコル切り替えの条件（Upgrade ヘッダーの有無やボディの有無など）を調査したい場合は、h2_switch.c 内の h2_protocol_propose 関数を参照してください。
  • 本ファイルは非常に軽量であり、主にモジュールのセットアップフェーズでのみ関与します。
