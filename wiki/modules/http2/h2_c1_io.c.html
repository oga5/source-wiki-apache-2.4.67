modules/http2/h2_c1_io.c

概要

HTTP/2 セッションにおけるクライアント接続（C1）の入出力処理を担うモジュールです。Apache HTTP Server のフィルタチェーンと nghttp2 ライブラリの間で、バッファリング、TLS 最適化、およびデータ転送の仲介を行います。

モジュール内での役割

h2_session の下位レイヤーとして機能し、接続ごとの入出力バッファ（h2_c1_io）を管理します。特に TLS 接続時のパケットサイズ最適化や、効率的なデータ転送のためのスクラッチバッファリングを提供します。

責務

  • **出力バッファリング**: TLS 接続時に MTU を考慮したチャンクサイズ（WRITE_SIZE_INITIAL / MAX）でデータをまとめ、効率的な TLS レコード生成を支援する。
  • **入力フィード**: ap_get_brigade を通じてクライアントからのデータを読み込み、nghttp2_session_mem_recv へ渡す。
  • **フィルタ管理**: apr_bucket_brigade を使用して、接続フィルタチェーンへのデータ投入とフラッシュ制御を行う。
  • **デバッグ支援**: C1_IO_BB_LOG マクロによる、バケットブリゲードの状態可視化。

重要なシンボル

ライフサイクル・初期化

  • h2_c1_io_init: セッションに関連付けられた IO 構造体の初期化。TLS の有無やウォームアップ設定を適用する。

データ操作・転送

  • h2_c1_io_add_data: 生データをバッファに追加。バッファリングが有効な場合はスクラッチ領域へコピー。
  • h2_c1_io_append: apr_bucket_brigade を受け取り、メタデータや非バッファ対象バケットを適切に処理して出力キューへ追加。
  • h2_c1_io_pass: バッファリングされたデータを接続フィルタへ書き出す。
  • h2_c1_io_assure_flushed: 保留中のデータがある場合に強制フラッシュを行う。

入力処理

  • h2_c1_read: クライアントからの入力を読み込み、nghttp2 セッションへフィードするメインループ。

制御フローとデータフロー

1. **出力フロー**:

  • h2_c1_io_append がブリゲードを受け取り、assure_scratch_space を経由してスクラッチバッファにデータを集約。
  • pass_output が ap_pass_brigade を呼び出し、フィルタチェーンへデータを流す。
  • 転送量に応じて write_size を動的に調整し、TLS のウォームアップ/クールダウンを制御する。

    • 2. **入力フロー**:

  • h2_c1_read が read_and_feed を呼び出し、ap_get_brigade でデータを取得。
  • c1_in_feed_brigade がブリゲード内の各バケットを nghttp2_session_mem_recv に渡してプロトコル解析を進行させる。

読者向けメモ

  • **メモリ所有権**: h2_c1_io が管理する output ブリゲードは、h2_session のライフサイクルに依存します。スクラッチバッファは apr_bucket_alloc を使用し、append_scratch 時にヒープバケットとしてブリゲードへ所有権が移転します。
  • **再帰呼び出しの防止**: is_passing フラグにより、pass_output 内での再帰的な呼び出しを検知し、エラーを返します。
  • **TLS 最適化**: WRITE_SIZE_INITIAL (1300バイト) から開始し、warmup_size を超えると WRITE_SIZE_MAX (16KB) に拡大するロジックが含まれています。
  • **省略された詳細**: nghttp2 の内部的なストリーム状態遷移については本ファイルでは扱わず、h2_session.c に委譲されています。
