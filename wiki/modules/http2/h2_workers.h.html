modules/http2/h2_workers.h

概要

h2_workers.h は、HTTP/2 モジュールにおける「セカンダリ接続」を並列実行するための専用スレッドプール管理機能を提供します。動的なスレッド生成・破棄をサポートし、負荷に応じたスケーラブルな接続処理を実現します。

モジュール内での役割

modules/http2 サブシステムにおいて、h2_mplx（多重化）と連携し、実際の接続処理を担うワーカー群を抽象化します。Apache HTTP Server の MPM（Multi-Processing Module）とは独立した、HTTP/2 セッション内での接続処理の並列化を担当します。

責務

  • **スレッドプール管理**: 指定された最小・最大スレッド数に基づき、ワーカーのライフサイクルを制御する。
  • **接続プロデューサーの抽象化**: ap_conn_producer_t を介して、接続ソース（プロデューサー）とワーカーを疎結合に接続する。
  • **負荷適応**: アイドル状態のスレッドを一定時間後に終了させることで、リソース消費を最適化する。

重要なシンボル

  • h2_workers: ワーカープール全体を管理するメイン構造体。
  • ap_conn_producer_t: 接続（conn_rec）を提供するソースを定義する構造体。
  • h2_workers_register(): 新しい接続プロデューサーを登録する。
  • h2_workers_activate(): プロデューサーをアクティブ化し、ワーカーによる接続取得を許可する。
  • h2_workers_join(): プロデューサーの処理を停止し、完了を待機する。

制御フローとデータフロー

1. **登録**: h2_workers_register でプロデューサーを登録し、コールバック（next, done, shutdown）を紐付ける。 2. **活性化**: h2_workers_activate を呼び出すと、プロデューサーがアクティブリストに移動し、アイドル中のワーカーが next コールバックをポーリングする。 3. **処理**: ワーカーは取得した conn_rec を処理し、完了後に done コールバックを呼び出す。 4. **終了**: h2_workers_join を呼び出すと、プロデューサーはアクティブリストから外れ、進行中の処理が完了するまで待機する。

直接依存

  • apr_thread_mutex.h, apr_thread_cond.h: スレッド同期用。
  • http_connection.h: conn_rec の定義に使用。
  • h2.h: HTTP/2 モジュール共通定義。

関連コード

  • h2_workers.c: 本ヘッダーで宣言された関数の実装およびスレッドのメインループ（slot_run）を含む。
  • h2_mplx.h: ワーカープールを利用してリクエストの多重化を行う上位層。

読者向けメモ

  • 本機能は、HTTP/2 のストリーム処理を効率化するために設計されています。
  • ap_conn_producer_t のコールバック関数は、スレッドセーフである必要があります。
  • h2_workers_create で指定する idle_limit は、リソースの節約とスレッド生成コストのバランスを決定する重要なパラメータです。
