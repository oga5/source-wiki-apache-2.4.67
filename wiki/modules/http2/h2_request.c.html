modules/http2/h2_request.c

概要

HTTP/2 のリクエスト表現 (h2_request) を Apache HTTP Server のネイティブなリクエスト構造体 (request_rec) へ変換・構築するためのブリッジ層です。HTTP/2 のストリームから受け取ったヘッダーや擬似ヘッダーを解析し、Apache のリクエスト処理パイプラインに適合する形式へ変換する役割を担います。

モジュール内での役割

mod_http2 において、HTTP/2 プロトコル層と Apache のコアリクエスト処理エンジンを接続する重要なコンポーネントです。HTTP/2 のストリーム状態を Apache の request_rec にマッピングすることで、既存のモジュール（認証、アクセス制御、コンテンツ生成など）が HTTP/2 リクエストを透過的に扱えるようにします。

責務

  • **リクエスト構築**: h2_request オブジェクトの生成、クローン、および初期化。
  • **ヘッダー変換**: HTTP/2 の擬似ヘッダー（:method, :path 等）を Apache の request_rec 構造体へマッピング。
  • **プロトコル適合**: HTTP/2 特有の制約（CONNECT メソッドの扱い、Host ヘッダーの補完）を Apache のリクエスト処理フローに統合。
  • **エラーハンドリング**: 不正なリクエストに対する ap_die を用いたエラー応答の生成と、リソースのクリーンアップ。

重要なシンボル

リクエスト生成・管理

  • h2_request_create: 新規 h2_request オブジェクトのメモリ確保と初期化。
  • h2_request_rcreate: 既存の request_rec から h2_request を生成する変換関数。
  • h2_request_clone: h2_request のディープコピーを作成。

ヘッダー処理

  • h2_request_add_header: HTTP/2 ヘッダーフレームの各行を処理し、擬似ヘッダーを解析・格納。
  • h2_request_end_headers: ヘッダー受信完了後の Host ヘッダー補完や検証を行う。

Apache 統合

  • h2_create_request_rec: h2_request を基に Apache の request_rec を構築し、リクエスト処理パイプラインへ投入する。
  • my_ap_create_request: Apache のバージョンが古い場合に、手動で request_rec を初期化するフォールバック関数。

制御フローとデータフロー

1. **初期化**: h2_request_create で構造体を確保し、h2_request_add_header で逐次ヘッダーを蓄積します。 2. **変換**: h2_create_request_rec が呼び出されると、ap_create_request を経由して request_rec を作成します。 3. **ヘッダー注入**: assign_headers が h2_request のヘッダーを request_rec->headers_in にコピーし、Host ヘッダーの整合性を調整します。 4. **パイプライン投入**: ap_run_pre_read_request や ap_post_read_request を呼び出し、Apache の標準的なリクエスト処理フックを起動します。 5. **終了処理**: エラー発生時は die ラベルへジャンプし、ap_die でエラー応答を生成後、EOR (End of Request) バケットを挿入してリクエストを終了します。

読者向けメモ

  • **メモリ管理**: すべてのオブジェクトは apr_pool_t に紐付いており、リクエスト終了時に自動的に解放されます。
  • **擬似ヘッダー**: HTTP/2 の擬似ヘッダーは h2_request 内で個別のフィールドとして保持され、request_rec 構築時に標準的な HTTP ヘッダーへ変換されます。
  • **互換性**: AP_MODULE_MAGIC_AT_LEAST マクロを使用して、Apache のバージョンによる ap_create_request の挙動の違いを吸収しています。
  • **制約**: h2_request_add_header は擬似ヘッダーがリクエストの先頭に来ることを期待しており、順序違反はエラーとして扱われます。
  • **省略箇所**: h2_request の内部構造体定義そのものは h2_request.h にあるため、本ファイルでは操作ロジックに集中しています。
