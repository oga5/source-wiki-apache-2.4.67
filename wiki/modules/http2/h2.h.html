modules/http2/h2.h

概要

h2.h は Apache HTTP Server の HTTP/2 モジュール (mod_http2) における中心的なヘッダーファイルです。HTTP/2 プロトコル仕様（RFC 7540）に基づいた定数、列挙型、およびモジュール全体で共有されるデータ構造の定義を保持しています。

モジュール内での役割

本ファイルは、mod_http2 の全コンポーネントに対する「共通言語」を提供します。セッション管理、ストリーム制御、プロトコル変換といった各サブシステムが依存する基盤定義を集中管理することで、モジュール全体の整合性を維持しています。

責務

  • **プロトコル定数の定義**: HTTP/2 エラーコード、擬似ヘッダー名、フレーム関連の定数を提供。
  • **ビルド時環境の抽象化**: APR や nghttp2 のバージョンに基づいた機能フラグ（WebSockets 対応、パイプ利用可否など）の判定。
  • **コアデータ構造の宣言**: セッション状態、ストリーム状態、リクエスト変換用構造体などの型定義。
  • **ユーティリティマクロ**: ストリーム ID 判定や最小/最大値計算などの汎用マクロの提供。

重要なシンボル

プロトコル定数

  • H2_ERR_*: RFC 7540 で定義された HTTP/2 エラーコード群。
  • H2_HEADER_*: :method, :path 等の HTTP/2 擬似ヘッダー名およびその長さ。
  • H2_FRAME_HDR_LEN: HTTP/2 フレームヘッダーの固定長（9バイト）。

セッション・ストリーム状態

  • h2_session_state: セッションのライフサイクル（INIT, BUSY, WAIT, CLEANUP 等）。
  • h2_stream_state_t: ストリームの RFC 準拠状態（IDLE, OPEN, CLOSED 等）。
  • h2_session_props: セッションの統計情報（ストリーム ID 管理、エラー状態）。

リクエスト変換

  • h2_request: HTTP/2 ストリームを HTTP/1.1 形式へ変換するための中間構造体。request_rec 生成前のメタデータを保持。

機能フラグ

  • H2_USE_PIPES, H2_USE_WEBSOCKETS: コンパイル環境の依存ライブラリに基づいた機能有効化フラグ。

制御フローとデータフロー

本ファイル自体はデータ構造と定数の定義のみであり、実行コードは含みません。

  • **データフロー**: h2_request 構造体が HTTP/2 フレームから抽出されたヘッダーを保持し、後続の mod_http2 フィルタチェーンを通じて request_rec へと変換されます。
  • **制御フロー**: h2_session_state や h2_stream_state_t は、h2_session.c や h2_stream.c における状態遷移マシンの基盤として利用されます。

直接依存

  • apr_version.h: APR バージョン判定用。
  • ap_mmn.h: Apache モジュールマジックナンバー判定用。
  • nghttp2/nghttp2ver.h: nghttp2 ライブラリのバージョン判定用。

関連コード

  • h2_private.h: モジュール内部でのみ使用される非公開定義。
  • h2_session.h / h2_stream.h: 本ファイルで定義された型を実際に操作する主要コンポーネント。
  • h2_util.h: 本ファイルの定数を利用したユーティリティ関数群。

読者向けメモ

  • **メモリ管理**: h2_request 構造体に含まれる apr_table_t 等は、通常 request_pool に紐付けられて管理されます。
  • **レイアウト**: H2_DATA_CHUNK_SIZE は TLS レコードサイズを考慮して計算されており、ネットワーク I/O の効率化に寄与しています。
  • **注意点**: H2_USE_WEBSOCKETS 等のフラグは、ビルド時の環境（APR/nghttp2 のバージョン）に強く依存します。環境構築時にはこれらの依存関係が満たされているか確認してください。
  • **省略範囲**: 本ファイルは定義のみであり、実装ロジックは含まれていません。詳細なロジックは各 .c ファイルを参照してください。
