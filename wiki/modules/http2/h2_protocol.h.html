modules/http2/h2_protocol.h

概要

HTTP/2 プロトコル仕様（RFC 7540）に基づいた、接続要件の検証およびプロトコル識別子を管理するためのヘッダーファイルです。

モジュール内での役割

mod_http2 におけるプロトコルレベルの制約定義と、接続が HTTP/2 を利用可能か判断するためのゲートキーパーの役割を果たします。

責務

  • **プロトコル識別子の定義**: TLS (ALPN) およびクリアテキスト (h2c) で使用される識別子の公開。
  • **エラーコードの可読化**: RFC 7540 で定義されたエラーコードに対する人間が読める説明文の提供。
  • **接続要件の検証**: 接続が HTTP/2 のセキュリティ要件（暗号スイートの制限など）を満たしているかの判定。

重要なシンボル

  • h2_protocol_ids_tls / h2_protocol_ids_clear: ALPN 等で使用されるプロトコル文字列の配列。
  • h2_protocol_err_description(): エラーコードから説明文字列を引くための関数。
  • h2_protocol_is_acceptable_c1(): 接続が HTTP/2 を許可する条件を満たしているかを確認する主要な判定関数。

制御フローとデータフロー

本ファイルは定義と宣言のみを提供します。実際の判定ロジックは h2_protocol.c に実装されており、接続確立時や ALPN ネゴシエーション時に呼び出されます。 1. 接続初期化時に h2_protocol_init が呼ばれる。 2. 接続のセキュリティ要件チェックが必要な際、h2_protocol_is_acceptable_c1 が conn_rec を引数として実行される。 3. エラー発生時には h2_protocol_err_description を通じてログ出力用の文字列が取得される。

直接依存

  • apr_pools.h: メモリプール管理用。
  • httpd.h / http_connection.h: Apache HTTP Server の接続構造体 (conn_rec) への依存。

関連コード

  • h2_protocol.c: 本ヘッダーの関数実装および RFC 7540 で禁止されている暗号スイートのブラックリスト定義が含まれます。
  • h2_session.h: プロトコルが受け入れられた後に生成される HTTP/2 セッションの管理。

読者向けメモ

  • 本ファイルは RFC 7540 の付録 A にある「禁止された暗号スイート」のチェックロジックの入り口です。
  • h2_protocol_is_acceptable_c1 の require_all 引数は、TLS ハンドシェイクが完了しているか（暗号スイートが確定しているか）を考慮するかどうかを制御します。ハンドシェイク途中の判定に使用する際は注意してください。
