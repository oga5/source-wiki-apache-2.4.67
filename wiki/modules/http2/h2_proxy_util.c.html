modules/http2/h2_proxy_util.c

概要

h2_proxy_util.c は、Apache HTTP Server の HTTP/2 プロキシ機能（mod_proxy_http2）で使用される汎用的なデータ構造とユーティリティ関数を提供します。主に、HTTP/2 フレームのデバッグ出力、整数識別子を持つ構造体のハッシュ管理、ソート済み整数キュー、および HTTP/1 と HTTP/2 間のヘッダー変換ロジックを担います。

モジュール内での役割

mod_proxy_http2 サブシステムにおける「基盤ユーティリティ層」です。ビジネスロジック（セッション管理やストリーム制御）から切り離された、再利用可能なデータ構造と変換処理を分離することで、コードの保守性を高めています。

責務

  • **データ構造の提供**: 整数 ID をキーとするハッシュテーブル（ihash）、ソート済み整数キュー（iqueue）、およびスレッドセーフな FIFO キュー（h2_proxy_fifo）の実装。
  • **ヘッダー変換**: HTTP/1 の apr_table_t と HTTP/2 の nghttp2_nv 間の相互変換、およびプロキシ特有のヘッダーフィルタリング。
  • **デバッグ支援**: nghttp2_frame の内容を人間が読める形式に変換するログ出力機能。
  • **リンクヘッダー処理**: RFC 5988 に基づく Link ヘッダーのパースと、リバースプロキシ環境下での URI 書き換え。

重要なシンボル

ハッシュ管理 (ihash)

  • h2_proxy_ihash_create: 構造体内の整数メンバーをキーとするハッシュを生成。
  • h2_proxy_ihash_add/remove: ハッシュへの要素追加・削除。
  • h2_proxy_ihash_shift: ハッシュから要素をバッファへ取り出し削除する（キュー的利用）。

キュー管理 (iqueue, fifo)

  • h2_proxy_iq_create: ソート順序を定義可能な整数キューの生成。
  • h2_proxy_iq_sort: 挿入ソートアルゴリズムを用いたキューの再整列。
  • h2_proxy_fifo_create: スレッドセーフな FIFO キュー。apr_thread_cond_t を用いたブロッキング操作をサポート。

ヘッダー変換

  • h2_proxy_util_nghd_make_req: h2_proxy_request から HTTP/2 用のヘッダーリストを生成。
  • h2_proxy_res_ignore_header: プロキシ転送時に無視すべきヘッダーの判定。

制御フローとデータフロー

  • **ハッシュ/キュー操作**: ihash は apr_hash_t をラップし、構造体のオフセットを利用してキーを抽出します。iqueue は動的配列を用い、iq_bubble_up/down を使ってソート状態を維持します。
  • **FIFO 処理**: h2_proxy_fifo はミューテックスと条件変数で保護されており、fifo_push/fifo_pull は block フラグに応じて待機または即時復帰します。
  • **リンクヘッダー解析**: h2_proxy_link_reverse_map は、文字列をスキャンする link_ctx を使用し、トークン単位でパースして URI を書き換える再帰的でないステートマシンとして動作します。

直接依存

  • apr_thread_mutex.h, apr_thread_cond.h: スレッドセーフなキューの実装に使用。
  • mod_proxy.h: プロキシ設定やリバースプロキシの URI マッピング機能を利用。
  • nghttp2/nghttp2.h: HTTP/2 フレーム構造体と定数定義。

読者向けメモ

  • **メモリ管理**: すべてのデータ構造は apr_pool_t を介して割り当てられます。ライフサイクルはプールに依存するため、明示的な free は不要ですが、fifo のように apr_pool_cleanup_register でクリーンアップが登録されているものもあります。
  • **パフォーマンス**: iqueue のソートは、変更が最小限であることを前提とした最適化が行われています。
  • **省略された詳細**: link_ctx 内のパースロジックは非常に詳細なステートマシンですが、RFC 5988 の仕様に厳密に従うため、複雑なネスト構造になっています。
  • **不変条件**: ihash を使用する際は、構造体内の整数 ID メンバーが変更されないことが前提です。変更が必要な場合は、一度 remove してから add する必要があります。
