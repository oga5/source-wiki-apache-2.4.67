modules/http2/h2_c1_io.h

概要

h2_c1_io.h は、HTTP/2 セッションにおけるクライアント接続（C1）の入出力処理を抽象化するコンポーネントです。Apache HTTP Server の接続フィルタスタックと、HTTP/2 プロトコル層の橋渡しを担います。

モジュール内での役割

mod_http2 における「C1（クライアント接続）」層の入出力管理を担当します。h2_session が管理する接続に対して、バッファリングされたデータ転送と、フィルタチェーンを通じた効率的な読み書きを提供します。

責務

  • **出力バッファリング**: apr_bucket_brigade を使用し、ネットワークへの書き込みを最適化するためのバッファ管理。
  • **フロー制御**: TLS レコードサイズや MTU を考慮した書き込みサイズの調整。
  • **フィルタ連携**: httpd の接続フィルタスタックに対するデータの投入とフラッシュ処理。
  • **入力処理**: クライアントからの入力を読み取り、HTTP/2 プロトコル層（nghttp2）へ渡すためのインターフェース提供。

重要なシンボル

  • h2_c1_io: 入出力状態を保持するメイン構造体。セッション情報、出力ブリゲード、バッファ統計、TLS 状態などを管理します。
  • h2_c1_io_add_data / h2_c1_io_add: 出力バッファへデータやバケットを追加する関数。
  • h2_c1_io_pass: バッファされたデータを接続フィルタへ送信（フラッシュ）する関数。
  • h2_c1_read: クライアントからの入力を読み取り、セッションへ供給する関数。

制御フローとデータフロー

1. **出力**: h2_c1_io_add 等でデータが h2_c1_io の出力ブリゲードに蓄積されます。h2_c1_io_pass が呼び出されると、蓄積されたデータが ap_pass_brigade を通じて下位の接続フィルタへ渡されます。 2. **入力**: h2_c1_read が呼び出されると、接続フィルタからデータを読み取り、HTTP/2 セッションの処理ロジックへ渡します。 3. **フラッシュ制御**: h2_c1_io_needs_flush や h2_c1_io_assure_flushed により、バッファの閾値やプロトコル上の必要性に応じて適切なタイミングでフラッシュが行われます。

直接依存

  • apr_bucket_brigade (APR): データ転送の基本単位。
  • h2_session (構造体宣言): 関連するセッションコンテキスト。
  • httpd.h 関連: Apache HTTP Server の接続およびフィルタ処理基盤。

関連コード

  • h2_c1_io.c: 本ヘッダーで定義された関数の実装。デバッグ用のログ出力関数 h2_c1_io_bb_log などが含まれます。
  • h2_session.h: この IO コンポーネントを所有し、セッション全体のライフサイクルを管理する上位モジュール。
  • h2_c1.h: クライアント接続の定義。

読者向けメモ

  • 本モジュールは、HTTP/2 のパフォーマンスを左右する「書き込みの集約」を担っています。WRITE_SIZE_MAX などの定数が TLS レコードサイズと密接に関係している点に注意してください。
  • h2_c1_io は h2_session と対になって動作します。デバッグ時には h2_c1_io_bb_log を活用して、ブリゲード内のバケット状態（EOS, FLUSH 等）を追跡することをお勧めします。
