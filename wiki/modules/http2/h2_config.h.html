modules/http2/h2_config.h

概要

h2_config.h は、Apache HTTP Server の mod_http2 モジュールにおける設定管理の基盤となるヘッダーファイルです。HTTP/2 固有の動作パラメータを定義し、サーバー全体（server_rec）およびディレクトリ単位（dir_config）での設定の保持、マージ、取得インターフェースを提供します。

モジュール内での役割

mod_http2 の設定システムの中核です。Apache の標準的な設定メカニズム（create_dir, merge_dir 等）を HTTP/2 の要件に合わせて拡張し、他のコンポーネントが実行時に必要な設定値へ安全にアクセスするための抽象化レイヤーとして機能します。

責務

  • **設定構造の定義**: HTTP/2 のストリーム数、ウィンドウサイズ、TLS 関連設定、プッシュ設定などのパラメータを保持するデータ構造の定義。
  • **設定の階層管理**: サーバーレベルとディレクトリレベルの設定を分離し、Apache の設定継承ルールに従ってマージする仕組みの提供。
  • **型安全なアクセス**: h2_config_var_t 列挙型を用いた、設定値取得のための統一的な API の提供。

重要なシンボル

  • h2_config_var_t: 設定項目を識別するための列挙型。H2_CONF_MAX_STREAMS や H2_CONF_PUSH など、モジュール全体で利用される設定キーを定義します。
  • h2_push_res: HTTP/2 Server Push を構成するためのリソース定義構造体。
  • h2_config_rgeti / h2_config_cgeti: それぞれリクエストおよび接続コンテキストから、指定した設定値を取得するためのアクセサ関数。

制御フローとデータフロー

1. **初期化**: h2_config_init がモジュールロード時に呼ばれ、設定の初期状態を準備します。 2. **設定読み込み**: Apache の設定解析フェーズで h2_cmds を通じて値が設定構造体に格納されます。 3. **マージ**: h2_config_merge_dir 等により、親ディレクトリの設定が子ディレクトリへ継承されます。 4. **取得**: 実行時、各コンポーネント（セッションやストリーム処理）が h2_config_rgeti などの関数を呼び出し、現在のリクエスト/接続に適用される設定値を取得します。

直接依存

本ファイルは純粋なインターフェース定義であり、直接的なソースコード依存（#include）は持っていません。ただし、Apache HTTP Server のコア構造体（server_rec, request_rec, conn_rec）および APR（Apache Portable Runtime）の型定義に依存しています。

関連コード

  • h2_config.c: 本ヘッダーで宣言された関数の実装および、設定構造体の実体定義が含まれます。
  • h2_session.c: セッション開始時に h2_config_cgeti を通じて接続ごとの設定値を取得し、動作を決定します。
  • h2_push.c: h2_push_res 構造体を利用して、プッシュ対象リソースの管理を行います。

読者向けメモ

  • 本ファイルは設定の「定義」です。設定値のデフォルト値や具体的なロジックは h2_config.c にあります。
  • 設定値を取得する際は、可能な限り rgeti（リクエスト単位）や cgeti（接続単位）を使用してください。これらは、サーバー全体の設定をベースにしつつ、特定のスコープで上書きされた値を正しく解決します。
  • h2_config_var_t に新しい設定項目を追加した場合は、必ず h2_config.c 側のアクセサ関数にもロジックを追加する必要があります。
