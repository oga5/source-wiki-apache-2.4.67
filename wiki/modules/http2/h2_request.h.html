modules/http2/h2_request.h

概要

h2_request.h は、HTTP/2 プロトコルにおけるリクエスト表現を定義し、Apache HTTP Server の内部構造（request_rec）との相互変換を行うためのインターフェースを提供します。

モジュール内での役割

mod_http2 において、HTTP/2 ストリーム上のリクエストを抽象化する中心的な役割を担います。HTTP/2 のフレームから抽出されたヘッダーやメタデータを、Apache のリクエスト処理パイプラインで扱える形式に橋渡しします。

責務

  • **リクエストの生成と管理**: HTTP/2 ストリームから h2_request 構造体を生成・複製する。
  • **ヘッダー/トレーラーの構築**: HTTP/2 のヘッダー圧縮（HPACK）を考慮しつつ、リクエストヘッダーおよびトレーラーを構築する。
  • **Apache 内部形式への変換**: h2_request を Apache の標準的な request_rec に変換し、サーバーのフィルタチェーンへ渡す準備を行う。

重要なシンボル

  • h2_request: HTTP/2 リクエストを保持する主要な構造体。
  • h2_request_create(): 新規リクエストオブジェクトの初期化。
  • h2_request_rcreate(): 既存の request_rec から h2_request を生成する（プロキシ処理等で使用）。
  • h2_create_request_rec(): h2_request を Apache の request_rec に変換する。
  • h2_request_add_header(): リクエストヘッダーの追加とバリデーション。

制御フローとデータフロー

1. **受信**: HTTP/2 セッションがフレームを受信すると、h2_request_create を通じてリクエストオブジェクトが生成されます。 2. **構築**: h2_request_add_header を繰り返し呼び出し、HPACK でデコードされたヘッダーをテーブルに格納します。 3. **変換**: リクエスト処理の開始時、h2_create_request_rec が呼び出され、h2_request の情報が Apache の request_rec にコピーされます。これにより、以降の Apache フィルタチェーンは HTTP/2 特有の処理を意識せずにリクエストを処理できます。

直接依存

  • h2.h: 基本的な型定義や定数。
  • apr_pools.h (APR): メモリ管理。
  • httpd.h / http_request.h: Apache のコアリクエスト構造体。

関連コード

  • h2_request.c: 上記関数の実装。ヘッダーのパースや URI の正規化ロジックが含まれます。
  • h2_stream.h: h2_request を保持し、ストリームのライフサイクルを管理する上位層。
  • h2_headers.h: ヘッダー処理の補助。

読者向けメモ

  • このファイルは、HTTP/2 の「ストリーム」と Apache の「リクエスト」という異なる抽象化レイヤーを繋ぐ重要なインターフェースです。
  • h2_hd_scratch 構造体は、ヘッダー処理中のメモリ効率を最適化するためのスクラッチパッドとして使用されます。
  • h2_request_rcreate は、主にプロキシモジュール（mod_proxy_http2）がリクエストを転送する際に使用されます。
