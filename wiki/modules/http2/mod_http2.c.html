modules/http2/mod_http2.c

概要

mod_http2.c は、Apache HTTP Server における HTTP/2 プロトコル実装のメインエントリポイントです。Apache モジュールとしてのライフサイクル管理、設定の初期化、および HTTP/2 関連の環境変数提供を担います。

モジュール内での役割

本ファイルは mod_http2 モジュールの「顔」であり、Apache のフックシステムと modules/http2 内の各サブコンポーネント（h2_session, h2_c1, h2_c2 等）を接続する役割を果たします。

責務

  • **モジュールライフサイクル管理**: post_config および child_init フックを通じた初期化。
  • **設定管理**: ディレクトリおよびサーバー単位の設定作成・マージの委譲。
  • **環境変数提供**: http2_var_lookup を通じた HTTP/2 関連情報（ストリームID、PUSH状態など）の公開。
  • **機能の統合**: h2_c1（クライアント接続）、h2_c2（バックエンド接続）、h2_switch（プロトコル切り替え）のフック登録。

重要なシンボル

モジュール定義

  • http2: module 構造体。Apache がモジュールを認識するための中心的な定義。

初期化・ライフサイクル

  • h2_post_config: サーバー起動時の初期化。nghttp2 のバージョン確認や MPM の互換性チェックを行う。
  • h2_child_init: 子プロセス生成時の初期化。h2_c1 および h2_c2 の初期化をトリガーする。
  • h2_hooks: Apache フックの登録関数。

環境変数ルックアップ（H2_VARS ファミリー）

  • http2_var_lookup: mod_rewrite 等から呼ばれる環境変数取得用関数。
  • val_H2_PUSH, val_H2_STREAM_ID 等: 各種 HTTP/2 関連変数の値を計算するヘルパー関数群。

制御フローとデータフロー

1. **初期化フロー**: h2_post_config が呼ばれ、nghttp2 の機能フラグ（myfeats）をセットし、h2_protocol_init と h2_switch_init を実行してプロトコルスタックを準備します。 2. **接続フロー**: h2_child_init が呼ばれると、h2_c1_child_init（フロントエンド）と h2_c2_child_init（バックエンド）が順次呼び出され、接続処理の準備が完了します。 3. **変数解決フロー**: h2_h2_fixups フックがリクエスト処理中に呼ばれ、H2_VARS 配列を走査して subprocess_env に HTTP/2 関連情報を注入します。

読者向けメモ

  • **メモリ管理**: apr_pool_t を使用したメモリ管理が行われています。h2_post_config は Apache の仕様上、設定確認のための「ドライラン」と「本番」の2回呼ばれる可能性があるため、apr_pool_userdata_get を用いて初期化の重複を防いでいます。
  • **設計の意図**: 本ファイルはロジックの集約場所ではなく、あくまで Apache のフックを各サブモジュールに委譲する「ハブ」として設計されています。
  • **省略された詳細**: h2_req_engine 関連のコードは、現在非推奨（deprecated）であるため、本ページでは詳細な解析を省略しています。
  • **注意点**: h2_c1（クライアント側）と h2_c2（サーバー側/バックエンド側）の初期化順序は、接続処理の安定性に直結するため重要です。
