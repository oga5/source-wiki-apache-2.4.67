modules/http2/h2_ws.h

概要

HTTP/2 上での WebSocket 通信（RFC 8441: Bootstrapping WebSockets with HTTP/2）をサポートするためのヘッダーファイルです。HTTP/2 の拡張 CONNECT メソッドを、Apache HTTP Server 内部で処理可能な標準的な WebSocket リクエスト形式へ変換するインターフェースを定義します。

モジュール内での役割

modules/http2 サブシステムにおいて、HTTP/2 ストリーム上で WebSocket を確立するためのブリッジ層として機能します。HTTP/2 固有のプロトコル変換を抽象化し、サーバーのコア機能が WebSocket リクエストを透過的に扱えるようにします。

責務

  • **リクエスト変換**: HTTP/2 の拡張 CONNECT メソッドを、HTTP/1.1 互換の GET リクエスト（Upgrade ヘッダー付き）へ書き換える。
  • **フック登録**: WebSocket 処理に必要な Apache フィルタやハンドラをシステムに登録する。

重要なシンボル

  • h2_ws_rewrite_request: HTTP/2 のリクエスト構造体を WebSocket 用に変換するメイン関数。
  • h2_ws_register_hooks: WebSocket 関連のフィルタや処理を Apache のリクエスト処理パイプラインに組み込むための関数。

制御フローとデータフロー

1. **リクエスト受信**: HTTP/2 セッションが CONNECT メソッドかつ protocol: websocket を含むリクエストを受信します。 2. **変換処理**: h2_ws_rewrite_request が呼び出され、リクエストメソッドを GET に変更し、必要な Upgrade および Connection ヘッダーを付与します。 3. **内部処理**: 変換されたリクエストは、通常の HTTP/1.1 WebSocket ハンドラと同様のパスを通り、サーバー内部で処理されます。 4. **レスポンス生成**: h2_ws.c 側の実装により、Sec-WebSocket-Accept ヘッダーの生成とレスポンスのフィルタリングが行われます。

直接依存

  • h2.h: HTTP/2 の基本定義および型定義。

関連コード

  • modules/http2/h2_ws.c: 上記関数の実装および、WebSocket フィルタ (ws_filter_ctx) の定義。
  • modules/http2/h2_request.h: h2_request 構造体の定義。
  • modules/http2/h2_conn_ctx.h: ストリームごとの接続コンテキスト管理。

読者向けメモ

  • 本機能は H2_USE_WEBSOCKETS マクロが有効な場合にのみコンパイルされます。
  • RFC 8441 に基づく実装であるため、HTTP/2 のストリーム多重化を維持したまま WebSocket をトンネリングする仕組みを理解する上で重要です。
  • h2_ws_rewrite_request は、元のリクエストを破壊せず、新しいリクエスト構造体を生成して返す設計になっています。
