modules/http2/h2_bucket_eos.c

概要

HTTP/2 ストリームの終了（EOS: End of Stream）を表現するためのカスタム APR バケット型を定義するファイルです。

モジュール内での役割

mod_http2 におけるストリーム処理において、データストリームの終端を明示的に通知するためのメタデータバケットを提供します。これは、HTTP/2 のストリーム状態遷移を Apache HTTP Server のバケットチェーン上で管理するために不可欠なコンポーネントです。

責務

  • **EOS バケットの生成と管理**: h2_stream オブジェクトと紐付いたカスタムバケットの作成。
  • **ライフサイクル同期**: ストリームの終了イベント（H2_SEV_EOS_SENT）を、バケットの破棄タイミングに合わせてディスパッチする。
  • **リソース保護**: ストリームのプールとバケットの生存期間を調整し、二重解放や不正なアクセスを防ぐ。

重要なシンボル

  • h2_bucket_type_eos: このバケットの挙動を定義する apr_bucket_type_t 構造体。
  • h2_bucket_eos_create(): 新しい EOS バケットを生成し、ストリームのクリーンアップ処理を登録する公開関数。
  • bucket_destroy(): バケット破棄時にストリームへ EOS 送信イベントを通知する内部関数。

制御フローとデータフロー

1. **生成**: h2_bucket_eos_create が呼ばれると、バケットが割り当てられ、ストリームのプールに bucket_cleanup が登録されます。 2. **伝搬**: バケットチェーンを EOS バケットが通過します。bucket_read はメタデータバケットであるため、データは返さず成功を返します。 3. **破棄**: バケットが解放される際、bucket_destroy が実行されます。

  • ストリームがまだ存在する場合、h2_stream_dispatch を通じて H2_SEV_EOS_SENT イベントが発行されます。
  • apr_pool_cleanup_kill により、プール側のクリーンアップ登録が解除され、二重処理を防止します。

読者向けメモ

  • このバケットは「メタデータ」として扱われます。bucket_read が呼び出されても、実際のデータは読み取られません。
  • ストリームの生存期間とバケットの生存期間が異なる可能性があるため、bucket_cleanup と bucket_destroy の両方でストリームのポインタを安全に管理する設計になっています。
  • H2_SEV_EOS_SENT イベントは、ストリームの終了処理をトリガーする重要なシグナルであることに注意してください。
