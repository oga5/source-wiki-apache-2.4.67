modules/core/mod_macro.c

概要

Apache HTTP Server の設定ファイル内でマクロ定義と展開を可能にするモジュールです。設定の再利用性を高め、冗長な設定記述を削減します。設定読み込み時（EXEC_ON_READ）に即時処理されるため、サーバー起動時や設定再読み込み時にマクロが展開されます。

モジュール内での役割

modules/core に属し、Apache の設定パーサーを拡張するユーティリティとして機能します。通常のモジュールとは異なり、リクエスト処理のフックは持たず、設定ファイルの解析フェーズに特化しています。

責務

  • **マクロ定義の管理**: <Macro> セクションを解析し、名前、引数リスト、本体を ap_macro_t 構造体としてグローバルなハッシュテーブルに保持する。
  • **マクロ展開**: Use ディレクティブを検出し、引数を置換した上で、その内容を一時的な設定ストリームとしてパーサーに注入する。
  • **構文検証**: 引数の重複チェック、未定義マクロの検出、再帰呼び出しの防止、およびマクロ本体の構文整合性を検証する。
  • **設定ストリームの制御**: ap_configfile_t を拡張し、マクロ展開後の内容を元の設定ファイルの一部としてパーサーに認識させる。

重要なシンボル

マクロ定義と管理

  • ap_macro_t: マクロの定義を保持する構造体。名前、引数名配列、本体行配列、定義場所（エラーメッセージ用）を保持。
  • ap_macros: マクロ定義を格納するグローバルな apr_hash_t。

設定コマンド (ディレクティブ)

  • macro_section: <Macro> を処理。定義の解析と検証を行う。
  • use_macro: Use を処理。引数置換を行い、展開後の内容をパーサーへ注入する。
  • undef_macro: UndefMacro を処理。ハッシュテーブルから定義を削除する。

設定ストリーム操作

  • array_contents_t: マクロ展開後の内容を保持し、パーサーが読み取れるようにするコンテキスト構造体。
  • make_array_config: array_contents_t を ap_configfile_t としてラップし、パーサーに提供する。

制御フローとデータフロー

1. **定義時**: macro_section が設定ファイルを読み込み、get_lines_till_end_token を用いて </Macro> までの行を ap_macro_t に格納する。 2. **展開時**: use_macro が Use を見つけると、process_content を呼び出す。 3. **置換**: process_content は内部で substitute_macro_args を呼び出し、各行に対して引数（$, %, @ プレフィックス付き）を置換する。 4. **注入**: 置換後の内容は make_array_config により ap_configfile_t 互換のストリームに変換され、現在の設定ファイル読み込み処理に割り込まれる。これにより、パーサーはマクロ展開後の内容を「元のファイルの一部」として読み続ける。

直接依存

  • httpd.h, http_config.h, http_log.h: Apache コア API。
  • apr.h, apr_strings.h, apr_hash.h: APR (Apache Portable Runtime) の文字列操作およびハッシュテーブル。

関連コード

  • server/config.c: Apache の設定ファイル解析のメインロジック。本モジュールはここでの解析処理に介入する。
  • include/http_config.h: ap_configfile_t や cmd_parms などの定義元。

読者向けメモ

  • **メモリ管理**: マクロ定義はグローバルな ap_macros ハッシュに保持されます。pre_config フックで初期化・リセットされる設計です。
  • **制約**: マクロの再帰的な使用は、設定ファイル名にマクロ名が含まれるかをチェックすることで簡易的に禁止されています。
  • **エッジケース**: マクロ引数に @ を使用すると、置換時にダブルクォートでエスケープされます。
  • **未実装/浅い部分**: get_lines_till_end_token は行ベースの解析に依存しており、複雑なネスト構造の解析には限界があります。また、エラーメッセージの生成には cmd->temp_pool を使用しており、メモリリークを防ぐためのライフサイクル管理に注意が必要です。
