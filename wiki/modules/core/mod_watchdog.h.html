modules/core/mod_watchdog.h

概要

mod_watchdog.h は、Apache HTTP Server におけるバックグラウンドタスク実行のための「ウォッチドッグ」機能を提供するヘッダーファイルです。定期的な処理や、特定のプロセス状態（開始・実行中・停止）に応じたコールバック実行を可能にするためのインターフェースを定義しています。

モジュール内での役割

modules/core 内で、他のモジュールが非同期または定期的なタスクを安全に実行するための共通基盤として機能します。MPM（Multi-Processing Module）環境下で、プロセスやスレッドを跨いだタスク管理の抽象化レイヤーを提供します。

責務

  • ウォッチドッグインスタンスの取得および管理インターフェースの提供。
  • 定期実行されるコールバック関数の登録・更新機能の定義。
  • プロセスライフサイクル（開始・停止）に連動したフックポイントの提供。
  • シングルトン（全プロセスで単一の実行）およびデフォルト（プロセス毎）の実行モードの定義。

重要なシンボル

  • ap_watchdog_t: ウォッチドッグインスタンスを保持する不透明な構造体。
  • ap_watchdog_callback_fn_t: 定期実行されるコールバック関数の型定義。
  • AP_WATCHDOG_STATE_*: コールバックに渡される現在の状態（STARTING, RUNNING, STOPPING）。
  • ap_watchdog_get_instance: ウォッチドッグインスタンスを取得するためのオプション関数。
  • ap_watchdog_register_callback: 特定のインターバルで実行される関数を登録するオプション関数。

制御フローとデータフロー

1. **初期化**: モジュールは ap_watchdog_get_instance を呼び出し、名前を指定してインスタンスを取得します。 2. **登録**: ap_watchdog_register_callback を使用して、実行間隔とコールバック関数を登録します。 3. **実行**: ウォッチドッグスレッドが内部タイマーに従い、登録されたコールバックを AP_WATCHDOG_STATE_RUNNING 状態で呼び出します。 4. **終了**: プロセス終了時、AP_WATCHDOG_STATE_STOPPING 状態がコールバックに通知され、クリーンアップが行われます。

読者向けメモ

  • 本モジュールは APR_DECLARE_OPTIONAL_FN を多用しています。これは、mod_watchdog がロードされていない環境でも他のモジュールがリンクエラーを起こさずに安全に呼び出せるようにするための設計です。
  • シングルトンモードを使用する場合、内部でプロセス間ミューテックスが使用されます。デッドロックを避けるため、コールバック内での重い処理や長時間のロック保持は推奨されません。
  • watchdog_step フックは、特定のインスタンスに依存しない汎用的な定期処理を実装する際に有用です。
