modules/core/mod_so.h

概要

mod_so.h は、Apache HTTP Server における動的共有オブジェクト（DSO）読み込み機能を提供する mod_so モジュールの公開インターフェースです。このヘッダーは、他のモジュールが実行時にロードされたモジュールのシンボルを検索するための仕組みを定義しています。

モジュール内での役割

mod_so は Apache のコア機能の一部であり、サーバーを再起動することなくモジュールを追加・削除するための基盤です。本ヘッダーは、その機能のうち「外部から利用可能な関数」を公開する役割を担います。

責務

  • mod_so が提供するオプション関数の宣言。
  • 実行時にロードされたモジュール構造体へのアクセス手段の提供。

重要なシンボル

  • MOD_SO_H: ヘッダーの多重インクルードを防ぐためのガードマクロ。
  • ap_find_loaded_module_symbol: ロード済みのモジュールから特定のシンボル（モジュール構造体）を検索するためのオプション関数。

制御フローとデータフロー

本ファイル自体は宣言のみを提供します。 1. mod_so.c が APR_REGISTER_OPTIONAL_FN を使用して ap_find_loaded_module_symbol を登録します。 2. 他のモジュールは APR_RETRIEVE_OPTIONAL_FN を使用してこの関数を取得し、実行時にロードされたモジュールの情報を動的に解決します。

直接依存

  • apr_optional.h: Apache Portable Runtime (APR) のオプション関数機能を利用するために必要。
  • httpd.h: Apache の基本データ構造（server_rec や module 型）の定義を参照するために必要。

関連コード

  • modules/core/mod_so.c: 本ヘッダーで宣言された関数の実装および DSO ロードのメインロジックが含まれます。
  • include/http_config.h: Apache のモジュール構造体定義の基点であり、mod_so が操作する対象です。

読者向けメモ

  • このヘッダーは非常に軽量であり、主に他のモジュールが mod_so の機能に依存するためのブリッジとして機能します。
  • ap_find_loaded_module_symbol を使用する際は、関数が NULL でないことを確認する（APR_RETRIEVE_OPTIONAL_FN の結果をチェックする）ことが推奨されます。これは、mod_so がロードされていない環境でも安全に動作させるためです。
