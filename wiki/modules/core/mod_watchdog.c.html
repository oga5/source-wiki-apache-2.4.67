modules/core/mod_watchdog.c

概要

Apache HTTP Server におけるバックグラウンド・タスク実行のための監視（Watchdog）フレームワークを提供します。定期的なコールバック実行や、プロセス間排他制御を伴うシングルトン・タスクの管理を担います。

モジュール内での役割

mod_watchdog は、他のモジュールが非同期かつ定期的な処理を安全に実行するためのインフラストラクチャとして機能します。ap_provider を介してインスタンスを公開し、MPM（Multi-Processing Module）のライフサイクルと同期して動作します。

責務

  • **監視スレッド管理**: 各 Watchdog インスタンス専用のワーカースレッドの生成と終了処理。
  • **コールバック実行**: 登録されたコールバック関数を STARTING, RUNNING, STOPPING の各状態で呼び出し。
  • **排他制御**: singleton モードにおけるプロセス間排他制御（apr_proc_mutex）の管理。
  • **設定管理**: WatchdogInterval ディレクティブによるグローバルな実行間隔の制御。
  • **フック提供**: watchdog_init, watchdog_step, watchdog_exit などの外部フックの実行。

重要なシンボル

データ構造

  • ap_watchdog_t: 監視インスタンスのメイン構造体。スレッド、mutex、コールバックリストを保持。
  • watchdog_list_t: 登録されたコールバック関数のリンクリスト。実行間隔や状態を管理。
  • wd_server_conf_t: モジュール全体のサーバー設定（プール、サーバーレコード）。

ライフサイクル・管理関数

  • wd_worker: スレッドのメインループ。コールバックの実行タイミングと終了判定を制御。
  • wd_startup: スレッドの生成とクリーンアップハンドラの登録。
  • ap_watchdog_get_instance: プロバイダ経由で Watchdog インスタンスを取得または生成。
  • ap_watchdog_register_callback: 特定の Watchdog にコールバックを登録。

制御フローとデータフロー

1. **初期化**: wd_pre_config_hook でデフォルトの Watchdog インスタンスを生成。wd_post_config_hook で設定に基づきスレッドを起動。 2. **スレッド実行**: wd_worker が起動し、apr_sleep を用いて AP_WD_TM_SLICE（100ms）ごとにループ。 3. **コールバック処理**: watchdog_list_t を走査し、経過時間が interval を超えた場合にコールバックを実行。 4. **終了処理**: wd_worker_cleanup がプール破棄時に呼ばれ、STOPPING 状態を通知してスレッドを join。

直接依存

  • apr.h, apr_atomic.h: アトミック操作と基本型。
  • mod_watchdog.h: 公開インターフェース定義。
  • ap_mpm.h: MPM 状態のクエリ（停止判定など）。
  • util_mutex.h: プロセス間排他制御の抽象化。
  • ap_provider.h
  • http_core.h

関連コード

  • modules/core/mod_watchdog.h: このモジュールの API 定義。
  • server/mpm_common.c: MPM 関連のフックやクエリの参照先。

読者向けメモ

  • **メモリ所有権**: ap_watchdog_t および watchdog_list_t は、登録時に指定されたプール（通常は pconf）のライフサイクルに従います。
  • **シングルトン制約**: singleton モードの Watchdog は、apr_proc_mutex を使用して全子プロセス中で唯一の実行を保証します。mutex のロック取得に失敗したプロセスは待機します。
  • **注意点**: コールバック関数内で長時間ブロックすると、その Watchdog インスタンス全体の遅延を招きます。
  • **省略箇所**: ap_run_watchdog_* フックの具体的な実装例は、このファイルではなく各利用モジュール側に存在するため、本ファイルでは定義のみを行っています。
