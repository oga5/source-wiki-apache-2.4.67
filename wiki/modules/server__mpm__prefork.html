server/mpm/prefork

システム内での役割

server/mpm/prefork は、Apache HTTP Server における Prefork MPM（Multi-Processing Module）の実装ディレクトリです。各リクエストを単一のプロセスで処理する非スレッド型のモデルを提供し、スレッドセーフではないライブラリとの互換性を維持しつつ、あらかじめ子プロセスを生成（プリフォーク）することでリクエストに対する応答性を高めます。

責務

  • **プロセス管理**: 子プロセスの生成 (fork)、死活監視、およびシグナルによるライフサイクル制御。
  • **動的スケーリング**: MinSpareServers / MaxSpareServers に基づくアイドルプロセスの監視と調整。
  • **接続処理**: accept() ミューテックスによる排他制御と、リスニングソケットのポーリング。
  • **設定管理**: プロセス数上限（ServerLimit 等）の検証と適用。

このモジュールのファイル

  • server/mpm/prefork/mpm_default.h: プロセス管理に関するデフォルトの閾値定義。
  • server/mpm/prefork/prefork.c: プロセス管理ロジックおよび MPM のメイン実装。

主要なデータ構造

  • prefork_retained_data: 再起動を跨いで保持される、世代管理やスケーリング状態を格納する構造体。
  • prefork_child_bucket: リスナー、パイプ（POD: Pipe of Death）、ミューテックスを保持する管理構造体。

主なフロー

1. **初期化**: prefork_pre_config にて設定の初期化と retained データの確保を行う。 2. **起動**: prefork_run がメインループを開始し、startup_children で初期プロセスを生成。 3. **監視**: 親プロセスが perform_idle_server_maintenance を通じて負荷に応じたプロセス増減を定期的に実行。 4. **接続処理**: 子プロセスが child_main 内で accept() を実行し、リクエストを処理。 5. **終了**: シグナル（SIGTERM 等）を受け取ると、POD を通じて子プロセスへ通知し、クリーンアップを行う。

外部依存

  • **APR (Apache Portable Runtime)**: apr.h, apr_thread_proc.h, apr_signal.h 等によるポータブルなプロセス・メモリ管理。
  • **Apache Core**: httpd.h, ap_config.h, scoreboard.h 等による MPM 共通インターフェースおよび共有メモリ管理。
  • **Unix 固有**: unixd.h による権限管理やシグナル処理。

内部構成

  • 子モジュール数: 0（ディレクトリ境界に基づき、単一の MPM 実装として構成）。

推奨読書順

1. server/mpm/prefork/mpm_default.h: プロセス管理の定数定義を把握。 2. server/mpm/prefork/prefork.c: プロセス管理のメインループとライフサイクル管理の実装を精読。

未解決の問い
