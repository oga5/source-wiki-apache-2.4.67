modules/http/http_etag.c

概要

HTTP プロトコルにおける ETag ヘッダーの生成と管理を担うモジュールです。ファイルシステム上の属性（inode, サイズ, 更新日時）や、ファイル内容の SHA1 ハッシュに基づいた一意な識別子を生成し、キャッシュの検証をサポートします。

モジュール内での役割

modules/http サブシステムの一部として、リクエスト処理の最終段階でレスポンスヘッダーに ETag を付与する役割を持ちます。http_protocol.c 等から呼び出され、リソースの同一性を保証するための計算ロジックを提供します。

責務

  • **ETag 生成ロジック**: ファイル属性（finfo）または内容のハッシュ（SHA1）を用いた文字列生成。
  • **検証情報の統合**: コンテンツネゴシエーション時に使用される vlist_validator (VLV) との結合。
  • **設定に基づく制御**: FileETag ディレクティブの設定（etag_bits）に従った、ETag に含める情報のフィルタリング。
  • **ヘッダー管理**: 生成された ETag を r->headers_out テーブルへ設定、または生成不可時のフラグ設定。

重要なシンボル

公開 API (ETag 生成・設定)

  • ap_make_etag_ex: etag_rec 構造体を受け取り、詳細な設定に基づいて ETag 文字列を生成するコア関数。
  • ap_make_etag: request_rec から簡易的に ETag を生成するラッパー。
  • ap_set_etag: リクエストのファイル情報から ETag を生成し、レスポンスヘッダーに自動設定する。
  • ap_set_etag_fd: ファイル記述子（apr_file_t）を直接指定して ETag を生成・設定する。

内部ヘルパー

  • make_digest_etag: ファイル内容を読み込み、SHA1 ハッシュによる強力な ETag を生成する。
  • etag_uint64_to_hex: apr_uint64_t を高速に 16 進数文字列へ変換する。
  • etag_start / etag_end: ETag 文字列のフォーマット（W/ プレフィックスや引用符の付与）を管理する。

制御フローとデータフロー

1. **入力**: ap_set_etag 等が request_rec から etag_rec を構築し、ap_make_etag_ex を呼び出します。 2. **設定評価**: core_dir_config から etag_bits を取得し、どの属性を含めるかを決定します。 3. **分岐**:

  • ETAG_DIGEST 指定時または強力な ETag が必要な場合: make_digest_etag がファイルを読み込み、apr_sha1 でハッシュ化します。
  • 通常時: ファイルの inode, size, mtime を連結し、etag_uint64_to_hex で文字列化します。

    • 4. **VLV 統合**: r->vlist_validator が存在する場合、ETag の末尾に結合し、ネゴシエーション情報を保持します。 5. **出力**: 生成された文字列を apr_table_setn で headers_out に格納します。

読者向けメモ

  • **メモリ管理**: すべての ETag 文字列は r->pool から割り当てられ、リクエスト終了時に自動解放されます。
  • **パフォーマンス**: make_digest_etag はファイル全体を読み込むため、大きなファイルに対してはコストが高い操作です。
  • **弱 ETag**: 更新から 1 秒以内のファイルや、VLV が弱い場合は W/ プレフィックスが付与されます。
  • **省略された詳細**: apr_mmap に関する最適化パスは、APR_HAS_MMAP が有効な環境でのみ動作するよう条件分岐されています。
  • **不変条件**: etag_bits が ETAG_NONE の場合、ETag は生成されず、no-etag ノートが設定されます。これは後続のフィルターが ETag を付与しないようにするための制御です。
