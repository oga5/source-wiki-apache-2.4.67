modules/http/http_request.c

概要

http_request.c は、Apache HTTP Server におけるリクエスト処理のメインラインを担う中核ファイルです。リクエストが読み込まれた後、適切なディレクトリ設定の適用、モジュールフックの実行、内部リダイレクト、およびエラー処理を統括します。

モジュール内での役割

modules/http サブシステムにおいて、リクエストのライフサイクル管理（非同期処理、サブクエスト、リダイレクト）を抽象化する役割を持ちます。http_protocol.c が低レベルなプロトコル解析を行うのに対し、本ファイルはリクエストの論理的な実行フローを制御します。

責務

  • **リクエスト実行制御**: メインリクエストおよび非同期リクエストの実行フロー制御。
  • **内部リダイレクト**: internal_redirect 等によるリクエストの再ルーティングとコンテキストの引き継ぎ。
  • **エラーハンドリング**: ap_die を通じたエラーレスポンスの生成と、カスタムエラードキュメントへのリダイレクト。
  • **パイプライン管理**: HTTP Keep-Alive 接続におけるパイプライン化されたリクエストのチェックとブランク行の消費。
  • **メソッド管理**: リクエストが許可する HTTP メソッドの動的な設定。

重要なシンボル

リクエスト処理のメインエントリ

  • ap_process_request: リクエスト処理の標準的なエントリポイント。
  • ap_process_async_request: 非同期リクエスト処理のコアロジック。
  • ap_process_request_after_handler: ハンドラ実行後の後処理（EORバケットの送信、パイプラインチェック）。

リダイレクト・サブクエスト

  • ap_internal_redirect: 内部リダイレクトを実行し、新しいリクエストコンテキストを構築。
  • ap_internal_fast_redirect: 既存のリクエスト構造を再利用してリダイレクトを行う最適化版。
  • internal_internal_redirect: リダイレクト時に新しい request_rec を生成する内部ヘルパー。

エラー・状態管理

  • ap_die: リクエスト処理中のエラー発生時に呼び出される汎用エラーハンドラ。
  • ap_check_pipeline: 接続上のパイプライン化された空行を消費し、Keep-Alive の継続可否を判定。

メソッド制御

  • ap_allow_methods / ap_allow_standard_methods: request_rec に許可されたメソッドリストを更新するユーティリティ。

制御フローとデータフロー

1. **リクエスト開始**: ap_process_async_request が呼び出され、quick_handler フックを試行後、ap_process_request_internal で設定適用とアクセスチェックを行います。 2. **ハンドラ実行**: ap_invoke_handler により、URI に対応するコンテンツハンドラが実行されます。 3. **後処理**: ap_process_request_after_handler が EOR (End of Request) バケットをフィルタチェーンに流し、リクエストプールの解放とパイプラインの確認を行います。 4. **リダイレクト**: internal_internal_redirect は、元のリクエストから設定やフィルタチェーンを継承しつつ、新しい URI 用の request_rec を生成します。この際、update_r_in_filters を用いてフィルタチェーン内の request_rec ポインタを新しいものに更新します。

読者向けメモ

  • **メモリ管理**: request_rec のライフサイクルは r->pool に依存します。内部リダイレクト時には apr_pool_join や apr_pcalloc を用いてメモリの生存期間を慎重に管理しています。
  • **フィルタチェーン**: update_r_in_filters は、リダイレクト時にフィルタが古い request_rec を参照し続けないようにするための重要な保守関数です。
  • **未実装・浅い部分**: ap_internal_fast_redirect は非常に脆弱であるとコメントされており、使用には注意が必要です。また、http_protocol.c との責務分担が曖昧な箇所（特にヘッダー設定など）がコード内に散見されます。
