modules/http/http_core.c

概要

http_core.c は、Apache HTTP Server における HTTP プロトコル処理の基盤となるモジュールです。接続のライフサイクル管理、HTTP 固有のフィルタチェーンの構築、および MPM（Multi-Processing Module）との連携を担います。

モジュール内での役割

modules/http ディレクトリにおける「HTTP プロトコルスタックの入口」として機能します。他のフィルタ（byterange, chunk 等）を登録し、接続が確立された後のリクエスト処理フローを制御します。

責務

  • **接続処理のディスパッチ**: MPM の特性（同期/非同期）に応じた接続処理関数の切り替え。
  • **HTTP フィルタの管理**: HTTP プロトコルに必要なフィルタ（入力フィルタ、ヘッダーフィルタ、チャンクフィルタ等）の登録とリクエストへの適用。
  • **設定管理**: KeepAlive 関連のディレクティブ（Timeout, MaxRequests）の解析とサーバー設定への反映。
  • **フックの提供**: post_config や process_connection などの主要なサーバーライフサイクルイベントへの介入。

重要なシンボル

フィルタハンドル (Public Data)

  • ap_http_input_filter_handle: リクエストボディの読み込み用。
  • ap_http_header_filter_handle: HTTP レスポンスヘッダーの生成用。
  • ap_chunk_filter_handle: チャンク転送エンコーディング用。
  • ap_http_outerror_filter_handle: エラー発生時のレスポンス処理用。
  • ap_byterange_filter_handle: バイトレンジリクエストの処理用。

接続処理関数

  • ap_process_http_connection: MPM の種類に応じて同期/非同期処理を振り分けるエントリポイント。
  • ap_process_http_async_connection: 非同期 MPM 用のステートマシン駆動処理。
  • ap_process_http_sync_connection: 同期 MPM 用のループベースの処理。

設定・初期化

  • http_cmds: KeepAlive, KeepAliveTimeout, MaxKeepAliveRequests を定義するコマンドテーブル。
  • register_hooks: モジュールのフック登録およびフィルタの登録を行うメイン関数。

制御フローとデータフロー

1. **初期化**: register_hooks が起動時に呼ばれ、フィルタの登録と MPM の非同期サポート状況（async_mpm）を確認します。 2. **接続開始**: ap_process_http_connection が呼ばれ、MPM の特性に基づき sync または async の処理関数へ分岐します。 3. **リクエスト処理**:

  • ap_read_request でリクエストを解析。
  • http_create_request フックにより、必要な出力フィルタがリクエストのフィルタスタックに追加されます。
  • ap_process_request を通じてリクエストのハンドラが実行されます。

    • 4. **終了処理**: 非同期モードでは CONN_STATE_LINGER への遷移、同期モードではループによる Keep-Alive 継続判定が行われます。

読者向けメモ

  • **メモリ管理**: ap_process_request 呼び出し後、リクエストプールは破棄されるため、関数内で r = NULL とすることで解放後の不正アクセスを防止する防御的プログラミングがなされています。
  • **非同期 MPM**: async_mpm 変数は post_config で決定されます。非同期対応の MPM（event など）では、接続状態（conn_state_t）を介してステートマシンが駆動されます。
  • **省略された詳細**: 本ファイルはフィルタの「登録」と「接続の振り分け」に特化しており、個々のフィルタの具体的なロジック（バイトレンジの計算やチャンクのエンコード）は各フィルタ専用のソースファイルに委譲されています。
