modules/cache

システム内での役割

modules/cache は、Apache HTTP Server における透過的なキャッシュサブシステムを構成するディレクトリです。HTTP リクエストのキャッシュ可能性の判定、ストレージへの保存、およびキャッシュからのレスポンス提供を担うポリシーエンジンと、それらを支える多様なストレージプロバイダー（ディスク、共有メモリ、Redis 等）で構成されています。

責務

  • **キャッシュポリシーの制御**: RFC 2616 等の HTTP 仕様に基づき、キャッシュの鮮度判定、Cache-Control の解釈、および条件付きリクエストの処理を行う。
  • **ストレージの抽象化**: cache_storage インターフェースを通じて、物理的な保存先（ディスクやメモリ）を意識せずにキャッシュ操作を行う仕組みの提供。
  • **リクエストパイプラインの統合**: クイックハンドラやフィルタチェーン（CACHE_SAVE, CACHE_OUT 等）を介した、透過的なキャッシュの挿入と管理。
  • **並行制御**: 「サンダリング・ハード（Thundering Herd）」問題を防ぐためのロック機構の提供。

このモジュールのファイル

  • **コアロジック**: mod_cache.c, cache_storage.c, cache_util.c
  • **ヘッダー定義**: mod_cache.h, cache_storage.h, cache_util.h, cache_common.h
  • **ディスクキャッシュ**: mod_cache_disk.c, mod_cache_disk.h, cache_disk_common.h
  • **共有オブジェクトキャッシュ**: mod_cache_socache.c, cache_socache_common.h
  • **socache バックエンド**: mod_socache_dbm.c, mod_socache_dc.c, mod_socache_memcache.c, mod_socache_redis.c, mod_socache_shmcb.c
  • **その他**: mod_file_cache.c

主要なデータ構造

  • cache_request_rec: リクエストごとのキャッシュコンテキスト。
  • cache_provider: ストレージバックエンドが実装すべき API 定義。
  • cache_control_t: Cache-Control ヘッダーの解析結果。
  • disk_cache_info_t / cache_socache_info_t: 各ストレージ固有のメタデータ構造。
  • cache_handle_t: キャッシュエントリへのアクセスハンドル。

主なフロー

1. **判定**: cache_quick_handler がリクエストをインターセプトし、キャッシュ可能か判定。 2. **検索**: cache_select を通じて各プロバイダーを走査し、既存のキャッシュエントリを検索。 3. **提供/保存**: ヒット時は CACHE_OUT フィルタでレスポンスを構築し、ミス時は CACHE_SAVE フィルタでバックエンドからのレスポンスをストレージへ永続化。 4. **無効化**: PUT/POST/DELETE メソッド時に cache_invalidate_filter が関連キャッシュを削除。

外部依存

  • APR (Apache Portable Runtime): メモリ管理、ファイル I/O、共有メモリ、アトミック操作、文字列操作。
  • httpd.h 系: Apache のリクエスト処理、設定フレームワーク、フィルタチェーン。
  • ap_socache.h: 共有オブジェクトキャッシュの抽象インターフェース。

内部構成

本モジュールは、コアとなるポリシーエンジン（mod_cache）と、それを利用する複数のストレージプロバイダー（mod_cache_disk, mod_cache_socache 等）というプラグイン構成をとっています。

推奨読書順

1. modules/cache/cache_util.h (共通定義とユーティリティ) 2. modules/cache/mod_cache.h (キャッシュの主要インターフェース) 3. modules/cache/cache_storage.h (ストレージ抽象化層) 4. modules/cache/mod_cache.c (メインのポリシーエンジン) 5. modules/cache/cache_storage.c (ストレージ操作の実装) 6. modules/cache/mod_cache_disk.c (ディスクキャッシュの実装例) 7. modules/cache/mod_cache_socache.c (共有オブジェクトキャッシュの統合)
