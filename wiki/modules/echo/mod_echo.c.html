modules/echo/mod_echo.c

概要

Apache HTTP Server 用のシンプルなエコーサーバーモジュールです。クライアントから受信したデータをそのまま送り返すプロトコルハンドラを実装しており、主に接続処理のデモンストレーションやテスト用途として機能します。

モジュール内での役割

本ファイルは mod_echo モジュールの唯一のソースコードであり、Apache の接続処理フック（process_connection）を乗っ取ることで、HTTP 以外の独自プロトコル処理を実装する際のテンプレートとしての役割を果たします。

責務

  • **設定管理**: ProtocolEcho ディレクティブによるモジュールの有効/無効の制御。
  • **接続処理**: process_connection フックを通じたクライアントとの入出力ループの実行。
  • **ステータス更新**: scoreboard を利用した、現在処理中の接続状態（読み込み中、書き込み中等）の可視化。
  • **データ転送**: apr_bucket_brigade を用いた効率的なストリームデータの読み書き。

重要なシンボル

  • echo_module: モジュールのエントリポイントとなる構造体。
  • process_echo_connection: 接続ごとのメインループを実行するコア関数。
  • EchoConfig: モジュールの有効状態を保持する設定構造体。
  • brigade_peek: scoreboard に表示する文字列をバッファから抽出するヘルパー関数。

制御フローとデータフロー

1. **初期化**: register_hooks が process_connection フックを登録します。 2. **接続開始**: process_echo_connection が呼び出され、設定が有効か確認します。 3. **ループ処理**:

  • ap_get_brigade でクライアントからデータを読み込みます。
  • update_echo_child_status でスコアボードを更新します。
  • ap_pass_brigade で受信したデータをクライアントへ送り返します。

    • 4. **終了**: 接続が切断されるまでループし、最後にリソースを解放します。

直接依存

  • httpd.h, http_connection.h: Apache 接続処理の基盤。
  • apr_buckets.h, util_filter.h: フィルタチェーンを通じたデータストリーム操作。
  • scoreboard.h: サーバーの稼働状況を外部（server-status 等）に公開するためのインターフェース。
  • ap_config.h
  • ap_mmn.h
  • apr_strings.h
  • http_config.h
  • http_core.h
  • http_log.h

関連コード

  • modules/http/http_core.c: 標準的な HTTP 処理の実装。本モジュールと比較することで、プロトコルハンドラの独自実装の差異を理解できます。
  • server/scoreboard.c: update_echo_child_status が内部で利用しているスコアボードの管理ロジック。

読者向けメモ

  • 本モジュールは ap_get_brigade を AP_MODE_GETLINE モードで使用しており、行単位の処理を前提としています。
  • brigade_peek はスコアボード表示用の文字列を生成するために memchr を使用して LF を探すなど、バッファ操作の典型的なパターンを含んでいます。
  • 独自プロトコルを実装する際は、process_connection フック内で ap_get_conn_socket を取得し、タイムアウト設定を適切に行う必要がある点に注意してください。
