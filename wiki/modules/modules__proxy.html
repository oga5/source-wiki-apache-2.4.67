modules/proxy

システム内での役割

modules/proxy は、Apache HTTP Server におけるプロキシサブシステムの中核です。HTTP、AJP、FastCGI、uWSGI、WebSocket、FTP など、多様なプロトコルを介したバックエンドへのリクエスト転送、ロードバランシング、および接続管理機能を提供します。このモジュールはディレクトリ境界に基づいて定義されています。

責務

  • **プロキシ基盤の提供**: 接続プール、ワーカー管理、共有メモリ同期、URL 正規化などの共通機能の提供。
  • **プロトコルハンドラの統合**: 各プロトコル固有の通信ロジックを mod_proxy のフックシステムに統合。
  • **負荷分散とヘルスチェック**: バックエンドサーバーの監視と、トラフィックの動的な振り分け。

このモジュールのファイル

  • mod_proxy.c, mod_proxy.h: プロキシのメインロジックと共通インターフェース。
  • proxy_util.c, proxy_util.h: 接続管理やURL正規化などの共通ユーティリティ。
  • mod_proxy_http.c, mod_proxy_ajp.c, mod_proxy_fcgi.c, mod_proxy_uwsgi.c, mod_proxy_scgi.c, mod_proxy_wstunnel.c, mod_proxy_connect.c, mod_proxy_ftp.c: 各プロトコルハンドラ。
  • mod_proxy_balancer.c, mod_proxy_hcheck.c: ロードバランサーおよびヘルスチェック機能。
  • mod_proxy_express.c, mod_proxy_fdpass.c: 動的ルーティングおよびFD受け渡し機能。
  • ajp_*.c, ajp_*.h, scgi.h: AJP/SCGI プロトコル固有の定義と実装。

主要なデータ構造

  • proxy_server_conf, proxy_dir_conf: サーバーおよびディレクトリ単位の設定構造体。
  • proxy_worker, proxy_balancer: バックエンドサーバーおよび負荷分散グループの定義。
  • proxy_conn_rec: バックエンドへの接続状態（ソケット、プール等）。
  • ajp_msg_t: AJP 通信用のメッセージバッファ。

主なフロー

1. **リクエスト翻訳**: proxy_translate_name が URL を解析し、プロキシ対象としてマーク。 2. **ワーカー選択**: pre_request フックにより、バランサーまたはワーカーを特定。 3. **接続取得**: ap_proxy_acquire_connection を呼び出し、接続プールから既存の接続を再利用または新規作成。 4. **バックエンド通信**: プロトコル固有のハンドラがリクエストを変換・送信し、レスポンスをフィルタリングしてクライアントへ転送。 5. **終了処理**: ap_proxy_release_connection により接続をプールへ返却。

外部依存

  • **APR (Apache Portable Runtime)**: メモリ管理、ネットワーク I/O、スレッド同期、共有メモリ（slotmem）。
  • **Apache Core**: httpd.h, http_config.h, http_request.h 等のリクエスト処理基盤。

内部構成

  • プロキシ基盤（mod_proxy, proxy_util）を核とし、複数のプロトコルハンドラおよび拡張モジュールがプラグインとして接続される構成。

推奨読書順

1. mod_proxy.h (インターフェース定義) 2. proxy_util.h / proxy_util.c (共通基盤) 3. mod_proxy.c (メインロジック) 4. mod_proxy_http.c (標準的なプロトコルハンドラ) 5. mod_proxy_balancer.c (負荷分散) 6. mod_proxy_hcheck.c (ヘルスチェック) 7. ajp.h / ajp_msg.c (AJP プロトコルスタック)
