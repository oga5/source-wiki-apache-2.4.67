test/modules/http2/mod_h2test

システム内での役割

mod_h2test は、Apache HTTP Server の HTTP/2 実装（mod_h2）を検証するためのテスト専用モジュールです。本モジュールはディレクトリ境界に基づき定義されており、実際の運用環境ではなく、テスト環境において HTTP/2 プロトコルの挙動をシミュレートするエンドポイントを提供します。

責務

  • **HTTP/2 挙動のシミュレーション**: ストリーム制御、トレーラーヘッダーの付与、意図的なエラー発生、レスポンスの遅延送信などを実行し、プロトコルスタックの堅牢性を検証します。
  • **テスト用ハンドラの提供**: h2test-echo、delay、trailer、error といった特定のハンドラ名に基づき、リクエストを処理します。

このモジュールのファイル

  • test/modules/http2/mod_h2test/mod_h2test.c: モジュールの主要ロジックおよびハンドラの実装。
  • test/modules/http2/mod_h2test/mod_h2test.h: モジュール全体で共有されるインターフェースや定数の定義。

主要なデータ構造

  • request_rec: Apache のリクエスト構造体。ハンドラ内でリクエストパラメータの解析に使用。
  • apr_bucket_brigade: レスポンス構築およびフィルタチェーンへのデータ送信に使用されるバケットブリゲード。

主なフロー

1. **リクエスト受信**: r->handler を確認し、担当するテストハンドラへ振り分けます。 2. **パラメータ解析**: r->args を解析し、テスト用の制御パラメータ（遅延時間やエラー種別など）を抽出します。 3. **レスポンス構築**: apr_bucket_brigade を操作し、必要に応じてエラーバケットやトレーラーを挿入します。 4. **データ送信**: ap_pass_brigade を介してフィルタチェーンへデータを流し込みます。 5. **クリーンアップ**: r->pool を利用したメモリ管理により、リクエスト終了時にリソースを自動解放します。

外部依存

  • **Apache HTTP Server API**: httpd.h, http_protocol.h, http_request.h
  • **APR (Apache Portable Runtime)**: apr_optional.h, apr_strings.h, apr_time.h

内部構成

  • 子モジュールは存在しません。
  • h2test_hooks を介して Apache のフックシステムに登録されます。

推奨読書順

1. test/modules/http2/mod_h2test/mod_h2test.h（インターフェース定義の確認） 2. test/modules/http2/mod_h2test/mod_h2test.c（ハンドラ実装および制御フローの確認）
