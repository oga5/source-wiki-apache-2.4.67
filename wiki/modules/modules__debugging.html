modules/debugging

システム内での役割

modules/debugging は、Apache HTTP Server のフィルタチェーンの挙動を可視化および操作するためのデバッグ用モジュール群を格納するディレクトリです。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • **データ監視**: ネットワーク I/O を傍受し、生のデータストリームをログへダンプする。
  • **フィルタ操作**: バケットの分割やフラッシュを強制的に実行し、フィルタチェーンの挙動をテスト・検証する。

このモジュールのファイル

  • modules/debugging/mod_bucketeer.c: バケット操作・インジェクションツール。
  • modules/debugging/mod_dumpio.c: 入出力データスニファー。

主要なデータ構造

  • dumpio_conf_t: mod_dumpio の入出力設定を保持。
  • bucketeer_filter_config_t: mod_bucketeer のデリミタ設定を保持。
  • bucketeer_ctx_t: mod_bucketeer のフィルタ実行状態を保持。

主なフロー

1. **mod_dumpio**: 接続フェーズでフィルタを挿入し、ap_get_brigade や ap_pass_brigade を通じて流れるバケットの内容を APLOG_TRACE7 レベルでログ出力する。 2. **mod_bucketeer**: 出力フィルタとして動作し、データ内の制御文字を検知してバケットの分割、フラッシュ、ブリゲードの送信をトリガーする。

外部依存

  • Apache コア: httpd.h, http_config.h, http_log.h, http_request.h, http_protocol.h, http_connection.h
  • APR: apr_strings.h, apr_general.h, apr_buckets.h
  • ユーティリティ: util_filter.h

内部構成

本ディレクトリは個別のデバッグツールをまとめたものであり、モジュール間の直接的な階層構造はありません。

推奨読書順

1. modules/debugging/mod_dumpio.c（フィルタチェーンの可視化を理解するため） 2. modules/debugging/mod_bucketeer.c（フィルタチェーンの操作を理解するため）
