modules/test/mod_dialup.c

概要

mod_dialup.c は、Apache HTTP Server において低速なネットワーク接続（ダイヤルアップ回線）をシミュレートするための実験的なハンドラモジュールです。レスポンスデータを一定のバイトレートで分割し、ap_mpm_register_timed_callback を用いて非同期に送信することで、帯域制限を模倣します。

モジュール内での役割

modules/test ディレクトリ内で、非同期 I/O と SUSPENDED ステータスを利用したリクエスト処理の制御フローを実証する役割を担います。

責務

  • **帯域制限の適用**: 設定されたモデム規格に基づき、レスポンスの送信速度を制限する。
  • **非同期処理の管理**: SUSPENDED 状態を利用し、1秒間隔のタイマーコールバックでデータを断続的に送信する。
  • **設定管理**: ModemStandard ディレクティブによる通信速度の定義と適用。

重要なシンボル

設定・状態管理

  • dialup_dcfg_t: ディレクトリごとの設定構造体。bytes_per_second を保持。
  • dialup_baton_t: 非同期処理のコンテキスト。送信中のバケットブリゲード、ファイル記述子、リクエスト情報を保持。
  • modem_bitrates: モデム規格名とバイトレートの対応テーブル。

ハンドラ・フック

  • dialup_handler: リクエストをインターセプトし、初期送信とタイマー登録を行うメインハンドラ。
  • dialup_send_pulse: 制限値分だけデータを送信するコアロジック。
  • dialup_callback: タイマーにより起動される非同期処理の再開点。
  • dialup_register_hooks: APR_HOOK_ALMOST_LAST でハンドラを登録。

制御フローとデータフロー

1. **初期化**: dialup_handler がリクエストを捕捉し、ファイルを開いて dialup_baton_t を初期化。 2. **パルス送信**: dialup_send_pulse が bb（全データ）から tmpbb（送信分）へバケットを移動し、ap_pass_brigade で送信。 3. **非同期ループ**:

  • データが残っている場合、SUSPENDED を返し、ap_mpm_register_timed_callback で1秒後に dialup_callback を予約。
  • dialup_callback は invoke_mtx で排他制御を行い、再度 dialup_send_pulse を呼び出す。

    • 4. **終了**: 全データ送信完了時に ap_finalize_request_protocol を呼び出し、リクエストを完了させる。

直接依存

  • httpd.h, http_core.h, http_protocol.h: Apache コア API。
  • util_filter.h: バケットブリゲード操作。
  • ap_mpm.h: 非同期タイマーコールバック用。
  • http_config.h
  • http_log.h
  • http_request.h

関連コード

  • modules/test/mod_optional_fn_export.c: 本モジュールと同様に、Apache の拡張機能（フックやオプション関数）のテストケースとして機能する。

読者向けメモ

  • **メモリ管理**: dialup_baton_t および関連するブリゲードはリクエストプール (r->pool) に割り当てられており、リクエスト終了時に自動解放されます。
  • **排他制御**: dialup_callback 内で db->r->invoke_mtx を使用しており、マルチスレッド環境での安全性を確保しています。
  • **制限事項**: 本モジュールは実験的であり、エラーハンドリング（特に ap_pass_brigade 失敗時）は最小限です。また、SUSPENDED 状態の利用は MPM の実装に依存するため、すべての環境で期待通りに動作するとは限りません。
  • **未実装**: dialup_send_pulse 内の XXXXXX: Log me. と記された箇所など、エラー時の詳細なログ出力は省略されています。
