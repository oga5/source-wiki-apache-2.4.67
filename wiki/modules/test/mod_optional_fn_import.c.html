modules/test/mod_optional_fn_import.c

概要

Apache HTTP Server のオプション関数（Optional Function）メカニズムを検証するためのテスト用モジュールです。他のモジュールがエクスポートした関数を動的に取得し、実行する手順を実証します。

モジュール内での役割

modules/test サブシステムにおいて、mod_optional_fn_export.c と対になる「コンシューマ」側の実装です。モジュール間での疎結合な関数呼び出しが、Apache のフックシステムを通じてどのように実現されるかを示します。

責務

  • APR_RETRIEVE_OPTIONAL_FN を使用した外部関数の動的解決。
  • optional_fn_retrieve フックを用いた、関数ポインタの遅延バインディング。
  • log_transaction フックへの介入による、取得した関数の実行。

重要なシンボル

  • pfn: エクスポートされた関数を保持する静的な関数ポインタ。
  • ImportFnRetrieve: optional_fn_retrieve フックで呼び出され、関数ポインタを解決する関数。
  • ImportLogTransaction: 実際にインポートした関数を呼び出すハンドラ。
  • optional_fn_import: モジュール定義構造体。

制御フローとデータフロー

1. **初期化**: モジュールロード時、ImportRegisterHooks がフックを登録します。 2. **解決フェーズ**: optional_fn_retrieve フックが実行されると ImportFnRetrieve が呼ばれ、APR_RETRIEVE_OPTIONAL_FN を通じて pfn に実体が格納されます。 3. **実行フェーズ**: リクエスト処理の log_transaction フェーズで ImportLogTransaction が起動し、pfn が非 NULL であればエクスポートされた関数を実行します。

直接依存

  • httpd.h: Apache コア定義。
  • http_config.h: モジュール構造体およびフック定義。
  • http_protocol.h: リクエスト処理関連の定義。
  • mod_optional_fn_export.h: エクスポートされる関数の型定義。

関連コード

  • modules/test/mod_optional_fn_export.c: 本モジュールがインポートする関数を定義・エクスポートしている対向モジュール。
  • modules/test/mod_optional_hook_import.c: 類似のメカニズム（フック版）を検証するコード。

読者向けメモ

  • 本ファイルは「インポート側」の実装です。コード内のコメントにある通り、エクスポート側（mod_optional_fn_export.c）と対比して読むことで、Apache のモジュール間通信の仕組みが理解しやすくなります。
  • APR_RETRIEVE_OPTIONAL_FN は、対象の関数がロードされていない場合でも NULL を返すだけで安全に動作するように設計されています。そのため、実行時に pfn の NULL チェックを行うことが重要です。
