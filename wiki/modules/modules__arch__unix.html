modules/arch/unix

システム内での役割

本モジュールは、Apache HTTP Server が Unix 系 OS 上で動作する際に必要な、プロセス権限管理、環境分離、およびシステムサービス統合のための基盤を提供します。ディレクトリ境界に基づき、Unix 特有のシステムコールや外部サービス（systemd）との連携機能を抽象化・集約しています。

責務

  • **権限管理**: プロセス起動時の root 権限から非特権ユーザーへの降格（setuid/setgid）。
  • **環境分離**: chroot による実行ディレクトリの制限および suEXEC の検証。
  • **特権の動的制御**: Solaris 等の特権機能を用いたリクエスト単位の権限分離。
  • **サービス統合**: systemd によるサービス管理（起動通知、ソケット引き継ぎ、稼働状況の報告）のサポート。

このモジュールのファイル

  • mod_unixd.c / mod_unixd.h: 権限管理とプロセス環境設定のコア実装。
  • mod_privileges.c: OS レベルの特権機能を活用したリクエスト単位の権限分離。
  • mod_systemd.c: systemd サービスマネージャとの連携インターフェース。

主要なデータ構造

  • unixd_config_t: ユーザー/グループ ID や chroot 設定を保持する共通設定構造体。
  • priv_cfg / priv_dir_cfg: mod_privileges における vhost 単位の権限セットおよび動作モード。
  • ap_scoreboard_image: mod_systemd が稼働統計を取得するために参照する共有メモリ構造。

主なフロー

1. **起動時**: mod_unixd が設定を読み込み、mod_systemd が起動完了をサービスマネージャへ通知。 2. **プロセス生成時**: ap_unixd_setup_child() が呼び出され、グループ権限の確定、chroot の実行、最終的な setuid による権限降格が行われる。 3. **リクエスト処理時**: mod_privileges が有効な場合、リクエストごとに権限セットの動的変更やプロセス分離（fork）が実行される。 4. **監視時**: mod_systemd が定期的にスコアボードから統計を読み取り、systemd へステータスを送信。

外部依存

  • **システム API**: unistd.h, pwd.h, grp.h, sys/prctl.h, priv.h (Solaris)
  • **systemd**: libsystemd (sd-daemon.h)
  • **Apache 内部**: httpd.h, mpm_common.h, unixd.h, scoreboard.h

内部構成

本モジュールは、Unix 系 MPM の動作を支える独立した機能モジュールの集合体です。各モジュールは MPM のフックポイントを利用して連携しますが、モジュール間での直接的な依存は最小限に抑えられています。

推奨読書順

1. mod_unixd.h: 権限管理の基本インターフェースを理解する。 2. mod_unixd.c: プロセス降格の標準的な実装を確認する。 3. mod_privileges.c: OS 特有の高度な権限分離手法を学ぶ。 4. mod_systemd.c: 外部サービスマネージャとの統合ロジックを確認する。
