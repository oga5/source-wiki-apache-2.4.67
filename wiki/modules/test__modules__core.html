test/modules/core

システム内での役割

Apache HTTP Server のコア機能（リクエストパスの正規化、プロセスライフサイクル管理）の堅牢性を検証するテストスイートです。pyhttpd フレームワークを利用し、サーバーの動的な挙動やセキュリティ境界条件を網羅的にテストします。

責務

  • **環境構築**: テスト実行に必要な Apache 環境（CoreTestEnv）の初期化とモジュール（cgid 等）のロード。
  • **セキュリティ検証**: URL エンコーディングやパストラバーサルに対するサーバーの応答正当性の保証。
  • **プロセス管理検証**: 負荷環境下における MPM（Multi-Processing Module）の再起動・世代交代の動的追跡。

このモジュールのファイル

  • __init__.py: パッケージ定義。
  • conftest.py: テスト環境のライフサイクル管理（pytest フィクスチャ）。
  • env.py: テスト環境の構成定義（CoreTestEnv, CoreTestSetup）。
  • test_001_encoding.py: エンコーディングおよびパス正規化テスト。
  • test_002_restarts.py: プロセス再起動と世代交代の動的テスト。

主要なデータ構造

  • CoreTestEnv: テスト環境の抽象化クラス。
  • Loader: h2load をラップした負荷生成ヘルパー。
  • ChildDynamics: ログ解析によるプロセス状態（PID, 世代）の抽出器。

主なフロー

1. **セットアップ**: conftest.py が CoreTestEnv を通じて Apache を起動し、env.py が必要なモジュールをロード。 2. **テスト実行**:

  • **静的検証**: test_001_encoding.py が多様なパス文字列を送信し、ステータスコードとエラーログを検証。
  • **動的検証**: test_002_restarts.py が Loader で負荷をかけつつ、apache_reload() 前後のログを ChildDynamics で解析し、プロセスの世代交代を確認。

    • 3. **クリーンアップ**: 全テスト終了後、conftest.py の _stop_package_scope が Apache プロセスを停止。

外部依存

  • pyhttpd: Apache 制御用基盤ライブラリ。
  • pytest: テストフレームワーク。
  • h2load: 負荷生成ツール（test_002_restarts.py で使用）。

内部構成

本モジュールは pyhttpd の拡張パターンに従い、CoreTestEnv を中心とした単一のパッケージ構成をとります。

推奨読書順

1. env.py（環境定義の理解） 2. conftest.py（テストライフサイクルの理解） 3. test_001_encoding.py（正常系・境界値テストの確認） 4. test_002_restarts.py（動的・負荷テストの確認） 5. __init__.py
