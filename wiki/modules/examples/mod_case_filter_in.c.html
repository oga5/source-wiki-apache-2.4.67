modules/examples/mod_case_filter_in.c

概要

mod_case_filter_in.c は、Apache HTTP Server の入力フィルタ機能を示すサンプルモジュールです。クライアントから送信されるリクエストボディの内容を読み取り、すべて大文字に変換して後続の処理へ渡すという単純な変換ロジックを実装しています。

モジュール内での役割

modules/examples ディレクトリにおいて、入力フィルタ（Input Filter）の登録と実行フローを解説するための教育的リソースとして機能します。リクエストヘッダーではなく、リクエストボディのストリーム処理に焦点を当てています。

責務

  • **フィルタの登録**: insert_filter フックを使用して、設定に応じて入力フィルタを動的に挿入する。
  • **データ変換**: ap_input_filter を通じて、バケットブリゲード（apr_bucket_brigade）内のデータを逐次読み込み、apr_toupper を用いて大文字化する。
  • **設定管理**: CaseFilterIn ディレクティブによるフィルタの有効/無効の切り替え機能を提供する。

重要なシンボル

  • CaseFilterInFilter: フィルタの本体。バケットを読み取り、変換し、出力ブリゲードへ再構築する。
  • CaseFilterInInsertFilter: フィルタをリクエスト処理チェーンに挿入するフック関数。
  • CaseFilterInConfig: モジュールの有効状態を保持するサーバー設定構造体。
  • CaseFilterInContext: フィルタ実行中の状態（一時的なブリゲードなど）を保持するコンテキスト。

制御フローとデータフロー

1. **初期化**: CaseFilterInRegisterHooks がフィルタを AP_FTYPE_RESOURCE として登録します。 2. **挿入**: リクエスト処理時、CaseFilterInInsertFilter が設定を確認し、有効であれば ap_add_input_filter を呼び出します。 3. **フィルタ処理**:

  • CaseFilterInFilter が呼び出されると、ap_get_brigade で入力データを取得します。
  • バケットを一つずつ走査し、apr_bucket_read でデータを取得、大文字に変換します。
  • 変換後のデータを apr_bucket_heap_create で新しいバケットとして作成し、出力ブリゲードへ追加します。
  • 最後に元のバケットを削除し、処理を完了します。

直接依存

  • httpd.h, http_config.h: Apache モジュール基盤。
  • apr_buckets.h: バケットブリゲード操作用。
  • util_filter.h: フィルタチェーン操作用。
  • apr_lib.h: apr_toupper などのユーティリティ。
  • ctype.h: 標準的な文字操作。
  • apr_general.h
  • http_request.h

関連コード

  • mod_case_filter.c: 出力フィルタ（Output Filter）のサンプル。対照的な実装として参照してください。
  • mod_example_hooks.c: フックの基本的な仕組みを理解するために有用です。

読者向けメモ

  • **バケットの取り扱い**: このコードはバケットの変換において、既存のバケットを直接書き換えるのではなく、新しいヒープバケットを作成して置き換える手法をとっています。これはファイルやソケットなどの特殊なバケットを破壊しないための安全な実装パターンです。
  • **制限事項**: このフィルタは AP_FTYPE_RESOURCE で動作するため、リクエストヘッダーの解析後に追加されます。そのため、ヘッダーの内容を変換することはできません。
  • **メモリ管理**: apr_bucket_heap_create に free 関数を渡すことで、変換後のバケットが破棄される際にメモリが適切に解放されるよう設計されています。
