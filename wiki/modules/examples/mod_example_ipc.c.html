modules/examples/mod_example_ipc.c

概要

Apache HTTP Server におけるプロセス間通信（IPC）の利用例を示すサンプルモジュールです。共有メモリ（Shared Memory）とグローバルミューテックス（Global Mutex）を使用して、マルチプロセス環境下で共有カウンタを安全にインクリメントする方法を提示します。

モジュール内での役割

modules/examples ディレクトリにおいて、APR（Apache Portable Runtime）の IPC 機能に特化した実装例を提供します。実運用向けではなく、開発者が Apache のプロセスモデルと共有リソースの同期メカニズムを理解するための教育用コードです。

責務

  • **共有メモリの管理**: apr_shm を用いた共有メモリセグメントの確保と初期化。
  • **同期制御**: apr_global_mutex を用いた、プロセス間での排他制御の実装。
  • **ライフサイクル管理**: 親プロセスでのリソース確保と、子プロセスでの再初期化、および終了時のクリーンアップ。
  • **リクエスト処理**: HTTP リクエストに応じて共有カウンタを更新し、結果を HTML で出力する。

重要なシンボル

  • exipc_shm: 共有メモリブロックを指すポインタ。
  • exipc_mutex: プロセス間で共有されるグローバルミューテックス。
  • exipc_data: 共有メモリ内に格納されるデータ構造（カウンタを保持）。
  • exipc_handler: リクエストを受け取り、ロック取得・カウンタ更新・レスポンス生成を行うメイン関数。

制御フローとデータフロー

1. **初期化**: pre_config でミューテックス型を登録し、post_config で共有メモリの確保とミューテックスの生成を行います。 2. **子プロセス起動**: child_init で各子プロセスがミューテックスを再オープンし、共有リソースへのアクセス準備を整えます。 3. **リクエスト処理**:

  • exipc_handler が呼び出されると、apr_global_mutex_trylock または lock を試行します。
  • ロック取得後、共有メモリ上の exipc_data を読み込み、カウンタをインクリメントします。
  • 処理終了後、apr_global_mutex_unlock でロックを解放します。

直接依存

  • apr.h, apr_shm.h, util_mutex.h: APR の IPC および同期プリミティブ。
  • httpd.h, http_config.h, http_protocol.h: Apache モジュール API。
  • unistd.h: getpid() 等のシステムコール用。

関連コード

  • modules/examples/mod_example_hooks.c: Apache のフック機構の基本を学ぶための関連サンプル。
  • util_mutex.h: Apache におけるミューテックスの抽象化レイヤーの詳細。

読者向けメモ

  • **ロックの挙動**: 本コードは trylock が未実装のプラットフォームを考慮したフォールバック処理を含んでおり、ロック取得の試行ロジックがやや複雑です。
  • **実運用上の注意**: このモジュールは教育用であり、高負荷環境でのパフォーマンスや、共有メモリの堅牢な管理については考慮されていません。
  • **デバッグ**: ab (Apache Benchmark) を使用して、同時接続数を変えながらカウンタの挙動を観察することをお勧めします。
