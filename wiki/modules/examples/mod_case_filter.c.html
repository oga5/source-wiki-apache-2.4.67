modules/examples/mod_case_filter.c

概要

Apache HTTP Server の出力フィルタ機能を示すサンプルモジュールです。レスポンスボディに含まれるすべての文字を大文字に変換してクライアントへ送信する、単純なフィルタ処理を実装しています。

モジュール内での役割

modules/examples ディレクトリにおいて、Apache のフィルタチェーン（Output Filter）の基本的な構築方法と、バケットブリゲード（Bucket Brigade）の操作方法を解説するための教育用コードです。

責務

  • **設定管理**: CaseFilter ディレクティブによるフィルタの有効/無効の切り替え。
  • **フィルタ登録**: insert_filter フックを用いた、リクエスト処理時のフィルタ挿入。
  • **データ変換**: CaseFilterOutFilter による、バケット単位のデータ読み込みと大文字変換処理。

重要なシンボル

  • CaseFilterOutFilter: フィルタの本体。バケットを走査し、データを変換して後続のフィルタへ渡す。
  • CaseFilterInsertFilter: リクエストに応じてフィルタをチェーンに追加するか判断するフック関数。
  • CaseFilterConfig: モジュールの有効状態を保持するサーバー設定構造体。
  • case_filter_module: Apache モジュール定義のメインエントリポイント。

制御フローとデータフロー

1. **初期化**: CaseFilterRegisterHooks がフィルタを登録し、CaseFilterCreateServerConfig が設定領域を確保します。 2. **挿入**: リクエスト発生時、CaseFilterInsertFilter が設定を確認し、有効であれば ap_add_output_filter を呼び出します。 3. **変換**: CaseFilterOutFilter が呼び出されると、入力バケットブリゲード（pbbIn）からバケットを順次取り出し、apr_toupper で変換後、新しいヒープバケットとして出力ブリゲード（pbbOut）に格納します。 4. **伝播**: 最後に ap_pass_brigade を通じて、変換後のデータを後続のフィルタへ引き渡します。

直接依存

  • httpd.h, http_config.h: Apache モジュール基盤。
  • util_filter.h: フィルタチェーン操作用 API。
  • apr_buckets.h: バケットブリゲードによるデータストリーム処理。
  • ctype.h: 文字変換関数（apr_toupper の利用）。
  • apr_general.h
  • apr_lib.h
  • http_request.h

関連コード

  • modules/examples/mod_case_filter_in.c: 本ファイルと対になる入力フィルタのサンプル。
  • modules/examples/mod_example_hooks.c: Apache フックの仕組みを理解するための基礎的なサンプル。

読者向けメモ

  • **メモリ効率**: この実装はバケットを新規作成してデータをコピーしていますが、大規模なレスポンスを扱う場合はメモリ消費量に注意が必要です。ソース内のコメントにある通り、バケットの消費（破棄）タイミングは効率に直結します。
  • **学習のポイント**: apr_bucket_read でデータを取得し、apr_bucket_heap_create で新しいバケットを作成する一連の流れは、Apache フィルタ開発の定石です。
  • **注意**: 本モジュールは教育用であり、実運用環境での使用は想定されていません。
