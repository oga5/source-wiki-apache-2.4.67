modules/examples/mod_example_hooks.c

概要

mod_example_hooks.c は、Apache HTTP Server の全処理フェーズにおけるフック呼び出しを可視化・追跡するための教育用モジュールです。サーバーの起動からリクエスト処理、接続管理に至るまで、ほぼすべての利用可能なフックに介入し、その実行順序とコンテキストを HTML 形式で出力します。

モジュール内での役割

本モジュールは、Apache モジュール開発者がフックの実行タイミングや設定の継承ルールを理解するための「リファレンス実装」として機能します。実運用環境での使用は推奨されません。

責務

  • **フックの可視化**: サーバーの全ライフサイクルにおけるコールバックを捕捉し、ログおよび HTML レポートとして蓄積する。
  • **設定管理**: ディレクトリおよびサーバー単位の設定構造体（x_cfg）の生成、マージ、および継承ルールのデモンストレーション。
  • **ディレクティブ提供**: Example ディレクティブを通じた設定の適用と、その影響範囲の追跡。

重要なシンボル

設定構造体と管理

  • struct x_cfg: モジュールの設定データ。cmode（スコープ）、local（ディレクティブ有無）、congenital（継承フラグ）、trace（ログ文字列）を保持。
  • x_create_dir_config / x_create_server_config: 設定構造体の初期化とメモリ割り当て。
  • x_merge_dir_config / x_merge_server_config: 親設定と子設定をマージするロジック。

フック登録とディレクティブ

  • x_register_hooks: ap_hook_* を用いて、全フェーズのコールバック関数をサーバーに登録する。
  • x_cmds: Example ディレクティブを定義する command_rec 配列。
  • example_hooks_module: モジュールのエントリポイントとなる module 構造体。

トレースユーティリティ

  • trace_startup: 起動時のフックをグローバル変数に記録。
  • trace_request: リクエスト単位のフックを request_rec->notes に記録。
  • trace_connection: 接続単位のフックを conn_rec->pool のユーザーデータに記録。

制御フローとデータフロー

1. **初期化**: x_register_hooks がサーバー起動時に呼ばれ、各フック関数を登録。 2. **設定**: Example ディレクティブが解析されると cmd_example が呼ばれ、設定構造体にフラグを立てる。 3. **実行**: 各フック関数（x_handler 等）が呼ばれるたびに、対応する trace_* 関数が呼び出され、現在のコンテキスト（request_rec や conn_rec）から情報を抽出して文字列を生成し、プールに保存する。 4. **出力**: x_handler が example-hooks-handler を受け取ると、蓄積されたトレース文字列を HTML としてクライアントに返却する。

直接依存

  • httpd.h, http_config.h, http_request.h: Apache コア API。
  • apr_pools.h (経由): メモリ管理およびユーザーデータ保存。
  • apr_strings.h: 文字列操作とトレース生成。

関連コード

  • modules/examples/mod_example_ipc.c: プロセス間通信の例。
  • modules/examples/mod_case_filter.c: フィルタ処理の例。

読者向けメモ

  • **メモリ管理**: すべてのトレース文字列は、関連する pool（request_pool や connection_pool）から割り当てられます。これにより、リクエスト終了時に自動的にメモリが解放されます。
  • **パフォーマンス**: 本モジュールはすべてのリクエストで文字列連結とログ出力を行うため、高負荷環境では深刻なパフォーマンス低下を招きます。
  • **制限事項**: EXAMPLE_LOG_EACH マクロを有効にすると error_log が肥大化します。また、マルチスレッド環境でのグローバル変数 trace の使用は競合の可能性があるため、注意が必要です。
  • **未実装**: 意図的に一部の複雑なフックや、非リクエスト関連の HTML バックトレースは簡略化されています。
