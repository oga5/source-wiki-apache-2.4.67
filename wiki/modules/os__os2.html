os/os2

システム内での役割

os/os2 は、Apache HTTP Server における OS/2 プラットフォーム固有の抽象化レイヤーです。ディレクトリ境界に基づき構成されており、OS/2 環境特有のファイルシステム特性やプロセス管理の差異を吸収します。

責務

  • OS/2 環境におけるファイルシステム特性（ドライブレター、大文字小文字の区別など）の定義。
  • OS/2 ネイティブ API と Apache コア機能の橋渡し。
  • プラットフォーム固有のプロセス生成処理の抽象化。

このモジュールのファイル

  • os/os2/os.h: OS/2 固有の構成定義およびプラットフォームフラグ。
  • os/os2/util_os2.c: OS/2 固有のユーティリティ関数（プロセス管理等）の実装。

主要なデータ構造

  • 本モジュールは主にマクロ定義と関数ラッパーで構成されており、独自の複雑なデータ構造は定義されていません。

主なフロー

  • **構成定義**: コンパイル時に os.h のマクロが参照され、条件付きコンパイルを通じて OS/2 向けの動作が決定されます。
  • **プロセス生成**: ap_os_create_privileged_process が呼び出されると、APR（Apache Portable Runtime）のプロセス生成関数を介して OS/2 の実行環境へ処理が委譲されます。

外部依存

  • httpd.h, http_log.h, os.h (Apache HTTPD)
  • apr_strings.h (APR)
  • <os2.h> (OS/2 システムヘッダ)

内部構成

  • 子モジュールは存在しません。

推奨読書順

1. os/os2/os.h: プラットフォームの前提条件（ファイルシステム特性等）を把握する。 2. os/os2/util_os2.c: OS/2 固有のプロセス管理実装を確認する。

未解決の問い

  • HAVE_* マクロと APR の機能重複に関する整理の必要性。
  • 将来的な OS/2 ネイティブ API（DosExecPgm 等）への直接的な拡張可能性。
