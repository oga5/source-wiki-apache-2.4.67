modules/aaa/mod_authn_anon.c

概要

mod_authn_anon.c は、Apache HTTP Server において「匿名ログイン」を許可するための認証プロバイダーモジュールです。FTP の匿名ログインと同様の仕組みを提供し、特定のユーザー名（anonymous など）とパスワード（メールアドレス形式を想定）による認証を可能にします。

モジュール内での役割

modules/aaa サブシステムの一部として、認証プロバイダー（authn）インターフェースを実装しています。mod_authn_core が提供する認証フレームワークに「anon」という名前のプロバイダーとして登録され、他の認証モジュールと組み合わせて使用されます。

責務

  • **設定管理**: Anonymous ディレクティブで指定された許可ユーザー名のリストと、メールアドレス検証などの動作フラグを保持する。
  • **認証ロジック**: 提供されたユーザー名が許可リストに含まれているか、およびパスワードが設定されたポリシー（メールアドレス形式の検証など）を満たしているかを判定する。
  • **ログ出力**: 設定に応じて、認証に使用されたパスワード（メールアドレス）をログに記録する。

重要なシンボル

  • authn_anon_config_rec: ディレクトリごとの設定を保持する構造体。許可ユーザーリストや各種フラグ（mustemail, verifyemail 等）を管理。
  • check_anonymous: 認証プロバイダーのメイン関数。ユーザー名とパスワードの検証を行い、AUTH_GRANTED または AUTH_DENIED を返却する。
  • authn_anon_provider: ap_register_auth_provider に渡される、認証関数へのポインタを含む構造体。

制御フローとデータフロー

1. **設定**: anon_set_string_slots が Anonymous ディレクティブを解析し、許可ユーザーリストを構築する。 2. **認証要求**: check_anonymous が呼び出される。 3. **ユーザー検証**: 設定されたリストと入力されたユーザー名を比較（大文字小文字を区別しない）。 4. **パスワード検証**: mustemail や verifyemail フラグに基づき、パスワードの有無や形式（@ と . の存在）をチェックする。 5. **結果返却**: 全ての条件を満たせば AUTH_GRANTED を返し、そうでなければ AUTH_DENIED または AUTH_USER_NOT_FOUND を返す。

直接依存

  • mod_auth.h: 認証プロバイダーの型定義とインターフェース。
  • httpd.h, http_config.h, http_log.h: Apache のコア機能、設定管理、ログ出力用。
  • ap_provider.h: 認証プロバイダー登録用。
  • apr_strings.h, apr_want.h: 文字列操作ユーティリティ。
  • http_core.h
  • http_protocol.h
  • http_request.h

関連コード

  • mod_authn_core.c: 認証プロバイダーの登録と呼び出しを統括するコアモジュール。
  • mod_auth_basic.c: 本モジュールと組み合わせて使用されることが多い、基本認証の実装。

読者向けメモ

  • 本モジュールは、認証の「ユーザー名」と「パスワード」の検証のみを担当します。認可（誰がどのリソースにアクセスできるか）は mod_authz_* 系モジュールが別途処理します。
  • Anonymous_VerifyEmail を有効にすると、パスワードフィールドに @ と . を含む文字列が必須となります。セキュリティ要件に応じて適切に設定してください。
  • ログ出力機能は ap_is_initial_req(r) をチェックしており、サブクエストによる重複ログ出力を防ぐ設計になっています。
