modules/aaa/mod_authn_dbm.c

概要

DBM（Database Manager）ファイルを使用してユーザー認証を行うための Apache HTTP Server モジュールです。ユーザー名とパスワード（またはハッシュ）を DBM 形式のデータベースから検索し、認証プロバイダーとして機能します。

モジュール内での役割

mod_authn_core が提供する認証フレームワークに「dbm」プロバイダーとして登録されます。他の認証モジュール（mod_authn_file など）と同様に、認証情報の取得元を外部ファイルに委譲する役割を担います。

責務

  • **設定管理**: AuthDBMUserFile および AuthDBMType ディレクティブによる DBM ファイルパスと形式の定義。
  • **認証情報の検索**: DBM ファイルからユーザー名に対応するパスワードハッシュを抽出。
  • **認証判定**: 提供されたパスワードと DBM 内のハッシュを比較し、認証結果（GRANTED/DENIED/NOT_FOUND）を返す。
  • **キャッシュ連携**: ap_authn_cache_store が存在する場合、検索結果をキャッシュへ保存。

重要なシンボル

  • authn_dbm_provider: 認証プロバイダーのインターフェース（check_dbm_pw, get_dbm_realm_hash を保持）。
  • fetch_dbm_value: DBM ファイルからキー（ユーザー名）に対応する値を読み出す共通関数。
  • authn_dbm_config_rec: ディレクトリごとの設定（ファイルパス、DBM タイプ）を保持する構造体。
  • authn_dbm_module: モジュールのメイン構造体。

制御フローとデータフロー

1. **初期化**: register_hooks で「dbm」プロバイダーを登録し、オプション関数 ap_authn_cache_store を取得。 2. **認証要求**: 認証フェーズで check_dbm_pw が呼び出される。 3. **DBM 検索**: fetch_dbm_value が apr_dbm API を使用してファイルを開き、ユーザー名で検索。 4. **検証**: 取得した文字列からパスワードを抽出し、apr_password_validate で検証。 5. **結果返却**: 成功時は AUTH_GRANTED、失敗時は AUTH_DENIED などを返却し、必要に応じてキャッシュへ書き込む。

関連コード

  • mod_authn_core.c: 認証プロバイダーの呼び出し元となるコアモジュール。
  • mod_authn_socache.c: 認証結果をキャッシュするモジュール（本モジュールが連携する authn_cache_store の提供元）。
  • apr_dbm: APR が提供する抽象化された DBM ライブラリ。

読者向けメモ

  • **DBM タイプ**: AuthDBMType で指定する値は、システムにインストールされている APR-util の DBM ドライバー（SDBM, GDBM, DB など）に依存します。
  • **互換性**: APU_MAJOR_VERSION 等の条件分岐により、古い APR-util と新しい API の両方に対応するよう設計されています。
  • **キャッシュ**: AUTHN_CACHE_STORE マクロは、キャッシュモジュールがロードされている場合にのみ機能します。キャッシュがない環境でも動作に支障はありません。
