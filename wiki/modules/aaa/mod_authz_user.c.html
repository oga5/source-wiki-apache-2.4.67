modules/aaa/mod_authz_user.c

概要

Apache HTTP Server の認可（Authorization）機能を提供するモジュールです。require user および require valid-user ディレクティブを実装し、認証済みユーザーがリソースへのアクセス権を持っているかを判定します。

モジュール内での役割

mod_authz_core が提供する認可プロバイダーフレームワークを利用するプラグインモジュールです。認証（Authentication）フェーズで特定されたユーザー名（r->user）に基づき、アクセス制御の最終判断を行います。

責務

  • **ユーザー単位のアクセス制御**: require user [name] で指定されたユーザー名と、リクエストのユーザー名を照合する。
  • **有効ユーザーの確認**: require valid-user により、認証済みであれば誰でもアクセスを許可する。
  • **設定のパース**: require 行に含まれる式（expression）を解析し、実行時に評価可能な形式に変換する。

重要なシンボル

  • user_check_authorization: require user の判定ロジック。
  • validuser_check_authorization: require valid-user の判定ロジック。
  • user_parse_config: 設定ファイル内の require 引数を式としてパースする。
  • authz_user_provider / authz_validuser_provider: mod_authz_core に登録されるプロバイダー構造体。

制御フローとデータフロー

1. **登録**: register_hooks により、起動時に user および valid-user プロバイダーを mod_authz_core へ登録します。 2. **パース**: 設定読み込み時、user_parse_config が require 行を ap_expr_info_t 形式に変換し、実行時の評価に備えます。 3. **判定**: リクエスト処理時、mod_authz_core から呼び出されます。

  • r->user が未設定（認証されていない）場合は AUTHZ_DENIED_NO_USER を返します。
  • user_check_authorization は式を評価し、結果の文字列と r->user を比較して AUTHZ_GRANTED または AUTHZ_DENIED を返します。

直接依存

  • mod_auth.h: 認可プロバイダーのインターフェース定義。
  • httpd.h, http_config.h, http_request.h: Apache のコア構造体およびリクエスト処理用ヘッダー。
  • ap_expr.h (間接的に http_core.h 経由): require 行の式評価機能。
  • ap_config.h
  • ap_provider.h
  • apr_strings.h
  • http_log.h
  • http_protocol.h

関連コード

  • modules/aaa/mod_authz_core.c: 本モジュールが提供するプロバイダーを呼び出し、認可フローを統括する中心的なモジュール。
  • modules/aaa/mod_authn_core.c: ユーザー認証を行い r->user を設定するモジュール。

読者向けメモ

  • 本モジュールは非常にシンプルで、状態を持たない（authz_user_config_rec はダミー）実装です。
  • require user は式評価（ap_expr）をサポートしているため、動的なユーザー指定が可能です。
  • 認証フェーズが完了していないリクエストに対しては、常に AUTHZ_DENIED_NO_USER を返すことで、認証モジュールへのフォールバックを促す設計になっています。
