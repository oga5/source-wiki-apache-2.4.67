modules/aaa/mod_auth_digest.c

概要

mod_auth_digest は、HTTP Digest 認証（RFC 2617）を実装する Apache HTTP Server モジュールです。パスワードを平文で送信せず、MD5 ハッシュとサーバー側で発行する「nonce（ナンス）」を使用して認証を行います。共有メモリ（SHM）を利用したクライアント追跡、リプレイ攻撃防止のための nonce カウント（nc）チェック、および認証成功後の Authentication-Info ヘッダー生成を担います。

モジュール内での役割

modules/aaa サブシステムにおける認証プロバイダーの一つです。mod_auth_basic が単純な Base64 エンコードによる認証を行うのに対し、本モジュールはよりセキュアなチャレンジ・レスポンス方式を提供します。

責務

  • **認証プロセスの管理**: check_authn フックを通じた Digest 認証の検証。
  • **共有メモリ管理**: クライアント状態（nonce カウント、opaque 値）を保持するための apr_shm および apr_rmm の管理。
  • **Nonce 生成と検証**: 時間ベースの nonce 生成と、リプレイ攻撃を防ぐための検証ロジック。
  • **ヘッダー処理**: Authorization ヘッダーのパースと、WWW-Authenticate / Authentication-Info ヘッダーの生成。

重要なシンボル

設定構造体

  • digest_config_rec: ディレクトリごとの設定（realm, qop リスト, nonce 寿命, 認証プロバイダー等）を保持。

クライアント管理

  • client_entry: 共有メモリ上のクライアント状態（nonce カウント、最後に使用した nonce）。
  • client_list: ハッシュテーブル構造。client_shm 上に配置され、クライアントの追跡に使用。
  • client_shm / client_rmm: 共有メモリセグメントとメモリ管理ハンドル。

ヘッダー処理

  • digest_header_rec: パースされた Authorization ヘッダーの内容を保持。
  • get_digest_rec(): ヘッダー文字列を解析し、構造体にマッピングする。

認証ロジック

  • authenticate_digest_user(): 認証のメインフック。ヘッダー検証、ハッシュ照合、nonce チェックを統合。
  • get_hash(): 設定された認証プロバイダーを介して、realm に紐づくパスワードハッシュを取得。

制御フローとデータフロー

1. **初期化**: initialize_module および initialize_child で共有メモリとグローバルミューテックスをセットアップ。 2. **リクエスト受信**: parse_hdr_and_update_nc が post_read_request フックで動作し、Authorization ヘッダーをパースして nonce カウントをインクリメント。 3. **認証検証**: authenticate_digest_user が呼ばれ、以下の順で検証：

  • ヘッダーの妥当性確認。
  • URI の整合性チェック。
  • get_hash によるプロバイダーからのハッシュ取得。
  • new_digest / old_digest によるレスポンスハッシュの計算と照合。
  • check_nonce による有効期限とリプレイ攻撃のチェック。

    • 4. **レスポンス**: 認証失敗時は note_digest_auth_failure で WWW-Authenticate を生成。成功時は add_auth_info で Authentication-Info を付与。

読者向けメモ

  • **メモリ所有権**: 共有メモリ上のデータは apr_rmm を介して管理されます。gc() 関数がメモリ不足時に古いエントリを削除する LRU 的な挙動をとります。
  • **制約**: qop=auth-int や MD5-sess は未実装です。
  • **エッジケース**: AuthDigestEnableQueryStringHack は MSIE のバグ対応用です。
  • **注意点**: 共有メモリを使用するため、マルチプロセス環境での同期（client_lock）が必須です。共有メモリが利用できない環境では、nonce カウントチェックなどの一部機能が制限されます。
  • **省略箇所**: 本ファイルは非常に長いため、詳細なハッシュ計算アルゴリズムの細部や、URI パースの複雑なエッジケース処理は省略しています。これらは RFC 2617 に準拠しています。
