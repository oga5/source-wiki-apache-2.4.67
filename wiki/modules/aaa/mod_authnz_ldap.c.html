modules/aaa/mod_authnz_ldap.c

概要

Apache HTTP Server の LDAP 認証・認可モジュールです。mod_ldap (util_ldap) が提供するキャッシュ機能と接続管理を利用し、LDAP ディレクトリに対する認証（パスワード照合）および認可（ユーザー、グループ、DN、属性、フィルタによる制限）を提供します。

モジュール内での役割

modules/aaa サブシステムにおいて、外部 LDAP サーバーを認証・認可のバックエンドとして統合する役割を担います。mod_authn_core や mod_authz_core が定義するプロバイダーインターフェースを実装し、HTTP リクエストの認証・認可フェーズで呼び出されます。

責務

  • **認証**: LDAP バインドによるユーザー資格情報の検証。
  • **認可**: LDAP 検索や比較操作を用いたアクセス制御（require ディレクティブの処理）。
  • **設定管理**: AuthLDAPURL 等のディレクティブを通じた LDAP 接続パラメータの保持と、ディレクトリごとの設定管理。
  • **環境変数設定**: 認可成功時に LDAP 属性をリクエスト環境変数へマッピング。
  • **文字コード変換**: charset.conv ファイルに基づくパスワードやユーザー名の UTF-8 変換。

重要なシンボル

設定構造体

  • authn_ldap_config_t: ディレクトリごとの設定（URL、ホスト、DN、フィルタ、グループ属性など）を保持。
  • authn_ldap_request_t: リクエスト単位のコンテキスト（DN、ユーザー名、パスワード、検索結果属性）。

プロバイダー関数

  • authn_ldap_check_password: 認証フェーズのメインエントリ。
  • ldapuser_check_authorization / ldapgroup_check_authorization: 各種認可プロバイダーの実装。

ユーティリティ・ヘルパー

  • authn_ldap_build_filter: LDAP 検索フィルタの動的構築とメタ文字のエスケープ。
  • get_connection_for_authz: 認可フェーズで使用する LDAP 接続の取得（ユーザー資格情報でのバインド対応）。
  • set_request_vars: LDAP 属性を subprocess_env に展開。

制御フローとデータフロー

1. **初期化**: register_hooks で認証・認可プロバイダーを登録し、ImportULDAPOptFn で mod_ldap の関数ポインタを取得。 2. **認証フロー**: authn_ldap_check_password が呼ばれると、authn_ldap_build_filter でフィルタを作成し、util_ldap_cache_checkuserid で LDAP サーバーへ問い合わせる。成功時は authn_ldap_request_t をリクエストコンテキストに保存。 3. **認可フロー**: ldap*_check_authorization 関数群が呼ばれる。認証済みでない場合は util_ldap_cache_getuserdn で DN を特定し、その後 util_ldap_cache_compare 等を用いてアクセス権を判定する。 4. **接続管理**: apr_pool_cleanup_register を使用し、リクエスト終了時に LDAP 接続を確実にクローズする。

読者向けメモ

  • **メモリ管理**: 設定は dir_config プール、リクエスト単位のデータは request_rec->pool で管理されます。
  • **スレッド安全性**: authn_ldap_config_t 内の lock は、マルチスレッド環境での設定更新時に使用されます。
  • **制約**: mod_ldap がロードされていない場合、post_config フックでエラーを返し起動を停止します。
  • **省略された詳細**: LDAP フィルタの複雑なエスケープ処理や、mod_ldap 内部のキャッシュアルゴリズムの詳細は本ファイルには含まれていません。これらは util_ldap 側に委譲されています。
  • **注意点**: AuthLDAPInitialBindAsUser を有効にする場合、ユーザーのパスワードが LDAP バインドに使用されるため、セキュリティ要件を十分に検討してください。
