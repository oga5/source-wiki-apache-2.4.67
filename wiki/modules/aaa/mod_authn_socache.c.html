modules/aaa/mod_authn_socache.c

概要

mod_authn_socache は、Apache HTTP Server の認証情報（パスワードやハッシュ）を共有オブジェクトキャッシュ（socache）に保存・取得するためのモジュールです。認証バックエンド（LDAP, DB, ファイル等）への負荷を軽減し、認証プロセスを高速化します。

モジュール内での役割

modules/aaa サブシステムにおいて、認証プロバイダーのキャッシュ層として機能します。他の認証モジュールが ap_authn_cache_store を呼び出すことで、認証成功時の結果をキャッシュし、次回以降の認証リクエストをキャッシュから解決します。

責務

  • **キャッシュ管理**: socache プロバイダーを介した認証情報の永続化と有効期限管理。
  • **設定管理**: ディレクトリ単位でのキャッシュ対象プロバイダーの指定、タイムアウト設定、コンテキスト定義。
  • **同期制御**: グローバルミューテックスによるキャッシュアクセス時の排他制御。
  • **認証インターフェース**: authn_provider を実装し、check_password および get_realm_hash を提供。

重要なシンボル

設定・ライフサイクル

  • authn_cache_dircfg: ディレクトリごとの設定構造体（タイムアウト、対象プロバイダー、コンテキスト）。
  • authn_cache_cmds: AuthnCacheSOCache 等のディレクティブ定義。
  • authn_cache_precfg / post_config: モジュール初期化、ミューテックス登録、キャッシュプロバイダーのバインド。

キャッシュ操作

  • ap_authn_cache_store: 外部モジュールから呼び出されるキャッシュ保存用関数（APR_REGISTER_OPTIONAL_FN で公開）。
  • check_password: authn_provider の実装。キャッシュからパスワードを取得し apr_password_validate で検証。
  • get_realm_hash: authn_provider の実装。ダイジェスト認証用のハッシュを取得。

グローバル状態

  • authn_cache_mutex: キャッシュへの書き込みを保護するグローバルミューテックス。
  • socache_provider / socache_instance: 選択されたキャッシュバックエンドの実装とインスタンス。

制御フローとデータフロー

1. **初期化**: post_config で socache プロバイダーを初期化し、グローバルミューテックスを作成。 2. **保存フロー**:

  • 認証成功後、ap_authn_cache_store が呼ばれる。
  • construct_key でコンテキストに応じた一意なキーを生成。
  • ミューテックスを trylock し、成功した場合のみ socache_provider->store を実行。

    • 3. **取得フロー**:

  • check_password 等が呼ばれると、キーを生成し socache_provider->retrieve を実行。
  • 取得した値を apr_password_validate に渡し、結果を返す。

読者向けメモ

  • **メモリ所有権**: r->pool を多用しており、リクエスト終了時にキャッシュ関連のメモリも解放される設計です。
  • **排他制御の注意**: check_password ではパフォーマンスを優先し、読み取り時のミューテックスロックを行っていません。これは socache 実装がスレッドセーフであることを前提としています。
  • **制約**: AuthnCacheSOCache はグローバル設定でのみ有効です。
  • **未実装・課題**: ソース内の FIXME にある通り、キャッシュとミューテックスは本来グローバルであるべきですが、現在の実装では設定の柔軟性と引き換えに複雑なライフサイクル管理を伴っています。また、retrieve 時の NOTFOUND エラーハンドリングに一部不整合がある可能性があります。
