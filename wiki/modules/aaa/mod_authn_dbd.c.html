modules/aaa/mod_authn_dbd.c

概要

mod_authn_dbd.c は、Apache HTTP Server の認証（Authentication）フェーズにおいて、データベース（DBD）をバックエンドとして利用するための認証プロバイダーを提供します。mod_dbd が提供するデータベース接続機能を利用し、SQL クエリを通じてユーザーのパスワードやレルム情報を検証します。

モジュール内での役割

modules/aaa サブシステムにおいて、認証プロバイダー（authn）の一種として機能します。mod_authn_core が管理する認証プロバイダーフレームワークに「dbd」という名前で登録され、設定ファイルで AuthBasicProvider dbd のように指定された際に呼び出されます。

責務

  • **設定管理**: ユーザー認証およびレルム認証のための SQL クエリを保持するディレクトリ設定の管理。
  • **認証プロバイダーの実装**: authn_provider インターフェースを実装し、パスワード検証およびレルムハッシュ取得を行う。
  • **環境変数注入**: クエリ結果の追加カラムを subprocess_env に展開し、後続の処理で利用可能にする。
  • **キャッシュ連携**: mod_authn_socache 等のキャッシュモジュールと連携し、認証結果のキャッシュを試みる。

重要なシンボル

設定構造体とコマンド

  • authn_dbd_conf: ディレクトリ単位の設定（SQL クエリ文字列）を保持。
  • authn_dbd_cmds: AuthDBDUserPWQuery および AuthDBDUserRealmQuery ディレクティブを定義。

認証プロバイダー関数

  • authn_dbd_password: AuthBasic 認証用。データベースからパスワードを取得し、apr_password_validate で検証。
  • authn_dbd_realm: AuthDigest 認証用。データベースからレルムハッシュを取得。

ライフサイクル・フック

  • authn_dbd_hooks: モジュールの初期化時にプロバイダーを登録し、オプション関数（キャッシュ用）を取得するフックを設定。
  • authn_dbd_cr_conf / authn_dbd_merge_conf: 設定の生成とマージ。

制御フローとデータフロー

1. **初期化**: authn_dbd_hooks がプロバイダーを登録。opt_retr でキャッシュ用関数を遅延解決。 2. **設定**: authn_dbd_prepare が SQL クエリを解析し、mod_dbd の準備済みステートメントとして登録。 3. **認証実行**:

  • authn_dbd_acquire_fn を介して mod_dbd から接続を取得。
  • apr_dbd_pvselect でクエリを実行。
  • 結果セットをループし、最初のカラムを認証情報として使用。
  • 残りのカラムは AUTHENTICATE_ プレフィックスを付与して環境変数にセット。
  • AUTHN_CACHE_STORE マクロ経由で結果をキャッシュへ保存。

直接依存

  • mod_dbd.h / apr_dbd.h: データベース接続およびクエリ実行の基盤。
  • mod_auth.h: 認証プロバイダーインターフェースの定義。
  • httpd.h / http_config.h: Apache モジュール基盤。

関連コード

  • modules/database/mod_dbd.c: データベース接続管理の提供元。
  • modules/aaa/mod_authn_core.c: 認証プロバイダーのディスパッチ管理。
  • modules/aaa/mod_authn_socache.c: 認証結果のキャッシュ機能。

読者向けメモ

  • **メモリ管理**: すべてのクエリ結果や一時文字列は request_rec のプール（r->pool）から割り当てられます。リクエスト終了時に自動解放されるため、明示的な解放は不要です。
  • **エラーパス**: データベース接続失敗やクエリ実行失敗は AUTH_GENERAL_ERROR を返し、認証プロセスを中断させます。
  • **制約**: AuthDBDUserPWQuery 等のクエリは、必ず最初のカラムにパスワード/ハッシュを返す必要があります。
  • **省略された詳細**: SQL インジェクション対策は apr_dbd_pvselect のパラメータバインディングに依存しています。本ファイル内ではクエリの構文解析は行わず、mod_dbd に委譲しています。
