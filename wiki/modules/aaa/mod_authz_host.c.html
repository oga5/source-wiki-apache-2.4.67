modules/aaa/mod_authz_host.c

概要

mod_authz_host.c は、Apache HTTP Server におけるホストベースおよびIPベースのアクセス制御を提供する認証モジュールです。Require ディレクティブを通じて、クライアントの IP アドレス、ホスト名、またはローカル接続判定に基づいた認可判断を行います。

モジュール内での役割

mod_authz_core が提供する認可プロバイダーインターフェースを実装するプラグインの一つです。他の認証モジュール（mod_authn_*）がユーザー認証を担うのに対し、本モジュールはネットワーク層のメタデータに基づいたアクセス制限を担当します。

責務

  • **IP アドレス制限**: CIDR 表記や単一 IP によるアクセス制御。
  • **ホスト名制限**: DNS 逆引き結果を用いたドメインマッチング。
  • **Forward DNS 制限**: クライアント IP が指定ホスト名の正引き結果に含まれるかの検証。
  • **ローカル接続判定**: リクエストがサーバー自身からのものかどうかの判定。
  • **設定の最適化**: 複数の仮想ホストで同一サブネットが指定された際、メモリ節約のためにハッシュテーブルを用いてサブネット情報を共有する。

重要なシンボル

プロバイダー定義 (authz_provider)

  • authz_ip_provider: ip プロバイダー。ip_check_authorization を使用。
  • authz_host_provider: host プロバイダー。host_check_authorization を使用。
  • authz_forward_dns_provider: forward-dns プロバイダー。forward_dns_check_authorization を使用。
  • authz_local_provider: local プロバイダー。local_check_authorization を使用。

設定解析・検証

  • ip_parse_config: Require ip の引数を解析し、apr_ipsubnet_t の配列に変換。
  • host_parse_config: Require host 等の引数を ap_expr_info_t（式）としてコンパイル。

内部ユーティリティ

  • in_domain: ホスト名が指定ドメインのサブドメインであるかを判定する文字列比較関数。
  • parsed_subnets: 設定解析フェーズで使用される、サブネット共有用の一時的なハッシュテーブル。

制御フローとデータフロー

1. **初期化**: authz_host_pre_config で parsed_subnets を初期化し、ローカルホスト用サブネットを登録。post_config でハッシュを破棄し、.htaccess 等での誤用を防ぐ。 2. **設定時**: parse_config 関数群が Require 行を解析し、認可時に使用する構造体（apr_ipsubnet_t 配列や ap_expr_info_t）を生成。 3. **認可時**: check_authorization 関数群がリクエストの r->useragent_addr を取得し、解析済みデータと照合。AUTHZ_GRANTED または AUTHZ_DENIED を返す。

読者向けメモ

  • **メモリ管理**: parsed_subnets は設定解析中（ptemp プール）のみ有効です。実行時の認可判断には影響しません。
  • **DNS 依存**: host プロバイダーは DNS 逆引き（REMOTE_DOUBLE_REV）に依存しており、DNS サーバーの応答速度や設定にパフォーマンスが左右されます。
  • **エッジケース**: forward-dns は正引き結果のリストを走査するため、DNS 構成によっては複数の IP がマッチする可能性があります。
  • **省略された詳細**: ap_expr の詳細な構文解析ロジックは core モジュール側に委譲されているため、本ファイルでは評価結果の利用に焦点を当てています。
