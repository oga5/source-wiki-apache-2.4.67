modules/aaa/mod_authz_groupfile.c

概要

mod_authz_groupfile.c は、Apache HTTP Server においてテキストファイルベースのグループ認証を提供するモジュールです。AuthGroupFile ディレクティブで指定されたファイルを参照し、require group および require file-group によるアクセス制御を実現します。

モジュール内での役割

modules/aaa サブシステムの一部として、認証・認可（Authz）プロバイダーを提供します。mod_authz_core が提供する認可フレームワークにプラグインとして登録され、ファイルシステム上のグループ定義とリクエストユーザーを照合します。

責務

  • **設定管理**: AuthGroupFile ディレクティブの解析とディレクトリ単位の設定保持。
  • **認可プロバイダーの提供**:
  • group: 指定されたグループ名とユーザーの所属を照合。
  • file-group: リクエスト対象ファイルの所有グループとユーザーの所属を照合。
  • **ファイル解析**: グループ定義ファイル（group: user1 user2 ... 形式）の読み込みとパース。

重要なシンボル

設定・ライフサイクル

  • authz_groupfile_config_rec: ディレクトリごとの設定構造体。groupfile パスを保持。
  • create_authz_groupfile_dir_config: 設定構造体の初期化関数。
  • authz_groupfile_cmds: AuthGroupFile ディレクティブを登録するコマンドテーブル。

認可ロジック

  • group_check_authorization: require group の認可処理。式評価を行い、ユーザーが指定グループに含まれるか判定。
  • filegroup_check_authorization: require file-group の認可処理。mod_authz_owner から取得したファイルグループとユーザーの所属を照合。
  • groups_for_user: 指定ユーザーが所属する全グループを抽出し、apr_table_t に格納するヘルパー。

プロバイダー登録

  • authz_group_provider / authz_filegroup_provider: 認可プロバイダーのインターフェース定義。
  • register_hooks: モジュール起動時にプロバイダーを登録し、authz_owner_get_file_group を取得するフック。

制御フローとデータフロー

1. **初期化**: register_hooks が authz_owner_get_file_group を取得し、認可プロバイダーを登録。 2. **認可実行**:

  • group_check_authorization が呼ばれると、groups_for_user を呼び出し、グループファイルを走査。
  • groups_for_user は ap_varbuf を使用して行単位でファイルを読み込み、ユーザー名に一致するグループを apr_table_t にキャッシュ。
  • 最後に ap_expr_str_exec で評価された要求グループ名と、キャッシュされたテーブルを照合。

    • 3. **ファイルグループ判定**: filegroup_check_authorization は authz_owner_get_file_group を呼び出し、対象ファイルの所有グループを取得した上で、同様に groups_for_user の結果と照合。

直接依存

  • mod_auth.h: 認証・認可の共通基盤。
  • mod_authz_owner.h: file-group 判定に必要なオプション関数を提供。
  • util_varbuf.h: 大規模なファイル行読み込みのためのバッファ管理。
  • http_config.h, http_core.h: 設定管理およびディレクティブ処理。

関連コード

  • mod_authz_core.c: 認可プロバイダーの呼び出し元。
  • mod_authz_owner.c: file-group の判定元となるファイル所有者情報の提供元。

読者向けメモ

  • **メモリ管理**: groups_for_user 内では apr_pool_t を使用し、一時的な文字列操作やテーブル生成を行っています。sp プールはループごとにクリアされ、メモリ消費を抑制しています。
  • **パフォーマンス**: グループファイルはリクエストごとにオープン・走査されます。大規模なグループファイルの場合、I/O負荷が高くなる可能性があるため注意が必要です。
  • **制約**: groupfile_parse_config は ap_expr を使用しており、動的なグループ指定が可能です。
  • **省略箇所**: エラーハンドリングの詳細や、ap_varbuf の内部実装の詳細は省略しています。これらは Apache HTTP Server の標準的なユーティリティであるためです。
