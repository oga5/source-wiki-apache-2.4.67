modules/aaa/mod_authn_core.c

概要

mod_authn_core.c は、Apache HTTP Server の認証（Authentication）サブシステムの基盤となるモジュールです。主に、認証設定の管理、認証プロバイダーのエイリアス機能（<AuthnProviderAlias>）、および認証タイプ（AuthType）やレルム名（AuthName）の動的な評価を担います。

モジュール内での役割

modules/aaa ディレクトリにおいて、認証プロバイダー（mod_authn_file 等）と Apache コアの仲介役を果たします。認証の実行そのものを行うのではなく、認証設定の構造化と、プロバイダーの抽象化レイヤーを提供します。

責務

  • **設定管理**: AuthType および AuthName のディレクティブを解析し、ap_expr を用いてリクエスト時に動的に評価可能な形式で保持する。
  • **プロバイダーエイリアス**: <AuthnProviderAlias> ブロックを処理し、既存の認証プロバイダーに対して別名を付与し、かつ特定のディレクトリ設定をカプセル化して適用する仕組みを提供する。
  • **認証フックの提供**: 認証タイプが設定されているが対応するモジュールがない場合のフォールバック処理（authenticate_no_user）を行う。

重要なシンボル

設定構造体

  • authn_core_dir_conf: ディレクトリごとの設定。AuthType と AuthName を ap_expr_info_t として保持。
  • authn_alias_srv_conf: サーバー全体の設定。定義されたプロバイダーエイリアスをハッシュテーブルで管理。
  • provider_alias_rec: エイリアスされたプロバイダーの定義。元のプロバイダーへの参照と、エイリアス内で有効なディレクトリ設定（sec_auth）を保持。

プロバイダー管理

  • authn_alias_check_password: エイリアス経由の認証実行関数。一時的に r->per_dir_config をエイリアスの設定に差し替えてから、元のプロバイダーを呼び出す。
  • authn_alias_get_realm_hash: エイリアス経由のレルムハッシュ取得関数。同様に設定を差し替えて実行。

ディレクティブ処理

  • authaliassection: <AuthnProviderAlias> ブロックの解析と登録を行う。
  • set_authname / set_authtype: AuthName / AuthType の解析と ap_expr への変換。

制御フローとデータフロー

1. **設定時**: authaliassection が呼び出されると、内部で ap_walk_config を実行し、ブロック内の設定を new_auth_config に蓄積します。これを provider_alias_rec に格納し、ap_register_auth_provider でエイリアス名を登録します。 2. **実行時**: 認証が必要なリクエストが発生すると、authn_alias_check_password 等が呼び出されます。

  • r->notes からプロバイダー名を取得。
  • authn_alias_srv_conf から該当する provider_alias_rec を検索。
  • r->per_dir_config を一時的に prvdraliasrec->sec_auth に差し替え（ap_merge_per_dir_configs）、元のプロバイダー関数を実行。
  • 終了後、r->per_dir_config を復元します。

読者向けメモ

  • **メモリ管理**: apr_pool_t を使用しており、リクエストごとの設定差し替えは r->pool を利用して安全に行われています。
  • **不変条件**: AuthnProviderAlias は GLOBAL_ONLY であり、サーバー起動時に確定します。
  • **制限事項**: r->ap_auth_type や r->ap_auth_name への直接アクセスは非推奨であり、本モジュールが提供する authn_ap_auth_type 等のオプション関数経由でのアクセスが推奨されています。
  • **省略箇所**: ap_expr の詳細な構文解析ロジックは ap_expr サブシステムに委譲されているため、本ファイルでは解析結果の保持と実行のみを扱っています。
