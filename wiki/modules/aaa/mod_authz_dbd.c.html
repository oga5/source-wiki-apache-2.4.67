modules/aaa/mod_authz_dbd.c

概要

mod_authz_dbd.c は、Apache HTTP Server においてデータベース（DBD）をバックエンドとした認可（Authorization）機能を提供するモジュールです。ユーザーのグループ所属確認、ログイン/ログアウト処理を SQL クエリを通じて実行します。

モジュール内での役割

mod_authz_core が提供する認可プロバイダーインターフェースを実装するプラグインです。mod_dbd に依存しており、データベース接続の取得やプリペアドステートメントの管理を mod_dbd のオプション関数経由で行います。

責務

  • **設定管理**: ディレクトリ単位の SQL クエリ設定（AuthzDBDQuery 等）の保持とマージ。
  • **認可プロバイダーの実装**: dbd-group（グループ判定）、dbd-login（ログイン処理）、dbd-logout（ログアウト処理）の提供。
  • **セッション管理**: ログイン成功時に client_login フックを呼び出し、他モジュール（mod_auth_cookie 等）と連携。
  • **リダイレクト処理**: ログイン成功時のリダイレクト先を SQL クエリから取得または Referer から決定。

重要なシンボル

設定管理

  • authz_dbd_cfg: ディレクトリごとの設定構造体。クエリ文字列とリダイレクト設定を保持。
  • authz_dbd_cr_cfg / authz_dbd_merge_cfg: 設定の初期化および親ディレクトリからの継承・マージ関数。
  • authz_dbd_prepare: AuthzDBDQuery 等のディレクティブを処理し、mod_dbd にクエリを登録する関数。

プロバイダー実装

  • dbdgroup_check_authorization: Require dbd-group の評価。SQL で取得したグループリストと照合。
  • dbdlogin_check_authorization / dbdlogout_check_authorization: ログイン/ログアウトの実行。内部で authz_dbd_login を呼び出す。

内部ヘルパー

  • authz_dbd_login: ログイン/ログアウトの SQL 実行とリダイレクト処理を行う共通ロジック。
  • authz_dbd_group_query: ユーザーの所属グループをデータベースから取得するクエリ実行関数。

制御フローとデータフロー

1. **初期化**: authz_dbd_hooks で 3 つのプロバイダーを登録。 2. **設定**: authz_dbd_prepare が mod_dbd の ap_dbd_prepare を呼び出し、SQL クエリをプリペアドステートメントとして登録。 3. **認可実行**:

  • dbdgroup_check_authorization は authz_dbd_group_query を呼び出し、結果を apr_array_header_t に格納して比較。
  • dbdlogin_check_authorization は authz_dbd_login を呼び出し、SQL 実行後に client_login フックをトリガー。

    • 4. **データフロー**: request_rec のプールを使用してメモリを管理。SQL 結果セットは apr_dbd_results_t を介して反復処理されます。

読者向けメモ

  • **メモリ管理**: すべての動的メモリ割り当て（クエリ結果、グループ配列）は r->pool に紐付いており、リクエスト終了時に自動解放されます。
  • **エラー処理**: データベース接続が失敗した場合やクエリが未設定の場合は HTTP_INTERNAL_SERVER_ERROR を返します。
  • **制約**: dbd_prepare は NOT_IN_HTACCESS コンテキストで実行されるため、.htaccess 内でのクエリ定義は許可されません。
  • **未実装/浅い部分**: client_login フックの設計意図についてソース内に疑問が投げられており、汎用的なセッション管理ではなく本モジュール固有のロジックに依存している点に注意してください。
