modules/aaa/mod_authz_dbm.c

概要

mod_authz_dbm.c は、Apache HTTP Server において DBM ファイルベースのグループ認可機能を提供するモジュールです。ユーザーが所属するグループ情報を DBM ファイルから読み取り、require dbm-group または require dbm-file-group ディレクティブに基づいたアクセス制御を行います。

モジュール内での役割

modules/aaa サブシステムの一部として、認証後の認可フェーズを担当します。mod_authz_core が提供する認可プロバイダーインターフェースを実装し、外部の DBM データベースと連携して認可判断を下す役割を担います。

責務

  • **設定管理**: AuthDBMGroupFile および AuthzDBMType ディレクティブによる DBM ファイルパスと形式の定義。
  • **DBM 検索**: ユーザー名（およびレルム）をキーとして、DBM ファイルからグループ情報を抽出する。
  • **認可ロジック**:
  • dbm-group: 指定されたグループ名とユーザーの所属グループを照合。
  • dbm-file-group: ファイルの所有グループとユーザーの所属グループを照合（mod_authz_owner に依存）。

重要なシンボル

設定構造体

  • authz_dbm_config_rec: ディレクトリごとの設定（DBM ファイルパス、DBM タイプ）を保持。

認可プロバイダー (Provider)

  • authz_dbmgroup_provider: require dbm-group 用のハンドラ。
  • authz_dbmfilegroup_provider: require dbm-file-group 用のハンドラ。

内部ヘルパー

  • get_dbm_grp: DBM ファイルを開き、ユーザーに対応するグループ文字列を抽出するコア関数。
  • get_dbm_entry_as_str: DBM から特定のキーに対応する値を文字列として取得する低レベルラッパー。

制御フローとデータフロー

1. **初期化**: register_hooks で認可プロバイダーを登録し、authz_owner_get_file_group 関数をオプション関数として取得。 2. **認可実行**:

  • dbmgroup_check_authorization が呼ばれると、get_dbm_grp を使用して DBM からグループリストを取得。
  • get_dbm_grp は apr_dbm_open2 (または open_ex) を使用して読み取り専用でファイルを開き、ユーザー名または user:realm をキーに検索。
  • 取得したグループ文字列をカンマ区切りでパースし、要求されたグループと一致するか検証。

    • 3. **終了**: apr_dbm_close でリソースを解放。

読者向けメモ

  • **メモリ管理**: すべてのメモリ割り当ては request_rec のプール (r->pool) を使用しており、リクエスト終了時に自動解放されます。
  • **DBM 形式**: DBM ファイル内の値に : が含まれる場合、key=username value=password:groupname のような形式を想定した特殊なパースが行われます。
  • **パフォーマンス**: DBM ファイルはリクエストごとに開閉されます。高負荷環境ではキャッシュの検討が必要です。
  • **省略箇所**: DBM の具体的なファイルフォーマット（Berkeley DB, GDBM 等）の詳細は apr_dbm に委譲されているため、本ソースコードでは詳細なバイナリレイアウトには触れていません。
