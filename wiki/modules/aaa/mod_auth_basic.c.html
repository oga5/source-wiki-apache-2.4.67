modules/aaa/mod_auth_basic.c

概要

HTTP Basic 認証を実装する Apache HTTP Server モジュールです。クライアントから送信された Authorization ヘッダーをデコードし、設定された認証プロバイダーを通じてユーザーの検証を行います。また、バックエンドへの転送用に認証情報を偽装（Fake）する機能も提供します。

モジュール内での役割

mod_authn_core 等の認証基盤と連携し、HTTP プロトコルレベルでの Basic 認証フローを担当します。認証の成否判定を mod_authn 系のプロバイダーに委譲する「認証の仲介役」として機能します。

責務

  • **設定管理**: ディレクトリ単位の認証設定（プロバイダーリスト、権限委譲フラグ、偽装設定）の保持とマージ。
  • **認証実行**: Authorization ヘッダーの解析、Base64 デコード、およびプロバイダーへの照会。
  • **認証失敗時の通知**: WWW-Authenticate ヘッダーの生成。
  • **認証情報の偽装**: AuthBasicFake ディレクティブに基づく、リクエストヘッダーの動的な書き換え。

重要なシンボル

設定構造体

  • auth_basic_config_rec: ディレクトリごとの設定を保持。プロバイダーリスト、権限委譲フラグ、偽装用式（ap_expr）を管理。

設定ハンドラ（ディレクティブ）

  • add_authn_provider: AuthBasicProvider を処理し、プロバイダーを検索・キャッシュ。
  • set_authoritative: AuthBasicAuthoritative を設定。
  • add_basic_fake: AuthBasicFake を解析し、式をコンパイル。
  • set_use_digest_algorithm: AuthBasicUseDigestAlgorithm を設定。

認証フック・ロジック

  • authenticate_basic_user: check_authn フックのメイン関数。認証フローの核心。
  • authenticate_basic_fake: fixups フックで実行され、認証ヘッダーを上書き。
  • note_basic_auth_failure: WWW-Authenticate ヘッダーをレスポンスに追加。

制御フローとデータフロー

1. **認証開始**: authenticate_basic_user がリクエストをフックし、Authorization ヘッダーを get_basic_auth で取得・デコード。 2. **プロバイダー照会**: 設定された authn_provider_list を順次走査。プロバイダーが未設定の場合はデフォルトプロバイダーを使用。 3. **検証**: check_password または get_realm_hash（Digest アルゴリズム指定時）を呼び出し、結果を判定。 4. **権限委譲**: authoritative がオフの場合、認証失敗時に DECLINED を返し、後続の認証モジュールに処理を委ねる。 5. **偽装**: authenticate_basic_fake が ap_expr を評価し、成功すれば Authorization ヘッダーを再構築して後続の処理へ渡す。

読者向けメモ

  • **メモリ管理**: すべてのメモリ割り当ては request_rec のプール（r->pool）または設定プール（cmd->pool）で行われます。明示的な解放は不要です。
  • **不変条件**: authoritative が 1 の場合、認証失敗は即座に HTTP_UNAUTHORIZED を返します。
  • **エッジケース**: AuthBasicFake が空の式を評価した場合、認証ヘッダーは削除され、認証はスキップされます。
  • **省略箇所**: 本ファイルは認証の「仲介」に特化しており、実際のパスワード照合ロジック（DB照会やファイル読み込み）は mod_authn_xxx モジュール側に隠蔽されています。
