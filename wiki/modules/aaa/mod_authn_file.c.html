modules/aaa/mod_authn_file.c

概要

mod_authn_file.c は、Apache HTTP Server における認証プロバイダーの一つであり、テキストファイル（AuthUserFile）に保存されたユーザー名とパスワード情報を使用して認証を行うためのモジュールです。

モジュール内での役割

mod_authn_core が提供する認証フレームワークに対し、「file」という名前の認証プロバイダーとして機能します。認証が必要なリクエストが発生した際、設定されたファイルから資格情報を検索し、認証の可否を判定します。

責務

  • **設定管理**: AuthUserFile ディレクティブを解析し、パスワードファイルへのパスを保持する。
  • **認証処理**: 指定されたファイルからユーザー名とパスワード（またはハッシュ）を読み取り、照合を行う。
  • **キャッシュ連携**: 認証結果を ap_authn_cache_store（存在する場合）へ通知し、後続の認証処理を高速化する。

重要なシンボル

  • authn_file_config_rec: パスワードファイルのパスを保持するディレクトリごとの設定構造体。
  • check_password: 基本認証（Basic Auth）で使用されるパスワード照合関数。
  • get_realm_hash: ダイジェスト認証（Digest Auth）で使用されるハッシュ取得関数。
  • authn_file_provider: mod_authn_core に登録される関数ポインタのセット。

制御フローとデータフロー

1. **初期化**: register_hooks が ap_register_auth_provider を呼び出し、「file」プロバイダーを登録します。 2. **リクエスト処理**:

  • 認証が必要なリクエストに対し、check_password または get_realm_hash が呼び出されます。
  • ap_pcfg_openfile を使用して設定されたパスワードファイルを開きます。
  • ap_cfg_getline で行単位に読み込み、ap_getword でユーザー名とパスワードを分割・抽出します。
  • ユーザー名が一致した場合、apr_password_validate でパスワードを検証するか、ハッシュ値を返します。

    • 3. **キャッシュ**: 成功時に AUTHN_CACHE_STORE マクロを通じて結果をキャッシュ層へ渡します。

  • apr_strings.h, apr_md5.h: 文字列操作およびパスワード検証用ユーティリティ。

関連コード

  • mod_authn_core.c: 認証プロバイダーの登録と呼び出しを管理する上位モジュール。
  • mod_auth_basic.c / mod_auth_digest.c: 本モジュールが提供するプロバイダーを利用する認証方式の実装。

読者向けメモ

  • 本モジュールは非常にシンプルで、ファイルベースの認証の標準的な実装例です。
  • パスワードファイルはプレーンテキストであるため、セキュリティ上の観点からファイルへのアクセス権限管理が重要です。
  • ap_cfg_getline を使用した行単位の読み込みは、ファイルサイズが大きくなるとパフォーマンスに影響を与える可能性があるため、大規模なユーザーリストには mod_authn_dbm や mod_authn_dbd の利用が推奨されます。
