modules/aaa/mod_allowmethods.c

概要

mod_allowmethods.c は、Apache HTTP Server において特定の HTTP メソッド（GET, POST 等）の実行をディレクトリ単位で制限するためのモジュールです。

モジュール内での役割

modules/aaa サブシステムの一部として、リクエスト処理の初期段階でアクセス制御を行うフィルタリング機能を提供します。

責務

  • AllowMethods ディレクティブによる許可メソッドリストの管理。
  • リクエストされたメソッドが許可リストに含まれているかの検証。
  • 許可されていないメソッドに対する 405 Method Not Allowed レスポンスの生成。

重要なシンボル

  • am_conf_t: ディレクトリごとの設定を保持する構造体。許可されたメソッドのビットマスクを管理します。
  • am_check_access: access_checker フックに登録される関数。リクエストのメソッドを検証します。
  • am_allowmethods: AllowMethods ディレクティブのハンドラ。設定値を解析し、ビットマスクを生成します。

制御フローとデータフロー

1. **設定**: am_allowmethods が設定ファイルからメソッド名を受け取り、am_conf_t にビットマスクとして格納します。 2. **フック**: am_register_hooks により、access_checker フックに am_check_access が APR_HOOK_REALLY_FIRST（最優先）で登録されます。 3. **検証**: リクエスト受信時、am_check_access が現在の設定を取得します。

  • 設定が空であれば DECLINED を返し、後続の処理に委ねます。
  • メソッドが許可リストにない場合、HTTP_METHOD_NOT_ALLOWED を返し、リクエストを拒否します。

直接依存

  • httpd.h, http_core.h, http_config.h: Apache モジュール開発の基本ヘッダー。
  • http_protocol.h, http_request.h: リクエスト情報およびメソッド番号の取得に使用。
  • apr_strings.h: 文字列操作用。
  • http_log.h

関連コード

  • http_core.c: Apache 標準のメソッド制限機能との比較対象。
  • mod_authz_core.c: アクセス制御のフレームワークとして関連します。

読者向けメモ

  • 本モジュールは APR_HOOK_REALLY_FIRST を使用しているため、他のアクセス制御モジュールよりも先に評価されます。
  • AllowMethods reset を指定すると、そのディレクトリでの制限を無効化（全許可）できます。
  • 許可リストにないメソッドがリクエストされた場合、ログにエラーが出力されます。
