modules/mappers

システム内での役割

modules/mappers は、Apache HTTP Server におけるリクエストの「翻訳（Translation）」および「修正（Fixup）」フェーズを担うモジュール群です。クライアントから送られた URI を物理的なファイルパスへ変換したり、リクエストの属性に基づいて動的なリダイレクトやハンドラの振り分けを行ったりする、サーバーのルーティング層を構成します。

責務

  • **URI マッピング**: URI をファイルシステム上のパスや外部リダイレクト先へ変換する。
  • **リクエスト調停**: コンテンツネゴシエーションやディレクトリインデックスの解決を行う。
  • **動的制御**: 正規表現による書き換え、スペル修正、バーチャルホストの動的決定、MIME タイプに応じたスクリプト実行などを提供する。

このモジュールのファイル

本モジュールは以下の 10 個のソースファイルで構成されています（ディレクトリ境界に基づき集約）。

  • mod_actions.c: MIME タイプ/メソッドに基づくスクリプト実行。
  • mod_alias.c: URI のパス変換およびリダイレクト。
  • mod_dir.c: ディレクトリインデックスおよびスラッシュ補完。
  • mod_imagemap.c: 画像マップに基づくリダイレクト。
  • mod_negotiation.c: コンテンツネゴシエーション（多言語・多形式）。
  • mod_rewrite.c / mod_rewrite.h: 高度な正規表現ベースの書き換えエンジン。
  • mod_speling.c: URL のスペルミス自動修正。
  • mod_userdir.c: ユーザーホームディレクトリへのマッピング。
  • mod_vhost_alias.c: 動的バーチャルホストのパス生成。

主要なデータ構造

各モジュールは独自のコンテキスト構造体を持ち、設定の保持やリクエスト処理に使用します。

  • **書き換え**: rewrite_server_conf, rewrite_perdir_conf
  • **エイリアス**: alias_entry, alias_dir_conf
  • **ネゴシエーション**: neg_dir_config, var_rec
  • **ディレクトリ**: dir_config_rec
  • **その他**: imap_conf_rec, action_dir_config, userdir_config, mva_sconf_t

主なフロー

1. **翻訳フェーズ (translate_name)**: mod_alias, mod_rewrite, mod_userdir, mod_vhost_alias が連携し、URI を物理パスへ変換。 2. **修正フェーズ (fixups)**: mod_dir (インデックス解決), mod_speling (スペル修正), mod_negotiation (バリアント選択) がリクエストを調整。 3. **ハンドラ実行**: mod_actions や mod_imagemap が特定の条件でリクエストをインターセプトし、内部リダイレクトやスクリプト実行を行う。

外部依存

Apache HTTP Server のコア API (httpd.h, http_config.h, http_request.h) および APR (apr_strings.h, apr_lib.h 等) に強く依存しています。また、mod_rewrite は ap_expr.h を介して式評価エンジンと連携します。

内部構成

各ソースファイルは独立したモジュールとして実装されており、Apache のフック機構を通じてリクエスト処理パイプラインに統合されます。

推奨読書順

1. mod_rewrite.h / mod_rewrite.c (最も複雑で強力なマッピング機能) 2. mod_negotiation.c (コンテンツ選択のロジック) 3. mod_imagemap.c (幾何学的判定の例) 4. mod_alias.c (基本的なマッピング) 5. mod_dir.c (ディレクトリ処理の標準) 6. mod_userdir.c (ユーザー単位のパス変換) 7. mod_speling.c (補完ロジック) 8. mod_vhost_alias.c (動的ホスト処理) 9. mod_actions.c (ハンドラ振り分け)

未解決の問い

  • 各モジュール間での translate_name フックの実行順序が競合した場合の挙動と、その優先順位付けの最適化。
  • 大規模な設定ファイルにおける mod_rewrite や mod_alias の線形探索によるパフォーマンスへの影響。
