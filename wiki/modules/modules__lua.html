modules/lua

システム内での役割

modules/lua は、Apache HTTP Server に Lua スクリプト実行環境を統合するモジュールです。HTTP リクエスト処理の各フェーズ（ハンドラ、フィルタ、認証など）を Lua スクリプトに委譲し、Apache 内部のデータ構造やデータベース操作を Lua から安全に利用可能にします。

責務

  • **VM ライフサイクル管理**: リクエスト、接続、サーバー単位での Lua 仮想マシン（VM）の生成・キャッシュ・破棄。
  • **データバインディング**: Apache の request_rec や apr_table_t などの C 構造体を Lua のオブジェクトとして公開。
  • **設定統合**: Apache 設定ディレクティブと Lua スクリプトのハンドラマッピングの管理。
  • **機能拡張**: パスワードハッシュ生成やデータベース操作（DBD）などのユーティリティ提供。

このモジュールのファイル

  • **コア**: mod_lua.c, mod_lua.h
  • **VM 管理**: lua_vmprep.c, lua_vmprep.h
  • **リクエスト・データ**: lua_request.c, lua_request.h, lua_apr.c, lua_apr.h
  • **設定・構成**: lua_config.c, lua_config.h
  • **ユーティリティ**: lua_dbd.c, lua_dbd.h, lua_passwd.c, lua_passwd.h

主要なデータ構造

  • ap_lua_vm_spec: Lua VM の生成パラメータとスコープ定義。
  • ap_lua_dir_cfg: ディレクトリ単位の Lua 設定（ハンドラやキャッシュ戦略）。
  • request_rec: Apache のリクエストコンテキスト（Lua へバインドされる主要対象）。
  • req_table_t: apr_table_t を Lua から操作するためのラッパー構造体。
  • passwd_ctx: パスワードハッシュ生成用のコンテキスト。
  • lua_db_handle: データベース接続状態を保持する構造体。

主なフロー

1. **初期化**: mod_lua がフックを登録し、設定に基づいて ap_lua_vm_spec を構築。 2. **VM 取得**: リクエスト発生時、ap_lua_get_lua_state がスコープに応じた VM をキャッシュから取得または新規生成。 3. **ディスパッチ**: Lua スクリプトが Apache API を呼び出すと、メタテーブル（Apache2.Request 等）を介して C 関数が実行される。 4. **データ変換**: apr_table_t 等の C 構造体と Lua テーブル間での相互変換が行われ、処理結果が Apache へ返却される。

外部依存

  • **Lua**: lua.h, lauxlib.h, lualib.h (コア言語機能)
  • **APR**: apr_pools.h, apr_tables.h, apr_dbd.h 等 (メモリ管理、テーブル操作、DB 抽象化)
  • **Apache Core**: httpd.h, http_config.h, http_request.h (サーバーコンテキスト)

内部構成

本モジュールは、Apache のフックシステムを Lua に接続する「ハーネス」と、C 構造体を Lua に公開する「バインディング層」で構成されています。子モジュールは存在せず、ディレクトリ内の各ファイルが役割ごとに分担するフラットな構成です。

推奨読書順

1. mod_lua.h (全体像とデータ構造) 2. lua_vmprep.h / lua_vmprep.c (VM 管理の基盤) 3. lua_request.h / lua_request.c (Apache API バインディング) 4. lua_config.h / lua_config.c (設定管理) 5. lua_apr.h / lua_apr.c (データ変換) 6. lua_dbd.h / lua_dbd.c (DB 連携) 7. lua_passwd.h / lua_passwd.c (認証ユーティリティ) 8. mod_lua.c (メインエントリーポイントとフック処理)
