os/win32

システム内での役割

Apache HTTP Server を Windows プラットフォーム上で動作させるための OS 抽象化レイヤーです。Windows 特有のシステムコール、レジストリ操作、およびビルド時のパス構成を管理し、プラットフォーム間の差異を吸収する役割を担います。本モジュールはディレクトリ境界に基づいて定義されています。

責務

  • **OS 抽象化**: Windows API の差異（Unicode/ANSI）やバージョン依存性を吸収し、Apache Core に一貫したインターフェースを提供する。
  • **リソース管理**: Windows レジストリ操作のライフサイクルを APR プールと統合し、安全なメモリ管理を実現する。
  • **動的リンク**: 特定の Windows バージョンに依存する API を実行時に遅延ロードするメカニズムの提供。
  • **静的構成**: サーバー起動時に必要なモジュールリストの管理およびビルド時のパスレイアウト定義。

このモジュールのファイル

  • os/win32/ap_regkey.c: Windows レジストリ操作の抽象化実装。
  • os/win32/modules.c: 静的リンクされるモジュール群の定義と登録。
  • os/win32/os.h: Windows プラットフォーム向け基底ヘッダーおよび動的リンクマクロ。
  • os/win32/util_win32.c: プロセス管理や DLL ロード等の OS 固有ユーティリティ。
  • os/win32/win32_config_layout.h: Windows ビルド環境におけるディレクトリパス定義。

主要なデータ構造

  • ap_regkey_t: レジストリキーハンドルと APR プールを保持するコンテナ。
  • ap_dlltoken_e: ロード対象のシステム DLL を識別する列挙型。
  • ap_prelinked_modules: サーバー起動時に初期化される静的モジュールの配列。

主なフロー

1. **初期化**: サーバー起動時に modules.c が参照され、core_module を含む必須モジュールがロードされます。 2. **遅延ロード**: os.h で定義された AP_DECLARE_LATE_DLL_FUNC を介して、必要な Windows API が初回呼び出し時に util_win32.c の機能で動的に解決されます。 3. **レジストリ操作**: ap_regkey_open 等の関数が呼び出されると、OS API をラップしつつ APR プールにクリーンアップハンドラを登録してリソースを保護します。

外部依存

  • **APR**: apr.h, apr_pools.h, apr_strings.h, arch/win32/ 配下のアーキテクチャ定義。
  • **Apache Core**: httpd.h, http_config.h, http_log.h, ap_mpm.h。

内部構成

推奨読書順

1. os/win32/os.h: プラットフォーム抽象化の基本方針を理解する。 2. os/win32/util_win32.c: OS 固有のユーティリティと DLL ロード機構を確認する。 3. os/win32/ap_regkey.c: APR プールと連携したリソース管理の実装例を確認する。 4. os/win32/modules.c: サーバーの静的構成を確認する。 5. os/win32/win32_config_layout.h: ビルド時のパス構成を確認する。
