modules/loggers

システム内での役割

modules/loggers は、Apache HTTP Server におけるリクエストの記録、診断、および統計収集を担うサブシステムです。このモジュール群は、標準的なアクセスログの記録から、デバッグ用の詳細なメッセージ出力、ネットワーク統計の追跡、セキュリティ監査のためのフォレンジックログまで、多角的なロギング機能を提供します。

責務

  • **アクセスログの柔軟な生成**: mod_log_config を中心に、リクエスト単位の情報を収集し、設定可能なフォーマットで出力する。
  • **診断とデバッグ**: mod_log_debug を通じて、リクエスト処理の各フェーズで任意の条件式に基づいたログを出力する。
  • **統計情報の収集**: mod_logio により、入出力バイト数や TTFB（Time To First Byte）を計測し、ログへ統合する。
  • **セキュリティ監査**: mod_log_forensic により、リクエストの開始から終了までを追跡可能な形式で記録する。

このモジュールのファイル

  • mod_log_config.c / .h: ログ記録のコアエンジンおよび外部拡張用インターフェース。
  • mod_log_debug.c: 条件付きデバッグメッセージ出力モジュール。
  • mod_log_forensic.c: リクエストのライフサイクルを追跡するフォレンジックログモジュール。
  • mod_logio.c: 入出力バイト数および TTFB 統計収集モジュール。

主要なデータ構造

  • **設定管理**: multi_log_state, config_log_state, log_debug_dirconf, fcfg
  • **ログ処理**: log_format_item, buffered_log, msg_entry
  • **統計・状態**: logio_config_t, logio_req_t
  • **共通**: request_rec, server_rec, apr_size_t

主なフロー

1. **初期化**: 各モジュールが register_hooks を通じて Apache のフック（open_logs, log_transaction 等）に登録される。 2. **設定解析**: LogFormat や LogMessage 等のディレクティブが解析され、内部的な設定構造体へ変換される。 3. **リクエスト処理**:

  • mod_log_debug が各フェーズで条件評価とログ出力を行う。
  • mod_logio がフィルタを通じて入出力バイト数を加算する。
  • mod_log_forensic がリクエストの開始と終了を記録する。

    • 4. **ログ出力**: リクエスト終了時、mod_log_config が収集された情報をフォーマットに従って文字列化し、登録されたライターを通じて出力する。

外部依存

  • **Apache Core**: httpd.h, http_config.h, http_protocol.h, http_log.h
  • **APR**: apr_strings.h, apr_hash.h, apr_optional.h, apr_lib.h, apr_want.h
  • **その他**: ap_config.h, ap_expr.h（式評価用）

内部構成

本モジュールは、mod_log_config を基盤として他のログモジュールが拡張機能を提供する構成をとっています。各モジュールは独立した責務を持ちつつ、mod_log_config が公開するオプション関数やフックを通じて連携します。

推奨読書順

1. mod_log_config.h（インターフェースの理解） 2. mod_log_config.c（コアエンジンの理解） 3. mod_logio.c（拡張モジュールの実装例） 4. mod_log_debug.c（条件付き処理の理解） 5. mod_log_forensic.c（監査ログの特殊なライフサイクル管理）
