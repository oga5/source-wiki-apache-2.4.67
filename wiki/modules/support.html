support

システム内での役割

support モジュールは、Apache HTTP Server の運用、管理、および性能測定を支援するスタンドアロンのユーティリティ群を提供します。これらのツールは、サーバー本体の実行プロセスとは独立して動作し、認証情報の管理、ログのローテーション、負荷試験、セキュリティ検証などのタスクを担います。

責務

  • **認証管理**: パスワードファイル（htpasswd）や DBM データベース（htdbm）の作成・更新・検証。
  • **ログ・キャッシュ管理**: ログの自動ローテーション（rotatelogs）、キャッシュのクリーンアップ（htcacheclean）、DNS 逆引きによるログ整形（logresolve）。
  • **プロセス・権限管理**: CGI 実行時の権限昇格（suexec）、FastCGI プロセスの起動（fcgistarter）、グループ権限の検証（checkgid）。
  • **性能測定**: HTTP ベンチマーク（ab）によるサーバー負荷試験。
  • **設定変換**: テキスト形式のマップファイルを DBM 形式へ変換（httxt2dbm）。

このモジュールのファイル

  • **認証・管理**: htpasswd.c, htdbm.c, htdigest.c, passwd_common.c/h
  • **ログ・キャッシュ**: rotatelogs.c, htcacheclean.c, logresolve.c
  • **プロセス・セキュリティ**: suexec.c/h, fcgistarter.c, checkgid.c
  • **性能・変換**: ab.c, httxt2dbm.c

主要なデータ構造

  • **認証コンテキスト**: passwd_ctx (パスワード生成・検証状態を保持)
  • **接続・統計**: connection, stats (ApacheBench 用)
  • **キャッシュ管理**: ENTRY, DIRENTRY (キャッシュエントリの管理)
  • **ローテーション**: rotate_config_t, rotate_status_t (ログローテーション設定と状態)
  • **DBM 管理**: htdbm_t (DBM データベース接続情報)

主なフロー

各ツールは独立した main() を持ち、以下の共通パターンで動作します。 1. **初期化**: apr_app_initialize による APR 環境の構築とメモリプールの作成。 2. **引数解析**: apr_getopt を用いたコマンドラインオプションの処理。 3. **実行**: 各ツールの責務に応じたメインループ（ファイル走査、ソケット通信、または認証処理）。 4. **終了**: atexit 等で登録された終了処理によるリソース解放とステータスコードの返却。

外部依存

  • **APR (Apache Portable Runtime)**: ファイル I/O、メモリ管理、ネットワーク通信、プロセス制御、ハッシュテーブル等の基盤機能。
  • **OpenSSL**: ab における HTTPS 通信のサポート。
  • **システムライブラリ**: pwd.h, grp.h, sys/stat.h 等（主に suexec や checkgid で使用）。

内部構成

本モジュールはディレクトリ境界によって定義されており、各ソースファイルは独立した実行バイナリとしてビルドされます。passwd_common.c/h が認証ツール間での共有ロジックを提供し、コードの重複を排除しています。

推奨読書順

1. support/passwd_common.h (認証ツールの共通インターフェース) 2. support/suexec.h (セキュリティポリシーの定義) 3. support/ab.c (非同期 I/O の実装例) 4. support/htcacheclean.c (ファイルシステム操作とキャッシュ管理) 5. support/rotatelogs.c (ログ管理とプロセス間通信) 6. support/htpasswd.c (認証ファイルの更新ロジック) 7. support/suexec.c (権限昇格とセキュリティ検証) 8. support/fcgistarter.c (プロセス管理とソケット操作) 9. support/checkgid.c (システム権限検証)
