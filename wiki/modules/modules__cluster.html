modules/cluster

システム内での役割

modules/cluster は、Apache HTTP Server のクラスター環境におけるサーバー負荷情報の共有を担うサブシステムです。ディレクトリ境界に基づき、ハートビート信号の送受信を行うモジュール群で構成されます。

責務

  • **負荷情報のブロードキャスト**: 自サーバーの稼働状況（Ready/Busy）を定期的にネットワークへ通知する。
  • **クラスター状態の監視・集約**: ネットワーク上のハートビート信号を監視し、バックエンドサーバーの稼働状態を収集・永続化する。
  • **ロードバランシングへのデータ提供**: 収集した情報を共有メモリ（slotmem）等に保持し、他モジュール（mod_lbmethod_heartbeat 等）が動的な負荷分散判断を行えるようにする。

このモジュールのファイル

  • mod_heartbeat.c: 負荷情報を送信する情報発信側モジュール。
  • mod_heartmonitor.c: ハートビート信号を監視・集約する情報受信側モジュール。

主要なデータ構造

  • hb_ctx_t: mod_heartbeat の設定および実行時状態。
  • hm_ctx_t: mod_heartmonitor のサーバー単位の設定と実行時状態。
  • hm_server_t: 個々のバックエンドサーバーの稼働状態（IP, ポート, 負荷統計）。
  • hm_slot_server_ctx_t: 共有メモリ上のサーバー状態管理用構造体。

主なフロー

1. **送信**: mod_heartbeat が mod_watchdog を介して定期的にスコアボードを走査し、UDP マルチキャストで負荷情報を送信する。 2. **受信・集約**: mod_heartmonitor が mod_watchdog を介してソケットをポーリングし、パケットを受信・解析する。 3. **永続化**: mod_heartmonitor が収集した情報を slotmem（共有メモリ）またはファイルへ書き出し、ロードバランサーが参照可能な状態にする。

外部依存

  • mod_watchdog.h: バックグラウンドタスクの定期実行。
  • scoreboard.h: サーバーの稼働状況（Ready/Busy）の取得。
  • ap_slotmem.h: 共有メモリによるプロセス間通信。
  • apr_network_io.h / apr_poll.h: UDP 通信およびイベント駆動の待機。
  • httpd.h, http_config.h, http_log.h 等の Apache HTTP Server コア API。

内部構成
