server

システム内での役割

server モジュールは Apache HTTP Server の心臓部であり、サーバーの起動、設定管理、リクエスト処理のライフサイクル、およびマルチプロセッシング（MPM）の基盤を提供します。このディレクトリは、HTTPD のコア機能と、それらを支えるユーティリティ群を集約しています。

責務

  • **設定管理**: 設定ファイルのパース、モジュールのロード、および階層的な設定マージ。
  • **リクエスト処理**: リクエストの受信からハンドラへの引き渡し、フィルタチェーンの管理。
  • **プロセス/スレッド管理**: MPM を介したワーカーのライフサイクル管理と共有メモリ（スコアボード）の維持。
  • **式評価エンジン**: 設定やリクエスト処理で動的に条件を評価する ap_expr エンジンの提供。
  • **共通ユーティリティ**: 文字列操作、パス正規化、ログ出力、CGI 関連処理の抽象化。

主要なデータ構造

  • request_rec: 単一の HTTP リクエストの全状態を保持。
  • server_rec: 仮想ホストを含むサーバー設定の基盤。
  • ap_directive_t: 設定ファイルから構築されるツリー構造。
  • ap_filter_t: フィルタチェーンを構成するノード。
  • ap_expr_parse_ctx_t: 式評価エンジンの解析コンテキスト。
  • fd_queue_t: MPM におけるスレッド間通信用キュー。

主なフロー

1. **起動**: main.c が設定を読み込み、config.c がモジュールをロードして設定ツリーを構築。 2. **接続**: listen.c がソケットを監視し、MPM が mpm_fdqueue.c を介してワーカースレッドへ接続を割り当て。 3. **リクエスト**: protocol.c がリクエストを解析し、request.c がディレクトリ/ファイル単位の設定をマージ（Walk）して処理フェーズを進行。 4. **評価**: 必要に応じて util_expr_eval.c が条件式を評価し、フィルタチェーンを通じてレスポンスを送信。

外部依存

  • **APR (Apache Portable Runtime)**: メモリ管理、スレッド、プロセス、I/O の抽象化レイヤー。
  • **システムライブラリ**: unistd.h, pwd.h 等の OS 固有機能。

内部構成

本モジュールは、以下の機能カテゴリで構成されています。

  • **コア・設定**: core.c, config.c, vhost.c
  • **リクエスト・プロトコル**: request.c, protocol.c, util_filter.c
  • **MPM・プロセス管理**: mpm_common.c, mpm_unix.c, scoreboard.c, listen.c, mpm_fdqueue.c
  • **式評価エンジン**: util_expr_parse.c, util_expr_scan.c, util_expr_eval.c
  • **ユーティリティ**: util.c, log.c, util_script.c 等

推奨読書順

1. core.c (サーバーの全体像と設定構造) 2. config.c (設定のロードとモジュール管理) 3. request.c (リクエスト処理のライフサイクル) 4. protocol.c (HTTP プロトコル解析) 5. util_filter.c (データ処理パイプライン) 6. mpm_common.c (プロセス管理の基盤) 7. util_expr_eval.c (式評価エンジンの動作)
