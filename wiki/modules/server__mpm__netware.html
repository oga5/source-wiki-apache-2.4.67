server/mpm/netware

システム内での役割

NetWare OS 上で Apache HTTP Server を動作させるためのマルチプロセッシングモジュール（MPM）です。NetWare の単一プロセス・マルチスレッドモデル（NKS: NetWare Kernel Services）を活用し、効率的なリクエスト処理を実現します。

責務

  • **スレッド管理**: ワーカースレッドの生成、監視、およびスレッドプールの動的調整（perform_idle_server_maintenance）。
  • **コンソール統合**: NetWare コンソールコマンド（RESTART, SHUTDOWN 等）の登録と処理。
  • **ライフサイクル管理**: サーバーの起動、リスニングソケットの監視、および正常終了/再起動の制御。
  • **設定管理**: スレッド数やスタックサイズなどの MPM 固有ディレクティブの解析と適用。

このモジュールのファイル

  • server/mpm/netware/mpm_default.h: スレッド数やスタックサイズ等のデフォルト値定義。
  • server/mpm/netware/mpm_netware.c: MPM のメインロジックおよび NetWare OS との統合実装。

主要なデータ構造

  • worker_main: ワーカースレッドのメインループ関数。
  • mpm_state: サーバーの現在の動作状態（起動中、実行中、停止中など）。
  • shutdown_pending / restart_pending: 終了や再起動を制御する状態遷移フラグ。

主なフロー

1. **初期化**: netware_mpm_hooks によりフックを登録し、コンソールハンドラをインストール。 2. **起動**: netware_run がリスニングソケットを準備し、初期スレッド群を生成。 3. **メインループ**: perform_idle_server_maintenance を定期実行し、スレッドプールの過不足を調整。 4. **リクエスト処理**: 各ワーカースレッドが select() で接続を待機し、リクエストを処理。 5. **終了**: 終了フラグがセットされると、全スレッドの終了を待機し、リソースを解放。

外部依存

  • **APR**: スレッド、メモリ、同期プリミティブの抽象化。
  • **NetWare SDK**: NKS を介したネイティブなスレッド制御およびコンソール操作。
  • **Apache Core**: httpd.h, mpm_common.h, scoreboard.h 等によるサーバー基盤との連携。

内部構成

  • 子モジュール数: 0
  • 本モジュールは NetWare 特有の環境に最適化されており、fork() を使用しない単一プロセスモデルを採用しています。

推奨読書順

1. server/mpm/netware/mpm_default.h（定数と制限値の把握） 2. server/mpm/netware/mpm_netware.c（スレッド管理とライフサイクル制御の理解）
