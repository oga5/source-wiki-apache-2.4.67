modules/session

システム内での役割

Apache HTTP Server におけるセッション管理の基盤を提供するモジュール群です。リクエスト間での状態保持を可能にし、セッションデータのロード、保存、および永続化を抽象化します。

責務

  • **セッションライフサイクル管理**: セッションの生成、ロード、保存、および有効期限の制御。
  • **ストレージの抽象化**: Cookie やデータベース（DBD）など、多様なバックエンドへのセッション永続化。
  • **データ保護**: 暗号化および認証（MAC）によるセッションデータの機密性と完全性の確保。
  • **フックによる拡張**: ストレージやシリアライズ形式をプラグインとして追加可能なインターフェースの提供。

このモジュールのファイル

  • mod_session.c: セッション管理のコアロジックとオーケストレーション。
  • mod_session.h: 共通のデータ構造（session_rec 等）とフックインターフェースの定義。
  • mod_session_cookie.c: クライアントサイド（Cookie）へのセッション保存実装。
  • mod_session_crypto.c: セッションデータの暗号化・復号化および改ざん検知。
  • mod_session_dbd.c: サーバーサイド（データベース）へのセッション保存実装。

主要なデータ構造

  • session_rec: セッションのキー・値ペア、有効期限、UUID 等を保持する中心的な構造体。
  • session_dir_conf: ディレクトリ単位のセッション設定（有効化、パス制限等）。
  • session_cookie_dir_conf / session_crypto_dir_conf / session_dbd_dir_conf: 各バックエンド固有の設定構造体。

主なフロー

1. **ロード**: リクエスト開始時に session_load フックが実行され、Cookie やデータベースからセッションを復元。 2. **操作**: ap_session_get / set を通じて session_rec を操作。変更時は dirty フラグがセットされる。 3. **保存**: リクエスト終了時の出力フィルタにて、dirty フラグを確認し、session_encode（シリアライズ/暗号化）および session_save（バックエンド書き込み）を実行。

外部依存

  • **APR**: apr_hooks.h, apr_crypto.h, apr_dbd.h, apr_tables.h 等によるメモリ管理、暗号化、DB接続、データ構造の提供。
  • **Apache Core**: httpd.h, http_config.h, util_filter.h 等によるモジュール基盤およびフィルタ処理。

内部構成

本モジュールは、コアとなる mod_session がフックを定義し、各ストレージ/機能モジュールがそれを実装する「プロバイダー・コンシューマー」モデルを採用しています。

推奨読書順

1. mod_session.h（インターフェースとデータ構造の理解） 2. mod_session.c（ライフサイクル管理の全体像） 3. mod_session_crypto.c（データ保護の仕組み） 4. mod_session_cookie.c（クライアントサイド保存の実装） 5. mod_session_dbd.c（サーバーサイド保存の実装）
