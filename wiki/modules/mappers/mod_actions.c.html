modules/mappers/mod_actions.c

概要

mod_actions.c は、特定の MIME タイプや HTTP メソッドに基づいて CGI スクリプトを自動的に実行するための Apache HTTP Server モジュールです。リクエストされたリソースの属性に応じ、内部リダイレクトを通じて処理を外部スクリプトへ委譲します。

モジュール内での役割

modules/mappers サブシステムの一部として、リクエストの「マッピング」フェーズにおいて、コンテンツの型やメソッドに応じた動的な処理の振り分けを担当します。

責務

  • **設定管理**: Action および Script ディレクティブによる、MIME タイプ/メソッドとスクリプトパスの紐付けを管理。
  • **リクエスト処理**: リクエストの MIME タイプやメソッドを評価し、該当するスクリプトが存在する場合に ap_internal_redirect_handler を呼び出す。
  • **環境変数の伝播**: 実行されるスクリプトに対し、元のリクエスト情報を PATH_INFO や PATH_TRANSLATED 等を通じて提供する。

重要なシンボル

  • action_dir_config: ディレクトリごとの設定を保持する構造体。MIME タイプごとのアクションと、メソッドごとのスクリプトを格納。
  • action_handler: リクエスト処理のメイン関数。APR_HOOK_LAST で登録され、他のハンドラが処理しなかったリクエストを評価する。
  • add_action / set_script: 設定ファイルからディレクティブを読み込み、設定構造体に反映させるコマンドハンドラ。

制御フローとデータフロー

1. **設定**: action_cmds により、設定ファイルから action_dir_config が構築される。 2. **フック**: register_hooks により action_handler がハンドラとして登録される。 3. **評価**: action_handler がリクエストをインターセプトし、以下の順序でスクリプトを決定する。

  • メソッドベースのスクリプト（Script ディレクティブ）を確認。
  • MIME タイプベースのアクション（Action ディレクティブ）を確認（こちらが優先される）。

    • 4. **実行**: スクリプトが特定された場合、ap_internal_redirect_handler を使用してリクエストをスクリプトへ転送する。

直接依存

  • httpd.h, http_config.h, http_request.h: Apache のコアリクエスト処理およびモジュール設定用。
  • util_script.h: CGI スクリプト実行に関連するユーティリティ。
  • apr_strings.h, apr_want.h: APR による文字列操作およびメモリ管理。
  • ap_config.h
  • http_core.h
  • http_log.h
  • http_main.h
  • http_protocol.h

関連コード

  • mod_cgi.c: 本モジュールが転送先として利用する CGI 実行エンジン。
  • http_core.h: ap_internal_redirect_handler 等のコア機能を提供。

読者向けメモ

  • **ループ防止**: r->prev をチェックすることで、アクションによる無限リダイレクトループを回避しています。
  • **優先順位**: Action ディレクティブによる MIME タイプベースの処理は、Script ディレクティブによるメソッドベースの処理を上書きします。
  • **仮想スクリプト**: Action ディレクティブで virtual オプションを指定すると、ファイルの実在チェックをスキップしてスクリプトを呼び出せます。
