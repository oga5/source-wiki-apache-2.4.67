modules/mappers/mod_rewrite.c

概要

mod_rewrite は、Apache HTTP Server の強力な URL 書き換えエンジンです。正規表現ベースのルールエンジンを用いて、リクエストされた URL を動的に書き換えます。サーバー全体（httpd.conf）およびディレクトリ単位（.htaccess）の両方のコンテキストで動作し、外部データベース（DBM, DBD）や外部プログラムとの連携による高度な URL 置換をサポートします。

モジュール内での役割

modules/mappers サブシステムにおいて、リクエストの変換（Translation）および修正（Fixup）フェーズを担う最も複雑なモジュールです。他のマッパーモジュール（mod_alias 等）よりも柔軟かつ動的なパス操作を提供し、リクエストの内部リダイレクト、外部リダイレクト、プロキシ転送のトリガーとなります。

責務

  • **ルールエンジン**: RewriteRule および RewriteCond の解析と実行。
  • **変数展開**: HTTP ヘッダー、環境変数、時刻、SSL 情報などの動的変数展開。
  • **外部マップ連携**: テキストファイル、DBM、DBD（SQL）、外部プログラム（PRG）を用いたルックアップ。
  • **リクエスト制御**: MIME タイプやハンドラーの強制設定、リダイレクト処理。
  • **キャッシュ管理**: 外部マップルックアップのパフォーマンス向上のためのキャッシュ機構。

重要なシンボル

設定構造体

  • rewrite_server_conf: サーバー全体のエンジン状態、マップ、ルールを保持。
  • rewrite_perdir_conf: ディレクトリ単位のエンジン状態、ルールを保持。
  • rewritemap_entry: 外部マップの定義（ファイルパス、タイプ、プログラム実行用パイプ等）。
  • rewriterule_entry / rewritecond_entry: 解析済みのルールと条件式。

実行コンテキスト

  • rewrite_ctx: ルール適用時のリクエスト状態、URI、バックリファレンス情報を保持。
  • backrefinfo: 正規表現のキャプチャ結果を保持。

主要フック関数

  • hook_uri2file: サーバーコンテキストの書き換え（Translation フェーズ）。
  • hook_fixup: ディレクトリコンテキストの書き換え（Fixup フェーズ）。
  • hook_mimetype: [T=...] や [H=...] フラグによる強制設定。

制御フローとデータフロー

1. **初期化**: pre_config でマップ関数を登録し、post_config で外部プログラムマップを起動。 2. **ルール適用**: apply_rewrite_list がルールセットを反復処理。 3. **条件評価**: apply_rewrite_cond が条件を評価し、一致すれば apply_rewrite_rule で置換を実行。 4. **展開**: do_expand が文字列内の変数（%{}, ${}）やバックリファレンス（$N）を再帰的に解決。 5. **結果適用**: splitout_queryargs や fully_qualify_uri を経て、r->filename や r->status を更新。 6. **リダイレクト**: 内部リダイレクトが必要な場合、handler_redirect を経由して ap_internal_redirect を呼び出す。

読者向けメモ

  • **メモリ管理**: ほとんどの動的割り当ては r->pool を使用しており、リクエスト終了時に自動解放されます。キャッシュ関連のみ cachep->pool を使用し、生存期間が異なります。
  • **スレッド安全性**: APR_HAS_THREADS が有効な場合、マップキャッシュや外部プログラムとの通信には apr_thread_mutex_t や apr_global_mutex_t による排他制御が行われます。
  • **エッジケース**: [N] フラグによる無限ループを防ぐため、REWRITE_MAX_ROUNDS による制限があります。また、[PT] フラグは Apache 1.x 互換のためのハックであり、使用には注意が必要です。
  • **省略箇所**: 本ページでは、複雑な正規表現置換の詳細や、各 OS 特有のファイルシステムパス処理の細部は省略しています。これらは server/util.c の実装に強く依存しています。
