modules/mappers/mod_dir.c

概要

mod_dir.c は、Apache HTTP Server におけるディレクトリリクエストの処理を担うモジュールです。主に「ディレクトリインデックス（DirectoryIndex）の解決」と「末尾スラッシュなしのディレクトリに対するリダイレクト処理」を制御します。また、ファイルが存在しない場合に備えた FallbackResource の処理も提供します。

モジュール内での役割

modules/mappers サブシステムの一部として、リクエストのパスがディレクトリを指している場合に、適切なコンテンツ（インデックスファイル）を特定し、内部リダイレクトを行う役割を持ちます。fixups フックの最後（APR_HOOK_LAST）で動作し、他のモジュールがリクエストを処理しなかった場合のフォールバックとして機能します。

責務

  • **設定管理**: DirectoryIndex, DirectorySlash, DirectoryCheckHandler, FallbackResource などのディレクティブを解析し、ディレクトリごとの設定構造体 dir_config_rec を構築・マージする。
  • **ディレクトリリダイレクト**: 末尾に / がないディレクトリリクエストに対し、DirectorySlash 設定に基づいて正規化（リダイレクト）を行う。
  • **インデックス解決**: 設定されたインデックスファイルリストを順次探索し、最初に見つかった有効なファイルを内部リダイレクトで提供する。
  • **フォールバック処理**: ファイルが存在しない場合に FallbackResource で指定されたリソースへ転送する。

重要なシンボル

設定構造体

  • dir_config_rec: ディレクトリごとの設定を保持。index_names（インデックスリスト）、do_slash（スラッシュ補完フラグ）、dflt（フォールバック先）などを含む。

設定コマンドハンドラ

  • add_index: DirectoryIndex を解析し、配列に格納する。
  • configure_slash / configure_checkhandler: フラグ設定を更新。
  • configure_redirect: DirectoryIndexRedirect のステータスコード（3xx）を検証・設定。

フックとメインロジック

  • dir_fixups: エントリポイント。ファイルタイプに基づき fixup_dir または fixup_dflt へ振り分ける。
  • fixup_dir: ディレクトリリクエストのメイン処理。スラッシュ補完とインデックス探索を行う。
  • fixup_dflt: FallbackResource の処理。

制御フローとデータフロー

1. **フック起動**: dir_fixups がリクエストの finfo を確認。 2. **ディレクトリ判定**: APR_DIR であれば fixup_dir を呼び出し、末尾スラッシュの確認と DirectoryIndex の探索を行う。 3. **サブリクエスト**: ap_sub_req_lookup_uri を使用して、インデックス候補が実際に存在するか（APR_REG か）を確認する。 4. **リダイレクト/転送**:

  • 条件を満たすインデックスが見つかれば ap_internal_fast_redirect で処理を完了。
  • DirectoryIndexRedirect が有効な場合は、クライアントへ 3xx レスポンスを返す。

    • 5. **フォールバック**: APR_NOFILE の場合、fixup_dflt が FallbackResource を確認し、同様にサブリクエスト経由で転送を試みる。

関連コード

読者向けメモ

  • **メモリ所有権**: dir_config_rec は Apache の設定プール（pool）上で管理されます。fixup 内で生成されるサブリクエストは、ap_destroy_sub_req を呼び出すことで適切にクリーンアップされます。
  • **不変条件**: fixup_dir は path_info が存在する場合、ディレクトリインデックス処理をスキップします。
  • **注意点**: FallbackResource を使用する場合、指定先が存在しないと無限ループに陥る可能性があるため、設定には注意が必要です。
  • **省略箇所**: 本ページでは、apr_array_header_t の詳細な操作や、ap_getword_conf による引数解析の細部については、標準的な Apache モジュール開発の範疇であるため省略しています。
