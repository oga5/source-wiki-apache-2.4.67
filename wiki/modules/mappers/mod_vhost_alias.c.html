modules/mappers/mod_vhost_alias.c

概要

mod_vhost_alias は、Apache HTTP Server において動的なバーチャルホスト（Mass Virtual Hosting）を実現するためのモジュールです。ホスト名や IP アドレスをテンプレート文字列（VirtualDocumentRoot 等）に埋め込むことで、ディレクトリ構造を自動的に決定し、リクエストのパスをマッピングします。

モジュール内での役割

modules/mappers サブシステムの一部として、リクエストの translate_name フェーズで動作します。mod_alias や mod_userdir と競合しないよう、フックの実行順序を制御しつつ、静的な設定ファイル記述を最小化する役割を担います。

責務

  • **設定管理**: VirtualDocumentRoot および VirtualScriptAlias の設定をサーバー単位で保持・マージする。
  • **パス変換**: リクエストされたホスト名/IP を解析し、設定されたテンプレートに従って物理パスを生成する。
  • **コンテキスト設定**: 生成されたパスを r->filename に反映し、DocumentRoot や cgi-script ハンドラを適切に設定する。

重要なシンボル

設定構造体とライフサイクル

  • mva_sconf_t: サーバー単位の設定を保持する構造体。doc_root / cgi_root のパスと、それらがホスト名ベースか IP ベースか（mva_mode_e）を管理。
  • mva_create_server_config: 設定構造体の初期化。
  • mva_merge_server_config: 親サーバー設定と子サーバー設定の継承処理。

コマンドハンドラ

  • vhost_alias_set: VirtualDocumentRoot 等のディレクティブを処理する汎用ハンドラ。テンプレート文字列の構文チェック（% 記法）を行う。

変換ロジック

  • mva_translate: translate_name フックのメイン関数。リクエストが CGI かドキュメントルートかを判定し、適切なモードで補間処理を呼び出す。
  • vhost_alias_interpolate: テンプレート文字列を解析し、ホスト名/IP のパーツを抽出して物理パスを構築するコア関数。
  • vhost_alias_checkspace: r->filename のバッファ溢れを防ぐためのユーティリティ。

制御フローとデータフロー

1. **フック登録**: register_hooks により ap_hook_translate_name に mva_translate を登録。 2. **判定**: mva_translate がリクエストの URI を確認し、CGI か通常パスかを判断。 3. **補間**: vhost_alias_interpolate がホスト名/IP を dots 配列で分割し、テンプレートの % 指定子（例: %1.2）に基づいてパスを組み立てる。 4. **適用**: 生成されたパスを r->filename に格納し、必要に応じて r->handler を cgi-script に設定して終了。

直接依存

  • httpd.h, http_config.h, http_request.h: Apache コア API。
  • apr_strings.h, apr_lib.h: 文字列操作およびメモリ管理。

関連コード

  • mod_alias.c: 実装の着想元。
  • mod_cgi.c: cgi-script ハンドラの挙動に関連。
  • http_core.c: DocumentRoot の標準的な扱い。

読者向けメモ

  • **メモリ管理**: apr_pool_t を使用しており、リクエスト処理中の文字列生成はすべてリクエストプールに依存します。
  • **制約**: テンプレート文字列の構文チェックは厳密に行われますが、HUGE_STRING_LEN を超えるパス生成時の挙動には注意が必要です。
  • **最適化の余地**: ソース内のコメントにある通り、パスの固定部分（% 以前のパス）を事前に計算してキャッシュする最適化は未実装です。
  • **未実装領域**: このモジュールは Directory 単位の設定オーバーライドをサポートしていません（サーバー設定のみ）。
