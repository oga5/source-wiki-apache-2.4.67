modules/mappers/mod_rewrite.h

概要

mod_rewrite.h は、Apache HTTP Server の URL 書き換えエンジンである mod_rewrite モジュールのパブリックインターフェースを定義するヘッダーファイルです。主に、外部モジュールが mod_rewrite の拡張機能を利用するためのフックを提供します。

モジュール内での役割

modules/mappers サブシステムにおいて、本ファイルは mod_rewrite.c（実装本体）と、それを利用する他のモジュールとの間の契約を定義します。特に、動的な書き換えマップ（RewriteMap）を外部から登録するための標準的な手段を公開しています。

責務

  • mod_rewrite が提供する拡張ポイント（カスタムマップ関数）の型定義。
  • 外部モジュールが独自の書き換えロジックを登録するためのオプション関数 ap_register_rewrite_mapfunc の宣言。
  • モジュール間連携のための定数定義。

重要なシンボル

  • rewrite_mapfunc_t: 書き換えマップ関数が従うべき関数ポインタ型。request_rec を受け取り、変換後の文字列を返します。
  • ap_register_rewrite_mapfunc: APR_DECLARE_OPTIONAL_FN を介して提供される関数。外部モジュールが独自のマップ関数を mod_rewrite に登録するために使用します。
  • REWRITE_REDIRECT_HANDLER_NAME: リダイレクトハンドラを識別するための定数文字列。

制御フローとデータフロー

本ファイル自体は宣言のみを行うため、実行時の制御フローは持ちません。

  • **データフロー**: 外部モジュールが ap_register_rewrite_mapfunc を呼び出すことで、mod_rewrite 内部のマップ関数テーブルにカスタム関数が登録されます。その後、URL 書き換え処理中に mod_rewrite がこのテーブルを参照し、登録された関数を実行することでデータ（URL）の変換が行われます。

直接依存

  • apr_optional.h: Apache Portable Runtime のオプション関数機能を利用するために必要です。
  • httpd.h: Apache のコア構造体である request_rec を参照するために必要です。

関連コード

  • modules/mappers/mod_rewrite.c: 本ヘッダーで宣言された関数の実装および、書き換えエンジンのメインロジックが含まれます。
  • ap_expr.h: mod_rewrite が式評価エンジンと連携する際に参照される関連モジュールです。

読者向けメモ

  • 本ファイルは非常に軽量であり、主に API のインターフェース定義に特化しています。
  • mod_rewrite の内部ロジックを理解したい場合は、本ファイルではなく mod_rewrite.c を参照してください。
  • APR_DECLARE_OPTIONAL_FN を使用しているため、mod_rewrite がロードされていない環境でも、このヘッダーをインクルードするモジュールはリンクエラーを回避しつつ安全に動作する設計になっています。
