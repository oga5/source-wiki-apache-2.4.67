modules/mappers/mod_userdir.c

概要

mod_userdir は、Apache HTTP Server において ~username 形式の URI を各ユーザーのホームディレクトリ内の特定のサブディレクトリ（例: public_html）へマッピングする機能を提供します。

モジュール内での役割

modules/mappers サブシステムの一部として、リクエストの URI をファイルシステム上のパスへ変換する「翻訳フェーズ（Translate Name）」を担当します。mod_alias 等と連携し、リクエスト処理の初期段階で動作します。

責務

  • **URI マッピング**: ~user を設定されたディレクトリパスへ変換。
  • **アクセス制御**: UserDir ディレクティブによる、特定のユーザーに対する有効化/無効化の管理。
  • **リダイレクト処理**: 設定に応じて、ローカルパスへの変換だけでなく、外部 URL への 302 リダイレクトを実行。
  • **suEXEC 連携**: Unix 環境において、該当ユーザーの権限でコンテンツを配信するための ID 解決（get_suexec_identity フック）。

重要なシンボル

  • userdir_config: モジュールの設定を保持する構造体。有効/無効ユーザーのテーブルを含む。
  • translate_userdir: リクエストの URI を解析し、ファイルパスを決定するメイン関数。
  • set_user_dir: UserDir ディレクティブを解析し、設定を構築するハンドラ。
  • get_suexec_id_doer: suEXEC 使用時にユーザーの UID/GID を取得する関数。

制御フローとデータフロー

1. **設定**: set_user_dir が UserDir 設定を読み込み、userdir_config に格納。 2. **フック**: translate_userdir が translate_name フックで起動。 3. **検証**: URI が /~ で始まるか確認し、無効化リストに含まれていないかチェック。 4. **変換**: 設定された UserDir パターンに基づき、パスを構築。

  • パスが絶対パスや URL の場合、リダイレクトまたは直接パスを生成。
  • それ以外の場合、apr_uid_homepath_get を使用してユーザーのホームディレクトリを特定。

    • 5. **適用**: 成功時、r->filename を更新し、OK を返して処理を完了。

直接依存

  • apr_strings.h, apr_user.h: 文字列操作およびユーザー情報取得。
  • httpd.h, http_config.h, http_request.h: Apache のコア API。
  • unixd.h: Unix 環境での suEXEC 関連定義。
  • ap_config.h

関連コード

  • mod_alias.c: 類似の URI マッピングを行うモジュール。register_hooks での実行順序の基準となっている。
  • mod_vhost_alias.c: 同様にパス変換を行うモジュール。

読者向けメモ

  • 本モジュールは ~user という特定のパターンに特化しています。複雑な書き換えが必要な場合は mod_rewrite の使用が推奨されます。
  • UserDir に複数の値を指定した場合、順に評価され、最初に存在が確認された（または最後の）パスが採用されます。
  • HAVE_UNIX_SUEXEC が定義されている環境では、セキュリティ上の理由から suEXEC との連携が重要になります。コードを読む際は、apr_uid_homepath_get が OS 依存である点に注意してください。
