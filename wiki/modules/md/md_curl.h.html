modules/md/md_curl.h

概要

md_curl.h は、mod_md モジュールにおける HTTP 通信バックエンドとして libcurl を利用するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

modules/md サブシステムにおいて、HTTP リクエストの抽象化レイヤー（md_http）の具体的な実装の一つとして機能します。md_http が定義する汎用的な HTTP 操作を、libcurl の機能を用いて実行するためのブリッジを提供します。

責務

  • md_http 実装構造体（md_http_impl_t）の取得関数の宣言。
  • HTTP 通信バックエンドの切り替えを可能にするための抽象化インターフェースの提供。

重要なシンボル

  • md_curl_get_impl(apr_pool_t *p): libcurl を使用した HTTP 実装を返す関数。md_http レイヤーが通信処理を初期化する際に呼び出されます。

制御フローとデータフロー

本ファイル自体は宣言のみを提供します。実際の処理は対となる md_curl.c に記述されており、以下のフローを担います。 1. md_curl_get_impl を通じて実装が取得される。 2. libcurl のコールバック（req_data_cb, resp_data_cb）が、Apache APR のバケットブリゲード（apr_bucket_brigade）と連携し、ストリームデータを変換する。 3. curl_status 関数により、libcurl のエラーコードが APR のステータスコードにマッピングされる。

直接依存

  • apr_pools.h: メモリ管理のための APR プール定義。

関連コード

  • modules/md/md_http.h: HTTP 通信の抽象化定義。本実装が準拠すべきインターフェースを定義しています。
  • modules/md/md_curl.c: 本ヘッダーで宣言された関数の実装および libcurl との統合ロジック。

読者向けメモ

  • 本ファイルは非常に軽量なインターフェース定義です。実装の詳細を確認する場合は、必ず対となる md_curl.c を参照してください。
  • libcurl のマルチインターフェース（CURLM）を利用した非同期的な通信処理が md_curl.c 内で行われているため、バケット処理のコールバック実装には注意が必要です。
