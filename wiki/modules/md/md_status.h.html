modules/md/md_status.h

概要

md_status.h は、Managed Domain (MD) の現在の状態や実行中のジョブに関する情報を、JSON 形式で集約・提供するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

modules/md サブシステムにおいて、管理対象ドメインのステータス監視およびジョブ管理の基盤となります。主に mod_md_status.c などから呼び出され、外部（管理者や監視ツール）に対して現在の証明書の状態やバックグラウンドジョブの進捗状況を可視化する役割を担います。

責務

  • **ステータス集約**: 個別または全 MD の状態を JSON 形式で生成する。
  • **ジョブ管理**: 証明書更新などのバックグラウンドタスク（md_job_t）のライフサイクル管理（ロード、保存、ログ記録、再試行制御）。
  • **統計情報の提供**: システム全体の稼働状況（完了数、エラー数など）のサマリー生成。

重要なシンボル

  • md_status_get_md_json(): 特定の MD に関する詳細なステータス情報を取得する。
  • md_status_take_stock(): システム内の全 MD の状態を要約し、統計情報を出力する。
  • md_job_t: バックグラウンドジョブの状態を保持する構造体。実行時間、結果、ログ、再試行回数などを管理する。
  • md_job_save() / md_job_load(): ジョブ状態の永続化と復元を行う。
  • md_job_log_append(): ジョブ実行履歴を記録する。

制御フローとデータフロー

1. **ステータス取得**: 外部からの要求に応じて md_status_get_json が呼ばれ、md_reg_t（レジストリ）から情報を抽出して JSON を構築します。 2. **ジョブ実行**: md_job_t インスタンスが作成され、md_store を介してディスク上の状態と同期されます。 3. **ログと結果**: ジョブの各ステップで md_job_log_append が呼ばれ、実行結果が md_result_t を通じて保存されます。 4. **再試行制御**: md_job_delay_on_errors がエラー発生時のバックオフ時間を計算し、次の実行タイミングを制御します。

直接依存

  • md_store.h: ジョブの永続化ストレージ操作に必須。
  • apr_time.h, apr_strings.h: APR (Apache Portable Runtime) による時間管理および文字列操作。
  • md_json.h (前方宣言): JSON 構造体の操作。

関連コード

  • md_status.c: 本ヘッダーで定義された関数の実装。
  • mod_md_status.c: HTTP レスポンスとしてステータスを返すための上位層の実装。
  • md_store.h: ジョブが永続化される基盤ストレージの定義。
  • md_result.h: ジョブ実行結果の構造体定義。

読者向けメモ

  • 本ファイルは「ステータス表示」と「ジョブ管理」という2つの側面を持っています。ジョブの永続化ロジックを追う場合は md_job_t 構造体のフィールド定義から読み始めるのが効率的です。
  • MD_STATUS_WITH_SCTS などのマクロにより、ビルド設定に応じて詳細な証明書透明性（CT）ログの出力有無が切り替わる設計になっています。
  • ジョブのログは md_json_t として保持されるため、デバッグ時には md_json 関連のユーティリティと併せて確認してください。
