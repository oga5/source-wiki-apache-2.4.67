modules/md/md_json.c

概要

md_json.c は、Jansson ライブラリをラップし、Apache Portable Runtime (APR) のメモリ管理と統合した JSON 操作レイヤーです。ACME プロトコル等で頻出する複雑な JSON 構造の読み書き、パス指定による要素へのアクセス、およびファイル/HTTP ストリームとの相互変換を提供します。

モジュール内での役割

modules/md サブシステムにおけるデータ交換フォーマットの標準化を担います。生ポインタの管理を隠蔽し、APR プールと連動した自動クリーンアップを提供することで、メモリリークを防ぎつつ、ACME 関連の複雑な設定やレスポンスを安全に処理します。

責務

  • **ライフサイクル管理**: APR プールと json_t の紐付け、およびクリーンアップの自動化。
  • **パスベースの操作**: 可変長引数（...）を用いた、ネストされた JSON オブジェクトへの直感的なアクセスと変更。
  • **型変換とシリアライズ**: JSON と C の基本型（数値、文字列、時間、配列）間の変換およびファイル/HTTP への入出力。
  • **イテレーション**: 配列やオブジェクトの要素を走査するためのコールバックベースのインターフェース提供。

重要なシンボル

ライフサイクルとコンテナ

  • md_json_t: json_t をラップする構造体。APR プールへのポインタを保持。
  • md_json_create, md_json_destroy: インスタンスの生成と破棄。
  • md_json_copy, md_json_clone: JSON ツリーのコピー（浅いコピーと深いコピー）。

セレクタ（パス操作）

  • jselect: 可変長引数で指定されたキーパスを辿り、対象の json_t を取得。
  • jselect_parent: パス上の親ノードを取得し、必要に応じてノードを生成する。
  • jselect_set_new: パスを指定して新しい値をセットする（所有権を移動）。

データ操作

  • md_json_getb, md_json_setb: 真偽値の取得・設定。
  • md_json_getn, md_json_setn: 数値（double）の取得・設定。
  • md_json_gets, md_json_sets: 文字列の取得・設定。
  • md_json_get_time, md_json_set_time: RFC 形式の日付文字列と apr_time_t の相互変換。

入出力

  • md_json_writef, md_json_readf: ファイルシステムとの読み書き。
  • md_json_read_http: HTTP レスポンスボディからの JSON 解析。

制御フローとデータフロー

  • **パス解決**: すべてのアクセサは jselect または jselect_parent を経由します。これらは va_list を走査し、JSON ツリーを再帰的に辿ります。
  • **メモリ所有権**: md_json_t は json_t の参照カウントを管理します。json_create で登録されたクリーンアップ関数により、APR プールが破棄される際に json_decref が自動的に呼ばれます。
  • **エラーパス**: jselect 系関数は、パスが存在しない場合や型が不一致の場合に APR_EINVAL や APR_ENOENT を返し、呼び出し元で適切にハンドリングされます。

読者向けメモ

  • **メモリ所有権**: md_json_t が保持する json_t は、APR プールの生存期間に依存します。プールを跨いで md_json_t を渡す場合は注意が必要です。
  • **可変長引数の終端**: md_json_has_key 等の可変長引数関数は、必ず NULL で終端する必要があります。
  • **Jansson の制約**: JSON_INLINE を無効化してインクルードしており、コンパイラの警告を抑制するためのプリプロセッサディレクティブが冒頭に記述されています。
  • **省略された詳細**: JSON パッチや複雑なマージ操作は実装されておらず、主に「設定ファイルの読み書き」と「ACME レスポンスの抽出」に特化した設計となっています。
