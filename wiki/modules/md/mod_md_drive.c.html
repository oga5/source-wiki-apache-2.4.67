modules/md/mod_md_drive.c

概要

本ファイルは、Apache HTTP Server の mod_md におけるマネージドドメイン（MD）の証明書自動更新プロセスを制御するバックグラウンド実行エンジンです。mod_watchdog を利用して、証明書の有効期限監視と更新処理を非同期に実行します。

モジュール内での役割

mod_md サブシステムにおいて、証明書更新の「駆動（Drive）」を担当します。設定の読み込みや証明書の管理を行う md_reg や md_store と連携し、定期的なタスクとして更新処理をスケジュールします。

責務

  • **監視の開始**: mod_watchdog を利用した更新用スレッドの登録と初期化。
  • **更新ジョブの実行**: 各 MD に対して、証明書の更新が必要か（期限切れ、OCSP 破棄、ARI による推奨など）を判定し、必要に応じて更新処理を実行。
  • **エラーリカバリ**: 再起動時に前回の更新失敗が残っている場合、STAGING 領域をパージして再試行を促す。
  • **イベント通知**: 更新の開始、成功、失敗などの状態変化をイベントハンドラ経由で通知。

重要なシンボル

  • md_renew_start_watching: 更新監視プロセスを開始するメインエントリーポイント。
  • run_watchdog: mod_watchdog から呼び出されるコールバック関数。更新ジョブのループを制御。
  • process_drive_job: 個別の MD に対して更新判定と実行を行うコアロジック。
  • md_renew_ctx_t: 更新プロセス全体の状態を保持するコンテキスト構造体。

制御フローとデータフロー

1. **初期化**: md_renew_start_watching が呼ばれ、監視対象の MD ごとに md_job_t を作成し、mod_watchdog にコールバックを登録。 2. **ループ実行**: run_watchdog が定期的に起動。各ジョブの next_run を確認し、期限に達したジョブに対して process_drive_job を実行。 3. **判定と更新**: process_drive_job 内で md_will_renew_cert を呼び出し、更新が必要か判断。必要であれば md_reg_renew を実行し、結果を STAGING 領域に保存。 4. **再スケジュール**: 全ジョブの処理後、最も早い next_run を計算し、wd_set_interval で次回の実行タイミングを更新。

直接依存

  • mod_watchdog.h: 非同期タスク実行のための基盤。
  • md_reg.h, md_store.h: 証明書登録管理およびストレージ操作。
  • md_acme.h: ACME プロトコルによる証明書取得ロジック。
  • mod_md_private.h, mod_md_config.h: モジュール内部設定とプライベート定義。

関連コード

  • mod_md_drive.h: 本ファイルの公開インターフェース。
  • md_reg.c: 証明書更新の具体的なロジック（md_reg_renew など）。
  • md_store.c: 更新中の証明書データが一時保存される STAGING 領域の管理。

読者向けメモ

  • **マルチプロセス環境**: mod_watchdog を使用しているため、サーバー内の単一プロセスでのみ更新処理が実行されることが保証されています。
  • **状態の永続化**: 更新処理の進捗は STAGING 領域の JSON ファイルに保存されます。これにより、プロセスが再起動しても更新状態が維持されます。
  • **ARI (ACME Renewal Information)**: 本ファイルは ARI をサポートしており、CA からの推奨更新タイミングを考慮した柔軟なスケジュール調整が行われます。
