modules/md/md_crypt.c

概要

md_crypt.c は、OpenSSL をラップして ACME プロトコルに必要な暗号化操作（鍵生成、署名、証明書解析、DER/PEM 変換）を提供するモジュールです。Apache HTTP Server の mod_md における暗号化の抽象化層として機能します。

モジュール内での役割

modules/md サブシステムにおいて、暗号化の低レイヤー処理を担います。md_acme.c や md_reg.c が高レベルな ACME トランザクションを扱うのに対し、本ファイルはそれらが利用する「鍵の生成」「証明書の検証」「署名の作成」といった具体的な暗号処理をカプセル化しています。

責務

  • **乱数生成**: プロセス固有のシードを用いた安全な乱数生成の初期化。
  • **鍵管理**: RSA/EC 鍵の生成、ファイルシステムへの保存・読み込み、HTTP レスポンスからの抽出。
  • **証明書処理**: X.509 証明書の解析、有効期限チェック、SAN（Subject Alternative Names）の抽出、チェーンの読み込み。
  • **署名とハッシュ**: SHA-256 ダイジェスト計算、HMAC、および CSR（証明書署名要求）の作成。
  • **ASN.1 変換**: ASN.1 時間形式の APR 時間への変換。

重要なシンボル

鍵管理

  • md_pkey_t: 秘密鍵を保持する構造体。EVP_PKEY をラップし、APR プールと紐付けられたライフサイクル管理を行う。
  • md_pkey_gen(): 指定された仕様（RSA/EC）に基づき秘密鍵を生成する。
  • md_pkey_fsave() / md_pkey_fload(): 秘密鍵のファイル入出力。

証明書管理

  • md_cert_t: X.509 証明書を保持する構造体。
  • md_cert_read_chain(): PEM 形式の証明書チェーンを解析し、配列に格納する。
  • md_cert_req_create(): CSR を生成し、Base64URL エンコードされた DER 形式で返す。
  • md_cert_self_sign(): 自己署名証明書を作成する（チャレンジ検証用）。

ユーティリティ

  • md_crypt_init(): OpenSSL の初期化と乱数シードのセットアップ。
  • md_asn1_generalized_time_get(): ASN.1 形式の時間を apr_time_t に変換。

制御フローとデータフロー

1. **初期化**: md_crypt_init が呼ばれると、getpid() を含むスタックデータを用いて RAND_seed を実行し、OpenSSL の乱数状態を確立する。 2. **鍵生成**: md_pkey_gen は md_pkey_spec_t を受け取り、EVP_PKEY_CTX を用いて RSA または EC 鍵を生成し、md_pkey_t に格納する。 3. **証明書解析**: md_cert_read_http や md_cert_fload は、DER/PEM データを d2i_X509 または PEM_read_bio_X509 で解析し、md_cert_t を作成する。 4. **クリーンアップ**: すべての構造体は APR プールと紐付いており、apr_pool_cleanup_register を介して EVP_PKEY_free や X509_free が自動的に呼び出される。

読者向けメモ

  • **メモリ管理**: すべての OpenSSL オブジェクトは APR プールに関連付けられたクリーンアップハンドラによって管理されます。手動で free を呼ぶ必要はありませんが、プールが破棄されるタイミングに注意してください。
  • **OpenSSL バージョン**: OpenSSL 1.0.x と 1.1.x/3.0.x の API 差異を吸収するためのマクロ（MD_OPENSSL_10x 等）が多用されています。
  • **スレッド安全性**: md_crypt_init は静的フラグ initialized を使用して一度だけ実行されるよう保護されています。
  • **省略箇所**: 本ファイルは OpenSSL の複雑なエラーハンドリングをラップしていますが、一部の低レベルな ASN.1 拡張の解析は、特定の ACME 要件に特化して実装されています。
