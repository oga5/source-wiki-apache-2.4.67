modules/md/md.h

概要

modules/md/md.h は、Apache HTTP Server の Managed Domain (MD) サブシステムにおける中心的なデータ定義と定数、およびドメイン管理のための基本操作を提供するヘッダーファイルです。MD は、証明書の自動取得・更新を管理するための論理的なドメイン単位を指します。

モジュール内での役割

本ファイルは modules/md モジュール全体の「共通言語」として機能します。ACME プロトコルによる証明書管理、ストレージ操作、設定管理など、他のすべてのサブコンポーネントが参照するデータ構造 md_t を定義しています。

責務

  • **データモデルの定義**: Managed Domain の状態、設定、および証明書関連のメタデータを保持する構造体の定義。
  • **定数管理**: ACME プロトコル、HSTS、タイムアウト、および JSON シリアライズ用のキー文字列の集約。
  • **ドメイン操作の抽象化**: 複数のドメインリストに対する包含関係の判定や、重複チェックなどのユーティリティ関数の提供。
  • **シリアライズ**: md_t 構造体と JSON 形式間の相互変換インターフェースの提供。

重要なシンボル

データ構造

  • md_t: MD の設定と状態を保持するメイン構造体。ドメイン名、連絡先、CA 設定、更新ポリシー、現在の状態（md_state_t）を保持。
  • md_pubcert_t: 公開証明書チェーンと対応する秘密鍵のファイルパスを保持する構造体。
  • md_state_t: MD のライフサイクル状態（COMPLETE, INCOMPLETE, ERROR 等）。

定数・マクロ

  • **設定キー**: MD_KEY_* で始まる多数のマクロ。JSON ストレージにおけるフィールド名を定義。
  • **タイムアウト**: MD_TIME_* マクロ。更新ウィンドウや警告ウィンドウのデフォルト値を定義。
  • **プロトコル**: PROTO_ACME_TLS_1 等のプロトコル識別子。

操作関数

  • **ライフサイクル**: md_create, md_create_empty, md_clone, md_copy。
  • **検索・判定**: md_contains, md_domains_overlap, md_get_by_name 等。
  • **変換**: md_to_json, md_from_json。

制御フローとデータフロー

  • **データフロー**: 設定ファイルから読み込まれた MD 定義は md_t 構造体に格納されます。この構造体は md_to_json を通じて永続化され、md_from_json を通じて再構築されます。
  • **制御フロー**: 多くの関数は apr_array_header_t を引数に取り、ドメインリストの走査や比較を行います。md_contains などの述語関数は、設定されたドメインとリクエストされたドメインの照合に使用されます。

直接依存

  • apr_time.h: 時間計算用。
  • md_time.h: md_timeslice_t 等の MD 固有の時間管理定義。
  • md_version.h: モジュールバージョン情報。

関連コード

  • md_reg.h: MD の登録と管理を行うレジストリ層。
  • md_store.h: md_t を JSON としてディスクに保存・読み込みを行うストレージ層。
  • md_json.h: JSON 操作の基盤。

読者向けメモ

  • **メモリ管理**: md_t 構造体およびその内部の配列は、生成時に渡される apr_pool_t に依存します。md_clone を使用する際は、適切なプールが指定されていることを確認してください。
  • **不変条件**: md_t の name は一意である必要があり、state は md_reg などの上位モジュールによって更新されます。
  • **注意点**: 本ファイルは定義のみを提供します。実装の詳細は各 .c ファイル（md_core.c 等）に委譲されています。
  • **省略された詳細**: 複雑な ACME 認証フローや HTTP 通信の詳細は、このヘッダーファイルでは抽象化されており、md_acme.h や md_http.h で定義されています。
