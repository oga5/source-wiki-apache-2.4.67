modules/md/mod_md_os.h

概要

mod_md_os.h は、Apache HTTP Server の Managed Domains (MD) モジュールにおいて、OS 固有の低レイヤー操作を抽象化するためのヘッダーファイルです。主にファイルシステムの権限管理と、サーバープロセスのライフサイクル制御に関するインターフェースを提供します。

モジュール内での役割

modules/md サブシステム内で、プラットフォーム依存の処理を分離する役割を担います。mod_md が実行環境（Unix系、Windows等）に依存する操作を行う際、直接システムコールを呼び出すのではなく、このヘッダーで定義された関数を経由することで、コードの移植性と保守性を維持しています。

責務

  • **ファイル権限の調整**: MD が管理する証明書や設定ファイルに対し、HTTPD ワーカープロセスがアクセス可能な権限（所有者変更など）を付与する。
  • **プロセス間シグナル制御**: サーバーの再起動や状態遷移をトリガーするためのシグナル送信機能の提供。
  • **プラットフォーム抽象化**: OS ごとに異なる実装（または未実装）を APR ステータスコードでラップし、呼び出し元に統一的なインターフェースを提供する。

重要なシンボル

  • md_try_chown: 指定されたパスの所有権を変更する。ルート権限で実行されている場合のみ有効。
  • md_make_worker_accessible: HTTPD ワーカープロセスが MD 関連ファイルにアクセスできるよう、適切な所有権を設定する。
  • md_server_graceful: サーバーに対してグレースフル再起動のシグナルを送信する。

制御フローとデータフロー

  • **権限設定**: md_make_worker_accessible が呼び出されると、内部で ap_unixd_config.user_id を参照し、md_try_chown を通じてファイルシステム上の所有者を更新します。
  • **シグナル送信**: md_server_graceful は、現在のプロセスから親プロセスに対して AP_SIG_GRACEFUL を送信します。これにより、サーバーの再起動プロセスが開始されます。
  • **エラーハンドリング**: 各関数は apr_status_t を返し、OS が操作をサポートしていない場合は APR_ENOTIMPL を返して呼び出し元が適切にフォールバックできるようにしています。

直接依存

  • apr_pools.h (APR プール管理)
  • httpd.h (Apache HTTP Server コア定義)

関連コード

  • modules/md/mod_md_os.c: 本ヘッダーで宣言された関数の実装。プラットフォームごとの #ifdef 分岐が集中しています。
  • modules/md/md_store.c: MD が管理するファイルストレージの操作において、これらの権限管理関数が利用されます。

読者向けメモ

  • 本ファイルは、主に Unix 系環境での権限管理（chown）を想定しています。Windows 等のプラットフォームでは、多くの関数が APR_ENOTIMPL を返すように設計されています。
  • md_try_chown を呼び出す際は、ルート権限での実行かどうかが内部でチェックされるため、呼び出し側で過度な権限チェックを行う必要はありません。
  • プロセス制御（md_server_graceful）は、サーバーの運用状態に直接影響を与えるため、呼び出しタイミングには注意が必要です。
