modules/md/md_acme.h

概要

md_acme.h は、ACME (Automated Certificate Management Environment) プロトコルを Apache HTTP Server の mod_md モジュール内で実装するための中心的なインターフェースです。ACME サーバーとの通信、アカウント管理、およびリクエストのライフサイクルを定義します。

モジュール内での役割

modules/md サブシステムにおける ACME プロトコル層の基盤です。このヘッダーは、ACME サーバーとの対話に必要なデータ構造と、非同期的なリクエスト処理のためのコールバック型を定義し、上位のドライブロジック（md_acme_drive.c 等）と下位の HTTP/JWS 通信層を仲介します。

責務

  • **ACME サーバー状態管理**: サーバーのディレクトリ情報、バージョン（v1/v2）、および認証状態の保持。
  • **アカウント管理**: ACME アカウントのロード、保存、利用規約（ToS）の同意管理。
  • **リクエスト抽象化**: HTTP POST/GET リクエストの構築、JWS 署名、およびレスポンス処理のコールバックベースのディスパッチ。
  • **エラーハンドリング**: ACME プロトコル固有のエラーコードと Apache ステータスコードの変換。

重要なシンボル

サーバーと状態管理

  • md_acme_t: ACME サーバーインスタンスを保持するメイン構造体。サーバーURL、アカウント情報、APIエンドポイント（v1/v2）、および通信用 HTTP ハンドルを管理。
  • md_acme_state_t: ACME プロセスの進行状況（登録済み、検証済み、証明書取得済みなど）を示す列挙型。

リクエスト処理

  • md_acme_req_t: 個別の ACME リクエストのコンテキスト。リクエストのメソッド、JSON ペイロード、レスポンスヘッダー、およびコールバック関数を保持。
  • md_acme_post_fn / md_acme_POST: ACME サーバーへの POST リクエストを送信し、結果をコールバックで処理する主要なディスパッチ関数。
  • md_acme_req_init_cb 等: リクエストの初期化、JSON レスポンス処理、エラー処理のためのコールバック型定義。

アカウント管理

  • md_acme_use_acct / md_acme_save_acct: ローカルストアと ACME インスタンス間でのアカウントデータの同期。
  • md_acme_agree / md_acme_check_agreement: ToS 同意の検証と更新。

制御フローとデータフロー

1. **初期化**: md_acme_create でインスタンスを生成し、md_acme_setup でサーバーのディレクトリ情報を取得して md_acme_t を初期化します。 2. **リクエスト実行**: md_acme_POST が呼び出されると、md_acme_req_t が生成されます。on_init コールバックでペイロードが準備され、JWS 署名を経て送信されます。 3. **レスポンス処理**: HTTP レスポンス受信後、on_json または on_res コールバックが呼び出され、結果が md_result_t に格納されます。 4. **エラー変換**: md_acme_problem_is_input_related により、ACME サーバーからのエラーが設定ミスによるものか、一時的なサーバーエラーかを判定します。

直接依存

  • apr_pool_t, apr_hash_t, apr_table_t: Apache Portable Runtime によるメモリ管理とデータ構造。
  • md_json.h: ACME プロトコルで多用される JSON メッセージの処理。
  • md_http.h: 低レベルの HTTP 通信。
  • md_jws.h: ACME リクエストに必要な JWS 署名生成。

関連コード

  • md_acme.c: 本ヘッダーで定義されたエラー判定ロジックの実装。
  • md_acme_acct.c: アカウントの永続化と管理の詳細。
  • md_acme_authz.c: ドメイン検証（チャレンジ）フロー。
  • md_acme_order.c: 証明書発行オーダーの管理。

読者向けメモ

  • **メモリ管理**: md_acme_t および md_acme_req_t は、関連付けられた apr_pool_t の生存期間に依存します。リクエスト処理中のメモリリークを防ぐため、コールバック内での適切なプール利用が求められます。
  • **スレッド安全性**: 本モジュールは Apache のリクエスト処理コンテキスト内で動作することを前提としており、グローバルな状態変更には注意が必要です。
  • **省略された詳細**: 本ヘッダーはインターフェース定義に集中しており、JWS 署名の具体的な暗号化アルゴリズムや、HTTP 通信の再試行ロジックの詳細は md_jws.c や md_http.c に委譲されています。
