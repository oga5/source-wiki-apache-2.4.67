modules/md/md_jws.c

概要

本ファイルは、ACMEプロトコル等で使用される **JWS (JSON Web Signature)** の生成および関連する鍵情報の処理を実装しています。RSA署名およびHMAC署名を用いたメッセージの構築を担います。

モジュール内での役割

modules/md サブシステムにおいて、暗号化ユーティリティ（md_crypt）とJSON操作（md_json）を橋渡しし、ACMEサーバーとの通信に必要な署名済みメッセージ形式を生成する層です。

責務

  • **JWK (JSON Web Key) の生成**: 公開鍵からACMEで要求されるJWK構造を作成する。
  • **JWS 署名生成**: ペイロードと保護ヘッダーを結合し、Base64URLエンコード後に署名を付与する。
  • **鍵の指紋 (Thumbprint) 計算**: RFC 8555 に準拠した公開鍵のハッシュ値を算出する。
  • **HMAC 署名**: 共有鍵を用いたメッセージ認証コードの生成。

重要なシンボル

  • md_jws_sign: RSA鍵を使用してJWSメッセージを構築する主要関数。
  • md_jws_get_jwk: 公開鍵からJWK JSONオブジェクトを生成する。
  • md_jws_pkey_thumb: 公開鍵の正規化されたJSON表現からSHA-256指紋を計算する。
  • md_jws_hmac: HMAC-SHA256を用いた署名生成を行う。

制御フローとデータフロー

1. **署名生成フロー**:

  • prot_fields (JSON) に alg (RS256/HS256) を追加。
  • key_id が未指定の場合、md_jws_get_jwk を呼び出し jwk フィールドをヘッダーに埋め込む。
  • ヘッダーとペイロードをそれぞれ Base64URL エンコードし、header.payload 形式の文字列を作成。
  • md_crypt を介して署名を行い、最終的な JSON オブジェクトを構築する。

    • 2. **指紋計算フロー**:

  • 公開鍵の e と n を取得し、特定の順序で JSON 文字列化する。
  • その文字列に対して SHA-256 ダイジェストを計算し、Base64URL エンコードする。

直接依存

  • apr_lib.h, apr_strings.h, apr_tables.h, apr_buckets.h: APR基本ユーティリティ。
  • md_crypt.h: 署名およびハッシュ計算の低レイヤー処理。
  • md_json.h: JSON構造の構築とシリアライズ。
  • md_util.h: Base64URLエンコード等の補助関数。
  • md_jws.h
  • md_log.h

関連コード

  • md_crypt.c: 本ファイルが呼び出す署名アルゴリズムの実装。
  • md_acme.c: 本ファイルで生成されたJWSメッセージを実際に送信する上位モジュール。

読者向けメモ

  • **順序の重要性**: md_jws_pkey_thumb で生成される文字列は、JSONのキー順序や空白が厳密に定義されています。指紋計算の際は、この関数が返す形式がRFC 8555の要件を満たしていることに依存しています。
  • **メモリ管理**: すべての関数は apr_pool_t を引数に取り、生成される JSON オブジェクトや文字列はプールによって管理されます。
  • **エラーハンドリング**: 署名プロセスで失敗した場合は APR_SUCCESS 以外のステータスが返され、pmsg は NULL に設定されます。呼び出し側での戻り値チェックが必須です。
