modules/md/mod_md_config.h

概要

本ヘッダーファイルは、Apache HTTP Server の mod_md モジュールにおける設定管理の基盤を定義します。サーバー全体（グローバル）および仮想ホスト単位での設定構造体と、それらを操作するためのインターフェースを提供します。

モジュール内での役割

mod_md の設定システムの中核です。Apache の設定ディレクティブ（<MDomain> 等）から読み込まれた値を保持し、実行時に他のコンポーネント（ACME 処理、証明書管理、OCSP ステープリング等）が設定値にアクセスするための窓口となります。

責務

  • **設定構造体の定義**: グローバル設定 (md_mod_conf_t) とサーバー/仮想ホスト単位の設定 (md_srv_conf_t) のスキーマ定義。
  • **設定のライフサイクル管理**: Apache の設定フェーズ（作成、マージ、ポストコンフィグ）で利用される関数の宣言。
  • **アクセサの提供**: 実行時に設定値を取得するための型安全なインターフェースの提供。

重要なシンボル

  • md_mod_conf_t: モジュール全体のグローバル設定。ポート番号、ベースディレクトリ、OCSP 関連のデフォルト値などを保持。
  • md_srv_conf_t: 特定のサーバー（仮想ホスト）ごとの設定。ACME 関連のプロトコル設定や更新ウィンドウなどが含まれる。
  • md_config_var_t: 設定値を取得する際に使用する列挙型。md_config_gets や md_config_geti の引数として利用。
  • md_config_get() / md_config_cget(): サーバーまたは接続コンテキストから有効な設定を取得する主要な関数。

制御フローとデータフロー

1. **初期化**: Apache 起動時に md_config_create_svr が呼ばれ、デフォルト値で構造体が初期化されます。 2. **設定読み込み**: 設定ファイル解析時に md_cmds を通じて値が構造体に格納されます。 3. **マージ**: md_config_merge_svr により、グローバル設定と仮想ホスト設定が階層的に統合されます。 4. **利用**: 実行時、各コンポーネントは md_config_get を呼び出し、現在のコンテキストに適用される設定値を取得します。

直接依存

本ファイルは構造体定義が中心であり、以下の型定義に依存しています。

  • apr_pool_t, apr_hash_t, apr_array_header_t (Apache Portable Runtime)
  • server_rec, conn_rec (Apache HTTP Server Core)
  • md_t, md_require_t, md_timeslice_t (モジュール内共通型)

関連コード

  • mod_md_config.c: 本ヘッダーで宣言された関数の実装およびデフォルト値の定義。
  • mod_md.c: モジュールのメインエントリポイント。設定のフック処理が記述されています。
  • md_reg.h: 証明書登録情報の管理に関連する定義。

読者向けメモ

  • 本ファイルは、mod_md がどのように設定を保持しているかを理解するための最優先の参照先です。
  • md_config_var_t を使用したアクセサ（md_config_gets 等）は、設定値の取得を抽象化しているため、直接構造体のメンバにアクセスするよりも推奨されます。
  • md_srv_conf_t 内の mc ポインタを通じて、常にグローバル設定へアクセス可能な構造になっています。
