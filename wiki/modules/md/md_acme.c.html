modules/md/md_acme.c

概要

md_acme.c は、ACME プロトコル（RFC 8555等）を用いた証明書管理の通信層を担う中心的なファイルです。ACME サーバーとのディレクトリ情報の取得、リクエストの署名（JWS）、ノンストップな通信のためのノンスト管理、およびサーバーからのエラー（Problem）の解釈を抽象化します。

モジュール内での役割

modules/md サブシステムにおいて、ACME プロトコルの「クライアント」としての役割を果たします。md_http.c を利用して低レベルな HTTP 通信を行い、md_jws.c を利用して署名付きリクエストを生成します。上位層（md_acme_order.c や md_acme_acct.c）が ACME のビジネスロジックを実装するのに対し、本ファイルはプロトコル特有の通信規約を隠蔽します。

責務

  • **ACME プロトコル通信**: GET/POST リクエストの抽象化と、ACME サーバーへの適応（v1/v2 バージョン判定）。
  • **リクエストのライフサイクル管理**: リクエストごとのメモリプール管理、ノンスト（Nonce）の更新、再試行ロジックの実行。
  • **エラーハンドリング**: ACME サーバーから返される application/problem+json の解析と、Apache 内部ステータスへのマッピング。
  • **アカウント管理**: アカウント情報のロード、検証、および保存のインターフェース提供。

重要なシンボル

リクエスト・通信制御

  • md_acme_req_create(): リクエスト用のメモリプールを確保し、初期状態を構築する。
  • md_acme_req_send(): ACME リクエストの送信エンジン。Nonce の付与、署名、再試行ロジックを内包する。
  • md_acme_POST() / md_acme_GET(): 上位層向けの公開 API。コールバック関数を介して非同期的なレスポンス処理を行う。

プロトコル・エラー解析

  • acme_problem_status_t: ACME エラー型と Apache ステータスコードの対応表。
  • inspect_problem(): サーバーからのエラーレスポンスを解析し、再試行が必要か（APR_EAGAIN）を判定する。
  • md_acme_problem_is_input_related(): エラーがユーザー入力に起因するものか判定する。

セットアップ・状態管理

  • md_acme_create(): ACME サーバーとの通信用コンテキスト（md_acme_t）を初期化する。
  • md_acme_setup(): サーバーのディレクトリ情報を取得し、API エンドポイントを特定する。

制御フローとデータフロー

1. **初期化**: md_acme_create でコンテキストを作成し、md_acme_setup でサーバーのディレクトリ情報を取得して md_acme_t を構成する。 2. **リクエスト送信**: md_acme_POST 等が呼ばれると、md_acme_req_create でリクエスト専用プールが作成される。 3. **署名と送信**: md_acme_req_send 内で acmev2_req_init が呼ばれ、md_jws_sign を用いてペイロードが署名される。 4. **レスポンス処理**: on_response コールバックが HTTP レスポンスを捕捉。Nonce を更新し、JSON 解析後に成功時は on_json、エラー時は inspect_problem へ委譲する。 5. **終了**: md_acme_req_done で結果を md_acme_t の last フィールドにコピーし、リクエスト用プールを破棄する。

直接依存

  • **APR**: apr_pool_t, apr_table_t 等によるメモリとデータ構造の管理。
  • **md_http.h**: HTTP 通信の実行。
  • **md_jws.h**: ACME リクエストの署名。
  • **md_json.h**: ACME プロトコルで多用される JSON の読み書き。
  • **md_acme.h**: 公開インターフェースとデータ構造の定義。

読者向けメモ

  • **メモリ管理**: md_acme_req_t はリクエストごとに独自の apr_pool_t を持ちます。md_acme_req_done で確実に破棄されるため、リクエスト処理中に確保したメモリはリクエストプールに紐付ける必要があります。
  • **再試行**: md_acme_req_send は APR_EAGAIN を受け取ると再帰的に再試行を行います。無限ループを防ぐため max_retries が設定されています。
  • **省略された詳細**: 本ファイルは通信の「実行」に特化しており、証明書の生成や秘密鍵の暗号化ロジックは md_crypt.c に委譲されています。また、ACME v1 の実装は v2 への移行に伴い、現在は v2 が優先される設計となっています。
