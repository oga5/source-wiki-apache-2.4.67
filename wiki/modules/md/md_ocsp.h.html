modules/md/md_ocsp.h

概要

md_ocsp.h は、Apache HTTP Server のマネージドドメイン（MD）モジュールにおける OCSP（Online Certificate Status Protocol）ステータス確認機能のインターフェースを定義するヘッダーファイルです。証明書の失効状態を検証し、その結果を管理するためのデータ構造と関数を提供します。

モジュール内での役割

modules/md サブシステムにおいて、証明書の有効性をリアルタイムに検証する役割を担います。mod_md が証明書の更新や管理を行う際、外部の OCSP レスポンダと通信して証明書が失効していないかを確認するためのバックエンドとして機能します。

責務

  • OCSP レスポンダへの問い合わせ管理（レジストリの作成と保持）。
  • 証明書 ID の生成と、それに基づくステータス取得。
  • 定期的な OCSP レスポンスの更新（renew）と古いレスポンスのクリーンアップ。
  • OCSP ステータスの要約および詳細情報の JSON 出力。

重要なシンボル

  • md_ocsp_reg_t: OCSP 処理を管理するメインのレジストリ構造体。設定やキャッシュ、スレッドセーフなアクセスを制御します。
  • md_ocsp_cert_stat_t: 証明書のステータス（GOOD, REVOKED, UNKNOWN）を表す列挙型。
  • md_ocsp_get_status: 指定された証明書の OCSP ステータスを取得し、コールバック経由で DER 形式のデータを渡す主要関数。
  • md_ocsp_renew: 期限切れに近い OCSP レスポンスを再取得するための更新処理。

制御フローとデータフロー

1. **初期化**: md_ocsp_reg_make によりレジストリが作成され、ストアやプロキシ設定が保持されます。 2. **検証**: md_ocsp_get_status または md_ocsp_get_meta が呼び出されると、内部で証明書の ID が計算され、キャッシュの確認またはネットワーク経由での OCSP 問い合わせが行われます。 3. **更新**: md_ocsp_renew が定期的に実行され、next_run に基づいてレスポンダから最新のステータスを取得し、結果をストアに保存します。 4. **出力**: md_ocsp_get_summary 等を通じて、現在の OCSP 状態が JSON 形式でシステムに提供されます。

直接依存

本ヘッダーファイル自体は他のヘッダーをインクルードしていませんが、前方宣言により以下の構造体を利用します。

  • md_data_t, md_json_t, md_store_t, md_timeslice_t, md_job_t

関連コード

  • modules/md/md_ocsp.c: 本ヘッダーで宣言された関数の実装。OpenSSL の OCSP API を直接操作します。
  • modules/md/mod_md_ocsp.c: Apache モジュールとしての OCSP 統合レイヤー。
  • modules/md/md_crypt.h: 証明書操作や X.509 関連のユーティリティを提供します。

読者向けメモ

  • 本ファイルは OpenSSL の OCSP 実装をラップする高レベルな抽象化層です。
  • OCSP レスポンスは DER 形式で扱われ、md_ocsp_copy_der コールバックを通じて外部に渡されます。
  • ネットワーク通信を伴うため、タイムアウトや再試行のロジックは md_ocsp.c 側の実装に依存しています。
  • 証明書の失効確認を自動化したい場合、md_ocsp_reg_t を中心としたライフサイクル管理を理解することが重要です。
