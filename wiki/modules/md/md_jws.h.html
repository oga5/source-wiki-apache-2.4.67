modules/md/md_jws.h

概要

md_jws.h は、ACME プロトコル等で使用される **JWS (JSON Web Signature)** の生成と管理を行うためのインターフェースを定義しています。RSA 鍵を用いた署名や、RFC 8555 で規定される鍵のサムプリント生成機能を提供します。

モジュール内での役割

modules/md サブシステムにおいて、暗号化された通信や認証情報の署名を行うためのユーティリティ層に位置します。md_crypt.h が提供する低レベルな暗号化プリミティブを、JWS という特定のデータ形式に適合させる役割を担います。

責務

  • **JWS メッセージの構築**: ペイロードと保護ヘッダーを組み合わせ、署名付き JSON オブジェクトを生成する。
  • **鍵の表現**: 公開鍵から JWK (JSON Web Key) 形式への変換およびサムプリントの計算。
  • **署名アルゴリズムの抽象化**: RS256 や HMAC 署名の実装を隠蔽し、上位モジュールへ簡潔な API を提供する。

重要なシンボル

  • md_jws_sign: ペイロードに署名を付与し、JWS 形式の JSON を作成する主要関数。
  • md_jws_pkey_thumb: RFC 8555 に準拠した公開鍵のサムプリント（指紋）を生成する。
  • md_jws_get_jwk: 公開鍵から JWK 構造体を生成する。
  • md_jws_hmac: HMAC (HS256) を用いた署名付きメッセージを生成する。

制御フローとデータフロー

1. **入力**: 署名対象の payload、ヘッダー情報 prot_fields、および秘密鍵 pkey を受け取ります。 2. **処理**:

  • md_jws_sign は、ヘッダーに alg: RS256 を付与し、必要に応じて kid または jwk を挿入します。
  • ペイロードとヘッダーを Base64URL エンコードし、連結した文字列に対して md_crypt_sign64 を呼び出して署名を生成します。

    • 3. **出力**: 署名結果を含む JSON オブジェクトを pmsg に格納して返却します。

直接依存

  • apr_pools.h (メモリ管理)
  • md_json.h (JSON 操作)
  • md_crypt.h (暗号化処理)
  • md_util.h (Base64URL エンコード等)

関連コード

  • md_jws.c: 上記ヘッダーで宣言された関数の実装。
  • md_crypt.h: 署名アルゴリズムの基盤となる暗号化関数群。
  • md_acme.h: 本モジュールで生成された JWS を利用して ACME サーバーと通信する上位モジュール。

読者向けメモ

  • 本ファイルで生成される JWS は、ACME プロトコルの仕様に厳密に従っています。特に md_jws_pkey_thumb で計算されるサムプリントは、JSON の空白や順序に依存するため、実装を変更する際は RFC 8555 の仕様を再確認してください。
  • md_jws_sign において key_id が NULL の場合、自動的に jwk フィールドが生成される設計になっています。
