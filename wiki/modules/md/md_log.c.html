modules/md/md_log.c

概要

md_log.c は、mod_md モジュール全体で使用される抽象化されたログ出力インターフェースを提供します。Apache HTTP Server の標準的なログ機能に依存しつつ、モジュール固有のログレベル管理や出力先制御を柔軟に行うためのラッパー層です。

モジュール内での役割

mod_md は ACME プロトコルを扱う複雑なモジュールであり、多くのサブコンポーネントからログ出力が発生します。本ファイルは、各コンポーネントが直接 Apache の ap_log_perror 等を呼び出すのではなく、md_log_* インターフェースを介すことで、ログの出力制御（フィルタリングや出力先の差し替え）を一元管理する役割を担います。

責務

  • ログレベルの名称管理とマッピング。
  • 実行時のログ出力可否判定（md_log_is_level）。
  • ログ出力関数の抽象化（md_log_perror）。
  • 外部からログ出力ロジックを注入するためのコールバック設定（md_log_set）。

重要なシンボル

  • md_log_set(): ログ出力のバックエンド（コールバック関数）を登録します。テストや環境に応じたログ出力の差し替えを可能にします。
  • md_log_is_level(): 指定されたログレベルが現在有効かどうかを判定します。不要な文字列構築処理を避けるために使用されます。
  • md_log_perror(): 実際にログを出力するメイン関数です。可変長引数を受け取り、登録されたコールバックへ転送します。
  • level_names: ログレベルの列挙型に対応する文字列配列です。

制御フローとデータフロー

1. **初期化**: モジュール起動時に md_log_set が呼び出され、ログ出力用のコールバック関数とコンテキスト（log_baton）が静的変数に保持されます。 2. **判定**: 各コンポーネントは md_log_is_level を呼び出し、現在の設定でログ出力が必要かを確認します。 3. **出力**: 出力が必要な場合、md_log_perror が呼び出されます。この関数は va_list を介して引数を展開し、登録済みの log_printv コールバックへ処理を委譲します。

直接依存

  • apr_lib.h, apr_strings.h, apr_buckets.h: APR (Apache Portable Runtime) の文字列操作およびメモリ管理機能。
  • md_log.h: 本ファイルのヘッダーであり、ログレベルの定義と関数プロトタイプを提供。

関連コード

  • mod_md.c: モジュール全体の初期化プロセスにおいて、md_log_set を呼び出し、Apache のログシステムと md_log を結びつける役割を果たします。

読者向けメモ

  • 本モジュールは「ログ出力の抽象化」に特化しており、実際のログ出力先（ファイル、コンソール、syslog 等）の決定は、md_log_set で渡されるコールバックの実装に依存します。
  • LOG_BUFFER_LEN は定義されていますが、現在の実装では直接使用されていません。将来的なバッファリングやフォーマット処理の拡張用と考えられます。
  • ログレベルの判定には apr_pool_t が渡されますが、これはコールバック側でメモリ管理やコンテキスト判定を行うために利用されます。
