modules/md/md_http.c

概要

md_http.c は、modules/md モジュールにおける HTTP 通信の抽象化レイヤーを提供します。特定の HTTP クライアント実装（例: libcurl）に依存せず、Apache Portable Runtime (APR) を介してリクエストの生成、設定、および実行を管理するインターフェースを定義します。

モジュール内での役割

md_http は、ACME プロトコル等の通信を行う上位レイヤーと、実際のネットワーク通信を行う下位実装（md_curl.c など）の間のブリッジとして機能します。プラグイン可能なアーキテクチャを採用しており、実行時に実装を切り替えることが可能です。

責務

  • **HTTP セッション管理**: md_http_t インスタンスの生成とライフサイクル管理。
  • **リクエストの構築**: GET/POST リクエストの生成、ヘッダー設定、ボディのバケットブリゲードへの変換。
  • **設定の抽象化**: タイムアウト、プロキシ、CA ファイル、Unix ソケットなどの通信パラメータの統一的な設定。
  • **実装のディスパッチ**: 登録された md_http_impl_t を通じて、実際の通信処理を委譲。

重要なシンボル

セッション管理

  • md_http_t: HTTP クライアントのコンテキスト。プール、バケットアロケータ、実装へのポインタを保持。
  • md_http_create(): セッションの初期化と実装の初期化を行う。
  • md_http_use_implementation(): 使用する HTTP 実装（md_http_impl_t）を登録する。

リクエスト操作

  • md_http_request_t: 個別の HTTP リクエスト状態。
  • md_http_GET_create() / md_http_POST_create(): リクエストの生成ファクトリ。
  • md_http_perform(): 単一リクエストの同期実行。
  • md_http_multi_perform(): 複数のリクエストを並行処理するためのディスパッチャ。

設定・アクセサ

  • md_http_set_timeout() / md_http_set_connect_timeout(): タイムアウト設定。
  • md_http_set_response_limit(): レスポンスサイズの制限設定。

制御フローとデータフロー

1. **初期化**: md_http_use_implementation で実装を注入し、md_http_create でセッションを開始。 2. **リクエスト構築**: req_create がリクエスト専用のサブプールを作成し、親セッションから設定を継承。 3. **ボディ処理**: req_set_body_data が md_data_t を apr_bucket_brigade に変換し、リクエストにアタッチ。 4. **実行**: md_http_perform が内部の impl->perform を呼び出し、実際のネットワーク I/O を実行。 5. **クリーンアップ**: md_http_req_destroy がリクエスト専用プールを破棄し、実装固有のクリーンアップ関数を呼び出す。

直接依存

  • apr_buckets.h: HTTP ボディのストリーム処理（バケットブリゲード）。
  • md_http.h: 公開インターフェース定義。
  • md_log.h / md_util.h: ログ出力およびユーティリティ関数。
  • apr_lib.h
  • apr_strings.h
  • assert.h

関連コード

  • md_curl.c: 本モジュールの主要な実装（libcurl を使用）。
  • md_http.h: データ構造の定義とコールバックシグネチャ。

読者向けメモ

  • **メモリ管理**: md_http_request_t は専用のサブプールを持ち、md_http_req_destroy で明示的に破棄する必要があります。
  • **実装の分離**: md_http_t 内の impl_data は、実装側が自由に状態を保持するために使用されます。
  • **制約**: cur_impl は静的変数であり、プロセス全体で単一の実装が共有されます。
  • **省略箇所**: md_http_impl_t の構造体定義自体は md_http.h に存在し、本ファイルではそのディスパッチロジックのみを扱っています。
