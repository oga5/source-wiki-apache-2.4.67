modules/md/mod_md_status.h

概要

mod_md_status.h は、Apache HTTP Server の mod_status モジュールと連携し、管理対象ドメイン（Managed Domains: MD）の証明書ステータス情報を外部へ提供するためのインターフェースを定義します。

モジュール内での役割

mod_md モジュールの一部として、サーバーの稼働状況レポート機能に証明書の状態（有効期限、OCSP ステータス等）を統合する役割を担います。

責務

  • 証明書ステータス取得用の HTTP リクエストハンドラの宣言。
  • mod_status の出力に MD 関連情報を追加するためのフック関数の提供。
  • サーバー設定に基づいた証明書情報の抽出とフォーマットの仲介。

重要なシンボル

  • md_http_cert_status(request_rec *r): 特定のパス（/.httpd/certificate-status）へのリクエストに対し、JSON 形式で証明書ステータスを返すハンドラ。
  • md_domains_status_hook(request_rec *r, int flags): mod_status の出力にドメイン情報を追加するためのフック。
  • md_ocsp_status_hook(request_rec *r, int flags): mod_status の出力に OCSP ステータス情報を追加するためのフック。

制御フローとデータフロー

1. **リクエスト受信**: mod_status がリクエストを処理する際、または特定のステータスパスへのアクセスが発生します。 2. **フック実行**: mod_md が登録したフック関数が呼び出され、md_reg（レジストリ）や md_ocsp から現在の証明書状態を取得します。 3. **データ生成**: md_status_get_md_json 等の内部関数を介して、証明書の有効性や OCSP 応答が JSON オブジェクトとして構築されます。 4. **レスポンス**: 構築されたデータが HTTP レスポンスとしてクライアントに返却されます。

直接依存

本ヘッダーファイル自体は他のヘッダーへの依存を持ちませんが、実装ファイル（mod_md_status.c）は以下に依存します。

  • httpd.h, mod_status.h: Apache サーバーのコアおよびステータスモジュール機能。
  • md.h, md_reg.h, md_ocsp.h: 管理対象ドメインの定義およびステータス管理機能。

関連コード

  • modules/md/mod_md_status.c: 上記関数の実装。
  • modules/md/md_status.c: ステータス情報の JSON 生成ロジック。
  • modules/md/md_reg.h: 証明書情報の管理と検索。

読者向けメモ

  • 本ファイルで宣言されている関数は、主に Apache の mod_status 拡張として機能します。
  • ステータス情報の取得には md_reg を介したディスク上のストアへのアクセスが発生するため、高頻度なリクエストには注意が必要です。
  • MD_STATUS_RESOURCE 定数（/.httpd/certificate-status）は、外部から証明書状態を確認する際の重要なエンドポイントとなります。
