modules/md/mod_md_status.c

概要

本ファイルは、Apache HTTP Server の mod_status 拡張として、Managed Domains (MD) の状態情報を JSON または HTML 形式で出力する機能を提供します。証明書の有効期限、OCSP ステータス、更新ジョブの進捗状況などを可視化する役割を担います。

モジュール内での役割

mod_md のステータスレポート層です。md_status.c が提供する低レベルな JSON 生成ロジックをラップし、Apache の request_rec コンテキストに合わせて、HTTP レスポンスとして整形・出力するブリッジとして機能します。

責務

  • **証明書ステータス公開**: 特定のドメインに対する証明書情報を JSON で提供するエンドポイント (/.httpd/certificate-status) の実装。
  • **サーバーステータス統合**: mod_status のフックを通じて、管理対象ドメインおよび OCSP ステータスの概要を HTML/テキスト形式で出力。
  • **データ整形**: md_json_t 構造体から、HTML テーブルまたは人間が読みやすいテキスト形式への変換。
  • **ハンドラ実装**: md-status ハンドラによる、詳細な JSON ステータス情報の提供。

重要なシンボル

リクエスト処理・ハンドラ

  • md_http_cert_status: 証明書ステータス用エンドポイントのハンドラ。
  • md_domains_status_hook: mod_status に統合されるドメイン状態表示フック。
  • md_ocsp_status_hook: mod_status に統合される OCSP 状態表示フック。
  • md_status_handler: md-status ハンドラによる JSON 出力の実装。

データ構造・コンテキスト

  • status_ctx: ステータス出力処理中に使用される一時的なコンテキスト（プール、設定、出力用ブリゲード、フォーマットフラグを保持）。
  • status_info: ステータス表示の各カラム定義（ラベル、JSON キー、表示用コールバック関数）。

表示用ヘルパー

  • add_status_cell: status_info に基づき、適切なフォーマット関数をディスパッチする。
  • print_date / print_time: タイムスタンプを人間が読みやすい形式（相対時間や ISO 形式）に変換。
  • add_json_val: JSON 値を再帰的に走査し、テキストまたは HTML として出力。

制御フローとデータフロー

1. **リクエスト受信**: 各ハンドラ/フックが request_rec から設定 (md_srv_conf_t) を取得。 2. **データ取得**: md_status_get_json 等の md_status.c の関数を呼び出し、管理対象ドメインの情報を JSON 形式で取得。 3. **フォーマット**:

  • HTML モードの場合: status_infos 配列をイテレートし、add_status_cell を通じてテーブル行を生成。
  • テキストモードの場合: ctx->prefix を用いて階層的なキー・値ペアを出力。

    • 4. **出力**: apr_bucket_brigade を使用して、生成されたコンテンツを r->output_filters に渡す。

読者向けメモ

  • **メモリ管理**: すべての出力操作は r->pool を使用しており、リクエスト終了時に自動的に解放されます。
  • **HTML/テキストの切り替え**: HTML_STATUS(ctx) マクロにより、mod_status の出力モードを判定しています。
  • **制約**: md_http_cert_status は GET メソッドのみをサポートします。
  • **省略箇所**: 本ファイルは表示ロジックに特化しており、証明書の更新処理や OCSP の検証ロジックそのものは含まれていません。それらは md_acme.c や md_ocsp.c に委譲されています。
