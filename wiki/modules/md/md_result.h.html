modules/md/md_result.h

概要

md_result.h は、mod_md モジュール内での処理結果、エラー状態、および進行状況を追跡・管理するための共通構造体と API を定義するヘッダーファイルです。

モジュール内での役割

本ファイルは、ACME プロトコルや証明書管理プロセスにおける非同期処理の結果を保持する「コンテナ」として機能します。各処理ステップで発生したステータスや詳細情報を一元管理し、ログ出力や JSON へのシリアライズを容易にします。

責務

  • **結果の保持**: 処理の成功・失敗ステータス、エラー詳細、ACME プロトコル特有のサブ問題（subproblems）の保持。
  • **状態変更の通知**: 処理結果が更新された際に呼び出されるコールバック（on_change 等）の管理。
  • **イベント通知**: 処理の進行に伴うイベント（raise や holler）のトリガー機能。
  • **シリアライズ**: 状態の JSON 変換および復元。

重要なシンボル

  • md_result_t: 処理結果を保持するメイン構造体。メモリ管理用の apr_pool_t を内包します。
  • md_result_make(): 新しい結果オブジェクトを生成するコンストラクタ。
  • md_result_problem_set(): エラー発生時に詳細な問題内容とサブ問題をセットする関数。
  • MD_RESULT_LOG_ID: ログ出力時に使用される一意な識別子を生成するマクロ。

制御フローとデータフロー

1. **初期化**: md_result_make でオブジェクトを生成し、処理対象の md_name を設定します。 2. **更新**: md_result_activity_set や md_result_problem_set を通じて、処理の進行状況やエラー内容を更新します。 3. **通知**: 更新メソッド内で on_change コールバックが実行され、外部の監視者（UI やログ出力など）に状態変化を伝えます。 4. **永続化/伝達**: md_result_to_json を使用して状態を保存するか、md_result_log で現在の状態をログに出力します。

直接依存

  • apr_pool_t (APR ライブラリ): メモリ管理に使用。
  • md_json.h: サブ問題の保持やシリアライズに使用。

関連コード

  • modules/md/md_result.c: 本ヘッダーで宣言された関数の実装。
  • modules/md/md_json.h: JSON 操作の基盤。
  • modules/md/md_log.h: ログ出力の基盤。

読者向けメモ

  • 本構造体は apr_pool_t に強く依存しています。md_result_t を使用する際は、必ず有効なプールが生存していることを確認してください。
  • on_change などのコールバックは、状態が更新されるたびに実行されるため、重い処理を記述しないよう注意が必要です。
  • ACME プロトコルのエラーハンドリングにおいて、subproblems フィールドはサーバーから返される詳細なエラー情報を扱うために重要です。
