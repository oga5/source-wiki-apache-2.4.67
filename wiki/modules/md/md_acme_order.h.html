modules/md/md_acme_order.h

概要

ACME プロトコルにおける「Order（注文）」オブジェクトを定義し、そのライフサイクル管理を行うためのヘッダーファイルです。証明書発行プロセスにおける状態遷移や、サーバーとの通信に必要な情報の保持を担います。

モジュール内での役割

modules/md モジュールにおいて、ACME サーバーとの証明書発行フローを管理する中心的な役割を果たします。md_acme_authz.h（認可）と連携し、証明書発行の各ステップを制御します。

責務

  • ACME Order オブジェクトの生成とメモリ管理。
  • Order 状態（pending, ready, valid 等）の定義と管理。
  • JSON 形式へのシリアライズ/デシリアライズによる永続化サポート。
  • サーバー側での Order 状態の更新、監視、および完了までの待機処理。

重要なシンボル

  • md_acme_order_t: Order 情報を保持するメイン構造体。URL、状態、認可 URL リスト、証明書取得用エンドポイント等を管理します。
  • md_acme_order_st: Order の現在の状態を示す列挙型。
  • MD_FN_ORDER: 永続化時に使用されるファイル名定数 (order.json)。
  • md_acme_order_update(): サーバーから最新の Order 情報を取得し、ローカル状態を同期する関数。

制御フローとデータフロー

1. **生成**: md_acme_order_create で初期化。 2. **登録**: md_acme_order_register を通じて ACME サーバーに新規 Order を作成。 3. **更新**: md_acme_order_update を定期的に呼び出し、サーバー上の状態をローカルの md_acme_order_t に反映。 4. **待機**: md_acme_order_await_ready や md_acme_order_await_valid を使用して、証明書発行が完了するまでポーリングを行う。 5. **永続化**: md_acme_order_save / md_acme_order_load により、ディスク上の order.json と同期。

直接依存

本ヘッダーファイル自体は他のヘッダーを直接インクルードしていませんが、構造体定義内で apr_pool_t や apr_array_header_t などの APR（Apache Portable Runtime）型に依存しています。

関連コード

  • modules/md/md_acme_order.c: 実装本体。JSON 変換ロジックや状態遷移の具体的な処理が含まれます。
  • modules/md/md_acme_authz.h: Order に紐づく各ドメインの認可プロセスを管理する関連モジュール。
  • modules/md/md_store.h: order.json の読み書きを行うストレージインターフェース。

読者向けメモ

  • 本ファイルは ACMEv2 プロトコルに強く依存しています。
  • md_acme_order_t の各フィールドは、ACME サーバーから返される JSON レスポンスと密接に対応しています。
  • 状態遷移（pending -> ready -> valid）のロジックを追う際は、md_acme_order_update の実装から読み始めることを推奨します。
