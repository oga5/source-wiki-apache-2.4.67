modules/md/mod_md_ocsp.h

概要

Apache HTTP Server における OCSP Stapling（証明書失効状態の検証結果をサーバーが代理で提供する機能）を管理するためのヘッダーファイルです。mod_md が管理する証明書に対して、OCSP レスポンスの取得と提供を統合します。

モジュール内での役割

mod_md サブシステムの一部として、サーバー設定と SSL/TLS レイヤーの橋渡しを行います。mod_md_ocsp.c と対になり、mod_ssl が必要とする OCSP ステータス情報の供給元として機能します。

責務

  • **ステータスの初期化**: 証明書チェーンを解析し、OCSP 照会に必要な情報を準備する。
  • **ステータスの提供**: mod_ssl からの要求に応じて、キャッシュされた OCSP レスポンスをコールバック経由で渡す。
  • **バックグラウンド監視**: ウォッチドッグプロセスを起動し、OCSP レスポンスの定期的な更新を管理する。

重要なシンボル

  • md_ocsp_prime_status: 証明書 PEM データから OCSP 照会用の初期状態を構築します。
  • md_ocsp_provide_status: 接続時に mod_ssl へ OCSP レスポンスを供給するためのインターフェースです。
  • md_ocsp_start_watching: OCSP レスポンスの更新を自動化するウォッチドッグを起動します。

制御フローとデータフロー

1. **初期化**: md_ocsp_prime_status がサーバー起動時や設定読み込み時に呼ばれ、証明書チェーンを解析して md_ocsp_prime を呼び出します。 2. **監視**: md_ocsp_start_watching により、バックグラウンドで OCSP レスポンスの有効期限を監視し、必要に応じて再取得を行います。 3. **提供**: クライアントからの TLS ハンドシェイク時、mod_ssl が md_ocsp_provide_status を呼び出し、メモリ上の OCSP レスポンスをコピーしてクライアントへ送信します。

直接依存

  • httpd.h, http_ssl.h: Apache HTTP Server のコアおよび SSL 関連の型定義。
  • apr_*.h: Apache Portable Runtime によるメモリ管理および時間管理。
  • mod_watchdog.h: バックグラウンドタスク実行のための基盤。

関連コード

  • modules/md/md_ocsp.c: OCSP レスポンスの取得ロジックやキャッシュ管理の実装。
  • modules/md/mod_md_ocsp.c: 本ヘッダーで宣言された関数の実装および mod_ssl との統合ロジック。
  • modules/md/md_crypt.h: 証明書解析や暗号化関連のユーティリティ。

読者向けメモ

  • 本ファイルは mod_ssl の OCSP コールバック機構を利用しています。
  • md_ocsp_provide_status は、mod_ssl が提供する ap_ssl_ocsp_copy_resp 型のコールバックと密接に連携するため、実装を読む際は mod_ssl の該当インターフェース定義を併せて参照することをお勧めします。
  • OCSP Stapling が有効な場合、mod_md は証明書の更新サイクルに合わせて自動的に OCSP レスポンスを更新します。
