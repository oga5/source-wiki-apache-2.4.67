modules/md/md_curl.c

概要

md_curl.c は、libcurl を使用した HTTP クライアント機能の実装です。md_http モジュールのバックエンドとして機能し、ACME プロトコル等の通信に必要な同期・非同期リクエスト処理を提供します。

モジュール内での役割

modules/md サブシステムにおいて、ネットワーク通信の抽象化レイヤーを担います。md_http.h で定義されたインターフェースを実装し、上位層（ACME ロジック等）が libcurl の詳細を意識せずに HTTP 通信を行えるようにします。

責務

  • **HTTP 実装の提供**: md_http_impl_t 構造体を通じて、初期化、リクエスト実行、クリーンアップの各操作を md_http に提供。
  • **libcurl 変換**: md_http_request_t（内部データ構造）と libcurl のオプション、コールバック、ステータスコード間のマッピング。
  • **リソース管理**: apr_pool_t を利用したメモリ管理と、libcurl ハンドルのライフサイクル管理（再利用または破棄）。
  • **非同期多重化**: curl_multi インターフェースを用いた、複数のリクエストを並行処理するイベントループの制御。

重要なシンボル

内部データ構造

  • md_curl_internals_t: リクエストごとの libcurl ハンドル、ヘッダーリスト、レスポンスバッファを保持するコンテキスト。

コールバック関数

  • req_data_cb: apr_bucket_brigade からデータを読み取り、libcurl へ渡す。
  • resp_data_cb: libcurl からのレスポンスデータを apr_brigade に書き込む。
  • header_cb: HTTP レスポンスヘッダーを解析し、apr_table_t に格納する。
  • curl_debug_log: libcurl の詳細な通信ログを md_log に出力する。

実装インターフェース

  • md_curl_get_impl(): md_http モジュールにこの実装を登録するためのエントリポイント。
  • md_curl_perform(): 単一リクエストの同期実行。
  • md_curl_multi_perform(): 複数のリクエストを curl_multi で並行実行するメインループ。

制御フローとデータフロー

1. **初期化**: md_curl_get_impl が呼ばれ、curl_global_init が実行される。 2. **セットアップ**: internals_setup が md_http_request_t を libcurl の easy_handle に変換。apr_pool_t のライフサイクルに紐づく。 3. **実行**:

  • 単一の場合: curl_easy_perform を呼び出し、完了後にコールバックを起動。
  • 多重の場合: curl_multi_perform を使用。nextreq コールバックで順次リクエストを追加し、curl_multi_wait でイベントを待機。

    • 4. **終了**: fire_status によりレスポンス処理が完了し、md_http_req_destroy でリソースが解放される。

読者向けメモ

  • **メモリ所有権**: md_http_request_t のプールに依存します。internals 構造体はリクエスト終了時に md_curl_req_cleanup で適切に解放される必要があります。
  • **スレッド安全性**: libcurl のハンドルはスレッドセーフではないため、リクエストごとのプール内で管理される設計です。
  • **エッジケース**: resp_limit を超えるレスポンスボディは resp_data_cb で拒否され、エラーとして処理されます。
  • **省略箇所**: curl_multi の詳細なエラーハンドリングや、特定の SSL オプション（CURLSSLOPT_NO_REVOKE 等）のプラットフォーム依存の挙動については、実装の深部を直接参照してください。
