modules/md/md_json.h

概要

md_json.h は、Apache HTTP Server の mod_md モジュールにおいて、JSON データの操作を抽象化・簡素化するためのラッパーライブラリです。内部で jansson ライブラリを利用し、APR（Apache Portable Runtime）のメモリ管理プールと統合されたインターフェースを提供します。

モジュール内での役割

mod_md は ACME プロトコルを介した証明書管理を行うため、JSON 形式のデータ交換が頻繁に発生します。本ファイルは、複雑な JSON 構造の走査や変換を、可変長引数を用いた直感的な API でラップし、他のモジュールコンポーネントが JSON の詳細を意識せずにデータを扱えるようにします。

責務

  • **JSON オブジェクトのライフサイクル管理**: APR プールと連動した JSON オブジェクトの生成・破棄。
  • **パスベースのデータアクセス**: 可変長引数を使用して、ネストされた JSON 構造へ直接アクセス（取得・設定）する機能の提供。
  • **型変換とシリアライズ**: JSON と C の基本型（数値、文字列、時間、配列、辞書）間の相互変換、およびファイルやネットワークバケットへの読み書き。
  • **反復処理の抽象化**: 配列やオブジェクトのキーに対するイテレータの提供。

重要なシンボル

  • md_json_t: JSON オブジェクトを保持する不透明な構造体。APR プールへの参照を保持します。
  • md_json_get* / md_json_set*: 可変長引数でパスを指定し、特定の型の値を取得・設定する一連の関数群。
  • md_json_read* / md_json_write*: ファイルやバケットからの JSON 解析、およびシリアライズを行う関数群。
  • md_json_itera / md_json_iterkey: 配列やオブジェクトを走査するためのコールバックベースのイテレータ。

制御フローとデータフロー

1. **生成**: md_json_create で空の JSON オブジェクトを生成し、APR プールに紐付けます。 2. **アクセス**: md_json_get* 系関数にパス（キーのリスト）を渡すと、内部で jansson の走査が行われ、対象ノードの値が返されます。 3. **操作**: md_json_set* 系関数は、指定されたパスが存在しない場合に自動的に中間ノードを作成する挙動を含みます。 4. **永続化**: md_json_writef 等により、メモリ上の JSON 構造がファイルシステム上のファイルやネットワークストリームへシリアライズされます。

直接依存

  • apr_file_io.h: ファイル操作および APR プール定義のため。
  • jansson.h (内部実装): JSON のコア処理エンジン。

関連コード

  • modules/md/md_json.c: 本ヘッダーで宣言された関数の実装。jansson の複雑なメモリ管理を隠蔽しています。
  • modules/md/md_util.h: JSON 以外の汎用的なユーティリティ関数群。
  • modules/md/md_http.h: HTTP レスポンスから JSON を読み込む際に連携します。

読者向けメモ

  • **可変長引数の利用**: md_json_get* 等の関数は、最後に NULL を置くことでパスを表現します（例: md_json_getb(json, "top", "sub", "key", NULL)）。
  • **メモリ管理**: md_json_t は生成時に渡された APR プールによって管理されます。明示的な md_json_destroy も可能ですが、プールがクリアされると自動的にクリーンアップされます。
  • **エラーハンドリング**: 多くの関数は apr_status_t を返し、成功時は APR_SUCCESS を返します。JSON のパスが不正な場合や型が一致しない場合はエラーが返されます。
