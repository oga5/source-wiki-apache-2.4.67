modules/md/md_core.c

概要

md_core.c は、マネージドドメイン（Managed Domain: md_t）のデータ構造を操作するための中心的なユーティリティ群を提供します。ドメインの照合、JSON へのシリアライズ/デシリアライズ、および CA（認証局）情報の管理を担う、モジュール内の基盤ファイルです。

モジュール内での役割

modules/md サブシステムにおいて、md_t オブジェクトのライフサイクル管理と、ドメイン設定の整合性チェックを行うコアレイヤーです。md_reg.c（登録管理）や md_store.c（永続化）が扱うドメイン情報の「データモデル」としての役割を果たします。

責務

  • **ドメイン照合**: 複数のドメインリスト間での重複チェックや包含関係の判定。
  • **ライフサイクル管理**: md_t オブジェクトの生成、コピー、クローン。
  • **シリアライズ**: md_t 構造体と JSON 形式間の相互変換（公開用 JSON の生成含む）。
  • **CA 管理**: 既知の CA URL と名称の相互解決および検証。

重要なシンボル

ドメイン操作

  • md_contains: 指定ドメインが md_t に含まれるか判定。
  • md_common_name: 2つの md_t 間で共通するドメイン名を抽出。
  • md_domains_overlap: 2つの md_t がドメインを共有しているか判定。
  • md_equal_domains: 2つの md_t のドメインリストが完全に一致するか判定。

ライフサイクル・生成

  • md_create_empty: apr_pool_t を使用して初期化された空の md_t を作成。
  • md_copy / md_clone: md_t のディープコピー。clone は設定値の複製を伴う完全な複製。

JSON 変換

  • md_to_json / md_from_json: md_t と md_json_t 間の変換。
  • md_to_public_json: EAB（External Account Binding）の HMAC などの機密情報をマスクした JSON を生成。

CA 解決

  • md_get_ca_name_from_url / md_get_ca_url_from_name: CA 名と URL のマッピング。KNOWN_CAs 配列に基づき解決。

制御フローとデータフロー

  • **ドメイン判定**: md_contains が md_array_str_index を呼び出し、一致しない場合は md_dns_domains_match でワイルドカード等の DNS マッチングを試行します。
  • **JSON 変換**: md_to_json は md_json_t を作成し、md_t の各フィールドを MD_KEY_* 定数を用いて JSON キーにマッピングします。md_from_json はその逆を行い、apr_pool_t 上で文字列を複製（md_json_dups）します。
  • **CA 解決**: md_get_ca_url_from_name は、まず既知の CA 名を検索し、見つからない場合は入力が絶対 URI であるかを md_util_abs_uri_check で検証します。

直接依存

  • apr_ 系ライブラリ: メモリ管理（pool）、文字列操作、配列操作、URI 解析。
  • md_json.h: JSON 構造体の操作。
  • md_util.h: ユーティリティ関数（URI チェック等）。
  • md.h: md_t 構造体の定義。
  • md_crypt.h: 暗号化関連の仕様（pks 等）。

関連コード

  • md_reg.c: md_t の登録と検索の管理。
  • md_store.c: md_t のディスクへの保存と読み込み。
  • md_json.c: JSON 操作の低レイヤー実装。

読者向けメモ

  • **メモリ管理**: すべての生成・クローン関数は apr_pool_t を引数に取り、メモリの生存期間はプールに依存します。
  • **不変条件**: md_t の domains 配列は、md_array_str_compact によって正規化されていることが期待されます。
  • **エッジケース**: md_from_json では、古いフォーマット（単一の CA URL）との互換性を維持するためのロジックが含まれています。
  • **省略箇所**: 本ファイルはデータ構造の操作に特化しており、実際の ACME プロトコル通信や証明書の署名処理は含まれていません。これらは md_acme.c や md_crypt.c に委譲されています。
