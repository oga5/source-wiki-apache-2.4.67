modules/md/md_status.c

概要

本ファイルは、Managed Domain (MD) のステータス情報の集約と、証明書更新ジョブ（md_job_t）の永続化・ライフサイクル管理を担当します。JSON 形式でのステータス出力機能と、ディスク上のジョブ状態を同期させるためのロジックを提供します。

モジュール内での役割

modules/md サブシステムにおいて、管理者が現在の証明書状態や更新ジョブの進捗を把握するための「レポート生成器」および「ジョブコントローラー」として機能します。md_reg.c（レジストリ）や md_store.c（ストレージ）と密接に連携し、高レベルな状態を JSON に変換します。

責務

  • **ステータスレポート生成**: MD および証明書の状態、OCSP ステータス、更新期限などを JSON 構造に集約する。
  • **ジョブ永続化**: md_job_t 構造体の JSON シリアライズ/デシリアライズ、およびストレージへの保存・読み込み。
  • **ジョブ実行管理**: ジョブの開始・終了時のログ記録、エラー発生時のバックオフ（再試行間隔）計算、イベント通知の管理。
  • **オブザーバビリティ**: md_result_t の変更を監視し、進捗状況をログに反映させるフック機能。

重要なシンボル

ステータス生成（JSON）

  • md_status_get_md_json: 単一 MD の詳細なステータス JSON を生成。
  • md_status_get_json: 全 MD のステータスを一覧化。
  • md_status_take_stock: 全 MD の状態（完了、更新中、エラー等）をカウントし、概要を生成。

ジョブ管理（md_job_t）

  • md_job_make: ジョブインスタンスの初期化。
  • md_job_load / md_job_save: ストレージとの間でのジョブ状態の同期。
  • md_job_start_run / md_job_end_run: ジョブ実行のライフサイクル管理。
  • md_job_log_append: ジョブ実行履歴の追記（最大件数制限あり）。

内部ヘルパー

  • md_job_delay_on_errors: エラー時の指数バックオフとジッター計算。
  • job_observation_start: md_result_t の変更を監視するコールバックの設定。

制御フローとデータフロー

1. **ステータス取得**: md_status_get_md_json が呼び出されると、md_reg から証明書情報を取得し、status_get_cert_json_ex を通じて各証明書の有効期限や OCSP 状態を JSON にマッピングします。 2. **ジョブ実行**: md_job_start_run で md_result_t にコールバック（job_result_update 等）を登録します。結果が更新されるたびに、このコールバックがトリガーされ、ログへの追記とストレージへの保存が行われます。 3. **エラー処理**: md_job_end_run でエラーが検出されると、md_job_delay_on_errors が呼び出され、次の実行時刻（next_run）が計算されます。

読者向けメモ

  • **メモリ管理**: すべての JSON オブジェクトとジョブ構造体は、渡された apr_pool_t に依存します。
  • **永続化**: ジョブのログは max_log（デフォルト128）で制限されており、古いログは自動的に破棄されます。
  • **エラー処理**: md_job_delay_on_errors は、ACME サーバーからの入力関連エラー（設定ミス等）を検知すると、即座に最大遅延（24時間）へ移行する安全策を備えています。
  • **未実装/浅い部分**: MD_STATUS_WITH_SCTS マクロは現在 0 に設定されており、証明書の SCT（Signed Certificate Timestamps）情報は出力されません。これは将来的な拡張用です。
