modules/md/md_http.h

概要

md_http.h は、mod_md モジュール内で HTTP 通信を行うための抽象化レイヤーを定義するヘッダーファイルです。特定の HTTP クライアントライブラリ（例: libcurl）に依存せず、プラグイン可能な実装（md_http_impl_t）を通じて通信機能を提供します。

モジュール内での役割

modules/md サブシステムにおけるネットワーク通信のフロントエンドです。ACME プロトコルなどの外部サービスとの通信を抽象化し、上位レイヤーが通信の詳細を意識せずに HTTP リクエストを発行できるようにします。

責務

  • **HTTP セッション管理**: md_http_t を通じた設定（タイムアウト、プロキシ、CA 証明書など）の保持。
  • **リクエストの構築**: GET/HEAD/POST リクエストの生成と、ボディ（apr_bucket_brigade）の管理。
  • **通信の抽象化**: 実装（md_http_impl_t）を切り替えるためのインターフェース提供。
  • **非同期/並列処理**: md_http_multi_perform による複数リクエストの並列実行サポート。

重要なシンボル

セッションとリクエスト

  • md_http_t: HTTP クライアントのインスタンス。設定と実装へのポインタを保持。
  • md_http_request_t: 個別の HTTP リクエスト。メソッド、URL、ヘッダー、ボディ、タイムアウト設定を含む。
  • md_http_response_t: 受信したレスポンス。ステータスコード、ヘッダー、ボディを保持。

実装インターフェース

  • md_http_impl_t: 通信バックエンドの関数テーブル。init, perform, multi_perform 等を定義。
  • md_http_use_implementation(): 使用するバックエンド実装を登録する関数。

設定・コールバック

  • md_http_timeouts_t: 接続、全体、ストール検出のタイムアウト設定。
  • md_http_callbacks_t: リクエスト完了時やステータス判明時に呼び出されるコールバック関数群。

制御フローとデータフロー

1. **初期化**: md_http_use_implementation でバックエンドを登録し、md_http_create でセッションを作成。 2. **リクエスト作成**: md_http_GET_create 等でリクエストオブジェクトを生成し、必要に応じてコールバックやタイムアウトを設定。 3. **実行**: md_http_perform を呼び出すと、内部で登録された md_http_impl_t の perform が実行される。 4. **終了**: リクエスト完了後、コールバックが実行され、リクエストに関連するメモリは自動的に解放される。

直接依存

  • apr_lib.h, apr_strings.h, apr_buckets.h: Apache Portable Runtime (APR) を使用してメモリ管理とデータストリームを処理。

関連コード

  • modules/md/md_http.c: 本ヘッダーのインターフェース実装。
  • modules/md/md_curl.c: md_http の具体的な実装例（libcurl を使用）。

読者向けメモ

  • **メモリ管理**: md_http_request_t は md_http_perform の終了時に自動的に破棄されます。呼び出し側で手動解放する必要はありません。
  • **実装の切り替え**: 本モジュールは cur_impl 静的変数を通じて実装を保持します。md_http_create 呼び出し前に必ず実装が登録されている必要があります。
  • **ストール検出**: md_http_set_stalling を使用することで、低速な通信をタイムアウトさせる安全機構が提供されています。
  • **省略箇所**: md_http_request_t 内部の internals ポインタは実装固有のデータ用であり、本ヘッダーでは詳細を定義していません。
