modules/md/md_acme_acct.c

概要

ACME プロトコルにおけるアカウントの管理、永続化、およびサーバーとの同期を担当するモジュールです。ACME アカウントの作成、更新、利用規約（ToS）への同意、およびアカウントの無効化を制御します。

モジュール内での役割

modules/md サブシステムにおいて、ACME サーバーとクライアント（Apache HTTP Server）を結びつける認証情報のライフサイクルを管理します。md_store を介してディスク上のアカウント情報（JSON）と秘密鍵（PEM）を管理し、md_acme の上位ロジックにアカウント状態を提供します。

責務

  • **アカウントの永続化**: アカウント情報（JSON）と秘密鍵（PEM）の読み書き。
  • **サーバー同期**: ACME サーバーに対するアカウント情報の更新および新規登録。
  • **検索と照合**: 特定のドメイン（MD）や URL に適合するアカウントの検索と検証。
  • **EAB（External Account Binding）**: 外部アカウントバインディングの署名と検証。
  • **利用規約管理**: サーバーが要求する ToS への同意状態の追跡と更新。

重要なシンボル

アカウント管理

  • md_acme_acct_t: アカウント情報を保持する主要構造体。
  • md_acme_acct_save / md_acme_acct_load: md_store を用いたアカウントの永続化操作。
  • md_acme_acct_register: 新規アカウントの作成とサーバーへの登録。
  • md_acme_acct_update: サーバーから最新のアカウント情報を取得し同期。

検索・照合

  • md_acme_find_acct_for_md: 指定されたドメインに最適なアカウントを検索。
  • md_acme_acct_matches_md: アカウントが特定のドメインの要件（CA URL、EAB 設定）を満たすか判定。

ユーティリティ

  • md_acme_acct_to_json / md_acme_acct_from_json: アカウント構造体と JSON 間のシリアライズ。
  • md_acme_agree: サーバーに対して ToS 同意を通知。

制御フローとデータフロー

1. **検索**: md_acme_find_acct_for_md が md_store_iter を呼び出し、find_acct コールバックで候補を走査。 2. **検証**: acct_find_and_verify が見つかったアカウントに対して md_acme_acct_validate を実行し、サーバー側での有効性を確認。 3. **更新**: md_acme_POST を経由して ACME サーバーと通信。on_init_acct_upd でリクエストを初期化し、acct_upd でレスポンスを解析して md_acme_acct_t を更新。 4. **EAB**: get_eab が JWS 署名を用いて外部アカウントバインディングを計算し、登録リクエストに付与。

読者向けメモ

  • **メモリ管理**: すべての操作は apr_pool_t に依存しており、ライフサイクルは呼び出し元のプールに紐付きます。
  • **エラー処理**: APR_EAGAIN はアカウントの再検証や再試行が必要な場合に頻繁に使用されます。
  • **永続化の不変条件**: アカウント ID は ACME-<servername>-<index> の形式で生成され、account.json と account.pem がペアで管理されます。
  • **制限事項**: 本ファイルは ACME プロトコルのアカウント管理に特化しており、証明書発行の注文（Order）ロジックは md_acme_order.c に分離されています。
