modules/md/mod_md_os.c

概要

mod_md_os.c は、Apache HTTP Server の mod_md モジュールにおいて、OS 固有の低レベルな操作を抽象化するユーティリティファイルです。主にファイルシステムの権限管理と、サーバープロセスの制御（シグナル送信）を担当します。

モジュール内での役割

mod_md は証明書の自動管理を行うため、証明書ファイルやディレクトリを生成・更新します。本ファイルは、それらのリソースが Apache のワーカプロセスから適切にアクセス可能であることを保証し、必要に応じてサーバーの再起動をトリガーする役割を担います。

責務

  • **権限管理**: 証明書ディレクトリやファイルの所有者（UID/GID）を、Apache の実行ユーザーに合わせて調整する。
  • **プロセス制御**: 証明書の更新完了時などに、親プロセスへシグナルを送り、サーバーのグレースフル再起動を要求する。
  • **プラットフォーム抽象化**: OS ごとの差異（Unix系 vs Windows）を吸収し、サポートされない環境では APR_ENOTIMPL を返す。

重要なシンボル

  • md_try_chown: 指定されたパスの所有者を変更するラッパー。ルート権限で実行されている場合のみ動作します。
  • md_make_worker_accessible: ap_unixd_config.user_id を使用して、ファイルやディレクトリをワーカプロセスから読み書き可能にします。
  • md_server_graceful: 親プロセスに対して AP_SIG_GRACEFUL を送信し、設定の再読み込みを促します。

制御フローとデータフロー

1. **権限変更**: md_make_worker_accessible が呼び出されると、内部で md_try_chown が実行されます。これは ap_unixd_config から現在の実行ユーザー情報を取得し、chown システムコールをラップします。 2. **シグナル送信**: md_server_graceful は getppid() で親プロセスを特定し、kill() を用いてシグナルを送信します。成功時は APR_SUCCESS を返し、失敗時はエラーをログ出力します。 3. **条件分岐**: コンパイル時のマクロ（AP_NEED_SET_MUTEX_PERMS や WIN32）により、OS がサポートしていない機能は即座に APR_ENOTIMPL を返すよう設計されています。

関連コード

  • mod_md_config.c: 実行ユーザー設定（User/Group ディレクティブ）の管理。
  • md_store.c: 証明書や鍵の永続化を担当しており、本ファイルの権限調整機能を頻繁に利用します。

読者向けメモ

  • 本ファイルは「特権が必要な操作」を扱うため、実行環境の権限設定に依存します。
  • md_try_chown は geteuid() が 0（root）でない場合、何もしない（成功を返す）設計になっています。これは、非特権ユーザーで実行されている場合に権限エラーで停止しないための安全策です。
  • Windows 環境では多くの機能が APR_ENOTIMPL となるため、クロスプラットフォームでの動作確認時には注意が必要です。
