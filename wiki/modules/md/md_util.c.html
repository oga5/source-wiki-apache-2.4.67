modules/md/md_util.c

概要

md_util.c は、mod_md モジュール全体で使用される汎用的なユーティリティ関数群を提供します。メモリ管理、文字列操作、ファイルシステム操作、DNS 名の検証、および HTTP Link ヘッダーの解析など、特定のドメインロジックに依存しない低レベルな補助機能を集約しています。

モジュール内での役割

modules/md サブシステムにおける「共通ライブラリ」として機能します。他の多くのファイル（md_reg.c, md_store.c など）が、APR（Apache Portable Runtime）の機能をラップしたこれらのユーティリティを呼び出し、コードの重複を排除し、一貫したエラー処理を実現しています。

責務

  • **メモリ管理**: md_data_t 構造体を用いたデータチャンクのライフサイクル管理とコピー操作。
  • **ファイルシステム**: ファイルの作成、置換、再帰的な削除、ディレクトリ走査などの抽象化。
  • **文字列・配列操作**: 大文字小文字を区別しない比較や、配列内の要素操作。
  • **ネットワーク・プロトコル**: DNS 名の検証、URI の妥当性チェック、Base64URL エンコード/デコード。
  • **HTTP ヘッダー解析**: Link ヘッダーの特定の関係性（rel）を持つ URL の抽出。

重要なシンボル

データチャンク管理 (md_data_t)

  • md_data_pinit, md_data_pmake: プールベースのメモリ割り当てと初期化。
  • md_data_clear: 保持しているデータの解放と構造体のクリア。
  • md_data_assign_copy, md_data_assign_pcopy: データの代入とコピー。

ファイルシステム操作

  • md_util_fcreatex: 排他的なファイル作成（O_EXCL 相当）。
  • md_util_freplace: 一時ファイル経由での安全なファイル置換。
  • md_util_rm_recursive, md_util_ftree_remove: ディレクトリツリーの再帰的削除。
  • md_util_files_do, md_util_tree_do: ファイルシステム走査とコールバックのディスパッチ。

DNS・URI 検証

  • md_dns_is_name, md_dns_is_wildcard: DNS 名の構文チェック。
  • md_util_abs_http_uri_check: URI のスキームやホスト名の妥当性検証。

HTTP Link ヘッダー

  • md_link_find_relation: Link ヘッダーから特定の rel 属性を持つ URL を検索。

制御フローとデータフロー

  • **プールベースの実行**: md_util_pool_do や md_util_pool_vdo は、一時的な apr_pool_t を作成し、コールバック関数を実行した後にプールを破棄するパターンを提供します。これにより、再帰的なファイル操作や複雑な処理中のメモリリークを防ぎます。
  • **ファイル操作**: md_util_freplace は、fpath.tmp を作成し、成功時に rename するというアトミックな更新フローを実装しています。
  • **再帰的走査**: match_and_do や tree_do は、apr_dir_read を用いてディレクトリを走査し、マッチした要素に対してコールバックを再帰的に呼び出します。

読者向けメモ

  • **メモリ所有権**: md_data_t は、free_data 関数ポインタを持つことで、動的に確保されたメモリの解放を管理します。md_data_clear を呼び出すことで、所有しているメモリが適切に解放される設計になっています。
  • **エラー処理**: 多くの関数は apr_status_t を返します。APR_SUCCESS 以外が返された場合、呼び出し元で適切にログ出力やリソース解放を行う必要があります。
  • **省略された詳細**: 本ファイルには、Base64URL 変換用の巨大なルックアップテーブルが含まれていますが、これらは RFC 準拠のための静的データであり、ロジック上の複雑さはありません。
  • **パフォーマンス**: md_util_freplace は、ファイル作成時に EEXIST が発生した場合、指数バックオフを伴うリトライ処理を行っており、競合の多い環境での堅牢性を確保しています。
