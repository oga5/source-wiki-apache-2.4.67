modules/md/mod_md_config.c

概要

Apache HTTP Server の mod_md における設定管理を担当するソースファイルです。Apache の設定ディレクティブを解析し、サーバー単位（md_srv_conf_t）およびモジュール全体（md_mod_conf_t）の設定構造体へ反映させる役割を持ちます。

モジュール内での役割

mod_md の設定層として、httpd.h の設定フレームワークと mod_md のコアロジックを橋渡しします。設定の継承、マージ、およびバリデーションを行い、他の md_* コンポーネントが利用する設定状態を構築します。

責務

  • **設定の定義**: md_cmds 配列による Apache 設定ディレクティブの登録。
  • **設定のライフサイクル管理**: 設定構造体の作成、マージ、およびクリーンアップ。
  • **バリデーション**: 設定値の妥当性チェック（コンテキスト制限、値の範囲、ファイル存在確認）。
  • **アクセスインターフェース**: md_config_get 等を通じた、実行時における設定値の取得。

重要なシンボル

設定構造体とライフサイクル

  • md_mod_conf_t: モジュール全体で共有されるグローバル設定（ストレージパス、プロキシ設定、OCSP設定など）。
  • md_srv_conf_t: 各バーチャルホスト単位の設定。md_config_create_svr で初期化され、md_config_merge で継承関係が解決されます。
  • md_config_get() / md_config_get_unique(): サーバーレコードから設定を取得する主要なアクセサ。

設定解析とディレクティブ

  • md_cmds: Apache に登録される全ディレクティブの定義テーブル。
  • md_config_sec_start: <MDomain> / <MDomainSet> セクションの開始処理。
  • md_config_set_*: 各種設定ディレクティブ（MDRenewMode, MDContactEmail 等）のハンドラ関数群。

ユーティリティ

  • md_conf_check_location: 設定が許可されたコンテキスト（グローバル、セクション内など）にあるかを確認。
  • srv_conf_props_apply: セクション内の設定を md_t オブジェクトへ適用する。

制御フローとデータフロー

1. **初期化**: Apache 起動時に md_config_create_svr が呼ばれ、デフォルト値（defmc, defconf）で初期化されます。 2. **解析**: 設定ファイル読み込み時、md_cmds に定義されたハンドラが順次実行されます。 3. **マージ**: md_config_merge が、ベースサーバーの設定とバーチャルホストの設定をマージし、継承階層を構築します。 4. **適用**: md_config_post_config で、最終的なパスの解決やヘッダーの事前計算が行われます。 5. **アクセス**: リクエスト処理中、md_config_gets や md_config_geti を通じて、継承された最終的な設定値が取得されます。

読者向けメモ

  • **メモリ管理**: 設定構造体は Apache の pool に紐付いており、リクエスト終了時や設定再読み込み時に自動的に解放されます。
  • **継承の仕組み**: md_config_merge は、子設定が未設定（DEF_VAL）の場合に親設定を採用する階層的なマージを行います。
  • **制約**: inside_section などの関数により、特定のディレクティブが <MDomain> セクション内でのみ有効であるといった制約が厳格に管理されています。
  • **注意点**: md_config_gets 等のアクセサは、設定が存在しない場合にデフォルト値（defconf）を返す設計になっています。
