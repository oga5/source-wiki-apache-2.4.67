modules/md/md_ocsp.c

概要

本ファイルは、マネージド証明書（Managed Domains）における OCSP ステープリングのためのクライアント機能を提供します。証明書の失効状態を定期的に取得・更新し、ローカルストレージにキャッシュして、TLS ハンドシェイク時に stapling 用の DER データを供給する役割を担います。

モジュール内での役割

modules/md サブシステムの一部として、証明書のライフサイクル管理（md_reg.c）と連携し、証明書の有効性検証を自動化します。mod_md_ocsp.c が Apache HTTP Server のフックと本モジュールを接続するブリッジとなります。

責務

  • **状態管理**: 証明書ごとの OCSP ステータス（md_ocsp_status_t）の保持と、メモリ上のハッシュテーブルによる管理。
  • **通信と更新**: OCSP レスポンダーへの HTTP POST リクエストの実行、レスポンスの解析（OpenSSL OCSP_RESPONSE）、および検証。
  • **永続化**: md_store を介した OCSP レスポンスの JSON 形式での保存と読み込み。
  • **スケジューリング**: md_ocsp_renew による、期限切れ間近のステータスの更新処理。

重要なシンボル

レジストリ管理

  • md_ocsp_reg_t: OCSP 処理のグローバルコンテキスト。ストレージ、スレッドミューテックス、証明書 ID マップを保持。
  • md_ocsp_reg_make(): レジストリの初期化とミューテックスの作成。

ステータス管理

  • md_ocsp_status_t: 個別の証明書に対する OCSP 状態。DER データ、有効期限、レスポンダー URL を含む。
  • md_ocsp_prime(): 証明書を OCSP 管理対象として登録し、初期状態をロードする。
  • md_ocsp_get_status(): ステープリング用の DER データを取得するコールバック関数。

更新・検証

  • ocsp_status_refresh(): ストレージから最新の OCSP レスポンスを読み込み、メモリ状態を更新する。
  • ostat_on_resp(): HTTP レスポンスのパースと検証（Nonce チェック、署名検証、ステータス確認）。

制御フローとデータフロー

1. **初期化**: md_ocsp_prime で証明書と issuer から OCSP_CERTID を生成し、管理対象に追加。 2. **定期更新**: md_ocsp_renew が呼び出されると、期限が近いステータスを抽出し、md_http を用いて非同期的にリクエストを送信。 3. **レスポンス処理**: ostat_on_resp が HTTP レスポンスを受け取り、d2i_OCSP_RESPONSE でデコード。検証後、ostat_set でメモリを更新し、ocsp_status_save でストレージへ書き出す。 4. **データ供給**: md_ocsp_get_status が呼ばれると、メモリ上の最新データをコールバック経由で提供。必要に応じて ocsp_status_refresh で最新化を試みる。

直接依存

  • **APR**: apr_hash_t, apr_thread_mutex_t, apr_pool_t 等によるメモリ・スレッド管理。

  • **md_store**: OCSP レスポンスの永続化（JSON）。
  • **md_http**: OCSP レスポンダーへの通信。

読者向けメモ

  • **メモリ管理**: md_ocsp_reg_t は apr_pool_t に紐付いており、プール破棄時に ocsp_reg_cleanup が呼ばれ、ハッシュテーブル内の全ステータスが解放されます。
  • **スレッド安全性**: reg->mutex により、複数スレッドからのステータス読み書きが保護されています。
  • **エッジケース**: OCSP レスポンダーが Nonce を返さない場合や、レスポンスがキャッシュされている場合でも、有効期限（nextUpdate）に基づいて更新を試みる設計になっています。
  • **省略箇所**: 本ファイル内の md_openssl_free などの単純なラッパーや、JSON 変換の細部は、OpenSSL/APR の標準的な利用パターンであるため詳細な解説を省略しています。
