modules/md/mod_md.c

概要

mod_md.c は Apache HTTP Server の Managed Domains (MD) モジュールのメインエントリーポイントです。サーバー設定の解析、証明書の自動管理（ACME プロトコル経由）、および SSL/TLS 関連のフックを統合し、HTTPD のライフサイクル全体で MD の状態を維持・監視します。

モジュール内での役割

mod_md サブシステムの「司令塔」として機能します。設定（mod_md_config.c）とレジストリ（md_reg.c）を橋渡しし、Apache の post_config フックから child_init、リクエスト処理に至るまで、MD のライフサイクルを管理します。

責務

  • **設定の統合**: サーバー設定を解析し、ドメインと MD のマッピングを構築する。
  • **ライフサイクル管理**: post_config での初期化、証明書ストアのセットアップ、監視プロセスの起動。
  • **SSL/TLS フック**: mod_ssl と連携し、証明書の提供、チャレンジ応答（HTTP-01/TLS-ALPN-01）、HTTPS 強制リダイレクトを処理する。
  • **通知とログ**: 証明書の更新やエラー発生時の外部コマンド実行（MDNotifyCmd 等）とログ出力。

重要なシンボル

モジュール定義

  • md_module: Apache モジュール構造体。コマンドハンドラやフックを登録。

設定・初期化

  • md_post_config_before_ssl: SSL 初期化前の MD レジストリとストアの準備。
  • md_post_config_after_ssl: SSL 初期化後の MD 監視開始と同期処理。
  • merge_srv_config: サーバー設定と MD 設定の統合。

SSL/TLS フック

  • md_add_cert_files: mod_ssl に証明書ファイルパスを注入する。
  • md_answer_challenge: TLS-ALPN-01 チャレンジに対する証明書応答。
  • md_http_challenge_pr: HTTP-01 チャレンジ（/.well-known/acme-challenge/）の応答。
  • md_require_https_maybe: HTTPS 強制リダイレクトと HSTS ヘッダーの付与。

通知・イベント

  • notify: 証明書更新等のイベント発生時に外部コマンドを実行する。
  • on_event: md_event サブシステムからのコールバック。

制御フローとデータフロー

1. **初期化**: post_config フックが before_ssl と after_ssl の2段階で実行されます。before_ssl でストアとレジストリを構築し、after_ssl で証明書の監視（Watchdog）を開始します。 2. **証明書提供**: mod_ssl が証明書を要求すると md_add_cert_files が呼ばれ、レジストリから有効な証明書パスを取得して返します。 3. **チャレンジ応答**: リクエストが /.well-known/ に到達すると md_http_challenge_pr がストアから応答データを読み取り、クライアントに返します。 4. **同期**: merge_mds_with_conf がサーバー設定を走査し、ドメインと MD をリンクさせ、設定の不整合を検証します。

読者向けメモ

  • **メモリ管理**: Apache の apr_pool_t を使用。post_config の dry_run フラグにより、設定チェック時と実際の起動時で処理を分岐させています。
  • **排他制御**: md_reg_lock_global を使用して、マルチプロセス環境でのレジストリ同期の整合性を確保しています。
  • **制約**: mod_ssl が必須です。AP_MODULE_MAGIC_AT_LEAST により Apache 2.4.48 以降を要求します。
  • **省略箇所**: mod_md_status.c 関連のステータスハンドラや、詳細な ACME プロトコル内部の通信ロジックは、このファイルでは呼び出し元としてのみ扱われています。
