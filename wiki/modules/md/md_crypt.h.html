modules/md/md_crypt.h

概要

md_crypt.h は、mod_md モジュールにおける暗号化操作の抽象化レイヤーです。OpenSSL をバックエンドとして利用し、秘密鍵の生成・管理、X.509 証明書の処理、および ACME プロトコルに必要な署名やダイジェスト計算機能を提供します。

モジュール内での役割

modules/md サブシステムにおいて、低レイヤーの OpenSSL API をラップし、Apache HTTP Server の apr_pool_t ベースのメモリ管理と統合する役割を担います。他のモジュール（md_acme.c や md_reg.c など）は、OpenSSL の複雑な構造体を直接操作せず、本ヘッダーで定義された md_pkey_t や md_cert_t を通じて暗号化機能を利用します。

責務

  • **乱数生成**: セキュアな乱数生成器の初期化と提供。
  • **秘密鍵管理**: RSA/EC 鍵の生成、JSON 形式との相互変換、ファイル入出力。
  • **証明書管理**: X.509 証明書の読み込み、検証、属性抽出、および自己署名証明書の作成。
  • **暗号学的ハッシュ・署名**: SHA256 ダイジェスト計算、HMAC 生成、JWS 等で使用する署名処理。
  • **ACME 特化機能**: TLS-ALPN-01 チャレンジ用証明書の生成や、証明書透明性（CT）情報の抽出。

重要なシンボル

秘密鍵管理

  • md_pkey_t: 秘密鍵を保持する不透明な構造体。apr_pool_t に紐付き、自動解放される。
  • md_pkey_spec_t: 鍵の生成要件（RSA ビット数や EC 曲線名）を定義する仕様構造体。
  • md_pkey_gen(): 仕様に基づき新しい秘密鍵を生成する。
  • md_pkey_fload() / md_pkey_fsave(): 秘密鍵のファイル入出力（パスフレーズ対応）。

証明書管理

  • md_cert_t: X.509 証明書をラップする構造体。
  • md_cert_make() / md_cert_wrap(): メモリ所有権の管理戦略（make は所有権取得、wrap は参照のみ）。
  • md_cert_self_sign(): ACME 認証等で使用する自己署名証明書の作成。
  • md_cert_state_get(): 証明書の有効期限状態（VALID/EXPIRED）を判定。

ユーティリティ

  • md_crypt_init(): OpenSSL ライブラリの初期化と乱数シードの生成。
  • md_crypt_sha256_digest64(): データに対する SHA256 ダイジェストの Base64URL エンコード。

制御フローとデータフロー

1. **初期化**: モジュール起動時に md_crypt_init() が呼ばれ、OpenSSL のアルゴリズム登録と乱数生成器のシード設定が行われます。 2. **鍵生成**: md_pkeys_spec_t で要件を定義し、md_pkey_gen() を呼び出すことで md_pkey_t が生成されます。 3. **証明書操作**: md_cert_read_http() 等で外部から取得した証明書を md_cert_t に変換し、md_cert_covers_domain() 等でドメイン整合性を検証します。 4. **メモリ管理**: すべての md_pkey_t および md_cert_t は apr_pool_t に紐付いており、プール破棄時に OpenSSL の内部オブジェクトも適切に解放されます。

直接依存

  • apr_file_io.h: ファイル入出力操作。
  • md.h: モジュール共通の型定義。
  • openssl/evp.h, openssl/x509v3.h: OpenSSL のコア機能。

関連コード

  • md_crypt.c: 本ヘッダーで宣言された関数の実装。乱数シード生成のロジックが含まれる。
  • md_jws.c: 本ヘッダーの署名機能を利用して ACME プロトコルの JWS ペイロードを構築する。
  • md_store.c: 証明書や秘密鍵の永続化ストレージ管理。

読者向けメモ

  • **メモリ所有権**: md_cert_make() は apr_pool_t による自動解放を前提としますが、md_cert_wrap() は所有権を移転しません。呼び出し側が OpenSSL オブジェクトの寿命を管理する必要があります。
  • **OpenSSL バージョン**: OPENSSL_VERSION_NUMBER に基づき、CT（Certificate Transparency）機能の有無や API の差異を吸収するマクロが実装されています。
  • **スレッド安全性**: md_crypt_init() は静的フラグ initialized を使用して一度だけ実行されるよう制御されています。
  • **省略された詳細**: 本ヘッダーには OpenSSL の詳細な内部構造体定義は含まれていません。これは mod_md が OpenSSL の実装詳細から分離されることを意図しているためです。
