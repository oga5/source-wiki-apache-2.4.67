os/netware

システム内での役割

os/netware は、Apache HTTP Server を NetWare プラットフォーム上で動作させるための OS 抽象化レイヤーおよびビルド構成を提供します。このモジュールはディレクトリ境界に基づいて定義されており、NetWare 特有の環境差異を吸収し、コア機能がプラットフォームを意識せずに動作できるよう支援します。

責務

  • **プラットフォーム適応**: NetWare 環境特有のコンパイラ挙動、型定義、およびシステムコールのマッピング。
  • **ビルド構成**: NetWare 向けビルドに必要なディレクトリレイアウトやプリプロセッサ定義の提供。
  • **ライフサイクル管理**: NLM (NetWare Loadable Module) としての起動・終了処理および OS イベント通知のハンドリング。
  • **静的モジュール管理**: NetWare 環境でサーバーバイナリに組み込むべき静的モジュールリストの定義。

このモジュールのファイル

  • os/netware/pre_nw.h: コンパイル初期化、プラットフォーム識別、型定義の調整。
  • os/netware/os.h: OS 固有の機能抽象化、マクロ定義（getpid 等）。
  • os/netware/netware_config_layout.h: ビルド時のディレクトリレイアウト定義。
  • os/netware/modules.c: 静的リンクされるモジュールの登録管理。
  • os/netware/util_nw.c: NLM ライフサイクル管理およびプロセス制御ユーティリティ。

主要なデータ構造

  • ap_prelinked_modules: サーバー起動時にロードされる静的モジュールのポインタ配列。
  • ap_prelinked_module_symbols: モジュール名と構造体を対応付ける検索用配列。
  • ap_module_symbol_t: モジュール名と構造体アドレスを保持する型。

主なフロー

1. **ビルド時**: pre_nw.h が環境を整え、netware_config_layout.h がパス構成を確定させる。 2. **起動時**: _NonAppStart が呼び出され、イベント通知の登録や終了ハンドラのセットアップを行う。 3. **実行中**: http_config サブシステムが modules.c で定義された配列を走査し、各モジュールのフックを初期化する。 4. **終了時**: EVENT_DOWN_SERVER イベントを捕捉し、ap_cb_destroy を通じてリソースの解放とクリーンアップを実行する。

外部依存

  • httpd.h, http_config.h, http_log.h, ap_mpm.h (Apache コア)
  • nks/netware.h, nks/vm.h (NetWare Kernel Services)

内部構成

  • 子モジュール数: 0

推奨読書順

1. os/netware/pre_nw.h (環境設定の基礎) 2. os/netware/os.h (OS 抽象化レイヤー) 3. os/netware/netware_config_layout.h (ビルド構成) 4. os/netware/modules.c (静的モジュール構成) 5. os/netware/util_nw.c (ライフサイクル管理)
