server/mpm/worker

システム内での役割

Apache HTTP Server におけるハイブリッド型マルチプロセッシングモジュール（MPM）の実装です。各子プロセスが複数のスレッドを生成し、各スレッドが単一の接続を処理することで、プロセスベースの MPM よりも効率的なリソース利用と高い同時接続処理能力を実現します。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • **プロセス管理**: fork による子プロセスの生成と、スコアボードを用いた死活監視。
  • **スレッド管理**: リスナー用スレッドと、リクエスト処理用ワーカープールスレッドの生成・管理。
  • **接続の受け渡し**: リスナーが受け取ったソケットを fd_queue を介してワーカーへ引き渡す。
  • **ライフサイクル制御**: 設定の検証、Graceful/Ungraceful な再起動・終了処理、リソースのクリーンアップ。
  • **デフォルト値の提供**: 設定ファイルで指定がない場合の動作パラメータの基準定義。

このモジュールのファイル

  • server/mpm/worker/mpm_default.h: プロセス・スレッド管理のデフォルト定数定義。
  • server/mpm/worker/worker.c: プロセス管理、スレッド生成、接続処理のメインロジック。

主要なデータ構造

  • worker_retained_data: モジュール再読み込み時にも保持される世代管理データ。
  • worker_child_bucket: 子プロセスの管理用構造。
  • worker_queue / worker_queue_info: リスナーとワーカー間でソケットをやり取りするためのキュー。
  • proc_info / thread_starter: プロセスおよびスレッド起動時の制御情報。

主なフロー

1. **初期化**: worker_run がメインループを開始し、startup_children が初期プロセスを生成。 2. **リスナー**: listener_thread が apr_pollset で接続を待機し、接続をキューへ投入。 3. **ワーカー**: worker_thread がキューからソケットを取得し、ap_process_connection を実行。 4. **終了処理**: signal_threads が終了フラグを伝播し、join_workers でスレッドを回収。

外部依存

  • **APR (Apache Portable Runtime)**: apr_thread_proc.h, apr_poll.h, apr_proc_mutex.h 等による OS 抽象化。
  • **Apache Core**: httpd.h, http_connection.h, scoreboard.h 等によるサーバー状態管理。
  • **MPM 共通**: mpm_common.h, mpm_fdqueue.h によるキュー管理とプロセス管理。

内部構成

  • 子モジュール数: 0

1. server/mpm/worker/mpm_default.h (設定パラメータの理解) 2. server/mpm/worker/worker.c (プロセス・スレッド管理のメインロジック)
