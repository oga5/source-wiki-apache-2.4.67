modules/metadata

システム内での役割

modules/metadata は、HTTP リクエストおよびレスポンスのメタデータ（ヘッダー、環境変数、IP 情報、キャッシュ制御、識別子など）を動的に操作・生成するためのモジュール群を収容するディレクトリです。

※本モジュールの定義は、ディレクトリ境界に基づき推論されたものです。

責務

  • **HTTP メッセージの加工**: ヘッダーの追加・削除・置換、および MIME タイプの判定。
  • **リクエストコンテキストの拡張**: 環境変数の設定、一意なリクエスト ID の生成、クライアント IP の特定。
  • **キャッシュとライフサイクルの制御**: 有効期限（Expires）の管理、Cookie による行動追跡、サーバーバージョンに応じた設定の条件分岐。
  • **外部メタデータの統合**: メタファイル（CERN 形式）や IDENT プロトコルによる外部情報の注入。

このモジュールのファイル

  • mod_cern_meta.c: メタファイルによるヘッダー注入
  • mod_env.c: CGI 環境変数の設定
  • mod_expires.c: キャッシュ有効期限の制御
  • mod_headers.c: HTTP ヘッダーの動的操作
  • mod_ident.c: IDENT プロトコルによるユーザー名照会
  • mod_mime_magic.c: マジックナンバーによる MIME タイプ判定
  • mod_remoteip.c: プロキシ経由のクライアント IP 特定
  • mod_setenvif.c: 条件付き環境変数設定
  • mod_unique_id.c: リクエスト一意 ID の生成
  • mod_usertrack.c: Cookie による行動追跡
  • mod_version.c: バージョンベースの設定条件分岐

主要なデータ構造

  • **設定構造体**: headers_conf, sei_cfg_rec, expires_dir_config, env_dir_config_rec など、各モジュールがディレクトリ単位の設定を保持。
  • **リクエスト情報**: request_rec を中心に、subprocess_env（環境変数）や headers_in/out（HTTP ヘッダー）を操作。
  • **解析用構造体**: magic (MIME 判定用ルール), header_entry (ヘッダー操作ルール), sei_entry (環境変数設定ルール)。

主なフロー

1. **設定読み込み**: 各モジュールがディレクティブを解析し、設定構造体にルールを蓄積。 2. **リクエスト初期化**: post_read_request フック等で mod_remoteip や mod_unique_id が接続情報や ID を確定。 3. **環境・ヘッダー処理**: fixups フック等で mod_headers, mod_env, mod_setenvif がリクエストコンテキストを修飾。 4. **レスポンス調整**: 出力フィルタを介して mod_expires や mod_headers が最終的なレスポンスヘッダーを調整。

外部依存

  • **Apache Core API**: httpd.h, http_config.h, http_request.h, http_protocol.h
  • **APR (Apache Portable Runtime)**: apr.h, apr_strings.h, apr_network_io.h, apr_hash.h
  • **その他**: ap_expr.h (式評価), util_filter.h (フィルタ処理)

内部構成

本ディレクトリは個別の機能を持つ独立したモジュールの集合体であり、ディレクトリ自体が共有する内部ライブラリは持ちません。各モジュールは Apache のフック機構を通じて疎結合に連携します。

推奨読書順

1. mod_mime_magic.c (判定ロジックの基礎) 2. mod_headers.c (ヘッダー操作の標準) 3. mod_remoteip.c (接続情報の重要性) 4. mod_setenvif.c (条件付き処理の理解) 5. mod_ident.c 6. mod_usertrack.c 7. mod_expires.c 8. mod_cern_meta.c 9. mod_unique_id.c 10. mod_env.c 11. mod_version.c

未解決の問い

  • 各モジュール間でのフック実行順序（APR_HOOK_FIRST 等）の競合や依存関係の網羅的な可視化。
  • 高負荷環境における mod_mime_magic や mod_unique_id のパフォーマンス最適化の余地。
