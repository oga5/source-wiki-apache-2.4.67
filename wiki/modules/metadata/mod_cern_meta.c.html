modules/metadata/mod_cern_meta.c

概要

mod_cern_meta.c は、CERN HTTPD のメタファイルセマンティクスをエミュレートする Apache HTTP Server モジュールです。リクエストされたファイルに関連付けられた「メタファイル」を読み込み、その内容を HTTP レスポンスヘッダーに追加します。主に Expires ヘッダーの制御や、カスタム MIME ヘッダーの付与に使用されます。

モジュール内での役割

modules/metadata サブシステムの一部として、リクエスト処理の fixups フェーズで動作します。他のヘッダー操作モジュール（mod_headers 等）と異なり、ファイルシステム上の外部ファイルからヘッダー情報を注入する点が特徴です。

責務

  • **設定管理**: ディレクトリ単位でのメタファイル処理の有効化、メタファイル格納ディレクトリ名、およびファイルサフィックスの構成。
  • **メタデータ注入**: リクエストされたファイルに対応するメタファイルを検索し、その内容を解析してレスポンスヘッダーへ反映する。
  • **エラーハンドリング**: メタファイルが存在しない場合は透過的に処理を継続（DECLINED）し、ファイルが破損している場合はログを出力してエラーを返す。

重要なシンボル

設定構造体と管理

  • cern_meta_dir_config: ディレクトリごとの設定（メタディレクトリ名、サフィックス、有効フラグ）を保持する構造体。
  • create_cern_meta_dir_config / merge_cern_meta_dir_configs: 設定の初期化および親ディレクトリからの継承・マージ処理。
  • cern_meta_cmds: MetaFiles, MetaDir, MetaSuffix ディレクティブを定義するコマンドテーブル。

リクエスト処理

  • add_cern_meta_data: fixups フックのメイン関数。メタファイルのパスを構築し、存在確認後に scan_meta_file を呼び出す。
  • scan_meta_file: メタファイルを開き、行単位で解析して r->headers_out にヘッダーをマージする。Content-type や Status の特殊処理も行う。

制御フローとデータフロー

1. **フック**: register_hooks により add_cern_meta_data が fixups フェーズで登録される。 2. **パス構築**: add_cern_meta_data が現在のリクエストパスからメタファイルのパス（例: dir/.web/file.html.meta）を生成する。 3. **検証**: ap_sub_req_lookup_file を使用してメタファイルへのアクセス権限を検証する。 4. **解析**: scan_meta_file がファイルを読み込み、: を区切り文字としてヘッダー名と値を分離。apr_table_t を介してレスポンスヘッダーへ追加する。 5. **終了**: ファイルハンドルをクローズし、処理を完了する。

読者向けメモ

  • **メモリ管理**: すべての動的メモリ割り当ては request_rec のプール (r->pool) を使用しており、リクエスト終了時に自動解放されます。
  • **セキュリティ**: メタファイルは Web サーバーからアクセス可能な場所に配置する必要があるため、Location ブロック等で適切に保護することを推奨します。
  • **制限事項**: scan_meta_file 内の解析ロジックは、ap_scan_script_header_err_core と類似していますが、コードの重複と経年劣化（bit rot）の可能性がある旨がソース内にコメントされています。
  • **パフォーマンス**: 各リクエストでファイルシステムへのアクセスが発生するため、高負荷環境では注意が必要です。
