modules/metadata/mod_mime_magic.c

概要

mod_mime_magic は、ファイルの内容（マジックナンバー）を解析して MIME タイプを判定する Apache HTTP Server モジュールです。UNIX の file コマンドのロジックをベースにしており、設定ファイルで定義されたパターンマッチングに基づき、ファイル先頭のバイト列からコンテンツタイプとエンコーディングを推論します。

モジュール内での役割

Apache の type_checker フックとして動作します。通常は mod_mime（拡張子ベースの判定）が先に実行され、そこで判定できなかった場合にフォールバックとして機能するよう設計されています。

責務

  • **設定管理**: MimeMagicFile ディレクティブによるマジック定義ファイルの読み込みと解析。
  • **ファイル解析**: ファイル先頭のバイト列に対するマジックナンバー照合。
  • **結果の集約**: 解析結果（MIME タイプ等）を一時的なリスト（RSL: Result String List）に保持し、最終的に request_rec へ反映。
  • **特殊処理**: 圧縮ファイル（gzip 等）の展開、tar アーカイブの識別、およびドキュメント管理システム用のリビジョンサフィックス処理。

重要なシンボル

設定・データ構造

  • struct magic: マジックナンバーのルールを保持する単方向連結リスト。
  • magic_server_config_rec: サーバー単位の設定（マジックファイルパス、ルールリスト）。
  • magic_req_rec: リクエスト単位の解析結果保持用構造体。
  • magic_rsl: 解析結果の断片を保持する連結リスト。

主要な処理関数

  • apprentice(): マジックファイルをパースし、struct magic リストを構築。
  • magic_process(): ファイルを開き、先頭バイトを読み込んで解析を開始。
  • match(): マジックリストを走査し、ルールに合致するか判定。
  • mget() / mcheck(): 特定のルールに対するバイト抽出と条件比較。
  • ascmagic(): テキストファイルや特定のファイル形式（tar 等）のヒューリスティック判定。
  • zmagic(): 圧縮ファイルの識別と展開処理。

制御フローとデータフロー

1. **初期化**: magic_init が apprentice を呼び出し、設定ファイルをメモリ上の struct magic リストに変換。 2. **リクエスト処理**: magic_find_ct がフックされ、magic_process を起動。 3. **解析**:

  • fsmagic でファイルタイプ（ディレクトリ、シンボリックリンク等）をチェック。
  • tryit を経由して zmagic（圧縮）、softmagic（マジックルール）、ascmagic（テキスト/tar）の順に試行。

    • 4. **結果反映**: magic_rsl_to_request が RSL リストを走査し、ap_set_content_type_ex を通じて request_rec に MIME タイプをセット。

読者向けメモ

  • **メモリ管理**: すべてのメモリ割り当ては Apache の apr_pool_t を使用しており、リクエスト終了時に自動解放されます。
  • **スレッド安全性**: 元の file コマンドから移植される際、グローバル変数を排除し、設定構造体経由でデータにアクセスするように変更されていますが、スレッド環境での完全な検証は不十分であるとの注記があります。
  • **パフォーマンス**: HOWMANY（4096バイト）までの読み込みを行うため、大量のファイルアクセスが発生する環境では I/O 負荷に注意が必要です。
  • **未実装・浅い部分**: revision_suffix 関数は特定のイントラネット環境に依存した実装であり、汎用的な機能ではありません。また、mcheck 内の BOINK ログなど、デバッグ用の古いコードが残存しています。
