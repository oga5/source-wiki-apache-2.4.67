modules/metadata/mod_version.c

概要

mod_version は、Apache HTTP Server の設定ファイル内で、実行中のサーバーバージョンに基づいた条件分岐（<IfVersion> コンテナ）を可能にするモジュールです。

モジュール内での役割

modules/metadata ディレクトリに属し、設定の動的な制御を担当します。他のメタデータモジュールがリクエスト処理時のヘッダーや環境変数を操作するのに対し、本モジュールは設定の読み込み段階で適用範囲を制御します。

責務

  • **バージョン比較**: 実行中のサーバーバージョン（メジャー・マイナー・パッチ）と設定値の数値比較。
  • **正規表現マッチング**: サーバーの完全なバージョン文字列に対する正規表現による条件判定。
  • **コンテナ制御**: <IfVersion> ブロック内の設定を、条件が真の場合のみ有効化し、偽の場合はスキップする。

重要なシンボル

  • version_module: Apache モジュール構造体。command_rec を通じて <IfVersion> ディレクティブを登録します。
  • start_ifversion: <IfVersion> ディレクティブのハンドラ。引数を解析し、条件判定と設定の読み込み制御を行います。
  • compare_version: 数値ベースのバージョン比較を行う内部関数。
  • match_version: 正規表現ベースのバージョン比較を行う内部関数。

制御フローとデータフロー

1. **ディレクティブ解析**: 設定読み込み時に start_ifversion が呼び出されます。 2. **バージョン取得**: 初回実行時に ap_get_server_revision() を呼び出し、現在のサーバーバージョンを httpd_version にキャッシュします。 3. **条件判定**: 引数（演算子とバージョン文字列）に基づき、compare_version または match_version を実行します。 4. **設定の適用**:

  • 条件が真の場合: ap_build_cont_config() を呼び出し、ブロック内の設定を読み込みます。
  • 条件が偽の場合: ap_soak_end_container() を呼び出し、ブロック内の設定を無視（読み飛ばし）します。

直接依存

  • apr.h, apr_strings.h, apr_lib.h: APR（Apache Portable Runtime）の文字列操作およびユーティリティ。
  • httpd.h, http_config.h, http_log.h: Apache コアのサーバー定義、設定管理、ログ出力機能。

関連コード

  • ap_release.h: ap_version_t 構造体やサーバーのバージョン定義が格納されています。
  • core.c: <IfModule> などのコンテナ処理の実装が含まれており、本モジュールの設計の参考となっています。

読者向けメモ

  • **演算子の柔軟性**: 演算子の前に ! を付与することで条件を反転可能です。
  • **正規表現の指定**: 正規表現を使用する場合、/regex/ のようにスラッシュで囲むか、~ 演算子を使用する必要があります。
  • **読み込みタイミング**: EXEC_ON_READ フラグが設定されているため、設定ファイル読み込み時に即座に評価されます。実行時のリクエスト処理には影響を与えません。
