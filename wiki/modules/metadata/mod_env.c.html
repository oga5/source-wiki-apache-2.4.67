modules/metadata/mod_env.c

概要

mod_env は、Apache HTTP Server の CGI 環境変数（subprocess_env）を制御するためのモジュールです。設定ファイル（.htaccess や httpd.conf）を通じて、OS の環境変数を引き継いだり、任意の値を設定・削除したりする機能を提供します。

モジュール内での役割

modules/metadata サブシステムの一部として、リクエスト処理の初期段階（fixups フック）で環境変数を注入する役割を担います。他のモジュールが CGI 実行時に参照する request_rec->subprocess_env テーブルを構築します。

責務

  • **設定管理**: ディレクトリ単位の設定（env_dir_config_rec）の作成とマージ。
  • **ディレクティブ処理**: PassEnv（OS環境変数の取り込み）、SetEnv（変数の設定）、UnsetEnv（変数の削除）の解析と適用。
  • **環境変数注入**: リクエスト処理の fixups フェーズにおいて、設定された変数を subprocess_env にオーバーレイする。

重要なシンボル

設定構造体

  • env_dir_config_rec: ディレクトリごとの設定を保持。vars（設定済み変数）と unsetenv（削除対象変数）の2つの apr_table_t を管理。

ディレクティブハンドラ

  • add_env_module_vars_passed: PassEnv を処理。getenv() で OS 環境変数を取得し、設定テーブルにコピー。
  • add_env_module_vars_set: SetEnv を処理。キーと値をテーブルに登録。不正な = 記号の混入を警告。
  • add_env_module_vars_unset: UnsetEnv を処理。unsetenv テーブルに記録し、現在の設定テーブルからも削除。

フックとライフサイクル

  • create_env_dir_config: 設定構造体の初期化。
  • merge_env_dir_configs: 親ディレクトリの設定と子ディレクトリの設定をマージ。UnsetEnv を優先的に適用するロジックを含む。
  • fixup_env_module: fixups フックで呼び出され、subprocess_env に変数を追加。

制御フローとデータフロー

1. **設定時**: env_module_cmds で定義されたディレクティブが解析され、env_dir_config_rec に格納されます。 2. **マージ時**: merge_env_dir_configs が呼び出され、親の設定をベースに、子の UnsetEnv を適用し、その後に子の SetEnv を上書きします。 3. **リクエスト時**: fixup_env_module がリクエストごとに実行されます。apr_table_overlay を使用して、設定された変数をリクエスト固有の subprocess_env テーブルにマージします。

直接依存

  • apr_strings.h, apr.h: 文字列操作および APR 基本型。
  • httpd.h, http_config.h, http_request.h: Apache モジュール API、設定管理、リクエスト処理用。
  • http_log.h: エラーログ出力用。
  • ap_config.h

関連コード

  • modules/metadata/mod_setenvif.c: 条件付きで環境変数を設定するモジュール。mod_env と併用されることが多い。
  • server/util_script.c: subprocess_env を実際に CGI プロセスへ渡す処理が含まれる。

読者向けメモ

  • **メモリ管理**: 設定構造体およびテーブルは、Apache のプール（apr_pool_t）によって管理されます。リクエスト終了時に自動的に解放されます。
  • **マージの順序**: merge_env_dir_configs は、UnsetEnv を先に処理してから SetEnv を適用します。これにより、同じディレクトリ階層内での設定順序に関わらず、意図した結果が得られるよう設計されています。
  • **エッジケース**: PassEnv で指定した環境変数が OS 上に存在しない場合、警告ログが出力されますが、処理は継続されます。
  • **制限**: SetEnv で変数名に = を含めることは非推奨であり、警告が出力されます。これは設定ミスを防ぐための仕様です。
