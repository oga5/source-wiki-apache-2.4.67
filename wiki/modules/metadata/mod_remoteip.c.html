modules/metadata/mod_remoteip.c

概要

mod_remoteip は、プロキシ経由で接続されたクライアントの本来の IP アドレスを特定し、Apache の request_rec 内の接続情報を上書きするためのモジュールです。HTTP ヘッダー（X-Forwarded-For 等）の解析と、PROXY プロトコル（v1/v2）のバイナリ解析の両方をサポートします。

モジュール内での役割

modules/metadata ディレクトリに属し、リクエスト処理の初期段階（post_read_request）で接続情報を修正することで、後続のモジュール（アクセス制御やログ記録）が正しいクライアント IP を参照できるようにします。

責務

  • **設定管理**: プロキシの信頼リスト（内部/外部）および PROXY プロトコルの有効化設定を保持。
  • **HTTP ヘッダー解析**: 設定されたヘッダーを走査し、信頼できるプロキシ経由の IP を抽出・検証。
  • **PROXY プロトコル処理**: コネクションフィルタとして動作し、ストリームの先頭からバイナリヘッダーを解析して接続情報を抽出。
  • **接続情報の修正**: r->useragent_addr および r->useragent_ip を上書きし、必要に応じてプロキシのリストを headers_in に記録。

重要なシンボル

設定構造体

  • remoteip_config_t: サーバー単位の設定。信頼するプロキシのリストやヘッダー名を保持。
  • remoteip_conn_config_t: 接続単位のコンテキスト。PROXY プロトコルで解決されたクライアント情報を保持。

フィルタ・解析用

  • remoteip_filter_context: remoteip_input_filter の状態保持用。ヘッダーの受信状況やバッファを管理。
  • proxy_header: PROXY プロトコル v1/v2 の共用体。
  • remoteip_req_t: リクエスト処理中に解析された IP リストを一時保持。

フック関数

  • remoteip_hook_pre_connection: PROXY プロトコルフィルタを接続チェーンに挿入するか判断。
  • remoteip_modify_request: post_read_request フック。HTTP ヘッダーを解析し、接続情報を書き換える。

制御フローとデータフロー

1. **接続確立時**: remoteip_hook_pre_connection が実行され、設定に基づいて REMOTEIP_INPUT フィルタを接続チェーンに挿入。 2. **PROXY プロトコル解析**: remoteip_input_filter がストリームの先頭を読み取り、remoteip_determine_version で v1/v2 を判別。解析完了後、conn_config に結果を格納。 3. **リクエスト処理時**: remoteip_modify_request が実行される。

  • conn_config があればそれを優先。
  • なければ RemoteIPHeader で指定されたヘッダーを解析。
  • 信頼リスト（proxymatch_ip）に基づき、プロキシの連鎖を辿って最終的なクライアント IP を特定。
  • 内部プロキシと外部プロキシの境界を判定し、不正な IP を除外。

読者向けメモ

  • **メモリ管理**: 設定はサーバーのプール（pconf）で管理されますが、リクエストごとの解析には r->pool を使用します。
  • **不変条件**: remoteip_modify_request は post_read_request の APR_HOOK_FIRST で実行されるため、他のモジュールが IP を参照する前に確実に書き換えが行われます。
  • **エッジケース**: PROXY プロトコル使用時、ヘッダーが分割されて届く可能性があるため、remoteip_input_filter は HDR_NEED_MORE を用いて状態を保持します。
  • **制限事項**: remoteip_sockaddr_compat は、ポート番号が 0 の場合をワイルドカードとして扱うなど、柔軟なマッチングを行いますが、設定ミスによるセキュリティ低下には注意が必要です。
