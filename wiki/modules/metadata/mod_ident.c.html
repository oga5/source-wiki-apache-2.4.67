modules/metadata/mod_ident.c

概要

mod_ident は、RFC 1413 (IDENT プロトコル) に基づき、リモートホストの ident デーモンに対して接続ユーザー名の照会を行う Apache HTTP Server モジュールです。この情報は認証目的には使用すべきではなく、主にログ記録用として提供されます。

モジュール内での役割

modules/metadata ディレクトリに属し、接続に関連するメタデータ（リモートユーザー名）を外部ソースから取得する役割を担います。他のモジュールから ap_ident_lookup を呼び出せるよう、オプション関数として公開されています。

責務

  • **設定管理**: IdentityCheck および IdentityCheckTimeout ディレクティブによる動作制御とタイムアウト設定の管理。
  • **ネットワーク通信**: RFC 1413 準拠のソケット通信（接続、クエリ送信、レスポンス受信）。
  • **データ変換**: プロトコル上の ASCII/EBCDIC 変換およびレスポンスのパース。
  • **ライフサイクル管理**: リクエストごとの設定マージと、接続終了時のクリーンアップ。

重要なシンボル

設定管理

  • ident_config_rec: モジュール固有の設定構造体。do_rfc1413（有効フラグ）と timeout を保持。
  • create_ident_dir_config / merge_ident_dir_config: 設定の初期化およびディレクトリ階層間でのマージ処理。
  • ident_cmds: IdentityCheck および IdentityCheckTimeout を定義するコマンドテーブル。

ネットワーク・クエリ処理

  • rfc1413_connect: 指定されたタイムアウトでリモートの 113 番ポートへ接続を確立する。
  • rfc1413_query: クエリを送信し、レスポンスを読み取って conn->remote_logname を更新する。

公開インターフェース

  • ap_ident_lookup: 他モジュールから呼び出されるメインのエントリポイント。設定を確認し、クエリ処理をディスパッチする。

制御フローとデータフロー

1. **トリガー**: 他モジュールが ap_ident_lookup を呼び出す。 2. **設定確認**: ap_get_module_config で現在のディレクトリ設定を取得。無効であれば即座に終了。 3. **接続**: rfc1413_connect が apr_socket_t を作成し、リモートの 113 番ポートへ接続。 4. **通信**: rfc1413_query が conn からポート情報を取得し、\r\n で終端されたクエリを送信。 5. **解析**: レスポンスを読み取り、sscanf でフォーマットを検証。成功時に conn->remote_logname を apr_pstrdup でプールに割り当てる。 6. **終了**: 失敗時は FROM_UNKNOWN ("unknown") を設定し、ソケットを閉じる。

読者向けメモ

  • **メモリ管理**: conn->remote_logname は conn->pool を使用して割り当てられます。リクエスト終了時に自動的に解放されます。
  • **パフォーマンス**: IDENT 照会はネットワーク I/O を伴うため、非常に低速です。本番環境での使用には注意が必要です。
  • **制約**: sscanf によるパースは固定長 (RFC1413_USERLEN) に依存しており、バッファオーバーフローを防ぐために厳密なサイズ指定が行われています。
  • **省略箇所**: 本ドキュメントでは、ap_xlate_proto_to_ascii 等のプラットフォーム依存の変換関数の内部実装詳細は省略しています。これらは主に EBCDIC 互換性のためのラッパーです。
