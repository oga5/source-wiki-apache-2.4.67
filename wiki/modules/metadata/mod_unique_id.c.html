modules/metadata/mod_unique_id.c

概要

mod_unique_id は、Apache HTTP Server の各リクエストに対して、システム全体で一意となる識別子（UNIQUE_ID）を生成するモジュールです。この識別子は、ログ記録やリクエスト追跡、分散環境でのセッション管理などに利用されます。

モジュール内での役割

modules/metadata ディレクトリに属し、リクエスト処理の初期段階で環境変数 UNIQUE_ID を設定します。他のモジュールがこの値を参照することで、リクエストのライフサイクル全体を通じた一貫した追跡が可能になります。

責務

  • **一意性の保証**: タイムスタンプ、プロセス/スレッド情報、乱数（PRNG）を組み合わせ、クラスタ環境や複数プロセス環境でも衝突しにくいIDを生成する。
  • **URLセーフなエンコーディング**: 生成したバイナリデータを [A-Za-z0-9@-_] の文字セットでエンコードし、URLに含めても安全な形式に変換する。
  • **ライフサイクル管理**: リクエストの開始時にIDを生成し、内部リダイレクト時にもIDを維持する。

重要なシンボル

  • unique_id_rec: ID生成の元となる構造体。タイムスタンプ、乱数ルート、カウンタ、スレッドIDで構成される。
  • gen_unique_id(): unique_id_rec を構築し、UUエンコードを施して文字列を返すコア関数。
  • set_unique_id(): post_read_request フックで呼ばれ、環境変数 UNIQUE_ID を設定する。
  • generate_log_id(): エラーログ生成時にIDが必要な場合、既存のIDを再利用または新規生成する。

制御フローとデータフロー

1. **初期化**: unique_id_global_init で構造体のオフセットを計算し、unique_id_child_init でプロセス固有の乱数ルートを生成する。 2. **生成**: gen_unique_id が呼ばれると、現在の時刻、接続ID、カウンタを構造体に詰め、パディングを除去したバイナリを生成する。 3. **エンコード**: バイナリデータをカスタムのUUエンコードアルゴリズムで24文字の文字列に変換する。 4. **注入**: set_unique_id がリクエストの subprocess_env に値を格納し、後続のモジュールが利用可能にする。

直接依存

  • apr_want.h, apr_general.h: APRの基本機能とバイト順序変換。
  • apr_atomic.h: スレッドセーフなカウンタ操作。
  • httpd.h, http_config.h, http_protocol.h: ApacheのコアAPIおよびフック定義。

関連コード

  • modules/metadata/mod_headers.c: 生成された UNIQUE_ID をHTTPヘッダーとして出力する場合の参考。
  • server/util_expr_eval.c: ログフォーマット等で %{UNIQUE_ID}e を評価する際の仕組み。

読者向けメモ

  • **不透明なトークン**: このIDは内部構造が将来変更される可能性があるため、他のアプリケーションは中身を解析せず、不透明な文字列として扱うべきです。
  • **スレッド安全性**: カウンタのインクリメントには apr_atomic_inc32 を使用しており、マルチスレッド環境での競合を考慮しています。
  • **パフォーマンス**: cur_unique_id の更新はキャッシュ効率を考慮した設計が求められており、コード内の XXX コメントにある通り、さらなる最適化の余地が議論されています。
