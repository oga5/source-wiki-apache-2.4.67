server/mpm/event

システム内での役割

Apache HTTP Server における Event MPM（Multi-Processing Module）の実装です。リスナー専用スレッドと非同期 I/O（epoll/kqueue）を組み合わせることで、Keep-Alive 接続がスレッドを占有し続ける問題を解決し、高並列な接続処理を実現します。

責務

  • **プロセス・スレッド管理**: 子プロセスの生成・監視およびワーカースレッドのライフサイクル制御。
  • **接続状態管理**: 接続を「処理中」「書き込み完了待ち」「Keep-Alive 待ち」等の状態に分離し、効率的に再利用。
  • **非同期 I/O 統合**: apr_pollset を活用したイベント駆動型の接続ディスパッチ。
  • **リソース調整**: 設定値に基づいた動的な接続受け入れ制御と負荷分散。

このモジュールのファイル

  • server/mpm/event/event.c: MPM の中核となるロジックとスレッド管理の実装。
  • server/mpm/event/mpm_default.h: プロセス・スレッド数のデフォルト値など、MPM 動作の基準となる定数定義。

主要なデータ構造

  • event_conn_state_t: 接続ごとの状態、タイムアウト、ソケット情報を保持。
  • timeout_queue: 接続の期限管理を行うリングバッファ。
  • event_srv_cfg: MPM のサーバー設定情報。
  • event_pollset: 非同期 I/O 監視のための apr_pollset インスタンス。

主なフロー

1. **リスナー**: listener_thread が apr_pollset_poll でソケットを監視。 2. **受け入れ**: 新規接続を worker_queue 経由でワーカーへ渡す。 3. **処理**: worker_thread が process_socket を実行。処理後、状態に応じて timeout_queue へ登録し監視を再開。 4. **タイムアウト**: リスナーが定期的に process_timeout_queue を呼び出し、期限切れ接続をクローズまたは再処理。

外部依存

  • **APR (Apache Portable Runtime)**: apr_poll, apr_thread_proc, apr_queue, apr_ring 等のプラットフォーム抽象化層。
  • **Apache Core**: httpd.h, http_connection.h, mpm_common.h 等のサーバー基本構造。

内部構成

ディレクトリ境界に基づいたモジュール構成であり、現時点で独立した子モジュールは存在しません。

推奨読書順

1. server/mpm/event/mpm_default.h: MPM の動作パラメータの理解。 2. server/mpm/event/event.c: プロセス管理および接続状態遷移ロジックの解析。
