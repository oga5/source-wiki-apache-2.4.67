modules/arch/netware/mod_netware.c

概要

NetWare プラットフォームにおける Apache HTTP Server の動作を最適化・拡張するためのモジュールです。主に CGI 実行環境の制御と、ファイル拡張子に基づいた NLM (NetWare Loadable Module) のマッピング機能を提供します。

モジュール内での役割

NetWare 環境特有の実行モデル（アドレス空間の分離や NLM の起動）を Apache の標準的な CGI 処理パイプラインに統合するブリッジの役割を果たします。

責務

  • **CGI マッピング管理**: ファイル拡張子と対応する CGI 実行エンジン（NLM）の関連付けを管理します。
  • **実行環境の制御**: 特定の拡張子に対して、独立したアドレス空間でプロセスを起動するかどうかのフラグ管理を行います。
  • **CGI コマンド構築**: ap_cgi_build_command を提供し、リクエストされたファイルと設定されたハンドラを組み合わせて実行可能なコマンドラインを生成します。

重要なシンボル

  • netware_dir_config: ディレクトリ単位の設定構造体。拡張子マッピングや環境変数を保持します。
  • ap_cgi_build_command: CGI 実行時に呼び出されるオプション関数。NetWare 用のコマンド構築ロジックを実装しています。
  • set_extension_map: 設定ディレクティブ CGIMapExtension のハンドラ。
  • netware_module: モジュールのメイン構造体。

制御フローとデータフロー

1. **設定フェーズ**: CGIMapExtension ディレクティブにより、netware_dir_config 内の file_type_handlers テーブルが構築されます。 2. **リクエスト処理**: CGI リクエストが発生すると ap_cgi_build_command が呼び出されます。 3. **コマンド構築**:

  • リクエストされたファイルの拡張子を特定。
  • file_type_handlers から対応する NLM を検索。
  • file_handler_mode を確認し、必要に応じて e_info->addrspace を設定。
  • 最終的な実行コマンド文字列を生成し、引数をトークン化して返却します。

読者向けメモ

  • 本モジュールは #ifdef NETWARE で囲まれており、NetWare 以外のプラットフォームではコンパイルされません。
  • ap_cgi_build_command は APR_REGISTER_OPTIONAL_FN を介して公開されており、他のモジュール（主に mod_cgi）から動的に呼び出される設計になっています。
  • Windows 版のロジックと類似した拡張子解析を行っていますが、NetWare 特有の「アドレス空間の分離」という概念が e_info->addrspace に反映されている点に注意してください。
