modules/arch/netware/libprews.c

概要

NetWare NLM (NetWare Loadable Module) として動作する Apache モジュールのライフサイクル管理を行うための補助ライブラリです。標準的な main() 関数を使用せずにモジュールを初期化・終了させることで、ロード時間の短縮とメモリフットプリントの削減を実現します。

モジュール内での役割

modules/arch/netware サブシステムにおいて、OS レベルの初期化フックを提供します。他のモジュール（mod_netware.c 等）が Apache のモジュールとして機能する以前の、NLM としての基盤的なセットアップを担います。

責務

  • **NLM ライフサイクル管理**: モジュールのロード時およびアンロード時のエントリーポイントを提供。
  • **Winsock 初期化**: USE_WINSOCK が定義されている場合、ネットワークスタック（Winsock）の起動と終了を管理。
  • **アンロード可否判定**: モジュールが安全にアンロード可能かどうかのチェック。

重要なシンボル

  • _NonAppStart: NLM ロード時に呼び出される初期化関数。Winsock の初期化を行います。
  • _NonAppStop: NLM アンロード時に呼び出される終了関数。Winsock のリソース解放を行います。
  • _NonAppCheckUnload: モジュールのアンロード可否を判定する関数。現在は常に成功（0）を返します。

制御フローとデータフロー

1. **ロード時**: NLM ローダーが _NonAppStart を呼び出します。引数は多数ありますが、本ファイルでは主に USE_WINSOCK 定義下での WSAStartup 実行に利用されます。 2. **実行中**: モジュールはメモリ上に常駐し、Winsock が利用可能な状態となります。 3. **アンロード時**: NLM ローダーが _NonAppCheckUnload で確認を行い、その後 _NonAppStop を呼び出して WSACleanup を実行し、ネットワークリソースを解放します。

直接依存

  • netware.h: NetWare OS の基本定義。
  • novsock2.h: NetWare 環境における Winsock 2 API 定義（USE_WINSOCK 定義時）。

関連コード

  • mod_netware.c: 本ファイルが提供する基盤上で動作する、Apache モジュールとしてのメインロジック。
  • mod_nw_ssl.c: NetWare 環境特有の SSL 処理を行うモジュール。

読者向けメモ

  • 本ファイルは、Apache モジュール特有の module 構造体とは直接関係ありません。あくまで NetWare OS 上での NLM の挙動を最適化するためのラッパーです。
  • pragma unused が多用されていますが、これは NLM の標準的なインターフェース仕様に合わせつつ、実際には使用しない引数に対するコンパイラ警告を抑制するためのものです。
  • 独自の NetWare モジュールを開発する際、標準的な Apache モジュールの枠組み以外で OS レベルのクリーンアップが必要な場合は、このパターンを拡張して利用することが推奨されます。
