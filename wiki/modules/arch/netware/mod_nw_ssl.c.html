modules/arch/netware/mod_nw_ssl.c

概要

NetWare プラットフォーム専用の Apache HTTP Server SSL/TLS モジュールです。NetWare OS が提供するネイティブな SSL/TLS 実装（WinSock 拡張）を Apache に統合し、SecureListen ディレクティブによるセキュアなリスニングポートの作成や、RFC 2817 に基づく TLS アップグレード機能を提供します。

モジュール内での役割

NetWare 環境における SSL/TLS 処理のゲートウェイです。Apache の標準的な mod_ssl とは異なり、OS レベルの WinSock IOCTL を直接呼び出すことで、NetWare の SSL スタックを利用します。

責務

  • **リスナー管理**: SecureListen による専用 SSL ポートの作成と、既存リスナーとの統合。
  • **TLS アップグレード**: NWSSLUpgradeable による、平文接続から TLS への動的移行（RFC 2817）。
  • **接続の抽象化**: ssl_is_https や ssl_var_lookup などのオプション関数を提供し、他モジュール（mod_proxy 等）との連携を可能にする。
  • **証明書管理**: NetWare の証明書ストア（KMO）と連携するための証明書リストの構築。

重要なシンボル

設定構造体

  • NWSSLSrvConfigRec: サーバーごとの SSL 設定（リスナーテーブル、アップグレード可能ポートテーブル）を保持。
  • seclisten_rec / seclistenup_rec: セキュアリスナーおよびアップグレード可能リスナーの定義を保持するリンクリスト構造体。

接続・ソケット操作

  • make_secure_socket: WinSock IOCTL を使用して SSL 対応のリスニングソケットを生成。
  • convert_secure_socket: 既存の接続を SSL/TLS に変換する。
  • SSLize_Socket: アップグレード時にソケットを SSL モードへ切り替える。

フック関数

  • nwssl_pre_config / nwssl_post_config: 起動時のリスナー再構成とソケットの初期化。
  • nwssl_pre_connection: 接続開始時に SSL 状態を判定し、必要に応じて設定を適用。
  • ssl_io_filter_Upgrade: アップグレード要求を処理する出力フィルタ。

制御フローとデータフロー

1. **初期化**: nwssl_pre_config が既存のリスナーを一時退避させ、nwssl_post_config が SecureListen で定義された設定に基づき、WinSock を介して SSL 対応ソケットを生成・登録します。 2. **接続時**: nwssl_pre_connection が接続を監視し、SSL 接続であれば secsocket_data を接続コンテキストに付与します。 3. **アップグレード**: ssl_io_filter_Upgrade が Upgrade: TLS/1.0 ヘッダーを検知すると、101 Switching Protocols を返送し、SSLize_Socket を呼び出してソケットを暗号化状態へ遷移させます。 4. **変数参照**: ssl_var_lookup がリクエストや接続情報から SSL 関連環境変数を抽出します。

直接依存

  • httpd.h 系: Apache コア API。
  • apr_portable.h / apr_strings.h: APR 抽象化レイヤー。
  • unilib.h: NetWare の Unicode 変換ライブラリ。
  • WinSock (WSAIoctl): NetWare ネイティブの SSL/TLS 制御インターフェース。

関連コード

  • modules/arch/netware/mod_netware.c: NetWare MPM のメインモジュール。
  • modules/arch/netware/libprews.c: NetWare 固有の WinSock 補助関数。

読者向けメモ

  • **メモリ管理**: ほとんどの動的メモリは Apache の pool に紐付けられており、明示的な free は不要です。
  • **制約**: NetWare の SSL スタックに依存しているため、mod_ssl のような OpenSSL ベースの柔軟な設定（暗号スイートの細かな制御など）は提供されません。
  • **未実装部分**: ssl_var_lookup 内のいくつかの SSL 固有変数（SSL_CIPHER 等）は、NetWare の API 制限により取得できず、コメントアウトされています。
  • **注意点**: nwssl_pre_config でリスナーを操作しているため、このモジュールのロード順序や MPM との整合性は非常に重要です。
