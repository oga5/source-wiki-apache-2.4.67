modules/arch/win32/mod_isapi.h

概要

mod_isapi.h は、Apache HTTP Server 上で ISAPI (Internet Server Application) 拡張を動作させるためのインターフェース定義ヘッダーです。Microsoft IIS の ISAPI 2.0 仕様をベースに、Apache 側でエミュレーションまたはサポートする構造体、関数ポインタ、および定数を定義しています。

モジュール内での役割

本ヘッダーは mod_isapi.c が実装する ISAPI ハンドラアーキテクチャの契約を規定します。Windows 環境において、外部の ISAPI DLL が Apache と通信するための共通言語を提供し、Apache の内部リクエスト処理と ISAPI のコールバック関数を橋渡しします。

責務

  • **ISAPI 仕様の定義**: EXTENSION_CONTROL_BLOCK (ECB) を中心とした、ISAPI 拡張が利用するデータ構造の定義。
  • **通信インターフェースの提供**: GetServerVariable や ServerSupportFunction などの関数ポインタ型の定義。
  • **拡張機能のサポート**: HSE_TF_INFO や HSE_SEND_HEADER_EX_INFO など、IIS 5.0 相当の拡張機能の定義。
  • **状態管理の抽象化**: HSE_STATUS_* 定数による、非同期処理やセッション終了の制御フローの定義。

重要なシンボル

ライフサイクル・エントリポイント

  • GetExtensionVersion: DLL ロード時に呼び出される初期化関数。
  • HttpExtensionProc: 各リクエストのメイン処理関数。
  • TerminateExtension: DLL アンロード時に呼び出される終了処理関数。

通信・制御構造体

  • EXTENSION_CONTROL_BLOCK: ISAPI 拡張に渡されるリクエストコンテキストの核。
  • HCONN: isapi_cid への不透明なポインタ。Apache 側のリクエスト状態を保持。
  • HSE_VERSION_INFO: バージョン情報および拡張機能の説明用構造体。

サーバーサポート関数 (ServerSupportFunction)

  • HSE_REQ_*: ServerSupportFunction を通じて実行される各種操作（リダイレクト、パス変換、ログ追加、非同期完了通知など）。

I/O 拡張

  • HSE_TF_INFO: TransmitFile 相当のファイル転送用構造体。
  • HSE_SEND_HEADER_EX_INFO: 拡張ヘッダー送信用の構造体。

制御フローとデータフロー

  • **リクエスト処理**: Apache が HttpExtensionProc を呼び出す際、EXTENSION_CONTROL_BLOCK を介してリクエスト情報とコールバック関数群（WriteClient 等）を渡します。
  • **非同期エミュレーション**: 本モジュールはネイティブな非同期 I/O を持たないため、HSE_IO_SYNC フラグが指定されない場合、完了関数を即時実行する「偽装非同期」フローを採用しています。
  • **セッション管理**: HSE_STATUS_PENDING が返された場合、Apache は HSE_REQ_DONE_WITH_SESSION が呼び出されるまでセッションを維持します。

直接依存

  • apr_uint32_t (APR ライブラリ経由の型定義)
  • apr_os_file_t (ファイルハンドル操作用)

関連コード

  • modules/arch/win32/mod_isapi.c: 本ヘッダーで定義されたインターフェースの実際の実装。
  • modules/arch/win32/mod_win32.c: Windows プラットフォーム固有の機能を提供する関連モジュール。

読者向けメモ

  • **メモリ所有権**: EXTENSION_CONTROL_BLOCK などの構造体は、Apache がリクエストのライフサイクルに合わせて管理します。ISAPI 側で永続化してはなりません。
  • **未サポート機能**: HSE_REQ_GET_SSPI_INFO や HSE_REQ_GET_CERT_INFO_EX など、一部の IIS 固有機能は Apache のアーキテクチャと整合しないため、意図的にサポートされていません。
  • **スレッド安全性**: APR_THREAD_FUNC が指定されている通り、各エントリポイントはマルチスレッド環境で呼び出されることを前提としています。
  • **ヘッダーの制約**: HSE_SEND_HEADER_EX_INFO のヘッダー文字列には NULL 文字を含めないでください。Apache の内部処理と競合する可能性があります。
