modules/arch/unix/mod_unixd.c

概要

mod_unixd.c は、Apache HTTP Server が Unix 系 OS 上で動作する際に必要な、特権管理およびプロセス環境設定を行うコアモジュールです。主に、サーバー起動後の特権降格（root から指定ユーザーへの切り替え）、chroot 環境の構築、および suEXEC の有効性検証を担当します。

モジュール内での役割

Unix 系 MPM（Multi-Processing Module）の共通基盤として機能します。サーバーの起動フェーズで設定を読み込み、プロセス生成時に安全な実行環境を整えるためのフックを提供します。

責務

  • **特権管理**: setuid/setgid および initgroups を用いたプロセス実行ユーザーの切り替え。
  • **環境分離**: chroot によるプロセス実行ディレクトリの制限。
  • **設定管理**: User, Group, ChrootDir, Suexec ディレクティブの解析と検証。
  • **初期化と検証**: サーバー起動時の suEXEC バイナリの存在確認と権限チェック。

重要なシンボル

設定管理（ディレクティブハンドラ）

  • unixd_set_user: User ディレクティブを処理し、ユーザー名を ID に変換。
  • unixd_set_group: Group ディレクティブを処理し、グループ名を ID に変換。
  • unixd_set_chroot_dir: ChrootDir ディレクティブの検証と設定。
  • unixd_set_suexec: Suexec フラグの有効化と制約チェック。

特権・環境制御

  • ap_unixd_setup_child: 子プロセス生成時に呼び出される公開関数。特権降格と chroot を実行。
  • set_group_privs: グループ ID の設定と initgroups による補助グループの初期化。
  • unixd_drop_privileges: サーバー全体の特権降格プロセスを管理する内部フック。

ライフサイクル・フック

  • unixd_pre_config: 起動時のデフォルト値設定と suEXEC の環境チェック。
  • unixd_hooks: モジュールのフック登録（pre_config, test_config, drop_privileges）。

制御フローとデータフロー

1. **初期化**: unixd_pre_config が APR_HOOK_FIRST で実行され、ap_unixd_config 構造体をデフォルト値で初期化し、suEXEC バイナリのパーミッションを検証します。 2. **設定**: unixd_cmds で定義されたディレクティブが解析され、ap_unixd_config が更新されます。 3. **実行時**: 子プロセス生成時、ap_unixd_setup_child が呼び出されます。

  • set_group_privs でグループ権限を確定。
  • chroot が設定されていれば実行。
  • setuid でユーザー権限を降格。
  • Linux 環境では prctl を使用してコアダンプの可否を設定。

読者向けメモ

  • **メモリ所有権**: ap_unixd_config は静的またはグローバルな領域で管理されており、モジュールのライフサイクル全体で生存します。
  • **制約**: chroot や setuid は root 権限で実行されている場合にのみ成功します。失敗時は APLOG_ALERT でログ出力されます。
  • **未実装・浅い部分**: OS2 や _OSD_POSIX などのレガシー環境向けの分岐が含まれていますが、現代的な Linux/Unix 環境では標準的なパスのみが実行されます。
  • **注意点**: set_group_privs 内で getpwuid を使用してユーザー名を取得するロジックは、設定で #uid 形式が指定された場合の互換性維持のために存在します。
