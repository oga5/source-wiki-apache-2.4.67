modules/arch/unix/mod_unixd.h

概要

mod_unixd.h は、Apache HTTP Server の Unix 系プラットフォーム向けマルチプロセッシングモジュール（MPM）が共通して利用する、権限管理および子プロセス初期化のためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

modules/arch/unix ディレクトリにおいて、Unix 特有のシステムコール（ユーザー/グループ ID の設定や権限の降格など）を抽象化する基盤を提供します。MPM が子プロセスを起動する際、安全に実行権限を切り替えるための標準的な窓口として機能します。

責務

  • Unix MPM が子プロセス生成時に呼び出すべき共通関数の宣言。
  • プラットフォーム依存の権限管理ロジックを、各 MPM から分離して提供する。

重要なシンボル

  • ap_unixd_setup_child(): 子プロセスが実行を開始する直前に呼び出される関数。プロセスが適切なユーザー権限で動作するように設定を行い、必要に応じて権限の降格（privilege dropping）を実行します。

制御フローとデータフロー

本ファイル自体は宣言のみを提供します。実際のフローは以下の通りです： 1. MPM が子プロセスをフォークする。 2. 子プロセス内で ap_unixd_setup_child() が呼び出される。 3. mod_unixd.c 内の実装が、設定ファイルで指定されたユーザー/グループ ID を取得し、setgid や initgroups などのシステムコールを介してプロセスの権限を確定させる。

直接依存

  • ap_config.h: Apache の基本設定およびプラットフォーム固有の定義をインポートするために使用されます。

関連コード

  • modules/arch/unix/mod_unixd.c: ap_unixd_setup_child の実体および権限管理の具体的なロジックが含まれています。
  • include/unixd.h: 権限管理に関連するデータ構造（unixd_config_t 等）が定義されており、本モジュールと密接に関連しています。

読者向けメモ

  • 本ファイルは非常に軽量ですが、Apache のセキュリティモデルの根幹に関わる部分です。
  • mod_unixd.c 内の TODO コメントにある通り、unixd.h との役割分担が歴史的に複雑な状態です。コードを追う際は、mod_unixd.h が提供するインターフェースと、unixd.h が提供するデータ構造の両方を参照する必要があります。
  • 権限の降格処理は、root 権限で起動されたプロセスが安全に非特権ユーザーへ移行するために不可欠なプロセスです。デバッグ時には setuid/setgid の戻り値チェックに注目してください。
