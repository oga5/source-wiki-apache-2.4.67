modules/arch/unix/mod_privileges.c

概要

Solaris などの Unix 系システムにおける「特権（Privileges）」機能を活用し、仮想ホスト（vhost）単位でプロセス権限を分離・制限するための Apache モジュールです。プロセスが持つ権限セットを動的に変更することで、セキュリティを強化します。

モジュール内での役割

modules/arch/unix ディレクトリに属し、OS レベルの特権管理を Apache のリクエスト処理パイプラインに統合します。mod_unixd と密接に関連し、サーバー起動時およびリクエスト処理時の権限降格（Privilege Dropping）を制御します。

責務

  • **権限設定の管理**: vhost ごとの UID/GID および特権セット（priv_set_t）の保持。
  • **リクエスト時の権限降格**: リクエスト処理開始時に、設定された権限セットへプロセスを移行。
  • **セキュアな分離**: PRIV_SECURE モードにおいて、リクエストを別プロセスで処理（fork）し、権限を最小化して実行。
  • **ライフサイクル管理**: サーバー起動時の初期化、リクエスト終了時の権限復元、および設定メモリのクリーンアップ。

重要なシンボル

設定構造体

  • priv_cfg: サーバー/vhost レベルの設定。UID, GID, 特権セット（自身用および子プロセス用）、動作モードを保持。
  • priv_dir_cfg: ディレクトリ単位の設定。PrivilegesMode を保持。

ライフサイクル・フック

  • privileges_hooks: モジュールのフック登録（post_read_request, drop_privileges 等）。
  • privileges_postconf: 起動時の初期化。DTrace 設定の統合や priv_setid の準備。
  • privileges_init: MPM がスレッドセーフでないことを確認（本モジュールは非スレッド環境専用）。

権限操作関数

  • privileges_req: リクエストごとの権限降格処理。fork の判断と setppriv による権限適用。
  • privileges_end_req: リクエスト終了時に元の権限へ復元。
  • privileges_drop_first / privileges_drop_last: サーバー起動時の権限降格シーケンス。

制御フローとデータフロー

1. **初期化**: privileges_postconf でシステム全体の特権セットを準備。 2. **リクエスト処理**: privileges_req が post_read_request で呼ばれる。

  • PRIV_SECURE モードの場合、apr_proc_fork で子プロセスを生成。
  • 子プロセス内で setppriv を呼び出し、PRIV_EFFECTIVE, PRIV_INHERITABLE, PRIV_LIMIT を設定。

    • 3. **終了処理**: apr_pool_cleanup_register により privileges_end_req が登録され、リクエスト完了後に権限を復元。 4. **データフロー**: priv_cfg は ap_get_module_config を通じてリクエスト処理中に参照され、setppriv の引数として渡される。

関連コード

  • mod_unixd.c: サーバー全体のユーザー/グループ権限管理。本モジュールはこれと連携し、権限降格のタイミングを調整する。

読者向けメモ

  • **スレッド制限**: 本モジュールは setppriv を使用するため、マルチスレッド MPM（Worker/Event）とは互換性がありません。privileges_init で厳格にチェックされます。
  • **メモリ管理**: priv_set_t は priv_allocset で確保され、priv_cfg_cleanup や privileges_term で適切に解放されます。
  • **セキュリティ警告**: BIG_SECURITY_HOLE マクロで囲まれた機能は、権限を直接操作する危険な機能です。デフォルトでは無効化されています。
  • **省略箇所**: priv_set_t の詳細な内部構造や、OS 固有の setppriv の挙動については、Solaris 等のシステムドキュメントを参照してください。本モジュールはそれらの API を Apache の設定と結びつけるラッパーとして機能します。
