modules/dav/main

システム内での役割

modules/dav/main は、Apache HTTP Server における WebDAV (RFC 4918) 実装の中核を担うディレクトリです。リポジトリやストレージの実装から独立したプロトコル処理の抽象レイヤーを提供し、WebDAV メソッドのディスパッチ、プロパティ管理、ロック制御、および拡張可能なプロバイダーインターフェースを定義します。

責務

  • **プロトコル実装**: WebDAV メソッド（PROPFIND, PUT, LOCK 等）のハンドリングと XML レスポンスの生成。
  • **抽象化レイヤー**: リポジトリ、プロパティDB、ロック管理、バージョン管理のためのフック（dav_hooks_*）の定義。
  • **プロバイダー管理**: 外部モジュールが DAV 機能を拡張するための登録・検索メカニズムの提供。
  • **共通ユーティリティ**: XML 解析、If ヘッダーのパース、自動バージョン管理、エラーハンドリングの共通化。

このモジュールのファイル

  • mod_dav.h: API 定義、データ構造、フックの宣言。
  • mod_dav.c: メインハンドラ、リクエストディスパッチ、設定管理。
  • util.c: 汎用ユーティリティ、If ヘッダー処理、自動バージョン管理。
  • props.c: プロパティデータベース（propdb）の管理とトランザクション処理。
  • util_lock.c: ロック管理、再帰的ロック適用、ロック・ヌルリソースの制御。
  • liveprop.c: ライブプロパティの名前空間管理と検索。
  • std_liveprop.c: 標準的なライブプロパティ（DAV: 名前空間）の実装。
  • providers.c: プロバイダー登録・検索のための Apache API ラッパー。

主要なデータ構造

  • **フック構造体**: dav_hooks_repository, dav_hooks_propdb, dav_hooks_locks, dav_hooks_vsn, dav_hooks_liveprop
  • **リソース管理**: dav_resource (リソースの抽象表現), dav_propdb (プロパティ操作コンテキスト)
  • **設定**: dav_dir_conf (ディレクトリごとの DAV 設定)
  • **エラー**: dav_error (スタックベースのエラー管理構造体)

主なフロー

1. **リクエスト捕捉**: mod_dav.c が Apache の handler フックを介してリクエストを捕捉。 2. **リソース解決**: 設定されたプロバイダーの get_resource を呼び出し、リクエスト URI を dav_resource に変換。 3. **メソッド実行**: メソッドに応じた dav_method_* 関数が呼び出され、必要に応じて util.c や props.c のユーティリティを利用。 4. **プロパティ/ロック処理**: props.c や util_lock.c がプロバイダーのフックを介してバックエンドと連携。 5. **応答生成**: dav_send_multistatus を通じて XML 形式のレスポンスをクライアントへ返却。

外部依存

  • **Apache Core**: httpd.h, http_config.h, http_protocol.h, http_request.h
  • **APR**: apr_pools.h, apr_hash.h, apr_strings.h, apr_tables.h
  • **XML**: util_xml.h (Apache XML 解析 API)

内部構成

本ディレクトリは mod_dav のコアロジックを構成しており、サブディレクトリ（fs など）に配置される具体的なストレージ実装を統合する役割を持ちます。

推奨読書順

1. mod_dav.h (API とフック定義の理解) 2. mod_dav.c (全体的なリクエストフローの把握) 3. util.c (共通ユーティリティと検証ロジック) 4. props.c (プロパティ管理とトランザクション) 5. util_lock.c (ロック管理の仕組み) 6. liveprop.c / std_liveprop.c (ライブプロパティの拡張) 7. providers.c (プロバイダー登録メカニズム)
