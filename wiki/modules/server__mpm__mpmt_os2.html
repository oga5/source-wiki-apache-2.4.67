server/mpm/mpmt_os2

システム内での役割

OS/2 プラットフォーム専用のマルチプロセッシングモジュール（MPM）です。Apache HTTP Server のプロセス管理およびスレッド管理を OS/2 ネイティブ API（Dos* 系）を用いて実装し、マルチプロセス・マルチスレッド環境を提供します。

責務

  • **プロセス管理**: 親プロセスによる子プロセスの生成、監視、および終了処理。
  • **スレッド管理**: 子プロセス内でのスレッドプール生成と、アイドル状態に応じた動的なスレッド数調整。
  • **リソース共有**: 共有メモリを用いたリスナーソケットとスコアボードのプロセス間受け渡し。
  • **接続処理**: OS/2 ネイティブのキューを用いた接続の受け入れとワーカーへのディスパッチ。

このモジュールのファイル

  • mpm_default.h: プロセス・スレッド管理に関するデフォルト設定値の定義。
  • mpmt_os2.c: 親プロセスのメインループおよびプロセス管理ロジック。
  • mpmt_os2_child.c: 子プロセスの実行環境、スレッドプール管理、およびリクエスト処理。

主要なデータ構造

  • listen_socket_t: 子プロセスへ引き継ぐリスナーソケット情報。
  • parent_info_t: 共有メモリ上に配置される、プロセス間で共有する同期用ミューテックスやソケット情報。
  • worker_args_t: 子プロセス内のワーカーへ接続情報を伝達するための構造体。

主なフロー

1. **起動**: mpmt_os2_run が呼び出され、親プロセスが共有メモリを確保してリスナーソケットを準備します。 2. **分岐**: DosGetNamedSharedMem の成否により、親プロセス（master_main）と子プロセス（ap_mpm_child_main）に分岐します。 3. **親プロセス**: DosWaitChild で子プロセスを監視し、必要に応じて spawn_child で補充します。 4. **子プロセス**: 共有メモリからソケットを取得し、OS/2 キューを介して接続をワーカーへ割り当てます。server_maintenance が定期的にスレッド数を最適化します。

外部依存

  • **OS/2 API**: os2.h（DosAllocSharedMem, DosExecPgm, DosCreateQueue 等）。
  • **Apache/APR**: httpd.h, ap_mpm.h, scoreboard.h, apr_portable.h, apr_poll.h 等。

内部構成

本モジュールは OS/2 特有の API に強く依存しており、プラットフォーム固有のプロセスモデルを前提としています。子プロセス数は HARD_SERVER_LIMIT（10）、スレッド数は HARD_THREAD_LIMIT（256）によって制限されています。

推奨読書順

1. mpm_default.h（設定パラメータの把握） 2. mpmt_os2.c（親プロセスの制御フローとプロセス間通信の理解） 3. mpmt_os2_child.c（子プロセス内のスレッドプール管理とリクエスト処理の詳細）

未解決の問い

  • OS/2 以外のプラットフォームへの移植性や、現代の OS/2 環境におけるパフォーマンス特性の検証。
  • Dos* API のエラーハンドリングにおける、より堅牢なリカバリ戦略の策定。
