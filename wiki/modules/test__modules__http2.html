test/modules/http2

システム内での役割

test/modules/http2 は、Apache HTTP Server の mod_http2 モジュールおよび関連するプロキシ機能（mod_proxy_http2）の品質保証を担う統合テストスイートです。HTTP/2 プロトコル仕様（RFC 9113 等）への適合性、セキュリティ境界、および高負荷時の安定性を検証します。

責務

  • **プロトコル適合性検証**: HTTP/2 の基本機能（GET/POST）、条件付きリクエスト、Range リクエスト、トレーラー、パディング等の仕様準拠の確認。
  • **プロキシ機能の検証**: mod_proxy_http2 を介したバックエンドとの通信、接続再利用、ヘッダー転送、エラーハンドリングのテスト。
  • **セキュリティと制限の検証**: 不正なヘッダー処理、SSL 再ネゴシエーションの禁止、タイムアウト制御、リクエスト制限の検証。
  • **高度な機能の検証**: Server Push、Early Hints (103)、WebSocket (RFC 8441) サポートの検証。

このモジュールのファイル

本モジュールは、環境設定用の env.py、conftest.py および 37 個のテストファイルで構成されています。

  • **環境基盤**: env.py (設定・環境管理), conftest.py (フィクスチャ定義)
  • **基本通信**: test_001_httpd_alive.py ～ test_009_timing.py
  • **接続・セキュリティ**: test_100_conn_reuse.py ～ test_107_frame_lengths.py
  • **ヘッダー・プロトコル**: test_200_header_invalid.py ～ test_203_rfc9113.py
  • **プッシュ・ヒント**: test_400_push.py, test_401_early_hints.py
  • **プロキシ**: test_500_proxy.py ～ test_601_h2proxy_twisted.py
  • **負荷・その他**: test_700_load_get.py ～ test_800_websockets.py

主要なデータ構造

  • H2TestEnv: テスト環境の構成と Apache のライフサイクルを管理する基底クラス。
  • H2Conf: Apache の設定ファイル（httpd.conf）を動的に生成・適用するためのヘルパークラス。
  • WsFrameReader / WsFrame: WebSocket テストで使用されるフレーム解析用構造。

主なフロー

1. **セットアップ**: _class_scope フィクスチャが H2Conf を用いて Apache を設定し、テスト用モジュールをロードして再起動。 2. **リクエスト実行**: curl や nghttp、h2load を用いて、定義されたシナリオに従いサーバーへリクエストを送信。 3. **検証**: レスポンスコード、ヘッダー、ボディ、およびサーバーのログ（error_log）を解析し、期待値と一致するかをアサート。 4. **クリーンアップ**: 必要に応じて apache_restart() を呼び出し、ログのフラッシュや状態のリセットを行う。

外部依存

  • pytest: テストフレームワーク。
  • curl, nghttp, h2load: HTTP/2 通信および負荷試験用クライアントツール。
  • pyhttpd: Apache テスト用ユーティリティフレームワーク。

内部構成

ディレクトリ境界に基づき、機能ごとにテストファイルが分割されています。

  • **プロキシ系**: test_5xx, test_6xx
  • **プロトコル・ヘッダー系**: test_2xx
  • **負荷試験系**: test_7xx

推奨読書順

1. test/modules/http2/env.py（基盤理解） 2. test/modules/http2/test_003_get.py（基本 GET） 3. test/modules/http2/test_004_post.py（基本 POST） 4. test/modules/http2/test_400_push.py（高度な機能） 5. test/modules/http2/test_600_h2proxy.py（プロキシ機能）
