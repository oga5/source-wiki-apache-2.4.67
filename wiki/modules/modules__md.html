modules/md

システム内での役割

modules/md は、Apache HTTP Server における Managed Domain (MD) サブシステムを構成するモジュールです。ACME プロトコルを用いた証明書の自動取得、更新、および管理を担い、サーバーの運用負荷を軽減します。このモジュールはディレクトリ境界に基づいて定義されています。

責務

  • **証明書ライフサイクル管理**: 証明書の自動更新判定、取得、および適用。
  • **ACME プロトコル実装**: サーバーとの通信、アカウント管理、ドメイン認証（チャレンジ）、証明書発行オーダーの制御。
  • **データ永続化**: 設定、証明書、秘密鍵、ACME アカウント情報の安全な保存と読み込み。
  • **抽象化レイヤーの提供**: HTTP 通信、JSON 操作、暗号化処理（OpenSSL）、ログ出力の共通インターフェース化。

このモジュールのファイル

58個のソースファイルで構成され、主に以下のカテゴリに分類されます。

  • **コア・レジストリ**: mod_md.c, md_reg.c, md_core.c
  • **ACME プロトコル**: md_acme.c, md_acme_acct.c, md_acme_authz.c, md_acme_order.c, md_acme_drive.c
  • **ストレージ・設定**: md_store.c, md_store_fs.c, mod_md_config.c
  • **ユーティリティ・基盤**: md_crypt.c, md_http.c, md_json.c, md_ocsp.c, md_util.c, md_log.c, md_time.c

主要なデータ構造

  • md_t: 管理対象ドメインの設定と状態を保持するメイン構造体。
  • md_reg_t: ドメイン登録情報のレジストリハンドル。
  • md_acme_t: ACME サーバーとの通信セッション状態。
  • md_store_t: 永続化層の抽象インターフェース。
  • md_result_t: 処理結果、エラー詳細、サブ問題の管理コンテナ。
  • md_job_t: バックグラウンドジョブの実行状態と履歴管理。

主なフロー

1. **初期化**: mod_md が設定を読み込み、md_reg を通じてドメイン情報を管理下に置く。 2. **更新判定**: md_reg_should_renew により、証明書の有効期限に基づき更新が必要か判断。 3. **ACME 処理**: md_acme_drive を介してサーバーへオーダーを発行し、チャレンジ（認証）をクリア。 4. **証明書取得**: 発行された証明書を md_store を通じてステージング領域に保存。 5. **適用**: md_reg_load_staging により、検証済みの証明書を本番領域へ昇格。

外部依存

  • **APR (Apache Portable Runtime)**: メモリ管理、ファイル I/O、データ構造（ハッシュ、配列）。
  • **OpenSSL**: 暗号化、秘密鍵生成、X.509 証明書処理。
  • **jansson**: JSON データの解析および生成。
  • **libcurl (推奨)**: md_http を介したネットワーク通信バックエンド。

内部構成

本モジュールは、特定のビジネスロジックを担う層と、それを支える基盤層に分かれています。

  • **ビジネス層**: md_reg, md_acme, md_ocsp
  • **基盤層**: md_store, md_http, md_crypt, md_json, md_util

推奨読書順

1. modules/md/md.h (基本データモデル) 2. modules/md/md_reg.h (レジストリ管理) 3. modules/md/md_acme.h (ACME プロトコル基盤) 4. modules/md/md_store.h (ストレージ抽象化) 5. modules/md/md_http.h (通信抽象化) 6. modules/md/md_crypt.h (暗号化・証明書処理) 7. modules/md/mod_md_config.h (設定管理)
