test/modules/http1

システム内での役割

Apache HTTP Server (httpd) の HTTP/1.1 プロトコル実装に対する回帰テストおよび機能検証を行うテストスイートです。本モジュールは、サーバーのプロトコル準拠性、リクエスト処理の堅牢性、および設定変更に対する動的な応答を検証します。

責務

  • HTTP/1.1 プロトコルの正常系（GET/POST）および境界条件（トレーラー、厳格なパース設定）の検証。
  • テスト実行環境（Apache HTTP Server）の動的な構築、設定、およびライフサイクル管理。
  • 不正なリクエストやエッジケースに対するサーバーの耐性確認。

このモジュールのファイル

  • __init__.py: パッケージ定義。
  • conftest.py: 共有フィクスチャおよびテスト実行フックの定義。
  • env.py: テスト環境構築、Apache 設定管理、およびデータ生成の基盤。
  • test_001_alive.py: 疎通確認（スモークテスト）。
  • test_003_get.py: GET リクエストおよび環境変数伝播の検証。
  • test_004_post.py: POST リクエストおよびファイルアップロードの検証。
  • test_005_trailers.py: HTTP トレーラーの送受信検証。
  • test_006_unsafe.py: HttpProtocolOptions Unsafe 設定下の挙動検証。
  • test_007_strict.py: HttpProtocolOptions Strict 設定下の挙動検証。

主要なデータ構造

  • H1TestEnv: テスト実行環境を抽象化するクラス。
  • H1Conf: Apache の設定ファイル（httpd.conf）を動的に生成・管理するクラス。
  • curl レスポンスオブジェクト: HTTP 通信の結果を保持し、ステータスコードやヘッダー、JSON ボディを検証するために使用。

主なフロー

1. **環境セットアップ**: conftest.py が H1TestEnv を通じて Apache を起動し、テスト用モジュール（mod_h1test）をコンパイル・ロードする。 2. **設定適用**: 各テストクラスが H1Conf を使用して特定のテスト用設定（Strict/Unsafe モードや CGI 有効化など）を適用し、必要に応じて Apache を再起動する。 3. **リクエスト実行**: curl や低レイヤーの socket 通信を用いて、サーバーに対してテストケースに応じたリクエストを送信する。 4. **検証**: レスポンスのステータスコード、ヘッダー、またはボディをアサーションで検証する。 5. **クリーンアップ**: テスト終了時に Apache を停止し、リソースを解放する。

外部依存

  • pytest: テストフレームワーク。
  • httpd (Apache HTTP Server): テスト対象。
  • apxs: テスト用モジュール（mod_h1test）のコンパイルに必要。
  • curl: HTTP リクエスト送信ツール。

内部構成

  • env.py を中心とした環境管理層と、test_*.py による機能テスト層の2階層で構成されています。

推奨読書順

1. test/modules/http1/env.py（環境構築の基盤を理解する） 2. test/modules/http1/conftest.py（テスト実行のライフサイクルを理解する） 3. test/modules/http1/test_001_alive.py（最もシンプルなテストケース） 4. test/modules/http1/test_003_get.py / test_004_post.py（標準的なリクエスト処理） 5. test/modules/http1/test_006_unsafe.py / test_007_strict.py（プロトコル厳格性の境界条件）
