test/modules/md

システム内での役割

test/modules/md は、Apache HTTP Server の mod_md モジュールに対する統合テストスイートです。ACME プロトコルを用いた証明書の自動取得・管理、および Apache 設定との整合性を検証する 42 個のテストファイルで構成されています。このモジュールはディレクトリ境界に基づき定義されています。

責務

  • mod_md の主要機能（証明書ライフサイクル、ACME チャレンジ、OCSP ステープリング、EAB）の統合テスト。
  • a2md 管理ツールおよび Apache 設定ディレクティブの妥当性検証。
  • テスト環境（ACME サーバー、証明書ストア、Apache プロセス）のライフサイクル管理。

このモジュールのファイル

  • **基盤・ユーティリティ**: md_env.py (環境制御), md_certs.py (証明書生成), md_cert_util.py (検証), md_conf.py (設定生成)
  • **主要テスト**:
  • test_001_store.py, test_310_conf_store.py: ストア管理と設定の永続化
  • test_100_reg_add.py, test_110_reg_update.py: ドメイン登録・更新
  • test_202_acmev2_regs.py: ACMEv2 アカウント管理
  • test_502_acmev2_drive.py, test_702_auto.py: 証明書取得（Drive）プロセス
  • test_720_wildcard.py: ワイルドカード証明書
  • test_750_eab.py, test_751_sectigo.py, test_752_zerossl.py: EAB 関連
  • test_801_stapling.py, test_810_ec.py: OCSP ステープリングと EC 鍵
  • test_901_message.py, test_920_status.py: 通知機能とステータスレポート

主要なデータ構造

  • **MDTestEnv**: テスト環境の抽象化クラス。Apache の起動、ACME サーバーのポーリング、ストアのパージを管理。
  • **MDConf**: Apache 設定ファイル（httpd.conf）を動的に構築・インストールするヘルパー。
  • **CertificateSpec**: 証明書の属性（ドメイン、鍵タイプ、有効期限）を定義するデータクラス。

主なフロー

1. **環境構築**: MDTestEnv が ACME サーバーを起動し、ストアを初期化。 2. **設定適用**: MDConf で Apache 設定を生成し、apache_restart() で反映。 3. **駆動**: mod_md が ACME サーバーと通信し、証明書を取得（a2md drive または自動駆動）。 4. **検証**: env.await_completion() で完了を待ち、MDCertUtil で証明書の整合性やステータスを検証。 5. **クリーンアップ**: env.clear_store() でファイルシステム上の状態をリセット。

外部依存

  • **ACME サーバー**: Pebble または Boulder（テスト実行に必須）。
  • **外部ツール**: a2md バイナリ、openssl、curl。
  • **Python ライブラリ**: pytest, cryptography, PyOpenSSL。

内部構成

  • 本モジュールは pyhttpd フレームワークを拡張しており、各テストファイルは pytest のフィクスチャを使用して Apache のライフサイクルを制御します。
  • 構成は「基盤（md_env 等）」「CLI テスト（test_0xx）」「ACME フローテスト（test_5xx, test_7xx）」「機能テスト（test_8xx, test_9xx）」に分類されます。

推奨読書順

1. test/modules/md/md_env.py (環境の基盤を理解) 2. test/modules/md/md_conf.py (設定生成ロジック) 3. test/modules/md/test_001_store.py (基本操作) 4. test/modules/md/test_502_acmev2_drive.py (主要な駆動フロー) 5. test/modules/md/test_702_auto.py (自動更新の検証)
