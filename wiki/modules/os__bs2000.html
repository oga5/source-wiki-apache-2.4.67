os/bs2000

システム内での役割

os/bs2000 は、BS2000（OSD/POSIX）メインフレーム環境において Apache HTTP Server を動作させるための OS 固有の抽象化レイヤーです。ディレクトリ境界に基づき構成されており、プラットフォーム特有のプロセス管理と文字コード変換機能を提供します。

責務

  • **プロセス管理の抽象化**: BS2000 特有のジョブ環境切り替え（ufork）をラップし、サーバーの実行権限やプロセス生成を制御する。
  • **文字コード変換**: ネイティブな EBCDIC 環境と、ネットワーク標準である ASCII との間の相互変換機能を提供する。

このモジュールのファイル

  • os/bs2000/ebcdic.c: 文字コード変換関数の実装と変換テーブル。
  • os/bs2000/ebcdic.h: 文字コード変換のインターフェース定義。
  • os/bs2000/os.c: プロセス生成およびジョブ環境管理の実装。
  • os/bs2000/os.h: プラットフォーム定義および OS 固有関数のプロトタイプ。

主要なデータ構造

  • bs2_ForkType: プロセス生成手法（noFORK, FORK, UFORK）を決定するための列挙型。

主なフロー

1. **プロセス生成**: os_fork() が呼び出されると、os_forktype() が現在の実行環境（デバッグモードや権限）を判定し、標準の fork() または BS2000 特有の ufork() を選択して実行します。 2. **文字コード変換**: ebcdic2ascii() 等の関数が呼び出されると、静的に定義された変換テーブルを参照し、入力バッファをバイト単位で変換して出力バッファへ書き込みます。

外部依存

  • ../unix/os.h: Unix 互換レイヤーの基本定義。
  • httpd.h, http_config.h, http_log.h: Apache コア機能およびログ出力。
  • apr_lib.h: APR ユーティリティ。
  • ap_config.h, sys/types.h: システム設定および型定義。

内部構成

  • 子モジュールは存在しません。

推奨読書順

1. os/bs2000/os.h: プラットフォームの基本定義とインターフェースの把握。 2. os/bs2000/ebcdic.h: 文字コード変換の役割と方針の理解。 3. os/bs2000/os.c: プロセス生成ロジックの詳細確認。 4. os/bs2000/ebcdic.c: 変換テーブルと実装の詳細確認。
