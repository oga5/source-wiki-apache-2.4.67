test/pyhttpd

システム内での役割

test/pyhttpd は、Apache HTTP Server (httpd) の統合テストを実行するための Python ベースのテストフレームワークです。テスト環境の構築から、サーバーのライフサイクル管理、HTTP/1.1 や HTTP/2、WebSocket を用いた通信テスト、およびログ検証までを網羅的にサポートします。

責務

  • **テスト環境のオーケストレーション**: 設定ファイルの動的生成、証明書発行、サーバープロセスの制御。
  • **クライアントシミュレーション**: curl や nghttp をラップし、リクエスト送信とレスポンスの構造化解析を行う。
  • **検証基盤の提供**: 通信の遅延検証、ログの差分監視、プロトコルフレームの解析。

このモジュールのファイル

  • __init__.py: パッケージ定義。
  • env.py: テスト環境の司令塔（環境構築・サーバー管理）。
  • conf.py: httpd.conf の動的生成と設定変換。
  • certs.py: TLS 用の証明書・鍵の生成と管理。
  • curl.py: curl コマンドによる通信テスト・遅延検証。
  • nghttp.py: nghttp コマンドによる HTTP/2 テスト。
  • ws_util.py: WebSocket フレームの構築・解析。
  • result.py: コマンド実行結果の構造化データクラス。
  • log.py: エラーログの監視とフィルタリング。

主要なデータ構造

  • HttpdTestEnv: テスト環境のグローバル状態を保持する中心的なクラス。
  • ExecResult: 外部コマンドの実行結果（終了コード、出力、時間）を保持するコンテナ。
  • Credentials: 証明書と秘密鍵を管理するデータクラス。
  • WsFrame: WebSocket フレームのバイナリ構造を表現するクラス。

主なフロー

1. **初期化**: HttpdTestEnv が設定を読み込み、ディレクトリ構造を構築。 2. **構築**: HttpdConf を用いて設定ファイルを生成し、certs.py で必要な証明書を発行。 3. **実行**: curl や nghttp を介してサーバーへリクエストを送信。 4. **解析**: 実行結果を ExecResult として取得し、必要に応じてログ（log.py）やストリーム状態を検証。

外部依存

  • cryptography: 証明書生成。
  • subprocess: 外部コマンド実行。
  • re, json, struct: 出力解析およびバイナリ操作。

内部構成

本モジュールは、環境管理（env.py）、設定（conf.py）、証明書（certs.py）、クライアント（curl.py, nghttp.py）、結果管理（result.py）、ログ監視（log.py）、プロトコル補助（ws_util.py）の各コンポーネントで構成される疎結合な設計です。

推奨読書順

1. env.py（全体像の把握） 2. conf.py / certs.py（環境構築の理解） 3. curl.py / nghttp.py（テスト実行の理解） 4. result.py / log.py（検証ロジックの理解） 5. ws_util.py（特殊プロトコル対応）

未解決の問い

  • 現在の curl のヘッダー解析は一時ファイルに依存しており、並列実行時の競合リスクがある。将来的にメモリベースの解析へ移行可能か。
  • nghttp の出力解析は正規表現に依存しているため、ツール側の出力フォーマット変更に対する堅牢性をどう確保するか。
