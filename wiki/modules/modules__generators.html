modules/generators

システム内での役割

modules/generators は、Apache HTTP Server においてリクエストに対するレスポンスを生成（生成・出力）する主要なハンドラ群をまとめたディレクトリです。CGI スクリプトの実行、ディレクトリ索引の生成、サーバー状態の可視化など、クライアントへ返すコンテンツを動的に構築するモジュールが配置されています。

責務

  • **コンテンツ生成**: リクエストに応じて動的なレスポンスを生成・送信する。
  • **実行環境の提供**: CGI や SSI 実行のためのプロセス管理および環境構築。
  • **診断・監視**: サーバーの稼働状況や設定情報の可視化。
  • **共通ロジックの集約**: CGI 実行系モジュール間での共通処理（バケット管理、ヘッダー解析等）の共有。

このモジュールのファイル

  • **CGI 実行系**: mod_cgi.c, mod_cgid.c, mod_cgi.h, cgi_common.h, mod_suexec.c, mod_suexec.h
  • **索引生成**: mod_autoindex.c
  • **診断・監視**: mod_status.c, mod_status.h, mod_info.c
  • **その他**: mod_asis.c

主要なデータ構造

  • **CGI 関連**: cgi_exec_info_t (プロセス生成情報), cgi_bucket_data (非同期 I/O 用), suexec_config_t (権限設定)
  • **索引関連**: autoindex_config_rec (ディレクトリ設定), ent (ファイルメタデータ)
  • **診断関連**: info_svr_conf (設定情報), status_hook (拡張用フック)

主なフロー

1. **リクエスト受信**: 各モジュールのハンドラがリクエストをフック。 2. **設定・権限確認**: mod_suexec 等が実行権限を解決し、ディレクトリごとの設定を適用。 3. **コンテンツ生成**:

  • CGI: mod_cgi/mod_cgid がプロセスを起動し、cgi_common.h のロジックで入出力をパイプ処理。
  • 索引: mod_autoindex がディレクトリを走査し、HTML を動的生成。
  • 診断: mod_status/mod_info がサーバー内部状態を走査し、レポートを出力。

    • 4. **レスポンス送信**: フィルタチェーンを通じてクライアントへデータをストリーム。

外部依存

  • **APR**: apr_proc_t, apr_pollset_t, apr_bucket_brigade 等によるプロセス・I/O 管理。
  • **Apache Core**: httpd.h, http_config.h, http_protocol.h 等のサーバー基盤 API。
  • **OS 依存**: Unix ドメインソケット (mod_cgid) や権限管理 (mod_suexec)。

内部構成

本モジュールは、特定の階層構造を持たないフラットな構成です。CGI 実行系は cgi_common.h を中心にロジックを共有し、診断系は scoreboard.h や ap_conftree といったコアのデータ構造を読み取ることで機能を実現しています。

推奨読書順

1. modules/generators/cgi_common.h (CGI 実行の基盤) 2. modules/generators/mod_cgi.h / mod_suexec.h (インターフェース定義) 3. modules/generators/mod_cgi.c (標準的な CGI 実装) 4. modules/generators/mod_cgid.c (デーモン型 CGI 実装) 5. modules/generators/mod_autoindex.c (索引生成ロジック) 6. modules/generators/mod_status.c / mod_info.c (診断機能) 7. modules/generators/mod_asis.c (特殊なレスポンス生成)
