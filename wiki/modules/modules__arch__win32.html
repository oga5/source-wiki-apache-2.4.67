modules/arch/win32

システム内での役割

modules/arch/win32 は、Apache HTTP Server を Windows 環境で運用するためのプラットフォーム固有の拡張機能群です。主にレガシーな ISAPI 拡張のサポートと、Windows 特有の CGI 実行環境の最適化を担います。

責務

  • **ISAPI サポート**: Microsoft IIS 2.0/5.0 互換の ISAPI 拡張を Apache 上で実行するためのエミュレーション層の提供。
  • **CGI 実行の最適化**: Windows レジストリやシバン行に基づくスクリプトインタープリタの自動解決と、プロセス生成時のコマンドライン構築。
  • **プラットフォーム統合**: Windows 特有のファイルシステム、レジストリ、およびプロセス管理と Apache のリクエスト処理パイプラインの橋渡し。

このモジュールのファイル

  • mod_isapi.c: ISAPI 拡張のロード、ライフサイクル管理、およびコールバック関数の実装。
  • mod_isapi.h: ISAPI 拡張との通信に必要な構造体（EXTENSION_CONTROL_BLOCK 等）およびインターフェース定義。
  • mod_win32.c: Windows 環境における CGI 実行支援およびインタープリタ解決ロジック。

主要なデータ構造

  • **ISAPI 関連**: EXTENSION_CONTROL_BLOCK (リクエストコンテキスト), isapi_cid (接続状態), isapi_loaded (DLL キャッシュ), HSE_VERSION_INFO。
  • **CGI/設定関連**: win32_dir_conf (ディレクトリごとの設定), interpreter_source_e (インタープリタ解決戦略)。

主なフロー

1. **ISAPI リクエスト**: isapi_handler が DLL をロードし、Apache のリクエスト情報を EXTENSION_CONTROL_BLOCK に変換して HttpExtensionProc を呼び出します。 2. **CGI 実行**: ap_cgi_build_command フックが呼び出されると、レジストリまたはシバン行から適切な実行バイナリを特定し、Windows 形式のコマンドラインを構築します。

外部依存

  • **APR (Apache Portable Runtime)**: apr_dso.h (DLL ロード), apr_regkey.h (レジストリ操作), apr_thread_mutex.h 等。
  • **Apache Core**: httpd.h, http_config.h, http_protocol.h 等の標準的なリクエスト処理 API。

内部構成

本モジュールはディレクトリ境界によって定義された論理的なグループであり、個別のサブモジュールは持ちません。各ソースファイルが独立した機能（ISAPI または CGI 支援）を実装しています。

推奨読書順

1. modules/arch/win32/mod_isapi.h (インターフェース定義の理解) 2. modules/arch/win32/mod_isapi.c (ISAPI 実装の詳細) 3. modules/arch/win32/mod_win32.c (Windows 固有の CGI 支援機能)
