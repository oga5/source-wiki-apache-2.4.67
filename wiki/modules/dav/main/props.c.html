modules/dav/main/props.c

概要

WebDAV プロパティデータベース（propdb）のハンドリングを担う中心的なファイルです。リポジトリ非依存のプロパティ管理、ライブプロパティ（liveprop）とデッドプロパティ（deadprop）の統合、および PROPPATCH 操作のトランザクション管理（検証・実行・コミット・ロールバック）を実装しています。

モジュール内での役割

mod_dav のプロパティサブシステムの中核です。liveprop.c で定義されるライブプロパティプロバイダーと、DBM ベースのデッドプロパティストレージを抽象化し、リクエスト処理層に対して統一されたインターフェースを提供します。

責務

  • **プロパティデータベース管理**: dav_propdb 構造体のライフサイクル管理（オープン、クローズ、遅延初期化）。
  • **プロパティ解決**: ライブプロパティとコアプロパティ（DAV: 名前空間）の検索と解決。
  • **XML 変換**: プロパティ値の XML 出力生成と名前空間（xmlns）の管理。
  • **トランザクション処理**: PROPPATCH に伴う検証、実行、ロールバックの各フェーズの制御。

重要なシンボル

データベース管理

  • dav_propdb: プロパティ操作のコンテキストを保持する主要構造体。
  • dav_open_propdb / dav_popen_propdb: propdb の初期化。DB アクセスは遅延評価（deferred）される。
  • dav_close_propdb: リソースの解放とサブリクエストの破棄。

プロパティ解決・挿入

  • dav_find_liveprop: XML 要素からライブプロパティプロバイダーを特定し、priv にキャッシュする。
  • dav_insert_liveprop / dav_insert_coreprop: プロパティ値を XML 形式で出力バッファに挿入する。
  • dav_get_allprops / dav_get_props: PROPFIND リクエストに対するプロパティ一覧の生成。

トランザクション（PROPPATCH）

  • dav_prop_validate: 操作の妥当性確認と、必要に応じた DB のオープン。
  • dav_prop_exec: プロパティの更新または削除の実行。
  • dav_prop_commit / dav_prop_rollback: トランザクションの確定または失敗時の復旧。

制御フローとデータフロー

1. **初期化**: dav_popen_propdb でコンテキストを作成。DB オープンは deferred フラグにより、実際にプロパティが必要になるまで先送りされます。 2. **解決**: dav_find_liveprop がライブプロパティプロバイダーを検索し、見つからない場合はデッドプロパティとして扱います。 3. **出力**: dav_get_props は、ライブプロパティとデッドプロパティをマージし、dav_xmlns_info を用いて名前空間の競合を解決しながら XML を構築します。 4. **更新**: PROPPATCH 時には validate -> exec -> commit/rollback の順で呼び出され、dav_rollback_item を介して失敗時の状態復旧を保証します。

直接依存

  • mod_dav.h: DAV の主要なフックとデータ構造。
  • apr_strings.h, apr_want.h: 文字列操作と標準型。
  • http_log.h, http_request.h: Apache HTTPD のリクエスト処理とログ出力。
  • apr.h

関連コード

  • liveprop.c: ライブプロパティプロバイダーの登録と実行フック。
  • std_liveprop.c: 標準的なライブプロパティの実装。
  • util.c: XML 名前空間管理やユーティリティ関数。

読者向けメモ

  • **メモリ管理**: dav_propdb はリクエストプール（r->pool）または指定されたプールを使用します。サブリクエスト（subreq）は dav_close_propdb で明示的に破棄する必要があります。
  • **名前空間**: プロパティ値内の名前空間は ns<index> 形式に変換されます。クライアントは特定のプレフィックスに依存してはなりません。
  • **制約**: DAV_DISABLE_WRITABLE_PROPS が有効な場合、一部のコアプロパティは書き込み不可となります。
  • **未実装・浅い部分**: 名前空間の「ガベージコレクション」は未実装であり、将来的な課題としてコメントされています。また、PROPPATCH のエラーハンドリングにおいて、一部の複雑なケースでエラーの伝播が不完全な箇所があります。
