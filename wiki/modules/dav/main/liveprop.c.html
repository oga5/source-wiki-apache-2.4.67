modules/dav/main/liveprop.c

概要

WebDAV における「ライブプロパティ（Live Property）」の名前空間管理およびプロパティ検索を司るモジュールです。サーバーがサポートするライブプロパティの URI を一元管理し、プロパティ名と名前空間から対応するフックや ID を解決する機能を提供します。

モジュール内での役割

mod_dav のプロパティ処理サブシステムの一部です。props.c がプロパティの読み書きを抽象化するのに対し、本ファイルは「どのプロパティがどの名前空間に属し、どのプロバイダによって処理されるか」というメタデータ構造の解決を担当します。

責務

  • **名前空間の登録とインデックス化**: ライブプロパティで使用される URI を一意な数値 ID にマッピングし、グローバルな名前空間テーブルを維持する。
  • **プロパティ検索**: 指定された名前空間とプロパティ名から、該当する dav_liveprop_spec および処理用フックを特定する。
  • **XML 出力支援**: プロパティのシリアライズ時に必要な xmlns 属性の生成を補助する。

重要なシンボル

名前空間管理

  • dav_liveprop_uris: 登録された URI をキー、インデックスを値とする apr_hash_t。
  • dav_register_liveprop_namespace(): URI をハッシュに登録し、インデックスを割り当てる。
  • dav_get_liveprop_ns_index(): URI に対応するグローバルインデックスを取得する。

プロパティ検索・解決

  • dav_do_find_liveprop(): 名前空間とプロパティ名から、対応するプロパティ ID とフックを検索する。
  • dav_get_liveprop_info(): プロパティ ID から仕様情報（dav_liveprop_spec）を逆引きする。
  • dav_register_liveprop_group(): プロバイダが提供するプロパティ群を登録し、関連する名前空間を初期化する。

ユーティリティ

  • dav_add_all_liveprop_xmlns(): 登録済みの全名前空間を xmlns:lpN="URI" 形式で出力バッファに追加する。

制御フローとデータフロー

1. **登録フェーズ**: モジュール初期化時に dav_register_liveprop_group が呼ばれ、プロバイダが定義する名前空間リストが dav_liveprop_uris ハッシュに登録されます。 2. **検索フェーズ**: dav_do_find_liveprop は、提供された dav_liveprop_group 内の namespace_uris 配列を走査して名前空間を特定し、次に specs 配列を走査してプロパティ名の一致を確認します。 3. **データフロー**: dav_add_all_liveprop_xmlns は apr_hash_t をイテレートし、メモリプール p を介して apr_text_header に文字列を蓄積します。

直接依存

  • apr_pools.h, apr_hash.h: 名前空間の永続化とハッシュ管理。
  • util_xml.h: XML 出力用テキストヘッダー操作。
  • mod_dav.h: dav_liveprop_group や dav_liveprop_spec などのデータ構造定義。
  • apr_errno.h
  • apr_strings.h

関連コード

  • std_liveprop.c: 標準的なライブプロパティの実装例。本モジュールの検索機能を利用する主要なクライアント。
  • props.c: プロパティデータベースの操作。本モジュールで特定したフックを使用して実際の値を取得する。

読者向けメモ

  • **メモリ管理**: dav_liveprop_uris はプロセス生存期間中保持される静的変数ですが、apr_pool_cleanup_register によりプール破棄時にクリーンアップされる設計です。
  • **インデックスの性質**: dav_liveprop_count は 1 から始まる連番であり、XML 出力時の lp1, lp2 といったプレフィックス生成に使用されます。
  • **制約**: dav_do_find_liveprop は線形探索を行うため、プロパティ数が多い場合はパフォーマンスに影響する可能性がありますが、通常は小規模なセットであるため許容されています。
  • **省略箇所**: 本ファイルはメタデータの管理に特化しており、実際のプロパティ値の取得ロジック（get や set）は各プロバイダのフックに委譲されているため、ここには含まれません。
