modules/dav/main/mod_dav.h

概要

mod_dav.h は、Apache HTTP Server の WebDAV (RFC 4918 および関連拡張) 実装における中心的なヘッダーファイルです。リポジトリ非依存のコアロジックと、具体的なストレージやバージョン管理機能を提供するプロバイダー間のインターフェース（フック）を定義します。

モジュール内での役割

本ファイルは mod_dav の API 定義です。リポジトリ、プロパティデータベース、ロック管理、バージョン管理、バインディング、検索といった各機能の抽象化レイヤーを提供し、mod_dav.c をはじめとする各実装ファイルが共通の規約に従うための基盤となります。

責務

  • **API 定義**: WebDAV プロバイダーが実装すべきフック構造体の定義。
  • **データ構造**: リソース、エラー、ロック、プロパティ、XML 名前空間管理などの共通データ構造の提供。
  • **エラー管理**: WebDAV 特有のエラーコードと、エラースタック処理の定義。
  • **ユーティリティ**: XML 解析、バッファ操作、マルチステータスレスポンス生成などの共通ヘルパー関数の提供。

重要なシンボル

エラー管理

  • dav_error: エラー状態を保持するスタック構造体。dav_new_error 等で生成・管理。
  • dav_handle_err(): WebDAV エラーを Apache のレスポンスに変換するメインハンドラ。

リソースとプロバイダー

  • dav_resource: WebDAV リソースの抽象表現。リポジトリ固有の info とフックを保持。
  • dav_provider: リポジトリ、プロパティDB、ロック、バージョン管理等のフックを束ねる構造体。
  • dav_hooks_*: 各機能（repository, propdb, locks, vsn 等）の関数ポインタテーブル。

ロックとプロパティ

  • dav_lock: リソースに対するロック情報を保持。
  • dav_prop_ctx: プロパティ変更（validate/exec/commit/rollback）の 3 フェーズ処理用コンテキスト。

XML ユーティリティ

  • dav_xmlns_info: XML 名前空間の動的生成と管理用構造体。

制御フローとデータフロー

  • **リクエスト処理**: mod_dav はリクエストを受け取ると、設定されたプロバイダーの get_resource を呼び出し、リソースを特定します。
  • **フック呼び出し**: メソッド（PROPFIND, PUT 等）に応じて、dav_provider 内の適切なフック（repos_hooks, propdb_hooks 等）が呼び出されます。
  • **エラーフロー**: 各フックは dav_error* を返します。エラーが発生した場合、dav_handle_err を通じて適切な HTTP ステータスコードと XML レスポンスが生成されます。
  • **プロパティ変更**: dav_prop_validate -> dav_prop_exec -> dav_prop_commit の順で、トランザクション的なプロパティ更新が行われます。

直接依存

  • apr_hooks.h, apr_hash.h, apr_dbm.h, apr_tables.h: APR ユーティリティ。
  • httpd.h, util_filter.h, util_xml.h: Apache HTTP Server コアおよび XML 解析機能。
  • limits.h, time.h: 標準 C ライブラリ。

関連コード

  • mod_dav.c: 本ヘッダーで定義された API を利用するメインハンドラ。
  • util.c, util_lock.c: 本ヘッダーで宣言されたユーティリティ関数の実装。
  • props.c, liveprop.c: プロパティ処理のロジック。
  • providers.c: プロバイダーの登録と検索の実装。

読者向けメモ

  • **メモリ管理**: 多くの構造体は apr_pool_t を引数に取ります。リソースやプロバイダーの生存期間は、関連付けられたリクエストプールに依存します。
  • **ABI の注意**: dav_resource の uri フィールドなど、一部にエンコード状態の曖昧さが残る ABI の課題があります。
  • **拡張性**: 本ヘッダーは非常に広範なフックを定義していますが、一部（特にバージョン管理の高度な機能）はオプションであり、プロバイダーが実装しない場合は NULL を設定することで機能が無効化されます。
  • **未実装領域**: 本ヘッダーはインターフェース定義に特化しており、具体的なストレージ実装（mod_dav_fs 等）の詳細は含まれていません。
