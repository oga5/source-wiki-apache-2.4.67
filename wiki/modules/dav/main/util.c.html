modules/dav/main/util.c

概要

WebDAV プロトコル処理におけるリポジトリ非依存の汎用ユーティリティ群を提供します。エラーハンドリング、バッファ管理、URI 解析、XML 操作、If ヘッダーのパース、および自動バージョン管理（Auto-versioning）のロジックを担います。

モジュール内での役割

mod_dav の中核となる補助モジュールです。リポジトリ固有の実装（repository.c 等）に依存せず、HTTP リクエストの検証や WebDAV 特有のヘッダー処理を共通化することで、各プロバイダーの実装負荷を軽減します。

責務

  • **エラー管理**: dav_error 構造体の生成、スタック管理、連結。
  • **バッファ操作**: 可変長バッファの動的確保と文字列/メモリの追加。
  • **リクエスト検証**: If ヘッダーのパース、リソース状態とロックトークンの照合。
  • **XML ユーティリティ**: apr_xml 構造体からの要素検索、属性取得、CDATA 抽出。
  • **自動バージョン管理**: リソースの自動チェックアウト/チェックイン処理。

重要なシンボル

エラーハンドリング

  • dav_new_error, dav_new_error_tag: dav_error オブジェクトの生成。
  • dav_push_error, dav_join_error: エラーのスタック化と連結。

バッファ管理

  • dav_buffer_init, dav_buffer_append: dav_buffer 構造体の初期化と追記。
  • dav_check_bufsize: 必要に応じたバッファの拡張。

XML 操作

  • dav_find_child, dav_find_attr: 名前空間を考慮した要素・属性の検索。
  • dav_xml_get_cdata: XML 要素内の CDATA を連結して取得。

リクエスト検証

  • dav_process_if_header: If ヘッダーのパースと構造化。
  • dav_validate_request: リクエスト全体（リソースおよび親）の検証フローの統括。

自動バージョン管理

  • dav_auto_checkout, dav_auto_checkin: バージョン管理プロバイダーと連携した自動チェックアウト/イン処理。

制御フローとデータフロー

1. **検証フロー**: dav_validate_request が起点となり、dav_process_if_header でヘッダーを解析後、dav_validate_resource_state を呼び出してリソースのロック状態と If 条件を照合します。 2. **XML 解析**: apr_xml_doc を受け取り、dav_find_child_ns 等のヘルパーを用いて階層を辿り、dav_xml_get_cdata で最終的な値を抽出します。 3. **自動バージョン管理**: dav_auto_checkout はリソースの auto_versionable フックを確認し、必要に応じて checkout フックを呼び出します。エラー発生時は dav_auto_checkin（undo モード）で状態をロールバックします。

読者向けメモ

  • **メモリ管理**: すべての動的確保は apr_pool_t を介して行われます。リクエスト終了時に自動解放されるため、明示的な free は不要です。
  • **バッファの不変性**: dav_buffer は cur_len と alloc_len を持ち、dav_check_bufsize で自動拡張されます。
  • **If ヘッダーの複雑性**: dav_process_if_header は非常に複雑なステートマシンです。RFC 4918 の If ヘッダー仕様に厳密に従う必要があるため、修正時は注意が必要です。
  • **省略された詳細**: 本ファイル内の dav_xmlns_* 系関数は XML 名前空間の管理用ですが、実装はハッシュテーブルベースの単純なマッピングであり、特筆すべき複雑なロジックは含まれていません。
