modules/dav/main/util_lock.c

概要

WebDAV プロトコルにおけるリポジトリ非依存のロック管理機能を提供します。リソースのロック状態のシリアライズ、XML 解析、ロックの継承、および「ロック・ヌル（lock-null）」リソースの管理を担当します。

モジュール内での役割

mod_dav サブシステムにおいて、リポジトリの実装（dav_hooks_repository）とロックデータベースの実装（dav_hooks_locks）を繋ぐ仲介層です。リポジトリ固有の詳細はフック経由で委譲し、このファイルはプロトコルレベルのロジック（Depthヘッダーの解釈やXML生成など）を統括します。

責務

  • **ロック情報の変換**: dav_lock 構造体と XML（<activelock>）間の相互変換。
  • **ロックの伝播**: DAV_INFINITY 深さを持つロックの再帰的な適用および継承。
  • **ライフサイクル管理**: ロックの追加（dav_add_lock）、解除（dav_unlock）、およびリソース生成時の状態遷移（dav_notify_created）。
  • **ロック・ヌル管理**: 存在しないリソースに対するロック状態の判定と、リソース作成時のロック状態の昇格。

重要なシンボル

ロック操作・変換

  • dav_lock_get_activelock: dav_lock リストを XML 文字列に変換します。
  • dav_lock_parse_lockinfo: クライアントからの LOCK リクエスト XML を解析し、dav_lock 構造体を生成します。
  • dav_add_lock: 指定リソースにロックを追加し、必要に応じて子リソースへ再帰的に適用します。
  • dav_unlock: 指定されたトークンに基づき、リソースおよびその配下のロックを解除します。

状態管理・継承

  • dav_get_resource_state: リソースが通常、ロック・ヌル、または存在しない状態かを判定します。
  • dav_notify_created: リソース作成時に、親からのロック継承やロック・ヌル状態の解消をトリガーします。
  • dav_inherit_locks: 親リソースから子リソースへロックを伝播させます。

ウォーカー（内部ヘルパー）

  • dav_lock_walker / dav_unlock_walker / dav_inherit_walker: walk フックで使用されるコールバック関数群。再帰的なロック操作の単位処理を担当します。

制御フローとデータフロー

1. **ロック追加**: dav_add_lock が呼び出されると、まず対象リソースに直接ロックを適用し、depth > 0 の場合は resource->hooks->walk を介して dav_lock_walker を全子孫に適用します。 2. **ロック解除**: dav_unlock は dav_get_direct_resource を使用して、間接ロックの起点となる直接ロックのリソースを特定し、そこから dav_unlock_walker を用いて再帰的に削除します。 3. **継承**: dav_inherit_locks は親リソースのロックを取得し、dav_inherit_walker を通じて新しいリソースへ再帰的に適用します。

読者向けメモ

  • **メモリ管理**: r->pool を多用しており、リクエストのライフサイクルに依存します。
  • **設計上の注意**: dav_lock_get_activelock 内の dav_buffer は、効率的な文字列構築のために使用されています。
  • **未実装・課題**: コード内の ### コメントで示されている通り、エラーハンドリングの粒度や、一部の再帰処理におけるマルチステータスレスポンスの生成には改善の余地があります。
  • **ロック・ヌル**: r->path_info を使用した判定は、リポジトリ実装によっては正しく機能しない可能性があるため、注意が必要です。
