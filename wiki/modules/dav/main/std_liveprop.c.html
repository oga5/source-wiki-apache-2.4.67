modules/dav/main/std_liveprop.c

概要

WebDAV プロトコルにおける「ライブプロパティ（Live Property）」のコア実装を提供します。主に DAV: 名前空間に属する標準的なリソースプロパティの定義と、それらを XML レスポンスへ挿入するためのフックを管理します。

モジュール内での役割

mod_dav のライブプロパティサブシステムの一部です。他のプロバイダが提供するライブプロパティとは異なり、本ファイルは DAV 仕様で定義された基本的なリソース属性（resourcetype 等）を処理する「コア」プロバイダとして機能します。

責務

  • **プロパティ定義**: DAV: 名前空間内の標準プロパティ（resourcetype, displayname 等）の仕様を保持。
  • **XML 生成**: リソースの状態に基づき、適切な XML 形式でプロパティ値を生成・挿入。
  • **プロバイダ登録**: dav_core_liveprop_group を通じて、DAV エンジンへライブプロパティのフックを公開。
  • **書き込み制御**: プロパティの書き込み可否判定および検証ロジックの提供。

重要なシンボル

  • dav_core_liveprop_group: 本ファイルが管理するライブプロパティの仕様、名前空間、フックを統合した構造体。
  • dav_core_insert_prop: プロパティ値を XML 形式で apr_text_header に追加するメイン関数。
  • dav_core_props: DAV: 名前空間でサポートされるプロパティのリスト。
  • dav_core_hooks_liveprop: DAV エンジンが呼び出すライブプロパティ操作用の関数ポインタ群。

制御フローとデータフロー

1. **プロパティ検索**: dav_core_find_liveprop が呼び出されると、dav_do_find_liveprop を介して dav_core_liveprop_group から対象プロパティを特定します。 2. **値の挿入**: dav_core_insert_prop がリソース型（collection, version 等）を判定し、XML 文字列を構築します。特に resourcetype の場合、外部プロバイダからの拡張情報も統合されます。 3. **検証**: dav_core_patch_validate は、書き込み要求があった際に、その処理を「デッドプロパティ（データベース保存）」へ委譲するようフラグを立てます。

直接依存

  • mod_dav.h: DAV モジュールの主要なデータ構造とフック定義。
  • util_xml.h: XML 生成および解析ユーティリティ。
  • ap_provider.h: リソース型プロバイダの動的取得に使用。
  • httpd.h, apr_strings.h: Apache HTTP Server の基本 API および文字列操作。

関連コード

  • modules/dav/main/liveprop.c: ライブプロパティの基盤ロジックと検索アルゴリズム。
  • modules/dav/main/props.c: プロパティデータベース（デッドプロパティ）の操作。
  • modules/dav/main/mod_dav.c: モジュールの初期化とリクエスト処理の起点。

読者向けメモ

  • 本ファイルは DAV: 名前空間のプロパティのみを扱います。他の名前空間のプロパティは別のプロバイダによって処理されます。
  • resourcetype の生成ロジックは、リソースの属性（コレクションか、バージョン管理下にあるか等）に強く依存しているため、DAV のリソースモデルを理解する上で重要な箇所です。
  • dav_core_patch_validate が常に *defer_to_dead = 1 を設定している点に注意してください。これは、コアライブプロパティの書き込みが直接このファイルで完結せず、バックエンドのプロパティデータベースへ委ねられることを示しています。
