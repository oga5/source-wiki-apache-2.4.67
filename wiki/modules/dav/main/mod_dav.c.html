modules/dav/main/mod_dav.c

概要

mod_dav.c は Apache HTTP Server における WebDAV (RFC 4918) プロトコルの実装の中核となるモジュールです。リポジトリの実装から独立した抽象レイヤーを提供し、HTTP メソッドのディスパッチ、リクエストの検証、および XML レスポンスの生成を管理します。

モジュール内での役割

本ファイルは mod_dav のエントリーポイントであり、Apache のフック（handler, fixups）を登録して WebDAV リクエストを捕捉します。具体的なリポジトリ操作（ファイルシステムやバージョン管理システムなど）は、dav_provider インターフェースを介して外部モジュールに委譲します。

責務

  • **リクエストハンドリング**: WebDAV メソッド（PROPFIND, PUT, DELETE, LOCK 等）のディスパッチ。
  • **設定管理**: DAV ディレクティブによるプロバイダーの選択と、ディレクトリごとの設定（dav_dir_conf）の管理。
  • **レスポンス生成**: XML 形式のマルチステータスレスポンス（dav_send_multistatus）の構築。
  • **リソース解決**: dav_get_resource を用いたリクエスト URI からリポジトリリソースへの変換。
  • **フック管理**: ライブプロパティやレポート生成のための外部拡張ポイントの提供。

重要なシンボル

設定構造体

  • dav_dir_conf: ディレクトリごとの設定。プロバイダー名、ロックタイムアウト、DAV 有効フラグを保持。
  • dav_server_conf: サーバー全体の設定（現在は未使用）。

メソッドハンドラ

  • dav_handler: Apache のリクエストハンドラ。r->method_number に基づき各メソッド関数へ振り分ける。
  • dav_method_*: 各 WebDAV メソッド（dav_method_get, dav_method_put, dav_method_propfind 等）の実装。

ユーティリティ・ヘルパー

  • dav_get_provider: 現在のリクエストに対する dav_provider を取得。
  • dav_send_multistatus: WebDAV のマルチステータスレスポンスをストリーム出力する。
  • dav_handle_err: dav_error 構造体から適切な HTTP ステータスまたは XML エラーレスポンスを生成。

制御フローとデータフロー

1. **初期化**: register_hooks が Apache の handler および fixups フックを登録。 2. **リクエスト捕捉**: dav_fixups がリクエストを評価し、DAV が有効な場合に r->handler を dav-handler に設定。 3. **ディスパッチ**: dav_handler が呼び出され、メソッドに応じた dav_method_* 関数を実行。 4. **リソース解決**: 各メソッドは dav_get_resource を呼び出し、プロバイダー経由で dav_resource オブジェクトを取得。 5. **実行と応答**: プロバイダーのフック（repos, locks, vsn 等）を呼び出し、結果を dav_send_multistatus 等でクライアントへ送信。

読者向けメモ

  • **メモリ管理**: Apache の request_rec に紐付く pool を多用します。dav_send_multistatus 等ではサブプールを作成し、反復処理中のメモリ消費を抑制しています。
  • **エラー処理**: dav_error 構造体によるスタックベースのエラー管理が行われています。dav_handle_err はエラー発生時にリクエストを終了させるための標準的な出口です。
  • **未実装・浅い部分**: dav_server_conf は現在未使用です。また、一部のメソッド（BASELINE_CONTROL 等）はスタブ状態であり、将来の拡張や特定のプロバイダー実装に依存します。
  • **注意点**: dav_method_copymove 等の複雑なメソッドは、リソースのロック状態やバージョン管理状態に強く依存するため、プロバイダー側の実装との整合性が重要です。
