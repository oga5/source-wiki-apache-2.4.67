modules/dav/main/providers.c

概要

本ファイルは、Apache HTTP Server の DAV モジュールにおける「プロバイダー」の登録および検索メカニズムを抽象化したインターフェースを提供します。Apache の ap_provider API をラップし、DAV 固有の拡張機能やリソースタイプをプラグインとして管理するための基盤です。

モジュール内での役割

mod_dav のコア機能と、それを利用する外部モジュール（ファイルシステムバックエンドやカスタムプロパティハンドラなど）を疎結合にするための仲介役です。DAV 関連の機能拡張を動的に登録・取得する際の標準的な窓口として機能します。

責務

  • **プロバイダーの登録**: ap_provider API を介して、DAV 関連のプロバイダー（リポジトリ、オプション、リソースタイプ等）をグローバルなレジストリに登録する。
  • **プロバイダーの検索**: 名前を指定して、登録済みのプロバイダー実装を取得する。
  • **名前空間の管理**: DAV_PROVIDER_GROUP などの定数を用いて、DAV 関連のプロバイダーを他の Apache モジュールと分離して管理する。

重要なシンボル

  • dav_register_provider / dav_lookup_provider: DAV プロバイダーの汎用的な登録・取得関数。
  • dav_options_provider_register / dav_get_options_providers: OPTIONS メソッドの拡張プロバイダーを扱う関数。
  • dav_resource_type_provider_register / dav_get_resource_type_providers: DAV リソースタイプを定義するプロバイダーを扱う関数。
  • DAV_PROVIDER_GROUP: プロバイダー検索時に使用されるグループ識別子。

制御フローとデータフロー

本ファイルは純粋なラッパーとして動作します。 1. **登録時**: 呼び出し元が apr_pool_t とプロバイダー構造体を渡すと、ap_register_provider を呼び出し、Apache の内部ハッシュテーブルに格納します。 2. **検索時**: 呼び出し元が名前（文字列）を指定すると、ap_lookup_provider を通じて該当するプロバイダー構造体へのポインタを返します。 データフローはシンプルで、メモリ管理は apr_pool_t に依存しています。

直接依存

  • apr_pools.h: メモリプール管理。
  • apr_hash.h: プロバイダー管理の内部構造で使用。
  • ap_provider.h: Apache の汎用プロバイダー API。
  • mod_dav.h: DAV 固有の構造体定義。

関連コード

  • mod_dav.h: ここで定義されている dav_provider や dav_options_provider などの構造体定義が、本ファイルで扱うデータの型となります。
  • modules/dav/fs/repos.c: 本ファイルで定義されたインターフェースを利用して、ファイルシステムバックエンドがプロバイダーとして登録される典型的な例です。

読者向けメモ

  • 本ファイルは非常に薄いラッパーです。複雑なロジックは含まれていません。
  • プロバイダーのバージョン管理（ap_register_provider の第 4 引数）は、現状すべて "0" に固定されています。
  • 新しい DAV 拡張機能を追加する際は、本ファイルに新しい登録関数を追加するのではなく、既存の ap_provider API を活用する設計が推奨されます。
