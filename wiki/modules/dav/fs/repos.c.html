modules/dav/fs/repos.c

概要

repos.c は、Apache HTTP Server の mod_dav におけるファイルシステムベースのストレージプロバイダーの実装です。ローカルファイルシステム上のディレクトリやファイルを WebDAV リソースとして公開し、プロパティ管理、リソースのコピー/移動/削除、およびディレクトリ走査機能を提供します。

モジュール内での役割

mod_dav_fs モジュールの中心的なリポジトリフック実装です。repos.h で宣言されたインターフェースを具現化し、mod_dav コアに対してファイルシステム固有の操作を抽象化して提供します。

責務

  • **リソース管理**: ファイルシステムパスと WebDAV リソースの相互変換、およびリソースの存在確認。
  • **ファイル操作**: ストリーム（読み書き）の抽象化、ファイルのコピー・移動・削除の実行。
  • **プロパティ管理**: DAV: 名前空間およびカスタム名前空間（executable 属性など）のライブプロパティの定義と操作。
  • **ディレクトリ走査**: dav_fs_walk を通じた再帰的なリソース探索と、それに応じたコールバック実行。

重要なシンボル

リポジトリフック (dav_hooks_repository)

  • dav_hooks_repository_fs: リポジトリ操作のメインエントリーポイント。get_resource, open_stream, copy_resource, walk 等の関数ポインタを保持。

コンテキスト構造体

  • dav_resource_private: リソースの物理パス、apr_finfo_t、リクエスト情報を保持するプライベートデータ。
  • dav_fs_walker_context: ディレクトリ走査中に状態を維持するためのコンテキスト。
  • dav_stream: ファイル読み書きのためのストリーム状態（一時ファイルパスやクリーンアップ情報を含む）。

ライブプロパティ

  • dav_fs_liveprop_group: サポートするライブプロパティ（creationdate, getcontentlength 等）と名前空間の定義。
  • dav_fs_patch_*: executable プロパティの更新、検証、ロールバックを行う一連のフック関数。

制御フローとデータフロー

  • **リソース探索**: dav_fs_get_resource がリクエストからパスを構築し、dav_resource を初期化します。
  • **ディレクトリ走査**: dav_fs_walk が dav_fs_internal_walk を経由して dav_fs_walker を呼び出します。これは再帰的にディレクトリを探索し、各リソースに対して params->func を実行します。
  • **コピー/移動**: dav_fs_copymove_resource が走査を行い、各ファイルに対して dav_fs_copymove_file を実行します。プロパティの移動は dav_fs_copymoveset が担当し、DBM ファイルを操作します。
  • **ストリーム処理**: dav_fs_open_stream でファイルを開き、dav_fs_write_stream で書き込み、dav_fs_close_stream でコミット（一時ファイルからのリネーム）またはロールバック（一時ファイルの削除）を行います。

直接依存

  • apr_file_io.h, apr_buckets.h: ファイル I/O およびバケット操作。
  • mod_dav.h: WebDAV プロバイダーインターフェース。
  • repos.h: 本モジュールの公開ヘッダー。
  • httpd.h, http_protocol.h: Apache HTTP Server のリクエスト処理およびヘッダー操作。

  • apr_strings.h

読者向けメモ

  • **メモリ管理**: すべての操作は request_rec または dav_resource に関連付けられた apr_pool_t を使用します。明示的な free は不要ですが、プール寿命に注意が必要です。
  • **エラー処理**: dav_error 構造体を使用してエラーを伝播します。MAP_IO2HTTP マクロにより、APR のエラーコードを適切な HTTP ステータスコードに変換しています。
  • **一貫性**: MOVE や COPY 操作において、ファイルとプロパティ（DBM）の両方を更新する必要があるため、一部の操作で「不整合な状態」が発生するリスクがコメントで指摘されています。
  • **未実装/浅い部分**: DEBUG_GET_HANDLER が 0 に設定されており、標準的な GET ハンドリングは mod_dav コアに依存しています。また、シンボリックリンクの扱いは明示的にスキップされています。
