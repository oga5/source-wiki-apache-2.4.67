modules/dav/fs/lock.c

概要

本ファイルは、Apache HTTP Server の DAV モジュールにおけるファイルシステムベースのロック管理機能を実装しています。DAVLockDB ディレクティブで指定された DBM データベースを利用し、リソースに対する直接ロックおよび間接ロック（継承ロック）の永続化、取得、更新、削除を担います。

モジュール内での役割

mod_dav のロックプロバイダインターフェース（dav_hooks_locks）を実装するバックエンドです。repos.c が提供するリソース抽象化層と連携し、ファイルシステム上のリソースに対する WebDAV ロック状態を管理します。

責務

  • **ロックデータベースの管理**: DBM を用いたロック情報の読み書き、および遅延オープンによる効率的なリソース管理。
  • **ロックのシリアライズ**: dav_lock_discovery（直接ロック）および dav_lock_indirect（間接ロック）構造体のバイナリ形式への変換と保存。
  • **ロックnullリソースの管理**: 存在しないリソースに対するロック（lock-null）を追跡するための補助ファイル（.locknull）の操作。
  • **ロックのライフサイクル管理**: 有効期限切れロックの自動クリーンアップと、トークンベースのロック照合。

重要なシンボル

ロックデータベース構造

  • dav_lockdb_combined: dav_lockdb とプライベート情報（DBM ハンドル、パス）を統合した構造体。
  • dav_hooks_locks_fs: mod_dav に登録されるロックフックのメインエントリポイント。

ロックデータ構造

  • dav_lock_discovery: 直接ロックの情報を保持。有効期限、スコープ、所有者情報を含む。
  • dav_lock_indirect: 親コレクションから継承されたロックの参照。
  • dav_locktoken: UUID をラップしたロック識別子。

主要な操作関数

  • dav_fs_save_lock_record / dav_fs_load_lock_record: DBM へのロックリストの永続化と復元。
  • dav_fs_resolve: 間接ロックを辿り、対応する直接ロックを解決する。
  • dav_fs_remove_locknull_member: ロックnull状態の解消時に補助ファイルを更新する。

制御フローとデータフロー

1. **初期化**: dav_fs_open_lockdb が呼ばれ、dav_lockdb 構造体が初期化されます。実際の DBM オープンは dav_fs_really_open_lockdb により、最初のアクセス時に遅延実行されます。 2. **データ読み込み**: dav_fs_load_lock_record がキー（ファイルパス）に基づき DBM からデータをフェッチし、メモリ上のリンクドリスト（dav_lock_discovery / dav_lock_indirect）を構築します。この際、期限切れロックは自動的に除外されます。 3. **データ保存**: dav_fs_save_lock_record がリンクドリストをシリアライズし、DBM に書き戻します。 4. **間接ロック解決**: dav_fs_resolve は間接ロックのキーを辿り、再帰的に dav_fs_load_lock_record を呼び出して元の直接ロックを特定します。

読者向けメモ

  • **メモリ管理**: すべての動的メモリ割り当ては request_rec に紐付くプール（r->pool）で行われます。明示的な free は不要ですが、長時間のプロセスでは注意が必要です。
  • **ロックnull**: 存在しないファイルに対するロックは、ディレクトリ内の .locknull ファイルに記録されます。このファイルは dav_fs_load_locknull_list 等で管理されます。
  • **制約**: DAV_GETLOCKS_COMPLETE は未実装です。また、ロックデータベースの破損チェックは最小限であり、DBM の整合性は外部要因に依存します。
  • **設計上の注意**: dav_fs_save_lock_record は、ロックデータベースが読み取り専用で開かれている場合にエラーを返す設計になっています。
