modules/dav/fs/repos.h

概要

repos.h は、Apache HTTP Server の mod_dav モジュールにおけるファイルシステムベースのリポジトリプロバイダーのインターフェースを定義するヘッダーファイルです。DAV リソースとローカルファイルシステム上の実体とのマッピング、およびメタデータ管理のための抽象化を提供します。

モジュール内での役割

modules/dav/fs サブシステムの中核となるヘッダーであり、repos.c で実装されるリポジトリ操作の公開 API を定義します。DAV のリソース操作（プロパティ管理、ロック、ファイルアクセス）をファイルシステム上の操作に変換するためのブリッジとして機能します。

責務

  • **リソース管理**: DAV リソースからファイルシステム上のパスやディレクトリ構造を解決する。
  • **状態管理**: DAV 固有のメタデータ（ロック情報やプロパティ）を格納する .DAV ディレクトリおよび関連ファイルの管理。
  • **DBM インターフェース**: プロパティやロック情報を永続化するための DBM 操作のラッパーを提供。
  • **フックの公開**: mod_dav が必要とするライブプロパティやロック操作のフックを登録・提供する。

重要なシンボル

  • DAV_FS_STATE_DIR: DAV 関連のメタデータを格納するディレクトリ名 (.DAV)。
  • dav_fs_ensure_state_dir(): 必要なメタデータ用ディレクトリの存在を保証するユーティリティ。
  • dav_dbm_*: DBM データベースを操作するための抽象化関数群。
  • dav_fs_register(): モジュール初期化時にリポジトリプロバイダーを登録する関数。

制御フローとデータフロー

本ファイルは宣言のみを提供するため、具体的なフローは repos.c に依存します。

  • **リソース解決**: dav_fs_pathname() 等を通じて、dav_resource 構造体からファイルシステム上の絶対パスを導出します。
  • **メタデータアクセス**: dav_dbm_* 関数群を介して、リソースに関連付けられたプロパティやロック情報を DBM ファイルから読み書きします。
  • **初期化**: dav_fs_register() が呼び出されることで、ファイルシステムプロバイダーが mod_dav に認識されます。

直接依存

本ファイルは直接的な #include を持たず、純粋な宣言ヘッダーとして設計されています。ただし、使用される型（apr_pool_t, dav_resource, dav_error 等）は、mod_dav.h や APR ライブラリによって提供されることを前提としています。

読者向けメモ

  • 本ファイルは mod_dav のプロバイダーインターフェースを理解するための出発点です。
  • DAV_FS_STATE_DIR などの定数は、ファイルシステム上に DAV 固有の隠しディレクトリを作成する際の規約を定義しています。
  • DBM 関連の関数は、リポジトリとロックプロバイダーの両方から利用される共通基盤である点に注意してください。
