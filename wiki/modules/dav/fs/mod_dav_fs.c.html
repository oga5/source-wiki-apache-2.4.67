modules/dav/fs/mod_dav_fs.c

概要

mod_dav_fs.c は、Apache HTTP Server の WebDAV モジュール (mod_dav) に対し、ファイルシステムベースのバックエンド機能を提供するエントリーポイントです。WebDAV のロック管理用データベースのパス設定や、モジュールのライフサイクル管理を担います。

モジュール内での役割

mod_dav が提供する抽象インターフェースに対し、ファイルシステム固有の実装をバインドする役割を持ちます。他のファイル（repos.c, lock.c など）で定義された具体的なロジックを、Apache のモジュール構造に統合するための「接着剤」として機能します。

責務

  • **設定管理**: DAVLockDB ディレクティブを処理し、ロックデータベースのパスをサーバー設定として保持・マージする。
  • **モジュール登録**: Apache のモジュール構造体 (dav_fs_module) を定義し、フック関数を登録する。
  • **インターフェース提供**: dav_get_lockdb_path を通じて、他のファイルがロックDBのパスを参照できるようにする。

重要なシンボル

  • dav_fs_server_conf: サーバーごとの設定を保持する構造体。lockdb_path を管理します。
  • dav_fs_cmd_davlockdb: DAVLockDB ディレクティブのハンドラ。設定値を解析し、絶対パスに変換します。
  • dav_fs_module: Apache に認識されるモジュール定義構造体。
  • register_hooks: モジュールの初期化時に呼び出され、WebDAV 関連のフックを登録します。

制御フローとデータフロー

1. **設定読み込み**: Apache 起動時、DAVLockDB ディレクティブが検出されると dav_fs_cmd_davlockdb が呼ばれ、パスが dav_fs_server_conf に格納されます。 2. **初期化**: register_hooks が実行され、dav_fs_register を通じてファイルシステムバックエンドが mod_dav に登録されます。 3. **実行時**: WebDAV リクエスト処理中に dav_get_lockdb_path が呼び出され、設定されたパスに基づきロック操作が行われます。

直接依存

  • httpd.h, http_config.h: Apache モジュール基盤。
  • apr_strings.h: 文字列操作ユーティリティ。
  • mod_dav.h: WebDAV 抽象インターフェース。
  • repos.h: ファイルシステムバックエンドの内部定義。

関連コード

  • repos.c: ファイルシステムリポジトリの主要な実装。
  • lock.c: ロックデータベースの操作ロジック。
  • dbm.c: DBM を用いたプロパティ管理の実装。

読者向けメモ

  • 本ファイルは主に「設定」と「登録」に特化しています。WebDAV のリクエスト処理そのもの（ファイル読み書きやロックのセマンティクス）を理解したい場合は、repos.c や lock.c を先に参照してください。
  • DAVLockDB はサーバー単位の設定であり、ディレクトリごとのオーバーライドは行われません。設定の継承は dav_fs_merge_server_config で制御されています。
