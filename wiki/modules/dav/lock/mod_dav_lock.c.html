modules/dav/lock/mod_dav_lock.c

概要

mod_dav_lock.c は、Apache HTTP Server の DAV モジュール（mod_dav）に対して、ロックデータベースの管理機能を提供する汎用プロバイダーモジュールです。主に DAV ロックの永続化先となるデータベースパスの設定と、そのプロバイダーの登録を担います。

モジュール内での役割

mod_dav サブシステムにおける「ロック管理」のフロントエンドです。mod_dav 本体がロック操作を必要とする際、このモジュールが提供する dav-lock プロバイダーを介して、実際のロックデータベース（locks.c で実装）へアクセスします。

責務

  • **設定管理**: DAVGenericLockDB ディレクティブを定義し、ディレクトリごとのロックデータベースパスを保持する。
  • **プロバイダー登録**: dav-lock プロバイダーとして dav_hooks_locks_generic を mod_dav に公開する。
  • **設定の継承**: 親ディレクトリの設定を子ディレクトリが適切に継承・マージする仕組みの提供。

重要なシンボル

  • dav_lock_dir_conf: ロックデータベースのパスを保持するディレクトリ設定構造体。
  • dav_lock_cmd_davlockdb: DAVGenericLockDB ディレクティブのハンドラ。パスをサーバールート相対パスに変換して保存する。
  • dav_hooks_locks_generic: locks.c で定義されたロック操作のフック関数群。
  • register_hooks: モジュール初期化時に dav-lock プロバイダーを登録する関数。

制御フローとデータフロー

1. **設定読み込み**: Apache 起動時または設定再読み込み時に dav_lock_cmd_davlockdb が呼ばれ、設定ファイルからパスを読み込み dav_lock_dir_conf に格納します。 2. **プロバイダー登録**: register_hooks が実行され、ap_register_provider を通じて dav-lock プロバイダーが mod_dav に認識されます。 3. **実行時アクセス**: mod_dav がロック操作を行う際、dav_generic_get_lockdb_path を呼び出し、現在のリクエストコンテキストからロックデータベースのパスを取得します。

直接依存

  • httpd.h, http_config.h: Apache コア機能および設定フレームワーク。
  • ap_provider.h: プロバイダー登録用インターフェース。
  • mod_dav.h: DAV モジュール共通インターフェース。
  • locks.h: ロック機能の内部定義およびフック構造体。
  • apr_strings.h

関連コード

  • locks.c: このモジュールが提供するプロバイダーの実際のロジック（ロックの読み書き、データベース操作）が実装されています。
  • mod_dav.c: mod_dav 本体。ここで登録されたプロバイダーを利用してロック処理を委譲します。

読者向けメモ

  • 本ファイルは主に「設定」と「プロバイダーの登録」に特化しています。ロックの具体的なアルゴリズムやファイル I/O を追いたい場合は locks.c を参照してください。
  • DAVGenericLockDB で指定するパスは、サーバーの実行ユーザーが書き込み可能である必要があります。
  • dav_lock_merge_dir_config は、子ディレクトリで設定がない場合に親の設定をそのまま引き継ぐシンプルなマージロジックを採用しています。
