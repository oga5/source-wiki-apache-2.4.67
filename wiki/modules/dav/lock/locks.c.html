modules/dav/lock/locks.c

概要

本ファイルは、Apache HTTP Server の mod_dav における汎用的なロック管理実装を提供します。DBM（Database Manager）を利用して、リソースに対する直接ロックおよび間接ロックの永続化、検索、更新、削除を行うためのフック関数群を定義しています。

モジュール内での役割

mod_dav のロックプロバイダインターフェース（dav_hooks_locks）を実装するバックエンドです。mod_dav_lock.c がモジュールのエントリポイントとして機能するのに対し、本ファイルはロックデータベースの物理的な読み書きとデータ構造のシリアライズを担います。

責務

  • **DBM 管理**: ロックデータベースの遅延オープンおよびクローズ。
  • **データ永続化**: ロック情報をバイナリ形式で DBM にシリアライズ/デシリアライズ。
  • **ロック操作**: ロックの作成、検索、削除、およびタイムアウトによる期限切れロックの自動クリーンアップ。
  • **間接ロック解決**: 親コレクションから継承された間接ロックの追跡と解決。

重要なシンボル

データ構造

  • dav_lockdb_private: DBM ハンドルとリクエストコンテキストを保持する内部構造体。
  • dav_lock_discovery: 直接ロックの情報を保持するリスト構造。
  • dav_lock_indirect: 間接ロック（継承ロック）の情報を保持するリスト構造。
  • dav_locktoken: ロックを一意に識別するための UUID ラッパー。

フック実装 (dav_hooks_locks_generic)

  • dav_generic_open_lockdb: ロック DB を初期化し、dav_lockdb を構築。
  • dav_generic_get_locks: リソースに関連付けられた全ロックを取得。
  • dav_generic_append_locks: 新規ロックをデータベースに追加。
  • dav_generic_refresh_locks: 既存ロックのタイムアウトを更新。

内部ユーティリティ

  • dav_generic_save_lock_record: メモリ上のロックリストを DBM に書き込み。
  • dav_generic_load_lock_record: DBM からロックリストを読み込み、期限切れを排除。
  • dav_generic_resolve: 間接ロックが参照する親の直接ロックを解決。

制御フローとデータフロー

1. **初期化**: dav_generic_open_lockdb が呼ばれると、DBM ハンドルは即座に開かれず、opened フラグで管理されます。実際の I/O は dav_generic_really_open_lockdb が呼ばれた時に発生します。 2. **読み込み**: dav_generic_load_lock_record が DBM からデータを取得し、dav_lock_discovery または dav_lock_indirect のリストを構築します。この際、time(NULL) と比較して期限切れのロックはリストから除外されます。 3. **書き込み**: dav_generic_save_lock_record は、リストをバイナリバッファにシリアライズします。直接ロックは DAV_LOCK_DIRECT、間接ロックは DAV_LOCK_INDIRECT というプレフィックスを付与して連結します。 4. **解決**: 間接ロックの操作時、dav_generic_resolve が再帰的に親リソースのロックレコードを読み込み、トークンの一致を確認します。

読者向けメモ

  • **メモリ管理**: すべてのロック構造体は request_rec のプールから割り当てられます。リクエスト終了時に自動的に解放されます。
  • **データレイアウト**: DBM 内のデータは構造体パッキングを考慮したバイナリ列です。dav_size_direct 等の計算ロジックと、dav_generic_load_lock_record 内のオフセット計算は密結合しており、変更時は注意が必要です。
  • **制約**: DAV_GETLOCKS_COMPLETE は未実装です。また、ロック DB が読み取り専用で開かれている場合、期限切れロックの自動削除（need_save）はスキップされます。
  • **設計上の注意**: ### で始まるコメントは、将来的な拡張やリファクタリングの余地（特にエラーハンドリングの強化やメモリ効率の改善）を示唆しています。
