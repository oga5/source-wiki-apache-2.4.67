modules/ldap/util_ldap_cache_mgr.c

概要

util_ldap_cache_mgr.c は、Apache HTTP Server の LDAP モジュールにおけるキャッシュ管理の基盤層です。共有メモリ（SHM）またはヒープメモリを利用した汎用的なハッシュテーブルの実装と、LDAP 固有のキャッシュ操作（検索、比較、DN 比較）のライフサイクル管理を提供します。

モジュール内での役割

util_ldap_cache.c が高レベルな LDAP キャッシュロジックを扱うのに対し、本ファイルはメモリ管理、ハッシュ計算、キャッシュのパージ（掃除）、および統計情報の可視化といった「インフラストラクチャ」の役割を担います。

責務

  • **メモリ管理**: apr_rmm（共有メモリ）または標準の malloc/calloc を抽象化したアロケータの提供。
  • **キャッシュ構造の管理**: キャッシュの作成、破棄、およびエントリの挿入・削除・検索。
  • **キャッシュの健全性維持**: キャッシュが満杯になった際のパージ処理（TTL 期限切れエントリの削除）。
  • **統計と可視化**: キャッシュのヒット率やチェーン長などの統計計算と、HTML 形式でのダンプ出力。

重要なシンボル

メモリ管理

  • util_ald_alloc, util_ald_free, util_ald_strdup: キャッシュ専用のアロケータ。共有メモリ利用時は apr_rmm を介し、それ以外は標準ライブラリを使用する。

キャッシュライフサイクル

  • util_ald_create_cache: キャッシュ構造体の初期化とハッシュテーブルの割り当て。
  • util_ald_destroy_cache: キャッシュ内の全エントリを解放し、構造体自体を破棄。
  • util_ald_create_caches: URL ごとの検索・比較・DN 比較キャッシュをセットで生成。

キャッシュ操作

  • util_ald_cache_fetch: ハッシュテーブルからエントリを検索。
  • util_ald_cache_insert: キャッシュへの挿入。満杯時は util_ald_cache_purge を呼び出す。
  • util_ald_cache_remove: 特定エントリの削除。

ユーティリティ

  • util_ald_hash_string: 可変長引数を受け取る文字列ハッシュ計算関数。
  • util_ald_cache_purge: キャッシュのパージ処理。marktime を基準に古いエントリを削除。

制御フローとデータフロー

1. **挿入フロー**: util_ald_cache_insert が呼ばれると、まず numentries を確認。上限に達していれば util_ald_cache_purge を実行。その後、copyfunc でペイロードを複製し、ハッシュ値を計算してリストの先頭に挿入する。 2. **パージフロー**: util_ald_cache_purge は全バケットを走査し、add_time が marktime より古いノードを順次解放する。 3. **メモリレイアウト**: util_ald_cache_t はハッシュテーブル（nodes）を保持し、各ノードは util_cache_node_t として連結リストを形成する。共有メモリ使用時は、これらすべてが apr_rmm 管理下のオフセットとして配置される。

関連コード

  • util_ldap.c: モジュールのメインエントリポイント。

読者向けメモ

  • **メモリ所有権**: キャッシュに挿入されるデータは、copyfunc によってキャッシュ専用メモリ領域に複製されます。呼び出し元は元のデータを解放して構いません。
  • **共有メモリの制約**: APR_HAS_SHARED_MEMORY が有効な場合、すべてのデータは apr_rmm を通じて割り当てられる必要があります。ポインタの直接代入は共有メモリ境界を越える可能性があるため厳禁です。
  • **パージの挙動**: パージは O(n) の単純な走査です。キャッシュサイズが非常に大きい場合、パージ実行時のレイテンシに注意が必要です。
  • **未実装/浅い部分**: util_ald_cache_insert 内のコメントにある通り、現在のキャッシュ管理はサイズ制限に達した際の挙動がやや強引（強制パージ）であり、将来的なリファクタリングの余地が残されています。
