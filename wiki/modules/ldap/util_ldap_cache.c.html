modules/ldap/util_ldap_cache.c

概要

本ファイルは、Apache HTTP Server の LDAP モジュールにおけるキャッシュノードの管理ロジックを実装しています。LDAP 検索、比較、DN 比較といった各操作に対応するキャッシュノードのハッシュ計算、比較、メモリ確保・解放、およびデバッグ表示用のコールバック関数を定義します。

モジュール内での役割

util_ldap_cache.h で定義された汎用キャッシュ構造体（util_ald_cache_t）に対し、具体的なデータ型（URL、検索、比較、DN 比較）ごとの振る舞いを注入する役割を担います。

責務

  • **ノード操作の抽象化**: 各キャッシュノード型に対して、ハッシュ関数、比較関数、コピー関数、解放関数、表示関数を提供。
  • **キャッシュのライフサイクル管理**: util_ldap_cache_init を通じたキャッシュの初期化と、共有メモリ（SHM/RMM）のセットアップ。
  • **リソースのクリーンアップ**: モジュール終了時のキャッシュ破棄と共有メモリの解放。

重要なシンボル

ノード操作ファミリー

各データ型（URL, Search, Compare, DN Compare）ごとに以下の関数群が実装されています。

  • util_ldap_[type]_node_hash: キャッシュキーのハッシュ値を算出。
  • util_ldap_[type]_node_compare: キャッシュキーの等価性判定。
  • util_ldap_[type]_node_copy: キャッシュへの挿入時にメモリを確保し、データを複製。
  • util_ldap_[type]_node_free: キャッシュから削除する際、ノード内の動的確保メモリを再帰的に解放。
  • util_ldap_[type]_node_display: キャッシュの状態を HTML テーブル形式で出力。

キャッシュ管理

  • util_ldap_cache_init: キャッシュの初期化。共有メモリが有効な場合、apr_shm および apr_rmm を用いてメモリ領域を確保し、キャッシュ構造体を構築。
  • util_ldap_cache_module_kill: プールクリーンアップハンドラ。キャッシュと共有メモリを安全に破棄。

制御フローとデータフロー

1. **初期化**: util_ldap_cache_init が呼ばれると、共有メモリ領域が確保され、util_ald_create_cache に各ノード操作関数が関数ポインタとして渡されます。 2. **データ挿入**: キャッシュへの挿入時、_copy 関数が呼ばれ、util_ald_alloc を介してキャッシュ専用メモリ領域にデータが複製されます。 3. **データ削除**: キャッシュの TTL 切れや容量制限によるパージ時、_free 関数が呼ばれ、ノード内の文字列やサブグループリストが util_ald_free で解放されます。 4. **表示**: 管理画面等でキャッシュ内容を表示する際、_display 関数が request_rec を通じて HTML を生成します。

直接依存

  • httpd.h: Apache コア機能（request_rec, ap_rprintf 等）。
  • util_ldap.h: LDAP モジュール共通定義。
  • util_ldap_cache.h: キャッシュ構造体定義。
  • apr_strings.h: 文字列操作ユーティリティ。

関連コード

  • util_ldap_cache_mgr.c: 本ファイルで定義された関数を利用して、キャッシュの管理や統計情報の集計を行う上位層。
  • util_ldap.c: LDAP 検索や比較のメインロジック。キャッシュのヒット判定や挿入をトリガーする。

読者向けメモ

  • **メモリ管理**: 本ファイル内の _copy および _free 関数は、標準の malloc/free ではなく、util_ald_alloc/util_ald_free を使用しています。これは共有メモリ（apr_rmm）上の領域を操作するためであり、このルールを破るとメモリ破壊やセグメンテーション違反を引き起こします。
  • **共有メモリ**: APR_HAS_SHARED_MEMORY が定義されている場合、キャッシュはプロセス間で共有されます。そのため、ノード内のポインタは絶対アドレスではなく、共有メモリ内のオフセットとして管理される必要があります（util_ald_ 系関数がこれを隠蔽しています）。
  • **省略された詳細**: 本ファイルはノードの「振る舞い」に特化しており、キャッシュの衝突解決アルゴリズムや、共有メモリの具体的なアロケーションロジック（util_ald_ 内部）については、util_ldap_cache_mgr.c 側に委譲されています。
