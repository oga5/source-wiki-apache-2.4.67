modules/ldap/util_ldap_cache.h

概要

本ヘッダーファイルは、Apache HTTP Server の LDAP モジュールにおけるキャッシュ機構のデータ構造とインターフェースを定義します。LDAP 検索、比較、DN 比較の結果をメモリ上に保持し、バックエンド LDAP サーバーへの負荷を軽減するための基盤を提供します。

モジュール内での役割

modules/ldap サブシステムにおけるキャッシュ層の定義部です。util_ldap_cache.c（ノード操作の実装）および util_ldap_cache_mgr.c（キャッシュ管理・メモリ管理の実装）と密接に連携し、LDAP 関連のデータ永続化と検索効率化を担います。

責務

  • **キャッシュ構造の定義**: URL ごとのキャッシュノードおよび、検索・比較結果を保持するデータ構造の定義。
  • **抽象化されたキャッシュ操作**: ハッシュ、比較、コピー、解放、表示を行う関数ポインタ群の定義。
  • **メモリ管理の抽象化**: 共有メモリ（SHM）とプロセス内メモリの双方を透過的に扱うためのアロケータインターフェースの提供。
  • **統計情報の保持**: キャッシュのヒット率、パージ回数、TTL 管理などの運用メトリクスの保持。

重要なシンボル

キャッシュ管理構造体

  • util_ald_cache_t: キャッシュのメタデータと統計、および操作関数ポインタを保持するメイン構造体。
  • util_cache_node_t: キャッシュ内の個別のエントリを管理するリンクリスト構造体。

キャッシュノード型

  • util_url_node_t: LDAP URL をキーとし、検索・比較・DN 比較の各キャッシュを保持するルートノード。
  • util_search_node_t: ユーザー名ベースの検索結果（DN、パスワード、属性値）を保持。
  • util_compare_node_t: DN、属性、値をキーとした比較結果を保持。
  • util_dn_compare_node_t: DN 同士の比較結果を保持。

操作関数群

  • util_ald_cache_fetch/insert/remove: キャッシュに対する基本的な CRUD 操作。
  • util_ald_alloc/free/strdup: キャッシュ専用のメモリ管理関数（SHM 対応）。

制御フローとデータフロー

1. **初期化**: util_ald_create_cache により、特定のデータ型に対するハッシュ関数や比較関数を登録したキャッシュインスタンスが生成されます。 2. **検索**: util_ald_cache_fetch が呼び出されると、登録された hash 関数でインデックスを特定し、compare 関数でペイロードを照合します。 3. **挿入**: キャッシュミス時、copy 関数を介して新しいノードがキャッシュメモリ領域に複製・格納されます。 4. **パージ**: キャッシュが fullmark（3/4 容量）に達すると、util_ald_cache_purge がトリガーされ、TTL 期限切れのノードが解放されます。

直接依存

  • util_ldap.h: モジュール共通の型定義（util_ldap_state_t 等）を参照。
  • apr_shm.h / apr_rmm.h (間接的): APR_HAS_SHARED_MEMORY が有効な場合、共有メモリ管理に使用。

関連コード

  • util_ldap_cache.c: 各ノード型（URL, Search, Compare, DN Compare）の具体的なハッシュ・比較・コピー・解放ロジックの実装。
  • util_ldap_cache_mgr.c: キャッシュのライフサイクル管理、メモリ割り当て、統計表示の実装。

読者向けメモ

  • **メモリ所有権**: キャッシュ内のデータは、util_ald_alloc を使用してキャッシュ専用のメモリ領域（共有メモリまたはヒープ）に確保されます。util_ald_free を通じて明示的に解放する必要があります。
  • **共有メモリ**: APR_HAS_SHARED_MEMORY が定義されている場合、キャッシュはプロセス間で共有されます。この際、ポインタの直接参照ではなく、apr_rmm（Relocatable Memory Management）を介したオフセット管理が行われる点に注意してください。
  • **制約**: util_url_node_t は、LDAP URL ごとに独立したキャッシュインスタンスを保持する階層構造をとっています。
  • **省略された詳細**: 本ファイルはインターフェース定義に特化しており、各ノードのハッシュアルゴリズムの具体的な実装や、共有メモリのロック制御の詳細については util_ldap_cache_mgr.c を参照してください。
