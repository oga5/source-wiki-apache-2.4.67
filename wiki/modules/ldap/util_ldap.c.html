modules/ldap/util_ldap.c

概要

util_ldap.c は、Apache HTTP Server における LDAP 接続管理、キャッシュ制御、および認証・認可操作のバックエンド機能を提供する中心的なソースファイルです。LDAP SDK（OpenLDAP, Novell, Solaris 等）の差異を抽象化し、接続プールと共有メモリキャッシュを統合して LDAP 操作のパフォーマンスを最適化します。

モジュール内での役割

modules/ldap サブシステムの基盤であり、util_ldap.h で定義されたインターフェースを実装します。他のモジュール（mod_authnz_ldap など）から APR_REGISTER_OPTIONAL_FN を通じて呼び出される API を提供し、LDAP サーバーとの通信を仲介します。

責務

  • **接続管理**: LDAP 接続のライフサイクル（初期化、バインド、再利用、切断）の管理。
  • **キャッシュ管理**: 検索結果、DN 比較、サブグループ情報の共有メモリキャッシュへの読み書き。
  • **設定管理**: httpd.conf で指定された LDAP 関連ディレクティブの解析とサーバー状態への反映。
  • **SDK 抽象化**: 異なる LDAP SDK 間の API 差異（特に SSL/TLS 設定や再バインド処理）の吸収。

重要なシンボル

接続管理

  • uldap_connection_find: キャッシュから既存の接続を検索、または新規作成する。
  • uldap_connection_open: LDAP サーバーへの接続とバインドを実行する。
  • uldap_connection_unbind: 接続を破棄し、リソースを解放する。

キャッシュ操作

  • uldap_cache_compare: LDAP 比較操作を実行し、結果をキャッシュする。
  • uldap_cache_checkuserid: ユーザー認証のための検索と結果のキャッシュ。
  • uldap_cache_check_subgroups: ネストされたグループの再帰的な検索とキャッシュ。

設定・フック

  • ldap_module: モジュールのメイン構造体。
  • util_ldap_cmds: 設定ディレクティブの定義テーブル。
  • util_ldap_register_hooks: Apache フック（post_config, handler 等）の登録。

制御フローとデータフロー

1. **接続取得**: uldap_connection_find が呼び出されると、スレッドセーフなミューテックスで保護されたリストから、条件（ホスト、ポート、証明書等）に一致する接続を探します。 2. **操作実行**: uldap_cache_compare 等の関数は、まず共有メモリキャッシュを検索します。キャッシュミスが発生した場合、uldap_connection_open を経由して LDAP サーバーへリクエストを送信します。 3. **再試行ロジック**: サーバーダウンやタイムアウト発生時、st->retries 回数まで再接続を試みるループ構造を持ちます。 4. **キャッシュ更新**: 操作成功後、結果は util_ald_cache_insert を通じて共有メモリに書き込まれます。

読者向けメモ

  • **メモリ管理**: 接続構造体は apr_pool_t で管理されます。接続の再利用時には must_rebind フラグを監視し、認証情報が変更された場合に適切に再バインドを行う必要があります。
  • **スレッド安全性**: st->mutex および l->lock を使用して、マルチスレッド環境での接続リストと個々の接続状態を保護しています。
  • **共有メモリ**: キャッシュは util_ald_cache を介して共有メモリ上に構築されます。post_config フックで初期化され、子プロセス間で共有されます。
  • **省略された詳細**: 本ファイルは非常に大規模であるため、個別の SDK 特有の ifdef 分岐（OpenLDAP vs Novell 等）の詳細な網羅は省略しています。これらは主に uldap_connection_init 内の SSL/TLS 設定に集中しています。
