modules/lua/lua_vmprep.h

概要

lua_vmprep.h は、Apache HTTP Server 上で動作する Lua 仮想マシン（VM）のライフサイクル管理と構成定義を担うヘッダーファイルです。Lua スクリプトの実行環境を構築するための仕様定義と、VM インスタンスの取得・初期化インターフェースを提供します。

モジュール内での役割

mod_lua モジュールにおいて、リクエストや接続、サーバー単位で Lua VM をどのように生成・保持するかを規定する基盤層です。mod_lua.c がリクエストを処理する際、このヘッダーで定義された仕様に基づいて VM を準備します。

責務

  • **VM 仕様の定義**: スコープ（リクエスト、スレッド、サーバー等）やキャッシュ戦略、パッケージパスを含む ap_lua_vm_spec 構造体の提供。
  • **ハンドラ定義**: マップされた Lua ハンドラやフィルタハンドラの仕様定義。
  • **VM 管理インターフェース**: ライフサイクルプールに基づいた lua_State の取得および初期化。
  • **同期制御**: マルチスレッド環境下での VM 共有を保護するためのミューテックス初期化。

重要なシンボル

  • ap_lua_vm_spec: Lua VM の生成パラメータを保持する主要構造体。
  • ap_lua_get_lua_state(): 指定された仕様に基づき、既存の VM を再利用または新規作成して返す関数。
  • AP_LUA_SCOPE_*: VM の生存期間を定義する定数群（REQUEST, CONN, SERVER など）。
  • AP_LUA_CACHE_*: スクリプトのキャッシュ戦略を定義する定数群。

制御フローとデータフロー

1. **構成**: ap_lua_vm_spec にスクリプトパスやスコープを設定します。 2. **取得**: ap_lua_get_lua_state() が呼び出されると、指定されたプールと仕様に基づき、キャッシュされた lua_State を検索します。 3. **初期化**: 存在しない場合、新しい VM を生成し、必要に応じて ap_lua_state_open_callback を実行して環境を構築します。 4. **同期**: マルチスレッド環境では、ap_lua_init_mutex() を通じて初期化されたミューテックスにより、VM へのアクセスが保護されます。

関連コード

読者向けメモ

  • 本ファイルは VM の「仕様」を定義するものであり、実際の実行ロジックは lua_vmprep.c にあります。
  • ap_lua_get_lua_state はスレッドセーフではないため、呼び出し側で適切なロック管理を行う必要があります。
  • bytecode フィールドを使用すると、ファイルシステムから読み込む代わりにメモリ上のバイトコードから直接 VM をロード可能です。
