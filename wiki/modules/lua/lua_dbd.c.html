modules/lua/lua_dbd.c

概要

Apache HTTP Server の mod_lua において、データベース接続（DBD: Database Driver）機能を提供するモジュールです。Lua スクリプトから APR (Apache Portable Runtime) の DBD フレームワークを介して、SQL クエリの実行、結果の取得、プリペアドステートメントの管理を可能にします。

モジュール内での役割

mod_lua の拡張機能の一部として、Lua VM 内のオブジェクトと Apache の DBD インフラストラクチャを橋渡しします。lua_dbd.h で定義された構造体を実装し、Lua のテーブルをデータベースハンドルや結果セットとしてラップします。

責務

  • **接続管理**: mod_dbd または直接的な APR DBD ドライバを用いたデータベース接続の取得と解放。
  • **クエリ実行**: SQL クエリの実行および結果セットの Lua テーブルへの変換。
  • **プリペアドステートメント**: SQL 文の事前コンパイルと、パラメータのバインド実行。
  • **リソース管理**: Lua のガベージコレクション (__gc) を利用したデータベース接続の自動クローズ。

重要なシンボル

接続・ライフサイクル管理

  • lua_db_acquire: Lua から呼び出される接続開始関数。mod_dbd または指定ドライバを使用。
  • lua_db_close: 接続を明示的に閉じる。
  • lua_db_gc: Lua オブジェクト破棄時に呼ばれるクリーンアップ関数。
  • lua_db_active: 接続の生存確認。

クエリ・データ操作

  • lua_db_query: 非選択系クエリ（INSERT/UPDATE等）の実行。
  • lua_db_select: 選択系クエリの実行と結果セットの生成。
  • lua_db_get_row: 結果セットから行データを取得（イテレータ的役割）。
  • lua_db_escape: SQL インジェクション防止のための文字列エスケープ。

プリペアドステートメント

  • lua_db_prepare: SQL 文をコンパイルしてステートメントオブジェクトを作成。
  • lua_db_prepared: 既存の DBDPrepareSQL 定義済みステートメントを取得。
  • lua_db_prepared_select / lua_db_prepared_query: バインド変数を用いた実行。

制御フローとデータフロー

1. **初期化**: lua_db_acquire が request_rec からコンテキストを取得し、lua_push_db_handle で Lua テーブルを生成。このテーブルには __gc メタメソッドが設定され、接続の生存期間を管理します。 2. **クエリ実行**: lua_db_select 等が呼ばれると、lua_get_db_handle で内部の lua_db_handle を取り出し、APR DBD API を呼び出します。 3. **結果処理**: lua_db_select は結果セットを lua_db_result_set 構造体としてラップし、__call メタメソッドに lua_db_get_row を割り当てます。これにより、Lua 側で resultset(row_no) のような呼び出しが可能になります。 4. **エラーハンドリング**: APR DBD から返されるエラーコードを apr_dbd_error で文字列化し、Lua の nil とエラーメッセージとして返します。

直接依存

  • mod_lua.h: mod_lua の基本定義。
  • lua_dbd.h: 共有データ構造と関数プロトタイプ。
  • apr_dbd.h / mod_dbd.h: APR のデータベース抽象化レイヤー。

関連コード

  • modules/database/mod_dbd.c: mod_lua が利用する Apache 側の DBD 管理モジュール。
  • modules/lua/mod_lua.c: mod_lua のメインエントリポイント。

読者向けメモ

  • **メモリ管理**: lua_db_handle は apr_pool_t を保持しており、接続が閉じられるかガベージコレクションされる際にプールが破棄されます。
  • **制約**: lua_db_get_row において、alpha フラグが真の場合、カラム名によるアクセスが可能ですが、それ以外はインデックスベースとなります。
  • **注意点**: mod_dbd 経由の接続は Apache のライフサイクルに依存するため、Lua 側で close を呼んでも即座に物理接続が切断されない場合があります。
  • **省略箇所**: プリペアドステートメントのパラメータバインドにおける型変換の詳細は、APR DBD の実装に依存するため、本ファイルでは単純な文字列変換 (lua_tostring) に留めています。
