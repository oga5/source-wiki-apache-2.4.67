modules/lua/lua_config.h

概要

Apache HTTP Server の mod_lua モジュールにおける設定管理インターフェースを定義するヘッダーファイルです。Lua スクリプトから Apache の設定を操作・拡張するためのブリッジ機能を提供します。

モジュール内での役割

mod_lua の設定処理において、Lua 側から呼び出される設定関数や、ハンドラマッピングの定義を仲介します。lua_config.c と対になっており、本ファイルは公開 API の宣言を担います。

責務

  • Lua インタープリタに対して、Apache 設定操作用のモジュールをロードする関数の提供。
  • URI パターンと Lua スクリプトの関数を紐付ける「ハンドラマッピング」の登録機能の定義。

重要なシンボル

  • ap_lua_load_config_lmodule(lua_State *L): Lua 環境に Apache 設定操作用の関数群を登録する初期化関数。
  • ap_lua_map_handler(...): 特定の URI パターンに対して Lua ハンドラを割り当てるための内部 API。

制御フローとデータフロー

本ファイル自体は宣言のみですが、lua_config.c を通じて以下のフローをサポートします： 1. **初期化**: ap_lua_load_config_lmodule が呼ばれ、Lua 側から ap_lua_dir_cfg 等の構造体へアクセス可能になる。 2. **マッピング**: Lua スクリプト内で定義されたハンドラ情報が ap_lua_map_handler を経由して ap_lua_dir_cfg 構造体内の配列に格納される。

直接依存

  • mod_lua.h: mod_lua の主要なデータ構造（ap_lua_dir_cfg 等）および共通定義へのアクセスに必須。

関連コード

  • modules/lua/lua_config.c: 本ヘッダーで宣言された関数の実装。Lua スタック操作や apr_pool を用いたメモリ管理の詳細が含まれます。
  • modules/lua/mod_lua.c: モジュールのメインエントリーポイント。設定の読み込み時に本ヘッダーの関数が利用されます。

読者向けメモ

  • 本ファイルは _APL_CONFIG_H_ マクロによって多重インクルードを防いでいます。
  • ap_lua_map_handler の引数にある scope は、Lua スクリプトの実行ライフサイクル（once, request, conn, thread）を決定する重要なパラメータです。
  • Lua 側から Apache の設定を操作する際は、lua_config.c 内の check_dir_config 等のヘルパー関数がどのように Lua のユーザーデータ（udata）をアンボックスしているかを確認すると理解が深まります。
