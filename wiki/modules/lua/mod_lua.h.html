modules/lua/mod_lua.h

概要

mod_lua.h は、Apache HTTP Server の Lua モジュール (mod_lua) における中心的なヘッダーファイルです。モジュール全体で共有されるデータ構造、型定義、および Lua C API との互換性レイヤーを提供します。

モジュール内での役割

mod_lua の各コンポーネント（設定管理、リクエスト処理、VM 管理）が共通して参照する定義を保持しています。他のソースファイルがこのヘッダーをインクルードすることで、Apache のリクエスト処理と Lua VM の橋渡しに必要な構造体やマクロにアクセスします。

責務

  • **API 互換性:** Lua 5.1 から 5.4 までのバージョン差異を吸収するマクロ定義。
  • **データ構造の定義:** ディレクトリ設定 (ap_lua_dir_cfg) やリクエスト設定 (ap_lua_request_cfg) など、モジュールの状態管理に必要な構造体の定義。
  • **エクスポート制御:** プラットフォームごとのシンボル公開用マクロ (AP_LUA_DECLARE) の提供。
  • **フック定義:** mod_lua が提供する外部フックの宣言。

重要なシンボル

  • ap_lua_dir_cfg: ディレクトリ単位の設定（Lua スクリプトのパス、VM スコープ、キャッシュ設定など）を保持する構造体。
  • ap_lua_request_cfg: リクエスト単位で保持される Lua VM の詳細情報。
  • lua_boxpointer / lua_unboxpointer: C ポインタを Lua の userdata として安全にラップ・アンラップするためのマクロ。
  • AP_LUA_DECLARE: モジュール外から利用可能な関数を宣言するためのエクスポートマクロ。

制御フローとデータフロー

本ファイルは定義のみを行うため、直接的な制御フローは持ちません。ただし、ここで定義された ap_lua_dir_cfg などの構造体は、mod_lua.c や lua_config.c において設定の読み込みや VM の初期化フローの基盤として利用されます。

読者向けメモ

  • **バージョン互換性:** LUA_VERSION_NUM を用いたマクロ定義は、異なる Lua バージョン間でのビルドを可能にするための重要なハックです。コードを追う際は、これらのマクロがどのバージョンを対象にしているか注意してください。
  • **ポインタの扱い:** lua_boxpointer は、C のポインタを Lua 側で保持する際の標準的な手法です。userdata のライフサイクル管理には特に注意が必要です。
  • **設定構造体:** ap_lua_dir_cfg は非常に多くのフィールドを持つため、設定の継承 (ap_lua_inherit_t) の仕組みを理解する際にこの構造体の定義をまず確認することをお勧めします。
