modules/lua/lua_vmprep.c

概要

lua_vmprep.c は、Apache HTTP Server 内で動作する Lua 仮想マシン（VM）のライフサイクル管理、初期化、および設定を行う中核的な実装ファイルです。Apache のメモリプール（apr_pool_t）と Lua VM の状態を統合し、リクエストやサーバーのスコープに応じた VM の生成・キャッシュ・破棄を制御します。

モジュール内での役割

mod_lua のインフラ層に位置し、他の Lua 関連モジュール（lua_request.c, lua_dbd.c 等）が利用する lua_State を提供します。特に、Apache のマルチスレッド環境における VM のスレッドセーフな共有や、設定ファイルで定義された VM 仕様（ap_lua_vm_spec）に基づく VM の構築を担います。

責務

  • **VM ライフサイクル管理**: apr_pool_t のクリーンアップハンドラを利用した lua_State の自動破棄。
  • **VM 構築と初期化**: luaL_newstate による生成、標準ライブラリのロード、および package.path/cpath の動的調整。
  • **Apache 統合**: apache2 Lua モジュールのロードと、HTTP ステータスコード等の定数定義の注入。
  • **キャッシュ戦略**: AP_LUA_CACHE_STAT 等のポリシーに基づく、ファイル変更検知と VM の再ロード制御。
  • **同期制御**: マルチスレッド環境下での lua_ivm_mutex 等を用いた排他制御。

重要なシンボル

VM 構築・管理

  • ap_lua_get_lua_state: VM を取得するメインエントリポイント。キャッシュの確認、必要に応じた vm_construct の呼び出しを行う。
  • vm_construct: lua_State を新規作成し、パス設定や初期スクリプト実行を行う低レベル関数。
  • server_vm_construct: サーバーレベルの VM を構築し、リソースリスト（apr_reslist）に登録するためのラッパー。

Apache 統合

  • ap_lua_load_apache2_lmodule: Lua VM 内に apache2 テーブルを作成し、HTTP 定数（OK, DECLINED 等）を登録する。
  • makeintegerfield: Lua スタック上に整数定数を定義するためのマクロ。

同期・クリーンアップ

  • ap_lua_init_mutex: グローバルな Lua ミューテックスを初期化する。
  • cleanup_lua / server_cleanup_lua: プール破棄時に lua_close を呼び出すためのコールバック。

制御フローとデータフロー

1. **VM 取得**: ap_lua_get_lua_state が呼び出されると、スコープ（AP_LUA_SCOPE_SERVER 等）に応じてキャッシュを検索。 2. **構築**: キャッシュミス時は vm_construct が呼ばれ、luaL_newstate で VM を生成。munge_path で Lua の検索パスを Apache の設定に合わせて修正。 3. **初期化**: spec->cb が指定されていれば、ユーザー定義の初期化コールバックを実行。 4. **ロード**: bytecode があれば直接ロードし、なければ luaL_loadfile でファイルを読み込む。 5. **キャッシュ管理**: codecache ポリシーに従い、apr_stat でファイルの更新日時を確認し、必要に応じて luaL_loadfile を再実行する。

読者向けメモ

  • **メモリ所有権**: lua_State のライフサイクルは、apr_pool_t のクリーンアップハンドラに依存しています。プールが破棄されると自動的に lua_close が呼ばれます。
  • **排他制御**: AP_LUA_SCOPE_SERVER スコープでは apr_reslist を使用して VM をプールしますが、これには ap_lua_mutex による排他制御が伴います。
  • **未実装・省略**: pstack_dump 関数は #if 0 で無効化されており、デバッグ目的で残されています。また、HTTP ステータスコードの定数定義は非常に多いため、一部がコメントアウトされています。
  • **注意点**: munge_path は Lua の package.path を文字列操作で書き換えるため、パスの形式が期待通りでない場合にエラーが発生する可能性があります。
