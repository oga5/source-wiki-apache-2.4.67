modules/lua/lua_apr.c

概要

lua_apr.c は、Apache Portable Runtime (APR) のテーブル構造を Lua スクリプトから透過的に操作可能にするためのブリッジ層です。HTTP リクエストやレスポンスのヘッダー、環境変数などを Lua のテーブルとして扱うためのインターフェースを提供します。

モジュール内での役割

mod_lua において、C 言語側のデータ構造（apr_table_t）と Lua VM 側のオブジェクトを接続する役割を担います。lua_apr.h で定義されたインターフェースを実装し、Lua スクリプトが Apache の内部テーブルを安全かつ直感的に読み書きできるようにします。

責務

  • **データバインディング**: apr_table_t をラップした req_table_t を Lua のユーザーデータとして公開する。
  • **メタテーブル管理**: Lua 上で Apr.Table 型として振る舞うためのメタテーブル定義とメソッド登録。
  • **入力バリデーション**: set 操作時に改行コードの混入を検知・置換し、HTTP ヘッダーインジェクション等のリスクを軽減する。

重要なシンボル

  • ap_lua_check_apr_table: Lua スタック上の値が Apr.Table であるか検証し、内部ポインタを取り出す。
  • ap_lua_push_apr_table: req_table_t を Lua のユーザーデータとしてスタックにプッシュする。
  • lua_table_methods: Lua から呼び出される get/set メソッドの定義テーブル。
  • ap_lua_init: Lua VM 初期化時に Apr.Table メタテーブルを登録する。

制御フローとデータフロー

1. **初期化**: ap_lua_init が呼ばれ、Lua VM に Apr.Table メタテーブルが登録されます。ここで __index と __newindex が get/set メソッドにマッピングされます。 2. **アクセス**: Lua スクリプトからテーブル操作が行われると、lua_table_get または lua_table_set が起動します。 3. **変換**: ap_lua_check_apr_table を介して Lua のユーザーデータから C の req_table_t 構造体へ変換されます。 4. **実行**: apr_table_get / apr_table_set を呼び出し、必要に応じて改行コードのサニタイズ処理を経て、実際の APR テーブルを操作します。

直接依存

  • mod_lua.h: モジュール共通のデータ構造およびロギング機能。
  • lua_apr.h: 公開インターフェース定義。
  • lua.h / lualib.h / lauxlib.h: Lua C API。

関連コード

  • lua_request.c: req_table_t の具体的な生成や、リクエストに関連するテーブルの管理を行っています。
  • mod_lua.c: モジュールのメインエントリーポイントであり、ap_lua_init を呼び出すタイミングを制御します。

読者向けメモ

  • **改行コードの扱い**: lua_table_set 内で notes 以外のテーブルに対して改行コードをスペースに置換するロジックがあります。これは HTTP ヘッダーの整合性を保つための重要な防御的実装です。
  • **Lua バージョン互換性**: ap_lua_init 内で LUA_VERSION_NUM を用いて Lua 5.1 と 5.2 以降の API 差異（luaL_register vs luaL_newlib）を吸収しています。
  • **メタテーブル**: Apr.Table は Lua 側で __index と __newindex が設定されているため、スクリプトからは通常のテーブルのように t["key"] = "val" といった構文でアクセス可能です。
