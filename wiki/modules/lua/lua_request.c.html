modules/lua/lua_request.c

概要

Apache HTTP Server の mod_lua において、Lua スクリプトから Apache の内部構造（request_rec, conn_rec, server_rec）へアクセスするためのブリッジ機能を提供します。Lua の userdata を介して、HTTP リクエストの属性取得、ヘッダー操作、ボディの読み書き、および各種ユーティリティ関数を公開します。

モジュール内での役割

mod_lua の中核となる API 提供レイヤーです。Lua VM がリクエスト処理中に呼び出すメソッドのディスパッチテーブルを構築し、Apache の C 言語ベースのデータ構造を Lua のテーブルや型に変換する役割を担います。

責務

  • **リクエスト・メタデータ公開**: request_rec のフィールド（URI, メソッド, ステータス等）を Lua から読み書き可能にする。
  • **データ変換**: apr_table_t を Lua のテーブルへ変換し、ヘッダーや環境変数の操作を可能にする。
  • **I/O 制御**: リクエストボディの読み込み、レスポンスの書き込み、フィルタ操作、WebSocket 通信のサポート。
  • **ユーティリティ提供**: 暗号化（MD5/SHA1）、パス操作、ログ出力、設定ツリーの走査などの補助機能。
  • **IVM（Inter-VM）共有**: apr_shm を用いたプロセス間でのデータ共有機能の提供。

重要なシンボル

リクエスト・ディスパッチ

  • ap_lua_load_request_lmodule: Lua VM 初期化時に呼び出され、Apache2.Request メタテーブルとディスパッチハッシュを登録する。
  • req_dispatch: Lua の __index メタメソッド。Lua から r:field が呼ばれた際に、ハッシュから対応する C 関数を検索・実行する。
  • req_newindex: Lua の __newindex メタメソッド。リクエストフィールドへの書き込みを制御する。

データ変換・ヘルパー

  • req_aprtable2luatable: apr_table_t を Lua のテーブル（キー・値ペアおよび配列）に変換する。
  • ap_lua_check_request_rec: Lua スタック上の userdata が Apache2.Request であることを検証し、request_rec ポインタを抽出する。

WebSocket 関連

  • lua_websocket_greet: WebSocket ハンドシェイク（Sec-WebSocket-Key の検証とレスポンス生成）。
  • lua_websocket_read / lua_websocket_write: フレームの解析と構築、マスク処理を含む WebSocket I/O。

制御フローとデータフロー

1. **初期化**: ap_lua_load_request_lmodule が Apache2.Request メタテーブルを生成し、Apache2.Request.dispatch レジストリに各メソッドを登録する。 2. **アクセス**: Lua スクリプトが r:method() を呼び出すと、req_dispatch が実行される。 3. **ディスパッチ**: req_dispatch は req_fun_t 構造体（関数ポインタと型情報）をハッシュから取得し、型に応じて C 関数を呼び出し、結果を Lua スタックにプッシュする。 4. **データフロー**: apr_table_t などの Apache 内部データは、コールバック関数（req_aprtable2luatable_cb 等）を通じて Lua テーブルへ逐次コピーされる。

読者向けメモ

  • **メモリ管理**: すべてのメモリ割り当ては request_rec のプール（r->pool）を使用しており、リクエスト終了時に自動的に解放されます。
  • **スレッド安全性**: lua_ivm_set/get は lua_ivm_mutex を使用して共有メモリへのアクセスを保護しています。
  • **制約**: req_newindex で設定可能なフィールドは限定されています。また、MODLUA_MAX_REG_MATCH により正規表現のキャプチャ数に制限があります。
  • **省略**: WebSocket のフレーム解析ロジックは非常に詳細ですが、本ファイルでは基本的な HTTP リクエスト処理と混在しています。エラーハンドリングは lua_error を介して Lua 側に伝播されます。
