modules/lua/lua_dbd.h

概要

lua_dbd.h は、Apache HTTP Server の mod_lua において、データベース操作機能（DBD: Database Driver）を Lua スクリプトから利用可能にするためのインターフェース定義ファイルです。

モジュール内での役割

mod_lua のサブシステムとして、Lua VM と Apache の mod_dbd および apr_dbd を仲介します。Lua スクリプトからデータベース接続、クエリ実行、結果取得を抽象化して提供します。

責務

  • データベース接続（lua_db_handle）、結果セット（lua_db_result_set）、プリペアドステートメント（lua_db_prepared_statement）の構造体定義。
  • Lua から呼び出されるデータベース操作関数（select, query, prepare 等）のプロトタイプ宣言。
  • apr_dbd（低レベル）と mod_dbd（Apache 統合型）の両方のバックエンドをサポートするための型定義。

重要なシンボル

  • lua_db_handle: データベース接続状態を保持する構造体。接続先やドライバ情報、生存フラグを管理します。
  • LUA_DBTYPE_APR_DBD / LUA_DBTYPE_MOD_DBD: 使用するバックエンドの種別を識別するフラグ。
  • lua_db_result_set: クエリ実行後の結果セットを保持し、Lua 側での行取得をサポートします。

制御フローとデータフロー

Lua スクリプトがデータベース操作を要求すると、lua_db_handle を介して apr_dbd または mod_dbd の関数が呼び出されます。データフローは以下の通りです： 1. **初期化**: lua_db_acquire で接続を取得し、Lua のユーザーデータとしてスタックに積む。 2. **実行**: lua_db_query や lua_db_select が呼ばれ、Lua の引数を C の型に変換してバックエンドへ渡す。 3. **終了**: lua_db_close またはガベージコレクタ（lua_db_gc）により、リソースが解放される。

直接依存

  • mod_lua.h: mod_lua の基本定義。
  • apr.h: Apache Portable Runtime の基本型。
  • apr_dbd.h: APR のデータベース抽象化レイヤー。
  • mod_dbd.h: Apache のデータベース管理モジュール。

関連コード

  • modules/lua/lua_dbd.c: 本ヘッダーで定義された関数の実装。
  • modules/lua/mod_lua.c: モジュールのエントリポイント。
  • modules/database/mod_dbd.c: 実際にデータベース接続を管理する Apache モジュール。

読者向けメモ

  • 本ファイルは主に C 言語側のデータ構造定義です。Lua 側から見た API の詳細は lua_dbd.c の実装と併せて確認してください。
  • lua_db_gc は Lua のガベージコレクションによって自動的に呼び出されるため、リソースリークを防ぐ重要な役割を担っています。
  • 接続の生存確認には alive フラグが使用されます。マルチスレッド環境下での接続の再利用については mod_dbd の仕様に依存します。
