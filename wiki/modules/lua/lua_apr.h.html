modules/lua/lua_apr.h

概要

lua_apr.h は、Apache Portable Runtime (APR) のデータ構造を Lua スクリプトから透過的に操作するためのブリッジ機能を提供するヘッダーファイルです。主に apr_table_t を Lua のテーブルとして扱うためのインターフェースを定義しています。

モジュール内での役割

mod_lua モジュールの一部として、C 言語側の Apache 内部データ構造と Lua VM 間のデータ変換層を担います。特に、HTTP リクエストやレスポンスのヘッダー操作などで頻繁に使用される apr_table_t を、Lua のオブジェクトとして公開する役割を持ちます。

責務

  • **Lua メタテーブルの管理**: Apr.Table 型のメタテーブルを定義し、Lua からのアクセスを制御します。
  • **データ変換**: C の req_table_t 構造体と Lua のスタック間でのポインタのボックス化・アンボックス化を行います。
  • **安全な操作の提供**: apr_table_t へのアクセス時に、改行コードのサニタイズなど、Apache 内部の整合性を保つためのラッパー関数を提供します。

重要なシンボル

  • ap_lua_init: Lua VM 初期化時に Apr.Table メタテーブルを登録する関数。
  • ap_lua_check_apr_table: Lua スタック上の値が Apr.Table であるかを確認し、内部ポインタを取得する。
  • ap_lua_push_apr_table: C 側の req_table_t を Lua オブジェクトとしてスタックにプッシュする。

制御フローとデータフロー

1. **初期化**: ap_lua_init が呼ばれ、Apr.Table メタテーブルが作成されます。この際、__index と __newindex がそれぞれ get と set メソッドにマッピングされます。 2. **データアクセス**: Lua スクリプトから table:get(key) や table:set(key, val) が呼び出されると、lua_apr.c 内の静的関数が実行されます。 3. **変換と検証**: ap_lua_check_apr_table を介して C の構造体へアクセスし、必要に応じて値の検証（改行コードの置換など）を行った後、apr_table_t API を呼び出します。

直接依存

  • apr_tables.h: apr_table_t 操作の基盤。
  • apr.h および各種 APR ユーティリティ (apr_md5.h, apr_sha1.h 等): 汎用的なデータ処理および型定義。
  • scoreboard.h, http_main.h: Apache サーバーのコアコンテキストへのアクセス。
  • ap_mpm.h
  • apr_base64.h
  • apr_poll.h

関連コード

  • modules/lua/lua_apr.c: 本ヘッダーで宣言された関数の実装および Lua メソッドの定義。
  • modules/lua/mod_lua.h: req_table_t 構造体の定義元。
  • modules/lua/lua_request.c: 本機能を利用してリクエストヘッダー等を操作する上位層。

読者向けメモ

  • 本ファイルは req_table_t を対象としていますが、これは単なる apr_table_t ではなく、Apache のリクエストコンテキスト (request_rec) を保持するラッパー構造体である点に注意してください。
  • set メソッド内では、notes テーブル以外に対して改行コードの混入をチェックし、スペースに置換する安全策が講じられています。これは HTTP ヘッダーインジェクションを防ぐための重要な実装です。
