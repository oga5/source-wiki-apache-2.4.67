modules/lua/lua_config.c

概要

Apache HTTP Server の mod_lua における設定管理および Lua スクリプト環境への設定情報の公開を担当します。Apache のディレクトリ設定（ap_lua_dir_cfg）やコマンドパラメータ（cmd_parms）を Lua から操作可能にするためのブリッジ機能を提供します。

モジュール内での役割

mod_lua の設定フェーズと実行フェーズを繋ぐ役割を担います。Apache の設定構造体を Lua のユーザーデータ（userdata）としてラップし、Lua スクリプトから Apache の設定やログ出力機能へ安全にアクセスできるようにします。

責務

  • **設定のバインディング**: ap_lua_dir_cfg および cmd_parms を Lua のメタテーブルとして登録し、メソッドを公開する。
  • **ハンドラマッピング**: Lua スクリプト内で定義されたハンドラ設定（ファイル、関数名、スコープ、正規表現パターン）を解析し、Apache 内部の構造体に格納する。
  • **ログ出力の統合**: Lua スクリプトから Apache のログシステム（ap_log_error）を呼び出すためのインターフェースを提供し、トレースレベルから緊急レベルまでのログ出力をサポートする。

重要なシンボル

設定バインディング・登録

  • ap_lua_load_config_lmodule: Lua VM 起動時に呼び出され、Apache2.DirConfig および Apache2.CommandParameters メタテーブルを登録する。
  • cfg_methods: match_handler や directory を含む DirConfig 用のメソッドテーブル。
  • cmd_methods: ログ出力用メソッド（debug, err, trace1 等）を網羅した CommandParameters 用のメソッドテーブル。

ハンドラ管理

  • ap_lua_map_handler: C 言語側からハンドラを登録するための汎用関数。
  • cfg_lua_map_handler: Lua スクリプトから呼び出されるハンドラ登録用関数。テーブル引数を解析して ap_lua_mapped_handler_spec を構築する。

内部ヘルパー

  • check_dir_config / check_cmd_parms: Lua のスタックから userdata を取り出し、型チェックを行う。
  • apl_toscope: 文字列（"request", "once" 等）を内部のスコープ定数に変換する。
  • cmd_log_at: ログレベルを引数に取り、Lua のスタック情報からソースファイル名と行番号を取得して Apache ログへ出力する。

制御フローとデータフロー

1. **初期化**: ap_lua_load_config_lmodule が呼ばれ、Lua VM に Apache 固有の型を認識させる。 2. **設定登録**: Lua スクリプトが match_handler を呼び出すと、cfg_lua_map_handler が実行される。 3. **データ変換**: Lua のテーブル引数が解析され、apr_pcalloc で確保された ap_lua_mapped_handler_spec に値がコピーされる。 4. **永続化**: 生成された仕様は cfg->mapped_handlers 配列にプッシュされ、Apache のプール（cfg->pool）の寿命に従って管理される。 5. **ログ出力**: cmd_log_at が呼ばれると、lua_getinfo で現在の実行コンテキストを特定し、ap_log_error を通じて Apache のログパイプラインへ流し込む。

直接依存

  • lua_config.h: 公開インターフェース定義。
  • lua_vmprep.h: Lua VM 構築および互換性レイヤー。
  • mod_lua.h: mod_lua の主要な構造体定義。

関連コード

  • mod_lua.c: モジュールのエントリポイント。ここで ap_lua_load_config_lmodule が呼び出される。
  • lua_vmprep.c: Lua VM のライフサイクル管理。

読者向けメモ

  • **メモリ管理**: すべての動的確保（apr_pcalloc, apr_pstrdup）は cfg->pool を使用しており、Apache のリクエストまたは設定ライフサイクルに紐付いています。明示的な free は不要です。
  • **エラーハンドリング**: luaL_error を使用して Lua 側へ例外を伝播させます。正規表現のコンパイル失敗などはここで捕捉されます。
  • **省略された詳細**: cfg_root など一部の関数はコメントアウトされており、現時点では使用されていません。
  • **注意点**: cmd_log_at 内の APLOGNO マクロは意図的に省略されています。これは動的なログ出力を行うための設計上の判断です。
