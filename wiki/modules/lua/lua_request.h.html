modules/lua/lua_request.h

概要

Apache HTTP Server の mod_lua において、HTTP リクエスト、接続、およびサーバー情報を Lua 環境へ公開するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

mod_lua の中核となるデータバインディング層です。Apache の内部構造体（request_rec, conn_rec, server_rec）を Lua のオブジェクトとして操作可能にするためのブリッジ機能を提供します。

責務

  • Apache のリクエスト処理コンテキストを Lua スタックへプッシュする関数の宣言。
  • Lua スクリプトから Apache の内部データへアクセスするための型定義と定数の提供。
  • リクエストに関連するテーブルやフィールドを抽象化するための構造体定義。

重要なシンボル

  • ap_lua_push_request(lua_State *L, request_rec *r): request_rec を Lua の userdata としてスタックにプッシュします。
  • req_fun_t: Lua から呼び出される Apache 内部関数の型とポインタを保持する構造体。
  • req_table_t: request_rec 内の apr_table_t（ヘッダーや環境変数など）を Lua から操作するためのラッパー構造体。
  • APL_REQ_FUNTYPE_*: Lua 側からアクセスするリクエストフィールドのデータ型を識別する定数群。

制御フローとデータフロー

本ファイルは定義のみを提供します。実際のデータフローは、lua_request.c 内でこれらの構造体を利用して行われます。 1. mod_lua がリクエストを処理する際、ap_lua_push_request が呼び出されます。 2. Lua 側でリクエストオブジェクトのメソッドが呼ばれると、req_fun_t を介して対応する C 関数がディスパッチされます。 3. req_table_t を通じて、Apache の apr_table_t と Lua のテーブル間でのデータ同期が行われます。

直接依存

  • mod_lua.h: モジュール共通の定義と request_rec 等の型定義。
  • util_varbuf.h: 可変長バッファのサポート（lua_ivm_object 内で使用）。

関連コード

  • modules/lua/lua_request.c: 本ヘッダーで宣言された関数の実装および、Lua スタックダンプ等のデバッグユーティリティが含まれます。
  • modules/lua/mod_lua.c: モジュールのメインエントリーポイントであり、リクエスト処理のライフサイクルを管理します。

読者向けメモ

  • 本ファイルは主に「Lua から Apache の内部状態をどう見せるか」というマッピング層です。
  • lua_ivm_object はプロセス間共有メモリや Lua VM 間でのデータ受け渡しに関連する構造体であり、通常のスクリプト実行フローとは別に注意が必要です。
  • Lua スタック上の userdata を操作するため、メモリ管理（apr_pool_t）のライフサイクルと Lua のガベージコレクションの整合性に注意してください。
