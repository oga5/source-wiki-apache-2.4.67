システム内での役割

本モジュールは、Apache HTTP Server 2.4 のモジュール開発を学習するためのリファレンス実装群です。ディレクトリ境界に基づき、開発者が Apache のフック機構やリクエスト処理パイプラインの基本を理解するためのサンプルコードを提供します。

責務

  • **開発者向け教育**: Apache モジュールのライフサイクル（登録・フック・応答）の最小構成を提示する。
  • **リクエスト処理の実演**: example-handler を通じたリクエストの捕捉、判定、およびレスポンス生成の標準的な作法を示す。
  • **API 利用例の提供**: APR (Apache Portable Runtime) を用いたファイル操作やハッシュ計算の実装例を提示する。

このモジュールのファイル

  • docs/manual/developer/mod_example_1.c: 最小構成のハンドラ実装（"Hello, world!" の返却）。
  • docs/manual/developer/mod_example_2.c: 発展的なハンドラ実装（ファイルハッシュ計算と HTML レスポンス）。

主要なデータ構造

  • request_rec: Apache のリクエスト情報を保持するコア構造体。
  • module: AP_DECLARE_MODULE マクロにより定義される、モジュールのエントリポイントとなる構造体。

主なフロー

1. **フック登録**: register_hooks 関数が Apache 起動時に呼び出され、ap_hook_handler を通じてハンドラを登録する。 2. **リクエスト判定**: 各ハンドラ関数内で r->handler を確認し、対象外であれば DECLINED を返して処理を委譲する。 3. **処理実行**: 対象リクエストの場合、ap_rprintf 等を用いてレスポンスを生成し、OK を返して処理を完了する。 4. **リソース管理**: apr_pool_t を活用し、リクエスト終了時に自動的なメモリ解放を行う。

外部依存

  • **Apache Core API**: httpd.h, http_config.h, http_protocol.h, ap_config.h
  • **APR (Apache Portable Runtime)**: apr_md5.h, apr_sha1.h, apr_file_io.h, apr_tables.h

内部構成

  • サンプルモジュール 2 個で構成される学習用ディレクトリ。

推奨読書順

1. docs/manual/developer/mod_example_1.c（基本構造の理解） 2. docs/manual/developer/mod_example_2.c（実用的な API 利用と処理フローの理解）
