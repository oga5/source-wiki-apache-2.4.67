support/win32

システム内での役割

support/win32 は、Windows 環境における Apache HTTP Server の運用管理を支援するツール群を提供するモジュールです。ディレクトリ境界に基づき、サービス監視用 GUI アプリケーションと、サービスプロセス向けのコンソールリダイレクトユーティリティを包含しています。

責務

  • **サービス監視と制御**: Windows サービスとして動作する Apache の状態（起動・停止・再起動）を GUI を通じて管理する。
  • **対話的コンソールの提供**: サービスセッション下で実行されるプロセスに対し、標準入出力を扱うためのコンソールウィンドウを動的に割り当てる。

このモジュールのファイル

  • support/win32/ApacheMonitor.c: タスクトレイ常駐型管理ツールの実装。
  • support/win32/ApacheMonitor.h: GUI リソースおよび識別子の定義。
  • support/win32/wintty.c: サービスプロセス用コンソールリダイレクトユーティリティ。

主要なデータ構造

  • ST_APACHE_SERVICE: サービス名やパス、PID 等の情報を保持する監視対象構造体。
  • ST_MONITORED_COMP: リモート監視対象のコンピュータ情報とレジストリハンドルを保持。
  • feedback_args_t: wintty における入出力ハンドル管理用のスレッド引数構造体。

主なフロー

1. **ApacheMonitor**: レジストリからサービスを特定し、WM_TIMER によるポーリングで状態を監視。ユーザー操作時は SCM (Service Control Manager) を介してサービスを制御し、必要に応じて管理者権限で自身を再起動する。 2. **wintty**: サービスセッションからコンソールを分離・再生成し、メインスレッドとフィードバックスレッド間で標準入出力のストリームを仲介する。

外部依存

  • windows.h, WtsApi32.h, commctrl.h, shlobj.h: Windows システムおよび UI/シェル API。
  • ApacheMonitor.h: リソース ID 定義（内部依存）。

内部構成

本モジュールは、GUI 管理ツールとコンソール補助ツールの 2 つの独立した機能で構成されています。

推奨読書順

1. support/win32/ApacheMonitor.h (リソース定義の把握) 2. support/win32/ApacheMonitor.c (サービス管理ロジックの理解) 3. support/win32/wintty.c (サービス環境下での入出力制御の理解)

未解決の問い

  • 大規模環境における MAX_APACHE_SERVICES (128) 等の固定長配列による制限の緩和。
  • ネットワークブラウズ機能の安定性向上およびエラーハンドリングの精緻化。
