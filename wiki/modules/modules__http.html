modules/http

システム内での役割

modules/http は、Apache HTTP Server における HTTP プロトコル処理の中核を担うモジュール群です。リクエストの解析から、フィルタチェーンを通じたレスポンスの生成、条件付きリクエストの評価、MIME タイプの決定まで、HTTP セマンティクスの実装を網羅しています。

責務

  • **プロトコル解析**: HTTP リクエストの読み込みとメソッドの管理。
  • **フィルタリング**: チャンク転送、バイトレンジリクエスト、入力/出力フィルタの適用。
  • **メタデータ管理**: MIME タイプ、ETag、キャッシュ制御ヘッダーの生成と解決。
  • **ライフサイクル制御**: リクエストの実行フロー、内部リダイレクト、エラーハンドリングの統括。

このモジュールのファイル

  • http_protocol.c: プロトコルセマンティクスとステータス管理の核心。
  • http_filters.c: 入出力フィルタリングとストリーム処理のゲートキーパー。
  • http_request.c: リクエストの実行フローとライフサイクル管理。
  • http_core.c: HTTP フィルタの登録と接続処理のディスパッチ。
  • mod_mime.c: ファイル拡張子に基づく MIME タイプ解決。
  • byterange_filter.c: Range リクエストの処理と断片化。
  • http_etag.c: キャッシュ検証用の ETag 生成。
  • chunk_filter.c: チャンク転送エンコーディングの実装。

主要なデータ構造

  • http_ctx_t: 入力フィルタの状態管理。
  • indexes_t: バイトレンジの範囲指定情報。
  • mime_dir_config: ディレクトリごとの MIME マッピング設定。
  • extension_info: 拡張子に関連付けられたメタデータ。
  • header_struct / header_filter_ctx: ヘッダー送信時のコンテキスト。

主なフロー

1. **接続・解析**: http_core.c が接続を受け取り、http_protocol.c がリクエストを解析。 2. **メタデータ解決**: mod_mime.c がリソースの MIME タイプを決定。 3. **リクエスト実行**: http_request.c がフックを順次実行し、ハンドラを呼び出す。 4. **フィルタ処理**: http_filters.c を基点に、byterange_filter.c や chunk_filter.c がレスポンスを加工。 5. **ヘッダー生成**: http_etag.c 等がキャッシュ情報を付与し、クライアントへ送出。

外部依存

  • **APR**: apr_buckets.h (ストリーム処理), apr_hash.h, apr_tables.h 等。
  • **Apache Core**: httpd.h, http_config.h, http_request.h, util_filter.h 等。

内部構成

本モジュールはディレクトリ境界に基づき構成されており、個別のソースファイルが特定の HTTP 機能（フィルタ、プロトコル、メタデータ）を分担する疎結合な設計となっています。

推奨読書順

1. http_protocol.c (プロトコルの基礎) 2. http_filters.c (フィルタの仕組み) 3. http_request.c (リクエストのライフサイクル) 4. http_core.c (接続とフィルタの登録) 5. mod_mime.c (メタデータ解決) 6. byterange_filter.c (レンジリクエスト) 7. http_etag.c (キャッシュ検証) 8. chunk_filter.c (チャンク転送)
