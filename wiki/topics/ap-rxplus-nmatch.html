ap_rxplus_nmatch

概念

ap_rxplus_nmatch は、Apache HTTP Server において、コンパイル済みの正規表現オブジェクト（ap_rxplus_t）に対して直近に実行されたマッチング操作から、検出されたキャプチャグループの総数を取得するための関数およびマクロです。

なぜ重要か

正規表現マッチングの実行後、マッチした部分文字列（キャプチャグループ）の数を正確に把握することは、後続の文字列抽出や、認証・認可処理（例: mod_authnz_fcgi での処理）などを安全かつ適切に行うために重要です。

関与するモジュール

本機能には以下のモジュールが関与しています。

  • module:include
  • module:server

主要エンティティ

本トピックに関連する主要なエンティティは以下の通りです。

  • symbol:include/ap_regex.h#ap_rxplus_nmatch：マクロ定義（ap_regex.h:267）
  • symbol:server/util_regex.c#ap_rxplus_nmatch：関数定義（server/util_regex.c:183）

エンドツーエンドの流れ

1. 正規表現オブジェクト ap_rxplus_t を用いてマッチング処理（ap_rxplus_exec など）が実行されます。 2. マッチング処理の結果、マッチング状態やキャプチャグループ数（nmatch）が ap_rxplus_t 構造体内部に格納されます。 3. ap_rxplus_nmatch が呼び出されます。 4. 内部で rx->match の存在を確認し、存在する場合は構造体内部の rx->nmatch フィールドの値を返し、存在しない場合は 0 を返します。 5. 呼び出し元（mod_authnz_fcgi.c や util_regex.c 内部など）は、この戻り値を利用してキャプチャグループの処理を行います。

※なお、マッチング処理自体の詳細な内部実装や、呼び出し元における具体的な後続処理の全容など、エンドツーエンドの完全なフローは現時点ではすべて明らかになっていません。

重要な不変条件

  • rx->match が NULL である場合、ap_rxplus_nmatch は常に 0 を返します。
  • rx->nmatch の値は、直近のマッチング処理（ap_rxplus_exec の実行結果など）に依存します。

よくある落とし穴

  • **NULLポインタの未チェック**: 引数 rx 自体が NULL である場合、関数内部でガードされていないため、セグメンテーションフォールトが発生する可能性があります。
  • **マッチング未実行時の呼び出し**: マッチングが一度も実行されていない、または前回のマッチングが失敗している場合、rx->match が NULL となり、戻り値は 0 になります。本関数はマッチング処理の直後に呼び出すことが想定されています。
