ap_os_create_privileged_process

概念

ap_os_create_privileged_process は、Apache HTTP Server において、プラットフォーム固有の権限管理やプロセス生成モデルを抽象化し、特権プロセス（CGIスクリプトなど）を生成するための共通インターフェースを提供する関数です。

なぜ重要か

Apache HTTP Server はマルチプラットフォームで動作するため、CGIなどの外部プロセスを起動する際の権限管理モデルがOSごとに異なります。

  • **Unix環境**では、suexec を利用したユーザー/グループ権限の切り替えが必要になります。
  • **NetWare、OS/2、Windows環境**では、現時点では標準的なプロセス生成（apr_proc_create）の単純なラッパーとして機能していますが、将来的にプラットフォーム固有の特権昇格や制御ロジックを組み込むための拡張ポイント（抽象化レイヤー）として定義されています。

これにより、呼び出し元である mod_cgi や mod_cgid は、プラットフォームごとの差異を意識せずに一貫した方法でプロセスを生成できます。

エンドツーエンドの流れ

1. **呼び出し**: modules/generators/mod_cgi.c または modules/generators/mod_cgid.c から、リクエストコンテキスト（request_rec）やプログラム名、引数、環境変数、プロセス属性（apr_procattr_t）を伴って ap_os_create_privileged_process が呼び出されます。 2. **プラットフォーム別の処理**:

  • **Unix環境 (os/unix/unixd.c)**:

    • 1. ap_run_get_suexec_identity(r) フックを呼び出し、リクエストに対応する suexec 用のユーザー/グループ識別情報（ugid）を取得します。 2. ugid が取得できた場合は、ap_unix_create_privileged_process を呼び出して suexec 経由で特権プロセスを生成します。 3. ugid が NULL（取得できない）の場合は、通常の apr_proc_create を直接呼び出します。

  • **NetWare, OS/2, Windows環境**:
  • 受け取った引数をそのまま APR（Apache Portable Runtime）の apr_proc_create 関数に転送して実行します。

    • 3. **結果の返却**: プロセス生成の成否を示すステータスコード（apr_status_t）を呼び出し元に返します。

※ ap_unix_create_privileged_process の内部実装詳細や、各プラットフォームにおける apr_proc_create の具体的なOSレベルでの挙動など、上記以外の詳細なエンドツーエンドの流れは提供された情報からは未確定です。

重要な不変条件

  • **引数の有効性**: 呼び出し元は、newproc（apr_proc_t）および attr（apr_procattr_t）に対して、適切に初期化または割り当てられた有効なポインタを渡す必要があります。
  • **Unix環境におけるフォールバック**: Unix環境において suexec の識別情報（ugid）が取得できない場合は、処理を失敗させるのではなく、通常の apr_proc_create によるプロセス生成へ安全にフォールバックしなければなりません。
  • **非Unix環境での等価性**: 現時点の実装において、NetWare、OS/2、Windows環境では、本関数の動作は apr_proc_create を直接呼び出すことと等価です。

よくある落とし穴

  • **プラットフォーム間のセキュリティ挙動の不一致**: Unix環境では suexec による厳密な権限分離が実行されますが、NetWare、OS/2、Windows環境では単なる apr_proc_create のラッパーであるため、プラットフォーム間でセキュリティや権限の挙動が異なる点に注意が必要です。
  • **エラーコードの透過性**: 本関数は内部で発生したエラー（apr_proc_create や ap_unix_create_privileged_process の失敗）の apr_status_t コードをそのまま返却するため、呼び出し元で適切にエラーハンドリングを行う必要があります。
