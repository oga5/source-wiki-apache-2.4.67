ap_os_create_privileged_process

概念

ap_os_create_privileged_process は、Apache HTTP Serverにおいて、プラットフォーム（OS）ごとの特性や権限モデルの違いを抽象化し、特権プロセス（CGIプロセスなど）を生成するための共通インターフェースを提供する関数です。

なぜ重要か

Apache HTTP Serverはマルチプラットフォームで動作するため、外部プロセスを起動する際の権限管理方法がOSごとに異なります。例えば、Unix系OSでは suexec を使用したユーザー/グループの切り替えが必要ですが、他のOSでは異なるアプローチが取られます。この関数は、mod_cgi や mod_cgid などのプロセス生成を行うモジュールに対して、プラットフォーム固有のプロセス生成ロジックを隠蔽した統一的なAPIを提供するために重要です。

エンドツーエンドの流れ

1. **呼び出し**: modules/generators/mod_cgi.c または modules/generators/mod_cgid.c から、リクエスト情報（request_rec）、生成するプロセス情報（apr_proc_t）、プログラム名、引数、環境変数、プロセス属性（apr_procattr_t）、メモリプールを引数として本関数が呼び出されます。 2. **プラットフォーム別の処理**:

  • **Unix環境 (os/unix/unixd.c)**:

    • 1. ap_run_get_suexec_identity(r) を呼び出し、対象リクエストに対する suexec 用のユーザー/グループ識別子（ugid）を取得します。 2. ugid が取得できた場合、ap_unix_create_privileged_process を呼び出して suexec を経由した特権プロセス生成を行います。 3. ugid が NULL の場合（suexec が適用不可の場合）は、通常の apr_proc_create を直接呼び出して標準的なプロセス生成を行います。

  • **NetWare, OS/2, Windows環境 (os/netware/util_nw.c, os/os2/util_os2.c, os/win32/util_win32.c)**:
  • 受け取った引数をそのまま APR（Apache Portable Runtime）ライブラリの apr_proc_create に転送し、プロセスを生成します。

    • 3. **返却**: プロセス生成処理の結果（成功時は APR_SUCCESS、失敗時はエラーコード）を呼び出し元に返します。

重要な不変条件

  • **引数の有効性**: 呼び出し元は、newproc（apr_proc_t）および attr（apr_procattr_t）に対して、適切に初期化または割り当てられた有効なメモリ領域を渡す必要があります。
  • **Unix環境における前提条件**: ap_run_get_suexec_identity フックが利用可能である必要があります。
  • **非Unix環境における前提条件**: APRのプロセス管理機能（apr_proc_create）が正常に動作する環境である必要があります。

よくある落とし穴

  • **非Unix環境での特権処理の欠如**: NetWare、OS/2、Windows環境における現在の実装は、単に apr_proc_create を呼び出すだけのラッパーとなっています。そのため、これらのプラットフォームで固有の特権昇格や詳細なセキュリティ制御が必要になった場合、現行の実装では対応できず、将来的に関数内部の実装を拡張する必要があります。
  • **エラーハンドリングの依存**: 内部で呼び出される apr_proc_create や ap_unix_create_privileged_process が失敗した場合、そのエラーコードがそのまま呼び出し元に返されるため、呼び出し元で適切にエラーをハンドリングする必要があります。
