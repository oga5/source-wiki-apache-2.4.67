handle_headers

概念

handle_headers は、Apache HTTP Server（httpd）の FastCGI 関連モジュールにおいて、バックエンドから受信したデータストリームから HTTP ヘッダの終端（空行）を検出するための関数です。

なぜ重要か

FastCGI バックエンドからのレスポンスはネットワークの状況などにより分割されて届く可能性があるため、ヘッダの境界（\r\n\r\n または \n\n）を正確に識別し、ヘッダ部分とボディ部分を正しく分離して処理を遷移させるために重要です。

エンドツーエンドの流れ

mod_proxy_fcgi.c における処理の流れは以下の通りです： 1. バックエンドからデータを受信すると、handle_headers が呼び出されます。 2. readbuf 内のデータを1バイトずつ走査し、改行コード（\r や \n）の出現パターンに応じて state（状態変数）を更新します。 3. \r\n\r\n または \n\n を検出して HDR_STATE_DONE_WITH_HEADERS に達すると、関数は 1 を返してヘッダの終端を通知します。終端に達しない場合は 0 を返し、次のデータ受信を待ちます。

なお、mod_authnz_fcgi.c 側の handle_headers における詳細なエンドツーエンドの流れや、両モジュール間での共通の呼び出しフローの全容はまだ完全には解析されていません。

重要な不変条件

  • **状態の一貫性**: mod_proxy_fcgi.c の handle_headers において、解析状態を追跡する state 変数は、データの断片化に対応するため、呼び出し間で一貫して保持・更新される必要があります。
  • **バッファの有効性**: readbuf は readlen で指定された長さの有効なデータを含んでいなければなりません。
  • mod_authnz_fcgi.c 側の不変条件については、現時点では未解析です。

よくある落とし穴

  • **データの断片化**: ヘッダが複数回に分割されて届く場合、state の引き継ぎが正しく行われないと、ヘッダの終端を誤認するか検出に失敗する可能性があります。
