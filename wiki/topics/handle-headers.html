handle_headers

概念

handle_headers は、FastCGI バックエンドから受信したストリームデータから HTTP ヘッダの終端（空行）を検出するためのステートマシンとして機能する関数です。

なぜ重要か

FastCGI プロトコルにおいて、バックエンドから送られてくるレスポンスのヘッダ領域とボディ領域を正しく分離するために不可欠です。ネットワークの都合によりデータが複数回に断片化されて受信される場合でも、解析状態を維持しながら正確にヘッダの終端を特定する役割を担います。

関与するモジュール

  • module:modules/aaa
  • module:modules/proxy

主要エンティティ

  • symbol:modules/proxy/mod_proxy_fcgi.c#handle_headers：FastCGI プロキシモジュールにおけるヘッダ解析関数の定義および実装。
  • symbol:modules/aaa/mod_authnz_fcgi.c#handle_headers：FastCGI 認証・認可モジュールにおける呼び出し、または関連するヘッダ解析処理。

エンドツーエンドの流れ

1. **データの受信**: バックエンドからデータを受信するたびに、呼び出し元（mod_proxy_fcgi.c や mod_authnz_fcgi.c）から handle_headers が呼び出されます。 2. **状態遷移の走査**: 入力バッファ（readbuf）をバイト単位で走査し、\r または \n の出現パターンに応じて状態（state）を遷移させます。 3. **終端検出**: \r\n\r\n または \n\n に相当するシーケンスを検出すると、状態が HDR_STATE_DONE_WITH_HEADERS に移行し、関数は 1 を返してヘッダの解析完了を通知します。 4. **継続待機**: 終端に達しない場合は 0 を返し、後続のデータ受信と再度の解析を待機します。

*注意: バックエンドとの接続確立からレスポンス全体の処理完了に至る完全なエンドツーエンドの制御フローは、提供された情報からは詳細が判明していません。*

重要な不変条件

  • **状態の維持**: データの断片化に対応するため、解析状態を示す state ポインタの指す値は、呼び出し間で一貫して保持・更新されなければなりません。
  • **バッファの整合性**: 入力される readbuf は、指定された readlen バイト分の有効なデータを常に含んでいる必要があります。

よくある落とし穴

  • **状態の初期化ミス**: 複数回に分けてデータを受信する際、呼び出し間で state を正しく維持・引き継がないと、ヘッダの終端を誤判定したり検出できなくなったりする可能性があります。
  • **状態遷移の不整合**: ヘッダの定義や解析ロジックを変更する際、HDR_STATE_* 定数の遷移ルールと実装の整合性が崩れると、不正なパケットで無限ループや誤検出が発生する恐れがあります。
