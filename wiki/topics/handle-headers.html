handle_headers

概念

handle_headers は、FastCGI バックエンドから受信したストリームデータから HTTP ヘッダの終端（空行）を検出するためのステートマシンとして機能する関数です。

なぜ重要か

FastCGI プロトコルにおいて、バックエンドからのレスポンスはヘッダとボディが連続したストリームとして送信されます。レスポンスを正しく処理するためには、ヘッダの境界（\r\n\r\n または \n\n）を正確に識別し、ヘッダ部分の受信完了を検知する必要があります。この関数は、データの断片化に対応しながら低レベルな状態遷移を管理し、ヘッダの終端を判定する役割を担っています。

関与するモジュール

  • module:modules/aaa
  • module:modules/proxy

主要エンティティ

  • symbol:modules/proxy/mod_proxy_fcgi.c#handle_headers: FastCGI バックエンドからの受信データからヘッダ終端を検出する関数。
  • symbol:modules/aaa/mod_authnz_fcgi.c#handle_headers: mod_authnz_fcgi.c 内で定義または利用される同名の処理（mod_proxy_fcgi.c の handle_headers は mod_authnz_fcgi.c:557 からも呼び出されます）。

エンドツーエンドの流れ

1. FastCGI バックエンドからデータを受信する際、受信したデータバッファ（readbuf）とその長さ（readlen）が、現在の解析状態（state）とともに handle_headers に渡されます。 2. handle_headers はバッファをバイト単位で走査し、\r や \n の出現パターンに応じて state を更新します。 3. \r\n\r\n または \n\n に相当するシーケンスを検出すると、状態が HDR_STATE_DONE_WITH_HEADERS に遷移し、関数は 1 を返してヘッダの終端に達したことを呼び出し元に知らせます。 4. 終端に達しない場合は 0 を返し、呼び出し元は後続のデータ受信を待機します。

なお、mod_authnz_fcgi および mod_proxy_fcgi における、ヘッダ解析後のレスポンス転送やエラーハンドリングを含む完全なエンドツーエンドの制御フローの詳細は現時点では判明していません。

重要な不変条件

  • **状態の継続性**: データの断片化によりヘッダが複数回に分けて受信される場合、呼び出し間で state ポインタが指す状態値が一貫して保持・更新される必要があります。
  • **バッファの整合性**: 入力される readbuf は、指定された readlen 分の有効なデータを保持していなければなりません。

よくある落とし穴

  • **状態のリセット**: \r や \n 以外の文字が入力された場合、状態は即座に HDR_STATE_READING_HEADERS にリセットされます。不完全な改行シーケンスや不正な文字が混入すると、ヘッダの終端検出が正しく行われない可能性があります。
  • **状態遷移の不整合**: HDR_STATE_* 定数の遷移ロジックを変更する際、状態遷移図との不整合が生じると、ヘッダの終端を誤判定したり、無限にデータ受信を待ち続けたりする危険性があります。
