log_virtual_host

概念

log_virtual_host は、Apache HTTP Server（httpd-2.4.67）において、バーチャルホストに関する情報をログに記録するために用いられる関数群です。この名前を持つ関数は、アクセスログのフォーマット処理を行うモジュールと、サーバーコアのエラーログ処理を行うモジュールの2箇所に、それぞれ独立した static 関数として定義されています。

なぜ重要か

この関数群は、マルチホスト環境において、どのバーチャルホストに対するリクエストやエラーであるかをログ出力に正しく反映させるために重要です。アクセスログ（mod_log_config）とエラーログ（server/log）のそれぞれで、バーチャルホスト情報を適切にフォーマットしてバッファに格納またはエスケープする役割を担っています。

主要エンティティ

  • **symbol:modules/loggers/mod_log_config.c#log_virtual_host**
  • **シグネチャ**: static const char *log_virtual_host(request_rec *r, char *a)
  • **説明**: リクエスト情報（request_rec）からバーチャルホスト情報を取得し、ap_escape_logitem を呼び出してエスケープ処理を施したログ項目を返します。
  • **symbol:server/log.c#log_virtual_host**
  • **シグネチャ**: static int log_virtual_host(const ap_errorlog_info *info, const char *arg, char *buf, int buflen)
  • **説明**: エラーログ情報（ap_errorlog_info）からバーチャルホスト情報を取得し、cpystrn を用いて指定されたバッファ（buf）に安全にコピーします。
