passwd_ctx

概念

passwd_ctx は、Apache HTTP Server（httpd-2.4.67）において、パスワードのハッシュ生成、検証、または暗号化処理のコンテキスト（状態や設定情報）を管理するために使用される構造体です。

ただし、この構造体は共通の単一定義ではなく、関与するモジュール（modules/lua、modules/md、support）ごとに異なる定義と役割を持って個別に実装されています。

なぜ重要か

パスワードのハッシュ化や暗号化処理（htpasswd や htdbm などのユーティリティ、Luaスクリプトからのパスワード操作、MDモジュールでの暗号化処理など）において、必要なパラメータ（メモリプール、入力パスワード、アルゴリズム、出力バッファ、エラー情報など）をカプセル化し、一貫したインターフェースで処理関数に引き渡す役割を果たすため重要です。

主要エンティティ

  • **symbol:modules/lua/lua_passwd.h#passwd_ctx**: Luaモジュール用のパスワードハッシュ生成コンテキスト。
  • **symbol:modules/md/md_crypt.c#passwd_ctx**: MD（Managed Domains）モジュール用のパスフレーズ保持コンテキスト。
  • **symbol:support/passwd_common.h#passwd_ctx**: htpasswd や htdbm などのサポートツールで共有される、パスワード生成・検証用の共通コンテキスト。

エンドツーエンドの流れ

システム全体を横断する単一のエンドツーエンドフローは存在せず、各モジュール内で独立したフローが実行されます。

1. support モジュールにおける流れ

1. 呼び出し元（htpasswd.c の mkrecord や htdbm.c など）が passwd_ctx を初期化し、メモリプールや入力ソース（passwd_src）を設定します。 2. get_password() を介して、指定されたソース（プロンプト、引数、標準入力など）からパスワードを取得し、passwd メンバに格納します。 3. mkhash() が ctx 内のアルゴリズム（alg）や計算コスト（cost）を参照してハッシュを計算し、out に書き込みます。成功時は 0、失敗時はエラーコードを返し、errstr にエラー詳細をセットします。

2. modules/lua モジュールにおける流れ

1. 呼び出し元（lua_passwd.c の mk_password_hash や lua_request.c など）が passwd_ctx を初期化し、メモリプール（pool）と入力パスワード（passwd）を設定します。 2. mk_password_hash(passwd_ctx *ctx) が呼び出され、ctx->alg に基づいて apr_md5_encode、apr_sha1_base64、またはシステム標準の crypt() 等へ処理が分岐します。 3. 生成されたハッシュ文字列が ctx->out に書き込まれます。

3. modules/md モジュールにおける流れ

1. 呼び出し元が passwd_ctx を初期化し、パスフレーズ（pass_phrase）とその長さ（pass_len）を設定します。 2. OpenSSL のコールバック関数やパスワード入力を必要とする暗号化ルーチンにおいて、認証情報を一括して保持するコンテナとして機能し、暗号化操作に引き渡されます。

重要な不変条件

それぞれの定義において、以下の不変条件を維持する必要があります。

  • **support/passwd_common.h**:
  • pool は apr_pool_t によって管理される有効なメモリプールでなければなりません。
  • 処理を実行する前に errfile が適切に初期化されている必要があります。
  • **modules/lua/lua_passwd.h**:
  • pool はエラーメッセージ生成などのために有効な APR プールである必要があります。
  • パスワードの長さは MAX_PASSWD_LEN（256）を超えてはなりません。
  • 呼び出し元は、ハッシュ生成関数を呼び出す前に構造体を適切に初期化する必要があります。
  • **modules/md/md_crypt.c**:
  • pass_phrase は有効な文字列を指している必要があります。
  • pass_len は pass_phrase の実際の長さを正確に反映している必要があります。

よくある落とし穴

  • **名前の衝突と定義の不一致**: passwd_ctx という同名の構造体が3つの異なるファイルで個別に定義されています。これらは互換性がないため、ソースコードの変更やリファクタリング時に混同しないよう注意が必要です。
  • **初期化不備によるクラッシュ**:
  • support ツール群において errfile が未初期化の状態で putline 等が呼び出されると、セグメンテーション違反が発生します。
  • modules/md において pass_phrase が NULL の場合や pass_len が負の値の場合、利用する関数側で未定義の動作を引き起こす可能性があります。
  • **メモリ不足時の強制終了**: support モジュールでは abort_on_oom() が登録されている場合、メモリ確保失敗時にプロセスが即座に終了します。また、pool のライフサイクル管理は呼び出し元の責任です。
