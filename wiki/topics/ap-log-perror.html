ap_log_perror

概念

ap_log_perror は、Apache HTTP Server (httpd) において、メモリプール (apr_pool_t) に関連付けてエラーログを出力するためのAPIおよびマクロ群です。サーバー構造体 (server_rec) やリクエスト構造体 (request_rec) が利用できない、あるいは存在しないコンテキストにおいて、プール情報を利用してログを記録する際に用いられます。

なぜ重要か

エラーログ出力時に、特定の接続やリクエストのコンテキストがない場合でも、メモリプールに関連付けることで、適切なモジュールインデックス、ログレベル、エラーステータス（apr_status_t）を含めた詳細なログ記録を可能にするため重要です。

関与するモジュール

  • module:include
  • module:server

主要エンティティ

  • **symbol:include/http_log.h#ap_log_perror**: ヘッダーファイルで定義されている公開API/マクロ。
  • **symbol:include/http_log.h#ap_log_perror__**: ログ記録を支援するための内部マクロまたはヘルパー。
  • **symbol:server/log.c#ap_log_perror_**: 実際のログ処理を行う実体関数。以下のシグネチャを持ちます。

    • AP_DECLARE(void) ap_log_perror_(const char *file, int line, int module_index, int level, apr_status_t status, apr_pool_t *p, const char *fmt, ...)

エンドツーエンドの流れ

1. 開発者が ap_log_perror (または ap_log_perror__) を呼び出します。 2. マクロ展開等を経由して、server/log.c に定義されている実体関数 ap_log_perror_ が呼び出されます。 3. ap_log_perror_ は、渡されたファイル名、行番号、モジュールインデックス、ログレベル、ステータス、プール、フォーマット文字列などの引数を引き連れて、内部関数 log_error_core を呼び出します。
