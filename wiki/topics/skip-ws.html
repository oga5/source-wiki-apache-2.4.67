skip_ws

概念

skip_ws は、HTTP/2 や ACME/TLS などのモジュールにおいて、Link ヘッダーなどの文字列バッファを解析する際に、現在の読み取り位置から連続する空白文字（スペースやタブ）をスキップして次の非空白文字へポインタを進めるためのヘルパー関数です。

なぜ重要か

HTTP ヘッダー（特に Link ヘッダー）の解析において、トークンやパラメータの間に存在する任意の空白文字（スペース ' ' やタブ '\t'）を適切に無視し、正確な構文解析を行うために重要です。これにより、堅牢なヘッダー処理が実現されます。

エンドツーエンドの流れ

Link ヘッダーなどの解析処理において、各トークン（リンク、パラメータ名、パラメータ値など）を読み取る前後に skip_ws が呼び出されます。

各モジュール間での詳細な連携フローや、システム全体におけるエンドツーエンドの完全な制御フローは、現時点ではまだ解析されていません。ただし、modules/http2/h2_push.c における基本的な処理の流れは以下の通りです：

1. link_ctx 構造体に解析対象の文字列（ctx->s）、現在のインデックス（ctx->i）、文字列長（ctx->slen）を格納して解析を開始する。 2. pv_contains や read_link などの呼び出し元関数が、次のトークンを処理する前に skip_ws(ctx) を呼び出す。 3. skip_ws 内で、現在のインデックス ctx->i が文字列長 ctx->slen 未満であり、かつ文字が ' ' または '\t' である間、インデックスをインクリメントする。 4. 文字列の終端に達したかどうかの判定結果（ctx->i < ctx->slen）を戻り値（真/偽）として返す。

重要な不変条件

  • modules/http2/h2_push.c の実装において、入力される ctx は有効なポインタでなければならず、インデックス ctx->i は常に 0 以上 ctx->slen 以下の範囲内にある必要があります。
  • modules/http2/h2_proxy_util.c および modules/md/md_util.c における具体的な前提条件や不変条件は、現時点ではまだ解析されていません。

よくある落とし穴

  • **終端のハンドリング**: 空白のみで構成される文字列や空文字列が渡された場合、ctx->i が ctx->slen に達して戻り値が 0（偽）となります。呼び出し元は、戻り値が 0 の場合に文字列の終端に達したことを正しくハンドリングする必要があります。
  • **重複定義**: 同名の static 関数が複数のソースファイル（h2_proxy_util.c、h2_push.c、md_util.c）に個別に定義されているため、共通の仕様変更を行う際には、すべての実装箇所への影響を考慮する必要があります。
