skip_ws

概念

skip_ws は、文字列解析処理において、現在の読み取り位置から連続する空白文字（スペース ' ' およびタブ '\t'）をスキップし、次の非空白文字へポインタ（インデックス）を進めるためのユーティリティ関数です。主に HTTP/2 関連の Link ヘッダー解析などで、トークン間の不要な空白を除去するために使用されます。

なぜ重要か

HTTP ヘッダー（特に Link ヘッダーなど）の解析において、仕様に準拠した正確なトークン抽出を行うためには、要素間の空白文字を適切に無視する必要があります。skip_ws は、解析状態を保持するコンテキストのインデックスを安全に進めることで、堅牢な構文解析を支えています。

エンドツーエンドの流れ

modules/http2 における具体的なエンドツーエンドの流れは以下の通りです。

1. Link ヘッダーなどの解析対象文字列が link_ctx 構造体に格納され、初期化されます。 2. 解析処理の過程で、各トークン（URIや属性など）の境界に達した際、skip_ws が呼び出されます。 3. skip_ws は、現在のインデックス ctx->i から連続するスペース ' ' またはタブ '\t' をループ処理でスキップし、インデックスを進めます。 4. 次の非空白文字の位置、または文字列の終端（ctx->slen）に達した時点でループを抜け、終端に達していない場合は真（非ゼロ）、達した場合は偽（ゼロ）を返します。 5. 呼び出し元は、この戻り値や更新されたインデックスを基に、次のトークンの抽出処理を継続します。

なお、modules/md における skip_ws の具体的なエンドツーエンドの流れについては、提供されたコンテキストからは未詳です。

重要な不変条件

  • **インデックスの境界条件**: ctx->i は常に 0 以上 ctx->slen 以下の範囲内に維持される必要があります。
  • **入力の妥当性**: ctx は有効なポインタでなければなりません。

よくある落とし穴

  • **文字列終端の判定**: 空白文字のみで文字列が終了した場合、ctx->i は ctx->slen と等しくなり、戻り値は 0（偽）を返します。呼び出し元がこの戻り値を適切にハンドリングしない場合、バッファオーバーランや不正な解析処理につながる恐れがあります。
  • **複数ファイル間での重複定義・参照**: h2_proxy_util.c と h2_push.c の両方から参照・定義されているため、一方の挙動や link_ctx 構造体の定義を変更する際は、他方への影響を慎重に考慮する必要があります。
