skip_ws

概念

skip_ws は、文字列解析において現在の読み取り位置から連続する空白文字（スペース ' ' およびタブ '\t'）をスキップし、次の非空白文字へポインタを進めるためのユーティリティ関数です。主に Link ヘッダーなどの解析において、トークン間の不要な空白を除去するために使用されます。

なぜ重要か

HTTP/2 などのプロトコル処理において、ヘッダー値を正確にパースするためには、仕様に準拠した空白のハンドリングが不可欠です。skip_ws は、解析対象の文字列バッファのインデックスを安全に進め、パース処理の堅牢性を担保する役割を担っています。

エンドツーエンドの流れ

Link ヘッダーなどの解析処理において、以下の流れで呼び出されます。

1. 解析対象の文字列、現在のオフセット（ctx->i）、および文字列長（ctx->slen）を保持する link_ctx 構造体が用意されます。 2. トークンの境界に達した際、skip_ws が呼び出されます。 3. skip_ws は、現在の位置からスペース ' ' またはタブ '\t' が続く限り、インデックス ctx->i をインクリメントします。 4. 次の非空白文字の位置、または文字列の終端（ctx->slen）に達した時点で処理を終了し、終端に達していなければ非ゼロ（真）、達していればゼロ（偽）を返します。

なお、modules/md/md_util.c#skip_ws を含む全体的なエンドツーエンドの流れや、他のモジュールとの詳細な連携フローについては、提供された情報からは未解明です。

重要な不変条件

  • link_ctx 構造体のポインタ ctx は常に有効（非ヌル）でなければなりません。
  • 解析インデックス ctx->i は、常に 0 以上 ctx->slen（文字列の長さ）以下の範囲内に維持されます。

よくある落とし穴

  • **終端のハンドリング**: 空白のみで構成された文字列や空文字列が渡された場合、ctx->i は ctx->slen と等しくなり、戻り値として 0（偽）が返されます。呼び出し側はこの戻り値を適切にハンドリングし、バッファ範囲外へのアクセスを防ぐ必要があります。
  • **複数箇所からの参照**: h2_proxy_util.c と h2_push.c の両方から参照されているため、共通の挙動を変更する際は両方のコンテキストへの影響を考慮する必要があります。
