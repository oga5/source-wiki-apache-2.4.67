skip_ws

概念

skip_ws は、文字列解析において現在の読み取り位置から連続する空白文字（スペース ' ' およびタブ '\t'）をスキップし、次の非空白文字へポインタを進めるためのユーティリティ関数です。主に HTTP/2 関連の Link ヘッダー解析などで使用されます。

なぜ重要か

HTTP ヘッダー（特に Link ヘッダー）の解析において、トークン間に存在する任意の空白文字を正確に読み飛ばすことは、プロトコル仕様に準拠した正しいパースを行うために不可欠です。この共通処理をカプセル化することで、コードの重複を防ぎ、解析処理の堅牢性を高めています。

エンドツーエンドの流れ

modules/http2 における具体的な流れは以下の通りです。 1. Link ヘッダーなどの解析対象文字列と、現在のオフセット位置 i、文字列長 slen を保持する link_ctx 構造体が用意されます。 2. skip_ws(ctx) が呼び出されます。 3. ctx->s[ctx->i] がスペース ' ' またはタブ '\t' である間、インデックス ctx->i をインクリメントします。 4. スキップ処理後、インデックスが文字列の終端に達していなければ真（非ゼロ）、達していれば偽（ゼロ）を返します。

なお、modules/md における skip_ws の具体的なエンドツーエンドの流れや、システム全体における詳細な統合フローは現時点では不明です。

重要な不変条件

  • modules/http2/h2_push.c#skip_ws において、入力となる link_ctx 構造体のポインタは有効でなければなりません。
  • 解析中のインデックス ctx->i は、常に 0 以上 ctx->slen（文字列の長さ）以下の範囲内に維持される必要があります。
  • その他のモジュール（modules/md など）における不変条件は現時点では不明です。

よくある落とし穴

  • **文字列終端の判定**: 空白文字のみで文字列が終了した場合、ctx->i は ctx->slen と等しくなり、戻り値は 0（偽）を返します。呼び出し側でこの戻り値を適切にハンドリングしないと、終端以降の不正なメモリアクセスや解析エラーに繋がる可能性があります。
  • **モジュール間の差異**: modules/http2 と modules/md のそれぞれに skip_ws が存在しますが、これらが同一のシグネチャや振る舞いを持つか、あるいは完全に独立した実装であるかは提供された情報からは特定できません。変更を加える際は、対象のモジュールに閉じた影響範囲であるかを確認する必要があります。
