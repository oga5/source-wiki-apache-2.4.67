skip_ws

概念

skip_ws は、解析対象の文字列バッファにおいて、現在の読み取り位置から連続する空白文字（スペース ' ' およびタブ '\t'）をスキップし、次の非空白文字へポインタを進めるためのユーティリティ関数です。主に Link ヘッダーなどの解析処理において、トークン間の不要な空白を除去するために使用されます。

なぜ重要か

HTTP/2 などのプロトコル処理において、Link ヘッダーなどのメタデータを正確にパースすることは、リソースのプッシュ（Server Push）やプロキシ連携を正しく行うために不可欠です。skip_ws は、仕様に準拠した柔軟なヘッダー解析を支える共通の低レベル処理を提供します。

エンドツーエンドの流れ

提供された情報からは、skip_ws が呼び出される全体的なエンドツーエンドの制御フロー（HTTP/2 リクエストの受信から Link ヘッダーのパース、プッシュのトリガーに至る詳細なシーケンス）の全容は完全には明らかになっていません。

判明している局所的な流れは以下の通りです： 1. Link ヘッダー解析などの処理中に、link_ctx 構造体（解析対象文字列 s、現在のインデックス i、文字列長 slen を保持）を伴って skip_ws が呼び出されます。 2. skip_ws は ctx->i が ctx->slen 未満である限り、ctx->s[ctx->i] がスペース ' ' またはタブ '\t' である間、インデックス ctx->i をインクリメントします。 3. スキップ処理後、読み取り位置が文字列の終端に達していなければ非ゼロ（真）を返し、達していればゼロ（偽）を返して呼び出し元に制御を戻します。

重要な不変条件

  • **インデックスの範囲**: ctx->i は常に 0 以上 ctx->slen 以下の範囲内に維持される必要があります。
  • **ポインタの有効性**: 引数として渡される link_ctx *ctx は、常に有効なポインタでなければなりません。

よくある落とし穴

  • **文字列の終端到達**: 空白文字のみで文字列が終了した場合、または空文字列が渡された場合、ctx->i は ctx->slen と等しくなり、関数は 0 を返します。呼び出し元はこの戻り値を適切にハンドリングし、バッファ外アクセスを防ぐ必要があります。
  • **複数ファイル間での共有**: h2_push.c で定義されている skip_ws は、h2_proxy_util.c からも参照されています。この関数の挙動や link_ctx の構造を変更する場合、両方のファイルにおける解析ロジックへの影響を考慮する必要があります。
  • **modules/md における実装**: modules/md/md_util.c にも skip_ws が存在しますが、その具体的な仕様や不変条件、modules/http2 側の実装との共通性・差異については現時点で情報が不足しています。
