dav_namespace_map

概念

dav_namespace_map は、WebDAV プロパティ管理において XML 名前空間 URI を内部的な数値 ID にマッピングするためのテーブル構造を保持する構造体です。

なぜ重要か

WebDAV では、プロパティが XML 名前空間によって識別されます。文字列である名前空間 URI をそのまま扱う代わりに、内部的な数値 ID にマッピングして管理することで、プロパティデータベースにおけるキーの構築（dav_build_key など）や検索処理を効率化します。

エンドツーエンドの流れ

詳細なエンドツーエンドの制御フローは完全には明らかになっていませんが、提供された情報から以下の流れが確認できます。 1. dav_propdb_map_namespaces 等の関数において、XML 名前空間 URI を内部 ID に変換する処理が行われます。 2. この変換の際、dav_namespace_map 構造体がマッピングテーブルとして参照・更新されます。 3. 変換された数値 ID は、dav_build_key などの関数でデータベースキーを構築する際などに利用されます。

重要な不変条件

  • ns_map ポインタは、apr_palloc 等によって適切にメモリ確保された整数配列を指している必要があります。
  • dav_build_key が使用する名前空間インデックスと、dav_namespace_map が保持するマッピングの整合性が常に保たれている必要があります。

よくある落とし穴

  • ns_map が NULL の状態でアクセスすると、セグメンテーションフォールトが発生する可能性があります。
  • メモリ管理は apr_pool_t に紐付いているため、プールおよびマッピングテーブルのライフサイクル管理を誤ると、ダングリングポインタやメモリリークの原因となります。
