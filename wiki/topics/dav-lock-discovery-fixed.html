dav_lock_discovery_fixed

概念

dav_lock_discovery_fixed は、WebDAVの直接ロック（direct lock）情報をDBMデータベースへ永続化・復元する際に、固定長属性を保持・シリアライズするためのデータ構造（struct）です。スコープ、ロックタイプ、深度、有効期限といった固定長の属性をバイナリ形式で扱うためのメモリレイアウトを定義します。

なぜ重要か

WebDAVのロック情報をDBMファイルに効率的に保存・復元するために、メモリ上の構造体をそのままバイナリデータとして扱う必要があります。この構造体は、シリアライズおよびデシリアライズ時の固定長ヘッダーとして機能し、データの永続化レイアウトを決定づける役割を果たします。

関与するモジュール

  • module:modules/dav/fs
  • module:modules/dav/lock

主要エンティティ

本トピックに関与する主要なエンティティは以下の通りです。

  • **symbol:modules/dav/fs/lock.c#dav_lock_discovery_fixed**
  • **symbol:modules/dav/lock/locks.c#dav_lock_discovery_fixed**

これらは、それぞれのファイルで以下のように定義されている同名の構造体です。

typedef struct dav_lock_discovery_fixed
{
    char scope;
    char type;
    int depth;
    time_t timeout;
} dav_lock_discovery_fixed;

エンドツーエンドの流れ

1. **ロック情報の保存（シリアライズ）** メモリ上のロック構造体（dav_lock_discovery）から、dav_lock_discovery_fixed の各フィールド（scope, type, depth, timeout）へ値がコピーされます。その後、dav_generic_save_lock_record などの処理において、DBMレコードのバイナリバッファの一部として書き込まれます。 2. **ロック情報の読み込み（デシリアライズ）** DBMから読み込まれたバイナリデータから、dav_lock_discovery_fixed へキャストまたはコピーされ、有効期限やスコープ、深さが復元されます（dav_generic_load_lock_record などの処理）。

※ なお、ファイルシステムロック処理（modules/dav/fs/lock.c）とロック管理ロジック（modules/dav/lock/locks.c）の間における、詳細なエンドツーエンドの呼び出しフロー全体は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **サイズ計算の同期**

    • dav_size_direct マクロにより計算されるサイズ（本構造体のサイズに1バイトを加えた値）が、シリアライズ時のデータ長やオフセット計算の基準として使用されます。そのため、本構造体の定義と dav_size_direct マクロの定義は常に同期していなければなりません。

よくある落とし穴

  • **バイナリ互換性の喪失**

    • 本構造体のサイズやフィールド順序を変更すると、既存のDBMファイルとの互換性が失われます。

  • **プラットフォーム依存性**

    • time_t や int のサイズ、およびコンパイラによる構造体のパディング（アライメント）がプラットフォーム間で異なる場合、DBMファイルの互換性やバイナリ列のオフセットに影響を与える可能性があります。

  • **重複定義の不整合**

    • modules/dav/fs/lock.c と modules/dav/lock/locks.c の両方に同名の構造体が定義されているため、一方に変更を加える際は、必ず両方のファイルで整合性を保つ必要があります。

次に読む場所

  • **modules/dav/fs/lock.c**

    • ファイルシステムバックエンドにおける dav_lock_discovery_fixed の定義と、dav_size_direct マクロによるサイズ計算を確認してください。

  • **modules/dav/lock/locks.c**

    • ロック管理ロジックにおける dav_lock_discovery_fixed の定義、および dav_generic_save_lock_record / dav_generic_load_lock_record によるシリアライズ・デシリアライズ処理を確認してください。
