dav_lock_discovery_fixed

概念

dav_lock_discovery_fixed は、WebDAVの直接ロック（direct lock）情報をDBMデータベースなどの永続ストレージにバイナリ形式で保存・復元する際に、固定長属性を保持するためのデータ構造（struct）です。

なぜ重要か

WebDAVのロック情報を永続化する際、メモリ上のロック構造体からバイナリ形式へシリアライズ・デシリアライズするための固定長ヘッダー/データレイアウトとして機能します。これにより、効率的なロック情報の保存と読み込みが可能になります。また、ファイルシステムバックエンドと汎用ロック管理ロジックの双方で一貫したデータ構造を維持するために重要な役割を果たしています。

関与するモジュール

  • module:modules/dav/fs
  • module:modules/dav/lock

主要エンティティ

  • **symbol:modules/dav/fs/lock.c#dav_lock_discovery_fixed**

    • ファイルシステムバックエンドにおける直接ロックの固定長属性を保持する構造体です。

  • **symbol:modules/dav/lock/locks.c#dav_lock_discovery_fixed**

    • 汎用ロック管理ロジックにおいて、DBMファイルへのシリアライズ/デシリアライズ時に使用される構造体です。

両ファイルにおいて、以下の同一のシグネチャで定義されています。

typedef struct dav_lock_discovery_fixed
{
    char scope;
    char type;
    int depth;
    time_t timeout;
} dav_lock_discovery_fixed;

エンドツーエンドの流れ

1. **ロック情報の保存（シリアライズ）** メモリ上のロック構造体（dav_lock_discovery など）から dav_lock_discovery_fixed の各フィールド（scope, type, depth, timeout）へ値がコピーされ、DBMレコードのバイナリバッファの一部として書き込まれます（dav_generic_save_lock_record など）。 2. **ロック情報の読み込み（デシリアライズ）** DBMから読み込まれたバイナリデータから本構造体へキャストまたはコピーされ、有効期限（timeout）やスコープ、深さ（depth）などの固定長属性が復元されます（dav_generic_load_lock_record など）。

※なお、ファイルシステムロック処理と汎用ロック処理の間における、詳細な呼び出しシーケンスや完全なエンドツーエンドの連携フローの全容は、提供されたコンテキストからは明らかになっていません。

重要な不変条件

  • **サイズ計算の同期**: dav_size_direct マクロにより、本構造体のサイズ（またはそれに1バイトを加えた値）がシリアライズ時のデータ長やオフセット計算に使用されます。このマクロの定義と構造体のサイズは常に同期していなければなりません。
  • **構造体定義の整合性**: modules/dav/fs/lock.c と modules/dav/lock/locks.c の両方に同名の構造体が定義されており、両者の間でフィールド構成やレイアウトの整合性が保たれている必要があります。

よくある落とし穴

  • **バイナリ互換性の喪失**: 本構造体のサイズやフィールド順序を変更すると、既存のDBMファイルとの互換性が失われ、保存されたロック情報を正しく読み込めなくなります。
  • **プラットフォーム依存性**: time_t や int のサイズ、およびコンパイラによる構造体のパディング（アライメント）はプラットフォームによって異なるため、異なる環境間でDBMファイルを移行する際に互換性の問題が発生する可能性があります。
  • **重複定義の同期漏れ**: 2つのファイルに同名の構造体が定義されているため、片方のみを変更するとモジュール間で不整合が発生します。変更を加える際は、必ず両方のファイルを更新する必要があります。
