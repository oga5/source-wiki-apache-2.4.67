WebDAV Repository and Lock Management (mod_dav)

概念

mod_dav は、Apache HTTP Server における WebDAV（Web-based Distributed Authoring and Versioning）プロトコルの実装を提供します。このシステムは、リソースのロック管理（排他制御）や、リソースに関連付けられたプロパティ（メタデータ）の永続化を担います。ファイルシステムベースのバックエンドでは、ロック情報を管理するために DBM などのロックデータベースが使用され、プロパティ情報はプロパティデータベースを介して管理されます。

なぜ重要か

WebDAV 環境において、複数クライアントによる同時編集時の競合を防ぐための「ロック管理」と、リソースのメタデータを保持する「プロパティ管理」は、データの整合性を維持するために不可欠です。これらが適切に機能しない場合、データの破損、意図しない上書き、あるいはメタデータの消失が発生する可能性があります。

関与するモジュール

本機能は以下のモジュール群によって構成されています。

  • module:modules/dav
  • module:modules/dav/fs
  • module:modules/dav/lock
  • module:modules/dav/main

主要エンティティ

本トピックに関与する主要な構造体は以下の通りです。

  • **symbol:modules/dav/fs/lock.c#dav_lock_private**

    • ロックデータベース内での検索や識別に使用される、ロックに関連付けられたプライベートなキー情報（apr_datum_t key）を保持する構造体です。

  • **symbol:modules/dav/fs/lock.c#dav_lockdb_private**

    • ファイルシステムベースの DAV ロックデータベース（DBM）へのアクセスに必要な状態情報（リクエスト、プール、パス、オープン状態、DB ハンドルなど）を保持するプライベート構造体です。

  • **symbol:modules/dav/fs/lock.c#dav_locktoken**

    • WebDAV ロックを一意に識別するための UUID（apr_uuid_t）を保持するラッパー構造体です。

  • **symbol:modules/dav/main/mod_dav.h#dav_propdb**

    • WebDAV プロパティデータベースへのアクセスおよび操作を管理するためのコンテキスト構造体です。

エンドツーエンドの流れ

提供されたソースコード情報からは、リクエストの受信からロック獲得・プロパティ更新、そしてレスポンス返却に至る詳細なエンドツーエンドの完全な制御フローは完全には明らかになっていません。ただし、部分的な処理の流れとして以下の動作が定義されています。

1. **ロックデータベースの操作**: dav_lockdb_private を用いて、設定されたパス（lockdb_path）の DBM を必要に応じて遅延オープン（opened フラグで制御）し、dav_lock_private の key（apr_datum_t）を用いてロック情報の検索や保存を行います。ロックの識別には dav_locktoken 内の UUID が使用されます。 2. **プロパティ操作**: リクエスト処理フロー（modules/dav/main/mod_dav.c など）において dav_propdb が初期化され、プロパティの検索や変更（validate/exec/commit/rollback）が行われた後、dav_close_propdb によって明示的にクローズされリソースが解放されます。

重要な不変条件

  • **メモリ管理の整合性**:

    • dav_lockdb_private におけるメモリ割り当ては、リクエストのライフサイクルと整合性を保つため、必ず構造体内の pool メンバ（apr_pool_t）を使用して行われなければなりません。

  • **プロパティDBのクローズ**:

    • dav_propdb を用いたプロパティ操作のライフサイクル終了時には、リソース解放のために dav_close_propdb による明示的なクローズが必要です。

  • **ロックデータベースの状態**:

    • dav_lockdb_private の db ハンドルが非 NULL であることは、データベースが正常に開かれていることを示します。

よくある落とし穴

  • **DBM 破損の影響**:

    • dav_lock_private の key が指す DBM 内のデータが破損している場合、後続のデータベース操作が失敗する可能性があります。

  • **未定義プロバイダーや接続失敗**:

    • dav_propdb の操作において、プロバイダーが未定義の場合やデータベースへの接続が確立できない場合、プロパティ操作全体が失敗します。

  • **読み取り専用エラー**:

    • 読み取り専用で開かれているロックデータベースに対して書き込み操作を試みるとエラーとなります。
