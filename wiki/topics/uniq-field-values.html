uniq_field_values

概念

uniq_field_values は、HTTP ヘッダー（主に Vary ヘッダー）の値を解析し、重複を除去して配列に格納するためのコールバック関数です。カンマ区切りのヘッダー値文字列をトークンに分割し、大文字小文字を区別せずに重複を排除した一意な値のリストを作成します。

なぜ重要か

HTTP 仕様において、Vary などのヘッダーは複数の同一フィールド名、または単一のフィールド内にカンマ区切りで複数の値を持つことができます。キャッシュ処理やプロトコル変換（HTTP/2など）において、これらの値を一意に整理（正規化）して処理することは、正確なキャッシュ制御や効率的なヘッダー処理を行うために重要です。

エンドツーエンドの流れ

1. 各モジュールのフィルター処理（modules/http/http_filters.c および modules/http2/h2_c2_filter.c）において、対象となるヘッダーテーブルに対して apr_table_do(uniq_field_values, ...) が呼び出されます。 2. apr_table_do は、ヘッダーテーブル内の各エントリに対して uniq_field_values を実行します。 3. uniq_field_values は、渡されたヘッダー値（val）をメモリプール上に複製し、カンマや空白を区切り文字としてトークンに分割します。 4. 分割された各トークンについて、すでに格納先配列（apr_array_header_t）に存在するかを ap_cstr_casecmp を用いて大文字小文字を区別せずに線形探索します。 5. 配列内に存在しないトークンのみを apr_array_push で配列に追加します。 6. すべてのヘッダー値の処理が完了すると、重複のない一意な値のリストが格納された配列が得られます。

なお、この処理の前後における具体的な上位の呼び出しシーケンスや、抽出された配列がその後どのように利用されるかといった完全なエンドツーエンドの流れは、提供されたコンテキストからは未だ完全には明らかになっていません。

重要な不変条件

  • **入力データの構造**: uniq_field_values の第1引数 d は、有効な apr_array_header_t 構造体へのポインタであり、その pool メンバが有効なメモリプールを指している必要があります。
  • **戻り値の不変性**: apr_table_do によるテーブル走査を継続させるため、本関数は常に 1 を返します。
  • **比較の不変性**: トークンの重複判定は、常に ap_cstr_casecmp を用いた大文字小文字を区別しない（case-insensitive）比較で行われます。

よくある落とし穴

  • **パフォーマンスの懸念**: 重複チェックは、配列内の既存要素に対する線形探索（ループ処理）によって行われます。ヘッダー値のトークン数が非常に多い場合、探索の計算量が累積し、パフォーマンスに影響を与える可能性があります。
  • **メモリ消費**: 解析対象の文字列は apr_pstrdup を用いてプール上に複製されるため、処理するヘッダーの量に応じてメモリ消費が増加します。
  • **キーの無視**: 本関数は apr_table_do のコールバック仕様に合わせるため key 引数を受け取りますが、関数内部では使用されず無視されます。
