max_workers

概念

max_workers は、Apache HTTP Server のマルチプロセッシングモジュール（MPM）である event および worker において、同時に動作可能な最大ワーカー数を管理するための設定値、およびそれを保持する変数です。

なぜ重要か

この値は、サーバーが同時に処理できるリクエストの最大数を決定し、システムの最大負荷やリソース（メモリやCPU）の消費量を制御するために極めて重要です。適切な制限を設けることで、過負荷によるサーバーのダウンを防ぎます。

関与するモジュール

  • module:server/mpm/event
  • module:server/mpm/worker

主要エンティティ

  • symbol:server/mpm/event/event.c#max_workers：event MPM において最大ワーカー数を保持する変数。
  • symbol:server/mpm/event/event.c#set_max_workers：event MPM において設定ファイル等から最大ワーカー数を設定するための関数。
  • symbol:server/mpm/worker/worker.c#max_workers：worker MPM において最大ワーカー数を保持する変数。
  • symbol:server/mpm/worker/worker.c#set_max_workers：worker MPM において設定ファイル等から最大ワーカー数を設定するための関数。

エンドツーエンドの流れ

設定ファイルの解析時に set_max_workers が呼び出され、設定された値が各 MPM の max_workers 変数に格納されると推測されます。しかし、具体的なパース処理や、実行時におけるスレッド生成・制限の適用といった詳細なエンドツーエンドの流れは、現時点では不明です。

重要な不変条件

max_workers が維持すべき他の設定値（例: ThreadsPerChild や ServerLimit など）との相関関係や、システムリソースに基づく最小値・最大値の制限などの具体的な不変条件は、現時点では不明です。

よくある落とし穴

設定値が極端に小さすぎる場合の接続拒否や、大きすぎる場合のメモリ枯渇など、一般的な挙動は予想されますが、提供されたソースコードのシンボル情報のみからは、具体的な落とし穴や制約は現時点では不明です。
