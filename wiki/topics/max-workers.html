max_workers

概念

max_workers は、Apache HTTP Server（httpd）のマルチプロセッシングモジュール（MPM）である event および worker において、同時に稼働できる最大ワーカー数を管理・保持するための変数、およびそれを設定するための仕組みです。

なぜ重要か

Webサーバーの並行処理能力とリソース消費を適切に制御するために重要です。event MPM および worker MPM はマルチスレッドモデルを採用しており、システムリソース（メモリやCPU）の枯渇を防ぎつつ、最大同時接続数を制限するために max_workers の値が利用されます。

関与するモジュール

  • module:server/mpm/event
  • module:server/mpm/worker

主要エンティティ

  • **symbol:server/mpm/event/event.c#max_workers**: event MPM における最大ワーカー数を保持する内部変数。
  • **symbol:server/mpm/event/event.c#set_max_workers**: event MPM において、設定ファイル等から max_workers の値を設定するための関数。
  • **symbol:server/mpm/worker/worker.c#max_workers**: worker MPM における最大ワーカー数を保持する内部変数。
  • **symbol:server/mpm/worker/worker.c#set_max_workers**: worker MPM において、設定ファイル等から max_workers の値を設定するための関数。

エンドツーエンドの流れ

設定ファイル（httpd.conf など）の解析時に、対応するディレクティブ（例: MaxRequestWorkers または旧 MaxClients）が検出されると、各MPMの set_max_workers 関数が呼び出され、内部変数 max_workers に値が格納されます。

ただし、この設定値がプロセスの起動やスレッドプールの生成、および実際のリクエスト処理時にどのように適用されるかという具体的なエンドツーエンドの制御フローの詳細は、現時点では不明です。

重要な不変条件

現時点では、max_workers が維持すべき具体的な不変条件（例: 他の設定値 ThreadsPerChild や ServerLimit との相関関係、許容される最小値・最大値の制約など）は不明です。

よくある落とし穴

現時点では、max_workers の設定値に起因する具体的な競合状態や、設定誤りによるリソース枯渇などの落とし穴の詳細は不明です。

  • **server/mpm/worker/worker.c**: worker MPM における max_workers の定義と set_max_workers による設定ロジック。
