on_event

概念

on_event は、Apache HTTP Server（httpd-2.4.67）の複数の独立したモジュール（mod_http2 および mod_md）において、イベント駆動型の処理や状態遷移、通知処理を行うために定義されている内部関数またはコールバック関数です。これらは同名ですが、それぞれ異なる目的と文脈で使用される独立したシンボルです。

なぜ重要か

  • **mod_http2 における重要性**: HTTP/2 ストリームの状態遷移ロジックをカプセル化し、発生したイベント（h2_stream_event_t）に応じた適切な状態遷移を保証するために不可欠です。また、外部モニターへのイベント通知も担います。
  • **mod_md における重要性**: ACME などのマネージドドメイン（Managed Domains）管理において、md_event サブシステムからのイベント通知を受け取り、関連するジョブの通知処理（notify）へ転送する役割を果たします。

関与するモジュール

  • module:modules/http2
  • module:modules/md

主要エンティティ

  • symbol:modules/http2/h2_stream.c#on_event: HTTP/2 ストリームの状態遷移およびモニター通知を行う関数。
  • symbol:modules/md/mod_md.c#on_event: md_event サブシステムからのイベントを notify 関数へ転送するコールバック関数。

エンドツーエンドの流れ

これら2つの関数は独立して動作するため、システム全体を横断する単一のエンドツーエンドの流れは存在しません。それぞれのモジュールにおける流れは以下の通りです。

mod_http2 における流れ

1. ストリームの状態遷移を伴う処理（h2_stream.c 内の各所、例: 399, 469, 523 行目など）から on_event が呼び出されます。 2. ストリームに monitor が登録されている場合、stream->monitor->on_event コールバックが呼び出され、イベントが通知されます。 3. イベント ev が状態遷移テーブル trans_on_event の範囲内にあるか検証されます。 4. 範囲内であれば、on_map を介して現在の stream->state とイベントから次の状態を算出して返します。範囲外の場合は現在の状態をそのまま返します。

mod_md における流れ

1. modules/md/mod_md.c:891 において、md_event_subscribe を通じて本関数がイベント購読者として登録されます。 2. md_event サブシステムからイベントが通知されると、本関数が呼び出されます。 3. 本関数は、受け取った引数（job、event、result、p、baton）をそのまま notify 関数に引き渡して実行し、その結果（apr_status_t）を返します。なお、mdomain 引数は無視されます。

重要な不変条件

  • **mod_http2**:
  • 関数開始時に H2_STRM_ASSERT_MAGIC(stream, H2_STRM_MAGIC_OK) が実行され、ストリーム構造体の整合性が常に検証されます。
  • イベント ev が定義範囲外（trans_on_event の範囲外）の場合、状態は遷移せず、現在の stream->state が維持されます。
  • **mod_md**:
  • mdomain 引数は関数内で使用されず、明示的にキャストされて無視されます。
  • イベント処理の結果は、呼び出し先の notify 関数の実行結果に完全に依存します。

よくある落とし穴

  • **名前の混同**: modules/http2 の on_event と modules/md の on_event は、名前は同じですが完全に異なるシグネチャと目的を持つ独立した関数です。これらを混同しないよう注意が必要です。
  • **mod_http2 における変更**: 状態遷移ルールを変更する際、on_event 自体ではなく、状態遷移テーブル trans_on_event の定義を適切に更新する必要があります。
  • **mod_md における拡張**: 本関数は現状、notify への単なるパススルー（転送）として実装されています。将来的にイベントのフィルタリングやログ出力を追加する場合は、この関数内に実装を追加する必要があります。
