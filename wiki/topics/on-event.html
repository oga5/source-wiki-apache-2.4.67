on_event

概念

on_event は、Apache HTTP Server 内の異なるモジュール（mod_http2 および mod_md）において、それぞれのコンテキストで発生したイベントを処理するために定義されている、同名の独立した静的（static）関数です。

  • **HTTP/2 ストリーム領域**: HTTP/2 ストリームの状態遷移を管理し、イベントに応じた次の状態を決定します。
  • **Managed Domains (MD) 領域**: md_event サブシステムからのイベント通知を受け取り、関連するジョブの通知処理へ転送します。

なぜ重要か

それぞれのモジュールにおいて、イベント駆動型の処理や状態管理をカプセル化するために重要な役割を果たしています。

  • **mod_http2 における重要性**: HTTP/2 ストリームの複雑な状態遷移ロジックを1箇所に集約し、プロトコルの整合性を保ちながら安全に状態を遷移させるために不可欠です。
  • **mod_md における重要性**: 証明書管理などのバックグラウンドジョブで発生するイベントを検知し、適切な通知処理（notify）へと仲介・パススルーするために必要となります。

関与するモジュール

  • module:modules/http2
  • module:modules/md

主要エンティティ

  • **symbol:modules/http2/h2_stream.c#on_event**: HTTP/2 ストリーム（h2_stream）のイベント処理と状態遷移を担当する関数。
  • **symbol:modules/md/mod_md.c#on_event**: md_event サブシステムからのイベントを notify 関数へ転送するコールバック関数。

エンドツーエンドの流れ

これら2つの関数は、それぞれのモジュール内で独立して動作しており、モジュール間を横断する共通のエンドツーエンドフローは存在しません。それぞれの個別フローは以下の通りです。

mod_http2 における流れ

1. ストリームの状態遷移を伴う各処理（h2_stream.c 内の各所）から on_event が呼び出されます。 2. ストリームに監視用モニター（stream->monitor）が登録されている場合、その on_event コールバックを呼び出してイベントを通知します。 3. イベントが状態遷移テーブル trans_on_event の範囲内である場合、on_map 関数を呼び出して現在の状態（stream->state）とイベントから次の遷移先状態を計算して返します。

mod_md における流れ

1. mod_md.c:891 において、md_event_subscribe に対し本関数がコールバックとして登録されます。 2. md_event サブシステムからイベントが通知されると、本関数が呼び出されます。 3. 入力された mdomain 引数を無視し、受け取った job、event、result、p、baton をそのまま notify 関数に引き渡してその結果を返します。

重要な不変条件

  • **mod_http2**:
  • 処理の開始時に H2_STRM_ASSERT_MAGIC(stream, H2_STRM_MAGIC_OK) が実行され、対象となるストリーム構造体の整合性が常に検証されます。
  • **mod_md**:
  • mdomain 引数は関数内部で明示的にキャストされて破棄され、使用されません。
  • notify 関数が正常に呼び出せる状態である必要があります。

よくある落とし穴

  • **同名関数の混同**: modules/http2/h2_stream.c の on_event と modules/md/mod_md.c の on_event は、名前が同一であるだけの全く異なる静的関数です。これらに関連性はありません。
  • **mod_http2 での範囲外イベント**: 発生したイベントが trans_on_event テーブルの定義範囲外である場合、状態遷移は行われず現在の状態がそのまま返されます。意図しないイベントが渡された場合の挙動に注意してください。
  • **mod_md でのエラー伝播**: 本関数は notify 関数の単なるパススルーとして動作するため、notify が失敗した場合はそのエラーコードがそのまま呼び出し元に返されます。
