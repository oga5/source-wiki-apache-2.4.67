on_event

概念

on_event は、Apache HTTP Server の複数の独立したモジュール（mod_http2 および mod_md）において、特定のイベントが発生した際に呼び出される内部関数（コールバックまたは状態遷移ハンドラ）の名称です。これらは同名ですが、それぞれ異なる目的と文脈で実装されています。

  • **HTTP/2 ストリーム管理 (mod_http2)**: HTTP/2 ストリームの状態遷移を制御するためのイベントハンドラ。
  • **Managed Domains (mod_md)**: md_event サブシステムからのイベント通知を受け取り、関連するジョブ通知処理（notify）へ転送するためのコールバック。

なぜ重要か

それぞれのモジュールにおいて、非同期または動的なイベント駆動型の処理を適切に制御するために重要です。

  • **mod_http2 における重要性**: ストリームのライフサイクル（状態遷移）を正しく管理し、HTTP/2 プロトコルの整合性を維持するために不可欠です。
  • **mod_md における重要性**: 証明書管理などのバックグラウンドジョブの進捗や結果を、適切な通知処理（notify）へ確実に連携するために機能します。

関与するモジュール

  • module:modules/http2
  • module:modules/md

主要エンティティ

  • symbol:modules/http2/h2_stream.c#on_event: HTTP/2 ストリームの状態遷移を処理する静的関数。
  • symbol:modules/md/mod_md.c#on_event: md_event サブシステムからのイベントを notify へパススルーする静的コールバック関数。

エンドツーエンドの流れ

これら2つの on_event は完全に独立して動作し、システム全体を横断する単一のエンドツーエンドフローは存在しません。それぞれの個別フローは以下の通りです。

HTTP/2 ストリームイベントフロー (mod_http2)

1. ストリームの状態遷移を伴う処理（h2_stream.c 内の各所）から on_event が呼び出されます。 2. ストリームに monitor が設定されている場合、stream->monitor->on_event コールバックが実行され、イベントが通知されます。 3. イベント ev が状態遷移テーブル trans_on_event の範囲内であるか確認します。 4. 範囲内であれば、on_map を呼び出して現在の stream->state とイベントに対応する遷移先を計算して返します。範囲外の場合は現在の状態を維持します。

MD イベント通知フロー (mod_md)

1. modules/md/mod_md.c:891 において、md_event_subscribe に on_event が登録されます。 2. md_event サブシステムからイベントが通知されると、on_event が呼び出されます。 3. on_event は受け取った引数のうち mdomain を無視し、残りの引数（job、event、result、p、baton）をそのまま notify 関数に引き渡して、その実行結果を返します。

重要な不変条件

  • **mod_http2**:
  • 関数呼び出し時に H2_STRM_ASSERT_MAGIC(stream, H2_STRM_MAGIC_OK) により、ストリーム構造体の整合性が検証されている必要があります。
  • イベントが trans_on_event の定義範囲外の場合、状態遷移は行われず、現在の状態が維持されます。
  • **mod_md**:
  • mdomain 引数は関数内で使用されず、明示的にキャストされて無視されます。
  • notify 関数が正常に呼び出せる状態である必要があります。

よくある落とし穴

  • **名前の混同**: modules/http2/h2_stream.c の on_event と modules/md/mod_md.c の on_event は、名前は同じですが完全に無関係な別個のシンボルです。コードリーディングやデバッグの際にこれらを混同しないよう注意が必要です。
  • **mod_http2 の遷移テーブル**: 状態遷移ルールを変更する場合、on_event 自体ではなく trans_on_event テーブルの定義を修正する必要があります。
  • **mod_md のパススルー**: 現在の mod_md の on_event は単なるパススルー（転送）として実装されています。将来的にイベント処理のフィルタリングやログ出力を追加する場合は、この関数内に実装を挟む必要があります。
