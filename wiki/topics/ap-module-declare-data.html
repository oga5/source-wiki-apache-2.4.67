AP_MODULE_DECLARE_DATA

概念

AP_MODULE_DECLARE_DATA は、Apache HTTP Server（httpd-2.4.67）において、各モジュールが自身の module 構造体などのデータシンボルを宣言・定義する際に付与するマクロです。

このマクロは、プラットフォーム固有のコンパイル属性（例えば、Windows環境におけるDLLからのシンボルエクスポート用の __declspec(dllexport) や、インポート用の __declspec(dllimport) など）を抽象化し、マルチプラットフォームでのビルドと動的リンクを可能にします。

なぜ重要か

Apache HTTP Serverは、コア機能に対して動的にモジュールを追加・ロードするDSO（Dynamic Shared Object）アーキテクチャを採用しています。

Windowsなどの特定のオペレーティングシステムでは、共有ライブラリ（DLL）からデータシンボルを外部に公開するために明示的なエクスポート宣言が必要です。AP_MODULE_DECLARE_DATA を用いてモジュール構造体を宣言することで、ビルド環境の違いを吸収し、mod_so などのロード機構がシンボルを正しく解決してモジュールをロードできるようになります。

主要エンティティ

  • **symbol:include/ap_config.h#AP_MODULE_DECLARE_DATA**: プラットフォームやビルド設定に応じて、適切なストレージクラス属性（エクスポート/インポート属性など）にマクロを展開する定義元です。
  • **symbol:include/httpd.h#AP_MODULE_DECLARE_DATA**: モジュール構造体の宣言や関連ヘッダーで参照される定義です。
  • **各モジュールファイル内の AP_MODULE_DECLARE_DATA**:
  • symbol:modules/aaa/mod_auth_basic.c#AP_MODULE_DECLARE_DATA
  • symbol:modules/ssl/mod_ssl.c#AP_MODULE_DECLARE_DATA
  • symbol:modules/core/mod_so.c#AP_MODULE_DECLARE_DATA
  • symbol:server/mpm/event/event.c#AP_MODULE_DECLARE_DATA

これらをはじめとする各Cファイル内で、以下のようにモジュール構造体の実体定義に付与されます。 ``c module AP_MODULE_DECLARE_DATA auth_basic_module = { ... }; ``

エンドツーエンドの流れ

1. **プリプロセス段階**: コンパイル時、ターゲットプラットフォーム（Unix、Windows、NetWare等）およびビルド構成（静的リンクか動的共有オブジェクトか）に応じて、include/ap_config.h 内で AP_MODULE_DECLARE_DATA が適切な属性に展開されます。 2. **コンパイル・リンク段階**: 各モジュール（例: mod_auth_basic.c）がコンパイルされ、auth_basic_module などのシンボルがエクスポートテーブルに登録されます。 3. **実行時のロード段階**: Apacheの起動時または設定再読み込み時、mod_so（modules/core/mod_so.c）が設定ファイル（httpd.conf）の LoadModule 指令に基づき、共有ライブラリからこのエクスポートされたモジュール構造体シンボルを検索・ロードし、Apacheコアに登録します。

*注意: プラットフォームごとの詳細なマクロ展開の全分岐や、リンカレベルでのシンボル解決の厳密な挙動は、提供されたソースコードメタデータのみからは完全には特定されていません。*

重要な不変条件

  • **シンボルの可視性**: 動的にロードされるすべてのモジュールは、Apacheコアや mod_so から参照可能にするため、モジュール構造体の定義時に必ず AP_MODULE_DECLARE_DATA を付与して宣言しなければなりません。
  • **一貫性**: 同一プラットフォーム上でのビルドにおいて、マクロが展開される属性は、モジュールをコンパイルする際と、それをインポートする側（コア側）で一貫している必要があります。

*注意: ビルドシステム（AutotoolsやCMakeなど）が提供するプリプロセッサマクロと本マクロの連動関係における詳細な不変条件は、現時点では完全には特定されていません。*

よくある落とし穴

  • **マクロの付与漏れ**:

    • 新規にカスタムモジュールを作成する際、module 構造体の定義に AP_MODULE_DECLARE_DATA を付与し忘れると、Unix系環境では動作しても、Windows環境などの特定のプラットフォームでシンボルがエクスポートされず、起動時に「シンボルが見つからない」といったロードエラーが発生します。

  • **他の宣言マクロとの混同**:

    • 関数を宣言するための AP_DECLARE や AP_DECLARE_NONSTD などのマクロと混同し、データシンボルに対して誤ったマクロを適用すると、呼び出し規約（Calling Convention）やリンクエラーの原因になります。

次に読む場所

  • **include/ap_config.h**: プラットフォームごとのマクロの具体的な定義と展開ロジックを確認できます。
  • **include/httpd.h**: module 構造体の定義と、本マクロがどのように適用されているかを確認できます。
  • **modules/core/mod_so.c**: エクスポートされたモジュールシンボルがどのように動的にロードされるかを確認できます。
