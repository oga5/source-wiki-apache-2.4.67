ap_daemons_to_start

概念

ap_daemons_to_start は、Apache HTTP Server の各種マルチプロセッシングモジュール（MPM）において、サーバー起動時に生成すべき初期子プロセス（デーモン）の数を保持・管理するための変数です。

なぜ重要か

Apache の起動直後からクライアントからのリクエストに対して迅速に応答するためには、あらかじめ適切な数のプロセスを起動しておく必要があります。この変数は、設定ファイル（通常は StartServers ディレクティブなど）から読み込まれた初期プロセス数を保持し、各 MPM が起動処理を行う際の基準値として機能するため重要です。

関与するモジュール

本変数に関与するモジュールは以下の通りです。

  • module:server/mpm/event
  • module:server/mpm/mpmt_os2
  • module:server/mpm/prefork
  • module:server/mpm/worker

主要エンティティ

各 MPM のソースコードにおいて、以下のシンボルとして定義・参照されています。

  • symbol:server/mpm/event/event.c#ap_daemons_to_start
  • symbol:server/mpm/mpmt_os2/mpmt_os2.c#ap_daemons_to_start
  • symbol:server/mpm/prefork/prefork.c#ap_daemons_to_start
  • symbol:server/mpm/worker/worker.c#ap_daemons_to_start

エンドツーエンドの流れ

現時点では、設定ファイルのパースからプロセス生成に至る具体的なエンドツーエンドのコールフローや詳細な実行シーケンスは完全には判明していません。

一般的には、サーバーの初期化フェーズにおいて設定ディレクティブから値が ap_daemons_to_start に格納され、その後各 MPM のメインループまたはプロセス生成処理がこの値を参照して、指定された数の子プロセスをフォーク（起動）する流れになると推測されます。

重要な不変条件

提供されたモジュール情報からは、ap_daemons_to_start に関する具体的な不変条件（設定可能な最小値・最大値の制限や、他のスレッド関連設定値との依存関係など）は明らかになっていません。

よくある落とし穴

各 MPM（prefork、worker、event、mpmt_os2）によって、1プロセスあたりのスレッド数やリクエスト処理モデルが大きく異なります。そのため、ap_daemons_to_start（起動プロセス数）に同じ値を設定しても、システム全体で同時に処理できる初期リクエスト数や消費されるメモリ量が MPM ごとに大きく異なる点に注意が必要です。具体的なコードレベルでの制約や落とし穴については、現時点では特定されていません。
