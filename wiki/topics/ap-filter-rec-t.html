ap_filter_rec_t

概念

ap_filter_rec_t は、Apache HTTP Server におけるフィルタの定義情報を保持する構造体です。フィルタの名前、実行関数、型、およびプロトコル制約などのメタデータを管理し、フィルタチェーンへの登録と識別を可能にします。内部的には next ポインタを含んでおり、登録されたフィルタのリスト構造を形成します。

なぜ重要か

この構造体は、フィルタチェーン内でのフィルタの順序付けや、特定のフィルタハンドルの検索において中心的な役割を果たします。ap_core_output_filter_handle や ap_http_input_filter_handle といったグローバルなフィルタハンドルとして外部公開され、各モジュール（mod_deflate など）が自身の動作を定義・登録する際の基盤となります。

エンドツーエンドの流れ

フィルタの登録から実行に至る完全なエンドツーエンドのフローは、提供されたソース情報からはすべて明らかになっていません。現時点で判明している流れは以下の通りです。

1. **登録**: ap_register_input_filter や ap_register_output_filter_protocol などの登録関数により、ap_filter_rec_t が初期化・参照され、システムに登録されます。 2. **チェーン構築**: 登録されたフィルタは next メンバを介してリスト化され、フィルタチェーン構築時の参照情報として利用されます。 3. **実行**: フィルタチェーンの実行時に、本構造体が保持する関数ポインタが呼び出され、apr_bucket_brigade を介したデータ処理が行われます。

重要な不変条件

  • フィルタは ap_filter_type に基づいてソートされます。
  • AP_FTYPE_CONNECTION 以上のフィルタはサブリクエスト内では使用できません。

よくある落とし穴

  • フィルタの登録順序や型定義（ap_filter_type）が不適切な場合、フィルタチェーンの実行順序が意図しない状態になる可能性があります。
  • フィルタハンドルが正しく初期化されていない状態でチェーンに追加しようとした場合の挙動や詳細なエラーハンドリングについては、提供された情報からは特定されていません。

次に読む場所

  • include/util_filter.h：ap_filter_rec_t の構造体定義、および関連する登録インターフェースが宣言されています。
  • modules/http/http_core.c：HTTPプロトコル固有のフィルタハンドルの定義と利用方法が記述されています。
  • server/core.c：コアフィルタの登録と管理の仕組みが実装されています。
  • server/protocol.c：プロトコル処理におけるフィルタの適用フローが記述されています。
