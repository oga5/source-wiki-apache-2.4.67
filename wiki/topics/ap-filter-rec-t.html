ap_filter_rec_t

概念

ap_filter_rec_t は、Apache HTTP Server におけるフィルタの定義情報を保持する構造体です（include/util_filter.h）。フィルタの名前、実行関数、型、およびプロトコル制約などのメタデータを管理し、フィルタチェーンへの登録と識別を可能にします。内部に struct ap_filter_rec_t *next ポインタを持ち、登録済みフィルタのリスト構造を形成します。

なぜ重要か

Apacheのフィルタ機構において、フィルタチェーン内でのフィルタの順序付けや、特定のフィルタハンドルの検索を行うための基盤となるためです。ap_core_output_filter_handle や ap_http_input_filter_handle といったグローバルなフィルタハンドルとして外部公開され、各フィルタの具体的な実装（例: mod_deflate など）やコア機能が自身の動作を定義・登録する際に参照されます。

エンドツーエンドの流れ

1. **登録**: ap_register_input_filter や ap_register_output_filter_protocol などの登録関数により、ap_filter_rec_t が初期化され、システムへ登録されます。 2. **チェーン構築**: 登録された ap_filter_rec_t のリスト構造（next ポインタによる連結）を基に、フィルタチェーンが構築されます。 3. **実行**: フィルタチェーンの実行時に、本構造体が保持する関数ポインタが呼び出され、apr_bucket_brigade を介してデータが処理されます。

※ 登録から実行に至る詳細な制御フローの全容は、提供されたソース抜粋からは完全には特定されていません。

重要な不変条件

  • フィルタは ap_filter_type に基づいてソートされます。
  • AP_FTYPE_CONNECTION 以上のフィルタはサブリクエスト内では使用できません。

よくある落とし穴

  • フィルタの登録順序や型定義（ap_filter_type）が不適切な場合、フィルタチェーンの実行順序が意図した通りにならない可能性があります。
  • フィルタハンドルが正しく初期化されていない状態でチェーンに追加しようとした場合の挙動については、提供された情報からは特定されていません。
