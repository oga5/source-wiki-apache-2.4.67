is_graceful

概念

is_graceful は、Apache HTTP Server (httpd-2.4.67) の特定のマルチプロセッシングモジュール (MPM) において、サーバーが「Graceful（優雅な）」シャットダウンまたは再起動の状態にあるかどうかを追跡・判定するために使用されるフラグまたは変数です。これにより、実行中のリクエストやアクティブな接続を強制切断することなく、安全にプロセスを終了または再起動させる制御が行われます。

なぜ重要か

Webサーバーの運用において、設定の再読み込みやメンテナンス時にクライアントとの接続を突然切断することは避けるべきです。is_graceful の状態を適切に管理・参照することで、新規接続の受付を停止しつつ、既存の処理中リクエストが完了するのを待ってから安全にプロセスを終了する「Graceful」な移行プロセスを実現できます。

関与するモジュール

本トピックに関与するモジュールは以下の通りです。

  • module:server/mpm/mpmt_os2 (OS/2 プラットフォーム向けのマルチスレッド・マルチプロセス MPM)

  • symbol:server/mpm/mpmt_os2/mpmt_os2_child.c#is_graceful : OS/2 MPM の子プロセス側における Graceful 状態の参照・管理
  • symbol:server/mpm/netware/mpm_netware.c#is_graceful : NetWare MPM における Graceful 状態の管理

エンドツーエンドの流れ

現時点では、シグナル受信から is_graceful がセットされ、各スレッドやプロセスが終了するまでの具体的なエンドツーエンドの制御フローは完全には判明していません。

一般的には、親プロセスが再起動やシャットダウンのシグナルを検知した際に is_graceful が有効化され、各ワーカースレッドや子プロセスがこのフラグを検知して新規接続の受け入れ（accept）を停止し、既存の接続処理が完了した段階で順次終了していく流れを辿ると推測されます。

重要な不変条件

現時点では、is_graceful の状態遷移や、プロセス・スレッド間での同期に関する具体的な不変条件（インバリアント）は判明していません。

よくある落とし穴

現時点では、OS/2 や NetWare などの特定プラットフォームにおける is_graceful 制御特有のバグや、シグナルハンドリングにおける競合状態などの具体的な落とし穴は判明していません。
