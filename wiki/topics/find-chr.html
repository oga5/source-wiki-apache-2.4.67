find_chr

概念

find_chr は、指定された解析コンテキスト（link_ctx）の現在位置から、特定の文字 c を検索するためのユーティリティ関数です。主に Link ヘッダーなどの文字列解析において、クォートや区切り文字を探索するために利用されます。

なぜ重要か

この関数は、HTTP/2 関連のプッシュ機能やプロキシユーティリティ、および Managed Domains（MD）モジュールにおいて、文字列パース処理（クォート文字列の読み込みやスキップなど）を共通のパターンで安全に行うための基礎的なビルディングブロックとして機能しています。

関与するモジュール

  • module:modules/http2
  • module:modules/md

主要エンティティ

各モジュールで同名の静的関数として定義されており、シグネチャや型に若干の差異があります。

  • **symbol:modules/http2/h2_proxy_util.c#find_chr**
  • シグネチャ: static int find_chr(link_ctx *ctx, char c, int *pidx)
  • **symbol:modules/http2/h2_push.c#find_chr**
  • シグネチャ: static int find_chr(link_ctx *ctx, char c, size_t *pidx)
  • **symbol:modules/md/md_util.c#find_chr**
  • シグネチャ: static unsigned int find_chr(link_ctx *ctx, char c, apr_size_t *pidx)

エンドツーエンドの流れ

1. **呼び出し**: read_link、read_qstring、skip_qstring などの上位のパース関数から呼び出されます。 2. **線形探索**:

  • ctx->i（現在のオフセット位置）から ctx->slen（文字列全体の長さ）までループを回します。
  • 文字列 ctx->s の各要素を順次確認し、引数 c と一致する文字を探索します。

    • 3. **結果の返却**:

  • 一致する文字が見つかった場合、そのインデックスを *pidx に格納し、成功を示す値（1 など）を返して即座に終了します。
  • 文字列の末尾まで見つからなかった場合は 0 を返します。

※ なお、システム全体における完全なエンドツーエンドのデータフローや、各モジュール間での詳細な連携フローは現時点では完全には判明していません。

重要な不変条件

  • **引数の有効性**: ctx は有効な link_ctx 構造体を指し、pidx は書き込み可能なメモリ領域を指している必要があります。
  • **検索開始位置**: 検索範囲は常に ctx->i から開始されるため、呼び出し側で ctx->i が適切に管理・更新されている必要があります。
  • その他の詳細な不変条件はまだ解析されていません。

よくある落とし穴

  • **境界条件**: ctx->i がすでに ctx->slen 以上である場合、ループは実行されずに 0 が返されます。呼び出し側でインデックスの境界チェックや更新を誤ると、意図しないパース失敗の原因になります。
  • **型の不一致**: モジュールごとに戻り値の型（int / unsigned int）や、インデックスを格納するポインタの型（int * / size_t * / apr_size_t *）が微妙に異なるため、コードを移植・参照する際には注意が必要です。
