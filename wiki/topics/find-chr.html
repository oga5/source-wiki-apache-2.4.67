find_chr

概念

find_chr は、Apache HTTP Server の複数モジュール（http2 および md）において、Link ヘッダーなどの文字列解析を行うために使用されるユーティリティ関数です。解析コンテキスト（link_ctx）が保持する文字列の現在位置から、指定された特定の文字（クォートや区切り文字など）を線形探索します。

なぜ重要か

HTTP/2 のサーバープッシュ（h2_push.c、h2_proxy_util.c）や、Managed Domains（md_util.c）における Link ヘッダーのパース処理において、特定の区切り文字やクォート文字を正確に検出するために使用されます。これにより、ヘッダー値のトークン切り出しや属性解析が正しく行われます。

関与するモジュール

  • module:modules/http2
  • module:modules/md

主要エンティティ

  • **symbol:modules/http2/h2_push.c#find_chr**: link_ctx 構造体を用いて、指定された文字を検索する静的関数。
  • **symbol:modules/http2/h2_proxy_util.c#find_chr**: HTTP/2 プロキシユーティリティにおける find_chr の呼び出し、または同様の文字検索処理。
  • **symbol:modules/md/md_util.c#find_chr**: Managed Domains ユーティリティにおける find_chr の呼び出し、または同様の文字検索処理。

エンドツーエンドの流れ

1. Link ヘッダーなどの解析対象文字列と現在のオフセット（i）を保持する link_ctx 構造体が初期化されます。 2. 解析処理の中で、特定の文字（例: , や ;、" など）の位置を特定するために find_chr が呼び出されます。 3. find_chr は ctx->i から ctx->slen（文字列長）までループを回し、一致する文字を探索します。 4. 一致する文字が見つかった場合、そのインデックスを pidx に格納して 1 を返し、呼び出し側はこれを利用して文字列を切り出します。 5. 文字が見つからない場合は 0 を返します。

※なお、各モジュール（http2、md）における具体的な呼び出し元から全体のパース処理に至る完全なエンドツーエンドの制御フローの詳細は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • ctx は有効な link_ctx 構造体を指している必要があります。
  • pidx は書き込み可能なメモリ領域を指している必要があります。
  • 検索範囲は常に ctx->i（現在のオフセット）から開始され、ctx->slen（文字列長）を超えない範囲で実行されます。

よくある落とし穴

  • ctx->i（現在のオフセット）が ctx->slen 以上の場合、ループは実行されず即座に 0 を返します。呼び出し側で ctx->i のインクリメントや更新が適切に行われていない場合、無限ループや意図しない解析漏れが発生する可能性があります。
  • 検索対象の文字が見つからなかった場合、pidx の値は更新されず 0 が返されるため、呼び出し側は戻り値を必ず確認する必要があります。
