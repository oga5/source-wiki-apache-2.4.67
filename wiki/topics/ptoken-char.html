ptoken_char

概念

ptoken_char は、HTTP/2 の Link ヘッダーなどの解析において、対象の文字がトークンの一部として許容される文字（英数字および特定の記号）であるかを判定する関数です。

なぜ重要か

HTTP/2 のプッシュ機能やプロキシ処理、およびマネージドドメイン（MD）モジュールにおいて、ヘッダー値からトークンの境界を特定するための走査処理（for ループの条件式など）で頻繁に使用され、正確な構文解析を支えています。

エンドツーエンドの流れ

詳細なエンドツーエンドのパースフロー全体は提供された情報からは未解明ですが、部分的な処理の流れは以下の通りです。

1. **ヘッダー走査**: h2_push.c、h2_proxy_util.c、md_util.c 内の解析処理において、ヘッダー文字列を1文字ずつ走査します。 2. **文字判定**: 各文字に対して ptoken_char(c) を呼び出します。 3. **判定アルゴリズム**:

  • 入力文字 c が以下の記号セットに含まれるか判定します。

    • !, #, $, &, ', (, ), *, +, -, ., /, :, <, =, >, ?, @, [, ], ^, _, ` `, {, |, }, ~`

  • 含まれる場合は 1 を返します。
  • 含まれない場合は apr_isalnum(c) を呼び出し、英数字であれば 1、それ以外は 0 を返します。

    • 4. **境界特定**: 戻り値が 0 になった位置をトークンの境界として処理を継続します。

重要な不変条件

  • **ロケール依存性**: 内部で apr_isalnum を使用しているため、英数字の判定結果が実行環境のロケール設定に依存する可能性があります。
  • **仕様との整合性**: 許容される記号セットは、HTTP/2 の仕様（RFC 7540）および関連する Link ヘッダーの構文定義と整合している必要があります。

よくある落とし穴

  • **型境界の挙動**: 入力 c が char 型の範囲外である場合の挙動は、APR ライブラリの apr_isalnum の実装に依存します。
  • **仕様変更時の影響**: 判定対象の記号を変更（追加・除外）すると、複数のモジュール（http2 および md）のパース挙動に横断的な影響を与えるため、慎重な変更が必要です。
