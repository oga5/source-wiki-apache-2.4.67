ptoken_char

概念

ptoken_char は、HTTP/2 の Link ヘッダーなどの解析において、対象の文字がトークンの一部として許容される有効な文字であるかどうかを判定するユーティリティ関数です。

なぜ重要か

HTTP ヘッダーの構文解析において、トークンの境界を正しく識別し、仕様に準拠したパース処理を安全かつ正確に行うために必要となります。

  • symbol:modules/md/md_util.c#ptoken_char （呼び出し元）

エンドツーエンドの流れ

ヘッダー解析処理におけるエンドツーエンドの完全な制御フローは、提供された情報からは明らかになっていません。

判明している範囲では、Link ヘッダー等の解析時にトークンの境界を特定するための走査処理（for ループの条件式など）の中で本関数が呼び出され、文字がトークンの一部であるかどうかが1文字ずつ判定されます。

重要な不変条件

  • **許容文字の定義**: 以下のいずれかの条件を満たす文字に対してのみ 1 を返し、それ以外は 0 を返します。
  • 指定された記号セット：!, #, $, &, ', (, ), *, +, -, ., /, :, <, =, >, ?, @, [, ], ^, _, ` `, {, |, }, ~`
  • apr_isalnum(c) が真を返す英数字
  • **ロケール依存性**: 内部で APR ライブラリの apr_isalnum を使用しているため、判定結果が実行環境のロケール設定に依存する可能性があります。

よくある落とし穴

  • **範囲外の入力**: 入力 c が char 型の範囲外である場合の挙動は、内部で呼び出している apr_isalnum の実装に依存します。
  • **仕様との不整合**: 判定対象の記号を変更（追加・除外）する場合、HTTP/2 の仕様（RFC 7540）や Link ヘッダーの構文定義との整合性が失われる可能性があります。
