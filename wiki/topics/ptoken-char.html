ptoken_char

概念

ptoken_char は、HTTP/2 の Link ヘッダーなどの解析において、対象の文字がトークンの一部として許容される文字（英数字および特定の記号）であるかを判定するための関数です。

なぜ重要か

HTTP/2 プロトコルや管理メタデータ（MD）の処理において、ヘッダー値などの文字列からトークンを正確に切り出す必要があります。ptoken_char は、トークンの境界を特定するための走査処理（read_ptoken や skip_ptoken など）において、文字の妥当性を一貫して判定するために重要な役割を果たします。

エンドツーエンドの流れ

各モジュール内のトークン解析処理（read_ptoken や skip_ptoken など）において、文字列を走査するループ処理の条件式などで ptoken_char が呼び出されます。 ただし、各モジュールにおける呼び出し元の詳細な連携や、システム全体におけるエンドツーエンドの完全な制御フローは、現時点では明らかになっていません。

重要な不変条件

  • **許容される記号セット**: 以下の記号は常にトークン文字として有効（戻り値 1）と判定されます。

    • !, #, $, &, ', (, ), *, +, -, ., /, :, <, =, >, ?, @, [, ], ^, _, ` `, {, |, }, ~`

  • **英数字の判定**: 上記の記号以外の文字については、apr_isalnum の判定結果（真であれば 1、偽であれば 0）に依存します。
  • その他の詳細な不変条件については、現時点では未解析です。

よくある落とし穴

  • **ロケール依存性**: 内部で apr_isalnum を使用しているため、実行環境のロケール設定によって英数字の判定挙動が影響を受ける可能性があります。
  • **境界値の挙動**: 引数 c の値が char 型の範囲外である場合の挙動は、APR ライブラリの apr_isalnum の実装に依存します。
  • **仕様との不整合**: 判定対象の記号を変更すると、HTTP/2 仕様（RFC 7540）や関連する Link ヘッダーの構文定義との整合性が失われる可能性があります。
