CGI and FastCGI Process Execution

概念

CGI and FastCGI Process Execution は、Apache HTTP Server（httpd-2.4.67）において、外部のCGI（Common Gateway Interface）スクリプトやプログラムを起動・管理し、動的コンテンツを生成するための仕組みです。

Apacheでは、実行環境のスレッドモデル（MPM）に応じて、直接プロセスをフォークする方式（mod_cgi）と、専用のデーモンプロセスを介してフォークする方式（mod_cgid）を使い分けます。これらは共通のヘッダー定義（cgi_common.h）を共有して実装されています。

なぜ重要か

Webサーバーにおいて外部プロセスを安全かつ効率的に実行することは、システムの安定性とセキュリティに直結します。 特にマルチスレッドMPM（worker や event など）環境下では、リクエスト処理スレッドから直接 fork() を呼び出すと、デッドロックやリソース消費の肥大化を招く危険性があります。これを防ぐために、専用の実行モデル（mod_cgid）を提供し、プロセスの生成と通信を適切に分離することが極めて重要です。

関与するモジュール

  • module:modules/generators

主要エンティティ

本トピックに関与する主要なファイルおよびエンティティは以下の通りです。

  • **modules/generators/mod_cgi.c**:

    • シングルスレッドまたは非スレッドMPM（prefork など）向けのCGI実行モジュール。リクエストを処理するプロセスが直接CGIプロセスをフォークします。

  • **modules/generators/mod_cgid.c**:

    • マルチスレッドMPM向けのCGI実行モジュール。バックグラウンドで動作する専用の cgid デーモンプロセスを介してCGIプロセスをフォークします。

  • **modules/generators/cgi_common.h**:

    • mod_cgi と mod_cgid の間で共有される、CGI実行に関する共通の構造体定義やヘルパー関数を提供するヘッダーファイル。

エンドツーエンドの流れ

1. **リクエストの受信とハンドラの決定**: クライアントからのリクエストがCGIスクリプトにマッチすると、設定されたMPMに応じて mod_cgi または mod_cgid のハンドラが起動します。 2. **プロセスの起動**:

  • **mod_cgi の場合**: リクエスト処理プロセスが直接 fork() および exec() を実行し、CGIスクリプトを起動します。
  • **mod_cgid の場合**: リクエスト処理スレッドが、UNIXドメインソケットを介して cgid デーモンに起動要求を送信します。cgid デーモンが代わりに fork() および exec() を実行します。

    • 3. **入出力のパイプ処理**: CGIプロセスの標準入力、標準出力、標準エラー出力がApacheの接続ソケットやエラーログにバインドされ、データの送受信が行われます。

*注意: 詳細な通信プロトコルやシグナル処理、FastCGIプロセス実行に関する具体的なエンドツーエンドフローの全容は、提供されたモジュールソースのメタデータからは完全には特定されていません。*

重要な不変条件

  • **MPMに応じたモジュールの選択**:

    • マルチスレッドMPM環境では、スレッドセーフティとパフォーマンスを維持するため、mod_cgi ではなく必ず mod_cgid を使用しなければなりません。

  • **共通インターフェースの維持**:

    • mod_cgi と mod_cgid は、CGI環境変数のセットアップやヘッダーのパース処理において、cgi_common.h で定義された共通のデータ構造とロジックを共有しなければなりません。

*注意: 内部的なパイプのクローズ順序や、プロセス監視に関するその他の不変条件は、現時点では未検証です。*

よくある落とし穴

  • **スレッドモデルでの mod_cgi の誤用**:

    • マルチスレッド環境で誤って mod_cgi を有効にすると、フォーク時のスレッド状態の不整合や、パフォーマンスの大幅な低下を引き起こす可能性があります。

  • **mod_cgid デーモンソケットの通信失敗**:

    • mod_cgid はUNIXドメインソケットを使用してデーモンと通信するため、ソケットファイルの配置ディレクトリのパーミッション設定が不適切な場合、CGIの起動に失敗します。

次に読む場所

  • modules/generators/cgi_common.h — CGIモジュール間で共有される共通定義の確認。
  • modules/generators/mod_cgi.c — 直接フォーク方式のCGI実行実装の確認。
  • modules/generators/mod_cgid.c — デーモン経由方式のCGI実行およびソケット通信実装の確認。
