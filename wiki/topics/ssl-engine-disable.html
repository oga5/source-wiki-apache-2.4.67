ssl_engine_disable

概念

ssl_engine_disable は、Apache HTTP Server において特定の接続（conn_rec）に対して SSL/TLS エンジンを動的に無効化するための関数、およびオプション関数（Optional Function）です。

なぜ重要か

この関数は、特定の接続における SSL 処理を無効化するための統一されたインターフェースを提供します。オプション関数として登録されるため、コアサーバー、プロキシモジュール（mod_proxy）、プラットフォーム固有のモジュール（mod_nw_ssl）などの間で、強固なコンパイル時依存関係を持たずに機能を共有・呼び出すことができます。

関与するモジュール

  • module:server
  • module:modules/arch/netware

主要エンティティ

  • symbol:server/ssl.c#ssl_engine_disable: 関数の実体定義。APR_REGISTER_OPTIONAL_FN を用いてオプション関数として登録されます。
  • symbol:modules/arch/netware/mod_nw_ssl.c#ssl_engine_disable: NetWare 環境用の SSL モジュールにおける、本オプション関数の登録または利用。

エンドツーエンドの流れ

1. **登録**: server/ssl.c 内で APR_REGISTER_OPTIONAL_FN(ssl_engine_disable) により、オプション関数としてシステムに登録されます。 2. **取得と呼び出し**: mod_proxy や mod_nw_ssl などのモジュールが、このオプション関数を動的に取得して呼び出します。 3. **実行**: ssl_engine_disable(conn_rec *c) が呼び出されると、内部で ap_ssl_bind_outgoing(c, NULL, 0) を実行します。 4. **返却**: ap_ssl_bind_outgoing の実行結果（ステータスコード）がそのまま呼び出し元に返されます。

※ 各モジュールがどのような条件でこの無効化をトリガーするのか、および ap_ssl_bind_outgoing 内部の具体的な処理フローの詳細は、提供されたコンテキストからは未詳です。

重要な不変条件

  • 引数 c は有効な conn_rec 構造体へのポインタである必要があります。
  • 本関数は、ap_ssl_bind_outgoing の第2引数に NULL、第3引数に 0 を渡して呼び出すラッパーとして機能します。
  • ※ c が NULL の場合の挙動や、無効化された接続状態が下流のフィルタやハンドラでどのように維持されるかについての不変条件は、現時点では不明です。

よくある落とし穴

  • **オプション関数の未登録**: 本関数はオプション関数として公開されているため、呼び出し側モジュールは関数ポインタが NULL でないことを確認してから呼び出す必要があります。
  • **エラーハンドリング**: ap_ssl_bind_outgoing が失敗した場合、そのエラーコードがそのまま返されるため、呼び出し元で戻り値を適切に評価する必要があります。
