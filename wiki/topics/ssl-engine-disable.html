ssl_engine_disable

概念

ssl_engine_disable は、Apache HTTP Server において特定の接続（conn_rec）に対する SSL エンジン（SSL/TLS 処理）を無効化するための関数、およびオプション関数（Optional Function）です。

なぜ重要か

この関数は、特定の接続において動的に SSL 処理を無効化する統一された手段を提供します。オプション関数として登録・公開されているため、コアサーバー（server）やプラットフォーム固有のモジュール（mod_nw_ssl）、プロキシモジュール（mod_proxy）などの異なるコンポーネント間で、密結合を避けながら SSL の無効化機能を共有・利用できます。

関与するモジュール

  • module:server (コアサーバー機能)
  • module:modules/arch/netware (NetWare 向け SSL モジュール mod_nw_ssl)

※ 呼び出し元として modules/proxy (プロキシモジュール mod_proxy) も関与します。

主要エンティティ

  • symbol:server/ssl.c#ssl_engine_disable: ap_ssl_bind_outgoing を用いて SSL を無効化する実体。オプション関数として登録されます。
  • symbol:modules/arch/netware/mod_nw_ssl.c#ssl_engine_disable: NetWare 環境向けに定義されている同名関数。
  • conn_rec: 接続状態を保持し、本関数の引数として渡される Apache の構造体。

エンドツーエンドの流れ

1. **登録**: server/ssl.c 内で APR_REGISTER_OPTIONAL_FN(ssl_engine_disable) により、オプション関数として登録されます。 2. **取得と呼び出し**: mod_proxy などの他モジュールが、このオプション関数を動的に取得して呼び出します。 3. **実行**: ssl_engine_disable(conn_rec *c) が呼び出されると、内部で ap_ssl_bind_outgoing(c, NULL, 0) が実行されます。 4. **返却**: ap_ssl_bind_outgoing の実行結果（ステータスコード）が呼び出し元に返されます。

※ NetWare 環境における modules/arch/netware/mod_nw_ssl.c 内の ssl_engine_disable の具体的な振る舞いや、呼び出しの完全なエンドツーエンドのフローは現時点では解析されておらず、未詳です。

重要な不変条件

  • 引数として渡される conn_rec *c は、有効な接続構造体を指している必要があります。
  • server/ssl.c の実装においては、ap_ssl_bind_outgoing の第2引数に NULL、第3引数に 0 を指定して呼び出すことで、SSL 設定の無効化を保証します。
  • c が NULL の場合の挙動や、NetWare 環境における不変条件は現時点では未詳です。

よくある落とし穴

  • **オプション関数の解決**: ssl_engine_disable はオプション関数として登録されているため、呼び出し元モジュールは実行時にこの関数が正しくロード・取得できているかを確認する必要があります。
  • **エラーハンドリング**: ap_ssl_bind_outgoing が失敗した場合、そのエラーコードがそのまま返されるため、呼び出し側で戻り値を適切にハンドリングする必要があります。
  • **プラットフォーム依存**: modules/arch/netware/mod_nw_ssl.c 側の実装詳細や失敗モードについては、現時点で解析が進んでいないため注意が必要です。
