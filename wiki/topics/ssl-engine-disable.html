ssl_engine_disable

概念

ssl_engine_disable は、Apache HTTP Server において特定の接続（conn_rec）に対する SSL エンジンを動的に無効化するための関数です。この関数はオプション関数（Optional Function）として登録され、コアの SSL モジュール以外の別モジュールから動的に呼び出すことができます。

なぜ重要か

この関数は、プロキシ処理やプラットフォーム固有の SSL 処理（NetWare など）において、特定の接続に対して SSL を無効化する統一されたインターフェースを提供します。ap_ssl_bind_outgoing を直接呼び出す代わりに、このオプション関数を介することで、モジュール間の結合度を下げつつ、安全に SSL 設定を解除できます。

関与するモジュール

  • module:server
  • module:modules/arch/netware

主要エンティティ

  • **symbol:server/ssl.c#ssl_engine_disable**

    • SSL エンジンを無効化する実体関数。内部で ap_ssl_bind_outgoing(c, NULL, 0) を呼び出します。APR_REGISTER_OPTIONAL_FN を用いてオプション関数として登録されます。

  • **symbol:modules/arch/netware/mod_nw_ssl.c#ssl_engine_disable**

    • NetWare 環境向けの SSL モジュール。このオプション関数を登録または取得して利用します。

エンドツーエンドの流れ

1. **登録**: server/ssl.c 内で APR_REGISTER_OPTIONAL_FN(ssl_engine_disable) により、オプション関数としてシステムに登録されます。 2. **取得と呼び出し**: mod_nw_ssl.c などの外部モジュールが、登録された ssl_engine_disable オプション関数を動的に取得し、対象の接続（conn_rec *c）を引数にして呼び出します。 3. **実行**: ssl_engine_disable は、受け取った接続ポインタ c を使用して ap_ssl_bind_outgoing(c, NULL, 0) を呼び出します。 4. **返却**: ap_ssl_bind_outgoing の実行結果（ステータスコード）が、そのまま呼び出し元に返されます。

※ 各モジュールがどのタイミングでこの関数を呼び出すか、および ap_ssl_bind_outgoing 内部の具体的な処理フローの全容については、提供された情報からは未確定です。

重要な不変条件

  • **引数の有効性**: 引数として渡される c は、有効な conn_rec 構造体を指している必要があります。
  • **ラッパーとしての動作**: 本関数は常に ap_ssl_bind_outgoing の第2引数に NULL、第3引数に 0 を指定して呼び出す不変のラッパーとして機能します。

※ 接続状態の遷移やスレッドセーフ性に関する詳細な不変条件は、現時点では不明です。

よくある落とし穴

  • **戻り値の無視**: ap_ssl_bind_outgoing が失敗した場合、そのエラーコードがそのまま返されます。呼び出し側で戻り値を適切にチェックしないと、SSL が無効化されていない状態で後続の通信が処理される可能性があります。
  • **NULL ポインタの伝播**: 引数 c が NULL の場合の挙動は ap_ssl_bind_outgoing の実装に依存するため、呼び出し前に c が有効であることを確認する必要があります。
