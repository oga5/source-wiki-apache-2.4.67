os_fork

概念

os_fork は、Apache HTTP Server（httpd-2.4.67）において、実行環境（OS）の特性に応じた適切なプロセス生成（フォーク）処理を抽象化・カプセル化するための関数です。プラットフォームごとのプロセス生成手法の違いを吸収し、共通のインターフェースを提供します。

なぜ重要か

マルチプロセスモデルを採用するWebサーバー（例: prefork MPM）において、リクエストを処理するための子プロセスを安全かつ適切に生成することは不可欠です。 例えば、BS2000（OSD/POSIX）環境では標準の fork() だけでなく、特定のユーザー権限でプロセスを起動するための ufork() を使い分ける必要があり、os_fork を通じてこれらのプラットフォーム固有の挙動を隠蔽します。

エンドツーエンドの流れ

1. **プロセスの生成要求**: server/mpm/prefork/prefork.c などの呼び出し元から、対象のユーザー名（user）を指定して os_fork(user) が呼び出されます。 2. **BS2000環境における処理**:

  • os_forktype(0) を呼び出し、プロセス生成モード（bs2_ForkType）を判定します。
  • モードが bs2_FORK の場合、標準の fork() を実行します。
  • モードが bs2_UFORK の場合、指定された user 名を大文字に変換（ap_str_toupper()）し、ufork(username) を実行します。
  • それ以外のモードでは、PIDを 0 に設定します。

    • 3. **Unix環境における処理**:

  • Unix環境（os/unix/unixd.c）における具体的なエンドツーエンドの流れや、Unix固有の os_fork の内部実装の詳細は、提供されたコンテキストからは未詳です。

重要な不変条件

  • **BS2000環境における不変条件**:
  • bs2_UFORK モードで動作する場合、渡される user 引数は大文字に変換されて ufork() に渡されなければなりません。また、user 名の長さは USER_LEN に制限されます。
  • **Unix環境における不変条件**:
  • 提供されたコンテキストからは未詳です。

よくある落とし穴

  • **BS2000環境における権限エラー**:
  • ufork() の実行時に EPERM（権限エラー）が発生した場合、設定不備とみなされ、サーバーはエラーログを出力して exit(1) により緊急停止します。実行ユーザーの権限設定に注意する必要があります。
