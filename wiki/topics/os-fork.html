os_fork

概念

os_fork は、Apache HTTP Server が動作するプラットフォーム（特に BS2000/OSD/POSIX 環境や Unix 環境）において、適切なプロセス生成処理を抽象化・カプセル化するためのラッパー関数です。プラットフォーム固有のプロセス生成ロジック（標準の fork() や、BS2000 固有の ufork() など）を呼び出し元から隠蔽します。

なぜ重要か

マルチプロセスモデル（例: prefork MPM）を採用する Web サーバーにおいて、プロセスの安全かつ適切な生成は不可欠です。特に BS2000 環境では、実行環境の設定や指定されたユーザー権限に応じて fork() と ufork() を適切に使い分ける必要があり、os_fork はこのプラットフォーム固有の複雑な分岐処理を一元管理する役割を担っています。

関与するモジュール

  • module:os/bs2000 : BS2000（OSD/POSIX）環境向けのプロセス生成処理を実装しています。
  • module:os/unix : Unix 環境向けの os_fork 実装を提供します。

主要エンティティ

  • symbol:os/bs2000/os.c#os_fork : BS2000 環境用の実装。os_forktype の結果に基づき、fork() または ufork() を選択実行します。
  • symbol:os/unix/unixd.c#os_fork : Unix 環境用の実装（詳細な振る舞いは未解析）。

エンドツーエンドの流れ

BS2000 環境における os_fork の処理フローは以下の通りです： 1. server/mpm/prefork/prefork.c などの呼び出し元から、切り替え対象のユーザー名（user）を引数として os_fork が呼び出されます。 2. os_forktype(0) を呼び出し、現在のプロセス生成モード（bs2_ForkType）を判定します。 3. 判定されたモードに応じて以下の処理を行います：

  • **bs2_FORK**: 標準ライブラリの fork() を実行します。
  • **bs2_UFORK**: 引数 user を ap_str_toupper() で大文字に変換し、ufork(username) を実行します。
  • **その他**: 生成された PID を 0 に設定します。

    • 4. 生成されたプロセスの PID（またはエラー時の -1）を呼び出し元に返します。

※ Unix 環境（os/unix/unixd.c）における具体的なエンドツーエンドの流れや、呼び出し元との詳細な連携フローについては、現時点では解析されておらず未詳です。

重要な不変条件

  • **BS2000 環境における不変条件**:
  • bs2_UFORK モードで動作する場合、渡された user 引数は必ず大文字に変換されてから ufork() に渡されます。
  • user 名の長さは USER_LEN を超えてはなりません。
  • ufork() の実行時に EPERM（権限エラー）が発生した場合、プロセスは即座に緊急ログを出力し、exit(1) で異常終了しなければなりません（サーバーの安全な停止を保証するため）。

※ Unix 環境における不変条件については、現時点では解析されておらず未詳です。

よくある落とし穴

  • **BS2000 環境**:
  • ufork() が EPERM を返した場合、関数はエラーを呼び出し元に返すのではなく、プロセス自体を exit(1) で強制終了します。このため、呼び出し元でエラーハンドリングを行う余地がなく、設定不備がある場合はサーバーが即座に緊急停止する点に注意が必要です。
  • user 名の長さ制限（USER_LEN）を超えた場合の挙動や、BS2000 のバージョン依存による ufork() の挙動変化に注意する必要があります。
  • **Unix 環境**:
  • 現時点では詳細な解析が行われていないため、Unix 環境特有の落とし穴や制約は未詳です。
