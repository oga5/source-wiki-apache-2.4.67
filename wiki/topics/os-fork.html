os_fork

概念

os_fork は、動作するオペレーティングシステム環境の特性に応じて、適切なプロセス生成手法を選択・実行するためのラッパー関数です。特に BS2000（OSD/POSIX）環境においては、標準の fork() と、ユーザー切り替えを伴う固有の ufork() を動的に切り替えてプロセスを生成する役割を担います。

なぜ重要か

Apache HTTP Server などのマルチプロセスモデル（MPM）において、プラットフォーム固有のプロセス生成ロジックや制約（BS2000 における大文字ユーザー名要件や権限管理など）を隠蔽し、上位レイヤーに対して統一的なプロセス生成インターフェースを提供するために重要です。

関与するモジュール

  • **symbol:os/bs2000/os.c#os_fork**: BS2000 環境向けの実装。fork() または ufork() を選択的に呼び出します。

エンドツーエンドの流れ

1. **呼び出し**: server/mpm/prefork/prefork.c などの呼び出し元から、対象ユーザー名（user）を指定して os_fork が呼び出されます。 2. **BS2000 環境での処理**:

  • os_forktype(0) を呼び出し、現在のプロセス生成モード（bs2_ForkType）を判定します。
  • **bs2_FORK の場合**: 標準の fork() を実行します。
  • **bs2_UFORK の場合**: user 引数を ap_str_toupper() で大文字に変換し、ufork(username) を実行します。
  • **その他の場合**: PID を 0 に設定します。
  • **エラーハンドリング**: ufork() が EPERM を返した場合、緊急ログを出力して exit(1) によりプロセスを終了します。

※ Unix 環境（symbol:os/unix/unixd.c#os_fork）における具体的なエンドツーエンドの流れや、他プラットフォームとの統合フローの全貌は現時点では不明です。

重要な不変条件

  • **BS2000 環境における不変条件**:
  • bs2_UFORK モードの際、ufork() に渡されるユーザー名は必ず大文字に変換されていなければなりません。
  • ufork() 実行時に EPERM（権限エラー）が発生した場合、プロセスは必ず exit(1) で即座に終了しなければなりません。
  • 入力される user 名の長さは USER_LEN 以下に制限されます。
  • **Unix 環境における不変条件**:
  • 現時点では情報が不足しており不明です。

よくある落とし穴

  • **BS2000 環境での権限不備**: ufork() 実行時に EPERM が返されると、サーバープロセスが緊急停止（exit(1)）するため、実行ユーザーの権限設定や OS 側の設定不備がシステム全体の停止に直結します。
  • **ユーザー名の長さ制限**: user 引数の長さが USER_LEN を超過した場合の境界挙動に注意する必要があります。
  • **Unix 環境の挙動**: os/unix/unixd.c における os_fork の具体的な失敗モードや制限事項は現時点では不明です。
