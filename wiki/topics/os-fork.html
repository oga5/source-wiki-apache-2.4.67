os_fork

概念

os_fork は、Apache HTTP Server（httpd-2.4.67）において、プラットフォームや実行環境の特性に応じて適切なプロセス生成処理を抽象化・カプセル化するためのラッパー関数です。特にBS2000（OSD/POSIX）環境などでは、標準の fork() だけでなく、特定のユーザー権限でプロセスを起動するための ufork() などの固有のAPIを切り替えて使用する役割を担います。

なぜ重要か

マルチプロセスモデル（prefork MPMなど）を採用するWebサーバーにおいて、リクエストを処理する子プロセスの生成は基盤となる処理です。プラットフォームごとのプロセス生成APIの差異（標準の fork とBS2000固有の ufork など）を os_fork という共通のインターフェースで隠蔽することで、サーバーコアやMPM（Multi-Processing Module）のコードのポータビリティと保守性を向上させています。

エンドツーエンドの流れ

BS2000環境における os_fork の処理の流れは以下の通りです。なお、Unix環境における詳細なエンドツーエンドの流れは提供された情報からは不明です。

1. **呼び出し**: server/mpm/prefork/prefork.c などの呼び出し元から、切り替え対象のユーザー名（user）を引数として os_fork(user) が呼び出されます。 2. **モード判定**: os_forktype(0) を呼び出し、現在のプロセス生成モード（bs2_ForkType）を判定します。 3. **プロセス生成の実行**:

  • **bs2_FORK の場合**: 標準ライブラリの fork() を実行します。
  • **bs2_UFORK の場合**: 引数 user を大文字に変換（ap_str_toupper）し、ufork(username) を実行します。
  • **その他の場合**: PIDを 0 に設定します。

    • 4. **エラーハンドリング**: ufork() の実行時に EPERM エラーが発生した場合、緊急ログ（ap_log_error）を出力し、exit(1) でプロセスを即座に終了します。 5. **返却**: 生成されたプロセスのPID（またはエラー時の -1、デフォルト時の 0）を呼び出し元に返します。

重要な不変条件

  • **BS2000環境における不変条件**:
  • os_fork は _OSD_POSIX マクロが定義されている環境でのみ有効に動作します。
  • bs2_UFORK モードで動作する場合、渡されるユーザー名は大文字に変換されて ufork() に渡される必要があります。
  • user 引数として渡されるユーザー名の長さは、内部バッファの制限である USER_LEN を超えてはなりません。
  • **Unix環境における不変条件**:
  • 提供された情報からは、Unix環境における具体的な不変条件は不明です。

よくある落とし穴

  • **BS2000環境における権限エラーによる即時終了**: ufork() の呼び出しが EPERM（権限エラー）を返した場合、関数は単にエラーコードを返すのではなく、exit(1) を呼び出してサーバープロセスを緊急停止させます。このため、実行ユーザーの権限設定に不備があると、システムが予期せず終了する原因となります。
  • **Unix環境における詳細**: Unix環境の実装（os/unix/unixd.c#os_fork）における落とし穴や制限事項については、提供された情報からは不明です。
