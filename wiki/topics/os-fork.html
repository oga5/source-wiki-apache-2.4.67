os_fork

概念

os_fork は、Apache HTTP Server が動作するプラットフォームの違いを吸収し、それぞれの実行環境に適したプロセス生成を行うためのラッパー関数です。特に BS2000（OSD/POSIX）環境などにおいて、標準の fork() とプラットフォーム固有の ufork() を適切に切り替えてプロセスを生成する役割を担います。

なぜ重要か

マルチプロセスモデルを採用する Web サーバーにおいて、プロセスの生成は基盤となる処理です。プラットフォーム（Unix や BS2000 など）によってプロセス生成の仕組みや必要な権限、ユーザー切り替えの作法が異なるため、これらを os_fork という共通のインターフェースで抽象化することで、MPM（マルチプロセッシングモジュール）などの上位レイヤーがプラットフォーム非依存で動作できるようにしています。

関与するモジュール

主要エンティティ

  • **symbol:os/bs2000/os.c#os_fork**: BS2000 環境向けの実装。実行環境のモードに応じて fork() または ufork() を選択的に呼び出します。
  • **symbol:os/unix/unixd.c#os_fork**: Unix 環境向けの実装（詳細な振る舞いは提供されたコンテキストからは未詳）。

エンドツーエンドの流れ

BS2000 環境における処理の流れは以下の通りです。

1. **呼び出し**: server/mpm/prefork/prefork.c などの呼び出し元から、対象のユーザー名（user）を指定して os_fork(user) が呼び出されます。 2. **モード判定**: os_forktype(0) を呼び出し、現在のプロセス生成モード（bs2_ForkType）を取得します。 3. **プロセス生成の実行**:

  • **bs2_FORK の場合**: 標準の fork() を実行します。
  • **bs2_UFORK の場合**: 入力された user 名を ap_str_toupper() で大文字に変換し、ufork(username) を実行します。
  • **その他の場合**: PID を 0 に設定します。

    • 4. **結果の返却**: 生成されたプロセスの PID（エラー時は -1）を呼び出し元に返します。

※ Unix 環境（os/unix/unixd.c）における具体的なエンドツーエンドの流れ、およびシステム全体における完全な統合フローは、提供されたコンテキストからは未詳です。

重要な不変条件

  • **BS2000 環境における制約**:
  • _OSD_POSIX マクロが定義されている環境でのみ有効です。
  • bs2_UFORK モードの際、user 名の長さは USER_LEN 以下に制限されます。また、ufork() に渡すユーザー名は必ず大文字に変換されます。
  • **Unix 環境における制約**:
  • 提供されたコンテキストからは未詳です。

よくある落とし穴

  • **BS2000 環境での緊急停止**:

    • ufork() の実行時に EPERM（権限エラー）が発生した場合、os_fork は呼び出し元にエラーを返すのではなく、緊急ログを出力した上で exit(1) を呼び出し、プロセスを強制終了します。実行環境の権限設定に不備がある場合、サーバーが予期せず停止する原因となります。

  • **Unix 環境との差異**:

    • Unix 環境の実装（os/unix/unixd.c）における具体的な失敗モードや挙動の差異については、提供されたコンテキストからは未詳です。
