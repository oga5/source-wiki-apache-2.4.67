attr_char

概念

attr_char は、HTTPヘッダーやトークンの解析処理において、指定された文字が「属性（attribute）」の一部として許容される文字であるかどうかを判定するためのヘルパー関数です。この関数は、Apache httpdの複数のモジュール（HTTP/2およびACME/Managed Domains管理用のmod_md）において、それぞれ static 関数として個別に定義されています。

なぜ重要か

HTTP仕様（RFC）に準拠した厳格なトークン解析を行うために重要です。特定の記号（ホワイトリストに含まれるもの）および英数字のみを属性文字として許可することで、不正な文字の混入を防ぎ、プロトコル解析の安全性と正確性を担保します。

関与するモジュール

  • module:modules/http2
  • module:modules/md

主要エンティティ

  • **symbol:modules/http2/h2_proxy_util.c#attr_char**: h2_proxy_util.c 内で定義される静的関数。read_pname や skip_pname から呼び出されます。
  • **symbol:modules/http2/h2_push.c#attr_char**: h2_push.c 内で定義される静的関数。英数字および特定の記号（!, #, $, &, +, -, ., ^, _, ` `, |, ~`）を許容します。
  • **symbol:modules/md/md_util.c#attr_char**: md_util.c 内で定義される静的関数。read_pname や skip_pname から呼び出されます。

エンドツーエンドの流れ

現時点では、これらの関数が呼び出される全体的なエンドツーエンドの処理フロー（リクエスト受信から解析、レスポンス送信にいたる一連の流れ）の詳細は完全には判明していません。

判明している範囲では、HTTP/2のプッシュ機能（h2_push.c）やプロキシユーティリティ（h2_proxy_util.c）、およびManaged Domainsユーティリティ（md_util.c）におけるヘッダーやパラメータ名（read_pname, skip_pname など）のパース処理の過程で、1文字ずつ文字を走査して属性値として妥当かを検証する際に呼び出されます。

重要な不変条件

  • **ホワイトリスト方式の文字判定**: h2_push.c の実装においては、標準的な英数字（apr_isalnum による判定）に加え、特定の記号（!, #, $, &, +, -, ., ^, _, ` `, |, ~）のみを許容し、それ以外の制御文字や NULL 文字などは常に 0`（拒否）を返します。
  • **スコープの限定**: 各ファイルで static 関数として定義されているため、スコープはその定義ファイル内に限定されます。ただし、各ファイル間での実装の不変条件や同期関係の詳細は完全には判明していません。

よくある落とし穴

  • **文字コード依存性**: apr_isalnum の挙動に依存しているため、入力文字が char 型の範囲外である場合の挙動は apr_isalnum に依存します。
  • **仕様変更時の不整合**: 許容する記号を変更する場合、RFC等のHTTP仕様と整合しているか確認する必要があります。また、複数のファイルに同名の static 関数として分散して定義されているため、一方のみを変更するとモジュール間で判定基準に不整合が生じる可能性があります。
