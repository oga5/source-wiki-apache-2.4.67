log_local_address

概念

log_local_address は、Apache HTTP Server において、リクエストに関連付けられたローカル（サーバー側）のIPアドレスを取得し、ログ出力用に提供するための関数です。

なぜ重要か

この関数は、アクセスログやエラーログのフォーマット文字列（例: %A）に対応するハンドラとして登録されます。これにより、複数のIPアドレスを持つサーバー環境において、クライアントが実際にどのローカルIPアドレスに接続してきたかを正確に記録・追跡することが可能になります。

関与するモジュール

  • module:modules/loggers
  • module:server

主要エンティティ

  • symbol:modules/loggers/mod_log_config.c#log_local_address: ローカルIPアドレスを取得する実体関数。
  • symbol:server/log.c#log_local_address: エラーログハンドラとして登録・利用されるシンボル。

エンドツーエンドの流れ

1. **ハンドラ登録**:

  • modules/loggers/mod_log_config.c にて、log_pfn_register を介してアクセスログのフォーマット項目 "A" として log_local_address が登録されます。
  • server/log.c にて、ap_register_errorlog_handler を介してエラーログハンドラ "A" として登録されます。

    • 2. **ログ出力時**:

  • ログ記録フェーズにおいて、登録されたハンドラが呼び出されます。
  • log_local_address は、引数として渡された request_rec 構造体から r->connection->local_ip を参照し、その文字列を返します。

※なお、接続確立時に r->connection->local_ip が設定される具体的な詳細フローや、エラーログハンドラとしての呼び出しコンテキストの全容は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **ポインタの有効性**: 呼び出し時に渡される request_rec（r）および r->connection が有効なメモリ領域を指している必要があります。
  • **値の参照のみ**: 本関数自体はIPアドレスの解決や生成を行わず、接続フェーズで既に設定されている r->connection->local_ip の値をそのまま参照して返します。

よくある落とし穴

  • **NULLポインタアクセス**: r が NULL の状態で本関数が呼び出されると、セグメンテーション違反が発生します。
  • **未初期化状態**: 接続が確立されていない、あるいは local_ip が初期化されていない状態での動作は、r->connection の状態に依存します。

次に読む場所

  • modules/loggers/mod_log_config.c: アクセスログ用フォーマット登録と log_local_address の実装。
  • server/log.c: エラーログハンドラとしての登録処理。
