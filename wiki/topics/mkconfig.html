mkconfig

概念

mkconfig は、Apache HTTP Server（httpd-2.4.67）の特定モジュールにおいて、モジュール固有の設定構造体をメモリ上に確保し、初期化するための静的（static）関数です。この関数名は複数のモジュールで独立して定義されており、それぞれのモジュールが持つ設定データを管理するための初期化処理を担っています。

なぜ重要か

各モジュール（mod_suexec や mod_speling）が正しく動作するためには、リクエスト処理や設定適用に先立ち、設定構造体が適切なデフォルト値で初期化されている必要があります。mkconfig は、APR（Apache Portable Runtime）のメモリプールを利用して安全にメモリを確保し、初期状態を定義する役割を持つため重要です。

関与するモジュール

  • module:modules/generators
  • module:modules/mappers

主要エンティティ

  • **symbol:modules/generators/mod_suexec.c#mkconfig**: suexec_config_t 構造体のメモリを apr_palloc で確保し、active フラグを 0（無効）に初期化します。
  • **symbol:modules/mappers/mod_speling.c#mkconfig**: spconfig 構造体のメモリを apr_pcalloc で確保し、enabled を 0、check_case_only を 0、check_basename_match を 1（有効）に初期化します。

エンドツーエンドの流れ

各モジュールの初期化フェーズにおいて、Apacheのコアまたは設定パーサーからこれらの関数が呼び出されると推測されますが、具体的な呼び出しシーケンスや全体的なエンドツーエンドの流れは、提供された情報からは完全には明らかになっていません。

現時点で判明している局所的な流れは以下の通りです： 1. **mod_suexec 内の流れ**:

  • apr_pool_t ポインタを受け取ります。
  • apr_palloc を用いて suexec_config_t のメモリを確保します。
  • active メンバに 0 を設定して、初期化された構造体へのポインタを返します。

    • 2. **mod_speling 内の流れ**:

  • apr_pool_t ポインタを受け取ります。
  • apr_pcalloc を用いて spconfig のメモリをゼロクリア状態で確保します。
  • enabled を 0、check_case_only を 0、check_basename_match を 1 に設定して、初期化された構造体へのポインタを返します。

重要な不変条件

  • **メモリ確保の成否**: メモリ確保には APR のプール（apr_palloc / apr_pcalloc）が使用されます。メモリ不足が発生した場合、APR の仕様によりプロセス自体が終了する可能性があります。
  • **初期値の保証**:
  • mod_suexec の mkconfig が返す構造体は、active メンバが必ず 0 で初期化されます。
  • mod_speling の mkconfig が返す構造体は、check_basename_match が必ず 1（有効）であり、その他の指定メンバは 0 で初期化されます。

よくある落とし穴

  • **名前の衝突とスコープの混同**: 両関数は static として定義されているため、それぞれのソースファイル内でのみ有効です。しかし、静的解析ツールなどの検出において、mod_suexec.c から mod_speling.c の mkconfig を呼び出している（あるいはその逆）かのような誤検出が発生することがあります。これらは完全に独立した別個の関数であり、相互に直接呼び出すことはできない点に注意が必要です。
