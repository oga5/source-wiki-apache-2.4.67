mkconfig

概念

mkconfig は、Apache HTTP Serverの特定モジュールにおいて、モジュール固有の設定構造体をメモリ上に確保し、初期値を設定するための静的（static）なヘルパー関数です。同名の関数が複数のモジュールで個別に定義されています。

なぜ重要か

各モジュール（mod_suexec や mod_speling）が動作するにあたり、リクエスト処理や動作制御に必要な設定値を保持する構造体を、APR（Apache Portable Runtime）のメモリプールを用いて安全に初期化するために重要です。これにより、未初期化メモリによる予期しない動作を防ぎます。

関与するモジュール

  • module:modules/generators
  • module:modules/mappers

主要エンティティ

  • **symbol:modules/generators/mod_suexec.c#mkconfig**: suexec_config_t 構造体のメモリを apr_palloc で確保し、active フラグを 0（無効）に初期化します。
  • **symbol:modules/mappers/mod_speling.c#mkconfig**: spconfig 構造体のメモリを apr_pcalloc で確保し、enabled を 0、check_case_only を 0、check_basename_match を 1（有効）に初期化します。

エンドツーエンドの流れ

各モジュールの初期化フェーズにおいて、Apacheのコアまたは設定パーサーからこれらの関数が呼び出され、設定構造体が生成されます。 ただし、具体的な設定ファイルのパースからこれら mkconfig が呼び出され、最終的にリクエスト処理へ適用されるまでの詳細なエンドツーエンドの制御フロー全体は、提供された情報からは未解明です。

重要な不変条件

  • **メモリ確保の成否**: いずれの関数もAPRのプール（apr_pool_t）からメモリを確保します。メモリ不足が発生した場合、APRの仕様に従ってプロセスが終了する可能性があります。
  • **初期値の保証**:
  • mod_suexec の mkconfig では、戻り値の active メンバは常に 0 で初期化されます。
  • mod_speling の mkconfig では、apr_pcalloc によるゼロ初期化に加え、check_basename_match が明示的に 1 に設定されます。

よくある落とし穴

これらの関数は static 宣言されたファイルローカルな関数です。解析ツールなどのヒューリスティックな判定により、mod_suexec.c 内の mkconfig と mod_speling.c 内の mkconfig が相互に呼び出し合っているように検出される場合がありますが、実際にはそれぞれのモジュール内で独立して定義・実行される別個の関数です。コードの変更やリファクタリングを行う際は、これらが共有コードではない点に注意してください。
