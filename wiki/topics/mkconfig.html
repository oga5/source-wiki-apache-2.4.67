mkconfig

概念

mkconfig は、Apache HTTP Serverの特定モジュール（mod_suexec および mod_speling）において、モジュール固有の設定構造体をメモリ上に確保し、デフォルト値で初期化するための静的（static）なヘルパー関数です。

なぜ重要か

各モジュールが動作する際、動作制御に必要な設定値を保持する構造体（suexec_config_t や spconfig）を適切に初期化する必要があります。mkconfig は、APR（Apache Portable Runtime）のメモリプールを利用して安全にメモリを確保し、初期状態の不整合を防ぐ役割を担っています。

関与するモジュール

  • module:modules/generators
  • module:modules/mappers

主要エンティティ

  • symbol:modules/generators/mod_suexec.c#mkconfig: suexec_config_t 構造体のメモリを apr_palloc で確保し、active フラグを 0（無効）に初期化する関数。
  • symbol:modules/mappers/mod_speling.c#mkconfig: spconfig 構造体のメモリを apr_pcalloc で確保し、enabled を 0、check_case_only を 0、check_basename_match を 1（有効）に初期化する関数。

エンドツーエンドの流れ

Apache HTTP Serverの起動時や設定読み込みフェーズにおいて、各モジュールの設定構造体を生成するために呼び出されます。ただし、Apacheコアからこれらの関数が呼び出される正確なライフサイクルパスや全体的なエンドツーエンドの流れは、提供された情報からは完全には明らかになっていません。

現時点で判明している局所的な流れは以下の通りです： 1. メモリプール（apr_pool_t）へのポインタを引数として mkconfig が呼び出される。 2. mod_suexec では apr_palloc、mod_speling では apr_pcalloc を使用して、対応する設定構造体のメモリを確保する。 3. 構造体の各メンバにデフォルト値を代入する。 4. 初期化された構造体へのポインタを呼び出し元に返す。

重要な不変条件

  • **メモリ確保の成否**: メモリ確保には APR のプール関数（apr_palloc / apr_pcalloc）が使用されます。これらの関数がメモリ不足により失敗した場合、APR の仕様に従ってプロセス自体が終了する可能性があります。
  • **初期値の保証**:
  • mod_suexec の mkconfig では、active メンバが常に 0 で初期化されます。
  • mod_speling の mkconfig では、enabled が 0、check_case_only が 0、check_basename_match が 1 で初期化されます。

よくある落とし穴

  • **静的スコープとツールの誤検出**: 両関数はそれぞれのソースファイル内で static として定義された独立した関数です。しかし、静的解析ツールなどのヒューリスティックな検出により、mod_suexec.c 内の mkconfig が mod_speling.c から呼び出されている（またはその逆）と誤って判定される場合があります。実際には、これらは共有ライブラリ関数ではなく、各モジュール内に閉じられた独立した実装です。コードの保守やリファクタリングの際は、これらが別個のエンティティであることを認識する必要があります。
