CGI and FastCGI Execution Lifecycle

概念

CGI and FastCGI Execution Lifecycle は、Apache HTTP Server が外部の CGI (Common Gateway Interface) スクリプトや関連するバックエンドプロセスを起動・管理し、リクエストおよびレスポンスの仲介を行うライフサイクルです。これには、環境変数のセットアップ、プロセスのフォーク（または専用デーモンへの処理委譲）、および標準入出力を介した通信の管理が含まれます。

なぜ重要か

動的コンテンツを生成する外部プロセスと安全かつ効率的に連携することは、Webサーバーのパフォーマンスと安定性に直結します。特にマルチスレッド型のMPM（Multi-Processing Module）環境では、リクエストごとに直接プロセスをフォークするとシステム全体のパフォーマンス低下やスレッド安全性の問題が生じるため、専用のデーモンプロセスを介して実行を制御する仕組みが極めて重要になります。

関与するモジュール

  • module:modules/generators

主要エンティティ

本ライフサイクルは、主に以下のファイル群によって構成されています：

  • **modules/generators/cgi_common.h**: CGI および CGID モジュール間で共有される共通のデータ構造やヘルパー定義を提供します。
  • **modules/generators/mod_cgi.c**: 主にシングルスレッド（Preforkなど）環境向けに、Webサーバープロセスから直接CGIスクリプトをフォークして実行する標準CGIモジュールです。
  • **modules/generators/mod_cgid.c**: マルチスレッド環境向けに、専用のCGIデーモンプロセス（cgid）をバックグラウンドで起動し、ソケット通信を介してCGIスクリプトの起動と通信を処理するモジュールです。

エンドツーエンドの流れ

現時点では、提供されたソースコードの詳細な解析が不足しているため、具体的な関数呼び出しレベルでのエンドツーエンドの制御フローは完全には判明していません。

一般的な動作モデルとしては、以下の流れが想定されます： 1. クライアントからのリクエストを Apache が受信し、CGIハンドラが起動します。 2. mod_cgi または mod_cgid が、リクエスト情報に基づいて環境変数を構築します。 3. mod_cgi の場合は直接子プロセスをフォークしてスクリプトを実行し、mod_cgid の場合はUNIXドメインソケットを介して cgid デーモンに実行を依頼します。 4. 外部CGIスクリプトが実行され、標準出力を介してレスポンスがWebサーバーに返送されます。

重要な不変条件

現時点では、モジュール間における厳密な不変条件（スレッド安全性、ソケット記述子の生存期間、シグナルハンドリングの制約など）は完全には特定されていません。

よくある落とし穴

  • **MPMとモジュールの不適合**: マルチスレッドMPM（WorkerやEventなど）を使用しているにもかかわらず、mod_cgid ではなく mod_cgi を使用すると、フォーク時のオーバーヘッドやシグナル処理の競合が発生する可能性があります。
  • **環境変数の管理**: 外部プロセスに引き渡される環境変数のフィルタリングやサニタイズが不十分な場合、セキュリティ上の脆弱性につながる恐れがあります。
