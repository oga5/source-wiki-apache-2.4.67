COOKIE_NAME

概念

COOKIE_NAME は、Apache HTTP Server の複数のモジュール（mod_usertrack、mod_session_cookie、mod_session_dbd）において、クライアントとの間でやり取りされる Cookie の名前を設定・管理するための概念および設定項目です。各モジュールは、設定ディレクティブを介して指定された Cookie 名を解析・検証し、それぞれの設定構造体に格納します。

なぜ重要か

Cookie 名は、ユーザー追跡（mod_usertrack）やセッション管理（mod_session 関連モジュール）において、リクエストから特定の Cookie データを識別・抽出するためのキーとなります。正しく設定・検証されない場合、Cookie のパースエラーや、セッション情報の不整合、セキュリティ上の問題（不正な文字の混入など）を引き起こす可能性があります。また、複数のモジュール間で設定関数の再利用や参照関係があるため、変更時の影響範囲が広いという特徴があります。

エンドツーエンドの流れ

1. **設定のロード**: Apache の起動時または設定再読み込み時に、設定ファイルから CookieName、SessionCookieName、または SessionDBDCookieName ディレクティブが読み込まれます。 2. **設定の解析と検証**:

  • mod_usertrack の set_cookie_name では、指定された名前を cookie_dir_rec に保存し、Cookie 検索用の正規表現をコンパイルして、キャプチャグループ数が NUM_SUBS (3) と一致するか検証します。
  • mod_session_cookie の set_cookie_name では、引数から Cookie 名とオプションの属性（name_attrs）を分離し、check_string を用いて Cookie 名に不正な文字（= や & など）が含まれていないか検証します。

    • 3. **リクエスト処理時の利用**:

  • 設定された Cookie 名（および生成された正規表現）は、クライアントからのリクエストヘッダーから該当する Cookie を検索・抽出するために使用されます。ただし、リクエスト受信から Cookie 抽出、セッション復元に至る具体的なエンドツーエンドの実行フローは、提供されたコンテキストからは完全には明らかになっていません。

重要な不変条件

  • **正規表現のキャプチャグループ数**: mod_usertrack の set_cookie_name において、コンパイルされた正規表現のキャプチャグループ数（re_nsub + 1）は、必ず NUM_SUBS（3）と一致しなければなりません。
  • **Cookie 名の文字制限**: mod_session_cookie の set_cookie_name において、Cookie 名に = や & などの不正な文字が含まれていてはなりません（check_string による検証を通過する必要があります）。

よくある落とし穴

  • **モジュール間の相互影響**: set_cookie_name 関数は、mod_usertrack、mod_session_cookie、mod_session_dbd の間で相互に参照または再利用されています。そのため、1つのモジュールにおける set_cookie_name のシグネチャや挙動を変更すると、他のモジュールの設定処理に予期せぬ影響を与える可能性があります。
  • **正規表現の不整合**: mod_usertrack において NUM_SUBS 定数や set_and_comp_regexp の仕様を変更した場合、キャプチャグループ数の不整合により設定エラーが発生し、サーバーが起動しなくなります。
