COOKIE_NAME

概念

COOKIE_NAME は、Apache HTTP Server の複数のモジュール（mod_usertrack、mod_session_cookie、mod_session_dbd）において、クライアントとの間でやり取りされる Cookie の名前を設定・管理するための概念および設定値です。各モジュールは、設定ディレクティブ（CookieName、SessionCookieName、SessionDBDCookieName）を介して指定された Cookie 名を解析し、それぞれの設定構造体に格納して利用します。

なぜ重要か

Cookie 名は、ユーザートラッキング（mod_usertrack）やセッション管理（mod_session_cookie、mod_session_dbd）において、対象となる Cookie を特定するための識別子として機能します。正しい Cookie 名の設定と検証は、リクエストからの正確なデータ抽出や、不正な文字の混入防止などのセキュリティ・動作安定性の面で重要です。また、複数のモジュール間で同様の名称の設定関数（set_cookie_name）が定義・参照されているため、コードの保守時にも注意を払う必要があります。

関与するモジュール

主要エンティティ

  • **symbol:modules/metadata/mod_usertrack.c#COOKIE_NAME**: 横断トピックの対象となるシンボル。
  • **symbol:modules/metadata/mod_usertrack.c#set_cookie_name**: CookieName ディレクティブ等の設定時に呼び出され、Cookie 名を cookie_dir_rec に保存し、検索用の正規表現を生成・検証する関数。
  • **symbol:modules/session/mod_session_cookie.c#set_cookie_name**: SessionCookieName ディレクティブ等の設定時に呼び出され、Cookie 名とオプションの属性を解析して session_cookie_dir_conf に格納し、妥当性を検証する関数。
  • **symbol:modules/session/mod_session_dbd.c#set_cookie_name**: SessionDBDCookieName ディレクティブ等の設定に関連する関数（詳細な実装コードは未提供ですが、呼び出し元/呼び出し先として言及されています）。

エンドツーエンドの流れ

1. **設定のロード**: Apache の起動時または設定再読み込み時に、設定ファイル中の CookieName、SessionCookieName、または SessionDBDCookieName ディレクティブが検出されます。 2. **引数の解析と格納**:

  • mod_usertrack では、set_cookie_name が呼び出され、指定された名前を cookie_dir_rec->cookie_name に保存します。
  • mod_session_cookie では、set_cookie_name が呼び出され、引数から Cookie 名（conf->name）と属性（conf->name_attrs）を分割して格納します。

    • 3. **検証**:

  • mod_usertrack では、Cookie 名に基づく正規表現をコンパイルし、キャプチャグループ数が期待値（NUM_SUBS）と一致するか検証します。
  • mod_session_cookie では、check_string を用いて Cookie 名に不正な文字が含まれていないか検証します。

※なお、リクエスト処理時にこれらの設定された Cookie 名を用いて実際に Cookie を抽出・処理する具体的なエンドツーエンドの実行時フローの詳細は、提供された情報からは未だ不明です。

重要な不変条件

  • **正規表現のキャプチャグループ数**: mod_usertrack の set_cookie_name において、コンパイルされた正規表現のキャプチャグループ数（re_nsub + 1）は、必ず NUM_SUBS（値は 3）と一致しなければなりません。
  • **Cookie 名の文字制限**: mod_session_cookie の set_cookie_name において、check_string による検証により、Cookie 名に = や & などの不正な文字が含まれていてはなりません。

よくある落とし穴

  • **モジュール間の相互影響**: 提供された情報によると、mod_usertrack.c の set_cookie_name と mod_session_cookie.c の set_cookie_name は、互いのモジュールや mod_session_dbd.c から設定ディレクティブ（CookieName、SessionCookieName、SessionDBDCookieName）を介して参照・再利用されているとされています。そのため、一方の関数のシグネチャや振る舞いを変更すると、他方のモジュールに予期せぬ影響を及ぼす可能性があります。
  • **正規表現の不整合**: mod_usertrack において NUM_SUBS は正規表現の構造に直結しているため、これに関連するコードを変更する際は、set_and_comp_regexp との整合性を維持する必要があります。
