defconf

概念

defconf は、Apache HTTP Server の特定モジュール（mod_http2 および mod_md）において、デフォルトの設定値（Default Configuration）を定義・保持するために使用されるシンボル（静的変数または構造体）です。

なぜ重要か

各モジュールが初期化される際、または設定ファイル（httpd.conf など）で明示的なディレクティブが指定されていない場合に、システムが安全かつ一貫したデフォルトの挙動を適用するために重要となります。これにより、未設定の項目に対して適切なフォールバック値が提供されます。

関与するモジュール

  • module:modules/http2
  • module:modules/md

主要エンティティ

  • symbol:modules/http2/h2_config.c#defconf：HTTP/2 プロトコル制御モジュールにおけるデフォルト設定。
  • symbol:modules/md/mod_md_config.c#defconf：Managed Domains（ACME/Let's Encrypt 連携など）モジュールにおけるデフォルト設定。

エンドツーエンドの流れ

現時点では、これらの defconf シンボルがサーバー起動時にどのように初期化され、他の設定（バーチャルホスト固有の設定など）とどのようにマージされて最終的なランタイム設定に適用されるかという、具体的なエンドツーエンドの流れは判明していません。

重要な不変条件

現時点では、defconf の値が不変（Read-Only）として扱われるべきか、あるいは特定の設定フェーズ以降に変更が禁止されるかなどの、具体的な不変条件は判明していません。

よくある落とし穴

現時点では、これらのデフォルト設定シンボルを直接参照・変更する際、または設定のマージ処理（merge_dir_config や merge_server_config など）において発生しうる具体的な落とし穴は判明していません。
