ap_my_generation

概念

ap_my_generation は、Apache HTTP Server (httpd) の特定のマルチプロセッシングモジュール (MPM) において、現在のプロセスまたはスレッドが属する「世代 (generation)」を識別・保持するための変数（シンボル）です。

なぜ重要か

Apacheの再起動（特に graceful restart）の際、古い設定で動作しているプロセスと、新しい設定で起動された新しい世代のプロセスを区別する必要があります。ap_my_generation を参照することで、プロセスは自身が最新の世代に属しているか、あるいは古い世代として安全に終了すべきかを判断できます。

エンドツーエンドの流れ

親プロセスが起動または再起動を検知した際に世代番号を更新し、新しく生成される子プロセスやスレッドに対してその世代番号が ap_my_generation として設定されます。 ただし、これらのプラットフォーム（OS/2、NetWare）間での具体的な値の伝播方法や、再起動シグナル受信時の詳細な遷移フローなど、具体的なエンドツーエンドの流れは現時点では完全には判明していません。

重要な不変条件

各子プロセスまたはスレッドが保持する ap_my_generation は、そのプロセスが生成された時点の親プロセスの世代番号と一致している必要があります。 ただし、世代の更新タイミングや排他制御に関する具体的な不変条件は、現時点では完全には判明していません。

よくある落とし穴

OS/2 や NetWare といった特定のプラットフォーム向けMPM固有の実装であるため、Unix系の主要なMPM（event や worker など）やWindows向けの mpm_winnt とは、世代管理の仕組みや変数のスコープが異なる可能性があります。また、世代の不一致判定に誤りがあると、古いプロセスが終了せずに残り続けたり、新しい接続を処理中のプロセスが意図せず早期終了したりするリスクがあります。
