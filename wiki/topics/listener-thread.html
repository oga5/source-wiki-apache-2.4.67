listener_thread

概念

listener_thread は、Apache HTTP Server のマルチプロセッシングモジュール（MPM）である event および worker において、クライアントからの接続要求を監視・受付（accept）し、それらを処理スレッドへ分配する役割を持つスレッドのエントリポイント関数です。

なぜ重要か

マルチスレッド型のMPMにおいて、ネットワークソケットのリスニングと実際の接続処理（リクエスト処理）を分離し、効率的な並行処理を実現するために極めて重要な役割を果たします。これにより、接続の受付がボトルネックになるのを防ぎます。

  • **worker MPM の場合**:

    • 1. unblock_signal によりシグナルを解除します。 2. ap_queue_info_wait_for_idler を用いて、処理可能なアイドル状態のワーカースレッドが存在するか確認・待機します。 3. 接続を受け付けた後、ap_queue_push_socket を使用してソケットをキューにプッシュし、ワーカースレッドに処理を委ねます。 4. 終了時やエラー時には ap_close_listeners_ex や ap_queue_term、signal_threads を呼び出してクリーンアップとスレッド間連携を行います。

  • **event MPM の場合**:

    • 1. unblock_signal によりシグナルを解除します。 2. 非同期イベントやタイマーの管理（push_timer2worker、TO_QUEUE_REMOVE など）と連携しながら、接続の受付や管理を行います。 3. 終了時やエラー時には close_listeners や close_connection、signal_threads を呼び出します。
