create_config

概念

create_config は、Apache HTTP Server の各モジュールにおいて、サーバーごとの設定構造体を生成・初期化するためのコールバック関数（フック）です。提供されたコードベースからは、mod_cache_disk、mod_cache_socache、mod_proxy_connect などのモジュールで同名の静的関数（static void *create_config(apr_pool_t *p, server_rec *s)）として定義されていることが確認できます。

なぜ重要か

Apache HTTP Server の起動時や設定読み込みフェーズにおいて、各モジュールが独自の設定パラメータを保持するためのメモリ領域を適切に確保・初期化するために不可欠です。例えば、mod_proxy_connect では、接続を許可するポート範囲を管理する構造体（connect_conf）をこの段階で初期化します。

エンドツーエンドの流れ

Apache HTTP Server の設定フェーズにおいて、登録されたモジュールの設定作成フックとして呼び出されます。

mod_proxy_connect における具体的な流れは以下の通りです： 1. Apache の設定フェーズから、登録された create_config が呼び出されます（modules/proxy/mod_proxy_connect.c:400 にて構造体登録を確認）。 2. apr_pcalloc を使用して、メモリプール p から connect_conf 構造体分のメモリを確保します。 3. apr_array_make を呼び出し、許可されたポート範囲を格納するための配列 allowed_connect_ports を初期容量 10 で初期化します。 4. 初期化された構造体へのポインタを返します。

なお、mod_cache_disk および mod_cache_socache における具体的なエンドツーエンドの流れや、これらを含むシステム全体での詳細な呼び出しシーケンスは現時点では未解明です。

重要な不変条件

  • **メモリのゼロクリア**: mod_proxy_connect において、返される connect_conf 構造体は apr_pcalloc によりゼロクリアされています。
  • **初期容量の確保**: mod_proxy_connect の allowed_connect_ports 配列は、初期容量 10 の apr_array_header_t として確保されます。
  • mod_cache_disk および mod_cache_socache における不変条件は現時点では未解明です。

よくある落とし穴

  • **メモリ不足時の挙動**: mod_proxy_connect において、メモリ不足時には apr_pcalloc または apr_array_make がプロセスを終了させる可能性があります（APR の標準的な挙動）。
  • **構造体変更時の追従**: connect_conf の構造変更や、許可ポート以外の設定項目を追加する場合は、本関数での初期化処理も同期して更新する必要があります。
  • mod_cache_disk および mod_cache_socache における落とし穴は現時点では未解明です。
