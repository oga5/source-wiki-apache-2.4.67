substitute

概念

substitute（置換）は、Apache HTTP Serverの複数の異なるコンテキストにおいて、文字列やパターンの置き換え処理を実行するための共通的な概念および関数群です。主に、設定ファイル内のマクロ展開、レスポンスボディに対するフィルタリング、およびsedストリーム編集の3つの領域で個別に実装されています。

なぜ重要か

この概念は、サーバーの設定および出力コンテンツの動的な制御において極めて重要です。

  • **設定の効率化**: mod_macro において設定ファイルの重複を排除し、パラメータ化されたマクロを展開します。
  • **コンテンツの動的書き換え**: mod_substitute や mod_sed を通じて、バックエンドからのレスポンスボディをリアルタイムに検索・置換し、リンクの修正やコンテンツのカスタマイズを可能にします。

関与するモジュール

主要エンティティ

  • **symbol:modules/core/mod_macro.c#substitute**: 設定マクロのパースおよび展開時に、マクロ引数を実際の値に置換する関数。
  • **symbol:modules/filters/mod_substitute.c#substitute**: レスポンスフィルタとして動作し、設定されたルール（固定文字列または正規表現）に基づいてレスポンスボディのデータを置換する関数。
  • **symbol:modules/filters/sed1.c#substitute**: mod_sed フィルタの内部で、sed互換の置換コマンド（s/regexp/replacement/）を実行する関数。

エンドツーエンドの流れ

これらは異なるモジュールで独立して動作するため、単一の統合されたエンドツーエンドの流れは存在しません。それぞれのコンテキストにおける大まかな流れは以下の通りですが、詳細な内部フローはまだ完全には判明していません。

1. **設定マクロ展開時 (mod_macro)**:

  • サーバー起動時または再起動時に設定ファイルが読み込まれます。
  • マクロの使用箇所が検出されると、mod_macro.c の substitute が呼び出され、定義されたパラメータが実引数に置換された設定テキストが生成されます。

2. **レスポンスフィルタリング時 (mod_substitute / mod_sed)**:

  • クライアントへのレスポンス送信時、データバケットがフィルタチェーンを通過します。
  • mod_substitute.c または sed1.c の substitute が呼び出され、バケット内のデータに対してパターンマッチングと置換処理が適用されます。
  • 置換後のデータが次のフィルタへと渡されます。

重要な不変条件

各モジュールにおける置換処理の正確性やメモリ安全性を保証するための不変条件（例：バッファ境界の維持、置換ループの防止など）が存在すると考えられますが、現時点では具体的な不変条件は判明していません。

よくある落とし穴

  • **パフォーマンスへの影響**: レスポンスボディ全体に対して大規模な正規表現置換（mod_substitute や mod_sed）を適用すると、CPUおよびメモリの消費量が著しく増加する可能性があります。
  • **意図しない置換**: 置換ルールの適用順序や正規表現のパターン設計が不適切な場合、意図しない箇所まで書き換えられてしまうリスクがあります。
  • ※具体的なコードレベルでの落とし穴の詳細は、現時点では判明していません。
