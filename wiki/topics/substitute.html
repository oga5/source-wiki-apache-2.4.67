substitute

概念

substitute は、Apache HTTP Server 内の複数のモジュールにおいて、文字列の置換や変換を行うために定義されている内部関数群です。主に、設定ファイルにおけるマクロの展開処理や、レスポンスフィルタにおける文字列・正規表現の置換処理などで利用されます。

なぜ重要か

この概念は、サーバーの設定を動的かつ柔軟に生成する機能（マクロ展開）や、クライアントに返却されるコンテンツを動的に書き換える機能（フィルタ処理）を支える基盤となるため重要です。それぞれのモジュールが独自のコンテキストで substitute 処理を実装しています。

関与するモジュール

  • module:modules/core
  • module:modules/filters

主要エンティティ

  • **symbol:modules/core/mod_macro.c#substitute** (関数)

    • マクロ引数の置換を行う関数。substitute_macro_args から呼び出され、エスケープ文字数をカウントする number_of_escapes などを呼び出します。

  • **symbol:modules/filters/mod_substitute.c#substitute** (シンボル)

    • レスポンスフィルタモジュール mod_substitute における置換処理に関連するエンティティです。

  • **symbol:modules/filters/sed1.c#substitute** (関数)

    • sed フィルタにおける置換処理を実行する関数。dosub や match を呼び出して実際のパターンマッチングと置換を行います。

エンドツーエンドの流れ

各モジュールにおける substitute の具体的な呼び出しライフサイクルや、リクエスト・設定解析の全体にわたるエンドツーエンドの流れは、現時点では完全には解明されていません。

判明している範囲では、以下の呼び出し関係が存在します。

  • mod_macro.c 内では、マクロ展開時に substitute_macro_args から substitute が呼び出されます。
  • sed1.c 内では、sed 評価処理の過程で substitute が呼び出され、内部で match によるパターン検出と dosub による置換が実行されます。
