core_server_config

概念

core_server_config は、Apache HTTP Server においてサーバー単位（グローバルおよび仮想ホスト単位）の設定情報を保持するための構造体です。ドキュメントルート、アクセス制御、リクエスト処理の制限、プロトコル関連の動作設定、エラーログのフォーマットなどの設定項目を格納します。

なぜ重要か

この構造体は、Apache HTTP Server のコア機能（core_module）および各種プロトコル処理モジュール（HTTP や HTTP/2 など）が、リクエスト処理の制限（リダイレクトやサブreqの制限）やプロトコル適合性チェックを行う際の基準値として参照するため、サーバー全体の動作を制御する上で極めて重要です。

関与するモジュール

  • module:include
  • module:server

主要エンティティ

  • symbol:include/http_core.h#core_server_config（構造体）
  • symbol:server/core.c#create_core_server_config（初期化関数）

エンドツーエンドの流れ

1. **初期化**: サーバー起動時または設定読み込み時に、server/core.c 内の create_core_server_config が呼び出されます。

  • メモリプール（apr_pool_t）から apr_pcalloc を用いて core_server_config 構造体のメモリが割り当てられます。
  • グローバルサーバー（メインサーバー）か仮想ホスト（s->is_virtual）かに応じて、デフォルト値の設定やメインサーバーからの設定継承（error_log_format など）が行われます。

    • 2. **設定の適用**: 設定ファイル（httpd.conf 等）のディレクティブに基づいて、構造体の各フィールドに値が設定されます（※設定ファイルから構造体メンバーへの具体的なマッピングやパースのフローは、提供された情報からは未詳です）。 3. **参照**: リクエスト処理中、modules/http/http_filters.c、modules/http/http_protocol.c、modules/http2/h2_c2_filter.c、modules/http2/h2_headers.c などの各モジュールが ap_get_module_config を介してこの構造体を取得し、設定値を参照して動作を決定します。

重要な不変条件

  • trace_enable、http09_enable、http_conformance、http_methods などのフィールドは、それぞれ定義された定数（例: AP_TRACE_*）の範囲内の値でなければなりません。
  • sec_dir および sec_url は、apr_array_header_t として適切に初期化されている必要があります。
  • 仮想ホストの設定を初期化する際、メインサーバーの設定が未初期化である場合は ap_get_core_module_config の結果に依存します。
  • プラットフォームが Windows（WIN32）の場合のみ、構造体に unc_list フィールドが存在します。

よくある落とし穴

  • redirect_limit や subreq_limit の設定値を超えた場合、リクエスト処理が意図せず中断される可能性があります。
  • http_conformance を AP_HTTP_CONFORMANCE_STRICT に設定すると、非準拠な HTTP リクエストが拒否されるようになるため、クライアントの互換性に影響を及ぼす可能性があります。
  • 構造体に新しいメンバーを追加する場合、ap_mmn.h で定義されるモジュールバイナリ互換性（MMN）に影響を与えるため、注意が必要です。
  • プラットフォーム固有のアクセプトフィルタ設定（accf_map）が含まれているため、OS 間の挙動差異に注意する必要があります。
