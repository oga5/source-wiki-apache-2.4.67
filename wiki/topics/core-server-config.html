core_server_config

概念

core_server_config は、Apache HTTP Server においてサーバー単位（グローバルおよび仮想ホスト単位）の設定情報を保持するための構造体です。ドキュメントルート、アクセス制御、リクエスト処理の制限、およびプロトコル関連の動作設定など、コアとなる設定項目を格納します。

なぜ重要か

この構造体は、サーバーのライフサイクル全体を通じて、HTTPプロトコルの適合性チェック（http_conformance）、リクエストの再帰制限（redirect_limit, subreq_limit）、ドキュメントルート（ap_document_root）の参照など、リクエスト処理の基準値を提供するために不可欠です。各モジュールは ap_get_module_config を介してこの設定を参照し、動作を決定します。

関与するモジュール

  • module:include
  • module:server

主要エンティティ

  • symbol:include/http_core.h#core_server_config: サーバー単位の設定を保持する構造体定義。
  • symbol:server/core.c#create_core_server_config: core_server_config 構造体を生成・初期化し、メモリを割り当てる関数。

エンドツーエンドの流れ

1. **初期化**: サーバー起動時または仮想ホスト設定時に、server/core.c 内の create_core_server_config が呼び出されます。 2. **メモリ確保とデフォルト設定**: apr_pcalloc を用いて core_server_config のメモリが確保され、グローバルサーバーか仮想ホストかに応じてデフォルト値（アクセプトフィルタ、ドキュメントルート、配列の初期化など）が設定されます。 3. **設定の適用**: 設定ファイル（httpd.conf 等）のディレクティブに基づいて、構造体の各フィールドに値が設定されます（※設定ファイルから構造体へ値をマッピングする具体的なパース処理のフローは、提供されたコンテキストからは未詳です）。 4. **参照**: リクエスト処理中、各モジュール（modules/http/http_filters.c や modules/http/http_protocol.c など）が ap_get_module_config を介してこの構造体を取得し、設定値を参照して処理を行います。

重要な不変条件

  • trace_enable、http09_enable、http_conformance、http_methods などのフィールドは、定義された特定の定数（例: AP_TRACE_*）の範囲内で使用される必要があります。
  • sec_dir および sec_url は、apr_array_header_t として適切に初期化されている必要があります（create_core_server_config 内で apr_array_make により初期化されます）。
  • 仮想ホスト設定の初期化時、メインサーバーの設定から error_log_format などの一部の設定が継承されます。

よくある落とし穴

  • **プラットフォーム依存**: Windows環境（WIN32）では unc_list メンバーが追加されるため、プラットフォーム固有のコードを扱う際は #ifdef WIN32 による条件分岐を考慮する必要があります。また、アクセプトフィルタ（accf_map）の設定もOS依存の挙動を示します。
  • **バイナリ互換性**: core_server_config 構造体にメンバーを追加すると、ap_mmn.h で定義されるモジュールバイナリ互換性（MMN）に影響を与える可能性があります。
  • **制限値の超過と拒否**: redirect_limit や subreq_limit を超えた場合、リクエスト処理が中断される可能性があります。また、http_conformance が AP_HTTP_CONFORMANCE_STRICT に設定されている場合、非準拠なHTTPリクエストが拒否される可能性があります。
