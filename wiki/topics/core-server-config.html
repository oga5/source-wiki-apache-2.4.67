core_server_config

概念

core_server_config は、Apache HTTP Server においてサーバー単位（グローバルおよび仮想ホスト単位）の設定情報を保持するための構造体です。ドキュメントルート、アクセス制御、リクエスト処理の制限、およびプロトコル関連の動作設定などを格納します。

なぜ重要か

サーバーのライフサイクル全体を通じて、リクエスト処理の制限（リダイレクトやサブリクエストの最大回数など）やHTTPプロトコルの適合性チェック（http_conformance）の基準値として、様々なモジュール（modules/http/http_filters.c や modules/http/http_protocol.c など）から参照されるため、サーバー全体の動作を制御する基盤となります。

関与するモジュール

  • module:include
  • module:server

主要エンティティ

  • symbol:include/http_core.h#core_server_config：サーバー単位の設定を保持する構造体定義。
  • symbol:server/core.c#create_core_server_config：core_server_config 構造体を生成・初期化する関数。

エンドツーエンドの流れ

1. **初期化**: サーバー起動時、server/core.c 内の create_core_server_config が呼び出され、apr_pcalloc を用いて core_server_config 構造体のメモリが割り当てられます。

  • メインサーバー（グローバル）の場合は、デフォルトのドキュメントルートやアクセプトフィルタ（accf_map）などが設定されます。
  • 仮想ホストの場合は、メインサーバーからエラーログフォーマットを継承するなどの初期化が行われます。

    • 2. **設定の適用**: 設定ファイル（httpd.conf 等）のディレクティブに基づいて値が設定されます（※設定ファイルから本構造体への具体的なマッピングやパースの全体フローは、提供された情報からは未詳です）。 3. **参照**: リクエスト処理中、各モジュールが ap_get_module_config を介してこの構造体を取得し、設定値を参照して動作を決定します。

重要な不変条件

  • trace_enable、http09_enable、http_conformance、http_methods などの設定値は、それぞれ定義された定数（AP_TRACE_* など）の範囲内で使用される必要があります。
  • sec_dir および sec_url は apr_array_header_t として適切に初期化されている必要があります（create_core_server_config 内で初期化されます）。
  • 仮想ホスト設定の初期化時、メインサーバーの設定が未初期化である場合の挙動は、ap_get_core_module_config の結果に依存します。

よくある落とし穴

  • **リクエストの中断**: redirect_limit や subreq_limit の設定値を超えた場合、リクエスト処理が中断される可能性があります。
  • **リクエストの拒否**: http_conformance を AP_HTTP_CONFORMANCE_STRICT に設定すると、非準拠なHTTPリクエストが拒否される可能性があります。
  • **プラットフォーム依存**: Windows環境（WIN32）では unc_list メンバーが追加されるなど、プラットフォーム依存のコード（#ifdef WIN32）が存在するため、クロスプラットフォーム開発時に注意が必要です。
  • **バイナリ互換性**: 構造体にメンバーを追加すると、ap_mmn.h で定義されるモジュールバイナリ互換性（MMN）に影響を与える可能性があります。
