core_server_config

概念

core_server_config は、Apache HTTP Server においてサーバー単位（グローバルおよび仮想ホスト単位）の設定情報を保持するための構造体です。ドキュメントルート、アクセス制御、リクエスト処理の制限、プロトコル関連の動作設定など、サーバー全体の挙動を決定づけるコアな設定項目を管理します。

なぜ重要か

この構造体は、Apacheのコア機能（core_module）や各種プロトコル処理モジュール（HTTP、HTTP/2など）が、リクエスト処理の制限（リダイレクト制限やサブリクエスト制限など）やプロトコルの適合性チェックを行う際の基準値として参照するため、サーバー全体の動作の一貫性と安全性を担保する上で極めて重要です。

関与するモジュール

エンドツーエンドの流れ

1. **初期化**: サーバー起動時または仮想ホスト作成時に、server/core.c の create_core_server_config が呼び出されます。

  • グローバルサーバー（!is_virtual）の場合、デフォルトのドキュメントルートやアクセプトフィルタ（accf_map）などが設定されます。
  • 仮想ホストの場合、メインサーバーの設定からエラーログフォーマットなどを継承し、一部の設定を初期化します。

    • 2. **設定の適用**: 設定ファイル（httpd.conf 等）のディレクティブにより、構造体内の各フィールドに値が設定されます。*（※設定ファイルからの詳細なパースやマージの具体的なフローは、提供された情報からは未確定です）* 3. **参照**: リクエスト処理中、modules/http/http_filters.c や modules/http/http_protocol.c などの各モジュールが ap_get_module_config を介してこの構造体を取得し、設定値（redirect_limit や http_conformance など）を参照して処理を制御します。

重要な不変条件

  • trace_enable、http09_enable、http_conformance、http_methods などの設定値は、それぞれ定義された定数（例: AP_TRACE_*）の範囲内で使用される必要があります。
  • sec_dir および sec_url は apr_array_header_t として適切に初期化されている必要があります。
  • 仮想ホスト設定の生成時、メインサーバーの設定が未初期化である場合は ap_get_core_module_config の結果に依存します。*（※実行時におけるその他の詳細な不変条件は未確定です）*

よくある落とし穴

  • **プラットフォーム依存**: Windows環境では unc_list が追加される（#ifdef WIN32）ため、プラットフォーム固有のコードを扱う際に注意が必要です。また、アクセプトフィルタ（accf_map）の設定もOS依存の挙動を示します。
  • **バイナリ互換性**: 構造体 core_server_config にメンバーを追加すると、ap_mmn.h で定義されるモジュールバイナリ互換性（MMN）に影響を与える可能性があります。
  • **制限値超過と拒否**: redirect_limit や subreq_limit の設定値を超えた場合、リクエスト処理が中断される可能性があります。また、http_conformance を AP_HTTP_CONFORMANCE_STRICT に設定すると、非準拠なHTTPリクエストが拒否されるようになります。
