ap_socache_instance_t

概念

ap_socache_instance_t は、Apache HTTP Server における小規模オブジェクトキャッシュ（socache）の各バックエンド実装（プロバイダー）において、個別のキャッシュインスタンスの状態やコンテキストを保持するための構造体です。

共通ヘッダーである include/ap_socache.h においては、具体的な内部定義を持たない不透明な（opaque）型として前方宣言されています。実際の構造体のメンバ定義は、各キャッシュプロバイダーのソースコード（mod_socache_dbm.c、mod_socache_dc.c、mod_socache_memcache.c、mod_socache_redis.c、mod_socache_shmcb.c）の中で個別に定義されています。

なぜ重要か

この型は、キャッシュを利用する上位モジュール（mod_authn_socache や mod_cache_socache など）に対して、バックエンドの具体的な実装詳細を隠蔽する抽象化レイヤーとして機能します。

上位モジュールは、バックエンドが共有メモリ（SHMCB）、データベース（DBM）、Memcached、Redis、あるいは分散キャッシュ（distcache）のいずれであるかを意識することなく、統一された ap_socache_provider_t インターフェースを介して、このインスタンスへのポインタを受け渡しすることでキャッシュ操作を行うことができます。

エンドツーエンドの流れ

1. **生成**: 上位モジュールがプロバイダーの create メソッド（例: socache_mc_create）を呼び出し、対応するプロバイダー固有の ap_socache_instance_t 構造体をメモリプール上に確保します。 2. **初期化**: init メソッド（例: socache_rd_init）を呼び出し、接続の確立、共有メモリの確保、あるいは内部構造体の初期化を行います。 3. **操作**: store、retrieve、remove などのキャッシュ操作メソッドの第一引数としてこのインスタンスへのポインタを渡し、データの読み書きを行います。 4. **破棄**: destroy メソッドを呼び出して、確保したリソースを解放します（実装によっては no-op の場合もあります）。

※ただし、上位モジュール（mod_cache_socache や mod_authn_socache）における具体的な呼び出しシーケンスの全容や、エラーハンドリングを含む完全なエンドツーエンドのフローの詳細は、提供されたコンテキストからは完全には明らかになっていません。

重要な不変条件

  • **初期化の必須性**: init メソッドが正常に完了するまで、このインスタンスを他のキャッシュ操作メソッドに渡して使用することはできません。
  • **排他制御の前提**:
  • mod_socache_dbm.c では、pool メンバは対応する mutex が保持されている間のみ使用可能です。
  • mod_socache_shmcb.c は AP_SOCACHE_FLAG_NOTMPSAFE を指定しており、インスタンス自体にロック機構は含まれません。呼び出し側での排他制御が前提となります。
  • **サイズ・長さの制限**:
  • mod_socache_redis.c では、名前空間タグの長さ（taglen）が 16 バイト以下である必要があります（デバッグビルドでは AP_DEBUG_ASSERT で検証されます）。
  • mod_socache_memcache.c では、tag の長さが MC_KEY_LEN（254バイト）を超えるとキー生成時にエラーが発生します。

よくある落とし穴

  • **直接的なメンバアクセスの禁止**: ap_socache_instance_t は不透明な型として定義されているため、呼び出し側から直接構造体のメンバ（shm や rc など）にアクセスすることはできません。必ず ap_socache_provider_t の関数ポインタを経由する必要があります。
  • **スレッドセーフティの欠如**: DBM や SHMCB などのプロバイダーは自身で排他制御を行わないため、呼び出し側で適切にロックを管理しないと、メモリ競合や未定義動作を招く原因になります。
  • **初期化未完了でのアクセス**: init が失敗した、あるいは呼び出されていない状態でインスタンスを操作すると、未定義の動作やアサーションエラー（例: Redis での rc 未初期化、SHMCB での shm が NULL など）を引き起こす可能性があります。
