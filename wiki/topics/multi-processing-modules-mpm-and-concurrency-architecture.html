Multi-Processing Modules (MPM) and Concurrency Architecture

概念

Multi-Processing Modules (MPM) は、Apache HTTP Server がネットワークポートをバインドし、クライアントからのリクエストを受け付け、それらを処理する子プロセスやスレッドに割り当てるための並行処理アーキテクチャを抽象化したモジュール群です。システムのリソースや要求されるパフォーマンス特性に応じて、異なる並行処理モデルを選択できるように設計されています。

なぜ重要か

Webサーバーが動作するオペレーティングシステムの特性や、デプロイされるアプリケーションの要件（スレッドセーフティの有無、同時接続数など）に適合させるために重要です。適切なMPMを選択・構成することで、メモリ消費の最適化、スループットの向上、高並行接続時のリソース枯渇の防止が可能になります。

関与するモジュール

本アーキテクチャには以下のモジュールが関与しています。

  • module:server/mpm/event: 非同期イベント駆動モデルを採用し、キープアライブ接続などの処理を効率化するMPM。
  • module:server/mpm/prefork: スレッドを使用せず、リクエストごとに独立したプロセスを事前にフォークして処理する、スレッドセーフでない環境に適したMPM。
  • module:server/mpm/worker: マルチプロセスとマルチスレッドのハイブリッドモデルを採用し、高スループットと省メモリを両立するMPM。

主要エンティティ

本アーキテクチャを構成する主要なソースファイルは以下の通りです。

  • server/mpm/event/event.c: Event MPM のメイン実装。
  • server/mpm/prefork/prefork.c: Prefork MPM のメイン実装。
  • server/mpm/worker/worker.c: Worker MPM のメイン実装。

※ 各ファイル内の具体的な関数やデータ構造などの詳細なエンティティ定義は、提供されたコンテキストからは特定されていません。
