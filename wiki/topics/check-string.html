check_string

概念

check_string は、Apache HTTP Serverの特定のモジュール（認証やセッション管理など）において、設定ディレクティブで指定された文字列（Cookie名やフォーム認証の設定値など）が、仕様上禁止されている文字を含んでいないか、または空でないかを検証するためのバリデーション関数です。

なぜ重要か

この関数は、クッキーやフォーム認証の処理において、区切り文字として使用される特別な文字（=、&、; など）が設定値に混入することを防ぎます。これにより、設定の不正なパースや、意図しない動作、セキュリティ上の脆弱性を未然に防ぐ役割を果たします。

関与するモジュール

  • module:modules/aaa
  • module:modules/session

主要エンティティ

  • symbol:modules/aaa/mod_auth_form.c#check_string: フォーム認証の設定値（location, username, passwordなど）に = や & が含まれていないか、また空でないかを検証します。
  • symbol:modules/session/mod_session_cookie.c#check_string: クッキー名などの設定値に = や & が含まれていないか、また空でないかを検証します。
  • symbol:modules/session/mod_session_dbd.c#check_string: ap_cookie_check_string を使用して、設定値にクッキーの仕様上禁止されている文字（=, ;, &）が含まれていないかを検証します。

エンドツーエンドの流れ

1. **設定のロードと解析**: Apacheの起動時または設定再読み込み時に、各モジュールの設定ディレクティブ（例: mod_session_cookie の set_cookie_name や mod_session_dbd の設定など）が処理されます。 2. **検証の実行**: 各設定ディレクティブの解析処理から、それぞれのモジュール内に定義された check_string が呼び出されます。 3. **文字列の検査**:

  • mod_auth_form.c および mod_session_cookie.c では、対象文字列が NULL または空でないかを確認し、ap_strchr_c を用いて = または & が含まれていないかを直接検査します。
  • mod_session_dbd.c では、ap_cookie_check_string を呼び出して =, ;, & の有無を検査します。

    • 4. **結果の返却**:

  • 検証を通過した場合、NULL を返して処理を継続します。
  • 検証に失敗した場合、cmd->pool を用いてエラーメッセージを構築し、それを返却して設定エラーとします。

※なお、提供された情報には各 static 関数が他モジュールからも呼び出されている旨の記述がありますが、これらが完全に独立して動作しているのか、あるいは何らかの形で共有・連携しているのかの正確なエンドツーエンドの呼び出し関係の全貌は完全には判明していません。

重要な不変条件

  • **メモリプールの有効性**: 呼び出し時に渡される cmd->pool が有効なメモリプールでなければなりません（エラーメッセージの生成に使用されるため）。
  • **禁止文字の不検出**: 検証を通過するためには、対象文字列に以下の文字が含まれていてはなりません。
  • mod_auth_form.c / mod_session_cookie.c: = および &
  • mod_session_dbd.c: =, ;, および &

よくある落とし穴

  • **実装と検証ルールの差異**: 同名の check_string 関数が複数のファイルに static 関数として定義されていますが、mod_session_dbd.c の実装のみ ; の混入も禁止（ap_cookie_check_string を使用）しているなど、検証ルールや内部実装に差異があります。共通のバリデーターとして認識して一括で変更を加える際には、モジュールごとの仕様の違いに注意する必要があります。
  • **エラーハンドリング**: 検証失敗時に返されるエラーメッセージの生成は apr_pstrcat によるメモリ割り当てに依存しているため、メモリ不足時の挙動に注意が必要です。
