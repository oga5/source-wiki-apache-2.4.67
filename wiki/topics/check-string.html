check_string

概念

check_string は、Apache HTTP Server の認証（AAA）およびセッション管理モジュールにおいて、設定ディレクティブで指定された文字列（クッキー名やフォーム認証用のパラメータなど）の妥当性を検証するためのヘルパー関数です。主に、クッキーやフォームの処理において特別な意味を持つ禁止文字（=, &, ; など）が設定値に含まれていないこと、および値が空でないことを検証する役割を持ちます。

なぜ重要か

設定値に =、&、; などの文字が含まれていると、クッキーのパースやフォームデータの解析処理において、キーと値の境界が曖昧になり、意図しない動作やセキュリティ上の脆弱性を引き起こす可能性があります。check_string は、設定の読み込み（パース）段階でこれらの不正な文字列を検出し、エラーメッセージを返すことで、安全な設定のみが適用されることを保証します。

関与するモジュール

  • module:modules/aaa
  • module:modules/session

主要エンティティ

  • symbol:modules/aaa/mod_auth_form.c#check_string: フォーム認証の設定（location, username, password など）において、空文字列や =、& の有無を検証します。
  • symbol:modules/session/mod_session_cookie.c#check_string: クッキーセッション設定（Cookie名など）において、空文字列や =、& の有無を検証します。
  • symbol:modules/session/mod_session_dbd.c#check_string: DBDセッション設定において、ap_cookie_check_string を使用して =, ;, & の有無を検証します。

エンドツーエンドの流れ

設定ファイル（httpd.confなど）の解析時に、各モジュールの設定ディレクティブハンドラから check_string が呼び出されます。

1. **設定の読み込み**: mod_auth_form、mod_session_cookie、mod_session_dbd の各設定ディレクティブ（例: クッキー名やフォームパラメータの設定）が処理される際、引数として渡された文字列が check_string に渡されます。 2. **検証処理**:

  • mod_auth_form.c および mod_session_cookie.c では、文字列が NULL または空でないかを確認し、ap_strchr_c を用いて = や & が含まれていないかを直接検証します。
  • mod_session_dbd.c では、外部関数 ap_cookie_check_string を呼び出して =, ;, & の有無を検証します。

    • 3. **結果の返却**:

  • 検証を通過した場合、NULL を返して処理を継続します。
  • 検証に失敗した場合、cmd->pool を用いてエラーメッセージを生成して返し、設定エラーとして処理を中断します。

※なお、各モジュール間での具体的な呼び出し関係（提供された情報にある、あるモジュールの check_string が他モジュールから呼び出されているとされる記述）の物理的な結合や、詳細な実行時エラーハンドリングの全体像（エンドツーエンドの完全なフロー）については、提供されたシンボル情報からは不変の事実として特定できていません。

重要な不変条件

  • **禁止文字の排除**: 検証対象の文字列に、仕様上禁止されている文字（= および &、mod_session_dbd ではさらに ;）が含まれていてはなりません。
  • **非空値の保証**: mod_auth_form.c および mod_session_cookie.c の検証において、対象文字列は NULL または空文字列であってはなりません。
  • **メモリプールの使用**: エラーメッセージの生成には、常に cmd->pool から割り当てられた有効なメモリ（apr_pstrcat 等による）を使用しなければなりません。

よくある落とし穴

  • **実装の不一致**: mod_auth_form.c / mod_session_cookie.c と mod_session_dbd.c とで、禁止文字の定義や検証方法（直接の文字検索か、ap_cookie_check_string の使用か）が微妙に異なります。特に ; の扱いなどに差異があるため、共通のバリデーターとして扱う際には注意が必要です。
  • **静的（static）関数のスコープ**: 各ファイルで static 関数として定義されているため、モジュールを跨いだ変更を行う場合は、それぞれのファイルに定義された check_string の挙動を個別に同期させる必要があります。
