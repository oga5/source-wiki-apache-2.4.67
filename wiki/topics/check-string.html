check_string

概念

check_string は、Apache HTTP Server の認証（AAA）およびセッション管理モジュールにおいて、設定ディレクティブ（Cookie名、フォーム認証設定、DBDセッション設定など）に指定された文字列が、仕様上禁止されている文字を含んでいないか、または空でないかを検証するためのバリデーション関数です。

なぜ重要か

この関数は、クッキーやフォーム認証の内部処理において、セパレータや特殊文字として扱われる文字（=、&、; など）が設定値に混入することを防ぎます。これにより、設定値の不正なパースや、セッション・認証処理における予期しない挙動、セキュリティ上の脆弱性を未然に防ぐ役割を果たします。

関与するモジュール

  • module:modules/aaa
  • module:modules/session

主要エンティティ

  • **symbol:modules/aaa/mod_auth_form.c#check_string**

    • フォーム認証の設定値（location, username, password, method, mimetype, body）を検証します。= および & を禁止文字とします。

  • **symbol:modules/session/mod_session_cookie.c#check_string**

    • クッキー設定（Cookie名など）を検証します。= および & を禁止文字とします。

  • **symbol:modules/session/mod_session_dbd.c#check_string**

    • DBDセッション設定を検証します。ap_cookie_check_string を用いて、=, ;, & を禁止文字として検証します。

エンドツーエンドの流れ

設定ファイルの読み込み時（設定ディレクティブの解析フェーズ）において、各モジュールの設定ハンドラから check_string が呼び出されます。

1. **入力**: 設定コマンドのパラメータ構造体（cmd_parms）と、検証対象の文字列（string）が渡されます。 2. **検証**:

  • mod_auth_form.c および mod_session_cookie.c では、文字列が NULL または空でないかを確認し、ap_strchr_c を用いて = または & が含まれていないかを直接検査します。
  • mod_session_dbd.c では、ap_cookie_check_string を呼び出して、=, ;, & の有無を検査します。

    • 3. **出力**:

  • 検証を通過した場合、NULL を返して処理を継続します。
  • 検証に失敗した場合、cmd->pool を使用してエラーメッセージを構築し、それを返します。これにより、Apache の起動時または再起動時に設定エラーとして報告されます。

※なお、リクエスト処理時の動的なバリデーションなど、設定解析フェーズ以外での詳細なエンドツーエンドの実行フローは現時点では不明です。

重要な不変条件

  • **メモリプールの有効性**: 検証エラー時にエラーメッセージを生成するため、cmd->pool が有効なメモリプールを指している必要があります。
  • **禁止文字の不検出**:
  • mod_auth_form.c および mod_session_cookie.c では、対象文字列に = および & が含まれていてはなりません。
  • mod_session_dbd.c では、対象文字列に =, ;, & が含まれていてはなりません。

よくある落とし穴

  • **モジュール間での実装の差異**: mod_session_dbd.c の check_string は ap_cookie_check_string を使用して ; も禁止文字として検証しますが、他の2つの実装では ; の検証が明示されていません。共通のバリデーターとして扱われる一方で、禁止文字の定義が微妙に異なる点に注意が必要です。
  • **静的（static）関数の重複**: 各ファイルに static const char *check_string として定義されているため、グローバルな共通関数ではなく、ファイルごとに独立して実装されています。変更を加える際は、それぞれのファイルに個別に修正を適用する必要があります。
