Module: modules/ssl

概念

modules/ssl は、Apache HTTP Server（httpd）において SSL/TLS プロトコルを用いた暗号化通信（HTTPS）を実現するためのモジュール（主に mod_ssl）の実装コード群です。OpenSSL などの暗号ライブラリと連携し、セキュアな接続、クライアント証明書による認証、セッションキャッシュ、OCSP ステープリングなどの機能を提供します。

なぜ重要か

現代の Web サーバーにおいて、通信の暗号化（HTTPS）は必須の要件です。modules/ssl は、機密データの保護、通信相手の認証、データの改ざん防止を担うコアコンポーネントであり、パフォーマンス向上のためのセッション再利用や、証明書の有効性検証（OCSP）など、セキュアな Web サービスを運用するための必須機能を提供します。

関与するモジュール

主要エンティティ

提供されたソースファイル群から、以下の主要なコンポーネントが構成されています。

  • **mod_ssl.c / mod_ssl.h**: モジュールの登録、フックの設定、および外部公開 API の定義。
  • **ssl_private.h**: モジュール内部で共有されるデータ構造、マクロ、および関数プロトタイプ。
  • **ssl_engine_init.c**: サーバー起動時における SSL コンテキストの作成、証明書・秘密鍵のロード、および OpenSSL の初期化。
  • **ssl_engine_io.c**: 接続に対する入力/出力フィルタの挿入と、SSL/TLS を介したデータの送受信処理。
  • **ssl_engine_kernel.c**: TLS ハンドシェイクの制御、アクセス制御（ディレクトリごとの SSL 設定適用）、およびプロトコル処理のコアロジック。
  • **ssl_engine_pphrase.c**: 起動時に暗号化された秘密鍵をロードするためのパスフレーズ取得処理。
  • **ssl_scache.c**: SSL セッションのキャッシュ管理（パフォーマンス向上のためのセッション再利用）。
  • **ssl_engine_mutex.c**: 複数プロセス/スレッド間での排他制御（セッションキャッシュや OpenSSL 内部ロック用）。
  • **ssl_util_stapling.c / ssl_engine_ocsp.c**: OCSP（Online Certificate Status Protocol）および OCSP ステープリングの処理。

エンドツーエンドの流れ

一般的な SSL/TLS 処理の流れは以下の通り推論されますが、詳細な内部状態遷移や OpenSSL API との厳密な相互作用のフローは完全には検証されていません。

1. **初期化フェーズ (ssl_engine_init.c, ssl_engine_pphrase.c)**: サーバー起動時に設定ファイルを読み込み、SSL コンテキストを初期化します。秘密鍵が暗号化されている場合は、パスフレーズの問い合わせが発生します。 2. **接続確立フェーズ (ssl_engine_io.c, ssl_engine_kernel.c)**: クライアントからの接続を検知すると、SSL I/O フィルタが接続ラインに挿入され、TLS ハンドシェイクが開始されます。 3. **リクエスト処理フェーズ (ssl_engine_kernel.c, ssl_engine_vars.c)**: ハンドシェイク完了後、暗号化された HTTP リクエストが復号され、後続のハンドラに渡されます。必要に応じて、SSL 関連の環境変数が提供されます。

重要な不変条件

ファイル構成から推論される不変条件は以下の通りですが、コードレベルでの厳密な不変条件はまだ完全には特定されていません。

  • **鍵と証明書の整合性**: 初期化時（ssl_engine_init.c）に、ロードされた証明書と秘密鍵がペアとして正しく一致している必要があります。
  • **スレッド安全性**: マルチプロセス/マルチスレッド環境において、OpenSSL のグローバルな状態や共有セッションキャッシュへのアクセスは、ssl_engine_mutex.c で管理されるロックによって保護されていなければなりません。

よくある落とし穴

  • **パスフレーズのブロック**: 起動時に秘密鍵のパスフレーズ入力を自動化（スクリプト等を使用）していない場合、バックグラウンド起動時にプロセスがハングしたように見えることがあります（ssl_engine_pphrase.c 関連）。
  • **セッションキャッシュの不整合**: 共有メモリ（shmcb）などのセッションキャッシュ（ssl_scache.c）の設定が不適切な場合、マルチプロセス間でセッションの再利用に失敗し、ハンドシェイクのオーバーヘッドが増大する可能性があります。
  • **OCSP ステープリングのタイムアウト**: 外部 OCSP レスポンダーへの通信が遮断されている場合、ハンドシェイク時に遅延が発生したり、接続エラーが発生したりすることがあります（ssl_util_stapling.c 関連）。
