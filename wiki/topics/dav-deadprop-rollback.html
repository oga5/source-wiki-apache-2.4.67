dav_deadprop_rollback

概念

dav_deadprop_rollback は、Apache HTTP Serverの mod_dav モジュールにおいて、WebDAVのデッドプロパティ（Dead Properties）を更新する際、処理の失敗時に元のプロパティ値を復元（ロールバック）するために使用されるデータ構造です。

なぜ重要か

WebDAVプロパティの更新処理において、途中でエラーが発生した場合にシステムの一貫性を保つためには、変更前の状態にロールバックする必要があります。dav_deadprop_rollback は、変更前のキーと値（apr_datum_t）を一時的に退避させておくことで、このロールバック処理を確実に実行できるようにするために重要です。

関与するモジュール

  • module:modules/dav/fs
  • module:modules/dav/main

主要エンティティ

  • symbol:modules/dav/fs/dbm.c#dav_deadprop_rollback: DBMプロパティデータベース用のロールバック情報を保持する構造体。
  • symbol:modules/dav/main/mod_dav.h#dav_deadprop_rollback: mod_dav の共通ヘッダーにおける定義。

エンドツーエンドの流れ

具体的な関数呼び出しの連鎖や、上位レイヤー（modules/dav/main/props.c など）から本構造体が操作される詳細なエンドツーエンドの制御フローは、提供されたソース情報からは完全には特定されていません。

一般的には、プロパティ更新の開始時に変更前の key と value が dav_deadprop_rollback に退避され、更新が失敗した際にこの構造体から元の値がDBMに再書き込みされることでロールバックが行われます。

重要な不変条件

  • **メモリ管理の依存性**: apr_datum_t 型のメンバ（key、value）を使用しているため、格納されるデータの生存期間は関連する apr_pool_t のライフサイクルに依存します。
  • **プロバイダー固有のプライベート性**: 本構造体は mod_dav のプロバイダー実装におけるプライベートなデータ構造として扱われます。

よくある落とし穴

  • **メモリ不足時の挙動**: apr_pcalloc 等によるロールバック構造体自体のメモリ確保に失敗した場合、ロールバック情報を生成できず、エラー発生時に不整合な状態が残る可能性があります。
  • **DBMの破損**: ロールバック処理の実行時にDBMファイル自体が破損している場合、復元操作そのものが失敗する可能性があります。
  • **定義の二重管理**: modules/dav/fs/dbm.c と modules/dav/main/mod_dav.h の両方に定義が存在するため、構造体を変更する際は両者の整合性を保つ必要があります。
