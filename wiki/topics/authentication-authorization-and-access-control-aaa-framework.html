Authentication, Authorization, and Access Control (AAA) Framework

概念

Authentication, Authorization, and Access Control (AAA) Framework は、Apache HTTP Serverにおけるセキュリティ制御の基盤です。このフレームワークは、ユーザーの身元を検証する「認証（Authentication）」、検証されたユーザーやリクエストに権限を付与する「認可（Authorization）」、およびIPアドレスなどの条件に基づく「アクセス制御（Access Control）」を、モジュール化されたプロバイダアーキテクチャを通じて統合的に管理します。

なぜ重要か

AAAフレームワークは、セキュリティの関心事をコアロジックから分離し、柔軟で拡張可能なアクセス制御を可能にするために重要です。mod_authn_core や mod_authz_core を通じて、認証・認可プロバイダの登録、エイリアスの作成、および複雑な論理条件（<RequireAll>、<RequireAny>など）の評価を統一された方法で処理できます。これにより、Basic認証（mod_auth_basic）などの具体的な認証方式を容易に組み込むことができます。

関与するモジュール

  • module:include (共通定義の提供)
  • module:modules/aaa (AAA機能のコアおよび具体的なプロバイダ実装)

主要エンティティ

  • **include/mod_auth.h**: 認証および認可プロバイダが実装すべき共通の構造体やインターフェースを定義します。
  • **modules/aaa/mod_authn_core.c**: 認証プロバイダの登録、管理、および共通ディレクティブの処理を担当するコアモジュールです。
  • **modules/aaa/mod_authz_core.c**: 認可プロバイダの管理、および Require ディレクティブによる論理コンテナの評価を担当するコアモジュールです。
  • **modules/aaa/mod_auth_basic.c**: HTTP Basic認証スキームを実装し、登録された認証プロバイダを利用してユーザー資格情報を検証します。

エンドツーエンドの流れ

リクエスト処理における詳細な関数呼び出しのシーケンスやエンドツーエンドの完全な流れは、提供されたソースコードの範囲からは**完全には判明していません**。

概念的には以下の流れで処理が行われます： 1. クライアントからのリクエストを受信すると、mod_auth_basic などの認証ハンドラが起動し、クライアントから送られた資格情報を抽出します。 2. mod_authn_core に登録された認証プロバイダ（例: ファイル、データベースなど）を呼び出し、資格情報の妥当性を検証します。 3. 認証が成功すると、mod_authz_core が設定された Require ルール（およびその論理コンテナ）に基づいて、認証されたユーザーまたはリクエスト属性がアクセス基準を満たしているかを評価します。

重要な不変条件

詳細なコードレベルの不変条件は、提供された情報からは**完全には判明していません**。

一般的な設計上の不変条件として以下が挙げられます：

  • 認可（Authorization）フェーズは、認証（Authentication）フェーズが成功するか、またはアクセス制御ルールによって明示的にバイパスが許可されない限り、アクセスを拒否しなければなりません。

よくある落とし穴

具体的な実装上の落とし穴やバグの傾向は、提供された情報からは**完全には判明していません**。

一般的な構成上の注意点として以下が挙げられます：

  • <RequireAll> や <RequireAny> などの認可コンテナをネストする際、評価順序やデフォルトの挙動（すべて拒否か、いずれか許可か）を誤解すると、意図しないアクセス許可や拒否が発生する可能性があります。
  • 認証プロバイダ（AuthBasicProvider など）が正しく登録または設定されていない場合、認証フェーズでエラーが発生し、リクエストが拒否されます。

次に読む場所

  • include/mod_auth.h (プロバイダインターフェースの定義)
  • modules/aaa/mod_authn_core.c (認証コアの登録と管理)
  • modules/aaa/mod_authz_core.c (認可コアとRequireコンテナの処理)
  • modules/aaa/mod_auth_basic.c (Basic認証の実装例)
