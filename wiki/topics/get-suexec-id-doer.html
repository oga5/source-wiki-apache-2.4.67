get_suexec_id_doer

概念

get_suexec_id_doer は、Apache HTTP Server（httpd-2.4.67）において、suEXEC（ユーザー権限でのプログラム実行）を適用する際に、リクエストに対応する実行ユーザー（UID）およびグループ（GID）の識別情報（ap_unix_identity_t）を取得・解決するための関数です。

この名前を持つ関数は、mod_suexec と mod_userdir の2つの異なるモジュールにそれぞれ定義されており、それぞれの文脈（ディレクトリごとの設定、またはユーザーディレクトリの解決）に応じて適切なユーザー/グループ情報を返します。

なぜ重要か

Apache が CGI などの外部プログラムを実行する際、セキュリティ上の理由から特定のユーザー権限（suEXEC）で実行する必要があります。get_suexec_id_doer は、ap_hook_get_suexec_identity フックを介して、リクエストに応じた適切な UNIX 実行権限情報を動的に提供する役割を担うため重要です。これにより、ディレクトリごとの設定（mod_suexec）や、ユーザーのホームディレクトリ（mod_userdir）に応じた適切な権限分離が実現されます。

関与するモジュール

  • module:modules/generators
  • module:modules/mappers

主要エンティティ

  • **symbol:modules/generators/mod_suexec.c#get_suexec_id_doer**: ディレクトリごとの設定（suexec_config_t）から、有効化されている場合に設定済みの ap_unix_identity_t を返します。
  • **symbol:modules/mappers/mod_userdir.c#get_suexec_id_doer**: リクエストに関連付けられたユーザー名から、システム（OS）のユーザーデータベース（apr_uid_get）を介して UID/GID を動的に解決し、ap_unix_identity_t を生成して返します。

エンドツーエンドの流れ

1. **フックの登録**: 両モジュールは、初期化時に ap_hook_get_suexec_identity フックに対してそれぞれの get_suexec_id_doer をハンドラとして登録します。 2. **リクエスト処理時の呼び出し**: リクエスト処理中（例えば mod_cgi などが suEXEC 権限を必要とする際）、このフックが呼び出されます。 3. **個別モジュールでの解決**:

  • **mod_suexec 側**: リクエストのディレクトリ設定から suexec_config_t を取得し、cfg->active が真であれば設定済みの cfg->ugid を返します。
  • **mod_userdir 側**: リクエストの設定からユーザー名を取得し、apr_uid_get を用いて OS から UID/GID を取得して、userdir フラグを 1 に設定した ap_unix_identity_t を返します。

    • 4. **権限の適用**: 取得された ap_unix_identity_t に基づき、最終的に外部の suexec バイナリなどを通じて権限が切り替えられます。

*注意: フック呼び出し以降の具体的なエンドツーエンドの実行フローや、戻り値が NULL の場合の呼び出し側（mod_cgi 等）の厳密なフォールバック挙動の詳細は、提供された情報からは完全には明らかになっていません。*

重要な不変条件

  • **mod_suexec における不変条件**:
  • r->per_dir_config から suexec_module に関連付けられた設定が取得可能である必要があります。
  • suexec_config_t の active フラグが真である場合のみ、有効な ugid が返されます。
  • **mod_userdir における不変条件**:
  • APR_HAS_USER が定義されている環境でのみ動作します。未定義の場合は常に NULL を返します。
  • リクエストの request_config に userdir_module キーでユーザー名が格納されている必要があります。

よくある落とし穴

  • **設定の無効化による NULL 返却**: mod_suexec で cfg->active が偽の場合、または mod_userdir でユーザー名が設定されていない場合、関数は NULL を返します。この場合、suEXEC は適用されませんが、その後のエラーハンドリングや挙動は呼び出し側の実装に依存します。
  • **OS依存のユーザー解決**: mod_userdir の実装は apr_uid_get を介して OS のユーザー管理に依存しているため、プラットフォーム固有のユーザー設定（/etc/passwd や LDAP 等）に問題がある場合、ユーザー名の解決に失敗して NULL を返す原因になります。
