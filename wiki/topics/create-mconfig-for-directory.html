create_mconfig_for_directory

概念

create_mconfig_for_directory は、Apache HTTP Server の各モジュールにおいて、ディレクトリ単位の設定構造体を生成および初期化するためのコールバック関数です。Apache のモジュール設定フレームワークに適合するシグネチャを持ち、モジュールごとに固有の設定領域を動的に確保する役割を持ちます。

なぜ重要か

Apache では、ディレクトリごとに異なる設定（<Directory> セクションや .htaccess など）を適用する仕組みがあります。各モジュールがディレクトリ固有の設定を保持するためには、Apache コアからの要求に応じて専用の設定構造体を割り当てる必要があります。create_mconfig_for_directory は、この割り当て処理をカプセル化し、Apache のモジュール構造体（module）にフックとして登録するために重要です。

関与するモジュール

  • **module:modules/generators**: mod_suexec.c を含み、CGI 実行時の権限を設定する suexec 機能を担当します。
  • **module:modules/mappers**: mod_speling.c を含み、URL の綴りミスを自動修正するスペリング機能を担当します。

主要エンティティ

  • **symbol:modules/generators/mod_suexec.c#create_mconfig_for_directory**: mod_suexec 内の実装。ディレクトリ単位の設定構造体 suexec_config_t を生成します。
  • **symbol:modules/mappers/mod_speling.c#create_mconfig_for_directory**: mod_speling 内の実装。ディレクトリ単位の設定構造体 spconfig を生成します。
  • **apr_pool_t**: メモリ割り当てに使用される Apache ポータブルランタイム（APR）のメモリプール。
  • **mkconfig**: 各モジュール内で実際に構造体のメモリ確保と初期化を行う内部関数。

エンドツーエンドの流れ

1. **設定フェーズの開始**: Apache の起動時または設定再読み込み時、ディレクトリごとの設定が必要なタイミングで Apache コアが各モジュールの設定フックを呼び出します。 2. **関数の呼び出し**: Apache コアは、有効なメモリプール（apr_pool_t *p）と対象のディレクトリパス（char *dir）を引数として create_mconfig_for_directory を呼び出します。 3. **メモリ確保と初期化**:

  • mod_suexec では、mkconfig(p) を呼び出して suexec_config_t を確保します。
  • mod_speling では、mkconfig(p) を呼び出して spconfig を確保します。

    • 4. **ポインタの返却**: 初期化された構造体へのポインタ（void *）を Apache コアに返却し、コア側でディレクトリ設定として管理されます。

*注意: Apache コアがこの関数を呼び出す詳細なライフサイクルや、他モジュールとの詳細な相互作用の全貌は、提供されたコンテキストからは完全には明らかになっていません。*

重要な不変条件

  • **シグネチャの維持**: Apache の API 規約に従うため、関数は static void *create_mconfig_for_directory(apr_pool_t *p, char *dir) の形状を維持しなければなりません。
  • **dir 引数の扱い**: 渡される dir 引数は、現在の mod_suexec および mod_speling の実装内では使用されませんが、API 互換性のためにシグネチャ内に残す必要があります。
  • **メモリプールの利用**: 構造体の割り当てには、必ず引数として渡されたメモリプール p を使用しなければなりません。

よくある落とし穴

  • **未使用引数の誤削除**: dir 引数は関数内で使用されていないため、未使用変数警告の排除などの目的で誤って削除すると、Apache のモジュール API とのシグネチャ不一致によりコンパイルエラーが発生します。
  • **モジュール間の混同**: mod_suexec と mod_speling に同名の static 関数が存在しますが、これらは各ファイル内に閉じられたスコープを持ちます。ツールによる解析で相互参照が検知されることがありますが、これらは独立した別個の実装であり、直接相互に呼び出すことはできません。
  • **メモリ不足時の挙動**: メモリプール p が枯渇している場合、内部で呼び出される mkconfig の挙動（APR のメモリ割り当てエラーハンドリング）に依存してプロセスが終了する可能性があります。

次に読む場所

  • **modules/generators/mod_suexec.c**: suexec_config_t の構造や、suexec_module 構造体への登録方法を確認できます。
  • **modules/mappers/mod_speling.c**: spconfig の構造や、speling_module 構造体への登録方法を確認できます。
