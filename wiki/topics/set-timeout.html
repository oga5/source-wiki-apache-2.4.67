set_timeout

概念

set_timeout は、Apache HTTP Server において設定ディレクティブ（例: IdentityCheckTimeout や Timeout）から指定されたタイムアウト値を解析し、内部データ構造に設定するための関数（設定ハンドラ）です。

なぜ重要か

設定ファイルで指定された秒単位のタイムアウト設定値を、APR（Apache Portable Runtime）が扱える内部時間形式に変換し、各モジュールの設定構造体に正しく反映させる役割を担うため重要です。これにより、IDENT プロトコルによる照会などのネットワーク通信におけるタイムアウト制御が正確に行われます。

関与するモジュール

  • module:modules/metadata
  • module:server

主要エンティティ

  • symbol:modules/metadata/mod_ident.c#set_timeout: IdentityCheckTimeout ディレクティブのハンドラ。ident_config_rec 構造体の timeout および timeout_unset メンバを更新します。
  • symbol:server/core.c#set_timeout: Timeout ディレクティブに関連する設定ハンドラ（詳細な内部実装は現時点で未検証ですが、mod_ident.c の保守メモ等から参照が示唆されています）。

エンドツーエンドの流れ

1. Apache の設定ファイル解析フェーズにおいて、IdentityCheckTimeout ディレクティブが検出されます。 2. AP_INIT_TAKE1 マクロによる登録に基づき、modules/metadata/mod_ident.c の set_timeout が呼び出されます。 3. 引数として渡された文字列（秒数）が atoi によって整数に変換されます。 4. apr_time_from_sec を用いて、秒数が APR 内部形式のタイムアウト値に変換されます。 5. 変換された値が ident_config_rec 構造体の timeout に格納され、明示的に設定されたことを示すために timeout_unset フラグが 0 に設定されます。

※ server/core.c 側の set_timeout における具体的なエンドツーエンドの流れや、両モジュール間での詳細な連携フローは現時点では完全には解明されていません。

重要な不変条件

  • modules/metadata/mod_ident.c の set_timeout において、入力される設定構造体ポインタ d_ は ident_config_rec 型としてキャスト可能でなければなりません。
  • タイムアウト値が明示的に設定された場合、timeout_unset フラグは必ず 0 に設定されます。
  • その他のモジュール間をまたぐ共通の不変条件については、現時点では未検証です。

よくある落とし穴

  • **不適切な入力値**: 設定値 arg が数値以外の文字列である場合、atoi の仕様により 0 が設定される可能性があります。
  • **負の値の指定**: 負の値を指定した場合の挙動は apr_time_from_sec の入力値依存となり、意図しないタイムアウト挙動を引き起こす可能性があります。
  • **名前の重複と参照**: mod_ident.c の set_timeout は、server/core.c からも参照されている、あるいは同名の関数が存在するため、設定の共有や意図しない副作用に注意する必要があります。
