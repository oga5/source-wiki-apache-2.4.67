set_timeout

概念

set_timeout は、Apache HTTP Server の設定ファイル（httpd.conf など）からタイムアウト値に関するディレクティブを解析し、内部の設定構造体に格納するための設定ハンドラ関数です。主に mod_ident モジュールにおける IdentityCheckTimeout ディレクティブや、コアサーバーにおける Timeout ディレクティブの処理に関連しています。

なぜ重要か

設定ファイルで指定された秒単位の文字列（タイムアウト値）を、APR（Apache Portable Runtime）が扱える内部時間形式（apr_time_t）に変換し、各モジュールの動作（例：IDENT プロトコルによる問い合わせのタイムアウトなど）に反映させる役割を持つため重要です。

関与するモジュール

  • module:modules/metadata
  • module:server

主要エンティティ

  • **symbol:modules/metadata/mod_ident.c#set_timeout**

    • IdentityCheckTimeout ディレクティブのハンドラ。入力された秒数を apr_time_from_sec で変換し、ident_config_rec 構造体の timeout に設定します。

  • **symbol:server/core.c#set_timeout**

    • Timeout ディレクティブのハンドラとして登録されている関数。

エンドツーエンドの流れ

1. 設定ファイルの解析中に IdentityCheckTimeout または Timeout ディレクティブが検出されます。 2. AP_INIT_TAKE1 マクロによる登録に基づき、対応する set_timeout 関数が呼び出されます。 3. 引数として渡された文字列（arg）を atoi で整数に変換します。 4. apr_time_from_sec を使用して秒数を APR 内部形式に変換し、対応する設定構造体（mod_ident では ident_config_rec の timeout メンバ）に格納します。 5. 設定が明示的に行われたことを示すため、timeout_unset フラグを 0 に設定します。

※ server/core.c 側の set_timeout における具体的な内部処理や、設定されたタイムアウト値が実際にソケット通信等でどのように適用されるかという詳細なエンドツーエンドの流れは、提供された情報からは未解明です。

重要な不変条件

  • mod_ident.c において、渡される設定構造体ポインタ（d_）は ident_config_rec 型としてキャスト可能でなければなりません。
  • arg は atoi で変換可能な数値文字列である必要があります。
  • タイムアウト値が正常に設定された場合、timeout_unset フラグは必ず 0 に更新されます。

よくある落とし穴

  • **不正な入力値**: arg が数値以外の文字列である場合、atoi の仕様により 0 が設定される可能性があります。また、負の値を指定した場合の動作は apr_time_from_sec の入力値に依存します。
  • **名前の重複と副作用**: 本関数は mod_ident 固有の IdentityCheckTimeout だけでなく、server/core.c からも参照（または同名定義）されています。設定の共有や意図しない副作用がないか注意する必要があります。

次に読む場所

  • server/core.c：Timeout ディレクティブのハンドラ登録と実装。
