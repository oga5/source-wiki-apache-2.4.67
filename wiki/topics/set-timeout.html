set_timeout

概念

set_timeout は、Apache HTTP Server において設定ファイルからタイムアウト値を読み込み、内部の設定構造体に反映するためのディレクティブハンドラ関数です。同名の関数または参照が、メタデータモジュール（mod_ident）およびサーバーコアに存在します。

なぜ重要か

Webサーバーにおいて、外部プロトコル（IDENTなど）の照会や接続維持におけるタイムアウト値を適切に制御することは、リソースの枯渇を防ぎ、サーバーの安定性を維持するために重要です。set_timeout は、設定ファイル内のディレクティブ（IdentityCheckTimeout や Timeout）から指定された秒数を受け取り、システムが利用可能な形式に変換して保持する役割を担います。

関与するモジュール

  • module:modules/metadata （mod_ident.c を含む）
  • module:server （core.c を含む）

主要エンティティ

  • **symbol:modules/metadata/mod_ident.c#set_timeout**: IdentityCheckTimeout ディレクティブのハンドラ関数。IDENTプロトコル照会時のタイムアウト値を設定します。
  • **symbol:server/core.c#set_timeout**: Timeout ディレクティブのハンドラとして登録されている、または参照されているエンティティ。

エンドツーエンドの流れ

1. **ディレクティブの検出**: Apache の設定解析フェーズにおいて、IdentityCheckTimeout または Timeout ディレクティブが検出されます。 2. **ハンドラの呼び出し**: AP_INIT_TAKE1 マクロの登録に基づき、set_timeout が呼び出されます。 3. **値の変換と格納 (mod_ident.c の場合)**:

  • 設定ファイルから渡された文字列引数（秒数）を atoi で整数に変換します。
  • apr_time_from_sec を用いて APR 内部形式のタイムアウト値に変換し、ident_config_rec 構造体の timeout メンバに格納します。
  • timeout_unset フラグを 0 に設定し、値が明示的に設定されたことを示します。

※ server/core.c 側における具体的な内部処理フローや、両モジュール間での詳細な連携フローは現時点では完全には判明していません。

重要な不変条件

  • mod_ident.c の set_timeout において、渡される設定構造体ポインタ d_ は ident_config_rec 型としてキャスト可能でなければなりません。
  • 設定が正常に適用された場合、timeout_unset フラグは必ず 0 に更新されます。
  • モジュール間をまたぐ共通の不変条件や、server/core.c 側における不変条件は現時点では不明です。

よくある落とし穴

  • **不正な入力値**: 設定値として数値以外の文字列が渡された場合、atoi の挙動により 0 秒として設定される可能性があります。また、負の値を指定した場合の動作は apr_time_from_sec の入力値に依存します。
  • **名前の衝突と意図しない副作用**: mod_ident.c 内の set_timeout は、server/core.c からも参照されているとされています。設定の共有や意図しない副作用がないか、コード変更時には注意が必要です。
