Authentication, Authorization, and Access Control (AAA)

概念

Apache HTTP Server (httpd) における AAA は、認証（Authentication: authn）、認可（Authorization: authz）、およびアクセス制御を処理するためのモジュール式フレームワークです。

  • **認証 (authn)**: ユーザーの身元を検証します（例: mod_auth_basic.c によるBasic認証）。
  • **認可 (authz)**: 認証されたユーザーやリクエスト属性（IPアドレスなど）に対して、特定のリソースへのアクセス権限を付与するかを決定します（例: mod_authz_core.c）。
  • **コア管理**: mod_authn_core.c と mod_authz_core.c が、それぞれのプロバイダーの登録やディレクティブの解決を担います。

なぜ重要か

AAA フレームワークは、Webサーバーのセキュリティの根幹をなします。認証と認可を分離し、それぞれを独立したプロバイダーとしてモジュール化することで、管理者はBasic認証、LDAP、データベース連携などの多様な認証ソースと、複雑な認可ルール（Require グループなど）を柔軟に組み合わせて安全なアクセス制御を構成できます。

関与するモジュール

本トピックに関与するモジュールは以下の通りです。

  • module:include: 共通の認証・認可定義を提供するヘッダーファイル群。
  • module:modules/aaa: AAA フレームワークのコアおよび具体的な認証・認可プロバイダーの実装群。

主要エンティティ

本フレームワークを構成する主要なファイルと役割は以下の通りです。

  • **include/mod_auth.h**: 認証・認可モジュール間で共有される共通のデータ構造、定数、およびAPI関数を定義します。
  • **modules/aaa/mod_auth_basic.c**: HTTP Basic認証スキームを実装し、クライアントから送られたユーザー名とパスワードを抽出します。
  • **modules/aaa/mod_authn_core.c**: 認証プロバイダー（ユーザーデータベースなど）の登録と、認証プロセスの共通制御ロジックを提供します。
  • **modules/aaa/mod_authz_core.c**: Require ディレクティブや、<RequireAll>, <RequireAny> などの認可コンテナを処理し、アクセス可否の最終決定を行います。

エンドツーエンドの流れ

リクエスト処理時における詳細な関数呼び出しやフックの正確な遷移順序など、具体的なエンドツーエンドのフローは**現時点では完全には判明していません**。

一般的な想定フローは以下の通りです： 1. クライアントからのリクエストを mod_auth_basic などの認証ハンドラがインターセプトします。 2. 認証ハンドラは mod_authn_core を介して、設定された認証プロバイダーを呼び出し、資格情報を検証します。 3. 認証が成功すると、mod_authz_core が Require 規則に基づき、対象リソースへのアクセス権限（認可）を検証します。

重要な不変条件

モジュール間における詳細な状態遷移や、スレッドセーフティに関する不変条件は**現時点では完全には判明していません**。

よくある落とし穴

  • **認証と認可の混同**: 認証（ユーザーの特定）が成功しても、適切な認可ルール（Require）が設定されていない場合、アクセスが拒否される、あるいは意図せず全許可される可能性があります。
  • **コアモジュールへの依存**: mod_auth_basic などの個別モジュールは、mod_authn_core や mod_authz_core が提供する共通フレームワークに依存しているため、これらが適切にロード・配置されている必要があります。
