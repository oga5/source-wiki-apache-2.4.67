set_threads_per_child

概念

set_threads_per_child は、Apache HTTP Serverの設定ディレクティブ ThreadsPerChild の値を解析し、子プロセスあたりのスレッド数を表すグローバル変数 ap_threads_per_child に設定するためのコマンドハンドラ関数です。

なぜ重要か

この関数は、Apacheの並行処理能力を決定する重要なパラメータである ThreadsPerChild を設定ファイルから読み込み、システムに適用する役割を担っています。複数の主要なマルチプロセッシングモジュール（MPM）で共通のディレクティブ名と設定ロジックを提供するために、それぞれのMPM実装内に定義されています。

エンドツーエンドの流れ

1. **設定ファイルの解析**: Apacheの起動時または再起動時に、設定ファイル中の ThreadsPerChild ディレクティブが検出されます。 2. **ハンドラの呼び出し**: AP_INIT_TAKE1 マクロを介して登録された set_threads_per_child が呼び出されます。 3. **コンテキスト検証**: ap_check_cmd_context を呼び出し、設定がグローバルスコープで行われているか検証します。 4. **値の変換と格納**: 引数として渡された文字列（arg）を atoi で整数に変換し、グローバル変数 ap_threads_per_child に代入します。

※ なお、設定された ap_threads_per_child がその後どのように各MPMの子プロセス生成やスレッドプール初期化に反映されるかという、詳細なエンドツーエンドの実行時フローは現時点では不明です。

重要な不変条件

  • **グローバルスコープの強制**: ThreadsPerChild ディレクティブはグローバルコンテキスト（バーチャルホストやディレクトリコンテキスト以外）で設定されなければなりません。これは ap_check_cmd_context による検証によって保証されます。
  • **グローバル変数への反映**: 正常に処理された場合、設定値は必ずグローバル変数 ap_threads_per_child に格納されます。

よくある落とし穴

  • **非数値の指定**: atoi を使用して文字列から整数への変換を行っているため、設定ファイルに数値として解釈できない無効な文字列（例: ThreadsPerChild abc）が指定された場合、エラーにならずに 0 が設定されます。
  • **スコープ外での定義**: グローバルスコープ以外（例: <VirtualHost> 内など）で ThreadsPerChild を定義しようとすると、ap_check_cmd_context による検証でエラーが返されます。
