set_threads_per_child

概念

set_threads_per_child は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）において、設定ディレクティブ ThreadsPerChild の値を解析し、子プロセスあたりのスレッド数を表すグローバル変数 ap_threads_per_child に設定するための設定コマンドハンドラ関数です。

なぜ重要か

Apache HTTP Serverの並行処理性能やリソース消費量は、子プロセスが生成するスレッド数（ThreadsPerChild）に大きく依存します。set_threads_per_child は、設定ファイルからこの重要なパラメータを読み込んでシステムに適用するための共通のインターフェースを提供します。複数の主要なMPM（event, winnt, worker）で同様の処理が定義・利用されており、各MPMにおけるスレッド管理の起点となります。

関与するモジュール

エンドツーエンドの流れ

1. **設定の読み込み**: Apacheの起動または再起動時に、設定ファイル中の ThreadsPerChild ディレクティブが検出されます。 2. **マクロ経由の呼び出し**: 各MPM（event.c、mpm_winnt.c、worker.c）で定義された AP_INIT_TAKE1 マクロを介して、set_threads_per_child が呼び出されます。 3. **コンテキスト検証**: ap_check_cmd_context を呼び出し、設定がグローバルスコープで行われているか検証します。 4. **値の変換と設定**: 入力された文字列引数（arg）を atoi によって整数に変換し、グローバル変数 ap_threads_per_child に代入します。 5. **結果の返却**: 正常に設定された場合は NULL を返し、コンテキストエラーなどの不整合がある場合はエラーメッセージを返します。

※なお、設定された ap_threads_per_child がその後どのように各MPMの子プロセス生成やスレッドプール初期化のライフサイクル内で利用されるかという、詳細なエンドツーエンドの実行時フローは現時点では提供された情報からは不明です。

重要な不変条件

  • **グローバルスコープの強制**: ThreadsPerChild ディレクティブはグローバルコンテキスト（バーチャルホスト内などではなく、メインサーバー設定）でのみ設定されなければなりません。これは ap_check_cmd_context によって検証されます。
  • **グローバル変数への格納**: 解析された値は、システム全体で参照されるグローバル変数 ap_threads_per_child に格納されます。

よくある落とし穴

  • **非数値の指定**: 入力値の検証において atoi が使用されているため、設定ファイルに数値として解釈できない不正な文字列（例: ThreadsPerChild abc）が指定された場合、エラーにならずに 0 が設定されてしまいます。
  • **スコープ外での定義**: グローバルスコープ以外（<VirtualHost> 内など）で ThreadsPerChild を設定しようとすると、ap_check_cmd_context による検証でエラーとなり、設定の読み込みに失敗します。
