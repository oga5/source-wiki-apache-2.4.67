set_threads_per_child

概念

set_threads_per_child は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）において、設定ディレクティブ ThreadsPerChild の値を解析し、子プロセスあたりのスレッド数を表すグローバル変数 ap_threads_per_child に設定するための設定コマンドハンドラ関数です。

なぜ重要か

Apache HTTP Serverの並行処理モデル（特にスレッドを使用するMPM）において、各子プロセスが生成するスレッド数はパフォーマンスとリソース消費に直結します。set_threads_per_child は、設定ファイルからこの重要なパラメータを安全に読み込み、システム全体に適用する役割を担っています。

関与するモジュール

本機能は、以下のMPM（Multi-Processing Module）に関連しています。

  • module:server/mpm/event
  • module:server/mpm/winnt
  • module:server/mpm/worker

エンドツーエンドの流れ

1. **設定の読み込み**: Apacheの起動または再起動時に設定ファイルが解析され、ThreadsPerChild ディレクティブが検出されます。 2. **ハンドラの呼び出し**: 各MPM（event、winnt、worker）で AP_INIT_TAKE1 マクロを介して登録された set_threads_per_child が呼び出されます。 3. **コンテキスト検証**: ap_check_cmd_context を使用して、ディレクティブが適切なグローバルスコープで定義されているか検証します。 4. **値の変換と設定**: 引数として渡された文字列（arg）を atoi によって整数に変換し、グローバル変数 ap_threads_per_child に代入します。

※ なお、設定された ap_threads_per_child がその後どのように各MPMの子プロセス生成やスレッドプール初期化のライフサイクルで利用されるか、その詳細なエンドツーエンドの実行フローは現時点では提供された情報からは未確定です。

重要な不変条件

  • **グローバルスコープの強制**: ThreadsPerChild ディレクティブは、バーチャルホストやディレクトリなどの局所的なコンテキストではなく、グローバルな設定コンテキストで定義されていなければなりません（ap_check_cmd_context による検証）。

よくある落とし穴

  • **非数値の指定**: 設定値のパースに標準Cライブラリの atoi を使用しているため、ThreadsPerChild に数値として解釈できない無効な文字列が指定された場合、エラーにならずに 0 が設定されます。
  • **コンテキストエラー**: グローバルスコープ以外（例: <Directory> や <Location> ブロック内）で ThreadsPerChild を指定した場合、ap_check_cmd_context によってエラーが返され、サーバーの起動に失敗します。
