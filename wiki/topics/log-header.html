log_header

概念

log_header は、Apache HTTP Server（httpd）の異なるモジュールにおいて、ヘッダー情報のログ記録や処理を行うために定義されている同名の静的（static）関数群です。これらは modules/http2 モジュールと server コアモジュールにそれぞれ個別のシグネチャと役割を持って存在しています。

なぜ重要か

これらの関数は、それぞれのコンテキストにおいてヘッダー情報を適切に処理・出力するために重要です。

  • **server/log.c の log_header**: エラーログ出力の仕組みにおいて、ヘッダー情報をフォーマットしてバッファに格納する役割を担っていると推測されます。
  • **modules/http2/h2_proxy_session.c の log_header**: HTTP/2 プロキシセッションの文脈において、ヘッダーの走査やログ記録を行うためのコールバック等として機能していると推測されます。
