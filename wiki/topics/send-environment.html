send_environment

概念

send_environment は、Apache HTTP Server（httpd）において、HTTPリクエストに関連付けられた環境変数を FastCGI プロトコルの PARAMS レコードとしてシリアライズし、FastCGI バックエンドへ送信するための処理（関数）です。

なぜ重要か

FastCGI バックエンド（アプリケーションサーバーなど）がリクエストを正しく処理するためには、CGI環境変数やリクエスト固有のメタデータが必要です。send_environment は、これらの環境変数を FastCGI 規格に準拠した形式（PARAMS レコード）に変換して確実にバックエンドへ伝達する役割を担っています。

関与するモジュール

  • module:modules/proxy

主要エンティティ

  • symbol:modules/proxy/mod_proxy_fcgi.c#send_environment: 実際の環境変数シリアライズと送信処理を実装している関数。
  • symbol:modules/aaa/mod_authnz_fcgi.c#send_environment: mod_proxy_fcgi.c の同関数を呼び出す、または関連する処理を行うエンティティ。

エンドツーエンドの流れ

1. **環境変数の構築**: ap_add_common_vars および ap_add_cgi_vars を呼び出し、CGI 関連の環境変数を構築します。 2. **動的調整**: fix_cgivars を呼び出し、設定に基づいた環境変数の書き換えを行います。 3. **シリアライズ**: r->subprocess_env から環境変数を取得し、最大 16KB 単位の PARAMS レコードに分割してエンコードします。 4. **送信**: send_data を使用して、ヘッダとボディ（環境変数）をバックエンドへ送信します。 5. **終了通知**: すべての環境変数の送信後、データ長 0 の PARAMS レコードを送信して送信完了を通知します。

※なお、mod_authnz_fcgi.c から呼び出される際のエンドツーエンドの全体フローや、認証・認可フェーズにおける詳細なコンテキストとの統合フローについては、提供された情報からは完全には判明していません。

重要な不変条件

  • **環境変数の存在**: 呼び出し時に r->subprocess_env に環境変数が適切に格納されている必要があります。
  • **メモリ管理**: temp_pool は呼び出し元で管理され、本関数内で apr_pool_clear により適宜解放されます。
  • **単一セッション前提**: FastCGI の多重化（Multiplexing）はサポートされておらず、request_id は単一のセッションを前提として動作します。

よくある落とし穴

  • **パス処理の整合性**: 関数内で r->filename の一時的な書き換えと復元を行っているため、この周辺のコードを修正する際はパス処理の整合性が崩れないよう注意が必要です。
  • **エンコード失敗時の挙動**: 環境変数のエンコードに失敗した場合、警告ログを出力して該当の変数をスキップします。一部の環境変数がバックエンドに届かない場合は、このスキップ処理が発生していないか確認する必要があります。
  • **エラーの即時伝播**: send_data がエラーを返した場合、処理は中断され、即座にそのエラーが呼び出し元へ伝播されます。
