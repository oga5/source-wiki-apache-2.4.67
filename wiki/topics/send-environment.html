send_environment

概念

send_environment は、Apache HTTP Server（httpd）において、HTTP リクエストに関連付けられた環境変数を FastCGI プロトコルの PARAMS レコードとしてシリアライズし、FastCGI バックエンドへ送信するための処理および関数です。

なぜ重要か

FastCGI バックエンド（アプリケーションサーバーなど）がリクエストを正しく処理するためには、CGI 規格に準拠した環境変数（メタデータ）が必要です。send_environment は、リクエスト情報からこれらの環境変数を構築・エンコードし、プロトコル仕様に従って確実にバックエンドへ伝達する役割を担うため、FastCGI 連携において極めて重要です。

  • symbol:modules/aaa/mod_authnz_fcgi.c#send_environment: mod_authnz_fcgi.c から呼び出される、または同等の役割を持つエンティティ。

エンドツーエンドの流れ

1. **環境変数の準備**: ap_add_common_vars および ap_add_cgi_vars を呼び出し、CGI 関連の環境変数を構築します。 2. **動的調整**: fix_cgivars を呼び出し、設定に基づいた環境変数の書き換えを行います。 3. **シリアライズ**: r->subprocess_env から環境変数を取得し、最大 16KB 単位の PARAMS レコードに分割してエンコードします。 4. **送信**: send_data を使用して、ヘッダとボディ（環境変数）をバックエンドへ送信します。 5. **終了通知**: 全ての環境変数の送信後、データ長 0 の PARAMS レコードを送信して終了を通知します。

※なお、mod_authnz_fcgi.c 側における詳細なエンドツーエンドの呼び出しコンテキストや、認証・認可フェーズ全体における完全なフローの詳細は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **環境変数の存在**: r->subprocess_env に環境変数が格納されている必要があります。
  • **単一セッション前提**: FastCGI の多重化はサポートしておらず、request_id は単一のセッションを前提とします。
  • **メモリ管理**: temp_pool は呼び出し元で管理され、本関数内で apr_pool_clear により適宜解放されます。

よくある落とし穴

  • **パス処理の整合性**: 関数内で r->filename の一時的な書き換えと復元を行っているため、修正時はパス処理の整合性に注意する必要があります。
  • **エラーの伝播**: 環境変数のエンコードに失敗した場合は警告ログを出力して該当変数をスキップしますが、send_data がエラーを返した場合は即座にそのエラーが呼び出し元へ伝播されます。
  • **将来的な課題**: ソースコード内の XXX コメントにある通り、FastCGI 特有の環境変数追加の必要性については将来的な検討課題として残されています。
