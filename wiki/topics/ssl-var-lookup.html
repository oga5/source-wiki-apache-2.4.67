ssl_var_lookup

概念

ssl_var_lookup は、Apache HTTP Server において SSL/TLS 関連の環境変数、リクエスト情報、接続情報、サーバー設定、および時刻情報などの各種変数を動的に解決し、文字列として取得するための関数群および互換レイヤーです。

なぜ重要か

この機能は、SSL/TLS 接続のステータスやクライアント証明書情報、暗号化スイートなどの詳細なメタデータを、他のモジュール（mod_lua など）やリクエスト処理パイプラインから透過的に参照できるようにするために重要です。また、古い API を使用するレガシーなモジュールとの後方互換性を維持するためのブリッジとしても機能します。

関与するモジュール

提供された情報に基づき、以下のモジュールが関与しています。

  • module:modules/ssl (ssl_engine_vars.c): 実際の変数解決ロジックの本体を実装しています。
  • module:server (ssl.c): 後方互換性のためのラッパー（ap_ssl_var_lookup への委譲）を提供しています。
  • module:modules/arch/netware (mod_nw_ssl.c): NetWare 環境向けの SSL 処理において、この関数を呼び出すか、またはオプション関数として登録・参照しています。

主要エンティティ

  • **symbol:modules/ssl/ssl_engine_vars.c#ssl_var_lookup**:

    • SSL 関連変数や環境変数、時刻情報などを解決する実体関数。

  • **symbol:server/ssl.c#ssl_var_lookup**:

    • ap_ssl_var_lookup への委譲を行う、後方互換性のための静的（static）なラッパー関数。

  • **symbol:modules/arch/netware/mod_nw_ssl.c#ssl_var_lookup**:

    • NetWare 向けの実装において本機能と連携するシンボル（詳細な内部仕様は提供されたコンテキストからは未確定）。

エンドツーエンドの流れ

1. **呼び出しの発生**: 各モジュール（mod_lua や NetWare 向けモジュールなど）またはフック経由で ssl_var_lookup が呼び出されます。 2. **互換レイヤー（server/ssl.c 経由の場合）**: 古い API 経由で呼び出された場合、server/ssl.c の ssl_var_lookup が呼び出しを受け取り、推奨される API である ap_ssl_var_lookup へ処理を委譲します。 3. **変数の解決（modules/ssl/ssl_engine_vars.c の場合）**:

  • メモリ割り当て用のプール（apr_pool_t）が指定されていない場合は、リクエスト（r）、接続（c）、またはモジュール設定から自動的にプールを特定します。
  • リクエスト r が存在する場合、HTTP_ ヘッダ、REQUEST_ 属性、REMOTE_ 属性、SERVER_ 属性、および ENV: 変数を順次検索します。
  • 接続 c が存在する場合、SSL_ で始まる変数や HTTPS 変数を ssl_var_lookup_ssl へ委譲して解決します。
  • 上記で解決できない場合、SSL_VERSION_、SERVER_SOFTWARE、API_VERSION、および現在時刻（TIME_ 系）を解決します。

    • 4. **結果の返却**: 解決された値は apr_pool_t 上に確保され、呼び出し元に返されます。

*注意: NetWare 環境における具体的なエンドツーエンドの統合フローや、ap_ssl_var_lookup 自体の内部実装の詳細、およびフックの完全な登録プロセスは、提供されたコンテキストからは未確定です。*

重要な不変条件

  • **メモリ管理**:

    • modules/ssl/ssl_engine_vars.c の実装において、戻り値は常に apr_pool_t 上に確保されます。解決された値が存在しない（NULL の）場合、最終的に空文字列 "" が返されます。

  • **コンテキスト依存性**:

    • リクエスト r や接続 c が NULL の場合、それらに依存する変数（SSL_ プレフィックスを持つ変数など）は解決されません。

よくある落とし穴

  • **メモリ消費**:

    • ssl_var_lookup は内部で apr_pstrdup などのメモリ複製を多用するため、高頻度で呼び出すとメモリ消費量（プールメモリの累積）が増大する可能性があります。

  • **API の選択**:

    • server/ssl.c の ssl_var_lookup は後方互換性のためのブリッジです。新規開発においては、この互換レイヤーを介さず、直接 ap_ssl_var_lookup を使用することが推奨されます。

  • **変数の未解決**:

    • 必要なコンテキスト（s, c, r）が不足している場合、該当する変数が存在していても解決できず、空文字列または NULL が返される原因になります。

次に読む場所
