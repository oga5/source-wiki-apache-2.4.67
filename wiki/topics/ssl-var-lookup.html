ssl_var_lookup

概念

ssl_var_lookup は、Apache HTTP Server において、リクエスト、接続、またはサーバー情報に基づいて SSL 関連の変数、環境変数、時刻情報などの値を解決し、文字列として取得するための関数および仕組みです。

なぜ重要か

この仕組みは、SSL/TLS 接続の状態や証明書情報、サーバー設定などの動的なメタデータを、他のモジュール（mod_lua や NetWare 向け SSL モジュールなど）や内部処理から統一的に参照できるようにするために重要です。また、古い API を使用するモジュールとの後方互換性を維持するためのブリッジ（互換レイヤー）としても機能します。

関与するモジュール

提供された情報に基づき、以下のモジュールが関与しています。

  • module:modules/arch/netware (NetWare 環境向けの SSL 処理)
  • module:modules/ssl (コアとなる SSL 変数解決ロジックの実装)
  • module:server (後方互換性のためのラッパー/ブリッジの実装)

主要エンティティ

  • **symbol:modules/ssl/ssl_engine_vars.c#ssl_var_lookup**:

    • 実際の変数解決ロジックを実装する中心的な関数。リクエスト、接続、サーバーのコンテキストから、指定された変数名（SSL_、HTTP_、TIME_ など）に対応する値を解決します。

  • **symbol:server/ssl.c#ssl_var_lookup**:

    • 古い API を使用するモジュール向けの後方互換性レイヤー。内部的に ap_ssl_var_lookup を呼び出して処理を委譲します。

  • **symbol:modules/arch/netware/mod_nw_ssl.c#ssl_var_lookup**:

    • NetWare 環境において、この変数解決の仕組みを呼び出す、またはオプション関数として登録・参照するエンティティ。

エンドツーエンドの流れ

1. **呼び出しの発生**: mod_lua などの外部モジュール、NetWare 向け SSL モジュール、または内部の SSL 処理（ssl_engine_init.c や ssl_engine_io.c）から ssl_var_lookup が呼び出されます。 2. **互換レイヤー（必要な場合）**: 古い API 経由で呼び出された場合、server/ssl.c の ssl_var_lookup が仲介し、ap_ssl_var_lookup へ処理を委譲します。 3. **変数解決の実装 (modules/ssl/ssl_engine_vars.c)**:

  • メモリ割り当て用のプールが未指定の場合は、コンテキストから自動的に特定します。
  • リクエスト r が存在する場合、HTTP_ ヘッダ、REQUEST_ 属性、REMOTE_ 属性、SERVER_ 属性、ENV: 変数を検索します。
  • 接続 c が存在する場合、SSL_ で始まる変数や HTTPS 変数を ssl_var_lookup_ssl に委譲して解決します。
  • それ以外の場合、SSL_VERSION_、SERVER_SOFTWARE、API_VERSION、現在時刻（TIME_ 系）などを解決します。

    • 4. **結果の返却**: 解決された値は apr_pool_t 上に確保（必要に応じて apr_pstrdup で複製）され、呼び出し元に返されます。

*注意: 呼び出し元から各実装への完全なバインディングや、オプション関数としての登録・解決の厳密なエンドツーエンドのライフサイクルは、提供された情報からは完全には特定されていません。*

重要な不変条件

  • **メモリ管理**: 戻り値の文字列は、常に apr_pool_t 上に確保されます。
  • **フォールバック**: 変数名が一致しない、または解決できない場合、最終的に空文字列 ""（または NULL）が返されます。
  • **コンテキスト依存**: r（リクエスト）や c（接続）が NULL の場合、それらに依存する変数（SSL_ プレフィックスなど）は解決されません。

よくある落とし穴

  • **メモリ消費**: apr_pstrdup を多用してプール上にメモリを確保するため、高頻度で呼び出すとメモリ消費量が増大する可能性があります。
  • **古い API の使用**: server/ssl.c の ssl_var_lookup は互換性のためのブリッジであるため、新規開発では直接 ap_ssl_var_lookup を使用することが推奨されます。
  • **コンテキストの欠落**: 必要なコンテキスト（request_rec や conn_rec）を渡さない場合、特定の変数が解決できず空文字列が返される原因になります。
