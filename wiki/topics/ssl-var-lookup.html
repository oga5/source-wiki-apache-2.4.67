ssl_var_lookup

概念

ssl_var_lookup は、Apache HTTP Server において、リクエスト、接続（Connection）、またはサーバー情報に基づいて、SSL/TLS 関連の変数や環境変数、時刻情報などの値を解決し、文字列として取得するための関数群およびインターフェースです。

この関数は複数のモジュール（modules/ssl/ssl_engine_vars.c、server/ssl.c、modules/arch/netware/mod_nw_ssl.c）に同名のシンボルとして存在し、それぞれコアの実装、後方互換性維持のためのレイヤー、プラットフォーム固有の実装としての役割を担っています。

なぜ重要か

SSL/TLS 接続のメタデータ（暗号スイート、証明書情報、プロトコルバージョンなど）や、サーバー・リクエストの属性情報を、ログ出力、アクセス制御、Lua スクリプト、SSI（Server Side Includes）などの様々なコンテキストから統一的に参照できるようにするために重要です。また、古い API を使用する外部モジュールとの互換性を維持するためのブリッジとしても機能します。

関与するモジュール

提供されたコンテキストに基づき、以下のモジュールが関与しています。

  • module:modules/ssl（コアの SSL 変数解決ロジック）
  • module:server（後方互換レイヤー）
  • module:modules/arch/netware（NetWare プラットフォーム向けの実装）

主要エンティティ

  • **symbol:modules/ssl/ssl_engine_vars.c#ssl_var_lookup**

    • SSL 関連変数（SSL_ で始まる変数や HTTPS）、HTTP ヘッダ、リクエスト/接続/サーバー属性、時刻情報などを解決するコア実装です。

  • **symbol:server/ssl.c#ssl_var_lookup**

    • 古い API を使用するモジュール向けの後方互換性レイヤーです。内部的に推奨 API である ap_ssl_var_lookup を呼び出すラッパーとして機能します。

  • **symbol:modules/arch/netware/mod_nw_ssl.c#ssl_var_lookup**

    • NetWare 環境用の SSL 変数解決関数です。isSecure や ssl_var_lookup_ssl などを呼び出して解決を行います。

エンドツーエンドの流れ

詳細なエンドツーエンドの呼び出しフローの全容は完全には解明されていませんが、提供された情報から以下の流れが確認できます。

1. **呼び出しの発生**: expr_func_fn、ssl_hook_Access、ssl_hook_Fixup、Lua リクエスト処理（lua_request.c）などから ssl_var_lookup が呼び出されます。 2. **互換レイヤー経由（server/ssl.c の場合）**: 古い API 経由の呼び出しは server/ssl.c の ssl_var_lookup で受け取られ、ap_ssl_var_lookup へ処理が委譲されます。 3. **コア解決処理（ssl_engine_vars.c の場合）**:

  • プール p が未指定の場合は、利用可能なコンテキストから自動的にプールを特定します。
  • リクエスト r が存在する場合、HTTP_ ヘッダ、REQUEST_、REMOTE_、SERVER_ 属性、ENV: 変数を順次検索します。
  • 接続 c が存在する場合、SSL_ 変数を ssl_var_lookup_ssl に委譲して解決します。
  • それ以外の場合、SSL_VERSION_、SERVER_SOFTWARE、時刻情報（TIME_ 系）などを解決します。

    • 4. **値の返却**: 解決された文字列は apr_pool_t 上に確保（必要に応じて apr_pstrdup で複製）され、呼び出し元に返されます。

重要な不変条件

  • **メモリ管理**: 戻り値の文字列は常に apr_pool_t 上に確保されます。resdup フラグが真の場合は apr_pstrdup により複製されます。
  • **フォールバック**: 変数名が一致しない、または解決できない場合、ssl_engine_vars.c の実装は空文字列 "" を返します。
  • **コンテキスト依存**: r（リクエスト）や c（接続）が NULL の場合、それらに依存する変数（例: SSL_ プレフィックスを持つ変数）は解決されません。

よくある落とし穴

  • **メモリ消費**: ssl_var_lookup（特に ssl_engine_vars.c）は内部で apr_pstrdup や apr_psprintf を多用するため、頻繁に呼び出すとメモリプール内のメモリ消費量が累積・増加する可能性があります。
  • **API の混同**: 後方互換用の server/ssl.c#ssl_var_lookup は古い API とのブリッジです。新規開発においては、直接 ap_ssl_var_lookup を使用することが推奨されます。
  • **コンテキストの欠落**: 必要なコンテキスト（s, c, r）が NULL で渡された場合、特定の変数（特に SSL 接続状態に依存するもの）が解決できず、意図せず空文字列や NULL が返される原因になります。
