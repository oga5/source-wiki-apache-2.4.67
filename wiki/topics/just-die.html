just_die

概念

just_die は、Apache HTTP Server (httpd) のマルチプロセッシングモジュール (MPM) において、シグナル（SIGHUP や SIGTERM など）を受信した際に子プロセスを安全かつ正常に終了させるためのシグナルハンドラ関数です。

なぜ重要か

この関数は、シグナル受信時にプロセスを即座に、かつクリーンに終了させる役割を担います。just_die という名称でありながら、内部的には clean_child_exit(0) を呼び出すことで、リソースのクリーンアップ処理を経由した正常終了（ステータス0）を保証します。これにより、プロセスの異常終了を防ぎ、システムの安定性を維持します。

エンドツーエンドの流れ

1. **シグナルの受信**: 子プロセスが SIGHUP や SIGTERM などのシグナルを受信します。 2. **ハンドラの起動**: apr_signal を介して登録された just_die が起動します。 3. **クリーンアップと終了**: just_die は受信したシグナル番号を無視し、clean_child_exit(0) を呼び出して子プロセスのクリーンアップ処理を実行し、プロセスを終了させます。

※なお、event および worker MPM における具体的な登録・呼び出しのライフサイクルや、詳細なエンドツーエンドのフローの全容は現時点では完全には判明していません。

重要な不変条件

  • just_die が呼び出された場合、制御は呼び出し元には戻らず、必ずプロセスが終了します。
  • 終了ステータスは常に 0（正常終了）として clean_child_exit に渡されます。

※その他の詳細な不変条件については、現時点では完全には判明していません。

よくある落とし穴

  • **名称による誤解**: just_die という名前から強制終了や異常終了を連想しがちですが、実際には clean_child_exit(0) を呼び出すため、正常終了として処理されます。
  • **MPM間の共通性**: prefork だけでなく event や worker でも同様のハンドラとして参照されているため、この関数を変更する際はすべての対象MPMへの影響を考慮する必要があります。
