just_die

概念

just_die は、Apache HTTP Server（httpd）のマルチプロセッシングモジュール（MPM）において、シグナル（SIGHUP や SIGTERM など）を受信した際に子プロセスを安全かつ正常に終了させるためのシグナルハンドラ関数です。

なぜ重要か

シグナル受信時にプロセスを即座に強制終了するのではなく、clean_child_exit(0) を呼び出すことで、必要なクリーンアップ処理を適切に実行し、ステータス 0（正常終了）として子プロセスを安全に終了させるために重要です。

エンドツーエンドの流れ

1. 子プロセスが SIGHUP や SIGTERM などのシグナルを受信します。 2. apr_signal を介して登録されたシグナルハンドラ just_die(sig) が起動します。 3. just_die は受信したシグナル番号を無視し、clean_child_exit(0) を呼び出します。 4. clean_child_exit により、子プロセスのクリーンアップ処理が実行され、プロセスが終了します。

※ event および worker MPM における具体的なシグナル登録状況や、詳細なエンドツーエンドのフローの差異については、提供された情報からはまだ完全には判明していません。

重要な不変条件

  • **制御の非返却**: just_die が呼び出された後、プロセスは必ず終了するため、呼び出し元へ制御が戻ることはありません。
  • **正常終了の保証**: 関数名に反して、実行されるのは clean_child_exit(0) であり、常に正常終了（ステータス0）として扱われます。

※ event および worker MPM における固有の不変条件については、提供された情報からはまだ完全には判明していません。

よくある落とし穴

  • **名称による誤解**: just_die という即時強制終了を想起させる名称ですが、実際にはクリーンアップを伴う正常終了（ステータス0）処理が実行されます。
  • **複数MPM間での影響**: prefork、event、worker の各MPMで同様のシグナルハンドラとして参照・定義されているため、挙動を変更する際にはすべての対象MPMへの影響範囲に注意する必要があります。
