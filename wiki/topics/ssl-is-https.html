ssl_is_https

概念

ssl_is_https は、Apache HTTP Server において特定の接続が SSL/TLS を使用しているかどうかを判定するための互換性レイヤー関数です。古い API を使用するモジュールとの後方互換性を維持するために提供されており、内部的には推奨される新しい API である ap_ssl_conn_is_ssl を呼び出して判定を行います。

なぜ重要か

この関数は、古い API に依存している既存のモジュールやサードパーティ製モジュールが、新しいバージョンの Apache HTTP Server でも修正なしに動作し続けられるようにするための後方互換性を提供します。オプション関数（Optional Function）として登録されることで、モジュール間の結合度を下げつつ、SSL 判定機能を安全に提供します。

関与するモジュール

  • module:modules/arch/netware（NetWare プラットフォーム向け SSL モジュール）
  • module:server（コアサーバー機能）

主要エンティティ

  • symbol:server/ssl.c#ssl_is_https: コアサーバー側で定義されている ssl_is_https 関数。ap_ssl_conn_is_ssl へのリダイレクトを行います。
  • symbol:modules/arch/netware/mod_nw_ssl.c#ssl_is_https: NetWare 環境における ssl_is_https の実装または利用に関連するシンボル。

エンドツーエンドの流れ

1. 古い API を使用する呼び出し元が、オプション関数として登録された ssl_is_https を呼び出します。 2. server/ssl.c 内の ssl_is_https(conn_rec *c) が呼び出しを受け取ります。 3. 本関数は、受け取った接続構造体 c をそのまま推奨される新しい API である ap_ssl_conn_is_ssl(c) に渡します。 4. ap_ssl_conn_is_ssl からの戻り値（SSL/TLS を使用している場合は非ゼロ、それ以外は 0）をそのまま呼び出し元に返します。

※なお、NetWare プラットフォーム（mod_nw_ssl.c）における具体的な呼び出し関係や、その他のモジュールからの詳細なエンドツーエンドの呼び出しフローの全容は、提供されたコンテキストからは完全には特定されていません。

重要な不変条件

  • 引数として渡される conn_rec *c は、有効な接続構造体ポインタである必要があります。
  • ssl_is_https の動作および不変条件は、内部で呼び出す ap_ssl_conn_is_ssl の実装とその前提条件に直接依存します。

よくある落とし穴

  • **非推奨 API の使用**: ssl_is_https は後方互換性のために残されている古い API です。新規にモジュールを開発または改修する場合は、本関数ではなく ap_ssl_conn_is_ssl を直接使用する必要があります。
  • **NULL ポインタのハンドリング**: 引数 c が NULL の場合の挙動は、内部で呼び出される ap_ssl_conn_is_ssl の実装に依存するため、呼び出し側で適切なポインタを渡すように注意する必要があります。

次に読む場所

  • modules/arch/netware/mod_nw_ssl.c: NetWare 環境における SSL 関連の実装ファイル。
