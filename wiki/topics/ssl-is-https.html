ssl_is_https

概念

ssl_is_https は、Apache HTTP Server において特定の接続（conn_rec）が SSL/TLS を使用しているかどうかを判定するための互換性レイヤー関数です。古い API を使用するモジュールとの後方互換性を維持するために提供されており、内部的には推奨される新しい API である ap_ssl_conn_is_ssl へ処理を委譲します。

なぜ重要か

この関数は、古いモジュールや NetWare などの特定プラットフォーム向けモジュール（mod_nw_ssl.c など）が、新しい接続判定 API（ap_ssl_conn_is_ssl）へ直接移行することなく、既存のコードベースのまま動作できるようにするための後方互換性（互換性レイヤー）を提供するために重要です。

関与するモジュール

  • module:modules/arch/netware
  • module:server

主要エンティティ

  • symbol:server/ssl.c#ssl_is_https: conn_rec を受け取り、ap_ssl_conn_is_ssl を呼び出して接続が SSL/TLS かどうかを判定する互換性関数。
  • symbol:modules/arch/netware/mod_nw_ssl.c#ssl_is_https: NetWare 環境において本機能に関連する、あるいは依存するシンボル（詳細な実装は提供されたコンテキストからは未詳）。

エンドツーエンドの流れ

1. 古い API またはオプション関数経由で ssl_is_https(conn_rec *c) が呼び出されます。 2. ssl_is_https は、受け取った接続構造体 c をそのまま推奨 API である ap_ssl_conn_is_ssl(c) に転送します。 3. ap_ssl_conn_is_ssl が接続状態を判定し、SSL/TLS を使用している場合は非ゼロ、そうでない場合は 0 を返します。 4. ssl_is_https はその戻り値をそのまま呼び出し元に返却します。

※なお、mod_nw_ssl.c 側における具体的な呼び出しや、オプション関数としての登録・解決の完全なエンドツーエンドのライフサイクルは、提供されたコンテキストからは完全には明らかになっていません。

重要な不変条件

  • 引数として渡される conn_rec *c は、有効な接続レコードのポインタである必要があります。
  • 本関数の挙動および不変条件は、委譲先である ap_ssl_conn_is_ssl の実装と不変条件に直接依存します。

よくある落とし穴

  • **新規開発での誤用**: 本関数は後方互換性のために残されている古い API です。新規にモジュールを開発または改修する場合は、本関数ではなく ap_ssl_conn_is_ssl を直接使用する必要があります。
  • **NULL ポインタの扱い**: 引数 c が NULL の場合の挙動は、内部で呼び出される ap_ssl_conn_is_ssl の実装に依存するため、呼び出し側で事前に有効性を担保することが推奨されます。
  • **シグネチャ変更の影響**: APR_DECLARE_OPTIONAL_FN を介してオプション関数として公開されているため、この関数のシグネチャ変更や削除を行うと、mod_nw_ssl.c などの外部モジュールに影響を及ぼす可能性があります。
