ssl_is_https

概念

ssl_is_https は、Apache HTTP Server において特定の接続（conn_rec）が SSL/TLS（HTTPS）を使用しているかどうかを判定するための関数です。プラットフォーム固有の実装や、古い API との互換性を維持するためのレイヤーとして複数のモジュールに存在します。

なぜ重要か

  • **後方互換性の維持**: server/ssl.c において、古い API を使用するモジュールが接続のセキュア状態を判定できるようにし、推奨される新しい API (ap_ssl_conn_is_ssl) への仲介役として機能します。
  • **プラットフォーム固有の対応**: NetWare 環境（mod_nw_ssl.c）など、特定のプラットフォームにおいて適切にセキュア接続の判定を行うために個別の実装が提供されています。

関与するモジュール

  • module:modules/arch/netware
  • module:server

主要エンティティ

  • symbol:server/ssl.c#ssl_is_https: 互換性維持のための静的関数。ap_ssl_conn_is_ssl を呼び出し、オプション関数として登録されます。
  • symbol:modules/arch/netware/mod_nw_ssl.c#ssl_is_https: NetWare 環境向けの静的関数。ap_get_module_config や isSecureConn を呼び出して判定を行います。

エンドツーエンドの流れ

1. **互換レイヤー（server/ssl.c）経由のフロー**:

  • 呼び出し元が ssl_is_https(c) を呼び出します。
  • 本関数が ap_ssl_conn_is_ssl(c) を呼び出し、その結果をそのまま返します。

    • 2. **NetWare環境（mod_nw_ssl.c）のフロー**:

  • ssl_is_https(c) が呼び出されると、内部で ap_get_module_config や isSecureConn を用いて接続状態を判定します。

※ どのモジュールがどのタイミングでこれらの関数を呼び出すかなど、システム全体における詳細なエンドツーエンドのフローは現時点では完全には判明していません。

重要な不変条件

  • 引数 c は有効な conn_rec ポインタである必要があります。
  • server/ssl.c の実装は、委譲先である ap_ssl_conn_is_ssl が提供する不変条件に依存します。
  • ※ その他の詳細な不変条件は現時点では解析されていません。

よくある落とし穴

  • **非推奨 API の使用**: server/ssl.c の ssl_is_https は後方互換性のために提供されています。新規開発においては、本関数ではなく ap_ssl_conn_is_ssl を直接使用する必要があります。
  • **シグネチャ変更の影響**: 本関数はオプション関数（APR_REGISTER_OPTIONAL_FN）として登録され外部から参照される可能性があるため、削除やシグネチャの変更を行うと、依存する外部モジュールに影響を与えるリスクがあります。
  • **NULL ポインタ**: 引数 c が NULL の場合の挙動は、内部で呼び出される ap_ssl_conn_is_ssl やプラットフォーム固有の実装に依存します。
