process_proxy_header

概念

process_proxy_header は、Apache HTTP Server (httpd) のプロキシ処理において、バックエンドから受信した、あるいは転送対象となるHTTPヘッダーのキーと値のペアを個別に処理・変換するための内部関数です。modules/http2 と modules/proxy の2つの異なるモジュールに同名の static 関数として定義されており、それぞれのプロトコルやコンテキストに応じたヘッダーの書き換えや正規化を行います。

なぜ重要か

プロキシを介した通信では、バックエンドサーバーが返したヘッダー（例: Location や日付ヘッダーなど）をそのままクライアントに転送すると、内部ネットワークの構造が露出したり、クライアント側で正しく解釈できなかったりする可能性があります。process_proxy_header は、リバースマッピング（ヘッダーの書き換え）や日付の正規化などを適用し、プロキシとしての整合性とセキュリティを維持するために重要な役割を果たします。

関与するモジュール

主要エンティティ

  • **symbol:modules/http2/h2_proxy_session.c#process_proxy_header**
  • HTTP/2 プロキシセッションにおいて、ヘッダーテーブル（apr_table_t）やプロキシストリーム（h2_proxy_stream）に対して、ヘッダー名 n と値 v を処理します。
  • **symbol:modules/proxy/mod_proxy_http.c#process_proxy_header**
  • 標準的な HTTP プロキシにおいて、リクエストレコード（request_rec）やディレクトリ設定（proxy_dir_conf）を用いて、ヘッダーの key と value を処理します。

エンドツーエンドの流れ

現時点では、詳細なエンドツーエンドの制御フロー全体は完全には解明されていません。ただし、呼び出し関係から以下の部分的な流れが確認されています。

1. バックエンドからのヘッダー読み込み時、または出力ヘッダーの追加時に、ap_proxy_read_headers や h2_proxy_stream_add_header_out から各モジュールの process_proxy_header が呼び出されます。 2. **HTTP/2 側 (h2_proxy_session.c)**:

  • ap_cstr_casecmp を用いてヘッダー名を比較判定します。
  • h2_proxy_link_reverse_map などのヘルパーを呼び出し、必要に応じてヘッダー値のリバースマッピング（書き換え）を行います。

    • 3. **HTTP/1.x 側 (mod_proxy_http.c)**:

  • ap_cstr_casecmp を用いてヘッダー名を比較判定します。
  • date_canon を呼び出し、日付ヘッダーのフォーマットを正規化します。

重要な不変条件

現時点では、これらの関数が維持すべき詳細な不変条件（メモリの所有権、スレッド安全性、特定のヘッダーに対する書き換え規則など）は完全には解明されていません。

よくある落とし穴

現時点では、具体的な実装コードの詳細な解析が行われていないため、この関数群に関する特有の落とし穴は完全には解明されていません。しかし、同名の関数が異なるモジュールで独立して定義されているため、一方のモジュールで行われたヘッダー処理の修正やバグ修正が、もう一方のモジュールに自動的に反映されない点に注意する必要があります。
