process_proxy_header

概念

process_proxy_header は、Apache HTTP Server (httpd-2.4.67) において、プロキシ接続を介して送受信されるヘッダー情報を処理・変換するための内部関数です。HTTP/1.1 プロキシ処理を行う mod_proxy_http と、HTTP/2 プロキシ処理を行う mod_http2 の双方に同名のシンボルが存在します。

なぜ重要か

プロキシを通過するリクエストおよびレスポンスのヘッダーを適切に解析・制御することは、プロキシとしての正しい動作、プロトコル準拠、およびセキュリティ（不要なヘッダーの削除や書き換えなど）を担保するために極めて重要です。
