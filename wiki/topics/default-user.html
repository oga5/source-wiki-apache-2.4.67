DEFAULT_USER

概念

DEFAULT_USER は、Apache HTTP Server（httpd）のUnix系オペレーティングシステム向けの実装において、サーバープロセスが特権（rootなど）を放棄した後に実行されるデフォルトのシステムユーザーを定義するためのシンボル（マクロ）です。

なぜ重要か

Unix環境におけるWebサーバーのセキュリティモデルでは、特権ポート（80や443など）をバインドするためにroot権限で起動した後、リクエストの処理を行う子プロセスやスレッドの権限を速やかに非特権ユーザーに切り替える必要があります。DEFAULT_USER は、明示的な設定がない場合にフォールバック先となる安全なデフォルトユーザーを定義するために重要です。

現時点では、DEFAULT_USER に関連する具体的な不変条件（例：コンパイル時に必ず定義されている必要があるか、特定のユーザーID形式を満たす必要があるかなど）は判明していません。

よくある落とし穴

ビルド環境やターゲットプラットフォーム（OS）によって DEFAULT_USER のデフォルト値（例: "nobody" や "#-1"）が異なる場合があり、適切な実行ユーザーがシステムに存在しない場合、サーバーの起動失敗や予期しない権限エラーを引き起こす可能性があります。ただし、具体的なエラーハンドリングの挙動は現時点では未検証です。
