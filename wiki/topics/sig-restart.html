sig_restart

概念

sig_restart は、Apache HTTP Server においてサーバーの再起動（通常の再起動およびグレースフル再起動）を要求するシグナルを受信した際に、状態遷移をトリガーするためのシグナルハンドラ関数です。OS/Unix 環境と OS/2 環境のそれぞれで実装が存在し、受信したシグナルに応じて再起動フラグや状態変数を更新します。

なぜ重要か

サーバープロセスのライフサイクル管理において、稼働中のサーバーに対して安全かつ迅速に再起動を指示するために不可欠です。特に、実行中のリクエストを中断せずに再起動を行う「グレースフル再起動」と、強制的に再起動を行う「通常の再起動」をシグナル経由で識別し、適切な状態フラグをセットする役割を担っています。

関与するモジュール

  • module:os/unix
  • module:server/mpm/mpmt_os2

主要エンティティ

  • symbol:os/unix/unixd.c#sig_restart: Unix 環境における再起動シグナルハンドラ。共有メモリ領域 retained_data を介して状態を管理します。
  • symbol:server/mpm/mpmt_os2/mpmt_os2.c#sig_restart: OS/2 MPM 環境における再起動シグナルハンドラ。グローバル変数 restart_pending および is_graceful を介して状態を管理します。

エンドツーエンドの流れ

1. **シグナルの受信**: 外部またはプロセス自身から再起動シグナル（Unix では SIGHUP や AP_SIG_GRACEFUL、OS/2 では SIGUSR1 など）が送信されます。 2. **ハンドラの起動**: 登録された sig_restart ハンドラが起動します。 3. **状態の更新**:

  • **Unix 環境 (unixd.c)**: retained_data が存在することを確認し、mpm_state を AP_MPMQ_STOPPING に設定します。重複処理を避ける判定を行った後、restart_pending を 1 に設定します。シグナルが AP_SIG_GRACEFUL でない場合は is_ungraceful を 1 に設定します。
  • **OS/2 環境 (mpmt_os2.c)**: 受信シグナルが SIGUSR1 の場合は is_graceful を 1 に設定し、restart_pending を 1 に設定します。

    • 4. **メインループへの伝播**:

  • シグナルハンドラ内ではフラグの更新のみを行い、実際の再起動処理はメインループ側でこれらのフラグを検知して実行されます。ただし、メインループがこれらのフラグをどのように評価し、最終的にプロセスを再起動するかの詳細なエンドツーエンドのフローは、提供された情報からは不明です。

重要な不変条件

  • **Unix 環境**:
  • retained_data（ap_unixd_mpm_retained_data 構造体）が初期化されている必要があります。初期化されていない場合、関数は処理を行わずに即座に終了します。
  • すでに再起動が保留中であり、かつ現在のシグナルが AP_SIG_GRACEFUL であるか、すでに非優雅（ungraceful）な再起動が要求されている場合は、重複処理を避けるため状態の上書きは行われません。
  • **OS/2 環境**:
  • restart_pending および is_graceful グローバル変数は、本関数が呼び出される前に適切に初期化されている必要があります。

よくある落とし穴

  • **非同期シグナル安全性**: sig_restart はシグナルハンドラとして動作するため、関数内では非同期シグナル安全（async-signal-safe）な操作のみを行う必要があります。複雑な処理やブロックする処理を記述すると、デッドロックや未定義動作を引き起こす可能性があります。
  • **環境間の差異**: Unix 環境と OS/2 環境で、使用するシグナル（AP_SIG_GRACEFUL と SIGUSR1）や、状態を保持するデータ構造（共有メモリ retained_data とグローバル変数）が異なるため、クロスプラットフォームのコードを保守する際には注意が必要です。また、OS/2 環境における os/unix/unixd.c からの参照など、環境をまたぐシグナル伝播の整合性については、提供された範囲では詳細が不明です。
