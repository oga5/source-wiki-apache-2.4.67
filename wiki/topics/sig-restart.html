sig_restart

概念

sig_restart は、Apache HTTP Server においてサーバーの再起動（通常の再起動およびグレースフル再起動）を要求するためのシグナルハンドラ関数です。プラットフォームや MPM（Multi-Processing Module）の実装（UNIX系OSおよびOS/2）に応じて、それぞれ異なるファイルで定義されています。

なぜ重要か

サーバープロセスが外部から再起動シグナル（SIGHUPやSIGUSR1など）を受け取った際に、安全かつ適切に再起動処理へ移行するためのトリガーとして機能します。このハンドラが状態フラグを適切に更新することで、メインループが再起動要求を検知し、サーバーの状態遷移（停止や再起動）を開始できます。

関与するモジュール

  • module:os/unix

主要エンティティ

  • symbol:os/unix/unixd.c#sig_restart

    • UNIX環境におけるシグナルハンドラ。共有メモリ領域である retained_data（ap_unixd_mpm_retained_data 構造体）の状態を更新します。

  • symbol:server/mpm/mpmt_os2/mpmt_os2.c#sig_restart

    • OS/2環境におけるシグナルハンドラ。グローバル変数 restart_pending および is_graceful を更新します。

エンドツーエンドの流れ

シグナル受信から再起動完了に至る完全なエンドツーエンドの流れは、提供された情報からは完全には明らかになっていません。現時点で判明している流れは以下の通りです。

1. **シグナルハンドラの登録**

  • **UNIX環境**: os/unix/unixd.c 内で apr_signal 等を用いて SIGHUP や AP_SIG_GRACEFUL に対するハンドラとして登録されます。
  • **OS/2環境**: server/mpm/mpmt_os2/mpmt_os2.c の set_signals 関数内でシグナルハンドラとして登録されます。

    • 2. **シグナルの受信とフラグ更新**

  • **UNIX環境**:
  • シグナルを受信すると、retained_data が存在することを確認し、retained_data->mpm_state を AP_MPMQ_STOPPING に設定します。
  • すでに再起動が保留中で、かつ重複処理の回避条件を満たしていれば、retained_data->restart_pending を 1 に設定します。
  • 受信シグナルが AP_SIG_GRACEFUL でない場合は、非優雅な再起動として retained_data->is_ungraceful を 1 に設定します。
  • **OS/2環境**:
  • シグナルを受信すると、受信したシグナルが SIGUSR1 であればグレースフル再起動を示す is_graceful を 1 に設定します。
  • 再起動の保留状態を示す restart_pending を 1 に設定します。

    • 3. **メインループでの検知**

  • これらのフラグ更新を契機として、メインループ側で実際の再起動ロジックが制御されます（ただし、メインループ側の具体的な処理フローは提供された範囲外のため不明です）。

重要な不変条件

  • **UNIX環境**:
  • retained_data が初期化されている必要があります。初期化されていない場合、関数は即座に終了します。
  • すでに再起動が保留されており、かつ現在のシグナルが AP_SIG_GRACEFUL であるか、すでに非優雅な再起動が要求されている場合は、重複処理を避けるため状態を上書きせずに終了します。
  • **OS/2環境**:
  • restart_pending および is_graceful が、本関数が呼び出される前に適切に初期化されている必要があります。
  • **共通**:
  • シグナルハンドラとして動作するため、非同期シグナル安全（async-signal-safe）な操作のみを行う必要があります。

よくある落とし穴

  • **非同期シグナル安全性**: 本関数はシグナルハンドラとして直接呼び出されるため、内部で非同期シグナル安全ではない処理（複雑な関数呼び出しやロック取得など）を行うと、デッドロックや未定義動作を引き起こす可能性があります。
  • **プラットフォーム間の状態管理の差異**: UNIX環境では retained_data 構造体を用いて状態を管理するのに対し、OS/2環境では個別のグローバル変数（restart_pending, is_graceful）を使用しており、データ構造が異なります。
  • **OS/2環境におけるUNIX参照**: os/unix/unixd.c から OS/2 の sig_restart への参照が確認されていますが、OS/2環境下でのシグナル伝播の整合性や、直接的な呼び出しフローの詳細は提供されたソース範囲外のため不明です。
