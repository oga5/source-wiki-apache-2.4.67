Shared Memory Slot Management (slotmem)

概念

Shared Memory Slot Management (slotmem) は、Apache HTTP Serverにおいて、複数の子プロセス間で共有される固定サイズの実体（スロット）の配列を管理するための抽象化APIおよびその実装です。この仕組みにより、各プロセスは一貫したメモリ領域に対して動的にデータの読み書きを行うことができます。

なぜ重要か

マルチプロセスモデル（MPM）を採用するApacheにおいて、ロードバランサ（mod_proxy_balancer など）のステータスや、クラスタのハートビート情報などの動的な状態情報を、プロセス境界を越えて安全かつ効率的に共有・同期するために不可欠です。プロバイダモデルを採用しているため、バッキングストアの実装（共有メモリか、プロセスローカルなプレーンメモリか）を透過的に切り替えることができます。

関与するモジュール

  • module:include (include/ap_slotmem.h)
  • module:modules/slotmem (modules/slotmem/mod_slotmem_plain.c, modules/slotmem/mod_slotmem_shm.c)

主要エンティティ

  • **ap_slotmem_provider_t** (include/ap_slotmem.h): スロットメモリの作成、アタッチ、データの取得・設定などの操作（create, attach, get, put など）を定義するプロバイダ構造体。
  • **ap_slotmem_instance_t** (include/ap_slotmem.h): 個々のスロットメモリセグメントを表す不透明な構造体。
  • **mod_slotmem_shm** (modules/slotmem/mod_slotmem_shm.c): 共有メモリ（SHM）をバッキングストアとして使用するプロバイダ実装。プロセス間でのデータ共有を可能にします。
  • **mod_slotmem_plain** (modules/slotmem/mod_slotmem_plain.c): 通常のプロセス内メモリ（プレーンメモリ）を使用するプロバイダ実装。プロセス間での共有は行われません。

エンドツーエンドの流れ

1. **プロバイダの取得**: 利用側モジュール（例: ロードバランサ）が、ap_lookup_provider を用いて "slotmem" のプロバイダ（shm または plain）を取得します。 2. **インスタンスの初期化**: create または attach APIを呼び出し、必要なスロット数と各スロットのサイズを指定して ap_slotmem_instance_t を生成または参照します。 3. **データの操作**: 実行時に get や put などのAPIを介して、特定のスロットインデックスに対してデータの読み書きを行います。

*注意: 利用側モジュール（ロードバランサ等）との具体的な連携シーケンスや、OSレベルでの共有メモリ確保の厳密なライフサイクルなど、詳細なエンドツーエンドのフローは提供されたソースコードの範囲内では完全には特定されていません。*

重要な不変条件

  • **固定スロットサイズ**: スロットメモリのインスタンス作成時に指定された各スロットのサイズおよびスロット数は、作成後に動的に変更することはできません。
  • **プロバイダの抽象化**: 利用側は ap_slotmem_provider_t を介して操作するため、背後の実装が shm であるか plain であるかに関わらず、同一のAPIコードで動作する必要があります。

*注意: 複数プロセスから同時に書き込みが発生した場合の排他制御（ロック）の保証や、クラッシュ時のデータ整合性に関する不変条件の詳細は、提供されたモジュール情報からは完全には特定されていません。*

よくある落とし穴

  • **mod_slotmem_plain の誤用**: mod_slotmem_plain はプロセス間でメモリを共有しないため、マルチプロセス環境（workerやevent MPMなど）でプロセス間共有を期待して使用すると、データが同期されず不整合が発生します。
  • **永続化ファイル（.shm）の残存**: mod_slotmem_shm を使用している場合、サーバーが異常終了した際にディスク上に古い .shm ファイルが残ることがあり、再起動時のアタッチ失敗や古いデータの意図しない読み込みの原因となる可能性があります。

次に読む場所

  • include/ap_slotmem.h: スロットメモリAPIおよびプロバイダ構造体の定義。
  • modules/slotmem/mod_slotmem_shm.c: 共有メモリベースの実装詳細。
  • modules/slotmem/mod_slotmem_plain.c: プレーンメモリベースの実装詳細。
