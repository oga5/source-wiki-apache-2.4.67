server_main_loop

概念

server_main_loop は、Apache HTTP Server のマルチプロセッシングモジュール（MPM）である event および worker において、親プロセスが子プロセスの生存状態を監視し、プロセスの生成や回収を制御するためのメインループ処理を実行する関数です。

なぜ重要か

このループは、Webサーバー全体のプロセス管理の中核を担います。負荷状況に応じた適切な子プロセスの維持、異常終了したプロセスの再起動、およびサーバーのシャットダウンや再起動シグナルのハンドリングを統制するために不可欠な役割を果たします。

主要エンティティ

  • symbol:server/mpm/event/event.c#server_main_loop: event MPMにおけるメインループの実装。非同期接続処理を特徴とするMPMでのプロセス管理を行います。
  • symbol:server/mpm/worker/worker.c#server_main_loop: worker MPMにおけるメインループの実装。マルチスレッド/マルチプロセス型のハイブリッドMPMでのプロセス管理を行います。

現時点では、提供されたソースコードの詳細な内部ロジック（シグナル受信から各子プロセスのフォーク、終了ステータスの監視、およびループの終了条件に至る具体的な制御フロー）は解析されておらず、詳細なエンドツーエンドの流れは未確定です。

現時点では、ループ実行中やプロセス監視において維持されるべき具体的な不変条件（例えば、稼働すべき最小/最大プロセス数の維持ルールや、シグナルハンドラとの排他制御など）は特定されていません。
