server_main_loop

概念

server_main_loop は、Apache HTTP Server (httpd-2.4.67) のマルチプロセッシングモジュール (MPM) である event および worker において、親プロセスが子プロセスのライフサイクルを監視・管理し、サーバーの実行状態を維持するためのメインループ処理を担当する内部関数です。

なぜ重要か

この関数は、サーバーの安定稼働と適切なリソース管理を維持するために極めて重要です。起動すべき残りの子プロセス数（remaining_children_to_start）を引数として受け取り、子プロセスの生成、終了したプロセスの検知と回収、スコアボードの更新、および負荷に応じたプロセス数の維持管理（メンテナンス）を一元的に制御します。

エンドツーエンドの流れ

現時点では詳細なソースコードの解析が完了していないため、完全なエンドツーエンドの実行フローは確定していません。しかし、呼び出し関係から以下の大まかな流れが推測されます。

1. **メインループの開始**: event_run または worker_run から server_main_loop が呼び出されます。 2. **子プロセスの監視と待機**: ap_wait_or_timeout を呼び出して、子プロセスの状態変化（終了など）を待機、またはタイムアウトが発生するまでブロックします。 3. **ステータス処理**:

  • 子プロセスが終了した場合、ap_find_child_by_pid や ap_get_scoreboard_process を用いて対象のプロセス情報を取得し、ap_process_child_status で終了ステータスを処理します。
  • event MPMでは event_note_child_stopped、worker MPMでは worker_note_child_killed などの個別関数を呼び出し、終了したプロセスの後処理を行います。

    • 4. **プロセスの維持管理**: perform_idle_server_maintenance を呼び出し、アイドル状態のプロセス数が設定値に適合するよう、必要に応じて make_child を介して新たな子プロセスを生成します。
