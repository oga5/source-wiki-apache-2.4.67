ap_preloaded_modules

概念

ap_preloaded_modules は、Apache HTTP Server (httpd-2.4.67) において、特定のプラットフォーム向けに事前ロード（プリロード）されるモジュールのリストを定義または管理するためのシンボルです。NetWare および Windows (Win32) といった特定のオペレーティングシステム環境のビルドにおいて、起動時にあらかじめ組み込んでおくべきモジュール群を指し示すために使用されます。

なぜ重要か

NetWare や Windows などのプラットフォームでは、標準的な Unix 系 OS とは異なるモジュールロード機構や初期化手順が必要となる場合があります。ap_preloaded_modules は、これらのプラットフォーム固有の初期化コード（modules.c）において、起動時にロードされるべきコアモジュール群を定義することで、プラットフォームごとの適切な起動とモジュール管理を実現するために重要です。

エンドツーエンドの流れ

現時点では、ap_preloaded_modules が具体的にどのように参照され、どのタイミングで各モジュールの初期化関数が呼び出されるかなど、詳細なエンドツーエンドの制御フローは完全には判明していません。一般的には、サーバーの起動初期段階において、この配列に登録されたモジュールが順次ロード・登録されるものと推測されます。

重要な不変条件

現時点では、ap_preloaded_modules の要素数、終端のシグナル（例: NULL 終端など）、またはスレッド安全性に関する重要な不変条件は明らかになっていません。

よくある落とし穴

プラットフォーム固有の modules.c に定義されているため、Unix系プラットフォーム（Linuxなど）向けのビルドや一般的なモジュール追加手順とは異なる管理が必要になる可能性があります。NetWare や Windows 向けにカスタムモジュールを静的に追加・削除する場合、これらのファイルを直接編集する必要があるか、あるいはビルドシステムの生成物に依存しているかについて注意が必要です。現時点では具体的な落とし穴の詳細は判明していません。
