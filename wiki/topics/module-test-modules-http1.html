Module: test/modules/http1

概念

test/modules/http1 は、Apache HTTP Server における HTTP/1.x プロトコルの挙動、仕様への適合性、および堅牢性を検証するための自動テストスイートです。Python の pytest フレームワークをベースにしており、テスト用の Apache インスタンスを起動し、実際にリクエストを送信して挙動を検証します。

なぜ重要か

HTTP/1.1 プロトコル（RFC 準拠）の正確な動作を担保し、リグレッションを防止するために不可欠です。特に、GET/POST などの基本メソッド、CGI を介したデータアップロード、トレイラーヘッダー（Trailers）の処理、およびセキュリティに関わる厳格なパース（Strict/Unsafe）の挙動を自動検証することで、Web サーバーとしての信頼性と安全性を維持します。

関与するモジュール

  • module:test/modules/http1 : テストスイートの本体およびテストケース群。
  • module:test/modules/http1/htdocs : テスト用の静的コンテンツおよび CGI スクリプト。
  • module:test/modules/http1/mod_h1test : HTTP/1.x の特殊な挙動を検証するために使用されるテスト専用の Apache モジュール。

主要エンティティ

  • **test/modules/http1/conftest.py**: pytest の設定ファイル。テスト実行に必要なフィクスチャや初期化処理を定義します。
  • **test/modules/http1/env.py**: テスト環境（Apache の設定、起動、ディレクトリ構成など）を管理・提供します。
  • **test/modules/http1/mod_h1test/mod_h1test.c**: HTTP/1.1 の特定のプロトコル挙動（トレイラーヘッダーのハンドリングなど）をシミュレート・検証するためのカスタム C モジュール。
  • **CGI スクリプト群 (test/modules/http1/htdocs/cgi/)**:
  • hello.py: 基本的な応答テスト用。
  • requestparser.py: 送信されたリクエストの解析テスト用。
  • upload.py: POST 等によるデータアップロードの検証用。
  • **テストケース群 (test_*.py)**:
  • test_001_alive.py: サーバーの生存確認。
  • test_003_get.py / test_004_post.py: GET/POST リクエストの検証。
  • test_005_trailers.py: トレイラーヘッダーの処理検証。
  • test_006_unsafe.py / test_007_strict.py: 許容されない、または厳格にパースされるべきリクエストの挙動検証。

エンドツーエンドの流れ

1. **テスト環境の起動**: pytest 実行時、conftest.py と env.py を通じてテスト用の Apache HTTP Server インスタンスが設定・起動されます。この際、mod_h1test モジュールがロードされます。 2. **リクエストの送信**: 各テストスクリプト（test_*.py）が、起動したサーバーに対して特定の HTTP/1.x リクエストを送信します。 3. **リクエストの処理**:

  • 静的ファイルや CGI スクリプト（upload.py など）がリクエストを処理します。
  • 特殊なプロトコルテストでは、mod_h1test が直接リクエストをハンドリングします。

    • 4. **検証**: テストスクリプトが、サーバーから返されたレスポンス（ステータスコード、ヘッダー、ボディ）をアサート（検証）します。

*※テスト実行時の詳細な内部状態遷移や、個々のテストケースにおける厳密なアサーション条件などのエンドツーエンドのフローの全容は、提供されたファイル構成からは完全には特定されていません。*

重要な不変条件

  • **サーバーの生存性**: テストスイートが実行される際、対象の Apache インスタンスが正常に応答可能な状態である必要があります（test_001_alive.py で担保）。
  • **厳格なパース規則**: test_007_strict.py などのテストにおいて、仕様に準拠しない不正な HTTP リクエストは、サーバーによって適切に拒否（または定義されたエラー応答を返却）されなければなりません。

*※テストスイート全体における詳細な不変条件や、各テストが前提とする具体的なサーバー設定の不変条件は、ソースコードの内部実装に依存するため、現時点では完全には特定されていません。*

よくある落とし穴

  • **環境依存の CGI 失敗**: テスト環境に Python インタープリタが正しくインストールされていない、または CGI スクリプト（hello.py など）の実行権限が不足している場合、CGI 関連のテストが予期せず失敗することがあります。
  • **カスタムモジュールの未ビルド**: mod_h1test.c が事前に正しくコンパイル・配置されていない場合、トレイラーヘッダーなどの特殊なテストケースが動作しません。

次に読む場所

  • test/modules/http1/conftest.py : テストの初期化フローと共通フィクスチャの理解。
  • test/modules/http1/env.py : テスト用 Apache インスタンスの設定方法の確認。
  • test/modules/http1/mod_h1test/mod_h1test.c : テスト用カスタムモジュールが提供する検証用機能の把握。
  • test/modules/http1/test_005_trailers.py : トレイラーヘッダーなど、HTTP/1.1 固有の挙動を検証する具体的なテスト実装の参照。
