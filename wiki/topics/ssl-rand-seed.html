ssl_rand_seed

概念

ssl_rand_seed は、Apache HTTP Server（httpd）およびその付属ツールである ab（ApacheBench）において、OpenSSL の擬似乱数生成器（PRNG）を初期化・強化するためにエントロピー（シード値）を供給する関数、およびその仕組みです。

この名称を持つ関数は、サーバーモジュール（mod_ssl）とクライアントツール（ab）のそれぞれに、異なるシグネチャと役割を持つ別個の実装として存在しています。

なぜ重要か

SSL/TLS 暗号通信の安全性は、予測不可能な高品質の乱数生成器に依存しています。

  • **サーバー側（mod_ssl）**: 設定された多様なエントロピーソース（ファイル、外部コマンド、EGD、ビルトイン）から適切にシードを供給することで、暗号強度の担保と安全なセッション確立を可能にします。
  • **ツール側（ab）**: クライアントとして SSL/TLS 接続テストを迅速に行うために、簡易的かつ確実に PRNG を初期化する必要があります。

関与するモジュール

  • module:modules/ssl（Apache の SSL/TLS サポートモジュール）
  • module:support（Apache 付属のサポートツール群）

主要エンティティ

  • **symbol:modules/ssl/ssl_engine_rand.c#ssl_rand_seed**

    • サーバー側で動作する関数。設定ファイルに基づき、複数のソースから高度なエントロピーを収集して OpenSSL に供給します。

  • **symbol:support/ab.c#ssl_rand_seed**

    • ab ツール内で動作する静的（static）関数。現在時刻、プロセスID、およびスタックデータを用いて簡易的にシードを供給します。

エンドツーエンドの流れ

mod_ssl 側と ab 側はそれぞれ独立して動作し、直接的なデータのやり取りはありません。それぞれの実行フローは以下の通りです。

1. サーバー側（mod_ssl）のフロー

1. サーバー起動時または接続時に、mod_ssl.c や ssl_engine_init.c から ssl_rand_seed が呼び出されます。 2. 設定構造体 SSLModConfigRec に登録されたシードソースを順次走査します。 3. ソースの種別に応じて以下の処理を行います：

  • **SSL_RSSRC_FILE**: 指定ファイルを読み込み、ssl_rand_feedfp を経由して供給。
  • **SSL_RSSRC_EXEC**: 外部コマンドを実行し、その標準出力を ssl_rand_feedfp を経由して供給。
  • **SSL_RSSRC_EGD**: RAND_egd() を呼び出し、Entropy Gathering Daemon から取得。
  • **SSL_RSSRC_BUILTIN**: 現在時刻、PID、および実行時のスタックメモリの一部を RAND_seed() に供給。

    • 4. 供給した総バイト数をログに出力し、RAND_status() でエントロピーの充足度を検証します。

2. ツール側（ab）のフロー

1. support/ab.c の処理（1453行目付近）から、引数なしの ssl_rand_seed() が呼び出されます。 2. time(NULL) から現在時刻を取得し、RAND_seed に渡します。 3. getpid() からプロセスIDを取得し、RAND_seed に渡します。 4. ssl_rand_choosenum を使用して決定したスタック上のランダムなオフセットから、一時バッファ（stackdata）の 128 バイトデータを抽出し、RAND_seed に渡して初期化を完了します。

重要な不変条件

  • **mod_ssl 側**:
  • SSLModConfigRec に乱数シードソースのリストが適切に設定されている必要があります。
  • 処理完了後、OpenSSL の RAND_status() が呼び出され、エントロピーの充足状態が検証される必要があります（不足時は警告ログが出力されますが、関数自体は正常終了します）。
  • **ab 側**:
  • ssl_rand_choosenum が返すスタックオフセットは、確保された一時バッファ stackdata（256バイト）の範囲を超えないよう、sizeof(stackdata) - 128 - 1 を上限として計算される必要があります。

よくある落とし穴

  • **同名関数の混同**:

    • modules/ssl/ssl_engine_rand.c の ssl_rand_seed（引数あり）と support/ab.c の ssl_rand_seed（引数なし、static）は、名前は同じですがシグネチャも実装も異なります。コードの追跡やデバッグの際には、どちらのコンテキストを扱っているか注意してください。

  • **パフォーマンスへの影響**:

    • サーバー側で外部コマンド実行（SSL_RSSRC_EXEC）をシードソースに指定している場合、コマンドの実行遅延がサーバーの起動時間に直接影響を与える可能性があります。

  • **エントロピーの質と現代の OpenSSL**:

    • ビルトイン（SSL_RSSRC_BUILTIN）や ab で使用される「スタックデータやPID、時刻」によるシード生成は補助的なものであり、これらのみに依存することは推奨されません。現代の OS および OpenSSL 環境では、OS のエントロピー源が自動的に利用されるため、これらの手動シード供給機能の重要性は以前よりも低下しています。

次に読む場所

  • **modules/ssl/ssl_engine_rand.c**: サーバー側における多様なエントロピー収集と供給の具体的な実装コード。
  • **support/ab.c**: ベンチマークツールにおける簡易的な PRNG 初期化処理の実装コード。
