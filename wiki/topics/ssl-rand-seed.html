ssl_rand_seed

概念

ssl_rand_seed は、Apache HTTP Server（httpd）およびその付属ベンチマークツール（ab）において、OpenSSL の擬似乱数生成器（PRNG）に対してエントロピー（シード値）を供給し、初期化および強化を行うための関数です。

この関数は、サーバーモジュール（mod_ssl）とクライアントツール（ab）のそれぞれに異なるシグネチャと実装で存在しています。

なぜ重要か

SSL/TLS 通信における暗号鍵の生成やハンドシェイクの安全性は、予測不可能な乱数生成器に依存しています。ssl_rand_seed は、起動時や接続時に十分なエントロピーを OpenSSL に供給することで、暗号強度の低下を防ぎ、セキュアな通信を担保するために不可欠な役割を果たします。

関与するモジュール

  • module:modules/ssl （サーバー側の SSL/TLS サポート）
  • module:support （ベンチマークツール ab）

主要エンティティ

  • **symbol:modules/ssl/ssl_engine_rand.c#ssl_rand_seed**

    • サーバー側で動作する関数。設定ファイルで指定された複数のソース（ファイル、外部コマンド、EGD、ビルトイン等）からエントロピーを収集し、OpenSSL に供給します。

  • **symbol:support/ab.c#ssl_rand_seed**

    • ab ツール内で動作する静的（static）関数。実行環境のシステム情報（時刻、PID、スタックメモリ）を利用して、簡易的にシード値を供給します。

エンドツーエンドの流れ

サーバー側とツール（ab）側はそれぞれ独立して動作し、直接的なデータのやり取りはありません。それぞれの実行フローは以下の通りです。

1. サーバー側（mod_ssl）のフロー

1. **呼び出し**: サーバーの初期化時（ssl_engine_init.c）または接続処理時（mod_ssl.c）に呼び出されます。 2. **ソースの走査**: 設定構造体 SSLModConfigRec に登録されたシードソースを順次走査します。 3. **エントロピーの収集と供給**:

  • **ファイル (SSL_RSSRC_FILE)**: 指定ファイルを読み込み、OpenSSL に供給します。
  • **外部コマンド (SSL_RSSRC_EXEC)**: 指定コマンドを実行し、その標準出力を供給します。
  • **EGD (SSL_RSSRC_EGD)**: Entropy Gathering Daemon からデータを取得します。
  • **ビルトイン (SSL_RSSRC_BUILTIN)**: 現在時刻、PID、および実行時スタックの一部を供給します。

    • 4. **検証**: 供給後、OpenSSL の RAND_status() を用いてエントロピーが十分であるかを確認します。

2. ツール側（ab）のフロー

1. **呼び出し**: ab の実行開始時（ab.c:1453）に呼び出されます。 2. **シードの生成**:

  • time(NULL) による現在時刻。
  • getpid() によるプロセスID。
  • ssl_rand_choosenum で決定したスタック上のランダムなオフセットから抽出した 128 バイトのデータ（stackdata）。

    • 3. **供給**: 上記のデータを RAND_seed() に渡して OpenSSL を初期化します。

重要な不変条件

  • **サーバー側**:
  • SSLModConfigRec に乱数シードソースのリストが適切に設定されている必要があります。
  • 処理完了後、RAND_status() が 0（エントロピー不足）を返した場合は警告ログが出力されますが、関数自体は処理を中断せず、供給した総バイト数を返して正常終了します。
  • **ツール側**:
  • スタックからデータを抽出する際、ssl_rand_choosenum が返すオフセットは、確保されたバッファ stackdata（256バイト）の境界（sizeof(stackdata) - 128 - 1）を超えないように制御される必要があります。

よくある落とし穴

  • **同名関数の混同**:

    • modules/ssl/ssl_engine_rand.c の ssl_rand_seed（引数あり）と、support/ab.c の ssl_rand_seed（引数なし、static）は、名前は同じですがシグネチャもスコープも異なる別個の実装です。コードリーディングの際は注意してください。

  • **起動遅延の発生**:

    • サーバー側で外部コマンド実行（SSL_RSSRC_EXEC）をシードソースに指定している場合、コマンドの実行完了を待つため、サーバーの起動時間が大幅に延びる可能性があります。

  • **エントロピーの質と現代の OpenSSL**:

    • ビルトイン（SSL_RSSRC_BUILTIN）や ab で用いられる「スタックデータやPID、時刻」に依存する手法は補助的なものであり、これらのみに依存することは推奨されません。現代の OpenSSL 環境では OS のエントロピー源が自動的に利用されるため、これらの手動シード供給の重要性は以前よりも低下しています。
