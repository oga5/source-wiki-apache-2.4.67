ssl_rand_seed

概念

ssl_rand_seed は、Apache HTTP Server（httpd）およびその付属ツールにおいて、OpenSSL の擬似乱数生成器（PRNG）を初期化・強化するためにエントロピー（シード値）を供給する関数群です。

この名前を持つ関数は、サーバーモジュール（mod_ssl）とベンチマークツール（ab）の2箇所にそれぞれ異なるシグネチャと役割で実装されています。

なぜ重要か

暗号通信の安全性は、暗号鍵やセッションキーの生成に使用される乱数の予測不可能性に依存しています。ssl_rand_seed は、OpenSSL の PRNG に対して十分なエントロピーを供給し、暗号強度の高い乱数生成を可能にするために重要です。特に、OS の自動エントロピー供給が不十分な環境や、起動初期段階での安全な乱数生成器のセットアップにおいて役割を果たします。

関与するモジュール

  • module:modules/ssl: Apache HTTP Server の SSL/TLS サポートモジュール（mod_ssl）。
  • module:support: Apache 付属のベンチマークツール ab（ApacheBench）。

主要エンティティ

  • **symbol:modules/ssl/ssl_engine_rand.c#ssl_rand_seed**
  • **シグネチャ**: int ssl_rand_seed(server_rec *s, apr_pool_t *p, ssl_rsctx_t nCtx, char *prefix)
  • **役割**: 設定ファイルで指定された複数のソース（ファイル、外部コマンド、EGD、BUILTIN）からエントロピーを収集し、OpenSSL の PRNG に供給します。
  • **symbol:support/ab.c#ssl_rand_seed**
  • **シグネチャ**: static void ssl_rand_seed(void)
  • **役割**: ab ツール専用の静的関数。現在時刻、PID、およびスタック上のメモリ領域から簡易的なシード値を生成し、OpenSSL の PRNG に供給します。

エンドツーエンドの流れ

1. **サーバー起動時または接続時 (mod_ssl)**:

  • modules/ssl/mod_ssl.c や modules/ssl/ssl_engine_init.c から ssl_rand_seed が呼び出されます。
  • 設定された乱数ソース（SSLModConfigRec 内の ssl_randseed_t リスト）を走査します。
  • ソースのタイプ（ファイル、外部コマンド、EGD、BUILTIN）に応じて、データを読み込むかコマンドを実行し、OpenSSL の RAND_seed() や RAND_egd() を用いてエントロピーを供給します。
  • 供給後、RAND_status() を用いてエントロピーが充足しているか確認します。

2. **ベンチマーク実行時 (ab)**:

  • support/ab.c の実行中（行1453付近）に、静的関数 ssl_rand_seed() が呼び出されます。
  • 現在時刻 (time(NULL))、プロセスID (getpid())、およびスタック上のバッファ (stackdata) から ssl_rand_choosenum を用いて決定したオフセットから128バイトのデータを抽出し、RAND_seed() に供給します。

*注意: ツールによる検出では support/ab.c:1453 から modules/ssl 側の ssl_rand_seed も呼び出されているとされていますが、引数の不一致があるため、両モジュールを跨ぐ完全なエンドツーエンドの相互作用フローの詳細は未解明です。*

重要な不変条件

  • **エントロピーの充足検証**: modules/ssl 側の実装では、シード供給処理の完了後に RAND_status() を呼び出し、エントロピーが充足しているか（戻り値が 0 以外か）を検証します。
  • **バッファ境界の維持**: support/ab.c 側の実装では、スタックからシードを抽出する際、ssl_rand_choosenum が返すオフセットが stackdata の範囲（sizeof(stackdata) - 128 - 1 が上限）を超えないように制御されます。

よくある落とし穴

  • **同名関数の混同**: modules/ssl のグローバル関数と support/ab.c の静的関数は、名前は同じですが引数や実装が全く異なります。コードリーディングやデバッグ時にこれらを混同しないよう注意が必要です。
  • **パフォーマンスへの影響**: mod_ssl において外部コマンド（SSL_RSSRC_EXEC）をシードソースとして設定している場合、サーバーの起動時間や接続処理の遅延につながる可能性があります。
  • **エントロピー不足の警告**: RAND_status() が 0 を返した場合、エントロピー不足の警告ログが出力されますが、関数自体はエラーを返さず正常終了するため、ログを監視していないとエントロピー不足に気づかない可能性があります。
  • **BUILTIN ソースの限界**: SSL_RSSRC_BUILTIN や ab で使われるスタックデータ、PID、時刻によるシード生成は補助的なものであり、これらのみに依存することは推奨されません。現代の OpenSSL 環境では OS のエントロピー源が自動利用されるため、本機能の重要性は以前より低下しています。

次に読む場所

  • modules/ssl/ssl_engine_rand.c: mod_ssl における詳細なエントロピー収集ロジックと、対応する各種ソース（ファイル、コマンド、EGD）の処理。
  • support/ab.c: ab ツールにおける静的なシード生成処理（ssl_rand_seed および ssl_rand_choosenum）。
