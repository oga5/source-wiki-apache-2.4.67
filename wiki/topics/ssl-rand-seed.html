ssl_rand_seed

概念

ssl_rand_seed は、Apache HTTP Server（httpd）およびその付属ツールにおいて、OpenSSLの擬似乱数生成器（PRNG）を初期化・強化するためにエントロピー（シード値）を供給する関数、およびその仕組みです。

この処理は、暗号論的に安全な乱数生成を保証するために不可欠であり、サーバーモジュール（mod_ssl）とベンチマークツール（ab）のそれぞれで、同名でありながら異なるシグネチャと実装を持つ関数として存在しています。

なぜ重要か

SSL/TLSプロトコルにおける鍵交換やセッション暗号化などのセキュリティ処理は、予測不可能な乱数に依存しています。OpenSSLのPRNGが適切に初期化（シード）されていない場合、暗号の安全性が著しく低下します。

ssl_rand_seed は、システム時刻、プロセスID（PID）、スタックメモリ、外部ファイル、外部コマンド、またはEGD（Entropy Gathering Daemon）などの多様なソースからエントロピーを収集してOpenSSLに供給することで、この安全性を担保します。

関与するモジュール

  • module:modules/ssl: Apache HTTP ServerのSSL/TLSサポートモジュール（mod_ssl）。
  • module:support: Apacheの補助ツール群。ベンチマークツール ab（ApacheBench）が含まれます。

主要エンティティ

  • symbol:modules/ssl/ssl_engine_rand.c#ssl_rand_seed: mod_ssl 内で定義されている関数。サーバー設定（SSLModConfigRec）に基づき、複数のエントロピーソースから動的にシードを供給します。
  • symbol:support/ab.c#ssl_rand_seed: ab ツール内で定義されている静的（static）関数。現在時刻、PID、およびスタック上のデータをシードとしてOpenSSLに供給します。

エンドツーエンドの流れ

mod_ssl と ab はそれぞれ独立して動作し、個別にOpenSSLの初期化を行います。両者の間で直接データが連携するエンドツーエンドのフローは存在しません。それぞれの実行フローは以下の通りです。

1. mod_ssl におけるフロー（サーバー起動時または接続時）

1. modules/ssl/mod_ssl.c または ssl_engine_init.c から ssl_rand_seed が呼び出されます。 2. 設定された乱数ソースリスト（SSLModConfigRec）を順次走査します。 3. ソースの種別に応じて以下の処理を行い、OpenSSLにエントロピーを供給します。

  • SSL_RSSRC_FILE: 指定ファイルを読み込み、ssl_rand_feedfp を経由して供給。
  • SSL_RSSRC_EXEC: 指定コマンドを実行し、その標準出力を ssl_rand_feedfp を経由して供給。
  • SSL_RSSRC_EGD: RAND_egd() を呼び出し、EGDから取得。
  • SSL_RSSRC_BUILTIN: 現在時刻、PID、および実行時のスタックメモリの一部を RAND_seed() に供給。

    • 4. 供給した総バイト数をログに出力し、RAND_status() を用いてエントロピーの充足度を検証します。

2. ab におけるフロー（ベンチマーク実行時）

1. support/ab.c の処理（位置 1453）から、引数なしの ssl_rand_seed が呼び出されます。 2. time(NULL) から取得した現在時刻を RAND_seed に渡します。 3. getpid() から取得したプロセスIDを RAND_seed に渡します。 4. ssl_rand_choosenum で決定されたスタック上のランダムなオフセットから、128バイトのデータを RAND_seed に渡してPRNGを初期化します。

重要な不変条件

  • **エントロピーの充足検証（mod_ssl）**: 処理の最後に必ず RAND_status() を呼び出し、エントロピーが十分に供給されたか（戻り値が0以外か）を検証します。
  • **スタックバッファの境界（ab）**: スタックからシードを抽出する際、ssl_rand_choosenum が返すオフセットは、確保された一時バッファ stackdata（256バイト）の範囲を超えないよう、sizeof(stackdata) - 128 - 1 を上限として制限されます。

よくある落とし穴

  • **パフォーマンスへの影響**: mod_ssl で外部コマンド実行（SSL_RSSRC_EXEC）をシードソースとして設定している場合、外部プロセスの起動オーバーヘッドにより、サーバーの起動時間に悪影響を及ぼす可能性があります。
  • **エントロピー不足の警告**: RAND_status() が 0 を返した場合、エントロピー不足として警告ログが出力されますが、関数自体はエラーを返さずに正常終了するため、ログを監視していないと潜在的なセキュリティリスクを見落とす可能性があります。
  • **同名関数の混同**: modules/ssl/ssl_engine_rand.c の ssl_rand_seed（引数あり）と support/ab.c の ssl_rand_seed（引数なし、static）は、名前は同じですがシグネチャも役割も異なる別個の実装です。

次に読む場所

  • modules/ssl/ssl_engine_rand.c: mod_ssl における詳細なエントロピー収集アルゴリズムの実装。
  • support/ab.c: ab ツールにおける簡易的なPRNG初期化処理の実装。
