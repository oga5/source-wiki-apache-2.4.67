clean_child_exit

概念

clean_child_exit は、Apache HTTP Server (httpd) の各種マルチプロセッシングモジュール (MPM) において、子プロセスまたはワーカースレッドを安全かつクリーンに終了させるための関数です。各MPMのアーキテクチャ（プロセスモデル、スレッドモデル、OS依存の挙動など）に合わせて、リソースの解放や状態の更新、最終的な終了処理をカプセル化しています。

なぜ重要か

子プロセスやスレッドが終了する際、割り当てられたメモリプールやバケットアロケータなどのリソースを適切に解放しないと、リソースリークが発生します。また、スコアボードの状態更新（例: WORKER_DEAD への変更）や、シグナルハンドリングの調整、POD (Pipe of Death) のクローズなどを確実に行うことで、サーバー全体の安定稼働とクリーンなシャットダウンシーケンスを保証するために極めて重要です。

エンドツーエンドの流れ

各MPMにおける具体的なエンドツーエンドの呼び出しフロー（どのようなトリガーで本関数が呼び出され、どのように全体が遷移するか）の全容は、提供された情報からは**完全には判明していません**。

ただし、判明している個々の実装における内部フローは以下の通りです。

  • **OS/2 (mpmt_os2_child.c)**:

    • 1. 子プロセスのメモリプール pchild が存在すれば破棄する (apr_pool_destroy)。 2. exit(code) を呼び出してプロセスを終了する。

  • **NetWare (mpm_netware.c)**:

    • 1. バケットアロケータを破棄する。 2. サーバー停止中 (shutdown_pending) でなければ、トランザクションプール ptrans を破棄する。 3. ワーカースレッド数を減算し、スコアボード上のスレッド状態を WORKER_DEAD に更新する。 4. NXThreadExit を呼び出してスレッドを終了する。

  • **Prefork (prefork.c)**:

    • 1. MPM状態を AP_MPMQ_STOPPING に設定し、SIGHUP/SIGTERM シグナルを無視する。 2. 正常終了（コード0）の場合は ap_run_child_stopping フックを実行する。 3. pchild メモリプールを破棄する。 4. one_process モードの場合は終了を記録し、PODを閉じる。 5. exit(code) を呼び出してプロセスを終了する。

重要な不変条件

  • **プロセスの終了保証**: 本関数は呼び出し後に制御を戻しません（exit や NXThreadExit を呼び出すため）。
  • **メモリプールの存在確認**: pchild などのメモリプールを破棄する際、NULLチェックを行い、存在する場合のみ破棄を実行します。
  • **NetWareにおける条件付き破棄**: NetWare環境において shutdown_pending が真（サーバー停止中）の場合、トランザクションプール ptrans の破棄はスキップされます。

よくある落とし穴

  • **MPM間での混同**: clean_child_exit は複数のMPMで同名で定義されていますが、それぞれシグネチャや内部処理（プロセス終了かスレッド終了かなど）が異なります。これらを混同して他モジュールの呼び出し元と結びつけないよう注意が必要です。
  • **リターンしない設計**: 本関数を呼び出した後は、呼び出し元に制御が戻りません。関数の後にクリーンアップ処理などを記述しても実行されません。
