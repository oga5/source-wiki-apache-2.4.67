process_socket

概念

process_socket は、Apache HTTP Server のマルチプロセッシングモジュール（MPM）である event および worker において、クライアントから受け入れた個々のネットワークソケット接続を処理するための内部関数です。

なぜ重要か

この関数は、各MPMのワーカースレッド（worker_thread）から直接呼び出され、接続の初期化、ステータスの更新、および実際の接続処理をキックする重要なエントリーポイントです。MPMのアーキテクチャ（スレッド駆動の worker と、イベント駆動の event）に応じて、ソケットのライフサイクルを適切に管理する役割を担っています。

エンドツーエンドの流れ

詳細なエンドツーエンドの流れは完全には解析されていませんが、静的解析から以下の流れが確認されています。

1. **呼び出し元**: どちらのMPMでも、ワーカースレッドである worker_thread から process_socket が呼び出されます。 2. **共通の初期処理**: どちらの関数も、スコアボードハンドルの作成や更新（ap_create_sb_handle, ap_update_sb_handle）を行います。 3. **worker MPM での流れ**:

  • ap_process_connection を呼び出して接続を同期的に処理します。
  • 処理終了後、ap_lingering_close を呼び出してソケットを閉じます。

    • 4. **event MPM での流れ**:

  • 非同期イベント駆動処理に対応するため、接続状態（event_conn_state_t）を管理します。
  • ap_pre_connection による事前処理、キューへの追加・削除（TO_QUEUE_APPEND, TO_QUEUE_REMOVE）、サスペンドやレジュームの通知（notify_suspend, notify_resume）など、イベントループと連携した複雑な制御を行います。
