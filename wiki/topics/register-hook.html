register_hook

概念

register_hook は、Apache HTTP Server の各モジュールにおいて、モジュールロード時に自身の機能（ハンドラ、プロバイダー、初期化関数など）を Apache のフックシステムやプロバイダー機構に登録するためのエントリポイントとなる関数です。

各モジュール（HTTP/2 プロキシ、ロードバランサーなど）が個別に同名の static void register_hook(apr_pool_t *p) 関数を定義し、それぞれの役割に応じた登録処理を行います。

なぜ重要か

Apache HTTP Server はモジュール式のアーキテクチャを採用しており、コア機能と各モジュール間の連携はフック（Hook）やプロバイダー（Provider）の登録を介して行われます。register_hook は、モジュールがロードされた直後に呼び出され、以下のような登録を行うため、モジュールが正しく機能するための起点として極めて重要です。

  • **プロキシハンドラの登録**: HTTP/2 プロキシなどのスキームハンドラや URL 正規化ハンドラを登録します。
  • **ロードバランシング手法の登録**: mod_proxy のバランサーに対して、特定のアルゴリズム（bybusyness, byrequests, bytraffic など）をプロバイダーとして登録します。
  • **初期化処理の登録**: 設定読み込み完了後（post_config）に実行すべき初期化関数を登録します。

エンドツーエンドの流れ

1. **モジュールのロード**: Apache HTTP Server の起動または再起動時に、各モジュールがロードされます。 2. **登録関数の呼び出し**: Apache のモジュールロード機構が、各モジュール内に定義された register_hook を呼び出します。 3. **フック・プロバイダーの登録**:

  • **HTTP/2 プロキシ (mod_proxy_http2.c)**: proxy_hook_scheme_handler や proxy_hook_canon_handler を用いて、優先度 APR_HOOK_FIRST でハンドラを登録します。また、ap_hook_post_config で設定完了後の初期化関数を登録します。
  • **ロードバランサー (mod_lbmethod_*.c)**: ap_register_provider を用いて、PROXY_LBMETHOD カテゴリにそれぞれのアルゴリズム名（"bybusyness", "byrequests", "bytraffic") で構造体を登録します。さらに、ap_hook_post_config を用いて、優先度 APR_HOOK_MIDDLE で設定完了後の初期化関数を登録します。

    • 4. **設定完了後の初期化**: Apache の設定読み込みが完了すると、登録された post_config フック（h2_proxy_post_config や各 lbmethod_*_post_config）が呼び出され、実際の初期化処理が実行されます。

※なお、Apache のモジュールロード機構がどのようにして各モジュールの register_hook を検出し、呼び出すかの具体的な制御フロー（モジュール構造体とのマッピングなど）は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **実行タイミング**: register_hook は、Apache のモジュールロードプロセス中に呼び出される必要があります。
  • **登録対象の存在**: 登録される各ハンドラ（例: proxy_http2_handler）やプロバイダー構造体（例: byrequests）は、登録時点で定義済みでなければなりません。
  • **メモリプール**: 登録処理には、引数として渡される apr_pool_t *p（メモリプール）が使用されます。このプールは有効でなければなりません。

よくある落とし穴

  • **同名関数の混同**: register_hook は各モジュール内で static 関数として定義されている同名の別関数です。例えば、mod_cgid.c の register_hook と mod_proxy_http2.c の register_hook は全く無関係であり、ソースコードの静的解析やデバッグ時にこれらを混同しないよう注意が必要です。
  • **優先度の競合**: mod_proxy_http2.c では APR_HOOK_FIRST が指定されており、他のプロキシモジュールよりも優先して実行されます。一方、ロードバランサーモジュールでは APR_HOOK_MIDDLE が指定されています。これらの優先度設定を変更すると、モジュール間の実行順序に予期しない影響を与える可能性があります。
  • **プロバイダー名の競合**: ap_register_provider で登録する名称（"byrequests" など）が他のモジュールと重複した場合、登録が競合して予期しない動作を引き起こす可能性があります。
