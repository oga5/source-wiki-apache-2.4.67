Module: include

概念

include ディレクトリは、Apache HTTP Server (httpd) のコアシステム、マルチプロセッシングモジュール (MPM)、プロトコル処理、および各種ユーティリティが共有する公開API、データ構造、マクロ、およびフックの定義を格納するヘッダーファイル群で構成されています。

なぜ重要か

このモジュール（ヘッダー群）は、Apacheのコア機能と外部モジュール（サードパーティ製モジュールを含む）との間のインターフェースを定義します。一貫したAPI（例: httpd.h, http_config.h）を提供することで、モジュールの独立性を保ちつつ、サーバー全体の拡張性と互換性を維持するために不可欠な役割を果たしています。

関与するモジュール

  • module:include

主要エンティティ

提供されたヘッダーファイル群は、主に以下のカテゴリに分類されます。

  • **コアおよび設定定義**:
  • include/httpd.h: サーバーの基本データ構造とコアAPI。
  • include/http_config.h: モジュール設定およびコマンド処理。
  • include/ap_config.h: プラットフォーム固有の設定と互換性。
  • **リクエスト・プロトコル処理**:
  • include/http_protocol.h: HTTPプロトコルハンドラ。
  • include/http_request.h: リクエスト処理のフェーズとディスパッチ。
  • include/http_connection.h: 接続レベルの処理。
  • **ランタイム・マルチプロセッシング (MPM)**:
  • include/ap_mpm.h / include/mpm_common.h: MPM共通インターフェース。
  • include/scoreboard.h: プロセス・スレッド状態の監視。
  • **ユーティリティ・共通機能**:
  • include/util_filter.h: フィルタチェーンの管理。
  • include/ap_hooks.h: フックメカニズムの定義。
  • include/ap_mmn.h: モジュールマジックナンバー（API互換性管理）。

エンドツーエンドの流れ

これらのヘッダーファイルはコンパイル時に参照される静的な定義であるため、実行時の具体的なエンドツーエンドの制御フローは、これらを参照する各モジュール（例: mod_core, MPMなど）の実装に依存します。現時点では、ヘッダー定義単体からの詳細なエンドツーエンドの流れは未確定です。

重要な不変条件

  • **API互換性の維持**: include/ap_mmn.h で定義されるModule Magic Number (MMN) により、モジュールとサーバーコア間のバイナリ互換性が厳格に管理されます。
  • その他の詳細な不変条件（構造体のスレッドセーフ性や初期化順序など）は、現時点では未確定です。

よくある落とし穴

  • **MMNの不一致**: サーバーコアとビルドされたモジュール間でMMNが一致しない場合、ロード時にクラッシュや未定義動作を引き起こす可能性があります。
  • その他の具体的な落とし穴や依存関係の競合については、現時点では未確定です。

次に読む場所

主要な定義やAPI仕様を理解するために、以下のヘッダーファイルから読み進めることを推奨します。

  • include/httpd.h (コアデータ構造)
  • include/http_config.h (設定とモジュール構造)
  • include/ap_hooks.h (フックシステム)
  • include/ap_mmn.h (モジュール互換性)
  • include/util_filter.h (フィルタシステム)
