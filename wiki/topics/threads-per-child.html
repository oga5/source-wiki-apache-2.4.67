threads_per_child

概念

threads_per_child は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）において、1つの子プロセスが生成・維持するスレッド数を決定するための設定値および関連する処理群です。設定ファイル内の ThreadsPerChild ディレクティブを介して指定されます。

なぜ重要か

この設定は、サーバーの並行処理能力に直接影響を与えます。各MPM（event、worker、winnt）は、この値に基づいて子プロセス内のスレッド数を制御し、リクエストを処理するスレッドのプールを管理します。

関与するモジュール

  • symbol:server/mpm/event/event.c#set_threads_per_child (関数): event MPMにおける ThreadsPerChild ディレクティブのハンドラ。
  • symbol:server/mpm/event/event.c#threads_per_child (変数): event MPMでスレッド数を保持する変数。
  • symbol:server/mpm/winnt/mpm_winnt.c#set_threads_per_child (関数): winnt MPMにおける ThreadsPerChild ディレクティブのハンドラ。
  • symbol:server/mpm/worker/worker.c#set_threads_per_child (関数): worker MPMにおける ThreadsPerChild ディレクティブのハンドラ。
  • symbol:server/mpm/worker/worker.c#threads_per_child (変数): worker MPMでスレッド数を保持する変数。

エンドツーエンドの流れ

1. 設定ファイルの解析中に ThreadsPerChild ディレクティブが検出されると、各MPM（event, winnt, worker）に定義された set_threads_per_child 関数が AP_INIT_TAKE1 マクロ経由で呼び出されます。 2. set_threads_per_child 内で ap_check_cmd_context を使用して、設定がグローバルスコープで行われているか検証されます。 3. 入力された文字列引数が atoi によって整数に変換され、グローバル変数 ap_threads_per_child に代入されます。

※ 設定された ap_threads_per_child の値が、実際に子プロセスの起動やスレッドプールの生成においてどのように使用されるか、その詳細なエンドツーエンドのライフサイクルや流れは、提供された情報からは未だ明らかになっていません。

重要な不変条件

  • **グローバルスコープでの設定**: ThreadsPerChild ディレクティブは、グローバルコンテキスト（バーチャルホストやディレクトリコンテキスト内ではなく）で設定されなければなりません。これは ap_check_cmd_context によって検証されます。

よくある落とし穴

  • **非数値の指定**: set_threads_per_child は入力文字列の変換に atoi を使用しているため、数値として解釈できない無効な文字列が指定された場合、エラーにならずに 0 が設定されます。
  • **MPM間の共通性**: ThreadsPerChild は複数のMPM（event, worker, winnt）で共通して使用されるディレクティブですが、それぞれのMPMのソースコード（event.c, mpm_winnt.c, worker.c）で個別にハンドラ関数が定義されています。
