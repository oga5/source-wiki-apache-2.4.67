worker_thread

概念

worker_thread は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）である event および worker において、クライアントからの接続やリクエストを処理するために生成されるワーカースレッドのエントリポイント関数です。

なぜ重要か

この関数は、サーバーが並行してリクエストを処理するためのコアロジックを担います。子プロセス生成時（make_child）に起動され、キューからソケットを取り出して実際の接続処理（process_socket）へ配送する役割を持っています。

エンドツーエンドの流れ

1. **スレッドの起動**: 子プロセス（make_child）によって worker_thread が作成・起動されます。 2. **タスクの取得と処理**:

  • event MPM では、ap_queue_pop_something などを介して処理対象を取得します。
  • 取得したソケットは process_socket に渡され、実際のリクエスト処理が実行されます。
  • 処理中、ap_update_child_status_from_indexes を用いてスレッドのステータス更新が行われます。

    • 3. **アイドル状態の管理**: スレッドが待機状態に入ると、ap_queue_info_set_idle を呼び出してアイドル状態として登録されます。 4. **終了処理**: シャットダウンやシグナル受信時には、signal_threads や ap_queue_interrupt_all を介してスレッドの終了処理が行われます。
