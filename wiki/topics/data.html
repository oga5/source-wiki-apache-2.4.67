data

概念

Apache HTTP Server（httpd）のソースツリーにおいて、data という名称は複数の異なる文脈で使用されます。

1. **ベンチマーク測定データ (support/ab.c)**: ApacheBench (ab) ツールにおいて、個別の HTTP リクエスト実行時に計測された各種時間メトリクスを保持するための構造体 (struct data) です。 2. **データフィルタ (modules/filters/mod_data.c)**: フィルタモジュールにおける処理に関与するシンボルですが、本コンテキストにおける具体的な役割や構造は未定義です。

なぜ重要か

  • **ベンチマークの正確性**: support/ab.c の struct data は、ApacheBench がリクエストごとの詳細な時間（接続時間、待機時間など）を追跡し、最終的な統計情報（平均、標準偏差、パーセンタイルなど）を正しく算出するための基盤となります。
  • **フィルタ処理**: mod_data はレスポンスデータの変換等に関与しますが、具体的な重要性の詳細は現時点では未定義です。

関与するモジュール

  • module:modules/filters
  • module:support

主要エンティティ

  • symbol:support/ab.c#data: 接続開始時刻 (starttime)、待機時間 (waittime)、接続確立時間 (ctime)、総処理時間 (time) を保持する構造体。
  • symbol:modules/filters/mod_data.c#data: フィルタモジュール内のシンボル（詳細未定義）。

エンドツーエンドの流れ

ApacheBench (ab) における測定フロー

1. **データ記録**: 接続のライフサイクル（接続開始、リクエスト送信、レスポンス受信）の各フェーズにおいて、APR (Apache Portable Runtime) の時間計測関数を用いてタイムスタンプが struct data に記録されます。 2. **統計集計**: 全てのリクエストが終了した後、これらのデータが格納された配列がソートされ、output_results() 関数によって統計処理が実行されます。

mod_data におけるフロー

  • modules/filters/mod_data.c におけるデータ処理のエンドツーエンドの流れは、現時点では不明です。

重要な不変条件

  • **動的確保**: ab の実行中、struct data の配列は同時接続数（concurrency）に基づいたサイズで動的に確保されます。
  • **時間整合性**: 各フィールドには、APR の時間計測関数によって得られた一貫性のある apr_time_t および apr_interval_time_t 型の値が格納される必要があります。
  • **フィルタ側の不変条件**: modules/filters/mod_data.c に関する不変条件は現時点では不明です。

よくある落とし穴

  • **不完全なデータによる統計の歪み**: ab において接続エラーなどが発生した場合、計測値が初期値のままとなり、統計計算時に不完全なデータとして処理される可能性があります。
  • **同名シンボルの混同**: include/ap_expr.h などに存在する data という名前のシンボルは、本トピックで扱う構造体やフィルタとは無関係な別名シンボルです。

次に読む場所

  • support/ab.c: struct data の定義、および output_results() による統計集計ロジックの実装。
  • modules/filters/mod_data.c: フィルタとしての data 処理の具体的な実装。
