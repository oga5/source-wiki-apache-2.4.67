dav_propdb

概念

dav_propdb は、Apache HTTP Server (httpd-2.4.67) の mod_dav モジュールにおいて、WebDAV プロパティ（ライブプロパティおよびデッドプロパティ）の操作コンテキストを保持・管理するための構造体、およびそれに関連するデータベース操作機構です。

なぜ重要か

WebDAV プロトコルにおける PROPFIND や PROPPATCH などのプロパティ操作要求を処理する際、リソース情報、データベース接続、トランザクション状態、名前空間マッピング、ロック情報などを一元的に管理する必要があります。dav_propdb はこれらすべてのコンテキストを統合し、プロパティの解決、検証、更新、および XML 出力の生成を円滑に行うための中核的な役割を担っています。

関与するモジュール

  • module:modules/dav/fs
  • module:modules/dav/main

主要エンティティ

  • symbol:modules/dav/main/props.c#dav_propdb (および symbol:modules/dav/main/mod_dav.h#dav_propdb): プロパティ操作のコンテキストを保持する構造体。
  • symbol:modules/dav/fs/dbm.c#dav_propdb_open: プロパティデータベースを開くための関数。
  • symbol:modules/dav/fs/dbm.c#dav_propdb_close: プロパティデータベースを閉じるための関数。

エンドツーエンドの流れ

1. **初期化**: mod_dav.c 内の各メソッドハンドラ（PROPFIND, PROPPATCH 等）から、dav_popen_propdb 等の初期化関数を通じて dav_propdb コンテキストが作成されます。 2. **データベースのオープン**: deferred フラグが設定されている場合は、実際にプロパティ操作が必要になるまでデータベースのオープンが遅延されます。オープン処理には dav_propdb_open が関与します。 3. **プロパティの解決と操作**:

  • dav_find_liveprop を通じてライブプロパティプロバイダーを特定し、見つからない場合はデッドプロパティとして処理します。
  • PROPPATCH 操作時には、validate -> exec -> commit/rollback の順序でトランザクション処理が行われ、失敗時には dav_rollback_item を用いて状態が復旧されます。

    • 4. **XML 構築**: dav_get_props 等により、名前空間（ns_xlate, mapping）を管理しながら XML 形式のプロパティリストを生成します。 5. **クローズと解放**: 処理の終了時、dav_close_propdb（および dav_propdb_close）を呼び出して、データベースのクローズ、リソースの解放、および内部で生成されたサブリクエスト（subreq）の破棄を確実に行います。

*注意: dav_propdb_open および dav_propdb_close の具体的な内部処理や、mod_dav.c からの完全なエンドツーエンドの呼び出しフローの詳細は、提供された情報からは完全には特定されていません。*

重要な不変条件

  • **メモリ管理**: dav_propdb は、リクエストプール（r->pool）または指定されたプールを使用してメモリ管理を行います。
  • **リソース解放の保証**: サブリクエスト（subreq）が生成された場合、メモリリークやリソースの残留を防ぐために、必ず dav_close_propdb を呼び出して適切に破棄しなければなりません。
  • **名前空間の変換**: プロパティ値内の名前空間は ns<index> 形式に変換されるため、クライアント側で特定のプレフィックスに依存しないようにする必要があります。

よくある落とし穴

  • **遅延オープンの考慮**: deferred フラグによる遅延オープンが有効な場合、コンテキスト初期化時点でデータベースが即座に開かれないため、実際の操作時に初めてオープンエラーが発生する可能性があります。
  • **エラーハンドリングの不完全性**: PROPPATCH のエラーハンドリングにおいて、複雑なケースではエラー伝播が不完全になる可能性があります。
  • **書き込み制限**: DAV_DISABLE_WRITABLE_PROPS が有効な場合、一部のコアプロパティへの書き込みが制限されるため、書き込み操作が予期せず失敗することがあります。
