ap_sb_handle_t

概念

ap_sb_handle_t は、Apache HTTP Server において特定のワーカー（プロセスおよびスレッド）のスコアボード（実行時状態）を識別・参照するためのハンドルです。

公開ヘッダ（include/scoreboard.h）においては、構造体の内部定義が隠蔽された不透明なポインタ（Opaque Pointer）として定義されています。一方、実装ファイル（server/scoreboard.c）の内部において、具体的なインデックス情報を保持する構造体として定義されています。

なぜ重要か

このハンドルは、マルチプロセッシングモジュール（MPM）や各種モジュール（mod_echo など）が、共有メモリ上に配置されたスコアボード内の特定のワーカー状態（worker_score）を特定し、安全に更新・参照するために重要です。

構造体を不透明な型として扱うことで、利用側にスコアボードの内部レイアウトを意識させることなく、状態更新 API（ap_update_child_status など）やアクセサ API（ap_get_scoreboard_worker など）を介した安全な操作を提供します。

関与するモジュール

主要エンティティ

  • **symbol:include/scoreboard.h#ap_sb_handle_t**

    • 外部モジュール向けに公開されている不透明な構造体定義（typedef struct ap_sb_handle_t ap_sb_handle_t;）です。

  • **symbol:server/scoreboard.c#ap_sb_handle_t**

    • server/scoreboard.c 内部で定義されている具体的な構造体です。以下のメンバを持ち、スコアボード内の特定のスロットを指し示すオフセット情報として機能します。 ``c struct ap_sb_handle_t { int child_num; int thread_num; }; ``

エンドツーエンドの流れ

1. **ハンドルの生成**: ap_create_sb_handle などの関数を通じて、特定の child_num および thread_num を保持する ap_sb_handle_t が生成されます。 2. **API への受け渡し**: 生成されたハンドルは、ap_update_child_status、ap_get_scoreboard_worker、ap_time_process_request などのスコアボード操作 API に引数として渡されます。 3. **スロットの特定と更新**: 各 API は、ハンドル内の child_num と thread_num を使用して共有メモリ上の該当する worker_score 構造体を特定し、統計情報や状態の読み書きを行います。

*注意: ハンドルの生成から破棄に至る完全なライフサイクルや、MPM 初期化時の詳細なエンドツーエンドフローは、提供された情報からは完全には判明していません。*

重要な不変条件

  • **有効なインデックスの保持**: ハンドルが保持する child_num および thread_num は、常にスコアボード内の有効なインデックスを指していなければなりません。
  • **単一所有者による書き込み**: 本ハンドルが指すスコアボード上のスロットは、単一の所有者（ワーカー）のみが書き込むことを想定しているため、排他制御（ロック）なしでの更新が可能です。
  • **インデックスの不変性**: メモリレイアウトの設計上、このハンドルが指すインデックスは初期化後に変更されない前提で運用されます。

よくある落とし穴

  • **直接的なメンバアクセス**: ap_sb_handle_t は不透明な型として設計されているため、呼び出し側から直接メンバにアクセスすることは避けてください。worker_score へのアクセスが必要な場合は、必ず ap_get_scoreboard_worker などの公開 API を経由する必要があります。
  • **未初期化・無効なハンドルの使用**: ハンドルが適切に初期化されていない場合や、無効なインデックスを保持している場合、スコアボードのメモリ領域外へのアクセスや、誤った統計情報の更新など、未定義動作を引き起こす可能性があります。
  • **定義の二重管理**: 本構造体は server/scoreboard.c と include/scoreboard.h の両方で定義されているため、構造体を変更する際は両方の整合性を確認する必要があります。

次に読む場所

  • **include/scoreboard.h**: スコアボード操作 API の宣言と、不透明な型としての ap_sb_handle_t の定義を確認できます。
  • **server/scoreboard.c**: ap_sb_handle_t の具体的な構造体定義と、それを利用したスコアボード操作の実装を確認できます。
