Module: modules/dav

概念

modules/dav は、Apache HTTP Server において WebDAV (Web-based Distributed Authoring and Versioning) プロトコル（RFC 4918等）のサポートを提供するモジュール群です。

このモジュール群は、WebDAV のプロトコルハンドリングや共通処理を行うコア部分（main）と、実際のデータ保存やロック管理を行うバックエンドプロバイダ（fs、lock）に分離された、モジュール化設計を採用しています。

なぜ重要か

WebDAV は、HTTP を拡張してサーバー上のファイルやディレクトリを直接編集・管理できるようにするプロトコルです。modules/dav が重要な理由は以下の通りです。

  • **プロトコルとストレージの分離**: コアの mod_dav がプロトコル処理（XMLのパースやレスポンス生成）を担当し、実際のファイル操作やロック処理はプロバイダ（mod_dav_fs など）に委譲するため、異なるストレージバックエンドへの拡張が容易です。
  • **標準的な共同編集環境の提供**: ロック管理（排他制御）やプロパティ管理（メタデータ付与）をサポートし、安全なリモートファイル編集環境を実現します。

関与するモジュール

本機能は以下のサブモジュールおよびディレクトリに分割されて実装されています。

  • **module:modules/dav**: 全体を統括するメタモジュール。
  • **module:modules/dav/main**: WebDAV のコアプロトコル処理、共通ユーティリティ、およびプロバイダインターフェースの定義。
  • **module:modules/dav/fs**: ローカルファイルシステムをリポジトリとして使用するための DAV プロバイダ実装。
  • **module:modules/dav/lock**: 汎用的な DAV ロック管理を提供するプロバイダ実装。

主要エンティティ

提供されたソースファイル群に基づく主要なエンティティと役割は以下の通りです。

  • **mod_dav.c / mod_dav.h (main)**:

    • WebDAV のメインハンドラ。リクエストのディスパッチ、設定ディレクティブの処理、およびプロバイダ登録のライフサイクルを管理します。

  • **providers.c (main)**:

    • リポジトリ、ロック、ライブプロパティなどの各種 DAV プロバイダを登録・解決するためのレジストリ機構を提供します。

  • **props.c / liveprop.c / std_liveprop.c (main)**:

    • WebDAV プロパティ（PROPFIND や PROPPATCH で扱われるメタデータ）の取得・更新処理を担当します。

  • **util.c / util_lock.c (main)**:

    • URI 解析、エラーハンドリング、およびロック検証などの共通ユーティリティ関数群。

  • **mod_dav_fs.c / repos.c / repos.h (fs)**:

    • ファイルシステムベースのリポジトリプロバイダ。実際のファイルやディレクトリに対する操作を抽象化します。

  • **mod_dav_lock.c / locks.c / locks.h (lock)**:

    • リソースに対するロック（LOCK / UNLOCK）を管理する汎用ロックプロバイダ。

エンドツーエンドの流れ

詳細なコードレベルの実行シーケンスや内部状態遷移の全容は、提供されたソースコードメタデータからは**完全には特定されていません**。しかし、構成ファイルから以下の一般的な処理フローが推測されます。

1. **リクエスト受信**: クライアントから WebDAV メソッド（例: PROPFIND, LOCK, PUT）のリクエストが届くと、mod_dav.c のハンドラが起動します。 2. **プロバイダの解決**: mod_dav は、対象のロケーションに設定されている DAV プロバイダ（例: mod_dav_fs）を providers.c を介して解決します。 3. **プロトコル処理と委譲**: mod_dav がリクエストボディ（XML等）を解析し、必要な操作をプロバイダのAPI（repos.c や locks.c の関数）に委譲します。 4. **バックエンド操作**: mod_dav_fs などのプロバイダが、ローカルファイルシステムやロックデータベース（dbm.c 等）に対して実際の操作を実行します。 5. **レスポンス返却**: 操作結果を mod_dav が WebDAV 準拠の XML レスポンス等に整形し、クライアントに返却します。

重要な不変条件

モジュール間の詳細な不変条件（インバリアント）や排他制御の厳密なルールは、提供されたファイル一覧からは**直接検証されていません**。

一般的な設計上の前提として、リポジトリプロバイダ（fs）とロックプロバイダ（lock）は、コアモジュール（main）が定義するプロバイダAPI（mod_dav.h 内の構造体）に厳密に準拠して実装されている必要があります。

よくある落とし穴

  • **プロバイダのロード忘れ**:

    • mod_dav（コア）を有効化するだけでは WebDAV は動作しません。ファイルシステムを対象とする場合は mod_dav_fs を、ロック管理を行う場合は mod_dav_lock などの具体的なプロバイダモジュールを明示的にロードおよび設定する必要があります。

  • **権限不足によるロック/プロパティの失敗**:

    • mod_dav_fs や mod_dav_lock は、ロック情報やプロパティ情報を管理するために DBM ファイル（dbm.c）等を使用します。これらのファイルを格納するディレクトリに対して、Apache の実行権限（www-data や apache など）が適切に付与されていない場合、LOCK や PROPPATCH リクエストが 500 Internal Server Error で失敗します。

次に読む場所

各サブモジュールの具体的な実装を理解するために、以下のファイルから読み進めることを推奨します。

  • **コアプロトコルとプロバイダAPIの理解**:
  • modules/dav/main/mod_dav.h （主要な構造体とプロバイダAPIの定義）
  • modules/dav/main/mod_dav.c （メインハンドラとディレクティブ処理）
  • **ファイルシステムプロバイダの実装**:
  • modules/dav/fs/mod_dav_fs.c （FSプロバイダの登録）
  • modules/dav/fs/repos.c （ファイルシステム操作の実装）
  • **ロック管理の実装**:
  • modules/dav/lock/mod_dav_lock.c （ロックプロバイダの登録）
  • modules/dav/lock/locks.c （ロックデータベースの操作）
