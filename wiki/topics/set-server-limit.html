set_server_limit

概念

set_server_limit は、Apache HTTP Server（httpd-2.4.67）のマルチプロセッシングモジュール（MPM）において、生成可能な最大サーバープロセス数（ServerLimit）を設定・制御するための処理です。

なぜ重要か

Apache HTTP Serverでは、クライアントからのリクエストを処理するために複数のプロセスやスレッドを使用します。event、prefork、worker といった異なる並行処理モデルを提供する各MPMにおいて、システムリソースの過剰消費を防ぎ安定稼働を維持するために、起動するプロセス数の上限を適切に制限・管理することが極めて重要です。

関与するモジュール

本機能には、以下のMPMモジュールが関与しています。

  • module:server/mpm/event
  • module:server/mpm/prefork
  • module:server/mpm/worker

主要エンティティ

各MPMの実装ファイル内に、それぞれ対応する設定ハンドラ関数が定義されています。

  • symbol:server/mpm/event/event.c#set_server_limit
  • symbol:server/mpm/prefork/prefork.c#set_server_limit
  • symbol:server/mpm/worker/worker.c#set_server_limit

エンドツーエンドの流れ

設定ファイルの解析時に ServerLimit ディレクティブが検出された際、該当するMPMの set_server_limit が呼び出されて内部の制限値が適用されると推測されますが、起動フェーズにおける具体的なコールスタックや詳細なエンドツーエンドの処理フローは、現時点では未確定です。

重要な不変条件

各MPMにおける ServerLimit の設定値が、コンパイル時のハードリミットや他の設定値（MaxRequestWorkers など）とどのように整合性を保つべきかといった、具体的な不変条件は現時点では未確定です。

よくある落とし穴

選択したMPM（prefork、worker、event）の特性に応じて、設定値がシステム全体のスレッド数やプロセス数に与える影響が異なりますが、具体的な設定ミスや競合などの落とし穴の詳細は現時点では未確定です。
