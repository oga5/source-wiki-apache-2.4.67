set_server_limit

概念

set_server_limit は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）において、生成可能なプロセス数の上限（ServerLimit）を設定ファイルから読み込んで適用するための設定ディレクティブハンドラ関数です。この関数は、各MPM（event、prefork、worker）のソースコード内にそれぞれ定義されています。

なぜ重要か

この関数は、Apacheの並行処理能力を制御する重要なパラメータである ServerLimit を適切に処理するために必要です。各MPMが起動するプロセスの最大数を制限し、システムのメモリやリソース消費を適切に管理する役割を担っています。

関与するモジュール

本機能には、以下のMPM（マルチプロセッシングモジュール）が関与しています。

  • module:server/mpm/event
  • module:server/mpm/prefork
  • module:server/mpm/worker

エンドツーエンドの流れ

設定ファイルの解析中に ServerLimit ディレクティブが検出されると、該当するMPMの set_server_limit 関数が呼び出されます。 1. 設定パーサーから cmd_parms および設定値の文字列である引数 arg が渡されます。 2. 各関数内では、ap_check_cmd_context を呼び出して、ディレクティブが適切なコンテキスト（グローバル設定など）で記述されているかを検証します。
