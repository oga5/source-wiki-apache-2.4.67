os_forktype

概念

os_forktype は、BS2000環境（_OSD_POSIX マクロ定義下）において、現在の実行環境（デバッグモードの有無や実行ユーザーの権限）に適したプロセス生成手法（bs2_ForkType）を判定し、その結果をキャッシュして返却する関数です。

なぜ重要か

BS2000環境において、特権ユーザー（root）か非特権ユーザーか、あるいは単一プロセス実行（デバッグモード等）が強制されているかによって、最適なプロセス生成手法（bs2_noFORK, bs2_FORK, bs2_UFORK）が異なります。os_forktype はこれらを動的に判定し、適切なプロセス生成を保証するために重要です。

関与するモジュール

  • module:os/bs2000
  • module:os/unix

主要エンティティ

  • symbol:os/bs2000/os.c#os_forktype: static bs2_ForkType os_forktype(int one_process) として定義される関数。プロセス生成手法を判定・キャッシュします。
  • symbol:os/unix/unixd.c#os_forktype: os/unix/unixd.c 内のプロセス管理処理から呼び出される参照。

エンドツーエンドの流れ

1. **呼び出し**: os/bs2000/os.c 内のプロセス生成処理、または os/unix/unixd.c 内のプロセス管理処理から os_forktype(one_process) が呼び出されます。 2. **キャッシュ確認**: 静的変数 forktype が bs2_unknown 以外（初期化済み）であれば、キャッシュされた値を即座に返します。 3. **判定**: 未初期化の場合、以下の順に判定します。

  • one_process が真の場合: bs2_noFORK を設定。
  • 実行ユーザーが非特権（getuid() != 0）の場合: bs2_FORK を設定。
  • 特権ユーザーの場合: bs2_UFORK を設定。

    • 4. **キャッシュと返却**: 決定した値を forktype に格納し、呼び出し元に返します。

※なお、呼び出し元である os/bs2000/os.c や os/unix/unixd.c の具体的なプロセス生成・管理処理の全体像や、返された bs2_ForkType がどのようにプロセス生成システムコール等に適用されるかの詳細なエンドツーエンドのフローは、提供された情報からは不明です。

重要な不変条件

  • **キャッシュの不変性**: 静的変数 forktype は一度 bs2_unknown 以外の値に決定されると、以降の呼び出しでは再判定を行わず、常に同じ値を返します。
  • **権限依存**: 特権ユーザー（getuid() == 0）かつ one_process が偽の場合にのみ bs2_UFORK が選択されます。

よくある落とし穴

  • **実行権限の動的変更への非追従**: forktype は静的変数にキャッシュされるため、プロセス起動後に実行権限（UID）が変更されても、os_forktype の返す値は更新されません。
  • **環境制限**: 本関数は _OSD_POSIX マクロが定義された環境でのみ有効です。
  • **エラーハンドリング**: 関数自体はエラーを返さないため、getuid() の結果や実行環境の権限設定が意図しない状態であっても、そのまま判定結果（bs2_ForkType）が返されます。
