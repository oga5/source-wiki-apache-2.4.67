ap_filter_provider_t

概念

ap_filter_provider_t は、Apache HTTP Server（特に mod_filter）において、特定の条件に基づいて動的に適用される「スマートフィルタ」のプロバイダを管理するためのデータ構造です。

ヘッダーファイル include/util_filter.h においては不完全型として宣言されており、具体的な構造体のメンバ定義は modules/filters/mod_filter.c の内部に隠蔽（カプセル化）されています。

なぜ重要か

この構造体により、フィルタチェーンを静的に固定するのではなく、リクエスト実行時に条件式（ap_expr_info_t）の評価結果やコンテンツタイプ（types）に応じて、適切なフィルタ（ap_filter_rec_t）を動的に選択・適用することが可能になります。これにより、柔軟なコンテンツ処理パイプラインが実現されます。

関与するモジュール

  • module:include （include/util_filter.h を提供）
  • module:modules/filters （modules/filters/mod_filter.c を提供）

主要エンティティ

  • **symbol:include/util_filter.h#ap_filter_provider_t**

    • 外部公開用の不完全型宣言です。 ``c typedef struct ap_filter_provider_t ap_filter_provider_t; ``

  • **symbol:modules/filters/mod_filter.c#ap_filter_provider_t**

    • mod_filter.c 内で定義される具体的な構造体です。 ``c struct ap_filter_provider_t { ap_expr_info_t *expr; /* フィルタ適用条件を評価する式 */ const char **types; /* 適用対象となるコンテンツタイプのリスト */ ap_filter_rec_t *frec; /* このプロバイダが実装するフィルタの実体 */ ap_filter_provider_t *next; /* 連結リスト内の次のプロバイダへのポインタ */ }; ``

エンドツーエンドの流れ

1. **設定フェーズ**: 設定の解析時（add_filter などの実行時）に、設定用プール（cmd->pool）から apr_palloc を用いて ap_filter_provider_t が割り当てられます。条件式やコンテンツタイプ、対応する ap_filter_rec_t が設定され、next ポインタを介して単方向連結リストが構築されます。 2. **リクエスト処理フェーズ**: mod_filter のディスパッチャ（filter_lookup など）が、このプロバイダの連結リストを走査します。 3. **評価と適用**: 各プロバイダの expr の評価や types の一致確認を行い、リクエストに最適なプロバイダを特定します。マッチしたプロバイダが保持する frec の関数ポインタを介して、実際のフィルタ処理が呼び出されます。

*注意: 設定コマンドの詳細なバインド処理や、ディスパッチャとコアフィルタシステムとの完全な相互作用の全容は、提供された情報からは未解明です。*

重要な不変条件

  • **単方向連結リストの終端**: リストの末尾ノードの next ポインタは必ず NULL でなければなりません。
  • **メモリのライフサイクル**: プロバイダのメモリは設定時のプール（cmd->pool）から割り当てられるため、リクエスト処理中の個別のメモリ解放は不要であり、ライフサイクルは自動的に管理されます。
  • **優先順位**: 複数のプロバイダが条件に合致する場合、連結リストの先頭に近い（先に登録された）ものが優先して適用されます。

よくある落とし穴

  • **不完全型によるコンパイルエラー**:

    • 本構造体は include/util_filter.h では不完全型として定義されているため、mod_filter.c 以外のモジュールから直接構造体のメンバ（expr や next など）にアクセスしようとするとコンパイルエラーになります。構造体の拡張や操作を行う場合は、必ず mod_filter.c の実装を確認する必要があります。

  • **条件式のスキップ**:

    • expr が NULL の場合、条件式による評価はスキップされます。意図しないスキップを防ぐため、設定時の適切な初期化が必要です。

次に読む場所

  • include/util_filter.h （関連する ap_filter_t や ap_filter_rec_t の定義）
  • modules/filters/mod_filter.c （filter_lookup や設定関数などの具体的な管理ロジック）
