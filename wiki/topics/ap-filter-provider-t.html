ap_filter_provider_t

概念

ap_filter_provider_t は、Apache HTTP Server（特に mod_filter）において、特定の条件に基づいて適用されるフィルタプロバイダーを管理するための構造体です。

ヘッダーファイル include/util_filter.h では不完全型として宣言されており、具体的な構造体のメンバ定義は modules/filters/mod_filter.c 内に隠蔽されています。

なぜ重要か

この構造体は、実行時に適切なフィルタを動的に選択・適用する「スマートフィルタ」機能の核となるデータ構造です。特定のフィルタ名に対して、条件式やコンテンツタイプに基づいた複数のプロバイダー（実装）を連結リストとして紐付け、リクエストのコンテキストに応じて最適なフィルタ（ap_filter_rec_t）を選択できるようにします。

関与するモジュール

  • module:include （include/util_filter.h を提供）
  • module:modules/filters （modules/filters/mod_filter.c を提供）

主要エンティティ

  • **symbol:include/util_filter.h#ap_filter_provider_t**

    • 外部モジュール向けに公開されている不完全型の宣言です。

  • **symbol:modules/filters/mod_filter.c#ap_filter_provider_t**

    • mod_filter.c 内で定義されている具体的な構造体です。以下のメンバを持ちます。

  • ap_expr_info_t *expr: フィルタを適用するための条件式。
  • const char **types: 適用対象となるコンテンツタイプのリスト。
  • ap_filter_rec_t *frec: このプロバイダーが実装するフィルタのレコード。
  • ap_filter_provider_t *next: リスト内の次のプロバイダーへのポインタ。

エンドツーエンドの流れ

1. **設定フェーズ**: 設定関数（add_filter など）の実行時に、メモリプール（cmd->pool）から apr_palloc を用いて ap_filter_provider_t のメモリが割り当てられ、プロバイダーが登録されます。 2. **リスト構築**: 登録されたプロバイダーは、next ポインタを介して単方向連結リストを形成します。 3. **リクエスト処理フェーズ**: mod_filter のディスパッチャ（filter_lookup）が、このプロバイダーの連結リストを走査します。 4. **評価と選択**: 各プロバイダーの条件式（expr）の評価結果や、コンテンツタイプ（types）の一致を確認し、リクエストに最適なプロバイダーを特定します。 5. **実行**: マッチしたプロバイダーの frec が持つ関数ポインタを介して、実際のフィルタ処理が呼び出されます。

重要な不変条件

  • **リストの終端**: 連結リストの末尾は必ず NULL で終端されていなければなりません。
  • **優先順位**: 複数のプロバイダーが条件に合致する場合、リストの先頭に近い（先に登録された）ものが優先して適用されます。
  • **メモリのライフサイクル**: 設定時に cmd->pool を使用して割り当てられるため、リクエスト終了後のライフサイクル管理は Apache のプール機構によって自動的に行われます。

よくある落とし穴

  • **定義の二重管理**:

    • 本構造体は include/util_filter.h と modules/filters/mod_filter.c の両方に定義が存在するため、構造体を変更・拡張する際は両ファイルの整合性に注意する必要があります。

  • **条件式のスキップ**:

    • expr が NULL の場合、条件式による評価はスキップされます。

  • **メモリ割り当て失敗**:

    • apr_palloc によるメモリ割り当てが失敗した場合の挙動は、呼び出し側のメモリ管理ポリシーに依存します。

次に読む場所

  • include/util_filter.h: 不完全型としての宣言と、関連するフィルタ構造体（ap_filter_t、ap_filter_rec_t）の定義。
  • modules/filters/mod_filter.c: 構造体の詳細な定義、および filter_lookup や add_filter などの具体的な管理・適用ロジック。
