cmds

概念

cmds は、Apache HTTP Server (httpd) の各モジュールにおいて、設定ファイル（httpd.conf など）から読み込まれる設定ディレクティブ（コマンド）を定義・登録するための構造体配列（通常 command_rec 型）を指します。各モジュールは、自身が処理可能なディレクティブ、その引数の数、適用可能なコンテキスト、および対応する設定ハンドラ関数をこの配列に定義します。

なぜ重要か

Apacheのコアシステムが各モジュール固有の設定ディレクティブを認識し、設定ファイルのパース時に適切なハンドラを呼び出すために不可欠です。cmds が正しく定義され、モジュール構造体に登録されていない場合、モジュール独自の設定ディレクティブをApacheが解釈できず、起動エラーや設定の無視が発生します。

関与するモジュール

  • module:modules/filters
  • module:modules/proxy
  • module:modules/proxy/balancers

主要エンティティ

各モジュールファイル内で定義されている cmds シンボルが主要なエンティティです。

  • symbol:modules/filters/mod_brotli.c#cmds : Brotli圧縮フィルターの設定ディレクティブ群
  • symbol:modules/filters/mod_charset_lite.c#cmds : 文字コード変換フィルターの設定ディレクティブ群
  • symbol:modules/filters/mod_ext_filter.c#cmds : 外部フィルター定義の設定ディレクティブ群
  • symbol:modules/proxy/balancers/mod_lbmethod_heartbeat.c#cmds : Heartbeatベースのロードバランシング手法の設定ディレクティブ群
  • symbol:modules/proxy/mod_proxy_connect.c#cmds : Proxy CONNECTメソッドの設定ディレクティブ群

エンドツーエンドの流れ

Apacheの起動時および設定再読み込み時に、コアシステムは登録された各モジュールの cmds 配列を走査し、設定ファイル内のディレクティブと照合します。一致するディレクティブが見つかると、cmds 内で指定されたハンドラ関数が呼び出され、設定値がモジュールの設定構造体に格納されます。

ただし、提供された情報からは、具体的なパース処理や登録処理の内部実装における詳細なエンドツーエンドの制御フローは未だ不明です。

重要な不変条件

  • **センチネルによる終端**: cmds 配列の終端は、通常、すべてのフィールドが NULL または 0 で初期化された空の要素（センチネル）で終わる必要があります。これにより、ループ処理が配列の境界を越えてメモリを読み込むのを防ぎます。

ただし、本ソースコード群における具体的な不変条件の検証状況は現時点では不明です。

よくある落とし穴

  • **終端センチネルの欠如**: cmds 配列の最後にセンチネル要素を置き忘れると、メモリの範囲外アクセスが発生し、サーバー起動時にセグメンテーションフォールトを引き起こす原因になります。
  • **引数指定の不一致**: ディレクティブの引数タイプ（TAKE1, ITERATE など）と、ハンドラ関数のシグネチャが一致していない場合、実行時に不正なメモリ参照やクラッシュを引き起こす可能性があります。
