ap_cgi_build_command

概念

ap_cgi_build_command は、Apache HTTP Serverにおいて、プラットフォーム固有のCGI実行コマンドラインおよび引数リスト（argv）を構築するための関数です。NetWareやWindowsなどの非UNIX系OS環境において、CGIスクリプトや実行ファイルを適切に起動するためのインタープリタの特定、パスの解決、引数のトークン化などの処理を抽象化します。

なぜ重要か

UNIX系OSとは異なり、WindowsやNetWare環境では、CGIスクリプト（例: .pl, .py, .bat）を直接実行する仕組みや、シバン（#!）の解釈、レジストリによる関連付け、NLM（NetWare Loadable Module）の起動方法などがOS固有の仕様に依存します。ap_cgi_build_command は、これらのプラットフォーム固有の差異を吸収し、mod_cgi などのジェネリックなCGI実行モジュールに対して統一されたインターフェースを提供するために重要です。

エンドツーエンドの流れ

1. **呼び出しのトリガー**: modules/generators/mod_cgi.c が、オプション関数（Optional Function）として登録された ap_cgi_build_command を APR_RETRIEVE_OPTIONAL_FN を介して動的に取得し、呼び出します。 2. **プラットフォーム固有の解析**:

  • **NetWareの場合**: リクエストされたファイルの拡張子に基づき、netware_dir_config 内の file_type_handlers から対応するハンドラ（NLMなど）を検索します。
  • **Windowsの場合**: ファイル拡張子（.exe, .bat など）の判定、レジストリ（ScriptInterpreterSource 設定に基づく）の検索、またはファイル先頭のシバン（#!）行の解析を行い、適切なインタープリタを特定します。

    • 3. **コマンド・引数の構築**:

  • **NetWareの場合**: [ハンドラ] [ファイル名] の形式でコマンドを生成し、apr_tokenize_to_argv で引数配列に変換後、ap_server_root_relative で絶対パスを解決します。
  • **Windowsの場合**: split_argv を用いて引数テンプレートを展開・配列化し、prep_string で環境変数をWindows向けに調整します。

    • 4. **結果の返却**: 構築されたコマンドパス（cmd）と引数配列（argv）を呼び出し元に返し、CGIプロセスが起動されます。

※なお、mod_cgi 側での具体的なプロセス起動処理や、その他のプラットフォーム（UNIX等）における詳細なエンドツーエンドの統合フローの全容は、提供されたコンテキストからは完全には明らかになっていません。

重要な不変条件

  • **メモリ管理**: すべての動的メモリ確保は、引数として渡されるメモリプール（apr_pool_t *p）に依存して行われなければなりません。
  • **インターフェースの一貫性**: プラットフォームごとの実装（NetWare、Windows）は、同一の関数シグネチャ（apr_status_t ap_cgi_build_command(...)）を持ち、mod_cgi から透過的に呼び出せる必要があります。
  • **NetWareの不変条件**: netware_dir_config がリクエストのディレクトリ設定から取得可能であること、および拡張子に対応するハンドラが登録されている必要があります。
  • **Windowsの不変条件**: e_info->cmd_type が APR_SHELLCMD 以外でインタープリタが特定された場合、APR_PROGRAM_PATH に設定される必要があります。

よくある落とし穴

  • **NetWare環境**: 拡張子に対応するコマンドが file_type_handlers に登録されていない場合、APR_EBADF を返して処理が失敗します。
  • **Windows環境**:
  • シバン解析において、UTF-8のBOM（\xEF\xBB\xBF）の考慮漏れや、ファイル読み込み失敗（apr_file_open の失敗）により、インタープリタが特定できず APR_EBADF となることがあります。
  • prep_string によるUTF-8からWindowsローカルコードページへの変換は限定的な実装に留まるため、文字コード関連の不整合が発生する可能性があります。
  • split_argv 内の引数解析は複雑なエスケープ処理を含むため、Windowsのコマンドライン引数ルールに反する入力を与えると予期しない挙動を示す恐れがあります。
