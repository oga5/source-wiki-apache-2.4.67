ap_cgi_build_command

概念

ap_cgi_build_command は、Apache HTTP Serverにおいて、プラットフォーム固有（NetWare および Windows）の環境下でCGIスクリプトや実行ファイルを起動するために必要なコマンドライン文字列および引数配列（argv）を構築するための関数です。

なぜ重要か

CGIの実行方法はオペレーティングシステムによって大きく異なります。例えば、NetWareではNLM（NetWare Loadable Module）の実行やアドレス空間の制御が必要であり、Windowsでは拡張子の判別、レジストリ検索、シバン（#!）解析、複雑な引数のエスケープ処理が必要です。ap_cgi_build_command は、これらのプラットフォーム固有の差異を吸収し、mod_cgi などのジェネリックなCGI実行モジュールに対して統一されたインターフェース（オプショナル関数）を提供するために重要です。

エンドツーエンドの流れ

mod_cgi などの外部モジュールからプロセス起動に至る完全なエンドツーエンドのフローは、提供されたコンテキストからは未だ不明ですが、本関数内における処理の流れは以下の通りです。

1. **呼び出し**: modules/generators/mod_cgi.c が APR_RETRIEVE_OPTIONAL_FN を介して本関数を動的に取得し、呼び出します。 2. **プラットフォーム固有のコマンド構築**:

  • **NetWare環境**:

    • 1. リクエスト設定から netware_dir_config を取得します。 2. r->filename の拡張子を抽出します。 3. 拡張子をキーに d->file_type_handlers からハンドラ（NLM）を検索します。 4. ハンドラに基づき [ハンドラ] [ファイル名] 形式のコマンドを構築し、apr_tokenize_to_argv で引数配列に変換します。 5. ap_server_root_relative を使用して実行ファイルの絶対パスを解決します。

  • **Windows環境**:

    • 1. ファイル拡張子（.exe, .com, .bat, .cmd）を判定し、標準的な実行形式であればインタープリタを空に設定します。 2. 必要に応じて、設定（ScriptInterpreterSource）に基づきレジストリからインタープリタを検索します。 3. レジストリで特定できない場合、ファイル先頭のシバン（#!）行を解析してインタープリタを抽出します（UTF-8 BOMはスキップ）。 4. split_argv を呼び出し、引数テンプレートを実際のパスで展開して配列化します。 5. prep_string を使用して環境変数をWindows向けに調整し、e_info->detached を 1 に設定します。 3. **返却**: 構築されたコマンドパス（cmd）と引数配列（argv）を呼び出し元に返し、成功時は APR_SUCCESS を、失敗時は APR_EBADF などのステータスコードを返します。

重要な不変条件

プラットフォームを跨いだ共通のグローバルな不変条件の全貌は、提供されたコンテキストからは未だ不明ですが、各プラットフォームの実装において以下の条件が課されています。

  • **メモリ管理**: すべての動的メモリ確保は、引数として渡される apr_pool_t（p）に依存しなければなりません。
  • **NetWare環境**:
  • netware_dir_config がリクエストのディレクトリ設定から取得可能である必要があります。
  • e_info->process_cgi が真の場合、r->filename をベースにコマンドが初期化されます。
  • **Windows環境**:
  • e_info->cmd_type が APR_SHELLCMD でない場合、インタープリタが特定されれば APR_PROGRAM_PATH に設定される必要があります。

よくある落とし穴

  • **NetWare環境**:
  • 拡張子に対応するハンドラが file_type_handlers に登録されていない場合、APR_EBADF を返して処理が失敗します。
  • **Windows環境**:
  • インタープリタが特定できず、シバン行も存在しない場合は APR_EBADF で失敗します。
  • prep_string によるUTF-8からWindowsローカルコードページへの変換は限定的な実装に留まっています。
  • split_argv 内の引数解析は複雑なエスケープ処理を伴うため、Windowsのコマンドライン引数ルールに注意する必要があります。
