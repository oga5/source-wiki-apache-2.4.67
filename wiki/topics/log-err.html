log_err

概念

log_err は、Apache HTTP Serverプロジェクト（httpd-2.4.67）内の異なるコンテキストにおいて、エラー情報を記録するために定義されている静的（static）関数です。具体的には、特権実行ヘルパーである suexec と、テストクライアントである h2ws の2つの異なるプログラムで、それぞれ独自のエラーログ出力処理をカプセル化するために同名の関数が定義されています。

なぜ重要か

エラーの発生箇所や原因を正確に記録することは、システムのセキュリティ監査やデバッグにおいて極めて重要です。

  • **suexec における重要性**: セキュリティ検証の失敗（UID/GID不一致、不正なコマンド実行、メモリ確保失敗など）をシステムログや指定ファイルに記録し、不正アクセスの検知や監査を可能にします。
  • **h2ws における重要性**: テスト実行中のエラーを標準出力やログ出力先に一貫した形式で記録し、テストの失敗原因の特定を容易にします。

関与するモジュール

  • module:support
  • module:test/clients

主要エンティティ

  • symbol:support/suexec.c#log_err: suexec 内でエラーを記録する可変長引数対応の関数。内部で err_output を呼び出します。
  • symbol:test/clients/h2ws.c#log_err: h2ws 内でエラー箇所とメッセージを記録するラッパー関数。内部で log_out を呼び出します。

エンドツーエンドの流れ

これら2つの関数は独立したプログラムに存在するため、統一された単一のエンドツーエンドの流れは存在しません。それぞれの流れは以下の通りです。

1. suexec におけるエラー記録の流れ

1. suexec.c 内の各検証ロジック（メモリ確保失敗時、UID/GID不一致時、引数不足時、不正なコマンド実行時など）でエラーを検知します。 2. log_err が呼び出され、va_list を用いて可変長引数を受け取ります。 3. 内部で err_output(1, fmt, ap) を呼び出し、エラーフラグ（1）を立てて実際のログ出力処理（syslog または指定ファイルへの書き込み）を委譲します。

2. h2ws におけるエラー記録の流れ

1. テストクライアント実行中にエラーが発生します（ただし、提供されたソースコード内には本関数を直接呼び出している箇所は明示されていません）。 2. log_err(where, msg) が呼び出されます。 3. 引数として受け取った where と msg を、固定のラベル "ERROR" と共に log_out 関数へ渡し、ログ出力を委譲します。

重要な不変条件

  • **suexec のコンパイル時条件**:
  • コンパイル時に AP_LOG_EXEC マクロが定義されている場合のみ、実際のログ出力処理が実行されます。定義されていない場合、この関数は何も行わず即座に復帰します。
  • 可変長引数は stdarg.h の規約に従う必要があります。
  • **h2ws の引数条件**:
  • 引数 where および msg は、有効なヌル終端文字列である必要があります。

よくある落とし穴

  • **同名関数の混同**: support/suexec.c の log_err と test/clients/h2ws.c の log_err は、名前は同じですがシグネチャ（引数の数や型）も目的も全く異なる独立した関数です。これらを混同しないよう注意が必要です。
  • **suexec でのログ無効化**: AP_LOG_EXEC が未定義の状態でコンパイルされると、suexec の log_err は何も行わずに復帰するため、エラーログが一切記録されなくなります。
  • **出力ストリームの閉塞**: h2ws の log_err が依存する log_out の出力先ストリームがクローズされている場合、ログ出力が失敗する可能性があります。
