log_err

概念

log_err は、Apache HTTP Server（httpd-2.4.67）の異なるコンポーネントにおいて、エラー情報を記録するために定義されているローカルなヘルパー関数です。具体的には、suexec ユーティリティにおけるセキュリティ監査用のエラーログ出力と、h2ws テストクライアントにおけるエラーメッセージ出力という、2つの独立した実装が存在します。

なぜ重要か

エラーの発生箇所や原因を正確に記録することは、システムのセキュリティ監査やデバッグにおいて極めて重要です。

  • **support/suexec.c の log_err**: 権限昇格を伴う suexec の実行時エラー（UID/GID不一致、不正コマンド、メモリ確保失敗など）を記録し、セキュリティ監査の信頼性を担保します。
  • **test/clients/h2ws.c の log_err**: テストクライアント実行時のエラーを整理された形式で出力し、問題の特定を容易にします。

関与するモジュール

  • module:support
  • module:test/clients

主要エンティティ

  • **symbol:support/suexec.c#log_err**: suexec 内でエラー（is_error = 1）を記録するための可変長引数関数。内部で err_output を呼び出します。
  • **symbol:test/clients/h2ws.c#log_err**: h2ws クライアント内でエラー箇所とメッセージを記録するための関数。内部で log_out を呼び出します。

エンドツーエンドの流れ

これら2つの関数は独立して動作しており、システム全体を通じた単一のエンドツーエンドのフローは存在しません。それぞれのコンテキストにおける処理の流れは以下の通りです。

1. suexec におけるエラー記録の流れ

1. suexec 内の各検証ロジック（UID/GIDの不一致や不正なコマンド実行など）でエラーを検知します。 2. log_err(fmt, ...) が呼び出されます。 3. コンパイル時に AP_LOG_EXEC マクロが定義されている場合、可変長引数を va_list として処理し、err_output(1, fmt, ap) へ委譲してシステムログ（syslog）または指定されたログファイルへ出力します。

2. h2ws におけるエラー記録の流れ

1. テストクライアント実行中にエラーが発生します。 2. エラーが発生した箇所（where）とメッセージ（msg）を指定して log_err(where, msg) が呼び出されます。 3. 固定ラベル "ERROR" を付与して log_out 関数へ処理を委譲し、標準出力または指定のログ出力先へ記録します。

重要な不変条件

  • **support/suexec.c**:
  • コンパイル時に AP_LOG_EXEC マクロが定義されている場合のみ、実際のログ出力処理が実行されます。定義されていない場合、この関数は何も行わずに即座に復帰します。
  • 可変長引数は stdarg.h の規約に従う必要があります。
  • **test/clients/h2ws.c**:
  • 引数 where および msg は、有効なヌル終端文字列でなければなりません。

よくある落とし穴

  • **同名関数の混同**: support/suexec.c の log_err と test/clients/h2ws.c の log_err は、名前は同じですがシグネチャ（引数の構成）も目的も全く異なる独立した関数です。ソースコードの静的解析や修正の際には、これらを混同しないよう注意が必要です。
  • **マクロ定義への依存**: suexec の log_err は AP_LOG_EXEC マクロの定義状況に依存するため、ビルド設定によってはログが一切出力されない状態になります。
  • **出力ストリームの切断**: h2ws の log_err が依存する log_out の出力先ストリームがクローズされている場合、ログ出力が失敗する可能性があります。

次に読む場所

  • **support/suexec.c**: suexec におけるエラーログ出力の具体的な委譲先である err_output の実装を確認してください。
  • **test/clients/h2ws.c**: h2ws における実際のログ出力先やフォーマットを制御している log_out の実装を確認してください。
