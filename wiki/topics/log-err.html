log_err

概念

log_err は、Apache HTTP Server（httpd-2.4.67）の異なるコンポーネントにおいて、エラー情報を記録するために定義されている静的（static）な関数です。この関数は、セキュリティ監査に関わるシステム管理ユーティリティ（suexec）と、テスト用クライアント（h2ws）の双方で、それぞれ独立したエラーログ出力のインターフェースとして実装されています。

なぜ重要か

エラーログの記録は、システムのデバッグやセキュリティ監査において極めて重要です。

  • **suexec における重要性**: suexec は権限を変更してプログラムを実行するセキュリティ上重要なコンポーネントであり、メモリ確保失敗や不正なコマンド実行などのエラーを確実にシステムログ（syslog）や指定のログファイルに記録する必要があります。
  • **h2ws における重要性**: テスト用クライアントにおいて、エラーが発生した箇所（コンテキスト）とメッセージを整理して出力することで、テスト実行時の問題特定を容易にします。

関与するモジュール

  • module:support
  • module:test/clients

主要エンティティ

  • **symbol:support/suexec.c#log_err**: suexec 内で発生したエラーを err_output を介してシステムログ等に記録する可変長引数関数。
  • **symbol:test/clients/h2ws.c#log_err**: テストクライアント h2ws 内で、エラー発生箇所とメッセージを log_out を介して出力するラッパー関数。

エンドツーエンドの流れ

これら2つのモジュールは独立して動作しており、共通するエンドツーエンドのフローは存在しません。それぞれのモジュール内における処理の流れは以下の通りです。

1. support/suexec.c における流れ

1. suexec 内の検証ロジック（UID/GID不一致、引数不足、不正コマンド実行など）でエラーを検知します。 2. log_err(fmt, ...) が呼び出されます。 3. va_list を用いて可変長引数を処理し、err_output(1, fmt, ap) を呼び出して実際のログ出力処理を委譲します。

2. test/clients/h2ws.c における流れ

1. エラー発生時に log_err(where, msg) が呼び出されます（ただし、提供されたソースコード内での具体的な呼び出し元は明示されていません）。 2. 引数として受け取った where（発生箇所）と msg（メッセージ）を、固定ラベル "ERROR" と共に log_out 関数へ渡して出力を委譲します。

重要な不変条件

  • **support/suexec.c#log_err**:
  • コンパイル時に AP_LOG_EXEC マクロが定義されている場合のみ、実際のログ出力処理が実行されます。定義されていない場合、関数は何も行わずに即座に復帰します。
  • 可変長引数は stdarg.h の規約に従う必要があります。
  • **test/clients/h2ws.c#log_err**:
  • 引数 where および msg は、有効なヌル終端文字列でなければなりません。

よくある落とし穴

  • **同名関数の混同**: support/suexec.c の log_err と test/clients/h2ws.c の log_err は、名前は同じですがシグネチャ（引数の数や型）も目的も全く異なる独立した関数です。これらを混同しないよう注意が必要です。
  • **マクロ定義の依存**: suexec.c の log_err は AP_LOG_EXEC マクロが定義されていないと機能しないため、ビルド設定によってはログが出力されない落とし穴があります。
  • **出力ストリームの閉塞**: h2ws.c の log_err が依存する log_out の出力先ストリームがクローズされている場合、ログ出力が失敗する可能性があります。
