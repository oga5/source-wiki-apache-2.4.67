log_err

概念

log_err は、Apache HTTP Server（httpd-2.4.67）の異なるモジュールにおいて、エラー情報を記録するために定義されている静的（static）なヘルパー関数です。

本プロジェクト内では、セキュリティ実行ラッパーである suexec と、テスト用クライアントである h2ws の2箇所で同名の関数が定義されています。これらは名前こそ同じですが、それぞれ独立した目的、シグネチャ、および振る舞いを持つ異なる関数です。

なぜ重要か

エラーの発生箇所や詳細を適切に記録することは、システムのセキュリティ監査やデバッグにおいて極めて重要です。

  • **suexec における重要性**: 権限昇格を伴う実行時の検証失敗（UID/GIDの不一致、引数不足、不正なコマンド実行など）をシステムログ（syslog）や指定のログファイルに記録し、セキュリティ監査トレイルを提供します。
  • **h2ws における重要性**: テストクライアントの実行中に発生したエラーを標準出力やログ出力先に記録し、テスト失敗時の原因特定を容易にします。

関与するモジュール

  • module:support
  • module:test/clients

主要エンティティ

  • **symbol:support/suexec.c#log_err**:

    • suexec プログラム内でエラーを記録する関数。可変長引数を受け取り、内部で err_output を呼び出します。

  • **symbol:test/clients/h2ws.c#log_err**:

    • h2ws クライアント内でエラーメッセージを記録するためのラッパー関数。エラー発生箇所（where）とメッセージ（msg）を受け取り、内部で log_out を呼び出します。

エンドツーエンドの流れ

各コンポーネントにおけるエラー記録の流れは以下の通りです。

1. suexec におけるエラー記録

1. support/suexec.c 内の各検証ロジック（メモリ確保失敗、UID/GID不一致、不正なコマンド実行など）でエラーが検出されます。 2. log_err(fmt, ...) が呼び出されます。 3. log_err は va_list を用いて可変長引数を処理し、err_output(1, fmt, ap) を呼び出して実際のログ出力処理を委譲します。 4. ※ err_output の内部実装、およびシステムログやログファイルへの最終的な書き込みに至る詳細なエンドツーエンドの流れは、提供されたコンテキストからは未解明です。

2. h2ws におけるエラー記録

1. test/clients/h2ws.c 内でエラーが発生します（具体的な呼び出し元は提供されたソースコードからは明示されていません）。 2. log_err(where, msg) が呼び出されます。 3. log_err は、固定のラベル "ERROR" とともに where および msg を log_out 関数へ渡します。 4. log_out を呼び出すことで、実際のログ出力処理を委譲します。 5. ※ log_out の内部実装、および標準出力やログ出力先への最終的な書き込みに至る詳細なエンドツーエンドの流れは、提供されたコンテキストからは未解明です。

重要な不変条件

  • **support/suexec.c#log_err**:
  • コンパイル時に AP_LOG_EXEC マクロが定義されている場合のみ、実際のログ出力処理が実行されます。定義されていない場合、この関数は何も行わずに即座に復帰します。
  • 可変長引数は stdarg.h の規約に従う必要があります。
  • **test/clients/h2ws.c#log_err**:
  • 引数 where および msg は、有効なヌル終端文字列でなければなりません。

よくある落とし穴

  • **同名関数の混同**: support/suexec.c と test/clients/h2ws.c に存在する log_err は、シグネチャも目的も異なる全く別の関数です。これらを混同しないよう注意が必要です。
  • **suexec でのログ未出力**: AP_LOG_EXEC マクロが定義されずにコンパイルされた場合、suexec の log_err は実質的に機能しません。エラーが発生してもログが記録されない状態になります。
  • **類似関数との混同（suexec）**: suexec にはエラー（is_error = 1）として記録する log_err のほかに、通常のトランザクション記録を行う log_no_err が存在します。監査目的が異なるため、混同しないよう注意してください。
  • **出力ストリームのクローズ（h2ws）**: h2ws の log_err が依存する log_out において、出力先ストリームがクローズされている場合、ログ出力が失敗する可能性があります。

次に読む場所

  • **support/suexec.c**: 実際のログ出力を行う err_output の実装、および通常のトランザクション記録を行う log_no_err の実装を確認してください。
  • **test/clients/h2ws.c**: ログ出力を担当する log_out の実装、および log_err が実際に呼び出されている箇所を確認してください。
