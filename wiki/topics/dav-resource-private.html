dav_resource_private

概念

dav_resource_private は、Apache HTTP Server の WebDAV モジュール（mod_dav）において、各 WebDAV リソース（dav_resource）に関連付けられたプロバイダー固有のプライベート情報を保持するための構造体です。

mod_dav コアのヘッダー（mod_dav.h）上では不透明な（opaque）構造体として前方宣言されており、具体的な実装（例えばファイルシステムプロバイダーである mod_dav_fs の repos.c）においてその詳細なメンバが定義されます。

なぜ重要か

この構造体は、mod_dav コアと具体的なリポジトリプロバイダー（mod_dav_fs など）との間のカプセル化（疎結合）を実現するために重要です。

mod_dav コアはリソースのプライベートな詳細を直接知る必要がなく、プロバイダー側が dav_resource->info を介して自身に必要なコンテキスト情報（ファイルシステム上の絶対パスやリクエストプールなど）を安全に保持・参照できます。

関与するモジュール

  • module:modules/dav/main (mod_dav コア)
  • module:modules/dav/fs (mod_dav_fs プロバイダー)

主要エンティティ

  • **symbol:modules/dav/main/mod_dav.h#dav_resource_private**

    • 不透明な構造体としての前方宣言。 ``c typedef struct dav_resource_private dav_resource_private; ``

  • **symbol:modules/dav/fs/repos.c#dav_resource_private**

    • mod_dav_fs における具体的な構造体定義。以下のメンバを含みます。

  • apr_pool_t *pool: リクエストに関連付けられたメモリプール。
  • const char *pathname: リソースのフルパス。
  • apr_finfo_t finfo: ファイルシステム情報。
  • request_rec *r: Apache リクエストレコード。

エンドツーエンドの流れ

1. **リソースの生成**: リクエスト処理中、プロバイダー（mod_dav_fs の dav_fs_get_resource 相当の処理など）が dav_resource を生成する際、dav_resource_private のメモリを割り当てて初期化し、dav_resource->info に設定します。 2. **プロバイダー内での利用**: mod_dav_fs の各関数（ファイルコピー、移動、プロパティ管理など）は、受け取った dav_resource->info を dav_resource_private* にキャストして、ファイルパス（pathname）やリクエスト情報（r）を取得し、実際のファイルシステム操作を行います。 3. **コアとの連携**: mod_dav コアは、この構造体の内部を直接操作せず、単に dav_resource の一部として引き回します。

※なお、プロバイダーの初期化からリクエスト終了に伴うメモリ解放に至る完全なライフサイクルや、他プロバイダーにおける詳細なエンドツーエンドの流れは、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **生存期間の同期**: pool は、リクエストの生存期間と同期した apr_pool_t である必要があります。また、本構造体自体も dav_resource のライフサイクルと同期して管理される必要があります。
  • **有効なリクエスト参照**: r は、常に有効な Apache request_rec を指している必要があります。

よくある落とし穴

  • **NULLポインタアクセス**: dav_resource->info が NULL の状態でプロバイダー固有のデータにアクセスしようとすると、セグメンテーション違反が発生します。
  • **不正なキャスト**: mod_dav コアは不透明型として扱うため、異なるプロバイダー間で誤ってキャストを行うと、未定義の動作を引き起こす可能性があります。
  • **メモリの有効期限**: pathname などの文字列は pool から割り当てられているため、リクエスト終了に伴うプール破棄の後にアクセスすると未定義の動作になります。

次に読む場所

  • modules/dav/main/mod_dav.h: 不透明構造体としての宣言と、親となる dav_resource の定義を確認できます。
  • modules/dav/fs/repos.c: mod_dav_fs における具体的な構造体定義と、それを利用したファイルシステム操作の実装を確認できます。
