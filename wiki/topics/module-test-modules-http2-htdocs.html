Module: test/modules/http2/htdocs

概念

test/modules/http2/htdocs は、Apache HTTP Server（httpd）の HTTP/2 モジュール（mod_http2）における統合テスト用のドキュメントルートディレクトリです。このディレクトリ、特にその配下の cgi サブディレクトリには、HTTP/2 経由でのリクエスト処理、ヘッダー操作、環境変数の伝播、およびファイルアップロードなどの挙動を検証するためのテスト用 CGI スクリプト（Python 製）が配置されています。

なぜ重要か

HTTP/2 プロトコルは、従来の HTTP/1.1 と比較してフレーム構造やマルチプレクシング、ヘッダー圧縮などの仕組みが大きく異なります。このディレクトリ内の CGI スクリプト群は、mod_http2 がリクエストを正しく解釈し、バックエンドの CGI プロセスに対して正確にデータを引き渡せているか、また CGI からのレスポンスを正しく HTTP/2 フレームに変換してクライアントに返せているかを自動テストで検証するために不可欠なエンドポイントを提供します。

関与するモジュール

  • module:test/modules/http2/htdocs
  • module:test/modules/http2/htdocs/cgi

主要エンティティ

test/modules/http2/htdocs/cgi 配下に存在する以下の Python スクリプト群が、テスト時の主要なモック・検証用エンドポイントとして機能します。

  • **hello.py**: シンプルな応答を返す、基本的な疎通確認用の CGI。
  • **echo.py / necho.py**: クライアントから送信されたリクエストボディをそのまま（または特定のフォーマットで）エコーバックし、データ整合性を検証する CGI。
  • **echohd.py**: 受信したリクエストヘッダーを解析して応答に含めることで、ヘッダーの伝播を検証する CGI。
  • **env.py**: CGI 実行環境に渡された環境変数を一覧出力し、HTTP/2 固有のメタデータやサーバー変数が正しく設定されているかを検証する CGI。
  • **hecho.py**: カスタムヘッダーの送受信や特殊なヘッダー操作を検証するための CGI。
  • **upload.py**: HTTP/2 ストリーム経由でのファイルアップロードおよびマルチパートデータのハンドリングを検証する CGI。
  • **requestparser.py**: リクエストの構造をパースし、テストアサーション用のメタデータを提供するヘルパー。
  • **mnot164.py**: 特定のテストケース（RFC 関連の特殊な挙動など）を検証するための専用 CGI。

エンドツーエンドの流れ

1. **テストの起動**: テストランナー（pytest など）が、HTTP/2 経由で Apache サーバーに対して特定のリクエストを送信します。 2. **リクエストの受信**: mod_http2 がリクエストを受信し、ストリームを処理して、対象の CGI スクリプト（例: upload.py や echo.py）へのルーティングを行います。 3. **CGI の実行**: Apache が Python インタープリタを介して test/modules/http2/htdocs/cgi/ 内のスクリプトを実行し、標準入力や環境変数を経由してリクエストデータを渡します。 4. **レスポンスの返却**: CGI スクリプトが処理結果を標準出力に書き出し、mod_http2 がそれを HTTP/2 レスポンスフレームに変換してテストクライアントに返却します。

*注意: テストフレームワーク（pytest 側）の具体的なアサーションロジックや、テスト実行時の詳細な Apache 設定との完全な連携フローについては、提供されたファイル群の範囲外であるため現時点では未検証です。*

重要な不変条件

  • **環境変数の整合性**: env.py などの実行時、HTTP/2 接続特有の環境変数（HTTP2 や H2PUSH など）が Apache から CGI 側へ正しく伝播されていなければなりません。
  • **データの同一性**: echo.py や upload.py を通じたデータの送受信において、HTTP/2 のフレーム境界やストリーム制御に影響されることなく、送信データと受信データが完全に一致する必要があります。

*注意: 各 CGI スクリプトが内部で要求する具体的な入力フォーマットや、テストアサーションにおける厳密な期待値（不変条件）の全容は、現時点では完全には特定されていません。*

よくある落とし穴

  • **Python 環境への依存**: これらのテスト用スクリプトは Python で記述されているため、テストを実行するホスト環境に適切な Python インタープリタがインストールされており、かつ CGI としての実行権限（実行パーミッション）が正しく付与されている必要があります。
  • **バッファリングとタイムアウト**: HTTP/2 のストリーム制御やウィンドウサイズの設定によっては、CGI スクリプト側でのバッファリングや出力遅延が原因でテストがハングアップしたり、タイムアウトが発生したりする可能性があります。
