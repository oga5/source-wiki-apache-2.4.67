kept_body_filter

概念

kept_body_filter は、Apache HTTP Server（httpd-2.4.67）の mod_request モジュールにおいて、r->kept_body に保存されたリクエストボディをサブリクエストの入力ストリームとして再挿入するための入力フィルタです。これにより、一度読み込まれたリクエストボディを後続の処理やサブリクエストで再利用することが可能になります。

なぜ重要か

通常のHTTPリクエスト処理において、リクエストボディは一度読み取ると消費されてしまい、サブリクエストなどで再利用することができません。kept_body_filter を使用することで、保存されたリクエストボディ（r->kept_body）をサブリクエストの入力ストリームとして透過的に再挿入できます。また、再挿入時に Transfer-Encoding を除去して Content-Length を再設定するため、サブリクエスト側はチャンク転送などの元の転送符号化を意識せずにボディデータを再読み込みできるようになります。

  • symbol:modules/filters/mod_request.c#kept_body_filter_init: サブリクエスト開始時に呼び出され、保持データのメタデータをヘッダー情報へ反映させる初期化関数。

エンドツーエンドの流れ

1. **初期化フェーズ (kept_body_filter_init)** サブリクエストの開始時に呼び出されます。f->r->kept_body が存在する場合、r->headers_in から Transfer-Encoding ヘッダーを削除し、apr_brigade_length で計算した保持ボディの総バイト数を Content-Length ヘッダーに設定します。 2. **フィルタ実行フェーズ (kept_body_filter)** フィルタチェーンの実行エンジン（ap_get_brigade 経由）から呼び出されます。

  • kept_body が存在しない、または読み込みモードが AP_MODE_READBYTES や AP_MODE_GETLINE 以外の場合は、後続のフィルタへ処理を委譲します。
  • 初回呼び出し時に f->ctx（kept_body_ctx_t）を初期化し、残量 ctx->remaining に総長を設定します。
  • ctx->remaining が 0 以下の場合は、後続のフィルタへ処理を委譲します。
  • 要求された readbytes に応じて kept_body からバケットを特定（apr_brigade_partition）し、コピーして出力用ブリゲード b に挿入します。
  • ctx->offset と ctx->remaining を更新して APR_SUCCESS を返します。

*注意: フィルタチェーンへの登録や、r->kept_body にデータが保存される具体的なトリガーなど、システム全体における完全なエンドツーエンドのフローの詳細は、提供された情報からは完全には明らかになっていません。*

重要な不変条件

  • **データの有効性**: r->kept_body はリクエストのライフサイクル（r->pool）に紐付いており、サブリクエストが終了するまで有効であることが保証されています。
  • **ヘッダーの整合性**: kept_body_filter_init により、Transfer-Encoding が除去され、Content-Length が保持データの正確な長さに固定されます。
  • **フィルタコンテキストの維持**: f->ctx は初回呼び出し時に r->pool から割り当てられ、以降の呼び出しで状態（offset, remaining）を追跡するために再利用されます。

よくある落とし穴

  • **バケットコピーの失敗**: apr_bucket_copy が失敗した場合、apr_bucket_read による強制読み込みを試みるフォールバック処理が行われますが、現在の実装では最小限のフォールバックに留まっています。
  • **ヘッダー書き換えの影響**: サブリクエストの初期化フェーズで Transfer-Encoding や Content-Length が書き換えられるため、後続のフィルタやアプリケーションの挙動に影響を与える可能性があります。
  • **巨大なボディの長さ計算**: apr_brigade_length の計算結果が apr_off_t の範囲を超える場合の明示的なエラーハンドリングは実装されていません。
