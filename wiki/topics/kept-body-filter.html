kept_body_filter

概念

kept_body_filter は、Apache HTTP Server（mod_request）において、事前に保存されたリクエストボディ（r->kept_body）をサブリクエストなどの入力ストリームとして再挿入するための入力フィルタです。これにより、一度読み取られたリクエストボディを後続の処理やサブリクエストで再利用することが可能になります。

なぜ重要か

HTTPリクエストボディは通常、一度しか読み取ることができません。しかし、サブリクエストの実行や特定のフィルタ処理において、同じリクエストボディを再度読み込ませたい場合があります。kept_body_filter は、保存されたボディデータをバケットブリゲード経由で再注入し、さらに Transfer-Encoding ヘッダーの削除や Content-Length ヘッダーの再設定を行うことで、後続の処理がチャンク転送などを意識せずに透過的にデータを再利用できるようにします。

関与するモジュール

  • module:include
  • module:modules/filters

主要エンティティ

  • symbol:include/mod_request.h#KEPT_BODY_FILTER : フィルタの識別子。
  • symbol:modules/filters/mod_request.c#kept_body_filter : 保存されたリクエストボディを読み出して出力ブリゲードにコピーする、入力フィルタの本体関数。
  • symbol:modules/filters/mod_request.c#kept_body_filter_init : サブリクエスト開始時に、保持されたボディのメタデータをヘッダー情報（Content-Length など）に反映させる初期化関数。

エンドツーエンドの流れ

1. **初期化フェーズ**: サブリクエストの開始時などに kept_body_filter_init が呼び出されます。f->r->kept_body が存在する場合、入力ヘッダー（r->headers_in）から Transfer-Encoding を削除し、保持されているボディの総バイト数を計算して Content-Length ヘッダーに設定します。 2. **データ読み出しフェーズ**: フィルタチェーンの実行エンジンから ap_get_brigade 経由で kept_body_filter が呼び出されます。 3. **データの抽出とコピー**:

  • 初回呼び出し時に、フィルタコンテキスト f->ctx（kept_body_ctx_t）を初期化し、残量（ctx->remaining）を設定します。
  • 要求された読み込みバイト数（readbytes）に応じて、apr_brigade_partition を用いて kept_body から該当範囲のバケットを特定し、コピーして出力用ブリゲード b に挿入します。
  • 読み出しに伴い、ctx->offset と ctx->remaining を更新します。

    • 4. **フォールスルー**: 保持されたデータがない場合、またはすべてのデータを読み出し終えた（ctx->remaining <= 0）後は、後続のフィルタへ処理を委譲します。

※なお、r->kept_body にデータが保存される具体的なトリガーや、フィルタチェーン全体における厳密な登録・実行タイミングの全容など、詳細なエンドツーエンドのフローは提供された情報からは完全には判明していません。

重要な不変条件

  • **データの有効性**: r->kept_body はリクエストのライフサイクル（r->pool）に紐付いており、サブリクエストが終了するまで有効であることが保証されます。
  • **フィルタの適用条件**: r->kept_body が存在しない場合、または読み込みモードが AP_MODE_READBYTES もしくは AP_MODE_GETLINE 以外の場合は、データのコピー処理は行われず、処理は即座に後続のフィルタへ委譲されます。
  • **ヘッダーの整合性**: kept_body_filter_init の実行により、Transfer-Encoding が除去され、Content-Length が固定値に置き換わることで、サブリクエスト側は一貫した長さのボディとしてデータを処理できます。

よくある落とし穴

  • **バケットコピーの失敗**: apr_bucket_copy が失敗した場合、apr_bucket_read による強制読み込みを試みるフォールバック処理が存在しますが、現在の実装では最小限の対応に留まっているため、特殊なバケットタイプを扱う際には注意が必要です。
  • **巨大なボディの長さ計算**: kept_body_filter_init 内で apr_brigade_length を用いてボディの総バイト数を計算する際、計算結果が apr_off_t の範囲を超える場合の明示的なエラーハンドリングは実装されていません。
  • **ヘッダー書き換えの影響**: 初期化フェーズで Transfer-Encoding や Content-Length が書き換えられるため、後続のフィルタやアプリケーションがオリジナルのヘッダーに依存している場合に予期しない挙動を引き起こす可能性があります。
