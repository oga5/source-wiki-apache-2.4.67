server_rec

概念

server_rec は、Apache HTTP Server において、メインサーバーおよび各バーチャルホスト（仮想ホスト）の設定情報、状態、および実行時情報を保持するための中心的なデータ構造（構造体および typedef）です。サーバーのライフサイクル全体を通じて、リクエストや接続がどのバーチャルホスト設定に属するかを特定・管理するために使用されます。

なぜ重要か

  • **仮想ホスト設定の基点**: サーバー名、ポート、タイムアウト設定、スキーム、コンテキスト情報など、仮想ホストごとの設定を保持し、各モジュールがこれらにアクセスするための基点となります。
  • **モジュールデータの格納**: モジュールがサーバー単位で保持するデータ（module_config）の格納場所として機能します。
  • **リクエストルーティング**: 接続時やリクエスト処理時に、適切なバーチャルホスト設定（server_rec）を特定・割り当てるために不可欠です。
  • **コアシステムへの影響**: 本構造体の変更は、モジュールマジックナンバー（MMN）の更新を伴うほど、サーバーの根幹に関わる重要な位置を占めています。

関与するモジュール

  • module:include
  • module:modules/loggers

主要エンティティ

  • symbol:include/httpd.h#server_rec: 構造体の実体定義（struct server_rec）。サーバーの構成情報や状態を保持します。
  • symbol:include/http_vhost.h#server_rec: バーチャルホストの初期化、検索、およびリクエストへの割り当てに関連する参照。
  • symbol:modules/loggers/mod_log_config.h#server_rec: ログモジュール（mod_log_config）におけるサーバー設定やログ出力先設定の参照。

エンドツーエンドの流れ

提供された情報に基づく、server_rec を介した処理の流れは以下の通りです。

1. **初期化と設定読み込み**: サーバー起動時に ap_init_vhost_config で初期化され、設定ファイル（httpd.conf 等）の内容に基づいて各仮想ホストに対応する server_rec インスタンスが生成されます。 2. **検索テーブルの構築**: 設定読み込み後、ap_fini_vhost_config によって実行時の高速検索用テーブルが構築されます。 3. **接続時の特定**: クライアントからの接続時、ap_update_vhost_given_ip により、接続の IP アドレスとポートからベースとなる server_rec が特定されます。 4. **リクエスト時の切り替え**: リクエスト処理時、ap_update_vhost_from_headers が HTTP の Host ヘッダーを解析し、名前ベースのバーチャルホストに対応する server_rec へ r->server（request_rec 内）を切り替えます。 5. **モジュールによる参照**: mod_log_config などのモジュールが、割り当てられた server_rec の設定を参照して、ログ出力先やフォーマットの決定などの処理を行います。

*注意: MPM（マルチプロセッシングモジュール）のフック関数や、詳細な内部マッチングアルゴリズムにおける完全なデータ伝播の全容は、提供された情報からは一部未確定です。*

重要な不変条件

  • **生存期間**: サーバーのライフサイクル全体を通じて生存するオブジェクトです。
  • **構成の確定**: open_logs フック等の初期化フェーズにおいて、サーバー構成が確定している必要があります。
  • **初期化手順**: サーバー起動時に ap_init_vhost_config で初期化され、設定読み込み後に ap_fini_vhost_config によって実行時の高速検索用テーブルが構築されていなければなりません。

よくある落とし穴

  • **設定の不整合**: 設定ファイルに不整合がある場合、ap_fini_vhost_config によるテーブル構築時にエラーが発生したり、不完全な状態で初期化されたりする可能性があります。
  • **Hostヘッダーの欠如**: リクエスト時に適切な Host ヘッダーが存在しない場合、デフォルトのバーチャルホスト設定が適用されます。
  • **構造体の巨大さとMMN**: server_rec は非常に巨大な構造体であり、メンバーの追加や変更は include/ap_mmn.h のモジュールマジックナンバー（MMN）の更新を伴います。モジュール開発者は必要なフィールドのみにアクセスし、不必要な変更を避けることが推奨されます。
  • **スレッド同期**: 複数のスレッドから同時にアクセスされる場合、構成情報の変更には適切な同期が必要です。

次に読む場所

  • include/httpd.h: server_rec の主要な構造体定義（struct server_rec）が存在します。
  • include/http_vhost.h: バーチャルホストの初期化や検索に関するAPI定義。
  • modules/loggers/mod_log_config.h: ログモジュールにおける server_rec の利用例。
  • server/vhost.c: ハッシュテーブルの構造やマッチングアルゴリズムの実装詳細（ソースコード内の参照先）。
