server_rec

概念

server_rec は、Apache HTTP Server において、メインサーバーおよび各バーチャルホスト（仮想ホスト）の設定情報、状態、および実行時の構成情報を保持するための中心的なデータ構造です。

なぜ重要か

サーバーのライフサイクル全体を通じて、リクエストや接続がどのバーチャルホスト設定に属するかを特定・管理するために不可欠です。タイムアウト設定、サーバー名、スキーム、コンテキスト情報、およびモジュールごとの設定（module_config）などを保持し、リクエスト処理やログ出力（mod_log_config など）の各フェーズで参照される基点となります。

関与するモジュール

主要エンティティ

  • **symbol:include/httpd.h#server_rec**

    • 構造体の本体定義（include/httpd.h:818）。仮想ホストごとの設定情報やサーバーの状態を保持するコアデータ構造です。

  • **symbol:include/http_vhost.h#server_rec**

    • バーチャルホスト設定の特定や、実行時の高速検索用テーブル構築に関連する定義です。

  • **symbol:modules/loggers/mod_log_config.h#server_rec**

    • ログモジュール（mod_log_config）における参照用の定義です。ログ出力先の設定やフォーマットの決定において、この構造体が保持するサーバー設定が参照されます。

エンドツーエンドの流れ

提供された情報に基づく、サーバー起動からリクエスト処理、ログ出力に至る流れは以下の通りです。

1. **初期化と設定読み込み**: サーバー起動時に ap_init_vhost_config で初期化され、設定ファイル（httpd.conf 等）に基づいて各仮想ホストに対応する server_rec インスタンスが生成されます。設定読み込み後、ap_fini_vhost_config によって実行時の高速検索用テーブルが構築されます。 2. **接続時**: クライアントからの接続時、ap_update_vhost_given_ip により、接続の IP アドレスとポートからベースとなる server_rec が特定されます。 3. **リクエスト処理時**: ap_update_vhost_from_headers が HTTP の Host ヘッダーを解析し、名前ベースのバーチャルホストに対応する server_rec へ r->server（request_rec 内）を切り替えます。 4. **ログ出力時**: mod_log_config などのログモジュールにおいて、server_rec が保持するサーバー設定を参照してログ出力先やフォーマットを決定します。

*注意: 詳細な内部フロー（ハッシュテーブルの完全なマッチングアルゴリズムや、すべてのフックにおける具体的な伝播など）の全容は、提供された情報からは完全には判明していません。*

重要な不変条件

  • **生存期間**: サーバーのライフサイクル全体を通じて生存するオブジェクトです。
  • **初期化の確定**: open_logs フック等の初期化フェーズにおいて、サーバー構成が確定している必要があります。
  • **設定の整合性**: サーバー起動時に ap_fini_vhost_config によるテーブル構築が正常に完了している必要があります。

よくある落とし穴

  • **設定の不整合**: 設定ファイルに不整合がある場合、起動時のテーブル構築（ap_fini_vhost_config）でエラーが発生する可能性があります。
  • **Hostヘッダーの欠如**: リクエスト時に適切な Host ヘッダーが存在しない場合、デフォルトのバーチャルホスト設定が適用されます。
  • **構造体の変更とMMN**: server_rec の構造変更（メンバーの追加など）は、include/ap_mmn.h に定義されているモジュールマジックナンバー（MMN）の更新を伴う場合があります。
  • **直接アクセスの制限**: 非常に巨大な構造体であるため、モジュール開発者は必要なフィールドのみにアクセスし、不必要な変更を避けることが推奨されます。また、複数スレッドから同時にアクセスされる場合、構成情報の変更には適切な同期が必要です。
