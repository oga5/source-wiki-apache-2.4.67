server_rec

概念

server_rec は、Apache HTTP Server においてメインサーバーおよび各バーチャルホスト（仮想ホスト）の設定情報、状態、および実行時情報を保持するための中心的なデータ構造です。

なぜ重要か

サーバーのライフサイクル全体を通じて、リクエストや接続がどのバーチャルホスト設定に属するかを特定し、適切にルーティングするために不可欠です。また、タイムアウト設定、サーバー名、スキーム、ログ出力先の設定、モジュールごとのサーバー単位の設定（module_config）などを保持する基点となります。

関与するモジュール

  • module:include (コアヘッダー群)
  • module:modules/loggers (ログ構成モジュール)

主要エンティティ

  • **symbol:include/httpd.h#server_rec**: 構造体の本体定義（include/httpd.h:818）。タイムアウトやサーバー名などの基本情報を保持します。
  • **symbol:include/http_vhost.h#server_rec**: バーチャルホスト設定としての参照と、関連する解決処理を定義します。
  • **symbol:modules/loggers/mod_log_config.h#server_rec**: ログモジュール（mod_log_config）におけるサーバー設定やログ出力先決定のための参照を定義します。

エンドツーエンドの流れ

1. **初期化**: サーバー起動時に設定ファイル（httpd.conf 等）の内容に基づいて server_rec インスタンスが生成・初期化されます。ap_init_vhost_config で初期化され、設定読み込み後に ap_fini_vhost_config によって実行時の高速検索用テーブルが構築されます。 2. **接続時**: クライアントからの接続時、ap_update_vhost_given_ip により、接続の IP アドレスとポートからベースとなる server_rec が特定されます。 3. **リクエスト処理時**: ap_update_vhost_from_headers が HTTP の Host ヘッダーを解析し、名前ベースのバーチャルホストに対応する server_rec へ r->server を切り替えます。 4. **ログ出力・終了**: リクエスト処理やログ出力（mod_log_config など）の際、該当する server_rec の設定（ログフォーマットや出力先など）が参照されます。

*注意: 提供された情報に基づく流れであり、詳細な内部関数呼び出しの全容やエラーハンドリングの全パスは完全には網羅されていません。*

重要な不変条件

  • **生存期間**: サーバーのライフサイクル全体を通じて生存するオブジェクトである必要があります。
  • **構成の確定**: open_logs フック等の初期化フェーズにおいて、サーバー構成（server_rec の内容）が確定している必要があります。
  • **インスタンスの分離**: 複数の仮想ホストが存在する場合、各ホストに対応する個別の server_rec インスタンスが生成され、適切に分離・管理される必要があります。

よくある落とし穴

  • **設定の不整合**: 設定ファイルに不整合がある場合、ap_fini_vhost_config によるテーブル構築時やサーバー起動時の設定解析時にエラーが発生し、不完全な状態で初期化される可能性があります。
  • **Hostヘッダーの欠如**: リクエスト時に適切な Host ヘッダーが存在しない場合、デフォルトのバーチャルホスト設定が適用されます。
  • **構造体の巨大さと互換性**: server_rec は非常に巨大な構造体であり、メンバーの追加（例: server_scheme や context）は include/ap_mmn.h に定義されるモジュールマジックナンバー（MMN）の更新を伴う場合があります。モジュール開発者は必要なフィールドのみにアクセスし、不必要な変更を避けることが推奨されます。
  • **マルチスレッド同期**: 複数のスレッドから同時にアクセスされる場合、構成情報の変更には適切な同期が必要です。

次に読む場所

  • **include/httpd.h**: 構造体の詳細なフィールド定義。
  • **include/http_vhost.h**: バーチャルホスト関連のAPI定義。
  • **modules/loggers/mod_log_config.h**: ログモジュールでの利用方法。
  • **server/vhost.c** (外部参照): バーチャルホストのマッチングアルゴリズムやハッシュテーブルの実装詳細。
