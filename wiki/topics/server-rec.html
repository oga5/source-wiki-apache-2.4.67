server_rec

概念

server_rec は、Apache HTTP Server においてメインサーバーおよび各バーチャルホスト（仮想ホスト）の設定情報、状態、および実行時情報を保持するための中心的なデータ構造です。

なぜ重要か

  • **バーチャルホストの特定**: サーバーのライフサイクル全体を通じて、リクエストや接続がどのバーチャルホスト設定に属するかを特定・管理するための基点となります。
  • **設定コンテキストの提供**: タイムアウト設定、サーバー名、ポート、スキーム、コンテキスト情報、およびモジュールごとのサーバー単位の設定（module_config）などを一元管理します。
  • **モジュール間連携**: ログモジュール（mod_log_config）をはじめとする各モジュールが、サーバー単位の設定やログ出力先、フォーマットを決定する際のリファレンスとして機能します。

関与するモジュール

  • module:include （include/httpd.h, include/http_vhost.h など）
  • module:modules/loggers （modules/loggers/mod_log_config.h）

主要エンティティ

  • **symbol:include/httpd.h#server_rec**: 構造体の実体定義（include/httpd.h:818）。仮想ホストごとの設定や状態を保持するコアデータ構造。
  • **symbol:include/http_vhost.h#server_rec**: バーチャルホスト設定の解決やルックアップに関連する定義。
  • **symbol:modules/loggers/mod_log_config.h#server_rec**: ログ設定などで参照される、サーバー構造体の前方宣言および型定義。

エンドツーエンドの流れ

提供された情報に基づく、接続からリクエスト処理、ログ記録に至る大まかな流れは以下の通りです。

1. **初期化**: サーバー起動時に設定ファイル（httpd.conf 等）が読み込まれ、ap_init_vhost_config によって初期化されます。設定読み込み完了後、ap_fini_vhost_config により実行時の高速検索用テーブルが構築されます。 2. **接続時**: クライアントからの接続が発生すると、ap_update_vhost_given_ip により、接続の IP アドレスとポートに基づいてベースとなる server_rec が特定されます。 3. **リクエスト処理時**: リクエストの解析中、ap_update_vhost_from_headers が HTTP の Host ヘッダーを解析し、名前ベースのバーチャルホストに対応する server_rec へ r->server を切り替えます。 4. **ログ出力・処理**: mod_log_config などのモジュールが、server_rec に保持されたサーバー設定を参照してログ出力先やフォーマットを決定し、処理を行います。

*注意: MPM（マルチプロセッシングモジュール）のフックや、詳細な内部マッチングアルゴリズムにおける具体的なエンドツーエンドの全伝播フローの完全な詳細は、提供された情報からは一部未確定です。*

重要な不変条件

  • **生存期間**: サーバーのライフサイクル全体を通じて生存するオブジェクトです。
  • **初期化のタイミング**: open_logs フックなどの初期化フェーズにおいて、サーバー構成が確定している必要があります。
  • **設定の整合性**: サーバー起動時の設定解析エラーや不整合がある場合、ap_fini_vhost_config によるテーブル構築時にエラーが発生する可能性があります。

よくある落とし穴

  • **MMN（Module Magic Number）の更新**:

    • server_rec は非常に巨大な構造体であり、メンバーの追加や変更（例: server_scheme や context の追加など）を行う場合は、include/ap_mmn.h に定義されているモジュールマジックナンバー（MMN）の更新を伴う必要があります。

  • **不必要な変更の回避**:

    • 構造体が巨大かつコアなデータ構造であるため、モジュール開発者が不必要な変更を加えると、リクエスト処理や接続管理において予期せぬ動作を引き起こす可能性があります。必要なフィールドのみにアクセスすることが推奨されます。

  • **マルチスレッド環境での同期**:

    • 複数のスレッドから同時にアクセスされる場合、構成情報の変更には適切な同期が必要です。

  • **Hostヘッダーの欠如**:

    • リクエスト時に適切な Host ヘッダーが存在しない場合、デフォルトのバーチャルホスト設定が適用されます。

次に読む場所

  • **include/httpd.h**: server_rec の詳細な構造体定義とフィールド構成を確認できます。
  • **include/http_vhost.h**: バーチャルホストの解決やルックアップに関するAPIを確認できます。
  • **modules/loggers/mod_log_config.h**: ログモジュールにおける server_rec の参照方法を確認できます。
  • **server/vhost.c** *(推奨)*: 実装の詳細、ハッシュテーブルの構造やマッチングアルゴリズムの詳細を確認できます。
