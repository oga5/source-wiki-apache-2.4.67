dav_lock_indirect

概念

dav_lock_indirect は、Apache HTTP Server（httpd-2.4.67）の WebDAV（mod_dav）モジュールにおいて、親コレクションから継承された「間接ロック（Indirect Lock）」の情報を保持・管理するためのデータ構造（struct）です。

なぜ重要か

WebDAV プロトコルでは、ディレクトリ（コレクション）に対して設定されたロックが、その配下にあるすべてのリソース（ファイルやサブディレクトリ）に継承されます。dav_lock_indirect は、これらの継承された間接ロックを追跡し、ファイルシステム上のリソースに関連付けられたロック状態を正しく解決・検証するために不可欠な役割を担っています。

関与するモジュール

  • module:modules/dav/fs （ファイルシステムベースのロックプロバイダ実装）
  • module:modules/dav/lock （ジェネリックなロック管理実装）

主要エンティティ

  • symbol:modules/dav/fs/lock.c#dav_lock_indirect
  • symbol:modules/dav/lock/locks.c#dav_lock_indirect

両ファイルにおいて、ほぼ同一の形状を持つ以下の構造体として定義されています。

typedef struct dav_lock_indirect
{
    dav_locktoken *locktoken;
    apr_datum_t key;
    struct dav_lock_indirect *next;
    time_t timeout;
} dav_lock_indirect;

エンドツーエンドの流れ

1. **DBM からの構築**: DBM（データベース）にシリアライズされて保存されているロックレコードから、dav_fs_load_lock_record（fs 側）または dav_generic_load_lock_record（lock 側）によってデータが読み込まれ、メモリ上に dav_lock_indirect 構造体が構築されます。 2. **有効期限の判定**: 読み込み処理の過程で、構造体内の timeout が現在時刻（time(NULL)）と比較され、期限切れのロックは自動的にリストから除外されます。 3. **ロックの解決**: dav_fs_resolve（fs 側）または dav_generic_resolve（lock 側）が呼び出され、dav_lock_indirect の key（apr_datum_t）を使用して、対応する直接ロック（Direct Lock）を DBM から再帰的に特定し、間接ロックの有効性を検証します。 4. **DBM への保存**: ロック状態の変更時には、dav_generic_save_lock_record などの関数により、構造体の内容がバイナリ形式にシリアライズされて DBM へ書き戻されます。

*注意: WebDAV リクエストの受付から最終的なレスポンス送信に至る、HTTP 処理全体と連動した完全なエンドツーエンドのライフサイクルや呼び出しフローの全容は、提供されたコンテキストからは完全には明らかになっていません。*

重要な不変条件

  • **単方向リストの形成**: next ポインタにより、同一リソースに関連付けられた間接ロックの単方向リストを形成します。
  • **直接ロックへの参照**: key メンバは、対応する直接ロック（dav_lock_discovery）を DBM から検索・特定するための有効な識別子でなければなりません。
  • **メモリ管理**: すべてのインスタンスは、リクエスト処理に関連付けられたメモリプール（request_rec に紐付くプール）から割り当てられ、リクエスト終了時に自動的に解放されます。明示的な個別解放は不要です。

よくある落とし穴

  • **シリアライズの不整合**:

    • 本構造体は DBM 内のバイナリシリアライズロジックと密結合しています。構造体のメンバを変更する場合、dav_generic_save_lock_record や dav_generic_load_lock_record 内のオフセット計算を正確に更新しないと、バイナリ列のデシリアライズ時に不整合やデータ破損を引き起こします。

  • **キーの破損による解決失敗**:

    • 構造体内の key が無効または破損している場合、dav_fs_resolve や dav_generic_resolve による直接ロックの解決が失敗し、ロック状態の不整合が発生します。
