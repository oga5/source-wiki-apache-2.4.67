usage

概念

usage は、Apache HTTP Server（httpd）のメインプログラム、各種サポートユーティリティ、およびテスト用ツールにおいて、コマンドライン引数の指定に誤りがあった場合やヘルプオプションが指定された際に、正しい使用方法（ヘルプメッセージ）を標準エラー出力等に表示し、プロセスを終了させるために定義されている関数群の総称です。

なぜ重要か

各実行可能バイナリ（httpd 本体、ab、htpasswd、rotatelogs など）が受け付けるコマンドライン引数やオプションは多岐にわたります。また、プラットフォーム（WIN32、NETWARE、OS/2 等）やビルド時のマクロ定義（例: APR_FILES_AS_SOCKETS や DBM サポート状況）によって、利用可能なオプションが変化することもあります。

usage 関数は、ユーザーに対してその実行環境で利用可能な正しい構文とオプション一覧を提示し、不正な引数による誤動作を防ぎつつ、安全かつ即座にプログラムを終了させる役割を担うため重要です。

主要エンティティ

各モジュールおよびファイルで定義されている主な usage 関数は以下の通りです：

  • **symbol:server/main.c#usage**: httpd 本体のヘルプメッセージを APLOG_STARTUP を介して出力し、destroy_and_exit_process でプロセスを終了します。
  • **symbol:support/ab.c#usage**: ab（ApacheBench）ツールの使用方法を表示します。
  • **symbol:support/fcgistarter.c#usage**: fcgistarter ツールの使用方法（usage_message）を stderr に出力し、exit(EXIT_FAILURE) で終了します。
  • **symbol:support/htcacheclean.c#usage**: htcacheclean ツールの使用方法を表示します。
  • **symbol:support/htdigest.c#usage**: htdigest ツールの使用方法を errfile（標準エラー出力）に出力し、exit(1) で終了します。
  • **symbol:support/htpasswd.c#usage**: htpasswd ツールの使用方法を errfile に出力し、exit(ERR_SYNTAX) で終了します。bcrypt のデフォルトコスト値（BCRYPT_DEFAULT_COST）を動的に埋め込んで表示します。
  • **symbol:support/httxt2dbm.c#usage**: httxt2dbm ツールの使用方法に加え、ビルド時に有効化された DBM フォーマット（GDBM, SDBM, DB, NDBM）のサポート状況（AVAIL / UNAVAIL）を判定して出力します。
  • **symbol:support/logresolve.c#usage**: logresolve ツールの使用方法を errfile に出力し、exit(1) で終了します。
  • **symbol:support/rotatelogs.c#usage**: rotatelogs ツールの使用方法を stderr に出力し、exit(1) で終了します。マクロ APR_FILES_AS_SOCKETS の有無やプラットフォームに応じた設定例を出力します。
  • **symbol:test/clients/h2ws.c#usage**: h2ws テストクライアントの使用方法を表示します。
  • **symbol:test/test_limits.c#usage**: test_limits ツールの使用方法を stderr に出力し、exit(1) で終了します。

エンドツーエンドの流れ

各バイナリの起動時における一般的な流れは以下の通りです：

1. **引数のパース**: ユーザーまたは親プロセスがコマンドライン引数を指定してバイナリを実行すると、main 関数や引数解析関数（例: check_args）が引数をパースします。 2. **不正入力・ヘルプ要求の検知**: 不正なオプションが指定された場合、またはヘルプオプション（-h など）が指定された場合、対応する usage 関数が呼び出されます。 3. **メッセージ出力**: usage 関数は、標準エラー出力（stderr、APLOG_STARTUP、または APR の errfile）に対して、プログラム名、構文、利用可能なオプション一覧を出力します。 4. **プロセスの終了**: メッセージ出力後、exit または destroy_and_exit_process を呼び出して、プロセスを即座に終了させます。

*注: 各ツールにおける詳細な引数解析ロジックや、すべての呼び出し元との完全な結合関係など、詳細なエンドツーエンドの流れは完全には判明していません。*

重要な不変条件

  • **呼び出し後のプロセス終了**: usage 関数が呼び出された場合、呼び出し元に制御を戻すことなく、必ずプロセスを終了（exit または destroy_and_exit_process）させなければなりません。
  • **出力先は標準エラー出力**: ユーザーへの通知およびエラーハンドリングの観点から、メッセージは標準出力ではなく、標準エラー出力（stderr、APLOG_STARTUP、または errfile）に対して出力されます。
  • **引数の有効性**: server/main.c#usage のように process_rec を受け取る場合、process 引数は NULL でない有効な構造体を指している必要があります（NULL の場合はセグメンテーション違反の恐れがあります）。

よくある落とし穴

  • **メッセージの同期漏れ**: 新しいコマンドラインオプションを追加または変更した際、対応する usage 関数内のヘルプメッセージ（フォーマット文字列や usage_message 定数など）を手動で更新し忘れると、ユーザーに古い情報が提示される原因になります。
  • **ビルドマクロやプラットフォームによる差異**: httxt2dbm や rotatelogs のように、ビルド時のマクロ定義（例: APU_HAVE_* や APR_FILES_AS_SOCKETS）やプラットフォーム（WIN32, NETWARE, OS/2 等）によって表示内容や有効なオプションが変化するため、条件付きコンパイルの整合性に注意する必要があります。
  • **未初期化の出力先**: errfile などのファイルストリームが適切に初期化される前に usage が呼び出された場合、出力が失敗するか、未定義動作を引き起こす可能性があります。

次に読む場所

  • **server/main.c**: httpd 本体の引数処理と usage の実装。
  • **support/htpasswd.c**: bcrypt コスト値などの動的埋め込みを含む usage の実装。
  • **support/httxt2dbm.c**: ビルドマクロによる DBM サポート状況の動的判定を含む usage の実装。
  • **support/rotatelogs.c**: プラットフォームやマクロによる条件付き出力を含む usage の実装。
