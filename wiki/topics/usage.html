usage

概念

usage は、Apache HTTP Server（httpd）本体、および各種サポートツールやテストプログラムにおいて、コマンドライン引数の解析エラーが発生した場合やヘルプオプションが指定された際に、正しいコマンドの使用方法（ヘルプメッセージ）を標準エラー出力やログシステムに表示し、プロセスを安全に終了させるための関数群です。

なぜ重要か

各実行バイナリ（httpd 本体、fcgistarter、htpasswd など）が受け付けるコマンドライン引数やオプションの仕様はそれぞれ異なります。usage 関数は、ユーザーに対してそのバイナリがサポートする正しい構文やオプション一覧を提示し、不正な入力による誤動作を防ぎつつ、プロセスを適切な終了ステータスで終了させる役割を持つため重要です。

関与するモジュール

提供された情報に基づき、以下のモジュールが関与します。

  • module:server
  • module:support
  • module:test
  • module:test/clients

主要エンティティ

各モジュールおよびファイルで定義されている主な usage 関数は以下の通りです。

  • **symbol:server/main.c#usage**: httpd 本体のヘルプメッセージを APLOG_STARTUP を介して出力し、destroy_and_exit_process でプロセスを終了します。
  • **symbol:support/fcgistarter.c#usage**: fcgistarter 用のヘルプメッセージを出力し、exit(EXIT_FAILURE) で終了します。
  • **symbol:support/htdigest.c#usage**: htdigest 用のヘルプメッセージを出力し、exit(1) で終了します。
  • **symbol:support/htpasswd.c#usage**: htpasswd 用のヘルプメッセージを出力し、exit(ERR_SYNTAX) で終了します。
  • **symbol:support/httxt2dbm.c#usage**: httxt2dbm 用のヘルプメッセージと、ビルド時に有効な DBM フォーマット（SDBM, GDBM など）のサポート状況を出力します。
  • **symbol:support/logresolve.c#usage**: logresolve 用のヘルプメッセージを出力し、exit(1) で終了します。
  • **symbol:support/rotatelogs.c#usage**: rotatelogs 用のヘルプメッセージを出力し、exit(1) で終了します。
  • **symbol:test/test_limits.c#usage**: test_limits 用のヘルプメッセージを出力し、exit(1) で終了します。
  • **symbol:test/clients/h2ws.c#usage**: h2ws クライアント用のヘルプメッセージを表示します。

エンドツーエンドの流れ

各バイナリの起動時における一般的な流れは以下の通りです。

1. **起動と引数解析**: ユーザーまたはシステムがコマンドライン引数を指定してバイナリを実行し、main 関数（または引数解析ヘルパー）が argv を解析します。 2. **エラーまたはヘルプの検出**: 不正なオプションが指定された場合、あるいはヘルプオプション（-h など）が指定された場合に、それぞれの usage 関数が呼び出されます。 3. **メッセージ出力**: usage 関数は、標準エラー出力（stderr、errfile、または ap_log_error）に対して、そのバイナリ固有のヘルプメッセージを出力します。 4. **プロセスの終了**: メッセージ出力後、exit または destroy_and_exit_process を呼び出してプロセスを終了します。

*注: 個々のバイナリにおける詳細な引数解析ロジックや、すべての呼び出し元との完全なエンドツーエンドの連携フローは、提供された情報からは完全には特定されていません。*

重要な不変条件

  • **プロセスの強制終了**: すべての usage 関数は、呼び出された後に呼び出し元へ制御を戻さず、プロセスを終了させます（exit や destroy_and_exit_process の呼び出し）。
  • **出力先の制限**: ヘルプメッセージは原則として標準エラー出力（stderr、errfile、または APLOG_STARTUP を用いたエラーログ）に対して出力されます。

*注: システム全体にわたる共通の不変条件は、提供された情報からは完全には特定されていません。*

よくある落とし穴

  • **手動更新の漏れ**: コマンドラインオプションを追加・変更した際、対応する usage 関数内のメッセージ文字列を手動で更新しないと、ユーザーに古い情報や誤った情報が表示される原因となります。
  • **プラットフォーム依存の条件分岐**: server/main.c#usage や support/rotatelogs.c#usage のように、ビルド環境やプラットフォーム（WIN32, NETWARE, APR_FILES_AS_SOCKETS など）によって表示されるオプションが変化する場合があります。プラットフォームごとのビルド確認を怠ると、特定の環境で不正確なヘルプが表示される、あるいはビルドエラーが発生する可能性があります。
  • **NULLポインタ参照**: server/main.c#usage において、引数 process が NULL の場合に process->argv[0] へのアクセスでセグメンテーション違反が発生するリスクがあります。
