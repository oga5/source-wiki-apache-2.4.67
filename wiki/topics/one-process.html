one_process

概念

one_process は、Apache HTTP Server (httpd-2.4.67) の各種マルチプロセッシングモジュール (MPM) において定義・参照されているグローバル変数またはシンボルです。通常、サーバーを単一プロセスモード（デバッグモードなど）で実行するかどうかを制御・判定するために使用されます。

なぜ重要か

Apache HTTP Serverは通常、リクエストを並行処理するために複数のプロセスやスレッドを生成します。しかし、デバッグや特定の制限環境下での動作においては、単一のプロセスで動作させることが極めて有用です。one_process シンボルは、各MPM（event, mpmt_os2, prefork, worker）において、この単一プロセス動作モードを識別し、プロセスのフォークやスレッド生成の挙動を切り替えるためのフラグとして機能します。

現時点では、コマンドライン引数（例: -X オプション）の解析から、これら各MPMファイル内の one_process 変数へ値が設定され、最終的にプロセス生成処理がスキップされるまでの具体的な制御フローの詳細は未確定です。

重要な不変条件

現時点では、one_process が有効な場合に維持されるべき具体的な不変条件（例: 生成される子プロセス数が厳密に0または1であること、シグナルハンドリングの挙動など）は未確定です。
