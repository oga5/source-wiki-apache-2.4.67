one_process

概念

one_process は、Apache HTTP Server (httpd-2.4.67) の各種マルチプロセッシングモジュール（MPM）において定義・参照されているシンボル（通常は実行モードを制御するフラグ変数）です。これは、サーバーをマルチプロセス/マルチスレッドではなく、単一のプロセスとして実行するモード（主にデバッグ用途）を制御するために使用されます。

なぜ重要か

通常、Apacheは高い並行性を実現するために複数のプロセスやスレッドを起動しますが、デバッグ時や特定の制限環境下では、動作を追跡しやすくするために単一プロセスで実行する必要があります。one_process は、異なる並行処理モデルを持つ複数のMPM（event, mpmt_os2, prefork, worker）において、このシングルプロセス動作を統一的または個別に制御するための共通のフラグとして機能するため重要です。

重要な不変条件

one_process が有効（真）である場合、各MPMは追加の子プロセスをフォークしない、あるいはシグナルハンドリングやスレッド生成の挙動をシングルプロセス向けに制限するという不変条件が存在すると推測されますが、具体的なコード上の不変条件の詳細は現時点では特定されていません。

よくある落とし穴

one_process モードはデバッグや開発を目的として設計されているため、本番環境で誤って有効化すると、マルチプロセスの恩恵（耐障害性やスケーラビリティ）が得られなくなります。また、MPMの実装（prefork と worker/event など）の違いにより、one_process が有効な場合のスレッドの生成有無やシグナル処理の挙動に差異が存在する可能性がありますが、その詳細な落とし穴は現時点では未検証です。
