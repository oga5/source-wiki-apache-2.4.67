dav_lockdb_private

概念

dav_lockdb_private は、Apache HTTP Server の mod_dav 関連モジュールにおいて、WebDAV ロックデータベース（DBM）へのアクセスに必要なコンテキスト、状態情報、およびデータベースハンドルを保持するための内部（プライベート）構造体です。

なぜ重要か

この構造体は、WebDAV のロック管理において、データベースの遅延オープン（Lazy Open）やリクエストごとの状態追跡を効率的に行うために重要です。データベースのパス、メモリプール、およびオープン状態（opened フラグと db ハンドル）を一元管理することで、不要な I/O 操作を減らし、リクエスト終了時の適切なリソース解放を保証します。

関与するモジュール

  • module:modules/dav/fs (ファイルシステムベースのDAVロック実装)
  • module:modules/dav/lock (汎用的なDAVロック実装)
  • module:modules/dav/main (WebDAVコアモジュール)

主要エンティティ

  • symbol:modules/dav/fs/lock.c#dav_lockdb_private : modules/dav/fs/lock.c 内で定義される構造体。dav_db * 型のデータベースハンドルを持ちます。
  • symbol:modules/dav/lock/locks.c#dav_lockdb_private : modules/dav/lock/locks.c 内で定義される構造体。apr_dbm_t * 型のDBMハンドルを持ちます。
  • symbol:modules/dav/main/mod_dav.h#dav_lockdb_private : dav_lockdb 構造体内の info メンバ（不透明なポインタ）などとして参照される、共通のプライベートデータ型。

エンドツーエンドの流れ

詳細なエンドツーエンドの呼び出しフローの全容は完全には提示されていませんが、提供された情報に基づく部分的な流れは以下の通りです：

1. **初期化**: dav_fs_open_lockdb や dav_generic_open_lockdb などの関数によって dav_lockdb_private が初期化されます。この時点ではデータベースは即座には開かれません（遅延オープン）。 2. **遅延オープン**: 実際の I/O 操作（ロックの読み書きなど）が発生するタイミングで、dav_generic_really_open_lockdb などの関数を通じてデータベースが実際に開かれ、db フィールドにハンドルが設定され、opened フラグが更新されます。 3. **データベース操作**: dav_generic_load_lock_record などの関数が、この構造体を介して DBM 操作（apr_dbm API の呼び出しなど）を実行します。 4. **リソース解放**: リクエスト終了時に、関連付けられたメモリプール（pool）が破棄されることで、リソースが自動的に解放されます。

重要な不変条件

  • **データベースの有効性**: db フィールドが非NULLである場合、データベースは正常に開かれており、有効な状態である必要があります。
  • **メモリ管理の整合性**: 構造体内のメモリ割り当ては、必ず pool メンバ（通常は request_rec に関連付けられたプール）を使用して行われなければなりません。
  • **遅延オープンの整合性**: opened フラグと db ハンドルの状態は、実際のデータベースのオープン状態と一致している必要があります。

よくある落とし穴

  • **未オープン状態での操作**: db が NULL の状態で（かつ適切なオープン処理を経ずに）データベース操作を直接試みると、操作は失敗します。
  • **読み取り専用時の書き込み**: データベースが読み取り専用で開かれている場合、期限切れロックの自動削除（クリーンアップ）などの書き込みを伴う処理はスキップされるか、エラーになります。
  • **モジュール間の定義の相違**: modules/dav/fs/lock.c と modules/dav/lock/locks.c で同名の構造体が定義されていますが、db メンバの型（dav_db * と apr_dbm_t *）などが異なるため、これらを混同してキャストしたり、不適切に共有したりしないよう注意が必要です。

次に読む場所

  • modules/dav/fs/lock.c : ファイルシステムベースのロックデータベース実装と dav_lockdb_private の定義。
  • modules/dav/lock/locks.c : 汎用的な DBM ベースのロックデータベース実装と dav_lockdb_private の定義。
  • modules/dav/main/mod_dav.h : dav_lockdb などの上位構造体における info ポインタとしての参照箇所。
