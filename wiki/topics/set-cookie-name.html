set_cookie_name

概念

set_cookie_name は、Apache HTTP Server（httpd）の複数のモジュールにおいて、設定ファイル（httpd.conf など）のディレクティブから指定された Cookie 名を解析・検証し、各モジュールのディレクトリ設定構造体に保存するための設定ハンドラ関数です。

なぜ重要か

Apache におけるユーザー追跡（mod_usertrack）やセッション管理（mod_session_cookie、mod_session_dbd）では、どの Cookie 名を対象にするかを正しく設定・検証することが不可欠です。set_cookie_name は、設定された Cookie 名が不正な文字を含んでいないか、あるいは内部の正規表現パターンと合致しているかを初期化時に検証し、実行時の安全な Cookie 処理を保証する役割を担っています。

関与するモジュール

提供された情報に基づき、以下のモジュールが関与します。

  • module:modules/metadata (mod_usertrack.c)
  • module:modules/session (mod_session_cookie.c, mod_session_dbd.c)

主要エンティティ

  • **symbol:modules/metadata/mod_usertrack.c#set_cookie_name**:

    • CookieName ディレクティブ（AP_INIT_TAKE1）の処理を担当します。指定された Cookie 名を cookie_dir_rec 構造体に保存し、リクエストから Cookie を検索するための正規表現を生成・検証します。

  • **symbol:modules/session/mod_session_cookie.c#set_cookie_name**:

    • SessionCookieName ディレクティブ（AP_INIT_RAW_ARGS）の処理を担当します。引数から Cookie 名とオプションの属性（name_attrs）を抽出し、session_cookie_dir_conf 構造体に格納して検証します。

  • **symbol:modules/session/mod_session_dbd.c#set_cookie_name**:

    • SessionDBDCookieName ディレクティブ（AP_INIT_RAW_ARGS）に関連する設定ハンドラです（詳細な実装は提供されていませんが、呼び出し関係から存在が確認されています）。

エンドツーエンドの流れ

設定ファイルの解析フェーズにおいて、Apache の設定エンジンが対象のディレクティブを検出した際に呼び出されます。

1. mod_usertrack における流れ

1. CookieName ディレクティブが検出されると、mod_usertrack.c の set_cookie_name が呼び出されます。 2. 引数 name を cookie_dir_rec の cookie_name フィールドに代入します。 3. set_and_comp_regexp を呼び出し、指定された名前を組み込んだ正規表現をコンパイルして dcfg->regexp に格納します。 4. コンパイルされた正規表現のキャプチャグループ数を検証します。

2. mod_session_cookie における流れ

1. SessionCookieName ディレクティブが検出されると、mod_session_cookie.c の set_cookie_name が呼び出されます。 2. 引数文字列 args を apr_pstrdup で複製し、apr_strtok を用いて最初の空白までを Cookie 名（conf->name）として抽出します。 3. 残りの文字列から先頭の空白を除去し、conf->name_attrs（Cookie 属性）として格納します。 4. check_string を呼び出し、Cookie 名に不正な文字が含まれていないか検証します。

※ mod_session_dbd における具体的な設定解析の流れや、リクエスト処理時にこれらの設定された Cookie 名がどのように使用されるかといったエンドツーエンドの全体フローは、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **正規表現のキャプチャグループ数 (mod_usertrack)**:

    • set_and_comp_regexp によってコンパイルされた正規表現のキャプチャグループ数（re_nsub + 1）は、必ず NUM_SUBS（値は3）と一致しなければなりません。一致しない場合は設定エラー（無効な Cookie 名）となります。

  • **Cookie 名の文字制限 (mod_session_cookie)**:

    • check_string による検証により、Cookie 名に = や & などの不正な文字が含まれていてはなりません。

  • **メモリの生存期間**:

    • mod_session_cookie において、抽出された Cookie 名および属性は、設定のライフサイクル中保持されるよう、Apache のプールメモリ（apr_pstrdup）に複製されている必要があります。

よくある落とし穴

  • **モジュール間の暗黙的な依存関係**:

    • 提供された保守メモによると、mod_usertrack の set_cookie_name は mod_session_cookie や mod_session_dbd からも参照（または再利用）されているとされています。しかし、これらは各ファイル内で static 関数として定義されているため、変更を加える際は各モジュールでの定義や挙動の差異に注意を払う必要があります。

  • **NUM_SUBS 定数の変更**:

    • mod_usertrack において、NUM_SUBS は正規表現の構造に直結しています。この定数を変更する場合は、set_and_comp_regexp の実装との整合性を維持しないと、設定読み込み時にエラーが発生します。

次に読む場所

  • modules/metadata/mod_usertrack.c：CookieName の処理と正規表現コンパイルのロジック。
  • modules/session/mod_session_cookie.c：SessionCookieName のパースと check_string による検証ロジック。
  • modules/session/mod_session_dbd.c：SessionDBDCookieName ディレクティブに関連する設定処理。
