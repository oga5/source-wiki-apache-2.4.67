set_cookie_name

概念

set_cookie_name は、Apache HTTP Server の複数のモジュール（mod_usertrack、mod_session_cookie、mod_session_dbd）において、設定ディレクティブ（CookieName、SessionCookieName、SessionDBDCookieName など）を介して指定された Cookie 名を解析・検証し、それぞれの設定構造体に保存するための関数（設定ハンドラ）です。

なぜ重要か

Web アプリケーションやセッション管理、ユーザートラッキングにおいて、Cookie の名前はクライアントとサーバー間でデータを正しく識別するためのキーとなります。set_cookie_name は、設定ファイルから渡された Cookie 名が正しいフォーマットであるかを検証し、不正な文字の混入を防ぐとともに、後続の処理（正規表現によるマッチングや属性の付与など）が正しく動作するための設定情報を構築する役割を担っています。

関与するモジュール

提供された情報に基づき、以下のモジュールが関与しています。

  • module:modules/metadata (mod_usertrack.c)
  • module:modules/session (mod_session_cookie.c, mod_session_dbd.c)

主要エンティティ

  • **symbol:modules/metadata/mod_usertrack.c#set_cookie_name**: CookieName ディレクティブ等で指定された Cookie 名を cookie_dir_rec に保存し、検索用の正規表現を生成・検証する関数。
  • **symbol:modules/session/mod_session_cookie.c#set_cookie_name**: SessionCookieName ディレクティブ等から渡された引数を解析し、Cookie 名と属性を session_cookie_dir_conf に格納する関数。
  • **symbol:modules/session/mod_session_dbd.c#set_cookie_name**: SessionDBDCookieName ディレクティブに関連する設定関数（詳細な実装は提供されていませんが、呼び出し元として関与が示されています）。

エンドツーエンドの流れ

設定ファイル（httpd.conf など）のパース時に、各ディレクティブに対応する set_cookie_name が呼び出されます。

1. **mod_usertrack における流れ**:

  • CookieName ディレクティブ（AP_INIT_TAKE1）により呼び出されます。
  • 引数として渡された Cookie 名を cookie_dir_rec 構造体の cookie_name に格納します。
  • set_and_comp_regexp を呼び出して、Cookie 名に基づく正規表現をコンパイルし、dcfg->regexp に格納します。
  • コンパイルされた正規表現のキャプチャグループ数を検証します。

2. **mod_session_cookie における流れ**:

  • SessionCookieName ディレクティブ（AP_INIT_RAW_ARGS）により呼び出されます。
  • 引数文字列を複製し、apr_strtok を用いて最初の空白までを Cookie 名（conf->name）として抽出します。
  • 残りの文字列を Cookie の属性（conf->name_attrs）として格納します。
  • check_string を呼び出して、Cookie 名に不正な文字が含まれていないか検証します。

*注意*: mod_session_dbd における具体的な設定処理の流れや、リクエスト処理フェーズにおいてこれらの設定された Cookie 名を用いて実際に Cookie を抽出・操作するエンドツーエンドの具体的な流れは、提供された情報からは未確定です。

重要な不変条件

  • **mod_usertrack における正規表現の整合性**:

    • 生成・コンパイルされた正規表現のキャプチャグループ数（re_nsub + 1）は、必ず NUM_SUBS（値は3）と一致しなければなりません。一致しない場合は設定エラーとなります。

  • **mod_session_cookie における文字制限**:

    • Cookie 名には = や & などの不正な文字が含まれていてはなりません（check_string による検証）。

  • **メモリの生存期間**:

    • mod_session_cookie において抽出された Cookie 名および属性は、Apache のプールメモリ（apr_pstrdup による複製）に保持され、設定のライフサイクル中維持される必要があります。

よくある落とし穴

  • **モジュール間の相互参照と影響範囲**:

    • 提供された情報によると、mod_usertrack.c の set_cookie_name と mod_session_cookie.c の set_cookie_name は、互いのモジュールや mod_session_dbd.c から相互に参照・再利用されているとされています。そのため、一方のモジュールで関数のシグネチャや検証ロジック（例: NUM_SUBS の変更や check_string の制約）を変更すると、予期せず他のモジュールの動作や設定パースに影響を与える可能性があります。

  • **Cookie 名の文字制限による起動失敗**:

    • mod_session_cookie では = や & が含まれる Cookie 名はエラーとなるため、設定ファイルにこれらの文字が含まれていると Apache の起動時または再起動時に設定エラーとなります。
