set_cookie_name

概念

set_cookie_name は、Apache HTTP Server の複数のモジュール（mod_usertrack、mod_session_cookie、mod_session_dbd）において、設定ディレクティブを介して指定された Cookie 名を解析・検証し、それぞれのモジュールの設定構造体に保存するための関数群です。

なぜ重要か

Apache のセッション管理やユーザートラッキング機能において、クライアントとやり取りする Cookie の名前を正しく設定・検証することは、機能の正常な動作とセキュリティの確保に直結します。各モジュールは、設定された Cookie 名に基づいてリクエストから対象の Cookie を抽出したり、レスポンスに Cookie を付与したりするため、この初期設定フェーズが極めて重要です。

関与するモジュール

  • module:modules/metadata (mod_usertrack.c)
  • module:modules/session (mod_session_cookie.c, mod_session_dbd.c)

主要エンティティ

  • **symbol:modules/metadata/mod_usertrack.c#set_cookie_name**: CookieName ディレクティブ等で指定された Cookie 名を cookie_dir_rec に保存し、Cookie 検索用の正規表現を生成・検証する関数。
  • **symbol:modules/session/mod_session_cookie.c#set_cookie_name**: SessionCookieName ディレクティブ等から渡された引数を解析し、Cookie 名および関連属性を session_cookie_dir_conf に格納・検証する関数。
  • **symbol:modules/session/mod_session_dbd.c#set_cookie_name**: SessionDBDCookieName ディレクティブ等に関連するエンティティ（詳細な実装は提供されたコンテキストからは未詳）。

エンドツーエンドの流れ

1. **設定の読み込み**: Apache の起動時または設定再読み込み時に、設定ファイル内のディレクティブ（CookieName、SessionCookieName、SessionDBDCookieName）が検出されます。 2. **関数の呼び出し**:

  • mod_usertrack では、CookieName ディレクティブにより mod_usertrack.c の set_cookie_name が呼び出されます。
  • mod_session_cookie では、SessionCookieName ディレクティブにより mod_session_cookie.c の set_cookie_name が呼び出されます。

    • 3. **解析と検証**:

  • **mod_usertrack 側**: 指定された名前を cookie_dir_rec に保存し、set_and_comp_regexp を用いて Cookie 検索用の正規表現をコンパイル・検証します。
  • **mod_session_cookie 側**: 引数文字列をトークン化して Cookie 名と属性（name_attrs）に分割し、check_string を用いて Cookie 名に不正な文字が含まれていないか検証します。

    • 4. **設定の保存**: 検証を通過した Cookie 名や正規表現、属性情報が各モジュールのディレクトリ設定構造体に格納され、その後のリクエスト処理で利用されます。

*注意*: リクエスト処理フェーズにおけるこれらの設定値の具体的な利用方法など、エンドツーエンドの完全な実行フローは提供されたコンテキストからは不明です。

重要な不変条件

  • **mod_usertrack における不変条件**:
  • set_and_comp_regexp によって生成される正規表現のキャプチャグループ数（re_nsub + 1）は、必ず NUM_SUBS（値は3）と一致しなければなりません。一致しない場合は設定エラーとなります。
  • **mod_session_cookie における不変条件**:
  • Cookie 名に = や & などの不正な文字が含まれていてはなりません（check_string による検証）。
  • 抽出された Cookie 名および属性は、設定のライフサイクル中保持されるよう、メモリプールに複製（apr_pstrdup）される必要があります。

よくある落とし穴

  • **モジュール間の相互参照と影響**:

    • コンテキストの保守担当者向けメモによると、mod_usertrack の set_cookie_name や mod_session_cookie の set_cookie_name は、互いのモジュールや mod_session_dbd から参照・再利用されているとされています。そのため、一方の関数のシグネチャや振る舞い、あるいは NUM_SUBS などの定数を変更すると、他のモジュールのビルドや動作に予期せぬ影響を与える可能性があります。

  • **正規表現のコンパイル失敗**:

    • mod_usertrack において、指定した Cookie 名に起因して正規表現のコンパイルに失敗した場合、Apache の起動時または設定適用時にエラーとなり起動に失敗します。
