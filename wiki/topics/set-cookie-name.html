set_cookie_name

概念

set_cookie_name は、Apache HTTP Server の複数のモジュール（mod_usertrack、mod_session_cookie、mod_session_dbd）において、設定ファイルから Cookie 名（および関連する属性）を設定・検証するために使用される、同名の設定コマンドハンドラ関数群です。

なぜ重要か

Apache のセッション管理やユーザートラッキング機能において、クライアントとの間でやり取りする Cookie の名前を正しく設定し、検証することは極めて重要です。set_cookie_name は、設定された Cookie 名が不正な文字（= や & など）を含んでいないか、あるいはトラッキング用の正規表現パターンとして正しくコンパイルできるかといった検証を、サーバー起動時（設定解析時）に保証する役割を担っています。

関与するモジュール

  • module:modules/metadata
  • module:modules/session

主要エンティティ

  • **symbol:modules/metadata/mod_usertrack.c#set_cookie_name**: CookieName ディレクティブ（AP_INIT_TAKE1）を処理し、トラッキング用 Cookie 名の保存と正規表現の生成・検証を行います。
  • **symbol:modules/session/mod_session_cookie.c#set_cookie_name**: SessionCookieName ディレクティブ（AP_INIT_RAW_ARGS）を処理し、セッション用 Cookie 名と属性（name_attrs）を解析・検証して session_cookie_dir_conf に格納します。
  • **symbol:modules/session/mod_session_dbd.c#set_cookie_name**: SessionDBDCookieName ディレクティブ（AP_INIT_RAW_ARGS）に関連する Cookie 名の設定処理を行います。

エンドツーエンドの流れ

設定ファイルの解析フェーズにおいて、Apache コアが対応するディレクティブを検出した際に、それぞれのモジュールに定義された set_cookie_name が呼び出されます。

1. **mod_usertrack の場合**:

  • CookieName ディレクティブから渡された名前を cookie_dir_rec->cookie_name に格納します。
  • set_and_comp_regexp を呼び出して正規表現をコンパイルし、キャプチャグループ数が NUM_SUBS と一致するか検証します。

    • 2. **mod_session_cookie の場合**:

  • SessionCookieName ディレクティブから渡された引数を apr_strtok で Cookie 名と属性に分割します。
  • check_string を呼び出して、Cookie 名に不正な文字が含まれていないか検証します。

    • 3. **mod_session_dbd の場合**:

  • SessionDBDCookieName ディレクティブの処理時に呼び出され、check_string による検証などを行います。

※なお、リクエスト処理時におけるこれらの Cookie 名を用いた実際の Cookie 抽出やセッション復元などのエンドツーエンドの実行時フローについては、提供された情報からは未だ完全には明らかになっていません。

重要な不変条件

  • **mod_usertrack**:
  • set_and_comp_regexp によって生成される正規表現のキャプチャグループ数（re_nsub + 1）は、必ず NUM_SUBS（3）と一致しなければなりません。
  • **mod_session_cookie**:
  • 設定される Cookie 名には、check_string の制約により = や & などの不正な文字を含めることはできません。
  • **mod_session_dbd**:
  • 詳細な不変条件はまだ解析されていません。

よくある落とし穴

  • **モジュール間の同名関数の混同**: set_cookie_name という名前の関数は複数のモジュールに独立して定義されています。静的解析やコード変更の際、これらが相互に呼び出されているように見えることがありますが、実際には各モジュール独自の設定ハンドラです。変更を加える際は、対象のモジュール（mod_usertrack、mod_session_cookie、mod_session_dbd）を正しく区別する必要があります。
  • **正規表現の不整合**: mod_usertrack において NUM_SUBS の定義や set_and_comp_regexp の挙動を変更すると、set_cookie_name での検証が失敗し、サーバーの起動エラーにつながる可能性があります。
