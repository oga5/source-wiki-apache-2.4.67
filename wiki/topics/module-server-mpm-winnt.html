Module: server/mpm/winnt

概念

server/mpm/winnt は、Windows（WinNT）オペレーティングシステム向けに最適化された Apache HTTP Server のマルチプロセッシングモジュール（MPM）です。Windows のネイティブ機能（サービス管理、イベントログ、マルチスレッドAPIなど）を最大限に活用し、シングルプロセス・マルチスレッドモデルをベースに設計されています。

なぜ重要か

Windows 環境において Apache を高パフォーマンスかつ安定して動作させるために不可欠なモジュールです。Unix 系 OS のプロセスベースまたはハイブリッド型の MPM とは異なり、Windows のスレッドモデルやサービスコントロールマネージャー（SCM）に適合するように作られており、Windows 上でのデーモン（サービス）としてのライフサイクル管理やリソース効率を最適化します。

関与するモジュール

  • module:server/mpm/winnt

主要エンティティ

本モジュールは以下の主要なソースファイル群で構成されています。

  • **mpm_winnt.c / mpm_winnt.h**: MPM のメインエントリーポイント。親プロセスの初期化、コマンドライン引数の処理、および子プロセスの監視を担当します。
  • **child.c**: 子プロセス内での実際の処理を制御します。リクエストを処理するワーカースレッドの生成や管理を行います。
  • **service.c**: Windows のサービスコントロールマネージャー（SCM）と連携し、Apache を Windows サービスとして登録、起動、停止、制御するためのインターフェースを提供します。
  • **nt_eventlog.c**: Windows のイベントログにエラーや警告、ステータス情報を記録するためのヘルパーです。
  • **mpm_default.h**: スレッド数（ThreadsPerChild）など、WinNT MPM におけるデフォルトの動作パラメータを定義します。

エンドツーエンドの流れ

親プロセスが起動してからリクエストを処理するまでの大まかな流れは以下の通りですが、詳細なスレッド間同期やソケットの受け渡しなどの具体的な実装フローは現時点では未検証です。

1. **起動とサービス登録**: service.c を介して Windows サービスとして、またはコマンドラインから直接起動されます。 2. **親プロセスの初期化**: mpm_winnt.c が設定ファイルを読み込み、ログや共有メモリなどの基本リソースを初期化します。 3. **子プロセスの生成**: 親プロセスは、実際にリクエストを処理するための子プロセスを1つ生成します。 4. **ワーカースレッドの起動**: 子プロセス（child.c）内で、設定された数のワーカースレッドが起動し、接続の待ち受けとリクエスト処理を開始します。

重要な不変条件

Windows 環境におけるプロセスモデルの制約上、以下の不変条件が存在すると推測されますが、具体的なコードレベルでの同期プリミティブや状態遷移の不変条件は現時点では未検証です。

  • **2プロセスモデル**: 原則として、制御を行う1つの親プロセスと、リクエストを処理する1つの子プロセスのみがペアで動作します。
  • **サービス状態の同期**: Windows SCM からの停止・再起動要求に対し、サービスの状態（Running, Stopping, Stopped など）が正確に反映される必要があります。

よくある落とし穴

  • **権限の不整合**: Windows サービスとして実行する場合、実行アカウント（SYSTEM や特定のネットワークサービスアカウント）のアクセス権限不足により、ファイルシステムやネットワークリソースへのアクセスに失敗することがあります。
  • **Unix系MPMとの設定の乖離**: プロセス数やスレッド数のスケーリングモデルが Unix 系（event や worker）と大きく異なるため、Unix 向けの設定をそのまま流用するとパフォーマンスが低下したり、リソースが枯渇したりする原因になります。

次に読む場所

  • **server/mpm/winnt/mpm_winnt.c**: 親プロセスのライフサイクルと初期化処理を理解するために最初に読むべきファイルです。
  • **server/mpm/winnt/child.c**: 子プロセスがどのようにワーカースレッドを管理し、リクエストを処理するかを把握するのに適しています。
  • **server/mpm/winnt/service.c**: Windows サービスとしての統合ロジックを調査する際に参照してください。
