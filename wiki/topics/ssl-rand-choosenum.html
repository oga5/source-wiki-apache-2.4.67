ssl_rand_choosenum

概念

ssl_rand_choosenum は、Apache HTTP Server（httpd）のSSLモジュール（mod_ssl）およびベンチマークツール（ab）において、指定された範囲 [l, h] 内の擬似乱数（整数）を生成するために使用される内部関数です。

なぜ重要か

この関数は、SSL接続の初期化プロセス（例：ssl_rand_seed におけるスタックメモリのオフセット決定など）や、テストツールにおけるランダムな動作の制御など、特定の範囲に収まる乱数値を必要とする場面で使用されます。ただし、暗号学的に安全な乱数生成器（CSPRNG）ではないため、非暗号用途の補助的なランダム化に限定して利用されます。

関与するモジュール

  • module:modules/ssl
  • module:support

主要エンティティ

  • symbol:modules/ssl/ssl_engine_rand.c#ssl_rand_choosenum: mod_ssl 内部でスタックオフセット決定などに使用される乱数生成関数。
  • symbol:support/ab.c#ssl_rand_choosenum: ベンチマークツール ab で使用される乱数生成関数。

エンドツーエンドの流れ

1. **初期化**:

  • support/ab.c 側の実装では、呼び出し時に srand((unsigned int)time(NULL)) を用いて乱数シードを初期化します。
  • modules/ssl/ssl_engine_rand.c 側では、呼び出し前に rand() が初期化されていることを前提とします。

    • 2. **乱数生成とスケーリング**:

  • rand() を呼び出して得られた値を RAND_MAX で割るなどして 0.0 から 1.0 の範囲の浮動小数点数を得ます。
  • この値に範囲幅 (h - l) を乗算します。

    • 3. **文字列を経由した整数変換**:

  • 乗算結果を apr_snprintf を用いて文字列（%.0f など）としてフォーマットし、整数部を抽出します。
  • 抽出した文字列を atoi() で整数に変換し、1 を加算します。

    • 4. **範囲補正**:

  • 得られた値が下限 l 未満の場合は l に、上限 h を超える場合は h に補正して返却します。

※なお、modules/ssl/ssl_engine_rand.c と support/ab.c の間における相互の呼び出し関係や、具体的なシステム全体における完全なエンドツーエンドのデータフローの詳細は、現時点では完全には判明していません。

重要な不変条件

  • **範囲保証**: どのような入力であっても、戻り値は必ず l <= result <= h の範囲内に収まります。
  • **上限・下限逆転時の挙動**: l > h の場合、補正ロジックにより常に h が返されます。

よくある落とし穴

  • **暗号学的強度の欠如**: 標準ライブラリの rand() に依存しているため、暗号学的に安全な乱数が必要な用途には適していません。
  • **パフォーマンスのオーバーヘッド**: 浮動小数点数から整数への変換に apr_snprintf と atoi を経由する冗長な処理を行っているため、頻繁に呼び出されるパスではパフォーマンス上のボトルネックになる可能性があります。
  • **シード再初期化による重複 (ab.c)**: support/ab.c では呼び出しのたびに time(NULL) で srand() を初期化するため、秒未満の極めて短い間隔で連続して呼び出された場合、同一の乱数列（同じ値）が生成され続ける問題があります。
  • **スレッドセーフティ**: rand() および srand() はスレッドセーフではない可能性があり、マルチスレッド環境での並行呼び出し時に予期しない挙動を示す恐れがあります。
