ssl_rand_choosenum

概念

ssl_rand_choosenum は、指定された範囲 [l, h] 内の整数を擬似乱数として生成するヘルパー関数です。この関数は、Apache HTTP Server（httpd）のSSLモジュール（mod_ssl）およびベンチマークツールである ab（ApacheBench）の双方において、ほぼ同様のアルゴリズムで個別に実装されています。

なぜ重要か

この関数は、SSL接続の初期化やシード生成プロセス（例：ssl_rand_seed）において、スタックメモリのオフセット決定など、ランダムな変動要素を導入するために使用されます。これにより、特定の処理における予測不可能性を向上させる役割を担っています。

エンドツーエンドの流れ

1. **呼び出し**: ssl_engine_rand.c または ab.c 内の処理から、範囲の下限 l と上限 h を指定して ssl_rand_choosenum(l, h) が呼び出されます。 2. **初期化（ab.c のみ）**: srand((unsigned int)time(NULL)) を呼び出して乱数生成器を初期化します。 3. **乱数生成とスケーリング**: rand() を用いて 0 から RAND_MAX の範囲の乱数を生成し、RAND_MAX で割ることで 0.0 から 1.0 の間の浮動小数点数を得ます。これに範囲幅 (h - l) を乗算します。 4. **文字列を経由した整数変換**: apr_snprintf を用いて浮動小数点数を文字列（%.0f など）としてフォーマットし、それを atoi() で整数に変換した上で 1 を加算します。 5. **範囲補正**: 最終的な値が l 未満であれば l に、h を超えていれば h に調整して返却します。

※なお、呼び出し元（ab.c:655 や ssl_engine_rand.c:118）における具体的な上位コンテキストとの統合や、システム全体における完全なエンドツーエンドのデータフローの詳細は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **範囲保証**: 戻り値は必ず l <= result <= h の範囲内に収まります。計算後に境界チェックと補正ロジックが必ず実行されます。
  • **入力順序の前提**: l <= h であることが前提となります。もし l > h の状態で呼び出された場合、補正ロジックの挙動により常に h が返されます。

よくある落とし穴

  • **暗号学的な脆弱性**: 標準ライブラリの rand() に依存しているため、暗号学的に安全な乱数が必要な用途には適していません。
  • **時間依存による重複（ab.c）**: ab.c の実装では time(NULL) で乱数シードを初期化するため、秒単位以下の高速な連続呼び出しが行われた場合、同一の乱数列が生成される特性があります。
  • **非スレッドセーフ**: rand() および srand() はマルチスレッド環境においてスレッドセーフではない可能性があり、競合が発生する恐れがあります。
  • **パフォーマンスのオーバーヘッド**: 浮動小数点数から整数への変換に apr_snprintf による文字列フォーマットと atoi による再変換を経由しているため、純粋な整数演算に比べて処理のオーバーヘッドが存在します。
