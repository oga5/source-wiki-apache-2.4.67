ssl_rand_choosenum

概念

ssl_rand_choosenum は、Apache HTTP Server（httpd）のSSLモジュール（mod_ssl）およびベンチマークツール（ab）において、指定された範囲 [l, h]（下限 l、上限 h）の擬似乱数を生成するために使用されるヘルパー関数です。

なぜ重要か

この関数は、SSL接続の初期化時やテストツールの動作において、ランダムな要素（例えば、ssl_rand_seed におけるスタックメモリのオフセット決定など）を導入するために使用されます。これにより、予測可能性を排除し、セキュリティやテストのランダム性を担保する役割を果たします。ただし、暗号学的に強力な乱数生成器ではないため、補助的な用途に限定されています。

関与するモジュール

  • module:support

主要エンティティ

  • symbol:modules/ssl/ssl_engine_rand.c#ssl_rand_choosenum: mod_ssl 内部でスタックオフセット決定などのシード処理に利用される静的関数。
  • symbol:support/ab.c#ssl_rand_choosenum: ベンチマークツール ab 内で利用される静的関数。

エンドツーエンドの流れ

本関数は、modules/ssl と support の間で直接データをやり取りするようなモジュール間のエンドツーエンドの連携フローは持たず、それぞれのコンテキストで独立して動作します。

1. **初期化**:

  • support/ab.c では、呼び出し時に srand((unsigned int)time(NULL)) を用いて乱数生成器を初期化します。
  • modules/ssl/ssl_engine_rand.c では、呼び出し前に rand() が初期化されていることを前提とします。

    • 2. **乱数生成**: rand() を呼び出して 0 から RAND_MAX の範囲の乱数を取得し、RAND_MAX で除算して 0.0 から 1.0 の浮動小数点数を得ます。 3. **スケーリングと変換**: 得られた浮動小数点数に範囲幅 (h - l) を乗算し、apr_snprintf を用いて文字列（整数部）にフォーマットします。その後、atoi で整数に変換し、1 を加算します。 4. **範囲補正**: 計算された値が l 未満の場合は l に、h を超える場合は h に補正して返却します。

重要な不変条件

  • **範囲保証**: 戻り値は常に l <= result <= h の範囲内に収まることが保証されます。
  • **上限・下限の逆転時の挙動**: l > h の場合、補正ロジックにより常に h が返されます。

よくある落とし穴

  • **暗号学的強度の欠如**: rand() に依存しているため、暗号学的に安全な乱数が必要な用途には適していません。
  • **時間依存性**: support/ab.c では time(NULL) をシードに使用しているため、秒未満の非常に短い間隔で連続して呼び出された場合、同一の乱数列が生成される可能性があります。
  • **スレッドセーフティ**: rand() および srand() はスレッドセーフではない可能性があり、マルチスレッド環境での使用には注意が必要です。
  • **パフォーマンス**: 浮動小数点数の演算や apr_snprintf を経由した文字列変換、atoi による再変換を行っているため、パフォーマンス上のオーバーヘッドが存在します。
