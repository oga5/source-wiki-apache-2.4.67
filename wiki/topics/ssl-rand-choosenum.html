ssl_rand_choosenum

概念

ssl_rand_choosenum は、指定された範囲 [l, h]（下限 l から上限 h まで）の整数を擬似乱数として生成・返却するヘルパー関数です。Apache HTTP Server（httpd）のSSLモジュールおよびベンチマークツール（ab）の双方において、同様のアルゴリズムを持つ関数として個別に定義されています。

なぜ重要か

この関数は、SSL/TLSの初期化処理やテストツールにおいて、ランダムな値（例えば、ssl_engine_rand.c におけるスタックメモリのオフセット決定など）を簡易的に生成するために使用されます。ただし、暗号学的に安全な乱数生成器（CSPRNG）ではなく、軽量な擬似乱数生成を目的としています。

エンドツーエンドの流れ

現時点では、この関数が呼び出される全体的なエンドツーエンドの制御フロー（特に ab.c における具体的な呼び出しコンテキストや、SSL接続確立のどのフェーズでどのように統合されているか）の完全な詳細は判明していません。

判明している局所的な処理の流れは以下の通りです： 1. **初期化（support/ab.c のみ）**: srand((unsigned int)time(NULL)) を呼び出して乱数生成器を初期化します。 2. **乱数生成**: rand() を用いて 0 から RAND_MAX の範囲の乱数を取得し、0.0 から 1.0 の浮動小数点数に変換します。 3. **範囲スケーリングと文字列変換**: 範囲幅 (h - l) を乗算し、apr_snprintf を用いて文字列（%.0f 形式など）にフォーマットすることで整数部を抽出します。 4. **整数化と補正**: atoi で整数に変換して 1 を加算し、最終的な値が [l, h] の範囲に収まるように境界チェックと補正を行います。

重要な不変条件

  • **範囲の保証**: 戻り値は必ず l <= result <= h の範囲内に収まるように補正されます。
  • **入力順序の前提**: l <= h であることが前提とされています（不変条件が崩れた場合の挙動は「よくある落とし穴」を参照）。

よくある落とし穴

  • **暗号学的安全性の欠如**: rand() に依存しているため、暗号学的に強力な乱数が必要な用途には適していません。
  • **パフォーマンスオーバーヘッド**: 浮動小数点数の演算結果を apr_snprintf で一度文字列にフォーマットし、さらに atoi で整数に逆変換するという冗長な処理を行っているため、パフォーマンス上のオーバーヘッドが存在します。
  • **l > h 時の挙動**: 下限 l が上限 h より大きい場合、補正ロジックにより常に h が返される挙動となります。
  • **時間依存のシード（ab.c）**: support/ab.c では time(NULL) をシードに使用しているため、秒単位以下の極めて短い間隔で連続して呼び出された場合、同一の乱数列が生成される可能性があります。また、rand() や srand() はスレッドセーフではない可能性があるため、マルチスレッド環境での利用には注意が必要です。
