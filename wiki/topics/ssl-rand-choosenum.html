ssl_rand_choosenum

概念

ssl_rand_choosenum は、Apache HTTP Server (httpd) の SSL モジュールおよびベンチマークツール ab (ApacheBench) において、指定された範囲 [l, h] 内の擬似乱数（整数）を生成するために用意されているヘルパー関数です。

なぜ重要か

この関数は、SSL/TLS 接続の初期化時（例えば ssl_rand_seed におけるスタックメモリのオフセット決定など）や、ab ツール内での処理において、ランダムな数値を安全な範囲に収めて取得するために使用されます。暗号学的に強力な乱数ではないものの、処理のランダム性を担保するための簡便な手段として機能します。

エンドツーエンドの流れ

現時点では、この関数が呼び出される全体的なエンドツーエンドの制御フローの全貌や、呼び出し元（ssl_rand_seed や ab 内の特定処理）との完全な相互作用の詳細は完全には明らかになっていません。

判明している部分的な流れは以下の通りです： 1. **呼び出し**: modules/ssl/ssl_engine_rand.c (行118) または support/ab.c (行655) から、範囲の下限 l と上限 h を指定して呼び出されます。 2. **初期化 (ab側)**: support/ab.c の実装では、呼び出し時に srand((unsigned int)time(NULL)) を用いて乱数生成器を初期化します。 3. **乱数生成とスケーリング**: rand() を用いて 0.0 から 1.0 の範囲の浮動小数点数を生成し、範囲幅 (h - l) を乗算します。 4. **文字列経由の変換**: apr_snprintf を用いて浮動小数点数を文字列としてフォーマットし、atoi を用いて整数に変換した上で 1 を加算します。 5. **範囲補正**: 得られた値が l 未満であれば l に、h を超えていれば h に補正して返却します。

重要な不変条件

  • **範囲保証**: 戻り値は常に l <= result <= h の範囲内に収まります。計算後に下限 l および上限 h を超えないよう厳密な境界チェックと補正が行われます。
  • **入力の逆転**: l > h の場合、補正ロジックにより常に h が返される挙動となります。

よくある落とし穴

  • **非暗号学的乱数**: rand() に依存しているため、暗号学的に安全な乱数が必要な用途には適していません。
  • **パフォーマンスのオーバーヘッド**: 浮動小数点数から整数への変換に apr_snprintf と atoi を経由する冗長な処理を行っているため、パフォーマンス上のオーバーヘッドが存在します。
  • **時間依存のシード (ab側)**: support/ab.c では time(NULL) をシードにしているため、秒未満の非常に短い間隔で連続して呼び出された場合、同じ乱数列が生成される可能性があります。
  • **スレッドセーフティ**: rand() および srand() はスレッドセーフではない可能性があり、マルチスレッド環境での利用には注意が必要です。
