ap_strrchr_c

概念

ap_strrchr_c は、標準Cライブラリの strrchr をラップし、const char * 型の文字列に対して型安全な文字検索インターフェースを提供するユーティリティです。

なぜ重要か

本関数は、const char * 型の引数に対する安全な操作を保証するために重要です。httpd.h においてマクロとして定義される一方で、server/util_debug.c にも実体として定義されており、デバッグの容易性やバイナリ互換性の維持に寄与しています。

関与するモジュール

  • module:include
  • module:server

主要エンティティ

  • symbol:include/httpd.h#ap_strrchr_c：マクロ定義（httpd.h:2418）
  • symbol:server/util_debug.c#ap_strrchr_c：デバッグ・互換性のためのラッパー関数定義

エンドツーエンドの流れ

1. 各種モジュール（mod_authn_core.c、mod_mime.c、mod_http2.c など）から ap_strrchr_c(s, c) が呼び出されます。 2. server/util_debug.c 内の実装では、標準ライブラリの strrchr(s, c) を呼び出し、その結果をそのまま返します。

※なお、コンパイル時にマクロ定義と関数定義のどちらが優先して選択されるか、およびその詳細な切り替え制御フローの全容は現時点では判明していません。

重要な不変条件

  • 入力引数 s は、有効なヌル終端文字列でなければなりません。
  • 戻り値として返されるポインタは、必ず入力文字列 s のメモリ範囲内を指すか、あるいは NULL になります。

よくある落とし穴

  • s に NULL を渡した場合の挙動は、標準ライブラリの strrchr に依存するため、セグメンテーションフォールトなどの未定義動作を引き起こす可能性があります。
  • 検索対象の文字 c が文字列内に存在しない場合は NULL が返されるため、呼び出し側で戻り値の NULL チェックを怠るとヌルポインタ参照の原因になります。
