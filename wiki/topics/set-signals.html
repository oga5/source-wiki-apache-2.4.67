set_signals

概念

set_signals は、Apache HTTP Serverの特定のマルチプロセッシングモジュール（MPM）において、OS/2やNetWareといった特定のプラットフォーム環境下で、プロセスの終了、再起動、割り込み、または異常終了などのシグナルに対するハンドラを登録するための関数です。

なぜ重要か

プラットフォーム固有のシグナル（SIGTERM、SIGHUP、SIGINT、SIGUSR1、SIGABRTなど）を適切に捕捉し、対応するハンドラ（sig_term、sig_restart、sig_hupなど）にマッピングすることで、サーバープロセスや子プロセスの安全な終了処理や設定の再読み込みといったライフサイクル管理を正しく行うために重要です。

関与するモジュール

  • module:server/mpm/mpmt_os2
  • module:server/mpm/netware

主要エンティティ

  • symbol:server/mpm/mpmt_os2/mpmt_os2.c#set_signals: OS/2環境の親プロセス（または共通処理）において、SIGTERM/SIGINT（終了用）および SIGHUP/SIGUSR1（再起動用）のハンドラを sigaction を用いて登録します。
  • symbol:server/mpm/mpmt_os2/mpmt_os2_child.c#set_signals: OS/2環境の子プロセスにおいて、SIGTERM（終了用）および SIGHUP（設定再読み込み用）のハンドラを sigaction を用いて登録します。
  • symbol:server/mpm/netware/mpm_netware.c#set_signals: NetWare環境において、SIGTERM および SIGABRT を apr_signal を用いて sig_term ハンドラにバインドします。

エンドツーエンドの流れ

各プラットフォームのMPM起動時またはプロセス初期化時に set_signals が呼び出されます。

1. **OS/2（親プロセス/共通）**: mpmt_os2.c 内の set_signals が呼び出され、sigaction を介して SIGTERM/SIGINT に sig_term、SIGHUP/SIGUSR1 に sig_restart を登録します。 2. **OS/2（子プロセス）**: mpmt_os2_child.c 内の set_signals が呼び出され、sigaction を介して SIGTERM に sig_term、SIGHUP に sig_hup を登録します。 3. **NetWare**: mpm_netware.c 内の set_signals が呼び出され、apr_signal を介して SIGTERM および SIGABRT に sig_term を登録します。

なお、これらの呼び出しがどのような上位の初期化シーケンスからトリガーされるか、またシグナル受信後の詳細なクリーンアップ処理を含む完全なエンドツーエンドのフローは、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **ハンドラの存在**: 登録対象となるハンドラ関数（sig_term、sig_restart、sig_hupなど）が、同一スコープまたはリンク可能な範囲で定義されている必要があります。
  • **初期化順序（OS/2子プロセス）**: ap_server_conf をログ出力に使用するため、mpmt_os2_child.c の set_signals が呼び出される時点で、サーバー設定が初期化済みでなければなりません。
  • **プラットフォーム依存性**: 同名の関数ですが、OS/2環境ではシステムコール sigaction を直接使用するのに対し、NetWare環境では apr_signal を使用して抽象化されています。

よくある落とし穴

  • **エラーハンドリングの不備**:
  • OS/2環境（mpmt_os2.c / mpmt_os2_child.c）では、sigaction が失敗した際に ap_log_error で警告ログを出力しますが、プロセスの実行自体は継続されます。この場合、シグナルによる終了や再起動が期待通りに動作しない可能性があります。
  • NetWare環境（mpm_netware.c）では、apr_signal が内部的にエラーを返した場合の明示的なエラーハンドリングが行われていません。
  • **プラットフォーム間の混同**: 同名の set_signals 関数が複数のファイルに定義されていますが、プラットフォームごとにシグナルの種類やバインド方法が異なるため、一方の変更が他方に影響を与えないよう注意する必要があります。
