match

概念

match は、Apache HTTP Server（httpd-2.4.67）において、テキストフィルタリング（sed）およびメタデータ判定（MIME Magic）という異なる文脈で、パターンやデータの合致判定を実行する静的（static）関数です。

なぜ重要か

この関数は、リクエストやレスポンスの処理において、コンテンツの変換やメディアタイプの自動判定を行うためのコアとなるマッチングロジックを提供します。

  • modules/filters/sed1.c の match は、sedフィルタ内での正規表現やパターンの評価に寄与します。
  • modules/metadata/mod_mime_magic.c の match は、ファイル内容（マジック番号など）に基づいたMIMEタイプの判定に寄与します。

関与するモジュール

  • module:modules/metadata

主要エンティティ

  • **symbol:modules/filters/sed1.c#match**
  • シグネチャ: static int match(sed_eval_t *eval, char *expbuf, int gf, step_vars_storage *step_vars)
  • 役割: sed評価器（sed_eval_t）において、コンパイル済みの正規表現バッファ（expbuf）に対するマッチングを行います。
  • **symbol:modules/metadata/mod_mime_magic.c#match**
  • シグネチャ: static int match(request_rec *r, unsigned char *s, apr_size_t nbytes)
  • 役割: リクエスト（request_rec）に紐づくデータバッファ（s）に対して、MIMEマジックパターンのマッチングを行います。

エンドツーエンドの流れ

現時点では、これらの関数が呼び出される全体的なエンドツーエンドの制御フローは完全には解明されていません。ただし、呼び出し関係から以下の流れが示唆されています。

  • **sedフィルタ側**: do_pattmatch, execute, softmagic, spdist, substitute などの処理から match が呼び出され、内部で sed_step を呼び出してマッチングを段階的に進めます。
  • **MIME Magic側**: 同様に do_pattmatch, execute, softmagic, spdist, substitute などの処理から match が呼び出され、内部で ap_get_module_config による設定取得や、mcheck, mget, mprint, magic_rsl_putchar を用いたマジックチェックおよび結果出力を処理します。
