get_line

概念

get_line は、Apache HTTP Server 内の複数の異なるモジュールやユーティリティにおいて、入力ソース（バケットブリゲードやファイルストリーム）からテキストを1行単位で読み取るために定義されている、同名の独立したヘルパー関数群です。これらは共通のライブラリ関数ではなく、それぞれのコンテキスト（HTTP/2 接続フィルタ、OCSP クライアント、htdigest ツール）のデータ構造や要件に合わせて個別に実装されています。

なぜ重要か

各モジュールにおいて、プロトコルヘッダーや設定ファイルなどの行指向データを解析する際の基礎的なビルディングブロックとして機能します。それぞれの処理系（バケットブリゲードのフラット化、メモリプール管理、ファイルI/O）に最適化された行切り出しロジックを提供することで、安全かつ効率的なパース処理を支えています。

関与するモジュール

  • module:modules/ssl

主要エンティティ

  • **symbol:modules/http2/h2_c2_filter.c#get_line**: HTTP/1.1 形式のレスポンスデータ（ブリゲード）から CRLF で終了する1行を読み取り、指定バッファにコピーします。
  • **symbol:modules/ssl/ssl_util_ocsp.c#get_line**: OCSP レスポンダからの HTTP レスポンスストリームから1行読み取り、改行を除去してメモリプール上に割り当てた文字列を返します。
  • **symbol:support/htdigest.c#get_line**: htdigest ユーティリティにおいて、ファイルストリームから1行を読み込み、指定バッファに格納します。

エンドツーエンドの流れ

これらは独立したヘルパー関数であり、システム全体を横断する単一の「エンドツーエンドの流れ」は存在しません。それぞれのコンテキストにおける個別の流れは以下の通りです。

1. **HTTP/2 接続フィルタ (h2_c2_filter.c)**

  • h2_c2_filter_catch_h1_out などの解析ロジックから呼び出されます。
  • apr_brigade_split_line で入力ブリゲードから1行分を一時ブリゲードに切り出します。
  • apr_brigade_flatten でバッファにコピーし、行末の CRLF を除去してヌル終端します。
  • データが不完全な場合は、一時ブリゲードの内容を退避させて APR_EAGAIN で中断し、次回呼び出し時に再開します。

2. **OCSP クライアント (ssl_util_ocsp.c)**

  • OCSP レスポンスの解析処理から呼び出されます。
  • apr_brigade_split_line を用いて、最大 8192 バイトまで入力ブリゲードから切り出します。
  • apr_brigade_pflatten でメモリプール上にフラット化し、行末の LF（および直前の CR）をヌル文字に置換して返します。

3. **htdigest ユーティリティ (htdigest.c)**

  • ファイルから行を読み込むループ内で呼び出されます。
  • apr_file_getc を用いて、改行（\n）に達するかバッファサイズ制限に達するまで1文字ずつ読み込み、バッファに格納してヌル終端します。

重要な不変条件

各関数は、それぞれのコンテキストにおいて以下の不変条件を維持または前提とします。

  • **h2_c2_filter.c#get_line**
  • 不完全な行の断片は、次回呼び出しまで parser->saveto に保持されなければなりません。
  • 正常に取得された行は、必ず末尾の CRLF が除去され、ヌル終端されます。
  • **ssl_util_ocsp.c#get_line**
  • 読み取られた行は、必ず APR_ASCII_LF で終了している必要があります（そうでない場合はエラー）。
  • 一時格納用のブリゲード bbout は、呼び出しのたびにクリーンアップされます。
  • **htdigest.c#get_line**
  • 格納先バッファ s は、少なくとも n バイトの領域が確保されている必要があります。

よくある落とし穴

  • **名前の混同**: 同名でありながら、シグネチャ、戻り値の型、エラーハンドリングの挙動が全く異なるため、コードリーディングや静的解析の際に混同しないよう注意が必要です。
  • **バッファサイズ制限とエラー挙動の違い**:
  • h2_c2_filter.c では、行の長さがバッファサイズを超えると APR_ENOSPC を返します。また、LF のみで CR がない場合の挙動に未定義の部分があります。
  • ssl_util_ocsp.c では、8192 バイトを超える行、または LF で終わらない行はエラーとして NULL を返します。
  • htdigest.c では、バッファサイズ n - 2 を超える文字は読み込まれず、途中で打ち切られてヌル終端されます（呼び出し側で EOF とエラーの区別が必要です）。
