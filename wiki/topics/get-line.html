get_line

概念

get_line は、Apache HTTP Server 内の複数の異なるコンテキストにおいて、入力ストリームやファイルから改行区切りのデータ（1行）を読み取るために定義されている、同名かつ独立したヘルパー関数群です。

なぜ重要か

各モジュール（HTTP/2 フィルタ、SSL/OCSP ユーティリティ、htdigest サポートツール）は、それぞれ異なるデータソース（バケットブリゲードやファイルストリーム）やメモリ管理方式（APR プール、一時ブリゲード）を使用します。get_line は、それぞれのコンテキストに最適化されたバッファリング、改行コード（CRLF/LF）の除去、およびエラーハンドリングをカプセル化するために重要です。

関与するモジュール

  • module:modules/http2
  • module:modules/ssl
  • module:support

主要エンティティ

  • symbol:modules/http2/h2_c2_filter.c#get_line: HTTP/1.1 形式のレスポンスデータ（ブリゲード）から 1 行を読み取り、指定バッファへコピーする関数。
  • symbol:modules/ssl/ssl_util_ocsp.c#get_line: OCSP レスポンダからの HTTP レスポンスストリームから 1 行を読み取り、メモリプール上に改行除去済みの文字列を生成して返す関数。
  • symbol:support/htdigest.c#get_line: ファイルストリームから 1 行を読み込み、指定されたバッファに格納する静的関数。

エンドツーエンドの流れ

これら 3 つの関数は、それぞれ独立したコンテキストで動作しており、**システム全体を横断する単一のエンドツーエンドのフローは存在しません**。それぞれの個別フローは以下の通りです。

1. **HTTP/2 フィルタ (h2_c2_filter.c)**

  • h2_c2_filter_catch_h1_out などの解析ロジックから呼び出されます。
  • apr_brigade_split_line を使用して入力ブリゲードから 1 行分を parser->tmp に切り出します。
  • apr_brigade_flatten でバッファにコピーし、行末の CRLF を除去します。
  • 行が不完全な場合は ap_save_brigade で parser->saveto に退避し、次回呼び出し時に継続処理します。

2. **SSL/OCSP ユーティリティ (ssl_util_ocsp.c)**

  • OCSP レスポンスの解析処理から呼び出されます。
  • apr_brigade_split_line を使用して、入力ブリゲードから最大 8192 バイトを一時ブリゲード bbout に切り出します。
  • apr_brigade_pflatten でメモリプール上にフラット化し、行末の LF（および直前の CR）をヌル文字 \0 に置換して返します。

3. **htdigest ツール (htdigest.c)**

  • ファイルから行を読み込むループ内で呼び出されます。
  • apr_file_getc を用いて、改行文字（\n）に達するかバッファ制限（n - 2 文字）に達するまで 1 文字ずつ読み込み、バッファをヌル終端します。

重要な不変条件

  • **HTTP/2**:
  • parser->tmp は行の断片の一時保持用、parser->saveto は不完全な行の次回持ち越し用として一貫して使用されます。
  • 正常終了時、行末の CRLF は除去され、バッファは必ずヌル終端されます。
  • **SSL/OCSP**:
  • 一時ブリゲード bbout は、呼び出しのたびに必ず apr_brigade_cleanup で初期化されます。
  • 読み取られた行は必ず APR_ASCII_LF で終了している必要があります。
  • **htdigest**:
  • バッファ s は少なくとも n バイトの領域を確保している必要があります。
  • 最大 n - 2 文字までしか読み込まず、バッファオーバーフローを防ぎます。

よくある落とし穴

  • **名前の混同**:
  • これらはシグネチャ、戻り値、エラーハンドリング、およびメモリ管理方法が全く異なります。同名ですが、直接の呼び出し関係や共通の親インターフェースはありません。
  • **バッファサイズ制限**:
  • HTTP/2 では HUGE_STRING_LEN を超える行、SSL/OCSP では 8192 バイトを超える行、htdigest では n - 2 バイトを超える行は、それぞれエラー（APR_ENOSPC や NULL）または読み込み中断となります。
  • **htdigest の戻り値**:
  • htdigest の get_line が返す 1 は、エラーだけでなく EOF（ファイル終端）も意味するため、呼び出し側で詳細な状態判別を行う際には注意が必要です。
  • **未定義の挙動**:
  • HTTP/2 の実装において、LF のみで CRLF が存在しない場合の挙動は未定義（ソースコード上に警告コメントあり）となっています。
