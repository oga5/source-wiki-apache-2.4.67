requests_this_child

概念

requests_this_child は、Apache HTTP Server（httpd）のマルチプロセッシングモジュール（MPM）において、**個々の子プロセスが起動してから現在までに処理した累積リクエスト数**を追跡・管理するための変数（シンボル）です。

なぜ重要か

Apache HTTP Serverでは、長期稼働に伴うメモリリークの防止やリソースの健全性を維持するために、1つの子プロセスが処理できる最大リクエスト数（MaxConnectionsPerChild 設定値）を制限する仕組みが備わっています。

requests_this_child は、各子プロセスがこの制限値に達したかどうかを判定し、安全にプロセスを終了（リタイア）させて新しい子プロセスへ交代させるためのカウンタとして極めて重要な役割を担っています。

関与するモジュール

  • module:server/mpm/prefork
  • module:server/mpm/worker

主要エンティティ

  • **symbol:server/mpm/prefork/prefork.c#requests_this_child**

    • シングルスレッド・マルチプロセスモデルである prefork MPMにおいて、子プロセスが処理したリクエスト数をカウントします。

  • **symbol:server/mpm/worker/worker.c#requests_this_child**

    • マルチスレッド・マルチプロセスモデルである worker MPMにおいて、子プロセス内の複数のワーカースレッドが共同で処理したリクエストの総数をカウントします。

エンドツーエンドの流れ

現時点では、リクエストを受信した際における requests_this_child の具体的なインクリメントのタイミングや、MaxConnectionsPerChild との比較によるプロセス終了処理の厳密なコードレベルでのエンドツーエンドの流れは完全には解析されていません。

重要な不変条件

現時点では、この変数が満たすべき厳密な不変条件（例：スレッド間での排他制御の有無や、アトミック操作の保証など）は完全には特定されていません。

よくある落とし穴

  • **マルチスレッド環境での競合**: worker MPMのようなマルチスレッドモデルでは、複数のスレッドが同時に requests_this_child を更新しようとするため、適切なアトミック操作やロックが施されていない場合、カウントの不整合（ロストアップデート）が発生する可能性があります（具体的な同期機構の実装状況は未検証です）。
  • **プロセスの早期終了**: 設定値が極端に小さく見積もられている場合、プロセスの頻繁な再起動が発生し、パフォーマンス低下を招く原因になります。
