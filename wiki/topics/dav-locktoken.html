dav_locktoken

概念

dav_locktoken は、Apache HTTP Serverの mod_dav において、WebDAV（Web Distributed Authoring and Versioning）のロックを一意に識別するためのトークンを保持するデータ構造（構造体）です。内部的には、APR（Apache Portable Runtime）が提供する apr_uuid_t 型のUUIDをラップしています。

なぜ重要か

WebDAV仕様では、複数クライアントによる同時編集時の競合を防ぐために排他制御（ロック）が定義されています。dav_locktoken は、特定のロックを一意に識別・照合・比較・永続化するための基盤となる識別子を提供します。これにより、リソースに対する安全なロック操作（ロックの作成、検索、解決、解除など）が可能になります。

関与するモジュール

  • module:modules/dav/fs (ファイルシステムベースのロック実装)
  • module:modules/dav/lock (汎用的なロック管理)
  • module:modules/dav/main (WebDAVのコア定義)

主要エンティティ

  • symbol:modules/dav/fs/lock.c#dav_locktoken: ファイルシステムロックプロバイダにおける dav_locktoken 構造体の定義。
  • symbol:modules/dav/lock/locks.c#dav_locktoken: 汎用ロック管理における dav_locktoken 構造体の定義。
  • symbol:modules/dav/main/mod_dav.h#dav_locktoken: mod_dav の共通ヘッダーにおける宣言。

modules/dav/fs/lock.c と modules/dav/lock/locks.c の両方において、以下の形状で定義されています。

struct dav_locktoken {
    apr_uuid_t uuid;
};

エンドツーエンドの流れ

WebDAVクライアントからロック要求（LOCKメソッドなど）やロック解除要求（UNLOCKメソッドなど）が送信された際、サーバー側でロックトークンが処理されます。 ただし、リクエストの受信から、トークンの生成、解析、永続化、そしてクライアントへのレスポンス送信に至る具体的なエンドツーエンドの制御フローの全容は、提供されたコンテキストからは完全には明らかになっていません。

部分的に判明している流れとして、以下の処理が行われます。 1. **解析と生成**: dav_fs_parse_locktoken などの関数を用いて、リクエストからロックトークンを解析・生成します。 2. **照合とロード**: dav_fs_load_lock_record などの関数が、永続化されたロック情報からトークンをロードし、dav_fs_compare_locktoken を用いてクライアントから提示されたトークンと照合します。 3. **フォーマット**: dav_fs_format_locktoken を用いて、トークンをクライアントへ返却する文字列形式に変換します。

重要な不変条件

  • **一意性**: 内部に保持する apr_uuid_t により、システム全体でロックトークンが一意であることが保証される必要があります。
  • **不変性**: ロック操作の文脈において、この構造体へのポインタは const として扱われることが一般的であり、一度生成されたトークン自体の値（UUID）は変更されません。

よくある落とし穴

  • **構造体の重複定義**: dav_locktoken は modules/dav/fs/lock.c と modules/dav/lock/locks.c の両方で個別に定義されています。これらは同一の形状（apr_uuid_t のみを持つ）をしていますが、モジュール間で構造体を変更・拡張する際には、双方の整合性を維持する必要があります。
  • **UUIDの妥当性**: 構造体自体は単なるデータコンテナであるため、無効なUUIDが渡された場合の検証は、本構造体を利用する関数（dav_fs_compare_locktoken など）の呼び出し側で適切に行う必要があります。

次に読む場所

  • modules/dav/fs/lock.c: ファイルシステムベースのロックトークンの解析、比較、フォーマット処理（dav_fs_parse_locktoken など）の実装。
  • modules/dav/lock/locks.c: 汎用ロック管理におけるトークンの扱い。
  • modules/dav/main/mod_dav.h: mod_dav のロックプロバイダインターフェース定義。
