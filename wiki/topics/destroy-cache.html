destroy_cache

概念

destroy_cache は、Apache HTTP Server（httpd-2.4.67）の複数のモジュールにおいて、共有オブジェクトキャッシュ（socache）インスタンスの破棄および関連リソースの解放を行うクリーンアップ関数です。主にメモリプールのクリーンアップ処理と連携して動作します。

なぜ重要か

サーバーの停止時や設定の再読み込み（リスタート）の際、古いキャッシュインスタンスやリソースを適切に解放し、メモリリークやリソースの二重解放を防ぐために重要です。

関与するモジュール

  • module:modules/aaa
  • module:modules/cache

主要エンティティ

  • **symbol:modules/aaa/mod_authn_socache.c#destroy_cache**: 認証用 socache インスタンスを破棄するクリーンアップ関数。
  • **symbol:modules/cache/mod_cache_socache.c#destroy_cache**: キャッシュモジュール側で定義されている destroy_cache 関数（詳細未解析）。

エンドツーエンドの流れ

1. **登録**: mod_authn_socache.c では、設定用プール（pconf）に対して apr_pool_cleanup_register を使用し、本関数をクリーンアップハンドラとして登録します。 2. **トリガー**: サーバーの停止または設定再読み込み時に、登録された destroy_cache が自動的に呼び出されます。 3. **破棄処理**:

  • mod_authn_socache.c 内の処理では、グローバル変数 socache_instance が非 NULL である場合、socache_provider->destroy を呼び出してキャッシュインスタンスを破棄し、socache_instance を NULL にリセットします。
  • mod_cache_socache.c 内の destroy_cache は内部で ap_get_module_config を呼び出しますが、現時点では詳細なエンドツーエンドの実行フローは解析されていません。

重要な不変条件

  • **mod_authn_socache.c における不変条件**:
  • socache_instance を破棄した後に必ず NULL にリセットすること。これにより、二重解放を防ぎます。
  • socache_provider が初期化済みであること。
