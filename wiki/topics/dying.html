dying

概念

dying は、Apache HTTP Server のマルチプロセッシングモジュール（MPM）である event および worker において、プロセスまたはスレッドが終了処理中（シャットダウンやクローズのプロセス）であることを示す状態、あるいはそれを管理するためのフラグ（変数）です。

なぜ重要か

マルチスレッドおよびマルチプロセスモデルを採用する event や worker MPM において、プロセスやスレッドが安全かつクリーンに終了（グレースフルシャットダウンなど）するためには、新規リクエストの受付を停止し、既存の処理を完了させる必要があります。dying 状態を適切に管理することは、リソースのリークを防ぎ、クライアントへの接続を途切れさせずにプロセスを終了させるために重要です。

関与するモジュール

  • module:server/mpm/event
  • module:server/mpm/worker

主要エンティティ

  • symbol:server/mpm/event/event.c#dying: event MPM において、スレッドやプロセスの終了状態を追跡・制御するためのシンボル。
  • symbol:server/mpm/worker/worker.c#dying: worker MPM において、スレッドやプロセスの終了状態を追跡・制御するためのシンボル。

エンドツーエンドの流れ

現時点では、dying フラグがセットされる具体的なトリガー（シグナル受信、親プロセスからの指示など）から、実際にスレッドやプロセスが完全に破棄されるまでの詳細なエンドツーエンドの流れは特定されていません。

重要な不変条件

現時点では、dying 状態の遷移時におけるスレッドセーフティや、他の状態フラグとの排他関係などの具体的な不変条件は特定されていません。

よくある落とし穴

現時点では、dying 状態のハンドリングにおけるデッドロックや、シグナルハンドラ内での安全でない操作などの具体的な落とし穴は特定されていません。
