h2_session_send

概念

h2_session_send は、Apache HTTP Server（httpd）の HTTP/2 実装（mod_http2）およびテスト用クライアントツール（h2ws）において、HTTP/2 セッションを介したデータ送信処理を担う関数、またはその処理単位です。サーバー側での実際のセッション送信制御と、クライアント側でのテスト送信シミュレーションの双方に同名のシンボルが存在します。

なぜ重要か

HTTP/2 プロトコルでは、単一の接続上で複数のストリームが多重化されます。h2_session_send は、フレーム（HEADERS、DATA、WINDOW_UPDATE など）を適切にスケジュールし、対向のアピア（クライアントまたはサーバー）へ送信するためのコア機能を担うため、通信の効率性とプロトコル準拠の維持において極めて重要です。

関与するモジュール

  • module:modules/http2: Apache HTTP Server の HTTP/2 プロトコルサポートモジュール。
  • module:test/clients: HTTP/2 や WebSocket などの動作を検証するためのテストクライアント。

主要エンティティ

  • symbol:modules/http2/h2_session.c#h2_session_send: サーバー側でセッションの状態を管理し、保留中のデータをクライアントへ送信する関数。
  • symbol:test/clients/h2ws.c#h2_session_send: テストクライアント（h2ws）において、検証用の HTTP/2 セッションデータを送信する関数。

エンドツーエンドの流れ

現時点では、サーバー側の h2_session_send がどのようなイベント（I/O フィルター、書き込みイベントなど）によってトリガーされ、テストクライアント側の h2_session_send とどのように通信を確立してデータをやり取りするかについての、詳細なエンドツーエンドの制御フローは特定されていません。

重要な不変条件

送信時におけるフロー制御（ウィンドウサイズ制限）の遵守や、マルチスレッド環境におけるセッション状態の排他制御など、h2_session_send が維持すべき具体的な不変条件は現時点では未検証です。

よくある落とし穴

ノンブロッキング I/O における送信バッファの枯渇や、不適切なウィンドウ更新によるデッドロックなどが一般的な HTTP/2 送信処理の課題として考えられますが、本実装における具体的な落とし穴は現時点では特定されていません。
