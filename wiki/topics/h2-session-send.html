h2_session_send

概念

h2_session_send は、Apache HTTP Server（httpd）の HTTP/2 実装（mod_http2）およびそのテストクライアント（h2ws）において、HTTP/2 セッションのデータ送信処理を担当する関数またはシンボルです。サーバー側のコア実装と、テスト用のクライアント実装の双方に同名のシンボルが存在します。

なぜ重要か

HTTP/2 プロトコルでは、単一の接続（セッション）上で複数のストリームが多重化されて送信されます。h2_session_send は、これらのストリームのフレームや制御フレームを適切にネットワークへ送信するためのコアとなる処理を担うため、通信の確立と維持において極めて重要です。また、テストクライアント側でも同様の送信処理をシミュレートするために必要となります。

関与するモジュール

  • module:modules/http2 : Apache HTTP Server の HTTP/2 プロトコルサポートモジュール。
  • module:test/clients : HTTP/2 や WebSocket などの動作を検証するためのテストクライアント。

主要エンティティ

  • symbol:modules/http2/h2_session.c#h2_session_send : サーバー側で HTTP/2 セッションの送信バッファやフレームのフラッシュ、ソケットへの書き込みを制御する関数。
  • symbol:test/clients/h2ws.c#h2_session_send : テストクライアント（h2ws）において、HTTP/2 セッションデータを送信するための関数。

エンドツーエンドの流れ

現時点では、具体的なソースコードの詳細な解析が行われていないため、詳細なエンドツーエンドのデータフロー（バッファリング、ウィンドウ制御、ソケット書き込みの正確な順序など）は未確定です。

重要な不変条件

現時点では、セッション状態の遷移やスレッドセーフ性、送信ウィンドウサイズに関する具体的な不変条件（Invariants）は特定されていません。

よくある落とし穴

現時点では、エラーハンドリングやブロッキング/ノンブロッキングI/Oの挙動、送信バッファのオーバーフローに関する具体的な落とし穴は特定されていません。

  • test/clients/h2ws.c : テストクライアントにおける送信処理のシミュレーション方法を確認してください。
