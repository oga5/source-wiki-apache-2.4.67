dav_stream

概念

dav_stream は、Apache HTTP Server の WebDAV モジュール（mod_dav および mod_dav_fs）において、WebDAV リソースに対する読み書き操作のストリーム状態を管理するための構造体です。特にファイルシステムをバックエンドとするリポジトリ（mod_dav_fs）において、一時ファイルを用いた安全な書き込みやエラー時のクリーンアップ処理を制御する役割を持ちます。

なぜ重要か

WebDAV リソースへの書き込み処理において、直接対象ファイルを書き換えるのではなく、一時ファイル（temppath）へ書き込みを行い、正常終了時にアトミックにリネーム（コミット）する仕組みを提供します。これにより、書き込み途中のエラー発生時に不完全なファイルが残ることを防ぎ、データの整合性を保つために極めて重要です。

関与するモジュール

  • module:modules/dav/fs (ファイルシステムベースのリポジトリプロバイダ)
  • module:modules/dav/main (WebDAV コアモジュール)

主要エンティティ

  • **symbol:modules/dav/fs/repos.c#dav_stream**

    • ファイルシステムリポジトリにおけるストリーム状態を保持する構造体。以下のフィールドを持ちます。

  • apr_pool_t *p: 操作に関連付けられたメモリプール。
  • apr_file_t *f: 現在開いているファイルハンドル。
  • const char *pathname: 対象ファイルの本来のパス。
  • char *temppath: 書き込み用の一時ファイルパス。
  • int unlink_on_error: エラー発生時に一時ファイルを削除するかどうかを示すフラグ。
  • **symbol:modules/dav/main/mod_dav.h#dav_stream**

    • mod_dav コアとバックエンドプロバイダ間のインターフェースで言及されるストリーム型（詳細な構造は提供されたコンテキストからは未詳）。

エンドツーエンドの流れ

1. **ストリームのオープン**: mod_dav コアからフック経由で dav_fs_open_stream が呼び出され、dav_stream 構造体が初期化されます。 2. **データの書き込み/シーク**: dav_fs_write_stream や dav_fs_seek_stream を通じて、APR (Apache Portable Runtime) のファイル I/O 関数群（apr_file_t）を用いた操作が行われます。 3. **ストリームのクローズとコミット**: dav_fs_close_stream が呼び出されます。

  • **正常終了（コミット時）**: temppath が設定されている場合、本来のパス（pathname）へのリネーム処理が行われます。
  • **異常終了（エラー時）**: unlink_on_error が真であれば、一時ファイルが削除されます。

※なお、mod_dav コア側（mod_dav.h）における具体的な呼び出しライフサイクルや、他プロバイダとの共通フローの全容は提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **メモリプールの生存期間**: apr_pool_t *p は、関連する WebDAV リクエストのライフサイクルと同期していなければなりません。
  • **コミット時のリネーム**: 書き込み操作において temppath が設定されている場合、dav_fs_close_stream でのコミット時に必ずリネーム処理が行われなければなりません。
  • **エラー時のクリーンアップ**: ファイルオープン失敗や書き込みエラーが発生し、かつ unlink_on_error が真である場合、クローズ処理時に一時ファイルが適切に破棄されなければなりません。

よくある落とし穴

  • **クローズ処理の欠落**: dav_stream のクリーンアップや一時ファイルのリネーム・削除は dav_fs_close_stream の呼び出しに依存しています。ストリームが適切に閉じられない場合、一時ファイルが残存したり、リソースリークが発生したりする可能性があります。
  • **内部構造への依存**: 本構造体は主に repos.c 内で完結する内部実装です。フィールドを変更する際は、dav_fs_close_stream 内のコミットおよびクリーンアップロジックへの影響を慎重に確認する必要があります。
