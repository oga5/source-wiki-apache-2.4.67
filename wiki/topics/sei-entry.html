sei_entry

概念

sei_entry は、Apache HTTP Server において特定の条件（リクエストヘッダ、属性、式など）に基づいて環境変数を動的に設定・操作するためのルールを保持するデータ構造です。

この名前の構造体は、mod_setenvif モジュールと mod_proxy_fcgi モジュールの両方に個別に定義されており、それぞれ異なる形状と目的を持っています。

なぜ重要か

HTTP リクエストの属性や条件式に応じて環境変数を制御することは、柔軟なリクエスト処理やバックエンド（FastCGI など）への適切なパラメータ伝達において不可欠です。sei_entry は、これらの条件付き環境変数設定（SetEnvIf や ProxyFCGISetEnvIf など）の評価ルールをメモリ上に保持し、実行時に高速にマッチング・適用するための基盤となります。

関与するモジュール

  • module:modules/metadata
  • module:modules/proxy

主要エンティティ

  • **symbol:modules/metadata/mod_setenvif.c#sei_entry**: SetEnvIf ディレクティブの条件ルール（ヘッダ名、正規表現、式、設定する環境変数テーブルなど）を保持する構造体。
  • **symbol:modules/proxy/mod_proxy_fcgi.c#sei_entry**: ProxyFCGISetEnvIf などのディレクティブ向けに、条件式（cond）、置換式（subst）、環境変数名（envname）を保持する構造体。

エンドツーエンドの流れ

両モジュールを跨ぐ統一されたエンドツーエンドの連携フローや、直接的なバイナリレベルでの相互作用の詳細は完全には明らかになっていません。各モジュール内における個別の処理の流れは以下の通りです。

1. mod_setenvif における流れ

  • **設定フェーズ**: add_setenvif_core 関数が SetEnvIf ディレクティブの引数を解析し、sei_entry 構造体を生成・初期化します。正規表現は ap_pregcomp で、単純文字列は apr_strmatch_precompile で事前コンパイルされます。
  • **実行フェーズ**: リクエスト処理の初期段階で match_headers 関数が呼び出され、sei_entry の各ルール（name, regex, expr など）をリクエスト情報と照合します。マッチした場合、features テーブルに基づいてリクエストの subprocess_env が更新されます。

2. mod_proxy_fcgi における流れ

  • **設定フェーズ**: ProxyFCGISetEnvIf 等のディレクティブにより、条件式（cond）や置換式（subst）を含む sei_entry が生成され、配列（apr_array_header_t）に格納されます。
  • **実行フェーズ**: リクエスト処理時にリストを走査し、cond が真と評価された場合に subst を評価して envname に値を割り当てます。

重要な不変条件

  • **mod_setenvif の不変条件**:
  • 正規表現マッチングを使用する場合、preg には ap_pregcomp でコンパイルされた有効なオブジェクトが格納されていなければなりません。
  • 高速文字列マッチングを使用する場合、pattern に apr_strmatch_precompile で生成されたパターンが格納されている必要があります。
  • **mod_proxy_fcgi の不変条件**:
  • cond および subst は、Apache の式評価エンジン（ap_expr）によってコンパイルされた有効な ap_expr_info_t である必要があります。
  • **モジュール間の不変条件**:
  • 同名の構造体ですが、定義されているフィールドや形状が異なります。両者間の直接的なバイナリ互換性や連携ロジックの詳細は未確定です。

よくある落とし穴

  • **名前の衝突とバイナリ互換性**: mod_setenvif.c と mod_proxy_fcgi.c の両方で sei_entry という同名の構造体が異なる定義で存在しています。mod_proxy_fcgi.c における定義は mod_setenvif のロジックを模倣する目的で配置されているとされていますが、修正を行う際は両モジュール間でのバイナリ互換性や定義の乖離に注意する必要があります。
  • **評価のタイミング**: mod_setenvif の SetEnvIf はリクエスト処理の初期段階で実行されるため、後続のモジュールが動的に生成したヘッダに対してはマッチングを行えません。
  • **コンパイルエラー**: 設定フェーズでの正規表現や式のコンパイルに失敗した場合、Apache の起動（設定読み込み）時にエラーが発生します。
