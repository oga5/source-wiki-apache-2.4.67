sei_entry

概念

sei_entry は、Apache HTTP Server において特定の条件に基づいて環境変数を動的に設定・管理するための構造体です。この名前の構造体は mod_setenvif モジュールと mod_proxy_fcgi モジュールの両方に定義されており、それぞれ異なる形状を持ちながらも、条件付き環境変数設定という共通の目的のために使用されています。

なぜ重要か

リクエストのヘッダや属性、あるいはカスタムの条件式（ap_expr）を評価し、それに応じて環境変数を動的に操作する仕組みを提供するためです。これにより、後続の処理やバックエンド（FastCGI など）へのリクエスト転送時に適切な環境変数を引き渡すことが可能になります。また、同名の構造体が異なるモジュールで定義・参照されているため、モジュール間の連携やバイナリ互換性の観点からも重要です。

関与するモジュール

  • module:modules/metadata (mod_setenvif.c)
  • module:modules/proxy (mod_proxy_fcgi.c)

主要エンティティ

  • **symbol:modules/metadata/mod_setenvif.c#sei_entry**

    • SetEnvIf ディレクティブによって定義された個別の条件ルールを保持する構造体です。ヘッダ名、正規表現（preg）、高速マッチング用パターン（pattern）、解析済み式（expr）、設定・解除する環境変数のテーブル（features）などを保持します。

  • **symbol:modules/proxy/mod_proxy_fcgi.c#sei_entry**

    • 環境変数の条件付き設定（ProxyFCGISetEnvIf 等）を管理するための構造体です。条件式（cond）、置換式（subst）、環境変数名（envname）を保持します。

エンドツーエンドの流れ

1. **設定フェーズ**:

  • **mod_setenvif.c**: add_setenvif_core 関数により、ディレクティブから解析された情報が sei_entry 構造体に格納され、sei_cfg_rec 内の配列として保持されます。
  • **mod_proxy_fcgi.c**: 設定解析時に、sei_entry のリスト（apr_array_header_t）が生成されます。

    • 2. **実行フェーズ**:

  • **mod_setenvif.c**: リクエスト処理の初期段階で match_headers 関数が呼び出され、sei_entry の各フィールドを用いてリクエスト情報を検査します。apr_strmatch や ap_regexec による判定が成功すると、features に基づいて subprocess_env テーブルが更新されます。
  • **mod_proxy_fcgi.c**: リクエスト処理時に sei_entry のリストを順次走査し、条件式 cond が真と評価された場合に subst を評価して envname に値を割り当てます。

※なお、mod_proxy_fcgi.c の sei_entry が mod_setenvif.c 内の関数群（add_envvars 等）からどのように直接参照・連携されているか、その詳細なエンドツーエンドの呼び出しフローの全容は完全には明らかになっていません。

重要な不変条件

  • **正規表現の有効性**: mod_setenvif.c の sei_entry で正規表現を使用する場合、preg には ap_pregcomp でコンパイルされた有効なオブジェクトが格納されていなければなりません。
  • **高速マッチングの排他性**: 単純文字列マッチングの場合は、pattern に apr_strmatch_precompile で生成されたパターンが格納されます。
  • **式の有効性**: mod_proxy_fcgi.c の sei_entry における cond および subst は、Apache の式評価エンジン（ap_expr）によってコンパイルされた有効な式情報である必要があります。

よくある落とし穴

  • **同名異構造による混乱と互換性**: mod_setenvif.c と mod_proxy_fcgi.c で同名の sei_entry 構造体が定義されていますが、メンバ構成が異なります。mod_proxy_fcgi.c における定義は mod_setenvif のロジックを再利用または模倣する目的で配置されているため、修正を行う際は両モジュール間でのバイナリ互換性に注意する必要があります。
  • **評価タイミングの制約**: SetEnvIf はリクエスト処理の初期段階で実行されるため、後続のモジュールが動的に生成したヘッダに対してはマッチングを行えません。
  • **コンパイルエラー**: 設定フェーズでの正規表現や式のコンパイルに失敗した場合、Apache の設定読み込み時にエラーが発生し起動に失敗します。
