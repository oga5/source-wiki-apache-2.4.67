core_dir_config

概念

core_dir_config は、Apache HTTP Server におけるディレクトリ単位の設定情報を保持するための中心的なデータ構造です。Options、AllowOverride、フィルタ設定、リソース制限、アクセス制御ルールなど、リクエスト処理の基盤となる設定状態を管理します。

なぜ重要か

Apache HTTP Server では、ディレクトリ階層ごとに異なる設定（<Directory> ブロックや .htaccess ファイルなど）を適用する必要があります。core_dir_config は、これらのディレクトリ単位の設定をメモリ上で表現し、親ディレクトリから子ディレクトリへの設定継承やマージ処理を可能にするための基盤として機能します。

関与するモジュール

  • module:include
  • module:server

主要エンティティ

  • **symbol:include/http_core.h#core_dir_config**: ディレクトリ単位の設定情報を保持する構造体定義。
  • **symbol:server/core.c#create_core_dir_config**: core_dir_config 構造体をメモリ上に確保し、デフォルト値で初期化する関数。

エンドツーエンドの流れ

1. **メモリ確保と初期化**: ディレクトリ単位の設定が必要になると、create_core_dir_config が呼び出されます。この関数は apr_pool_t から apr_pcalloc を用いて core_dir_config のメモリを確保し、各種フィールドを未設定状態を示す定数（_UNSET など）やデフォルト値で初期化します。 2. **設定の適用とマージ**: 設定ファイルの解析やリクエスト処理の過程において、親ディレクトリの設定と子ディレクトリの設定がマージされます。

※なお、設定ファイル解析からリクエスト処理時に至る完全なエンドツーエンドの呼び出しフローや、具体的なマージ関数（merge_core_dir_configs など）の内部実装の詳細は、提供されたモジュール情報の範囲内では完全には特定されていません。

重要な不変条件

  • **オプションの排他性**: opts_add と opts_remove は排他的でなければなりません（opts_add & opts_remove == 0）。各ビットはどちらか一方にのみ記録され、相対的なオプション変更の最終状態を保持します。
  • **初期化時のデフォルト値**:
  • apr_pcalloc によるメモリ確保が行われるため、明示的に初期化されないメンバは NULL または 0 になります。
  • dir 引数が NULL の場合、opts の初期値は OPT_UNSET|OPT_SYM_LINKS となり、dir が指定されている場合は OPT_UNSET となります。

よくある落とし穴

  • **初期化の欠落**: core_dir_config に新しい設定項目を追加した際、create_core_dir_config で適切な _UNSET 値やデフォルト値による初期化を怠ると、マージ処理時に予期しない動作を引き起こす原因になります。
  • **バイナリ互換性（MMN）**: 本構造体にフィールドを追加・変更すると、モジュールバイナリ互換性（MMN）のバージョン更新が必要になる可能性があります。
  • **未使用フィールド**: response_code_strings フィールドは定義されていますが、現在は使用されていません。
  • **プラットフォーム依存性**: リソース制限（rlimit）などの一部のフィールドはプラットフォーム依存の定義に依存するため、環境によって構造体のサイズや有効性が異なる場合があります。

次に読む場所

  • **include/http_core.h**: core_dir_config 構造体の詳細なフィールド定義や、関連するビットマスク型（allow_options_t など）を確認できます。
  • **server/core.c**: create_core_dir_config における各フィールドの初期化処理の具体的な実装を確認できます。
