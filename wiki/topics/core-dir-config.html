core_dir_config

概念

core_dir_config は、Apache HTTP Server においてディレクトリ単位の設定情報を保持するための中心的な構造体です。Options、AllowOverride、フィルタ設定、リソース制限、アクセス制御ルールなど、リクエスト処理の基盤となる設定状態を管理します。

なぜ重要か

Apache HTTP Server の柔軟なディレクトリ単位の設定（<Directory> や .htaccess など）を実現するための基盤データ構造だからです。親ディレクトリから子ディレクトリへの設定の継承やマージにおいて中心的な役割を果たします。また、この構造体の変更はモジュールバイナリ互換性（MMN: Module Magic Number）に直接影響を与えるため、コア開発において極めて重要な意味を持ちます。

関与するモジュール

  • module:include （ヘッダファイル include/http_core.h を提供）
  • module:server （コア実装 server/core.c を提供）

主要エンティティ

  • symbol:include/http_core.h#core_dir_config

    • ディレクトリ単位の設定情報を保持する構造体定義。

  • symbol:server/core.c#create_core_dir_config

    • core_dir_config 構造体をメモリ上に確保し、デフォルト値で初期化する関数。

エンドツーエンドの流れ

1. **初期化**: ディレクトリ単位の設定構造体が必要になると、server/core.c 内で create_core_dir_config が呼び出されます。この関数は apr_pcalloc を用いてメモリを確保し、各種フラグや制限値を未設定状態を示す定数（_UNSET など）やデフォルト値で初期化します。 2. **設定の適用とマージ**: 設定ファイル（httpd.conf 等）の解析結果や親ディレクトリからの継承設定に基づき、ディレクトリ単位の設定マージ（directory_walk 等）が行われ、親から子へ設定が継承されます。 3. **リクエスト処理時の参照**: リクエスト処理時に core モジュールや各モジュールがこの設定値を参照して動作を決定します。

*注意: 設定ファイルの解析からリクエスト処理に至る完全なエンドツーエンドの制御フローの全容は、提供されたソースコード抜粋からは完全には特定されていません。*

重要な不変条件

  • **オプションの排他性**: opts_add と opts_remove は排他的である必要があります（opts_add & opts_remove == 0）。各ビットはどちらか一方にのみ記録され、相対的なオプション変更の最終状態を保持します。
  • **初期値の条件分岐**: create_core_dir_config において、dir 引数が指定されている場合、conf->opts は OPT_UNSET に初期化されます。指定がない（NULL の）場合は OPT_UNSET|OPT_SYM_LINKS が適用されます。
  • **メモリ確保の前提**: apr_pcalloc を使用してメモリを確保するため、構造体のメンバは明示的に初期化されない限り NULL または 0 で初期化されます。

よくある落とし穴

  • **バイナリ互換性（MMN）の破壊**: core_dir_config 構造体に新しいフィールドを追加すると、include/ap_mmn.h で定義されるモジュールバイナリ互換性（MMN）のバージョン更新が必要になります。サードパーティ製モジュールとの互換性を壊す可能性があるため、フィールドの追加は慎重に行う必要があります。
  • **未設定状態のハンドリング**: max_ranges などの数値設定において、AP_MAXRANGES_UNSET などの定数による未設定状態の適切なハンドリングを怠ると、意図しないデフォルト値が適用される恐れがあります。
  • **プラットフォーム依存性**: リソース制限（rlimit）はプラットフォーム依存の定義（RLIMIT_CPU 等）に依存するため、環境によって構造体のサイズや有効性が異なる場合があります。
  • **非推奨フィールド**: response_code_strings は構造体内に存在しますが、現在は使用されていません。
  • **初期化漏れ**: 新しい設定項目を core_dir_config に追加した際、create_core_dir_config で適切な初期化（_UNSET 値など）を行わないと、後続のマージ処理（merge_core_dir_configs など）で未定義の挙動を示す原因になります。

次に読む場所

  • include/http_core.h

    • core_dir_config 構造体の詳細なフィールド定義や、関連するビットマスク（allow_options_t など）を確認できます。

  • server/core.c

    • create_core_dir_config の具体的な初期化ロジックや、設定値がどのようにデフォルト設定されるかを確認できます。
