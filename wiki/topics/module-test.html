Module: test

概念

test ディレクトリは、Apache HTTP Server (httpd) の機能検証およびリグレッションテストを行うための統合テストスイートです。主に Python の pytest フレームワークを利用した統合テスト群（test/modules/）と、C言語で記述された低レベルのユニットテストやユーティリティ（test/test_parser.c など）で構成されています。

なぜ重要か

httpd のコア機能（core）、HTTP/1.1（http1）、HTTP/2（http2）、プロキシ（proxy）、および ACME/Let's Encrypt 連携を行う Managed Domains（md）などの各モジュールが、プロトコル仕様や期待される挙動通りに動作するかを自動検証するために不可欠です。テスト専用の Apache モジュール（mod_h2test など）を動的にロードして、異常系や特殊なプロトコル挙動のシミュレーションを可能にしています。

関与するモジュール

  • module:test (ルートテストディレクトリ)
  • module:test/clients (テスト用クライアントプログラム)
  • module:test/modules (各機能モジュールごとの pytest テスト)
  • module:test/pyhttpd (Python テストフレームワークのヘルパー群)

主要エンティティ

  • **テスト環境管理 (test/pyhttpd/env.py, test/conftest.py)**: テスト用 Apache インスタンスの起動、設定生成、証明書管理（certs.py）などを抽象化します。
  • **テスト用カスタムモジュール**:
  • test/modules/http2/mod_h2test/mod_h2test.c
  • test/modules/http1/mod_h1test/mod_h1test.c
  • test/pyhttpd/mod_aptest/mod_aptest.c

    • これらはテスト中にのみ httpd にロードされ、特殊なヘッダー処理やエラー注入を行います。

  • **テスト用クライアント**: test/clients/h2ws.c (HTTP/2 WebSocket 検証用)
  • **CGI テストスクリプト**: test/modules/http2/htdocs/cgi/hello.py や upload.py など、リクエスト処理の検証用 CGI。
  • **C言語ユニットテスト**: test/test_parser.c, test/test_limits.c などのスタンドアロンな検証プログラム。

エンドツーエンドの流れ

1. **テストの初期化**: pytest 実行時、test/conftest.py および各モジュールの conftest.py が読み込まれ、テスト環境（pyhttpd/env.py）がセットアップされます。 2. **サーバー起動**: テスト用の設定ファイル（conf.py）や自己署名証明書（certs.py）が動的に生成され、テスト用ポートで httpd プロセスが起動します。 3. **リクエスト送信と検証**: テストコード（例: test_003_get.py）から pyhttpd/curl.py や pyhttpd/nghttp.py を介してリクエストが送信され、CGI やテスト用モジュールがそれを処理します。 4. **アサーション**: 受信したレスポンスのステータス、ヘッダー、ボディが期待値と一致するかを pytest が検証します。

*注意: C言語単体テスト（test_parser.c など）のビルド・実行に関する詳細なエンドツーエンドの連携フローは、提供されたソースメタデータからは完全には特定されていません。*

重要な不変条件

  • **テスト環境の隔離**: テスト用 httpd インスタンスは、本番環境に影響を与えないよう、動的に割り当てられたポートと一時ディレクトリ（pyhttpd/env.py）を使用しなければなりません。
  • **テスト専用モジュールの非公開**: mod_h2test などの検証用モジュールは、テスト実行時のみロードされ、本番のビルドやデプロイメントに含まれてはなりません。
  • *注意: その他の詳細な内部不変条件（メモリ管理やプロセス間同期など）は、提供されたファイル一覧からは直接特定されていません。*

よくある落とし穴

  • **外部依存の欠如**: テスト実行環境に curl、nghttp2、または必要な Python パッケージがインストールされていない場合、多くのテストが暗黙的にスキップされるか失敗します。
  • **CGI 実行エラー**: テスト用 CGI（hello.py など）が実行権限の不足や Python パスの不整合により 500 Internal Server Error を返し、テストが失敗することがあります。
  • **ポート競合**: テスト用 httpd が使用しようとするポートが既に他のプロセスによって占有されている場合、サーバーの起動に失敗します。
