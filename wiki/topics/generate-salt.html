generate_salt

概念

generate_salt は、Apache HTTP Server（httpd-2.4.67）において、パスワードハッシュの生成に使用されるランダムなソルト文字列を生成するための関数です。この関数は、Luaモジュール（modules/lua）およびパスワード管理サポートツール（support）の双方で、それぞれ類似したシグネチャとロジックを持つ実装として存在しています。

なぜ重要か

安全なパスワードハッシュを生成するためには、暗号論的に安全な疑似乱数を用いたソルトが不可欠です。generate_salt は、APR（Apache Portable Runtime）ライブラリの apr_generate_random_bytes を利用して乱数を取得し、それをハッシュ生成に適したBase64風の文字列（itoa64文字セット）に変換する役割を担っています。これにより、パスワードハッシュの安全性が担保されます。

関与するモジュール

  • module:modules/lua
  • module:support

主要エンティティ

  • symbol:modules/lua/lua_passwd.c#generate_salt: Luaスクリプトからのパスワードハッシュ生成（mk_password_hashなど）で利用される静的（static）関数。
  • symbol:support/passwd_common.c#generate_salt: パスワード管理ユーティリティ（htpasswdなどの背後にある mkhash など）で共通して利用される関数。

エンドツーエンドの流れ

1. **呼び出し**: 呼び出し元（modules/lua/lua_passwd.c の mk_password_hash や、support/passwd_common.c の mkhash など）が、必要なソルトの長さ（size）と格納用バッファ（s）を指定して generate_salt を呼び出します。 2. **バイト数計算**: 指定された size に基づいて、必要な乱数バイト数 n を算出します。 3. **乱数生成**: apr_generate_random_bytes を呼び出し、乱数データを取得します。 4. **Base64風エンコード**: 取得した乱数データを、itoa64 テーブル（./0-9A-Za-z または ./0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz）を用いて文字に変換し、バッファ s に格納します。 5. **終端処理**: 文字列の末尾にヌル文字（\0）を付与して終了します。

※なお、これら呼び出し元（mk_password_hash や mkhash）より上位の、システム全体におけるエンドツーエンドの完全な制御フロー（例えば、どのようなHTTPリクエストやコマンドライン引数がトリガーとなるかなど）の詳細は、提供されたコンテキストからは未詳です。

重要な不変条件

  • **バッファサイズ制限**: 乱数を一時的に格納する内部バッファ rnd のサイズは 32 バイトに固定されています。そのため、size から計算される必要バイト数 n が 32 バイトを超えてはなりません。
  • **バッファの確保**: 呼び出し元が渡すバッファ s は、少なくとも size + 1 バイト以上の領域を確保している必要があります。
  • **乱数生成の成功**: apr_generate_random_bytes が正常に乱数を取得できる必要があります。

よくある落とし穴

  • **バッファオーバーフロー時の挙動の差異**:
  • modules/lua/lua_passwd.c の実装では、要求されたサイズが内部バッファ（32バイト）を超えた場合にエラー（ERR_RANDOM）を返します。
  • 一方、support/passwd_common.c の実装では、サイズ超過時に abort() を呼び出してプロセスを即座に強制終了します。
  • **グローバル変数への依存**: support/passwd_common.c の実装は、エラー出力のためにグローバル変数 errfile を使用するため、呼び出し前にこれが適切に初期化されている必要があります。
  • **非標準Base64**: 使用される itoa64 変換テーブルは標準的なBase64とは異なるため、他の一般的なBase64デコーダとの互換性はありません。
