generate_salt

概念

generate_salt は、Apache HTTP Server（httpd-2.4.67）において、パスワードハッシュ生成時に使用されるランダムなソルト（Salt）文字列を生成するための内部ユーティリティ関数です。この関数は、APR（Apache Portable Runtime）ライブラリの乱数生成機能を利用し、取得した乱数を特定のBase64風文字セット（itoa64）を用いてエンコードし、指定されたバッファに格納します。

なぜ重要か

パスワードの安全なハッシュ化には、レインボーテーブル攻撃などの事前計算攻撃を防ぐための「ソルト」が不可欠です。generate_salt は、暗号論的に安全な乱数（apr_generate_random_bytes 経由）から一意で予測不可能なソルトを一貫した方法で生成することにより、support モジュール（htpasswdなどのツール）や modules/lua モジュール（Luaスクリプトによる認証処理など）におけるパスワードハッシュの安全性を担保する役割を担っています。

関与するモジュール

  • module:modules/lua
  • module:support

主要エンティティ

  • symbol:modules/lua/lua_passwd.c#generate_salt : Luaモジュール内でのパスワードハッシュ生成（mk_password_hash など）に使用される静的（static）関数。
  • symbol:support/passwd_common.c#generate_salt : 共通のパスワード管理ツール（mkhash など）で広く使用される関数。

エンドツーエンドの流れ

1. **呼び出し**: パスワードハッシュ生成処理（modules/lua/lua_passwd.c の mk_password_hash や support/passwd_common.c の mkhash など）から、必要なソルトの長さ（size）を指定して generate_salt が呼び出されます。 2. **バイト数計算**: 要求された size に基づいて、必要な乱数バイト数 n を算出します。 3. **乱数生成**: APRライブラリの apr_generate_random_bytes を呼び出し、暗号論的に安全なランダムバイト列を取得します。 4. **Base64風エンコード**: 取得したバイト列を、itoa64 テーブル（./0-9A-Za-z）を用いてBase64風の文字列に変換します。 5. **格納と終端**: 変換された文字列をバッファ s に書き込み、末尾にヌル文字（\0）を付与して呼び出し元に返します。

※なお、呼び出し元（mk_password_hash や mkhash）のさらに上位に位置する、システム全体における完全なエンドツーエンドの認証フローや、これらがどのように統合されて動作するかについての詳細な全体像は、提供されたコンテキストからは未だ完全には判明していません。

重要な不変条件

  • **バッファサイズ制限**: 生成に必要な乱数バイト数 n は、内部の一時バッファ rnd のサイズ（32バイト）を超えてはなりません。
  • modules/lua/lua_passwd.c の実装では、これを超えた場合に ERR_RANDOM エラーを返します。
  • support/passwd_common.c の実装では、これを超えた場合に abort() を呼び出してプロセスを即座に終了させます。
  • **出力バッファの確保**: 呼び出し元は、s に対して size + 1 以上のメモリ領域を確保しておく必要があります。

よくある落とし穴

  • **バッファ不足時のアボート**: support/passwd_common.c の実装では、要求するソルトのサイズが内部制限（32バイト）を超えると、エラーを返すのではなく abort() によりプロセスが強制終了します。呼び出し側でのサイズ指定には注意が必要です。
  • **非標準のBase64エンコード**: 変換に使用される itoa64 テーブル（./0-9A-Za-z）は、標準的なBase64（RFC 4648等）とは文字順序や文字セットが異なります。このため、一般的なBase64デコーダで直接デコードすることはできません。
  • **グローバル変数の依存**: support/passwd_common.c の実装は、エラー出力のために errfile グローバル変数に依存しています。呼び出し前にこれが適切に初期化されている必要があります。
