generate_salt

概念

generate_salt は、パスワードハッシュ生成に使用される、Base64風にエンコードされたランダムなソルト文字列を生成する関数です。Apache HTTP Serverの複数のモジュール（Luaモジュールおよびサポートツール用の共通パスワード処理）において、それぞれ同名の関数として定義・利用されています。

なぜ重要か

パスワードを安全にハッシュ化するためには、レインボーテーブルなどの事前計算攻撃を防ぐためのランダムなソルトが不可欠です。この関数は、APR（Apache Portable Runtime）の安全な乱数生成機能（apr_generate_random_bytes）を利用し、ハッシュ生成アルゴリズムと互換性のある itoa64 テーブルを用いたソルト文字列を生成する共通の仕組みを提供します。

エンドツーエンドの流れ

1. **呼び出し**: 呼び出し元（modules/lua/lua_passwd.c の mk_password_hash や、support/passwd_common.c の mkhash など）が、生成するソルトの長さ（size）を指定して generate_salt を呼び出します。 2. **乱数生成**: 指定された size から必要なバイト数 n を算出し、apr_generate_random_bytes を用いて乱数を取得します。 3. **エンコード**: 取得した乱数を itoa64 文字列（./0-9A-Za-z）を用いて Base64 風の文字に変換します。 4. **出力**: 変換された文字をバッファ s に格納し、ヌル終端（\0）します。

*注意: 呼び出し元からハッシュ生成、そして最終的なパスワード検証や保存に至るシステム全体のエンドツーエンドの完全なフロー（例えば、どの設定やリクエストがトリガーとなってこれらの関数が呼び出されるかなど）の詳細は、提供されたコンテキストからは完全には明らかになっていません。*

重要な不変条件

  • **バッファサイズ制限**: size に基づいて計算される必要なバイト数が、内部バッファ（rnd 配列）のサイズである 32 バイトを超えてはなりません。
  • **出力バッファの確保**: 呼び出し元が渡すバッファ s は、少なくとも size + 1 バイト以上の領域を確保している必要があります。
  • **乱数生成の成功**: apr_generate_random_bytes が正常に乱数を取得できる必要があります。

よくある落とし穴

  • **サイズ超過時の挙動の違い**:
  • modules/lua/lua_passwd.c の実装では、size が大きすぎて内部バッファ（32バイト）に収まらない場合、エラー（ERR_RANDOM）を返してエラーメッセージを設定します。
  • 一方、support/passwd_common.c の実装では、同様のバッファ不足が発生した場合に abort() を呼び出してプロセスを強制終了します。
  • **グローバル変数への依存**: support/passwd_common.c の実装は errfile グローバル変数を使用するため、呼び出し側で適切に初期化されている必要があります。
  • **非標準のBase64エンコーディング**: 使用される itoa64 テーブルは標準的な Base64 とは異なるため、変更時や外部連携時にはハッシュ生成アルゴリズムとの整合性に注意が必要です。
