generate_salt

概念

generate_salt は、Apache HTTP Server（httpd）において、パスワードハッシュを生成する際に使用されるランダムなソルト文字列を生成するためのユーティリティ関数です。APR（Apache Portable Runtime）の乱数生成機能を利用し、Base64風の文字セット（itoa64）でエンコードされたソルトを生成します。

なぜ重要か

セキュアなパスワードハッシュ（MD5やSHAなど）を生成するためには、予測不可能なソルトが不可欠です。generate_salt は、暗号論的に安全な乱数生成器（apr_generate_random_bytes）をラップし、ハッシュアルゴリズムと互換性のある形式（itoa64）の文字列に変換する共通の手段を提供することで、パスワード管理機能（Luaモジュールやコマンドラインツールなど）のセキュリティを支えています。

関与するモジュール

  • module:modules/lua
  • module:support

主要エンティティ

  • symbol:modules/lua/lua_passwd.c#generate_salt: Luaモジュール内部で定義・使用される静的（static）関数。
  • symbol:support/passwd_common.c#generate_salt: サポートツール群で共通して使用される関数。

エンドツーエンドの流れ

1. **呼び出し**: modules/lua/lua_passwd.c の mk_password_hash や、support/passwd_common.c の mkhash などの上位関数から、必要なソルト長（size）を指定して generate_salt が呼び出されます。 2. **乱数生成**: 指定された size から必要なバイト数 n を算出し、apr_generate_random_bytes を用いて乱数バイト列を取得します。 3. **エンコード**: 取得した乱数バイト列を、itoa64 テーブル（./0-9A-Za-z）を用いて6ビットずつ抽出し、Base64風の文字列に変換します。 4. **出力**: 変換された文字列をバッファ s に格納し、ヌル終端（\0）します。

※ なお、システム全体における具体的な呼び出し元の全容や、Luaスクリプトからこの関数に至るまでの完全なエンドツーエンドの制御フローは、現時点のコンテキストからは完全には明らかになっていません。

重要な不変条件

  • **バッファサイズ制限**: size に基づいて計算される必要なバイト数は、内部の一時バッファ rnd のサイズ（32バイト）を超えてはなりません。
  • **バッファの確保**: 呼び出し元が渡すバッファ s は、少なくとも size + 1 バイト以上の領域が確保されている必要があります。

よくある落とし穴

  • **バッファオーバーフロー時の挙動の違い**: support/passwd_common.c の実装では、要求された size に対応するバイト数が32バイトを超えた場合、エラーを返すのではなく abort() を呼び出してプロセスを強制終了します。一方、modules/lua/lua_passwd.c の実装では ERR_RANDOM を返します。コンテキストによってエラーハンドリングの挙動が異なる点に注意が必要です。
  • **グローバル変数の依存**: support/passwd_common.c の実装は errfile グローバル変数を使用するため、呼び出し側でこれが適切に初期化されている必要があります。
  • **非標準のBase64**: 使用される itoa64 テーブルは標準的なBase64（RFC 4648等）とは文字順序が異なるため、一般的なBase64デコーダで直接デコードすることはできません。

次に読む場所

  • modules/lua/lua_passwd.c: Lua環境におけるパスワードハッシュ生成と generate_salt の利用方法を確認できます。
  • support/passwd_common.c: コマンドラインツール等で共有される generate_salt の実装と、mkhash での利用方法を確認できます。
