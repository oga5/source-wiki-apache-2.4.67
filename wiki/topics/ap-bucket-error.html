ap_bucket_error

概念

ap_bucket_error は、Apache HTTP Server のフィルタスタック内において、エラー状態（HTTPステータスコードなど）を伝播させるための特殊なメタデータバケット型です。フィルタ処理中に問題が発生した際、このバケットをバケットブリゲード（apr_bucket_brigade）に挿入することで、後続のフィルタ処理をバイパスし、エラーレスポンス生成処理へとエラー情報を伝達します。

なぜ重要か

リクエスト処理やフィルタチェーンの実行中にエラーが発生した場合、通常のコンテンツ処理を中断し、クライアントへ適切なHTTPエラーレスポンスを安全に返す必要があります。ap_bucket_error を使用することで、エラーコードとエラーメッセージをカプセル化したバケットをストリーム内に流し、安全かつ一貫した方法でエラーハンドリングをトリガーできます。

関与するモジュール

  • module:include （ヘッダー定義の提供）
  • module:server （エラーバケットの生成・操作ロジックの実装）

主要エンティティ

  • **symbol:include/http_protocol.h#ap_bucket_error**

    • エラー状態を伝播させるためのバケット構造体です。ヘッダー内では typedef struct ap_bucket_error ap_bucket_error; として前方宣言されています。

  • **symbol:server/error_bucket.c#ap_bucket_error_create**

    • HTTPステータスコード、エラーメッセージ、メモリプール、およびバケットアロケータを受け取り、初期化された新しいエラーバケット（apr_bucket）を生成して返す関数です。

エンドツーエンドの流れ

1. **エラーの発生とバケット生成**: フィルタ処理中（例: modules/http/http_filters.c や modules/http2/h2_c2_filter.c など）にエラーが検出されると、ap_bucket_error_create が呼び出され、HTTPステータスコードとエラーメッセージを格納したエラーバケットが生成されます。 2. **ブリゲードへの挿入**: 生成されたエラーバケットは、現在処理中の apr_bucket_brigade に挿入されます。 3. **バイパスと伝播**: 後続のフィルタは、このエラーバケットを検知すると通常のデータ処理をスキップし、エラー情報を下流へそのまま伝達します。 4. **エラーレスポンスの生成**: 最終的に、保持されているエラー状態（HTTPステータスコード等）に基づいて、クライアントへ適切なエラーレスポンスが生成・送信されます。

*注意: フィルタスタックのバイパスから最終的なエラーレスポンス生成に至る具体的な関数呼び出し関係や、詳細なハンドリングの全容は、提供されたソース抜粋からは完全には明らかになっていません。*

重要な不変条件

  • **ステータスコードの自動修正**:

    • ap_bucket_error_create に渡されたエラーコード（error）が有効なHTTPレスポンスコードでない場合（ap_is_HTTP_VALID_RESPONSE が偽を返す場合）、内部的に HTTP_INTERNAL_SERVER_ERROR（500）へ自動的に置換されます。

  • **有効なアロケータの要求**:

    • バケット生成時に渡される list 引数は、有効な apr_bucket_alloc_t インスタンスでなければなりません。

よくある落とし穴

  • **コピーと分割の非サポート**:

    • エラーバケットはメタデータバケットであり、コピーや分割（split）操作がサポートされていません。フィルタチェーン内でこれらの操作を試みないよう注意する必要があります。

  • **伝播の阻害**:

    • カスタムフィルタがブリゲード内のバケットを走査する際、エラーバケットを適切に処理（スルーまたはハンドリング）せずに破棄または無視してしまうと、エラー状態が正しく伝播されず、クライアントへ適切なエラーレスポンスが返されなくなる可能性があります。
