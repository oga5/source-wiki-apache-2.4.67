ap_bucket_error

概念

ap_bucket_error は、Apache HTTP Server のフィルタスタック内において、エラー状態（HTTPステータスコードやエラーメッセージなど）を伝播させるために設計された特殊なメタデータバケット型です。

なぜ重要か

フィルタ処理中に問題が発生した際、このバケットを apr_bucket_brigade に挿入することで、後続の通常のフィルタ処理をスキップ（バイパス）し、最終的に適切なエラーレスポンスを生成する処理へエラー情報を安全かつ確実に伝達できるためです。

関与するモジュール

  • module:include
  • module:server

主要エンティティ

  • **symbol:include/http_protocol.h#ap_bucket_error**:

    • フィルタスタック内でエラー状態を伝播させるための構造体（前方宣言）。

  • **symbol:server/error_bucket.c#ap_bucket_error_create**:

    • HTTPステータスコードとエラーメッセージを保持する、新しいエラーバケットを生成・初期化する関数。

エンドツーエンドの流れ

1. **エラーの発生とバケット生成**: フィルタ処理中（modules/http/http_filters.c や modules/http2/h2_bucket_beam.c など）にエラーを検出すると、ap_bucket_error_create を呼び出して、HTTPステータスコードとエラーメッセージを格納した apr_bucket を生成します。 2. **ブリゲードへの挿入と伝播**: 生成されたエラーバケットは apr_bucket_brigade に挿入され、フィルタスタックを流れます。 3. **エラーレスポンスの生成**: 後続のフィルタチェーンにおいて、このバケットが保持するエラー状態（HTTPステータスコード等）が参照され、最終的なエラーレスポンスの生成がトリガーされます。

*注意: フィルタチェーンが具体的にどのように後続処理をバイパスするのか、および最終的なエラーレスポンス生成処理との完全な結合フローの詳細は、提供された情報からは未解明です。*

重要な不変条件

  • **有効なHTTPステータスコードの保証**:

    • ap_bucket_error_create に渡された error が有効なHTTPレスポンスコードでない場合（ap_is_HTTP_VALID_RESPONSE が偽を返す場合）、内部的に HTTP_INTERNAL_SERVER_ERROR (500) に自動的に置換されます。

  • **有効なアロケータの要求**:

    • バケット生成時に渡される list（apr_bucket_alloc_t）は、有効なバケットアロケータでなければなりません。

よくある落とし穴

  • **コピー・分割の非サポート**:

    • エラーバケットはコピーや分割（split）をサポートしていません。フィルタチェーン内でこれらの操作を試みると、予期しない動作を引き起こす可能性があります。

  • **伝播の阻害**:

    • カスタムフィルタなどでエラーバケットが適切に処理・転送されずに破棄された場合、エラー状態が正しく伝播されず、クライアントへ適切なエラーレスポンスが返されなくなる可能性があります。
