close_worker_sockets

概念

close_worker_sockets は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）において、ワーカープロセスが保持しているすべてのアクティブなソケットを強制的にクローズし、リソースを解放するための関数です。

なぜ重要か

プロセスの強制終了処理時などに、開いたままのソケットを確実にクローズしてシステムリソースを解放し、ソケットのリークや不正な参照（二重解放など）を防ぐために重要です。

関与するモジュール

  • module:server/mpm/event
  • module:server/mpm/worker

主要エンティティ

  • symbol:server/mpm/worker/worker.c#close_worker_sockets : worker MPMにおける実装。
  • symbol:server/mpm/event/event.c#close_worker_sockets : event MPMにおける実装（worker とのアーキテクチャ上の類似性により参照・定義されています）。
  • worker_sockets : 各ワーカーが使用するソケットを格納する配列。
  • threads_per_child : 子プロセスあたりのスレッド数（ループの終了条件として使用）。

エンドツーエンドの流れ

1. 強制終了処理が発生すると、server/mpm/worker/worker.c または server/mpm/event/event.c から close_worker_sockets() が呼び出されます。 2. 0 から threads_per_child - 1 までのインデックスでループを実行します。 3. 各インデックス i について、worker_sockets[i] が有効（NULL ではない）かを確認します。 4. 有効な場合、APRライブラリ関数 apr_socket_close() を呼び出してソケットを閉じます。 5. クローズ後、worker_sockets[i] を NULL に設定し、二重解放や不正な参照を防ぎます。

※強制終了処理が開始されるトリガーや、プロセス全体のシャットダウンシーケンスにおける詳細なエンドツーエンドのフローの全容は、提供された情報からは完全には特定されていません。

重要な不変条件

  • threads_per_child が正しく初期化されている必要があります。
  • worker_sockets 配列が、各ワーカーのソケット状態を保持している必要があります。
  • ソケットをクローズした後は、二重解放や不正な参照を防ぐために、対応する worker_sockets[i] が必ず NULL に設定される必要があります。

よくある落とし穴

  • apr_socket_close 自体が失敗した場合のエラーハンドリングは、本関数内では行われません。
  • worker_sockets が初期化されていない場合や、既に閉じられたソケットに対しては、NULL チェックにより安全にスキップされますが、初期化状態の管理に不備があると予期しない挙動につながる可能性があります。
