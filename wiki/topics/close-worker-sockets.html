close_worker_sockets

概念

close_worker_sockets は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）である worker および event において、ワーカープロセスが保持しているすべてのソケットを強制的にクローズし、リソースを解放するための関数です。

なぜ重要か

この関数は、プロセスの強制終了処理時などに呼び出されます。アクティブなソケットを確実にクローズし、関連するシステムリソースを即座に解放することで、リソースリークを防ぎ、プロセスの安全な終了を保証するために重要です。

関与するモジュール

  • module:server/mpm/event
  • module:server/mpm/worker

主要エンティティ

  • symbol:server/mpm/worker/worker.c#close_worker_sockets: worker MPMにおける実装。グローバルな worker_sockets 配列を走査し、有効なソケットをクローズします。
  • symbol:server/mpm/event/event.c#close_worker_sockets: event MPMにおける実装（worker とのアーキテクチャ上の類似性により存在・参照されています）。

エンドツーエンドの流れ

1. **トリガー**: プロセスの強制終了処理が発生します。 2. **呼び出し**: server/mpm/worker/worker.c（強制終了処理時）または server/mpm/event/event.c（強制終了処理時）から close_worker_sockets() が呼び出されます。 3. **ループ処理**: 0 から threads_per_child - 1 までのインデックスでループを実行します。 4. **ソケットの確認**: 各インデックス i について、worker_sockets[i] が NULL でないか確認します。 5. **クローズ**: 有効なソケットが存在する場合、apr_socket_close() を呼び出してソケットを閉じます。 6. **クリア**: クローズ後、worker_sockets[i] を NULL に設定します。

※なお、強制終了処理の全体像や、event MPM側での具体的な呼び出しコンテキストなど、詳細なエンドツーエンドのフローは完全には提供されていません。

重要な不変条件

  • **前提条件**: threads_per_child が正しく初期化されていること、および worker_sockets 配列が各ワーカーのソケット状態を適切に保持していること。
  • **事後条件**: 処理された worker_sockets 配列の要素はすべて NULL に設定され、二重解放や不正な参照が防止されること。

よくある落とし穴

  • **エラーハンドリングの欠如**: apr_socket_close が失敗した場合、本関数内ではエラーハンドリングやログ記録を行っていません。
  • **未初期化状態での動作**: worker_sockets が初期化されていない場合や、既に閉じられているソケットに対しては NULL チェックにより安全にスキップされますが、初期化フローとの整合性に注意する必要があります。
