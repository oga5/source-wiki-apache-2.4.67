close_worker_sockets

概念

close_worker_sockets は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）である worker および event において、ワーカープロセスが保持しているすべてのソケットを強制的にクローズし、システムリソースを解放するための内部関数です。

なぜ重要か

プロセスの強制終了処理時などに、開いたままになっているソケットを確実にクローズすることで、ファイル記述子のリークやポートの占有を防ぎ、システムのクリーンな終了および再起動を保証するために重要です。

  • 各インデックス i において、worker_sockets[i] が NULL でない（有効なソケットが存在する）かを確認します。
  • 有効な場合、APR（Apache Portable Runtime）ライブラリの apr_socket_close() を呼び出してソケットを閉じます。

    • 4. **状態のクリア**: クローズ後、worker_sockets[i] に NULL を代入して参照をクリアします。

重要な不変条件

  • **初期化の前提**: 本関数の実行時、threads_per_child が正しく初期化されており、worker_sockets 配列が各ワーカーのソケット状態を正しく保持している必要があります。
  • **安全な初期化（NULLクリア）**: ソケットをクローズした後は、必ず worker_sockets[i] を NULL に設定することで、二重解放や不正なポインタ参照（ダングリングポインタ）を防止します。

よくある落とし穴

  • **エラーハンドリングの欠如**: apr_socket_close が失敗した場合、本関数内ではエラーハンドリングやログ出力を行いません。
  • **未初期化時の挙動**: worker_sockets が初期化されていない場合や、既に閉じられたソケットに対しては NULL チェックにより安全にスキップされますが、配列自体の境界外アクセスに対する保護は threads_per_child の正確性に依存します。
