close_worker_sockets

概念

close_worker_sockets は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）において、ワーカープロセスが保持しているすべてのアクティブなソケットを強制的にクローズし、リソースを解放するための関数です。

なぜ重要か

プロセスの強制終了処理時などに、開いたままのソケットを確実にクローズしてシステムリソースを解放し、リソースリークや不正なソケット参照（二重解放など）を防ぐために重要です。

関与するモジュール

  • module:server/mpm/event
  • module:server/mpm/worker

主要エンティティ

  • symbol:server/mpm/worker/worker.c#close_worker_sockets: worker MPMにおける実装。
  • symbol:server/mpm/event/event.c#close_worker_sockets: event MPMにおける実装（worker MPMとのアーキテクチャ上の類似性により存在・参照されます）。
  • worker_sockets: 各ワーカーが使用するソケットを格納する配列。
  • threads_per_child: 子プロセスあたりのスレッド数（ループの終了条件として使用）。

エンドツーエンドの流れ

1. 強制終了処理時（worker.c:331 または event.c:645）に、本関数が呼び出されます。 2. 0 から threads_per_child - 1 までのインデックスでループを実行します。 3. 各インデックス i について、worker_sockets[i] が有効（NULL ではない）かを確認します。 4. 有効な場合、apr_socket_close() を呼び出してソケットを閉じます。 5. クローズ後、worker_sockets[i] を NULL に設定し、二重解放や不正な参照を防ぎます。

※ 強制終了処理が開始される契機（シグナル受信や親プロセスからの指示など）に関する詳細なエンドツーエンドのフローは、提供された情報からは判明していません。

重要な不変条件

  • threads_per_child が正しく初期化されている必要があります。
  • worker_sockets 配列が、各ワーカーのソケット状態を正しく保持している必要があります。
  • ソケットのクローズ後、worker_sockets[i] は必ず NULL に設定され、再利用や二重解放が防止されます。

よくある落とし穴

  • apr_socket_close 自体が失敗した場合、本関数内ではエラーハンドリングやリトライは行われません。
  • worker_sockets が初期化されていない場合や、既に閉じられたソケットに対しては NULL チェックにより安全にスキップされますが、配列自体の整合性が崩れている場合の挙動は保証されません。
