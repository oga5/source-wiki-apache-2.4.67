close_worker_sockets

概念

close_worker_sockets は、Apache HTTP Serverのマルチプロセッシングモジュール（MPM）である worker および event において、ワーカープロセスが保持しているアクティブなソケットを強制的にクローズし、リソースを解放するための内部関数です。

なぜ重要か

プロセスの強制終了時やシャットダウン時に、未処理の接続やオープンなままのソケットを確実にクローズすることで、ソケットリソースのリークを防ぎ、OSのリソースを迅速に解放するために重要です。

関与するモジュール

  • module:server/mpm/event
  • module:server/mpm/worker

主要エンティティ

  • symbol:server/mpm/event/event.c#close_worker_sockets
  • symbol:server/mpm/worker/worker.c#close_worker_sockets
  • worker_sockets: 各ワーカーが使用するソケットを格納する配列。
  • threads_per_child: 子プロセスあたりのスレッド数（ループの終了条件）。

エンドツーエンドの流れ

1. 強制終了処理（worker.c や event.c の強制終了時）や、スレッドへのシグナル送信（signal_threads や close_socket_nonblocking_ など）が発生します。 2. close_worker_sockets が呼び出されます。 3. 0 から threads_per_child - 1 までのインデックスでループを実行します。 4. 各インデックス i について、worker_sockets[i] が有効（非NULL）であるかを確認します。 5. 有効な場合、apr_socket_close() を呼び出してソケットを閉じます。 6. クローズ後、worker_sockets[i] を NULL に設定して参照をクリアします。

※なお、event MPMにおける詳細なエンドツーエンドの呼び出しフローや、close_socket_nonblocking_ との具体的な連携の全容は、現時点の解析情報では完全には明らかになっていません。

重要な不変条件

  • threads_per_child が正しく初期化されていること。
  • worker_sockets 配列が各ワーカーのソケット状態を保持していること。
  • ソケットクローズ後は、二重解放や不正な参照を防ぐために worker_sockets[i] が必ず NULL に設定されること。

よくある落とし穴

  • apr_socket_close 自体が失敗した場合のエラーハンドリングは本関数内では行われません。
  • worker_sockets が初期化されていない場合や、既に閉じられたソケットに対しては、NULL チェックにより安全にスキップされますが、初期化フローとの整合性が崩れると予期せぬ挙動を招く可能性があります。
