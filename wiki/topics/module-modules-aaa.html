Module: modules/aaa

概念

modules/aaa は、Apache HTTP Server における **AAA**（Authentication: 認証、Authorization: 認可、Access Control: アクセス制御）フレームワークおよびその具体的なプロバイダ群を格納するディレクトリです。

このモジュール群は、クライアントの識別（認証）、リソースへのアクセス権限の検証（認可）、およびホストやHTTPメソッドに基づくアクセス制限（アクセス制御）を協調して処理するための仕組みを提供します。

なぜ重要か

Webサーバーのセキュリティにおいて、リソースの保護とユーザー管理は中核的な機能です。modules/aaa は、認証方式（Basic、Digest、Formなど）と、バックエンドのデータストア（テキストファイル、DBM、SQLデータベース、LDAPなど）を疎結合に保ちながら柔軟に組み合わせるための、拡張性の高いアーキテクチャを提供します。

関与するモジュール

  • module:modules/aaa

主要エンティティ

提供されたファイル群から、以下の主要なコンポーネント（エンティティ）が識別されます。

  • **認証プロトコル / 方式**:
  • mod_auth_basic.c: Basic認証の処理
  • mod_auth_digest.c: Digest認証の処理
  • mod_auth_form.c: フォームベース認証の処理
  • **認証コアおよびプロバイダ**:
  • mod_authn_core.c: 認証プロバイダの登録と管理を行うコア
  • mod_authn_file.c / mod_authn_dbm.c / mod_authn_dbd.c: それぞれテキストファイル、DBM、SQLデータベース（DBD）をバックエンドとする認証プロバイダ
  • mod_authn_anon.c: 匿名ユーザー認証
  • mod_authn_socache.c: 認証資格情報のキャッシュ
  • **認可コアおよびプロバイダ**:
  • mod_authz_core.c: <RequireAll> や <RequireAny> などの論理コンテナを含む認可コア
  • mod_authz_user.c / mod_authz_groupfile.c / mod_authz_dbm.c / mod_authz_dbd.c: ユーザー、グループ、DBM、SQLデータベースに基づく認可プロバイダ
  • mod_authz_host.c: IPアドレスやホスト名に基づくアクセス制御
  • mod_authz_owner.c (mod_authz_owner.h): ファイルの所有者に基づく認可
  • **統合・その他**:
  • mod_authnz_ldap.c: LDAPを用いた認証・認可の統合
  • mod_authnz_fcgi.c: FastCGIオーサライザとの連携
  • mod_access_compat.c: 旧バージョン（Apache 2.2以前）の Order/Allow/Deny 形式との互換性維持
  • mod_allowmethods.c: HTTPメソッド（GET, POSTなど）の制限
