ap_conn_producer_t

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_workers.c:42

目的

HTTP/2 サブシステムにおいて、接続（conn_rec）の供給元を抽象化し、ワーカープールが接続をプル（取得）および完了通知するためのインターフェースを提供します。

シグネチャまたは形状

struct ap_conn_producer_t {
    APR_RING_ENTRY(ap_conn_producer_t) link;
    const char *name;
    void *baton;
    ap_conn_producer_next *fn_next;
    ap_conn_producer_done *fn_done;
    ap_conn_producer_shutdown *fn_shutdown;
    volatile prod_state_t state;
    volatile int conns_active;
};

入力と出力

  • **入力**: 接続取得用のコールバック（fn_next）、完了通知用（fn_done）、シャットダウン用（fn_shutdown）の関数ポインタを保持します。baton は各コールバックに渡されるコンテキストデータです。
  • **出力**: fn_next を通じて conn_rec をワーカーへ提供します。

前提条件と不変条件

  • link フィールドにより、h2_workers 内の prod_active または prod_idle リングに登録される必要があります。
  • state および conns_active は volatile 修飾されており、スレッド間での可視性が保証される必要があります。
  • 状態遷移およびリング操作は、h2_workers が保持するミューテックスによって保護される必要があります。

アルゴリズムまたは振る舞い

1. **登録**: h2_workers_register を介してプールに登録されます。 2. **タスク供給**: ワーカーが get_next を呼び出すと、fn_next が実行され、処理対象の接続が提供されます。 3. **完了通知**: 接続処理終了後、fn_done が呼び出され、アクティブな接続数が更新されます。 4. **シャットダウン**: fn_shutdown により、プロデューサーの終了処理が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_mplx がこの構造体を保持し、h2_workers へ登録します。
  • **呼び出し先**: h2_workers.c 内の get_next 関数が fn_next を呼び出します。
  • **注記**: 具体的なコールバック関数の実装（fn_next 等の実体）については、本構造体の定義箇所からは直接参照されておらず、外部モジュールで定義されている可能性があります。

関連データ構造

  • h2_workers: プロデューサーを管理するメイン構造体。
  • prod_state_t: プロデューサーの状態を定義する列挙型。
  • APR_RING: プロデューサーをリスト管理するための Apache Portable Runtime のリング構造。

失敗モードと境界条件

  • fn_next が NULL を返した場合、プロデューサーから取得可能な接続がないことを示します。
  • conns_active が 0 になった際の fn_done の挙動や、シャットダウン時の同期処理が不完全な場合、デッドロックやリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は h2_workers の中核となるタスク供給インターフェースです。変更時は h2_workers.c 内のリング操作および h2_mplx との整合性に注意してください。
  • volatile 修飾が施されていますが、複雑な状態遷移を行う場合は、メモリバリアや適切なアトミック操作が適用されているか再確認を推奨します。
