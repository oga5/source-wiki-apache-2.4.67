get_dbm_grp

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbm.c:100

目的

指定された DBM ファイルから、ユーザー名（またはレルム付きユーザー名）をキーとしてグループ情報を検索し、抽出する。

シグネチャまたは形状

static apr_status_t get_dbm_grp(request_rec *r, char *key1, char *key2,
                                const char *dbmgrpfile, const char *dbtype,
                                const char ** out)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体（メモリプール r->pool を使用）。
  • key1, key2: 検索キー（通常はユーザー名や user:realm）。
  • dbmgrpfile: DBM ファイルのパス。
  • dbtype: 使用する DBM ドライバーの型名。
  • **出力**:
  • out: 抽出されたグループ文字列へのポインタを格納するアドレス。
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • すべてのメモリ割り当ては r->pool に対して行われ、リクエスト終了時に自動的に解放される。
  • DBM ファイルは読み取り専用 (APR_DBM_READONLY) で開かれる。
  • DBM の値に : が含まれる場合、password:groupname のような形式を想定してパースされる。

アルゴリズムまたは振る舞い

1. dbtype に基づき DBM ドライバーを取得し、apr_dbm_open2（または apr_dbm_open_ex）でファイルを開く。 2. key1 で検索を試み、失敗した場合は key2 で検索を行う。 3. 取得した値からグループ部分を抽出する:

  • 値に : が含まれる場合、最初の : 以降をグループ文字列として扱う。
  • さらに : が続く場合は、その位置で文字列を終端させ、グループ名のみを抽出する。

    • 4. apr_dbm_close を呼び出し、ファイルを閉じる。 5. 抽出したグループ文字列を out にセットして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authz_dbm.c:191 (dbmgroup_check_authorization 内)
  • modules/aaa/mod_authz_dbm.c:263 (dbmfilegroup_check_authorization 内)
  • **呼び出し先**:
  • get_dbm_entry_as_str: DBM から値を文字列として取得する内部ヘルパー。
  • apr_dbm_* 系関数: APR が提供する DBM 操作 API。
  • ap_log_rerror: エラーログ出力。

関連データ構造

  • request_rec: リクエストごとの状態を保持。
  • apr_dbm_t: DBM ファイルのハンドル。
  • apr_dbm_driver_t: DBM ドライバーの定義。

失敗モードと境界条件

  • DBM ドライバーのロード失敗: APR_SUCCESS 以外のステータスを返し、エラーログを出力。
  • DBM ファイルのオープン失敗: APR_SUCCESS 以外のステータスを返し、エラーログを出力。
  • キーが見つからない場合: out に NULL が設定される。
  • 値の形式が不正な場合: : が含まれない値はそのまま返される。

保守担当者向けメモ

  • DBM ファイルはリクエストごとに開閉されるため、高負荷時にはパフォーマンスへの影響を考慮すること。
  • APU_MAJOR_VERSION 等の条件分岐により、APR-util のバージョンに応じたドライバー取得方法の差異を吸収している。
