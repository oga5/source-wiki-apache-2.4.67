isapi_lookup

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_isapi.c:326

目的

ISAPI 拡張 DLL を検索・ロードし、その状態を管理するキャッシュ機構へのアクセスを提供します。DLL のロード状況を追跡し、複数スレッドからの同時ロード要求を排他制御します。

シグネチャまたは形状

apr_status_t isapi_lookup(apr_pool_t *p, server_rec *s, request_rec *r,
                          const char *fpath, isapi_loaded** isa)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • s: サーバーレコード。
  • r: リクエストレコード（NULL の場合あり）。
  • fpath: ロード対象 DLL のファイルパス。
  • isa: isapi_loaded 構造体へのポインタを格納する出力先。
  • **出力**:
  • apr_status_t: 操作結果（APR_SUCCESS 等）。
  • *isa: 検索またはロードされた isapi_loaded 構造体。

前提条件と不変条件

  • loaded.lock ミューテックスによるグローバルな排他制御下でハッシュ検索が行われます。
  • isapi_loaded 構造体内の in_progress ロックは、DLL のロード中における他スレッドの待機を制御します。
  • ロード成功後、in_progress は NULL に設定されます。

アルゴリズムまたは振る舞い

1. loaded.hash から fpath をキーに検索。 2. **キャッシュヒット時**:

  • ロード完了済みであれば即時返却。
  • ロード中であれば in_progress ロックで完了を待機。
  • ロード失敗時、ISAPI_RETRY 期間経過後であれば再ロードを試行。

    • 3. **キャッシュミス時**:

  • 新規 isapi_loaded 構造体を生成しハッシュに登録。
  • isapi_load を呼び出し DLL をロード。
  • ロード結果をキャッシュに反映し、待機中のスレッドを解放。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/win32/mod_isapi.c:80 (プロトタイプ宣言)
  • modules/arch/win32/mod_isapi.c:156 (設定処理)
  • modules/arch/win32/mod_isapi.c:1448 (リクエストハンドラ)
  • **呼び出し先**:
  • isapi_load (DLL の実ロード)
  • apr_thread_mutex_lock/unlock
  • apr_thread_rwlock_rdlock/wrlock/unlock
  • apr_hash_get/set

関連データ構造

  • isapi_loaded: DLL のハンドル、関数ポインタ、ロード状態を保持する構造体。
  • loaded: グローバルなキャッシュ管理構造体（ハッシュテーブルとミューテックスを含む）。

失敗モードと境界条件

  • apr_dso_load が失敗した場合、last_load_rv にエラーコードが保持され、再試行ロジックが適用されます。
  • r が NULL の場合、リクエスト固有のロック処理をスキップする分岐が存在します。
  • ISAPI_RETRY 期間内の再試行はブロックされます。

保守担当者向けメモ

  • in_progress ロックの生存期間は loaded.pool に依存しており、リクエストプール (r->pool) を使用する場合の競合リスクに注意が必要です。
  • fake_async 関連の処理は同期的な待機を伴うため、高負荷時のスレッド枯渇に留意してください。
