ap_queue_info_push_pool

定義

  • 種別: function
  • 定義箇所: server/mpm_fdqueue.c:217

目的

ワーカーが使用したメモリプール（apr_pool_t）を再利用可能なリストに登録し、メモリ割り当てのオーバーヘッドを削減します。

シグネチャまたは形状

void ap_queue_info_push_pool(fd_queue_info_t *queue_info, apr_pool_t *pool_to_recycle)

入力と出力

  • queue_info: メモリプールを管理する fd_queue_info_t 構造体へのポインタ。
  • pool_to_recycle: 再利用対象の apr_pool_t。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • pool_to_recycle が NULL の場合、関数は即座に終了します。
  • queue_info->max_recycled_pools が 0 以上の場合、現在の再利用プール数が上限に達していれば、当該プールは再利用されず apr_pool_destroy により破棄されます。
  • 再利用されるプールは、リストへの追加前に apr_pool_clear によって初期化されます。

アルゴリズムまたは振る舞い

1. 入力されたプールが NULL でないか確認します。 2. 再利用プール数の上限チェックを行い、上限に達している場合はプールを破棄して終了します。 3. apr_pool_clear を呼び出し、プールを空の状態にします。 4. recycled_pool 構造体をプール内に確保し、対象プールを格納します。 5. apr_atomic_casptr を使用したループにより、queue_info->recycled_pools の先頭に新しい要素をアトミックに挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c (複数箇所)
  • server/mpm_fdqueue.c:110
  • **呼び出し先**:
  • apr_atomic_read32, apr_atomic_inc32, apr_atomic_casptr (アトミック操作)
  • apr_pool_destroy, apr_pool_clear, apr_palloc (APR メモリ管理)

関連データ構造

  • fd_queue_info_t: 再利用プールのリストと上限設定を保持する構造体。
  • recycled_pool: 再利用プールを連結リストで管理するためのノード構造体。

失敗モードと境界条件

  • max_recycled_pools の上限に達した場合、プールは再利用されずに破棄されます。
  • アトミック操作の競合が発生した場合、ループ内で再試行が行われます。

保守担当者向けメモ

  • 本関数に渡されたプールは、以降 queue_info の管理下に入ります。
  • recycled_pools_count のアトミックな加算と、recycled_pools リストへの挿入は別々の操作であるため、厳密な整合性が必要な場合は注意してください。
  • PR 44402 に関連し、apr_atomic_casptr 呼び出し後のポインタ変更を考慮した実装になっています。
