authz_dbm_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authz_dbm.c:39

目的

Apache HTTP Server の mod_authz_dbm モジュールにおいて、ディレクトリごとの設定情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    const char *grpfile;
    const char *dbmtype;
} authz_dbm_config_rec;

入力と出力

本構造体は、Apache の設定解析フェーズで生成され、認可フェーズで参照されます。

  • grpfile: AuthDBMGroupFile ディレクティブで指定された DBM ファイルのパスを保持します。
  • dbmtype: AuthzDBMType ディレクティブで指定された DBM の形式（例: dbm, gdbm, sdbm）を保持します。

前提条件と不変条件

  • 設定値は request_rec のプール (r->pool) に割り当てられ、リクエスト終了時に自動的に解放されます。
  • grpfile および dbmtype は、設定解析時に ap_get_module_config 等を通じて適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

本構造体は、認可プロバイダー（dbmgroup_check_authorization 等）が実行される際、現在のディレクトリ設定を取得するために使用されます。モジュールは、この構造体から得られたパスとタイプを用いて apr_dbm_open2 を呼び出し、認可判断に必要なグループ情報を DBM ファイルから抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authz_dbm.c:68 (設定生成時)
  • modules/aaa/mod_authz_dbm.c:162, 243 (認可チェック実行時)
  • **呼び出し先**:
  • APR_OFFSETOF マクロ（設定コマンドの定義に使用）
  • ap_get_module_config（実行時の設定取得に使用）

関連データ構造

  • request_rec: リクエストごとのコンテキストを保持する Apache の構造体。
  • apr_dbm_t: apr_dbm ライブラリが提供する DBM ファイルのハンドル。

失敗モードと境界条件

  • 設定が未定義の場合、grpfile や dbmtype が NULL になる可能性があります。
  • DBM ファイルが存在しない、または読み取り権限がない場合、認可処理において apr_dbm_open2 が失敗し、認可判断が拒否される（またはエラーとなる）境界条件が存在します。

保守担当者向けメモ

  • 本構造体は mod_authz_dbm の設定管理の基盤です。新しい設定項目を追加する場合は、create_authz_dbm_dir_config 関数での初期化処理と、コマンドテーブルの定義を併せて更新してください。
