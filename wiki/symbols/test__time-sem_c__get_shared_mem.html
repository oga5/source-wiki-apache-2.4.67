get_shared_mem

定義

  • 種別: function
  • 定義箇所: test/time-sem.c:420

目的

プロセス間で共有されるカウンタ等のデータを保持するための共有メモリ領域を確保する。

シグネチャまたは形状

static void *get_shared_mem(apr_size_t size)

入力と出力

  • **入力**: size (確保するメモリサイズ)
  • **出力**: 確保された共有メモリ領域へのポインタ。失敗時は exit(1) によりプロセスが終了する。

前提条件と不変条件

  • mmap が MAP_ANON および MAP_SHARED をサポートする環境であること。
  • 確保されたメモリは、親プロセスから子プロセスへ継承されることを前提としている。

アルゴリズムまたは振る舞い

1. mmap システムコールを呼び出し、匿名共有メモリ（MAP_ANON | MAP_SHARED）を確保する。 2. 読み書き権限（PROT_READ | PROT_WRITE）を付与する。 3. 確保に失敗した場合（戻り値が (void *)-1）、perror でエラーを出力し、exit(1) で即時終了する。 4. 成功した場合、確保したメモリ領域の先頭アドレスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: test/time-sem.c:524 の main 関数内にて、shared_counter の初期化のために呼び出されている。
  • **呼び出し先**: mmap (システムコール), perror, exit

関連データ構造

  • shared_counter: 本関数で確保されたメモリ領域を指すポインタとして使用される。

失敗モードと境界条件

  • mmap が失敗した場合、エラーメッセージを出力してプログラム全体が終了する。
  • 確保サイズが 0 の場合や、システムのリソース制限を超過した場合は mmap が失敗する可能性がある。

保守担当者向けメモ

  • 本関数はベンチマークツール専用であり、エラーハンドリングは exit(1) による即時終了を基本としている。
  • 現代のシステムでは mmap が標準的だが、古いOS環境では shmget/shmat への切り替えが必要になる場合がある。
