sendv_data

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:213

目的

FastCGI バックエンドに対して、iovec 配列で指定された複数のデータバッファをソケット経由で送信する。

シグネチャまたは形状

apr_status_t sendv_data(const fcgi_provider_conf *conf,
                        request_rec *r,
                        apr_socket_t *s,
                        struct iovec *vec,
                        int nvec,
                        apr_size_t *len)

入力と出力

  • **入力**:
  • conf: バックエンド設定情報（エラーログ出力に使用）。
  • r: 現在のリクエストレコード。
  • s: 送信先のソケット。
  • vec: 送信するデータの iovec 配列。
  • nvec: vec 配列の要素数。
  • **出力**:
  • len: 実際に送信された合計バイト数。
  • 戻り値: apr_status_t 型のステータス（成功時は APR_SUCCESS）。

前提条件と不変条件

  • vec に含まれる各バッファの合計サイズが to_write として計算される。
  • apr_socket_sendv が成功する限り、ループを継続して全データを送信する。

アルゴリズムまたは振る舞い

1. 全 iovec の合計長を計算し、デバッグビルド時（APLOG_TRACE5）には ap_log_rdata で送信内容をログ出力する。 2. apr_socket_sendv を呼び出し、ソケットへデータを書き込む。 3. 送信が部分的（ショートライト）であった場合、送信済みバイト数に応じて vec のポインタと長さを更新し、残りのデータを送信するためにループを継続する。 4. 送信エラーが発生した場合は、ap_log_rerror でエラーを記録し、ループを中断する。 5. 最終的な送信バイト数を *len に格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:288
  • modules/aaa/mod_authnz_fcgi.c:373
  • modules/aaa/mod_authnz_fcgi.c:388
  • **呼び出し先**:
  • apr_socket_sendv (APR ライブラリ)
  • ap_log_rerror (Apache HTTP Server API)
  • ap_log_rdata (Apache HTTP Server API, 条件付き)

関連データ構造

  • fcgi_provider_conf: バックエンド接続情報を保持する構造体。
  • request_rec: Apache のリクエスト状態を保持する構造体。
  • apr_socket_t: APR のソケットオブジェクト。
  • iovec: 送信データバッファの配列構造体。

失敗モードと境界条件

  • **ソケットエラー**: apr_socket_sendv が APR_SUCCESS 以外の値を返した場合、エラーログを出力して処理を中断する。
  • **部分送信**: n が to_write に満たない場合、内部で offset と vec の各要素を調整し、残りのデータを再送するロジックが働く。

保守担当者向けメモ

  • 本関数は mod_authnz_fcgi.c 内の FastCGI 通信における低レベルなデータ転送を担当している。
  • AP_MODULE_MAGIC_AT_LEAST マクロにより、特定のバージョン以降でのみトレースログが出力されるよう制御されている。
