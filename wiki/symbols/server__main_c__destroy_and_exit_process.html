destroy_and_exit_process

定義

  • 種別: function
  • 定義箇所: server/main.c:257

目的

サーバープロセスの終了処理を安全に実行し、リソースの解放とプロセスの終了を行う。

シグネチャまたは形状

static void destroy_and_exit_process(process_rec *process, int process_exit_value)

入力と出力

  • process (入力): 終了対象のプロセス情報を保持する process_rec 構造体へのポインタ。
  • process_exit_value (入力): プロセス終了時に OS へ返す終了ステータスコード。
  • 出力: なし（本関数は exit() を呼び出し、呼び出し元へは戻らない）。

前提条件と不変条件

  • process ポインタは有効な process_rec 構造体を指している必要がある。
  • 本関数はプロセスのライフサイクルの最終段階で呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. TASK_SWITCH_SLEEP マイクロ秒間待機する。これは、パイプ経由のロガーが入力データを処理するための猶予を与え、ログの欠落を防ぐための措置である。 2. グローバル変数 ap_main_state を AP_SQ_MS_EXITING に更新する。 3. process->pool を破棄し、関連するすべての子孫メモリプールを解放する。 4. apr_terminate() を呼び出し、APR (Apache Portable Runtime) のリソースを解放する。 5. exit(process_exit_value) を呼び出し、プロセスを終了する。

呼び出し元と呼び出し先

  • 呼び出し元: server/main.c 内の各所（エラー発生時や正常終了時の分岐）。
  • 呼び出し先: apr_sleep, apr_pool_destroy, apr_terminate, exit。

関連データ構造

  • process_rec: プロセス全体のメモリプールや設定情報を保持する構造体。

失敗モードと境界条件

  • process->pool が既に破棄されている場合、未定義の動作を引き起こす可能性がある。
  • TASK_SWITCH_SLEEP 待機中、OS のスケジューリング状況によってはロガーの処理が完了しない可能性がある。

保守担当者向けメモ

  • 本関数は main() ループの終了処理を担うため、変更時はサーバーのシャットダウンシーケンス全体に影響を与える点に注意すること。
  • ログの欠落を防ぐための待機処理は、将来的な非同期ロギングの実装状況に応じて調整が必要となる可能性がある。
