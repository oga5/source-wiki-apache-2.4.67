disk_cache_info_t

定義

  • 種別: struct
  • 定義箇所: modules/cache/cache_disk_common.h:43

目的

ディスクキャッシュ（mod_cache_disk）において、キャッシュエントリのメタデータをディスクへ永続化するための構造体です。キャッシュの有効性検証や、対応する実体ファイルとの紐付けに使用されます。

シグネチャまたは形状

typedef struct {
    apr_uint32_t format;
    int status;
    apr_size_t name_len;
    apr_size_t entity_version;
    apr_time_t date;
    apr_time_t expire;
    apr_time_t request_time;
    apr_time_t response_time;
    apr_ino_t inode;
    apr_dev_t device;
    unsigned int has_body:1;
    unsigned int header_only:1;
    cache_control_t control;
} disk_cache_info_t;

入力と出力

  • **入力**: キャッシュ保存時に mod_cache_disk によって各フィールドに値が設定されます。
  • **出力**: ディスク上のキャッシュファイルの一部としてバイナリ形式で書き込まれます。読み込み時には、ディスク上のデータから本構造体へパースされます。

前提条件と不変条件

  • 本構造体はディスクへ直接バイナリ書き込みされるため、プラットフォーム間でのアライメントやエンディアンの差異に注意が必要です。
  • inode および device フィールドは、キャッシュヘッダーと実体ファイルが一致しているかを検証するために使用されます。

アルゴリズムまたは振る舞い

1. **保存**: mod_cache_disk がキャッシュ生成時に構造体を初期化し、ディスクへ書き込みます。 2. **検証**: キャッシュ読み込み時に、ディスク上のファイルから本構造体を読み取り、inode や device 等の情報を基にキャッシュの整合性を確認します。 3. **識別**: name_len を使用して、構造体に続くエンティティ名の長さを特定します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/cache/mod_cache_disk.c: キャッシュの読み書きおよびサイズ計算に使用。
  • modules/cache/mod_cache_disk.h: ヘッダー情報として保持。
  • support/htcacheclean.c: キャッシュのクリーンアップ処理において、ディスク上のメタデータ読み取りに使用。
  • **呼び出し先**: 構造体自体はデータ定義であり、特定の関数を呼び出すものではありません。

関連データ構造

  • cache_control_t: control フィールドとして保持され、キャッシュ制御ヘッダーの解析結果を格納します。

失敗モードと境界条件

  • **バージョン不整合**: format フィールドが期待値と異なる場合、ディスク上のキャッシュ構造が古い、または互換性がないと判断されます。
  • **整合性エラー**: inode や device が現在のファイルシステム上の値と一致しない場合、キャッシュは無効とみなされます。

保守担当者向けメモ

  • DISK_FORMAT_VERSION が変更された場合、本構造体のレイアウト変更に伴い既存キャッシュのパージが必要になる可能性があります。
  • 構造体のパディングやサイズについては、mod_cache_disk.c でのシリアライズ実装を必ず確認してください。
