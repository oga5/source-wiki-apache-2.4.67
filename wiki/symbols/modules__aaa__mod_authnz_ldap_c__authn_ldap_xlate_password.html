authn_ldap_xlate_password

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_ldap.c:170

目的

LDAP 認証に使用するパスワードを、設定された文字コードから UTF-8 へ変換する。

シグネチャまたは形状

static const char* authn_ldap_xlate_password(request_rec *r, const char* sent_password)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • sent_password: クライアントから送信された変換前のパスワード文字列。
  • **出力**:
  • 変換後の UTF-8 パスワード文字列、または変換が不要・失敗した場合は元の sent_password を返す。

前提条件と不変条件

  • グローバル変数 charset_conversions が有効であり、かつ get_conv_set(r) が有効な変換セット（apr_xlate_t）を返す場合にのみ変換処理が行われる。
  • 変換後のバッファは r->pool から割り当てられる。

アルゴリズムまたは振る舞い

1. charset_conversions が有効かつ get_conv_set(r) で変換ハンドルが取得できるか確認する。 2. 変換が必要な場合、入力文字列の長さの3倍のメモリを apr_pcalloc で確保する。 3. apr_xlate_conv_buffer を使用して、パスワードを UTF-8 に変換する。 4. 変換が APR_SUCCESS で完了すれば、変換後のバッファを返す。 5. 上記条件を満たさない場合、または変換に失敗した場合は、入力された sent_password をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authnz_ldap.c:563
  • **呼び出し先**:
  • get_conv_set (内部関数)
  • apr_pcalloc (APR)
  • apr_xlate_conv_buffer (APR)

関連データ構造

  • request_rec: Apache のリクエストコンテキスト。
  • apr_xlate_t: APR の文字コード変換ハンドル。

失敗モードと境界条件

  • get_conv_set(r) が NULL を返す場合、変換はスキップされる。
  • apr_xlate_conv_buffer が APR_SUCCESS 以外を返した場合、変換処理は中断され、元のパスワードが返される。

保守担当者向けメモ

  • 本関数はパスワードの文字コード変換のみを担う。LDAP フィルタの構築やエスケープ処理は authn_ldap_build_filter が担当する。
  • 変換後のバッファサイズは (inbytes+1)*3 として確保されており、UTF-8 への変換に伴うバイト数増加を考慮している。
