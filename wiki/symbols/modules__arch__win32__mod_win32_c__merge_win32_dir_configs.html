merge_win32_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_win32.c:62

目的

ディレクトリ単位の設定構造体（win32_dir_conf）をマージし、親ディレクトリの設定を継承しつつ、子ディレクトリで明示的に指定された設定を優先させる。

シグネチャまたは形状

static void *merge_win32_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定（win32_dir_conf *）。
  • addv: 現在のディレクトリの設定（win32_dir_conf *）。
  • **出力**:
  • マージされた新しい win32_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は win32_dir_conf 型にキャスト可能であること。
  • apr_pcalloc により、戻り値の構造体はゼロ初期化された状態で確保される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい win32_dir_conf 構造体をメモリ上に確保する。 2. add->script_interpreter_source が INTERPRETER_SOURCE_UNSET でない場合、その値を新しい設定として採用する。 3. add 側が未設定（UNSET）であれば、base（親）側の設定値を継承する。 4. 最終的にマージされた構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_win32.c:556（モジュール設定の登録プロセスの一部として参照）。
  • **呼び出し先**: apr_pcalloc（APRライブラリ）。

関連データ構造

  • win32_dir_conf: ディレクトリごとの設定を保持する構造体。
  • interpreter_source_e: script_interpreter_source が取りうる値の列挙型。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、Apache のメモリ管理ポリシーに従ってプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は Apache の設定マージ機構の一部である。script_interpreter_source 以外の設定項目が win32_dir_conf に追加された場合は、本関数内でも同様にマージロジックを更新する必要がある。
