h2_c1_setup

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1.c:92

目的

HTTP/2 接続の初期化を行い、h2_session オブジェクトを生成して接続コンテキストに割り当てる。また、HTTP/1.1 向けフィルタである reqtimeout を除去し、HTTP/2 への移行を完了させる。

シグネチャまたは形状

apr_status_t h2_c1_setup(conn_rec *c, request_rec *r, server_rec *s)

入力と出力

  • **入力**:
  • c: 現在の接続レコード (conn_rec)。
  • r: リクエストレコード (request_rec)。初期化時に利用される。
  • s: サーバーレコード (server_rec)。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、ワーカーが未初期化の場合は APR_EGENERAL、セッション生成失敗時はそのエラーコードを返す。

前提条件と不変条件

  • グローバル変数 workers が初期化されている必要がある。
  • 接続コンテキスト (h2_conn_ctx_t) が h2_conn_ctx_get(c) により取得可能であること。

アルゴリズムまたは振る舞い

1. workers の存在を確認し、未初期化であればエラーを返す。 2. h2_session_create を呼び出し、セッションオブジェクトを生成する。 3. 接続コンテキストを取得し、生成したセッションを h2_conn_ctx_assign_session で割り当てる。 4. ap_remove_input_filter_byhandle を呼び出し、HTTP/1.x 向けに設定された reqtimeout フィルタを除去する。 5. 成功時に APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c:302
  • modules/http2/h2_switch.c:200
  • **呼び出し先**:
  • h2_session_create
  • h2_conn_ctx_get
  • h2_conn_ctx_assign_session
  • ap_remove_input_filter_byhandle
  • ap_log_cerror (エラー時)

関連データ構造

  • h2_session: HTTP/2 セッションのメイン構造体。
  • h2_conn_ctx_t: 接続ごとの HTTP/2 コンテキスト。
  • conn_rec: Apache の接続レコード。

失敗モードと境界条件

  • workers が未初期化の場合: APR_EGENERAL を返し、ログを出力して終了する。
  • h2_session_create が失敗した場合: そのエラーコードを返し、処理を中断する。
  • h2_conn_ctx_get が NULL を返す場合: ap_assert(ctx) によりアボートする。

保守担当者向けメモ

  • reqtimeout フィルタの除去は、HTTP/2 確立後の状態管理が HTTP/1.x と異なるために必須の処理である。
  • 本関数は h2_c1.h にて公開されており、プロトコル切り替えの重要な接点となっている。
