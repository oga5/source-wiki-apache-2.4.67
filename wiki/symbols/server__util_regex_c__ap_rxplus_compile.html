ap_rxplus_compile

定義

  • 種別: function
  • 定義箇所: server/util_regex.c:31

目的

Perl形式の正規表現パターン（m/rx/、/rx/、または置換用の s/rx/subs/）を解析し、実行可能な ap_rxplus_t 構造体を生成します。

シグネチャまたは形状

ap_rxplus_t* ap_rxplus_compile(apr_pool_t *pool, const char *pattern)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • pattern: 解析対象の正規表現文字列。
  • **出力**:
  • 成功時: 初期化済みの ap_rxplus_t 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • pool は有効な APR プールである必要があります。
  • pattern が s/ で始まる場合は置換文字列が必要であり、欠落している場合は失敗します。
  • 成功した場合、ap_rxplus_t 内の rx メンバは rxplus_cleanup をクリーンアップ関数としてプールに登録されます。

アルゴリズムまたは振る舞い

1. **構文解析**: パターンの先頭文字から区切り文字（delimiter）とアクション（MATCH または SUBSTITUTE）を特定します。 2. **正規表現抽出**: 区切り文字を用いて正規表現部分を抽出します。 3. **置換文字列抽出**: アクションが SUBSTITUTE の場合、置換文字列を抽出します。 4. **フラグ解析**: 区切り文字以降の文字をフラグとして解釈し、ap_rxplus_t->flags に設定します（例: i=大文字小文字無視, g=グローバルマッチ等）。 5. **コンパイル**: ap_regcomp を呼び出し、正規表現をコンパイルします。 6. **メモリ管理**: AP_REG_NOMEM フラグが指定されていない場合、パターン内の ( の数をカウントし、キャプチャグループ用の ap_regmatch_t 配列をプールから割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:1182
  • modules/filters/mod_proxy_html.c:1319
  • **呼び出し先**:
  • ap_regcomp (Apache 正規表現コンパイル関数)
  • apr_pcalloc, apr_pstrmemdup, apr_palloc (APR メモリ管理)
  • apr_pool_cleanup_register (クリーンアップ登録)
  • rxplus_cleanup (クリーンアップ関数)

関連データ構造

  • ap_rxplus_t: 正規表現、置換文字列、フラグ、マッチング結果を保持するメイン構造体。
  • ap_regmatch_t: マッチした部分文字列のオフセットを保持する構造体。

失敗モードと境界条件

  • パターンが不正な形式（置換文字列の欠落など）である場合は NULL を返します。
  • ap_regcomp が失敗した場合（正規表現の構文エラーなど）は NULL を返します。
  • 区切り文字が不明確な場合や、エスケープシーケンスの複雑な組み合わせには制限があります。

保守担当者向けメモ

  • rxplus_cleanup は ap_rxplus_compile が成功した際にプールに登録されます。
  • ret->pmatch のサイズ計算は、正規表現内の ( を数える単純な手法をとっています。
