dbd_save

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_dbd.c:258

目的

セッションデータをデータベースへ永続化します。既存のセッションがあれば更新（UPDATE）を試み、存在しない場合は新規挿入（INSERT）を行います。

シグネチャまたは形状

apr_status_t dbd_save(request_rec *r, const char *oldkey, const char *newkey, const char *val, apr_int64_t expiry)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • oldkey: 更新対象の既存セッションキー（新規作成時は NULL 可）。
  • newkey: 保存するセッションの新しいキー。
  • val: シリアライズされたセッションデータ。
  • expiry: セッションの有効期限（UNIXタイムスタンプ）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は APR_EGENERAL または dbd_init が返すエラーコード。

前提条件と不変条件

  • r->per_dir_config に session_dbd_dir_conf が設定されていること。
  • mod_dbd がロードされ、データベース接続が確立可能であること。
  • 設定において SessionDBDupdatelabel および SessionDBDinsertlabel が定義されていること。

アルゴリズムまたは振る舞い

1. conf->updatelabel を使用して dbd_init を呼び出し、更新用プリペアドステートメントを取得。 2. oldkey が指定されている場合、apr_dbd_pvbquery で更新を実行。更新行数が 0 でなければ成功として終了。 3. 更新が行われなかった（または oldkey が NULL の）場合、conf->insertlabel を使用して dbd_init を呼び出し、挿入用プリペアドステートメントを取得。 4. apr_dbd_pvbquery で挿入を実行。挿入行数が 0 でなければ成功として終了。 5. いずれの操作も行われなかった場合はエラーをログ出力し、APR_EGENERAL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session_dbd.c:432
  • modules/session/mod_session_dbd.c:464
  • **呼び出し先**:
  • dbd_init (内部ヘルパー)
  • ap_get_module_config
  • ap_log_rerror
  • apr_dbd_pvbquery
  • apr_dbd_error

関連データ構造

  • session_dbd_dir_conf: ディレクトリ単位の設定（SQLラベル等）を保持。
  • request_rec: リクエストのプール管理および設定取得に使用。
  • ap_dbd_t: mod_dbd のデータベース接続ハンドル。

失敗モードと境界条件

  • **設定欠如**: updatelabel や insertlabel が未定義の場合、エラーログを出力して APR_EGENERAL を返す。
  • **DBエラー**: dbd_init や apr_dbd_pvbquery が失敗した場合、エラーログを出力してそのステータスを返す。
  • **挿入失敗**: 更新も挿入も行われなかった場合、セッションが保存されなかったと判断し APR_EGENERAL を返す。

保守担当者向けメモ

  • 本関数は r->pool を使用してメモリを管理しているため、リクエスト終了時にリソースが解放されます。
  • oldkey と newkey が同一の場合でも、ロジック上は更新処理が優先されます。
