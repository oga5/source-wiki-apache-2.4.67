brigade_peek

定義

  • 種別: function
  • 定義箇所: modules/echo/mod_echo.c:54

目的

apr_bucket_brigade 内のデータを読み取り、スコアボード表示用の文字列としてバッファに抽出します。

シグネチャまたは形状

static apr_status_t brigade_peek(apr_bucket_brigade *bbIn, char *buff, apr_size_t bufflen)

入力と出力

  • **入力**:
  • bbIn: 読み取り対象の apr_bucket_brigade。
  • buff: 抽出した文字列を格納するバッファ。
  • bufflen: バッファのサイズ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS を返し、失敗時は APR_EGENERAL または apr_bucket_read のエラーコードを返します。

前提条件と不変条件

  • bufflen は NULL 終端文字を含めたサイズとして扱われます。
  • bbIn が空の場合、APR_EGENERAL を返します。
  • バッファサイズが 0 の場合、APR_EGENERAL を返します。

アルゴリズムまたは振る舞い

1. バッファの先頭を NULL 終端し、初期化します。 2. bbIn 内のバケットを先頭から順に走査します。 3. apr_bucket_read を用いてバケットからデータを読み込みます。 4. memchr を使用してデータ内の LF (改行コード) を検索し、LF が見つかった場合はそこまでを抽出対象とします。 5. バッファの残り容量を超えない範囲でデータをコピーし、都度 NULL 終端を更新します。 6. 全てのバケットを走査するか、バッファが満杯になるまで処理を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/echo/mod_echo.c:123 (直接的な呼び出しを確認)
  • **呼び出し先**: apr_bucket_read (APR ライブラリ関数)

関連データ構造

  • apr_bucket_brigade: データのストリームを保持する構造体。
  • apr_bucket: バッファ内の個別のデータ断片。

失敗モードと境界条件

  • bbIn が空の場合: APR_EGENERAL を返します。
  • bufflen が 0 の場合: APR_EGENERAL を返します。
  • apr_bucket_read が失敗した場合: そのエラーコードを返します。
  • バッファサイズを超えるデータが存在する場合: バッファの容量分のみをコピーし、切り捨てます。

保守担当者向けメモ

  • 本関数は memchr を使用して LF を探すため、行単位のデータ抽出を前提としています。
  • APR_NONBLOCK_READ を使用しているため、非ブロッキング環境での動作を想定しています。
