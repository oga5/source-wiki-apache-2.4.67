apr_status_t

定義

  • 種別: typedef
  • 定義箇所: modules/http2/h2_proxy_session.h:70

目的

Apache Portable Runtime (APR) における操作結果やエラーコードを表現するための標準的な整数型です。

シグネチャまたは形状

typedef int apr_status_t; (APR ライブラリの定義に準拠)

入力と出力

  • 入力: なし（型定義）
  • 出力: なし（型定義）

前提条件と不変条件

  • APR_SUCCESS (通常は 0) は操作の成功を示します。
  • 0 以外の値はエラーまたは特定の状態（APR_EAGAIN, APR_EOF 等）を示します。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、具体的なアルゴリズムは持ちません。ただし、mod_proxy_http2 においては、h2_proxy_session_process() などの関数の戻り値として使用され、イベントループの制御フローを決定する指標となります。

呼び出し元と呼び出し先

  • 呼び出し元: h2_proxy_request_done コールバック関数の引数として使用されています。
  • 呼び出し先: 提供されたソース excerpt 内では、この型を直接呼び出す関係は定義されていません。

関連データ構造

  • h2_proxy_session: 本型を戻り値や引数として利用するセッション管理構造体。
  • request_rec: h2_proxy_request_done コールバック内で本型と共に使用されるリクエスト構造体。

失敗モードと境界条件

  • 型自体に失敗モードはありませんが、この型が保持する値として APR_EAGAIN（再試行が必要）、APR_EOF（接続終了）などがプロキシ処理の境界条件として頻繁に使用されます。

保守担当者向けメモ

  • 本型は APR のコア定義であり、mod_proxy_http2 固有の型ではありません。
  • h2_proxy_session_process の戻り値として本型が返される際、その値がイベントループの継続条件に直結するため、エラーハンドリングの変更時には注意が必要です。
