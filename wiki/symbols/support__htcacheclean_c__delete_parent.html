delete_parent

定義

  • 種別: function
  • 定義箇所: support/htcacheclean.c:267

目的

キャッシュディレクトリ構造において、不要になった親ディレクトリを再帰的に削除し、inode 統計情報を更新する。

シグネチャまたは形状

static void delete_parent(const char *path, const char *basename,
                          apr_off_t *nodes, apr_pool_t *pool)

入力と出力

  • path: キャッシュのベースディレクトリパス。
  • basename: 削除対象のファイルまたはディレクトリのパス名。
  • nodes: 削除されたディレクトリ数（inode 数）を減算するためのポインタ。
  • pool: メモリ割り当てに使用する親プール。

前提条件と不変条件

  • deldirs フラグが真であり、かつ dryrun フラグが偽である場合にのみディレクトリ削除が実行される。
  • 削除処理には一時的なサブプールが使用され、関数終了時に破棄される。

アルゴリズムまたは振る舞い

1. basename を基に一時プール上でパスを構築する。 2. パスを末尾から / で区切り、ディレクトリ階層を遡る。 3. apr_dir_remove を試行し、成功した場合は nodes をデクリメントする。 4. CACHE_VDIR_SUFFIX を含むディレクトリの場合、対応するファイルも削除を試み、成功時に nodes をデクリメントする。 5. apr_dir_remove が失敗した場合は、そのディレクトリにまだ他のエントリが存在すると判断し、遡行を停止する。 6. benice フラグが有効な場合、DELETE_NICE 回の削除ごとに NICE_DELAY 分の待機を行い、システム負荷を抑制する。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htcacheclean.c 内の delete_entry 関数（352行目および400行目）。
  • **呼び出し先**: apr_pool_create, apr_pstrdup, apr_pstrcat, apr_dir_remove, apr_file_remove, apr_pool_destroy, apr_sleep。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_off_t: nodes カウンタの型として使用。

失敗モードと境界条件

  • apr_dir_remove の失敗はエラーとして扱わず、ディレクトリが空でない（削除不可）とみなして処理を終了する。
  • basename に / が含まれない場合、ループは実行されない。

保守担当者向けメモ

  • 本関数は delete_entry から呼び出され、ファイル削除後のディレクトリ整理を担う。
  • benice によるスロットリングは、大量のファイル削除時にシステム全体の応答性を維持するために重要である。
