read_to_scratch

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:208

目的

apr_bucket の内容を h2_c1_io 構造体が保持するスクラッチバッファへコピーし、後続の処理のためにデータを集約する。

シグネチャまたは形状

static apr_status_t read_to_scratch(h2_c1_io *io, apr_bucket *b)

入力と出力

  • **入力**:
  • io: ターゲットとなる h2_c1_io 構造体へのポインタ。
  • b: 読み込み対象の apr_bucket。
  • **出力**:
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。
  • 副作用: io->scratch バッファにデータが追記され、io->slen（現在のバッファ長）が更新される。

前提条件と不変条件

  • b->length がスクラッチバッファの残り容量（io->ssize - io->slen）以下であること（ap_assert により検証）。
  • io->scratch が十分なサイズで確保されていること。

アルゴリズムまたは振る舞い

1. バケット長が 0 の場合は即座に APR_SUCCESS を返す。 2. バケット種別に応じて以下の処理を行う：

  • **FILE バケット**: ファイル記述子から直接 io->scratch のオフセット位置へデータを読み込む。
  • **MMAP バケット**: ログ出力を行った後、apr_bucket_read を経由してメモリをコピーする。
  • **その他**: apr_bucket_read を経由してメモリをコピーする。

    • 3. 読み込み成功時、io->slen を読み込んだバイト数分増加させる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c1_io.c:420（直接的な呼び出しを確認）。
  • **呼び出し先**:
  • apr_file_seek, apr_file_read (FILE バケット時)
  • apr_bucket_read (MMAP およびその他バケット時)
  • ap_log_cerror (MMAP バケット時)

関連データ構造

  • h2_c1_io: 入出力状態とスクラッチバッファを管理する構造体。
  • apr_bucket: Apache のデータ転送単位。

失敗モードと境界条件

  • apr_file_seek または apr_file_read が失敗した場合、そのステータスを返す。
  • apr_bucket_read がエラーを返した場合、そのステータスを返す。
  • バケット長がスクラッチバッファの空き容量を超える場合、アサーション違反によりプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数は h2_c1_io の内部バッファリングを最適化するための低レイヤー処理である。
  • FILE バケットの処理において、apr_file_read を直接呼び出すことで、中間バッファを介さない効率的な転送を行っている。
