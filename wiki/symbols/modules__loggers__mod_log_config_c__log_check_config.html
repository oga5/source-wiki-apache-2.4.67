log_check_config

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_config.c:1804

目的

サーバー設定に含まれるすべてのログ設定に対し、出力先ディレクトリの有効性を検証する。

シグネチャまたは形状

int log_check_config(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • pconf, plog, ptemp: メモリプール。
  • s: サーバー設定構造体（server_rec）。
  • **出力**:
  • int: 成功時は OK、検証に失敗したログ設定が一つでも存在する場合は !OK を返す。

前提条件と不変条件

  • s が指すサーバー設定から log_config_module の設定（multi_log_state）が取得可能であること。
  • multi_log_state 内の config_logs 配列が初期化されていること。

アルゴリズムまたは振る舞い

1. サーバーリストを s = s->next で順次走査する。 2. 各サーバー設定から multi_log_state を取得し、保持されている config_logs 配列を反復処理する。 3. 各ログ設定（config_log_state）に対して check_log_dir を呼び出し、ディレクトリの妥当性を確認する。 4. いずれかの check_log_dir が失敗した場合、戻り値を !OK に更新する。 5. 全てのサーバーのチェック完了後、最終的なステータスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_check_config により、Apache の check_config フックとして登録されている（modules/loggers/mod_log_config.c:1830）。
  • **呼び出し先**: check_log_dir（各ログ設定のディレクトリ検証用）。

関連データ構造

  • multi_log_state: サーバーごとのログ設定を保持する構造体。
  • config_log_state: 個別のログファイル設定を保持する構造体。
  • server_rec: Apache のサーバー設定構造体。

失敗モードと境界条件

  • check_log_dir が失敗した場合、関数全体としては !OK を返すが、処理は中断されず全てのサーバー設定の検証を継続する。
  • サーバーリストが空の場合、ループは実行されず OK を返す。

保守担当者向けメモ

  • 本関数は設定読み込みフェーズで実行されるため、ファイルシステムへのアクセスは ptemp（一時プール）を使用してメモリリークを防ぐこと。
  • mls->server_config_logs は親サーバーの設定を参照しているため、本関数では明示的なチェック対象外としている。
