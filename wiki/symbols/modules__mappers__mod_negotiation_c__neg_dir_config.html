neg_dir_config

定義

  • **種別**: struct
  • **定義箇所**: modules/mappers/mod_negotiation.c:48

目的

ディレクトリ単位でのコンテンツネゴシエーション設定（言語優先度など）を保持するための設定構造体です。

シグネチャまたは形状

typedef struct {
    int forcelangpriority;
    apr_array_header_t *language_priority;
} neg_dir_config;

入力と出力

  • **入力**: Apache の設定ファイルから読み込まれたディレクティブ（LanguagePriority や ForceLanguagePriority）の値。
  • **出力**: モジュール内のネゴシエーション処理において、バリアント選択アルゴリズムの挙動を制御するパラメータとして利用されます。

前提条件と不変条件

  • language_priority は apr_array_header_t 型であり、メモリはリクエストプールまたは設定プールから割り当てられます。
  • 本構造体は、Apache の設定マージ処理を通じてディレクトリ階層ごとに継承・統合されることが想定されています。

アルゴリズムまたは振る舞い

  • forcelangpriority: ネゴシエーション時に言語優先度を強制するかどうかを決定するフラグとして機能します。
  • language_priority: 優先すべき言語のリストを保持し、バリアント選択時の品質値計算において参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_neg_dir_config（構造体の初期化）
  • merge_neg_dir_configs（設定のマージ）
  • set_language_priority / set_force_priority（設定値のセット）
  • ap_get_module_config を介したリクエスト処理時の設定取得
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • negotiation_state: リクエストごとのネゴシエーション状態を保持するコンテキストであり、本構造体の設定を参照します。

失敗モードと境界条件

  • 設定が未定義の場合、language_priority が NULL になる可能性があるため、参照時にはヌルポインタチェックが必要です。

保守担当者向けメモ

  • 本構造体は mod_negotiation の設定管理の核となります。設定の追加や変更を行う際は、merge_neg_dir_configs におけるマージロジックが正しく継承されることを確認してください。
