set_extension_map

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_netware.c:72

目的

Apache HTTP Server の設定ディレクティブ CGIMapExtension を処理し、特定のファイル拡張子とそれに対応する CGI 実行エンジン（NLM）および実行モードをディレクトリ設定に登録します。

シグネチャまたは形状

static const char *set_extension_map(cmd_parms *cmd, netware_dir_config *m,
                                     char *CGIhdlr, char *ext, char *detach)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • m: 拡張子マッピングを保持する netware_dir_config 構造体。
  • CGIhdlr: 実行対象の NLM パス。
  • ext: マッピング対象のファイル拡張子。
  • detach: 独立したアドレス空間で実行するかを指定するフラグ（指定があれば "y"）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • ext に先頭の . が含まれている場合、自動的に除去されます。
  • CGIhdlr 内のバックスラッシュ (\) は、内部処理のためにスラッシュ (/) に置換されます。

アルゴリズムまたは振る舞い

1. ext の先頭に . があればスキップします。 2. CGIhdlr が指定されている場合、文字列内の \ を / に置換します。 3. m->file_type_handlers テーブルに、拡張子をキーとして CGIhdlr を登録します。 4. detach 引数が提供されている場合、m->file_handler_mode テーブルに、当該拡張子に対して "y" を設定し、分離実行モードを有効化します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_netware.c:188 において、AP_INIT_TAKE23 マクロを介して CGIMapExtension ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: apr_table_set (APR ライブラリ) を呼び出し、設定テーブルへの書き込みを行います。

関連データ構造

  • netware_dir_config: ディレクトリ単位の設定を保持する構造体。本関数はこの中の file_type_handlers および file_handler_mode テーブルを更新します。

失敗モードと境界条件

  • CGIhdlr が NULL の場合、パスの置換処理はスキップされます。
  • detach が NULL の場合、分離実行モードの設定は行われません。

保守担当者向けメモ

  • 本関数は NetWare プラットフォーム専用のモジュールに含まれており、他のプラットフォームではコンパイルされません。
  • file_handler_mode に設定される "y" は、後の CGI 実行フェーズにおいて ap_cgi_build_command がアドレス空間の分離を判断する際のフラグとして利用されます。
