uwsgi_canon

定義

  • **種別**: 関数
  • **定義箇所**: modules/proxy/mod_proxy_uwsgi.c:60

目的

uwsgi:// スキームで始まるリクエスト URL を解析し、バックエンドへ転送するための内部的なプロキシ URL 形式（proxy:uwsgi://...）へ正規化する。

シグネチャまたは形状

static int uwsgi_canon(request_rec *r, char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • url: 正規化対象の URL 文字列。
  • **出力**:
  • OK: 正規化成功。r->filename に正規化後の文字列が格納される。
  • DECLINED: uwsgi:// スキームではない場合。
  • HTTP_BAD_REQUEST: URL 解析エラーまたはパスのエンコード失敗時。
  • HTTP_FORBIDDEN: パスに不正な制御文字やスペースが含まれる場合。

前提条件と不変条件

  • url は uwsgi:// で始まる必要がある。
  • r->pool を使用してメモリを割り当てる。
  • proxy-nocanon または proxy-noencode ノートが設定されている場合、パスのエンコードをスキップする。

アルゴリズムまたは振る舞い

1. **スキーム検証**: uwsgi:// で始まらない場合は DECLINED を返す。 2. **ネットワークロケーション解析**: ap_proxy_canon_netloc を使用してホスト名とポートを抽出。 3. **IPv6 対応**: ホスト名に : が含まれる場合、[] で囲む。 4. **パス正規化**:

  • proxy-nocanon 等が設定されていない場合、ap_proxy_canonenc_ex を使用してパスをエンコード。
  • 不正な制御文字やスペースが含まれる場合は HTTP_FORBIDDEN を返す。

    • 5. **URL 再構築**: proxy:uwsgi:// プレフィックス、ホスト、ポート、パスを結合し、r->filename に格納。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_uwsgi.c 内の register_hooks にて proxy_hook_canon_handler に登録される。
  • **呼び出し先**:
  • ap_proxy_canon_netloc (URL 解析)
  • ap_proxy_canonenc_ex (パスエンコード)
  • ap_log_rerror (エラーログ出力)
  • apr_pstrcat / apr_snprintf (文字列操作)

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • core_dir_config: ディレクトリ設定（エンコードフラグの判定に使用）。

失敗モードと境界条件

  • **URL 解析失敗**: ap_proxy_canon_netloc がエラーを返すと HTTP_BAD_REQUEST となる。
  • **パスエンコード失敗**: ap_proxy_canonenc_ex が NULL を返すと HTTP_BAD_REQUEST となる。
  • **不正な文字**: パスに制御文字やスペースが含まれる場合、セキュリティ上の理由で HTTP_FORBIDDEN となる。

保守担当者向けメモ

  • 本関数は mod_proxy のフックとして動作するため、URL の書き換えロジックを変更する際は mod_proxy の正規化ルールとの整合性に注意すること。
  • IPv6 リテラルの扱いやエンコードフラグの判定は、他のプロキシモジュールと共通のパターンに従っている。
