bucket_socket_ex_create

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:162

目的

SOCKET_EX 型のカスタムバケットを新規作成し、初期化を行う。

シグネチャまたは形状

static apr_bucket *bucket_socket_ex_create(socket_ex_data *data, apr_bucket_alloc_t *list)

入力と出力

  • **入力**:
  • data: ソケット関連の統計情報等を保持する socket_ex_data 構造体へのポインタ。
  • list: バケット割り当て用の apr_bucket_alloc_t リスト。
  • **出力**:
  • 初期化済みの apr_bucket 構造体へのポインタ。

前提条件と不変条件

  • list は有効なバケットアロケータであること。
  • data は bucket_socket_ex_make に渡されるまで有効であること。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を使用して、apr_bucket 構造体分のメモリを確保する。 2. APR_BUCKET_INIT マクロでバケットの初期化を行う。 3. バケットの解放関数 (free) に apr_bucket_free を設定する。 4. バケットのリスト (list) を設定する。 5. bucket_socket_ex_make を呼び出し、バケットの型定義やデータ紐付けを行い、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_scgi.c:134
  • modules/proxy/mod_proxy_scgi.c:377
  • **呼び出し先**:
  • apr_bucket_alloc (APR API)
  • APR_BUCKET_INIT (APR API)
  • bucket_socket_ex_make (同一ファイル内)

関連データ構造

  • apr_bucket: Apache バケット構造体。
  • socket_ex_data: SOCKET_EX バケットが保持するカスタムデータ構造。
  • apr_bucket_alloc_t: バケットメモリ管理用のアロケータ。

失敗モードと境界条件

  • メモリ不足時には apr_bucket_alloc が失敗し、呼び出し元へエラーが伝播する可能性がある。

保守担当者向けメモ

  • 本関数は SOCKET_EX バケットのライフサイクル管理の入り口です。バケットの型定義や具体的なデータ構造の変更については、bucket_socket_ex_make および socket_ex_data を併せて参照してください。
