h2_ihash_t

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.h:42

目的

整数 ID をキーとして保持し、関連するデータ構造（ストリーム等）を効率的に管理・検索するためのハッシュテーブルです。

シグネチャまたは形状

不透明な構造体として定義されており、具体的な内部メンバはヘッダーファイル上では公開されていません。

入力と出力

  • **生成**: h2_ihash_create(apr_pool_t *pool, size_t offset_of_int) により、メモリプールを指定して生成されます。
  • **操作**: 整数 ID をキーとして、値（void *）の追加、取得、削除が可能です。

前提条件と不変条件

  • h2_ihash_create の offset_of_int 引数は、ハッシュに格納される構造体内部での整数 ID フィールドのオフセットを指定する必要があります。
  • スレッド安全性については、本構造体自体は明示的なロックを保持しないため、呼び出し元で同期制御を行う必要があります。

アルゴリズムまたは振る舞い

  • 整数 ID をキーとしたハッシュテーブル操作を提供します。
  • h2_ihash_iter を通じて、登録されている全要素に対する反復処理が可能です。
  • h2_ihash_count や h2_ihash_empty により、現在の格納状態を検査できます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_mplx.h: h2_mplx 構造体内でアクティブなストリームや処理済みストリームの管理に使用。
  • modules/http2/h2_session.h: セッション管理に関連して参照。
  • **呼び出し先**:
  • apr_pcalloc (メモリ確保)
  • h2_util.c 内の各操作関数（h2_ihash_add, h2_ihash_get 等）が実装を提供。

関連データ構造

  • h2_ihash_iter_t: 反復処理時に使用されるコールバック関数型。
  • apr_pool_t: メモリ管理に使用される Apache Portable Runtime のプール。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数経由でエラーが発生する可能性があります。
  • 存在しない ID に対する h2_ihash_get は NULL を返します。

保守担当者向けメモ

  • 本構造体は h2_util.c に実装が隠蔽されています。要素のメモリ所有権はコンテナ側にある場合が多いため、追加・削除時のライフサイクル管理に注意してください。
