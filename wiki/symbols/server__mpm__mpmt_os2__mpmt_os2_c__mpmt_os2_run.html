mpmt_os2_run

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:111

目的

OS/2 プラットフォームにおける MPM のメインエントリーポイントとして、親プロセスと子プロセスの分岐、およびプロセスごとの初期化処理を行う。

シグネチャまたは形状

int mpmt_os2_run(apr_pool_t *_pconf, apr_pool_t *plog, server_rec *s)

入力と出力

  • **入力**:
  • _pconf: 設定用プール。
  • plog: ログ用プール。
  • s: サーバー設定レコード。
  • **出力**:
  • OK: 正常終了（再起動時）。
  • DONE: 子プロセス終了時。
  • !OK: エラー発生時。

前提条件と不変条件

  • ap_thread_limit に基づき、プロセスあたりの最大ファイルハンドル数が設定される。
  • 共有メモリ名 /sharemem/httpd/parent_info.<ppid> の存在有無により、現在のプロセスが親か子かを判定する。

アルゴリズムまたは振る舞い

1. **初期化**: DosSetMaxFH でファイルハンドル制限を設定し、共有メモリ名を作成する。 2. **プロセス判定**: DosGetNamedSharedMem を試行。

  • **子プロセスの場合**:
  • ap_mpm_accept_mutex を共有メモリから取得。
  • リスナーソケットを共有メモリから復元し、apr_os_sock_put で APR ハンドルへ変換。
  • DosFreeMem で共有メモリを解放。
  • ap_mpm_child_main を呼び出し、リクエスト処理を開始。
  • **親プロセスの場合**:
  • ap_setup_listeners でリスナーソケットを準備。
  • master_main を呼び出し、子プロセスの監視と管理を行う。
  • 終了時に ap_my_generation を更新し、必要に応じてエラーログを出力。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_mpm により登録（server/mpm/mpmt_os2/mpmt_os2.c:530）。
  • **呼び出し先**:
  • master_main（親プロセス時）
  • ap_mpm_child_main（子プロセス時）
  • ap_setup_listeners, ap_log_error, ap_log_pid, ap_remove_pid 等の Apache API。
  • DosSetMaxFH, DosGetNamedSharedMem, DosFreeMem 等の OS/2 API。

関連データ構造

  • parent_info_t: プロセス間で共有されるリスナー情報とミューテックスを保持する構造体。
  • ap_listen_rec: リスナーソケットの情報を管理。

失敗モードと境界条件

  • DosGetNamedSharedMem が失敗した場合は親プロセスとして動作する。
  • ap_setup_listeners が 1 未満のソケットを返した場合、エラーログを出力して終了する。
  • master_main が OK 以外を返した場合、PID ファイルを削除して終了する。

保守担当者向けメモ

  • 本関数は OS/2 の Dos 系 API に強く依存している。
  • 共有メモリの所有権は親プロセスにあり、子プロセスは読み取り後に解放する設計となっている。
  • HARD_SERVER_LIMIT（10）等の制限値がプロセス管理に影響するため注意が必要。
