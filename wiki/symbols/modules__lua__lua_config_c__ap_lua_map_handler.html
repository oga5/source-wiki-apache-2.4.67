ap_lua_map_handler

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:54

目的

Apache HTTP Server のディレクトリ設定構造体（ap_lua_dir_cfg）に対し、Lua スクリプトのハンドラ情報を登録します。

シグネチャまたは形状

apr_status_t ap_lua_map_handler(ap_lua_dir_cfg *cfg,
                                const char *file,
                                const char *function,
                                const char *pattern,
                                const char *scope)

入力と出力

  • **入力**:
  • cfg: 設定を保持する ap_lua_dir_cfg 構造体へのポインタ。
  • file: 実行対象の Lua ファイルパス。
  • function: 呼び出す Lua 関数名。
  • pattern: マッチングに使用する正規表現パターン。
  • scope: スクリプトの実行スコープ（文字列）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、正規表現のコンパイル失敗時はエラーコードを返します。

前提条件と不変条件

  • cfg->pool を使用してメモリを確保するため、cfg は有効な Apache プールに関連付けられている必要があります。
  • cfg->mapped_handlers は apr_array_header_t として初期化されている必要があります。

アルゴリズムまたは振る舞い

1. ap_lua_mapped_handler_spec 構造体を cfg->pool から確保します。 2. 指定された pattern を ap_regcomp でコンパイルし、成功すれば uri_pattern として格納します。 3. file および function を apr_pstrdup で複製し、構造体に保持します。 4. scope 文字列を apl_toscope 関数で内部定数に変換して格納します。 5. 構築した ap_lua_mapped_handler_spec を cfg->mapped_handlers 配列の末尾に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_config.h に定義があり、設定読み込みプロセスから呼び出されることが想定されています。具体的な呼び出し箇所はソースコード内で明示されていません。
  • **呼び出し先**:
  • apr_pcalloc, apr_palloc, apr_pstrdup, apr_array_push (APR メモリ・配列管理)
  • ap_regcomp (Apache 正規表現エンジン)
  • apl_toscope (スコープ変換ヘルパー)

関連データ構造

  • ap_lua_dir_cfg: 設定を保持するメイン構造体。
  • ap_lua_mapped_handler_spec: ハンドラの仕様を定義する構造体。
  • ap_regex_t: コンパイル済みの正規表現オブジェクト。

失敗モードと境界条件

  • ap_regcomp が失敗した場合、関数は即座にエラーコードを返し、ハンドラの登録は行われません。
  • メモリ確保失敗時は APR の規約に従い、プールのアロケータが例外を発生させる可能性があります。

保守担当者向けメモ

  • 本関数は cfg->pool を使用するため、明示的な free は不要です。
  • cfg->mapped_handlers への追加は、Apache の設定ライフサイクルに依存します。
  • APLOGNO マクロは含まれていません。
