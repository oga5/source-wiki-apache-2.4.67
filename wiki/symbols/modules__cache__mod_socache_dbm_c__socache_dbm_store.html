socache_dbm_store

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:211

目的

キャッシュデータを DBM ファイル形式で永続化ストレージに書き込みます。

シグネチャまたは形状

apr_status_t socache_dbm_store(ap_socache_instance_t *ctx,
                               server_rec *s, const unsigned char *id,
                               unsigned int idlen, apr_time_t expiry,
                               unsigned char *ucaData,
                               unsigned int nData, apr_pool_t *pool)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンスのコンテキスト。
  • s: サーバーレコード。
  • id: キャッシュキー。
  • idlen: キーの長さ。
  • expiry: 有効期限（apr_time_t）。
  • ucaData: 保存するデータ本体。
  • nData: データ本体の長さ。
  • pool: メモリプール。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。失敗時は apr_status_t 型のエラーコードを返します。

前提条件と不変条件

  • idlen + nData が約 1KB（950バイト）を超えてはなりません。
  • ctx->pool は、本関数内で apr_pool_clear によりクリアされます。
  • DBM ファイルへの書き込みには DBM_FILE_MODE で指定された権限が適用されます。

アルゴリズムまたは振る舞い

1. **サイズ検証**: キーとデータの合計サイズが制限値（PAIRMAX または 950バイト）を超えていないか確認し、超過時は APR_ENOSPC を返します。 2. **データ構築**: apr_time_t（期限）を値の先頭に配置し、その後に ucaData を連結したバッファを作成します。 3. **DBM オープン**: ctx->data_file を APR_DBM_RWCREATE モードで開きます（APR のバージョンにより apr_dbm_open2 または apr_dbm_open を使用）。 4. **書き込み**: apr_dbm_store を使用してキーと値を保存します。 5. **クリーンアップ**: DBM ファイルを閉じ、一時バッファを解放します。 6. **期限切れ処理**: socache_dbm_expire(ctx, s) を呼び出し、キャッシュ全体の期限切れエントリの整理を試みます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dbm.c:746 に参照の記述があります。
  • **呼び出し先**: socache_dbm_expire を呼び出します。また、APR DBM ライブラリ関数（apr_dbm_open 等）を呼び出します。

関連データ構造

  • ap_socache_instance_t: キャッシュのコンテキスト（ファイルパス等を保持）。
  • apr_dbm_t: DBM ファイルのハンドル。
  • apr_datum_t: DBM のキーおよび値を保持する構造体。

失敗モードと境界条件

  • **サイズ超過**: 合計サイズが制限を超えると APR_ENOSPC を返します。
  • **ファイルオープン失敗**: DBM ファイルが開けない場合、エラーログを出力し、その時点でのエラーコードを返します。
  • **書き込み失敗**: apr_dbm_store が失敗した場合、ファイルを閉じてエラーコードを返します。

保守担当者向けメモ

  • 本関数はスレッドセーフではありません。呼び出し側で適切な排他制御が行われていることを前提としています。
  • DBM ファイルの特性上、大きなデータの保存には適していません。
  • APR バージョンによって DBM ドライバのロード方法が分岐するため、修正時は条件コンパイルブロックに注意してください。
