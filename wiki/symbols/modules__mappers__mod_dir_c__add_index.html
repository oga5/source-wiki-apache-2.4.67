add_index

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_dir.c:55

目的

Apache 設定ディレクティブ DirectoryIndex を解析し、ディレクトリごとの設定構造体 dir_config_rec にインデックスファイル名のリストを格納します。

シグネチャまたは形状

static const char *add_index(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: dir_config_rec 構造体へのポインタ。
  • arg: 設定ファイルから読み込まれた引数文字列。
  • **出力**:
  • 成功時は NULL を返します。
  • 内部で dir_config_rec->index_names 配列を更新します。

前提条件と不変条件

  • dummy は dir_config_rec 型としてキャスト可能である必要があります。
  • cmd->pool は設定の生存期間中有効なメモリプールである必要があります。

アルゴリズムまたは振る舞い

1. d->index_names が未初期化の場合、cmd->pool を使用して配列を生成します。 2. ap_getword_conf を用いて arg からトークンを順次取得します。 3. 最初のトークンが "disabled" であり、かつ他に引数がない場合、index_names 配列をクリアして終了します。 4. それ以外のトークンは、index_names 配列の末尾に順次追加されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_dir.c:131 にて AP_INIT_RAW_ARGS("DirectoryIndex", ...) マクロを通じて登録されています。
  • **呼び出し先**: ap_getword_conf, apr_array_make, apr_array_clear, apr_array_push (すべて APR/Apache API)。

関連データ構造

  • dir_config_rec: ディレクトリごとの設定を保持する構造体。index_names メンバを更新します。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • 引数に "disabled" が指定された場合、既存のインデックスリストが破棄されます。
  • ap_getword_conf が空文字列を返した時点で解析を終了します。

保守担当者向けメモ

  • 本関数は DirectoryIndex ディレクティブの解析専用です。
  • index_names 配列のメモリ管理は cmd->pool に依存しているため、設定の再読み込み時に適切に処理されます。
