create_core_server_config

定義

  • 種別: function
  • 定義箇所: server/core.c:467

目的

Apache HTTP Server のサーバー単位（仮想ホスト含む）の設定構造体 core_server_config を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_core_server_config(apr_pool_t *a, server_rec *s)

入力と出力

  • **入力**:
  • a: 設定構造体の割り当てに使用する apr_pool_t。
  • s: 設定対象となる server_rec 構造体。
  • **出力**:
  • 初期化済みの core_server_config 構造体へのポインタ（void * 型）。

前提条件と不変条件

  • s->is_virtual の値に基づき、グローバルサーバー設定か仮想ホスト設定かを判定する。
  • 構造体内の各フィールドは apr_pcalloc によりゼロ初期化される。

アルゴリズムまたは振る舞い

1. core_server_config 構造体分のメモリを apr_pcalloc で確保する。 2. **グローバルサーバーの場合 (!is_virtual)**:

  • ap_document_root および access_name をデフォルト値で設定。
  • accf_map テーブルを作成し、プラットフォーム（APR_HAS_SO_ACCEPTFILTER, WIN32 等）に応じて http/https のアクセプトフィルタを登録。
  • flush_max_threshold および flush_max_pipelined を設定。

    • 3. **仮想ホストの場合**:

  • メインサーバーの設定から error_log_format を継承。
  • flush_max_pipelined を -1 に設定。

    • 4. **共通処理**:

  • sec_dir および sec_url 用の配列を初期化。
  • trace_enable を AP_TRACE_UNSET に設定。
  • protocols 配列を初期化し、protocols_honor_order、merge_slashes、strict_host_check を未設定状態（UNSET または -1）で初期化。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:5826（サーバー設定構造体作成時）。
  • **呼び出し先**: apr_pcalloc, apr_table_make, apr_table_setn, ap_get_core_module_config, apr_array_make。

関連データ構造

  • core_server_config: 本関数が生成する主要な設定構造体。
  • server_rec: サーバーの属性を保持する構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足で失敗した場合、APR の仕様に従いプロセスが終了する可能性がある。
  • 仮想ホスト設定時にメインサーバーの設定が未初期化である場合、ap_get_core_module_config の結果に依存する。

保守担当者向けメモ

  • 本関数は core_module の設定作成フックとして機能する。
  • プラットフォーム固有のアクセプトフィルタ設定が含まれているため、OS 間の挙動差異に注意すること。
  • merge_slashes や strict_host_check などの新しい設定項目を追加する際は、ここで適切なデフォルト値（UNSET 等）を割り当てること。
