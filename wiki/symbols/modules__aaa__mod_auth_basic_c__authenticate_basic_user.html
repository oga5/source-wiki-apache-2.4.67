authenticate_basic_user

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_basic.c:291

目的

HTTP Basic 認証スキームにおけるユーザー認証の仲介を行います。設定された認証プロバイダーを順次呼び出し、クライアントから提供された資格情報の検証を試みます。

シグネチャまたは形状

int authenticate_basic_user(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のHTTPリクエスト情報)
  • **出力**: int (Apache HTTP Server のステータスコード: OK, DECLINED, HTTP_UNAUTHORIZED, HTTP_INTERNAL_SERVER_ERROR)

前提条件と不変条件

  • AuthType が Basic に設定されている必要があります。
  • AuthName が設定されている必要があります（未設定時は HTTP_INTERNAL_SERVER_ERROR を返します）。
  • authoritative 設定が有効な場合、認証失敗は即座に HTTP_UNAUTHORIZED を返します。無効な場合は DECLINED を返し、後続の認証モジュールへ処理を委譲します。

アルゴリズムまたは振る舞い

1. **初期チェック**: リクエストの認証タイプが Basic か確認し、非該当なら DECLINED を返します。 2. **資格情報取得**: get_basic_auth を呼び出し、Authorization ヘッダーからユーザー名とパスワードを抽出します。 3. **アルゴリズム適用**: AuthBasicUseDigestAlgorithm が指定されている場合、ユーザー名・レルム・パスワードから MD5 ハッシュを生成します。 4. **プロバイダー照会**: 設定されたプロバイダーリストを走査します。プロバイダー未設定時はデフォルトプロバイダーを使用します。 5. **検証**: プロバイダーの check_password または get_realm_hash を呼び出し、認証結果（AUTH_GRANTED, AUTH_DENIED, AUTH_USER_NOT_FOUND 等）を取得します。 6. **結果判定**: 認証成功時は OK を返します。失敗時は設定に応じてエラーログを出力し、必要に応じて note_basic_auth_failure を呼び出して WWW-Authenticate ヘッダーを付与し、適切な HTTP ステータスコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_check_authn に登録されており、Apache の認証フックとして呼び出されます。
  • **呼び出し先**:
  • get_basic_auth (資格情報抽出)
  • ap_lookup_provider (デフォルトプロバイダー取得)
  • note_basic_auth_failure (認証失敗時のレスポンスヘッダー付与)
  • 各種認証プロバイダーの check_password または get_realm_hash

関連データ構造

  • request_rec: リクエストごとの状態とプールを保持。
  • auth_basic_config_rec: ディレクトリ単位の認証設定（プロバイダーリスト、権限委譲フラグ等）。
  • authn_provider_list: 認証プロバイダーの連結リスト。

失敗モードと境界条件

  • **認証失敗**: AUTH_DENIED または AUTH_USER_NOT_FOUND の場合、authoritative 設定に基づき HTTP_UNAUTHORIZED または DECLINED を返します。
  • **設定不備**: AuthName がない場合や、プロバイダーがロードできない場合は HTTP_INTERNAL_SERVER_ERROR を返します。
  • **プロバイダーエラー**: AUTH_GENERAL_ERROR が発生した場合は HTTP_INTERNAL_SERVER_ERROR を返します。

保守担当者向けメモ

  • 本関数は認証の「仲介」に特化しています。実際のパスワード照合ロジックは mod_authn_xxx モジュール側に隠蔽されています。
  • メモリ割り当ては r->pool を使用しているため、明示的な解放は不要です。
  • 認証結果のキャッシュや詳細なプロバイダーの挙動については、mod_authn_core の仕様を参照してください。
