set_cookie_form_method

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:229

目的

Apache HTTP Server の設定ディレクティブ AuthFormMethod を処理し、フォーム認証時に使用する HTTP メソッドを auth_form_config_rec 構造体に設定する。

シグネチャまたは形状

static const char *set_cookie_form_method(cmd_parms *cmd, void *config, const char *method)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • config: 現在のディレクトリ設定構造体 (auth_form_config_rec)。
  • method: 設定ファイルから読み込まれたメソッド文字列。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • config 引数は auth_form_config_rec 型としてキャスト可能であること。
  • check_string 関数が提供するバリデーションルールに従うこと。

アルゴリズムまたは振る舞い

1. config を auth_form_config_rec 型にキャストする。 2. conf->method に指定された method を代入する。 3. conf->method_set フラグを 1 に設定し、メソッドが明示的に設定されたことを記録する。 4. check_string(cmd, method) を呼び出し、結果を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c:363 において、AP_INIT_TAKE1 マクロを通じて AuthFormMethod ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: check_string 関数。

関連データ構造

  • auth_form_config_rec: ディレクトリごとの認証設定を保持する構造体。method および method_set フィールドを更新する。

失敗モードと境界条件

  • check_string がエラーを返した場合、そのエラーメッセージがそのまま本関数の戻り値となる。

保守担当者向けメモ

  • 本関数は AuthFormMethod ディレクティブ専用のセッターである。
  • method_set フラグは、設定のマージ処理やデフォルト値の判定に使用される可能性があるため、変更時は注意すること。
