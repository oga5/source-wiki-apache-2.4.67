secsocket_data

定義

  • 種別: struct
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:106

目的

NetWare 環境における SSL/TLS 接続のコンテキスト情報を保持するための構造体です。

シグネチャまたは形状

ソースコード上では typedef struct secsocket_data secsocket_data; として宣言されています。具体的なメンバ変数の構成は提供されたソース抜粋には含まれていません。

入力と出力

  • **入力**: 接続確立時やアップグレード時に、Apache の接続プール（c->pool）からメモリが割り当てられます。
  • **出力**: 接続コンテキスト（conn_config）に格納され、後続のリクエスト処理やフィルタ処理で参照されます。

前提条件と不変条件

  • Apache の接続プール（apr_pool_t）のライフサイクルに紐付いて管理されます。
  • nwssl_module 設定として接続コンテキストに登録されていることが前提となります。

アルゴリズムまたは振る舞い

  • 接続開始時（nwssl_pre_connection）に SSL 接続であると判定された場合、この構造体が生成・付与されます。
  • ap_get_module_config を介して、接続に関連付けられた SSL 状態やソケット情報を取得するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/netware/mod_nw_ssl.c: 接続初期化時およびリクエスト処理時の各所で、ap_palloc によるメモリ確保や ap_get_module_config による取得が行われています。
  • **呼び出し先**:
  • 構造体内部の具体的な関数ポインタやメソッドの呼び出しについては、提供されたソース抜粋からは確認できません。

関連データ構造

  • conn_rec: Apache の接続レコード。この構造体は conn_rec->conn_config に格納されます。
  • nwssl_module: 本構造体を管理するための Apache モジュール構造体。

失敗モードと境界条件

  • メモリ確保失敗: apr_palloc が失敗した場合、接続処理が継続できない可能性があります。
  • 設定未取得: ap_get_module_config が NULL を返す場合、SSL コンテキストが正しく初期化されていない状態となります。

保守担当者向けメモ

  • 本構造体のメンバ定義は modules/arch/netware/mod_nw_ssl.c の 132 行目付近から開始される定義ブロックを参照してください。
  • NetWare の SSL スタックに依存しているため、構造体の変更は WinSock IOCTL を介した通信処理に直接影響を与える可能性があります。
