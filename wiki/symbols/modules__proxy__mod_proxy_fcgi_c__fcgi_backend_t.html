fcgi_backend_t

定義

  • 種別: enum
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:35

目的

FastCGI バックエンドサーバーの種別を識別し、プロトコル処理の挙動を切り替えるために使用されます。

シグネチャまたは形状

typedef enum {
    BACKEND_DEFAULT_UNKNOWN = 0,
    BACKEND_FPM,
    BACKEND_GENERIC,
} fcgi_backend_t;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: BACKEND_DEFAULT_UNKNOWN (0), BACKEND_FPM, BACKEND_GENERIC のいずれかの値。

前提条件と不変条件

  • BACKEND_DEFAULT_UNKNOWN がデフォルト値として使用されます。
  • fcgi_dirconf_t 構造体内でバックエンドタイプを保持するために利用されます。

アルゴリズムまたは振る舞い

  • BACKEND_FPM: PHP-FPM 特有の挙動を制御する際に使用されます。
  • BACKEND_GENERIC: 標準的な FastCGI バックエンドとして処理されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/mod_proxy_fcgi.c:48 にて fcgi_dirconf_t 構造体のメンバ backend_type として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • fcgi_dirconf_t: ディレクトリ単位の設定構造体であり、本列挙型を保持します。

失敗モードと境界条件

  • 未初期化状態では BACKEND_DEFAULT_UNKNOWN が設定されます。これ以外の値が代入された場合の動作は、本ソース抜粋範囲内では定義されていません。

保守担当者向けメモ

  • 本シンボルは mod_proxy_fcgi におけるバックエンドの差異を吸収するためのフラグです。新しいバックエンドタイプを追加する場合は、この列挙型を拡張し、対応する処理を fcgi_do_request 等の制御フローに追加してください。
