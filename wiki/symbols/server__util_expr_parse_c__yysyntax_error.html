yysyntax_error

定義

  • 種別: function
  • 定義箇所: server/util_expr_parse.c:1050

目的

Bison によって生成されたパーサーにおいて、構文エラー発生時にエラーメッセージを構築する。

シグネチャまたは形状

static int yysyntax_error (YYSIZE_T *yymsg_alloc, char **yymsg, yytype_int16 *yyssp, int yytoken)

入力と出力

  • **入力**:
  • yymsg_alloc: メッセージバッファの現在のサイズ（ポインタ）。
  • yymsg: エラーメッセージを格納するバッファ（ポインタ）。
  • yyssp: パーサースタックの現在の状態。
  • yytoken: 検出された予期しないトークン。
  • **出力**:
  • 成功時 0。
  • バッファの再割り当てが必要な場合 1。
  • メモリ制限超過等のエラー時 2。

前提条件と不変条件

  • YYERROR_VERBOSE が有効な環境で呼び出される。
  • メモリ管理は apr_pool_t を介して行われるが、本関数内では YYSTACK_ALLOC_MAXIMUM を基準としたバッファサイズの検証を行う。

アルゴリズムまたは振る舞い

1. 現在のパーサースタック状態 (yyssp) に基づき、予期されるトークンを特定する。 2. 予期されるトークンの数に応じて、適切なエラーメッセージフォーマット（syntax error, unexpected %s, expecting %s... 等）を選択する。 3. メッセージの合計サイズを計算し、提供された yymsg バッファが不足している場合は 1 を返して呼び出し元に再割り当てを促す。 4. バッファが十分な場合、yytnamerr を使用してトークン名を文字列化し、フォーマット文字列に埋め込んでエラーメッセージを生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_expr_parse.c 内の YYSYNTAX_ERROR マクロ（yyparse 実行中のエラーハンドリング時）。
  • **呼び出し先**: yytnamerr（トークン名の変換）、yystrlen（文字列長計算）。

関連データ構造

  • yytname: トークン名の文字列配列。
  • yypact, yycheck, yytable: パーサーの遷移テーブル。

失敗モードと境界条件

  • **メモリ不足**: yysize が YYSTACK_ALLOC_MAXIMUM を超える場合、またはバッファ拡張が不可能な場合は 2 を返す。
  • **トークン数制限**: 予期されるトークンが YYERROR_VERBOSE_ARGS_MAXIMUM (5) を超える場合、リストを切り詰めて処理する。

保守担当者向けメモ

  • 本関数は Bison が自動生成するコードの一部であるため、直接編集せず util_expr_parse.y を修正すること。
  • sprintf を使用せず、手動で文字列を構築することで名前空間の汚染を回避している。
