cache_negotiated_docs

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_negotiation.c:142

目的

Apache HTTP Server の設定ディレクティブ CacheNegotiatedDocs を処理し、ネゴシエーションされたドキュメントのキャッシュ可否をサーバー設定に反映させる。

シグネチャまたは形状

static const char *cache_negotiated_docs(cmd_parms *cmd, void *dummy, int arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • arg: CacheNegotiatedDocs ディレクティブの引数（On であれば非ゼロ、Off であれば 0）。
  • **出力**:
  • 常に NULL を返す（設定エラーがないことを示す）。

前提条件と不変条件

  • cmd->server->module_config が初期化されている必要がある。
  • negotiation_module がサーバー設定に登録されている必要がある。

アルゴリズムまたは振る舞い

1. arg が真（On）の場合、negotiation_module のサーバー設定領域に "Cache" という文字列を設定する。 2. arg が偽（Off）の場合、設定領域を NULL にリセットする。 3. ap_set_module_config を使用して、サーバーレベルの設定として値を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_negotiation.c:158 において、AP_INIT_FLAG マクロを通じて設定解析時に呼び出される。
  • **呼び出し先**: ap_set_module_config（Apache HTTP Server API）。

関連データ構造

  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。
  • negotiation_module: 本モジュールのモジュール定義構造体。

失敗モードと境界条件

  • 本関数自体はエラーを返さないが、ap_set_module_config が失敗する環境下では設定が正しく反映されない可能性がある。

保守担当者向けメモ

  • この関数は CacheNegotiatedDocs ディレクティブ専用のハンドラである。
  • 設定値は単なる文字列 "Cache" の有無として保持されるため、キャッシュの有効・無効判定を行う際は、この設定値が NULL でないかを確認するロジックが他箇所に存在すると推測される。
