ap_internal_redirect_handler

定義

  • 種別: function
  • 定義箇所: modules/http/http_request.c:800

目的

既存のリクエストコンテキストを基に、新しい URI への内部リダイレクトを実行し、そのリクエストの処理とハンドラ呼び出しを完結させる。

シグネチャまたは形状

void ap_internal_redirect_handler(const char *new_uri, request_rec *r)

入力と出力

  • **入力**:
  • new_uri: リダイレクト先の URI 文字列。
  • r: 現在のリクエスト構造体 (request_rec)。
  • **出力**: なし（戻り値 void）。

前提条件と不変条件

  • r は有効なリクエスト構造体であること。
  • internal_internal_redirect が内部的に呼び出され、新しいリクエスト構造体が生成される。
  • リダイレクト先のリクエスト処理が完了した時点で、ap_die を通じてレスポンスが確定される。

アルゴリズムまたは振る舞い

1. internal_internal_redirect(new_uri, r) を呼び出し、新しいリクエスト構造体 new を生成する。 2. new が生成できなかった場合（エラー発生時）、処理を中断して終了する。 3. 元のリクエスト r にハンドラが設定されている場合、そのコンテンツタイプを新しいリクエスト new に引き継ぐ。 4. ap_process_request_internal(new) を実行し、リクエストの内部処理（設定適用等）を行う。 5. 内部処理が OK であれば、ap_invoke_handler(new) を呼び出し、コンテンツハンドラを実行する。 6. 最後に ap_die(access_status, new) を呼び出し、リクエストの終了処理またはエラーレスポンスの生成を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/cgi_common.h
  • modules/generators/mod_asis.c
  • modules/mappers/mod_actions.c
  • modules/proxy/mod_proxy_scgi.c
  • **呼び出し先**:
  • internal_internal_redirect (内部ヘルパー)
  • ap_set_content_type_ex
  • ap_process_request_internal
  • ap_invoke_handler
  • ap_die

関連データ構造

  • request_rec: リクエストのライフサイクルとコンテキストを保持する主要構造体。

失敗モードと境界条件

  • internal_internal_redirect が失敗して NULL を返した場合、関数は即座にリターンし、リダイレクト処理は行われない。
  • ap_process_request_internal または ap_invoke_handler でエラーが発生した場合、そのステータスコードが ap_die に渡され、適切なエラーレスポンスが生成される。

保守担当者向けメモ

  • 本関数は、リダイレクト後のリクエストに対してハンドラ実行までを一貫して行うため、呼び出し側はリダイレクト後の制御フローを意識する必要がない。
  • ap_die は内部でリクエストの終了処理を担うため、本関数の最後で呼び出すことでリクエストのライフサイクルを完結させている。
