menu_header

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_imagemap.c:476

目的

画像マップの代替メニューを表示する際、HTML ドキュメントのヘッダー部分（<html> から <body> 開始タグまで）を生成する。

シグネチャまたは形状

static void menu_header(request_rec *r, char *menu)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • menu: メニューの表示形式を指定する文字列（例: "formatted"）。
  • **出力**: なし（ap_rvputs を通じてレスポンスバッファへ直接書き出す）。

前提条件と不変条件

  • r は有効なリクエストオブジェクトであること。
  • r->pool を使用してメモリ管理が行われるため、呼び出し元で明示的な解放は不要。
  • menu 引数が "formatted" の場合、追加のヘッダー見出しが出力される。

アルゴリズムまたは振る舞い

1. ap_set_content_type_ex を呼び出し、コンテンツタイプを text/html; charset=ISO-8859-1 に設定する。 2. ap_rvputs を使用して、HTML の標準的なヘッダー（DOCTYPE、<html>、<head>、<title>）を出力する。タイトルには r->uri を HTML エスケープして埋め込む。 3. menu 引数が "formatted" と一致する場合、さらに <h1> 見出しと水平線（<hr />）を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_imagemap.c:640
  • **呼び出し先**:
  • ap_set_content_type_ex (Apache API)
  • ap_rvputs (Apache API)
  • ap_escape_html (Apache API)
  • strcasecmp (標準ライブラリ)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する主要な構造体。

失敗モードと境界条件

  • r->uri が NULL の場合、ap_escape_html の挙動に依存する。
  • menu 引数が NULL の場合、strcasecmp での比較時に未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は HTML の開始タグのみを生成する。対応する終了タグ（</body></html>）の生成については、menu_footer 等の別の関数が担当している。
  • 文字コードとして ISO-8859-1 がハードコードされている点に注意。
