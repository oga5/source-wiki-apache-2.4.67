md_acme_order_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme_order.h:22

目的

ACME プロトコルにおける「Order（注文）」オブジェクトを保持し、証明書発行プロセスにおける状態管理とサーバーとの同期を担います。

シグネチャまたは形状

typedef struct md_acme_order_t md_acme_order_t;

※構造体の内部フィールドは非公開（不透明ポインタとして扱われることが多い）ですが、URL、状態、認可 URL リスト、証明書取得用エンドポイント等の情報を保持します。

入力と出力

  • **入力**: ACME サーバーから受信した JSON レスポンス（md_acme_order_from_json 等を介して入力）。
  • **出力**: JSON 形式へのシリアライズ（md_acme_order_to_json）や、メモリ上の状態更新。

前提条件と不変条件

  • apr_pool_t によるメモリ管理下で生成される必要があります。
  • ACMEv2 プロトコルの仕様に基づき、サーバー側の状態とローカルの保持状態が同期されていることが期待されます。

アルゴリズムまたは振る舞い

1. **生成**: md_acme_order_create により初期化されます。 2. **状態更新**: md_acme_order_update を通じて、サーバー上の最新情報とローカルの md_acme_order_t を同期します。 3. **管理**: 認可 URL の追加（md_acme_order_add）や削除（md_acme_order_remove）をサポートします。 4. **永続化**: JSON 変換を介してディスク上の order.json と同期されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_drive.h（証明書取得フローの管理）、modules/md/md_acme_order.c（内部ロジック）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、具体的な処理は md_acme_order.c 内の関数群が操作します。

関連データ構造

  • md_acme_order_st: Order の状態（pending, ready, valid 等）を示す列挙型。
  • md_json_t: 永続化およびサーバー通信用 JSON データ構造。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • JSON 解析失敗時や、サーバーとの通信エラー時に状態更新が失敗する可能性があります。
  • サーバーから返される JSON レスポンスが期待するスキーマと一致しない場合、整合性が失われる可能性があります。

保守担当者向けメモ

  • 本構造体は ACMEv2 プロトコルに強く依存しています。
  • フィールドの変更や追加を行う際は、md_acme_order_to_json および order_update_from_json のシリアライズ/デシリアライズ処理を必ず更新してください。
  • 状態遷移（pending -> ready -> valid）のデバッグには md_acme_order_update の実装を参照してください。
