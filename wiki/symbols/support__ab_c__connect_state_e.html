connect_state_e

定義

  • 種別: enum
  • 定義箇所: support/ab.c:251

目的

ApacheBench (ab) における各接続のライフサイクル状態を定義し、非同期 I/O 処理の進捗を管理するために使用されます。

シグネチャまたは形状

typedef enum {
    STATE_UNCONNECTED = 0,
    STATE_CONNECTING,
    STATE_CONNECTED,
    STATE_READ
} connect_state_e;

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: 接続状態を示す整数値（0〜3）

前提条件と不変条件

  • STATE_UNCONNECTED が初期状態として使用されます。
  • STATE_CONNECTING は TCP 接続が開始されたが、完了が未確認の状態を指します。
  • STATE_CONNECTED は TCP 接続の完了が確認された状態を指します。

アルゴリズムまたは振る舞い

ab のイベントループにおいて、ソケットの読み書きイベントが発生するたびに、この列挙値を用いて接続の現在のフェーズを追跡します。これにより、非同期に進行する接続確立からレスポンス読み込みまでの遷移を制御します。

呼び出し元と呼び出し先

  • 呼び出し元: set_conn_state 関数（support/ab.c:540）が、接続状態を更新する際に引数として使用します。
  • 呼び出し先: 情報なし。

関連データ構造

  • struct connection: 各接続の情報を保持する構造体。この構造体内の状態管理フィールドとして connect_state_e が利用されます。

失敗モードと境界条件

  • 接続が失敗した場合やタイムアウトが発生した場合の遷移先については、本シンボルの定義範囲外です。

保守担当者向けメモ

  • 状態遷移のロジックを変更する場合は、set_conn_state 関数内の実装と整合性を取る必要があります。
  • STATE_CONNECTING から STATE_CONNECTED への遷移は、非同期ポーリングの結果に基づいて行われるため、イベントループの挙動に依存します。
