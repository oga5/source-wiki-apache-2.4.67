magic

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_mime_magic.c:128

目的

ファイルの内容（マジックナンバー）を判定するためのルールを保持する単方向連結リストのノード。

シグネチャまたは形状

struct magic {
    struct magic *next;
    int lineno;
    short flag;
    short cont_level;
    struct { char type; long offset; } in;
    long offset;
    unsigned char reln;
    char type;
    char vallen;
    union VALUETYPE value;
    unsigned long mask;
    char nospflag;
    char desc[MAXDESC];
};

入力と出力

  • **入力**: apprentice() 関数により、設定ファイルから読み込まれた各ルールが本構造体に格納される。
  • **出力**: match() 関数がこの構造体のリストを走査し、ファイル内容との照合結果を判定する。

前提条件と不変条件

  • next ポインタにより、単方向連結リストを構成する。
  • value 共用体は、type フィールドで指定された型（BYTE, SHORT, LONG, STRING 等）に従って解釈される。
  • MAXDESC サイズの desc フィールドは、バッファオーバーランの可能性があることがソースコード内で指摘されている。

アルゴリズムまたは振る舞い

  • mod_mime_magic モジュールにおいて、ファイル先頭のバイト列と value を比較するためのパターンマッチング定義として機能する。
  • cont_level は、ネストされたルール（> 記号で示される条件）の階層を示す。
  • mask フィールドは、比較前に値に対して適用されるビットマスクを保持する。

呼び出し元と呼び出し先

  • **呼び出し元**: apprentice() がリスト構築時に使用。match() が判定処理時に使用。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出さない。
  • **注記**: ツールが提示した http_config.h 等の magic は、本モジュールの構造体とは異なる Apache 内部のモジュールバージョン管理用シンボルであり、本構造体とは無関係である。

関連データ構造

  • magic_server_config_rec: 本構造体のリストを保持するサーバー設定構造体。
  • union VALUETYPE: magic 構造体内で比較対象の値を保持する共用体。

失敗モードと境界条件

  • desc フィールドのバッファオーバーランリスク。
  • MAXstring を超える文字列値の取り扱い。
  • cont_level が深いネスト構造を持つ場合、再帰的なマッチング処理におけるスタックや計算量の制限。

保守担当者向けメモ

  • desc フィールドのオーバーランに関する警告がソース内に存在するため、修正を検討すること。
  • 本構造体は file(1) コマンドのロジックを移植したものであり、スレッドセーフな設計については検証が不十分である点に注意が必要。
