md_pkey_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_crypt.c:78

目的

Apache HTTP Server の mod_md において、OpenSSL の EVP_PKEY をラップし、APR プールと紐付けられたライフサイクル管理を行うための構造体です。

シグネチャまたは形状

struct md_pkey_t {
    apr_pool_t *pool;
    EVP_PKEY   *pkey;
};

入力と出力

  • pool: 構造体のメモリ管理に使用される APR プールへのポインタ。
  • pkey: 実際の秘密鍵を保持する OpenSSL の EVP_PKEY 構造体へのポインタ。

前提条件と不変条件

  • 本構造体に関連付けられた OpenSSL オブジェクトは、APR プールのクリーンアップハンドラを介して自動的に解放されます。
  • 手動での EVP_PKEY_free 呼び出しは不要ですが、プールが破棄されるタイミングに依存します。

アルゴリズムまたは振る舞い

  • md_crypt.c 内の鍵管理関数（md_pkey_gen 等）によって生成・操作されます。
  • 秘密鍵の保持および、ACME プロトコルにおける署名や暗号化操作の抽象化層として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: md_acme_acct.c（アカウント鍵の読み込み・管理）、md_acme.c（ACME トランザクション処理）等から参照されています。
  • **呼び出し先**: OpenSSL の EVP_PKEY API を利用します。

関連データ構造

  • EVP_PKEY: OpenSSL の鍵保持構造体。
  • apr_pool_t: APR のメモリ管理プール。

失敗モードと境界条件

  • OpenSSL の初期化（md_crypt_init）が完了していない場合、本構造体を用いた暗号操作は失敗する可能性があります。
  • メモリ不足時には apr_pool_t の割り当てに失敗する可能性があります。

保守担当者向けメモ

  • OpenSSL 1.0.x と 1.1.x/3.0.x 間の API 差異を吸収するマクロが md_crypt.c 内で使用されているため、本構造体を利用する際はバージョン互換性に注意してください。
  • スレッド安全性は md_crypt_init の初期化ロジックによって担保されています。
