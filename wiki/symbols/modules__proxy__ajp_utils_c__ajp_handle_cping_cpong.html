ajp_handle_cping_cpong

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_utils.c:24

目的

バックエンドサーバーとのAJP接続の生存確認（ヘルスチェック）を行います。CPINGメッセージを送信し、CPONG応答を受信することで接続の有効性を検証します。

シグネチャまたは形状

apr_status_t ajp_handle_cping_cpong(apr_socket_t *sock,
                                    request_rec *r,
                                    apr_interval_time_t timeout)

入力と出力

  • **入力**:
  • sock: 通信に使用するソケット。
  • r: 現在のリクエスト情報（ログ出力およびメモリプール用）。
  • timeout: Ping/Pong 処理に適用する一時的なタイムアウト値。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時はエラーコード（ソケット操作失敗、タイムアウト設定失敗、またはCPONG以外の応答受信時など）。

前提条件と不変条件

  • 呼び出し時にソケット sock は接続済みである必要があります。
  • 処理終了時には、ソケットのタイムアウト設定が呼び出し前の状態に復元されることが保証されます。

アルゴリズムまたは振る舞い

1. **初期化**: ajp_msg_create でメッセージバッファを作成し、ajp_msg_serialize_cping でCPINGメッセージをシリアライズします。 2. **送信**: ajp_ilink_send を使用してバックエンドへCPINGを送信します。 3. **タイムアウト変更**: 現在のソケットタイムアウト値を apr_socket_timeout_get で保存し、引数 timeout を apr_socket_timeout_set で適用します。 4. **受信と検証**: ajp_ilink_receive で応答を待ち、ajp_msg_get_uint8 で受信したコマンドが CMD_AJP13_CPONG であるかを確認します。 5. **クリーンアップ**: 成功・失敗に関わらず、cleanup ラベルを経由して元のソケットタイムアウト値を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述はありません。
  • **呼び出し先**:
  • ajp_msg_create, ajp_msg_serialize_cping, ajp_msg_reuse, ajp_msg_get_uint8, ajp_msg_log
  • ajp_ilink_send, ajp_ilink_receive
  • apr_socket_timeout_get, apr_socket_timeout_set
  • ap_log_rerror

関連データ構造

  • ajp_msg_t: AJP通信用メッセージバッファ。
  • apr_socket_t: APRソケット構造体。
  • request_rec: Apacheリクエスト構造体。
  • CMD_AJP13_CPONG: AJPプロトコルで定義されたCPONGコマンド定数。

失敗モードと境界条件

  • **メッセージ作成失敗**: ajp_msg_create が失敗した場合、即座にエラーを返します。
  • **通信エラー**: 送信または受信時にエラーが発生した場合、タイムアウトを復元した上でエラーを返します。
  • **プロトコル違反**: 受信したメッセージが CMD_AJP13_CPONG ではない場合、APR_EGENERAL を返し、接続が期待通りでないことを示します。
  • **タイムアウト設定失敗**: apr_socket_timeout_set が失敗した場合、処理を中断してエラーを返します。

保守担当者向けメモ

  • 本関数はソケットのタイムアウトを一時的に変更します。エラー発生時も確実に元の設定に戻るよう cleanup ラベルによる制御フローが実装されています。
  • ログ出力には APLOG_TRACE8 が使用されており、詳細なデバッグが可能です。
