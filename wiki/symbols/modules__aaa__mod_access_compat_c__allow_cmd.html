allow_cmd

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_access_compat.c:143

目的

Apache 設定ファイル内の Allow および Deny ディレクティブを解析し、ディレクトリごとの設定構造体（access_compat_dir_conf）にアクセス制御ルールを追加します。

シグネチャまたは形状

static const char *allow_cmd(cmd_parms *cmd, void *dv, const char *from, const char *where_c)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • dv: ディレクトリ設定構造体（access_compat_dir_conf）。
  • from: "from" 文字列（必須）。
  • where_c: 許可または拒否する対象（IP、ホスト名、環境変数など）。
  • **出力**:
  • 成功時は NULL を返します。
  • 構文エラーや解析失敗時は、エラーメッセージ文字列を返します。

前提条件と不変条件

  • from 引数は必ず "from" という文字列である必要があります。
  • cmd->info の値によって、ルールが allows 配列に追加されるか denys 配列に追加されるかが決定されます。

アルゴリズムまたは振る舞い

1. from 引数が "from" でない場合、エラーを返します。 2. cmd->info に基づき、設定構造体の allows または denys 配列に新しい allowdeny 構造体を確保します。 3. where_c の内容を解析し、以下のいずれかのタイプとして a->type を設定します。

  • env=!: 環境変数の否定（T_NENV）
  • env=: 環境変数（T_ENV）
  • all: 全許可/拒否（T_ALL）
  • IPアドレスまたはサブネット（T_IP）: apr_ipsubnet_create を使用して検証。
  • ホスト名（T_HOST）: 上記以外の場合。

    • 4. 解析中に apr_ipsubnet_create が失敗した場合、適切なエラーメッセージを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_access_compat.c 内の AP_INIT_ITERATE2 マクロにより、設定ファイル内の Allow および Deny ディレクティブから呼び出されます。
  • **呼び出し先**:
  • apr_pstrdup, apr_array_push, apr_ipsubnet_create, ap_strchr, apr_psprintf などの APR/Apache API を呼び出します。

関連データ構造

  • access_compat_dir_conf: ディレクトリごとの設定を保持する構造体。
  • allowdeny: 個別のアクセス制御ルールを格納する構造体。

失敗モードと境界条件

  • "from" キーワードが欠落している場合、エラーを返します。
  • IP アドレスの形式が不正な場合、エラーを返します。
  • 設定値に # が含まれる場合、コメントとして扱わずエラーを返します。

保守担当者向けメモ

  • 本関数は mod_access_compat における設定解析の要です。mod_authz_host への移行を検討する際、この解析ロジックがレガシーな構文をどのように現在の認可構造にマッピングしているかを確認してください。
  • in_domain 関数と同様、ホスト名ベースの判定は DNS 逆引きに依存するため、セキュリティ上の注意が必要です。
