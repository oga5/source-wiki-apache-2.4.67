get_fn_info

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_info.c:134

目的

リクエストまたはサーバー設定の文脈から、現在処理中のファイル名情報を取得します。

シグネチャまたは形状

static const char *get_fn_info(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト構造体、またはNULL）
  • **出力**: const char * （ファイル名情報を示す文字列）

前提条件と不変条件

  • r が有効な場合、info_module に関連付けられたリクエスト設定から情報を取得します。
  • r が NULL の場合、静的変数 dump_config_fn_info を返します。

アルゴリズムまたは振る舞い

1. 引数 r が真（非NULL）であるかを確認します。 2. r が存在する場合、ap_get_module_config を呼び出し、r->request_config から info_module に紐付く設定情報を取得して返します。 3. r が NULL の場合、グローバルな設定情報である dump_config_fn_info を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_info.c:147 にて、const char *prevfn = get_fn_info(r); として呼び出されています。
  • **呼び出し先**: ap_get_module_config （Apache HTTP Server コア API）を呼び出します。

関連データ構造

  • request_rec: リクエスト処理のコンテキストを保持する構造体。
  • info_module: mod_info モジュールの識別子。
  • dump_config_fn_info: 設定ダンプ時に使用されるファイル名情報の静的変数。

失敗モードと境界条件

  • r が NULL の場合、リクエスト固有の設定ではなく、フォールバックとして dump_config_fn_info が返されます。

保守担当者向けメモ

  • 本関数は mod_info における設定情報の追跡に使用されます。リクエスト処理中か、コマンドラインからの設定ダンプ（-DDUMP_CONFIG）かによって取得元を切り替える設計になっています。
