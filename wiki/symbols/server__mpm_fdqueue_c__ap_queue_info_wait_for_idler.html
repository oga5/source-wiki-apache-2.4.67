ap_queue_info_wait_for_idler

定義

  • 種別: function
  • 定義箇所: server/mpm_fdqueue.c:148

目的

ワーカーが利用可能になるまでリスナー・スレッドを待機させ、スレッド間の負荷調整を行う。

シグネチャまたは形状

apr_status_t ap_queue_info_wait_for_idler(fd_queue_info_t *queue_info, int *had_to_block)

入力と出力

  • **入力**:
  • queue_info: 管理対象の fd_queue_info_t 構造体へのポインタ。
  • had_to_block: 待機が発生したかを示すフラグを格納するポインタ（任意）。
  • **出力**:
  • APR_SUCCESS: 成功。
  • APR_EOF: キューが終了状態（terminated）である場合。
  • その他 apr_status_t エラーコード（Mutex/CondVar 操作失敗時）。

前提条件と不変条件

  • queue_info->idlers はアトミックなカウンタであり、zero_pt を基準として負の値は待機中のスレッド数を表す。
  • 本関数はスレッド環境 (APR_HAS_THREADS) でのみ有効。

アルゴリズムまたは振る舞い

1. apr_atomic_add32 を使用し、idlers カウンタをデクリメントする。 2. カウンタが zero_pt 以下であれば、ワーカーが不足していると判断し、idlers_mutex を取得して条件変数 wait_for_idler で待機する。 3. ミューテックス取得後、競合状態を避けるため idlers の値を再確認する。 4. 待機解除後、または最初からアイドルが存在した場合は、terminated 状態を確認して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • apr_atomic_add32, apr_atomic_inc32
  • apr_thread_mutex_lock, apr_thread_mutex_unlock
  • apr_thread_cond_wait

関連データ構造

  • fd_queue_info_t: アイドル状態管理構造体。
  • apr_thread_mutex_t: 同期用ミューテックス。
  • apr_thread_cond_t: 待機用条件変数。

失敗モードと境界条件

  • apr_thread_mutex_lock または apr_thread_cond_wait が失敗した場合、エラーコードを返し、必要に応じてアトミックカウンタを復元する。
  • terminated フラグがセットされている場合、待機せずに APR_EOF を返す。

保守担当者向けメモ

  • idlers カウンタの zero_pt はマジックナンバーであり、アトミック操作による「待機中」と「アイドル中」の区別に不可欠。
  • had_to_block フラグは、呼び出し側が実際にブロッキングが発生したかを追跡するために使用される。
