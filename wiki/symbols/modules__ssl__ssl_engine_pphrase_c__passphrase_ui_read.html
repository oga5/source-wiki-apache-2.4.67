passphrase_ui_read

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:696

目的

OpenSSL の UI_METHOD インターフェースの一部として、暗号化された秘密鍵を復号するためのパスフレーズを収集します。

シグネチャまたは形状

static int passphrase_ui_read(UI *ui, UI_STRING *uis)

入力と出力

  • **入力**:
  • ui: OpenSSL の UI 構造体。ユーザーデータとして pphrase_cb_arg_t を保持。
  • uis: 入力対象の UI_STRING 構造体。
  • **出力**:
  • 成功時は 1 を返し、UI_set_result を介してパスフレーズを OpenSSL に渡します。
  • 失敗時は 0 を返します。

前提条件と不変条件

  • UI_get0_user_data(ui) から取得される pphrase_cb_arg_t が有効であること。
  • パスフレーズは apr_pool_t 上で管理され、処理終了時に OPENSSL_cleanse によってメモリから消去されること。

アルゴリズムまたは振る舞い

1. UI_get0_user_data からコンテキストを取得し、設定された SSLPassPhraseDialog タイプを確認します。 2. **組み込み/パイプ方式**: EVP_read_pw_string または pipe_get_passwd_cb を使用してパスフレーズを取得します。空入力の場合は再試行を繰り返します。 3. **フィルタ方式**: ssl_util_readfilter を呼び出し、外部プログラムからパスフレーズを取得します。 4. 取得したパスフレーズを ppcb->cpPassPhraseCur に格納し、UI_set_result で OpenSSL に設定します。 5. 最後に OPENSSL_cleanse を呼び出し、一時バッファ上の機密情報を破棄します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_pphrase.c:821 にて UI_method_set_reader に登録され、OpenSSL の内部処理から呼び出されます。
  • **呼び出し先**:
  • pipe_get_passwd_cb (パイプ方式時)
  • EVP_read_pw_string (組み込み方式時)
  • ssl_util_readfilter (フィルタ方式時)
  • OPENSSL_cleanse (機密データ消去)

関連データ構造

  • pphrase_cb_arg_t: パスフレーズ収集の状態を保持するコンテキスト。
  • SSLSrvConfigRec: サーバーごとの SSL 設定。
  • UI, UI_STRING: OpenSSL の対話型 UI API 構造体。

失敗モードと境界条件

  • パスフレーズが空の場合、エラーメッセージを出力し、入力を再要求します。
  • 外部フィルタプログラムが失敗した場合、またはパスフレーズ取得処理でエラーが発生した場合は 0 を返して処理を中断します。

保守担当者向けメモ

  • 本関数は OpenSSL の UI_METHOD 実装の一部です。メモリ管理は apr_pool_t に依存しているため、プール生存期間に注意してください。
  • Windows 環境では builtin 方式がサポートされないため、パイプ経由の入力が必須となる点に留意してください。
