curl_status

定義

  • 種別: function
  • 定義箇所: modules/md/md_curl.c:35

目的

libcurl 固有のエラーコードを、Apache Portable Runtime (APR) の標準的なステータスコードに変換します。

シグネチャまたは形状

static apr_status_t curl_status(unsigned int curl_code)

入力と出力

  • **入力**: unsigned int curl_code (libcurl が返す CURLcode 値)
  • **出力**: apr_status_t (対応する APR エラーコード)

前提条件と不変条件

  • 入力値が CURLE_OK の場合は APR_SUCCESS を返します。
  • 定義済みの CURLcode に該当しない値が入力された場合、デフォルトとして APR_EGENERAL を返します。

アルゴリズムまたは振る舞い

switch 文を用いて、以下のマッピングを実行します。

  • CURLE_UNSUPPORTED_PROTOCOL, CURLE_NOT_BUILT_IN → APR_ENOTIMPL
  • CURLE_URL_MALFORMAT → APR_EINVAL
  • CURLE_COULDNT_RESOLVE_PROXY, CURLE_COULDNT_RESOLVE_HOST, CURLE_COULDNT_CONNECT → APR_ECONNREFUSED
  • CURLE_REMOTE_ACCESS_DENIED → APR_EACCES
  • CURLE_OUT_OF_MEMORY → APR_ENOMEM
  • CURLE_OPERATION_TIMEDOUT → APR_TIMEUP
  • CURLE_SSL_CONNECT_ERROR → APR_ECONNABORTED
  • CURLE_AGAIN → APR_EAGAIN

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_curl.c:369
  • modules/md/md_curl.c:412
  • modules/md/md_curl.c:419
  • modules/md/md_curl.c:577 ( fire_status 経由)
  • **呼び出し先**: 情報なし。

関連データ構造

  • apr_status_t (APR エラーコード型)
  • CURLcode (libcurl エラーコード列挙型)

失敗モードと境界条件

  • 未定義の curl_code が渡された場合、APR_EGENERAL が返されます。

保守担当者向けメモ

  • 新しい libcurl のエラーコードが導入された場合、必要に応じて switch 文にケースを追加してください。
  • APR_EGENERAL は汎用的なエラーであるため、特定のデバッグが必要な場合は、ログ出力と併用することを推奨します。
