lua_get_cookie

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:2053

目的

HTTP リクエストから指定された名前のクッキー値を取得し、Lua スタックへプッシュする。

シグネチャまたは形状

static int lua_get_cookie(lua_State *L)

入力と出力

  • **入力**:
  • L: Lua 状態（スタック第1引数に Apache2.Request userdata、第2引数にクッキー名文字列を期待）。
  • **出力**:
  • 成功時: クッキー値を Lua スタックにプッシュし、1 を返す。
  • 失敗時（クッキー未検出）: 0 を返す。

前提条件と不変条件

  • スタック第1引数は ap_lua_check_request_rec により検証可能な request_rec であること。
  • スタック第2引数は luaL_checkstring で取得可能な文字列であること。

アルゴリズムまたは振る舞い

1. ap_lua_check_request_rec を呼び出し、スタックから request_rec ポインタを取得する。 2. luaL_checkstring を呼び出し、スタックからクッキーのキー名を取得する。 3. ap_cookie_read を呼び出し、リクエストから該当キーのクッキー値を検索する。 4. クッキーが見つかった場合、その値を lua_pushstring でスタックに積んで 1 を返す。 5. クッキーが見つからない場合、何も積まずに 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_request.c:2965（makefun を介した登録）。
  • **呼び出し先**:
  • ap_lua_check_request_rec
  • luaL_checkstring
  • ap_cookie_read
  • lua_pushstring

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • lua_State: Lua インタプリタの状態。

失敗モードと境界条件

  • ap_lua_check_request_rec が失敗した場合、Lua エラーが発生し処理が中断される。
  • luaL_checkstring が失敗した場合、Lua エラーが発生する。
  • 指定されたキーのクッキーが存在しない場合、戻り値 0 を返し、Lua 側では nil として扱われる。

保守担当者向けメモ

  • 本関数は ap_cookie_read に依存しており、クッキーの解析ロジックは Apache の util_cookies.h 経由で提供される機能に準拠する。
