abort_on_oom

定義

  • 種別: function
  • 定義箇所: server/main.c:275

目的

メモリ割り当て失敗（OOM: Out of Memory）が発生した際に、サーバーを緊急停止させるためのハンドラです。

シグネチャまたは形状

static int abort_on_oom(int retcode)

入力と出力

  • **入力**: retcode (int) - 呼び出し元から渡される終了コード。
  • **出力**: int - 原則として呼び出し元へは戻りません。

前提条件と不変条件

  • 本関数は apr_pool_abort_set を介して APR プールのエラーハンドラとして登録されることを前提としています。
  • 内部で ap_abort_on_oom() を呼び出し、プロセスを終了させるため、正常な復帰は想定されていません。

アルゴリズムまたは振る舞い

1. ap_abort_on_oom() を呼び出し、メモリ不足時の致命的エラー処理を実行します。 2. 引数として受け取った retcode を返しますが、このコードに到達することは期待されていません。

呼び出し元と呼び出し先

  • **呼び出し先**: ap_abort_on_oom()
  • **呼び出し元**:
  • server/main.c (複数箇所)
  • modules/http2/h2_mplx.c
  • support/ab.c
  • support/htdbm.c
  • support/htpasswd.c
  • support/passwd_common.h

    • (注: これらは apr_pool_abort_set の引数として参照されている直接的な証拠に基づくものです)

関連データ構造

  • apr_pool_t: APR のメモリプール。本関数はこのプールのエラーハンドラとして設定されます。

失敗モードと境界条件

  • メモリ不足が発生した際に呼び出されることが唯一の目的です。
  • 呼び出し元が apr_pool_abort_set を使用して本関数を登録していない場合、メモリ不足時に本関数は実行されません。

保守担当者向けメモ

  • 本関数は static で定義されており、server/main.c 内のプロセス初期化処理において重要な役割を果たします。
  • 複数のサポートツール（ab, htdbm 等）からも参照されているため、変更時はこれらのツールへの影響を考慮してください。
