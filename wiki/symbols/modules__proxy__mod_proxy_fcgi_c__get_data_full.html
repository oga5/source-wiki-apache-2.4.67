get_data_full

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:301

目的

バックエンド接続から指定されたバイト数（buflen）のデータを完全に読み込むまで、内部の get_data 関数を繰り返し呼び出すラッパー関数です。

シグネチャまたは形状

static apr_status_t get_data_full(proxy_conn_rec *conn, char *buffer, apr_size_t buflen)

入力と出力

  • **入力**:
  • conn: バックエンド接続を保持する proxy_conn_rec 構造体。
  • buffer: 読み込んだデータを格納するメモリ領域。
  • buflen: 読み込むべき合計バイト数。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS を返し、get_data でエラーが発生した場合はそのエラーコードを返します。

前提条件と不変条件

  • buffer は少なくとも buflen バイト以上の容量を持っている必要があります。
  • get_data が成功し続ける限り、cumulative_len が buflen に達するまでループが継続されます。

アルゴリズムまたは振る舞い

1. cumulative_len を 0 に初期化します。 2. cumulative_len が buflen に達するまで以下のループを実行します。

  • 残りの読み込み必要量（buflen - cumulative_len）を計算します。
  • get_data を呼び出し、データを buffer のオフセット位置に読み込みます。
  • get_data が APR_SUCCESS 以外を返した場合は、直ちにそのエラーコードを返して終了します。
  • 読み込んだバイト数を cumulative_len に加算します。

    • 3. 指定された全バイト数の読み込みが完了した時点で APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_fcgi.c:766
  • modules/proxy/mod_proxy_fcgi.c:985
  • **呼び出し先**:
  • get_data (同ファイル内のユーティリティ関数)

関連データ構造

  • proxy_conn_rec: mod_proxy が管理するバックエンド接続情報。

失敗モードと境界条件

  • get_data がエラーを返した場合、そのエラーがそのまま伝播されます。
  • buflen が 0 の場合、ループは実行されず即座に APR_SUCCESS を返します。

保守担当者向けメモ

  • 本関数はブロッキング I/O を前提としており、指定された全量を読み込むまで制御を戻しません。
