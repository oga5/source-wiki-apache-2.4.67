ssl_verify_t

定義

  • 種別: enum
  • 定義箇所: modules/ssl/ssl_private.h:463

目的

SSL/TLS クライアント認証の検証レベルを定義する列挙型です。

シグネチャまたは形状

typedef enum {
    SSL_CVERIFY_UNSET           = UNSET,
    SSL_CVERIFY_NONE            = 0,
    SSL_CVERIFY_OPTIONAL        = 1,
    SSL_CVERIFY_REQUIRE         = 2,
    SSL_CVERIFY_OPTIONAL_NO_CA  = 3
} ssl_verify_t;

入力と出力

  • 本シンボル自体は列挙型であり、関数ではありません。
  • 設定値として SSLConnRec 構造体や設定処理内で使用されます。

前提条件と不変条件

  • SSL_CVERIFY_UNSET は設定が未指定であることを示します。
  • 各値は mod_ssl の検証ロジックにおいて、クライアント証明書を要求するか、検証に失敗した場合に接続を拒否するかを決定する基準となります。

アルゴリズムまたは振る舞い

  • SSL_CVERIFY_NONE: クライアント証明書を要求しません。
  • SSL_CVERIFY_OPTIONAL: クライアント証明書を要求しますが、提示されなくても接続を許可します。
  • SSL_CVERIFY_REQUIRE: 有効なクライアント証明書を必須とします。
  • SSL_CVERIFY_OPTIONAL_NO_CA: 証明書を要求しますが、CA 証明書による検証を行いません。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ssl/ssl_engine_config.c 内の構成解析処理で、設定値の保持や初期化に使用されています。
  • 呼び出し先: 本シンボルはデータ型であるため、直接的な呼び出し先はありません。

関連データ構造

  • SSLConnRec (modules/ssl/ssl_private.h): 接続ごとの検証モード（verify_mode, nVerifyClient）として本型を保持します。

失敗モードと境界条件

  • SSL_CVERIFY_UNSET が設定されたまま検証処理に到達した場合、モジュール側でデフォルト値へのフォールバックまたはエラーハンドリングが必要です。

保守担当者向けメモ

  • 新しい検証モードを追加する場合は、OpenSSL の検証コールバック関数との整合性を確認してください。
  • ssl_engine_config.c での型キャストや比較処理において、列挙型の値が正しく扱われているか注意してください。
