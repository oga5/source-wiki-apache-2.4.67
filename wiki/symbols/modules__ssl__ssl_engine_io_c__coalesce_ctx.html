coalesce_ctx

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_engine_io.c:1723

目的

出力データの結合（coalesce）を行い、TLS レコードの断片化を抑制して転送効率を最適化するためのバッファ構造体です。

シグネチャまたは形状

struct coalesce_ctx {
    char buffer[COALESCE_BYTES];
    apr_size_t bytes;
};

入力と出力

  • buffer: COALESCE_BYTES サイズの固定長文字配列。出力データを一時的に保持します。
  • bytes: buffer 内に現在格納されている有効なデータ量（バイト数）。

前提条件と不変条件

  • bytes は常に 0 以上 COALESCE_BYTES 以下である必要があります。
  • COALESCE_BYTES は TLS レコードの効率化を目的とした定数です。

アルゴリズムまたは振る舞い

ssl_io_filter_coalesce 関数において、小さなバケットをこの構造体の buffer に蓄積し、一定量に達した際やフラッシュ時にまとめて下位フィルタへ渡すことで、TLS レコードの断片化を最小限に抑えます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ssl/ssl_engine_io.c:1733 にて、フィルタコンテキスト (f->ctx) から本構造体へのポインタが取得されています。
  • 呼び出し先: 提供されたソース抜粋からは、本構造体自体が関数を呼び出す関係は確認できません。

関連データ構造

  • ssl_filter_ctx_t: SSL 接続のメインコンテキスト。
  • apr_bucket_brigade: HTTPD のデータ転送単位。本構造体で結合されたデータは、最終的にこのブリゲードに格納されます。

失敗モードと境界条件

  • buffer が満杯になった場合、既存のデータをフラッシュして新しいデータを書き込む必要があります。
  • COALESCE_BYTES を超えるデータ入力時の挙動については、本抜粋からは詳細不明です。

保守担当者向けメモ

  • 本構造体は ssl_io_filter_coalesce による最適化処理専用です。
  • bytes の更新と buffer への書き込みが同期していることを確認してください。
