set_cookie_name

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_cookie.c:213

目的

Apache設定ディレクティブ（SessionCookieName等）から渡された引数を解析し、Cookie名および関連属性をセッション設定構造体に格納します。

シグネチャまたは形状

static const char *set_cookie_name(cmd_parms *cmd, void *config, const char *args)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: session_cookie_dir_conf 構造体へのポインタ。
  • args: 設定ファイルから渡される引数文字列（Cookie名およびオプションの属性）。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は check_string が返すエラーメッセージ文字列を返します。

前提条件と不変条件

  • config は session_cookie_dir_conf 型としてキャスト可能である必要があります。
  • args は apr_pstrdup によりリクエストプールに複製され、設定のライフサイクル中保持されます。

アルゴリズムまたは振る舞い

1. args を apr_pstrdup で複製します。 2. apr_strtok を使用し、最初の空白文字までを Cookie 名として抽出して conf->name に格納します。 3. conf->name_set フラグを 1 に設定します。 4. 残りの文字列から先頭の空白を除去し、残りの部分を conf->name_attrs（Cookie属性）として格納します。 5. 最後に check_string を呼び出し、Cookie 名に不正な文字が含まれていないか検証して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_usertrack.c:477 (AP_INIT_TAKE1)
  • modules/session/mod_session_cookie.c:258 (AP_INIT_RAW_ARGS)
  • modules/session/mod_session_dbd.c:626 (AP_INIT_RAW_ARGS)
  • **呼び出し先**:
  • apr_pstrdup
  • apr_strtok
  • apr_isspace
  • check_string

関連データ構造

  • session_cookie_dir_conf: Cookie設定を保持する構造体。
  • cmd_parms: Apache設定コマンドのコンテキスト。

失敗モードと境界条件

  • check_string の制約により、Cookie 名に = や & が含まれる場合はエラーとなります。
  • args が空または不正な形式の場合、apr_strtok の挙動に依存しますが、後続の check_string が検証を行います。

保守担当者向けメモ

  • 本関数は SessionCookieName だけでなく、他のモジュール（mod_usertrack や mod_session_dbd）からも再利用されています。変更時はこれらへの影響を考慮してください。
  • conf->name_attrs に格納される文字列は、Apacheのプールメモリ管理下にあるため、明示的な解放は不要です。
