cache_out_filter

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:641

目的

キャッシュされたレスポンスをストレージから読み出し、フィルタチェーンを通じてクライアントへ送信する。

シグネチャまたは形状

apr_status_t cache_out_filter(ap_filter_t *f, apr_bucket_brigade *in)

入力と出力

  • **入力**:
  • f: 現在のフィルタコンテキスト。f->ctx に cache_request_rec が格納されていることを期待する。
  • in: 処理対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS または ap_pass_brigade の戻り値）。

前提条件と不変条件

  • f->ctx が cache_request_rec 構造体として初期化されていること。
  • キャッシュの検索（cache_select）が先行して完了しており、cache->handle が有効であること。

アルゴリズムまたは振る舞い

1. **コンテキスト検証**: f->ctx が空の場合、キャッシュが無効と判断し、フィルタを削除して後続のフィルタへブリゲードを渡す。 2. **EOS（End of Stream）待機**: 入力ブリゲード in を走査し、EOS バケットが見つかるまで既存のバケットを破棄する。 3. **レスポンス復元**: EOS を検知すると以下を実行する。

  • キャッシュされた Content-Type をレスポンスヘッダーに設定。
  • キャッシュされたステータスコードを r->status に反映。
  • cache->provider->recall_body を呼び出し、キャッシュされたボディデータをブリゲードへ読み込む。

    • 4. **終了処理**: フィルタ自身をチェーンから削除し、復元したボディを含むブリゲードを後続のフィルタへ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache.c 内の2箇所（行2668, 2678）から参照されている。
  • **呼び出し先**:
  • ap_remove_output_filter: フィルタの終了時に自身を削除。
  • cache->provider->recall_body: キャッシュストレージからのデータ読み出し。
  • ap_pass_brigade: 後続のフィルタへのデータ転送。

関連データ構造

  • cache_request_rec: キャッシュリクエストの管理構造体。
  • ap_filter_t: Apache フィルタ構造体。
  • apr_bucket_brigade: データ転送用のバケットリスト。

失敗モードと境界条件

  • **キャッシュコンテキスト欠如**: cache が NULL の場合、エラーログを出力し、フィルタを無効化して処理を継続する。
  • **EOS 未到達**: EOS バケットがブリゲード内に存在しない間は、入力バケットを破棄し続ける。

保守担当者向けメモ

  • 本フィルタは AP_FTYPE_CONTENT_SET 相当のフェーズで動作する。
  • recall_body はプロバイダー依存の実装であるため、ストレージ固有の挙動については各プロバイダー（mod_cache_disk 等）のコードを参照すること。
