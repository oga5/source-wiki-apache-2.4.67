socache_mc_status

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_memcache.c:288

目的

Memcached バックエンドの稼働状況および統計情報を取得し、mod_status 経由で HTTP レスポンスとして出力する。

シグネチャまたは形状

static void socache_mc_status(ap_socache_instance_t *ctx, request_rec *r, int flags)

入力と出力

  • **入力**:
  • ctx: Memcached インスタンス情報を含む ap_socache_instance_t 構造体。
  • r: 現在のリクエストを処理する request_rec 構造体。
  • flags: 出力形式を制御するフラグ（AP_STATUS_SHORT の有無で HTML タグの有無を切り替え）。
  • **出力**: なし（ap_rprintf 等を通じてリクエスト出力ストリームへ直接書き出す）。

前提条件と不変条件

  • ctx->mc が有効な apr_memcache_t オブジェクトを指していること。
  • r->pool が統計情報取得のためのメモリ割り当てに利用可能であること。

アルゴリズムまたは振る舞い

1. ctx->mc に登録された全 Memcached サーバー（rc->live_servers）をループで走査する。 2. 各サーバーに対して apr_memcache_stats を呼び出し、統計情報を取得する。 3. 取得に成功した場合、flags に応じて以下の情報を出力する：

  • サーバーの接続状態（Up/Down）。
  • バージョン、PID、稼働時間。
  • クライアント接続数（構造体数、総数、現在数）。
  • ストレージ使用状況（アイテム数、バイト数）。
  • CPU 使用率（システム/ユーザー）。
  • キャッシュ操作統計（Gets, Sets, Hits, Misses）。
  • ネットワーク統計（入出力バイト数）。
  • その他（Evictions, MaxMem, Threads）。

    • 4. flags が AP_STATUS_SHORT でない場合は、HTML タグ（<br />, <b> 等）を付与して整形する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_memcache.c:361（socache_mc プロバイダー構造体の一部として登録されている）。
  • **呼び出し先**: apr_memcache_stats（統計取得）、ap_rprintf / ap_rputs（出力）。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスのコンテキスト。
  • apr_memcache_t: apr_memcache ライブラリのメイン構造体。
  • apr_memcache_server_t: 個別の Memcached サーバー情報。
  • apr_memcache_stats_t: サーバーから取得した統計データ。

失敗モードと境界条件

  • apr_memcache_stats が APR_SUCCESS を返さない場合、そのサーバーの統計出力はスキップされ、次のサーバーの処理へ移行する。
  • サーバーリストが空の場合、ループは実行されず何も出力されない。

保守担当者向けメモ

  • 本関数は mod_status の出力生成に依存しているため、出力フォーマットの変更は mod_status の仕様と整合させる必要がある。
  • apr_memcache_stats の呼び出しはネットワーク I/O を伴う可能性があるため、多数のサーバーが登録されている場合はレスポンス時間に影響を与える可能性がある。
