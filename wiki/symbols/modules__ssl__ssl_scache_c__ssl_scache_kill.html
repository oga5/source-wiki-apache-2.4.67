ssl_scache_kill

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_scache.c:97

目的

SSL セッションキャッシュおよび OCSP ステープリングキャッシュの破棄（クリーンアップ）を行います。

シグネチャまたは形状

void ssl_scache_kill(server_rec *s)

入力と出力

  • **入力**: server_rec *s（現在のサーバー設定を保持する構造体へのポインタ）
  • **出力**: なし（void）

前提条件と不変条件

  • myModConfig(s) を通じて取得される SSLModConfigRec が有効である必要があります。
  • キャッシュプロバイダーが初期化されている場合、その destroy メソッドが呼び出されます。

アルゴリズムまたは振る舞い

1. myModConfig(s) を呼び出し、モジュールの設定構造体 mc を取得します。 2. mc->sesscache が存在する場合、その destroy メソッドを呼び出し、セッションキャッシュを破棄します。 3. HAVE_OCSP_STAPLING が定義されている場合、mc->stapling_cache が存在すればその destroy メソッドを呼び出し、OCSP ステープリングキャッシュを破棄します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:2484
  • **呼び出し先**:
  • mc->sesscache->destroy（セッションキャッシュプロバイダーの破棄関数）
  • mc->stapling_cache->destroy（OCSP キャッシュプロバイダーの破棄関数、HAVE_OCSP_STAPLING 定義時のみ）

関連データ構造

  • SSLModConfigRec: モジュール設定を保持する構造体。sesscache および stapling_cache プロバイダーへのポインタを保持します。
  • server_rec: Apache HTTP Server のサーバー設定構造体。

失敗モードと境界条件

  • mc->sesscache または mc->stapling_cache が NULL の場合、対応する破棄処理はスキップされます。
  • サーバー終了処理の過程で呼び出されることを想定しており、キャッシュが既に破棄されている状態での二重呼び出しに対する保護は、プロバイダー側の destroy 実装に依存します。

保守担当者向けメモ

  • 本関数は mod_ssl の終了処理の一部として実行されます。
  • HAVE_OCSP_STAPLING マクロの有無により、コンパイル時に OCSP 関連のコードパスが変化することに注意してください。
