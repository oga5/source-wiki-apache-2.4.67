md_duration_parse

定義

  • 種別: function
  • 定義箇所: modules/md/md_time.c:159

目的

人間が読みやすい形式の期間指定文字列（例: "10d", "50mi"）を、APR が扱う apr_interval_time_t（マイクロ秒単位の期間）に変換します。

シグネチャまたは形状

apr_status_t md_duration_parse(apr_interval_time_t *ptimeout, const char *value, const char *def_unit)

入力と出力

  • **入力**:
  • ptimeout: 変換結果を格納する apr_interval_time_t へのポインタ。
  • value: 解析対象の文字列（数値と単位）。
  • def_unit: 単位が省略された場合に適用されるデフォルト単位（"d", "h", "mi", "s", "ms" 等）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時は APR_EINVAL（解析不能）や APR_EGENERAL（単位不明）を返します。

前提条件と不変条件

  • value には数値が含まれている必要があります。
  • apr_strtoi64 が内部で使用されており、数値変換エラー時は errno が返されます。

アルゴリズムまたは振る舞い

1. apr_strtoi64 を使用して文字列から数値を抽出します。 2. 単位文字列が省略されている場合、def_unit を使用します。 3. 単位文字に基づき以下の変換を行います：

  • d / D: 日単位（MD_SECS_PER_DAY を乗算）
  • h / H: 時間単位（MD_SECS_PER_HOUR を乗算）
  • s / S: 秒単位
  • m / M: 次の文字を判定
  • s / S: ミリ秒単位
  • i / I: 分単位（60秒を乗算）

    • 4. 変換後の値を apr_time_from_sec 等でマイクロ秒に換算し、*ptimeout に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_reg.c:1118
  • modules/md/md_time.c:259
  • modules/md/mod_md_config.c:689, 707, 725, 769, 1236
  • **呼び出し先**:
  • apr_strtoi64
  • apr_time_from_sec

関連データ構造

  • apr_interval_time_t: APR で期間を表す標準的な型。

失敗モードと境界条件

  • value が数値で始まらない場合: APR_EINVAL を返します。
  • サポートされていない単位が指定された場合: APR_EGENERAL を返します。
  • 数値変換時にオーバーフロー等が発生した場合: errno が返されます。

保守担当者向けメモ

  • 本関数は md_time.h で宣言されています。
  • ミリ秒の指定には "ms" ではなく "ms" のように 'm' に続く 's' を用いる仕様です。
  • 単位の判定ロジックは switch 文による文字比較で行われており、大文字小文字を区別しない設計となっています。
