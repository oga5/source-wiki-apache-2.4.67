init_process

定義

  • 種別: function
  • 定義箇所: server/main.c:306

目的

Apache HTTP Server のプロセス初期化を行い、APR (Apache Portable Runtime) の環境設定、メモリプールの作成、および process_rec 構造体の構築を行う。

シグネチャまたは形状

static process_rec *init_process(int *argc, const char * const * *argv)

入力と出力

  • **入力**:
  • argc: コマンドライン引数の数へのポインタ。
  • argv: コマンドライン引数配列へのポインタ。
  • **出力**:
  • 戻り値: 初期化された process_rec 構造体へのポインタ。

前提条件と不変条件

  • APR が未初期化の状態から呼び出されることを想定している。
  • メモリ割り当て失敗時には abort_on_oom がハンドラとして設定される。
  • 成功時には、サーバープロセスの生存期間中有効なメモリプール (process->pool) が確保される。

アルゴリズムまたは振る舞い

1. apr_app_initialize を呼び出し、APR 環境を初期化する。 2. apr_pool_create でグローバルなメモリプールを作成する。初期化失敗時は stderr にエラーを出力し、apr_terminate を経て終了する。 3. apr_pool_abort_set でメモリ不足時の緊急停止ハンドラを登録し、apr_pool_tag でプールにタグ付けを行う。 4. ap_open_stderr_log で標準エラー出力のログ設定を行う。 5. process_rec 構造体を割り当て、プール、引数情報、実行ファイル名を格納する。 6. reset_process_pconf を呼び出し、設定用メモリプール (pconf) を初期化する。 7. AP_HAS_THREAD_LOCAL が定義されている場合、ap_thread_main_create を呼び出しスレッドコンテキストを初期化する。失敗時はエラーを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/main.c 内の main 関数。
  • **呼び出し先**: apr_app_initialize, apr_pool_create, apr_ctime, apr_time_now, apr_pool_abort_set, apr_pool_tag, ap_open_stderr_log, apr_palloc, reset_process_pconf, apr_filepath_name_get, ap_thread_main_create (条件付き), apr_terminate (エラー時)。

関連データ構造

  • process_rec: サーバープロセスの状態を保持する構造体。
  • apr_pool_t: APR のメモリ管理用プール。

失敗モードと境界条件

  • apr_app_initialize または apr_pool_create が失敗した場合、stderr にエラーメッセージを出力し、exit(1) でプロセスを終了する。
  • スレッドコンテキストの初期化（AP_HAS_THREAD_LOCAL 有効時）に失敗した場合も同様に終了する。

保守担当者向けメモ

  • 本関数は APR が初期化される前の段階で実行されるため、APR の機能に依存したエラー報告には制限がある。
  • XXX コメントにある通り、apr_ctime や apr_time_now が将来的にグローバルプール等に依存するようになった場合、初期化順序の再検討が必要となる。
