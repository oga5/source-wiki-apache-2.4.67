fcfg

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_log_forensic.c:42

目的

Apache HTTP Server におけるフォレンジックログ（証拠保全ログ）のサーバーごとの設定を保持する構造体です。

シグネチャまたは形状

typedef struct fcfg {
    const char *logname;
    apr_file_t *fd;
} fcfg;

入力と出力

  • **入力**: logname（ログファイル名）、fd（ファイルディスクリプタ）。
  • **出力**: なし（本構造体は設定保持用であり、関数ではない）。

前提条件と不変条件

  • logname はログ出力先のファイルまたはパイプを指定します。
  • fd は log_init 実行時に open_log を通じて初期化される必要があります。

アルゴリズムまたは振る舞い

  • サーバーごとの設定を管理し、log_init によるログファイルのオープン、およびリクエスト処理時のログ書き込み先として利用されます。
  • リクエスト処理中、ap_get_module_config を介してサーバー設定から取得され、ログ出力の制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • modules/loggers/mod_log_forensic.c:51, 61: 設定の初期化時に使用。
  • modules/loggers/mod_log_forensic.c:62, 63: 設定のマージ時に使用。
  • modules/loggers/mod_log_forensic.c:73, 184, 235, 257: サーバー設定の取得時に使用。
  • **呼び出し先**: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • apr_file_t: APR（Apache Portable Runtime）のファイルハンドル。
  • server_rec: サーバー設定を保持する Apache コア構造体。

失敗モードと境界条件

  • fd が適切に初期化されていない場合、ログ書き込み時にエラーが発生する可能性があります。
  • logname が無効なパスや書き込み権限のない場所を指定している場合、log_init 段階でログファイルのオープンに失敗します。

保守担当者向けメモ

  • 本構造体は mod_log_forensic モジュールの設定管理の基盤です。
  • リクエスト処理時に r->server->module_config から取得されるため、設定の変更や拡張を行う際は、マージ関数（merge_forensic_log_scfg）との整合性に注意してください。
