destroy_cache

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:61

目的

共有オブジェクトキャッシュ（socache）のインスタンスを破棄し、リソースを解放する。

シグネチャまたは形状

apr_status_t destroy_cache(void *data)

入力と出力

  • **入力**: void *data（server_rec* としてキャストされるサーバーレコードへのポインタ）
  • **出力**: apr_status_t（常に APR_SUCCESS を返す）

前提条件と不変条件

  • socache_instance がグローバル変数として存在すること。
  • socache_provider が初期化済みであること。

アルゴリズムまたは振る舞い

1. グローバル変数 socache_instance が非 NULL であるかを確認する。 2. socache_provider->destroy を呼び出し、キャッシュインスタンスとサーバーレコードを渡して破棄処理を実行する。 3. socache_instance を NULL にリセットする。 4. APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: apr_pool_cleanup_register を介して、設定用プール（pconf）のクリーンアップハンドラとして登録されている（modules/aaa/mod_authn_socache.c:134）。
  • **呼び出し先**: socache_provider->destroy（ap_socache.h で定義されたプロバイダー関数）。

関連データ構造

  • socache_instance: キャッシュバックエンドのインスタンスを保持するグローバル変数。
  • socache_provider: キャッシュ操作関数群を保持する構造体。
  • server_rec: Apache サーバー設定情報。

失敗モードと境界条件

  • socache_instance が既に NULL の場合、何もしない（安全に終了する）。
  • socache_provider が未設定の場合の動作は定義されていない（セグメンテーション違反の可能性がある）。

保守担当者向けメモ

  • 本関数は apr_pool_cleanup_register によって登録されるため、サーバーの停止時や設定の再読み込み時に自動的に呼び出される。
  • socache_instance のリセットは、二重解放を防ぐための重要な不変条件である。
