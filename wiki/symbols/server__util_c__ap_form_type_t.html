ap_form_type_t

定義

  • 種別: enum
  • 定義箇所: server/util.c:2777

目的

application/x-www-form-urlencoded 形式のデータをパースする際の状態マシンにおける、現在の解析状態を表現するために使用されます。

シグネチャまたは形状

typedef enum {
    FORM_NORMAL,
    FORM_AMP,
    FORM_NAME,
    FORM_VALUE,
    FORM_PERCENTA,
    FORM_PERCENTB,
    FORM_ABORT
} ap_form_type_t;

入力と出力

  • 入力: 状態遷移のトリガーとなる入力文字。
  • 出力: 状態遷移後の ap_form_type_t 型の値。

前提条件と不変条件

  • ap_parse_form_data 関数内で状態変数として利用されます。
  • FORM_ABORT 状態に遷移した場合、解析処理は中断される必要があります。

アルゴリズムまたは振る舞い

  • FORM_NAME および FORM_VALUE を中心に、URL エンコードされたデータのキーと値を抽出します。
  • FORM_PERCENTA および FORM_PERCENTB は、パーセントエンコードされた文字（%XX）をデコードするためのサブ状態として機能します。
  • FORM_AMP は区切り文字（&）の検出を管理します。

呼び出し元と呼び出し先

  • 呼び出し元: server/util.c 内の ap_parse_form_data 関数がこの列挙型をローカル変数 state として使用しています。
  • 呼び出し先: 情報なし。

関連データ構造

  • ap_parse_form_data: この列挙型を利用してフォームデータをパースする関数。

失敗モードと境界条件

  • FORM_ABORT: 解析中にエラーが発生した場合や、不正なデータ形式を検出した場合に遷移する終端状態です。

保守担当者向けメモ

  • 本列挙型は server/util.c 内のフォーム解析ロジックに密結合しています。状態遷移のロジックを変更する場合は、ap_parse_form_data 内のスイッチ文または条件分岐を併せて確認してください。
