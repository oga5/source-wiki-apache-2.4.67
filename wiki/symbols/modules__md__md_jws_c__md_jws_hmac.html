md_jws_hmac

定義

  • 種別: function
  • 定義箇所: modules/md/md_jws.c:112

目的

HMAC-SHA256 アルゴリズムを使用して、指定されたペイロードに対する JWS (JSON Web Signature) メッセージを生成する。

シグネチャまたは形状

apr_status_t md_jws_hmac(md_json_t **pmsg, apr_pool_t *p,
                         md_data_t *payload, md_json_t *prot_fields,
                         const md_data_t *hmac_key)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • payload: 署名対象のデータ。
  • prot_fields: 保護ヘッダーに追加するフィールド。
  • hmac_key: 署名に使用する共有鍵。
  • **出力**:
  • pmsg: 生成された JWS JSON オブジェクトへのポインタ（成功時）。失敗時は NULL。
  • 戻り値: apr_status_t 型のステータスコード（成功時 APR_SUCCESS）。

前提条件と不変条件

  • prot_fields は md_json_t 形式で提供される必要がある。
  • 署名アルゴリズムとして強制的に "HS256" がヘッダーに設定される。

アルゴリズムまたは振る舞い

1. prot_fields を複製し、"alg" フィールドに "HS256" を設定する。 2. 保護ヘッダーを JSON 文字列化し、Base64URL エンコードする。 3. payload を Base64URL エンコードする。 4. prot64.pay64 形式の文字列を作成し、md_crypt_hmac64 を呼び出して HMAC 署名を生成する。 5. "protected"、"payload"、"signature" の各フィールドを持つ JSON オブジェクトを構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_acct.c:539
  • **呼び出し先**:
  • md_json_create, md_json_clone, md_json_sets, md_json_writep
  • md_util_base64url_encode
  • md_crypt_hmac64
  • md_log_perror

関連データ構造

  • md_json_t: JWS メッセージの構造体。
  • md_data_t: ペイロードおよび HMAC 鍵のデータコンテナ。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • 保護ヘッダーのシリアライズに失敗した場合、APR_EINVAL を返す。
  • md_crypt_hmac64 がエラーを返した場合、そのステータスコードを伝播し、pmsg は NULL に設定される。

保守担当者向けメモ

  • 本関数は HS256 専用である。他の HMAC アルゴリズムへの拡張が必要な場合は、alg フィールドの設定箇所を修正する必要がある。
  • 戻り値の pmsg が NULL かどうかのチェックは呼び出し側で必須である。
