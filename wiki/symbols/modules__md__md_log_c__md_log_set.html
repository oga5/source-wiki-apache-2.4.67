md_log_set

定義

  • 種別: function
  • 定義箇所: modules/md/md_log.c:53

目的

mod_md モジュール全体で使用されるログ出力のバックエンド（コールバック関数）およびコンテキストを登録し、ログ出力処理を抽象化・差し替え可能にする。

シグネチャまたは形状

void md_log_set(md_log_level_cb *level_cb, md_log_print_cb *print_cb, void *baton)

入力と出力

  • level_cb: ログレベルの判定を行うコールバック関数へのポインタ。
  • print_cb: 実際にログを出力する処理を行うコールバック関数へのポインタ。
  • baton: コールバック関数に渡される不透明なコンテキストデータ（void *）。
  • 出力: なし（void）。

前提条件と不変条件

  • 本関数はモジュールの初期化プロセスにおいて呼び出されることを想定している。
  • 渡されたコールバックおよびコンテキストは、モジュール内の静的変数（log_printv, log_level, log_baton）に保持される。

アルゴリズムまたは振る舞い

1. 引数として受け取った print_cb を静的変数 log_printv に代入する。 2. 引数として受け取った level_cb を静的変数 log_level に代入する。 3. 引数として受け取った baton を静的変数 log_baton に代入する。 これにより、以降の md_log_perror 等の呼び出しが、設定されたバックエンドへ委譲されるようになる。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md.c:883（モジュール初期化時）
  • 呼び出し先: なし（本関数はコールバックを登録するのみ）

関連データ構造

  • md_log_level_cb: ログレベル判定用の関数型。
  • md_log_print_cb: ログ出力用の関数型。

失敗モードと境界条件

  • 特筆すべき失敗モードは定義されていない。引数に NULL を渡した場合の挙動は、後続のログ出力関数側での実装に依存する。

保守担当者向けメモ

  • 本関数は mod_md のログ出力ロジックを外部から注入するためのエントリポイントである。テスト環境や異なるログ出力先への切り替えが必要な場合は、この関数を通じて適切なコールバックを注入すること。
