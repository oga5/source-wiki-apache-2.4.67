get_arguments

定義

  • 種別: function
  • 定義箇所: modules/core/mod_macro.c:91

目的

設定行から引数リストを抽出し、文字列の配列として返します。

シグネチャまたは形状

apr_array_header_t *get_arguments(apr_pool_t *pool, const char *line)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • line: 解析対象の文字列（引数を含む設定行）。
  • **出力**:
  • apr_array_header_t *: 抽出された引数文字列を格納した配列。

前提条件と不変条件

  • line は trim 関数によって前後の空白が除去されます。
  • 返される配列の各要素は、ap_getword_conf によって設定ファイルの構文規則に従って切り出された文字列です。

アルゴリズムまたは振る舞い

1. apr_array_make を使用して、文字列ポインタを格納する空の配列を初期化します。 2. trim を呼び出し、入力文字列の先頭と末尾の空白を削除します。 3. 文字列が空になるまで ap_getword_conf を繰り返し呼び出し、設定ファイル形式の単語（引数）を順次取得します。 4. 取得した各引数を apr_array_push で配列に追加します。 5. 処理のたびに trim を呼び出し、次の引数取得に備えます。 6. 最終的に構築された配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/core/mod_macro.c:767: マクロ定義時の引数解析に使用。
  • modules/core/mod_macro.c:847: Use ディレクティブ実行時の引数置換用リストの生成に使用。
  • **呼び出し先**:
  • trim (内部ユーティリティ)
  • ap_getword_conf (Apache Core API)
  • apr_array_make / apr_array_push (APR)

関連データ構造

  • apr_array_header_t: 引数リストを保持する APR 配列構造体。

失敗モードと境界条件

  • 入力行が空、または空白のみの場合、空の配列が返されます。
  • ap_getword_conf の仕様に基づき、引用符で囲まれた引数やエスケープされた文字は適切に処理されます。

保守担当者向けメモ

  • 本関数は mod_macro における設定解析の基盤であり、引数のパースロジックを集中管理しています。
  • trim 関数が line ポインタを直接操作するため、呼び出し後の line の状態に依存する処理を行う場合は注意が必要です。
