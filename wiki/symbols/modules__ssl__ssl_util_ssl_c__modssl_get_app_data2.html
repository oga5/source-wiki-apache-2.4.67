modssl_get_app_data2

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ssl.c:60

目的

OpenSSL の SSL オブジェクトに関連付けられた、mod_ssl 固有のカスタムアプリケーションデータを取得します。

シグネチャまたは形状

void *modssl_get_app_data2(SSL *ssl);

入力と出力

  • **入力**: SSL *ssl - 対象となる OpenSSL の SSL 接続オブジェクト。
  • **出力**: void * - SSL オブジェクトに格納されているアプリケーションデータへのポインタ。データが設定されていない場合は NULL を返します。

前提条件と不変条件

  • 本関数を呼び出す前に、modssl_init_app_data2_idx が実行され、グローバルなインデックス app_data2_idx が初期化されている必要があります。
  • ssl 引数は有効な SSL オブジェクトを指している必要があります。

アルゴリズムまたは振る舞い

1. 内部で OpenSSL の SSL_get_ex_data 関数を呼び出します。 2. 事前に modssl_init_app_data2_idx で確保されたインデックス app_data2_idx をキーとして使用し、SSL オブジェクトから拡張データを抽出して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_kernel.c:1572: リクエスト処理中に request_rec 構造体を取得するために呼び出されます。
  • **呼び出し先**:
  • OpenSSL API: SSL_get_ex_data

関連データ構造

  • SSL (OpenSSL): 接続状態を保持する構造体。
  • request_rec (Apache): modssl_get_app_data2 の戻り値として格納されることが一般的です。

失敗モードと境界条件

  • ssl が NULL の場合、OpenSSL の SSL_get_ex_data の挙動に従い、未定義の動作またはエラーが発生する可能性があります。
  • インデックス app_data2_idx が初期化されていない状態で呼び出した場合、意図しないデータが返されるか、エラーが発生します。

保守担当者向けメモ

  • 本関数は modssl_set_app_data2 と対で使用されます。
  • 取得されるデータは void * 型であるため、呼び出し側で適切な型（主に request_rec *）へのキャストが必要です。
  • app_data2_idx はモジュール起動時に一度だけ初期化されるため、マルチスレッド環境下でも安全に利用可能です。
