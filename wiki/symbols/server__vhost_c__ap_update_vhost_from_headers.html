ap_update_vhost_from_headers

定義

  • 種別: function
  • 定義箇所: server/vhost.c:1168

目的

リクエストヘッダー（Host ヘッダー等）に基づき、リクエストに関連付けられたサーバー設定（server_rec）を動的に更新し、適切な仮想ホストを決定します。

シグネチャまたは形状

void ap_update_vhost_from_headers(request_rec *r)

入力と出力

  • **入力**: request_rec *r - 現在処理中のリクエスト情報を保持する構造体。
  • **出力**: なし（副作用として r->server が更新されます）。

前提条件と不変条件

  • ap_update_vhost_given_ip が既に呼び出され、接続に関連する仮想ホストの候補リストが r->connection->vhost_lookup_data に格納されている必要があります。
  • r は有効なリクエスト構造体である必要があります。

アルゴリズムまたは振る舞い

1. 内部関数 ap_update_vhost_from_headers_ex(r, 0) を呼び出します。 2. ap_update_vhost_from_headers_ex 内で以下の処理が行われます：

  • fix_hostname を使用してリクエストの Host ヘッダーを正規化します。
  • update_server_from_aliases を使用し、接続時に特定された仮想ホスト候補リスト（vhost_lookup_data）と、リクエストのホスト名・ポートを照合します。
  • 一致する ServerName または ServerAlias を持つ仮想ホストが見つかった場合、r->server をそのサーバー設定に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_request.c:505 (直接の呼び出しを確認)
  • **呼び出し先**:
  • ap_update_vhost_from_headers_ex (本関数から直接呼び出される)

関連データ構造

  • request_rec: リクエストの状態と、更新対象となる server_rec を保持します。
  • server_rec: 仮想ホストの設定情報（ServerName, ServerAlias 等）を保持します。

失敗モードと境界条件

  • Host ヘッダーが存在しない、または不正な形式である場合、デフォルトのサーバー設定が維持されます。
  • 候補リスト内に一致する仮想ホストが存在しない場合、r->server は更新されず、接続時に割り当てられたサーバー設定がそのまま使用されます。

保守担当者向けメモ

  • 本関数は ap_update_vhost_from_headers_ex のラッパーです。詳細なロジックの変更が必要な場合は、server/vhost.c 内の ap_update_vhost_from_headers_ex を参照してください。
  • r->connection->vhost_lookup_data が初期化されていない状態で呼び出された場合の挙動については、ap_update_vhost_from_headers_ex の実装依存となります。
