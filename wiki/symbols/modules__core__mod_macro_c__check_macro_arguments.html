check_macro_arguments

定義

  • 種別: function
  • 定義箇所: modules/core/mod_macro.c:233

目的

マクロ定義における引数リストの妥当性を検証し、重複や命名規則違反を検出する。

シグネチャまたは形状

static const char *check_macro_arguments(apr_pool_t *pool, const ap_macro_t *macro);

入力と出力

  • **入力**:
  • pool: エラーメッセージ生成に使用するメモリプール。
  • macro: 検証対象のマクロ定義（ap_macro_t 構造体）。
  • **出力**:
  • 正常時: NULL を返す。
  • 異常時: エラー内容を示す文字列（const char *）を返す。

前提条件と不変条件

  • macro->arguments が有効な配列であることを前提とする。
  • マクロ名および定義場所（macro->location）がエラーメッセージ生成のために設定されている必要がある。

アルゴリズムまたは振る舞い

1. マクロの引数リストを順次走査する。 2. 引数名が空文字列である場合、エラーを返す。 3. looks_like_an_argument を用いて、引数名が期待されるプレフィックス（ARG_PREFIX）で始まっていない場合に警告をログ出力する。 4. 引数リスト内で重複する名前がないか比較する。重複がある場合はエラーを返す。 5. 引数名同士で共通のプレフィックスを持つ場合、警告をログ出力する。 6. すべてのチェックを通過した場合、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_macro.c:769
  • **呼び出し先**:
  • looks_like_an_argument (引数名の形式チェック)
  • strlen, strcmp, strncmp (標準文字列関数)
  • apr_psprintf (エラーメッセージ生成)
  • ap_log_error (警告ログ出力)

関連データ構造

  • ap_macro_t: マクロの定義（名前、引数リスト、定義場所）を保持する構造体。

失敗モードと境界条件

  • **引数名が空**: ltabi == 0 の場合にエラーを返す。
  • **引数名の重複**: !strcmp(tab[i], tab[j]) が真の場合にエラーを返す。
  • **プレフィックスの欠如**: looks_like_an_argument が偽を返す場合、警告ログを出力するが処理は継続する。

保守担当者向けメモ

  • 本関数は設定解析フェーズで実行される。エラーメッセージは cmd->temp_pool 等のライフサイクルに依存するため、呼び出し元でのメモリ管理に注意すること。
  • 警告ログの出力には APLOGNO(02796) および APLOGNO(02797) が使用されている。
