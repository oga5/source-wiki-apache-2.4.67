SSLModConfigRec

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:666

目的

Apache HTTP Server の mod_ssl モジュールにおいて、プロセス生存期間を通じて保持されるグローバルな設定および状態を管理するための構造体です。

シグネチャまたは形状

typedef struct {
    pid_t           pid;
    apr_pool_t     *pPool;
    BOOL            bFixed;
    long            sesscache_mode;
    const ap_socache_provider_t *sesscache;
    ap_socache_instance_t *sesscache_context;
    apr_global_mutex_t   *pMutex;
    apr_array_header_t   *aRandSeed;
    apr_hash_t     *tVHostKeys;
    apr_hash_t     *tPrivateKey;
    const char     *szCryptoDevice;
#ifdef HAVE_OCSP_STAPLING
    const ap_socache_provider_t *stapling_cache;
    ap_socache_instance_t *stapling_cache_context;
    apr_global_mutex_t   *stapling_cache_mutex;
    apr_global_mutex_t   *stapling_refresh_mutex;
#endif
#ifdef HAVE_OPENSSL_KEYLOG
    apr_file_t      *keylog_file;
#endif
#ifdef HAVE_FIPS
    BOOL             fips;
#endif
    modssl_snivhpolicy_t snivh_policy;
} SSLModConfigRec;

入力と出力

  • **入力**: サーバー起動時に ssl_config_global_create を通じてメモリが割り当てられ、設定ディレクティブに基づいて各フィールドが初期化されます。
  • **出力**: モジュール内の各フック関数が myModConfig マクロを介して本構造体を参照し、セッションキャッシュの制御や秘密鍵の取得、ミューテックスの排他制御を行います。

前提条件と不変条件

  • 本構造体はサーバーのライフサイクル全体で生存します。
  • bFixed フラグは、設定が確定（固定）されたかどうかを示します。
  • tPrivateKey は、DER 形式の秘密鍵を vhost-id:index 形式のキーで管理するハッシュテーブルです。

アルゴリズムまたは振る舞い

  • サーバー起動時に初期化され、パスフレーズのキャッシュやセッションキャッシュのプロバイダ設定を保持します。
  • HAVE_OCSP_STAPLING が定義されている場合、OCSP ステープリング用のキャッシュとミューテックスを管理します。
  • tPrivateKey を通じて、プロセス全体で共有される秘密鍵データへのアクセスを提供します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_config.c: ssl_config_global_create, ssl_config_global_fix, ssl_config_global_isfixed, および myModConfig マクロ経由での各設定関数。
  • modules/ssl/ssl_engine_init.c: ssl_init_Module。
  • **呼び出し先**:
  • apr ライブラリ（メモリプール、ハッシュテーブル、ミューテックス操作）。
  • ap_socache（セッションキャッシュプロバイダ）。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_hash_t: 秘密鍵および仮想ホスト鍵の格納に使用。
  • ap_socache_provider_t: セッションキャッシュおよび OCSP ステープリングのプロバイダ定義。
  • apr_global_mutex_t: プロセス間同期に使用。

失敗モードと境界条件

  • メモリ割り当て失敗時（apr_palloc）の挙動は Apache の標準的なエラー処理に依存します。
  • tPrivateKey や tVHostKeys へのアクセス時にキーが存在しない場合、ハッシュテーブルの検索結果は NULL となります。

保守担当者向けメモ

  • 本構造体はプロセス生存期間中保持されるため、ABI 互換性に注意が必要です。
  • OpenSSL 3.0 移行に伴い、機能フラグ（HAVE_*）による条件分岐が構造体定義に含まれています。変更時はこれらのマクロ定義を確認してください。
