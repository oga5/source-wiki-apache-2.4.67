ap_get_protocol

定義

  • 種別: function
  • 定義箇所: server/protocol.c:2397

目的

現在のコネクションに関連付けられたプロトコル名を特定します。

シグネチャまたは形状

const char *ap_get_protocol(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c (現在のコネクション構造体)
  • **出力**: プロトコル名を表す const char * 文字列。

前提条件と不変条件

  • c は有効な conn_rec 構造体を指している必要があります。
  • 戻り値は ap_run_protocol_get(c) の結果、またはデフォルト値のいずれかとなります。

アルゴリズムまたは振る舞い

1. ap_run_protocol_get(c) を呼び出し、登録されているプロトコル取得フックを実行します。 2. フックがプロトコル名を返した場合はその値を返します。 3. フックが NULL を返した場合は、デフォルト値として AP_PROTOCOL_HTTP1 を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: ap_run_protocol_get (フック関数)
  • **呼び出し元**:
  • modules/http2/h2_c1.c
  • modules/http2/h2_switch.c
  • modules/ssl/ssl_engine_kernel.c
  • server/core.c
  • server/protocol.c
  • server/scoreboard.c

関連データ構造

  • conn_rec: コネクションの状態を保持する構造体。

失敗モードと境界条件

  • c が NULL の場合の挙動は定義されていません。
  • フックが未登録または NULL を返す場合、常に AP_PROTOCOL_HTTP1 が返されます。

保守担当者向けメモ

  • 本関数はプロトコルネゴシエーション（ALPN等）の結果を確認するために広く利用されています。
  • 戻り値の文字列は静的メモリまたはプール上のメモリを指しているため、呼び出し側での解放は不要です。
