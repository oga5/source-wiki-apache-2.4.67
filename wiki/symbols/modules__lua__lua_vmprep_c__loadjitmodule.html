loadjitmodule

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:314

目的

LuaJIT モジュールを初期化し、Lua VM 内で jit ライブラリをロードしてその start 関数を取得する。

シグネチャまたは形状

int loadjitmodule(lua_State *L, apr_pool_t *lifecycle_pool)

入力と出力

  • **入力**:
  • L: 操作対象の lua_State。
  • lifecycle_pool: ログ出力に使用する apr_pool_t。
  • **出力**:
  • 成功時: 0 を返し、Lua スタックのトップに jit.start 関数を配置する。
  • 失敗時: 1 を返し、エラーメッセージをログに出力する。

前提条件と不変条件

  • L は有効な lua_State であること。
  • スタックのトップに、ロード対象となるモジュール名（文字列）が配置されていること。

アルゴリズムまたは振る舞い

1. グローバルな require 関数をスタックに取得する。 2. 文字列 "jit." をプッシュし、スタックトップにあるモジュール名と連結して "jit.<name>" を生成する。 3. lua_pcall を使用して当該モジュールをロードする。 4. 失敗した場合はエラーメッセージを lifecycle_pool を用いてログ出力し、終了する。 5. 成功した場合は、ロードされたモジュールテーブルから start フィールドを取得する。 6. 不要になったモジュールテーブルをスタックから削除し、start 関数をスタックトップに残して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_vmprep.c:389
  • **呼び出し先**:
  • lua_getglobal, lua_pushliteral, lua_pushvalue, lua_concat, lua_pcall, lua_getfield, lua_remove (Lua C API)
  • ap_log_perror (Apache ログ出力)

関連データ構造

  • lua_State: Lua VM の状態を保持する構造体。
  • apr_pool_t: Apache のメモリプール。

失敗モードと境界条件

  • lua_pcall が失敗した場合（モジュールが見つからない、または実行時エラーが発生した場合）、関数は 1 を返し、ap_log_perror によって APLOGNO(01480) としてエラー内容が記録される。

保守担当者向けメモ

  • 本関数は LuaJIT 特有の初期化処理を行うため、標準の Lua VM を使用している場合には注意が必要である。
  • スタック操作（lua_remove 等）は、スタックの状態に強く依存しているため、変更を加える際はスタックインデックスの整合性に留意すること。
