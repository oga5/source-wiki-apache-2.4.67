request_rec

定義

  • 種別: typedef
  • 定義箇所: server/config.c:236
  • 構造体定義: httpd.h:822

目的

HTTPリクエストの処理過程において、リクエスト固有の情報を保持・管理するための主要なデータ構造です。

シグネチャまたは形状

typedef として定義されており、ハンドラ関数等の引数として request_rec * の形式で頻繁に渡されます。

入力と出力

  • 入力: HTTPリクエスト処理の各フェーズにおいて、リクエストコンテキストとして関数に渡されます。
  • 出力: 構造体内のメンバ（リクエストヘッダ、URI、ステータス、フィルタチェーン等）が、処理の進行に伴い更新されます。

前提条件と不変条件

  • 構造体のメモリ管理は、関連する apr_pool_t に依存します。
  • リクエストのライフサイクルを通じて有効であることが期待されます。

アルゴリズムまたは振る舞い

リクエストの解析、認証、アクセス制御、コンテンツ生成、およびレスポンス送信の各段階で、サーバーおよび各モジュールがこの構造体を参照・変更することで、リクエストの状態を共有します。

呼び出し元と呼び出し先

  • 呼び出し元（例）: ap_expr_exec や ap_expr_str_exec などの式評価関数、および各モジュールのハンドラ関数（mod_example_1.c, mod_example_2.c 等）。
  • 呼び出し先: 構造体そのものはデータ構造であり、直接的な呼び出し先はありませんが、この構造体を引数に取る関数群が多数存在します。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • ap_expr_info_t: 式評価に関連して参照されます。
  • ap_directive_t: 設定ツリー構造。

失敗モードと境界条件

  • 構造体が NULL の状態で関数に渡された場合、多くの関数でセグメンテーション違反が発生する可能性があります。
  • 過去のバージョン（ap_mmn.h 参照）でメンバの追加や削除が行われており、バイナリ互換性には注意が必要です。

保守担当者向けメモ

  • server/config.c での定義は typedef ですが、実体は httpd.h で定義されています。
  • リクエスト処理の根幹をなす構造体であるため、メンバの変更はモジュール間のバイナリ互換性に重大な影響を及ぼします。変更時は ap_mmn.h（Module Magic Number）の更新を検討してください。
