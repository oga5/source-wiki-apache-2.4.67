ap_update_child_status_from_indexes

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:575

目的

指定された子プロセスおよびスレッドのインデックスに基づき、スコアボード上のワーカー状態を更新します。

シグネチャまたは形状

int ap_update_child_status_from_indexes(int child_num,
                                        int thread_num,
                                        int status,
                                        request_rec *r)

入力と出力

  • **入力**:
  • child_num: 更新対象のプロセスインデックス。
  • thread_num: 更新対象のスレッドインデックス。
  • status: 設定するステータスコード（SERVER_READY, SERVER_DEAD 等）。
  • r: リクエスト情報を含む request_rec 構造体（NULL 可）。
  • **出力**:
  • 成功時は update_child_status_internal の戻り値を返します。
  • child_num が負の値の場合、更新を行わず -1 を返します。

前提条件と不変条件

  • スコアボードの各スロットは単一の所有者によって書き込まれる設計であるため、本関数呼び出し時に明示的なロックは不要です。
  • r が提供される場合、その接続情報およびサーバー情報が内部更新に使用されます。

アルゴリズムまたは振る舞い

1. child_num が 0 未満であれば、エラーとして -1 を返します。 2. r が存在する場合、その connection および server メンバを抽出し、update_child_status_internal を呼び出します。 3. r が NULL の場合、接続およびサーバー情報として NULL を渡して update_child_status_internal を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/mpmt_os2/mpmt_os2_child.c
  • server/mpm/netware/mpm_netware.c
  • **呼び出し先**:
  • update_child_status_internal (同一ファイル内の静的関数)

関連データ構造

  • request_rec: リクエスト処理のコンテキスト情報。
  • worker_score: スコアボード内のワーカーごとの状態保持構造体。

失敗モードと境界条件

  • child_num < 0: 不正なインデックスとして処理を中断し -1 を返します。
  • r が NULL の場合: 接続やサーバー情報に依存する詳細なステータス更新はスキップされます。

保守担当者向けメモ

  • 本関数は ap_update_child_status 等のファミリー関数の一種であり、インデックスベースでの直接指定が必要な MPM 実装から利用されます。
  • 内部処理の詳細は update_child_status_internal に集約されています。
