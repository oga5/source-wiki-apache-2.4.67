create_ef_dir_conf

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ext_filter.c:85

目的

ディレクトリ単位の外部フィルタ設定構造体（ef_dir_t）を生成し、初期値を設定します。

シグネチャまたは形状

static void *create_ef_dir_conf(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t ポインタ。
  • dummy: 使用されない引数（ディレクトリパス等）。
  • **出力**:
  • void *: 初期化された ef_dir_t 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体はゼロクリアされた状態で割り当てられます。
  • 戻り値の ef_dir_t は、リクエスト処理の生存期間中有効なメモリプール上に確保されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して ef_dir_t サイズのメモリを確保します。 2. log_stderr メンバを -1 に初期化します。 3. onfail メンバを -1 に初期化します。 4. 初期化済みの構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_ext_filter.c:948（module 構造体のディレクティブ定義内での参照が示唆されています）。
  • **呼び出し先**: apr_pcalloc（APR API）。

関連データ構造

  • ef_dir_t: ディレクトリごとの動作設定（エラー時の挙動やログ出力設定）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボートする可能性があります（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は Apache の設定マージプロセスの一部として呼び出されます。
  • log_stderr および onfail の初期値 -1 は、設定が未指定であることを示すフラグとして後続の処理で使用されます。
