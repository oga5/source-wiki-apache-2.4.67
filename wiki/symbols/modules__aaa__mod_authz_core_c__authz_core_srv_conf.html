authz_core_srv_conf

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authz_core.c:94

目的

サーバーレベルでの認可設定を保持するための構造体です。主に AuthzProviderAlias によって定義された認可プロバイダーのエイリアス情報を管理します。

シグネチャまたは形状

typedef struct authz_core_srv_conf {
    apr_hash_t *alias_rec;
} authz_core_srv_conf;

入力と出力

  • **入力**: alias_rec は apr_hash_t 型であり、認可プロバイダーのエイリアス名（キー）とそれに対応する設定情報（値）を格納します。
  • **出力**: サーバー設定の一部として、他の認可関連関数から参照されます。

前提条件と不変条件

  • alias_rec は APR プールを使用して割り当てられる必要があります。
  • サーバーのライフサイクルを通じて、認可プロバイダーのエイリアスを解決するためのハッシュテーブルとして機能します。

アルゴリズムまたは振る舞い

  • サーバー設定の初期化時およびマージ時に、認可プロバイダーのエイリアスを登録・保持するために使用されます。
  • AuthzProviderAlias ディレクティブが解析される際、この構造体の alias_rec にエントリが追加されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の箇所でこの構造体へのポインタが使用されています。
  • modules/aaa/mod_authz_core.c:180
  • modules/aaa/mod_authz_core.c:206
  • modules/aaa/mod_authz_core.c:309
  • **呼び出し先**: 構造体内部の alias_rec は、APR ハッシュテーブル操作関数（apr_hash_get 等）によって操作されます。

関連データ構造

  • provider_alias_rec: alias_rec ハッシュテーブルに格納される値の型として関連しています。
  • apr_hash_t: APR が提供するハッシュテーブル構造体。

失敗モードと境界条件

  • メモリ不足により apr_hash_t の初期化やエントリの追加が失敗する可能性があります。
  • 存在しないエイリアスを参照した場合の挙動は、この構造体を利用する側のロジックに依存します。

保守担当者向けメモ

  • 本構造体はサーバー設定（server_rec）に関連付けられます。
  • authz_core_srv_conf を操作する関数（create_authz_core_svr_config や merge_authz_core_svr_config）と併せて確認してください。
