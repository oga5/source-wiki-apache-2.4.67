pstack_dump

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:66

目的

Lua スタックの内容をデバッグログとして出力する。

シグネチャまたは形状

static void pstack_dump(lua_State *L, apr_pool_t *r, int level, const char *msg)

入力と出力

  • L: 操作対象の lua_State。
  • r: ログ出力に使用する Apache メモリプール。
  • level: ログレベル（APLOG_DEBUG 等）。
  • msg: ログに付与する識別メッセージ。
  • 出力: 戻り値なし。副作用として ap_log_perror を通じてエラーログへスタック内容を書き出す。

前提条件と不変条件

  • lua_gettop(L) を使用してスタックの現在の深さを取得する。
  • スタック内の各要素の型を lua_type で判定し、型に応じた文字列変換を行う。

アルゴリズムまたは振る舞い

1. 指定された msg を含むヘッダーログを出力する。 2. スタックのトップ（top）までループし、各インデックス i の要素を順次処理する。 3. switch 文により、LUA_TSTRING, LUA_TNUMBER, LUA_TBOOLEAN 等の型を判定する。 4. 各型に対応する lua_tostring, lua_tonumber 等の API を呼び出し、値をログに記録する。 5. テーブルや関数などの複雑な型については、型名のみをログに出力する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt 内では確認できません。
  • 呼び出し先: lua_gettop, lua_type, lua_tostring, lua_toboolean, lua_tonumber, lua_typename (Lua C API)、および ap_log_perror (Apache API)。

関連データ構造

  • lua_State: Lua VM の状態を保持する構造体。
  • apr_pool_t: Apache のメモリ管理用プール。

失敗モードと境界条件

  • スタックが空の場合、ループは実行されずヘッダーのみが出力される。
  • 未知の型（default ケース）に遭遇した場合、lua_typename を使用して型名を表示する。

保守担当者向けメモ

  • 本関数は #if 0 で無効化されている可能性がある（モジュール概要の「読者向けメモ」参照）。
  • デバッグ目的の関数であり、本番環境でのパフォーマンスには影響しないよう設計されている。
