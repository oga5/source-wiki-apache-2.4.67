configure_redirect

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_dir.c:98

目的

DirectoryIndexRedirect ディレクティブの設定値を解析し、ディレクトリインデックス解決時のリダイレクトステータスコードを dir_config_rec に設定する。

シグネチャまたは形状

static const char *configure_redirect(cmd_parms *cmd, void *d_, const char *arg1)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • d_: 現在のディレクトリ設定構造体 (dir_config_rec)。
  • arg1: 設定値文字列（"ON", "OFF", "permanent", "temp", "seeother", または 300-399 の数値）。
  • **出力**:
  • 成功時: NULL。
  • 失敗時: エラーメッセージ文字列。

前提条件と不変条件

  • d_ は dir_config_rec 型としてキャスト可能であること。
  • arg1 は NULL でないこと。

アルゴリズムまたは振る舞い

1. arg1 を大文字小文字を区別せずに比較し、対応する HTTP ステータスコードを決定する。

  • "ON", "temp": HTTP_MOVED_TEMPORARILY (302)
  • "permanent": HTTP_MOVED_PERMANENTLY (301)
  • "seeother": HTTP_SEE_OTHER (303)
  • "OFF": REDIRECT_OFF

    • 2. 数値の場合、atoi で変換し、ap_is_HTTP_REDIRECT マクロを用いて 300-399 の範囲内か検証する。 3. 妥当な値であれば d->redirect_index にステータスコードを格納し、NULL を返す。 4. 不正な値の場合はエラーメッセージを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_dir.c:137 にて AP_INIT_TAKE1 マクロを通じて設定ディレクティブ DirectoryIndexRedirect に登録されている。
  • **呼び出し先**: strcasecmp, apr_isdigit, atoi, ap_is_HTTP_REDIRECT (Apache API)。

関連データ構造

  • dir_config_rec: ディレクトリごとの設定を保持する構造体。本関数はこの中の redirect_index メンバを更新する。

失敗モードと境界条件

  • arg1 が指定されたキーワードのいずれにも該当せず、かつ数値でもない場合にエラーメッセージを返す。
  • 数値が 300-399 の範囲外である場合にエラーメッセージを返す。

保守担当者向けメモ

  • 本関数は設定読み込み時にのみ実行される。
  • REDIRECT_OFF は mod_dir.c 内で定義された内部定数であり、リダイレクトが無効であることを示す。
