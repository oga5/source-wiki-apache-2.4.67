imap_conf_rec

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_imagemap.c:84

目的

Apache HTTP Server の mod_imagemap モジュールにおいて、ディレクトリ単位の設定情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    char *imap_menu;
    char *imap_default;
    char *imap_base;
} imap_conf_rec;

入力と出力

  • **入力**: Apache の設定ファイル（httpd.conf 等）で指定された ImapMenu、ImapDefault、ImapBase ディレクティブの値が、設定読み込み時に各メンバに格納されます。
  • **出力**: モジュール内のリクエスト処理において、リダイレクト先やメニュー表示の挙動を制御するための設定値として参照されます。

前提条件と不変条件

  • 各メンバは apr_palloc 等によりリクエストプールまたは設定プールからメモリが割り当てられます。
  • imap_menu、imap_default、imap_base は、設定されていない場合 NULL となる可能性があります。

アルゴリズムまたは振る舞い

  • 本構造体は、create_imap_dir_config で初期化され、merge_imap_dir_configs で親ディレクトリの設定とマージされます。
  • imap_cmds テーブルにおいて、APR_OFFSETOF を使用して各メンバへのオフセットが指定されており、Apache の設定解析機構が直接値を書き込めるようになっています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_imap_dir_config (メモリ確保)
  • merge_imap_dir_configs (設定マージ)
  • imap_cmds 定義テーブル (設定値の格納先として参照)
  • imap_handler_internal (設定値の読み取り)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • command_rec (設定ディレクティブ定義用)
  • request_rec (リクエスト処理時に本構造体を参照する際に使用)

失敗モードと境界条件

  • メモリ不足時には apr_palloc が失敗する可能性があります。
  • 設定値が未定義（NULL）の場合、モジュール側でデフォルトの挙動（メニュー表示の有無やベースURLの解釈）を決定するロジックが必要です。

保守担当者向けメモ

  • 本構造体にメンバを追加する場合は、imap_cmds テーブルの定義および merge_imap_dir_configs 内のマージロジックを更新する必要があります。
  • imap_menu 等の文字列はリクエストプールに依存するため、手動でのメモリ解放は不要です。
