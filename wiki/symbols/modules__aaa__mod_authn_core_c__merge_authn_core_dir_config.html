merge_authn_core_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_core.c:83

目的

ディレクトリ単位の認証設定（authn_core_dir_conf）をマージし、親ディレクトリの設定と子ディレクトリの設定を統合する。

シグネチャまたは形状

static void *merge_authn_core_dir_config(apr_pool_t *a, void *basev, void *newv)

入力と出力

  • **入力**:
  • a: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定（authn_core_dir_conf へのポインタ）。
  • newv: 子ディレクトリの設定（authn_core_dir_conf へのポインタ）。
  • **出力**:
  • マージされた新しい authn_core_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および newv は authn_core_dir_conf 型としてキャスト可能であること。
  • 戻り値の構造体は、メモリプール a から新たに割り当てられる。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい設定構造体 conf をメモリプール上に確保する。 2. AuthType のマージ:

  • new 側で設定されている場合、new の値を優先する。
  • 設定されていない場合、base の値を継承する。

    • 3. AuthName のマージ:

  • new 側で値が存在する場合、new の値を優先する。
  • 値が存在しない場合、base の値を継承する。

    • 4. マージ済みの conf を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_core.c:421（Apache の設定マージ処理の一部として呼び出される）。
  • **呼び出し先**: 特になし（標準的なメモリ割り当て関数 apr_pcalloc を除く）。

関連データ構造

  • authn_core_dir_conf: ディレクトリごとの認証設定を保持する構造体。ap_auth_type や ap_auth_name を含む。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性がある。
  • basev または newv が NULL の場合の挙動については、本ソースコード内では明示的なチェックは行われていない。

保守担当者向けメモ

  • 本関数は、Apache の設定継承メカニズムにおいて「上書き（override）」を基本とするマージロジックを実装している。
  • AuthType の設定有無は auth_type_set フラグで管理されており、マージ時にこのフラグも適切に引き継がれる。
