send_416

定義

  • 種別: function
  • 定義箇所: modules/http/byterange_filter.c:385

目的

HTTP Range リクエストが要求する範囲が不適切な場合に、416 (Range Not Satisfiable) エラーレスポンスを生成し、フィルタチェーンへ送信する。

シグネチャまたは形状

static apr_status_t send_416(ap_filter_t *f, apr_bucket_brigade *tmpbb)

入力と出力

  • **入力**:
  • f: 現在のフィルタコンテキスト (ap_filter_t)。
  • tmpbb: エラーバケットを格納するためのバケットブリゲード (apr_bucket_brigade)。
  • **出力**:
  • apr_status_t: ap_pass_brigade の実行結果を返す。

前提条件と不変条件

  • f->r（リクエストレコード）が有効であること。
  • tmpbb は呼び出し元によって適切に初期化されていること。

アルゴリズムまたは振る舞い

1. ap_remove_output_filter(f) を呼び出し、現在のフィルタをチェーンから削除する。 2. f->r->status を HTTP_OK に設定する。 3. ap_bucket_error_create を使用して HTTP_RANGE_NOT_SATISFIABLE エラーバケットを作成し、tmpbb に挿入する。 4. apr_bucket_eos_create を使用して EOS（End of Stream）バケットを作成し、tmpbb に挿入する。 5. ap_pass_brigade を呼び出し、構築したブリゲードを下流のフィルタへ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/byterange_filter.c:470
  • modules/http/byterange_filter.c:556
  • **呼び出し先**:
  • ap_remove_output_filter
  • ap_bucket_error_create
  • APR_BRIGADE_INSERT_TAIL
  • apr_bucket_eos_create
  • ap_pass_brigade

関連データ構造

  • ap_filter_t: フィルタチェーンのノード。
  • apr_bucket_brigade: バケットのコンテナ。
  • apr_bucket: HTTP レスポンスを構成するデータ単位。

失敗モードと境界条件

  • ap_pass_brigade がエラーを返した場合、そのステータスがそのまま呼び出し元に伝播される。
  • メモリ不足時には ap_bucket_error_create や apr_bucket_eos_create が失敗する可能性がある。

保守担当者向けメモ

  • 本関数はエラー発生時にフィルタチェーンから自身を切り離すため、呼び出し後は当該フィルタが再実行されることはない。
  • f->r->status を HTTP_OK に設定しているのは、エラーバケットが下流で適切に処理され、最終的な HTTP ステータスコードとして 416 が設定されることを前提としている。
