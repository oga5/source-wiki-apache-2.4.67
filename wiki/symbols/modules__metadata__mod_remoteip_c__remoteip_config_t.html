remoteip_config_t

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_remoteip.c:51

目的

mod_remoteip モジュールにおけるサーバー単位の設定情報を保持するための構造体です。信頼するプロキシのリストや、クライアント IP を特定するためのヘッダー名、PROXY プロトコルの有効化状態などを管理します。

シグネチャまたは形状

typedef struct {
    const char *header_name;
    const char *proxies_header_name;
    apr_array_header_t *proxymatch_ip;
    remoteip_addr_info *proxy_protocol_enabled;
    remoteip_addr_info *proxy_protocol_disabled;
    apr_array_header_t *disabled_subnets;
    apr_pool_t *pool;
} remoteip_config_t;

入力と出力

  • **入力**: Apache の設定ディレクティブ（RemoteIPHeader, RemoteIPProxyProtocol 等）を通じて値が設定されます。
  • **出力**: モジュール内の各フック関数（remoteip_modify_request 等）がこの構造体を参照し、リクエストの接続情報（r->useragent_addr 等）を上書きするための判断基準として使用します。

前提条件と不変条件

  • 設定はサーバーのプール（pconf）で管理されます。
  • proxymatch_ip に格納される信頼プロキシのリストは、頻繁に遭遇するプロキシが先頭に来るように構成することが推奨されます。

アルゴリズムまたは振る舞い

  • RemoteIPHeader で指定されたヘッダーからプロキシ経由の IP リストを抽出し、proxymatch_ip に基づいて信頼できるプロキシを辿ることで、最終的なクライアント IP を特定します。
  • PROXY プロトコルが有効な場合、接続フィルタを通じてバイナリヘッダーを解析し、接続情報を抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_remoteip.c 内の各種設定関数（ap_get_module_config を介して取得）およびリクエスト処理フック。
  • **呼び出し先**: apr_array_header_t や remoteip_addr_info などの APR/モジュール内データ構造を操作します。

関連データ構造

  • remoteip_addr_info: PROXY プロトコルの有効/無効状態を保持。
  • apr_array_header_t: 信頼するプロキシの IP リストおよび無効化されたサブネットの保持に使用。
  • apr_pool_t: 設定情報のメモリ管理に使用。

失敗モードと境界条件

  • 設定ミスにより信頼リストが不適切に構成された場合、セキュリティ低下を招く可能性があります。
  • PROXY プロトコル使用時、ヘッダーが分割されて届く場合は remoteip_filter_context と連携して状態を保持する必要があります。

保守担当者向けメモ

  • remoteip_modify_request は post_read_request の APR_HOOK_FIRST で実行されるため、他のモジュールが IP を参照する前に確実に書き換えが行われる設計となっています。
  • remoteip_sockaddr_compat を用いたマッチングではポート番号 0 をワイルドカードとして扱うなど、柔軟なマッチングが行われます。
