h2_bucket_eos_make

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_eos.c:59

目的

HTTP/2 ストリームの終了（EOS）を表現するカスタム APR バケットを初期化し、指定されたストリームと関連付ける。

シグネチャまたは形状

apr_bucket *h2_bucket_eos_make(apr_bucket *b, h2_stream *stream)

入力と出力

  • **入力**:
  • b: 初期化対象の apr_bucket 構造体へのポインタ。
  • stream: このバケットが終了を通知する対象となる h2_stream 構造体へのポインタ。
  • **出力**:
  • 初期化済みの apr_bucket 構造体へのポインタを返す。

前提条件と不変条件

  • b は apr_bucket_alloc 等でメモリが確保された有効なバケットであること。
  • stream は有効な h2_stream オブジェクトであること。
  • バケットの型は h2_bucket_type_eos に設定され、メタデータバケットとして振る舞う。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を使用して、バケットのプライベートデータ領域として h2_bucket_eos 構造体を割り当てる。 2. 割り当てた構造体の stream メンバに、引数の stream を格納する。 3. apr_bucket_shared_make を呼び出し、バケットを共有可能な状態に初期化する。 4. バケットの type を h2_bucket_type_eos に設定する。 5. 初期化されたバケットを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_bucket_eos.c:80 (h2_bucket_eos_create 内から呼び出される)
  • **呼び出し先**:
  • apr_bucket_alloc (APR)
  • apr_bucket_shared_make (APR)

関連データ構造

  • h2_bucket_eos: ストリームへの参照を保持する内部構造体。
  • h2_bucket_type_eos: このバケットの挙動（破棄処理など）を定義する apr_bucket_type_t 構造体。
  • h2_stream: HTTP/2 ストリームの状態管理オブジェクト。

失敗モードと境界条件

  • メモリ不足時に apr_bucket_alloc が失敗した場合の挙動は、APR の標準的なメモリ割り当てエラーに準ずる。

保守担当者向けメモ

  • 本関数はバケットの型定義とストリームの紐付けのみを行う。実際のストリーム終了イベントのディスパッチは、バケット破棄時に bucket_destroy を通じて行われる。
