cachedmap

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:397

目的

外部マップルックアップの結果を保持し、パフォーマンス向上のためのキャッシュエントリとして機能します。

シグネチャまたは形状

typedef struct {
    apr_time_t mtime;
    apr_pool_t *pool;
    apr_hash_t *entries;
} cachedmap;

入力と出力

  • 入力: apr_pool_t を用いたメモリ割り当て時に初期化されます。
  • 出力: entries フィールドを通じて、キャッシュされたキーと値のペア（apr_hash_t）を提供します。

前提条件と不変条件

  • pool はキャッシュの生存期間を管理するために使用されます。
  • mtime はキャッシュの有効期限や更新時刻の判定に使用されます。

アルゴリズムまたは振る舞い

  • mod_rewrite の外部マップルックアップにおいて、結果をメモリ上に保持するために使用されます。
  • apr_palloc を介して cachep->pool からメモリが割り当てられ、リクエストのライフサイクルとは独立した生存期間を持つことが可能です。

呼び出し元と呼び出し先

  • 呼び出し元: modules/mappers/mod_rewrite.c 内のキャッシュ管理ロジック（1146行目付近でインスタンス化）。
  • 呼び出し先: apr_hash_t 関連の APR 関数（apr_hash_get, apr_hash_set 等）が entries に対して使用されます。

関連データ構造

  • cache: キャッシュ全体の管理構造体。
  • apr_hash_t: キャッシュエントリの格納に使用。
  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • メモリ不足時: apr_palloc が失敗する可能性があります。
  • キャッシュの競合: マルチスレッド環境下では、apr_thread_mutex_t や apr_global_mutex_t による排他制御が必須となります。

保守担当者向けメモ

  • キャッシュ関連のメモリはリクエストプールではなく cachep->pool を使用しているため、メモリリークには特に注意してください。
  • mtime の更新ロジックが、外部マップの変更検知と正しく同期しているか確認してください。
