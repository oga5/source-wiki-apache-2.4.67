stapling_get_issuer

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_stapling.c:93

目的

指定された証明書（X509）の発行者（Issuer）証明書を検索し、取得します。OCSP Stapling において、証明書の検証チェーンを構築するために使用されます。

シグネチャまたは形状

static X509 *stapling_get_issuer(modssl_ctx_t *mctx, X509 *x)

入力と出力

  • **入力**:
  • mctx: SSL コンテキスト情報を含む modssl_ctx_t 構造体。
  • x: 発行者を特定したい対象の X509 証明書。
  • **出力**:
  • 発行者が見つかった場合はその X509 オブジェクトへのポインタ。
  • 見つからない場合は NULL。

前提条件と不変条件

  • mctx および x は有効なポインタである必要があります。
  • OpenSSL のバージョンに応じて、参照カウントの操作（CRYPTO_add または X509_up_ref）が適切に行われます。

アルゴリズムまたは振る舞い

1. **チェーン証明書の検索**: mctx->ssl_ctx に設定された extra_certs（中間証明書リスト）を走査し、X509_check_issued を用いて発行者を特定します。一致した場合は参照カウントを増やして返却します。 2. **ストア検索**: extra_certs に存在しない場合、SSL_CTX に関連付けられた X509_STORE を使用します。 3. X509_STORE_CTX を初期化し、X509_STORE_CTX_get1_issuer を呼び出して発行者を検索します。 4. 検索に使用した X509_STORE_CTX をクリーンアップし、結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_util_stapling.c:145
  • **呼び出し先**:
  • SSL_CTX_get_cert_store
  • sk_X509_num / sk_X509_value
  • X509_check_issued
  • X509_STORE_CTX_new / init / get1_issuer / cleanup / free
  • CRYPTO_add または X509_up_ref (OpenSSL バージョン依存)

関連データ構造

  • modssl_ctx_t: SSL 設定とコンテキストを保持する構造体。
  • X509: OpenSSL の証明書構造体。
  • X509_STORE: 証明書ストア。

失敗モードと境界条件

  • X509_STORE_CTX_init が失敗した場合、0 を返します。
  • 発行者がチェーン内にもストア内にも存在しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は OpenSSL のバージョン（0x10100000L を境界）により、参照カウントのインクリメント方法が異なります。
  • X509_STORE_CTX_get1_issuer は、成功時に発行者証明書の参照カウントをインクリメントして返すため、呼び出し側で適切に解放する必要があります。
