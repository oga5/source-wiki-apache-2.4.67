md_http_response_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme.h:60

目的

ACME プロトコル通信において、サーバーから返却された HTTP レスポンスを保持・参照するための構造体です。

シグネチャまたは形状

ソースコード上では不完全型（opaque pointer）として扱われることが多く、主に const struct md_http_response_t * の形式で参照されます。

入力と出力

  • **入力**: HTTP 通信層から受信したレスポンスデータ。
  • **出力**: ACME プロトコル処理におけるコールバック関数（md_acme_req_res_cb 等）の引数として渡され、レスポンス内容の解析やエラー判定に使用されます。

前提条件と不変条件

  • 構造体へのポインタは、関連付けられた HTTP リクエストのライフサイクル内でのみ有効です。
  • コールバック関数に渡される際は const 修飾されており、レスポンスの内容は読み取り専用として扱われます。

アルゴリズムまたは振る舞い

ACME サーバーからのレスポンス受信後、md_acme_req_t に登録されたコールバック関数を通じて、HTTP ステータスコードの確認、JSON ペイロードの抽出、および ACME 固有のエラー（Problem）判定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: md_acme.c および md_acme_drive.c 内の各コールバック関数（http_update_nonce, inspect_problem, on_response, update_directory, on_add_cert, csr_req, on_add_chain 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • md_acme_t: ACME セッション状態を保持する構造体。
  • md_acme_req_t: リクエストコンテキスト。
  • md_acme_req_res_cb: 本構造体を引数として受け取るコールバック型。

失敗モードと境界条件

  • HTTP 通信エラーやタイムアウトが発生した場合、レスポンスが空（NULL）または無効な状態でコールバックに渡される可能性があります。
  • サーバーが不正な JSON を返した場合、解析処理においてエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体の詳細なメンバ定義は md_http.h 側に存在します。本ヘッダーはインターフェース定義として前方宣言を利用しています。
  • メモリ管理は Apache Portable Runtime (APR) のプールに依存しているため、コールバック外でのポインタ保持には注意が必要です。
