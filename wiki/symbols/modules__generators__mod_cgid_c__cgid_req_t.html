cgid_req_t

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_cgid.c:195

目的

mod_cgid において、メインのサーバープロセスから CGI デーモンプロセスへリクエスト情報を転送するためのヘッダー構造体です。

シグネチャまたは形状

typedef struct {
    int req_type;
    unsigned long conn_id;
    pid_t ppid;
    int env_count;
    ap_unix_identity_t ugid;
    apr_size_t filename_len;
    apr_size_t argv0_len;
    apr_size_t uri_len;
    apr_size_t args_len;
    int loglevel;
#ifdef AP_CGID_USE_RLIMIT
    cgid_rlimit_t limits;
#endif
} cgid_req_t;

入力と出力

本構造体は、send_req() を介して Unix ドメインソケット経由でデーモンプロセスへ送信されます。デーモン側では get_req() を通じてこの情報を受け取り、CGI プロセスの生成や環境設定に使用します。

前提条件と不変条件

  • conn_id はデーモンプロセス側でハッシュ値として利用され、プロセス終了時のクリーンアップ対象を特定するために使用されます。
  • ppid は設定不備によるソケットの誤用を検知するためのサニティチェック用として保持されます。

アルゴリズムまたは振る舞い

1. cgid_handler 等の呼び出し元で構造体が初期化されます。 2. リクエストに関連するパス長や環境変数数、ユーザー識別情報（ugid）が格納されます。 3. ソケット経由でデーモンへ送信され、デーモン側で apr_proc_create を用いた CGI プロセス生成のパラメータとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cgid.c 内の cgid_handler およびリクエスト処理フロー（行 582, 801, 1425 付近）。
  • **呼び出し先**: send_req() および get_req() 関数が本構造体を引数として受け取ります（行 501 付近）。

関連データ構造

  • ap_unix_identity_t: 実行ユーザー/グループ情報の保持に使用。
  • cgid_rlimit_t: AP_CGID_USE_RLIMIT 定義時にリソース制限情報を保持。

失敗モードと境界条件

  • ppid の不一致は、設定誤りによるソケットの不正利用を示唆します。
  • filename_len 等の各長さフィールドは、ソケット通信時のデータ読み取りサイズを決定するため、不整合がある場合、デーモン側でのデシリアライズ失敗を招きます。

保守担当者向けメモ

  • 本構造体は mod_cgid の通信プロトコルの根幹であるため、フィールドの追加や変更を行う際は、デーモン側とサーバー側の両方で send_req/get_req のシリアライズロジックを同期させる必要があります。
