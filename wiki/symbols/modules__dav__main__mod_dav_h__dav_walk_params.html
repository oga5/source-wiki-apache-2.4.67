dav_walk_params

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1825

目的

WebDAV リソースツリーを走査（walk）する際のパラメータおよび実行コンテキストを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    int walk_type;
    dav_error * (*func)(dav_walk_resource *wres, int calltype);
    void *walk_ctx;
    apr_pool_t *pool;
    const dav_resource *root;
    dav_lockdb *lockdb;
} dav_walk_params;

入力と出力

  • **入力**: 走査の起点となる root リソース、走査対象を制御する walk_type フラグ、各ノードで実行されるコールバック関数 func、およびその実行コンテキスト walk_ctx を受け取ります。
  • **出力**: 走査処理の結果として dav_error 構造体（成功時は NULL）を返します。

前提条件と不変条件

  • pool は走査中に必要なメモリ割り当てに使用されます。
  • walk_type には以下のフラグを組み合わせて指定可能です：
  • DAV_WALKTYPE_AUTH: 認可されたファイルのみに限定。
  • DAV_WALKTYPE_NORMAL: 通常のファイルを走査。
  • DAV_WALKTYPE_LOCKNULL: LOCKNULL リソースを走査。
  • DAV_WALKTYPE_TOLERANT: 非致命的なエラーを許容。
  • lockdb は LOCKNULL リソースを走査する場合に必要となります。

アルゴリズムまたは振る舞い

指定された root からツリーを走査し、各リソースに対して func コールバックを呼び出します。走査の範囲や挙動は walk_type によって制御され、メモリ管理は pool に依存します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/repos.c 内の dav_fs_walk や dav_fs_internal_walk、および modules/dav/main/mod_dav.c で利用されています。
  • **呼び出し先**: 構造体内の func フィールドに設定された関数ポインタを介して、走査中の各リソースに対する処理が実行されます。

関連データ構造

  • dav_resource: 走査の起点となるリソース。
  • dav_lockdb: ロックデータベースへのアクセス用。
  • dav_walk_resource: コールバック関数 func に渡される走査対象リソース情報。
  • dav_error: 処理結果のステータス。

失敗モードと境界条件

  • func がエラーを返した場合、走査が中断される可能性があります（DAV_WALKTYPE_TOLERANT が設定されていない場合）。
  • lockdb が必要な走査（LOCKNULL 含む）において、lockdb が適切に初期化されていない場合は動作が未定義となります。

保守担当者向けメモ

  • 本構造体は mod_dav のリポジトリ走査機能の核となるため、フィールドの追加や変更は dav_fs_walk 等の各プロバイダー実装に影響を与えます。
  • walk_ctx は不透明なポインタ（void *）であるため、コールバック関数側での適切な型キャストが必要です。
