dav_lockdb_private

定義

  • 種別: struct
  • 定義箇所: modules/dav/lock/locks.c:175

目的

WebDAV ロックデータベースの操作に必要なコンテキストと、DBM ハンドルを保持するための内部構造体です。

シグネチャまたは形状

struct dav_lockdb_private
{
    request_rec *r;           /* UUID 状態アクセス用 */
    apr_pool_t *pool;         /* メモリ割り当て用プール */
    const char *lockdb_path;  /* ロックデータベースのパス */
    int opened;               /* データベースが開かれているかのフラグ */
    apr_dbm_t *db;            /* DBM ハンドル（未オープン時は NULL） */
};

入力と出力

本構造体は、mod_dav のロックプロバイダーがロックデータベースへアクセスする際のセッション状態として機能します。

  • **入力**: request_rec および apr_pool_t を初期化時に受け取り、ロックデータベースのパスを保持します。
  • **出力**: DBM ハンドル（apr_dbm_t）を保持し、後続のロック読み書き操作に提供します。

前提条件と不変条件

  • db フィールドが NULL でない場合、データベースは有効な状態であるとみなされます。
  • opened フラグは、データベースの遅延オープン状態を管理するために使用されます。
  • 構造体内のメモリは、request_rec に関連付けられたプールから割り当てられることが前提です。

アルゴリズムまたは振る舞い

  • **遅延オープン**: dav_generic_open_lockdb によって初期化されますが、DBM ハンドルは即座には開かれません。実際の I/O が発生するタイミングで dav_generic_really_open_lockdb を通じて db フィールドが設定されます。
  • **リソース管理**: リクエスト終了時にプールが破棄されることで、関連するリソースも自動的に解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/lock.c:192
  • modules/dav/lock/locks.c:188
  • modules/dav/main/mod_dav.h:1343（dav_lockdb 構造体内の info メンバとして参照）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。本構造体を利用する関数（dav_generic_load_lock_record 等）が apr_dbm API を呼び出します。

関連データ構造

  • apr_dbm_t: DBM データベースへのアクセスに使用されるハンドル。
  • request_rec: Apache HTTP Server のリクエストコンテキスト。
  • dav_lockdb: 本構造体を内部情報として保持する上位のロックデータベース構造体。

失敗モードと境界条件

  • db が NULL の状態でデータベース操作を試みた場合、dav_generic_really_open_lockdb が呼び出されない限り、操作は失敗します。
  • 読み取り専用で開かれた場合、期限切れロックの自動削除（クリーンアップ）処理はスキップされます。

保守担当者向けメモ

  • 本構造体は mod_dav のロックプロバイダー実装と密結合しています。フィールドの追加や変更を行う際は、locks.c 内のシリアライズロジック（dav_generic_save_lock_record 等）との整合性に注意してください。
  • opened フラグと db ハンドルの状態管理は、ロックデータベースの遅延初期化において重要です。
