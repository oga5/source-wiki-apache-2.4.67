create_dir_expires_config

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:185

目的

ディレクトリごとの mod_expires 設定構造体（expires_dir_config）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_dir_expires_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定の割り当てに使用するメモリプール。
  • char *dummy: 使用されない引数。
  • **出力**:
  • void *: 初期化された expires_dir_config 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体の全メンバはゼロ初期化される。
  • expiresbytype テーブルは、4つのエントリを初期容量として作成される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して expires_dir_config サイズのメモリを確保する。 2. active メンバを ACTIVE_DONTCARE に設定する。 3. wildcards を 0 に初期化する。 4. expiresdefault を NULL に設定する。 5. apr_table_make を呼び出し、MIME タイプごとのマッピング用テーブルを作成する。 6. 構築した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定読み込みプロセス（modules/metadata/mod_expires.c:565 にて、ディレクトリ設定作成ハンドラとして登録されている）。
  • **呼び出し先**: apr_pcalloc, apr_table_make（いずれも APR API）。

関連データ構造

  • expires_dir_config: 本関数が初期化する対象の構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がプロセスを終了させる可能性がある（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。
  • expiresbytype テーブルの初期サイズ（4）は、一般的な設定数に基づいているが、必要に応じて調整可能である。
