ap_xlate_proto_from_ascii

定義

  • 種別: function
  • 定義箇所: server/util_ebcdic.c:82
  • マクロ定義: util_ebcdic.h:81

目的

ASCII形式のデータを、EBCDIC環境で動作するApache HTTP Serverのローカル文字セットへ変換します。

シグネチャまたは形状

void ap_xlate_proto_from_ascii(char *buffer, apr_size_t len)

入力と出力

  • **入力**:
  • buffer: 変換対象のASCIIデータが格納されたバッファ。
  • len: 変換するデータのバイト数。
  • **出力**:
  • なし（buffer を直接書き換えるインプレース変換）。

前提条件と不変条件

  • 本関数は APR_CHARSET_EBCDIC が定義されている環境でのみ有効です。
  • ap_hdrs_from_ascii ハンドルが ap_init_ebcdic によって適切に初期化されている必要があります。
  • 非EBCDIC環境では、ヘッダーファイルにより空のマクロに置換されます。

アルゴリズムまたは振る舞い

1. inbytes_left および outbytes_left を len で初期化します。 2. apr_xlate_conv_buffer を呼び出し、グローバル変数 ap_hdrs_from_ascii を使用して、buffer 内のデータをASCIIからローカル文字セットへ変換します。 3. 変換結果は元の buffer に上書きされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/debugging/mod_dumpio.c
  • modules/http/http_filters.c
  • modules/metadata/mod_ident.c
  • modules/proxy/ajp_header.c
  • modules/proxy/proxy_util.c
  • modules/ssl/ssl_engine_io.c
  • modules/ssl/ssl_util_ssl.c
  • server/protocol.c
  • **呼び出し先**:
  • apr_xlate_conv_buffer (APRライブラリ)

関連データ構造

  • ap_hdrs_from_ascii: 変換に使用される apr_xlate_t 型のグローバルハンドル。

失敗モードと境界条件

  • apr_xlate_conv_buffer の内部エラーについては本関数内で明示的なエラーハンドリングを行っていません。
  • buffer がNULLの場合や、len が実際のバッファサイズを超えている場合の動作は、apr_xlate_conv_buffer の実装に依存します。

保守担当者向けメモ

  • 本関数はインプレース変換を行うため、呼び出し側は buffer が書き換え可能であることを保証する必要があります。
  • 非EBCDIC環境では呼び出しが無視されるため、条件分岐なしで呼び出し可能です。
