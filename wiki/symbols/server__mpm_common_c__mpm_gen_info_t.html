mpm_gen_info_t

定義

  • 種別: struct
  • 定義箇所: server/mpm_common.c:138

目的

マルチプロセッシングモジュール（MPM）において、プロセス世代（Generation）ごとの状態を追跡・管理するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct mpm_gen_info_t {
    APR_RING_ENTRY(mpm_gen_info_t) link;
    int gen;          /* 世代番号 */
    int active;       /* アクティブなプロセス数 */
    int done;         /* 世代終了フラグ */
} mpm_gen_info_t;

入力と出力

本構造体は関数ではなくデータ型です。server/mpm_common.c 内のプロセス管理ロジックにおいて、世代情報の保持およびリスト操作の単位として利用されます。

前提条件と不変条件

  • APR_RING 構造体の一部として管理されます。
  • active は現在稼働中のプロセス数を表し、done は世代が終了処理に入ったかどうかを示します。
  • サーバーの生存期間中、s->process->pool に割り当てられ保持されます。

アルゴリズムまたは振る舞い

  • ap_core_child_status() 等のプロセス監視ロジックにおいて、特定の世代のプロセスが終了した際に active カウンタを更新します。
  • active が 0 になった際、end_gen() 関数が呼び出され、世代終了フックの発火とリストからの除外が行われます。
  • APR_RING を用いて、アクティブな世代と未使用の世代をそれぞれリストとして管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm_common.c 内のプロセス管理関数群（ap_core_child_status 等）が、世代情報の参照や更新のために本構造体を利用します。
  • **呼び出し先**: APR_RING マクロ群（APR_RING_INSERT_HEAD, APR_RING_INIT, APR_RING_SENTINEL など）が、リスト操作のために本構造体の link メンバを操作します。

関連データ構造

  • APR_RING: 本構造体をリストとして連結するために使用される APR のリングデータ構造。
  • mpm_gen_info_head_t: APR_RING_HEAD によって定義される、本構造体のリストの先頭を保持する型。

失敗モードと境界条件

  • 世代管理リストが空の状態で APR_RING_SENTINEL を参照した場合の挙動は、APR のリング実装に依存します。
  • プロセス数が正しく追跡されない場合、done フラグが適切にセットされず、世代終了フックが発火しない可能性があります。

保守担当者向けメモ

  • 本構造体は server/mpm_common.c 内のプロセスライフサイクル管理の核となります。
  • gen_head_init 変数による初期化判定など、古い設計の名残が含まれているため、変更を加える際は既存のリスト操作ロジックとの整合性に注意してください。
