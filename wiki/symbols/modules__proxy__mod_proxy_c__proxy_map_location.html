proxy_map_location

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:1135

目的

Apache HTTP Server の map_to_storage フックとして動作し、リクエストがプロキシ対象である場合に、コアモジュールによる標準的なファイルシステムマッピングをバイパスしてプロキシ固有の設定適用（proxy_walk）を行う。

シグネチャまたは形状

static int proxy_map_location(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: int （Apache のステータスコード: OK, DECLINED, または ap_die によるエラー終了）

前提条件と不変条件

  • リクエストが r->proxyreq を持ち、かつ r->filename が "proxy:" で始まる文字列である必要がある。
  • 上記条件を満たさない場合、本関数は何も行わず DECLINED を返す。

アルゴリズムまたは振る舞い

1. リクエストがプロキシリクエストか、および r->filename が "proxy:" で始まるかを判定する。条件を満たさない場合は DECLINED を返す。 2. proxy_walk(r) を呼び出し、<Proxy> セクションの設定をリクエストに適用する。 3. proxy_walk がエラーを返した場合、ap_die を実行してエラーレスポンスを生成し、そのステータスを返す。 4. 正常終了時は OK を返す。これにより、コアモジュールや mod_http による後続の map_to_storage フック処理を抑制する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の map_to_storage フック（modules/proxy/mod_proxy.c:3369 にて ap_hook_map_to_storage に登録）。
  • **呼び出し先**: proxy_walk（設定適用）、ap_die（エラー処理）。

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • proxy_dir_conf: proxy_walk を通じて適用されるディレクトリ単位のプロキシ設定。

失敗モードと境界条件

  • proxy_walk が失敗した場合、ap_die を介してエラーがクライアントに返される。
  • r->filename が NULL の場合、または "proxy:" で始まらない場合は、本関数は処理をスキップする。

保守担当者向けメモ

  • 本関数は APR_HOOK_FIRST として登録されており、他のモジュールよりも優先的にプロキシ処理を確定させる意図がある。
  • TRACE メソッドなどのリクエスト処理において、コアのファイルマッピングロジックと競合しないよう設計されている。
