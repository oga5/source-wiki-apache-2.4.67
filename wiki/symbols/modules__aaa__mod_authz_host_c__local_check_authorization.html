local_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:293

目的

リクエストがサーバー自身（ローカル接続）からのものかどうかを判定し、アクセスを認可または拒否する。

シグネチャまたは形状

static authz_status local_check_authorization(request_rec *r,
                                              const char *require_line,
                                              const void *parsed_require_line)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報を保持する request_rec 構造体。
  • require_line: 設定ファイル内の Require ディレクティブの引数文字列（本関数では未使用）。
  • parsed_require_line: 解析済みの設定データ（本関数では未使用）。
  • **出力**:
  • AUTHZ_GRANTED: リクエストがローカル接続であると判定された場合。
  • AUTHZ_DENIED: それ以外の場合。

前提条件と不変条件

  • r->connection->local_addr が有効なソケットアドレスであること。
  • r->useragent_addr がクライアントの IP アドレスとして正しく設定されていること。
  • グローバル変数 localhost_v4（および APR_HAVE_IPV6 定義時は localhost_v6）が、ローカルホストのサブネット情報として初期化済みであること。

アルゴリズムまたは振る舞い

1. リクエストの接続先アドレス (r->connection->local_addr) とクライアントアドレス (r->useragent_addr) を apr_sockaddr_equal で比較し、一致すれば AUTHZ_GRANTED を返す。 2. クライアントアドレスが localhost_v4 サブネットに含まれるか apr_ipsubnet_test で確認し、一致すれば AUTHZ_GRANTED を返す。 3. IPv6 が有効な場合、同様に localhost_v6 サブネットに含まれるか確認し、一致すれば AUTHZ_GRANTED を返す。 4. 上記のいずれにも該当しない場合、AUTHZ_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_host.c:350 にて authz_local_provider 構造体の一部として参照されている。
  • **呼び出し先**:
  • apr_sockaddr_equal (APR)
  • apr_ipsubnet_test (APR)

関連データ構造

  • request_rec: リクエストのコンテキスト情報。
  • authz_status: 認可結果を示す列挙型。
  • localhost_v4, localhost_v6: ローカルホストを定義する apr_ipsubnet_t 型の静的変数。

失敗モードと境界条件

  • クライアントアドレスが取得できない場合や、localhost サブネットの初期化が不完全な場合は、意図せず AUTHZ_DENIED が返される可能性がある。

保守担当者向けメモ

  • 本関数は mod_authz_core の認可プロバイダーインターフェースに準拠している。
  • require_line および parsed_require_line は引数として受け取るが、ロジック上使用していないため、将来的な拡張がない限り無視される。
