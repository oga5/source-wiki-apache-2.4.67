mod_auth_ldap_groupattr_entry_t

定義

  • 種別: struct
  • 定義箇所: include/util_ldap.h:186

目的

LDAP 認可処理において、グループ属性（Group Attribute）の名前を保持するためのデータ構造です。

シグネチャまたは形状

struct mod_auth_ldap_groupattr_entry_t {
    char *name;
};

入力と出力

  • **入力**: name メンバにグループ属性名（文字列）を格納します。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • name は有効な文字列へのポインタである必要があります。
  • 複数のエントリが apr_array_header_t 等の配列構造に格納されて利用されることが想定されています。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持に特化しており、特定のアルゴリズムは実装していません。主に mod_authnz_ldap や util_ldap において、設定されたグループ属性のリストを走査・参照するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/aaa/mod_authnz_ldap.c: 設定値の管理およびグループ属性の検証時に参照。
  • modules/ldap/util_ldap.c: サブグループクラスの処理において、配列要素としてキャスト・参照。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_array_header_t: 本構造体のインスタンスを複数保持するコンテナとして利用されます。

失敗モードと境界条件

  • name が NULL の場合、参照時にセグメンテーションフォールトが発生する可能性があります。
  • メモリ割り当て失敗時、本構造体を含む配列の初期化や拡張が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_authnz_ldap と util_ldap の間で共有されるインターフェースの一部です。変更を加える際は、両モジュールへの影響を確認してください。
  • modules/aaa/mod_authnz_ldap.c および modules/ldap/util_ldap.c 内で elts メンバから本構造体へキャストして利用されているため、メンバ構成を変更する場合はキャスト箇所の整合性に注意してください。
