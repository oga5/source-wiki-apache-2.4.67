cache_status

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:1776

目的

HTTP キャッシュ処理のステータスを追跡し、リクエストの環境変数およびレスポンスヘッダー（X-Cache, X-Cache-Detail）にキャッシュ状態を反映させる。

シグネチャまたは形状

int cache_status(cache_handle_t *h, request_rec *r, apr_table_t *headers, ap_cache_status_e status, const char *reason);

入力と出力

  • **入力**:
  • h: キャッシュハンドル（未使用）。
  • r: 現在のリクエスト構造体。
  • headers: レスポンスヘッダーテーブル。
  • status: キャッシュ状態を示す列挙型 (AP_CACHE_HIT, AP_CACHE_REVALIDATE, AP_CACHE_MISS, AP_CACHE_INVALIDATE)。
  • reason: ステータスの理由を示す文字列。
  • **出力**:
  • 常に OK を返す。
  • 副作用として r->subprocess_env に環境変数を設定し、headers にキャッシュ関連ヘッダーを追加する。

前提条件と不変条件

  • r->server->module_config から cache_server_conf が取得可能であること。
  • r->per_dir_config から cache_dir_conf が取得可能であること。
  • r->pool を使用してメモリ割り当てが行われるため、リクエストのライフサイクルに依存する。

アルゴリズムまたは振る舞い

1. status に応じて、対応する環境変数（AP_CACHE_HIT_ENV 等）に reason を格納する。 2. AP_CACHE_STATUS_ENV に reason を格納する。 3. 設定（dconf または conf）に基づき、X-Cache ヘッダーの付与が必要か判定する。必要であれば、ステータスとホスト名を含む文字列を生成して追加する。 4. 同様に、設定に基づき X-Cache-Detail ヘッダーの付与が必要か判定する。必要であれば、reason とホスト名を含む文字列を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cache の外部フック cache_status として登録されており、キャッシュ処理の各フェーズから呼び出される（直接的な呼び出し箇所はソース抜粋外）。
  • **呼び出し先**: ap_get_module_config, apr_table_setn, apr_psprintf などの Apache/APR API を呼び出す。

関連データ構造

  • cache_server_conf: サーバー全体の設定。
  • cache_dir_conf: ディレクトリ単位の設定。
  • request_rec: リクエスト情報および環境変数の保持に使用。

失敗モードと境界条件

  • dconf が NULL の場合、conf の設定値がフォールバックとして使用される。
  • headers が NULL の場合、ヘッダー追加処理でクラッシュする可能性がある（呼び出し元が有効なテーブルを渡すことが前提）。

保守担当者向けメモ

  • 本関数は cache_status フックの実装である。
  • X-Cache および X-Cache-Detail の出力制御は、設定ファイルでの有効化状態に依存する。
  • ステータス列挙型 ap_cache_status_e に新しい状態を追加した場合は、switch 文の更新が必要。
