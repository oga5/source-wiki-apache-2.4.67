set_cookie_form_size

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:253

目的

Apache 設定ファイル内の AuthFormSize ディレクティブを解析し、フォームデータの最大サイズ設定を auth_form_config_rec 構造体に格納する。

シグネチャまたは形状

static const char *set_cookie_form_size(cmd_parms *cmd, void *config, const char *arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • config: 現在のディレクトリ設定構造体 (auth_form_config_rec)。
  • arg: 設定値として渡された文字列（バイト数）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • config は auth_form_config_rec 型としてキャスト可能であること。
  • arg は数値として解釈可能であること。

アルゴリズムまたは振る舞い

1. apr_strtoff を使用して、入力文字列 arg を apr_off_t 型の数値に変換する。 2. 変換結果が負の値、または APR_SIZE_MAX を超える場合はエラーメッセージを返す。 3. 変換が成功した場合、conf->form_size に値を代入し、conf->form_size_set フラグを 1 に設定する。 4. 正常終了として NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c:370 において、AP_INIT_TAKE1 マクロを通じて AuthFormSize ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: apr_strtoff (APR ライブラリ関数)。

関連データ構造

  • auth_form_config_rec: ディレクトリごとの認証設定を保持する構造体。form_size および form_size_set フィールドを更新する。

失敗モードと境界条件

  • apr_strtoff が APR_SUCCESS を返さない場合（数値以外の入力など）。
  • 入力値が負数である場合。
  • 入力値が APR_SIZE_MAX を超過する場合。
  • 上記のいずれかの場合、設定エラーとしてメッセージが返される。

保守担当者向けメモ

  • 本関数は AuthFormSize ディレクティブ専用のパーサーである。設定値のバリデーションは apr_size_t の範囲内で行われる。
