set_metadir

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_cern_meta.c:169

目的

Apache 設定ファイル内の MetaDir ディレクティブを処理し、メタファイルが格納されるディレクトリ名をモジュールのディレクトリ設定構造体に保存します。

シグネチャまたは形状

static const char *set_metadir(cmd_parms *parms, void *in_dconf, const char *arg)

入力と出力

  • **入力**:
  • parms: コマンドパラメータ構造体。
  • in_dconf: 現在のディレクトリ設定構造体 (cern_meta_dir_config) へのポインタ。
  • arg: 設定ファイルから読み込まれたディレクトリ名の文字列。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • in_dconf は cern_meta_dir_config 型としてキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. in_dconf を cern_meta_dir_config 構造体としてキャストします。 2. 引数 arg（ディレクトリ名）を構造体の metadir メンバに代入します。 3. NULL を返し、設定処理が正常に完了したことを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_cern_meta.c:198 において、AP_INIT_TAKE1 マクロを介して MetaDir ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • cern_meta_dir_config: ディレクトリごとの設定を保持する構造体。本関数はこの構造体の metadir フィールドを更新します。

失敗モードと境界条件

  • 引数 arg が NULL の場合や、メモリ割り当てに関する境界条件については、Apache の設定解析フレームワークに依存します。本関数自体は単純な代入を行うため、特有の失敗モードは定義されていません。

保守担当者向けメモ

  • 本関数は MetaDir ディレクティブ専用のセッターです。設定の継承やマージについては、merge_cern_meta_dir_configs が別途担当します。
