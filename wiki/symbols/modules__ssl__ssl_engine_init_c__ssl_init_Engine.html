ssl_init_Engine

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_init.c:595

目的

OpenSSL の暗号化エンジン（Crypto Device）を初期化し、設定されたデバイスをロードしてシステム全体で利用可能にする。

シグネチャまたは形状

apr_status_t ssl_init_Engine(server_rec *s, apr_pool_t *p)

入力と出力

  • **入力**:
  • s: サーバーレコード（設定情報 SSLModConfigRec の取得に使用）。
  • p: メモリプール。
  • **出力**:
  • APR_SUCCESS: 正常終了、またはエンジン設定が不要な場合。
  • ssl_die(s) の戻り値: エンジンのロードまたは有効化に失敗した場合（プロセス終了）。

前提条件と不変条件

  • MODSSL_HAVE_ENGINE_API が定義されている環境でのみ有効。
  • mc->szCryptoDevice にデバイス名が設定されている場合、そのデバイスが OpenSSL によってロード可能であること。

アルゴリズムまたは振る舞い

1. mc->szCryptoDevice が設定されているか確認する。 2. ENGINE_by_id を使用して指定されたエンジンをロードする。失敗時はエラーログを出力し ssl_die を呼び出す。 3. 特定のエンジン（"chil"）の場合、ENGINE_CTRL_CHIL_SET_FORKCHECK を設定する。 4. ENGINE_set_default を呼び出し、すべてのメソッドに対して当該エンジンをデフォルトとして有効化する。失敗時はエラーログを出力し ssl_die を呼び出す。 5. 成功時はログを出力し、ENGINE_free で参照カウントを減らして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:435（初期化プロセス内）。
  • **呼び出し先**:
  • OpenSSL API: ENGINE_by_id, ENGINE_ctrl, ENGINE_set_default, ENGINE_free。
  • Apache API: ap_log_error, ssl_log_ssl_error, ssl_die。

関連データ構造

  • SSLModConfigRec: モジュール設定を保持する構造体。szCryptoDevice メンバにエンジン名が格納される。

失敗モードと境界条件

  • **エンジンロード失敗**: 指定された ID のエンジンが見つからない場合、APLOG_EMERG レベルでログ出力し、ssl_die によりサーバー起動を停止する。
  • **有効化失敗**: ENGINE_set_default が失敗した場合も同様に ssl_die を呼び出し、サーバー起動を停止する。

保守担当者向けメモ

  • 本関数は OpenSSL のエンジン API に強く依存している。
  • MODSSL_HAVE_ENGINE_API マクロにより、ビルド環境によって挙動が完全に無効化される可能性がある。
  • ENGINE_free は ENGINE_set_default 後の参照カウント管理のために呼び出されている。
