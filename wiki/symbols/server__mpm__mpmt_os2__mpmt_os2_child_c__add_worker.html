add_worker

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2_child.c:293

目的

新規ワーカー（スレッド）を生成し、スコアボードを初期化して実行を開始する。

シグネチャまたは形状

void add_worker(void)

入力と出力

  • 入力: なし（グローバルな ap_scoreboard_image および child_slot を参照）
  • 出力: なし

前提条件と不変条件

  • HARD_THREAD_LIMIT までのスレッドスロットが存在すること。
  • ap_scoreboard_image が初期化済みであること。
  • ap_thread_stacksize が設定されているか、デフォルト値（128KB）が使用可能であること。

アルゴリズムまたは振る舞い

1. ap_thread_stacksize に基づき、スレッドのスタックサイズを決定する。 2. HARD_THREAD_LIMIT までスコアボードの servers[child_slot] を走査する。 3. SERVER_DEAD 状態のスロットを発見した場合：

  • ステータスを SERVER_STARTING に更新する。
  • _beginthread を呼び出し、worker_main をエントリポイントとして新規スレッドを起動する。
  • 引数として thread_slot を渡す。

    • 4. スレッド起動後、ループを終了する。

呼び出し元と呼び出し先

  • 呼び出し元: server_maintenance (server/mpm/mpmt_os2/mpmt_os2_child.c:444)
  • 呼び出し先: _beginthread (OS/2 ランタイム関数), worker_main

関連データ構造

  • ap_scoreboard_image: プロセスおよびスレッドの状態を保持する共有メモリ上の構造体。
  • HARD_THREAD_LIMIT: スレッド数の上限定数。

失敗モードと境界条件

  • HARD_THREAD_LIMIT を超えるスレッドは生成されない（ループ終了により何もしない）。
  • _beginthread が失敗した場合の明示的なエラーハンドリングはソースに含まれていない。

保守担当者向けメモ

  • 本関数は OS/2 の _beginthread に依存している。
  • child_slot はこの関数が実行されるコンテキスト（子プロセス）において有効である必要がある。
