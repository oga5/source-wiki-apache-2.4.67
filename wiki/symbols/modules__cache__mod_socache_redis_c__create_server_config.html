create_server_config

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_redis.c:402

目的

Redis キャッシュモジュール用のサーバー単位設定構造体（socache_rd_svr_cfg）をメモリ上に確保し、デフォルト値で初期化する。

シグネチャまたは形状

static void* create_server_config(apr_pool_t* p, server_rec* s)

入力と出力

  • **入力**:
  • apr_pool_t* p: 設定構造体を割り当てるためのメモリプール。
  • server_rec* s: 設定対象となるサーバーレコード（本関数内では未使用）。
  • **出力**:
  • void*: 初期化済みの socache_rd_svr_cfg 構造体へのポインタ。

前提条件と不変条件

  • apr_palloc が成功すること。
  • socache_rd_svr_cfg 構造体が定義されていること。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、socache_rd_svr_cfg サイズのメモリをプール p から割り当てる。 2. 割り当てた構造体の ttl メンバに RD_DEFAULT_SERVER_TTL を代入する。 3. 割り当てた構造体の rwto メンバに RD_DEFAULT_SERVER_RWTO を代入する。 4. 初期化した構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の設定読み込みプロセス（server/config.c）から、モジュール構造体を通じて間接的に呼び出される。
  • **呼び出し先**: apr_palloc（APR ライブラリ）。

関連データ構造

  • socache_rd_svr_cfg: サーバーごとの接続設定（TTL, 読み書きタイムアウト）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc がアボートする可能性がある（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は module 構造体の create_server_config フックとして登録されることを前提としている。
  • server_rec* s 引数は将来的な拡張のために用意されているが、現在の実装では使用されていない。
