ap_sb_handle_t

定義

  • 種別: struct
  • 定義箇所: server/scoreboard.c:107
  • 関連ヘッダ: include/scoreboard.h

目的

Apache HTTP Server のスコアボードにおいて、特定のワーカー（プロセスおよびスレッド）を識別するためのハンドルとして機能します。

シグネチャまたは形状

struct ap_sb_handle_t {
    int child_num;
    int thread_num;
};

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではありません。
  • ap_create_sb_handle を通じて生成され、スコアボード関連の各関数（ap_update_child_status 等）の引数として渡されます。

前提条件と不変条件

  • child_num および thread_num は、スコアボード内の有効なインデックスを指している必要があります。
  • 本ハンドルが指すスコアボード上のスロットは、単一の所有者（ワーカー）によってのみ書き込まれることが想定されており、排他制御なしでの更新が可能です。

アルゴリズムまたは振る舞い

  • スコアボード内の特定の worker_score 構造体を特定するためのオフセット情報を提供します。
  • ap_get_scoreboard_worker などのアクセサ関数は、このハンドルを使用して共有メモリ上の該当スロットを特定し、統計情報や状態の読み書きを行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/echo/mod_echo.c
  • include/scoreboard.h 内の各種 API（ap_increment_counts, ap_update_child_status 等）の利用側。
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、直接的な呼び出し先はありません。ただし、本構造体を引数として受け取る関数群（ap_update_child_status, ap_get_scoreboard_worker 等）が、このハンドルを介してスコアボードの操作を行います。

関連データ構造

  • worker_score: スコアボード内のワーカーごとの詳細な状態を保持する構造体。
  • process_score: プロセスごとの状態を保持する構造体。

失敗モードと境界条件

  • 無効な child_num や thread_num を保持したハンドルを使用した場合、スコアボードのメモリ領域外アクセスや、誤った統計情報の更新を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は server/scoreboard.c と include/scoreboard.h の両方で定義されています。変更時は両方の整合性を確認してください。
  • メモリレイアウトの設計上、このハンドルが指すインデックスは初期化後に変更されない前提で運用されています。
