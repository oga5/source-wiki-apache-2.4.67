set_expiresdefault

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:335

目的

Apache の ExpiresDefault ディレクティブを処理し、ディレクトリ設定構造体にデフォルトの有効期限情報を格納する。

シグネチャまたは形状

static const char *set_expiresdefault(cmd_parms *cmd, void *in_dir_config, const char *code)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • in_dir_config: 現在のディレクトリ設定構造体 (expires_dir_config)。
  • code: 設定ファイルから読み込まれた有効期限文字列（例: "access plus 1 month"）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • in_dir_config は expires_dir_config 型としてキャスト可能であること。
  • check_code 関数が有効期限文字列を正しく解析できること。

アルゴリズムまたは振る舞い

1. check_code を呼び出し、入力された code を内部形式（[AM]<seconds>）へ変換する。 2. 変換が成功した場合、結果の real_code を dir_config->expiresdefault に代入する。 3. 変換が失敗した場合、check_code が返したエラーメッセージを apr_pstrcat で整形して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: AP_INIT_TAKE1 マクロを介して、Apache の設定読み込み時に呼び出される（modules/metadata/mod_expires.c:355）。
  • **呼び出し先**: check_code（有効期限文字列の解析）、apr_pstrcat（エラーメッセージの生成）。

関連データ構造

  • expires_dir_config: ディレクトリごとの設定を保持する構造体。本関数はこの中の expiresdefault メンバを更新する。

失敗モードと境界条件

  • check_code が解析に失敗した場合、エラーメッセージを返し、設定の更新は行われない。
  • code が不正なフォーマットである場合、check_code の戻り値に基づいたエラーメッセージが生成される。

保守担当者向けメモ

  • check_code 関数が atoi() を使用しているため、非常に大きな数値や不正なフォーマットに対する堅牢性に注意が必要である。
  • 本関数は ExpiresDefault ディレクティブ専用のハンドラである。
