session_dbd_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/session/mod_session_dbd.c:33

目的

Apache HTTP Server の mod_session_dbd モジュールにおいて、ディレクトリ単位でのセッションデータベース設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const char *name;           /* クッキー名 */
    int name_set;               /* クッキー名設定フラグ */
    const char *name_attrs;     /* クッキー属性 */
    const char *name2;          /* 予備のクッキー名 */
    int name2_set;              /* 予備クッキー名設定フラグ */
    const char *name2_attrs;    /* 予備クッキー属性 */
    int peruser;                /* ユーザー紐付けフラグ */
    int peruser_set;            /* ユーザー紐付け設定フラグ */
    int remove;                 /* セッション削除フラグ */
    int remove_set;             /* 削除設定フラグ */
    const char *selectlabel;    /* SELECT用SQLクエリラベル */
    const char *insertlabel;    /* INSERT用SQLクエリラベル */
    const char *updatelabel;    /* UPDATE用SQLクエリラベル */
    const char *deletelabel;    /* DELETE用SQLクエリラベル */
} session_dbd_dir_conf;

入力と出力

  • 本構造体は ap_get_module_config(r->per_dir_config, ...) を通じてリクエスト処理中に取得されます。
  • 設定値は SessionDBD* 系ディレクティブによって初期化・マージされます。

前提条件と不変条件

  • mod_dbd がロードされ、データベース接続が確立されている必要があります。
  • 各 *label フィールドは、mod_dbd で定義された SQL クエリのラベルと一致している必要があります。

アルゴリズムまたは振る舞い

  • ディレクトリ単位の設定を保持し、セッションの読み込み（selectlabel）、保存（insertlabel/updatelabel）、削除（deletelabel）時に使用する SQL クエリを特定します。
  • peruser フラグが有効な場合、セッションはユーザー名に基づいて管理されます。
  • 設定の継承およびマージ処理において、apr_pcalloc でメモリ確保されたインスタンスが使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • session_dbd_load, session_dbd_save 等のセッション処理関数（r->per_dir_config から取得）。
  • 設定マージ関数 merge_session_dbd_dir_config。
  • 設定生成関数 create_session_dbd_dir_config。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • session_rec: セッションデータ本体を保持する構造体。
  • request_rec: リクエストごとの設定コンテキストを保持。

失敗モードと境界条件

  • 設定が未定義の場合、各ポインタは NULL となり、後続のデータベース操作でエラーが発生する可能性があります。
  • mod_dbd が利用できない環境では、本構造体を用いたクエリ実行は失敗します。

保守担当者向けメモ

  • 本構造体は mod_session_dbd.c 内の各フック関数で頻繁に参照されます。設定項目を追加する場合は、merge_session_dbd_dir_config での適切なマージロジックの実装を忘れないでください。
