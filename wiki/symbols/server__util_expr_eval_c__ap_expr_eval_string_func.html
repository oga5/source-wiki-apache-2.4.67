ap_expr_eval_string_func

定義

  • 種別: function
  • 定義箇所: server/util_expr_eval.c:208

目的

式評価エンジンにおいて、登録された文字列関数（string function）を呼び出し、その実行結果を文字列として取得します。

シグネチャまたは形状

static const char *ap_expr_eval_string_func(ap_expr_eval_ctx_t *ctx,
                                            const ap_expr_t *info,
                                            const ap_expr_t *arg)

入力と出力

  • **入力**:
  • ctx: 評価コンテキスト（ap_expr_eval_ctx_t）。
  • info: 関数情報を含む AST ノード。node_arg1 に関数ポインタ、node_arg2 に関数固有のデータが格納されている。
  • arg: 関数に渡す引数となる AST ノード。
  • **出力**:
  • 評価結果の文字列（const char *）。

前提条件と不変条件

  • info->node_op は op_StringFuncInfo であること。
  • info->node_arg1（関数ポインタ）および info->node_arg2（データ）は NULL であってはならない。

アルゴリズムまたは振る舞い

1. info ノードから関数ポインタ（func）とデータ（data）を取り出します。 2. ap_expr_eval_word(ctx, arg) を呼び出し、引数ノードを評価して文字列値を取得します。 3. 取得した引数値、コンテキスト、および data を引数として func を実行し、その戻り値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_expr_eval.c 内の式評価ロジック（ap_expr_eval_word 等からの呼び出しが示唆されています）。
  • **呼び出し先**: ap_expr_eval_word（引数の評価のため）。

関連データ構造

  • ap_expr_eval_ctx_t: 評価コンテキスト。
  • ap_expr_t: 抽象構文木（AST）のノード。
  • ap_expr_string_func_t: 文字列関数を定義する関数ポインタ型。

失敗モードと境界条件

  • AP_DEBUG_ASSERT により、不正なノードタイプや NULL ポインタが渡された場合にアサーションエラーが発生します。
  • 引数の評価（ap_expr_eval_word）が失敗した場合の挙動は、その関数の実装に依存します。

保守担当者向けメモ

  • 本関数は op_StringFuncInfo 型のノード専用のディスパッチャです。
  • 評価中に生成される文字列は、ctx->p（リクエストプール）から割り当てられることが前提となっています。
