dav_rollback_item

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/props.c:220

目的

PROPPATCH 操作におけるトランザクションのロールバック情報を保持するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct dav_rollback_item {
    dav_deadprop_rollback *deadprop;
    dav_liveprop_rollback *liveprop;
} dav_rollback_item;

入力と出力

  • **入力**: dav_prop_ctx.is_liveprop の値に基づき、いずれか一方のロールバックコンテキストが設定されます。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • deadprop と liveprop のどちらか一方が、現在のプロパティ種別に応じて使用されます。
  • 呼び出し元で is_liveprop フラグによる適切な選択が行われている必要があります。

アルゴリズムまたは振る舞い

  • PROPPATCH の実行フェーズにおいて、操作が失敗した際に元の状態へ復旧するために必要な情報を格納します。
  • ライブプロパティの場合は liveprop メンバを、デッドプロパティの場合は deadprop メンバを介して復旧処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.h:1781 にて、ロールバック情報としてこの構造体へのポインタが定義されています。
  • **呼び出し先**: 具体的なロールバック処理を行う関数（dav_prop_rollback 等）が、この構造体内のポインタを参照します。

関連データ構造

  • dav_deadprop_rollback: デッドプロパティのロールバック用コンテキスト。
  • dav_liveprop_rollback: ライブプロパティのロールバック用コンテキスト。
  • dav_prop_ctx: プロパティ操作のコンテキスト。

失敗モードと境界条件

  • 構造体内のポインタが NULL の場合、ロールバック情報が存在しないことを示します。
  • 適切な型（liveprop か deadprop か）を選択せずに参照した場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は PROPPATCH のトランザクション管理において不可欠です。
  • 新たなプロパティタイプを追加する際は、ロールバック用の構造体と本構造体へのメンバ追加を検討してください。
