iter_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.c:102

目的

ハッシュテーブルの反復処理（イテレーション）において、現在のイテレータ状態と、処理中に参照すべきユーザー定義のコンテキストを保持するためのラッパー構造体です。

シグネチャまたは形状

typedef struct {
    h2_proxy_ihash_iter_t *iter;
    void *ctx;
} iter_ctx;

入力と出力

  • **入力**:
  • iter: h2_proxy_ihash_iter_t 型のポインタ。ハッシュテーブルの反復処理状態を指します。
  • ctx: void * 型の汎用ポインタ。コールバック関数等に渡す任意のコンテキストデータです。
  • **出力**: なし（構造体定義）。

前提条件と不変条件

  • iter は有効なハッシュテーブルのイテレータを指している必要があります。
  • ctx は、この構造体を利用するコールバック関数が期待する型と一致している必要があります。

アルゴリズムまたは振る舞い

ハッシュテーブルの走査中に、現在のイテレータ位置を保持しつつ、外部から渡されたコンテキスト（ctx）をコールバック関数へ橋渡しする役割を担います。これにより、状態を維持したまま特定の処理を反復的に実行可能です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_util.c 内の関数（110, 116 行付近で参照）。
  • modules/http2/h2_util.c 内の関数（251, 256, 262 行付近で参照）。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • h2_proxy_ihash_iter_t: ハッシュテーブルの反復処理に使用されるイテレータ型。
  • h2_proxy_ihash_t: 本構造体が反復処理の対象とするハッシュテーブル構造体。

失敗モードと境界条件

  • iter が NULL の状態で反復処理を開始した場合、未定義の動作を引き起こす可能性があります。
  • ctx に不正なポインタが設定されている場合、コールバック関数内での参照時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は h2_proxy_util.c と h2_util.c の両方で定義・利用されています。変更を加える際は、両ファイル間での互換性に注意してください。
  • ctx は void * であるため、型安全性が保証されていません。キャスト時の型定義が正しいことを確認してください。
