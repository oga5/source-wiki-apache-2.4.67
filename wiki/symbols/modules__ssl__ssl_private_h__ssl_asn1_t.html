ssl_asn1_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:575

目的

DER形式でエンコードされたASN.1データ（主に秘密鍵や証明書）を、モジュール内のハッシュテーブルで管理・保持するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct {
    long int       nData;
    unsigned char *cpData;
    apr_time_t     source_mtime;
} ssl_asn1_t;

入力と出力

  • nData: 格納されているデータのバイト長。
  • cpData: 実際のASN.1データへのポインタ。
  • source_mtime: データのソースファイル（鍵ファイル等）の最終更新時刻。

前提条件と不変条件

  • cpData が指すメモリ領域は、適切なメモリプール（通常は SSLModConfigRec に関連付けられたプール）によって管理されている必要があります。
  • ssl_asn1_table_* 関数群を通じて操作されることが前提です。

アルゴリズムまたは振る舞い

  • 本構造体は、apr_hash_t を用いたハッシュテーブルのバリューとして格納されます。
  • ssl_asn1_table_set を通じてテーブルに登録され、ssl_asn1_table_get を通じて取得されます。
  • データの生存期間は、ハッシュテーブルが保持されるプロセス生存期間（SSLModConfigRec のライフサイクル）に依存します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_init.c
  • modules/ssl/ssl_engine_pphrase.c
  • modules/ssl/ssl_util.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SSLModConfigRec: 本構造体を格納するハッシュテーブルを保持するグローバル設定構造体。
  • apr_hash_t: 本構造体を管理するためのハッシュテーブル。

失敗モードと境界条件

  • cpData が NULL の場合、または nData が負の値である場合の動作は、本構造体単体では定義されていません。これらを扱う ssl_asn1_table_* 関数側の実装に依存します。

保守担当者向けメモ

  • 本構造体は、パスフレーズ保護された秘密鍵をメモリ上にキャッシュする際に使用されます。
  • source_mtime は、ディスク上のファイルが更新されたかどうかを判定するために使用されます。
  • OpenSSLのデータ構造（X509等）とは異なり、生のDERバイト列を保持する低レイヤーの構造体である点に注意してください。
