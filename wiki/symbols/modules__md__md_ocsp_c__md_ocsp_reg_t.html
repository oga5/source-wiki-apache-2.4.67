md_ocsp_reg_t

定義

  • **種別**: struct
  • **定義箇所**: modules/md/md_ocsp.c:64

目的

マネージド証明書（Managed Domains）における OCSP ステープリング処理のグローバルコンテキストを保持し、証明書の失効状態の管理、更新、および永続化を統合的に制御します。

シグネチャまたは形状

struct md_ocsp_reg_t {
    apr_pool_t *p;
    md_store_t *store;
    const char *user_agent;
    const char *proxy_url;
    apr_hash_t *id_by_external_id;
    apr_hash_t *ostat_by_id;
    apr_thread_mutex_t *mutex;
    md_timeslice_t renew_window;
    md_job_notify_cb *notify;
    void *notify_ctx;
    apr_time_t min_delay;
};

入力と出力

  • **入力**: md_ocsp_reg_make() を通じて初期化され、apr_pool_t に紐付けられます。
  • **出力**: 保持するハッシュテーブル（ostat_by_id 等）を介して、証明書ごとの OCSP ステータス情報を提供します。

前提条件と不変条件

  • mutex フィールドにより、複数スレッドからのステータス読み書きが保護されています。
  • p（プール）が破棄される際、関連する全ステータスが解放されるライフサイクル管理が行われます。

アルゴリズムまたは振る舞い

  • **状態管理**: 証明書 ID をキーとして、メモリ上のハッシュテーブルで OCSP ステータスを管理します。
  • **更新処理**: md_ocsp_renew() を通じて、期限切れ間近のステータスを抽出し、非同期的な更新を試みます。
  • **データ供給**: md_ocsp_get_status() を通じて、TLS ハンドシェイクに必要な DER データをコールバック経由で提供します。

呼び出し元と呼び出し先

  • **呼び出し元**: md_ocsp_reg_make, md_ocsp_prime, md_ocsp_get_status, md_ocsp_count, md_ocsp_renew 等の modules/md/md_ocsp.c 内の関数から参照されています。
  • **呼び出し先**: 構造体内部の md_store_t を介して永続化処理を呼び出します。

関連データ構造

  • md_ocsp_status_t: 個別の証明書の OCSP 状態を保持する構造体。
  • md_store_t: OCSP レスポンスの JSON 形式での永続化に使用。
  • apr_hash_t: 証明書 ID とステータスのマッピングに使用。

失敗モードと境界条件

  • **スレッド競合**: mutex の初期化失敗やロック取得の失敗が境界条件となります。
  • **メモリ不足**: apr_pool_t の枯渇により、ハッシュテーブルの操作が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は modules/md/md_ocsp.c の中心的なコンテキストです。
  • 変更を加える際は、mutex による排他制御が全アクセスパスで適切に行われているか確認してください。
  • md_ocsp_reg_make による初期化時に、必要なリソースが正しく割り当てられているか注意が必要です。
