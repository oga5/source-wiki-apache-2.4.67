a_file

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_file_cache.c:98

目的

mod_file_cache モジュールにおいて、キャッシュ対象の静的ファイルに関するリソース（ファイルハンドルまたはメモリマップ）およびメタデータを保持するための構造体です。

シグネチャまたは形状

typedef struct {
#if APR_HAS_SENDFILE
    apr_file_t *file;
#endif
    const char *filename;
    apr_finfo_t finfo;
    int is_mmapped;
#if APR_HAS_MMAP
    apr_mmap_t *mm;
#endif
    char mtimestr[APR_RFC822_DATE_LEN];
    char sizestr[21];
} a_file;

入力と出力

本構造体は、設定読み込み時（cachefilehandle / cachefilemmap）に初期化され、リクエスト処理時（file_cache_xlat）にハッシュテーブルから取得されます。出力として、mmap_handler や sendfile_handler に渡され、レスポンス生成に使用されます。

前提条件と不変条件

  • APR_HAS_SENDFILE または APR_HAS_MMAP が有効な環境で動作します。
  • サーバー起動時にファイルがオープンされ、サーバーの生存期間中保持されます。
  • mtimestr および sizestr は、リクエストごとの計算コストを削減するため、キャッシュ登録時に事前計算された値が格納されます。

アルゴリズムまたは振る舞い

1. **登録**: cache_the_file 関数により、指定されたファイルの stat 情報とハンドル/メモリマップが取得され、本構造体に格納されます。 2. **検索**: file_cache_xlat フックにおいて、リクエストされたパスをキーとしてハッシュテーブルから本構造体を検索します。 3. **提供**: file_cache_handler が本構造体を参照し、is_mmapped フラグに応じて mmap_handler または sendfile_handler を呼び出し、効率的なファイル転送を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: cache_the_file（初期化時）、file_cache_xlat（検索時）、file_cache_handler（レスポンス生成時）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • a_server_config: キャッシュされた a_file を格納するハッシュテーブル（fileht）を保持するコンテナ。
  • apr_finfo_t: ファイル属性情報。
  • apr_mmap_t: メモリマップ領域（APR_HAS_MMAP 有効時）。
  • apr_file_t: ファイルハンドル（APR_HAS_SENDFILE 有効時）。

失敗モードと境界条件

  • **ディスク変更**: サーバー起動後にディスク上のファイルが変更されても、本構造体は古い情報を保持し続けます（再起動が必要）。
  • **メモリ制限**: 大量のファイルをキャッシュする場合、メモリ消費量が増大します。
  • **環境依存**: sendfile や mmap が利用できない環境では、本構造体の該当フィールドは無効化されます。

保守担当者向けメモ

  • sizestr は 64bit ファイルサイズを保持するために 21 バイト確保されています。
  • Alias や RewriteRule を使用した環境では、キャッシュヒット後も stat() が発生する可能性があるため、パフォーマンス測定時には注意してください。
