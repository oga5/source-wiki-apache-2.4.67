register_hooks

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_substitute.c:741

目的

Apache HTTP Server のフィルタチェーンに対し、mod_substitute が提供する出力フィルタを登録します。

シグネチャまたは形状

static void register_hooks(apr_pool_t *pool)

入力と出力

  • **入力**: apr_pool_t *pool（フック登録に使用されるメモリプール）
  • **出力**: なし（void）

前提条件と不変条件

  • 本関数は Apache のモジュール初期化フェーズで呼び出されることを前提としています。
  • substitute_filter_name が定義済みであり、substitute_filter 関数がフィルタの実装として有効である必要があります。

アルゴリズムまたは振る舞い

1. ap_register_output_filter を呼び出します。 2. substitute_filter_name をフィルタ名として指定し、substitute_filter をフィルタ関数として登録します。 3. フィルタの型を AP_FTYPE_RESOURCE に設定し、リソースレベルのフィルタとしてチェーンに組み込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュールロードプロセス（直接の呼び出し元コードは本ソース抜粋内にはありません）。
  • **呼び出し先**: ap_register_output_filter（Apache API）。

関連データ構造

  • substitute_filter（フィルタのメインロジック関数）

失敗モードと境界条件

  • フィルタ登録時にメモリ不足等の問題が発生した場合、Apache の起動プロセスが失敗する可能性があります。

保守担当者向けメモ

  • 本関数は mod_substitute のエントリポイントの一つです。フィルタの優先順位や型を変更する場合は、ap_register_output_filter の引数を確認してください。
