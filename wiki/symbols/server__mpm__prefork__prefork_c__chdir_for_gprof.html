chdir_for_gprof

定義

  • 種別: function
  • 定義箇所: server/mpm/prefork/prefork.c:163

目的

gprof によるプロファイリング出力ファイルを保存するためのディレクトリへ、カレントワーキングディレクトリを変更します。

シグネチャまたは形状

static void chdir_for_gprof(void)

入力と出力

  • **入力**: 引数はなし。ap_server_conf から取得される core_server_config 内の gprof_dir 設定値を使用します。
  • **出力**: なし（副作用としてプロセスのカレントディレクトリを変更）。

前提条件と不変条件

  • pconf（設定用メモリプール）が有効であること。
  • ap_server_conf が初期化済みであること。
  • gprof_dir が設定されている場合、そのディレクトリを作成する権限が必要。

アルゴリズムまたは振る舞い

1. ap_get_core_module_config を介して gprof_dir 設定を取得します。 2. gprof_dir が指定されている場合:

  • パス末尾が % で終わる場合、ap_append_pid を使用してプロセスIDを付与したディレクトリ名を生成します。
  • ap_server_root_relative で絶対パスを解決します。
  • apr_dir_make を呼び出し、適切なパーミッション（読み書き実行権限）でディレクトリを作成します。作成失敗時はエラーログを出力します（既存ディレクトリの場合は無視）。

    • 3. gprof_dir が指定されていない場合:

  • ap_runtime_dir_relative を使用してランタイムディレクトリを対象とします。

    • 4. 最後に chdir() を実行し、対象ディレクトリへ移動します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/prefork/prefork.c:239（child_main 内から呼び出される）。
  • **呼び出し先**: ap_get_core_module_config, ap_append_pid, ap_server_root_relative, apr_dir_make, ap_log_error, ap_runtime_dir_relative, chdir。

関連データ構造

  • core_server_config: gprof_dir を保持する設定構造体。

失敗モードと境界条件

  • apr_dir_make が失敗し、かつその理由が「ディレクトリが既に存在する (APR_STATUS_IS_EEXIST)」以外である場合、エラーログが出力されます。
  • gprof_dir が未設定の場合、ランタイムディレクトリへの移動が試行されます。

保守担当者向けメモ

  • 本関数はプロファイリング用であり、通常の運用環境では使用されません。
  • pconf を使用しているため、設定フェーズでの利用を前提としています。
  • chdir はプロセス全体の状態を変更するため、マルチスレッド環境では注意が必要ですが、本 MPM は非スレッド型であるため、この実装が許容されています。
