ap_create_request

定義

  • 種別: function
  • 定義箇所: server/protocol.c:1356

目的

HTTP リクエスト処理のための request_rec 構造体をメモリ確保および初期化し、リクエストのライフサイクルを開始する準備を行う。

シグネチャまたは形状

request_rec *ap_create_request(conn_rec *conn)

入力と出力

  • **入力**: conn_rec *conn（現在のコネクション情報）
  • **出力**: 初期化済みの request_rec * 構造体。メモリは conn->pool から作成されたサブプールに割り当てられる。

前提条件と不変条件

  • conn が有効な conn_rec であること。
  • request_rec の各テーブル（headers_in, headers_out 等）は、この関数内で apr_table_make を用いて初期化される。
  • r->status はデフォルトで HTTP_OK に設定される。
  • r->pool はリクエスト終了時に自動的に解放される。

アルゴリズムまたは振る舞い

1. conn->pool を親として新しいメモリプール p を作成し、タグ付けする。 2. request_rec 構造体をゼロクリアして確保する。 3. AP_READ_REQUEST_ENTRY マクロを呼び出し、リクエスト開始を記録する。 4. conn からサーバー情報、フィルタチェーン、クライアントアドレス情報をコピーする。 5. 各種テーブル（headers_in, trailers_in, subprocess_env, headers_out, err_headers_out, trailers_out, notes）を生成する。 6. ap_create_request_config を呼び出し、リクエスト設定を初期化する。 7. ap_run_create_request(r) を実行し、登録されたフックを呼び出す。 8. デフォルトのパス情報、ステータス、ボディ読み取り状態を初期化する。 9. 初期化済みの request_rec を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/protocol.c:1428
  • modules/http2/h2_request.c:381
  • **呼び出し先**:
  • apr_pool_create, apr_pool_tag, apr_pcalloc, apr_table_make (APR API)
  • ap_make_method_list
  • ap_create_request_config
  • ap_run_create_request (フック)

関連データ構造

  • request_rec: リクエストの全状態を保持する主要構造体。
  • conn_rec: コネクションレベルの情報を保持。
  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • メモリ不足時には apr_pool_create や apr_pcalloc がアボートする可能性がある（APR の標準的な挙動）。
  • ap_run_create_request フック内でエラーが発生した場合の挙動は、各フックの実装に依存する。

保守担当者向けメモ

  • 本関数は request_rec の初期化における「テンプレート」として機能する。新しいフィールドを request_rec に追加した場合は、本関数での初期化漏れがないか確認すること。
  • r->per_dir_config は r->server->lookup_defaults で初期化されるため、サーバー設定の変更が正しく反映されているか注意が必要。
