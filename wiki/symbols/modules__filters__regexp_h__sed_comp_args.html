sed_comp_args

定義

  • **種別**: struct
  • **定義箇所**: modules/filters/regexp.h:65

目的

正規表現のコンパイル処理において、コンパイル時の状態やフラグを保持・伝達するために使用されます。

シグネチャまたは形状

typedef struct _sed_comp_args {
    int circf; /* Regular expression starts with ^ */
    int nbra;  /* braces count */
} sed_comp_args;

入力と出力

本構造体は、正規表現コンパイル関数（sed_compile や comple）の引数として渡されます。

  • **入力**: コンパイル対象の正規表現パターン解析時に、行頭一致フラグや括弧のネスト数が設定されます。
  • **出力**: コンパイル完了後、解析された正規表現の特性が呼び出し元へ伝達されます。

前提条件と不変条件

  • nbra（括弧のネスト数）は、libsed の仕様により最大 9 に制限されています。
  • コンパイル関数へ渡される前に、呼び出し元で適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • sed_compile 等の関数が正規表現をバイトコードへ変換する際、この構造体を参照して正規表現のメタ情報を更新します。
  • circf は、正規表現が ^ で始まる場合に真（非ゼロ）となります。
  • nbra は、正規表現内の括弧のペア数をカウントし、後方参照の管理に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・設定）**:
  • modules/filters/regexp.c: sed_compile 関数
  • modules/filters/sed0.c: comple 関数および関連するコンパイル処理
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • sed_commands_t: libsed.h で定義されるコマンド構造体。コンパイルの入力元となります。
  • step_vars_storage: 正規表現実行時のマッチング状態を管理する構造体。

失敗モードと境界条件

  • nbra が 9 を超える場合、libsed の制限により正常なコンパイルが保証されません。

保守担当者向けメモ

  • 本構造体は AT&T sed の実装を継承しており、非常に限定的な機能セットを前提としています。
  • 現代的な正規表現ライブラリ（PCRE 等）への移行を検討する際は、この構造体が保持するフラグが現在のバイトコード体系と密結合している点に注意してください。
