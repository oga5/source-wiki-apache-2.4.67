register_hooks

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_data.c:243

目的

Apache HTTP Server のフィルタチェーンに対し、mod_data モジュールが提供する出力フィルタを登録するために使用されます。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p (モジュールのライフサイクルを管理するメモリプール)
  • **出力**: なし（戻り値は void）

前提条件と不変条件

  • 本関数は Apache のモジュール初期化フェーズで呼び出されることを前提としています。
  • DATA_FILTER という識別子（定数）が定義されている必要があります。

アルゴリズムまたは振る舞い

1. ap_register_output_filter を呼び出します。 2. DATA_FILTER をフィルタ名として指定し、フィルタ本体である data_out_filter を登録します。 3. フィルタの型として AP_FTYPE_RESOURCE を指定し、リソースレベルのフィルタとしてチェーンに組み込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュールロードプロセス（直接的な呼び出し元コードは提供されたソース内にはありません）。
  • **呼び出し先**: ap_register_output_filter (Apache API) を呼び出します。
  • **注記**: 提供されたツール出力に含まれる他の register_hooks は、本シンボルとは異なるモジュール（mod_auth_basic 等）の同名関数であり、本シンボルの呼び出し元ではありません。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • data_out_filter: 登録対象となるフィルタ関数。

失敗モードと境界条件

  • ap_register_output_filter が失敗した場合の挙動については、Apache API の仕様に依存します。本関数自体はエラーコードを返さないため、登録失敗はサーバー起動時のエラーログとして記録されることが一般的です。

保守担当者向けメモ

  • 本関数は mod_data のエントリーポイントの一つです。フィルタの適用順序を変更したい場合は、AP_FTYPE_RESOURCE の指定を検討してください。
  • フィルタ名 DATA_FILTER の定義箇所と整合性が取れていることを確認してください。
