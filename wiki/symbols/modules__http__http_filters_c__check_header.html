check_header

定義

  • 種別: function
  • 定義箇所: modules/http/http_filters.c:697

目的

レスポンスヘッダの名称および値の妥当性を検証し、必要に応じてヘッダ値の折り返し（folding）を正規化する。

シグネチャまたは形状

static int check_header(struct check_header_ctx *ctx, const char *name, const char **val)

入力と出力

  • **入力**:
  • ctx: 検証コンテキスト（strict モードフラグおよびリクエスト情報を含む）。
  • name: 検証対象のヘッダ名。
  • val: 検証対象のヘッダ値へのポインタ。
  • **出力**:
  • 戻り値: 検証に成功した場合は 1、失敗した場合は 0。
  • val: 修正が必要な場合、メモリプールから確保された正規化済みの文字列へのポインタに更新される。

前提条件と不変条件

  • name は空文字列であってはならない。
  • ctx->strict が真の場合、ヘッダ名は HTTP トークンとして妥当である必要がある。
  • ctx->strict が偽の場合、ヘッダ名は VCHAR または obs-text で構成される必要がある。

アルゴリズムまたは振る舞い

1. **ヘッダ名の検証**: name が空でないか確認し、ctx->strict の設定に応じて ap_scan_http_token または ap_scan_vchar_obstext を用いて不正な文字が含まれていないか走査する。 2. **ヘッダ値の走査と正規化**:

  • ap_scan_http_field_content を用いてヘッダ値を走査する。
  • 折り返し（CRLF + 空白/タブ）が検出された場合、それを単一のスペースに置換する。
  • 置換が発生する場合、ctx->r->pool からメモリを確保し、正規化された文字列を生成する。

    • 3. **エラー処理**: 不正な文字や不正な形式が検出された場合、ap_log_rerror でエラーを記録し、0 を返して処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_filters.c:765（ソースコード内の直接的な呼び出し）。
  • **呼び出し先**:
  • ap_log_rerror (ログ出力)
  • ap_scan_http_token / ap_scan_vchar_obstext / ap_scan_http_field_content (解析用ユーティリティ)
  • apr_palloc (メモリ確保)
  • memcpy, strspn (文字列操作)
  • **補足**: modules/filters/mod_deflate.c:25 からの参照がツールにより示唆されているが、本ソース excerpt 内での直接的な証拠はない。

関連データ構造

  • struct check_header_ctx: 検証状態を保持する構造体。

失敗モードと境界条件

  • **空のヘッダ名**: name[0] == '\0' の場合、エラーとして 0 を返す。
  • **不正な文字**: ヘッダ名またはヘッダ値に許可されない文字が含まれる場合、エラーとして 0 を返す。
  • **不正な折り返し**: 折り返しシーケンスが正しくない（CRLF の直後に空白/タブがない）場合、エラーとして 0 を返す。

保守担当者向けメモ

  • 本関数はレスポンスヘッダのサニタイズを担うため、RFC 7230 等のプロトコル仕様変更時には、走査関数（ap_scan_*）の挙動と併せて検証ロジックを確認すること。
  • メモリ確保は ctx->r->pool に対して行われるため、リクエストのライフサイクルを超えたメモリ保持は行われない。
