set_keep_alive_max

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:90

目的

HTTP 接続における「Keep-Alive」状態の最大リクエスト数を設定するための設定ディレクティブハンドラです。

シグネチャまたは形状

static const char *set_keep_alive_max(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: 設定値として渡される文字列（数値に変換される）。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時はエラーメッセージを示す文字列を返します。

前提条件と不変条件

  • ap_check_cmd_context により、ディレクトリコンテキスト（.htaccess 等）での設定が禁止されています（NOT_IN_DIR_CONTEXT）。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、現在の設定コンテキストが許可されているか検証します。エラーがあればそのメッセージを即座に返します。 2. arg を atoi で整数に変換し、cmd->server->keep_alive_max に代入します。 3. NULL を返し、正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_core.c:105 にて、AP_INIT_TAKE1 マクロを通じて「MaxKeepAliveRequests」ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: ap_check_cmd_context（Apache Core API）、atoi（標準ライブラリ）。

関連データ構造

  • cmd_parms: 設定コマンドの実行時パラメータを保持する構造体。
  • server_rec (cmd->server): サーバー設定を保持する構造体。keep_alive_max フィールドを更新します。

失敗モードと境界条件

  • ディレクトリコンテキストでこのディレクティブが使用された場合、ap_check_cmd_context がエラーを返します。
  • arg に数値以外の文字列が渡された場合、atoi の仕様に従い 0 として処理されます。

保守担当者向けメモ

  • 本関数は MaxKeepAliveRequests ディレクティブの値をサーバー設定に反映する役割のみを持ちます。
  • 実際の Keep-Alive 処理のロジックは、この設定値を参照する他のモジュールや http_protocol.c 等の接続処理フローに依存します。
