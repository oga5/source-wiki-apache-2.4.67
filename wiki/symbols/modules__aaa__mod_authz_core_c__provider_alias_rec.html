provider_alias_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authz_core.c:50

目的

AuthzProviderAlias ディレクティブによって定義された認可プロバイダーのエイリアス設定を保持するために使用されます。

シグネチャまたは形状

typedef struct provider_alias_rec {
    char *provider_name;
    char *provider_alias;
    char *provider_args;
    const void *provider_parsed_args;
    ap_conf_vector_t *sec_auth;
    const authz_provider *provider;
} provider_alias_rec;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。設定フェーズにおいて、AuthzProviderAlias の解析結果が各フィールドに格納されます。

前提条件と不変条件

  • provider フィールドは、登録済みの認可プロバイダー（authz_provider）を指す必要があります。
  • sec_auth は、エイリアスに関連付けられたディレクトリ設定ベクトルを保持します。

アルゴリズムまたは振る舞い

本構造体は、認可プロバイダーのエイリアス化および再設定の管理に使用されます。リクエスト処理時には、この構造体を通じてエイリアス化されたプロバイダーの check_authorization 関数が呼び出されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_core.c および modules/aaa/mod_authz_core.c 内のエイリアス設定処理関数（apr_hash_get や apr_pcalloc を使用）。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出しません。ただし、provider フィールドが指す authz_provider 構造体を通じて、認可ロジックが実行されます。

関連データ構造

  • authz_provider: 認可プロバイダーのインターフェースを定義する構造体。
  • ap_conf_vector_t: Apache の設定ベクトル。

失敗モードと境界条件

  • エイリアス名が重複している場合や、参照先のプロバイダーが存在しない場合の挙動については、本構造体定義からは判断できません。

保守担当者向けメモ

  • 本構造体は mod_authz_core.c と mod_authn_core.c の両方で参照されています。変更を加える際は、両モジュール間での整合性に注意してください。
