cached_request_time

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_log_config.c:610

目的

リクエスト時刻の文字列変換処理を最適化するため、計算済みの時刻情報をキャッシュする構造体です。

シグネチャまたは形状

typedef struct {
    unsigned t;
    char timestr[DEFAULT_REQUEST_TIME_SIZE];
    unsigned t_validate;
} cached_request_time;

入力と出力

  • 入力: 構造体内の t（時刻値）および t_validate（検証用値）。
  • 出力: timestr（フォーマット済みの時刻文字列）。

前提条件と不変条件

  • timestr は DEFAULT_REQUEST_TIME_SIZE のサイズ制限に従う必要があります。
  • t と t_validate は、キャッシュの有効性を判定するためのペアとして使用されます。

アルゴリズムまたは振る舞い

リクエスト処理中に時刻文字列を生成する際、この構造体を用いて計算コストの高い文字列変換処理をスキップします。キャッシュされた t と現在の時刻を比較し、一致する場合に timestr を再利用します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/loggers/mod_log_config.c 内の時刻処理ロジック。
  • 呼び出し先: apr_palloc を介してリクエストプール (r->pool) からメモリを割り当てる際に使用されます。
  • 補足: 具体的な関数呼び出しの連鎖については、提供されたソース excerpt には詳細な記述がありません。

関連データ構造

  • request_time_cache: TIME_CACHE_SIZE をサイズとする cached_request_time 型の静的配列。
  • request_rec: リクエストのメモリプール (r->pool) を提供し、キャッシュの生存期間を管理します。

失敗モードと境界条件

  • DEFAULT_REQUEST_TIME_SIZE を超える時刻文字列は生成できません。
  • キャッシュの衝突や無効化のロジックが不完全な場合、古い時刻文字列が返される可能性があります。

保守担当者向けメモ

  • 本構造体は mod_log_config.c 内の時刻キャッシュメカニズムの基盤です。
  • 変更を加える際は、TIME_CACHE_SIZE および DEFAULT_REQUEST_TIME_SIZE の定義と整合性を取ってください。
