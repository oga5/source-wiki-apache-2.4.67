proxy_header

定義

  • 種別: typedef (共用体)
  • 定義箇所: modules/metadata/mod_remoteip.c:114

目的

PROXY プロトコル（v1 および v2）のヘッダー情報を保持するための共用体です。

シグネチャまたは形状

typedef union {
    proxy_v1 v1;
    proxy_v2 v2;
} proxy_header;

入力と出力

  • 入力: PROXY プロトコル形式のバイナリデータ。
  • 出力: v1 または v2 メンバを介した、プロトコルバージョンごとの構造化された接続情報。

前提条件と不変条件

  • proxy_v1 および proxy_v2 構造体のサイズを包含するメモリ領域が必要です。
  • データの解釈には、事前に remoteip_determine_version 等で判別されたバージョン情報が必要です。

アルゴリズムまたは振る舞い

  • proxy_v1 と proxy_v2 のメモリレイアウトを重ね合わせることで、プロトコルバージョンに応じた柔軟なデータアクセスを提供します。
  • remoteip_input_filter において、ストリームから読み取ったバイナリデータをこの型にキャストして解析します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/metadata/mod_remoteip.c:131 (バッファ確保)
  • modules/metadata/mod_remoteip.c:771, 946, 1018, 1027, 1146, 1152, 1163 (解析および型変換)
  • 呼び出し先:
  • 呼び出し先情報は提供されていません。

関連データ構造

  • proxy_v1: PROXY プロトコル v1 形式の構造体。
  • proxy_v2: PROXY プロトコル v2 形式の構造体。
  • remoteip_filter_context: 本構造体を保持するフィルタコンテキスト。

失敗モードと境界条件

  • 不正なプロトコルデータが渡された場合、共用体のメンバアクセスによってメモリ上の不整合な値が読み取られる可能性があります。
  • remoteip_get_v2_len 等でサイズを検証せずにアクセスすると、バッファオーバーランや不正なメモリ参照が発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は mod_remoteip の PROXY プロトコル解析の基盤です。proxy_v1 または proxy_v2 の定義が変更された場合、この共用体のサイズとアライメントに影響がないか確認してください。
