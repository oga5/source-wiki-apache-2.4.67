cgi_bucket_dup

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:294

目的

既存の cgi_bucket_data を参照する新しい CGI 型のバケットを生成し、バケットチェーンへの挿入を可能にする。

シグネチャまたは形状

static apr_bucket *cgi_bucket_dup(struct cgi_bucket_data *data,
                                  apr_bucket_alloc_t *list)

入力と出力

  • **入力**:
  • data: バケットに関連付ける cgi_bucket_data 構造体へのポインタ。
  • list: バケットの割り当てに使用する apr_bucket_alloc_t プール。
  • **出力**:
  • 初期化済みの apr_bucket 構造体へのポインタ。

前提条件と不変条件

  • list は有効な apr_bucket_alloc_t であること。
  • data は、バケットの生存期間中有効な cgi_bucket_data を指していること。
  • バケットの長さ (length) は (apr_size_t)(-1)（不定）、開始位置 (start) は -1 に固定される。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を使用してバケット構造体メモリを確保する。 2. APR_BUCKET_INIT マクロでバケットを初期化する。 3. バケットの解放関数を apr_bucket_free に設定する。 4. バケットの型を bucket_type_cgi に設定する。 5. 入力された data をバケットの data フィールドに格納する。 6. 初期化したバケットを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/cgi_common.h:336 (APR_BUCKET_INSERT_AFTER マクロ内での呼び出しが確認されている)
  • **呼び出し先**: apr_bucket_alloc (APR API)

関連データ構造

  • struct cgi_bucket_data: バケットが保持するポーリング用データ。
  • apr_bucket: Apache のバケット構造体。
  • bucket_type_cgi: CGI 型バケットの定義（本関数内で参照）。

失敗モードと境界条件

  • メモリ不足時には apr_bucket_alloc が失敗し、プログラムが異常終了する可能性がある（APR のメモリ割り当てポリシーに依存）。

保守担当者向けメモ

  • 本関数は static インライン関数として定義されており、cgi_common.h をインクルードするモジュールごとに実体が生成される。
  • バケットのデータポインタはコピーされず、参照が共有されるため、cgi_bucket_data のライフサイクル管理には注意が必要である。
