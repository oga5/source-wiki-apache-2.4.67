should_send_brigade

定義

  • 種別: function
  • 定義箇所: server/core_filters.c:360

目的

出力フィルタにおいて、バケットブリゲードをソケットへ書き出す（フラッシュする）べきか否かを、設定された閾値やバケットの種類に基づいて判断するヒューリスティック関数です。

シグネチャまたは形状

static int should_send_brigade(apr_bucket_brigade *bb, conn_rec *c, int *flush)

入力と出力

  • **入力**:
  • bb: 評価対象のバケットブリゲード。
  • c: 現在の接続情報 (conn_rec)。
  • flush: フラッシュの必要性を格納する出力用ポインタ（NULL可）。
  • **出力**:
  • 戻り値: int 型。直ちに送信を行うべき場合は非ゼロ（真）、そうでない場合は 0（偽）を返す。
  • *flush: 呼び出し元がフラッシュを明示的に要求する場合、内部判定結果が書き込まれる。

前提条件と不変条件

  • c->base_server->module_config からコアモジュールの設定 (core_server_config) が取得可能であること。
  • bb は apr_bucket_brigade として有効であること。

アルゴリズムまたは振る舞い

ブリゲード内のバケットを走査し、以下の条件のいずれかに該当する場合、need_flush を真として判定を終了します。 1. **フラッシュバケット**: APR_BUCKET_IS_FLUSH が含まれる場合。 2. **モルフィングバケット**: 長さが不明 (-1) なバケットが含まれる場合。 3. **閾値超過**: メモリ上のバケット（非ファイルバケット）の合計サイズが flush_max_threshold を超える場合。 4. **パイプライン制限**: EOR (End of Request) バケット数が flush_max_pipelined を超える場合。

最終的な戻り値は、上記の need_flush が真であるか、または合計データサイズが THRESHOLD_MIN_WRITE を超えるか、あるいはファイルバケットが含まれている場合に真となります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core_filters.c:497
  • server/core_filters.c:509
  • server/core_filters.c:514
  • **呼び出し先**:
  • ap_get_core_module_config (マクロ)
  • ap_log_cerror (ログ出力)
  • APR_BRIGADE_FIRST, APR_BRIGADE_SENTINEL, APR_BUCKET_NEXT (APR マクロ)
  • APR_BUCKET_IS_METADATA, APR_BUCKET_IS_FILE, APR_BUCKET_IS_FLUSH, AP_BUCKET_IS_EOR (APR/HTTPD マクロ)

関連データ構造

  • apr_bucket_brigade: バケットの連結リスト。
  • conn_rec: 接続状態を保持する構造体。
  • core_server_config: flush_max_threshold や flush_max_pipelined を保持するコア設定構造体。

失敗モードと境界条件

  • flush_max_pipelined が負の値の場合、EOR バケットによるフラッシュ判定はスキップされます。
  • total_bytes が THRESHOLD_MIN_WRITE に達しない場合でも、ファイルバケットが存在すれば送信が優先されます。

保守担当者向けメモ

  • 本関数は core_output_filter のフロー制御の要です。DoS 対策としてパイプライン化されたリクエストのバッファリングを制限しているため、閾値の変更には注意が必要です。
  • flush 引数が NULL の場合、ログ出力は行われません。
