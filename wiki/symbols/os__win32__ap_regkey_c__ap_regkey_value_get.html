ap_regkey_value_get

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:179

目的

Windows レジストリから文字列値（REG_SZ または REG_EXPAND_SZ）を取得し、必要に応じて環境変数を展開して呼び出し元に返します。

シグネチャまたは形状

apr_status_t ap_regkey_value_get(char **result,
                                 ap_regkey_t *key,
                                 const char *valuename,
                                 apr_pool_t *pool)

入力と出力

  • **入力**:
  • key: 操作対象のレジストリキー（ap_regkey_t 型）。
  • valuename: 取得する値の名前。
  • pool: メモリ割り当てに使用する APR プール。
  • **出力**:
  • result: 取得した文字列が格納されるバッファへのポインタ。
  • 戻り値: 成功時は APR_SUCCESS、失敗時は apr_status_t 形式のエラーコード。

前提条件と不変条件

  • key は有効な ap_regkey_t オブジェクトである必要があります。
  • pool はメモリ割り当てのために有効である必要があります。
  • 取得対象のレジストリ値は REG_SZ または REG_EXPAND_SZ 型である必要があります。それ以外の型の場合、エラーとなります。

アルゴリズムまたは振る舞い

1. **プラットフォーム判定**: コンパイル時の設定（APR_HAS_UNICODE_FS / APR_HAS_ANSI_FS）に基づき、Unicode または ANSI API を選択します。 2. **サイズ取得**: RegQueryValueEx を呼び出し、必要なバッファサイズを特定します。 3. **メモリ確保**: apr_palloc を使用して、特定したサイズ分のメモリを pool から確保します。 4. **値の取得**: 再度 RegQueryValueEx を呼び出し、データを取得します。 5. **環境変数展開**: 型が REG_EXPAND_SZ の場合、ExpandEnvironmentStrings を使用して %SystemRoot% 等の環境変数を解決します。 6. **変換（Unicode時）**: Unicode 環境では、取得したデータを UTF-8 に変換して result に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/win32/mod_win32.c
  • server/mpm/winnt/service.c
  • **呼び出し先**:
  • Win32 API: RegQueryValueEx (W/A), ExpandEnvironmentStrings (W/A)
  • APR: apr_palloc, apr_conv_utf8_to_ucs2, apr_conv_ucs2_to_utf8, APR_FROM_OS_ERROR

関連データ構造

  • ap_regkey_t: レジストリキーハンドル (HKEY) を保持するコンテナ。

失敗モードと境界条件

  • レジストリ値が存在しない、またはアクセス権限がない場合、Win32 エラーコードを apr_status_t に変換して返します。
  • 指定された値が REG_SZ / REG_EXPAND_SZ 以外の場合、ERROR_INVALID_PARAMETER を返します。
  • 文字列変換時にバッファが不足する場合、APR_ENAMETOOLONG を返します。

保守担当者向けメモ

  • REG_EXPAND_SZ の処理において、ExpandEnvironmentStrings を2回呼び出すことで必要なバッファサイズを動的に決定しています。
  • Unicode 版では、メモリ効率と CPU サイクルのトレードオフとして、バッファのオーバーアロケーションが行われています。
