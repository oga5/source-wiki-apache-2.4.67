clean_env

定義

  • 種別: function
  • 定義箇所: support/suexec.c:224

目的

CGI プログラム実行前に環境変数を安全な状態へリセットし、ホワイトリストに基づいた環境変数のみを再構築することで、環境変数経由の攻撃を防止する。

シグネチャまたは形状

static void clean_env(void)

入力と出力

  • **入力**: グローバル変数 environ（現在の環境変数）、および safe_env_lst（許可される環境変数のリスト）。
  • **出力**: なし（グローバル変数 environ を書き換える）。

前提条件と不変条件

  • environ を一時的に空にすることで、環境変数に依存するライブラリ関数（malloc や sprintf 等）の実行時リスクを排除する。
  • AP_ENVBUF で定義された上限数まで環境変数を保持する。
  • 実行対象のプログラムへ渡される環境変数は、safe_env_lst に合致するもののみに制限される。

アルゴリズムまたは振る舞い

1. 現在の environ を退避し、environ を空ポインタに設定して環境を隔離する。 2. AP_ENVBUF サイズのメモリを確保し、PATH 環境変数を AP_SAFE_PATH で初期化する。 3. 退避した元の環境変数を走査し、各変数が safe_env_lst のいずれかの接頭辞と一致する場合のみ、新しい環境リストに追加する。 4. 新しい環境リストの末尾に NULL を設定し、environ をこのリストで更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: support/suexec.c:291 (main 関数内)
  • **呼び出し先**: log_err（エラー発生時）、calloc, strdup, strncmp, strlen

関連データ構造

  • safe_env_lst: 許可される環境変数の接頭辞を保持する配列。
  • AP_ENVBUF: 環境変数リストの最大保持数。

失敗モードと境界条件

  • メモリ確保（calloc または strdup）に失敗した場合、log_err を呼び出し、それぞれ終了コード 123 または 124 でプロセスを終了する。
  • 環境変数が AP_ENVBUF の上限を超えた場合、それ以降の環境変数は無視される。

保守担当者向けメモ

  • environ の操作はセキュリティに直結するため、safe_env_lst の定義変更には細心の注意を払うこと。
  • strdup で確保されたメモリは execv 呼び出しまで保持されるため、メモリリークの懸念はないが、この関数内での動的確保の失敗は即座に致命的なエラーとして扱う。
