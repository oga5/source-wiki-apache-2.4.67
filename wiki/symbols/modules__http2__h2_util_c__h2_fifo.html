h2_fifo

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.c:526

目的

スレッドセーフな FIFO（先入れ先出し）キューを提供し、マルチスレッド環境下でのデータ受け渡しを管理します。

シグネチャまたは形状

struct h2_fifo {
    void **elems;
    int capacity;
    int set;
    int in;
    int out;
    int count;
    int aborted;
    apr_thread_mutex_t *lock;
    apr_thread_cond_t  *not_empty;
    apr_thread_cond_t  *not_full;
};

入力と出力

  • **入力**: void **elems（要素格納用バッファ）、int capacity（最大容量）。
  • **出力**: apr_status_t（操作の成功または失敗を示すステータスコード）。

前提条件と不変条件

  • lock によって保護されたクリティカルセクション内で操作される必要があります。
  • not_empty および not_full 条件変数を用いて、スレッド間の同期を行います。
  • aborted フラグが設定された場合、キューの操作は中断されます。

アルゴリズムまたは振る舞い

  • apr_thread_mutex_lock を使用して排他制御を行います。
  • in（挿入位置）と out（取り出し位置）のインデックスを管理する循環バッファとして動作します。
  • キューが満杯の場合は not_full で待機し、空の場合は not_empty で待機するブロッキング操作をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_fifo_create, h2_fifo_set_create, h2_fifo_term, h2_fifo_count, h2_fifo_push 等のユーティリティ関数から参照されます。
  • **呼び出し先**: APR のスレッド同期プリミティブ（apr_thread_mutex_t, apr_thread_cond_t）を呼び出します。

関連データ構造

  • apr_thread_mutex_t: 排他制御用。
  • apr_thread_cond_t: スレッド待機および通知用。

失敗モードと境界条件

  • **容量超過**: capacity に達した状態で push を行うと、非ブロッキングモードではエラーを返し、ブロッキングモードでは not_full で待機します。
  • **空の状態**: count が 0 の状態で pull を行うと、同様に待機またはエラーが発生します。
  • **中断**: aborted 状態に遷移した場合、待機中のスレッドは解放され、操作は失敗します。

保守担当者向けメモ

  • 本構造体は h2_util.c 内の静的関数および公開 API によって操作されます。
  • メモリ管理は apr_pool_t に依存しているため、作成時に適切なプールが割り当てられていることを確認してください。
