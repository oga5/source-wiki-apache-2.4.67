is_best_byrequests

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_byrequests.c:28

目的

リクエスト数ベースのロードバランシングにおいて、各ワーカーの累積負荷状況（lbstatus）を更新し、現時点で最も優先すべきワーカーを判定するコールバック関数です。

シグネチャまたは形状

static int is_best_byrequests(proxy_worker *current, proxy_worker *prev_best, void *baton)

入力と出力

  • **入力**:
  • current: 現在評価中の proxy_worker 構造体。
  • prev_best: これまでに評価された中で最も優先度が高いと判断された proxy_worker（初回は NULL）。
  • baton: 全ワーカーの lbfactor の合計値を保持する int * 型のポインタ。
  • **出力**:
  • int: current が prev_best よりも優先されるべき場合は非ゼロ（真）、それ以外は 0（偽）を返します。

前提条件と不変条件

  • current および prev_best の s メンバ（proxy_worker_shared）が有効であること。
  • baton は int 型の有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. current->s->lbstatus に current->s->lbfactor を加算し、現在のワーカーの負荷状況を更新します。 2. baton が指す total_factor に current->s->lbfactor を加算します。 3. 以下のいずれかの場合に真を返します。

  • prev_best が NULL である（最初のワーカーである）。
  • current->s->lbstatus が prev_best->s->lbstatus よりも大きい。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/balancers/mod_lbmethod_byrequests.c:87 にて、ap_proxy_balancer_get_best_worker_fn の引数として渡されます。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、proxy_worker 構造体のメンバへアクセスします。

関連データ構造

  • proxy_worker: ロードバランサーのバックエンドサーバーを表す構造体。
  • proxy_worker_shared: 共有メモリ上に保持されるワーカーの状態（lbstatus, lbfactor を含む）。

失敗モードと境界条件

  • prev_best が NULL の場合、最初のワーカーが自動的に選出されます。
  • lbstatus の加算により整数オーバーフローが発生する可能性については、本関数単体では制御していません。

保守担当者向けメモ

  • 本関数は ap_proxy_balancer_get_best_worker_fn によってワーカーリストを走査する際に呼び出されます。
  • lbstatus は共有メモリ上に保持されるため、この更新はプロセス間で永続化されます。
  • total_factor の計算は、選出後の lbstatus 調整（全ワーカーの合計値減算）のために使用されます。
