merge_dbd_config

定義

  • 種別: function
  • 定義箇所: modules/database/mod_dbd.c:116

目的

Apache HTTP Server の設定マージフェーズにおいて、親コンテキスト（basev）と子コンテキスト（addv）のデータベース設定を統合し、新しいサーバー設定構造体（svr_cfg）を生成する。

シグネチャまたは形状

static void *merge_dbd_config(apr_pool_t *pool, void *basev, void *addv)

入力と出力

  • **入力**:
  • pool: 設定メモリを割り当てるための APR プール。
  • basev: 親コンテキストの svr_cfg 構造体。
  • addv: 現在のコンテキストの svr_cfg 構造体。
  • **出力**:
  • 統合された設定を含む新しい svr_cfg 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は svr_cfg 型にキャスト可能であること。
  • add 側の設定値が未設定（デフォルト値）の場合、base 側の値が優先的に継承される。
  • queries はハッシュのオーバーレイ、init_queries は配列の結合によって統合される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して新しい svr_cfg および内部の dbd_cfg_t をメモリ確保する。 2. server は add 側の値を採用する。 3. name（ドライバ名）および params（接続パラメータ）は、add 側が空でない場合に採用し、それ以外は base 側の値を継承する。 4. persist（持続接続フラグ）は、add 側が明示的に設定されている場合のみ採用する。 5. APR_HAS_THREADS が有効な場合、nmin, nkeep, nmax, exptime の各設定値について、add 側でフラグ（set）が立っているもののみを上書きする。 6. queries ハッシュと init_queries 配列をそれぞれ apr_hash_overlay および apr_array_append で統合する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/database/mod_dbd.c:991（Apache の設定マージ処理から呼び出される）。
  • **呼び出し先**:
  • apr_pcalloc
  • apr_hash_overlay
  • apr_array_append

関連データ構造

  • svr_cfg: サーバーごとの設定コンテナ。
  • dbd_cfg_t: データベース接続設定の詳細を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がエラーを発生させる可能性がある。
  • add 側の設定が空文字列やデフォルト値（-1）の場合、正しく base 側の値が選択されるよう条件分岐が設計されている。

保守担当者向けメモ

  • 本関数は Apache の設定マージ機構の一部であり、mod_dbd のディレクティブが階層構造（サーバー間など）で継承される際に使用される。
  • APR_HAS_THREADS の有無によってコンパイルされるメンバが異なるため、スレッド関連の設定変更時には注意が必要。
