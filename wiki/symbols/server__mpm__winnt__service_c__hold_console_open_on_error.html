hold_console_open_on_error

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/service.c:111

目的

エラー発生時にコンソールウィンドウを即座に閉じず、ユーザーがエラーメッセージを確認し、キー入力またはマウス操作によって明示的に終了できるようにする。

シグネチャまたは形状

void hold_console_open_on_error(void)

入力と出力

  • 入力: なし（グローバル変数 ap_real_exit_code を参照）
  • 出力: なし（副作用としてコンソールへのメッセージ出力と入力待機を行う）

前提条件と不変条件

  • グローバル変数 ap_real_exit_code が真であること。
  • 標準入力および標準エラー出力が有効なハンドルとして取得可能であること。

アルゴリズムまたは振る舞い

1. ap_real_exit_code を確認し、偽であれば即座に終了する。 2. 標準入力および標準エラー出力のハンドルを取得する。 3. コンソールに「エラーを確認し、ESCキーを押して終了せよ」という旨のメッセージを出力する。 4. コンソールの入力モードをマウス入力および特定フラグ付きの状態に設定する。 5. ループ内で以下のいずれかが発生するまで待機する：

  • ESCキー（ASCII 27）の押下。
  • マウスのダブルクリック。

    • 6. ループ中、30秒のカウントダウンを標準エラー出力に表示し続ける。 7. 30秒経過、または待機中にエラーが発生した場合は終了する。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/winnt/mpm_winnt.c:1113 にて atexit() を介して登録されている。
  • 呼び出し先: Windows API (GetStdHandle, WriteConsole, GetConsoleScreenBufferInfo, SetConsoleMode, PeekConsoleInput, ReadConsoleInput, SetConsoleCursorPosition, WaitForSingleObject)。

関連データ構造

  • HANDLE: コンソールハンドル。
  • CONSOLE_SCREEN_BUFFER_INFO: コンソール画面バッファ情報。
  • INPUT_RECORD: コンソール入力イベント。

失敗モードと境界条件

  • ap_real_exit_code が偽の場合、何も行わずに戻る。
  • コンソールハンドルが INVALID_HANDLE_VALUE の場合、何も行わずに戻る。
  • WriteConsole や GetConsoleScreenBufferInfo 等のWindows APIが失敗した場合、その時点で処理を中断して戻る。
  • 30秒経過すると、ユーザー入力がなくてもループを抜けて終了する。

保守担当者向けメモ

  • 本関数は atexit で登録されるため、プロセス終了時のクリーンアップ処理として実行されます。
  • ap_real_exit_code フラグの制御により、サービス実行時とコンソール実行時で挙動を切り替えています。
  • 30秒のタイムアウト処理は time(NULL) と WaitForSingleObject を組み合わせて実装されています。
