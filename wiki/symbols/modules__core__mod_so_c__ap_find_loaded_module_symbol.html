ap_find_loaded_module_symbol

定義

  • 種別: function
  • 定義箇所: modules/core/mod_so.c:342

目的

指定された名前を持つロード済みモジュールの構造体ポインタ（module *）を、サーバー設定から検索して返します。

シグネチャまたは形状

module *ap_find_loaded_module_symbol(server_rec *s, const char *modname)

入力と出力

  • **入力**:
  • s: 検索対象のサーバー設定を保持する server_rec 構造体へのポインタ。
  • modname: 検索するモジュール名（文字列）。
  • **出力**:
  • 見つかった場合は対応する module 構造体へのポインタ。
  • 見つからない場合は NULL。

前提条件と不変条件

  • s が有効な server_rec を指していること。
  • s のモジュール設定から so_module に関連付けられた so_server_conf が取得可能であること。
  • sconf->loaded_modules が初期化済みであること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、サーバー設定から so_server_conf を取得します。 2. loaded_modules 配列内の各要素（ap_module_symbol_t）を順次走査します。 3. 各要素の name フィールドと引数の modname を strcmp で比較します。 4. 一致する名前が見つかった場合、その要素が保持する modp（module 構造体ポインタ）を返します。 5. 配列の最後まで一致が見つからなかった場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core.c: オプション関数として取得され、モジュールシンボルの存在確認に使用されます。
  • **呼び出し先**:
  • ap_get_module_config（Apache Core API）
  • strcmp（標準Cライブラリ）

関連データ構造

  • so_server_conf: ロードされたモジュールリストを保持する構造体。
  • ap_module_symbol_t: モジュール名と module 構造体ポインタを保持する内部管理構造体。
  • module: Apache のモジュール定義構造体。

失敗モードと境界条件

  • modname が NULL の場合、比較は行われずスキップされます。
  • loaded_modules が空の場合、ループは実行されず NULL が返されます。
  • サーバー設定が正しく初期化されていない場合、ap_get_module_config の結果が不正となり、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は APR_DECLARE_OPTIONAL_FN を通じて公開されており、他のモジュールから動的に取得可能です。
  • server/core.c での利用時にセグメンテーションフォールトを避けるためのチェックが行われているため、呼び出し側での引数の妥当性確認が重要です。
