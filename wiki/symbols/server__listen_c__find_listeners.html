find_listeners

定義

  • 種別: function
  • 定義箇所: server/listen.c:405

目的

指定されたアドレス、ポート、スコープIDに一致するリスナーをリストから検索し、必要に応じてそのリスナーを別のリストへ移動させる。

シグネチャまたは形状

static int find_listeners(ap_listen_rec **from, ap_listen_rec **to,
                          const char *addr, apr_port_t port,
                          const char *scope_id, apr_pool_t *temp_pool)

入力と出力

  • **入力**:
  • from: 検索対象となるリスナー連結リストのポインタ。
  • to: 一致したリスナーを移動させる先のリストのポインタ（NULL指定可）。
  • addr, port, scope_id: 検索条件となるネットワーク情報。
  • temp_pool: 比較処理に使用する一時的なメモリプール。
  • **出力**:
  • 戻り値: 一致するリスナーが見つかった場合は 1、それ以外は 0。
  • 副作用: to が指定されている場合、一致したリスナーは from リストから取り除かれ、to リストの先頭に挿入される。

前提条件と不変条件

  • from が指すリスト内の各 ap_listen_rec は、bind_addr を保持している場合と保持していない場合がある。
  • bind_addr が NULL のリスナーは「実体を持たない」とみなされ、比較対象から除外される。

アルゴリズムまたは振る舞い

1. from リストを順次走査する。 2. 各ノードの bind_addr が存在する場合、match_address を呼び出し、引数の addr, port, scope_id と比較する。 3. 一致した場合:

  • 戻り値を 1 に設定する。
  • to が NULL であれば、検索を終了する。
  • to が指定されていれば、現在のノードを from リストから切り離し、to リストの先頭に付け替える。

    • 4. 一致しなかった場合、次のノードへ進む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/listen.c:449: ap_listeners を対象に検索を実行。
  • server/listen.c:454: old_listeners から ap_listeners へリスナーを移動。
  • **呼び出し先**:
  • match_address: アドレス情報の比較を実行。

関連データ構造

  • ap_listen_rec: リスナー情報を保持する連結リスト構造体。
  • apr_sockaddr_t: APR のソケットアドレス構造体。

失敗モードと境界条件

  • from リストが空の場合、何もせず 0 を返す。
  • to が NULL の場合、リストの移動は行われず、存在確認のみが行われる。
  • bind_addr が NULL のノードはスキップされる。

保守担当者向けメモ

  • 本関数はリスナーの再利用やリスト間の移動を制御する重要なユーティリティである。
  • match_address の比較ロジックに依存しているため、アドレス解決や比較仕様の変更時には注意が必要である。
