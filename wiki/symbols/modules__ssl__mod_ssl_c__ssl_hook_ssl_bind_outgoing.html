ssl_hook_ssl_bind_outgoing

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:542

目的

接続に対して SSL/TLS の有効化または無効化を試み、接続コンテキスト (SSLConnRec) をバインドする。

シグネチャまたは形状

static int ssl_hook_ssl_bind_outgoing(conn_rec *c, ap_conf_vector_t *per_dir_config, int enable_ssl)

入力と出力

  • **入力**:
  • c: 接続情報を保持する conn_rec 構造体。
  • per_dir_config: ディレクトリごとの設定ベクトル。
  • enable_ssl: SSL を有効にするか否かを示すフラグ。
  • **出力**:
  • OK: 正常終了（SSL が既にバインド済み、または有効化成功）。
  • DECLINED: SSL の有効化が拒否された、または無効化された。

前提条件と不変条件

  • ssl_init_connection_ctx を呼び出し、接続に関連付けられた SSLConnRec を取得する。
  • 既に sslconn->ssl が存在する場合、再バインドは行わず OK を返す。

アルゴリズムまたは振る舞い

1. ssl_init_connection_ctx を呼び出し、接続コンテキストを取得する。 2. 既に SSL オブジェクトがバインドされている場合は、即座に OK を返す。 3. ssl_engine_status を呼び出し、現在の接続状態を確認する。 4. enable_ssl が真の場合:

  • エンジン状態が OK であれば、sslconn->disabled = 0 とし OK を返す。
  • それ以外の場合は、sslconn->disabled = 1 とし、ログを出力して DECLINED を返す。

    • 5. enable_ssl が偽の場合:

  • sslconn->disabled = 1 とし、DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c:750 にて ap_hook_ssl_bind_outgoing に登録されている。
  • **呼び出し先**:
  • ssl_init_connection_ctx (内部関数)
  • ssl_engine_status (内部関数)
  • ap_log_cerror (Apache Core API)
  • mySrvConfig (マクロ)

関連データ構造

  • conn_rec: Apache の接続構造体。
  • SSLConnRec: mod_ssl 内部の接続ごとの SSL 状態保持構造体。
  • SSLSrvConfigRec: サーバーごとの SSL 設定構造体。

失敗モードと境界条件

  • ssl_engine_status が OK を返さない場合、SSL は無効化され DECLINED が返される。
  • enable_ssl が偽の場合、明示的に sslconn->disabled が設定され DECLINED となる。

保守担当者向けメモ

  • 本関数は ap_hook_ssl_bind_outgoing フックとして登録されており、プロキシ接続等のアウトバウンド SSL 処理の制御に使用される。
  • sslconn->disabled フラグの管理が、後続のフィルタ処理における SSL 有効性の判定基準となる。
