scoreboard

定義

  • 種別: struct
  • 定義箇所: include/scoreboard.h:161

目的

Apache HTTP Server の実行時状態を保持する共有メモリ領域へのアクセスポイントとして機能します。サーバー全体の統計情報、プロセス単位の管理情報、および各ワーカー（スレッド）の稼働状況を統合的に管理します。

シグネチャまたは形状

typedef struct {
    global_score *global;
    process_score *parent;
    worker_score **servers;
} scoreboard;

入力と出力

  • **入力**: 共有メモリ上に配置された各スコアボード構造体（global_score, process_score, worker_score）へのポインタ。
  • **出力**: サーバーの稼働統計、プロセス状態、およびワーカーの処理状況を指し示す構造体。

前提条件と不変条件

  • 本構造体は共有メモリ（apr_shm）上に配置されることを前提としています。
  • servers は worker_score 構造体へのポインタの配列であり、各ワーカーのステータスを追跡するために使用されます。
  • 共有メモリ上のデータであるため、アクセス時には適切な同期制御が必要です。

アルゴリズムまたは振る舞い

1. **初期化**: ap_create_scoreboard および ap_init_scoreboard を通じて共有メモリが確保・初期化される際に、本構造体の各ポインタが設定されます。 2. **状態更新**: MPM やプロトコルモジュールが ap_update_child_status 等の API を介して、本構造体が保持するポインタ経由で worker_score 等の値を更新します。 3. **参照**: mod_status 等の監視モジュールが、本構造体を通じて共有メモリ上の統計情報を読み取り、外部へ出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: MPM（Multi-Processing Module）および mod_status 等の監視モジュール。
  • **呼び出し先**: global_score, process_score, worker_score 構造体。
  • 注: 具体的な関数レベルの呼び出し元・呼び出し先関係については、提供されたソース excerpt には記載がありません。

関連データ構造

  • global_score: サーバー全体の稼働統計を保持。
  • process_score: プロセス単位の管理情報（PID、世代など）を保持。
  • worker_score: 個別のスレッド/ワーカーの統計（アクセス数、ステータスなど）を保持。

失敗モードと境界条件

  • **メモリ競合**: 共有メモリへの同時アクセスが発生するため、同期制御が不十分な場合、データ不整合が生じる可能性があります。
  • **ポインタの無効化**: サーバーの再起動や世代交代時に、共有メモリの再確保が行われるため、古いポインタを参照し続けるとクラッシュの原因となります。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、ポインタの扱いに注意してください。
  • worker_score 内の文字列フィールドはサイズが固定されており、オーバーフローを防ぐ設計となっています。
  • include/ap_mmn.h に記載がある通り、本構造体の変更は Module Magic Number (MMN) の更新を伴うバイナリ互換性に影響する変更となります。
