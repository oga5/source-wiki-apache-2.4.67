ap_get_protocol_upgrades

定義

  • 種別: function
  • 定義箇所: server/protocol.c:2403

目的

現在の接続において、現在使用中のプロトコルからアップグレード可能なプロトコルのリストを取得します。

シグネチャまたは形状

apr_status_t ap_get_protocol_upgrades(conn_rec *c, request_rec *r, 
                                      server_rec *s, int report_all, 
                                      const apr_array_header_t **pupgrades)

入力と出力

  • **入力**:
  • c: 現在の接続構造体。
  • r: 現在のリクエスト構造体（NULL可）。
  • s: サーバー設定構造体（NULLの場合、r または c から取得）。
  • report_all: 0以外の場合、現在使用中のプロトコルも含めて全候補を報告する。
  • **出力**:
  • pupgrades: アップグレード可能なプロトコル名の配列 (const char *) へのポインタ。候補がない場合は NULL が設定される。
  • 戻り値: APR_SUCCESS を返す。

前提条件と不変条件

  • r または c が有効であること。
  • サーバー設定 (core_server_config) にプロトコルリストが定義されていること。

アルゴリズムまたは振る舞い

1. サーバー設定からプロトコルリストを取得します。 2. リストが空でない場合、現在使用中のプロトコル (ap_get_protocol(c)) を特定します。 3. report_all が偽の場合、現在使用中のプロトコル以外の候補を収集します。 4. report_all が真の場合、設定されている全プロトコルを収集します。 5. 収集したプロトコル名を pool から確保した配列に格納し、pupgrades に出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core.c:5622 (確認済み)
  • **呼び出し先**:
  • ap_get_core_module_config
  • ap_get_protocol
  • ap_array_str_contains
  • apr_array_make
  • APR_ARRAY_IDX
  • APR_ARRAY_PUSH

関連データ構造

  • conn_rec: 接続状態を保持。
  • request_rec: リクエスト状態を保持。
  • core_server_config: サーバーのプロトコル設定を保持。
  • apr_array_header_t: プロトコル名のリストを格納する動的配列。

失敗モードと境界条件

  • プロトコル設定が空の場合、pupgrades は NULL となります。
  • r が NULL の場合、メモリ確保には c->pool が使用されます。

保守担当者向けメモ

  • 本関数はプロトコルアップグレードの候補リストを生成するのみであり、実際のアップグレード処理（426 Upgrade レスポンスの生成など）は呼び出し側で行う必要があります。
