fetch_dbm_value

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_dbm.c:80

目的

指定されたDBMファイルから、特定のユーザー名（キー）に対応する値を読み出し、リクエストプールに複製して取得する。

シグネチャまたは形状

apr_status_t fetch_dbm_value(request_rec *r, const char *dbmtype,
                             const char *dbmfile,
                             const char *user, char **value)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • dbmtype: 使用するDBMドライバーの型（例: "SDBM"）。
  • dbmfile: DBMファイルのパス。
  • user: 検索対象のユーザー名（キー）。
  • **出力**:
  • value: 検索結果の文字列を格納するポインタ。見つからない場合は NULL が設定される。
  • 戻り値: apr_status_t 型。処理の成功・失敗を示す。

前提条件と不変条件

  • r->pool は有効なメモリプールであること。
  • dbmtype および dbmfile は apr_dbm が認識可能な形式であること。
  • 戻り値が APR_SUCCESS であっても、*value が NULL の場合は「キーが存在しなかった」ことを意味する。

アルゴリズムまたは振る舞い

1. APR-utilのバージョンに応じて、apr_dbm_get_driver または apr_dbm_open_ex を使用してDBMファイルを開く。 2. ユーザー名をキーとして apr_dbm_fetch を実行する。 3. NETSCAPE_DBM_COMPAT 定義の有無により、キーのサイズ計算（strlen）を調整する。 4. 値が見つかった場合、apr_pstrmemdup を用いてリクエストプール上に値を複製する。 5. apr_dbm_close でファイルを閉じる。 6. 検索失敗（NOT FOUND）はエラーとみなさず、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authn_dbm.c:147
  • modules/aaa/mod_authn_dbm.c:181
  • **呼び出し先**:
  • apr_dbm_get_driver / apr_dbm_open_ex
  • apr_dbm_fetch
  • apr_dbm_close
  • apr_pstrmemdup
  • ap_log_rerror

関連データ構造

  • apr_dbm_t: DBMファイルハンドル。
  • apr_datum_t: DBMのキーおよび値を保持する構造体。
  • request_rec: Apacheのリクエスト情報。

失敗モードと境界条件

  • DBMファイルのオープンに失敗した場合、ap_log_rerror でログを出力し、そのエラーコードを返す。
  • 指定されたキーが存在しない場合、*value に NULL を設定し、エラーなし（APR_SUCCESS）として終了する。

保守担当者向けメモ

  • APU_MAJOR_VERSION 等の条件分岐により、APR-utilのバージョン差異を吸収している。
  • 認証情報の検索において、NOT FOUNDは正常な制御フローの一部として扱われるため、呼び出し側で *value が NULL かどうかの判定が必要である。
