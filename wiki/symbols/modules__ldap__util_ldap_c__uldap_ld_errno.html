uldap_ld_errno

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:532

目的

LDAP 接続オブジェクトから、直近の操作で発生したエラーコードを取得する。LDAP SDK の差異を抽象化し、プラットフォーム固有のオプションを使用してエラー番号を抽出する。

シグネチャまたは形状

static int uldap_ld_errno(util_ldap_connection_t *ldc)

入力と出力

  • **入力**: util_ldap_connection_t *ldc （LDAP 接続状態を保持する構造体へのポインタ）
  • **出力**: int 型の LDAP エラーコード

前提条件と不変条件

  • 引数 ldc は有効な接続構造体を指している必要がある。
  • ldc->ldap が初期化済みであることを前提とする。

アルゴリズムまたは振る舞い

1. LDAP_OPT_ERROR_NUMBER が定義されている場合、ldap_get_option を使用してエラーコードの取得を試みる。成功すればその値を返す。 2. 上記が未定義または失敗した場合、LDAP_OPT_RESULT_CODE が定義されていれば同様に取得を試みる。 3. いずれの手段でも取得できない場合、デフォルト値として LDAP_OTHER を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap.c:559
  • modules/ldap/util_ldap.c:565
  • modules/ldap/util_ldap.c:573
  • **呼び出し先**:
  • ldap_get_option (LDAP SDK 提供の関数)

関連データ構造

  • util_ldap_connection_t: LDAP 接続情報を管理する構造体。

失敗モードと境界条件

  • ldap_get_option が LDAP_SUCCESS を返さない場合、次の条件分岐へ進む。
  • すべての取得手段が失敗した場合は LDAP_OTHER を返すため、呼び出し側は常に有効なエラーコードを受け取る。

保守担当者向けメモ

  • 本関数は SDK 間の API 差異を吸収するためのラッパーである。新しい LDAP SDK をサポートする際は、エラー取得オプションの定義を確認し、必要に応じてプリプロセッサ分岐を追加すること。
