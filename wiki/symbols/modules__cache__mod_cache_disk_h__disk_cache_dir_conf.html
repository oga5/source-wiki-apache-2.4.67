disk_cache_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache_disk.h:79

目的

mod_cache_disk モジュールにおいて、ディレクトリ単位で適用されるキャッシュ設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_off_t minfs;             /* キャッシュ対象ファイルの最小サイズ */
    apr_off_t maxfs;             /* キャッシュ対象ファイルの最大サイズ */
    apr_off_t readsize;          /* 一括キャッシュ試行時の最大データサイズ */
    apr_time_t readtime;         /* 一括キャッシュ試行時の最大処理時間 */
    unsigned int minfs_set:1;    /* minfs 設定フラグ */
    unsigned int maxfs_set:1;    /* maxfs 設定フラグ */
    unsigned int readsize_set:1; /* readsize 設定フラグ */
    unsigned int readtime_set:1; /* readtime 設定フラグ */
} disk_cache_dir_conf;

入力と出力

本構造体は設定値のコンテナであり、直接的な関数呼び出しの入力や出力には使用されません。設定値は mod_cache_disk.c 内で ap_get_module_config を通じて取得され、キャッシュの保存可否やI/O制限の判断に使用されます。

前提条件と不変条件

  • minfs_set 等のビットフィールドは、対応する設定値が明示的に構成されたかどうかを追跡するために使用されます。
  • apr_off_t および apr_time_t 型を使用しており、APR（Apache Portable Runtime）の環境に依存します。

アルゴリズムまたは振る舞い

  • ディレクトリごとの設定（CacheDirLevels や CacheDirLength 等に関連する設定）を管理します。
  • mod_cache_disk.c において、リクエスト処理時に ap_get_module_config(r->per_dir_config, &cache_disk_module) を介して現在のディレクトリ設定を取得するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache_disk.c 内の各関数（設定の取得、マージ、初期化処理など）。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数を呼び出す側ではありません。

関連データ構造

  • disk_cache_conf: モジュール全体の設定を保持する構造体。
  • disk_cache_object_t: キャッシュされたリソースのメタデータを保持する構造体。

失敗モードと境界条件

  • 設定値が未設定（_set フラグが 0）の場合、モジュールはデフォルト値を使用するロジックを別途持つ必要があります。
  • apr_pcalloc によるメモリ確保が失敗した場合、プロセスは異常終了する可能性があります。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を管理するため、mod_cache_disk.c 内の create_dir_config や merge_dir_config 関数と密接に関連しています。変更を加える際は、設定の継承ロジックに影響がないか確認してください。
