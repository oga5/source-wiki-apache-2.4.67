config_log_state

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_log_config.c:262

目的

単一のログファイルに関する設定状態（ファイルパス、フォーマット、条件式など）を保持する。

シグネチャまたは形状

typedef struct {
    const char *fname;
    const char *format_string;
    apr_array_header_t *format;
    void *log_writer;
    char *condition_var;
    int inherit;
    ap_expr_info_t *condition_expr;
    const ap_directive_t *directive;
} config_log_state;

入力と出力

  • 入力: CustomLog や TransferLog ディレクティブの設定値。
  • 出力: ログ記録処理時に config_log_transaction へ渡され、ログの書き込み制御に使用される。

前提条件と不変条件

  • format フィールドは、parse_log_string によって解析された log_format_item の配列を保持する。
  • log_writer は、open_logs フック実行前に設定されている必要がある。

アルゴリズムまたは振る舞い

  • リクエスト終了時の log_transaction フックにおいて、この構造体に格納された condition_expr が評価される。
  • 条件が真であれば、format に基づいてリクエスト情報を文字列化し、log_writer を介して出力する。

呼び出し元と呼び出し先

  • 呼び出し元: multi_log_transaction（config_log_transaction を呼び出す）、および設定解析時の add_custom_log 等。
  • 呼び出し先: ap_expr_exec（条件評価用）、および log_writer 関数ポインタ。

関連データ構造

  • multi_log_state: サーバー単位で複数の config_log_state を管理する配列を含む。
  • log_format_item: format 配列の各要素。
  • ap_expr_info_t: condition_expr で使用される式評価情報。

失敗モードと境界条件

  • log_writer が未設定の場合、ログ出力は行われない。
  • condition_expr の評価に失敗した場合、ログ記録はスキップされる可能性がある。

保守担当者向けメモ

  • 本構造体は request_rec のプールに関連付けられて管理されることが一般的である。
  • directive フィールドは定義箇所を指し、設定チェックが完了すると NULL になる場合がある。
