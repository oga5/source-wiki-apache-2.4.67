resp_data_cb

定義

  • 種別: function
  • 定義箇所: modules/md/md_curl.c:106

目的

libcurl から受信した HTTP レスポンスボディのデータを、apr_bucket_brigade を介して内部のレスポンス構造体に書き込むためのコールバック関数です。

シグネチャまたは形状

size_t resp_data_cb(void *data, size_t len, size_t nmemb, void *baton)
  • data: 受信したデータブロックへのポインタ。
  • len: データ要素のサイズ。
  • nmemb: データ要素の数。
  • baton: md_curl_internals_t 構造体へのポインタ。

入力と出力

  • **入力**: libcurl から渡されるデータチャンクおよび md_curl_internals_t コンテキスト。
  • **出力**: 正常に処理された場合は書き込んだバイト数（len * nmemb）、エラーまたは制限超過時は 0 を返します。

前提条件と不変条件

  • baton は md_curl_internals_t 型としてキャスト可能であること。
  • internals->response および res->body が初期化されていること。

アルゴリズムまたは振る舞い

1. 受信データサイズ（blen = len * nmemb）を算出します。 2. res->req->resp_limit が設定されている場合、現在の res->body の長さに blen を加算し、制限を超えていないか確認します。制限を超える場合は 0 を返し、libcurl に処理を中断させます。 3. apr_brigade_write を使用して、受信データを res->body に書き込みます。 4. 書き込みに失敗した場合は 0 を返し、成功した場合は blen を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: libcurl の内部イベントループ（curl_easy_setopt で CURLOPT_WRITEFUNCTION に登録）。
  • **呼び出し先**: apr_brigade_length, apr_brigade_write（APR ライブラリ関数）。

関連データ構造

  • md_curl_internals_t: リクエストのコンテキストを保持。
  • md_http_response_t: レスポンスデータとボディバッファ（apr_bucket_brigade）を保持。

失敗モードと境界条件

  • **レスポンスサイズ制限**: resp_limit を超えるデータを受信した場合、0 を返して転送を停止します。
  • **書き込みエラー**: apr_brigade_write が APR_SUCCESS 以外を返した場合、0 を返して転送を停止します。

保守担当者向けメモ

  • 本関数は libcurl の書き込みコールバックとして動作するため、戻り値が len * nmemb と一致しない場合、libcurl はエラーとしてリクエストを終了させます。
  • resp_limit のチェックは、メモリ消費を抑えるための重要な保護機能です。
