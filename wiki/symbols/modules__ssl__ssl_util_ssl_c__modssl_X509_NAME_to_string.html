modssl_X509_NAME_to_string

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ssl.c:235

目的

OpenSSL の X509_NAME 構造体を、RFC 2253 形式の文字列に変換して返します。Apache のメモリ管理（apr_pool_t）下で文字列を生成し、呼び出し側がメモリ解放を意識せずに利用できるようにします。

シグネチャまたは形状

char *modssl_X509_NAME_to_string(apr_pool_t *p, X509_NAME *dn, int maxlen)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • dn: 変換対象の X509_NAME 構造体。
  • maxlen: 文字列の最大長。0 以下の場合は制限なし。
  • **出力**:
  • 変換された文字列（char *）。失敗した場合は NULL を返します。

前提条件と不変条件

  • dn は有効な X509_NAME 構造体である必要があります。
  • 生成される文字列は apr_pool_t に紐付くため、プールが破棄されるまで有効です。
  • maxlen が指定された場合、出力文字列は maxlen を超えないように切り詰められます。

アルゴリズムまたは振る舞い

1. メモリバッファとして BIO を作成します。 2. X509_NAME_print_ex を使用し、XN_FLAG_RFC2253 フラグで BIO に名前情報を書き出します。 3. BIO_pending で書き込まれたデータ長を取得します。 4. maxlen の指定に従い、apr_palloc で必要なメモリを確保します。 5. maxlen が指定されており、かつデータ長が maxlen を超える場合、切り詰めた上で末尾に ... を付与します。 6. 最後に BIO_free で一時バッファを解放し、結果の文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_log.c: 証明書情報のログ出力時に利用。
  • modules/ssl/ssl_engine_vars.c: SSL 環境変数の生成時に利用。
  • **呼び出し先**:
  • OpenSSL API: BIO_new, X509_NAME_print_ex, BIO_pending, BIO_read, BIO_free。
  • APR API: apr_palloc, apr_snprintf。

関連データ構造

  • X509_NAME: OpenSSL の証明書識別名構造体。
  • apr_pool_t: Apache のメモリ管理用プール。
  • BIO: OpenSSL の抽象 I/O 構造体。

失敗モードと境界条件

  • BIO_new が失敗した場合、NULL を返します。
  • maxlen が 0 以下の場合、文字列長に制限を設けず全データをコピーします。
  • maxlen が 3 未満の場合、切り詰め時の ... 付与は行われません。

保守担当者向けメモ

  • 本関数は ssl_util_ssl.h で宣言されており、モジュール内の広範なログ出力や変数解決で使用されています。
  • OpenSSL の X509_NAME_print_ex に依存しているため、OpenSSL のバージョンアップに伴う出力形式の変更に注意してください。
