dav_propdb

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1691

目的

WebDAV プロパティデータベースへのアクセスおよび操作を管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct dav_propdb dav_propdb;

入力と出力

  • 入力: プロパティ操作の開始時に、リソースやプール情報とともに初期化されます。
  • 出力: dav_close_propdb(dav_propdb *db) を通じてリソースの解放が行われます。

前提条件と不変条件

  • プロパティ操作のライフサイクル（validate/exec/commit/rollback）において、一貫したコンテキストとして保持される必要があります。
  • dav_close_propdb による明示的なクローズが必要です。

アルゴリズムまたは振る舞い

  • mod_dav のプロパティ管理ロジック（props.c）において、ライブプロパティの検索やプロバイダーの特定に使用されます。
  • プロパティ操作のトランザクション的な処理において、バックエンドのプロパティデータベースとの橋渡しを行います。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/dav/main/mod_dav.c (リクエスト処理フロー内)
  • modules/dav/main/props.c (ライブプロパティ検索時)
  • 呼び出し先:
  • dav_close_propdb (終了処理)
  • その他、dav_hooks_propdb で定義されたプロバイダー関数群（具体的な呼び出し先はプロバイダーの実装に依存します）。

関連データ構造

  • dav_hooks_propdb: プロパティデータベース操作のためのフックテーブル。
  • dav_prop_ctx: プロパティ変更の3フェーズ処理用コンテキスト。

失敗モードと境界条件

  • プロバイダーが未定義の場合や、データベースへの接続が確立できない場合にエラーが発生します。
  • 構造体の初期化に失敗した場合、後続のプロパティ操作は実行できません。

保守担当者向けメモ

  • 本構造体は不透明ポインタとして扱われることが多く、具体的な内部フィールドは mod_dav.c や props.c の実装詳細に依存します。
  • メモリ管理は Apache の apr_pool_t に依存しているため、関連するリクエストのライフサイクルを考慮してください。
