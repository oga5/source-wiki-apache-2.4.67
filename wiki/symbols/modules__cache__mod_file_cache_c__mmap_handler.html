mmap_handler

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_file_cache.c:270

目的

キャッシュされたメモリマップ（mmap）領域を apr_bucket_brigade に格納し、出力フィルタチェーンを通じてクライアントへ送信する。

シグネチャまたは形状

static int mmap_handler(request_rec *r, a_file *file)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • file: キャッシュされたファイル情報を含む a_file 構造体。
  • **出力**:
  • 成功時は OK を返す。
  • ap_pass_brigade が失敗した場合は AP_FILTER_ERROR を返す。

前提条件と不変条件

  • APR_HAS_MMAP が定義されている環境でのみ有効。
  • file->mm に有効なメモリマップオブジェクトが保持されていること。

アルゴリズムまたは振る舞い

1. リクエストのプールから apr_bucket_brigade を作成する。 2. file->mm をリクエストプール用に複製（apr_mmap_dup）する。 3. 複製したメモリマップから apr_bucket_mmap_create を用いてバケットを作成し、ブリゲードの末尾に挿入する。 4. ストリームの終端を示す EOS（End of Stream）バケットを作成し、ブリゲードの末尾に挿入する。 5. ap_pass_brigade を呼び出し、構築したブリゲードを出力フィルタへ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_file_cache.c:375 (file_cache_handler 内)
  • **呼び出し先**:
  • apr_brigade_create
  • apr_mmap_dup
  • apr_bucket_mmap_create
  • APR_BRIGADE_INSERT_TAIL
  • apr_bucket_eos_create
  • ap_pass_brigade

関連データ構造

  • request_rec: リクエスト情報。
  • a_file: キャッシュされたファイルメタデータおよびメモリマップハンドルを保持する構造体。
  • apr_bucket_brigade: 送信データのバケット列。
  • apr_mmap_t: メモリマップオブジェクト。

失敗モードと境界条件

  • APR_HAS_MMAP が無効な場合、本関数は何も行わず OK を返す。
  • ap_pass_brigade が APR_SUCCESS 以外の値を返した場合、AP_FILTER_ERROR を返して処理を中断する。

保守担当者向けメモ

  • 本関数は mod_file_cache のメモリマップ送信専用のハンドラである。
  • ファイルハンドルベースの送信については sendfile_handler が別途担当する。
  • apr_mmap_dup を使用しているため、リクエスト終了時にメモリマップの複製が適切にクリーンアップされることを前提としている。
