proxy_fdpass_canon

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fdpass.c:31

目的

fd:// スキームを持つ URL を識別し、プロキシ処理のために内部的なファイル名形式へ正規化します。

シグネチャまたは形状

static int proxy_fdpass_canon(request_rec *r, char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • url: 正規化対象の URL 文字列。
  • **出力**:
  • 成功時: OK を返し、r->filename を proxy:fd://<path> 形式に更新。
  • スキーム不一致時: DECLINED を返す。

前提条件と不変条件

  • 入力された url が fd:// で始まる場合のみ処理を継続します。
  • ap_server_root_relative を使用して、パスをサーバーのルートディレクトリからの相対パスとして解決します。

アルゴリズムまたは振る舞い

1. url が fd:// で始まるか確認し、一致しない場合は DECLINED を返して終了します。 2. fd:// プレフィックスを除去します。 3. ap_server_root_relative を呼び出し、残りのパスをサーバーのルートディレクトリを基準とした絶対パスに変換します。 4. 変換後のパスを proxy:fd:// というプレフィックスと結合し、r->filename に格納します。 5. 処理結果をデバッグログに出力し、OK を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_fdpass.c:230 にて proxy_hook_canon_handler に登録されています。
  • **呼び出し先**:
  • ap_cstr_casecmpn (Apache API)
  • ap_server_root_relative (Apache API)
  • apr_pstrcat (APR API)
  • ap_log_rerror (Apache API)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • fd:// 以外のスキームが渡された場合、DECLINED を返し、他のハンドラへ処理を委譲します。
  • パス解決に失敗した場合の挙動については、ap_server_root_relative の仕様に依存します。

保守担当者向けメモ

  • 本関数は mod_proxy の正規化フックとして機能します。URL の形式変更やパス解決のロジックを変更する際は、proxy_fdpass_handler が期待する r->filename の形式と整合性が取れているか確認してください。
