socache_dbm_iterate

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:661

目的

DBM ファイルに格納されたキャッシュエントリを全件走査し、期限切れでないエントリに対して指定されたコールバック関数を実行します。

シグネチャまたは形状

apr_status_t socache_dbm_iterate(ap_socache_instance_t *ctx,
                                 server_rec *s, void *userctx,
                                 ap_socache_iterator_t *iterator,
                                 apr_pool_t *pool)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンスのコンテキスト（データファイルパス等を含む）。
  • s: サーバーレコード。
  • userctx: コールバック関数に渡されるユーザー定義コンテキスト。
  • iterator: 走査時に実行されるコールバック関数。
  • pool: メモリ割り当て用プール。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS を返し、エラー発生時はエラーコードを返します。

前提条件と不変条件

  • ctx->data_file が有効な DBM ファイルパスを指していること。
  • DBM ファイル内の各値は、先頭に apr_time_t 型の期限情報が格納されていること。
  • 本関数はスレッドセーフではありません。呼び出し側で適切な排他制御が必要です。

アルゴリズムまたは振る舞い

1. 現在時刻を取得します。 2. apr_dbm_open（または apr_dbm_open2）を使用して DBM ファイルを読み取り専用で開きます。 3. apr_dbm_firstkey で最初のキーを取得し、apr_dbm_nextkey を用いて全エントリをループ処理します。 4. 各エントリに対し、値の先頭から apr_time_t を読み取り、現在時刻と比較します。 5. 期限切れでない場合のみ、iterator コールバックを呼び出します。 6. 最後に apr_dbm_close でファイルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は modules/cache/mod_socache_dbm.c:750 です。
  • **呼び出し先**: apr_dbm 系関数（open, firstkey, fetch, nextkey, close）、iterator コールバック関数、ap_log_error。

関連データ構造

  • ap_socache_instance_t: キャッシュのインスタンス情報。
  • apr_dbm_t: DBM ファイルのハンドル。
  • apr_datum_t: DBM のキーおよび値を保持する構造体。

失敗モードと境界条件

  • DBM ファイルのオープン失敗: APR_SUCCESS 以外のステータスを返し、エラーログを出力します。
  • 値のサイズが sizeof(apr_time_t) 以下の場合: 期限情報が不正とみなされ、そのエントリはスキップ（期限切れ扱い）されます。
  • コールバック関数がエラーを返した場合: 走査を中断し、そのエラーコードを返します。

保守担当者向けメモ

  • 本関数は AP_SOCACHE_FLAG_NOTMPSAFE を前提としており、並行アクセス時の整合性は呼び出し側の排他制御に依存します。
  • DBM ライブラリのバージョン（APU 1.7 以上か否か）により、ドライバのロード方法が分岐します。
