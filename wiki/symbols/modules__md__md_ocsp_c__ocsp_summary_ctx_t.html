ocsp_summary_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_ocsp.c:950

目的

OCSP ステータスの集計処理において、状態のカウントとコンテキスト情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    apr_pool_t *p;
    md_ocsp_reg_t *reg;
    int good;
    int revoked;
    int unknown;
} ocsp_summary_ctx_t;

入力と出力

  • **入力**: 処理対象のメモリプール (p) および OCSP レジストリ (reg) への参照。
  • **出力**: good（正常）、revoked（失効）、unknown（不明）の各状態の集計値。

前提条件と不変条件

  • p は有効な apr_pool_t である必要があります。
  • reg は初期化済みの md_ocsp_reg_t を指している必要があります。

アルゴリズムまたは振る舞い

OCSP ステータスの集計処理中に、各証明書のステータスを走査し、対応するカウンタ（good, revoked, unknown）をインクリメントするために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_ocsp.c 内の集計ロジックにて、ローカル変数としてインスタンス化され、バトン（baton）として渡されることが確認されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • md_ocsp_reg_t: OCSP 処理のグローバルコンテキスト。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • 構造体内のポインタが NULL の場合、参照時にセグメンテーションフォールトが発生する可能性があります。
  • カウンタ変数は初期化されていない場合、不定値を持つため、使用前にゼロクリアが必要です。

保守担当者向けメモ

  • 本構造体は md_ocsp.c 内の集計処理専用です。集計ロジックの変更に伴い、カウンタの定義や種類を追加・修正する際は、この構造体のメンバを更新してください。
