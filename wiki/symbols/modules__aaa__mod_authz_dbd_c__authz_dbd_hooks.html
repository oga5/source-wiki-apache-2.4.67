authz_dbd_hooks

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbd.c:384

目的

Apache HTTP Server の認可プロバイダーインターフェースに対し、データベース（DBD）を利用した認可機能（グループ確認、ログイン、ログアウト）を登録する。

シグネチャまたは形状

void authz_dbd_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（プロバイダー登録に使用するメモリプール）
  • **出力**: なし（副作用としてプロバイダーが登録される）

前提条件と不変条件

  • ap_register_auth_provider を呼び出すため、Apache の認可フレームワークが初期化されている必要がある。
  • 登録される各プロバイダー（dbd-group, dbd-login, dbd-logout）に対応するプロバイダー構造体が定義されていること。

アルゴリズムまたは振る舞い

1. ap_register_auth_provider を使用して、以下の 3 つの認可プロバイダーを AUTHZ_PROVIDER_GROUP カテゴリに登録する。

  • dbd-group: authz_dbdgroup_provider を紐付け。
  • dbd-login: authz_dbdlogin_provider を紐付け。
  • dbd-logout: authz_dbdlogout_provider を紐付け。

    • 2. 各登録において、バージョン AUTHZ_PROVIDER_VERSION を指定し、AP_AUTH_INTERNAL_PER_CONF フラグを付与することで、設定単位での内部認可処理として登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbd.c:408（モジュールの登録フック内で呼び出される）
  • **呼び出し先**: ap_register_auth_provider（Apache HTTP Server のコア API）

関連データ構造

  • authz_dbdgroup_provider
  • authz_dbdlogin_provider
  • authz_dbdlogout_provider
  • apr_pool_t

失敗モードと境界条件

  • メモリ不足により apr_pool_t が確保できない場合、Apache の標準的なエラーハンドリングに従いプロセスが異常終了する可能性がある。
  • 既に同一名のプロバイダーが登録されている場合、登録が失敗または上書きされる可能性がある。

保守担当者向けメモ

  • 本関数はモジュールの初期化時に一度だけ実行される。
  • 新たな認可機能を追加する場合は、本関数内で ap_register_auth_provider を追加し、対応するプロバイダー構造体を定義する必要がある。
