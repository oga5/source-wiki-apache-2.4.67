array_contents_t

定義

  • 種別: struct
  • 定義箇所: modules/core/mod_macro.c:550

目的

マクロ展開後の設定内容を保持し、Apache の設定パーサーが読み取れる形式（ap_configfile_t 互換）として提供するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    int index;                    /* 現在処理中の要素インデックス */
    int char_index;               /* 現在の要素内の文字位置 */
    int length;                   /* 現在の行のキャッシュされた長さ */
    apr_array_header_t *contents; /* char * の配列（マクロ展開後の行データ） */
    ap_configfile_t *next;        /* この内容の処理完了後に続く設定ファイル */
    ap_configfile_t **upper;      /* 必要に応じて更新するための上位設定ファイルへのポインタ */
} array_contents_t;

入力と出力

  • 入力: マクロ展開処理によって生成された文字列配列（contents）を受け取ります。
  • 出力: ap_configfile_t としてラップされ、設定パーサーに対してマクロ展開後の内容を「元の設定ファイルの一部」として供給します。

前提条件と不変条件

  • contents は apr_array_header_t 型であり、各要素は char * である必要があります。
  • next は、本構造体による処理が終了した後にパーサーが読み込むべき次の設定ストリームを指す必要があります。

アルゴリズムまたは振る舞い

1. make_array_config 関数によって初期化され、ap_configfile_t の読み込み用コールバック（next_one 等）から参照されます。 2. パーサーが設定行を要求するたびに、index と char_index を進めながら contents 配列から文字列を順次提供します。 3. すべての要素の処理が完了すると、next に保持された次の設定ストリームへ制御を移します。

呼び出し元と呼び出し先

  • 呼び出し元: make_array_config（初期化）、next_one、および mod_macro.c 内の各設定処理関数が param として受け取ります。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。パーサー側のコールバック関数が本構造体のメンバを操作します。

関連データ構造

  • ap_configfile_t: Apache の設定ファイル読み込み用抽象構造体。
  • apr_array_header_t: APR の動的配列。マクロ展開後の各行を保持するために使用されます。

失敗モードと境界条件

  • contents が空の場合、あるいは index が配列の範囲外に達した場合の挙動は、パーサー側の読み込みロジックに依存します。
  • upper ポインタの更新が適切に行われない場合、設定ファイルの読み込みチェーンが断絶する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_macro における設定ストリーム注入の要です。メモリ管理は apr_palloc を使用しているため、ライフサイクルは関連する pool に依存します。
  • upper メンバはハック的な実装であるため、設定パーサーの構造変更時には特に注意して検証してください。
