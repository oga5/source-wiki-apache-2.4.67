mod_filter_chain

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_filter.c:62

目的

設定されたフィルタ名を保持し、それらを連結リストとして管理するためのデータ構造です。FilterChain ディレクティブ等で指定されたフィルタの適用順序を定義します。

シグネチャまたは形状

typedef struct mod_filter_chain {
    const char *fname;
    struct mod_filter_chain *next;
} mod_filter_chain;

入力と出力

  • fname: フィルタ名を指す文字列ポインタ。
  • next: リスト内の次の要素を指すポインタ。

前提条件と不変条件

  • mod_filter_chain の各インスタンスは、設定フェーズにおいて cmd->pool からメモリが割り当てられます。
  • リストの終端は next が NULL であることで示されます。

アルゴリズムまたは振る舞い

  • フィルタチェーンの構築時に、新しいフィルタ名がリストの末尾または適切な位置に追加されます。
  • 実行時には、このリストを辿ることで適用すべきフィルタの順序を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_filter.c 内の FilterChain ディレクティブ処理関数（506, 518, 543, 550, 552, 558 行目付近）から、設定の構築時に呼び出されます。
  • **呼び出し先**: 情報なし。

関連データ構造

  • mod_filter_cfg: モジュール全体の設定を保持する構造体であり、この中に mod_filter_chain へのポインタが含まれます。

失敗モードと境界条件

  • メモリ割り当て失敗時: apr_pcalloc や apr_palloc が失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本構造体は設定フェーズでのみ使用されます。リクエスト処理中のフィルタ実行時には、このリストから構築された実行用コンテキストが参照されます。
  • フィルタ名の重複や存在しないフィルタの指定については、設定読み込み時に検証される必要があります。
