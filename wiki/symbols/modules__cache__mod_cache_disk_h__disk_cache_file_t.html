disk_cache_file_t

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache_disk.h:28

目的

ディスクベースのキャッシュにおけるファイルI/O操作を抽象化し、実ファイルと書き込み用の一時ファイルを管理するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_pool_t *pool;
    const char *file;
    apr_file_t *fd;
    char *tempfile;
    apr_file_t *tempfd;
} disk_cache_file_t;

入力と出力

  • pool: メモリ割り当てに使用されるAPRプール。
  • file: キャッシュ対象のファイルパス。
  • fd: 読み込みまたは書き込みに使用される実ファイルのAPRファイルハンドル。
  • tempfile: 書き込み処理中に使用される一時ファイルのパス。
  • tempfd: 一時ファイルのAPRファイルハンドル。

前提条件と不変条件

  • fd および tempfd は、使用されていない場合は NULL である必要があります。
  • pool は、この構造体が生成された際に有効なAPRプールが割り当てられている必要があります。

アルゴリズムまたは振る舞い

  • mod_cache_disk において、キャッシュデータの読み書きや一時ファイルから実ファイルへの切り替え（コミット）の際に使用されます。
  • close_disk_cache_fd 関数により、保持しているファイルハンドル（fd, tempfd）のクローズ処理が行われます。
  • file_cache_create や file_cache_el_final などの関数を通じて、キャッシュファイルの生成や最終化処理のコンテキストとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache_disk.c 内の file_cache_el_final、file_cache_create、close_disk_cache_fd など。
  • **呼び出し先**: APRのファイルI/O関数（apr_file_t を操作する関数群）。
  • **補足**: mod_cache_disk.h 内で disk_cache_object_t のメンバ（data, hdrs, vary）として利用されています。

関連データ構造

  • disk_cache_object_t: 本構造体を保持し、キャッシュエントリのメタデータを管理する上位構造体。
  • disk_cache_conf: モジュール設定を保持する構造体。

失敗モードと境界条件

  • ファイルハンドルが NULL の状態でクローズ処理や読み書き操作を行った場合の挙動は、APRのファイルI/O実装に依存します。
  • 一時ファイルから実ファイルへのリネームや移動が失敗した場合、キャッシュの整合性が損なわれる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cache_disk のファイルI/Oの基盤です。ファイルハンドルを扱う際は、必ず pool の生存期間と整合性を確認してください。
  • disk_cache_object_t のメンバとして複数定義されているため、各インスタンスがどのファイル（データ、ヘッダー、Vary）を指しているかを混同しないよう注意してください。
