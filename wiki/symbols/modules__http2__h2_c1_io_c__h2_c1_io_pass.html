h2_c1_io_pass

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:339

目的

HTTP/2 クライアント接続（C1）において、バッファリングされている出力データを接続フィルタチェーンへ書き出します。

シグネチャまたは形状

apr_status_t h2_c1_io_pass(h2_c1_io *io)

入力と出力

  • **入力**: io (h2_c1_io *) - 処理対象の入出力管理構造体。
  • **出力**: apr_status_t - 処理結果（成功時は APR_SUCCESS、失敗時はエラーコード）。

前提条件と不変条件

  • io は有効な h2_c1_io 構造体を指している必要があります。
  • 内部的に is_passing フラグを使用して再帰呼び出しを防止する設計となっています。

アルゴリズムまたは振る舞い

1. h2_c1_io_pending(io) を呼び出し、送信待ちのデータが存在するか確認します。 2. データが存在する場合、pass_output(io, 0) を呼び出してフィルタチェーンへの書き出しを実行します。 3. 処理結果を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:1373
  • modules/http2/h2_stream.c:1463
  • **呼び出し先**:
  • h2_c1_io_pending (判定)
  • pass_output (実行)

関連データ構造

  • h2_c1_io: 接続ごとの入出力バッファおよび状態を管理する構造体。

失敗モードと境界条件

  • 送信待ちデータがない場合、何もせず APR_SUCCESS を返します。
  • pass_output 内でエラーが発生した場合、そのエラーコードがそのまま返されます。
  • 再帰的な呼び出しが発生した場合、pass_output 内部のチェックによりエラーが返される可能性があります（ソース内のログメッセージより推測）。

保守担当者向けメモ

  • 本関数は h2_session や h2_stream からの出力要求をトリガーする主要なインターフェースです。
  • フィルタチェーンへの書き出し処理の詳細については pass_output を参照してください。
