SHMCBIndex

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_shmcb.c:94

目的

共有メモリベースのキャッシュ（SHMCB）において、個別のキャッシュエントリのメタデータおよびデータ配置情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_time_t expires;        /* 有効期限（絶対時間） */
    unsigned int data_pos;     /* サブキャッシュのデータ領域内での位置 */
    unsigned int data_used;    /* 使用データサイズ */
    unsigned int id_len;       /* IDを含む使用データ長 */
    unsigned char removed;     /* 明示的な削除フラグ */
} SHMCBIndex;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。各フィールドは、キャッシュのストア時および検索時に SHMCB_INDEX マクロを介してアクセスされます。

前提条件と不変条件

  • SHMCBIndex の配列は、SHMCBSubcache 構造体の直後に配置されます。
  • data_pos は、対応するサブキャッシュのデータ領域内を指す有効なオフセットである必要があります。
  • removed フィールドは、エントリが論理的に削除されたことを示すために使用されます。

アルゴリズムまたは振る舞い

  • **エントリ管理**: 各 SHMCBIndex インスタンスは、単一のキャッシュオブジェクトを表現します。
  • **メモリ最適化**: data_used フィールドは、メモリコピー処理を最小限に抑えるために保持されています。
  • **期限切れ処理**: expires フィールドに基づき、shmcb_subcache_expire 関数がエントリの有効性を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_socache_shmcb.c 内のキャッシュ操作関数（socache_shmcb_store, socache_shmcb_retrieve 等）が、SHMCB_INDEX マクロを通じて本構造体を参照・更新します。
  • **呼び出し先**: 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • SHMCBSubcache: 本構造体をインデックスとして保持する親構造体。
  • SHMCBHeader: 共有メモリ全体のメタデータを管理する構造体。

失敗モードと境界条件

  • **オーバーフロー**: unsigned int を使用しているため、非常に大きなキャッシュサイズを指定した場合、サイズ計算時にオーバーフローが発生するリスクがあります。
  • **整合性**: 共有メモリ上のデータであるため、プロセス間での適切な同期が行われない場合、インデックスとデータ領域の不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は SHMCB_INDEX マクロによってポインタ演算が行われるため、構造体のメンバ順序やサイズを変更する際は、関連するマクロ定義およびメモリレイアウト計算への影響を十分に確認してください。
  • data_used の保持はメモリコピーの最適化を目的としていますが、ロジック上は必ずしも必須ではないため、将来的なリファクタリングの対象となり得ます。
