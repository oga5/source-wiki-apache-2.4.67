load_file

定義

  • 種別: function
  • 定義箇所: modules/core/mod_so.c:327

目的

Apache HTTP Server の設定ファイルにおいて LoadFile ディレクティブが指定された際、指定された共有ライブラリをプロセス空間に動的にロードする。

シグネチャまたは形状

static const char *load_file(cmd_parms *cmd, void *dummy, const char *filename)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • filename: ロード対象の共有ライブラリのパス。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラー内容を示す文字列（const char *）を返す。

前提条件と不変条件

  • dso_load を介してライブラリをロードするため、プラットフォームが apr_dso による動的ロードをサポートしている必要がある。
  • ロードされたライブラリは Apache モジュールとしての登録は行われない。

アルゴリズムまたは振る舞い

1. dso_load を呼び出し、指定された filename をロードする。 2. dso_load がエラーを返した場合は、そのエラーメッセージをそのまま呼び出し元へ返す。 3. ロードに成功した場合、ap_log_error を使用してデバッグレベルでロード成功のログを出力する。 4. NULL を返して正常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_so.c 内の AP_INIT_ITERATE マクロ定義（LoadFile ディレクティブのハンドラとして登録）。
  • **呼び出し先**: dso_load（共有ライブラリのロード処理）、ap_log_error（ログ出力）。

関連データ構造

  • cmd_parms: 設定解析時のコンテキスト情報を保持。
  • apr_dso_handle_t: ロードされた共有ライブラリのハンドル（dso_load 経由で使用）。

失敗モードと境界条件

  • dso_load が失敗した場合（ファイルが存在しない、権限がない、バイナリ形式が不正など）、そのエラーが伝播される。
  • NO_DLOPEN が定義されている環境では、動的ロード機能が無効化されている可能性がある。

保守担当者向けメモ

  • 本関数はライブラリをロードするのみであり、Apache モジュールとしての初期化や登録は行わない。
  • エラーメッセージは apr_dso_error に依存しており、プラットフォーム固有の文字列が返される。
