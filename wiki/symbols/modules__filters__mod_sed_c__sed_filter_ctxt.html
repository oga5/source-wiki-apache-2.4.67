sed_filter_ctxt

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_sed.c:45

目的

mod_sed フィルタがリクエストまたはレスポンスのストリームを処理する際、その実行状態を保持するために使用されるコンテキスト構造体です。

シグネチャまたは形状

typedef struct sed_filter_ctxt
{
    sed_eval_t eval;             /* libsed の評価エンジン状態 */
    ap_filter_t *f;              /* フィルタインスタンス */
    request_rec *r;              /* リクエストレコード */
    apr_bucket_brigade *bb;      /* 出力用バケットブリゲード */
    apr_bucket_brigade *bbinp;   /* 入力用バケットブリゲード */
    char *outbuf;                /* 出力バッファの先頭ポインタ */
    char *curoutbuf;             /* 出力バッファの現在位置ポインタ */
    apr_size_t bufsize;          /* バッファのサイズ */
    apr_pool_t *tpool;           /* 一時的なバケット生成用プール */
    int numbuckets;              /* 現在のバケット数 */
} sed_filter_ctxt;

入力と出力

  • **入力**: libsed エンジンによる変換対象データおよび Apache のフィルタチェーンから渡される apr_bucket_brigade。
  • **出力**: 変換後のデータを含む apr_bucket_brigade。sed_write_output を通じてバッファリングされ、後続のフィルタへ渡されます。

前提条件と不変条件

  • tpool はメモリリークを防ぐため、定期的に clear_ctxpool によってクリアされる必要があります。
  • sed_request_filter は AP_MODE_READBYTES モードでのみ動作します。
  • sed_finalize_eval 実行時に改行がない場合、libsed の仕様により自動的に改行が付与されます。

アルゴリズムまたは振る舞い

1. **初期化**: リクエストごとに apr_pcalloc で割り当てられ、libsed の評価エンジンが初期化されます。 2. **データ処理**: sed_response_filter 等がバケットを反復し、sed_eval_buffer を呼び出して変換を行います。 3. **バッファリング**: 変換結果は outbuf に蓄積され、MODSED_OUTBUF_SIZE に達するか、EOS/Flush バケットを受信した時点で append_bucket を介してブリゲードへ書き出されます。 4. **終了処理**: EOS バケット受信時に sed_finalize_eval が呼ばれ、内部バッファの残存データがフラッシュされます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_sed.c 内のフィルタ初期化処理（apr_pcalloc による生成）およびフィルタ処理関数（f->ctx からの取得）。
  • **呼び出し先**: libsed の各関数（sed_eval_buffer, sed_finalize_eval 等）、および mod_sed.c 内のユーティリティ関数（clear_ctxpool, alloc_outbuf, append_bucket, flush_output_buffer）。

関連データ構造

  • sed_eval_t: libsed の評価状態を保持。
  • apr_bucket_brigade: Apache のストリームデータ管理構造。
  • sed_config / sed_expr_config: モジュールの設定情報を保持。

失敗モードと境界条件

  • libsed のコンパイルエラーや実行時エラーが発生した場合、log_sed_errf や sed_compile_errf を通じてログ出力が行われます。
  • sed_request_filter において AP_MODE_READBYTES 以外のモードが指定された場合、処理は透過的にパスされます。

保守担当者向けメモ

  • tpool の管理はメモリ効率に直結するため、新しいバケット操作を追加する際は clear_ctxpool のタイミングを考慮してください。
  • libsed の仕様上、POST データ処理時に意図しない改行付与が発生する可能性があるため、データ整合性が重要な場合は注意が必要です。
