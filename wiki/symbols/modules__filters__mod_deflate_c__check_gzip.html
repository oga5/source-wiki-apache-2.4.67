check_gzip

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_deflate.c:107

目的

HTTP リクエストまたはレスポンスの Content-Encoding ヘッダーを解析し、Gzip 圧縮が適用されているかを判定する。Gzip が検出された場合は、該当するヘッダーをテーブルから削除または更新し、圧縮済みであることを示すフラグを返す。

シグネチャまたは形状

int check_gzip(request_rec *r, apr_table_t *hdrs1, apr_table_t *hdrs2)

入力と出力

  • **入力**:
  • r: リクエスト構造体。
  • hdrs1, hdrs2: 検査対象のヘッダーテーブル（hdrs2 はオプション）。
  • **出力**:
  • int: Gzip 圧縮が検出された場合は 1、それ以外は 0 を返す。

前提条件と不変条件

  • hdrs1 または hdrs2 に Content-Encoding ヘッダーが存在する場合、その値を評価する。
  • r->content_encoding は、ヘッダーテーブルの更新後に同期される。

アルゴリズムまたは振る舞い

1. hdrs1 および hdrs2 から Content-Encoding を取得し、優先順位に従って評価対象を決定する。 2. エンコーディング値が "gzip" または "x-gzip" と一致するか確認する。 3. カンマ区切りのリストが含まれる場合、末尾のトークンから順に解析し、"gzip" または "x-gzip" を探す。

  • "identity" トークンは無視してスキップする。
  • Gzip を検出した場合、該当するトークンを削除または更新し、ヘッダーテーブルを修正する。

    • 4. 最後に、r->content_encoding が設定されている場合、修正後のヘッダーテーブルの値と同期させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_deflate.c:1184
  • modules/filters/mod_deflate.c:1574
  • **呼び出し先**:
  • apr_table_get, apr_table_unset, apr_table_setn (APR テーブル操作)
  • ap_cstr_casecmp, ap_strchr_c, ap_strrchr (Apache 文字列操作)
  • apr_pstrdup, apr_isspace (APR ユーティリティ)

関連データ構造

  • request_rec: Apache のリクエスト状態を保持する構造体。
  • apr_table_t: HTTP ヘッダーを保持するテーブル構造体。

失敗モードと境界条件

  • Content-Encoding が存在しない場合、0 を返す。
  • 複数のエンコーディングが指定されている場合、最も外側（右側）の非 identity トークンのみを評価する。
  • ヘッダーテーブルが更新された場合、r->content_encoding も整合性を保つよう更新される。

保守担当者向けメモ

  • 本関数は Content-Encoding の書き換えを行うため、後続の http_header_filter 等でヘッダーが再評価される際に矛盾が生じないよう注意が必要である。
  • hdrs2 が指定されている場合、出力フィルタのコンテキスト（headers_out と err_headers_out）を考慮した二段構えのチェックが行われる。
