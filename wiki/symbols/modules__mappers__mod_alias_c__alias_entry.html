alias_entry

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_alias.c:46

目的

Apache HTTP Server における Alias、ScriptAlias、および Redirect ディレクティブの設定情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    const char *real;
    const char *fake;
    char *handler;
    ap_regex_t *regexp;
    int redir_status;
} alias_entry;

入力と出力

本構造体は設定解析時に生成され、リクエスト処理フェーズで参照されます。

  • real: マッピング先の物理パスまたはリダイレクト先 URL。
  • fake: マッピング元の URI プレフィックスまたは正規表現パターン。
  • handler: 関連付けられたハンドラ名（存在する場合）。
  • regexp: 正規表現マッチング用のコンパイル済みオブジェクト。
  • redir_status: リダイレクト時の HTTP ステータスコード（301, 302, 303, 410 等）。

前提条件と不変条件

  • apr_array_header_t 内の要素として格納され、リクエスト処理時に elts からキャストしてアクセスされます。
  • redir_status はリダイレクト処理において有効な HTTP ステータスコードである必要があります。

アルゴリズムまたは振る舞い

  • 設定フェーズにおいて apr_array_make で確保された配列に格納されます。
  • リクエストの翻訳フェーズ（translate_name）および修正フェーズ（fixups）において、配列を走査し、URI と fake（または regexp）を照合することでマッピングを決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_alias.c 内の add_alias_internal および add_redirect_internal が本構造体のインスタンスを生成・追加します。また、try_alias_list 等の関数が配列内の要素を走査します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • alias_server_conf: サーバー全体の設定を保持し、alias_entry の配列を管理します。
  • alias_dir_conf: ディレクトリごとの設定を保持します。
  • apr_array_header_t: alias_entry を格納するコンテナとして使用されます。

失敗モードと境界条件

  • regexp が設定されている場合、正規表現のコンパイル失敗やマッチングエラーが発生する可能性があります。
  • alias_entry の配列を線形探索するため、定義数が多い場合にパフォーマンスが低下する可能性があります。

保守担当者向けメモ

  • alias_entry は mod_alias.c の設定管理の基盤です。
  • redir_status の値が適切に設定されているか、特に Redirect 系のディレクティブ追加時に注意してください。
  • apr_pool_t によるメモリ管理が行われているため、構造体内の文字列ポインタはリクエスト終了時に自動的に解放されます。
