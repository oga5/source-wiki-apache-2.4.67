dav_options_provider_register

定義

  • 種別: function
  • 定義箇所: modules/dav/main/providers.c:35

目的

WebDAV の OPTIONS メソッドに対する拡張プロバイダーを、Apache のグローバルなプロバイダーレジストリに登録します。

シグネチャまたは形状

void dav_options_provider_register(apr_pool_t *p,
                                   const char *name,
                                   const dav_options_provider *provider)

入力と出力

  • p: プロバイダーの生存期間を管理するためのメモリプール。
  • name: 登録するプロバイダーを一意に識別するための名称。
  • provider: 登録対象となる dav_options_provider 構造体へのポインタ。
  • 戻り値: なし（void）。

前提条件と不変条件

  • provider は mod_dav.h で定義された dav_options_provider 構造体の要件を満たしている必要があります。
  • プロバイダーのバージョンは内部的に "0" として固定登録されます。

アルゴリズムまたは振る舞い

1. Apache の ap_register_provider 関数を呼び出します。 2. グループ識別子として DAV_OPTIONS_EXTENSION_GROUP を使用し、指定された name でプロバイダーをハッシュテーブルに登録します。 3. バージョン番号として "0" を指定します。

呼び出し元と呼び出し先

  • 呼び出し元: 特定のソースコード内での直接的な呼び出し箇所は、提供された情報からは特定できません。
  • 呼び出し先: ap_register_provider (Apache Core API)。

関連データ構造

  • dav_options_provider: OPTIONS メソッドの拡張機能を提供する構造体。
  • DAV_OPTIONS_EXTENSION_GROUP: プロバイダー検索時に使用されるグループ定数。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • ap_register_provider の内部仕様に依存します。同一の name で既にプロバイダーが登録されている場合、上書きまたはエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は ap_provider API の薄いラッパーです。
  • プロバイダーのバージョン管理は現在 "0" に固定されています。将来的にバージョン管理が必要となった場合、この引数の変更を検討してください。
