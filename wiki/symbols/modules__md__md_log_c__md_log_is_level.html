md_log_is_level

定義

  • 種別: function
  • 定義箇所: modules/md/md_log.c:60

目的

指定されたログレベルが現在有効かどうかを判定します。高コストなログメッセージの構築処理（文字列生成など）を実行する前に、そのログ出力が必要かどうかを事前に確認するために使用されます。

シグネチャまたは形状

int md_log_is_level(apr_pool_t *p, md_log_level_t level)

入力と出力

  • **入力**:
  • p: 判定に使用する apr_pool_t 型のメモリプール。
  • level: 判定対象のログレベル（md_log_level_t 型）。
  • **出力**:
  • int: ログレベルが有効な場合は非ゼロ（真）、無効な場合は 0（偽）を返します。

前提条件と不変条件

  • 内部で保持されているログ出力用コールバック関数 log_level が初期化されている必要があります。
  • log_level が設定されていない場合、本関数は常に 0 を返します。

アルゴリズムまたは振る舞い

1. 静的変数 log_level が存在するか確認します。存在しない場合は 0 を返します。 2. 存在する場合、登録済みの log_level コールバック関数を呼び出します。 3. その際、保持されている log_baton（コンテキスト）、引数のメモリプール p、および判定対象の level をコールバックへ渡します。 4. コールバックの戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c
  • modules/md/md_acme_acct.c
  • modules/md/md_acme_authz.c
  • modules/md/md_acme_drive.c
  • modules/md/md_curl.c
  • modules/md/md_reg.c
  • **呼び出し先**:
  • 静的関数ポインタ log_level（md_log_set によって設定されるコールバック）。

関連データ構造

  • md_log_level_t: ログレベルを定義する列挙型。
  • apr_pool_t: APR のメモリプール。
  • log_level: ログレベル判定を委譲する内部的な関数ポインタ。
  • log_baton: ログ出力設定に関連付けられたコンテキストデータ。

失敗モードと境界条件

  • log_level が初期化されていない（NULL である）場合、安全に 0 を返します。
  • 渡された apr_pool_t *p が NULL の場合、コールバック関数の実装に依存しますが、通常は予期せぬ動作を引き起こす可能性があるため注意が必要です。

保守担当者向けメモ

  • 本関数は md_log_perror を呼び出す前のガード条件として広く利用されています。
  • ログ出力のバックエンドを動的に差し替える設計となっているため、判定ロジック自体は log_level コールバックの実装に完全に依存しています。
