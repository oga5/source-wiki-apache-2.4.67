deregister_all_hooks

定義

  • 種別: function
  • 定義箇所: server/main.c:285

目的

サーバーの設定リセット時において、既存のフックを解除し、内部認証状態をクリアするために使用されます。

シグネチャまたは形状

static apr_status_t deregister_all_hooks(void *unused)

入力と出力

  • **入力**: void *unused（apr_pool_pre_cleanup_register のコールバックシグネチャに合わせるための引数。本関数内では使用されません）。
  • **出力**: apr_status_t 型の戻り値。常に APR_SUCCESS を返します。

前提条件と不変条件

  • 本関数は apr_pool_pre_cleanup_register を介してメモリプールのクリーンアップハンドラとして登録されることを前提としています。
  • 実行により、グローバルなフック登録状態および内部認証状態がリセットされます。

アルゴリズムまたは振る舞い

1. 引数 unused を破棄します。 2. ap_clear_auth_internal() を呼び出し、内部認証状態をクリアします。 3. apr_hook_deregister_all() を呼び出し、登録されているすべてのフックを解除します。 4. APR_SUCCESS を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/main.c:303 において、apr_pool_pre_cleanup_register の引数として登録されています。
  • **呼び出し先**:
  • ap_clear_auth_internal()
  • apr_hook_deregister_all()

関連データ構造

  • process->pconf: 本関数がクリーンアップハンドラとして登録される設定用メモリプール。

失敗モードと境界条件

  • 本関数自体はエラーを返さない設計ですが、内部で呼び出される ap_clear_auth_internal および apr_hook_deregister_all の実行結果に依存します。

保守担当者向けメモ

  • 本関数は設定の再読み込み（graceful restart 等）に伴う pconf プールのリセット時に実行されます。フックの再登録が必要な場合は、このクリーンアップ後に別途処理が行われるフローであることを確認してください。
