htdbm_save

定義

  • 種別: function
  • 定義箇所: support/htdbm.c:155

目的

ユーザー名とハッシュ化されたパスワード（およびオプションのコメント）を DBM データベースに永続化する。

シグネチャまたは形状

apr_status_t htdbm_save(htdbm_t *htdbm, int *changed)

入力と出力

  • **入力**:
  • htdbm: データベース接続情報およびユーザー情報を含む htdbm_t 構造体へのポインタ。
  • changed: 既存レコードの更新が発生したかどうかを記録するフラグへのポインタ。
  • **出力**:
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。
  • changed: 既存のユーザー名がデータベース内に存在した場合、1 が設定される。

前提条件と不変条件

  • htdbm->dbm は apr_dbm_open によって既に開かれている必要がある。
  • htdbm->username が NULL の場合、処理は何も行わず APR_SUCCESS を返す。
  • DBM 内のデータは username をキーとし、password[:comment] を値として保存される。

アルゴリズムまたは振る舞い

1. htdbm->username が存在するか確認し、なければ終了する。 2. htdbm->username をキーとして apr_dbm_exists を呼び出し、存在する場合は changed フラグを 1 に設定する。 3. htdbm->ctx.passwd を値のベースとして設定する。 4. htdbm->comment が存在する場合、apr_pstrcat を使用して password:comment の形式で連結し、値を更新する。 5. apr_dbm_store を呼び出し、キーと値を DBM ファイルに書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htdbm.c:453
  • **呼び出し先**:
  • apr_dbm_exists (APR)
  • apr_pstrcat (APR)
  • apr_dbm_store (APR)
  • strlen (標準ライブラリ)

関連データ構造

  • htdbm_t: データベース接続およびコンテキストを保持するメイン構造体。
  • apr_datum_t: DBM のキーおよび値を保持する APR のデータ構造。

失敗モードと境界条件

  • apr_dbm_store が失敗した場合、そのエラーコードがそのまま戻り値として返される。
  • htdbm->username が NULL の場合は早期リターンする。
  • comment が存在する場合、: を挟んで連結するため、val.dsize には区切り文字分（+1）が加算される。

保守担当者向けメモ

  • 本関数は passwd_common.c で生成されたハッシュ値を永続化する役割を担う。
  • apr_pstrcat を使用しているため、メモリ割り当ては htdbm->ctx.pool から行われる。
  • DBM のキー・値には \0 終端が含まれないため、サイズ計算には注意が必要である。
