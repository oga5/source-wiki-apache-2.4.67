main

定義

  • 種別: function
  • 定義箇所: support/htdigest.c:193

目的

htdigest コマンドのエントリポイントとして、ダイジェスト認証用パスワードファイルの作成およびユーザー情報の追加・更新を行う。

シグネチャまたは形状

int main(int argc, const char * const argv[])

入力と出力

  • **入力**: コマンドライン引数 (argc, argv)。
  • 新規作成モード: -c オプション、ファイル名、レルム、ユーザー名。
  • 更新モード: ファイル名、レルム、ユーザー名。
  • **出力**: 成功時に 0 を返し、エラー発生時に 1 を返して終了する。標準エラー出力 (errfile) に進捗やエラーメッセージを出力する。

前提条件と不変条件

  • APR (Apache Portable Runtime) 環境が初期化されていること。
  • ユーザー名およびレルムの長さは MAX_STRING_LEN (256) 未満であること。
  • パスワードファイル内の各行は MAX_LINE_LEN (768) 未満であること。

アルゴリズムまたは振る舞い

1. **初期化**: apr_app_initialize で環境を構築し、終了処理 terminate を atexit に登録。 2. **モード判定**:

  • 引数が5つで -c が指定された場合、新規ファイルを作成し add_password を実行して終了。
  • それ以外の場合、既存ファイルを読み込み、一時ファイル (apr_file_mktemp) を作成して更新処理を開始。

    • 3. **ファイル処理**:

  • 既存ファイルを1行ずつ読み込み、対象ユーザーが見つかれば新しいハッシュで更新し、それ以外は一時ファイルへコピー。
  • ファイル末尾まで対象ユーザーが見つからなければ、新規ユーザーとして追加。

    • 4. **確定**: 一時ファイルを元のファイル名へ apr_file_copy で上書きし、リソースを解放して終了。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（コマンド実行時）。
  • **呼び出し先**:
  • apr_app_initialize, apr_pool_create, apr_file_open, apr_file_mktemp, apr_file_copy, apr_file_close, apr_file_printf, apr_cpystrn, apr_psprintf, apr_temp_dir_get, apr_signal
  • terminate, usage, add_password, get_line, putline, getword, cleanup_tempfile_and_exit
  • ※注: ツールによる解析結果には、Apache HTTP Server 内部の main 関連シンボルが含まれていますが、本ファイル htdigest.c 内の main との直接的な呼び出し関係は確認されていません。

関連データ構造

  • apr_file_t: ファイル操作用ハンドル。
  • apr_pool_t (cntxt): メモリ管理用プール。
  • MAX_STRING_LEN, MAX_LINE_LEN: 文字列および行の長さ制限定数。

失敗モードと境界条件

  • 引数の数が不正な場合、usage() を呼び出して終了。
  • ファイルのオープンや一時ファイルの作成に失敗した場合、エラーメッセージを出力して終了。
  • 行内の文字列が制限を超えた場合、cleanup_tempfile_and_exit(1) を呼び出して終了。
  • ユーザー名やレルムの不一致により更新対象が見つからない場合、新規追加として処理。

保守担当者向けメモ

  • 本関数は htdigest のメインロジックを担う。ファイル更新はアトミック性を確保するため、一時ファイル経由のコピー方式を採用している。
  • APR_CHARSET_EBCDIC 定義時の変換処理が含まれているが、現代の標準環境では影響しない。
  • 排他制御（ロック）は明示的に実装されていないため、同時実行時の競合には注意が必要。
