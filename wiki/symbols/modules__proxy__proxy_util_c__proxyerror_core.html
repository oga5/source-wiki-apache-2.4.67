proxyerror_core

定義

  • 種別: function
  • 定義箇所: modules/proxy/proxy_util.c:412

目的

プロキシ処理中に発生したエラーをログに記録し、HTTP レスポンスとしてクライアントに返すためのエラー情報をリクエストノートに設定する。

シグネチャまたは形状

static int proxyerror_core(request_rec *r, int statuscode, const char *message, apr_status_t rv)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • statuscode: 返すべき HTTP ステータスコード。
  • message: エラーの理由を示す文字列。
  • rv: 発生した APR エラーコード。
  • **出力**:
  • 戻り値: 引数として渡された statuscode をそのまま返す。

前提条件と不変条件

  • r->pool が有効であること。
  • message は HTML エスケープ処理に適した文字列であること。

アルゴリズムまたは振る舞い

1. ap_log_rerror を使用し、APLOG_ERR レベルでエラーメッセージとリクエスト URI をログに出力する。 2. r->notes テーブルに error-notes キーで HTML エスケープされたエラー理由を追加する。 3. r->notes テーブルに verbose-error-to キーを * に設定し、ap_send_error_response によるエラー詳細の出力を許可する。 4. r->status_line を「[statuscode] Proxy Error」という形式で更新する。 5. statuscode を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/proxy_util.c:434
  • modules/proxy/proxy_util.c:2894
  • modules/proxy/proxy_util.c:2988
  • **呼び出し先**:
  • ap_log_rerror (Apache API)
  • apr_table_setn (APR API)
  • apr_pstrcat (APR API)
  • ap_escape_html (Apache API)
  • apr_psprintf (APR API)

関連データ構造

  • request_rec: Apache のリクエスト処理用構造体。

失敗モードと境界条件

  • r->pool が枯渇している場合、apr_pstrcat や apr_psprintf の呼び出しでメモリ割り当てエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は static であり、proxy_util.c 内でのみ使用される。
  • エラーメッセージは HTML エスケープされるため、呼び出し側で事前にエスケープする必要はない。
