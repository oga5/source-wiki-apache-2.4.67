authn_cache_child_init

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:138

目的

子プロセス生成時に、認証キャッシュ用のグローバルミューテックスを初期化し、プロセス間での同期を可能にする。

シグネチャまたは形状

static void authn_cache_child_init(apr_pool_t *p, server_rec *s)

入力と出力

  • p: 子プロセスのライフサイクルを管理するメモリプール。
  • s: サーバー設定レコード。
  • 出力: なし（副作用として authn_cache_mutex を初期化）。

前提条件と不変条件

  • モジュールが設定済み（configured が真）であること。未設定の場合は処理をスキップする。
  • authn_cache_mutex がグローバルに定義されていること。

アルゴリズムまたは振る舞い

1. configured フラグを確認し、偽であれば即座に終了する。 2. apr_global_mutex_lockfile を使用して、キャッシュ保護用ミューテックスのロックファイルパスを取得する。 3. apr_global_mutex_child_init を呼び出し、子プロセス内でミューテックスを再初期化する。 4. 初期化に失敗した場合、ap_log_error を使用して APLOG_CRIT レベルでエラーログを出力する。

呼び出し元と呼び出し先

  • 呼び出し元: ap_hook_child_init を介して Apache HTTP Server の子プロセス初期化フェーズで呼び出される（modules/aaa/mod_authn_socache.c:462）。
  • 呼び出し先: apr_global_mutex_lockfile, apr_global_mutex_child_init, ap_log_error（いずれも APR/Apache API）。

関連データ構造

  • authn_cache_mutex: キャッシュアクセスを排他制御するためのグローバルミューテックス。
  • server_rec: サーバー設定情報。

失敗モードと境界条件

  • apr_global_mutex_child_init が APR_SUCCESS 以外の値を返した場合、エラーログが出力されるが、プロセスは終了せずに継続する。
  • configured が偽の場合、ミューテックスの初期化は行われない。

保守担当者向けメモ

  • 本関数は子プロセスごとの初期化を担当する。マルチプロセス環境において、親プロセスで作成されたミューテックスを子プロセスで正しく再接続するために不可欠である。
  • configured フラグの管理状態と、ミューテックスのライフサイクルが整合していることを確認すること。
