h2_conn_ctx_create_for_c1

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_conn_ctx.c:54

目的

クライアント接続（C1）に対して HTTP/2 固有のコンテキストを生成し、ソケットのポーリング設定およびノンブロッキング設定を行う。

シグネチャまたは形状

h2_conn_ctx_t *h2_conn_ctx_create_for_c1(conn_rec *c1, server_rec *s, const char *protocol)

入力と出力

  • **入力**:
  • c1: クライアント接続を表す conn_rec 構造体。
  • s: サーバー設定を表す server_rec 構造体。
  • protocol: プロトコル名を示す文字列。
  • **出力**:
  • 生成および初期化された h2_conn_ctx_t 構造体へのポインタ。

前提条件と不変条件

  • c1 は有効な接続オブジェクトであること。
  • ctx_create が内部で呼び出され、c1 のプールを使用してコンテキストが割り当てられる。

アルゴリズムまたは振る舞い

1. ctx_create を呼び出し、c1 の ID を用いてコンテキストを初期化する。 2. 引数で受け取ったサーバー情報 s とプロトコル名をコンテキストに格納する。 3. c1 からソケットを取得し、ctx->pfd（apr_pollfd_t）に設定する。 4. ポーリング対象のイベントとして APR_POLLIN、APR_POLLERR、APR_POLLHUP を指定する。 5. pfd.client_data に現在のコンテキストを保持させる。 6. 対象ソケットを APR_SO_NONBLOCK モードに設定する。 7. 初期化済みのコンテキストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c:286
  • modules/http2/h2_switch.c:177
  • **呼び出し先**:
  • ctx_create（内部関数）
  • apr_psprintf, apr_pstrdup（APR 文字列操作）
  • ap_get_conn_socket（Apache HTTP Server API）
  • apr_socket_opt_set（APR ソケット操作）

関連データ構造

  • h2_conn_ctx_t: 接続ごとの状態を保持する主要な構造体。
  • conn_rec: Apache の接続管理構造体。
  • apr_pollfd_t: ソケットのポーリング設定を保持する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（apr_palloc 等の内部挙動に依存）の動作は本関数内では明示的にハンドリングされていない。
  • ap_get_conn_socket が NULL を返す場合、apr_socket_opt_set 等でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は C1（クライアント接続）専用である。C2（ストリーム処理用接続）の初期化には h2_conn_ctx_init_for_c2 を使用すること。
  • ctx->pfd の設定は、mod_http2 がイベント駆動で接続を監視するために不可欠である。
