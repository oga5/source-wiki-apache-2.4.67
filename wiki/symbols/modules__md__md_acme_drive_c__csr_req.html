csr_req

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:298

目的

ACME サーバーからの証明書発行レスポンスを処理し、証明書の場所（Location）を記録および永続化します。また、レスポンス内に証明書が含まれている場合はその読み込みとチェーンの更新を試みます。

シグネチャまたは形状

static apr_status_t csr_req(md_acme_t *acme, const md_http_response_t *res, void *baton)

入力と出力

  • **入力**:
  • acme: ACME セッション状態（未使用）。
  • res: サーバーからの HTTP レスポンス。
  • baton: md_proto_driver_t へのポインタ。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS。エラー時は md_acme_order_save 等の戻り値。

前提条件と不変条件

  • baton は md_proto_driver_t 型としてキャスト可能であること。
  • res->headers に "location" ヘッダーが存在する場合、その値は証明書の取得先 URL として扱われます。
  • MD_SG_STAGING グループへの書き込み権限が必要です。

アルゴリズムまたは振る舞い

1. **Location の取得**: レスポンスヘッダーから location を取得します。存在しない場合は即座に終了します。 2. **永続化**: location を ad->order->certificate に格納し、md_acme_order_save を呼び出してステージング領域に保存します。 3. **証明書の読み込み**: md_cert_read_http を使用してレスポンスから証明書を抽出します。

  • 成功時: ad->cred->chain をクリアし、取得した証明書を格納した後、get_up_link を呼び出してチェーンのリンクを補完します。
  • ENOENT 時: 証明書がレスポンスに含まれていないと判断し、ポーリングが必要であることをログ出力して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_drive.c:385（md_acme_POST のコールバックとして指定）。
  • **呼び出し先**:
  • md_acme_order_save: 注文情報の保存。
  • md_cert_read_http: レスポンスからの証明書抽出。
  • get_up_link: 証明書チェーンのリンク取得。
  • md_log_perror: ログ出力。

関連データ構造

  • md_acme_t: ACME セッション管理。
  • md_proto_driver_t: プロトコル駆動のコンテキスト。
  • md_acme_driver_t: ACME 固有の駆動状態。
  • md_http_response_t: HTTP レスポンスデータ。

失敗モードと境界条件

  • location ヘッダーが欠落している場合、何も行わず APR_SUCCESS を返します。
  • md_acme_order_save が失敗した場合、エラーコードを返し処理を中断します。
  • md_cert_read_http が ENOENT 以外のエラーを返した場合、そのエラーが呼び出し元に伝播します。

保守担当者向けメモ

  • 本関数は ACMEv1 スタイルのレスポンス処理を想定したロジックを含んでいます。
  • ad->chain_up_link をリセットしてから処理を開始するため、呼び出しごとに状態が初期化される点に注意してください。
