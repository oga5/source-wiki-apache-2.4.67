ap_macro_t

定義

  • 種別: struct
  • 定義箇所: modules/core/mod_macro.c:55

目的

Apache HTTP Server の設定ファイル内で定義されたマクロの情報を保持し、設定の再利用と展開を管理するために使用されます。

シグネチャまたは形状

typedef struct {
    char *name;                    /* マクロの小文字名称 */
    apr_array_header_t *arguments; /* char* の配列：マクロの引数名 */
    apr_array_header_t *contents;  /* char* の配列：マクロの本体 */
    char *location;                /* 定義箇所（エラーメッセージ用） */
} ap_macro_t;

入力と出力

  • 本構造体は、設定解析フェーズにおいて <Macro> ディレクティブの処理結果として生成されます。
  • 展開時には、contents 内の文字列に対して引数の置換が行われ、設定パーサーへ注入されます。

前提条件と不変条件

  • name は小文字で保持される必要があります。
  • arguments および contents は apr_array_header_t 型の配列として管理されます。
  • メモリは apr_pool_t を介して割り当てられます。

アルゴリズムまたは振る舞い

  • 設定ファイル解析時に macro_section 関数によってインスタンス化されます。
  • マクロの定義内容はグローバルなハッシュテーブル ap_macros に格納され、Use ディレクティブが検出された際に参照されます。
  • 展開処理では substitute_macro_args を通じて引数の置換が行われ、make_array_config を経由してパーサーに提供されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_macro.c 内の macro_section（定義時）、use_macro（展開時）、およびマクロの検証・処理を行う各関数（check_macro_arguments 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_hash_t (ap_macros): マクロ名をキー、ap_macro_t へのポインタを値として保持するグローバルなハッシュテーブル。
  • apr_array_header_t: 引数リストおよびマクロ本体の行データを保持するために使用。

失敗モードと境界条件

  • マクロの再帰的な使用は、設定ファイル名にマクロ名が含まれるかをチェックすることで制限されています。
  • エラーメッセージ生成時に location が参照されますが、メモリ管理（cmd->temp_pool）が不適切な場合、ライフサイクル管理に注意が必要です。

保守担当者向けメモ

  • 本構造体は mod_macro モジュール内でのみ使用されます。
  • マクロ定義は pre_config フックで初期化・リセットされる設計である点に留意してください。
  • get_lines_till_end_token を使用した行ベースの解析に依存しているため、複雑なネスト構造の解析には限界があります。
