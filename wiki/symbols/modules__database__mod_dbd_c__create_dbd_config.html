create_dbd_config

定義

  • 種別: function
  • 定義箇所: modules/database/mod_dbd.c:94

目的

サーバーごとのデータベース設定コンテナ（svr_cfg）を初期化し、デフォルト値を設定する。

シグネチャまたは形状

static void *create_dbd_config(apr_pool_t *pool, server_rec *s)

入力と出力

  • **入力**:
  • pool: 設定の割り当てに使用するメモリプール。
  • s: 設定対象のサーバーレコード。
  • **出力**:
  • 初期化された svr_cfg 構造体へのポインタ。

前提条件と不変条件

  • pool は有効な apr_pool_t であること。
  • svr_cfg および内部の dbd_cfg_t は apr_pcalloc によりゼロクリアされた状態で割り当てられる。

アルゴリズムまたは振る舞い

1. svr_cfg 構造体と、その内部の dbd_cfg_t 構造体をメモリプールから割り当てる。 2. cfg->server にサーバーレコードを格納する。 3. cfg->name を no_dbdriver に、cfg->params を空文字列に初期化し、未設定時のセグメンテーションフォールトを防止する。 4. cfg->persist を -1 に設定する。 5. APR_HAS_THREADS が定義されている場合、接続プーリング用のデフォルト値（nmin, nkeep, nmax, exptime）を設定する。 6. クエリ管理用のハッシュテーブル（cfg->queries）および初期化用 SQL 配列（cfg->init_queries）を生成する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/database/mod_dbd.c 内のサーバー設定生成ロジック（151行目、324行目）。
  • modules/database/mod_dbd.c 内のコマンドテーブル定義（990行目）。
  • **呼び出し先**:
  • apr_pcalloc, apr_hash_make, apr_array_make（APR ライブラリ関数）。

関連データ構造

  • svr_cfg: サーバーごとの設定コンテナ。
  • dbd_cfg_t: データベース接続設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数が例外を発生させる可能性がある。
  • APR_HAS_THREADS が定義されていない環境では、プーリング関連のパラメータは初期化されない。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。設定値のデフォルト変更が必要な場合は、本関数の初期化ロジックを更新すること。
