log_find_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_status.c:455

目的

ジョブのログ検索処理において、検索対象のジョブ情報、ログタイプ、および現在の検索位置を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    md_job_t *job;
    const char *type;
    md_json_t *entry;
    size_t index;
} log_find_ctx;

入力と出力

本構造体は関数間のデータ受け渡しに使用されます。

  • 入力: job（対象ジョブ）、type（ログタイプ）、index（検索インデックス）
  • 出力: entry（検索結果として見つかった JSON エントリ）

前提条件と不変条件

  • job は有効な md_job_t 構造体を指している必要があります。
  • entry は検索処理の結果として設定されるため、初期状態では NULL であることが想定されます。

アルゴリズムまたは振る舞い

ジョブに関連付けられたログエントリを走査する際、現在の検索状態（どのジョブの、どのタイプのログを、何番目まで確認したか）を保持するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_status.c 内のログ検索関連関数（baton として渡されるケースが確認されています）。
  • 呼び出し先: 情報なし。

関連データ構造

  • md_job_t: 管理対象のバックグラウンドジョブ構造体。
  • md_json_t: JSON データの保持に使用される構造体。

失敗モードと境界条件

  • job が NULL の場合、または type が無効な文字列を指している場合、検索処理が失敗する可能性があります。
  • index がログの総数を超過した場合の挙動は、この構造体を使用する側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は md_status.c 内のローカルな検索処理でのみ使用されています。
  • ログの最大件数制限（max_log）に関連するロジックと密接に関係しているため、ログの構造に変更を加える際は本構造体の利用箇所を確認してください。
