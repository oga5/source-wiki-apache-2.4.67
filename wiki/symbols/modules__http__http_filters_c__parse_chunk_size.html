parse_chunk_size

定義

  • 種別: function
  • 定義箇所: modules/http/http_filters.c:137

目的

HTTP チャンク転送エンコーディングにおけるチャンクサイズ行を解析し、状態遷移を管理します。

シグネチャまたは形状

apr_status_t parse_chunk_size(http_ctx_t *ctx, const char *buffer, apr_size_t len, int linelimit, int strict)

入力と出力

  • **入力**:
  • ctx: フィルタの状態を保持する http_ctx_t 構造体。
  • buffer: 解析対象のデータバッファ。
  • len: バッファの長さ。
  • linelimit: 許容される最大行長。
  • strict: RFC 準拠の厳格なチェックを行うかどうかのフラグ。
  • **出力**:
  • APR_SUCCESS: 解析成功。
  • APR_EINVAL: プロトコル違反（不正な文字、CRLF 不足など）。
  • APR_ENOSPC: サイズオーバーフローまたは行長制限超過。
  • APR_EGENERAL: 想定外の状態遷移。

前提条件と不変条件

  • ctx->state は、現在のチャンク解析フェーズ（BODY_CHUNK, BODY_CHUNK_PART, BODY_CHUNK_END 等）を正確に反映している必要があります。
  • ctx->remaining は、現在のチャンクの残りのサイズを保持します。
  • ctx->chunk_bws は、チャンクヘッダ内の空白（LWS）の数を追跡し、10を超えないことを保証します。

アルゴリズムまたは振る舞い

1. バッファ内の各文字を ASCII に変換し、ctx->state に基づいて逐次処理します。 2. **チャンク終了処理**: BODY_CHUNK_END 状態では CRLF を確認し、次のチャンク開始へ遷移します。 3. **サイズ解析**: BODY_CHUNK_PART 状態では、16進数を読み取り ctx->remaining を更新します。オーバーフロー時は APR_ENOSPC を返します。 4. **拡張フィールド**: ; が現れると BODY_CHUNK_EXT へ遷移し、制御文字を排除しつつ拡張情報をスキップします。 5. **行長制限**: 処理した総バイト数 ctx->chunk_used が linelimit を超えた場合、エラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_filters.c:545（ap_http_filter 内から呼び出される）。
  • **呼び出し先**: ap_xlate_proto_from_ascii（文字変換）、apr_isxdigit / apr_iscntrl（文字種判定）。

関連データ構造

  • http_ctx_t: チャンク解析の状態（state）、残余サイズ（remaining）、ビット数（chunkbits）、使用量（chunk_used）を保持。

失敗モードと境界条件

  • **オーバーフロー**: チャンクサイズが apr_off_t の範囲を超える場合、または linelimit を超える場合に APR_ENOSPC を返します。
  • **フォーマット違反**: 16進数以外の文字、CRLF の欠落、または strict モードでの不正な改行シーケンスに対し APR_EINVAL を返します。
  • **空白制限**: チャンクヘッダ内の空白が10個を超えると APR_EINVAL を返します。

保守担当者向けメモ

  • ctx->state の遷移は RFC 7230 に基づく厳密なステートマシンです。変更時は http_ctx_t の定義と整合性を確認してください。
  • strict フラグは、プロトコル違反に対してどの程度寛容であるかを制御します。
