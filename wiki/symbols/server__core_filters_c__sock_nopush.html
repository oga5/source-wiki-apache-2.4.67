sock_nopush

定義

  • 種別: function
  • 定義箇所: server/core_filters.c:637

目的

ソケットに対して TCP_NOPUSH（または TCP_CORK）オプションを設定し、パケットの送信タイミングを制御する。

シグネチャまたは形状

static APR_INLINE void sock_nopush(apr_socket_t *s, int to)

入力と出力

  • s: 操作対象の apr_socket_t 構造体へのポインタ。
  • to: 1 を指定すると TCP_NOPUSH を有効化し、0 を指定すると無効化する。
  • 戻り値: なし。

前提条件と不変条件

  • APR_TCP_NOPUSH_FLAG が定義されている環境でのみ動作する。
  • macOS (__APPLE__) 環境では、OS 側の挙動（TCP_NOPUSH 解除時に保留データが即座に送信されない問題）を回避するため、本関数による操作は無効化される。

アルゴリズムまたは振る舞い

1. コンパイル時に APR_TCP_NOPUSH_FLAG が定義されており、かつターゲットが macOS ではないことを確認する。 2. apr_socket_opt_set を呼び出し、ソケット s に対して APR_TCP_NOPUSH オプションを to の値で設定する。 3. macOS の場合は何もしない。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/core_filters.c:670 (sock_nopush(s, 1))
  • server/core_filters.c:700 (sock_nopush(s, 0))
  • server/core_filters.c:727 (sock_nopush(s, 1))
  • server/core_filters.c:766 (sock_nopush(s, 1))
  • server/core_filters.c:780 (sock_nopush(s, 0))
  • 呼び出し先:
  • apr_socket_opt_set (APR ライブラリ関数)

関連データ構造

  • apr_socket_t: APR が提供するソケット抽象化構造体。

失敗モードと境界条件

  • apr_socket_opt_set の戻り値は無視されるため、ソケットオプションの設定失敗は呼び出し元に伝播しない。
  • macOS 環境では、to の値に関わらずソケットの状態は変更されない。

保守担当者向けメモ

  • 本関数は APR_INLINE として定義されており、ヘッダの依存関係やコンパイル環境に強く依存する。
  • macOS における TCP_NOPUSH の挙動に関するコメントは、KeepAlive 接続時の遅延問題に関連しているため、将来的な OS の挙動変更がない限り削除しないこと。
