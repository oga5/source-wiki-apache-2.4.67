host_parse_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:311

目的

Require host ディレクティブで指定された引数を解析し、実行時に評価可能な式（ap_expr_info_t）へコンパイルします。

シグネチャまたは形状

static const char *host_parse_config(cmd_parms *cmd, const char *require_line,
                                     const void **parsed_require_line);

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • require_line: Require ディレクティブに続く設定文字列。
  • parsed_require_line: 解析結果の式構造体を格納するための出力用ポインタ。
  • **出力**:
  • 成功時: NULL を返し、*parsed_require_line にコンパイル済みの ap_expr_info_t を設定。
  • 失敗時: エラーメッセージ文字列を返し、*parsed_require_line は更新されません。

前提条件と不変条件

  • cmd->temp_pool が有効である必要があります。
  • ap_expr_parse_cmd を介して式を解析するため、Apache の式評価エンジンの構文規則に従う必要があります。

アルゴリズムまたは振る舞い

1. ap_expr_parse_cmd を呼び出し、require_line を AP_EXPR_FLAG_STRING_RESULT フラグ付きで解析します。 2. 解析中にエラーが発生した場合、cmd->temp_pool を使用してエラーメッセージを生成し、呼び出し元へ返します。 3. 解析が成功した場合、生成された ap_expr_info_t 構造体を *parsed_require_line に格納し、NULL を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_host.c 内の authz_host_provider 構造体定義において、Require host および Require forward-dns の設定解析用関数として登録されています（339行目、345行目）。
  • **呼び出し先**: ap_expr_parse_cmd（Apache コアの式評価エンジン）。

関連データ構造

  • ap_expr_info_t: 解析された式を保持する構造体。
  • cmd_parms: 設定解析時のコンテキスト情報。

失敗モードと境界条件

  • require_line が不正な構文である場合、ap_expr_parse_cmd がエラーを返し、本関数はそれを伝播させます。
  • メモリ不足等の理由で式がコンパイルできない場合、エラーメッセージが返されます。

保守担当者向けメモ

  • 本関数は ap_expr エンジンを利用しているため、構文解析の詳細は core モジュール側の実装に依存します。
  • parsed_require_line に格納されるデータは、設定解析フェーズのメモリプールに関連付けられている点に注意してください。
