reqtimeout_srv_cfg

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_reqtimeout.c:50

目的

Apache HTTP Server における mod_reqtimeout モジュールのサーバー単位の設定を保持する構造体です。リクエスト処理の各段階におけるタイムアウト閾値を管理します。

シグネチャまたは形状

typedef struct
{
    reqtimeout_stage_t handshake;   /* TLSハンドシェイク用 */
    reqtimeout_stage_t header;      /* HTTPヘッダー読み取り用 */
    reqtimeout_stage_t body;        /* HTTPボディ読み取り用 */
} reqtimeout_srv_cfg;

入力と出力

本構造体は設定値のコンテナであり、直接的な関数入出力は持ちません。設定値は RequestReadTimeout ディレクティブを通じて設定され、モジュール内の各フィルタ処理で参照されます。

前提条件と不変条件

  • 各フィールドは reqtimeout_stage_t 型であり、個別のタイムアウト値および最小データ転送レートを保持します。
  • サーバー設定のライフサイクルを通じて生存します。

アルゴリズムまたは振る舞い

  • サーバーごとの設定を保持し、接続ごとの状態（reqtimeout_con_cfg）を初期化する際のベースラインとして使用されます。
  • apr_pcalloc を用いてメモリ確保が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_reqtimeout.c 内の set_reqtimeout_param（設定値の適用）、reqtimeout_create_srv_config（構造体の生成）、および設定のマージ処理（base_, add_）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • reqtimeout_stage_t: 各フェーズのタイムアウト設定を定義する構造体。
  • reqtimeout_con_cfg: 接続ごとの実行時状態を保持する構造体。

失敗モードと境界条件

  • 設定が未定義の場合、デフォルト値が適用されます。
  • 構造体のメモリ確保（apr_pcalloc）が失敗した場合の挙動は、Apache のメモリ管理ポリシーに依存します。

保守担当者向けメモ

  • 本構造体は RequestReadTimeout ディレクティブの各パラメータを保持します。設定値の変更や新しいフェーズの追加を行う際は、本構造体の定義とマージロジックを更新してください。
