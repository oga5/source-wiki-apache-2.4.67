standard_flush

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fdpass.c:194

目的

プロキシ接続において、保留中のレスポンスデータを強制的にフラッシュし、接続をクローズ状態に設定する。

シグネチャまたは形状

static int standard_flush(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (成功時は OK、失敗時は ap_pass_brigade が返すエラーコード)

前提条件と不変条件

  • r->connection が有効であること。
  • r->output_filters が適切に設定されていること。

アルゴリズムまたは振る舞い

1. リクエストの接続状態を AP_CONN_CLOSE に設定し、キープアライブを無効化する。 2. apr_brigade_create を用いて新しいバケットブリゲードを作成する。 3. apr_bucket_flush_create でフラッシュバケットを作成し、ブリゲードの末尾に挿入する。 4. ap_pass_brigade を呼び出し、出力フィルタチェーンを通じてフラッシュバケットを送信する。 5. 送信結果が OK でない場合、エラーログを出力してそのステータスを返す。 6. 正常終了時は OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_fdpass.c:222 (静的解析による参照)
  • **呼び出し先**:
  • apr_brigade_create
  • apr_bucket_flush_create
  • APR_BRIGADE_INSERT_TAIL
  • ap_pass_brigade
  • ap_log_rerror

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • apr_bucket_brigade: データ転送用のバケット管理構造体。
  • apr_bucket: バケット（ここではフラッシュバケット）。

失敗モードと境界条件

  • ap_pass_brigade がエラーを返した場合、そのエラーコードをそのまま呼び出し元へ伝播する。
  • メモリ不足等によりブリゲードの作成に失敗した場合、APR の標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数は mod_proxy_fdpass におけるデフォルトのフラッシュプロバイダとして機能する。
  • 接続を強制的に閉じるため、呼び出し後の通信継続は想定されていない。
