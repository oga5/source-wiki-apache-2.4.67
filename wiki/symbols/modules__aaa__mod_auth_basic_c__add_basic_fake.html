add_basic_fake

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_basic.c:142

目的

AuthBasicFake ディレクティブの設定を解析し、認証情報を偽装するための式をコンパイルして設定構造体に格納する。

シグネチャまたは形状

static const char *add_basic_fake(cmd_parms *cmd, void *config, const char *user, const char *pass)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: auth_basic_config_rec 構造体へのポインタ。
  • user: 偽装するユーザー名（または "off"）。
  • pass: 偽装するパスワード（省略可）。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • config は auth_basic_config_rec 型としてキャスト可能であること。
  • pass が指定されない場合、互換性のためにデフォルト値 "password" が使用される。

アルゴリズムまたは振る舞い

1. user 引数が "off" の場合、設定内の偽装情報をクリアし fake_set を 1 に設定して終了する。 2. それ以外の場合、user および pass を ap_expr_parse_cmd を用いて式としてコンパイルする。 3. コンパイルされた式を conf->fakeuser および conf->fakepass に格納する。 4. 解析中にエラーが発生した場合は、cmd->pool から割り当てられたエラーメッセージを返す。 5. 正常終了時に conf->fake_set を 1 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_basic.c:207 における AP_INIT_TAKE12 マクロ経由で、Apache の設定読み込み時に呼び出される。
  • **呼び出し先**: ap_expr_parse_cmd（式評価エンジンのコンパイル関数）、apr_psprintf（エラーメッセージ生成）。

関連データ構造

  • auth_basic_config_rec: ディレクトリごとの設定を保持する構造体。fakeuser, fakepass, fake_set フィールドを更新する。

失敗モードと境界条件

  • user または pass の式が不正な場合、ap_expr_parse_cmd がエラーを返し、関数はエラーメッセージを返して終了する。
  • user に "off" を指定すると、既存の偽装設定が破棄される。

保守担当者向けメモ

  • 本関数は設定フェーズでのみ実行される。
  • ap_expr の評価は、後のリクエスト処理フェーズにおいて authenticate_basic_fake 関数内で行われる。
  • メモリ管理は cmd->pool に依存しており、明示的な解放は不要。
