sock_readhdr

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgid.c:352

目的

Unixドメインソケットから指定されたサイズのデータ（ヘッダー情報）を読み取り、オプションで補助的なファイルディスクリプタ（errfd）を受信します。

シグネチャまたは形状

apr_status_t sock_readhdr(int fd, int *errfd, void *vbuf, size_t buf_size)

入力と出力

  • **入力**:
  • fd: 読み取り対象のソケットファイルディスクリプタ。
  • errfd: 補助的なファイルディスクリプタを受け取るためのポインタ（不要な場合は NULL）。
  • vbuf: 受信データを格納するバッファへのポインタ。
  • buf_size: 読み取るデータのバイト数。
  • **出力**:
  • 戻り値: 成功時は APR_SUCCESS。エラー時は errno または ECONNRESET、APR_INCOMPLETE。
  • errfd: 補助的なファイルディスクリプタが受信された場合、その値が格納されます。

前提条件と不変条件

  • HAVE_CGID_FDPASSING が定義されている場合、recvmsg を使用してデータと補助データ（SCM_RIGHTS）を同時に取得します。
  • HAVE_CGID_FDPASSING が定義されていない場合、read を使用して指定バイト数に達するまでループします。

アルゴリズムまたは振る舞い

1. **データ読み取り**:

  • HAVE_CGID_FDPASSING 未定義時: read を呼び出し、EINTR を考慮しつつ buf_size に達するまで読み取りを継続します。
  • HAVE_CGID_FDPASSING 定義時: recvmsg に MSG_WAITALL を指定して呼び出し、指定されたデータ長を一度の操作で取得しようと試みます。

    • 2. **エラーハンドリング**:

  • 読み取り中に接続が切断された場合は ECONNRESET を返します。
  • システムエラー発生時は errno を返します。
  • recvmsg で指定バイト数と異なるデータが返された場合は APR_INCOMPLETE を返します。

    • 3. **補助ディスクリプタの抽出**:

  • errfd が指定されている場合、cmsghdr を解析し、SCM_RIGHTS 型のデータからファイルディスクリプタを抽出して *errfd に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgid.c:426 (補助ディスクリプタなしのラッパー関数内)
  • modules/generators/mod_cgid.c:512
  • **呼び出し先**:
  • read (システムコール)
  • recvmsg (システムコール)
  • CMSG_FIRSTHDR, CMSG_LEN, CMSG_DATA (マクロ)

関連データ構造

  • struct msghdr
  • struct iovec
  • struct cmsghdr

失敗モードと境界条件

  • read/recvmsg が 0 を返した場合、接続が予期せず閉じられたと判断し ECONNRESET を返します。
  • read/recvmsg が負の値を返した場合、errno を返します。
  • MSG_WAITALL を使用してもなお読み取りサイズが一致しない場合、APR_INCOMPLETE を返します。

保守担当者向けメモ

  • HAVE_CGID_FDPASSING の有無により実装が大きく分岐するため、プラットフォームごとのソケット通信仕様に依存します。
  • errfd の抽出ロジックは、cmsg_len や cmsg_level 等の厳密なチェックを行っており、変更時は msghdr 構造体の整合性に注意してください。
