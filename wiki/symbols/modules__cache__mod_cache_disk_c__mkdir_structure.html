mkdir_structure

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:110

目的

指定されたファイルパスの親ディレクトリ構造を再帰的に作成します。キャッシュファイルを保存する前に、必要なディレクトリ階層がディスク上に存在することを保証するために使用されます。

シグネチャまたは形状

apr_status_t mkdir_structure(disk_cache_conf *conf, const char *file, apr_pool_t *pool)

入力と出力

  • **入力**:
  • conf: キャッシュ設定構造体（cache_root_len を使用してパスの基点を特定）。
  • file: 作成対象のファイルパス（フルパス）。
  • pool: メモリ割り当てに使用する APR プール。
  • **出力**:
  • APR_SUCCESS: ディレクトリ構造の作成に成功した場合。
  • apr_status_t: エラー発生時の APR エラーコード。

前提条件と不変条件

  • file 引数は conf->cache_root_len よりも長いパスである必要があります。
  • conf->cache_root_len は、キャッシュルートディレクトリの長さとして正しく設定されている必要があります。

アルゴリズムまたは振る舞い

1. file パスの cache_root_len 以降の文字列を走査します。 2. / を見つけるたびに、その位置を一時的に終端文字 (\0) に置き換えてディレクトリパスを切り出します。 3. apr_dir_make を呼び出し、APR_UREAD|APR_UWRITE|APR_UEXECUTE（所有者による読み書き実行権限）でディレクトリを作成します。 4. ディレクトリが既に存在する場合（APR_STATUS_IS_EEXIST）はエラーとみなさず処理を継続します。 5. 切り出したパスを / に戻し、次の階層へ進みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c:150
  • modules/cache/mod_cache_disk.c:996
  • **呼び出し先**:
  • strchr (標準ライブラリ)
  • apr_dir_make (APR)

関連データ構造

  • disk_cache_conf: キャッシュ設定を保持する構造体。

失敗モードと境界条件

  • apr_dir_make が EEXIST 以外のエラーを返した場合、そのエラーコードを即座に返して処理を中断します。
  • パス内に / が含まれない場合や、ルートディレクトリのみの場合はループが終了します。

保守担当者向けメモ

  • 本関数は、ディレクトリ作成時に所有者権限のみを付与します。グループや他人の権限が必要な環境では、この権限設定の調整が必要になる可能性があります。
  • conf->cache_root_len を基準にしているため、キャッシュルート外のディレクトリ作成には使用できません。
