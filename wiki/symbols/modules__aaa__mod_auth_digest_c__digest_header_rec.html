digest_header_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_auth_digest.c:142

目的

HTTP Digest 認証において、クライアントから送信された Authorization ヘッダーの内容をパースし、認証検証プロセス全体で利用可能な形式で保持するために使用されます。

シグネチャまたは形状

typedef struct digest_header_struct {
    const char           *scheme;
    const char           *realm;
    const char           *username;
          char           *nonce;
    const char           *uri;
    const char           *method;
    const char           *digest;
    const char           *algorithm;
    const char           *cnonce;
    const char           *opaque;
    unsigned long         opaque_num;
    const char           *message_qop;
    const char           *nonce_count;
    const char           *raw_request_uri;
    apr_uri_t            *psd_request_uri;
    apr_time_t            nonce_time;
    enum hdr_sts          auth_hdr_sts;
    int                   needed_auth;
    const char           *ha1;
    client_entry         *client;
} digest_header_rec;

入力と出力

  • **入力**: get_digest_rec() 関数により、リクエスト内の Authorization ヘッダー文字列から各フィールドが抽出・代入されます。
  • **出力**: 認証ロジック（check_nonce や check_nc など）において、検証対象のデータソースとして参照されます。

前提条件と不変条件

  • apr_pcalloc 等でメモリ確保された後に使用される必要があります。
  • 共有メモリ上の client_entry へのポインタ（client）は、共有メモリの有効期間内でのみ有効です。

アルゴリズムまたは振る舞い

1. **パース**: get_digest_rec がヘッダーを解析し、構造体の各メンバを埋めます。 2. **拡張情報**: ヘッダーから直接得られない情報（raw_request_uri、psd_request_uri、nonce_time、client 等）は、認証処理の過程で計算または検索され、本構造体に格納されます。 3. **状態管理**: auth_hdr_sts により、ヘッダーの解析状態や認証の進捗状況を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: get_digest_rec（初期化）、authenticate_digest_user（認証フロー）、check_nc（カウント検証）、check_nonce（ナンス検証）などで参照・更新されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • client_entry: 共有メモリ上のクライアント状態を保持する構造体。本構造体の client メンバから参照されます。
  • apr_uri_t: URI 解析結果を保持する APR の標準構造体。
  • hdr_sts: ヘッダー解析の状態を示す列挙型。

失敗モードと境界条件

  • ヘッダーのパースに失敗した場合、auth_hdr_sts にエラー状態が記録されます。
  • client メンバが NULL の場合、共有メモリ上に該当するクライアントエントリが存在しないことを示します。

保守担当者向けメモ

  • 本構造体は mod_auth_digest の認証ライフサイクル全体を通じて受け渡されるため、メンバの追加や変更は認証ロジック全体に影響を与えます。
  • ha1 や client など、ヘッダーから直接得られないフィールドは、認証の各フェーズで適切に設定されていることを確認してください。
