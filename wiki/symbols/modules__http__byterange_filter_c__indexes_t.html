indexes_t

定義

  • 種別: struct
  • 定義箇所: modules/http/byterange_filter.c:72

目的

HTTP レスポンスにおけるバイト範囲（Byte-range）の開始位置と終了位置を保持し、Range リクエストの処理結果を管理するために使用されます。

シグネチャまたは形状

typedef struct indexes_t {
    apr_off_t start;
    apr_off_t end;
} indexes_t;

入力と出力

  • 本構造体は、ap_set_byterange 関数によって生成される apr_array_header_t 配列の要素として格納されます。
  • start および end は apr_off_t 型であり、ファイルやバケット内のオフセット値を表します。

前提条件と不変条件

  • start は範囲の開始バイト位置、end は範囲の終了バイト位置を示します。
  • ap_set_byterange による解析過程で、範囲の逆転や重複がチェックされ、有効な範囲のみが本構造体に格納されます。

アルゴリズムまたは振る舞い

  • ap_set_byterange で解析された各範囲が indexes_t インスタンスとして配列にプッシュされます。
  • フィルタ処理（ap_byterange_filter）において、この構造体を参照してバケットの切り出し（copy_brigade_range）が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: ap_set_byterange（配列への追加）、ap_byterange_filter（配列の走査）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_array_header_t: indexes_t のインスタンスを複数保持するために使用される APR 配列。
  • apr_off_t: ファイルオフセットを表現するための APR 型。

失敗モードと境界条件

  • 範囲指定が不正な場合や、設定された最大範囲数（max_ranges）を超える場合は、ap_set_byterange が処理を中断するため、本構造体が生成されない、あるいは処理がスキップされます。

保守担当者向けメモ

  • r->pool を使用してメモリ確保が行われるため、リクエスト終了時に自動的に解放されます。
  • 範囲が非常に多い場合、この構造体を用いたバケットの分割・コピー処理が CPU およびメモリ負荷の要因となるため、AP_DEFAULT_MAX_RANGES による制限が適用されています。
