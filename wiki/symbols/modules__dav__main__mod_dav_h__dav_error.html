dav_error

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:121

目的

WebDAV 操作中に発生したエラー状態を保持および伝播するためのスタック構造体です。複数のエラーを連結して管理し、最終的に HTTP レスポンスへ変換することを可能にします。

シグネチャまたは形状

typedef struct dav_error {
    int status;
    int error_id;
    const char *desc;
    apr_status_t aprerr;
    const char *namespace;
    const char *tagname;
    struct dav_error *prev;
    const char *childtags;
} dav_error;

入力と出力

  • **入力**: dav_new_error 等の関数を介して各フィールドが初期化されます。
  • **出力**: dav_handle_err() 関数に渡され、HTTP ステータスコードおよび XML レスポンスボディとしてクライアントへ返されます。

前提条件と不変条件

  • status が 0 の場合、エラーは発生していないとみなされます。
  • prev フィールドにより、エラーのスタック（連結リスト）を構成できます。
  • aprerr は APR 関連のエラーコード、または成功を示す APR_SUCCESS (0) を保持します。

アルゴリズムまたは振る舞い

  • WebDAV プロバイダーの各フック関数は、処理の失敗時に dav_error 構造体へのポインタを返します。
  • dav_handle_err() は、この構造体を解析し、適切な HTTP ステータスコードを設定して、エラー詳細を XML 形式でレスポンスに書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/dbm.c 内の dav_fs_dbm_error、dav_dbm_open、dav_dbm_fetch 等のプロバイダー実装関数が、エラー発生時に本構造体を生成して返します。
  • **呼び出し先**: mod_dav のメインハンドラ（mod_dav.c）が、フックから返された dav_error を受け取り、dav_handle_err() を呼び出して処理を完結させます。

関連データ構造

  • dav_response: エラー発生時のマルチステータスレスポンス生成に関連します。

失敗モードと境界条件

  • メモリ不足により dav_error 自体の割り当てに失敗する可能性があります。
  • prev を介したスタックの深さが過度になった場合、エラー処理の再帰的な走査でスタックオーバーフローのリスクがある可能性があります（ただし、一般的な WebDAV 操作では発生しにくい）。

保守担当者向けメモ

  • include/ap_mmn.h の記録によれば、バージョン 2.3.4-dev で apr_status_t を使用するようにインターフェースが変更され、2.4.24-dev で childtags フィールドが追加されました。ABI 互換性に注意してください。
