md_timeperiod_has_started

定義

  • 種別: function
  • 定義箇所: modules/md/md_time.c:37

目的

指定された時刻が、管理対象の期間（md_timeperiod_t）の開始時刻以降であるかを判定します。

シグネチャまたは形状

int md_timeperiod_has_started(const md_timeperiod_t *period, apr_time_t time)

入力と出力

  • **入力**:
  • period: 判定対象となる期間構造体へのポインタ。
  • time: 判定基準となる時刻（apr_time_t）。
  • **出力**:
  • int: time が period->start 以上であれば非ゼロ（真）、そうでなければ 0（偽）を返します。

前提条件と不変条件

  • period は有効な md_timeperiod_t 構造体を指している必要があります。
  • apr_time_t は APR（Apache Portable Runtime）で定義された時間形式である必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った time と、period 構造体内の start メンバを比較します。 2. time >= period->start の評価結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_ocsp.c:176
  • modules/md/md_reg.c:759
  • modules/md/md_time.c:33
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、構造体のメンバアクセスと比較演算のみを実行します。

関連データ構造

  • md_timeperiod_t: 期間の開始と終了を保持する構造体。
  • apr_time_t: APR で定義される時刻表現。

失敗モードと境界条件

  • period が NULL の場合、プログラムはクラッシュする可能性があります。
  • time と period->start が等しい場合、本関数は真（1）を返します。

保守担当者向けメモ

  • 本関数は単純な比較演算子をラップしたものです。将来的に期間の定義が複雑化した場合でも、このインターフェースを維持することで呼び出し側のロジックを保護できます。
