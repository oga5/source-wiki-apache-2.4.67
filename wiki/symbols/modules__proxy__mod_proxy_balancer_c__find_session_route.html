find_session_route

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:272

目的

リクエストに含まれるスティッキーセッション情報（パスパラメータまたはクッキー）を解析し、対応するバックエンドワーカーを特定する。

シグネチャまたは形状

proxy_worker *find_session_route(proxy_balancer *balancer,
                                 request_rec *r,
                                 char **route,
                                 const char **sticky_used,
                                 char **url)

入力と出力

  • **入力**:
  • balancer: 対象のロードバランサ構造体。
  • r: 現在のリクエスト情報。
  • url: 検索対象のURL（クエリ文字列を含む場合がある）。
  • **出力**:
  • 戻り値: 特定された proxy_worker。見つからない場合は NULL。
  • route: 抽出されたルート文字列。
  • sticky_used: 使用されたスティッキーセッションのキー名。

前提条件と不変条件

  • balancer->s->sticky が設定されている必要がある（未設定の場合は即座に NULL を返す）。
  • url はクエリ文字列を含まないことが想定されているが、関数内で r->args を用いて一時的に結合される。

アルゴリズムまたは振る舞い

1. **スティッキーセッションの検索**:

  • まず get_path_param を使用し、URLからパスパラメータとしてルートを検索する。
  • 見つからない場合、get_cookie_param を使用し、クッキーからルートを検索する。

    • 2. **ルートの抽出**:

  • 取得した値から balancer->s->sticky_separator を探し、それ以降の文字列をルートとして抽出する。

    • 3. **ワーカーの特定**:

  • find_route_worker を呼び出し、抽出したルートに対応するワーカーを検索する。
  • クライアントが提供したルートと、実際に特定されたワーカーのルートが異なる場合、環境変数 BALANCER_ROUTE_CHANGED を "1" に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_balancer.c:514 (proxy_balancer_pre_request 内)
  • **呼び出し先**:
  • get_path_param
  • get_cookie_param
  • find_route_worker

関連データ構造

  • proxy_balancer: バランサの設定と状態を保持。
  • proxy_worker: 負荷分散先のバックエンドワーカー。
  • request_rec: Apache のリクエスト構造体。

失敗モードと境界条件

  • スティッキーセッションが有効でない場合、NULL を返す。
  • ルート情報がリクエストに含まれていない、または対応するワーカーが存在しない場合、NULL を返す。
  • sticky_separator が設定されていない場合、抽出ロジックがスキップされる。

保守担当者向けメモ

  • 本関数は、クライアントのセッション維持を優先するルーティングの要です。
  • find_route_worker の第4引数に 1 を渡しており、ルート不一致時のリトライ処理を有効にしています。
  • ログ出力には APLOGNO が付与されており、デバッグ時には APLOG_DEBUG レベルで追跡可能です。
