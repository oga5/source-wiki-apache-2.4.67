md_config_create_svr

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_config.c:241

目的

Apache HTTP Server のバーチャルホスト単位の設定構造体（md_srv_conf_t）を生成し、初期化する。

シグネチャまたは形状

void *md_config_create_svr(apr_pool_t *pool, server_rec *s)

入力と出力

  • **入力**:
  • pool: 設定構造体の割り当てに使用するメモリプール。
  • s: 設定対象となるサーバーレコード。
  • **出力**:
  • 初期化済みの md_srv_conf_t 構造体へのポインタ（void * 型として返却）。

前提条件と不変条件

  • pool は有効な apr_pool_t であること。
  • s は有効な server_rec であること。
  • 内部で md_mod_conf_get(pool, 1) を呼び出し、モジュール全体のグローバル設定を取得する。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して md_srv_conf_t 構造体をゼロクリアした状態でメモリ確保する。 2. サーバー名を含む識別文字列を conf->name に生成・格納する。 3. サーバーレコード s を conf->s に保持する。 4. md_mod_conf_get を呼び出し、モジュール共通設定を conf->mc に割り当てる。 5. srv_conf_props_clear(conf) を呼び出し、サーバー固有の設定プロパティをデフォルト値で初期化する。 6. 初期化された構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c:64（サーバー設定作成関数として登録）。
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_pstrcat (APR)
  • md_mod_conf_get (mod_md_config.c)
  • srv_conf_props_clear (mod_md_config.c)

関連データ構造

  • md_srv_conf_t: バーチャルホスト単位の設定を保持する主要構造体。
  • server_rec: Apache のサーバーレコード。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性がある。
  • md_mod_conf_get が失敗した場合の挙動は当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。
  • srv_conf_props_clear による初期化は、将来的に設定項目が増えた場合、必ずデフォルト値の定義と同期させる必要がある。
