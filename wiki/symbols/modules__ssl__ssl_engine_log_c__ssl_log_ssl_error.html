ssl_log_ssl_error

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_log.c:94

目的

OpenSSL のエラーキューを走査し、Apache のログシステムを通じて診断情報を出力する。エラー内容に応じたアノテーション（補足説明）を自動的に付与することで、SSL/TLS 関連の問題解決を支援する。

シグネチャまたは形状

void ssl_log_ssl_error(const char *file, int line, int level, server_rec *s)

入力と出力

  • **入力**:
  • file: ログ出力元ファイル名（SSLLOG_MARK 等で指定）。
  • line: ログ出力元行番号。
  • level: Apache ログレベル（APLOG_ERR, APLOG_EMERG 等）。
  • s: エラーに関連する server_rec 構造体。
  • **出力**: なし（戻り値なし）。副作用として Apache のエラーログへメッセージを書き出す。

前提条件と不変条件

  • OpenSSL のエラーキューにエラーが蓄積されていることを前提とする。
  • modssl_ERR_peek_error_data が OpenSSL バージョン間の差異を吸収し、エラーデータとフラグを正しく取得できること。
  • ssl_log_annotation がエラー文字列に対応するヒントを返せること。

アルゴリズムまたは振る舞い

1. modssl_ERR_peek_error_data をループ内で呼び出し、エラーキューが空になるまで以下の処理を繰り返す。 2. ERR_error_string_n を使用してエラーコードを文字列に変換する。 3. ssl_log_annotation を呼び出し、変換されたエラー文字列に基づいた補足説明を取得する。 4. ap_log_error を使用し、エラー文字列、付随するデータ（存在する場合）、およびアノテーションを整形してログに出力する。 5. ERR_get_error を呼び出し、処理したエラーをキューから削除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/mod_ssl.c
  • modules/ssl/ssl_engine_init.c
  • **呼び出し先**:
  • modssl_ERR_peek_error_data
  • ERR_error_string_n
  • ssl_log_annotation
  • ap_log_error
  • ERR_get_error

関連データ構造

  • server_rec: Apache のサーバー設定構造体。
  • ERR_TXT_STRING: OpenSSL のエラーデータフラグ。

失敗モードと境界条件

  • エラーデータが存在しない場合、ループは即座に終了する。
  • flags に ERR_TXT_STRING が含まれていない場合、data は NULL として扱われ、ログ出力時にデータ部分は省略される。
  • アノテーションが見つからない場合、ログ出力にはアノテーション部分は含まれない。

保守担当者向けメモ

  • 本関数は OpenSSL 3.0 以降の API 変更を考慮した modssl_ERR_peek_error_data に依存しているため、OpenSSL のバージョンアップ時にはこの互換レイヤーの挙動を確認すること。
  • ログ出力のパフォーマンスを考慮し、呼び出し側で APLOG_IS_LEVEL 等による事前チェックを行うことが推奨される。
