socache_dbm_init

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:119

目的

DBM ベースの共有オブジェクトキャッシュ（socache）インスタンスを初期化し、バックエンドとなる DBM ファイルの作成、権限設定、および初期の期限切れ処理を実行する。

シグネチャまたは形状

apr_status_t socache_dbm_init(ap_socache_instance_t *ctx,
                              const char *namespace,
                              const struct ap_socache_hints *hints,
                              server_rec *s, apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンスのコンテキスト（データファイルパス等を保持）。
  • namespace: キャッシュの識別子。
  • hints: 期限切れ間隔などの設定ヒント。
  • s: サーバーレコード（ログ出力用）。
  • p: メモリプール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS を返し、失敗時はエラーコードを返す。

前提条件と不変条件

  • ctx->data_file が未設定の場合、namespace を用いてデフォルトパスが生成される。
  • ctx->pool は初期化の過程でクリアされる。
  • DBM ファイルは APR_DBM_RWCREATE モードでオープンされる。

アルゴリズムまたは振る舞い

1. **パス生成**: ctx->data_file が空であれば、DEFAULT_DBM_PREFIX と namespace を結合してパスを生成する。 2. **DBM オープン**: apr_dbm_open（または apr_dbm_open2）を使用して DBM ファイルを作成・オープンし、即座にクローズしてファイル作成の可否を確認する。 3. **設定適用**: hints から expiry_interval を設定する（デフォルトは 30 秒）。 4. **権限管理**: スーパーユーザー権限で実行されている場合、try_chown を呼び出し、生成された DBM ファイル（および関連するサフィックスファイル）の所有権を Apache 子プロセスがアクセス可能なものに変更する。 5. **クリーンアップ**: socache_dbm_expire を呼び出し、既存の期限切れエントリを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dbm.c:744（直接の呼び出し元として確認）。
  • **呼び出し先**: ap_runtime_dir_relative, apr_dbm_open, apr_dbm_close, try_chown, socache_dbm_expire 等。

関連データ構造

  • ap_socache_instance_t: キャッシュのコンテキストを保持する主要構造体。
  • apr_dbm_t: APR が提供する DBM ファイル操作用ハンドル。

失敗モードと境界条件

  • **パス生成失敗**: ap_runtime_dir_relative が失敗した場合、APR_EINVAL を返す。
  • **DBM オープン失敗**: ファイル作成権限がない場合やパスが無効な場合、エラーログを出力し、エラーコードまたは DECLINED を返す。
  • **権限変更失敗**: try_chown は内部でエラーを処理するが、ファイルシステム上の制約により所有権変更ができない場合がある。

保守担当者向けメモ

  • 本関数はスレッドセーフではないため、呼び出し側で適切な排他制御が行われていることを前提とする。
  • DBM ファイルのサフィックス（.dir, .pag, .db 等）はプラットフォーム依存であるため、try_chown による権限設定は複数の可能性を網羅する設計となっている。
