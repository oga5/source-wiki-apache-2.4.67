ap_reset_module_loglevels

定義

  • 種別: function
  • 定義箇所: server/config.c:1564

目的

指定されたログ設定構造体 (ap_logconf) 内のモジュール別ログレベル設定を、指定された値で初期化（リセット）します。

シグネチャまたは形状

void ap_reset_module_loglevels(struct ap_logconf *l, int val)

入力と出力

  • **入力**:
  • l: ログ設定を保持する ap_logconf 構造体へのポインタ。
  • val: モジュールごとのログレベルに設定する整数値。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • l が指す構造体は有効である必要があります。
  • conf_vector_length は、モジュール設定ベクトルの長さを定義するグローバルな定数として存在している必要があります。

アルゴリズムまたは振る舞い

1. l->module_levels が NULL でないことを確認します。 2. memset を使用し、l->module_levels が指すメモリ領域の先頭から conf_vector_length バイト分を、引数 val で指定された値で埋めます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/config.c:1576
  • server/config.c:1579
  • server/core.c:3483
  • **呼び出し先**:
  • memset (標準ライブラリ関数)

関連データ構造

  • struct ap_logconf: ログ設定を管理する構造体。module_levels メンバを介して各モジュールのログレベルを保持します。

失敗モードと境界条件

  • l->module_levels が NULL の場合、何も実行されません。
  • conf_vector_length のサイズを超えるメモリ操作は行われません。

保守担当者向けメモ

  • 本関数は、ログレベルをデフォルト値や未設定状態に一括リセットするために使用されます。
  • conf_vector_length は動的モジュールのロード数に依存するため、本関数が操作するメモリ範囲は実行時のモジュール構成に左右されます。
