dav_unlock_walker

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:386

目的

walk フック処理の一環として、指定されたリソースおよびその配下のリソースから間接ロックを再帰的に解除します。

シグネチャまたは形状

static dav_error * dav_unlock_walker(dav_walk_resource *wres, int calltype)

入力と出力

  • **入力**:
  • wres: 現在処理中のリソース情報を含む dav_walk_resource 構造体。
  • calltype: 呼び出しの種類を示す整数値。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時はエラー情報を含む dav_error 構造体へのポインタを返します。

前提条件と不変条件

  • wres->walk_ctx には、ロックデータベースのフックおよび解除対象のロックトークンを含む dav_walker_ctx が格納されている必要があります。

アルゴリズムまたは振る舞い

1. wres->resource->working が真の場合、dav_auto_checkin を呼び出し、ロック解除前に必要な自動チェックイン処理を実行します。 2. ctx->w.lockdb->hooks->remove_lock を呼び出し、現在のリソースから ctx->locktoken で指定されたロックを削除します。 3. 処理中にエラーが発生した場合は即座にエラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/util_lock.c 内の dav_unlock 関数から、walk フックのコールバックとして設定され呼び出されます。
  • **呼び出し先**:
  • dav_auto_checkin (条件付き)
  • ctx->w.lockdb->hooks->remove_lock (ロック削除フック)

関連データ構造

  • dav_walk_resource: ウォーカーが走査するリソースのコンテキスト。
  • dav_walker_ctx: ロックデータベースのフックやロックトークンを保持する構造体。
  • dav_resource: 操作対象のリソース表現。

失敗モードと境界条件

  • dav_auto_checkin または remove_lock フックがエラーを返した場合、そのエラーが伝播されます。
  • 失敗時の挙動として、現在の実装では処理を中断してエラーを返す設計となっています。

保守担当者向けメモ

  • dav_auto_checkin 呼び出し時の型キャスト（dav_resource *）は将来的に削除することが推奨されています。
  • エラー発生時の挙動について、現在の「即時停止」が適切か、あるいはマルチステータスレスポンスを生成すべきか検討の余地があります。
