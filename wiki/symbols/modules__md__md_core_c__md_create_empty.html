md_create_empty

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:103

目的

md_t 構造体のメモリを確保し、各フィールドをデフォルト値で初期化して空のマネージドドメインオブジェクトを作成します。

シグネチャまたは形状

md_t *md_create_empty(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（メモリ割り当てに使用する APR プール）
  • **出力**: 初期化された md_t 構造体へのポインタ。メモリ確保に失敗した場合は NULL を返します。

前提条件と不変条件

  • 引数 p は有効な apr_pool_t である必要があります。
  • 戻り値の md_t 構造体は、指定されたプール p の生存期間に依存します。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して md_t サイズのメモリをゼロクリアして確保します。 2. メモリ確保が成功した場合、以下の初期化を行います。

  • domains、contacts、acme_tls_1_domains 配列を初期サイズ 5 で作成。
  • renew_mode を MD_RENEW_DEFAULT に設定。
  • require_https を MD_REQUIRE_UNSET に設定。
  • must_staple、transitive、stapling、profile_mandatory、ari_renewals を -1 に設定。
  • defn_name を "unknown"、defn_line_number を 0 に設定。

    • 3. 初期化済みの構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_core.c:202
  • modules/md/md_core.c:333
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_array_make (APR)

関連データ構造

  • md_t: 本関数が生成するメインのデータ構造。
  • apr_pool_t: メモリ管理に使用される APR プール。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足により NULL を返した場合、関数はそのまま NULL を返します。

保守担当者向けメモ

  • 本関数は md_t のフィールド追加やデフォルト値の変更があった際に、必ず更新する必要があります。
  • domains 等の配列は初期サイズ 5 で作成されますが、必要に応じて自動拡張されます。
