cache_handle

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache.h:91

目的

キャッシュされたエンティティ（リクエストおよびレスポンスのヘッダー情報と、対応するキャッシュオブジェクト）へのアクセスを保持するための構造体です。

シグネチャまたは形状

struct cache_handle {
    cache_object_t *cache_obj;
    apr_table_t *req_hdrs;
    apr_table_t *resp_hdrs;
};

入力と出力

  • **入力**: 本構造体は、キャッシュされたコンテンツのメタデータおよび実体への参照を保持します。
  • **出力**: cache_object_t を通じてキャッシュの実体へ、apr_table_t を通じてキャッシュされた HTTP ヘッダーへアクセス可能です。

前提条件と不変条件

  • cache_obj は、キャッシュされたリソースのストレージ上の表現を指す必要があります。
  • req_hdrs および resp_hdrs は、キャッシュされたリクエストおよびレスポンスのヘッダーテーブルを指します。

アルゴリズムまたは振る舞い

本構造体は、キャッシュのヒット時にストレージから取得された情報を保持し、後続のフィルタチェーンやリクエスト処理において、キャッシュされたヘッダーやボディデータにアクセスするためのコンテキストとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache_disk.c:571 にて、キャッシュハンドルのコールバック関数初期化に関連する言及があります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • **補足**: modules/cache/mod_cache.h:90 にて cache_handle_t として typedef されています。これ以外の具体的な呼び出し関係は、提供されたソースからは特定できません。

関連データ構造

  • cache_handle_t: 本構造体の別名（typedef）。
  • cache_object_t: キャッシュされたオブジェクトの実体。
  • apr_table_t: Apache Portable Runtime (APR) のテーブル構造体。ヘッダーの保持に使用されます。

失敗モードと境界条件

  • 各ポインタが NULL である場合、無効なキャッシュハンドルとして扱われる可能性があります。
  • メモリ不足時、apr_table_t の操作が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cache のコアインターフェースの一部です。ストレージプロバイダ（mod_cache_disk 等）を実装する際は、このハンドルを通じてキャッシュデータがどのように公開されるかを考慮してください。
