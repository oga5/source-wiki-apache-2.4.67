merge_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_connect.c:65

目的

親サーバーの設定と、現在のコンテキスト（オーバーライド）の設定をマージし、新しい connect_conf 構造体を生成します。

シグネチャまたは形状

static void *merge_config(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する APR プール。
  • basev: 親サーバーの設定構造体（connect_conf 型）。
  • overridesv: 現在のコンテキストで上書きされる設定構造体（connect_conf 型）。
  • **出力**:
  • マージ済みの新しい connect_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は connect_conf 構造体としてキャスト可能であること。
  • apr_pool_t は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい connect_conf 構造体をプール p 上に割り当てます。 2. base（親）と overrides（上書き）の各 allowed_connect_ports を取得します。 3. apr_array_append を使用して、両者のポートリストを結合し、新しい構造体の allowed_connect_ports に格納します。 4. マージされた構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_connect.c:401 にて、サーバー設定のマージ処理として参照されています。
  • **呼び出し先**: apr_pcalloc および apr_array_append（APR ライブラリ関数）。

関連データ構造

  • connect_conf: mod_proxy_connect モジュールのサーバー設定を保持する構造体。
  • allowed_connect_ports: connect_conf 内の許可ポートリストを管理する apr_array_header_t。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが終了する可能性があります。
  • basev または overridesv が NULL の場合、apr_array_append の動作は APR の仕様に依存します。

保守担当者向けメモ

  • 本関数は AllowCONNECT ディレクティブで設定されたポート範囲を継承・統合するために使用されます。
  • mod_cache_socache.c:1542 にも同名のシンボル参照が見られますが、これは別のモジュール内の同名関数である可能性が高く、本モジュールのロジックとは直接関係ありません。
