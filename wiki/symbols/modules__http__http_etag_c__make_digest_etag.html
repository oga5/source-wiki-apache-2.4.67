make_digest_etag

定義

  • 種別: function
  • 定義箇所: modules/http/http_etag.c:94

目的

ファイル内容の SHA1 ハッシュを計算し、強力な（Strong）ETag 文字列を生成する。

シグネチャまたは形状

static char *make_digest_etag(request_rec *r, etag_rec *er, char *vlv,
                              apr_size_t vlv_len, char *weak, apr_size_t weak_len)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • er: ETag 生成に必要なファイル情報（ファイル記述子またはパス）。
  • vlv, vlv_len: バリデータ情報（VLV）とその長さ。
  • weak, weak_len: 弱 ETag プレフィックス（W/ 等）とその長さ。
  • **出力**:
  • 生成された ETag 文字列（r->pool から割り当て）。失敗時は空文字列 "" を返す。

前提条件と不変条件

  • er->fd または er->pathname が有効であること。
  • ファイルの読み込みおよびシーク操作が可能であること。
  • 生成される ETag は r->pool の生存期間に依存する。

アルゴリズムまたは振る舞い

1. **ファイル準備**: er->fd があれば使用し、なければ er->pathname を開く。 2. **シーク**: 現在のオフセットを保存し、ファイル先頭へ移動する。 3. **ハッシュ計算**: apr_brigade を使用してファイル内容を読み込み、apr_sha1 コンテキストでハッシュを算出する。

  • APR_HAS_MMAP が有効な場合、設定に応じてメモリマップを制御する。

    • 4. **復元**: ファイルの読み込み完了後、元のオフセット位置へシークを戻す。 5. **文字列構築**: etag_start でプレフィックスを付与し、SHA1 ダイジェストを Base64 エンコードして連結し、etag_end で VLV 情報を付与する。 6. **後処理**: 必要に応じてファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_etag.c:273（ap_make_etag_ex 内からの呼び出しが確認されている）。
  • **呼び出し先**: apr_file_open, apr_file_seek, apr_brigade_create, apr_brigade_insert_file, apr_sha1_init, apr_bucket_read, apr_sha1_update, apr_sha1_final, apr_base64_encode_binary, etag_start, etag_end 等。

関連データ構造

  • request_rec: リクエスト情報。
  • etag_rec: ETag 生成用メタデータ。
  • core_dir_config: FileETag 設定を含むディレクトリ設定。
  • apr_sha1_ctx_t: SHA1 ハッシュ計算用コンテキスト。
  • apr_bucket_brigade: ファイル読み込み用バケットブリゲード。

失敗モードと境界条件

  • ファイルのオープン、シーク、読み込みに失敗した場合、エラーログを出力して空文字列 "" を返す。
  • er->pathname を使用して開いたファイルは、処理終了時に必ず閉じる。

保守担当者向けメモ

  • 本関数はファイル全体を読み込むため、大きなファイルに対してはパフォーマンス上のコストが高い。
  • apr_palloc を使用しているため、メモリリークの心配はないが、リクエストごとのメモリ消費量に注意すること。
  • APR_HAS_MMAP 環境下での挙動は core_dir_config の設定に依存する。
