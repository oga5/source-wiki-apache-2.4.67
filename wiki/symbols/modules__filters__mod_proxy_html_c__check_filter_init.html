check_filter_init

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_proxy_html.c:822

目的

mod_proxy_html フィルタの実行コンテキスト (saxctxt) を初期化し、リクエストがフィルタ処理の対象として適切かどうかを検証する。

シグネチャまたは形状

static saxctxt *check_filter_init (ap_filter_t *f)

入力と出力

  • **入力**: ap_filter_t *f (現在のフィルタチェーン情報)
  • **出力**: 初期化済みの saxctxt * 構造体、またはフィルタ適用不要と判断された場合の NULL。

前提条件と不変条件

  • f->ctx が未設定である場合にのみ初期化処理が実行される。
  • proxy_html_conf 設定において links が定義されている必要がある。
  • リクエストがプロキシ経由であるか、または PROXY_HTML_FORCE 環境変数が設定されている必要がある。
  • コンテンツタイプが text/html または application/xhtml+xml である必要がある。

アルゴリズムまたは振る舞い

1. **検証**: プロキシリクエストか、コンテンツタイプが HTML か、設定ルールが存在するかを確認する。条件を満たさない場合は ap_remove_output_filter(f) を呼び出し、フィルタを解除して NULL を返す。 2. **コンテキスト生成**: apr_pcalloc を使用して saxctxt を確保し、f->ctx に割り当てる。 3. **リソース初期化**:

  • apr_brigade_create でバケットブリゲードを作成。
  • Content-Length ヘッダーを削除（置換によるサイズ変動のため）。

    • 4. **ルール適用**: 設定に応じて fixup_rules を実行し、動的ルールを評価するか、静的マップを割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_proxy_html.c:898 (proxy_html_filter 内)
  • **呼び出し先**: ap_get_module_config, apr_table_get, ap_remove_output_filter, apr_pcalloc, apr_brigade_create, apr_table_unset, fixup_rules

関連データ構造

  • saxctxt: フィルタ実行時の状態保持構造体。
  • proxy_html_conf: モジュール設定構造体。
  • ap_filter_t: Apache フィルタフレームワークの構造体。

失敗モードと境界条件

  • **フィルタ解除**: プロキシリクエストでない、コンテンツタイプが不一致、または置換ルールが未設定の場合、フィルタは自身を削除し処理を中断する。
  • **ログ出力**: GO_FASTER マクロが定義されていない場合、フィルタ適用除外の理由がトレースログに出力される。

保守担当者向けメモ

  • charset_out の処理は、charset_in のスニッフィング後に遅延して行われる設計となっている。
  • fctx->bb はリクエストプール (f->r->pool) に紐付いているため、リクエスト終了時に自動的に解放される。
