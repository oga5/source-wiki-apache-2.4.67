util_ald_cache_fetch

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache_mgr.c:438

目的

指定されたキャッシュから、ハッシュテーブルの検索キー（payload）に一致するエントリを検索し、存在する場合はそのデータを返却する。

シグネチャまたは形状

void *util_ald_cache_fetch(util_ald_cache_t *cache, void *payload)

入力と出力

  • **入力**:
  • cache: 検索対象の util_ald_cache_t 構造体へのポインタ。
  • payload: 検索キーとして使用されるデータ構造へのポインタ。
  • **出力**:
  • 一致するエントリが見つかった場合: そのエントリのペイロード（void *）を返却。
  • 見つからない場合、または cache が NULL の場合: NULL を返却。

前提条件と不変条件

  • cache が NULL の場合、関数は安全に NULL を返す。
  • cache 構造体内の hash 関数および compare 関数ポインタが適切に初期化されている必要がある。
  • 共有メモリ環境下では、返却されるポインタは apr_rmm 管理下の有効なアドレスである。

アルゴリズムまたは振る舞い

1. cache が NULL であれば直ちに NULL を返す。 2. キャッシュの統計情報 fetches をインクリメントする。 3. cache->hash 関数を用いて payload からハッシュ値を計算し、バケットインデックス（hashval）を算出する。 4. cache->nodes[hashval] を起点に連結リストを走査する。 5. cache->compare 関数を用いて、リスト内のノードと payload を比較する。 6. 一致するノードが見つかった場合、cache->hits をインクリメントし、そのノードのペイロードを返す。 7. リストの末尾まで一致が見つからなければ NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c 内の各所（util_url_node_t の検索、dn_compare_cache や compare_cache の検索など、計12箇所で確認）。
  • **呼び出し先**:
  • cache->hash: キャッシュ構造体に登録されたハッシュ計算関数。
  • cache->compare: キャッシュ構造体に登録された比較関数。

関連データ構造

  • util_ald_cache_t: キャッシュの管理構造体。ハッシュテーブルの配列と統計情報を含む。
  • util_cache_node_t: ハッシュテーブルの各バケットに連結されるノード構造体。

失敗モードと境界条件

  • cache が NULL の場合、エラーとして NULL を返す。
  • キャッシュが空、または該当するハッシュバケットにノードが存在しない場合、走査は即座に終了し NULL を返す。
  • ハッシュ衝突が発生した場合、連結リストを順次走査して比較を行うため、リストの長さに応じた計算コストが発生する。

保守担当者向けメモ

  • 本関数はキャッシュの読み取り専用操作を行う。
  • 統計情報（fetches, hits）の更新はアトミックではないため、マルチスレッド環境下での厳密な整合性が必要な場合は注意が必要である。
