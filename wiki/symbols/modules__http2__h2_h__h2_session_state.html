h2_session_state

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2.h:121

目的

HTTP/2 セッションのライフサイクルにおける現在の状態を定義し、状態遷移マシンによる制御を可能にする。

シグネチャまたは形状

typedef enum {
    H2_SESSION_ST_INIT,
    H2_SESSION_ST_DONE,
    H2_SESSION_ST_IDLE,
    H2_SESSION_ST_BUSY,
    H2_SESSION_ST_WAIT,
    H2_SESSION_ST_CLEANUP,
} h2_session_state;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力を持たない。
  • h2_session_state_str() 関数を通じて、列挙値を人間が読み取り可能な文字列へ変換可能。

前提条件と不変条件

  • セッションは常にこれら6つの状態のいずれか一つに属する。
  • h2_session 構造体の state メンバとして保持され、セッションの生存期間を通じて管理される。

アルゴリズムまたは振る舞い

  • h2_session.c 内の transit() 関数により、セッションのアクションに応じて状態遷移が実行される。
  • 各状態の意味:
  • H2_SESSION_ST_INIT: 初期設定（SETTINGSフレーム送信等）中。
  • H2_SESSION_ST_DONE: 処理完了、接続切断準備。
  • H2_SESSION_ST_IDLE: 書き込み待ち、データ受信待機中。
  • H2_SESSION_ST_BUSY: 読み書きが継続的に発生している状態。
  • H2_SESSION_ST_WAIT: クライアントからの入力と、内部処理からの出力の両方を待機中。
  • H2_SESSION_ST_CLEANUP: メモリプール等のクリーンアップ中。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_session.c および modules/http2/h2_session.h がこの型を参照し、状態遷移や文字列変換を行う。
  • 呼び出し先: 本シンボルは型定義であるため、直接的な呼び出し先は存在しない。

関連データ構造

  • h2_session: セッション管理のメイン構造体。state メンバとして本型を保持。

失敗モードと境界条件

  • 未定義の列挙値が代入された場合の動作は未定義。
  • 状態遷移の順序は h2_session.c の実装ロジックに依存しており、不正な遷移が発生した際の挙動は当該ソースコードのロジックに準拠する。

保守担当者向けメモ

  • 新たな状態を追加する場合は、h2_session_state_str() 関数にも対応する文字列定義を追加すること。
  • 状態遷移のロジックを変更する際は、h2_session.c 内の transit 関数および関連する状態判定ロジックを必ず確認すること。
