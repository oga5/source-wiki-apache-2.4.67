dav_generic_get_lockdb_path

定義

  • 種別: function
  • 定義箇所: modules/dav/lock/mod_dav_lock.c:34

目的

現在のリクエストコンテキストに関連付けられた、DAV ロックデータベースのファイルパスを取得します。

シグネチャまたは形状

const char *dav_generic_get_lockdb_path(const request_rec *r)

入力と出力

  • **入力**: r (Apache のリクエストレコード request_rec へのポインタ)
  • **出力**: ロックデータベースのパスを示す const char * 型の文字列。

前提条件と不変条件

  • r->per_dir_config に dav_lock_module に関連付けられた設定構造体 dav_lock_dir_conf が格納されている必要があります。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、リクエストのディレクトリ設定から dav_lock_module の設定構造体を取得します。 2. 取得した設定構造体 conf 内の lockdb_path メンバを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/lock/locks.c:387 にて、ロックデータベースのパスを初期化する際に呼び出されています。
  • **呼び出し先**: Apache API の ap_get_module_config を呼び出します。

関連データ構造

  • dav_lock_dir_conf: ロックデータベースのパスを保持するディレクトリ設定構造体。
  • request_rec: Apache のリクエストコンテキスト。

失敗モードと境界条件

  • r が NULL の場合や、リクエストに dav_lock_module の設定が含まれていない場合の挙動については、ソースコード内に明示的なエラーハンドリングの記述はありません。

保守担当者向けメモ

  • 本関数は locks.h で宣言されており、mod_dav のロックプロバイダーがデータベースの場所を特定するために使用されます。
  • パスは DAVGenericLockDB ディレクティブによって設定されます。
