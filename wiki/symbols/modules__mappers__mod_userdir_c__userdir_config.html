userdir_config

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_userdir.c:90

目的

mod_userdir モジュールにおける設定情報を保持するための構造体です。ユーザーディレクトリ機能の有効・無効状態や、マッピング先のパス設定を管理します。

シグネチャまたは形状

typedef struct {
    int globally_disabled;
    const char *userdir;
    apr_table_t *enabled_users;
    apr_table_t *disabled_users;
} userdir_config;

入力と出力

本構造体は直接的な関数ではありませんが、設定ディレクティブ（UserDir 等）の解析結果を格納し、リクエスト処理時の判断材料として出力（参照）されます。

前提条件と不変条件

  • enabled_users および disabled_users は apr_table_t 型であり、ユーザー名のリストを保持するために使用されます。
  • globally_disabled はフラグとして機能し、全ユーザーに対する機能を一括で制御します。

アルゴリズムまたは振る舞い

  • set_user_dir 関数等を通じて、設定ファイルから読み込まれた値が本構造体の各メンバに格納されます。
  • リクエスト処理時には、この構造体を参照して、対象ユーザーが有効化されているか、あるいは無効化されているかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_userdir_config (初期化)
  • merge_userdir_config (設定のマージ)
  • set_user_dir (設定値のセット)
  • translate_userdir (リクエスト処理時の参照)
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • apr_table_t: ユーザー名のリスト管理に使用される Apache Portable Runtime のテーブル構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc によるメモリ確保が失敗した場合、プロセスが終了する可能性があります。
  • 設定の競合: enabled_users と disabled_users の両方に同一ユーザーが指定された場合の優先順位は、モジュールの実装ロジックに依存します。

保守担当者向けメモ

  • 本構造体は mod_userdir の設定状態を保持する中心的な役割を担います。
  • UserDir ディレクティブの挙動を変更する際は、本構造体のメンバ定義と、それを利用する translate_userdir 内の判定ロジックを併せて確認してください。
