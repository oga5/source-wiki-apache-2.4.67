ap_bucket_error_create

定義

  • 種別: function
  • 定義箇所: server/error_bucket.c:55

目的

HTTPレスポンス処理中に発生したエラー情報を保持する、新しいエラーバケットを生成および初期化します。

シグネチャまたは形状

apr_bucket *ap_bucket_error_create(int error, const char *buf,
                                   apr_pool_t *p,
                                   apr_bucket_alloc_t *list)

入力と出力

  • **入力**:
  • error: HTTPステータスコード。
  • buf: エラーメッセージ文字列（NULL可）。
  • p: メモリ割り当てに使用するプール。
  • list: バケット割り当て用のアロケータ。
  • **出力**:
  • 初期化済みの apr_bucket 構造体へのポインタ。

前提条件と不変条件

  • list は有効な apr_bucket_alloc_t である必要があります。
  • error が有効なHTTPレスポンスコードでない場合、内部的に HTTP_INTERNAL_SERVER_ERROR に置換されます。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を使用して、バケット構造体用のメモリを確保します。 2. APR_BUCKET_INIT を呼び出し、バケットの初期化を行います。 3. バケットの free メソッドに apr_bucket_free を設定し、アロケータを割り当てます。 4. ap_is_HTTP_VALID_RESPONSE を用いて error の妥当性を検証し、不正な場合は HTTP_INTERNAL_SERVER_ERROR に修正します。 5. ap_bucket_error_make を呼び出し、エラーコードとメッセージをバケットに格納して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_ext_filter.c
  • modules/filters/mod_request.c
  • modules/http/byterange_filter.c
  • modules/http/http_filters.c
  • modules/http2/h2_bucket_beam.c
  • modules/http2/h2_c2_filter.c
  • modules/http2/h2_proxy_session.c
  • modules/proxy/proxy_util.c
  • server/protocol.c
  • test/modules/http2/mod_h2test/mod_h2test.c
  • **呼び出し先**:
  • ap_bucket_error_make (直接の初期化処理)

関連データ構造

  • apr_bucket: APRフレームワークの汎用バケット構造体。
  • ap_bucket_type_error: エラーバケットの型定義。

失敗モードと境界条件

  • メモリ確保失敗時: apr_bucket_alloc が失敗した場合の挙動はAPRの仕様に依存します。
  • 不正なステータスコード: ap_is_HTTP_VALID_RESPONSE が偽を返す場合、自動的に 500 Internal Server Error へフォールバックされます。

保守担当者向けメモ

  • 本関数はメタデータバケットを生成する公開APIです。
  • バケットのコピーや分割はサポートされていないため、フィルタチェーン内での取り扱いに注意してください。
