cleanup_ctx

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_brotli.c:180

目的

Brotli エンコーダのインスタンスを破棄し、関連するメモリリソースを解放する。

シグネチャまたは形状

static apr_status_t cleanup_ctx(void *data)

入力と出力

  • **入力**: void *data - brotli_ctx_t 構造体へのポインタ。
  • **出力**: apr_status_t - 常に APR_SUCCESS を返す。

前提条件と不変条件

  • 引数 data は有効な brotli_ctx_t 構造体を指している必要がある。
  • ctx->state が BrotliEncoderCreateInstance によって初期化済みであること。

アルゴリズムまたは振る舞い

1. 引数 data を brotli_ctx_t * 型にキャストする。 2. BrotliEncoderDestroyInstance(ctx->state) を呼び出し、エンコーダの状態を破棄する。 3. ctx->state を NULL に設定し、二重解放を防止する。 4. APR_SUCCESS を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_brotli.c:201: apr_pool_cleanup_register を介してプールクリーンアップ時に登録される。
  • modules/filters/mod_brotli.c:531: apr_pool_cleanup_run を介して明示的に呼び出される。
  • **呼び出し先**:
  • BrotliEncoderDestroyInstance (Brotli ライブラリ関数)

関連データ構造

  • brotli_ctx_t: リクエスト単位のコンテキストを保持する構造体。本関数はこの中の state メンバを操作する。

失敗モードと境界条件

  • ctx が NULL の場合、ctx->state へのアクセスによりセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は Apache のプールクリーンアップ機構と連携して動作する。リクエスト終了時に BrotliEncoderDestroyInstance を確実に呼び出すことで、メモリリークを防止する役割を担っている。
