cache_generate_key_default

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_storage.c:665

目的

リクエスト URI およびクエリ文字列に基づき、キャッシュの識別に使用する一意なキーを生成するデフォルトの実装を提供します。

シグネチャまたは形状

apr_status_t cache_generate_key_default(request_rec *r, apr_pool_t* p, const char **key)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • p: キー生成に使用するメモリプール。
  • **出力**:
  • key: 生成されたキャッシュキー文字列へのポインタ。
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • r->parsed_uri が適切に初期化されている必要があります。
  • キャッシュキーの生成ロジックは、リクエストの処理フェーズ（初期段階か否か）に依存します。

アルゴリズムまたは振る舞い

1. cache_use_early_url(r) を呼び出し、リクエストが初期処理段階（クイックハンドラやフォワードプロキシなど）にあるか判定します。 2. 初期処理段階であれば、r->parsed_uri からパスとクエリ文字列を取得します。それ以外の場合は、現在の r->uri および r->args を使用します。 3. 取得したパスとクエリ文字列を引数として cache_canonicalise_key を呼び出し、正規化されたキャッシュキーを生成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c:2523: cache_generate_key 関数ポインタへの代入先として参照されています。
  • **呼び出し先**:
  • cache_use_early_url (判定用)
  • cache_canonicalise_key (キー生成の核となる内部関数)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • apr_pool_t: メモリ管理に使用されるプール。

失敗モードと境界条件

  • cache_canonicalise_key が失敗した場合、そのステータスコードがそのまま返されます。
  • r->parsed_uri が未設定の場合、正規化プロセスで予期しない動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は cache_generate_key インターフェースのデフォルト実装です。カスタムキー生成が必要な場合は、この関数をラップまたは置換する実装が検討されます。
  • cache_canonicalise_key の複雑な正規化ロジックに依存しているため、ホスト名やポートの解釈変更時には注意が必要です。
