log_bad_create_options

定義

  • 種別: function
  • 定義箇所: server/util_mutex.c:374

目的

ミューテックス生成時に不正なオプションが指定された際、エラーログを出力する。

シグネチャまたは形状

static void log_bad_create_options(server_rec *s, const char *type)

入力と出力

  • **入力**:
  • s: ログ出力対象となる server_rec 構造体へのポインタ。
  • type: 不正なオプションが指定されたミューテックスの型を示す文字列。
  • **出力**: なし（void）。

前提条件と不変条件

  • s は有効な server_rec 構造体を指している必要がある。
  • type は NULL でない文字列であること。

アルゴリズムまたは振る舞い

1. ap_log_error を呼び出し、APLOG_EMERG レベルでエラーログを記録する。 2. ログメッセージには APLOGNO(00021) が付与され、「Invalid options were specified when creating the %s mutex」という形式で、引数 type を埋め込んだメッセージを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_mutex.c:419
  • server/util_mutex.c:466
  • **呼び出し先**:
  • ap_log_error (Apache HTTP Server のログ出力 API)

関連データ構造

  • server_rec: サーバー設定情報を保持する構造体。

失敗モードと境界条件

  • s が NULL の場合、ap_log_error の仕様に従い、ログ出力時にセグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数は static であり、server/util_mutex.c 内でのみ使用される。
  • ログ番号 00021 はこのエラーメッセージ専用に割り当てられているため、変更時には注意が必要。
