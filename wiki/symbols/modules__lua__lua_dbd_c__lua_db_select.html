lua_db_select

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:308

目的

Lua スクリプトから SQL の SELECT クエリを実行し、結果セットを Lua テーブルとしてラップして返す。

シグネチャまたは形状

int lua_db_select(lua_State *L);
  • 引数: Lua スタック上に request_rec オブジェクト（インデックス2）と SQL ステートメント文字列（インデックス3）を要求する。

入力と出力

  • **入力**:
  • request_rec: ap_lua_check_request_rec を介して検証されたリクエストコンテキスト。
  • statement: 実行する SQL クエリ文字列。
  • **出力**:
  • 成功時: 結果セットを含む Lua テーブルをスタックに積む（戻り値 1）。
  • 結果が空の場合: 0 を返す。
  • 失敗時: nil とエラーメッセージをスタックに積む（戻り値 2）、または false を返す（戻り値 1）。

前提条件と不変条件

  • lua_get_db_handle によって取得されるデータベースハンドルが有効（db->alive が真）であること。
  • request_rec が有効であること。

アルゴリズムまたは振る舞い

1. ap_lua_check_request_rec でリクエストを検証し、SQL 文を取得する。 2. apr_dbd_select を呼び出し、データベースから結果セットを取得する。 3. 結果セットが存在する場合、lua_newtable でテーブルを作成し、lua_db_result_set 構造体をユーザーデータとして生成する。 4. lua_apr.dbselect メタテーブルを適用し、__call メタメソッドに lua_db_get_row を割り当てることで、Lua 側から行データへのアクセスを可能にする。 5. 成功時は結果テーブルをスタックに積んで終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_dbd.c:694 にて Lua の関数として登録されている。
  • **呼び出し先**:
  • ap_lua_check_request_rec
  • lua_get_db_handle
  • apr_dbd_select
  • apr_dbd_num_cols
  • apr_dbd_num_tuples
  • lua_db_get_row (メタメソッドとして登録)
  • apr_dbd_error

関連データ構造

  • lua_db_handle: データベース接続状態を保持。
  • lua_db_result_set: 結果セットのメタデータ（カラム数、行数、結果ポインタ）を保持。
  • lua_apr.dbselect: 結果セット操作用のメタテーブル。

失敗モードと境界条件

  • db が無効または alive でない場合: false を返し終了する。
  • apr_dbd_select が失敗した場合: apr_dbd_error を取得し、nil とエラーメッセージを返して終了する。
  • カラム数が 0 の場合: 0 を返し、結果セットを生成しない。

保守担当者向けメモ

  • 本関数は lua_db_result_set を生成し、Lua のガベージコレクションと連携する __gc 処理の基盤となる。
  • 結果セットの行アクセスは lua_db_get_row に委譲されているため、行取得ロジックを変更する場合はそちらを確認すること。
