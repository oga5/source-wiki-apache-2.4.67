literal

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.c:1596

目的

文字列とその長さをペアとして保持するための軽量なデータ構造です。主に、特定のヘッダー名などを静的な配列として定義し、比較や検索を行う際に使用されます。

シグネチャまたは形状

typedef struct {
    const char *name;
    size_t len;
} literal;

入力と出力

  • **入力**: name（文字列へのポインタ）、len（文字列の長さ）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • name はヌル終端文字列を指すことが想定されます。
  • len は name が指す文字列の長さを保持する必要があります。

アルゴリズムまたは振る舞い

この構造体自体はデータ保持のみを行います。関連する処理として、h2_proxy_util.c 等で定義される ignore_header 関数のように、literal の配列を走査して特定のヘッダー名と一致するかを判定する用途で利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_util.c: IgnoredRequestHeaders 配列の定義に使用。
  • modules/http2/h2_proxy_util.c: IgnoredRequestHeaders および IgnoredProxyRespHds 配列の定義、および ignore_header 関数での引数として使用。
  • **呼び出し先**: 情報なし。

関連データ構造

  • IgnoredRequestHeaders（literal 型の静的配列）
  • IgnoredProxyRespHds（literal 型の静的配列）

失敗モードと境界条件

  • len が実際の文字列長と一致しない場合、比較処理において誤判定やメモリ安全性の問題を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は h2_util.c と h2_proxy_util.c の両方で定義されています。変更を加える際は、両方のファイルでの整合性に注意してください。
  • 汎用的な構造体ですが、主に HTTP ヘッダーのフィルタリングリストを管理するために利用されています。
