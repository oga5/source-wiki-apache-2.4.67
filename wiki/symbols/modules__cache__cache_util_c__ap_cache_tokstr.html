ap_cache_tokstr

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_util.c:809

目的

カンマ区切りの文字列リストから、最初のトークンを切り出して返します。主に HTTP ヘッダー値の解析に使用されます。

シグネチャまたは形状

const char *ap_cache_tokstr(apr_pool_t *p, const char *list, const char **str);

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • list: 解析対象のカンマ区切り文字列。
  • str: 次のトークンの開始位置を格納するためのポインタ。
  • **出力**:
  • 戻り値: 切り出された最初のトークン（apr_pstrmemdup によりプールから割り当てられた文字列）。トークンが存在しない場合は NULL。
  • *str: リスト内の次のトークンの開始位置（カンマ以降の空白を除去した位置）。

前提条件と不変条件

  • list は NULL であってはなりません。
  • 戻り値の文字列は、呼び出し元のリクエストプール p の生存期間に依存します。

アルゴリズムまたは振る舞い

1. list 内の最初のカンマ位置を検索します。 2. カンマが見つかった場合、その位置までをトークンとして扱い、カンマ以降の空白をスキップして次のトークンの開始位置を *str に設定します。 3. カンマが見つからない場合、文字列全体をトークンとし、*str には NULL を設定します。 4. トークンの前後にある空白をトリミングします。 5. 切り出したトークンを apr_pstrmemdup で複製して返します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt には具体的な呼び出し元は記載されていません。
  • 呼び出し先: ap_strchr_c、apr_isspace、strlen、apr_pstrmemdup を呼び出します。

関連データ構造

  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • list が空文字列の場合、戻り値は NULL となり、*str には NULL が設定されます。
  • トークンが空白のみで構成されている場合、戻り値は NULL となる可能性があります。

保守担当者向けメモ

  • 本関数は modules/cache/mod_cache.h:139 で宣言されています。
  • apr_pstrmemdup を使用しているため、呼び出し側でメモリリークを気にする必要はありませんが、大量のトークンを処理する場合はプール消費量に注意してください。
