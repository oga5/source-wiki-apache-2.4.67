stapling_get_cached_response

定義

  • **種別**: 関数
  • **定義箇所**: modules/ssl/ssl_util_stapling.c:324

目的

OCSP Stapling 用のキャッシュから、特定の証明書に対応する OCSP レスポンスを取得し、デコードして呼び出し元へ提供する。

シグネチャまたは形状

static void stapling_get_cached_response(server_rec *s, OCSP_RESPONSE **prsp,
                                         BOOL *pok, certinfo *cinf,
                                         apr_pool_t *pool)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • cinf: キャッシュキー（idx）を含む証明書情報構造体。
  • pool: メモリ割り当て用 APR プール。
  • **出力**:
  • prsp: 取得・デコードされた OCSP_RESPONSE オブジェクトへのポインタ（キャッシュミス時は NULL）。
  • pok: レスポンスが「有効」として保存されていたかを示すフラグ（TRUE または FALSE）。

前提条件と不変条件

  • mc->stapling_cache が初期化されていること。
  • cinf->idx が有効なキャッシュキーとして設定されていること。
  • MAX_STAPLING_DER（10KB）を超えるレスポンスはキャッシュから取得できない。

アルゴリズムまたは振る舞い

1. キャッシュバックエンドがスレッドセーフでない場合、stapling_cache_mutex_on を用いて排他制御を行う。 2. mc->stapling_cache->retrieve を呼び出し、キャッシュから DER 形式のデータを取得する。 3. 取得したデータの先頭 1 バイトを「有効フラグ」として読み取り、残りのデータを d2i_OCSP_RESPONSE で OpenSSL 構造体に変換する。 4. 処理終了後、必要に応じて stapling_cache_mutex_off で排他制御を解除する。 5. 成功時は *prsp にレスポンスを格納し、失敗時はログを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_util_stapling.c:744（stapling_cb 内での呼び出しが確認されている）。
  • **呼び出し先**:
  • stapling_cache_mutex_on / off
  • mc->stapling_cache->retrieve
  • d2i_OCSP_RESPONSE（OpenSSL API）
  • ap_log_error

関連データ構造

  • certinfo: 証明書とキャッシュキーを保持。
  • SSLModConfigRec: モジュールの設定とキャッシュバックエンドへのアクセスを保持。
  • OCSP_RESPONSE: OpenSSL の OCSP レスポンス構造体。

失敗モードと境界条件

  • **キャッシュミス**: rv != APR_SUCCESS の場合、ログを出力して終了する。
  • **データ長異常**: resp_derlen <= 1 の場合、不正なキャッシュデータとしてエラーログを出力する。
  • **パース失敗**: d2i_OCSP_RESPONSE が NULL を返した場合、パースエラーとしてログを出力する。

保守担当者向けメモ

  • キャッシュバックエンドの AP_SOCACHE_FLAG_NOTMPSAFE フラグに基づき、明示的なミューテックス操作が行われる点に注意すること。
  • resp_der バッファはスタック上に MAX_STAPLING_DER サイズで確保されている。
