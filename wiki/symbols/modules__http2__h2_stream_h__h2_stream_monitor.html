h2_stream_monitor

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_stream.h:55

目的

HTTP/2 ストリームの状態遷移やイベント発生を外部から監視するためのコールバック構造体です。

シグネチャまたは形状

typedef struct h2_stream_monitor {
    void *ctx;
    h2_stream_state_cb *on_state_enter;
    h2_stream_state_cb *on_state_invalid;
    h2_stream_event_cb *on_state_event;
    h2_stream_event_cb *on_event;
} h2_stream_monitor;

入力と出力

  • **入力**: 監視対象のストリームに関連付けられたコンテキスト（ctx）および各イベントに対応するコールバック関数ポインタ。
  • **出力**: 状態遷移やイベント発生時に、登録されたコールバック関数を通じて通知が行われます。

前提条件と不変条件

  • h2_stream 構造体内にオプションとして保持されます。
  • 各コールバックは、ストリームのライフサイクルにおける特定のタイミング（状態遷移時やイベント発生時）に呼び出されます。

アルゴリズムまたは振る舞い

  • on_state_enter: 新しい状態に遷移した際に呼び出されます。
  • on_state_invalid: 不正な状態遷移が検出された際に呼び出されます。
  • on_state_event: ストリームの状態を変化させるイベントの直前に呼び出されます。
  • on_event: 状態遷移を伴わないイベントが発生した際に呼び出されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_session.c にてインスタンスが生成され、h2_stream_set_monitor を通じてストリームに設定されます。
  • **呼び出し先**: 状態遷移やイベント発生時に、ストリーム処理ロジックから各コールバック関数が実行されます（具体的なコールバック関数の実装は本構造体定義外に依存します）。

関連データ構造

  • h2_stream: 本構造体を保持し、監視対象となるメイン構造体。
  • h2_session: 本構造体のインスタンスを管理するセッション層。

失敗モードと境界条件

  • コールバック関数ポインタが NULL の場合、対応するイベント発生時の通知はスキップされます。
  • 状態遷移の整合性チェックは h2_stream_dispatch 等の呼び出し元ロジックに依存しており、本構造体自体は通知の仲介のみを行います。

保守担当者向けメモ

  • 本構造体は h2_stream のデバッグや統計情報の収集に利用されます。
  • h2_stream_set_monitor 関数を使用して、ストリーム生成後に監視用コールバックを動的に割り当てることが可能です。
