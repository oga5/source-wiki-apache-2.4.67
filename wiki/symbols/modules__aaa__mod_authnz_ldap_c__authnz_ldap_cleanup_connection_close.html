authnz_ldap_cleanup_connection_close

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_ldap.c:369

目的

LDAP 接続のクリーンアップ処理として、リクエスト終了時に接続を確実にクローズするために使用されます。

シグネチャまたは形状

apr_status_t authnz_ldap_cleanup_connection_close(void *param)

入力と出力

  • **入力**: void *param - util_ldap_connection_t 型の LDAP 接続オブジェクトへのポインタ。
  • **出力**: apr_status_t - 常に APR_SUCCESS を返します。

前提条件と不変条件

  • param は有効な util_ldap_connection_t 構造体を指している必要があります。
  • 本関数は apr_pool_cleanup_register を介してプールクリーンアップハンドラとして登録されることを想定しています。

アルゴリズムまたは振る舞い

1. 引数 param を util_ldap_connection_t 型にキャストします。 2. util_ldap_connection_close(ldc) を呼び出し、LDAP 接続をクローズします。 3. APR_SUCCESS を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_authnz_ldap.c 内の各所（行 674, 829, 981, 986, 1059, 1186, 1319, 1409, 1474 付近）。主に apr_pool_cleanup_register の引数として渡されるか、直接呼び出されています。
  • **呼び出し先**: util_ldap_connection_close (外部モジュール mod_ldap が提供)。

関連データ構造

  • util_ldap_connection_t: mod_ldap が管理する LDAP 接続状態を保持する構造体。

失敗モードと境界条件

  • param が NULL の場合、util_ldap_connection_close 内部の挙動に依存します。
  • 戻り値は常に APR_SUCCESS であり、クローズ処理自体の成否は呼び出し元へ伝播されません。

保守担当者向けメモ

  • 本関数は、リクエストのライフサイクル終了時に LDAP 接続を解放するためのコールバック関数です。
  • apr_pool_cleanup_kill を使用して、明示的にクリーンアップをキャンセルする箇所（行 982, 1409）が存在するため、二重解放が発生しないよう注意してください。
