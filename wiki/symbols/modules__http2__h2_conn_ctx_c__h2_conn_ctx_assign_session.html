h2_conn_ctx_assign_session

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_conn_ctx.c:71

目的

HTTP/2 接続コンテキスト（h2_conn_ctx_t）に対し、対応する HTTP/2 セッション（h2_session）を割り当て、識別子を生成・設定します。

シグネチャまたは形状

void h2_conn_ctx_assign_session(h2_conn_ctx_t *ctx, struct h2_session *session)

入力と出力

  • **入力**:
  • ctx: 更新対象の h2_conn_ctx_t 構造体へのポインタ。
  • session: 割り当てる h2_session 構造体へのポインタ。
  • **出力**: なし（副作用として ctx のメンバを更新）。

前提条件と不変条件

  • ctx および session は有効なポインタである必要があります。
  • session->pool が有効であり、識別子生成のためのメモリ割り当てに使用されます。

アルゴリズムまたは振る舞い

1. ctx->session に引数の session を代入します。 2. session->child_num と session->id を使用して、session->pool 上に文字列形式の識別子（"%d-%lu"）を生成し、ctx->id に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c1.c:110 からの呼び出しが確認されています。
  • **呼び出し先**: apr_psprintf（APRライブラリ）を呼び出し、識別子文字列を生成します。

関連データ構造

  • h2_conn_ctx_t: 接続ごとの状態を保持する構造体。
  • h2_session: HTTP/2 セッションの状態を管理する構造体。

失敗モードと境界条件

  • session->pool が枯渇している場合、apr_psprintf によるメモリ割り当てが失敗する可能性があります。

保守担当者向けメモ

  • 本関数は h2_conn_ctx と h2_session の紐付けを行う重要なステップです。識別子 ctx->id はデバッグログ等で接続を特定するために使用されます。
