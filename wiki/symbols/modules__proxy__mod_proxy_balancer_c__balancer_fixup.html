balancer_fixup

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:405

目的

リクエストの転送先となるワーカーのURLとリクエストパスを結合し、プロキシ用の正規化されたURLを生成・更新します。

シグネチャまたは形状

static int balancer_fixup(request_rec *r, proxy_worker *worker, char **url)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • worker: 選択されたプロキシワーカー構造体。
  • url: 転送先URLへのポインタ。
  • **出力**:
  • int: 処理結果（OK またはエラーコード）。
  • url: 正規化された新しいURL文字列（更新される）。

前提条件と不変条件

  • worker および worker->s が有効であること。
  • r->pool がメモリ割り当てに利用可能であること。
  • 成功時、r->filename は proxy: で始まる文字列に設定されます。

アルゴリズムまたは振る舞い

1. *url からスキーム以降のパス部分を抽出します。 2. worker->s->name_ex（ワーカー名）と抽出したパスを結合し、r->filename を proxy:<worker_name><path> の形式で構築します。 3. ap_proxy_canon_url(r) を呼び出し、スキーム固有の正規化処理を実行します。 4. 正規化が成功した場合、r->filename から proxy: を除いた文字列を *url に格納して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_balancer.c:98 (コメント上の言及)
  • modules/proxy/mod_proxy_balancer.c:630
  • **呼び出し先**:
  • ap_proxy_canon_url (プロキシ基盤の正規化フック)

関連データ構造

  • request_rec: Apacheのリクエスト情報。
  • proxy_worker: 負荷分散対象のワーカー情報。

失敗モードと境界条件

  • ap_proxy_canon_url が OK 以外の値を返した場合、そのエラーコードがそのまま戻り値となります。
  • *url に :// が含まれない場合、パスの抽出に失敗し、意図しないURL生成が行われる可能性があります。

保守担当者向けメモ

  • 本関数はリクエスト処理の最終段階でURLを確定させるために使用されます。
  • r->filename の先頭に proxy: を付与する規約に依存しているため、変更時はプロキシ基盤側の期待値と整合させる必要があります。
