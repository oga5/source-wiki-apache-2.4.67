cgid_rlimit_t

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_cgid.c:178

目的

CGI プロセス実行時に適用するリソース制限（RLIMIT）の値を保持するための構造体です。

シグネチャまたは形状

typedef struct {
#ifdef RLIMIT_CPU
    int    limit_cpu_set;
    struct rlimit limit_cpu;
#endif
#if defined (RLIMIT_DATA) || defined (RLIMIT_VMEM) || defined(RLIMIT_AS)
    int    limit_mem_set;
    struct rlimit limit_mem;
#endif
#ifdef RLIMIT_NPROC
    int    limit_nproc_set;
    struct rlimit limit_nproc;
#endif
} cgid_rlimit_t;

入力と出力

  • **入力**: 本構造体は cgid_req_t の一部として、メインサーバーから CGI デーモンプロセスへ Unix ドメインソケット経由で転送されます。
  • **出力**: デーモンプロセス側で読み取られ、apr_proc_create を用いた CGI プロセス生成時のリソース制限設定に使用されます。

前提条件と不変条件

  • 各制限項目（CPU, メモリ, プロセス数）は、対応する OS のマクロ（RLIMIT_CPU 等）が定義されている場合にのみ有効となります。
  • limit_xxx_set フラグは、該当する制限が設定されているかどうかを示すブール値として機能します。

アルゴリズムまたは振る舞い

1. **シリアライズ**: メインサーバー側で sock_write を使用し、cgid_rlimit_t 構造体全体をソケットへ書き込みます。 2. **デシリアライズ**: CGI デーモン側で sock_read を使用し、ソケットから構造体データを読み取ります。 3. **適用**: デーモンは読み取ったフラグを確認し、設定されている制限値を CGI プロセスに適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgid.c: リクエスト送信時 (sock_write) および受信時 (sock_read) に参照されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • cgid_req_t: 本構造体をメンバとして保持し、デーモンへのリクエスト情報を構成します。
  • struct rlimit: OS 標準のリソース制限構造体。

失敗モードと境界条件

  • **通信エラー**: sock_read または sock_write が APR_SUCCESS を返さない場合、リソース制限情報の転送に失敗し、CGI 実行フローが中断されます。
  • **プラットフォーム依存**: コンパイル環境で RLIMIT_* マクロが定義されていない場合、該当するメンバは構造体に含まれず、制限を適用できません。

保守担当者向けメモ

  • 本構造体はバイナリ形式でソケット通信されるため、構造体のメンバ変更やパディングの差異がプロセス間通信の互換性に影響を与える可能性があります。変更時は注意してください。
