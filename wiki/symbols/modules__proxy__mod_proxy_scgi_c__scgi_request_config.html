scgi_request_config

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:60

目的

SCGI プロキシリクエスト処理中における、内部リダイレクトやファイル送信（X-Sendfile）に関連する指示情報を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const char *location;    /* target URL */
    scgi_request_type type;  /* type of request */
} scgi_request_config;

入力と出力

  • **入力**: リクエスト処理中に apr_palloc を用いて r->pool 上に割り当てられます。
  • **出力**: 構造体の各フィールド（location, type）は、後続の scgi_request_status フック等でリクエストの制御判断に使用されます。

前提条件と不変条件

  • この構造体は request_rec のメモリプール (r->pool) に紐付いており、リクエストの終了とともに自動的に解放されます。

アルゴリズムまたは振る舞い

  • リクエスト処理の過程で、バックエンドからのレスポンスヘッダーに基づき、内部リダイレクトやファイル送信が必要と判断された際にインスタンス化されます。
  • scgi_request_status フックにおいて、この構造体に格納された type（リクエストの種類）と location（ターゲットURL）を参照し、サブリクエストの実行などの制御が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_scgi.c 内の以下の箇所でインスタンス化（apr_palloc）されています。
  • 411行目
  • 445行目
  • 476行目
  • **呼び出し先**: scgi_request_status フックにおいて、リクエスト状態の判定のために参照されます。

関連データ構造

  • scgi_request_type: リクエストの種類を定義する列挙型または型定義。
  • request_rec: Apache HTTPd のリクエスト構造体（メモリプールの提供元）。

失敗モードと境界条件

  • メモリ割り当て失敗時: apr_palloc が失敗した場合、Apache の標準的なメモリ不足エラーハンドリングに従います。
  • 未初期化状態: scgi_request_config が割り当てられていない場合、scgi_request_status フックはリクエストの制御を行いません。

保守担当者向けメモ

  • 本構造体は mod_proxy_scgi におけるリクエスト固有の状態管理用です。
  • location フィールドは const char * であるため、文字列の生存期間が r->pool の生存期間と一致していることを確認してください。
