disk_cache_object_t

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache_disk.h:40

目的

ディスクベースのキャッシュモジュール（mod_cache_disk）において、キャッシュされた個々のリソースのメタデータおよびファイルハンドルを保持し、ディスク上の表現を管理するために使用されます。

シグネチャまたは形状

typedef struct disk_cache_object {
    const char *root;
    apr_size_t root_len;
    const char *prefix;
    disk_cache_file_t data;
    disk_cache_file_t hdrs;
    disk_cache_file_t vary;
    const char *hashfile;
    const char *name;
    const char *key;
    apr_off_t file_size;
    disk_cache_info_t disk_info;
    apr_table_t *headers_in;
    apr_table_t *headers_out;
    apr_off_t offset;
    apr_time_t timeout;
    unsigned int done:1;
} disk_cache_object_t;

入力と出力

本構造体はデータ保持用であり、関数ではありません。mod_cache_disk.c 内の各処理において、キャッシュの読み書きに必要なパス情報、ファイルハンドル、およびヘッダー情報を提供します。

前提条件と不変条件

  • mod_cache の cache_object_t 構造体の vobj メンバとして格納されることを前提としています。
  • done ビットフィールドは、キャッシュ保存処理の完了状態を示します。

アルゴリズムまたは振る舞い

  • キャッシュ対象のURIから生成されたハッシュ値（hashfile）に基づき、ディスク上のファイルパスを特定します。
  • data、hdrs、vary の各 disk_cache_file_t メンバを通じて、実データおよびメタデータのI/O操作を抽象化します。
  • headers_in および headers_out を用いて、保存対象のHTTPヘッダーを管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache_disk.c 内の複数の関数（例: キャッシュの初期化、検索、保存処理）でインスタンス化および参照が行われています。
  • **呼び出し先**: 本構造体自体はデータ保持構造体であり、直接的な関数呼び出しは行いません。

関連データ構造

  • disk_cache_file_t: ファイルI/Oを抽象化する構造体。
  • disk_cache_info_t: ディスクへシリアライズされるメタデータ情報。
  • cache_object_t: mod_cache が管理する上位のキャッシュオブジェクト。
  • apr_table_t: APRが提供するヘッダー保持用テーブル。

失敗モードと境界条件

  • done フラグが適切に更新されない場合、キャッシュの整合性が失われる可能性があります。
  • ファイルシステム上のパス制限や権限により、root や hashfile を用いたファイルアクセスが失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cache の汎用的なキャッシュオブジェクトと密結合しています。
  • disk_cache_file_t メンバの管理状況については、mod_cache_disk.c の実装を参照してください。
