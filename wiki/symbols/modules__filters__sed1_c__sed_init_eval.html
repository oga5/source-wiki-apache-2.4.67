sed_init_eval

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:294

目的

sed_eval_t 構造体を初期化し、実行に必要なコマンドセット、エラーハンドラ、出力関数、およびメモリプールを関連付ける。

シグネチャまたは形状

apr_status_t sed_init_eval(sed_eval_t *eval, sed_commands_t *commands, sed_err_fn_t *errfn, void *data, sed_write_fn_t *writefn, apr_pool_t* p)

入力と出力

  • **入力**:
  • eval: 初期化対象の sed_eval_t 構造体へのポインタ。
  • commands: コンパイル済みの sed_commands_t 構造体。
  • errfn: エラー発生時に呼び出されるコールバック関数。
  • data: エラーハンドラに渡されるユーザーデータ。
  • writefn: 出力処理を行うコールバック関数。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は sed_reset_eval が返すステータスコード。

前提条件と不変条件

  • eval ポインタは有効なメモリ領域を指している必要がある。
  • 内部で memset を使用して eval をゼロクリアするため、既存のデータは破棄される。

アルゴリズムまたは振る舞い

1. memset を用いて eval 構造体全体を 0 で初期化する。 2. 引数として受け取ったメモリプール p および出力関数 writefn を eval 構造体に格納する。 3. sed_reset_eval を呼び出し、コマンドセットやエラーハンドラの設定を含む残りの初期化処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_sed.c:255 (フィルタコンテキスト作成時)
  • **呼び出し先**: sed_reset_eval (modules/filters/sed1.c 内)

関連データ構造

  • sed_eval_t: libsed の実行時評価コンテキスト。
  • sed_commands_t: コンパイル済みコマンド構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • sed_reset_eval が失敗した場合、そのステータスコードをそのまま返す。
  • eval が NULL の場合、memset 実行時にセグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数は sed_eval_t の初期化の入り口であり、実際の状態設定は sed_reset_eval に委譲されている。
  • eval 内のバッファは apr_pool_t に紐付いているため、個別のメモリ解放は不要だが、ファイルハンドル等のリソースは sed_destroy_eval で適切に解放する必要がある。
