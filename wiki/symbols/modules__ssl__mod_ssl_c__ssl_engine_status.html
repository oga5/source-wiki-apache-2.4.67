ssl_engine_status

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:511

目的

特定の接続 (conn_rec) に対して SSL エンジンを有効化すべきか、あるいは無効化・スキップすべきかを判定する。

シグネチャまたは形状

static int ssl_engine_status(conn_rec *c, SSLConnRec *sslconn)

入力と出力

  • **入力**:
  • c: 現在の接続を表す conn_rec 構造体。
  • sslconn: 接続に関連付けられた SSLConnRec 構造体（NULL の場合あり）。
  • **出力**:
  • OK: SSL エンジンを有効化して処理を継続する。
  • DECLINED: SSL エンジンを適用せず、処理をスキップする。
  • SUSPENDED: SSL エンジンが明示的に無効化されている。

前提条件と不変条件

  • c->master が真（マスター接続）である場合、常に DECLINED を返す。
  • sslconn が提供されている場合、その設定状態（disabled や proxy_enabled）が優先される。
  • sslconn が提供されていない場合、c->base_server の設定に基づき判定を行う。

アルゴリズムまたは振る舞い

1. **マスター接続チェック**: c->master ならば直ちに DECLINED を返す。 2. **既存コンテキストの評価**: sslconn が存在する場合:

  • sslconn->disabled が真なら SUSPENDED を返す。
  • アウトバウンド接続 (c->outgoing) の場合、sslconn->dc->proxy_enabled が偽なら DECLINED を返す。
  • インバウンド接続の場合、サーバー設定の enabled が SSL_ENABLED_TRUE でなければ DECLINED を返す。

    • 3. **新規コンテキストの評価**: sslconn が存在しない場合:

  • アウトバウンド接続、または base_server の enabled が SSL_ENABLED_TRUE でない場合は DECLINED を返す。

    • 4. 上記のいずれにも該当しない場合、OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/mod_ssl.c:557
  • modules/ssl/mod_ssl.c:666
  • **呼び出し先**:
  • mySrvConfig マクロ（サーバー設定の取得に使用）。

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • SSLConnRec: mod_ssl 固有の接続ごとの SSL 状態。

失敗モードと境界条件

  • sslconn が NULL の場合、c->base_server を参照して判定を行うため、c->base_server が有効であることを前提としている。
  • アウトバウンド接続とインバウンド接続で判定ロジックが分岐するため、c->outgoing フラグの正確性に依存する。

保守担当者向けメモ

  • 本関数は接続の初期段階で SSL を適用するかどうかのゲートキーパーとして機能する。
  • SSL_ENABLED_TRUE 以外の設定値（SSL_ENABLED_FALSE 等）が渡された場合の挙動は、mySrvConfig の定義に依存する。
