check_string

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:195

目的

設定値として指定された文字列が、フォーム認証の内部処理で禁止されている文字（= および &）を含んでいないか、また空でないかを検証する。

シグネチャまたは形状

static const char *check_string(cmd_parms *cmd, const char *string)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • string: 検証対象の文字列。
  • **出力**:
  • 検証に成功した場合: NULL を返す。
  • 検証に失敗した場合: エラーメッセージを含む文字列（const char *）を返す。

前提条件と不変条件

  • cmd->pool が有効なメモリプールとして割り当てられていること。
  • cmd->directive->directive がエラーメッセージ生成のために参照可能であること。

アルゴリズムまたは振る舞い

1. string が NULL または空文字列であるかを確認する。 2. string 内に = または & が含まれているかを ap_strchr_c で検索する。 3. 上記のいずれかの条件に該当する場合、cmd->pool からメモリを割り当て、エラーメッセージを構築して返す。 4. いずれの条件にも該当しない場合、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_form.c: 設定ディレクティブ（location, username, password, method, mimetype, body）の解析時に呼び出される。
  • modules/session/mod_session_cookie.c: クッキー設定の検証時に呼び出される。
  • modules/session/mod_session_dbd.c: DBD セッション設定の検証時に呼び出される。
  • **呼び出し先**:
  • ap_strchr_c (Apache API)
  • apr_pstrcat (APR API)

関連データ構造

  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。

失敗モードと境界条件

  • **失敗モード**:
  • 文字列が空、または = や & を含む場合、検証エラーとしてメッセージを返す。
  • **境界条件**:
  • string が NULL の場合、即座にエラーと判定される。

保守担当者向けメモ

  • 本関数は mod_auth_form だけでなく、mod_session 関連のモジュールからも汎用的なバリデーターとして利用されている。変更を加える際は、これら複数のモジュールへの影響を考慮すること。
