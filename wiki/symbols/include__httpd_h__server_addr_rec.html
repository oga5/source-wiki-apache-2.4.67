server_addr_rec

定義

  • 種別: struct
  • 定義箇所: include/httpd.h:1313

目的

仮想ホストがリッスンするネットワークアドレス情報を保持するための構造体です。

シグネチャまたは形状

typedef struct server_addr_rec server_addr_rec;
struct server_addr_rec {
    server_addr_rec *next;
    /* ... 他のメンバ ... */
};

入力と出力

  • 入力: サーバー設定の初期化時や仮想ホストの構築時に、アドレス情報が本構造体に格納されます。
  • 出力: サーバーがリッスンするアドレスのリストとして、server_rec 構造体などを通じて参照されます。

前提条件と不変条件

  • server_rec 構造体の addrs フィールドからリンクされるリスト形式で管理されます。
  • リストの終端は NULL です。

アルゴリズムまたは振る舞い

  • サーバーの起動時や設定読み込み時に、apr_pcalloc を用いてメモリが割り当てられ、リスト構造として構築されます。
  • 仮想ホストの照合やリッスンソケットの管理において、このリストを走査して対象のアドレスを特定します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • server/config.c: 設定読み込み時に構造体のメモリ確保を実行。
  • server/vhost.c: 仮想ホストの解決時にリストを走査。
  • server/listen.c: リッスンアドレスの管理に使用。
  • modules/md/mod_md.c, modules/metadata/mod_remoteip.c, modules/proxy/mod_proxy_balancer.c: 各モジュール内でアドレス情報の参照に使用。
  • 呼び出し先:
  • APR（Apache Portable Runtime）のメモリ管理関数（apr_pcalloc 等）に依存しています。

関連データ構造

  • server_rec: 仮想ホストごとの設定を保持する構造体であり、本構造体のリストを addrs フィールドとして保持します。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc が失敗した場合、サーバーの起動が中断される可能性があります。
  • リストの不整合: 複数の仮想ホスト間でアドレスリストのポインタが正しく管理されていない場合、設定の誤認やクラッシュを招く可能性があります。

保守担当者向けメモ

  • 本構造体は server_rec の一部として機能するため、変更を加える際は仮想ホストの解決ロジック（server/vhost.c）への影響を考慮してください。
  • 構造体のメンバ定義は include/httpd.h の 1314 行目以降に続きます。
