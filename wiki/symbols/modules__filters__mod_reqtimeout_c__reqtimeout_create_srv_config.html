reqtimeout_create_srv_config

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reqtimeout.c:462

目的

サーバーごとの設定構造体（reqtimeout_srv_cfg）を生成し、初期化する。

シグネチャまたは形状

static void *reqtimeout_create_srv_config(apr_pool_t *p, server_rec *s)

入力と出力

  • 入力:
  • apr_pool_t *p: 設定構造体の割り当てに使用するメモリプール。
  • server_rec *s: 関連付けられるサーバーレコード（本関数内では未使用）。
  • 出力:
  • void *: 初期化済みの reqtimeout_srv_cfg 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全フィールドはゼロ初期化される。
  • 戻り値の構造体は、handshake、header、body の各ステージが UNSET_STAGE マクロによって未設定状態にマークされる。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、reqtimeout_srv_cfg サイズのメモリをプールから確保する。 2. UNSET_STAGE マクロを各ステージ（handshake, header, body）に対して実行し、各ステージの設定を無効値に設定する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_reqtimeout.c:666（サーバー設定構造体の作成時）。
  • 呼び出し先: apr_pcalloc（APRライブラリ）。

関連データ構造

  • reqtimeout_srv_cfg: 本関数が生成するサーバー設定構造体。
  • reqtimeout_stage_t: 各ステージのタイムアウト値を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc はアボートする可能性がある（APRのメモリ割り当てポリシーに依存）。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。
  • server_rec *s が未使用であるため、将来的にサーバーごとの動的な設定変更が必要になった場合に活用可能。
