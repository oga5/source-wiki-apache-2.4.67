fix_vary

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2_filter.c:224

目的

HTTPレスポンスヘッダーに含まれる複数の Vary フィールドを正規化し、重複を除去して単一のヘッダーに統合する。

シグネチャまたは形状

static void fix_vary(request_rec *r)

入力と出力

  • **入力**: request_rec *r （レスポンスヘッダー r->headers_out を含むリクエスト構造体）
  • **出力**: なし（副作用として r->headers_out を直接更新する）

前提条件と不変条件

  • r->headers_out が有効なテーブルとして存在していること。
  • r->pool がメモリ割り当てのために利用可能であること。

アルゴリズムまたは振る舞い

1. r->pool 上に一時的な配列 varies を作成する。 2. apr_table_do を使用して、r->headers_out 内のすべての Vary フィールドを走査する。 3. 各フィールド値をカンマ区切りで分解し、重複がないように varies 配列へ格納する（uniq_field_values 関数を利用）。 4. varies 配列に要素が追加された場合、既存の Vary フィールドを削除し、統合された値をカンマ区切り文字列として r->headers_out に再設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c2_filter.c:271 （直接的な呼び出しを確認）
  • **呼び出し先**:
  • apr_array_make
  • apr_table_do
  • uniq_field_values
  • apr_array_pstrcat
  • apr_table_setn

関連データ構造

  • request_rec: Apache のリクエスト状態を保持する構造体。
  • apr_array_header_t: Vary 値のリストを保持する動的配列。

失敗モードと境界条件

  • Vary ヘッダーが存在しない場合、何も変更を行わない。
  • Vary ヘッダーが既に単一で重複がない場合、実質的な変更は発生しない。
  • メモリ不足時は apr_array_make 等の内部関数によりアボートまたはエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は Vary ヘッダーの正規化のみを行う。ヘッダーの解析ロジックを変更する場合は uniq_field_values の実装も併せて確認すること。
  • r->pool を使用しているため、リクエストのライフサイクルを超えたメモリ保持は行われない。
