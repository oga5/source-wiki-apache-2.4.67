suexec_post_config

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_suexec.c:91

目的

サーバー起動時に suEXEC メカニズムが有効であるかを確認し、有効な場合にその旨をエラーログへ通知します。

シグネチャまたは形状

static int suexec_post_config(apr_pool_t *p, apr_pool_t *plog,
                              apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • p, plog, ptemp: メモリ管理用のアロケータプール。
  • s: サーバー設定を保持する server_rec 構造体。
  • **出力**:
  • 戻り値: 常に OK を返します。

前提条件と不変条件

  • ap_unixd_config.suexec_enabled が suEXEC の有効状態を示している必要があります。
  • サーバープロセスプール (s->process->pool) を介して、実行済みフラグ (SUEXEC_POST_CONFIG_USERDATA) を管理します。

アルゴリズムまたは振る舞い

1. apr_pool_userdata_get を使用し、サーバープロセスプールから SUEXEC_POST_CONFIG_USERDATA キーで以前の実行状態を取得します。 2. 以下の条件を両方満たす場合のみ、ログ出力を行います。

  • reported が NULL である（初回実行である）。
  • ap_unixd_config.suexec_enabled が真である。

    • 3. ログ出力時、APLOG_NOTICE レベルで suEXEC が有効であることと、使用されるバイナリパス (SUEXEC_BIN) を記録します。 4. apr_pool_userdata_set を使用し、次回以降の重複実行を防ぐためにフラグをセットします。 5. 最後に OK を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: suexec_hooks 関数内にて ap_hook_post_config に登録され、サーバー起動時の設定フェーズで呼び出されます。
  • **呼び出し先**:
  • apr_pool_userdata_get / apr_pool_userdata_set (APR API)
  • ap_log_error (Apache Core API)

関連データ構造

  • server_rec: サーバー設定情報。
  • apr_pool_t: メモリ管理用プール。
  • ap_unixd_config: UNIX 権限関連の設定を保持するグローバル構造体。

失敗モードと境界条件

  • 既に実行済みの場合、ログ出力はスキップされます。
  • ap_unixd_config.suexec_enabled が偽の場合、ログ出力は行われません。

保守担当者向けメモ

  • 本関数はサーバー起動時に一度だけ実行されることを意図しています。
  • SUEXEC_BIN はコンパイル時に定義されるマクロであり、環境によって異なります。
