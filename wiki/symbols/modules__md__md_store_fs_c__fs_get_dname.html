fs_get_dname

定義

  • 種別: function
  • 定義箇所: modules/md/md_store_fs.c:418

目的

指定されたストレージグループと名前から、ファイルシステム上のディレクトリパスを生成します。

シグネチャまたは形状

apr_status_t fs_get_dname(const char **pdname, 
                          md_store_t *store, md_store_group_t group, 
                          const char *name, apr_pool_t *p)

入力と出力

  • pdname: 生成されたディレクトリパス文字列を格納するポインタの出力先。
  • store: ファイルシステムストレージのハンドル。
  • group: 対象のストレージグループ（md_store_group_t）。
  • name: ディレクトリ名または識別子。
  • p: パス生成に使用するメモリプール。
  • 戻り値: 成功時は APR_SUCCESS、失敗時はエラーコードを返します。

前提条件と不変条件

  • store は FS_STORE マクロによって md_store_fs_t 型へキャスト可能である必要があります。
  • p はパス文字列の割り当てに十分な生存期間を持つ必要があります。

アルゴリズムまたは振る舞い

1. store からファイルシステムのベースパスを取得します。 2. group が MD_SG_NONE の場合、ベースパスをそのまま pdname に設定して終了します。 3. それ以外の場合、md_util_path_merge を使用して、ベースパス、グループ名、および name を結合した絶対パスを生成し、pdname に格納します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_store_fs.c:523
  • 呼び出し先: md_store_group_name（グループ名取得）、md_util_path_merge（パス結合）

関連データ構造

  • md_store_t: ストレージの抽象インターフェース。
  • md_store_fs_t: ファイルシステム実装の内部構造体。
  • md_store_group_t: ストレージグループの列挙型。

失敗モードと境界条件

  • md_util_path_merge が失敗した場合、そのエラーコードがそのまま返されます。
  • group が MD_SG_NONE の場合、name 引数は無視されます。

保守担当者向けメモ

  • 本関数は md_store_fs.c 内の内部的なパス解決ロジックの一部です。
  • パス生成には apr_pool_t を使用しているため、呼び出し側で適切なプールのライフサイクル管理が必要です。
