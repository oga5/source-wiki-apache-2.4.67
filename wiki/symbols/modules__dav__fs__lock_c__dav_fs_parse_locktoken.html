dav_fs_parse_locktoken

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/lock.c:242

目的

WebDAV の opaquelocktoken 形式の文字列を解析し、dav_locktoken 構造体に変換する。

シグネチャまたは形状

dav_error * dav_fs_parse_locktoken(
    apr_pool_t *p,
    const char *char_token,
    dav_locktoken **locktoken_p)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する APR プール。
  • char_token: 解析対象のロックトークン文字列。
  • **出力**:
  • locktoken_p: 解析結果の dav_locktoken 構造体へのポインタ。
  • 戻り値: 成功時は NULL、失敗時はエラー詳細を含む dav_error 構造体。

前提条件と不変条件

  • char_token は "opaquelocktoken:" で始まる必要がある。
  • char_token の後半部分は apr_uuid_parse が受理可能な UUID 形式である必要がある。

アルゴリズムまたは振る舞い

1. char_token が "opaquelocktoken:" で開始されているかを確認する。一致しない場合は HTTP_BAD_REQUEST を返す。 2. 文字列ポインタを 16 文字分進め、UUID 部分を指すようにする。 3. apr_pcalloc を使用して dav_locktoken 構造体をプールから割り当てる。 4. apr_uuid_parse を呼び出し、UUID 部分を解析して構造体に格納する。解析に失敗した場合は HTTP_BAD_REQUEST を返す。 5. 成功した場合、locktoken_p に結果を代入し NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/lock.c 内のコード（行 238, 1429 付近）から参照されている。
  • **呼び出し先**: ap_strstr_c, apr_pcalloc, apr_uuid_parse, dav_new_error。

関連データ構造

  • dav_locktoken: ロック識別子を保持する構造体。
  • dav_error: エラー情報を保持する構造体。

失敗モードと境界条件

  • **形式エラー**: "opaquelocktoken:" で始まらない場合、DAV_ERR_LOCK_UNK_STATE_TOKEN エラーを返す。
  • **UUID 解析エラー**: UUID 部分が不正な場合、DAV_ERR_LOCK_PARSE_TOKEN エラーを返す。

保守担当者向けメモ

  • 本関数は opaquelocktoken スキームのみをサポートしている。他のスキームの追加が必要な場合はロジックの拡張が必要。
  • メモリ割り当ては引数 p に依存しているため、呼び出し側で適切なライフサイクルのプールを渡すこと。
