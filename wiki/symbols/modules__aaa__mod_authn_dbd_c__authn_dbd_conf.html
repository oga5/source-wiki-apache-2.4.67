authn_dbd_conf

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authn_dbd.c:32

目的

ディレクトリ単位の設定において、データベース認証に使用する SQL クエリ文字列を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    const char *user;
    const char *realm;
} authn_dbd_conf;

入力と出力

  • user: AuthDBDUserPWQuery ディレクティブで指定された、パスワード検証用 SQL クエリ文字列。
  • realm: AuthDBDUserRealmQuery ディレクティブで指定された、レルムハッシュ取得用 SQL クエリ文字列。

前提条件と不変条件

  • request_rec のプール（r->pool）から割り当てられるメモリ管理下にあります。
  • クエリ文字列は mod_dbd の準備済みステートメントとして利用されることを前提としています。

アルゴリズムまたは振る舞い

  • 設定生成時（authn_dbd_cr_conf）にメモリが確保されます。
  • 設定マージ時（authn_dbd_merge_conf）に、親ディレクトリの設定と統合されます。
  • 認証実行時、ap_get_module_config を介してリクエストごとの設定として取得され、SQL クエリの実行に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • authn_dbd_cr_conf（構造体の初期化）
  • authn_dbd_merge_conf（設定のマージ）
  • authn_dbd_password および authn_dbd_realm（認証実行時の設定取得）
  • authn_dbd_cmds（ディレクティブ定義におけるオフセット指定）
  • 呼び出し先: 情報なし。

関連データ構造

  • authn_dbd_cmds: 本構造体のフィールドを埋めるためのディレクティブ定義。

失敗モードと境界条件

  • 設定が未定義の場合、各フィールドは NULL となります。認証実行時にこれらのフィールドが NULL である場合、対応する認証機能は実行できません。

保守担当者向けメモ

  • 本構造体は mod_authn_dbd の設定管理の基盤です。フィールドの追加や変更を行う際は、authn_dbd_merge_conf でのマージロジックが適切に更新されているか確認してください。
