ap_get_sload

定義

  • 種別: function
  • 定義箇所: server/util.c:3305

目的

サーバー全体の負荷状況（アイドル状態およびビジー状態の割合）を算出し、提供された構造体に格納します。

シグネチャまたは形状

void ap_get_sload(ap_sload_t *ld)

入力と出力

  • **入力**: ld (ap_sload_t *) - 負荷情報を書き込むための出力先構造体へのポインタ。
  • **出力**: なし（ld が指す構造体の idle, busy, bytes_served, access_count フィールドが更新されます）。

前提条件と不変条件

  • ap_scoreboard_image が初期化されている必要があります。
  • ap_mpm_query が正常に MPM の制限値および世代情報を取得できる必要があります。

アルゴリズムまたは振る舞い

1. ld の各フィールドを初期化します（idle, busy は -1、他は 0）。 2. ap_mpm_query を使用して、現在の世代情報 (mpm_generation) およびスレッド/プロセスのハードリミットを取得します。 3. スコアボードを全プロセスおよび全スレッドについて走査します。 4. 現在の世代と一致し、かつ停止中（quiescing）でないプロセス内のスレッドを対象に、状態（SERVER_READY か否か等）を判定して ready および busy カウンタを増分します。 5. ap_extended_status が有効な場合、各スレッドのアクセス数と転送バイト数を集計します。 6. 合計値が 0 でない場合、ready および busy の割合（パーセンテージ）を計算して ld に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/unix/mod_systemd.c:108
  • modules/metadata/mod_headers.c:228, 235
  • **呼び出し先**:
  • ap_mpm_query
  • ap_get_scoreboard_process

関連データ構造

  • ap_sload_t: 負荷情報を保持する構造体。
  • process_score: プロセスごとのスコアボード情報。
  • worker_score: スレッドごとのスコアボード情報。
  • ap_scoreboard_image: サーバー全体のスコアボード状態を保持するグローバル構造体。

失敗モードと境界条件

  • total（busy + ready）が 0 の場合、計算を行わず idle と busy は初期値（-1）のままとなります。
  • ap_extended_status が無効な場合、access_count および bytes_served の集計はスキップされます。

保守担当者向けメモ

  • 本関数はスコアボードの全走査を行うため、スレッド数やプロセス数が多い環境では呼び出し頻度に注意してください。
  • ap_mpm_query で取得する制限値と、実際のスコアボードのサイズが一致していることを前提としています。
