fileowner_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_owner.c:40

目的

Apache HTTP Server の Require file-owner ディレクティブに対する認可判定を行う。リクエストされたファイルの所有者名と、認証済みユーザー名（r->user）を比較し、アクセス可否を決定する。

シグネチャまたは形状

static authz_status fileowner_check_authorization(request_rec *r,
                                                  const char *require_args,
                                                  const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエストコンテキスト。
  • require_args: ディレクティブの引数（本関数では未使用）。
  • parsed_require_args: 解析済みの引数（本関数では未使用）。
  • **出力**:
  • AUTHZ_GRANTED: 所有者が一致した場合。
  • AUTHZ_DENIED: 所有者が一致しない、またはファイル情報の取得に失敗した場合。
  • AUTHZ_DENIED_NO_USER: 認証済みユーザーが存在しない場合。

前提条件と不変条件

  • プラットフォームが APR_HAS_USER をサポートしている必要がある。
  • r->user が設定されている（認証済みである）こと。
  • r->filename が有効なファイルパスを指していること。

アルゴリズムまたは振る舞い

1. **プラットフォーム確認**: APR_HAS_USER が未定義の場合、エラーをログ出力し AUTHZ_DENIED を返す。 2. **ユーザー確認**: r->user が空であれば AUTHZ_DENIED_NO_USER を返す。 3. **ファイル情報取得**: apr_stat を使用して対象ファイルの所有者IDを取得する。失敗時は AUTHZ_DENIED を返す。 4. **所有者名解決**: apr_uid_name_get を使用して、取得した所有者IDをユーザー名文字列に変換する。 5. **比較**: 解決された所有者名と r->user を strcmp で比較する。一致すれば AUTHZ_GRANTED を返し、不一致であれば AUTHZ_DENIED を返す。 6. **ログ出力**: 失敗時はすべて APLOGNO を付与したエラーログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_owner.c 内の authz_fileowner_provider 構造体を通じて、mod_authz_core から呼び出される。
  • **呼び出し先**:
  • ap_log_rerror: エラーログ出力。
  • apr_stat: ファイル情報の取得。
  • apr_uid_name_get: ユーザーIDから名前への解決。
  • apr_pstrcat / apr_psprintf: エラーメッセージ生成のためのメモリ割り当て。

関連データ構造

  • request_rec: リクエスト情報を保持する Apache の主要構造体。
  • apr_finfo_t: ファイル情報を保持する APR 構造体。
  • authz_status: 認可結果を示す列挙型。

失敗モードと境界条件

  • **プラットフォーム非対応**: APR_HAS_USER がない環境では常に拒否される。
  • **ファイル欠損**: r->filename が NULL の場合、認可失敗となる。
  • **所有者情報取得失敗**: ファイルが存在しない、または権限不足で apr_stat が失敗した場合、認可失敗となる。
  • **ユーザー名解決失敗**: apr_uid_name_get が失敗した場合、認可失敗となる。

保守担当者向けメモ

  • 本関数は Unix 系プラットフォームでの動作を前提としている。Windows 等の環境では機能しないため注意すること。
  • 認可失敗時のログには APLOGNO が付与されており、トラブルシューティング時には ErrorLog を参照すること。
