lua_db_prepared_query

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:473

目的

Lua スクリプトから、事前に準備された SQL ステートメント（プリペアドステートメント）を実行し、その影響を受けた行数を返します。

シグネチャまたは形状

int lua_db_prepared_query(lua_State *L)

入力と出力

  • **入力**:
  • Lua スタック第1引数: プリペアドステートメントを保持するテーブル（lua_db_prepared_statement を含む）。
  • 第2引数以降: SQL クエリのバインド変数（可変長）。
  • **出力**:
  • 成功時: 影響を受けた行数（整数）をスタックにプッシュし、1 を返す。
  • 引数不足時: false とエラーメッセージをスタックにプッシュし、2 を返す。
  • 実行失敗時: nil とエラーメッセージ（存在する場合）をスタックにプッシュし、1 または 2 を返す。
  • 接続切断時: false とエラーメッセージをスタックにプッシュし、2 を返す。

前提条件と不変条件

  • 第1引数は LUA_TTABLE 型であり、そのインデックス 0 に lua_db_prepared_statement 型のユーザーデータが格納されていること。
  • データベース接続（st->db）が有効（alive）であること。

アルゴリズムまたは振る舞い

1. スタックからプリペアドステートメントオブジェクトを取得します。 2. 必要なバインド変数の数と、スタックから渡された引数の数を比較し、不足している場合はエラーを返します。 3. 渡された引数を apr_pcalloc で確保したメモリ領域にコピーします。 4. apr_dbd_pquery を呼び出し、SQL を実行します。 5. 実行結果が APR_SUCCESS であれば影響行数を返し、それ以外の場合は apr_dbd_error を用いてエラーメッセージを取得して返します。 6. 接続が切断されている場合は、再接続を促すエラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_dbd.c 内で lua_pushcfunction を介して Lua VM に登録されています（行600, 654）。
  • **呼び出し先**:
  • apr_dbd_pquery (APR DBD API)
  • apr_dbd_error (APR DBD API)
  • apr_pcalloc (APR メモリ管理)

関連データ構造

  • lua_db_prepared_statement: プリペアドステートメントの定義とデータベースハンドルを保持。
  • lua_db_handle: データベース接続状態を管理。

失敗モードと境界条件

  • **引数不足**: 期待されるバインド変数数に満たない場合、実行を中断します。
  • **接続喪失**: st->db->alive が偽の場合、クエリを実行せずエラーを返します。
  • **実行エラー**: apr_dbd_pquery が失敗した場合、ドライバから取得したエラーメッセージを Lua 側に伝播させます。

保守担当者向けメモ

  • 本関数は lua_db_prepared_statement の variables フィールドを使用して引数の妥当性を検証します。この値が -1 の場合は検証をスキップします。
  • バインド変数はすべて lua_tostring を介して文字列として扱われます。型変換の詳細は APR DBD ドライバの実装に依存します。
