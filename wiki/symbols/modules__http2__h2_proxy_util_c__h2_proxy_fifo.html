h2_proxy_fifo

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.c:1082

目的

スレッドセーフな FIFO（先入れ先出し）キューを提供します。マルチスレッド環境において、複数のスレッド間でデータを安全に受け渡すための同期機構として機能します。

シグネチャまたは形状

struct h2_proxy_fifo {
    void **elems;
    int nelems;
    int set;
    int head;
    int count;
    int aborted;
    apr_thread_mutex_t *lock;
    apr_thread_cond_t  *not_empty;
    apr_thread_cond_t  *not_full;
};

入力と出力

  • **入力**: void **elems（要素格納用バッファ）、int nelems（容量）。
  • **出力**: int count（現在の格納数）、int aborted（中断状態フラグ）。
  • 状態管理として apr_thread_mutex_t および apr_thread_cond_t を保持し、スレッド間のブロッキング制御を行います。

前提条件と不変条件

  • lock ミューテックスにより、構造体メンバーへのアクセスは排他制御されます。
  • not_empty および not_full 条件変数により、キューの空/満杯状態に応じたスレッドの待機・起床が保証されます。
  • aborted が設定された場合、待機中のスレッドは即座に復帰する必要があります。

アルゴリズムまたは振る舞い

  • **FIFO 動作**: head インデックスと count を用いて循環バッファとして動作します。
  • **スレッド同期**: fifo_push および fifo_pull 操作において、block フラグが有効な場合、キューが満杯または空の際に条件変数を用いてスレッドを待機させます。
  • **中断処理**: h2_proxy_fifo_interrupt を呼び出すことで、待機中の全スレッドを強制的に起床させ、処理を中断させることが可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_proxy_fifo_create, h2_proxy_fifo_set_create, h2_proxy_fifo_term, h2_proxy_fifo_interrupt, h2_proxy_fifo_count, h2_proxy_fifo_capacity 等のユーティリティ関数から参照されます。
  • **呼び出し先**: apr_thread_mutex_t および apr_thread_cond_t の APR API を呼び出し、スレッド同期を実現します。

関連データ構造

  • apr_thread_mutex_t: 排他制御用。
  • apr_thread_cond_t: スレッド待機用。

失敗モードと境界条件

  • **容量超過**: nelems を超える要素の追加は、not_full で待機するか、エラーを返します。
  • **空のキュー**: 要素がない状態で pull を行うと、not_empty で待機します。
  • **中断**: aborted フラグがセットされると、キュー操作はエラーを返し、待機中のスレッドは解放されます。

保守担当者向けメモ

  • 本構造体は h2_proxy_util.c 内の静的関数および公開関数によって管理されます。
  • メモリ管理は apr_pool_t に依存しており、生成時に指定されたプールが破棄される際に適切にクリーンアップされる必要があります。
  • set メンバーが何を指すかについては、ソースコード内の利用箇所（h2_proxy_fifo_set_create 等）を詳細に確認してください。
