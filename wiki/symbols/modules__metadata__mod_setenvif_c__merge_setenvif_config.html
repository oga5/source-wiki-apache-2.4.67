merge_setenvif_config

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_setenvif.c:151

目的

サーバー設定とディレクトリ設定、または異なるスコープの設定間で sei_cfg_rec 構造体を結合し、条件ルールのリストを統合します。

シグネチャまたは形状

static void *merge_setenvif_config(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • basev: 親スコープの設定（sei_cfg_rec 型）。
  • overridesv: 現在のスコープの設定（sei_cfg_rec 型）。
  • **出力**:
  • 結合された新しい sei_cfg_rec 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は sei_cfg_rec 構造体としてキャスト可能である必要があります。
  • 結合後の conditionals 配列は、base のルールが先に、overrides のルールが後に配置されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい sei_cfg_rec 構造体 a をプール p 上に確保します。 2. basev と overridesv をそれぞれ sei_cfg_rec 型としてキャストします。 3. apr_array_append を呼び出し、base->conditionals と overrides->conditionals を結合して a->conditionals に格納します。 4. 結合された構造体 a を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_setenvif.c:659（ディレクトリ設定の結合用）
  • modules/metadata/mod_setenvif.c:661（サーバー設定の結合用）
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_array_append (APR)

関連データ構造

  • sei_cfg_rec: 設定を保持するコンテナ。
  • sei_entry: conditionals 配列内に格納される個別の条件ルール。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc または apr_array_append がエラーを発生させる可能性があります。
  • 入力ポインタが NULL の場合の挙動については、APR の配列操作関数の仕様に依存します。

保守担当者向けメモ

  • 本関数は Apache の設定マージ機構の一部として機能します。
  • conditionals 配列の順序は、リクエスト評価時のルール適用順序に直結するため、変更時には注意が必要です。
