ap_down_server_cb

定義

  • 種別: function
  • 定義箇所: os/netware/util_nw.c:50

目的

NetWare 環境において、サーバー停止イベント（EVENT_DOWN_SERVER）が発生した際に、NLM（NetWare Loadable Module）のアンロード処理を開始するために呼び出されるコールバック関数です。

シグネチャまたは形状

void ap_down_server_cb(void *a, void *b)

入力と出力

  • 入力:
  • void *a: 未使用（NetWare のイベント通知シグネチャに適合させるための引数）。
  • void *b: 未使用。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • 本関数は NetWare OS 上で実行されることを前提としています。
  • nlmUnloadSignaled が利用可能な環境で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. nlmUnloadSignaled(0) を呼び出し、サーバーのアンロード処理をトリガーします。 2. 処理完了後、直ちに制御を戻します。

呼び出し元と呼び出し先

  • 呼び出し元: NetWare のイベント通知システム（EVENT_DOWN_SERVER 発生時）。
  • 呼び出し先: nlmUnloadSignaled。
  • 参照: os/netware/util_nw.c:101 にて NX_WRAP_INTERFACE を介してインターフェースとしてラップされています。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • nlmUnloadSignaled の内部状態に依存します。本関数自体は引数の検証を行わず、常に 0 を渡してアンロードを通知します。

保守担当者向けメモ

  • 本関数は NetWare のイベントハンドラとして登録されるため、シグネチャを変更する場合は NX_WRAP_INTERFACE の呼び出し箇所（util_nw.c:101）との整合性に注意してください。
  • 引数 a および b は NetWare の標準的なコールバック形式に合わせるためのものであり、関数内部では使用されていません。
