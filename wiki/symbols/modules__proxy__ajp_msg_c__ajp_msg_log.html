ajp_msg_log

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_msg.c:115

目的

AJPメッセージの内容をデバッグログとして出力します。トレースレベル（TRACE7/8）が有効な場合にのみ実行され、メッセージのバイナリダンプを人間が読める形式でログに記録します。

シグネチャまたは形状

apr_status_t ajp_msg_log(request_rec *r, ajp_msg_t *msg, char *err)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報（request_rec）。ログ出力およびメモリプール参照に使用。
  • msg: ダンプ対象の ajp_msg_t 構造体。
  • err: ログ出力時に付与するエラーメッセージまたは説明文。
  • **出力**:
  • apr_status_t: 操作の成功状態（APR_SUCCESS または ajp_msg_dump が返すエラーコード）。

前提条件と不変条件

  • APLOGrtrace7(r) が真である場合にのみ、ログ出力処理が実行されます。
  • r->pool が有効である必要があります。

アルゴリズムまたは振る舞い

1. **トレースレベル判定**: APLOGrtrace7 が有効か確認し、さらに APLOGrtrace8 が有効であればダンプサイズを AJP_MAX_BUFFER_SZ に、そうでなければ 1024 バイトに設定します。 2. **ダンプ生成**: ajp_msg_dump を呼び出し、メッセージ内容を文字列バッファに変換します。 3. **ログ出力**: 生成された文字列を改行コード（\n）で分割し、各行を ap_log_rerror を用いて個別にログ出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/ajp_header.c (ajp_send_header, ajp_read_header)
  • modules/proxy/ajp_utils.c (ajp_handle_cping_cpong)
  • modules/proxy/mod_proxy_ajp.c (ajp_send_data_msg)
  • **呼び出し先**:
  • ajp_msg_dump: メッセージのバイナリ変換。
  • ap_log_rerror: Apache のログ出力関数。
  • ap_strchr: 文字列内の改行検索。

関連データ構造

  • ajp_msg_t: AJPメッセージを保持する構造体。
  • request_rec: Apache のリクエスト状態を管理する構造体。

失敗モードと境界条件

  • ajp_msg_dump が失敗した場合、そのエラーコードをそのまま返します。
  • トレースレベルが設定されていない場合、何も処理を行わず APR_SUCCESS を返します。

保守担当者向けメモ

  • 本関数はデバッグ専用であり、パフォーマンスへの影響を避けるため APLOGrtrace チェックで囲まれています。
  • ログ出力時に APLOGNO マクロが含まれていないのは意図的な仕様です。
