ap_lua_get_lua_state

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:436

目的

Apache HTTP Server のリクエスト処理において、指定されたスコープと仕様に基づき、Lua 仮想マシン（lua_State）を取得または生成します。

シグネチャまたは形状

lua_State *ap_lua_get_lua_state(apr_pool_t *lifecycle_pool,
                                ap_lua_vm_spec *spec,
                                request_rec *r);

入力と出力

  • **入力**:
  • lifecycle_pool: VM のライフサイクルを管理するメモリプール。
  • spec: VM の生成パラメータ（スコープ、キャッシュポリシー、ファイルパス等）を保持する構造体。
  • r: 現在のリクエストコンテキスト。
  • **出力**:
  • 成功時: 初期化済みの lua_State ポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • spec->scope が AP_LUA_SCOPE_SERVER の場合、内部で ap_lua_mutex を使用した排他制御が行われます。
  • VM のライフサイクルは lifecycle_pool に紐付けられ、プール破棄時に自動的に lua_close が実行されるようクリーンアップハンドラが登録されます。

アルゴリズムまたは振る舞い

1. **キャッシュ検索**: スコープに応じてキャッシュ（apr_reslist または apr_pool_userdata）から既存の lua_State を検索します。 2. **VM 生成**: キャッシュミス時は vm_construct を呼び出し、新規 VM を作成します。 3. **キャッシュ検証**: spec->codecache 設定に基づき、ファイルの更新日時（apr_stat）や実行回数をチェックし、必要に応じてスクリプトの再ロード（luaL_loadfile および lua_pcall）を行います。 4. **返却**: 準備完了した lua_State を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/mod_lua.c 内の各フック関数（リクエスト処理の各フェーズ）。
  • **呼び出し先**: vm_construct, copy_vm_spec, server_vm_construct, server_cleanup_lua, cleanup_lua, luaL_loadfile, lua_pcall。

関連データ構造

  • ap_lua_vm_spec: VM の動作定義。
  • ap_lua_finfo: ファイルのキャッシュ状態（更新日時、サイズ、実行回数）を保持。
  • apr_reslist_t: サーバーレベルの VM プール管理に使用。

失敗モードと境界条件

  • vm_construct が失敗した場合、NULL を返します。
  • luaL_loadfile でエラーが発生した場合、エラーログを出力し 0（NULL）を返します。
  • AP_LUA_SCOPE_SERVER でスレッド競合が発生した場合、apr_reslist_acquire の失敗により NULL を返す可能性があります。

保守担当者向けメモ

  • spec->scope が AP_LUA_SCOPE_ONCE の場合、キャッシュ検証ロジックはスキップされます。
  • munge_path によるパス操作が失敗すると、Lua スクリプトのロードに影響が出るため注意してください。
  • AP_LUA_CACHE_STAT 使用時は apr_stat のオーバーヘッドが発生するため、高負荷環境ではキャッシュポリシーの選定が重要です。
