log_print

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md.c:90

目的

Managed Domains (MD) サブシステムにおけるログ出力の抽象化インターフェースとして機能し、設定されたログレベルに基づいてメッセージを Apache のエラーログへ転送します。

シグネチャまたは形状

static void log_print(const char *file, int line, md_log_level_t level,
                      apr_status_t rv, void *baton, apr_pool_t *p, const char *fmt, va_list ap)

入力と出力

  • **入力**:
  • file, line: ログ出力元のソースファイル名と行番号。
  • level: md_log_level_t 型のログレベル。
  • rv: APR ステータスコード。
  • baton: ログ出力コンテキスト（log_is_level に渡される）。
  • p: メモリ管理用プール。
  • fmt, ap: フォーマット文字列と可変長引数リスト。
  • **出力**: なし（副作用としてログ出力を行う）。

前提条件と不変条件

  • log_is_level によるレベル判定が真である場合にのみ出力処理が実行されます。
  • LOG_BUF_LEN サイズのバッファを使用し、フォーマット後の文字列は必ずヌル終端されます。

アルゴリズムまたは振る舞い

1. log_is_level を呼び出し、現在のログレベルで出力すべきか判定します。 2. apr_vsnprintf を使用して、フォーマット文字列と引数を buffer に展開します。 3. グローバル変数 log_server が設定されている場合、ap_log_error を使用してサーバー単位のログを出力します。 4. log_server が未設定の場合、ap_log_perror を使用してプールに関連付けられたログを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c:883 にて md_log_set を通じて登録されています。
  • **呼び出し先**:
  • log_is_level (判定用)
  • apr_vsnprintf (文字列整形)
  • ap_log_error / ap_log_perror (Apache ログ出力 API)

関連データ構造

  • md_log_level_t: ログレベルを定義する列挙型。
  • apr_pool_t: メモリ管理用プール。
  • log_server: モジュール内で保持されるサーバーコンテキスト（server_rec）。

失敗モードと境界条件

  • LOG_BUF_LEN を超えるメッセージは切り詰められます。
  • log_server が NULL の場合、ap_log_perror にフォールバックするため、サーバーコンテキストが利用できない初期化フェーズでもログ出力が可能です。

保守担当者向けメモ

  • 本関数は md_log_set を介して md サブシステム全体に提供されるログ出力バックエンドです。
  • ログ出力の制御は log_is_level に依存しているため、出力内容を調整する場合はそちらのロジックを確認してください。
