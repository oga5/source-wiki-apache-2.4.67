h2_protocol_propose

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_switch.c:52

目的

サーバーがサポートするプロトコルを提案し、HTTP/1.1 から HTTP/2 への切り替えが可能かを判定する。

シグネチャまたは形状

static int h2_protocol_propose(conn_rec *c, request_rec *r,
                               server_rec *s,
                               const apr_array_header_t *offers,
                               apr_array_header_t *proposals)

入力と出力

  • **入力**:
  • c: 現在の接続情報 (conn_rec)
  • r: リクエスト情報 (request_rec, アップグレード時のみ存在)
  • s: サーバー情報 (server_rec)
  • offers: クライアントが提示したプロトコルリスト
  • proposals: サーバーが提案するプロトコルを格納する配列
  • **出力**:
  • OK (提案が成功した場合) または DECLINED (条件不適合や提案なしの場合)

前提条件と不変条件

  • 現在のプロトコルが http/1.1 であること。
  • h2_mpm_supported() が真であること。
  • h2_protocol_is_acceptable_c1 による接続要件を満たしていること。
  • アップグレードリクエストの場合、HTTP2-Settings ヘッダーの存在と、リクエストボディが空（Content-Length: 0）であることが必須。

アルゴリズムまたは振る舞い

1. SSL/TLS の有無に基づき、対応するプロトコル ID リストを選択する。 2. 接続状態やリクエストヘッダー（Upgrade, HTTP2-Settings 等）を検証し、条件を満たさない場合は DECLINED を返す。 3. offers に含まれるプロトコルと、サーバーがサポートするプロトコルを照合する。 4. 一致するプロトコルを proposals 配列に追加する。 5. 提案が一つでも追加されれば OK を、そうでなければ DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_protocol_propose により登録され、Apache のプロトコルネゴシエーションフェーズで呼び出される。
  • **呼び出し先**: ap_ssl_conn_is_ssl, ap_get_protocol, h2_mpm_supported, h2_protocol_is_acceptable_c1, h2_c1_can_upgrade, ap_table_get, ap_find_token, APR_ARRAY_PUSH 等。

関連データ構造

  • conn_rec, request_rec: Apache HTTP Server のコア構造体。
  • h2_protocol_ids_tls, h2_protocol_ids_clear: モジュール内で定義されたサポートプロトコル識別子の配列。

失敗モードと境界条件

  • HTTP/1.1 以外のプロトコルからの切り替えは拒否される。
  • リクエストボディが存在するアップグレード要求は拒否される。
  • HTTP2-Settings ヘッダーが欠落しているアップグレード要求は拒否される。

保守担当者向けメモ

  • 本関数はプロトコルの「提案」のみを行い、実際の切り替え処理は h2_protocol_switch が担当する。
  • フィルターの操作は行わないため、安全に呼び出し可能。
