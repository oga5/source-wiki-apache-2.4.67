check_group_dir

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md.c:264

目的

Managed Domains (MD) サブシステムにおいて、指定されたストレージグループに対応するディレクトリが存在することを確認し、存在しない場合は作成する。

シグネチャまたは形状

static apr_status_t check_group_dir(md_store_t *store, md_store_group_t group,
                                    apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • store: md_store_t 構造体へのポインタ。
  • group: 作成対象のストレージグループ識別子 (md_store_group_t)。
  • p: メモリ割り当てに使用する apr_pool_t。
  • s: サーバー設定レコード server_rec。
  • **出力**:
  • apr_status_t: 処理結果。成功時は APR_SUCCESS を返す。

前提条件と不変条件

  • store は有効な md_store_t インスタンスであること。
  • md_store_get_fname が指定された group に対してディレクトリパスを解決できること。

アルゴリズムまたは振る舞い

1. md_store_get_fname を呼び出し、指定された group に対応するディレクトリパスを取得する。 2. apr_dir_make_recursive を使用し、取得したパスに対して MD_FPROT_D_UALL_GREAD 権限でディレクトリを再帰的に作成する。 3. ディレクトリ作成が成功した場合、store_file_ev を呼び出し、MD_S_FS_EV_CREATED イベントを記録する。 4. 処理結果のステータスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md.c 内の setup_store 関数（推測される文脈上の初期化処理）。
  • 具体的には、MD_SG_CHALLENGES, MD_SG_STAGING, MD_SG_ACCOUNTS, MD_SG_OCSP の各グループに対して本関数が呼び出されている。
  • **呼び出し先**:
  • md_store_get_fname
  • apr_dir_make_recursive
  • store_file_ev

関連データ構造

  • md_store_t: ストレージ管理の抽象構造体。
  • md_store_group_t: ストレージ内の論理グループ（チャレンジ、ステージング、アカウント等）。
  • server_rec: Apache HTTP Server のサーバー設定構造体。

失敗モードと境界条件

  • md_store_get_fname が失敗した場合、そのエラーコードがそのまま返される。
  • apr_dir_make_recursive がディレクトリ作成に失敗した場合（権限不足やパスの競合など）、そのエラーコードが返される。
  • store_file_ev が失敗した場合、そのエラーコードが返される。

保守担当者向けメモ

  • 本関数は mod_md の初期化フェーズにおいて、必要なディレクトリ構造が確実に存在することを保証するために使用される。
  • MD_FPROT_D_UALL_GREAD 定数により、作成されるディレクトリのパーミッションが制御されている点に注意すること。
