read_heartbeats

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_heartbeat.c:243

目的

バックエンドサーバーの負荷情報を取得するための抽象化レイヤーとして機能し、共有メモリまたはファイルからハートビート情報を読み込みます。

シグネチャまたは形状

apr_status_t read_heartbeats(const char *path, apr_hash_t *servers, apr_pool_t *pool)

入力と出力

  • **入力**:
  • path: ハートビート情報が格納されたファイルパス（ファイルベース読み込み時に使用）。
  • servers: 読み込んだ情報を格納する apr_hash_t 構造体。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • hm_serversmem が有効な場合、共有メモリから情報を取得します。
  • hm_serversmem が無効な場合、path で指定されたファイルから情報を取得します。

アルゴリズムまたは振る舞い

1. hm_serversmem が存在するかを確認します。 2. 存在する場合、ctx_servers_t を初期化し、現在時刻を取得した上で readslot_heartbeats を呼び出します。 3. 存在しない場合、readfile_heartbeats を呼び出し、指定されたファイルから情報を解析します。 4. 呼び出し先からの戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: find_best_hb (modules/proxy/balancers/mod_lbmethod_heartbeat.c:286)
  • **呼び出し先**:
  • readslot_heartbeats (共有メモリが有効な場合)
  • readfile_heartbeats (共有メモリが無効な場合)

関連データ構造

  • apr_hash_t: サーバー情報を保持するハッシュテーブル。
  • ctx_servers_t: 共有メモリ読み込み時に使用されるコンテキスト構造体。

失敗モードと境界条件

  • 共有メモリまたはファイルへのアクセス権限がない場合、あるいはファイルが存在しない場合、呼び出し先の関数が返すエラーコードがそのまま伝播されます。

保守担当者向けメモ

  • 本関数は mod_heartmonitor との連携を抽象化しています。新しいデータソースを追加する場合は、この分岐ロジックを拡張する必要があります。
