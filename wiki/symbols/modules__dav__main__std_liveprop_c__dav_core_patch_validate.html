dav_core_patch_validate

定義

  • 種別: function
  • 定義箇所: modules/dav/main/std_liveprop.c:183

目的

WebDAV リソースに対するプロパティ書き込み要求を検証し、処理の委譲先を決定します。

シグネチャまたは形状

dav_error * dav_core_patch_validate(const dav_resource *resource,
                                    const apr_xml_elem *elem,
                                    int operation, void **context,
                                    int *defer_to_dead)

入力と出力

  • **入力**:
  • resource: 対象となる DAV リソース。
  • elem: 書き込み対象の XML 要素。
  • operation: 実行される操作の種類。
  • context: 操作コンテキスト（本関数内では未使用）。
  • **出力**:
  • defer_to_dead: 書き込み処理をデッドプロパティデータベースへ委譲するかどうかを示すフラグ。
  • 戻り値: 常に NULL を返します。

前提条件と不変条件

  • 本関数は DAV: 名前空間のライブプロパティに対する書き込み要求時に呼び出されます。
  • *defer_to_dead は常に 1 に設定されます。

アルゴリズムまたは振る舞い

1. 書き込み対象のプロパティを「デッドプロパティ（データベース保存）」として扱うよう、defer_to_dead 引数に 1 を代入します。 2. エラーを発生させることなく NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/std_liveprop.c:198 にて、dav_core_liveprop_group 構造体を通じて参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • dav_resource: DAV リソースの抽象表現。
  • apr_xml_elem: XML 要素の構造体。
  • dav_error: エラー情報を保持する構造体。

失敗モードと境界条件

  • 本関数は常に成功（NULL を返却）し、失敗モードはありません。

保守担当者向けメモ

  • 本関数は、コアライブプロパティの書き込みがこのファイル内では完結せず、バックエンドのプロパティデータベースへ委ねられることを強制する役割を持ちます。ロジック変更時は、この委譲方針が維持されるか確認してください。
