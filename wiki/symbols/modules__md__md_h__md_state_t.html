md_state_t

定義

  • 種別: enum
  • 定義箇所: modules/md/md.h:52

目的

Managed Domain (MD) のライフサイクル状態を表現し、証明書の取得・更新プロセスにおける現在の進行状況を管理する。

シグネチャまたは形状

typedef enum {
    MD_S_UNKNOWN = 0,
    MD_S_INCOMPLETE = 1,
    MD_S_COMPLETE = 2,
    MD_S_EXPIRED_DEPRECATED = 3,
    MD_S_ERROR = 4,
    MD_S_MISSING_INFORMATION = 5,
} md_state_t;

入力と出力

  • 本シンボルは列挙型であり、関数への入力（状態の判定）や出力（状態の保持）として使用される。
  • md_t 構造体のメンバとして保持され、JSON シリアライズ時には数値として永続化される。

前提条件と不変条件

  • md_t 構造体の state メンバとして管理される。
  • MD_S_COMPLETE は、ドメインが稼働可能な状態であることを示す。
  • MD_S_MISSING_INFORMATION は、利用規約（ToS）への同意不足など、ユーザーによる追加操作が必要な状態を示す。

アルゴリズムまたは振る舞い

  • md_json_getl を介して JSON データから状態を復元する際に使用される。
  • md_reg などの上位モジュールが、ACME プロトコルの進捗やエラー発生に応じてこの値を更新する。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/md/md.h:111: md_t 構造体内で状態保持用として使用。
  • modules/md/md_core.c:354: JSON からの読み込み時にキャストして代入。
  • modules/md/md_reg.c:229: 初期状態の定義に使用。
  • 呼び出し先:
  • 本シンボルは列挙型であり、特定の関数を呼び出すことはない。

関連データ構造

  • md_t: 本状態を保持するメインのデータ構造体。

失敗モードと境界条件

  • MD_S_ERROR: データに不備があり、処理が継続できない状態を示す。
  • MD_S_INCOMPLETE: 必要な情報が不足しており、稼働状態へ移行できない状態を示す。

保守担当者向けメモ

  • MD_S_EXPIRED_DEPRECATED は非推奨であるため、新規ロジックでの使用は避けること。
  • 状態遷移のロジックは md_reg.c 等のレジストリ層に実装されているため、状態変更のトリガーを追跡する際は当該ファイルを調査すること。
