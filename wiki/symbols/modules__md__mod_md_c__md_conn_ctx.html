md_conn_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/mod_md.c:1043

目的

Apache HTTP Server の接続（connection）単位で、Managed Domains (MD) モジュールに関連するコンテキスト情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    const char *protocol;
} md_conn_ctx;

入力と出力

  • **入力**: 接続に関連付けられた設定データ。
  • **出力**: 接続コンテキスト内の protocol 文字列（ACME チャレンジ等のプロトコル識別に使用）。

前提条件と不変条件

  • ap_get_module_config(c->conn_config, &md_module) を介して、接続ごとの設定として取得されることが想定されています。

アルゴリズムまたは振る舞い

  • 接続オブジェクト（conn_rec）に対して、MD モジュール固有の情報を付与・参照するためのデータコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c 内の接続処理ロジックにおいて、ap_get_module_config を通じて本構造体へのポインタが取得されています。
  • **呼び出し先**: 構造体内部の protocol メンバが、接続に関連するプロトコル判定や処理の分岐に使用されます。

関連データ構造

  • conn_rec (Apache HTTPD の接続構造体): md_conn_ctx が設定として格納される対象。
  • md_module: ap_get_module_config で指定されるモジュール識別子。

失敗モードと境界条件

  • ap_get_module_config が NULL を返す場合、接続に MD コンテキストが未設定である可能性があります。

保守担当者向けメモ

  • 本構造体は接続のライフサイクルに依存します。接続終了時にメモリが適切に解放されるよう、Apache のプール管理と整合性を保つ必要があります。
