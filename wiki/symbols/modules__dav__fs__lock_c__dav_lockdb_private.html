dav_lockdb_private

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/lock.c:180

目的

ファイルシステムベースのDAVロックデータベース（DBM）へのアクセスに必要な状態情報を保持するためのプライベート構造体です。

シグネチャまたは形状

struct dav_lockdb_private
{
    request_rec *r;                  /* UUID状態へのアクセス用 */
    apr_pool_t *pool;                /* 使用するメモリプール */
    const char *lockdb_path;         /* ロックデータベースのパス */

    int opened;                      /* データベースが開かれているか */
    dav_db *db;                      /* ロックデータベースへのハンドル（非NULL時） */
};

入力と出力

本構造体は関数引数として渡されるのではなく、主に dav_lockdb_combined 構造体の一部として、ロック操作関数内でデータベースの状態を追跡するために使用されます。

前提条件と不変条件

  • r はリクエスト処理に関連する request_rec を指す必要があります。
  • db が非NULLの場合、データベースは正常に開かれています。
  • メモリ管理は pool に紐付いて行われます。

アルゴリズムまたは振る舞い

  • **遅延オープン**: opened フラグと db ハンドルにより、データベースの実際のオープン処理を初回アクセス時まで遅延させる制御をサポートします。
  • **状態管理**: ロックの読み込みや保存を行う際、この構造体を通じてデータベースのパスやハンドルを参照します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/lock.c:192
  • modules/dav/lock/locks.c:188
  • modules/dav/main/mod_dav.h:1343（info メンバとして参照）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。操作は dav_fs_open_lockdb 等の関数が本構造体のメンバを操作することで行われます。

関連データ構造

  • dav_lockdb_combined: 本構造体を内包し、ロックデータベースの統合的な管理を行う構造体。
  • dav_db: DBM操作のための抽象化ハンドル。
  • request_rec: Apache HTTP Server のリクエスト情報。

失敗モードと境界条件

  • db がNULLの状態でデータベース操作を試みた場合、操作は失敗します。
  • 読み取り専用で開かれているデータベースに対して書き込み操作を行うとエラーとなります。

保守担当者向けメモ

  • 本構造体は mod_dav のファイルシステムプロバイダー内部でのみ使用されるプライベートな定義です。
  • r->pool との整合性を保つため、メモリ割り当て時には必ず pool メンバを使用してください。
  • ロックデータベースのパス（lockdb_path）は、設定ファイルで指定された値が格納されます。
