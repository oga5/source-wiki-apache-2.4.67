ebcdic2ascii_strictly

定義

  • 種別: function
  • 定義箇所: os/bs2000/ebcdic.c:196

目的

EBCDIC形式のメモリブロックを、厳密な変換ルールに従ってASCII形式へ変換します。

シグネチャまたは形状

void ebcdic2ascii_strictly(unsigned char *dest, const unsigned char *srce, size_t count)

入力と出力

  • dest: 変換後のデータを格納するバッファ（出力）。
  • srce: 変換元のEBCDICデータが格納されたバッファ（入力）。
  • count: 変換するバイト数。
  • 戻り値: なし。

前提条件と不変条件

  • dest と srce が指すメモリ領域は、互いにオーバーラップしてはなりません。
  • count バイト分の読み取りおよび書き込みが可能なメモリ領域が確保されている必要があります。

アルゴリズムまたは振る舞い

1. count が0になるまで、以下の処理を繰り返します。 2. srce が指す現在のバイトをインデックスとして、静的変換テーブル os_toascii_strictly を参照します。 3. 変換後の値を dest が指す位置に書き込みます。 4. dest および srce のポインタをそれぞれインクリメントします。

  • os_toascii と異なり、改行コード（0x0A）を含むすべての文字を厳密なテーブル定義に基づいて変換します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • 呼び出し先: 静的変換テーブル os_toascii_strictly を参照します。

関連データ構造

  • os_toascii_strictly: os/bs2000/ebcdic.c 内で定義された、厳密なEBCDIC-ASCII変換用ルックアップテーブル。

失敗モードと境界条件

  • count が0の場合、ループは実行されず何も変換されません。
  • メモリ領域のオーバーラップが発生した場合、変換結果は未定義となります。

保守担当者向けメモ

  • 本関数は os_toascii と異なり、プロトコル上の改行コード（LF）の特別な扱いを行いません。プロトコル文字列の変換には適さない場合があります。
  • 変換テーブルは os/bs2000/ebcdic.c 内で静的に定義されています。テーブルの更新が必要な場合は、同ファイルの定義を確認してください。
