ostat_req_cleanup

定義

  • 種別: function
  • 定義箇所: modules/md/md_ocsp.c:145

目的

OCSP リクエストに関連するメモリリソースを解放し、ステータス構造体内のポインタを安全にリセットする。

シグネチャまたは形状

static void ostat_req_cleanup(md_ocsp_status_t *ostat)

入力と出力

  • **入力**: ostat (md_ocsp_status_t *) - クリーンアップ対象の OCSP ステータス構造体へのポインタ。
  • **出力**: なし（副作用として ostat が指すメモリの内容が更新される）。

前提条件と不変条件

  • ostat は有効な md_ocsp_status_t 構造体を指している必要がある。
  • ostat->ocsp_req が非 NULL の場合、それは OCSP_REQUEST_free で解放可能な有効な OpenSSL オブジェクトであること。

アルゴリズムまたは振る舞い

1. ostat->ocsp_req が存在する場合、OCSP_REQUEST_free() を呼び出して OpenSSL のリクエストオブジェクトを解放し、ostat->ocsp_req を NULL に設定する。 2. md_data_clear() を呼び出し、ostat->req_der（DER 形式のリクエストデータ）に関連付けられたメモリをクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_ocsp.c:162
  • modules/md/md_ocsp.c:767
  • **呼び出し先**:
  • OCSP_REQUEST_free (OpenSSL)
  • md_data_clear (内部ユーティリティ)

関連データ構造

  • md_ocsp_status_t: OCSP ステータスを保持する構造体。本関数はこの構造体内の ocsp_req および req_der フィールドを操作する。

失敗モードと境界条件

  • ostat が NULL の場合、未定義の動作を引き起こす可能性がある（ソースコード上に明示的な NULL チェックは存在しない）。
  • ostat->ocsp_req が既に NULL の場合、そのフィールドに対する操作はスキップされるため安全である。

保守担当者向けメモ

  • 本関数は、OCSP リクエストのライフサイクル終了時や再試行前のクリーンアップとして利用される。
  • md_data_clear の詳細については、modules/md/md_util.h 等の基盤層の定義を参照すること。
