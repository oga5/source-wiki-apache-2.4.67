session_crypto_init

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:542

目的

Apache HTTP Server の起動時に apr_crypto ドライバを初期化し、セッション暗号化に必要なコンテキストをサーバープロセスに保持させる。

シグネチャまたは形状

static int session_crypto_init(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定用プール。
  • plog: ログ用プール。
  • ptemp: 一時作業用プール。
  • s: サーバーレコード。
  • **出力**:
  • OK (0): 正常終了。
  • APR エラーコード: 初期化失敗時。

前提条件と不変条件

  • mod_session_crypto の設定構造体 session_crypto_conf が s->module_config から取得可能であること。
  • APU_HAVE_CRYPTO が有効である環境であること。
  • 本関数は post_config フックとして登録されており、サーバー起動時に2回呼び出される仕様である。

アルゴリズムまたは振る舞い

1. **二重呼び出しの回避**: AP_SQ_MS_CREATE_PRE_CONFIG 状態であれば、初期化を行わず即座に OK を返す。 2. **ライブラリのロード**: 設定された暗号ライブラリ（conf->library）に対し、apr_crypto_init および apr_crypto_get_driver を実行する。 3. **コンテキスト生成**: apr_crypto_make を使用して暗号化コンテキスト apr_crypto_t を生成する。 4. **状態保持**: 生成されたコンテキストを apr_pool_userdata_set を介してサーバープロセスプール (s->process->pconf) に格納し、後続のリクエスト処理で利用可能にする。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_config により、Apache HTTP Server の起動プロセスから呼び出される。
  • **呼び出し先**: apr_crypto_init, apr_crypto_get_driver, apr_crypto_make, ap_log_error, apr_pool_userdata_set 等の APR/Apache API。

関連データ構造

  • session_crypto_conf: サーバー全体の設定（ライブラリ名、パラメータ等）を保持。
  • apr_crypto_t: 暗号化操作のコンテキスト。
  • apr_crypto_driver_t: 暗号化ドライバのインターフェース。

失敗モードと境界条件

  • **ライブラリロード失敗**: 指定された暗号ライブラリが見つからない、または初期化できない場合、エラーログを出力してエラーコードを返す。
  • **再初期化**: APR_EREINIT が発生した場合は警告ログを出力し、既存の設定を継続利用する。

保守担当者向けメモ

  • 本関数で生成された apr_crypto_t コンテキストは、CRYPTO_KEY というキーでプールに保存されるため、他の関数からこのキーを用いてコンテキストを取得する設計となっている。
  • 起動時の二重呼び出しは仕様であり、初回呼び出し時のリソース消費を避けるためのガード句が含まれている。
