slotmem_get

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:166

目的

指定されたスロットメモリインスタンスから、特定のIDに対応するスロットデータを読み込み、呼び出し元が提供したバッファへコピーします。

シグネチャまたは形状

apr_status_t slotmem_get(ap_slotmem_instance_t *slot, unsigned int id, unsigned char *dest, apr_size_t dest_len)

入力と出力

  • **入力**:
  • slot: 操作対象の ap_slotmem_instance_t 構造体へのポインタ。
  • id: 読み込み対象のスロットインデックス。
  • dest: データを格納する宛先バッファ。
  • dest_len: コピーするデータのサイズ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時はエラーコード（APR_ENOSHMAVAIL, APR_EINVAL, APR_NOTFOUND 等）。
  • dest: 成功時、対象スロットの内容がコピーされます。

前提条件と不変条件

  • slot が NULL でないこと。
  • id が slot->num（スロット総数）未満であること。
  • dest_len は、対象スロットのサイズを超えないこと（関数内で境界チェックは行われません）。
  • マルチスレッド環境で利用する場合、呼び出し側で排他制御を行う必要があります。

アルゴリズムまたは振る舞い

1. slot が有効か確認し、無効なら APR_ENOSHMAVAIL を返します。 2. id が範囲内か確認し、範囲外なら APR_EINVAL を返します。 3. AP_SLOTMEM_TYPE_PREGRAB フラグが設定されている場合、対象スロットの inuse フラグを確認し、未使用であれば APR_NOTFOUND を返します。 4. slotmem_dptr を呼び出し、対象スロットのメモリ位置を取得します。 5. 対象スロットの inuse フラグを 1 に設定します。 6. memcpy を使用して、スロットの内容を dest へコピーします。 7. APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/slotmem/mod_slotmem_plain.c:309 (プロバイダー構造体への登録)
  • modules/slotmem/mod_slotmem_shm.c:738 (プロバイダー構造体への登録)
  • **呼び出し先**:
  • slotmem_dptr

関連データ構造

  • ap_slotmem_instance_t: スロットメモリの管理構造体。

失敗モードと境界条件

  • slot が NULL: APR_ENOSHMAVAIL を返します。
  • id が範囲外: APR_EINVAL を返します。
  • AP_SLOTMEM_TYPE_PREGRAB が有効かつスロットが未使用: APR_NOTFOUND を返します。
  • slotmem_dptr の失敗: そのエラーコードを返します。
  • バッファオーバーフロー: dest_len がスロットサイズより大きい場合、メモリ破壊が発生する可能性があります。

保守担当者向けメモ

  • 本関数は memcpy を使用しており、境界チェックを行いません。呼び出し側で dest_len の妥当性を保証してください。
  • inuse フラグの更新（*inuse=1）は、PREGRAB フラグの有無に関わらず実行されます。
