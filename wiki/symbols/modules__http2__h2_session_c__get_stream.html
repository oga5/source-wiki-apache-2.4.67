get_stream

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:81

目的

HTTP/2 セッション内において、指定されたストリーム ID に対応する h2_stream オブジェクトを取得します。

シグネチャまたは形状

static h2_stream *get_stream(h2_session *session, int stream_id)

入力と出力

  • **入力**:
  • session: 現在の HTTP/2 セッションを管理する h2_session 構造体へのポインタ。
  • stream_id: 取得対象のストリームを識別する整数値。
  • **出力**:
  • 指定された ID に対応する h2_stream 構造体へのポインタ。該当するストリームが存在しない場合は NULL を返します。

前提条件と不変条件

  • session ポインタは有効である必要があります。
  • 内部で nghttp2 ライブラリのセッション状態 (session->ngh2) を参照するため、セッションが初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. session->ngh2（nghttp2 セッションオブジェクト）に対し、nghttp2_session_get_stream_user_data を呼び出します。 2. nghttp2 が管理するストリームに関連付けられたユーザーデータ（h2_stream 構造体）を検索し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c 内の各コールバックおよび処理関数（on_frame_recv_cb 等）から、フレーム処理時のストリーム特定のために頻繁に呼び出されます。
  • modules/http2/h2_mplx.c において、ストリームの多重化処理に関連して呼び出されます。
  • **呼び出し先**:
  • nghttp2_session_get_stream_user_data（nghttp2 ライブラリ関数）。

関連データ構造

  • h2_session: HTTP/2 セッションの全体状態を保持する構造体。
  • h2_stream: 個別の HTTP/2 ストリームの状態を保持する構造体。
  • nghttp2_session: nghttp2 ライブラリ側のセッション管理構造体。

失敗モードと境界条件

  • stream_id が無効な場合や、該当するストリームが nghttp2 側で管理されていない場合、nghttp2_session_get_stream_user_data は NULL を返し、本関数もそのまま NULL を返します。

保守担当者向けメモ

  • 本関数は nghttp2 のユーザーデータ機能に依存した単純なラッパーです。
  • h2_stream のライフサイクルは h2_session のプール管理と密接に関連しているため、取得したポインタをセッションの生存期間を超えて保持しないよう注意してください。
