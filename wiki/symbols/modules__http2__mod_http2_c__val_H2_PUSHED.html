val_H2_PUSHED

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_http2.c:241

目的

現在のリクエストがサーバープッシュによって開始されたものかどうかを判定し、環境変数用の文字列を返します。

シグネチャまたは形状

static const char *val_H2_PUSHED(apr_pool_t *p, server_rec *s,
                                 conn_rec *c, request_rec *r,
                                 h2_conn_ctx_t *conn_ctx)

入力と出力

  • **入力**:
  • conn_ctx: 接続コンテキスト（h2_conn_ctx_t）。
  • p, s, c, r: Apache HTTP Server の標準的なプール、サーバー、接続、リクエスト構造体（本関数内では直接使用されません）。
  • **出力**:
  • サーバープッシュによって生成されたストリームである場合は "PUSHED" を返します。
  • それ以外の場合は空文字列 "" を返します。

前提条件と不変条件

  • conn_ctx が NULL の場合、関数は安全に空文字列を返します。

アルゴリズムまたは振る舞い

1. conn_ctx が存在するか確認します。 2. conn_ctx->stream_id が設定されており、かつ H2_STREAM_CLIENT_INITIATED(conn_ctx->stream_id) が偽（クライアントが開始したストリームではない）である場合に、サーバープッシュと判断して "PUSHED" を返します。 3. 上記条件を満たさない場合は "" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c:303 の h2_var_def 配列内で参照されています。
  • **呼び出し先**: H2_STREAM_CLIENT_INITIATED マクロ（詳細な定義元はソース抜粋外）。

関連データ構造

  • h2_conn_ctx_t: 接続ごとのHTTP/2メタデータを保持する構造体。

失敗モードと境界条件

  • conn_ctx が NULL の場合、クラッシュすることなく空文字列を返します。
  • stream_id が 0 の場合、サーバープッシュとはみなされません。

保守担当者向けメモ

  • 本関数は http2_var_lookup を通じて環境変数として公開される値の計算に使用されます。
  • H2_STREAM_CLIENT_INITIATED マクロの判定ロジックに変更があった場合、本関数の挙動も影響を受けます。
