set_authtype

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_core.c:315

目的

Apache HTTP Server の設定ディレクティブ AuthType を解析し、その結果をディレクトリごとの設定構造体（authn_core_dir_conf）に格納する。

シグネチャまたは形状

static const char *set_authtype(cmd_parms *cmd, void *mconfig, const char *word1)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • mconfig: 現在のディレクトリ設定構造体（authn_core_dir_conf）。
  • word1: AuthType ディレクティブに指定された値（文字列）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • mconfig は authn_core_dir_conf 型としてキャスト可能であること。
  • word1 は ap_expr_parse_cmd が処理可能な式であること。

アルゴリズムまたは振る舞い

1. mconfig を authn_core_dir_conf 構造体にキャストする。 2. ap_expr_parse_cmd を呼び出し、word1 を式として解析する。この際、AP_EXPR_FLAG_STRING_RESULT フラグを指定する。 3. 解析が失敗した場合、エラーメッセージを生成して返す。 4. 解析が成功した場合、結果を aconfig->ap_auth_type に格納し、aconfig->auth_type_set を 1 に設定して完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_core.c:380 において、AP_INIT_TAKE1 マクロを通じて AuthType ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: ap_expr_parse_cmd（式評価エンジン）。

関連データ構造

  • authn_core_dir_conf: AuthType の設定値を保持する構造体。
  • cmd_parms: 設定解析時のコンテキスト情報。

失敗モードと境界条件

  • ap_expr_parse_cmd が構文エラーを検出した場合、エラーメッセージを返し、設定の適用を中断する。

保守担当者向けメモ

  • 本関数は AuthType の解析結果を ap_expr_info_t 形式で保持するため、実行時に動的な評価が可能となっている。
  • auth_type_set フラグは、設定が明示的に行われたことを示すために使用される。
