register_hooks

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:551

目的

mod_expires モジュールが必要とする出力フィルタおよびフィルタ挿入フックを Apache のフックシステムに登録する。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（設定用メモリプール）
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • Apache のモジュール読み込みプロセスにおいて、フック登録フェーズで呼び出されることを前提とする。
  • フィルタ MOD_EXPIRES は、キャッシュ保存フィルタ（AP_FTYPE_CONTENT_SET-1）よりも先に実行される必要があるため、AP_FTYPE_CONTENT_SET-2 として登録される。

アルゴリズムまたは振る舞い

1. ap_register_output_filter を呼び出し、expires_filter を MOD_EXPIRES という名前で出力フィルタチェーンに登録する。この際、優先順位を AP_FTYPE_CONTENT_SET-2 に設定する。 2. ap_hook_insert_error_filter を呼び出し、エラー発生時に expires_insert_filter が実行されるよう登録する。 3. ap_hook_insert_filter を呼び出し、通常のリクエスト処理時に expires_insert_filter が実行されるよう登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール初期化プロセス（直接的な呼び出し元コードは提供されたソース内には存在しない）。
  • **呼び出し先**:
  • ap_register_output_filter
  • ap_hook_insert_error_filter
  • ap_hook_insert_filter
  • expires_filter（フィルタとして登録）
  • expires_insert_filter（フックとして登録）

関連データ構造

  • apr_pool_t: 設定情報の保持に使用されるメモリプール。

失敗モードと境界条件

  • フィルタ登録関数が失敗した場合の挙動は Apache Core API に依存する。本関数自体はエラーハンドリングを行わない。

保守担当者向けメモ

  • 本関数は mod_expires の動作の起点となる。フィルタの実行順序を変更する場合は、AP_FTYPE_CONTENT_SET のオフセット値を慎重に調整すること。
  • ツールが提示した docs/manual/... 等の register_hooks への参照は、同名の関数を持つ他のモジュールとの混同であり、本シンボルの直接の呼び出し元ではない。
