priv_dir_cfg

定義

  • 種別: struct
  • 定義箇所: modules/arch/unix/mod_privileges.c:55

目的

ディレクトリ単位（<Directory> 等）の設定において、mod_privileges の動作モードを保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    priv_mode mode;
} priv_dir_cfg;

入力と出力

  • **入力**: 設定ファイル内の PrivilegesMode ディレクティブから読み込まれた値が mode フィールドに格納されます。
  • **出力**: リクエスト処理時に ap_get_module_config を通じて取得され、リクエストごとの権限降格処理の挙動を決定します。

前提条件と不変条件

  • mode は priv_mode 型（列挙型）であり、有効な動作モード（PRIV_SECURE 等）を保持している必要があります。
  • 本構造体は apr_palloc を用いてリクエストプールまたは設定プール上に割り当てられます。

アルゴリズムまたは振る舞い

  • privileges_create_dir_cfg 関数により初期化され、privileges_merge_dir_cfg 関数により親ディレクトリの設定とマージされます。
  • リクエスト処理中、privileges_req 関数内でこの構造体を参照し、現在のディレクトリコンテキストにおける権限分離の実行可否やモードを判断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • privileges_create_dir_cfg (メモリ確保)
  • privileges_merge_dir_cfg (設定マージ)
  • privileges_req (リクエスト時の設定取得)
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • priv_mode: mode フィールドの型。
  • priv_cfg: サーバー/vhost レベルの設定構造体。

失敗モードと境界条件

  • 設定ファイルで PrivilegesMode が指定されていない場合、デフォルト値が適用されます（privileges_create_dir_cfg の実装に依存）。
  • メモリ不足時には apr_palloc が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を管理するため、mod_privileges の設定マージロジックにおいて、親ディレクトリの設定を継承する際の挙動を適切に維持してください。
  • 現在のところ mode フィールドのみを保持していますが、将来的にディレクトリ単位で追加の権限制御が必要となった場合は、この構造体を拡張してください。
