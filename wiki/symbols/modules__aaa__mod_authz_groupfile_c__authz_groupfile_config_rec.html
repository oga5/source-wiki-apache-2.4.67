authz_groupfile_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authz_groupfile.c:60

目的

Apache HTTP Server の mod_authz_groupfile モジュールにおいて、ディレクトリ単位の設定情報を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    char *groupfile;
} authz_groupfile_config_rec;

入力と出力

  • **入力**: AuthGroupFile ディレクティブにより設定ファイルから読み込まれたグループ定義ファイルのパス文字列。
  • **出力**: なし（設定保持用構造体）。

前提条件と不変条件

  • groupfile メンバは、AuthGroupFile ディレクティブが指定された場合に、そのファイルパスを指す文字列として初期化されます。
  • ap_get_module_config を介してリクエストごとの設定として取得されます。

アルゴリズムまたは振る舞い

  • 本構造体は、認可プロバイダー（group_check_authorization や filegroup_check_authorization）が実行される際、対象となるグループ定義ファイルの場所を特定するために参照されます。
  • 設定の初期化時には apr_palloc を使用してメモリが割り当てられます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authz_groupfile.c:66 (初期化時)
  • modules/aaa/mod_authz_groupfile.c:140 (認可チェック時)
  • modules/aaa/mod_authz_groupfile.c:213 (認可チェック時)
  • **呼び出し先**:
  • APR_OFFSETOF マクロ（コマンドテーブル定義時）

関連データ構造

  • command_rec（authz_groupfile_cmds 内で本構造体のメンバオフセットを指定するために使用）

失敗モードと境界条件

  • AuthGroupFile ディレクティブが設定されていない場合、groupfile は NULL になる可能性があります。認可プロバイダー側でこの値が適切にチェックされている必要があります。

保守担当者向けメモ

  • 本構造体は非常にシンプルですが、mod_authz_groupfile の認可ロジックの起点となります。
  • groupfile のパスが相対パスの場合、サーバーのルートディレクトリからの相対パスとして扱われます。
