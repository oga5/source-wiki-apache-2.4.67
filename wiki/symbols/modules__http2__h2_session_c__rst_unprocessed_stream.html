rst_unprocessed_stream

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:92

目的

HTTP/2 セッションにおいて、処理が完了していない（未処理の）ストリームを特定し、それらを H2_ERR_NO_ERROR でリセット（RST_STREAM）するために使用されます。

シグネチャまたは形状

static int rst_unprocessed_stream(h2_stream *stream, void *ctx)

入力と出力

  • **入力**:
  • stream: 対象となる h2_stream 構造体へのポインタ。
  • ctx: コンテキスト（呼び出し元では h2_session が渡される）。
  • **出力**:
  • int: ストリームがリセットされた場合は 0 を返し、それ以外の場合は 1 を返します。

前提条件と不変条件

  • ストリームが H2_SS_CLOSED 状態に達していないことが条件となります。
  • クライアント開始ストリームの場合、セッションが local.accepting 状態ではなく、かつストリーム ID が local.accepted_max を超えている必要があります。
  • リモート開始ストリームの場合、セッションが remote.accepting 状態ではなく、かつストリーム ID が remote.accepted_max を超えている必要があります。

アルゴリズムまたは振る舞い

1. ストリームの状態とセッションの受け入れ状況を照合し、未処理のストリームであるかを判定します。 2. 未処理と判定された場合、h2_stream_rst を呼び出して H2_ERR_NO_ERROR エラーコードでストリームをリセットし、0 を返します。 3. 未処理でない場合は、何もせず 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_session.c:111 において、h2_mplx_c1_streams_do のコールバックとして呼び出されます。
  • **呼び出し先**: h2_stream_rst を呼び出します。

関連データ構造

  • h2_stream: ストリームの状態管理に使用されます。
  • h2_session: セッションの受け入れ状態（local/remote）の確認に使用されます。

失敗モードと境界条件

  • ストリームが既にクローズされている場合、リセット処理はスキップされます。
  • セッションがまだ新しいストリームを受け入れている最中の場合、この関数による強制リセットは行われません。

保守担当者向けメモ

  • 本関数は h2_mplx_c1_streams_do を介してイテレータとして動作するため、戻り値の 0 はイテレーションの継続・停止の制御として機能している点に注意してください。
