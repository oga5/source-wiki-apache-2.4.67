sed_commands_t

定義

  • 種別: typedef struct
  • 定義箇所: modules/filters/libsed.h:65
  • 実体: sed_commands_s

目的

コンパイル済みの sed スクリプト全体を保持するデータ構造です。ラベルテーブル、コマンドリスト、および実行に必要なメモリプール情報を格納し、スクリプトの解析結果を保持します。

シグネチャまたは形状

typedef struct sed_commands_s sed_commands_t;

※構造体の内部メンバは libsed.h の当該行では非公開です。

入力と出力

  • **入力**: sed_compile_string や sed_compile_file を通じて、テキスト形式の sed スクリプトが本構造体にコンパイルされます。
  • **出力**: コンパイルされた本構造体は、sed_eval_t を初期化する際の引数として使用され、ストリーム処理の実行基盤となります。

前提条件と不変条件

  • 本構造体は apr_pool_t に依存してメモリ管理が行われます。
  • sed_init_commands による初期化が必須です。
  • sed_destroy_commands を呼び出すことで、関連付けられたメモリリソースを解放する必要があります。

アルゴリズムまたは振る舞い

  • sed スクリプトのコンパイルフェーズにおいて、解析されたコマンド群（sed_reptr_t の連結リスト）を保持します。
  • 実行フェーズでは、この構造体を参照して各行に対するマッチングとコマンド適用が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_sed.c において、フィルタ設定の初期化時に apr_pcalloc で割り当てられ、sed_init_commands 等で操作されます。
  • **呼び出し先**: sed_init_commands, sed_compile_string, sed_compile_file, sed_get_finalize_error, sed_canbe_finalized, sed_destroy_commands, sed_init_eval, sed_reset_eval などの libsed API が本構造体を引数として受け取ります。

関連データ構造

  • sed_commands_s: 本型の実体となる構造体定義。
  • sed_reptr_t: コンパイルされた個々の sed コマンドノード。
  • sed_eval_t: 本構造体を使用して実行状態を管理するコンテキスト。
  • apr_pool_t: メモリ管理に使用される Apache Portable Runtime のプール。

失敗モードと境界条件

  • sed_init_commands 等の関数が APR_SUCCESS 以外のステータスを返す場合、コンパイルや初期化に失敗しています。
  • スクリプトの複雑さやネストは、SED_NLINES, SED_DEPTH, SED_LABSIZE などの定数による制限を受けます。

保守担当者向けメモ

  • 本構造体は純粋な C インターフェースとして定義されています。
  • メモリ所有権は apr_pool_t に帰属するため、ライフサイクル管理には注意してください。
  • 内部構造（sed_commands_s）の詳細は libsed.h の他の箇所または実装ファイルを参照してください。
