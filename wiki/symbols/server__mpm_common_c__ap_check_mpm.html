ap_check_mpm

定義

  • 種別: function
  • 定義箇所: server/mpm_common.c:559

目的

現在ロードされているマルチプロセッシングモジュール（MPM）の整合性を検証し、サーバーの再起動時にMPMが変更されていないことを確認します。

シグネチャまたは形状

const char *ap_check_mpm(void)

入力と出力

  • **入力**: なし（内部のフック状態および ap_show_mpm() の結果を参照）。
  • **出力**:
  • 正常時: NULL
  • エラー時: エラー内容を示す文字列（const char *）。

前提条件と不変条件

  • サーバーの起動および再起動プロセス中に呼び出されることを想定しています。
  • _hooks.link_mpm が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. **MPMロード確認**: _hooks.link_mpm を参照し、MPMがロードされていない場合、または複数のMPMがロードされている場合はエラーメッセージを返します。 2. **MPM変更チェック**:

  • 初回呼び出し時: 現在のMPM名を last_mpm_name として静的変数に保存します。
  • 2回目以降: ap_show_mpm() が返す現在のMPM名と last_mpm_name を比較します。名前が異なる場合、再起動によるMPM変更は許可されないため、エラーメッセージを返します。

    • 3. **正常終了**: 整合性が確認できれば NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/config.c:2348 にて呼び出されています。
  • **呼び出し先**:
  • ap_show_mpm()（MPM名取得）
  • apr_pstrdup()（メモリ確保）

関連データ構造

  • _hooks.link_mpm: ロードされたMPMを保持するフック構造体。

失敗モードと境界条件

  • **MPM未ロード**: _hooks.link_mpm が空の場合、「No MPM loaded.」を返します。
  • **MPM重複ロード**: _hooks.link_mpm->nelts > 1 の場合、「More than one MPM loaded.」を返します。
  • **再起動時のMPM変更**: 実行中にMPMが切り替わった場合、「The MPM cannot be changed during restart.」を返します。

保守担当者向けメモ

  • 本関数は静的変数 last_mpm_name を使用して状態を保持しているため、プロセス生存期間中を通じてMPMの変更を禁止する不変条件を維持します。
  • ap_show_mpm() の戻り値に依存しているため、MPM側の実装が正しく名前を返していることが前提となります。
