md_subscription

定義

  • 種別: struct
  • 定義箇所: modules/md/md_event.c:25

目的

イベント通知システムにおいて、登録されたコールバック関数とその実行コンテキストを保持するためのリンクリストノードです。

シグネチャまたは形状

typedef struct md_subscription {
    struct md_subscription *next;
    md_event_cb *cb;
    void *baton;
} md_subscription;

入力と出力

  • **入力**: 構造体メンバーとして、次のノードへのポインタ (next)、実行されるコールバック関数 (cb)、およびコールバックに渡される任意のデータポインタ (baton) を保持します。
  • **出力**: なし（データ保持用の構造体）。

前提条件と不変条件

  • next はリストの終端で NULL となります。
  • cb はイベント発生時に呼び出される関数ポインタです。
  • baton はコールバック関数が必要とする任意のコンテキストデータへのポインタであり、NULL も許容されます。

アルゴリズムまたは振る舞い

  • 本構造体は md_event_subscribe によってリストに追加され、md_event_raise または md_event_holler によってリストを辿りながら各ノードの cb が実行されます。
  • リストの先頭は EVNT グローバル構造体によって管理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: md_event_subscribe が本構造体のインスタンスを生成し、リストに連結します。md_event_raise および md_event_holler がリストを走査して参照します。
  • **呼び出し先**: 本構造体自体は関数を呼び出しませんが、保持している cb メンバーがイベント発生時に実行されます。

関連データ構造

  • EVNT: イベント管理用のグローバル構造体。本構造体のリスト (subs) を保持します。
  • md_event_cb: cb メンバーが指すコールバック関数の型定義。

失敗モードと境界条件

  • 購読解除機能が存在しないため、一度登録されたノードはプロセスの生存期間中保持されます。
  • メモリ割り当て失敗時の挙動については、本構造体定義からは直接判断できません。

保守担当者向けメモ

  • 本構造体はスレッドセーフではありません。マルチスレッド環境で利用する場合は、リスト操作時に外部での排他制御が必要です。
  • baton ポインタのライフサイクルは、この構造体が保持される期間（プロセス終了まで）有効である必要があります。
