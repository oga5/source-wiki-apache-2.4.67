sed_get_finalize_error

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:127

目的

コンパイル済みの sed スクリプトが、構文的に正しく終了しているか（未解決のラベルや不完全なブロック構造がないか）を検証する。

シグネチャまたは形状

char* sed_get_finalize_error(const sed_commands_t *commands, apr_pool_t* pool)

入力と出力

  • **入力**:
  • commands: 検証対象の sed_commands_t 構造体へのポインタ。
  • pool: エラーメッセージ生成時に使用する apr_pool_t。
  • **出力**:
  • 正常終了時: NULL。
  • エラー時: エラー内容を示す文字列（SEDERR_TMOMES、SEDERR_ULMES、または SEDERR_INTERNAL）。

前提条件と不変条件

  • commands は sed_init_commands によって初期化済みであること。
  • pool は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. **ブロック構造の検証**: commands->depth が 0 でない場合、閉じられていないブロックがあると判断し SEDERR_TMOMES を返す。 2. **ラベルの検証**: commands->labtab から commands->lab までの範囲でラベルを走査する。

  • address == 0: ラベルが定義されていない（未解決）と判断し、SEDERR_ULMES を用いてエラーメッセージを生成して返す。
  • chain != 0: ラベルチェーンが残っている場合、内部エラーとして SEDERR_INTERNAL を返す。

    • 3. **成功**: すべてのチェックを通過した場合、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/sed1.c:415 にて、実行エンジンがスクリプトの最終化状態を確認するために呼び出している。
  • **呼び出し先**: apr_psprintf（エラーメッセージ生成時）。

関連データ構造

  • sed_commands_t: sed スクリプトのコンパイル済みコマンドとラベルテーブルを保持する構造体。
  • sed_label_t: ラベルの定義と参照チェーンを保持する構造体。

失敗モードと境界条件

  • **ブロック不整合**: { で開始されたブロックが } で閉じられていない場合。
  • **未定義ラベル**: b や t コマンドで参照されているラベルが、スクリプト内に存在しない場合。
  • **内部エラー**: ラベル解決の過程で予期せぬチェーンが残存している場合。

保守担当者向けメモ

  • 本関数はコンパイルの最終段階で呼び出される。エラーメッセージの定数（SEDERR_...）は libsed.h 等で定義されている。
  • commands->labtab + 1 から開始しているのは、インデックス 0 が特殊な用途（あるいは未使用）であるためと推測される。
