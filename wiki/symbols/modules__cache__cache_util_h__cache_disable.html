cache_disable

定義

  • 種別: struct
  • 定義箇所: modules/cache/cache_util.h:114

目的

キャッシュを無効化すべきURLパターンを保持するための設定構造体です。

シグネチャまたは形状

struct cache_disable {
    apr_uri_t url;
    apr_size_t pathlen;
};

入力と出力

  • **入力**: mod_cache の設定フェーズにおいて、CacheDisable ディレクティブにより指定されたURLパスが apr_uri_t 形式で格納されます。
  • **出力**: キャッシュの有効性判定時に、リクエストされたURLと照合するために使用されます。

前提条件と不変条件

  • url フィールドには、キャッシュを無効化する対象のURI情報が格納されます。
  • pathlen は、対応するURLパスの長さを示します。

アルゴリズムまたは振る舞い

本構造体は、サーバーまたはディレクトリ単位の設定（cache_server_conf 等）において、キャッシュ対象から除外するURLリストを管理するために使用されます。リクエスト処理時に、このリストを走査して現在のリクエストURLが除外対象に該当するかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • modules/cache/cache_util.c: 設定構造体から配列要素として取得し、URL照合に使用。
  • modules/cache/mod_cache.c: 設定の初期化および CacheDisable ディレクティブの処理時にインスタンス化。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • cache_server_conf: サーバー全体のキャッシュ設定を保持する構造体。本構造体の配列を保持します。
  • apr_uri_t: APRが提供するURI解析用構造体。

失敗モードと境界条件

  • 設定時に不正なURLが指定された場合、照合ロジックが意図通りに機能しない可能性があります。
  • メモリ不足時には、apr_array_make による配列確保が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cache の設定管理の一部です。CacheDisable ディレクティブの追加・削除に伴い、mod_cache.c 内の配列操作と整合性を保つ必要があります。
  • pathlen の値が実際の url.path の長さと一致していることを確認してください。
