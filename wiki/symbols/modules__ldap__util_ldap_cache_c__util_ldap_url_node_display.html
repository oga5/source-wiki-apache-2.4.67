util_ldap_url_node_display

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache.c:79

目的

LDAP URL キャッシュノードに関連付けられた検索、比較、および DN 比較の各キャッシュ統計情報を HTML テーブルの行形式で出力します。

シグネチャまたは形状

void util_ldap_url_node_display(request_rec *r, util_ald_cache_t *cache, void *n)

入力と出力

  • **入力**:
  • r: Apache のリクエストレコード（出力先）。
  • cache: キャッシュ構造体（本関数内では未使用）。
  • n: 表示対象の util_url_node_t 構造体へのポインタ。
  • **出力**: なし。ap_rprintf を通じて HTTP レスポンスボディに HTML テーブル行を出力します。

前提条件と不変条件

  • n は util_url_node_t 型としてキャスト可能である必要があります。
  • node 内の search_cache、compare_cache、dn_compare_cache が有効な util_ald_cache_t を指している必要があります。

アルゴリズムまたは振る舞い

1. 内部ループで 0 から 2 まで反復し、以下の順序でキャッシュノードを処理します。

  • 0: search_cache ("Searches")
  • 1: compare_cache ("Compares")
  • 2: dn_compare_cache ("DN Compares")

    • 2. 各キャッシュノードについて、marktime が存在する場合は apr_ctime で文字列化し、存在しない場合は空文字列とします。 3. ap_rprintf を使用し、以下の情報を HTML の <tr> 要素として出力します。

  • URL 文字列、キャッシュタイプ名、キャッシュサイズ、最大エントリ数、現在のエントリ数、TTL（秒）、フルマーク、最終更新時刻。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap_cache.c:462 (直接的な参照を確認)
  • **呼び出し先**:
  • apr_ctime (APR ライブラリ)
  • ap_rprintf (Apache HTTP Server API)
  • apr_time_sec (APR ライブラリ)

関連データ構造

  • util_url_node_t: キャッシュノードの基底構造。
  • util_ald_cache_t: 各種キャッシュの統計情報（サイズ、TTL、エントリ数等）を保持する構造体。
  • request_rec: Apache のリクエストコンテキスト。

失敗モードと境界条件

  • cache_node が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • node->url が NULL の場合、ap_rprintf の出力時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は HTML 生成に特化しています。出力フォーマットを変更する場合は、util_ldap_cache_mgr.c 側のテーブルヘッダー定義と整合性を取る必要があります。
  • cache 引数は現在の実装では使用されていませんが、インターフェースの互換性のため保持されています。
