ping_new_session

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_proxy_session.c:146

目的

新規プロキシセッションにおける PING 状態の初期化および生存確認の開始を行います。

シグネチャまたは形状

static void ping_new_session(h2_proxy_session *session, proxy_conn_rec *p_conn)

入力と出力

  • **入力**:
  • session: 操作対象の h2_proxy_session 構造体。
  • p_conn: 接続設定を含む proxy_conn_rec 構造体。
  • **出力**: なし（副作用として session の状態を更新）。

前提条件と不変条件

  • p_conn および p_conn->worker が有効であること。
  • session が初期化済みであること。

アルゴリズムまたは振る舞い

1. session->save_timeout を -1 に設定します。 2. p_conn の設定に基づき、session->ping_timeout を設定します（設定がない場合は -1）。 3. session->ping_state を H2_PING_ST_NONE にリセットします。 4. enter_ping_state を呼び出し、セッションを H2_PING_ST_AWAIT_ANY 状態へ遷移させます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c:784
  • **呼び出し先**: enter_ping_state

関連データ構造

  • h2_proxy_session: セッションの生存確認状態やタイムアウト値を保持。
  • proxy_conn_rec: バックエンド接続のワーカースレッド設定（ping_timeout 等）を保持。

失敗モードと境界条件

  • p_conn->worker が NULL の場合、あるいは ping_timeout_set が偽の場合、ping_timeout は -1 となり、タイムアウト機能は無効化されます。

保守担当者向けメモ

  • 本関数はセッション開始時の初期化フローの一部です。PING タイムアウトの挙動を変更する場合は、enter_ping_state の実装と併せて確認してください。
