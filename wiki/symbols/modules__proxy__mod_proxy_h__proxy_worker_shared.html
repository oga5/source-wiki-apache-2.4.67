proxy_worker_shared

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:424

目的

バックエンドサーバー（ワーカー）に関する共有状態を保持するための構造体です。マルチプロセス環境において、ap_slotmem を介してプロセス間で共有される情報を定義します。

シグネチャまたは形状

typedef struct {
    /* 識別子および接続先情報 */
    char      name[PROXY_WORKER_MAX_NAME_SIZE];
    char      scheme[PROXY_WORKER_MAX_SCHEME_SIZE];
    char      hostname[PROXY_WORKER_MAX_HOSTNAME_SIZE];
    char      hostname_ex[PROXY_RFC1035_HOSTNAME_SIZE];
    char      uds_path[PROXY_WORKER_MAX_NAME_SIZE];
    apr_port_t port;
    
    /* 負荷分散および接続設定 */
    int       lbset, retries, lbstatus, lbfactor;
    int       min, smax, hmax;
    apr_interval_time_t ttl, retry, timeout, acquire, ping_timeout, conn_timeout;
    
    /* 統計および状態管理 */
    unsigned int status;
    apr_size_t   elected, busy;
    apr_off_t    transferred, read;
    apr_time_t   updated, error_time;
    
    /* ヘルスチェックおよびその他設定 */
    char      hcuri[PROXY_WORKER_MAX_ROUTE_SIZE];
    char      hcexpr[PROXY_WORKER_MAX_SCHEME_SIZE];
    hcmethod_t method;
    /* ... (ビットフィールドおよびフラグ群) ... */
} proxy_worker_shared;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。ただし、プロキシサブシステム内では、共有メモリ上のこの構造体に対して、設定の読み込みや、負荷分散アルゴリズムによる統計情報の更新が行われます。

前提条件と不変条件

  • **共有メモリ**: 本構造体は ap_slotmem を介して管理されるため、マルチプロセス環境での更新には proxy_mutex による排他制御が必須です。
  • **バイナリ互換性**: 構造体のメンバ追加は Apache のマイナーバージョンアップ等で発生するため、バイナリ互換性に注意が必要です。

アルゴリズムまたは振る舞い

  • **状態共有**: プロセス間でワーカーの負荷状況（lbstatus）や統計情報（elected, busy 等）を同期します。
  • **接続管理**: min, smax, hmax などのパラメータに基づき、接続プール（proxy_conn_pool）の挙動を制御します。
  • **ヘルスチェック**: hcuri や method を用いて、バックエンドの生存確認を行うための設定値を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy および各プロトコルハンドラ（mod_proxy_http 等）が、設定の参照や状態更新のために本構造体を利用します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは含みません。

関連データ構造

  • proxy_worker: プロセスローカルな状態と本構造体（共有メモリ）を保持する上位構造体。
  • proxy_conn_pool: 接続管理のためのプール構造体。
  • proxy_balancer: 複数のワーカーを束ねる負荷分散グループ。

失敗モードと境界条件

  • **共有メモリ競合**: 排他制御が不十分な場合、プロセス間で統計情報が不整合になる可能性があります。
  • **サイズ制限**: name や hostname_ex などの文字列フィールドは固定長であり、これを超える値は切り詰められるか、設定エラーとなります。

保守担当者向けメモ

  • 本構造体は mod_proxy のインターフェース契約の一部です。メンバの追加や変更を行う際は、include/ap_mmn.h に記載されているモジュール・マジック・ナンバー（MMN）の更新が必要となる場合があります。
  • proxy_worker 構造体内の s ポインタを通じてアクセスされます。
