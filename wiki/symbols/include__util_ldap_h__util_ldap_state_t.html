util_ldap_state_t

定義

  • 種別: struct
  • 定義箇所: include/util_ldap.h:147

目的

Apache HTTP Server の mod_ldap における、モジュール全体のグローバルな設定状態およびキャッシュ状態を保持するための構造体です。

シグネチャまたは形状

typedef struct util_ldap_state_t {
    apr_pool_t *pool;
#if APR_HAS_THREADS
    apr_thread_mutex_t *mutex;
#endif
    apr_global_mutex_t *util_ldap_cache_lock;
    apr_size_t cache_bytes;
    char *cache_file;
    long search_cache_ttl;
    long search_cache_size;
    long compare_cache_ttl;
    long compare_cache_size;
    struct util_ldap_connection_t *connections;
    apr_array_header_t *global_certs;
    int ssl_supported;
    int secure;
    int secure_set;
    int verify_svr_cert;
#if APR_HAS_SHARED_MEMORY
    apr_shm_t *cache_shm;
    apr_rmm_t *cache_rmm;
#endif
    void *util_ldap_cache;
    long connectionTimeout;
    struct timeval *opTimeout;
    int debug_level;
    apr_interval_time_t connection_pool_ttl;
    int retries;
    apr_interval_time_t retry_delay;
} util_ldap_state_t;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。ただし、util_ldap_cache_init や util_ald_cache_display などの関数において、設定値の読み取りやキャッシュ状態の更新対象として引数に渡されます。

前提条件と不変条件

  • APR_HAS_LDAP が定義されている環境で有効です。
  • 共有メモリキャッシュを使用する場合、APR_HAS_SHARED_MEMORY が有効である必要があります。
  • マルチスレッド環境下では、mutex による接続リストの保護が必須です。

アルゴリズムまたは振る舞い

  • **キャッシュ管理**: 共有メモリ（cache_shm）とリソース管理（cache_rmm）を用いて、検索および比較結果を保持します。
  • **接続管理**: connections リストを保持し、LDAP サーバーへの接続プールを管理します。
  • **再試行ロジック**: バインド、検索、比較操作が失敗した場合、retries 回数まで retry_delay の間隔を空けて再試行を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: include/ap_mmn.h で定義されるモジュール間インターフェースの変更履歴において、本構造体のフィールド追加・変更が記録されています。また、mod_authnz_ldap 等のクライアントモジュールが設定保持のために参照します。
  • **呼び出し先**: util_ldap_cache_init、util_ald_cache_display、ldap_cache_lock 等の mod_ldap 内部関数が本構造体を操作します。

関連データ構造

  • util_ldap_connection_t: LDAP 接続状態を保持する構造体。
  • apr_pool_t: メモリ割り当て用プール。
  • apr_global_mutex_t: キャッシュ操作の排他制御用。
  • apr_shm_t, apr_rmm_t: 共有メモリ管理用。

失敗モードと境界条件

  • APR_HAS_THREADS や APR_HAS_SHARED_MEMORY が無効な環境では、対応するフィールドが構造体から除外されます。
  • キャッシュサイズ（cache_bytes）や TTL 設定が不適切な場合、メモリ不足やキャッシュ効率の低下を招く可能性があります。

保守担当者向けメモ

  • 本構造体は Apache のモジュール間インターフェース（MMN）の一部であり、フィールドの変更はバイナリ互換性に影響を与える可能性があります。
  • connection_pool_ttl や retries などの設定値は、パフォーマンスチューニングの主要なパラメータです。
