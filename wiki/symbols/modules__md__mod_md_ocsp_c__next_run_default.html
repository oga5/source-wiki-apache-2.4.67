next_run_default

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_ocsp.c:151

目的

バックグラウンドタスク（主に OCSP レスポンスの更新）の次回の実行タイミングを計算し、デフォルトの待機時間を算出する。

シグネチャまたは形状

apr_time_t next_run_default(void)

入力と出力

  • **入力**: なし。
  • **出力**: 現在時刻に 1 時間を加算した apr_time_t 型の時刻値を返す。

前提条件と不変条件

  • apr_time_now() が有効なシステム時刻を返す環境であること。
  • MD_SECS_PER_HOUR が 3600 秒として定義されていること。

アルゴリズムまたは振る舞い

1. apr_time_now() を呼び出し、現在のシステム時刻を取得する。 2. MD_SECS_PER_HOUR を用いて 1 時間分の時間を計算する。 3. 現在時刻に 1 時間を加算した値を戻り値として返す。これにより、最低でも 1 時間間隔での実行を保証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_ocsp.c:182 (直接の呼び出し)
  • modules/md/mod_md_drive.c:279 (引数 dctx を渡して呼び出しているが、関数定義側は引数を取らないため、実際の動作には注意が必要)
  • **呼び出し先**:
  • apr_time_now (APR ライブラリ)
  • apr_time_from_sec (APR ライブラリ)

関連データ構造

  • apr_time_t: APR で定義される時刻表現型。

失敗モードと境界条件

  • システム時刻が遡るような環境では、期待される間隔で実行されない可能性がある。
  • apr_time_from_sec のオーバーフローについては、apr_time_t の範囲に依存する。

保守担当者向けメモ

  • mod_md_drive.c:279 での呼び出しにおいて引数 dctx が渡されているが、本関数の定義は引数を取らない。この不一致が意図的なものか、あるいは将来的な拡張の残滓かを確認すること。
  • 実行間隔を調整する場合は MD_SECS_PER_HOUR の定義を確認すること。
