authz_dbd_prepare

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbd.c:78

目的

Apache HTTP Server の設定ディレクティブ（AuthzDBDQuery および AuthzDBDRedirectQuery）で指定された SQL クエリを、mod_dbd のプリペアドステートメントとして登録する。

シグネチャまたは形状

static const char *authz_dbd_prepare(cmd_parms *cmd, void *cfg, const char *query)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • cfg: 設定構造体（authz_dbd_cfg）。
  • query: 設定ファイルから読み込まれた SQL クエリ文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • mod_dbd がロードされている必要がある。
  • .htaccess コンテキストでの使用は禁止されている（ap_check_cmd_context により制限）。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、NOT_IN_HTACCESS コンテキストであることを確認する。 2. mod_dbd の ap_dbd_prepare 関数が未取得であれば、APR_RETRIEVE_OPTIONAL_FN を使用して取得を試みる。失敗した場合はエラーを返す。 3. label_num をインクリメントし、authz_dbd_N という形式のユニークなラベルを生成する。 4. dbd_prepare を呼び出し、サーバーコンテキスト、クエリ、ラベルを渡してプリペアドステートメントを登録する。 5. ap_set_string_slot を使用して、生成したラベルを設定構造体に保存する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authz_dbd.c 内の AP_INIT_TAKE1 マクロ定義（AuthzDBDQuery および AuthzDBDRedirectQuery ディレクティブ）。
  • **呼び出し先**:
  • ap_check_cmd_context
  • APR_RETRIEVE_OPTIONAL_FN
  • apr_psprintf
  • dbd_prepare（mod_dbd が提供する ap_dbd_prepare）
  • ap_set_string_slot

関連データ構造

  • cmd_parms: 設定コマンドのコンテキスト。
  • authz_dbd_cfg: 設定を保持する構造体。
  • mod_dbd が管理するプリペアドステートメント（内部的に label で識別）。

失敗モードと境界条件

  • mod_dbd がロードされていない場合、エラーメッセージを返す。
  • .htaccess 内で呼び出された場合、ap_check_cmd_context によりエラーが返される。

保守担当者向けメモ

  • label_num は静的変数であり、プロセス内で一意のラベルを生成するために使用される。
  • dbd_prepare 関数ポインタはモジュール内でキャッシュされるため、初回呼び出し時のみ APR_RETRIEVE_OPTIONAL_FN が実行される。
