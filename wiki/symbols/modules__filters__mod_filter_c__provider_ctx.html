provider_ctx

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_filter.c:50

目的

フィルタプロバイダが filter_init 実行時に設定したコンテキスト値を保持し、後続の処理で利用可能にするための構造体です。

シグネチャまたは形状

struct provider_ctx {
    provider_ctx *next;
    void *init_ctx;
};

入力と出力

  • 入力: apr_pcalloc を介してリクエストプール (f->r->pool) からメモリが割り当てられます。
  • 出力: 連結リストのノードとして、プロバイダ固有の初期化コンテキスト (init_ctx) を保持します。

前提条件と不変条件

  • filter_init の実行過程で生成されることが想定されています。
  • 連結リスト構造を維持するため、next ポインタは適切に管理される必要があります。

アルゴリズムまたは振る舞い

  • プロバイダが filter_init で生成したコンテキストを、mod_filter の実行コンテキスト内で永続化するために使用されます。
  • 複数のプロバイダが関与する場合、連結リストとして順次保持されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_filter.c 内のフィルタ初期化および実行ロジック（filter_init 等）。
  • 呼び出し先: 特定されていません（プロバイダ固有のコンテキストを保持するコンテナとして機能します）。

関連データ構造

  • ap_filter_provider_t: プロバイダの定義を保持する構造体。
  • harness_ctx: 実行時のコンテキスト。

失敗モードと境界条件

  • メモリ割り当て失敗時: apr_pcalloc が失敗した場合の挙動は Apache のメモリ管理ポリシーに依存します。

保守担当者向けメモ

  • 本構造体は、プロバイダが filter_init で設定した値を保存するための「入れ物」です。
  • プロバイダの実装によっては init_ctx に複雑な構造体が格納されるため、型キャストの整合性に注意してください。
