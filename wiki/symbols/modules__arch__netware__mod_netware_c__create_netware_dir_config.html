create_netware_dir_config

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_netware.c:46

目的

NetWare モジュール用のディレクトリ単位の設定構造体 (netware_dir_config) を生成し、初期化します。

シグネチャまたは形状

static void *create_netware_dir_config(apr_pool_t *p, char *dir)

入力と出力

  • **入力**:
  • p: 設定情報の割り当てに使用するメモリプール (apr_pool_t*)。
  • dir: 設定が適用されるディレクトリパス (char*)。
  • **出力**:
  • 初期化済みの netware_dir_config 構造体へのポインタを返します。

前提条件と不変条件

  • 指定されたメモリプール p は有効である必要があります。
  • 構造体内の各テーブル (file_type_handlers, file_handler_mode, extra_env_vars) は、それぞれ 10 要素の初期容量で作成されます。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して netware_dir_config 構造体分のメモリを確保します。 2. apr_table_make を呼び出し、以下のテーブルを初期化します。

  • file_type_handlers: ファイル拡張子とハンドラの対応表。
  • file_handler_mode: ファイルハンドラの実行モード管理表。
  • extra_env_vars: 追加の環境変数格納表。

    • 3. デフォルト設定として、file_type_handlers テーブルにキー "NLM"、値 "OS" を設定します。 4. 初期化された構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_netware.c:198 (モジュールの設定初期化フェーズにて呼び出されます)。
  • **呼び出し先**: apr_palloc, apr_table_make, apr_table_setn (すべて APR ライブラリ関数)。

関連データ構造

  • netware_dir_config: 本関数が生成するディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性があります。
  • テーブル作成時の初期容量 (10) を超える設定が追加された場合、APR テーブルの仕様に従い自動的に拡張されます。

保守担当者向けメモ

  • 本関数は NetWare 環境専用です。他のプラットフォームではコンパイルされません。
  • デフォルトで設定される {"NLM", "OS"} は、NetWare 上での標準的な実行形式を指します。
