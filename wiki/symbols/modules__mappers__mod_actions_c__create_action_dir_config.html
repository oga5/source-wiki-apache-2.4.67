create_action_dir_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_actions.c:62

目的

ディレクトリごとの設定構造体 action_dir_config を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_action_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定の割り当てに使用するメモリプール。
  • char *dummy: 使用されない引数（Apache の設定コールバックシグネチャに適合させるためのもの）。
  • **出力**:
  • void *: 初期化された action_dir_config 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体はゼロクリアされた状態で割り当てられる。
  • action_types フィールドは、4つのエントリを保持可能な apr_table_t として初期化される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、action_dir_config サイズのメモリをプール p から確保する。 2. apr_table_make を呼び出し、action_types テーブルを初期サイズ 4 で作成して構造体に格納する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_actions.c:226 （ディレクトリ設定作成用ハンドラとして参照されている）。
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_table_make (APR)

関連データ構造

  • action_dir_config: 本関数が初期化対象とする構造体。MIME タイプごとのアクションとメソッドごとのスクリプトを保持する。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がプロセスを終了させる（APR の標準的な挙動）。
  • dummy 引数は無視されるため、値の内容は影響しない。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズでディレクトリごとの設定を生成するために使用される。
  • action_types の初期サイズ（4）は、一般的な設定数に基づいている。設定項目が増加する傾向がある場合は、この初期値を調整することを検討すること。
