open_listeners

定義

  • 種別: function
  • 定義箇所: server/listen.c:548

目的

設定済みのリスナーリスト（ap_listeners）を走査し、各リスナーに対してソケットの作成、バインド、およびリスン状態への移行を行う。また、IPv6/IPv4 のバインド順序の最適化や、不要なリスナーの除外、マルチリスナー環境でのノンブロッキング設定を適用する。

シグネチャまたは形状

static int open_listeners(apr_pool_t *pool)

入力と出力

  • **入力**: pool (メモリ割り当て用の APR プール)
  • **出力**: 成功時は 0、失敗時は -1 を返す。また、正常にオープンされたリスナーの総数が内部的にカウントされる。

前提条件と不変条件

  • ap_listeners は ap_listen_rec 構造体の連結リストとして構築済みであること。
  • old_listeners は、本関数呼び出し時にクリーンアップ対象として参照可能であること。
  • IPv6 環境下では、::（IPv6）と 0.0.0.0（IPv4）のバインド順序が適切に調整される。

アルゴリズムまたは振る舞い

1. **リストの走査と最適化**: ap_listeners を走査し、IPv6/IPv4 のバインド順序を調整する。IPv6 ソケットが IPv4 を包含できる場合、IPv4 のバインドをスキップする等の重複排除を行う。 2. **ソケット生成**: make_sock() を呼び出し、各リスナーに対してソケットの作成と設定を行う。 3. **旧リスナーの破棄**: ap_close_listeners_ex() を呼び出し、古いリスナーをクローズする。 4. **ノンブロッキング設定**: 複数のリスナーが存在する場合、apr_socket_opt_set を用いて全リスナーをノンブロッキングモードに設定する。 5. **クリーンアップ登録**: プロセス終了時にリスナーをクローズするためのクリーンアップハンドラ（close_listeners_on_exec）をプールに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/listen.c:771（ap_setup_listeners 内から呼び出されることが示唆されている）
  • **呼び出し先**: make_sock, ap_close_listeners_ex, apr_socket_opt_set, apr_pool_userdata_get, apr_pool_cleanup_register

関連データ構造

  • ap_listen_rec: リスナー情報を保持する連結リスト構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • make_sock() が失敗した場合、致命的なエラーとして -1 を返す。
  • IPv6/IPv4 のバインド競合が発生した場合、リストの並び替えや特定のリスナーの除外が行われる。
  • ノンブロッキング設定の適用に失敗した場合、エラーログを出力して -1 を返す。

保守担当者向けメモ

  • AP_NONBLOCK_WHEN_MULTI_LISTEN マクロが定義されている環境では、リスナー数に応じてソケットのブロッキングモードが動的に変更される。
  • systemd ソケットアクティベーションに関連するロジックは、HAVE_SYSTEMD マクロの有無により挙動が分岐する可能性がある。
  • make_sock の戻り値が APR_SUCCESS でない場合、致命的エラーとして処理を中断する設計となっている。
