extract_responder_uri

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_ocsp.c:24

目的

X.509 証明書の AIA (Authority Information Access) 拡張から、OCSP レスポンダの URI を抽出します。

シグネチャまたは形状

static const char *extract_responder_uri(X509 *cert, apr_pool_t *pool)

入力と出力

  • **入力**:
  • cert: 抽出対象の OpenSSL X509 証明書オブジェクト。
  • pool: URI 文字列を複製するために使用する apr_pool_t メモリプール。
  • **出力**:
  • 成功時: 抽出された OCSP レスポンダの URI 文字列（pool 上に割り当て）。
  • 失敗時: NULL（AIA 拡張が存在しない、または OCSP 用の URI が見つからない場合）。

前提条件と不変条件

  • cert は有効な X509 オブジェクトである必要があります。
  • 抽出された URI は apr_pstrdup により提供されたメモリプールにコピーされるため、呼び出し元はメモリ管理をプールに依存できます。

アルゴリズムまたは振る舞い

1. X509_get_ext_d2i を使用して、証明書から NID_info_access (AIA) 拡張を取得します。 2. 拡張が存在しない場合は NULL を返します。 3. 取得した ACCESS_DESCRIPTION のスタックを反復処理します。 4. 各要素について、メソッドが NID_ad_OCSP であり、かつ場所のタイプが GEN_URI であるものを探します。 5. 条件に合致する最初の URI を見つけた場合、その値を apr_pstrdup で複製して返します。 6. 最後に AUTHORITY_INFO_ACCESS_free を呼び出し、スタックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_ocsp.c:68 (ソースコード内の直接的な参照として確認)
  • **呼び出し先**:
  • OpenSSL API: X509_get_ext_d2i, sk_ACCESS_DESCRIPTION_num, sk_ACCESS_DESCRIPTION_value, OBJ_obj2nid, AUTHORITY_INFO_ACCESS_free
  • APR API: apr_pstrdup

関連データ構造

  • X509: OpenSSL の証明書構造体。
  • ACCESS_DESCRIPTION: AIA 拡張内の各エントリを保持する OpenSSL 構造体。
  • apr_pool_t: APR のメモリ管理用プール。

失敗モードと境界条件

  • 証明書に AIA 拡張が含まれていない場合、NULL を返します。
  • AIA 拡張内に OCSP 用の URI が含まれていない場合、NULL を返します。
  • メモリ不足時には apr_pstrdup が例外を発生させる可能性があります。

保守担当者向けメモ

  • 本関数は OPENSSL_NO_OCSP が定義されていない環境でのみコンパイルされます。
  • 複数の OCSP URI が存在する場合、スタックの先頭から検索し、最初に見つかったものを返します。
