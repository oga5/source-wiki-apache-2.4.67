md_acme_order_to_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:101

目的

ACME Order オブジェクト（md_acme_order_t）の現在の状態を、永続化可能な JSON 形式へシリアライズします。

シグネチャまたは形状

md_json_t *md_acme_order_to_json(md_acme_order_t *order, apr_pool_t *p)

入力と出力

  • **入力**:
  • order: シリアライズ対象の md_acme_order_t 構造体へのポインタ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • 生成された JSON データを保持する md_json_t 構造体へのポインタ。

前提条件と不変条件

  • order ポインタは有効である必要があります。
  • p は JSON オブジェクトの生存期間を管理するために使用されます。
  • order->status は order_st_to_str によって文字列変換可能である必要があります。

アルゴリズムまたは振る舞い

1. md_json_create(p) を呼び出し、空の JSON オブジェクトを生成します。 2. order 構造体の各フィールドを以下のキーで JSON に格納します。

  • url: MD_KEY_URL
  • status: MD_KEY_STATUS（文字列変換後）
  • authz_urls: MD_KEY_AUTHORIZATIONS
  • challenge_setups: MD_KEY_CHALLENGE_SETUPS
  • finalize: MD_KEY_FINALIZE（存在する場合のみ）
  • certificate: MD_KEY_CERTIFICATE（存在する場合のみ）

    • 3. 構築した JSON オブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_order.c:212（p_save 関数内での呼び出しが確認されています）
  • **呼び出し先**:
  • md_json_create
  • md_json_sets / md_json_setsa
  • order_st_to_str

関連データ構造

  • md_acme_order_t: ACME Order の状態を保持するメイン構造体。
  • md_json_t: JSON データを保持する抽象構造体。

失敗モードと境界条件

  • order の各フィールドが NULL の場合、オプションのフィールド（url, finalize, certificate）は JSON に含まれません。
  • メモリ不足時には md_json_create または md_json_sets 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は md_acme_order_from_json と対になるシリアライズ処理です。構造体のフィールドが追加された場合は、本関数とデシリアライズ側の両方を更新する必要があります。
