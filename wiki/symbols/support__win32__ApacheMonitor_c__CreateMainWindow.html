CreateMainWindow

定義

  • 種別: function
  • 定義箇所: support/win32/ApacheMonitor.c:1509

目的

ApacheMonitor アプリケーションのメインウィンドウを作成し、ウィンドウクラスの登録と初期化を行う。

シグネチャまたは形状

HWND CreateMainWindow(HINSTANCE hInstance)

入力と出力

  • **入力**: hInstance (現在のアプリケーションインスタンスハンドル)
  • **出力**: 作成されたウィンドウのハンドル (HWND)。失敗した場合は NULL を返す。

前提条件と不変条件

  • g_szWindowClass および g_szTitle がグローバル変数として定義されている必要がある。
  • g_hCursorArrow が初期化済みであること。
  • IDI_APSRVMON リソース ID が定義されていること。

アルゴリズムまたは振る舞い

1. WNDCLASSEX 構造体を初期化し、ウィンドウプロシージャとして WndProc を指定する。 2. LoadImage を使用して、32x32 および 16x16 のアイコンをリソースから読み込む。 3. RegisterClassEx を呼び出し、ウィンドウクラスを登録する。 4. 登録が成功した場合、CreateWindow を呼び出して不可視（サイズ 0x0）のメインウィンドウを作成する。 5. 作成されたウィンドウハンドルを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/win32/ApacheMonitor.c:1643 (WinMain 内)
  • **呼び出し先**:
  • LoadImage (Windows API)
  • RegisterClassEx (Windows API)
  • CreateWindow (Windows API)
  • WndProc (ウィンドウプロシージャとして登録)

関連データ構造

  • WNDCLASSEX: ウィンドウクラスの属性定義に使用。
  • HWND: ウィンドウハンドル。

失敗モードと境界条件

  • RegisterClassEx が失敗した場合、ウィンドウは作成されず NULL が返される。
  • LoadImage がリソースの読み込みに失敗した場合、アイコンは設定されない可能性がある。

保守担当者向けメモ

  • 本関数は不可視ウィンドウを作成する設計となっている。
  • g_szWindowClass や g_szTitle のグローバル変数への依存が強いため、変更時は注意が必要。
  • LoadImage で使用している IDI_APSRVMON は ApacheMonitor.h で定義されている。
