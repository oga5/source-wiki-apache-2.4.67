get_dbm_realm_hash

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_dbm.c:172

目的

DBM ファイルから指定されたユーザー名とレルム（realm）に対応するパスワードハッシュを取得し、認証プロバイダーとして提供する。

シグネチャまたは形状

static authn_status get_dbm_realm_hash(request_rec *r, const char *user,
                                       const char *realm, char **rethash)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • user: 認証対象のユーザー名。
  • realm: 認証対象のレルム名。
  • **出力**:
  • rethash: 取得したパスワードハッシュ文字列を格納するポインタのポインタ。
  • 戻り値: authn_status 型（AUTH_USER_FOUND, AUTH_USER_NOT_FOUND, AUTH_GENERAL_ERROR のいずれか）。

前提条件と不変条件

  • r->per_dir_config から authn_dbm_module の設定が取得可能であること。
  • fetch_dbm_value が DBM ファイルへのアクセスを正常に完了できること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、ディレクトリごとの DBM 設定を取得する。 2. user と realm を user:realm の形式で連結し、fetch_dbm_value を呼び出して DBM ファイルから値を検索する。 3. DBM 検索でエラーが発生した場合は AUTH_GENERAL_ERROR を返す。 4. 値が見つからない場合は AUTH_USER_NOT_FOUND を返す。 5. 取得した値にコロン（:）が含まれる場合、最初のコロン以降を切り捨ててハッシュ部分のみを抽出する。 6. 抽出したハッシュを rethash に格納し、AUTHN_CACHE_STORE マクロを通じてキャッシュへの保存を試みる。 7. 成功時に AUTH_USER_FOUND を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_dbm.c:207 にて、authn_dbm_provider 構造体の一部として登録されている（直接的な呼び出し元コードの詳細はソース内に明示されていない）。
  • **呼び出し先**:
  • ap_get_module_config
  • apr_pstrcat
  • fetch_dbm_value
  • ap_strchr
  • AUTHN_CACHE_STORE

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • authn_dbm_config_rec: DBM ファイルパスやタイプを保持する設定構造体。
  • authn_status: 認証結果を示す列挙型。

失敗モードと境界条件

  • DBM ファイルの読み込み失敗やシステムエラー時は AUTH_GENERAL_ERROR を返す。
  • 指定されたユーザーとレルムの組み合わせが DBM 内に存在しない場合は AUTH_USER_NOT_FOUND を返す。
  • DBM から取得した値が空、または形式が不正な場合、後続の処理で予期せぬ動作をする可能性がある。

保守担当者向けメモ

  • 本関数は mod_authn_dbm の認証プロバイダーインターフェースの一部である。
  • AUTHN_CACHE_STORE はキャッシュモジュールが有効な場合にのみ機能するマクロであるため、キャッシュの有無に関わらず動作する設計となっている。
