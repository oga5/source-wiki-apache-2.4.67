ap_get_loadavg

定義

  • 種別: function
  • 定義箇所: server/util.c:3360

目的

システムのロードアベレージ（1分、5分、15分平均）を取得し、指定された構造体に格納します。

シグネチャまたは形状

void ap_get_loadavg(ap_loadavg_t *ld)

入力と出力

  • **入力**: ap_loadavg_t 構造体へのポインタ ld。
  • **出力**: なし（引数 ld を通じて値を書き戻す）。

前提条件と不変条件

  • ld は有効な ap_loadavg_t 構造体を指している必要があります。
  • 関数呼び出しの直後、ld の各フィールド（loadavg, loadavg5, loadavg15）は、システムがロードアベレージを取得できない場合、初期値として -1.0 に設定されます。

アルゴリズムまたは振る舞い

1. ld が指す構造体の loadavg, loadavg5, loadavg15 フィールドをすべて -1.0 で初期化します。 2. コンパイル時に HAVE_GETLOADAVG が定義されている場合、システムコール getloadavg(la, 3) を呼び出します。 3. getloadavg が成功し、取得できた値の数に応じて、la 配列の値を ld の各フィールドに float 型にキャストして代入します。 4. HAVE_GETLOADAVG が定義されていない環境では、初期値の -1.0 がそのまま保持されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_status.c:419
  • modules/metadata/mod_headers.c:220
  • **呼び出し先**:
  • getloadavg() (システムコール、HAVE_GETLOADAVG 定義時)

関連データ構造

  • ap_loadavg_t: ロードアベレージの値を保持する構造体（定義元は httpd.h）。

失敗モードと境界条件

  • getloadavg がサポートされていない環境では、常に -1.0 が返されます。
  • getloadavg がシステム上で失敗した場合、または値を取得できない場合、フィールドは初期値の -1.0 のままとなります。

保守担当者向けメモ

  • 本関数はプラットフォーム依存の getloadavg をラップする抽象化レイヤーです。
  • ap_loadavg_t 構造体の定義変更や、ロードアベレージ取得ロジックの拡張を行う際は、include/httpd.h の定義と整合性を確認してください。
