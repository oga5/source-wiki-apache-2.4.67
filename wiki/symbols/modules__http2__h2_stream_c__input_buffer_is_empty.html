input_buffer_is_empty

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_stream.c:221

目的

指定された HTTP/2 ストリームの入力バッファが空であるかどうかを判定します。

シグネチャまたは形状

static int input_buffer_is_empty(h2_stream *stream)

入力と出力

  • **入力**: h2_stream *stream - 状態を確認する対象のストリーム構造体へのポインタ。
  • **出力**: int - バッファが空（または未初期化）であれば非ゼロ（真）、データが存在すれば 0（偽）を返します。

前提条件と不変条件

  • stream ポインタは有効である必要があります。
  • stream->in_buffer が NULL であるか、または apr_bucket_brigade 型の構造体として初期化されていることを前提としています。

アルゴリズムまたは振る舞い

1. stream->in_buffer が NULL であるかを確認します。NULL の場合は空とみなします。 2. stream->in_buffer が存在する場合、APR_BRIGADE_EMPTY マクロを使用して、バケットブリゲード内にデータが含まれていないかを判定します。 3. 上記のいずれかの条件を満たす場合に真を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_stream.c:231 にて、ストリーム処理のクリーンアップ判定条件として使用されています。
  • **呼び出し先**: APR_BRIGADE_EMPTY マクロ（APRライブラリ）。

関連データ構造

  • h2_stream: ストリームのコンテキストを保持する構造体。
  • apr_bucket_brigade: Apache Portable Runtime (APR) のバケット管理構造体。

失敗モードと境界条件

  • stream が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • in_buffer が初期化されていない（かつ NULL でない）場合、APR_BRIGADE_EMPTY の挙動は未定義です。

保守担当者向けメモ

  • 本関数は static であり、h2_stream.c 内でのみ使用されます。
  • APR_BRIGADE_EMPTY は APR のマクロであり、ブリゲードの先頭と末尾のポインタを比較して空かどうかを判定します。
