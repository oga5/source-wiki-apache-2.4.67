md_jws_sign

定義

  • 種別: function
  • 定義箇所: modules/md/md_jws.c:42

目的

RSA秘密鍵を使用して、ACMEプロトコル等で利用されるJWS (JSON Web Signature) メッセージを構築する。

シグネチャまたは形状

apr_status_t md_jws_sign(md_json_t **pmsg, apr_pool_t *p,
                         md_data_t *payload, md_json_t *prot_fields,
                         struct md_pkey_t *pkey, const char *key_id)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • payload: 署名対象のデータ。
  • prot_fields: 保護ヘッダーに追加するJSONフィールド。
  • pkey: 署名に使用するRSA秘密鍵。
  • key_id: (オプション) 鍵識別子。指定がない場合は pkey からJWKを生成してヘッダーに埋め込む。
  • **出力**:
  • pmsg: 成功時に構築されたJWSメッセージ（JSONオブジェクト）が格納される。失敗時は NULL。
  • 戻り値: APR_SUCCESS またはエラーコード。

前提条件と不変条件

  • prot_fields には alg フィールドが強制的に RS256 として設定される。
  • 署名対象のヘッダーとペイロードはBase64URLエンコードされる。
  • 署名処理は md_crypt_sign64 を介して行われる。

アルゴリズムまたは振る舞い

1. prot_fields を複製し、alg を RS256 に設定する。 2. key_id があればヘッダーに kid を追加し、なければ md_jws_get_jwk で公開鍵からJWKを生成してヘッダーに含める。 3. 保護ヘッダーをJSON文字列化し、Base64URLエンコードする。 4. payload をBase64URLエンコードする。 5. protected.payload 形式の文字列を作成し、md_crypt_sign64 で署名を生成する。 6. protected、payload、signature を含むJSONオブジェクトを構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c:249
  • **呼び出し先**:
  • md_json_create, md_json_clone, md_json_sets, md_json_setj, md_json_writep
  • md_jws_get_jwk (条件付き)
  • md_data_init, md_util_base64url_encode
  • apr_psprintf
  • md_crypt_sign64
  • md_log_perror

関連データ構造

  • md_json_t: JSONオブジェクト構造体。
  • md_data_t: データバッファ。
  • md_pkey_t: 秘密鍵構造体。

失敗モードと境界条件

  • md_jws_get_jwk が失敗した場合、エラーを返して終了する。
  • 保護ヘッダーのシリアライズに失敗した場合、APR_EINVAL を返す。
  • 署名生成 (md_crypt_sign64) が失敗した場合、エラーを返して終了する。
  • 失敗時は *pmsg が NULL に設定される。

保守担当者向けメモ

  • 本関数は RS256 アルゴリズムに固定されている。
  • 戻り値が APR_SUCCESS でない場合、呼び出し側で pmsg が NULL であることを確認する必要がある。
  • メモリ管理はすべて p (apr_pool_t) に依存している。
