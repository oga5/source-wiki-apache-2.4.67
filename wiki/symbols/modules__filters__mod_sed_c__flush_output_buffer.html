flush_output_buffer

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_sed.c:134

目的

sed_filter_ctxt 内の出力バッファに蓄積されたデータを、新しいバケットとしてブリゲードへ書き出し、バッファをリセットする。

シグネチャまたは形状

static apr_status_t flush_output_buffer(sed_filter_ctxt *ctx)

入力と出力

  • **入力**: sed_filter_ctxt *ctx（フィルタの実行コンテキスト）
  • **出力**: apr_status_t（処理結果。成功時は APR_SUCCESS）

前提条件と不変条件

  • ctx->outbuf が初期化されていること。
  • ctx->tpool がバケット生成のためのメモリプールとして有効であること。
  • 処理終了後、ctx->curoutbuf は ctx->outbuf の先頭位置にリセットされる。

アルゴリズムまたは振る舞い

1. 現在のバッファ使用量（size）を算出する。 2. バッファが空（outbuf が NULL または size <= 0）であれば、何もせず APR_SUCCESS を返す。 3. ctx->tpool を使用して、現在のバッファ内容をメモリ上に複製する。 4. append_bucket を呼び出し、複製したデータをブリゲードに追加する。 5. ctx->curoutbuf を ctx->outbuf の先頭に再設定し、バッファを空の状態に戻す。 6. append_bucket の戻り値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_sed.c 内の各所（行 334, 343, 369, 455, 471 付近）。
  • **呼び出し先**:
  • apr_pmemdup (APR API)
  • append_bucket (同ファイル内の関数)

関連データ構造

  • sed_filter_ctxt: フィルタの状態、出力バッファ、および一時プールを保持する構造体。

失敗モードと境界条件

  • ctx->outbuf が NULL の場合、またはバッファ内のデータサイズが 0 以下の場合、処理はスキップされ成功として扱われる。
  • append_bucket がエラーを返した場合、そのステータスがそのまま呼び出し元へ伝播される。

保守担当者向けメモ

  • 本関数は sed_write_output 等から呼び出され、蓄積された変換結果をストリームへフラッシュする役割を担う。
  • メモリ管理は ctx->tpool に依存しているため、呼び出し元で適切にプールがクリアされていることを確認すること。
