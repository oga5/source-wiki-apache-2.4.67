ap_lua_rstack_dump

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:50

目的

Lua スタックの内容をデバッグ目的で Apache のエラーログ（APLOG_INFO レベル）に出力する。

シグネチャまたは形状

void ap_lua_rstack_dump(lua_State *L, request_rec *r, const char *msg)

入力と出力

  • L: 検査対象の Lua VM 状態。
  • r: ログ出力に使用する Apache リクエスト構造体。
  • msg: ログ出力時に付与する識別用メッセージ。
  • 出力: 戻り値なし。副作用として Apache のエラーログへスタックの内容を書き出す。

前提条件と不変条件

  • L および r は有効なポインタであること。
  • APLOG_MARK を使用するため、Apache のログシステムが初期化されている必要がある。

アルゴリズムまたは振る舞い

1. lua_gettop(L) で現在のスタックトップを取得する。 2. 指定された msg を含むヘッダーログを出力する。 3. スタックの 1 番目からトップまでをループし、lua_type(L, i) で各要素の型を判定する。 4. 型に応じて以下の値をログ出力する:

  • LUA_TSTRING: 文字列の内容。
  • LUA_TBOOLEAN: true または false。
  • LUA_TNUMBER: 数値。
  • LUA_TUSERDATA, LUA_TLIGHTUSERDATA, LUA_TTABLE, LUA_TFUNCTION: 型名またはプレースホルダー。
  • LUA_TNIL, LUA_TNONE: それぞれの定数名。
  • その他: lua_typename を用いた型名。

呼び出し元と呼び出し先

  • 呼び出し元: modules/lua/mod_lua.h に宣言が存在するが、提供されたソース内での具体的な呼び出し箇所は特定されていない。
  • 呼び出し先: lua_gettop, lua_type, lua_tostring, lua_toboolean, lua_tonumber, lua_typename (Lua API)、および ap_log_rerror (Apache API)。

関連データ構造

  • lua_State: Lua VM の状態。
  • request_rec: Apache のリクエスト構造体。

失敗モードと境界条件

  • スタックが空の場合、ループは実行されずヘッダーログのみが出力される。
  • lua_State が無効な場合、Lua API 呼び出しにより未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はデバッグ用であり、本番環境で頻繁に呼び出すとログが肥大化するため注意が必要である。
  • 新しい Lua 型が追加された場合、switch 文の default 節で捕捉される。
