socache_rd_svr_cfg

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_redis.c:30

目的

Redis バックエンドを使用する共有オブジェクトキャッシュ（socache）において、サーバーごとの接続設定パラメータを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_uint32_t ttl;
    apr_uint32_t rwto;
} socache_rd_svr_cfg;

入力と出力

本構造体は設定値のコンテナであり、直接的な入力・出力関数はありません。各フィールドは以下の通りです。

  • ttl: キャッシュの生存期間（Time To Live）。
  • rwto: 読み書きのタイムアウト値。

前提条件と不変条件

  • mod_socache_redis モジュールのサーバー設定の一部として、Apache の設定フレームワークを通じて管理されます。

アルゴリズムまたは振る舞い

本構造体は、RedisConnPoolTTL および RedisTimeout ディレクティブによって設定された値を保持します。これらの値は、apr_redis を用いた接続プールの初期化や、キャッシュ操作時のタイムアウト制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_socache_redis.c 内のサーバー設定生成関数（create_server_config）にてメモリ確保が行われます。
  • ap_get_module_config を介して、設定値の読み取りが行われます（行 105, 416, 439）。
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ap_socache_instance_t: Redis インスタンスのコンテキストを保持する構造体であり、本設定構造体と組み合わせて使用されます。

失敗モードと境界条件

  • 構造体内の各フィールドは apr_uint32_t 型であるため、設定値がこの範囲を超える場合はオーバーフローの可能性があります。

保守担当者向けメモ

  • 本構造体は mod_socache_redis 専用の設定保持用です。新しい設定項目を追加する場合は、この構造体を拡張し、対応するディレクティブのハンドラを更新してください。
