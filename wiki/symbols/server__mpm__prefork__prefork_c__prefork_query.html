prefork_query

定義

  • 種別: function
  • 定義箇所: server/mpm/prefork/prefork.c:295

目的

Prefork MPM の動作設定や現在の状態に関する情報を、クエリコードに基づいて取得する。

シグネチャまたは形状

int prefork_query(int query_code, int *result, apr_status_t *rv)

入力と出力

  • query_code: 取得したい情報の種類を示す定数（AP_MPMQ_*）。
  • result: 取得した値を格納するポインタ。
  • rv: 処理結果のステータス（APR_SUCCESS または APR_ENOTIMPL）。
  • 戻り値: 常に OK を返す。

前提条件と不変条件

  • rv は呼び出し元で初期化されている必要がある。
  • retained グローバル変数が初期化されていること（AP_MPMQ_MPM_STATE 等のクエリ時）。

アルゴリズムまたは振る舞い

1. rv を APR_SUCCESS に設定する。 2. query_code に基づき switch 文で分岐する。 3. 対応する MPM の設定値（ap_daemons_limit 等）や、retained 構造体内の状態（mpm_state, my_generation）を result に代入する。 4. 未定義の query_code が渡された場合は rv を APR_ENOTIMPL に設定する。 5. 常に OK を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/prefork/prefork.c:1473 にて ap_hook_mpm_query に登録されている。
  • 呼び出し先: 直接的な関数呼び出しは行わず、グローバル変数および retained 構造体を参照する。

関連データ構造

  • prefork_retained_data: MPM の状態や世代管理情報を保持する構造体。
  • ap_daemons_limit 等のグローバル設定変数。

失敗モードと境界条件

  • 未サポートの query_code が指定された場合、*rv に APR_ENOTIMPL が設定される。

保守担当者向けメモ

  • 新しいクエリコードを追加する場合は、switch 文にケースを追加すること。
  • retained 構造体へのアクセスは、マルチプロセス環境下での整合性に注意すること。
