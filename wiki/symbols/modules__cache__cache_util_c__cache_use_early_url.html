cache_use_early_url

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_util.c:130

目的

リクエストに対して「早期 URL（early URL）」を使用すべきかどうかを判定します。ここでいう早期 URL とは、リクエストの正規化や処理の初期段階で利用される、r->parsed_uri に基づくパスやクエリを指します。

シグネチャまたは形状

int cache_use_early_url(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト構造体)
  • **出力**: int 型。早期 URL を使用すべき場合は 1、そうでない場合は 0 を返します。

前提条件と不変条件

  • r は有効な request_rec 構造体である必要があります。
  • r->server->module_config から cache_module の設定を取得できる必要があります。

アルゴリズムまたは振る舞い

1. リクエストがプロキシリクエスト（PROXYREQ_PROXY）である場合、直ちに 1 を返します。 2. キャッシュモジュールの設定（cache_server_conf）を取得します。 3. 設定内の quick フラグが有効である場合、1 を返します。 4. 上記のいずれにも該当しない場合、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/cache_storage.c:674
  • modules/cache/cache_util.c:197
  • modules/cache/mod_cache.c:931
  • **呼び出し先**:
  • ap_get_module_config (Apache HTTP Server API)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • cache_server_conf: mod_cache のサーバー設定を保持する構造体。

失敗モードと境界条件

  • r->server が NULL の場合、ap_get_module_config の呼び出しでクラッシュする可能性があります。
  • 設定が適切に初期化されていない場合、conf->quick の参照で予期せぬ動作をする可能性があります。

保守担当者向けメモ

  • この関数は、キャッシュのキー生成やリクエストのパス解決において、r->uri（生の URI）を使うか r->parsed_uri（解析済み URI）を使うかの分岐点となります。
  • quick ハンドラの設定と密接に関連しているため、キャッシュの動作モードを変更する際は本関数のロジックを確認してください。
