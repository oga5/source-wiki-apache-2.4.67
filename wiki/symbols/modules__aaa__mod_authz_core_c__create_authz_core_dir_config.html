create_authz_core_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_core.c:102

目的

ディレクトリごとの認可設定構造体（authz_core_dir_conf）を生成し、初期化する。

シグネチャまたは形状

static void *create_authz_core_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体の割り当てに使用するメモリプール。
  • char *dummy: Apache 設定フレームワークの規約による未使用の引数。
  • **出力**:
  • void *: 初期化された authz_core_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全メンバはゼロ初期化される。
  • 呼び出しごとに新しい authz_core_dir_conf がメモリプールから割り当てられる。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して authz_core_dir_conf サイズのメモリを確保する。 2. 認可ロジック演算子 op を AUTHZ_LOGIC_UNSET に設定する。 3. 認可失敗時の挙動 authz_forbidden_on_fail を UNSET に設定する。 4. グローバル変数 authz_core_first_dir_conf を参照し、新しく作成した設定をリストの先頭に繋ぐ（リンクリストの構築）。 5. 構築した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_core.c:1166（ディレクトリ設定作成用ハンドラとして登録）。
  • **呼び出し先**: apr_pcalloc（APRライブラリ）。

関連データ構造

  • authz_core_dir_conf: 本関数が生成するディレクトリごとの認可設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボートする可能性がある（APRの標準的な挙動）。

保守担当者向けメモ

  • 本関数はリンクリストの先頭を更新する副作用を持つ。設定の読み込み順序やライフサイクルに依存するため、グローバル変数 authz_core_first_dir_conf の操作には注意すること。
