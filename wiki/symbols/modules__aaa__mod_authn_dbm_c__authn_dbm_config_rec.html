authn_dbm_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authn_dbm.c:52

目的

mod_authn_dbm モジュールにおいて、ディレクトリごとの認証設定（DBMファイルパスおよびDBM形式）を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    const char *pwfile;
    const char *dbmtype;
} authn_dbm_config_rec;

入力と出力

  • **入力**: AuthDBMUserFile および AuthDBMType ディレクティブによって設定値が格納されます。
  • **出力**: 認証処理時に ap_get_module_config を通じて取得され、DBM検索時のパラメータとして利用されます。

前提条件と不変条件

  • pwfile は DBM ファイルのパスを指す文字列である必要があります。
  • dbmtype はシステムでサポートされている DBM ドライバー名（SDBM, GDBM, DB 等）である必要があります。

アルゴリズムまたは振る舞い

本構造体は、Apache の設定コンテキスト（per_dir_config）に紐付けられ、認証フェーズにおいて fetch_dbm_value 等の関数が DBM ファイルへアクセスする際の参照先として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_authn_dbm_dir_config: 構造体のメモリ確保時に使用。
  • check_dbm_pw / get_dbm_realm_hash: 認証実行時に設定値を取得するために使用。
  • Apache 設定コマンド（APR_OFFSETOF を使用して設定値を構造体にマッピング）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • mod_authn_dbm.c 内の各認証関数（check_dbm_pw 等）が、この構造体を通じて設定を参照します。

失敗モードと境界条件

  • 設定が未定義の場合、pwfile や dbmtype が NULL になる可能性があります。これらは認証処理関数内で適切にチェックされる必要があります。

保守担当者向けメモ

  • 本構造体は mod_authn_dbm の設定管理の基盤です。新しい設定項目を追加する場合は、この構造体と command_rec テーブルの両方を更新してください。
