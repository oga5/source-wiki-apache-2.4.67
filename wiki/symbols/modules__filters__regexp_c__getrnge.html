getrnge

定義

  • 種別: function
  • 定義箇所: modules/filters/regexp.c:593

目的

正規表現における {n,m} 形式の範囲指定を解析し、その結果を step_vars_storage 構造体に格納する。

シグネチャまたは形状

static void getrnge(char *str, step_vars_storage *vars)

入力と出力

  • str: 範囲指定が格納されているバイトコード内の文字列ポインタ。
  • vars: 解析結果を書き込む step_vars_storage 構造体へのポインタ。
  • vars->low: 範囲の最小値。
  • vars->size: 範囲の最大値と最小値の差（または特殊値）。

前提条件と不変条件

  • str は {n,m} 形式の範囲指定データを含む有効なメモリ領域を指している必要がある。
  • vars は呼び出し元によって確保された有効なメモリ領域であること。

アルゴリズムまたは振る舞い

1. str が指す最初のバイトを vars->low に代入する。 2. str をインクリメントし、次のバイトを評価する。 3. 次のバイトが 255 である場合、vars->size に 20000 を設定する（無限大の表現）。 4. それ以外の場合、次のバイトから vars->low を減算した値を vars->size に設定する。

  • 呼び出し先:
  • 外部関数呼び出しは確認されていない。

関連データ構造

  • step_vars_storage: regexp.h で定義される、マッチング状態および範囲指定を保持する構造体。

失敗モードと境界条件

  • str が指すメモリ位置が不正な場合、未定義動作を引き起こす可能性がある。
  • *str の値が 255 の場合に 20000 を代入する特殊な境界条件を持つ。

保守担当者向けメモ

  • 本関数は AT&T sed 由来の古い実装であり、バイトコードの直接的な操作に依存している。
  • vars->size に設定される 20000 は、正規表現エンジン内での繰り返し回数の上限として機能する。
