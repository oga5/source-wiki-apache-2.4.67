count_headers

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_forensic.c:159

目的

リクエストヘッダーをシリアライズする際に必要なバッファサイズを計算するため、各ヘッダーのキーと値の長さを累積加算する。

シグネチャまたは形状

static int count_headers(void *h_, const char *key, const char *value)

入力と出力

  • **入力**:
  • h_: hlog 構造体へのポインタ（型消去された void *）。
  • key: 現在処理中のヘッダー名。
  • value: 現在処理中のヘッダー値。
  • **出力**:
  • 常に 1 を返す（apr_table_do のイテレーションを継続させるため）。

前提条件と不変条件

  • h_ は有効な hlog 構造体へのポインタであること。
  • h->count は、ヘッダーの累積サイズを保持するカウンタとして初期化されていること。

アルゴリズムまたは振る舞い

1. hlog 構造体からカウンタ h->count を取得する。 2. count_string(key) と count_string(value) を呼び出し、それぞれの長さを取得する。 3. キーと値の長さに、区切り文字および終端文字分として 2 を加算し、h->count に累積する。 4. 1 を返し、次のヘッダー要素の処理へ進む。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_log_forensic.c:206 において apr_table_do のコールバックとして呼び出される。
  • **呼び出し先**: count_string（ヘッダーの各要素の長さを算出するために使用）。

関連データ構造

  • hlog: ログ出力用のバッファサイズ計算結果を保持する構造体。

失敗モードと境界条件

  • key または value が NULL の場合、count_string の実装に依存する（ソースコード上では count_string が適切に処理することを前提としている）。

保守担当者向けメモ

  • 本関数は apr_table_do を通じてリクエストヘッダーの全要素に対して反復実行される。
  • h->count への加算はアトミックではないため、単一リクエストのコンテキスト内でのみ使用すること。
