calc_space_left

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:181

目的

h2_bucket_beam において、現在バッファされているデータ量を考慮し、設定された上限に対して追加可能な空き容量を算出する。

シグネチャまたは形状

static apr_size_t calc_space_left(h2_bucket_beam *beam)

入力と出力

  • **入力**: beam (h2_bucket_beam *) - 対象となるバケットビームのポインタ。
  • **出力**: apr_size_t - 追加可能なバイト数。上限が設定されていない場合は APR_SIZE_MAX を返す。

前提条件と不変条件

  • beam は有効なポインタであること。
  • beam->max_buf_size は、ビームのバッファ上限（バイト単位）を保持している。

アルゴリズムまたは振る舞い

1. beam->max_buf_size が 0 より大きい場合：

  • calc_buffered(beam) を呼び出し、現在バッファされているデータ量を取得する。
  • 上限から現在量を差し引いた値を返す（現在量が上限を超えている場合は 0 を返す）。

    • 2. beam->max_buf_size が 0 以下の場合：

  • バッファ制限がないものとみなし、APR_SIZE_MAX を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_bucket_beam.c:228 (送信待機ループ内)
  • modules/http2/h2_bucket_beam.c:550
  • **呼び出し先**:
  • calc_buffered (同一ファイル内)

関連データ構造

  • h2_bucket_beam: バケット転送と同期を管理する構造体。

失敗モードと境界条件

  • max_buf_size が 0 の場合、制限なしとして APR_SIZE_MAX を返す。
  • 現在のバッファ量が max_buf_size を超えている場合、負の値を返さず 0 を返す安全な設計となっている。

保守担当者向けメモ

  • 本関数は max_buf_size の設定値に依存する。動的なバッファサイズ変更が行われる環境下での整合性に注意すること。
