set_compression_quality

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_brotli.c:94

目的

Apache HTTP Server の設定ファイル（httpd.conf）から指定された Brotli 圧縮品質（0〜11）を解析し、サーバー設定構造体に適用する。

シグネチャまたは形状

static const char *set_compression_quality(cmd_parms *cmd, void *dummy, const char *arg);

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • dummy: 使用されない引数。
  • arg: 設定ファイルから渡される品質値の文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • cmd->server->module_config から brotli_module の設定構造体を取得可能であること。
  • arg は atoi で整数変換可能な文字列であること。
  • 設定される品質値 val は、Brotli ライブラリの仕様に基づき 0 から 11 の範囲内であること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定構造体 brotli_server_config_t を取得する。 2. atoi(arg) を用いて入力文字列を整数に変換する。 3. 変換後の値が 0 未満または 11 を超える場合、エラーメッセージを返して終了する。 4. 正常な範囲であれば、設定構造体の quality メンバに値を代入し、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_brotli.c:581 において、AP_INIT_TAKE1 マクロを通じて BrotliCompressionQuality ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: ap_get_module_config (Apache API), atoi (標準ライブラリ)。

関連データ構造

  • brotli_server_config_t: サーバー単位の Brotli 設定を保持する構造体。本関数はこの構造体の quality フィールドを更新する。

失敗モードと境界条件

  • **範囲外の値**: arg が 0〜11 の範囲外である場合、設定の適用を拒否し、エラーメッセージを返す。
  • **不正な入力**: atoi が非数値文字列を処理した場合の挙動は atoi の仕様に依存する（通常は 0 が返される）。

保守担当者向けメモ

  • 本関数は BrotliCompressionQuality ディレクティブ専用のパーサーである。
  • 設定値の検証ロジックは Brotli エンコーダの許容範囲（0-11）に直接依存しているため、将来的にライブラリの仕様が変更された場合はこの定数を確認すること。
