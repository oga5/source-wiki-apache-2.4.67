trie_node_link

定義

  • 種別: function
  • 定義箇所: server/util_filter.c:75

目的

フィルタ名を管理するトライ木において、親ノードに子ノードをリンクし、子ノードのリストを文字コード順に維持する。

シグネチャまたは形状

static void trie_node_link(apr_pool_t *p, filter_trie_node *parent,
                           filter_trie_node *child, int c)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t。
  • parent: 子ノードを追加する対象の filter_trie_node。
  • child: 追加する filter_trie_node。
  • c: この子ノードに対応する文字コード。
  • 出力: なし（parent 構造体内部の状態を直接更新する）。

前提条件と不変条件

  • parent->children は parent->size 分の領域を確保済みであること。
  • parent->children リストは常に文字コード c の昇順でソートされていること。
  • parent->nchildren は現在の有効な子ノード数と一致していること。

アルゴリズムまたは振る舞い

1. **容量確認**: parent->nchildren が parent->size に達している場合、parent->size を2倍に拡張し、apr_palloc で新しいメモリ領域を確保して既存のデータをコピーする。 2. **重複チェック**: 指定された文字 c が既に存在する場合は何もせず終了する。 3. **挿入位置の特定**: c を挿入すべき位置 i を、昇順を維持するように探索する。 4. **シフトと挿入**: i 以降の要素を後ろにずらし、空いた parent->children[i] に c と child を格納する。 5. **更新**: parent->nchildren をインクリメントする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/util_filter.c:128
  • server/util_filter.c:225
  • 呼び出し先:
  • apr_palloc (APRライブラリ)
  • memcpy (標準ライブラリ)

関連データ構造

  • filter_trie_node: トライ木のノード構造体。
  • filter_trie_child_ptr: 子ノードへのポインタと対応する文字を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: apr_palloc が失敗した場合、APR の規約に従いプロセスが終了する可能性がある。
  • 既存の文字: 同一の文字 c を持つ子ノードが既に存在する場合、挿入は行われず、既存のリンクは上書きされない。

保守担当者向けメモ

  • 本関数はトライ木の動的な拡張を伴うため、頻繁な呼び出しはメモリ断片化を招く可能性がある。
  • parent->size の倍加戦略は、ノード数が増加する際の再割り当てコストを抑えるための標準的な手法である。
