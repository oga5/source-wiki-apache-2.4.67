set_cookie_form_username

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:213

目的

Apache 設定ファイル内の AuthFormUsername ディレクティブを処理し、フォーム認証で使用するユーザー名フィールド名を指定された設定構造体に格納します。

シグネチャまたは形状

static const char *set_cookie_form_username(cmd_parms *cmd, void *config, const char *username)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • config: 現在のディレクトリ設定構造体（auth_form_config_rec）。
  • username: 設定ファイルから読み取られたユーザー名フィールド名。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は check_string が返すエラーメッセージ文字列を返します。

前提条件と不変条件

  • config 引数は auth_form_config_rec 型としてキャスト可能である必要があります。
  • username は設定ファイルから渡される有効な文字列である必要があります。

アルゴリズムまたは振る舞い

1. config を auth_form_config_rec 型にキャストします。 2. conf->username に引数の username を代入します。 3. conf->username_set フラグを 1 に設定し、値が設定済みであることを示します。 4. check_string(cmd, username) を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c:356 において、AP_INIT_TAKE1 マクロを通じて AuthFormUsername ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: check_string 関数を呼び出します。

関連データ構造

  • auth_form_config_rec: フォーム認証の設定を保持する構造体。username フィールドおよび username_set フラグを保持します。

失敗モードと境界条件

  • check_string がエラーを返した場合、そのエラーがそのまま呼び出し元に伝播します。

保守担当者向けメモ

  • 本関数は AuthFormUsername ディレクティブ専用のセッターです。設定の変更や検証ロジックを追加する場合は、check_string の仕様と併せて確認してください。
