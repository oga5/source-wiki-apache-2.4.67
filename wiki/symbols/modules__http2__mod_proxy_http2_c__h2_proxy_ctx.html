h2_proxy_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/mod_proxy_http2.c:51

目的

Apache HTTP Server のプロキシモジュールにおいて、バックエンドへの HTTP/2 通信を行う際のリクエスト処理状態を保持・管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct h2_proxy_ctx {
    const char *id;
    conn_rec *cfront;
    apr_pool_t *pool;
    server_rec *server;
    const char *proxy_func;
    char server_portstr[32];
    proxy_conn_rec *p_conn;
    proxy_worker *worker;
    proxy_server_conf *conf;
    
    apr_size_t req_buffer_size;
    int capacity;
    
    unsigned is_ssl : 1;
    
    request_rec *r;
    apr_status_t r_status;
    int r_done;
    int r_may_retry;
    int has_reusable_session;
} h2_proxy_ctx;

入力と出力

  • **入力**: リクエスト処理開始時に生成され、リクエストのプール (r->pool) に割り当てられます。
  • **出力**: リクエストの完了状態 (r_done)、処理結果 (r_status)、再試行可否 (r_may_retry)、およびセッションの再利用性 (has_reusable_session) を保持します。

前提条件と不変条件

  • リクエストのライフサイクルと同期しており、リクエスト終了時に自動的に解放されます。
  • has_reusable_session が真の場合、セッションはオープンなストリームを持たないクリーンな状態であることを示します。

アルゴリズムまたは振る舞い

  • バックエンドとの接続情報、ワーカースレッド、およびプロキシ設定を保持し、ctx_run によるセッション実行の基盤となります。
  • request_done や session_req_done などのコールバックを通じて、リクエストの完了判定や再試行ロジックの制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_proxy_http2.c 内の proxy_http2_handler や ctx_run 等でインスタンス化・参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、h2_proxy_session 関連の関数に引数として渡されます。

関連データ構造

  • proxy_conn_rec: バックエンド接続情報。
  • request_rec: 処理対象のクライアントリクエスト。
  • proxy_worker: プロキシワーカースレッド。
  • h2_proxy_session: HTTP/2 セッション管理ロジック（本構造体は session->user_data として保持されるケースがあります）。

失敗モードと境界条件

  • r_may_retry フラグにより、最大1回の再試行が許可されています。
  • 接続エラーやセッション切断が発生した場合、r_status にエラーコードが格納され、r_done が更新されます。

保守担当者向けメモ

  • 本構造体はリクエストのプールに依存しているため、メモリリークの懸念は少ない設計です。
  • nghttp2 の詳細なフレーム処理は h2_proxy_session.c に委譲されているため、本構造体の変更時はセッション管理側との整合性に注意してください。
