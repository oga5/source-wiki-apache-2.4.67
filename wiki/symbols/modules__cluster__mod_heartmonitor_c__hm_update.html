hm_update

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartmonitor.c:169

目的

共有メモリ上のサーバー状態（hm_slot_server_t）を、最新のハートビート統計情報で更新するコールバック関数です。

シグネチャまたは形状

static apr_status_t hm_update(void* mem, void *data, apr_pool_t *p)

入力と出力

  • **入力**:
  • mem: 共有メモリ内の対象サーバーエントリ（hm_slot_server_t* にキャスト）。
  • data: 更新元となるコンテキスト（hm_slot_server_ctx_t* にキャスト）。
  • p: メモリプール（本関数内では未使用）。
  • **出力**:
  • 常に APR_SUCCESS を返します。

前提条件と不変条件

  • mem は hm_slot_server_t 型としてキャスト可能であること。
  • data は hm_slot_server_ctx_t 型としてキャスト可能であり、そのメンバ s が有効な hm_server_t を指していること。

アルゴリズムまたは振る舞い

1. 共有メモリ上のエントリ（old）と更新対象のサーバー情報（new）の IP アドレスを比較します。 2. IP アドレスが一致した場合：

  • s->found フラグを 1 に設定します。
  • old の busy、ready、seen フィールドを new の値で上書き更新します。

    • 3. IP アドレスが一致しない場合は何も行わず終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cluster/mod_heartmonitor.c:201 において、storage->doall メソッドの引数として渡され、共有メモリ内の全エントリに対して反復実行されます。
  • **呼び出し先**: なし。

関連データ構造

  • hm_slot_server_t: 共有メモリ上に配置されるサーバー状態構造体。
  • hm_slot_server_ctx_t: 更新処理中に使用される一時的なコンテキスト構造体。
  • hm_server_t: ハートビートにより収集された最新のサーバー統計情報。

失敗モードと境界条件

  • IP アドレスが一致しない場合、共有メモリの更新はスキップされます。
  • slotmem の doall 反復処理内で呼び出されるため、本関数自体はエラーを返さず、成功として処理を継続します。

保守担当者向けメモ

  • 本関数は slotmem の反復処理用コールバックとして設計されています。
  • s->found フラグは、共有メモリ内のエントリが更新されたかどうかを呼び出し元で追跡するために使用されています。
