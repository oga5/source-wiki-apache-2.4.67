merge_session_dbd_dir_config

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_dbd.c:508

目的

Apache HTTP Server の設定マージプロセスにおいて、親ディレクトリの設定（basev）と現在のディレクトリの設定（addv）を統合し、新しい session_dbd_dir_conf 構造体を生成します。

シグネチャまたは形状

static void *merge_session_dbd_dir_config(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定構造体（session_dbd_dir_conf）。
  • addv: 現在のディレクトリの設定構造体（session_dbd_dir_conf）。
  • **出力**:
  • 統合された新しい session_dbd_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は session_dbd_dir_conf 型としてキャスト可能である必要があります。
  • 戻り値の構造体は p プールから割り当てられます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい session_dbd_dir_conf 構造体をメモリ上に確保します。 2. 各設定項目について、addv 側で設定が行われている（*_set フラグが有効、または値が存在する）場合は addv の値を、そうでない場合は basev の値を採用します。 3. 具体的には以下の項目をマージします：

  • クッキー名および属性 (name, name_attrs, name_set)
  • セカンダリクッキー名および属性 (name2, name2_attrs, name2_set)
  • ユーザー紐付けフラグ (peruser, peruser_set)
  • 削除フラグ (remove, remove_set)
  • SQL クエリラベル (selectlabel, updatelabel, insertlabel, deletelabel)

    • 4. 最終的に統合された構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_dbd.c:644 にて、モジュールの設定マージ用関数として登録されています。
  • **呼び出し先**: 直接的な関数呼び出しは行わず、メモリ確保のために apr_pcalloc を使用します。

関連データ構造

  • session_dbd_dir_conf: ディレクトリ単位のセッション設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが終了する可能性があります。
  • addv または basev が NULL の場合、未定義の動作を引き起こす可能性があります（Apache の設定マージの慣習上、通常は有効なポインタが渡されます）。

保守担当者向けメモ

  • 本関数は設定の継承を制御します。新しい設定項目を session_dbd_dir_conf に追加した場合は、必ず本関数内でもマージ処理を記述してください。
