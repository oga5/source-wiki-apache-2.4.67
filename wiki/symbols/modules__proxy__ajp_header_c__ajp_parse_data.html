ajp_parse_data

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_header.c:804

目的

AJP メッセージからボディチャンクの長さを検証し、データ本体へのポインタを抽出します。

シグネチャまたは形状

apr_status_t ajp_parse_data(request_rec *r, ajp_msg_t *msg, apr_uint16_t *len, char **ptr)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • msg: 解析対象の ajp_msg_t メッセージバッファ。
  • **出力**:
  • len: 抽出されたボディチャンクの長さ（apr_uint16_t）。
  • ptr: メッセージバッファ内のデータ開始位置を指すポインタ。
  • 戻り値: 成功時は APR_SUCCESS、失敗時はエラーコード（AJP_EBAD_HEADER 等）。

前提条件と不変条件

  • msg は有効な AJP メッセージ構造体であること。
  • メッセージの先頭バイトが CMD_AJP13_SEND_BODY_CHUNK (0x03) であること。
  • メッセージ全体の長さ（msg->len）が、ヘッダーおよび制御バイトの合計サイズ（AJP_HEADER_LEN + AJP_HEADER_SZ_LEN + 1 + 1）以上であること。

アルゴリズムまたは振る舞い

1. ajp_msg_get_uint8 を呼び出し、メッセージタイプを確認する。タイプが CMD_AJP13_SEND_BODY_CHUNK でない場合はエラーを返す。 2. ajp_msg_get_uint16 を呼び出し、チャンク長 len を取得する。 3. メッセージ全体の長さからヘッダー等の固定長を差し引き、期待されるチャンク長を計算する。 4. 取得した len が計算値と一致するか検証し、不一致の場合はエラーを返す。 5. メッセージバッファの現在位置（msg->pos）を ptr に格納し、成功を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ajp.c:509
  • **呼び出し先**:
  • ajp_msg_get_uint8
  • ajp_msg_get_uint16
  • ap_log_rerror (エラーログ出力用)
  • ajp_type_str (エラーメッセージ生成用)

関連データ構造

  • ajp_msg_t: AJP 通信用のメッセージバッファ構造体。
  • request_rec: Apache のリクエスト情報構造体。

失敗モードと境界条件

  • **メッセージタイプ不一致**: 期待する CMD_AJP13_SEND_BODY_CHUNK 以外の場合、AJP_EBAD_HEADER を返す。
  • **サイズ不足**: メッセージ長が最小ヘッダーサイズを下回る場合、AJP_EBAD_HEADER を返す。
  • **長さ不一致**: 取得したチャンク長が計算上の期待値と異なる場合、AJP_EBAD_HEADER を返す。
  • **読み取りエラー**: ajp_msg_get_* 関数が失敗した場合、その戻り値をそのまま返す。

保守担当者向けメモ

  • 本関数は AJP プロトコルの SEND_BODY_CHUNK パケットの構造に強く依存しています。パケットフォーマットの変更時には、固定長計算（AJP_HEADER_LEN 等の加算値）の整合性を確認してください。
  • msg->pos を使用してポインタを算出しているため、本関数呼び出し前に msg の読み取り位置が適切に管理されている必要があります。
