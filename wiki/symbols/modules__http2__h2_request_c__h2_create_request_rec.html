h2_create_request_rec

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:374

目的

HTTP/2 のリクエスト表現 (h2_request) を Apache HTTP Server のネイティブなリクエスト構造体 (request_rec) に変換し、Apache のリクエスト処理パイプラインへ投入する。

シグネチャまたは形状

request_rec *h2_create_request_rec(const h2_request *req, conn_rec *c, int no_body);

入力と出力

  • **入力**:
  • req: 変換元の h2_request オブジェクト。
  • c: 接続を表す conn_rec オブジェクト。
  • no_body: ボディの有無を示すフラグ。
  • **出力**:
  • 成功時: 初期化済みの request_rec 構造体へのポインタ。
  • 失敗時: NULL (内部で ap_die を呼び出し、エラー応答を送信済み)。

前提条件と不変条件

  • req->authority が存在すること (AP_DEBUG_ASSERT による検証)。
  • Apache のバージョンに応じて ap_create_request または my_ap_create_request を使用してメモリが確保される。
  • 処理の過程で r->per_dir_config が r->server->lookup_defaults に設定される。

アルゴリズムまたは振る舞い

1. **リクエスト生成**: ap_create_request 等を用いて request_rec を作成。 2. **ヘッダー設定**: assign_headers を呼び出し、HTTP/2 ヘッダーを headers_in にマッピング。 3. **リクエストライン構築**: メソッドやスキーム、パスに基づき r->the_request を生成。CONNECT メソッドやプロキシ設定に応じた特殊処理を行う。 4. **バリデーション**: ap_parse_request_line および ap_check_request_header でリクエストの整合性を確認。 5. **パイプライン投入**: ap_add_input_filter_handle で HTTP 入力フィルタを追加し、ap_post_read_request を実行してフックを起動。 6. **エラー処理**: 異常発生時は die ラベルへ遷移し、ap_die でエラー応答を生成後、EOR (End of Request) バケットを挿入してリクエストを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2.c:748
  • modules/http2/h2_c2_filter.c:658
  • **呼び出し先**:
  • ap_create_request / my_ap_create_request
  • assign_headers
  • ap_run_pre_read_request
  • ap_parse_request_line
  • ap_check_request_header
  • ap_post_read_request
  • ap_die
  • ap_bucket_eor_create

関連データ構造

  • h2_request: HTTP/2 固有のリクエスト情報。
  • request_rec: Apache の標準リクエスト構造体。
  • conn_rec: Apache の接続構造体。
  • apr_bucket_brigade: EOR バケットの送出に使用。

失敗モードと境界条件

  • req->http_status が設定されている場合、即座に die へ遷移しエラー応答を返す。
  • CONNECT メソッドでスキームやパスが含まれている場合、HTTP_BAD_REQUEST を返す。
  • H2ProxyRequests が有効でスキームや権限情報が欠落している場合、HTTP_BAD_REQUEST を返す。
  • ap_post_read_request が失敗した場合、エラーログを出力して ap_die を実行する。

保守担当者向けメモ

  • Apache のバージョンによる API の差異を AP_MODULE_MAGIC_AT_LEAST マクロで吸収しているため、修正時はバージョン互換性に注意すること。
  • ap_die 呼び出し後は EOR バケットを挿入してリクエストのライフサイクルを正しく終了させる必要がある。
