md_get_ca_url_from_name

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:443

目的

CA（認証局）の名称から、対応する ACME サーバーの URL を解決します。

シグネチャまたは形状

apr_status_t md_get_ca_url_from_name(const char **purl, apr_pool_t *p, const char *name)

入力と出力

  • **入力**:
  • name: 解決対象の CA 名または URL 文字列。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • purl: 解決された URL、またはエラーメッセージが格納されるポインタ。
  • 戻り値: 成功時は APR_SUCCESS、失敗時はエラーコード。

前提条件と不変条件

  • KNOWN_CAs 配列がモジュール内で定義されており、CA 名と URL のマッピングを保持していること。
  • name が KNOWN_CAs に存在しない場合、md_util_abs_uri_check による絶対 URI の検証が行われます。

アルゴリズムまたは振る舞い

1. KNOWN_CAs 配列を走査し、name と一致する CA 名を検索します（大文字小文字を区別しない比較）。 2. 一致するものが見つかった場合、対応する URL を *purl に設定し終了します。 3. 一致しない場合、name をそのまま URL と見なして md_util_abs_uri_check で検証します。 4. 検証に失敗した場合、既知の CA 名一覧を生成し、エラー詳細を含むメッセージを *purl に格納してエラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md_config.c:524
  • **呼び出し先**:
  • apr_cstr_casecmp
  • md_util_abs_uri_check
  • apr_array_make
  • apr_psprintf
  • apr_array_pstrcat

関連データ構造

  • KNOWN_CAs: CA 名と URL を保持する静的配列（定義元は md_core.c 内）。
  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • name が既知の CA 名でも有効な絶対 URI でもない場合、APR_SUCCESS 以外のステータスコードを返し、*purl にはエラー内容を示す文字列が設定されます。

保守担当者向けメモ

  • 本関数は KNOWN_CAs 配列の構造に依存しています。新しい CA を追加する場合は KNOWN_CAs の定義を更新してください。
  • エラーメッセージ生成時に apr_pool_t を使用して動的に文字列を構築するため、呼び出し元でのプール管理に注意してください。
