CaseFilterRegisterHooks

定義

  • 種別: function
  • 定義箇所: modules/examples/mod_case_filter.c:123

目的

Apache HTTP Server のモジュールライフサイクルにおいて、フィルタの挿入フックおよび出力フィルタ自体を登録する。

シグネチャまたは形状

static void CaseFilterRegisterHooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p (モジュール初期化に使用されるメモリプール)
  • **出力**: なし（戻り値は void）

前提条件と不変条件

  • 本関数は Apache のモジュール読み込みプロセスにおいて、フック登録フェーズで呼び出されることを前提とする。
  • s_szCaseFilterName がフィルタ名として定義されている必要がある。

アルゴリズムまたは振る舞い

1. ap_hook_insert_filter を呼び出し、リクエスト処理時に CaseFilterInsertFilter が実行されるよう登録する（優先度は APR_HOOK_MIDDLE）。 2. ap_register_output_filter を呼び出し、s_szCaseFilterName という名称で CaseFilterOutFilter を出力フィルタとして登録する。フィルタのタイプは AP_FTYPE_RESOURCE に設定される。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュールロードプロセス（case_filter_module 定義内から参照される）。
  • **呼び出し先**:
  • ap_hook_insert_filter (Apache API)
  • ap_register_output_filter (Apache API)
  • CaseFilterInsertFilter (本モジュール内の関数)
  • CaseFilterOutFilter (本モジュール内の関数)

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • ap_filter_t (間接的): 登録されるフィルタの型。

失敗モードと境界条件

  • Apache API (ap_hook_insert_filter 等) が失敗した場合の戻り値チェックは本関数内には存在しない。

保守担当者向けメモ

  • 本関数はモジュールの初期化時に一度だけ実行される。フィルタの挙動を変更する場合は、登録される関数 (CaseFilterInsertFilter や CaseFilterOutFilter) の実装を確認すること。
