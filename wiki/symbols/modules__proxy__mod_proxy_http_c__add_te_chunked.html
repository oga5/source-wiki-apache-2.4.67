add_te_chunked

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_http.c:246

目的

HTTP リクエストヘッダーのブリゲードに Transfer-Encoding: chunked ヘッダーを追加し、バックエンドへのチャンク転送を指示する。

シグネチャまたは形状

static void add_te_chunked(apr_pool_t *p,
                           apr_bucket_alloc_t *bucket_alloc,
                           apr_bucket_brigade *header_brigade)

入力と出力

  • p: メモリ割り当てに使用するプール。
  • bucket_alloc: バケット割り当て用のアロケータ。
  • header_brigade: ヘッダーを格納するバケットブリゲード（入力および出力）。このブリゲードの末尾に Transfer-Encoding ヘッダーバケットが追加される。

前提条件と不変条件

  • header_brigade は有効なブリゲードであること。
  • p および bucket_alloc は、バケットの生成とメモリ管理のために有効であること。

アルゴリズムまたは振る舞い

1. Transfer-Encoding: chunked という文字列を定義する。 2. apr_pmemdup を使用して、指定されたプール p 上にヘッダー文字列のコピーを作成する。 3. ap_xlate_proto_to_ascii を呼び出し、ヘッダー文字列をプロトコル標準の ASCII 形式に変換する。 4. apr_bucket_pool_create を使用して、変換後の文字列を含むプールバケットを作成する。 5. APR_BRIGADE_INSERT_TAIL マクロを使用して、作成したバケットを header_brigade の末尾に挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_http.c:640 にて呼び出されている。
  • **呼び出し先**: apr_pmemdup, ap_xlate_proto_to_ascii, apr_bucket_pool_create, APR_BRIGADE_INSERT_TAIL (いずれも APR/Apache API)。

関連データ構造

  • apr_bucket_brigade: HTTP ヘッダーやボディを保持するバケットの連結リスト。
  • apr_bucket: ブリゲードを構成する個々のデータ単位。

失敗モードと境界条件

  • メモリ不足時には apr_pmemdup や apr_bucket_pool_create がエラーを返す可能性があるが、本関数自体は戻り値を持たないため、呼び出し元でのエラーハンドリングに依存する。

保守担当者向けメモ

  • 本関数は Transfer-Encoding ヘッダーを強制的に追加する。既存のヘッダーとの重複チェックは行わないため、呼び出し側で適切に制御すること。
  • modules/proxy/mod_proxy_http.c:523 にあるコメントの通り、本関数のロジックを変更する際は注意が必要である。
