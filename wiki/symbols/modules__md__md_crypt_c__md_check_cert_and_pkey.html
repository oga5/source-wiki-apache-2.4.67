md_check_cert_and_pkey

定義

  • 種別: function
  • 定義箇所: modules/md/md_crypt.c:2202

目的

提供された証明書チェーンの先頭にある証明書と、指定された秘密鍵が数学的に一致するか（ペアであるか）を検証する。

シグネチャまたは形状

apr_status_t md_check_cert_and_pkey(struct apr_array_header_t *certs, md_pkey_t *pkey)

入力と出力

  • **入力**:
  • certs: 証明書（md_cert_t）を格納する apr_array_header_t 構造体へのポインタ。
  • pkey: 検証対象の秘密鍵（md_pkey_t）。
  • **出力**:
  • APR_SUCCESS: 証明書と秘密鍵が一致した場合。
  • APR_ENOENT: 証明書配列が空の場合。
  • APR_EGENERAL: 証明書と秘密鍵が一致しない場合。

前提条件と不変条件

  • certs 配列には少なくとも1つの証明書が含まれている必要がある。
  • pkey は有効な md_pkey_t 構造体であること。

アルゴリズムまたは振る舞い

1. certs 配列の要素数を確認し、0であれば APR_ENOENT を返す。 2. 配列のインデックス 0 から最初の証明書（md_cert_t）を取得する。 3. OpenSSL の X509_check_private_key 関数を呼び出し、証明書の公開鍵と秘密鍵の整合性を検証する。 4. 検証結果が 1（成功）であれば APR_SUCCESS を返し、それ以外であれば APR_EGENERAL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_drive.c:834
  • modules/md/md_acme_drive.c:985
  • **呼び出し先**:
  • X509_check_private_key (OpenSSL)
  • APR_ARRAY_IDX (APR)

関連データ構造

  • md_cert_t: 証明書情報を保持する構造体。
  • md_pkey_t: 秘密鍵情報を保持する構造体。
  • apr_array_header_t: 証明書チェーンを保持する APR 配列。

失敗モードと境界条件

  • 配列が空の場合、検証を行わずに APR_ENOENT を返す。
  • OpenSSL 側の検証で不一致が検出された場合、APR_EGENERAL を返す。

保守担当者向けメモ

  • 本関数は証明書チェーンの先頭（リーフ証明書）のみを検証対象とする。
  • OpenSSL の X509_check_private_key に依存しているため、OpenSSL のバージョンやビルド設定による挙動の変化に注意すること。
