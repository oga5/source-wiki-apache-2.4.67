am_check_access

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_allowmethods.c:53

目的

HTTP リクエストのメソッドが、ディレクトリ単位で設定された許可リストに含まれているかを検証し、アクセス制御を行う。

シグネチャまたは形状

static int am_check_access(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）
  • **出力**: int（Apache のステータスコード）
  • DECLINED: 許可リストが未設定、またはメソッドが許可されている場合。
  • HTTP_METHOD_NOT_ALLOWED: メソッドが許可されていない場合。

前提条件と不変条件

  • allowmethods_module の設定が r->per_dir_config に存在すること。
  • r->method_number が Apache の標準メソッド番号として有効であること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリ設定 am_conf_t を取得する。 2. 設定が存在しない、または conf->allowed が 0（制限なし）の場合は DECLINED を返す。 3. r->allowed に conf->allowed を設定し、後続の処理へ情報を伝達する。 4. conf->allowed ビットマスクと AP_METHOD_BIT << method を比較し、許可されている場合は DECLINED を返す。 5. 許可されていない場合、エラーログを出力し、HTTP_METHOD_NOT_ALLOWED を返してリクエストを拒否する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の access_checker フック経由で呼び出される。
  • **呼び出し先**:
  • ap_get_module_config（設定取得）
  • ap_log_rerror（エラーログ出力）

関連データ構造

  • am_conf_t: ディレクトリごとの許可メソッドをビットマスクで保持する構造体。
  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • conf が NULL の場合、アクセス制限は適用されず DECLINED となる。
  • conf->allowed が 0 の場合、制限は適用されず DECLINED となる。
  • 許可リストにないメソッドがリクエストされた場合、405 Method Not Allowed が返される。

保守担当者向けメモ

  • 本関数は APR_HOOK_REALLY_FIRST で登録されており、他のアクセス制御モジュールよりも優先して実行される。
  • 許可リストの更新や変更を行う際は、am_conf_t のビットマスク操作が正しく行われているか確認すること。
