appendmem_to_linebuf

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:169

目的

sed_eval_t 構造体が保持する行バッファ（linebuf）に対し、指定されたメモリ領域の内容を追記します。

シグネチャまたは形状

apr_status_t appendmem_to_linebuf(sed_eval_t *eval, const char* sz, apr_size_t len)

入力と出力

  • **入力**:
  • eval: 評価コンテキストへのポインタ。
  • sz: 追記するデータのソースポインタ。
  • len: 追記するデータのバイト長。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • バッファ拡張に失敗した場合は、grow_line_buffer が返すエラーコード（apr_status_t）を返します。

前提条件と不変条件

  • eval は有効な sed_eval_t 構造体を指している必要があります。
  • eval->linebuf は grow_line_buffer によって管理される動的メモリ領域です。
  • 処理後、eval->lspend は追記されたデータの末尾を指すように更新されます。

アルゴリズムまたは振る舞い

1. 現在のバッファ使用量と追加分 len を合計し、必要なサイズ（reqsize）を算出します。 2. 現在のバッファ容量 eval->lsize が reqsize 未満の場合、grow_line_buffer を呼び出してバッファを拡張します。 3. 拡張に失敗した場合は即座にエラーを返します。 4. memcpy を使用して、eval->lspend が指す位置に sz から len バイトをコピーします。 5. eval->lspend ポインタを len 分だけ進め、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed1.c:194
  • modules/filters/sed1.c:450
  • modules/filters/sed1.c:468
  • **呼び出し先**:
  • grow_line_buffer (バッファ拡張が必要な場合)
  • memcpy (標準ライブラリ)

関連データ構造

  • sed_eval_t: 評価状態を保持する構造体。linebuf（バッファ開始位置）、lspend（バッファ末尾ポインタ）、lsize（バッファ総容量）が含まれます。

失敗モードと境界条件

  • **メモリ不足**: grow_line_buffer がメモリ確保に失敗した場合、そのエラーコードがそのまま返されます。
  • **バッファ上限**: grow_line_buffer の内部仕様により、8MB（MAX_BUF_SIZE）を超える拡張は APR_ENOMEM を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は sed_eval_t 内のポインタを直接操作するため、バッファが再確保された際のポインタ整合性に注意してください。
  • lspend は常に書き込み可能な次の位置を指すよう維持されます。
