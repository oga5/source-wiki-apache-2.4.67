ap_condition_if_match

定義

  • 種別: function
  • 定義箇所: modules/http/http_protocol.c:327

目的

HTTP リクエストに含まれる If-Match ヘッダーを評価し、リソースの ETag との整合性を判定する。

シグネチャまたは形状

ap_condition_e ap_condition_if_match(request_rec *r, apr_table_t *headers)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体（r->headers_in から If-Match を取得）。
  • headers: 比較対象の ETag を含むヘッダーテーブル。
  • **出力**:
  • AP_CONDITION_STRONG: 条件が一致した場合。
  • AP_CONDITION_NOMATCH: 条件が不一致の場合。
  • AP_CONDITION_NONE: リクエストに If-Match ヘッダーが存在しない場合。

前提条件と不変条件

  • RFC 準拠のため、ETag の比較には強力な比較関数（ap_find_etag_strong）を使用する。
  • If-Match ヘッダーの値が * の場合は、常に一致（AP_CONDITION_STRONG）とみなす。

アルゴリズムまたは振る舞い

1. リクエストヘッダーから If-Match を取得する。存在しない場合は AP_CONDITION_NONE を返す。 2. If-Match が * であるか、または headers 内の ETag と If-Match の値が ap_find_etag_strong で一致するかを確認する。 3. 一致する場合は AP_CONDITION_STRONG を返し、それ以外の場合は AP_CONDITION_NOMATCH を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/cache_storage.c:324
  • modules/http/http_protocol.c:571
  • **呼び出し先**:
  • apr_table_get (APR)
  • ap_find_etag_strong (Apache HTTP Server 内部関数)

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • apr_table_t: ヘッダー情報を保持するテーブル構造体。
  • ap_condition_e: 評価結果を示す列挙型。

失敗モードと境界条件

  • If-Match ヘッダーが欠落している場合は、エラーではなく AP_CONDITION_NONE を返す。
  • ETag ヘッダーが headers に存在しない場合、If-Match が * でなければ AP_CONDITION_NOMATCH となる。

保守担当者向けメモ

  • 本関数は If-Match の評価のみを担当する。他の条件付きリクエストヘッダーについては、ap_condition_if_none_match 等の対応する関数を参照すること。
