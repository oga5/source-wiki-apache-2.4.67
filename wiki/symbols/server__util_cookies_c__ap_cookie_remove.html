ap_cookie_remove

定義

  • 種別: function
  • 定義箇所: server/util_cookies.c:120

目的

RFC 2109 準拠のクッキー削除用ヘッダーを生成し、指定されたテーブルに追加することで、クライアント側でのクッキー破棄を促します。

シグネチャまたは形状

apr_status_t ap_cookie_remove(request_rec *r, const char *name, const char *attrs, ...);

入力と出力

  • **入力**:
  • r: リクエストレコード（メモリプール r->pool を使用）。
  • name: 削除対象のクッキー名。
  • attrs: クッキー属性文字列（NULL の場合はデフォルトの CLEAR_ATTRS が使用される）。
  • ...: 可変長引数として渡される apr_table_t * のリスト（末尾は NULL で終端）。
  • **出力**:
  • 戻り値: APR_SUCCESS を返します。

前提条件と不変条件

  • r->pool を使用してメモリが確保されるため、呼び出し側での明示的な解放は不要です。
  • 可変長引数で渡されるテーブルは、apr_table_t 型である必要があります。

アルゴリズムまたは振る舞い

1. name と Max-Age=0 を含む RFC 2109 形式のクッキー文字列を生成します。 2. 生成された文字列をデバッグログに出力します。 3. 可変長引数で渡された各 apr_table_t に対して、SET_COOKIE キーを用いて apr_table_addn を実行し、クッキー削除ヘッダーを追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_cookie.c:71 からの呼び出しが確認されています。
  • **呼び出し先**:
  • apr_pstrcat（文字列連結）
  • ap_log_rerror（ログ出力）
  • apr_table_addn（ヘッダー追加）

関連データ構造

  • request_rec: リクエスト状態およびメモリプールを保持。
  • apr_table_t: HTTP ヘッダーを格納するテーブル構造。

失敗モードと境界条件

  • attrs が NULL の場合、内部定義の CLEAR_ATTRS が使用されます。
  • 可変長引数の終端に NULL が指定されていない場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は RFC 2109 に準拠しています。
  • ap_cookie_remove2 との機能的な差異については、関連シンボルを参照してください。
