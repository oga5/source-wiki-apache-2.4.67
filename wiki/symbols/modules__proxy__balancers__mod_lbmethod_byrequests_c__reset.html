reset

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_byrequests.c:97

目的

ロードバランサーに登録されている全ワーカーの累積負荷状況（lbstatus）をリセットし、初期状態に戻します。

シグネチャまたは形状

static apr_status_t reset(proxy_balancer *balancer, server_rec *s)

入力と出力

  • **入力**:
  • balancer: 操作対象のロードバランサー構造体。
  • s: サーバーレコード（本関数内では未使用）。
  • **出力**:
  • APR_SUCCESS: 常に成功を返します。

前提条件と不変条件

  • balancer->workers が有効なワーカー配列を保持していること。
  • 本関数は、呼び出し元においてミューテックス等で適切に保護されている必要があります（共有メモリ上のワーカー状態を更新するため）。

アルゴリズムまたは振る舞い

1. balancer->workers からワーカー配列を取得します。 2. 配列内の全ワーカーを走査します。 3. 各ワーカーの s->lbstatus（累積負荷状況）を 0 に設定します。 4. APR_SUCCESS を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内には、本関数を呼び出している箇所は明示されていません。
  • **呼び出し先**: 本関数は標準的なポインタ操作のみを行い、他の関数を呼び出しません。

関連データ構造

  • proxy_balancer: ロードバランサーの構成情報とワーカーリストを保持します。
  • proxy_worker: 各バックエンドサーバーの状態を保持し、lbstatus を含みます。

失敗モードと境界条件

  • ワーカーが一つも登録されていない場合、ループは実行されず即座に APR_SUCCESS を返します。
  • 共有メモリへのアクセスにおいて、呼び出し元で排他制御が行われていない場合、競合状態が発生する可能性があります。

保守担当者向けメモ

  • 本関数は mod_lbmethod_byrequests における proxy_balancer_method プロバイダーの reset フックとして機能します。
  • lbstatus はリクエスト分配の優先度を決定する重要な指標であるため、本関数によるリセットはバランサーの負荷分散ロジックを一時的に初期化する効果を持ちます。
