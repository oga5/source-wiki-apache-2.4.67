ftp_get_PWD

定義

  • **種別**: 関数
  • **定義箇所**: modules/proxy/mod_proxy_ftp.c:888

目的

FTP サーバーに対して PWD (Print Working Directory) コマンドを発行し、現在の作業ディレクトリパスを取得する。

シグネチャまたは形状

char *ftp_get_PWD(request_rec *r, conn_rec *ftp_ctrl, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • ftp_ctrl: FTP 制御接続用の接続レコード。
  • bb: コマンド送受信に使用するバケットブリゲード。
  • **出力**:
  • 成功時: サーバーから返された現在のディレクトリパス（文字列）。
  • 失敗時: NULL。

前提条件と不変条件

  • ftp_ctrl は既に確立された有効な FTP 制御接続である必要がある。
  • 戻り値の文字列は r->pool から割り当てられるため、リクエストの生存期間に依存する。

アルゴリズムまたは振る舞い

1. proxy_ftp_command を使用して PWD コマンドをサーバーに送信する。 2. サーバーからのレスポンスコードを評価する:

  • 257: 成功。レスポンスメッセージからディレクトリパスを抽出し、ap_getword_conf を用いて解析して返す。
  • -1, 421, 550: 失敗。ap_proxyerror を呼び出し、HTTP_BAD_GATEWAY エラーを記録して NULL を返す。

    • 3. その他のレスポンスコードについては、明示的な処理を行わず NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_ftp.c:1719
  • modules/proxy/mod_proxy_ftp.c:1853
  • **呼び出し先**:
  • proxy_ftp_command (FTP コマンド送信とレスポンス取得)
  • ap_proxyerror (エラーログ記録)
  • ap_getword_conf (文字列解析)

関連データ構造

  • request_rec: Apache リクエスト情報。
  • conn_rec: 接続情報。
  • apr_bucket_brigade: データ転送用バッファ。

失敗モードと境界条件

  • サーバーが PWD コマンドをサポートしていない場合（500, 502 等）、または認証エラー等で失敗した場合（550 等）は NULL を返す。
  • 接続が切断された場合（421）はエラーとして処理される。

保守担当者向けメモ

  • PWD のレスポンス形式（257 コード）は RFC 959 に基づいているが、サーバーの実装により引用符の扱いが異なる場合があるため、ap_getword_conf による解析が適切か注意すること。
  • 本関数は r->pool を使用しているため、メモリリークの心配はない。
