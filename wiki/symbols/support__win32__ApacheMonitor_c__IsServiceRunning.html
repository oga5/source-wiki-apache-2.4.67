IsServiceRunning

定義

  • 種別: function
  • 定義箇所: support/win32/ApacheMonitor.c:696

目的

指定されたコンピュータ上の Apache サービスが現在実行中であるかどうかを判定し、その状態を返します。

シグネチャまたは形状

BOOL IsServiceRunning(LPCTSTR szServiceName, LPCTSTR szComputerName, LPDWORD lpdwPid)

入力と出力

  • **入力**:
  • szServiceName: 監視対象のサービス名。
  • szComputerName: サービスが動作するコンピュータ名（ローカルの場合は NULL）。
  • lpdwPid: 状態取得時に値を書き込むためのポインタ（呼び出し元が提供する場合）。
  • **出力**:
  • 戻り値: サービスが SERVICE_RUNNING 状態であれば TRUE、それ以外（停止中やエラー）は FALSE。
  • lpdwPid: サービスが正常にクエリできた場合、値 1 が書き込まれます。

前提条件と不変条件

  • サービス制御マネージャ（SCM）への接続権限が必要です。
  • サービス名が正しく指定されている必要があります。

アルゴリズムまたは振る舞い

1. OpenSCManager を使用して、指定されたコンピュータの SCM に接続します。 2. OpenService を使用して、対象サービスを SERVICE_QUERY_STATUS 権限で開きます。 3. QueryServiceStatus を呼び出し、現在のサービス状態を取得します。 4. 取得した状態が SERVICE_RUNNING と一致するかを判定し、結果を返します。 5. サービスが開けなかった場合、グローバル変数 g_bRescanServices を TRUE に設定し、サービスリストの再スキャンを促します。 6. 最後に、開いたすべてのハンドルを CloseServiceHandle で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: support/win32/ApacheMonitor.c:741（サービス状態のポーリング処理内）。
  • **呼び出し先**: Windows API (OpenSCManager, OpenService, QueryServiceStatus, CloseServiceHandle)。

関連データ構造

  • SERVICE_STATUS: Windows API の構造体。サービスの状態情報を保持します。

失敗モードと境界条件

  • SCM への接続に失敗した場合、直ちに FALSE を返します。
  • サービスが存在しない、またはアクセス権限がない場合、g_bRescanServices を TRUE に設定し、FALSE を返します。
  • lpdwPid が NULL の場合、ポインタの逆参照は行われません。

保守担当者向けメモ

  • 本関数はサービスの状態判定のみを行い、実際のサービス制御（起動・停止）は ApacheManageService が担当します。
  • lpdwPid 引数は現状、値が 1 に固定されており、実際のプロセス ID を取得する用途には使用されていません。
