add_alias_regex

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_alias.c:240

目的

Apache HTTP Server の設定ディレクティブ AliasMatch および ScriptAliasMatch を処理し、正規表現を用いたエイリアス定義を内部構造体へ登録する。

シグネチャまたは形状

static const char *add_alias_regex(cmd_parms *cmd, void *dummy, const char *fake, const char *real)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • fake: マッチング対象となる正規表現パターン。
  • real: 置換後のパスまたはURL。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ（文字列）を返す。

前提条件と不変条件

  • add_alias_internal を呼び出す際、正規表現モードを示すフラグとして 1 を渡す。
  • RSRC_CONF コンテキストでのみ使用可能。

アルゴリズムまたは振る舞い

1. 受け取った引数 fake（正規表現）と real（置換先）を、正規表現フラグ 1 と共に add_alias_internal へ転送する。 2. add_alias_internal の戻り値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_alias.c:723 (AliasMatch ディレクティブ登録時)
  • modules/mappers/mod_alias.c:725 (ScriptAliasMatch ディレクティブ登録時)
  • **呼び出し先**:
  • add_alias_internal

関連データ構造

  • cmd_parms: 設定解析用パラメータ。
  • alias_entry: add_alias_internal 経由で生成・格納されるエイリアス定義構造体。

失敗モードと境界条件

  • add_alias_internal が内部的に行う ap_check_cmd_context により、ディレクトリコンテキスト内での使用は拒否される。
  • 正規表現のコンパイルエラーやメモリ割り当て失敗は add_alias_internal の挙動に依存する。

保守担当者向けメモ

  • 本関数は AliasMatch と ScriptAliasMatch の共通ラッパーである。
  • ScriptAliasMatch の場合、第3引数として "cgi-script" が渡されるが、本関数のシグネチャ上は dummy 引数として無視される（add_alias_internal 側で適切に処理される）。
