dav_dbm_open_direct

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/dbm.c:132

目的

指定されたパスの DBM データベースを低レベルで開き、dav_db 構造体としてラップして提供する。

シグネチャまたは形状

dav_error * dav_dbm_open_direct(apr_pool_t *p, const char *pathname, int ro, dav_db **pdb)

入力と出力

  • p: メモリ割り当てに使用する APR プール。
  • pathname: 開くデータベースファイルのパス。
  • ro: 読み取り専用モードを指定するフラグ（非ゼロで読み取り専用）。
  • pdb: 開かれたデータベース状態を格納する dav_db 構造体へのポインタ（出力）。
  • 戻り値: 成功時は NULL、失敗時は dav_error 構造体へのポインタを返す。

前提条件と不変条件

  • pdb は呼び出し元で初期化され、関数内で NULL にリセットされる。
  • APU_MAJOR_VERSION が 1.7 以上の場合、apr_dbm_get_driver を使用してドライバをロードする。
  • データベースが正常に開かれた場合、*pdb には apr_pcalloc で割り当てられた dav_db 構造体が格納される。

アルゴリズムまたは振る舞い

1. *pdb を NULL に初期化する。 2. APR バージョンに応じて apr_dbm_open2 または apr_dbm_open を呼び出し、データベースを開く。

  • ro が真の場合、APR_DBM_READONLY を使用する。
  • ro が偽の場合、APR_DBM_RWCREATE を使用する。

    • 3. データベースのオープンに失敗し、かつ ro が偽（書き込みが必要）である場合、dav_fs_dbm_error を呼び出してエラーを返す。 4. データベースが正常に開かれた場合、apr_pcalloc で dav_db を確保し、プールとファイルハンドルをセットして返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/dav/fs/dbm.c:209
  • modules/dav/fs/lock.c:307
  • 呼び出し先:
  • apr_dbm_get_driver (条件付き)
  • apr_dbm_open2 / apr_dbm_open
  • dav_fs_dbm_error
  • apr_pcalloc
  • ap_log_error

関連データ構造

  • dav_db: データベース接続状態、プール、および apr_dbm_t を保持する構造体。
  • apr_dbm_t: APR が提供する DBM データベースハンドル。

失敗モードと境界条件

  • データベースファイルのオープン失敗: ro が偽の場合、dav_fs_dbm_error を経由してエラーが返される。
  • ドライバロード失敗: APU_MAJOR_VERSION が 1.7 以上でドライバのロードに失敗した場合、HTTP_INTERNAL_SERVER_ERROR を含むエラーが返される。
  • 読み取り専用モードで存在しないファイルを開いた場合: file が NULL となり、*pdb は NULL のまま正常終了する。

保守担当者向けメモ

  • 本関数は低レベルのオープン処理を担当する。上位の dav_propdb_open 等でメタデータの検証が行われる前の段階である。
  • DBM の並行書き込み時の再試行ロジックは含まれていないため、高負荷環境での挙動に注意が必要である。
