ap_get_conn_server_module_loglevel

定義

  • 種別: function
  • 定義箇所: server/util_debug.c:172

目的

コネクションおよびサーバーのコンテキストに基づき、特定のモジュールに対するログレベルを解決します。モジュール固有のログレベルが設定されていない場合、親コンテキストのデフォルトレベルへフォールバックします。

シグネチャまたは形状

int ap_get_conn_server_module_loglevel(const conn_rec *c,
                                       const server_rec *s,
                                       int module_index);

入力と出力

  • **入力**:
  • c: 現在のコネクション情報 (conn_rec)。
  • s: サーバー設定情報 (server_rec)。
  • module_index: 対象モジュールを識別するインデックス。
  • **出力**:
  • 解決されたログレベル（整数値）。

前提条件と不変条件

  • c および s は有効なポインタである必要があります。
  • module_index は、モジュール登録時に割り当てられた有効なインデックスである必要があります。

アルゴリズムまたは振る舞い

1. **ログ設定の選択**: コネクション c が独自のログ設定 (c->log) を持ち、かつそれがベースサーバーのログ設定と異なる場合は c->log を使用します。それ以外の場合はサーバー s のログ設定 (s->log) を使用します。 2. **フォールバック判定**: 以下のいずれかの条件に該当する場合、モジュール固有のレベルではなく、選択されたログ設定のデフォルトレベル (l->level) を返します。

  • module_index が負の値である。
  • ログ設定内の module_levels 配列が NULL である。
  • 指定された module_index に対応するログレベルが負の値である。

    • 3. **値の返却**: 上記のフォールバック条件に該当しない場合、l->module_levels[module_index] に格納されているモジュール固有のログレベルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/log.c:1203 (直接の呼び出しを確認)
  • include/http_log.h:193 および 209 (マクロ経由の利用)
  • **呼び出し先**:
  • 本関数はロジックの解決を行うのみであり、他の関数を呼び出しません。

関連データ構造

  • conn_rec: コネクション固有の状態を保持。
  • server_rec: サーバーの構成情報を保持。
  • ap_logconf: ログレベル設定およびモジュールごとのレベル配列を保持する構造体。

失敗モードと境界条件

  • module_index が範囲外の場合、または module_levels が初期化されていない場合は、安全にデフォルトのログレベルへフォールバックします。
  • c や s が NULL の場合の挙動については、本ソースコード内では明示的なガード処理は記述されていません。

保守担当者向けメモ

  • 本関数は server/util_debug.c で定義されていますが、http_config.h で同名のマクロとしても定義されており、ビルド環境に応じて最適化や差し替えが行われる可能性があります。
  • ログレベルの階層構造（Server > Conn）を扱うため、c->log の参照先が c->base_server->log と同一かどうかの判定が重要です。
