si_val_ca_urls

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:384

目的

Managed Domain (MD) のステータス情報から認証局 (CA) の URL 情報を抽出し、フォーマットして出力する。

シグネチャまたは形状

static void si_val_ca_urls(status_ctx *ctx, md_json_t *mdj, const status_info *info)

入力と出力

  • ctx: ステータス出力用のコンテキスト（メモリプールや出力先を含む）。
  • mdj: 処理対象の JSON データ構造。
  • info: 現在のステータス表示カラムの定義情報。
  • 出力: 戻り値なし。副作用として ctx を通じて HTTP レスポンス等へ CA URL を書き出す。

前提条件と不変条件

  • mdj から info->key に基づいて取得される JSON オブジェクトが存在すること。
  • val_url_print 関数が ctx に対して有効な出力先を提供していること。

アルゴリズムまたは振る舞い

1. mdj から info->key で指定された JSON オブジェクト (jcert) を取得する。存在しない場合は終了する。 2. jcert 内の MD_KEY_PROTO および MD_KEY_URL を取得する。 3. MD_KEY_URL が存在する場合、val_url_print を呼び出し、有効な CA URL を 1 件出力する。 4. MD_KEY_URL が存在しない場合、MD_KEY_URLS 配列を取得し、含まれるすべての URL に対してループ処理を行い、各々を val_url_print で出力する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md_status.c:707 (ステータス表示用の status_info 配列内)
  • 呼び出し先: val_url_print

関連データ構造

  • status_ctx: ステータス出力の制御コンテキスト。
  • md_json_t: JSON データの保持および走査用構造体。
  • status_info: ステータス表示の定義（キーや表示ロジックを保持）。

失敗モードと境界条件

  • jcert が取得できない場合、何も出力せずに正常終了する。
  • MD_KEY_URLS 配列が空の場合、ループは実行されず何も出力されない。

保守担当者向けメモ

  • 本関数は mod_status の出力フックの一部として機能する。
  • CA URL の表示ロジックを変更する場合は、val_url_print の仕様と併せて確認すること。
