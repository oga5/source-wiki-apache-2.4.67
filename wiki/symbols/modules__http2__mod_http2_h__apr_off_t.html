apr_off_t

定義

  • 種別: typedef
  • 定義箇所: modules/http2/mod_http2.h:71

目的

ファイルサイズやオフセット、バイト数など、ファイルシステムやネットワーク通信における「バイト単位の量」を表現するための型です。Apache Portable Runtime (APR) で定義される標準的な型であり、プラットフォームに応じて適切なサイズ（通常は 64 ビット）に解決されます。

シグネチャまたは形状

typedef [platform-dependent-integer] apr_off_t;

入力と出力

  • **入力**: 整数値（バイト数やオフセット値）。
  • **出力**: 符号付き整数として扱われ、ファイル内の位置やデータ長を表現します。

前提条件と不変条件

  • 負の値は、特定の API においてエラーや特殊な状態を示すために使用される場合があります。
  • 64 ビット環境では、非常に大きなファイルサイズを扱うことが可能です。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、ロジックを持ちません。Apache HTTP Server 全体で、ファイルポインタの移動（apr_file_seek 等）や、送信済みバイト数のカウント、リクエストボディの制限値管理などに広く利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_http2 内の http2_output_consumed 関数など、データ転送量を扱う関数で引数として利用されます。
  • **呼び出し先**: 本シンボルは型定義であるため、呼び出し先はありません。
  • **補足**: include/http_core.h や include/httpd.h 等のコアヘッダーで頻繁に参照されており、Apache の広範な API で利用されています。

関連データ構造

  • conn_rec: 接続情報を保持する構造体。bytes_sent などのメンバで apr_off_t が使用されます。
  • request_rec: リクエスト情報を保持する構造体。clength（コンテンツ長）や remaining（残りのボディサイズ）などで使用されます。
  • apr_file_t: ファイル操作を行うための構造体。オフセット指定に apr_off_t を使用します。

失敗モードと境界条件

  • **オーバーフロー**: 非常に大きなファイル（数テラバイト以上）を扱う際、プラットフォームが 64 ビットの apr_off_t をサポートしていない場合、値が切り詰められるリスクがあります。
  • **負の値**: 負の値を期待していない関数に負の値を渡した場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは APR のコア機能の一部です。mod_http2 固有の定義ではなく、Apache 全体で一貫して使用されるべき型です。
  • 修正や変更を行う際は、APR の仕様および Apache HTTP Server のコア API との整合性に注意してください。
