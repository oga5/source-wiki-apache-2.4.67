ssl_randseed_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:552

目的

OpenSSL の乱数生成器（PRNG）を初期化するためのシードソース情報を保持する構造体です。

シグネチャまたは形状

typedef struct {
    ssl_rsctx_t  nCtx;    /* シードのコンテキスト */
    ssl_rssrc_t  nSrc;    /* シードのソースタイプ */
    char        *cpPath;  /* ソースのパス（ファイルやデバイス等） */
    int          nBytes;  /* 読み込むバイト数 */
} ssl_randseed_t;

入力と出力

  • 入力: 設定ファイル等から読み込まれたシードのコンテキスト、ソース種別、パス、バイト数。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • cpPath は、nSrc がファイルやデバイスを指す場合に有効な文字列である必要があります。
  • nBytes は、乱数生成器に供給する適切なバイト数である必要があります。

アルゴリズムまたは振る舞い

本構造体は、mod_ssl が起動時に OpenSSL の乱数プールを初期化する際、どのソースからどの程度のデータを取得するかを定義するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/ssl/ssl_engine_config.c: 設定のメモリ確保および初期化時に使用。
  • modules/ssl/ssl_engine_rand.c: 乱数シードの適用処理において、配列要素として走査・参照。
  • 呼び出し先: 構造体定義のため、直接的な関数呼び出しはありません。

関連データ構造

  • ssl_rsctx_t: シードのコンテキスト定義。
  • ssl_rssrc_t: シードのソースタイプ定義。

失敗モードと境界条件

  • cpPath が NULL または無効なパスである場合、ssl_engine_rand.c でのシード読み込み処理が失敗する可能性があります。
  • nBytes が負の値や不適切な値である場合、乱数生成器の初期化が不完全になる可能性があります。

保守担当者向けメモ

  • 本構造体は ssl_engine_rand.c において配列として扱われることが多いため、メモリレイアウトやサイズ変更には注意が必要です。
  • OpenSSL 3.0 への移行に伴い、乱数生成器の初期化要件が変更される可能性があるため、ssl_engine_rand.c の実装と併せて確認してください。
