ap_mpm_dump_pidfile

定義

  • 種別: function
  • 定義箇所: server/mpm_common.c:316

目的

現在設定されている PID ファイルのパスを、指定された出力ストリームへ書き出します。

シグネチャまたは形状

void ap_mpm_dump_pidfile(apr_pool_t *p, apr_file_t *out)

入力と出力

  • **入力**:
  • p: パス解決に使用するメモリプール (apr_pool_t*)。
  • out: 書き出し先のファイルハンドル (apr_file_t*)。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • グローバル変数 ap_pid_fname が初期化されている必要があります。
  • out は書き込み可能な有効なファイルハンドルである必要があります。

アルゴリズムまたは振る舞い

1. ap_server_root_relative を使用し、ap_pid_fname（PID ファイル名）をサーバーのルートディレクトリからの相対パスとして解決します。 2. 解決されたパスを "PidFile: \"%s\"\n" というフォーマットで out に書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:5553 から呼び出されていることが確認されています。
  • **呼び出し先**:
  • ap_server_root_relative (サーバーのルートディレクトリを基準としたパス解決)
  • apr_file_printf (ファイルへのフォーマット出力)

関連データ構造

  • ap_pid_fname: PID ファイルのパスを保持するグローバル変数（server/mpm_common.c 内で定義）。
  • apr_pool_t: メモリ管理用構造体。
  • apr_file_t: ファイル I/O 用構造体。

失敗モードと境界条件

  • ap_pid_fname が未設定（NULL）の場合、ap_server_root_relative の挙動に依存します。
  • apr_file_printf が失敗した場合の戻り値は無視されます。

保守担当者向けメモ

  • 本関数は主にサーバー設定のダンプ出力機能の一部として利用されます。
  • パス解決には apr_pool_t を使用するため、呼び出し元で適切なライフサイクルのプールを提供してください。
