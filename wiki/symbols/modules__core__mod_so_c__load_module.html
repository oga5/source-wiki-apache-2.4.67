load_module

定義

  • 種別: function
  • 定義箇所: modules/core/mod_so.c:182

目的

Apache HTTP Server の実行時に、共有オブジェクト（DSO）を動的にロードし、Apache のモジュール構造として統合する。

シグネチャまたは形状

static const char *load_module(cmd_parms *cmd, void *dummy, const char *modname, const char *filename)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されないポインタ（内部で NULL に初期化される）。
  • modname: ロードするモジュール名。
  • filename: ロードする共有オブジェクトのファイルパス。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージを示す文字列（const char *）を返す。

前提条件と不変条件

  • ロード対象のモジュールが既にロードされていないこと。
  • ロード対象のモジュールが静的リンクされたモジュールと重複していないこと。
  • ロードされた DSO が MODULE_MAGIC_COOKIE を保持する有効な Apache モジュール構造体であること。

アルゴリズムまたは振る舞い

1. **重複チェック**: 既にロード済みのモジュールリストおよび静的リンク済みモジュールリストを走査し、名前の衝突を確認する。 2. **DSO ロード**: dso_load を呼び出し、指定されたファイルをプロセス空間にマッピングする。 3. **シンボル解決**: apr_dso_sym を使用して、共有オブジェクトからモジュール構造体（module）を取得する。 4. **検証**: モジュールの magic フィールドが MODULE_MAGIC_COOKIE と一致するか確認する。 5. **統合**: ap_add_loaded_module を呼び出し、Apache のグローバルモジュールリストへ追加する。 6. **クリーンアップ登録**: apr_pool_cleanup_register を使用し、サーバー終了時に unload_module が実行されるよう予約する。 7. **設定適用**: ap_single_module_configure を呼び出し、モジュールの設定を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: AP_INIT_TAKE2 マクロにより LoadModule ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: dso_load, ap_add_loaded_module, apr_pool_cleanup_register, ap_single_module_configure, apr_dso_sym 等。

関連データ構造

  • so_server_conf: ロードされたモジュールを追跡する設定構造体。
  • ap_module_symbol_t: ロードされたモジュールの名前と構造体ポインタを保持する。
  • module: Apache モジュールの基本構造体。

失敗モードと境界条件

  • **重複エラー**: 同名のモジュールが既に存在する場合、警告を出力して終了する。
  • **組み込みエラー**: 静的リンク済みのモジュールと名前が衝突する場合、エラーを返す。
  • **ロード失敗**: dso_load が失敗した場合、そのエラーを返す。
  • **シンボル解決失敗**: 指定されたモジュール名に対応するシンボルが見つからない場合、エラーを返す。
  • **互換性エラー**: magic 番号が一致しない場合、バージョン不整合としてエラーを返す。

保守担当者向けメモ

  • 本関数は cmd->pool（通常は pconf）を使用してメモリを管理しており、サーバーの再起動時に自動的にクリーンアップされる。
  • dummy 引数は execute_now への戻り値として利用されるため、関数冒頭で NULL に設定する必要がある。
  • apr_dso_error を通じてプラットフォーム固有のエラーメッセージを取得している。
