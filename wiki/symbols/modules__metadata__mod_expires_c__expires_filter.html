expires_filter

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:446

目的

HTTPレスポンスに対して、設定されたルールに基づき Expires ヘッダーおよび Cache-Control: max-age を付与する出力フィルタ。

シグネチャまたは形状

static apr_status_t expires_filter(ap_filter_t *f, apr_bucket_brigade *b)

入力と出力

  • **入力**:
  • f: 現在のフィルタコンテキスト（request_rec を含む）。
  • b: 処理対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS 等）。
  • フィルタチェーンの次段へブリゲードを渡す。

前提条件と不変条件

  • Expires ヘッダーが既に存在する場合、本関数はヘッダーの追加・上書きを行わない。
  • HTTPエラーレスポンス（ap_is_HTTP_ERROR が真）の場合、本フィルタは即座に自身を削除し、処理をスキップする。

アルゴリズムまたは振る舞い

1. **エラーチェック**: ステータスコードがエラーであれば、フィルタを解除してブリゲードを次段へ渡す。 2. **ヘッダー確認**: r->err_headers_out または r->headers_out に既存の Expires があるか確認。存在すれば何もしない。 3. **有効期限の決定**:

  • r->content_type に基づき expiresbytype テーブルを検索。
  • 一致しない場合、wildcards 設定が有効であればワイルドカード（type/*）による検索を試みる。
  • それでも見つからない場合、expiresdefault を使用する。

    • 4. **ヘッダー注入**: 有効期限が決定した場合、set_expiration_fields を呼び出してヘッダーをセットする。 5. **終了処理**: フィルタを解除し、ブリゲードを次段へ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apacheの出力フィルタチェーン（ap_register_output_filter により登録）。
  • **呼び出し先**:
  • ap_remove_output_filter
  • ap_pass_brigade
  • ap_field_noparam
  • apr_table_get
  • set_expiration_fields

関連データ構造

  • expires_dir_config: ディレクトリごとの設定（expiresbytype テーブル、expiresdefault、wildcards フラグを保持）。
  • request_rec: リクエスト情報（ヘッダーテーブル、コンテンツタイプ等）。

失敗モードと境界条件

  • content_type が設定されていない場合、expiresbytype の検索は失敗し、デフォルト値が適用される。
  • Expires ヘッダーが既に存在する場合、本フィルタは介入せず、既存の値を優先する。

保守担当者向けメモ

  • 本関数は ap_remove_output_filter を呼び出し、一度実行されるとフィルタチェーンから自身を削除する設計となっている。
  • check_code 関数と同様に、設定値の解析には注意が必要。
  • r->err_headers_out と r->headers_out の両方を走査することで、CGI等の他モジュールが生成したヘッダーとの競合を回避している。
