ap_slotmem_instance_t

定義

  • 種別: struct (不透明なハンドル)
  • 定義箇所: include/ap_slotmem.h:75

目的

共有メモリ領域を管理するための不透明なハンドル（インスタンス）です。slotmem プロバイダーが確保またはアタッチした特定の共有メモリ領域を識別し、その領域に対する操作の起点として機能します。

シグネチャまたは形状

typedef struct ap_slotmem_instance_t ap_slotmem_instance_t;

入力と出力

  • **入力**: ap_slotmem_provider_t の各関数（create, attach, dptr, get, put, num_slots, doall）において、操作対象のインスタンスとして引数 s または inst に渡されます。
  • **出力**: create および attach 関数によって、初期化されたインスタンスへのポインタが inst 引数を通じて返されます。

前提条件と不変条件

  • 本構造体は不透明（opaque）であり、直接のメンバアクセスは行わず、ap_slotmem_provider_t で定義された API を通じて操作する必要があります。
  • create または attach が成功した後にのみ、有効なインスタンスとして他の関数で使用可能です。

アルゴリズムまたは振る舞い

  • 共有メモリ領域のライフサイクル（確保、アタッチ、解放）を抽象化します。
  • dptr を介したメモリの直接参照や、get/put を介した安全なデータ転送の対象となります。
  • 内部的には apr_shm や apr_global_mutex を利用した排他制御やメモリ管理と結びついていますが、本構造体自体はそれらの詳細を隠蔽します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy_balancer 等の slotmem プロバイダーを利用するモジュール。
  • **呼び出し先**: mod_slotmem_shm.c 等のプロバイダー実装（具体的な関数ポインタの実装先）。
  • **補足**: ソースコード内の関数ポインタ定義（doall, create, attach, dptr, get, put, num_slots）において、本構造体が引数として参照されています。

関連データ構造

  • ap_slotmem_provider_t: 本インスタンスを操作するための関数ポインタ群を保持する構造体。
  • ap_slotmem_type_t: インスタンス作成時の特性を指定するフラグ。
  • apr_pool_t: インスタンスのメモリ管理に使用されるプール。

失敗モードと境界条件

  • インスタンスが初期化されていない状態で API に渡された場合の挙動は、プロバイダーの実装に依存します。
  • attach 失敗時や、無効な item_id を指定した操作を行った場合、各 API は apr_status_t 型のエラーコードを返します。

保守担当者向けメモ

  • 本構造体はインターフェース定義の一部であり、メモリレイアウトの詳細は実装モジュール（mod_slotmem_shm 等）に依存します。
  • 構造体の定義を変更する場合は、すべてのプロバイダー実装との互換性を維持する必要があります。
