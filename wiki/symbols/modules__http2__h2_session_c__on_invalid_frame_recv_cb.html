on_invalid_frame_recv_cb

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:214

目的

HTTP/2 セッションにおいて、nghttp2 が無効なフレームを受信した際に呼び出されるコールバック関数です。主にデバッグログの出力を行います。

シグネチャまたは形状

static int on_invalid_frame_recv_cb(nghttp2_session *ngh2,
                                    const nghttp2_frame *frame,
                                    int error, void *userp);

入力と出力

  • **入力**:
  • ngh2: nghttp2 セッションポインタ。
  • frame: 受信した無効なフレーム構造体。
  • error: エラーコード。
  • userp: h2_session 構造体へのポインタ。
  • **出力**:
  • 常に 0 を返します。

前提条件と不変条件

  • userp は有効な h2_session 構造体を指している必要があります。
  • nghttp2 ライブラリのコールバックとして登録されていることが前提です。

アルゴリズムまたは振る舞い

1. userp を h2_session 型にキャストします。 2. 接続レベルのデバッグログが有効（APLOGcdebug）であるか確認します。 3. 有効な場合、h2_util_frame_print を使用してフレーム情報をバッファに書き出します。 4. ap_log_cerror を使用し、受信したフレーム情報、およびセッションの累計送受信フレーム数を含むデバッグログを出力します。 5. 処理を終了し、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: nghttp2 ライブラリ内部（nghttp2_session_callbacks_set_on_invalid_frame_recv_callback で登録されたタイミング）。
  • **呼び出し先**:
  • h2_util_frame_print（ログ出力用）
  • ap_log_cerror（Apache ログ出力用）

関連データ構造

  • h2_session: HTTP/2 セッションの状態を保持する構造体。
  • nghttp2_frame: nghttp2 が定義するフレーム構造体。

失敗モードと境界条件

  • ログ出力設定がオフの場合、関数は何も行わず即座に 0 を返します。
  • フレーム情報の文字列化において、バッファサイズ（256バイト）を超える情報は切り詰められます。

保守担当者向けメモ

  • 本関数はエラーハンドリングのロジックを含まず、ログ出力のみを目的としています。
  • ngh2 引数は現在使用されていません。
