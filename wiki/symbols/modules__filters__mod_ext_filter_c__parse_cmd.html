parse_cmd

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ext_filter.c:154

目的

ExtFilterDefine ディレクティブで指定された外部コマンド文字列を解析し、ef_filter_t 構造体にコマンドパスと引数リストを格納する。

シグネチャまたは形状

static const char *parse_cmd(apr_pool_t *p, const char **args, ef_filter_t *filter)

入力と出力

  • **入力**:
  • p: メモリ割り当て用のプール。
  • args: 解析対象の文字列ポインタ。
  • filter: 設定を格納する ef_filter_t 構造体。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ（const char *）を返す。

前提条件と不変条件

  • args が指す文字列がダブルクォーテーションで囲まれている場合、エスケープされたクォーテーションを正しく処理する必要がある。
  • 成功時、filter->args にはトークン化された引数配列が、filter->command には実行コマンドが格納される。

アルゴリズムまたは振る舞い

1. **クォーテーション処理**: 文字列が " で始まる場合、対応する閉じクォーテーションまでを抽出する。その際、\ によるエスケープを考慮する。 2. **トークン化**:

  • クォーテーションで囲まれている場合、apr_tokenize_to_argv を使用して引数を分割する。
  • クォーテーションがない場合、ap_getword_white を使用して最初の単語をコマンドとして取得する。

    • 3. **格納**: 取得したコマンドを filter->command に、引数リストを filter->args に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_ext_filter.c:309 (define_filter 関数内)
  • **呼び出し先**:
  • apr_pstrndup, apr_tokenize_to_argv, apr_palloc (APR API)
  • ap_getword_white (Apache Core API)

関連データ構造

  • ef_filter_t: 外部フィルタの定義情報を保持する構造体。

失敗モードと境界条件

  • **エラー条件**:
  • 閉じクォーテーションが見つからない場合: "Expected cmd= delimiter" を返す。
  • apr_tokenize_to_argv が失敗した場合: "cmd= parse error" を返す。
  • コマンドが空の場合: "Invalid cmd= parameter" を返す。

保守担当者向けメモ

  • 本関数は ExtFilterDefine の引数解析に特化している。
  • filter->args のメモリはリクエストまたは設定プールから割り当てられるため、呼び出し側のライフサイクルに依存する。
