slotmem_attach

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:476

目的

既存の共有メモリ（SHM）セグメントにアタッチし、ap_slotmem_instance_t 構造体を初期化してスロット管理を可能にする。

シグネチャまたは形状

apr_status_t slotmem_attach(ap_slotmem_instance_t **new,
                            const char *name, apr_size_t *item_size,
                            unsigned int *item_num, apr_pool_t *pool)

入力と出力

  • **入力**:
  • name: アタッチ対象の共有メモリ名。
  • pool: インスタンス管理用のメモリプール。
  • **出力**:
  • new: 初期化された ap_slotmem_instance_t へのポインタ。
  • item_size: 取得したスロットのサイズ。
  • item_num: 取得したスロットの個数。
  • 戻り値: apr_status_t（成功時は APR_SUCCESS）。

前提条件と不変条件

  • グローバルプール gpool が初期化されていること。
  • slotmem_filenames により、指定された name から有効なファイルパスが生成できること。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: globallistmem を走査し、指定された名前のインスタンスが既にメモリ上に存在する場合はそれを再利用する。 2. **SHMアタッチ**: 存在しない場合、apr_shm_attach を呼び出し、共有メモリセグメントをプロセス空間にマッピングする。 3. **構造体構築**: 共有メモリの先頭にある sharedslotdesc_t を読み取り、オフセット計算（AP_SLOTMEM_OFFSET 等）を用いてデータ領域、利用状況フラグ配列（inuse）、およびメタデータへのポインタを ap_slotmem_instance_t に格納する。 4. **結果返却**: 構築したインスタンスを new にセットし、成功を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/slotmem/mod_slotmem_shm.c（プロバイダーテーブル内）
  • modules/slotmem/mod_slotmem_plain.c（プロバイダーテーブル内）
  • ※ソースコード上、mod_proxy_balancer.c および proxy_util.c で本関数失敗時のログ出力が確認できる。
  • **呼び出し先**:
  • apr_shm_attach
  • apr_shm_baseaddr_get
  • apr_pcalloc / apr_pstrdup
  • slotmem_filenames

関連データ構造

  • ap_slotmem_instance_t: インスタンス管理構造体。
  • sharedslotdesc_t: 共有メモリ内の構成情報ヘッダー。

失敗モードと境界条件

  • gpool が未初期化の場合: APR_ENOSHMAVAIL を返す。
  • slotmem_filenames が失敗した場合: APR_ENOSHMAVAIL を返す。
  • apr_shm_attach が失敗した場合: そのエラーコードをそのまま返す。

保守担当者向けメモ

  • 共有メモリのレイアウトは [desc] [num_free] [data_slots] [inuse_array] の順序で固定されており、オフセット計算は AP_SLOTMEM_OFFSET 等の定数に依存しているため、レイアウト変更時は注意が必要。
  • globallistmem によるキャッシュ機構があるため、同一プロセス内での重複アタッチは抑制される。
