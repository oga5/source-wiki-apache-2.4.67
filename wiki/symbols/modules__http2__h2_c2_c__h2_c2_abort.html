h2_c2_abort

定義

  • **種別**: 関数
  • **定義箇所**: modules/http2/h2_c2.c:157

目的

セカンダリ接続（c2）の I/O 処理を強制的に停止し、関連するデータ転送用バッファ（beam）をエラー状態に遷移させることで、デッドロックやリソースの不整合を防ぐ。

シグネチャまたは形状

void h2_c2_abort(conn_rec *c2, conn_rec *from)

入力と出力

  • **入力**:
  • c2: 終了対象となるセカンダリ接続の conn_rec 構造体。
  • from: 終了を要求した接続元（conn_rec）。
  • **出力**: なし（void）。

前提条件と不変条件

  • c2 には h2_conn_ctx_t コンテキストが関連付けられている必要がある。
  • conn_ctx->stream_id が設定されていること（AP_DEBUG_ASSERT による検証）。

アルゴリズムまたは振る舞い

1. h2_conn_ctx_get(c2) を呼び出し、接続コンテキストを取得する。 2. 接続がまだ中断されておらず、かつデータ送信済みであれば、h2_c2_log_io を呼び出して I/O 統計を記録する。 3. コンテキスト内の beam_in および beam_out が存在する場合、それぞれ h2_beam_abort を呼び出し、from を引数としてエラー状態に遷移させる。 4. 最後に c2->aborted フラグを 1 に設定し、接続が中断されたことを明示する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2.c:415
  • modules/http2/h2_mplx.c:185
  • modules/http2/h2_stream.c:643
  • **呼び出し先**:
  • h2_conn_ctx_get
  • h2_c2_log_io
  • h2_beam_abort

関連データ構造

  • conn_rec: Apache HTTP Server の接続構造体。
  • h2_conn_ctx_t: HTTP/2 接続ごとのメタデータ管理構造体。
  • h2_bucket_beam: ストリーム間データ転送用バッファ。

失敗モードと境界条件

  • conn_ctx が NULL の場合、AP_DEBUG_ASSERT によりデバッグビルド環境でアボートする。
  • beam_in または beam_out が NULL の場合、その beam に対する h2_beam_abort はスキップされる。

保守担当者向けメモ

  • 本関数は、異常終了時に beam を確実にクローズしてデッドロックを回避するために不可欠である。
  • c2->aborted フラグの更新は、接続のライフサイクル管理における最終的な状態確定を意味する。
