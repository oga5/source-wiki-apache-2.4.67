cmd_func

定義

  • 種別: typedef union
  • 定義箇所: include/http_config.h:82

目的

Apache HTTP Server の設定ディレクティブを処理するための、引数形式に応じた関数ポインタ群を保持する共用体です。

シグネチャまたは形状

以下の関数ポインタをメンバとして持ちます。

  • no_args: const char *(*)(cmd_parms *, void *)
  • raw_args: const char *(*)(cmd_parms *, void *, const char *)
  • take_argv: const char *(*)(cmd_parms *, void *, int, char *const [])
  • take1: const char *(*)(cmd_parms *, void *, const char *)
  • take2: const char *(*)(cmd_parms *, void *, const char *, const char *)
  • take3: const char *(*)(cmd_parms *, void *, const char *, const char *, const char *)
  • flag: const char *(*)(cmd_parms *, void *, int)

入力と出力

  • **入力**: 各関数は共通して cmd_parms *parms（設定コンテキスト）と void *mconfig（モジュール固有の設定データ）を受け取ります。
  • **出力**: 成功時は NULL を返し、エラー発生時はエラーメッセージを示す文字列（const char *）を返します。

前提条件と不変条件

  • 呼び出される関数は、設定ファイルの解析フェーズにおいて ap_walk_config を通じて実行されます。
  • mconfig は、対象モジュールが保持する設定構造体へのポインタである必要があります。

アルゴリズムまたは振る舞い

設定ファイルの解析中、ディレクティブの種類（cmd_how で定義される引数形式）に応じて、適切なメンバ関数が選択・実行されます。これにより、引数の数や形式に応じた柔軟な設定値の取り込みを実現します。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_walk_config が設定ツリーを走査する際に、対応する command_rec 構造体内の cmd_func を呼び出します。
  • **呼び出し先**: 不明（具体的なハンドラー実装は各モジュールに依存します）。

関連データ構造

  • command_rec: この共用体を保持し、ディレクティブの定義を構成します。
  • cmd_parms: ハンドラーに渡されるコンテキスト情報。
  • cmd_how: 引数の解析方法を決定する列挙型。

失敗モードと境界条件

  • ハンドラーがエラーメッセージ（文字列）を返した場合、設定解析プロセスは中断され、エラーとして報告されます。
  • mconfig が不正なポインタである場合、セグメンテーションフォールト等の未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • include/http_config.h:160 に別の cmd_func 型定義（typedef const char *(*cmd_func) ()）が存在しますが、これは本共用体とは別物であるため混同に注意してください。
  • 本共用体は command_rec 構造体の一部として使用されるため、変更時は command_rec への影響を考慮する必要があります。
