ap_cookie_remove2

定義

  • 種別: function
  • 定義箇所: server/util_cookies.c:148

目的

RFC 2965 準拠のクッキーを無効化するための Set-Cookie2 ヘッダーを生成し、指定されたヘッダーテーブルに追加します。

シグネチャまたは形状

apr_status_t ap_cookie_remove2(request_rec *r, const char *name2, const char *attrs2, ...);

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • name2: 削除対象のクッキー名。
  • attrs2: クッキーに付与する属性文字列（NULL の場合はデフォルトの CLEAR_ATTRS が使用される）。
  • ...: 可変長引数として、Set-Cookie2 ヘッダーを追加する apr_table_t 型のテーブルポインタを渡す（リストの末尾は NULL で終端）。
  • **出力**:
  • 常に APR_SUCCESS を返します。

前提条件と不変条件

  • r->pool を使用してメモリが割り当てられるため、呼び出し側での明示的な解放は不要です。
  • 可変長引数で渡されるテーブルは、apr_table_t 型である必要があります。

アルゴリズムまたは振る舞い

1. name2 と Max-Age=0 を含む RFC 2965 形式のクッキー文字列を生成します。 2. 生成された文字列をデバッグログに出力します。 3. 可変長引数で渡された各 apr_table_t に対して、apr_table_addn を使用して Set-Cookie2 ヘッダーを追加します。 4. 処理完了後、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session_cookie.c:84 からの呼び出しが確認されています。
  • **呼び出し先**:
  • apr_pstrcat (メモリ確保と文字列連結)
  • ap_log_rerror (ログ出力)
  • apr_table_addn (テーブルへのヘッダー追加)

関連データ構造

  • request_rec: リクエストの状態管理に使用。
  • apr_table_t: ヘッダーの格納先として使用。

失敗モードと境界条件

  • 本関数自体はエラーを返さない設計ですが、内部で使用する apr_table_addn 等の APR 関数がメモリ不足等の理由で失敗した場合の挙動については、APR の仕様に依存します。

保守担当者向けメモ

  • 本関数は RFC 2965 準拠のクッキー削除専用です。RFC 2109 準拠の削除には ap_cookie_remove を使用してください。
  • 可変長引数の終端に NULL を渡し忘れると、メモリ破壊やセグメンテーション違反を引き起こす可能性があるため注意してください。
