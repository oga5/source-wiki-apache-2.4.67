set_enabled

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_express.c:61

目的

Apache HTTP Server の設定ディレクティブ ProxyExpressEnable を処理し、サーバーごとの express_server_conf 構造体における有効/無効フラグを更新します。

シグネチャまたは形状

static const char *set_enabled(cmd_parms *cmd, void *dconf, int flag)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dconf: ディレクトリ設定（本関数では未使用）。
  • flag: ディレクティブから渡される真偽値（int）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • cmd->server->module_config から proxy_express_module に関連付けられた express_server_conf が取得可能である必要があります。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定 (express_server_conf) を取得します。 2. 引数 flag の値を sconf->enabled に代入します。 3. NULL を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_express.c:224 にて AP_INIT_FLAG マクロを通じて ProxyExpressEnable ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: ap_get_module_config（Apache Core API）。

関連データ構造

  • express_server_conf: サーバーごとの設定を保持する構造体。本関数はこの中の enabled メンバを更新します。

失敗モードと境界条件

  • ap_get_module_config が適切な設定構造体を返せない場合、セグメンテーションフォールトが発生する可能性があります（ただし、Apache の設定フェーズでは通常正しく初期化されています）。

保守担当者向けメモ

  • 本関数は ProxyExpressEnable ディレクティブ専用のセッターです。
  • dconf 引数は AP_INIT_FLAG の仕様上存在しますが、このモジュールではサーバー単位の設定 (sconf) を操作するため使用されていません。
