dir_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_dir.c:45

目的

ディレクトリごとの設定情報を保持し、mod_dir モジュールによるディレクトリインデックスの解決、スラッシュ補完、およびフォールバックリソースの処理を制御するために使用されます。

シグネチャまたは形状

typedef struct dir_config_struct {
    apr_array_header_t *index_names;
    moddir_cfg do_slash;
    moddir_cfg checkhandler;
    int redirect_index;
    const char *dflt;
} dir_config_rec;

入力と出力

  • **入力**: 設定ファイル内のディレクティブ（DirectoryIndex, DirectorySlash, DirectoryCheckHandler, DirectoryIndexRedirect, FallbackResource）から値が設定されます。
  • **出力**: fixup_dir や fixup_dflt などのフック関数内で、リクエスト処理の挙動を決定するためのパラメータとして参照されます。

前提条件と不変条件

  • Apache の設定プール（pool）上で管理されます。
  • fixup フック内で使用される際、リクエストの per_dir_config から ap_get_module_config を通じて取得されます。

アルゴリズムまたは振る舞い

  • index_names: DirectoryIndex で指定されたファイル名のリストを保持し、ディレクトリリクエスト時に順次探索されます。
  • do_slash: DirectorySlash 設定に基づき、末尾スラッシュのないディレクトリリクエストに対してリダイレクトを行うか否かを制御します。
  • dflt: FallbackResource で指定されたリソースパスを保持し、ファイルが存在しない場合のフォールバック先として使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_dir.c 内の各種設定ハンドラ（add_index, configure_slash 等）が構造体の値を設定します。
  • dir_fixups およびその下位関数（fixup_dir, fixup_dflt）が設定値を読み取ります。
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_array_header_t: index_names を格納するために使用される APR の配列構造体。
  • moddir_cfg: do_slash および checkhandler の状態を保持する列挙型または整数型。

失敗モードと境界条件

  • FallbackResource が設定されている場合、指定先が存在しないと無限ループに陥る可能性があるため、設定には注意が必要です。
  • path_info が存在する場合、ディレクトリインデックス処理はスキップされます。

保守担当者向けメモ

  • 本構造体は Apache の設定マージ処理（merge_dir_configs）の対象となります。
  • メモリ管理は Apache のプール機構に依存しているため、手動での解放は不要です。
