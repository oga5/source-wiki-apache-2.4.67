logio_config_t

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_logio.c:47

目的

接続ごとの入出力バイト数および直近のリクエストにおけるバイト数を保持し、mod_logio モジュールによる統計情報の追跡を管理する。

シグネチャまたは形状

typedef struct logio_config_t {
    apr_off_t bytes_in;         /* 受信バイト数 */
    apr_off_t bytes_out;        /* 送信バイト数 */
    apr_off_t bytes_last_request; /* 直近のリクエストのバイト数 */
} logio_config_t;

入力と出力

  • **入力**: ap_logio_add_bytes_out や logio_in_filter を通じて、ネットワーク経由の入出力バイト数が加算される。
  • **出力**: log_bytes_in, log_bytes_out, log_bytes_combined などのログハンドラが、本構造体から値を読み取り文字列として出力する。

前提条件と不変条件

  • 接続プール (c->pool) に割り当てられ、接続の生存期間中保持される。
  • logio_transaction フックにより、リクエスト終了時にカウンタがリセットされる。

アルゴリズムまたは振る舞い

1. **初期化**: logio_pre_conn にて apr_pcalloc を使用してメモリが確保される。 2. **追跡**: 接続に関連付けられた conn_config に格納され、フィルタや外部関数から参照・更新される。 3. **リセット**: リクエスト処理の完了後、次のリクエストの計測に備えて状態が更新される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • logio_pre_conn: 構造体の初期化。
  • logio_in_filter: 受信バイト数の加算。
  • log_bytes_in, log_bytes_out, log_bytes_combined: ログ出力時の値参照。
  • logio_transaction: カウンタのリセット。
  • **呼び出し先**:
  • ap_get_module_config: conn_config から本構造体を取得するために使用。
  • apr_pcalloc: 構造体のメモリ確保に使用。

関連データ構造

  • conn_rec: 接続情報を保持し、conn_config を通じて本構造体を保持する。
  • logio_module: モジュール設定の識別に使用。

失敗モードと境界条件

  • メモリ確保失敗時: apr_pcalloc が失敗した場合の挙動は APR の仕様に依存する。
  • 接続終了後: 接続が閉じられた後のアクセスは未定義の動作となる。

保守担当者向けメモ

  • 本構造体は接続の生存期間に依存するため、リクエストを跨ぐ統計の累積については logio_transaction での適切なリセット処理が不可欠である。
  • ログ出力の順序は mod_log_config に依存するため、フック順序の変更には注意が必要。
