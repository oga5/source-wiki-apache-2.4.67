ssl_init_connection_ctx

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:461

目的

接続 (conn_rec) に紐づく SSL 接続コンテキスト (SSLConnRec) を生成、取得、または再初期化し、適切なディレクトリ設定 (SSLDirConfigRec) を適用する。

シグネチャまたは形状

static SSLConnRec *ssl_init_connection_ctx(conn_rec *c,
                                           ap_conf_vector_t *per_dir_config,
                                           int reinit)

入力と出力

  • **入力**:
  • c: 現在の接続構造体 (conn_rec)。
  • per_dir_config: ディレクトリごとの設定ベクトル。NULL の場合はサーバーのデフォルト設定が使用される。
  • reinit: 再初期化フラグ。0 以外の場合、既存のコンテキストがあっても設定を更新する。
  • **出力**:
  • 接続に関連付けられた SSLConnRec 構造体へのポインタ。

前提条件と不変条件

  • SSLConnRec は接続の生存期間中、conn_rec のプール (c->pool) を使用して管理される。
  • reinit が 0 の場合、既存の SSLConnRec があれば即座にそれが返される。
  • per_dir_config が提供される場合、ssl_module の設定が sslconn->dc に格納される。

アルゴリズムまたは振る舞い

1. myConnConfig(c) を使用して既存の SSLConnRec を検索する。 2. コンテキストが存在しない場合は apr_pcalloc で新規作成し、初期化フラグを立てる。 3. reinit フラグが立っているか、新規作成された場合に per_dir_config またはサーバーのデフォルト設定から sslconn->dc を更新する。 4. 新規作成時のみ、sslconn->server を設定し、cipher_suite を接続タイプ（プロキシか否か）に応じて初期化し、myConnConfigSet(c, sslconn) で接続に保存する。 5. 最終的に sslconn を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/mod_ssl.c:549
  • modules/ssl/mod_ssl.c:590
  • **呼び出し先**:
  • ap_get_module_config: 設定ベクトルの取得に使用。
  • apr_pcalloc: メモリ確保に使用。
  • myConnConfig / myConnConfigSet: 接続コンテキストの取得・保存に使用。
  • mySrvConfig: サーバー設定の取得に使用。

関連データ構造

  • SSLConnRec: 接続ごとの SSL 状態を保持する主要構造体。
  • conn_rec: Apache の接続構造体。
  • SSLSrvConfigRec: サーバーごとの SSL 設定。
  • SSLDirConfigRec: ディレクトリごとの SSL 設定。

失敗モードと境界条件

  • c->pool が破棄された後の呼び出しは未定義の動作を引き起こす。
  • per_dir_config が NULL かつサーバーのデフォルト設定が存在しない場合、ap_get_module_config の結果が不正になる可能性がある。

保守担当者向けメモ

  • mod_proxy はリクエストごとに per_dir_config を変更するため、reinit フラグを利用して sslconn->dc を適切に更新する設計となっている。
  • cipher_suite の初期化ロジックは、c->outgoing (プロキシ接続) かどうかで参照先が分岐するため、変更時は注意が必要。
