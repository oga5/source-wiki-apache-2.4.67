setup_signal_names

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:193

目的

プロセス間通信（IPC）で使用されるシャットダウンおよび再起動用イベント名の接頭辞を設定し、それに基づいた具体的なイベント名を生成する。

シグネチャまたは形状

static void setup_signal_names(char *prefix)

入力と出力

  • **入力**: prefix (char型ポインタ) - イベント名の基点となる文字列（通常はプロセスIDを含む文字列）。
  • **出力**: なし（副作用として静的変数を更新）。

前提条件と不変条件

  • signal_name_prefix、signal_shutdown_name、signal_restart_name の各静的バッファが、生成される文字列を格納するのに十分なサイズを持っていること。

アルゴリズムまたは振る舞い

1. 引数 prefix を signal_name_prefix にコピーする。 2. signal_name_prefix に _shutdown を付加し、signal_shutdown_name に格納する。 3. signal_name_prefix に _restart を付加し、signal_restart_name に格納する。 4. 各操作には apr_snprintf を使用し、バッファオーバーフローを防止する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/mpm_winnt.c:1578
  • server/mpm/winnt/mpm_winnt.c:1665
  • **呼び出し先**:
  • apr_snprintf (APRライブラリ関数)

関連データ構造

  • signal_name_prefix (静的バッファ)
  • signal_shutdown_name (静的バッファ)
  • signal_restart_name (静的バッファ)

失敗モードと境界条件

  • prefix が NULL の場合、apr_snprintf の挙動に依存する。
  • prefix が長すぎる場合、apr_snprintf により切り詰められる。

保守担当者向けメモ

  • 本関数は、親プロセスと子プロセス間で同期オブジェクト（イベント）を特定するための名前空間を構築する。
  • 呼び出し元で apr_psprintf を使用して生成された動的な文字列が渡されるため、メモリ管理は呼び出し側のプール（pconf や pchild）に依存する。
