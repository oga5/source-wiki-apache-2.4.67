mpmt_os2_check_config

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:483

目的

MPM 固有の設定ディレクティブ（StartServers および MinSpareThreads）の値を検証し、許容範囲外の場合は警告を出力した上で安全な最小値に補正する。

シグネチャまたは形状

static int mpmt_os2_check_config(apr_pool_t *p, apr_pool_t *plog,
                                 apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • p, plog, ptemp: APR プール。
  • s: サーバーレコード。
  • **出力**:
  • 常に OK を返す。
  • 副作用として、グローバル変数 ap_daemons_to_start および ap_min_spare_threads が修正される場合がある。

前提条件と不変条件

  • サーバー起動時および再起動時に呼び出される。
  • restart_num 静的変数により、初回起動時のみ startup フラグが有効になる。

アルゴリズムまたは振る舞い

1. **起動判定**: restart_num をインクリメントし、初回呼び出しであれば startup を真とする。 2. **ap_daemons_to_start の検証**: 値が 0 未満の場合、警告ログを出力し、値を 1 に強制設定する。 3. **ap_min_spare_threads の検証**: 値が 1 未満の場合、警告ログを出力し、値を 1 に強制設定する。 4. ログ出力時、初回起動時は APLOG_STARTUP フラグを付与し、それ以外は通常のサーバーログとして出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/mpmt_os2/mpmt_os2.c:529 にて ap_hook_check_config に登録されている。
  • **呼び出し先**: ap_log_error（ログ出力用 API）。

関連データ構造

  • server_rec: サーバー設定情報を保持する構造体。

失敗モードと境界条件

  • 設定値が負数または 0 の場合、サーバーの動作を維持するために自動的に 1 へ補正される。
  • ログ出力は APLOG_WARNING レベルで行われる。

保守担当者向けメモ

  • 本関数は設定の妥当性チェックのみを行う。OS/2 特有の制限である HARD_SERVER_LIMIT（10）との整合性は、本関数内では直接検証されていない。
