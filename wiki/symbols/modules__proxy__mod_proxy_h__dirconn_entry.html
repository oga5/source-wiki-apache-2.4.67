dirconn_entry

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:142

目的

プロキシの接続制御（ProxyBlock 等）において、接続先ホストやネットワークの許可・拒否を判定するためのエントリを保持する構造体です。

シグネチャまたは形状

struct dirconn_entry {
    char *name;
    struct in_addr addr, mask;
    struct apr_sockaddr_t *hostaddr;
    int (*matcher) (struct dirconn_entry * This, request_rec *r);
};

入力と出力

  • name: 識別名またはホスト名。
  • addr, mask: IPアドレスベースの判定用アドレスとマスク。
  • hostaddr: ホスト名解決後のソケットアドレス情報。
  • matcher: 判定ロジックを実装する関数ポインタ。request_rec を受け取り、判定結果（整数）を返します。

前提条件と不変条件

  • matcher 関数は、dirconn_entry 構造体へのポインタとリクエスト情報を引数に取る必要があります。
  • mod_proxy.c 内で apr_array_make を用いて配列として管理されます。

アルゴリズムまたは振る舞い

  • matcher フィールドに設定された関数（proxy_match_ipaddr, proxy_match_domainname, proxy_match_hostname, proxy_match_word 等）を呼び出すことで、リクエストが対象の接続制限ルールに合致するかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy.c 内で設定の初期化や参照が行われています。
  • **呼び出し先**: proxy_util.c 内の proxy_match_ipaddr, proxy_match_domainname, proxy_match_hostname, proxy_match_word が matcher として割り当てられることが確認されています。

関連データ構造

  • request_rec: Apache のリクエスト情報構造体。
  • apr_sockaddr_t: APR のソケットアドレス構造体。

失敗モードと境界条件

  • matcher 関数が未設定（NULL）の場合の動作については、本ソース excerpt からは判断できません。
  • ホスト名解決に失敗した場合の hostaddr の状態については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 本構造体は mod_proxy の設定の一部として配列管理されています。判定ロジックを追加・変更する場合は proxy_util.c の matcher 実装を確認してください。
