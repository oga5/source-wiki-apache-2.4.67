set_cookie_style

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_usertrack.c:416

目的

Apache 設定ディレクティブ CookieStyle の値を解析し、cookie_dir_rec 構造体内の Cookie 形式（style）を設定します。

シグネチャまたは形状

static const char *set_cookie_style(cmd_parms *cmd, void *mconfig, const char *name)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • mconfig: 現在のディレクトリ設定構造体 (cookie_dir_rec)。
  • name: 設定値として渡された文字列（"Netscape", "Cookie", "RFC2109", "Cookie2", "RFC2965" のいずれか）。
  • **出力**:
  • 成功時: NULL を返します。
  • 失敗時: エラーメッセージを含む文字列（apr_psprintf で生成）を返します。

前提条件と不変条件

  • mconfig は cookie_dir_rec 型としてキャスト可能である必要があります。
  • name は大文字小文字を区別せずに比較されます。

アルゴリズムまたは振る舞い

1. mconfig を cookie_dir_rec 型にキャストします。 2. name を以下のキーワードと比較し、対応する列挙値を dcfg->style に格納します。

  • "Netscape" → CT_NETSCAPE
  • "Cookie" または "RFC2109" → CT_COOKIE
  • "Cookie2" または "RFC2965" → CT_COOKIE2

    • 3. 上記のいずれにも一致しない場合、エラーメッセージを生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_usertrack.c:472 にて AP_INIT_TAKE1 マクロを通じて設定ディレクティブ CookieStyle のハンドラとして登録されています。
  • **呼び出し先**: apr_psprintf（エラー発生時のみ）。

関連データ構造

  • cookie_dir_rec: ディレクトリごとの設定を保持する構造体。style メンバを更新します。

失敗モードと境界条件

  • 未定義のキーワードが指定された場合、Invalid CookieStyle keyword: '...' という形式のエラーメッセージを返し、設定処理を中断します。

保守担当者向けメモ

  • 本関数は CookieStyle ディレクティブのバリデーションを担います。新しい Cookie 形式を追加する場合は、比較ロジックおよび cookie_type_e（関連ヘッダーで定義）の拡張が必要です。
