stream_iter_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_mplx.c:386

目的

ストリームの反復処理（イテレーション）を行う際に、コールバック関数とその実行コンテキストを保持するための構造体です。

シグネチャまたは形状

typedef struct {
    h2_mplx_stream_cb *cb;
    void *ctx;
} stream_iter_ctx_t;

入力と出力

  • cb: ストリームに対して実行されるコールバック関数へのポインタ。
  • ctx: コールバック関数に渡される任意のユーザーデータ（コンテキスト）。

前提条件と不変条件

  • cb は h2_mplx_stream_cb 型の関数ポインタである必要があります。
  • 本構造体は、ストリームのリストやキューを走査する際の引数として利用されます。

アルゴリズムまたは振る舞い

ストリームの反復処理において、この構造体を通じて対象のストリームに対する操作（コールバック）をカプセル化します。反復処理の実行者は、この構造体からコールバックを取り出し、保持しているコンテキストと共にストリームへ適用します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_mplx.c 内のストリーム走査処理（stream_iter_ctx_t *x = ctx; および stream_iter_ctx_t x; の参照箇所）。
  • 呼び出し先: cb フィールドに格納された関数が、反復処理の過程で呼び出されます。

関連データ構造

  • h2_mplx_stream_cb: ストリーム処理用コールバックの型定義。

失敗モードと境界条件

  • cb が NULL の状態で反復処理が行われた場合、未定義の動作（セグメンテーション違反等）を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は h2_mplx.c 内の局所的な反復処理のために設計されています。
  • 構造体の拡張が必要な場合は、h2_mplx_stream_cb のシグネチャとの整合性に注意してください。
