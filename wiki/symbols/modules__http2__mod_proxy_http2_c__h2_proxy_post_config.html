h2_proxy_post_config

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_proxy_http2.c:74

目的

mod_proxy_http2 モジュールの初期化処理を行い、nghttp2 ライブラリのバージョン情報をログ出力します。また、モジュール初期化の二重実行を防止するためのカウンター制御を行います。

シグネチャまたは形状

static int h2_proxy_post_config(apr_pool_t *p, apr_pool_t *plog,
                                apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • p, plog, ptemp: Apache のメモリプール。
  • s: 現在のサーバー設定レコード。
  • **出力**:
  • int: 成功時は APR_SUCCESS (0) を返します。

前提条件と不変条件

  • s->process->pool を使用して、プロセス生存期間中に永続的な初期化フラグを保持します。
  • nghttp2_version(0) が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. apr_pool_userdata_get を呼び出し、mod_proxy_http2_init_counter キーで初期化済みかを確認します。 2. 未初期化の場合、apr_pool_userdata_set でフラグをセットし、即座に APR_SUCCESS を返します。 3. 二回目以降の呼び出し時（子プロセス等）に、nghttp2_version を取得し、モジュールバージョンと nghttp2 のバージョンを APLOG_INFO レベルでログ出力します。 4. APR_RETRIEVE_OPTIONAL_FN(http2_is_h2) を呼び出し、is_h2 関数ポインタを取得します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_proxy_http2.c:483 にて ap_hook_post_config に登録されています。
  • **呼び出し先**:
  • apr_pool_userdata_get / apr_pool_userdata_set (APR)
  • nghttp2_version (nghttp2)
  • ap_log_error (Apache Core)
  • APR_RETRIEVE_OPTIONAL_FN (Apache APR)

関連データ構造

  • server_rec: サーバー設定情報。
  • nghttp2_info: nghttp2 ライブラリのバージョン情報構造体。

失敗モードと境界条件

  • nghttp2_version が NULL を返す場合、ログ出力時にバージョン文字列として "unknown" を使用します。
  • APR_RETRIEVE_OPTIONAL_FN が関数を見つけられない場合、is_h2 は NULL になります。

保守担当者向けメモ

  • 本関数は post_config フックとして動作するため、サーバー起動時および再起動時に実行されます。
  • is_h2 の取得はオプション機能への依存解決であり、mod_http2 がロードされていない環境でもモジュール自体がクラッシュしないよう設計されています。
