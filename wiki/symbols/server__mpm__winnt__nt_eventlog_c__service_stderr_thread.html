service_stderr_thread

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/nt_eventlog.c:29

目的

標準エラー出力（stderr）のパイプを監視し、受信した文字列をWindowsイベントログへ転送するバックグラウンドスレッドとして動作します。

シグネチャまたは形状

static DWORD WINAPI service_stderr_thread(LPVOID hPipe)
  • 引数 hPipe: 監視対象となるパイプの読み取りハンドル。

入力と出力

  • **入力**: hPipe を通じて渡されるパイプからのバイトストリーム。
  • **出力**: Windowsイベントログへの書き込み（ReportEvent API経由）。戻り値は常に 0。

前提条件と不変条件

  • stderr がパイプにリダイレクトされている環境で実行されること。
  • stderr_ready イベントが適切に管理されていること。
  • イベントログのソースとして「Apache Service」が登録可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: イベントログのレジストリ設定（EventMessageFile等）を行い、RegisterEventSourceW でイベントソースを登録します。 2. **同期**: SetEvent(stderr_ready) を呼び出し、スレッドの準備完了をメインプロセスに通知します。 3. **監視ループ**: ReadFile でパイプから1バイトずつ読み取ります。

  • 改行コードを検知するか、バッファ（256バイト）が満杯になると、蓄積された文字列を ReportEvent でイベントログへ送信します。

    • 4. **終了処理**: パイプがクローズされた際、エラー発生時、または終了シグナル受信時に、イベントソースの登録解除、ハンドルクローズ、メモリプール（apr_pool_t）の破棄を行い終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/nt_eventlog.c 内の関数（CreateThread を介して起動）。
  • **呼び出し先**:
  • Windows API: RegisterEventSourceW, ReadFile, ReportEvent, DeregisterEventSource, SetEvent, CloseHandle。
  • APR: apr_pool_create_ex, apr_pool_destroy, ap_regkey_open, ap_regkey_value_set 等。

関連データ構造

  • ap_regkey_t: レジストリ操作用構造体。
  • apr_pool_t: メモリ管理用プール。
  • stderr_ready: スレッド同期用イベントハンドル（外部定義）。
  • stderr_thread: スレッドハンドル（外部定義）。

失敗モードと境界条件

  • **パイプ読み取りエラー**: ERROR_BROKEN_PIPE 以外のエラーが発生した場合、その旨をイベントログに記録します。
  • **バッファ制限**: 256バイトを超えるメッセージは、バッファ上限に達した時点で強制的にイベントログへ出力されます。
  • **レジストリ権限**: レジストリへの書き込み権限がない場合、設定更新はスキップされますが、イベントログ出力自体は継続を試みます。

保守担当者向けメモ

  • 本関数は stderr をパイプへ差し替えるため、他のモジュールが stderr を直接操作すると競合する可能性があります。
  • メッセージ定義には netmsg.dll を流用しており、イベントID 3299 を使用しています。
  • stderr_ready イベントによる同期は、サービス起動時の初期化順序を保証するために不可欠です。
