mapped_request_details

定義

  • 種別: struct
  • 定義箇所: modules/lua/mod_lua.h:156

目的

HTTP リクエストに関連付けられた Lua スクリプトの実行関数名と、その実行に必要な Lua VM の仕様（ap_lua_vm_spec）を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
    const char *function_name;
    ap_lua_vm_spec *spec;
} mapped_request_details;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。

  • function_name: 実行対象となる Lua 関数名（文字列）。
  • spec: VM の生成やスコープを定義する ap_lua_vm_spec 構造体へのポインタ。

前提条件と不変条件

  • function_name は、Lua VM 内で解決可能な関数名である必要があります。
  • spec は、有効な ap_lua_vm_spec 構造体を指している必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータの格納のみを行います。mod_lua のリクエスト処理フローにおいて、特定のハンドラやフックがどの Lua 関数をどの VM 設定で実行すべきかを決定する際のコンテキスト情報として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/lua/mod_lua.c:1996 にて、設定構造体内の本構造体ポインタを NULL に初期化する処理が確認できます。
  • 呼び出し先: 本構造体から直接呼び出される関数や、本構造体を引数に取る関数の詳細は、提供されたソース excerpt には含まれていません。

関連データ構造

  • ap_lua_vm_spec: VM の生成パラメータを定義する構造体。
  • ap_lua_request_cfg: 本構造体をメンバとして保持する可能性があるリクエスト設定構造体（modules/lua/mod_lua.h:164 に参照あり）。

失敗モードと境界条件

  • spec が NULL の場合、Lua VM の準備ができないため、スクリプトの実行に失敗する可能性があります。
  • function_name が NULL または空文字列の場合、Lua VM 側で関数呼び出しが失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_lua の設定管理において、リクエストと Lua スクリプトを紐付ける重要な役割を担います。
  • modules/lua/mod_lua.c 内での初期化処理以外の具体的な利用箇所については、引き続きソースコードの追跡が必要です。
