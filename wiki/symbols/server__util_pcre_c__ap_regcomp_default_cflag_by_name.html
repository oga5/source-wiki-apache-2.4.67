ap_regcomp_default_cflag_by_name

定義

  • 種別: function
  • 定義箇所: server/util_pcre.c:173

目的

正規表現のコンパイルフラグ名を文字列から対応する内部整数値（AP_REG_* 定数）へ変換します。

シグネチャまたは形状

int ap_regcomp_default_cflag_by_name(const char *name)

入力と出力

  • **入力**: name (文字列) - 変換対象のフラグ名（例: "ICASE"）。
  • **出力**: int - 対応する AP_REG_* フラグ値。一致するものがない場合は 0 を返します。

前提条件と不変条件

  • name は大文字小文字を区別せずに比較されます。
  • 戻り値の 0 は、指定された名前が既知のフラグ名と一致しなかったことを示します。

アルゴリズムまたは振る舞い

1. ap_cstr_casecmp を使用して、入力された name を以下の既知のフラグ名と比較します。

  • "ICASE" -> AP_REG_ICASE
  • "DOTALL" -> AP_REG_DOTALL
  • "DOLLAR_ENDONLY" -> AP_REG_DOLLAR_ENDONLY
  • "EXTENDED" -> AP_REG_EXTENDED

    • 2. 一致した場合は対応する定数を返し、いずれにも一致しない場合は初期値の 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:3021 で呼び出されていることが確認されています。
  • **呼び出し先**: ap_cstr_casecmp（文字列比較ユーティリティ）を呼び出します。

関連データ構造

  • include/ap_regex.h に定義されている AP_REG_* 定数群。

失敗モードと境界条件

  • 未知の文字列が渡された場合、エラーを発生させるのではなく 0 を返します。
  • name が NULL の場合の挙動については、ソースコード上では明示的なガードがありません。

保守担当者向けメモ

  • 新しい正規表現フラグをサポートする場合は、本関数の if-else ブロックに条件を追加してください。
  • ap_cstr_casecmp を使用しているため、フラグ名の大文字小文字は無視されます。
