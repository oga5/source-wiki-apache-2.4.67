generate_log_id

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_unique_id.c:276

目的

エラーログ生成時にリクエストを一意に識別するためのIDを取得または生成します。

シグネチャまたは形状

static int generate_log_id(const conn_rec *c, const request_rec *r, const char **id)

入力と出力

  • **入力**:
  • c: 接続情報（本関数内では使用されません）。
  • r: リクエスト情報。
  • id: 生成されたID文字列を格納するためのポインタのポインタ。
  • **出力**:
  • OK: IDの取得または生成に成功した場合。
  • DECLINED: r が NULL の場合。

前提条件と不変条件

  • r が NULL の場合、処理は即座に終了します。
  • UNIQUE_ID が subprocess_env に既に存在する場合はその値が優先されます。

アルゴリズムまたは振る舞い

1. r が NULL の場合、DECLINED を返します。 2. r->subprocess_env から UNIQUE_ID を検索し、存在すればそれを *id に代入します。 3. UNIQUE_ID が見つからない場合、gen_unique_id(r) を呼び出して新しいIDを生成し、*id に代入します。 4. OK を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/log.c 内の generate_log_id フック経由で呼び出されます。
  • **呼び出し先**:
  • gen_unique_id(): IDが未設定の場合に新規生成のために呼び出されます。
  • apr_table_get(): 環境変数からのID取得に使用されます。

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。subprocess_env テーブルを介してIDを管理します。

失敗モードと境界条件

  • r が NULL の場合、IDは生成されず DECLINED が返されます。
  • gen_unique_id() が失敗した場合の挙動については、本関数のソースコード内では明示的なエラーハンドリングは行われていません。

保守担当者向けメモ

  • 本関数は ap_hook_generate_log_id を通じて登録されています。
  • set_unique_id() との実行順序が前後する可能性があるため、既存のIDの再利用を優先する設計となっています。
  • 内部リダイレクト時のID維持に関する特別な処理が必要かどうかが検討事項として挙げられています。
