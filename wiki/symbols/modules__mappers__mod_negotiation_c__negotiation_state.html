negotiation_state

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_negotiation.c:246

目的

HTTP コンテンツネゴシエーション処理において、単一のリクエストに関連する状態、設定、および利用可能なバリアント情報を保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    apr_pool_t *pool;
    request_rec *r;
    neg_dir_config *conf;
    char *dir_name;
    int accept_q;
    float default_lang_quality;
    apr_array_header_t *accepts;
    apr_array_header_t *accept_encodings;
    apr_array_header_t *accept_charsets;
    apr_array_header_t *accept_langs;
    apr_array_header_t *avail_vars;
    int count_multiviews_variants;
    int is_transparent;
    int dont_fiddle_headers;
    int ua_supports_trans;
    int send_alternates;
    int may_choose;
    int use_rvsa;
} negotiation_state;

入力と出力

  • **入力**: request_rec を通じてリクエストヘッダー（Accept-* 等）およびディレクトリ設定を受け取ります。
  • **出力**: 選択されたバリアントの評価結果や、レスポンスヘッダー（Vary, Alternates 等）の生成に必要な情報を保持します。

前提条件と不変条件

  • r->pool からメモリが割り当てられる必要があります。
  • accepts, accept_encodings, accept_charsets, accept_langs の各配列ポインタは、対応するヘッダーが存在しない場合は NULL となります。
  • バリアントの MIME タイプや言語タグは、解析時に小文字に正規化されていることが前提です。

アルゴリズムまたは振る舞い

1. **初期化**: parse_accept_headers によりリクエストヘッダーから構築されます。 2. **収集**: read_type_map または read_types_multi を通じて、利用可能なバリアントが avail_vars に格納されます。 3. **評価**: set_language_quality や set_accept_quality 等の関数により、各バリアントの品質値が計算・更新されます。 4. **決定**: 最終的に best_match 等のアルゴリズムで最適なバリアントが選択されます。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_accept_headers, parse_negotiate_header, maybe_add_default_accepts, read_type_map, read_types_multi, set_default_lang_quality, set_language_quality, find_content_length, set_accept_quality 等の関数で引数として使用されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • neg_dir_config: ディレクトリ単位の設定。
  • accept_rec: 各 Accept ヘッダーの要素。
  • var_rec: サーバー側で利用可能な各バリアントの属性。

失敗モードと境界条件

  • type-map ファイルの読み込み失敗や、ディレクトリ内のバリアント列挙失敗時に、配列が空または不完全な状態になる可能性があります。
  • ForceLanguagePriority 設定がある場合、標準的な品質値計算ロジックが上書きされます。

保守担当者向けメモ

  • 本構造体は request_rec のライフサイクルに依存します。
  • is_transparent や use_rvsa などのフラグは、RFC 2296 準拠のネゴシエーションを行うかどうかの制御に使用されます。
  • count_multiviews_variants は MultiViews 処理時にディスクから検出されたバリアント数を追跡します。
