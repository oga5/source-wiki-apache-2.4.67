prefork_check_config

定義

  • 種別: function
  • 定義箇所: server/mpm/prefork/prefork.c:1345

目的

Prefork MPM の設定値（ServerLimit, MaxRequestWorkers, StartServers, MinSpareServers）を検証し、コンパイル時の制限や論理的な整合性に基づいて値を補正する。

シグネチャまたは形状

static int prefork_check_config(apr_pool_t *p, apr_pool_t *plog,
                                apr_pool_t *ptemp, server_rec *s);

入力と出力

  • **入力**:
  • p, plog, ptemp: 設定用メモリプール。
  • s: 現在のサーバー設定レコード。
  • **出力**: 常に OK を返す。

前提条件と不変条件

  • retained->mpm->module_loads が 1 の場合、初回起動（startup）とみなされる。
  • ServerLimit は再起動を跨いで変更できない（retained->first_server_limit に固定される）。

アルゴリズムまたは振る舞い

1. **ServerLimit の検証**:

  • MAX_SERVER_LIMIT を超える場合は上限値に切り詰め、1 未満の場合は 1 に引き上げる。
  • 再起動時に retained->first_server_limit と異なる値が指定された場合、警告を出して元の値に戻す。

    • 2. **MaxRequestWorkers (ap_daemons_limit) の検証**:

  • ServerLimit を超える場合は ServerLimit に切り詰め、1 未満の場合は 1 に引き上げる。

    • 3. **StartServers (ap_daemons_to_start) の検証**:

  • 1 未満の場合は 1 に引き上げる。

    • 4. **MinSpareServers (ap_daemons_min_free) の検証**:

  • 1 未満の場合は 1 に引き上げる。

    • 5. **ログ出力**: 各補正処理において、初回起動時は APLOG_STARTUP フラグ付きでコンソールへ、それ以外はエラーログへ警告を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_check_config により登録され、設定チェックフェーズで呼び出される。
  • **呼び出し先**: ap_log_error（ログ出力用）。

関連データ構造

  • retained: prefork_retained_data 型。再起動を跨ぐ設定状態（first_server_limit 等）を保持。
  • グローバル変数: server_limit, ap_daemons_limit, ap_daemons_to_start, ap_daemons_min_free。

失敗モードと境界条件

  • 設定値が許容範囲外の場合、プログラムを停止させるのではなく、警告を出力した上で安全な最小値または最大値に自動補正する。
  • 再起動時の ServerLimit 変更試行は無視され、初回起動時の値が強制される。

保守担当者向けメモ

  • 本関数は設定の整合性を保つためのガードレールとして機能する。
  • ap_daemons_to_start > ap_daemons_limit や ap_daemons_max_free < ap_daemons_min_free + 1 といった複雑な依存関係のチェックは、この関数ではなく prefork_run() 内で行われる。
