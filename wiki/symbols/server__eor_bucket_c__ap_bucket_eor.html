ap_bucket_eor

定義

  • 種別: struct
  • 定義箇所: server/eor_bucket.c:22

目的

Apache HTTP Server における「End of Request (EOR)」バケットのデータ構造を定義します。このバケットは、フィルタチェーンを通じてリクエストの終了を通知し、リソースのクリーンアップやログ記録をトリガーするためのメタデータとして機能します。

シグネチャまたは形状

typedef struct {
    apr_bucket_refcount refcount;
    request_rec *data;
} ap_bucket_eor;

入力と出力

  • **入力**: request_rec *data は、このバケットが関連付けられたリクエストの情報を保持します。
  • **出力**: 本構造体自体はデータ転送を行わず、バケットの破棄時に request_rec に関連付けられたプールを解放し、ログ記録や統計更新をトリガーします。

前提条件と不変条件

  • refcount はバケットの参照カウントを管理します。
  • data は有効な request_rec 構造体を指す必要があります。
  • 本バケットはデータ転送用ではないため、読み込み操作（eor_bucket_read）は常に空を返すように設計されています。

アルゴリズムまたは振る舞い

1. **生成**: ap_bucket_eor_create により生成され、リクエストプールに対して eor_bucket_cleanup が登録されます。 2. **伝播**: フィルタチェーンを通過し、リクエストの終了地点まで到達します。 3. **破棄**: バケットが破棄される際、eor_bucket_destroy が呼び出されます。これによりリクエストプールが破棄され、登録済みの eor_bucket_cleanup が実行されます。 4. **終了処理**: eor_bucket_cleanup 内で ap_run_log_transaction が実行され、ログ出力とスコアボードの統計更新が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/eor_bucket.c 内の関数（ap_bucket_eor_create, eor_bucket_destroy, eor_bucket_read 等）から参照されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_bucket_refcount: バケットのライフサイクル管理用。
  • request_rec: リクエストの状態を保持する構造体。

失敗モードと境界条件

  • **二重解放**: eor_bucket_cleanup 内で *rp = NULL とすることで、バケット破棄とプール破棄の順序が前後しても安全に動作するよう設計されています。

保守担当者向けメモ

  • 本バケットはメタデータ専用です。eor_bucket_read が空を返す仕様を変更しないでください。
  • apr_pool_pre_cleanup_register を使用しているのは、リクエスト終了時にもモジュールがサブプールを必要とする可能性があるためです。
