set_dbmfile

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_express.c:35

目的

Apache HTTP Server の設定ファイルで指定された DBM ファイルのパスを読み込み、サーバー設定構造体 express_server_conf に格納する。

シグネチャまたは形状

static const char *set_dbmfile(cmd_parms *cmd, void *dconf, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dconf: ディレクトリ設定（本関数では未使用）。
  • arg: 設定ファイルから渡される DBM ファイルのパス文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • cmd->server->module_config から express_server_conf が取得可能であること。
  • ap_server_root_relative により、パスがサーバーのルートディレクトリからの相対パスとして解決される。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定構造体 sconf を取得する。 2. ap_server_root_relative を呼び出し、引数 arg を絶対パスに変換して sconf->dbmfile に代入する。 3. パスの解決に失敗した場合（NULL が返された場合）、エラーメッセージを生成して返す。 4. 正常終了時は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_express.c:226 にて AP_INIT_TAKE1 マクロを通じて ProxyExpressDBMFile ディレクティブのハンドラとして登録されている。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core)
  • ap_server_root_relative (Apache Core)
  • apr_pstrcat (APR)

関連データ構造

  • express_server_conf: サーバーごとの設定を保持する構造体。本関数はこの中の dbmfile メンバを更新する。

失敗モードと境界条件

  • ap_server_root_relative がパスの解決に失敗した場合、エラーメッセージを返し設定処理を中断する。

保守担当者向けメモ

  • 本関数は ProxyExpressDBMFile ディレクティブ専用のハンドラである。
  • パス解決には cmd->pool が使用されており、メモリ管理は Apache のリクエスト/設定ライフサイクルに依存している。
