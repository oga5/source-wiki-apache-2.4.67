md_store_move

定義

  • 種別: function
  • 定義箇所: modules/md/md_store.c:120

目的

Managed Domain (MD) ストレージ内の特定のデータエントリを、あるストレージグループから別のグループへ移動（またはアーカイブ）します。

シグネチャまたは形状

apr_status_t md_store_move(md_store_t *store, apr_pool_t *p, 
                           md_store_group_t from, md_store_group_t to,
                           const char *name, int archive)

入力と出力

  • store: ストレージ抽象化レイヤーのハンドル。
  • p: メモリ割り当てに使用するプール。
  • from: 移動元のストレージグループ。
  • to: 移動先のストレージグループ。
  • name: 移動対象のドメイン名。
  • archive: アーカイブフラグ（非ゼロの場合、アーカイブ処理として実行）。
  • 戻り値: 操作の成功または失敗を示す apr_status_t。

前提条件と不変条件

  • store は有効な md_store_t 構造体へのポインタであり、その move 関数ポインタが初期化されている必要があります。

アルゴリズムまたは振る舞い

  • 引数として渡された md_store_t 構造体の関数ポインタ store->move を呼び出し、実際の移動処理を具体的なストレージ実装（md_store_fs.c 等）へ委譲します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/md/md_reg.c:1029
  • modules/md/md_reg.c:1299
  • 呼び出し先:
  • store->move（md_store_t 構造体内の関数ポインタ）

関連データ構造

  • md_store_t: ストレージ操作のインターフェースを定義する構造体。
  • md_store_group_t: ストレージグループ（MD_SG_DOMAINS, MD_SG_TMP, MD_SG_ARCHIVE 等）を識別する列挙型。

失敗モードと境界条件

  • 基盤となるストレージ実装（md_store_fs 等）がエラーを返した場合、そのエラーコードがそのまま返されます。
  • ファイルシステム上の権限不足や、移動元エントリの不在などが失敗の原因となります。

保守担当者向けメモ

  • 本関数は抽象化レイヤーであり、実際のファイル移動やディレクトリ操作は md_store_t の実装側に依存します。
  • 呼び出し元で archive フラグを適切に設定することで、古い証明書等の履歴管理を制御しています。
