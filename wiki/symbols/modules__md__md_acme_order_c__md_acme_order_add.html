md_acme_order_add

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:148

目的

ACME Order オブジェクトに、指定された認証（Authz）URL を追加します。

シグネチャまたは形状

apr_status_t md_acme_order_add(md_acme_order_t *order, const char *authz_url)

入力と出力

  • **入力**:
  • order: 認証 URL を追加する対象の md_acme_order_t 構造体へのポインタ。
  • authz_url: 追加対象の認証 URL 文字列。
  • **出力**:
  • APR_SUCCESS: 正常終了。

前提条件と不変条件

  • authz_url は NULL であってはなりません（assert による検証あり）。
  • order->authz_urls は、apr_array_header_t として初期化されている必要があります。

アルゴリズムまたは振る舞い

1. authz_url が既に order->authz_urls 配列内に存在するかを md_array_str_index で確認します。 2. 配列内に存在しない場合のみ、order->p（メモリプール）を使用して authz_url を複製し、APR_ARRAY_PUSH を用いて配列の末尾に追加します。 3. 既に存在する場合は何もしません。 4. 常に APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内で直接的な呼び出し元は特定されていません。
  • **呼び出し先**:
  • assert()
  • md_array_str_index()
  • apr_pstrdup()
  • APR_ARRAY_PUSH()

関連データ構造

  • md_acme_order_t: ACME Order の状態を保持する構造体。本関数はこの構造体内の authz_urls 配列を操作します。

失敗モードと境界条件

  • authz_url が NULL の場合、プログラムはアサート失敗により終了します。
  • メモリ不足が発生した場合、apr_pstrdup 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は重複排除を行うため、同一の認証 URL を複数回追加しようとしても配列は肥大化しません。
  • order->p プールを使用しているため、追加された文字列の生存期間は order オブジェクトの生存期間と同期します。
