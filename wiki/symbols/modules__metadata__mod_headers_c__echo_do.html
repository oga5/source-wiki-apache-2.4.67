echo_do

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_headers.c:139

目的

Header ディレクティブの echo アクションにおいて、リクエストヘッダーを反復処理するためのコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct {
    request_rec *r;
    header_entry *hdr;
} echo_do;

入力と出力

  • **入力**: request_rec 構造体へのポインタ（処理対象のリクエスト）および header_entry 構造体へのポインタ（適用するヘッダー操作ルール）。
  • **出力**: なし（構造体定義のため）。

前提条件と不変条件

  • r は有効なリクエストコンテキストを指している必要があります。
  • hdr は適用対象となるヘッダー操作ルールを指している必要があります。

アルゴリズムまたは振る舞い

本構造体は、Header echo アクション実行時に、リクエストヘッダーのテーブルを走査する際の引数として使用されます。mod_headers.c 内の処理において、この構造体を介して現在のリクエスト情報とヘッダー操作ルールが渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_headers.c 内の Header echo アクションを処理する関数群（echo_do v; としてスタック上に生成される）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すものではありません。

関連データ構造

  • request_rec: Apache のリクエスト情報を保持するコア構造体。
  • header_entry: ヘッダー操作のルール（アクション、対象ヘッダー名など）を定義する構造体。

失敗モードと境界条件

  • r または hdr が NULL の状態で参照された場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は Header echo アクション専用の補助データ構造です。
  • mod_headers.c の 666 行目付近で、void *v から echo_do *ed へのキャストが行われており、コールバック関数等での受け渡しに使用されています。
