ap_version_t

定義

  • 種別: struct
  • 定義箇所: include/httpd.h:413

目的

Apache HTTP Server のバージョン情報を保持するためのデータ構造です。メジャー、マイナー、パッチ番号および追加の文字列情報を格納します。

シグネチャまたは形状

typedef struct {
    int major;              /* メジャー番号 */
    int minor;              /* マイナー番号 */
    int patch;              /* パッチ番号 */
    const char *add_string; /* "-dev" のような追加文字列 */
} ap_version_t;

入力と出力

  • 本構造体は、ap_get_server_revision(ap_version_t *version) 関数への出力引数として使用されます。

前提条件と不変条件

  • add_string は定数文字列を指すか、NULL である必要があります。
  • 各数値フィールドは、サーバーのリリースバージョンを反映した整数値です。

アルゴリズムまたは振る舞い

  • ap_get_server_revision 関数を呼び出すことで、実行中のサーバーのバージョン情報が本構造体に書き込まれます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_version.c（httpd_version 変数として保持）
  • **呼び出し先**: server/core.c において ap_get_server_revision が実装されています。

関連データ構造

  • なし。

失敗モードと境界条件

  • ap_get_server_revision に NULL ポインタを渡した場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 本構造体は include/ap_mmn.h（Module Magic Number）の定義と密接に関連しており、サーバーの API バージョン管理に使用されます。変更時は互換性に注意してください。
