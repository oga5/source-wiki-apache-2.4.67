set_worker_param

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:96

目的

Apache HTTP Server のプロキシワーカー（proxy_worker）に対する設定パラメータを解析し、対応する構造体フィールドに値を適用する。

シグネチャまたは形状

static const char *set_worker_param(apr_pool_t *p,
                                    server_rec *s,
                                    proxy_worker *worker,
                                    const char *key,
                                    const char *val)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • s: サーバー設定レコード。
  • worker: 設定対象の proxy_worker 構造体。
  • key: 設定項目名（文字列）。
  • val: 設定値（文字列）。
  • **出力**: 成功時は NULL を返し、エラー時はエラー内容を示す文字列（const char *）を返す。

前提条件と不変条件

  • worker および worker->s が初期化済みであること。
  • key は大文字小文字を区別しない（strcasecmp を使用）。
  • 数値パラメータは、指定された範囲外の場合にエラーを返す。

アルゴリズムまたは振る舞い

1. key に基づき、対応する proxy_worker の内部構造体（worker->s）のフィールドを更新する。 2. loadfactor, retry, ttl, min, max, smax, acquire, timeout, iobuffersize, receivebuffersize, keepalive, disablereuse, enablereuse, addressttl, route, redirect, status, flushpackets, flushwait, ping, lbset, connectiontimeout, flusher, upgrade, responsefieldsize, secret の各パラメータを処理する。 3. 一部のパラメータ（acquire, ping, connectiontimeout など）は ap_timeout_parameter_parse を使用して単位付きの時間を解析する。 4. status パラメータは + または - の接頭辞を解析し、ap_proxy_set_wstatus を呼び出して状態を更新する。 5. 既知のパラメータに一致しない場合、set_worker_hc_param_f が定義されていればヘルスチェック用パラメータとして委譲し、そうでなければエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy.c:2165
  • modules/proxy/mod_proxy.c:2701
  • modules/proxy/mod_proxy.c:2801
  • modules/proxy/mod_proxy.c:2953
  • **呼び出し先**:
  • ap_timeout_parameter_parse
  • ap_proxy_set_wstatus
  • set_worker_hc_param_f (関数ポインタ経由)

関連データ構造

  • proxy_worker: プロキシワーカーの定義を保持する構造体。
  • server_rec: Apache のサーバー設定構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • 不正な数値や範囲外の値が渡された場合、エラーメッセージ文字列を返す。
  • PROXY_STRNCPY を使用するフィールド（route, redirect, flusher, upgrade, secret）で、値がバッファサイズを超える場合はエラーを返す。
  • status パラメータで未知のオプションが指定された場合、エラーを返す。

保守担当者向けメモ

  • 新しいワーカーパラメータを追加する場合は、この関数の if-else チェーンに新しい条件を追加する必要がある。
  • set_worker_hc_param_f は外部モジュール（mod_proxy_hcheck 等）によって設定される可能性のあるフックであるため、呼び出し時の存在確認が必須である。
