log_create_failure

定義

  • 種別: function
  • 定義箇所: server/util_mutex.c:387

目的

ミューテックスの生成に失敗した際、エラーログを出力する。

シグネチャまたは形状

static void log_create_failure(apr_status_t rv, server_rec *s, const char *type, const char *fname)

入力と出力

  • rv (apr_status_t): APR が返すエラーコード。
  • s (server_rec *): エラーログの出力先となるサーバーレコード。
  • type (const char *): 失敗したミューテックスの型名。
  • fname (const char *): ミューテックスに関連付けられたファイル名（NULL の場合もある）。
  • 戻り値: なし。

前提条件と不変条件

  • ap_log_error を使用するため、呼び出し時に server_rec が有効である必要がある。
  • fname が NULL の場合、ログメッセージにはファイル名に関する記述は含まれない。

アルゴリズムまたは振る舞い

1. ap_log_error を呼び出し、APLOG_EMERG レベルでエラーを記録する。 2. ログメッセージには、ミューテックスの型 (type) を含める。 3. fname が指定されている場合、(file <fname>) という形式でファイルパスをログに付加する。 4. APR のエラーコード rv をログに含め、具体的な失敗原因を記録する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/util_mutex.c:437
  • server/util_mutex.c:484
  • 呼び出し先:
  • ap_log_error (Apache HTTP Server のログ出力 API)

関連データ構造

  • apr_status_t: APR のステータスコード。
  • server_rec: サーバー設定情報を保持する構造体。

失敗モードと境界条件

  • fname が NULL の場合、三項演算子によりログ出力文字列からファイル名部分が除外される。
  • s が NULL の場合、ap_log_error の仕様に従い、ログ出力が適切に処理される必要がある。

保守担当者向けメモ

  • 本関数は server/util_mutex.c 内のミューテックス生成処理におけるエラーハンドリングを共通化するために使用されている。
  • ログメッセージのフォーマットを変更する場合は、APLOGNO(00023) の識別子を維持すること。
