readfile_heartbeats

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_heartbeat.c:92

目的

指定されたファイルパスからハートビート情報を読み込み、バックエンドサーバーの負荷状態（busy, ready, lastseen, port）を解析してハッシュテーブルに格納する。

シグネチャまたは形状

apr_status_t readfile_heartbeats(const char *path, apr_hash_t *servers, apr_pool_t *pool)

入力と出力

  • **入力**:
  • path: ハートビート情報が記述されたファイルパス。
  • servers: サーバー情報を格納するための apr_hash_t。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • path が NULL の場合は即座に APR_SUCCESS を返す。
  • ファイル内の各行は <IP> <query_string> の形式である必要がある。
  • # で始まる行はコメントとして無視される。
  • サーバーの seen 値が更新されない場合、初期値は -1 となる。

アルゴリズムまたは振る舞い

1. ファイルをバイナリ・バッファモードでオープンする。 2. apr_brigade を使用してファイル内容を読み込み、行単位（apr_brigade_split_line）で分割する。 3. 各行を解析し、IPアドレスをキーとして servers ハッシュから hb_server_t を取得または新規作成する。 4. argstr_to_table を用いてクエリ文字列を解析し、busy, ready, lastseen, port の各値をサーバー構造体に反映する。 5. **ヒューリスティック処理**: busy が 0 かつ ready が 0 でない場合、サーバーが起動直後であると判断し、過負荷を防ぐために ready 値を 1/4 に調整する。 6. 全行の処理完了後、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/balancers/mod_lbmethod_heartbeat.c:253（ソースコード内の直接的な参照）。
  • **呼び出し先**:
  • apr_file_open, apr_file_info_get, apr_brigade_split_line 等の APR 関数。
  • argstr_to_table: クエリ文字列の解析に使用。
  • apr_hash_get, apr_hash_set: サーバー情報の管理に使用。

関連データ構造

  • hb_server_t: 各バックエンドサーバーの負荷状態を保持する構造体。
  • apr_hash_t: サーバーリストを管理するハッシュテーブル。
  • apr_bucket_brigade: ファイル読み込み用の一時バッファ。

失敗モードと境界条件

  • ファイルが開けない場合、またはファイル情報の取得に失敗した場合は、そのエラーコードを返す。
  • 行が空の場合や、IPアドレスとクエリ文字列の区切り（スペース）が存在しない行はスキップされる。
  • bsize が 0 の場合は読み込み終了とみなす。

保守担当者向けメモ

  • 本関数はリクエスト毎に呼び出される可能性があるため、メモリ管理には pool を適切に使用している。
  • ready 値の 1/4 調整は、起動直後のサーバーへの急激な負荷集中を避けるための重要なヒューリスティックである。
  • ファイルフォーマットの変更や解析ロジックの拡張を行う際は、argstr_to_table の仕様と整合性をとること。
