merge_session_cookie_dir_config

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_cookie.c:179

目的

親コンテキストの設定と、現在のディレクトリコンテキストで追加された設定をマージし、新しい設定構造体を生成します。

シグネチャまたは形状

static void *merge_session_cookie_dir_config(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親コンテキストの設定構造体（session_cookie_dir_conf）。
  • addv: 現在のディレクトリコンテキストの設定構造体（session_cookie_dir_conf）。
  • **出力**:
  • マージされた新しい session_cookie_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は session_cookie_dir_conf 型としてキャスト可能であること。
  • 設定値の継承において、_set フラグが 0（未設定）である場合、親の設定が優先されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい session_cookie_dir_conf 構造体をプール p 上に確保します。 2. name、name_attrs、name2、name2_attrs、remove の各フィールドについて、addv 側で設定が行われていない（_set == 0）場合は basev の値を採用し、それ以外は addv の値を採用します。 3. name_set、name2_set、remove_set の各フラグは、basev または addv のいずれかが設定されていれば 1 となるよう論理和（OR）をとります。 4. マージ済みの新しい構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apacheの設定システム（modules/session/mod_session_cookie.c:278 にて、ディレクトリマージ用ハンドラとして登録）。
  • **呼び出し先**: apr_pcalloc（APRライブラリ）。

関連データ構造

  • session_cookie_dir_conf: ディレクトリ単位のセッション設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、Apacheの標準的なエラーハンドリングに従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は、Apacheのコンフィグマージフェーズで呼び出されます。新しい設定項目を session_cookie_dir_conf に追加した場合は、必ず本関数内でマージロジックを更新してください。
