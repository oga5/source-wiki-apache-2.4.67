saxctxt

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_proxy_html.c:120

目的

mod_proxy_html フィルタの実行時コンテキストを保持し、HTML 解析から置換処理に至る一連のストリーム処理状態を管理する。

シグネチャまたは形状

typedef struct {
    ap_filter_t *f;             /* フィルタ情報 */
    proxy_html_conf *cfg;       /* モジュール設定 */
    htmlParserCtxtPtr parser;   /* libxml2 パーサコンテキスト */
    apr_bucket_brigade *bb;     /* 出力用バケットブリゲード */
    char *buf;                  /* 処理用バッファ */
    size_t offset;              /* バッファ内の現在位置 */
    size_t avail;               /* バッファの空き容量 */
    const char *encoding;       /* 文字エンコーディング */
    urlmap *map;                /* 適用される置換ルール */
    char rbuf[4];               /* マルチバイト文字処理用一時バッファ */
    apr_size_t rlen;            /* rbuf の使用長 */
    apr_size_t rmin;            /* rbuf の最小保持長 */
} saxctxt;

入力と出力

  • **入力**: ap_filter_t を通じて受け取るバケットブリゲード内の HTML データ。
  • **出力**: bb (バケットブリゲード) を介して後続のフィルタへ渡される、置換済みの HTML データ。

前提条件と不変条件

  • saxctxt 内のバッファは apr_pool_t に紐付いており、リクエスト終了時に自動的に解放される。
  • realloc を伴う操作を行う場合は、apr_pool_cleanup_register によるクリーンアップ登録が必須である。

アルゴリズムまたは振る舞い

1. check_filter_init により初期化され、リクエストごとにコンテキストが生成される。 2. libxml2 の SAX コールバック（pstartElement 等）が発火する際、ctxt 引数から saxctxt が復元され、現在の解析状態や置換ルールを参照する。 3. dump_content を通じて、バッファ内のテキストに対して urlmap に基づく置換処理が実行される。 4. EOS (End of Stream) バケット受信時に、パーサの解放とバッファのフラッシュが行われる。

呼び出し元と呼び出し先

  • **呼び出し元**: check_filter_init (生成元)、および pstartElement, pcharacters, pcomment, pcdata などの SAX コールバック関数。
  • **呼び出し先**: fixup_rules (ルール適用時)、preserve, pappend, dump_content (バッファ操作時)。

関連データ構造

  • proxy_html_conf: モジュール全体の設定を保持。
  • urlmap: 置換ルールの定義。
  • ap_filter_t: Apache フィルタチェーンの制御構造。
  • htmlParserCtxtPtr: libxml2 のパーサ状態。

失敗モードと境界条件

  • **メモリ消費**: 大規模な HTML 処理時に ProxyHTMLBufSize を超えるメモリ消費が発生する可能性がある。
  • **解析不完全**: libxml2 の特性上、HTML の断片処理においてタグが正しく閉じられない場合がある。
  • **マルチバイト文字**: rbuf を使用してマルチバイト文字の分断を回避するが、エンコーディングの不一致により処理が失敗する場合がある。

保守担当者向けメモ

  • saxctxt はリクエスト単位のライフサイクルを持つ。メモリリークを防ぐため、apr_pool_t の管理ポリシーを遵守すること。
  • libxml2 のパーサ状態遷移は複雑であるため、SAX コールバック内での saxctxt の操作は慎重に行うこと。
