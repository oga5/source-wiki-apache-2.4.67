ap_relieve_child_processes

定義

  • 種別: function
  • 定義箇所: server/mpm_unix.c:286

目的

終了処理中の子プロセスを非同期に回収し、各プロセスの終了状態をコールバック関数を通じて通知します。

シグネチャまたは形状

void ap_relieve_child_processes(ap_reclaim_callback_fn_t *mpm_callback)

入力と出力

  • **入力**: mpm_callback - プロセス回収時に呼び出されるコールバック関数へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • ap_mpm_query(AP_MPMQ_MAX_DAEMON_USED, ...) により取得された最大プロセス数に基づき動作します。
  • スコアボード上のプロセスおよび extras リストに登録されたプロセスを対象とします。

アルゴリズムまたは振る舞い

1. AP_MPMQ_MAX_DAEMON_USED を取得し、スコアボード内の全プロセスを走査します。 2. reclaim_one_pid(pid, DO_NOTHING) を呼び出し、プロセスが終了していれば mpm_callback を実行します。 3. 続いて extras リストを走査し、同様に reclaim_one_pid で終了を確認します。 4. extras 内のプロセスが終了している場合、ap_unregister_extra_mpm_process を呼び出してリストから削除し、コールバックを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/prefork/prefork.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • ap_mpm_query
  • ap_get_scoreboard_process
  • reclaim_one_pid
  • ap_unregister_extra_mpm_process
  • 引数として渡された mpm_callback

関連データ構造

  • process_score: スコアボード内のプロセス状態管理構造体。
  • extra_process_t: 追加のMPMプロセスを管理するリンクリスト構造体。

失敗モードと境界条件

  • pid == 0 のスコアボードエントリはスキップされます。
  • ap_unregister_extra_mpm_process が失敗した場合（戻り値が1以外）、AP_DEBUG_ASSERT(1 == 0) によりアサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は非同期回収を行うため、呼び出し側はプロセスが完全に終了するまで複数回呼び出す必要があります。
  • extras リストの走査中に ap_unregister_extra_mpm_process を呼び出すと、現在の cur_extra ポインタが指すメモリ領域が無効になる可能性があるため、ループ内でのポインタ管理に注意してください。
