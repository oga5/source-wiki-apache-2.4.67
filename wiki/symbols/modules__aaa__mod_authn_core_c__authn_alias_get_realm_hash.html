authn_alias_get_realm_hash

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_core.c:140

目的

認証プロバイダーのエイリアス（<AuthnProviderAlias>）経由で、レルムハッシュの取得を代行する。

シグネチャまたは形状

static authn_status authn_alias_get_realm_hash(request_rec *r, const char *user,
                                               const char *realm, char **rethash)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。r->notes からプロバイダー名を取得し、r->per_dir_config を一時的に差し替える。
  • user: 認証対象のユーザー名。
  • realm: 認証レルム名。
  • **出力**:
  • rethash: 取得されたハッシュ値が格納されるポインタ。
  • 戻り値: authn_status 型の認証結果（成功時は AUTH_GRANTED 等、失敗時は AUTH_USER_NOT_FOUND 等）。

前提条件と不変条件

  • r->notes に AUTHN_PROVIDER_NAME_NOTE キーでプロバイダー名が設定されている必要がある。
  • サーバー設定 authn_alias_srv_conf に該当するエイリアスが登録されている必要がある。

アルゴリズムまたは振る舞い

1. r->notes からプロバイダー名を取得し、サーバー設定から対応する provider_alias_rec を検索する。 2. エイリアスが見つかった場合、現在の r->per_dir_config を保存する。 3. ap_merge_per_dir_configs を使用して、リクエストの設定をエイリアス内の設定（prvdraliasrec->sec_auth）で一時的に上書きする。 4. エイリアスが参照する本来のプロバイダーの get_realm_hash 関数を呼び出す。 5. 処理終了後、r->per_dir_config を元の設定に復元する。 6. プロバイダーの実行結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_core.c 内の認証処理フローから参照される（直接の呼び出し箇所はソース抜粋外）。
  • **呼び出し先**: prvdraliasrec->provider->get_realm_hash（エイリアス先の実際の認証プロバイダー）。

関連データ構造

  • authn_alias_srv_conf: エイリアスを管理するサーバー設定構造体。
  • provider_alias_rec: エイリアスされたプロバイダーの定義と、そのエイリアスが保持するディレクトリ設定。
  • request_rec: リクエストごとの状態管理に使用。

失敗モードと境界条件

  • provider_name が r->notes に存在しない場合、またはエイリアスが未登録の場合は、何も実行せず AUTH_USER_NOT_FOUND を返す。
  • プロバイダー関数が呼び出せない場合、または設定の差し替えに失敗した場合の挙動は、呼び出し先のプロバイダー実装に依存する。

保守担当者向けメモ

  • 本関数は r->per_dir_config を一時的に書き換えるため、呼び出し先でエラーが発生しても確実に復元されるよう注意が必要である。
  • ap_merge_per_dir_configs は r->pool を使用してメモリを確保するため、リクエスト終了時に自動的に解放される。
