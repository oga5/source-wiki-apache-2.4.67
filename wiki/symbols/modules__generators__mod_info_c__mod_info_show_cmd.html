mod_info_show_cmd

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_info.c:183

目的

Apache HTTP Server の設定ディレクティブ情報を、インデントを伴う形式で出力（HTML または標準出力）します。

シグネチャまたは形状

static void mod_info_show_cmd(request_rec *r, const ap_directive_t *dir, int nest)

入力と出力

  • **入力**:
  • r: リクエストレコード（NULL の場合、標準出力へ出力）。
  • dir: 出力対象のディレクティブ構造体。
  • nest: インデントレベルを示す整数。
  • **出力**: なし（副作用として HTML またはファイルストリームへの書き込みを行う）。

前提条件と不変条件

  • dir は有効な ap_directive_t 構造体を指している必要があります。
  • r が NULL の場合、グローバル変数 out が有効なファイルハンドルとして初期化されている必要があります。

アルゴリズムまたは振る舞い

1. mod_info_indent を呼び出し、指定されたネストレベル (nest) に基づくインデントと、ディレクティブの定義元ファイル名・行番号を出力します。 2. r が有効な場合:

  • ap_escape_html を使用してディレクティブ名と引数をエスケープし、HTML 形式でレスポンスへ書き込みます。

    • 3. r が NULL の場合:

  • apr_file_printf を使用し、標準出力（out）へプレーンテキスト形式でディレクティブ名と引数を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_info.c:267
  • **呼び出し先**:
  • mod_info_indent
  • ap_escape_html (r が有効な場合)
  • ap_rprintf (r が有効な場合)
  • apr_file_printf (r が NULL の場合)

関連データ構造

  • request_rec: Apache のリクエスト情報構造体。
  • ap_directive_t: 設定ツリー内のディレクティブ情報を保持する構造体。

失敗モードと境界条件

  • dir が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • r が NULL かつ out が初期化されていない場合、書き込みエラーが発生します。

保守担当者向けメモ

  • 本関数は mod_info の表示ロジックにおいて、設定ツリーの末端ノードを表示する役割を担います。
  • r の有無による出力先の切り替えは、リクエスト処理時と -DDUMP_CONFIG によるコマンドライン出力の両方をサポートするための設計です。
