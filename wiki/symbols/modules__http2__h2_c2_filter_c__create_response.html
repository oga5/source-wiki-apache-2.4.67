create_response

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2_filter.c:244

目的

HTTP/1.1 形式のレスポンス情報を HTTP/2 ストリーム用の h2_headers 構造体に変換し、ヘッダーの正規化およびプロトコル適合処理を行う。

シグネチャまたは形状

h2_headers *create_response(request_rec *r)

入力と出力

  • **入力**: request_rec *r (処理対象のリクエストレコード)
  • **出力**: h2_headers * (HTTP/2 ヘッダーバケットとして送信可能な構造体)

前提条件と不変条件

  • r->pool が有効であり、メモリ割り当てに使用可能であること。
  • r->headers_out および r->err_headers_out が適切に設定されていること。

アルゴリズムまたは振る舞い

1. **ヘッダー統合**: err_headers_out を headers_out に統合し、err_headers_out をクリアする。 2. **Vary ヘッダー処理**: force-no-vary が設定されている場合は Vary を削除し、プロトコルを HTTP/1.0 に強制する。それ以外は fix_vary(r) を呼び出す。 3. **ETag 制御**: no-etag ノートが存在する場合、ETag ヘッダーを削除する。 4. **プロトコル設定**: ap_set_keepalive(r) を呼び出し、Keep-Alive 状態を決定する。 5. **コンテンツ制御**:

  • ヘッダーのみのレスポンスの場合、Transfer-Encoding や Content-Length をクリアする。
  • チャンク化されている場合、Transfer-Encoding: chunked を追加する。

    • 6. **メタデータ付与**: Content-Type、Content-Encoding、Content-Language をヘッダーテーブルに反映する。 7. **キャッシュ制御**: no_cache が設定されている場合、Expires ヘッダーを生成する。 8. **プロキシ/サーバー情報**: 必要に応じて Date および Server ヘッダーを補完する。 9. **変換**: h2_headers_rcreate を呼び出し、最終的な h2_headers を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c2_filter.c:739 (直接的な呼び出しを確認)
  • **呼び出し先**:
  • apr_table_overlay, apr_table_clear, apr_table_get, apr_table_unset, apr_table_setn, apr_table_mergen, apr_table_add (APR テーブル操作)
  • fix_vary (関連関数)
  • ap_set_keepalive, ap_make_content_type, ap_get_list_item, ap_recent_rfc822_date, ap_get_server_banner (Apache HTTP Server API)
  • h2_headers_rcreate (HTTP/2 ヘッダー生成)

関連データ構造

  • request_rec: Apache のリクエストコンテキスト。
  • h2_headers: HTTP/2 ストリーム用のヘッダー表現。

失敗モードと境界条件

  • r->header_only かつ Content-Length が "0" の場合、誤ったゼロ長ヘッダーを送信しないよう Content-Length を削除する回避策が含まれる。
  • Content-Language の重複排除処理において、配列操作を行う。

保守担当者向けメモ

  • 本関数は HTTP/1.1 のレガシーなヘッダー処理を HTTP/2 に適合させるための翻訳レイヤーとして機能する。
  • AP_STATUS_IS_HEADER_ONLY や r->chunked の判定ロジックは、HTTP/2 フィルタスタックにおけるレスポンスの正当性を担保する重要な箇所である。
