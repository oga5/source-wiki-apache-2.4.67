mk_password_hash

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_passwd.c:87

目的

passwd_ctx 構造体で指定されたアルゴリズムに基づき、平文パスワードからハッシュ値を生成し、結果をバッファに格納する。

シグネチャまたは形状

int mk_password_hash(passwd_ctx *ctx);

入力と出力

  • **入力**: passwd_ctx *ctx（パスワード、アルゴリズム種別、出力バッファ、メモリプール等を含む構造体）。
  • **出力**: int 型のステータスコード（成功時は 0、失敗時はエラーコード）。
  • **副作用**: ctx->out へのハッシュ値書き込み、ctx->errstr へのエラーメッセージ設定、および処理後の平文パスワードのメモリゼロクリア。

前提条件と不変条件

  • ctx->passwd に有効な平文パスワード文字列が格納されていること。
  • ctx->out にハッシュ値を格納するための十分なメモリ領域が確保されていること。
  • 処理終了時には、必ず memset により平文パスワードがメモリ上から消去される。

アルゴリズムまたは振る舞い

1. ctx->alg に基づき、以下のいずれかのアルゴリズムを選択する。

  • ALG_APSHA: apr_sha1_base64 を使用。
  • ALG_APMD5: generate_salt でソルトを生成後、apr_md5_encode を使用。
  • ALG_CRYPT: generate_salt でソルトを生成後、crypt を使用（8文字を超えるパスワードの切り詰め警告チェックを含む）。
  • ALG_BCRYPT: apr_generate_random_bytes でソルトを生成後、apr_bcrypt_encode を使用（ctx->cost が未設定の場合はデフォルト値を使用）。

    • 2. 未サポートのアルゴリズムが指定された場合はエラーを返す。 3. 最後に memset を実行し、平文パスワードをゼロクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_request.c:1008
  • **呼び出し先**: generate_salt（内部関数）、apr_sha1_base64、apr_md5_encode、crypt、apr_bcrypt_encode、apr_generate_random_bytes、apr_psprintf、apr_cpystrn、apr_pstrdup、memset、strlen、strcmp。

関連データ構造

  • passwd_ctx: ハッシュ化のパラメータと結果を保持する主要構造体。

失敗モードと境界条件

  • **アルゴリズム未サポート**: default ケースによりエラーを返す。
  • **ソルト生成失敗**: generate_salt または apr_generate_random_bytes の失敗時にエラーを返す。
  • **エンコード失敗**: apr_md5_encode や apr_bcrypt_encode の失敗時にエラーを返す。
  • **CRYPT失敗**: crypt 関数が NULL を返した場合、OSエラーを取得しエラーを返す。
  • **パスワード切り詰め**: ALG_CRYPT 使用時に8文字を超えた場合、警告メッセージを ctx->errstr に設定する。

保守担当者向けメモ

  • CRYPT_ALGO_SUPPORTED および BCRYPT_ALGO_SUPPORTED マクロにより、コンパイル環境に応じて利用可能なアルゴリズムが制限される。
  • 平文パスワードのメモリ保護のため、関数終了時の memset は必須である。
