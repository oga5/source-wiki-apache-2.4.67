authn_dbd_realm

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_dbd.c:191

目的

AuthDigest 認証において、データベースから指定されたユーザーとレルムに対応するハッシュ値を取得し、認証プロバイダーとして提供する。

シグネチャまたは形状

authn_status authn_dbd_realm(request_rec *r, const char *user,
                             const char *realm, char **rethash)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • user: 認証対象のユーザー名。
  • realm: 認証対象のレルム名。
  • **出力**:
  • rethash: データベースから取得したハッシュ値を格納するポインタ。
  • 戻り値: AUTH_USER_FOUND（成功）、AUTH_USER_NOT_FOUND（ユーザー不在）、または AUTH_GENERAL_ERROR（エラー）。

前提条件と不変条件

  • mod_dbd が利用可能であり、データベース接続が確立されていること。
  • 設定ファイルにて AuthDBDUserRealmQuery が定義されていること。
  • SQL クエリの結果セットの最初のカラムは、必ずパスワードハッシュであること。
  • メモリ管理は r->pool に依存しており、リクエスト終了時に自動的に解放される。

アルゴリズムまたは振る舞い

1. authn_dbd_acquire_fn を介してデータベース接続を取得する。 2. 設定された AuthDBDUserRealmQuery に基づき、準備済みステートメントを検索する。 3. apr_dbd_pvselect を実行し、ユーザー名とレルムをバインドしてクエリを実行する。 4. 結果セットを走査し、最初のカラムを rethash として抽出する。 5. 2番目以降のカラムが存在する場合、AUTHENTICATE_ プレフィックスを付与して subprocess_env に環境変数として展開する（カラム名は英数字に変換される）。 6. 取得したハッシュを AUTHN_CACHE_STORE マクロ経由でキャッシュに保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_authn_core が管理する認証プロバイダーフレームワークから呼び出される（直接の呼び出し元は modules/aaa/mod_authn_dbd.c:289 に登録されたプロバイダー構造体経由）。
  • **呼び出し先**: authn_dbd_acquire_fn（接続取得）、apr_dbd_pvselect（クエリ実行）、apr_dbd_get_row（結果取得）、apr_table_set（環境変数設定）、AUTHN_CACHE_STORE（キャッシュ保存）。

関連データ構造

  • authn_dbd_conf: ディレクトリ設定を保持する構造体。
  • request_rec: リクエストごとの状態とプールを管理。
  • ap_dbd_t: データベース接続ハンドル。

失敗モードと境界条件

  • データベース接続の取得失敗: AUTH_GENERAL_ERROR を返す。
  • AuthDBDUserRealmQuery 未設定: AUTH_GENERAL_ERROR を返す。
  • クエリ実行失敗: AUTH_GENERAL_ERROR を返す。
  • ユーザーが見つからない場合: AUTH_USER_NOT_FOUND を返す。
  • 結果セットの取得エラー: AUTH_GENERAL_ERROR を返す。

保守担当者向けメモ

  • SQL インジェクション対策は apr_dbd_pvselect のパラメータバインディングに依存しているため、クエリ文字列の構築には注意すること。
