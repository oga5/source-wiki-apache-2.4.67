md_event_subscribe

定義

  • 種別: function
  • 定義箇所: modules/md/md_event.c:50

目的

イベント発生時に実行されるコールバック関数を、グローバルなイベント管理構造体（EVNT）の購読リストに追加します。

シグネチャまたは形状

void md_event_subscribe(md_event_cb *cb, void *baton)

入力と出力

  • cb: 登録するコールバック関数へのポインタ。
  • baton: コールバック実行時に渡される任意のデータへのポインタ。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • EVNT 構造体が md_event_init によって初期化されている必要があります。
  • 本関数はスレッドセーフではありません。マルチスレッド環境で使用する場合は、呼び出し元で排他制御を行う必要があります。
  • 購読解除機能は提供されておらず、登録されたコールバックはプロセスの生存期間中有効です。

アルゴリズムまたは振る舞い

1. EVNT.p（メモリプール）から md_subscription 構造体分のメモリを確保します。 2. 引数として渡された cb と baton を確保した構造体に格納します。 3. 新しい購読エントリの next ポインタを現在の購読リストの先頭（EVNT.subs）に設定します。 4. EVNT.subs を新しいエントリで更新し、リストの先頭に挿入します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md.c:891（md_event_subscribe(on_event, mc);）
  • 呼び出し先: apr_pcalloc（APRメモリ管理関数）

関連データ構造

  • md_subscription: 購読情報を保持するリスト構造体。
  • EVNT: イベント管理用のグローバル構造体。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足により失敗した場合、APRの規約に従いプロセスが異常終了する可能性があります。

保守担当者向けメモ

  • 本関数はリストの先頭に要素を追加する単純な実装です。順序は登録の逆順となります。
  • 購読の解除機能が必要な場合は、md_subscription リストの走査と削除ロジックを別途実装する必要があります。
