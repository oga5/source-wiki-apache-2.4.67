ap_get_local_host

定義

  • server/util.c:2429

目的

サーバーの完全修飾ドメイン名（FQDN）を特定し、特定できない場合にフォールバック値（127.0.0.1）を返すことで、サーバーのホスト名を決定する。

シグネチャまたは形状

char *ap_get_local_host(apr_pool_t *a)

入力と出力

  • **入力**: apr_pool_t *a（メモリ割り当て用プール）
  • **出力**: 決定されたホスト名を含む文字列（apr_pstrdup によりプールから割り当てられる）。

前提条件と不変条件

  • apr_pool_t が有効であること。
  • 内部的に MAXHOSTNAMELEN（デフォルト 256）までのホスト名バッファを使用する。

アルゴリズムまたは振る舞い

1. apr_gethostname を呼び出し、ローカルホスト名を取得する。失敗した場合は警告ログを出力する。 2. 取得したホスト名を apr_sockaddr_info_get で解決し、FQDN の特定を試みる。 3. apr_getnameinfo で名前解決を行い、ドット（.）が含まれていればそれを FQDN とみなして返す。 4. ドットが含まれない場合、元のホスト名にドットが含まれていればそれを使用し、それ以外の場合は IP アドレスを解決して返す。 5. 上記のいずれでもホスト名が確定しない場合、127.0.0.1 を使用し、警告ログ（APLOG_ALERT）を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/vhost.c:585
  • **呼び出し先**: apr_gethostname, apr_sockaddr_info_get, apr_getnameinfo, apr_sockaddr_ip_get, apr_pstrdup, ap_log_perror, ap_strchr_c

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_sockaddr_t: ソケットアドレス情報の保持に使用。

失敗モードと境界条件

  • apr_gethostname が失敗した場合、警告ログを出力しつつ処理を継続する。
  • apr_sockaddr_info_get が失敗した場合、警告ログを出力し、最終的に 127.0.0.1 にフォールバックする。
  • ホスト名が FQDN として信頼できない（ドットを含まない）場合、警告メッセージが表示される。

保守担当者向けメモ

  • 本関数は ServerName ディレクティブが設定されていない場合に自動的にホスト名を決定するためのフォールバックとして機能する。
  • ログ出力には ap_server_argv0 を使用しており、実行時のプロセス名がログに含まれる。
