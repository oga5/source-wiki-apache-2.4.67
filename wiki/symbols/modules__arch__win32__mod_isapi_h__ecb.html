ecb

定義

  • 種別: typedef
  • 定義箇所: modules/arch/win32/mod_isapi.h:209

目的

ISAPI 拡張が Apache 上でリクエストを処理する際に使用する、EXTENSION_CONTROL_BLOCK 構造体へのポインタ型を定義します。

シグネチャまたは形状

typedef apr_uint32_t (APR_THREAD_FUNC *PFN_HTTPEXTENSIONPROC)(EXTENSION_CONTROL_BLOCK *ecb);

※ ecb は上記関数ポインタ型の引数として定義されています。

入力と出力

  • **入力**: EXTENSION_CONTROL_BLOCK 構造体へのポインタ。
  • **出力**: なし（EXTENSION_CONTROL_BLOCK 構造体自体は、Apache のリクエスト処理パイプラインと ISAPI DLL 間のデータ受け渡しに使用されます）。

前提条件と不変条件

  • EXTENSION_CONTROL_BLOCK は Apache によってリクエストのライフサイクルに合わせて管理されます。
  • ISAPI 拡張側でこの構造体を永続化してはなりません。

アルゴリズムまたは振る舞い

  • mod_isapi.c において、Apache のリクエスト状態（request_rec）と ISAPI 拡張間の橋渡しを行います。
  • dwHttpStatusCode メンバを介して、Apache のリクエストステータスと ISAPI 側のステータスコードの同期が行われます。
  • 非同期処理の完了通知や、リダイレクト等のサーバーサポート機能の呼び出し時に、この構造体が引数として渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_isapi.c 内の各ハンドラおよびコールバック関数。
  • **呼び出し先**: 情報なし。

関連データ構造

  • EXTENSION_CONTROL_BLOCK: ISAPI 拡張に渡されるリクエストコンテキストの核となる構造体。
  • isapi_cid: Apache 側のリクエスト状態を保持する構造体（HCONN）。

失敗モードと境界条件

  • dwHttpStatusCode の設定値が Apache の期待する HTTP ステータスコードと不整合を起こす可能性があります。
  • HSE_REQ_DONE_WITH_SESSION 等の制御フローにおいて、セッション終了のタイミングを誤るとリクエスト処理が正しく完了しません。

保守担当者向けメモ

  • 本シンボルは EXTENSION_CONTROL_BLOCK への参照を指します。
  • mod_isapi.c 内で cid->ecb として頻繁に参照されており、Apache のリクエスト状態と ISAPI 拡張のステータスを同期させるための重要な接点です。
