hm_create_config

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartmonitor.c:801

目的

mod_heartmonitor モジュールのサーバー単位の設定構造体（hm_ctx_t）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *hm_create_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定割り当て用のメモリプール。
  • server_rec *s: 関連付けられるサーバーレコード。
  • **出力**:
  • void *: 初期化された hm_ctx_t 構造体へのポインタ。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • s は有効なサーバーレコードであること。
  • 戻り値の ctx->p は、p から作成されたサブプールとして管理される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して hm_ctx_t 構造体分のメモリを確保する。 2. active フラグを 0 に初期化する。 3. storage_path を DEFAULT_HEARTBEAT_STORAGE に基づいて設定する。 4. interval を HM_UPDATE_SEC マクロ値から設定する。 5. s を構造体に保持する。 6. ctx->p としてサブプールを作成し、デバッグ用に "hm_ctx" タグを付与する。 7. ctx->servers として空のハッシュテーブルを作成する。 8. 初期化済みの ctx を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cluster/mod_heartmonitor.c:916（サーバー設定作成時に呼び出される）。
  • **呼び出し先**:
  • apr_palloc
  • ap_runtime_dir_relative
  • apr_time_from_sec
  • apr_pool_create
  • apr_pool_tag
  • apr_hash_make

関連データ構造

  • hm_ctx_t: 本関数が初期化するメインの設定構造体。
  • server_rec: Apache HTTP Server のサーバー設定構造体。
  • apr_pool_t: メモリ管理用プール。
  • apr_hash_t: ctx->servers で使用されるハッシュテーブル。

失敗モードと境界条件

  • メモリ割り当て失敗時、apr_palloc および apr_pool_create は Apache のメモリ管理ポリシーに従い、通常はプロセスを終了させる（例外をスローしない）。

保守担当者向けメモ

  • HM_UPDATE_SEC は現在マクロ定義であるが、将来的に実行時設定へ移行する計画がある。
  • ctx->servers は ctx->p プール上で管理されているため、サーバー設定の破棄時に自動的にクリーンアップされる。
