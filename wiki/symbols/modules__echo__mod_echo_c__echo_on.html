echo_on

定義

  • 種別: function
  • 定義箇所: modules/echo/mod_echo.c:45

目的

Apache HTTP Server の設定ディレクティブ ProtocolEcho に対応し、モジュールの有効/無効状態を EchoConfig 構造体に反映させるための設定ハンドラです。

シグネチャまたは形状

static const char *echo_on(cmd_parms *cmd, void *dummy, int arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: ディレクティブで指定された値（真偽値）。
  • **出力**:
  • 常に NULL を返します（成功を示す）。

前提条件と不変条件

  • cmd->server->module_config から取得される設定が EchoConfig 型であることを前提としています。
  • echo_module が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定から EchoConfig 構造体を取得します。 2. 引数 arg の値を pConfig->bEnabled に代入し、モジュールの有効状態を更新します。 3. NULL を返して処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/echo/mod_echo.c:199 において、AP_INIT_FLAG マクロを通じて ProtocolEcho ディレクティブの設定時に呼び出されます。
  • **呼び出し先**: ap_get_module_config（Apache API）を呼び出します。

関連データ構造

  • EchoConfig: モジュールの設定状態を保持する構造体。
  • cmd_parms: Apache の設定コマンドパラメータを保持する構造体。

失敗モードと境界条件

  • ap_get_module_config が無効なコンテキストで呼び出された場合、あるいは設定が正しく初期化されていない場合の挙動については、本ソースコード内では明示されていません。

保守担当者向けメモ

  • 本関数は AP_INIT_FLAG で登録されているため、設定ファイルで ProtocolEcho On または Off が指定された際に実行されます。
  • dummy 引数は現在の実装では使用されていませんが、cmd_func 型のシグネチャに合わせるために保持されています。
