cgi_bucket_data

定義

  • 種別: struct
  • 定義箇所: modules/generators/cgi_common.h:229

目的

CGI スクリプトの入出力パイプを監視するためのカスタムバケット（CGI 型）に関連付けられた、非同期 I/O 処理用の状態データを保持します。

シグネチャまたは形状

struct cgi_bucket_data {
    apr_pollset_t *pollset;
    request_rec *r;
    apr_interval_time_t timeout;
};

入力と出力

  • **入力**: request_rec を介してリクエストのコンテキストを保持し、apr_pollset_t を介して監視対象のパイプ状態を管理します。
  • **出力**: 本構造体自体はデータ保持用であり、バケット読み取り処理（cgi_bucket_read）において、スクリプトの stdout/stderr を取得するための制御情報として機能します。

前提条件と不変条件

  • request_rec のメモリプール（r->pool）を使用して割り当てられます。
  • バケットの生存期間は、関連付けられたリクエストの生存期間に依存します。
  • WANT_CGI_BUCKET マクロが定義されている環境でのみ有効です。

アルゴリズムまたは振る舞い

  • cgi_bucket_read において、poll() を実行し、スクリプトの出力が完了するまで監視を行うための基盤となります。
  • pollset を用いて stdout と stderr を非同期的に監視し、stderr にデータがある場合は log_script_err を通じてエラーログへ転送します。

呼び出し元と呼び出し先

  • **呼び出し元**: cgi_bucket_create（構造体の初期化）、cgi_bucket_dup（複製）、cgi_bucket_read（データ読み取り）から参照されます。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_pollset_t: I/O 監視用。
  • request_rec: リクエストコンテキスト。
  • apr_bucket: 本構造体を内部データとして保持するバケット。

失敗モードと境界条件

  • poll() 実行時のタイムアウト設定が重要であり、スクリプトの出力が完了しない場合にブロックする可能性があります。
  • メモリ割り当て失敗時は、apr_palloc の挙動に従いリクエストのプールからエラーが伝播します。

保守担当者向けメモ

  • cgi_bucket_read 内での poll() はブロッキングの可能性があるため、高負荷時の挙動に注意してください。
  • log_scripterror での apr_stat 呼び出しはパフォーマンスコストとなる場合があります。
