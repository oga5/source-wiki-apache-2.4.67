ap_signal_parent_e

定義

  • 種別: enum
  • 定義箇所: server/mpm/winnt/mpm_winnt.h:83

目的

Windows 版 Apache HTTP Server において、子プロセスから親プロセスに対して送信するシグナル（制御要求）の種類を定義します。

シグネチャまたは形状

typedef enum {
    SIGNAL_PARENT_SHUTDOWN,
    SIGNAL_PARENT_RESTART,
    SIGNAL_PARENT_RESTART_GRACEFUL
} ap_signal_parent_e;

入力と出力

  • 本シンボル自体は列挙型であり、関数ではありません。
  • ap_signal_parent(ap_signal_parent_e type) 関数の引数として使用され、親プロセスへの通知内容を決定します。

前提条件と不変条件

  • Windows 環境（WIN32）でのみ使用されます。
  • 定義された3つの値（SHUTDOWN, RESTART, RESTART_GRACEFUL）のいずれかが、親プロセスへの制御要求として使用されることを想定しています。

アルゴリズムまたは振る舞い

  • SIGNAL_PARENT_SHUTDOWN: 親プロセスに対してサーバーの停止を要求します。
  • SIGNAL_PARENT_RESTART: 親プロセスに対してサーバーの再起動を要求します。
  • SIGNAL_PARENT_RESTART_GRACEFUL: 親プロセスに対してグレースフル（緩やかな）再起動を要求します。

呼び出し元と呼び出し先

  • 呼び出し元: 子プロセス内の処理ロジック（server/mpm/winnt/mpm_winnt.c 等）が、親プロセスへイベントを通知する際に使用します。
  • 呼び出し先: ap_signal_parent 関数がこの列挙型を引数として受け取り、内部で適切なシグナル処理を実行します。

関連データ構造

  • ap_signal_parent 関数（server/mpm/winnt/mpm_winnt.c:202 で宣言・実装）

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本列挙型は mpm_winnt モジュールにおけるプロセス間通信のインターフェースの一部です。新しいシグナルタイプを追加する場合は、ap_signal_parent 関数の実装側でも対応する処理を追加する必要があります。
