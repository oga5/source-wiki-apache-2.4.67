in_domain

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_access_compat.c:215

目的

ホスト名（what）が指定されたドメイン（domain）に属しているかを検証します。

シグネチャまたは形状

static int in_domain(const char *domain, const char *what)

入力と出力

  • **入力**:
  • domain: 比較対象のドメイン文字列。
  • what: 検証対象のホスト名文字列。
  • **出力**:
  • 1: what が domain に属している（または一致する）場合。
  • 0: 属していない場合。

前提条件と不変条件

  • domain および what はヌル終端文字列である必要があります。
  • what の長さが domain より短い場合、常に 0 を返します。

アルゴリズムまたは振る舞い

1. domain と what の長さを取得します。 2. what の末尾が domain と一致するかを strcasecmp で比較します。 3. 一致しない場合は 0 を返します。 4. 一致する場合、サブドメインの境界を検証します：

  • what と domain が完全に一致する場合は 1 を返します。
  • それ以外の場合、domain の先頭文字が . であるか、または what 内の domain の直前の文字が . であることを確認し、境界が正しく区切られている場合にのみ 1 を返します。これにより、good.com が nogood.com にマッチするのを防ぎます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_access_compat.c:293
  • modules/aaa/mod_authz_host.c:217
  • **呼び出し先**:
  • 標準ライブラリ関数: strlen, strcasecmp

関連データ構造

  • なし。

失敗モードと境界条件

  • what が domain より短い場合、即座に 0 を返します。
  • DNS 逆引きの結果に依存するため、REMOTE_DOUBLE_REV 設定等のセキュリティ上の考慮が必要です。

保守担当者向けメモ

  • 本関数は単純な文字列比較に基づいています。ホスト名の正規化や DNS 解決の信頼性は呼び出し側のコンテキストに依存します。
