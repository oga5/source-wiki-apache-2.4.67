ap_strrchr_c

定義

  • 種別: function
  • 定義箇所: server/util_debug.c:73

目的

標準ライブラリの strrchr をラップし、const char * 型の文字列に対して型安全な文字検索インターフェースを提供します。

シグネチャまたは形状

const char *ap_strrchr_c(const char *s, int c)

入力と出力

  • **入力**:
  • s: 検索対象のヌル終端文字列（const char *）。
  • c: 検索する文字（int）。
  • **出力**:
  • 文字列 s 内で文字 c が最後に出現する位置を指すポインタを返します。文字が見つからない場合は NULL を返します。

前提条件と不変条件

  • s は有効なヌル終端文字列である必要があります。
  • 戻り値のポインタは、入力された文字列 s の範囲内、または NULL です。

アルゴリズムまたは振る舞い

標準ライブラリ関数 strrchr(s, c) を呼び出し、その結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し先**: 標準ライブラリ関数 strrchr。
  • **呼び出し元（直接的な証拠）**:
  • modules/aaa/mod_authn_core.c
  • modules/aaa/mod_authn_socache.c
  • modules/aaa/mod_authz_core.c
  • modules/cache/cache_storage.c
  • modules/core/mod_macro.c
  • modules/generators/mod_autoindex.c
  • modules/http/mod_mime.c
  • modules/http2/mod_http2.c
  • modules/lua/mod_lua.c

関連データ構造

なし。

失敗モードと境界条件

  • s が NULL の場合、標準ライブラリの strrchr の挙動に依存します（通常は未定義動作）。
  • 文字 c が文字列内に存在しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は httpd.h においてマクロとしても定義されています（httpd.h:2418）。
  • server/util_debug.c 内での定義は、デバッグやバイナリ互換性のためのラッパーとして機能しています。
