ap_socache_instance_t

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_memcache.c:57 （および ap_socache.h:49）

目的

Memcached バックエンドを用いた共有オブジェクトキャッシュ（socache）のインスタンス状態を保持するための構造体です。

シグネチャまたは形状

struct ap_socache_instance_t {
    const char *servers;        /* Memcached サーバーリスト */
    apr_memcache_t *mc;         /* apr_memcache クライアントオブジェクト */
    const char *tag;            /* 名前空間タグ */
    apr_size_t taglen;          /* tag の長さ + 1 */
};

入力と出力

  • 本構造体は、socache_mc_create 等のプロバイダー関数において、キャッシュ操作のコンテキストとして渡されます。
  • mc メンバは apr_memcache ライブラリの操作対象として利用されます。

前提条件と不変条件

  • taglen は strlen(tag) + 1 と一致している必要があります。
  • mc は socache_mc_init によって初期化された有効な apr_memcache_t ポインタである必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、mod_socache_memcache におけるキャッシュの接続管理および名前空間の分離を担います。
  • tag を用いてキャッシュキーのプレフィックスを生成し、複数のキャッシュエントリ間での名前衝突を防ぎます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cache_socache.c や mod_authn_socache.c などの上位モジュールが、ap_socache API を通じて本構造体を参照します。
  • **呼び出し先**: apr_memcache ライブラリの各関数（apr_memcache_set, apr_memcache_getp 等）が、mc メンバを引数として呼び出されます。

関連データ構造

  • apr_memcache_t: apr_memcache ライブラリのメインハンドル。
  • socache_mc_svr_cfg: 本モジュールのサーバー単位の設定を保持する構造体。

失敗モードと境界条件

  • servers が未設定または不正な形式の場合、socache_mc_init での初期化に失敗します。
  • tag の長さが MC_KEY_LEN（254バイト）の制限を超える場合、キー生成時にエラーが発生します。

保守担当者向けメモ

  • 本構造体は ap_socache.h で定義されたインターフェースに準拠しています。
  • mc メンバのライフサイクルは、Apache のプール管理に依存しています。socache_mc_destroy は現在 no-op ですが、将来的なリソース解放の際は本構造体のメンバを適切に破棄する必要があります。
