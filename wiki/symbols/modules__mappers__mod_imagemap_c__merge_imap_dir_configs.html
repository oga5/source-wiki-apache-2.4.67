merge_imap_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_imagemap.c:102

目的

ディレクトリ単位の設定構造体 (imap_conf_rec) をマージし、子コンテキスト（addv）で設定が未定義の場合に親コンテキスト（basev）の値を継承させる。

シグネチャまたは形状

static void *merge_imap_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定 (imap_conf_rec*)。
  • addv: 現在のディレクトリの設定 (imap_conf_rec*)。
  • **出力**:
  • マージ済みの新しい imap_conf_rec 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は imap_conf_rec 型としてキャスト可能であること。
  • 戻り値の構造体は、提供されたメモリプール p 上に確保される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、新しい imap_conf_rec 構造体用のメモリを確保する。 2. imap_menu、imap_default、imap_base の各フィールドについて、addv 側の値が設定されていればそれを採用し、未設定（NULL）であれば basev 側の値を採用する。 3. マージされた新しい構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_imagemap.c:892 （Apache の設定マージ処理の一部として参照）。
  • **呼び出し先**: 直接的な関数呼び出しはなし（メモリ確保のための apr_palloc を除く）。

関連データ構造

  • imap_conf_rec: ディレクトリごとの設定（ImapMenu, ImapDefault, ImapBase）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある。
  • basev または addv が NULL の場合の挙動については、ソースコード上での明示的なガードは存在しない。

保守担当者向けメモ

  • 本関数は Apache の設定マージ機構の一部であり、新しい設定ディレクティブを imap_conf_rec に追加した場合は、本関数内でもマージロジックを更新する必要がある。
