keep_body_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:65

目的

HTTP リクエストボディをメモリ（r->kept_body）に一時的にコピーし、後続のサブリクエストやフィルタが再利用できるようにする入力フィルタです。

シグネチャまたは形状

static apr_status_t keep_body_filter(ap_filter_t *f, apr_bucket_brigade *b,
                                     ap_input_mode_t mode,
                                     apr_read_type_e block,
                                     apr_off_t readbytes);

入力と出力

  • **入力**:
  • f: フィルタコンテキスト。
  • b: 上流から取得したバケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。
  • 成功時、リクエストボディが f->r->kept_body に格納されます。

前提条件と不変条件

  • r->kept_body は r->pool に紐付き、リクエスト終了時に解放されます。
  • dconf->keep_body（設定された保持サイズ制限）を超過した場合、フィルタはエラーを返し、保持されたデータは破棄されます。
  • kept_body_filter は、r->kept_body が存在する場合にのみ動作します。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時に keep_body_ctx_t を作成し、Content-Length を検証します。制限を超える場合は bail_out_on_error を呼び出し終了します。 2. **データ取得**: ap_get_brigade を通じて上流からデータを読み込みます。 3. **サイズ検証**: 読み込んだデータの合計が設定制限を超えた場合、r->kept_body をクリーンアップし、HTTP_REQUEST_ENTITY_TOO_LARGE を返します。 4. **保存**: 読み込んだバケットを apr_bucket_copy で複製し、r->kept_body ブリゲードの末尾に追加します。 5. **透過**: 取得したブリゲード b はそのまま後続のフィルタへ渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache フィルタチェーン（ap_get_brigade 経由）。
  • **呼び出し先**:
  • ap_get_brigade（上流フィルタ）
  • bail_out_on_error（エラー発生時）
  • apr_bucket_copy / apr_brigade_create（データ保持用）

関連データ構造

  • keep_body_ctx_t: 保持サイズ制限と現在の残り容量を管理するコンテキスト。
  • request_dir_conf: KeptBodySize ディレクティブの設定値。
  • request_rec: kept_body ブリゲードを保持するリクエスト構造体。

失敗モードと境界条件

  • Content-Length が設定値を超える場合: HTTP_REQUEST_ENTITY_TOO_LARGE を返します。
  • apr_bucket_copy 失敗時: 呼び出し元へエラーを伝播します。
  • 既に r->kept_body が存在する場合: フィルタは自身を削除し、透過的に動作します。

保守担当者向けメモ

  • r->kept_body のメモリ所有権はリクエストプールにあるため、手動での解放は apr_brigade_cleanup を通じて行われます。
  • Content-Length の解析には ap_parse_strict_length を使用しており、不正な形式やオーバーフローを厳格にチェックしています。
