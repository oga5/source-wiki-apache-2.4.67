si_val_valid_time

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:298

目的

JSON 形式で保持された証明書等の有効期間（開始日時および終了日時）を解析し、status_ctx の出力モード（HTML またはテキスト）に応じて整形して出力する。

シグネチャまたは形状

static void si_val_valid_time(status_ctx *ctx, md_json_t *mdj, const status_info *info)

入力と出力

  • **入力**:
  • ctx: 出力先バッファやフォーマット設定を保持するコンテキスト。
  • mdj: 有効期間情報を含む JSON オブジェクト。
  • info: JSON キーやラベル情報を含むメタデータ構造体。
  • **出力**: なし（副作用として ctx->bb に文字列を出力）。

前提条件と不変条件

  • mdj は有効な md_json_t オブジェクトであること。
  • info->key は mdj 内の有効期間データ（MD_KEY_FROM, MD_KEY_UNTIL）を指すキーとして機能すること。
  • apr_date_parse_rfc が RFC 形式の日付文字列を正しく解析できること。

アルゴリズムまたは振る舞い

1. md_json_gets を使用し、info->key 配下の from（開始）および until（終了）日時を RFC 形式文字列として取得する。 2. apr_date_parse_rfc で各文字列を apr_time_t に変換する。 3. HTML_STATUS(ctx) が真の場合（HTML 出力時）:

  • 現在時刻より後の開始日時であれば "from [日付]" を出力。
  • 終了日時が存在すれば、"until [日付]" を出力。

    • 4. それ以外（テキスト出力時）:

  • 現在時刻より後の開始日時であれば、ラベル付きで開始日時を出力。
  • 終了日時が存在すれば、ラベル付きで終了日時を出力。

    • 5. print_date を介して、人間が読みやすい形式でバッファへ書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_status.c:349 (直接の呼び出し)
  • modules/md/mod_md_status.c:835 (構造体 status_info のメンバとして参照)
  • **呼び出し先**:
  • md_json_gets
  • apr_date_parse_rfc
  • HTML_STATUS (マクロ)
  • apr_brigade_puts
  • print_date
  • apr_pstrcat / apr_psprintf

関連データ構造

  • status_ctx: 出力状態を管理する構造体。
  • md_json_t: JSON データを保持する構造体。
  • status_info: ステータス表示用のメタデータ定義。

失敗モードと境界条件

  • from や until が JSON 内に存在しない場合、apr_date_parse_rfc は 0 を返し、その項目は出力されない。
  • from が過去の日付である場合、HTML/テキスト出力のいずれにおいても開始日時の表示はスキップされる。

保守担当者向けメモ

  • 本関数は mod_status の出力形式に強く依存している。HTML 出力時は sep 変数を用いて、開始と終了の間にスペースを挿入する制御を行っている。
  • print_date 関数が内部でどのようにフォーマットを行うかによって、出力される文字列の可読性が決定される。
