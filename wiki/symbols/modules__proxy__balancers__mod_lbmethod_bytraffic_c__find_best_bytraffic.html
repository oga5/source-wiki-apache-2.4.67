find_best_bytraffic

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_bytraffic.c:60

目的

トラフィック量（送受信バイト数）に基づき、負荷が最小であるバックエンドワーカーを選択する。

シグネチャまたは形状

static proxy_worker *find_best_bytraffic(proxy_balancer *balancer, request_rec *r)

入力と出力

  • **入力**:
  • balancer: 負荷分散対象のバランサー構造体。
  • r: 現在のリクエスト情報。
  • **出力**:
  • 選択された proxy_worker 構造体へのポインタ。

前提条件と不変条件

  • ap_proxy_balancer_get_best_worker_fn が mod_proxy から提供されていること。
  • is_best_bytraffic がワーカーの比較基準として正しく機能すること。

アルゴリズムまたは振る舞い

1. 最小トラフィック量を保持する変数 min_traffic を 0 で初期化する。 2. ap_proxy_balancer_get_best_worker_fn を呼び出し、全ワーカーに対して is_best_bytraffic を適用する。 3. is_best_bytraffic の評価結果に基づき、最も負荷の低いワーカーを特定して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/balancers/mod_lbmethod_bytraffic.c:90 （bytraffic 構造体のメンバとして登録）
  • **呼び出し先**:
  • ap_proxy_balancer_get_best_worker_fn （ワーカー走査用関数）
  • is_best_bytraffic （各ワーカーの負荷比較用コールバック）

関連データ構造

  • proxy_balancer: バランサーの状態を保持。
  • proxy_worker: 負荷分散対象のワーカー情報。
  • request_rec: リクエストコンテキスト。

失敗モードと境界条件

  • ワーカーが一つも存在しない場合、ap_proxy_balancer_get_best_worker_fn の挙動に依存する。
  • 統計情報がリセット直後の場合、全てのワーカーの負荷が 0 として扱われる。

保守担当者向けメモ

  • 本関数は bytraffic アルゴリズムのメインエントリポイントです。
  • 負荷計算のロジックを変更する場合は、本関数ではなくコールバックである is_best_bytraffic を修正してください。
