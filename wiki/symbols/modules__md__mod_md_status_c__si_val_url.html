si_val_url

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:200

目的

JSON データから URL を抽出し、対応する認証局（CA）名とともに HTML またはテキスト形式でステータス出力へ書き出す。

シグネチャまたは形状

static void si_val_url(status_ctx *ctx, md_json_t *mdj, const status_info *info)

入力と出力

  • **入力**:
  • ctx: ステータス出力用のコンテキスト（出力先ブリゲード、プール、フォーマットフラグを保持）。
  • mdj: 情報を取得する対象の JSON オブジェクト。
  • info: 現在処理中のカラム定義（key および label を参照）。
  • **出力**: なし（ctx->bb に直接書き込まれる）。

前提条件と不変条件

  • mdj および info は有効なポインタであること。
  • ctx->p（プール）はリクエストの生存期間中有効であること。
  • md_json_gets により取得された URL が存在しない場合、関数は何も出力せずに終了する。

アルゴリズムまたは振る舞い

1. info->key を用いて mdj から URL 文字列を取得する。 2. md_get_ca_name_from_url を呼び出し、URL から CA 名を導出する。 3. HTML_STATUS(ctx) が真の場合:

  • HTML の <a> タグとして、URL をリンク先、CA 名を表示テキストとして出力する。

    • 4. それ以外（テキストモード）の場合:

  • ctx->prefix と info->label を付与し、CA 名と URL をそれぞれ別行で出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md_status.c:836（status_info 配列の初期化において参照）。
  • **呼び出し先**:
  • md_json_gets
  • md_get_ca_name_from_url
  • ap_escape_html2
  • apr_brigade_printf

関連データ構造

  • status_ctx: 出力先やフォーマット状態を管理する構造体。
  • status_info: ステータス表示のメタデータ（キー名、ラベル、表示関数）を保持する構造体。
  • md_json_t: JSON データを保持する抽象構造体。

失敗モードと境界条件

  • URL が JSON 内に見つからない場合、処理は中断され何も出力されない。
  • HTML 出力時には ap_escape_html2 を使用し、XSS 等のインジェクションを防ぐ設計となっている。

保守担当者向けメモ

  • 本関数は status_info 配列のコールバックとして利用されるため、シグネチャの変更には mod_md_status.c 内の定義テーブルの更新が必要。
  • CA 名の導出ロジック（md_get_ca_name_from_url）は md_util 側に依存しているため、URL 形式の変更時にはそちらの確認も推奨される。
