proxy_hashes

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:417

目的

ワーカー名などの識別子をハッシュ化して保持するための構造体です。

シグネチャまたは形状

typedef struct {
    unsigned int def;
    unsigned int fnv;
} proxy_hashes;

入力と出力

  • 入力: 構造体の各メンバ（def, fnv）は、ハッシュ計算の結果を格納するために使用されます。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • def および fnv は、ワーカー名のハッシュ値を格納する目的で利用されます。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持のみを行い、具体的なハッシュ計算アルゴリズムは proxy_util.c 等の呼び出し側で実装されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/proxy/mod_proxy.h 内の proxy_worker 関連構造体（441, 503, 540, 566行目）
  • modules/proxy/proxy_util.c:1142
  • 呼び出し先: なし（データ構造のため）

関連データ構造

  • proxy_worker: バックエンドサーバー定義。本構造体をメンバとして保持し、ワーカー名のハッシュを管理します。

失敗モードと境界条件

  • 構造体メンバの型が unsigned int であるため、ハッシュ値の衝突（collision）が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はワーカー名のハッシュ値を保持するために複数の箇所で利用されています。ハッシュ計算ロジックを変更する際は、proxy_util.c の該当箇所と整合性を確認してください。
