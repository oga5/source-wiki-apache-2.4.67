check_values

定義

  • 種別: function
  • 定義箇所: modules/md/md_reg.c:140

目的

管理対象ドメイン（MD）の設定値が、指定されたカテゴリにおいて妥当であるかを検証する。

シグネチャまたは形状

apr_status_t check_values(md_reg_t *reg, apr_pool_t *p, const md_t *md, int fields)

入力と出力

  • **入力**:
  • reg: レジストリハンドル。
  • p: メモリプール。
  • md: 検証対象のドメイン設定構造体。
  • fields: 検証対象を指定するビットマスク（MD_UPD_DOMAINS, MD_UPD_CONTACTS, MD_UPD_CA_URL 等）。
  • **出力**:
  • APR_SUCCESS: 検証成功。
  • APR_EINVAL: 検証失敗（無効なドメイン名、重複、または不正なURI形式）。

前提条件と不変条件

  • fields で指定された各フィールドが md 構造体内に存在する場合、その値が検証される。
  • MD_UPD_DOMAINS 指定時、ドメインリストは空であってはならない。

アルゴリズムまたは振る舞い

1. **ドメイン検証**: MD_UPD_DOMAINS 指定時、各ドメイン名がDNS名またはワイルドカードとして妥当か判定し、md_reg_find_overlap を用いて既存のMDとの重複がないか確認する。 2. **連絡先検証**: MD_UPD_CONTACTS 指定時、各連絡先URIが絶対URI形式であるか検証する。 3. **CA設定検証**: MD_UPD_CA_URL 指定時、CA URLが絶対URI形式であるか検証する。 4. **合意事項検証**: MD_UPD_AGREEMENT 指定時、ca_agreement が "accepted" 以外であれば、その値が絶対URI形式であるか検証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_reg.c:436
  • modules/md/md_reg.c:474
  • **呼び出し先**:
  • md_reg_find_overlap (ドメイン重複チェック)
  • md_dns_is_name / md_dns_is_wildcard (DNS名妥当性)
  • md_util_abs_uri_check (URI形式妥当性)
  • md_log_perror (エラーログ出力)

関連データ構造

  • md_t: 検証対象のドメイン設定を保持。
  • md_reg_t: レジストリのメイン構造体。

失敗モードと境界条件

  • ドメインリストが空の場合、APR_EINVAL を返す。
  • ドメイン名がDNS規則に違反する場合、APR_EINVAL を返す。
  • 既存のMDとドメインが重複する場合、APR_EINVAL を返す。
  • 連絡先やCA URLが絶対URIでない場合、APR_EINVAL を返す。

保守担当者向けメモ

  • MD_UPD_CA_PROTO および MD_UPD_CA_ACCOUNT の検証ロジックは現在空であり、将来的な拡張ポイントとなっている。
  • 本関数は設定変更時の整合性保証の要であるため、新しい設定項目を追加する際は必ず本関数での検証を検討すること。
