ap_logio_add_bytes_out

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_logio.c:67

目的

接続ごとの送信バイト数統計を更新するために使用される関数です。

シグネチャまたは形状

static void ap_logio_add_bytes_out(conn_rec *c, apr_off_t bytes)

入力と出力

  • **入力**:
  • c: 接続情報を保持する conn_rec 構造体へのポインタ。
  • bytes: 加算する送信バイト数 (apr_off_t)。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • c は有効な conn_rec 構造体を指している必要があります。
  • c->conn_config に logio_module に関連付けられた logio_config_t 構造体が格納されている必要があります。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、接続設定から logio_config_t 構造体を取得します。 2. 取得した構造体内の bytes_out メンバに対し、引数 bytes を加算します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c および server/core_filters.c 等から、APR_RETRIEVE_OPTIONAL_FN を介して間接的に呼び出されることが示唆されています。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core API) を呼び出します。

関連データ構造

  • conn_rec: 接続状態を保持する構造体。
  • logio_config_t: 接続ごとの入出力バイト数を保持する構造体。

失敗モードと境界条件

  • c->conn_config に logio_module の設定が登録されていない場合、ap_get_module_config は NULL を返し、その後のメンバアクセスでセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN によりオプション関数として公開されており、他のモジュールから動的に利用可能です。
  • 統計の加算はアトミック操作ではないため、マルチスレッド環境下での整合性は呼び出し側の制御に依存します。
