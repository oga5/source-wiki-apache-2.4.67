dialup_handler

定義

  • 種別: function
  • 定義箇所: modules/test/mod_dialup.c:132

目的

Apache HTTP Server において、低速なネットワーク接続（ダイヤルアップ回線）をシミュレートするためのメインハンドラです。リクエストをインターセプトし、帯域制限を適用するための初期設定と非同期送信の予約を行います。

シグネチャまたは形状

static int dialup_handler(request_rec *r);

入力と出力

  • **入力**: request_rec *r （処理対象のリクエスト情報）
  • **出力**: int （Apache のステータスコード。成功時は SUSPENDED、条件不一致時は DECLINED、エラー時はエラーコードを返す）

前提条件と不変条件

  • リクエストメソッドが GET であること。
  • 対象がファイルであること。
  • 設定（dialup_dcfg_t）で bytes_per_second が 0 以外に設定されていること。
  • ap_meets_conditions による条件チェックを通過すること。

アルゴリズムまたは振る舞い

1. **条件判定**: メソッドやファイルタイプを確認し、対象外であれば DECLINED を返します。 2. **設定取得**: ディレクトリ設定から帯域制限値を取得します。 3. **ファイルオープン**: 対象ファイルを読み込みモードで開き、リクエストプールに紐付けます。 4. **メタデータ設定**: 最終更新日時、ETag、Accept-Ranges、Content-Length を設定します。 5. **コンテキスト初期化**: dialup_baton_t を生成し、バケットブリゲード（bb, tmpbb）を作成してファイル内容を挿入します。 6. **初期送信**: dialup_send_pulse を呼び出し、最初のデータパルスを送信します。 7. **非同期予約**: 処理が完了していない場合、ap_mpm_register_timed_callback を用いて 1 秒後に dialup_callback が実行されるよう予約し、SUSPENDED を返してリクエスト処理を一時停止します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のハンドラフック（ap_hook_handler により APR_HOOK_ALMOST_LAST で登録）。
  • **呼び出し先**: ap_get_module_config, apr_file_open, ap_meets_conditions, dialup_send_pulse, ap_mpm_register_timed_callback 等の Apache/APR API。

関連データ構造

  • request_rec: リクエスト情報。
  • dialup_dcfg_t: ディレクトリごとの設定（帯域制限値）。
  • dialup_baton_t: 非同期処理のコンテキスト（バケットブリゲード、ファイル記述子、リクエスト情報）。

失敗モードと境界条件

  • ファイルが開けない場合や、条件チェック（ap_meets_conditions）で失敗した場合は DECLINED を返します。
  • dialup_send_pulse が SUSPENDED または DONE 以外の値を返した場合は、そのエラーコードをそのまま返します。

保守担当者向けメモ

  • 本関数は SUSPENDED ステータスを利用して非同期 I/O を実現しています。MPM の実装に依存するため、環境によっては挙動が異なる可能性があります。
  • エラーハンドリングは最小限であり、ap_pass_brigade 等の失敗時の詳細なログ出力は省略されています。
