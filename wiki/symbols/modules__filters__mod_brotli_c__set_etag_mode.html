set_etag_mode

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_brotli.c:139

目的

Apache HTTP Server の設定ファイル（httpd.conf）で指定された BrotliAlterETag ディレクティブの引数を解析し、サーバー設定構造体 brotli_server_config_t の etag_mode を更新します。

シグネチャまたは形状

static const char *set_etag_mode(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: 設定値文字列（"AddSuffix", "NoChange", "Remove" のいずれか）。
  • **出力**:
  • 成功時: NULL を返します。
  • 失敗時: エラーメッセージ文字列を返します。

前提条件と不変条件

  • cmd->server->module_config から brotli_module の設定構造体が取得可能である必要があります。
  • arg は大文字小文字を区別しない比較（ap_cstr_casecmp）で判定されます。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定構造体を取得します。 2. arg の値に応じて conf->etag_mode を以下のいずれかに設定します。

  • "AddSuffix": ETAG_MODE_ADDSUFFIX
  • "NoChange": ETAG_MODE_NOCHANGE
  • "Remove": ETAG_MODE_REMOVE

    • 3. 指定された値が上記以外の場合は、エラーメッセージを返します。 4. 正常終了時は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_brotli.c:593 において、AP_INIT_TAKE1 マクロを通じて BrotliAlterETag ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: ap_get_module_config および ap_cstr_casecmp（Apache API）を呼び出します。

関連データ構造

  • brotli_server_config_t: 設定値を保持する構造体。
  • etag_mode_e: etag_mode が取りうる値の列挙型。

失敗モードと境界条件

  • サポートされていない文字列が arg として渡された場合、エラーメッセージを返し、設定の更新は行われません。

保守担当者向けメモ

  • 本関数は BrotliAlterETag ディレクティブ専用のハンドラです。新しい ETag 処理モードを追加する場合は、列挙型 etag_mode_e の拡張と本関数の条件分岐の更新が必要です。
