ap_die_r

定義

  • 種別: function
  • 定義箇所: modules/http/http_request.c:76

目的

リクエスト処理中に発生したエラーに対するレスポンス生成を統括する。カスタムエラードキュメントへのリダイレクトや、接続のクローズ判定、プロキシ認証の調整を行う。

シグネチャまたは形状

static void ap_die_r(int type, request_rec *r, int recursive_error)

入力と出力

  • type: 発生した HTTP ステータスコード（または OK/DONE）。
  • r: 現在のリクエスト構造体。
  • recursive_error: 再帰的なエラー発生時のステータス（正常時は HTTP_OK）。
  • 出力: なし（内部的にレスポンスを送信またはリダイレクトを実行）。

前提条件と不変条件

  • type が OK または DONE の場合、ap_finalize_request_protocol(r) を呼び出して即座に終了する。
  • type が有効な HTTP レスポンスコードでない場合、ap_http_header_filter が適用済みであれば HTTP_INTERNAL_SERVER_ERROR に変換し、そうでなければ処理を中断する。

アルゴリズムまたは振る舞い

1. **エラー検証**: type が有効な HTTP ステータスか確認し、無効な場合はログ出力後に HTTP_INTERNAL_SERVER_ERROR へ昇格させる。 2. **再帰エラー処理**: recursive_error が HTTP_OK でない場合、元のリクエストまで遡り、フィルタチェーンを復元して再帰的なリダイレクトを回避する。 3. **ステータス調整**: プロキシ認証が必要な場合、ステータスを HTTP_PROXY_AUTHENTICATION_REQUIRED に変更する。 4. **接続管理**: ap_status_drops_connection に基づき、Keep-Alive の可否を判定する。 5. **リダイレクト処理**:

  • URL が指定されている場合、外部リダイレクト（Location ヘッダー付与）または内部リダイレクト（ap_internal_redirect）を実行する。
  • 内部リダイレクト時はメソッドを GET に強制し、ERROR_NOTES を設定する。

    • 6. **最終応答**: ap_send_error_response を呼び出し、エラーレスポンスを送信する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/http/http_request.c 内の複数箇所
  • modules/http2/h2_c2_filter.c
  • 呼び出し先:
  • ap_finalize_request_protocol, ap_filter_t 関連操作
  • ap_log_rerror, ap_index_of_response, ap_response_code_string
  • update_r_in_filters
  • ap_is_url, ap_internal_redirect, ap_send_error_response

関連データ構造

  • request_rec: リクエストのコンテキストおよび状態管理に使用。
  • ap_filter_t: フィルタチェーンの走査に使用。

失敗モードと境界条件

  • custom_response が不正な形式（URL でもなく / で始まらない）の場合、ログ出力後に HTTP_INTERNAL_SERVER_ERROR として再帰的に処理される。
  • 再帰エラー発生時、フィルタチェーンの復元に失敗すると予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • ap_internal_redirect を呼び出す際、メソッドを GET に強制変更するロジックが含まれているため、メソッドの変更が意図通りか注意が必要。
  • update_r_in_filters によるフィルタチェーンの書き換えは、エラー処理の整合性を保つための重要なステップである。
