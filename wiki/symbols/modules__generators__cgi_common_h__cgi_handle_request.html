cgi_handle_request

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:557

目的

HTTP リクエストのボディを読み取り、CGI スクリプトの標準入力（パイプ）へ転送します。

シグネチャまたは形状

static apr_status_t cgi_handle_request(request_rec *r, apr_file_t *script_out,
                                       apr_bucket_brigade *bb,
                                       char *logbuf, apr_size_t logbufbytes)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • script_out: スクリプトの標準入力に接続されたファイルハンドル。
  • bb: データを読み取るためのバケットブリゲード。
  • logbuf: リクエストボディの先頭部分を記録するためのバッファ（NULL 可）。
  • logbufbytes: logbuf のサイズ。
  • **出力**:
  • 戻り値: 処理成功時は APR_SUCCESS、エラー時は apr_status_t エラーコード。
  • logbuf: 読み取ったボディデータの一部が格納される。

前提条件と不変条件

  • r->input_filters からデータが取得可能であること。
  • script_out は書き込み可能な状態であること。
  • bb は呼び出し元で初期化されていること。

アルゴリズムまたは振る舞い

1. ap_get_brigade を使用してリクエストボディをループで読み取ります。 2. 各バケットを走査し、EOS（End of Stream）に達するまで以下の処理を繰り返します。

  • FLUSH バケットはスキップします。
  • apr_bucket_read でデータを取得し、logbuf が指定されていればその内容をコピーします。
  • apr_file_write_full を使用して、取得したデータを script_out へ書き込みます。

    • 3. 書き込み中にエラーが発生した場合、child_stopped_reading フラグを立ててエラーログを出力し、残りのリクエストボディを消費（破棄）し続けます。 4. 最後に logbuf を使用している場合は終端文字を付与します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:612
  • modules/generators/mod_cgid.c:1614
  • **呼び出し先**:
  • ap_get_brigade
  • apr_bucket_read
  • apr_file_write_full
  • ap_log_rerror
  • apr_brigade_cleanup

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • apr_bucket_brigade: データストリームを管理するバケットの集合。
  • apr_file_t: スクリプトの入出力パイプ。

失敗モードと境界条件

  • ap_get_brigade が失敗した場合、そのステータスを即座に返します。
  • logbuf の容量を超えるデータは切り捨てられます。

保守担当者向けメモ

  • 本関数は mod_cgi および mod_cgid の両方から利用される共通ロジックです。
  • 書き込みエラー発生時の「読み取り継続」処理は、クライアントからの接続を適切に終了させるために重要です。
