fcgi_dirconf_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:47

目的

Apache HTTP Server のディレクトリ単位における FastCGI プロキシ設定を保持する構造体です。バックエンドの種類や、環境変数の動的な修正ルールを管理します。

シグネチャまたは形状

typedef struct {
    fcgi_backend_t backend_type;
    apr_array_header_t *env_fixups;
} fcgi_dirconf_t;

入力と出力

  • **入力**: ap_get_module_config を介してリクエストごとのディレクトリ設定として取得されます。
  • **出力**: fix_cgivars 関数等で環境変数の調整に使用されるほか、設定の結合（マージ）時に利用されます。

前提条件と不変条件

  • env_fixups は apr_array_header_t 型であり、ProxyFCGISetEnvIf ディレクティブによって設定された環境変数の修正ルールを格納します。
  • backend_type は fcgi_backend_t 型で、FPM や Generic といったバックエンドの種別を識別します。

アルゴリズムまたは振る舞い

  • 設定の初期化およびマージ時に apr_pcalloc を使用してメモリが割り当てられます。
  • リクエスト処理中、ap_get_module_config(r->per_dir_config, &proxy_fcgi_module) を通じて現在のディレクトリ設定として読み込まれます。
  • fix_cgivars 関数内で、この構造体に格納された env_fixups を参照し、リクエストの環境変数を動的に書き換える処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_fcgi.c 内の各種設定ハンドラおよびリクエスト処理関数（fix_cgivars, proxy_fcgi_handler 等）。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • fcgi_backend_t: backend_type フィールドの型。
  • apr_array_header_t: env_fixups フィールドの型。
  • request_rec: 本構造体を取得するためのコンテキストとして使用されます。

失敗モードと境界条件

  • 設定が未定義の場合、ap_get_module_config は NULL を返す可能性があるため、呼び出し側での適切なチェックが必要です。
  • apr_pcalloc によるメモリ割り当て失敗時は、APR の規約に従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_proxy_fcgi モジュール固有の設定を保持します。設定項目の追加や変更を行う際は、modules/proxy/mod_proxy_fcgi.c 内のディレクトリ設定マージ関数（create_dir_config 等）の更新を忘れないでください。
