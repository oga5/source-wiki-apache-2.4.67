cmd_crit

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:184

目的

Lua スクリプトから Apache のログシステムを呼び出し、APLOG_CRIT（致命的なエラー）レベルでメッセージを出力します。

シグネチャまたは形状

static int cmd_crit(lua_State *L)

入力と出力

  • **入力**: lua_State *L（Lua の実行状態）。スタックにはログ出力対象のメッセージや情報が含まれます。
  • **出力**: int（Lua スタックへ戻す戻り値の数）。内部で呼び出す cmd_log_at の戻り値をそのまま返します。

前提条件と不変条件

  • L は有効な Lua 状態である必要があります。
  • 内部で cmd_log_at を呼び出すため、Lua スタックにはログ出力に必要なコンテキスト情報（ソースファイル名や行番号など）がスタック上に存在している必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った lua_State *L を cmd_log_at 関数に渡します。 2. その際、ログレベルとして APLOG_CRIT を指定します。 3. cmd_log_at の実行結果を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_config.c:255 にある cmd_methods テーブル内で、Lua からのメソッド呼び出し用として登録されています。
  • **呼び出し先**: cmd_log_at（同ファイル内のヘルパー関数）。

関連データ構造

  • cmd_methods: CommandParameters 用のメソッドテーブル。本関数はこのテーブルの要素として定義されています。

失敗モードと境界条件

  • Lua スタックが不正な場合、cmd_log_at 内部でエラーが発生する可能性があります。
  • ログ出力先となる Apache のログパイプラインが利用できない場合、ログは記録されません。

保守担当者向けメモ

  • 本関数は cmd_log_at のラッパーです。ログレベルの変更や共通のログ出力ロジックの修正が必要な場合は、cmd_log_at を参照してください。
  • APLOGNO マクロは意図的に省略されています。
