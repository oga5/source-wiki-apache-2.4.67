proxy_schemes_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/proxy_util.c:5122

目的

プロキシがサポートするプロトコルスキーム名と、そのスキームで利用されるデフォルトのポート番号を関連付けるためのデータ構造です。

シグネチャまたは形状

typedef struct proxy_schemes_t {
    const char *name;
    apr_port_t default_port;
} proxy_schemes_t;

入力と出力

  • 入力: スキーム名（文字列）およびデフォルトポート（apr_port_t）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • name はプロトコルスキームを表すヌル終端文字列である必要があります。
  • default_port は apr_port_t 型で定義され、当該スキームの標準的な通信ポートを保持します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行います。提供されたソースコード内では、この構造体の配列（pschemes[]）を定義し、特定のスキームに対するデフォルトポートを検索・照合するための静的テーブルとして利用されています。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/proxy_util.c 内の静的配列 pschemes の定義および、スキーム情報を走査する関数（pscheme ポインタを使用する箇所）から参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • apr_port_t: APR（Apache Portable Runtime）で定義されるポート番号型。

失敗モードと境界条件

  • 構造体定義自体に失敗モードはありません。
  • この構造体を利用した検索処理において、未知のスキームが指定された場合の挙動は、利用側の実装に依存します。

保守担当者向けメモ

  • 本構造体は proxy_util.c 内の静的テーブルを構成するために使用されています。スキームの追加やデフォルトポートの変更が必要な場合は、pschemes 配列の定義箇所を確認してください。
