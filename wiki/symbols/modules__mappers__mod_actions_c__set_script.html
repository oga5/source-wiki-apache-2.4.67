set_script

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_actions.c:110

目的

Apache 設定ファイル内の Script ディレクティブを解析し、指定された HTTP メソッドとスクリプトパスの紐付けを action_dir_config 構造体に登録します。

シグネチャまたは形状

static const char *set_script(cmd_parms *cmd, void *m_v,
                              const char *method, const char *script)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • m_v: ディレクトリごとの設定構造体 (action_dir_config) へのポインタ。
  • method: 紐付ける HTTP メソッド名。
  • script: 実行するスクリプトのパス。
  • **出力**: 成功時は NULL を返し、エラー時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • m_v は action_dir_config 型としてキャスト可能である必要があります。
  • cmd->pool が設定のメモリ管理に使用されます。

アルゴリズムまたは振る舞い

1. **メソッドの登録**:

  • .htaccess 内（cmd->pool == cmd->temp_pool）であれば、既存のメソッド番号を検索します。
  • それ以外の場合は ap_method_register を呼び出し、必要に応じて新しいメソッドを登録します。

    • 2. **バリデーション**:

  • TRACE メソッドが指定された場合はエラーを返します。
  • メソッドの登録に失敗した場合（M_INVALID）はエラーを返します。

    • 3. **設定の保存**:

  • 指定された methnum に対応する m->scripted 配列の要素に script を格納します。
  • m->configured フラグを 1 に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_actions.c:144 において、AP_INIT_TAKE2 マクロにより Script ディレクティブのハンドラとして登録されています。
  • **呼び出し先**:
  • ap_method_number_of
  • ap_method_register
  • apr_pstrcat

関連データ構造

  • action_dir_config: 設定を保持する構造体。scripted 配列を通じてメソッドとスクリプトの対応を管理します。

失敗モードと境界条件

  • TRACE メソッドの指定: 許可されていないためエラーとなります。
  • メソッド登録失敗: ap_method_register が失敗した場合、エラーメッセージを生成して終了します。

保守担当者向けメモ

  • 本関数は Script ディレクティブ専用です。Action ディレクティブによる MIME タイプベースの処理とは別のパスで管理されています。
  • .htaccess 内でのメソッド登録制限があるため、コンテキストに応じたメソッド解決ロジックが実装されています。
