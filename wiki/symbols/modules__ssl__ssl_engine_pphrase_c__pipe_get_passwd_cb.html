pipe_get_passwd_cb

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:426

目的

外部パイプ経由でパスフレーズを読み取るためのコールバック関数です。

シグネチャまたは形状

static int pipe_get_passwd_cb(char *buf, int length, char *prompt, int verify)

入力と出力

  • **入力**:
  • buf: パスフレーズを格納するバッファ。
  • length: バッファの最大長。
  • prompt: ユーザーへのプロンプト文字列。
  • verify: 検証フラグ（本関数内では未使用）。
  • **出力**:
  • 成功時: 0 を返し、buf にパスフレーズが格納される。
  • 失敗時: 1 を返す。

前提条件と不変条件

  • グローバル変数 writetty および readtty が、有効な apr_file_t として初期化されている必要があります。
  • buf は length 分のメモリ領域が確保されている必要があります。

アルゴリズムまたは振る舞い

1. prompt を writetty に出力します。 2. readtty から length バイト分を buf に読み込みます。 3. 読み込み成功後、writetty に改行コードを出力します。 4. 読み込みエラーまたは EOF の場合、buf をゼロクリアして 1 を返します。 5. 読み込んだ文字列に含まれる改行文字（\n）を除去します。 6. Windows 環境の場合、さらに復帰文字（\r）を除去します。 7. 正常終了時に 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_pphrase.c:551
  • modules/ssl/ssl_engine_pphrase.c:725
  • **呼び出し先**:
  • apr_file_puts (APR)
  • apr_file_gets (APR)
  • apr_file_eof (APR)
  • strchr (標準ライブラリ)
  • memset (標準ライブラリ)

関連データ構造

  • apr_file_t: ファイル入出力用構造体（APR）。

失敗モードと境界条件

  • apr_file_gets が APR_SUCCESS 以外を返す、または EOF に達した場合、失敗として扱われます。
  • 失敗時には memset を用いて buf をゼロクリアし、機密情報の漏洩を防ぎます。

保守担当者向けメモ

  • 本関数は ssl_pipe_child_create 等で生成されたパイプとの通信を前提としています。
  • verify 引数は現在の実装では使用されていません。
  • Windows 環境での改行コード処理（\r）が条件コンパイルで含まれています。
