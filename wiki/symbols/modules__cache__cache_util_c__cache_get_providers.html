cache_get_providers

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_util.c:184

目的

現在のリクエストに対して適用可能なキャッシュプロバイダーのリストを構築し、返却する。

シグネチャまたは形状

cache_provider_list *cache_get_providers(request_rec *r, cache_server_conf *conf)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • conf: キャッシュサーバー設定を保持する cache_server_conf 構造体。
  • **出力**:
  • 適用可能なキャッシュプロバイダーの連結リスト（cache_provider_list *）。該当するものがない場合やキャッシュが無効化されている場合は NULL を返す。

前提条件と不変条件

  • r->per_dir_config から取得されるディレクトリ設定（cache_dir_conf）が有効であること。
  • conf にはグローバルなキャッシュ設定が含まれていること。

アルゴリズムまたは振る舞い

1. **ディレクトリ単位の無効化確認**: dconf->disable が真であれば、即座に NULL を返す。 2. **パスの決定**: cache_use_early_url(r) の結果に基づき、対象パスを r->parsed_uri.path または r->uri から選択する。 3. **グローバル無効化リストの走査**: conf->cachedisable に含まれる各エントリに対し、uri_meets_conditions を用いてリクエスト URI との照合を行う。一致した場合はキャッシュ対象外として NULL を返す。 4. **ディレクトリ単位の有効化リストの走査**: dconf->cacheenable に含まれる全エントリに対し、get_provider を呼び出し、プロバイダーリストを構築する。 5. **グローバル有効化リストの走査**: conf->cacheenable に含まれる各エントリに対し、uri_meets_conditions で照合を行い、一致したものを get_provider でリストに追加する。 6. 最終的に構築されたプロバイダーリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c:106
  • modules/cache/mod_cache.c:416
  • **呼び出し先**:
  • ap_get_module_config
  • cache_use_early_url
  • uri_meets_conditions
  • get_provider

関連データ構造

  • cache_provider_list: プロバイダーの連結リスト構造体。
  • cache_server_conf: サーバー全体の設定。
  • cache_dir_conf: ディレクトリ単位の設定。
  • request_rec: Apache のリクエスト情報。

失敗モードと境界条件

  • ディレクトリ設定でキャッシュが明示的に無効化されている場合、他の設定に関わらず NULL を返す。
  • グローバルな cachedisable 設定に一致した場合、その時点で処理を中断し NULL を返す。
  • どのプロバイダーも条件に合致しない場合、空のリスト（NULL）を返す。

保守担当者向けメモ

  • 本関数は mod_cache のメイン処理フローにおいて、キャッシュの適格性を判断する重要なステップである。
  • get_provider を介してプロバイダーをリストに追加する際、メモリは r->pool から割り当てられる設計となっている。
