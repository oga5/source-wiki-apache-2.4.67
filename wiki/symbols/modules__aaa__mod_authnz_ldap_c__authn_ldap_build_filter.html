authn_ldap_build_filter

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_ldap.c:209

目的

LDAP 検索用のフィルタ文字列を動的に構築し、ユーザー入力に含まれるメタ文字をエスケープすることで、LDAP インジェクション攻撃を防止する。

シグネチャまたは形状

apr_status_t authn_ldap_build_filter(char filtbuf[FILTER_LENGTH],
                                     request_rec *r,
                                     const char *user,
                                     const char *filter,
                                     authn_ldap_config_t *sec)

入力と出力

  • **入力**:
  • filtbuf: 構築されたフィルタ文字列を格納するバッファ。
  • r: 現在のリクエストレコード。
  • user: 検索対象のユーザー名。
  • filter: 使用するベースフィルタ（NULL の場合は sec->filter を使用）。
  • sec: LDAP 設定構造体。
  • **出力**:
  • filtbuf に結果が書き込まれる。
  • 成功時は APR_SUCCESS、バッファ不足等のエラー時は APR_EGENERAL を返す。

前提条件と不変条件

  • filtbuf は FILTER_LENGTH のサイズを持つ必要がある。
  • sec->attribute が有効な LDAP 属性名として設定されていること。
  • charset_conversions が有効な場合、ユーザー名は UTF-8 に変換される。

アルゴリズムまたは振る舞い

1. **フィルタ初期化**: filter 引数が指定されていない場合、sec->filter を使用する。 2. **文字コード変換**: charset_conversions が有効な場合、get_conv_set を用いてユーザー名を UTF-8 に変換する。 3. **フィルタ構築**:

  • filter が "none" または空の場合、(attribute=user) 形式のフィルタを作成する。
  • それ以外の場合、(&(filter)(attribute=user)) 形式のフィルタを作成する。

    • 4. **エスケープ処理**: ユーザー名に含まれる *, (, ), \ を LDAP 準拠の形式にエスケープする。APR_HAS_MICROSOFT_LDAPSDK が定義されている場合は数値エスケープを行い、それ以外はバックスラッシュによるエスケープを行う。 5. **終了処理**: フィルタの末尾に閉じ括弧を付与し、ヌル終端する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_ldap.c 内の各認証・認可フェーズ（行 551, 704, 893, 1087, 1214, 1347, 1392）。
  • **呼び出し先**:
  • get_conv_set (文字コード変換用)
  • apr_pcalloc, apr_snprintf, apr_xlate_conv_buffer (APR ユーティリティ)

関連データ構造

  • authn_ldap_config_t: LDAP 設定（属性名やベースフィルタを保持）。
  • request_rec: リクエストコンテキスト。

失敗モードと境界条件

  • **バッファオーバーラン**: フィルタ構築中に FILTER_LENGTH を超える場合、APR_EGENERAL を返し処理を中断する。
  • **文字コード変換失敗**: apr_xlate_conv_buffer が失敗した場合、変換前のユーザー名がそのまま使用される。

保守担当者向けメモ

  • 本関数は LDAP インジェクションを防ぐための重要なセキュリティ境界である。エスケープ対象文字の追加や変更を行う際は、APR_HAS_MICROSOFT_LDAPSDK の条件分岐に注意すること。
  • FILTER_LENGTH の定義値が、想定される最大フィルタ長に対して十分であることを確認すること。
