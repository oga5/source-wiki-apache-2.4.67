md_json_destroy

定義

  • 種別: function
  • 定義箇所: modules/md/md_json.c:102

目的

md_json_t 構造体が保持する JSON オブジェクトの参照カウントを減らし、メモリリソースを解放する。

シグネチャまたは形状

void md_json_destroy(md_json_t *json)

入力と出力

  • **入力**: md_json_t *json - 破棄対象の JSON ラッパー構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • json が指す構造体内の j（json_t）は、有効な参照カウント（refcount > 0）を保持している必要がある。
  • 実行後、json->j は NULL に設定される。

アルゴリズムまたは振る舞い

1. json ポインタおよび json->j が NULL でないことを確認する。 2. json->j->refcount > 0 であることを assert で検証する。 3. json_decref(json->j) を呼び出し、Jansson ライブラリ側の参照カウントをデクリメントする。 4. json->j を NULL に更新し、二重解放を防止する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_json.c:68 での呼び出しが確認されている。
  • **呼び出し先**: Jansson ライブラリの json_decref を呼び出す。

関連データ構造

  • md_json_t: json_t をラップし、APR プールとのライフサイクル管理を行う構造体。

失敗モードと境界条件

  • json が NULL の場合、または json->j が既に NULL の場合は何も行わない。
  • json->j->refcount が 0 以下の状態で呼び出された場合、assert により異常終了する。

保守担当者向けメモ

  • 本関数は md_json_t の所有する json_t の参照カウントを操作する。APR プールによる自動クリーンアップと併用される設計であるため、手動で呼び出す際は所有権の移動に注意すること。
