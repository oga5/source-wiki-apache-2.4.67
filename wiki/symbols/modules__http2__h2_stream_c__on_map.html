on_map

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_stream.c:132

目的

HTTP/2 ストリームの状態遷移テーブルをルックアップし、現在の状態とイベントに基づいて次の状態を決定します。

シグネチャまたは形状

static int on_map(h2_stream_state_t state, int map[H2_SS_MAX])

入力と出力

  • **入力**:
  • state: 現在のストリーム状態 (h2_stream_state_t)。
  • map: 状態遷移を定義する整数配列 (H2_SS_MAX サイズ)。
  • **出力**:
  • 遷移後の新しい状態を示す整数値を返します。

前提条件と不変条件

  • map 配列は、state をインデックスとしてアクセス可能な有効なメモリ領域である必要があります。
  • map 内の値は、S_XXX（エラー）、S_ERR、S_NOP、または有効な状態遷移先として解釈される数値である必要があります。

アルゴリズムまたは振る舞い

1. map[state] から操作値 op を取得します。 2. op の値に応じて以下の処理を行います：

  • S_XXX または S_ERR の場合: op をそのまま返します。
  • S_NOP の場合: 現在の state をそのまま返します。
  • それ以外の場合: op - 1 を計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_stream.c:154
  • modules/http2/h2_stream.c:174
  • **呼び出し先**:
  • 本関数から直接呼び出している関数はありません。

関連データ構造

  • h2_stream_state_t: ストリームの状態定義。
  • H2_SS_MAX: 状態配列の最大サイズ。

失敗モードと境界条件

  • state が H2_SS_MAX の範囲外である場合、配列外アクセスが発生する可能性があります。
  • map に不正な値が含まれている場合、意図しない状態遷移先が返される可能性があります。

保守担当者向けメモ

  • 本関数は状態遷移ロジックの共通部品です。遷移テーブル（frame_map や trans_on_event）の定義内容が、本関数の switch 文の挙動と整合していることを確認してください。
