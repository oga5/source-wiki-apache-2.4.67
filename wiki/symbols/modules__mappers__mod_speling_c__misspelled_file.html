misspelled_file

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_speling.c:139

目的

スペルチェック機能において、ファイルシステムから検出された候補ファイル名とその一致品質を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    const char *name;
    sp_reason quality;
} misspelled_file;

入力と出力

  • name: 候補となるファイル名（文字列）。
  • quality: sp_reason 型の列挙値。spdist() 関数によって算出された、リクエストされたパスとの類似度や差異の性質を保持します。

前提条件と不変条件

  • apr_array_header_t 配列の要素として格納されることを前提としています。
  • quality フィールドは、sort_by_quality 関数内で比較対象として使用されるため、一貫した品質評価値が格納されている必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用ですが、以下の処理フローで利用されます。 1. check_speling() 内でディレクトリ走査時にインスタンス化されます。 2. apr_array_push() を通じて候補リストに追加されます。 3. sort_by_quality() 関数において、quality フィールドに基づき候補リストのソートが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_speling.c 内の check_speling() 関数が、候補の蓄積およびソート時に本構造体を参照・操作します。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • sp_reason: quality フィールドの型として使用される列挙型。
  • apr_array_header_t: 複数の misspelled_file インスタンスを管理するために使用される APR 配列。

失敗モードと境界条件

  • メモリ不足時: apr_array_push() が失敗する可能性があります。
  • 候補なし: candidates 配列が空の状態で sort_by_quality が呼び出されないよう、呼び出し側で制御する必要があります。

保守担当者向けメモ

  • name フィールドは r->pool から割り当てられたメモリを指すため、リクエスト終了時に自動的に解放されます。
  • quality の比較ロジックを変更する場合は、sort_by_quality 関数と sp_reason の定義を併せて確認してください。
