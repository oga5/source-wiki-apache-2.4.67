session_dbd_save

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_dbd.c:404

目的

セッションデータをデータベース（DBD）に永続化し、必要に応じてクライアントへクッキーを書き出す。

シグネチャまたは形状

apr_status_t session_dbd_save(request_rec *r, session_rec *z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報。
  • z: 保存対象のセッションデータ（session_rec）。
  • **出力**:
  • apr_status_t: 処理結果。成功時は OK または APR_SUCCESS、失敗時はエラーコード、対象外の場合は DECLINED を返す。

前提条件と不変条件

  • mod_dbd がロードされ、データベース接続が利用可能であること。
  • r->per_dir_config から取得した session_dbd_dir_conf 設定に基づき動作する。

アルゴリズムまたは振る舞い

1. **匿名セッション処理**:

  • 設定でクッキー名が指定されている場合、UUIDをキーとして使用。
  • セッションが新規または変更（dirty）されている場合、新しいUUIDを生成。
  • z->encoded が存在すれば dbd_save で保存、なければ dbd_remove で削除。
  • 設定に応じて ap_cookie_write / ap_cookie_write2 でクッキーをレスポンスヘッダに追加。

    • 2. **ユーザー紐付けセッション処理**:

  • peruser 設定が有効な場合、Cache-Control に no-cache, private を付与。
  • r->user が存在する場合のみ dbd_save を実行。ユーザー未ログイン時はエラーログを出力し保存を中止。

    • 3. **終了**: 処理が適用された場合は OK を返し、いずれの条件にも合致しない場合は DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_session_save に登録されており、セッション保存フェーズで呼び出される。
  • **呼び出し先**:
  • dbd_save, dbd_remove: データベース操作を実行。
  • ap_cookie_write, ap_cookie_write2: クッキーの書き出し。
  • ap_log_rerror: エラーログ出力。

関連データ構造

  • session_rec: セッション内容、UUID、有効期限、エンコード済みデータを保持。
  • session_dbd_dir_conf: SQLクエリラベルやクッキー属性などのディレクトリ設定。

失敗モードと境界条件

  • **データベースエラー**: dbd_save または dbd_remove が失敗した場合、そのステータスをそのまま返す。
  • **未ログイン**: peruser 設定時に r->user が空の場合、セッションは保存されずエラーログが記録される。
  • **メモリ不足**: apr_pcalloc 等のメモリ割り当て失敗時は APR の挙動に従う。

保守担当者向けメモ

  • 本関数は mod_session の保存フックとして動作する。
  • 匿名セッションの UUID 生成およびクッキー管理と、ユーザー紐付けセッションのロジックが分岐している点に注意。
