log_bytes_in

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_logio.c:100

目的

現在の接続における受信バイト数を取得し、ログ出力用の文字列として返します。

シグネチャまたは形状

static const char *log_bytes_in(request_rec *r, char *a)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • a: ログフォーマットの引数（本関数では未使用）。
  • **出力**:
  • 接続の受信バイト数（cf->bytes_in）を文字列に変換したものを返します。

前提条件と不変条件

  • r->connection->conn_config に logio_module に関連付けられた logio_config_t 構造体が格納されている必要があります。
  • apr_off_t_toa を使用してメモリプール r->pool 上に文字列が生成されます。

アルゴリズムまたは振る舞い

1. ap_get_module_config を呼び出し、現在の接続設定から logio_config_t 構造体を取得します。 2. 取得した構造体内の bytes_in フィールド（受信バイト数）を参照します。 3. apr_off_t_toa を用いて、数値を文字列に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_logio.c:197 にて log_pfn_register を通じてログフォーマットトークン "I" に登録されています。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core API)
  • apr_off_t_toa (APR API)

関連データ構造

  • logio_config_t: 接続ごとの入出力統計を保持する構造体。
  • request_rec: リクエスト情報および接続情報へのアクセスに使用。

失敗モードと境界条件

  • conn_config から logio_module の設定が取得できない場合、cf が NULL となり、その後の cf->bytes_in 参照でセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は mod_log_config が提供するログフォーマットのハンドラとして機能します。
  • 戻り値の文字列はリクエストのメモリプール (r->pool) 上に確保されるため、リクエスト終了時に自動的に解放されます。
