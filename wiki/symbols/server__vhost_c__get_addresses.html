get_addresses

定義

  • 種別: function
  • 定義箇所: server/vhost.c:148

目的

設定ファイル内の仮想ホスト定義からアドレスとポートを解析し、server_addr_rec 構造体のリストを構築します。

シグネチャまたは形状

static const char *get_addresses(apr_pool_t *p, const char *w_,
                                 server_addr_rec ***paddr,
                                 apr_port_t default_port)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • w_: 解析対象の文字列（例: "127.0.0.1:80"）。
  • paddr: server_addr_rec リストの末尾を指すポインタのポインタ。
  • default_port: ホスト指定のみの場合に使用するデフォルトポート。
  • **出力**:
  • 成功時は NULL を返し、paddr が指すリストに解析結果を追加。
  • 失敗時はエラーメッセージ文字列を返却。

前提条件と不変条件

  • w_ が空文字列の場合は即座に NULL を返す。
  • apr_pool_t を介してメモリが割り当てられる。
  • paddr はリストの末尾を保持し、呼び出しごとに更新される。

アルゴリズムまたは振る舞い

1. 文字列末尾の :* を検出し、ワイルドカードポートとしてフラグを立てる。 2. apr_parse_addr_port を使用してホスト名、スコープID、ポートを分離。 3. * または _default_ の場合はワイルドカードアドレスとして解決。それ以外は apr_sockaddr_info_get で名前解決を実行。 4. 解決されたアドレスが複数ある場合（DNS ラウンドロビン等）、do-while ループで全アドレスを server_addr_rec 構造体に格納し、リストを連結する。 5. APR バージョン 1.7.0 未満の場合、スコープIDが含まれているとエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/vhost.c:252 (設定解析プロセス内)
  • **呼び出し先**: apr_pstrmemdup, apr_parse_addr_port, apr_sockaddr_info_get, apr_pcalloc, ap_log_error (APR および Apache HTTPD ユーティリティ)

関連データ構造

  • server_addr_rec: 仮想ホストのアドレス情報を保持する構造体。
  • apr_sockaddr_t: APR のソケットアドレス構造体。

失敗モードと境界条件

  • アドレスやポートの形式が不正な場合、エラーメッセージを返す。
  • ホスト名の解決に失敗した場合、ログを出力して NULL を返す（無視される）。
  • scope_id が指定されているが、環境（APR バージョン）がサポートしていない場合はエラーを返す。

保守担当者向けメモ

  • 本関数は設定読み込み時にのみ実行されるため、実行時のパフォーマンス影響は限定的です。
  • apr_sockaddr_info_get の結果がリスト形式で返されるため、do-while ループで全ノードを走査して server_addr_rec に変換する設計になっています。
