stapling_get_certinfo

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_stapling.c:244

目的

グローバルな証明書情報ハッシュテーブルから、指定されたインデックス（証明書ハッシュ）に対応する certinfo 構造体を取得します。

シグネチャまたは形状

certinfo *stapling_get_certinfo(server_rec *s, UCHAR *idx, apr_size_t idx_len,
                                modssl_ctx_t *mctx, SSL *ssl)

入力と出力

  • **入力**:
  • s: サーバーレコード。ログ出力に使用。
  • idx: 証明書を識別するハッシュ値（バイト列）。
  • idx_len: idx の長さ。
  • mctx: SSL コンテキスト（未使用）。
  • ssl: SSL オブジェクト（未使用）。
  • **出力**:
  • 成功時: 対応する certinfo 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • stapling_certinfo ハッシュテーブルが初期化済みであること。
  • 返される certinfo は cid（OCSP CERTID）を保持していることが保証されます。

アルゴリズムまたは振る舞い

1. stapling_certinfo ハッシュテーブルから idx をキーとして certinfo を検索します。 2. 取得した cinf が存在し、かつ cinf->cid が有効であれば、そのポインタを返します。 3. 上記条件を満たさない場合、APLOG_INFO レベルで「stapling not supported for certificate」というエラーログを出力し、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_util_stapling.c:855
  • **呼び出し先**:
  • apr_hash_get (APR ライブラリ関数)
  • ap_log_error (Apache HTTP Server API)

関連データ構造

  • certinfo: 証明書ハッシュ、OCSP CERTID、レスポンダー URI を保持する構造体。
  • stapling_certinfo: 証明書情報を保持するグローバルハッシュテーブル。

失敗モードと境界条件

  • 指定された idx がハッシュテーブルに存在しない場合、または certinfo は存在するが cid が設定されていない場合、NULL を返しログを出力します。

保守担当者向けメモ

  • 本関数は mctx および ssl 引数を受け取りますが、現在の実装では使用されていません。将来的な拡張やデバッグ目的で保持されています。
  • ログ出力には APLOGNO(01926) が割り当てられています。
