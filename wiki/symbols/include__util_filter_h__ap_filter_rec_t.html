ap_filter_rec_t

定義

  • 種別: struct
  • 定義箇所: include/util_filter.h:199

目的

Apache HTTP Server におけるフィルタの定義情報を保持する構造体です。フィルタの名前、実行関数、型、およびプロトコル制約などのメタデータを管理し、フィルタチェーンへの登録と識別を可能にします。

シグネチャまたは形状

struct ap_filter_rec_t {
    /* 内部構造体メンバ */
    struct ap_filter_rec_t *next;
    /* その他、フィルタ名、関数ポインタ、型情報等を含む */
};

※ソース抜粋に基づき、next ポインタを含む構造体として定義されています。

入力と出力

本構造体自体はデータ処理の入出力を行いません。フィルタの登録時にシステムへ提供され、フィルタチェーン構築時の参照情報として利用されます。

前提条件と不変条件

  • フィルタは ap_filter_type に基づいてソートされます。
  • AP_FTYPE_CONNECTION 以上のフィルタはサブリクエスト内では使用できません。

アルゴリズムまたは振る舞い

  • フィルタチェーン内でのフィルタの順序付けや、特定のフィルタハンドルの検索に使用されます。
  • next メンバにより、登録済みフィルタのリスト構造を形成します。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_register_input_filter や ap_register_output_filter_protocol 等の登録関数により初期化・参照されます。
  • **呼び出し先**: フィルタチェーンの実行時に、本構造体が保持する関数ポインタが呼び出されます。
  • **参照元**: http_core.h や mod_core.h などで、ap_core_output_filter_handle や ap_http_input_filter_handle といったグローバルなフィルタハンドルとして外部公開されています。

関連データ構造

  • ap_filter_t: インストールされたフィルタのインスタンス構造体。
  • ap_filter_type: フィルタの優先順位を決定する分類。
  • apr_bucket_brigade: フィルタ間でのデータ受け渡しに使用されるデータ構造。

失敗モードと境界条件

  • フィルタの登録順序や型定義が不適切な場合、フィルタチェーンの実行順序が意図した通りにならない可能性があります。
  • フィルタハンドルが正しく初期化されていない状態でチェーンに追加しようとした場合の挙動については、本ソース抜粋からは特定できません。

保守担当者向けメモ

  • 本構造体は server/util_filter.c で管理されるフィルタチェーンの基盤です。
  • フィルタの追加や削除を行う際は、include/util_filter.h で定義されたインターフェースを介して行ってください。
  • フィルタの具体的な実装（mod_deflate 等）は本構造体を参照して自身の動作を定義します。
