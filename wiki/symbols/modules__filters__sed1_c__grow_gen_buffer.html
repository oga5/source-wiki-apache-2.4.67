grow_gen_buffer

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:151

目的

sed_eval_t コンテキスト内の汎用バッファ（genbuf）を、指定されたサイズまで動的に拡張します。

シグネチャまたは形状

apr_status_t grow_gen_buffer(sed_eval_t *eval, apr_size_t newsize, char **gspend)

入力と出力

  • **入力**:
  • eval: 評価状態を保持する sed_eval_t 構造体へのポインタ。
  • newsize: 確保すべきバッファの最小サイズ。
  • gspend: バッファ内の現在位置を示すポインタの参照。NULL の場合は eval->genbuf が使用されます。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • 失敗時は grow_buffer が返すエラーコード（メモリ不足等）を返します。

前提条件と不変条件

  • eval->pool は有効な apr_pool_t である必要があります。
  • 成功時、eval->lcomend は eval->genbuf のオフセット 71 の位置に更新されます。

アルゴリズムまたは振る舞い

1. gspend が NULL の場合、eval->genbuf を指すように設定します。 2. grow_buffer を呼び出し、eval->genbuf を newsize に合わせて拡張します。 3. grow_buffer が成功した場合、eval->lcomend を &eval->genbuf[71] に再設定します。 4. 最終的なステータスコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed1.c:149 (定義直前での参照)
  • modules/filters/sed1.c:262
  • modules/filters/sed1.c:282
  • modules/filters/sed1.c:728
  • modules/filters/sed1.c:754
  • **呼び出し先**:
  • grow_buffer (内部ヘルパー)

関連データ構造

  • sed_eval_t: 評価コンテキスト。genbuf（汎用バッファ）、gsize（バッファサイズ）、pool（メモリプール）を保持します。

失敗モードと境界条件

  • grow_buffer の制約により、MAX_BUF_SIZE（8MB）を超える拡張要求は APR_ENOMEM を返します。
  • メモリ確保に失敗した場合、grow_buffer から返されるエラーコードがそのまま伝播されます。

保守担当者向けメモ

  • 本関数は sed1.c 内のバッファ管理ヘルパーの一つです。
  • eval->lcomend の固定オフセット更新は、特定のコマンド処理におけるポインタ整合性を維持するためのものです。変更時は影響範囲に注意してください。
