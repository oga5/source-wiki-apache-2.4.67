ap_logio_get_last_bytes

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_logio.c:89

目的

接続（conn_rec）に関連付けられた直近のリクエストにおける入出力バイト数を取得します。

シグネチャまたは形状

apr_off_t ap_logio_get_last_bytes(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c - 統計情報を取得する対象の接続構造体。
  • **出力**: apr_off_t - 直近のリクエストで処理されたバイト数。

前提条件と不変条件

  • c->conn_config に logio_module の設定（logio_config_t）が格納されている必要があります。
  • logio_config_t 構造体内の bytes_last_request フィールドが、リクエスト終了時に適切に更新されていることを前提とします。

アルゴリズムまたは振る舞い

1. 指定された接続構造体 c から ap_get_module_config を使用して logio_config_t 構造体を取得します。 2. 取得した構造体内の bytes_last_request メンバの値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/scoreboard.c 内で APR_RETRIEVE_OPTIONAL_FN を介して取得および参照されています。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core API) を呼び出します。

関連データ構造

  • logio_config_t: 接続ごとの統計情報を保持する構造体。
  • conn_rec: Apache の接続状態を保持するコア構造体。

失敗モードと境界条件

  • c が NULL の場合、または c->conn_config に logio_module の設定が紐付いていない場合、ap_get_module_config の挙動に依存し、予期せぬ動作やクラッシュを引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は APR_DECLARE_OPTIONAL_FN として公開されており、他のモジュールから動的にリンク可能です。
  • mod_logio.c 内で APR_REGISTER_OPTIONAL_FN(ap_logio_get_last_bytes) により登録されています。
  • 変更を加える際は、include/http_core.h で定義されているオプション関数のシグネチャと整合性を保つ必要があります。
