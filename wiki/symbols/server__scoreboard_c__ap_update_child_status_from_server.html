ap_update_child_status_from_server

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:613

目的

スコアボードハンドル（ap_sb_handle_t）を使用して、特定のワーカープロセスの状態を更新します。

シグネチャまたは形状

int ap_update_child_status_from_server(ap_sb_handle_t *sbh, int status, conn_rec *c, server_rec *s)

入力と出力

  • **入力**:
  • sbh: 更新対象のワーカーを特定するスコアボードハンドル。
  • status: 設定する新しい状態（例: SERVER_BUSY_READ）。
  • c: 接続情報（conn_rec）。
  • s: サーバー情報（server_rec）。
  • **出力**:
  • 成功時は update_child_status_internal の戻り値を返し、失敗時は -1 を返します。

前提条件と不変条件

  • sbh が有効であること。
  • sbh->child_num が 0 以上であること。
  • スコアボードの各スロットは単一の所有者によって書き込まれるため、排他制御は不要です。

アルゴリズムまたは振る舞い

1. sbh が NULL であるか、sbh->child_num が負の値であれば、更新を行わず -1 を返します。 2. 上記のチェックを通過した場合、sbh から抽出した child_num および thread_num を使用して、内部関数 update_child_status_internal を呼び出します。 3. update_child_status_internal には、request_rec および pool 引数として NULL を渡します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:762
  • modules/ssl/ssl_engine_kernel.c:2408
  • **呼び出し先**:
  • update_child_status_internal (同一ファイル内)

関連データ構造

  • ap_sb_handle_t: スコアボード上のワーカー位置を特定するハンドル。
  • conn_rec: 接続状態を保持する構造体。
  • server_rec: サーバー設定を保持する構造体。

失敗モードと境界条件

  • sbh が NULL の場合、または sbh->child_num が負の場合、関数は即座に -1 を返して終了します。

保守担当者向けメモ

  • 本関数は update_child_status_internal のラッパーです。詳細な状態更新ロジックは同内部関数に依存しています。
  • ap_extended_status が有効な場合、内部でリクエスト文字列のコピー等が発生するため、高負荷時の呼び出しには注意が必要です。
