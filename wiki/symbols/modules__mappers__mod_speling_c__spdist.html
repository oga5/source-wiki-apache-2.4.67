spdist

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_speling.c:163

目的

2つの文字列を比較し、スペルミスや大文字小文字の不一致の度合いを判定して、修正の可能性（sp_reason）を返します。

シグネチャまたは形状

static sp_reason spdist(const char *s, const char *t)

入力と出力

  • **入力**: 比較対象となる2つの文字列 s（リクエストされたパス）と t（ファイルシステム上の候補名）。
  • **出力**: sp_reason 型の列挙値。一致の度合い（大文字小文字の違い、転置、1文字の誤り、余分な文字、欠落文字、または修正不能）を返します。

前提条件と不変条件

  • 大文字小文字を区別しない比較を行います。
  • 編集距離が「1文字の誤り」の範囲内である場合にのみ、具体的な修正理由を返します。

アルゴリズムまたは振る舞い

1. **先頭一致のスキップ**: 両文字列の先頭から大文字小文字を無視して一致する部分をスキップします。 2. **完全一致判定**: 文字列の終端まで一致した場合、SP_MISCAPITALIZED を返します。 3. **編集距離の評価**:

  • **転置**: 隣接する2文字が入れ替わっている場合、SP_TRANSPOSITION を返します。
  • **単純ミス**: 1文字のみが異なる場合、SP_SIMPLETYPO を返します。
  • **余分な文字**: s に余分な文字がある場合、SP_EXTRACHAR を返します。
  • **欠落文字**: t に文字が不足している場合、SP_MISSINGCHAR を返します。

    • 4. **デフォルト**: 上記のいずれにも該当しない場合、SP_VERYDIFFERENT を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_speling.c:309（check_speling 関数内）。ディレクトリ走査時にファイル名とリクエストパスを比較するために使用されます。
  • **呼び出し先**: apr_tolower（APRライブラリ）、strcasecmp（標準Cライブラリ）。

関連データ構造

  • sp_reason: 判定結果を保持する列挙型。

失敗モードと境界条件

  • 編集距離が1文字を超える差異がある場合、SP_VERYDIFFERENT を返します。
  • 文字列が空の場合や、極端に短い場合の挙動は strcasecmp およびポインタ演算の範囲内に依存します。

保守担当者向けメモ

  • 本関数は Kernighan & Pike のアルゴリズムをベースに、大文字小文字を無視するように拡張されています。
  • 1文字の誤りまでしか許容しないため、複雑なスペルミスには対応していません。
