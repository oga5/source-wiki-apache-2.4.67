worker_args_t

定義

  • 種別: struct
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2_child.c:56

目的

子プロセス内において、メインスレッドが受け入れた接続情報をワーカー（worker_main）へ伝達するためのデータコンテナです。

シグネチャまたは形状

typedef struct {
    apr_pool_t *pconn;
    apr_socket_t *conn_sd;
} worker_args_t;

入力と出力

  • **入力**: 接続受け入れ時に apr_palloc で割り当てられ、pconn および conn_sd が設定されます。
  • **出力**: OS/2 のキュー（workq）を介してワーカーへ渡され、ワーカー側で接続処理に使用されます。

前提条件と不変条件

  • pconn は接続の生存期間を管理するメモリプールである必要があります。
  • conn_sd は有効な接続済みソケットである必要があります。

アルゴリズムまたは振る舞い

1. メインスレッドが接続を受け入れると、apr_palloc を用いて worker_args_t 構造体をメモリ上に確保します。 2. 構造体に接続情報（プールとソケット）を格納します。 3. DosWriteQueue を呼び出し、WORKTYPE_CONN 型のデータとしてキューに投入します。 4. ワーカー側で DosReadQueue を通じて本構造体を受け取り、接続処理を開始します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/mpmt_os2/mpmt_os2_child.c 内のメイン接続受け入れループ（DosWriteQueue の引数として使用）。
  • **呼び出し先**: worker_main（キューから読み出された後、接続処理の引数として利用）。

関連データ構造

  • apr_pool_t: 接続ごとのメモリ管理に使用。
  • apr_socket_t: 処理対象のネットワークソケット。
  • workq: OS/2 ネイティブの FIFO キュー。

失敗モードと境界条件

  • メモリ不足により apr_palloc が失敗した場合、接続処理が中断されます。
  • キューが満杯の場合や DosWriteQueue が失敗した場合、接続の受け渡しが正常に行われません。

保守担当者向けメモ

  • 本構造体は OS/2 のプロセス内通信（キュー）に依存しています。
  • 接続終了後、pconn プールが破棄されることで関連リソースが解放される設計です。
