decrypt_string

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:344

目的

Base64 エンコードされた暗号化セッション文字列を復号し、元のデータに変換する。複数のパスフレーズを順次試行することで、キーローテーションをサポートする。

シグネチャまたは形状

apr_status_t decrypt_string(request_rec *r, const apr_crypto_t *f,
                            session_crypto_dir_conf *dconf, const char *in,
                            char **out)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • f: apr_crypto コンテキスト。
  • dconf: 暗号化設定（パスフレーズリストを含む）。
  • in: Base64 エンコードされた入力文字列。
  • **出力**:
  • out: 復号された文字列（成功時）。
  • 戻り値: 成功時は APR_SUCCESS、失敗時はエラーコード（APR_ECRYPT 等）。

前提条件と不変条件

  • in は [MAC][Salt][IV][Ciphertext] の形式で構成されている必要がある。
  • dconf->passphrases に少なくとも1つのパスフレーズが登録されていること。
  • メモリ割り当ては r->pool を使用する。

アルゴリズムまたは振る舞い

1. 入力文字列を Base64 デコードする。 2. デコード後のデータ長が AP_SIPHASH_DSIZE（MACサイズ）と apr_uuid_t（ソルトサイズ）の合計未満であればエラーとする。 3. dconf 内のパスフレーズを順次ループする。 4. compute_auth を用いて MAC を計算し、ap_memeq_timingsafe でペイロード内の MAC と比較する。 5. MAC が一致した場合、apr_crypto_passphrase で鍵を生成し、apr_crypto_block_decrypt を実行して復号する。 6. 復号成功後、apr_crypto_block_decrypt_finish でパディングを処理し、ループを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_crypto.c:526（session_crypto_decode 内）
  • **呼び出し先**: crypt_init, compute_auth, ap_memeq_timingsafe, apr_crypto_passphrase, apr_crypto_block_decrypt_init, apr_crypto_block_decrypt, apr_crypto_block_decrypt_finish

関連データ構造

  • session_crypto_dir_conf: パスフレーズリストを保持。
  • apr_crypto_t: 暗号化ライブラリのコンテキスト。
  • request_rec: メモリプール管理に使用。

失敗モードと境界条件

  • **MAC 不一致**: ap_memeq_timingsafe が失敗した場合、そのパスフレーズをスキップして次を試行する。
  • **復号失敗**: apr_crypto_block_decrypt 等の各ステップでエラーが発生した場合、ログを出力し、次のパスフレーズを試行する。
  • **全パスフレーズ試行失敗**: 最終的に APR_ECRYPT を返す。
  • **入力長不足**: 最小サイズを満たさない場合は即座にエラーを返す。

保守担当者向けメモ

  • ap_memeq_timingsafe を使用しているため、タイミング攻撃に対する耐性が確保されている。
  • 複数のパスフレーズを試行する設計のため、古いパスフレーズをリストの末尾に追加することでキーローテーションが可能である。
  • 復号されたデータは r->pool に割り当てられるため、リクエスト終了時に自動的に解放される。
