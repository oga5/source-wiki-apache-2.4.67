parse_otherName_value

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ssl.c:263

目的

X.509 証明書の otherName フィールドに含まれる特定の値を解析し、Apache のメモリプールに割り当てられた文字列配列へ抽出・格納します。

シグネチャまたは形状

static void parse_otherName_value(apr_pool_t *p, ASN1_TYPE *value,
                                  const char *onf, apr_array_header_t **entries)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t。
  • value: 解析対象の ASN1_TYPE 構造体。
  • onf: otherName の形式を示す識別子（例: "msUPN", "id-on-dnsSRV"）。
  • entries: 抽出された文字列を格納する apr_array_header_t へのポインタ。
  • 出力: なし（entries が指す配列を直接更新）。

前提条件と不変条件

  • value が NULL、onf から導出される NID が不明、または entries が NULL の場合は何も行わずに終了します。
  • 抽出された文字列は apr_pool_t の生存期間に依存します。

アルゴリズムまたは振る舞い

1. onf を OBJ_txt2nid で NID に変換します。 2. 識別子に応じて以下の形式を処理します：

  • msUPN (NID_ms_upn): V_ASN1_UTF8STRING 型の値を UTF-8 文字列として抽出。
  • id-on-dnsSRV: V_ASN1_IA5STRING 型の値を UTF-8 文字列として抽出。

    • 3. 抽出に成功した場合、APR_ARRAY_PUSH を使用して entries 配列に追加します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ssl/ssl_util_ssl.c:333 (証明書内の otherName を走査する過程で呼び出されます)。
  • 呼び出し先: OBJ_txt2nid, asn1_string_to_utf8, APR_ARRAY_PUSH。

関連データ構造

  • ASN1_TYPE: OpenSSL の ASN.1 型定義。
  • apr_array_header_t: Apache Portable Runtime の動的配列。

失敗モードと境界条件

  • サポートされていない onf 識別子が渡された場合、または value の ASN.1 型が期待と異なる場合は、配列への追加は行われません。
  • asn1_string_to_utf8 が失敗（NULL を返却）した場合、配列には追加されません。

保守担当者向けメモ

  • 現在サポートされている形式は "msUPN" と "id-on-dnsSRV" のみです。新しい otherName 形式を追加する場合は、条件分岐を追加してください。
  • strEQ マクロを使用して文字列比較を行っているため、onf 引数の値には注意してください。
