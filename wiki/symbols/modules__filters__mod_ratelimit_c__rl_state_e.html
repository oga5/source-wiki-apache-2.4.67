rl_state_e

定義

  • 種別: enum
  • 定義箇所: modules/filters/mod_ratelimit.c:27

目的

mod_ratelimit フィルタにおける現在の転送制御モードを識別するために使用されます。

シグネチャまたは形状

typedef enum rl_state_e
{
    RATE_LIMIT,
    RATE_FULLSPEED
} rl_state_e;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: RATE_LIMIT または RATE_FULLSPEED のいずれかの状態値

前提条件と不変条件

  • rl_ctx_t コンテキスト内の状態変数として保持されます。
  • RATE_LIMIT モード時は、データ転送に制限が適用されます。
  • RATE_FULLSPEED モード時は、制限を適用せずデータを通過させます。

アルゴリズムまたは振る舞い

  • RATE_LIMIT: RL_START バケットが検出されるまで、またはデフォルトで適用されるモード。データをチャンク単位に分割し、apr_sleep を用いて転送速度を調整します。
  • RATE_FULLSPEED: RL_END バケットが検出されるまで、データを制限なしで後続のフィルタへ渡します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_ratelimit.c 内の rl_ctx_t 構造体定義（39行目）にて、メンバ変数 state の型として参照されています。
  • 呼び出し先: なし（列挙型のため、直接的な関数呼び出しは発生しません）。

関連データ構造

  • rl_ctx_t: 本列挙型をメンバとして保持し、フィルタの実行状態を管理します。

失敗モードと境界条件

  • 該当なし（列挙型定義のため、本シンボル自体に失敗モードはありません）。

保守担当者向けメモ

  • 本列挙型は mod_ratelimit の内部状態管理に特化しています。新しい転送モードを追加する場合は、rate_limit_filter 関数内の状態遷移ロジックを更新する必要があります。
