get_url_scheme

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_http.c:34

目的

入力されたURL文字列からプロトコルスキーム（http, https, ws, wss）を判定し、URLポインタをスキーム部分の直後まで進めるとともに、SSL使用の有無をフラグとして設定する。

シグネチャまたは形状

static const char *get_url_scheme(const char **url, int *is_ssl)

入力と出力

  • **入力**:
  • url: 解析対象のURL文字列へのポインタ。解析後、スキーム部分（例: http://）を除いた残りの文字列を指すように更新される。
  • is_ssl: SSL/TLSを使用するスキーム（https, wss）の場合に 1、それ以外（http, ws）の場合に 0 が書き込まれる出力用整数ポインタ。
  • **出力**:
  • 判定されたスキーム名（"http", "https", "ws", "wss"）の文字列リテラルを返す。スキームが特定できない場合は NULL を返す。

前提条件と不変条件

  • url および is_ssl は有効なメモリ領域を指している必要がある。
  • url が指す文字列は、スキーム判定に必要な長さ（最低3〜6文字）を保持している必要がある。

アルゴリズムまたは振る舞い

1. URLの先頭文字を判定する。 2. 'h' または 'H' の場合:

  • http に続く文字を検証し、: なら http (SSLなし)、s: なら https (SSLあり) と判定する。

    • 3. 'w' または 'W' の場合:

  • ws に続く文字を検証し、: なら ws (SSLなし)、s: なら wss (SSLあり) と判定する。

    • 4. 判定成功時、url ポインタをスキームと区切り文字の直後まで進め、スキーム名を返す。 5. 判定失敗時、is_ssl を 0 に設定し、NULL を返す。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 未知のスキームや不正な形式のURLが渡された場合、is_ssl を 0 に初期化した上で NULL を返す。
  • 文字列の終端チェックは行われないため、極端に短い文字列が渡された場合の挙動は呼び出し元に依存する。

保守担当者向けメモ

  • 本関数は http, https, ws, wss の4種類のみをサポートする。他のプロトコル（AJP等）を追加する場合は、このスイッチ文を拡張する必要がある。
  • 大文字小文字を区別しない判定を行っているため、スキーム名が混在していても正しく動作する。
