cgi_handle_response

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:420

目的

CGIスクリプトからの出力を解析し、HTTPレスポンスヘッダの処理、リダイレクトの制御、および標準エラー出力のログ転送を行う。

シグネチャまたは形状

static int cgi_handle_response(request_rec *r, int nph, apr_bucket_brigade *bb,
                               apr_interval_time_t timeout, cgi_server_conf *conf,
                               char *logdata, apr_file_t *script_err)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • nph: NPH（Non-Parsed Header）スクリプトかどうかを示すフラグ。
  • bb: スクリプトの出力を含むバケットブリゲード。
  • timeout: エラー出力読み取り時のタイムアウト値。
  • conf: CGIサーバー設定。
  • logdata: ログ出力用の補助データ。
  • script_err: スクリプトの標準エラー出力パイプ。
  • **出力**:
  • 戻り値: 処理結果を示すステータスコード（OK またはエラーコード）。

前提条件と不変条件

  • nph が偽の場合、ap_scan_script_header_err_brigade_ex を用いてヘッダ解析を行う。
  • script_err が提供されている場合、処理終了時に必ずクローズされる。
  • リダイレクト発生時は、残りの標準出力を破棄し、ブリゲードを破棄する。

アルゴリズムまたは振る舞い

1. **ヘッダ解析**: nph が偽の場合、ap_scan_script_header_err_brigade_ex でヘッダを解析する。解析失敗時は log_script を呼び出し、エラーを処理する。 2. **ヘッダ調整**: Content-Length および Transfer-Encoding を適切に調整する。 3. **リダイレクト処理**: Location ヘッダが存在しステータスが200の場合、内部リダイレクトを実行するか、一時的なリダイレクト（302）として処理する。 4. **出力転送**: ap_pass_brigade を使用して、解析済みのブリゲードをフィルタチェーンへ渡す。 5. **エラーログ転送**: 接続が中断されておらず、かつ書き込みが成功している場合、log_script_err を用いて標準エラー出力をログへ転送する。 6. **クリーンアップ**: script_err をクローズして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c
  • modules/generators/mod_cgid.c
  • **呼び出し先**:
  • ap_scan_script_header_err_brigade_ex
  • log_script
  • discard_script_output
  • apr_brigade_destroy
  • apr_file_pipe_timeout_set
  • log_script_err
  • ap_internal_redirect_handler
  • ap_pass_brigade
  • apr_file_close

関連データ構造

  • request_rec: Apacheのリクエスト情報。
  • apr_bucket_brigade: スクリプト出力のバッファ。
  • cgi_server_conf: CGIモジュールの設定構造体。

失敗モードと境界条件

  • **ヘッダ解析失敗**: ret != OK の場合、エラーログ出力を行い、特定のステータス（HTTP_NOT_MODIFIED 等）を除きエラーを返す。
  • **タイムアウト**: ret == HTTP_GATEWAY_TIME_OUT の場合、ブリゲードをクリーンアップして再読み取りを回避する。
  • **接続中断**: r->connection->aborted が真の場合、エラー出力の転送をスキップする。

保守担当者向けメモ

  • 本関数は mod_cgi および mod_cgid の両方から呼び出される共通ロジックである。
  • nph モード時はフィルタチェーンの再構築が行われるため、ヘッダ解析ロジックをバイパスする。
  • 戻り値は常に OK を返す設計となっており、実際のステータスコードは r->status に反映される。
