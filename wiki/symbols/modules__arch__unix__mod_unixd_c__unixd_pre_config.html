unixd_pre_config

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_unixd.c:294

目的

Apache HTTP Server の起動フェーズにおいて、Unix 関連のデフォルト設定を初期化し、suEXEC バイナリの実行権限および存在確認を行う。

シグネチャまたは形状

static int unixd_pre_config(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • plog: ログ用メモリプール。
  • ptemp: 一時的な割り当て用メモリプール。
  • **出力**:
  • 戻り値: OK (成功時)。

前提条件と不変条件

  • ap_unixd_config グローバル構造体が書き込み可能であること。
  • DEFAULT_USER および DEFAULT_GROUP マクロが定義されていること。
  • SUEXEC_BIN パスがシステム上に存在し、適切なパーミッション設定が期待されること。

アルゴリズムまたは振る舞い

1. **デフォルト設定の適用**: ap_unixd_config のユーザー名、グループ名、およびそれぞれの ID をデフォルト値で初期化する。chroot_dir を NULL に設定する。 2. **suEXEC の検証**:

  • apr_stat を使用して SUEXEC_BIN の情報を取得する。
  • ファイルが存在し、かつ SETUID_TEST（所有者が root かつ setuid ビットが立っていること）および access による読み取り・実行権限の確認をすべて満たす場合、suexec_enabled を 1 に設定する。
  • 検証に失敗した場合は、理由を suexec_disabled_reason に格納する。

    • 3. **特権ハンドラの初期化**: ap_sys_privileges_handlers(1) を呼び出し、システム権限ハンドラを有効化する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_pre_config を通じて Apache の設定フックとして登録される。
  • **呼び出し先**:
  • ap_uname2id, ap_gname2id: ユーザー/グループ名から ID への変換。
  • apr_stat: ファイル情報の取得。
  • access: ファイルのアクセス権限確認。
  • ap_sys_privileges_handlers: システム特権ハンドラの初期化。

関連データ構造

  • ap_unixd_config: unixd.h で定義された共有設定構造体。
  • apr_finfo_t: ファイル情報を保持する構造体。

失敗モードと境界条件

  • SUEXEC_BIN が存在しない、または所有者が root でない、あるいは実行権限がない場合、suexec_enabled は 0 となり、エラー理由が記録されるが、関数自体は OK を返す。

保守担当者向けメモ

  • 本関数は APR_HOOK_FIRST として登録されることを想定している。
  • suexec_disabled_reason は、設定のダンプやログ出力時に suEXEC が無効化された理由を特定するために使用される。
