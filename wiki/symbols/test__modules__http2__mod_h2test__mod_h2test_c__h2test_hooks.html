h2test_hooks

定義

  • 種別: function
  • 定義箇所: test/modules/http2/mod_h2test/mod_h2test.c:568

目的

Apache HTTP Server のフックシステムに対し、mod_h2test モジュールが必要とする設定フック、子プロセス初期化フック、および各種テスト用ハンドラを登録する。

シグネチャまたは形状

static void h2test_hooks(apr_pool_t *pool)

入力と出力

  • **入力**: pool (メモリ割り当て用の apr_pool_t へのポインタ)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • 本関数はモジュール読み込み時に呼び出される。
  • mod_h2.c が存在する場合、post_config フックがその後に実行されるよう依存関係が定義されている。

アルゴリズムまたは振る舞い

1. ログ出力: APLOG_TRACE1 レベルでフックインストール開始を記録する。 2. post_config フックの登録: h2test_post_config を APR_HOOK_MIDDLE で登録。 3. child_init フックの登録: h2test_child_init を APR_HOOK_MIDDLE で登録。 4. ハンドラの登録: 以下の各関数を ap_hook_handler を介して APR_HOOK_MIDDLE で登録する。

  • h2test_echo_handler
  • h2test_delay_handler
  • h2test_trailer_handler
  • h2test_error_handler

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュールロードプロセス（ソース内の宣言および静的初期化構造体から参照）。
  • **呼び出し先**:
  • ap_log_perror
  • ap_hook_post_config
  • ap_hook_child_init
  • ap_hook_handler
  • h2test_post_config
  • h2test_child_init
  • h2test_echo_handler
  • h2test_delay_handler
  • h2test_trailer_handler
  • h2test_error_handler

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • mod_h2: post_config の実行順序を制御するための依存モジュール名リスト。

失敗モードと境界条件

  • フック登録関数（ap_hook_*）自体は戻り値を返さないため、登録失敗時のエラーハンドリングは本関数内では行われない。

保守担当者向けメモ

  • 本関数はテスト専用モジュールのエントリーポイントである。ハンドラを追加する際は、必ずここで ap_hook_handler を呼び出す必要がある。
  • mod_h2 への依存関係は mod_h2[] 配列で定義されており、必要に応じて調整可能である。
