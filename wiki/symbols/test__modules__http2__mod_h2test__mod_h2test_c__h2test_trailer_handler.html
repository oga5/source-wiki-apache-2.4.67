h2test_trailer_handler

定義

  • 種別: function
  • 定義箇所: test/modules/http2/mod_h2test/mod_h2test.c:352

目的

HTTP/2 のトレーラーヘッダー付与機能を検証するためのテスト用ハンドラ。指定されたバイト数のレスポンスボディを送信し、その後にトレーラーヘッダーを付与する。

シグネチャまたは形状

static int h2test_trailer_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r（Apache リクエスト構造体）。r->handler が "h2test-trailer" である必要があり、r->args にボディサイズ（整数）を指定可能。
  • **出力**: int（Apache ステータスコード: OK または AP_FILTER_ERROR）。

前提条件と不変条件

  • r->handler が "h2test-trailer" であること。
  • リクエストメソッドが GET または POST であること。
  • r->args が指定された場合、数値として解釈され、負の値は 0 として扱われる。

アルゴリズムまたは振る舞い

1. リクエストのハンドラ名とメソッドを検証し、不一致なら DECLINED を返す。 2. r->args からボディサイズを解析し、レスポンスヘッダーに Content-Length を設定する。 3. Trailer ヘッダーに trailer-content-length を追加し、r->trailers_out に同じキーでボディサイズを格納する。 4. apr_bucket_brigade を作成し、指定されたサイズ分だけ 0 で埋めたバケットを ap_pass_brigade で順次送信する。 5. 最後に EOS（End of Stream）バケットを送信して終了する。 6. 処理中にエラーが発生した場合は cleanup ラベルへジャンプし、リソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフックシステム（test/modules/http2/mod_h2test/mod_h2test.c:585 にて ap_hook_handler に登録）。
  • **呼び出し先**: ap_log_rerror, ap_set_content_length, ap_set_content_type, apr_table_mergen, apr_table_set, apr_brigade_create, apr_brigade_write, ap_pass_brigade, apr_bucket_eos_create, apr_brigade_cleanup 等の Apache/APR API。

関連データ構造

  • request_rec: リクエスト情報の保持。
  • apr_bucket_brigade: レスポンスデータのバッファリングと送信に使用。
  • r->trailers_out: トレーラーヘッダーを格納するテーブル。

失敗モードと境界条件

  • apr_brigade_write または ap_pass_brigade が APR_SUCCESS 以外を返した場合、AP_FILTER_ERROR を返す。
  • r->args が数値以外の場合、apr_atoi64 の挙動に依存する（ソース上は明示的なエラーハンドリングなし）。

保守担当者向けメモ

  • 本関数はテスト専用であり、意図的にトレーラーヘッダーを生成する。
  • メモリ管理は r->pool に依存しており、明示的な解放は不要。
  • cleanup ラベルでの終了判定において、r->status が HTTP_OK でない場合や接続異常時は OK を返して終了する仕様となっている。
