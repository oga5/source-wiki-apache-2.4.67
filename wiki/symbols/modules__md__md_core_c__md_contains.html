md_contains

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:35

目的

指定されたドメイン名が、マネージドドメイン（md_t）の管理対象ドメインリストに含まれているかを判定します。

シグネチャまたは形状

int md_contains(const md_t *md, const char *domain, int case_sensitive)

入力と出力

  • **入力**:
  • md: 検索対象の md_t 構造体へのポインタ。
  • domain: 検索するドメイン名の文字列。
  • case_sensitive: 大文字小文字を区別するかを指定するフラグ（0で区別しない）。
  • **出力**:
  • int: 指定ドメインが含まれる場合は 1、含まれない場合は 0 を返します。

前提条件と不変条件

  • md->domains 配列は、md_array_str_compact によって正規化されていることが期待されます。

アルゴリズムまたは振る舞い

1. md_array_str_index を呼び出し、md->domains 内に指定ドメインが直接存在するかを検索します。 2. 直接の一致が見つかった場合は 1 を返します。 3. 一致しない場合、md_dns_domains_match を呼び出し、ワイルドカードを含む DNS マッチングによる判定を試行してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_core.c (複数箇所)
  • modules/md/md_reg.c:362
  • modules/md/mod_md.c:381
  • modules/md/mod_md_config.c:1534
  • **呼び出し先**:
  • md_array_str_index
  • md_dns_domains_match

関連データ構造

  • md_t: 管理対象ドメインの設定を保持する構造体。domains メンバ（配列）が検索対象となります。

失敗モードと境界条件

  • md が NULL の場合や、domains 配列が初期化されていない場合の挙動については、本ソースコード内では明示的なガード句は確認できません。

保守担当者向けメモ

  • 本関数は、完全一致検索（md_array_str_index）と DNS 形式の照合（md_dns_domains_match）の二段階で判定を行います。
  • ワイルドカード等の複雑なドメイン照合ロジックは md_dns_domains_match に依存しているため、照合ルールの変更が必要な場合はそちらを確認してください。
