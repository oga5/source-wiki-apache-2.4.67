store_table

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_socache.c:248

目的

HTTP ヘッダーテーブルの内容を、指定されたバッファへシリアライズ（パック）します。

シグネチャまたは形状

apr_status_t store_table(apr_table_t *table, unsigned char *buffer,
                         apr_size_t buffer_len, apr_size_t *slider);

入力と出力

  • table: シリアライズ対象の apr_table_t。
  • buffer: 書き込み先のバッファ。NULL の場合、書き込みを行わずに必要なサイズのみを計算します。
  • buffer_len: バッファの総容量。
  • slider: バッファ内の現在の書き込みオフセット（入出力両用）。
  • 戻り値: 成功時は APR_SUCCESS、バッファ不足時は APR_EOF を返します。

前提条件と不変条件

  • slider は、バッファ内の有効な書き込み開始位置を指している必要があります。
  • ヘッダーの各要素は Key: Value 形式でパックされ、最後に CRLF が付与されます。

アルゴリズムまたは振る舞い

1. テーブル内の各エントリを走査し、Key: Value 文字列を生成します。 2. buffer が非 NULL の場合、apr_snprintf を用いてバッファへ書き込みます。 3. 各ステップで slider を更新し、残りのバッファ容量をチェックします。 4. 全エントリの処理後、テーブルの終端を示す CRLF をバッファに追加します。 5. バッファ容量を超える場合は即座に APR_EOF を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c: dobj->headers_out および dobj->headers_in の保存時に使用。
  • modules/cache/mod_cache_socache.c: r->headers_out, r->headers_in のサイズ計算（buffer=NULL）および sobj->headers_out, sobj->headers_in のバッファへの書き込み時に使用。
  • **呼び出し先**:
  • apr_table_elts: テーブル要素の取得。
  • apr_snprintf: 文字列のフォーマット書き込み。
  • memcpy: CRLF のコピー。

関連データ構造

  • apr_table_t: Apache のヘッダー管理用データ構造。
  • apr_table_entry_t: テーブル内の個別のキーと値のペア。

失敗モードと境界条件

  • key_len + val_len + 5 が残りのバッファ容量を超える場合、APR_EOF を返します。
  • 終端の CRLF を書き込むための 3 バイトが確保できない場合、APR_EOF を返します。

保守担当者向けメモ

  • 本関数は read_table と対になるシリアライズ関数です。データレイアウトを変更する場合は、必ず read_table 側のデシリアライズロジックも更新してください。
  • buffer に NULL を渡すことで、必要なバッファサイズを事前に計算する用途にも利用可能です。
