check_time_left

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reqtimeout.c:91

目的

接続のタイムアウト期限（timeout_at）と現在時刻を比較し、残り時間を算出および検証する。

シグネチャまたは形状

static apr_status_t check_time_left(reqtimeout_con_cfg *ccfg,
                                    apr_time_t *time_left_p,
                                    apr_time_t now)

入力と出力

  • **入力**:
  • ccfg: 接続ごとの状態を保持する reqtimeout_con_cfg 構造体へのポインタ。
  • time_left_p: 残り時間を格納するための出力用ポインタ。
  • now: 現在時刻（0が指定された場合は内部で apr_time_now() を呼び出す）。
  • **出力**:
  • APR_SUCCESS: 期限内であり、残り時間が算出された場合。
  • APR_TIMEUP: 期限が経過している場合。

前提条件と不変条件

  • ccfg->timeout_at には、有効な絶対期限時刻が格納されている必要がある。
  • time_left_p は有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. now が 0 の場合、apr_time_now() を使用して現在時刻を取得する。 2. ccfg->timeout_at から now を減算し、残り時間を time_left_p に格納する。 3. 残り時間が 0 以下であれば APR_TIMEUP を返す。 4. 残り時間が 1 秒未満の場合、下限値として 1 秒を time_left_p に設定する。 5. 正常終了時は APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_reqtimeout.c:216
  • modules/filters/mod_reqtimeout.c:298
  • **呼び出し先**:
  • apr_time_now() (条件付き)
  • apr_time_from_sec()

関連データ構造

  • reqtimeout_con_cfg: 接続ごとのタイムアウト状態（timeout_at を保持）を管理する構造体。

失敗モードと境界条件

  • **タイムアウト超過**: 残り時間が 0 以下の場合、APR_TIMEUP を返し、呼び出し元での接続切断処理をトリガーする。
  • **最小値補正**: 1 秒未満の残り時間は 1 秒に切り上げられる。

保守担当者向けメモ

  • 本関数は reqtimeout_filter 内で頻繁に呼び出されるため、計算コストは最小限に抑えられている。
  • now 引数に 0 を渡すことで、関数内で現在時刻取得のオーバーヘッドを制御可能である。
