ap_byterange_filter

定義

  • 種別: function
  • 定義箇所: modules/http/byterange_filter.c:399

目的

HTTP Range リクエストを処理し、レスポンスボディを要求されたバイト範囲に切り出して送信する出力フィルタです。

シグネチャまたは形状

apr_status_t ap_byterange_filter(ap_filter_t *f, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • f: フィルタコンテキスト（request_rec を含む）。
  • bb: 送信対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（APR_SUCCESS またはエラーコード）。
  • フィルタチェーンの下流へ加工済みのバケットブリゲードを渡す。

前提条件と不変条件

  • ブリゲード内に EOS（End of Stream）バケットが存在し、かつ全バケットの長さが確定していること。
  • 条件を満たさない場合、フィルタは自身を削除し、データを変更せずに下流へ渡す。
  • 設定値（max_ranges 等）に基づく制限を超えた場合、処理を中断して透過的に振る舞う。

アルゴリズムまたは振る舞い

1. **検証**: ブリゲード内の全バケット長を計算し、EOS の有無と長さを確認。 2. **解析**: ap_set_byterange を呼び出し、Range ヘッダーを解析して範囲リストを取得。 3. **抽出**: copy_brigade_range を使用して、指定範囲のデータを新しいブリゲード tmpbb にコピー。 4. **構築**:

  • 単一範囲の場合: Content-Range ヘッダーを付与。
  • 複数範囲の場合: multipart/byteranges 形式に変換し、境界文字列（boundary）とヘッダーを挿入。

    • 5. **送信**: ap_pass_brigade を通じて下流へ送信。複数範囲の場合は適宜バッファをフラッシュする。 6. **終了**: 最後に EOS バケットを追加し、リソースを解放。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache フィルタチェーン（ap_register_output_filter により BYTERANGE として登録）。
  • **呼び出し先**:
  • ap_set_byterange
  • copy_brigade_range
  • send_416
  • ap_pass_brigade
  • ap_remove_output_filter

関連データ構造

  • indexes_t: バイト範囲の開始・終了位置を保持。
  • core_dir_config: 範囲制限（max_ranges 等）の設定値。
  • apr_bucket_brigade: バケットの管理用コンテナ。

失敗モードと境界条件

  • **416 Range Not Satisfiable**: 範囲指定が不適切な場合、send_416 を呼び出してエラーを返す。
  • **制限超過**: max_ranges 等の制限を超えた場合、フィルタを削除して元のデータをそのまま通過させる。
  • **メモリ不足**: バケットコピー時にエラーが発生した場合はログを出力し、処理を継続またはスキップする。

保守担当者向けメモ

  • 本フィルタは r->pool を使用してメモリを確保するため、リクエスト終了時に自動的に解放されます。
  • バケットの長さが不明（-1）なバケットが含まれる場合、安全のためバイトレンジ処理はスキップされます。
  • 複数範囲の処理では multipart/byteranges 形式の構築に伴うオーバーヘッドが発生するため、AP_DEFAULT_MAX_RANGES 等の制限値が重要です。
