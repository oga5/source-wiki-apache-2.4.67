dav_register_provider

定義

  • 種別: function
  • 定義箇所: modules/dav/main/providers.c:24

目的

DAV 関連のプロバイダーを Apache のグローバルなプロバイダーレジストリに登録します。

シグネチャまたは形状

DAV_DECLARE(void) dav_register_provider(apr_pool_t *p, const char *name,
                                        const dav_provider *provider);

入力と出力

  • **入力**:
  • p: プロバイダーの生存期間を管理するための apr_pool_t。
  • name: 登録するプロバイダーの識別名。
  • provider: 登録対象となる dav_provider 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • プロバイダーのバージョンは "0" に固定されます。
  • DAV_PROVIDER_GROUP 定数を使用して、DAV 固有のプロバイダーグループとして登録されます。

アルゴリズムまたは振る舞い

1. Apache の汎用プロバイダー登録関数 ap_register_provider を呼び出します。 2. 引数として受け取った p、name、provider をそのまま渡し、グループ識別子に DAV_PROVIDER_GROUP を、バージョンに "0" を指定して登録を完了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/repos.c:2282 （ファイルシステムバックエンドの登録時）。
  • **呼び出し先**: ap_register_provider （Apache Core API）。

関連データ構造

  • dav_provider: 登録されるプロバイダーの定義構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • ap_register_provider の内部実装に依存します。本関数自体は戻り値を持たず、エラー発生時の挙動は Apache Core のプロバイダー管理機構に委ねられます。

保守担当者向けメモ

  • 本関数は ap_register_provider の薄いラッパーです。
  • プロバイダーのバージョン管理（第 4 引数）は現在 "0" に固定されています。将来的にバージョン管理が必要となった場合でも、本関数を直接変更するのではなく、必要に応じて新しい API を検討してください。
