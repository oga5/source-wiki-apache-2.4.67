h2_proxy_request

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.h:176

目的

バックエンドサーバーへのプロキシ転送を行う前に、リクエスト情報を正規化および保持するための中間表現として使用される構造体です。

シグネチャまたは形状

struct h2_proxy_request {
    /* 構造体のメンバ詳細はソースコードの定義箇所を参照 */
};

入力と出力

  • **入力**: h2_proxy_req_create または h2_proxy_req_createn を介して、ストリームIDおよびメモリプールからインスタンス化されます。
  • **出力**: h2_proxy_req_make を通じて、プロキシ転送に必要な形式へリクエスト情報が構築されます。

前提条件と不変条件

  • h2_proxy_util.h で定義されたヘルパー関数群によって管理されることを前提としています。
  • インスタンス化には apr_pool_t によるメモリ管理が必須です。

アルゴリズムまたは振る舞い

1. **生成**: h2_proxy_req_create により、指定されたIDとプールを用いて構造体が初期化されます。 2. **構築**: h2_proxy_req_make を呼び出すことで、リクエストの各要素がプロキシ転送に適した状態に変換・設定されます。 3. **管理**: h2_proxy_util 内の関数群により、プロキシセッションにおけるリクエスト情報のライフサイクルが制御されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_session.c: セッション管理において本構造体を参照。
  • modules/http2/h2_proxy_util.c: ヘルパー関数内での生成および操作。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出し先は定義されていません。操作は h2_proxy_req_create や h2_proxy_req_make などのユーティリティ関数を介して行われます。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • h2_proxy_session: 本構造体を利用してプロキシ通信を管理する上位コンポーネント。

失敗モードと境界条件

  • メモリ割り当て失敗時（apr_pcalloc 等の内部エラー）の挙動については、APRの標準的なエラーハンドリングに依存します。
  • 構造体の初期化や構築時に無効なIDやNULLポインタが渡された場合の挙動は、各ヘルパー関数の実装に依存します。

保守担当者向けメモ

  • 本構造体は mod_proxy_http2 のプロキシロジックにおける基盤です。構造体のメンバを変更する場合は、h2_proxy_util.c 内の関連するヘルパー関数（h2_proxy_req_create 等）の整合性を確認してください。
