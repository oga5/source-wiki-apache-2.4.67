h2_proxy_session

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_session.h:69

目的

バックエンドサーバーとの HTTP/2 セッションを管理するためのメイン構造体です。mod_proxy_http2 において、フロントエンドからのリクエストをバックエンドへ橋渡しする際のセッション層を抽象化します。

シグネチャまたは形状

typedef struct h2_proxy_session h2_proxy_session;

※構造体の内部メンバは公開されたソース excerpt に含まれていません。

入力と出力

  • **入力**: nghttp2_session インスタンス、バックエンド接続状態、および Apache のバケットブリゲードを介した I/O データ。
  • **出力**: バックエンドへの HTTP/2 フレーム送信、および処理結果を示す apr_status_t（APR_EAGAIN, APR_SUCCESS, APR_EOF 等）。

前提条件と不変条件

  • nghttp2 ライブラリと連携して動作します。
  • セッションの健全性を維持するため、PING 状態管理（h2_ping_state_t）が組み込まれています。

アルゴリズムまたは振る舞い

1. **初期化**: h2_proxy_session_setup によりセッションが生成され、バックエンド接続が確立されます。 2. **リクエスト投入**: h2_proxy_session_submit を通じてリクエストがストリームキューに追加されます。 3. **処理ループ**: h2_proxy_session_process が呼び出されるたびに、nghttp2 のイベントループと Apache のバケットブリゲード間でデータの読み書きが行われます。 4. **状態遷移**: PING 応答、GOAWAY フレームの受信、またはタイムアウトに応じて h2_proxys_state が遷移します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c 内の各関数（dispatch_event, ping_arrived, check_suspended, set_ping_timeout, unset_ping_timeout, enter_ping_state, ping_new_session, ping_reuse_session, ping_ev_frame_received）が本構造体を引数として操作します。
  • **呼び出し先**: nghttp2 ライブラリの関数群（フレーム処理用）。

関連データ構造

  • h2_proxys_state: セッションの現在のフェーズ。
  • h2_ping_state_t: 接続の健全性確認用 PING 状態。
  • h2_proxys_event_t: セッションの状態遷移イベント。
  • proxy_conn_rec: バックエンド接続情報。

失敗モードと境界条件

  • **タイムアウト**: バックエンドが PING に応答しない場合、セッションは適切にクローズされます。
  • **I/O エラー**: h2_proxy_session_process の戻り値が APR_EAGAIN 以外の場合、接続の再試行や終了の判断が必要です。

保守担当者向けメモ

  • 本構造体は mod_proxy_http2 の心臓部です。
  • h2_proxy_session_process の戻り値は、呼び出し元のイベントループが次に取るべきアクションを決定する重要な指標となります。
  • 構造体の詳細なメンバ定義については、modules/http2/h2_proxy_session.c の実装を参照してください。
