allowdeny

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_access_compat.c:55

目的

Apache HTTP Server のレガシーなアクセス制御ディレクティブ（Allow および Deny）における個別のルール定義を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_int64_t limited;
    union {
        char *from;
        apr_ipsubnet_t *ip;
    } x;
    enum allowdeny_type type;
} allowdeny;

入力と出力

本構造体は設定ファイルから読み込まれたアクセス制御ルールを格納します。

  • limited: 適用対象となる HTTP メソッドのビットマスク。
  • x: ホスト名（char *from）または IP サブネット（apr_ipsubnet_t *ip）のいずれか一方を保持する共用体。
  • type: ルールの種別（allow か deny かを識別する allowdeny_type 列挙型）。

前提条件と不変条件

  • x 共用体内の値は、設定解析時に決定されます。
  • apr_array_make を用いて生成された配列内に格納され、apr_pool_t を介してメモリ管理されます。

アルゴリズムまたは振る舞い

リクエスト処理時に find_allowdeny 関数がこの構造体の配列を走査し、現在のリクエストの IP アドレスやホスト名が x に格納された条件と一致するかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_access_compat.c:87-88: apr_array_make による配列の初期化。
  • modules/aaa/mod_access_compat.c:155: allow_cmd ハンドラ内でのルール追加。
  • modules/aaa/mod_access_compat.c:245: find_allowdeny 関数内での配列走査。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • access_compat_dir_conf: 本構造体の配列（allows, denys）を保持するディレクトリごとの設定構造体。
  • allowdeny_type: ルールの種類を定義する列挙型。
  • apr_ipsubnet_t: IP サブネット情報を保持する APR のデータ構造。

失敗モードと境界条件

  • メモリ不足: apr_array_push 実行時にメモリ確保が失敗する可能性があります。
  • 設定不整合: limited マスクが適切に設定されていない場合、意図しないメソッドに対してルールが適用される可能性があります。

保守担当者向けメモ

  • 本構造体は mod_access_compat 専用の内部データ構造です。
  • x 共用体の使い分けは、設定ディレクティブの引数解析時に行われます。ホスト名ベースの判定は DNS 逆引きに依存するため、セキュリティ上の制約（REMOTE_DOUBLE_REV 等）に注意してください。
