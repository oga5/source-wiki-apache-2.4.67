unixd_set_chroot_dir

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_unixd.c:249

目的

Apache HTTP Server の設定ファイルにおいて ChrootDir ディレクティブを処理し、指定されたディレクトリを ap_unixd_config 構造体に格納する。

シグネチャまたは形状

static const char* unixd_set_chroot_dir(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • arg: 設定ファイルで指定されたディレクトリパスの文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラー内容を示す文字列（const char*）を返す。

前提条件と不変条件

  • 本関数はグローバルコンテキストでのみ呼び出し可能であること（ap_check_cmd_context による検証）。
  • 指定された arg がディレクトリとして存在すること。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、GLOBAL_ONLY コンテキスト以外での設定を拒否する。 2. ap_is_directory を使用して、引数 arg が有効なディレクトリであるかを確認する。無効な場合はエラーメッセージを返す。 3. 検証に成功した場合、グローバル変数 ap_unixd_config の chroot_dir メンバに arg を代入する。 4. NULL を返して正常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/unix/mod_unixd.c:417 において、AP_INIT_TAKE1 マクロを通じて設定ディレクティブ ChrootDir のハンドラとして登録されている。
  • **呼び出し先**:
  • ap_check_cmd_context (Apache API)
  • ap_is_directory (Apache API)

関連データ構造

  • cmd_parms: 設定コマンドのコンテキスト情報を保持する構造体。
  • ap_unixd_config: Unix 系モジュールの設定を保持するグローバル構造体。

失敗モードと境界条件

  • **コンテキストエラー**: GLOBAL_ONLY 以外の場所（例: <Directory> ブロック内など）で ChrootDir が指定された場合、エラーメッセージを返す。
  • **パス検証エラー**: 指定されたパスが存在しない、またはディレクトリではない場合、「ChrootDir must be a valid directory」というエラーメッセージを返す。

保守担当者向けメモ

  • 本関数はサーバー起動時の設定読み込みフェーズでのみ実行される。
  • ap_unixd_config.chroot_dir に格納されたパスは、後のプロセス生成フェーズにおいて ap_unixd_setup_child 内で chroot() システムコールを実行するために使用される。
