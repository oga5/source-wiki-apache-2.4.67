ST_MONITORED_COMP

定義

  • 種別: struct
  • 定義箇所: support/win32/ApacheMonitor.c:85

目的

リモート監視対象となるコンピュータの識別情報および、そのコンピュータのレジストリへアクセスするためのハンドルを保持する。

シグネチャまたは形状

typedef struct _st_MONITORED_COMPUTERS
{
    LPTSTR  szComputerName;
    HKEY    hRegistry;
} ST_MONITORED_COMP;

入力と出力

  • szComputerName: コンピュータ名を指す文字列ポインタ（入力）。
  • hRegistry: レジストリ操作に使用するハンドル（出力/状態保持）。

前提条件と不変条件

  • g_stComputers 配列を通じて管理される。
  • 構造体内の各メンバは、初期化時に memset 等でゼロクリアされることが想定されている。

アルゴリズムまたは振る舞い

  • 本構造体は、ApacheMonitor アプリケーションがリモートサーバーの状態をポーリングする際の基点として機能する。
  • hRegistry は、リモートコンピュータのレジストリ情報を取得・監視するために使用される。

呼び出し元と呼び出し先

  • 呼び出し元: support/win32/ApacheMonitor.c 内の初期化処理および終了処理（memset によるクリア操作）。
  • 呼び出し先: 情報なし。

関連データ構造

  • g_stComputers: ST_MONITORED_COMP 型の固定長配列（MAX_APACHE_COMPUTERS 個）。

失敗モードと境界条件

  • MAX_APACHE_COMPUTERS (32) の固定長配列であるため、監視対象がこれを超える場合は制限を受ける。
  • szComputerName に動的に割り当てられたメモリは、アプリケーション終了時に適切に解放する必要がある（メモリリークの可能性がある）。

保守担当者向けメモ

  • am_ClearComputersSt 関数が定義されている場合、本構造体のメンバの解放処理に使用されるため、メモリ管理の際は当該関数を確認すること。
