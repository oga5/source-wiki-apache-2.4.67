util_ldap_search_node_display

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache.c:220

目的

LDAP 検索キャッシュノードの内容を HTML テーブルの行（<tr>）として出力し、管理画面等での可視化を支援します。

シグネチャまたは形状

void util_ldap_search_node_display(request_rec *r, util_ald_cache_t *cache, void *n)

入力と出力

  • **入力**:
  • r: Apache のリクエストレコード（出力先）。
  • cache: キャッシュ管理構造体（本関数内では未使用）。
  • n: 表示対象の util_search_node_t 構造体へのポインタ。
  • **出力**: なし（ap_rprintf を通じてリクエストの出力バッファに HTML を書き込みます）。

前提条件と不変条件

  • 引数 n は util_search_node_t 型としてキャスト可能である必要があります。
  • r および n は有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 引数 n を util_search_node_t 型にキャストします。 2. ノード内の lastbind（最終バインド時刻）を apr_ctime を用いて人間が読み取り可能な文字列に変換します。 3. ap_rprintf を使用し、以下の情報を HTML テーブルのセルとして出力します。

  • username: ap_escape_html でエスケープ処理されたユーザー名。
  • dn: ap_escape_html でエスケープ処理された識別名（DN）。
  • lastbind: 変換済みの時刻文字列。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap_cache_mgr.c:296（キャッシュ管理ロジックから参照されています）。
  • **呼び出し先**:
  • apr_ctime
  • ap_rprintf
  • ap_escape_html

関連データ構造

  • util_search_node_t: 本関数が処理する対象のデータ構造。
  • request_rec: Apache のリクエスト状態を保持する構造体。
  • util_ald_cache_t: キャッシュ管理用構造体。

失敗モードと境界条件

  • node->username や node->dn が NULL の場合、ap_escape_html の挙動に依存します。
  • 出力バッファ（r）への書き込みが失敗した場合の戻り値チェックは行われていません。

保守担当者向けメモ

  • 本関数は HTML 生成に特化しています。出力フォーマットを変更する場合は、util_ldap_cache_mgr.c 側のテーブルヘッダー定義との整合性に注意してください。
  • cache 引数は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
