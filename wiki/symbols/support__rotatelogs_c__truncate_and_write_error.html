truncate_and_write_error

定義

  • 種別: function
  • 定義箇所: support/rotatelogs.c:352

目的

ログファイルの書き込みエラーが発生した際、既存のファイル内容を破棄（切り詰め）し、エラーメッセージをファイルに記録する。

シグネチャまたは形状

static void truncate_and_write_error(rotate_status_t *status, const char *message)

入力と出力

  • **入力**:
  • status: 現在のログファイルの状態を保持する rotate_status_t 構造体へのポインタ。
  • message: ファイルに書き込むエラーメッセージ文字列。
  • **出力**: なし（副作用としてファイル操作を実行）。

前提条件と不変条件

  • status->current.fd が有効なファイル記述子としてオープンされていること。
  • message は有効なヌル終端文字列であること。

アルゴリズムまたは振る舞い

1. message の長さを取得する。 2. apr_file_trunc を使用して、現在のログファイル（status->current.fd）のサイズを 0 に切り詰める。

  • 失敗した場合は標準エラー出力にエラーメッセージを出力し、終了コード 2 でプロセスを終了する。

    • 3. apr_file_write_full を使用して、message をファイルに書き込む。

  • 失敗した場合は標準エラー出力にエラーメッセージを出力し、終了コード 2 でプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/rotatelogs.c:523
  • support/rotatelogs.c:795
  • **呼び出し先**:
  • apr_file_trunc (APR)
  • apr_file_write_full (APR)
  • strlen (標準ライブラリ)
  • fprintf (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • rotate_status_t: ログファイルの現在の状態（ファイル記述子やパス）を保持する。

失敗モードと境界条件

  • apr_file_trunc または apr_file_write_full が APR_SUCCESS を返さない場合、エラーメッセージを stderr に出力して即座に終了する。

保守担当者向けメモ

  • 本関数は致命的な書き込みエラー発生時に呼び出される。ログの欠損を許容し、エラー情報を優先して記録する設計である。
  • 終了コード 2 は、このユーティリティにおけるエラー終了を示す標準的な値として扱われている。
