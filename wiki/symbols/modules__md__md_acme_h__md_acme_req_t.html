md_acme_req_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme.h:56

目的

ACME プロトコルにおける個別のリクエスト実行コンテキストを保持するための構造体です。

シグネチャまたは形状

typedef struct md_acme_req_t md_acme_req_t;

※構造体の内部メンバは不透明（opaque）として定義されています。

入力と出力

  • **入力**: md_acme_req_create を介して md_acme_t インスタンス、HTTP メソッド、および対象 URL を受け取ります。
  • **出力**: リクエストの処理結果は、関連付けられたコールバック関数を通じて md_result_t 等に反映されます。

前提条件と不変条件

  • 関連付けられた apr_pool_t の生存期間内に存在する必要があります。
  • リクエスト処理中のメモリ管理は、このプールに依存します。

アルゴリズムまたは振る舞い

1. **初期化**: md_acme_req_create により生成され、リクエストのメソッドと URL が設定されます。 2. **ペイロード準備**: md_acme_req_body_init を使用して JSON ペイロードが設定されます。 3. **送信**: md_acme_req_send により、JWS 署名を含む HTTP リクエストが実行されます。 4. **コールバック処理**: リクエストのライフサイクル（初期化、JSON レスポンス処理、エラー処理）は、登録されたコールバック関数によって制御されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme.c 内の各関数（md_acme_req_create, inspect_problem, acmev2_req_init, md_acme_req_body_init, md_acme_req_done, acmev2_GET_as_POST_init, md_acme_req_send 等）。
  • **呼び出し先**: md_acme_req_init_cb 型のコールバック関数が定義されており、リクエスト処理の各段階で呼び出されます。

関連データ構造

  • md_acme_t: ACME セッション状態。
  • md_json_t: リクエスト/レスポンスの JSON ペイロード。
  • md_http_response_t: HTTP レスポンスデータ。
  • md_result_t: 処理結果およびエラー情報。

失敗モードと境界条件

  • リクエスト生成時のメモリ不足（APR プール依存）。
  • ACME サーバーからのエラー応答（inspect_problem による判定）。
  • JWS 署名失敗やネットワーク通信エラー。

保守担当者向けメモ

  • 本構造体は md_acme.h で前方宣言されており、実装詳細は md_acme.c に隠蔽されています。
  • リクエスト処理中のメモリリークを防ぐため、コールバック内でのプール利用には注意してください。
