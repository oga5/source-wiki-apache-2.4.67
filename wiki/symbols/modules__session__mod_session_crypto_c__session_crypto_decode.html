session_crypto_decode

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:513

目的

mod_session の復号化フックとして、暗号化されたセッションデータを復号し、元のセッション文字列へ変換する。

シグネチャまたは形状

apr_status_t session_crypto_decode(request_rec *r, session_rec *z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報。
  • z: 復号対象のセッション構造体（z->encoded に暗号化された文字列が格納されている）。
  • **出力**:
  • 戻り値: 成功時は OK、失敗時は apr_status_t エラーコード。
  • z->encoded: 復号成功時、元の平文文字列に更新される。

前提条件と不変条件

  • dconf->passphrases_set が真であること。
  • z->encoded が存在し、かつ空ではないこと。
  • サーバープロセスプールに CRYPTO_KEY で識別される apr_crypto_t コンテキストが格納されていること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリ設定 (session_crypto_dir_conf) を取得する。 2. サーバープロセスプールから apr_crypto_t コンテキストを取得する。 3. decrypt_string を呼び出し、Base64 デコード、MAC 検証、および復号化を実行する。 4. 復号に成功した場合、z->encoded を復号後の文字列で更新する。 5. 復号に失敗した場合はエラーログを出力し、エラーコードを返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_session_decode を介して mod_session から呼び出される（modules/session/mod_session_crypto.c:775）。
  • **呼び出し先**:
  • ap_get_module_config
  • apr_pool_userdata_get
  • decrypt_string
  • ap_log_rerror

関連データ構造

  • request_rec: リクエストごとのコンテキスト。
  • session_rec: セッションデータを保持する構造体。
  • session_crypto_dir_conf: 暗号化設定（パスフレーズ等）を保持する構造体。

失敗モードと境界条件

  • decrypt_string が APR_SUCCESS 以外を返した場合、復号失敗としてエラーログを出力し、処理を中断する。
  • パスフレーズが設定されていない、またはセッションデータが空の場合は、復号処理をスキップして OK を返す。

保守担当者向けメモ

  • 本関数は APR_HOOK_FIRST としてフックに登録されており、他のセッションデコード処理よりも優先して実行される。
  • メモリ管理は r->pool に依存しており、リクエスト終了時に自動的に解放される。
