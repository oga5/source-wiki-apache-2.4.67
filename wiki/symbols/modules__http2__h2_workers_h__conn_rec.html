conn_rec

定義

  • 種別: typedef
  • 定義箇所: modules/http2/h2_workers.h:87

目的

Apache HTTP Server における接続（Connection）のコンテキストを保持する主要なデータ構造です。HTTP/2 モジュールにおいては、ワーカーが処理する個別の接続状態を表現するために使用されます。

シグネチャまたは形状

構造体の詳細なメンバ構成は本ヘッダーファイル内では定義されておらず、include/http_connection.h 等のコアヘッダーに依存します。

入力と出力

  • 入力: h2_workers において、ワーカーが接続を取得する際の対象として渡されます。
  • 出力: 処理完了後、ap_conn_producer_done コールバックを通じて、処理済み接続として通知されます。

前提条件と不変条件

  • h2_workers の文脈では、ワーカーが取得した conn_rec は、処理が完了するまで有効である必要があります。
  • 接続処理のライフサイクル管理は、h2_mplx および h2_workers の連携によって制御されます。

アルゴリズムまたは振る舞い

1. **取得**: h2_workers のワーカーが next コールバックを介して conn_rec を取得します。 2. **処理**: ワーカーは取得した conn_rec に対して、HTTP/2 のストリーム処理やフィルタ処理を実行します。 3. **完了**: 処理終了後、done コールバックが呼び出され、接続の解放や後処理が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_workers.c 内のワーカー処理ループ（slot_run）が、ap_conn_producer_done 等の引数としてこの型を参照します。
  • 呼び出し先: Apache HTTP Server コアの ap_process_connection や ap_shutdown_conn 等の接続管理関数が、この構造体を操作します。

関連データ構造

  • ap_conn_producer_t: conn_rec を提供するソースを定義する構造体。
  • h2_workers: conn_rec を処理するワーカープール。

失敗モードと境界条件

  • 複数のスレッドから同一の conn_rec を操作することは禁止されており、h2_mplx による排他制御が前提となります。

保守担当者向けメモ
