sharedslotdesc_t

定義

  • 種別: struct
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:35

目的

共有メモリ領域の先頭に配置され、その領域で管理されるスロットの構成情報（サイズ、個数、型）を保持するヘッダー構造体です。

シグネチャまたは形状

typedef struct {
    apr_size_t size;             /* 各メモリスロットのサイズ */
    unsigned int num;            /* メモリスロットの総数 */
    ap_slotmem_type_t type;      /* 型固有のフラグ */
} sharedslotdesc_t;

入力と出力

本構造体はデータ構造であり、関数ではありません。共有メモリの初期化時や復元時に、メモリレイアウトのメタデータとして読み書きされます。

前提条件と不変条件

  • 共有メモリの先頭（apr_shm_baseaddr_get で取得されるアドレス）に配置されます。
  • AP_SLOTMEM_OFFSET マクロにより、本構造体のサイズを考慮したアライメントが計算され、後続のデータ領域の開始位置が決定されます。

アルゴリズムまたは振る舞い

  • slotmem_create 実行時、共有メモリの確保直後に本構造体が書き込まれます。
  • restore_slotmem 実行時、永続化ファイルから読み込まれ、共有メモリの構成情報の復元に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: slotmem_create（初期化時）、restore_slotmem（復元時）、slotmem_attach（接続時）などで、共有メモリのベースアドレスからキャストして参照されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • ap_slotmem_instance_t: 共有メモリインスタンス全体を管理する構造体。
  • ap_slotmem_type_t: sharedslotdesc_t 内で利用される型定義。

失敗モードと境界条件

  • 共有メモリのベースアドレスが不正な場合、本構造体へのアクセスはメモリ保護違反を引き起こす可能性があります。
  • 永続化ファイルが破損している場合、restore_slotmem で読み込まれた本構造体の値が不正となり、後続のオフセット計算に影響を及ぼす可能性があります。

保守担当者向けメモ

  • 本構造体は共有メモリのレイアウトの基点となるため、メンバの変更は AP_SLOTMEM_OFFSET の計算や、既存の永続化ファイルとの互換性に直接影響します。
  • 構造体のサイズ変更を行う際は、バイナリ互換性の維持について慎重に検討してください。
