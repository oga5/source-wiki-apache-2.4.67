md_store_load

定義

  • 種別: function
  • 定義箇所: modules/md/md_store.c:70

目的

Managed Domain (MD) サブシステムにおいて、ストレージ層から特定のドメイン関連データ（証明書、秘密鍵、JSON設定など）を読み込むための汎用インターフェースです。

シグネチャまたは形状

apr_status_t md_store_load(md_store_t *store, md_store_group_t group, 
                           const char *name, const char *aspect, 
                           md_store_vtype_t vtype, void **pdata, 
                           apr_pool_t *p);

入力と出力

  • store: ストレージ抽象化ハンドル。
  • group: 読み込み対象のストレージグループ（例: MD_SG_ACCOUNTS）。
  • name: 対象ドメイン名または識別子。
  • aspect: 読み込むデータの側面（ファイル名や属性）。
  • vtype: データの型（JSON, PKEY, CHAIN, TEXT等）。
  • pdata: 読み込まれたデータを格納するポインタへのポインタ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **戻り値**: 成功時に APR_SUCCESS、失敗時にエラーコードを返します。

前提条件と不変条件

  • store 引数が有効な md_store_t 構造体であり、その load 関数ポインタが適切に初期化されている必要があります。
  • p プールは、読み込まれたデータが生存するために十分な期間保持される必要があります。

アルゴリズムまたは振る舞い

引数として渡された md_store_t *store が保持する関数ポインタ store->load を呼び出し、実際の読み込み処理を具体的なストレージ実装（md_store_fs.c 等）へ委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_acct.c (アカウント鍵のロード)
  • modules/md/md_acme_authz.c (チャレンジデータのロード)
  • modules/md/md_reg.c (HTTPD設定JSONのロード)
  • modules/md/md_store.c (内部ヘルパー関数 md_store_load_json, md_pkey_load, md_pubcert_load からの呼び出し)
  • modules/md/mod_md.c (チャレンジ関連データのロード)
  • **呼び出し先**:
  • store->load (引数で渡された md_store_t 構造体内の関数ポインタ)

関連データ構造

  • md_store_t: ストレージ操作の抽象化ハンドル。
  • md_store_group_t: ストレージグループの列挙型。
  • md_store_vtype_t: データの種類を示す列挙型。

失敗モードと境界条件

  • store が NULL の場合、または store->load が未初期化の場合はクラッシュする可能性があります。
  • 指定された group, name, aspect に対応するデータが存在しない場合、store->load の実装に従ったエラーコードが返されます。
  • ファイルシステムエラーやメモリ不足が発生した場合、APR のエラーコードが返されます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のI/Oロジックは含まれていません。デバッグ時は md_store_t の実装側（md_store_fs.c 等）を確認してください。
  • メモリ管理はすべて apr_pool_t *p に依存しています。呼び出し側でプールが適切に管理されていることを確認してください。
