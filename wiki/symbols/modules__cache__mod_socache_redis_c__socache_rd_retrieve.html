socache_rd_retrieve

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_redis.c:241

目的

Redis バックエンドからキャッシュデータを取得し、指定されたバッファへコピーする。

シグネチャまたは形状

apr_status_t socache_rd_retrieve(ap_socache_instance_t *ctx, server_rec *s,
                                 const unsigned char *id, unsigned int idlen,
                                 unsigned char *dest, unsigned int *destlen,
                                 apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: Redis インスタンスのコンテキスト。
  • s: サーバーレコード。
  • id, idlen: 取得対象のキャッシュ ID とその長さ。
  • dest: 取得したデータを格納するバッファ。
  • destlen: dest バッファのサイズ（入力時）および取得データの長さ（出力時）。
  • p: メモリ割り当て用プール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時はエラーコード。

前提条件と不変条件

  • ctx は初期化済みであること。
  • destlen は、取得されるデータサイズ以上の容量を指している必要がある。

アルゴリズムまたは振る舞い

1. socache_rd_id2key を呼び出し、内部 ID を Redis キー文字列へ変換する。失敗時は APR_EINVAL を返す。 2. apr_redis_getp を使用して Redis からデータを取得する。 3. Redis 側でエラーが発生した場合、APR_NOTFOUND 以外であればエラーログを出力し、そのステータスを返す。 4. 取得したデータのサイズが destlen を超える場合、エラーログを出力し APR_ENOMEM を返す。 5. 正常時は memcpy で dest にデータをコピーし、destlen を実際のデータ長で更新して APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_redis.c:394 (直接の参照を確認)
  • **呼び出し先**: socache_rd_id2key, apr_redis_getp, ap_log_error, memcpy

関連データ構造

  • ap_socache_instance_t: Redis 接続ハンドル (rc) を保持。
  • server_rec: エラーログ出力に使用。

失敗モードと境界条件

  • **キー変換失敗**: socache_rd_id2key が失敗すると APR_EINVAL を返す。
  • **データ未検出**: apr_redis_getp が APR_NOTFOUND を返した場合、そのまま呼び出し元へ伝播する。
  • **バッファオーバーフロー**: 取得データが destlen を超える場合、APR_ENOMEM を返す。
  • **Redis 通信エラー**: apr_redis_getp が返すその他のエラーコードをそのまま返す。

保守担当者向けメモ

  • apr_redis_getp はリクエストプール p からメモリを割り当てるため、大量のキャッシュ取得時にはメモリ消費量に注意が必要である。
  • socache_rd_id2key の変換結果を格納する buf は RD_KEY_LEN で固定されている。
