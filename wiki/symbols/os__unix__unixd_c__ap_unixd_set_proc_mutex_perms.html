ap_unixd_set_proc_mutex_perms

定義

  • 種別: function
  • 定義箇所: os/unix/unixd.c:228

目的

プロセスが root 権限で実行されている場合に、指定されたプロセスミューテックスの所有権（ユーザーIDおよびグループID）を ap_unixd_config で設定された非特権ユーザーのものに変更し、パーミッションを調整します。

シグネチャまたは形状

apr_status_t ap_unixd_set_proc_mutex_perms(apr_proc_mutex_t *pmutex)

入力と出力

  • **入力**: pmutex (操作対象のプロセスミューテックスへのポインタ)
  • **出力**: 成功時は APR_SUCCESS を返し、失敗時はシステムエラーコード（errno）を返します。

前提条件と不変条件

  • 呼び出し元プロセスが root 権限（geteuid() == 0）で実行されている必要があります。
  • ap_unixd_config が適切に初期化され、ターゲットとなるユーザーIDおよびグループIDが設定されている必要があります。

アルゴリズムまたは振る舞い

1. 現在の実行ユーザーが root かを確認します。root でない場合は何もせず APR_SUCCESS を返します。 2. ミューテックスのメカニズムを proc_mutex_mech で判定します。 3. メカニズムに応じた処理を実行します：

  • APR_LOCK_SYSVSEM: semctl を使用し、セマフォの所有権を ap_unixd_config のユーザー/グループに変更し、パーミッションを 0600 に設定します。
  • APR_LOCK_FLOCK: chown を使用し、ロックファイルの所有権を ap_unixd_config のユーザーに変更します。

    • 4. 上記以外のメカニズムの場合は何もしません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • os/unix/unixd.c:286
  • os/unix/unixd.c:289
  • server/util_mutex.c:492
  • **呼び出し先**:
  • proc_mutex_mech
  • apr_os_proc_mutex_get
  • semctl (SysV セマフォ時)
  • apr_proc_mutex_lockfile (flock 時)
  • chown (flock 時)

関連データ構造

  • apr_proc_mutex_t: APR のプロセスミューテックス構造体。
  • ap_unixd_config: 実行ユーザーおよびグループ情報を保持するグローバル構造体。

失敗モードと境界条件

  • semctl または chown が失敗した場合、その時点の errno を返して終了します。
  • root 権限以外で実行された場合、パーミッションの変更は行われず、成功として終了します。

保守担当者向けメモ

  • 本関数は、サーバー起動時の権限降格プロセスにおいて重要な役割を果たします。
  • APR_HAS_SYSVSEM_SERIALIZE や APR_HAS_FLOCK_SERIALIZE のコンパイル時フラグに依存しているため、プラットフォームごとのサポート状況に注意してください。
