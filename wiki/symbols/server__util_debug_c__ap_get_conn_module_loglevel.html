ap_get_conn_module_loglevel

定義

  • 種別: function
  • 定義箇所: server/util_debug.c:153

目的

指定されたコネクションコンテキストにおいて、特定のモジュールに対するログレベルを解決します。モジュール固有のログレベルが設定されていない場合は、サーバーのデフォルトレベルへフォールバックします。

シグネチャまたは形状

int ap_get_conn_module_loglevel(const conn_rec *c, int module_index)

入力と出力

  • **入力**:
  • c: ログ設定を保持する conn_rec 構造体へのポインタ。
  • module_index: ログレベルを取得する対象モジュールのインデックス。
  • **出力**:
  • 解決されたログレベル（整数値）。

前提条件と不変条件

  • c は有効な conn_rec 構造体を指している必要があります。
  • module_index は、module_levels 配列の範囲内であるか、あるいは負の値である必要があります。

アルゴリズムまたは振る舞い

1. c->log が存在する場合はそれを、存在しない場合は c->base_server->log を参照先としてログ設定構造体 l を特定します。 2. 以下のいずれかの条件を満たす場合、フォールバックとして l->level を返します。

  • module_index が負である。
  • l->module_levels が NULL である。
  • l->module_levels[module_index] が負の値である。

    • 3. 上記条件に該当しない場合、l->module_levels[module_index] に格納されているモジュール固有のログレベルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: include/http_log.h 内のログ出力マクロから参照されています。
  • **呼び出し先**: 外部関数への直接的な呼び出しはありません。

関連データ構造

  • conn_rec: コネクション情報を保持する構造体。
  • ap_logconf: ログ設定を保持する構造体。module_levels 配列を介してモジュールごとのレベルを管理します。

失敗モードと境界条件

  • c->log および c->base_server->log が NULL の場合、プログラムの動作は未定義となります（ソースコード上ではポインタ参照が発生するため）。
  • module_index が module_levels 配列の確保済みサイズを超過している場合、メモリの不正アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数は http_config.h でマクロとしても定義されており、server/util_debug.c ではマクロを undef してから関数として定義しています。
  • メモリの所有権は持たず、渡された構造体内の既存データを参照するのみです。
