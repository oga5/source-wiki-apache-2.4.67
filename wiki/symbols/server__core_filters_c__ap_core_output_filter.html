ap_core_output_filter

定義

  • 種別: function
  • 定義箇所: server/core_filters.c:453

目的

HTTPレスポンスデータをフィルタチェーンから受け取り、OSのソケットへ効率的に書き出すためのコア出力フィルタです。

シグネチャまたは形状

apr_status_t ap_core_output_filter(ap_filter_t *f, apr_bucket_brigade *new_bb)

入力と出力

  • **入力**:
  • f: フィルタチェーンのコンテキスト情報を含む ap_filter_t 構造体。
  • new_bb: 送信対象のデータを含む apr_bucket_brigade（NULL の場合もある）。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • 接続が既に中断されている場合、new_bb があればクリーンアップし、APR_ECONNABORTED を返します。
  • フィルタコンテキスト (ctx) が未初期化の場合は、接続プール (c->pool) を使用して初期化されます。
  • 内部バッファ (ctx->buffered_bb) にデータが存在する場合、new_bb の先頭に結合されます。

アルゴリズムまたは振る舞い

1. **バッファ結合**: 既存のバッファと新しいブリゲードを統合します。 2. **送信判断**: should_send_brigade を呼び出し、フラッシュが必要か判定します。 3. **非ブロッキング書き込み**:

  • ソケットのタイムアウトを一時的に 0（非ブロッキング）に設定します。
  • send_brigade_nonblocking を呼び出し、ソケットへデータを転送します。
  • EAGAIN が発生し、かつフラッシュが必要な場合は apr_poll を使用して書き込み可能になるまで待機します。

    • 4. **後処理**:

  • 書き込み完了後、元のソケットタイムアウトを復元します。
  • 書き込みエラー発生時は接続を中断 (c->aborted = 1) します。
  • 残存データは setaside_remaining_output を介して次回の呼び出しのために保持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: フィルタチェーンの実行エンジン（server/util_filter.c 経由）。
  • **呼び出し先**:
  • should_send_brigade (フラッシュ判定)
  • send_brigade_nonblocking (ソケット書き込み)
  • setaside_remaining_output (残存データの保存)
  • apr_socket_timeout_get/set, apr_poll (APR ソケット操作)

関連データ構造

  • ap_filter_t: フィルタチェーンのノード。
  • core_output_filter_ctx_t: 出力フィルタの永続状態（バッファ、iovec 等）。
  • apr_bucket_brigade: 送信データのバケットリスト。

失敗モードと境界条件

  • **接続中断**: c->aborted が真の場合、即座に終了します。
  • **書き込み失敗**: send_brigade_nonblocking が EAGAIN 以外のエラーを返した場合、接続を強制終了しエラーを返します。
  • **タイムアウト**: apr_poll 待機中にタイムアウトが発生した場合、その状態が呼び出し元へ伝播します。

保守担当者向けメモ

  • 本関数は非ブロッキング I/O を前提としています。EAGAIN 発生時の再試行ロジックは、should_send_brigade のフラッシュルールに依存しています。
  • ctx->buffered_bb の生存期間は c->pool に依存するため、接続終了時のメモリ管理に注意してください。
