os_fork

定義

  • 種別: function
  • 定義箇所: os/bs2000/os.c:98

目的

BS2000（OSD/POSIX）環境において、実行環境に応じた適切なプロセス生成手法（fork() または ufork()）を選択し、プロセスを生成するラッパー関数です。

シグネチャまたは形状

pid_t os_fork(const char *user)

入力と出力

  • **入力**: user (プロセス生成時に切り替える対象のユーザー名)
  • **出力**: pid_t (生成されたプロセスの PID、またはエラー時の -1、デフォルト時の 0)

前提条件と不変条件

  • os_forktype(0) が返す bs2_ForkType 列挙値に基づき動作します。
  • bs2_UFORK モードの場合、user 引数は ufork() に渡すために大文字へ変換されます。

アルゴリズムまたは振る舞い

1. os_forktype(0) を呼び出し、プロセス生成モードを判定します。 2. モードに応じた処理を実行します。

  • bs2_FORK: 標準の fork() を実行します。
  • bs2_UFORK: user を大文字に変換し、ufork(username) を実行します。
  • その他: pid を 0 に設定します。

    • 3. ufork() 実行時に EPERM エラーが発生した場合、緊急ログを出力して exit(1) でプロセスを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/prefork/prefork.c:716
  • **呼び出し先**:
  • os_forktype()
  • fork() (標準ライブラリ)
  • ufork() (BS2000 固有)
  • apr_cpystrn()
  • ap_str_toupper()
  • ap_log_error()

関連データ構造

  • bs2_ForkType (列挙型: noFORK, FORK, UFORK)

失敗モードと境界条件

  • ufork() が EPERM を返した場合、設定不備とみなしてサーバーを緊急停止（exit(1)）させます。
  • user 名の長さは USER_LEN に制限されます。

保守担当者向けメモ

  • 本関数は _OSD_POSIX マクロが定義されている環境でのみ有効です。
  • ufork() の挙動は BS2000 のバージョンに依存する可能性があるため、大文字変換処理が含まれています。
