ap_signal_parent

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:202

目的

親プロセスに対してシャットダウンまたは再起動のシグナルを送信する。

シグネチャまたは形状

void ap_signal_parent(ap_signal_parent_e type)

入力と出力

  • **入力**: type (ap_signal_parent_e 型) - 送信するシグナルの種類（SIGNAL_PARENT_SHUTDOWN, SIGNAL_PARENT_RESTART, SIGNAL_PARENT_RESTART_GRACEFUL）。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • parent_pid および my_pid が定義されている環境で動作する。
  • SIGNAL_PARENT_RESTART および SIGNAL_PARENT_RESTART_GRACEFUL は、本MPMにおいて同等の「グレースフル再起動」として扱われる。

アルゴリズムまたは振る舞い

1. **プロセス判定**: 呼び出し元が親プロセス（parent_pid == my_pid）である場合、グローバルなイベントハンドル（shutdown_event または restart_event）に対して直接 SetEvent を実行する。 2. **子プロセスからの呼び出し**: 子プロセスから呼び出された場合、シグナル名（signal_shutdown_name または signal_restart_name）を特定する。 3. **イベント通知**: OpenEvent で対象のイベントハンドルを取得し、SetEvent を実行して親プロセスに通知する。 4. **後処理**: 取得したイベントハンドルは CloseHandle で解放する。失敗時は ap_log_error を使用して緊急ログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/child.c
  • server/mpm/winnt/mpm_winnt.c
  • server/mpm/winnt/service.c
  • **呼び出し先**:
  • Windows API: SetEvent, OpenEvent, CloseHandle, ap_log_error（Apacheログ関数）。

関連データ構造

  • ap_signal_parent_e: シグナル種別を定義する列挙型。
  • shutdown_event, restart_event: 親プロセスが待機している同期用イベントハンドル。
  • signal_shutdown_name, signal_restart_name: 子プロセスが親プロセスを特定するために使用するイベント名文字列。

失敗モードと境界条件

  • OpenEvent が失敗した場合、親プロセスへのシグナル送信は行われない。この際、APLOG_EMERG レベルでエラーログが出力される。
  • SetEvent が失敗した場合も同様にエラーログが出力され、ハンドルがクローズされる。

保守担当者向けメモ

  • 本関数はWindows特有のイベントオブジェクトを利用したプロセス間通信（IPC）に依存している。
  • 子プロセスから親プロセスをシグナルする場合、イベント名による名前付きオブジェクトの解決が必要となるため、setup_signal_names 等で事前に名前が正しく設定されていることが前提となる。
