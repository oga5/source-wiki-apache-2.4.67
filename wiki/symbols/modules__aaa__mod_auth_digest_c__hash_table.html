hash_table

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_auth_digest.c:128

目的

HTTP Digest 認証におけるクライアント状態（client_entry）を管理するためのハッシュテーブル構造体です。共有メモリ（client_shm）上に配置され、クライアントの追跡に使用されます。

シグネチャまたは形状

struct hash_table {
    client_entry  **table;        /* エントリの配列 */
    unsigned long   tbl_len;      /* テーブルのサイズ */
    unsigned long   num_entries;  /* 現在の登録数 */
    unsigned long   num_created;  /* 生成された総数 */
    unsigned long   num_removed;  /* 削除された総数 */
    unsigned long   num_renewed;  /* 更新された総数 */
} *client_list;

入力と出力

  • 入力: client_list を介して共有メモリ上のクライアントエントリを操作します。
  • 出力: 認証プロセスにおいて、nonce カウントや opaque 値の検証結果を保持・更新します。

前提条件と不変条件

  • 共有メモリ（client_shm）およびメモリ管理ハンドル（client_rmm）が初期化されている必要があります。
  • マルチプロセス環境下では、client_lock を用いたプロセス間同期が必須です。

アルゴリズムまたは振る舞い

  • client_list としてグローバルに保持され、initialize_tables によって初期化されます。
  • メモリ不足時には gc() 関数が呼び出され、LRU（Least Recently Used）に近い挙動で古いエントリを削除します。
  • rmm_malloc および rmm_free を介して、共有メモリ領域の動的な割り当てと解放を行います。

呼び出し元と呼び出し先

  • 呼び出し元: initialize_tables（初期化時）、gc（メモリ管理時）、および authenticate_digest_user 等の認証ロジックから参照されます。
  • 呼び出し先: apr_rmm 関連関数を呼び出し、共有メモリ領域を操作します。
  • 注: 具体的な関数呼び出しの網羅的なトレースは、提供されたソース excerpt の範囲外です。

関連データ構造

  • client_entry: ハッシュテーブルに格納される個々のクライアント状態。
  • client_shm / client_rmm: 共有メモリセグメントおよびその管理ハンドル。

失敗モードと境界条件

  • 共有メモリが利用できない環境では、nonce カウントチェックなどの機能が制限されます。
  • メモリ不足時には gc() によるエントリ削除が発生し、追跡中のクライアント情報が失われる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_auth_digest の共有メモリ管理の要です。
  • num_created 等のカウンタは統計情報として利用されます。
  • 共有メモリの競合を避けるため、必ず client_lock による排他制御を行ってください。
