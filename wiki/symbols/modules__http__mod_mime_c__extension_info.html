extension_info

定義

  • 種別: struct
  • 定義箇所: modules/http/mod_mime.c:61

目的

ファイル名拡張子に関連付けられた、HTTP レスポンス生成に必要なメタデータ（MIME タイプ、エンコーディング、言語、ハンドラ、フィルタ等）を保持するための構造体です。

シグネチャまたは形状

typedef struct extension_info {
    char *forced_type;
    char *encoding_type;
    char *language_type;
    char *handler;
    char *charset_type;
    char *input_filters;
    char *output_filters;
} extension_info;

入力と出力

  • 本構造体は、AddType、AddEncoding、AddLanguage、AddHandler、AddCharset、AddInputFilter、AddOutputFilter といった Apache 設定ディレクティブによって各フィールドが設定されます。
  • mod_mime のリクエスト処理フェーズにおいて、ファイル名から抽出された拡張子に基づき、この構造体の値が request_rec の各フィールドに適用されます。

前提条件と不変条件

  • 各フィールドは char * 型であり、設定ファイルから読み込まれた文字列を指します。
  • 設定の継承やマージ時には、apr_pmemdup を用いてメモリの深いコピーが行われ、親設定の整合性が維持されます。

アルゴリズムまたは振る舞い

  • mod_mime は、リクエストされたファイル名から拡張子を特定し、ハッシュテーブルから該当する extension_info を検索します。
  • 複数の拡張子を持つファイル（例: file.tar.gz）の場合、右から左へ順次解析が行われ、メタデータが累積的に適用されます。
  • 設定の削除（RemoveType 等）が行われる場合、マージ時に remove_items 配列と照合され、該当するフィールドがクリアされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • add_extension_info: 設定ディレクティブの解析時に本構造体を生成・更新します。
  • merge_mime_dir_configs: ディレクトリ設定のマージ時に使用されます。
  • find_ct: リクエスト処理時にハッシュテーブルから本構造体を取得します。
  • **呼び出し先**:
  • apr_pmemdup, apr_pcalloc: メモリ確保に使用されます。
  • apr_hash_get: ハッシュテーブルからの検索に使用されます。

関連データ構造

  • mime_dir_config: 本構造体をハッシュテーブル（extension_mappings）として保持するディレクトリ設定構造体。
  • request_rec: 本構造体に格納されたメタデータが最終的に適用される Apache リクエスト構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等のメモリ割り当て関数が失敗する可能性があります。
  • 不正な設定や重複するディレクティブが指定された場合、設定の順序に依存してメタデータが上書きされます。
  • フィルタの重複チェックは行われないため、設定次第で同一フィルタが複数回追加される可能性があります。

保守担当者向けメモ

  • charset_type 等のメンバオフセットは APR_OFFSETOF マクロを用いて動的に参照されており、構造体のメンバ順序を変更する際は注意が必要です。
  • EBCDIC 環境における文字コード処理には一部課題が残されています。
  • フィルタの重複登録を防ぐためのロジックは現在の実装には含まれていません。
