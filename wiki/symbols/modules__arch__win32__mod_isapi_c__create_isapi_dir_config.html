create_isapi_dir_config

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_isapi.c:83

目的

Apache HTTP Server のディレクトリ単位の設定構造体 (isapi_dir_conf) を初期化し、メモリを割り当てるために使用されます。

シグネチャまたは形状

static void *create_isapi_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体を割り当てるためのメモリプール。
  • char *dummy: Apache の設定フックのシグネチャに合わせるための未使用引数。
  • **出力**:
  • void *: 初期化された isapi_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • 割り当てられた isapi_dir_conf の各設定項目は、初期値として ISAPI_UNDEF が設定されます。
  • メモリは指定されたプール p から割り当てられるため、リクエストまたはサーバーのライフサイクル終了時に自動的に解放されます。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、isapi_dir_conf サイズのメモリブロックをプールから確保します。 2. 以下の設定項目を ISAPI_UNDEF で初期化します。

  • read_ahead_buflen
  • log_unsupported
  • log_to_errlog
  • log_to_query
  • fake_async

    • 3. 初期化済みの構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_isapi.c:1722 (ディレクトリごとの設定作成フックとして登録)。
  • **呼び出し先**: apr_palloc (APR メモリ管理関数)。

関連データ構造

  • isapi_dir_conf: ディレクトリごとの設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させるか、プロセスを終了させる可能性があります（APR のメモリ管理ポリシーに依存）。

保守担当者向けメモ

  • 本関数は Apache の設定マージプロセスの一部として呼び出されます。設定項目を追加する場合は、必ず ISAPI_UNDEF で初期化し、merge_isapi_dir_configs 関数側でもマージロジックを更新してください。
