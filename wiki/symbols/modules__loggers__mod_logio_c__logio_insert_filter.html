logio_insert_filter

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_logio.c:224

目的

リクエスト処理において、TTFB（Time To First Byte）計測用の出力フィルタを条件付きで挿入します。

シグネチャまたは形状

static void logio_insert_filter(request_rec *r)

入力と出力

  • 入力: request_rec *r（現在のリクエスト情報）
  • 出力: なし（副作用としてフィルタの追加を行う）

前提条件と不変条件

  • r->per_dir_config から取得した logio_dirconf_t 設定構造体が有効であること。
  • logio_module が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリ設定から logio_dirconf_t を取得します。 2. 設定内の track_ttfb フラグを確認します。 3. track_ttfb が真の場合、ap_add_output_filter を呼び出し、logio_ttfb_filter_name で定義された出力フィルタをリクエストおよび接続に対して追加します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/loggers/mod_logio.c:266 にて ap_hook_insert_filter に登録されています。
  • 呼び出し先:
  • ap_get_module_config
  • ap_add_output_filter

関連データ構造

  • request_rec: リクエストごとの情報を保持する構造体。
  • logio_dirconf_t: ディレクトリごとの設定（TTFB追跡の有効/無効）を保持する構造体。

失敗モードと境界条件

  • conf->track_ttfb が偽の場合、フィルタは挿入されず、TTFBの計測は行われません。

保守担当者向けメモ

  • 本関数は APR_HOOK_LAST としてフックに登録されており、他のフィルタ挿入処理の後に実行されることが意図されています。
  • フィルタの削除は、挿入された logio_ttfb_filter 内部で自己削除（ap_remove_output_filter）される仕組みになっています。
