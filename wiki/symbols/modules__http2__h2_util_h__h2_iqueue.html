h2_iqueue

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.h:76

目的

ストリーム ID などの整数値を管理するための、ソート可能な整数キュー構造体です。

シグネチャまたは形状

typedef struct h2_iqueue {
    int *elts;           /* 整数配列 */
    int head;            /* 先頭インデックス */
    int nelts;           /* 格納されている要素数 */
    int nalloc;          /* 確保されている配列サイズ */
    apr_pool_t *pool;    /* メモリ管理用プール */
} h2_iqueue;

入力と出力

  • 入力: apr_pool_t を介したメモリ確保と、整数値の追加・操作。
  • 出力: キュー内の整数値の取得、およびソートされた状態の保持。

前提条件と不変条件

  • pool は apr_pool_t により管理されます。
  • elts 配列は nalloc に基づいて動的に拡張されます。
  • 本構造体自体はスレッドセーフではありません（明示的なロックが必要です）。

アルゴリズムまたは振る舞い

  • iq_grow: キューの容量を拡張します。
  • iq_swap: キュー内の要素を入れ替えます。
  • iq_bubble_up: ソート処理の一部として要素を適切な位置へ移動させます。
  • 内部的には動的配列として実装されており、ソート操作をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • modules/http2/h2_mplx.c: ready_to_process 等の管理。
  • modules/http2/h2_mplx.h: ストリーム ID 管理用メンバとして定義。
  • modules/http2/h2_session.h: out_c1_blocked, ready_to_process として利用。
  • **呼び出し先**:
  • modules/http2/h2_util.c: iq_grow, iq_swap, iq_bubble_up などの内部関数が操作を行います。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • h2_mplx: 本キューをストリーム管理のために保持する多重化器。

失敗モードと境界条件

  • メモリ不足時: apr_pool_t の制約に従い、メモリ確保に失敗した場合はプロセスが異常終了する可能性があります。
  • 境界条件: nelts が 0 の状態での操作や、nalloc を超える要素追加時の拡張処理が境界となります。

保守担当者向けメモ

  • 本構造体は h2_util.c 内の静的関数によって操作されることが前提です。
  • スレッド間での共有が必要な場合は、呼び出し側で適切な同期（mutex 等）を確保してください。
