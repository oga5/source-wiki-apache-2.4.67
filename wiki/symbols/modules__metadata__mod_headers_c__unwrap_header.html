unwrap_header

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_headers.c:182

目的

HTTPヘッダー文字列に含まれる改行コード（CRまたはLF）をスペースに置換し、ヘッダーの安全な処理を保証する。

シグネチャまたは形状

static const char *unwrap_header(apr_pool_t *p, const char *hdr)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • hdr: 処理対象のヘッダー文字列。
  • **出力**:
  • 改行がスペースに置換された文字列。改行が含まれない場合は元の文字列をそのまま返す。

前提条件と不変条件

  • 入力文字列 hdr が NULL でないこと。
  • 文字列の複製が必要な場合、指定されたプール p からメモリが割り当てられる。

アルゴリズムまたは振る舞い

1. ap_strchr_c を使用して、文字列内に APR_ASCII_LF または APR_ASCII_CR が存在するか確認する。 2. 改行が含まれる場合、apr_pstrdup を用いて文字列をプール上に複製する。 3. 複製した文字列を走査し、各改行文字を APR_ASCII_BLANK（スペース）に置換する。 4. 処理済みの文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_headers.c:202
  • modules/metadata/mod_headers.c:212
  • **呼び出し先**:
  • ap_strchr_c (Apache API)
  • apr_pstrdup (APR API)

関連データ構造

  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • 入力文字列に改行が含まれない場合、置換処理は行われず、入力ポインタがそのまま返される。
  • メモリ不足時は apr_pstrdup が例外を発生させる可能性がある（APRの標準的な挙動に依存）。

保守担当者向けメモ

  • 本関数はヘッダー値のサニタイズを目的としており、HTTPヘッダーインジェクション攻撃の防止に寄与する。
  • 処理対象の文字列が非常に長い場合、apr_pstrdup によるメモリコピーが発生するため、パフォーマンスへの影響を考慮すること。
