socache_rd_init

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_redis.c:93

目的

Redis バックエンドを利用した共有オブジェクトキャッシュ（socache）インスタンスを初期化し、接続先サーバーのプールを構築する。

シグネチャまたは形状

apr_status_t socache_rd_init(ap_socache_instance_t *ctx,
                             const char *namespace,
                             const struct ap_socache_hints *hints,
                             server_rec *s, apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: 初期化対象の ap_socache_instance_t。servers メンバにカンマ区切りのサーバーリストが格納されている必要がある。
  • namespace: キャッシュキーのプレフィックスとして使用する文字列。
  • s: サーバー設定を取得するための server_rec。
  • p: メモリ割り当てに使用するプール。
  • **出力**:
  • 成功時は APR_SUCCESS を返す。
  • 失敗時は apr_status_t エラーコードを返す。
  • ctx->rc に初期化済みの apr_redis_t ハンドルが設定される。
  • ctx->tag に namespace を含む文字列が設定される。

前提条件と不変条件

  • ctx->servers に有効な Redis サーバーリスト（host:port 形式）が含まれていること。
  • ctx->taglen は 16 バイト以下であること（AP_DEBUG_ASSERT による制約）。
  • socache_redis_module の設定構造体 socache_rd_svr_cfg がサーバー設定に紐付いていること。

アルゴリズムまたは振る舞い

1. ap_mpm_query を使用してスレッド制限数を取得する。 2. ctx->servers を解析し、サーバーの総数をカウントする。 3. apr_redis_create を呼び出し、Redis オブジェクトを生成する。 4. サーバーリストを再度走査し、各サーバーに対して以下の処理を行う：

  • apr_parse_addr_port でホスト名とポートを抽出（ポート未指定時は RD_DEFAULT_SERVER_PORT を使用）。
  • apr_redis_server_create でサーバー接続設定（タイムアウト、TTL 等）を生成。
  • apr_redis_add_server でプールにサーバーを追加。

    • 5. namespace を基に ctx->tag を生成し、その長さを ctx->taglen に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_redis.c:391（socache_redis_module のプロバイダー定義の一部として参照）。
  • **呼び出し先**: apr_redis_create, apr_redis_server_create, apr_redis_add_server, apr_parse_addr_port, ap_mpm_query, ap_get_module_config。

関連データ構造

  • ap_socache_instance_t: Redis インスタンスのコンテキスト。
  • socache_rd_svr_cfg: サーバーごとの接続設定（TTL, 読み書きタイムアウト）。
  • apr_redis_t: APR Redis クライアントハンドル。

失敗モードと境界条件

  • サーバーリストの解析失敗、または apr_redis_create / apr_redis_server_create / apr_redis_add_server のいずれかが失敗した場合、エラーログを出力して apr_status_t を返す。
  • ホスト名が指定されていない場合、APR_EINVAL を返す。
  • ctx->taglen が 16 バイトを超える場合、デバッグビルドではアサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は mod_socache_redis の初期化フェーズで一度だけ実行される。
  • ctx->taglen の制約（16 バイト以下）は、キャッシュキーの生成ロジックに直結するため、名前空間の設計時に注意が必要。
  • apr_redis ライブラリのバージョン要件（APU_MAJOR_VERSION 1.6 以上）が満たされている環境でのみ動作する。
