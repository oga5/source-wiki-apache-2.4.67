ap_regex_t

定義

  • 種別: struct
  • 定義箇所: include/ap_regex.h:108

目的

PCRE (Perl Compatible Regular Expressions) ライブラリをラップし、Apache HTTP Server 内で正規表現のコンパイル済み状態を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    void *re_pcre;
    int re_nsub;
    apr_size_t re_erroffset;
} ap_regex_t;

入力と出力

  • re_pcre: 内部的な PCRE コンパイル済みパターンへのポインタ。
  • re_nsub: 正規表現内のキャプチャグループ数。
  • re_erroffset: コンパイルエラー発生時のオフセット位置。

前提条件と不変条件

  • ap_regcomp() を使用して初期化する必要があります。
  • 使用後は ap_regfree() を呼び出し、関連するメモリを解放する必要があります。
  • re_nsub はコンパイル結果に基づき設定されます。

アルゴリズムまたは振る舞い

  • 本構造体は、PCRE の詳細を隠蔽し、POSIX 互換のインターフェースを提供するためのコンテナとして機能します。
  • ap_regcomp で生成され、ap_regexec や ap_regexec_len でマッチング処理に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_pregcomp (httpd.h) 等のコンパイル関数、および正規表現を利用する各モジュール（mod_rewrite 等）。
  • **呼び出し先**: 内部的に PCRE ライブラリを呼び出す util_pcre.c の各関数。

関連データ構造

  • ap_regmatch_t: マッチした文字列のオフセット情報を格納するために ap_regexec 等と併用されます。
  • apr_pool_t: ap_pregcomp を使用する場合、メモリ管理のために利用されます。

失敗モードと境界条件

  • コンパイルに失敗した場合、ap_regcomp は非ゼロの値を返し、re_erroffset にエラー位置が格納されます。
  • 未初期化の ap_regex_t を ap_regexec 等に渡した場合の挙動は未定義です。

保守担当者向けメモ

  • 本構造体は regex_t から名称変更された経緯があります (ap_mmn.h 参照)。
  • re_nsub は過去に apr_size_t から int へ変更されています。
  • PCRE の仕様に依存するため、ライブラリの更新時には挙動の変化に注意してください。
