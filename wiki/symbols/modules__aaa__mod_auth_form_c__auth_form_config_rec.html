auth_form_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_auth_form.c:50

目的

Apache HTTP Server の mod_auth_form モジュールにおいて、ディレクトリ単位のフォームベース認証設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    authn_provider_list *providers;
    char *dir;
    int authoritative;
    int authoritative_set;
    const char *site;
    int site_set;
    const char *username;
    int username_set;
    const char *password;
    int password_set;
    apr_size_t form_size;
    int form_size_set;
    int fakebasicauth;
    int fakebasicauth_set;
    const char *location;
    int location_set;
    const char *method;
    int method_set;
    const char *mimetype;
    int mimetype_set;
    const char *body;
    int body_set;
    int disable_no_store;
    int disable_no_store_set;
    ap_expr_info_t *loginsuccess;
    int loginsuccess_set;
    ap_expr_info_t *loginrequired;
    int loginrequired_set;
    ap_expr_info_t *logout;
    int logout_set;
} auth_form_config_rec;

入力と出力

本構造体は設定値のコンテナであり、直接的な関数入力・出力はありません。各フィールドは httpd.conf 等のディレクティブから読み込まれた設定値を保持します。_set で終わる各整数フィールドは、対応する設定値が明示的に指定されたかを示すフラグとして機能します。

前提条件と不変条件

  • mod_session がロードされている必要があります。
  • ap_expr_info_t 型のフィールド（loginsuccess, loginrequired, logout）は、Apache の式評価エンジンによって評価される動的な式を保持します。

アルゴリズムまたは振る舞い

  • 設定のマージ時（merge_auth_form_dir_config）に、親ディレクトリの設定と現在の設定を統合するために使用されます。
  • 認証フロー実行時、check_authn 等の関数がこの構造体を参照し、認証プロバイダーのリストやフォームフィールド名、リダイレクト先などのパラメータを取得します。

呼び出し元と呼び出し先

  • 呼び出し元: create_auth_form_dir_config（初期化）、merge_auth_form_dir_config（マージ）、および各ディレクティブ設定関数。
  • 呼び出し先: 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • authn_provider_list: 認証プロバイダーのリスト。
  • ap_expr_info_t: 動的式評価のための構造体。

失敗モードと境界条件

  • mod_session が利用できない場合、認証情報の永続化が失敗します。
  • form_size を超えるリクエストボディは、解析時に制限される可能性があります。

保守担当者向けメモ

  • 各設定項目には _set フラグが存在するため、設定の継承やマージを行う際は、これらのフラグを適切にチェックしてデフォルト値の上書きを制御してください。
  • メモリ割り当ては apr_pcalloc を使用しているため、リクエストプールまたは設定プールのライフサイクルに従います。
