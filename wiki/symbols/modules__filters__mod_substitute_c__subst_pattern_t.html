subst_pattern_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_substitute.c:47

目的

mod_substitute モジュールにおいて、レスポンスボディに対して適用する個別の置換ルールを保持するために使用されます。

シグネチャまたは形状

typedef struct subst_pattern_t {
    const apr_strmatch_pattern *pattern;
    const ap_regex_t *regexp;
    const char *replacement;
    apr_size_t replen;
    apr_size_t patlen;
    int flatten;
    const char *from;
} subst_pattern_t;

入力と出力

本構造体はデータ保持用であり、関数ではありません。set_pattern 関数によって設定値がコンパイル・格納され、do_pattmatch 関数によって置換ロジックのパラメータとして参照されます。

前提条件と不変条件

  • pattern（文字列検索用）または regexp（正規表現用）のいずれかが有効である必要があります。
  • flatten は、置換処理においてバッファを結合するモード（flatten モード）を使用するかどうかを示すフラグです。

アルゴリズムまたは振る舞い

  • do_pattmatch 関数において、この構造体に格納されたパターン情報に基づき、対象行に対する検索と置換が実行されます。
  • flatten が有効な場合、ap_varbuf を用いた文字列の結合・置換が行われます。無効な場合は、バケット分割によるインプレース置換が試行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: set_pattern（初期化時）、substitute_filter（フィルタ実行時）、do_pattmatch（置換実行時）。
  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • subst_dir_conf: 本構造体の配列を保持し、ディレクトリごとの設定を管理します。
  • apr_strmatch_pattern: 高速文字列検索用パターン。
  • ap_regex_t: 正規表現コンパイル済みオブジェクト。

失敗モードと境界条件

  • パターンや正規表現のコンパイルに失敗した場合、本構造体の初期化段階でエラーが発生します。
  • replen や patlen の値が不整合な場合、置換処理時にメモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • flatten モードはメモリ消費が激しいため、パフォーマンス上の制約がある環境では注意が必要です。
  • 本構造体は subst_dir_conf 内の配列として管理されるため、設定変更時のメモリ確保および解放のライフサイクルに注意してください。
