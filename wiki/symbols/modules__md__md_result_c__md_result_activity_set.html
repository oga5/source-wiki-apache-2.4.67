md_result_activity_set

定義

  • 種別: function
  • 定義箇所: modules/md/md_result.c:69

目的

md_result_t オブジェクトに、現在実行中の処理アクティビティを示す文字列を設定します。

シグネチャまたは形状

void md_result_activity_set(md_result_t *result, const char *activity)

入力と出力

  • result: 更新対象の md_result_t 構造体へのポインタ。
  • activity: 設定するアクティビティ名を表す文字列（NULL許容）。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • result は有効な md_result_t オブジェクトを指している必要があります。
  • result が保持する apr_pool_t を使用して、activity 文字列の複製が内部的に作成されます。

アルゴリズムまたは振る舞い

1. activity が NULL でない場合、apr_pstrdup を使用して result->p プール上に文字列を複製します。 2. activity が NULL の場合は NULL をそのまま使用します。 3. 複製した文字列（または NULL）を引数として md_result_activity_setn を呼び出し、内部状態を更新します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt には記載がありません。
  • 呼び出し先: md_result_activity_setn（内部関数）、apr_pstrdup（APRライブラリ）。

関連データ構造

  • md_result_t: 処理結果や状態を管理するメイン構造体。
  • apr_pool_t: メモリ管理に使用される APR プール。

失敗モードと境界条件

  • result が NULL の場合、apr_pstrdup 呼び出し時にクラッシュする可能性があります。
  • メモリ不足時には apr_pstrdup がアボートする可能性があります（APRの標準的な挙動）。

保守担当者向けメモ

  • 本関数は md_result_activity_setn のラッパーであり、文字列のメモリ管理（複製）を自動化する役割を担います。
  • 状態更新に伴うコールバック（on_change 等）の実行は、内部で呼び出される md_result_activity_setn に依存します。
