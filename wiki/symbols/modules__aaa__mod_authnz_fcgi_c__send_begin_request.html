send_begin_request

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:264

目的

FastCGI プロトコルにおける FCGI_BEGIN_REQUEST レコードを構築し、バックエンドサーバーへ送信することで、FastCGI 通信の開始を通知します。

シグネチャまたは形状

apr_status_t send_begin_request(request_rec *r,
                                const fcgi_provider_conf *conf,
                                apr_socket_t *s, int role,
                                apr_uint16_t request_id)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • conf: FastCGI プロバイダーの設定情報。
  • s: バックエンドへの接続済みソケット。
  • role: FastCGI ロール（例: AP_FCGI_AUTHORIZER）。
  • request_id: 通信を一意に識別する ID。
  • **出力**:
  • 戻り値: apr_status_t 型。送信処理の成功または失敗を示すステータスコード。

前提条件と不変条件

  • s は有効な接続済みソケットであること。
  • AP_FCGI_KEEP_CONN フラグは設定されず、接続の維持は行わない前提でリクエストボディが構築されます。

アルゴリズムまたは振る舞い

1. ap_fcgi_fill_in_header を使用し、AP_FCGI_BEGIN_REQUEST 型の FastCGI ヘッダーを生成します。 2. ap_fcgi_fill_in_request_body を使用し、指定された role を含むリクエストボディを生成します。 3. ヘッダーとボディをそれぞれバイト配列（farray, abrb）に変換します。 4. iovec 配列を用いてヘッダーとボディを結合し、sendv_data を呼び出してソケット経由でバックエンドへ送信します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:734
  • modules/proxy/mod_proxy_fcgi.c:1044（注: ツールによる発見であり、直接的なコードの依存関係はソース抜粋からは確認できません）
  • **呼び出し先**:
  • ap_fcgi_fill_in_header
  • ap_fcgi_fill_in_request_body
  • ap_fcgi_header_to_array
  • ap_fcgi_begin_request_body_to_array
  • sendv_data

関連データ構造

  • fcgi_provider_conf: プロバイダー設定を保持。
  • ap_fcgi_header: FastCGI プロトコルヘッダー構造体。
  • ap_fcgi_begin_request_body: BEGIN_REQUEST 用のボディ構造体。
  • struct iovec: 送信データバッファのベクトル。

失敗モードと境界条件

  • sendv_data が失敗した場合、そのエラーコードがそのまま返されます。
  • ソケット s が無効な場合や、ネットワークエラーが発生した場合は apr_status_t にエラーが反映されます。

保守担当者向けメモ

  • 本関数は util_fcgi.h で定義されたユーティリティ関数に強く依存しています。
  • AP_FCGI_KEEP_CONN を明示的に無効化しているため、接続の再利用が必要な場合はこのロジックの修正が必要です。
