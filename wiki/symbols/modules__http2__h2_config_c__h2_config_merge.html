h2_config_merge

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_config.c:179

目的

親コンテキスト（basev）と子コンテキスト（addv）の設定構造体をマージし、新しい設定構造体を生成する。

シグネチャまたは形状

static void *h2_config_merge(apr_pool_t *pool, void *basev, void *addv)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する APR プール。
  • basev: 親コンテキストの設定構造体（h2_config *）。
  • addv: 子コンテキストの設定構造体（h2_config *）。
  • **出力**:
  • マージ結果を保持する新しい h2_config 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は h2_config 構造体へのポインタとしてキャスト可能であること。
  • マージ処理は元の構造体を変更せず、常に新しいメモリ領域を確保して結果を返す。
  • H2_CONFIG_GET マクロにより、子設定が未設定（DEF_VAL）の場合に親設定が採用される階層的継承が保証される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して新しい h2_config 構造体 n を確保する。 2. add->name と base->name を連結し、新しい設定名を作成する。 3. 数値設定値（h2_max_streams, h2_window_size 等）は H2_CONFIG_GET マクロを用いて、子設定が未設定であれば親設定から値を継承する。 4. 複雑なデータ構造（priorities, push_list, early_headers）については、両方に値が存在する場合は apr_hash_overlay や apr_array_append 等で結合し、片方のみの場合は存在する方を採用する。 5. 最終的にマージ済みの構造体 n を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_config.c:234（h2_config_merge_svr 関数内での呼び出しが確認されている）。
  • **呼び出し先**:
  • apr_pcalloc, apr_pstrcat
  • apr_hash_overlay
  • apr_array_append
  • apr_table_overlay
  • H2_CONFIG_GET (マクロ)

関連データ構造

  • h2_config: 設定値を保持するメイン構造体。
  • apr_pool_t: メモリ管理用プール。
  • apr_hash_t, apr_array_header_t, apr_table_t: 継承される設定値の格納に使用される APR データ構造。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数が異常終了（abort）する可能性がある。
  • H2_CONFIG_GET マクロの判定基準である DEF_VAL (-1) が適切に設定されていない場合、意図しない値が継承される可能性がある。

保守担当者向けメモ

  • 本関数は h2_config 構造体の全フィールドを網羅的にマージするため、構造体にフィールドを追加した際は本関数にも反映が必要である。
  • priorities や push_list などの動的データ構造は、APR のオーバーレイ関数を使用しているため、メモリ消費量に注意すること。
