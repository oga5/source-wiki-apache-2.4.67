log_err

定義

  • 種別: function
  • 定義箇所: support/suexec.c:200

目的

suexec プログラムの実行中に発生したエラーを、システムログ（syslog）または指定されたログファイルへ記録する。

シグネチャまたは形状

static void log_err(const char *fmt, ...)

入力と出力

  • **入力**:
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（void）。内部で err_output を呼び出し、ログ出力を行う。

前提条件と不変条件

  • コンパイル時に AP_LOG_EXEC マクロが定義されている場合のみ、ログ出力処理が実行される。
  • 可変長引数は stdarg.h の規約に従う必要がある。

アルゴリズムまたは振る舞い

1. va_list を初期化し、可変長引数を受け取る。 2. err_output(1, fmt, ap) を呼び出し、エラーフラグ（1）を立ててログ出力処理へ委譲する。 3. va_end で引数リストを終了する。 4. 処理終了後、呼び出し元へ制御を戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/suexec.c 内の各検証ロジック（メモリ確保失敗時、UID/GID 不一致時、引数不足時、不正なコマンド実行時など）。
  • **呼び出し先**: err_output（support/suexec.c 内で定義）。

関連データ構造

  • va_list: 可変長引数を扱うための標準型。

失敗モードと境界条件

  • AP_LOG_EXEC が定義されていない場合、この関数は何も行わず即座に復帰する。
  • ログ出力先（syslog 等）が利用不可の場合の挙動は err_output の実装に依存する。

保守担当者向けメモ

  • 本関数は suexec のセキュリティ監査において重要な役割を果たす。ログ出力のフォーマットや出力先を変更する場合は、err_output と併せて確認すること。
  • log_err はエラー（is_error = 1）として記録するため、通常のトランザクション記録を行う log_no_err と混同しないよう注意が必要である。
