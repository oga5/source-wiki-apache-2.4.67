dav_notify_created

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:745

目的

リソース作成時に、ロック状態（ロック・ヌル状態の解消や親からのロック継承）を適切に更新・適用します。

シグネチャまたは形状

dav_error *dav_notify_created(request_rec *r,
                              dav_lockdb *lockdb,
                              const dav_resource *resource,
                              int resource_state,
                              int depth)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • lockdb: ロックデータベースのハンドル。
  • resource: 作成されたリソース。
  • resource_state: リソースの現在の状態（DAV_RESOURCE_LOCK_NULL または DAV_RESOURCE_NULL）。
  • depth: 再帰的なロック適用の深さ。
  • **出力**: 成功時は NULL、失敗時は dav_error 構造体へのポインタを返します。

前提条件と不変条件

  • resource_state が DAV_RESOURCE_LOCK_NULL の場合、当該リソースは既に親からロックを継承済みであるとみなされます。
  • resource_state が DAV_RESOURCE_NULL の場合、親リソースからロックを継承する必要があります。

アルゴリズムまたは振る舞い

1. **ロック・ヌル状態の処理**: resource_state が DAV_RESOURCE_LOCK_NULL の場合、lockdb->hooks->remove_locknull_state を呼び出して特殊なマーカーを削除します。depth > 0 の場合は dav_inherit_locks を呼び出し、子リソースへロックを伝播させます。 2. **通常状態の処理**: resource_state が DAV_RESOURCE_NULL の場合、dav_inherit_locks を呼び出して親リソースからロックを継承します。継承に失敗した場合は dav_push_error を介してエラー情報を付加して返却します。 3. **その他の状態**: リソースが既に存在する場合、ロック状態は正しいものとみなされ、何も行わずに NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c 内の各所（行 1209, 2738, 3155, 3632）から呼び出されます。
  • **呼び出し先**:
  • lockdb->hooks->remove_locknull_state
  • dav_inherit_locks
  • dav_push_error

関連データ構造

  • dav_lockdb: ロックデータベースのフックおよび状態を保持。
  • dav_resource: WebDAV リソースの抽象表現。
  • dav_error: エラー発生時の詳細情報を保持する構造体。

失敗モードと境界条件

  • dav_inherit_locks が失敗した場合、そのエラーがそのまま（またはラップされて）返されます。
  • depth が 0 の場合、ロック・ヌル状態の解消のみが行われ、再帰的な継承はスキップされます。

保守担当者向けメモ

  • resource_state が DAV_RESOURCE_NULL の場合、dav_inherit_locks に depth を渡して最適化する余地があります。
  • ロックプロバイダーによっては、ロック・ヌルリソースと通常リソースでキーの扱いが異なるため、状態遷移時にキーの変換が発生する点に注意してください。
