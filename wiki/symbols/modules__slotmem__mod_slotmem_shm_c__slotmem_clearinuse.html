slotmem_clearinuse

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:134

目的

共有メモリインスタンス内の全スロットの利用状況（in-use）フラグをリセットし、空きスロット数（num_free）を全スロット数に復元します。

シグネチャまたは形状

static void slotmem_clearinuse(ap_slotmem_instance_t *slot)

入力と出力

  • **入力**: slot (ap_slotmem_instance_t *) - 操作対象となる共有メモリインスタンスへのポインタ。
  • **出力**: なし（副作用として slot 構造体内部の inuse 配列および num_free カウンタを更新）。

前提条件と不変条件

  • slot が NULL の場合、関数は何もせず即座に復帰します。
  • slot->inuse は、スロット数分確保された配列を指している必要があります。
  • slot->num_free は、現在の空きスロット数を保持する有効なポインタである必要があります。

アルゴリズムまたは振る舞い

1. slot が NULL でないことを確認します。 2. slot->inuse 配列の先頭アドレスを取得します。 3. slot->desc->num で指定された全スロットに対してループ処理を行います。 4. 各スロットの inuse フラグが立っている（*inuse != 0）場合：

  • inuse フラグを 0 にクリアします。
  • slot->num_free が指す値をインクリメントし、空きスロット数を増やします。

    • 5. ループ終了後、すべてのスロットが未使用状態となります。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/slotmem/mod_slotmem_shm.c:176 にて呼び出されていることが確認されています。
  • **呼び出し先**: 本関数内での関数呼び出しはありません。

関連データ構造

  • ap_slotmem_instance_t: 共有メモリインスタンスの管理構造体。本関数はこの構造体内の inuse 配列と num_free を直接操作します。
  • sharedslotdesc_t: スロットの構成情報（num）を保持しており、ループの終了条件に使用されます。

失敗モードと境界条件

  • slot が NULL の場合、安全に処理を中断します。
  • slot->desc が初期化されていない場合、slot->desc->num へのアクセスによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は共有メモリ内のフラグを直接操作するため、呼び出し側で適切な排他制御が行われていることを前提としています。
  • num_free の更新は、スロットの解放数と正確に一致させる必要があるため、ループ内のロジック変更時には注意してください。
