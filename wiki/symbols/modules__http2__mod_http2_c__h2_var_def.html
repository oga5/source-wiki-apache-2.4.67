h2_var_def

定義

  • 種別: struct
  • 定義箇所: modules/http2/mod_http2.c:293

目的

HTTP/2 関連の環境変数を定義し、それらの解決ロジックと環境変数への注入可否を管理するための構造体です。

シグネチャまたは形状

typedef struct h2_var_def {
    const char *name;
    h2_var_lookup *lookup;
    unsigned int  subprocess : 1;    /* should be set in r->subprocess_env */
} h2_var_def;

入力と出力

  • **入力**: 構造体の各フィールド（変数名、値解決用関数ポインタ、環境変数設定フラグ）。
  • **出力**: なし（本構造体は定義のみ）。

前提条件と不変条件

  • lookup フィールドは、HTTP/2 関連の値を計算する関数（h2_var_lookup 型）を指す必要があります。
  • subprocess はビットフィールドとして定義されており、r->subprocess_env への注入制御に使用されます。

アルゴリズムまたは振る舞い

本構造体は、H2_VARS 配列の要素として使用されます。mod_http2 内の変数解決処理において、この構造体を通じて各変数の名前解決と、subprocess_env への値の反映が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/mod_http2.c 内で H2_VARS 配列の定義に使用されています。
  • modules/http2/mod_http2.c 内の変数解決ループ（vdef = &H2_VARS[i]）において、各変数の定義を参照するために使用されています。
  • **呼び出し先**:
  • 構造体自体はデータ定義であるため、直接の呼び出し先はありません。ただし、lookup フィールドに格納された関数ポインタが実行時に呼び出されます。

関連データ構造

  • H2_VARS: h2_var_def 型の静的配列。
  • h2_var_lookup: 変数値を計算するための関数型。

失敗モードと境界条件

  • H2_VARS 配列の走査時に、インデックスが範囲外にならないよう注意が必要です。
  • lookup 関数ポインタが NULL の場合、実行時にクラッシュする可能性があるため、初期化時に適切に設定されている必要があります。

保守担当者向けメモ

  • 本構造体は mod_http2 が提供する環境変数のインターフェースを定義しています。新しい HTTP/2 関連変数を追加する場合は、H2_VARS 配列に新しい h2_var_def インスタンスを追加してください。
  • subprocess フラグは、変数が Apache の subprocess_env に公開されるべきかどうかを制御します。外部モジュール（mod_rewrite 等）から参照される変数は、このフラグを適切に設定してください。
