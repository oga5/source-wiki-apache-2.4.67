ssl_config_server_merge

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_config.c:392

目的

親サーバー設定（basev）と子サーバー設定（addv）を統合し、新しいサーバー設定構造体を作成する。

シグネチャまたは形状

void *ssl_config_server_merge(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親となる SSLSrvConfigRec 構造体（voidポインタ）。
  • addv: 子となる SSLSrvConfigRec 構造体（voidポインタ）。
  • **出力**:
  • マージ結果を保持する新しい SSLSrvConfigRec 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は SSLSrvConfigRec 型としてキャスト可能であること。
  • ssl_config_server_new(p) が成功し、メモリが確保されること。

アルゴリズムまたは振る舞い

1. ssl_config_server_new(p) を呼び出し、マージ結果を格納する新しい構造体 mrg を生成する。 2. cfgMerge マクロ群を使用し、add が未設定（UNSET）の場合に base の値を採用する形式で各フィールドを統合する。

  • 対象フィールド: mc, enabled, session_cache_timeout, cipher_server_pref, insecure_reneg。
  • 条件付き対象: strict_sni_vhost_check（HAVE_TLSEXT 定義時）、compression（OPENSSL_NO_COMP 未定義時）。

    • 3. modssl_ctx_cfg_merge_server を呼び出し、サーバーコンテキスト固有の複雑な設定項目をマージする。 4. 統合された mrg 構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/mod_ssl.c:768（サーバー設定のマージ用ハンドラとして登録）。
  • **呼び出し先**:
  • ssl_config_server_new
  • modssl_ctx_cfg_merge_server

関連データ構造

  • SSLSrvConfigRec: サーバーごとの SSL 設定を保持する主要構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pool_t による割り当て失敗により、プロセスが異常終了する可能性がある。
  • basev または addv が NULL の場合: キャスト後のアクセスによりセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • cfgMerge マクロは単純な継承（add が UNSET なら base を採用）を行うため、配列や複雑なオブジェクトの統合が必要な場合は modssl_ctx_cfg_merge_server 側の実装を確認すること。
  • HAVE_TLSEXT や OPENSSL_NO_COMP といったプリプロセッサ条件により、コンパイル環境によってマージ対象のフィールドが変動する。
