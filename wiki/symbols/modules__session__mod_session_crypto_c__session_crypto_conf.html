session_crypto_conf

定義

  • 種別: struct
  • 定義箇所: modules/session/mod_session_crypto.c:55

目的

mod_session_crypto モジュールにおけるサーバー全体の設定を保持するための構造体です。暗号化に使用するライブラリ（ドライバ）およびそのパラメータを管理します。

シグネチャまたは形状

typedef struct {
    const char *library;
    const char *params;
    int library_set;
} session_crypto_conf;

入力と出力

  • **入力**: 設定ファイル（httpd.conf 等）で指定された SessionCryptoDriver および SessionCryptoParam ディレクティブの値。
  • **出力**: サーバープロセス全体で共有される暗号化設定の保持。

前提条件と不変条件

  • APU_HAVE_CRYPTO が有効である環境で利用されます。
  • library_set は、library が設定されたかどうかを示すフラグとして機能します。

アルゴリズムまたは振る舞い

  • サーバー起動時に session_crypto_init がこの構造体を参照し、指定された library を用いて apr_crypto ドライバをロードします。
  • サーバー設定の初期化およびマージ処理において、この構造体のインスタンスが生成・参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session_crypto.c: サーバー設定の取得時（ap_get_module_config）および設定の初期化時（apr_pcalloc）に参照されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • session_crypto_dir_conf: ディレクトリ単位の暗号化設定を保持する構造体。
  • apr_crypto_t: apr_crypto API によって生成される暗号化コンテキスト。

失敗モードと境界条件

  • library が指定されていない場合、暗号化機能の初期化が失敗する可能性があります。
  • 設定された library がシステムに存在しない、またはロードできない場合、サーバーの起動プロセスでエラーが発生します。

保守担当者向けメモ

  • 本構造体はサーバー全体の設定を管理します。ディレクトリ単位の設定が必要な場合は session_crypto_dir_conf を参照してください。
  • params フィールドは、使用する暗号化ドライバに渡される追加のパラメータ文字列を保持します。
