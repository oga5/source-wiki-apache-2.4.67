ap_recent_ctime_ex

定義

  • 種別: function
  • 定義箇所: server/util_time.c:164

目的

指定された時刻（apr_time_t）を、オプションで指定された形式（標準、コンパクト、マイクロ秒付与、タイムゾーンオフセット付与）に従って文字列へ変換します。

シグネチャまたは形状

apr_status_t ap_recent_ctime_ex(char *date_str, apr_time_t t, int option, int *len)

入力と出力

  • date_str: 変換後の文字列を格納するバッファ（呼び出し元で十分なサイズを確保する必要がある）。
  • t: 変換対象の時刻（apr_time_t）。
  • option: 以下のフラグのビットマスク。
  • AP_CTIME_OPTION_COMPACT: コンパクト形式（YYYY-MM-DD HH:MM:SS）を選択。
  • AP_CTIME_OPTION_USEC: マイクロ秒を付与。
  • AP_CTIME_OPTION_GMTOFF: タイムゾーンオフセットを付与。
  • len: 入力時はバッファの最大長、出力時は書き込まれた文字列の長さ（終端文字を含む）。
  • 戻り値: 成功時は APR_SUCCESS、バッファ不足時は APR_ENOMEM を返す。

前提条件と不変条件

  • len が指定されている場合、関数内でバッファサイズの検証が行われます。
  • AP_TIME_RECENT_THRESHOLD（デフォルト15秒）以内の時刻に対して最適化されています。
  • date_str は、指定されたオプションに応じた十分な長さが必要です。

アルゴリズムまたは振る舞い

1. **バッファ長計算**: 指定された option に基づき、必要なバッファ長を算出します。 2. **検証**: len が必要な長さを満たしているか確認し、不足していれば APR_ENOMEM を返します。 3. **時刻分解**: ap_explode_recent_localtime() を呼び出し、キャッシュを利用して時刻を分解します。 4. **文字列構築**:

  • COMPACT オプションの有無により、日付部分のフォーマットを切り替えます。
  • 時刻（時:分:秒）を書き込みます。
  • USEC オプションがあれば、マイクロ秒を付加します。
  • COMPACT でない場合は、曜日と月名を付加します。
  • GMTOFF オプションがあれば、タイムゾーンオフセット（+HHMM 形式）を付加します。

    • 5. **終端**: 文字列の最後に \0 を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_time.c:161 (ap_recent_ctime からの内部呼び出し)
  • server/log.c:691
  • **呼び出し先**:
  • ap_explode_recent_localtime()

関連データ構造

  • apr_time_t: APR の時刻表現。
  • apr_time_exp_t: 分解された時刻構造体。
  • apr_day_snames, apr_month_snames: 曜日および月名の定数配列。

失敗モードと境界条件

  • len で指定されたバッファサイズが、オプションで要求される長さに満たない場合、APR_ENOMEM を返し、len を 0 に設定します。

保守担当者向けメモ

  • 本関数は apr_ctime() のクローンですが、キャッシュを利用した ap_explode_recent_localtime() を使用することで高速化を図っています。
  • 新しいフォーマットオプションを追加する場合は、needed の計算ロジックと文字列構築ロジックの両方を更新する必要があります。
