ssl_pphrase_Handle_CB

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:453

目的

暗号化された秘密鍵を復号するためのパスフレーズを、設定されたダイアログ方式（組み込み、パイプ、または外部フィルタ）に従って収集し、OpenSSL へ提供する。

シグネチャまたは形状

int ssl_pphrase_Handle_CB(char *buf, int bufsize, int verify, void *srv)
  • buf: パスフレーズを格納するバッファ。
  • bufsize: バッファのサイズ。
  • verify: 検証フラグ（OpenSSL のコールバック仕様に準拠）。
  • srv: pphrase_cb_arg_t 構造体へのポインタ。

入力と出力

  • **入力**: srv 引数を通じて渡される pphrase_cb_arg_t（サーバー設定、鍵ID、試行回数など）。
  • **出力**: 成功時は取得したパスフレーズの長さ（int）、失敗時は -1 を返す。
  • **副作用**: buf へのパスフレーズ書き込み、pphrase_cb_arg_t 内のキャッシュ更新、および機密データのメモリ消去（memset）。

前提条件と不変条件

  • srv は pphrase_cb_arg_t 型としてキャスト可能であること。
  • 成功時には buf に有効なパスフレーズが格納され、その長さは 1 文字以上であること。
  • 失敗時には buf は memset によりゼロクリアされること。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: pphrase_array_get を使用し、既に保持されているパスフレーズがあればそれを返して終了する。 2. **ダイアログ実行**: sc->server->pphrase_dialog_type に基づき分岐する。

  • SSL_PPTYPE_BUILTIN / SSL_PPTYPE_PIPE: 端末またはパイプ経由で入力を要求。pipe_get_passwd_cb または EVP_read_pw_string を使用。空入力の場合はリトライする。
  • SSL_PPTYPE_FILTER: 外部プログラムを ssl_util_readfilter で実行し、その出力をパスフレーズとして取得する。

    • 3. **状態更新**: 取得したパスフレーズを ppcb_arg->cpPassPhraseCur に保存し、呼び出し元へ長さを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_pphrase.c 内の初期化処理（詳細はソースコード上の明示的な呼び出し箇所を参照）。
  • **呼び出し先**: pphrase_array_get, ssl_pipe_child_create, pipe_get_passwd_cb, EVP_read_pw_string, ssl_util_readfilter 等。

関連データ構造

  • pphrase_cb_arg_t: パスフレーズ収集のコンテキストを保持。
  • SSLSrvConfigRec: サーバーごとの SSL 設定（ダイアログタイプやパスを含む）。

失敗モードと境界条件

  • **パイプ作成失敗**: ssl_pipe_child_create が失敗した場合、エラーログを出力し -1 を返す。
  • **空パスフレーズ**: 1文字未満の入力はエラーとして扱われ、再入力を促す。
  • **Windows 環境**: SSL_PPTYPE_BUILTIN はサポートされておらず、エラーを返す。
  • **OpenSSL エラー**: 取得失敗時は PEMerr を呼び出し、バッファをクリアして -1 を返す。

保守担当者向けメモ

  • 本関数は OpenSSL の PEM コールバックインターフェースを実装している。
  • パスフレーズは apr_pool_t 上で管理されるため、メモリリークに注意すること。
  • 機密データは memset で消去しているが、OpenSSL のバージョンや環境によるメモリレイアウトの差異に留意が必要。
