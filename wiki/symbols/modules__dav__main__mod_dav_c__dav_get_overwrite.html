dav_get_overwrite

定義

  • 種別: function
  • 定義箇所: modules/dav/main/mod_dav.c:736

目的

HTTP リクエストヘッダー Overwrite の値を解析し、上書き許可フラグ（真偽値）を返します。

シグネチャまたは形状

static int dav_get_overwrite(request_rec *r)

入力と出力

  • **入力**: request_rec *r （Apache リクエスト構造体）
  • **出力**:
  • 1: 上書き許可（デフォルト値、またはヘッダーが "T"/"t" の場合）
  • 0: 上書き禁止（ヘッダーが "F"/"f" の場合）
  • -1: 不正なヘッダー値（エラー）

前提条件と不変条件

  • r->headers_in から Overwrite ヘッダーを取得します。
  • ヘッダー値は単一文字（'T', 't', 'F', 'f'）である必要があります。それ以外の文字列は不正とみなされます。

アルゴリズムまたは振る舞い

1. apr_table_get を使用して Overwrite ヘッダーを取得します。 2. ヘッダーが存在しない場合は、デフォルト値として 1 を返します。 3. ヘッダー値が "T" または "t" の場合は 1 を返します。 4. ヘッダー値が "F" または "f" の場合は 0 を返します。 5. 上記以外（値が空でない、または複数文字など）の場合は、ap_log_rerror を通じてエラーログを出力し、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.c:2874
  • modules/dav/main/mod_dav.c:4826
  • **呼び出し先**:
  • apr_table_get (APR API)
  • ap_log_rerror (Apache API)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • ヘッダー値が定義外の文字列（例: "True", "False", "1" など）である場合、-1 を返し、呼び出し元で HTTP_BAD_REQUEST を生成するためのトリガーとなります。

保守担当者向けメモ

  • 本関数が -1 を返した場合、呼び出し元は追加の診断情報を提供した上でリクエストを終了させる設計になっています。
