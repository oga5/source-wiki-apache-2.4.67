merge_netware_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_netware.c:59

目的

ディレクトリ単位の設定構造体 (netware_dir_config) をマージし、親ディレクトリの設定と現在のディレクトリの設定を統合した新しい設定インスタンスを生成します。

シグネチャまたは形状

static void *merge_netware_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定 (netware_dir_config 型へのポインタ)。
  • addv: 現在のディレクトリの設定 (netware_dir_config 型へのポインタ)。
  • **出力**:
  • 統合された新しい netware_dir_config 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は netware_dir_config 構造体としてキャスト可能である必要があります。
  • apr_palloc により、指定されたプール p 上に新しいメモリ領域が確保されます。

アルゴリズムまたは振る舞い

1. basev と addv を netware_dir_config 型にキャストします。 2. apr_palloc を使用して、新しい netware_dir_config 構造体用のメモリを確保します。 3. apr_table_overlay を使用して、以下のテーブルをマージします（add の設定が base の設定をオーバーライドします）。

  • file_type_handlers: 拡張子とハンドラの関連付け。
  • file_handler_mode: ハンドラの実行モード。
  • extra_env_vars: 追加の環境変数。

    • 4. 統合された新しい構造体を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_netware.c:199 (設定マージ処理として参照)。
  • **呼び出し先**: apr_palloc (APR), apr_table_overlay (APR)。

関連データ構造

  • netware_dir_config: ディレクトリ単位の設定を保持する構造体。

失敗モードと境界条件

  • apr_palloc がメモリ不足により失敗した場合、APR の規定に従い例外が発生します。
  • basev または addv が NULL の場合、apr_table_overlay の動作に依存します。

保守担当者向けメモ

  • 本関数は Apache の設定マージフェーズで呼び出されます。apr_table_overlay の性質上、add 側の設定が優先されるため、設定の継承順序（親から子へ）が正しく反映されていることを確認してください。
