ap_slotmem_type_t

定義

  • 種別: typedef
  • 定義箇所: include/ap_slotmem.h:55

目的

共有メモリ領域の特性や動作フラグを定義するための型です。

シグネチャまたは形状

typedef unsigned int ap_slotmem_type_t;

入力と出力

  • **入力**: 共有メモリの作成時（create 関数）や、プロバイダー実装内でのフラグ保持に使用されます。
  • **出力**: 該当するメモリ領域の永続化の有無やスレッドセーフティなどの特性を決定します。

前提条件と不変条件

  • unsigned int として定義されており、ビットフラグとして利用されます。
  • AP_SLOTMEM_TYPE_NOTMPSAFE フラグが指定されない場合、実装側で適切な排他制御を行うことが期待されます。

アルゴリズムまたは振る舞い

共有メモリの確保やアタッチを行う際、この型で指定されたフラグに基づいて、メモリの管理方法（永続化、同期の必要性など）が制御されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • include/ap_slotmem.h:111 (create 関数の引数)
  • modules/proxy/mod_proxy_balancer.c:908
  • modules/slotmem/mod_slotmem_plain.c:32, 66
  • modules/slotmem/mod_slotmem_shm.c:38, 341
  • **呼び出し先**:
  • 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • ap_slotmem_instance_t: 共有メモリ領域を指すハンドル。
  • ap_slotmem_provider_t: 本型を利用してメモリ操作を行う関数ポインタ群を保持する構造体。

失敗モードと境界条件

  • 不適切なフラグの組み合わせが指定された場合の挙動は、各プロバイダーの実装（mod_slotmem_shm 等）に依存します。

保守担当者向けメモ

  • 本型はビットフラグとして機能するため、新しいフラグを追加する際は既存の定義と重複しないように注意してください。
  • 実際のフラグ値の定義は include/ap_slotmem.h の 58 行目以降に記述されています。
