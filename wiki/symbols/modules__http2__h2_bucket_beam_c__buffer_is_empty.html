buffer_is_empty

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:190

目的

指定された h2_bucket_beam インスタンスにおいて、送信待ちのバケットリストが空であるかどうかを判定します。

シグネチャまたは形状

static int buffer_is_empty(h2_bucket_beam *beam)

入力と出力

  • **入力**: beam (h2_bucket_beam *) - 状態を確認する対象のバケットビーム。
  • **出力**: int - リストが空であれば非ゼロ（真）、そうでなければ 0（偽）を返します。

前提条件と不変条件

  • beam は有効な h2_bucket_beam 構造体へのポインタである必要があります。
  • 内部的に H2_BLIST_EMPTY マクロを使用して beam->buckets_to_send リストの状態を評価します。

アルゴリズムまたは振る舞い

1. beam 構造体内の buckets_to_send メンバ（h2_blist 型）を参照します。 2. H2_BLIST_EMPTY マクロを用いて、リストが空かどうかを判定し、その結果を即座に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_bucket_beam.c 内の複数の箇所から、ビームの状態監視やコールバックのトリガー判定のために呼び出されています（例: wait_not_empty 内のループ条件、beam->was_empty_cb の実行判定、h2_beam_send 後の状態チェックなど）。
  • **呼び出し先**:
  • H2_BLIST_EMPTY マクロ（リスト構造の判定用）。

関連データ構造

  • h2_bucket_beam: バケット転送を管理するメイン構造体。
  • h2_blist: APR_RING をラップした軽量なリスト構造。

失敗モードと境界条件

  • beam が NULL の場合、未定義の動作を引き起こす可能性があります。
  • 本関数自体は状態を読み取るのみであり、副作用や失敗モードは持ちません。

保守担当者向けメモ

  • 本関数は h2_bucket_beam.c 内で頻繁に利用されるユーティリティです。リストの空判定はスレッド同期の重要な判断基準となるため、変更時は同期ロジックへの影響を考慮してください。
