status_from_str

定義

  • 種別: function
  • 定義箇所: test/modules/http2/mod_h2test/mod_h2test.c:416

目的

文字列形式のテスト用エラー指定を、Apache Portable Runtime (APR) のステータスコード (apr_status_t) に変換します。

シグネチャまたは形状

static int status_from_str(const char *s, apr_status_t *pstatus)

入力と出力

  • **入力**:
  • s: 変換対象の文字列（"timeout" または "reset"）。
  • pstatus: 変換後のステータスコードを格納するポインタ。
  • **出力**:
  • 成功時（一致する文字列が見つかった場合）: 1 を返し、pstatus に対応する値を設定。
  • 失敗時（一致しない場合）: 0 を返し、pstatus は変更しない。

前提条件と不変条件

  • s は NULL であってはなりません（strcmp に渡されるため）。
  • pstatus は有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列 s を strcmp で比較します。 2. "timeout" と一致する場合、*pstatus に APR_TIMEUP を代入し 1 を返します。 3. "reset" と一致する場合、*pstatus に APR_ECONNRESET を代入し 1 を返します。 4. いずれにも一致しない場合、何も変更せず 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • test/modules/http2/mod_h2test/mod_h2test.c:465
  • test/modules/http2/mod_h2test/mod_h2test.c:476
  • **呼び出し先**:
  • strcmp (標準ライブラリ)

関連データ構造

  • apr_status_t: APR のエラーコード型。

失敗モードと境界条件

  • 入力文字列が定義されたキーワード以外の場合、関数は 0 を返し、pstatus は更新されません。呼び出し元はこの戻り値をチェックしてエラーハンドリングを行う必要があります。

保守担当者向けメモ

  • 本関数はテストモジュール専用です。新しいエラー種別を追加する場合は、この関数に else if ブロックを追加してください。
