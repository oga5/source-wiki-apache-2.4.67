util_dn_compare_node_t

定義

  • 種別: struct
  • 定義箇所: modules/ldap/util_ldap_cache.h:141

目的

LDAP の DN（Distinguished Name）同士の比較結果をキャッシュするためのデータ構造です。

シグネチャまたは形状

typedef struct util_dn_compare_node_t {
    const char *reqdn;          /* 要求された DN */
    const char *dn;             /* 検索で見つかった DN */
} util_dn_compare_node_t;

入力と出力

  • 入力: reqdn（比較対象の DN）、dn（比較基準となる DN）。
  • 出力: なし（構造体として値を保持）。

前提条件と不変条件

  • キャッシュ内のデータは util_ald_alloc を使用して確保されたメモリ領域に配置される必要があります。
  • reqdn および dn は、キャッシュのライフサイクル期間中、有効な文字列ポインタである必要があります。

アルゴリズムまたは振る舞い

本構造体は、LDAP 検索結果の DN と、要求された DN が一致するかどうかを判定するためのキャッシュエントリとして機能します。util_ldap_cache.c 内の比較関数やコピー関数において、この構造体を通じて DN の照合が行われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • modules/ldap/util_ldap.c: キャッシュ操作時のノード生成および参照。
  • modules/ldap/util_ldap_cache.c: キャッシュの比較、挿入、メモリ確保処理。
  • 呼び出し先:
  • util_ald_alloc（modules/ldap/util_ldap_cache.c 内でメモリ確保時に使用）。

関連データ構造

  • util_cache_node_t: キャッシュ内のエントリを管理する汎用リンクリスト構造体。
  • util_url_node_t: 本ノードを保持する親階層のキャッシュノード。

失敗モードと境界条件

  • メモリ不足: util_ald_alloc が失敗した場合、ノードの生成に失敗します。
  • 文字列の不一致: reqdn と dn の比較ロジックにおいて、LDAP の DN 形式の正規化が不十分な場合、誤ったキャッシュヒットまたはミスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は util_ldap_cache.c で実装されるキャッシュ操作関数群（比較・コピー・解放）の対象となります。
  • 共有メモリ環境下では、ポインタの指す先が apr_rmm によって管理されるオフセットであることを考慮する必要があります。
