cleanup_iter_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_session.c:1672

目的

HTTP/2 プロキシセッションにおいて、ストリームのクリーンアップ処理やイテレーションを行う際に、関連するセッション情報と完了通知用コールバックを保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    h2_proxy_session *session;
    h2_proxy_request_done *done;
} cleanup_iter_ctx;

入力と出力

  • session: 処理対象となる h2_proxy_session 構造体へのポインタ。
  • done: 処理完了時に呼び出される h2_proxy_request_done 型の関数ポインタ。

前提条件と不変条件

  • 本構造体は、h2_proxy_session.c 内のイテレーション処理において、ユーザーデータ（udata）として渡されることを前提としています。
  • session および done は、クリーンアップ処理が実行される時点で有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、nghttp2 のイテレーションコールバック等で、セッション状態と完了通知関数を安全に受け渡すためのコンテナとして機能します。
  • 呼び出し側で本構造体をスタック上に確保し、udata として渡すことで、コールバック関数内で session と done を参照可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_proxy_session.c 内の複数の箇所で、イテレーション処理の開始時に本構造体がインスタンス化されています（1689行目、1714行目など）。
  • 呼び出し先: 本構造体内の done フィールドに格納された関数が、イテレーションの過程で呼び出されます。

関連データ構造

  • h2_proxy_session: セッション管理のメイン構造体。
  • h2_proxy_request_done: ストリーム処理完了時に実行されるコールバックの型定義。

失敗モードと境界条件

  • udata として渡されたポインタが NULL である場合、またはキャスト先が不適切な場合、セッションのクリーンアップ処理が正しく行われず、メモリリークや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は h2_proxy_session.c 内でのみ使用されるプライベートなコンテキストです。
  • 構造体のフィールドを追加・変更する場合は、この構造体を利用しているすべてのイテレーションコールバック（udata を受け取る関数）の整合性を確認してください。
