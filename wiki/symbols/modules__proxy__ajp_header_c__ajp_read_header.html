ajp_read_header

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_header.c:725

目的

AJPソケットからパケットを読み込み、メッセージバッファを初期化または再利用して、受信したAJPメッセージの型を特定する。

シグネチャまたは形状

apr_status_t ajp_read_header(apr_socket_t *sock,
                             request_rec  *r,
                             apr_size_t buffsize,
                             ajp_msg_t **msg)

入力と出力

  • **入力**:
  • sock: 通信に使用するAPRソケット。
  • r: 現在のリクエスト構造体。
  • buffsize: メッセージバッファを新規作成する場合のサイズ。
  • msg: AJPメッセージを格納するポインタのポインタ。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS、失敗時はエラーコードを返す。
  • msg: 受信したパケットデータで更新される。

前提条件と不変条件

  • msg が指すポインタが NULL の場合、r->pool を使用して新規に ajp_msg_t が割り当てられる。
  • msg が既存のメッセージを指している場合、ajp_msg_reuse により再利用される。
  • 処理の最後には必ず ajp_msg_peek_uint8 によりパケットの先頭バイト（型情報）が検証される。

アルゴリズムまたは振る舞い

1. **バッファ準備**: *msg の有無を確認し、ajp_msg_reuse または ajp_msg_create でメモリを確保・準備する。 2. **リセット**: ajp_msg_reset を呼び出し、バッファを初期状態にする。 3. **受信**: ajp_ilink_receive を通じてソケットからバイナリパケットを読み込む。 4. **ログ出力**: 受信したパケットのダンプを記録する。 5. **型特定**: ajp_msg_peek_uint8 でパケットの型コードを読み取り、ajp_type_str を用いてログに記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ajp.c 内の proxy_ajp_handler 等（リクエスト処理ループ内）。
  • **呼び出し先**:
  • ajp_msg_reuse, ajp_msg_create, ajp_msg_reset
  • ajp_ilink_receive
  • ajp_msg_log
  • ajp_msg_peek_uint8
  • ajp_type_str
  • ap_log_rerror

関連データ構造

  • ajp_msg_t: AJPプロトコルのメッセージを保持する構造体。
  • request_rec: Apacheのリクエスト情報。
  • apr_socket_t: APRのソケット抽象化。

失敗モードと境界条件

  • ajp_msg_reuse または ajp_msg_create が失敗した場合、エラーログを出力し即時復帰する。
  • ajp_ilink_receive が失敗した場合（ソケット切断等）、エラーログを出力し即時復帰する。
  • ajp_msg_peek_uint8 が失敗した場合、パケットの型情報が取得できないためエラーとして復帰する。

保守担当者向けメモ

  • 本関数はパケットの読み込みと型特定までを担当する。パケットの内容解析（アンマーシャリング）は、この後に呼び出される関数（ajp_parse_header 等）で行われる。
  • buffsize の指定は、バックエンドからの予期される最大パケットサイズと整合している必要がある。
