parse_node_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_include.c:96

目的

SSI（Server Side Includes）ディレクティブの解析過程において、条件式や構造を表現するツリー構造のノードとして機能します。

シグネチャまたは形状

typedef struct parse_node {
    struct parse_node *parent;
    struct parse_node *left;
    struct parse_node *right;
    token_t token;
    int value;
    int done;
#ifdef DEBUG_INCLUDE
    int dump_done;
#endif
} parse_node_t;

入力と出力

  • **入力**: 解析対象のトークン（token_t）およびツリー構築時の親ノード情報。
  • **出力**: 構築されたノード構造体。value フィールドには評価結果や状態値が格納されます。

前提条件と不変条件

  • parent, left, right ポインタは、ツリー構造を維持するために使用されます。
  • done フラグは、当該ノードの処理完了状態を示します。

アルゴリズムまたは振る舞い

  • SSI の条件式解析において、ノードを連結して構文木を形成します。
  • debug_dump_tree 関数等でツリー構造を走査する際に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_include.c 内の解析ロジック（1169 行目付近でインスタンス化）。
  • **呼び出し先**: debug_dump_tree 関数（276 行目）がデバッグ出力のために本構造体を参照します。
  • **注記**: 呼び出し元・呼び出し先の詳細なフローは、ソースコードの断片からは限定的です。

関連データ構造

  • token_t: ノードが保持するトークン情報。
  • include_ctx_t: フィルタの実行状態を保持する構造体（debug_dump_tree の引数として関連）。

失敗モードと境界条件

  • メモリ割り当て失敗時の挙動については、本構造体の定義からは不明です。
  • ツリーの深さやノード数に関する制限は、mod_include.c の解析ロジックに依存します。

保守担当者向けメモ

  • DEBUG_INCLUDE マクロが定義されている場合のみ dump_done メンバが追加されるため、バイナリ互換性に注意してください。
  • 解析ロジックの変更時には、parent, left, right のポインタ整合性が保たれていることを確認してください。
