add_env_module_vars_unset

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_env.c:134

目的

Apache 設定ファイル内の UnsetEnv ディレクティブを処理し、指定された環境変数を削除対象として記録および現在の設定テーブルから除外する。

シグネチャまたは形状

static const char *add_env_module_vars_unset(cmd_parms *cmd, void *sconf_, const char *arg)

入力と出力

  • cmd: 設定コマンドのパラメータ構造体。
  • sconf_: ディレクトリごとの設定構造体（env_dir_config_rec）。
  • arg: 削除対象となる環境変数名。
  • 戻り値: 成功時は NULL を返す。

前提条件と不変条件

  • sconf_ は env_dir_config_rec 型としてキャスト可能であること。
  • sconf->unsetenv および sconf->vars テーブルが初期化済みであること。

アルゴリズムまたは振る舞い

1. 引数 arg で指定された環境変数名をキーとして、sconf->unsetenv テーブルにエントリを追加する。 2. 同時に、sconf->vars テーブルから該当するキーを削除する。 3. これにより、後続の merge_env_dir_configs 処理において、親設定からの継承時に当該変数が除外されるよう制御する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/metadata/mod_env.c:155 において、AP_INIT_ITERATE マクロを通じて UnsetEnv ディレクティブのハンドラとして登録されている。
  • 呼び出し先: apr_table_set および apr_table_unset（APR ライブラリ関数）。

関連データ構造

  • env_dir_config_rec: ディレクトリ単位の設定を保持する構造体。vars（設定済み変数）と unsetenv（削除対象変数）の2つの apr_table_t を管理する。

失敗モードと境界条件

  • 削除対象の変数が sconf->vars に存在しない場合でも、apr_table_unset はエラーを返さず正常に終了する。
  • 戻り値は常に NULL であり、この関数内でエラーハンドリングによる中断は行われない。

保守担当者向けメモ

  • 本関数は UnsetEnv が出現した時点での設定状態を操作する。
  • UnsetEnv による削除は、同じコンテキスト内で SetEnv や PassEnv より後に記述された場合にのみ意図した通りに機能する。これはマージロジックの制約によるものである。
