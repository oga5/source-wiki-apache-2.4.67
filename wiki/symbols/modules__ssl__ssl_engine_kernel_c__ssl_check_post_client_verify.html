ssl_check_post_client_verify

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:311

目的

SSL再ネゴシエーション後のクライアント証明書の検証結果を評価し、設定されたセキュリティポリシー（SSLVerify）に適合しているかを確認する。

シグネチャまたは形状

static int ssl_check_post_client_verify(request_rec *r, SSLSrvConfigRec *sc, 
                                        SSLDirConfigRec *dc, SSLConnRec *sslconn,
                                        SSL *ssl)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • sc, dc: サーバーおよびディレクトリ単位のSSL設定。
  • sslconn: SSL接続固有のデータ構造。
  • ssl: OpenSSLのSSLセッションオブジェクト。
  • **出力**:
  • OK (0): 検証成功。
  • HTTP_FORBIDDEN: 検証失敗（証明書不正または欠落）。

前提条件と不変条件

  • sslconn->client_cert が既に存在する場合、新しい証明書で更新される。
  • dc->nVerifyClient または sc->server->auth.verify_mode が SSL_CVERIFY_REQUIRE に設定されている場合、証明書の存在と検証結果が必須となる。

アルゴリズムまたは振る舞い

1. **証明書の取得**: SSL_get_peer_certificate(ssl) を呼び出し、ピア証明書を取得する。存在する場合は sslconn->client_cert を更新し、古い証明書を解放する。 2. **検証要否の判定**: ディレクトリまたはサーバー設定で検証が有効か確認する。 3. **検証結果の確認**: do_verify が真の場合、以下のチェックを行う：

  • SSL_get_verify_result(ssl) が X509_V_OK でない場合、エラーログを出力し HTTP_FORBIDDEN を返す。
  • 証明書が取得できていない場合、エラーログを出力し HTTP_FORBIDDEN を返す。

    • 4. **正常終了**: 上記の条件に抵触しない場合、OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_kernel.c:881
  • modules/ssl/ssl_engine_kernel.c:1036
  • **呼び出し先**:
  • SSL_get_peer_certificate (OpenSSL)
  • X509_free (OpenSSL)
  • SSL_get_verify_result (OpenSSL)
  • ap_log_rerror (Apache API)

関連データ構造

  • request_rec: Apacheリクエスト情報。
  • SSLSrvConfigRec, SSLDirConfigRec: SSL設定情報。
  • SSLConnRec: SSL接続状態（client_cert を保持）。
  • SSL: OpenSSLセッションオブジェクト。

失敗モードと境界条件

  • **証明書検証失敗**: SSL_get_verify_result が X509_V_OK 以外を返すと HTTP_FORBIDDEN となる。
  • **証明書欠落**: SSL_CVERIFY_REQUIRE 設定時に証明書が提示されない場合、HTTP_FORBIDDEN となる。

保守担当者向けメモ

  • 本関数は再ネゴシエーション後の事後処理として呼び出される。
  • sslconn->client_cert のメモリ管理（X509_free）は本関数内で行われるため、呼び出し元での二重解放に注意すること。
