md_timeperiod_has_ended

定義

  • 種別: function
  • 定義箇所: modules/md/md_time.c:42

目的

指定された時刻が、期間（md_timeperiod_t）の範囲内にあるかどうかを判定します。

シグネチャまたは形状

int md_timeperiod_has_ended(const md_timeperiod_t *period, apr_time_t time)

入力と出力

  • **入力**:
  • period: 判定対象となる期間構造体へのポインタ。
  • time: 判定対象となる apr_time_t 形式の時刻。
  • **出力**:
  • int: 指定時刻が期間の開始時刻以上かつ終了時刻以下であれば非ゼロ（真）、そうでなければ 0（偽）を返します。

前提条件と不変条件

  • period は有効な md_timeperiod_t 構造体を指している必要があります。
  • period->start および period->end は、apr_time_t として比較可能な値で初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った time が period->start 以上であるかを確認します。 2. 同時に、time が period->end 以下であるかを確認します。 3. 両方の条件を満たす場合（start <= time <= end）に真を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_time.c:34 で参照されています。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • md_timeperiod_t: 期間を定義する構造体（start と end を保持）。
  • apr_time_t: APR（Apache Portable Runtime）で定義される時刻型。

失敗モードと境界条件

  • period が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • period->start が period->end より大きい場合、この関数は常に 0（偽）を返します。

保守担当者向けメモ

  • 関数名から「期間が終了したか」を判定するように見えますが、実装上の論理は「指定時刻が期間内に含まれるか」を確認するものです。
  • 戻り値の解釈において、期間の境界値（start および end）は期間に含まれるものとして扱われます。
