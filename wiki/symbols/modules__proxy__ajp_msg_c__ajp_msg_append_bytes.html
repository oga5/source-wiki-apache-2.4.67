ajp_msg_append_bytes

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_msg.c:369

目的

AJPメッセージバッファの末尾に、指定された長さのバイナリデータを追加します。

シグネチャまたは形状

apr_status_t ajp_msg_append_bytes(ajp_msg_t *msg, const apr_byte_t *value, apr_size_t valuelen)

入力と出力

  • **入力**:
  • msg: 操作対象の ajp_msg_t 構造体へのポインタ。
  • value: 追加するデータが格納されたバイト配列。
  • valuelen: 追加するデータの長さ（バイト単位）。
  • **出力**:
  • APR_SUCCESS: 成功時。
  • AJP_EOVERFLOW (または ajp_log_overflow が返す値): バッファ容量を超過した場合。

前提条件と不変条件

  • msg は有効な ajp_msg_t 構造体を指している必要があります。
  • msg->len は現在のメッセージの有効なデータ長を正しく保持している必要があります。
  • msg->max_size はバッファの物理的な最大容量を示します。

アルゴリズムまたは振る舞い

1. valuelen が 0 の場合、何もせず APR_SUCCESS を返します。 2. 現在のメッセージ長 msg->len に追加分 valuelen を加えた値が、バッファの最大容量 msg->max_size を超えるか確認します。 3. 超過する場合は ajp_log_overflow を呼び出し、その戻り値を返して終了します。 4. 容量に余裕がある場合、memcpy を使用して msg->buf の msg->len オフセット位置にデータをコピーします。 5. msg->len を valuelen 分だけインクリメントし、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていませんが、modules/proxy/ajp.h:256 にプロトタイプ宣言が存在します。
  • **呼び出し先**:
  • memcpy (標準ライブラリ)
  • ajp_log_overflow (バッファ超過時のエラーハンドリング)

関連データ構造

  • ajp_msg_t: AJPメッセージを管理する構造体。buf（データバッファ）、len（現在の長さ）、max_size（最大容量）を保持します。

失敗モードと境界条件

  • **バッファオーバーフロー**: msg->len + valuelen > msg->max_size となる場合、データは追加されずエラーが返されます。
  • **空データ**: valuelen が 0 の場合はエラーとして扱われず、成功として処理されます。

保守担当者向けメモ

  • 本関数はバイナリデータのコピーを行うため、value ポインタが指すメモリ領域が valuelen 分確保されていることを呼び出し側で保証する必要があります。
  • valuelen が 0 の場合の挙動について、ソースコード内に「エラーを示すべきではないか？」というコメントが残されています。
