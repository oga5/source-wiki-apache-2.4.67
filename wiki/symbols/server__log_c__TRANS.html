TRANS

定義

  • 種別: struct
  • 定義箇所: server/log.c:71

目的

syslog のファシリティ名や優先度名と、それに対応する整数値をマッピングするためのデータ構造です。

シグネチャまたは形状

typedef struct {
    const char *t_name;
    int t_val;
} TRANS;

入力と出力

  • 入力: t_name（文字列の名称）、t_val（対応する整数値）を保持する構造体。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • t_name は const char * であり、静的に定義された文字列リテラルを指すことが想定されています。
  • t_val は syslog のファシリティや優先度を表す整数値です。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行います。server/log.c 内では、この構造体の配列（facilities[] や priorities[]）を定義し、文字列から対応する整数値への変換テーブルとして利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • server/log.c:85: facilities[] 配列の定義に使用。
  • server/log.c:148: priorities[] 配列の定義に使用。
  • server/log.c:411: const TRANS *fac として、変換テーブルを走査する際のポインタ変数として使用。
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • facilities[]: syslog ファシリティの変換テーブル。
  • priorities[]: syslog 優先度の変換テーブル。

失敗モードと境界条件

  • 変換テーブルの末尾は、通常 t_name が NULL であるエントリで終端される必要があります（コード内の走査ロジックに依存）。

保守担当者向けメモ

  • syslog 関連の設定を拡張する際は、この構造体を用いて新しい変換テーブルを追加してください。
  • 構造体定義の変更は、server/log.c 内のすべての変換テーブル定義に影響するため注意してください。
