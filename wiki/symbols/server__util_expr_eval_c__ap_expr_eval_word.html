ap_expr_eval_word

定義

  • 種別: function
  • 定義箇所: server/util_expr_eval.c:88

目的

抽象構文木（AST）のノードを評価し、リクエストコンテキストに基づいて文字列値を算出します。

シグネチャまたは形状

static const char *ap_expr_eval_word(ap_expr_eval_ctx_t *ctx, const ap_expr_t *node);

入力と出力

  • **入力**:
  • ctx: 評価コンテキスト（ap_expr_eval_ctx_t）。リクエスト情報やメモリプールを保持。
  • node: 評価対象の AST ノード。
  • **出力**: 評価結果の文字列（const char *）。結果が空の場合やエラー時は空文字列を返します。

前提条件と不変条件

  • ctx->p（リクエストプール）からメモリが割り当てられます。
  • 評価の再帰深さは inc_rec(ctx) によって監視され、制限を超えると評価を中断します。
  • 評価終了時には必ず ctx->reclvl をデクリメントし、再帰レベルを維持します。

アルゴリズムまたは振る舞い

1. inc_rec を呼び出し、再帰制限チェックを行います。 2. node->node_op に基づき、以下のいずれかの処理を実行します。

  • op_Digit / op_String: ノード内の値をそのまま返します。
  • op_Var: ap_expr_eval_var を呼び出し、変数を解決します。
  • op_Concat: 文字列連結を行います。連結対象が複数ある場合は iovec を使用して効率的に結合します。
  • op_StringFuncCall: ap_expr_eval_string_func を呼び出し、関数実行結果を取得します。
  • op_RegexBackref: ap_expr_eval_re_backref を呼び出し、正規表現の後方参照を取得します。

    • 3. 評価結果が NULL の場合は空文字列に置換します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_expr_eval.c 内の比較演算関数群（intstrcmp を含む評価ロジック）や、式評価の再帰処理から呼び出されます。
  • **呼び出し先**:
  • inc_rec, ap_expr_eval_var, ap_expr_eval_string_func, ap_expr_eval_re_backref, apr_pstrcat, apr_pstrcatv, apr_palloc。

関連データ構造

  • ap_expr_eval_ctx_t: 評価コンテキスト。
  • ap_expr_t: AST ノード構造体。
  • struct iovec: 文字列連結時に使用されるバッファ構造体。

失敗モードと境界条件

  • **未知のノード**: default ケースに到達した場合、ctx->err にエラーメッセージが格納され、空文字列が返されます。
  • **再帰制限**: inc_rec が真を返した場合、評価は即座に中断されます。
  • **メモリ不足**: apr_palloc 等のメモリ割り当て失敗時は、APR の規約に従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、スタックオーバーフローを防ぐための inc_rec の呼び出しは必須です。
  • op_Concat の処理は、連結対象のノード構成に応じて最適化されています。変更を加える際は、iovec の構築ロジックが AST の構造と一致しているか確認してください。
