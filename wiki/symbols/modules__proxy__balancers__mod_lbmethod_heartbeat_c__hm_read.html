hm_read

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_heartbeat.c:212

目的

共有メモリ（slotmem）から読み取ったサーバーのハートビート情報を、内部のハッシュテーブル形式（hb_server_t）へ変換・同期します。

シグネチャまたは形状

apr_status_t hm_read(void* mem, void *data, apr_pool_t *pool)

入力と出力

  • **入力**:
  • mem: hm_slot_server_t 型の共有メモリ上のデータ。
  • data: ctx_servers_t 型のコンテキスト（サーバー情報のハッシュテーブルと現在時刻を含む）。
  • pool: メモリ割り当て用プール。
  • **出力**:
  • APR_SUCCESS: 処理成功。

前提条件と不変条件

  • data は ctx_servers_t 構造体へのポインタとしてキャスト可能であること。
  • mem は hm_slot_server_t 構造体として解釈可能であること。
  • サーバーの IP アドレスがハッシュテーブルに存在しない場合、pool を使用して新規作成されます。

アルゴリズムまたは振る舞い

1. mem から hm_slot_server_t を取得し、対応する IP アドレスの hb_server_t をハッシュテーブルから検索します。 2. 該当するサーバー情報がない場合、apr_pcalloc で新規作成し、ハッシュテーブルに登録します。 3. 共有メモリ上の busy、ready、id、および経過時間（seen）を hb_server_t にコピーします。 4. **ヒューリスティック処理**: サーバーが busy == 0 かつ ready != 0 の場合、過負荷防止のため ready の値を 1/4 に調整します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/balancers/mod_lbmethod_heartbeat.c:238 において、storage->doall を介して呼び出されます。
  • **呼び出し先**:
  • apr_hash_get / apr_hash_set (ハッシュテーブル操作)
  • apr_pcalloc / apr_pstrdup (メモリ管理)
  • apr_time_sec (時間計算)

関連データ構造

  • hm_slot_server_t: 共有メモリ上の生データ構造。
  • ctx_servers_t: 処理中のコンテキスト。
  • hb_server_t: 内部で管理されるサーバー状態。

失敗モードと境界条件

  • サーバー情報がハッシュテーブルに存在しない場合、新規作成が行われます。
  • seen の計算において、ctx->now と slotserver->seen の差分が負になる可能性についてはソースコード上で明示的なガードはありません。

保守担当者向けメモ

  • 本関数は slotmem のイテレータとして動作します。
  • ready 値の調整ロジックは、起動直後のサーバーが過剰に選択されることを防ぐための重要なヒューリスティックです。変更時は負荷分散の公平性に影響するため注意してください。
