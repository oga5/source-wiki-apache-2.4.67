authz_dbd_login

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbd.c:115

目的

データベース（DBD）を使用してユーザーのログインまたはログアウト処理を実行し、必要に応じてリダイレクト処理とセッション関連フックの呼び出しを行う。

シグネチャまたは形状

static int authz_dbd_login(request_rec *r, authz_dbd_cfg *cfg, const char *action)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • cfg: データベースクエリ設定を含む authz_dbd_cfg 構造体。
  • action: 実行するアクション名（ログ出力用）。
  • **出力**:
  • OK: 処理成功。
  • HTTP_INTERNAL_SERVER_ERROR: 設定欠如、DB接続失敗、またはクエリ実行失敗時。

前提条件と不変条件

  • cfg->query が設定されていること。
  • mod_dbd が有効であり、dbd_handle(r) が有効なデータベース接続を返すこと。
  • すべてのメモリ割り当ては r->pool に依存し、リクエスト終了時に自動解放される。

アルゴリズムまたは振る舞い

1. **クエリ実行**: 設定された cfg->query をプリペアドステートメントとして取得し、r->user を引数として実行。 2. **リダイレクト判定**:

  • cfg->redirect が有効な場合、Referer ヘッダーからリダイレクト先を取得。
  • cfg->redir_query が設定されている場合、データベースからリダイレクト先を取得。

    • 3. **リダイレクト設定**: newuri が特定された場合、r->status を HTTP_MOVED_TEMPORARILY に設定し、Location ヘッダーを付与。 4. **フック実行**: authz_dbd_run_client_login を呼び出し、外部モジュールへログイン成功を通知。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • dbdlogin_check_authorization (アクション: "login")
  • dbdlogout_check_authorization (アクション: "logout")
  • **呼び出し先**:
  • dbd_handle (DB接続取得)
  • apr_dbd_pvquery / apr_dbd_pvselect (クエリ実行)
  • authz_dbd_run_client_login (フック通知)

関連データ構造

  • request_rec: リクエスト情報およびメモリプール管理。
  • authz_dbd_cfg: データベースクエリおよびリダイレクト設定。
  • ap_dbd_t: データベース接続ハンドル。

失敗モードと境界条件

  • **設定不備**: クエリが未定義の場合、エラーログを出力し HTTP_INTERNAL_SERVER_ERROR を返す。
  • **DB接続エラー**: dbd_handle が失敗した場合、エラーログを出力し HTTP_INTERNAL_SERVER_ERROR を返す。
  • **クエリ実行失敗**: SQL エラー発生時、エラーメッセージをログに記録し HTTP_INTERNAL_SERVER_ERROR を返す。
  • **リダイレクトクエリ失敗**: リダイレクト先取得クエリが失敗しても、ログイン処理自体は継続される（非クリティカルなエラーとして処理）。

保守担当者向けメモ

  • client_login フックの設計意図は本モジュール固有のロジックに依存しているため、変更時は注意が必要。
  • リダイレクトクエリの反復処理において、apr_dbd_get_row のループ内で res のクリーンアップが適切に行われるよう注意すること。
