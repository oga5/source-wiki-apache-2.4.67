set_and_comp_regexp

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_usertrack.c:175

目的

Cookie 名に基づいた検索用正規表現を構築し、コンパイルして設定構造体に格納する。

シグネチャまたは形状

static void set_and_comp_regexp(cookie_dir_rec *dcfg, apr_pool_t *p, const char *cookie_name)

入力と出力

  • **入力**:
  • dcfg: 設定を保持する cookie_dir_rec 構造体へのポインタ。
  • p: メモリ割り当てに使用する apr_pool_t プール。
  • cookie_name: 追跡対象とする Cookie の名称。
  • **出力**: なし（副作用として dcfg のメンバを更新）。

前提条件と不変条件

  • cookie_name に英数字以外の文字が含まれる場合、正規表現として機能するようにエスケープ処理が行われる。
  • コンパイルされた正規表現は dcfg->regexp に格納され、ap_assert により NULL でないことが保証される。

アルゴリズムまたは振る舞い

1. cookie_name を走査し、英数字以外の文字数をカウントする。 2. 英数字以外の文字が存在する場合、それらを \ でエスケープした新しい文字列を生成する。 3. ^cookie_name=([^;,]+)|[;,][ \t]*cookie_name=([^;,]+) という形式の正規表現文字列を構築する。 4. ap_pregcomp を使用し、AP_REG_EXTENDED フラグを付与して正規表現をコンパイルする。 5. 結果を dcfg->regexp_string および dcfg->regexp に保存する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_usertrack.c:293
  • modules/metadata/mod_usertrack.c:373
  • **呼び出し先**:
  • apr_isalnum (APR)
  • apr_palloc (APR)
  • apr_pstrcat (APR)
  • ap_pregcomp (Apache Core)
  • ap_assert (Apache Core)

関連データ構造

  • cookie_dir_rec: ディレクトリごとの設定を保持する構造体。本関数により regexp および regexp_string が更新される。

失敗モードと境界条件

  • ap_pregcomp が失敗した場合、ap_assert によりプロセスが停止する。
  • メモリ不足時は apr_palloc 等の APR 関数がエラーを返す（通常は Apache のメモリ管理ポリシーに従い終了する）。

保守担当者向けメモ

  • 正規表現内のキャプチャグループ数は NUM_SUBS (3) と一致させる必要がある。
  • 本関数は cookie_name が変更されるたびに呼び出されるため、動的な設定変更に対応している。
