ap_lua_server_cfg

定義

  • 種別: struct
  • 定義箇所: modules/lua/mod_lua.h:150

目的

Apache HTTP Server の mod_lua モジュールにおいて、サーバー単位（server_rec）の設定情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
    /* value of the LuaRoot directive */
    const char *root_path;
} ap_lua_server_cfg;

入力と出力

  • **入力**: LuaRoot ディレクティブによって設定されるパス文字列。
  • **出力**: サーバー設定コンテキスト内で root_path として参照可能。

前提条件と不変条件

  • root_path は、Lua スクリプトの検索ルートとして使用されることを想定しています。
  • apr_pcalloc 等でメモリ確保された後、適切に初期化される必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、Apache の設定フェーズにおいてサーバーごとの設定としてインスタンス化されます。
  • mod_lua.c 内の各関数において、ap_get_module_config(r->server->module_config, ...) を介して取得され、Lua スクリプトの実行環境構築やパス解決の基盤として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/mod_lua.c 内の複数の箇所で、サーバー設定の取得（ap_get_module_config）やメモリ確保（apr_pcalloc）が行われています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • server_rec: Apache のサーバー設定コンテキスト。
  • ap_lua_dir_cfg: ディレクトリ単位の設定を保持する構造体（本モジュールの他の主要な設定構造体）。

失敗モードと境界条件

  • root_path が設定されていない場合、NULL ポインタとなる可能性があります。
  • メモリ割り当て失敗時は apr_pcalloc が例外を発生させるか、呼び出し元で適切に処理される必要があります。

保守担当者向けメモ

  • 本構造体は現在 root_path のみを保持していますが、将来的にサーバー単位のグローバルな Lua 設定項目が増える場合はここに追加されます。
  • mod_lua.c の 1701 行目や 2005 行目などでインスタンス化および参照が行われているため、変更時はこれらの箇所での整合性に注意してください。
