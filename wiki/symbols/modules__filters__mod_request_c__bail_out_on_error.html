bail_out_on_error

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:40

目的

リクエスト処理中にエラーが発生した際、バケットブリゲードを初期化し、エラーバケットとストリーム終了（EOS）バケットを挿入して、クライアントへエラーレスポンスを送信する。

シグネチャまたは形状

static apr_status_t bail_out_on_error(apr_bucket_brigade *bb,
                                      ap_filter_t *f,
                                      int http_error)

入力と出力

  • **入力**:
  • bb: 操作対象のバケットブリゲード。
  • f: 現在のフィルタコンテキスト。
  • http_error: 送信する HTTP ステータスコード（例: HTTP_REQUEST_ENTITY_TOO_LARGE）。
  • **出力**:
  • apr_status_t: ap_pass_brigade の実行結果を返す。

前提条件と不変条件

  • f および f->r が有効であること。
  • bb は apr_brigade_cleanup によって既存の内容が破棄される。

アルゴリズムまたは振る舞い

1. apr_brigade_cleanup(bb) を呼び出し、ブリゲード内の既存バケットをすべて削除する。 2. ap_bucket_error_create を使用して、指定された http_error を含むエラーバケットを作成する。 3. APR_BRIGADE_INSERT_TAIL を使用して、エラーバケットをブリゲードの末尾に追加する。 4. apr_bucket_eos_create を使用して EOS バケットを作成し、ブリゲードの末尾に追加する。 5. ap_pass_brigade を呼び出し、構築したブリゲードを出力フィルタチェーンへ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_request.c: keep_body_filter 内の各所（リクエストボディサイズ制限超過時）。
  • modules/http/http_filters.c: ap_http_filter 等の HTTP フィルタ処理内。
  • **呼び出し先**:
  • apr_brigade_cleanup
  • ap_bucket_error_create
  • apr_bucket_eos_create
  • ap_pass_brigade

関連データ構造

  • apr_bucket_brigade: バケットの管理用コンテナ。
  • ap_filter_t: フィルタチェーンのノード情報。
  • request_rec: リクエスト情報（f->r 経由で参照）。

失敗モードと境界条件

  • ap_pass_brigade がエラーを返した場合、そのステータスがそのまま呼び出し元に伝播する。
  • f->r->pool や f->c->bucket_alloc が無効な場合、メモリ確保時に異常終了する可能性がある。

保守担当者向けメモ

  • 本関数はエラー発生時の定型的なレスポンス生成を目的としている。
  • ap_bucket_error_create で生成されるエラーバケットは、後続のフィルタチェーンで適切に処理される必要がある。
