ssl_log_rxerror

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_log.c:236

目的

リクエストコンテキスト (request_rec) において、X.509 証明書に関連するエラー情報を Apache のログシステムへ出力する。

シグネチャまたは形状

void ssl_log_rxerror(const char *file, int line, int level, apr_status_t rv,
                     request_rec *r, X509 *cert, const char *fmt, ...)

入力と出力

  • **入力**:
  • file, line: ログ出力元のソースファイル名と行番号。
  • level: Apache ログレベル。
  • rv: APR ステータスコード。
  • r: リクエストレコード (request_rec)。
  • cert: エラーに関連する X.509 証明書構造体。
  • fmt, ...: ログメッセージのフォーマット文字列と可変長引数。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • r が有効な request_rec であること。
  • APLOG_R_IS_LEVEL(r, level) が真である場合にのみログ出力が実行される。

アルゴリズムまたは振る舞い

1. APLOG_R_IS_LEVEL マクロにより、指定されたログレベルが現在の設定で有効か確認する。 2. 有効な場合、va_list を初期化し、ssl_log_cert_error を呼び出す。 3. ssl_log_cert_error に r および r->pool を渡し、証明書情報の文字列化とログ出力の統合処理を委譲する。 4. 処理終了後、va_end を呼び出して可変長引数リストをクリーンアップする。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は本抜粋からは特定できない。modules/ssl/ssl_private.h にて宣言されている。
  • **呼び出し先**: ssl_log_cert_error（内部関数）。

関連データ構造

  • request_rec: Apache のリクエストコンテキスト。
  • X509: OpenSSL の証明書構造体。
  • apr_pool_t: メモリ管理用プール（r->pool を使用）。

失敗モードと境界条件

  • r が NULL の場合、APLOG_R_IS_LEVEL の挙動に依存する（通常はセグメンテーション違反の可能性がある）。
  • ログレベルが設定より低い場合、関数は何も行わずに即座に復帰する。

保守担当者向けメモ

  • 本関数は ssl_log_xerror や ssl_log_cxerror と同様のラッパーであり、リクエスト単位のロギングに特化している。
  • 証明書情報の詳細出力は ssl_log_cert_error に依存しているため、出力形式の変更が必要な場合は同関数を確認すること。
