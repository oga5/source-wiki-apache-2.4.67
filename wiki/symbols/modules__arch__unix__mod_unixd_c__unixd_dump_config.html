unixd_dump_config

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_unixd.c:382

目的

Apache HTTP Server の設定ダンプ機能（DUMP_RUN_CFG）が有効な場合に、Unix 関連の実行設定（ユーザー、グループ、chroot ディレクトリ）を標準出力へ書き出します。

シグネチャまたは形状

static void unixd_dump_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • s: 現在のサーバー設定を保持する server_rec（本関数内では直接参照せず、グローバルな ap_unixd_config を使用）。
  • **出力**: なし（標準出力への書き出しのみ）。

前提条件と不変条件

  • ap_unixd_config 構造体が初期化されている必要があります。
  • 実行時に DUMP_RUN_CFG 定義が存在しない場合、本関数は何も出力せずに終了します。

アルゴリズムまたは振る舞い

1. ap_exists_config_define("DUMP_RUN_CFG") を確認し、定義がなければ即座にリターンします。 2. 現在のプロセスが root 権限（geteuid() != 0）で実行されていない場合、出力時に not_used という文字列を付与する準備をします。 3. apr_file_open_stdout を介して標準出力を取得します。 4. ap_unixd_config から以下の情報を取得し、フォーマットして出力します。

  • User: ユーザー名と ID。
  • Group: グループ名と ID。
  • ChrootDir: 設定されている場合のみ、そのパス。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/unix/mod_unixd.c:406 にて ap_hook_test_config に登録されています。
  • **呼び出し先**:
  • ap_exists_config_define
  • geteuid
  • apr_file_open_stdout
  • apr_file_printf

関連データ構造

  • ap_unixd_config: Unix 関連の設定値を保持するグローバル構造体（unixd.h 定義）。

失敗モードと境界条件

  • geteuid() が root 以外を返す場合、出力される各設定行の末尾に not_used が付加されます。
  • ap_unixd_config.chroot_dir が NULL の場合、ChrootDir 行は出力されません。

保守担当者向けメモ

  • 本関数は設定の検証やデバッグ目的でのみ使用されます。
  • ap_unixd_config のフィールド（user_name, group_name 等）は、設定読み込みフェーズで適切に設定されていることが前提です。
