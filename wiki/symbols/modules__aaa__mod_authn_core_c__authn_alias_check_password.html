authn_alias_check_password

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_core.c:108

目的

認証プロバイダーのエイリアス（<AuthnProviderAlias>）経由で認証を実行する。リクエストのディレクトリ設定をエイリアス定義時の設定で一時的に上書きし、本来の認証プロバイダーを呼び出す。

シグネチャまたは形状

static authn_status authn_alias_check_password(request_rec *r, const char *user, const char *password)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。r->notes からプロバイダー名を取得し、r->per_dir_config を一時的に変更する。
  • user: 認証対象のユーザー名。
  • password: 認証に使用するパスワード。
  • **出力**:
  • authn_status: 認証結果（成功、失敗、ユーザー不在など）。

前提条件と不変条件

  • r->notes に AUTHN_PROVIDER_NAME_NOTE キーでプロバイダー名が格納されている必要がある。
  • エイリアス設定はサーバー起動時に authn_alias_srv_conf 内のハッシュテーブルに登録されている必要がある。

アルゴリズムまたは振る舞い

1. r->notes からプロバイダー名を取得する。 2. サーバー設定から該当する provider_alias_rec を検索する。 3. エイリアスが見つかった場合:

  • 現在の r->per_dir_config を保存する。
  • ap_merge_per_dir_configs を使用し、リクエストの設定をエイリアス内の設定（sec_auth）でマージして差し替える。
  • 本来のプロバイダーの check_password 関数を呼び出し、結果を保持する。
  • r->per_dir_config を元の設定に戻す。

    • 4. 認証結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_authn_core.c 内の認証プロバイダー登録箇所（&authn_alias_check_password として参照）。
  • **呼び出し先**:
  • ap_get_module_config
  • apr_hash_get
  • ap_merge_per_dir_configs
  • prvdraliasrec->provider->check_password（動的ディスパッチ）

関連データ構造

  • authn_alias_srv_conf: エイリアスを管理するサーバー設定。
  • provider_alias_rec: エイリアスされたプロバイダーの定義と、そのエイリアスに紐づくディレクトリ設定を保持。
  • request_rec: リクエストごとの状態と設定を保持。

失敗モードと境界条件

  • provider_name が r->notes に存在しない場合、認証は実行されず AUTH_USER_NOT_FOUND が返される。
  • エイリアス名がハッシュテーブルに見つからない場合、認証は実行されず AUTH_USER_NOT_FOUND が返される。

保守担当者向けメモ

  • 本関数は r->per_dir_config を一時的に書き換えるため、呼び出し先でエラーが発生しても確実に復元されるよう注意が必要である。
  • r->pool を使用したメモリ管理が行われており、リクエスト終了時に一時的な設定マージ結果は解放される。
