modssl_read_ocsp_certificates

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ocsp.c:354

目的

指定されたファイルから PEM 形式の証明書を読み込み、STACK_OF(X509) 形式の証明書リストとして構築する。

シグネチャまたは形状

STACK_OF(X509) *modssl_read_ocsp_certificates(const char *file)

入力と出力

  • **入力**: file (読み込み対象の PEM ファイルパス)
  • **出力**: 読み込みに成功した場合は STACK_OF(X509)* を返し、失敗またはファイルが空の場合は NULL を返す。

前提条件と不変条件

  • 指定されたファイルは PEM 形式で記述されている必要がある。
  • 戻り値が NULL でない場合、呼び出し元は sk_X509_pop_free(certs, X509_free) を使用してメモリを解放する責任がある。

アルゴリズムまたは振る舞い

1. BIO_new(BIO_s_file()) を用いてファイル BIO を作成し、指定されたパスを開く。 2. PEM_read_bio_X509 をループ内で呼び出し、ファイル内のすべての証明書を順次読み込む。 3. 読み込んだ証明書を sk_X509_push でスタックに追加する。 4. 読み込み終了後、ERR_peek_error を確認する。エラーが「PEM ファイルの終端（EOF）」によるもの以外であれば、エラーとみなしてスタックを解放し NULL を返す。 5. 正常終了時は、構築したスタックを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_util_ocsp.c:414 (推定)
  • **呼び出し先**: OpenSSL API (BIO_new, BIO_read_filename, PEM_read_bio_X509, sk_X509_new_null, sk_X509_push, ERR_peek_error 等)

関連データ構造

  • X509: OpenSSL の証明書構造体。
  • STACK_OF(X509): OpenSSL の証明書スタック。

失敗モードと境界条件

  • ファイルが開けない場合、またはファイルが存在しない場合は NULL を返す。
  • 証明書の読み込み中にメモリ確保に失敗した場合、スタックを解放し NULL を返す。
  • PEM 形式として不正なデータが含まれる場合、エラーを検出し NULL を返す。
  • ファイルが空の場合、NULL を返す。

保守担当者向けメモ

  • 本関数は ssl_init_ocsp_certificates 等から呼び出されることを想定している。
  • エラーハンドリングにおいて、PEM_R_NO_START_LINE（ファイル終端）以外のエラーは致命的として処理される。
