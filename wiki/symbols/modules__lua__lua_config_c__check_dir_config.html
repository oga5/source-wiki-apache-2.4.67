check_dir_config

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:23

目的

Lua スタック上の指定されたインデックスから Apache2.DirConfig 型のユーザーデータを取得し、対応する C 言語の構造体ポインタへ変換します。

シグネチャまたは形状

static ap_lua_dir_cfg *check_dir_config(lua_State *L, int index)

入力と出力

  • **入力**:
  • L: Lua 仮想マシンの状態 (lua_State)。
  • index: ユーザーデータが配置されている Lua スタック上のインデックス。
  • **出力**:
  • 成功時: ap_lua_dir_cfg 構造体へのポインタを返します。
  • 失敗時: luaL_checkudata により、型が一致しない場合は Lua 側でエラーが発生し、制御が戻りません。

前提条件と不変条件

  • スタックの指定された index に Apache2.DirConfig メタテーブルを持つユーザーデータが存在している必要があります。
  • lua_unboxpointer を使用してポインタを取り出すため、当該ユーザーデータは内部的にポインタを保持している必要があります。

アルゴリズムまたは振る舞い

1. luaL_checkudata を呼び出し、スタック上の値が Apache2.DirConfig 型であることを検証します。 2. lua_unboxpointer を使用して、ユーザーデータから ap_lua_dir_cfg 構造体へのポインタを抽出します。 3. 抽出したポインタを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_config.c:84
  • modules/lua/lua_config.c:139
  • modules/lua/lua_config.c:146
  • **呼び出し先**:
  • luaL_checkudata (Lua API)
  • lua_unboxpointer (Lua API)

関連データ構造

  • ap_lua_dir_cfg: Apache のディレクトリ設定を保持する構造体。

失敗モードと境界条件

  • 指定されたインデックスにデータが存在しない、または型が Apache2.DirConfig ではない場合、luaL_checkudata が Lua のエラーハンドリング機構を通じて例外をスローします。

保守担当者向けメモ

  • 本関数は mod_lua の設定バインディング層における型安全性を担保するヘルパーです。
  • lua_unboxpointer を使用しているため、Lua 側で作成されたユーザーデータが正しく初期化されていることに依存しています。
