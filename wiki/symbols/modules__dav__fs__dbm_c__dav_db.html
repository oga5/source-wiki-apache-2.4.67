dav_db

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/dbm.c:52

目的

mod_dav_fs におけるファイルシステムリポジトリのプロパティ（Dead Properties）を管理するための、DBM データベース接続状態および関連メタデータを保持する構造体です。

シグネチャまたは形状

struct dav_db {
    apr_pool_t *pool;
    apr_dbm_t *file;
    int version;
    dav_buffer ns_table;
    short ns_count;
    int ns_table_dirty;
    apr_hash_t *uri_index;
    dav_buffer wb_key;
    apr_datum_t iter;
};

入力と出力

  • **入力**: 構造体内の各フィールドは、データベースのオープン時に初期化され、プロパティの読み書きや反復処理の過程で更新されます。
  • **出力**: dav_db を介して DBM ファイルへの永続的な書き込み（dav_dbm_store 等）や、メモリ上の名前空間テーブルの更新が行われます。

前提条件と不変条件

  • pool: データベース接続のライフサイクルを管理する apr_pool_t に紐付いています。
  • ns_table: XML 名前空間 URI を保持し、DBM キーの最適化に使用されます。
  • uri_index: URI からテーブルインデックスへのマッピングを保持するハッシュテーブルです。
  • wb_key: dav_build_key で使用される静的バッファとして機能します。

アルゴリズムまたは振る舞い

  • **データベース管理**: dav_propdb_open により初期化され、METADATA キーからバージョン情報と名前空間テーブルを構築します。
  • **キー生成**: dav_build_key が uri_index を参照し、名前空間 ID とプロパティ名から #:name 形式の DBM キーを生成します。
  • **反復処理**: iter フィールドを使用して DBM 内のキーを順次走査し、メタデータキーをスキップしながらプロパティを抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**: dav_fs_dbm_error, dav_dbm_close, dav_dbm_fetch, dav_dbm_store, dav_dbm_delete, dav_dbm_exists, dav_dbm_firstkey, dav_dbm_nextkey, dav_dbm_freedatum, dav_build_key 等の dbm.c 内の関数。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • apr_dbm_t: 基盤となる DBM ファイルハンドル。
  • dav_buffer: 名前空間テーブルおよびキー生成用バッファ。
  • apr_hash_t: URI インデックス用ハッシュマップ。
  • apr_datum_t: DBM のキーおよび値のデータ型。

失敗モードと境界条件

  • **メモリ管理**: dav_db は apr_pool_t に依存しており、プール解放時に適切にクローズされない場合、リソースリークが発生します。
  • **並行性**: DBM の性質上、読み取りと書き込みが重なる際の再試行ロジックが不足しており、整合性が損なわれる可能性があります。
  • **バッファ制約**: dav_build_key が使用する wb_key は静的バッファであるため、呼び出し直後に結果をコピーする必要があります。

保守担当者向けメモ

  • METADATA のマイナーバージョン更新ロジックは現在不完全です。
  • 高負荷環境下での並行書き込みに対する堅牢性を向上させる必要があります。
  • ns_table_dirty フラグは、名前空間テーブルが変更された際にデータベースへ書き戻す必要があることを示します。
