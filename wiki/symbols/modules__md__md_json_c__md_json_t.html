md_json_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_json.c:56

目的

Jansson ライブラリの json_t オブジェクトをラップし、Apache Portable Runtime (APR) プールによるメモリ管理と統合するためのコンテナ構造体です。

シグネチャまたは形状

struct md_json_t {
    apr_pool_t *p;
    json_t *j;
};

入力と出力

  • **入力**: 構造体生成時に apr_pool_t へのポインタと json_t オブジェクトを保持します。
  • **出力**: 保持する json_t オブジェクトへのアクセスを提供し、APR プールの破棄に伴い自動的にメモリが解放されます。

前提条件と不変条件

  • p は有効な apr_pool_t を指している必要があります。
  • j は Jansson ライブラリによって管理される有効な JSON オブジェクトである必要があります。
  • md_json_t の生存期間は、保持している apr_pool_t の生存期間に依存します。

アルゴリズムまたは振る舞い

  • md_json_t は json_t の参照カウントを管理します。
  • md_json_create 等の生成関数を通じて、APR プールにクリーンアップ関数（json_pool_cleanup）が登録されます。これにより、プールが破棄される際に json_decref が自動的に呼び出され、メモリリークを防止します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md.h で定義される md_to_json や md_from_json、md_to_public_json 等の関数がこの構造体を利用します。また、modules/md/md_acme.c 内の各 ACME プロトコル処理関数（acmev2_req_init、md_acme_req_body_init、md_acme_get_json 等）で頻繁に使用されます。
  • **呼び出し先**: 内部で Jansson ライブラリの関数（json_decref 等）を呼び出します。

関連データ構造

  • json_t: Jansson ライブラリのコア JSON オブジェクト。
  • apr_pool_t: APR のメモリプール。
  • md_t: md_to_json 等を通じて本構造体と相互変換される Managed Domain のメイン構造体。

失敗モードと境界条件

  • json_t が NULL の場合、適切にハンドリングされないとセグメンテーション違反を引き起こす可能性があります。
  • APR プールを跨いで md_json_t を受け渡す場合、メモリの生存期間管理に注意が必要です。

保守担当者向けメモ

  • 本構造体は modules/md/md_json.h にも関連定義が存在します。
  • JSON パッチや複雑なマージ操作は実装されておらず、主に設定ファイルの読み書きや ACME レスポンスの抽出に特化しています。
  • json_t の所有権は APR プールに委譲されているため、手動での json_decref は原則として不要です。
