proxy_wstunnel_post_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_wstunnel.c:477

目的

mod_proxy_http モジュールがロードされているかどうかを判定し、その結果をグローバル変数 can_fallback_to_proxy_http に格納することで、WebSocket トンネルからのフォールバック可否を初期化します。

シグネチャまたは形状

static int proxy_wstunnel_post_config(apr_pool_t *pconf, apr_pool_t *plog,
                                      apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • pconf, plog, ptemp: APR プール（設定、ログ、一時作業用）。
  • s: 現在のサーバー設定構造体。
  • **出力**:
  • 戻り値: OK (常に成功を返します)。

前提条件と不変条件

  • 本関数は Apache の post_config フックとして実行されることを前提としています。
  • ap_find_linked_module 関数が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. ap_find_linked_module("mod_proxy_http.c") を呼び出し、mod_proxy_http がサーバーにリンクされているかを確認します。 2. 判定結果（真または偽）を静的変数 can_fallback_to_proxy_http に代入します。 3. OK を返して処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の post_config フック機構（modules/proxy/mod_proxy_wstunnel.c:499 にて ap_hook_post_config を通じて登録）。
  • **呼び出し先**: ap_find_linked_module（Apache Core API）。

関連データ構造

  • server_rec: サーバー設定を保持する構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • mod_proxy_http.c がロードされていない場合、can_fallback_to_proxy_http は 0（偽）に設定されます。この関数自体がエラーを返すことはありません。

保守担当者向けメモ

  • 本関数はモジュールの初期化フェーズで一度だけ実行されます。
  • can_fallback_to_proxy_http の値は、実行時のフォールバック判断に使用されるため、この初期化ロジックを変更する場合は依存関係の整合性に注意してください。
