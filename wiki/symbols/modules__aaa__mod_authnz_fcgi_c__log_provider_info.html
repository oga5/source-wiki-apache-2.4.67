log_provider_info

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:125

目的

FastCGI プロバイダーの設定情報をデバッグログとして出力します。

シグネチャまたは形状

static void log_provider_info(const fcgi_provider_conf *conf, request_rec *r)

入力と出力

  • **入力**:
  • conf: 出力対象の FastCGI プロバイダー設定構造体へのポインタ。
  • r: 現在のリクエスト情報を保持する request_rec 構造体へのポインタ。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • conf および r は有効なポインタである必要があります。
  • Apache のログレベルが APLOG_DEBUG に設定されている場合にのみ、ログが記録されます。

アルゴリズムまたは振る舞い

1. ap_log_rerror を使用して、以下の情報をデバッグログに出力します。

  • プロバイダー名 (conf->name)
  • バックエンド識別子 (conf->backend)
  • ホスト名 (conf->host)
  • ポート番号 (conf->port)
  • 最初の接続先アドレス (conf->backend_addrs)
  • 認証 (is_authn) および認可 (is_authz) の有効状態（'N'/'Z' または '_' で表示）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:857
  • modules/aaa/mod_authnz_fcgi.c:961
  • modules/aaa/mod_authnz_fcgi.c:1022
  • **呼び出し先**:
  • ap_log_rerror (Apache HTTP Server API)

関連データ構造

  • fcgi_provider_conf: プロバイダーの接続設定および役割を保持する構造体。
  • request_rec: Apache のリクエスト処理状態を保持する構造体。

失敗モードと境界条件

  • conf が指す構造体の各フィールドが未初期化の場合、ログ出力時に不正な値やメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数はデバッグ専用です。ログ出力のフォーマットを変更する場合は、APLOGNO(02496) の識別子を維持してください。
