find_secure_listener

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:219

目的

グローバルなセキュアリスナーリスト（ap_seclisteners）から、指定されたアドレスと一致するリスナーを検索し、そのファイル記述子（fd）を取得します。

シグネチャまたは形状

static int find_secure_listener(seclisten_rec *lr)

入力と出力

  • **入力**: lr (seclisten_rec *) - 検索対象のアドレス情報を持つセキュアリスナー構造体へのポインタ。
  • **出力**: int - 一致したリスナーのファイル記述子（fd）。一致するものが見つからない場合は -1 を返します。

前提条件と不変条件

  • ap_seclisteners が初期化されている必要があります。
  • lr->local_addr に有効なアドレス情報が格納されている必要があります。

アルゴリズムまたは振る舞い

1. グローバル変数 ap_seclisteners を先頭から順に走査します。 2. 各要素の local_addr と引数 lr の local_addr を memcmp で比較します。 3. 一致した場合、該当するリスナーの used フラグを 1 に設定し、そのリスナーのファイル記述子 sl->fd を返します。 4. リストの末尾まで一致するものが見つからなかった場合、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_nw_ssl.c:782 にて、リスナーの再構成処理中に呼び出されています。
  • **呼び出し先**: 本関数内での関数呼び出しはありません。

関連データ構造

  • seclisten_rec: セキュアリスナーの定義を保持するリンクリスト構造体。local_addr（アドレス情報）、used（使用フラグ）、fd（ファイル記述子）、next（次要素へのポインタ）を含みます。

失敗モードと境界条件

  • **一致なし**: 指定されたアドレスに対応するリスナーが存在しない場合、-1 を返します。
  • **リストの空**: ap_seclisteners が NULL の場合、ループは実行されず即座に -1 を返します。

保守担当者向けメモ

  • memcmp を使用してアドレス構造体を比較しているため、local_addr のメモリレイアウトが完全に一致していることが前提となります。
  • used フラグの更新は副作用として発生するため、呼び出し側でこのフラグの状態に依存する処理がある場合は注意してください。
