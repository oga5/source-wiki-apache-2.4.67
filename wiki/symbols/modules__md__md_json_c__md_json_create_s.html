md_json_create_s

定義

  • 種別: function
  • 定義箇所: modules/md/md_json.c:97

目的

指定された文字列から JSON 文字列オブジェクトを生成し、APR プールに関連付けられた md_json_t コンテナに格納して返します。

シグネチャまたは形状

md_json_t *md_json_create_s(apr_pool_t *pool, const char *s)

入力と出力

  • **入力**:
  • pool: メモリ管理に使用する apr_pool_t へのポインタ。
  • s: JSON 文字列として格納する C 文字列。
  • **出力**:
  • 生成された md_json_t 構造体へのポインタ。

前提条件と不変条件

  • pool は有効な APR プールである必要があります。
  • s は有効な文字列ポインタである必要があります。
  • 内部で json_string(s) を呼び出すため、Jansson ライブラリが初期化されている必要があります。

アルゴリズムまたは振る舞い

1. json_string(s) を呼び出し、入力文字列から Jansson の JSON 文字列ノードを作成します。 2. 作成したノードを引数として json_create(pool, ...) を呼び出します。 3. json_create は、APR プールのクリーンアップハンドラに json_decref を登録し、メモリ管理を自動化します。 4. 生成された md_json_t インスタンスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_json.h:70 に宣言が存在します。具体的な呼び出し箇所は提供されたソース内にはありません。
  • **呼び出し先**:
  • json_string (Jansson ライブラリ)
  • json_create (modules/md/md_json.c 内の内部関数)

関連データ構造

  • md_json_t: json_t をラップし、APR プールとの紐付けを管理する構造体。
  • apr_pool_t: メモリのライフサイクルを管理する Apache Portable Runtime のプール。

失敗モードと境界条件

  • pool が NULL の場合や、メモリ不足により json_string または json_create が失敗した場合の挙動は、依存する Jansson および json_create の実装に依存します。

保守担当者向けメモ

  • 本関数は文字列専用のラッパーです。他の JSON 型（数値、オブジェクト等）を作成する場合は、別の md_json_create 系関数を使用してください。
  • md_json_t のメモリ所有権は pool に帰属するため、呼び出し元で明示的な解放は不要です。
