find_directory

定義

  • 種別: function
  • 定義箇所: support/htcacheclean.c:1144

目的

キャッシュディレクトリを再帰的に探索し、指定されたパスに対応するキャッシュファイル（ヘッダーおよびデータファイル）を特定して削除する。

シグネチャまたは形状

apr_status_t find_directory(apr_pool_t *pool, const char *base, const char *rest)

入力と出力

  • **入力**:
  • pool: メモリ割り当て用プール。
  • base: 探索を開始するディレクトリパス。
  • rest: 探索対象の残りのパスセグメント。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • dryrun フラグが偽の場合、対象ファイルおよびディレクトリの削除が実行される。
  • deldirs フラグが真の場合、空になったディレクトリの削除が試行される。
  • apr_pool_t は呼び出し元で適切に管理されている必要がある。

アルゴリズムまたは振る舞い

1. 指定された base ディレクトリを開き、内容を走査する。 2. ディレクトリ内の各エントリに対し、rest と一致するサブディレクトリがあれば再帰的に find_directory を呼び出す。 3. キャッシュのヘッダーファイルまたはデータファイルと一致するエントリを発見した場合、files フラグを立てる。 4. 探索終了後、files が真であれば、dryrun が無効な場合に限り、ヘッダーファイル、データファイル、および関連するディレクトリ（vdir）を削除する。 5. deldirs が有効な場合、探索したディレクトリ自体が空であれば削除を試みる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/htcacheclean.c:1172 (再帰呼び出し)
  • support/htcacheclean.c:1271 (外部からの呼び出し)
  • **呼び出し先**:
  • apr_dir_open, apr_dir_read, apr_dir_close (APR ディレクトリ操作)
  • apr_file_remove (ファイル削除)
  • remove_directory (ディレクトリ削除)
  • apr_dir_remove (ディレクトリ削除)
  • find_directory (自身を再帰呼び出し)

関連データ構造

  • apr_dir_t: ディレクトリ操作用ハンドル。
  • apr_finfo_t: ディレクトリ内のファイル情報。

失敗モードと境界条件

  • ディレクトリのオープン失敗時はエラーを返す。
  • ファイル削除失敗時はエラーを記録するが、ENOENT（ファイルが存在しない）の場合は無視して処理を継続する。
  • 再帰探索中にエラーが発生した場合、そのステータスが伝播される。

保守担当者向けメモ

  • 本関数は htcacheclean におけるキャッシュ削除の核心的な再帰ロジックである。
  • dryrun モード時は削除処理がスキップされるため、テストやシミュレーション時にはこのフラグの状態に注意すること。
  • vdir（ディレクトリ形式のキャッシュ）の削除には remove_directory が使用されており、ファイル削除とは異なるパスを通る点に留意。
