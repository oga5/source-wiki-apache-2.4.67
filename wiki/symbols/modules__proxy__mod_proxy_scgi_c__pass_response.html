pass_response

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:363

目的

バックエンド（SCGIサーバー）からのレスポンスを読み込み、ヘッダー解析および後続のフィルタリング処理（X-Sendfile や内部リダイレクトの判定）を行う。

シグネチャまたは形状

static int pass_response(request_rec *r, proxy_conn_rec *conn)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • conn: バックエンドへの接続情報。
  • **出力**:
  • OK: 処理成功。
  • AP_FILTER_ERROR: フィルタリング失敗。
  • その他: ap_scan_script_header_err_brigade_ex が返すエラーコード。

前提条件と不変条件

  • r->pool はリクエストの生存期間中有効であること。
  • conn->sock は有効なバックエンドソケットであること。
  • SCGI プロトコル特有のボディフレーミングが使用されるため、Content-Length や Transfer-Encoding ヘッダーは、コアによる自動決定を優先するために削除される。

アルゴリズムまたは振る舞い

1. **バケット作成**: SOCKET_EX バケットを作成し、バックエンドソケットと統計カウンタを紐付けてブリゲードに挿入する。 2. **ヘッダー解析**: ap_scan_script_header_err_brigade_ex を呼び出し、バックエンドからのレスポンスヘッダーを解析する。 3. **ヘッダー調整**: Content-Length および Transfer-Encoding をヘッダーから削除する。 4. **X-Sendfile 処理**: 設定されている場合、指定されたヘッダーを検索し、存在すれば scgi_request_config を設定してサブリクエストへ移行する。 5. **内部リダイレクト処理**: 設定されている場合、Location ヘッダー等を検索し、内部リダイレクトが必要であれば同様に scgi_request_config を設定する。 6. **転送**: 上記の特殊処理に該当しない場合、ap_pass_brigade を介してレスポンスボディをクライアントへ転送する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_scgi.c:582 (SCGIハンドラのメインフロー内)。
  • **呼び出し先**:
  • apr_palloc, apr_brigade_create, apr_bucket_eos_create, apr_table_get, apr_table_unset, apr_brigade_destroy (APR API)
  • bucket_socket_ex_create (カスタムバケット生成)
  • ap_scan_script_header_err_brigade_ex, ap_log_rerror, ap_get_module_config, ap_set_module_config, ap_pass_brigade (Apache HTTPd API)

関連データ構造

  • proxy_conn_rec: バックエンド接続状態。
  • scgi_config: ディレクトリ単位の設定（ProxySCGISendfile 等）。
  • scgi_request_config: リクエスト処理中の内部リダイレクトやファイル送信指示。
  • socket_ex_data: ソケット読み込みと統計更新のためのデータ構造。

失敗モードと境界条件

  • ヘッダー解析失敗時: ap_scan_script_header_err_brigade_ex がエラーを返し、ログ出力後にブリゲードを破棄して終了する。
  • フィルタリング失敗時: ap_pass_brigade がエラーを返した場合、AP_FILTER_ERROR を返す。

保守担当者向けメモ

  • 本関数は scgi_handler から呼び出される。
  • X-Sendfile や内部リダイレクトの判定ロジックは、r->err_headers_out と r->headers_out の両方を走査する。
  • SOCKET_EX バケットを使用することで、ソケット読み込み時の統計カウンタ更新を透過的に行っている。
