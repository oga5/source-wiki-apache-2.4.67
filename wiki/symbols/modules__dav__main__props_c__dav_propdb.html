dav_propdb

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/props.c:172

目的

WebDAV プロパティ操作におけるコンテキストを保持し、ライブプロパティおよびデッドプロパティの管理、トランザクション処理、名前空間の解決を統合的に行うための構造体です。

シグネチャまたは形状

struct dav_propdb {
    apr_pool_t *p;
    request_rec *r;
    const dav_resource *resource;
    int deferred;
    dav_db *db;
    apr_array_header_t *ns_xlate;
    dav_namespace_map *mapping;
    dav_lockdb *lockdb;
    dav_buffer wb_lock;
    int flags;
    request_rec *subreq;
    const dav_hooks_db *db_hooks;
};

入力と出力

  • **入力**: dav_popen_propdb 等の初期化関数を通じて、リクエストレコード (request_rec) や対象リソース (dav_resource) が設定されます。
  • **出力**: プロパティ操作の結果として、データベース (dav_db) へのアクセスや、XML 出力バッファへのプロパティ挿入が行われます。

前提条件と不変条件

  • dav_propdb はリクエストプール (r->pool) または指定されたプールを使用してメモリ管理を行います。
  • deferred フラグが設定されている場合、データベースのオープンは実際にプロパティ操作が必要になるまで遅延されます。
  • dav_close_propdb を呼び出すことで、リソースの解放および内部で生成されたサブリクエスト (subreq) の破棄を確実に行う必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: dav_popen_propdb によりコンテキストが作成されます。 2. **解決**: dav_find_liveprop を通じてライブプロパティプロバイダーを特定し、見つからない場合はデッドプロパティとして処理します。 3. **トランザクション**: PROPPATCH 操作時、validate -> exec -> commit/rollback の順序で処理され、失敗時には dav_rollback_item を用いて状態を復旧します。 4. **XML 構築**: dav_get_props 等により、名前空間 (ns_xlate, mapping) を管理しながら XML 形式のプロパティリストを生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_dav.c 内の各メソッドハンドラ（PROPFIND, PROPPATCH 等）から参照されます。
  • **呼び出し先**: props.c 内の dav_find_liveprop や dav_find_liveprop_provider 等の関数が、この構造体を引数として受け取り操作を行います。

関連データ構造

  • dav_resource: 操作対象のリソース。
  • dav_db: 基盤となるプロパティデータベース。
  • dav_hooks_db: プロパティ操作のためのフック関数群。
  • dav_lockdb: ロック管理用データベース。

失敗モードと境界条件

  • DAV_DISABLE_WRITABLE_PROPS が有効な場合、一部のコアプロパティへの書き込みが制限されます。
  • PROPPATCH のエラーハンドリングにおいて、複雑なケースではエラー伝播が不完全になる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dav のプロパティサブシステムの中核です。
  • サブリクエスト (subreq) を使用した場合は、必ず dav_close_propdb で適切に破棄してください。
  • プロパティ値内の名前空間は ns<index> 形式に変換されるため、クライアント側で特定のプレフィックスに依存しないよう注意が必要です。
