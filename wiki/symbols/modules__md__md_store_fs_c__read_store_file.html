read_store_file

定義

  • 種別: function
  • 定義箇所: modules/md/md_store_fs.c:203

目的

ファイルシステム上のストアメタデータファイルを読み込み、バージョン検証、暗号化キーの抽出、および必要に応じたデータ構造のマイグレーションを実行します。

シグネチャまたは形状

apr_status_t read_store_file(md_store_fs_t *s_fs, const char *fname, 
                             apr_pool_t *p, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • s_fs: ストレージ状態を保持する md_store_fs_t 構造体。
  • fname: 読み込むファイルのパス。
  • p: メモリ割り当て用プール。
  • ptemp: 一時的な処理用プール。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS、バージョン不整合やキー欠損等のエラー時は APR_EINVAL 等のステータスコードを返します。

前提条件と不変条件

  • fname で指定されたファイルは有効な JSON 形式である必要があります。
  • s_fs->key には、ファイルから抽出された Base64URL デコード済みのデータが格納されます。
  • ファイル内のバージョンが MD_STORE_VERSION を超えている場合、読み込みは拒否されます。

アルゴリズムまたは振る舞い

1. md_json_readf を使用して JSON ファイルを読み込みます。 2. MD_KEY_STORE および MD_KEY_VERSION からストアバージョンを取得します（未定義の場合は 1.0 とみなします）。 3. バージョンが現在の MD_STORE_VERSION より新しい場合はエラーを返します。 4. JSON 内の MD_KEY_KEY を取得し、Base64URL デコードして s_fs->key に設定します。キー長が FS_STORE_KLEN と一致しない場合はエラーとなります。 5. バージョンが古い場合、upgrade_from_1_0 等を呼び出してマイグレーションを実行し、更新後の JSON をファイルに書き戻します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_store_fs.c:277 (関数名等の詳細はソース抜粋外のため不明)
  • **呼び出し先**:
  • md_json_readf, md_json_getn, md_json_dups, md_json_del, md_json_setn, md_json_freplace
  • md_log_perror
  • md_util_base64url_decode
  • upgrade_from_1_0

関連データ構造

  • md_store_fs_t: ストレージのベースパスや暗号化キーを管理するメイン構造体。
  • md_json_t: JSON データの解析結果を保持する構造体。

失敗モードと境界条件

  • **バージョン不整合**: ファイルのバージョンが現在のシステムより新しい場合、APR_EINVAL を返します。
  • **キー欠損または不正**: JSON 内にキーが存在しない、またはデコード後の長さが FS_STORE_KLEN と異なる場合、APR_EINVAL を返します。
  • **マイグレーション失敗**: upgrade_from_1_0 等の処理が失敗した場合、そのステータスが返されます。

保守担当者向けメモ

  • 本関数はストアの初期化および互換性維持の要です。マイグレーションロジックを変更する際は、MD_STORE_VERSION の定義と整合性を取る必要があります。
  • ptemp はファイル書き戻し時のメモリ管理に使用されます。
