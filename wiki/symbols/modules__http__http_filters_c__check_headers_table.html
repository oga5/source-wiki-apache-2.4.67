check_headers_table

定義

  • 種別: function
  • 定義箇所: modules/http/http_filters.c:754

目的

HTTPレスポンスヘッダのテーブル（apr_table_t）を走査し、各ヘッダがプロトコル仕様に適合しているかを検証する。

シグネチャまたは形状

static int check_headers_table(apr_table_t *t, struct check_header_ctx *ctx)

入力と出力

  • **入力**:
  • t: 検証対象のヘッダテーブル。
  • ctx: 検証状態および設定を保持するコンテキスト構造体。
  • **出力**:
  • int: 検証に成功した場合は 1、一つでも不正なヘッダが含まれていた場合は 0 を返す。

前提条件と不変条件

  • t は apr_table_t として有効であること。
  • ctx は check_header 関数が期待する形式で初期化されていること。

アルゴリズムまたは振る舞い

1. apr_table_elts(t) を使用してテーブル内の全ヘッダエントリを取得する。 2. 各エントリをループで走査する。

  • キーが存在しないエントリはスキップする。
  • check_header(ctx, header->key, &header->val) を呼び出し、個別のヘッダを検証する。

    • 3. check_header が 0 を返した場合、直ちに検証を中断し 0 を返す。 4. 全てのエントリが正常に検証された場合、1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_filters.c:796 (r->headers_out の検証)
  • modules/http/http_filters.c:797 (r->err_headers_out の検証)
  • **呼び出し先**:
  • apr_table_elts (APRライブラリ)
  • check_header (同一ファイル内の検証関数)

関連データ構造

  • apr_table_t: ヘッダを保持するAPRのテーブル構造体。
  • check_header_ctx: 検証プロセスで共有されるコンテキスト。
  • apr_table_entry_t: テーブル内の個別のキー・値ペア。

失敗モードと境界条件

  • テーブルが空の場合、ループは実行されず 1 を返す。
  • ヘッダのキーが NULL の場合、そのエントリは無視される。
  • check_header が失敗（不正なヘッダを検出）した場合、即座に処理を停止する。

保守担当者向けメモ

  • 本関数は headers_out と err_headers_out の両方の検証に使用されるため、変更時は両方の出力ヘッダに対する影響を考慮すること。
  • check_header の戻り値が 0 の場合、呼び出し元はプロトコルエラーとして処理を継続する可能性があるため、検証ロジックの厳密さを維持すること。
