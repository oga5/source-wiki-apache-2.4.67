socache_dc_store

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dc.c:94

目的

分散キャッシュ（distcache）プロバイダーを使用して、セッションデータ等のキャッシュエントリを保存します。

シグネチャまたは形状

apr_status_t socache_dc_store(ap_socache_instance_t *ctx, server_rec *s,
                              const unsigned char *id, unsigned int idlen,
                              apr_time_t expiry,
                              unsigned char *der, unsigned int der_len,
                              apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンス（DC_CTX を保持）。
  • s: サーバーレコード。
  • id: キャッシュキーとなる識別子。
  • idlen: id の長さ。
  • expiry: 絶対有効期限（apr_time_t）。
  • der: 保存するデータ本体。
  • der_len: der の長さ。
  • p: メモリプール。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • 失敗時は APR_EGENERAL を返します。

前提条件と不変条件

  • expiry は絶対時間として渡されますが、内部で apr_time_now() を減算し、相対ミリ秒に変換して処理されます。
  • DC_CTX_add_session が正常に完了することが期待されます。

アルゴリズムまたは振る舞い

1. 引数 expiry から現在の時刻（apr_time_now()）を減算し、有効期限までの相対時間を算出します。 2. apr_time_msec() を用いて、算出された時間をミリ秒単位に変換します。 3. DC_CTX_add_session を呼び出し、分散キャッシュへデータを送信します。 4. 処理結果に応じてログを出力し、ステータスコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dc.c:179 に参照の記述があります。
  • **呼び出し先**: distcache ライブラリの DC_CTX_add_session を呼び出します。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスを保持する構造体。
  • server_rec: Apache のサーバー設定情報。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • DC_CTX_add_session が 0（失敗）を返した場合、エラーログを出力し APR_EGENERAL を返します。

保守担当者向けメモ

  • 本関数は絶対時間を相対時間に変換して扱っています。これは distcache API の仕様に合わせたものです。
  • 接続の永続化については、mod_socache_dc.c 全体として fork() 後のファイルディスクリプタ共有問題を避けるため、現在は一時的な接続を行う設計となっています。
