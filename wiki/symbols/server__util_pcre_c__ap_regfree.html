ap_regfree

定義

  • 種別: function
  • 定義箇所: server/util_pcre.c:145

目的

コンパイル済みの正規表現オブジェクト（ap_regex_t）に関連付けられた PCRE ライブラリのリソースを解放する。

シグネチャまたは形状

void ap_regfree(ap_regex_t *preg)

入力と出力

  • 入力: preg - ap_regcomp によって初期化された ap_regex_t 構造体へのポインタ。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • preg は有効な ap_regex_t オブジェクトを指している必要がある。
  • 本関数呼び出し後、preg->re_pcre は無効化されるため、当該オブジェクトに対するマッチング操作は行えない。

アルゴリズムまたは振る舞い

1. コンパイル時に使用された PCRE のバージョン（PCRE1 または PCRE2）を判定する。 2. PCRE2 が有効な場合、pcre2_code_free を呼び出してコンパイル済みコードを解放する。 3. PCRE1 が有効な場合、pcre_free を呼び出してコンパイル済みコードを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/util.c:258
  • server/util.c:281
  • server/util_regex.c:27
  • 呼び出し先:
  • pcre2_code_free (PCRE2 使用時)
  • pcre_free (PCRE1 使用時)

関連データ構造

  • ap_regex_t: 正規表現のコンパイル結果を保持する構造体。

失敗モードと境界条件

  • preg が NULL の場合の挙動については、本ソースコード内では明示的なガード処理は記述されていない。

保守担当者向けメモ

  • 本関数は ap_regcomp で確保されたリソースの解放のみを担当する。マッチング時に使用されるスレッドローカルなワークスペース（match_data 等）の解放は、別のメカニズム（free_match_data 等）によって管理される。
