find_code_page

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_charset_lite.c:193

目的

HTTP リクエストの fixup フックとして動作し、設定に基づいた文字セット変換が必要かどうかを判定し、必要な場合に変換用コンテキスト（charset_req_t および charset_filter_ctx_t）を初期化します。

シグネチャまたは形状

static int find_code_page(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）
  • **出力**: int（Apache のステータスコード。成功または変換不要時は DECLINED、変換設定エラー時は HTTP_INTERNAL_SERVER_ERROR を返す）

前提条件と不変条件

  • r->per_dir_config から取得した charset_dir_t 設定が存在すること。
  • プロキシリクエスト、または特定の mod_rewrite 指標（redirect:, gone: 等）が含まれるリクエストは変換対象外とする。
  • ソース文字セットとデフォルト文字セットが同一の場合は変換を行わない。

アルゴリズムまたは振る舞い

1. **設定確認**: ディレクトリ設定に charset_source と charset_default が存在するか確認し、欠落していれば DECLINED を返す。 2. **除外判定**: プロキシリクエストや特定のファイル名プレフィックスを持つリクエストをスキップ。 3. **変換不要判定**: ソースとデフォルトの文字セットが一致する場合は DECLINED を返す。 4. **コンテキスト初期化**:

  • charset_req_t と出力用コンテキストをリクエストプールから確保し、request_config に格納。
  • 出力用ブリゲード (tmpbb) を作成。

    • 5. **入力変換のセットアップ**: メソッドが M_PUT または M_POST の場合、入力用コンテキストを確保し、apr_xlate_open で変換ハンドルを初期化。エラー時は HTTP_INTERNAL_SERVER_ERROR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の fixup フックとして登録されている（modules/filters/mod_charset_lite.c:1124）。
  • **呼び出し先**:
  • ap_get_module_config, ap_set_module_config
  • apr_pcalloc, apr_palloc
  • apr_brigade_create
  • apr_xlate_open, apr_xlate_sb_get
  • ap_log_rerror

関連データ構造

  • charset_dir_t: ディレクトリごとの設定。
  • charset_req_t: リクエストごとの変換管理情報。
  • charset_filter_ctx_t: フィルタごとの変換コンテキスト（apr_xlate_t ハンドルやバッファを含む）。

失敗モードと境界条件

  • **設定不備**: ソースまたはデフォルト文字セットが未設定の場合、変換をスキップ。
  • **変換初期化失敗**: apr_xlate_open が失敗した場合、HTTP_INTERNAL_SERVER_ERROR を返し、リクエスト処理を中断する。
  • **メソッド制限**: OPTIONS メソッドにリクエストボディが含まれる場合、本関数では変換設定を行わない。

保守担当者向けメモ

  • 本関数はリクエストごとに実行されるため、メモリ確保には r->pool を使用しています。
  • 入力フィルタのコンテキストは M_PUT/M_POST 時のみ生成されるため、他のメソッドでは reqinfo->input_ctx が NULL であることを前提としています。
