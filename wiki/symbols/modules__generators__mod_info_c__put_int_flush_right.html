put_int_flush_right

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_info.c:108

目的

整数値を指定された桁数で右寄せし、HTMLレスポンスまたはファイルストリームへ出力するユーティリティ関数です。

シグネチャまたは形状

static void put_int_flush_right(request_rec *r, int i, int field)

入力と出力

  • r: リクエストレコード（NULLの場合はファイルストリーム out を使用）。
  • i: 出力対象の整数値。
  • field: 最小桁数（再帰的に減少する）。
  • 出力: r が有効な場合は ap_rputc / ap_rputs を介してクライアントへ、それ以外の場合は apr_file_putc / apr_file_printf を介して out へ出力。

前提条件と不変条件

  • r が NULL の場合、グローバルなファイルハンドル out が有効である必要があります。
  • 再帰呼び出しにより、field が 1 より大きいか、または i が 9 より大きい間、上位桁の処理が先行して実行されます。

アルゴリズムまたは振る舞い

1. **再帰的桁処理**: field > 1 または i > 9 の条件を満たす場合、field - 1 を指定して自身を再帰呼び出しし、上位桁を先に処理します。 2. **数値出力**: i が 0 でない場合、i % 10 の値を文字に変換して出力します。 3. **パディング**: i が 0 の場合（再帰の深さや数値の不足分）、r があれば &nbsp; を、なければスペースを出力して右寄せを維持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_info.c:164
  • modules/generators/mod_info.c:171
  • **呼び出し先**:
  • 自身（再帰呼び出し）
  • ap_rputc / ap_rputs (Apache API)
  • apr_file_putc / apr_file_printf (APR API)

関連データ構造

  • request_rec: Apache のリクエスト状態を保持する構造体。

失敗モードと境界条件

  • i が負の値である場合の挙動は明示されていません。
  • field が現在の数値の桁数より小さい場合、数値の切り捨ては行われず、全桁が出力されます。

保守担当者向けメモ

  • 本関数は mod_info における行番号表示などの整形に使用されています。
  • r が NULL の場合に参照される out は、ファイルスコープのグローバル変数として定義されている前提の設計です。
