winnt_get_connection

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/child.c:733

目的

I/O Completion Port (IOCP) を監視し、新しい接続要求またはシャットダウン通知を待機して、処理対象の接続コンテキストを取得します。

シグネチャまたは形状

winnt_conn_ctx_t *winnt_get_connection(winnt_conn_ctx_t *context)

入力と出力

  • **入力**: context (再利用対象の winnt_conn_ctx_t 構造体へのポインタ)
  • **出力**: 成功時は有効な winnt_conn_ctx_t 構造体へのポインタを返し、シャットダウン時やエラー時は NULL を返します。

前提条件と不変条件

  • ThreadDispatchIOCP が初期化済みであること。
  • g_blocked_threads アトミックカウンタがスレッドの待機状態を正確に反映していること。
  • workers_may_exit フラグがスレッド終了の制御に使用されること。

アルゴリズムまたは振る舞い

1. mpm_recycle_completion_context(context) を呼び出し、前回のコンテキストを再利用キューへ戻します。 2. g_blocked_threads をインクリメントし、待機状態に入ります。 3. GetQueuedCompletionStatus を用いて ThreadDispatchIOCP からの通知を無限待機します。

  • workers_may_exit が真であれば、カウンタをデクリメントして NULL を返します。
  • GetQueuedCompletionStatus が失敗した場合はエラーログを出力し、待機を継続します。

    • 4. CompKey の値に応じて分岐します。

  • IOCP_CONNECTION_ACCEPTED: CONTAINING_RECORD マクロを使用して OVERLAPPED 構造体から winnt_conn_ctx_t を復元します。
  • IOCP_SHUTDOWN またはその他: カウンタをデクリメントして NULL を返します。

    • 5. 正常終了時に g_blocked_threads をデクリメントし、取得したコンテキストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/child.c 内の worker_main 関数（行344および812）。
  • **呼び出し先**:
  • mpm_recycle_completion_context
  • apr_atomic_inc32 / apr_atomic_dec32
  • GetQueuedCompletionStatus (Windows API)
  • ap_log_error
  • apr_get_os_error

関連データ構造

  • winnt_conn_ctx_t: 接続状態を保持する主要構造体。
  • OVERLAPPED: Windows の非同期 I/O 用構造体。
  • ThreadDispatchIOCP: 接続受け渡しに使用される I/O Completion Port ハンドル。

失敗モードと境界条件

  • GetQueuedCompletionStatus が失敗した場合、エラーコードをログに記録し、ループを継続して再試行します。
  • workers_may_exit が設定されている場合、または IOCP_SHUTDOWN を受信した場合は、即座に待機を終了し NULL を返します。
  • 予期しない CompKey を受信した場合、安全のため NULL を返して終了します。

保守担当者向けメモ

  • winnt_conn_ctx_t の再利用ロジックにおいて、エラーパスでのメモリリークの可能性がある旨がソース内に示唆されています。
  • _WIN64 定義の有無により CompKey の型が ULONG_PTR または DWORD に切り替わるため、プラットフォーム依存の型定義に注意してください。
