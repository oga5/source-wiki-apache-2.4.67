expr_provider_multi

定義

  • 種別: struct
  • 定義箇所: server/util_expr_eval.c:1671

目的

複数の名前（キー）を単一の関数ポインタに関連付けるためのプロバイダー定義構造体です。ap_expr エンジンにおいて、変数や関数を検索・ディスパッチする際のテーブル定義に使用されます。

シグネチャまたは形状

struct expr_provider_multi {
    const void *func;
    const char *const *names;
};

入力と出力

  • func: 実行時に呼び出される関数へのポインタ。
  • names: この関数に関連付けられた名前のリスト（NULL 終端の文字列配列）。

前提条件と不変条件

  • names は NULL で終了する文字列配列である必要があります。
  • func は、評価エンジンが期待する特定のシグネチャを持つ関数を指している必要があります。

アルゴリズムまたは振る舞い

この構造体は、var_providers などの静的配列を構築するために使用されます。評価エンジンは、検索対象の名前と一致するエントリを names 配列から探索し、見つかった場合に func を実行します。

呼び出し元と呼び出し先

  • 呼び出し元: server/util_expr_eval.c 内の var_providers 配列定義および検索ロジック（prov = var_providers 等）。
  • 呼び出し先: 評価時に func が指す関数が実行されます。

関連データ構造

  • var_providers: expr_provider_multi 型の静的配列。
  • expr_provider_single: 単一の名前と関数をマッピングする対照的な構造体。

失敗モードと境界条件

  • names 配列が適切に NULL 終端されていない場合、検索ロジックでメモリ違反が発生する可能性があります。
  • 検索対象の名前がどのエントリの names にも含まれない場合、プロバイダーは見つかりません。

保守担当者向けメモ

  • 本構造体は util_expr_eval.c 内のプロバイダー登録テーブルの基盤です。新しい変数や関数を追加する際は、この構造体を用いて var_providers または対応するテーブルを拡張してください。
