worker_child_bucket

定義

  • 種別: struct
  • 定義箇所: server/mpm/worker/worker.c:174

目的

worker MPM において、リスナーソケットのグループとそれに関連するプロセス間通信（POD: Pipe of Death）および同期用ミューテックスを保持するための構造体です。

シグネチャまたは形状

typedef struct worker_child_bucket {
    ap_pod_t *pod;
    ap_listen_rec *listeners;
    apr_proc_mutex_t *mutex;
} worker_child_bucket;

入力と出力

  • **入力**: 構造体自体はデータ保持用であり、関数ではないため入力引数はありません。
  • **出力**: 戻り値はありません。

前提条件と不変条件

  • pod は ap_pod_t 型のポインタであり、子プロセスへのシグナル伝達に使用されます。
  • listeners は ap_listen_rec 型のポインタであり、このバケットが担当するリスニングソケットのリストを指します。
  • mutex は apr_proc_mutex_t 型のポインタであり、リスナー間の競合を制御するために使用されます。

アルゴリズムまたは振る舞い

本構造体は、複数のリスナーソケットをグループ化し、それらを管理するためのコンテナとして機能します。各バケットは、特定のリスナーセットに対する排他制御（mutex）と、子プロセスに対するシグナル制御（pod）をカプセル化します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/worker/worker.c 内の静的変数 all_buckets の定義（179行目）において、この構造体の配列として参照されています。
  • **呼び出し先**: 構造体定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • ap_pod_t: 子プロセスへの終了シグナル伝達用パイプ構造体。
  • ap_listen_rec: リスニングソケットの情報を保持する構造体。
  • apr_proc_mutex_t: プロセス間排他制御用ミューテックス。

失敗モードと境界条件

  • 構造体内の各ポインタが NULL である場合、適切に初期化されていない状態となります。
  • 実行時に mutex のロック取得に失敗した場合、リスナーの競合が発生する可能性があります。

保守担当者向けメモ

  • all_buckets として静的配列で管理されているため、この構造体のレイアウト変更は MPM のリスナー管理ロジック全体に影響を与えます。
  • pod や mutex のライフサイクルは、worker MPM のプロセス生成および終了処理と密接に関連しています。
