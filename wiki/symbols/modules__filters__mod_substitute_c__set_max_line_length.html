set_max_line_length

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_substitute.c:706

目的

Apache HTTP Server の設定ディレクティブ SubstituteMaxLineLength を解析し、subst_dir_conf 構造体に最大行長制限を設定する。

シグネチャまたは形状

static const char *set_max_line_length(cmd_parms *cmd, void *cfg, const char *arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • cfg: subst_dir_conf 構造体へのポインタ。
  • arg: 設定値を含む文字列（数値およびオプションの単位）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • arg は非負の整数である必要がある。
  • arg には単位として b (byte), k (kilobyte), m (megabyte), g (gigabyte) を付与可能。
  • 設定された max_line_length は apr_size_t 型にキャストされる。

アルゴリズムまたは振る舞い

1. apr_strtoff を使用して arg を数値に変換する。 2. 単位接尾辞（k, m, g）を判定し、対応する倍率（KBYTE, MBYTE, GBYTE）を乗算する。 3. 不正な単位や負の数値が指定された場合、エラーメッセージを返す。 4. 正常な場合、dcfg->max_line_length に値を格納し、dcfg->max_line_length_set フラグを 1 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_substitute.c:750 の AP_INIT_TAKE1 マクロにより、Apache 設定読み込み時に呼び出される。
  • **呼び出し先**: apr_strtoff (APR ライブラリ関数)。

関連データ構造

  • subst_dir_conf: mod_substitute のディレクトリ設定を保持する構造体。

失敗モードと境界条件

  • **失敗モード**:
  • 数値変換失敗時（apr_strtoff がエラーを返す場合）。
  • 負の数値が指定された場合。
  • 不正な単位（b, k, m, g 以外）が指定された場合。
  • **境界条件**:
  • 単位なしの数値はバイト単位として扱われる。
  • 単位指定の直後に余分な文字がある場合はエラーとなる。

保守担当者向けメモ

  • 本関数は SubstituteMaxLineLength ディレクティブ専用のハンドラである。
  • 単位の倍率定数（KBYTE 等）は mod_substitute.c 内で定義されている。
  • apr_size_t へのキャストを行っているため、プラットフォームのポインタサイズを超える値の指定には注意が必要である。
