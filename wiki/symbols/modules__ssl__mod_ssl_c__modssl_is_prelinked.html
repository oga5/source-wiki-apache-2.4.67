modssl_is_prelinked

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:316

目的

Apache HTTP Server の実行バイナリに mod_ssl が静的にリンクされているかどうかを判定します。

シグネチャまたは形状

static int modssl_is_prelinked(void)

入力と出力

  • 入力: なし
  • 出力: int 型。静的にリンクされている場合は 1、そうでない場合は 0 を返します。

前提条件と不変条件

  • Apache のグローバル変数 ap_prelinked_modules が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. ap_prelinked_modules 配列を先頭から順に走査します。 2. 各モジュール構造体の name フィールドを "mod_ssl.c" と比較します。 3. 一致するモジュールが見つかった時点で 1 を返し、配列の終端（NULL）に達した場合は 0 を返します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ssl/mod_ssl.c:395 にて、静的変数 modssl_running_statically の初期化時に呼び出されています。
  • 呼び出し先: 標準ライブラリ関数 strcmp を使用しています。

関連データ構造

  • module (Apache 内部構造体): モジュール情報を保持します。
  • ap_prelinked_modules (Apache 外部変数): 静的にリンクされたモジュールのリスト。

失敗モードと境界条件

  • ap_prelinked_modules が空の場合、ループは即座に終了し 0 を返します。
  • モジュール名が "mod_ssl.c" と一致しない場合、リスト全体を走査した後に 0 を返します。

保守担当者向けメモ

  • 本関数は、OpenSSL のクリーンアップ処理など、モジュールが静的か動的かによって挙動を変える必要がある箇所での判定フラグ生成に使用されます。
  • mod_ssl のソースファイル名が変更された場合、文字列比較の定数も更新する必要があります。
