acme_problem_status_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme.c:43

目的

ACME サーバーから返されるエラー（Problem）と、Apache HTTP Server 内部で扱うステータスコードとの対応を定義するための構造体です。

シグネチャまたは形状

struct acme_problem_status_t {
    /* 構造体の詳細はソースコードの定義箇所を参照 */
};

入力と出力

  • **入力**: ACME サーバーからのエラーレスポンス（application/problem+json）。
  • **出力**: Apache 内部で利用可能なステータスコードへのマッピング。

前提条件と不変条件

  • modules/md/md_acme.c 内の静的配列 Problems によって、既知の ACME エラータイプとステータスコードのペアが保持されています。

アルゴリズムまたは振る舞い

  • inspect_problem() 関数などがこの構造体を利用し、サーバーからのエラーを解析して再試行の必要性（APR_EAGAIN 等）やエラーの性質を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme.c 内の Problems 配列定義（45行目、51行目）にて参照されています。
  • **呼び出し先**: 具体的な関数呼び出し関係については、提供されたソース excerpt には記載がありません。

関連データ構造

  • md_result_t: 処理結果やエラー詳細を管理するコンテナ。
  • Problems: 本構造体を用いて定義された、エラー対応表を保持する静的配列。

失敗モードと境界条件

  • 定義されていないエラータイプがサーバーから返された場合、マッピングが失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は md_acme.c 内で完結しており、ACME プロトコルのエラーハンドリングの基盤となります。
  • 新しい ACME エラータイプに対応する場合は、Problems 配列にエントリを追加してください。
