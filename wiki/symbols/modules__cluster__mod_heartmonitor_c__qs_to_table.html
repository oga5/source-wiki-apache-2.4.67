qs_to_table

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartmonitor.c:130

目的

クエリ文字列形式（key=value&key2=value2）の入力文字列を解析し、apr_table_t 構造体にキーと値のペアとして格納するユーティリティ関数です。

シグネチャまたは形状

static void qs_to_table(const char *input, apr_table_t *parms, apr_pool_t *p)

入力と出力

  • **入力**:
  • input: 解析対象のクエリ文字列（const char *）。
  • parms: 解析結果を格納する apr_table_t オブジェクト。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**: なし（parms が更新される）。

前提条件と不変条件

  • input が NULL の場合、関数は何もせず即座に復帰します。
  • input は apr_pstrdup により内部で複製されるため、呼び出し元が保持する元の文字列には影響を与えません。
  • キーまたは値に含まれる URL エンコードされた文字は、ap_unescape_url によってデコードされます。

アルゴリズムまたは振る舞い

1. input が NULL であれば終了します。 2. input を apr_pstrdup で複製し、apr_strtok を使用して & を区切り文字としてトークン化します。 3. 各トークンについて = を検索します。

  • = が存在する場合、その位置を \0 で置換してキーと値に分割します。
  • = が存在しない場合、値は "1" とみなされます。

    • 4. キーと値の両方に対して ap_unescape_url を実行し、デコードします。 5. apr_table_set を使用して、解析したキーと値を parms テーブルに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cluster/mod_heartmonitor.c:314

  • **呼び出し先**:

関連データ構造

  • apr_table_t: キーと値のペアを保持する Apache Portable Runtime のテーブル構造体。

失敗モードと境界条件

  • input が空文字列の場合、ループは実行されず何も追加されません。
  • 値が指定されていないキー（例: key&key2=val）は、値 "1" としてテーブルに格納されます。
  • ap_unescape_url が失敗した場合の挙動は、当該 API の仕様に依存します。

保守担当者向けメモ

  • 本関数はクエリ文字列のパースに apr_strtok を使用しており、破壊的な文字列操作（\0 の挿入）を行いますが、apr_pstrdup で複製した領域に対して行われるため安全です。
  • 将来的にクエリ文字列の仕様が変更された場合、区切り文字やデフォルト値のロジックを調整する必要があります。
