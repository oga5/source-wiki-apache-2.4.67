header_state

定義

  • 種別: enum
  • 定義箇所: modules/mappers/mod_negotiation.c:740

目的

type-map ファイルの解析プロセスにおいて、現在のヘッダー行の読み取り状態を追跡するために使用されます。

シグネチャまたは形状

enum header_state {
    header_eof,
    header_seen,
    header_sep
};

入力と出力

  • 本シンボルは状態遷移の列挙型であり、関数への入力（状態の保持）および出力（状態の更新）として利用されます。

前提条件と不変条件

  • type-map ファイルの読み込み処理において、ファイル終端、ヘッダーの存在、またはセパレーターの検出という3つの排他的な状態のいずれかを取ります。

アルゴリズムまたは振る舞い

  • get_header_line 関数において、ファイルから読み取った行の内容に応じて、以下のいずれかの状態を返します。
  • header_eof: ファイルの終端に到達した場合。
  • header_seen: 有効なヘッダー行が読み取られた場合。
  • header_sep: ヘッダーセクションの区切り（空行など）を検出した場合。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_negotiation.c 内の get_header_line 関数（定義元）および、同ファイル内のヘッダー解析ロジック（hstate 変数として使用）。
  • **呼び出し先**: なし（列挙型のため）。
  • 注: 他のモジュール（mod_authnz_fcgi.c や mod_proxy_fcgi.c）にも同名の変数や類似の概念が存在しますが、これらは本シンボルとは別の定義を参照している可能性が高いです。

関連データ構造

  • negotiation_state: mod_negotiation.c におけるリクエストごとのネゴシエーション状態を保持する構造体。

失敗モードと境界条件

  • ファイル読み込みエラーや不正なフォーマットの type-map ファイルに対して、解析ロジックが header_eof を早期に返すことで処理を終了させる制御フローの一部として機能します。

保守担当者向けメモ

  • 本シンボルは mod_negotiation.c 内部でのみ使用されるローカルな状態定義です。他モジュールの同名変数と混同しないよう注意してください。
