read_chr

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_push.c:153

目的

解析対象の文字列コンテキストにおいて、現在の位置にある文字が指定された文字と一致するかを判定し、一致した場合は解析位置を前進させる。

シグネチャまたは形状

static int read_chr(link_ctx *ctx, char c)

入力と出力

  • **入力**:
  • ctx: 解析状態を保持する link_ctx 構造体へのポインタ。
  • c: 比較対象の文字。
  • **出力**:
  • int: 一致した場合は 1 を返し、一致しない場合または終端に達している場合は 0 を返す。

前提条件と不変条件

  • ctx は有効な link_ctx 構造体を指している必要がある。
  • ctx->i は現在の解析位置を示し、ctx->slen は文字列の長さを保持している。

アルゴリズムまたは振る舞い

1. 現在の解析位置 ctx->i が文字列の長さ ctx->slen 未満であることを確認する。 2. ctx->s[ctx->i] が指定された文字 c と等しいか比較する。 3. 一致する場合、ctx->i をインクリメント（前進）させ、1 を返す。 4. 一致しない場合、何も変更せず 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_util.c (複数箇所)
  • modules/http2/h2_push.c (複数箇所)
  • modules/md/md_util.c (複数箇所)
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わない。

関連データ構造

  • link_ctx: 解析中の文字列 (s)、その長さ (slen)、および現在のインデックス (i) を保持する構造体。

失敗モードと境界条件

  • ctx->i が ctx->slen に達している場合、比較を行わずに 0 を返す（境界外アクセスの防止）。

保守担当者向けメモ

  • 本関数は Link ヘッダー等の構文解析において、特定の区切り文字や記号を消費するために頻繁に使用されるユーティリティである。
  • 類似の関数として skip_ws などがあり、これらと組み合わせてパーサーが構成されている。
