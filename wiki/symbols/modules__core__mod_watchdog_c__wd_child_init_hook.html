wd_child_init_hook

定義

  • 種別: function
  • 定義箇所: modules/core/mod_watchdog.c:551

目的

子プロセスの初期化フェーズにおいて、アクティブなすべての Watchdog インスタンスに対してワーカースレッドを起動する。

シグネチャまたは形状

static void wd_child_init_hook(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 初期化に使用するメモリプール。
  • s: サーバーレコード。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • wd_server_conf が初期化済みであること。
  • AP_WATCHDOG_PGROUP プロバイダグループに Watchdog インスタンスが登録されていること。

アルゴリズムまたは振る舞い

1. wd_server_conf->child_workers が設定されていない場合、デバッグログを出力して終了する。 2. ap_list_provider_names を使用して、登録されているすべての Watchdog プロバイダ名を取得する。 3. 取得した各プロバイダに対し、ap_lookup_provider で ap_watchdog_t インスタンスを取得する。 4. インスタンスが active である場合、wd_startup を呼び出してワーカースレッドを生成する。 5. wd_startup が失敗した場合はエラーログを出力し、処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_child_init に登録されており、子プロセス起動時に Apache HTTP Server から呼び出される。
  • **呼び出し先**:
  • ap_list_provider_names: プロバイダ一覧の取得。
  • ap_lookup_provider: 特定の Watchdog インスタンスの取得。
  • wd_startup: ワーカースレッドの生成。
  • ap_log_error: ログ出力。

関連データ構造

  • ap_watchdog_t: 監視対象のインスタンス構造体。
  • wd_server_conf_t: モジュール全体のサーバー設定。
  • ap_list_provider_names_t: プロバイダ名リストの保持に使用。

失敗モードと境界条件

  • Watchdog が一つも設定されていない場合、処理は即座に終了する。
  • wd_startup によるスレッド生成に失敗した場合、エラーログを出力して処理を中断する。

保守担当者向けメモ

  • 本関数は子プロセスごとに実行されるため、スレッド生成の成否は各プロセスの動作に直結する。
  • wd_server_conf->pool を使用してスレッドのライフサイクルを管理している点に注意すること。
