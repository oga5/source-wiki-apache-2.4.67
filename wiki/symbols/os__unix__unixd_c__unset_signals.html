unset_signals

定義

  • 種別: function
  • 定義箇所: os/unix/unixd.c:500

目的

MPM の再起動や終了処理の過程で、retained_data に保持されているシグナル状態をリセットし、次回の実行サイクルに向けて状態を初期化する。

シグネチャまたは形状

static apr_status_t unset_signals(void *unused)

入力と出力

  • **入力**: void *unused（apr_pool_cleanup_register のインターフェースに合わせるための未使用引数）。
  • **出力**: apr_status_t 型のステータスコード（常に APR_SUCCESS を返す）。

前提条件と不変条件

  • retained_data が有効なメモリ領域を指している場合、その内部のシグナルフラグをリセットする。
  • retained_data が NULL の場合は、メインプロセス（ap_pglobal）が終了中であるとみなし、何も行わず正常終了する。

アルゴリズムまたは振る舞い

1. retained_data が NULL か確認し、NULL ならば即座に APR_SUCCESS を返す。 2. retained_data->shutdown_pending および retained_data->restart_pending を 0 に設定する。 3. retained_data->was_graceful を !retained_data->is_ungraceful の値で更新する。 4. retained_data->is_ungraceful を 0 にリセットする。 5. APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: os/unix/unixd.c:624 において、apr_pool_cleanup_register のクリーンアップハンドラとして登録されている。
  • **呼び出し先**: 外部関数呼び出しなし。

関連データ構造

  • ap_unixd_mpm_retained_data（retained_data）: MPM のライフサイクルを跨いで保持される状態構造体。

失敗モードと境界条件

  • retained_data が NULL の場合、安全に早期リターンする。
  • 本関数自体はエラーを発生させない設計となっている。

保守担当者向けメモ

  • 本関数は apr_pool_cleanup_register を介して呼び出されるため、プールの破棄タイミングと連動して実行される。
  • was_graceful の更新ロジックは、前回の終了が「グレースフル（緩やかな停止）」であったかどうかを判定するために使用される。
