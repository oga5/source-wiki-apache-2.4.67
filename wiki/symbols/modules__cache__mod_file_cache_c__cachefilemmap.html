cachefilemmap

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_file_cache.c:217

目的

mmapfile ディレクティブを処理し、指定されたファイルをメモリマップ（mmap）としてキャッシュに登録するコマンドハンドラです。

シグネチャまたは形状

static const char *cachefilemmap(cmd_parms *cmd, void *dummy, const char *filename)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • filename: キャッシュ対象のファイルパス。
  • **出力**:
  • 成功時またはOSがmmapをサポートしていない場合: NULL を返します。

前提条件と不変条件

  • APR_HAS_MMAP が定義されている環境（OSがmmapをサポートしていること）が必要です。
  • サーバー起動時の設定読み込みフェーズで呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. APR_HAS_MMAP マクロを確認します。 2. サポートされている場合、cache_the_file 関数を呼び出し、第3引数に 1（mmap有効フラグ）を渡してファイルをキャッシュに登録します。 3. サポートされていない場合、エラーログに警告を出力し、キャッシュ登録を行わずに終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_file_cache.c:387 において、AP_INIT_ITERATE マクロにより mmapfile ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: cache_the_file（APR_HAS_MMAP が真の場合）。

関連データ構造

  • cmd_parms: Apache の設定コマンドパラメータ構造体。

失敗モードと境界条件

  • **OS非対応**: APR_HAS_MMAP が無効な環境では、キャッシュ登録は行われず、エラーログに unable to cache file という警告が出力されます。
  • **ファイルアクセスエラー**: cache_the_file 内部でファイルが開けない、または統計情報が取得できない場合、その挙動は cache_the_file の実装に依存します。

保守担当者向けメモ

  • 本関数は mmapfile ディレクティブ専用のラッパーです。ファイルハンドルのみをキャッシュする cachefile ディレクティブについては cachefilehandle が担当します。
  • APR_HAS_MMAP のチェックはコンパイル時に決定されるため、実行環境のOSがmmapをサポートしているかどうかに依存します。
