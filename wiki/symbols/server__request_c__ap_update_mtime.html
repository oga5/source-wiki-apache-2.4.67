ap_update_mtime

定義

  • 種別: function
  • 定義箇所: server/request.c:2557

目的

リクエストに関連付けられた最終更新時刻（mtime）を、指定された依存時刻と比較し、後者が新しい場合に更新する。

シグネチャまたは形状

void ap_update_mtime(request_rec *r, apr_time_t dependency_mtime)

入力と出力

  • **入力**:
  • r: 更新対象のリクエスト構造体（request_rec）。
  • dependency_mtime: 比較対象となる時刻（apr_time_t）。
  • **出力**: なし（副作用として r->mtime を更新）。

前提条件と不変条件

  • r は有効な request_rec 構造体へのポインタであること。
  • r->mtime は、リクエストの最終更新時刻を保持するフィールドとして適切に初期化されていること。

アルゴリズムまたは振る舞い

1. r->mtime と dependency_mtime を比較する。 2. r->mtime が dependency_mtime より小さい場合、r->mtime を dependency_mtime に更新する。 3. それ以外の場合は何もしない。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のモジュールから呼び出されていることが確認されている。
  • modules/cache/cache_storage.c
  • modules/cache/mod_file_cache.c
  • modules/dav/fs/repos.c
  • modules/filters/mod_include.c
  • modules/filters/mod_reflector.c
  • modules/generators/mod_autoindex.c
  • modules/mappers/mod_negotiation.c
  • modules/test/mod_dialup.c
  • **呼び出し先**: なし（標準的な比較と代入のみを実行）。

関連データ構造

  • request_rec: リクエストのコンテキスト情報を保持する構造体。mtime フィールドを介して本関数と相互作用する。

失敗モードと境界条件

  • r が NULL の場合、未定義の動作（セグメンテーション違反）が発生する可能性がある。
  • dependency_mtime が 0 の場合、r->mtime が 0 より大きい限り更新は行われない。

保守担当者向けメモ

  • 本関数は、リクエストの最終更新時刻を「より新しい時刻」へ進めるためのユーティリティである。
  • 複数のソースから時刻情報を集約してリクエストの mtime を決定する際に使用される。
