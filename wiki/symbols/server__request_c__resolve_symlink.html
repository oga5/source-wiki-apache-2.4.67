resolve_symlink

定義

  • 種別: function
  • 定義箇所: server/request.c:533

目的

シンボリックリンクのアクセス制御（FollowSymLinks や SymLinksIfOwnerMatch）を検証し、リンク先のファイル情報を取得する。

シグネチャまたは形状

static int resolve_symlink(char *d, apr_finfo_t *lfi, int opts, apr_pool_t *p)

入力と出力

  • **入力**:
  • d: 対象のファイルパス。
  • lfi: シンボリックリンクのファイル情報（apr_finfo_t）。
  • opts: 設定オプション（OPT_SYM_OWNER, OPT_SYM_LINKS）。
  • p: メモリプール。
  • **出力**:
  • 成功時は OK を返し、lfi にリンク先のファイル情報を格納する。
  • 失敗時は HTTP_FORBIDDEN を返す。

前提条件と不変条件

  • 呼び出し元は、対象が APR_LNK（シンボリックリンク）であることを確認していることが期待される。
  • opts に OPT_SYM_OWNER または OPT_SYM_LINKS が含まれていない場合、即座に HTTP_FORBIDDEN を返す。

アルゴリズムまたは振る舞い

1. **オプション検証**: opts に許可されたフラグが含まれない場合、アクセスを拒否する。 2. **リンク先情報の取得**:

  • OPT_SYM_OWNER が指定されていない場合: apr_stat でリンク先の情報を取得し、lfi を更新する。
  • OPT_SYM_OWNER が指定されている場合:
  • リンク自体の所有者情報が不足していれば apr_stat で補完する。
  • リンク先の情報を取得し、apr_uid_compare でリンクとリンク先の所有者を比較する。所有者が一致しない場合は HTTP_FORBIDDEN を返す。

    • 3. **結果の反映**: 成功時、lfi にリンク先の情報をコピーし、元のファイル名（savename）を保持したまま呼び出し元へ戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/request.c 内の複数箇所（520行目付近のコメント、813行目、1249行目、1408行目、2391行目）。
  • **呼び出し先**: apr_stat（APRライブラリ）、apr_uid_compare（APRライブラリ）、memcpy。

関連データ構造

  • apr_finfo_t: ファイル情報を保持する構造体。
  • request_rec: 呼び出し元で利用されるリクエスト情報構造体。

失敗モードと境界条件

  • apr_stat が失敗した場合、アクセス権限がないとみなして HTTP_FORBIDDEN を返す。
  • OPT_SYM_OWNER 指定時に所有者が一致しない場合、HTTP_FORBIDDEN を返す。
  • opts に適切なフラグが設定されていない場合、HTTP_FORBIDDEN を返す。

保守担当者向けメモ

  • 本関数はシンボリックリンクのセキュリティチェックを担うため、変更時は認可ロジックへの影響を慎重に評価すること。
  • lfi->name の保持ロジックは、apr_stat 後の情報更新時にファイル名が失われないようにするための重要な処理である。
