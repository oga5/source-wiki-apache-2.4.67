set_cookie_name2

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_cookie.c:229

目的

Apache設定ファイル内の SessionCookieName2（または SessionDBDCookieName2）ディレクティブを解析し、セッションCookieの名前および属性を設定構造体に格納します。

シグネチャまたは形状

static const char *set_cookie_name2(cmd_parms *cmd, void *config, const char *args)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: session_cookie_dir_conf 構造体へのポインタ。
  • args: 設定ファイルから渡される引数文字列（Cookie名と属性のセット）。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • config は session_cookie_dir_conf 型としてキャスト可能である必要があります。
  • args に含まれるCookie名は、check_string による検証（= や & を含まないこと）を通過する必要があります。

アルゴリズムまたは振る舞い

1. args を apr_pstrdup で複製し、作業用文字列を作成します。 2. apr_strtok を使用して、最初のトークンをCookie名として抽出し、conf->name2 に格納します。 3. conf->name2_set フラグを 1 に設定します。 4. 残りの文字列から先頭の空白を除去し、残りの部分をCookie属性として conf->name2_attrs に格納します。 5. 最後に check_string を呼び出し、Cookie名の妥当性を検証した結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session_cookie.c:260 (AP_INIT_RAW_ARGS による登録)
  • modules/session/mod_session_dbd.c:628 (AP_INIT_RAW_ARGS による登録)
  • **呼び出し先**:
  • apr_pstrdup
  • apr_strtok
  • apr_isspace
  • check_string

関連データ構造

  • session_cookie_dir_conf: ディレクトリ単位のセッション設定を保持する構造体。name2（Cookie名）および name2_attrs（属性）を更新します。

失敗モードと境界条件

  • check_string が失敗した場合（Cookie名に不正な文字が含まれる場合）、そのエラーメッセージがそのまま呼び出し元に返されます。
  • args が空または不正な形式の場合、apr_strtok の挙動に依存しますが、check_string が空文字列を検証することでエラーを捕捉します。

保守担当者向けメモ

  • 本関数は mod_session_cookie と mod_session_dbd の両方から参照されています。設定の変更が両モジュールの挙動に影響を与える点に注意してください。
  • name2_attrs は args 文字列内のポインタを直接参照するため、設定のライフサイクルがリクエスト終了後も維持されることを前提としています。
