feedback

定義

  • 種別: function
  • 定義箇所: support/win32/wintty.c:361

目的

別スレッドとして動作し、コンソール入力（CONIN$）を監視して標準出力（stdout）へ転送することで、サービス環境下での双方向通信を仲介します。

シグネチャまたは形状

DWORD WINAPI feedback(LPVOID arg)

入力と出力

  • **入力**: LPVOID arg（feedback_args_t 構造体へのポインタ）。この構造体には、読み取り元の入力ハンドル（in）と書き込み先の出力ハンドル（out）が含まれます。
  • **出力**: 成功時は 0 を返します。

前提条件と不変条件

  • 引数 arg は feedback_args_t 型としてキャスト可能である必要があります。
  • feed->in および feed->out は、有効な Windows ハンドルである必要があります。

アルゴリズムまたは振る舞い

1. feed->in から ReadFile を使用してデータをバッファ（1024バイト）に読み込みます。 2. 読み込みが成功している間、feed->out に対して WriteFile を実行し、データを転送します。 3. ReadFile が失敗するか、読み込みデータ長が 0 になった時点でループを終了します。 4. 終了時に printerr を呼び出し、GetLastError を用いて EOF 到達時のエラー情報を出力します。 5. 最後に 0 を返してスレッドを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: support/win32/wintty.c:326 において CreateThread により呼び出されます。
  • **呼び出し先**: ReadFile, WriteFile（Windows API）、および printerr（同ファイル内の関数）。

関連データ構造

  • feedback_args_t: 入出力ハンドルを保持するために使用されます。

失敗モードと境界条件

  • ReadFile が失敗した場合、ループを抜けて終了します。
  • WriteFile が失敗した場合、ループを抜けて終了します。
  • EOF 到達時は printerr によりエラーコードと共にログが出力されます。

保守担当者向けメモ

  • 本関数はスレッドエントリポイントとして設計されています。
  • str バッファは 1024 バイトの固定長です。
  • サービス環境特有のハンドル継承や権限設定については、main 関数側の初期化処理と密接に関連しているため、変更時は注意が必要です。
