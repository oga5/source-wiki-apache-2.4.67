ap_list_provider_names_t

定義

  • 種別: struct
  • 定義箇所: include/ap_provider.h:35

目的

プロバイダー名の一覧を取得する際に、個々のプロバイダー名を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    const char *provider_name;
} ap_list_provider_names_t;

入力と出力

  • 入力: provider_name（プロバイダー名を指す文字列ポインタ）
  • 出力: なし（構造体として値を保持）

前提条件と不変条件

  • provider_name は、プロバイダーが登録された際に指定された名前を指します。
  • 通常、この構造体は配列の要素として使用され、apr_array_header_t 等のコンテナを介して管理されます。

アルゴリズムまたは振る舞い

  • プロバイダーの検索結果をリストとして呼び出し側に提供する際、各要素の型として利用されます。
  • 呼び出し側は、取得した配列の elts メンバを本構造体にキャストして各プロバイダー名にアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/core/mod_watchdog.c
  • modules/dav/main/mod_dav.c
  • modules/dav/main/std_liveprop.c
  • 呼び出し先:
  • 情報なし（本構造体はデータ保持用であり、関数呼び出しを含まないため）

関連データ構造

  • ap_list_provider_groups_t: プロバイダーのグループとバージョンを保持する対となる構造体。

失敗モードと境界条件

  • リストが空の場合、配列の要素数は 0 となります。
  • provider_name は NULL になる可能性があります（実装依存）。

保守担当者向けメモ

  • 本構造体は ap_list_provider_names 関数が返す配列の各要素として使用されます。
  • 構造体の拡張を行う場合は、バイナリ互換性に注意してください。
