add_cern_meta_data

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_cern_meta.c:267

目的

リクエストされたファイルに対応する「メタファイル」をファイルシステムから検索し、その内容を解析して HTTP レスポンスヘッダーに注入する。

シグネチャまたは形状

static int add_cern_meta_data(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）
  • **出力**: int（Apache HTTP Server のステータスコード。成功時は scan_meta_file の戻り値、メタファイルが不要または存在しない場合は DECLINED、権限エラー時は HTTP_FORBIDDEN）

前提条件と不変条件

  • dconf->metafiles が有効であること。
  • リクエスト対象がディレクトリではなく、ファイルであること。
  • メタファイルは、リクエストされたファイルと同じディレクトリ内の指定されたサブディレクトリ（デフォルトは .web）に、指定されたサフィックス（デフォルトは .meta）を付与した名前で存在する必要がある。

アルゴリズムまたは振る舞い

1. ディレクトリごとの設定 (cern_meta_dir_config) を取得し、メタファイル機能が無効であれば DECLINED を返す。 2. リクエストされたファイルがディレクトリであるか、ファイルシステム上に存在しない場合は DECLINED を返す。 3. リクエストされたファイルのパスを解析し、メタファイルのパスを構築する。 4. ap_sub_req_lookup_file を使用してメタファイルへのアクセス権限を検証する。検証失敗時は DECLINED を返す。 5. apr_file_open でメタファイルを開く。ファイルが存在しない場合は DECLINED を返し、その他のエラー時は HTTP_FORBIDDEN を返す。 6. scan_meta_file を呼び出してメタファイルの内容を解析し、レスポンスヘッダーに反映させる。 7. ファイルハンドルをクローズし、解析結果のステータスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_fixups により fixups フェーズのフックとして登録されている。
  • **呼び出し先**:
  • ap_sub_req_lookup_file (検証用)
  • scan_meta_file (内容解析用)
  • apr_file_open, apr_file_close (ファイル操作)
  • ap_log_rerror (エラーログ出力)

関連データ構造

  • request_rec: リクエスト情報およびメモリプール (r->pool) の管理に使用。
  • cern_meta_dir_config: メタディレクトリ名やサフィックスなどの設定保持。

失敗モードと境界条件

  • **メタファイル不在**: APR_STATUS_IS_ENOENT を検知し、透過的に DECLINED を返す。
  • **権限不足**: メタファイルへのアクセスが拒否された場合、エラーログを出力し HTTP_FORBIDDEN を返す。
  • **パス解析失敗**: ファイルパスにスラッシュが含まれない等の異常系では、エラーログを出力し DECLINED を返す。

保守担当者向けメモ

  • 本関数は fixups フェーズで動作する。
  • メタファイルは Web サーバーからアクセス可能な場所に配置する必要があるため、セキュリティ上の保護（<Location> ディレクティブ等）を検討すること。
  • メモリ管理は r->pool に依存しており、リクエスト終了時に自動解放される。
  • scan_meta_file 内の解析ロジックは ap_scan_script_header_err_core と類似しているが、コードの重複による経年劣化の可能性がある。
