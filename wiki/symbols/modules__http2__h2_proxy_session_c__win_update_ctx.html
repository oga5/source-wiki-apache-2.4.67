win_update_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_session.c:1751

目的

HTTP/2 プロキシセッションにおけるウィンドウ更新処理のコンテキストを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    h2_proxy_session *session;
    conn_rec *c;
    apr_off_t bytes;
    int updated;
} win_update_ctx;

入力と出力

  • **入力**: h2_proxy_session へのポインタ、conn_rec（接続情報）、および更新対象のバイト数（bytes）。
  • **出力**: updated フラグにより、更新が実行されたかどうかの状態を保持します。

前提条件と不変条件

  • session および c は、有効なセッションおよび接続コンテキストを指している必要があります。

アルゴリズムまたは振る舞い

HTTP/2 のフロー制御におけるウィンドウ更新処理において、セッションおよび接続に関連する状態を追跡します。具体的な更新ロジックの詳細は、本構造体を利用する関数群に依存します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数名は、提供されたソース抜粋からは特定できません。

関連データ構造

  • h2_proxy_session: プロキシセッションの管理構造体。
  • conn_rec: Apache HTTP Server の接続管理構造体。

失敗モードと境界条件

  • 提供されたソース抜粋には、失敗モードや境界条件に関する情報は含まれていません。

保守担当者向けメモ

  • 本構造体は modules/http2/h2_proxy_session.c 内のウィンドウ更新処理に関連する断片的な定義です。
  • 構造体の各フィールド（特に bytes や updated）がどのように更新・参照されるかについては、同ファイル内の関連関数を参照してください。
