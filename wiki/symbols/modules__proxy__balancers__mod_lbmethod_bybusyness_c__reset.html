reset

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_bybusyness.c:61

目的

ロードバランサーに紐付く全ワーカーの負荷状態（lbstatus および busy カウント）をゼロに初期化します。

シグネチャまたは形状

apr_status_t reset(proxy_balancer *balancer, server_rec *s)

入力と出力

  • **入力**:
  • balancer: 初期化対象のワーカー群を保持する proxy_balancer 構造体へのポインタ。
  • s: サーバーレコード（本関数内では未使用）。
  • **出力**:
  • APR_SUCCESS: 常に成功として返されます。

前提条件と不変条件

  • balancer->workers が有効な apr_array_header_t であり、内部に proxy_worker 構造体へのポインタが格納されている必要があります。

アルゴリズムまたは振る舞い

1. balancer->workers からワーカーの配列を取得します。 2. 配列内の全ワーカーをループで走査します。 3. 各ワーカーの s->lbstatus（重み付け状態）を 0 に設定します。 4. 各ワーカーの s->busy（現在の処理中リクエスト数）を 0 に設定します。 5. 処理完了後、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述はありません。mod_proxy のロードバランサー管理機能から呼び出されることが想定されます。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • proxy_balancer: ロードバランサーの構成情報を保持する構造体。
  • proxy_worker: 各バックエンドサーバーの状態を保持する構造体。

失敗モードと境界条件

  • balancer->workers が空の場合、ループは実行されず即座に APR_SUCCESS を返します。
  • 共有メモリ上のワーカー情報が破損している場合、本関数によるリセットは期待通りに動作しない可能性があります。

保守担当者向けメモ

  • 本関数は bybusyness アルゴリズムのライフサイクル管理の一部です。
  • ツールによる呼び出し元調査結果（include/ap_mmn.h 等）に列挙されている reset は、本関数とは異なるコンテキストのシンボルであり、本関数を直接呼び出すものではありません。
