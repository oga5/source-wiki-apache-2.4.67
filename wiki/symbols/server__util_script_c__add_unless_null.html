add_unless_null

定義

  • 種別: function
  • 定義箇所: server/util_script.c:88

目的

apr_table_t に対して、キーと値の両方が NULL でない場合にのみエントリを追加する安全なラッパー関数です。

シグネチャまたは形状

static void add_unless_null(apr_table_t *table, const char *name, const char *val)

入力と出力

  • **入力**:
  • table: 追加先の apr_table_t 構造体へのポインタ。
  • name: 追加するエントリのキー名。
  • val: 追加するエントリの値。
  • **出力**: なし（void）。

前提条件と不変条件

  • table は有効な apr_table_t である必要があります。
  • name または val が NULL の場合、テーブルへの追加操作はスキップされます。

アルゴリズムまたは振る舞い

1. name および val が共に非 NULL であるかを確認します。 2. 条件を満たす場合、apr_table_addn を呼び出し、テーブルにキーと値のペアを追加します。 3. 条件を満たさない場合は何もしません。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_script.c 内の以下の箇所から呼び出されています。
  • 環境変数の構築時（http2env の結果を渡すケースなど）
  • REMOTE_HOST, AUTH_TYPE, REDIRECT_URL, REDIRECT_QUERY_STRING などの CGI 環境変数設定時
  • **呼び出し先**: apr_table_addn (APR ライブラリ関数)

関連データ構造

  • apr_table_t: APR が提供するキーと値のペアを保持するテーブル構造体。

失敗モードと境界条件

  • name または val が NULL の場合、関数は静かに終了し、エラーは発生しません。
  • table が NULL の場合、apr_table_addn の内部動作に依存しますが、本関数自体にはガードロジックは含まれていません。

保守担当者向けメモ

  • 本関数は static であり、server/util_script.c 内でのみ使用されます。
  • apr_table_addn を使用しているため、name および val の文字列は、テーブルの生存期間中有効である必要があります（通常はリクエストプール r->pool に割り当てられた文字列が使用されます）。
