set_language_priority

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_negotiation.c:96

目的

Apache 設定ファイル内の LanguagePriority ディレクティブを処理し、指定された言語優先度をディレクトリごとの設定構造体 (neg_dir_config) に登録する。

シグネチャまたは形状

static const char *set_language_priority(cmd_parms *cmd, void *n_, const char *lang)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • n_: ディレクトリ設定構造体 (neg_dir_config) へのポインタ。
  • lang: 設定ファイルから読み込まれた言語タグ文字列。
  • **出力**: 成功時は NULL を返し、エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • n_ は neg_dir_config 型としてキャスト可能であること。
  • cmd->pool は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. n->language_priority が未初期化の場合、cmd->pool を使用して 4 要素分の配列を新規作成する。 2. apr_array_push を使用して、配列の末尾に新しい言語タグ用の領域を確保する。 3. 確保した領域に引数 lang を格納する。 4. NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_negotiation.c:160 において、AP_INIT_ITERATE マクロ経由で設定解析時に呼び出される。
  • **呼び出し先**: APR 配列操作関数 (apr_array_make, apr_array_push) を呼び出す。

関連データ構造

  • neg_dir_config: ディレクトリごとのネゴシエーション設定を保持する構造体。language_priority フィールド（apr_array_header_t）を更新する。

失敗モードと境界条件

  • メモリ不足時には apr_array_make または apr_array_push が内部的にエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は AP_INIT_ITERATE で登録されているため、設定ファイル内で LanguagePriority が複数回指定された場合、その都度呼び出されて配列に言語が追加される。
