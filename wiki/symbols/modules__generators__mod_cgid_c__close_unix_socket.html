close_unix_socket

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgid.c:341

目的

APR プールクリーンアップ処理として、Unix ドメインソケットのファイル記述子を閉じるためのラッパー関数です。

シグネチャまたは形状

apr_status_t close_unix_socket(void *thefd)

入力と出力

  • **入力**: thefd (ファイル記述子を void* にキャストしたもの)
  • **出力**: close() システムコールの戻り値（成功時は 0、失敗時は -1）を apr_status_t 型で返します。

前提条件と不変条件

  • thefd は int 型のファイル記述子を long 型にキャストし、さらに void* にキャストして渡されることを想定しています。
  • 呼び出し元は、この関数がクリーンアップハンドラとして登録されていることを前提としています。

アルゴリズムまたは振る舞い

1. 引数 thefd を long 型を経由して int 型のファイル記述子 fd に復元します。 2. 標準の close(fd) システムコールを呼び出し、ソケットを閉じます。 3. close の結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgid.c 内の各箇所（apr_pool_cleanup_register や apr_pool_cleanup_kill の引数として参照）。
  • **呼び出し先**:
  • close() (システムコール)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • thefd が無効なファイル記述子である場合、close() は -1 を返し、errno が設定されます。

保守担当者向けメモ

  • 本関数は apr_pool_cleanup_register 等のクリーンアップインターフェースに適合させるためのアダプターです。
  • void* からのキャストを行っているため、ポインタサイズと整数サイズの互換性に依存しています。
