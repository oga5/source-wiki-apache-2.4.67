ap_set_byterange

定義

  • 種別: function
  • 定義箇所: modules/http/byterange_filter.c:80

目的

HTTP Range ヘッダーを解析し、レスポンスボディから抽出するバイト範囲のリストを生成します。

シグネチャまたは形状

int ap_set_byterange(request_rec *r, apr_off_t clength,
                     apr_array_header_t **indexes,
                     int *overlaps, int *reversals);

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • clength: コンテンツの総バイト数。
  • **出力**:
  • indexes: 抽出対象の範囲（indexes_t）を格納した配列へのポインタ。
  • overlaps: 重複した範囲の数。
  • reversals: 範囲の逆転が発生した回数。
  • **戻り値**: 抽出する範囲の数。エラー時は 0 または -1（416 Range Not Satisfiable を示す）。

前提条件と不変条件

  • r->assbackwards が真の場合は処理をスキップします。
  • Range ヘッダーが存在しない、または bytes= で始まらない場合は処理をスキップします。
  • 既に Content-Range ヘッダーが付与されている、または multipart/byteranges 形式である場合は処理をスキップします。
  • If-Range ヘッダーの条件に一致しない場合は処理をスキップします。

アルゴリズムまたは振る舞い

1. Range ヘッダーを解析し、カンマ区切りの各範囲を抽出します。 2. 各範囲の開始・終了位置を計算し、clength に基づいてバリデーションを行います。 3. 範囲の重複や包含関係を判定し、必要に応じてマージします。 4. 全ての範囲が clength に対して無効（unsatisfiable）な場合は -1 を返します。 5. 範囲の合計長が clength を超える場合は無視します。 6. 成功時、r->status を HTTP_PARTIAL_CONTENT に設定し、r->range に解析結果を文字列として格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/byterange_filter.c 内のフィルタ処理（行454付近）。
  • **呼び出し先**: ap_getword, ap_parse_strict_length, ap_condition_if_range, apr_array_make, apr_array_push, apr_psprintf, ap_log_rerror 等のユーティリティ関数。

関連データ構造

  • indexes_t: 範囲の開始位置と終了位置を保持する構造体。
  • request_rec: HTTP リクエスト情報を保持する構造体。

失敗モードと境界条件

  • **416 エラー**: 全ての範囲が clength を超えているなど、範囲指定が不適切な場合は -1 を返します。
  • **解析失敗**: 構文的に無効な範囲指定が含まれる場合、ヘッダー全体を無視して 0 を返します。
  • **制限**: MAX_PREALLOC_RANGES を超える範囲指定は切り詰められます。

保守担当者向けメモ

  • 本関数は r->pool を使用してメモリを割り当てます。
  • 0- のような特殊な範囲指定は、他の範囲を無視して単一の範囲として扱われます。
  • 範囲の合計がファイルサイズを超える場合は、安全のため処理を中断します。
