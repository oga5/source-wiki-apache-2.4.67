cgid_server_conf

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_cgid.c:171

目的

mod_cgid モジュールにおけるサーバー単位の設定情報を保持するための構造体です。主に CGI デーモンとの通信やログ出力に関連する設定を管理します。

シグネチャまたは形状

typedef struct {
    const char *logname; /* CGI スクリプトのログファイルパス */
    long logbytes;       /* ログに記録する最大バイト数 */
    int bufbytes;        /* CGI 出力バッファのサイズ */
} cgid_server_conf;

入力と出力

  • **入力**: サーバー起動時または設定読み込み時に、ap_get_module_config を介してサーバー設定から取得されます。
  • **出力**: log_script 関数等でログ出力の判断に使用されるほか、CGI デーモンとの通信におけるバッファ制御に利用されます。

前提条件と不変条件

  • apr_pcalloc を使用してメモリが割り当てられることが想定されています。
  • サーバー設定のライフサイクル（server_rec の生存期間）に依存します。

アルゴリズムまたは振る舞い

  • サーバー設定の初期化時、apr_pcalloc によりゼロ初期化された状態で生成されます。
  • mod_cgid の各ハンドラやデーモン管理関数において、現在のサーバー設定を参照するために引数として渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgid.c 内の各種設定フックおよびリクエスト処理関数（log_script, get_cgi_pid 等）。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行いません。データ保持用の構造体です。

関連データ構造

  • cgid_dirconf: ディレクトリ単位の設定構造体。
  • cgid_req_t: デーモンへ送信されるリクエスト情報構造体。

失敗モードと境界条件

  • 設定が正しく初期化されていない場合、logname が NULL となりログ出力がスキップされる可能性があります。
  • bufbytes の値が不適切な場合、CGI プロセスの出力処理においてバッファリングの挙動に影響を与える可能性があります。

保守担当者向けメモ

  • cgi_server_conf という名前でマクロ定義（#define cgi_server_conf cgid_server_conf）が提供されており、コード内で混在している可能性があるため注意してください。
  • 本構造体は mod_cgid の設定管理の基盤であるため、設定項目を追加する際は mod_cgid の設定マージ関数（merge_cgid_server_config 等）の更新が必要です。
