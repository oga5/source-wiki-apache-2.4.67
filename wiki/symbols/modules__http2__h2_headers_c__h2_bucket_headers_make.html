h2_bucket_headers_make

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_headers.c:59

目的

既存の apr_bucket を H2HEADERS 型のバケットとして再初期化し、HTTP/2 レスポンスヘッダー情報を保持するバケットへと変換します。

シグネチャまたは形状

apr_bucket *h2_bucket_headers_make(apr_bucket *b, h2_headers *r)

入力と出力

  • **入力**:
  • b: 変換対象の apr_bucket ポインタ。
  • r: バケットに格納する h2_headers 構造体へのポインタ。
  • **出力**:
  • 変換後の apr_bucket ポインタを返します。

前提条件と不変条件

  • b は有効な apr_bucket である必要があります。
  • b->list を使用して h2_bucket_headers 構造体のメモリ割り当てが行われます。
  • バケットの長さ（b->length）は 0 に設定されます（メタデータ専用バケットであるため）。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を使用して、h2_bucket_headers 構造体分のメモリを確保します。 2. 確保した構造体の headers メンバに引数 r を代入します。 3. apr_bucket_shared_make を呼び出し、バケットの共有状態を更新します。 4. バケットの型（b->type）を h2_bucket_type_headers に設定します。 5. バケットのデータ長を 0 に設定し、更新されたバケットを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_headers.c:81 (h2_bucket_headers_create 関数内から呼び出されます)
  • **呼び出し先**:
  • apr_bucket_alloc
  • apr_bucket_shared_make

関連データ構造

  • h2_bucket_headers: ヘッダー情報を保持するラッパー構造体。
  • apr_bucket: Apache のバケット管理用基本構造体。
  • h2_headers: HTTP/2 レスポンスヘッダー情報を保持する構造体。
  • h2_bucket_type_headers: H2HEADERS 型バケットの定義（バケットの振る舞いを規定する定数）。

失敗モードと境界条件

  • apr_bucket_alloc がメモリ不足により失敗した場合、APR のメモリ管理ポリシーに従いプロセスが終了するか、例外が発生する可能性があります。

保守担当者向けメモ

  • 本関数は H2HEADERS 型バケットの初期化を担当します。bucket_read は意図的に空（NULL）として定義されているため、このバケットから直接データを読み取ることはできません。
  • AP_HAS_RESPONSE_BUCKETS が定義されている環境では、このロジックは無効化される点に注意してください。
