status_info

定義

  • 種別: struct
  • 定義箇所: modules/md/mod_md_status.c:160

目的

Managed Domains (MD) のステータス表示における各カラムの定義を保持する構造体です。ラベル、対応する JSON キー、および表示用コールバック関数を関連付け、ステータス出力の生成ロジックを抽象化します。

シグネチャまたは形状

struct status_info {
    const char *label;
    const char *key;
    add_status_fn *add;
};

※ add_status_fn は void (status_ctx *ctx, md_json_t *mdj, const status_info *info) 型の関数ポインタです。

入力と出力

  • 入力: ステータス出力処理時に、カラム定義として status_info のインスタンスが参照されます。
  • 出力: add コールバック関数を通じて、status_ctx に基づく HTML テーブル行またはテキスト形式のデータが出力されます。

前提条件と不変条件

  • add コールバック関数は、status_ctx および md_json_t が有効な状態で呼び出される必要があります。
  • label および key は、ステータス表示のライフサイクルを通じて不変であると想定されます。

アルゴリズムまたは振る舞い

status_info は、add_status_cell 関数において、現在の出力モード（HTML/テキスト）に応じた適切なフォーマット関数をディスパッチするためのメタデータとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md_status.c 内のステータス生成ロジック（add_status_cell 等）。
  • 呼び出し先: si_val_status, si_val_url, si_val_valid_time, si_val_cert_valid_time, si_val_ca_urls, si_val_activity, si_val_remote_check などの各値表示用関数が add フィールドに設定されます。

関連データ構造

  • status_ctx: ステータス出力のコンテキストを保持する構造体。
  • md_json_t: ステータス情報のソースとなる JSON データ構造。

失敗モードと境界条件

  • add コールバック関数が NULL の場合、または ctx が不正な場合、実行時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は mod_md_status.c 内の静的なカラム定義配列で使用されます。新しいステータス項目を追加する場合は、この構造体に適合する新しい add 関数を実装し、定義配列に追加してください。
