md_common_name

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:43

目的

2つのマネージドドメイン（md_t）間で共通するドメイン名を1つ特定し、返却します。

シグネチャまたは形状

const char *md_common_name(const md_t *md1, const md_t *md2)

入力と出力

  • **入力**:
  • md1: 比較対象のマネージドドメイン構造体へのポインタ。
  • md2: 比較対象のマネージドドメイン構造体へのポインタ。
  • **出力**:
  • 共通するドメイン名（const char *）。共通するものがない場合、または入力が不正な場合は NULL を返します。

前提条件と不変条件

  • md1 および md2 は有効な md_t 構造体である必要があります。
  • md_t 構造体内の domains 配列は、md_array_str_compact によって正規化されていることが期待されます。

アルゴリズムまたは振る舞い

1. md1 または md2 が NULL、あるいはそれぞれの domains メンバが NULL の場合は NULL を返します。 2. md1 のドメインリストを順次走査します。 3. 各ドメイン名について md_contains(md2, name1, 0) を呼び出し、md2 に含まれているかを確認します。 4. 最初に一致したドメイン名を即座に返します。 5. 全ての走査が終了しても一致が見つからない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_core.c:63
  • modules/md/md_core.c:182
  • modules/md/md_reg.c:395
  • modules/md/mod_md.c:699
  • **呼び出し先**:
  • md_contains（modules/md/md_core.c 内の関数）

関連データ構造

  • md_t: 管理対象ドメインの設定と状態を保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、安全に NULL を返します。
  • domains 配列が空の場合、一致は検出されず NULL を返します。
  • 共通ドメインが複数存在する場合、md1 のリスト順で最初に見つかったものが返されます。

保守担当者向けメモ

  • 本関数は md_contains に依存しており、DNS ワイルドカード等のマッチングロジックは md_contains 側に委譲されています。
  • 戻り値の文字列は md1 構造体が保持するメモリ領域を指しているため、md1 の生存期間に依存します。
