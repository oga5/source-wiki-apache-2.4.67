bucket_destroy

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_eos.c:88

目的

HTTP/2 EOS（End of Stream）バケットが破棄される際に、関連付けられたストリームに対して終了イベントを通知し、リソースのクリーンアップを行う。

シグネチャまたは形状

static void bucket_destroy(void *data)

入力と出力

  • **入力**: void *data - h2_bucket_eos 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • data は h2_bucket_eos 型としてキャスト可能であること。
  • apr_bucket_shared_destroy(h) が真を返す場合のみ、バケットの解放およびイベント通知が実行される。

アルゴリズムまたは振る舞い

1. apr_bucket_shared_destroy を呼び出し、バケットの参照カウントを減らして破棄の可否を確認する。 2. 破棄が許可された場合、以下の処理を行う：

  • ストリームのプールが存在すれば、bucket_cleanup のクリーンアップ登録を apr_pool_cleanup_kill で解除し、二重処理を防止する。
  • apr_bucket_free(h) を呼び出し、バケットメモリを解放する。
  • ストリームが存在する場合、h2_stream_dispatch(stream, H2_SEV_EOS_SENT) を呼び出し、ストリームに対して EOS 送信イベントを発行する。

呼び出し元と呼び出し先

  • **呼び出し元**: apr_bucket の破棄メカニズムから呼び出される（h2_bucket_type_eos 構造体を通じて参照）。
  • **呼び出し先**:
  • apr_bucket_shared_destroy
  • apr_pool_cleanup_kill
  • apr_bucket_free
  • h2_stream_dispatch

関連データ構造

  • h2_bucket_eos: 本関数が操作するバケットのデータ構造。
  • h2_stream: イベント通知先となるストリームオブジェクト。
  • apr_bucket_type_t: 本関数を保持するバケット型定義。

失敗モードと境界条件

  • h->stream が NULL の場合、ストリーム関連のクリーンアップおよびイベント通知はスキップされる。
  • stream->pool が NULL の場合、apr_pool_cleanup_kill は実行されない。

保守担当者向けメモ

  • 本関数は bucket_cleanup と対になる処理であり、二重解放を防ぐために apr_pool_cleanup_kill を使用している。修正時は h2_stream_dispatch の呼び出し順序とストリームの生存期間に注意すること。
