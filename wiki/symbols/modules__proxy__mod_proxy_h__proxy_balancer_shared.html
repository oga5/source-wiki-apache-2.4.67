proxy_balancer_shared

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:527

目的

ロードバランサーの共有メモリ状態を保持するための構造体です。マルチプロセス環境において、複数のプロセス間でバランサーの設定や状態を同期・共有するために使用されます。

シグネチャまたは形状

typedef struct {
    char      sticky_path[PROXY_BALANCER_MAX_STICKY_SIZE];
    char      sticky[PROXY_BALANCER_MAX_STICKY_SIZE];
    char      lbpname[PROXY_MAX_PROVIDER_NAME_SIZE];
    char      nonce[APR_UUID_FORMATTED_LENGTH + 1];
    char      name[PROXY_BALANCER_MAX_NAME_SIZE];
    char      sname[PROXY_BALANCER_MAX_NAME_SIZE];
    char      vpath[PROXY_BALANCER_MAX_ROUTE_SIZE];
    char      vhost[PROXY_BALANCER_MAX_HOSTNAME_SIZE];
    apr_interval_time_t timeout;
    apr_time_t      wupdated;
    int             max_attempts;
    int             index;
    proxy_hashes hash;
    unsigned int    sticky_force:1;
    unsigned int    scolonsep:1;
    unsigned int    max_attempts_set:1;
    unsigned int    was_malloced:1;
    unsigned int    need_reset:1;
    unsigned int    vhosted:1;
    unsigned int    inactive:1;
    unsigned int    forcerecovery:1;
    char      sticky_separator;
    unsigned int    forcerecovery_set:1;
    unsigned int    scolonsep_set:1;
    unsigned int    sticky_force_set:1; 
    unsigned int    nonce_set:1;
    unsigned int    sticky_separator_set:1;
} proxy_balancer_shared;

入力と出力

本構造体は関数ではなくデータ型です。ap_proxy_find_balancershm 関数などを通じて、共有メモリ上のインスタンスが取得されます。

前提条件と不変条件

  • ap_slotmem を介してプロセス間で共有されるため、更新時には proxy_mutex による排他制御が必須です。
  • 構造体内の各フラグ（_set サフィックスを持つもの）は、対応する設定値が明示的に指定されたかどうかを管理します。

アルゴリズムまたは振る舞い

  • 負荷分散アルゴリズム（lbpname）の指定、スティッキーセッションの識別子（sticky）、およびフェイルオーバーの挙動（max_attempts）を保持します。
  • wupdated フィールドにより、ワーカーリストの最終更新時刻を追跡します。

呼び出し元と呼び出し先

  • 呼び出し元: mod_proxy_balancer.c や proxy_util.c 内のバランサー管理ロジックから参照・更新されます。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • proxy_balancer: プロセスローカルなバランサー管理構造体。
  • proxy_worker: バランサー配下のバックエンドサーバー定義。
  • ap_slotmem_provider_t: 共有メモリ領域へのアクセスプロバイダー。

失敗モードと境界条件

  • 共有メモリのサイズ制限（PROXY_BALANCER_MAX_NAME_SIZE 等）を超える文字列設定は切り詰められる可能性があります。
  • 排他制御が適切に行われない場合、マルチプロセス環境で状態の不整合が発生します。

保守担当者向けメモ

  • 本構造体はバイナリ互換性に影響するため、フィールドの追加や変更を行う際は Apache HTTP Server の MMN (Module Magic Number) の更新を検討してください。
  • ALIGNED_PROXY_BALANCER_SHARED_SIZE マクロを使用して、共有メモリ確保時のサイズ計算を行ってください。
