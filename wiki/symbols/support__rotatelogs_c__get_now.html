get_now

定義

  • 種別: function
  • 定義箇所: support/rotatelogs.c:166

目的

ローテーション判定に使用する現在時刻を、設定されたタイムゾーン（UTCまたはローカル時刻）に基づいて取得します。

シグネチャまたは形状

adjusted_time_t get_now(rotate_config_t *config, apr_int32_t *offset)

入力と出力

  • **入力**:
  • config: ローテーション設定を保持する rotate_config_t 構造体へのポインタ。
  • offset: UTCオフセット値を格納するための apr_int32_t へのポインタ（NULL指定可能）。
  • **出力**:
  • 戻り値: 調整済みの現在時刻（秒単位の adjusted_time_t）。
  • *offset: offset が非NULLの場合、計算に使用したUTCオフセット値が書き込まれます。

前提条件と不変条件

  • config->use_localtime が真の場合、サマータイム等の切り替えを考慮し、呼び出しの都度 apr_time_exp_lt を用いて現在のUTCオフセットを再計算します。
  • config->use_localtime が偽の場合、config->utc_offset に保持された固定値を使用します。

アルゴリズムまたは振る舞い

1. apr_time_now() で現在のシステム時刻を取得します。 2. config->use_localtime の設定に基づき、UTCオフセットを決定します。

  • 真の場合: apr_time_exp_lt を呼び出し、現在のローカル時刻からオフセットを取得します。
  • 偽の場合: config->utc_offset を使用します。

    • 3. offset 引数が指定されていれば、決定したオフセット値を格納します。 4. システム時刻（秒単位）にオフセットを加算し、調整後の時刻を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/rotatelogs.c:251
  • support/rotatelogs.c:257
  • support/rotatelogs.c:389
  • support/rotatelogs.c:739
  • **呼び出し先**:
  • apr_time_now()
  • apr_time_exp_lt() (条件付き)
  • apr_time_sec()

関連データ構造

  • rotate_config_t: ローテーション設定を保持。
  • adjusted_time_t: 時刻計算結果を保持する型。

失敗モードと境界条件

  • offset が NULL の場合、オフセット値の書き込みはスキップされます。
  • apr_time_exp_lt の実行結果に依存するため、システム時刻の取得や変換が失敗する環境では不正確な値を返す可能性があります。

保守担当者向けメモ

  • 本関数はローテーションの判定ロジックの基盤となるため、時刻の不連続性（システムクロックの変更等）がローテーション動作に与える影響を考慮してください。
