ssl_hook_process_connection

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:689

目的

SSL接続において、入力フィルタを初期化することでTLSハンドシェイクをトリガーし、SNIやALPNなどの必要な処理を開始させる。

シグネチャまたは形状

static int ssl_hook_process_connection(conn_rec* c)

入力と出力

  • **入力**: conn_rec* c (処理対象の接続構造体)
  • **出力**: int (常に DECLINED を返す)

前提条件と不変条件

  • myConnConfig(c) を通じて取得した SSLConnRec が存在し、かつ disabled フラグが立っていない場合にのみハンドシェイク処理が実行される。

アルゴリズムまたは振る舞い

1. 接続に関連付けられた SSLConnRec を取得する。 2. 接続が有効かつ無効化されていない場合、一時的な apr_bucket_brigade を作成する。 3. ap_get_brigade を AP_MODE_INIT モードで呼び出し、入力フィルタを初期化する。この操作により、TLSハンドシェイクが強制的に実行される。 4. 作成したブリゲードを破棄する。 5. DECLINED を返し、後続のモジュールによる処理を許可する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の process_connection フック（modules/ssl/mod_ssl.c:728 にて ap_hook_process_connection に登録）。
  • **呼び出し先**:
  • myConnConfig (マクロ)
  • apr_brigade_create
  • ap_get_brigade
  • apr_brigade_destroy

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • SSLConnRec: mod_ssl が接続ごとに保持する SSL 関連のコンテキスト情報。
  • apr_bucket_brigade: フィルタチェーン間でデータを渡すためのバッファ構造体。

失敗モードと境界条件

  • sslconn が NULL の場合、または sslconn->disabled が真の場合、ハンドシェイク処理はスキップされる。
  • ap_get_brigade の内部でエラーが発生した場合の戻り値については、本関数内では明示的にハンドリングされていない。

保守担当者向けメモ

  • 本関数はハンドシェイクを強制的に開始させるためのトリガーとして機能する。
  • AP_MODE_INIT を使用した初期化は、接続の早い段階で SNI や ALPN を確定させるために重要である。
  • 戻り値が常に DECLINED であるため、本モジュールが接続処理を完全に占有することはない。
