add_action

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_actions.c:92

目的

Apache の設定ファイルで指定された Action ディレクティブを解析し、特定の MIME タイプとスクリプトパスの紐付けをモジュールの設定構造体に登録します。

シグネチャまたは形状

static const char *add_action(cmd_parms *cmd, void *m_v,
                              const char *type, const char *script,
                              const char *option)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • m_v: action_dir_config 構造体へのポインタ。
  • type: アクションを適用する MIME タイプ。
  • script: 実行するスクリプトのパス。
  • option: オプション文字列（"virtual" が指定可能）。
  • **出力**: 成功時は NULL を返し、エラー時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • m_v は action_dir_config 型としてキャスト可能である必要があります。
  • option が指定された場合、その値は "virtual" である必要があります。それ以外の場合はエラーとなります。

アルゴリズムまたは振る舞い

1. option が指定されている場合、それが "virtual" であるかを確認し、異なればエラーを返します。 2. option の有無に基づき、type をキーとして、"0"（通常）または "1"（virtual）を接頭辞に付与した script パスを m->action_types テーブルに格納します。 3. m->configured フラグを 1 に設定し、設定が有効であることを示します。 4. 正常終了として NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_actions.c:142 において、AP_INIT_TAKE23 マクロを通じて Action ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: Apache APR の文字列操作関数（apr_pstrcat）およびテーブル操作関数（apr_table_setn）を呼び出します。

関連データ構造

  • action_dir_config: ディレクトリごとの設定を保持する構造体。本関数はこの中の action_types テーブルを更新します。

失敗モードと境界条件

  • option に "virtual" 以外の文字列が渡された場合、"unrecognized option '...'" というエラーメッセージを返します。

保守担当者向けメモ

  • 本関数は Action ディレクティブの処理専用です。Script ディレクティブの処理には set_script が使用されます。
  • action_types テーブルに格納される値の先頭文字（0 または 1）は、後のリクエスト処理時に virtual オプションの有無を判定するために利用されます。
