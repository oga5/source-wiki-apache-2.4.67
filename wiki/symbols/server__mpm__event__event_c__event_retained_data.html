event_retained_data

定義

  • 種別: struct
  • 定義箇所: server/mpm/event/event.c:375

目的

Apache HTTP Server のグレースフル再起動（AP_SIG_GRACEFUL）を跨いで、Event MPM のプロセス管理状態や統計情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct event_retained_data {
    ap_unixd_mpm_retained_data *mpm;
    int first_server_limit;
    int first_thread_limit;
    int sick_child_detected;
    int maxclients_reported;
    int near_maxclients_reported;
    int max_daemon_used;
    int total_daemons;
    int active_daemons;
    int *idle_spawn_rate;
    int hold_off_on_exponential_spawning;
} event_retained_data;

入力と出力

  • 本構造体は、再起動時に共有メモリまたは保持されたメモリ領域から復元されます。
  • max_daemon_used や total_daemons 等の統計値は、プロセス管理サイクルを通じて更新・参照されます。

前提条件と不変条件

  • max_daemon_used は、これまでに割り当てられた最大の子プロセススロット数を保持し、スコアボードのスキャン最適化に使用されます。
  • idle_spawn_rate はリスナーバケットごとに管理され、MAX_SPAWN_RATE（32）を上限として指数的に増加します。

アルゴリズムまたは振る舞い

  • グレースフル再起動時、既存のプロセス情報を維持しつつ、新しい設定（MaxRequestWorkers 等）への移行をサポートします。
  • idle_spawn_rate は、アイドル状態のサーバーが不足している場合に子プロセスを生成する際のレートを制御し、生成の必要がないサイクルが続くとリセットされます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の詳細は、提供されたソース抜粋からは特定できません。
  • server/mpm/event/event.c:413 にて static event_retained_data *retained; として静的ポインタが定義されています。

関連データ構造

  • ap_unixd_mpm_retained_data: Unix MPM 共通の保持データ構造。
  • scoreboard: プロセス状態を追跡する共有メモリ構造（max_daemon_used によりスキャン範囲が最適化される）。

失敗モードと境界条件

  • MAX_SPAWN_RATE を超える生成レートの増加は抑制されます。
  • シャットダウンが保留されている間は、total_daemons は最新の状態を反映しない場合があります。

保守担当者向けメモ

  • 本構造体は、再起動を跨ぐ状態保持が必須であるため、メンバーの追加や変更を行う際は、グレースフル再起動時の互換性に注意してください。
  • idle_spawn_rate はポインタとして定義されており、動的な割り当てが行われる点に留意してください。
