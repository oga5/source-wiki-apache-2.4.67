dav_check_bufsize

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:107

目的

dav_buffer 構造体が保持するメモリ領域を、指定された追加サイズ分だけ拡張可能にする。

シグネチャまたは形状

void dav_check_bufsize(apr_pool_t *p, dav_buffer *pbuf, apr_size_t extra_needed)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t。
  • pbuf: 拡張対象の dav_buffer 構造体（入出力）。
  • extra_needed: 現在のデータ長に加えて必要となる追加バイト数。
  • 戻り値: なし。pbuf が指す構造体内部の buf および alloc_len が更新される。

前提条件と不変条件

  • pbuf は初期化済みであること。
  • pbuf->cur_len は現在格納されているデータの長さ、pbuf->alloc_len は確保済みの容量を指す。
  • 拡張が必要な場合、apr_palloc を使用して新しいメモリ領域を確保する。

アルゴリズムまたは振る舞い

1. pbuf->cur_len + extra_needed が現在の pbuf->alloc_len を超えるか判定する。 2. 超える場合、pbuf->alloc_len を extra_needed + DAV_BUFFER_PAD 分だけ増加させる。 3. apr_palloc を用いて新しいバッファ領域を確保する。 4. 既存のデータが存在する場合、memcpy を使用して新しい領域へデータをコピーする。 5. pbuf->buf を新しいバッファ領域で更新する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • modules/dav/fs/dbm.c:617
  • modules/dav/fs/repos.c:1529, 1535
  • modules/dav/main/util.c:157, 168, 177
  • modules/dav/main/util_lock.c:79
  • 呼び出し先:
  • apr_palloc (APR API)
  • memcpy (標準ライブラリ)

関連データ構造

  • dav_buffer: バッファのポインタ、現在の長さ、確保済みサイズを管理する構造体。

失敗モードと境界条件

  • apr_palloc がメモリ不足で失敗した場合、Apache の標準的なメモリ割り当てポリシーに従い、プロセスが終了または例外が発生する可能性がある。
  • extra_needed が 0 の場合、バッファの拡張は行われない。

保守担当者向けメモ

  • DAV_BUFFER_PAD はバッファの断片化を防ぐためのパディング値として機能する。
  • 拡張時には常に新しいメモリ領域を確保するため、頻繁な呼び出しはメモリ消費量に影響を与える可能性がある。
  • 本関数は dav_buffer の内容をコピーするが、古いバッファ領域は apr_pool_t の管理下にあるため、明示的な解放は不要である。
