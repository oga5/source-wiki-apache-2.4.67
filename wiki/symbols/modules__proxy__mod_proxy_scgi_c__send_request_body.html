send_request_body

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:334

目的

クライアントから受け取ったリクエストボディを読み込み、バックエンドサーバーへ転送する。

シグネチャまたは形状

static int send_request_body(request_rec *r, proxy_conn_rec *conn)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • conn: バックエンドへの接続情報を保持する proxy_conn_rec 構造体。
  • **出力**:
  • OK: 正常終了。
  • HTTP_SERVICE_UNAVAILABLE: バックエンドへの送信失敗。
  • HTTP_INTERNAL_SERVER_ERROR: クライアントからのデータ読み込み失敗。

前提条件と不変条件

  • r->pool が有効であり、バッファ割り当てに使用される。
  • conn は既にバックエンドへの接続が確立されている状態であること。

アルゴリズムまたは振る舞い

1. ap_should_client_block(r) を呼び出し、リクエストボディが存在するか確認する。 2. 存在する場合、AP_IOBUFSIZE サイズのバッファを r->pool から確保する。 3. ap_get_client_block を用いてクライアントからデータを読み込む。 4. 読み込んだデータがなくなるまで、sendall を介してバックエンドへ順次送信する。 5. 送信中に sendall が OK 以外を返した場合は HTTP_SERVICE_UNAVAILABLE を返す。 6. 読み込み中にエラー（-1）が発生した場合は、エラーログを出力し HTTP_INTERNAL_SERVER_ERROR を返す。 7. すべての処理が完了した時点で OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_scgi.c:581 (SCGIハンドラ内)
  • **呼び出し先**:
  • ap_should_client_block (Apache API)
  • apr_palloc (APR API)
  • ap_get_client_block (Apache API)
  • sendall (モジュール内関数)
  • ap_log_rerror (Apache API)

関連データ構造

  • request_rec: リクエストの状態管理。
  • proxy_conn_rec: バックエンド接続の管理。

失敗モードと境界条件

  • **送信失敗**: バックエンドとの通信が途絶した場合、HTTP_SERVICE_UNAVAILABLE を返す。
  • **読み込み失敗**: クライアントからのデータ取得が異常終了した場合、HTTP_INTERNAL_SERVER_ERROR を返し、エラーログを記録する。

保守担当者向けメモ

  • 本関数は sendall を介してデータを転送する。sendall の実装詳細については当該関数の定義を参照すること。
  • メモリ割り当てはリクエストプール (r->pool) に依存しており、リクエスト終了時に自動的に解放される。
