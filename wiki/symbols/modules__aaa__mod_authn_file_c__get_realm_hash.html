get_realm_hash

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_file.c:113

目的

テキストファイル（AuthUserFile）から、指定されたユーザー名とレルム（realm）に対応するダイジェスト認証用のハッシュ値を取得します。

シグネチャまたは形状

static authn_status get_realm_hash(request_rec *r, const char *user,
                                   const char *realm, char **rethash)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • user: 認証対象のユーザー名。
  • realm: 認証対象のレルム名。
  • **出力**:
  • rethash: 成功時に見つかったハッシュ値が格納されるポインタ。
  • 戻り値: authn_status 型。成功時は AUTH_USER_FOUND、ユーザーが見つからない場合は AUTH_USER_NOT_FOUND、設定不備やファイル読み込み失敗時は AUTH_GENERAL_ERROR を返します。

前提条件と不変条件

  • r->per_dir_config に authn_file_module の設定が保持されている必要があります。
  • AuthUserFile ディレクティブにより、パスワードファイルへのパスが設定されている必要があります。

アルゴリズムまたは振る舞い

1. 設定からパスワードファイルのパスを取得し、ファイルを開きます。 2. ファイルを1行ずつ読み込み、: を区切り文字として「ユーザー名」「レルム」「ハッシュ値」を抽出します。 3. 指定された user および realm と一致する行を検索します。 4. 一致した行が見つかった場合、そのハッシュ値を rethash に代入し、AUTHN_CACHE_STORE マクロを呼び出してキャッシュに保存します。 5. ファイルを閉じ、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_auth_basic.c および mod_auth_digest.c（認証プロバイダー経由）
  • mod_authn_core.c（プロバイダーエイリアスの解決時）
  • **呼び出し先**:
  • ap_get_module_config, ap_pcfg_openfile, ap_cfg_getline, ap_getword, ap_cfg_closefile, ap_log_rerror, AUTHN_CACHE_STORE

関連データ構造

  • authn_file_config_rec: パスワードファイルのパスを保持するディレクトリごとの設定構造体。
  • request_rec: Apache のリクエスト情報。

失敗モードと境界条件

  • AuthUserFile が未設定の場合: AUTH_GENERAL_ERROR を返し、エラーログを出力します。
  • ファイルが開けない場合: AUTH_GENERAL_ERROR を返し、エラーログを出力します。
  • ユーザーまたはレルムがファイル内に存在しない場合: AUTH_USER_NOT_FOUND を返します。
  • 行のフォーマットが不正な場合: 該当行はスキップされます。

保守担当者向けメモ

  • 本関数は行単位でファイルを走査するため、ユーザー数が多い場合はパフォーマンスが低下します。大規模環境では mod_authn_dbm や mod_authn_dbd への移行を検討してください。
  • ap_getword を使用した文字列抽出は、ファイル内の区切り文字が : であることを前提としています。
