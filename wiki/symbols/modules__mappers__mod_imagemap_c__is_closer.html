is_closer

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_imagemap.c:203

目的

画像マップの point 指令において、指定された座標がこれまでに記録された候補点よりもターゲットに近いかを判定し、最小距離を更新する。

シグネチャまたは形状

static int is_closer(const double point[2], double coords[MAXVERTS][2], double *closest)

入力と出力

  • **入力**:
  • point[2]: 判定対象の座標（X, Y）。
  • coords[MAXVERTS][2]: 比較対象となる座標群（本関数では coords[0] のみを使用）。
  • closest: 現在の最小二乗距離を保持するポインタ。
  • **出力**:
  • int: 判定結果。1 は point が以前の候補より近い（または初回）ことを示し、0 はそうでないことを示す。
  • *closest: 更新された最小二乗距離（1 を返す場合のみ）。

前提条件と不変条件

  • point の各座標値が負である場合、関数は直ちに 0 を返す。
  • *closest が負の値である場合、初回判定として現在の距離を最小値として採用する。

アルゴリズムまたは振る舞い

1. point と coords[0] との間のユークリッド距離の二乗 (dist_squared) を算出する。 2. point の X または Y が負であれば、計算を行わず 0 を返す。 3. *closest が負（初期状態）であるか、算出された dist_squared が現在の *closest より小さい場合：

  • *closest を dist_squared で更新する。
  • 1 を返す。

    • 4. 上記以外の場合、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_imagemap.c:813
  • **呼び出し先**: 本関数は標準的な算術演算のみを行い、他の関数を呼び出さない。

関連データ構造

  • MAXVERTS: coords 配列のサイズを定義する定数。
  • X, Y: 座標配列のインデックスとして使用される定数。

失敗モードと境界条件

  • **負の座標**: point に負の値が含まれる場合、距離判定は行われず 0 が返される。
  • **初期化**: *closest に負の値が渡された場合、最初の比較対象として必ず採用される。

保守担当者向けメモ

  • 本関数は coords 配列の先頭要素 (coords[0]) のみを比較に使用している。
  • 距離の比較には平方根をとらず、二乗値 (dist_squared) を使用することで計算コストを削減している。
