copy_to_linebuf

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:217

目的

現在の行バッファの内容を破棄し、指定された文字列で置き換える（上書きする）ための関数です。

シグネチャまたは形状

static apr_status_t copy_to_linebuf(sed_eval_t *eval, const char* sz, step_vars_storage *step_vars);

入力と出力

  • **入力**:
  • eval: 評価状態を保持する sed_eval_t 構造体へのポインタ。
  • sz: 行バッファにコピーするソース文字列。
  • step_vars: 正規表現処理に関連する変数ストレージ。
  • **出力**:
  • apr_status_t: 操作の結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • eval->linebuf が有効なメモリ領域を指している必要があります。
  • eval->lspend は、行バッファの末尾を指すポインタとして更新されます。

アルゴリズムまたは振る舞い

1. eval->lspend を eval->linebuf（バッファの先頭）にリセットし、既存の行データを論理的にクリアします。 2. append_to_linebuf を呼び出し、指定された文字列 sz を空になった行バッファへコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed1.c:740
  • modules/filters/sed1.c:812
  • modules/filters/sed1.c:823
  • modules/filters/sed1.c:1042
  • **呼び出し先**:
  • append_to_linebuf

関連データ構造

  • sed_eval_t: 評価コンテキスト（行バッファやバッファ末尾ポインタを保持）。
  • step_vars_storage: 正規表現マッチング時の状態保持用構造体。

失敗モードと境界条件

  • append_to_linebuf が内部でバッファ拡張（grow_line_buffer）を行う際、メモリ不足が発生した場合は APR_ENOMEM が返されます。
  • sz が NULL の場合の挙動については、append_to_linebuf の実装に依存します。

保守担当者向けメモ

  • 本関数は eval->lspend を強制的に先頭へ戻すため、呼び出し前の行バッファの内容は完全に破棄されます。
  • バッファ再確保時にポインタが移動する可能性があるため、step_vars を介したオフセット管理が重要です。
