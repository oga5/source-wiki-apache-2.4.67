md_acme_acct_load

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_acct.c:202

目的

指定されたストレージから ACME アカウント情報（JSON）と対応する秘密鍵（PEM）を読み込み、メモリ上の構造体に復元します。

シグネチャまたは形状

apr_status_t md_acme_acct_load(md_acme_acct_t **pacct, md_pkey_t **ppkey,
                               md_store_t *store, md_store_group_t group, 
                               const char *name, apr_pool_t *p)

入力と出力

  • **入力**:
  • store: アカウントが保存されている md_store_t インスタンス。
  • group: ストレージグループ（例: MD_SG_STAGING）。
  • name: アカウント名。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • pacct: 読み込まれた md_acme_acct_t 構造体へのポインタ。
  • ppkey: 読み込まれた md_pkey_t（秘密鍵）へのポインタ。
  • 戻り値: 成功時に APR_SUCCESS、失敗時にエラーコードを返します。

前提条件と不変条件

  • name に対応する account.json および account.pem が store 内の指定 group に存在する必要があります。
  • アカウント ID は ACME-<servername>-<index> の形式で管理されていることが想定されています。

アルゴリズムまたは振る舞い

1. md_store_load_json を呼び出し、アカウントの JSON データを読み込みます。 2. JSON が存在しない場合（APR_ENOENT）、処理を終了します。 3. md_acme_acct_from_json を使用して、JSON を md_acme_acct_t 構造体に変換します。 4. 変換成功後、md_store_load を使用して MD_FN_ACCT_KEY（秘密鍵）を読み込みます。 5. 処理中にエラーが発生した場合、pacct および ppkey を NULL に初期化して戻ります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c (複数箇所)
  • modules/md/md_acme_acct.c (内部)
  • modules/md/md_acme_drive.c (複数箇所)
  • **呼び出し先**:
  • md_store_load_json
  • md_acme_acct_from_json
  • md_store_load
  • md_log_perror

関連データ構造

  • md_acme_acct_t: アカウント情報を保持する主要構造体。
  • md_pkey_t: 秘密鍵を保持する構造体。
  • md_store_t: 永続化ストレージの抽象インターフェース。

失敗モードと境界条件

  • 指定されたアカウント名が見つからない場合、APR_ENOENT を返します。
  • JSON の解析失敗や秘密鍵の読み込み失敗が発生した場合、適切なエラーコードを返し、出力ポインタを NULL に設定します。

保守担当者向けメモ

  • 本関数は md_store を介したファイル I/O に依存しています。
  • 読み込み失敗時のクリーンアップとして、出力ポインタの NULL 初期化が保証されています。
