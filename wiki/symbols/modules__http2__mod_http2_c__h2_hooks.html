h2_hooks

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_http2.c:179

目的

Apache HTTP Server のフックシステムに対し、HTTP/2 モジュールが必要とする各種コールバック関数およびオプション関数を登録する。

シグネチャまたは形状

static void h2_hooks(apr_pool_t *pool)

入力と出力

  • **入力**: apr_pool_t *pool（モジュール初期化用のメモリプール）
  • **出力**: なし（副作用としてフックの登録を行う）

前提条件と不変条件

  • 本関数はモジュールロード時に呼び出される。
  • mod_ssl が存在する場合、post_config フックの実行順序において mod_ssl の後に配置される。

アルゴリズムまたは振る舞い

1. http2_is_h2, http2_var_lookup, http2_get_num_workers をオプション関数として登録。 2. ログに "installing hooks" を出力。 3. ap_hook_post_config に h2_post_config を登録（APR_HOOK_MIDDLE）。 4. ap_hook_child_init に h2_child_init を登録（APR_HOOK_MIDDLE）。 5. 条件付きで ap_hook_child_stopping に h2_c1_child_stopping を登録。 6. 各サブコンポーネント（h2_c1, h2_switch, h2_c2, h2_ws）のフック登録関数を呼び出す。 7. ap_hook_fixups に h2_h2_fixups を登録し、環境変数の解決を可能にする。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c 内のモジュール定義（module 構造体）から参照される。
  • **呼び出し先**:
  • APR_REGISTER_OPTIONAL_FN 関連関数
  • ap_hook_* 関連関数
  • h2_c1_register_hooks, h2_switch_register_hooks, h2_c2_register_hooks, h2_ws_register_hooks

関連データ構造

  • apr_pool_t: メモリ管理用。
  • module 構造体（mod_http2.c 内）: 本関数をフック登録用として保持。

失敗モードと境界条件

  • AP_MODULE_MAGIC_AT_LEAST マクロにより、サーバーのバージョンが古い場合は child_stopping フックの登録がスキップされる。

保守担当者向けメモ

  • 本関数は mod_http2 の初期化の起点となるため、新しいサブコンポーネントを追加した際は、ここで register_hooks を呼び出す必要がある。
  • mod_ssl との依存関係（mod_ssl が先に実行されること）が明示的に指定されている点に注意すること。
