req_aprtable2luatable_cb

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:128

目的

apr_table_t の各要素を Lua のテーブル構造へ変換・コピーするためのコールバック関数です。

シグネチャまたは形状

static int req_aprtable2luatable_cb(void *l, const char *key, const char *value);
  • l: Lua ステート (lua_State *)。スタック上には変換先となる2つのテーブル（複合用と単純用）が配置されている必要があります。
  • key: APR テーブルのキー。
  • value: APR テーブルの値。
  • 戻り値: 処理継続を示す 1 を返します。

入力と出力

  • 入力: Lua スタック上の2つのテーブル（[table<s,t>, table<s,s>]）および APR テーブルのキー・値ペア。
  • 出力: Lua スタック上のテーブルが更新されます。キーが重複する場合、table<s,t> には配列として値が蓄積され、table<s,s> には最初の値のみが保持されます。

前提条件と不変条件

  • Lua スタックのトップに、変換先となる2つのテーブルが配置されていること。
  • l は有効な lua_State であること。

アルゴリズムまたは振る舞い

1. **複合テーブルの更新**: key を用いて table<s,t> を検索します。

  • 未定義の場合: 新規テーブルを作成し、値を配列の1番目に格納します。
  • 既存の場合: 既存の配列の末尾に値を追加します。

    • 2. **単純テーブルの更新**: key を用いて table<s,s> を検索します。

  • 未定義の場合のみ、値を文字列として格納します。

    • 3. 処理完了後、常に 1 を返して apr_table_do による反復処理を継続させます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/lua/lua_request.c:323 (apr_table_do 経由)
  • modules/lua/lua_request.c:360 (apr_table_do 経由)
  • modules/lua/lua_request.c:444 (直接呼び出し)
  • 呼び出し先: Lua C API (lua_getfield, lua_newtable, lua_settable, lua_setfield 等)。

関連データ構造

  • apr_table_t: APR のキー・値ペアを保持する構造体。
  • lua_State: Lua インタプリタの状態。

失敗モードと境界条件

  • Lua スタックの状態が期待と異なる場合、lua_getfield や lua_setfield 等の操作により Lua VM がエラーを発生させる可能性があります。
  • メモリ不足時には Lua のアロケータが例外をスローします。

保守担当者向けメモ

  • 本関数は apr_table_do からのコールバックとして設計されています。
  • Lua スタックのインデックス操作（-1, -2 等）に強く依存しているため、スタック構成を変更する際は注意が必要です。
