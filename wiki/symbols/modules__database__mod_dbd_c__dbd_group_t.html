dbd_group_t

定義

  • 種別: struct
  • 定義箇所: modules/database/mod_dbd.c:54

目的

データベース接続のグループを管理するための構造体です。接続プール（apr_reslist）または単一のデータベース接続を保持し、サーバー設定単位での接続管理を抽象化します。

シグネチャまたは形状

struct dbd_group_t {
    dbd_group_t *next;
    /* その他、接続プールや設定に関連するメンバを含む */
};

※ソースコードの構造体定義行（54行目）および参照箇所（58行目）に基づく。

入力と出力

  • **入力**: サーバー設定（svr_cfg）の初期化時や、dbd_setup / dbd_setup_lock などの関数を通じて設定値やリソースが渡されます。
  • **出力**: データベース接続の管理状態を保持し、必要に応じて接続プールへのアクセスを提供します。

前提条件と不変条件

  • mod_dbd の設定フェーズにおいて、apr_pcalloc を用いてメモリが確保されます。
  • サーバー起動時の dbd_post_config フックにより初期化されることが想定されています。

アルゴリズムまたは振る舞い

  • **接続管理**: 内部で apr_reslist を使用し、スレッド環境下での接続プーリングを実現します。
  • **ライフサイクル**: サーバーのライフサイクルに紐付き、dbd_setup や dbd_setup_lock を介して接続の確立やロック管理が行われます。
  • **連結リスト**: next ポインタにより、複数のグループがリスト構造として管理可能です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/database/mod_dbd.c: create_dbd_config（構造体の割り当て）、dbd_setup、dbd_setup_lock、および設定解析関数群。
  • **呼び出し先**:
  • apr_reslist（接続プーリング機能）
  • apr_pcalloc（メモリ確保）

関連データ構造

  • svr_cfg: 本構造体を保持するサーバー設定コンテナ。
  • ap_dbd_t: データベース接続ハンドル。本グループが管理する接続の実体。

失敗モードと境界条件

  • **メモリ不足**: apr_pcalloc が失敗した場合、サーバーの初期化が中断される可能性があります。
  • **接続確立失敗**: dbd_setup 実行時にデータベースへの接続が確立できない場合、エラー状態が返されます。

保守担当者向けメモ

  • 本構造体は mod_dbd の接続管理の核となるため、変更時は apr_reslist のライフサイクル管理への影響を考慮してください。
  • 構造体の全メンバ定義は modules/database/mod_dbd.c の56行目以降を参照してください。
