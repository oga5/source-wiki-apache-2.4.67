HCONN

定義

  • 種別: typedef (構造体 isapi_cid へのポインタ)
  • 定義箇所: modules/arch/win32/mod_isapi.h:49

目的

Apache HTTP Server 上で動作する ISAPI 拡張に対し、内部的な接続状態（isapi_cid）を不透明なハンドルとして提供するために使用されます。

シグネチャまたは形状

typedef struct isapi_cid *HCONN;

入力と出力

  • **入力**: ISAPI 拡張のコールバック関数（GetServerVariable, WriteClient, ReadClient, ServerSupportFunction など）の第一引数として渡されます。
  • **出力**: 該当しません。

前提条件と不変条件

  • ユーザー（ISAPI 拡張側）に対しては常に不透明（opaque）なポインタとして扱われます。
  • Apache 側のリクエストライフサイクルと密接に結びついており、ISAPI 拡張側で永続化してはなりません。

アルゴリズムまたは振る舞い

  • isapi_cid 構造体へのポインタとして、Apache のリクエスト処理コンテキストを保持します。
  • ISAPI 拡張がサーバー機能を呼び出す際、このハンドルを介して Apache 内部のリクエスト状態を特定します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_isapi.c 内の ISAPI ハンドラが、ISAPI 拡張の各関数を呼び出す際にこのハンドルを渡します。
  • **呼び出し先**: ISAPI 拡張が提供するコールバック関数群（PFN_GETSERVERVARIABLE, PFN_WRITECLIENT, PFN_READCLIENT, PFN_SERVERSUPPORTFUNCTION）の引数として利用されます。
  • **構造体メンバ**: EXTENSION_CONTROL_BLOCK 構造体の ConnID メンバとして保持されます。

関連データ構造

  • isapi_cid: HCONN が指し示す実体。
  • EXTENSION_CONTROL_BLOCK: ISAPI 拡張に渡されるリクエストコンテキストの核となる構造体。

失敗モードと境界条件

  • 接続終了後やリクエスト処理完了後にこのハンドルを保持し続けると、無効なメモリ参照を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは IIS の ISAPI 仕様における HCONN を Apache 上でエミュレートするためのものです。
  • 内部実装である isapi_cid 構造体の定義は mod_isapi.c に依存しているため、変更時は両ファイルの整合性に注意してください。
