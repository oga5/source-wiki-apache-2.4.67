extend_timeout

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reqtimeout.c:74

目的

受信したデータ量に基づいて、接続のタイムアウト期限（timeout_at）を動的に延長します。

シグネチャまたは形状

static void extend_timeout(reqtimeout_con_cfg *ccfg, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • ccfg: 接続ごとの状態を保持する reqtimeout_con_cfg 構造体へのポインタ。
  • bb: 受信データを含む apr_bucket_brigade 構造体へのポインタ。
  • **出力**: なし（ccfg->timeout_at を直接更新）。

前提条件と不変条件

  • bb からデータ長が取得可能であること。
  • ccfg->cur_stage.rate_factor が設定されていること。
  • ccfg->max_timeout_at が 0 より大きい場合、延長後の期限はそれを超えないこと。

アルゴリズムまたは振る舞い

1. apr_brigade_length を用いて bb 内のデータ長 len を算出します。 2. len が 0 以下、または長さの取得に失敗した場合は何もせず終了します。 3. 現在の期限 ccfg->timeout_at に len * ccfg->cur_stage.rate_factor を加算し、新しい期限を算出します。 4. ccfg->max_timeout_at が設定されており、算出した期限がそれを超える場合は、期限を ccfg->max_timeout_at に制限します。 5. それ以外の場合は、算出した期限を ccfg->timeout_at に適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_reqtimeout.c:224
  • modules/filters/mod_reqtimeout.c:258
  • modules/filters/mod_reqtimeout.c:320
  • **呼び出し先**:
  • apr_brigade_length (APR API)

関連データ構造

  • reqtimeout_con_cfg: 接続ごとのタイムアウト状態（期限やレート係数）を保持する構造体。
  • apr_bucket_brigade: フィルタ間で受け渡されるデータバケットのリスト。

失敗モードと境界条件

  • apr_brigade_length が APR_SUCCESS を返さない場合、タイムアウトの延長は行われません。
  • len が 0 以下の場合は延長処理をスキップします。
  • max_timeout_at が 0 の場合は上限チェックを行いません。

保守担当者向けメモ

  • 本関数は AP_MODE_GETLINE 等のデータ読み取り後に呼び出されることを想定しています。
  • タイムアウトの計算式は rate_factor に依存するため、設定変更時の挙動に注意してください。
