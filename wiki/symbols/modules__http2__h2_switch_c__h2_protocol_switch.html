h2_protocol_switch

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_switch.c:154

目的

HTTP/1.1 接続を HTTP/2 セッションへ昇格させるための判断と、接続コンテキストの初期化およびフィルターチェーンの再構成を行う。

シグネチャまたは形状

static int h2_protocol_switch(conn_rec *c, request_rec *r, server_rec *s, const char *protocol)

入力と出力

  • **入力**:
  • c: 現在の接続情報 (conn_rec)
  • r: 現在のリクエスト情報 (request_rec)
  • s: サーバー設定情報 (server_rec)
  • protocol: 切り替え対象のプロトコル文字列
  • **出力**:
  • OK: プロトコル切り替えが成功した場合
  • DECLINED: プロトコルがサポート対象外、または MPM が HTTP/2 をサポートしていない場合
  • !OK: セッションのセットアップに失敗した場合

前提条件と不変条件

  • h2_mpm_supported() が真であること。
  • 指定された protocol が、SSL/TLS の有無に応じて h2_protocol_ids_tls または h2_protocol_ids_clear に含まれていること。

アルゴリズムまたは振る舞い

1. **プロトコル検証**: 接続の SSL 状態に基づき、サポートされているプロトコルリストと引数の protocol を照合する。 2. **コンテキスト作成**: h2_conn_ctx_create_for_c1 を呼び出し、接続用の HTTP/2 コンテキストを生成する。 3. **フィルター再構成**: リクエスト r が存在する場合、既存の HTTP/1.1 フィルター（http_in, HTTP_HEADER 等）を削除し、HTTP/2 用のスタックへ置き換える。 4. **セッション起動**: h2_c1_setup でセッションを初期化し、成功すれば h2_c1_run を呼び出して HTTP/2 セッションのメインループへ制御を移す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_switch.c:231 (Apache フック ap_hook_protocol_switch 経由で登録)
  • **呼び出し先**:
  • h2_mpm_supported()
  • h2_conn_ctx_create_for_c1()
  • remove_input_filters_below() / remove_output_filters_below() (または ap_remove_... 系関数)
  • h2_c1_setup()
  • h2_c1_run()
  • h2_conn_ctx_detach()

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • request_rec: Apache のリクエスト状態を保持する構造体。
  • h2_conn_ctx: 接続ごとの HTTP/2 コンテキスト管理用データ構造。

失敗モードと境界条件

  • **MPM 非対応**: h2_mpm_supported() が偽の場合、即座に DECLINED を返す。
  • **プロトコル不一致**: サポートリストにないプロトコルが指定された場合、DECLINED を返す。
  • **セットアップ失敗**: h2_c1_setup が APR_SUCCESS 以外を返した場合、コンテキストをデタッチし !OK を返す。

保守担当者向けメモ

  • 本関数は Apache のフィルターシステムを直接操作するため、AP_FTYPE_CONNECTION 以下のフィルター削除処理は非常に繊細です。
  • AP_HAS_RESPONSE_BUCKETS の定義有無により、フィルター削除のロジックが分岐するため、環境ごとの挙動に注意してください。
  • HTTP/2 の詳細なフレーム処理は h2_session.c 等に委譲されており、本関数はあくまで「切り替え」の境界処理を担当します。
