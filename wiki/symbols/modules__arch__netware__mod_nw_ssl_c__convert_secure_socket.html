convert_secure_socket

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:329

目的

既存の接続ソケットに対して、NetWare ネイティブの SSL/TLS 機能を有効化し、暗号化通信へ移行させる。

シグネチャまたは形状

static int convert_secure_socket(conn_rec *c, apr_socket_t *csd)

入力と出力

  • **入力**:
  • c: 現在の接続を表す conn_rec 構造体。
  • csd: SSL を適用する apr_socket_t 型のソケット。
  • **出力**:
  • int: WSAIoctl の実行結果（成功時は 0、失敗時は SOCKET_ERROR）。

前提条件と不変条件

  • numcerts（グローバル変数）の状態に基づき、証明書リストを使用するか、KMO（Key Management Object）のデフォルト証明書を使用するかが決定される。
  • apr_os_sock_get を介して、APR ソケットからネイティブな SOCKET ハンドルが取得可能であること。

アルゴリズムまたは振る舞い

1. apr_os_sock_get でネイティブソケットを取得。 2. WSAIoctl を呼び出し、SO_TLS_SET_FLAGS で SO_TLS_ENABLE（証明書がない場合は SO_TLS_BLIND_ACCEPT を併用）を設定。 3. SO_TLS_UNCLEAN_SHUTDOWN フラグを設定。 4. numcerts の有無に応じて tlsclientopts 構造体を構築：

  • numcerts > 0: certarray を使用し、WAL_PROV_DER プロバイダを指定。
  • numcerts == 0: SSL CertificateIP を使用し、WAL_PROV_KMO プロバイダを指定。

    • 5. WSAIoctl の SO_TLS_SET_CLIENT を呼び出し、SSL 設定をソケットに適用。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_nw_ssl.c:717（直接的な呼び出しを確認）。
  • **呼び出し先**: WSAIoctl（WinSock API）、apr_os_sock_get（APR API）、memset、unicpy、unilen。

関連データ構造

  • conn_rec: Apache の接続情報。
  • tlsclientopts, nwtlsopts, sslserveropts: NetWare ネイティブの SSL 設定構造体。

失敗モードと境界条件

  • WSAIoctl が SOCKET_ERROR を返した場合、WSAGetLastError() を用いてエラーログを出力し、そのエラーコードを呼び出し元へ返却する。
  • 証明書リストが空の場合、デフォルトの KMO 証明書へのフォールバックが行われる。

保守担当者向けメモ

  • 本関数は NetWare の WinSock 拡張に強く依存している。
  • numcerts はモジュール内で管理されるグローバルな証明書数である。
  • SO_TLS_BLIND_ACCEPT の使用は、証明書が提供されない環境での接続を許可するが、セキュリティ要件に応じて注意が必要である。
