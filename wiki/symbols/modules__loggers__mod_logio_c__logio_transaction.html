logio_transaction

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_logio.c:139

目的

HTTPリクエスト終了時に、接続ごとの入出力バイト数カウンタをリセットし、次回の計測に備える。

シグネチャまたは形状

static int logio_transaction(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在処理中のリクエスト情報)
  • **出力**: int (常に OK を返す)

前提条件と不変条件

  • r->connection->conn_config から logio_module に関連付けられた logio_config_t 構造体が取得可能であること。
  • mod_log_config の後に実行されるようフック順序が制御されていること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在の接続に関連付けられた logio_config_t 構造体を取得する。 2. 直前のリクエストの合計バイト数（入力と出力の和）を cf->bytes_last_request に保存する。 3. cf->bytes_in および cf->bytes_out を 0 にリセットする。 4. OK を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_log_transaction を介して Apache のログ記録フェーズから呼び出される (modules/loggers/mod_logio.c:261)。
  • **呼び出し先**: ap_get_module_config (Apache API) を呼び出す。

関連データ構造

  • logio_config_t: 接続ごとの入出力統計を保持する構造体。
  • request_rec: リクエストごとの状態を保持する Apache コア構造体。

失敗モードと境界条件

  • 接続設定が取得できない場合、ap_get_module_config は NULL を返す可能性があるが、本関数内での NULL チェックは明示されていない。

保守担当者向けメモ

  • 本関数はリクエスト終了時に実行されるため、接続の生存期間中に複数回呼び出される可能性がある。
  • bytes_last_request は、ap_increment_counts 等の統計処理で利用されるための値を保持する。
