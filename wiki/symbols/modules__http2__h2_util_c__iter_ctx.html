iter_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.c:248

目的

ハッシュテーブルの反復処理（イテレーション）において、反復関数ポインタとユーザー定義のコンテキストデータを保持するためのラッパー構造体です。

シグネチャまたは形状

typedef struct {
    h2_ihash_iter_t *iter;
    void *ctx;
} iter_ctx;

入力と出力

  • iter: 反復処理時に呼び出されるコールバック関数へのポインタ。
  • ctx: 反復処理中にコールバック関数へ渡される任意のユーザーデータ。

前提条件と不変条件

  • iter は h2_ihash_iter_t 型の関数ポインタである必要があります。
  • ctx は void * 型であり、呼び出し側のコンテキストに応じて適切な型にキャストして使用されます。

アルゴリズムまたは振る舞い

ハッシュテーブルの走査を行う際、この構造体を通じてコールバック関数とそれに付随する状態（コンテキスト）をカプセル化します。反復処理の実行中、この構造体から iter 関数が取り出され、ctx を引数として渡すことで、ハッシュテーブルの各要素に対する処理が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_util.c および modules/http2/h2_proxy_util.c 内の関数が、ハッシュテーブルの反復処理を行う際にこの構造体をインスタンス化し、利用しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。保持されている iter 関数ポインタが、反復処理の過程で実行されます。

関連データ構造

  • h2_ihash_t: 本構造体が反復処理の対象とするハッシュテーブル。
  • h2_ihash_iter_t: iter メンバーが指す関数ポインタの型定義。

失敗モードと境界条件

  • iter が NULL の場合、反復処理中にデリファレンスが発生する可能性があります。
  • ctx が指すメモリ領域が、反復処理の完了前に解放された場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は h2_util.c と h2_proxy_util.c の両方で利用されています。変更を加える際は、両ファイルでの整合性に注意してください。
  • h2_ihash の反復処理ロジックと密結合しているため、ハッシュテーブルの走査仕様に変更がある場合は本構造体の利用箇所を確認してください。
