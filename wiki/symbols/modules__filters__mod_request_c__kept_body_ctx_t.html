kept_body_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_request.c:165

目的

サブリクエストへリクエストボディを再挿入する際、保持されたボディデータの読み出し位置と残量を管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct kept_body_filter_ctx {
    apr_off_t offset;
    apr_off_t remaining;
} kept_body_ctx_t;

入力と出力

  • offset: 現在の読み出し開始位置（オフセット）。
  • remaining: 読み出し可能な残りのデータサイズ。

前提条件と不変条件

  • 本構造体は、r->kept_body に保持された apr_bucket_brigade を対象として動作します。
  • kept_body_filter が呼び出される際、f->ctx に本構造体のインスタンスが格納されている必要があります。

アルゴリズムまたは振る舞い

1. サブリクエストの開始時に kept_body_filter_init によって初期化されます。 2. kept_body_filter が呼び出されると、ctx を参照して r->kept_body からデータをパーティション分割し、後続のフィルタへ渡します。 3. 読み出しが進むにつれ、offset が更新され、remaining が減少します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_request.c 内の kept_body_filter 関数が f->ctx を通じて本構造体を参照しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • apr_bucket_brigade: 保持されたリクエストボディの実体。
  • request_rec: リクエスト情報を保持する構造体（r->kept_body を含む）。
  • keep_body_ctx_t: リクエストボディを保存する側のコンテキスト構造体。

失敗モードと境界条件

  • remaining が 0 に達した場合、データの読み出しは終了します。
  • r->kept_body が存在しない状態で本構造体が参照された場合の挙動は、kept_body_filter の実装に依存します。

保守担当者向けメモ

  • 本構造体は kept_body_filter の状態管理専用です。
  • apr_off_t を使用しているため、非常に大きなリクエストボディに対してもオフセット計算が可能です。
