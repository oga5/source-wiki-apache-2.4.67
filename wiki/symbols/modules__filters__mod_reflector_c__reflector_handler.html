reflector_handler

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reflector.c:46

目的

受信した POST リクエストのボディをそのままレスポンスとしてクライアントに送り返す（リフレクションする）ためのハンドラ。

シグネチャまたは形状

static int reflector_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r（Apache リクエスト構造体）
  • **出力**: int（HTTP ステータスコードまたは DECLINED）

前提条件と不変条件

  • リクエストのハンドラ名が "reflector" であること。
  • POST メソッドのみをサポートする（OPTIONS メソッドは ap_send_http_options で処理される）。
  • サブリクエストや接続が閉じている場合は処理を行わず OK を返す。

アルゴリズムまたは振る舞い

1. **メソッド検証**: POST または OPTIONS 以外は HTTP_METHOD_NOT_ALLOWED を返す。 2. **ヘッダー転送**: 設定に基づき header_do を呼び出し、リクエストヘッダーをレスポンスヘッダーへコピーする。 3. **メタデータ設定**: Last-Modified、Accept-Ranges、Content-Length、Content-Type をリクエストから引き継ぐ。 4. **ストリーム転送**:

  • ap_get_brigade で入力フィルタからデータを読み取る。
  • EOS (End of Stream) バケットを検出するまで、読み取ったデータを bbout ブリゲードへ書き込み、ap_pass_brigade で出力フィルタへ渡すループを実行する。

    • 5. **終了**: 正常終了時に OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のハンドラフック（ap_hook_handler により reflector_hooks で登録）。
  • **呼び出し先**:
  • header_do（ヘッダーコピー用）
  • ap_get_brigade, ap_pass_brigade（ストリーム処理用）
  • ap_parse_strict_length, ap_set_content_length 等の Apache API

関連データ構造

  • reflector_cfg: ディレクトリ単位の設定（ヘッダーマッピングルール等）を保持。
  • apr_bucket_brigade: 入出力データのバッファリングに使用。

失敗モードと境界条件

  • Content-Length が不正な場合: HTTP_BAD_REQUEST を返す。
  • ap_get_brigade または ap_pass_brigade が失敗した場合: エラーログを出力し、適切なエラーコード（HTTP_BAD_REQUEST または AP_FILTER_ERROR）を返す。
  • POST 以外のメソッド: HTTP_METHOD_NOT_ALLOWED を返す。

保守担当者向けメモ

  • 本関数は POST リクエストのボディをメモリ経由で転送するストリーミング実装です。
  • bucket->length == 0 のメタデータバケットはスキップされます。
  • ap_pass_brigade の戻り値は AP_FILTER_ERROR として処理されます。
