ap_scan_script_header_err

定義

  • 種別: function
  • 定義箇所: server/util_script.c:771

目的

ファイルベースのスクリプト出力から HTTP ヘッダーを解析し、リクエスト構造体へ反映させるためのラッパー関数です。

シグネチャまたは形状

int ap_scan_script_header_err(request_rec *r, apr_file_t *f, char *buffer);

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • f: スクリプトの出力を読み取るための apr_file_t ファイルポインタ。
  • buffer: ヘッダー行を読み取るために使用されるバッファ領域。
  • **出力**:
  • 戻り値: 解析結果を示す整数値（成功時は OK 等、失敗時はエラーコード）。

前提条件と不変条件

  • r および f は有効なポインタである必要があります。
  • buffer はヘッダー行を格納するのに十分なサイズを確保している必要があります。
  • すべてのメモリ割り当ては r->pool を介して行われ、リクエスト終了時に自動的に解放されます。

アルゴリズムまたは振る舞い

1. getsfunc_FILE をリーダー関数として指定します。 2. ap_scan_script_header_err_core_ex を呼び出し、ファイル f からスクリプト出力を 1 行ずつ読み取ります。 3. 空行（\0）に到達するまでヘッダーを解析し、Status や Location などの特殊ヘッダーを処理します。 4. 正常に解析されたヘッダーは r->err_headers_out にマージされます。

呼び出し元と呼び出し先

  • **呼び出し先**: ap_scan_script_header_err_core_ex および getsfunc_FILE。
  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供された情報には含まれていませんが、include/util_script.h で公開 API として宣言されています。

関連データ構造

  • request_rec: リクエストの状態管理に使用。
  • apr_file_t: スクリプト出力の I/O ストリームとして使用。

失敗モードと境界条件

  • スクリプト出力が不正な形式である場合や、解析中にタイムアウトが発生した場合はエラーを返します。
  • スクリプトの出力が完了するまでブロックする可能性があるため、非同期 I/O が必要な環境では注意が必要です。
  • APR_CHARSET_EBCDIC が定義されている環境では、ASCII との変換ロジックがヘッダー解析に介入します。

保守担当者向けメモ

  • 本関数は ap_scan_script_header_err_core_ex の単なるラッパーです。ロジックの変更はコア関数側で行うことが推奨されます。
  • getsfunc_FILE はファイルポインタから行を抽出するためのヘルパー関数であり、本関数と対で使用されます。
