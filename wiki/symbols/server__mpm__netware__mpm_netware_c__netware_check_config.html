netware_check_config

定義

  • 種別: function
  • 定義箇所: server/mpm/netware/mpm_netware.c:1011

目的

Apache HTTP Server の起動および再起動時に、MPM 固有の設定値（スレッド数関連）が妥当な範囲内にあるかを確認・修正する。

シグネチャまたは形状

static int netware_check_config(apr_pool_t *p, apr_pool_t *plog,
                                apr_pool_t *ptemp, server_rec *s)

入力と出力

  • 入力:
  • p, plog, ptemp: APR メモリプール。
  • s: 現在のサーバー設定レコード。
  • 出力:
  • 常に OK を返す。
  • 副作用として、グローバル変数 ap_threads_limit, ap_threads_to_start, ap_threads_min_free を妥当な値に修正し、警告ログを出力する。

前提条件と不変条件

  • サーバーの起動時（初回）と再起動時でログ出力の形式（APLOG_STARTUP フラグの有無）が切り替わる。
  • HARD_THREAD_LIMIT がコンパイル時に定義されている必要がある。

アルゴリズムまたは振る舞い

1. 実行回数をカウントし、初回起動時かどうかを判定する。 2. ap_threads_limit が HARD_THREAD_LIMIT を超える場合、上限値に切り詰め、警告を記録する。 3. ap_threads_limit が 1 未満の場合、1 に引き上げ、警告を記録する。 4. ap_threads_to_start が 0 未満の場合、1 に引き上げ、警告を記録する。 5. ap_threads_min_free が 1 未満の場合、1 に引き上げ、警告を記録する。

呼び出し元と呼び出し先

  • 呼び出し元: ap_hook_check_config を介して Apache の設定確認フェーズで呼び出される（server/mpm/netware/mpm_netware.c:1095）。
  • 呼び出し先: ap_log_error（エラーログ出力用）。

関連データ構造

  • server_rec: サーバー設定情報。
  • ap_threads_limit, ap_threads_to_start, ap_threads_min_free: MPM のスレッド管理用グローバル変数。

失敗モードと境界条件

  • 設定値が許容範囲外であっても、関数自体は失敗せず、値を強制的に修正して OK を返す。
  • startup フラグにより、サーバー起動時のコンソール出力と、実行中のログファイルへの出力を適切に使い分ける。

保守担当者向けメモ

  • 本関数は設定の「修正」を行うため、ユーザーが誤った設定値を指定してもサーバーが起動できるように設計されている。
  • ap_threads_max_free に関するチェックは本関数内ではなく ap_mpm_run() 内で行われるため、ここには含まれていない。
