store_slotmem

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:153

目的

共有メモリ上のスロット管理インスタンスの内容を、整合性検証用のMD5ハッシュと共にファイルへ永続化する。

シグネチャまたは形状

static void store_slotmem(ap_slotmem_instance_t *slotmem)

入力と出力

  • **入力**: ap_slotmem_instance_t *slotmem（永続化対象の共有メモリインスタンス）。
  • **出力**: なし（ファイルシステムへの書き込みによる副作用）。

前提条件と不変条件

  • slotmem->pname に有効なファイルパスが設定されていること。
  • 共有メモリ領域が初期化済みであること。
  • AP_SLOTMEM_IS_CLEARINUSE(slotmem) が真の場合、書き出し前に slotmem_clearinuse が実行される。

アルゴリズムまたは振る舞い

1. slotmem->pname で指定されたパスにファイルを開く（存在する場合は削除して再作成）。 2. 必要に応じて slotmem_clearinuse を呼び出し、利用状況フラグをクリアする。 3. データ領域（persist）のサイズを計算し、apr_md5 を用いてハッシュ値を算出する。 4. ファイルへ順に「データ領域」「MD5ハッシュ」「記述子（desc）」を書き込む。 5. 書き込み失敗時は、作成したファイルを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/slotmem/mod_slotmem_shm.c:301（cleanup_slotmem 内のループ処理から参照）。
  • **呼び出し先**: apr_file_open, apr_file_remove, slotmem_clearinuse, apr_md5, apr_file_write_full, apr_file_close。

関連データ構造

  • ap_slotmem_instance_t: インスタンスの管理構造体。
  • sharedslotdesc_t: スロットの構成情報。

失敗モードと境界条件

  • ファイルオープン失敗時や書き込み失敗時は、処理を中断する。
  • ファイルが既に存在する場合、一度削除してから再作成を試みる。
  • storename が NULL の場合は何も行わない。

保守担当者向けメモ

  • 本関数はサーバー終了時のクリーンアップ処理の一部として呼び出される。
  • 書き込み順序（データ→ハッシュ→記述子）は restore_slotmem と対になっているため、変更時は整合性に注意すること。
