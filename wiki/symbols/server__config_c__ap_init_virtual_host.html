ap_init_virtual_host

定義

  • 種別: function
  • 定義箇所: server/config.c:2083

目的

仮想ホスト用の server_rec 構造体を初期化し、指定されたホスト名に基づいてアドレス情報を解析・設定する。

シグネチャまたは形状

const char *ap_init_virtual_host(apr_pool_t *p,
                                 const char *hostname,
                                 server_rec *main_server,
                                 server_rec **ps)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • hostname: 仮想ホストのホスト名。
  • main_server: テンプレートとして使用するメインサーバーの server_rec。
  • ps: 初期化された server_rec を格納するためのポインタのポインタ。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ文字列を返す。
  • *ps に初期化済みの server_rec 構造体が格納される。

前提条件と不変条件

  • main_server は有効な server_rec 構造体を指していること。
  • p は有効なメモリプールであること。
  • 生成される server_rec は is_virtual フラグが 1 に設定される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して server_rec 構造体をゼロ初期化し、メモリを確保する。 2. main_server からプロセス情報、エラーログ、ポート番号、リクエスト制限値などの基本設定をコピーする。 3. 仮想ホスト用の名前リスト（names, wild_names）を apr_array_make で初期化する。 4. create_empty_config および ap_create_per_dir_config を呼び出し、モジュール設定とディレクトリ設定を初期化する。 5. ap_parse_vhost_addrs を呼び出し、ホスト名に関連付けられたアドレス情報を解析・登録する。 6. 最終的に初期化された構造体を *ps に代入し、解析結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:2969
  • **呼び出し先**: ap_parse_vhost_addrs, apr_pcalloc, apr_array_make, create_empty_config, ap_create_per_dir_config

関連データ構造

  • server_rec: 仮想ホストの構成情報を保持する主要な構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • apr_pcalloc や apr_array_make がメモリ不足で失敗した場合の挙動は、APR の標準的なエラー処理に依存する。
  • ap_parse_vhost_addrs がエラーを返した場合、そのエラーメッセージがそのまま本関数の戻り値となる。

保守担当者向けメモ

  • ソース内のコメントにある通り、本関数のロジックの一部は本来 http_core に属するべき内容である。
  • server_rec の初期化において、main_server からのコピー漏れがないか注意すること。
