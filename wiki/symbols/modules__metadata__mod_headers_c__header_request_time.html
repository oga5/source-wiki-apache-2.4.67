header_request_time

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_headers.c:175

目的

HTTPリクエストの受信時刻（request_time）を文字列形式に変換し、ヘッダー値として利用可能な形式で提供する。

シグネチャまたは形状

static const char *header_request_time(request_rec *r, char *a)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報を保持する request_rec 構造体へのポインタ。
  • a: 未使用の引数（char *）。
  • **出力**:
  • apr_psprintf によって生成された "t=<time>" 形式の文字列を返す。メモリは r->pool から割り当てられる。

前提条件と不変条件

  • r が有効な request_rec 構造体を指していること。
  • r->request_time に有効なリクエスト開始時刻が格納されていること。

アルゴリズムまたは振る舞い

1. r->request_time からリクエスト時刻を取得する。 2. apr_psprintf を使用し、t= というプレフィックスを付与した文字列を生成する。 3. 生成された文字列を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_headers.c:980 にて、register_format_tag_handler("t", header_request_time) を通じてフォーマットタグハンドラとして登録されている。
  • **呼び出し先**: apr_psprintf（APRライブラリ関数）。

関連データ構造

  • request_rec: Apacheのリクエスト情報を保持する主要な構造体。

失敗モードと境界条件

  • メモリ不足時には apr_psprintf が異常終了する可能性がある（APRのメモリ管理ポリシーに依存）。

保守担当者向けメモ

  • 本関数は mod_headers におけるフォーマットタグ %t の実装である。
  • a 引数は現在の実装では使用されていないが、フォーマットタグハンドラのシグネチャに合わせるために保持されている。
