add_options

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ext_filter.c:127

目的

ExtFilterOptions ディレクティブで指定された設定引数を解析し、ディレクトリ単位の設定構造体 (ef_dir_t) に反映させる。

シグネチャまたは形状

static const char *add_options(cmd_parms *cmd, void *in_dc, const char *arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • in_dc: 設定対象の ef_dir_t 構造体（void* として渡される）。
  • arg: 解析対象のオプション文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • in_dc は ef_dir_t 型のポインタとしてキャスト可能であること。
  • arg は大文字小文字を区別しない比較（strcasecmp）が行われる。

アルゴリズムまたは振る舞い

1. arg の値に応じて以下のフラグを dc に設定する:

  • "LogStderr": dc->log_stderr を 1 に設定。
  • "NoLogStderr": dc->log_stderr を 0 に設定。
  • "Onfail=remove": dc->onfail を 1 に設定。
  • "Onfail=abort": dc->onfail を 0 に設定。

    • 2. 上記のいずれにも一致しない場合、cmd->temp_pool を使用してエラーメッセージを生成し、返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_ext_filter.c:339（command_rec 構造体経由でディレクティブ解析時に呼び出される）。
  • **呼び出し先**: apr_pstrcat（エラーメッセージ生成時）。

関連データ構造

  • ef_dir_t: ディレクトリ単位の設定を保持する構造体。
  • cmd_parms: Apache のコマンド解析用パラメータ構造体。

失敗モードと境界条件

  • 未知のオプション文字列が渡された場合、apr_pstrcat を用いてエラーメッセージを返し、設定の適用を拒否する。

保守担当者向けメモ

  • 本関数は ExtFilterOptions ディレクティブのハンドラとして機能する。新しいオプションを追加する場合は、strcasecmp の条件分岐を拡張すること。
  • エラーメッセージのメモリ確保には cmd->temp_pool が使用されており、リクエストや設定読み込みのライフサイクルに依存している。
