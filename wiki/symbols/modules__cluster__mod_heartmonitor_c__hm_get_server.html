hm_get_server

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartmonitor.c:511

目的

指定された IP アドレスとポート番号に対応するサーバー状態構造体（hm_server_t）を取得、または存在しない場合は新規作成して返します。

シグネチャまたは形状

static hm_server_t *hm_get_server(hm_ctx_t *ctx, const char *ip, const int port)

入力と出力

  • **入力**:
  • ctx: モジュールの実行時状態を保持する hm_ctx_t 構造体へのポインタ。
  • ip: サーバーの IP アドレスを示す文字列。
  • port: サーバーのポート番号。
  • **出力**:
  • 該当する hm_server_t 構造体へのポインタ。

前提条件と不変条件

  • ctx->servers は初期化済みのハッシュテーブルである必要があります。
  • 新規作成される hm_server_t は ctx->p プールからメモリが割り当てられます。
  • 戻り値の hm_server_t は、ip、port、ready、busy、seen の各フィールドが初期化された状態であることが保証されます。

アルゴリズムまたは振る舞い

1. ctx->servers ハッシュテーブルから、ip をキーとして既存の hm_server_t を検索します。 2. 該当するエントリが存在しない場合:

  • ctx->p プールから hm_server_t 用のメモリを確保します。
  • ip を複製し、port、ready、busy、seen を初期化します。
  • 新規作成した構造体を ctx->servers ハッシュテーブルに登録します。

    • 3. 取得または作成した構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cluster/mod_heartmonitor.c:558 (直接の呼び出しを確認)
  • modules/cluster/mod_heartmonitor.c:771 (コメント内で言及)
  • **呼び出し先**:
  • apr_hash_get / apr_hash_set (APR ハッシュテーブル操作)
  • apr_palloc / apr_pstrdup (APR メモリプール操作)

関連データ構造

  • hm_ctx_t: モジュールの設定とサーバーハッシュテーブルを保持。
  • hm_server_t: 個々のバックエンドサーバーの稼働状態を保持。

失敗モードと境界条件

  • ctx が NULL の場合、または ctx->servers が初期化されていない場合はセグメンテーション違反が発生する可能性があります。
  • メモリ不足時には apr_palloc が例外を発生させる可能性があります。

保守担当者向けメモ

  • 本関数で生成されるサーバー情報は ctx->p プールで管理されています。
  • hm_processmsg 等のメッセージ処理ロジックから頻繁に呼び出されるため、ハッシュテーブルの検索効率がパフォーマンスに直結します。
