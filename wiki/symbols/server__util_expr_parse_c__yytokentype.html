yytokentype

目的

Apache HTTP Server の式評価エンジン（ap_expr）において、構文解析器が扱うトークンの種類を定義する列挙型です。

シグネチャまたは形状

enum yytokentype {
    T_TRUE = 258,
    T_FALSE = 259,
    T_EXPR_BOOL = 260,
    // ... (中略)
    T_OP_NOT = 293
};

各トークンには 258 から 293 までの整数値が割り当てられています。

入力と出力

  • 入力: 字句解析器（ap_expr_yylex）によって生成されるトークン種別。
  • 出力: 構文解析器（ap_expr_yyparse）の内部状態遷移および還元処理における識別子として利用されます。

前提条件と不変条件

  • GNU Bison 2.5 によって生成されたコードの一部であり、util_expr_parse.y の文法定義と整合している必要があります。
  • 各列挙値は一意であり、パーサーのスタック操作においてトークンを識別するために使用されます。

アルゴリズムまたは振る舞い

  • 構文解析器が入力ストリームからトークンを読み取る際、この列挙型を使用して演算子（T_OP_EQ 等）、変数（T_VAR_BEGIN 等）、リテラル（T_STRING 等）を分類します。
  • 一致したトークンに基づき、対応する文法規則のアクションが実行され、抽象構文木（AST）が構築されます。

呼び出し元と呼び出し先

  • 呼び出し元: ap_expr_yylex（字句解析器）がトークンを返す際に使用します。
  • 呼び出し先: ap_expr_yyparse（構文解析器）がトークンを消費し、文法規則の照合に使用します。

関連データ構造

  • YYSTYPE: パーサーのスタックで使用される共用体。トークンに関連付けられた値（文字列、AST ノード、数値）を保持します。

失敗モードと境界条件

  • 未定義のトークンが字句解析器から返された場合、パーサーは構文エラーとして ap_expr_yyerror を呼び出します。
