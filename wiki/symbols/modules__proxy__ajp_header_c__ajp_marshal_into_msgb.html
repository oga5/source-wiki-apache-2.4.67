ajp_marshal_into_msgb

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_header.c:217

目的

Apache の request_rec 構造体から情報を抽出し、AJP v13 プロトコルの FORWARD_REQUEST パケット形式へ直列化（マーシャリング）して ajp_msg_t バッファに格納する。

シグネチャまたは形状

apr_status_t ajp_marshal_into_msgb(ajp_msg_t *msg,
                                   request_rec *r,
                                   apr_uri_t *uri,
                                   const char *secret)

入力と出力

  • **入力**:
  • msg: 操作対象の ajp_msg_t バッファ。
  • r: Apache のリクエスト情報構造体。
  • uri: 解析済みのリクエスト URI。
  • secret: AJP 認証用シークレット（オプション）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は APR_EGENERAL または AJP_EOVERFLOW を返す。

前提条件と不変条件

  • msg は事前に確保されていること。
  • r は有効なリクエストコンテキストであること。
  • ajp_msg_reset(msg) が実行され、バッファが初期化される。

アルゴリズムまたは振る舞い

1. **初期化**: リクエストメソッド、SSL 状態、ヘッダー数を特定。 2. **ヘッダー構築**: CMD_AJP13_FORWARD_REQUEST を先頭に、プロトコル、パス、IP、ホスト名、ポート、SSL フラグ、ヘッダー数を順次追加。 3. **ヘッダーのシリアライズ**: headers_in テーブルを走査し、既知のヘッダーは ID に変換、未知のヘッダーは文字列として追加。 4. **属性の追加**: シークレット、リモートユーザー、認証タイプ、クエリ文字列、JVM ルートを順次追加。 5. **SSL 情報**: HTTPS 通信時のみ、証明書、暗号スイート、セッション、キーサイズ、プロトコル名を属性として追加。 6. **環境変数**: AJP_ で始まる環境変数をリクエスト属性として追加。 7. **終了**: SC_A_ARE_DONE を付与して終了。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていませんが、mod_proxy_ajp.c 等のプロキシ処理ループから呼び出されることが想定されます。
  • **呼び出し先**:
  • ajp_msg_* 系関数（ajp_msg_reset, ajp_msg_append_*）
  • sc_for_req_method_by_id, sc_for_req_header
  • ap_proxy_conn_is_https, ap_proxy_ssl_val
  • ap_get_useragent_host, ap_get_server_name

関連データ構造

  • ajp_msg_t: AJP 通信用のメッセージバッファ。
  • request_rec: Apache のリクエスト情報。
  • apr_table_t: ヘッダーや環境変数の保持に使用。

失敗モードと境界条件

  • **AJP_EOVERFLOW**: メッセージバッファへの書き込み時に容量不足が発生した場合。
  • **APR_EGENERAL**: メッセージの初期構築やシークレット、SSL キーサイズの追加に失敗した場合。
  • **未知のメソッド**: メソッドが特定できない場合、SC_M_JK_STORED として扱い、属性としてメソッド名を追加する。

保守担当者向けメモ

  • ajp_msg_t のメモリは r->pool に依存するため、リクエスト終了時に自動解放される。
  • sc_for_req_header は固定長バッファと memcmp を使用した最適化が行われているため、ヘッダー定義の変更時は注意が必要。
  • EBCDIC 環境における文字列変換の考慮が一部コメントアウトされている（XXXX）。
