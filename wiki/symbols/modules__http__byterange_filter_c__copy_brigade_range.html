copy_brigade_range

定義

  • 種別: function
  • 定義箇所: modules/http/byterange_filter.c:288

目的

元のバケットブリゲード（bb）から指定されたバイト範囲（start から end まで）のデータを抽出し、新しいブリゲード（bbout）へコピーします。

シグネチャまたは形状

apr_status_t copy_brigade_range(apr_bucket_brigade *bb,
                                apr_bucket_brigade *bbout,
                                apr_off_t start,
                                apr_off_t end)

入力と出力

  • **入力**:
  • bb: 読み込み元のバケットブリゲード。
  • bbout: 抽出したバケットを格納する出力用ブリゲード。
  • start: 抽出開始位置（オフセット）。
  • end: 抽出終了位置（オフセット）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時は APR_EINVAL やバケットコピー時のエラーコードを返します。

前提条件と不変条件

  • start および end は 0 以上であること。
  • start <= end であること。
  • ブリゲード内の各バケットの長さが定義されていること（e->length != -1）。
  • 処理終了後、bbout に格納されたバケットの合計長は end - start + 1 と一致すること。

アルゴリズムまたは振る舞い

1. 入力された start と end を apr_uint64_t に変換し、範囲の妥当性を検証します。 2. bb 内のバケットを走査し、指定範囲に含まれるバケットの開始位置と終了位置を特定します。 3. 該当するバケットを apr_bucket_copy で複製し、bbout に追加します。 4. 範囲の境界に位置するバケットに対しては apr_bucket_split を実行し、指定された範囲外のデータを切り捨てます。 5. 処理中にエラーが発生した場合は、bbout をクリーンアップしてエラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/byterange_filter.c 内の ap_byterange_filter（505行目付近）。
  • **呼び出し先**:
  • apr_bucket_copy
  • apr_bucket_split
  • apr_bucket_delete
  • apr_brigade_cleanup
  • apr_brigade_length

関連データ構造

  • apr_bucket_brigade: バケットのリストを管理する構造体。
  • apr_bucket: データの断片を保持する構造体。

失敗モードと境界条件

  • start < 0 または end < 0 の場合: APR_EINVAL を返します。
  • start > end の場合: APR_EINVAL を返します。
  • 範囲に該当するバケットが見つからない場合: APR_EINVAL を返します。
  • バケットのコピーや分割に失敗した場合: apr_brigade_cleanup を実行し、そのエラーコードを返します。

保守担当者向けメモ

  • 本関数はバケットの分割とコピーを伴うため、範囲指定が細分化されるとメモリおよび CPU のオーバーヘッドが増加します。
  • AP_DEBUG_ASSERT を使用して、バケットの長さが不明なケースや、抽出後のブリゲード長が期待値と異なるケースを開発時に検知するように設計されています。
