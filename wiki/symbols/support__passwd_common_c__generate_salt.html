generate_salt

定義

  • 種別: function
  • 定義箇所: support/passwd_common.c:71

目的

パスワードハッシュ生成に使用するランダムなソルト文字列を生成します。

シグネチャまたは形状

int generate_salt(char *s, size_t size, const char **errstr, apr_pool_t *pool);

入力と出力

  • **入力**:
  • s: 生成されたソルトを格納するバッファ（呼び出し元で確保）。
  • size: 生成するソルトの文字数。
  • pool: エラーメッセージ生成に使用する APR メモリプール。
  • **出力**:
  • s: 生成された Base64 風のソルト文字列（ヌル終端）。
  • errstr: 失敗時にエラーメッセージが格納されるポインタ。
  • 戻り値: 成功時は 0、失敗時は ERR_RANDOM を返します。

前提条件と不変条件

  • size に基づいて計算される必要バイト数が 32 バイト（rnd 配列のサイズ）を超えてはなりません。超えた場合は abort() します。
  • s は size + 1 以上のメモリ領域を確保している必要があります。

アルゴリズムまたは振る舞い

1. size から必要なバイト数 n を算出します。 2. apr_generate_random_bytes を呼び出し、乱数を取得します。 3. 取得した乱数を itoa64 文字列（./0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz）を用いて Base64 風の文字に変換します。 4. 変換した文字を s に順次格納し、最後にヌル文字で終端します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/passwd_common.c 内の mkhash 関数（複数箇所）。
  • modules/lua/lua_passwd.c 内の関数。
  • **呼び出し先**:
  • apr_generate_random_bytes (APR)
  • apr_psprintf (APR)
  • apr_file_printf (APR)

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • itoa64: ソルト変換用の静的文字配列。

失敗モードと境界条件

  • **バッファ不足**: size が大きすぎて rnd 配列（32バイト）に収まらない場合、abort() を呼び出しプロセスが終了します。
  • **乱数生成失敗**: apr_generate_random_bytes がエラーを返した場合、ERR_RANDOM を返し、errstr にエラー詳細をセットします。

保守担当者向けメモ

  • 本関数は errfile グローバル変数を使用するため、呼び出し側で適切に初期化されていることを確認してください。
  • itoa64 の変換ロジックは標準的な Base64 とは異なるため、変更時は注意が必要です。
