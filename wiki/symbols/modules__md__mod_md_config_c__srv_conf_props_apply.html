srv_conf_props_apply

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_config.c:214

目的

サーバー設定構造体（md_srv_conf_t）から、管理対象ドメインオブジェクト（md_t）へ設定プロパティを適用・同期します。

シグネチャまたは形状

static void srv_conf_props_apply(md_t *md, const md_srv_conf_t *from, apr_pool_t *p)

入力と出力

  • md (入出力): 適用先となる md_t オブジェクト。
  • from (入力): 設定値のソースとなる md_srv_conf_t オブジェクト。
  • p (入力): メモリ割り当てに使用する apr_pool_t。

前提条件と不変条件

  • from の各フィールドが DEF_VAL または MD_REQUIRE_UNSET ではない場合にのみ、md への値のコピーが行われます。
  • p は、配列のコピーや秘密鍵仕様のクローンなど、メモリ確保が必要な操作に使用されます。

アルゴリズムまたは振る舞い

1. require_https, transitive, renew_mode, must_staple 等のスカラ値を、ソース側が未設定でない場合に限り md に上書きします。 2. pks（秘密鍵仕様）は md_pkeys_spec_clone を用いて複製します。 3. ca_urls や ca_challenges 等の配列は apr_array_copy を用いて複製します。 4. ca_contact が存在する場合、既存の md->contacts をクリアし、md_util_schemify を適用した値を新たにプッシュします。 5. その他、ca_proto, ca_agreement, profile, dns01_cmd 等の値を直接代入します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md_config.c:432 にて呼び出されています。
  • **呼び出し先**: md_pkeys_spec_clone, apr_array_copy, apr_array_clear, md_util_schemify 等のユーティリティ関数を呼び出します。

関連データ構造

  • md_t: 管理対象ドメインのメイン構造体。
  • md_srv_conf_t: バーチャルホスト単位の設定構造体。

失敗モードと境界条件

  • from が指す構造体の各フィールドがデフォルト値（DEF_VAL 等）である場合、そのフィールドはスキップされ、md 側の既存値は保持されます。
  • メモリ不足時には apr_pool_t の動作に従い、アボートまたはエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は設定の「マージ」ではなく、特定の md_t インスタンスへの「適用」を目的としています。
  • 新しい設定項目を md_srv_conf_t に追加した際は、本関数にも対応するコピーロジックを追加する必要があります。
