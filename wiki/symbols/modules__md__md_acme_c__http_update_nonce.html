http_update_nonce

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme.c:124

目的

HTTP レスポンスヘッダーから ACME ノンス（Nonce）を抽出し、内部状態を更新するためのコールバック関数です。

シグネチャまたは形状

static apr_status_t http_update_nonce(const md_http_response_t *res, void *data)

入力と出力

  • **入力**:
  • res: HTTP レスポンスを保持する md_http_response_t 構造体へのポインタ。
  • data: ノンス更新対象のコンテキストデータ（void *）。
  • **出力**:
  • 常に APR_SUCCESS を返します。

前提条件と不変条件

  • res および res->headers が有効である必要があります。
  • data は req_update_nonce が期待する型と互換性がある必要があります。

アルゴリズムまたは振る舞い

1. res->headers を引数として req_update_nonce(data, res->headers) を呼び出します。 2. 処理結果に関わらず APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c:160 において、md_http_HEAD_perform のコールバック引数として指定されています。
  • **呼び出し先**:
  • req_update_nonce（同ファイル内の関数）

関連データ構造

  • md_http_response_t: HTTP レスポンス情報を保持する構造体。
  • apr_table_t (間接的): res->headers として参照されるヘッダーテーブル。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • 内部で呼び出される req_update_nonce がヘッダーの解析に失敗した場合の挙動については、当該関数の仕様に依存します。

保守担当者向けメモ

  • 本関数は md_http レイヤーのコールバックインターフェースに適合させるためのラッパーです。
  • ノンスの更新ロジックそのものは req_update_nonce に集約されているため、ロジックの変更が必要な場合はそちらを確認してください。
