authz_host_post_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:372

目的

設定完了フェーズにおいて、サブネット情報の共有用ハッシュテーブルを無効化し、.htaccess ファイルの解析時における誤用を防止する。

シグネチャまたは形状

static int authz_host_post_config(apr_pool_t *p, apr_pool_t *plog,
                                  apr_pool_t *ptemp, server_rec *s)

入力と出力

  • 入力:
  • p: 設定用プール
  • plog: ログ用プール
  • ptemp: 一時的な設定解析用プール
  • s: サーバーレコード
  • 出力:
  • 常に OK を返す。

前提条件と不変条件

  • 本関数は post_config フックとして実行される。
  • 実行後、モジュール内のグローバル変数 parsed_subnets は NULL に設定される。

アルゴリズムまたは振る舞い

1. モジュール内で設定解析時に使用される一時的なハッシュテーブル parsed_subnets を NULL にリセットする。 2. これにより、設定解析フェーズ終了後のリクエスト処理や .htaccess の再解析時に、古いサブネット情報が誤って参照されることを防ぐ。 3. 成功を示す OK を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server の post_config フック（modules/aaa/mod_authz_host.c:384 にて ap_hook_post_config に登録）。
  • 呼び出し先: なし。

関連データ構造

  • parsed_subnets: 設定解析フェーズで使用されるサブネット共有用ハッシュテーブル。

失敗モードと境界条件

  • 本関数自体はエラーを返さない設計となっている。

保守担当者向けメモ

  • parsed_subnets は設定解析中（ptemp プール）のみ有効なデータ構造であるため、本関数によるリセットはメモリ安全性の観点から重要である。
