ap_session_load

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:91

目的

リクエストに関連付けられたセッションデータをロードし、session_rec 構造体として提供します。セッションが存在しない場合や期限切れの場合は、新規の空セッションを作成します。

シグネチャまたは形状

apr_status_t ap_session_load(request_rec *r, session_rec **z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報 (request_rec)。
  • z: ロードされたセッション構造体 (session_rec) を格納するためのポインタ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。エラー時はエラーコードを返します。
  • *z: 取得または生成された session_rec へのポインタ。

前提条件と不変条件

  • dconf->enabled が有効であること。
  • session_included により、対象リクエストがセッション処理の対象パスに含まれていること。
  • セッションデータはリクエストプール (r->pool) 上で管理され、リクエスト終了時に自動的に解放されます。

アルゴリズムまたは振る舞い

1. **設定確認**: ディレクトリ設定を確認し、セッションが有効かつ対象パスか判定します。 2. **ロード**: ap_run_session_load フックを呼び出し、バックエンドからセッションデータを取得します。 3. **デコード**: ap_run_session_decode フックを呼び出し、取得したデータを session_rec の entries テーブルに展開します。 4. **有効期限検証**: セッションの expiry を現在時刻と比較し、期限切れの場合はセッションを破棄（zz = NULL）します。 5. **初期化**: セッションが存在しない場合、apr_pcalloc で新規 session_rec を作成し、entries テーブルを初期化します。 6. **設定適用**: 設定に基づき maxage を適用し、必要に応じて expiry を計算します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session.c 内の各処理（session_fixups 等）。
  • modules/aaa/mod_auth_form.c（APR_RETRIEVE_OPTIONAL_FN を介して利用）。
  • **呼び出し先**:
  • ap_run_session_load (フック)
  • ap_run_session_decode (フック)
  • session_included (内部関数)

関連データ構造

  • session_rec: セッションのキー・値ペアや有効期限を保持する構造体。
  • session_dir_conf: ディレクトリ単位のセッション設定。

失敗モードと境界条件

  • **ストレージ未設定**: ap_run_session_load が DECLINED を返した場合、APR_EGENERAL を返します。
  • **デコード失敗**: ap_run_session_decode がエラーを返した場合、セッションはロードされず、空のセッションとして扱われます。
  • **期限切れ**: セッションが期限切れの場合、既存データは無視され、新規の空セッションが作成されます。

保守担当者向けメモ

  • dirty フラグが立っていない場合、保存処理はスキップされます。
  • 本関数は r->pool を使用するため、呼び出し側でのメモリ解放は不要です。
  • 内部リダイレクトが発生した場合、連鎖するリクエストに対してもセッション処理が適用される点に注意してください。
