merge_env_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_env.c:47

目的

親ディレクトリの設定（basev）と子ディレクトリの設定（addv）をマージし、新しいディレクトリ設定構造体を作成する。

シグネチャまたは形状

static void *merge_env_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定（env_dir_config_rec*）。
  • addv: 子ディレクトリの設定（env_dir_config_rec*）。
  • **出力**:
  • マージ結果を保持する新しい env_dir_config_rec 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は env_dir_config_rec 型としてキャスト可能であること。
  • apr_pool_t は有効であり、メモリ割り当てが可能であること。
  • add->unsetenv に含まれる変数は、add->vars から既に削除済みであることを前提としている。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して新しい設定構造体 res を確保する。 2. 親の設定 base->vars を新しいテーブル res->vars にコピーする。 3. 子の設定 add->unsetenv に含まれるすべてのキーを、res->vars から apr_table_unset を用いて削除する。 4. 子の設定 add->vars に含まれるすべてのキーと値を、res->vars に apr_table_setn を用いて上書き追加する。 5. マージされた res を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_env.c:184（Apache の設定マージ機構から呼び出される）。
  • **呼び出し先**:
  • apr_palloc
  • apr_table_copy
  • apr_table_elts
  • apr_table_unset
  • apr_table_setn

関連データ構造

  • env_dir_config_rec: ディレクトリごとの環境変数設定を保持する構造体。vars（設定テーブル）と unsetenv（削除対象テーブル）を管理する。

失敗モードと境界条件

  • add->unsetenv または add->vars が空（NULL）の場合、それぞれの処理はスキップされる。
  • メモリ不足時は apr_palloc が例外を発生させる可能性がある。

保守担当者向けメモ

  • 本関数は UnsetEnv を先に処理してから SetEnv を適用する順序を守っている。これにより、設定ファイル内での記述順序に関わらず、子ディレクトリの設定が親の設定を正しくオーバーライドする。
