ssl_scache_status_register

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_scache.c:234

目的

mod_status モジュールに対し、SSL セッションキャッシュの利用状況を表示するためのフック関数を登録します。

シグネチャまたは形状

void ssl_scache_status_register(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p (フック登録に使用されるメモリプール)
  • **出力**: なし (void)

前提条件と不変条件

  • mod_status がサーバーにロードされている環境で実行されることを前提とします。
  • APR_OPTIONAL_HOOK を使用しているため、mod_status が存在しない場合でもコンパイルおよび実行が可能です。

アルゴリズムまたは振る舞い

1. APR_OPTIONAL_HOOK マクロを呼び出し、ap 名前空間の status_hook に対して ssl_ext_status_hook 関数を登録します。 2. 登録時の優先順位は APR_HOOK_MIDDLE に設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c:447 (モジュール初期化フェーズにて呼び出されます)
  • **呼び出し先**: ssl_ext_status_hook (登録されるフック関数)

関連データ構造

  • apr_pool_t: Apache Portable Runtime のメモリ管理構造体。
  • mod_status.h: status_hook の定義元。

失敗モードと境界条件

  • mod_status がロードされていない場合、フックの登録は行われませんが、エラーにはなりません。

保守担当者向けメモ

  • 本関数は mod_ssl の初期化プロセスの一部です。ssl_ext_status_hook の実装に変更を加える際は、本関数による登録が正しく維持されているか確認してください。
