socache_mc_store

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_memcache.c:198

目的

Memcached バックエンドを使用して、指定されたキャッシュキーとデータ、有効期限をキャッシュストレージに保存します。

シグネチャまたは形状

apr_status_t socache_mc_store(ap_socache_instance_t *ctx, server_rec *s,
                              const unsigned char *id, unsigned int idlen,
                              apr_time_t expiry,
                              unsigned char *ucaData, unsigned int nData,
                              apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: Memcached インスタンスコンテキスト。
  • s: サーバーレコード。
  • id, idlen: キャッシュキーのバイナリデータとその長さ。
  • expiry: キャッシュの有効期限（絶対時刻）。
  • ucaData, nData: 保存するデータ本体とそのサイズ。
  • p: メモリ割り当て用プール。
  • **出力**:
  • apr_status_t: 操作の成功（APR_SUCCESS）または失敗を示すステータスコード。

前提条件と不変条件

  • socache_mc_id2key により、id は MC_KEY_LEN（254バイト）以内の文字列キーに変換可能である必要があります。
  • expiry は現在時刻よりも未来である必要があります。

アルゴリズムまたは振る舞い

1. socache_mc_id2key を呼び出し、内部IDを Memcached 用の文字列キーに変換します。変換に失敗した場合は APR_EINVAL を返します。 2. expiry から現在時刻を減算し、Memcached が要求する「有効期限までの秒数」を算出します。 3. 算出された秒数が 0 以下の場合、APR_EINVAL を返します。 4. apr_memcache_set を呼び出し、データを Memcached に送信します。 5. 保存に失敗した場合は ap_log_error でエラーログを出力し、エラーコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_memcache.c:358（socache_mc プロバイダー構造体の一部として参照）。
  • **呼び出し先**:
  • socache_mc_id2key（キー変換）
  • apr_time_now, apr_time_sec（時刻計算）
  • apr_memcache_set（Memcached へのデータ書き込み）
  • ap_log_error（エラーログ出力）

関連データ構造

  • ap_socache_instance_t: Memcached 接続情報を含むインスタンスコンテキスト。
  • server_rec: Apache サーバー設定情報。

失敗モードと境界条件

  • **キー変換失敗**: id が長すぎる場合、APR_EINVAL を返します。
  • **有効期限切れ**: expiry が現在時刻以前の場合、APR_EINVAL を返します。
  • **通信エラー**: apr_memcache_set が失敗した場合、そのステータスコードを返し、エラーログを記録します。

保守担当者向けメモ

  • 本関数は apr_memcache_set をラップしており、Memcached の仕様に従い有効期限を秒単位で扱います。
  • MC_KEY_LEN を超えるキーは処理できないため、将来的にキーの仕様変更がある場合は注意が必要です。
