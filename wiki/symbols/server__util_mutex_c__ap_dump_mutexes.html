ap_dump_mutexes

定義

  • 種別: function
  • 定義箇所: server/util_mutex.c:501

目的

現在設定されているすべてのミューテックスの構成情報を、指定された出力ストリームへダンプする。デバッグおよび設定確認用に使用される。

シグネチャまたは形状

void ap_dump_mutexes(apr_pool_t *p, server_rec *s, apr_file_t *out)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • s: サーバーレコード（本関数内では未使用）。
  • out: ダンプ結果を書き込むファイルハンドル。
  • **出力**: なし（out に対してテキスト形式で情報を書き出す）。

前提条件と不変条件

  • mxcfg_by_type ハッシュテーブルが初期化済みであること。
  • p は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. mxcfg_by_type ハッシュテーブルを走査し、登録されているすべてのミューテックス型を取得する。 2. mxcfg_lookup を用いて各ミューテックスの有効な設定を解決する。 3. デフォルト設定と同一の場合は using_defaults と出力する。 4. none が設定されている場合は none と出力する。 5. それ以外の場合、メカニズム（fcntl, flock, posixsem, sysvsem, pthread 等）およびディレクトリパス、omit_pid フラグの有無を整形して出力する。 6. 未知のメカニズムに遭遇した場合は ap_assert(0) により異常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:5552
  • **呼び出し先**:
  • apr_hash_first, apr_hash_next, apr_hash_this, apr_hash_get (APR ハッシュ操作)
  • mxcfg_lookup (設定解決)
  • apr_file_printf (出力)
  • ap_server_root_relative (パス解決)

関連データ構造

  • mutex_cfg_t: ミューテックスの設定情報を保持する構造体。
  • mxcfg_by_type: ミューテックス型をキーとする内部ハッシュテーブル。

失敗モードと境界条件

  • mxcfg->mech が定義された列挙値のいずれにも該当しない場合、ap_assert によりプロセスが停止する。
  • out が無効なファイルハンドルの場合、apr_file_printf の呼び出しが失敗する可能性がある。

保守担当者向けメモ

  • 本関数は server/util_mutex.c 内の内部ハッシュテーブルに直接依存している。新しいミューテズムメカニズムを APR に追加した際は、本関数の switch 文にも対応するケースを追加する必要がある。
