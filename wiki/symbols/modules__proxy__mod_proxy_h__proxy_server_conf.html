proxy_server_conf

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:154

目的

Apache HTTP Server のプロキシサブシステムにおいて、サーバー全体のプロキシ設定を保持するための構造体です。プロキシの動作、負荷分散設定、バッファサイズ、タイムアウト、およびバックエンド接続に関連するグローバルな構成情報を管理します。

シグネチャまたは形状

typedef struct {
    /* 設定リスト */
    apr_array_header_t *proxies;
    apr_array_header_t *sec_proxy;
    apr_array_header_t *aliases;
    apr_array_header_t *noproxies;
    apr_array_header_t *dirconn;
    apr_array_header_t *workers;
    apr_array_header_t *balancers;
    
    /* ワーカーおよび基本設定 */
    proxy_worker *forward;
    proxy_worker *reverse;
    const char *domain;
    const char *id;
    apr_pool_t *pool;
    int req;
    int max_balancers;
    int bgrowth;
    
    /* 動作オプション（Viaヘッダー、バッファ、タイムアウト等） */
    enum { via_off, via_on, via_block, via_full } viaopt;
    apr_size_t recv_buffer_size;
    apr_size_t io_buffer_size;
    long maxfwd;
    apr_interval_time_t timeout;
    enum { bad_error, bad_ignore, bad_body } badopt;
    enum { status_off, status_on, status_full } proxy_status;
    
    /* ネットワークおよび同期 */
    apr_sockaddr_t *source_address;
    apr_global_mutex_t *mutex;
    ap_slotmem_instance_t *bslot;
    ap_slotmem_provider_t *storage;

    /* 設定フラグ（ビットフィールド） */
    unsigned int req_set:1;
    /* ... 他多数の _set フラグおよび継承設定 */
} proxy_server_conf;

入力と出力

  • **入力**: Apache の設定ファイル（httpd.conf 等）から読み込まれたディレクティブ値が、設定プール（pool）を介して本構造体に格納されます。
  • **出力**: プロキシ処理の各ステージにおいて、リクエスト処理の挙動を決定するためのパラメータとして参照されます。

前提条件と不変条件

  • **メモリ管理**: 本構造体およびそのメンバ（apr_array_header_t 等）は、pool メンバで指定された Apache 設定プールによってライフサイクルが管理されます。
  • **排他制御**: マルチプロセス環境において共有メモリ（bslot）を更新する場合、mutex メンバによる排他制御が必須です。

アルゴリズムまたは振る舞い

  • **設定管理**: サーバー全体のプロキシ動作（req）、バランサーの最大数（max_balancers）、およびバックエンドとの通信タイムアウト（timeout）を定義します。
  • **状態保持**: bslot を通じて、実行時のバランサー状態を共有メモリ上で管理します。
  • **フラグ管理**: 各設定項目に対して _set 接尾辞を持つビットフィールドが用意されており、設定が明示的に指定されたかどうかを追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy.c の設定処理関数（set_balancer_param 等）や、mod_proxy_http2.c などの各プロトコルハンドラが設定参照として利用しています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を直接呼び出すことはありません。ただし、storage メンバを通じて ap_slotmem プロバイダーの関数群が間接的に利用されます。

関連データ構造

  • proxy_worker: バックエンドサーバーの定義。
  • proxy_balancer: 負荷分散グループの定義。
  • apr_pool_t: メモリ管理用プール。
  • ap_slotmem_instance_t: 共有メモリ管理用インスタンス。

失敗モードと境界条件

  • **メモリ不足**: 設定読み込み時に pool からのメモリ確保が失敗する可能性があります。
  • **共有メモリ競合**: mutex による保護が不適切な場合、マルチプロセス環境での bslot 更新時にデータ不整合が発生する可能性があります。
  • **設定値の範囲**: max_balancers や bgrowth の値が適切でない場合、バランサーの動的な追加や管理に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はバイナリ互換性に影響を与えるため、メンバの追加や変更を行う際は include/ap_mmn.h で定義されるモジュールマジックナンバー（MMN）の更新を検討してください。
  • _set フラグは、設定の継承やデフォルト値の適用を判断するために重要です。新しい設定項目を追加する際は、対応する _set ビットフィールドも併せて定義してください。
