server_rec

定義

  • 種別: typedef
  • 定義箇所: include/http_vhost.h:73

目的

Apache HTTP Server におけるバーチャルホスト（vhost）設定を保持する主要なデータ構造です。サーバーのライフサイクル全体を通じて、リクエストや接続がどのバーチャルホスト設定に属するかを特定するために使用されます。

シグネチャまたは形状

構造体の詳細なメンバ構成は include/httpd.h に定義されています。本シンボルは、Apache のコアモジュールにおいてサーバー設定を表現する型として参照されます。

入力と出力

  • **入力**: サーバー起動時の設定読み込み、およびリクエスト処理時のルーティング判断に使用されます。
  • **出力**: ap_update_vhost_from_headers 等の関数により、リクエスト処理中に r->server として適切な server_rec が割り当てられます。

前提条件と不変条件

  • サーバー起動時に ap_init_vhost_config で初期化され、設定読み込み後に ap_fini_vhost_config によって実行時の高速検索用テーブルが構築される必要があります。

アルゴリズムまたは振る舞い

  • **接続時**: ap_update_vhost_given_ip により、接続の IP アドレスとポートからベースとなる server_rec が特定されます。
  • **リクエスト処理時**: ap_update_vhost_from_headers が HTTP の Host ヘッダーを解析し、名前ベースのバーチャルホストへ r->server を切り替えます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • include/ap_listen.h: ap_setup_listeners や ap_duplicate_listeners の引数として使用。
  • include/ap_mpm.h: mpm フックや child_status フックの引数として使用。
  • include/ap_expr.h: 式評価のコンテキストとして参照。
  • **呼び出し先**:
  • 提供されたソース情報からは、特定の関数呼び出し先は特定できません。

関連データ構造

  • conn_rec: 接続情報を保持し、server_rec と関連付けられます。
  • request_rec: リクエスト処理中に server_rec を保持します。
  • ap_vhost_iterate_conn_cb: 接続に関連する server_rec を走査するためのコールバック型。

失敗モードと境界条件

  • 設定ファイルに不整合がある場合、ap_fini_vhost_config によるテーブル構築時にエラーが発生する可能性があります。
  • リクエスト時に適切な Host ヘッダーが存在しない場合、デフォルトのバーチャルホスト設定が適用されます。

保守担当者向けメモ

  • server_rec の構造変更は include/ap_mmn.h に記載されているモジュールマジックナンバー（MMN）の更新を伴う場合があります。
  • 実装の詳細（ハッシュテーブルの構造やマッチングアルゴリズム）については server/vhost.c を参照してください。
