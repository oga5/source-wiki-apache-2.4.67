dav_fs_merge_server_config

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/mod_dav_fs.c:45

目的

Apache HTTP Server の設定マージフェーズにおいて、親サーバー設定と子（オーバーライド）サーバー設定を統合し、新しいサーバー設定構造体を生成する。

シグネチャまたは形状

static void *dav_fs_merge_server_config(apr_pool_t *p, void *base, void *overrides)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • base: 親コンテキストの dav_fs_server_conf 構造体。
  • overrides: 現在のコンテキストで上書き設定された dav_fs_server_conf 構造体。
  • **出力**:
  • 統合された新しい dav_fs_server_conf 構造体へのポインタ。

前提条件と不変条件

  • base および overrides は dav_fs_server_conf 型としてキャスト可能であること。
  • p は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい dav_fs_server_conf 構造体をメモリプール p 上に確保する。 2. lockdb_path フィールドについて、overrides（子）側に値が存在すればそれを採用し、存在しなければ base（親）側の値を継承する。 3. 統合された構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のモジュール設定マージ機構（modules/dav/fs/mod_dav_fs.c:105 にて dav_fs_module 構造体の一部として登録）。
  • **呼び出し先**: 直接的な関数呼び出しはなし（apr_pcalloc を除く）。

関連データ構造

  • dav_fs_server_conf: サーバーごとの設定を保持する構造体。lockdb_path を管理する。

失敗モードと境界条件

  • メモリ不足時、apr_pcalloc はアボートする可能性がある。
  • base または overrides が NULL の場合の挙動については、ソースコード上での明示的なガードは存在しない。

保守担当者向けメモ

  • 本関数は DAVLockDB ディレクティブの設定継承を制御する。
  • DAVLockDB はサーバー単位の設定であり、ディレクトリごとのオーバーライドは行われない仕様を反映している。
