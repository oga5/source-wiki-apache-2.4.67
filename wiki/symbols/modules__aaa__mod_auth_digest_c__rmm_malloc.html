rmm_malloc

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_digest.c:243

目的

APR共有メモリ管理（apr_rmm）を使用してメモリを割り当て、その論理オフセットをプロセス空間内の絶対アドレスに変換して返す。

シグネチャまたは形状

void *rmm_malloc(apr_rmm_t *rmm, apr_size_t size)

入力と出力

  • **入力**:
  • rmm: 割り当て対象の共有メモリ管理ハンドル。
  • size: 割り当てるメモリサイズ（バイト単位）。
  • **出力**:
  • 成功時: 割り当てられたメモリ領域のポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • rmm は有効な apr_rmm_t ハンドルであること。
  • 割り当て失敗時には apr_rmm_malloc が 0（オフセットなし）を返すため、関数は NULL を返す。

アルゴリズムまたは振る舞い

1. apr_rmm_malloc を呼び出し、共有メモリセグメント内でのオフセットを取得する。 2. オフセットが 0 の場合は割り当て失敗とみなし、NULL を返す。 3. オフセットが有効な場合、apr_rmm_addr_get を使用して、そのオフセットに対応するプロセス空間内の絶対アドレスを算出し、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_digest.c 内の以下の箇所から呼び出されていることが確認されている。
  • line 314: client_list の割り当て
  • line 337: opaque_cntr の割り当て
  • line 354: otn_counter の割り当て
  • line 883, 892: client_entry の割り当て
  • **呼び出し先**:
  • apr_rmm_malloc (APRライブラリ)
  • apr_rmm_addr_get (APRライブラリ)

関連データ構造

  • apr_rmm_t: 共有メモリ管理のための不透明なハンドル。
  • apr_rmm_off_t: 共有メモリ内の相対オフセット値。

失敗モードと境界条件

  • 共有メモリの空き容量が不足している場合、apr_rmm_malloc が失敗し、本関数は NULL を返す。呼び出し元はこの戻り値をチェックする必要がある。

保守担当者向けメモ

  • 本関数は apr_rmm の低レイヤー操作をラップし、ポインタ変換を簡略化するユーティリティである。
  • 共有メモリを使用するため、マルチプロセス環境での競合を避けるために適切なロック（client_lock 等）が呼び出し元で確保されていることを確認すること。
