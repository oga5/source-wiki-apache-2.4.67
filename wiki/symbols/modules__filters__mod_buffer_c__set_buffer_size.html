set_buffer_size

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_buffer.c:320

目的

Apache HTTP Server の BufferSize ディレクティブに対する設定値のバリデーションおよび、ディレクトリ単位の設定構造体への反映を行う。

シグネチャまたは形状

static const char *set_buffer_size(cmd_parms *cmd, void *dconf, const char *arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • dconf: 設定対象となる buffer_conf 構造体へのポインタ。
  • arg: 設定値として渡された文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • dconf は buffer_conf 型としてキャスト可能であること。
  • arg は apr_strtoff で数値変換可能な文字列であること。

アルゴリズムまたは振る舞い

1. apr_strtoff を使用して、入力された arg を conf->size に数値として格納する。 2. 変換に失敗した場合、または conf->size が 0 以下である場合は、エラーメッセージを返して終了する。 3. 正常に変換された場合、conf->size_set を 1 に設定し、NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_buffer.c:334 (コマンドテーブル定義内での参照が確認されている)。
  • **呼び出し先**: apr_strtoff (APR ライブラリ関数)。

関連データ構造

  • buffer_conf: size（バッファ上限）および size_set（設定有無）を保持する構造体。
  • cmd_parms: Apache の設定コマンド処理用構造体。

失敗モードと境界条件

  • arg が数値として解釈できない場合、または 0 以下の値を指定した場合はエラーメッセージを返す。

保守担当者向けメモ

  • 本関数は BufferSize ディレクティブのハンドラとして機能する。
  • 設定値は apr_off_t 型として扱われるため、プラットフォームの制限に依存する。
