rewrite_ctx

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:422

目的

mod_rewrite におけるルール適用時のリクエスト状態、URI、およびバックリファレンス情報を保持するための実行コンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    request_rec *r;
    const char  *uri;
    const char  *vary_this;
    const char  *vary;
    char        *perdir;
    backrefinfo briRR;
    backrefinfo briRC;
} rewrite_ctx;

入力と出力

本構造体は、ルールエンジン（apply_rewrite_cond や do_expand 等）の引数として渡され、リクエスト処理中の動的な状態を保持・更新するために使用されます。

前提条件と不変条件

  • r は有効な request_rec を指す必要があります。
  • briRR（RewriteRule用）および briRC（RewriteCond用）は、正規表現のキャプチャ結果を保持します。

アルゴリズムまたは振る舞い

リクエストの書き換え処理において、以下の役割を果たします。

  • do_expand 等の関数を通じて、文字列内の変数やバックリファレンスを解決するための参照先となります。
  • vary フィールドは、レスポンスの Vary ヘッダーを構築するための情報を蓄積します。

呼び出し元と呼び出し先

  • **呼び出し元**: apply_rewrite_cond、do_expand、do_expand_env、do_expand_cookie、lookup_header、lookup_variable、la_u 等の mod_rewrite.c 内の関数群から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • backrefinfo: 正規表現のバックリファレンス情報を保持する構造体。

失敗モードと境界条件

  • r->pool を使用したメモリ割り当てに依存しているため、リクエストのライフサイクルを超えた保持はできません。
  • [N] フラグによる再帰的な書き換えが発生する場合、REWRITE_MAX_ROUNDS による制限が適用されます。

保守担当者向けメモ

  • 本構造体はリクエスト処理のコンテキストをカプセル化する重要な役割を担っています。関数シグネチャを変更する際は、これらを参照するすべてのヘルパー関数への影響を確認してください。
