data_item

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:339

目的

環境変数やクッキーなど、動的に追加される文字列データを連結リスト形式で保持するために使用されます。

シグネチャまたは形状

typedef struct data_item {
    struct data_item *next;
    char *data;
} data_item;

入力と出力

  • 入力: char *data（保持する文字列データ）および struct data_item *next（リストの次要素へのポインタ）。
  • 出力: なし（構造体定義）。

前提条件と不変条件

  • next はリストの終端で NULL を指す必要があります。
  • data はリクエストの生存期間（r->pool）に依存して割り当てられることが一般的です。

アルゴリズムまたは振る舞い

  • 連結リストのノードとして機能し、do_expand_env や do_expand_cookie 関数において、設定された環境変数やクッキーのリストを走査・展開するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/mappers/mod_rewrite.c:353 (構造体メンバとして使用)
  • modules/mappers/mod_rewrite.c:354 (構造体メンバとして使用)
  • modules/mappers/mod_rewrite.c:2694 (do_expand_env の引数)
  • modules/mappers/mod_rewrite.c:2830 (do_expand_cookie の引数)
  • modules/mappers/mod_rewrite.c:3738, 3767 (設定構造体からの取得)
  • 呼び出し先: なし（データ構造体のため）。

関連データ構造

  • rewrite_server_conf: この構造体をリストのヘッドとして保持する設定構造体。
  • rewrite_perdir_conf: 同上。
  • rewrite_ctx: 展開処理時にこのデータ構造を操作するコンテキスト。

失敗モードと境界条件

  • メモリ不足: apr_palloc 等によるメモリ割り当て失敗時に Apache のエラー処理へ依存します。
  • リストの循環: 連結リスト構築時に循環参照が発生しないよう、呼び出し側のロジックで管理する必要があります。

保守担当者向けメモ

  • 本構造体は単純な文字列リストであるため、複雑なデータ保持には向きません。
  • mod_rewrite 内の環境変数やクッキーの動的追加処理において頻繁に参照されるため、リスト操作ロジックを変更する際は do_expand_* 系関数への影響を確認してください。
