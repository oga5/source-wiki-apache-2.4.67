get_listeners_from_parent

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:442

目的

親プロセスから継承されたリスニングソケット情報をパイプ経由で取得し、子プロセス内で再構築する。

シグネチャまたは形状

void get_listeners_from_parent(server_rec *s)

入力と出力

  • **入力**: server_rec *s（サーバー設定構造体）。また、グローバルな pipe（親プロセスから継承された標準入力ハンドル）および ap_listeners リストを使用する。
  • **出力**: なし（副作用として ap_listeners 内の各 lr->sd にソケットが割り当てられる）。

前提条件と不変条件

  • 親プロセスが事前にソケットをリスニング状態にし、その情報をパイプ経由で送信する準備ができていること。
  • pipe ハンドルが有効であること。

アルゴリズムまたは振る舞い

1. ap_listeners が空の場合、デフォルトのリスナー構造体を初期化する。 2. ap_listeners リストを走査し、各要素に対して以下の処理を行う。

  • ReadFile を使用して、親プロセスから WSAPROTOCOL_INFO 構造体を読み取る。
  • WSASocket を呼び出し、取得したプロトコル情報からソケットを再生成する。
  • SetHandleInformation を呼び出し、継承フラグを解除する。
  • apr_os_sock_put を使用して、生成したソケットを ap_listen_rec 構造体に格納する。

    • 3. 読み取り失敗やソケット生成失敗時には、エラーログを出力して APEXIT_CHILDINIT でプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/mpm_winnt.c:437（直接の言及あり）
  • server/mpm/winnt/mpm_winnt.c:1674（get_listeners_from_parent(s); の呼び出し）
  • **呼び出し先**:
  • ReadFile, WSASocket, SetHandleInformation（Windows API）
  • apr_os_sock_put, apr_palloc, ap_log_error（APR/Apache API）

関連データ構造

  • ap_listen_rec: リスニングソケット情報を保持する構造体。
  • WSAPROTOCOL_INFO: Windowsソケットのプロトコル情報を格納する構造体。
  • server_rec: サーバー設定を保持する構造体。

失敗モードと境界条件

  • **パイプ読み取り失敗**: 親プロセスからのデータ受信に失敗した場合、APEXIT_CHILDINIT で異常終了する。
  • **ソケット生成失敗**: WSASocket が INVALID_SOCKET を返した場合、エラーログを出力して異常終了する。
  • **継承フラグ設定失敗**: SetHandleInformation が失敗した場合、エラーログを出力するが処理は継続する。

保守担当者向けメモ

  • 本関数はWindows特有のソケット継承メカニズムに依存している。
  • ソース内のコメントによれば、パイプのオープン処理は現在 winnt_rewrite_args に移動されているため、本関数は読み取り処理のみに専念している。
