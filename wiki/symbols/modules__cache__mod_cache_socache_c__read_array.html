read_array

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_socache.c:139

目的

シリアライズされたバッファから文字列の配列を読み込み、apr_array_header_t に復元します。

シグネチャまたは形状

apr_status_t read_array(request_rec *r, apr_array_header_t *arr,
                        unsigned char *buffer, apr_size_t buffer_len, 
                        apr_size_t *slider);

入力と出力

  • r: リクエストレコード（メモリ割り当て用プールとして使用）。
  • arr: 読み込み先の配列ヘッダー。
  • buffer: 読み込み元のデータバッファ。
  • buffer_len: バッファの総サイズ。
  • slider: バッファ内の現在の読み込み位置（オフセット）。関数実行後に更新されます。
  • 戻り値: 成功時は APR_SUCCESS、終端に達した場合は APR_EOF を返します。

前提条件と不変条件

  • slider はバッファ内の有効なオフセットを指している必要があります。
  • arr は apr_array_make 等で初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. slider が指す位置から buffer_len に達するまでループします。 2. \r を検出した場合:

  • 直前の位置と現在の位置が同じ（空行）であれば、slider を進めて APR_SUCCESS を返します。
  • それ以外の場合、buffer 内の文字列を apr_pstrndup で複製し、arr に追加します。
  • \r の直後に \n がある場合はそれもスキップします。

    • 3. \0 を検出した場合:

  • slider を進め、APR_SUCCESS を返して終了します。

    • 4. 上記以外の場合は slider をインクリメントして次へ進みます。 5. ループ終了まで条件を満たさない場合は APR_EOF を返します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/cache/mod_cache_socache.c:537 にて sobj->buffer を対象に呼び出されています。
  • 呼び出し先: apr_array_push および apr_pstrndup（APRライブラリ関数）。

関連データ構造

  • apr_array_header_t: 文字列を格納する配列構造。
  • request_rec: メモリプール管理に使用。

失敗モードと境界条件

  • バッファの終端に達した際に \r や \0 が見つからない場合は APR_EOF が返されます。
  • buffer_len が不正な場合、バッファ外参照の可能性があります。

保守担当者向けメモ

  • 本関数は mod_cache_disk.c でも同名の関数が定義されており、類似のシリアライズ形式を扱っています。
  • slider は参照渡しであり、呼び出し側で状態が保持されるため、連続的な読み込みが可能です。
