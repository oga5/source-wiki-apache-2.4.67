apr_off_t

定義

  • 種別: typedef
  • 定義箇所: include/util_filter.h:136

目的

ファイルサイズ、オフセット、バイト数などの「ファイルやストリーム上の位置・長さ」を表現するための整数型です。

シグネチャまたは形状

typedef により定義された整数型です。

入力と出力

  • 入力: 該当なし（型定義のため）。
  • 出力: 該当なし。

前提条件と不変条件

  • 負の値を取り得るか、符号なしであるかはプラットフォームおよび APR の定義に依存します。
  • ファイルシステムやネットワークストリームの境界を扱うため、システムがサポートする最大ファイルサイズに準拠します。

アルゴリズムまたは振る舞い

  • 数値としての加算、減算、比較演算が可能です。
  • 主に apr_bucket_brigade を介したデータ転送量や、ファイル読み込み時のオフセット指定に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元/呼び出し先の直接的な関係はありません（型定義のため）。
  • 参照箇所:
  • include/http_core.h: ap_get_limit_req_body, limit_req_body 等で使用。
  • include/http_protocol.h: ap_set_content_length, ap_send_fd 等で使用。
  • include/httpd.h: sent_bodyct, bytes_sent, clength, remaining 等の構造体メンバで使用。

関連データ構造

  • apr_bucket_brigade: フィルタチェーン内でデータを保持する構造体。
  • request_rec: HTTP リクエスト情報を保持する構造体。
  • conn_rec: 接続情報を保持する構造体。

失敗モードと境界条件

  • 型の最大値を超過した場合のオーバーフロー。
  • プラットフォームの off_t 制限に依存するファイルサイズ制限。

保守担当者向けメモ

  • 本シンボルは APR (Apache Portable Runtime) 由来の型であり、クロスプラットフォームでの一貫性を保証するために使用されています。
  • 64bit 環境と 32bit 環境でサイズが異なる可能性があるため、バイナリ互換性が重要な箇所では注意が必要です。
