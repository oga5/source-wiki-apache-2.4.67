ap_add_if_conf

定義

  • 種別: function
  • 定義箇所: server/core.c:644

目的

設定ファイル内の <If>, <ElseIf>, <Else> セクションを core_dir_config 構造体に登録し、条件付き設定の階層を構築します。

シグネチャまたは形状

const char *ap_add_if_conf(apr_pool_t *p, core_dir_config *conf, void *if_config)

入力と出力

  • **入力**:
  • p: 設定割り当て用のメモリプール。
  • conf: 設定を追加する対象の core_dir_config 構造体。
  • if_config: 追加される <If> 等のセクション設定（void *）。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時はエラーメッセージ（const char *）を返します。

前提条件と不変条件

  • conf->sec_if が未初期化の場合は、内部で apr_array_make を用いて初期化されます。
  • <Else> または <ElseIf> を追加する場合、直前のセクションが <If> または <ElseIf> である必要があります。

アルゴリズムまたは振る舞い

1. conf->sec_if が存在しない場合、初期化します。 2. 追加対象の if_config が AP_CONDITION_ELSE フラグを持つ場合（<Else> または <ElseIf> の場合）:

  • conf->sec_if の末尾を確認し、直前のセクションが AP_CONDITION_IF フラグを持っているか検証します。
  • 直前に対応する <If> 等が存在しない場合、エラーメッセージを返して終了します。

    • 3. 検証を通過した場合、if_config を conf->sec_if 配列の末尾に追加します。 4. 成功時は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:2785 （設定解析時のディレクティブ処理から呼び出されます）。
  • **呼び出し先**: ap_get_module_config（core_module の設定取得）、apr_array_make、apr_array_push、APR_ARRAY_IDX。

関連データ構造

  • core_dir_config: ディレクトリ単位の設定構造体。sec_if メンバで条件付き設定を保持します。
  • ap_conf_vector_t: 設定ベクタ。

失敗モードと境界条件

  • <If> 等の先行セクションが存在しない状態で <Else> / <ElseIf> を追加しようとした場合、エラーメッセージが返されます。
  • メモリ割り当て失敗時は apr_pool_t の挙動に従い、プロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は http_core.h で宣言されており、モジュール間での利用が想定されています。
  • sec_if 配列の順序は、設定の評価順序に直結するため、追加時の整合性チェックは重要です。
