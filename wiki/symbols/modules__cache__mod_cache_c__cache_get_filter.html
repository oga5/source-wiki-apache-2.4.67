cache_get_filter

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:377

目的

フィルタチェーン内から、指定されたフィルタレコード（ap_filter_rec_t）とコンテキスト（ctx）を持つ特定のフィルタインスタンスを検索して取得する。

シグネチャまたは形状

ap_filter_t *cache_get_filter(ap_filter_t *next, ap_filter_rec_t *rec);

入力と出力

  • **入力**:
  • next: 検索を開始するフィルタチェーンの先頭（ap_filter_t *）。
  • rec: 検索対象のフィルタ定義レコード（ap_filter_rec_t *）。
  • **出力**:
  • 条件に一致するフィルタインスタンス（ap_filter_t *）。見つからない場合は NULL を返す。

前提条件と不変条件

  • next から始まるフィルタチェーンが有効であること。
  • 検索対象のフィルタは、next->frec が rec と一致し、かつ next->ctx が非 NULL である必要がある。

アルゴリズムまたは振る舞い

1. next が指すフィルタから順に、チェーンの終端まで走査する。 2. 各フィルタにおいて、frec が引数の rec と一致し、かつ ctx が設定されている（非 NULL）かを確認する。 3. 条件を満たす最初のフィルタを見つけた時点で走査を終了し、そのポインタを返す。 4. チェーンの最後まで一致するものがない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c:534（cache->save_filter の初期化時に使用）
  • **呼び出し先**:
  • 直接的な関数呼び出しの証拠はなし。

関連データ構造

  • ap_filter_t: Apache HTTP Server のフィルタチェーンを構成する構造体。
  • ap_filter_rec_t: フィルタの型や名前を定義するレコード構造体。

失敗モードと境界条件

  • next が NULL の場合、直ちに NULL を返す。
  • フィルタチェーン内に該当する rec を持つフィルタが存在しない場合、あるいは ctx が設定されていない場合は NULL を返す。

保守担当者向けメモ

  • 本関数は、キャッシュ関連のフィルタ（CACHE_SAVE 等）が既にフィルタチェーンに挿入されているかを確認し、そのインスタンスを取得するために使用される。
  • ctx の存在チェックが必須条件となっているため、コンテキストが初期化されていないフィルタは無視される点に注意すること。
