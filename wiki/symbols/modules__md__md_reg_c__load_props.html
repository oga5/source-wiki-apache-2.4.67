load_props

定義

  • 種別: function
  • 定義箇所: modules/md/md_reg.c:65

目的

レジストリの永続化層からHTTP/HTTPSプロトコルの設定（httpd.json）を読み込み、md_reg_t 構造体のプロトコル有効フラグを初期化する。

シグネチャまたは形状

static apr_status_t load_props(md_reg_t *reg, apr_pool_t *p)

入力と出力

  • **入力**:
  • reg: レジストリハンドル（md_reg_t）。
  • p: メモリ割り当てに使用するプール。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS。ファイルが存在しない場合は APR_SUCCESS を返し、読み込みエラー時はエラーコードを返す。

前提条件と不変条件

  • reg->store が初期化済みであること。
  • 読み込み対象のファイルは MD_FN_HTTPD_JSON（httpd.json）である。

アルゴリズムまたは振る舞い

1. md_store_load を使用して、ストアから MD_FN_HTTPD_JSON を読み込む。 2. ファイルが存在する場合、JSON内の proto キー配下の http および https の値を解析し、reg->can_http および reg->can_https に反映させる。 3. ファイルが存在しない場合（APR_STATUS_IS_ENOENT）、エラーとは見なさず正常終了する。 4. その他の読み込みエラーが発生した場合、該当ファイルを削除しようと試みる。削除に成功すれば APR_SUCCESS を返し、失敗した場合はエラーを伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_reg.c:125（直接的な呼び出しを確認）
  • **呼び出し先**:
  • md_store_load
  • md_json_has_key, md_json_getb
  • md_log_perror
  • md_store_remove

関連データ構造

  • md_reg_t: レジストリのメイン構造体。
  • md_json_t: 設定ファイルの解析に使用されるJSONオブジェクト。

失敗モードと境界条件

  • **ファイル不在**: ENOENT は正常系として扱われ、フラグは更新されない。
  • **読み込みエラー**: ファイルが破損している等の理由で読み込みに失敗した場合、ログを出力し、該当ファイルを削除（md_store_remove）する。削除失敗時はエラーを返す。

保守担当者向けメモ

  • 本関数は httpd.json の読み込みに特化している。プロトコル設定のスキーマ変更時は、md_json_getb のキー指定を確認すること。
  • 読み込み失敗時にファイルを削除する挙動は、設定の不整合を自動修復するためのものである。
