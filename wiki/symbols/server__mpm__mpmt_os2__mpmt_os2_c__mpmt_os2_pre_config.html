mpmt_os2_pre_config

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:465

目的

OS/2 MPM の設定フェーズにおける初期化処理を実行し、グローバルな MPM 設定変数をデフォルト値で構成する。

シグネチャまたは形状

static int mpmt_os2_pre_config(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • plog: ログ用メモリプール。
  • ptemp: 一時的な割り当て用メモリプール。
  • **出力**:
  • 戻り値: 常に OK を返す。

前提条件と不変条件

  • 本関数は Apache の設定読み込みプロセス中に呼び出される。
  • ap_exists_config_define を使用して、実行環境がシングルプロセスモードかデバッグモードかを判定する。

アルゴリズムまたは振る舞い

1. ONE_PROCESS または DEBUG 定義の有無に基づき、グローバル変数 one_process を設定する。 2. is_graceful フラグを 0 にリセットする。 3. ap_listen_pre_config() を呼び出し、リスナーソケットの初期設定を行う。 4. MPM 関連のグローバル変数を初期化する:

  • ap_daemons_to_start を DEFAULT_START_DAEMON に設定。
  • ap_thread_limit を HARD_THREAD_LIMIT に設定。
  • ap_extended_status を 0 に設定。
  • ap_min_spare_threads を DEFAULT_MIN_SPARE_THREAD に設定。
  • ap_max_spare_threads を DEFAULT_MAX_SPARE_THREAD に設定。

    • 5. ap_sys_privileges_handlers(1) を呼び出し、システム権限ハンドラを有効化する。 6. OK を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_pre_config により登録され、設定フェーズで呼び出される。
  • **呼び出し先**:
  • ap_exists_config_define
  • ap_listen_pre_config
  • ap_sys_privileges_handlers

関連データ構造

  • apr_pool_t (APR メモリプール)

失敗モードと境界条件

  • 本関数自体はエラーを返さない設計となっている。
  • 依存する ap_listen_pre_config 等の内部でエラーが発生した場合の挙動は、それらの関数に依存する。

保守担当者向けメモ

  • 本関数は OS/2 MPM の設定の起点である。設定値の変更が必要な場合は、mpm_default.h で定義されている定数を確認すること。
