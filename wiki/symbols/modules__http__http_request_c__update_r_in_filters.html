update_r_in_filters

定義

  • 種別: function
  • 定義箇所: modules/http/http_request.c:64

目的

フィルタチェーン内の各フィルタが保持する request_rec ポインタを、古いリクエスト構造から新しいリクエスト構造へ一括更新するために使用されます。

シグネチャまたは形状

static void update_r_in_filters(ap_filter_t *f, request_rec *from, request_rec *to);

入力と出力

  • f: 更新対象のフィルタチェーンの先頭ポインタ。
  • from: 現在フィルタが参照している古い request_rec 構造体。
  • to: フィルタが参照すべき新しい request_rec 構造体。
  • 出力: なし（副作用としてフィルタチェーン内の f->r を書き換える）。

前提条件と不変条件

  • f が NULL の場合、何も行われません。
  • フィルタチェーン内の f->r が from と一致する場合のみ、to に置き換えられます。

アルゴリズムまたは振る舞い

1. フィルタチェーン f を末尾まで順次走査します。 2. 各フィルタノードにおいて、f->r が from と一致するか確認します。 3. 一致する場合、f->r を to に更新します。 4. f = f->next を繰り返し、チェーン全体を処理します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/http/http_request.c:140, 141 (エラー処理時)
  • modules/http/http_request.c:663, 664 (内部リダイレクト時)
  • modules/http/http_request.c:735, 736 (サブクエスト終了時)
  • 呼び出し先:
  • 外部関数呼び出しは行いません（フィルタチェーンの走査のみ）。

関連データ構造

  • ap_filter_t: フィルタチェーンを構成するノード。r メンバ（request_rec）を保持します。
  • request_rec: HTTP リクエストのコンテキストを保持する構造体。

失敗モードと境界条件

  • f が NULL の場合、安全に終了します。
  • フィルタチェーン内に from と一致する request_rec が存在しない場合、チェーンは変更されません。

保守担当者向けメモ

  • 本関数は、内部リダイレクトやエラーハンドリングにおいて、古いリクエストプールに関連付けられたフィルタが誤ったコンテキストを参照し続けることを防ぐために不可欠です。
  • フィルタチェーンの構造を変更するものではなく、あくまでポインタの付け替えのみを行います。
