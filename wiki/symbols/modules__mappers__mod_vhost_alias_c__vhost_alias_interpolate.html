vhost_alias_interpolate

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_vhost_alias.c:253

目的

ホスト名やIPアドレスをテンプレート文字列（VirtualDocumentRoot 等）に埋め込み、リクエストに対応する物理的なディレクトリパスを構築する。

シグネチャまたは形状

void vhost_alias_interpolate(request_rec *r, const char *name,
                             const char *map, const char *uri)

入力と出力

  • **入力**:
  • r: リクエストレコード。
  • name: 補間対象のホスト名またはIPアドレス。
  • map: テンプレート文字列（% 記法を含む）。
  • uri: リクエストされたURI。
  • **出力**:
  • r->filename: 生成された物理パスが格納される。
  • r のコンテキスト情報およびドキュメントルートが更新される。

前提条件と不変条件

  • map 内のテンプレート構文（% 記法）は、呼び出し前に vhost_alias_set 等で検証済みであること。
  • name はドットで区切られた文字列として処理される。
  • HUGE_STRING_LEN を超えるパス生成時の挙動は vhost_alias_checkspace に依存する。

アルゴリズムまたは振る舞い

1. **解析**: name をドットで分割し、最大19個のパーツとして保持する。 2. **テンプレート展開**: map を走査し、以下の指定子を処理する。

  • %%: リテラルの % に置換。
  • %p: サーバーポート番号に置換。
  • %N, %M 等: ホスト名/IPのパーツを抽出（1ベースのインデックス指定）。

    • 3. **正規化**: 生成されたパスの末尾が / の場合は削除する。 4. **パス結合**: r->filename が既に存在する場合はその末尾に結合し、存在しない場合は新規作成する。 5. **適用**: ap_set_context_info および ap_set_document_root を呼び出し、リクエストに反映する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_vhost_alias.c:428（mva_translate 内から呼び出される）。
  • **呼び出し先**:
  • vhost_alias_checkspace: バッファ溢れ防止。
  • ap_get_server_port: ポート番号取得。
  • apr_tolower: 文字列の小文字化。
  • apr_pstrcat, apr_pstrdup: メモリ確保と文字列結合。
  • ap_set_context_info, ap_set_document_root: リクエスト設定の更新。

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • HUGE_STRING_LEN: パス構築用バッファの最大長。

失敗モードと境界条件

  • N や M がパーツ数を超える場合、_（アンダースコア）が代用される。
  • map が空の場合、空のパスが生成される。
  • r->filename が既に設定されている場合、その値がベースパスとして使用される。

保守担当者向けメモ

  • name の解析において dots[ndots++] = name-1 というポインタ操作を行っているため、変更時は注意が必要。
  • Directory 単位の設定オーバーライドには対応していない。
