get_handles_from_parent

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:270

目的

親プロセスからパイプ経由で渡された、子プロセスの実行に必要な同期オブジェクトおよび共有メモリのハンドルを取得・初期化します。

シグネチャまたは形状

static void get_handles_from_parent(server_rec *s, HANDLE *child_exit_event,
                                    apr_proc_mutex_t **child_start_mutex,
                                    apr_shm_t **scoreboard_shm)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • pipe: グローバル変数（winnt_rewrite_argsで設定された標準入力ハンドル）。
  • **出力**:
  • child_exit_event: 親プロセスから受け取った終了イベントハンドル。
  • child_start_mutex: 初期化済みのプロセス間ミューテックス。
  • scoreboard_shm: 初期化および再接続済みの共有メモリ領域。

前提条件と不変条件

  • pipe が有効なハンドルとしてオープンされていること。
  • 呼び出し側で s->process->pool が有効であること。
  • 親プロセスが send_handles_to_child を通じて、定義された順序（readyイベント、exitイベント、mutex、scoreboard）でハンドルをパイプに書き込んでいること。

アルゴリズムまたは振る舞い

1. **Readyイベントの取得**: パイプから ready_event を読み取り、シグナルを送ってからクローズします。 2. **終了イベントの取得**: パイプから child_exit_event を読み取ります。 3. **ミューテックスの取得**: パイプからミューテックスハンドルを読み取り、apr_os_proc_mutex_put を用いて APR オブジェクトに変換します。 4. **スコアボードの取得**: パイプから共有メモリハンドルを読み取り、apr_os_shm_put で変換します。 5. **スコアボードの再接続**: ap_reopen_scoreboard を呼び出し、共有メモリをプロセス空間にマッピングします。 6. **スコアボードの初期化**: ap_init_scoreboard を呼び出し、共有メモリ内のポインタ配列を再リンクします。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/mpm_winnt.c:1670
  • **呼び出し先**:
  • ReadFile (Windows API)
  • SetEvent, CloseHandle (Windows API)
  • apr_os_proc_mutex_put, apr_os_shm_put (APR)
  • ap_reopen_scoreboard, ap_init_scoreboard (Apache MPM)
  • ap_log_error (Apache Logging)

関連データ構造

  • server_rec: サーバー設定情報。
  • apr_proc_mutex_t: プロセス間同期用ミューテックス。
  • apr_shm_t: プロセス間共有メモリ。

失敗モードと境界条件

  • 各 ReadFile でデータの読み取りに失敗、またはサイズが不一致の場合、APEXIT_CHILDINIT でプロセスを異常終了させます。
  • apr_os_proc_mutex_put や apr_os_shm_put が失敗した場合、エラーログを出力し APEXIT_CHILDINIT で終了します。
  • スコアボードの再オープンまたは初期化に失敗した場合、同様に終了します。

保守担当者向けメモ

  • 本関数は子プロセスの初期化フェーズで一度だけ実行されます。
  • パイプからの読み取り順序は send_handles_to_child と厳密に一致させる必要があります。
  • pipe 変数はグローバルスコープで管理されているため、並列実行は想定されていません。
