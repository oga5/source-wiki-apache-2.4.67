sig_hup

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2_child.c:467

目的

SIGHUP シグナルを受信した際に、子プロセスに対してグレースフルシャットダウン（緩やかな終了）を要求するためのシグナルハンドラです。

シグネチャまたは形状

static void sig_hup(int sig)

入力と出力

  • 入力: int sig（受信したシグナル番号）
  • 出力: なし（void）

前提条件と不変条件

  • 本関数はシグナルハンドラとして呼び出されることを前提としています。
  • グローバル変数 shutdown_pending および is_graceful が定義されている必要があります。

アルゴリズムまたは振る舞い

1. shutdown_pending を 1 に設定し、終了処理が保留中であることを示します。 2. is_graceful を 1 に設定し、強制終了ではなくグレースフルシャットダウンを行うようフラグを立てます。

呼び出し元と呼び出し先

  • 呼び出し元: OS/2 のシグナル配送メカニズムにより、SIGHUP 受信時に呼び出されます。
  • 呼び出し先: 直接的な関数呼び出しはありません。
  • 参照元: server/mpm/mpmt_os2/mpmt_os2_child.c:486 にて、sa.sa_handler に本関数が割り当てられています。

関連データ構造

  • shutdown_pending（グローバル変数）
  • is_graceful（グローバル変数）

失敗モードと境界条件

  • シグナルハンドラ内での実行であるため、非同期シグナル安全な操作のみを行う必要があります。本関数は単純なフラグ代入のみを行うため、この制約を満たしています。

保守担当者向けメモ

  • 本関数は set_signals 関数内でシグナルハンドラとして登録されます。
  • is_graceful が 1 に設定されることで、メインループ側で接続の新規受け入れを停止し、既存の処理を完了させてから終了する挙動がトリガーされます。
