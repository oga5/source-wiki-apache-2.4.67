parse_state_t

定義

  • 種別: enum
  • 定義箇所: modules/filters/mod_include.c:131

目的

SSI（Server Side Includes）ディレクティブを解析するためのステートマシンにおける、現在の解析状態を保持する。

シグネチャまたは形状

typedef enum {
    PARSE_PRE_HEAD,
    PARSE_HEAD,
    PARSE_DIRECTIVE,
    PARSE_DIRECTIVE_POSTNAME,
    PARSE_DIRECTIVE_TAIL,
    PARSE_DIRECTIVE_POSTTAIL,
    PARSE_PRE_ARG,
    PARSE_ARG,
    PARSE_ARG_NAME,
    PARSE_ARG_POSTNAME,
    PARSE_ARG_EQ,
    PARSE_ARG_PREVAL,
    PARSE_ARG_VAL,
    PARSE_ARG_VAL_ESC,
    PARSE_ARG_POSTVAL,
    PARSE_TAIL,
    PARSE_TAIL_SEQ,
    PARSE_EXECUTE
} parse_state_t;

入力と出力

  • 入力: ストリームから読み込まれた文字データおよびバケット。
  • 出力: 状態遷移の結果として、後続の解析処理（引数抽出やディレクティブ実行）へ制御を渡す。

前提条件と不変条件

  • mod_include.c 内の解析ループにおいて、現在の解析コンテキスト（ssi_internal_ctx 等）の一部として保持される。
  • 解析は PARSE_PRE_HEAD から開始され、最終的に PARSE_EXECUTE へ到達することでディレクティブの処理が確定する。

アルゴリズムまたは振る舞い

  • SSI タグ（<!--#...-->）の開始シーケンス検出から、ディレクティブ名、引数名、値の抽出、および終了シーケンスの確認までを段階的に追跡する。
  • バッファ境界をまたぐ解析において、現在の状態を維持することでストリーム処理を継続可能にする。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_include.c 内の解析ロジック（send_parsed_content 等）。
  • 呼び出し先: 状態遷移に基づき、各引数抽出関数やディレクティブハンドラが呼び出される。

関連データ構造

  • ssi_internal_ctx: この列挙型をメンバとして保持し、解析の進捗を管理する。

失敗モードと境界条件

  • バッファ境界でのタグ分割（find_partial_start_sequence）において、状態の正確な保持が不可欠である。

保守担当者向けメモ

  • PARSE_ARG_VAL_ESC はエスケープされた文字の処理に使用される。
  • PARSE_EXECUTE は解析が完了し、ディレクティブを実行する準備が整った状態を示す。
  • 状態遷移のロジックは mod_include.c のメインループに密結合しているため、変更時はステートマシンの網羅的なテストが必要である。
