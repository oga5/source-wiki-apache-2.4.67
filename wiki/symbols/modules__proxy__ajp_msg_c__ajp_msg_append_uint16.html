ajp_msg_append_uint16

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_msg.c:285

目的

AJPメッセージのバッファ末尾に、16ビットの符号なし整数をネットワークバイトオーダー（ビッグエンディアン）で追加します。

シグネチャまたは形状

apr_status_t ajp_msg_append_uint16(ajp_msg_t *msg, apr_uint16_t value)

入力と出力

  • **入力**:
  • msg: 操作対象の ajp_msg_t 構造体へのポインタ。
  • value: 追加する apr_uint16_t 型の数値。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • バッファの容量不足時は ajp_log_overflow の戻り値（エラーコード）を返します。

前提条件と不変条件

  • msg->len は現在のバッファ内の有効データ長を示しており、この関数は msg->len + 2 が msg->max_size を超えないことを前提とします。
  • 成功時には msg->len が 2 増加し、バッファの更新後の位置を指すようになります。

アルゴリズムまたは振る舞い

1. 現在のデータ長 len を取得します。 2. len + 2 が msg->max_size を超えるか検証し、超える場合は ajp_log_overflow を呼び出してエラーを返します。 3. value をビッグエンディアン形式に分解し、msg->buf の len および len + 1 の位置に書き込みます。

  • 上位8ビット: (value >> 8) & 0xFF
  • 下位8ビット: value & 0xFF

    • 4. msg->len を 2 増加させます。 5. APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/ajp_header.c: リクエストヘッダーの構築時にポート番号やヘッダー数、環境変数の追加で使用。
  • modules/proxy/ajp_msg.c: メッセージの末尾マークや長さ情報の追加で使用。
  • **呼び出し先**:
  • ajp_log_overflow (エラー発生時)

関連データ構造

  • ajp_msg_t: AJPメッセージを保持する構造体。バッファ本体 (buf)、現在の長さ (len)、最大サイズ (max_size) を含みます。

失敗モードと境界条件

  • **バッファオーバーフロー**: len + 2 > max_size となる場合、書き込みを行わずにエラーを返します。

保守担当者向けメモ

  • 本関数はネットワークバイトオーダーへの変換を明示的に行っています。プラットフォームのエンディアンに関わらず、AJPプロトコル仕様に従ったビッグエンディアンで書き込まれます。
