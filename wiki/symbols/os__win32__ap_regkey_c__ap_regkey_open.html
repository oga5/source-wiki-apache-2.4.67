ap_regkey_open

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:58

目的

Windows レジストリキーを開く、または作成し、そのハンドルを APR プールに関連付けて管理する。

シグネチャまたは形状

apr_status_t ap_regkey_open(ap_regkey_t **newkey,
                            const ap_regkey_t *parentkey,
                            const char *keyname,
                            apr_int32_t flags,
                            apr_pool_t *pool)

入力と出力

  • **入力**:
  • parentkey: 親レジストリキーへのポインタ。
  • keyname: 開く/作成するキーの名称。
  • flags: 操作フラグ（APR_READ, APR_WRITE, APR_CREATE, APR_EXCL）。
  • pool: リソース管理に使用する APR プール。
  • **出力**:
  • newkey: 成功時に初期化された ap_regkey_t 構造体へのポインタが格納される。
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • parentkey は有効な ap_regkey_t を指している必要がある。
  • pool は newkey の生存期間を管理するために使用される。
  • flags に APR_CREATE が含まれる場合、キーが存在しなければ作成される。
  • flags に APR_EXCL が含まれる場合、キーが既に存在するとエラー（APR_EEXIST）となる。

アルゴリズムまたは振る舞い

1. flags に基づき、Win32 API のアクセス権限（KEY_QUERY_VALUE, KEY_READ, KEY_WRITE）を決定する。 2. 実行環境に応じて Unicode (RegCreateKeyExW / RegOpenKeyExW) または ANSI (RegCreateKeyEx / RegOpenKeyEx) の Win32 API を選択して呼び出す。 3. Win32 API の戻り値が成功でない場合、APR_FROM_OS_ERROR を用いてエラーを返却する。 4. APR_EXCL が指定されており、かつキーが既に存在していた場合は RegCloseKey を実行し APR_EEXIST を返却する。 5. 成功時、pool から ap_regkey_t 用のメモリを確保し、ハンドルを格納する。 6. apr_pool_cleanup_register を呼び出し、プール破棄時に regkey_cleanup が自動実行されるよう登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/win32/mod_win32.c
  • server/mpm/winnt/nt_eventlog.c
  • server/mpm/winnt/service.c
  • **呼び出し先**:
  • Win32 API: RegCreateKeyEx, RegOpenKeyEx, RegCloseKey
  • APR: apr_palloc, apr_pool_cleanup_register, apr_conv_utf8_to_ucs2 (Unicode環境時)
  • 内部関数: regkey_cleanup (クリーンアップハンドラとして登録)

関連データ構造

  • ap_regkey_t: レジストリキーハンドル (HKEY) と関連付けられた apr_pool_t を保持する構造体。

失敗モードと境界条件

  • apr_conv_utf8_to_ucs2 が失敗した場合、変換エラーを返却する。
  • keyname が長すぎる場合、APR_ENAMETOOLONG を返却する。
  • Win32 API が失敗した場合、対応する OS エラーコードを apr_status_t に変換して返却する。
  • APR_EXCL 指定時にキーが既に存在する場合、APR_EEXIST を返却する。

保守担当者向けメモ

  • 本関数は ap_regkey_t のライフサイクル管理の起点であり、regkey_cleanup と対になる。
  • Unicode/ANSI の切り替えは APR_HAS_UNICODE_FS および APR_HAS_ANSI_FS マクロに依存している。
