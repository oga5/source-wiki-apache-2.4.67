server_maintenance

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2_child.c:416

目的

子プロセス内のスレッドプールを監視し、アイドルスレッド数と保留中の接続数に基づいて、ワーカーの生成または終了を動的に調整する管理スレッドのメインループ。

シグネチャまたは形状

static void server_maintenance(void *vpArg)

入力と出力

  • **入力**: vpArg (voidポインタ、未使用)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • pchild プールが初期化されていること。
  • shutdown_event イベントセマフォが有効であること。
  • ap_scoreboard_image が共有メモリ上にマップされ、アクセス可能であること。
  • OS/2 の名前付きキュー /queues/httpd/work.<pid> が存在すること。

アルゴリズムまたは振る舞い

1. **キューのオープン**: DosOpenQueue を使用して、現在のプロセスIDに対応する作業キューを開く。失敗時はエラーログを出力して終了する。 2. **監視ループ**: shutdown_event がシグナルされるか、500ms のタイムアウトが発生するまで以下の処理を繰り返す。

  • **アイドル数集計**: スコアボードを走査し、SERVER_READY 状態のスレッド数をカウントする。
  • **キュー確認**: DosQueryQueue で保留中の接続数を取得する。
  • **スレッド生成**: 必要数（ap_min_spare_threads - アイドル数 + 保留数）が正の場合、add_worker() を呼び出して新規ワーカーを生成する。
  • **スレッド終了**: アイドル数が ap_max_spare_threads + 保留数 を超える場合、DosWriteQueue に WORKTYPE_EXIT を書き込み、ワーカーの終了を促す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/mpmt_os2/mpmt_os2_child.c 内の _beginthread（スレッド起動時）。
  • **呼び出し先**: add_worker()（スレッド生成）、DosOpenQueue, DosQueryQueue, DosWriteQueue, DosWaitEventSem（OS/2 API）。

関連データ構造

  • ap_scoreboard_image: スレッドのステータスを保持する共有メモリ構造体。
  • workq: OS/2 の FIFO キュー。
  • shutdown_event: プロセス終了を通知するイベントセマフォ。

失敗モードと境界条件

  • **キューオープン失敗**: DosOpenQueue が失敗した場合、ログを出力してスレッドを即時終了する。
  • **スレッド上限**: HARD_THREAD_LIMIT を超えるスレッド生成は行われない（add_worker 側の実装に依存）。
  • **タイムアウト**: DosWaitEventSem は 500ms ごとにタイムアウトし、ループを継続して再評価を行う。

保守担当者向けメモ

  • 本関数は _beginthread で起動されるため、スタックサイズやスレッドコンテキストの管理に注意すること。
  • ap_min_spare_threads および ap_max_spare_threads の設定値がスレッド増減の閾値となる。
  • OS/2 API のエラーコードは APR_FROM_OS_ERROR(rc) を介して APR エラーに変換されている。
