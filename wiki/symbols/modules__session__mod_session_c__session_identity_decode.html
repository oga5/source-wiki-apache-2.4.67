session_identity_decode

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:391

目的

URLエンコードされた文字列形式のセッションデータを解析し、session_rec 構造体の entries テーブルへ展開する。

シグネチャまたは形状

apr_status_t session_identity_decode(request_rec *r, session_rec *z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報。
  • z: デコード対象のセッション構造体。z->encoded にエンコード済み文字列が格納されている必要がある。
  • **出力**:
  • apr_status_t: 成功時は OK を返す。
  • 副作用: z->entries にキー・値ペアが設定され、z->encoded は NULL にリセットされる。

前提条件と不変条件

  • z->encoded が NULL の場合、処理は即座に終了する。
  • z->entries は初期化済みであること。
  • セッションデータは key=value のペアが & で連結された形式であること。

アルゴリズムまたは振る舞い

1. z->encoded が存在する場合、リクエストプールを使用して文字列を複製する。 2. apr_strtok を用いて & 区切りでペアを順次抽出する。 3. 各ペアを = で分割し、キーと値を取得する。 4. 値が空の場合は apr_table_unset で該当キーを削除する。 5. 値が存在する場合、ap_unescape_urlencoded でデコードを行い、キーが SESSION_EXPIRY であれば z->expiry に数値を格納し、それ以外は apr_table_set でテーブルに保存する。 6. 処理完了後、z->encoded を NULL にクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session.c:480 (内部処理)
  • modules/session/mod_session.c:694 (フック登録時)
  • **呼び出し先**:
  • apr_pstrdup, apr_strtok, apr_table_unset, apr_table_set, apr_atoi64 (APRユーティリティ)
  • ap_unescape_urlencoded (Apache HTTPDコア)

関連データ構造

  • session_rec: セッション情報を保持する構造体。
  • request_rec: リクエストごとのメモリプール（r->pool）を提供。

失敗モードと境界条件

  • z->encoded が空または NULL の場合は OK を返して終了する。
  • key が存在しない、あるいは val が空の場合はテーブルから該当キーを削除する。
  • デコード処理（ap_unescape_urlencoded）が失敗したペアは無視される。

保守担当者向けメモ

  • 本関数はデフォルトのシリアライズ形式を扱う。カスタム形式が必要な場合は、別の session_decode フック実装を検討すること。
  • SESSION_EXPIRY キーは特別扱いされ、セッションの有効期限として z->expiry に直接反映される。
