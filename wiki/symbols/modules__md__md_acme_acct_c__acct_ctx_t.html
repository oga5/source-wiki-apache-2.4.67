acct_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme_acct.c:403

目的

ACME アカウント関連の操作（検索、登録、更新など）を実行する際に、必要なコンテキスト情報とリソースを保持するための構造体です。

シグネチャまたは形状

typedef struct {
    md_acme_t *acme;        /* ACME サーバーとの通信セッション */
    apr_pool_t *p;          /* 操作に使用するメモリプール */
    const char *agreement;  /* 同意済みの利用規約（ToS）URL */
    const char *eab_kid;    /* 外部アカウントバインディング用キーID */
    const char *eab_hmac;   /* 外部アカウントバインディング用HMACキー */
} acct_ctx_t;

入力と出力

本構造体は関数間で「バトン（baton）」として受け渡されるコンテキストオブジェクトです。

  • **入力**: ACME セッション、メモリプール、およびオプションの EAB 設定情報。
  • **出力**: 構造体自体が変更されることはなく、主に各処理関数が ACME サーバーとの通信やデータ検証を行うための参照先として機能します。

前提条件と不変条件

  • acme および p は有効なポインタである必要があります。
  • 保持される文字列（agreement, eab_kid, eab_hmac）は、関連付けられたメモリプール p の生存期間に依存します。

アルゴリズムまたは振る舞い

主に md_store を介したアカウント検索や、ACME サーバーとの通信処理において、コールバック関数や補助関数へ設定情報を伝達するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_acct.c 内の各アカウント操作関数（acct_find_and_verify 等）でインスタンス化され、baton として渡されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • md_acme_t: ACME プロトコル通信のセッション状態を管理。
  • apr_pool_t: メモリ管理のライフサイクルを制御。

失敗モードと境界条件

  • acme ポインタが NULL の場合、ACME サーバーとの通信を伴う処理でクラッシュする可能性があります。
  • メモリプール p が破棄された後に本構造体を参照すると、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は md_acme_acct.c 内のローカルなスコープで頻繁に生成・利用されています。
  • EAB（External Account Binding）設定が必要な場合、eab_kid と eab_hmac が適切に設定されていることを確認してください。
