ap_lua_mapped_handler_spec

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_vmprep.h:88

目的

Apache HTTP Server において、特定の URI パターンやリクエストに対して実行される Lua ハンドラの構成情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    const char *function_name;
    const char *file_name;
    int scope;
    ap_regex_t *uri_pattern;
    const char *bytecode;
    apr_size_t bytecode_len;
    int codecache;
} ap_lua_mapped_handler_spec;

入力と出力

  • **入力**: 設定ファイルから読み込まれたスクリプトパス、関数名、スコープ、キャッシュ設定、およびオプションのバイトコードデータ。
  • **出力**: mod_lua がリクエスト処理時に参照するハンドラ実行仕様。

前提条件と不変条件

  • file_name または bytecode のいずれかが有効なスクリプトソースとして提供される必要があります。
  • scope は AP_LUA_SCOPE_* 定数に従う必要があります。

アルゴリズムまたは振る舞い

  • mod_lua はリクエスト発生時にこの構造体を参照し、指定された file_name または bytecode を用いて Lua VM を準備します。
  • uri_pattern が設定されている場合、リクエスト URI との照合に使用されます。
  • codecache 設定に基づき、スクリプトのコンパイル済みバイトコードをキャッシュするかどうかが決定されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_config.c: 設定ファイル解析時に構造体の割り当てと初期化が行われます。
  • modules/lua/mod_lua.c: リクエスト処理のフック実行時に、登録されたハンドラ仕様として参照されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。実行ロジックは mod_lua.c 等の呼び出し側が本構造体のフィールドを参照して制御します。

関連データ構造

  • ap_lua_vm_spec: VM の生成パラメータを定義する関連構造体。
  • ap_regex_t: URI パターンマッチングに使用される正規表現構造体。

失敗モードと境界条件

  • bytecode が指定されている場合、bytecode_len が正しく設定されていないとロード時にエラーが発生する可能性があります。
  • メモリ不足時には apr_pcalloc による割り当てが失敗します。

保守担当者向けメモ

  • 本構造体は mod_lua の設定層と実行層を繋ぐ重要なインターフェースです。フィールドを追加・変更する場合は、lua_config.c での初期化ロジックと mod_lua.c での参照ロジックの両方を更新してください。
