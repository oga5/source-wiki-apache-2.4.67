ap_replace_stderr_log

定義

  • 種別: function
  • 定義箇所: server/log.c:248

目的

標準エラー出力（stderr）を、指定されたファイルパスのログファイルへリダイレクトします。

シグネチャまたは形状

apr_status_t ap_replace_stderr_log(apr_pool_t *p, const char *fname)

入力と出力

  • **入力**:
  • p: パス解決に使用するメモリプール。
  • fname: リダイレクト先となるファイルパス。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS、失敗時はエラーコードを返します。

前提条件と不変条件

  • fname は ap_server_root_relative によりサーバーのルートディレクトリからの相対パスとして解決されます。
  • stderr_pool が未設定の場合、一時的に引数 p が使用されます。
  • 処理の最後に stderr_pool が引数 p と一致する場合は NULL にリセットされます。

アルゴリズムまたは振る舞い

1. ap_server_root_relative を用いて fname を絶対パスに変換します。失敗時は APR_EBADPATH を返します。 2. apr_file_open を使用し、追記・書き込み・作成モードで対象ファイルを開きます。 3. apr_file_open_stderr で現在の標準エラー出力のハンドルを取得します。 4. apr_file_flush で既存の stderr バッファをフラッシュします。 5. apr_file_dup2 を使用して、標準エラー出力を開いたファイルへ複製（リダイレクト）します。 6. 複製成功後、一時的に開いたファイルハンドルを閉じます。 7. 処理中にエラーが発生した場合は、ap_log_error を通じてログを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/main.c:679 (ap_replace_stderr_log(process->pool, temp_error_log);)
  • **呼び出し先**:
  • ap_server_root_relative
  • ap_log_error
  • apr_file_open
  • apr_file_open_stderr
  • apr_file_flush
  • apr_file_dup2
  • apr_file_close

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_file_t: ファイルハンドルおよび標準エラー出力の保持に使用。
  • stderr_log: server/log.c 内で管理される標準エラー出力用のグローバルなファイル記述子。

失敗モードと境界条件

  • パス解決失敗: fname が無効な場合、APR_EBADPATH を返し終了します。
  • ファイルオープン失敗: 指定されたパスへの書き込み権限がない場合やパスが存在しない場合、apr_file_open のエラーコードを返します。
  • リダイレクト失敗: apr_file_dup2 が失敗した場合、そのエラーコードを返します。

保守担当者向けメモ

  • stderr_pool の生存期間に関する注意書きがソース内に存在します。stderr_pool は次のリサイクルまで破棄できない可能性があるため、呼び出し側のプール管理には注意が必要です。
  • APLOGNO(00085) から 00087 までのエラー番号が割り当てられています。
