ap_regkey_value_set

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:283

目的

Windows レジストリの文字列値を設定します。指定されたフラグに応じて、値を REG_SZ または REG_EXPAND_SZ として書き込みます。

シグネチャまたは形状

apr_status_t ap_regkey_value_set(ap_regkey_t *key,
                                 const char *valuename,
                                 const char *value,
                                 apr_int32_t flags,
                                 apr_pool_t *pool)

入力と出力

  • **入力**:
  • key: 操作対象のレジストリキー（ap_regkey_t 構造体）。
  • valuename: 設定する値の名前。
  • value: 設定する文字列データ。
  • flags: AP_REGKEY_EXPAND を指定すると REG_EXPAND_SZ として扱われる。
  • pool: メモリ割り当てに使用する APR プール。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。失敗時は apr_status_t 型のエラーコードを返します。

前提条件と不変条件

  • key は有効な ap_regkey_t ハンドルである必要があります。
  • APR_HAS_UNICODE_FS が有効な場合、内部で UTF-8 から UCS-2 への変換が行われます。
  • pool は、変換後の文字列を保持するために十分なメモリを確保できる必要があります。

アルゴリズムまたは振る舞い

1. flags に基づき、レジストリ型を REG_EXPAND_SZ または REG_SZ に決定します。 2. OS 環境（Unicode/ANSI）に応じて以下の処理を行います。

  • **Unicode 環境**: apr_conv_utf8_to_ucs2 を使用して入力文字列を変換し、RegSetValueExW を呼び出します。
  • **ANSI 環境**: RegSetValueEx を呼び出します。

    • 3. Win32 API の戻り値が ERROR_SUCCESS でない場合、APR_FROM_OS_ERROR を介してエラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/nt_eventlog.c:65 での呼び出しが確認されています。
  • **呼び出し先**:
  • apr_palloc（メモリ割り当て）
  • apr_conv_utf8_to_ucs2（Unicode 変換）
  • RegSetValueExW / RegSetValueEx（Win32 API）

関連データ構造

  • ap_regkey_t: レジストリキーハンドルを保持するコンテナ。

失敗モードと境界条件

  • 文字列変換に失敗した場合、APR_ENAMETOOLONG または変換エラーが返されます。
  • レジストリへの書き込み権限がない場合やキーが存在しない場合、Win32 エラーが apr_status_t に変換されて返されます。

保守担当者向けメモ

  • 本関数は REG_SZ および REG_EXPAND_SZ の書き込みに特化しています。バイナリデータや配列データの書き込みには ap_regkey_value_raw_set や ap_regkey_value_array_set を使用してください。
  • Unicode 変換時のバッファサイズ計算は、apr_conv_utf8_to_ucs2 の仕様に依存しているため、変更時は注意が必要です。
