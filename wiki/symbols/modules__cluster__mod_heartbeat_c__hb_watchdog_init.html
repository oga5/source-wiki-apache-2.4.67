hb_watchdog_init

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartbeat.c:115

目的

mod_watchdog の初期化フェーズにおいて、MPM（Multi-Processing Module）のハードリミット設定を取得し、モジュールのコンテキストに格納する。

シグネチャまたは形状

static int hb_watchdog_init(server_rec *s, const char *name, apr_pool_t *pool)

入力と出力

  • **入力**:
  • s: 現在のサーバー設定構造体。
  • name: ウォッチドッグ名（使用されず）。
  • pool: 初期化用メモリプール。
  • **出力**:
  • 戻り値: 常に OK を返す。
  • 副作用: hb_ctx_t 構造体内の thread_limit および server_limit が更新される。

前提条件と不変条件

  • s->module_config から heartbeat_module に対応する hb_ctx_t が取得可能であること。
  • MPM がロードされており、ap_mpm_query が有効な値を返せる状態であること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を呼び出し、サーバー設定から hb_ctx_t 構造体を取得する。 2. ap_mpm_query を使用して、現在の MPM のスレッド上限値 (AP_MPMQ_HARD_LIMIT_THREADS) を取得し、ctx->thread_limit に代入する。 3. ap_mpm_query を使用して、現在の MPM のプロセス上限値 (AP_MPMQ_HARD_LIMIT_DAEMONS) を取得し、ctx->server_limit に代入する。 4. OK を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_watchdog の初期化フック (modules/cluster/mod_heartbeat.c:154 にて ap_hook_watchdog_init に登録)。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_mpm_query

関連データ構造

  • hb_ctx_t: モジュールの設定と実行時状態を保持する構造体。
  • server_rec: Apache のサーバー設定構造体。

失敗モードと境界条件

  • ap_mpm_query が失敗した場合の戻り値チェックは行われていない。
  • ctx が NULL の場合のガード句は存在しない。

保守担当者向けメモ

  • 本関数は mod_watchdog の初期化時に一度だけ実行されることを想定している。
  • MPM の制限値は静的な値として取得されるため、実行中に MPM の設定が変更された場合、本関数は再実行されない限り追従しない。
