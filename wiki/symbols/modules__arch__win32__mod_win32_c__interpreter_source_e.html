interpreter_source_e

定義

  • 種別: enum
  • 定義箇所: modules/arch/win32/mod_win32.c:40

目的

Windows環境におけるCGIスクリプトの実行時、どの手法でインタープリタ（実行バイナリ）を特定するかという解決戦略を定義する列挙型です。

シグネチャまたは形状

typedef enum {
    INTERPRETER_SOURCE_UNSET,
    INTERPRETER_SOURCE_REGISTRY_STRICT,
    INTERPRETER_SOURCE_REGISTRY,
    INTERPRETER_SOURCE_SHEBANG
} interpreter_source_e;

入力と出力

  • 本シンボル自体は設定値の選択肢を定義する型です。
  • win32_dir_conf 構造体内の script_interpreter_source メンバとして保持され、設定値として利用されます。

前提条件と不変条件

  • ScriptInterpreterSource ディレクティブの設定値に基づき、いずれかの値が割り当てられます。
  • INTERPRETER_SOURCE_UNSET は初期状態を示します。

アルゴリズムまたは振る舞い

  • INTERPRETER_SOURCE_REGISTRY_STRICT: レジストリ検索を必須とし、見つからない場合はエラーとします。
  • INTERPRETER_SOURCE_REGISTRY: レジストリ検索を試行し、失敗した場合はフォールバックを許容します。
  • INTERPRETER_SOURCE_SHEBANG: ファイル先頭のシバン（#!）行を解析してインタープリタを特定します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/arch/win32/mod_win32.c 内の win32_dir_conf 構造体定義（51行目）で型として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • win32_dir_conf: 本列挙型を保持するディレクトリ単位の設定構造体。

失敗モードと境界条件

  • 設定が INTERPRETER_SOURCE_UNSET のまま実行された場合、モジュール内のロジックでデフォルトの挙動（通常はレジストリ検索）が適用される必要があります。

保守担当者向けメモ

  • 新しいインタープリタ解決戦略を追加する場合は、本列挙型に要素を追加し、set_interpreter_source ハンドラおよび ap_cgi_build_command 内のロジックを更新してください。
