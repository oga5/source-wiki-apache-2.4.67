worker_note_child_killed

定義

  • 種別: function
  • 定義箇所: server/mpm/worker/worker.c:390

目的

子プロセスの終了をスコアボードおよびシステム状態に反映させるためのコールバック関数です。

シグネチャまたは形状

static void worker_note_child_killed(int childnum, pid_t pid, ap_generation_t gen)

入力と出力

  • **入力**:
  • childnum: スコアボード上のインデックス（該当しない場合は -1）。
  • pid: 終了したプロセスのプロセスID。
  • gen: プロセスの世代番号。
  • **出力**: なし（void）。

前提条件と不変条件

  • childnum が -1 でない場合、ap_scoreboard_image が有効であり、指定されたインデックスにアクセス可能である必要があります。

アルゴリズムまたは振る舞い

1. childnum が -1 でない場合:

  • ap_run_child_status を呼び出し、スコアボードに記録されたプロセス情報（PID、世代）を用いて MPM_CHILD_EXITED ステータスを通知します。
  • スコアボード上の該当インデックスの PID を 0 にリセットします。

    • 2. childnum が -1 の場合:

  • 引数として渡された pid と gen を使用して、直接 ap_run_child_status を呼び出し、MPM_CHILD_EXITED ステータスを通知します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/worker/worker.c 内のプロセス管理ロジック（ap_relieve_child_processes や ap_reclaim_child_processes への引数として渡されるケースを含む）。
  • **呼び出し先**:
  • ap_run_child_status（フック関数）。

関連データ構造

  • ap_scoreboard_image: プロセスおよびスレッドの状態を保持するグローバルなスコアボード構造体。
  • ap_server_conf: サーバー設定情報。

失敗モードと境界条件

  • childnum が -1 の場合、スコアボードの更新はスキップされ、引数で渡された情報のみが通知に使用されます。

保守担当者向けメモ

  • 本関数は ap_relieve_child_processes 等のプロセス管理関数からコールバックとして利用されることを想定しています。
  • スコアボードの整合性を維持するため、childnum が有効な場合は必ず PID のリセットが行われる点に注意してください。
