deflate_set_buffer_size

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_deflate.c:246

目的

Apache 設定ディレクティブ DeflateBufferSize の値を解析し、サーバー設定構造体 deflate_filter_config に適用する。

シグネチャまたは形状

static const char *deflate_set_buffer_size(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • arg: 設定ファイルから読み込まれたバッファサイズを表す文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列（const char *）を返す。

前提条件と不変条件

  • arg は整数値として解釈可能である必要がある。
  • 設定されるバッファサイズは正の整数（n > 0）でなければならない。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定から deflate_filter_config を取得する。 2. atoi(arg) を用いて入力文字列を整数に変換する。 3. 変換された値が 0 以下の場合、エラーメッセージを返して終了する。 4. 値が正の場合、c->bufferSize にその値を格納する。 5. 正常終了として NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_deflate.c:1925 において、AP_INIT_TAKE1 マクロを通じて DeflateBufferSize ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: ap_get_module_config (Apache API) および atoi (標準ライブラリ)。

関連データ構造

  • deflate_filter_config: サーバー全体の設定を保持する構造体。bufferSize メンバを更新する。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • arg が非数値または 0 以下の数値である場合、エラーメッセージ "DeflateBufferSize should be positive" を返す。

保守担当者向けメモ

  • 本関数は DeflateBufferSize ディレクティブの解析専用である。
  • deflate_ctx 内のバッファ確保は、この設定値に基づいて動的に行われるため、設定変更はフィルタの初期化時に反映される。
