md_common_name_count

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:66

目的

2つのマネージドドメイン（md_t）構造体間で、共通して含まれるドメイン名の数を算出します。

シグネチャまたは形状

apr_size_t md_common_name_count(const md_t *md1, const md_t *md2)

入力と出力

  • **入力**:
  • md1: 比較対象となる1つ目の md_t 構造体へのポインタ。
  • md2: 比較対象となる2つ目の md_t 構造体へのポインタ。
  • **出力**:
  • 共通するドメイン名の数（apr_size_t）。

前提条件と不変条件

  • md1 および md2 の domains メンバは、md_array_str_compact によって正規化されていることが期待されます。
  • 入力された md_t 構造体またはその domains 配列が NULL の場合、関数は即座に 0 を返します。

アルゴリズムまたは振る舞い

1. md1 または md2 が NULL、あるいはそれぞれの domains 配列が NULL であるかを確認し、該当する場合は 0 を返します。 2. md1->domains 内の各ドメイン名についてループ処理を行います。 3. 各ドメイン名に対し、md_contains(md2, name1, 0) を呼び出し、md2 に含まれているかを判定します。 4. 一致が見つかるたびにカウンタ hits をインクリメントします。 5. 全ての走査終了後、hits を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_reg.c:810
  • **呼び出し先**:
  • md_contains（modules/md/md_core.c 内の関数）

関連データ構造

  • md_t: 管理対象ドメインの設定を保持する構造体（modules/md/md.h で定義）。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、安全に 0 を返します。
  • 共通するドメイン名が一つも存在しない場合、0 を返します。

保守担当者向けメモ

  • 本関数は md_contains に依存しており、DNS ワイルドカード等のマッチングロジックは md_contains の実装に準拠します。
  • md_contains の第3引数に 0 を渡しているため、厳密なマッチングまたは md_contains 内部で定義されたデフォルトの比較ルールが適用されます。
