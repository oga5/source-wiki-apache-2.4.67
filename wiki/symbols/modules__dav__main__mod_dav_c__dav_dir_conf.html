dav_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.c:80

目的

ディレクトリ単位での WebDAV 設定を保持するための構造体です。プロバイダーの指定や、DAV 操作に関連する各種制限・挙動を管理します。

シグネチャまたは形状

typedef struct {
    const char *provider_name;
    const dav_provider *provider;
    const char *dir;
    const char *base;
    int locktimeout;
    int allow_depthinfinity;
    int allow_lockdiscovery;
} dav_dir_conf;

入力と出力

本構造体は、Apache の設定プロセスにおいてディレクトリ単位で生成・初期化されます。

  • **入力**: ap_get_module_config を介してリクエスト処理時に取得されます。
  • **出力**: 構造体内の各フィールドは、WebDAV メソッド（PROPFIND, PUT 等）の実行判断や、プロバイダーの特定に使用されます。

前提条件と不変条件

  • provider フィールドは、provider_name に基づいて解決された有効な dav_provider を指す必要があります。
  • apr_pcalloc を使用してメモリが割り当てられるため、リクエストプールまたは設定プールに関連付けられた生存期間を持ちます。

アルゴリズムまたは振る舞い

  • ディレクトリごとの設定を保持し、リクエスト処理時に ap_get_module_config(r->per_dir_config, &dav_module) を通じてアクセスされます。
  • 設定の継承やオーバーライドの際に、親ディレクトリの設定から新しい構造体へ値がコピーまたはマージされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.c 内の各種設定ハンドラおよびリクエスト処理関数（dav_handler 等）。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、保持している provider フィールドを通じてプロバイダーのフック関数が呼び出されます。

関連データ構造

  • dav_provider: WebDAV のリポジトリ操作を抽象化したインターフェース。
  • dav_module: 本構造体を使用する Apache モジュール定義。

失敗モードと境界条件

  • provider_name が設定されていない場合、WebDAV リクエストの処理が適切にディスパッチされない可能性があります。
  • メモリ割り当て（apr_pcalloc）が失敗した場合、Apache の標準的なエラー処理に従いリクエストが中断されます。

保守担当者向けメモ

  • dav_server_conf と異なり、本構造体はディレクトリ単位の設定として頻繁に参照されます。
  • フィールドの追加や変更を行う際は、設定の継承ロジック（dav_create_dir_config 等の関連関数）に影響がないか確認してください。
