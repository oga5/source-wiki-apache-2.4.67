ajp_msg_check_header

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_msg.c:151

目的

受信した AJP メッセージのヘッダー署名およびサイズを検証し、メッセージの有効性を確認します。

シグネチャまたは形状

apr_status_t ajp_msg_check_header(ajp_msg_t *msg, apr_size_t *len)

入力と出力

  • **入力**:
  • msg: 検証対象の ajp_msg_t 構造体へのポインタ。
  • len: メッセージのデータ長（ヘッダーを除く）を格納するための出力用ポインタ。
  • **出力**:
  • APR_SUCCESS: 検証成功。
  • AJP_EBAD_SIGNATURE: 署名が不正。
  • AJP_ETOBIG: メッセージサイズがバッファ許容範囲を超過。

前提条件と不変条件

  • msg->buf には少なくとも AJP_HEADER_LEN 分のデータが格納されている必要があります。
  • msg->max_size はバッファの物理的な最大サイズを保持している必要があります。

アルゴリズムまたは振る舞い

1. **署名検証**: バッファの先頭2バイトが 0x4142 または 0x1234 であるかを確認します。一致しない場合はエラーを返します。 2. **サイズ抽出**: バッファの3-4バイト目からネットワークバイトオーダーでメッセージ長を読み取ります。 3. **サイズ検証**: 抽出したメッセージ長が msg->max_size - AJP_HEADER_LEN を超えていないかを確認します。超えている場合はエラーを返します。 4. **状態更新**: 検証成功時、msg->len にヘッダーを含めた全長を、msg->pos にヘッダー終了位置を設定し、出力引数 len にメッセージ長を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/ajp_link.c:90 にて呼び出されています。
  • **呼び出し先**: ap_log_error（エラーログ出力用）。

関連データ構造

  • ajp_msg_t: AJP メッセージを保持する構造体。
  • AJP_HEADER_LEN: ヘッダーの固定長（ajp.h で定義）。

失敗モードと境界条件

  • **署名不一致**: 0x4142 または 0x1234 以外の場合、AJP_EBAD_SIGNATURE を返します。
  • **サイズ超過**: メッセージ長がバッファの許容サイズを超える場合、AJP_ETOBIG を返します。

保守担当者向けメモ

  • 本関数は ajp_msg_t の内部カーソル pos をヘッダー直後にリセットするため、受信直後の解析処理の起点として使用してください。
