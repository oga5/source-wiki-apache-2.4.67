proxy_wstunnel_handler

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_wstunnel.c:346

目的

WebSocket プロトコル（ws: / wss:）のアップグレードリクエストを処理し、クライアントとバックエンド間の双方向トンネリングを確立・実行するメインエントリポイントです。

シグネチャまたは形状

static int proxy_wstunnel_handler(request_rec *r, proxy_worker *worker,
                                  proxy_server_conf *conf,
                                  char *url, const char *proxyname,
                                  apr_port_t proxyport)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • worker: ターゲットとなるプロキシワーカー。
  • conf: プロキシサーバー設定。
  • url: 接続先 URL。
  • proxyname, proxyport: プロキシ設定情報。
  • **出力**:
  • int: 処理結果ステータス（OK, DECLINED, HTTP_UPGRADE_REQUIRED, HTTP_SERVICE_UNAVAILABLE 等）。

前提条件と不変条件

  • r->per_dir_config から proxyws_dir_conf を取得可能であること。
  • URL が ws: または wss: で始まること。
  • Upgrade ヘッダーが適切に設定されていること。

アルゴリズムまたは振る舞い

1. **フォールバック判定**: 設定により mod_proxy_http への委譲が有効な場合は DECLINED を返す。 2. **スキーム検証**: URL が ws/wss かを確認し、SSL 使用フラグをセットする。 3. **アップグレード検証**: Upgrade ヘッダーをチェックし、不適合な場合は HTTP_UPGRADE_REQUIRED を返す。 4. **接続管理**:

  • ap_proxy_acquire_connection でバックエンド接続を取得。
  • ap_proxy_determine_connection で接続先を特定。
  • ap_proxy_connect_backend でバックエンドへの接続を確立。
  • ap_proxy_connection_create_ex で接続レコードを作成。

    • 5. **トンネリング**: proxy_wstunnel_request を呼び出し、双方向データ転送ループを実行。 6. **終了処理**: 接続を解放し、ステータスを返却。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy のフックシステム（proxy_hook_scheme_handler に登録）。
  • **呼び出し先**:
  • ap_proxy_acquire_connection, ap_proxy_determine_connection, ap_proxy_connect_backend, ap_proxy_connection_create_ex, ap_proxy_release_connection (プロキシ基盤 API)
  • proxy_wstunnel_request (トンネリング実行)

関連データ構造

  • proxyws_dir_conf: ディレクトリ単位の設定（フォールバック制御）。
  • proxy_conn_rec: バックエンド接続状態。
  • request_rec: リクエスト情報。

失敗モードと境界条件

  • **URL 不適合**: ws/wss 以外は DECLINED。
  • **アップグレード失敗**: Upgrade ヘッダーが期待値と異なる場合、HTTP_UPGRADE_REQUIRED を返し、Connection: Upgrade ヘッダーを付与。
  • **接続失敗**: バックエンドへの接続確立に失敗した場合、HTTP_SERVICE_UNAVAILABLE を返す。

保守担当者向けメモ

  • 本関数はトンネリングが確立されると proxy_wstunnel_request 内でループし、リクエスト処理を占有します。
  • backend->close = 1 を設定することで、終了時に接続が再利用されないよう制御しています。
