report_lua_error

定義

  • 種別: function
  • 定義箇所: modules/lua/mod_lua.c:101

目的

Lua スクリプトの実行中に発生したエラーを捕捉し、HTTP 500 エラーレスポンスとしてクライアントへ通知するとともに、Apache のエラーログへ記録する。

シグネチャまたは形状

static void report_lua_error(lua_State *L, request_rec *r)

入力と出力

  • **入力**:
  • lua_State *L: エラー情報がスタックトップに格納されている Lua 状態。
  • request_rec *r: 現在のリクエスト情報。
  • **出力**: なし（副作用としてレスポンスボディの生成とログ出力を行う）。

前提条件と不変条件

  • L のスタックトップ（インデックス -1）にエラーメッセージ文字列が存在すること。
  • r は有効なリクエストレコードであること。

アルゴリズムまたは振る舞い

1. リクエストのステータスコードを HTTP_INTERNAL_SERVER_ERROR に設定する。 2. コンテンツタイプを text/html に設定する。 3. ap_rputs を使用して、HTML 形式のエラーメッセージ（<h3>Error!</h3> および <pre> タグで囲まれたエラー内容）をレスポンスボディとして書き出す。 4. lua_tostring でスタックトップのエラー文字列を取得し、ap_escape_html でエスケープ処理を行った上で出力する。 5. ap_log_perror を呼び出し、APLOG_WARNING レベルでエラー内容を Apache のエラーログに記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/mod_lua.c:324
  • modules/lua/mod_lua.c:726
  • modules/lua/mod_lua.c:848
  • **呼び出し先**:
  • lua_tostring (Lua API)
  • ap_rputs (Apache API)
  • ap_escape_html (Apache API)
  • ap_log_perror (Apache API)

関連データ構造

  • lua_State: Lua インタプリタの状態。
  • request_rec: Apache のリクエスト処理用構造体。

失敗モードと境界条件

  • スタックトップに文字列以外の値がある場合、lua_tostring の挙動に依存する。
  • r->pool が無効な場合、ap_escape_html や ap_log_perror の呼び出しでクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数はエラー発生時の定型処理であるため、エラー表示形式を変更する場合はここを修正する。
  • ログ出力には APLOGNO(01471) が付与されている。
