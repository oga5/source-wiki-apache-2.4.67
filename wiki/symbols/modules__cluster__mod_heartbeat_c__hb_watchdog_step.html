hb_watchdog_step

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartbeat.c:130

目的

mod_watchdog の定期実行フックとして、サーバーの負荷情報をネットワークへ通知する処理をトリガーする。

シグネチャまたは形状

static int hb_watchdog_step(server_rec *s, const char *name, apr_pool_t *pool)

入力と出力

  • **入力**:
  • s: サーバー設定構造体。
  • name: ウォッチドッグの識別名。
  • pool: 処理に使用するメモリプール。
  • **出力**:
  • OK: 処理が正常終了、または実行条件を満たさなかった場合。
  • hb_monitor の戻り値: 負荷情報の送信結果。

前提条件と不変条件

  • hb_ctx_t 設定構造体が heartbeat_module に関連付けられていること。
  • mod_watchdog によって呼び出されること。

アルゴリズムまたは振る舞い

1. heartbeat_module から設定構造体 hb_ctx_t を取得する。 2. ctx->active が無効であるか、name が AP_WATCHDOG_SINGLETON と一致しない場合は、何もせず OK を返す。 3. 上記条件を満たす場合、hb_monitor(ctx, pool) を呼び出し、負荷情報の集計と送信を実行してその結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_watchdog（ap_hook_watchdog_step を介して登録）。
  • **呼び出し先**: hb_monitor。

関連データ構造

  • hb_ctx_t: モジュールの設定および実行状態を保持する構造体。
  • server_rec: Apache サーバー設定構造体。

失敗モードと境界条件

  • ctx->active が偽の場合、送信処理はスキップされる。
  • name が AP_WATCHDOG_SINGLETON 以外の場合、処理はスキップされる。
  • hb_monitor 内部でソケット通信エラーが発生した場合、そのエラーコードがそのまま返される。

保守担当者向けメモ

  • 本関数は mod_watchdog のフックとして動作するため、AP_WATCHDOG_SINGLETON 以外の名前で呼び出された場合は無視する設計となっている。
  • 負荷情報の送信ロジックは hb_monitor に委譲されているため、送信プロトコルや集計ロジックの変更は hb_monitor を参照すること。
