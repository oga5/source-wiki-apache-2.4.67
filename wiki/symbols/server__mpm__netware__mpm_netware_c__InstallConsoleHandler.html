InstallConsoleHandler

定義

  • 種別: function
  • 定義箇所: server/mpm/netware/mpm_netware.c:1271

目的

NetWare コンソールからの入力を Apache HTTP Server (NLM) に橋渡しするためのコマンドハンドラを登録します。

シグネチャまたは形状

static int InstallConsoleHandler(void)

入力と出力

  • 入力: なし
  • 出力: 成功時に 0、リソースタグの割り当て失敗時に 1 を返します。

前提条件と不変条件

  • CommandLineInterpreter 関数が定義されている必要があります。
  • NetWare の RegisterConsoleCommand API が利用可能な環境である必要があります。

アルゴリズムまたは振る舞い

1. NX_WRAP_INTERFACE を使用して CommandLineInterpreter をコンソールハンドラのパーサーとしてラップします。 2. AllocateResourceTag を呼び出し、コマンドライン処理用のリソースタグを取得します。取得に失敗した場合はエラーメッセージを出力して終了します。 3. RegisterConsoleCommand を呼び出し、設定済みの ConsoleHandler をシステムに登録します。 4. 正常終了時に 0 を返します。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/netware/mpm_netware.c 内の初期化処理（1114行目付近）。
  • 呼び出し先: NX_WRAP_INTERFACE, AllocateResourceTag, getnlmhandle, RegisterConsoleCommand (いずれも NetWare SDK)。

関連データ構造

  • ConsoleHandler: コンソールコマンドの登録情報を保持する構造体。
  • CommandLineInterpreter: コンソール入力を解析するコールバック関数。

失敗モードと境界条件

  • AllocateResourceTag が NULL を返した場合、リソースの割り当て失敗として 1 を返し、ハンドラの登録は行われません。

保守担当者向けメモ

  • 本関数は NetWare 特有のコンソール統合機能です。
  • RemoveConsoleHandler が対となる登録解除処理を担う設計となっています。
