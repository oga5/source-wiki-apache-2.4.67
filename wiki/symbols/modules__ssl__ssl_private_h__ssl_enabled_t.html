ssl_enabled_t

定義

  • 種別: enum
  • 定義箇所: modules/ssl/ssl_private.h:525

目的

mod_ssl における SSL/TLS 機能の有効化状態を表現するための列挙型です。設定値が未定義であるか、明示的に有効か無効かを区別するために使用されます。

シグネチャまたは形状

typedef enum {
    SSL_ENABLED_UNSET = UNSET,
    SSL_ENABLED_FALSE = 0,
    SSL_ENABLED_TRUE  = 1,
} ssl_enabled_t;

入力と出力

  • 入力: 設定ファイルや内部状態から読み取られた値。
  • 出力: SSL_ENABLED_UNSET（未設定）、SSL_ENABLED_FALSE（無効）、SSL_ENABLED_TRUE（有効）のいずれかの状態値。

前提条件と不変条件

  • UNSET マクロが定義されている環境下で利用されることを前提とします。
  • 各列挙子は、設定の継承や上書きの判定において、論理的な三値論理として機能します。

アルゴリズムまたは振る舞い

  • 設定の初期化時やマージ時に、現在の設定が明示的に指定されているか（TRUE/FALSE）、あるいは親設定を継承すべきか（UNSET）を判定するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ssl/ssl_private.h 内の構造体定義（SSLSrvConfigRec 等）でメンバとして参照されています。
  • 呼び出し先: 本シンボル自体はデータ型であり、関数呼び出しは含みません。

関連データ構造

  • SSLSrvConfigRec: 仮想ホストごとの設定構造体において、enabled や strict_sni_vhost_check などのメンバとして利用されています。

失敗モードと境界条件

  • UNSET 以外の値が期待される箇所で UNSET が渡された場合、後続のロジックでデフォルト値へのフォールバックが必要となる場合があります。

保守担当者向けメモ

  • ssl_private.h 内の 852 行目および 860 行目で使用されています。設定の追加や変更を行う際は、この三値状態が適切に処理されているか確認してください。
