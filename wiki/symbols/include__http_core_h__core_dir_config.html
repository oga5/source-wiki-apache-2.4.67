core_dir_config

定義

  • 種別: struct
  • 定義箇所: include/http_core.h:527

目的

Apache HTTP Server におけるディレクトリ単位の設定情報を保持する構造体です。Options、AllowOverride、フィルタ設定、リソース制限、アクセス制御ルールなど、リクエスト処理の基盤となる設定状態を管理します。

入力と出力

  • **入力**: 設定ファイル（httpd.conf 等）のディレクティブ解析結果、または親ディレクトリからの継承設定。
  • **出力**: リクエスト処理時に core モジュールが参照する設定値。

前提条件と不変条件

  • opts_add と opts_remove は排他的である必要があります（opts_add & opts_remove == 0）。
  • 各ビットは opts_add または opts_remove のいずれか一方にのみ記録され、相対的なオプション変更の最終状態を保持します。

アルゴリズムまたは振る舞い

  • ディレクトリ単位の設定マージ（directory_walk 等）において、親から子へ設定が継承されます。
  • d_is_fnmatch フィールドは、ディレクトリパス d がワイルドカードを含むかを事前に判定し、頻繁な呼び出しを最適化するために使用されます。
  • condition フィールドにより、<If> などの条件付き設定ブロックの評価とマージを制御します。

呼び出し元と呼び出し先

  • 呼び出し元: include/ap_mmn.h に記載されている通り、モジュールバイナリ互換性（MMN）に関連する多くの変更履歴が存在し、コアモジュールおよび設定解析ロジックから参照されます。
  • 呼び出し先: apr_table_t、apr_hash_t、ap_expr_info_t などの APR および Apache ユーティリティ構造体を利用します。
  • ※具体的な関数レベルの呼び出し関係は本ソース抜粋からは特定できません。

関連データ構造

  • allow_options_t, overrides_t, etag_components_t: 設定用のビットマスク型。
  • ap_expr_info_t: 条件式評価用構造体。
  • apr_table_t, apr_hash_t: 設定値保持用のコンテナ。

失敗モードと境界条件

  • max_ranges 等の数値設定において、AP_MAXRANGES_UNSET などの定数による未設定状態のハンドリングが必要です。
  • リソース制限（rlimit）はプラットフォーム依存の定義（RLIMIT_CPU 等）に依存するため、環境によって構造体のサイズや有効性が異なります。

保守担当者向けメモ

  • 本構造体へのフィールド追加は、include/ap_mmn.h で定義されるモジュールバイナリ互換性（MMN）のバージョン更新を伴う可能性があります。
  • response_code_strings は現在使用されていません。
  • opts 関連のビットマスク操作は、不変条件を維持するため慎重に行う必要があります。
