check_speling

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_speling.c:197

目的

リクエストされた URL がサーバー上に存在しない場合に、ファイルシステムを走査してスペルミスや大文字小文字の不一致を補完し、リダイレクトまたは候補の提示を行う。

シグネチャまたは形状

static int check_speling(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在の HTTP リクエスト情報）
  • **出力**: int （Apache のステータスコード。成功時は HTTP_MOVED_PERMANENTLY または HTTP_MULTIPLE_CHOICES、処理対象外やエラー時は DECLINED を返す）

前提条件と不変条件

  • CheckSpelling ディレクティブによりモジュールが有効化されていること。
  • リクエストメソッドが GET であること。
  • リクエストされたパスがファイルシステム上に存在しないこと（r->finfo.filetype == APR_NOFILE）。
  • サブリクエストではなく、プロキシリクエストでもないこと。
  • リクエスト URI が / ではないこと。

アルゴリズムまたは振る舞い

1. **パス分解**: リクエストされた r->filename をディレクトリパスとファイル名に分割する。 2. **ディレクトリ走査**: 親ディレクトリを apr_dir_open で開き、全ファイルを読み込む。 3. **類似度判定**: 各ファイル名に対し以下の順で判定を行う。

  • 大文字小文字の不一致 (strcasecmp)。
  • spdist() による編集距離判定（cfg->check_case_only が無効な場合）。
  • cfg->check_basename_match が有効な場合、拡張子を除いたベース名での比較。

    • 4. **レスポンス決定**:

  • 候補が 1 つかつ確実性が高い場合、301 Moved Permanently を返し、Location ヘッダで修正先へリダイレクトする。
  • 候補が複数ある場合、300 Multiple Choices を返し、r->notes に候補リストを格納して HTML 形式の選択肢を生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の fixups フック（register_hooks にて ap_hook_fixups に登録）。
  • **呼び出し先**: spdist()（類似度計算）、apr_dir_open/apr_dir_read（ファイル探索）、qsort（候補のソート）、ap_escape_uri/ap_escape_html（出力のサニタイズ）。

関連データ構造

  • spconfig: モジュールの設定保持。
  • misspelled_file: 候補ファイル名と品質（sp_reason）を保持する構造体。
  • apr_array_header_t: 候補リストの格納に使用。

失敗モードと境界条件

  • **ディレクトリ走査失敗**: apr_dir_open が失敗した場合は DECLINED を返す。
  • **無限ループ回避**: 修正後の URL がリクエスト元と一致する場合（壊れたシンボリックリンク等）、リダイレクトせずに OK を返して終了する。
  • **パフォーマンス**: ディレクトリ内のファイル数に比例して走査コストが増大する。
  • **曖昧性**: 候補が複数ある場合は 300 Multiple Choices となり、自動リダイレクトは行われない。

保守担当者向けメモ

  • r->pool を使用してメモリ管理を行っているため、明示的な解放は不要。
  • spdist は「1文字の誤り」までを許容する設計となっている。
  • mod_negotiation と同様の notes テーブル利用パターンを採用しているため、修正時は関連性に注意すること。
