ssl_hook_pre_connection

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:658

目的

接続確立の初期段階において、SSL/TLS エンジンの有効性を判定し、接続に対する SSL コンテキストの初期化と準備を行う。

シグネチャまたは形状

static int ssl_hook_pre_connection(conn_rec *c, void *csd)

入力と出力

  • **入力**:
  • c: 現在の接続を表す conn_rec 構造体。
  • csd: クライアントソケット記述子（本関数内では直接使用されない）。
  • **出力**:
  • OK: SSL 接続の初期化が正常に開始された場合。
  • DECLINED: SSL が無効であると判定された場合。
  • その他: ssl_init_ssl_connection の戻り値に準ずる。

前提条件と不変条件

  • conn_rec 構造体が有効であること。
  • ssl_engine_status により、対象接続で SSL が有効かどうかが判定される。

アルゴリズムまたは振る舞い

1. ssl_engine_status を呼び出し、SSL が無効な接続であれば DECLINED を返して終了する。 2. 接続情報からサーバー設定 (SSLSrvConfigRec) を取得する。 3. 接続確立をログに記録する。 4. ssl_init_ssl_connection(c, NULL) を呼び出し、OpenSSL オブジェクトの生成と接続へのバインドを実行して、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の pre_connection フック（modules/ssl/mod_ssl.c:727 にて ap_hook_pre_connection を通じて登録）。
  • **呼び出し先**:
  • ssl_engine_status: SSL 有効性の確認。
  • myConnConfig / mySrvConfig: 設定情報の取得。
  • ap_log_cerror: ログ出力。
  • ssl_init_ssl_connection: SSL 接続の初期化。

関連データ構造

  • conn_rec: Apache の接続状態管理構造体。
  • SSLConnRec: 接続ごとの SSL 状態を保持する構造体。
  • SSLSrvConfigRec: サーバーごとの SSL 設定構造体。

失敗モードと境界条件

  • ssl_engine_status が OK 以外を返した場合、SSL 処理はスキップされる。
  • ssl_init_ssl_connection が失敗した場合、その戻り値がそのまま本関数の結果となる。

保守担当者向けメモ

  • 本関数は接続ライフサイクルの極めて早い段階で実行される。
  • ssl_init_ssl_connection への依存が強いため、SSL コンテキストの初期化ロジックを変更する場合は同関数を確認すること。
