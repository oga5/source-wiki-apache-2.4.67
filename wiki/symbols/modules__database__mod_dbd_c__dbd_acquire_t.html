dbd_acquire_t

定義

  • 種別: struct
  • 定義箇所: modules/database/mod_dbd.c:861

目的

データベース接続ハンドル（ap_dbd_t）と、その接続を管理するリソースリスト（apr_reslist_t）をペアとして保持し、リクエストまたはコネクションのライフサイクル内での接続管理を容易にするための構造体です。

シグネチャまたは形状

typedef struct {
    ap_dbd_t *rec;
    apr_reslist_t *reslist;
} dbd_acquire_t;

入力と出力

  • **入力**: 構造体生成時に ap_dbd_t へのポインタと apr_reslist_t へのポインタが割り当てられます。
  • **出力**: 構造体自体は、リクエストプール（r->pool）またはコネクションプール（c->pool）からメモリ確保されます。

前提条件と不変条件

  • rec は有効なデータベース接続ハンドルを指している必要があります。
  • reslist は、当該接続が属する有効な接続プールを指している必要があります。
  • この構造体は、ap_dbd_acquire や ap_dbd_cacquire の内部処理において、接続の解放（プールへの返却）を追跡するために使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: リクエストまたはコネクションのコンテキスト内で、apr_palloc を用いてメモリが確保されます。 2. **保持**: 取得したデータベース接続（rec）と、その接続元であるプール（reslist）を保持します。 3. **ライフサイクル管理**: リクエスト終了時に登録されたクリーンアップ関数（dbd_release）がこの構造体を参照し、rec を reslist へ返却することで接続の再利用を可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/database/mod_dbd.c 内の接続取得関数（ap_dbd_acquire および ap_dbd_cacquire）から、接続管理用コンテキストとしてインスタンス化されます。
  • **呼び出し先**:
  • 接続解放時に dbd_release 関数が本構造体を引数として受け取り、内部の reslist に対して接続を返却します。

関連データ構造

  • ap_dbd_t: データベース接続ハンドル。
  • apr_reslist_t: APR が提供するリソースプーリング構造体。

失敗モードと境界条件

  • メモリ不足: apr_palloc が失敗した場合、接続管理コンテキストの生成に失敗します。
  • 接続無効化: rec が指す接続が既に切断されている場合、dbd_release による返却処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dbd.c 内部でのみ使用されるプライベートな構造体です。
  • リクエストまたはコネクションの終了時に自動的にクリーンアップされるよう設計されているため、手動でのメモリ解放は不要です。
  • ap_dbd_acquire を使用する際は、この構造体がリクエストプールに紐付いているため、明示的な ap_dbd_close を呼び出さないよう注意してください。
