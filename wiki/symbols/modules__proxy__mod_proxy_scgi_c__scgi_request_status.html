scgi_request_status

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:474

目的

バックエンドからのレスポンスに基づき、内部リダイレクトまたはファイル送信（X-Sendfile）のサブリクエストを実行するフック関数です。

シグネチャまたは形状

static int scgi_request_status(int *status, request_rec *r)

入力と出力

  • **入力**:
  • status: 現在のリクエストステータスへのポインタ。
  • r: 現在のリクエストレコード。
  • **出力**:
  • OK: 処理が成功、またはサブリクエストが実行された場合。
  • DECLINED: 処理対象外（設定が存在しない、またはステータスが OK 以外）の場合。
  • HTTP_INTERNAL_SERVER_ERROR: ファイルサブリクエストの生成または実行に失敗した場合。

前提条件と不変条件

  • *status が OK であること。
  • r->request_config から proxy_scgi_module に関連付けられた scgi_request_config が取得可能であること。

アルゴリズムまたは振る舞い

1. **条件判定**: *status が OK かつ設定が存在する場合に処理を開始します。 2. **分岐処理**: req_conf->type に基づき以下のいずれかを行います。

  • scgi_internal_redirect:
  • r->status_line をクリアします。
  • メソッドが GET 以外の場合、GET に強制変更します。
  • Content-Length ヘッダーを削除し、ap_internal_redirect_handler を呼び出してリダイレクトを実行します。
  • scgi_sendfile:
  • ap_sub_req_lookup_file を使用してファイルサブリクエストを生成します。
  • ファイルが存在し、ステータスが HTTP_OK であれば ap_run_sub_req を実行します。
  • 失敗した場合は HTTP_INTERNAL_SERVER_ERROR を返します。

    • 3. **終了**: いずれのケースにも該当しない、または条件を満たさない場合は DECLINED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: APR_OPTIONAL_HOOK(proxy, request_status, ...) により、mod_proxy の request_status フックとして登録されています（modules/proxy/mod_proxy_scgi.c:671）。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_log_rerror
  • apr_table_unset
  • ap_internal_redirect_handler
  • ap_sub_req_lookup_file
  • ap_run_sub_req

関連データ構造

  • scgi_request_config: リクエストごとの設定（リダイレクト先やファイルパス、処理タイプを保持）。
  • request_rec: Apache のリクエストレコード。

失敗モードと境界条件

  • scgi_sendfile 処理において、サブリクエスト対象のファイルが存在しない、またはアクセスできない場合、HTTP_INTERNAL_SERVER_ERROR を返し処理を中断します。
  • Content-Length はサブリクエスト実行時に明示的に操作されず、バックエンドの挙動またはチャンク転送に依存します。

保守担当者向けメモ

  • 本関数は mod_proxy のフックとして動作するため、他のプロキシモジュールとの競合や実行順序に注意してください。
  • scgi_sendfile における Content-Length の扱いは、バックエンドとの整合性を保つための設計上の判断です。
