event_conn_state_t

定義

  • 種別: struct
  • 定義箇所: server/mpm/event/event.c:215

目的

Event MPM における個々の接続の状態を追跡・管理するための構造体です。接続のライフサイクル（処理中、書き込み完了待ち、Keep-Alive 待ち、遅延クローズなど）を保持し、タイムアウト管理や非同期 I/O 処理の基盤となります。

シグネチャまたは形状

不完全な型として typedef struct event_conn_state_t event_conn_state_t; で宣言され、同ファイル内で構造体定義が行われています。内部には APR_RING_ENTRY(event_conn_state_t) timeout_list; を含み、リングバッファによるキュー管理をサポートする形状をしています。

入力と出力

本シンボル自体はデータ構造ですが、process_socket() 等の関数において接続状態のコンテキストとして渡されます。また、notify_suspend(event_conn_state_t *cs) や notify_resume(event_conn_state_t *cs, int cleanup) などの関数で引数として利用されます。

前提条件と不変条件

  • ptrans（トランザクションプール）から割り当てられ、接続終了時にクリーンアップされる必要があります。
  • timeout_queue に登録される際、timeout_mutex による排他制御が必要です。
  • apr_pollset への登録・削除と連動して状態が遷移します。

アルゴリズムまたは振る舞い

  • **状態管理**: 接続の現在の状態を保持し、timeout_queue（waitio_q, write_completion_q, keepalive_q 等）を通じて期限管理を行います。
  • **キューイング**: TO_QUEUE_APPEND や TO_QUEUE_REMOVE マクロを介して、リングバッファへの追加・削除が行われます。
  • **非同期処理**: リスナーおよびワーカー間で受け渡され、イベント駆動による接続の再開やクローズの判断に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: listener_thread がタイムアウト処理時に参照するほか、worker_thread が接続処理のコンテキストとして利用します。
  • **呼び出し先**: APR_RING 関連の操作関数（APR_RING_INSERT_TAIL 等）が本構造体のメンバを操作します。
  • **注記**: 具体的な関数呼び出しの全容は、ソースコードの各フック関数の詳細に依存します。

関連データ構造

  • timeout_queue: 接続を期限ごとに管理するリングバッファ。
  • apr_pollset: 非同期 I/O 監視のための構造体。
  • event_srv_cfg: MPM の設定情報。

失敗モードと境界条件

  • メモリ割り当て失敗（ptrans 不足時）。
  • タイムアウト期限切れ時のクローズ処理における競合（timeout_mutex による保護が必須）。
  • apr_pollset への登録失敗時、接続が適切にクローズされないリスクがあります。

保守担当者向けメモ

  • 本構造体は event.c 内で広範囲に参照されています。メンバ変数の追加や変更を行う際は、timeout_queue の整合性と APR_RING の定義に影響がないか確認してください。
  • 接続のライフサイクル管理は ptrans の生存期間と密接に関連しているため、メモリリークを防ぐためにクリーンアップ処理の順序を厳守してください。
