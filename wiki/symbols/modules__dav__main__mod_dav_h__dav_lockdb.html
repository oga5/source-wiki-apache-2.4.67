dav_lockdb

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1338

目的

WebDAV ロックデータベースへのアクセスを抽象化し、ロック管理機能を提供するプロバイダーへのインターフェースを保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const dav_hooks_locks *hooks;
    int ro;
    dav_lockdb_private *info;
} dav_lockdb;

入力と出力

  • **入力**:
  • hooks: ロック操作を実行するための関数ポインタテーブル。
  • ro: 読み取り専用モードで開かれたかどうかを示すフラグ（0以外で読み取り専用）。
  • info: プロバイダー固有の内部データ構造へのポインタ。

前提条件と不変条件

  • hooks は、ロックデータベースの操作に必要な関数群を提供する必要があります。
  • info は、特定のストレージ実装（例: mod_dav_fs）によって管理されるプライベートな状態を指します。

アルゴリズムまたは振る舞い

本構造体は、WebDAV のロック操作（LOCK, UNLOCK 等）を行う際に、プロバイダーが提供するフックと、その実行に必要なコンテキストを保持します。mod_dav の各メソッドは、この構造体を通じてバックエンドのロックデータベースと対話します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/lock.c 内の各関数（dav_fs_really_open_lockdb, dav_fs_close_lockdb, dav_fs_get_locks 等）が、この構造体を引数として受け取り、操作対象として利用しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、内部の hooks メンバを介してプロバイダーの実装関数が呼び出されます。

関連データ構造

  • dav_hooks_locks: ロック操作のフック定義。
  • dav_lockdb_private: プロバイダー固有の内部データ構造。
  • dav_lock: ロック情報を保持する構造体。

失敗モードと境界条件

  • hooks が適切に初期化されていない場合、ロック操作の呼び出し時に未定義の動作を引き起こす可能性があります。
  • ro フラグが設定されている状態で書き込み操作を試みた場合、プロバイダー側の実装でエラーが返されることが想定されます。

保守担当者向けメモ

  • 本構造体は mod_dav のロック管理の基盤です。新しいロックプロバイダーを追加する際は、この構造体の hooks に適切な関数テーブルを割り当てる必要があります。
  • dav_lockdb_private の定義は、各プロバイダーの実装に依存するため、変更時には関連するプロバイダーモジュール（mod_dav_fs 等）との整合性に注意してください。
