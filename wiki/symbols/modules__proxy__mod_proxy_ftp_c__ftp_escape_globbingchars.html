ftp_escape_globbingchars

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:141

目的

FTP パス文字列に含まれるグロブ文字（ワイルドカード）をエスケープし、FTP サーバー側で特殊文字として解釈されないようにする。

シグネチャまたは形状

static const char *ftp_escape_globbingchars(apr_pool_t *p, const char *path, proxy_ftp_dir_conf *dconf)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • path: エスケープ対象のパス文字列。
  • dconf: 設定構造体。ftp_escape_wildcards フラグを参照する。
  • **出力**:
  • エスケープ処理後の新しい文字列（apr_palloc で確保）、またはエスケープ不要な場合は元の path ポインタを返す。

前提条件と不変条件

  • dconf->ftp_escape_wildcards が偽の場合、入力された path がそのまま返される。
  • 戻り値の文字列は、元のパスの最大 2 倍の長さまで確保される。

アルゴリズムまたは振る舞い

1. dconf->ftp_escape_wildcards を確認し、無効であれば即座に path を返す。 2. 有効な場合、apr_palloc を使用して、元のパス長の 2 倍のメモリ領域を確保する。 3. 入力 path を 1 文字ずつ走査し、FTP_GLOBBING_CHARS に含まれる文字であれば、前にバックスラッシュ (\) を挿入する。 4. 最後にヌル終端文字を付与して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ftp.c 内の以下の箇所から呼び出されている。
  • 1320行目、1675行目、1694行目、1755行目（MDTM コマンド）、1794行目（RETR コマンド）、1831行目。
  • **呼び出し先**:
  • apr_palloc (APR)
  • strchr (標準ライブラリ)

関連データ構造

  • proxy_ftp_dir_conf: FTP 固有の設定を保持する構造体。ftp_escape_wildcards フラグを含む。

失敗モードと境界条件

  • path が空文字列の場合、空文字列が返される。
  • FTP_GLOBBING_CHARS に該当する文字がない場合、元の文字列がそのまま返される（dconf が有効であっても）。
  • メモリ不足時は apr_palloc の挙動に従い、通常はプロセスが終了する。

保守担当者向けメモ

  • FTP_GLOBBING_CHARS の定義は本関数外で行われている。エスケープ対象文字を変更する場合は、その定数定義を確認すること。
  • 戻り値は apr_pool_t に依存しているため、呼び出し側で適切なプールが管理されていることを前提としている。
