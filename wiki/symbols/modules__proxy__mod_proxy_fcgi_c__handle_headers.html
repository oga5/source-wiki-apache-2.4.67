handle_headers

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:543

目的

FastCGI バックエンドから受信したストリームデータから、HTTP ヘッダの終端（空行）を検出するステートマシンとして機能します。

シグネチャまたは形状

static int handle_headers(request_rec *r, int *state, const char *readbuf, apr_size_t readlen)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • state: 現在の解析状態を保持する整数へのポインタ。
  • readbuf: 受信したデータのバッファ。
  • readlen: バッファ内のデータ長。
  • **出力**:
  • 戻り値: ヘッダの終端に到達した場合は 1、それ以外は 0 を返します。
  • state: 終端検出に伴い、値が更新されます。

前提条件と不変条件

  • state は、ヘッダ解析の進行状況を追跡するために、呼び出し間で一貫して保持される必要があります。
  • readbuf は readlen 分の有効なデータを含んでいる必要があります。

アルゴリズムまたは振る舞い

1. readbuf をバイト単位で走査します。 2. \r または \n の出現に基づき、state を遷移させます。

  • \r\n\r\n または \n\n に相当するシーケンスを検出すると、HDR_STATE_DONE_WITH_HEADERS 状態へ移行します。
  • それ以外の文字が入力された場合、状態は HDR_STATE_READING_HEADERS にリセットされます。

    • 3. HDR_STATE_DONE_WITH_HEADERS に到達した時点で走査を終了し、1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_fcgi.c:822
  • modules/aaa/mod_authnz_fcgi.c:557
  • **呼び出し先**:
  • 本関数は内部で他の関数を呼び出しません。

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • HDR_STATE_*: ヘッダ解析の状態を定義する定数群（mod_proxy_fcgi.c 内で定義）。

失敗モードと境界条件

  • データの断片化によりヘッダが複数回に分けて受信される場合、state を維持することで正しく終端を検出します。
  • 終端が見つからない場合は 0 を返し、後続のデータ受信を待機する設計となっています。

保守担当者向けメモ

  • 本関数は FastCGI のストリーム解析における低レベルな状態遷移を担当しています。ヘッダの定義や解析ロジックを変更する場合は、HDR_STATE 定数の遷移図を更新してください。
