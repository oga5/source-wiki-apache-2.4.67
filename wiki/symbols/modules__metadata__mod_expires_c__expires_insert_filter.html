expires_insert_filter

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:526

目的

リクエストのコンテキストを評価し、mod_expires による出力フィルタ（MOD_EXPIRES）を適用すべきかどうかを判定して、必要に応じてフィルタチェーンに追加します。

シグネチャまたは形状

static void expires_insert_filter(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: なし（副作用として出力フィルタチェーンを操作）

前提条件と不変条件

  • HTTP エラーレスポンス（ap_is_HTTP_ERROR が真）の場合はフィルタを追加しません。
  • サブリクエスト（r->main != NULL）の場合はフィルタを追加しません。
  • ディレクトリ設定において ExpiresActive が on であり、かつ ExpiresDefault または ExpiresByType のいずれかが設定されている場合にのみフィルタが追加されます。

アルゴリズムまたは振る舞い

1. リクエストのステータスを確認し、エラーであれば即座に終了します。 2. サブリクエストであるかを確認し、該当する場合は終了します。 3. ap_get_module_config を使用して、現在のディレクトリ設定（expires_dir_config）を取得します。 4. 設定値を確認し、フィルタが有効かつ適用可能なルールが存在するかを判定します。 5. 条件を満たす場合、ap_add_output_filter を呼び出し、MOD_EXPIRES フィルタをリクエストの出力チェーンに登録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ap_hook_insert_error_filter（modules/metadata/mod_expires.c:558）
  • ap_hook_insert_filter（modules/metadata/mod_expires.c:559）
  • **呼び出し先**:
  • ap_is_HTTP_ERROR
  • ap_get_module_config
  • apr_is_empty_table
  • ap_add_output_filter

関連データ構造

  • request_rec: リクエストのコンテキストを保持する構造体。
  • expires_dir_config: ディレクトリごとの設定（active フラグ、expiresbytype テーブル、expiresdefault）を保持する構造体。

失敗モードと境界条件

  • ExpiresActive が off の場合、または設定が空の場合はフィルタが追加されず、ヘッダーの注入は行われません。
  • 既に Expires ヘッダーが存在する場合の挙動については、本関数ではなく、後続の expires_filter 関数で制御されます。

保守担当者向けメモ

  • 本関数はフィルタの「挿入」のみを担います。実際のヘッダー生成ロジックは expires_filter に依存しています。
  • expires_module を介して設定を取得しているため、モジュール構成の変更時には注意が必要です。
