close_handle_in_child

定義

  • 種別: function
  • 定義箇所: server/log.c:221

目的

子プロセスにおいて、後でクローズ処理を行うためにファイル記述子を追跡リストへ追加します。

シグネチャまたは形状

static void close_handle_in_child(apr_pool_t *p, apr_file_t *f)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t ポインタ。
  • f: 追跡対象となる apr_file_t ポインタ。
  • 出力: なし（副作用としてグローバルな追跡リストを更新）。

前提条件と不変条件

  • WIN32 が定義されていない環境でのみ動作します。
  • read_handles という静的変数（リストのヘッド）が、この関数からアクセス可能である必要があります。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい read_handle_t 構造体をプール p 上に割り当てます。 2. 新しい構造体の handle メンバに引数 f を格納します。 3. 新しい構造体の next メンバに現在の read_handles リストの先頭を代入します。 4. read_handles を新しい構造体で更新し、リストの先頭に挿入します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/log.c:357
  • server/log.c:1807
  • 呼び出し先:
  • apr_pcalloc (APR ライブラリ関数)

関連データ構造

  • read_handle_t: ファイル記述子を保持し、リスト構造を形成するための構造体。
  • apr_file_t: APR が提供するファイルハンドル抽象化構造体。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足により失敗した場合、プログラムは異常終了する可能性があります（APR のメモリ割り当てポリシーに依存）。
  • WIN32 環境では、この関数は何も行わない空の関数として振る舞います。

保守担当者向けメモ

  • 本関数は read_handles という静的変数に依存しています。マルチスレッド環境下でのこのリストへのアクセス保護については、呼び出し側のコンテキストを確認してください。
  • WIN32 以外のプラットフォームにおけるプロセスフォーク後のファイル記述子管理において重要な役割を果たします。
