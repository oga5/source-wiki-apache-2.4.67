dav_get_direct_resource

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:423

目的

指定されたロックトークンに対応する「直接ロック（Direct Lock）」が適用されているリソースを特定します。間接ロック（継承されたロック）が指定された場合、その起点を遡って探索します。

シグネチャまたは形状

dav_error * dav_get_direct_resource(apr_pool_t *p,
                                    dav_lockdb *lockdb,
                                    const dav_locktoken *locktoken,
                                    const dav_resource *resource,
                                    const dav_resource **direct_resource)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • lockdb: ロックデータベースへのハンドル。
  • locktoken: 検索対象のロックトークン。
  • resource: 探索を開始する対象リソース。
  • **出力**:
  • direct_resource: 直接ロックを保持するリソースへのポインタ（成功時）。
  • 戻り値: 成功時は NULL、失敗時は dav_error 構造体。

前提条件と不変条件

  • lockdb->hooks->find_lock が実装されている必要があります。
  • resource->hooks->get_parent_resource が親リソースの取得に利用可能である必要があります。

アルゴリズムまたは振る舞い

1. lockdb->hooks->lookup_resource が定義されている場合、それを優先して呼び出し、結果を返します。 2. 定義されていない場合、現在の resource から順に親リソースを遡りながら以下の処理を繰り返します。

  • find_lock を呼び出し、現在のリソースに指定されたトークンのロックが存在するか確認します。
  • ロックが見つからない場合は HTTP_BAD_REQUEST を返します。
  • 見つかったロックが DAV_LOCKREC_DIRECT（直接ロック）であれば、そのリソースを direct_resource に設定して終了します。
  • 間接ロックであれば、get_parent_resource を用いて親リソースへ移動し、探索を継続します。

    • 3. ルートに到達しても直接ロックが見つからない場合は、ロックデータベースの不整合とみなし HTTP_INTERNAL_SERVER_ERROR を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/util_lock.c 内の dav_unlock 関数（行533付近）から呼び出されます。
  • **呼び出し先**:
  • lockdb->hooks->lookup_resource
  • lockdb->hooks->find_lock
  • resource->hooks->get_parent_resource

関連データ構造

  • dav_lockdb: ロックデータベースのフックと状態を保持。
  • dav_locktoken: ロックを一意に識別するトークン。
  • dav_resource: WebDAV リソースの抽象表現。
  • dav_lock: ロックの属性（rectype 等）を保持する構造体。

失敗モードと境界条件

  • **ロック未検出**: 指定トークンがリソースに存在しない場合、HTTP_BAD_REQUEST を返します。
  • **データベース不整合**: 間接ロックの起点となる直接ロックが探索過程で見つからない場合、HTTP_INTERNAL_SERVER_ERROR を返します。
  • **フックエラー**: 各フック関数がエラーを返した場合、そのエラーを伝播します。

保守担当者向けメモ

  • 本関数は、間接ロックの解除時にその起点を特定するために不可欠です。
