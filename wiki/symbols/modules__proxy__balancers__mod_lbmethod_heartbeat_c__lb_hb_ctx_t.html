lb_hb_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_heartbeat.c:44

目的

mod_lbmethod_heartbeat モジュールにおいて、ハートビート情報の読み込み元となるパス（HeartbeatStorage）を保持するための設定コンテキスト構造体です。

シグネチャまたは形状

typedef struct lb_hb_ctx_t
{
    const char *path;
} lb_hb_ctx_t;

入力と出力

  • **入力**: path メンバは、ハートビート情報が格納されているファイルパスまたは共有メモリ識別子を保持します。
  • **出力**: なし（本構造体は設定保持用であり、直接的な出力は行いません）。

前提条件と不変条件

  • path は apr_palloc 等で割り当てられたメモリ領域を指す必要があります。
  • モジュール設定として使用される際、ap_get_module_config を通じてサーバー設定から取得されることが想定されています。

アルゴリズムまたは振る舞い

  • 本構造体は、HeartbeatStorage ディレクティブで指定された値を保持し、readfile_heartbeats や readslot_heartbeats がハートビート情報を取得する際の参照先として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_lbmethod_heartbeat.c 内の各種設定関数（ap_get_module_config を使用する箇所）および初期化関数。
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • hb_server_t: ハートビート情報によって更新される各バックエンドサーバーの負荷状態を保持する構造体。

失敗モードと境界条件

  • path が NULL または無効なパスである場合、readfile_heartbeats 等のデータ読み込み関数が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はモジュールの設定コンテキストとして機能します。設定の継承やマージ処理（apr_pcalloc を使用した初期化など）を行う際は、mod_proxy の設定管理ルールに従ってください。
