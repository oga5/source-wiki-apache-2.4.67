create_buffer_config

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_buffer.c:298

目的

ディレクトリ単位の設定構造体 buffer_conf を初期化し、デフォルト値を割り当てるために使用されます。

シグネチャまたは形状

static void *create_buffer_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体を割り当てるためのメモリプール。
  • char *dummy: Apache の設定ディレクティブハンドラから渡される引数（本関数では使用されません）。
  • **出力**:
  • void *: 初期化された buffer_conf 構造体へのポインタ。

前提条件と不変条件

  • 指定されたメモリプール p が有効である必要があります。
  • apr_pcalloc により、構造体内の全メンバはゼロ初期化されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、リクエストプール p 上に buffer_conf サイズのメモリを確保します。 2. size_set を 0（設定未適用）に設定します。 3. size を DEFAULT_BUFFER_SIZE で初期化します。 4. 確保した構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_buffer.c:347 にて、ディレクトリ単位の設定構造体を作成する際に呼び出されます。
  • **呼び出し先**: apr_pcalloc（APRライブラリ関数）。

関連データ構造

  • buffer_conf: 本関数が初期化対象とする設定構造体。size（バッファ上限）と size_set（設定有無）を保持します。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが終了する可能性があります（APRのメモリ割り当てポリシーに依存）。

保守担当者向けメモ

  • 本関数は mod_buffer の設定ライフサイクルの初期段階で実行されます。設定の継承ロジックについては merge_buffer_config を参照してください。
