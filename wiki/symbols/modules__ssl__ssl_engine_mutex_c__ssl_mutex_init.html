ssl_mutex_init

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_mutex.c:33

目的

セッションキャッシュの保護に必要なグローバル Mutex を初期化します。キャッシュプロバイダがマルチプロセス/マルチスレッド環境で安全でない場合にのみ Mutex を作成します。

シグネチャまたは形状

int ssl_mutex_init(server_rec *s, apr_pool_t *p)

入力と出力

  • **入力**:
  • s: サーバー設定構造体（server_rec）。
  • p: 初期化に使用するプール（apr_pool_t）。
  • **出力**:
  • TRUE (1): Mutex が不要、または初期化に成功した場合。
  • FALSE (0): Mutex の作成に失敗した場合。

前提条件と不変条件

  • myModConfig(s) を介して取得した SSLModConfigRec が有効であること。
  • mc->sesscache が設定されている場合、そのフラグに基づいて Mutex の必要性が判定されます。

アルゴリズムまたは振る舞い

1. SSLModConfigRec から設定を取得します。 2. 以下のいずれかの場合、Mutex は不要と判断し TRUE を返します。

  • セッションキャッシュが設定されていない。
  • キャッシュプロバイダが AP_SOCACHE_FLAG_NOTMPSAFE フラグを持たない（内部的に安全）。

    • 3. 既に mc->pMutex が存在する場合は、再作成せず TRUE を返します。 4. ap_global_mutex_create を呼び出し、SSL_CACHE_MUTEX_TYPE 型で Mutex を生成します。 5. 生成結果が APR_SUCCESS であれば TRUE を、それ以外であれば FALSE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:478
  • **呼び出し先**:
  • ap_global_mutex_create (APR API)
  • myModConfig (マクロ)

関連データ構造

  • SSLModConfigRec: モジュールの設定情報を保持する構造体。
  • server_rec: Apache サーバーの構成情報。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • ap_global_mutex_create が APR_SUCCESS 以外を返した場合、関数は FALSE を返し、初期化は失敗します。
  • キャッシュプロバイダが AP_SOCACHE_FLAG_NOTMPSAFE を指定しているにもかかわらず Mutex 作成に失敗した場合、セッションキャッシュの排他制御が機能しない状態となります。

保守担当者向けメモ

  • 本関数はサーバー起動時に呼び出されることを想定しています。
  • SSL_CACHE_MUTEX_TYPE の定義については ssl_private.h を参照してください。
