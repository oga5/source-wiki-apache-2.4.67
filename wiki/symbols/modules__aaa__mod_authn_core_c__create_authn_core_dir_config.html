create_authn_core_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_core.c:75

目的

ディレクトリ単位の認証設定を保持するための authn_core_dir_conf 構造体をメモリ上に確保し、初期化する。

シグネチャまたは形状

static void *create_authn_core_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体の割り当てに使用するメモリプール。
  • char *dummy: 使用されない引数（Apache の設定コールバックシグネチャへの適合用）。
  • **出力**:
  • void *: 確保およびゼロクリアされた authn_core_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • Apache の設定フェーズにおいて、ディレクトリ設定が生成される際に呼び出される。
  • apr_pcalloc を使用しているため、返される構造体はゼロクリアされていることが保証される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、authn_core_dir_conf 構造体のサイズ分のメモリをプール p から確保する。 2. 確保したメモリ領域を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_core.c:420（ディレクトリ設定作成用ハンドラとして登録されている）。
  • **呼び出し先**: apr_pcalloc（APRライブラリ）。

関連データ構造

  • authn_core_dir_conf: 本関数が生成するディレクトリ設定構造体。AuthType や AuthName を保持する。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がプロセスを終了させる（APRの標準的な挙動）。

保守担当者向けメモ

  • 本関数は Apache の設定システムから呼び出される標準的なコンストラクタである。
  • dummy 引数は将来的な拡張や API 互換性のために存在しており、現在の実装では無視されている。
