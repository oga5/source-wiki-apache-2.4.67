ap_recent_rfc822_date

定義

  • **種別**: function
  • **定義箇所**: server/util_time.c:282

目的

RFC 822 形式の時刻文字列を生成します。apr_rfc822_date() と同等の機能を提供しますが、キャッシュを利用した高速な時刻分解関数 ap_explode_recent_gmt() を使用することで、システムコール呼び出しを削減し、高頻度な時刻変換を最適化します。

シグネチャまたは形状

apr_status_t ap_recent_rfc822_date(char *date_str, apr_time_t t)

入力と出力

  • **入力**:
  • date_str: RFC 822 形式の文字列を格納するためのバッファ（十分なサイズが必要）。
  • t: 変換対象の時刻（apr_time_t）。
  • **出力**:
  • date_str: 変換された文字列（例: "Sat, 08 Jan 2000 18:31:41 GMT"）が書き込まれます。
  • 戻り値: APR_SUCCESS を返します。

前提条件と不変条件

  • date_str は、終端文字を含む RFC 822 形式の文字列を格納するために十分なメモリ領域を指している必要があります。
  • ap_explode_recent_gmt() の制約により、t がキャッシュの有効範囲（デフォルトで直近15秒程度）にある場合に最大のパフォーマンスを発揮します。

アルゴリズムまたは振る舞い

1. ap_explode_recent_gmt() を呼び出し、入力時刻 t を apr_time_exp_t 構造体に分解します。 2. 分解された時刻情報（曜日、日、月、年、時、分、秒）を、apr_day_snames および apr_month_snames を参照して文字列バッファへ順次コピーします。 3. 年の計算において 1900 + xt.tm_year を使用し、4桁の数値としてバッファに書き込みます。 4. 最後に " GMT" を付与し、ヌル終端文字で閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_filters.c
  • modules/http2/h2_c2_filter.c
  • modules/http2/h2_headers.c
  • **呼び出し先**:
  • ap_explode_recent_gmt()

関連データ構造

  • apr_time_exp_t: 時刻の分解結果を保持する構造体。
  • apr_day_snames, apr_month_snames: 曜日および月の短縮名テーブル。

失敗モードと境界条件

  • **西暦1万年問題**: 実装内のコメントにある通り、このルーチンは西暦10,000年以降の年数表記には対応していません。
  • **バッファオーバーフロー**: date_str が十分に確保されていない場合、メモリ破壊が発生します。

保守担当者向けメモ

  • 本関数は apr_rfc822_date() のクローンです。将来的なフォーマット変更が必要な場合は、この実装と標準の APR 実装の両方に注意を払う必要があります。
  • キャッシュの有効範囲外の時刻を扱う場合、ap_explode_recent_gmt() が内部でフォールバック処理を行いますが、パフォーマンス特性が変化することに留意してください。
