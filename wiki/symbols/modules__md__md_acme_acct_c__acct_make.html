acct_make

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_acct.c:41

目的

ACME アカウント情報を保持する md_acme_acct_t 構造体をメモリ上に新規作成し、初期化します。

シグネチャまたは形状

static apr_status_t acct_make(md_acme_acct_t **pacct, apr_pool_t *p, 
                              const char *ca_url, apr_array_header_t *contacts)

入力と出力

  • **入力**:
  • pacct: 生成されたアカウント構造体を格納するためのポインタのポインタ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • ca_url: アカウントに関連付ける ACME サーバーの URL。
  • contacts: アカウントの連絡先情報（文字列配列）。
  • **出力**:
  • pacct: 初期化された md_acme_acct_t 構造体へのポインタが設定されます。
  • 戻り値: 成功時は APR_SUCCESS を返します。

前提条件と不変条件

  • p は有効な apr_pool_t である必要があります。
  • ca_url は NULL であってはなりません。
  • 生成された md_acme_acct_t の contacts フィールドは、contacts が空または NULL の場合は空の配列として初期化され、それ以外の場合は元の配列のコピーが保持されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、指定されたプール p 上に md_acme_acct_t 構造体分のメモリを確保し、ゼロクリアします。 2. 引数 ca_url を構造体の ca_url フィールドに代入します。 3. contacts が NULL または空の配列である場合、容量 5 の空の文字列配列を apr_array_make で作成します。 4. contacts に値が存在する場合、apr_array_copy を使用してその内容を構造体にコピーします。 5. 最後に、生成した構造体へのポインタを pacct に格納し、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_acct.c:154

  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_is_empty_array (APR)
  • apr_array_make (APR)
  • apr_array_copy (APR)

関連データ構造

  • md_acme_acct_t: 本関数が生成・初期化する主要なアカウント構造体。

失敗モードと境界条件

  • メモリ不足が発生した場合、apr_pcalloc 等の APR 関数が異常終了（アボート）する可能性があります。
  • ca_url に NULL を渡した場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は static であり、md_acme_acct.c 内でのみ使用されます。
  • 連絡先情報のコピーには apr_array_copy を使用しているため、呼び出し元の contacts 配列が変更されても、生成されたアカウント構造体には影響しません。
