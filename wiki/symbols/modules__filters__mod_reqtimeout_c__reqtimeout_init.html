reqtimeout_init

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reqtimeout.c:370

目的

接続初期化時に mod_reqtimeout のフィルタ（reqtimeout_filter）をフィルタチェーンへ挿入し、接続ごとの状態管理構造体（reqtimeout_con_cfg）を初期化する。

シグネチャまたは形状

static int reqtimeout_init(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（現在の接続情報）
  • **出力**: int（OK または DECLINED）

前提条件と不変条件

  • c->base_server から取得した reqtimeout_srv_cfg が存在すること。
  • タイムアウト設定（ハンドシェイク、ヘッダー、ボディ）がすべて無効（0以下）な場合、本関数は何もせず DECLINED を返す。

アルゴリズムまたは振る舞い

1. サーバー設定（reqtimeout_srv_cfg）を取得し、タイムアウト設定が有効か確認する。無効であれば DECLINED を返す。 2. 接続設定（reqtimeout_con_cfg）が未割り当ての場合、apr_pcalloc でメモリを確保する。 3. 確保した ccfg を c->conn_config に設定する。 4. ap_add_output_filter および ap_add_input_filter を呼び出し、reqtimeout_filter_name をフィルタチェーンに挿入する。 5. 初期状態として ccfg->type を "handshake" に設定し、ハンドシェイクのタイムアウトが設定されていれば INIT_STAGE マクロで初期化する。 6. 最終的に DECLINED を返し、後続の接続処理を継続させる。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_process_connection により、接続処理の初期段階で呼び出される（modules/filters/mod_reqtimeout.c:634）。
  • **呼び出し先**:
  • ap_get_module_config / ap_set_module_config
  • apr_pcalloc
  • ap_add_output_filter / ap_add_input_filter
  • INIT_STAGE（マクロ）

関連データ構造

  • reqtimeout_con_cfg: 接続ごとの状態を保持する構造体。
  • reqtimeout_srv_cfg: サーバーごとの設定を保持する構造体。
  • conn_rec: Apache の接続情報構造体。

失敗モードと境界条件

  • reqtimeout_con_cfg が既に存在する場合、再初期化は行われない。
  • タイムアウト設定がすべて無効な場合、フィルタは挿入されず、本モジュールは接続に対して何の影響も及ぼさない。

保守担当者向けメモ

  • 本関数はフィルタの登録のみを行い、実際のタイムアウト監視は reqtimeout_filter が担う。
  • AP_DEBUG_ASSERT により、サーバー設定の欠落を開発時に検知するようになっている。
