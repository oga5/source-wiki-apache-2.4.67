is_header_regex

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_setenvif.c:170

目的

指定されたHTTPヘッダ名が正規表現を含んでいるかどうかを判定する。

シグネチャまたは形状

static int is_header_regex(apr_pool_t *p, const char* name)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール（使用箇所なし）。
  • name: 判定対象のヘッダ名文字列。
  • **出力**:
  • 1: ヘッダ名が正規表現であると判定された場合。
  • 0: ヘッダ名が正規表現ではない（標準的なヘッダ名文字のみで構成されている）と判定された場合。

前提条件と不変条件

  • is_header_regex_regex が事前にコンパイル済みである必要がある。
  • ヘッダ名に -, _, [A-Z], [a-z], [0-9] 以外の文字が含まれる場合、正規表現として扱う。

アルゴリズムまたは振る舞い

1. ap_regexec を使用し、is_header_regex_regex に対して name を照合する。 2. マッチングが失敗した場合（ap_regexec が非ゼロを返す場合）、正規表現が含まれていると判断し 1 を返す。 3. それ以外の場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_setenvif.c:383 (add_setenvif_core 内で呼び出されていることを確認)
  • **呼び出し先**: ap_regexec (Apache Core API)

関連データ構造

  • is_header_regex_regex: 判定に使用されるコンパイル済み正規表現オブジェクト（ファイルスコープ変数）。

失敗モードと境界条件

  • name が NULL の場合の挙動は、ap_regexec の実装に依存する。
  • is_header_regex_regex が初期化されていない場合、未定義の動作となる可能性がある。

保守担当者向けメモ

  • 本関数はヘッダ名が「正規表現か否か」を判定するヒューリスティックな役割を持つ。
  • apr_pool_t *p 引数は現在の実装では使用されていないが、将来的な拡張のためにシグネチャに含まれている。
