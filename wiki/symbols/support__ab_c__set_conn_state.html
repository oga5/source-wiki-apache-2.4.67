set_conn_state

定義

  • 種別: function
  • 定義箇所: support/ab.c:540

目的

HTTP接続の状態（connect_state_e）を更新し、対応するポーリングイベントを apr_pollset_t に反映させる。

シグネチャまたは形状

static void set_conn_state(struct connection *c, connect_state_e new_state)

入力と出力

  • **入力**:
  • c: 更新対象の接続構造体（struct connection *）。
  • new_state: 遷移先の接続状態（connect_state_e）。
  • **出力**: なし（void）。副作用として c->state が更新され、set_polled_events を通じてポーリング設定が変更される。

前提条件と不変条件

  • new_state は connect_state_e で定義された有効な値であること。
  • c は有効な接続構造体へのポインタであること。

アルゴリズムまたは振る舞い

1. 状態に応じたポーリングイベントのマップ（events_by_state）を参照する。

  • STATE_UNCONNECTED: 0（イベントなし）
  • STATE_CONNECTING: APR_POLLOUT
  • STATE_CONNECTED: APR_POLLIN
  • STATE_READ: APR_POLLIN

    • 2. c->state を new_state に更新する。 3. set_polled_events(c, events_by_state[new_state]) を呼び出し、APRのポーリング設定を同期する。

呼び出し元と呼び出し先

  • **呼び出し元**: support/ab.c 内の各所（行: 249, 922, 1478, 1483, 1504, 1552, 1707, 1829, 2079, 2091）。
  • **呼び出し先**: set_polled_events。

関連データ構造

  • struct connection: 接続状態を保持する構造体。
  • connect_state_e: 接続のライフサイクルを定義する列挙型。

失敗モードと境界条件

  • new_state が events_by_state 配列の範囲外である場合、未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • STATE_CONNECTED 状態では本来ポーリングを行わない設計意図があるが、events_by_state では APR_POLLIN が割り当てられている。これは STATE_READ への遷移を見越した設定である。
  • 状態遷移のロジックを変更する場合は、events_by_state 配列のインデックスと connect_state_e の定義順序が一致していることを確認すること。
