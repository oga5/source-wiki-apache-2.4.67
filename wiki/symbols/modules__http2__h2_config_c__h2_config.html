h2_config

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_config.c:54

目的

Apache HTTP Server の mod_http2 モジュールにおいて、HTTP/2 通信に関連するサーバーレベルの設定パラメータを保持・管理するための構造体です。

シグネチャまたは形状

typedef struct h2_config {
    const char *name;
    int h2_max_streams;
    int h2_window_size;
    int min_workers;
    int max_workers;
    apr_interval_time_t idle_limit;
    int stream_max_mem_size;
    int h2_direct;
    int modern_tls_only;
    int h2_upgrade;
    apr_int64_t tls_warmup_size;
    int tls_cooldown_secs;
    int h2_push;
    struct apr_hash_t *priorities;
    int push_diary_size;
    int copy_files;
    apr_array_header_t *push_list;
    apr_table_t *early_headers;
    int early_hints;
    int padding_bits;
    int padding_always;
    int output_buffered;
    apr_interval_time_t stream_timeout;
    int max_data_frame_len;
    int max_hd_block_len;
    int max_stream_errors;
    int proxy_requests;
    int h2_websockets;
} h2_config;

入力と出力

  • 本構造体は、Apache の設定フェーズでメモリプール上に確保されます。
  • h2_config_merge 等の関数を通じて、親コンテキストの設定値とマージされ、最終的な設定値が決定されます。

前提条件と不変条件

  • すべての設定値は Apache のメモリプール（apr_pool_t）上で管理されます。
  • マージ処理は既存の構造体を変更せず、新しい構造体を生成して返します。
  • 値 -1 は「未設定（デフォルト値）」を意味するマジックナンバーとして扱われます。

アルゴリズムまたは振る舞い

  • mod_http2 の設定システムにおける中心的なデータ保持構造です。
  • h2_config_rgeti 等のゲッター関数により、リクエストやコネクションのコンテキストに応じて、ディレクトリ設定とサーバー設定を階層的に解決するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_config.c 内の初期化関数（apr_pcalloc による確保）およびマージ関数。
  • 呼び出し先: 直接的な関数呼び出し関係は本構造体の定義からは特定できませんが、h2_session.c 等のモジュール内コンポーネントがこの構造体を参照して動作パラメータを取得します。

関連データ構造

  • h2_dir_config: ディレクトリ単位の設定を保持する構造体。
  • apr_hash_t *priorities: コンテンツタイプと優先度のマッピング。
  • apr_array_header_t *push_list: サーバープッシュ設定のリスト。

失敗モードと境界条件

  • 設定値が未設定（-1）のままマージされた場合、階層構造の上位（サーバー設定）へフォールバックされます。
  • メモリ不足時には apr_pcalloc が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は h2_config.h で公開インターフェースとして定義されています。
  • 構造体メンバを追加・変更した際は、h2_config.c 内のマージ関数およびゲッター/セッター関数への反映が必要です。
