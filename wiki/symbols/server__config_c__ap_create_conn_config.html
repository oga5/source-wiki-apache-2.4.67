ap_create_conn_config

定義

  • 種別: function
  • 定義箇所: server/config.c:361

目的

接続（connection）単位の設定を保持するための空の設定ベクトルを作成します。

シグネチャまたは形状

ap_conf_vector_t *ap_create_conn_config(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（設定ベクトルを割り当てるためのメモリプール）
  • **出力**: ap_conf_vector_t *（初期化された空の設定ベクトルへのポインタ）

前提条件と不変条件

  • 引数 p は有効な apr_pool_t である必要があります。
  • 戻り値として返される設定ベクトルは、モジュールごとの設定を格納するために使用されます。

アルゴリズムまたは振る舞い

内部的に create_empty_config(p) を呼び出し、指定されたメモリプールを使用して空の設定ベクトルを生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2.c:839
  • server/core.c:5256
  • **呼び出し先**:
  • create_empty_config (同一ファイル内)

関連データ構造

  • ap_conf_vector_t: モジュールごとの設定データを保持する不透明なベクトル型。
  • apr_pool_t: Apache Portable Runtime のメモリ管理プール。

失敗モードと境界条件

  • メモリ不足時には apr_pool_t の割り当てメカニズムに従い、例外またはエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は http_config.h で宣言されており、接続ライフサイクルにおける設定の初期化ポイントとして機能します。
  • create_empty_config への単なるラッパーですが、将来的に接続固有の初期化ロジックが必要となった場合はこの関数を拡張してください。
