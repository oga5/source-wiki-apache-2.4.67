merge_cern_meta_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_cern_meta.c:155

目的

ディレクトリ単位の設定構造体 cern_meta_dir_config をマージし、親ディレクトリの設定を継承または子ディレクトリの設定で上書きするための関数です。

シグネチャまたは形状

static void *merge_cern_meta_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定 (cern_meta_dir_config へのポインタ)。
  • addv: 現在のディレクトリの設定 (cern_meta_dir_config へのポインタ)。
  • **出力**:
  • マージされた新しい cern_meta_dir_config 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は cern_meta_dir_config 型にキャスト可能であること。
  • p はリクエストまたは設定のライフサイクルに紐付いた有効なプールであること。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、新しい cern_meta_dir_config 構造体用のメモリを確保します。 2. metadir および metasuffix については、add（子）側に値が存在すればそれを採用し、存在しなければ base（親）側の値を継承します。 3. metafiles については、add 側の値をそのまま採用します。 4. マージ済みの構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_cern_meta.c:366 において、Apache の設定マージ処理の一部として参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、メモリ管理のために apr_palloc を使用しています。

関連データ構造

  • cern_meta_dir_config: 本関数が操作するディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性があります。
  • basev または addv が NULL の場合、構造体メンバへのアクセス時にセグメンテーションフォールトが発生する可能性があります（Apache の設定フレームワーク上、通常は適切に初期化されたポインタが渡されます）。

保守担当者向けメモ

  • 本関数は、Apache の設定継承メカニズムにおいて「デフォルトは上書き」という方針に従って実装されています。
  • metafiles メンバは継承されず、常に add 側の値が優先される点に注意してください。
