dav_lock_query

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:369

目的

指定されたリソースに関連付けられたロックのリストを取得します。

シグネチャまたは形状

dav_error* dav_lock_query(dav_lockdb *lockdb, const dav_resource *resource, dav_lock **locks);

入力と出力

  • **入力**:
  • lockdb: ロックデータベースへのポインタ。
  • resource: ロック情報を取得する対象の dav_resource。
  • locks: 取得したロックのリストを格納するためのポインタのポインタ。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は dav_error 構造体へのポインタを返します。
  • locks 引数には、解決済みのロックリストが設定されます。

前提条件と不変条件

  • lockdb が NULL の場合、関数は即座に locks を NULL に設定し、正常終了します。
  • lockdb が有効な場合、内部の get_locks フックが呼び出されます。

アルゴリズムまたは振る舞い

1. lockdb が NULL かどうかを判定し、該当する場合は空の結果を返して終了します。 2. lockdb が存在する場合、lockdb->hooks->get_locks を呼び出します。 3. この際、DAV_GETLOCKS_RESOLVED フラグを渡し、ロック情報を解決済み状態で取得します。 4. フックの戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/repos.c:1755
  • modules/dav/main/props.c:364
  • modules/dav/main/util.c:966
  • modules/dav/main/util.c:1945
  • **呼び出し先**:
  • lockdb->hooks->get_locks (関数ポインタ経由で実行)

関連データ構造

  • dav_lockdb: ロックデータベースのハンドル。
  • dav_resource: WebDAV リソースの抽象表現。
  • dav_lock: ロック情報を保持する連結リスト構造体。

失敗モードと境界条件

  • lockdb が NULL の場合はエラーにならず、空のロックリストが返されます。
  • lockdb->hooks->get_locks がエラーを返した場合、その dav_error がそのまま伝播されます。

保守担当者向けメモ

  • 本関数はロックデータベースのフックを介したラッパーです。
  • DAV_GETLOCKS_RESOLVED フラグにより、ロックの解決処理がバックエンドに委譲されています。
