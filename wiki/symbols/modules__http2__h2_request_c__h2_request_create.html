h2_request_create

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:47

目的

HTTP/2 リクエストを表現する h2_request 構造体のメモリ確保および初期化を行います。

シグネチャまたは形状

h2_request *h2_request_create(int id, apr_pool_t *pool, const char *method,
                              const char *scheme, const char *authority,
                              const char *path, apr_table_t *header)

入力と出力

  • **入力**:
  • id: ストリーム識別子。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • method, scheme, authority, path: リクエストの各要素を示す文字列。
  • header: リクエストヘッダーを格納する apr_table_t（NULL の場合は新規作成）。
  • **出力**:
  • 初期化済みの h2_request 構造体へのポインタ。

前提条件と不変条件

  • pool は有効なメモリプールである必要があります。
  • header が NULL の場合、内部で apr_table_make(pool, 10) を使用して新しいテーブルが生成されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して h2_request 構造体分のメモリをゼロクリアして確保します。 2. 引数として渡された method, scheme, authority, path を構造体の各フィールドに代入します。 3. header が提供されている場合はそれを、提供されていない場合は新規作成したテーブルを headers フィールドに設定します。 4. request_time に apr_time_now() を使用して現在時刻を記録します。 5. 初期化済みの構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_push.c:351
  • modules/http2/h2_stream.c:784
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_table_make (APR)
  • apr_time_now (APR)

関連データ構造

  • h2_request: HTTP/2 リクエストのメタデータを保持する構造体。
  • apr_table_t: HTTP ヘッダーを管理するための APR テーブル。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がプロセスを終了させる可能性があります（APR の標準的な挙動）。
  • header が NULL の場合、初期容量 10 のテーブルが自動的に生成されます。

保守担当者向けメモ

  • 本関数は h2_request のライフサイクルの起点となります。
  • id 引数は関数内で受け取りますが、現在の実装では構造体フィールドへの代入に使用されていません。将来的な拡張やデバッグ用途の可能性があります。
