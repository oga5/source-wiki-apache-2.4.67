modssl_pk_proxy_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:725

目的

プロキシ接続において、複数の証明書および秘密鍵のペアを保持・管理するための構造体です。

シグネチャまたは形状

typedef struct {
    const char  *cert_file;
    const char  *cert_path;
    const char  *ca_cert_file;
    STACK_OF(X509_INFO) *certs;
    STACK_OF(X509) **ca_certs;
} modssl_pk_proxy_t;

入力と出力

  • **入力**: cert_file（証明書ファイルパス）、cert_path（証明書ディレクトリパス）、ca_cert_file（CA証明書ファイルパス）を保持します。
  • **出力**: certs に読み込まれた証明書・鍵ペアのスタック、および ca_certs に対応する証明書チェーンのスタック配列を保持します。

前提条件と不変条件

  • ca_certs[n] は、certs スタックの n 番目の要素に対応する証明書チェーン（STACK_OF(X509) *）を指します。
  • チェーンが設定されていない場合、ca_certs[n] は NULL となります。

アルゴリズムまたは振る舞い

  • 複数の証明書および鍵ペアをスタックとして保持し、プロキシ通信時にこれらを動的に参照可能にします。
  • ca_certs は certs と同期して管理され、各証明書に対応する中間証明書チェーンを個別に保持する構造となっています。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c 内で mctx->pkp として参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは含まれません。OpenSSL の STACK_OF 型を操作する関数群（外部ライブラリ）と組み合わせて使用されます。

関連データ構造

  • STACK_OF(X509_INFO): 証明書と鍵のペアを格納する OpenSSL スタック。
  • STACK_OF(X509): 証明書チェーンを格納する OpenSSL スタック。

失敗モードと境界条件

  • certs または ca_certs が初期化されていない場合、あるいはメモリ不足によりスタックの割り当てに失敗した場合の挙動は、呼び出し側の実装に依存します。
  • ca_certs のインデックスが certs の要素数を超えてアクセスされた場合の境界チェックは、本構造体定義内では規定されていません。

保守担当者向けメモ

  • 本構造体はプロキシ機能の基盤となるため、証明書チェーンの追加や削除を行う際は certs と ca_certs の整合性を維持する必要があります。
  • OpenSSL 3.0 への移行に伴い、X509_INFO や STACK_OF の取り扱いに変更が生じた場合は、本構造体のメモリ管理方針を再確認してください。
