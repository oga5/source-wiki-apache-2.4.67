dav_format_time

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/repos.c:297

目的

指定された時刻（apr_time_t）を、WebDAV で使用される特定のフォーマット（ISO 8601 または RFC 822）に変換し、提供されたバッファに書き込みます。

シグネチャまたは形状

static void dav_format_time(int style, apr_time_t sec, char *buf, apr_size_t buflen)

入力と出力

  • style: フォーマット形式（DAV_STYLE_ISO8601 またはそれ以外）。
  • sec: 変換対象の時刻（apr_time_t）。
  • buf: 結果を格納する文字バッファ。
  • buflen: buf のサイズ。

前提条件と不変条件

  • apr_time_exp_gmt による時刻変換が成功することを前提としています（失敗時の挙動は未定義）。
  • 出力バッファ buf は、ISO 8601 形式の場合は少なくとも 21 バイト、RFC 822 形式の場合は少なくとも 30 バイトの容量が必要です。

アルゴリズムまたは振る舞い

1. apr_time_exp_gmt を使用して、入力時刻を GMT の分解時刻構造体（apr_time_exp_t）に変換します。 2. style が DAV_STYLE_ISO8601 の場合、YYYY-MM-DDThh:mm:ssZ 形式でバッファに書き込みます。 3. それ以外の場合、RFC 822 形式（Day, DD Mon YYYY hh:mm:ss GMT）でバッファに書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/repos.c:1967 (DAV_STYLE_ISO8601 を指定)
  • modules/dav/fs/repos.c:1987 (DAV_STYLE_RFC822 を指定)
  • **呼び出し先**:
  • apr_time_exp_gmt
  • apr_snprintf

関連データ構造

  • apr_time_t
  • apr_time_exp_t
  • apr_day_snames (グローバル定数)
  • apr_month_snames (グローバル定数)

失敗モードと境界条件

  • apr_time_exp_gmt が失敗した場合、tms 構造体の内容は不定のまま処理が継続されます。
  • バッファサイズ buflen が不足している場合、apr_snprintf の仕様に従い切り詰めが発生します。

保守担当者向けメモ

  • apr_time_exp_gmt の戻り値チェックは現在行われていません。
  • ISO 8601 形式において、タイムゾーンオフセットとして "Z" を使用するか "-00:00" を使用するかについて検討の余地がある旨のコメントがソース内に存在します。
