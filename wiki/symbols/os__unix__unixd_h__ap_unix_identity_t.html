ap_unix_identity_t

定義

  • 種別: struct
  • 定義箇所: os/unix/unixd.h:56

目的

suexec などの機能において、プロセス実行時のユーザーおよびグループの識別情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    uid_t uid;
    gid_t gid;
    int userdir;
} ap_unix_identity_t;

入力と出力

  • **入力**: 構造体の各フィールド（uid, gid, userdir）に値を設定することで、特定の実行権限コンテキストを定義します。
  • **出力**: 構造体全体を介して、特定のリクエストやプロセス生成に必要なユーザー・グループ情報を伝播させます。

前提条件と不変条件

  • uid および gid は、システム上の有効なユーザー ID およびグループ ID を指す必要があります。
  • userdir はフラグとして機能し、特定のディレクトリ制御に関連する状態を保持します。

アルゴリズムまたは振る舞い

本構造体はデータの保持を目的としており、それ自体にロジックは含まれません。主に以下の用途で使用されます。 1. suexec 実行時のユーザー・グループ情報の受け渡し。 2. mod_cgid や mod_userdir における実行権限の管理。 3. memcpy 等による構造体単位での状態コピーおよび保持。

呼び出し元と呼び出し先

  • **呼び出し元（参照・利用）**:
  • modules/generators/mod_cgid.c
  • modules/generators/mod_suexec.c
  • modules/mappers/mod_userdir.c
  • modules/examples/mod_example_hooks.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • unixd_config_rec: サーバー全体の Unix 実行環境設定を保持する構造体。

失敗モードと境界条件

  • uid や gid に -1 を設定することで、無効または未設定の状態を表現する慣習があります（例: mod_cgid.c での empty_ugid 定義）。

保守担当者向けメモ

  • 本構造体は suexec 関連のモジュール間で頻繁に受け渡されるため、フィールドの追加や変更を行う際は、関連するすべてのモジュール（mod_cgid, mod_suexec, mod_userdir 等）の整合性を確認してください。
