get_next

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_workers.c:204

目的

ワーカープール内のスロットに対して、アクティブな接続プロデューサーから処理すべき次の接続（conn_rec）を取得します。

シグネチャまたは形状

static conn_rec *get_next(h2_slot *slot)

入力と出力

  • **入力**: slot (h2_slot *) - 現在のワーカーの実行スロット。
  • **出力**: conn_rec * - 処理対象の接続。取得できない場合は NULL を返します。

前提条件と不変条件

  • workers->lock による排他制御下で呼び出されることを想定しています。
  • slot->prod は呼び出し時にリセットされます。
  • プロデューサーがアクティブな場合、prod->state は PROD_ACTIVE である必要があります。

アルゴリズムまたは振る舞い

1. workers->prod_active リングが空でないか確認します。 2. リングの先頭からプロデューサーを取り出し、slot->prod に割り当てます。 3. プロデューサーの fn_next コールバックを呼び出し、接続 c を取得します。 4. 取得した接続にまだ続きがある場合（has_more が真）:

  • プロデューサーを prod_active リングの末尾に再挿入します。
  • wake_idle_worker を呼び出し、他のアイドルワーカーを起床させます。

    • 5. 続きがない場合:

  • プロデューサーの状態を PROD_IDLE に変更し、prod_idle リングへ移動させます。

    • 6. 接続が取得できた場合、prod->conns_active をインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**: slot_run (modules/http2/h2_workers.c:256)
  • **呼び出し先**:
  • prod->fn_next (プロデューサーのコールバック関数)
  • wake_idle_worker (アイドルワーカーの起床)

関連データ構造

  • h2_workers: ワーカープール全体を管理する構造体。
  • h2_slot: ワーカーの状態とプロデューサーへの参照を保持。
  • ap_conn_producer_t: 接続の供給元。

失敗モードと境界条件

  • prod_active リングが空の場合、何も取得せず NULL を返します。
  • fn_next が NULL を返した場合、接続処理は行われません。

保守担当者向けメモ

  • 本関数は h2_workers のリング操作を直接行うため、ロックの整合性に注意してください。
  • prod->conns_active のインクリメントは、接続が正常に取得された場合にのみ行われます。
