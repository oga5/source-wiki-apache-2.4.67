uwsgi_handler

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_uwsgi.c:500

目的

uwsgi:// スキームのリクエストを処理し、Apache HTTP Server と uWSGI バックエンド間のプロキシ通信を統括するメインエントリポイントです。

シグネチャまたは形状

static int uwsgi_handler(request_rec *r, proxy_worker *worker,
                         proxy_server_conf *conf, char *url,
                         const char *proxyname, apr_port_t proxyport)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • worker: 接続先のバックエンドワーカー情報。
  • conf: プロキシサーバーの設定。
  • url: プロキシ対象の URL 文字列。
  • proxyname, proxyport: プロキシ設定値。
  • **出力**:
  • int: 処理結果ステータス（OK, DECLINED, HTTP_INTERNAL_SERVER_ERROR, HTTP_SERVICE_UNAVAILABLE 等）。

前提条件と不変条件

  • URL が uwsgi:// で始まる必要があります。
  • r->subprocess_env に PATH_INFO が設定されます。
  • 処理終了時には必ず ap_proxy_release_connection が呼び出されます。
  • 現在の実装では、バックエンドとの接続はリクエストごとに切断（backend->close = 1）されます。

アルゴリズムまたは振る舞い

1. **URL 検証**: uwsgi:// スキームを確認し、一致しない場合は DECLINED を返します。 2. **PATH_INFO 抽出**: URL からパス情報を抽出し、デコードして subprocess_env に追加します。 3. **接続確立**: ap_proxy_acquire_connection で接続を確保し、ap_proxy_determine_connection で接続先を特定後、ap_proxy_connect_backend でソケットを接続します。 4. **リクエスト処理**: 以下の順序で実行し、いずれかが失敗した場合は cleanup へ遷移します。

  • ap_setup_client_block: クライアントからのデータ受信準備。
  • uwsgi_send_headers: 環境変数をバイナリ形式に変換して送信。
  • uwsgi_send_body: リクエストボディをバックエンドへ転送。
  • uwsgi_response: バックエンドからのレスポンスを受信・解析。

    • 5. **終了処理**: 接続を閉じ、プールを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_uwsgi.c 内の proxy_hook_scheme_handler に登録されたフック経由で mod_proxy から呼び出されます。
  • **呼び出し先**:
  • ap_proxy_acquire_connection, ap_proxy_determine_connection, ap_proxy_connect_backend, ap_proxy_connection_create, ap_proxy_release_connection (mod_proxy API)
  • uwsgi_send_headers, uwsgi_send_body, uwsgi_response (同ファイル内ユーティリティ)

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • proxy_worker: バックエンドワーカーの設定。
  • proxy_conn_rec: バックエンドとの接続状態。

失敗モードと境界条件

  • **URL 不正**: uwsgi:// で始まらない場合は DECLINED。
  • **デコード失敗**: PATH_INFO のデコードに失敗すると HTTP_INTERNAL_SERVER_ERROR。
  • **接続失敗**: バックエンドへの接続確立に失敗すると HTTP_SERVICE_UNAVAILABLE。
  • **通信エラー**: ヘッダー送信、ボディ転送、レスポンス受信のいずれかでエラーが発生した場合、そのステータスを返して終了します。

保守担当者向けメモ

  • 本関数はリクエストごとにバックエンド接続をクローズします。Keep-Alive を実装する場合は backend->close の制御を見直す必要があります。
  • uwsgi_send_headers がパケットサイズ制限（APR_UINT16_MAX）を持つため、大規模な環境変数を扱う際は注意してください。
