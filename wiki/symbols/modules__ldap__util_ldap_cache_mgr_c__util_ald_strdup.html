util_ald_strdup

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache_mgr.c:117

目的

キャッシュ管理用のメモリ領域（共有メモリまたはヒープ）に対して、文字列の複製を作成する。

シグネチャまたは形状

const char *util_ald_strdup(util_ald_cache_t *cache, const char *s);

入力と出力

  • **入力**:
  • cache: キャッシュ管理構造体へのポインタ。
  • s: 複製対象のヌル終端文字列。
  • **出力**:
  • 複製された文字列へのポインタ。メモリ確保に失敗した場合は NULL を返す。

前提条件と不変条件

  • cache が有効な構造体であること。
  • s が有効な文字列であること。
  • 共有メモリ（APR_HAS_SHARED_MEMORY）が有効な場合、cache->rmm_addr が初期化されていれば、その領域からメモリが割り当てられる。

アルゴリズムまたは振る舞い

1. 共有メモリが有効かつ cache->rmm_addr が設定されている場合:

  • apr_rmm_calloc を使用して、文字列長（strlen(s) + 1）分のメモリを共有メモリ領域から確保する。
  • 確保したオフセットを apr_rmm_addr_get で実アドレスに変換する。
  • 成功すれば strcpy で文字列をコピーし、そのアドレスを返す。

    • 2. 上記以外（共有メモリが無効、または rmm_addr が未設定）の場合:

  • 標準の strdup(s) を呼び出し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap_cache.c（複数箇所: URL、DN、属性値、パスワード等の複製に使用）
  • modules/ldap/util_ldap_cache_mgr.c:159
  • **呼び出し先**:
  • strlen, strcpy, strdup（標準Cライブラリ）
  • apr_rmm_calloc, apr_rmm_addr_get（APR RMM API）

関連データ構造

  • util_ald_cache_t: キャッシュのメモリ管理状態を保持する構造体。

失敗モードと境界条件

  • メモリ不足: apr_rmm_calloc または strdup が失敗した場合、NULL を返す。
  • s が NULL の場合: strlen(s) でクラッシュする可能性があるため、呼び出し側でガードが必要。

保守担当者向けメモ

  • 共有メモリ環境下では、返されるポインタは apr_rmm 管理下の領域を指すため、解放には util_ald_free を使用する必要がある。
  • APR_HAS_SHARED_MEMORY の有無によりメモリ確保の挙動が切り替わるため、キャッシュの生存期間とメモリ所有権の管理には注意すること。
