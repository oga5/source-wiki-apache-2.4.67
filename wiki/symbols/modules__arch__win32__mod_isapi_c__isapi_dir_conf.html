isapi_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/arch/win32/mod_isapi.c:70

目的

Apache HTTP Server 上で動作する ISAPI 拡張のディレクトリごとの設定を保持するために使用されます。バッファサイズ、ログ出力設定、および非同期エミュレーションの有効化状態を管理します。

シグネチャまたは形状

typedef struct isapi_dir_conf {
    int read_ahead_buflen;
    int log_unsupported;
    int log_to_errlog;
    int log_to_query;
    int fake_async;
} isapi_dir_conf;

入力と出力

  • 入力: 設定ファイル（httpd.conf 等）で指定されたディレクティブ値。
  • 出力: モジュール内のリクエスト処理において、ISAPI 拡張の挙動を制御するためのパラメータとして参照されます。

前提条件と不変条件

  • リクエスト処理中、r->pool 上に割り当てられたインスタンスが使用されます。
  • fake_async が有効な場合、同期的な Apache パイプライン上で非同期 I/O を擬似的に実現するため、ミューテックスによる待機が発生します。

アルゴリズムまたは振る舞い

  • 設定の初期化およびマージ処理において、apr_palloc を介してメモリが確保されます。
  • APR_OFFSETOF マクロを使用して、各メンバ変数が Apache の設定解析用テーブルとマッピングされます。

呼び出し元と呼び出し先

  • 呼び出し元: mod_isapi.c 内のディレクトリ設定作成関数（create_isapi_dir_config）およびマージ関数（merge_isapi_dir_configs）から参照されます。
  • 呼び出し先: 直接的な関数呼び出しは行いません。設定値として isapi_handler 等の処理ロジックから読み取られます。

関連データ構造

  • EXTENSION_CONTROL_BLOCK: ISAPI 拡張に渡されるリクエストコンテキスト。
  • isapi_cid: リクエスト処理中の状態を保持するコンテキスト構造体。

失敗モードと境界条件

  • fake_async を有効にした場合、高負荷時にスレッド枯渇のリスクがあります。
  • 設定値が未定義の場合のデフォルト値については、本構造体定義単体では規定されていません。

保守担当者向けメモ

  • 本構造体は mod_isapi.c 内で完結しており、他のモジュールからは直接操作されません。
  • メンバ変数の追加や変更を行う際は、mod_isapi.c 内のコマンド定義テーブル（APR_OFFSETOF を使用している箇所）との整合性に注意してください。
