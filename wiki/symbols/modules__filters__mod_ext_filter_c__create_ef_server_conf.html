create_ef_server_conf

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ext_filter.c:95

目的

サーバー単位の外部フィルタ設定構造体（ef_server_t）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_ef_server_conf(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定情報の割り当てに使用するメモリプール。
  • s: 設定が適用されるサーバーレコード（本関数内では未使用）。
  • **出力**:
  • 初期化済みの ef_server_t 構造体へのポインタ。

前提条件と不変条件

  • p は有効な apr_pool_t であること。
  • 割り当てられた ef_server_t の p メンバには、引数 p が格納される。
  • フィルタ定義を保持するためのハッシュテーブル h が apr_hash_make により生成されていること。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、メモリプール p 上に ef_server_t サイズのメモリをゼロクリアして確保する。 2. 確保した構造体の p メンバにメモリプール p を保存する。 3. apr_hash_make を呼び出し、フィルタ定義を格納するためのハッシュテーブルを作成して h メンバに代入する。 4. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の設定読み込みフェーズにおいて、サーバー設定生成時に呼び出される（modules/filters/mod_ext_filter.c:950 付近のコマンド登録構造体経由）。
  • **呼び出し先**:
  • apr_pcalloc
  • apr_hash_make

関連データ構造

  • ef_server_t: サーバー単位の設定を保持する構造体。本関数で生成される。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がエラーを発生させる（Apache の標準的なメモリ管理に従う）。

保守担当者向けメモ

  • 本関数は mod_ext_filter の設定ライフサイクルの開始点です。
  • server_rec *s 引数は現在の実装では使用されていませんが、将来的なサーバーごとの設定拡張のためにシグネチャに含まれています。
