sniff_encoding

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_xml2enc.c:183

目的

HTTPレスポンスのコンテンツから文字エンコーディングを特定し、必要に応じて xml2ctx 構造体にエンコーディング情報を設定する。

シグネチャまたは形状

static void sniff_encoding(request_rec* r, xml2ctx* ctx)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報 (request_rec)。
  • ctx: フィルタの実行状態を保持する xml2ctx 構造体（バッファ buf や現在のエンコーディング状態を含む）。
  • **出力**:
  • ctx->encoding および ctx->xml2enc が更新される。
  • r->content_type が UTF-8 を示すように書き換えられる場合がある。

前提条件と不変条件

  • ctx->buf には解析対象のデータが格納されている必要がある。
  • libxml2 のエンコーディング検出機能および apr_xlate が利用可能であること。

アルゴリズムまたは振る舞い

1. **HTTPヘッダー解析**: Content-Type ヘッダーに charset= が含まれる場合、それを優先して採用する。 2. **BOM検出**: ヘッダーで特定できない場合、xmlDetectCharEncoding を用いてバイトオーダーマーク（BOM）からエンコーディングを推論する。 3. **HTML <meta> タグ解析**: 上記で特定できない場合、バッファ内の <meta> タグを正規表現で検索し、charset 属性を抽出する。該当タグはバッファから削除される。 4. **フォールバック**:

  • libxml2 でサポートされない場合、apr_xlate_open を試行して変換可能か確認する。
  • 最終的に特定できない場合は、設定ファイル (xml2cfg) のデフォルト値を使用する。

    • 5. **ヘッダー更新**: 検出または決定されたエンコーディングに基づき、r->content_type を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_xml2enc.c:389 (フィルタのメインループ内)
  • **呼び出し先**:
  • libxml2 関数群 (xmlParseCharEncoding, xmlDetectCharEncoding, xmlGetCharEncodingName)
  • Apache API (ap_log_rerror, ap_regexec, apr_pstrndup, apr_brigade_partition, apr_xlate_open 等)

関連データ構造

  • xml2ctx: フィルタの実行状態を保持。
  • xml2cfg: モジュールのディレクトリごとの設定。
  • request_rec: Apache のリクエスト情報。

失敗モードと境界条件

  • **エンコーディング未検出**: 設定されたデフォルト値（デフォルトは ISO-8859-1）にフォールバックする。
  • **変換不可**: apr_xlate が失敗した場合、エラーログを出力する。
  • **メタタグ削除**: <meta> タグの抽出時に apr_brigade_partition が失敗した場合、ap_assert により停止する。

保守担当者向けメモ

  • libxml2 の内部的な <meta> タグサポートとの競合を避けるため、本関数で明示的にタグの除去を行っている。
  • ap_assert の使用箇所は、将来的にエラーハンドリングへの置き換えが推奨される。
