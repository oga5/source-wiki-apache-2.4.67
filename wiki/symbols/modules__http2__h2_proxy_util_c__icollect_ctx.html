icollect_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.c:177

目的

ハッシュテーブルから整数 ID を収集し、指定されたバッファへ格納するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    h2_proxy_ihash_t *ih;
    int *buffer;
    size_t max;
    size_t len;
} icollect_ctx;

入力と出力

  • **入力**:
  • ih: 収集対象となる h2_proxy_ihash_t へのポインタ。
  • buffer: 収集した整数を格納する配列。
  • max: buffer の最大容量。
  • **出力**:
  • len: 実際に収集された要素数。

前提条件と不変条件

  • ih は初期化済みである必要があります。
  • buffer は max 以上の要素数を保持できるメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

ハッシュテーブル ih 内の要素を走査し、その整数 ID を buffer に順次コピーします。len は収集された要素の現在数を示します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_proxy_util.c 内の関数（x を介したキャストや、ローカル変数としての宣言が確認されています）。
  • 呼び出し先: 情報なし。

関連データ構造

  • h2_proxy_ihash_t: 整数 ID をキーとして管理するハッシュテーブル。

失敗モードと境界条件

  • len が max を超える場合、バッファオーバーフローが発生する可能性があるため、呼び出し側で max を超えないよう制御する必要があります。

保守担当者向けメモ

  • 本構造体は h2_proxy_ihash_t の内容を配列形式へ変換する際に使用されます。利用時は len が max を超えないことを保証してください。
