dav_lock_combined

定義

  • 種別: struct
  • 定義箇所: modules/dav/lock/locks.c:198

目的

ロック情報をメモリ上で保持するために、公開用のロック構造体、内部管理用のプライベート構造体、およびロックトークンを統合したコンテナ構造体です。

シグネチャまたは形状

typedef struct
{
    dav_lock pub;
    dav_lock_private priv;
    dav_locktoken token;
} dav_lock_combined;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。メモリ上のロックレコードを構築する際に、dav_lock（公開情報）、dav_lock_private（内部状態）、dav_locktoken（識別子）を一つのメモリブロックとして管理するために使用されます。

前提条件と不変条件

  • dav_lock、dav_lock_private、dav_locktoken の各フィールドが適切に初期化されている必要があります。
  • メモリ割り当ては、関連する request_rec のプールから行われることが想定されています。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用ですが、modules/dav/lock/locks.c 内のロック操作において、DBM から読み込まれたバイナリデータをデシリアライズして格納する際のターゲットとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/lock/locks.c および modules/dav/fs/lock.c 内の関数で、ロックレコードを扱う際にポインタとして参照されています。
  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • dav_lock: 公開されているロック情報。
  • dav_lock_private: ロックの内部的な管理状態。
  • dav_locktoken: ロックを一意に識別するための UUID ラッパー。

失敗モードと境界条件

  • 構造体のサイズは、各フィールドの合計サイズに依存します。シリアライズされたデータとの整合性が取れない場合、メモリ破壊や不正なデータ読み込みが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は locks.c 内で定義されていますが、modules/dav/fs/lock.c からも参照されています。構造を変更する場合は、これら両方のファイルにおけるメモリレイアウトへの影響を考慮してください。
