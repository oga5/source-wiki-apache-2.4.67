register_hooks

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_user.c:126

目的

Apache HTTP Server の起動時に、mod_authz_user モジュールが提供する認可プロバイダー（user および valid-user）を mod_authz_core フレームワークへ登録します。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（モジュールのライフサイクルに関連付けられたメモリプール）
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • ap_register_auth_provider が利用可能であること。
  • authz_user_provider および authz_validuser_provider が定義済みであること。

アルゴリズムまたは振る舞い

1. ap_register_auth_provider を呼び出し、AUTHZ_PROVIDER_GROUP グループに対して "user" プロバイダーを登録します。この際、authz_user_provider 構造体を指定します。 2. 同様に、ap_register_auth_provider を呼び出し、"valid-user" プロバイダーを authz_validuser_provider 構造体を用いて登録します。 3. 両登録において、フラグ AP_AUTH_INTERNAL_PER_CONF を使用し、設定ごとの内部認可プロバイダーとして構成します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール読み込みプロセス（直接の呼び出し元はソースコード上では明示されていませんが、Apache のモジュールフック機構により起動時に呼び出されます）。
  • **呼び出し先**: ap_register_auth_provider（Apache コアのプロバイダー登録関数）。

関連データ構造

  • authz_user_provider: user 認可プロバイダーのインターフェース定義。
  • authz_validuser_provider: valid-user 認可プロバイダーのインターフェース定義。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • 登録処理自体は戻り値を返さないため、関数内部での失敗は Apache の起動プロセス全体に影響を及ぼす可能性があります。

保守担当者向けメモ

  • 本関数はモジュールの初期化時に一度だけ実行されます。
  • 新しい認可プロバイダーを追加する場合は、この関数内で ap_register_auth_provider を追加してください。
