md_acme_order_from_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:140

目的

JSON 形式のデータから ACME Order オブジェクトを復元（デシリアライズ）します。

シグネチャまたは形状

md_acme_order_t *md_acme_order_from_json(md_json_t *json, apr_pool_t *p)

入力と出力

  • **入力**:
  • json: 復元元の JSON データ構造体。
  • p: メモリ割り当てに使用する APR プール。
  • **出力**:
  • 復元された md_acme_order_t 構造体へのポインタ。

前提条件と不変条件

  • p は有効な apr_pool_t である必要があります。
  • json は ACME Order を表現する有効な JSON データである必要があります。
  • 内部で md_acme_order_create を呼び出すため、返されるオブジェクトは指定されたプール p の生存期間に依存します。

アルゴリズムまたは振る舞い

1. md_acme_order_create(p) を呼び出し、新しい md_acme_order_t オブジェクトを初期化します。 2. order_update_from_json(order, json, p) を呼び出し、提供された JSON の内容をオブジェクトに反映させます。 3. 構築された order オブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_order.c:191
  • **呼び出し先**:
  • md_acme_order_create
  • order_update_from_json

関連データ構造

  • md_acme_order_t: ACME Order の状態を保持するメイン構造体。
  • md_json_t: JSON データを扱うための構造体。

失敗モードと境界条件

  • json が不正な形式である場合、内部の order_update_from_json の挙動に依存します。
  • メモリ不足時には apr_pool_t の仕様に従い、アロケーションエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は md_acme_order_t のデシリアライズにおけるエントリポイントです。
  • JSON スキーマの変更や Order 構造体のフィールド追加時は、order_update_from_json の更新が必要となる点に注意してください。
