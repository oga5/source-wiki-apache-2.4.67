note_basic_auth_failure

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_basic.c:230

目的

HTTP Basic 認証の失敗時に、クライアントに対して認証を再試行させるための WWW-Authenticate ヘッダー（またはプロキシ経由の場合は Proxy-Authenticate ヘッダー）をレスポンスに追加します。

シグネチャまたは形状

static void note_basic_auth_failure(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: なし（副作用として r->err_headers_out テーブルを更新）

前提条件と不変条件

  • r->pool が有効であること。
  • ap_auth_name(r) が現在のディレクトリ設定に基づいた認証レルム名を正しく返すこと。

アルゴリズムまたは振る舞い

1. リクエストがプロキシ経由（r->proxyreq == PROXYREQ_PROXY）であるかを確認します。 2. プロキシ経由であれば Proxy-Authenticate ヘッダーを、そうでなければ WWW-Authenticate ヘッダーを対象とします。 3. Basic realm="<認証名>" という形式の文字列を生成し、r->err_headers_out テーブルに設定します。 4. 文字列の連結には apr_pstrcat を使用し、メモリ管理はリクエストプール（r->pool）に依存します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_basic.c 内の認証ロジック（244, 260, 268, 425行目付近）から呼び出されています。
  • **呼び出し先**:
  • apr_table_setn (APR)
  • apr_pstrcat (APR)
  • ap_auth_name (Apache Core)

関連データ構造

  • request_rec: リクエスト処理のコンテキストを保持する構造体。

失敗モードと境界条件

  • ap_auth_name(r) が空文字列を返す場合、レルム名が空の WWW-Authenticate ヘッダーが生成されます。
  • メモリ不足時には apr_pstrcat がアボートする可能性があります（APRの標準的な挙動）。

保守担当者向けメモ

  • 本関数は認証失敗時のレスポンスヘッダー生成に特化しています。認証の成否判定ロジック自体は呼び出し元で行われます。
  • プロキシ環境下での挙動がハードコードされているため、将来的に認証スキームが拡張される場合は注意が必要です。
