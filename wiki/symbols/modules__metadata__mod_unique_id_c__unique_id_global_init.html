unique_id_global_init

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_unique_id.c:146

目的

unique_id_rec 構造体のメモリレイアウト（オフセットとサイズ）を計算し、エンコード後のサイズを決定して初期化を行う。

シグネチャまたは形状

int unique_id_global_init(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *main_server)

入力と出力

  • **入力**:
  • p, plog, ptemp: APRメモリプール。
  • main_server: サーバー設定構造体。
  • **出力**:
  • 戻り値: 成功時に OK を返す。
  • 副作用: グローバル変数 unique_id_rec_offset[], unique_id_rec_size[], unique_id_rec_total_size, unique_id_rec_size_uu を更新する。

前提条件と不変条件

  • unique_id_rec 構造体の定義がメモリ上で固定されていること。
  • cur_unique_id が静的変数として存在していること。

アルゴリズムまたは振る舞い

1. APR_OFFSETOF マクロを使用して、unique_id_rec 構造体内の各フィールド（stamp, root, counter, thread_index）のオフセットを算出する。 2. 各フィールドのサイズを sizeof で取得し、配列に格納する。 3. 全フィールドの合計サイズを unique_id_rec_total_size に加算する。 4. エンコード後のサイズ unique_id_rec_size_uu を、バイナリデータからUUエンコード形式へ変換する際の計算式 (total_size * 8 + 5) / 6 に基づいて算出する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_config を介してApacheの設定読み込みフェーズで呼び出される（modules/metadata/mod_unique_id.c:322）。
  • **呼び出し先**: APRのオフセット計算マクロおよびサイズ取得関数を使用。

関連データ構造

  • unique_id_rec: ID生成の元となる構造体。
  • cur_unique_id: 構造体のインスタンス。

失敗モードと境界条件

  • 戻り値として OK 以外を返すロジックは本関数内には存在しない。
  • 構造体のパディングやアライメントがプラットフォーム間で異なる場合、計算されるオフセットに影響を与える可能性がある。

保守担当者向けメモ

  • 本関数はモジュールの初期化時に一度だけ実行される。
  • unique_id_rec 構造体のメンバ構成を変更した場合は、必ず本関数のオフセット計算ロジックも更新すること。
