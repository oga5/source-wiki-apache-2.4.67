ap_reclaim_child_processes

定義

  • 種別: function
  • 定義箇所: server/mpm_unix.c:184

目的

終了処理において、すべての子プロセスおよび追加の管理対象プロセスを段階的に回収（reap）します。時間経過に応じて SIGTERM から SIGKILL へとシグナルをエスカレーションし、プロセスの終了を強制します。

シグネチャまたは形状

void ap_reclaim_child_processes(int terminate, ap_reclaim_callback_fn_t *mpm_callback)

入力と出力

  • terminate (int): 1 の場合、即座に SIGTERM を送信するフェーズを開始します。0 の場合は待機から開始します。
  • mpm_callback (ap_reclaim_callback_fn_t*): プロセスが回収された際に呼び出されるコールバック関数。
  • 戻り値: なし（void）。

前提条件と不変条件

  • max_daemons は ap_mpm_query(AP_MPMQ_MAX_DAEMON_USED, ...) により取得されます。
  • スコアボード上のプロセスおよび extras リストに登録されたプロセスを対象とします。
  • not_dead_yet が 0 になるか、GIVEUP アクションに到達するまでループが継続されます。

アルゴリズムまたは振る舞い

1. **アクションテーブルの定義**: 経過時間に応じて DO_NOTHING, SEND_SIGTERM, SEND_SIGKILL, GIVEUP を実行するスケジュールを保持します。 2. **ループ処理**:

  • waittime を指数関数的に増加させ（最大1秒）、apr_sleep で待機します。
  • 現在の経過時間に基づき、実行すべきアクション（cur_action）を決定します。
  • **スコアボードの走査**: ap_get_scoreboard_process で取得した各プロセスに対し reclaim_one_pid を実行し、成功時に mpm_callback を呼び出します。
  • **追加プロセスの走査**: extras リストを辿り、同様に reclaim_one_pid を実行して回収を試みます。
  • **その他**: APR_HAS_OTHER_CHILD が定義されている場合、apr_proc_other_child_refresh_all を呼び出します。

    • 3. **終了条件**: 全プロセスが回収されるか、GIVEUP アクションに到達した時点で終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/prefork/prefork.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • reclaim_one_pid (内部関数)
  • ap_mpm_query
  • ap_get_scoreboard_process
  • ap_unregister_extra_mpm_process
  • apr_sleep, apr_time_now, apr_time_from_sec
  • apr_proc_other_child_refresh_all (条件付き)
  • mpm_callback (引数として渡された関数)

関連データ構造

  • extra_process_t: 追加の管理対象プロセスを保持するリスト構造。
  • process_score: スコアボード内のプロセス状態情報。
  • action_t: 実行されるシグナル送信等のアクション種別。

失敗モードと境界条件

  • pid == 0 のスコアボードエントリは無視されます。
  • ap_unregister_extra_mpm_process が期待通りに動作しない場合、AP_DEBUG_ASSERT により停止します。
  • GIVEUP アクションに到達すると、回収が完了していなくても関数は終了します。

保守担当者向けメモ

  • waittime の増加ロジックは、最後のプロセスが終了した際に迅速に反応できるよう、最大1秒に制限されています。
  • extras リストの走査中に ap_unregister_extra_mpm_process を呼び出すと、cur_extra が指すポインタが無効になる可能性があるため、ループ内でのポインタ管理に注意が必要です。
