passphrase_ui_open

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:620

目的

OpenSSL の UI_METHOD における「オープン」コールバックとして、暗号化された秘密鍵の復号に必要なパスフレーズを収集するための対話型インターフェースを初期化します。

シグネチャまたは形状

static int passphrase_ui_open(UI *ui)

入力と出力

  • **入力**: UI *ui (OpenSSL の UI 構造体)
  • **出力**: 成功時に 1、失敗時に 0 を返します。

前提条件と不変条件

  • UI_get0_user_data(ui) を通じて pphrase_cb_arg_t 型のコンテキストが取得可能であること。
  • ppcb->s から取得されるサーバー設定に基づき、パスフレーズの対話方式（SSL_PPTYPE_BUILTIN または SSL_PPTYPE_PIPE）が決定されていること。

アルゴリズムまたは振る舞い

1. **状態更新**: ppcb->nPassPhraseDialog および ppcb->nPassPhraseDialogCur をインクリメントします。 2. **方式別処理**:

  • **PIPE**: readtty が未初期化の場合、ssl_pipe_child_create を呼び出し、外部プログラムとの通信用パイプを生成します。
  • **BUILTIN**: Windows 環境ではエラーとして終了します。それ以外の場合は apr_file_open_stdout を使用して標準出力を writetty に割り当てます。

    • 3. **ユーザー通知**: 初回呼び出し時（nPassPhraseDialog == 1）に、mod_ssl のパスフレーズダイアログである旨のヘッダーを表示します。 4. **鍵情報表示**: bPassPhraseDialogOnce が真の場合、対象となる鍵の ID とファイルパスを表示し、フラグをリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_pphrase.c:820 において、UI_method_set_opener を介して OpenSSL の UI メソッドに登録されます。
  • **呼び出し先**:
  • ssl_pipe_child_create (パイプ方式時)
  • apr_file_printf / apr_file_open_stdout (APR ユーティリティ)
  • ap_log_error (ログ出力)

関連データ構造

  • pphrase_cb_arg_t: パスフレーズ収集の状態を保持するコンテキスト。
  • SSLSrvConfigRec: サーバーごとの SSL 設定。
  • UI: OpenSSL の対話型 UI 構造体。

失敗モードと境界条件

  • **パイプ生成失敗**: ssl_pipe_child_create が APR_SUCCESS 以外を返した場合、エラーログを出力して 0 を返します。
  • **Windows 環境**: SSL_PPTYPE_BUILTIN が指定されている場合、サポート外のためエラーとして 0 を返します。

保守担当者向けメモ

  • 本関数は OpenSSL の UI_METHOD インターフェースの一部です。
  • writetty はグローバルまたは静的なファイルハンドルとして扱われており、マルチスレッド環境での競合や初期化順序に注意が必要です。
  • パスフレーズの入力自体は、後続の passphrase_ui_read 等で行われます。
