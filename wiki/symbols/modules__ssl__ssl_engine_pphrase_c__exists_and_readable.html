exists_and_readable

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:61

目的

指定されたファイルパスが通常のファイルとして存在し、かつ読み取り可能であることを確認し、必要に応じてファイルの最終更新時刻（mtime）を取得します。

シグネチャまたは形状

static apr_status_t exists_and_readable(const char *fname, apr_pool_t *pool, apr_time_t *mtime)

入力と出力

  • **入力**:
  • fname: 確認対象のファイルパス（文字列）。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • mtime: 最終更新時刻を格納するためのポインタ（NULL指定可能）。
  • **出力**:
  • 成功時: APR_SUCCESS を返します。mtime が指定されていれば、そこにファイルの更新時刻が書き込まれます。
  • 失敗時: apr_stat または apr_file_open のエラーコード、あるいはファイルが通常ファイルでない場合に APR_EGENERAL を返します。

前提条件と不変条件

  • fname は有効なファイルパスである必要があります。
  • 呼び出し元は、戻り値が APR_SUCCESS であることを確認してから mtime を参照する必要があります。

アルゴリズムまたは振る舞い

1. apr_stat を使用してファイルのメタデータを取得します。取得に失敗した場合はエラーを返します。 2. ファイルタイプが APR_REG（通常ファイル）であることを確認します。それ以外の場合は APR_EGENERAL を返します。 3. apr_file_open を用いて読み取りモードでファイルを開きます。開けない場合はエラーを返します。 4. mtime が NULL でない場合、apr_stat で取得した sbuf.mtime を代入します。 5. apr_file_close でファイルを閉じ、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_pphrase.c:162
  • **呼び出し先**:
  • apr_stat (APR)
  • apr_file_open (APR)
  • apr_file_close (APR)

関連データ構造

  • apr_finfo_t: ファイル情報を保持する APR 構造体。
  • apr_file_t: ファイルハンドルを保持する APR 構造体。

失敗モードと境界条件

  • 指定されたパスが存在しない、またはアクセス権限がない場合は apr_stat または apr_file_open がエラーを返します。
  • ディレクトリやデバイスファイルなど、通常ファイル（APR_REG）以外が指定された場合は APR_EGENERAL が返されます。

保守担当者向けメモ

  • 本関数は、秘密鍵のキャッシュ判定ロジックの一部として使用されています。
  • apr_pool_t を使用しているため、呼び出し元で適切なプール管理が行われていることを前提としています。
