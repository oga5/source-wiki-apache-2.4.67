ssl_proxy_enable

定義

  • 種別: function
  • 定義箇所: server/ssl.c:118

目的

アウトバウンド接続（プロキシ接続など）に対して、SSL/TLS 機能を有効化するためのインターフェースを提供します。

シグネチャまたは形状

static int ssl_proxy_enable(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c - SSL を適用する対象の接続構造体。
  • **出力**: int - 処理の結果を返します（ap_ssl_bind_outgoing の戻り値に依存）。

前提条件と不変条件

  • 引数 c は有効な conn_rec 構造体である必要があります。
  • 本関数は ap_ssl_bind_outgoing を呼び出すラッパーであり、SSL 関連のオプション関数が適切に初期化されていることを前提とします。

アルゴリズムまたは振る舞い

1. ap_ssl_bind_outgoing 関数を呼び出します。 2. その際、第 2 引数に NULL、第 3 引数（SSL 必須フラグ）に 1 を指定して実行します。 3. ap_ssl_bind_outgoing の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: ap_ssl_bind_outgoing
  • **呼び出し元**:
  • modules/proxy/mod_proxy.c および modules/arch/netware/mod_nw_ssl.c 等から、オプション関数として登録・取得され利用されることが示唆されています。

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。

失敗モードと境界条件

  • ap_ssl_bind_outgoing が内部で失敗した場合、そのエラーコードが返されます。
  • SSL モジュールがロードされていない、またはオプション関数が未登録の場合、実行時にエラーとなる可能性があります。

保守担当者向けメモ

  • 本関数は APR_DECLARE_OPTIONAL_FN を介して公開されるオプション関数です。変更を加える際は、これを利用している mod_proxy 等の外部モジュールへの影響を考慮してください。
  • server/ssl.c 内で module_ssl_proxy_enable としてキャッシュ・管理されています。
