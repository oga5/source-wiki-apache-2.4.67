cmd_parms

定義

  • 種別: typedef
  • 定義箇所: server/core.c:2846（http_config.h:74 に構造体定義あり）

目的

Apache HTTP Server の設定ファイル解析中に、ディレクティブの引数やコンテキスト情報を保持・伝達するための構造体です。

シグネチャまたは形状

構造体の詳細は http_config.h に定義されています。設定ディレクティブを処理する関数（command_rec のハンドラ）において、第一引数として受け渡されます。

入力と出力

  • **入力**: 設定ファイルから読み込まれたディレクティブの引数、現在の設定コンテキスト（プール、サーバー設定、ディレクトリ設定など）。
  • **出力**: ディレクティブの解析結果を反映した設定構造体へのポインタ、またはエラー発生時のエラーメッセージ文字列。

前提条件と不変条件

  • 設定解析フェーズにおいて、apr_pool_t を介したメモリ管理が行われます。
  • ディレクティブハンドラに渡される際、有効な設定コンテキストを指している必要があります。

アルゴリズムまたは振る舞い

設定ファイル解析時、cmd_parms は現在の解析位置や設定の継承状態を追跡します。ディレクティブハンドラは、この構造体を通じて現在の設定状態を読み取り、必要に応じて新しい設定値を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: 設定ファイル解析エンジン（server/config.c 等）。
  • **呼び出し先**: 各モジュールの設定ハンドラ（例: ap_set_listenbacklog, ap_set_listener など include/ap_listen.h で定義された関数群）。また、ap_expr_parse_cmd_mi 等の式評価関数にも渡されます。

関連データ構造

  • command_rec: ディレクティブの定義テーブル。
  • apr_pool_t: 設定情報のライフサイクルを管理するメモリプール。
  • core_dir_config / core_server_config: server/core.c で定義される設定構造体。

失敗モードと境界条件

  • 引数の解析に失敗した場合、ハンドラはエラーメッセージを返し、cmd_parms を通じて設定プロセスに通知されます。
  • 構造体内のポインタが NULL である場合、設定の適用時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • server/core.c の cmd_parms は http_config.h の構造体定義を参照しています。
  • 構造体のメンバ変更は、全モジュールの設定ハンドラに影響を与えるため注意が必要です。
