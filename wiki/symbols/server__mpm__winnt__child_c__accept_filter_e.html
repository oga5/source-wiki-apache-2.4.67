accept_filter_e

定義

  • 種別: enum
  • 定義箇所: server/mpm/winnt/child.c:282

目的

Windows 版 Apache HTTP Server における、接続受け入れ（Accept）処理のフィルタリング戦略を定義する列挙型です。

シグネチャまたは形状

typedef enum {
    ACCEPT_FILTER_NONE = 0,
    ACCEPT_FILTER_CONNECT = 1
} accept_filter_e;

入力と出力

  • ACCEPT_FILTER_NONE: 標準的な accept() API を使用することを示します。
  • ACCEPT_FILTER_CONNECT: Windows 固有の高性能な AcceptEx API を使用することを示します。

前提条件と不変条件

  • ACCEPT_FILTER_NONE は値 0、ACCEPT_FILTER_CONNECT は値 1 を保持します。

アルゴリズムまたは振る舞い

本列挙型は、get_accept_filter 関数によってプロトコル文字列からフィルタ種別を決定するために使用されます。また、accept_filter_to_string 関数により、デバッグやログ出力用に文字列へ変換可能です。

呼び出し元と呼び出し先

  • 呼び出し元:
  • get_accept_filter (戻り値として使用)
  • winnt_accept (ローカル変数 accf として使用)
  • accept_filter_to_string (引数として使用)
  • 呼び出し先:
  • 本シンボルは列挙型定義であり、関数呼び出しは行いません。

関連データ構造

  • winnt_conn_ctx_t: 接続コンテキスト管理において、選択されたフィルタに基づいた接続処理が行われます。

失敗モードと境界条件

  • 定義外の値をキャストして代入した場合の挙動は未定義です。

保守担当者向けメモ

  • AcceptFilter 設定により none と connect が切り替えられます。AcceptEx を使用する ACCEPT_FILTER_CONNECT は、Windows 環境における接続処理の最適化に寄与します。
