lua_authz_provider_func

定義

  • 種別: struct
  • 定義箇所: modules/lua/mod_lua.c:68

目的

Apache HTTP Server の認証プロバイダとして登録された Lua 関数と、その実行に必要な引数を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    lua_authz_provider_spec *spec;
    apr_array_header_t *args;
} lua_authz_provider_func;

入力と出力

  • spec: 認証プロバイダのメタデータ（lua_authz_provider_spec 型）へのポインタ。
  • args: 認証関数に渡される引数のリスト（apr_array_header_t 型）。

前提条件と不変条件

  • apr_pcalloc 等でメモリ確保された後に使用されることが想定されています。
  • spec は有効な認証プロバイダ定義を指している必要があります。

アルゴリズムまたは振る舞い

本構造体は、設定ファイルで指定された認証プロバイダの情報を保持し、実行時に spec を参照して Lua 関数を特定し、args を引数として渡すためのコンテナとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/lua/mod_lua.c:1726: 設定パース時に apr_pcalloc を用いてインスタンス化されます。
  • modules/lua/mod_lua.c:1758: 認証処理の過程で parsed_require_line から参照されます。
  • 呼び出し先:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • lua_authz_provider_spec: 認証プロバイダの定義詳細を保持する構造体。
  • apr_array_header_t: APR（Apache Portable Runtime）の配列構造体。

失敗モードと境界条件

  • spec が NULL の状態で参照された場合、セグメンテーション違反を引き起こす可能性があります。
  • args が初期化されていない場合、認証関数の実行時に引数処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_lua における認証プロバイダの登録と実行の橋渡しを行います。
  • 認証プロバイダの追加や引数の受け渡し仕様を変更する際は、lua_authz_provider_spec との整合性に注意してください。
