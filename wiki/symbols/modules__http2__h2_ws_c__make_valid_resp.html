make_valid_resp

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_ws.c:156

目的

WebSocket ハンドシェイク成功時に、HTTP/2 ストリーム上でクライアントへ返すための有効なレスポンスバケットを生成します。

シグネチャまたは形状

apr_bucket *make_valid_resp(conn_rec *c2, int status, apr_table_t *headers, apr_table_t *notes);

入力と出力

  • **入力**:
  • c2: 子接続の接続レコード。
  • status: レスポンスステータスコード（例: HTTP_OK）。
  • headers: 元のレスポンスヘッダーテーブル。
  • notes: 元のレスポンスノートテーブル。
  • **出力**:
  • 成功時: 生成された apr_bucket（レスポンスヘッダーを含む）。

前提条件と不変条件

  • headers 引数は NULL であってはなりません（ap_assert による検証）。
  • 生成されるヘッダーおよびノートは、c2->pool を使用してメモリ管理されます。

アルゴリズムまたは振る舞い

1. headers をクローンし、WebSocket 関連のヘッダー（Connection, Upgrade, Sec-WebSocket-Accept）を削除します。 2. notes が存在する場合はクローンし、存在しない場合は新規作成します。 3. AP_HAS_RESPONSE_BUCKETS マクロの定義状況に応じて、以下のいずれかの方法でバケットを生成します。

  • 定義されている場合: ap_bucket_response_create を使用。
  • 定義されていない場合: h2_bucket_headers_create を使用して h2_headers をラップ。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_ws.c:226（ws_handle_resp 関数内から呼び出される）。
  • **呼び出し先**:
  • apr_table_clone, apr_table_unset, apr_table_make (APR)
  • ap_bucket_response_create または h2_bucket_headers_create / h2_headers_create (HTTPD/H2)

関連データ構造

  • conn_rec: Apache HTTPD の接続コンテキスト。
  • apr_table_t: ヘッダーおよびノートの管理に使用される連想配列。
  • apr_bucket: Apache HTTPD のバケット抽象化レイヤー。

失敗モードと境界条件

  • headers が NULL の場合、ap_assert によりプログラムが異常終了します。
  • メモリ不足時には APR のメモリ割り当て関数がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は 101 Switching Protocols を 200 OK に変換する過程で、WebSocket 特有のヘッダーを排除するために使用されます。
  • AP_HAS_RESPONSE_BUCKETS の有無による条件分岐は、ビルド環境の Apache HTTPD バージョン差異を吸収するためのものです。
