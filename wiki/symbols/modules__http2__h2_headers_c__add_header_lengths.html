add_header_lengths

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_headers.c:132

目的

apr_table_t 内の全ヘッダー名と値の長さを合計し、指定されたメモリ領域に加算するコールバック関数です。

シグネチャまたは形状

static int add_header_lengths(void *ctx, const char *name, const char *value)

入力と出力

  • **入力**:
  • ctx: 合計値を保持する apr_size_t 型変数へのポインタ。
  • name: ヘッダー名（文字列）。
  • value: ヘッダー値（文字列）。
  • **出力**:
  • 常に 1 を返します（apr_table_do による走査を継続させるため）。
  • ctx が指すメモリ領域に strlen(name) + strlen(value) が加算されます。

前提条件と不変条件

  • ctx は apr_size_t 型の有効なメモリ領域を指している必要があります。
  • name および value は NULL でない文字列である必要があります。

アルゴリズムまたは振る舞い

1. ctx を apr_size_t * にキャストします。 2. name の長さと value の長さを計算し、ctx が指す値に加算します。 3. 1 を返し、テーブルの次の要素へ処理を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_headers.c:142
  • modules/http2/h2_util.c:1940
  • modules/http2/h2_util.c:1947
  • すべて apr_table_do の引数として呼び出されています。
  • **呼び出し先**:
  • strlen (標準ライブラリ)

関連データ構造

  • apr_table_t: Apache Portable Runtime のテーブル構造体。本関数はこのテーブルを走査するために使用されます。

失敗モードと境界条件

  • name または value が NULL の場合、strlen の呼び出しによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は apr_table_do のコールバックとして設計されています。
  • ヘッダーの合計バイト数を計算する際、ヘッダー名と値の間に挿入される区切り文字（: ）や行末文字（\r\n）は本関数では考慮されません。純粋に文字列の長さのみを合計します。
