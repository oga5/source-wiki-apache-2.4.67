ssl_hook_Fixup

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:1350

目的

CGI や SSI などの後続処理で使用される環境変数（subprocess_env）に対し、SSL/TLS 接続に関連するメタデータを注入します。

シグネチャまたは形状

int ssl_hook_Fixup(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）
  • **出力**: int（常に DECLINED を返して後続のフックへ処理を継続する）

前提条件と不変条件

  • modssl_request_is_tls(r, &sslconn) が真であること。TLS 接続でない場合、本関数は即座に DECLINED を返します。
  • r->subprocess_env が有効であること。

アルゴリズムまたは振る舞い

1. **HTTPS フラグ設定**: HTTPS 環境変数を "on" に設定します。 2. **SNI 情報の注入**: HAVE_TLSEXT が定義されている場合、SNI で指定されたサーバー名を SSL_TLS_SNI に設定します。 3. **標準 SSL 変数の抽出**: ディレクトリ設定 SSL_OPT_STDENVVARS が有効な場合、DNS 情報や SAN エントリ、および ssl_hook_Fixup_vars に定義された変数群を環境変数へ追加します。 4. **証明書データの抽出**: ディレクトリ設定 SSL_OPT_EXPORTCERTDATA が有効な場合、サーバー証明書、クライアント証明書、およびクライアント証明書チェーンを環境変数へ追加します。 5. **再ネゴシエーション情報の記録**: SSL_get_secure_renegotiation_support が利用可能な場合、セキュア再ネゴシエーションのサポート状況を r->notes に記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c において ap_hook_fixups に登録されており、Apache の Fixup フェーズで呼び出されます。
  • **呼び出し先**: ssl_var_lookup（変数取得）、modssl_var_extract_dns、modssl_var_extract_san_entries、OpenSSL API（SSL_get_servername, SSL_get_peer_cert_chain 等）。

関連データ構造

  • SSLDirConfigRec: ディレクトリ単位の設定（SSL_OPT_* フラグを含む）。
  • SSLConnRec: SSL 接続状態を保持する構造体。
  • request_rec: Apache のリクエスト構造体。

失敗モードと境界条件

  • TLS 接続でない場合は何も行わず DECLINED を返します。
  • 各種変数取得関数が空の値を返した場合、環境変数へのセットはスキップされます。
  • 証明書チェーンが存在しない場合、チェーン情報の抽出は行われません。

保守担当者向けメモ

  • 本関数は環境変数を設定するのみであり、リクエストの成否を決定するものではありません。
  • SSL_OPT_EXPORTCERTDATA を有効にすると環境変数が肥大化するため、パフォーマンスへの影響を考慮する必要があります。
  • ssl_hook_Fixup_vars 配列の定義内容が、注入される変数のリストを決定します。
