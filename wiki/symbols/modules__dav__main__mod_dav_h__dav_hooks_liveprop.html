dav_hooks_liveprop

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:271

目的

WebDAV リソースにおける「ライブプロパティ（Live Property）」の取得、検証、および管理を行うための関数ポインタテーブルを定義するインターフェース構造体です。

シグネチャまたは形状

ソースコード上では typedef struct dav_hooks_liveprop dav_hooks_liveprop; として不完全型で宣言されています。具体的なメンバ構成は mod_dav.h の後続定義または実装ファイルに依存します。

入力と出力

本構造体は、WebDAV プロバイダーがライブプロパティを操作するためのフックを提供します。入力としてリソースやプロパティ名を受け取り、出力としてプロパティ値や操作結果（dav_error 等）を返す関数群を保持することが想定されています。

前提条件と不変条件

  • mod_dav のプロバイダーモデルの一部として機能します。
  • プロバイダーがライブプロパティをサポートする場合、この構造体を実装し、mod_dav に登録する必要があります。

アルゴリズムまたは振る舞い

WebDAV の PROPFIND や PROPPATCH メソッド実行時に、リポジトリ固有のライブプロパティを解決するために使用されます。プロバイダーは、この構造体を通じて特定のプロパティがライブであるかどうかの判定や、値の読み書きを制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_dav.c および liveprop.c 内のプロパティ処理ロジックが、登録されたプロバイダーのフックを呼び出します。
  • **呼び出し先**: modules/dav/fs/repos.c において dav_hooks_liveprop_fs として実装が提供されています。

関連データ構造

  • dav_provider: プロバイダーが提供する各種フックを束ねる構造体。
  • dav_prop_ctx: プロパティ変更の 3 フェーズ処理用コンテキスト。

失敗モードと境界条件

  • プロバイダーがライブプロパティをサポートしない場合、対応するフックは NULL に設定されます。
  • プロパティの取得や更新に失敗した場合、dav_error 構造体を介してエラー情報が伝播されます。

保守担当者向けメモ

  • 本構造体のメンバ定義は mod_dav.h 内の他の箇所に存在します。詳細な関数ポインタのシグネチャを確認する場合は、同ファイルの定義箇所を参照してください。
  • modules/dav/fs/repos.c の実装が主要な参照例となります。
