cache_header_cmp

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:777

目的

2つの apr_table_t（HTTPヘッダーテーブル）間で、指定されたキーに対応する値が異なるかどうかを判定する。

シグネチャまたは形状

static int cache_header_cmp(apr_pool_t *pool, apr_table_t *left,
                            apr_table_t *right, const char *key)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • left, right: 比較対象の apr_table_t。
  • key: 比較するヘッダー名。
  • **出力**:
  • 1: 両方のテーブルにキーが存在し、かつその値が異なる場合。
  • 0: それ以外（いずれかのテーブルにキーが存在しない、または両方の値が一致する場合）。

前提条件と不変条件

  • cache_table_getm が提供する値の取得機能に依存している。
  • pool は有効なメモリプールである必要がある。

アルゴリズムまたは振る舞い

1. cache_table_getm を使用して、left および right テーブルから指定された key に対応する値を取得する。 2. 両方のテーブルから値が取得できた場合、strcmp を用いて文字列比較を行う。 3. 文字列が異なる場合は 1 を返し、一致する場合または取得できなかった場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c:1155
  • modules/cache/mod_cache.c:1159
  • **呼び出し先**:
  • cache_table_getm（cache_util.h で定義されていると推測されるユーティリティ関数）
  • strcmp（標準Cライブラリ）

関連データ構造

  • apr_table_t: Apache Portable Runtime (APR) のキー・バリューペアを保持するテーブル構造体。

失敗モードと境界条件

  • 指定された key がいずれかのテーブルに存在しない場合、関数は 0 を返す。
  • cache_table_getm が NULL を返す場合、比較は行われず 0 が返される。

保守担当者向けメモ

  • 本関数はヘッダーの不一致を検出するためのユーティリティであり、キャッシュの検証ロジックの一部として使用されている。
  • cache_table_getm の挙動が本関数の結果を左右するため、変更時は cache_util.h の定義を確認すること。
