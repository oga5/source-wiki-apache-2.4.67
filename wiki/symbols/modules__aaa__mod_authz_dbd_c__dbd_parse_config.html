dbd_parse_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbd.c:346

目的

Apache の Require ディレクティブで指定された文字列を解析し、実行時に評価可能な式（ap_expr_info_t）へ変換する。

シグネチャまたは形状

static const char *dbd_parse_config(cmd_parms *cmd, const char *require_line,
                                    const void **parsed_require_line)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのコンテキスト情報。
  • require_line: 解析対象となる Require 行の文字列。
  • **出力**:
  • parsed_require_line: 解析結果の ap_expr_info_t 構造体へのポインタが格納される。
  • 戻り値: 解析成功時は NULL。解析失敗時はエラーメッセージを含む文字列（cmd->temp_pool から割り当て）。

前提条件と不変条件

  • require_line は ap_expr_parse_cmd が許容する形式である必要がある。
  • 解析結果は cmd->temp_pool を使用してメモリ確保される。

アルゴリズムまたは振る舞い

1. ap_expr_parse_cmd を呼び出し、require_line を文字列結果を返す式として解析する。 2. 解析中にエラーが発生した場合、エラーメッセージを構築して返却する。 3. 解析が成功した場合、生成された ap_expr_info_t 構造体を *parsed_require_line に代入し、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbd.c:369 において、authz_dbd_hooks 内のプロバイダー登録時に参照されている。
  • **呼び出し先**: ap_expr_parse_cmd（Apache コアの式評価エンジン）。

関連データ構造

  • cmd_parms: 設定処理のパラメータ構造体。
  • ap_expr_info_t: 解析された式を保持する構造体。

失敗モードと境界条件

  • ap_expr_parse_cmd がエラーを返した場合、関数は即座にエラーメッセージを返し、処理を中断する。

保守担当者向けメモ

  • 本関数は mod_authz_dbd における Require 行の構文解析を担当する。式評価エンジン（ap_expr）に依存しているため、式構文の変更は ap_expr の仕様に従う。
