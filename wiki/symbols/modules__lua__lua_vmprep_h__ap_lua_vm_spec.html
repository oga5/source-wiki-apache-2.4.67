ap_lua_vm_spec

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_vmprep.h:58

目的

Apache HTTP Server 上で実行される Lua 仮想マシン（VM）の生成パラメータ、生存期間（スコープ）、およびキャッシュ戦略を定義するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *package_paths;
    apr_array_header_t *package_cpaths;
    const char *file;
    int scope;
    unsigned int vm_min;
    unsigned int vm_max;
    ap_lua_state_open_callback cb;
    void* cb_arg;
    apr_pool_t *pool;
    const char *bytecode;
    apr_size_t bytecode_len;
    int codecache;
} ap_lua_vm_spec;

入力と出力

  • **入力**: mod_lua の設定読み込み時やリクエスト処理時に、スクリプトパス、スコープ、キャッシュ設定などが格納されます。
  • **出力**: ap_lua_get_lua_state 等の関数へ渡され、VM の初期化や取得の仕様として利用されます。

前提条件と不変条件

  • bytecode_len > 0 の場合、file フィールドはロード目的では無視され、エラーメッセージ出力のみに使用されます。
  • scope が APL_SCOPE_ONCE に設定されている場合、pool フィールドがライフサイクル管理に使用されます。

アルゴリズムまたは振る舞い

  • VM の生成および管理の仕様を保持します。
  • bytecode が提供されている場合、ファイルシステムから読み込む代わりにメモリ上のバイトコードから直接 VM をロードします。
  • cb（コールバック）と cb_arg を通じて、VM 生成時の追加環境構築をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_vmprep.c（copy_vm_spec 等）および modules/lua/mod_lua.c（create_vm_spec 等）にて、設定のコピーや VM 生成のパラメータとして参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • apr_array_header_t: パッケージパスの保持に使用。
  • apr_pool_t: VM のライフサイクル管理用メモリプール。
  • ap_lua_state_open_callback: VM 初期化時に実行されるコールバック関数型。

失敗モードと境界条件

  • bytecode_len が 0 の場合、file フィールドがロード対象となります。
  • scope の指定が不適切な場合、VM の生存期間が意図した範囲（リクエスト、接続、サーバー等）を超過または早期終了する可能性があります。

保守担当者向けメモ

  • package_paths および package_cpaths は、現在ディレクトリ設定の一部として扱われていますが、将来的には仕様の一部として統合されることがコメントで示唆されています。
