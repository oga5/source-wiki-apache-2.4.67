md_get_by_name

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:153

目的

管理対象ドメインのリスト（apr_array_header_t）から、指定された名称（name）に一致する md_t オブジェクトを検索して返します。

シグネチャまたは形状

md_t *md_get_by_name(struct apr_array_header_t *mds, const char *name)

入力と出力

  • **入力**:
  • mds: 検索対象となる md_t ポインタを格納した apr_array_header_t 構造体。
  • name: 検索するドメイン名（文字列）。
  • **出力**:
  • 一致する md_t 構造体へのポインタ。
  • 一致するものがない場合は NULL。

前提条件と不変条件

  • mds は有効な apr_array_header_t 構造体である必要があります。
  • md_t 構造体の name フィールドは、比較可能な文字列として初期化されている必要があります。

アルゴリズムまたは振る舞い

1. mds 配列内の要素をインデックス 0 から nelts - 1 まで順次走査します。 2. 各要素を md_t * 型として取得します。 3. strcmp を使用して、指定された name と md->name を比較します。 4. 一致した時点でその md_t ポインタを即座に返します。 5. ループ終了まで一致が見つからない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_reg.c:794
  • modules/md/mod_md_drive.c:88
  • modules/md/mod_md_status.c:961
  • **呼び出し先**:
  • strcmp (標準ライブラリ)
  • APR_ARRAY_IDX (APR マクロ)

関連データ構造

  • md_t: 管理対象ドメインの情報を保持する構造体（modules/md/md.h で定義）。
  • apr_array_header_t: APR が提供する動的配列構造体。

失敗モードと境界条件

  • mds が空（nelts == 0）の場合、ループは実行されず NULL を返します。
  • name が NULL の場合、strcmp の挙動により未定義動作（セグメンテーション違反等）が発生する可能性があります。

保守担当者向けメモ

  • 本関数は線形探索を行うため、mds 配列の要素数が増加した場合にはパフォーマンスに影響を与える可能性があります。
  • md_t の名称比較は完全一致を前提としています。大文字小文字の区別や正規化が必要な場合は、呼び出し側で事前処理を行う必要があります。
