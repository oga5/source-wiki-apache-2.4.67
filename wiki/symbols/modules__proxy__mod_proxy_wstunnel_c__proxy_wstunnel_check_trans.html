proxy_wstunnel_check_trans

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_wstunnel.c:29

目的

WebSocket プロトコル（ws: / wss:）のリクエストをインターセプトし、Upgrade ヘッダーの有無に基づいて、本モジュールで処理するか、他のハンドラ（mod_proxy_http 等）へ委譲するかを判定する。

シグネチャまたは形状

static int proxy_wstunnel_check_trans(request_rec *r, const char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • url: 処理対象の URL 文字列。
  • **出力**:
  • OK: WebSocket リクエストとして処理を続行する場合。
  • DECLINED: 本モジュールでの処理を辞退し、他のハンドラに委譲する場合。
  • HTTP_UPGRADE_REQUIRED: Upgrade ヘッダーが存在せず、WebSocket トンネリングが不可能な場合。

前提条件と不変条件

  • r->per_dir_config から proxyws_dir_conf を取得し、設定されたフォールバック条件を確認する。
  • URL が ws: または wss: で始まらない場合、即座に DECLINED を返す。

アルゴリズムまたは振る舞い

1. **フォールバック判定**: 設定により mod_proxy_http へのフォールバックが有効かつ条件を満たす場合、DECLINED を返す。 2. **スキーム検証**: URL が ws: または wss: で開始されない場合、DECLINED を返す。 3. **ヘッダー検証**: r->headers_in 内に Upgrade ヘッダーが存在するか確認する。

  • 存在しない場合: HTTP_UPGRADE_REQUIRED を返し、処理を中断する。
  • 存在する場合: OK を返し、後続のプロキシ処理へ進む。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の check_trans フック（modules/proxy/mod_proxy_wstunnel.c:501 にて proxy_hook_check_trans に登録）。
  • **呼び出し先**:
  • ap_get_module_config: 設定構造体の取得。
  • ap_log_rerror: ログ出力。
  • apr_table_get: ヘッダーの確認。
  • ap_cstr_casecmpn: URL スキームの比較。

関連データ構造

  • proxyws_dir_conf: ディレクトリ単位の設定（フォールバック設定を保持）。
  • request_rec: リクエスト情報およびヘッダーテーブル。

失敗モードと境界条件

  • Upgrade ヘッダーが欠落している場合、HTTP_UPGRADE_REQUIRED を返して処理を終了する。
  • ws: / wss: 以外のスキームが渡された場合、DECLINED を返して他のモジュールに処理を委ねる。

保守担当者向けメモ

  • 本関数は proxy_hook_check_trans フックの一部として実行される。
  • HTTP_UPGRADE_REQUIRED を返すことで、WebSocket 以外のリクエストが誤ってトンネル処理されることを防いでいる。
  • フォールバックロジックは dconf->fallback_to_proxy_http に依存しているため、設定変更時にはこのフラグの挙動に注意すること。
