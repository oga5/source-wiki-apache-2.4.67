console_control_handler

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/service.c:164

目的

コンソールモードで実行されているApacheプロセスに対し、Windowsのコンソール制御イベント（Ctrl+Cやシャットダウン要求など）を捕捉し、適切なサーバー制御シグナル（再起動またはシャットダウン）へ変換して通知する。

シグネチャまたは形状

static BOOL CALLBACK console_control_handler(DWORD ctrl_type)

入力と出力

  • **入力**: ctrl_type (DWORD) - 発生したコンソールイベントの種類（CTRL_C_EVENT, CTRL_BREAK_EVENT, CTRL_CLOSE_EVENT 等）。
  • **出力**: BOOL - イベントを処理した場合は TRUE、それ以外は FALSE を返す。

前提条件と不変条件

  • Windows環境のコンソールアプリケーションとして実行されていること。
  • SetConsoleCtrlHandler を介して本関数が登録されていること。

アルゴリズムまたは振る舞い

受け取った ctrl_type に応じて以下の処理を行う： 1. **CTRL_BREAK_EVENT**: 再起動シグナル (SIGNAL_PARENT_RESTART) を送信し、TRUE を返す。 2. **CTRL_C_EVENT**: シャットダウンシグナル (SIGNAL_PARENT_SHUTDOWN) を送信し、TRUE を返す。 3. **CTRL_CLOSE_EVENT / CTRL_LOGOFF_EVENT / CTRL_SHUTDOWN_EVENT**: シャットダウンシグナル (SIGNAL_PARENT_SHUTDOWN) を送信し、Sleep(30000) で待機した後、TRUE を返す。 4. **その他**: FALSE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Windows OSのコンソール制御ディスパッチャ（SetConsoleCtrlHandler により登録）。
  • **呼び出し先**:
  • ap_signal_parent: 親プロセスへシグナルを送信する。
  • fprintf: 標準エラー出力へログを出力する。
  • Sleep: シャットダウン処理の猶予時間を確保する。

関連データ構造

  • DWORD: Windows APIの型定義。

失敗モードと境界条件

  • CTRL_CLOSE_EVENT 等の終了イベント発生時には、30秒間の待機（Sleep）が発生するため、即時のプロセス終了は行われない。
  • 予期しないイベントタイプが渡された場合は FALSE を返し、OS側のデフォルト動作に委ねる。

保守担当者向けメモ

  • 本関数は SetConsoleCtrlHandler のコールバックとして動作するため、スレッドセーフな実装が求められる。
  • シャットダウン時の Sleep(30000) は、システムがプロセスを強制終了する前にApacheがクリーンアップを試みるための猶予期間である。
