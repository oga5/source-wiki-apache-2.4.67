pre_init

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_digest.c:369

目的

Digest 認証モジュールの初期化処理として、プロセス間同期用のミューテックス登録と、nonce 生成等に使用する共有シークレットの生成・保持を行う。

シグネチャまたは形状

static int pre_init(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp)

入力と出力

  • 入力:
  • pconf: 設定用メモリプール。
  • plog: ログ用メモリプール。
  • ptemp: 一時作業用メモリプール。
  • 出力:
  • 成功時は OK (0)、失敗時は !OK を返す。

前提条件と不変条件

  • ap_mutex_register によるミューテックス登録が成功すること。
  • APR_HAS_RANDOM が定義されており、乱数生成がサポートされていること。
  • 共有シークレットは ap_retained_data_create を通じてプロセス間で保持される。

アルゴリズムまたは振る舞い

1. client_mutex_type および opaque_mutex_type を ap_mutex_register で登録する。 2. ap_retained_data_get を呼び出し、既存のシークレットデータが存在するか確認する。 3. 存在しない場合、ap_retained_data_create で領域を確保し、apr_generate_random_bytes を用いて SECRET_LEN 分の乱数を生成・格納する。 4. 生成された（または取得した）シークレットをグローバル変数 secret に代入する。

呼び出し元と呼び出し先

  • 呼び出し元: ap_hook_pre_config により、Apache の設定読み込みフェーズで呼び出される（modules/aaa/mod_auth_digest.c:1950）。
  • 呼び出し先: ap_mutex_register, ap_retained_data_get, ap_retained_data_create, ap_log_error, apr_generate_random_bytes。

関連データ構造

  • secret: モジュール内で使用される共有シークレットを保持するグローバル変数。
  • RETAINED_DATA_ID: プロセス間でデータを保持するための識別子。

失敗モードと境界条件

  • ミューテックス登録失敗時: !OK を返し、初期化を中断する。
  • 乱数生成失敗時: ap_log_error でエラーを記録し、!OK を返す。
  • APR_HAS_RANDOM が未定義の場合: コンパイルエラーとなる。

保守担当者向けメモ

  • 本関数は Apache の pre_config フックで実行されるため、マルチプロセス環境での初期化順序に依存する処理が含まれています。
  • secret の生成には APR の乱数生成機能が必須であり、環境依存の制約がある点に注意してください。
