ap_cookie_do

定義

  • 種別: struct
  • 定義箇所: include/util_cookies.h:48

目的

HTTP Cookie 処理の内部状態を保持し、リクエスト処理中に Cookie 情報を抽象化して管理するための構造体です。

シグネチャまたは形状

typedef struct {
    request_rec *r;
    const char *name;
    const char *encoded;
    apr_table_t *new_cookies;
    int duplicated;
} ap_cookie_do;

入力と出力

本構造体は関数引数として渡される内部状態のコンテナとして機能します。

  • r: 処理対象の request_rec 構造体へのポインタ。
  • name: 処理対象の Cookie 名。
  • encoded: エンコードされた Cookie 値。
  • new_cookies: 新規 Cookie を格納する apr_table_t へのポインタ。
  • duplicated: 重複発生時に使用されるフラグ（整数）。

前提条件と不変条件

  • request_rec 構造体は有効なリクエストコンテキストを指している必要があります。
  • apr_table_t は APR のメモリプール管理下にあるテーブルである必要があります。

アルゴリズムまたは振る舞い

Cookie 処理の内部状態を保持し、server/util_cookies.c 内の処理フローにおいて、Cookie の読み取りや書き込み時のコンテキストとして利用されます。特に、複数のテーブルを操作する際の引数として受け渡される役割を担います。

呼び出し元と呼び出し先

  • 呼び出し元: server/util_cookies.c:249 にてインスタンス化が確認されています。
  • 呼び出し先: server/util_cookies.c:179 にて、varg から ap_cookie_do 型へのキャストが行われています。

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する基本構造体。
  • apr_table_t: APR が提供するキー・値ペアのテーブル構造。

失敗モードと境界条件

  • duplicated フラグが設定される条件については、ソースコード内のロジック（server/util_cookies.c）に依存します。
  • 構造体内の各ポインタが NULL の状態でアクセスされた場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 変更を加える際は、server/util_cookies.c 内の ap_cookie_do を参照している箇所との整合性に注意してください。
