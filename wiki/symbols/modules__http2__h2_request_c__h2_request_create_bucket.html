h2_request_create_bucket

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:290

目的

HTTP/2 リクエスト表現 (h2_request) を Apache HTTP Server のリクエスト処理パイプラインで使用可能な apr_bucket に変換します。

シグネチャまたは形状

apr_bucket *h2_request_create_bucket(const h2_request *req, request_rec *r)

入力と出力

  • **入力**:
  • req: 変換対象の h2_request 構造体（読み取り専用）。
  • r: 現在の接続に関連付けられた request_rec 構造体。
  • **出力**:
  • 成功時: Apache のリクエスト処理パイプラインに投入可能な apr_bucket へのポインタ。
  • 失敗時: apr_bucket_request_create の仕様に従い、メモリ不足等の場合は NULL またはエラー状態のバケットを返します。

前提条件と不変条件

  • req->method および req->authority が NULL でないこと（AP_DEBUG_ASSERT により検証）。
  • r->pool はバケットの生存期間を管理するために使用されます。

アルゴリズムまたは振る舞い

1. req->headers のコピーを r->pool 上に作成します。 2. req->method が CONNECT の場合、uri を req->authority に設定します。 3. フォワードプロキシ設定 (H2_CONF_PROXY_REQUESTS) が有効な場合、または scheme が http/https 以外の場合は、scheme、authority、path を結合して絶対 URI を生成します。 4. 上記以外の場合は、req->path をそのまま uri として使用します。 5. 最終的に ap_bucket_request_create を呼び出し、メソッド、URI、プロトコルバージョン ("HTTP/2.0")、ヘッダーテーブルを格納したバケットを生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2_filter.c:126
  • **呼び出し先**:
  • apr_table_clone
  • ap_cstr_casecmp
  • h2_config_cgeti
  • apr_psprintf
  • ap_bucket_request_create

関連データ構造

  • h2_request: HTTP/2 のリクエスト情報を保持する構造体。
  • request_rec: Apache のリクエスト処理における中心的な構造体。
  • apr_bucket: Apache のデータ転送用バケット。

失敗モードと境界条件

  • req->method や req->authority が欠落している場合、デバッグビルドではアサーションエラーが発生します。
  • apr_psprintf や apr_table_clone がメモリ不足に陥った場合、Apache のメモリ管理ポリシーに従いプロセスが終了するか、呼び出し元へエラーが伝播します。

保守担当者向けメモ

  • 本関数は HTTP/2 の擬似ヘッダーを Apache の標準的なリクエスト形式へ適合させる重要なブリッジです。
  • プロキシ設定やスキームの検証ロジックに変更を加える際は、h2_config との整合性に注意してください。
