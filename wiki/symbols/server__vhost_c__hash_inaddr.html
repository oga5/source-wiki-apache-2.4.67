hash_inaddr

定義

  • 種別: function
  • 定義箇所: server/vhost.c:352

目的

IP アドレス（またはポート番号）をキーとして、仮想ホスト管理用のハッシュテーブル（iphash_table）におけるインデックスを算出します。

シグネチャまたは形状

static APR_INLINE unsigned hash_inaddr(unsigned key)

入力と出力

  • **入力**: unsigned key（ハッシュ化対象の IP アドレスまたはポート番号）
  • **出力**: unsigned（IPHASH_TABLE_SIZE を法とするハッシュ値）

前提条件と不変条件

  • IPHASH_TABLE_SIZE は、本関数が剰余演算に使用する定数として定義されている必要があります。
  • APR_INLINE 修飾により、コンパイル時に呼び出し元へインライン展開されることが期待されます。

アルゴリズムまたは振る舞い

1. 入力 key に対してビットシフトと XOR 演算を行い、ビットの偏りを解消します（key ^= (key >> 16)）。 2. さらにビットシフトと XOR 演算を適用します（(key >> 8) ^ key）。 3. 最終的に IPHASH_TABLE_SIZE で剰余をとり、ハッシュテーブルのインデックスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/vhost.c:367 にて、キーからハッシュ値を計算するために呼び出されています。
  • **呼び出し先**: なし。

関連データ構造

  • ipaddr_chain: 本関数で算出されたインデックスを基にアクセスされる、IP アドレスとポートのペアを保持するハッシュテーブルのノード構造体。

失敗モードと境界条件

  • IPHASH_TABLE_SIZE が 0 の場合、剰余演算によりゼロ除算が発生する可能性があります。

保守担当者向けメモ

  • 本関数はハッシュテーブルの衝突を最小化するための単純なビット演算を行っています。ハッシュ分布の偏りが問題となる場合は、定数やシフト量の調整を検討してください。
