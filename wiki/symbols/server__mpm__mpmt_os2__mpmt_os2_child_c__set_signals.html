set_signals

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2_child.c:475

目的

子プロセスにおいて、SIGTERM および SIGHUP シグナルに対するハンドラを登録し、プロセス終了や設定再読み込みの要求を適切に捕捉する。

シグネチャまたは形状

static void set_signals(void)

入力と出力

  • 入力: なし（グローバルな sig_term および sig_hup ハンドラ関数を参照）
  • 出力: なし（副作用としてシグナルアクションの設定を変更）

前提条件と不変条件

  • sig_term および sig_hup 関数が定義されていること。
  • sigaction システムコールが利用可能であること。

アルゴリズムまたは振る舞い

1. sigaction 構造体を初期化し、マスクを空に設定する。 2. sa_handler に sig_term を設定し、sigaction(SIGTERM, ...) を呼び出す。 3. sa_handler に sig_hup を設定し、sigaction(SIGHUP, ...) を呼び出す。 4. 各 sigaction 呼び出しが失敗した場合、ap_log_error を使用して警告ログを出力する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/mpm/mpmt_os2/mpmt_os2.c:202
  • server/mpm/mpmt_os2/mpmt_os2_child.c:109
  • server/mpm/netware/mpm_netware.c:896
  • 呼び出し先:
  • sigaction (システムコール)
  • sigemptyset (システムコール)
  • ap_log_error (Apache API)

関連データ構造

  • struct sigaction: シグナルハンドラ設定用構造体。

失敗モードと境界条件

  • sigaction がエラーを返した場合、errno を含む警告ログが出力されるが、プロセスは終了せず処理を継続する。

保守担当者向けメモ

  • 本関数は mpmt_os2 モジュールおよび mpm_netware モジュールから参照されている。変更時は両プラットフォームへの影響を考慮すること。
  • ap_server_conf をログ出力に使用しているため、本関数が呼び出される時点でサーバー設定が初期化済みである必要がある。
