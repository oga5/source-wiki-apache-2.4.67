ap_get_server_built

定義

  • 種別: function
  • 定義箇所: server/buildmark.c:26

目的

サーバーのビルド日時を示す文字列を取得するための公開インターフェースを提供します。

シグネチャまたは形状

const char *ap_get_server_built(void)

入力と出力

  • **入力**: なし
  • **出力**: ビルド日時を保持する静的文字列定数へのポインタ（const char *）を返します。

前提条件と不変条件

  • コンパイル時に __DATE__ および __TIME__ マクロが評価され、静的変数 server_built に格納されている必要があります。
  • 戻り値の文字列は、サーバーのライフサイクルを通じて不変です。

アルゴリズムまたは振る舞い

1. 呼び出されると、メモリ上に静的に保持されている server_built 変数のアドレスを返します。 2. ロジックは単純な参照の返却であり、条件分岐は含みません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/main.c（起動時の情報出力）
  • modules/examples/mod_example_hooks.c
  • modules/generators/mod_info.c
  • modules/generators/mod_status.c
  • modules/lua/lua_request.c
  • modules/proxy/mod_proxy_balancer.c
  • server/mpm/event/event.c
  • server/mpm/mpmt_os2/mpmt_os2.c
  • server/mpm/netware/mpm_netware.c
  • **呼び出し先**: なし（本関数は純粋なデータアクセサです）

関連データ構造

  • server_built: server/buildmark.c 内で定義されているビルド日時を格納する静的文字列定数。

失敗モードと境界条件

  • コンパイル環境で __DATE__ や __TIME__ が利用できない場合、フォールバックとして "unknown" を示す文字列が返される実装になっています。

保守担当者向けメモ

  • 本関数は AP_DECLARE マクロで修飾されており、共有ライブラリからエクスポートされる公開 API です。
  • サーバーのバージョン情報とは異なり、純粋にコンパイル時のタイムスタンプのみを扱います。
