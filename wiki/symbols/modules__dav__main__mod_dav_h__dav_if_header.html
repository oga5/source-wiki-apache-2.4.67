dav_if_header

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:866

目的

WebDAV の If ヘッダーに含まれる条件付きリクエスト情報を保持するための構造体です。リソース URI とそれに関連付けられた状態リスト（ロックトークンや ETag）を連結リスト形式で管理します。

シグネチャまたは形状

typedef struct dav_if_header
{
    const char *uri;
    apr_size_t uri_len;
    struct dav_if_header *next;
    struct dav_if_state_list *state;
    int dummy_header;
} dav_if_header;

入力と出力

  • **入力**: dav_process_if_header 等の関数により、HTTP リクエストヘッダーから解析された情報が格納されます。
  • **出力**: next ポインタを介して、複数のリソースや条件が連鎖したリスト構造として後続の検証処理へ渡されます。

前提条件と不変条件

  • uri が NULL の場合、タグなしリスト（Tagged-list）の条件として扱われます。
  • uri が指定されている場合、そのリソースに対する条件付きリクエストとして処理されます。
  • next ポインタにより、同一リクエスト内の複数の条件セットを順次参照可能です。

アルゴリズムまたは振る舞い

  • If ヘッダーのパース結果を保持し、ロックや ETag の検証プロセスにおいて、対象リソースと状態（state）の照合に使用されます。
  • dummy_header フィールドは、ロックや ETag の検証ロジックにおいて内部的に使用されるフラグです。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/util.c: dav_process_if_header（解析処理）、dav_add_if_resource（リスト構築）
  • **呼び出し先**:
  • modules/dav/main/mod_dav.h: dav_if_state_list（state フィールド経由で参照）

関連データ構造

  • dav_if_state_list: If ヘッダー内の個別の状態（ロックトークンや ETag）を保持するリスト構造体。

失敗モードと境界条件

  • uri が重複する場合、同一リソースに対する複数の条件が state リストとして連結される構造になります。
  • メモリ割り当ては apr_pool_t に依存しており、リクエストの生存期間を超えた保持は行われません。

保守担当者向けメモ

  • 本構造体は If ヘッダーの複雑な構文（タグ付き/タグなしリスト）を表現するために設計されています。
  • uri のエンコード状態や、uri_len の整合性については、util.c でのパース処理時に厳密に管理されていることを前提としています。
