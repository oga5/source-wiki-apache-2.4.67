ap_expr_lookup_parms

定義

  • 種別: struct
  • 定義箇所: include/ap_expr.h:255

目的

式評価エンジンにおいて、変数、関数、または演算子の名前を解決（lookup）する際に、対象の識別情報と解決結果を格納するためのパラメータ構造体です。

シグネチャまたは形状

typedef struct {
    int type;
    const char *name;
    int flags;
    apr_pool_t *pool;
    apr_pool_t *ptemp;
    const void **func;
    const void **data;
    const char **err;
    const char *arg;
} ap_expr_lookup_parms;

入力と出力

  • **入力**: 解決対象の name、type（AP_EXPR_FUNC_* 定数）、およびメモリ管理用の pool。
  • **出力**: 解決された関数ポインタを func に、関連するコンテキストデータを data に格納します。エラーが発生した場合は err にメッセージを代入します。

前提条件と不変条件

  • pool および ptemp は、解決処理中に必要なメモリ割り当てのために有効な apr_pool_t である必要があります。
  • type は定義された AP_EXPR_FUNC_* のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. パーサーが式内の識別子（変数や関数名）を検出した際、この構造体に情報をセットして expr_lookup フックを呼び出します。 2. フック実装側は name を基に実体を検索し、対応する関数ポインタを func に、実行時コンテキストを data に設定します。 3. 解決に失敗した場合、err にエラー文字列を格納して非ゼロの値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_expr_eval.c 内の式解析プロセス。
  • **呼び出し先**: expr_lookup フックに登録された各モジュールの関数（例: mod_authz_core.c の expr_lookup_fn、mod_include.c の include_expr_lookup など）。

関連データ構造

  • ap_expr_lookup_fn_t: この構造体を引数に取る関数型定義。
  • AP_EXPR_FUNC_*: type フィールドで使用される定数群。

失敗モードと境界条件

  • name が未知の識別子である場合、解決失敗として扱われます。
  • func や data へのポインタ設定が適切に行われない場合、後の評価フェーズでセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は expr_lookup フックのインターフェースとして固定されています。フィールドの追加や変更を行う際は、フックを実装している全モジュール（mod_authz_core, mod_include, mod_proxy_hcheck, mod_ssl 等）への影響を確認してください。
