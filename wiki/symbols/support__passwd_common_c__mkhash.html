mkhash

定義

  • 種別: function
  • 定義箇所: support/passwd_common.c:179

目的

passwd_ctx 構造体で指定されたアルゴリズムに基づき、パスワードのハッシュ化を行い、結果をコンテキスト内のバッファに格納します。

シグネチャまたは形状

int mkhash(struct passwd_ctx *ctx);

入力と出力

  • **入力**: passwd_ctx *ctx（アルゴリズム、パスワード、出力バッファ、メモリプール等を含むコンテキスト）。
  • **出力**: 成功時は 0 を返し、失敗時はエラーコード（ERR_GENERAL, ERR_PWMISMATCH, ERR_RANDOM 等）を返します。ハッシュ化された文字列は ctx->out に書き込まれます。

前提条件と不変条件

  • ctx->passwd が NULL の場合、内部で get_password(ctx) が呼び出され、パスワードが取得されます。
  • 処理の最後には、メモリ上のパスワード（pw）を memset でゼロクリアし、機密情報を破棄します。
  • errfile グローバル変数が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. **警告**: ALG_BCRYPT, ALG_CRYPT_SHA256, ALG_CRYPT_SHA512 以外のアルゴリズムでコスト値（ctx->cost）が指定されている場合、警告を出力します。 2. **ハッシュ計算**: ctx->alg に基づき以下のいずれかを選択します。

  • ALG_APSHA: apr_sha1_base64 を使用。
  • ALG_APMD5: generate_salt でソルトを生成後、apr_md5_encode を使用。
  • ALG_PLAIN: 平文をそのままコピー。
  • ALG_CRYPT: crypt() を使用（プラットフォーム依存）。8文字を超えるパスワードの切り詰め警告機能あり。
  • ALG_CRYPT_SHA256/512: crypt() を使用し、SHA-2 形式のソルトを生成。
  • ALG_BCRYPT: apr_bcrypt_encode を使用。

    • 3. **終了処理**: memset でパスワードバッファをクリアしてリターンします。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htdbm.c:254, support/htpasswd.c:82
  • **呼び出し先**: get_password, generate_salt, apr_sha1_base64, apr_md5_encode, apr_cpystrn, crypt, apr_psprintf, apr_generate_random_bytes, apr_bcrypt_encode, memset, apr_file_printf

関連データ構造

  • struct passwd_ctx: ハッシュ化の制御パラメータと結果を保持する主要構造体。

失敗モードと境界条件

  • **アルゴリズム未対応**: default ケースに到達した場合、abort() によりプロセスが異常終了します。
  • **ハッシュ生成失敗**: crypt() や apr_md5_encode 等の関数が失敗した場合、エラーメッセージを ctx->errstr に格納し、適切なエラーコードを返します。
  • **メモリ不足**: apr_psprintf 等の APR 関数がメモリ不足を返した場合、エラーとして処理されます。

保守担当者向けメモ

  • switch 文内の各アルゴリズムは、プラットフォームのサポート状況に応じて #if で条件付きコンパイルされています。
  • crypt() を使用する際は、プラットフォームごとの挙動の違いに注意してください。
  • 機密情報（パスワード）を扱うため、エラー発生時や処理終了時には必ず memset によるゼロクリアが行われていることを確認してください。
