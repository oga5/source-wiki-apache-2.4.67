find_best_worker

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:340

目的

ロードバランサアルゴリズム（lbmethod）を使用して、リクエストを処理するのに最適なバックエンドワーカーを特定する。

シグネチャまたは形状

static proxy_worker *find_best_worker(proxy_balancer *balancer, request_rec *r)

入力と出力

  • **入力**:
  • balancer: 負荷分散対象の proxy_balancer 構造体。
  • r: 現在のリクエストを保持する request_rec 構造体。
  • **出力**:
  • 成功時: 選択された proxy_worker へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • マルチスレッド環境下では、PROXY_THREAD_LOCK による排他制御が必須である。
  • 選択されたワーカーの elected カウンタは、関数内でインクリメントされる。

アルゴリズムまたは振る舞い

1. **排他制御**: balancer に対してスレッドロックを取得する。 2. **ワーカー選択**: balancer->lbmethod->finder を呼び出し、アルゴリズムに基づきワーカーを特定する。 3. **統計更新**: ワーカーが特定された場合、その elected カウンタをインクリメントする。 4. **ロック解除**: スレッドロックを解放する。 5. **再試行ロジック**: ワーカーが取得できず、かつ balancer->s->timeout が設定されている場合、タイムアウト時間まで再帰的に find_best_worker を呼び出し、ワーカーの空きを待機する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_balancer.c 内の proxy_balancer_pre_request（直接の呼び出し）。
  • find_best_worker 自身（タイムアウト時の再帰呼び出し）。
  • **呼び出し先**:
  • balancer->lbmethod->finder: 登録された負荷分散アルゴリズムの実行。
  • apr_sleep: タイムアウト待機時のスリープ処理。

関連データ構造

  • proxy_balancer: バランサの状態と設定を保持。
  • proxy_worker: 個別のバックエンドワーカーの状態を保持。
  • request_rec: Apache のリクエスト情報。

失敗モードと境界条件

  • **ロック失敗**: スレッドロックの取得に失敗した場合、エラーログを出力して NULL を返す。
  • **ワーカー不在**: すべてのワーカーがエラー状態または無効な場合、NULL を返す。
  • **タイムアウト**: timeout が設定されている場合、ループ内で再試行を行うが、最終的にワーカーが見つからなければ NULL を返す。

保守担当者向けメモ

  • タイムアウト時の再帰呼び出しにおいて、無限ループを避けるため一時的に balancer->s->timeout を 0 に設定する処理が含まれている。
  • 共有メモリ（slotmem）を介したプロセス間同期が行われるため、ロックの整合性に注意すること。
