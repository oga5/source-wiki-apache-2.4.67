bucket_cleanup

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_eos.c:37

目的

HTTP/2 ストリームの生存期間と EOS（End of Stream）バケットの生存期間が異なる場合に、ストリームの二重解放や不正なアクセスを防ぐためのクリーンアップ処理を行います。

シグネチャまたは形状

static apr_status_t bucket_cleanup(void *data)

入力と出力

  • **入力**: void *data - h2_stream オブジェクトへのポインタを保持する h2_stream ** 型のデータ。
  • **出力**: apr_status_t - 常に APR_SUCCESS を返します。

前提条件と不変条件

  • data は h2_stream ** 型としてキャスト可能である必要があります。
  • 本関数は APR プールクリーンアップとして登録されることを前提としています。

アルゴリズムまたは振る舞い

1. 引数 data を h2_stream ** 型として受け取ります。 2. ストリームポインタが有効（非NULL）であるかを確認します。 3. ストリームポインタを NULL に設定します。これにより、後続の bucket_destroy が実行された際に、ストリームの二重解放を防止します。 4. APR_SUCCESS を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_bucket_eos.c:83 (プールへのクリーンアップ登録時)
  • modules/http2/h2_bucket_eos.c:95 (プールからのクリーンアップ解除時)
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • h2_stream: ストリームの状態管理を行う構造体。
  • apr_pool_t: APR メモリプール。本関数はこのプールのクリーンアップハンドラとして利用されます。

失敗モードと境界条件

  • data が NULL の場合、あるいは指し示す先が既に NULL の場合は、何も行わず APR_SUCCESS を返します。

保守担当者向けメモ

  • 本関数は bucket_destroy と協調して動作します。bucket_destroy が先に呼ばれた場合と、プールが先に破棄された場合の両方でストリームの安全性を担保する設計となっています。
