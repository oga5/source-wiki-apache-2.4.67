header_struct

定義

  • 種別: struct
  • 定義箇所: modules/http/http_filters.c:811

目的

レスポンスヘッダを構築および送信する際、一時的なメモリ管理とデータバッファリングを行うためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct header_struct {
    apr_pool_t *pool;
    apr_bucket_brigade *bb;
} header_struct;

入力と出力

  • pool: ヘッダ構築に使用するメモリプール。
  • bb: ヘッダデータを格納し、フィルタチェーンへ渡すためのバケットブリゲード。

前提条件と不変条件

  • pool は、ヘッダ構築処理の生存期間中有効である必要があります。
  • bb は、apr_bucket_brigade API を通じて適切に初期化および破棄される必要があります。

アルゴリズムまたは振る舞い

この構造体は、apr_table_do 等の反復処理において、ヘッダフィールドをバッファリングするために使用されます。form_header_field でバッファへ書き込まれ、send_all_header_fields を通じて最終的にフィルタチェーンへ送出されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_filters.c 内の関数（ap_http_header_filter 等）でローカル変数としてインスタンス化されます。
  • **呼び出し先**:
  • form_header_field (引数として渡される)
  • send_all_header_fields (引数として渡される)

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_bucket_brigade: ストリームデータ（ヘッダ）の保持に使用。

失敗モードと境界条件

  • メモリ不足: pool からの割り当て失敗時に apr_bucket_brigade の操作が失敗する可能性があります。
  • フィルタチェーンの切断: bb を下位フィルタへ渡す際に接続が閉じている場合、エラーが発生します。

保守担当者向けメモ

  • 本構造体は http_filters.c 内のヘッダ送信ロジックに特化しています。
  • フィルタチェーン内で bb を操作する際は、バケットのメモリリークを防ぐため、必ず apr_brigade_destroy または適切なバケット削除処理を行ってください。
