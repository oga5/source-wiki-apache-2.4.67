is_idempotent

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ajp.c:119

目的

HTTPリクエストが冪等（idempotent）であるかを判定し、バックエンド通信失敗時の自動リトライ可否を決定する。

シグネチャまたは形状

static int is_idempotent(request_rec *r)

入力と出力

  • **入力**: request_rec *r （処理対象のリクエスト情報）
  • **出力**: 以下のいずれかの整数値を返す。
  • METHOD_IDEMPOTENT: 冪等であり、リトライ可能。
  • METHOD_IDEMPOTENT_WITH_ARGS: 冪等だが引数（r->args）が存在するため、自動リトライには慎重を要する状態。
  • METHOD_NON_IDEMPOTENT: 冪等ではない。

前提条件と不変条件

  • RFC 2616 (9.1.2) に基づき、GET, HEAD, PUT, DELETE, OPTIONS, TRACE メソッドを冪等とみなす。
  • HEAD リクエストは内部的に M_GET として扱われる。

アルゴリズムまたは振る舞い

1. r->method_number を評価する。 2. 対象メソッドが M_GET, M_DELETE, M_PUT, M_OPTIONS, M_TRACE のいずれかである場合:

  • r->args が存在すれば METHOD_IDEMPOTENT_WITH_ARGS を返す。
  • それ以外は METHOD_IDEMPOTENT を返す。

    • 3. 上記以外のメソッドはすべて METHOD_NON_IDEMPOTENT を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_ajp.c:254
  • modules/proxy/mod_proxy_ajp.c:398
  • modules/proxy/mod_proxy_ajp.c:708
  • **呼び出し先**:
  • 本関数内での関数呼び出しは確認されていない。

関連データ構造

  • request_rec: Apache のリクエスト構造体。method_number および args フィールドを参照する。

失敗モードと境界条件

  • r->args が存在する場合、メソッド自体が冪等であっても METHOD_IDEMPOTENT_WITH_ARGS を返すことで、副作用の可能性を考慮した制御フローへ誘導する。

保守担当者向けメモ

  • 本関数は RFC 2616 に準拠しているが、将来的に HTTP 仕様の更新や新しいメソッドのサポートが必要になった場合は、switch 文のケース追加を検討すること。
