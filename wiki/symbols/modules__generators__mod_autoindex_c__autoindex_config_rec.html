autoindex_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_autoindex.c:120

目的

ディレクトリごとの自動索引生成（mod_autoindex）に関する設定情報を保持・管理するための構造体です。

シグネチャまたは形状

typedef struct autoindex_config_struct {
    char *default_icon;
    char *style_sheet;
    char *head_insert;
    char *header;
    char *readme;
    apr_int32_t opts;
    apr_int32_t incremented_opts;
    apr_int32_t decremented_opts;
    int name_width;
    int name_adjust;
    int desc_width;
    int desc_adjust;
    int icon_width;
    int icon_height;
    char default_keyid;
    char default_direction;
    apr_array_header_t *icon_list;
    apr_array_header_t *alt_list;
    apr_array_header_t *desc_list;
    apr_array_header_t *ign_list;
    int ign_noinherit;
    char *ctype;
    char *charset;
    char *datetime_format;
} autoindex_config_rec;

入力と出力

  • 本構造体は、Apache の設定フレームワークを通じてディレクトリ単位で生成・マージされます。
  • create_autoindex_config 関数で初期化され、merge_autoindex_configs 関数で親ディレクトリの設定を継承します。

前提条件と不変条件

  • すべての動的メモリ割り当ては、リクエストまたは設定用の apr_pool_t に依存します。
  • apr_array_header_t 型の各リスト（icon_list, alt_list, desc_list, ign_list）は、設定ディレクティブの解析時に push_item 等を用いて動的に構築されます。

アルゴリズムまたは振る舞い

  • ディレクトリ内のファイル一覧を表示する際、本構造体に格納された opts（表示オプション）や ign_list（無視するファイルパターン）を参照して索引生成の挙動を制御します。
  • name_width や desc_width などのフィールドは、生成される HTML テーブルのレイアウト調整に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_get_module_config を介してリクエスト処理（handle_autoindex）や設定解析関数から参照されます。
  • **呼び出し先**: push_item 関数がリストへのデータ追加のために本構造体のメンバを引数として受け取ります。また、APR_OFFSETOF マクロを用いて設定ディレクティブのオフセット指定に使用されます。

関連データ構造

  • struct item: アイコンや代替テキストのパターンマッチング用データ構造。
  • struct ent: 索引生成時に各ファイル/ディレクトリのメタデータを保持する一時的な構造体。

失敗モードと境界条件

  • 設定の継承時、ign_noinherit が設定されている場合、親ディレクトリからの IndexIgnore リストの継承が停止されます。
  • 大規模なディレクトリにおいて、本構造体に基づく設定値を用いた走査やサブリクエストの実行がパフォーマンスに影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体は mod_autoindex の設定状態を完全に表現します。新しい設定ディレクティブを追加する場合は、この構造体にフィールドを追加し、create および merge 関数を更新する必要があります。
  • find_title 等の関数が本構造体の設定値を利用して HTML 生成を行うため、変更時は出力への影響範囲を確認してください。
