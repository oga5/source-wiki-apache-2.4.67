dav_fs_dir_file_name

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/repos.c:238

目的

WebDAVリソースの絶対パスから、親ディレクトリのパスとファイル名（またはディレクトリ名）を分離して抽出する。

シグネチャまたは形状

dav_error * dav_fs_dir_file_name(
    const dav_resource *resource,
    const char **dirpath_p,
    const char **fname_p)

入力と出力

  • **入力**:
  • resource: 対象となるWebDAVリソースへのポインタ。
  • dirpath_p: 親ディレクトリパスを格納するポインタのアドレス。
  • fname_p: ファイル名（またはディレクトリ名）を格納するポインタのアドレス（NULL指定可能）。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時は dav_error 構造体へのポインタを返す。

前提条件と不変条件

  • resource->info に dav_resource_private 構造体が格納されている必要がある。
  • resource->info->pathname は有効な絶対パスであること。
  • ctx->pool を使用してメモリが割り当てられる。

アルゴリズムまたは振る舞い

1. リソースがコレクション（ディレクトリ）の場合:

  • dirpath_p に ctx->pathname を設定し、fname_p が指定されていれば NULL を設定する。

    • 2. リソースがファイルの場合:

  • ap_make_dirstr_parent を使用して親ディレクトリパスを生成する。
  • apr_filepath_root を用いてパスのルートを検証する。
  • パス末尾の不要なスラッシュを除去する。
  • 成功時、dirpath_p に親ディレクトリパスを、fname_p にファイル名部分（パスの残余）を設定する。
  • パス解析に失敗した場合は HTTP_INTERNAL_SERVER_ERROR を含むエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/dbm.c:192
  • modules/dav/fs/lock.c:919, 936
  • modules/dav/fs/repos.c:571, 572, 624
  • **呼び出し先**:
  • ap_make_dirstr_parent
  • apr_filepath_root
  • dav_new_error

関連データ構造

  • dav_resource: WebDAVリソースの抽象化。
  • dav_resource_private: ファイルシステム固有のプライベートデータ（pathname を保持）。

失敗モードと境界条件

  • パスが解析不能な場合（apr_filepath_root が APR_SUCCESS または APR_ERELATIVE 以外を返した場合）、HTTP_INTERNAL_SERVER_ERROR を伴うエラーを返す。
  • fname_p が NULL の場合、ファイル名の抽出処理をスキップする。

保守担当者向けメモ

  • 本関数は repos.h で宣言されており、ファイルシステムプロバイダーの内部操作においてパスの分解が必要な箇所で広く利用されている。
  • パス操作には APR の関数を使用しているため、プラットフォーム固有のパス形式に依存する可能性がある。
