ap_fcgi_header_fields_from_array

定義

  • 種別: function
  • 定義箇所: server/util_fcgi.c:52

目的

FastCGI プロトコルのバイナリヘッダー（バイト配列）から、各フィールドを個別の変数に展開します。

シグネチャまたは形状

void ap_fcgi_header_fields_from_array(unsigned char *version,
                                      unsigned char *type,
                                      apr_uint16_t *request_id,
                                      apr_uint16_t *content_len,
                                      unsigned char *padding_len,
                                      unsigned char a[])

入力と出力

  • **入力**:
  • a[]: FastCGI ヘッダーデータを含むバイト配列。
  • **出力**:
  • version: プロトコルバージョン。
  • type: レコードタイプ。
  • request_id: リクエスト識別子（16ビット）。
  • content_len: コンテンツ長（16ビット）。
  • padding_len: パディング長。

前提条件と不変条件

  • 引数 a は、FastCGI ヘッダー構造を保持するのに十分なサイズ（少なくとも 8 バイト）の配列である必要があります。
  • 本関数はメモリ割り当てを行わず、提供されたポインタに対して値を書き込みます。

アルゴリズムまたは振る舞い

1. a 配列の固定オフセットから version、type、padding_len を直接抽出します。 2. request_id および content_len については、2つのバイト（B1, B0）をビットシフトおよび加算により 16 ビット整数として再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:508
  • modules/proxy/mod_proxy_fcgi.c:776
  • **呼び出し先**:
  • 本関数は外部関数を呼び出しません。

関連データ構造

  • include/util_fcgi.h: FastCGI プロトコル定義およびオフセット定数（AP_FCGI_HDR_*_OFFSET）が定義されています。

失敗モードと境界条件

  • 配列 a が不正なデータを含む場合、抽出される値も不正となります。本関数自体はバリデーションを行いません。

保守担当者向けメモ

  • FastCGI の仕様変更やオフセット定義の変更があった場合は、include/util_fcgi.h の定数定義と本関数のオフセット参照が一致していることを確認してください。
