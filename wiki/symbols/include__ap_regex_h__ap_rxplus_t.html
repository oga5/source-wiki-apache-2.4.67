ap_rxplus_t

定義

  • 種別: struct
  • 定義箇所: include/ap_regex.h:219

目的

Perl 形式の正規表現（s/match/replace/flags 等）を扱うための、高レベルな正規表現操作用構造体です。PCRE をラップした ap_regex_t を内包し、置換文字列やマッチング結果の管理を簡略化します。

シグネチャまたは形状

typedef struct {
    ap_regex_t rx;            /* コンパイル済み正規表現 */
    apr_uint32_t flags;       /* 動作フラグ */
    const char *subs;         /* 置換文字列 */
    const char *match;        /* マッチ対象文字列 */
    apr_size_t nmatch;        /* マッチング結果の数 */
    ap_regmatch_t *pmatch;    /* マッチしたオフセット情報 */
} ap_rxplus_t;

入力と出力

  • **入力**: ap_rxplus_compile 関数により、正規表現パターン文字列から本構造体が生成されます。
  • **出力**: ap_rxplus_exec 関数等を通じて、マッチングの成否や置換結果、および pmatch に格納されたオフセット情報が取得されます。

前提条件と不変条件

  • apr_pool_t を使用してメモリ管理が行われるため、本構造体の生存期間は割り当てられたプールの生存期間に依存します。
  • ap_rxplus_compile で初期化された後、ap_rxplus_exec 等の操作関数に渡す必要があります。

アルゴリズムまたは振る舞い

1. **コンパイル**: ap_rxplus_compile がパターンを解析し、rx フィールドを初期化します。 2. **実行**: ap_rxplus_exec が対象文字列に対してマッチングを行い、結果を pmatch に書き込みます。 3. **情報取得**: ap_rxplus_nmatch や ap_rxplus_match を用いて、マッチした範囲や置換後の文字列情報を抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authnz_fcgi.c や modules/filters/mod_proxy_html.c 等のモジュールから参照されています。
  • **呼び出し先**: server/util_regex.c 内の ap_rxplus_compile、ap_rxplus_exec、ap_rxplus_nmatch 等によって操作されます。

関連データ構造

  • ap_regex_t: 内部で使用されるコンパイル済み正規表現構造体。
  • ap_regmatch_t: マッチした文字列のオフセット情報を保持する配列要素。
  • apr_pool_t: メモリ管理に使用されるプール構造体。

失敗モードと境界条件

  • ap_rxplus_compile 実行時にパターンが不正な場合、NULL が返される可能性があります。
  • ap_rxplus_exec 実行時にメモリ不足やマッチングエラーが発生した場合、適切なエラーコードが返されます。

保守担当者向けメモ

  • 本構造体は include/ap_regex.h で定義されており、PCRE の詳細を隠蔽する抽象化層の一部です。
  • ap_rxplus 系関数を使用する際は、メモリリークを防ぐため、必ず適切な apr_pool_t のライフサイクルを考慮してください。
