cgi_bucket_read

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:348

目的

CGIスクリプトの標準出力（stdout）および標準エラー出力（stderr）を監視し、データを読み取るためのバケット読み取り関数です。

シグネチャまたは形状

apr_status_t cgi_bucket_read(apr_bucket *b, const char **str, apr_size_t *len, apr_read_type_e block)

入力と出力

  • **入力**:
  • b: 読み取り対象の apr_bucket。内部データとして cgi_bucket_data を保持。
  • str: 読み取ったデータを格納するバッファへのポインタ。
  • len: 読み取ったデータの長さを格納する変数へのポインタ。
  • block: ブロッキング読み取りを行うかどうかの指定（apr_read_type_e）。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • バケット b の data フィールドには、有効な struct cgi_bucket_data が格納されている必要があります。
  • data->pollset が初期化済みであり、監視対象のパイプが登録されている必要があります。

アルゴリズムまたは振る舞い

1. block 指定に基づき、タイムアウト時間を決定します（非ブロッキング時は0、それ以外は設定値またはサーバーのタイムアウト値）。 2. apr_pollset_poll を呼び出し、スクリプトの出力パイプを監視します。

  • タイムアウト発生時: 設定に応じて警告ログを出力し rv を返すか、APR_EAGAIN を返します。
  • 割り込み（EINTR）発生時: 再試行します。
  • エラー発生時: エラーログを出力し rv を返します。

    • 3. ポーリング結果をループで処理します。

  • stdout（client_data == 1）の場合: cgi_read_stdout を呼び出しデータを取得します。
  • stderr の場合: log_script_err を呼び出し、エラー出力をログへ転送します。EOFに達した場合は pollset から削除します。

    • 4. stdout からデータが取得できるまで、上記ポーリングを繰り返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は modules/generators/cgi_common.h:410 に存在することが示唆されています。
  • **呼び出し先**:
  • apr_pollset_poll
  • ap_log_rerror
  • cgi_read_stdout
  • log_script_err
  • apr_pollset_remove

関連データ構造

  • struct cgi_bucket_data: ポーリング対象の pollset、request_rec、タイムアウト値を含む管理構造体。
  • apr_bucket: Apache のバケット抽象化構造体。

失敗モードと境界条件

  • **タイムアウト**: スクリプトからの応答がない場合、設定されたタイムアウト時間経過後に APR_TIMEUP を返します。
  • **ポーリング失敗**: apr_pollset_poll が失敗した場合、エラーログを出力しそのステータスを返します。
  • **EOF**: stderr がEOFに達した場合は監視対象から除外されます。stdout がEOFの場合は APR_SUCCESS として処理されます。

保守担当者向けメモ

  • 本関数は cgi_bucket_data を介してリクエストコンテキストに依存しています。
  • poll() のブロック動作はスクリプトの実行時間に直結するため、タイムアウト設定の妥当性に注意してください。
  • client_data に (void *)1 を使用して stdout を識別する実装となっているため、バケット作成時の登録ロジックと整合性を保つ必要があります。
