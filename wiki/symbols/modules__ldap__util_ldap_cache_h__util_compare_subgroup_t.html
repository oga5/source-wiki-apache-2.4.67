util_compare_subgroup_t

定義

  • 種別: struct
  • 定義箇所: modules/ldap/util_ldap_cache.h:103

目的

LDAP 比較操作において、サブグループの識別名（DN）リストとその件数を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct util_compare_subgroup_t {
    const char **subgroupDNs;
    int len;
} util_compare_subgroup_t;

入力と出力

  • subgroupDNs: サブグループの DN を格納する文字列配列へのポインタ。
  • len: subgroupDNs 配列に含まれる要素数。

前提条件と不変条件

  • subgroupDNs は、キャッシュ管理用のメモリ領域（util_ald_alloc 等で確保された領域）またはリクエストプール上に配置されることが想定されています。
  • len は配列の有効な要素数と一致している必要があります。

アルゴリズムまたは振る舞い

本構造体は、LDAP 比較結果のキャッシュノードの一部として保持されます。データの複製には util_ald_sgl_dup が、メモリ解放には util_ald_sgl_free が使用され、キャッシュのライフサイクル管理に統合されています。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c 内の uldap_get_subgroups 関数などで、LDAP 検索結果からサブグループ情報を抽出する際にインスタンス化されます。
  • **呼び出し先**: util_ald_sgl_dup（複製用）および util_ald_sgl_free（解放用）が本構造体を引数として操作を行います。

関連データ構造

  • util_compare_node_t: 本構造体をメンバとして保持する可能性のある比較結果ノード。
  • util_ald_cache_t: 本構造体のメモリ管理（割り当て・解放）を制御するキャッシュ管理構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時、subgroupDNs が NULL になる可能性があります。
  • len が 0 の場合、subgroupDNs は空のリストまたは NULL を指すことが想定されます。

保守担当者向けメモ

  • 本構造体は util_ldap_cache_mgr.c で提供されるメモリ管理関数と密接に結合しています。構造体のメンバを変更する場合は、util_ald_sgl_dup および util_ald_sgl_free の実装も併せて更新してください。
