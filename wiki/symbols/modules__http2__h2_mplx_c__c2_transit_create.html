c2_transit_create

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_mplx.c:200

目的

HTTP/2 リクエスト処理用の独立したメモリ環境（h2_c2_transit）を生成する。この構造体は、親接続とは異なるスレッドでリクエストを処理するために必要なメモリプールとバケットアロケータを保持する。

シグネチャまたは形状

static h2_c2_transit *c2_transit_create(h2_mplx *m)

入力と出力

  • **入力**: h2_mplx *m（多重化器インスタンスへのポインタ）
  • **出力**: 初期化された h2_c2_transit 構造体へのポインタ。失敗時は NULL を返す。

前提条件と不変条件

  • m->pool が有効であること。
  • 内部で生成されるメモリプール ptrans は、m->pool を親として作成されるが、専用の apr_allocator_t を所有し、スレッド間でのメモリ分離を保証する。
  • ptrans には h2_c2_transit というタグが設定される。

アルゴリズムまたは振る舞い

1. apr_allocator_create を呼び出し、専用のアロケータを作成する。 2. apr_pool_create_ex を使用し、作成したアロケータを紐付けたメモリプール ptrans を生成する。 3. 生成に失敗した場合は ap_log_cerror でエラーを記録し、ap_abort_on_oom() を呼び出してプロセスを終了させる。 4. apr_allocator_owner_set でアロケータの所有者を ptrans に設定する。 5. apr_pool_abort_set でメモリ不足時のハンドラを登録する。 6. apr_pcalloc で h2_c2_transit 構造体を確保し、プールとバケットアロケータを初期化して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_mplx.c:247（c2_transit_get 内と推測されるが、ソースコード上では直接の呼び出し箇所のみ確認可能）
  • **呼び出し先**: apr_allocator_create, apr_pool_create_ex, apr_allocator_owner_set, apr_pool_abort_set, apr_pool_tag, apr_pcalloc, apr_bucket_alloc_create

関連データ構造

  • h2_c2_transit: リクエスト処理のライフサイクルを管理するコンテナ。
  • h2_mplx: 多重化器のメイン構造体。
  • apr_pool_t: リクエスト処理用の独立したメモリプール。
  • apr_allocator_t: スレッド間分離を実現するためのメモリ管理機構。

失敗モードと境界条件

  • メモリ割り当て（apr_allocator_create または apr_pool_create_ex）が失敗した場合、ap_abort_on_oom() が呼び出され、プログラムは異常終了する。

保守担当者向けメモ

  • 本関数は、リクエスト処理を親接続のスレッドから切り離すための重要な境界点である。
  • ap_abort_on_oom が設定されているため、メモリ不足は回復不能なエラーとして扱われる。
  • transit->bucket_alloc は ptrans に紐付いているため、ptrans の破棄時に自動的にクリーンアップされる。
