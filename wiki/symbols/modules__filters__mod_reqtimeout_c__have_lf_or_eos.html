have_lf_or_eos

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reqtimeout.c:107

目的

バケットブリゲード（apr_bucket_brigade）の末尾から逆方向に走査し、データ内に改行（LF）が含まれているか、またはストリームの終端（EOS）に到達したかを判定する。

シグネチャまたは形状

apr_status_t have_lf_or_eos(apr_bucket_brigade *bb)

入力と出力

  • **入力**: bb - 検査対象のバケットブリゲードへのポインタ。
  • **出力**: apr_status_t 型のステータスコード。
  • APR_SUCCESS: 改行（LF）が見つかった、または EOS バケットが検出された場合。
  • APR_INCOMPLETE: ブリゲード内に改行も EOS も見つからなかった場合。
  • その他: apr_bucket_read 実行時にエラーが発生した場合はそのエラーコード。

前提条件と不変条件

  • bb は有効なバケットブリゲードであること。
  • メタデータバケット（EOS 以外）はスキップされる。

アルゴリズムまたは振る舞い

1. ブリゲードの末尾から APR_BRIGADE_SENTINEL に達するまで、APR_BUCKET_PREV を用いて逆方向にバケットを走査する。 2. EOS バケットを検出した場合は直ちに APR_SUCCESS を返す。 3. メタデータバケットは無視して次のバケットへ進む。 4. データバケットの場合、apr_bucket_read で内容を読み取り、バッファの末尾（len-1）が APR_ASCII_LF であるかを確認する。 5. 改行が見つかった場合は APR_SUCCESS を返す。 6. 走査が完了しても条件を満たさない場合は APR_INCOMPLETE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_reqtimeout.c:261（reqtimeout_filter 関数内）。
  • **呼び出し先**: apr_bucket_read（APR API）。

関連データ構造

  • apr_bucket_brigade: バケットのリストを管理する構造体。
  • apr_bucket: データの断片やメタデータを保持する構造体。

失敗モードと境界条件

  • **読み取り失敗**: apr_bucket_read がエラーを返した場合、そのエラーコードを伝播する。
  • **空のバケット**: len == 0 のバケットはスキップされる。
  • **メタデータ**: EOS 以外のメタデータバケットは走査対象外となる。

保守担当者向けメモ

  • 本関数は AP_MODE_GETLINE 処理において、行の終端を効率的に検出するために使用される。
  • 逆方向走査を行うため、ブリゲードの構成が複雑な場合でも末尾のデータに迅速にアクセスできる。
