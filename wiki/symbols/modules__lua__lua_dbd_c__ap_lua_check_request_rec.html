ap_lua_check_request_rec

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:29

目的

Lua スタック上の指定されたインデックスにあるユーザーデータから、Apache の request_rec 構造体ポインタを安全に取得・検証します。

シグネチャまたは形状

request_rec *ap_lua_check_request_rec(lua_State *L, int index)

入力と出力

  • **入力**:
  • L: Lua 状態へのポインタ。
  • index: request_rec が格納されている Lua スタックのインデックス。
  • **出力**:
  • 成功時: request_rec 構造体へのポインタを返します。
  • 失敗時: luaL_checkudata により、型が一致しない場合に Lua エラーを発生させます。

前提条件と不変条件

  • 指定された index に Apache2.Request 型のユーザーデータが存在する必要があります。
  • lua_unboxpointer が呼び出される前に、luaL_checkudata による型チェックが必須です。

アルゴリズムまたは振る舞い

1. luaL_checkudata を呼び出し、スタックの index にあるデータが Apache2.Request 型であることを検証します。型が一致しない場合、この時点で処理が中断され Lua エラーがスローされます。 2. lua_unboxpointer を使用して、ユーザーデータ内に格納されている request_rec ポインタを取り出します。 3. 抽出したポインタを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_dbd.c: lua_db_acquire, lua_db_query, lua_db_select, lua_db_prepared_select, lua_db_prepared_query
  • modules/lua/lua_request.c: 複数のリクエスト処理関数
  • **呼び出し先**:
  • luaL_checkudata (Lua API)
  • lua_unboxpointer (Lua API)

関連データ構造

  • request_rec: Apache HTTP Server のリクエストコンテキストを保持する構造体。
  • lua_State: Lua インタプリタの状態を保持する構造体。

失敗モードと境界条件

  • index が無効な場合や、スタック上のデータが Apache2.Request 型でない場合、luaL_checkudata が Lua エラーを発生させ、呼び出し元の実行が停止します。

保守担当者向けメモ

  • 本関数は mod_lua 全体で request_rec を取得するための標準的なユーティリティです。変更を加える際は、lua_request.c 側の呼び出し元にも影響が及ぶ点に注意してください。
