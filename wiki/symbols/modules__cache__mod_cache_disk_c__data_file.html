data_file

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:92

目的

キャッシュ対象の URI に対応するディスク上のデータファイル（ボディ部分）の絶対パスを生成する。

シグネチャまたは形状

char *data_file(apr_pool_t *p, disk_cache_conf *conf, disk_cache_object_t *dobj, const char *name)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • conf: ディスクキャッシュの設定情報。
  • dobj: キャッシュオブジェクトの状態を保持する構造体。
  • name: キャッシュキーとなる文字列。
  • **出力**:
  • キャッシュデータファイルのフルパスを示す文字列を返す。

前提条件と不変条件

  • dobj->hashfile が未設定の場合、ap_cache_generate_name を用いてハッシュ値が生成され、dobj にキャッシュされる。
  • 返されるパスは、設定された cache_root または dobj->prefix を基点とし、CACHE_DATA_SUFFIX で終わる。

アルゴリズムまたは振る舞い

1. dobj->hashfile が存在しない場合、conf の設定（dirlevels, dirlength）に基づき、name からハッシュ化されたファイル名を生成し dobj->hashfile に格納する。 2. dobj->prefix が設定されている場合、dobj->prefix、CACHE_VDIR_SUFFIX、ハッシュファイル名、CACHE_DATA_SUFFIX を連結したパスを返す。 3. dobj->prefix が設定されていない場合、conf->cache_root、ハッシュファイル名、CACHE_DATA_SUFFIX を連結したパスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c:395
  • modules/cache/mod_cache_disk.c:522
  • modules/cache/mod_cache_disk.c:1050
  • **呼び出し先**:
  • ap_cache_generate_name (キャッシュキーからのハッシュ生成)
  • apr_pstrcat (パス文字列の構築)

関連データ構造

  • disk_cache_conf: ディスクキャッシュのルートパスやディレクトリ構造設定を保持。
  • disk_cache_object_t: キャッシュ対象のハッシュファイル名やプレフィックスを保持。

失敗モードと境界条件

  • メモリ不足時には apr_pstrcat 等の内部関数によりアボートまたはエラーが発生する可能性がある。
  • name が NULL または空の場合の挙動は ap_cache_generate_name の実装に依存する。

保守担当者向けメモ

  • mod_socache_dbm.c に同名のシンボルが存在するが、本関数とは別物である（mod_cache_disk 固有のパス生成ロジックである点に注意）。
  • CACHE_DATA_SUFFIX の定義変更は、既存のキャッシュファイルへのアクセス不能を招くため注意が必要。
