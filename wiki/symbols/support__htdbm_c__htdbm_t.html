htdbm_t

定義

  • 種別: struct
  • 定義箇所: support/htdbm.c:58

目的

Apache HTTP Server の DBM 形式パスワードデータベースを操作する際、データベース接続情報、ユーザー情報、および実行コンテキストや設定状態を保持するために使用されるメイン構造体です。

シグネチャまたは形状

typedef struct htdbm_t htdbm_t;

※構造体の内部フィールド定義は、提供されたソース抜粋には含まれていません。

入力と出力

  • **入力**: htdbm_init 関数により、APR プールを介してメモリが割り当てられます。
  • **出力**: データベース操作関数（htdbm_save, htdbm_del 等）において、現在の操作対象や状態を保持する引数として渡されます。

前提条件と不変条件

  • すべての動的メモリは apr_pool_t によって管理されます。
  • htdbm_t の各フィールドはプールから割り当てられるため、明示的な free は不要です。
  • DBM 内のデータは username をキーとし、password[:comment] を値として保存されます。

アルゴリズムまたは振る舞い

本構造体は、以下のライフサイクルおよび操作において状態保持の役割を果たします。

1. **初期化**: htdbm_init により構造体が生成され、APR 環境と紐付けられます。 2. **接続管理**: htdbm_open を通じて DBM ファイルとの接続状態が管理されます。 3. **データ操作**: htdbm_make で生成されたハッシュ値や、htdbm_save による永続化処理のコンテキストとして利用されます。 4. **終了**: htdbm_terminate により、保持している DBM 接続のリソースが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htdbm.c 内の main 関数および各操作関数（htdbm_init, htdbm_open, htdbm_save, htdbm_del, htdbm_verify, htdbm_list, htdbm_make, htdbm_valid_username）がこの構造体を引数として受け取ります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出す側です。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_dbm_t: htdbm_open 等で内部的に管理される DBM 接続情報（構造体内に保持されると推測されます）。
  • passwd_ctx: passwd_common.c で定義される認証コンテキスト（関連するハッシュ生成ロジックで使用）。

失敗モードと境界条件

  • apr_pcalloc によるメモリ割り当て失敗時、プログラムは異常終了する可能性があります。
  • DBM ファイルのオープンや書き込みに失敗した場合、htdbm_t が保持する接続状態が不正になる可能性があります。
  • \0 終端が保存されないため、データアクセス時には apr_datum_t の dsize を参照する必要があります。

保守担当者向けメモ

  • htdbm_t の内部構造は support/htdbm.c 内で完結しています。
  • EBCDIC 環境下では、ハッシュ計算前に文字コード変換が行われるため、構造体内のデータ保持形式に注意してください。
  • 構造体の詳細なフィールド定義については、support/htdbm.c のソースコードを参照してください。
