val_url_print

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:352

目的

管理対象ドメイン（Managed Domain）に関連する URL を、HTML またはテキスト形式でステータス出力用ブリゲード (apr_bucket_brigade) に書き出す。

シグネチャまたは形状

static void val_url_print(status_ctx *ctx, const status_info *info,
                          const char *url, const char *proto, int i)

入力と出力

  • **入力**:
  • ctx: ステータス出力コンテキスト（プール、出力先ブリゲード、フォーマットフラグを保持）。
  • info: 現在のステータスセルの定義情報。
  • url: 表示対象の URL 文字列。
  • proto: 未使用（void キャスト済み）。
  • i: インデックス番号（0 の場合は単一項目、それ以外は複数項目の識別子として使用）。
  • **出力**: なし（副作用として ctx->bb に文字列を書き出す）。

前提条件と不変条件

  • url が NULL の場合、関数は何も出力せず即座に復帰する。
  • ctx->p（プール）は有効である必要がある。
  • HTML_STATUS(ctx) マクロにより、出力形式が HTML かテキストかが動的に決定される。

アルゴリズムまたは振る舞い

1. md_get_ca_name_from_url を呼び出し、URL から認証局（CA）の名称を取得する。 2. **HTML モードの場合**:

  • ap_escape_html2 を使用して URL と CA 名をエスケープし、<a> タグを含む HTML リンクとして出力する。

    • 3. **テキストモードの場合**:

  • i == 0 の場合: info->label を用いて「Name: ...」「URL: ...」の形式で出力する。
  • i > 0 の場合: 「Name[i]: ...」「URL[i]: ...」の形式でインデックス付き出力を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_status.c:400
  • modules/md/mod_md_status.c:408
  • **呼び出し先**:
  • md_get_ca_name_from_url
  • ap_escape_html2
  • apr_brigade_printf

関連データ構造

  • status_ctx: ステータス生成の制御状態を保持。
  • status_info: ステータス表示のラベル定義。

失敗モードと境界条件

  • url が NULL の場合は早期リターンする。
  • ap_escape_html2 によるエスケープ処理が失敗した場合の挙動は、Apache HTTP Server の基盤ライブラリに依存する。

保守担当者向けメモ

  • proto 引数は現在使用されていないため、将来的な拡張が不要であれば削除を検討すること。
  • HTML 出力時のリンク生成において、ap_escape_html2 を使用してクロスサイトスクリプティング（XSS）を防ぐ設計となっている。
