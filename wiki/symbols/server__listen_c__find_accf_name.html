find_accf_name

定義

  • 種別: function
  • 定義箇所: server/listen.c:225

目的

指定されたプロトコルに関連付けられたアクセプトフィルタ（Accept Filter）の名前を、サーバー設定から取得する。

シグネチャまたは形状

static const char* find_accf_name(server_rec *s, const char *proto)

入力と出力

  • **入力**:
  • s: サーバー設定を保持する server_rec 構造体へのポインタ。
  • proto: プロトコル名を示す文字列。
  • **出力**:
  • 成功時: プロトコルに対応するアクセプトフィルタ名（文字列）。
  • 失敗または該当なし: NULL。

前提条件と不変条件

  • s->module_config から取得される core_server_config が有効であること。
  • proto が NULL の場合、関数は直ちに NULL を返す。

アルゴリズムまたは振る舞い

1. サーバー設定から core_server_config を取得する。 2. proto が NULL であれば NULL を返す。 3. conf->accf_map テーブルから proto に対応する値を検索する。 4. 取得した値が "none" という文字列と一致する場合、アクセプトフィルタが無効であると判断し NULL を返す。 5. それ以外の場合は、検索結果の文字列をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/listen.c:257（alloc_listener 関数内から呼び出される）。
  • **呼び出し先**:
  • ap_get_core_module_config
  • apr_table_get
  • strcmp

関連データ構造

  • server_rec: サーバー設定の基盤構造体。
  • core_server_config: コアモジュールの設定を保持する構造体。accf_map を通じてプロトコルとアクセプトフィルタの対応を管理する。

失敗モードと境界条件

  • proto が NULL の場合、安全に NULL を返す。
  • accf_map 内に該当するプロトコルが存在しない場合、apr_table_get の挙動に従い NULL を返す。
  • 設定値が "none" である場合、明示的に NULL を返すことでフィルタの無効化を表現する。

保守担当者向けメモ

  • 本関数は AcceptFilter ディレクティブの設定値を解決するために使用される。
  • accf_map は apr_table_t 型であり、apr_table_get による検索はケースセンシティブである点に注意すること。
