char_buffer_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_engine_io.c:318

目的

SSL I/O 処理において、読みすぎたデータや一時的なバッファリングが必要なデータを保持するための汎用的なメモリ領域を提供します。

シグネチャまたは形状

typedef struct {
    int length;
    char *value;
} char_buffer_t;

入力と出力

  • length: バッファ内の有効なデータ長（バイト単位）。
  • value: データを格納する文字配列へのポインタ。

前提条件と不変条件

  • value が指すメモリ領域は、外部で適切に割り当ておよび解放される必要があります。
  • length は value が指すバッファの有効な範囲を正確に反映している必要があります。

アルゴリズムまたは振る舞い

  • AP_MODE_GETLINE 等の操作でネットワークから読み込みすぎたデータを一時保持し、次回の読み取り要求時に再利用することで、効率的なデータ転送を実現します。
  • char_buffer_read および char_buffer_write 関数を介して、バッファに対するデータの読み書きが行われます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ssl/ssl_engine_io.c 内の各 I/O フィルタ関数。
  • 呼び出し先: char_buffer_read および char_buffer_write が本構造体を操作します。

関連データ構造

  • bio_filter_in_ctx_t: 入力フィルタのコンテキスト内で本構造体が利用されます。

失敗モードと境界条件

  • バッファの容量を超えた書き込みや、未初期化の value ポインタへのアクセスは未定義動作を引き起こします。
  • length が負の値になるような操作は想定されていません。

保守担当者向けメモ

  • 本構造体はメモリ管理を自動化していないため、使用するコンテキスト（bio_filter_in_ctx_t 等）のライフサイクルと同期してメモリを管理する必要があります。
