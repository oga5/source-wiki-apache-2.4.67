h2_conn_mpm_name

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2.c:119

目的

現在実行中の Apache MPM（Multi-Processing Module）の名前を取得します。

シグネチャまたは形状

const char *h2_conn_mpm_name(void)

入力と出力

  • 入力: なし
  • 出力: MPM 名を表す文字列（const char *）。MPM が特定できない場合は "unknown" を返します。

前提条件と不変条件

  • 内部で check_modules(0) を呼び出し、MPM モジュール情報の初期化状態を確認します。
  • mpm_module グローバル変数が有効であることに依存しています。

アルゴリズムまたは振る舞い

1. check_modules(0) を呼び出し、モジュール情報の整合性を確認します。 2. グローバル変数 mpm_module が存在するか判定します。 3. 存在する場合は mpm_module->name を返し、存在しない場合は "unknown" を返します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/mod_http2.c:137 からの呼び出しが確認されています。
  • 呼び出し先: check_modules（同ファイル内）を呼び出します。

関連データ構造

  • mpm_module: Apache HTTP Server の MPM 情報を保持する内部構造体。

失敗モードと境界条件

  • check_modules が正常に初期化を完了できない場合、または MPM がロードされていない場合、戻り値は "unknown" となります。

保守担当者向けメモ

  • 本関数は mod_http2 が動作する環境の MPM を特定するために使用されます。
  • check_modules は静的シンボルとして定義されていますが、本ページではその詳細な内部実装については言及しません。
