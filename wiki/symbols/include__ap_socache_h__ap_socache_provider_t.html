ap_socache_provider_t

定義

  • 種別: struct
  • 定義箇所: include/ap_socache.h:89

目的

Apache HTTP Server における「小規模オブジェクトキャッシュ」の抽象化レイヤーを定義する構造体です。メモリ、共有メモリ、ディスクなど、多様なバックエンド実装を統一的なインターフェースで利用可能にします。

シグネチャまたは形状

typedef struct ap_socache_provider_t {
    const char *name;
    unsigned int flags;
    const char *(*create)(ap_socache_instance_t **instance, const char *arg, apr_pool_t *tmp, apr_pool_t *p);
    apr_status_t (*init)(ap_socache_instance_t *instance, const char *cname, const struct ap_socache_hints *hints, server_rec *s, apr_pool_t *pool);
    void (*destroy)(ap_socache_instance_t *instance, server_rec *s);
    apr_status_t (*store)(ap_socache_instance_t *instance, server_rec *s, const unsigned char *id, unsigned int idlen, apr_time_t expiry, unsigned char *data, unsigned int datalen, apr_pool_t *pool);
    apr_status_t (*retrieve)(ap_socache_instance_t *instance, server_rec *s, const unsigned char *id, unsigned int idlen, unsigned char *data, unsigned int *datalen, apr_pool_t *pool);
    apr_status_t (*remove)(ap_socache_instance_t *instance, server_rec *s, const unsigned char *id, unsigned int idlen, apr_pool_t *pool);
    void (*status)(ap_socache_instance_t *instance, request_rec *r, int flags);
    apr_status_t (*iterate)(ap_socache_instance_t *instance, server_rec *s, void *userctx, ap_socache_iterator_t *iterator, apr_pool_t *pool);
} ap_socache_provider_t;

入力と出力

  • **入力**: キャッシュ操作に必要なインスタンス、サーバー構造体、バイナリデータ（ID/データ本体）、有効期限、メモリプール。
  • **出力**: 操作結果を示す apr_status_t（成功時は APR_SUCCESS）、または retrieve におけるデータ本体と書き込みサイズ。

前提条件と不変条件

  • init に渡す cname は最大16文字で、英数字（[a-z0-9_-]）のみの使用が推奨されます。
  • retrieve の datalen は入出力両用です。呼び出し時にバッファサイズを渡し、戻り値として実際に書き込まれたサイズを受け取ります。
  • AP_SOCACHE_FLAG_NOTMPSAFE フラグが設定されている場合、呼び出し側で明示的な排他制御が必要です。

アルゴリズムまたは振る舞い

1. **登録**: ap_register_provider を介して AP_SOCACHE_PROVIDER_GROUP に登録されます。 2. **生成**: create でインスタンスを生成し、init で識別名とヒントを適用して運用を開始します。 3. **操作**: store, retrieve, remove によりバイナリデータの読み書きを行います。 4. **走査**: iterate を通じてキャッシュ内の全オブジェクトをコールバック関数で処理します。 5. **破棄**: destroy によりリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_ssl（セッションキャッシュ）、mod_cache_socache、mod_authn_socache など。
  • **呼び出し先**: 各キャッシュ実装モジュール（mod_socache_shmcb.c, mod_socache_dbm.c, mod_socache_dc.c, mod_socache_memcache.c, mod_socache_redis.c）。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスを保持する不透明な型。
  • ap_socache_hints: キャッシュの期待される利用状況を記述するヒント構造体。
  • server_rec, request_rec: Apache のサーバーおよびリクエスト構造体。

失敗モードと境界条件

  • create は失敗時にエラー文字列を返します。
  • retrieve は対象が見つからない場合に APR_NOTFOUND を返します。
  • status メソッドは mod_status 向けですが、内部で非推奨の ap_r* インターフェースを使用しています。

保守担当者向けメモ

  • 本構造体はインターフェース定義であり、実装は各 mod_socache_*.c に存在します。
  • iterate メソッドのシグネチャは include/ap_mmn.h で定義される API バージョン変更の影響を受ける可能性があります。
