exipc_post_config

定義

  • 種別: function
  • 定義箇所: modules/examples/mod_example_ipc.c:119

目的

Apache HTTP Server の起動フェーズにおいて、プロセス間通信（IPC）に必要な共有メモリセグメントの確保と、グローバルミューテックスの生成を行う。

シグネチャまたは形状

static int exipc_post_config(apr_pool_t *pconf, apr_pool_t *plog,
                             apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • plog: ログ用メモリプール。
  • ptemp: 一時作業用メモリプール。
  • s: サーバーレコード。
  • **出力**:
  • 成功時は OK を返す。
  • 失敗時は HTTP_INTERNAL_SERVER_ERROR を返す。

前提条件と不変条件

  • サーバーのメイン状態が AP_SQ_MS_CREATE_PRE_CONFIG である場合は、リソース確保を行わず即座に終了する。
  • 共有メモリのファイル名は、OS の一時ディレクトリ内に現在のプロセス ID を含めて生成される。

アルゴリズムまたは振る舞い

1. **状態確認**: サーバー起動時の初期段階であれば処理をスキップする。 2. **ディレクトリ取得**: apr_temp_dir_get を使用し、共有メモリ用ファイルの配置先を取得する。 3. **共有メモリ確保**: apr_shm_create を呼び出し、exipc_data サイズの共有メモリを確保する。 4. **初期化**: 確保した共有メモリのベースアドレスを取得し、カウンタを 0 にリセットする。 5. **ミューテックス生成**: ap_global_mutex_create を呼び出し、プロセス間排他制御用のミューテックスを生成する。 6. **クリーンアップ登録**: apr_pool_cleanup_register を使用し、サーバー再起動時に共有メモリを解放するラッパー関数を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_config により、Apache の post_config フックとして登録されている。
  • **呼び出し先**:
  • ap_state_query, apr_temp_dir_get, ap_log_error, apr_psprintf, apr_shm_create, apr_shm_baseaddr_get, ap_global_mutex_create, apr_pool_cleanup_register。

関連データ構造

  • exipc_data: 共有メモリ内に配置されるカウンタ保持用構造体。
  • exipc_shm: 共有メモリブロックを指すグローバルポインタ。
  • exipc_mutex: プロセス間で共有されるグローバルミューテックス。

失敗モードと境界条件

  • 一時ディレクトリの取得に失敗した場合、エラーログを出力して終了する。
  • 共有メモリの作成に失敗した場合、エラーログを出力して終了する。
  • ミューテックスの生成に失敗した場合、エラーを返して終了する。

保守担当者向けメモ

  • 本関数は親プロセスで実行されることを前提としている。
  • 共有メモリのファイル名は shmfilename グローバル変数に格納され、子プロセスに継承される。
  • サーバー再起動時のリソース解放は shm_cleanup_wrapper を介して行われるため、クリーンアップ処理を変更する際は当該関数も確認すること。
