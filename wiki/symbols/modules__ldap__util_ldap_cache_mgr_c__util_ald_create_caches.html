util_ald_create_caches

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache_mgr.c:281

目的

特定のURLに関連付けられた「検索」「比較」「DN比較」の3種類のLDAPキャッシュを生成し、それらを管理するためのURLノードをキャッシュシステムに登録する。

シグネチャまたは形状

util_url_node_t *util_ald_create_caches(util_ldap_state_t *st, const char *url)

入力と出力

  • **入力**:
  • st: LDAP状態管理構造体（キャッシュサイズやTTLの設定値を含む）。
  • url: キャッシュのキーとなるURL文字列。
  • **出力**:
  • 成功時: 登録された util_url_node_t 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • st が有効な util_ldap_state_t を指していること。
  • 内部で呼び出される util_ald_create_cache が、各キャッシュタイプに対して正常にメモリを割り当てられること。

アルゴリズムまたは振る舞い

1. util_ald_create_cache を3回呼び出し、検索、比較、DN比較用の各キャッシュを初期化する。 2. 3つのキャッシュすべてが正常に生成された場合：

  • スタック上に util_url_node_t を確保し、URLと各キャッシュポインタを格納する。
  • util_ald_cache_insert を呼び出し、このノードをメインのキャッシュ管理構造に挿入する。

    • 3. いずれか一つでも生成に失敗した場合：

  • 生成済みのキャッシュに対して util_ald_destroy_cache を呼び出し、リソースを解放して NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap.c (複数箇所)
  • **呼び出し先**:
  • util_ald_create_cache
  • util_ald_cache_insert
  • util_ald_destroy_cache

関連データ構造

  • util_url_node_t: URLごとのキャッシュセットを保持する構造体。
  • util_ald_cache_t: 個別のキャッシュ管理構造体。
  • util_ldap_state_t: モジュール全体の設定と状態を保持する構造体。

失敗モードと境界条件

  • キャッシュ生成時にメモリ不足等で util_ald_create_cache が NULL を返した場合、生成済みのキャッシュを破棄し、関数全体として NULL を返す。
  • util_ald_cache_insert が失敗した場合も同様に NULL が返される。

保守担当者向けメモ

  • util_url_node_t の内容は挿入時に共有メモリへ複製されるため、スタックメモリを使用してもリークの心配はない。
  • キャッシュのパージ処理は O(n) であり、キャッシュサイズが大きい場合は注意が必要である。
