isapi_unload

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_isapi.c:220

目的

ロード済みの ISAPI DLL をアンロードし、関連するリソースを解放します。

シグネチャまたは形状

apr_status_t isapi_unload(isapi_loaded *isa, int force)

入力と出力

  • **入力**:
  • isa: アンロード対象の isapi_loaded 構造体へのポインタ。
  • force: 強制アンロードを行うかを示すフラグ（0以外で強制）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、アンロードが拒否された場合は APR_EGENERAL を返します。

前提条件と不変条件

  • isa->handle が NULL の場合、何もせず APR_SUCCESS を返します。
  • isa->TerminateExtension が定義されている場合、DLL のアンロード前にその関数を呼び出して終了処理を行います。

アルゴリズムまたは振る舞い

1. isa->handle が NULL であれば即座に終了します。 2. TerminateExtension 関数が存在する場合:

  • force が真であれば、HSE_TERM_MUST_UNLOAD を指定して呼び出します。
  • force が偽であれば、HSE_TERM_ADVISORY_UNLOAD を指定して呼び出します。戻り値が 1 でない場合はアンロードを中止し、APR_EGENERAL を返します。

    • 3. apr_dso_unload を呼び出して DLL をアンロードします。 4. isa->handle を NULL に設定し、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_isapi.c:251 (強制アンロード時)
  • **呼び出し先**:
  • isa->TerminateExtension (関数ポインタ経由)
  • apr_dso_unload

関連データ構造

  • isapi_loaded: DLL のハンドルおよび終了関数ポインタを保持する構造体。

失敗モードと境界条件

  • DLL が HSE_TERM_ADVISORY_UNLOAD を受け取った際にアンロードを拒否（戻り値が 0）した場合、関数は APR_EGENERAL を返し、アンロードは実行されません。

保守担当者向けメモ

  • 本関数は isapi_loaded 構造体の状態を更新するため、呼び出し側で適切な排他制御（apr_thread_rwlock_t 等）が行われていることを確認してください。
