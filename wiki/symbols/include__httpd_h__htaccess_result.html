htaccess_result

定義

  • 種別: struct
  • 定義箇所: include/httpd.h:792

目的

.htaccess ファイルによって適用される設定情報およびオーバーライド権限を保持するための構造体です。

シグネチャまたは形状

struct htaccess_result {
    const char *dir;
    int override;
    int override_opts;
    apr_table_t *override_list;
    struct ap_conf_vector_t *htaccess;
    const struct htaccess_result *next;
};

入力と出力

  • **入力**: server/config.c 等において、リクエストプール (r->pool) からメモリを割り当てて初期化されます。
  • **出力**: リンクされたリスト構造として、後続のディレクトリ設定や設定ベクトルを保持します。

前提条件と不変条件

  • next フィールドはリストの終端を示すために NULL を指す必要があります。
  • next フィールドの内容は変更してはならないという規約があります。

アルゴリズムまたは振る舞い

ディレクトリ階層を辿る際、各階層で適用される .htaccess の設定をリストとして蓄積します。server/config.c 内で apr_palloc を用いて動的に生成され、リクエスト処理の過程で参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/config.c においてインスタンス化が行われます。
  • **呼び出し先**: include/httpd.h 内の request_rec 構造体（htaccess フィールド）から参照されます。
  • **補足**: include/ap_mmn.h および include/http_config.h で言及されていますが、具体的な関数呼び出し関係の詳細はソース抜粋からは不明です。

関連データ構造

  • apr_table_t: 許可されたディレクティブのリストを保持するために使用されます。
  • ap_conf_vector_t: 設定ディレクティブのベクトルを保持します。
  • request_rec: この構造体を保持する上位のコンテキストです。

失敗モードと境界条件

  • メモリ割り当て失敗時: apr_palloc が失敗した場合の挙動は APR のメモリ管理ポリシーに依存します。
  • リストの終端: next が NULL であることを確認せずに走査するとセグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はリクエストプールに紐付いているため、明示的な解放は不要です。
  • next ポインタの不変性（N.B. never change this）を維持してください。
