cookie_log_state

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_usertrack.c:70

目的

mod_usertrack モジュールにおいて、サーバー全体の設定（Cookie の有効期限など）を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    int always;
    int expires;
} cookie_log_state;

入力と出力

  • 本構造体は設定保持用であり、直接的な関数入力や出力は持ちません。
  • expires フィールドは、Cookie の有効期限を秒単位で保持するために使用されます。

前提条件と不変条件

  • expires フィールドは、set_cookie_exp 関数によって解析された期間値が格納されます。
  • サーバーのモジュール設定（module_config）の一部として管理されます。

アルゴリズムまたは振る舞い

  • サーバー起動時または設定読み込み時に、make_cookie_log_state（推測される初期化関数）等を通じてインスタンス化され、ap_get_module_config を介してリクエスト処理中に参照されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/metadata/mod_usertrack.c:101: ap_get_module_config を介してサーバー設定を取得。
  • modules/metadata/mod_usertrack.c:273-274: apr_palloc を使用してメモリを割り当て。
  • modules/metadata/mod_usertrack.c:301: 設定ポインタとして参照。
  • 呼び出し先:
  • 情報なし（本構造体はデータ保持用であり、関数呼び出しは行いません）。

関連データ構造

  • cookie_dir_rec: ディレクトリごとの設定を保持する構造体。
  • request_rec: リクエスト処理中に本構造体を参照する際のコンテキスト。

失敗モードと境界条件

  • メモリ割り当て失敗時: apr_palloc が失敗した場合、Apache のエラーハンドリングに従いリクエスト処理が中断される可能性があります。
  • 設定値の不整合: expires に負の値や異常な値が設定された場合の挙動は、set_cookie_exp の解析ロジックに依存します。

保守担当者向けメモ

  • 本構造体はサーバー全体の設定を管理します。ディレクトリ単位の設定が必要な場合は cookie_dir_rec を参照してください。
  • always フィールドの具体的な意味と利用箇所については、ソースコード内のロジック（spot_cookie 等）との照合が必要です。
