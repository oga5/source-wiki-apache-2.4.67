dav_get_provider

定義

  • 種別: function
  • 定義箇所: modules/dav/main/mod_dav.c:215

目的

現在処理中のリクエストに関連付けられた WebDAV プロバイダー（dav_provider）を取得します。

シグネチャまたは形状

const dav_provider *dav_get_provider(request_rec *r)

入力と出力

  • **入力**: request_rec *r (Apache のリクエストレコード)
  • **出力**: const dav_provider * (リクエストのディレクトリ設定に紐付くプロバイダー構造体へのポインタ)

前提条件と不変条件

  • r->per_dir_config に dav_module の設定が格納されている必要があります。
  • WebDAV が有効なコンテキストで呼び出されることを前提としており、conf->provider_name および conf->provider が NULL でないことが保証されています。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、リクエストのディレクトリ設定 (dav_dir_conf) を取得します。 2. 設定構造体内に保持されている provider メンバを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.c 内の各フック関数（dav_get_lock_hooks, dav_get_propdb_hooks, dav_get_vsn_hooks, dav_get_binding_hooks, dav_get_search_hooks など）。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core API)

関連データ構造

  • dav_dir_conf: ディレクトリごとの設定を保持する構造体。provider メンバを保持します。
  • dav_provider: リポジトリ操作やフックを定義する構造体。

失敗モードと境界条件

  • 本関数自体はエラーチェックを行わず、設定が存在することを前提としています。DAV が無効な環境で呼び出された場合の挙動は未定義です（ソース内のアサーションコメント参照）。

保守担当者向けメモ

  • 本関数は、WebDAV プロバイダーの機能（ロック、プロパティ、バージョン管理等）へアクセスするためのゲートウェイです。
  • 戻り値の dav_provider は定数として扱われるべきです。
