on_stream_close_cb

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:264

目的

HTTP/2 ストリームが終了した際に、nghttp2 ライブラリから通知を受け取り、関連する内部リソースのクリーンアップとエラー処理を行う。

シグネチャまたは形状

static int on_stream_close_cb(nghttp2_session *ngh2, int32_t stream_id,
                              uint32_t error_code, void *userp)

入力と出力

  • **入力**:
  • ngh2: nghttp2_session へのポインタ（未使用）。
  • stream_id: 終了したストリームの ID。
  • error_code: ストリーム終了理由を示すエラーコード（0 は正常終了）。
  • userp: h2_session 構造体へのポインタ。
  • **出力**:
  • 常に 0 を返す。

前提条件と不変条件

  • userp は有効な h2_session 構造体を指している必要がある。
  • get_stream を通じて、指定された stream_id に対応する h2_stream が取得可能であること。

アルゴリズムまたは振る舞い

1. userp から h2_session を取得する。 2. get_stream を呼び出し、対象の h2_stream を検索する。 3. ストリームが存在する場合:

  • error_code が 0 以外（エラー発生）であれば、ログを出力する。
  • h2_stream_rst を呼び出し、ストリームをリセット状態にする。
  • h2_mplx_c1_client_rst を呼び出し、多重化器に対してクライアント側でのリセットを通知する。

    • 4. 処理を終了し 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: nghttp2 ライブラリのコールバック機構（modules/http2/h2_session.c:728 にて NGH2_SET_CALLBACK を通じて登録）。
  • **呼び出し先**:
  • get_stream
  • ap_log_cerror
  • h2_protocol_err_description
  • h2_stream_rst
  • h2_mplx_c1_client_rst

関連データ構造

  • h2_session: セッション状態を保持するメイン構造体。
  • h2_stream: 個別の HTTP/2 ストリーム状態を管理する構造体。
  • nghttp2_session: nghttp2 ライブラリのセッションオブジェクト。

失敗モードと境界条件

  • get_stream が NULL を返した場合（ストリームが見つからない場合）、エラーログ出力やリセット処理はスキップされ、何も行わずに終了する。

保守担当者向けメモ

  • 本関数は nghttp2 のコールバックとして登録されるため、シグネチャの変更は nghttp2 の仕様に準拠する必要がある。
  • error_code が 0 の場合、ログ出力やリセット処理は行われないため、正常終了時のクリーンアップ処理が必要な場合は別途考慮が必要。
