get_line

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2_filter.c:418

目的

HTTP/1.1 形式のレスポンスデータ（ブリゲード）から、CRLF で終了する 1 行を読み取り、指定されたバッファへコピーする。

シグネチャまたは形状

apr_status_t get_line(h2_response_parser *parser, apr_bucket_brigade *bb, char *line, apr_size_t len)

入力と出力

  • **入力**:
  • parser: 解析状態を保持する h2_response_parser 構造体。
  • bb: 読み込み元のバケットブリゲード。
  • line: 読み取った行を格納する文字配列。
  • len: line バッファのサイズ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、データが不完全な場合は APR_EAGAIN、行が長すぎる場合は APR_ENOSPC を返す。

前提条件と不変条件

  • parser->tmp は、行の断片を保持するためのブリゲードとして使用される。
  • parser->saveto は、不完全な行を次回の呼び出しまで保持するために使用される。
  • 行末の CRLF は除去され、line はヌル終端される。

アルゴリズムまたは振る舞い

1. apr_brigade_split_line を使用して、入力ブリゲード bb から 1 行分を parser->tmp へ切り出す。 2. 切り出したデータを apr_brigade_flatten で line バッファにコピーする。 3. 行末に H2_CRLF が存在する場合、それを除去して APR_SUCCESS を返す。 4. H2_CRLF が見つからない場合、ブリゲードの長さを確認し、バッファサイズを超える場合は APR_ENOSPC を返す。 5. 行が不完全な場合は、ap_save_brigade を用いて parser->tmp の内容を parser->saveto に退避し、APR_EAGAIN を返して処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c2_filter.c:597 (解析ロジック内)。
  • **呼び出し先**: apr_brigade_split_line, apr_brigade_flatten, apr_brigade_length, ap_save_brigade, ap_log_cerror 等の APR/Apache API。
  • *注記*: modules/ssl/ssl_util_ocsp.c や support/htdigest.c に同名の関数が存在するが、本シンボルとの直接的な呼び出し関係は確認されていない。

関連データ構造

  • h2_response_parser: 解析状態（プール、コネクション、一時ブリゲード等）を保持。

失敗モードと境界条件

  • **APR_ENOSPC**: 行の長さが len を超える場合、またはブリゲードの合計長がバッファサイズを超える場合に発生。
  • **APR_EAGAIN**: 行末の CRLF がまだブリゲード内に存在しない場合に発生し、後続のデータ到着を待機する。
  • **制限事項**: HUGE_STRING_LEN を超える行は処理できない。LF のみで CRLF が存在しない場合の挙動は未定義（ソースコード上の XXX コメント参照）。

保守担当者向けメモ

  • 本関数は h2_c2_filter_catch_h1_out からの呼び出しを想定している。
  • parser->tmp に一時的なバケットが含まれる可能性があるため、ap_save_brigade による退避処理が重要である。
  • HUGE_STRING_LEN を超える行への対応が必要な場合は、バッファ管理ロジックの拡張を検討すること。
