rline

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:714

目的

sed スクリプトまたは入力ストリームから、1行分のテキストを読み込み、バッファに格納する。

シグネチャまたは形状

static int rline(sed_commands_t *commands, apr_file_t *fin, char *lbuf, char *lbend)

入力と出力

  • **入力**:
  • commands: sed コマンドのコンテキスト構造体。
  • fin: 読み込み対象のファイルポインタ。
  • lbuf: 行データを格納するバッファの開始ポインタ。
  • lbend: バッファの終了ポインタ。
  • **出力**:
  • 成功時: 1 を返す。
  • 失敗時: -1 を返す。
  • 副作用: lbuf に読み込んだ文字列を書き込み、commands->saveq を更新する。

前提条件と不変条件

  • lbuf は lbend までの十分なメモリ領域を確保していること。
  • commands->eflag が設定されている場合、ファイル入力ではなく commands->earg からの読み込みが優先される。

アルゴリズムまたは振る舞い

1. **モード判定**: commands->eflag が有効な場合、メモリ上の文字列（earg）から行を読み込む。無効な場合、apr_file_read を使用してファイルから 1 バイトずつ読み込む。 2. **エスケープ処理**: 文字列中に \ が出現した場合、直後の文字をリテラルとして読み込む。 3. **行終端判定**: 改行文字（\n）を検出すると、そこまでの内容を lbuf に格納し、\0 で終端して終了する。 4. **バッファ境界チェック**: 読み込み中に lbend に達した場合、command_errf を呼び出し SEDERR_CLTL エラーを報告して -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/sed0.c 内の fcomp 関数（行単位のスクリプト解析時）。
  • **呼び出し先**: command_errf（エラー報告時）、apr_file_read（ファイル読み込み時）。

関連データ構造

  • sed_commands_t: sed スクリプトのコンパイル状態と入力バッファを保持する構造体。

失敗モードと境界条件

  • **バッファオーバーフロー**: lbuf が lbend に達した場合はエラーとなる。

保守担当者向けメモ

  • apr_file_read を用いた 1 バイトずつの読み込みはパフォーマンス上のボトルネックとなる可能性がある。
  • S5EMUL マクロの定義有無により、テキスト挿入時のインデント除去挙動が変化する可能性がある。
