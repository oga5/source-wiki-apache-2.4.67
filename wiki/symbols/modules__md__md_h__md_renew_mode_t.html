md_renew_mode_t

定義

  • 種別: enum
  • 定義箇所: modules/md/md.h:68

目的

Managed Domain (MD) における証明書更新プロセスの実行モードを定義します。

シグネチャまたは形状

typedef enum {
    MD_RENEW_DEFAULT = -1,
    MD_RENEW_MANUAL,
    MD_RENEW_AUTO,
    MD_RENEW_ALWAYS,
} md_renew_mode_t;

入力と出力

  • 本シンボルは列挙型であり、設定値として利用されます。
  • 入力: 設定ファイル等から読み込まれた値。
  • 出力: 証明書更新ロジックにおける動作モードの決定。

前提条件と不変条件

  • MD_RENEW_DEFAULT は -1 として定義されています。
  • 各列挙子は、証明書の更新が必要か否か、および更新を誰が（またはどのようなトリガーで）行うかを決定するポリシーとして機能します。

アルゴリズムまたは振る舞い

  • MD_RENEW_DEFAULT: デフォルトの更新動作。
  • MD_RENEW_MANUAL: 証明書の更新を手動でトリガーする必要があります。
  • MD_RENEW_AUTO: httpd によって自動的に更新プロセスが実行されます。
  • MD_RENEW_ALWAYS: 更新の必要性に関わらず、httpd によって常に更新が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md_config.c:575 にて md_renew_mode_t renew_mode; として変数が宣言されています。
  • 呼び出し先: ソースコードの文脈からは、具体的な関数呼び出し先は特定できません。

関連データ構造

  • md_t: Managed Domain の設定を保持する構造体。本列挙型はこの構造体の一部として設定を管理するために使用されることが想定されます。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本列挙型は mod_md_config.c で設定値として保持されています。更新ロジックの実装詳細については、md_reg.c 等のレジストリ層のコードを参照してください。
