privileges_create_dir_cfg

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_privileges.c:116

目的

ディレクトリ単位の設定構造体（priv_dir_cfg）をメモリ上に割り当て、初期化する。

シグネチャまたは形状

static void *privileges_create_dir_cfg(apr_pool_t *pool, char *dummy)

入力と出力

  • **入力**:
  • pool: 設定構造体の割り当てに使用するメモリプール。
  • dummy: Apache 設定フレームワークから渡される未使用の引数。
  • **出力**:
  • void *: 初期化された priv_dir_cfg 構造体へのポインタ。

前提条件と不変条件

  • pool は有効な apr_pool_t であること。
  • 戻り値の cfg->mode は必ず PRIV_UNSET で初期化される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、priv_dir_cfg サイズ分のメモリを pool から確保する。 2. 確保した構造体の mode メンバに PRIV_UNSET を代入する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/unix/mod_privileges.c:582（Apache 設定フックによる間接的な呼び出し）。
  • **呼び出し先**: apr_palloc（Apache Portable Runtime）。

関連データ構造

  • priv_dir_cfg: 本関数で生成される構造体。ディレクトリごとの PrivilegesMode を保持する。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある（Apache の標準的なメモリ管理動作に従う）。

保守担当者向けメモ

  • 本関数は Apache の設定読み込みフェーズでディレクトリ設定が生成される際に呼び出される。
  • dummy 引数は Apache の create_dir_config フックのシグネチャに合わせるために存在しており、本関数内では使用されない。
