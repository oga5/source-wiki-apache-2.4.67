ap_slotmem_instance_t

定義

  • 種別: struct
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:25

目的

単一プロセス内における固定サイズメモリ領域（スロット）の管理単位を保持する構造体です。

シグネチャまたは形状

struct ap_slotmem_instance_t {
    char                 *name;       /* セグメント名 */
    void                 *base;       /* データ領域の開始アドレス */
    apr_size_t           size;        /* 各スロットのサイズ */
    unsigned int         num;         /* スロットの総数 */
    apr_pool_t           *gpool;      /* セグメントごとのグローバルプール */
    char                 *inuse;      /* 利用状況フラグテーブル */
    ap_slotmem_type_t    type;        /* 型固有のフラグ */
    struct ap_slotmem_instance_t  *next; /* 次の割り当て済みセグメントへのポインタ */
};

入力と出力

本構造体はデータ操作関数（slotmem_get, slotmem_put, slotmem_dptr 等）の引数として渡され、対象となるメモリ領域の特定および状態管理に使用されます。

前提条件と不変条件

  • gpool はインスタンスのライフサイクル全体で生存します。
  • base は num 個のスロットを格納するのに十分な領域を指している必要があります。
  • inuse フラグテーブルは、各スロットの利用状態を追跡するために使用されます。

アルゴリズムまたは振る舞い

  • 本構造体は mod_slotmem_plain.c において、globallistmem リストの一部として管理されます。
  • 各インスタンスは、slotmem_create 時に gpool からメモリを確保し、初期化されます。
  • データアクセス時には、item_id からオフセットを計算し、base アドレスを基準としたメモリ操作が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: ap_slotmem プロバイダーインターフェースを利用する各モジュール。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。
  • 補足: 具体的な呼び出し元・呼び出し先の関係性は、本ソース抜粋からは特定できません。

関連データ構造

  • ap_slotmem_provider_t: 本構造体を利用して操作を行う関数ポインタ群。
  • globallistmem: モジュール内で作成された全インスタンスを保持する静的リスト。

失敗モードと境界条件

  • メモリ境界チェック: slotmem_get/put 関数において、dest_len や src_len が size を超える場合の保護は呼び出し側に委ねられています。
  • 排他制御: 本実装は単一プロセス内を想定しており、マルチスレッド環境での利用には呼び出し側での排他制御が必須です。

保守担当者向けメモ

  • 本構造体は ap_slotmem.h にも定義が存在します。
  • AP_SLOTMEM_TYPE_PREGRAB フラグが設定されている場合、inuse フラグの整合性維持が重要となります。
