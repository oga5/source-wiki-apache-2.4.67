lua_read_body

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:233

目的

HTTP リクエストのボディを読み込み、メモリバッファに格納する。主に POST や PUT リクエストで送信されたデータを Lua スクリプトから利用可能にするために使用される。

シグネチャまたは形状

int lua_read_body(request_rec *r, const char **rbuf, apr_off_t *size, apr_off_t maxsize);

入力と出力

  • **入力**:
  • r: リクエスト情報を保持する request_rec 構造体。
  • maxsize: 読み込みを許可する最大バイト数（0の場合は制限なし）。
  • **出力**:
  • rbuf: 読み込んだボディデータを格納するバッファへのポインタ。
  • size: 実際に読み込まれたデータのバイト数。
  • 戻り値: 処理結果を示すステータスコード（OK, DONE, またはエラーコード）。

前提条件と不変条件

  • メモリ割り当ては r->pool を使用して行われるため、リクエスト終了時に自動的に解放される。
  • maxsize が指定されており、リクエストの残りデータ長がこれを超える場合、読み込みは行われず APR_EINCOMPLETE が返される。

アルゴリズムまたは振る舞い

1. ap_setup_client_block を呼び出し、クライアントからのデータ受信準備を行う。 2. ap_should_client_block でボディの存在を確認する。存在しない場合は DONE を返す。 3. ボディが存在する場合、r->remaining を基に apr_pcalloc でメモリを確保する。 4. ap_get_client_block をループ内で呼び出し、バッファが満たされるまでデータを読み込む。 5. 読み込み中にエラーが発生した場合は APR_EINCOMPLETE を返す。 6. 正常終了時は OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_request.c 内の各箇所（行402、行478付近）。
  • **呼び出し先**:
  • ap_setup_client_block
  • ap_should_client_block
  • ap_get_client_block
  • apr_pcalloc

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • apr_off_t: ファイルサイズやオフセットを扱うための型。

失敗モードと境界条件

  • maxsize を超えるデータが送信された場合: APR_EINCOMPLETE を返す。
  • ap_get_client_block による読み込みが失敗した場合: APR_EINCOMPLETE を返す。
  • ap_setup_client_block がエラーを返した場合: そのエラーコードをそのまま返す。

保守担当者向けメモ

  • 本関数は r->pool を使用してメモリを確保するため、呼び出し側で明示的な解放は不要である。
  • maxsize の設定は、巨大なリクエストボディによるメモリ枯渇を防ぐための重要なセキュリティ境界である。
