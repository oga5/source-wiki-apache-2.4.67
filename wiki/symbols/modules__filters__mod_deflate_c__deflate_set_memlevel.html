deflate_set_memlevel

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_deflate.c:286

目的

Apache 設定ディレクティブ DeflateMemLevel の値を解析し、サーバー設定構造体 deflate_filter_config に適用する。

シグネチャまたは形状

static const char *deflate_set_memlevel(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • arg: 設定値として渡された文字列（数値）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • arg は整数に変換可能であること。
  • 設定値は 1 から 9 の範囲内である必要がある。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定 deflate_filter_config を取得する。 2. atoi(arg) で入力文字列を整数に変換する。 3. 変換後の値が 1 未満または 9 を超える場合、エラーメッセージを返して終了する。 4. 正常な値であれば、設定構造体の memlevel フィールドに値を格納し、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_deflate.c:1927 において、AP_INIT_TAKE1 マクロ経由で DeflateMemLevel ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: ap_get_module_config (Apache API)、atoi (標準ライブラリ)。

関連データ構造

  • deflate_filter_config: 圧縮レベルやメモリレベルなどの設定を保持する構造体。
  • cmd_parms: Apache の設定コマンド実行時に渡されるコンテキスト情報。

失敗モードと境界条件

  • 入力が 1〜9 の範囲外である場合、設定の適用に失敗し、エラーメッセージが返される。
  • atoi が非数値文字列を処理した場合の挙動は標準ライブラリの仕様に依存する（本関数内での明示的なバリデーションは範囲チェックのみ）。

保守担当者向けメモ

  • 本関数は DeflateMemLevel ディレクティブ専用のセッターである。
  • memlevel は zlib の deflateInit2 に渡されるパラメータであり、メモリ使用量と圧縮速度・圧縮率のトレードオフを制御する。
