pool_vado

定義

  • 種別: function
  • 定義箇所: modules/md/md_util.c:51

目的

一時的なメモリプール（サブプール）を生成し、指定されたコールバック関数を実行した後に当該プールを破棄することで、処理中のメモリリークを防止するラッパー関数です。

シグネチャまたは形状

static apr_status_t pool_vado(md_util_vaction *cb, void *baton, apr_pool_t *p, va_list ap)

入力と出力

  • **入力**:
  • cb: 実行するコールバック関数（md_util_vaction 型）。
  • baton: コールバックに渡す不透明なデータポインタ。
  • p: 親となる apr_pool_t。
  • ap: 可変長引数リスト（va_list）。
  • **出力**:
  • apr_status_t: コールバック関数の実行結果、またはプール生成失敗時のステータス。

前提条件と不変条件

  • p は有効な apr_pool_t である必要があります。
  • コールバック関数 cb は、ptemp（サブプール）を引数として受け取り、処理終了後にそのプールが破棄されることを前提として設計されている必要があります。

アルゴリズムまたは振る舞い

1. 親プール p からサブプール ptemp を作成します。 2. apr_pool_tag を使用して、サブプールに "md_pool_vado" というタグを付与します。 3. コールバック関数 cb を呼び出し、baton、親プール p、サブプール ptemp、および可変長引数 ap を渡します。 4. コールバックの実行完了後、apr_pool_destroy(ptemp) を呼び出してサブプールを破棄します。 5. コールバックの戻り値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_util.c:71
  • modules/md/md_util.c:682
  • **呼び出し先**:
  • apr_pool_create
  • apr_pool_tag
  • apr_pool_destroy
  • cb (引数として渡された関数ポインタ)

関連データ構造

  • apr_pool_t: Apache Portable Runtime のメモリ管理用プール構造体。
  • md_util_vaction: pool_vado が実行するコールバック関数の型定義。

失敗モードと境界条件

  • apr_pool_create が失敗した場合、コールバックは実行されず、そのエラーコードが即座に返されます。

保守担当者向けメモ

  • 本関数は va_list を受け取るため、可変長引数を利用する関数（md_util_pool_vdo など）から呼び出されることを想定しています。
  • サブプールは関数終了時に必ず破棄されるため、コールバック内でサブプールに割り当てたメモリを外部に保持させないよう注意してください。
