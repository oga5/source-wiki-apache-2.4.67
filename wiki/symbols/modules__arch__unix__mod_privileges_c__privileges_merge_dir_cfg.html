privileges_merge_dir_cfg

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_privileges.c:122

目的

Apache のディレクトリ単位の設定（priv_dir_cfg）をマージし、親コンテキストから子コンテキストへの設定継承を行う。

シグネチャまたは形状

static void *privileges_merge_dir_cfg(apr_pool_t *pool, void *BASE, void *ADD)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する Apache プール。
  • BASE: 親ディレクトリの設定（priv_dir_cfg *）。
  • ADD: 現在のディレクトリの設定（priv_dir_cfg *）。
  • **出力**:
  • マージされた新しい priv_dir_cfg 構造体へのポインタ。

前提条件と不変条件

  • BASE および ADD は priv_dir_cfg 型としてキャスト可能であること。
  • pool は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、新しい priv_dir_cfg 構造体用のメモリをプールから確保する。 2. add->mode が PRIV_UNSET（未設定）である場合、親の設定である base->mode を採用する。 3. add->mode が設定されている場合、その値を優先して採用する。 4. マージ結果を格納した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定マージ機構により呼び出される（modules/arch/unix/mod_privileges.c:583 に参照の痕跡あり）。
  • **呼び出し先**: apr_palloc（Apache APR ライブラリ）。

関連データ構造

  • priv_dir_cfg: ディレクトリ単位の権限設定を保持する構造体。mode フィールド（PrivilegesMode）を管理する。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある。
  • BASE または ADD が NULL の場合の挙動は、Apache の設定マージの呼び出し規約に依存する。

保守担当者向けメモ

  • 本関数は PrivilegesMode の継承ロジックのみを実装している。将来的に priv_dir_cfg に新しい設定項目が追加された場合は、本関数内で適切にマージ処理を拡張する必要がある。
