md_pkey_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_crypt.h:47

目的

mod_md モジュールにおいて、秘密鍵（RSA/EC）を保持するための不透明な（opaque）構造体です。OpenSSL の内部的な秘密鍵表現をラップし、Apache HTTP Server の apr_pool_t によるメモリ管理と統合します。

シグネチャまたは形状

typedef struct md_pkey_t md_pkey_t;

※不透明な構造体として定義されており、内部メンバは公開されていません。

入力と出力

  • **入力**: 本構造体は md_pkey_gen() 等の生成関数を通じて初期化されます。
  • **出力**: 署名生成、ファイルへの保存（md_pkey_fsave）、または ACME プロトコルにおけるアカウント認証等の暗号操作に使用されます。

前提条件と不変条件

  • apr_pool_t に紐付いており、関連付けられたプールが破棄されると、内部の OpenSSL オブジェクトも自動的に解放されます。
  • 秘密鍵の所有権は、生成または読み込みを行った関数およびそのプールに依存します。

アルゴリズムまたは振る舞い

  • OpenSSL の低レイヤー API を直接操作することなく、mod_md 内の他のコンポーネントが秘密鍵を安全に扱えるように抽象化を提供します。
  • 秘密鍵の生成、ファイル入出力、および JSON 形式との相互変換をサポートするインターフェースの基盤となります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c: ACME アカウント鍵として使用。
  • modules/md/md_acme_acct.c: アカウントのロードおよび署名処理で使用。
  • **呼び出し先**:
  • 本構造体は不透明な型であるため、具体的な OpenSSL 関数への直接の呼び出し先は md_crypt.c 内の実装に隠蔽されています。

関連データ構造

  • md_pkey_spec_t: 鍵の生成要件（RSA ビット数や EC 曲線名）を定義する仕様構造体。
  • md_cert_t: 秘密鍵と対になる X.509 証明書を保持する構造体。

失敗モードと境界条件

  • メモリ不足時や、OpenSSL の初期化が不十分な環境では、鍵の生成や読み込みが失敗する可能性があります。
  • 構造体自体は不透明であるため、直接的な境界条件の検証は提供される API（md_pkey_gen 等）の戻り値を通じて行う必要があります。

保守担当者向けメモ

  • 本構造体は OpenSSL の実装詳細を隠蔽することを目的としています。内部構造を変更する場合は、md_crypt.c の実装と整合性を取る必要があります。
  • 複数のソースファイル（md_acme.h, md.h 等）で前方宣言されているため、型定義の変更時は依存関係に注意してください。
