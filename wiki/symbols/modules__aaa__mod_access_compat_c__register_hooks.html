register_hooks

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_access_compat.c:359

目的

Apache HTTP Server のフックシステムに対し、mod_access_compat モジュールのアクセス制御機能およびオプション関数を登録する。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（モジュール初期化用のメモリプール）
  • **出力**: なし（副作用としてフックおよびオプション関数を登録）

前提条件と不変条件

  • 本関数は Apache HTTP Server のモジュール読み込みプロセスにおいて、フック登録フェーズで呼び出されることを前提とする。

アルゴリズムまたは振る舞い

1. APR_REGISTER_OPTIONAL_FN を使用し、access_compat_ap_satisfies をオプション関数として公開する。 2. ap_hook_check_access を呼び出し、check_dir_access 関数をアクセスチェックフックとして登録する。

  • 優先度は APR_HOOK_MIDDLE を指定。
  • AP_AUTH_INTERNAL_PER_CONF フラグを付与し、内部的な設定ごとのアクセス制御として動作させる。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール読み込み機構（直接的な呼び出し元はソースコード上では明示されていないが、Apache の標準的なモジュールライフサイクルに従う）。
  • **呼び出し先**:
  • APR_REGISTER_OPTIONAL_FN（マクロ）
  • ap_hook_check_access（Apache API）
  • check_dir_access（本モジュール内の関数）
  • access_compat_ap_satisfies（本モジュール内の関数）

関連データ構造

  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • フック登録の失敗に関する明示的なエラーハンドリングは本関数内には存在しない。

保守担当者向けメモ

  • 本関数は mod_access_compat のエントリポイントの一つであり、Apache 2.4 以降の環境でレガシーなアクセス制御構文を有効化するために不可欠である。
  • check_dir_access は r->user が設定されていなくても動作可能なため、アクセスチェッカーとして登録されている。
