SHMCBSubcache

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_shmcb.c:84

目的

共有メモリベースのキャッシュ（SHMCB）において、各サブキャッシュ内の循環バッファ（インデックス領域およびデータ領域）の状態を管理するために使用されます。

シグネチャまたは形状

typedef struct {
    unsigned int idx_pos, idx_used;
    unsigned int data_pos, data_used;
} SHMCBSubcache;

入力と出力

本構造体は関数ではなくデータ構造であるため、入力と出力の概念は適用されません。各メンバは以下の役割を持ちます。

  • idx_pos: インデックス循環バッファの開始位置。
  • idx_used: インデックス循環バッファ内で現在使用されているサイズ。
  • data_pos: データ循環バッファの開始位置。
  • data_used: データ循環バッファ内で現在使用されているサイズ。

前提条件と不変条件

  • 共有メモリレイアウトにおいて、各サブキャッシュの先頭に配置される必要があります。
  • SHMCBHeader に続くサブキャッシュ領域の一部として存在します。
  • 循環バッファの操作（読み書き）は、本構造体のメンバが保持する位置情報とサイズ情報に基づいて行われます。

アルゴリズムまたは振る舞い

  • 本構造体自体は状態を保持するコンテナです。
  • 循環バッファのラップアラウンド処理や、データ領域の確保・解放を行う関数（shmcb_subcache_expire 等）によって、メンバの値が更新されます。

呼び出し元と呼び出し先

  • 呼び出し元: socache_shmcb_init（初期化時）、shmcb_subcache_expire、shmcb_subcache_retrieve、shmcb_subcache_remove 等のサブキャッシュ操作関数。
  • 呼び出し先: 本構造体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • SHMCBHeader: 共有メモリ全体のメタデータを管理する構造体。
  • SHMCBIndex: サブキャッシュ内の個別のキャッシュエントリを管理する構造体。

失敗モードと境界条件

  • **オーバーフロー**: unsigned int を使用しているため、非常に大きなキャッシュサイズを指定した場合、サイズ計算時にオーバーフローが発生するリスクがあります。
  • **メモリ配置**: 共有メモリ内の固定長レイアウトに依存しているため、ポインタ演算が正しく行われない場合、メモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は AP_SOCACHE_FLAG_NOTMPSAFE が指定されたモジュールの一部であり、スレッドセーフではありません。排他制御は呼び出し側で行う必要があります。
  • 構造体のサイズは ALIGNED_SUBCACHE_SIZE マクロを通じてアライメント調整された値が使用されます。
