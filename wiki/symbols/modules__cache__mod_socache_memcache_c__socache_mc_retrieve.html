socache_mc_retrieve

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_memcache.c:230

目的

Memcached バックエンドからキャッシュデータを取得し、指定されたバッファへコピーします。

シグネチャまたは形状

apr_status_t socache_mc_retrieve(ap_socache_instance_t *ctx, server_rec *s,
                                 const unsigned char *id, unsigned int idlen,
                                 unsigned char *dest, unsigned int *destlen,
                                 apr_pool_t *p);

入力と出力

  • **入力**:
  • ctx: Memcached インスタンスコンテキスト。
  • s: サーバーレコード。
  • id, idlen: 取得対象のキャッシュキーを特定するバイナリデータとその長さ。
  • dest: 取得したデータを格納するバッファ。
  • destlen: dest バッファのサイズ（入力時）。
  • p: メモリ割り当て用プール。
  • **出力**:
  • dest: 取得されたデータが書き込まれます。
  • destlen: 実際に書き込まれたデータの長さが更新されます。
  • 戻り値: apr_status_t 型のステータスコード。

前提条件と不変条件

  • socache_mc_id2key によるキー生成が成功すること。
  • destlen は、取得するデータのサイズを保持するのに十分な大きさであること。

アルゴリズムまたは振る舞い

1. socache_mc_id2key を呼び出し、内部 ID を Memcached 用の文字列キーに変換します。失敗時は APR_EINVAL を返します。 2. apr_memcache_getp を呼び出し、Memcached からデータを取得します。 3. 取得失敗時、APR_NOTFOUND 以外であればエラーログを出力し、そのステータスを返します。 4. 取得したデータのサイズが destlen を超える場合、APLOG_ERR を記録し APR_ENOMEM を返します。 5. 正常時は memcpy で dest にデータをコピーし、destlen を更新して APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_memcache.c:359（socache_mc 構造体内の関数ポインタ経由）。
  • **呼び出し先**: socache_mc_id2key, apr_memcache_getp, ap_log_error, memcpy。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスのコンテキスト。
  • server_rec: Apache サーバー設定構造体。
  • apr_memcache: apr_memcache_getp で使用される Memcached クライアントオブジェクト。

失敗モードと境界条件

  • socache_mc_id2key 失敗: APR_EINVAL を返します。
  • Memcached 接続エラーまたはデータ不在: apr_memcache_getp の戻り値をそのまま返します。
  • バッファオーバーフロー: 取得データが destlen を超える場合、APR_ENOMEM を返します。

保守担当者向けメモ

  • apr_memcache_getp はメモリを大量に消費する可能性があるため、呼び出し時のメモリ管理に注意してください。
  • MC_KEY_LEN（254バイト）を超えるキーは socache_mc_id2key で拒否されます。
