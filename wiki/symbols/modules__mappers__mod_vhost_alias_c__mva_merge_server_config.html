mva_merge_server_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_vhost_alias.c:84

目的

親サーバー設定と子サーバー設定をマージし、新しいサーバー設定構造体を作成する。

シグネチャまたは形状

static void *mva_merge_server_config(apr_pool_t *p, void *parentv, void *childv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • parentv: 親サーバーの設定構造体 (mva_sconf_t*)。
  • childv: 子サーバーの設定構造体 (mva_sconf_t*)。
  • **出力**:
  • マージされた新しい mva_sconf_t 構造体へのポインタ。

前提条件と不変条件

  • parentv および childv は mva_sconf_t 型としてキャスト可能であること。
  • apr_pcalloc によりメモリがゼロ初期化された状態で構造体が生成される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して新しい mva_sconf_t 構造体をメモリ上に確保する。 2. doc_root_mode および cgi_root_mode を確認する。 3. 子設定の値が VHOST_ALIAS_UNSET である場合、親設定の値を継承する。 4. 子設定の値が設定されている場合、その値を優先して新しい構造体にコピーする。 5. マージされた構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の設定マージ処理（modules/mappers/mod_vhost_alias.c:453 にて参照）。
  • **呼び出し先**: apr_pcalloc（APR API）。

関連データ構造

  • mva_sconf_t: サーバー単位の設定を保持する構造体。
  • mva_mode_e: 設定モード（VHOST_ALIAS_UNSET 等を含む）。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗する可能性がある（Apache の標準的なメモリ管理に従う）。
  • VHOST_ALIAS_UNSET 以外の値が設定されている場合、親の設定は無視され、子設定が完全に上書きされる。

保守担当者向けメモ

  • 本関数は mod_vhost_alias の設定継承ロジックの核となる。
  • パス文字列（doc_root, cgi_root）のコピーはポインタの代入によって行われるため、メモリの生存期間は apr_pool_t に依存する。
