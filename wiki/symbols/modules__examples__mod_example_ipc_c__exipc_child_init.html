exipc_child_init

定義

  • 種別: function
  • 定義箇所: modules/examples/mod_example_ipc.c:196

目的

子プロセス起動時に、親プロセスで作成されたグローバルミューテックスを再オープンし、子プロセス内での利用を可能にする。

シグネチャまたは形状

static void exipc_child_init(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 子プロセスのライフサイクルを管理するプール。
  • s: サーバー設定レコード。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • グローバル変数 exipc_mutex が既に初期化されていること。
  • apr_global_mutex_lockfile(exipc_mutex) が有効なロックファイルパスを返すこと。

アルゴリズムまたは振る舞い

1. apr_global_mutex_child_init を呼び出し、既存の exipc_mutex を子プロセス用に再初期化する。 2. 初期化に失敗した場合、ap_log_error でエラーログを出力し、exit(1) を実行してプロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_child_init を通じて Apache の child_init フックに登録されている（modules/examples/mod_example_ipc.c:343）。
  • **呼び出し先**:
  • apr_global_mutex_child_init (APR)
  • apr_global_mutex_lockfile (APR)
  • ap_log_error (Apache Core)
  • exit (標準ライブラリ)

関連データ構造

  • exipc_mutex: プロセス間で共有されるグローバルミューテックス（apr_global_mutex_t）。
  • exipc_mutex_type: ミューテックスの種類を識別する文字列。

失敗モードと境界条件

  • apr_global_mutex_child_init が APR_SUCCESS 以外を返した場合、プロセスは即座に終了する。これは、同期機構が利用できない状態で子プロセスが動作し、共有メモリの競合が発生することを防ぐための安全策である。

保守担当者向けメモ

  • 本関数は戻り値を返さないため、失敗時はプロセス終了という極端な挙動をとる。
  • exipc_mutex ポインタを再利用しているため、親プロセスでの設定が正しく引き継がれていることを前提としている。
