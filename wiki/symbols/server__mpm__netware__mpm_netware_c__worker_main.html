worker_main

定義

  • 種別: function
  • 定義箇所: server/mpm/netware/mpm_netware.c:336

目的

NetWare 環境における Apache ワーカースレッドのメインループを実行し、リスニングソケットの監視、クライアント接続の受け入れ、およびリクエスト処理を行う。

シグネチャまたは形状

void worker_main(void *arg)
  • arg: スレッド識別番号（int 型としてキャストして使用）。

入力と出力

  • **入力**: arg（スレッド番号）。グローバル変数 listenfds（監視対象ソケット）、pmain（メインプール）、ap_max_requests_per_child 等を参照。
  • **出力**: なし（戻り値は void）。副作用としてスコアボードの更新、接続の処理、およびスレッドの終了を行う。

前提条件と不変条件

  • 呼び出し時に pmain プールが初期化されていること。
  • die_now フラグが偽である間、スレッドは生存し続ける。
  • requests_this_child が ap_max_requests_per_child を超えた場合、または終了シグナルが検知された場合にスレッドは終了する。

アルゴリズムまたは振る舞い

1. **初期化**: スレッド固有のメモリプール（ptrans）とバケットアロケータを作成し、worker_thread_count をインクリメントする。 2. **メインループ**: die_now が真になるまで以下を繰り返す。

  • ap_max_requests_per_child に達していないか確認し、超過時は終了処理へ移行。
  • select() を使用して listenfds を監視し、接続要求を待機する。
  • 接続要求を検知した場合、apr_socket_accept() でソケットを受け入れる。
  • ap_run_create_connection を呼び出し、接続を確立する。
  • ap_process_connection でリクエストを処理し、ap_lingering_close で接続を閉じる。

    • 3. **終了処理**: clean_child_exit を呼び出し、リソースを解放してスレッドを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/netware/mpm_netware.c:553 において、NXContextAlloc の引数として関数ポインタが渡されている。
  • **呼び出し先**: clean_child_exit, ap_update_child_status_from_indexes, select, apr_socket_accept, ap_run_create_connection, ap_process_connection, ap_lingering_close 等。

関連データ構造

  • apr_pool_t *ptrans: リクエストごとのトランザクション用メモリプール。
  • ap_listen_rec: リスニングソケットのリスト。
  • fd_set main_fds: select() 用のファイル記述子セット。

失敗モードと境界条件

  • select() がエラーを返した場合、ログを出力し apr_thread_yield() を実行して再試行する。
  • apr_socket_accept() が EAGAIN 以外のエラーを返した場合、エラーの種類に応じてログ出力または clean_child_exit によるスレッド終了を行う。
  • ネットワーク層が停止している場合（ENETDOWN）、APEXIT_CHILDFATAL を指定してサーバー全体の終了を試みる。

保守担当者向けメモ

  • 本関数は NetWare の NKS スレッドとして動作するため、ブロッキング操作には注意が必要。
  • last_lr を保持することで、複数のリスニングソケット間での負荷分散（スターベーション防止）を行っている。
  • DBINFO_ON が定義されている場合、デバッグ用の統計情報が収集される。
