log_bytes_out

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_logio.c:108

目的

Apache HTTP Server のログフォーマットにおいて、現在の接続で送信された合計バイト数を文字列として取得するために使用されます。

シグネチャまたは形状

static const char *log_bytes_out(request_rec *r, char *a)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • a: ログフォーマットの引数（本関数では使用されません）。
  • **出力**:
  • 送信済みバイト数（cf->bytes_out）を文字列に変換したポインタを返します。メモリは r->pool から割り当てられます。

前提条件と不変条件

  • r->connection->conn_config に logio_module に関連付けられた logio_config_t 構造体が格納されている必要があります。
  • cf->bytes_out は、接続の生存期間中に蓄積された送信バイト数を示します。

アルゴリズムまたは振る舞い

1. ap_get_module_config を呼び出し、現在の接続設定から logio_config_t 構造体を取得します。 2. apr_off_t_toa を使用して、構造体内の bytes_out（送信バイト数）を文字列に変換します。 3. 変換された文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_logio.c:198 にて log_pfn_register を通じてログフォーマットトークン "O" に登録されています。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core API)
  • apr_off_t_toa (APR API)

関連データ構造

  • logio_config_t: 接続ごとの入出力統計を保持する構造体。
  • request_rec: リクエスト情報を保持する Apache のコア構造体。

失敗モードと境界条件

  • r->connection->conn_config が適切に初期化されていない場合、ap_get_module_config が NULL を返す可能性があります。その後の cf->bytes_out へのアクセスは未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は mod_log_config が提供するログフォーマット拡張機能の一部です。
  • 送信バイト数の計測自体は、外部モジュールから ap_logio_add_bytes_out が呼び出されることで行われます。本関数はその値を読み取る専用のアクセサです。
