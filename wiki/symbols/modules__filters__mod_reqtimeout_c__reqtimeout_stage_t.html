reqtimeout_stage_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_reqtimeout.c:42

目的

HTTPリクエスト処理の各段階（ハンドシェイク、ヘッダー読み取り、ボディ読み取り）におけるタイムアウト制限と、データ転送レートに基づく動的な期限延長ルールを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    int timeout;            /* タイムアウト値 (秒) */
    int max_timeout;        /* 最大タイムアウト値 (秒) */
    int min_rate;           /* 最小データ転送レート (バイト/秒) */
    apr_time_t rate_factor; /* 最小レート計算用のスケーリング係数 */
} reqtimeout_stage_t;

入力と出力

本構造体は設定値の保持および状態管理に使用されます。

  • **入力**: RequestReadTimeout ディレクティブにより設定された値が格納されます。
  • **出力**: フィルタ処理において、現在の接続に対するタイムアウト期限の算出や、受信データ量に応じた期限の延長計算に利用されます。

前提条件と不変条件

  • timeout および max_timeout は秒単位で指定されます。
  • min_rate が設定されている場合、受信データ量に応じて timeout が max_timeout を上限として動的に延長されます。

アルゴリズムまたは振る舞い

  • 本構造体は、reqtimeout_srv_cfg 内で各フェーズ（ハンドシェイク、ヘッダー、ボディ）ごとの設定として保持されます。
  • 接続ごとの状態管理（reqtimeout_con_cfg）において、現在適用中のステージを指すポインタとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_reqtimeout.c 内の各フック関数およびフィルタ関数（reqtimeout_filter 等）が、設定値の参照や現在のステージの特定のために本構造体を利用します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • reqtimeout_srv_cfg: サーバーごとの設定として、本構造体のインスタンス（handshake, header, body）を保持します。
  • reqtimeout_con_cfg: 接続ごとの現在の状態を保持し、cur_stage として本構造体へのポインタを保持します。

失敗モードと境界条件

  • min_rate が 0 の場合、データ転送レートに基づく動的な延長は行われません。
  • max_timeout が timeout より小さい場合、意図しないタイムアウト期限の短縮が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_reqtimeout の設定および状態管理の核となるため、フィールドの追加や変更を行う際は、extend_timeout 関数および reqtimeout_filter での計算ロジックへの影響を考慮してください。
