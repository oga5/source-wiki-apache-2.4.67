util_url_node_t

定義

  • 種別: struct
  • 定義箇所: modules/ldap/util_ldap_cache.h:93

目的

LDAP URL をキーとして、その URL に関連付けられた検索、比較、DN 比較の各キャッシュを保持するためのルートノード構造体です。

シグネチャまたは形状

typedef struct util_url_node_t {
    const char *url;
    util_ald_cache_t *search_cache;
    util_ald_cache_t *compare_cache;
    util_ald_cache_t *dn_compare_cache;
} util_url_node_t;

入力と出力

  • 入力: LDAP URL 文字列および、各キャッシュ種別に対応する util_ald_cache_t インスタンス。
  • 出力: URL ごとに階層化されたキャッシュ管理構造体。

前提条件と不変条件

  • 本構造体は、LDAP キャッシュ層のルートノードとして機能します。
  • 各キャッシュポインタ（search_cache, compare_cache, dn_compare_cache）は、util_ald_create_cache によって初期化された有効なキャッシュインスタンスを指す必要があります。

アルゴリズムまたは振る舞い

  • LDAP URL をキーとしてキャッシュを検索する際、本構造体がエントリポイントとなります。
  • 検索、比較、DN 比較の各操作は、本構造体が保持する個別のキャッシュインスタンスに対して委譲されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ldap/util_ldap.c において、キャッシュのフェッチやノード操作の文脈で参照されています（例: util_ald_cache_fetch の戻り値としてキャストされるケースなど）。
  • 呼び出し先: 保持する util_ald_cache_t 型の各キャッシュインスタンスに対して操作が行われます。

関連データ構造

  • util_ald_cache_t: キャッシュのメタデータと操作関数ポインタを保持する構造体。
  • util_search_node_t, util_compare_node_t, util_dn_compare_node_t: 各キャッシュインスタンス内に格納される具体的なデータノード。

失敗モードと境界条件

  • キャッシュメモリが不足している場合、util_ald_alloc を介したメモリ確保に失敗する可能性があります。
  • 共有メモリ（APR_HAS_SHARED_MEMORY）環境下では、ポインタの直接参照ではなく apr_rmm を介したオフセット管理が必要となるため、不適切なポインタ操作はメモリ破壊を招きます。

保守担当者向けメモ

  • 本構造体は階層的なキャッシュ構造の頂点に位置します。キャッシュのパージやメモリ解放を行う際は、本構造体が保持する各キャッシュインスタンスのライフサイクルと整合性を取る必要があります。
  • 共有メモリ環境での利用時は、ポインタの有効性に注意してください。
