ap_explode_recent_gmt

定義

  • 種別: function
  • 定義箇所: server/util_time.c:152

目的

キャッシュされた時刻情報を用いて、指定された時刻（apr_time_t）をグリニッジ標準時（GMT）の分解済み時刻構造体（apr_time_exp_t）へ効率的に変換します。

シグネチャまたは形状

apr_status_t ap_explode_recent_gmt(apr_time_exp_t *tm, apr_time_t t)

入力と出力

  • **入力**:
  • tm: 分解結果を格納する apr_time_exp_t 構造体へのポインタ。
  • t: 変換対象の時刻（apr_time_t）。
  • **出力**:
  • 戻り値: apr_status_t 型。成功時は APR_SUCCESS を返します。

前提条件と不変条件

  • AP_TIME_RECENT_THRESHOLD（デフォルト15秒）以内の時刻に対して最適化されています。
  • 内部的に exploded_cache_gmt キャッシュ構造体を使用します。

アルゴリズムまたは振る舞い

1. 内部関数 cached_explode を呼び出します。 2. その際、GMT用のキャッシュ exploded_cache_gmt を指定し、GMT変換フラグとして 1 を渡します。 3. キャッシュヒット時はメモリコピーにより高速に変換を行い、ミス時は標準の apr_time_exp_gmt に相当する処理へフォールバックします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_time.c:291（内部的な時刻処理にて利用）
  • **呼び出し先**:
  • cached_explode（server/util_time.c 内の内部関数）

関連データ構造

  • apr_time_exp_t: APRの時刻分解構造体。
  • exploded_cache_gmt: GMT変換用のキャッシュインスタンス。

失敗モードと境界条件

  • キャッシュの有効範囲（AP_TIME_RECENT_THRESHOLD）を超えた古い時刻が指定された場合、キャッシュミスが発生し、標準の時刻変換処理へフォールバックします。
  • 戻り値の apr_status_t により、変換処理の成否が報告されます。

保守担当者向けメモ

  • 本関数はロックフリーなキャッシュ機構を利用しています。高負荷環境下でのパフォーマンス最適化を目的としているため、ロジック変更時は cached_explode の整合性検証メカニズムを維持してください。
