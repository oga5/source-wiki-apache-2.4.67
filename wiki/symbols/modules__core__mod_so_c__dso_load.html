dso_load

定義

  • 種別: function
  • 定義箇所: modules/core/mod_so.c:146

目的

Apache HTTP Server において、共有オブジェクト（DSO）ファイルを指定されたパスからロードし、ハンドルを取得する。サーバールートからの相対パス解決および、パス指定がない場合の検索パス利用をサポートする。

シグネチャまたは形状

static const char *dso_load(cmd_parms *cmd, apr_dso_handle_t **modhandlep,
                            const char *filename, const char **used_filename)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • modhandlep: ロードされた DSO ハンドルを格納するポインタのポインタ。
  • filename: ロード対象のファイル名またはパス。
  • used_filename: 実際にロードを試みたパスを格納する出力用ポインタ。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • cmd->temp_pool および cmd->pool が有効であること。
  • filename にパス区切り文字が含まれない場合、システム標準の検索パス（dlopen 等）が利用される。

アルゴリズムまたは振る舞い

1. ap_server_root_relative を使用し、filename をサーバールートからの絶対パスに変換する。 2. パス区切り文字が含まれない場合は、検索パスを利用するフラグ（retry）を立てる。 3. 絶対パスへの変換に失敗し、かつ retry が不要な場合はエラーを返す。 4. 変換されたパスで apr_dso_load を試行し、成功すれば終了する。 5. 失敗した場合、retry フラグが有効であれば、元の filename を使用して再度 apr_dso_load を試行する。 6. いずれの試行も失敗した場合は、apr_dso_error を用いてプラットフォーム固有のエラーメッセージを取得し、エラー文字列として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/core/mod_so.c:263 (load_module 内)
  • modules/core/mod_so.c:332 (load_file 内)
  • **呼び出し先**:
  • ap_server_root_relative
  • apr_dso_load
  • apr_dso_error
  • apr_psprintf / apr_pstrcat

関連データ構造

  • apr_dso_handle_t: DSO ハンドルを保持する APR の構造体。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • **パス解決失敗**: サーバールートからの相対パス解決ができない場合、エラーを返す。
  • **ロード失敗**: apr_dso_load が APR_SUCCESS を返さない場合、プラットフォーム固有のエラーメッセージを生成する。
  • **メモリ不足**: APR のメモリプール操作が失敗した場合の挙動は APR の仕様に依存する。

保守担当者向けメモ

  • 本関数は apr_dso_load のラッパーであり、エラーハンドリングとパス解決の抽象化を担う。
  • used_filename は呼び出し元でロード成功の確認に使用されるため、必ず適切に更新される必要がある。
