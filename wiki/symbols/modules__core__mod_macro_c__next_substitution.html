next_substitution

定義

  • 種別: function
  • 定義箇所: modules/core/mod_macro.c:384

目的

文字列バッファ内から、指定された引数リストに含まれるいずれかの文字列を検索し、最初に出現する位置を特定します。

シグネチャまたは形状

char *next_substitution(const char *buf, const apr_array_header_t *args, int *whichone);

入力と出力

  • **入力**:
  • buf: 検索対象の文字列バッファ。
  • args: 検索対象となる文字列の配列（apr_array_header_t）。
  • whichone: 検出された引数のインデックスを格納するための出力用ポインタ。
  • **出力**:
  • 戻り値: buf 内で最初に見つかった引数文字列へのポインタ。見つからない場合は NULL。
  • whichone: 見つかった引数の args 内でのインデックス番号。

前提条件と不変条件

  • args は有効な apr_array_header_t 構造体を指している必要があります。
  • whichone は書き込み可能な int 型のメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. args 配列内の各要素を順次走査します。 2. ap_strstr を使用して、現在のバッファ buf 内に各引数文字列が含まれているかを確認します。 3. 複数の引数がバッファ内に存在する場合、以下の優先順位で「最初」の出現を決定します。

  • バッファ内の出現位置がより前であること。
  • 出現位置が同じ場合は、文字列長がより長いこと（最長一致）。

    • 4. 条件を満たす最も早い出現位置を chosen に保持し、そのインデックスを whichone に書き込みます。 5. 全ての引数を走査した後、特定された位置のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_macro.c:427 (substitute_macro_args 関数内)
  • **呼び出し先**: ap_strstr (Apache APR ユーティリティ関数)

関連データ構造

  • apr_array_header_t: 検索対象の引数リストを保持する APR 配列構造体。

失敗モードと境界条件

  • buf 内に args のいずれの文字列も含まれない場合、NULL を返します。
  • args が空の場合、ループは実行されず NULL を返します。

保守担当者向けメモ

  • 本関数はマクロ展開時の引数置換処理において、置換対象を特定するために使用されます。
  • ap_strstr を利用しているため、大文字小文字を区別した検索が行われます。
  • 戻り値が NULL になるまでループさせることで、バッファ内の全ての置換対象を順次処理する設計となっています。
