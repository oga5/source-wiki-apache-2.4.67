merge_request_dir_config

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:339

目的

Apache HTTP Server のディレクトリ単位の設定構造体（request_dir_conf）をマージし、親ディレクトリの設定と現在の設定を統合する。

シグネチャまたは形状

static void *merge_request_dir_config(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: 設定の割り当てに使用するメモリプール。
  • basev: 親ディレクトリの設定構造体（request_dir_conf *）。
  • addv: 現在のディレクトリで追加された設定構造体（request_dir_conf *）。
  • **出力**:
  • マージされた新しい request_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は request_dir_conf 型としてキャスト可能であること。
  • apr_pcalloc により、新しい設定構造体はゼロ初期化された状態で確保される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい request_dir_conf 構造体をメモリプール p 上に確保する。 2. add->keep_body_set が設定されている（0以外）場合、現在の設定 add->keep_body を採用し、そうでない場合は親の設定 base->keep_body を継承する。 3. keep_body_set フラグ自体も、親と現在の設定の論理和（OR）をとることで、設定の有効状態を伝播させる。 4. マージ済みの構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の設定マージ機構（modules/filters/mod_request.c:388 にて command_rec 構造体を通じて登録）。
  • **呼び出し先**: 該当なし。

関連データ構造

  • request_dir_conf: mod_request におけるディレクトリごとの設定を保持する構造体。keep_body（保持サイズ制限）および keep_body_set（設定の有無）をメンバに持つ。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が異常終了する可能性がある。
  • basev や addv が NULL の場合の挙動については、本関数内では明示的なチェックを行っていない。

保守担当者向けメモ

  • 本関数は KeptBodySize ディレクティブの設定継承を制御する。設定の追加・変更を行う際は、request_dir_conf 構造体のメンバと本関数のマージロジックを同期させる必要がある。
