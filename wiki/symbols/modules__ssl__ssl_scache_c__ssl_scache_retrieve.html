ssl_scache_retrieve

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_scache.c:147

目的

キャッシュプロバイダーから SSL セッションデータを取得し、OpenSSL の SSL_SESSION 構造体として復元する。

シグネチャまたは形状

SSL_SESSION *ssl_scache_retrieve(server_rec *s, IDCONST UCHAR *id, int idlen, apr_pool_t *p)

入力と出力

  • **入力**:
  • s: サーバー設定レコード。
  • id: セッション識別子。
  • idlen: セッション識別子の長さ。
  • p: メモリ割り当て用のプール。
  • **出力**:
  • 成功時: 復元された SSL_SESSION 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • mc->sesscache が初期化されていること。
  • MODSSL_SESSION_MAX_DER バイト以内のセッションデータであること。

アルゴリズムまたは振る舞い

1. AP_SOCACHE_FLAG_NOTMPSAFE フラグが設定されている場合、ssl_mutex_on で排他制御を開始する。 2. キャッシュプロバイダーの retrieve メソッドを呼び出し、バイナリ形式（DER）でセッションデータを取得する。 3. 排他制御を開始した場合は ssl_mutex_off で解除する。 4. 取得に失敗した場合は NULL を返す。 5. 成功した場合、d2i_SSL_SESSION を使用してバイナリデータを SSL_SESSION 構造体にデシリアライズして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_kernel.c:1964
  • **呼び出し先**:
  • mc->sesscache->retrieve (プロバイダー固有の取得関数)
  • ssl_mutex_on / ssl_mutex_off (排他制御)
  • d2i_SSL_SESSION (OpenSSL API)

関連データ構造

  • SSLModConfigRec: キャッシュプロバイダーおよび設定情報を保持。
  • SSL_SESSION: OpenSSL のセッション構造体。

失敗モードと境界条件

  • キャッシュプロバイダーが APR_SUCCESS 以外のステータスを返した場合、関数は NULL を返す。
  • セッションデータが MODSSL_SESSION_MAX_DER を超える場合、取得は失敗する。

保守担当者向けメモ

  • 本関数は socache プロバイダーの抽象化レイヤーであり、実際の保存先ロジックはプロバイダー側に依存する。
  • スレッドセーフでないプロバイダーを使用する場合、ssl_mutex による排他制御が必須となる。
