socache_rd_status

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_redis.c:299

目的

Redis サーバーの統計情報を取得し、Apache の mod_status ページへ出力する。

シグネチャまたは形状

static void socache_rd_status(ap_socache_instance_t *ctx, request_rec *r, int flags)

入力と出力

  • ctx: Redis インスタンスのコンテキスト（ap_socache_instance_t）。
  • r: 現在のリクエストレコード。
  • flags: 出力形式を制御するフラグ（AP_STATUS_SHORT の有無で HTML タグの有無や改行コードが切り替わる）。
  • 出力: ap_rprintf を通じて、接続中の全 Redis サーバーの稼働状況および統計情報（メモリ、CPU、ヒット率、ネットワーク統計等）をレスポンスストリームに書き出す。

前提条件と不変条件

  • ctx->rc が有効な apr_redis_t ハンドルを保持していること。
  • apr_redis_stats が成功すること。

アルゴリズムまたは振る舞い

1. ctx->rc 内の全サーバー（live_servers）をループで走査する。 2. 各サーバーの接続状態（Up/Down）を表示する。 3. apr_redis_stats を呼び出し、サーバーから統計情報を取得する。 4. flags に応じて、HTML 形式またはプレーンテキスト形式で統計情報を整形し、ap_rprintf で出力する。 5. 統計情報には、バージョン、PID、稼働時間、クライアント数、メモリ使用量、CPU 使用率、接続数、キャッシュヒット/ミス率、ネットワーク入出力、および Redis のロール（master/slave）が含まれる。

呼び出し元と呼び出し先

  • 呼び出し元: modules/cache/mod_socache_redis.c:396（ap_socache_provider_t 構造体を通じて mod_status から間接的に呼び出される）。
  • 呼び出し先: apr_redis_stats（統計情報の取得）、ap_rprintf / ap_rputs（出力）。

関連データ構造

  • ap_socache_instance_t: Redis インスタンスの管理構造体。
  • apr_redis_t: Redis クライアントハンドル。
  • apr_redis_server_t: 個別の Redis サーバー情報。
  • apr_redis_stats_t: Redis サーバーから取得した統計データ。

失敗モードと境界条件

  • apr_redis_stats が APR_SUCCESS を返さない場合、そのサーバーの統計出力はスキップされ、次のサーバーの処理へ移行する。
  • flags に AP_STATUS_SHORT が含まれる場合、HTML タグ（<b>, <i>, <br />）は出力されず、簡潔なテキスト形式となる。

保守担当者向けメモ

  • 本関数は mod_status の出力の一部として機能するため、出力フォーマットの変更は mod_status の表示レイアウトに影響を与える。
  • apr_redis_stats が返す構造体のフィールド（major, minor, patch 等）は apr_redis ライブラリの仕様に依存する。
