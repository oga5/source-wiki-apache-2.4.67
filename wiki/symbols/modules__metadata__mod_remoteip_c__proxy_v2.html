proxy_v2

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_remoteip.c:106

目的

PROXY プロトコル v2 のバイナリヘッダー構造を表現するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    apr_byte_t  sig[12];
    apr_byte_t  ver_cmd;
    apr_byte_t  fam;
    apr_uint16_t len;
    proxy_v2_addr addr;
} proxy_v2;

入力と出力

  • **入力**: ネットワークストリームから読み込まれたバイナリデータ。
  • **出力**: 構造体の各フィールドに格納されたプロトコルバージョン、コマンド、アドレスファミリー、およびアドレス情報。

前提条件と不変条件

  • sig フィールドには、PROXY プロトコル v2 で規定された 12 バイトの固定シグネチャ（0D 0A 0D 0A 00 0D 0A 51 55 49 54 0A）が格納されている必要があります。
  • len フィールドは、ヘッダーの残りの部分（addr 以降）のバイト数を表します。

アルゴリズムまたは振る舞い

  • mod_remoteip において、ストリームの先頭から PROXY プロトコル v2 のヘッダーを解析するために使用されます。
  • remoteip_input_filter 内で、受信したデータが本構造体のサイズを満たしているかを確認し、バイナリデータをキャストして解析します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_remoteip.c 内の remoteip_input_filter 関数（1153行目付近）にて、バッファサイズの判定に使用。
  • **呼び出し先**: 不明（本構造体はデータ保持用であり、関数呼び出しは含まれません）。

関連データ構造

  • proxy_v2_addr: proxy_v2 構造体の一部として、送信元および宛先のアドレス情報を保持します。
  • proxy_header: PROXY プロトコル v1 および v2 を包含する共用体。

失敗モードと境界条件

  • 受信データが sizeof(proxy_v2) に満たない場合、remoteip_input_filter は HDR_NEED_MORE を用いてデータの追加入力を待機します。
  • 不正なシグネチャが sig フィールドに含まれる場合、プロトコル解析は失敗します。

保守担当者向けメモ

  • 本構造体は PROXY プロトコル v2 の仕様に厳密に従っています。フィールドの追加や変更を行う際は、プロトコル仕様書と整合性が取れているか確認してください。
  • len フィールドのバイトオーダー（ネットワークバイトオーダー）の扱いに注意してください。
