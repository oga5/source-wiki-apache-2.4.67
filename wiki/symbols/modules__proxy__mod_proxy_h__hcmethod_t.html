hcmethod_t

定義

  • 種別: enum
  • 定義箇所: modules/proxy/mod_proxy.h:83

目的

プロキシバックエンドサーバーに対するヘルスチェックの手法を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    NONE, TCP, OPTIONS, HEAD, GET, CPING, PROVIDER, OPTIONS11, HEAD11, GET11, EOT
} hcmethod_t;

入力と出力

  • 入力: なし（列挙型定数）
  • 出力: ap_proxy_show_hcmethod(hcmethod_t method) 関数により、対応する手法の名称（文字列）が返されます。

前提条件と不変条件

  • EOT は列挙の終端を示します。
  • OPTIONS11, HEAD11, GET11 は HTTP/1.1 プロトコルを用いたヘルスチェックを指します。

アルゴリズムまたは振る舞い

各列挙値は、バックエンドの生存確認を行うための具体的なプロトコルレベルの操作（TCP接続、HTTPメソッドの送信等）に対応しています。

呼び出し元と呼び出し先

  • 呼び出し元: mod_proxy_hcheck.c および mod_proxy.h 内の構造体（proxy_worker 等）で、ヘルスチェック手法の保持に使用されています。
  • 呼び出し先: proxy_util.c 内の ap_proxy_show_hcmethod 関数がこの型を引数として受け取ります。

関連データ構造

  • proxy_worker: ヘルスチェック手法を保持するフィールド method を持ちます。
  • proxy_hcmethods_t: 関連するヘルスチェック手法の定義群。

失敗モードと境界条件

  • NONE が指定された場合、ヘルスチェックは実行されません。
  • 範囲外の列挙値が渡された場合の挙動については、ソースコード内に明示的なガード条件の記述はありません。

保守担当者向けメモ

  • 新しいヘルスチェック手法を追加する場合は、本列挙型に値を追加し、ap_proxy_show_hcmethod 関数内の文字列変換処理を更新する必要があります。
  • include/ap_mmn.h に記録されている通り、本シンボルはバージョンアップに伴い拡張される可能性があります。
