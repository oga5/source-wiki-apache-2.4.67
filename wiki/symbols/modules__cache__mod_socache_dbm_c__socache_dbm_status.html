socache_dbm_status

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:586

目的

DBM ベースの共有オブジェクトキャッシュ（socache）の現在の統計情報（エントリ数、合計サイズ、平均エントリサイズ）を収集し、HTTP レスポンスとして出力する。

シグネチャまたは形状

static void socache_dbm_status(ap_socache_instance_t *ctx, request_rec *r, int flags)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンスのコンテキスト（データファイルパス等を含む）。
  • r: 現在のリクエストレコード。
  • flags: 出力形式を制御するフラグ（AP_STATUS_SHORT の有無で HTML 形式かプレーンテキスト形式かを切り替える）。
  • **出力**: なし（ap_rprintf 等を通じてクライアントへ直接書き出す）。

前提条件と不変条件

  • ctx->pool は、本関数内で apr_pool_clear によりクリアされる。
  • DBM ファイルは APR_DBM_RWCREATE モードでオープンされる。
  • スレッドセーフではないため、呼び出し側での排他制御が前提となる。

アルゴリズムまたは振る舞い

1. ctx->pool を初期化する。 2. apr_dbm_open（または apr_dbm_open2）を使用して DBM ファイルを開く。失敗時はエラーログを出力して終了する。 3. apr_dbm_firstkey および apr_dbm_nextkey を用いて全エントリを走査する。 4. 各エントリに対して apr_dbm_fetch を実行し、データサイズを累積してエントリ数をカウントする。 5. apr_dbm_close でファイルを閉じる。 6. flags に応じて、HTML またはプレーンテキスト形式で統計情報をレスポンスとして書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dbm.c:749（socache_dbm 構造体内の関数ポインタとして参照されている）。
  • **呼び出し先**:
  • apr_pool_clear, apr_dbm_get_driver, apr_dbm_open2, apr_dbm_open, apr_dbm_firstkey, apr_dbm_nextkey, apr_dbm_fetch, apr_dbm_close
  • ap_log_rerror, ap_rprintf, ap_rputs

関連データ構造

  • ap_socache_instance_t: キャッシュのコンテキストを保持。
  • apr_dbm_t: DBM ファイルのハンドル。
  • apr_datum_t: DBM のキーおよび値を保持する構造体。

失敗モードと境界条件

  • DBM ファイルのオープンに失敗した場合、エラーログを出力して処理を中断する。
  • エントリが 0 件の場合、平均サイズは 0 として計算される。
  • apr_dbm_firstkey や apr_dbm_fetch の戻り値チェックについては、ソースコード内に「TBD（要検討）」とのコメントがある。

保守担当者向けメモ

  • 本関数は AP_SOCACHE_FLAG_NOTMPSAFE を前提としており、呼び出し側での適切なロック管理が不可欠である。
  • APU_MAJOR_VERSION 等の条件分岐により、APR-util のバージョンに応じた DBM ドライバの取得方法を切り替えている。
