set_secure_listener

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:467

目的

Apache の設定ファイルで指定された SecureListen ディレクティブを解析し、NetWare 環境におけるセキュアなリスニングポートを構成・登録する。

シグネチャまたは形状

static const char *set_secure_listener(cmd_parms *cmd, void *dummy,
                                       const char *ips, const char* key,
                                       const char* mutual)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • ips: IPアドレスとポートの文字列（例: "127.0.0.1:443"）。
  • key: 証明書キー名。
  • mutual: 相互認証の有無を示すフラグ（指定があれば有効）。
  • **出力**: 成功時は NULL、エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • GLOBAL_ONLY コンテキストで呼び出される必要がある。
  • ips 引数は IP:PORT 形式、または PORT のみ（全インターフェース）であること。
  • ポート番号は数値である必要がある。

アルゴリズムまたは振る舞い

1. **コンテキスト検証**: ap_check_cmd_context を使用し、グローバル設定以外での使用を拒否する。 2. **引数解析**: ips 文字列から IP アドレスとポート番号を分離する。IP が省略された場合は 0.0.0.0 を使用する。 3. **既存リスナーの再利用**: nw_old_listeners リストを走査し、指定されたアドレスとポートが既に存在する場合は、そのレコードを ap_listeners に移動して再利用する。 4. **設定テーブルへの登録**: sc->sltable にポートとアドレスのペアを追加する。 5. **新規リスナーの作成**: 再利用されなかった場合、seclisten_rec 構造体を確保し、ap_seclisteners リストの先頭に挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_nw_ssl.c:1245 (AP_INIT_TAKE23 マクロ経由で SecureListen ディレクティブから呼び出される)。
  • **呼び出し先**: get_nwssl_cfg, ap_check_cmd_context, apr_pcalloc, parse_addr, apr_pstrdup, apr_table_add。

関連データ構造

  • NWSSLSrvConfigRec: サーバーごとの SSL 設定を保持。
  • seclisten_rec: セキュアリスナーの定義を保持するリンクリスト。
  • ap_listen_rec: Apache 標準のリスナー構造体。

失敗モードと境界条件

  • IP アドレスの指定が不完全（例: : のみ）な場合、エラーメッセージを返す。
  • ポート番号が数値でない場合、「Port must be numeric」を返す。
  • コンテキストがグローバルでない場合、ap_check_cmd_context が返すエラーを伝播する。

保守担当者向けメモ

  • 本関数は nw_old_listeners を操作してリスナーの再利用を試みるため、Apache の起動シーケンスにおけるリスナー初期化タイミングと密接に関連している。
  • seclisten_rec の確保には cmd->server->process->pool を使用しており、メモリ管理は Apache のプール機構に依存している。
