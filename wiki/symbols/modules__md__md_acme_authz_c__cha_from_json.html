cha_from_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_authz.c:175

目的

ACME サーバーから取得した JSON オブジェクトを解析し、チャレンジ情報を保持する md_acme_authz_cha_t 構造体を生成・初期化します。

シグネチャまたは形状

static md_acme_authz_cha_t *cha_from_json(apr_pool_t *p, size_t index, md_json_t *json)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • index: チャレンジのインデックス番号。
  • json: 解析対象の md_json_t オブジェクト。
  • **出力**:
  • 初期化済みの md_acme_authz_cha_t 構造体へのポインタ。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • json は ACME チャレンジの JSON 表現を含んでいること。
  • 生成された構造体の各フィールド（type, uri, token, key_authz）は、指定されたプール p 上に複製されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して md_acme_authz_cha_t 構造体をゼロ初期化し、メモリを確保します。 2. 引数 index を構造体の index フィールドに設定します。 3. JSON から MD_KEY_TYPE を取得し、type フィールドに設定します。 4. URI の取得において、ACME バージョンを判定します：

  • MD_KEY_URL キーが存在する場合（ACMEv2）、その値を uri に設定します。
  • 存在しない場合（ACMEv1）、MD_KEY_URI キーから値を取得し uri に設定します。

    • 5. JSON から MD_KEY_TOKEN および MD_KEY_KEYAUTHZ を取得し、それぞれ対応するフィールドに設定します。 6. 初期化された構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_authz.c:598 にて呼び出されています。
  • **呼び出し先**: apr_pcalloc、md_json_dups、md_json_has_key を呼び出します。

関連データ構造

  • md_acme_authz_cha_t: チャレンジ情報を保持する構造体。
  • md_json_t: ACME サーバーとの通信に使用される JSON 表現。

失敗モードと境界条件

  • json に必要なキー（type, token 等）が含まれていない場合、md_json_dups は NULL を返す可能性があります。
  • メモリ不足時には apr_pcalloc がアボートする可能性があります（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は ACMEv1 と ACMEv2 の両方の URI 形式に対応しています。将来的なプロトコル変更でキー名が変更された場合は、この判定ロジックの更新が必要です。
  • md_json_dups を使用しているため、返される構造体の各文字列フィールドは、呼び出し元が指定したプール p の生存期間に依存します。
