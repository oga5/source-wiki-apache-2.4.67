ap_load_dll_func

定義

  • 種別: function
  • 定義箇所: os/win32/util_win32.c:92

目的

指定されたWindowsシステムDLLを遅延ロードし、そのDLL内の関数アドレスを名前または序数（ordinal）で解決して返却する。

シグネチャまたは形状

FARPROC ap_load_dll_func(ap_dlltoken_e fnLib, char* fnName, int ordinal)

入力と出力

  • **入力**:
  • fnLib: ロード対象のDLLを識別する ap_dlltoken_e 型のトークン。
  • fnName: 解決する関数名（文字列）。
  • ordinal: 解決する関数の序数。0以外の場合、fnName よりも優先される。
  • **出力**:
  • 成功時: 解決された関数のアドレス（FARPROC）。
  • 失敗時: NULL。

前提条件と不変条件

  • lateDllHandle 配列が、DLLのロード状態を保持するキャッシュとして機能している。
  • lateDllName 配列（外部定義）に、各 ap_dlltoken_e に対応するDLL名が格納されている必要がある。

アルゴリズムまたは振る舞い

1. lateDllHandle[fnLib] を確認し、未ロードであれば LoadLibrary を呼び出してDLLをロードし、ハンドルをキャッシュする。 2. LoadLibrary が失敗した場合は NULL を返して終了する。 3. ordinal が0以外の場合、GetProcAddress を使用して序数から関数アドレスを解決する。 4. ordinal が0の場合、fnName を使用して関数名から関数アドレスを解決する。 5. 解決されたアドレスを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: os/win32/os.h 内の AP_DECLARE_LATE_DLL_FUNC マクロによって生成されるラッパー関数から呼び出される。
  • **呼び出し先**:
  • LoadLibrary (Windows API)
  • GetProcAddress (Windows API)

関連データ構造

  • ap_dlltoken_e: ロード対象DLLの列挙型。
  • lateDllHandle: ロード済みDLLハンドルを保持する内部キャッシュ配列。

失敗モードと境界条件

  • LoadLibrary が失敗した場合、関数は NULL を返す。
  • GetProcAddress が指定された関数名または序数を見つけられない場合、NULL を返す。
  • ordinal が0の場合に fnName が NULL である場合の挙動は、GetProcAddress の仕様に依存する。

保守担当者向けメモ

  • 本関数はAPR側の類似機能と重複しており、将来的にAPRへ統合される可能性がある。
  • lateDllHandle はプロセス内で共有されるため、マルチスレッド環境下での初期化タイミングについては、呼び出し側のマクロ実装（os.h）の同期処理に依存する。
