sig_term

定義

  • server/mpm/netware/mpm_netware.c:270

目的

NetWare環境におけるサーバー終了シグナル（SIGTERM, SIGABRT）を処理し、サーバーのシャットダウンプロセスを開始する。

シグネチャまたは形状

static void sig_term(int sig)

入力と出力

  • **入力**: int sig（受信したシグナル番号）
  • **出力**: なし（void）

前提条件と不変条件

  • shutdown_pending 変数がサーバーの終了状態を管理するフラグとして機能する。
  • wait_to_finish 変数がスレッド終了待ちの条件として使用される。

アルゴリズムまたは振る舞い

1. shutdown_pending が既に 1 の場合は、二重のシャットダウン要求と判断して即座に終了する。 2. shutdown_pending を 1 に設定し、シャットダウンを開始する。 3. wait_to_finish が真である間、apr_thread_yield() を呼び出してスレッドの終了を待機するループを実行する。 4. 処理の過程で DBPRINT0 マクロを用いてデバッグメッセージを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/netware/mpm_netware.c 内の apr_signal(SIGTERM, sig_term) および apr_signal(SIGABRT, sig_term) によりシグナルハンドラとして登録される。
  • ※注: 他のソースファイル（os/unix/unixd.c 等）にも同名のシンボルへの参照が見られるが、これらは別プラットフォームのモジュールであり、本関数との直接的な呼び出し関係は確認できない。
  • **呼び出し先**:
  • apr_thread_yield()
  • DBPRINT0 (マクロ)

関連データ構造

  • shutdown_pending (グローバル変数)
  • wait_to_finish (グローバル変数)

失敗モードと境界条件

  • 既にシャットダウンが進行中（shutdown_pending == 1）の場合、関数は副作用なしに早期リターンする。
  • wait_to_finish が適切に更新されない場合、ループが終了せずデッドロックする可能性がある。

保守担当者向けメモ

  • 本関数は NetWare MPM の終了シーケンスの起点となる。
  • wait_to_finish の状態管理がサーバーの正常終了に直結するため、スレッド管理ロジックと整合性が取れていることを確認すること。
