filter_protocol

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_filter.c:331

目的

Apache HTTP Server の設定ディレクティブ FilterProtocol を処理し、指定されたフィルタまたはフィルタプロバイダに対して、HTTP プロトコル上の制約や挙動（キャッシュ、バイトレンジ、プロキシ変換など）を定義するフラグを設定します。

シグネチャまたは形状

static const char *filter_protocol(cmd_parms *cmd, void *CFG, const char *fname,
                                   const char *pname, const char *proto)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • CFG: モジュールの設定構造体 (mod_filter_cfg)。
  • fname: 対象フィルタ名。
  • pname: 対象プロバイダ名（省略可能）。
  • proto: プロトコルフラグの文字列（change=yes, byteranges=no 等）。
  • **出力**: 成功時は NULL、エラー時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • fname で指定されたフィルタが cfg->live_filters に登録されている必要があります。
  • pname が指定された場合、そのフィルタに関連付けられたプロバイダリスト内に該当するプロバイダが存在する必要があります。

アルゴリズムまたは振る舞い

1. cfg->live_filters から fname に一致するフィルタを検索します。 2. proto 引数が NULL の場合、pname をプロトコル文字列として扱い、フィルタ全体にフラグを適用します。 3. pname が指定されている場合、フィルタのプロバイダリストを走査し、一致するプロバイダを特定してそのプロバイダ固有のフラグとして適用します。 4. proto 文字列を ;, ,, , \t を区切り文字としてトークン化し、以下のフラグをビット演算で設定します：

  • change=yes: AP_FILTER_PROTO_CHANGE | AP_FILTER_PROTO_CHANGE_LENGTH
  • change=no: 上記フラグを解除
  • change=1:1: AP_FILTER_PROTO_CHANGE
  • byteranges=no: AP_FILTER_PROTO_NO_BYTERANGE
  • proxy=no: AP_FILTER_PROTO_NO_PROXY
  • proxy=transform: AP_FILTER_PROTO_TRANSFORM
  • cache=no: AP_FILTER_PROTO_NO_CACHE

    • 5. 最終的に特定された対象（フィルタまたはプロバイダ）の proto_flags メンバを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_filter.c:753 において AP_INIT_TAKE23 マクロ経由で FilterProtocol ディレクティブとして登録されています。
  • **呼び出し先**: apr_hash_get（フィルタ検索）、apr_strtok（文字列解析）、strcasecmp（文字列比較）、apr_pstrdup（メモリ確保）等の APR API を利用しています。

関連データ構造

  • mod_filter_cfg: モジュール全体の設定を保持。
  • ap_filter_rec_t: フィルタの定義情報を保持。
  • ap_filter_provider_t: フィルタプロバイダの情報を保持。

失敗モードと境界条件

  • 指定したフィルタ名が存在しない場合、エラーメッセージを返します。
  • 指定したプロバイダ名がフィルタ内に存在しない場合、エラーメッセージを返します。

保守担当者向けメモ

  • 本関数は設定フェーズでのみ実行されます。
  • proto_flags は実行時の filter_harness におけるプロトコル整合性チェックに使用されます。
  • cmd->temp_pool を使用して文字列のコピーを行っているため、設定読み込み中のメモリ管理は適切に行われています。
