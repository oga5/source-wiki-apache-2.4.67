core_output_filter_ctx_t

定義

  • 種別: struct
  • 定義箇所: include/http_core.h:787

目的

Apache HTTP Server のコア出力フィルタ処理において、フィルタチェーンの実行状態や一時的なバッファリング情報を保持するためのコンテキスト構造体です。

シグネチャまたは形状

ソースコード上では不透明な構造体として宣言されています（typedef struct core_output_filter_ctx core_output_filter_ctx_t;）。具体的なメンバ構成は server/core_filters.c 等の実装側に依存します。

入力と出力

  • **入力**: フィルタチェーンを通過するバケットブリゲード（apr_bucket_brigade）の処理状態。
  • **出力**: フィルタ処理後のバケットブリゲード、および接続状態の更新。

前提条件と不変条件

  • server/core_filters.c 内のフィルタ関数において、net->out_ctx を通じてアクセスされます。
  • フィルタのライフサイクルを通じて、リクエストまたは接続のプール（pool）に関連付けられたメモリ管理が行われます。

アルゴリズムまたは振る舞い

  • 出力フィルタの実行中に、一時的なフラッシュ用バケット（tmp_flush_bb）の保持や、フィルタチェーンの進捗管理に使用されます。
  • 接続ごとの出力フィルタコンテキストとして、net->out_ctx にキャストして格納・参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core_filters.c 内の各出力フィルタ関数（ap_core_output_filter 等）が、コンテキストの取得および更新を行います。
  • **呼び出し先**: フィルタ処理の過程で、APR（Apache Portable Runtime）のバケット操作関数や、下位のネットワークフィルタへデータを渡します。

関連データ構造

  • core_net_rec: ネットワーク接続情報を保持する構造体。out_ctx メンバとして本構造体を保持します。
  • apr_bucket_brigade: フィルタ間で受け渡されるデータストリーム。

失敗モードと境界条件

  • メモリ不足時、プールからの割り当てに失敗する可能性があります。
  • フィルタチェーンの初期化前にアクセスした場合、NULL ポインタ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は include/ap_mmn.h で定義されるモジュールバイナリ互換性（MMN）の対象であり、メンバの変更はバイナリ互換性に影響を与える可能性があります。
  • 構造体の詳細定義は include/http_core.h ではなく server/core_filters.c に存在するため、メンバの追加・変更時は実装側の整合性を確認してください。
