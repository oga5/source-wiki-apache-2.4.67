shm_cleanup_wrapper

定義

  • 種別: function
  • 定義箇所: modules/lua/mod_lua.c:89

目的

Apache HTTP Server の終了時またはプール破棄時に、Lua モジュールが使用する共有メモリ（lua_ivm_shm）を安全に破棄するためのクリーンアップ関数。

シグネチャまたは形状

apr_status_t shm_cleanup_wrapper(void *unused)

入力と出力

  • **入力**: void *unused（apr_pool_cleanup_register から渡される引数。本関数内では使用されない）
  • **出力**: apr_status_t 型のステータスコード。共有メモリが破棄された場合は apr_shm_destroy の戻り値、共有メモリが存在しない場合は OK を返す。

前提条件と不変条件

  • グローバル変数 lua_ivm_shm が初期化済みであるか、あるいは NULL であることが前提。
  • 本関数は apr_pool_cleanup_register を介してプールに関連付けられることを想定している。

アルゴリズムまたは振る舞い

1. グローバル変数 lua_ivm_shm が非 NULL かどうかを確認する。 2. lua_ivm_shm が存在する場合、apr_shm_destroy を呼び出して共有メモリを破棄し、その結果を返す。 3. lua_ivm_shm が NULL の場合、何もしないで OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/mod_lua.c:2059（apr_pool_cleanup_register への登録）
  • modules/examples/mod_example_ipc.c:186（外部モジュールからの参照として索引化されているが、意図的な利用か確認が必要）
  • **呼び出し先**: apr_shm_destroy（APR ライブラリ関数）

関連データ構造

  • lua_ivm_shm: mod_lua.c 内で管理される共有メモリセグメントへのポインタ。

失敗モードと境界条件

  • lua_ivm_shm が既に破棄されている、または初期化されていない場合、関数は安全に OK を返す。
  • apr_shm_destroy が失敗した場合、そのエラーコードがそのまま呼び出し元に伝播する。

保守担当者向けメモ

  • 本関数は apr_pool_cleanup_register に登録されるコールバックであるため、直接呼び出すのではなく、プール管理のライフサイクルに委ねること。
  • mod_example_ipc.c からの参照が検出されているが、これが意図的な依存関係か、あるいはコードのコピー＆ペーストによる誤った参照かを確認すること。
