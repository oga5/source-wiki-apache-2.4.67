h2_protocol_is_acceptable_c1

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_protocol.c:432

目的

接続が HTTP/2 のセキュリティ要件（RFC 7540）を満たしているかを判定するゲートキーパー関数です。主に TLS 接続において、現代的なプロトコルバージョンと暗号スイートが使用されているかを確認します。

シグネチャまたは形状

int h2_protocol_is_acceptable_c1(conn_rec *c, request_rec *r, int require_all)

入力と出力

  • **入力**:
  • c: 現在の接続情報 (conn_rec)。
  • r: リクエスト情報 (request_rec)。
  • require_all: TLS パラメータが未確定の場合に失敗とみなすかを制御するフラグ。
  • **出力**:
  • 1: 接続が HTTP/2 の要件を満たしている場合。
  • 0: 要件を満たさない、または TLS パラメータが不適切/未確定の場合。

前提条件と不変条件

  • H2_CONF_MODERN_TLS_ONLY 設定が有効な場合のみ、厳格な TLS チェックが実行されます。
  • TLS 接続ではない場合、本関数は常に 1 を返します。

アルゴリズムまたは振る舞い

1. ap_ssl_conn_is_ssl(c) を使用して TLS 接続か判定します。 2. TLS 接続かつ H2_CONF_MODERN_TLS_ONLY が有効な場合：

  • SSL_PROTOCOL を取得し、TLSv1.2 以上であることを確認します（TLSv1/v1.1 は拒否）。
  • require_all が真でプロトコルが取得できない場合は 0 を返します。
  • SSL_PROTOCOL が TLSv1.2 の場合、SSL_CIPHER を取得し、cipher_is_blacklisted() を用いて RFC 7540 で禁止されている暗号スイートが含まれていないかチェックします。
  • require_all が真で暗号スイートが取得できない場合は 0 を返します。

    • 3. すべてのチェックを通過した場合、または TLS 以外の接続の場合は 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c
  • modules/http2/h2_session.c
  • modules/http2/h2_switch.c
  • **呼び出し先**:
  • ap_ssl_conn_is_ssl
  • h2_config_cgeti
  • ap_ssl_var_lookup
  • cipher_is_blacklisted
  • ap_log_cerror

関連データ構造

  • conn_rec: Apache の接続コンテキスト。
  • request_rec: Apache のリクエストコンテキスト。

失敗モードと境界条件

  • TLS プロトコルが TLSv1 または TLSv1.1 の場合、0 を返します。
  • ブラックリストに登録された暗号スイートが検出された場合、0 を返します。
  • require_all が 1 の状態で、TLS プロトコルや暗号スイートが特定できない（indetermined）場合、0 を返します。

保守担当者向けメモ

  • 本関数は RFC 7540 Appendix A に基づくブラックリスト管理に依存しています。新しい脆弱な暗号スイートが特定された場合は cipher_is_blacklisted 側の更新が必要です。
  • require_all フラグは、ハンドシェイクの進行度合いに応じて厳格さを調整するために使用されます。
