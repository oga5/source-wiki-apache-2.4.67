authn_cache_timeout

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:239

目的

Apache HTTP Server の設定ディレクティブ AuthnCacheTimeout を処理し、認証キャッシュの有効期限を設定する。

シグネチャまたは形状

static const char *authn_cache_timeout(cmd_parms *cmd, void *CFG, const char *arg)

入力と出力

  • cmd: 設定コマンドのパラメータ構造体。
  • CFG: ディレクトリごとの設定構造体 (authn_cache_dircfg)。
  • arg: 設定ファイルから渡されるタイムアウト値（秒単位の文字列）。
  • 戻り値: 成功時は NULL を返し、エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • CFG は authn_cache_dircfg 型としてキャスト可能であること。
  • arg は atoi で変換可能な数値文字列であること。

アルゴリズムまたは振る舞い

1. arg を atoi で整数値に変換する。 2. 変換された秒数を apr_time_from_sec を用いて APR 内部形式の時刻に変換する。 3. CFG が指す authn_cache_dircfg 構造体の timeout メンバに値を格納する。 4. NULL を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/aaa/mod_authn_socache.c:258 において、AP_INIT_TAKE1 マクロを通じて設定ディレクティブ AuthnCacheTimeout のハンドラとして登録されている。
  • 呼び出し先: atoi (標準ライブラリ), apr_time_from_sec (APR)。

関連データ構造

  • authn_cache_dircfg: ディレクトリごとの設定を保持する構造体。本関数はこの構造体の timeout フィールドを更新する。

失敗モードと境界条件

  • arg が数値以外の文字列である場合、atoi の仕様に従い 0 が設定される可能性がある。

保守担当者向けメモ

  • 本関数は設定フェーズでのみ実行される。
  • apr_time_from_sec を使用しているため、設定値は秒単位として扱われる。
