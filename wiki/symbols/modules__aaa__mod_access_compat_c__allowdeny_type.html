allowdeny_type

定義

  • 種別: enum
  • 定義箇所: modules/aaa/mod_access_compat.c:46

目的

mod_access_compat において、アクセス制御ルール（Allow または Deny）の判定条件の種類を識別するために使用されます。

シグネチャまたは形状

enum allowdeny_type {
    T_ENV,
    T_NENV,
    T_ALL,
    T_IP,
    T_HOST,
    T_FAIL
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • 各列挙子は、Allow または Deny ディレクティブで指定された条件の評価方法に対応します。
  • T_FAIL は、条件の解析や判定に失敗した状態を表すために使用されます。

アルゴリズムまたは振る舞い

  • T_ENV / T_NENV: 環境変数の有無による判定。
  • T_ALL: すべてのアクセスを許可または拒否。
  • T_IP: IPアドレスまたはサブネットによる判定。
  • T_HOST: ホスト名による判定。
  • T_FAIL: 異常系または未定義の条件。

呼び出し元と呼び出し先

  • 呼び出し元: modules/aaa/mod_access_compat.c:61 にて、この型を使用した構造体メンバが定義されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • allowdeny: allowdeny_type 型のメンバ type を含む構造体。
  • access_compat_dir_conf: allowdeny 構造体の配列を保持し、ディレクトリごとのアクセス制御設定を管理します。

失敗モードと境界条件

  • T_FAIL が設定された場合、アクセス判定ロジックにおいてエラーとして扱われる可能性があります。

保守担当者向けメモ

  • 本列挙型は mod_access_compat 内部の判定ロジックに密結合しています。新しい判定条件を追加する場合は、この列挙型を拡張し、対応する判定ロジックを find_allowdeny 関数に追加する必要があります。
