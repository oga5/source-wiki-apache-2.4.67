h2_ififo

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.c:852

目的

スレッドセーフな整数値の FIFO（先入れ先出し）キューを提供します。マルチスレッド環境下でのスレッド間のデータ受け渡しや同期を目的としています。

シグネチャまたは形状

struct h2_ififo {
    int *elems;
    int capacity;
    int set;
    int head;
    int count;
    int aborted;
    apr_thread_mutex_t *lock;
    apr_thread_cond_t  *not_empty;
    apr_thread_cond_t  *not_full;
};

入力と出力

  • **入力**: h2_ififo_create または h2_ififo_set_create を通じて、容量 (capacity) とメモリプール (apr_pool_t) を指定して初期化されます。
  • **出力**: キュー内の要素数 (h2_ififo_count) や、プッシュ/プル操作による整数値のキューイングおよび取り出しを行います。

前提条件と不変条件

  • lock メンバーにより、キューへのアクセスはスレッドセーフに保護されます。
  • not_empty および not_full 条件変数を用いて、キューの状態（空または満杯）に応じたブロッキング操作が可能です。
  • aborted フラグが設定された場合、キュー操作は中断されます。

アルゴリズムまたは振る舞い

  • **同期**: apr_thread_mutex_lock を用いてクリティカルセクションを保護し、条件変数でスレッド間の待ち合わせを行います。
  • **操作**: ififo_push_int 等の内部関数により、リングバッファとして実装された elems 配列に対して要素の追加・削除を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_util.c 内の h2_ififo_create, h2_ififo_set_create, h2_ififo_term, h2_ififo_count, ififo_push などから参照されます。
  • **呼び出し先**: APR のスレッド同期プリミティブ（apr_thread_mutex_t, apr_thread_cond_t）を呼び出します。

関連データ構造

  • apr_thread_mutex_t: 排他制御用。
  • apr_thread_cond_t: スレッド間通知用。
  • h2_fifo: 同様の目的を持つ汎用 FIFO キュー。

失敗モードと境界条件

  • **容量制限**: capacity を超える要素の追加は、not_full 条件変数によりブロッキングされるか、設定に応じてエラーを返します。
  • **中断**: aborted 状態になると、キュー操作は即座に終了します。
  • **メモリ不足**: h2_ififo_create 等の初期化時にメモリ確保に失敗した場合、エラーコードが返されます。

保守担当者向けメモ

  • 本構造体は h2_util.c 内で完結する内部的な同期機構です。
  • h2_ififo_set_create が存在することから、単なるキューだけでなく、要素の重複を排除するセットとしての利用も想定されています。
  • 構造体の各メンバーの更新は必ず lock を保持した状態で行う必要があります。
