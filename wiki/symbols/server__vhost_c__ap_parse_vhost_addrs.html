ap_parse_vhost_addrs

定義

  • 種別: function
  • 定義箇所: server/vhost.c:242

目的

<VirtualHost> ディレクティブで指定されたホスト名や IP アドレスの文字列を解析し、サーバー設定構造体（server_rec）に紐付くアドレスリストを構築します。

シグネチャまたは形状

const char *ap_parse_vhost_addrs(apr_pool_t *p,
                                 const char *hostname,
                                 server_rec *s)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • hostname: 解析対象のホスト名または IP アドレスを含む文字列。
  • s: 設定を格納する server_rec 構造体。
  • **出力**:
  • 戻り値: 解析成功時は NULL、エラー発生時はエラーメッセージ文字列を返します。
  • 副作用: s->addrs に解析されたアドレスリストが格納され、必要に応じて s->port が更新されます。

前提条件と不変条件

  • hostname は ap_getword_conf で分割可能な形式である必要があります。
  • s->addrs はリストの終端として NULL が設定されることが保証されます。

アルゴリズムまたは振る舞い

1. s->addrs を起点としてアドレスリストの構築を開始します。 2. hostname 文字列を ap_getword_conf で順次切り出し、get_addresses を呼び出してアドレス情報を生成・追加します。 3. get_addresses がエラーを返した場合は、リストを NULL で終端し、エラーメッセージを即座に返します。 4. すべての解析完了後、リストを NULL で終端します。 5. s->addrs が存在する場合、最初のアドレスのポート番号（host_port）を s->port に反映し、メインサーバーから継承されたデフォルトポートを上書きします。

呼び出し元と呼び出し先

  • **呼び出し元**: server/config.c:2120
  • **呼び出し先**: get_addresses（内部関数）、ap_getword_conf（APR/Apache ユーティリティ）

関連データ構造

  • server_rec: 仮想ホストの設定を保持する構造体。
  • server_addr_rec: 解析された IP アドレスとポート情報を保持するリストノード。

失敗モードと境界条件

  • get_addresses が解析に失敗した場合、エラーメッセージを返し、リストは不完全な状態で NULL 終端されます。
  • hostname が空文字列の場合、ループは実行されず、s->addrs は NULL のままとなります。

保守担当者向けメモ

  • 本関数は設定読み込みフェーズで実行されます。
  • s->port の上書きロジックは、<VirtualHost> で指定された最初のポートを優先する仕様に基づいています。
