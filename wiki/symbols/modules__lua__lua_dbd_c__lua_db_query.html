lua_db_query

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:152

目的

Lua スクリプトからデータベースに対して非選択系クエリ（INSERT, UPDATE, DELETE 等）を実行し、影響を受けた行数を返します。

シグネチャまたは形状

int lua_db_query(lua_State *L)

入力と出力

  • **入力**: Lua スタックの第 3 引数に SQL 文（文字列）を要求します。
  • **出力**:
  • 成功時: 影響を受けた行数（数値）をスタックにプッシュし、1 を返します。
  • 失敗時: nil をプッシュし、エラーメッセージが存在する場合はその文字列をスタックにプッシュして 2 を返します。

前提条件と不変条件

  • lua_get_db_handle(L) を介して取得したデータベースハンドルが有効（db->alive が真）である必要があります。
  • 第 3 引数は LUA_TSTRING 型である必要があります。

アルゴリズムまたは振る舞い

1. luaL_checktype で第 3 引数が文字列であることを確認します。 2. lua_get_db_handle でデータベースハンドルを取得します。 3. ハンドルが有効な場合、apr_dbd_query を呼び出し、SQL を実行します。 4. 実行結果が APR_SUCCESS であれば、影響を受けた行数を lua_pushnumber で返します。 5. 実行に失敗した場合、apr_dbd_error でエラーメッセージを取得し、lua_pushnil とエラー文字列を返します。 6. ハンドルが無効な場合は、行数として -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_dbd.c:698 にて lua_pushcfunction を通じて Lua 環境へ登録されています。
  • **呼び出し先**:
  • luaL_checktype (Lua API)
  • lua_tostring (Lua API)
  • lua_get_db_handle (内部関数)
  • apr_dbd_query (APR DBD API)
  • apr_dbd_error (APR DBD API)
  • lua_pushnumber / lua_pushnil / lua_pushstring (Lua API)

関連データ構造

  • lua_db_handle: データベース接続状態を保持する構造体。
  • lua_State: Lua 仮想マシンの状態。

失敗モードと境界条件

  • **ハンドル無効**: db が NULL または db->alive が偽の場合、エラーとして扱われ、行数 -1 が返されます。
  • **SQL 実行エラー**: apr_dbd_query が APR_SUCCESS 以外を返した場合、apr_dbd_error で取得したエラーメッセージが Lua 側に伝播されます。

保守担当者向けメモ

  • 本関数はプリペアドステートメントを使用しない直接的な SQL 実行を行います。パラメータを含むクエリが必要な場合は、lua_db_prepared_query の使用を検討してください。
  • 戻り値の数は成功時と失敗時で異なる（1 または 2）ため、Lua 側での呼び出し元コードは戻り値の数を確認する必要があります。
