util_search_node_t

定義

  • 種別: struct
  • 定義箇所: modules/ldap/util_ldap_cache.h:113

目的

LDAP 検索操作の結果をキャッシュするためのデータ構造です。ユーザー名に基づいた検索結果（DN、パスワード、属性値）を保持し、同一の検索クエリに対するバックエンド LDAP サーバーへの再問い合わせを抑制します。

シグネチャまたは形状

typedef struct util_search_node_t {
    const char *username;               /* キャッシュキー */
    const char *dn;                     /* 検索結果の DN */
    const char *bindpw;                 /* 直近のバインドパスワード (失敗時は NULL) */
    apr_time_t lastbind;                /* 最終成功バインド時刻 */
    const char **vals;                  /* 取得した属性値の配列 */
    int        numvals;                 /* 属性値の数 */
} util_search_node_t;

入力と出力

  • **入力**: キャッシュ挿入時に username がキーとして使用されます。
  • **出力**: キャッシュヒット時に dn、bindpw、vals 等の検索結果が後続の認証・認可処理へ提供されます。

前提条件と不変条件

  • vals は numvals 個の属性値を保持する配列である必要があります。
  • キャッシュ内のデータは util_ald_alloc を用いて確保されたメモリ領域に配置される必要があります。
  • bindpw が NULL の場合、そのユーザーに対するバインド操作は失敗したとみなされます。

アルゴリズムまたは振る舞い

  • util_ldap_cache.c 内の操作関数により、ハッシュ計算、比較、コピー、解放の対象となります。
  • キャッシュミス発生時、util_ald_alloc でメモリを確保し、検索結果を複製して格納します。
  • 共有メモリ環境下では、ポインタ値は apr_rmm を介したオフセット管理の対象となります。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c 内の検索ロジック（util_search_node_t 型の変数を宣言・利用）。
  • **呼び出し先**: modules/ldap/util_ldap_cache.c 内のキャッシュ管理関数（util_ald_alloc 等）。
  • 詳細はソースコード上の参照に基づきますが、具体的な関数呼び出しの連鎖については、提供された情報範囲内では限定的です。

関連データ構造

  • util_url_node_t: 本ノードを保持する親階層のキャッシュノード。
  • util_ald_cache_t: 本ノードを管理するキャッシュ全体のメタデータ構造体。

失敗モードと境界条件

  • numvals が 0 の場合、属性値は存在しないものとして扱われます。
  • メモリ不足時、util_ald_alloc が失敗する可能性があります。
  • 共有メモリ使用時、ポインタの不適切な参照はプロセス間での不正アクセスを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は util_ldap_cache.c で実装される各操作関数（ハッシュ、比較、コピー、解放）の対象です。変更を加える際は、必ず util_ldap_cache.c 側の対応する実装を更新してください。
  • 共有メモリ（APR_HAS_SHARED_MEMORY）環境では、構造体内のポインタが指すメモリ領域が共有メモリ内にあることを保証する必要があります。
