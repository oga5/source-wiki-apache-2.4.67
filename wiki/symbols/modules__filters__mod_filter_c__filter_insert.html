filter_insert

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_filter.c:637

目的

リクエスト処理の開始時に、設定されたフィルタチェーンを評価し、有効なフィルタをリクエストの出力フィルタスタックへ挿入する。

シグネチャまたは形状

static void filter_insert(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: なし（副作用としてリクエストのフィルタスタックを更新）

前提条件と不変条件

  • r->per_dir_config から filter_module の設定 (mod_filter_cfg) が取得可能であること。
  • NO_PROTOCOL が定義されていない場合、mod_filter_ctx がリクエストプール (r->pool) に割り当てられ、リクエスト設定に格納される。

アルゴリズムまたは振る舞い

1. **設定取得**: ap_get_module_config を用いて現在のディレクトリ設定を取得する。 2. **チェーンの整理**: 設定されたフィルタチェーンを走査し、fname が NULL のエントリをチェーンから除外する。 3. **フィルタの挿入**:

  • cfg->live_filters ハッシュからフィルタ名を検索する。
  • フィルタが見つからない場合は警告ログを出力しスキップする。
  • ap_add_output_filter_handle を呼び出し、フィルタをスタックに追加する。

    • 4. **プロトコル調整**: NO_PROTOCOL が未定義の場合、AP_FILTER_PROTO_NO_BYTERANGE または AP_FILTER_PROTO_CHANGE_LENGTH フラグを持つフィルタが挿入された際に、Range ヘッダをリクエストから削除し、コンテキストに保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_insert_filter により登録され、Apache のフィルタ挿入フェーズで呼び出される。
  • **呼び出し先**:
  • ap_get_module_config / ap_set_module_config
  • apr_pcalloc
  • ap_log_rerror
  • ap_add_output_filter_handle
  • apr_hash_get
  • apr_table_get / apr_table_unset

関連データ構造

  • mod_filter_cfg: モジュール設定（チェーンと登録済みフィルタのハッシュを保持）。
  • mod_filter_chain: フィルタの連結リスト。
  • mod_filter_ctx: リクエストごとの実行コンテキスト。
  • ap_filter_rec_t: フィルタの定義情報。

失敗モードと境界条件

  • **フィルタ未登録**: cfg->live_filters に存在しないフィルタ名がチェーンに含まれている場合、警告ログを出力して処理を継続する。
  • **メモリ不足**: apr_pcalloc が失敗した場合、Apache の標準的なメモリ割り当てエラーが発生する。

保守担当者向けメモ

  • 本関数は ap_hook_insert_filter を介して実行されるため、リクエスト処理の初期段階でフィルタスタックを構築する重要な役割を担う。
  • NO_PROTOCOL マクロによる条件付きコンパイルが含まれており、プロトコル制御（Range ヘッダの処理など）が環境によって無効化される可能性がある。
  • cfg->chain の整理ロジックは、設定マージ後の最終的なクリーンアップとして機能している。
