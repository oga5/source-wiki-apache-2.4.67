reqtimeout_before_header

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reqtimeout.c:404

目的

新しいリクエストの処理開始に先立ち、mod_reqtimeout の接続ごとの状態（reqtimeout_con_cfg）をヘッダー読み取りフェーズ用に初期化する。

シグネチャまたは形状

static void reqtimeout_before_header(request_rec *r, conn_rec *c)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • c: 現在の接続構造体。
  • **出力**: なし（副作用として c に紐付く接続設定構造体を更新）。

前提条件と不変条件

  • c->conn_config に reqtimeout_module の設定が格納されている必要がある。
  • c->base_server->module_config にサーバーごとの設定が存在する必要がある（AP_DEBUG_ASSERT により検証）。
  • 接続の生存期間に依存する ccfg->socket および ccfg->tmpbb は保持される。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、接続設定 ccfg を取得する。設定が存在しない場合は即座に終了する。 2. サーバー設定 cfg を取得する。 3. ccfg の状態を以下のように更新する:

  • type を "header" に設定。
  • timeout_at および max_timeout_at を 0 にリセット。
  • in_keep_alive を c->keepalives > 0 に基づいて設定。

    • 4. INIT_STAGE マクロを呼び出し、サーバー設定 cfg に基づいてヘッダー読み取り用のステージ設定を ccfg に適用する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_pre_read_request に登録されており、リクエスト読み取り直前のフックとして呼び出される（modules/filters/mod_reqtimeout.c:636）。
  • **呼び出し先**: ap_get_module_config（Apache API）、INIT_STAGE（マクロ）。

関連データ構造

  • reqtimeout_con_cfg: 接続ごとの状態管理構造体。
  • reqtimeout_srv_cfg: サーバーごとの設定構造体。
  • request_rec, conn_rec: Apache HTTP Server のコア構造体。

失敗モードと境界条件

  • ccfg が NULL の場合、何もせず正常終了する。
  • cfg が NULL の場合、AP_DEBUG_ASSERT によりデバッグビルド環境ではアサーションエラーが発生する。

保守担当者向けメモ

  • 本関数はリクエストごとに呼び出されるため、接続全体で共有されるリソース（ソケットや一時バッファ）を誤ってクリアしないよう注意すること。
  • INIT_STAGE マクロの定義および reqtimeout_con_cfg の構造変更時には、本関数での初期化ロジックとの整合性を確認すること。
