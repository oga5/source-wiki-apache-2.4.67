job_loadj

定義

  • 種別: function
  • 定義箇所: modules/md/md_status.c:125

目的

ストレージからジョブの JSON 状態を読み込み、検証および必要に応じたログのフィルタリングを行う。

シグネチャまたは形状

apr_status_t job_loadj(md_json_t **pjson, md_store_group_t group, const char *name,
                       struct md_reg_t *reg, int with_log, apr_pool_t *p)

入力と出力

  • **入力**:
  • pjson: JSON オブジェクトを格納するポインタのポインタ。
  • group: ジョブが属するストレージグループ（md_store_group_t）。
  • name: ジョブ名。
  • reg: レジストリハンドル（md_reg_t）。
  • with_log: ログを含めるかどうかのフラグ（真であれば保持）。
  • p: メモリ割り当て用プール。
  • **出力**:
  • rv: APR_SUCCESS（成功時）、またはエラーコード。
  • *pjson: 読み込まれた JSON オブジェクト（失敗時は NULL）。

前提条件と不変条件

  • reg から取得される md_store_t が有効であること。
  • 読み込まれた JSON は md_job_json_seems_valid による妥当性チェックを通過する必要がある。

アルゴリズムまたは振る舞い

1. md_reg_store_get を使用してストレージインスタンスを取得する。 2. md_store_load_json を呼び出し、指定されたグループと名前からジョブの JSON を読み込む。 3. 読み込み成功時、md_job_json_seems_valid で内容を検証する。検証に失敗した場合は *pjson を NULL にし、APR_ENOENT を返す。 4. with_log が偽の場合、md_json_del を呼び出して JSON から MD_KEY_LOG エントリを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_ocsp.c:1025
  • modules/md/md_status.c:170
  • modules/md/md_status.c:287
  • **呼び出し先**:
  • md_reg_store_get
  • md_store_load_json
  • md_job_json_seems_valid
  • md_json_del

関連データ構造

  • md_json_t: JSON データの保持に使用。
  • md_store_group_t: ストレージ内の配置グループ。
  • md_reg_t: レジストリ情報。

失敗モードと境界条件

  • md_store_load_json が失敗した場合、そのエラーコードをそのまま返す。
  • md_job_json_seems_valid が偽を返した場合、APR_ENOENT が返される。

保守担当者向けメモ

  • 本関数は md_status.c 内のステータス生成処理において、ジョブ情報の取得とフィルタリングを共通化するために使用されている。
  • with_log フラグによるログ削除は、JSON オブジェクトのメモリ構造を直接操作するため、呼び出し側で p プールが適切に管理されている必要がある。
