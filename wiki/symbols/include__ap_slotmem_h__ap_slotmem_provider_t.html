ap_slotmem_provider_t

定義

  • 種別: struct
  • 定義箇所: include/ap_slotmem.h:86

目的

共有メモリ領域を固定サイズの「スロット」として管理するための関数ポインタ群を保持する抽象インターフェースです。Apache モジュールが共有メモリの確保、アタッチ、およびスロット単位の読み書きを行うためのプロバイダーモデルを提供します。

シグネチャまたは形状

struct ap_slotmem_provider_t {
    const char *name;
    apr_status_t (*doall)(ap_slotmem_instance_t *s, ap_slotmem_callback_fn_t *func, void *data, apr_pool_t *pool);
    apr_status_t (*create)(ap_slotmem_instance_t **inst, const char *name, apr_size_t item_size, unsigned int item_num, ap_slotmem_type_t type, apr_pool_t *pool);
    apr_status_t (*attach)(ap_slotmem_instance_t **inst, const char *name, apr_size_t *item_size, unsigned int *item_num, apr_pool_t *pool);
    apr_status_t (*dptr)(ap_slotmem_instance_t *s, unsigned int item_id, void**mem);
    apr_status_t (*get)(ap_slotmem_instance_t *s, unsigned int item_id, unsigned char *dest, apr_size_t dest_len);
    apr_status_t (*put)(ap_slotmem_instance_t *slot, unsigned int item_id, unsigned char *src, apr_size_t src_len);
    unsigned int (*num_slots)(ap_slotmem_instance_t *s);
    unsigned int (*num_free_slots)(ap_slotmem_instance_t *s);
    apr_size_t (*slot_size)(ap_slotmem_instance_t *s);
    apr_status_t (*grab)(ap_slotmem_instance_t *s, unsigned int *item_id);
    apr_status_t (*release)(ap_slotmem_instance_t *s, unsigned int item_id);
    apr_status_t (*fgrab)(ap_slotmem_instance_t *s, unsigned int item_id);
};

入力と出力

  • **入力**: 各関数ポインタは、操作対象の ap_slotmem_instance_t ハンドル、スロット識別子 (item_id)、およびデータ操作用のバッファやサイズを受け取ります。
  • **出力**: 状態操作関数は apr_status_t を返し、成功時に APR_SUCCESS を返します。情報取得関数はスロット数やサイズを返します。

前提条件と不変条件

  • create 時に指定されたスロットサイズと数は固定されます。
  • AP_SLOTMEM_TYPE_NOTMPSAFE フラグが指定されていない限り、実装側で適切な排他制御を行うことが期待されます。
  • dptr はメモリへの直接ポインタを返しますが、その有効性はスロットのライフサイクルに依存します。

アルゴリズムまたは振る舞い

  • **初期化**: create で新規領域確保、attach で既存領域への接続を行います。
  • **アクセス**: dptr による直接アクセス、または get/put によるコピーベースのアクセスを提供します。
  • **管理**: grab/release によりスロットの使用状態を追跡します。doall は全スロットに対してコールバックを実行します。fgrab は特定の item_id を強制的に確保します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy_balancer や mod_heartmonitor などのモジュールが、共有メモリ管理のためにこの構造体を利用します。
  • **呼び出し先**: 具体的な実装（mod_slotmem_shm など）が各関数ポインタの実体を提供します。

関連データ構造

  • ap_slotmem_instance_t: 共有メモリ領域を指す不透明なハンドル。
  • ap_slotmem_type_t: メモリ特性を指定するフラグ。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • create/attach は共有メモリの確保や接続に失敗した場合、エラーコードを返します。
  • item_id が範囲外の場合、各操作は失敗する可能性があります。
  • 共有メモリの排他制御が競合した場合、apr_status_t を通じてエラーが通知されます。

保守担当者向けメモ

  • 本構造体はインターフェース定義であり、実装は mod_slotmem_shm.c 等に存在します。
  • fgrab は 2.4.4-dev (20120211.9) で追加されました。
  • 構造体の変更は ABI 互換性に影響するため注意してください。
