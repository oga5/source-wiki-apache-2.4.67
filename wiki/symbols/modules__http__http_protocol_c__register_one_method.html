register_one_method

定義

  • 種別: function
  • 定義箇所: modules/http/http_protocol.c:662

目的

HTTPメソッド名（文字列）とメソッド番号（整数）のペアを、内部的なメソッドレジストリ（methods_registry）に登録する。

シグネチャまたは形状

static void register_one_method(apr_pool_t *p, const char *methname, int methnum)

入力と出力

  • p: メソッド番号を格納するメモリを割り当てるための apr_pool_t。
  • methname: 登録する HTTP メソッド名（例: "GET"）。
  • methnum: 対応するメソッド番号（例: M_GET）。
  • 出力: なし（副作用として methods_registry を更新）。

前提条件と不変条件

  • methods_registry が初期化済みであること。
  • methname は APR_HASH_KEY_STRING として扱われるため、ヌル終端文字列であること。

アルゴリズムまたは振る舞い

1. 指定されたプール p から、整数型（int）のサイズ分のメモリを apr_palloc で確保する。 2. 確保したメモリ領域に methnum を書き込む。 3. methods_registry ハッシュテーブルに対し、methname をキー、確保したメモリ領域のポインタを値として登録する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http/http_protocol.c 内のメソッド初期化処理（GET, HEAD, PUT, POST, DELETE, CONNECT, OPTIONS, TRACE, PATCH, PROPFIND, PROPPATCH, MKCOL の各登録箇所）。
  • 呼び出し先: apr_palloc（メモリ確保）、apr_hash_set（ハッシュテーブルへの挿入）。

関連データ構造

  • methods_registry: 動的メソッドを保持する apr_hash_t。
  • apr_pool_t: メモリ管理に使用されるリソースプール。

失敗モードと境界条件

  • メモリ不足時: apr_palloc は通常、メモリ確保失敗時にプロセスを終了させる（APRのデフォルト動作）。
  • 重複登録: 同一の methname で呼び出した場合、apr_hash_set の仕様により既存の値が上書きされる。

保守担当者向けメモ

  • 本関数は静的関数であり、主にモジュール初期化時に標準メソッドをレジストリへ登録するために使用される。
  • methnum を保持するメモリはリクエストプールではなく、モジュールのライフサイクルに紐づくプールから割り当てる必要がある点に注意すること。
