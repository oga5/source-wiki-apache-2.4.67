md_will_renew_cert

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_drive.c:234

目的

指定されたマネージドドメイン（md_t）において、証明書の自動更新処理を実行すべきかどうかを判定します。

シグネチャまたは形状

int md_will_renew_cert(const md_t *md)

入力と出力

  • **入力**: md (マネージドドメイン設定構造体へのポインタ)
  • **出力**: int 型。更新が必要な場合は 1、不要な場合は 0 を返します。

前提条件と不変条件

  • 入力される md ポインタは有効である必要があります。
  • md->renew_mode が MD_RENEW_MANUAL の場合、自動更新は無効とみなされます。

アルゴリズムまたは振る舞い

1. md->renew_mode が MD_RENEW_MANUAL である場合、直ちに 0 を返します。 2. md->renew_mode が MD_RENEW_AUTO であり、かつ md->cert_files が存在し、その要素数が 0 より大きい（既に証明書ファイルが存在する）場合、0 を返します。 3. 上記のいずれにも該当しない場合、1 を返して更新が必要であると判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md.c:832
  • modules/md/mod_md_drive.c:102
  • **呼び出し先**:
  • 本関数内から直接呼び出している関数はありません。

関連データ構造

  • md_t: マネージドドメインの設定と状態を保持する構造体。renew_mode および cert_files フィールドを参照します。

失敗モードと境界条件

  • md が NULL の場合、ポインタ参照によるクラッシュの可能性があります（ソースコード上に明示的なガードはありません）。
  • md->cert_files が NULL の場合、md->cert_files->nelts へのアクセスで境界条件が発生する可能性があります。

保守担当者向けメモ

  • 本関数は更新の「可否」のみを判定します。実際の更新実行タイミングや期限の妥当性については、md_reg_should_renew 等の他の関数が担っています。
  • MD_RENEW_AUTO モードにおいて、証明書ファイルが既に存在する場合に更新をスキップするロジックとなっているため、証明書の更新期限が迫っている場合の挙動については、呼び出し側のロジックと併せて確認してください。
