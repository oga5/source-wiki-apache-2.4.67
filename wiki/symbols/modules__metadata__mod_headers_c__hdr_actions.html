hdr_actions

定義

  • 種別: enum
  • 定義箇所: modules/metadata/mod_headers.c:90

目的

HTTP ヘッダーに対して実行可能な操作の種類を定義する列挙型です。

シグネチャまたは形状

typedef enum {
    hdr_add = 'a',
    hdr_set = 's',
    hdr_append = 'm',
    hdr_merge = 'g',
    hdr_unset = 'u',
    hdr_echo = 'e',
    hdr_edit = 'r',
    hdr_edit_r = 'R',
    hdr_setifempty = 'i',
    hdr_note = 'n'
} hdr_actions;

入力と出力

  • 入力: 各列挙子は文字定数（char）に対応する値を持ちます。
  • 出力: ヘッダー操作の種別を識別するための識別子として機能します。

前提条件と不変条件

  • 各列挙子は、mod_headers モジュール内でのヘッダー処理ロジックにおけるアクションを規定します。
  • 値は文字コードと対応しており、内部的なスイッチ文や条件分岐の判定基準として使用されます。

アルゴリズムまたは振る舞い

  • hdr_add: ヘッダーを追加します。
  • hdr_set: 既存の値を置換します。
  • hdr_append: 既存の値にマージします。
  • hdr_merge: 重複を避けてマージします。
  • hdr_unset: ヘッダーを削除します。
  • hdr_echo: リクエストヘッダーをレスポンスに反映します。
  • hdr_edit / hdr_edit_r: 正規表現による置換を行います。
  • hdr_setifempty: ヘッダーが存在しない場合のみ設定します。
  • hdr_note: ヘッダー値を内部ノート（request_rec）に保存します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/metadata/mod_headers.c 内の header_entry 構造体でメンバとして使用されています（128行目）。
  • 呼び出し先: 具体的な関数呼び出し関係はソース抜粋からは特定できません。

関連データ構造

  • header_entry: 本列挙型を action メンバとして保持します。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 新しいヘッダー操作を追加する場合は、本列挙型に新しい文字コードを割り当て、do_headers_fixup 等の処理ロジックを更新する必要があります。
