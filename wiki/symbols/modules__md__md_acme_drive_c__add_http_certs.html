add_http_certs

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:208

目的

HTTP レスポンスから証明書チェーンを抽出し、指定された配列に追加する。

シグネチャまたは形状

static apr_status_t add_http_certs(apr_array_header_t *chain, apr_pool_t *p,
                                   const md_http_response_t *res)

入力と出力

  • **入力**:
  • chain: 証明書を格納する apr_array_header_t へのポインタ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • res: HTTP レスポンスデータを含む md_http_response_t 構造体。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • res が有効な HTTP レスポンスを保持していること。
  • chain は初期化済みの配列であること。

アルゴリズムまたは振る舞い

1. レスポンスヘッダーから Content-Type を取得し、解析する。 2. Content-Type が application/x-pkcs7-mime（ルート証明書とみなされる）である場合、チェーンへの追加を行わず終了する。 3. md_cert_chain_read_http を呼び出し、レスポンスから証明書を読み込んで chain に追加する。 4. 読み込みに失敗し、かつエラーが ENOENT である場合は、APR_EAGAIN を返してデバッグログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_drive.c:243
  • modules/md/md_acme_drive.c:410
  • **呼び出し先**:
  • md_util_parse_ct
  • md_log_perror
  • md_cert_chain_read_http

関連データ構造

  • apr_array_header_t: 証明書チェーンの保持に使用。
  • md_http_response_t: HTTP 通信結果の保持に使用。

失敗モードと境界条件

  • Content-Type がルート証明書を示す場合、正常終了（APR_SUCCESS）として処理をスキップする。
  • md_cert_chain_read_http が証明書を見つけられない場合、APR_EAGAIN を返す。

保守担当者向けメモ

  • 本関数は証明書チェーンの取得プロセスにおいて、特定の MIME タイプをフィルタリングする役割を担う。
  • md_cert_chain_read_http の戻り値に基づき、一時的な失敗（ENOENT）を APR_EAGAIN に変換して上位層へ通知する設計となっている。
