create_cern_meta_dir_config

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_cern_meta.c:143

目的

ディレクトリごとの設定構造体 cern_meta_dir_config を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_cern_meta_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • p: 設定を割り当てるためのメモリプール (apr_pool_t*)。
  • dummy: 使用されない引数 (char*)。
  • **出力**:
  • 初期化された cern_meta_dir_config 構造体へのポインタを void* として返す。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • 割り当てられた構造体の各メンバは以下の初期値を持つ:
  • metadir: NULL
  • metasuffix: NULL
  • metafiles: DEFAULT_METAFILES

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、cern_meta_dir_config 構造体サイズ分のメモリをプール p から確保する。 2. 構造体の各フィールドに対して、デフォルト値または NULL を代入する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_cern_meta.c:365 において、モジュールのディレクトリ設定作成用関数として登録されている。
  • **呼び出し先**: apr_palloc (APR API)。

関連データ構造

  • cern_meta_dir_config: 本関数が初期化対象とするディレクトリ設定構造体。

失敗モードと境界条件

  • apr_palloc がメモリ不足により失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。
  • DEFAULT_METAFILES はモジュール内で定義されたマクロであり、初期値として使用される。
