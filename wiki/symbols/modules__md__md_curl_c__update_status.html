update_status

定義

  • 種別: function
  • 定義箇所: modules/md/md_curl.c:362

目的

HTTP リクエストの実行結果から HTTP ステータスコードを取得し、内部のレスポンス構造体に反映させる。

シグネチャまたは形状

static apr_status_t update_status(md_http_request_t *req)

入力と出力

  • **入力**: md_http_request_t *req (HTTP リクエストコンテキスト)
  • **出力**: apr_status_t (成功時は APR_SUCCESS、curl_easy_getinfo が失敗した場合はそのエラーコード)

前提条件と不変条件

  • req は有効なポインタであること。
  • req->internals が初期化されていること。

アルゴリズムまたは振る舞い

1. req->internals が存在する場合、curl_easy_getinfo を呼び出し、CURLINFO_RESPONSE_CODE を取得する。 2. curl_status 関数を介して取得結果を apr_status_t に変換する。 3. 取得が成功した場合、internals->response->status に取得したステータスコードを格納する。 4. 成功時には、トレースログ (MD_LOG_TRACE3) にリクエスト ID とステータスコードを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_curl.c:576
  • **呼び出し先**:
  • curl_easy_getinfo (libcurl API)
  • curl_status (内部ヘルパー関数)
  • md_log_perror (ログ出力関数)

関連データ構造

  • md_http_request_t: リクエスト全体を管理する構造体。
  • md_curl_internals_t: libcurl ハンドルやレスポンス情報を保持する内部コンテキスト。

失敗モードと境界条件

  • internals が NULL の場合、何も更新せず APR_SUCCESS を返す。
  • curl_easy_getinfo が失敗した場合、そのエラーコードをそのまま呼び出し元に返す。

保守担当者向けメモ

  • 本関数は libcurl のハンドルからステータスを抽出する薄いラッパーである。
  • ログ出力には req->pool を使用しているため、リクエストのライフサイクルと同期している。
