ap_process_http_sync_connection

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:180

目的

同期型MPM（Multi-Processing Module）において、確立された接続上のHTTPリクエストをループ処理し、リクエストの読み込みから実行、およびKeep-Aliveの継続判定までを一元管理する。

シグネチャまたは形状

static int ap_process_http_sync_connection(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（現在の接続情報を保持する構造体）
  • **出力**: int（常に OK を返す）

前提条件と不変条件

  • ap_process_http_connection から呼び出されることを想定している。
  • ap_process_request(r) 呼び出し後、リクエストプールは破棄されるため、直後に r を NULL に設定して不正アクセスを防止する不変条件がある。

アルゴリズムまたは振る舞い

1. 接続状態を SERVER_BUSY_READ に更新する。 2. ap_read_request を用いてリクエストを順次読み込むループを実行する。 3. リクエストが正常（HTTP_OK）であれば、接続状態を CONN_STATE_HANDLER に変更し、ap_process_request を実行してリクエストを処理する。 4. 処理後、Keep-Alive が有効か、または接続が中断されていないかを確認する。 5. 継続する場合、ap_mpm_query でサーバーの停止状態を確認し、ソケットのタイムアウトを keep_alive_timeout に再設定して次のリクエストを待機する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_core.c:249 (ap_process_http_connection 内)
  • **呼び出し先**:
  • ap_update_child_status_from_conn / ap_update_child_status
  • ap_read_request
  • ap_process_request
  • ap_mpm_query
  • ap_get_conn_socket
  • apr_socket_opt_set / apr_socket_timeout_set

関連データ構造

  • conn_rec: 接続のメタデータおよび状態を保持。
  • request_rec: 個別のHTTPリクエスト情報。
  • conn_state_t: 接続の現在のステートマシン状態。

失敗モードと境界条件

  • ap_read_request が NULL を返した場合、ループを終了し接続を閉じる。
  • c->keepalive が AP_CONN_KEEPALIVE でない場合、または c->aborted が真の場合、ループを中断する。
  • ap_mpm_query がエラーを返す、またはサーバーが AP_MPMQ_STOPPING 状態の場合はループを終了する。

保守担当者向けメモ

  • 本関数は同期処理専用である。非同期MPMの場合は ap_process_http_async_connection が使用される。
  • keep_alive_timeout の決定には、サーバー設定の優先順位（仮想ホスト設定の有無）が考慮されている。
  • r = NULL の代入は、リクエストプール解放後の誤った参照を即座にセグメンテーション違反として検知するための防御的実装である。
