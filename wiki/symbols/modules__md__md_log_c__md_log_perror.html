md_log_perror

定義

  • 種別: function
  • 定義箇所: modules/md/md_log.c:68

目的

mod_md モジュール全体で使用される、抽象化されたログ出力インターフェースのメイン関数です。可変長引数を受け取り、登録済みのログ出力バックエンドへ処理を転送します。

シグネチャまたは形状

void md_log_perror(const char *file, int line, md_log_level_t level, 
                   apr_status_t rv, apr_pool_t *p, const char *fmt, ...)

入力と出力

  • file: ログ出力元のソースファイル名。
  • line: ログ出力元の行番号。
  • level: md_log_level_t 型のログレベル。
  • rv: 関連する apr_status_t エラーコード。
  • p: ログ出力に関連付けられた apr_pool_t。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • 戻り値: なし（void）。

前提条件と不変条件

  • md_log_set を介してログ出力バックエンド（log_printv）が設定されている必要があります。
  • va_list を使用するため、呼び出し側は標準的な可変長引数の規約に従う必要があります。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リスト ap を初期化します。 2. 静的変数 log_printv が設定されているか確認します。 3. 設定されている場合、引数 file, line, level, rv, log_baton（内部保持されたコンテキスト）、p, fmt, ap を log_printv コールバックへ渡して実行します。 4. va_end を呼び出して引数リストを終了します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_acme.c 内の各関数（例: md_acme.c:195, 199, 216, 247, 284, 295, 306, 347, 377, 394, 399, 403）。
  • 呼び出し先: log_printv（md_log_set で登録された関数ポインタ）。

関連データ構造

  • md_log_level_t: ログレベルを定義する列挙型。
  • apr_pool_t: メモリ管理およびコンテキスト保持に使用される APR プール。
  • log_printv: ログ出力の実装を保持する静的関数ポインタ。
  • log_baton: ログ出力時に渡される不透明なコンテキストデータ。

失敗モードと境界条件

  • log_printv が NULL の場合、ログ出力は行われず、関数は何もせずに終了します。

保守担当者向けメモ

  • 本関数はログ出力の抽象化層であり、実際の出力先（ファイル、コンソール等）は実行時に md_log_set によって注入されます。
  • LOG_BUFFER_LEN が定義されていますが、本関数内では使用されていません。将来的な拡張の余地があります。
