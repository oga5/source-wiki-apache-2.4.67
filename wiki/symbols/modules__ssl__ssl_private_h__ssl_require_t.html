ssl_require_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:534

目的

SSL/TLS 接続において、特定のアクセス制御条件（SSLRequire ディレクティブ等で指定される式）を保持および評価するための構造体です。

シグネチャまたは形状

typedef struct {
    const char     *cpExpr;        /* 式の文字列形式 */
    ap_expr_info_t *mpExpr;        /* コンパイル済みの式情報 */
} ssl_require_t;

入力と出力

  • 入力: 設定読み込み時に文字列形式の式（cpExpr）が提供され、解析後にコンパイル済み式（mpExpr）が格納されます。
  • 出力: 接続時のアクセス制御判定において、mpExpr が評価に使用されます。

前提条件と不変条件

  • mpExpr は Apache の ap_expr API によってコンパイルされた有効な式情報である必要があります。
  • cpExpr は設定ファイルから読み込まれた文字列を保持します。

アルゴリズムまたは振る舞い

  • 設定フェーズで apr_array_make を用いて配列として確保され、各要素に条件式が格納されます。
  • 実行フェーズ（ssl_engine_kernel.c）において、配列から順次 ssl_require_t を取り出し、格納された式を評価することでアクセス可否を判定します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/ssl/ssl_engine_config.c: 設定の初期化および配列への追加。
  • modules/ssl/ssl_engine_kernel.c: 接続時のアクセス制御判定処理。
  • 呼び出し先:
  • ap_expr API（ap_expr_info_t を介した評価処理）。

関連データ構造

  • ap_expr_info_t: Apache HTTP Server の式評価エンジンで使用される構造体。
  • apr_array_header_t: ssl_require_t のリストを保持するために使用されます。

失敗モードと境界条件

  • 式のコンパイルに失敗した場合、mpExpr が適切に初期化されない可能性があります。
  • 配列が空の場合、アクセス制御の評価はスキップされます。

保守担当者向けメモ

  • 本構造体は mod_ssl のアクセス制御ロジックの核となります。式評価の挙動を変更する場合は、ap_expr API の仕様と ssl_engine_kernel.c での評価タイミングを併せて確認してください。
