create_server_config

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_memcache.c:365

目的

mod_socache_memcache モジュールで使用されるサーバー単位の設定構造体（socache_mc_svr_cfg）を初期化し、デフォルト値を設定します。

シグネチャまたは形状

static void *create_server_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定情報の割り当てに使用するメモリプール。
  • s: 設定対象のサーバーレコード（本関数内では未使用）。
  • **出力**:
  • 初期化済みの socache_mc_svr_cfg 構造体へのポインタを void * として返します。

前提条件と不変条件

  • apr_pcalloc により、構造体はゼロクリアされた状態で割り当てられます。
  • MC_DEFAULT_SERVER_TTL が有効な値として定義されている必要があります。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、メモリプール p から socache_mc_svr_cfg 構造体分のメモリを確保します。 2. 確保した構造体の ttl メンバに、デフォルト値である MC_DEFAULT_SERVER_TTL を代入します。 3. 初期化された構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の設定フレームワーク（server/config.c 等）から、モジュール設定の初期化時に呼び出されます。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ）。

関連データ構造

  • socache_mc_svr_cfg: 本関数が初期化するサーバー設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させるか、プロセスを終了させる可能性があります（APR のメモリ管理規約に依存）。

保守担当者向けメモ

  • 本関数は module 構造体の create_server_config フックとして登録されることを想定しています。
  • 構造体のメンバが増えた場合は、apr_pcalloc によるゼロ初期化後のフィールド設定を適宜追加してください。
