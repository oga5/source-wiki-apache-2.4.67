session_dbd_load

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_dbd.c:167

目的

リクエストに関連付けられたセッションをデータベースから読み込み、session_rec 構造体として復元する。

シグネチャまたは形状

static apr_status_t session_dbd_load(request_rec * r, session_rec ** z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報。
  • z: 復元されたセッションを格納するためのポインタのポインタ。
  • **出力**:
  • apr_status_t: 成功時は OK または APR_SUCCESS、失敗時はエラーコード、対象外の場合は DECLINED を返す。
  • z: 成功時に session_rec 構造体へのポインタが設定される。

前提条件と不変条件

  • mod_dbd がロードされ、データベース接続が利用可能であること。
  • 設定（session_dbd_dir_conf）に基づき、クッキー名またはユーザー名が特定できること。
  • セッションデータはリクエストの生存期間（r->pool）に依存して管理される。

アルゴリズムまたは振る舞い

1. **設定取得**: r->per_dir_config から session_dbd_dir_conf を取得。 2. **キー特定**: 設定に基づき、クッキー名またはユーザー名をセッションキーとして特定。特定できない場合は DECLINED を返す。 3. **キャッシュ確認**: r->main->notes に該当セッションが既に存在するか確認し、あればそれを返して終了。 4. **DB読み込み**:

  • 匿名セッションの場合: ap_cookie_read でクッキーを読み取り、dbd_load を実行。
  • ユーザー紐付けセッションの場合: r->user をキーとして dbd_load を実行。

    • 5. **セッション生成**: データベースから取得した値を用いて session_rec を初期化。UUID が含まれる場合は解析して格納。 6. **キャッシュ保存**: 取得した session_rec を r->main->notes に保存し、再パースを回避。 7. **ヘッダー設定**: Cache-Control: no-cache, private を追加し、セッションを含むページのキャッシュを抑制。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_dbd.c:635 にて ap_hook_session_load に登録されている。
  • **呼び出し先**: ap_get_module_config, apr_pstrcat, apr_table_get, ap_cookie_read, dbd_load, apr_pcalloc, apr_table_make, apr_uuid_parse, apr_table_setn, apr_table_addn。

関連データ構造

  • session_rec: セッション情報を保持する構造体。
  • session_dbd_dir_conf: ディレクトリ単位の DBD 設定。

失敗モードと境界条件

  • **DECLINED**: 設定でキーが特定できない場合や、セッション対象外のリクエストの場合。
  • **エラー**: dbd_load が失敗した場合、そのエラーコードをそのまま返す。
  • **未ログイン**: peruser 設定時に r->user が存在しない場合、セッションは読み込まれない。

保守担当者向けメモ

  • 本関数は r->main->notes を利用して同一リクエスト内での重複読み込みを防止している。
  • dbd_load の結果が APR_SUCCESS 以外の場合、処理は中断される。
  • セッションの生存期間は r->pool に紐付いているため、リクエスト終了後にセッションデータへアクセスしないこと。
