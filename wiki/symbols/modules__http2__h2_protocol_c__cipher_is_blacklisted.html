cipher_is_blacklisted

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_protocol.c:417

目的

指定された暗号スイート（Cipher Suite）が、HTTP/2 プロトコルにおいて禁止されているブラックリストに含まれているかを判定します。

シグネチャまたは形状

static int cipher_is_blacklisted(const char *cipher, const char **psource)

入力と出力

  • **入力**:
  • cipher: チェック対象の暗号スイート名（文字列）。
  • psource: ブラックリストのソース情報を格納するためのポインタ。
  • **出力**:
  • 戻り値: ブラックリストに含まれている場合は 1（真）、含まれていない場合は 0（偽）を返します。
  • psource: ブラックリストに存在する場合、そのソース情報が格納されます。

前提条件と不変条件

  • BLCNames が初期化済みである必要があります（h2_protocol_init 内の cipher_init で構築されるハッシュテーブル）。
  • cipher 引数は有効な文字列である必要があります。

アルゴリズムまたは振る舞い

1. apr_hash_get を使用し、グローバルなハッシュテーブル BLCNames から cipher をキーとして検索します。 2. 検索結果を *psource に代入します。 3. 検索結果が非 NULL であれば 1 を、NULL であれば 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_protocol.c:469 (h2_protocol_is_acceptable_c1 関数内)
  • **呼び出し先**: apr_hash_get (APR ライブラリ関数)

関連データ構造

  • BLCNames: ブラックリストを保持するハッシュテーブル（apr_hash_t）。

失敗モードと境界条件

  • cipher が NULL の場合、apr_hash_get の挙動に依存します。
  • ブラックリストが空の場合、常に 0 を返します。

保守担当者向けメモ

  • 本関数は RFC 7540 Appendix A に基づく禁止暗号リストを参照します。新しい脆弱性が報告された際は、cipher_init で構築される BLCNames を更新してください。
