session_cookie_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/session/mod_session_cookie.c:30

目的

mod_session_cookie モジュールにおいて、ディレクトリ単位のセッションCookie設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const char *name;        /* Cookie名 */
    int name_set;            /* nameが設定されているか */
    const char *name_attrs;  /* Cookie属性文字列 */
    const char *name2;       /* Cookie2名 */
    int name2_set;           /* name2が設定されているか */
    const char *name2_attrs; /* Cookie2属性文字列 */
    int remove;              /* セッション削除フラグ */
    int remove_set;          /* removeが設定されているか */
} session_cookie_dir_conf;

入力と出力

本構造体は設定値の保持を目的としており、関数ではありません。設定ハンドラ（set_cookie_name 等）を通じて値が入力され、session_cookie_load や session_cookie_save などのフック関数がこの構造体を参照して動作を決定します。

前提条件と不変条件

  • すべてのメンバは Apache のプールメモリ（apr_pool_t）上で管理されます。
  • name および name2 に = や & を含めることはできません（check_string による検証）。

アルゴリズムまたは振る舞い

  • create_session_cookie_dir_config によりメモリが確保され、初期化されます。
  • merge_session_cookie_dir_config により、親コンテキストの設定が子コンテキストへ継承・マージされます。
  • 各フック関数は ap_get_module_config(r->per_dir_config, ...) を介して、現在のリクエストに対する設定を取得します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_session_cookie.c 内の各設定ハンドラ、フック関数（session_cookie_load, session_cookie_save）、および設定マージ関数から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • session_rec: mod_session が管理するセッションデータ本体。
  • per_dir_config: Apache のリクエストごとのディレクトリ設定コンテキスト。

失敗モードと境界条件

  • 設定値の検証（check_string）に失敗した場合、モジュールはエラーを返します。
  • メモリ確保失敗時は apr_pcalloc が例外を発生させる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_session_cookie の設定管理の要です。新しいCookie属性を追加する場合は、本構造体へのメンバ追加と、merge_session_cookie_dir_config での適切なマージロジックの実装が必要です。
