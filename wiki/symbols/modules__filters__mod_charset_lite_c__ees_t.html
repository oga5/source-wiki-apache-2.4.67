ees_t

定義

  • 種別: enum
  • 定義箇所: modules/filters/mod_charset_lite.c:58

目的

文字セット変換フィルタ処理中に発生したエラー状態を保持・識別するための拡張エラーコード定義です。

シグネチャまたは形状

typedef enum {
    EES_INIT = 0,
    EES_LIMIT,
    EES_INCOMPLETE_CHAR,
    EES_BUCKET_READ,
    EES_DOWNSTREAM,
    EES_BAD_INPUT
} ees_t;

入力と出力

  • 入力: 該当なし（列挙型定義）。
  • 出力: フィルタ処理コンテキスト内のエラー状態変数として利用されます。

前提条件と不変条件

  • EES_INIT は初期化を容易にするため、値が 0 である必要があります。

アルゴリズムまたは振る舞い

  • EES_INIT: エラー情報が未設定の状態。
  • EES_LIMIT: 組み込みの制限（例：8バイトを超える文字の跨ぎ）に抵触した場合。
  • EES_INCOMPLETE_CHAR: コンテンツ末尾に不完全なマルチバイト文字が存在する場合。
  • EES_BUCKET_READ: バケット読み込み時のエラー。
  • EES_DOWNSTREAM: 下流フィルタで発生したエラー。
  • EES_BAD_INPUT: 入力データが不正な場合。

呼び出し元と呼び出し先

  • 呼び出し元: mod_charset_lite.c 内のフィルタ処理ロジックが、エラー状態を記録するために使用します。
  • 呼び出し先: 情報なし。

関連データ構造

  • charset_filter_ctx_t: フィルタインスタンスごとのコンテキスト内で、ees フィールドとしてこの型が保持されます。

失敗モードと境界条件

  • EES_LIMIT は、FATTEST_CHAR（8バイト）を超える文字がバケット境界を跨ぐ際に発生します。

保守担当者向けメモ

  • 本シンボルは mod_charset_lite.c におけるエラーハンドリングの基盤です。新しいエラー状態を追加する場合は、既存の列挙値との整合性に注意してください。
