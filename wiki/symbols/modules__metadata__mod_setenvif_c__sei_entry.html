sei_entry

定義

  • **種別**: struct
  • **定義箇所**: modules/metadata/mod_setenvif.c:106

目的

SetEnvIf ディレクティブによって定義された個別の条件ルールを保持するための構造体です。HTTP リクエストのヘッダや属性を評価し、条件に合致した場合に環境変数を設定・解除するために使用されます。

シグネチャまたは形状

typedef struct {
    char *name;                          /* ヘッダ名 */
    ap_regex_t *pnamereg;                /* コンパイル済みヘッダ名正規表現 */
    char *regex;                         /* マッチ対象の正規表現 */
    ap_regex_t *preg;                    /* コンパイル済み正規表現 */
    const apr_strmatch_pattern *pattern; /* 非正規表現の高速マッチング用パターン */
    ap_expr_info_t *expr;                /* 解析済み式 */
    apr_table_t *features;               /* 設定・解除する環境変数のテーブル */
    enum special special_type;           /* 特殊ヘッダフラグ */
    int icase;                           /* 大文字小文字を無視するか */
} sei_entry;

入力と出力

本構造体は設定フェーズで生成され、実行フェーズでリクエスト情報と比較されます。

  • **入力**: SetEnvIf ディレクティブの引数（ヘッダ名、正規表現、環境変数名など）。
  • **出力**: 評価結果に基づき、リクエストの subprocess_env テーブルが更新されます。

前提条件と不変条件

  • 正規表現を使用する場合、preg には ap_pregcomp でコンパイルされた有効なオブジェクトが格納されます。
  • 単純文字列マッチングの場合、pattern に apr_strmatch_precompile で生成されたパターンが格納されます。
  • features テーブルは、環境変数の設定状態を管理するために使用されます。

アルゴリズムまたは振る舞い

1. **構成**: add_setenvif_core 関数により、ディレクティブから解析された情報が本構造体に格納されます。 2. **評価**: match_headers 関数がリクエスト処理の初期段階で呼び出され、本構造体の各フィールド（name, regex, expr 等）を用いてリクエスト情報を検査します。 3. **マッチング**: apr_strmatch または ap_regexec を使用して条件判定を行い、成功時に features に基づいて環境変数を操作します。

呼び出し元と呼び出し先

  • **呼び出し元**: add_setenvif_core（設定時）、match_headers（実行時）。
  • **呼び出し先**: ap_pregcomp, apr_strmatch_precompile, ap_regexec, apr_table_setn 等の Apache/APR API。
  • ※ソースコード内の参照として modules/proxy/mod_proxy_fcgi.c:28 にも同名の構造体定義が存在しますが、本モジュールとの直接的な連携関係は明示されていません。

関連データ構造

  • sei_cfg_rec: sei_entry の配列を保持するコンテナ構造体。
  • apr_table_t: 環境変数の保持に使用。
  • ap_regex_t: 正規表現のコンパイル結果。
  • apr_strmatch_pattern: 高速文字列検索用パターン。

失敗モードと境界条件

  • 正規表現のコンパイル失敗時は、設定読み込み時にエラーが発生します。
  • リクエスト処理中にヘッダが存在しない場合や、式評価が失敗した場合はマッチングがスキップされます。
  • SetEnvIf はリクエスト処理の初期段階で実行されるため、後続のモジュールが動的に生成したヘッダには反応できない制約があります。

保守担当者向けメモ

  • メモリ管理は cmd->pool（設定時）および r->pool（リクエスト処理時）に依存しています。
  • 同一ヘッダに対する連続したルールは、ポインタ比較による最適化が行われているため、変更時はこのロジックに注意してください。
  • ap_expr の評価ロジックは ap_expr_exec_re に委譲されています。
