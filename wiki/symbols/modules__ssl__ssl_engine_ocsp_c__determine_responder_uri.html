determine_responder_uri

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_ocsp.c:54

目的

OCSP 検証に使用するレスポンダの URI を、サーバー設定または証明書の AIA (Authority Information Access) 拡張から特定し、解析済みの apr_uri_t 構造体として返します。

シグネチャまたは形状

static apr_uri_t *determine_responder_uri(SSLSrvConfigRec *sc, X509 *cert,
                                          conn_rec *c, apr_pool_t *p)

入力と出力

  • **入力**:
  • sc: サーバー設定構造体。
  • cert: 対象の X.509 証明書。
  • c: 現在の接続情報。
  • p: メモリ割り当て用プール。
  • **出力**:
  • 成功時: 解析済みの apr_uri_t 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • p は有効な apr_pool_t であること。
  • sc および c は NULL でないこと。
  • 返される URI は http スキームである必要がある。

アルゴリズムまたは振る舞い

1. **URI の優先順位決定**:

  • 設定で ocsp_force_default が有効な場合、設定値の ocsp_responder を使用。
  • それ以外の場合、extract_responder_uri を呼び出し証明書から取得を試みる。取得できない場合は設定値の ocsp_responder にフォールバックする。

    • 2. **検証と解析**:

  • URI が特定できない場合はログを出力し NULL を返す。
  • apr_uri_parse を使用して文字列を解析。解析失敗またはホスト名が欠落している場合は NULL を返す。
  • スキームが http 以外の場合はエラーとして NULL を返す。

    • 3. **ポート補完**:

  • ポート番号が指定されていない場合、apr_uri_port_of_scheme を使用して http のデフォルトポート（80）を補完する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_ocsp.c:140 (verify_ocsp_status 内)
  • **呼び出し先**:
  • extract_responder_uri (証明書からの URI 抽出)
  • apr_uri_parse (URI 文字列の解析)
  • ap_log_cerror (エラーログ出力)
  • apr_uri_port_of_scheme (デフォルトポートの取得)

関連データ構造

  • SSLSrvConfigRec: サーバーの OCSP 設定を保持。
  • apr_uri_t: 解析された URI 情報を格納。
  • X509: OpenSSL の証明書オブジェクト。

失敗モードと境界条件

  • **URI 抽出失敗**: 証明書に AIA 拡張がなく、かつデフォルトの設定値も存在しない場合は NULL を返す。
  • **解析失敗**: apr_uri_parse が失敗、またはホスト名が含まれない場合は NULL を返す。
  • **スキーム不一致**: http 以外のスキーム（https 等）が指定された場合はエラーとして NULL を返す。

保守担当者向けメモ

  • 本関数は同期的に動作するため、ネットワーク I/O を伴う後続処理の前に URI の妥当性を検証する役割を担います。
  • apr_palloc を使用しているため、呼び出し元で提供されるプール p のライフサイクルに依存します。
