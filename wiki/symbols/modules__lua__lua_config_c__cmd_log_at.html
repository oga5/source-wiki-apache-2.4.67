cmd_log_at

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:159

目的

Lua スクリプトから Apache のログシステムへ、現在のスクリプト実行コンテキスト（ファイル名および行番号）を付与したログメッセージを出力する。

シグネチャまたは形状

static int cmd_log_at(lua_State *L, int level)

入力と出力

  • **入力**:
  • L: Lua 仮想マシンの状態。スタックの第1引数に cmd_parms オブジェクト、第2引数にログメッセージ文字列を期待する。
  • level: 出力するログのレベル（APLOG_EMERG 等）。
  • **出力**:
  • 戻り値: 常に 0 を返す。

前提条件と不変条件

  • スタックの第1引数は check_cmd_parms で検証可能な cmd_parms 型の userdata であること。
  • スタックの第2引数は luaL_checkstring で取得可能な文字列であること。

アルゴリズムまたは振る舞い

1. check_cmd_parms を呼び出し、スタックから cmd_parms 構造体を取得する。 2. lua_getstack および lua_getinfo を使用して、現在の Lua 実行スタックからソースファイル名と行番号を取得する。 3. luaL_checkstring でログメッセージを取得する。 4. ap_log_error を呼び出し、取得したコンテキスト情報とメッセージを Apache のログパイプラインへ渡す。この際、APLOGNO マクロは使用されない。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_config.c 内の各ログレベル用ラッパー関数（cmd_emerg, cmd_alert, cmd_crit, cmd_err, cmd_warn, cmd_notice, cmd_info, cmd_debug, cmd_trace1 から cmd_trace4）。
  • **呼び出し先**:
  • check_cmd_parms (Lua スタックからの型チェック)
  • lua_getstack / lua_getinfo (Lua デバッグ API)
  • luaL_checkstring (Lua 文字列取得)
  • ap_log_error (Apache ログ出力 API)

関連データ構造

  • cmd_parms: Apache のコマンドパラメータ構造体。
  • lua_State: Lua 仮想マシンの状態。
  • lua_Debug: Lua のデバッグ情報を保持する構造体。

失敗モードと境界条件

  • スタック引数が不足している場合や型が一致しない場合、check_cmd_parms または luaL_checkstring が Lua のエラーを発生させ、処理が中断される。

保守担当者向けメモ

  • APLOGNO マクロが意図的に省略されているため、ログの追跡性を高める必要がある場合は、将来的にログメッセージのフォーマットや識別子を検討すること。
  • lua_getinfo の呼び出しは、Lua のデバッグ情報が有効な場合にのみ正確なファイル名と行番号を返却する。
