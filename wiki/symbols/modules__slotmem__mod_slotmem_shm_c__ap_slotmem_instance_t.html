ap_slotmem_instance_t

定義

  • 種別: struct
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:44

目的

共有メモリベースの固定長スロット管理インスタンスを保持する管理構造体です。メモリセグメントのメタデータ、利用状況フラグ、およびデータ領域へのポインタを統合し、プロセス間でのスロット操作を可能にします。

シグネチャまたは形状

struct ap_slotmem_instance_t {
    char                 *name;       /* ファイルベースのSHMパス/名前 */
    char                 *pname;      /* 永続化ファイルパス/名前 */
    int                  fbased;      /* ファイルベースか否か */
    void                 *shm;        /* メモリセグメントへのポインタ (apr_shm_t *) */
    void                 *base;       /* データセットの開始位置 */
    apr_pool_t           *gpool;      /* セグメントごとのプール（世代交代でクリア） */
    char                 *inuse;      /* 利用状況フラグテーブル */
    unsigned int         *num_free;   /* このインスタンスの空きスロット数 */
    void                 *persist;    /* 永続化データセットの開始位置 */
    const sharedslotdesc_t *desc;     /* スロットごとの記述子 */
    struct ap_slotmem_instance_t  *next; /* 次の割り当て済みセグメントの位置 */
};

入力と出力

本構造体は、mod_slotmem_shm の各関数（slotmem_create, slotmem_attach, slotmem_get, slotmem_put 等）において、操作対象のインスタンスを特定するためのコンテキストとして入力されます。

前提条件と不変条件

  • gpool はインスタンスの生存期間を管理し、プロセス終了時のクリーンアップに使用されます。
  • 共有メモリは [desc] [num_free] [data_slots] [inuse_array] の順でレイアウトされることが前提です。
  • AP_SLOTMEM_OFFSET 等の定数により、メモリ内の各フィールドのアライメントが厳密に管理されています。

アルゴリズムまたは振る舞い

  • **インスタンス管理**: next ポインタにより、複数の共有メモリセグメントを連結して管理します。
  • **データアクセス**: base を基準点とし、スロットIDからオフセット計算を行うことで、特定のデータ領域への memcpy を実行します。
  • **ライフサイクル**: gpool を通じて、サーバーの再起動やMPMの世代交代に伴うメモリの解放・再確保を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_slotmem_shm.c 内のプロバイダー関数群（slotmem_create, slotmem_attach, slotmem_get, slotmem_put 等）が本構造体を操作します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • sharedslotdesc_t: スロットの構成情報（サイズ、個数、型）を保持します。
  • apr_shm_t: shm フィールドが指す共有メモリセグメントの基盤構造体です。

失敗モードと境界条件

  • **整合性エラー**: restore_slotmem 実行時、MD5チェックサムが一致しない場合はデータが破損しているとみなされます。
  • **メモリ不足**: apr_shm_create が失敗した場合、インスタンスの初期化が中断されます。

保守担当者向けメモ

  • 本構造体は mod_slotmem_shm.c と ap_slotmem.h の両方で定義されています。変更時は両方の整合性に注意してください。
  • is_child_process フラグ（WIN32環境等）との相互作用により、プロセス継承時のメモリ再アタッチ挙動が変化するため、プラットフォーム固有の挙動に留意してください。
