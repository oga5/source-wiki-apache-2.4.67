cgid_suexec_id_doer

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgid.c:113

目的

リクエストに関連付けられた suexec 実行用のユーザーおよびグループ識別情報（ap_unix_identity_t）を取得します。

シグネチャまたは形状

static ap_unix_identity_t *cgid_suexec_id_doer(const request_rec *r)

入力と出力

  • **入力**: r (現在のリクエスト情報を保持する request_rec 構造体へのポインタ)
  • **出力**: リクエスト設定から取得した ap_unix_identity_t 構造体へのポインタ。設定が存在しない場合は NULL が返される可能性があります。

前提条件と不変条件

  • r->request_config に cgid_module に関連付けられた設定データが格納されている必要があります。

アルゴリズムまたは振る舞い

1. ap_get_module_config を呼び出し、リクエスト設定 (r->request_config) から cgid_module に紐付いた設定情報を取得します。 2. 取得した設定を ap_unix_identity_t 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgid.c:736 にて ap_hook_get_suexec_identity に登録されています。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core API)

関連データ構造

  • request_rec: リクエスト処理のコンテキストを保持する構造体。
  • ap_unix_identity_t: suexec 実行時に使用されるユーザーID、グループID、およびユーザー名を保持する構造体。
  • cgid_module: 本モジュールの設定を識別するためのモジュール構造体。

失敗モードと境界条件

  • r->request_config が初期化されていない場合や、該当するモジュール設定がリクエストに付与されていない場合、ap_get_module_config は NULL を返し、結果として本関数も NULL を返します。

保守担当者向けメモ

  • 本関数は suexec 実行時の権限解決フックとして機能します。
  • 戻り値が有効な ugid を指していることが期待される箇所（mod_cgid.c:951）があるため、呼び出し側での NULL チェックが重要です。
