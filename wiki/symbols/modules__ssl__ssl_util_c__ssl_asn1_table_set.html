ssl_asn1_table_set

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util.c:199

目的

EVP_PKEY 構造体を DER 形式に変換し、指定されたハッシュテーブルにキャッシュとして格納または更新します。

シグネチャまたは形状

ssl_asn1_t *ssl_asn1_table_set(apr_hash_t *table, const char *key, EVP_PKEY *pkey)

入力と出力

  • **入力**:
  • table: 格納先となる apr_hash_t ハッシュテーブル。
  • key: キャッシュのキーとなる文字列。
  • pkey: DER 形式へ変換する対象の EVP_PKEY 構造体。
  • **出力**:
  • 格納されたデータを含む ssl_asn1_t 構造体へのポインタを返します。

前提条件と不変条件

  • key は strlen で長さを算出可能なヌル終端文字列である必要があります。
  • pkey は有効な EVP_PKEY 構造体である必要があります。
  • ssl_asn1_t 構造体の cpData メンバは malloc で確保されるため、呼び出し側で適切に管理（解放）する必要があります。

アルゴリズムまたは振る舞い

1. key に基づき、ハッシュテーブルから既存の ssl_asn1_t エントリを検索します。 2. i2d_PrivateKey を使用して、pkey を DER 形式に変換するために必要なバイト数を算出します。 3. エントリが既に存在する場合:

  • 既存のデータサイズと新しいサイズを比較し、必要に応じて ap_realloc でバッファを拡張します。

    • 4. エントリが存在しない場合:

  • ap_malloc で ssl_asn1_t 構造体とデータ用バッファを新規確保し、ハッシュテーブルに登録します。

    • 5. 確保したバッファに i2d_PrivateKey を用いて実際の DER データを書き込みます。 6. 更新された ssl_asn1_t 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_pphrase.c:359
  • **呼び出し先**:
  • i2d_PrivateKey (OpenSSL API)
  • apr_hash_get / apr_hash_set (APR API)
  • ap_malloc / ap_realloc (Apache API)

関連データ構造

  • ssl_asn1_t: ASN.1 データを保持する構造体。cpData（データ本体）、nData（サイズ）、source_mtime を含む。
  • apr_hash_t: キャッシュを管理するためのハッシュテーブル。
  • EVP_PKEY: OpenSSL の公開鍵/秘密鍵構造体。

失敗モードと境界条件

  • メモリ確保 (ap_malloc / ap_realloc) が失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性があります。
  • i2d_PrivateKey が失敗した場合の戻り値チェックは本関数内には含まれていません。

保守担当者向けメモ

  • 本関数で確保される cpData はプールメモリではなく malloc で確保されるため、モジュールのリロードを跨いで保持されます。メモリリークを防ぐため、不要になった際は必ず ssl_asn1_table_unset 等で解放してください。
