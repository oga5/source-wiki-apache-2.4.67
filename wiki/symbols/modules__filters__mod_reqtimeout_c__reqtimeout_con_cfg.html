reqtimeout_con_cfg

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_reqtimeout.c:58

目的

接続ごとのタイムアウト状態を保持し、リクエスト処理の各段階における期限管理と一時的なデータバッファリングを行うための構造体です。

シグネチャまたは形状

typedef struct {
    apr_time_t timeout_at;        /* 現在のタイムアウト期限 */
    apr_time_t max_timeout_at;    /* 許容される最大タイムアウト期限 */
    reqtimeout_stage_t cur_stage; /* 現在の処理段階 */
    int in_keep_alive;            /* Keep-Alive 状態フラグ */
    char *type;                   /* タイムアウトの種類を示す文字列 */
    apr_socket_t *socket;         /* 監視対象のソケット */
    apr_bucket_brigade *tmpbb;    /* 行読み取り用の一時バッファ */
} reqtimeout_con_cfg;

入力と出力

  • 本構造体は接続プール（c->pool）から割り当てられ、接続のライフサイクルを通じて状態を保持します。
  • extend_timeout 関数などの入力として渡され、タイムアウト期限の更新やチェックに使用されます。

前提条件と不変条件

  • 接続初期化時に reqtimeout_init によってフィルタコンテキストとして割り当てられます。
  • 接続終了まで生存し、reqtimeout_eor によってリクエスト間の状態リセットが行われます。

アルゴリズムまたは振る舞い

  • reqtimeout_filter 内で、現在の時刻と timeout_at を比較し、期限を超過している場合に接続を強制終了します。
  • extend_timeout を通じて、受信したデータ量に応じて timeout_at を動的に延長します。
  • tmpbb は AP_MODE_GETLINE モードでの行単位の読み取りにおいて、データを一時的に保持するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: reqtimeout_init（初期化）、reqtimeout_filter（フィルタ処理）、extend_timeout（期限延長）、check_time_left（期限チェック）などで参照・操作されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • reqtimeout_stage_t: 処理段階を定義する列挙型。
  • apr_bucket_brigade: tmpbb フィールドで使用されるデータ保持用構造体。
  • apr_socket_t: 監視対象のソケット。

失敗モードと境界条件

  • check_time_left が APR_TIMEUP を返した場合、接続は AP_CONN_CLOSE に設定され、即時切断されます。
  • AP_MODE_GETLINE 以外のモードでは、タイムアウトの延長処理は行われません。

保守担当者向けメモ

  • tmpbb は接続プールに紐付いているため、メモリリークを防ぐために適切なブリゲードのクリアが必要です。
  • APR_MAJOR_VERSION < 2 の環境では apr_poll を、それ以外では apr_socket_wait を使用する実装となっており、プラットフォーム依存の挙動に注意してください。
