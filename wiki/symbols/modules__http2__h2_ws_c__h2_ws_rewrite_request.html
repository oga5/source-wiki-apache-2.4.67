h2_ws_rewrite_request

定義

  • **種別**: function
  • **定義箇所**: modules/http2/h2_ws.c:76

目的

HTTP/2 の拡張 CONNECT メソッドを、内部的な HTTP/1.1 形式の GET リクエストに変換し、WebSocket ハンドシェイクを処理するための出力フィルタを挿入する。

シグネチャまたは形状

const h2_request *h2_ws_rewrite_request(const h2_request *req,
                                        conn_rec *c2, int no_body)

入力と出力

  • **入力**:
  • req: 変換対象の h2_request オブジェクト。
  • c2: 処理対象の conn_rec（子接続）。
  • no_body: ボディの有無を示すフラグ。
  • **出力**:
  • 変換後の h2_request オブジェクト。変換が不要な場合やエラー時は元の req または NULL を返す。

前提条件と不変条件

  • req->protocol が "websocket" であること。
  • req->method が "CONNECT" であること。
  • req->scheme および req->path が存在すること。
  • 処理は c2->pool を使用してメモリ管理を行う。

アルゴリズムまたは振る舞い

1. **検証**: プロトコル、メソッド、必須フィールド（scheme, path）をチェックし、条件を満たさない場合は入力をそのまま返す。 2. **変換**: h2_request_clone でリクエストを複製し、メソッドを "GET" に変更、Upgrade: websocket および Connection: Upgrade ヘッダーを追加する。 3. **キー生成**: Sec-WebSocket-Key を生成し、リクエストヘッダーに設定する。 4. **フィルタ設定**: gen_ws_accept で期待される Sec-WebSocket-Accept 値を計算し、ws_filter_ctx に格納する。 5. **フィルタ挿入**: H2_C2_WS_OUT フィルタを H2_C2_NET_OUT の直前に挿入する。 6. **状態更新**: conn_ctx->is_upgrade を 1 に設定し、アップグレード接続であることをマークする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2.c:741
  • modules/http2/h2_c2_filter.c:118
  • **呼び出し先**:
  • h2_request_clone
  • apr_generate_random_bytes
  • apr_pencode_base64_binary
  • gen_ws_accept
  • ap_remove_output_filter_byhandle
  • ap_add_output_filter

関連データ構造

  • h2_request: HTTP/2 リクエストの内部表現。
  • ws_filter_ctx: フィルタの実行状態を保持する構造体。
  • h2_conn_ctx_t: 接続ごとのコンテキスト（is_upgrade フラグ等）。

失敗モードと境界条件

  • Sec-WebSocket-Key の生成に失敗した場合、NULL を返す。
  • 必要なヘッダー（scheme, path）が欠落している場合、変換を行わず元のリクエストを返す。
  • H2_USE_WEBSOCKETS マクロが未定義の場合、本関数は実質的に NOP として動作する（モジュール概要より）。

保守担当者向けメモ

  • 本関数は WebSocket ハンドシェイクの変換に特化しており、実際のデータフレーム転送は行わない。
  • フィルタの挿入順序（H2_C2_WS_OUT が H2_C2_NET_OUT の前）は、レスポンスのインターセプトを確実にするために重要である。
