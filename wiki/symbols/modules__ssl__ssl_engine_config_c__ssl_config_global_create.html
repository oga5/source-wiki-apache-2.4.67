ssl_config_global_create

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_config.c:42

目的

Apache HTTP Server の生存期間中保持されるグローバルな SSL 設定構造体（SSLModConfigRec）を生成・初期化します。

シグネチャまたは形状

SSLModConfigRec *ssl_config_global_create(server_rec *s)

入力と出力

  • **入力**: server_rec *s（サーバー設定レコード）
  • **出力**: SSLModConfigRec *（初期化済みのグローバル設定構造体）

前提条件と不変条件

  • s->process->pool を使用してメモリを割り当て、サーバーの再起動を跨いで生存することを前提としています。
  • SSL_MOD_CONFIG_KEY をキーとして、プール内に既に設定が存在する場合はそれを再利用します。

アルゴリズムまたは振る舞い

1. apr_pool_userdata_get を呼び出し、既存の SSLModConfigRec があればそれを返します。 2. 存在しない場合、apr_palloc で構造体を割り当てます。 3. pPool にサーバープロセスプールを格納し、bFixed を FALSE に設定します。 4. セッションキャッシュモード、乱数シード用配列、仮想ホストキーハッシュ、秘密鍵ハッシュなどの各フィールドを初期化します。 5. コンパイルオプション（HAVE_TLSEXT, HAVE_OCSP_STAPLING, HAVE_FIPS 等）に応じて、関連するフィールドを初期値に設定します。 6. apr_pool_userdata_set を使用して、生成した構造体をプールに登録し、次回以降の呼び出しで再利用可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_config.c:251
  • modules/ssl/ssl_engine_init.c:377
  • **呼び出し先**:
  • apr_pool_userdata_get / apr_pool_userdata_set
  • apr_palloc
  • apr_array_make
  • apr_hash_make

関連データ構造

  • SSLModConfigRec: グローバルな SSL 設定を保持する構造体。
  • server_rec: Apache HTTP Server のサーバー設定レコード。

失敗モードと境界条件

  • メモリ割り当て失敗時は apr_palloc の挙動に従い、通常はプロセスが終了します。
  • SSL_MOD_CONFIG_KEY に紐付くデータが不正な型である場合のチェックは含まれていません。

保守担当者向けメモ

  • 本関数はサーバーの生存期間中一度だけ初期化されることを意図しています。
  • HAVE_ で囲まれたフィールドは、ビルド時の OpenSSL の機能や環境設定に依存するため、変更時は条件付きコンパイルの整合性に注意してください。
