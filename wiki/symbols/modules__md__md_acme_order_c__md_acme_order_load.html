md_acme_order_load

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:181

目的

永続化ストレージから ACME Order 情報を読み込み、メモリ上の md_acme_order_t 構造体として復元します。

シグネチャまたは形状

apr_status_t md_acme_order_load(struct md_store_t *store, md_store_group_t group, 
                                const char *md_name, md_acme_order_t **pauthz_set, 
                                apr_pool_t *p)

入力と出力

  • **入力**:
  • store: ストレージへのアクセスハンドル。
  • group: 読み込み対象のストレージグループ（例: MD_SG_STAGING）。
  • md_name: 対象ドメイン名。
  • p: メモリ割り当てに使用する APR プール。
  • **出力**:
  • pauthz_set: 成功時に読み込まれた md_acme_order_t へのポインタが格納されます。失敗時は NULL。
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • store は有効な md_store_t インスタンスである必要があります。
  • p は読み込まれたオブジェクトの生存期間を管理する有効なプールである必要があります。

アルゴリズムまたは振る舞い

1. md_store_load_json を呼び出し、指定されたグループとドメイン名に対応する JSON ファイル（MD_FN_ORDER）を読み込みます。 2. 読み込みが成功した場合、md_acme_order_from_json を呼び出して JSON データを md_acme_order_t 構造体に変換します。 3. 処理結果を pauthz_set に格納し、ステータスコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_order.c:238
  • modules/md/md_acmev2_drive.c:71
  • **呼び出し先**:
  • md_store_load_json
  • md_acme_order_from_json

関連データ構造

  • md_acme_order_t: ACME Order の状態を保持する構造体。
  • md_store_t: 永続化ストレージの抽象インターフェース。

失敗モードと境界条件

  • 指定されたパスにファイルが存在しない場合や、JSON のパースに失敗した場合は md_store_load_json からエラーが返され、本関数もそのエラーを伝播します。その際、pauthz_set は NULL に設定されます。

保守担当者向けメモ

  • 本関数は md_store を介したディスク I/O を伴います。
  • 読み込まれたオブジェクトのメモリは引数 p で指定されたプールから確保されます。
