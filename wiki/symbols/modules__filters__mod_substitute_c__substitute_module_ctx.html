substitute_module_ctx

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_substitute.c:64

目的

mod_substitute フィルタの実行中における状態保持およびメモリ管理を行うためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    apr_bucket_brigade *linebb;
    apr_bucket_brigade *linesbb;
    apr_bucket_brigade *passbb;
    apr_bucket_brigade *pattbb;
    apr_pool_t *tpool;
} substitute_module_ctx;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。フィルタ処理の過程で、各 apr_bucket_brigade メンバがバケットの蓄積、処理、転送のために使用されます。

前提条件と不変条件

  • tpool は、フィルタの各ループ終了時に apr_pool_clear を呼び出してメモリを再利用することが想定されています。
  • フィルタ処理の初回呼び出し時に生成され、フィルタコンテキスト（f->ctx）として保持されます。

アルゴリズムまたは振る舞い

  • linesbb, pattbb: 置換処理の過程で一時的に使用されるブリゲードです。
  • passbb: 処理済みで下流フィルタへ転送されるバケットを保持します。
  • tpool: 処理に伴う一時的なメモリ確保のためのプールです。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_substitute.c:381 (substitute_filter 関数内にて f->ctx として参照)
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • apr_bucket_brigade: Apache のバケット管理用データ構造。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • メモリ不足時やバッファ制限（SubstituteMaxLineLength）を超過した場合の挙動は、本構造体を使用する substitute_filter のロジックに依存します。

保守担当者向けメモ

  • tpool の定期的なクリアによりメモリリークを防ぐ設計となっています。
  • AP_MAX_BUCKETS を超えるバケット蓄積が発生した場合のフラッシュ処理が、本構造体を用いたフィルタの安全装置として機能します。
