bail_out_on_error

定義

  • 種別: function
  • 定義箇所: modules/http/http_filters.c:88

目的

HTTP フィルタ処理中に発生したプロトコル違反やリソース制限超過などのエラーに対し、エラーバケットの挿入と接続のクローズを行い、安全に処理を中断する。

シグネチャまたは形状

static apr_status_t bail_out_on_error(http_ctx_t *ctx, ap_filter_t *f, int http_error)

入力と出力

  • ctx: 入力フィルタの状態を保持する http_ctx_t 構造体。
  • f: 現在のフィルタコンテキスト。
  • http_error: 発生した HTTP エラーコード（例: HTTP_REQUEST_ENTITY_TOO_LARGE）。
  • 戻り値: プロキシリクエストの場合は対応する apr_status_t エラーコードを返し、それ以外の場合は ap_pass_brigade の実行結果を返す。

前提条件と不変条件

  • ctx->bb が有効なバケットブリゲードとして初期化されていること。
  • f->r が有効なリクエストオブジェクトを指していること。

アルゴリズムまたは振る舞い

1. ctx->bb を apr_brigade_cleanup で初期化する。 2. プロキシリクエスト（PROXYREQ_RESPONSE）の場合、http_error に応じて APR_ENOSPC、APR_INCOMPLETE、APR_ENOTIMPL、または APR_EGENERAL を即座に返す。 3. 通常のリクエストの場合:

  • ap_bucket_error_create でエラーバケットを作成し、ブリゲードの末尾に挿入する。
  • apr_bucket_eos_create で EOS（End of Stream）バケットを作成し、ブリゲードの末尾に挿入する。
  • ctx->eos_sent を 1 に設定する。
  • 接続の keepalive を AP_CONN_CLOSE に設定し、Keep-Alive を無効化する。
  • ap_pass_brigade を呼び出し、エラーバケットと EOS を下位フィルタへ送信する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_request.c: mod_request.c:102, 114, 139
  • modules/http/http_filters.c: http_filters.c:402, 418, 429, 556, 644
  • **呼び出し先**:
  • apr_brigade_cleanup
  • ap_bucket_error_create
  • APR_BRIGADE_INSERT_TAIL
  • apr_bucket_eos_create
  • ap_pass_brigade

関連データ構造

  • http_ctx_t: フィルタの状態管理に使用。
  • ap_filter_t: フィルタチェーンの制御に使用。
  • apr_bucket_brigade: データの送受信に使用。

失敗モードと境界条件

  • プロキシリクエスト時は、特定の HTTP エラーコードに対して APR 定義のエラーコードを優先して返す。
  • チャンクエンコーディングや Content-Length の不整合が発生した場合、Keep-Alive を強制的に無効化することで、後続のリクエストへのデータ混入を防ぐ。

保守担当者向けメモ

  • 本関数はエラー発生時に接続を閉じるため、呼び出し後はフィルタチェーンの処理を終了させる必要がある。
  • プロキシリクエストと通常リクエストでエラーハンドリングのパスが異なる点に注意すること。
