server_cleanup_lua

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:247

目的

サーバーレベルの Lua 仮想マシン（VM）に関連付けられたリソースを解放し、lua_State をクリーンアップするために使用されるコールバック関数です。

シグネチャまたは形状

apr_status_t server_cleanup_lua(void *resource, void *params, apr_pool_t *pool)

入力と出力

  • **入力**:
  • resource: ap_lua_server_spec* 型のポインタ。解放対象の VM 仕様構造体。
  • params: 使用されていません。
  • pool: 関連付けられた apr_pool_t。
  • **出力**:
  • APR_SUCCESS: 常に成功として返されます。

前提条件と不変条件

  • resource 引数は ap_lua_server_spec* 型としてキャスト可能である必要があります。
  • AP_DEBUG_ASSERT により、resource が NULL でないことが検証されます。

アルゴリズムまたは振る舞い

1. resource を ap_lua_server_spec* 型にキャストします。 2. spec->L が NULL でないことを確認します。 3. lua_close を呼び出し、保持されている lua_State を破棄します。 4. APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_vmprep.c:465 において、apr_reslist_destructor として登録・利用されています。
  • **呼び出し先**: lua_close (Lua C API)。

関連データ構造

  • ap_lua_server_spec: VM の仕様と lua_State (L) を保持する構造体。

失敗モードと境界条件

  • spec->L が NULL の場合、lua_close は呼び出されず、安全に終了します。
  • resource が NULL の場合、デバッグビルドではアサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は apr_reslist の破棄処理の一部として機能します。メモリプールが破棄されるタイミングで自動的に呼び出される設計となっています。
