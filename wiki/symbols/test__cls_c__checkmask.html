checkmask

定義

  • 種別: function
  • 定義箇所: test/cls.c:33

目的

文字列 data が、特定のマスク文字を含むパターン mask に一致するかを検証する。キャッシュファイルのヘッダーフォーマットの妥当性確認に使用される。

シグネチャまたは形状

static int checkmask(const char *data, const char *mask)

入力と出力

  • **入力**:
  • data: 検証対象の文字列。
  • mask: 比較パターン。以下の特殊文字を含む。
  • @: 大文字英字 (isupper)
  • $: 小文字英字 (islower)
  • #: 数字 (isdigit)
  • &: 16進数 (isxdigit)
  • *: ワイルドカード（以降の比較をスキップ）
  • **出力**:
  • 1: マッチした場合。
  • 0: マッチしなかった場合。

前提条件と不変条件

  • mask はヌル終端文字列であること。
  • data は mask の長さ以上の有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. mask の各文字を順次走査する。 2. mask が * に到達した場合、即座に 1 を返す。 3. 特殊文字（@, $, #, &）の場合は、対応する ctype.h の関数で data の該当文字を検証する。 4. 特殊文字以外の場合は、data の文字と直接比較する。 5. ループ終了後、mask が * で終わっていない場合は、data の対応位置がヌル文字（\0）であることを確認して一致を判定する。

呼び出し元と呼び出し先

  • **呼び出し元**: test/cls.c:134（キャッシュファイルの行解析時）
  • **呼び出し先**: ctype.h の isupper, islower, isdigit, isxdigit

関連データ構造

なし。

失敗モードと境界条件

  • data が mask より短い場合、比較中に予期せぬメモリ参照が発生する可能性がある（mask の長さが data のバッファサイズを超えないことが前提）。
  • mask に * が含まれない場合、data の長さが mask と完全に一致しないと 0 を返す。

保守担当者向けメモ

  • 本関数は test/cls.c 固有の仕様であり、汎用的なパターンマッチング関数ではない。
  • マスク文字の追加や変更を行う際は、test/cls.c 内の呼び出し箇所（main 関数内の検証ロジック）と整合性をとること。
