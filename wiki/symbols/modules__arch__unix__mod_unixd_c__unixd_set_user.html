unixd_set_user

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_unixd.c:206

目的

Apache HTTP Server の設定ディレクティブ User を処理し、指定されたユーザー名からユーザー ID を解決してグローバル設定構造体に格納する。

シグネチャまたは形状

static const char *unixd_set_user(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • arg: User ディレクティブで指定されたユーザー名（文字列）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • ap_check_cmd_context により、本関数はグローバルコンテキスト（GLOBAL_ONLY）でのみ呼び出し可能である。
  • BIG_SECURITY_HOLE または OS2 が定義されていない環境では、ユーザー ID 0（root）を指定することは許可されない。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、設定がグローバルスコープで行われているか検証する。 2. ap_unixd_config.user_name に引数 arg を代入する。 3. ap_uname2id(arg) を呼び出し、ユーザー名を ID に変換して ap_unixd_config.user_id に格納する。 4. セキュリティチェックとして、ユーザー ID が 0 かつ BIG_SECURITY_HOLE が未定義の場合、root 権限での実行を禁止する警告メッセージを返す。 5. 正常終了時は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/unix/mod_unixd.c:413（AP_INIT_TAKE1 マクロによる User ディレクティブのハンドラとして登録）。
  • **呼び出し先**:
  • ap_check_cmd_context
  • ap_uname2id

関連データ構造

  • ap_unixd_config: サーバーの Unix 関連設定を保持するグローバル構造体。
  • cmd_parms: 設定解析時に使用されるコマンドパラメータ構造体。

失敗モードと境界条件

  • **コンテキストエラー**: グローバル設定以外で User ディレクティブが使用された場合、ap_check_cmd_context が返すエラーメッセージが伝播される。
  • **root 権限の指定**: BIG_SECURITY_HOLE が定義されていない環境でユーザー ID 0 を指定すると、セキュリティ上の警告メッセージが返され、設定は拒否される。

保守担当者向けメモ

  • 本関数は mod_unixd.c 内で静的に定義されており、外部からの直接呼び出しは想定されていない。
  • ユーザー ID の解決には ap_uname2id を使用しているため、OS のユーザーデータベース（/etc/passwd 等）の状態に依存する。
