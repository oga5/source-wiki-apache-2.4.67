check_password

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_file.c:58

目的

テキストファイル（AuthUserFile）に保存されたユーザー名とパスワードを用いて、基本認証（Basic Auth）の資格情報を検証する。

シグネチャまたは形状

static authn_status check_password(request_rec *r, const char *user, const char *password)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • user: 認証対象のユーザー名。
  • password: 検証対象のプレーンテキストパスワード。
  • **出力**:
  • AUTH_GRANTED: 認証成功。
  • AUTH_DENIED: パスワード不一致。
  • AUTH_USER_NOT_FOUND: 指定されたユーザー名がファイル内に存在しない。
  • AUTH_GENERAL_ERROR: 設定エラー（AuthUserFile 未指定）またはファイル読み込み失敗。

前提条件と不変条件

  • r->per_dir_config から取得される authn_file_config_rec に、有効な pwfile パスが設定されていること。
  • パスワードファイルは username:password（またはハッシュ）の形式で記述されていること。

アルゴリズムまたは振る舞い

1. 設定からパスワードファイルのパスを取得し、ファイルを開く。 2. ファイルを1行ずつ読み込み、# で始まる行や空行をスキップする。 3. 各行を : で分割し、ユーザー名が一致するまで検索する。 4. ユーザー名が一致した場合、対応するパスワード（ハッシュ）を抽出してループを終了する。 5. ファイルを閉じる。 6. ユーザーが見つからない場合は AUTH_USER_NOT_FOUND を返す。 7. 成功時に AUTHN_CACHE_STORE を呼び出し、結果をキャッシュ層へ通知する。 8. apr_password_validate を使用して、提供されたパスワードとファイル内の値を照合する。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_auth_basic.c、mod_auth_form.c、mod_authn_core.c 等の認証フレームワーク。
  • **呼び出し先**: ap_pcfg_openfile、ap_cfg_getline、ap_getword、ap_cfg_closefile、apr_password_validate、AUTHN_CACHE_STORE。

関連データ構造

  • authn_file_config_rec: ディレクトリごとの設定を保持する構造体。
  • request_rec: Apache のリクエスト状態を保持する構造体。
  • ap_configfile_t: 設定ファイル読み込み用のハンドル。

失敗モードと境界条件

  • AuthUserFile が設定されていない場合、AUTH_GENERAL_ERROR を返す。
  • ファイルが開けない場合、AUTH_GENERAL_ERROR を返す。
  • ユーザー名がファイル内に存在しない場合、AUTH_USER_NOT_FOUND を返す。
  • パスワードが一致しない場合、AUTH_DENIED を返す。

保守担当者向けメモ

  • 本関数は行単位でファイルを走査するため、ユーザー数が多い場合はパフォーマンスが低下する可能性がある。
  • apr_password_validate は、ファイル内のパスワードがハッシュ化されている場合でも適切に処理を行う。
