md_proto_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_reg.h:210

目的

Managed Domain (MD) サブシステムにおいて、ACME 等の外部プロトコルによる証明書取得や更新処理を抽象化するためのプロトコルドライバ定義です。

シグネチャまたは形状

typedef struct md_proto_t md_proto_t;

struct md_proto_t {
    /* 構造体の詳細は modules/md/md_reg.h:247 以降に定義 */
};

入力と出力

  • **入力**: プロトコルドライバの実行コンテキストとして md_proto_driver_t が使用されます。
  • **出力**: 処理結果は md_result_t を通じて返されます。

前提条件と不変条件

  • md_reg_t レジストリを通じて管理されるドメインに対して、適切なプロトコル実装が紐付けられている必要があります。
  • 構造体自体は不透明なハンドルとして扱われることが多く、具体的な振る舞いは実装ファイル（md_reg.c 等）に依存します。

アルゴリズムまたは振る舞い

  • md_reg が証明書の更新（renew）やロード（preload）を必要とする際、この構造体で定義された関数ポインタを介してプロトコル固有のロジックを呼び出します。
  • ACME プロトコルの場合、md_acme_drive.c にて ACME_PROTO として実体が定義されています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_reg.c: レジストリ管理層がプロトコル操作を委譲する際に参照。
  • **呼び出し先**:
  • modules/md/md_acme_drive.c: ACME_PROTO の定義を通じて、ACME プロトコル固有の実装が提供されます。

関連データ構造

  • md_proto_driver_t: 更新実行時のコンテキスト（環境、ストア、MD情報）を保持。
  • md_reg_t: プロトコルドライバを保持するレジストリハンドル。

失敗モードと境界条件

  • プロトコルドライバが未定義または初期化されていない状態で呼び出された場合の挙動については、ソースコードの断片からは判断できません。

保守担当者向けメモ

  • 本構造体は modules/md/md_reg.h の247行目以降で詳細が定義されています。
  • プロトコルを追加・拡張する場合は、既存の ACME_PROTO の実装を参考にしてください。
  • 構造体のフィールド定義は md_reg.c の実装と密結合しているため、変更時は影響範囲に注意してください。
