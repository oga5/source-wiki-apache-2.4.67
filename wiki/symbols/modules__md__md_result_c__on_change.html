on_change

定義

  • 種別: function
  • 定義箇所: modules/md/md_result.c:64

目的

md_result_t オブジェクトの状態が更新された際に、登録済みのコールバック関数を呼び出して通知を行うための内部ヘルパー関数です。

シグネチャまたは形状

static void on_change(md_result_t *result)

入力と出力

  • **入力**: result (md_result_t *) - 状態変更を通知する対象の md_result_t 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • result ポインタは有効である必要があります。
  • result->on_change にコールバック関数が設定されている場合、その関数は result と result->on_change_data を引数として受け取れる必要があります。

アルゴリズムまたは振る舞い

1. result->on_change が NULL でないかを確認します。 2. NULL でない場合、登録されているコールバック関数を result および result->on_change_data を引数として実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • md_result.c 内の各セッター関数（md_result_set, md_result_problem_set, md_result_printf, md_result_activity_set 等）から、状態変更のトリガーとして呼び出されます。
  • md_result_assign 等のデータ操作関数からも呼び出されます。
  • **呼び出し先**:
  • result->on_change に登録された関数ポインタ。

関連データ構造

  • md_result_t: 状態管理を行うメイン構造体。
  • md_result_change_cb: on_change コールバックの型定義（md_result.h:38）。

失敗モードと境界条件

  • result->on_change が NULL の場合、何も実行されず安全に終了します。

保守担当者向けメモ

  • 本関数は static であり、モジュール内部のセッター関数からのみ利用されます。
  • 状態更新を外部へ即時反映させるための重要なフックであるため、セッター関数を新規追加する際は、必ずこの関数を呼び出すようにしてください。
