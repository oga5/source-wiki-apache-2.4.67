ap_regkey_value_raw_set

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:403

目的

Windows レジストリ内の指定されたキーに対して、型を問わないバイナリデータ（raw data）を書き込みます。

シグネチャまたは形状

apr_status_t ap_regkey_value_raw_set(ap_regkey_t *key,
                                     const char *valuename,
                                     const void *value,
                                     apr_size_t valuesize,
                                     apr_int32_t valuetype,
                                     apr_pool_t *pool)

入力と出力

  • key: 操作対象のレジストリキー（ap_regkey_t 構造体）。
  • valuename: 書き込む値の名前（文字列）。
  • value: 書き込むデータへのポインタ。
  • valuesize: 書き込むデータのサイズ（バイト単位）。
  • valuetype: レジストリ値の型（例: REG_BINARY, REG_DWORD 等）。
  • pool: メモリ割り当てに使用する APR プール。
  • 戻り値: 成功時は APR_SUCCESS、失敗時は OS エラーコードを apr_status_t に変換した値を返します。

前提条件と不変条件

  • key は ap_regkey_open 等で適切に初期化されている必要があります。
  • valuename は NULL 終端文字列である必要があります。
  • APR_HAS_UNICODE_FS が有効な場合、内部で apr_conv_utf8_to_ucs2 を使用して Unicode 変換が行われます。

アルゴリズムまたは振る舞い

1. OS 環境が Unicode 対応の場合、valuename を UTF-8 から UCS-2 に変換します。変換失敗時や名前が長すぎる場合はエラーを返します。 2. OS のレジストリ書き込み API を呼び出します（Unicode 環境では RegSetValueExW、ANSI 環境では RegSetValueEx）。 3. API の戻り値が ERROR_SUCCESS でない場合、APR_FROM_OS_ERROR を用いて APR 形式のエラーコードに変換して返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • os/win32/ap_regkey.c:605
  • server/mpm/winnt/nt_eventlog.c:69
  • 呼び出し先:
  • RegSetValueExW または RegSetValueEx (Win32 API)
  • apr_conv_utf8_to_ucs2 (APR)

関連データ構造

  • ap_regkey_t: レジストリキーハンドル (HKEY) を保持するコンテナ。

失敗モードと境界条件

  • valuename の変換失敗: apr_conv_utf8_to_ucs2 が失敗した場合、そのエラーコードを返します。
  • 名前長超過: 変換後の名前がバッファサイズ（256文字）を超える場合、APR_ENAMETOOLONG を返します。
  • OS レジストリ操作失敗: RegSetValueEx が失敗した場合、対応する Win32 エラーコードが返されます。

保守担当者向けメモ

  • 本関数は ap_regkey_value_set 等の他の値設定関数と異なり、型変換を行わずに直接バイナリとして書き込みます。
  • Unicode/ANSI の切り替えはコンパイル時のマクロ (APR_HAS_UNICODE_FS, APR_HAS_ANSI_FS) に依存します。
