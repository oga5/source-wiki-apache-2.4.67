digest_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_auth_digest.c:85

目的

HTTP Digest 認証モジュール（mod_auth_digest）において、ディレクトリ単位の設定情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct digest_config_struct {
    const char  *dir_name;
    authn_provider_list *providers;
    const char  *realm;
    apr_array_header_t *qop_list;
    apr_sha1_ctx_t  nonce_ctx;
    apr_time_t    nonce_lifetime;
    int          check_nc;
    const char  *algorithm;
    char        *uri_list;
} digest_config_rec;

入力と出力

本構造体は設定ファイル（httpd.conf 等）のディレクティブから読み込まれた値を格納します。

  • **入力**: AuthDigestProvider, AuthName (realm), AuthDigestQop, AuthDigestNonceLifetime, AuthDigestCheckNonceCount, AuthDigestAlgorithm 等のディレクティブ設定。
  • **出力**: 認証プロセス（authenticate_digest_user 等）において、検証ロジックのパラメータとして参照されます。

前提条件と不変条件

  • apr_pcalloc 等でメモリ確保された後、各ディレクティブハンドラによって初期化される必要があります。
  • 共有メモリを使用する環境では、本構造体の設定値に基づき nonce の有効期限やリプレイ攻撃チェックが実行されます。

アルゴリズムまたは振る舞い

  • ディレクトリごとの認証設定をカプセル化し、リクエスト処理時に config 引数として各関数へ渡されます。
  • nonce_lifetime は apr_time_from_sec を介して設定され、nonce の有効期限管理に使用されます。
  • check_nc はリプレイ攻撃防止のための nonce カウントチェックの有効/無効を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_auth_digest.c 内の各設定ディレクティブハンドラ（AuthDigest... 系）および認証フック関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありませんが、authenticate_digest_user 等の認証ロジック内で参照されます。
  • **注記**: 呼び出し元・先に関する詳細なコールグラフは、提供されたソース excerpt の範囲外です。

関連データ構造

  • authn_provider_list: 認証プロバイダーのリスト。
  • apr_sha1_ctx_t: nonce 生成に使用される SHA1 コンテキスト。
  • apr_array_header_t: qop_list（Quality of Protection）の保持に使用。

失敗モードと境界条件

  • 設定が未初期化の状態で参照された場合、未定義の動作を引き起こす可能性があります。
  • nonce_lifetime が適切に設定されていない場合、nonce の期限切れ判定が正しく機能しない可能性があります。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を保持するため、merge_dir_config 等の関数で設定の継承処理が適切に行われているか確認してください。
  • qop_list や algorithm の拡張を行う際は、RFC 2617 への準拠状況を再確認してください。
