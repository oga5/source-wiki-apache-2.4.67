authz_status

定義

  • 種別: enum
  • 定義箇所: include/mod_auth.h:72

目的

Apache HTTP Server の認可（Authorization）プロバイダーが、リクエストに対するアクセス判定結果をコアサーバーへ伝達するために使用する列挙型です。

シグネチャまたは形状

typedef enum {
    AUTHZ_DENIED,
    AUTHZ_GRANTED,
    AUTHZ_NEUTRAL,
    AUTHZ_GENERAL_ERROR,
    AUTHZ_DENIED_NO_USER
} authz_status;

入力と出力

  • 出力: 認可プロバイダーの check_authorization 関数が戻り値として返します。

前提条件と不変条件

  • 認可プロバイダーは、この列挙型のいずれかの値を返すことで、サーバーの認可フローを制御します。

アルゴリズムまたは振る舞い

  • AUTHZ_DENIED: アクセスを拒否します。
  • AUTHZ_GRANTED: アクセスを許可します。
  • AUTHZ_NEUTRAL: 認可判定を行わず、次のプロバイダーへ評価を委ねます。
  • AUTHZ_GENERAL_ERROR: 内部エラーが発生したことを示します。
  • AUTHZ_DENIED_NO_USER: r->user が NULL であるためにアクセスを拒否します。

呼び出し元と呼び出し先

  • 呼び出し元（例）: modules/aaa/mod_authz_core.c 等の認可コアモジュールが、登録されたプロバイダーの関数を呼び出す際に結果を受け取ります。
  • 呼び出し先: 認可プロバイダーの実装（mod_authnz_ldap.c, mod_authnz_fcgi.c 等の check_authorization 関数）がこの値を返します。

関連データ構造

  • authz_provider: 認可プロバイダーの関数ポインタを保持する構造体。
  • request_rec: リクエスト情報を保持する構造体。

失敗モードと境界条件

  • AUTHZ_GENERAL_ERROR は、プロバイダー内部での処理失敗時に使用されます。
  • AUTHZ_DENIED_NO_USER は、認証済みユーザーが存在しない場合に特化した拒否理由として使用されます。

保守担当者向けメモ

  • ap_mmn.h に記載がある通り、認可プロバイダーが複数回呼び出されるケースがあるため、戻り値の解釈には注意してください。
  • 新しい認可プロバイダーを追加する際は、既存の mod_authnz_ldap.c 等の実装例を参考にしてください。
