getsfunc_BRIGADE

定義

  • 種別: function
  • 定義箇所: server/util_script.c:786

目的

apr_bucket_brigade から 1 行分のデータを読み取り、バッファへ格納するコールバック関数です。主に CGI スクリプトの出力ヘッダーを解析する際に使用されます。

シグネチャまたは形状

static int getsfunc_BRIGADE(char *buf, int len, void *arg)

入力と出力

  • **入力**:
  • buf: 読み取ったデータを格納するバッファ。
  • len: バッファのサイズ。
  • arg: apr_bucket_brigade へのポインタ（void * として渡される）。
  • **出力**:
  • 戻り値: 1 行の読み取りが完了した場合は 1、それ以外（エラーまたは終了）は 0 または -1 を返します。

前提条件と不変条件

  • buf は len 分の領域を確保している必要があります。
  • arg は有効な apr_bucket_brigade を指している必要があります。
  • 読み取り処理中、buf の末尾には常にヌル終端が保証されます。

アルゴリズムまたは振る舞い

1. apr_bucket_brigade からバケットを順次読み取ります。 2. 改行文字（\n）を検出するまで、またはバッファがいっぱいになるまでデータをコピーします。 3. キャリッジリターン（\r）は無視して破棄します。 4. 改行を検出した時点でバケットを分割し、処理済みのバケットを削除します。 5. 成功時は 1 を、タイムアウト時は -1 を、それ以外は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ap_scan_script_header_err_core_ex (server/util_script.c:836, 845)
  • **呼び出し先**:
  • apr_bucket_read
  • apr_bucket_split
  • apr_bucket_delete
  • APR_BRIGADE_FIRST / APR_BRIGADE_SENTINEL / APR_BUCKET_NEXT / APR_BUCKET_IS_EOS

関連データ構造

  • apr_bucket_brigade: データのストリームを保持するバケットの集合。
  • apr_bucket: データの断片を保持する構造体。

失敗モードと境界条件

  • apr_bucket_read が失敗した場合、またはタイムアウト（APR_STATUS_IS_TIMEUP）が発生した場合はエラーを返します。
  • バッファサイズ len を超える行データが入力された場合、バッファの範囲内で切り詰められます。
  • EOS（End of Stream）バケットに到達した場合は読み取りを終了します。

保守担当者向けメモ

  • 本関数は ap_scan_script_header_err_core_ex のコールバックとして設計されています。
  • バケットの分割と削除を行うため、呼び出し側でブリゲードの整合性が保たれていることを確認してください。
