usage

定義

  • 種別: function
  • 定義箇所: support/rotatelogs.c:106

目的

rotatelogs ユーティリティのコマンドライン引数の使用方法を標準エラー出力に表示し、プログラムを終了させる。

シグネチャまたは形状

static void usage(const char *argv0, const char *reason)

入力と出力

  • **入力**:
  • argv0: 実行ファイル名（通常は argv[0]）。
  • reason: 表示するエラーメッセージ（NULL の場合は省略される）。
  • **出力**: なし（標準エラー出力へメッセージを出力後、exit(1) を呼び出す）。

前提条件と不変条件

  • 本関数が呼び出されると、プログラムは必ず終了する。
  • reason が指定されている場合、その内容が最初に表示される。

アルゴリズムまたは振る舞い

1. reason が存在する場合、それを標準エラー出力に出力する。 2. コンパイル時の APR_FILES_AS_SOCKETS 定義の有無に応じ、利用可能なオプションを含む使用方法の概要を出力する。 3. OS 固有の TransferLog 設定例（httpd.conf への記述例）を出力する。 4. 各コマンドラインオプション（-v, -l, -L, -p, -f, -D, -t, -T, -e, -c, -n）の詳細な説明を出力する。 5. -p オプションで指定するプログラムの引数形式について説明する。 6. exit(1) を実行し、プロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内で明示的な呼び出し箇所は提供されていない。
  • **呼び出し先**: fprintf, exit (標準ライブラリ)。

関連データ構造

  • なし。

失敗モードと境界条件

  • 本関数自体はエラーを返さず、常に終了する。
  • APR_FILES_AS_SOCKETS が定義されている環境とそうでない環境で、表示されるオプション（-c の有無）が異なる。

保守担当者向けメモ

  • 新しいオプションを追加した際は、本関数の出力メッセージにも必ず追記すること。
  • httpd.conf の設定例は、プラットフォーム（OS/2 等）ごとの差異を維持する必要がある。
