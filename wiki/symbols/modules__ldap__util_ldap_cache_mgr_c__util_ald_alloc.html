util_ald_alloc

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache_mgr.c:98

目的

LDAP キャッシュ専用のメモリ割り当てを行う。共有メモリ（SHM）が有効な場合は apr_rmm を介して割り当て、それ以外の場合は標準の calloc を使用する抽象化レイヤーである。

シグネチャまたは形状

void *util_ald_alloc(util_ald_cache_t *cache, unsigned long size)

入力と出力

  • **入力**:
  • cache: キャッシュ管理構造体へのポインタ。
  • size: 割り当てるメモリサイズ（バイト単位）。
  • **出力**:
  • 割り当てられたメモリ領域へのポインタ。失敗時または size が 0 の場合は NULL を返す。

前提条件と不変条件

  • size が 0 の場合、常に NULL を返す。
  • APR_HAS_SHARED_MEMORY が有効かつ cache->rmm_addr が設定されている場合、メモリは共有メモリ領域から割り当てられる。
  • 共有メモリ使用時は、返されるポインタは apr_rmm_addr_get によって変換されたアドレスである。

アルゴリズムまたは振る舞い

1. size が 0 であれば直ちに NULL を返す。 2. APR_HAS_SHARED_MEMORY が定義されており、かつ cache->rmm_addr が存在する場合:

  • apr_rmm_calloc を使用して共有メモリブロックを確保する。
  • 確保成功時、apr_rmm_addr_get を用いてオフセットを実アドレスに変換して返す。

    • 3. 上記以外の場合:

  • 標準の calloc(sizeof(char), size) を呼び出し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap_cache.c: 各種ノード（URL, 検索, 比較, DN比較）の生成時に利用。
  • modules/ldap/util_ldap_cache_mgr.c: キャッシュ構造体の初期化やサブグループDNの割り当て時に利用。
  • **呼び出し先**:
  • apr_rmm_calloc / apr_rmm_addr_get (共有メモリ有効時)
  • calloc (標準ライブラリ)

関連データ構造

  • util_ald_cache_t: キャッシュ管理構造体。rmm_addr を保持し、メモリ割り当てのコンテキストとして使用される。

失敗モードと境界条件

  • size が 0 の場合: NULL を返す。
  • メモリ不足時: apr_rmm_calloc または calloc が失敗し、NULL を返す。
  • 共有メモリ環境下で rmm_addr が初期化されていない場合: calloc にフォールバックする。

保守担当者向けメモ

  • 共有メモリ使用時は apr_rmm を介したポインタ操作が必要となるため、呼び出し元は返されたポインタを直接的なメモリ操作の起点として扱うこと。
  • calloc を使用しているため、割り当てられたメモリはゼロクリアされていることが保証される。
