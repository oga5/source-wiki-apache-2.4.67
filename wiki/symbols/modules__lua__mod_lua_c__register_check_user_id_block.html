register_check_user_id_block

定義

  • 種別: function
  • 定義箇所: modules/lua/mod_lua.c:1395

目的

Apache HTTP Server の設定ディレクティブ <LuaHookCheckUserID を処理し、認証フェーズ（check_user_id）で実行される Lua 関数ブロックを登録する。

シグネチャまたは形状

static const char *register_check_user_id_block(cmd_parms *cmd, void *_cfg, const char *line)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • _cfg: 現在のディレクトリ設定（ap_lua_dir_cfg）。
  • line: 設定ファイルから読み込まれた引数文字列。
  • **出力**:
  • 成功時は NULL を返し、エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • cmd および _cfg は有効なポインタである必要がある。
  • 本関数は Apache の設定パースフェーズで呼び出される。

アルゴリズムまたは振る舞い

1. register_named_block_function_hook を呼び出す。 2. 第1引数に "check_user_id" を指定し、該当するフックポイントに対するブロック関数登録処理を委譲する。 3. 委譲先の戻り値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/mod_lua.c:1904 において、AP_INIT_RAW_ARGS マクロにより <LuaHookCheckUserID ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: register_named_block_function_hook（詳細な定義箇所は提供されたソース範囲外）。

関連データ構造

  • cmd_parms: Apache の設定コマンドパラメータ。
  • ap_lua_dir_cfg: ディレクトリ単位の Lua 設定（_cfg として渡される）。

失敗モードと境界条件

  • register_named_block_function_hook がエラーを返した場合、そのエラーメッセージがそのまま伝播される。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的なロジックは register_named_block_function_hook に依存している。
  • LuaHookCheckUserID ディレクティブの仕様変更を行う際は、この関数ではなく register_named_block_function_hook の実装を確認すること。
