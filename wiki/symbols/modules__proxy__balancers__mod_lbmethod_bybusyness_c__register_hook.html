register_hook

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_bybusyness.c:110

目的

mod_proxy のロードバランシング手法として「bybusyness」を登録し、設定完了後の初期化フックを構成します。

シグネチャまたは形状

static void register_hook(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p（メモリ割り当て用のプール）
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • mod_proxy がロードされている環境で実行される必要があります。
  • bybusyness 構造体が定義済みである必要があります。

アルゴリズムまたは振る舞い

1. ap_register_provider を呼び出し、PROXY_LBMETHOD プロバイダーとして "bybusyness" という名称で bybusyness 構造体を登録します。 2. ap_hook_post_config を呼び出し、設定読み込み後の初期化関数として lbmethod_bybusyness_post_config を APR_HOOK_MIDDLE の優先度で登録します。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュール読み込みプロセス（modules/proxy/balancers/mod_lbmethod_bybusyness.c:123 にてフック登録の一部として参照）。
  • 呼び出し先:
  • ap_register_provider
  • ap_hook_post_config
  • lbmethod_bybusyness_post_config

関連データ構造

  • bybusyness: ロードバランシング手法を定義する proxy_balancer_method 構造体。

失敗モードと境界条件

  • ap_register_provider や ap_hook_post_config が失敗した場合の挙動は、Apache のフックシステムおよびプロバイダー登録機構の仕様に依存します。

保守担当者向けメモ

  • 本関数はモジュールのライフサイクル管理の起点です。post_config フックでの依存関係チェック（mod_proxy のコア関数取得など）とセットで機能するため、変更時は lbmethod_bybusyness_post_config との整合性に注意してください。
