md_acme_order_create

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:44

目的

ACME プロトコルにおける「Order（注文）」オブジェクトをメモリ上に新規生成し、初期化を行う。

シグネチャまたは形状

md_acme_order_t *md_acme_order_create(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（オブジェクトの生存期間を管理するメモリプール）
  • **出力**: 初期化済みの md_acme_order_t 構造体へのポインタ

前提条件と不変条件

  • 引数 p は有効な apr_pool_t であること。
  • 生成された md_acme_order_t 内の authz_urls および challenge_setups 配列は、初期容量 5 で確保される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して md_acme_order_t 構造体分のメモリをゼロクリアして確保する。 2. 構造体の p メンバに引数のプールを割り当てる。 3. authz_urls（認証 URL リスト）および challenge_setups（チャレンジセットアップトークンリスト）を apr_array_make で初期化する。 4. 初期化された構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_order.c:142
  • modules/md/md_acme_order.c:314
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_array_make (APR)

関連データ構造

  • md_acme_order_t: 本関数で生成されるメイン構造体。
  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc または apr_array_make の内部実装に従い、プロセスが終了（abort）する可能性がある。

保守担当者向けメモ

  • 本関数はメモリ確保のみを行うため、ACME サーバーへの登録処理が必要な場合は、別途 md_acme_order_register を呼び出す必要がある。
  • authz_urls と challenge_setups の初期容量は 5 に設定されているが、必要に応じて動的に拡張される。
