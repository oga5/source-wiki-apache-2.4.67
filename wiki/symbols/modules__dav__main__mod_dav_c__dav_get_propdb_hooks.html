dav_get_propdb_hooks

定義

  • 種別: function
  • 定義箇所: modules/dav/main/mod_dav.c:233

目的

現在のリクエストに関連付けられた WebDAV プロバイダーから、プロパティデータベース（propdb）操作用のフックセットを取得します。

シグネチャまたは形状

const dav_hooks_propdb *dav_get_propdb_hooks(request_rec *r)

入力と出力

  • **入力**: request_rec *r （Apache のリクエストレコード）
  • **出力**: const dav_hooks_propdb * （プロパティデータベース操作を定義するフック構造体へのポインタ）

前提条件と不変条件

  • dav_get_provider(r) が有効なプロバイダーを返すことが前提です。
  • 返されるフック構造体は、当該リクエストに対して設定されたプロバイダーの実装に依存します。

アルゴリズムまたは振る舞い

1. dav_get_provider(r) を呼び出し、現在のリクエストに対応する dav_provider 構造体を取得します。 2. 取得したプロバイダー構造体内の propdb メンバ（dav_hooks_propdb 型）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.h:799 （宣言）
  • modules/dav/main/mod_dav.h:812 （DAV_GET_HOOKS_PROPDB マクロ経由）
  • **呼び出し先**:
  • dav_get_provider （modules/dav/main/mod_dav.c 内の関数）

関連データ構造

  • dav_hooks_propdb: プロパティデータベース操作のための関数ポインタ群を保持する構造体。
  • dav_provider: リポジトリやプロパティ操作のフックを保持するプロバイダー定義構造体。
  • request_rec: Apache のリクエストコンテキスト。

失敗モードと境界条件

  • dav_get_provider(r) が NULL を返す場合、本関数はデリファレンスによりクラッシュする可能性があります。呼び出し側でプロバイダーの存在が保証されている必要があります。

保守担当者向けメモ

  • 本関数は DAV_GET_HOOKS_PROPDB(r) マクロを通じて利用されることが一般的です。
  • プロバイダーの切り替えや拡張を行う際は、dav_hooks_propdb 構造体の定義が mod_dav.h と整合していることを確認してください。
