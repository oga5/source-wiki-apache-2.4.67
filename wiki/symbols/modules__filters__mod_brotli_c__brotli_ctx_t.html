brotli_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_brotli.c:161

目的

リクエスト単位の Brotli 圧縮コンテキストを保持し、エンコーダの状態、出力バッファ、および統計情報を管理するために使用されます。

シグネチャまたは形状

typedef struct brotli_ctx_t {
    BrotliEncoderState *state;
    apr_bucket_brigade *bb;
    apr_off_t total_in;
    apr_off_t total_out;
} brotli_ctx_t;

入力と出力

  • state: Brotli ライブラリのエンコーダインスタンス。
  • bb: 圧縮済みデータを保持するバケットブリゲード。
  • total_in: 圧縮対象として入力された総バイト数。
  • total_out: 圧縮済みとして出力された総バイト数。

前提条件と不変条件

  • 本構造体はリクエストプール（request_pool）に紐付いて割り当てられます。
  • cleanup_ctx 関数により、プールクリーンアップ時に BrotliEncoderDestroyInstance が呼び出され、メモリリークが防止される不変条件を持ちます。

アルゴリズムまたは振る舞い

  • compress_filter によるリクエスト処理開始時に create_ctx を通じて初期化されます。
  • process_chunk 内で BrotliEncoderCompressStream を介してデータを圧縮し、BrotliEncoderTakeOutput で取得したデータを bb に蓄積します。
  • flush 呼び出しにより、エンコーダのバッファを強制的にフラッシュし、残りのデータをブリゲードへ送出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_ctx (初期化)
  • process_chunk (データ処理)
  • flush (フラッシュ処理)
  • compress_filter (フィルタのメイン処理)
  • **呼び出し先**:
  • BrotliEncoderState (Brotli ライブラリ)
  • apr_bucket_brigade (Apache APR)

関連データ構造

  • BrotliEncoderState: Brotli ライブラリの内部状態構造体。
  • apr_bucket_brigade: Apache HTTP Server のストリームデータ管理構造体。
  • brotli_server_config_t: サーバー単位の設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc が失敗する可能性があります。
  • 圧縮処理失敗時: BrotliEncoderCompressStream がエラーを返す場合、フィルタ処理が中断されます。
  • サブクエストや Content-Range が存在するリクエストは、本コンテキストを使用せず圧縮対象外となります。

保守担当者向けメモ

  • 本構造体は mod_deflate のコンテキスト構造をモデルにしています。
  • 統計情報（total_in, total_out）は、リクエスト終了時にノートとして記録されるため、正確な計測には cleanup_ctx までのライフサイクル維持が必須です。
