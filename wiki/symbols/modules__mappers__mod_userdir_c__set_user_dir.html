set_user_dir

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_userdir.c:134

目的

Apache HTTP Server の UserDir ディレクティブを解析し、userdir_config 構造体へ設定を反映させるための設定ハンドラです。

シグネチャまたは形状

static const char *set_user_dir(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: UserDir ディレクティブに指定された引数文字列。
  • **出力**:
  • 成功時: NULL を返します。
  • 失敗時: エラーメッセージ文字列を返します。

前提条件と不変条件

  • cmd->server->module_config から userdir_module の設定構造体を取得できる必要があります。
  • 引数 arg が空文字列である場合、エラーとして扱われます。

アルゴリズムまたは振る舞い

1. arg の最初の単語（キーワード）を抽出します。 2. キーワードが disable/disabled または enable/enabled の場合:

  • 後続の引数がない場合は、s_cfg->globally_disabled を更新して終了します。
  • 後続の引数がある場合は、それらを s_cfg->disabled_users または s_cfg->enabled_users テーブルに追加します。

    • 3. キーワードが上記以外の場合:

  • arg 全体を s_cfg->userdir に設定します。

    • 4. テーブルへの追加処理では、ap_getword_conf を用いて引数を順次解析し、apr_table_setn でテーブルに登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_userdir.c:191 にて AP_INIT_RAW_ARGS マクロを通じて登録されており、Apache の設定読み込み時に呼び出されます。
  • **呼び出し先**: ap_getword_conf（文字列解析用）、apr_table_setn（テーブル登録用）。

関連データ構造

  • userdir_config: モジュールの設定を保持する構造体。
  • cmd_parms: 設定コマンドのコンテキスト。
  • apr_table_t: ユーザーの有効/無効リストを保持するテーブル。

失敗モードと境界条件

  • 引数が指定されていない場合、"UserDir requires an argument." というエラーメッセージを返します。
  • disable/enable キーワードの後にユーザー名が指定されない場合、グローバルな設定として処理されます。

保守担当者向けメモ

  • 本関数は RAW_ARGS として登録されているため、引数のパースを自前で行っています。
  • apr_table_setn を使用しているため、テーブルに登録される値の寿命が cmd->pool に依存することに注意してください。
