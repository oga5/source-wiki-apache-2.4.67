get_cert_name

定義

  • 種別: function
  • 定義箇所: modules/md/md_status.c:188

目的

Managed Domain (MD) に関連付けられた証明書の識別名（またはインデックス）を取得し、JSON 出力用の文字列として提供する。

シグネチャまたは形状

static const char *get_cert_name(const md_t *md, int i, int from_staging, apr_pool_t *p)

入力と出力

  • **入力**:
  • md: 対象の Managed Domain 構造体。
  • i: 証明書のインデックス。
  • from_staging: ステージング領域からの取得かを示すフラグ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • 証明書を識別するための文字列（const char *）。

前提条件と不変条件

  • p は有効な apr_pool_t であること。
  • md は md_t 構造体として初期化されていること。

アルゴリズムまたは振る舞い

1. from_staging が偽であり、かつ md->cert_files が存在し要素を持つ場合、静的ファイルが設定されていると判断し、インデックス i を文字列に変換して返す。 2. それ以外の場合、md_pkey_spec_name を呼び出し、md->pks から取得した秘密鍵仕様に基づいた名称を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_status.c:218 (md_json_setj の引数として使用)
  • **呼び出し先**:
  • md_pkey_spec_name
  • md_pkeys_spec_get
  • apr_psprintf

関連データ構造

  • md_t: 管理対象ドメインの設定情報を保持。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • md が NULL の場合、または md->pks が不正な状態である場合の挙動については、本ソース抜粋からは判断できない。

保守担当者向けメモ

  • 本関数は md_status モジュールにおける JSON 生成プロセスの一部として利用される。
  • MD_STATUS_WITH_SCTS マクロに関連する拡張が行われる場合、本関数の戻り値の仕様が変更される可能性がある。
