md_acme_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme.h:54

目的

ACME (Automated Certificate Management Environment) サーバーとの通信セッションおよび状態を保持するためのメイン構造体です。

シグネチャまたは形状

typedef struct md_acme_t md_acme_t;

※不透明なポインタとして定義されており、具体的なメンバ構成はヘッダーファイル内では非公開です。

入力と出力

  • **入力**: ACME サーバーの URL、アカウント情報、API エンドポイント（v1/v2）、および通信用の HTTP ハンドルを保持します。
  • **出力**: ACME プロトコルに基づくリクエストの構築、署名、およびレスポンス処理のコンテキストとして機能します。

前提条件と不変条件

  • 本構造体は、関連付けられた apr_pool_t（Apache Portable Runtime プール）の生存期間に依存します。
  • Apache のリクエスト処理コンテキスト内で動作することを前提としています。

アルゴリズムまたは振る舞い

  • ACME サーバーとの対話における状態管理（ディレクトリ情報、バージョン、認証状態）を行います。
  • md_acme_POST や md_acme_GET などの関数を通じて、HTTP リクエストのディスパッチおよびレスポンスのコールバック処理を仲介します。
  • アカウントのロード、保存、および利用規約（ToS）の同意管理をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme.c 内の各関数（md_acme_POST, md_acme_GET, md_acme_req_create 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、具体的な通信処理は md_http.h や md_jws.h を利用する関数群へ委譲されます。

関連データ構造

  • md_acme_req_t: 個別の ACME リクエストコンテキスト。
  • md_acme_state_t: ACME プロセスの進行状況を示す列挙型。
  • apr_pool_t: メモリ管理に使用。
  • md_json_t: JSON ペイロード処理に使用。

失敗モードと境界条件

  • メモリ管理: apr_pool_t の生存期間を超えて本構造体を参照した場合、未定義の動作を引き起こします。
  • スレッド安全性: Apache のリクエスト処理コンテキスト外でのグローバルな状態変更には注意が必要です。

保守担当者向けメモ

  • 本構造体は不透明ポインタとして定義されているため、内部構造を変更する際は modules/md/md_acme.c の実装との整合性に注意してください。
  • JWS 署名や HTTP 再試行ロジックの詳細は md_jws.c や md_http.c に委譲されています。
