compute_auth

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:168

目的

セッションデータに対する改ざん検知用の認証タグ（MAC）を生成します。

シグネチャまたは形状

static void compute_auth(const void *src, apr_size_t len,
                         const char *passphrase, apr_size_t passlen,
                         unsigned char auth[AP_SIPHASH_DSIZE])

入力と出力

  • **入力**:
  • src: 認証対象のデータポインタ。
  • len: データの長さ。
  • passphrase: 認証キー生成に使用するパスフレーズ文字列。
  • passlen: パスフレーズの長さ。
  • **出力**:
  • auth: 生成された認証タグ（AP_SIPHASH_DSIZE バイト）を格納するバッファ。

前提条件と不変条件

  • apr_md5 を使用してパスフレーズから 128 ビットのキーを生成します。
  • 認証アルゴリズムとして SipHash-2-4 を使用します。

アルゴリズムまたは振る舞い

1. 指定された passphrase を apr_md5 に渡し、128 ビットのダイジェストを生成して SipHash 用のキーとします。 2. 生成したキーと入力データ src を用いて ap_siphash24_auth を呼び出し、認証タグを計算して auth バッファに書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session_crypto.c:327 (復号時の検証用)
  • modules/session/mod_session_crypto.c:389 (暗号化時のタグ生成用)
  • **呼び出し先**:
  • apr_md5
  • ap_siphash24_auth

関連データ構造

  • AP_SIPHASH_DSIZE: 認証タグのサイズ定数。
  • APR_MD5_DIGESTSIZE: MD5 ダイジェストのサイズ定数。

失敗モードと境界条件

  • 本関数自体は戻り値を返さないため、内部的なエラーハンドリングは行いません。
  • 入力データやパスフレーズの長さが 0 の場合、apr_md5 および ap_siphash24_auth の仕様に従った動作となります。

保守担当者向けメモ

  • 現在、パスフレーズから SipHash キーを生成するために MD5 を使用しています。これは apr_crypto_key_t から直接バイト列を取得する手段が APR-1.5.x 以前で提供されていないことによる回避策です。
  • ap_memeq_timingsafe を使用した比較処理は、本関数の呼び出し側（decrypt_string 等）で実施されています。
