ssl_hook_UserCheck

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:1193

目的

クライアント証明書のサブジェクトDN（Distinguished Name）をユーザー名として利用し、HTTP Basic認証ヘッダーを擬似的に生成（FakeBasicAuth）することで、証明書ベースの認証をアプリケーション層の認証機構と統合する。

シグネチャまたは形状

int ssl_hook_UserCheck(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のHTTPリクエスト構造体）
  • **出力**: int （Apacheのフック戻り値。DECLINED または HTTP_FORBIDDEN）

前提条件と不変条件

  • サブリクエスト（!ap_is_initial_req(r)）の場合は処理をスキップする。
  • SSL_OPT_STRICTREQUIRE が有効で、かつ ssl-access-forbidden ノートが存在する場合は HTTP_FORBIDDEN を返す。
  • TLSが有効であること、クライアント証明書が提示されていること、SSL_OPT_FAKEBASICAUTH が設定されていること、および r->user が未設定であることが実行の前提条件。

アルゴリズムまたは振る舞い

1. **不正利用の検知**: 既存の Authorization ヘッダーが、証明書DNを模倣した不正な形式（ユーザー名が / で始まりパスワードが password）でないか検証し、該当すれば HTTP_FORBIDDEN を返す。 2. **条件判定**: 前提条件を満たさない場合は DECLINED を返す。 3. **DNの取得**: sslconn->client_cert からサブジェクトDNを抽出し、sslconn->client_dn にキャッシュする。 4. **ヘッダーの偽装**: client_dn と固定パスワード password を用いて Basic 認証文字列を生成し、r->headers_in の Authorization ヘッダーを上書きする。 5. **終了**: 処理完了後、DECLINED を返して後続の認証フェーズへ制御を渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c の ap_hook_check_authn により登録され、認証フェーズで呼び出される。
  • **呼び出し先**: Apache API (ap_getword, ap_pbase64encode, ap_log_rerror 等) および OpenSSL API (X509_get_subject_name, X509_NAME_oneline) を利用。

関連データ構造

  • SSLConnRec: SSL接続固有の情報を保持。
  • SSLDirConfigRec: ディレクトリ単位のSSL設定（nOptions 等）。
  • request_rec: HTTPリクエスト情報。

失敗モードと境界条件

  • FakeBasicAuth を悪用したなりすまし試行を検知した場合、HTTP_FORBIDDEN を返す。
  • 必要なSSL情報が欠落している場合や、既に認証済みの場合は処理をスキップする。

保守担当者向けメモ

  • 本関数は FakeBasicAuth オプションに強く依存している。
  • client_dn の生成には X509_NAME_oneline を使用しており、OpenSSLのメモリ管理ルールに従い OPENSSL_free で解放している。
  • 生成される偽装ヘッダーのパスワード password は、crypt の仕様を考慮した固定値である。
