ap_logs_child_init

定義

  • 種別: function
  • 定義箇所: server/log.c:233

目的

子プロセスの初期化フェーズにおいて、親プロセスから継承された不要なログ読み取り用ファイルハンドルをクローズし、リソースを解放します。

シグネチャまたは形状

void ap_logs_child_init(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: メモリプール（本関数内では未使用）。
  • s: サーバーレコード（本関数内では未使用）。
  • **出力**: なし（副作用としてファイルハンドルをクローズ）。

前提条件と不変条件

  • グローバル変数 read_handles が、クローズ対象のハンドルリストを保持していること。
  • read_handle_t 構造体の handle メンバが有効な APR ファイルハンドルであること。

アルゴリズムまたは振る舞い

1. グローバル変数 read_handles を起点として、リストを順次走査します。 2. 各ノードが保持する handle に対して apr_file_close を実行します。 3. リストの終端に達するまで、次のノードへポインタを進めます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c において ap_hook_child_init に登録されており、子プロセスの初期化時に呼び出されます。
  • **呼び出し先**: apr_file_close（APR ライブラリ関数）。

関連データ構造

  • read_handle_t: ログ読み取りハンドルを管理するリンクリスト構造体。next ポインタと handle を保持します。

失敗モードと境界条件

  • read_handles が NULL の場合、何も行わずに正常終了します。
  • apr_file_close がエラーを返した場合でも、本関数はエラーを伝播させず、リストの走査を継続します。

保守担当者向けメモ

  • 本関数はモジュールによる直接呼び出しを意図していません（include/http_log.h の注釈による）。
  • 子プロセス生成時に親プロセスのファイル記述子を適切に整理するための重要なクリーンアップ処理です。
