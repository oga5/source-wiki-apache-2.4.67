authn_cache_enable

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:191

目的

AuthnCacheEnable ディレクティブの設定を処理し、モジュールが有効であることを示すグローバルな状態を更新します。

シグネチャまたは形状

static const char *authn_cache_enable(cmd_parms *cmd, void *CFG)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • CFG: ディレクトリ設定構造体（本関数では使用されません）。
  • **出力**:
  • 戻り値: 設定エラーが発生した場合はエラーメッセージの文字列、成功した場合は NULL を返します。

前提条件と不変条件

  • 本関数は GLOBAL_ONLY コンテキストで呼び出される必要があります。サーバー設定レベル以外での使用は許可されません。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、現在の設定コンテキストがグローバルであることを検証します。 2. グローバル変数 configured を 1 に設定し、モジュールが有効化されたことを記録します。 3. 検証結果（エラーメッセージまたは NULL）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_socache.c:253 において、AP_INIT_NO_ARGS マクロを通じて AuthnCacheEnable ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: ap_check_cmd_context（Apache HTTP Server コア関数）。

関連データ構造

  • cmd_parms: 設定コマンドの引数とコンテキストを保持する構造体。
  • configured: モジュールの有効状態を保持するグローバル変数。

失敗モードと境界条件

  • ap_check_cmd_context がグローバルコンテキスト以外での呼び出しを検知した場合、エラーメッセージが返され、設定処理が中断されます。

保守担当者向けメモ

  • 本関数は単に configured フラグを立てる役割を担っています。このフラグが他の関数でどのように参照・利用されているかについては、モジュール全体の初期化フローを確認してください。
