session_req_done

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_proxy_http2.c:254

目的

HTTP/2 プロキシセッションにおけるリクエスト完了時の後処理を仲介します。

シグネチャまたは形状

static void session_req_done(h2_proxy_session *session, request_rec *r,
                             apr_status_t status, int touched, int error_code)

入力と出力

  • **入力**:
  • session: 処理対象の h2_proxy_session 構造体。
  • r: 完了したリクエストの request_rec。
  • status: リクエスト処理の結果を示す apr_status_t。
  • touched: リクエストが処理されたかを示すフラグ。
  • error_code: 発生したエラーコード。
  • **出力**: なし（void）。

前提条件と不変条件

  • session が有効なポインタであり、そのメンバ user_data が request_done 関数で処理可能な形式であることを前提とします。

アルゴリズムまたは振る舞い

1. session->user_data を取得します。 2. 取得したデータを引数として、request_done 関数を呼び出します。 3. request_done に r、status、touched、error_code を引き継ぎ、ステータスの更新や再試行判定を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/mod_proxy_http2.c:273 (直接的な呼び出し)
  • modules/http2/mod_proxy_http2.c:299 (h2_proxy_session_cleanup への引数として参照)
  • **呼び出し先**:
  • request_done (同ファイル内)

関連データ構造

  • h2_proxy_session: HTTP/2 プロキシセッションの状態を保持する構造体。
  • request_rec: Apache HTTP Server のリクエスト状態を保持する構造体。

失敗モードと境界条件

  • session が NULL の場合、session->user_data へのアクセスによりセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数は request_done への単なるラッパーです。ロジックの変更が必要な場合は、主に request_done 側を確認してください。
