dav_register_liveprop_group

定義

  • 種別: function
  • 定義箇所: modules/dav/main/liveprop.c:131

目的

WebDAV プロバイダが提供するライブプロパティのグループを登録し、そのグループに含まれるすべての名前空間 URI をグローバルな名前空間管理テーブルへ登録します。

シグネチャまたは形状

void dav_register_liveprop_group(apr_pool_t *p, const dav_liveprop_group *group)

入力と出力

  • p: 名前空間の登録に使用されるメモリプール。
  • group: 登録対象となる dav_liveprop_group 構造体へのポインタ。この構造体は namespace_uris（NULL 終端の文字列配列）を保持している必要があります。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • group 引数は NULL であってはなりません。
  • group->namespace_uris は NULL で終了する文字列配列である必要があります。
  • 登録される名前空間 URI は、内部的に dav_register_liveprop_namespace を介して dav_liveprop_uris ハッシュへ追加されます。

アルゴリズムまたは振る舞い

1. group 構造体から名前空間 URI のリスト（namespace_uris）を取得します。 2. リスト内の各 URI に対してループ処理を行います。 3. 各 URI を dav_register_liveprop_namespace(p, *uris) に渡し、名前空間のインデックス割り当てと登録を実行します。 4. リストの終端（NULL）に達するまで処理を繰り返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/dav/fs/repos.c:2279
  • modules/dav/main/std_liveprop.c:225
  • 呼び出し先:
  • dav_register_liveprop_namespace

関連データ構造

  • dav_liveprop_group: ライブプロパティの仕様（specs）と名前空間（namespace_uris）を保持する構造体。
  • dav_liveprop_uris: 登録された URI を管理する内部ハッシュテーブル。

失敗モードと境界条件

  • group->namespace_uris が空（最初の要素が NULL）の場合、ループは即座に終了し、何も登録されません。
  • メモリ不足が発生した場合、apr_pool の挙動に従い、Apache のエラーハンドリング機構へ制御が移ります。

保守担当者向けメモ

  • 本関数はプロバイダの初期化フェーズで呼び出されることを想定しています。
  • 登録された名前空間はプロセス生存期間中保持されるため、メモリ管理は apr_pool のライフサイクルに依存します。
  • dav_liveprop_count は本関数経由で登録された名前空間の数に応じて増加し、XML 出力時のプレフィックス（lp1, lp2...）生成に利用されます。
