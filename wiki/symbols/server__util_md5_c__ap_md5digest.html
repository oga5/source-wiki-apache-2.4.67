ap_md5digest

定義

  • 種別: function
  • 定義箇所: server/util_md5.c:150

目的

指定されたファイルの内容を読み込み、その MD5 ダイジェストを Base64 形式の文字列として生成します。

シグネチャまたは形状

char *ap_md5digest(apr_pool_t *p, apr_file_t *infile)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • infile: 読み込み対象の apr_file_t ポインタ。
  • **出力**:
  • 戻り値: 生成された MD5 ダイジェスト（Base64 エンコード済み文字列）。メモリは p から割り当てられます。

前提条件と不変条件

  • infile は読み込み可能な状態で開かれている必要があります。
  • 処理終了後、infile のファイルポインタは処理開始時の位置（先頭）に復元されます。

アルゴリズムまたは振る舞い

1. apr_md5_init() を呼び出し、MD5 コンテキストを初期化します。 2. 4096 バイトのバッファを使用し、apr_file_read() でファイル内容を順次読み込みます。 3. 読み込んだデータに対して apr_md5_update() を実行し、コンテキストを更新します。 4. ファイルの読み込みが完了（またはエラー）した後、apr_file_seek() を使用してファイルポインタをオフセット 0（先頭）に戻します。 5. ap_md5contextTo64() を呼び出し、確定したコンテキストから Base64 形式の文字列を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:4986 での呼び出しが確認されています。
  • **呼び出し先**:
  • apr_md5_init
  • apr_file_read
  • apr_md5_update
  • apr_file_seek
  • ap_md5contextTo64

関連データ構造

  • apr_md5_ctx_t: MD5 計算の状態を保持する構造体。
  • apr_pool_t: メモリ管理用プール。
  • apr_file_t: ファイルハンドル。

失敗モードと境界条件

  • apr_file_read が APR_SUCCESS 以外の値を返した場合、ループを終了し、その時点までのコンテキストでダイジェストを生成します。
  • ファイルポインタの復元（apr_file_seek）が失敗した場合の挙動については、本ソースコード内では明示的なエラーハンドリングは行われていません。

保守担当者向けメモ

  • 本関数は apr_file_seek を用いてファイルポインタをリセットするため、呼び出し側はファイルの状態が維持されることを期待できます。
  • ap_md5contextTo64 は標準ライブラリではなく、本モジュール独自の Base64 実装を使用しています。
