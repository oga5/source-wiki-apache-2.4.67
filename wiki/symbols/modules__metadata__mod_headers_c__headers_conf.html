headers_conf

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_headers.c:155

目的

mod_headers モジュールにおける、ディレクトリごとの設定を保持するための構造体です。HTTP リクエストおよびレスポンスに対して適用されるヘッダー操作ルールを管理します。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *fixup_in;
    apr_array_header_t *fixup_out;
    apr_array_header_t *fixup_err;
} headers_conf;

入力と出力

  • **入力**: Header および RequestHeader ディレクティブの設定値。
  • **出力**: fixup_in（リクエストヘッダー用）、fixup_out（レスポンスヘッダー用）、fixup_err（エラーレスポンスヘッダー用）の各配列に格納された header_entry のリスト。

前提条件と不変条件

  • 各配列は apr_array_header_t 型であり、header_entry 構造体を保持します。
  • 設定はディレクトリ単位で管理され、リクエスト処理時に ap_get_module_config を通じて取得されます。

アルゴリズムまたは振る舞い

  • 設定読み込み時に apr_pcalloc でメモリが割り当てられます。
  • リクエスト処理の各フェーズ（fixups フックや出力フィルタ）において、この構造体に格納されたルールが do_headers_fixup 関数によって走査され、apr_table_t に対してヘッダーの追加・削除・置換等の操作が適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_headers.c 内のディレクティブ解析関数（設定生成時）。
  • ap_headers_fixup および ap_headers_output_filter（リクエスト処理時）。
  • **呼び出し先**:
  • apr_pcalloc（メモリ確保）。
  • ap_get_module_config（設定取得）。
  • do_headers_fixup（ルール適用）。

関連データ構造

  • header_entry: fixup_in/out/err 配列の各要素として格納される、個別のヘッダー操作ルールを定義する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗する可能性があります。
  • 設定ファイル内のディレクティブ記述順序が適用順序に直結するため、unset や append の順序が不適切な場合、意図しないヘッダー状態になる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_headers の設定の核となるため、新しいヘッダー操作アクションを追加する際は、この構造体の配列を走査するロジックとの整合性に注意してください。
  • fixup_in はリクエストヘッダー、fixup_out は通常のレスポンスヘッダー、fixup_err はエラー時のレスポンスヘッダーをそれぞれ担当します。
