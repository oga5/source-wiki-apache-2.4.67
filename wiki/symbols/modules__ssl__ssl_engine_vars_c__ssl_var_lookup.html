ssl_var_lookup

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:163

目的

リクエスト、接続、またはサーバー情報に基づき、SSL 関連変数や環境変数、時刻情報などの値を解決して文字列として返す。

シグネチャまたは形状

char *ssl_var_lookup(apr_pool_t *p, server_rec *s, conn_rec *c, request_rec *r, char *var)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール（NULL の場合、r、c、またはモジュール設定から自動選択）。
  • s: サーバーレコード。
  • c: 接続レコード。
  • r: リクエストレコード。
  • var: 取得対象の変数名。
  • **出力**:
  • 解決された値を含む文字列（apr_pool_t 上に確保）。該当なしの場合は空文字列 "" を返す。

前提条件と不変条件

  • 戻り値は常に apr_pool_t 上に確保される。
  • result が NULL の場合、最終的に空文字列が返される。
  • resdup フラグが真の場合、apr_pstrdup によりメモリが複製される。

アルゴリズムまたは振る舞い

1. **初期化**: プールが未指定の場合、利用可能なコンテキストからプールを特定する。 2. **リクエスト依存の解決**: r が存在する場合、HTTP_ ヘッダ、REQUEST_ 属性、REMOTE_ 属性、SERVER_ 属性、および ENV: 変数を順次検索する。 3. **接続依存の解決**: c が存在する場合、SSL_ で始まる変数や HTTPS 変数を ssl_var_lookup_ssl へ委譲して解決する。 4. **独立した解決**: 上記で解決できない場合、SSL_VERSION_、SERVER_SOFTWARE、API_VERSION、および現在時刻（TIME_ 系）を解決する。 5. **終了処理**: 解決された値をプールから複製し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • include/http_ssl.h (フック)
  • modules/arch/netware/mod_nw_ssl.c
  • modules/lua/lua_request.c
  • modules/ssl/ssl_engine_init.c
  • modules/ssl/ssl_engine_io.c
  • **呼び出し先**:
  • ssl_var_lookup_ssl (SSL 関連変数の解決)
  • ssl_var_lookup_ssl_version (バージョン情報の解決)
  • ap_get_server_name_for_url, ap_get_server_port, ap_document_root, ap_get_server_banner 等の Apache API
  • apr_table_get, apr_psprintf, apr_pstrdup 等の APR API

関連データ構造

  • SSLConnRec: 接続ごとの SSL 状態保持に使用。
  • SSLModConfigRec: モジュール設定の保持に使用。
  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • 変数名が一致しない場合、空文字列 "" を返す。
  • r や c が NULL の場合、それらに依存する変数は解決されない。
  • SSL_ プレフィックスを持つ変数は、sslconn が存在しない場合解決されない。

保守担当者向けメモ

  • apr_pstrdup を多用するため、頻繁な呼び出しはメモリ消費に影響する可能性がある。
  • MKTIMESTR マクロを使用して時刻関連の文字列生成を行っている。
  • OpenSSL のバージョン差異を吸収するロジックが含まれる場合があるため、変更時は ssl_engine_vars.c 内の条件分岐に注意すること。
