ap_update_vhost_from_headers_ex

定義

  • 種別: function
  • 定義箇所: server/vhost.c:1173

目的

リクエストヘッダー（Host）およびリクエスト URI に基づいて、現在のリクエスト (request_rec) に適切な仮想ホスト設定を適用し、r->server を更新します。

シグネチャまたは形状

int ap_update_vhost_from_headers_ex(request_rec *r, int require_match)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • require_match: 仮想ホストの適合を必須とするかを示すフラグ。
  • **出力**:
  • int: 処理結果のステータスコード（HTTP_OK など）。

前提条件と不変条件

  • r->server は初期状態で設定されている必要があります。
  • r->connection->vhost_lookup_data には、接続時に特定された仮想ホストの候補リストが格納されている場合があります。
  • conf->http_conformance が AP_HTTP_CONFORMANCE_UNSAFE でない場合、RFC 2616 5.2 に従い、絶対 URI と Host ヘッダーが競合した際に Host ヘッダーが上書きされます。

アルゴリズムまたは振る舞い

1. **ホスト名の特定**: リクエスト URI にホスト名が含まれる場合はそれを優先し、含まれない場合は Host ヘッダーを使用します。fix_hostname を呼び出し、ホスト名を正規化します。 2. **ヘッダーの整合性確保**: require_match が偽かつリクエストが既にエラー状態であれば、即座に HTTP_OK を返します。 3. **RFC 準拠の強制**: http_conformance 設定が安全な場合、絶対 URI と Host ヘッダーの両方が存在すれば、Host ヘッダーをリクエスト行の値で置換し、デバッグログを出力します。 4. **仮想ホストの解決**:

  • 接続時に特定された vhost_lookup_data が存在する場合、ホスト名の有無に応じて update_server_from_aliases または check_serverpath を実行します。
  • vhost_lookup_data が存在せず、require_match が真かつホスト名が存在する場合は、ベースサーバー設定に対して update_server_from_aliases を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/protocol.c:1013
  • server/vhost.c:1170
  • **呼び出し先**:
  • fix_hostname (内部関数)
  • construct_host_header (内部関数)
  • update_server_from_aliases (内部関数)
  • check_serverpath (内部関数)
  • ap_get_core_module_config
  • apr_table_get / apr_table_setn
  • ap_log_rerror

関連データ構造

  • request_rec: リクエストの状態管理。
  • core_server_config: サーバーのコア設定（http_conformance 等）。
  • conn_rec: 接続情報（vhost_lookup_data を保持）。

失敗モードと境界条件

  • Host ヘッダーが欠落している場合、fix_hostname の挙動に依存します。
  • require_match が真で、かつホスト名がどの仮想ホストにも適合しない場合、update_server_from_aliases の戻り値がエラーコードを返す可能性があります。

保守担当者向けメモ

  • 本関数は http_vhost.h で宣言されており、プロトコル処理の重要なフェーズで呼び出されます。
  • http_conformance の設定値によって Host ヘッダーの書き換え挙動が変化するため、変更時は RFC 2616 の要件との整合性に注意してください。
