ssl_init_ctx_session_cache

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_init.c:983

目的

SSL コンテキスト（SSL_CTX）に対して、セッションキャッシュのモード設定および関連するコールバック関数の登録を行う。

シグネチャまたは形状

static void ssl_init_ctx_session_cache(server_rec *s,
                                       apr_pool_t *p,
                                       apr_pool_t *ptemp,
                                       modssl_ctx_t *mctx)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p: メインプール。
  • ptemp: 一時プール。
  • mctx: SSL コンテキスト設定を保持する modssl_ctx_t 構造体。
  • **出力**: なし（mctx->ssl_ctx を直接更新する）。

前提条件と不変条件

  • mctx->ssl_ctx が有効な SSL_CTX オブジェクトを指していること。
  • myModConfig(s) を通じて取得される SSLModConfigRec が初期化済みであること。

アルゴリズムまたは振る舞い

1. myModConfig(s) を呼び出し、サーバーの SSL モジュール設定を取得する。 2. SSL_CTX_set_session_cache_mode を使用し、設定された sesscache_mode を SSL_CTX に適用する。 3. mc->sesscache が有効な場合、以下の OpenSSL セッションキャッシュ用コールバックを登録する。

  • 新規セッション保存用: ssl_callback_NewSessionCacheEntry
  • セッション取得用: ssl_callback_GetSessionCacheEntry
  • セッション削除用: ssl_callback_DelSessionCacheEntry

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:1386
  • **呼び出し先**:
  • myModConfig (マクロ)
  • SSL_CTX_set_session_cache_mode (OpenSSL API)
  • SSL_CTX_sess_set_new_cb (OpenSSL API)
  • SSL_CTX_sess_set_get_cb (OpenSSL API)
  • SSL_CTX_sess_set_remove_cb (OpenSSL API)
  • ssl_callback_NewSessionCacheEntry
  • ssl_callback_GetSessionCacheEntry
  • ssl_callback_DelSessionCacheEntry

関連データ構造

  • modssl_ctx_t: SSL コンテキスト設定を保持。
  • SSLModConfigRec: モジュール全体のセッションキャッシュ設定（sesscache_mode, sesscache）を保持。
  • SSL_CTX: OpenSSL の SSL コンテキスト構造体。

失敗モードと境界条件

  • mc->sesscache が NULL の場合、コールバックの登録はスキップされる。
  • OpenSSL API の戻り値チェックは本関数内では行われない。

保守担当者向けメモ

  • 本関数は SSL_CTX の構築パイプラインの一部として実行される。
  • セッションキャッシュの具体的なバックエンド実装（ssl_scache.c）との橋渡しを行うため、コールバック関数のシグネチャ変更時には注意が必要。
