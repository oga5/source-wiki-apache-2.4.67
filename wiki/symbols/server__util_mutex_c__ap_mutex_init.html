ap_mutex_init

定義

  • 種別: function
  • 定義箇所: server/util_mutex.c:145

目的

ミューテックス設定を管理するための内部ハッシュテーブル mxcfg_by_type を初期化し、デフォルトのミューテックス設定を登録します。

シグネチャまたは形状

void ap_mutex_init(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p - 設定のライフサイクルを管理するためのメモリプール。
  • **出力**: なし（副作用としてグローバルなハッシュテーブルを初期化）。

前提条件と不変条件

  • mxcfg_by_type が初期化済みである場合、関数は即座に終了します。
  • サーバーの起動期間中、設定は pconf プールによって保持されます。

アルゴリズムまたは振る舞い

1. mxcfg_by_type が既に存在するか確認し、存在すれば処理を終了します。 2. apr_hash_make(p) を使用してハッシュテーブルを生成します。 3. apr_pool_cleanup_register を呼び出し、プール破棄時にハッシュテーブルをクリアするクリーンアップ処理を登録します。 4. mutex_cfg_t 構造体を割り当て、デフォルト値（mech = APR_LOCK_DEFAULT, dir = ap_runtime_dir_relative(p, "")）を設定します。 5. キー "default" を用いて、生成したデフォルト設定をハッシュテーブルに格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core.c:5112
  • server/util_mutex.c:266
  • **呼び出し先**:
  • apr_hash_make
  • apr_pool_cleanup_register
  • apr_pcalloc
  • ap_runtime_dir_relative
  • apr_hash_set

関連データ構造

  • mutex_cfg_t: ミューテックスのメカニズムやディレクトリ情報を保持する構造体。
  • mxcfg_by_type: ミューテックス型をキーとして設定を管理する内部ハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc や apr_hash_make が APR の規約に従いエラーを発生させます。
  • 複数回呼び出された場合、最初の呼び出し以外はハッシュテーブルの再生成を行わず、安全に無視されます。

保守担当者向けメモ

  • 本関数はサーバー起動時の初期化フェーズで実行されます。
  • mxcfg_by_type は静的変数として定義されており、この関数を通じてのみ初期化されます。
