hc_create_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_hcheck.c:82

目的

mod_proxy_hcheck モジュールで使用される、サーバー単位の構成管理構造体（sctx_t）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *hc_create_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定割り当て用の APR プール。
  • s: 設定対象のサーバーレコード。
  • **出力**:
  • 初期化済みの sctx_t 構造体へのポインタ。

前提条件と不変条件

  • p は有効な APR プールであること。
  • 戻り値の sctx_t 構造体は、templates（配列）、conditions（テーブル）、hcworkers（ハッシュ）が初期化済みであること。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して sctx_t 構造体分のメモリをゼロクリアして確保する。 2. サーバーレコード s をコンテキストに保持する。 3. ctx->p としてサブプールを作成し、デバッグ用に "proxy_hcheck" というタグを付与する。 4. templates 配列（初期容量10）、conditions テーブル（初期容量10）、hcworkers ハッシュをそれぞれ作成する。 5. 初期化されたコンテキストポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_hcheck.c:120
  • modules/proxy/mod_proxy_hcheck.c:1363（モジュール設定の登録時）
  • **呼び出し先**:
  • apr_pcalloc
  • apr_pool_create
  • apr_pool_tag
  • apr_array_make
  • apr_table_make
  • apr_hash_make

関連データ構造

  • sctx_t: 本関数で生成されるサーバーコンテキスト。
  • hc_template_t: templates 配列に格納されるヘルスチェック設定。

失敗モードと境界条件

  • APR のメモリ割り当て関数（apr_pcalloc 等）が失敗した場合、APR の仕様に従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は mod_proxy_hcheck の設定ライフサイクルの起点となる。
  • ctx->p はサブプールとして作成されているため、サーバー設定の破棄時に適切にクリーンアップされる。
