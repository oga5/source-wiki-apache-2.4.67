ap_get_basic_auth_components

定義

  • 種別: function
  • 定義箇所: server/protocol.c:1802

目的

HTTPリクエストからBasic認証のユーザー名とパスワードを抽出し、デコードして取得する。ap_get_basic_auth_pw() の代替として推奨される関数。

シグネチャまたは形状

apr_status_t ap_get_basic_auth_components(const request_rec *r,
                                          const char **username,
                                          const char **password)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • username: ユーザー名を格納するポインタへのポインタ（NULL可）。
  • password: パスワードを格納するポインタへのポインタ（NULL可）。
  • **出力**:
  • 成功時は APR_SUCCESS を返し、username および password 引数に値を設定する。
  • 失敗時は APR_EINVAL を返す。

前提条件と不変条件

  • r->pool を使用してメモリを確保するため、リクエストのライフサイクル内で有効であること。
  • headers_in テーブルに認証ヘッダーが存在すること。

アルゴリズムまたは振る舞い

1. リクエストがプロキシ経由か否かに応じて Proxy-Authorization または Authorization ヘッダーを取得する。 2. ヘッダーが存在しない場合、またはスキームが "Basic" でない場合は APR_EINVAL を返す。 3. ヘッダー値から認証情報を抽出し、ap_pbase64decode を用いてBase64デコードを行う。 4. デコードされた文字列を : で分割し、ユーザー名とパスワードを分離する。 5. 分離した値をそれぞれ username および password 引数に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述なし。
  • **呼び出し先**: apr_table_get, ap_cstr_casecmp, ap_getword, ap_pbase64decode, ap_getword_nulls。

関連データ構造

  • request_rec: リクエスト情報およびヘッダーテーブル (headers_in) を保持。

失敗モードと境界条件

  • 認証ヘッダーが欠落している場合: APR_EINVAL を返す。
  • "Basic" 以外の認証スキームの場合: APR_EINVAL を返す。
  • Base64デコード処理: 入力文字列が不正な場合でもエラーを返さず、解析可能な範囲までを処理する（仕様上の制限）。

保守担当者向けメモ

  • 本関数は ap_get_basic_auth_pw() の後継として設計されている。
  • Base64デコードの不完全なエラーハンドリングは、2.4.x系では現状維持とする方針である。
