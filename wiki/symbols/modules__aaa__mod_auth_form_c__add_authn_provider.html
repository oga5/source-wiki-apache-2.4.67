add_authn_provider

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:144

目的

Apache HTTP Server の設定ファイルで指定された認証プロバイダーを解析し、auth_form_config_rec 構造体のプロバイダーリストに登録します。

シグネチャまたは形状

static const char *add_authn_provider(cmd_parms *cmd, void *config, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: 現在のディレクトリ設定構造体 (auth_form_config_rec)。
  • arg: 設定ファイルから渡されるプロバイダー名。
  • **出力**: 成功時は NULL を返し、失敗時はエラーメッセージを含む文字列を返します。

前提条件と不変条件

  • config は auth_form_config_rec 型としてキャスト可能である必要があります。
  • 指定されたプロバイダーは、実行時に ap_lookup_provider で解決可能である必要があります。
  • 登録されるプロバイダーは、check_password 関数を実装している必要があります。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して新しい authn_provider_list ノードを割り当てます。 2. ap_lookup_provider を呼び出し、指定された名前の認証プロバイダーを検索・キャッシュします。 3. プロバイダーが見つからない場合、または check_password 関数をサポートしていない場合はエラーを返します。 4. 既存の conf->providers リストの末尾を探索し、新しいノードをリストに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_form.c:354 (AP_INIT_ITERATE による AuthFormProvider ディレクティブ)
  • modules/aaa/mod_auth_basic.c:202 (AuthBasicProvider ディレクティブ)
  • modules/aaa/mod_auth_digest.c:694 (AuthDigestProvider ディレクティブ)
  • **呼び出し先**:
  • ap_lookup_provider (Apache API)
  • apr_pcalloc (APR API)
  • apr_psprintf (APR API)

関連データ構造

  • auth_form_config_rec: ディレクトリごとの認証設定を保持する構造体。
  • authn_provider_list: 認証プロバイダーを保持する連結リスト構造体。

失敗モードと境界条件

  • **プロバイダー未検出**: ap_lookup_provider が NULL を返した場合、「Unknown Authn provider」エラーを返します。
  • **機能不適合**: プロバイダーが check_password を実装していない場合、「doesn't support Form Authentication」エラーを返します。
  • **メモリ不足**: apr_pcalloc が失敗した場合、APR のメモリ管理ポリシーに従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は mod_auth_form だけでなく、mod_auth_basic や mod_auth_digest からも参照されています。変更を加える際は、これら複数のモジュールへの影響を考慮してください。
  • リストの末尾への追加は線形探索（while (last->next)）で行われるため、プロバイダー数が極端に多い場合はパフォーマンスに影響する可能性があります。
