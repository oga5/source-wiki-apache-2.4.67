uldap_connection_init

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:308

目的

LDAP 接続構造体 (util_ldap_connection_t) を初期化し、LDAP サーバーへの接続確立およびプロトコルオプション（SSL/TLS、リファラル追跡、タイムアウト等）の設定を行う。

シグネチャまたは形状

static int uldap_connection_init(request_rec *r, util_ldap_connection_t *ldc)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • ldc: 初期化対象の LDAP 接続構造体。
  • **出力**:
  • int: 成功時は 0 (または APR_SUCCESS)、失敗時は APR エラーコードを返す。
  • 副作用: ldc->ldap に LDAP ハンドルが格納され、ldc->reason にエラー理由が設定される。

前提条件と不変条件

  • ldc は呼び出し元で確保されていること。
  • r->pool を使用してメモリ管理が行われるため、リクエストのライフサイクルに依存する。
  • st->verify_svr_cert 等の設定値が util_ldap_state_t から取得可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: apr_ldap_init を呼び出し、LDAP ハンドルを取得する。 2. **リファラル設定**: ChaseReferrals が有効な場合、uldap_rebind_add を実行して再バインド用参照を追加する。 3. **プロトコル設定**: LDAP V3 をデフォルトとし、クライアント証明書、SSL/TLS オプションを適用する。 4. **オプション設定**: エイリアス参照解除 (LDAP_OPT_DEREF)、リファラル追跡、ホップ制限を設定する。 5. **セキュリティ設定**: サーバー証明書の検証モード (APR_LDAP_OPT_VERIFY_CERT 等) を設定する。 6. **タイムアウト設定**: LDAP_OPT_NETWORK_TIMEOUT および LDAP_OPT_TIMEOUT を設定する。 7. **エラーハンドリング**: 各ステップで失敗した場合は uldap_connection_unbind を呼び出し、リソースを解放してエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap.c:614
  • modules/ldap/util_ldap.c:668
  • **呼び出し先**:
  • apr_ldap_init, apr_ldap_set_option
  • uldap_rebind_add
  • uldap_connection_unbind
  • ldap_set_option (SDK 依存)

関連データ構造

  • util_ldap_connection_t: 接続状態と設定を保持。
  • util_ldap_state_t: モジュール全体の設定（タイムアウト、証明書検証等）。
  • apr_ldap_err_t: LDAP 操作の結果とエラー詳細。

失敗モードと境界条件

  • apr_ldap_init 失敗時: APR_EGENERAL を返す。
  • リファラル追加失敗時: メモリ不足等の理由で uldap_connection_unbind を実行しエラーを返す。
  • オプション設定失敗時: 接続を解放し、result->rc を返す。
  • Solaris SDK 等の環境依存: #ifdef による条件分岐があり、プラットフォームにより挙動が異なる。

保守担当者向けメモ

  • 本関数は apr-util のバージョンや LDAP SDK の種類に強く依存する #ifdef ブロックを含んでいる。
  • ldc->reason にエラー詳細を格納することで、呼び出し元がログ出力を行えるように設計されている。
  • 接続の再利用時には must_rebind フラグの確認が必要となるため、呼び出し元のロジックと併せて確認すること。
