lua_table_get

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_apr.c:72

目的

Lua スクリプトから Apache Portable Runtime (APR) のテーブル（apr_table_t）内の値をキー指定で取得する。

シグネチャまたは形状

static int lua_table_get(lua_State *L)

入力と出力

  • **入力**:
  • Lua スタック第1引数: Apr.Table 型のユーザーデータ（req_table_t）。
  • Lua スタック第2引数: 取得対象のキー名（文字列）。
  • **出力**:
  • Lua スタックへ取得した値をプッシュし、戻り値として 1 を返す。値が存在しない場合は nil がプッシュされる。

前提条件と不変条件

  • 第1引数は ap_lua_check_apr_table によって検証された有効な req_table_t 構造体である必要がある。
  • 第2引数は luaL_checkstring により文字列型であることが保証される。

アルゴリズムまたは振る舞い

1. ap_lua_check_apr_table を呼び出し、Lua スタックから req_table_t ポインタを取得する。 2. luaL_checkstring を呼び出し、キー名を取得する。 3. apr_table_get を使用して、内部の apr_table_t から指定されたキーに対応する値を取得する。 4. 取得した値を lua_pushstring で Lua スタックに積む。 5. スタックに積んだ値の数（1）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_apr.c:83 （lua_table_methods テーブル内でメソッドとして登録されている）。
  • **呼び出し先**:
  • ap_lua_check_apr_table
  • luaL_checkstring
  • apr_table_get
  • lua_pushstring

関連データ構造

  • req_table_t: apr_table_t をラップし、Lua ユーザーデータとして扱うための構造体。

失敗モードと境界条件

  • キーが存在しない場合、apr_table_get は NULL を返し、lua_pushstring は Lua の nil をスタックに積む。
  • 第1引数が Apr.Table 型でない場合、ap_lua_check_apr_table がエラーを発生させる。

保守担当者向けメモ

  • 本関数は Apr.Table メタテーブルの __index メソッドとして機能することを想定している。
  • apr_table_get の戻り値が NULL の場合の挙動は Lua の API 仕様に従い、適切に nil として処理される。
