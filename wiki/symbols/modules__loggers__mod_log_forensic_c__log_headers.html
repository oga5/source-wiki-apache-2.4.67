log_headers

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_forensic.c:168

目的

リクエストヘッダーのキーと値をシリアライズし、フォレンジックログ用のバッファへ書き込むためのコールバック関数です。

シグネチャまたは形状

static int log_headers(void *h_, const char *key, const char *value)

入力と出力

  • **入力**:
  • h_: hlog 構造体へのポインタ（void * として受け取り内部でキャスト）。
  • key: ログに記録するヘッダー名。
  • value: ログに記録するヘッダー値。
  • **出力**:
  • 常に 1 を返します（apr_table_do の反復処理を継続するため）。

前提条件と不変条件

  • h_ は hlog 構造体への有効なポインタであること。
  • hlog 構造体内の pos（現在の書き込み位置）および end（バッファ終端）が適切に初期化されていること。
  • log_escape 関数がバッファオーバーフローを防止しているため、本関数内での境界チェックは省略されています。

アルゴリズムまたは振る舞い

1. h_ を hlog 型にキャストします。 2. バッファの現在位置に区切り文字 | を書き込み、pos を進めます。 3. log_escape を呼び出し、key をエスケープ処理してバッファに書き込みます。 4. 区切り文字 : を書き込み、pos を進めます。 5. log_escape を呼び出し、value をエスケープ処理してバッファに書き込みます。 6. 処理成功として 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_log_forensic.c:219 において apr_table_do の引数として呼び出されます。
  • **呼び出し先**: log_escape

関連データ構造

  • hlog: ログ書き込み用のバッファポインタ（pos）、バッファ終端（end）を保持する構造体。

失敗モードと境界条件

  • バッファ容量が不足する場合、log_escape 内部でアサーションエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は apr_table_do から呼び出されるため、シグネチャを変更する場合は apr_table_do の仕様に準拠する必要があります。
  • log_escape がバッファの安全性を保証している前提に依存しているため、log_escape の実装を変更する際は、本関数が想定するバッファ管理の不変条件が維持されているか確認してください。
