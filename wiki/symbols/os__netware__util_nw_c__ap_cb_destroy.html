ap_cb_destroy

定義

  • 種別: function
  • 定義箇所: os/netware/util_nw.c:63

目的

NetWare Loadable Module (NLM) の終了処理において、サーバー停止イベントの監視解除およびインターフェース参照の解放を行うクリーンアップ関数です。

シグネチャまたは形状

void ap_cb_destroy(void *a)

入力と出力

  • **入力**: void *a（未使用。終了ハンドラとしてのシグネチャ適合用）
  • **出力**: なし（void）

前提条件と不変条件

  • 本関数は、NXVmRegisterExitHandler を介して終了ハンドラとして登録されている必要があります。
  • 内部で使用されるグローバル変数 eh（イベントハンドル）、ref、dum（インターフェース参照）が有効な状態で初期化されている必要があります。

アルゴリズムまたは振る舞い

1. UnRegisterEventNotification(eh) を呼び出し、サーバー停止イベント（EVENT_DOWN_SERVER）の購読を解除します。 2. NX_UNWRAP_INTERFACE(ref) を呼び出し、保持していたインターフェース参照を解放します。 3. NX_UNWRAP_INTERFACE(dum) を呼び出し、同様にインターフェース参照を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: os/netware/util_nw.c:108 において、NXVmRegisterExitHandler の引数として登録されています。
  • **呼び出し先**:
  • UnRegisterEventNotification (NetWare OS API)
  • NX_UNWRAP_INTERFACE (NetWare Kernel Services マクロ)

関連データ構造

  • eh: イベント通知ハンドル。
  • ref, dum: インターフェース参照を保持する変数。

失敗モードと境界条件

  • eh が無効な場合や、既に解放済みのインターフェースに対して NX_UNWRAP_INTERFACE を呼び出した場合の挙動は、NetWare OS API の仕様に依存します。

保守担当者向けメモ

  • 本関数は NetWare 環境特有の終了処理を担います。他のプラットフォームへの移植時には、このクリーンアップ機構を該当 OS の終了フックへ置き換える必要があります。
