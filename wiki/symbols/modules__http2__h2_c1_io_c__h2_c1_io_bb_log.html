h2_c1_io_bb_log

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:61

目的

HTTP/2 クライアント接続（C1）における apr_bucket_brigade の内容をデバッグログとして可視化する。

シグネチャまたは形状

static void h2_c1_io_bb_log(conn_rec *c, int stream_id, int level,
                            const char *tag, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • c: 接続情報 (conn_rec)。
  • stream_id: 関連するストリーム ID（関数内で使用されず無視される）。
  • level: ログレベル（APLOG_DEBUG 等）。
  • tag: ログ出力時に付与される識別タグ。
  • bb: 内容をダンプする対象の apr_bucket_brigade。
  • **出力**: なし（ap_log_cerror を介してサーバーログへ出力）。

前提条件と不変条件

  • bb が NULL の場合、ログには (null) と記録される。
  • ブリゲードが空の場合、ログには (empty) と記録される。
  • ログ出力用の内部バッファは 16KB に制限されており、これを超えるブリゲードの内容は切り詰められる。

アルゴリズムまたは振る舞い

1. 16KB のスタックバッファを初期化する。 2. bb が存在する場合、ブリゲード内の各バケットを順次走査する。 3. バケットがメタデータ（EOS, FLUSH, EOR, H2EOS）の場合は対応する文字列を、データバケットの場合は種類（file, heap, transient 等）と長さをバッファに追記する。 4. 走査完了後、ap_log_cerror を使用して h2_session(<conn_id>)-<tag>: <log_line> の形式でログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c1_io.c 内の各処理（マクロ C1_IO_BB_LOG を介して呼び出される）。
  • **呼び出し先**: apr_snprintf（文字列構築）、ap_log_cerror（ログ出力）、および APR_BRIGADE_* / APR_BUCKET_* マクロ群。

関連データ構造

  • apr_bucket_brigade: 転送されるデータ単位のリスト。
  • apr_bucket: ブリゲードを構成する個別のデータ要素。
  • conn_rec: Apache の接続コンテキスト。

失敗モードと境界条件

  • バッファサイズ（16KB）を超えるブリゲードの内容は、バッファの末尾で切り捨てられる。
  • bmax（バッファ残量）が 0 に達した時点で走査を終了する。
  • b->length が (apr_size_t)-1 の場合、長さは -1 として記録される。

保守担当者向けメモ

  • 本関数は APLOGNO を使用していない。
  • stream_id 引数は現在使用されていないが、将来的なデバッグ情報の拡充のために保持されている。
