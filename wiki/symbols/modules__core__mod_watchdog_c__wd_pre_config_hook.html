wd_pre_config_hook

定義

  • 種別: function
  • 定義箇所: modules/core/mod_watchdog.c:386

目的

Apache HTTP Server の設定フェーズにおいて、mod_watchdog が使用するデフォルトの Watchdog インスタンスを生成し、プロセス間排他制御用のミューテックスを登録する。

シグネチャまたは形状

static int wd_pre_config_hook(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • plog, ptemp: ログおよび一時用メモリプール（本関数内では未使用）。
  • **出力**:
  • 成功時は OK を返す。
  • 失敗時は apr_status_t 型の戻り値（エラーコード）を返す。

前提条件と不変条件

  • ap_mpm_query を用いて MPM がフォークをサポートしているかを確認する。
  • pconf プールは、Watchdog インスタンスおよびミューテックスのライフサイクルを管理するために使用される。

アルゴリズムまたは振る舞い

1. ap_mpm_query で AP_MPMQ_IS_FORKED を取得し、MPM の動作モードを確認する。 2. ap_watchdog_get_instance を呼び出し、以下のインスタンスを生成する:

  • AP_WATCHDOG_SINGLETON: シングルトンモードのインスタンス。
  • AP_WATCHDOG_DEFAULT: デフォルトのインスタンス。

    • 3. MPM がフォークをサポートしていない場合、親プロセス用の AP_WATCHDOG_DEFAULT インスタンスを追加で生成する。 4. ap_mutex_register を呼び出し、wd_proc_mutex_type に関連付けられたプロセス間ミューテックスを登録する。 5. 正常終了時に OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_watchdog.c:647 にて ap_hook_pre_config に登録されている。
  • **呼び出し先**:
  • ap_mpm_query
  • ap_watchdog_get_instance
  • ap_mutex_register

関連データ構造

  • ap_watchdog_t: 生成される監視インスタンスの構造体。
  • apr_pool_t: リソース管理に使用されるメモリプール。

失敗モードと境界条件

  • ap_watchdog_get_instance または ap_mutex_register が APR_SUCCESS 以外の値を返した場合、そのエラーコードを即座に返して処理を中断する。

保守担当者向けメモ

  • 本関数はサーバー起動時の設定フェーズで実行されるため、ここで生成されたインスタンスはサーバーのライフサイクル全体で有効となる。
  • mpm_is_forked の判定により、非フォーク型 MPM における親プロセス専用の Watchdog 生成ロジックが分岐するため、MPM 固有の挙動変更時には注意が必要である。
