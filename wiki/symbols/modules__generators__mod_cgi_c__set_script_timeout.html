set_script_timeout

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgi.c:164

目的

Apache 設定ファイル内の CGIScriptTimeout（または CGIDScriptTimeout）ディレクティブを解析し、ディレクトリごとの CGI スクリプト実行タイムアウト値を設定する。

シグネチャまたは形状

static const char *set_script_timeout(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: cgi_dirconf 構造体へのポインタ。
  • arg: 設定ファイルから読み込まれたタイムアウト値の文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列（"CGIScriptTimeout has wrong format"）を返す。

前提条件と不変条件

  • dummy は cgi_dirconf 型としてキャスト可能であること。
  • arg は秒単位の数値として解析可能であること。

アルゴリズムまたは振る舞い

1. ap_timeout_parameter_parse 関数を呼び出し、arg を解析して dc->timeout に格納する。 2. 解析単位として秒（"s"）を指定する。 3. 解析が成功した場合は NULL を返し、失敗した場合はエラーメッセージを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:183（AP_INIT_TAKE1 マクロ経由）
  • modules/generators/mod_cgid.c:1184（AP_INIT_TAKE1 マクロ経由）
  • **呼び出し先**:
  • ap_timeout_parameter_parse（Apache コア API）

関連データ構造

  • cgi_dirconf: CGI モジュールのディレクトリ単位の設定を保持する構造体。timeout メンバを更新する。

失敗モードと境界条件

  • arg が数値として不正な形式である場合、ap_timeout_parameter_parse が APR_SUCCESS 以外を返し、エラーメッセージを返却する。

保守担当者向けメモ

  • 本関数は mod_cgi と mod_cgid の両方から参照されている。
  • タイムアウト値の解析には Apache 標準の ap_timeout_parameter_parse を利用しているため、単位指定の変更や拡張が必要な場合はこの API の仕様を確認すること。
