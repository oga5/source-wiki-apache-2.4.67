set_ftp_list_on_wildcard

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:81

目的

Apache HTTP Server の設定ディレクティブ ProxyFtpListOnWildcard の値を解析し、ディレクトリ単位の設定構造体 proxy_ftp_dir_conf に反映させる。

シグネチャまたは形状

static const char *set_ftp_list_on_wildcard(cmd_parms *cmd, void *dconf, int flag)

入力と出力

  • cmd: 設定コマンドのパラメータ（cmd_parms 構造体）。
  • dconf: 設定対象のディレクトリ設定構造体（proxy_ftp_dir_conf）。
  • flag: ディレクティブで指定された真偽値（int）。
  • 戻り値: 成功時は NULL を返す。

前提条件と不変条件

  • dconf は proxy_ftp_dir_conf 型としてキャスト可能であること。
  • 設定変更後、ftp_list_on_wildcard_set フラグが 1 に設定される。

アルゴリズムまたは振る舞い

1. 引数 dconf を proxy_ftp_dir_conf 型のポインタにキャストする。 2. flag の値を conf->ftp_list_on_wildcard に代入する。 3. conf->ftp_list_on_wildcard_set を 1 に設定し、設定が明示的に行われたことを記録する。 4. NULL を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/mod_proxy_ftp.c:2098 において、AP_INIT_FLAG マクロを通じて設定ディレクティブ ProxyFtpListOnWildcard のハンドラとして登録されている。
  • 呼び出し先: なし。

関連データ構造

  • proxy_ftp_dir_conf: FTP プロキシのディレクトリ単位の設定を保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーチェックを行わず、常に NULL を返す。

保守担当者向けメモ

  • 本関数は Apache の設定解析フェーズでのみ使用される。
  • ftp_list_on_wildcard_set フラグは、設定の継承やマージの際に、デフォルト値と明示的な設定値を区別するために使用される。
