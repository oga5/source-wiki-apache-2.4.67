stream_pri_cmp

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:164

目的

HTTP/2 セッション内における2つのストリームの優先順位を比較し、ソートやスケジューリングのための順序を決定する。

シグネチャまたは形状

int stream_pri_cmp(int sid1, int sid2, void *ctx)
  • sid1, sid2: 比較対象のストリームID。
  • ctx: h2_session 構造体へのポインタ。

入力と出力

  • 入力: 2つのストリームIDとセッションコンテキスト。
  • 出力:
  • 0: 両ストリームが同一、または優先順位が等価。
  • 1: sid1 が存在しない場合、または sid1 の優先順位が sid2 より低い。
  • -1: sid2 が存在しない場合、または sid1 の優先順位が sid2 より高い。

前提条件と不変条件

  • ctx は有効な h2_session 構造体を指している必要がある。
  • nghttp2_session_find_stream を使用して、セッション内のストリームオブジェクトを取得できること。

アルゴリズムまたは振る舞い

1. nghttp2_session_find_stream を呼び出し、sid1 および sid2 に対応する nghttp2_stream オブジェクトを取得する。 2. 両者が同一オブジェクトであれば 0 を返す。 3. s1 が存在しない場合は 1 を返し、s2 が存在しない場合は -1 を返す。 4. 両者が存在する場合、spri_cmp を呼び出して詳細な優先順位比較を行い、その結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/http2/h2_mplx.c: ストリーム処理時の優先順位比較に使用。
  • modules/http2/h2_session.c: ストリーム取得および再優先順位付け（h2_mplx_c1_reprioritize）の引数として使用。
  • 呼び出し先:
  • nghttp2_session_find_stream (nghttp2 ライブラリ)
  • spri_cmp (内部関数)

関連データ構造

  • h2_session: セッション状態と nghttp2 セッションを保持する構造体。
  • nghttp2_stream: nghttp2 ライブラリが管理するストリーム構造体。

失敗モードと境界条件

  • ストリームIDが nghttp2 セッション内に存在しない場合、nghttp2_session_find_stream は NULL を返す。本関数はこのケースを 1 または -1 を返すことで処理する。

保守担当者向けメモ

  • 本関数は h2_stream_pri_cmp_fn 型のコールバックとして機能する。
  • spri_cmp への依存が強いため、優先順位付けのロジックを変更する場合は spri_cmp を確認すること。
