allowed_port

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_connect.c:117

目的

指定されたポート番号が、プロキシ経由の CONNECT メソッドで許可されているかを判定します。

シグネチャまたは形状

static int allowed_port(connect_conf *conf, int port)

入力と出力

  • **入力**:
  • conf: サーバーごとの設定を保持する connect_conf 構造体へのポインタ。
  • port: 接続先のポート番号（整数）。
  • **出力**:
  • 1: 許可されている場合。
  • 0: 許可されていない場合。

前提条件と不変条件

  • conf->allowed_connect_ports は apr_array_header_t 型の配列として初期化されている必要があります。
  • port_range 構造体は first および last メンバを持ち、ポート範囲を定義します。

アルゴリズムまたは振る舞い

1. conf->allowed_connect_ports が空の場合、デフォルトの HTTPS ポート（443）および SNEWS ポート（563）のみを許可して 1 を返します。 2. 配列が空でない場合、設定されたすべての port_range を走査します。 3. 指定された port がいずれかの範囲内（first <= port <= last）であれば 1 を返します。 4. どの範囲にも該当しない場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_connect.c:226 (proxy_connect_handler 内で接続先ポートの検証に使用)
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • connect_conf: 許可ポートリストを保持する設定構造体。
  • port_range: 許可ポートの範囲（first, last）を定義する構造体。

失敗モードと境界条件

  • 設定が空の場合、ハードコードされたデフォルトポート（443, 563）のみが許可されます。
  • port が負の値や範囲外の値であっても、ロジック上は単なる数値比較として処理されます。

保守担当者向けメモ

  • 本関数は AllowCONNECT ディレクティブで設定されたポート範囲を検証します。
  • 許可ポートの追加や変更は set_allowed_ports 関数を通じて行われます。
