log_after

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_forensic.c:233

目的

リクエストの終了時に、フォレンジックログへ完了を示すレコードを書き込む。

シグネチャまたは形状

static int log_after(request_rec *r)

入力と出力

  • 入力: request_rec *r（現在のリクエスト情報）
  • 出力: int（Apache のステータスコード OK または DECLINED）

前提条件と不変条件

  • r->server->module_config から取得した fcfg 構造体に有効なファイルディスクリプタ fd が存在すること。
  • r->request_config から取得したリクエスト ID id が NULL でないこと。
  • 書き込み操作は apr_file_write を経由し、書き込みバイト数が期待値と一致することを ap_assert で保証する。

アルゴリズムまたは振る舞い

1. サーバー設定 fcfg およびリクエスト ID を取得する。 2. 設定または ID が欠如している場合は DECLINED を返す。 3. apr_pstrcat を使用し、リクエスト ID を用いて "-<id>\n" という形式の文字列を生成する。 4. 生成した文字列を cfg->fd が指すファイルへ書き込む。 5. 書き込み成功を確認後、OK を返す。

呼び出し元と呼び出し先

  • 呼び出し元: ap_hook_log_transaction に登録されており、リクエスト処理の終了フェーズで呼び出される（modules/loggers/mod_log_forensic.c:277）。
  • 呼び出し先: ap_get_module_config, apr_pstrcat, apr_file_write, ap_assert（いずれも Apache/APR API）。

関連データ構造

  • request_rec: リクエストごとのコンテキストを保持。
  • fcfg: モジュール固有の設定構造体。ログ出力先のファイルディスクリプタを保持。

失敗モードと境界条件

  • cfg->fd が未設定、またはリクエスト ID が取得できない場合はログを出力せず DECLINED を返す。
  • 書き込み処理が失敗した場合、または書き込みバイト数が不一致の場合は ap_assert によりプロセスが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数はリクエスト終了時のフックとして動作するため、高負荷環境では I/O 負荷に注意すること。
  • r->pool を使用してメモリを確保しているため、リクエスト終了時に自動的にメモリが解放される。
