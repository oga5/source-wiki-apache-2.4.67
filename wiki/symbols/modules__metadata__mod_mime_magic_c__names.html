names

定義

  • 種別: struct 配列
  • 定義箇所: modules/metadata/mod_mime_magic.c:276

目的

ファイル内容のヒューリスティック判定に使用する、既知の文字列パターンとそれに対応するファイルタイプ識別子の静的マッピングテーブル。

シグネチャまたは形状

static const struct names {
    const char *name;
    short type;
} names[];

配列の末尾は {NULL, 0} で終端されます。

入力と出力

  • **入力**: 判定対象のファイル先頭部分の文字列。
  • **出力**: 一致したパターンに対応する short 型の識別子（L_HTML, L_C, L_MAKE, L_MACH, L_PAS, L_PLI, L_MAIL, L_NEWS など）。

前提条件と不変条件

  • 配列内の要素は、判定のヒット率を最適化するために順序付けられています。
  • 新しい要素を追加する際は、既存の判定ロジックへの影響を慎重に検討する必要があります。

アルゴリズムまたは振る舞い

ファイル内容の解析において、特定のキーワードやタグ（HTMLタグ、C言語のプリプロセッサ、Makefileの構文、メールヘッダー等）を検索し、ファイルの種類を推論するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内でこのシンボルを直接参照している箇所は、提供された抜粋内には存在しません。
  • **呼び出し先**: なし（データ構造であるため）。
  • 注記: ツールが提示した include/ 配列内の names への言及は、本シンボルとは無関係な汎用的な変数名や構造体名であり、本シンボルへの参照ではありません。

関連データ構造

  • struct magic: マジックナンバーのルールを保持する連結リスト。
  • magic_req_rec: リクエスト単位の解析結果を保持する構造体。

失敗モードと境界条件

  • 一致するパターンが見つからない場合、あるいはファイルが空の場合、判定は行われません。
  • 複数のパターンが重複する可能性があるため、配列の順序が判定結果に影響を与えます。

保守担当者向けメモ

  • NOTDEF マクロで囲まれた Fortran 関連の定義は現在無効化されています。
  • このテーブルは mod_mime_magic のヒューリスティック判定の核となるため、変更時には既存のファイルタイプ判定への影響を十分にテストしてください。
