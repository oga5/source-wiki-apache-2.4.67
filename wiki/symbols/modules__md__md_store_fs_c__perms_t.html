perms_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_store_fs.c:44

目的

ファイルシステム上のディレクトリおよびファイルに対して適用するアクセス権限（パーミッション）を保持するためのヘルパー構造体です。

シグネチャまたは形状

typedef struct {
    apr_fileperms_t dir;
    apr_fileperms_t file;
} perms_t;

入力と出力

  • **入力**: apr_fileperms_t 型の権限値（ディレクトリ用およびファイル用）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • apr_fileperms_t 型は APR（Apache Portable Runtime）のファイル権限定義に準拠している必要があります。
  • md_store_fs_t 構造体の一部として、ストア内の各グループに対する権限設定を保持するために使用されます。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持のみを行い、ロジックは持ちません。
  • md_store_fs.c 内の gperms 関数などが、この構造体を参照して特定のグループに対する適切な権限設定を取得するために利用します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/md/md_store_fs.c:54 (def_perms 変数)
  • modules/md/md_store_fs.c:55 (group_perms 配列)
  • modules/md/md_store_fs.c:396 (gperms 関数内)
  • modules/md/md_store_fs.c:518 (ローカル変数)
  • modules/md/md_store_fs.c:635 (ローカル変数)
  • **呼び出し先**:
  • 情報なし（本構造体はデータ保持用であり、関数呼び出しは行いません）。

関連データ構造

  • md_store_fs_t: 本構造体を保持し、ファイルシステムの権限管理を行うメイン構造体。
  • apr_fileperms_t: APR が提供するファイル権限の型。

失敗モードと境界条件

  • 構造体自体に失敗モードはありません。
  • 保持する apr_fileperms_t 値がプラットフォームでサポートされていない場合、ファイルシステム操作時にエラーが発生する可能性があります。

保守担当者向けメモ

  • md_store_fs_group_perms_set 等の関数を通じて、この構造体の値が動的に設定される点に注意してください。
  • WIN32 環境など、プラットフォーム固有の権限モデルとの差異については md_store_fs.c 内の関連する設定関数を確認してください。
