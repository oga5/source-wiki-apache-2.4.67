md_acme_acct_matches_md

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_acct.c:244

目的

ACME アカウントが、指定された管理対象ドメイン（md_t）の要件（CA URL および EAB 設定）に適合するかを判定します。

シグネチャまたは形状

int md_acme_acct_matches_md(md_acme_acct_t *acct, const md_t *md)

入力と出力

  • **入力**:
  • acct: 照合対象の md_acme_acct_t 構造体へのポインタ。
  • md: 比較基準となる md_t 構造体へのポインタ。
  • **出力**:
  • int: 適合する場合は非ゼロ（真）、適合しない場合は 0（偽）を返します。

前提条件と不変条件

  • md の ca_effective URL がアカウントの URL と一致している必要があります。
  • EAB（External Account Binding）が設定されていない場合、アカウント側も EAB を持たないもののみが適合とみなされます。

アルゴリズムまたは振る舞い

1. md_acme_acct_matches_url を呼び出し、CA URL の一致を確認します。不一致なら 0 を返します。 2. ドメイン側で EAB（ca_eab_kid および ca_eab_hmac）が指定されていない場合:

  • アカウント側も EAB を持たない（eab_kid または eab_hmac が NULL）場合にのみ適合と判定します。

    • 3. ドメイン側で EAB が指定されている場合:

  • アカウント側が EAB を持たない場合は 0 を返します。
  • アカウント側の eab_kid および eab_hmac が、ドメイン側の値と完全に一致するかを strcmp で検証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c:581
  • modules/md/md_acme_acct.c:285
  • **呼び出し先**:
  • md_acme_acct_matches_url
  • strcmp (標準ライブラリ)

関連データ構造

  • md_acme_acct_t: アカウント情報を保持する構造体。
  • md_t: 管理対象ドメインの設定を保持する構造体。

失敗モードと境界条件

  • acct または md が NULL の場合、メンバアクセス時にクラッシュする可能性があります（ソースコード上に明示的な NULL チェックはありません）。
  • EAB の有無がドメインとアカウントで混在している場合、意図しないアカウントの再利用を防ぐため、厳格に不適合（0）を返します。

保守担当者向けメモ

  • 本関数は、既存のアカウントが新しいドメインの要件を満たしているかを確認する際に使用されます。
  • EAB の比較ロジックは、セキュリティ上の理由から厳密な一致を要求する仕様となっています。
