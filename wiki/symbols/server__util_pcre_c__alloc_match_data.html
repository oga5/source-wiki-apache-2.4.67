alloc_match_data

定義

  • 種別: function
  • 定義箇所: server/util_pcre.c:284

目的

正規表現マッチングに必要なワークスペース（PCRE2 の match_data または PCRE1 のオフセットベクトル）を確保する。

シグネチャまたは形状

static APR_INLINE match_data_pt alloc_match_data(apr_size_t size, match_vector_pt small_vector)

入力と出力

  • **入力**:
  • size: 確保する領域のサイズ（PCRE2 では要素数、PCRE1 ではオフセットベクトルのサイズ）。
  • small_vector: size が小さい場合に再利用されるスタック上のバッファ。
  • **出力**:
  • match_data_pt: 確保されたメモリ領域へのポインタ。

前提条件と不変条件

  • HAVE_PCRE2 が定義されている場合、PCRE2 の pcre2_match_data_create を使用する。
  • HAVE_PCRE2 が定義されていない場合、size が POSIX_MALLOC_THRESHOLD を超える場合にのみ malloc を実行し、それ以外は small_vector を返す。

アルゴリズムまたは振る舞い

1. **PCRE2 環境**: pcre2_match_data_create(size, NULL) を呼び出し、PCRE2 用のマッチングデータ構造を生成して返す。 2. **PCRE1 環境**:

  • size が POSIX_MALLOC_THRESHOLD を超える場合、malloc を使用して size * sizeof(int) * 3 バイトのメモリを確保して返す。
  • それ以外の場合、引数として渡された small_vector をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_pcre.c:345
  • server/util_pcre.c:390
  • **呼び出し先**:
  • pcre2_match_data_create (PCRE2 使用時)
  • malloc (PCRE1 使用時、閾値超過時)

関連データ構造

  • match_data_pt: マッチング結果を保持するデータ型。
  • match_vector_pt: オフセットベクトルを保持するデータ型。

失敗モードと境界条件

  • malloc が失敗した場合、本関数は NULL を返す可能性がある（PCRE1 環境下）。
  • pcre2_match_data_create が失敗した場合、NULL を返す可能性がある（PCRE2 環境下）。

保守担当者向けメモ

  • 本関数は APR_INLINE で定義されており、パフォーマンスを重視した設計となっている。
  • PCRE1 と PCRE2 でメモリ確保のロジックが大きく異なるため、修正時は #ifdef HAVE_PCRE2 の分岐に注意すること。
