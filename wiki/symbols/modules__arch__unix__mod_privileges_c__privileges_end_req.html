privileges_end_req

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_privileges.c:131

目的

リクエスト処理終了時に、プロセスに適用されていた特権およびユーザー/グループ権限を、サーバーのデフォルト状態へ復元するために使用されます。

シグネチャまたは形状

static apr_status_t privileges_end_req(void *data)

入力と出力

  • **入力**: data (リクエスト構造体 request_rec * へのポインタ)
  • **出力**: APR_SUCCESS を返します。

前提条件と不変条件

  • data は有効な request_rec 構造体である必要があります。
  • 本関数は apr_pool_cleanup_register を通じてリクエストのクリーンアップ処理として登録されることを前提としています。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバーおよびディレクトリ設定（priv_cfg, priv_dir_cfg）を取得します。 2. 動作モードが PRIV_SECURE（または PRIV_SELECTIVE かつディレクトリ設定が PRIV_SECURE）である場合、リクエスト処理のために生成された子プロセスであると判断し、exit(0) を実行してプロセスを終了します。 3. 上記以外の場合、設定された UID または GID がデフォルトと異なる場合は、setppriv で特権を一時的に有効化し、setuid/setgid を用いてデフォルトのユーザー/グループ権限へ戻します。 4. 最後に、setppriv を呼び出し、プロセス全体の特権セットを priv_default（システム起動時のデフォルト）へ復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/unix/mod_privileges.c:254 において、apr_pool_cleanup_register の引数として登録されています。
  • **呼び出し先**: ap_get_module_config, exit, setppriv, setuid, setgid, ap_log_rerror。

関連データ構造

  • request_rec: リクエスト情報を保持する Apache 内部構造体。
  • priv_cfg: vhost 単位の特権設定構造体。
  • priv_dir_cfg: ディレクトリ単位の特権設定構造体。
  • unixd_config_rec: サーバー全体のデフォルト UID/GID を保持する構造体（ap_unixd_config）。

失敗モードと境界条件

  • setppriv, setuid, setgid が失敗した場合、ap_log_rerror を通じてエラーログが出力されますが、関数自体は APR_SUCCESS を返します。
  • PRIV_SECURE モードで実行されている場合、本関数はプロセスを終了させるため、呼び出し元へは戻りません。

保守担当者向けメモ

  • 本関数は apr_pool_cleanup_register を介して呼び出されるため、リクエスト終了時のメモリ解放や権限復元を確実に行う必要があります。
  • ap_unixd_config を外部参照しているため、mod_unixd の設定変更が本関数の動作に影響を与える可能性があります。
