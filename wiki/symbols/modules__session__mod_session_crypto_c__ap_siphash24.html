ap_siphash24

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:109

目的

入力データに対して SipHash-2-4 アルゴリズムを用いた 64 ビットのハッシュ値を計算し、データの完全性検証（MAC）のための認証タグを生成します。

シグネチャまたは形状

apr_uint64_t ap_siphash24(const void *src, apr_size_t len, const unsigned char key[AP_SIPHASH_KSIZE])

入力と出力

  • **入力**:
  • src: ハッシュ対象のデータバッファ。
  • len: src のバイト長。
  • key: 16 バイトの秘密鍵（AP_SIPHASH_KSIZE）。
  • **出力**:
  • 戻り値: 計算された 64 ビットのハッシュ値（apr_uint64_t）。

前提条件と不変条件

  • key は 16 バイト以上のメモリ領域を指している必要があります。
  • SIPROUND() マクロが定義されている環境下で実行されます。
  • 入力データはリトルエンディアンとして処理されます。

アルゴリズムまたは振る舞い

1. **初期化**: 16 バイトの key を 64 ビットの k0, k1 に分割し、内部状態 v0〜v3 を定数と鍵で初期化します。 2. **メインループ**: 8 バイト単位でデータを読み込み、v3 との XOR、SIPROUND の実行、v0 との XOR を繰り返します。 3. **パディング処理**: 残りのデータ（1〜7 バイト）とデータ長を m に格納し、v3 との XOR、SIPROUND の実行、v0 との XOR を行います。 4. **最終化**: v2 を反転させ、さらに 4 回の SIPROUND を実行します。 5. **結果生成**: v0 ^ v1 ^ v2 ^ v3 を計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_crypto.c:162（compute_auth 関数内での呼び出しが確認されています）。
  • **呼び出し先**: SIPROUND() マクロ（ソースコード内で定義されたインライン処理）。

関連データ構造

  • AP_SIPHASH_KSIZE: 鍵サイズを定義する定数。

失敗モードと境界条件

  • len が 8 バイト未満の場合、switch 文によるパディング処理が正しく実行されます。
  • len が 0 の場合、パディング処理のみが実行され、空データに対するハッシュ値が生成されます。

保守担当者向けメモ

  • 本関数は APR 1.6 未満の環境におけるフォールバック実装として提供されています。最新の APR 環境では apr_siphash.h の利用が推奨されます。
  • SIPROUND マクロの定義が本ファイルまたはインクルードヘッダー内に存在することを確認してください。
