md_acme_authz_cha_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme_authz.c:53

目的

ACME プロトコルにおけるドメイン認証（Authorization）プロセスで使用される、個別のチャレンジ（Challenge）情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    size_t index;
    const char *type;
    const char *uri;
    const char *token;
    const char *key_authz;
} md_acme_authz_cha_t;

入力と出力

本構造体は、ACME サーバーから取得した JSON レスポンスを解析して生成されます。

  • index: チャレンジの識別インデックス。
  • type: チャレンジタイプ（例: "http-01", "dns-01"）。
  • uri: チャレンジの応答を送信するためのサーバー側 URI。
  • token: チャレンジ検証に使用されるトークン文字列。
  • key_authz: 鍵認証トークン（setup_key_authz で生成）。

前提条件と不変条件

  • md_acme_authz_t オブジェクトのライフサイクルと密接に関連しており、メモリ管理は apr_pool_t に依存します。
  • key_authz フィールドは、チャレンジのセットアップ処理（setup_key_authz）が実行されるまで初期化されない場合があります。

アルゴリズムまたは振る舞い

  • cha_from_json 関数により、サーバーから取得した JSON データから本構造体がインスタンス化されます。
  • 各チャレンジタイプ（HTTP-01, TLS-ALPN-01, DNS-01）のセットアップ関数（cha_..._setup）の引数として渡され、認証環境の構築に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: cha_from_json が本構造体を生成します。また、cha_http_01_setup, cha_tls_alpn_01_setup, cha_dns_01_setup などのセットアップ関数が本構造体を参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • md_acme_authz_t: チャレンジを包含する上位の認証リソース構造体。
  • cha_type: チャレンジタイプごとの処理を定義するディスパッチテーブル。

失敗モードと境界条件

  • JSON 解析時に必須フィールドが欠落している場合、cha_from_json は不完全な構造体を返す可能性があります。
  • key_authz の生成に失敗した場合、認証プロセス全体が失敗モードへ移行します。

保守担当者向けメモ

  • 新しいチャレンジタイプを追加する場合は、cha_type ディスパッチテーブルに新しいセットアップ関数を登録し、本構造体を利用してデータを引き渡す設計になっています。
  • メモリ管理は apr_pool_t に依存しているため、明示的な解放は不要です。
