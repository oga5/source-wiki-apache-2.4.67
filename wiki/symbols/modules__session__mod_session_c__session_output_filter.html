session_output_filter

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:447

目的

レスポンス送信直前にセッションデータを永続化し、フィルタチェーンから自身を削除する出力フィルタとして機能します。

シグネチャまたは形状

static apr_status_t session_output_filter(ap_filter_t *f, apr_bucket_brigade *in);

入力と出力

  • **入力**:
  • f: 現在のフィルタコンテキスト。
  • in: 送信されるバケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（成功時は ap_pass_brigade の戻り値を返却）。

前提条件と不変条件

  • session_rec はリクエストプール（r->pool）に関連付けられており、明示的な解放は不要です。
  • 内部リダイレクトが発生した場合、r->main を起点に r->next を辿り、連鎖するすべてのリクエストに対してセッション保存を試みます。
  • dirty フラグが立っていないセッションは保存処理をスキップします。

アルゴリズムまたは振る舞い

1. リクエストチェーン（r から r->next）を順次走査します。 2. 各リクエストに対して ap_session_load を呼び出し、セッションを取得します。 3. 設定（session_dir_conf）で指定されたヘッダーが存在する場合、その値をセッションに反映し、dirty フラグを立てて session_identity_decode を実行します。 4. ap_session_save を呼び出し、セッションをバックエンドへ書き出します。 5. 処理完了後、ap_remove_output_filter(f) によりフィルタチェーンから自身を削除します。 6. 最後に ap_pass_brigade を呼び出し、データを後続のフィルタへ渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_register_output_filter を通じて登録され、Apache の出力フィルタチェーンから呼び出されます。
  • **呼び出し先**:
  • ap_session_load
  • ap_session_save
  • session_identity_decode
  • ap_remove_output_filter
  • ap_pass_brigade

関連データ構造

  • session_rec: セッション状態を保持する構造体。
  • session_dir_conf: ディレクトリ単位のセッション設定。
  • ap_filter_t: フィルタチェーンの管理構造体。

失敗モードと境界条件

  • ap_session_load が失敗または未設定の場合、そのリクエストの保存処理はスキップされます。
  • z->written が真の場合、既に書き込み済みと判断しスキップします。
  • ヘッダーによる上書き処理において、指定されたヘッダーが見つからない場合はスキップされます。

保守担当者向けメモ

  • 本関数はリクエストのライフサイクル終了時に実行されるため、セッションの最終的な永続化ポイントとなります。
  • 内部リダイレクト時の挙動（r->next の走査）が実装に含まれているため、リダイレクトを伴う複雑なリクエスト処理のデバッグ時にはこのループ構造を考慮してください。
