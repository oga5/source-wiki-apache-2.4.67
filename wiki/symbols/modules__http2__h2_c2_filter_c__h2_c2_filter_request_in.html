h2_c2_filter_request_in

定義

  • **種別**: 関数
  • **定義箇所**: modules/http2/h2_c2_filter.c:95

目的

HTTP/2 セカンダリコネクション（C2）において、ストリームから受け取ったリクエスト情報を Apache のフィルタスタックへ注入し、自身をフィルタチェーンから削除する「一度きり」の入力フィルタです。

シグネチャまたは形状

apr_status_t h2_c2_filter_request_in(ap_filter_t *f,
                                     apr_bucket_brigade *bb,
                                     ap_input_mode_t mode,
                                     apr_read_type_e block,
                                     apr_off_t readbytes)

入力と出力

  • **入力**:
  • f: フィルタコンテキスト。
  • bb: データが挿入されるバケットブリゲード。
  • mode, block, readbytes: フィルタ操作のモード指定。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。
  • bb: リクエストバケット（およびエラー発生時はエラーバケット）が末尾に挿入されます。

前提条件と不変条件

  • mode が AP_MODE_READBYTES または AP_MODE_GETLINE 以外の場合、本関数は即座に後続のフィルタへ処理を委譲します。
  • 実行後、ap_remove_input_filter(f) によりフィルタスタックから自身を確実に削除します。
  • conn_ctx->stream_id が存在する場合にのみ、リクエスト注入処理が実行されます。

アルゴリズムまたは振る舞い

1. **フィルタの削除**: 実行時に必ず自身をフィルタスタックから除去します。 2. **WebSocket の処理**: プロトコルが websocket の場合、h2_ws_rewrite_request を呼び出しリクエストを書き換えます。 3. **リクエスト注入**: h2_request_create_bucket を使用してリクエストバケットを作成し、ブリゲードの末尾に挿入します。 4. **エラーハンドリング**: リクエスト準備段階でエラー（http_status が設定済み）が発生している場合、ap_bucket_error_create を使用してエラーバケットを挿入し、処理を終了します。 5. **EOS 挿入**: conn_ctx->beam_in が存在しない場合、ストリームの終了を示す EOS バケットを挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c2.c 内で ap_register_input_filter を介して登録され、フィルタチェーンから呼び出されます。
  • **呼び出し先**:
  • ap_get_brigade (後続フィルタ)
  • ap_remove_input_filter
  • h2_ws_rewrite_request
  • h2_request_create_bucket
  • ap_bucket_error_create
  • apr_bucket_eos_create

関連データ構造

  • h2_conn_ctx_t: 接続ごとのコンテキスト。request や beam_in を保持。
  • h2_request: HTTP/2 ストリームのリクエスト情報。
  • ap_filter_t: Apache フィルタスタックの構造体。

失敗モードと境界条件

  • h2_ws_rewrite_request が失敗した場合、APR_EGENERAL を返します。
  • mode が不適切な場合、後続フィルタの戻り値をそのまま返します。
  • req->http_status が設定されている場合、エラーバケットを挿入して APR_SUCCESS を返します。

保守担当者向けメモ

  • 本フィルタは「一度きり」の動作を前提としており、処理完了後に必ず自身を削除する設計になっています。
  • AP_MODE_EATCRLF はサポートされていません。
  • conn_ctx->beam_in の有無によって EOS バケットの挿入タイミングが制御されている点に注意してください。
