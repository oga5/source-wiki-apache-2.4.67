mpm_nt_eventlog_stderr_open

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/nt_eventlog.c:133

目的

Windowsサービスとして実行されるApacheにおいて、標準エラー出力（stderr）をキャプチャし、Windowsイベントログへリダイレクトするための初期化を行う。

シグネチャまたは形状

void mpm_nt_eventlog_stderr_open(const char *argv0, apr_pool_t *p)

入力と出力

  • **入力**:
  • argv0: サービス名（イベントログの表示名として使用）。
  • p: メモリ割り当てに使用するAPRプール。
  • **出力**: なし（副作用としてグローバルなパイプとスレッドを初期化）。

前提条件と不変条件

  • サービス起動の初期段階で呼び出される必要がある。
  • stderr をパイプへ差し替えるため、他のモジュールが stderr を直接操作する場合に競合が発生する可能性がある。
  • stderr_ready イベントがシグナルされるまで、呼び出し元はスレッドの起動とイベントソースの登録完了を待機する。

アルゴリズムまたは振る舞い

1. **パイプ作成**: CreatePipe を使用して、stderr を監視するための匿名パイプ（読み取り端 hPipeRead、書き込み端 hPipeWrite）を作成する。 2. **監視スレッド起動**: service_stderr_thread を生成し、hPipeRead を渡してバックグラウンドでイベントログへの転送を開始する。 3. **同期**: WaitForSingleObject を使用し、監視スレッドがイベントソースの登録を完了するまで待機する。 4. **リダイレクト**: apr_file_open_stderr で取得した stderr を、apr_file_dup2 を用いてパイプの書き込み端（hPipeWrite）へリダイレクトする。 5. **ログ初期化**: ap_open_stderr_log(p) を呼び出し、標準エラーログの基盤を確立する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/mpm_winnt.c:1243
  • **呼び出し先**:
  • CreatePipe, CreateThread, WaitForSingleObject (Windows API)
  • service_stderr_thread (監視スレッド)
  • apr_file_open_stderr, apr_os_file_put, apr_file_dup2 (APR)
  • ap_open_stderr_log (Apache Core)

関連データ構造

  • stderr_thread: 監視スレッドのハンドルを保持するグローバル変数。
  • stderr_ready: スレッドの初期化完了を通知するためのイベントハンドル。
  • display_name: サービス名を保持するグローバル変数。

失敗モードと境界条件

  • CreatePipe や CreateThread が失敗した場合、ap_assert によりプロセスが異常終了する。
  • apr_file_open_stderr 等のAPR関数が失敗した場合、リダイレクト処理はスキップされる。

保守担当者向けメモ

  • 本関数は stderr をパイプへ差し替えるため、必ずサービス起動の初期段階で呼び出すこと。
  • イベントログのメッセージ定義には netmsg.dll が流用されている。
  • 監視スレッドの終了処理については mpm_nt_eventlog_stderr_flush を参照のこと。
