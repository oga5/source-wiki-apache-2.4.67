connect_conf

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_connect.c:49

目的

mod_proxy_connect モジュールにおいて、サーバーごとの設定情報を保持するための構造体です。主に AllowCONNECT ディレクティブで指定された許可ポート範囲を管理します。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *allowed_connect_ports;
} connect_conf;

入力と出力

  • **入力**: allowed_connect_ports は apr_array_header_t 型の配列であり、set_allowed_ports コマンドを通じて port_range 構造体のリストが格納されます。
  • **出力**: allowed_port 関数等の引数として渡され、特定のポート番号が許可されているかどうかの判定に使用されます。

前提条件と不変条件

  • allowed_connect_ports は apr_pcalloc 等でメモリ確保された後に使用されます。
  • merge_config を介して親サーバーの設定を継承可能です。

アルゴリズムまたは振る舞い

  • 本構造体は、mod_proxy_connect の設定コンテキストとして機能します。
  • allowed_port 関数は、この構造体内の allowed_connect_ports を走査し、リクエストされたポート番号が許可リストに含まれているかを検証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_connect.c 内の各種設定関数（create_config, merge_config, set_allowed_ports）およびハンドラ関数（proxy_connect_handler）でインスタンス化および参照が行われています。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。allowed_port 関数が本構造体を引数として受け取り、内部の allowed_connect_ports を参照します。

関連データ構造

  • port_range: allowed_connect_ports 配列内に格納される、ポートの範囲（first, last）を定義する構造体。
  • apr_array_header_t: APR が提供する動的配列構造体。

失敗モードと境界条件

  • 設定が初期化されていない場合、allowed_connect_ports が NULL になる可能性があります。
  • allowed_port 関数において、ポート範囲が正しく設定されていない場合、意図しないアクセス拒否が発生する可能性があります。

保守担当者向けメモ

  • connect_conf はサーバー設定（server_rec）に紐付いています。
  • メモリ管理は r->pool に依存しており、リクエスト終了時に自動的に解放されます。
