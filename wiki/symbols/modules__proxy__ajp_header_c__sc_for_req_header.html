sc_for_req_header

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_header.c:51

目的

HTTP ヘッダー名を AJP プロトコルで定義された内部 ID（整数値）に変換します。文字列比較を最適化し、高速なマッピングを実現することを目的としています。

シグネチャまたは形状

int sc_for_req_header(const char *header_name)

入力と出力

  • **入力**: header_name (変換対象の HTTP ヘッダー名文字列)
  • **出力**: AJP 内部ヘッダー ID（SC_ACCEPT 等の定数）、または該当しない場合は UNKNOWN_METHOD を返します。

前提条件と不変条件

  • 入力される header_name はヌル終端文字列である必要があります。
  • 内部処理において、ヘッダー名は一時的に大文字に変換されます。
  • ヘッダー名の長さが 4 文字未満、または 15 文字を超える場合は、即座に UNKNOWN_METHOD を返します。

アルゴリズムまたは振る舞い

1. 入力文字列の長さを検証します。 2. 文字列を大文字に変換し、固定長バッファ（16バイト）に格納します。 3. 先頭文字に基づいて switch 文で分岐し、残りの文字列を memcmp または strcmp で比較します。 4. 一致するヘッダー名が見つかった場合、対応する AJP 内部 ID を返します。 5. どのヘッダーにも一致しない場合は UNKNOWN_METHOD を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/ajp_header.c:272 にて、リクエストヘッダーの解析処理中に呼び出されています。
  • **呼び出し先**: apr_toupper (APR ライブラリ関数) を使用して文字変換を行います。

関連データ構造

  • ajp_header.h: AJP プロトコルで使用されるヘッダー ID 定数（SC_ACCEPT, SC_HOST 等）が定義されています。

失敗モードと境界条件

  • **長さ制限**: 4 文字未満または 15 文字を超えるヘッダー名は、プロトコル上の関心対象外として UNKNOWN_METHOD を返します。
  • **不一致**: 定義済みの AJP ヘッダーリストに含まれないヘッダー名は、すべて UNKNOWN_METHOD となります。

保守担当者向けメモ

  • 本関数は固定長バッファと memcmp を用いた最適化を行っています。新しいヘッダーをサポート対象に追加する際は、バッファサイズ（16バイト）と switch 分岐のロジックが整合しているか確認してください。
  • ACCEPT-LANGUAGE が現在サポートされている最長のヘッダーです。これより長いヘッダーを追加する場合はバッファサイズの拡張が必要です。
