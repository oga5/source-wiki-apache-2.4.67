dbd_param

定義

  • 種別: function
  • 定義箇所: modules/database/mod_dbd.c:161

目的

Apache 設定ファイル内の DBDriver および DBDParams ディレクティブを解析し、サーバー設定構造体 (svr_cfg) にデータベース接続用のドライバ名や接続パラメータを格納する。

シグネチャまたは形状

static const char *dbd_param(cmd_parms *cmd, void *dconf, const char *val)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのコンテキスト情報。cmd->info を通じて呼び出し元（cmd_name または cmd_params）を識別する。
  • dconf: ディレクトリ設定（本関数では未使用）。
  • val: 設定ファイルから読み込まれた値（ドライバ名または接続文字列）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • cmd->server->module_config から dbd_module に関連付けられた svr_cfg が取得可能であること。
  • cmd_name が指定された場合、apr_dbd_get_driver を用いて指定されたドライバのロードを試みる。

アルゴリズムまたは振る舞い

1. cmd->info の値に基づき処理を分岐する。 2. cmd_name の場合:

  • cfg->name に val を設定する。
  • apr_dbd_get_driver を呼び出し、ドライバの存在とロード可能性を検証する。
  • ドライバが見つからない、またはロードに失敗した場合は、具体的なエラーメッセージを生成して返す。

    • 3. cmd_params の場合:

  • cfg->params に val を設定する。

    • 4. 正常終了時は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/database/mod_dbd.c 内の AP_INIT_TAKE1 マクロ定義（DBDriver および DBDParams ディレクティブのハンドラとして登録）。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core)
  • apr_dbd_get_driver (APR-util)
  • apr_psprintf (APR)

関連データ構造

  • svr_cfg: サーバーごとの設定コンテナ。
  • dbd_cfg_t: svr_cfg 内で保持されるデータベース設定値。
  • cmd_parms: Apache 設定コマンドのパラメータ構造体。

失敗モードと境界条件

  • 指定されたドライバ名が存在しない場合 (APR_STATUS_IS_ENOTIMPL)。
  • ドライバの共有ライブラリ（DSO）がロードできない場合 (APR_STATUS_IS_EDSOOPEN)。
  • ドライバ内のシンボルが見つからない場合 (APR_STATUS_IS_ESYMNOTFOUND)。
  • 上記いずれの場合も、エラーメッセージを生成して設定処理を中断する。

保守担当者向けメモ

  • cmd->info による分岐は、同一の関数を複数のディレクティブで再利用するための設計である。
  • ドライバのロードはサーバー起動時に行われるため、設定変更にはサーバーの再起動が必要となる。
