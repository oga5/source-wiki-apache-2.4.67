authz_alias_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_core.c:191

目的

AuthzProviderAlias によって定義された認可プロバイダーのエイリアスを解決し、本来のプロバイダーの認可チェック関数を呼び出すためのラッパーとして機能します。

シグネチャまたは形状

static authz_status authz_alias_check_authorization(request_rec *r,
                                                    const char *require_args,
                                                    const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。r->notes から AUTHZ_PROVIDER_NAME_NOTE を取得してエイリアス名を特定します。
  • require_args, parsed_require_args: プロバイダーに渡される引数。
  • **出力**:
  • authz_status: 認可結果（AUTHZ_GRANTED, AUTHZ_DENIED 等）。エイリアスが見つからない場合は AUTHZ_DENIED を返します。

前提条件と不変条件

  • r->notes に AUTHZ_PROVIDER_NAME_NOTE キーでエイリアス名が格納されている必要があります。
  • サーバー設定 (authz_core_srv_conf) に該当する provider_alias_rec が存在する必要があります。

アルゴリズムまたは振る舞い

1. r->notes からプロバイダーのエイリアス名を取得します。 2. サーバー設定から該当する provider_alias_rec をハッシュ検索します。 3. エイリアスが見つかった場合:

  • 現在のディレクトリ設定 (r->per_dir_config) を一時保存します。
  • エイリアスに紐付く設定 (prvdraliasrec->sec_auth) を現在のリクエスト設定にマージします。
  • 本来のプロバイダーの check_authorization 関数を、エイリアスが保持する引数を用いて呼び出します。
  • 保存していた元のディレクトリ設定を復元し、プロバイダーの戻り値を返します。

    • 4. エイリアスが見つからない場合、エラーログを出力し AUTHZ_DENIED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_core.c:244 にて、認可プロバイダーの定義の一部として参照されています。
  • **呼び出し先**:
  • ap_get_module_config
  • apr_hash_get
  • ap_merge_per_dir_configs
  • prvdraliasrec->provider->check_authorization (動的呼び出し)
  • ap_log_rerror

関連データ構造

  • authz_core_srv_conf: サーバー全体の設定。エイリアスのハッシュテーブルを保持。
  • provider_alias_rec: エイリアス設定（プロバイダー本体、引数、ディレクトリ設定）を保持。
  • request_rec: リクエストごとの状態と設定を管理。

失敗モードと境界条件

  • **エイリアス未検出**: r->notes に該当するエイリアス名が存在しない、またはハッシュテーブルに登録されていない場合、エラーログを出力して AUTHZ_DENIED を返します。
  • **設定マージ**: ap_merge_per_dir_configs の失敗については本関数内では明示的にハンドリングされていません。

保守担当者向けメモ

  • 本関数は AuthzProviderAlias の実行時解決を行うためのプロキシです。
  • r->per_dir_config を一時的に書き換えるため、呼び出し後の復元が必須です。
  • エイリアスが見つからないケースは設定不備（BUG）の可能性があるため、ログ出力が重要です。
