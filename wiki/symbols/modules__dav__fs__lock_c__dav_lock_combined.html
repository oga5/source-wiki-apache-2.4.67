dav_lock_combined

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/lock.c:202

目的

WebDAV ロック情報をメモリ上で保持するための統合構造体です。公開用のロック情報、内部管理用のプライベート情報、およびロック識別子を一つのメモリブロックにまとめ、効率的なデータ操作を可能にします。

シグネチャまたは形状

typedef struct
{
    dav_lock pub;
    dav_lock_private priv;
    dav_locktoken token;
} dav_lock_combined;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。

  • **入力**: 構築時に dav_lock（公開情報）、dav_lock_private（内部情報）、dav_locktoken（UUID）の各フィールドが設定されます。
  • **出力**: 構造体へのポインタを通じて、ロックの属性や識別子にアクセス可能です。

前提条件と不変条件

  • pub フィールドは mod_dav が要求する公開インターフェースと互換性を持つ必要があります。
  • token は dav_locktoken 型であり、UUID に基づく一意な識別子であることが期待されます。

アルゴリズムまたは振る舞い

本構造体は、ロックの永続化や解決の過程で、ロック情報をメモリ上でリンクドリストとして構築する際のノードとして機能します。dav_fs_load_lock_record 等の関数により、DBM から読み込まれたデータが本構造体にマッピングされます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/lock.c および modules/dav/lock/locks.c 内の関数で、ロック情報の生成や参照時に dav_lock_combined *comb として利用されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • dav_lock: 公開用ロック情報。
  • dav_lock_private: ファイルシステム固有のロック管理情報。
  • dav_locktoken: ロックを識別するための UUID ラッパー。

失敗モードと境界条件

  • メモリ割り当て失敗時（apr_palloc 等の失敗）に本構造体の生成が失敗する可能性があります。
  • 構造体内の各フィールドが正しく初期化されていない場合、ロックの解決や検証時に不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dav のロックプロバイダインターフェースと密接に関連しています。構造体のメンバを変更する場合は、mod_dav 側の期待するデータレイアウトとの整合性に注意してください。
  • modules/dav/lock/locks.c でも同名の構造体が定義されており、クロスモジュールでの利用状況を考慮する必要があります。
