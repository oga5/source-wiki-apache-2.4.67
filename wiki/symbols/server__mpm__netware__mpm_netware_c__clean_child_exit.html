clean_child_exit

定義

  • 種別: function
  • 定義箇所: server/mpm/netware/mpm_netware.c:178

目的

NetWare MPM におけるワーカースレッドの終了処理を行い、リソースの解放とスコアボードの更新を確実に行うために使用されます。

シグネチャまたは形状

static void clean_child_exit(int code, int worker_num, apr_pool_t *ptrans,
                             apr_bucket_alloc_t *bucket_alloc)

入力と出力

  • **入力**:
  • code: 終了コード。
  • worker_num: 終了するスレッドのインデックス。
  • ptrans: スレッドに関連付けられたトランザクションプール。
  • bucket_alloc: バケットアロケータ。
  • **出力**: なし（関数は NXThreadExit を呼び出し、呼び出し元へは戻りません）。

前提条件と不変条件

  • worker_num が 0 以上の場合、スコアボードの更新が試行されます。
  • shutdown_pending が真（サーバー停止中）の場合、ptrans プールの破棄はスキップされます。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc_destroy を呼び出し、バケットアロケータを破棄します。 2. サーバーが停止中でない場合、apr_pool_destroy を呼び出し、トランザクションプールを解放します。 3. atomic_dec を使用して、グローバルな worker_thread_count を減算します。 4. worker_num が 0 以上であれば、ap_update_child_status_from_indexes を呼び出し、スコアボード上の当該スレッド状態を WORKER_DEAD に更新します。 5. NXThreadExit を呼び出し、スレッドを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内には、本関数を呼び出している箇所は明示されていません。
  • **呼び出し先**:
  • apr_bucket_alloc_destroy
  • apr_pool_destroy
  • atomic_dec
  • ap_update_child_status_from_indexes
  • NXThreadExit
  • **注意**: ツールが提示した server/mpm/event/event.c 内の clean_child_exit への参照は、本ファイル（mpm_netware.c）とは異なる MPM 実装のものであり、本シンボルの呼び出し元ではありません。

関連データ構造

  • apr_pool_t: トランザクションメモリ管理用。
  • apr_bucket_alloc_t: バケットメモリ管理用。
  • worker_thread_count: ワーカースレッド数を追跡するアトミック変数。
  • scoreboard: ap_update_child_status_from_indexes を介して更新されるスレッド状態管理領域。

失敗モードと境界条件

  • worker_num が負の値の場合、スコアボードの更新はスキップされます。
  • shutdown_pending が真の場合、メモリリークを防ぐため、呼び出し元で適切にプールが管理されている必要があります（本関数内では ptrans は破棄されません）。

保守担当者向けメモ

  • 本関数は NXThreadExit を呼び出すため、呼び出し後のコードは実行されません。
  • server/mpm/event/event.c に同名の関数が存在しますが、NetWare MPM の実装とは独立しています。混同しないよう注意してください。
