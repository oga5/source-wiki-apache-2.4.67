h2_bucket_headers_get

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_headers.c:85

目的

Apache バケットチェーン内の H2HEADERS 型バケットから、内部で保持されている h2_headers 構造体へのポインタを取得します。

シグネチャまたは形状

h2_headers *h2_bucket_headers_get(apr_bucket *b)

入力と出力

  • **入力**: apr_bucket *b - 検査対象のバケット。
  • **出力**: バケットが H2HEADERS 型であれば、格納されている h2_headers 構造体へのポインタを返します。それ以外の場合は NULL を返します。

前提条件と不変条件

  • 入力された apr_bucket *b は有効なメモリ領域を指している必要があります。
  • H2_BUCKET_IS_HEADERS(b) マクロによって、バケットの型が H2HEADERS であることが確認されます。

アルゴリズムまたは振る舞い

1. H2_BUCKET_IS_HEADERS(b) を使用して、バケットが H2HEADERS 型であるか判定します。 2. 型が一致する場合、バケットの data メンバを h2_bucket_headers * 型にキャストし、その構造体が保持する headers メンバを返します。 3. 型が一致しない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2_filter.c
  • modules/http2/h2_headers.c
  • modules/http2/h2_stream.c
  • modules/http2/h2_ws.c
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行いません（マクロ判定のみ）。

関連データ構造

  • apr_bucket: Apache のバケット構造体。
  • h2_headers: HTTP/2 レスポンスヘッダー情報を保持する構造体。
  • h2_bucket_headers: apr_bucket のデータ領域として使用される、h2_headers をラップする構造体。

失敗モードと境界条件

  • b が NULL の場合、H2_BUCKET_IS_HEADERS の実装に依存しますが、一般的にセグメンテーション違反を引き起こす可能性があります。
  • H2HEADERS 型以外のバケットが渡された場合、安全に NULL を返します。

保守担当者向けメモ

  • 本関数はアクセサとして機能します。バケットの型判定には H2_BUCKET_IS_HEADERS マクロが必須であるため、このマクロの定義が変更された場合は本関数も影響を受けます。
