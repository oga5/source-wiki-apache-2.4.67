apr_off_t

定義

  • 種別: typedef
  • 定義箇所: modules/http2/h2.h:192

目的

ファイルサイズ、オフセット、バイト数などの「ファイルやストリーム内の位置・長さ」を表現するための、プラットフォーム非依存な整数型です。

シグネチャまたは形状

Apache Portable Runtime (APR) によって定義される符号付き整数型です。

入力と出力

  • **入力**: 多くの API でバイト数やオフセットを指定する引数として使用されます。
  • **出力**: ファイルの読み込み位置や、転送されたバイト数の戻り値として使用されます。

前提条件と不変条件

  • 負の値は、エラー状態や特定のオフセット計算において特別な意味を持つ場合があります。
  • 64ビット環境では通常 64ビット整数として扱われ、大規模なファイルやデータストリームをサポートします。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、実行時のアルゴリズムは持ちません。主に apr_file_t 関連の I/O 操作や、HTTP ヘッダーの Content-Length などのバイト数計算において、データ長を保持・伝達するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2.h 内の h2_io_data_cb コールバック関数の引数として使用されています。
  • **呼び出し先**: Apache HTTP Server のコア機能（include/httpd.h, include/http_protocol.h, include/http_core.h）において、ファイル操作やリクエストボディの制限管理のために広く利用されています。

関連データ構造

  • apr_file_t: ファイル I/O 操作において、オフセット指定の引数として併用されます。
  • h2_io_data_cb: 本シンボルを引数に含むコールバック型定義。
  • request_rec: HTTP リクエストのメタデータ構造体において、clength（コンテンツ長）や bytes_sent（送信バイト数）などのフィールド型として使用されます。

失敗モードと境界条件

  • **オーバーフロー**: 非常に大きなファイルやストリームを扱う際、プラットフォームの型サイズ制限を超える可能性があります。
  • **負の値**: 負の値を許容しない API に負の値を渡した場合、予期しない動作やエラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは APR の標準型であるため、mod_http2 固有の定義ではありません。変更が必要な場合は、APR の仕様および Apache HTTP Server 全体の整合性に注意してください。
  • h2_io_data_cb で使用される際は、データチャンクの長さを表すため、負の値が渡されないことを前提とした設計になっているか確認してください。
