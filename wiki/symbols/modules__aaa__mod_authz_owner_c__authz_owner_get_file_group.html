authz_owner_get_file_group

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_owner.c:110

目的

リクエストされたファイルシステムのグループ名を取得し、他の認可モジュールがグループベースのアクセス制御を行えるようにするためのオプション関数を提供します。

シグネチャまたは形状

char *authz_owner_get_file_group(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエストコンテキスト)
  • **出力**: 成功時はファイルグループ名を表す char *、失敗時または APR_HAS_USER が無効な場合は NULL を返します。

前提条件と不変条件

  • APR_HAS_USER が定義されている環境（主に Unix 系）でのみ動作します。
  • r->filename が有効なファイルパスを指している必要があります。

アルゴリズムまたは振る舞い

1. APR_HAS_USER が無効な場合、直ちに NULL を返します。 2. r->filename が存在しない場合、エラーログを出力し NULL を返します。 3. apr_stat を使用してファイルのグループ情報を取得します。取得に失敗した場合はエラーログを出力し NULL を返します。 4. ファイル情報にグループ情報が含まれていない場合、エラーログを出力し NULL を返します。 5. apr_gid_name_get を使用してグループ ID をグループ名に解決します。解決に失敗した場合はエラーログを出力し NULL を返します。 6. 解決されたグループ名を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authz_dbm.c
  • modules/aaa/mod_authz_groupfile.c
  • **呼び出し先**:
  • apr_stat (APR)
  • apr_gid_name_get (APR)
  • ap_log_rerror (Apache HTTP Server)

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • apr_finfo_t: ファイル情報を保持する APR 構造体。

失敗モードと境界条件

  • ファイルパスが未設定の場合、または stat が失敗した場合は NULL を返します。
  • ファイルシステムからグループ情報が取得できない場合、またはグループ名への解決に失敗した場合は NULL を返します。
  • 各失敗時には APLOGNO を付与したエラーログが記録されます。

保守担当者向けメモ

  • 本関数は APR_DECLARE_OPTIONAL_FN を通じて公開されています。変更時は modules/aaa/mod_authz_owner.h のシグネチャと整合性を保ってください。
  • この関数はグループ名の解決のみを行い、実際の認可判定は呼び出し側のモジュールに委譲する設計です。
