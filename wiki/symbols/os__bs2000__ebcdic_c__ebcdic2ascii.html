ebcdic2ascii

定義

  • 種別: function
  • 定義箇所: os/bs2000/ebcdic.c:189

目的

EBCDIC でエンコードされたメモリブロックを ASCII に変換します。主に BS2000 環境下で、ネットワーク通信やプロトコル処理のために文字コードを変換する際に使用されます。

シグネチャまたは形状

void ebcdic2ascii(unsigned char *dest, const unsigned char *srce, size_t count);

入力と出力

  • dest: 変換後のデータを格納する出力バッファ（書き込み先）。
  • srce: 変換元の EBCDIC データが格納された入力バッファ。
  • count: 変換するバイト数。
  • 戻り値: なし（void）。

前提条件と不変条件

  • dest と srce が指すメモリ領域は、互いにオーバーラップしていない必要があります。
  • os_toascii テーブルがメモリ上に存在し、参照可能である必要があります。

アルゴリズムまたは振る舞い

1. count が 0 になるまでループを繰り返します。 2. 各反復において、srce が指す現在のバイトをインデックスとして os_toascii テーブルを参照します。 3. 変換された値を dest が指す位置に書き込みます。 4. dest と srce のポインタをそれぞれインクリメントし、次のバイトへ進めます。

呼び出し元と呼び出し先

  • 呼び出し元: os/bs2000/ebcdic.h で宣言されており、Apache HTTP Server の各モジュールから利用される可能性があります。
  • 呼び出し先: os_toascii（変換テーブル）を参照します。
  • 補足: modules/http/mod_mime.c にて、マクロ定義内で本関数名が言及されていますが、直接的な呼び出し関係の詳細は提供されていません。

関連データ構造

  • os_toascii: EBCDIC から ASCII への変換テーブル。改行コード（0x0A）を維持するプロトコル向けの変換ルールを保持しています。

失敗モードと境界条件

  • count が 0 の場合、ループは実行されず何も変換されません。
  • メモリ領域がオーバーラップしている場合、変換結果は未定義となります。

保守担当者向けメモ

  • os_toascii_strictly との混同に注意してください。本関数はプロトコル文字列向けであり、改行コードの扱いが異なります。
  • 変換はバイト単位で行われるため、マルチバイト文字のエンコーディングには対応していません。
