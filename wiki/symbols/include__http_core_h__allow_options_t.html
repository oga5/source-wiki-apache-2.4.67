allow_options_t

定義

  • 種別: typedef
  • 定義箇所: include/http_core.h:482

目的

Apache HTTP Server における Options ディレクティブの設定値を保持するための型定義です。

シグネチャまたは形状

typedef unsigned char allow_options_t;

入力と出力

  • **入力**: 設定ファイルから読み込まれた Options ディレクティブの各フラグ（ビットマスク）。
  • **出力**: 設定構造体（core_dir_config 等）の一部として保持され、アクセス制御やリクエスト処理時の権限判定に使用されます。

前提条件と不変条件

  • unsigned char 型であるため、最大 8 つのオプションフラグをビットマスクとして保持可能です。
  • core_dir_config 内で使用される場合、opts_add（追加）と opts_remove（削除）のビットマスクは排他的である必要があります。

アルゴリズムまたは振る舞い

  • 設定の継承時、親ディレクトリの設定から子ディレクトリへビットマスクがマージされます。
  • リクエスト処理時には、この型で保持されたビットマスクを評価することで、特定のディレクトリで許可されている操作（Indexes, FollowSymLinks 等）を判定します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • include/http_core.h: core_dir_config 構造体内の opts, opts_add, opts_remove, override_opts フィールド。
  • server/core.c: set_allow_opts 関数およびディレクティブ処理ロジック。
  • server/request.c: リクエスト処理時のオプション判定ロジック。
  • **呼び出し先**:
  • 本シンボル自体は型定義であり、関数ではないため呼び出し先はありません。

関連データ構造

  • core_dir_config: この型を保持する主要な設定構造体。
  • OPT_* 定数: この型に格納されるビットマスク値の定義群。

失敗モードと境界条件

  • 8 ビットを超えるオプションフラグを定義しようとした場合、型サイズによる制限が発生します。
  • 設定の継承において、opts_add と opts_remove の両方に同じビットが設定された場合の優先順位は、実装側のマージロジックに依存します。

保守担当者向けメモ

  • 本型は unsigned char で定義されているため、将来的にオプションの種類が 8 つを超える場合は、型定義の変更（unsigned int 等への拡張）が必要です。その際、関連するすべての構造体およびマージロジックの整合性を確認してください。
