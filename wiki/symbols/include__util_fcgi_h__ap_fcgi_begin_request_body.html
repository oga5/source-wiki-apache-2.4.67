ap_fcgi_begin_request_body

定義

  • 種別: struct
  • 定義箇所: include/util_fcgi.h:106

目的

FastCGI プロトコルにおける BEGIN_REQUEST レコードのボディ部分を表現するための構造体です。

シグネチャまたは形状

typedef struct {
    unsigned char roleB1;
    unsigned char roleB0;
    unsigned char flags;
    unsigned char reserved[5];
} ap_fcgi_begin_request_body;

入力と出力

本構造体はデータ保持用であり、関数ではありません。以下の関数を介してバイト配列との相互変換が行われます。

  • 入力（変換用）: ap_fcgi_begin_request_body_to_array
  • 出力（構築用）: ap_fcgi_fill_in_request_body

前提条件と不変条件

  • roleB1 および roleB0 は、FastCGI のロールを構成する2バイトの値を保持します。
  • reserved フィールドは 5 バイトの予約領域であり、プロトコル仕様に従い適切に扱う必要があります。
  • メモリ管理は呼び出し側に委ねられており、本構造体自体はメモリ割り当てを行いません。

アルゴリズムまたは振る舞い

FastCGI 通信において、リクエスト開始時に送信される BEGIN_REQUEST レコードのペイロードを定義します。ネットワークバイトオーダーを意識したパッキングを行うため、直接のメモリ操作ではなく、提供される変換関数（ap_fcgi_begin_request_body_to_array 等）を通じて操作することが推奨されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • modules/aaa/mod_authnz_fcgi.c:272
  • modules/proxy/mod_proxy_fcgi.c:327
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、直接の呼び出し先はありません。操作関数として ap_fcgi_begin_request_body_to_array および ap_fcgi_fill_in_request_body が定義されています。

関連データ構造

  • ap_fcgi_header: FastCGI レコードの固定長ヘッダー。

失敗モードと境界条件

  • コンパイラのパディングに依存する可能性があるため、構造体の直接的なバイナリコピーは避けるべきです。
  • reserved フィールドの不適切な操作は、将来的なプロトコル拡張との互換性を損なう可能性があります。

保守担当者向けメモ

  • 本構造体は RFC 3652 に準拠しています。
  • フィールドのオフセットや値の定義については、include/util_fcgi.h 内の関連マクロを参照してください。
