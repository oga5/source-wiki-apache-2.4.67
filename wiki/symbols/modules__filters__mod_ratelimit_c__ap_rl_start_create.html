ap_rl_start_create

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ratelimit.c:285

目的

mod_ratelimit フィルタにおいて、全速転送モードを開始するための制御用メタデータバケットを生成する。

シグネチャまたは形状

apr_bucket *ap_rl_start_create(apr_bucket_alloc_t *list)

入力と出力

  • **入力**: apr_bucket_alloc_t *list（バケット割り当て用プール）
  • **出力**: apr_bucket *（初期化された RL_START 型のバケット）

前提条件と不変条件

  • list は有効な apr_bucket_alloc_t であること。
  • 生成されたバケットの type は ap_rl_bucket_type_start に設定される。
  • バケットの長さ（length）および開始位置（start）は 0 に初期化される。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を使用してバケット構造体用のメモリを確保する。 2. APR_BUCKET_INIT マクロでバケットを初期化する。 3. バケットの解放関数を apr_bucket_free に設定する。 4. バケットの型を ap_rl_bucket_type_start に指定する。 5. 初期化済みのバケットポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は提供されていません。
  • **呼び出し先**: apr_bucket_alloc および apr_bucket_free（APRライブラリ関数）。

関連データ構造

  • apr_bucket: Apache フィルタフレームワークのデータ単位。
  • ap_rl_bucket_type_start: 本関数で割り当てられるバケットの型定義。

失敗モードと境界条件

  • メモリ不足時には apr_bucket_alloc が失敗し、プロセスが異常終了する可能性があります（APRのメモリ割り当てポリシーに依存）。

保守担当者向けメモ

  • 本関数は mod_ratelimit.h で宣言されており、外部モジュールから利用可能な公開インターフェースです。
  • バケットのデータ領域（data）は NULL に初期化されます。このバケットは主にストリーム内のマーカーとして機能します。
