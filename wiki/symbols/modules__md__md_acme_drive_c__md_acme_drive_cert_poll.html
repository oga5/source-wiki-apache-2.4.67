md_acme_drive_cert_poll

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:262

目的

ACME プロトコルを通じて発行された証明書を、ポーリングによって取得する。

シグネチャまたは形状

apr_status_t md_acme_drive_cert_poll(md_proto_driver_t *d, int only_once)

入力と出力

  • **入力**:
  • d: md_proto_driver_t 型のドライバインスタンス。
  • only_once: 1回のみ試行するかどうかを指定するフラグ。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • d->baton が md_acme_driver_t 型としてキャスト可能であること。
  • ad->md、ad->acme、ad->order、および ad->order->certificate が NULL でないことが assert により保証されている。

アルゴリズムまたは振る舞い

1. only_once が真の場合、get_cert を直接呼び出す。 2. only_once が偽の場合、md_util_try を使用して get_cert を繰り返し呼び出す。この際、ad->cert_poll_timeout で指定されたタイムアウト時間までポーリングを継続する。 3. 最後に、証明書の取得先 URL をデバッグログとして記録する。 4. 最終的な apr_status_t を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_drive.c:490
  • **呼び出し先**:
  • get_cert (内部関数)
  • md_util_try (ユーティリティ関数)
  • md_log_perror (ログ出力)

関連データ構造

  • md_proto_driver_t: ドライバのコンテキスト。
  • md_acme_driver_t: ACME 駆動用のデータ構造（d->baton に格納）。

失敗モードと境界条件

  • get_cert が失敗した場合、そのエラーコードがそのまま返される。
  • only_once が偽の場合、md_util_try の設定に従い、タイムアウトが発生するまで再試行が行われる。

保守担当者向けメモ

  • 本関数は md_acme_drive.h で公開されており、外部モジュールから呼び出される可能性がある。
  • ad->order->certificate が指す URL が有効であることを前提としているため、呼び出し前に注文状態が適切に更新されている必要がある。
