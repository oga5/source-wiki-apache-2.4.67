ReportStatusToSCMgr

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/service.c:223

目的

Windows サービスコントロールマネージャ (SCM) に対して、現在のサービス状態（実行中、停止中、遷移中など）を通知し、同期を行う。

シグネチャまたは形状

static int ReportStatusToSCMgr(int currentState, int waitHint, nt_service_ctx_t *ctx)

入力と出力

  • **入力**:
  • currentState: 設定するサービス状態（SERVICE_RUNNING, SERVICE_STOPPED 等）。
  • waitHint: 状態遷移に要する推定時間（ミリ秒）。
  • ctx: サービス状態を保持する nt_service_ctx_t 構造体へのポインタ。
  • **出力**:
  • int: Windows API SetServiceStatus の戻り値（成功時は 0 以外）。

前提条件と不変条件

  • ctx->hServiceStatus が有効なハンドルであること。
  • ctx->ssStatus 構造体は、呼び出し元によって適切に管理されていること。

アルゴリズムまたは振る舞い

1. ctx->hServiceStatus が存在する場合のみ処理を実行する。 2. currentState に応じて ctx->ssStatus を更新する:

  • SERVICE_RUNNING: waitHint と dwCheckPoint をリセットし、停止およびシャットダウンの受付を許可する。
  • SERVICE_STOPPED: waitHint と dwCheckPoint をリセットする。予期せぬ停止の場合、終了コードをエラーに設定する。
  • その他（遷移中）: dwCheckPoint をインクリメントし、waitHint が指定されていれば更新する。

    • 3. ctx->ssStatus.dwCurrentState を currentState に更新する。 4. Windows API SetServiceStatus を呼び出し、SCM に状態を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/service.c 内の各所（サービス開始、停止、シャットダウン処理など）。
  • **呼び出し先**:
  • SetServiceStatus (Windows API)

関連データ構造

  • nt_service_ctx_t: サービスの状態保持および SCM との通信用コンテキスト。

失敗モードと境界条件

  • ctx->hServiceStatus が NULL の場合、何も行わず APR_SUCCESS を返す。
  • SetServiceStatus が失敗した場合、その戻り値がそのまま返される。

保守担当者向けメモ

  • 本関数は SERVICE_STATUS 構造体の更新と SCM への通知をカプセル化している。
  • dwCheckPoint のインクリメントは、SCM に対してサービスが応答中であることを示すために重要である。
  • 状態遷移のロジックは nt_service_ctx_t 内の既存の状態に依存するため、呼び出し順序に注意すること。
