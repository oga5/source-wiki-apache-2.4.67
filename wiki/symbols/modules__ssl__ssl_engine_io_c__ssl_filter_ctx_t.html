ssl_filter_ctx_t

定義

  • **種別**: struct
  • **定義箇所**: modules/ssl/ssl_engine_io.c:104

目的

SSL 接続におけるメインコンテキストを保持する構造体です。OpenSSL のオブジェクト、BIO（Basic I/O）抽象化レイヤー、および HTTPD のフィルタスタックを統合し、SSL/TLS 通信の管理を行います。

シグネチャまたは形状

typedef struct {
    SSL                *pssl;          /* OpenSSL SSL オブジェクト */
    BIO                *pbioRead;      /* 入力用 BIO */
    BIO                *pbioWrite;     /* 出力用 BIO */
    ap_filter_t        *pInputFilter;  /* HTTPD 入力フィルタ */
    ap_filter_t        *pOutputFilter; /* HTTPD 出力フィルタ */
    SSLConnRec         *config;        /* SSL 接続設定レコード */
} ssl_filter_ctx_t;

入力と出力

  • **入力**: apr_palloc 等を介して接続プール (conn_rec->pool) 上に割り当てられます。
  • **出力**: 構造体内の各メンバは、SSL フィルタのライフサイクルを通じて、接続に関連付けられた OpenSSL セッションおよび HTTPD フィルタの状態を保持します。

前提条件と不変条件

  • conn_rec のプールに関連付けられており、フィルタのライフサイクルと同期してメモリが管理されます。
  • pbioRead および pbioWrite は、OpenSSL が HTTPD のフィルタスタックを介して読み書きを行うためのカスタム BIO として初期化されている必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、ssl_io_filter_input および ssl_io_filter_output において、暗号化・復号化処理の起点となるコンテキストとして参照されます。
  • ssl_io_filter_handshake 実行時のネゴシエーションや、接続終了時の ssl_filter_io_shutdown において、SSL セッションの状態管理に使用されます。

呼び出し元と呼び出し先

  • **主な呼び出し元**:
  • ssl_io_filter_input / ssl_io_filter_output (フィルタ処理)
  • ssl_io_filter_handshake (ハンドシェイク制御)
  • ssl_filter_io_shutdown (接続終了処理)
  • ssl_io_input_add_filter (フィルタ追加時)
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、メンバである pssl (OpenSSL) や pInputFilter/pOutputFilter (HTTPD) を介して間接的に各サブシステムを制御します。

関連データ構造

  • SSLConnRec: SSL 接続設定を保持。
  • bio_filter_in_ctx_t / bio_filter_out_ctx_t: BIO 内部のデータバッファリング用コンテキスト。
  • ap_filter_t: HTTPD のフィルタスタック構造体。

失敗モードと境界条件

  • 構造体自体のメモリ割り当て失敗は apr_palloc の挙動に依存します。
  • メンバの pssl や pbio が適切に初期化されていない状態でフィルタ関数が呼び出された場合、ヌルポインタ参照等の未定義動作が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_ssl の I/O サブシステムの中核です。変更を加える際は、ssl_engine_io.c 内の BIO ライフサイクル管理 (bio_filter_create/destroy) との整合性に注意してください。
  • フィルタのメモリ所有権は conn_rec に紐付いているため、接続終了時のクリーンアップ漏れはメモリリークに直結します。
