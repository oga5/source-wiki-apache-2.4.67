read_qstring

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_push.c:170

目的

Link ヘッダーの解析において、二重引用符（"）で囲まれた文字列（quoted-string）を抽出し、解析コンテキストのカーソルを進める。

シグネチャまたは形状

static int read_qstring(link_ctx *ctx, const char **ps)

入力と出力

  • **入力**:
  • ctx: 解析状態を保持する link_ctx 構造体へのポインタ。
  • ps: 抽出した文字列を格納するための const char * へのポインタ。
  • **出力**:
  • 戻り値: 成功時に 1、失敗時に 0 を返す。
  • ps: 成功時、抽出された文字列が割り当てられる。

前提条件と不変条件

  • ctx は有効な link_ctx オブジェクトを指している必要がある。
  • 内部で呼び出される skip_ws および read_chr が ctx の状態を適切に更新する。

アルゴリズムまたは振る舞い

1. skip_ws(ctx) を呼び出し、先行する空白をスキップする。 2. read_chr(ctx, '\"') を呼び出し、開始の二重引用符を確認する。 3. find_chr(ctx, '\"', &end) を呼び出し、終了の二重引用符の位置を検索する。 4. 成功した場合、mk_str(ctx, end) を用いて文字列を生成し、ps に格納する。 5. ctx->i を終了引用符の次の位置（end + 1）に更新し、1 を返す。 6. 条件を満たさない場合は何も変更せず 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_push.c:232 にて、read_ptoken と組み合わせて呼び出されている。
  • **呼び出し先**: skip_ws, read_chr, find_chr, mk_str（すべて modules/http2/h2_push.c 内）。

関連データ構造

  • link_ctx: 解析対象の文字列や現在のオフセット（i）を保持する構造体。

失敗モードと境界条件

  • 文字列が二重引用符で開始されていない場合、または終了の二重引用符が見つからない場合は 0 を返し、解析状態は変更されない。

保守担当者向けメモ

  • 本関数は Link ヘッダーの構文解析の一部であり、h2_push_collect によるプッシュ候補抽出の過程で使用される。
  • mk_str は apr_pool_t を使用してメモリを割り当てるため、呼び出し側でのメモリ管理に注意すること。
