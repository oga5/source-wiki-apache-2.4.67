SSLize_Socket

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:393

目的

既存のソケットを NetWare ネイティブの SSL/TLS モードへ遷移させ、暗号化通信を有効化します。

シグネチャまたは形状

static int SSLize_Socket(SOCKET socketHnd, char *key, request_rec *r)

入力と出力

  • **入力**:
  • socketHnd: 操作対象のソケットハンドル。
  • key: SSL 証明書ストア（KMO）を特定するためのキー文字列。
  • r: 現在のリクエストレコード（エラーログ出力に使用）。
  • **出力**:
  • int: 成功時は 0、失敗時は WSAIoctl が返すエラーコード。

前提条件と不変条件

  • socketHnd は有効な WinSock ソケットであること。
  • key は loc2uni 関数によって Unicode 形式の SASKey に変換可能であること。

アルゴリズムまたは振る舞い

1. WSAIoctl を使用し、SO_TLS_SET_FLAGS で SO_TLS_ENABLE を設定して TLS を有効化する。 2. 同様に SO_TLS_SERVER フラグを設定し、サーバーモードとして構成する。 3. key を Unicode に変換し、tlsserveropts 構造体に格納する。 4. nwtlsopts 構造体を初期化し、walletProvider を WAL_PROV_KMO に設定する。 5. WSAIoctl の SO_TLS_SET_SERVER を呼び出し、上記構造体を用いてソケットに SSL 設定を適用する。 6. 各ステップでエラーが発生した場合は、ap_log_error でログを出力し、エラーコードを返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_nw_ssl.c:1214 (TLS アップグレード処理内)
  • **呼び出し先**:
  • WSAIoctl (WinSock API)
  • loc2uni (Unicode 変換)
  • ap_log_error (Apache エラーログ出力)
  • unilen (文字列長計算)

関連データ構造

  • struct tlsserveropts: SSL サーバーオプションを保持。
  • struct nwtlsopts: NetWare 固有の TLS オプションを保持。
  • SOCKET: WinSock ソケットハンドル。

失敗モードと境界条件

  • WSAIoctl が SOCKET_ERROR を返した場合、エラーログを出力し、その時点での WSAGetLastError() を記録して処理を中断する。
  • key の変換や構造体の設定が不完全な場合、SSL 接続の確立に失敗する。

保守担当者向けメモ

  • 本関数は NetWare の WinSock 拡張に強く依存しています。
  • nwtlsopts の reserved フィールドは将来の拡張用であり、現在は使用されていません。
  • メモリ管理は Apache のプール機構に依存しているため、明示的な解放は不要です。
