filegroup_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_groupfile.c:209

目的

require file-group 認可プロバイダーとして、リクエストされたファイルの所有グループと、認証済みユーザーが所属するグループを照合し、アクセス可否を判定する。

シグネチャまたは形状

static authz_status filegroup_check_authorization(request_rec *r,
                                                  const char *require_args,
                                                  const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。r->user（ユーザー名）および r->per_dir_config を使用。
  • require_args, parsed_require_args: 本関数内では直接使用されていない。
  • **出力**:
  • AUTHZ_GRANTED: ユーザーがファイルグループに所属している場合。
  • AUTHZ_DENIED: ユーザーが未認証、グループファイル未設定、グループファイル読み込み失敗、または所属グループ不一致の場合。
  • AUTHZ_DENIED_NO_USER: r->user が設定されていない場合。

前提条件と不変条件

  • r->user が存在すること。
  • conf->groupfile（AuthGroupFile ディレクティブ）が設定されていること。
  • groups_for_user が成功し、ユーザーの所属グループ情報が grpstatus テーブルに格納されること。

アルゴリズムまたは振る舞い

1. ユーザー認証の有無を確認し、未認証なら AUTHZ_DENIED_NO_USER を返す。 2. 設定されたグループファイルパスを確認し、未設定ならエラーログを出力して AUTHZ_DENIED を返す。 3. groups_for_user を呼び出し、ユーザーが所属する全グループを grpstatus テーブルに取得する。取得失敗時は AUTHZ_DENIED を返す。 4. grpstatus が空であれば、ユーザーがグループファイルに存在しないとみなし AUTHZ_DENIED を返す。 5. authz_owner_get_file_group(r) を呼び出し、対象ファイルの所有グループ名を取得する。 6. 取得したファイルグループ名が grpstatus テーブルに含まれているか確認し、存在すれば AUTHZ_GRANTED を返す。 7. 一致しない場合はデバッグログを出力し、AUTHZ_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_groupfile.c:301 にてプロバイダー構造体経由で参照されている（直接的な呼び出し元は mod_authz_core と推測されるが、ソースコード上での明示的な呼び出し箇所は未確認）。
  • **呼び出し先**:
  • groups_for_user: ユーザーの所属グループ一覧取得。
  • authz_owner_get_file_group: 対象ファイルの所有グループ取得。
  • ap_log_rerror: エラーおよびデバッグログ出力。

関連データ構造

  • authz_groupfile_config_rec: ディレクトリごとの設定（groupfile パスを保持）。
  • apr_table_t: ユーザーの所属グループを保持するテーブル。

失敗モードと境界条件

  • **グループファイル未指定**: 設定エラーとして AUTHZ_DENIED を返す。
  • **ファイル読み込み失敗**: groups_for_user のエラーに基づき AUTHZ_DENIED を返す。
  • **ユーザーがグループファイルに不在**: grpstatus が空の場合、明示的にエラーログを出力して拒否する。
  • **ファイル所有グループ取得失敗**: authz_owner_get_file_group が NULL を返した場合、同関数内でエラーログが出力されるため、そのまま AUTHZ_DENIED を返す。

保守担当者向けメモ

  • 本関数は mod_authz_owner モジュールに強く依存している。
  • groups_for_user はリクエストごとにファイルを走査するため、大規模なグループファイルではパフォーマンスに影響を与える可能性がある。
