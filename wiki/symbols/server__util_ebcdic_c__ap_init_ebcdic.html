ap_init_ebcdic

定義

  • 種別: function
  • 定義箇所: server/util_ebcdic.c:31

目的

EBCDIC環境において、プロトコルデータとASCII間の文字コード変換を行うためのハンドルを初期化し、関連するライブラリ（MD5, Base64, SHA1）に変換設定を適用する。

シグネチャまたは形状

apr_status_t ap_init_ebcdic(apr_pool_t *pool)

入力と出力

  • **入力**: pool (メモリ割り当て用の apr_pool_t ポインタ)
  • **出力**: apr_status_t 型のステータスコード。成功時は APR_SUCCESS を返し、失敗時はエラーコードを返す。

前提条件と不変条件

  • 本関数は APR_CHARSET_EBCDIC が定義されている環境でのみ有効に機能する。
  • 呼び出し時に有効な apr_pool_t が提供されている必要がある。
  • 内部で保持するグローバルな変換ハンドル（ap_hdrs_to_ascii, ap_hdrs_from_ascii）の初期化を行う。

アルゴリズムまたは振る舞い

1. apr_xlate_open を使用し、ローカル文字セットからASCII（ISO-8859-1）への変換ハンドルを生成し、ap_hdrs_to_ascii に格納する。 2. 同様に、ASCIIからローカル文字セットへの変換ハンドルを生成し、ap_hdrs_from_ascii に格納する。 3. 生成したハンドルを用いて、以下のライブラリに対してEBCDIC変換設定を適用する。

  • apr_MD5InitEBCDIC
  • apr_base64init_ebcdic
  • apr_SHA1InitEBCDIC

    • 4. 各ステップでエラーが発生した場合は、ap_log_error でログを出力し、直ちにエラーコードを返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/main.c:521 (サーバー起動時の初期化プロセス)
  • **呼び出し先**:
  • apr_xlate_open
  • ap_log_error
  • apr_MD5InitEBCDIC
  • apr_base64init_ebcdic
  • apr_SHA1InitEBCDIC

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • apr_status_t: 処理結果のステータス。
  • ap_hdrs_to_ascii, ap_hdrs_from_ascii: 本関数内で初期化されるグローバルな変換ハンドル（apr_xlate_t 型）。

失敗モードと境界条件

  • apr_xlate_open が失敗した場合、変換ハンドルが生成できず、エラーログを出力して終了する。
  • MD5, Base64, SHA1 の各初期化関数が失敗した場合、同様にエラーログを出力して終了する。
  • 非EBCDIC環境では、ヘッダーファイル util_ebcdic.h により本関数は空のマクロに置換されるため、実行時の処理は発生しない。

保守担当者向けメモ

  • 本関数はサーバー起動時に一度だけ呼び出されることを想定している。
  • 変換ハンドルはグローバル変数として管理されているため、マルチスレッド環境での初期化順序に注意すること。
  • ISO-8859-1 を固定の変換対象としているため、将来的に他の文字セット対応が必要な場合はこの箇所を再検討する必要がある。
