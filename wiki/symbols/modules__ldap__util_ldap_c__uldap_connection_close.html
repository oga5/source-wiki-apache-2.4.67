uldap_connection_close

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:197

目的

LDAP 接続の利用終了処理を行い、接続を破棄するか、再利用可能な状態としてプールへ戻す。

シグネチャまたは形状

static void uldap_connection_close(util_ldap_connection_t *ldc)

入力と出力

  • **入力**: ldc (util_ldap_connection_t *) - 終了対象の LDAP 接続構造体。
  • **出力**: なし（副作用として接続状態を更新）。

前提条件と不変条件

  • ldc は有効な util_ldap_connection_t 構造体を指している必要がある。
  • 本関数呼び出し時点で、ldc->lock がロックされていることが期待される（関数末尾でアンロックされるため）。

アルゴリズムまたは振る舞い

1. **接続の破棄判定**: ldc->keep フラグを確認する。

  • keep が偽の場合: uldap_connection_unbind(ldc) を呼び出して LDAP 接続を物理的に切断し、ldc->r を NULL に設定する。
  • keep が真の場合: ldc->freed に現在の時刻（apr_time_now()）を記録し、再利用可能な状態としてマークする。その後 ldc->r を NULL に設定する。

    • 2. **スレッド同期**: APR_HAS_THREADS が定義されている場合、ldc->lock をアンロックして排他制御を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_ldap.c にて APR_RETRIEVE_OPTIONAL_FN を通じて取得・利用されている。
  • **呼び出し先**:
  • uldap_connection_unbind (条件付き)
  • apr_time_now
  • apr_thread_mutex_unlock (条件付き)

関連データ構造

  • util_ldap_connection_t: LDAP 接続の状態、プール管理フラグ、および同期用ミューテックスを保持する構造体。

失敗モードと境界条件

  • ldc が NULL の場合、構造体メンバへのアクセスによりセグメンテーション違反が発生する可能性がある。
  • ldc->lock が既にアンロックされている状態で呼び出された場合、スレッドライブラリの挙動に依存したエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN を通じて外部モジュール（mod_authnz_ldap 等）に公開されている。インターフェース変更時は注意が必要。
  • ldc->keep フラグは、接続をプール内に維持するかどうかを決定する重要な制御フラグである。
