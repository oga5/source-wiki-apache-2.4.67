uwsgi_send_headers

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_uwsgi.c:148

目的

Apache の subprocess_env に格納された環境変数を uWSGI プロトコル仕様のバイナリ形式に変換し、バックエンドへ送信します。

シグネチャまたは形状

static int uwsgi_send_headers(request_rec *r, proxy_conn_rec *conn)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報（subprocess_env を含む）。
  • conn: バックエンドへの接続情報。
  • **出力**:
  • 成功時: uwsgi_send の戻り値（通常は OK）。
  • 失敗時: HTTP_INTERNAL_SERVER_ERROR（パケットサイズ超過時）。

前提条件と不変条件

  • r->subprocess_env に必要な CGI 環境変数が設定されていること。
  • conn は有効な接続状態であること。
  • 送信されるヘッダーの総サイズ（pktsize）は APR_UINT16_MAX 以下であること。

アルゴリズムまたは振る舞い

1. **環境変数の準備**: ap_add_common_vars および ap_add_cgi_vars を呼び出し、Authorization ヘッダーを HTTP_AUTHORIZATION として環境変数に追加します。 2. **パス調整**: SCRIPT_NAME と PATH_INFO が存在する場合、SCRIPT_NAME を適切に調整します。 3. **サイズ計算**: 全ての環境変数のキーと値の長さを合計し、uWSGI ヘッダーに必要なバッファサイズを算出します。 4. **バリデーション**: 合計サイズが APR_UINT16_MAX を超える場合はエラーを返します。 5. **シリアライズ**: メモリを確保し、各キーと値を [2バイト長][データ] の形式でバイナリバッファに書き込みます。 6. **ヘッダー構築**: バッファの先頭4バイトに uWSGI プロトコル固有のヘッダー（パケットサイズ等）を書き込みます。 7. **送信**: uwsgi_send を呼び出し、構築したバッファをバックエンドへ送信します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_uwsgi.c:571（uwsgi_handler 内から呼び出される）。
  • **呼び出し先**:
  • ap_add_common_vars, ap_add_cgi_vars
  • apr_table_get, apr_table_set, apr_table_setn, apr_table_elts
  • apr_palloc, apr_pstrndup
  • ap_log_rerror
  • uwsgi_send

関連データ構造

  • request_rec: リクエスト処理のコンテキスト。
  • proxy_conn_rec: バックエンド接続の管理構造体。
  • apr_table_entry_t: 環境変数のキーと値を保持するテーブルエントリ。

失敗モードと境界条件

  • **パケットサイズ超過**: pktsize > APR_UINT16_MAX の場合、ログを出力して HTTP_INTERNAL_SERVER_ERROR を返します。
  • **メモリ不足**: apr_palloc が失敗した場合の挙動は APR のメモリ管理ポリシーに従います。

保守担当者向けメモ

  • 本関数は uWSGI プロトコルのバイナリ仕様に直接依存しています。仕様変更時には、ヘッダーのシリアライズ部分（特にビット操作）を慎重に確認してください。
  • SCRIPT_NAME の調整ロジックは、Apache の環境変数と uWSGI 側の期待値の差異を埋めるためのものです。
  • 接続の再利用（Keep-Alive）は行われない設計であるため、本関数はリクエストごとに一度だけ実行されます。
