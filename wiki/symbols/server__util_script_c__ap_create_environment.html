ap_create_environment

定義

  • 種別: function
  • 定義箇所: server/util_script.c:112

目的

apr_table_t に格納されたリクエスト環境変数を、OS の exec システムコール等で利用可能な char ** 形式の配列（環境変数ブロック）へ変換します。

シグネチャまたは形状

char **ap_create_environment(apr_pool_t *p, apr_table_t *t)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • t: 環境変数のキーと値のペアを保持する apr_table_t。
  • **出力**:
  • 変換された環境変数の文字列配列（char **）。末尾は NULL で終端されます。

前提条件と不変条件

  • すべての動的メモリ割り当ては、提供された apr_pool_t *p から行われます。
  • 入力テーブル t に TZ 変数が存在しない場合、現在のプロセス環境から TZ を取得して配列に追加します。
  • 出力される各文字列は KEY=VALUE の形式です。

アルゴリズムまたは振る舞い

1. **初期化**: TZ 変数の確認と追加を行います。 2. **変換と正規化**: 入力テーブル t の各要素を走査します。

  • キーの先頭が数字の場合、_ に置換します。
  • キー内の英数字および _ 以外の文字を _ に置換します（Windows 環境では ( および ) も許可されます）。

    • 3. **重複排除**: 正規化の結果、既存のキーと衝突する場合はその変数を破棄します。 4. **終端**: 配列の最後に NULL を格納して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_ext_filter.c
  • modules/generators/mod_cgi.c
  • modules/generators/mod_cgid.c
  • **呼び出し先**:
  • apr_table_elts, apr_palloc, apr_table_get, apr_pstrcat, apr_isdigit, apr_isalnum などの APR ユーティリティ関数。

関連データ構造

  • apr_table_t: 環境変数のソースとなるキー・値ペアのテーブル。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • メモリ不足時は apr_palloc によりアボートまたはエラーが発生する可能性があります。
  • キーが空（NULL）の要素はスキップされます。
  • 正規化後のキーが既存のキーと重複する場合、その変数は環境変数ブロックに含まれません。

保守担当者向けメモ

  • 本関数は CGI 仕様に準拠した環境変数の構築を目的としています。
  • WIN32 定義時とそれ以外で、キー名の正規化ルール（許可される文字）が異なる点に注意してください。
  • mod_proxy_fcgi 等の他モジュールでも同様の機能が必要とされる場合がありますが、本関数は util_script.c 内で完結しています。
