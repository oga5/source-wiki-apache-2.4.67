cipher_init

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_protocol.c:403

目的

HTTP/2 で禁止されている暗号スイート（Cipher Suite）を管理するためのブラックリスト用ハッシュテーブルを初期化し、RFC 7540 で定義された暗号名を設定します。

シグネチャまたは形状

static void cipher_init(apr_pool_t *pool)

入力と出力

  • **入力**: apr_pool_t *pool - ハッシュテーブルのメモリ割り当てに使用する APR プール。
  • **出力**: なし（副作用としてグローバル変数 BLCNames を更新）。

前提条件と不変条件

  • BLCNames は本関数によって初期化される必要があります。
  • RFC7540_names 配列および RFC7540_names_LEN が定義されている必要があります。

アルゴリズムまたは振る舞い

1. apr_hash_make(pool) を呼び出し、新しいハッシュテーブルを作成します。 2. RFC7540_names 配列を反復処理し、各暗号名をキーとして、値に "rfc7540" という文字列をハッシュテーブルに格納します。 3. 構築したハッシュテーブルをグローバル変数 BLCNames に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_protocol_init (modules/http2/h2_protocol.c:427)
  • **呼び出し先**:
  • apr_hash_make
  • apr_hash_set

関連データ構造

  • BLCNames: ブラックリストを保持する apr_hash_t 型のグローバル変数。
  • RFC7540_names: ブラックリスト対象の暗号スイート名が格納された静的配列。

失敗モードと境界条件

  • apr_hash_make がメモリ不足により失敗した場合、APR の規約に従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は h2_protocol_init を通じてモジュール初期化時に実行されます。
  • 新たな脆弱な暗号スイートが発見された場合、RFC7540_names 配列を更新することで本関数が自動的にブラックリストへ追加します。
