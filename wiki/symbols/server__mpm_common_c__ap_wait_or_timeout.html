ap_wait_or_timeout

定義

  • 種別: function
  • 定義箇所: server/mpm_common.c:181

目的

子プロセスの終了を非ブロックで待機し、終了していない場合は定期的な監視タスクの実行や待機（スリープ）を行うことで、サーバーの健全性を維持します。

シグネチャまたは形状

void ap_wait_or_timeout(apr_exit_why_e *status, int *exitcode,
                        apr_proc_t *ret, apr_pool_t *p,
                        server_rec *s)

入力と出力

  • **入力**:
  • status: プロセスの終了理由（apr_exit_why_e）を格納するポインタ。
  • exitcode: プロセスの終了コードを格納するポインタ。
  • ret: 終了したプロセス情報を格納する apr_proc_t 構造体へのポインタ。
  • p: メモリ割り当てに使用するプール。
  • s: サーバー設定レコード。
  • **出力**:
  • なし（引数ポインタ経由で状態を更新）。

前提条件と不変条件

  • apr_proc_wait_all_procs が呼び出されるため、呼び出し元は適切なプロセス管理環境にある必要があります。
  • 内部で wait_or_timeout_counter を使用しており、このカウンタはモジュール内で共有されています。

アルゴリズムまたは振る舞い

1. wait_or_timeout_counter をインクリメントし、INTERVAL_OF_WRITABLE_PROBES に達した場合は ap_run_monitor(p, s) を実行してサーバーの監視フックを呼び出します。 2. apr_proc_wait_all_procs を APR_NOWAIT モードで呼び出し、子プロセスの終了を非ブロックで確認します。 3. ap_update_global_status() を呼び出し、グローバルなプロセス状態を更新します。 4. 割り込み（EINTR）が発生した場合は ret->pid を -1 に設定して即座に復帰します。 5. 子プロセスが終了していれば（CHILD_DONE）、そのまま復帰します。 6. いずれも該当しない場合、apr_sleep で 1 秒間待機し、ret->pid を -1 に設定して復帰します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/prefork/prefork.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • ap_run_monitor
  • apr_proc_wait_all_procs
  • ap_update_global_status
  • apr_sleep

関連データ構造

  • apr_proc_t: プロセス情報を保持する構造体。
  • server_rec: サーバー設定レコード。
  • apr_exit_why_e: プロセスの終了理由を示す列挙型。

失敗モードと境界条件

  • 子プロセスが終了していない場合、1 秒間のブロッキングが発生します。高負荷環境ではこの待機時間がサーバーの応答性に影響を与える可能性があります。
  • APR_STATUS_IS_EINTR(rv) が真の場合、プロセス待機は中断されます。

保守担当者向けメモ

  • 本関数は MPM のメインループ内で頻繁に呼び出されるため、ap_run_monitor に登録されるフック関数の処理時間は最小限に抑える必要があります。
  • wait_or_timeout_counter は静的変数であり、マルチスレッド環境下での競合については server/mpm_common.c の実装文脈を確認してください。
