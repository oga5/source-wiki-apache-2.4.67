msg_entry

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_log_debug.c:28

目的

LogMessage ディレクティブによって設定された個別のデバッグログ出力ルールを保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    ap_expr_info_t *msg_expr;
    ap_expr_info_t *condition;
    const char *hook;
} msg_entry;

入力と出力

  • **入力**: cmd_log_message 関数において、設定ファイルから読み込まれた式およびフック名が本構造体の各フィールドに格納されます。
  • **出力**: do_debug_log 関数が本構造体を参照し、ap_expr エンジンを用いて条件判定およびメッセージ生成を行います。

前提条件と不変条件

  • msg_expr および condition は ap_expr_parse_cmd によって生成された有効な式情報である必要があります。
  • hook は Apache のリクエスト処理フェーズに対応する文字列である必要があります。
  • 設定データは cmd->pool（設定用プール）に割り当てられ、リクエスト処理中は r->pool のライフサイクルに依存します。

アルゴリズムまたは振る舞い

1. **設定時**: cmd_log_message が LogMessage ディレクティブをパースし、apr_pcalloc でメモリを確保して各フィールドを初期化します。 2. **実行時**: do_debug_log が log_debug_dirconf 内の msg_entry 配列を走査します。 3. **評価**: ap_expr_exec を用いて condition を評価し、真であれば ap_expr_str_exec を用いて msg_expr からログメッセージを生成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • cmd_log_message (設定の生成)
  • do_debug_log (設定の読み取り)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。評価には ap_expr API（ap_expr_exec, ap_expr_str_exec）が使用されます。

関連データ構造

  • log_debug_dirconf: 本構造体を apr_array_header_t 内に複数保持するディレクトリ単位の設定構造体。
  • ap_expr_info_t: ap_expr エンジンで使用される式情報構造体。

失敗モードと境界条件

  • **メモリ不足**: apr_pcalloc や apr_array_make が失敗した場合、設定の読み込みが中断されます。
  • **式評価エラー**: ap_expr_exec がエラーを返した場合、そのエントリのログ出力はスキップされます。
  • **無効なフック名**: 設定時に検証されない場合、実行時に do_debug_log が意図しないタイミングで呼ばれる可能性があります。

保守担当者向けメモ

  • apr_array_append を使用したマージ処理が行われるため、設定の追加時にはメモリ消費量に注意してください。
  • ap_expr の評価はリクエスト処理の各フェーズで行われるため、複雑な式を定義するとリクエストのレイテンシに直接影響します。
