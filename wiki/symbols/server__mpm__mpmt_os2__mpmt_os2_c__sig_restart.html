sig_restart

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:366

目的

Apache HTTP Server の再起動またはグレースフル再起動を要求するためのシグナルハンドラです。

シグネチャまたは形状

void sig_restart(int sig)

入力と出力

  • 入力: int sig（受信したシグナル番号）
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • restart_pending および is_graceful は、本関数が呼び出される前に適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 受信したシグナル sig が SIGUSR1 であるか確認します。 2. SIGUSR1 の場合、グレースフル再起動を示す is_graceful を 1 に設定します。 3. 再起動の保留状態を示す restart_pending を 1 に設定します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/mpm/mpmt_os2/mpmt_os2.c:391（set_signals 関数内でのシグナルハンドラ登録）
  • os/unix/unixd.c において、sig_restart への参照が確認されています（ただし、OS/2 MPM の文脈における直接的な呼び出しフローの詳細は、提供されたソース範囲外のため不明です）。
  • 呼び出し先: なし。

関連データ構造

  • restart_pending (グローバル変数): 再起動の実行を制御するフラグ。
  • is_graceful (グローバル変数): 再起動の種類（通常またはグレースフル）を識別するフラグ。

失敗モードと境界条件

  • 本関数はシグナルハンドラとして設計されているため、非同期シグナル安全性が求められます。
  • sig が SIGUSR1 以外の場合、is_graceful は更新されず、通常の再起動フラグのみが立てられます。

保守担当者向けメモ

  • 本関数は OS/2 MPM におけるシグナル処理の一部です。
  • os/unix/unixd.c からの参照が見られますが、OS/2 環境下でのシグナル伝播の整合性に注意してください。
