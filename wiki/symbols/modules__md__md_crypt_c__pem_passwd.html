pem_passwd

定義

  • 種別: function
  • 定義箇所: modules/md/md_crypt.c:188

目的

OpenSSL の PEM 形式秘密鍵読み込み処理において、パスフレーズを供給するためのコールバック関数として機能する。

シグネチャまたは形状

static int pem_passwd(char *buf, int size, int rwflag, void *baton)

入力と出力

  • **入力**:
  • buf: パスフレーズを書き込むためのバッファ。
  • size: buf のサイズ。
  • rwflag: 読み込み/書き込みのフラグ（本関数内では使用されない）。
  • baton: passwd_ctx 構造体へのポインタ（パスフレーズ情報を含む）。
  • **出力**:
  • 成功時: buf にコピーされたパスフレーズのバイト数。
  • 失敗時: 0。

前提条件と不変条件

  • baton は passwd_ctx 型としてキャスト可能であること。
  • ctx->pass_len が 0 より大きい場合、ctx->pass_phrase に有効なパスフレーズが格納されていること。

アルゴリズムまたは振る舞い

1. baton を passwd_ctx にキャストする。 2. rwflag を無視する。 3. ctx->pass_len が 0 より大きい場合:

  • ctx->pass_len が size より小さい場合は、コピーサイズを ctx->pass_len に制限する。
  • ctx->pass_phrase を buf にコピーする。
  • コピーしたバイト数を返す。

    • 4. ctx->pass_len が 0 以下の場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_crypt.c:645: PEM_read_bio_PrivateKey の引数として渡される。
  • modules/md/md_crypt.c:686: 関数ポインタとして変数 cb に代入される。
  • **呼び出し先**:
  • memcpy (標準ライブラリ)

関連データ構造

  • passwd_ctx: パスフレーズ文字列とその長さを保持する構造体。

失敗モードと境界条件

  • ctx->pass_len が 0 の場合、パスフレーズは提供されず 0 が返される。
  • ctx->pass_len が size を超える場合、バッファオーバーフローを防ぐため size 分のみがコピーされる。

保守担当者向けメモ

  • 本関数は OpenSSL の pem_password_cb 型に適合するように設計されている。
  • rwflag は将来的な拡張や OpenSSL API との互換性のために保持されているが、現在の実装では使用されていない。
