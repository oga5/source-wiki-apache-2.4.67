systemd_monitor

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_systemd.c:97

目的

Apache HTTP Server の稼働統計（リクエスト数、負荷状況、稼働時間など）を定期的に取得し、sd_notifyf を介して systemd にステータス文字列として送信する。

シグネチャまたは形状

static int systemd_monitor(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: メモリ割り当て用のプール。
  • s: サーバーレコード（未使用）。
  • **出力**:
  • 常に DECLINED を返す。

前提条件と不変条件

  • ap_extended_status が有効であること。無効な場合、本関数は何も行わず即座に終了する。
  • スコアボード（ap_scoreboard_image）が初期化済みであること。

アルゴリズムまたは振る舞い

1. ap_extended_status を確認し、無効であれば DECLINED を返して終了する。 2. ap_get_sload を呼び出し、現在のサーバー負荷統計（ap_sload_t）を取得する。 3. サーバーの再起動時刻から現在までの稼働時間（秒）を算出する。 4. 稼働時間と総バイト数から、1秒あたりの転送量（バイト/秒）を計算し、apr_strfsize で人間が読みやすい形式に変換する。 5. sd_notifyf を使用し、以下の情報を含むステータス文字列を systemd に送信する:

  • READY=1（サービス稼働中）
  • 総リクエスト数
  • アイドル/ビジー状態のワーカー数
  • 1秒あたりのリクエスト数
  • 1秒あたりの転送バイト数

    • 6. DECLINED を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/unix/mod_systemd.c:168 にて ap_hook_monitor に登録されており、サーバーの監視フェーズで定期的に呼び出される。
  • **呼び出し先**:
  • ap_get_sload: 統計情報の取得。
  • apr_time_now, apr_time_sec: 稼働時間の計算。
  • apr_strfsize: 転送量のフォーマット変換。
  • sd_notifyf: systemd への通知。

関連データ構造

  • ap_sload_t: サーバー負荷統計を保持する構造体。
  • ap_scoreboard_image: サーバー全体の稼働状況を保持する共有メモリ構造体。

失敗モードと境界条件

  • **稼働時間が0秒の場合**: ゼロ除算が発生する可能性がある（up_time が 0 の場合）。
  • **ExtendedStatus 無効時**: 処理をスキップし、systemd への通知は行われない。

保守担当者向けメモ

  • 本関数は ap_hook_monitor を通じて実行されるため、高頻度での呼び出しは避ける設計が望ましい。
  • sd_notifyf に渡される文字列のフォーマットは systemd のプロトコル仕様に依存している。
  • 統計情報の取得には ap_scoreboard_image を直接参照しているため、スコアボードの構造変更時には注意が必要である。
