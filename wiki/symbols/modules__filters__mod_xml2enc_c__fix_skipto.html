fix_skipto

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_xml2enc.c:145

目的

設定された特定のHTML要素が出現するまで、バッファ内のコンテンツをスキップ（破棄）し、ストリームの開始位置を調整する。

シグネチャまたは形状

static void fix_skipto(request_rec* r, xml2ctx* ctx)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • ctx: フィルタの実行状態を保持する xml2ctx 構造体。
  • **出力**: なし（副作用として ctx->buf および ctx->bbsave を更新）。

前提条件と不変条件

  • cfg->skipto が設定されており、かつ ctx->flags に ENC_SKIPTO フラグが立っている場合にのみ実行される。
  • ctx->bbsave に格納されたバケットブリゲードが、ctx->buf の内容と同期している必要がある。

アルゴリズムまたは振る舞い

1. cfg->skipto に定義された要素リストを走査する。 2. ctx->buf 内を検索し、リスト内のいずれかの要素タグ（<tag）が最初に出現する位置を特定する。 3. 要素が見つかった場合:

  • apr_brigade_partition を使用して、該当位置までのバケットをブリゲードから分離する。
  • 分離されたバケットを削除し、ctx->buf および ctx->bytes を更新して、スキップ後の位置から処理が継続されるようにする。

    • 4. 要素が見つからなかった場合、警告ログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_xml2enc.c:393
  • **呼び出し先**:
  • ap_get_module_config
  • ap_strchr
  • apr_brigade_partition
  • ap_assert
  • APR_BRIGADE_FIRST
  • apr_bucket_delete
  • ap_log_rerror

関連データ構造

  • xml2ctx: フィルタのバッファと状態を保持。
  • xml2cfg: モジュールのディレクトリ設定（スキップ対象要素リストを含む）。
  • tattr: スキップ対象の要素名を保持する構造体。

失敗モードと境界条件

  • **要素未検出**: バッファ内に指定された要素が存在しない場合、警告ログを出力し、スキップ処理を行わずに終了する。
  • **メモリ/ブリゲード操作**: apr_brigade_partition が失敗した場合、ap_assert により異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は ctx->buf を直接操作するため、バッファの整合性が崩れないよう注意が必要。
  • ap_assert の使用箇所については、将来的にエラーハンドリングへの置き換えが推奨される。
