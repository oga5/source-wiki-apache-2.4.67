rate_limit_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ratelimit.c:59

目的

Apache HTTP Server の出力フィルタとして、レスポンスの転送速度を環境変数 rate-limit に基づいて制限する。

シグネチャまたは形状

static apr_status_t rate_limit_filter(ap_filter_t *f, apr_bucket_brigade *bb)

入力と出力

  • **入力**: ap_filter_t *f (フィルタコンテキスト), apr_bucket_brigade *bb (送信データブリゲード)
  • **出力**: apr_status_t (処理結果。成功時は APR_SUCCESS)

前提条件と不変条件

  • サブリクエストに対してはフィルタを解除し、制限を適用しない。
  • 環境変数 rate-limit が未設定の場合、フィルタを解除する。
  • フィルタコンテキスト f->ctx は初回呼び出し時に初期化される。
  • RATE_INTERVAL_MS (200ms) ごとに apr_sleep を呼び出すため、スレッドがブロッキングされる可能性がある。

アルゴリズムまたは振る舞い

1. **初期化**: 初回実行時に rl_ctx_t を生成し、環境変数から制限速度とバースト量を設定。 2. **状態遷移**:

  • RATE_FULLSPEED: RL_END バケットを検出するまでデータを制限なしで通過させる。
  • RATE_LIMIT: RL_START バケットを検出するまで、chunk_size ごとにデータを分割し、apr_sleep を挟んで送信する。

    • 3. **データ処理**:

  • holdingbb を使用して、処理しきれなかったデータを次回の呼び出しへ持ち越す。
  • tmpbb を使用して、送信単位のチャンクを一時保持する。

    • 4. **終了処理**: EOS (End of Stream) バケットを検出した際、フィルタを解除する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache フィルタチェーン（ap_register_output_filter により登録）。
  • **呼び出し先**: ap_pass_brigade (後続フィルタへのデータ送出), ap_remove_output_filter (フィルタ解除), apr_sleep (速度制御のための待機)。

関連データ構造

  • rl_ctx_t: フィルタの実行状態、速度、バースト量、ブリゲード（tmpbb, holdingbb）を保持するコンテキスト。
  • rl_state_e: 動作モード（RATE_LIMIT または RATE_FULLSPEED）。

失敗モードと境界条件

  • ratelimit が 0 以下の場合、ログを出力してフィルタを解除する。
  • apr_brigade_partition が失敗した場合、エラーを返して処理を中断する。
  • ユーザーの切断等により ap_pass_brigade が失敗した場合、そのエラーを伝播する。

保守担当者向けメモ

  • RLFDEBUG マクロを定義することで、brigade_dump による詳細なデバッグ出力が可能。
  • 将来的に Optional Functions への移行が検討されており、モジュールのロード順序依存を解消する可能性がある。
