ssi_internal_ctx

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_include.c:174

目的

SSI（Server Side Includes）処理における解析状態、バッファ、引数リスト、および式評価コンテキストを保持する非公開の内部構造体です。

シグネチャまたは形状

struct ssi_internal_ctx {
    parse_state_t state;
    int           seen_eos;
    int           error;
    char          quote;
    apr_size_t    parse_pos;
    apr_size_t    bytes_read;
    apr_bucket_brigade *tmp_bb;
    const char   *start_seq;
    bndm_t       *start_seq_pat;
    const char   *end_seq;
    apr_size_t    end_seq_len;
    char         *directive;
    apr_size_t    directive_len;
    arg_item_t   *current_arg;
    arg_item_t   *argv;
    backref_t    *re;
    const char   *undefined_echo;
    apr_size_t    undefined_echo_len;
    char         legacy_expr;
    ap_expr_eval_ctx_t *expr_eval_ctx;
    const char         *expr_vary_this;
    const char         *expr_err;
    /* DEBUG_INCLUDE 定義時のみ debug メンバが存在 */
};

入力と出力

本構造体は関数引数として渡されるのではなく、include_ctx_t（公開コンテキスト）の内部メンバとして保持されます。各解析ステップで state や argv が更新され、最終的な出力は tmp_bb（バケットブリゲード）を介してフィルタチェーンへ渡されます。

前提条件と不変条件

  • include_ctx_t の一部としてライフサイクルが管理されます。
  • dpool（ディレクティブ実行用プール）と連動してメモリが確保されることが前提です。
  • legacy_expr フラグにより、式評価エンジン（ap_expr かレガシーか）が切り替えられます。

アルゴリズムまたは振る舞い

  • ストリームベースのステートマシン（parse_state_t）の状態を保持し、タグの開始・終了シーケンスの検出を追跡します。
  • bndm_t を用いた高速なパターンマッチングにより、SSI タグの開始位置を特定します。
  • argv リストを通じて、解析されたディレクティブの引数を保持し、実行時に参照します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_include.c 内の複数の関数（2841, 2916, 3011, 3110, 3134, 3177, 3391, 3849 行付近）が、ctx->intern を通じて本構造体にアクセスしています。
  • **呼び出し先**: ap_expr 関連の関数や、バケット操作関数が本構造体のメンバを参照・更新します。

関連データ構造

  • parse_state_t: 解析状態の遷移を定義。
  • arg_item_t: ディレクティブの引数リスト。
  • bndm_t: 高速検索用パターン。
  • ap_expr_eval_ctx_t: ap_expr による式評価コンテキスト。
  • include_ctx_t: 本構造体を保持する上位のコンテキスト。

失敗モードと境界条件

  • error メンバにより解析エラー状態を保持します。
  • parse_pos は、タグがバッファ境界をまたぐ際の不完全なマッチング状態を管理します。
  • re（正規表現）が未初期化の場合、NULL が設定されます。

保守担当者向けメモ

  • 本構造体は非公開であり、mod_include.c 内部でのみ使用されます。
  • DEBUG_INCLUDE が定義されている場合、構造体のサイズとレイアウトが変化するため注意してください。
  • メモリ管理は include_ctx_t のプールに依存しているため、サブリクエスト生成時の寿命管理には特に留意してください。
