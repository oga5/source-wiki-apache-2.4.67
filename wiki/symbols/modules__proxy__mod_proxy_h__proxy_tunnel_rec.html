proxy_tunnel_rec

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:1386

目的

WebSocket 等のトンネル接続を管理するための構造体です。クライアントとバックエンド（origin）間の双方向通信を仲介する際のコンテキストを保持します。

シグネチャまたは形状

typedef struct {
    request_rec *r;
    const char *scheme;
    apr_pollset_t *pollset;
    apr_array_header_t *pfds;
    apr_interval_time_t timeout;
    struct proxy_tunnel_conn *client,
                             *origin;
    apr_size_t read_buf_size;
    int replied;
    unsigned int nohalfclose :1;
} proxy_tunnel_rec;

入力と出力

  • **入力**: ap_proxy_tunnel_create 関数により初期化されます。
  • **出力**: ap_proxy_tunnel_run 関数に渡され、トンネル処理の実行に使用されます。

前提条件と不変条件

  • r (request_rec) は有効なリクエストコンテキストを指している必要があります。
  • client および origin は、トンネルの両端となる接続情報を保持します。

アルゴリズムまたは振る舞い

1. ap_proxy_tunnel_create で構造体が生成・初期化されます。 2. ap_proxy_tunnel_run が呼び出されると、pollset を使用してクライアントとバックエンド間の I/O を監視します。 3. proxy_tunnel_forward（内部関数）を介して、データが双方向に転送されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy_connect.c および mod_proxy_http.c で使用されています。
  • **呼び出し先**: ap_proxy_tunnel_create および ap_proxy_tunnel_run がこの構造体を操作します。内部的には proxy_tunnel_forward が呼び出されます。

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • apr_pollset_t: I/O 多重化のための APR 構造体。
  • proxy_tunnel_conn: トンネルの各端点（クライアント/バックエンド）を表す構造体。

失敗モードと境界条件

  • replied フラグは、レスポンスが既に送信されたかどうかを追跡します。
  • nohalfclose ビットフィールドにより、ハーフクローズ動作の制御が可能です。
  • タイムアウト値（timeout）を超過した場合、トンネル接続は切断されます。

保守担当者向けメモ

  • replied フィールドは将来的にビットマスクへ統合される予定です。
  • 本構造体は ap_mmn.h に定義される API バージョン 20120211.99 以降で利用可能です。
