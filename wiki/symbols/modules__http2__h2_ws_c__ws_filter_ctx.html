ws_filter_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_ws.c:51

目的

HTTP/2 上での WebSocket ハンドシェイクを処理する出力フィルタ H2_C2_WS_OUT の実行状態を保持するコンテキスト構造体です。

シグネチャまたは形状

struct ws_filter_ctx {
    const char *ws_accept_base64; /* 生成された Sec-WebSocket-Accept 値 */
    int has_final_response;       /* レスポンス完了フラグ */
    int override_body;            /* ボディ上書きフラグ */
};

入力と出力

  • **入力**: フィルタ処理時に f->ctx を通じて参照されます。
  • **出力**: フィルタの各処理ステップにおいて、ハンドシェイクの検証結果やレスポンスの書き換え状態を保持し、後続のフィルタ処理に影響を与えます。

前提条件と不変条件

  • c2->pool（子接続のメモリプール）を使用して割り当てられる必要があります。
  • 接続終了時に自動的に解放されるライフサイクルを持ちます。

アルゴリズムまたは振る舞い

  • H2_C2_WS_OUT フィルタ内で、レスポンスバケットの検査および書き換え状態を追跡します。
  • ws_accept_base64 は、gen_ws_accept 関数によって生成された値が格納されます。
  • override_body がセットされた場合、フィルタは以降のボディバケットを破棄し、エラーレスポンス（502 Bad Gateway）の生成を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_ws.c:83 (初期化時)
  • modules/http2/h2_ws.c:198 (フィルタ処理関数内)
  • modules/http2/h2_ws.c:283 (フィルタコンテキストの取得時)
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、フィルタ処理関数（h2_c2_ws_filter_out 等）が本構造体のメンバを読み書きします。

関連データ構造

  • h2_c2（子接続）: 本構造体が紐付く接続コンテキスト。
  • apr_bucket: フィルタが処理するレスポンスデータ。

失敗モードと境界条件

  • Sec-WebSocket-Accept が欠落している、または不一致の場合、本構造体のフラグを用いて 502 Bad Gateway を生成し、接続を終了させる制御が行われます。

保守担当者向けメモ

  • 本構造体は H2_USE_WEBSOCKETS マクロが定義されている環境でのみ有効に機能します。
  • 実際のデータ転送（フレーム処理）ではなく、ハンドシェイクの整合性検証に特化した状態管理を行っています。
