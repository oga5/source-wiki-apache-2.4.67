compile_sed_expr

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_sed.c:206

目的

sed_expr_config 構造体に対し、指定された sed 式をコンパイルして保持する。

シグネチャまたは形状

apr_status_t compile_sed_expr(sed_expr_config *sed_cfg, cmd_parms *cmd, const char *expr)

入力と出力

  • **入力**:
  • sed_cfg: コンパイル済みコマンドを格納する設定構造体。
  • cmd: 設定コマンドのパラメータ（メモリプール cmd->pool を使用）。
  • expr: コンパイル対象の sed 式文字列。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • sed_cfg->sed_cmds が未初期化の場合、本関数内で sed_commands_t のメモリ確保と初期化が行われる。
  • コンパイルに失敗した場合、sed_cfg->sed_cmds は破棄され、NULL にリセットされる。

アルゴリズムまたは振る舞い

1. sed_cfg->sed_cmds が存在しない場合、apr_pcalloc でメモリを確保し、sed_init_commands を呼び出して初期化する。初期化失敗時は即座に終了する。 2. sed_compile_string を呼び出し、expr をコンパイルして sed_cfg->sed_cmds に追加する。 3. コンパイルが失敗した場合は sed_destroy_commands でリソースを解放し、sed_cfg->sed_cmds を NULL に戻してエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_sed.c:495（設定ディレクティブ解析時）。
  • **呼び出し先**:
  • apr_pcalloc
  • sed_init_commands
  • sed_compile_string
  • sed_destroy_commands
  • sed_compile_errf（コールバックとして渡される）

関連データ構造

  • sed_expr_config: 設定を保持する構造体。
  • sed_commands_t: libsed のコンパイル済みコマンドセット。
  • cmd_parms: Apache の設定コマンドパラメータ。

失敗モードと境界条件

  • sed_init_commands または sed_compile_string が APR_SUCCESS 以外を返した場合、関数はエラーを伝播させる。
  • メモリ不足や不正な sed 式が指定された場合、コンパイルは失敗し、設定構造体はクリーンアップされる。

保守担当者向けメモ

  • 本関数は libsed のライフサイクル管理を担う。sed_cfg->sed_cmds が NULL の場合にのみ初期化を行うため、複数回の呼び出しで式が累積される設計となっている。
  • エラー発生時に sed_cfg->sed_cmds を NULL に戻すことで、不整合な状態での後続処理を防止している。
