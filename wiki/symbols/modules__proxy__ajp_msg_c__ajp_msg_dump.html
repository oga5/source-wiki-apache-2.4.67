ajp_msg_dump

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_msg.c:42

目的

ajp_msg_t 構造体が保持するバイナリメッセージの内容を、デバッグ用に人間が判読可能な16進数およびASCII形式の文字列へ変換します。

シグネチャまたは形状

apr_status_t ajp_msg_dump(apr_pool_t *pool, ajp_msg_t *msg, char *err,
                          apr_size_t count, char **buf)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • msg: ダンプ対象の ajp_msg_t 構造体。
  • err: 出力文字列の先頭に付与するエラーメッセージまたはラベル。
  • count: ダンプする最大バイト数。
  • **出力**:
  • buf: 生成された文字列を格納するバッファへのポインタ。
  • 戻り値: 成功時は APR_SUCCESS、メモリ不足時は APR_ENOMEM を返します。

前提条件と不変条件

  • msg は有効な ajp_msg_t 構造体を指している必要があります。
  • buf は呼び出し元で確保されたポインタの格納先です。
  • pool は apr_palloc によるメモリ確保が可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. msg->len と count を比較し、ダンプするバイト数を決定します。 2. ヘッダー情報（pos, len, max_size）とデータ本体を表示するために必要なバッファサイズを計算し、apr_palloc でメモリを確保します。 3. メッセージのメタデータを文字列としてバッファに書き込みます。 4. データを16バイト単位でループ処理し、以下の形式で出力します：

  • オフセット（16進数）
  • 16進数表現（2桁ずつスペース区切り）
  • ASCII表現（表示可能文字はそのまま、それ以外は . に置換）

    • 5. 最後に文字列をヌル終端し、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/ajp_msg.c:129 での呼び出しが確認されています。
  • **呼び出し先**: apr_palloc, strlen, apr_snprintf などのAPRユーティリティ関数。

関連データ構造

  • ajp_msg_t: AJPメッセージを保持する構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • apr_palloc がメモリ確保に失敗した場合、APR_ENOMEM を返します。
  • ダンプ処理中にバッファの残り容量が不足した場合、処理を中断し APR_ENOMEM を返します。
  • count がメッセージ全体の長さより大きい場合、メッセージの全内容がダンプされます。

保守担当者向けメモ

  • 本関数は hex_table という外部の静的テーブルに依存しています。
  • 出力フォーマットの定数（AJP_MSG_DUMP_LINE_LENGTH 等）は ajp_msg.c 内で定義されています。
