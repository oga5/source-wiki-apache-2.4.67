md_acme_init

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme.c:164

目的

ACME モジュールの初期化を行い、製品名（base）の設定および必要に応じて暗号化ライブラリの初期化を実行します。

シグネチャまたは形状

apr_status_t md_acme_init(apr_pool_t *p, const char *base, int init_ssl)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • base: 製品名を示す文字列。
  • init_ssl: SSL/暗号化の初期化を行うかを示すフラグ（非ゼロで実行）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS を返します。

前提条件と不変条件

  • base は静的変数 base_product に代入されます。
  • init_ssl が真の場合、内部で md_crypt_init(p) が呼び出されます。

アルゴリズムまたは振る舞い

1. 引数 base をモジュール内の静的変数 base_product に格納します。 2. init_ssl が真であれば md_crypt_init(p) を実行し、その戻り値を返します。 3. init_ssl が偽であれば、何もせず APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c:1561 にて呼び出されています。
  • **呼び出し先**: md_crypt_init（init_ssl が真の場合）。

関連データ構造

  • apr_pool_t: Apache Portable Runtime のメモリプール。

失敗モードと境界条件

  • init_ssl が真の場合、md_crypt_init の実行結果（エラーコード）がそのまま返されます。

保守担当者向けメモ

  • 本関数はモジュール全体の初期化フェーズで一度だけ呼び出されることを想定しています。
  • base_product はモジュール内でグローバルな製品識別子として利用されます。
