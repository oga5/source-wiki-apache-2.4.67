md_acme_authz_retrieve

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_authz.c:82

目的

ACME サーバーから指定された URL の認証リソース（Authorization）を取得し、ローカルの md_acme_authz_t オブジェクトとして初期化・構築する。

シグネチャまたは形状

apr_status_t md_acme_authz_retrieve(md_acme_t *acme, apr_pool_t *p, const char *url, 
                                    md_acme_authz_t **pauthz)

入力と出力

  • **入力**:
  • acme: ACME サーバーとの通信セッション状態を保持する md_acme_t 構造体。
  • p: メモリ割り当てに使用する apr_pool_t。
  • url: 取得対象の認証リソースを示す ACME サーバー上の URL。
  • **出力**:
  • pauthz: 成功時に構築された md_acme_authz_t 構造体へのポインタが格納される。失敗時は NULL。
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • p は有効な apr_pool_t であること。
  • url は ACME サーバーが認識する有効な認証リソースの URL であること。
  • 内部で md_acme_authz_update を呼び出すため、通信および JSON 解析の前提条件に依存する。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して md_acme_authz_t 構造体をメモリプール上に確保する。 2. 引数の url を構造体の url フィールドにコピーする。 3. md_acme_authz_update を呼び出し、サーバーから最新の認証情報をフェッチして構造体に反映させる。 4. md_acme_authz_update の戻り値が APR_SUCCESS であれば、構築したオブジェクトを pauthz にセットして返す。それ以外の場合は pauthz に NULL をセットしてエラーコードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_order.c:461
  • modules/md/md_acme_order.c:518
  • **呼び出し先**:
  • md_acme_authz_update (同モジュール内)
  • apr_pcalloc (APR)
  • apr_pstrdup (APR)

関連データ構造

  • md_acme_authz_t: 認証リソースの状態を保持する主要構造体。
  • md_acme_t: ACME サーバーとの通信セッションを管理する構造体。

失敗モードと境界条件

  • md_acme_authz_update が失敗した場合、本関数もそのエラーを伝播し、pauthz には NULL が設定される。
  • メモリ不足時は apr_pcalloc によりアボートまたはエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は md_acme_authz_update のラッパーとして機能しており、認証リソースの初期化と取得を単一の操作として提供している。
  • 認証リソースのライフサイクルは引数として渡された apr_pool_t に依存するため、呼び出し側でのプール管理に注意すること。
