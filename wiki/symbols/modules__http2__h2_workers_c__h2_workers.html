h2_workers

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_workers.c:75

目的

HTTP/2 サブシステムにおいて、セカンダリ接続（C2）を処理する専用スレッドプールを管理するための構造体です。動的なスレッド生成、アイドルスレッドの管理、および接続プロデューサーを介したタスクの割り当てを担います。

シグネチャまたは形状

struct h2_workers {
    server_rec *s;
    apr_pool_t *pool;
    apr_uint32_t max_slots;
    apr_uint32_t min_active;
    volatile apr_time_t idle_limit;
    volatile int aborted;
    volatile int shutdown;
    int dynamic;
    volatile apr_uint32_t active_slots;
    volatile apr_uint32_t idle_slots;
    apr_threadattr_t *thread_attr;
    h2_slot *slots;
    APR_RING_HEAD(h2_slots_free, h2_slot) free;
    APR_RING_HEAD(h2_slots_idle, h2_slot) idle;
    APR_RING_HEAD(h2_slots_busy, h2_slot) busy;
    APR_RING_HEAD(h2_slots_zombie, h2_slot) zombie;
    APR_RING_HEAD(ap_conn_producer_active, ap_conn_producer_t) prod_active;
    APR_RING_HEAD(ap_conn_producer_idle, ap_conn_producer_t) prod_idle;
    struct apr_thread_mutex_t *lock;
    struct apr_thread_cond_t *prod_done;
    struct apr_thread_cond_t *all_done;
};

入力と出力

  • **入力**: h2_workers_create 等の初期化関数を通じて server_rec や設定値が入力されます。
  • **出力**: 内部で管理される h2_slot（ワーカー）が、ap_conn_producer_t から接続を取得し処理を実行します。

前提条件と不変条件

  • すべてのリング操作および状態遷移は、lock メンバー（apr_thread_mutex_t）によって保護されています。
  • h2_workers は独自の apr_pool_t を所有し、スレッド生成時に各スロットへサブプールを割り当てます。

アルゴリズムまたは振る舞い

  • **スレッド管理**: min_active から max_slots の範囲でワーカーを動的に増減させます。
  • **タスクキューイング**: apr_ring を使用して、状態別（free, idle, busy, zombie）にスロットを管理します。
  • **同期**: prod_done や all_done などの条件変数を用いて、スレッドのウェイクアップやシャットダウン時の完了待ちを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_mplx.c や h2_session.c が本構造体を参照し、HTTP/2 ストリームの多重化処理に利用しています。
  • **呼び出し先**: h2_slot 構造体、ap_conn_producer_t 構造体、および apr_thread_* 関連の同期プリミティブを操作します。

関連データ構造

  • h2_slot: 個々のワーカーの状態（スレッドハンドル、条件変数等）を保持。
  • ap_conn_producer_t: 接続の供給元。fn_next や fn_done コールバックを保持。
  • apr_ring: スロットおよびプロデューサーの管理用リスト構造。

失敗モードと境界条件

  • aborted フラグが設定された場合、処理は中断されます。
  • slot_run 内の接続 ID 生成ロジックにおいて、非常に高い並列度では ID 衝突の可能性があります。

保守担当者向けメモ

  • workers_pool_cleanup は MPM シャットダウン時に呼ばれ、全スレッドの終了を待機します。
  • h2_workers はプロセス全体で共有されるインスタンスとして設計されています。
