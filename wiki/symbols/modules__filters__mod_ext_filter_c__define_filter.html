define_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ext_filter.c:202

目的

ExtFilterDefine ディレクティブを解析し、外部フィルタの定義をサーバー設定に登録する。

シグネチャまたは形状

static const char *define_filter(cmd_parms *cmd, void *dummy, const char *args)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • args: ディレクティブに続く引数文字列。
  • **出力**: 成功時は NULL、エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • フィルタ名は小文字に正規化されてハッシュテーブルに格納される。
  • フィルタ名が重複している場合はエラーを返す。
  • フィルタ定義は ef_server_t 構造体のハッシュテーブルに保持される。

アルゴリズムまたは振る舞い

1. ap_getword_white でフィルタ名を取得し、小文字に変換する。 2. 既存のフィルタ定義と重複がないか確認する。 3. ef_filter_t 構造体を割り当て、デフォルト値（OUTPUT_FILTER, AP_FTYPE_RESOURCE）を設定する。 4. 引数文字列を順次解析し、以下のパラメータを ef_filter_t に反映する:

  • preservescontentlength: コンテンツ長保持フラグ。
  • mode: input または output。
  • ftype: フィルタタイプ（数値）。
  • enableenv / disableenv: 環境変数による制御設定。
  • intype / outtype: MIME タイプ設定。
  • cmd: 外部コマンドのパスと引数（parse_cmd を呼び出し）。

    • 5. 解析完了後、mode に応じて ap_register_output_filter または ap_register_input_filter を呼び出し、フィルタを登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_ext_filter.c:344（コマンドテーブル登録時）。
  • **呼び出し先**: ap_getword_white, ap_str_tolower, apr_hash_get, apr_pcalloc, apr_hash_set, parse_cmd, ap_register_output_filter, ap_register_input_filter。

関連データ構造

  • ef_server_t: サーバー単位の設定（フィルタハッシュテーブルを保持）。
  • ef_filter_t: フィルタの定義情報。

失敗モードと境界条件

  • フィルタ名が指定されていない場合、「Filter name not found」を返す。
  • フィルタ名が既に定義されている場合、「ExtFilter ... is already defined」を返す。
  • 不明なパラメータが渡された場合、「Unexpected parameter」を返す。
  • モード指定が input または output 以外の場合、「Invalid mode」を返す。

保守担当者向けメモ

  • XXX コメントにある通り、全フィルタ間での一意性を保証する仕組みが不足している。
  • parse_cmd の呼び出しにより、コマンド引数の解析を委譲している。
  • フィルタ登録後の mode が不正な場合、ap_assert により異常終了する設計となっている。
