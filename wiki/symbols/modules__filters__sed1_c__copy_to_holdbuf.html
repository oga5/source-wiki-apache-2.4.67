copy_to_holdbuf

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:247

目的

sed_eval_t コンテキスト内のホールドバッファ（holdbuf）の内容を、指定された文字列で上書き（置換）します。

シグネチャまたは形状

apr_status_t copy_to_holdbuf(sed_eval_t *eval, const char* sz)

入力と出力

  • **入力**:
  • eval: sed_eval_t 構造体へのポインタ。
  • sz: ホールドバッファにコピーするソース文字列。
  • **出力**:
  • apr_status_t: 操作の結果（成功時は APR_SUCCESS、メモリ不足時はエラーコード）。

前提条件と不変条件

  • eval は初期化済みであること。
  • eval->holdbuf は有効なバッファ領域を指していること。

アルゴリズムまたは振る舞い

1. eval->hspend（ホールドバッファの現在位置ポインタ）を eval->holdbuf の先頭にリセットします。 2. append_to_holdbuf を呼び出し、sz の内容をホールドバッファに書き込みます。これにより、既存の内容が上書きされ、必要に応じてバッファの拡張が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed1.c:835
  • modules/filters/sed1.c:1044
  • **呼び出し先**:
  • append_to_holdbuf

関連データ構造

  • sed_eval_t: libsed の評価状態を保持する構造体。ホールドバッファの管理ポインタ（holdbuf, hspend）を含む。

失敗モードと境界条件

  • append_to_holdbuf が内部で呼び出す grow_buffer が失敗した場合（メモリ制限 MAX_BUF_SIZE 超過など）、そのエラーコードがそのまま返されます。

保守担当者向けメモ

  • 本関数はホールドバッファの「置換」操作を担います。バッファの再確保が発生した場合、eval->holdbuf のベースアドレスが変更される可能性があるため、ポインタの直接参照には注意してください。
