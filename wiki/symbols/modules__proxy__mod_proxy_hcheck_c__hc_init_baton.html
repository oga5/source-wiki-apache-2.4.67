hc_init_baton

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_hcheck.c:619

目的

ヘルスチェック実行用バトン（baton_t）の初期化を行い、監視対象ワーカーの接続状態を準備する。

シグネチャまたは形状

static apr_status_t hc_init_baton(baton_t *baton)

入力と出力

  • **入力**: baton_t *baton（ヘルスチェックのコンテキストと対象ワーカー情報を保持）
  • **出力**: apr_status_t（成功時は APR_SUCCESS、ワーカー初期化失敗時はエラーコードを返す）

前提条件と不変条件

  • baton が有効なメモリ領域を指していること。
  • baton->ctx および baton->worker が初期化済みであること。

アルゴリズムまたは振る舞い

1. **ワーカーのブートストラップ**: worker->cp が未初期化の場合、ap_proxy_initialize_worker を呼び出してワーカーの接続プール等のローカルデータを初期化する。 2. **ヘルスチェック用ワーカーの取得**: hc_get_hcworker を呼び出し、チェック実行用のワーカーインスタンスを baton->hc に格納する。 3. **アドレス解決の試行**: ワーカーのアドレスが再利用可能（is_address_reusable）な場合、hc_get_backend を用いて接続を一度確立し、直後に ap_proxy_release_connection で解放することで、アドレス解決を先行して行う。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_hcheck.c:1061
  • **呼び出し先**:
  • ap_proxy_initialize_worker
  • hc_get_hcworker
  • hc_get_backend
  • ap_proxy_release_connection

関連データ構造

  • baton_t: ヘルスチェックの実行状態を保持する構造体。
  • proxy_worker: プロキシ対象のバックエンドサーバー定義。
  • sctx_t: サーバー単位のモジュール設定。

失敗モードと境界条件

  • ap_proxy_initialize_worker が失敗した場合、エラーログを出力し、そのステータスをそのまま戻り値として返す。
  • hc_get_backend が失敗した場合、APR_EGENERAL を返す。

保守担当者向けメモ

  • 本関数は mod_watchdog のコンテキストで実行されるため、リクエスト処理時とは異なりワーカーのローカルデータが初期化されていない前提で動作する。
  • once フラグにより、ワーカーの初期化処理は初回のみ実行されるよう制御されている。
