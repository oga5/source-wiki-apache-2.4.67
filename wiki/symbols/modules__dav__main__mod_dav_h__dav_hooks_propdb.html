dav_hooks_propdb

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:267

目的

WebDAV プロパティデータベース（propdb）に対する操作を抽象化し、リポジトリ非依存のプロパティ管理インターフェースを提供します。

シグネチャまたは形状

ソースコード上では不完全型（opaque struct）として定義されています。具体的な関数ポインタのメンバ構成は、このヘッダーファイル内の他のフック構造体と同様に、プロバイダー実装によって定義されます。

入力と出力

  • 入力: 主に request_rec を受け取り、プロパティ操作に必要なコンテキストを生成または利用します。
  • 出力: 操作の結果として dav_error 構造体（またはそのポインタ）を返し、成功または失敗の状態を伝達します。

前提条件と不変条件

  • mod_dav のプロバイダーモデルに従い、適切なリクエストプール（apr_pool_t）の生存期間内で使用される必要があります。
  • プロバイダーが実装を提供しない場合、関連するフックは NULL となる可能性があります。

アルゴリズムまたは振る舞い

WebDAV のプロパティ処理（PROPFIND や PROPPATCH 等）において、リポジトリ固有のプロパティデータベースへアクセスするための抽象レイヤーとして機能します。プロパティの検証、実行、コミット、ロールバックといったトランザクション的な処理フローをサポートします。

呼び出し元と呼び出し先

  • 呼び出し元: mod_dav.c 内の dav_get_propdb_hooks を介して、リクエスト処理中に取得されます。
  • 呼び出し先: modules/dav/fs/repos.h で定義される dav_fs_get_propdb_hooks など、具体的なファイルシステムプロバイダーの実装がこの構造体を返します。

関連データ構造

  • dav_provider: プロパティデータベースフックを保持する上位構造体。
  • dav_prop_ctx: プロパティ変更の 3 フェーズ処理（validate/exec/commit/rollback）に使用されるコンテキスト。
  • dav_hooks_db: 本構造体の別名として定義されています（modules/dav/main/mod_dav.h:276）。

失敗モードと境界条件

  • プロバイダーが未定義（NULL）の場合、呼び出し元で適切なエラーハンドリングが必要です。
  • 各フック関数は dav_error を返すことで、データベース操作時のエラー（接続失敗や権限不足など）を通知します。

保守担当者向けメモ

  • 本構造体は mod_dav のコア API の一部です。変更を加える際は、mod_dav.c および各プロバイダー実装（mod_dav_fs 等）への影響を考慮してください。
  • dav_hooks_db とのエイリアス関係があるため、定義の整合性に注意してください。
