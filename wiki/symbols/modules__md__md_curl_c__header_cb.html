header_cb

定義

  • 種別: function
  • 定義箇所: modules/md/md_curl.c:130

目的

HTTP レスポンスヘッダーを解析し、md_http_response_t 構造体内のテーブルへ格納するコールバック関数です。

シグネチャまたは形状

size_t header_cb(void *buffer, size_t elen, size_t nmemb, void *baton)

入力と出力

  • **入力**:
  • buffer: 受信したヘッダー行を含むデータバッファ。
  • elen, nmemb: データサイズ（elen * nmemb が合計バイト数）。
  • baton: md_curl_internals_t へのポインタ。
  • **出力**:
  • 戻り値: 処理したバイト数（elen * nmemb）。

前提条件と不変条件

  • baton は md_curl_internals_t 型としてキャスト可能であること。
  • internals->response および res->req->pool が有効であること。

アルゴリズムまたは振る舞い

1. 受信した行末の \r および \n を除去します。 2. : を区切り文字としてヘッダー名と値を分離します。 3. : 以降の先頭の空白文字をスキップします。 4. apr_pstrndup を使用して、リクエストのメモリプール上にヘッダー名と値を複製します。 5. apr_table_add を使用し、解析したキーと値を res->headers テーブルに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: libcurl（CURLOPT_HEADERFUNCTION として設定）。
  • **呼び出し先**:
  • apr_pstrndup (APR)
  • apr_table_add (APR)

関連データ構造

  • md_curl_internals_t: コンテキスト保持用。
  • md_http_response_t: ヘッダー格納先。
  • apr_table_t: HTTP ヘッダーの保持に使用。

失敗モードと境界条件

  • : が含まれない行は無視され、テーブルには追加されません。
  • 空のヘッダー値は空文字列として扱われます。
  • メモリ割り当て失敗時の挙動は apr_pstrndup に依存します。

保守担当者向けメモ

  • 本関数は libcurl から複数回呼び出される可能性があるため、状態の保持は baton を介して行われます。
  • apr_table_add を使用しているため、同じ名前のヘッダーが複数存在する場合はテーブル内に重複して格納されます。
