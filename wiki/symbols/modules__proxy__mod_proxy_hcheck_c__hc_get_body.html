hc_get_body

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_hcheck.c:1262

目的

リクエストオブジェクト内に保持されているボディデータ（r->kept_body）をフラットな文字列として取得し、返却する。

シグネチャまたは形状

static const char *hc_get_body(request_rec *r)

入力と出力

  • **入力**: request_rec *r （リクエスト情報構造体）
  • **出力**: ボディデータを含む const char * 型の文字列。データが存在しない、または取得に失敗した場合は空文字列 "" を返す。

前提条件と不変条件

  • r が有効であること。
  • r->kept_body にバケットブリゲードが格納されていること。
  • 戻り値の文字列は、呼び出し元のリクエストプール（r->pool）から割り当てられる。

アルゴリズムまたは振る舞い

1. r または r->kept_body が NULL の場合、空文字列を返す。 2. apr_brigade_length を使用してボディの長さを算出する。 3. 長さが 0 の場合、空文字列を返す。 4. apr_palloc を使用して、ボディ長 + 1 バイトのメモリを確保する。 5. apr_brigade_flatten を使用して、バケットブリゲードの内容を確保したメモリ領域にコピーする。 6. 終端文字 \0 を付与し、文字列として返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_hcheck.c:1290
  • modules/proxy/mod_proxy_hcheck.c:1301
  • **呼び出し先**:
  • apr_brigade_length
  • apr_palloc
  • apr_brigade_flatten

関連データ構造

  • request_rec: Apache のリクエスト処理用構造体。
  • apr_bucket_brigade: ボディデータを保持する APR のデータ構造。

失敗モードと境界条件

  • r が NULL の場合、安全に空文字列を返す。
  • apr_brigade_length または apr_brigade_flatten が APR_SUCCESS 以外を返した場合、空文字列を返す。
  • ボディデータが空の場合、空文字列を返す。

保守担当者向けメモ

  • 本関数は r->pool を使用してメモリを割り当てるため、呼び出し元のリクエスト処理が終了した時点でメモリは自動的に解放される。
  • apr_brigade_flatten の実行時に len が更新される可能性があるため、戻り値の整合性に注意すること。
