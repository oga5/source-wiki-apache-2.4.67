h2_config_create_dir

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_config.c:237

目的

Apache HTTP Server のディレクトリ単位の設定構造体（h2_dir_config）をメモリプール上に生成し、デフォルト値で初期化します。

シグネチャまたは形状

void *h2_config_create_dir(apr_pool_t *pool, char *x)

入力と出力

  • **入力**:
  • pool: 設定構造体を割り当てるための apr_pool_t。
  • x: ディレクトリ名または識別子（char *）。
  • **出力**:
  • 初期化済みの h2_dir_config 構造体へのポインタ（void * として返却）。

前提条件と不変条件

  • pool は有効なメモリプールである必要があります。
  • 生成された構造体の各設定項目は、明示的な設定が行われるまで「未設定」を示す DEF_VAL で初期化されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して h2_dir_config サイズのメモリをゼロクリアして確保します。 2. 引数 x が NULL の場合は "unknown" という文字列を補完し、dir[<name>] という形式の識別名を生成して構造体に格納します。 3. 以下の各設定項目に DEF_VAL を代入します。

  • h2_upgrade
  • h2_push
  • early_hints
  • stream_timeout

    • 4. 初期化済みの構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c:52 にて、Apache のディレクトリ設定作成フックとして登録されています。
  • **呼び出し先**: apr_pcalloc および apr_pstrcat（APR ライブラリ関数）。

関連データ構造

  • h2_dir_config: 本関数が生成するディレクトリ設定構造体。
  • DEF_VAL: 未設定状態を表すマジックナンバー。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がプロセスを終了させる可能性があります（APR の標準的な挙動）。
  • x が NULL の場合、安全のために "unknown" という文字列が使用されます。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出されます。
  • DEF_VAL が設定されている項目は、後のマージ処理（h2_config_merge_dir）において親コンテキストの設定値で上書きされる対象となります。
