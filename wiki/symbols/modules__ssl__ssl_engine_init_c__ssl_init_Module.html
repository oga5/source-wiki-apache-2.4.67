ssl_init_Module

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_init.c:339

目的

Apache HTTP Server の起動および再起動時における、mod_ssl モジュール全体の初期化および設定の確定を行うエントリーポイント。

シグネチャまたは形状

apr_status_t ssl_init_Module(apr_pool_t *p, apr_pool_t *plog,
                             apr_pool_t *ptemp,
                             server_rec *base_server)

入力と出力

  • **入力**:
  • p: モジュールのライフサイクル用プール。
  • plog: ログ記録用プール。
  • ptemp: 一時的な初期化処理用プール。
  • base_server: サーバー設定のルート構造体。
  • **出力**:
  • apr_status_t: 成功時は OK、失敗時はエラーコードを返す。

前提条件と不変条件

  • mc (SSLModConfigRec) が有効であること (AP_DEBUG_ASSERT(mc) による検証)。
  • コンパイル時の OpenSSL ライブラリバージョンと実行時のバージョンが一致、または互換性があること。

アルゴリズムまたは振る舞い

1. **初期化準備**: プロセスIDの記録、ssl_init_ModuleKill のクリーンアップ登録、グローバル設定の固定。 2. **サーバー設定の調整**: 各バーチャルホストの設定をループし、vhost_id の生成、SSL 有効化の判定、タイムアウト値の設定を行う。 3. **サブシステム初期化**:

  • ssl_init_Engine による暗号デバイスの初期化。
  • ssl_rand_seed による PRNG のシード。
  • FIPS モードの有効化（HAVE_FIPS 定義時）。
  • ssl_mutex_init によるミューテックス初期化。
  • ssl_scache_init によるセッションキャッシュ初期化。

    • 4. **証明書と鍵の読み込み**: ssl_init_ConfigureServer を呼び出し、各サーバーの証明書・鍵をロード。 5. **整合性チェック**: ssl_init_CheckServers による設定の検証。 6. **最終適用**: ssl_run_init_server を呼び出し、各サーバーおよびプロキシの SSL コンテキストを確定。 7. **環境設定**: バージョン情報の追加、アプリケーションデータ用インデックスの初期化、DH パラメータまたは BIO メソッドの初期化。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c の ap_hook_post_config フックにより呼び出される。
  • **呼び出し先**: ssl_init_Engine, ssl_rand_seed, ssl_mutex_init, ssl_scache_init, ssl_init_ConfigureServer, ssl_init_CheckServers, ssl_run_init_server, ssl_add_version_components 等。

関連データ構造

  • SSLModConfigRec: モジュール全体のグローバル設定。
  • SSLSrvConfigRec: 各サーバーの SSL 設定。
  • server_rec: Apache のサーバー構造体。

失敗モードと境界条件

  • OpenSSL ライブラリのバージョン不一致時は警告をログ出力。
  • 設定ミス（証明書不一致など）が発生した場合、ssl_die() を経由してサーバー起動を停止する。
  • ssl_init_Engine や ssl_scache_init 等のサブシステム初期化に失敗した場合、そのエラーコードを返して終了する。

保守担当者向けメモ

  • MODSSL_USE_OPENSSL_PRE_1_1_API マクロにより、OpenSSL 1.1.0 前後の API 差異が吸収されている。
  • ptemp プールは初期化完了後に解放されるため、永続的なデータは p プールを使用すること。
  • SSLKEYLOGFILE 環境変数が設定されている場合、秘密鍵素材のログ出力が試行される。
