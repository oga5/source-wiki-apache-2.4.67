so_server_conf

定義

  • 種別: struct
  • 定義箇所: modules/core/mod_so.c:107

目的

Apache HTTP Server において、動的にロードされたモジュール（DSO）のリストを保持・管理するためのサーバー設定構造体です。

シグネチャまたは形状

typedef struct so_server_conf {
    apr_array_header_t *loaded_modules;
} so_server_conf;

入力と出力

  • **入力**: loaded_modules は apr_array_header_t 型の配列であり、ロードされた各モジュールの情報を格納します。
  • **出力**: 本構造体自体は設定コンテナとして機能し、サーバー設定の取得関数（ap_get_module_config）を通じて、ロード済みモジュールの一覧を後続の処理へ提供します。

前提条件と不変条件

  • loaded_modules は、so_sconf_create 等の初期化処理において apr_array_make 等で適切にメモリ確保されている必要があります。
  • サーバーのライフサイクル中、この構造体は pconf（設定プール）に紐付けられて管理されます。

アルゴリズムまたは振る舞い

  • サーバー設定の一部として、LoadModule ディレクティブによってロードされたモジュール情報を loaded_modules 配列に追加します。
  • サーバーの終了時には、プールに関連付けられたクリーンアップ処理を通じて、ロードされたモジュールが適切にアンロードされるための基盤となります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • so_sconf_create: 構造体のインスタンス化と初期化。
  • load_module: モジュールロード時に loaded_modules への追加を行う。
  • ap_get_module_config: サーバー設定から本構造体を取得する各関数（load_module, dump_loaded_modules 等）。
  • **呼び出し先**:
  • apr_array_header_t (APR): 配列管理に使用。

関連データ構造

  • apr_array_header_t: loaded_modules の実体。
  • module (Apache 内部構造体): loaded_modules 内に格納される各モジュールの定義。

失敗モードと境界条件

  • メモリ不足: apr_pcalloc による構造体確保時に失敗する可能性があります。
  • 重複ロード: load_module 処理において、既にロード済みのモジュールと名前が衝突する場合、ロードが拒否される制約があります。

保守担当者向けメモ

  • 本構造体は mod_so.c 内でのみ使用される内部的な設定管理用構造体です。
  • loaded_modules 配列の操作は、Apache のモジュール登録順序や依存関係に影響を与えるため、変更時は server/config.c 側のモジュール統合ロジックとの整合性に注意してください。
