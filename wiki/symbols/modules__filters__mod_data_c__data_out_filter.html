data_out_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_data.c:65

目的

HTTP レスポンスボディを RFC 2397 形式の「data URL」スキーム（data:[<mime-type>];base64,<payload>）に変換する出力フィルタです。

シグネチャまたは形状

apr_status_t data_out_filter(ap_filter_t *f, apr_bucket_brigade *bb)

入力と出力

  • **入力**: ap_filter_t *f（フィルタコンテキスト）、apr_bucket_brigade *bb（処理対象のバケットブリゲード）。
  • **出力**: apr_status_t（処理結果ステータス）。成功時は APR_SUCCESS を返します。

前提条件と不変条件

  • 初回呼び出し時に f->ctx が初期化され、data URL のヘッダー部分が生成されます。
  • サブリクエストに対してはフィルタを適用せず、そのまま後続のフィルタへパススルーします。
  • Base64 エンコードの性質上、3バイト単位の処理が必要であり、端数は ctx->overflow に保持されます。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時に data_ctx を作成し、Content-Type を text/plain に変更します。Content-Length が存在する場合は、Base64 エンコード後のサイズを再計算して更新します。 2. **変換ループ**:

  • EOS (End of Stream) バケット: overflow バッファに残った端数を処理し、EOS を下流へ渡してフィルタを解除します。
  • FLUSH バケット: 蓄積されたデータを下流へ流します。
  • メタデータバケット: 変換せずにそのまま下流へ渡します。
  • データバケット: 最大 6000 バイト単位で読み取り、Base64 エンコードを適用して ctx->bb に書き込み、ap_pass_brigade で後続へ渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_data.c:245 にて ap_register_output_filter を通じて登録されています。
  • **呼び出し先**: ap_pass_brigade を介してフィルタチェーンの後続フィルタを呼び出します。

関連データ構造

  • data_ctx: フィルタのコンテキスト。Base64 変換用の overflow バッファと出力用ブリゲード bb を保持します。

失敗モードと境界条件

  • **サブリクエスト**: フィルタを適用せずパススルーします。
  • **Content-Length**: 解析不能または APR_INT32_MAX を超える場合はヘッダーを削除します。
  • **メモリ**: APR_BUCKET_BUFF_SIZE を利用してメモリ消費を抑制しています。

保守担当者向けメモ

  • 本フィルタは EOS を基準に終了判定を行うため、サブリクエストでの動作には制限があります。
  • Base64 エンコードの端数処理（overflow）が正しく行われるよう、バケットの分割と連結のロジックを維持してください。
