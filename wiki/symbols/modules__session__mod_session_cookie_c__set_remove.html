set_remove

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_cookie.c:245

目的

Apache設定ディレクティブ SessionCookieRemove の値を解析し、セッションCookieの削除フラグを設定します。

シグネチャまたは形状

static const char *set_remove(cmd_parms *parms, void *dconf, int flag)

入力と出力

  • **入力**:
  • parms: 設定コマンドのパラメータ構造体。
  • dconf: session_cookie_dir_conf 型のディレクトリ設定構造体。
  • flag: ディレクティブで指定された真偽値（On/Off）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • dconf は session_cookie_dir_conf 構造体へのポインタとしてキャスト可能である必要があります。
  • 本関数は AP_INIT_FLAG を介して呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. 引数 dconf を session_cookie_dir_conf 型のポインタにキャストします。 2. flag の値を conf->remove に格納します。 3. 設定が明示的に行われたことを示すため、conf->remove_set を 1 に設定します。 4. NULL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apacheの設定解析フェーズにおいて、AP_INIT_FLAG("SessionCookieRemove", ...) を通じて設定ファイルから呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • session_cookie_dir_conf: mod_session_cookie のディレクトリ単位の設定を保持する構造体。本関数はこの構造体の remove および remove_set フィールドを更新します。

失敗モードと境界条件

  • 設定ファイル内で SessionCookieRemove が正しく指定されていない場合、Apacheの構成解析プロセスによりエラーが報告されます。関数内部でのエラーハンドリングは行われていません。

保守担当者向けメモ

  • 本関数は SessionCookieRemove ディレクティブ専用のハンドラです。設定の継承やマージについては merge_session_cookie_dir_config が担当するため、本関数は現在のコンテキストの設定更新のみに集中しています。
