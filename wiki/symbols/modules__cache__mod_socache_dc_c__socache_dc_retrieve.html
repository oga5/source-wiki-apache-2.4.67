socache_dc_retrieve

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dc.c:115

目的

分散キャッシュ（distcache）から、指定された識別子（ID）に対応するセッションデータを取得し、呼び出し元が提供したバッファへ格納します。

シグネチャまたは形状

apr_status_t socache_dc_retrieve(ap_socache_instance_t *ctx, server_rec *s,
                                 const unsigned char *id, unsigned int idlen,
                                 unsigned char *dest, unsigned int *destlen,
                                 apr_pool_t *p);

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンス（distcache コンテキストを含む）。
  • s: サーバーレコード（ログ出力用）。
  • id: 検索対象のセッション ID。
  • idlen: id の長さ。
  • dest: データを格納するバッファ。
  • destlen: dest バッファのサイズ（入力時）、および取得したデータの実際の長さ（出力時）。
  • p: メモリプール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時は APR_NOTFOUND（キャッシュミス）または APR_ENOSPC（バッファ不足）。

前提条件と不変条件

  • ctx および ctx->dc が初期化済みであること。
  • destlen が指す値は、dest バッファの有効なサイズであること。

アルゴリズムまたは振る舞い

1. DC_CTX_get_session を呼び出し、分散キャッシュからデータを取得します。 2. 取得に失敗した場合、APR_NOTFOUND を返し、デバッグログを出力します。 3. 取得したデータ長（data_len）が提供されたバッファサイズ（*destlen）を超えている場合、APR_ENOSPC を返し、エラーログを出力します。 4. 成功した場合、*destlen に実際のデータ長を代入し、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dc.c:180 にて、ap_socache_provider_t 構造体の一部として参照されています。
  • **呼び出し先**: distcache ライブラリの DC_CTX_get_session を呼び出します。

関連データ構造

  • ap_socache_instance_t: distcache コンテキスト（DC_CTX）を保持する構造体。

失敗モードと境界条件

  • **キャッシュミス**: DC_CTX_get_session が失敗した場合、APR_NOTFOUND を返します。
  • **バッファオーバーフロー**: 取得データが dest バッファに収まらない場合、APR_ENOSPC を返します。

保守担当者向けメモ

  • 本関数は distcache API を直接ラップしています。
  • ログ出力には APLOGNO マクロが使用されており、エラーコード 00742（MISS）、00743（OVERFLOW）、00744（HIT）が割り当てられています。
