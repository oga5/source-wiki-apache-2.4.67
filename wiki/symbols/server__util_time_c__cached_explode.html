cached_explode

定義

  • 種別: function
  • 定義箇所: server/util_time.c:55

目的

時刻分解処理（apr_time_t から apr_time_exp_t への変換）をリングバッファ形式のキャッシュを用いて高速化し、システムコール呼び出しを削減する。

シグネチャまたは形状

apr_status_t cached_explode(apr_time_exp_t *xt, apr_time_t t,
                            struct exploded_time_cache_element *cache,
                            int use_gmt);

入力と出力

  • **入力**:
  • xt: 分解後の時刻を格納する構造体へのポインタ。
  • t: 変換対象の時刻（apr_time_t）。
  • cache: 時刻キャッシュを保持するリングバッファ配列。
  • use_gmt: GMT変換を行う場合は非ゼロ、ローカルタイムの場合はゼロを指定。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS を返し、xt に結果が格納される。失敗時は APR のエラーコードを返す。

前提条件と不変条件

  • キャッシュは TIME_CACHE_MASK を用いたリングバッファとして実装されている。
  • キャッシュの整合性は、t（タイムスタンプ）と t_validate（検証用タイムスタンプ）の双方が一致することで保証される。
  • AP_TIME_RECENT_THRESHOLD（デフォルト15秒）以内の時刻に対してのみキャッシュが有効。

アルゴリズムまたは振る舞い

1. **キャッシュ探索**: 入力時刻からリングバッファのインデックスを計算し、該当要素を取得する。 2. **キャッシュヒット判定**: キャッシュ要素のタイムスタンプが現在の秒数以上であれば、スナップショットを採取する。 3. **整合性検証**: スナップショットの t と t_validate が入力時刻と一致するか確認する。

  • 一致する場合: キャッシュされた xt を使用する。
  • 不一致の場合: キャッシュを無視し、apr_time_exp_gmt または apr_time_exp_lt を直接呼び出す。

    • 4. **キャッシュミス/更新**: キャッシュが古い場合、標準の APR 関数で時刻を分解し、結果をキャッシュに書き込む。書き込み時は t を最後に更新することで、読み取り側の不整合検知を可能にする。 5. **最終処理**: xt->tm_usec にマイクロ秒部分を反映させて終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_time.c:149 (ap_explode_recent_localtime から)
  • server/util_time.c:155 (ap_explode_recent_gmt から)
  • **呼び出し先**:
  • apr_time_exp_gmt / apr_time_exp_lt (APR ライブラリ関数)

関連データ構造

  • exploded_time_cache_element: 時刻分解結果と整合性検証用のタイムスタンプを保持する構造体。

失敗モードと境界条件

  • **競合**: マルチスレッド環境で書き込み中に読み取りが発生した場合、整合性チェックで不整合が検知され、フォールバックとして標準の APR 関数が呼び出される。
  • **範囲外**: AP_TIME_RECENT_THRESHOLD を超える古い時刻が入力された場合、キャッシュは利用されず直接変換が行われる。

保守担当者向けメモ

  • 本関数はロックフリー設計であるため、スレッド間の競合を「検知して再計算する」ことで解決している。
  • t_validate は、書き込み完了を保証するためのマーカーとして機能する。この順序（t 更新 -> xt コピー -> t_validate 更新）を変更してはならない。
