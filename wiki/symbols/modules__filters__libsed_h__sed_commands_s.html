sed_commands_s

定義

  • 種別: struct
  • 定義箇所: modules/filters/libsed.h:68

目的

sed スクリプトのコンパイル結果を保持し、実行環境へ引き渡すためのメインコンテナ構造体です。

シグネチャまたは形状

struct sed_commands_s {
    sed_err_fn_t *errfn;
    void         *data;
    apr_size_t   lsize;
    char         *linebuf;
    char         *lbend;
    const char   *saveq;
    char         *cp;
    char         *lastre;
    char         *respace;
    char         sseof;
    char         *reend;
    const char   *earg;
    int          eflag;
    int          gflag;
    int          nflag;
    apr_int64_t  tlno[SED_NLINES];
    int          nlno;
    int          depth;
    char         *fname[NWFILES];
    int          nfiles;
    sed_label_t  ltab[SED_LABSIZE];
    sed_label_t  *labtab;
    sed_label_t  *lab;
    sed_label_t  *labend;
    sed_reptr_t  **cmpend[SED_DEPTH];
    sed_reptr_t  *ptrspace;
    sed_reptr_t  *ptrend;
    sed_reptr_t  *rep;
    int          nrep;
    apr_pool_t   *pool;
    int          canbefinal;
};

入力と出力

  • **入力**: コンパイル対象のスクリプト情報、エラーハンドラ (errfn)、およびメモリ管理用の apr_pool_t。
  • **出力**: コンパイルされたコマンドリスト (rep)、ラベルテーブル (ltab)、および実行に必要なバッファ状態。

前提条件と不変条件

  • すべての動的メモリは pool メンバに関連付けられた apr_pool_t によって管理されます。
  • SED_NLINES, SED_DEPTH, SED_LABSIZE などの定数により、スクリプトの複雑さやネストに上限が設けられています。

アルゴリズムまたは振る舞い

1. **コンパイルフェーズ**: sed_compile_string 等により、スクリプトが解析され、rep (連結リスト) にコマンドが格納されます。 2. **状態保持**: コンパイルされたコマンド群、ラベル、および実行時に必要なバッファ (linebuf) やフラグ (gflag, nflag 等) を保持します。 3. **ライフサイクル**: sed_destroy_commands が呼び出されるまで、コンパイルされたスクリプトの実行状態を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/libsed.h 内で typedef struct sed_commands_s sed_commands_t; として定義されており、sed_commands_t を介して利用されます。
  • **呼び出し先**: 構造体内部で sed_err_fn_t (関数ポインタ)、sed_label_t、sed_reptr_t を参照しています。

関連データ構造

  • sed_commands_t: 本構造体の型定義。
  • sed_reptr_t: コンパイルされた個々の sed コマンドを保持するノード。
  • sed_label_t: 分岐先ラベルを管理する構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • SED_NLINES, SED_DEPTH, SED_LABSIZE の上限を超えた場合、解析が失敗する可能性があります。
  • pool が破棄された後に本構造体へアクセスすると未定義動作となります。

保守担当者向けメモ

  • 本構造体は sed0.c（コンパイル）と sed1.c（実行）の間で共有されます。
  • メモリ所有権は pool にあるため、手動での free は行わず、必ず sed_destroy_commands を使用してください。
  • 正規表現エンジンの詳細は regexp.h に依存しており、本構造体には含まれていません。
