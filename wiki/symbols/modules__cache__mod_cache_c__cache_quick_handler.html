cache_quick_handler

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:83

目的

Apache HTTP Server のクイックハンドラフェーズにおいて、キャッシュのヒット判定、レスポンスの提供、またはキャッシュ保存のためのフィルタ挿入を行うメインエントリポイント。

シグネチャまたは形状

static int cache_quick_handler(request_rec *r, int lookup)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • lookup: キャッシュの存在確認のみを行うか（真）否か（偽）を示すフラグ。
  • **出力**:
  • OK: キャッシュヒット時、またはルックアップ成功時。
  • DECLINED: キャッシュ対象外、キャッシュミス、または処理を後続のハンドラに委譲する場合。
  • HTTP_* ステータスコード: エラー発生時。

前提条件と不変条件

  • cache_module の設定において quick が有効であること。
  • リクエストが Authorization ヘッダーを含まないこと（認証済みリクエストはキャッシュ対象外）。
  • メモリ割り当てはすべて r->pool に紐付いていること。

アルゴリズムまたは振る舞い

1. **初期化**: キャッシュ設定を確認し、cache_request_rec を初期化。 2. **メソッド判定**:

  • PUT/POST/DELETE: cache_invalidate_filter を挿入し DECLINED を返す。
  • GET: キャッシュ検索へ進む。
  • その他: DECLINED を返す。

    • 3. **キャッシュ検索 (cache_select)**:

  • **ミス**: cache_try_lock で排他制御を試み、成功すれば CACHE_SAVE および CACHE_REMOVE_URL フィルタを挿入。
  • **ヒット**: cache_run_cache_status でヒットを記録し、条件付きリクエスト（If-Modified-Since 等）を検証。

    • 4. **レスポンス提供**:

  • lookup が偽の場合、CACHE_OUT フィルタを挿入し、既存のフィルタスタックを整理してレスポンスを送信。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の quick_handler フック（ap_hook_quick_handler により登録）。
  • **呼び出し先**:
  • cache_get_providers, cache_select, cache_try_lock, cache_run_cache_status
  • ap_add_output_filter_handle (各種フィルタの挿入)
  • ap_pass_brigade_fchk (レスポンス送信)

関連データ構造

  • cache_server_conf: サーバー単位のキャッシュ設定。
  • cache_request_rec: リクエストごとのキャッシュ状態管理。
  • cache_provider_list: 利用可能なキャッシュストレージプロバイダーのリスト。

失敗モードと境界条件

  • **ロック競合**: cache_try_lock が失敗した場合、キャッシュ保存をスキップし、バックエンドへリクエストを透過させる。
  • **ルックアップモード**: lookup が真の場合、コンテンツの提供は行わず、キャッシュの有無のみを判定して終了する。
  • **フィルタ順序**: CACHE_OUT フィルタ挿入時に、それ以前のフィルタをすべて削除し、キャッシュからのレスポンスが正しく処理されるようスタックを再構築する。

保守担当者向けメモ

  • 本関数は mod_cache の最前線で動作するため、パフォーマンスへの影響が大きい。
  • r->main をチェックしてサブクエストか否かを判別し、適切なフィルタハンドルを選択している点に注意。
  • キャッシュキー生成やストレージの物理的な読み書きは、cache_storage.c および各プロバイダーモジュールに委譲されている。
