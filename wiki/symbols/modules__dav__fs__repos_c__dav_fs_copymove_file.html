dav_fs_copymove_file

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/repos.c:327

目的

ローカルファイルシステム上のファイルをコピーまたは移動する。移動操作の場合は、コピー完了後にソースファイルを削除する。

シグネチャまたは形状

static dav_error * dav_fs_copymove_file(
    int is_move,
    apr_pool_t * p,
    const char *src,
    const char *dst,
    const apr_finfo_t *src_finfo,
    const apr_finfo_t *dst_finfo,
    dav_buffer *pbuf)

入力と出力

  • **入力**:
  • is_move: 移動操作であれば非ゼロ、コピーであればゼロ。
  • p: メモリ割り当て用プール。
  • src, dst: ソースおよび宛先のファイルパス。
  • src_finfo, dst_finfo: ファイル情報（NULL 可）。
  • pbuf: バッファ構造体（NULL の場合は内部で初期化）。
  • **出力**: 成功時は NULL、失敗時は dav_error 構造体へのポインタを返す。

前提条件と不変条件

  • pbuf が NULL の場合、関数内で一時的な dav_buffer が使用される。
  • src_finfo が存在し、実行権限（APR_UEXECUTE）が設定されている場合、宛先ファイルにも同様の権限が適用される。
  • 読み込みおよび書き込み操作は DAV_FS_COPY_BLOCKSIZE 単位で行われる。

アルゴリズムまたは振る舞い

1. **権限設定**: ソースファイルの権限を確認し、宛先ファイル作成時の権限を決定する。 2. **ファイルオープン**: ソースを読み込み専用、宛先を書き込み・作成・切り詰めモードで開く。 3. **データ転送**: apr_file_read と apr_file_write_full を使用し、ブロック単位でデータをコピーする。 4. **クリーンアップ**: 読み書き完了後に両ファイルを閉じる。 5. **移動処理**: is_move が真の場合、apr_file_remove でソースファイルを削除する。削除失敗時は、整合性を保つために宛先ファイルを削除しようと試みる場合がある。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/repos.c 内の複数箇所（行 537, 548, 1190, 1273）から呼び出されている。
  • **呼び出し先**: apr_file_open, apr_file_read, apr_file_write_full, apr_file_close, apr_file_remove, apr_file_perms_set, dav_new_error, dav_set_bufsize 等の APR および mod_dav ユーティリティ関数。

関連データ構造

  • dav_buffer: データ転送用バッファ。
  • apr_finfo_t: ファイル属性情報。
  • dav_error: エラー報告用構造体。

失敗モードと境界条件

  • **ファイル操作失敗**: 読み込み、書き込み、または削除に失敗した場合、適切な HTTP エラーコード（主に HTTP_INTERNAL_SERVER_ERROR）を含む dav_error を返す。
  • **不整合状態**: 読み書き中のエラー発生時や、移動後のソース削除失敗時に、ファイルシステムが不整合な状態になる可能性がある旨がコメントで指摘されている。
  • **権限設定失敗**: 宛先ファイルの権限変更に失敗した場合、エラーを返す。

保守担当者向けメモ

  • 本関数はファイルシステムレベルのコピー/移動を担当する。プロパティ（DBM）の移動は別途 dav_fs_copymoveset 等が担うため、呼び出し側での整合性管理に注意すること。
  • MAP_IO2HTTP マクロを使用して APR のエラーコードを HTTP ステータスに変換している。
  • is_move 実行時のソース削除失敗時、宛先ファイルを削除して整合性を保とうとするロジックが含まれている。
