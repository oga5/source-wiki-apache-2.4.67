register_hooks

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_status.c:1036

目的

mod_status モジュールが提供するハンドラおよび設定フックを Apache HTTP Server のフックシステムに登録する。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（フック登録に使用されるメモリプール）
  • **出力**: なし（void）

前提条件と不変条件

  • Apache のモジュール読み込みプロセスにおいて、サーバー起動時に呼び出されることを前提とする。
  • HAVE_TIMES マクロの定義有無により、status_child_init の登録が条件分岐する。

アルゴリズムまたは振る舞い

以下の順序で Apache のフック関数を呼び出し、モジュールの各機能を登録する： 1. ap_hook_handler: status_handler を APR_HOOK_MIDDLE で登録。 2. ap_hook_pre_config: status_pre_config を APR_HOOK_LAST で登録。 3. ap_hook_post_config: status_init を APR_HOOK_MIDDLE で登録。 4. ap_hook_child_init: HAVE_TIMES が定義されている場合のみ、status_child_init を APR_HOOK_MIDDLE で登録。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュールロード機構（直接的な呼び出し元はソースコード上では明示されていないが、http_config.h の register_hooks 型定義に従い、サーバー起動時に実行される）。
  • **呼び出し先**:
  • ap_hook_handler
  • ap_hook_pre_config
  • ap_hook_post_config
  • ap_hook_child_init
  • status_handler
  • status_pre_config
  • status_init
  • status_child_init（条件付き）

関連データ構造

  • apr_pool_t: リソース管理用メモリプール。

失敗モードと境界条件

  • 登録処理自体は戻り値を返さないため、失敗時の挙動は Apache のフック登録機構に依存する。
  • HAVE_TIMES が未定義の環境では、status_child_init は登録されない。

保守担当者向けメモ

  • 本関数は mod_status のエントリーポイントである。フックの優先順位（APR_HOOK_MIDDLE 等）を変更する場合は、他のモジュールとの実行順序への影響を考慮すること。
