worker_check_config

定義

  • 種別: function
  • 定義箇所: server/mpm/worker/worker.c:2120

目的

Apache HTTP Server の worker MPM における設定値の妥当性を検証し、制約条件に基づいて値を自動調整する。

シグネチャまたは形状

static int worker_check_config(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • p, plog, ptemp: APR プール。
  • s: サーバー設定レコード。
  • **出力**:
  • 常に OK を返す。
  • **副作用**:
  • グローバルな設定変数（server_limit, thread_limit, threads_per_child, max_workers, ap_daemons_limit, ap_daemons_to_start, min_spare_threads）を検証し、必要に応じて修正する。
  • 設定値が範囲外の場合、エラーログを出力する。

前提条件と不変条件

  • ServerLimit および ThreadLimit は、再起動を跨いで変更することはできない。
  • MaxRequestWorkers は ThreadsPerChild の整数倍である必要がある。
  • MaxRequestWorkers は少なくとも ThreadsPerChild 以上である必要がある。

アルゴリズムまたは振る舞い

1. **起動判定**: retained->mpm->module_loads を確認し、初回起動時か否かを判定する。 2. **制限値の検証**:

  • server_limit と thread_limit がコンパイル時の最大値（MAX_SERVER_LIMIT, MAX_THREAD_LIMIT）を超えていないか、また 1 未満でないかを確認し、範囲内に収める。
  • 再起動時に server_limit や thread_limit が初回起動時の値と異なる場合、警告を出力して元の値に強制的に戻す。

    • 3. **スレッド・プロセス数の調整**:

  • threads_per_child が thread_limit を超えないよう調整する。
  • max_workers が threads_per_child 未満の場合、threads_per_child まで引き上げる。
  • max_workers を threads_per_child の倍数に切り捨てる。
  • ap_daemons_limit を計算し、server_limit を超える場合は制限する。

    • 4. **最小値の保証**: ap_daemons_to_start および min_spare_threads が 1 未満の場合、1 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_check_config によりフックとして登録されている（server/mpm/worker/worker.c:2348）。
  • **呼び出し先**: ap_log_error を呼び出し、設定の不整合をログに記録する。

関連データ構造

  • retained: モジュール再読み込み時にも保持されるデータ構造。first_server_limit および first_thread_limit を保持する。
  • server_rec: サーバー設定を保持する構造体。

失敗モードと境界条件

  • 設定値がコンパイル時の制限（MAX_SERVER_LIMIT, MAX_THREAD_LIMIT）を超える場合、警告ログを出力し、上限値に切り詰める。
  • 設定値が 1 未満の場合、警告ログを出力し、1 に引き上げる。
  • 再起動時に変更不可なパラメータ（ServerLimit, ThreadLimit）が変更された場合、変更を無視して元の値を維持する。

保守担当者向けメモ

  • 本関数は設定の整合性を保つためのガードレールとして機能する。
  • startup フラグにより、サーバー起動時のコンソール出力と、再起動時のエラーログ出力を切り分けている。
  • ap_daemons_limit の計算ロジックは max_workers と threads_per_child の関係に強く依存するため、変更時は注意が必要である。
