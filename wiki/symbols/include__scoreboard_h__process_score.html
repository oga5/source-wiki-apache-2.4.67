process_score

定義

  • 種別: struct
  • 定義箇所: include/scoreboard.h:136

目的

Apache HTTP Server におけるプロセス単位の管理情報を保持するための構造体です。MPM（Multi-Processing Module）がプロセスを管理する際の「状態のレジストリ」の一部として機能します。

シグネチャまたは形状

struct process_score {
    /* 内部フィールドは実装依存のため、ヘッダー定義を参照 */
};

※ソース excerpt には構造体のメンバ詳細は含まれていません。

入力と出力

  • 入力: プロセス単位の稼働状態、PID、世代（Generation）、非同期接続情報、リスナーバケット、I/O 待機状態などの更新。
  • 出力: ap_get_scoreboard_process(int x) を通じて、特定のプロセススロットに対応する process_score 構造体へのポインタが返されます。

前提条件と不変条件

  • 共有メモリ（apr_shm）上に配置されるため、アクセスには適切な同期制御が必要です。
  • worker_score と同様に、サーバーのライフサイクル全体で共有される状態管理の基盤となります。

アルゴリズムまたは振る舞い

  • MPM がプロセスを生成・管理する際、この構造体を使用してプロセスの生存状況や負荷状況を追跡します。
  • 非同期 MPM（event など）では、接続数や書き込み完了状態などの追跡に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照例）:
  • server/mpm/event/event.c: ap_scoreboard_image->parent[slot] を通じてアクセス。
  • modules/generators/mod_status.c: 統計情報の表示のために参照。
  • modules/lua/lua_request.c: リクエスト処理に関連して参照。
  • modules/cluster/mod_heartbeat.c: 状態監視のために参照。
  • 呼び出し先: 情報なし。

関連データ構造

  • scoreboard: 本構造体を含むメインの共有メモリ構造体。
  • worker_score: 個別のスレッド/ワーカーの統計情報。
  • ap_sb_handle_t: スレッド/プロセスを識別するためのハンドル。

失敗モードと境界条件

  • 共有メモリ上のデータであるため、プロセス間での競合が発生する可能性があります。
  • 構造体内のフィールドは固定サイズで設計されており、オーバーフローを防ぐ設計が求められます。

保守担当者向けメモ

  • include/ap_mmn.h の記述から、非同期接続情報、リスナーバケット、wait_io フィールドなどが過去の修正で追加されています。
  • 構造体のメンバ構成は MPM の要件に応じて変更される可能性があるため、変更時は ap_mmn.h のモジュールマジックナンバーを確認してください。
