timeout_msec

定義

  • 種別: function
  • 定義箇所: modules/md/md_curl.c:183

目的

apr_time_t 型のタイムアウト値を、libcurl が期待するミリ秒単位の long 型へ変換します。

シグネチャまたは形状

static long timeout_msec(apr_time_t timeout)

入力と出力

  • **入力**: apr_time_t timeout（APR 形式のタイムアウト時間）
  • **出力**: long（ミリ秒単位のタイムアウト値）

前提条件と不変条件

  • 入力 timeout が 0 の場合、出力は 0 となります。
  • 入力 timeout が 0 以外で、ミリ秒換算値が 0 になる場合（非常に短い時間）、出力は 1 となります。

アルゴリズムまたは振る舞い

1. apr_time_as_msec(timeout) を呼び出し、APR 形式の時間をミリ秒に変換します。 2. 変換結果が 0 以外であれば、その値をそのまま返します。 3. 変換結果が 0 で、かつ元の timeout が 0 でない場合は、最小値として 1 を返します。 4. 元の timeout が 0 であれば、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_curl.c:298 (curl_easy_setopt の CURLOPT_TIMEOUT_MS 設定時)
  • modules/md/md_curl.c:301 (curl_easy_setopt の CURLOPT_CONNECTTIMEOUT_MS 設定時)
  • **呼び出し先**:
  • apr_time_as_msec (APR ライブラリ関数)

関連データ構造

  • apr_time_t (APR ライブラリの時刻型)

失敗モードと境界条件

  • timeout が 0 の場合、タイムアウトなし（無限待機）として 0 を返します。
  • 非常に短い timeout 値が渡された場合、0 ではなく 1 を返すことで、libcurl 側でタイムアウトが無効化されるのを防いでいます。

保守担当者向けメモ

  • 本関数は libcurl の CURLOPT_TIMEOUT_MS および CURLOPT_CONNECTTIMEOUT_MS オプション専用の変換ユーティリティです。
  • apr_time_as_msec の戻り値が 0 になるケース（1ミリ秒未満の指定）を考慮した安全な変換を行っています。
