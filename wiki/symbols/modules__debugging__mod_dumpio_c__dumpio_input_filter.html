dumpio_input_filter

定義

  • 種別: function
  • 定義箇所: modules/debugging/mod_dumpio.c:126

目的

Apache HTTP Server の入力フィルタチェーンを流れるデータを傍受し、デバッグ用に APLOG_TRACE7 レベルでログ出力する。

シグネチャまたは形状

static int dumpio_input_filter(ap_filter_t *f, apr_bucket_brigade *bb, 
                               ap_input_mode_t mode, apr_read_type_e block, 
                               apr_off_t readbytes)

入力と出力

  • **入力**:
  • f: 現在のフィルタコンテキスト。
  • bb: データを格納するバケットブリゲード。
  • mode, block, readbytes: ap_get_brigade に渡される入力制御パラメータ。
  • **出力**:
  • 戻り値: APR_SUCCESS または ap_get_brigade から返されたエラーコード。
  • 副作用: bb 内のバケットデータがログへ出力される。

前提条件と不変条件

  • f->ctx に dumpio_conf_t 型の設定情報が格納されていること。
  • f->next が有効なフィルタチェーンの次段であること。

アルゴリズムまたは振る舞い

1. フィルタ名、読み取りモード、ブロック状態、要求バイト数を APLOG_TRACE7 でログ出力する。 2. ap_get_brigade を呼び出し、後続のフィルタからデータを取得する。 3. 取得が成功した場合、ブリゲード内の各バケットを走査し、dumpit 関数を呼び出して内容をダンプする。 4. 取得に失敗した場合はエラーログを出力し、そのエラーコードを返す。 5. 成功時は APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_register_input_filter を通じてフィルタチェーンに登録され、Apache の入力処理フローから呼び出される。
  • **呼び出し先**:
  • ap_get_brigade (後続フィルタの呼び出し)
  • dumpit (バケット内容のログ出力)
  • ap_log_cerror (ログ出力)

関連データ構造

  • ap_filter_t: フィルタチェーンのノード。
  • apr_bucket_brigade: データのバケットリスト。
  • dumpio_conf_t: モジュールの設定状態を保持する構造体。

失敗モードと境界条件

  • ap_get_brigade が APR_SUCCESS 以外を返した場合、ダンプ処理をスキップしてエラーコードを即座に返す。
  • バケットが空の場合、dumpit は何も出力しない。

保守担当者向けメモ

  • 本関数は APLOG_TRACE7 で動作するため、ログレベル設定が低いと出力されない。
  • dumpit 経由で apr_bucket_read を使用するため、メモリ消費やパフォーマンスへの影響に注意すること。
  • バイナリデータ内のヌル文字等のエスケープ処理については dumpit の実装に依存する。
