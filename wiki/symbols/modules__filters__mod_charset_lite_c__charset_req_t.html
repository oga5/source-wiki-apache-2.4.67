charset_req_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_charset_lite.c:103

目的

HTTP リクエスト単位で文字セット変換の設定およびフィルタコンテキストを保持するための構造体です。

シグネチャまたは形状

typedef struct charset_req_t {
    charset_dir_t *dc;
    charset_filter_ctx_t *output_ctx, *input_ctx;
} charset_req_t;

入力と出力

  • **入力**: find_code_page フックにおいて、リクエストプール (r->pool) からメモリが割り当てられ、ディレクトリ設定 (dc) が格納されます。
  • **出力**: ap_get_module_config(r->request_config, ...) を通じて、フィルタ処理中に設定情報や変換コンテキスト (output_ctx, input_ctx) を取得するために利用されます。

前提条件と不変条件

  • 本構造体は r->request_config に格納されます。
  • output_ctx および input_ctx は、変換処理の実行時に charset_filter_ctx_t として初期化される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: リクエスト処理の初期段階で find_code_page が呼び出され、本構造体が生成されます。 2. **保持**: ディレクトリごとの変換設定 (charset_dir_t) を参照し、フィルタチェーン内で使用される入出力用の変換コンテキストを管理します。 3. **参照**: xlate_out_filter や xlate_in_filter などのフィルタ関数が、リクエストに関連付けられた本構造体を取得し、変換処理の実行状態を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • find_code_page (生成)
  • xlate_out_filter (取得)
  • xlate_in_filter (取得)
  • **呼び出し先**:
  • 構造体内部で charset_dir_t および charset_filter_ctx_t を参照しています。

関連データ構造

  • charset_dir_t: ディレクトリ単位の変換設定を保持。
  • charset_filter_ctx_t: フィルタインスタンスごとの変換状態（apr_xlate_t ハンドルやバッファ）を保持。

失敗モードと境界条件

  • メモリ割り当て失敗: apr_pcalloc が失敗した場合、リクエスト処理が中断される可能性があります。
  • 設定の欠如: r->request_config から本構造体が取得できない場合、フィルタは変換を行わずにデータを通過させる必要があります。

保守担当者向けメモ

  • 本構造体はリクエストプール (r->pool) で管理されるため、リクエスト終了時に自動的に解放されます。
  • charset_filter_ctx_t の初期化状態については、各フィルタ関数側で適切にチェックを行う必要があります。
