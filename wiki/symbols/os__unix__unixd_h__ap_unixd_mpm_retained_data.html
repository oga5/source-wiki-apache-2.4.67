ap_unixd_mpm_retained_data

定義

  • 種別: struct
  • 定義箇所: os/unix/unixd.h:113

目的

Apache HTTP Server の MPM（Multi-Processing Module）が、サーバーの再起動（graceful restart）を跨いで状態を保持するために使用するデータ構造です。

シグネチャまたは形状

typedef struct {
    void            *baton;
    int volatile    mpm_state;
    int volatile    shutdown_pending;
    int volatile    restart_pending;
    int volatile    is_ungraceful;
    ap_generation_t my_generation;
    int             module_loads;
    int             was_graceful;
    int             num_buckets;
    int             max_buckets;
} ap_unixd_mpm_retained_data;

入力と出力

  • 本構造体は直接的な関数ではありません。
  • ap_unixd_mpm_get_retained_data() 関数を介して、保持されているインスタンスへのポインタを取得します。

前提条件と不変条件

  • mpm_state, shutdown_pending, restart_pending, is_ungraceful は volatile 修飾されており、シグナルハンドラから非同期に更新されることを前提としています。
  • 再起動時にもデータが維持されるよう、ap_retained_data_create を通じて管理される必要があります。

アルゴリズムまたは振る舞い

  • MPM のメインループは、シグナルハンドラによって更新された shutdown_pending や restart_pending などのフラグを監視し、停止や再起動の制御フローを決定します。
  • num_buckets および max_buckets は、動的なリスナーバケット数に応じてリソースを適切に割り当てるために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • os/unix/unixd.c: retained_data の生成および取得処理。
  • server/mpm/event/event.c, server/mpm/prefork/prefork.c, server/mpm/worker/worker.c: 各 MPM 実装において、状態保持のために本構造体を参照。
  • **呼び出し先**:
  • 情報なし（本構造体はデータ保持を目的としており、特定の関数を呼び出すものではありません）。

関連データ構造

  • ap_unix_identity_t: ユーザー/グループ ID 管理用。
  • unixd_config_rec: サーバーの Unix 実行環境設定用。

失敗モードと境界条件

  • ap_retained_data_create による初期化に失敗した場合、MPM は再起動時の状態引き継ぎができず、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体のメンバを変更する場合は、再起動時のバイナリ互換性および各 MPM 実装での整合性に注意してください。
  • シグナルハンドラから更新されるメンバについては、アトミック性やメモリバリアの要件を考慮する必要があります。
