cache_enable

定義

  • 種別: struct
  • 定義箇所: modules/cache/cache_util.h:108

目的

キャッシュ対象となるURLパスと、それに対応するキャッシュプロバイダーの型を関連付けるための設定構造体です。

シグネチャまたは形状

struct cache_enable {
    apr_uri_t url;
    const char *type;
    apr_size_t pathlen;
};

入力と出力

本構造体は設定データとして保持されます。

  • url: キャッシュ対象とするURI（apr_uri_t）。
  • type: 使用するキャッシュプロバイダーの名称（文字列）。
  • pathlen: url に関連付けられたパスの長さ。

前提条件と不変条件

  • mod_cache の設定フェーズにおいて、apr_array_make を用いて配列として確保されたメモリ領域に格納されます。
  • リクエスト処理中、dconf->cacheenable または conf->cacheenable から配列として参照されます。

アルゴリズムまたは振る舞い

本構造体は、mod_cache の設定ディレクティブ（CacheEnable）によって生成されます。リクエスト処理時には、この構造体の配列を走査し、現在のリクエストURLと一致するプロバイダーを特定するために利用されます（get_provider 関数等）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c: 設定読み込み時に apr_array_make で初期化され、エントリが追加されます。
  • **呼び出し先**:
  • modules/cache/cache_util.c: get_provider 関数内で、リクエストに対する適切なキャッシュプロバイダーを決定するために参照されます。

関連データ構造

  • cache_server_conf: サーバー全体の設定構造体。
  • cache_dir_conf: ディレクトリ単位の設定構造体。
  • apr_uri_t: URL情報を保持するAPRの構造体。

失敗モードと境界条件

  • 設定配列が空の場合、キャッシュプロバイダーの特定に失敗します。
  • pathlen が実際のURLパスと整合しない場合、マッチングロジックで予期せぬ挙動を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cache の設定管理において中心的な役割を果たします。
  • cache_util.c 内の get_provider 関数でこの構造体がどのように走査・比較されているかを確認することで、キャッシュの有効化ロジックを理解できます。
