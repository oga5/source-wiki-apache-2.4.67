on_event

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_stream.c:167

目的

HTTP/2 ストリームに対して発生したイベントを処理し、現在のストリーム状態に基づいた次状態への遷移を決定します。

シグネチャまたは形状

static int on_event(h2_stream* stream, h2_stream_event_t ev)

入力と出力

  • **入力**:
  • stream: 処理対象の h2_stream 構造体へのポインタ。
  • ev: 発生したイベントを示す h2_stream_event_t 型の値。
  • **出力**:
  • 遷移後の新しい状態（int 型）。イベントが定義範囲外の場合は現在の状態を返します。

前提条件と不変条件

  • H2_STRM_ASSERT_MAGIC(stream, H2_STRM_MAGIC_OK) により、ストリーム構造体の整合性が検証されます。

アルゴリズムまたは振る舞い

1. ストリームに monitor が設定されている場合、その on_event コールバックを呼び出し、イベントを通知します。 2. イベント ev が trans_on_event 配列の範囲内であるか確認します。 3. 範囲内であれば、on_map 関数を呼び出し、現在の stream->state とイベントに対応する遷移先を計算して返します。 4. 範囲外であれば、遷移を行わず現在の stream->state を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_stream.c: ストリームの状態遷移を伴う各処理（h2_stream.c:399, 469, 523 等）。
  • **呼び出し先**:
  • stream->monitor->on_event: ストリームに監視用モニターが登録されている場合に実行。
  • on_map: 状態遷移テーブルに基づいた次状態の算出。

関連データ構造

  • h2_stream: ストリームの管理構造体。
  • h2_stream_event_t: イベント種別の列挙型。
  • trans_on_event: 状態遷移を定義する内部テーブル。

失敗モードと境界条件

  • stream が無効な場合、H2_STRM_ASSERT_MAGIC によりアサーションエラーが発生します。
  • イベント ev が trans_on_event の定義範囲を超えている場合、状態遷移は行われず、現在の状態が維持されます。

保守担当者向けメモ

  • 本関数は状態遷移のロジックをカプセル化しています。遷移ルールを変更する場合は、trans_on_event テーブルの定義を確認してください。
  • monitor コールバックは、デバッグや外部監視目的で利用されています。
