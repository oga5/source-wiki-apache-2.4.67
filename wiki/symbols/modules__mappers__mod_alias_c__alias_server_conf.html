alias_server_conf

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_alias.c:54

目的

Apache HTTP Server におけるサーバーレベルの Alias および Redirect ディレクティブの設定情報を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *aliases;
    apr_array_header_t *redirects;
} alias_server_conf;

入力と出力

  • **入力**: サーバー設定ファイルで定義された Alias および Redirect ディレクティブ。
  • **出力**: サーバーの翻訳フェーズにおいて、URI をファイルパスやリダイレクト先へ変換するための設定データとして利用されます。

前提条件と不変条件

  • aliases および redirects は apr_array_header_t 型の配列であり、apr_pool_t を介してメモリ管理されます。
  • サーバー起動時に create_alias_config によって初期化され、設定解析時に add_alias_internal 等を通じてエントリが追加されます。

アルゴリズムまたは振る舞い

  • サーバー全体の設定として、定義されたエイリアスとリダイレクトのリストを保持します。
  • リクエストの翻訳フェーズ（translate_name フック）において、この構造体内のリストを走査し、リクエストされた URI と一致する定義を検索します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_alias_config (初期化)
  • merge_alias_config (設定のマージ)
  • translate_alias_redir (リクエスト翻訳時の設定取得)
  • try_alias_list (エイリアス検索時の参照)
  • **呼び出し先**:
  • apr_pcalloc (メモリ確保)
  • ap_get_module_config (設定の取得)

関連データ構造

  • alias_entry: aliases および redirects 配列の各要素として格納される、個別のエイリアス/リダイレクト定義を保持する構造体。
  • apr_array_header_t: APR が提供する動的配列。

失敗モードと境界条件

  • 設定が空の場合、aliases および redirects は空の配列として扱われます。
  • エイリアスの数が多い場合、線形探索によるパフォーマンス低下の可能性があります。

保守担当者向けメモ

  • 本構造体はサーバーコンテキスト用です。ディレクトリ単位の設定には alias_dir_conf が別途使用されます。
  • aliases および redirects の各要素は alias_entry 型であり、正規表現やパス情報を含みます。これらは apr_pool_t のライフサイクルに従うため、明示的な解放は不要です。
