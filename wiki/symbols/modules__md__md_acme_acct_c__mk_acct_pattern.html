mk_acct_pattern

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_acct.c:65

目的

ACME アカウントをファイルシステム上で検索・照合するための、ワイルドカードを含む検索パターン文字列を生成します。

シグネチャまたは形状

const char *mk_acct_pattern(apr_pool_t *p, md_acme_t *acme);

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • acme: アカウントに関連付けられたサーバー名（sname）を含む md_acme_t 構造体。
  • **出力**:
  • ACME-<servername>-* という形式の文字列を返します。

前提条件と不変条件

  • acme->sname が有効な文字列として設定されている必要があります。
  • 返される文字列は、呼び出し元で指定したプール p の生存期間に依存します。

アルゴリズムまたは振る舞い

1. apr_psprintf を使用して、acme->sname を埋め込んだパターン文字列を動的に生成します。 2. 生成されるパターンは、md_store におけるアカウントファイルの命名規則（ACME-<servername>-<index>）に適合するように設計されています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_acct.c:363
  • modules/md/md_acme_acct.c:631
  • **呼び出し先**:
  • apr_psprintf (APR ライブラリ)

関連データ構造

  • md_acme_t: サーバー名（sname）を保持する ACME セッション構造体。

失敗モードと境界条件

  • acme が NULL の場合、あるいは acme->sname が NULL の場合、apr_psprintf の動作によりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は、アカウントの永続化ファイル（account.json 等）を検索する際のフィルタリング用文字列を生成します。命名規則を変更する場合は、この関数と md_store 側の検索ロジックを同期させる必要があります。
