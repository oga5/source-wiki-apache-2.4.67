modssl_verify_ocsp

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_ocsp.c:262

目的

証明書の失効状態を OCSP (Online Certificate Status Protocol) を用いて検証し、その結果を OpenSSL の検証コンテキスト (X509_STORE_CTX) に反映させる。

シグネチャまたは形状

int modssl_verify_ocsp(X509_STORE_CTX *ctx, SSLSrvConfigRec *sc,
                       server_rec *s, conn_rec *c, apr_pool_t *pool)

入力と出力

  • **入力**:
  • ctx: 検証対象の証明書を含む OpenSSL の検証コンテキスト。
  • sc: SSL サーバー設定構造体。
  • s: サーバーレコード。
  • c: 接続レコード。
  • pool: メモリ割り当て用プール。
  • **出力**:
  • int: 検証が成功（V_OCSP_CERTSTATUS_GOOD）した場合は 1、それ以外は 0 を返す。

前提条件と不変条件

  • OPENSSL_NO_OCSP が定義されていない環境で動作する。
  • X509_STORE_CTX_get_current_cert(ctx) が有効な証明書を返す必要がある（NULL の場合は即時終了）。
  • 自己署名証明書（self-issued）に対しては OCSP チェックをスキップし、検証成功として扱う。

アルゴリズムまたは振る舞い

1. **初期化**: 一時的なメモリプール vpool を作成し、メモリ消費を抑制する。 2. **検証実行**: verify_ocsp_status を呼び出し、OCSP レスポンダとの通信および検証を行う。 3. **結果反映**: verify_ocsp_status の戻り値に基づき、X509_STORE_CTX_set_error を使用して OpenSSL の検証エラー状態を更新する。

  • V_OCSP_CERTSTATUS_GOOD: X509_V_OK を設定。
  • V_OCSP_CERTSTATUS_REVOKED: X509_V_ERR_CERT_REVOKED を設定。
  • V_OCSP_CERTSTATUS_UNKNOWN: X509_V_ERR_APPLICATION_VERIFICATION を設定。

    • 4. **終了処理**: vpool を破棄し、検証結果を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_kernel.c:1679
  • **呼び出し先**:
  • verify_ocsp_status (内部関数)
  • OpenSSL API (X509_STORE_CTX_get_current_cert, X509_check_issued, X509_STORE_CTX_set_error)
  • APR API (apr_pool_create, apr_pool_tag, apr_pool_destroy)

関連データ構造

  • X509_STORE_CTX: OpenSSL の証明書検証コンテキスト。
  • SSLSrvConfigRec: mod_ssl のサーバー設定。
  • apr_pool_t: APR メモリプール。

失敗モードと境界条件

  • 証明書が取得できない場合、エラーを発生させずに 1 を返して終了する。
  • 自己署名証明書は検証をスキップする。
  • OCSP レスポンダとの通信は同期的に行われるため、ネットワーク遅延が処理時間に影響する。

保守担当者向けメモ

  • 本関数は mod_ssl の証明書検証プロセスにおけるメインのエントリポイントである。
  • メモリリークを防ぐため、vpool の生成と破棄が確実に行われるよう注意すること。
  • X509_STORE_CTX_set_error によるエラーコードの伝播は、OpenSSL の検証エンジンが後続の処理で正しく解釈できるよう整合性を保つ必要がある。
