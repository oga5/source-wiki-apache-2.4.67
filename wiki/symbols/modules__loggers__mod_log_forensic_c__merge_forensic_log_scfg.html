merge_forensic_log_scfg

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_forensic.c:59

目的

サーバー設定の結合時に、親設定（parent）と新しい設定（new）をマージし、フォレンジックログ用の設定構造体（fcfg）を生成する。

シグネチャまたは形状

static void *merge_forensic_log_scfg(apr_pool_t *p, void *parent, void *new)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • parent: 親サーバーの設定構造体（fcfg）。
  • new: 現在のサーバーの設定構造体（fcfg）。
  • **出力**:
  • マージ済みの新しい fcfg 構造体へのポインタ。

前提条件と不変条件

  • parent および new は fcfg 型としてキャスト可能であること。
  • p は有効なメモリプールであること。
  • 戻り値の cfg->fd は常に NULL で初期化される（ファイルディスクリプタはマージされない）。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい fcfg 構造体用のメモリをプール p から確保する。 2. logname フィールドを決定する。new 側の logname が設定されていればそれを優先し、設定されていなければ parent 側の logname を使用する。 3. apr_pstrdup を使用して、選択された logname をプール p 上に複製する。 4. cfg->fd を明示的に NULL に設定する。 5. 構築した cfg を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_log_forensic.c:286（サーバー設定のマージ処理において参照されている）。
  • **呼び出し先**: apr_pcalloc, apr_pstrdup（APR API）。

関連データ構造

  • fcfg: フォレンジックログの設定を保持する構造体。logname（ログファイル名）と fd（ファイルディスクリプタ）を管理する。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc または apr_pstrdup が例外を発生させる可能性がある。
  • parent または new が NULL の場合の挙動については、ソースコード上では明示的なガードは確認できない。

保守担当者向けメモ

  • 本関数は Apache の設定マージフェーズで呼び出される。
  • fd が常に NULL で初期化されるため、ログファイルを開く処理は別途 log_init 等のフックで行う必要がある。
