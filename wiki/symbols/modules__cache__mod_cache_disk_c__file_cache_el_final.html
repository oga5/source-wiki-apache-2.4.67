file_cache_el_final

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:161

目的

一時ファイルとして保存されていたキャッシュデータを、最終的なキャッシュパスへアトミックに移動（リネーム）し、永続化を完了させる。

シグネチャまたは形状

apr_status_t file_cache_el_final(disk_cache_conf *conf, disk_cache_file_t *file, request_rec *r)

入力と出力

  • **入力**:
  • conf: ディスクキャッシュの設定情報。
  • file: 操作対象のキャッシュファイル構造体（一時ファイルパスと最終パスを保持）。
  • r: 現在のリクエストレコード。
  • **出力**:
  • apr_status_t: 操作の成功（APR_SUCCESS）または失敗を示すステータスコード。

前提条件と不変条件

  • 一時ファイルと最終的なキャッシュファイルは、同一のファイルシステム上に存在することを前提としている。
  • file->tempfd が有効（NULL 以外）である場合にのみ、リネーム処理が実行される。

アルゴリズムまたは振る舞い

1. file->tempfd が存在するか確認する。 2. safe_file_rename を呼び出し、一時ファイル（file->tempfile）を最終パス（file->file）へリネームする。 3. リネームに失敗した場合、警告ログを出力し、一時ファイルを削除（apr_file_remove）する。 4. 処理完了後、file->tempfd を NULL にリセットする。 5. 最終的なステータスを呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c:1339
  • modules/cache/mod_cache_disk.c:1342
  • modules/cache/mod_cache_disk.c:1346
  • **呼び出し先**:
  • safe_file_rename
  • ap_log_rerror
  • apr_file_remove

関連データ構造

  • disk_cache_conf: モジュールの設定情報を保持。
  • disk_cache_file_t: キャッシュファイルのパス、ファイルハンドル、プールを管理する構造体。

失敗モードと境界条件

  • **ファイルシステム境界**: 一時ファイルとキャッシュルートが異なるファイルシステムにある場合、safe_file_rename が失敗する可能性がある（ソース内のコメントによる指摘）。
  • **リネーム失敗**: リネーム処理が失敗した場合、エラーログが記録され、一時ファイルがクリーンアップされる。

保守担当者向けメモ

  • 本関数は、キャッシュのコミット処理の最終段階で呼び出される。
  • safe_file_rename は htcacheclean 等によるディレクトリ削除との競合を考慮した設計となっているため、この関数を修正する際はそのアトミック性を維持すること。
