thread_exception_handler

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2_child.c:311

目的

OS/2 環境下において、ワーカースレッドで発生した例外を捕捉し、プロセス全体の健全性を維持するために安全なシャットダウンを開始する。

シグネチャまたは形状

ULONG APIENTRY thread_exception_handler(EXCEPTIONREPORTRECORD *pReportRec,
                                        EXCEPTIONREGISTRATIONRECORD *pRegRec,
                                        CONTEXTRECORD *pContext,
                                        PVOID p)

入力と出力

  • **入力**:
  • pReportRec: 例外の詳細情報（例外番号やフラグ）。
  • pRegRec, pContext, p: OS/2 例外登録レコードおよびコンテキスト情報。
  • **出力**:
  • XCPT_CONTINUE_SEARCH: 例外を処理しなかった場合、またはネストされた例外の場合。

前提条件と不変条件

  • ネストされた例外（EH_NESTED_CALL）が発生した場合は、無限ループを避けるため即座に検索を継続する。
  • XCPT_ACCESS_VIOLATION または XCPT_INTEGER_DIVIDE_BY_ZERO のみが本ハンドラの処理対象である。

アルゴリズムまたは振る舞い

1. 例外がネストされているか確認し、該当する場合は XCPT_CONTINUE_SEARCH を返す。 2. 対象の例外（アクセス違反またはゼロ除算）である場合、エラーログを出力する。 3. スコアボードを走査し、現在のスレッドID（_gettid()）と一致するエントリを SERVER_DEAD に設定する。 4. プロセス全体の終了フラグ shutdown_pending を立て、スコアボードの quiescing を有効にする。 5. kill(getpid(), SIGHUP) を発行してプロセスに終了を通知する。 6. DosUnwindException を呼び出し、スタックを巻き戻して処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: OS/2 の例外処理機構（server/mpm/mpmt_os2/mpmt_os2_child.c:362 にて EXCEPTIONREGISTRATIONRECORD に登録される）。
  • **呼び出し先**: ap_log_error, getpid, _gettid, kill, DosUnwindException。

関連データ構造

  • EXCEPTIONREPORTRECORD: OS/2 の例外レポート構造体。
  • ap_scoreboard_image: プロセス間で共有されるスコアボード情報。

失敗モードと境界条件

  • **境界条件**: HARD_THREAD_LIMIT を超えるスレッドは走査対象外となる。
  • **失敗モード**: 例外発生時にプロセスが既に不安定な状態である場合、DosUnwindException による巻き戻しが完全に行われない可能性がある。

保守担当者向けメモ

  • 本関数は OS/2 ネイティブの例外処理機構に依存している。
  • DosUnwindException の呼び出しは、プロセスがリソースリークを起こす可能性があるほど「不健全」であるという判断に基づいている。
