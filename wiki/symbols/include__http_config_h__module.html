module

定義

  • 種別: struct
  • 定義箇所: include/http_config.h:344

目的

Apache HTTP Server における各モジュールの基底構造体です。モジュール固有の設定データ、フック登録、およびサーバーのライフサイクル（初期化、設定読み込み、リクエスト処理）への統合インターフェースを提供します。

シグネチャまたは形状

typedef struct module_struct module;

※ module_struct の具体的なメンバ構成は本ヘッダーファイル内の別定義に依存します。

入力と出力

  • **入力**: サーバー起動時にロードされる共有オブジェクトまたは静的リンクを通じて、サーバーのモジュールリストに登録されます。
  • **出力**: サーバーの構成管理システムに対し、モジュールが提供するディレクティブやフック関数を公開します。

前提条件と不変条件

  • module 構造体内の module_index は、サーバー起動時に割り当てられ、実行中に変更してはなりません。
  • メモリ管理には主に pconf (設定プール) が使用されます。

アルゴリズムまたは振る舞い

1. **登録**: サーバー起動時に module 構造体がロードされ、サーバーの内部リストに組み込まれます。 2. **設定処理**: 設定ファイル解析時、本構造体を通じて各モジュールの merge_dir_config や merge_server_config が呼び出され、コンテキストの継承が行われます。 3. **実行**: リクエスト処理中、ap_get_module_config を介して本構造体に関連付けられたモジュール固有のコンテキストが取得されます。

呼び出し元と呼び出し先

  • **呼び出し元**: サーバーのコア設定システム（http_core.c 等）が、設定ファイルの走査やリクエスト処理の各フェーズで本構造体を参照します。
  • **呼び出し先**: モジュールが実装するフック関数や設定マージ関数を指し示します。
  • **参照元**: mod_example_1.c, mod_example_2.c などの各モジュール実装、および ap_config.h 等の構成管理ヘッダー。

関連データ構造

  • command_rec: モジュールが提供するディレクティブの定義。
  • ap_conf_vector_t: モジュール固有の設定データを保持するベクトル。
  • module_struct: module の実体定義。

失敗モードと境界条件

  • ap_get_module_config はデバッグビルド以外ではマクロとして実装されており、インデックスの範囲外アクセスに対する保護はありません。

保守担当者向けメモ

  • 本構造体はインターフェース定義であり、具体的なメモリ確保や解析ロジックは server/config.c 等の実装側に委ねられています。
  • AP_DECLARE_MODULE マクロを使用して宣言することが推奨されます。
