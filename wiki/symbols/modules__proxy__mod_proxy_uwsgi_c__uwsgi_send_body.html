uwsgi_send_body

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_uwsgi.c:238

目的

クライアントから送信されたリクエストボディを読み取り、uWSGI バックエンドサーバーへ転送する。

シグネチャまたは形状

int uwsgi_send_body(request_rec *r, proxy_conn_rec *conn)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • conn: バックエンドへの接続状態を保持する proxy_conn_rec 構造体。
  • **出力**:
  • OK: 正常終了。
  • HTTP_SERVICE_UNAVAILABLE: バックエンドへのデータ送信失敗。
  • HTTP_INTERNAL_SERVER_ERROR: クライアントからのボディ読み取り失敗。

前提条件と不変条件

  • r->pool が有効であり、バッファ割り当てに使用可能であること。
  • conn がバックエンドへの有効な接続を保持していること。

アルゴリズムまたは振る舞い

1. ap_should_client_block(r) を呼び出し、転送すべきボディが存在するか確認する。 2. 存在する場合、AP_IOBUFSIZE サイズのバッファをリクエストプールから確保する。 3. ap_get_client_block を使用して、ボディをループで読み取る。 4. 読み取ったデータを uwsgi_send を介してバックエンドへ送信する。 5. 送信中に uwsgi_send が OK 以外を返した場合、直ちに HTTP_SERVICE_UNAVAILABLE を返す。 6. 読み取り中にエラー（readlen == -1）が発生した場合は、ログを出力し HTTP_INTERNAL_SERVER_ERROR を返す。 7. ボディの読み取りが完了した時点で OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_uwsgi.c:572 (uwsgi_handler 内)
  • **呼び出し先**:
  • ap_should_client_block
  • apr_palloc
  • ap_get_client_block
  • uwsgi_send
  • ap_log_rerror

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • proxy_conn_rec: プロキシ接続情報。

失敗モードと境界条件

  • **送信失敗**: uwsgi_send がエラーを返すと、処理を中断し HTTP_SERVICE_UNAVAILABLE を返す。
  • **読み取り失敗**: ap_get_client_block が -1 を返した場合、内部エラーとして処理される。

保守担当者向けメモ

  • 本関数は AP_IOBUFSIZE 単位でストリーミングを行うため、メモリ消費量は一定に保たれる。
  • uwsgi_send の戻り値チェックは厳密に行われており、通信エラー時は即座にリクエスト処理を終了させる設計となっている。
