h2_request_rcreate

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:79

目的

Apache のネイティブなリクエスト構造体 (request_rec) から、HTTP/2 用のリクエスト表現 (h2_request) を生成・変換します。

シグネチャまたは形状

apr_status_t h2_request_rcreate(h2_request **preq, apr_pool_t *pool, 
                                request_rec *r, h2_hd_scratch *scratch)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • r: 変換元の Apache request_rec 構造体。
  • scratch: ヘッダー処理用の一時領域 (h2_hd_scratch)。
  • **出力**:
  • preq: 生成された h2_request オブジェクトへのポインタ。
  • 戻り値: 成功時に APR_SUCCESS、失敗時にエラーコードを返します。

前提条件と不変条件

  • r の各フィールド（メソッド、ホスト名、URI）が有効であること。
  • h2_request 内の headers テーブルは、r->headers_in からのコピーによって初期化されます。

アルゴリズムまたは振る舞い

1. r->parsed_uri および r->hostname を基に、スキーム、権限（authority）、パスを抽出・複製します。 2. 権限情報にポート番号が含まれていない場合、r->parsed_uri.port_str や r->server->port を参照して補完を試みます。 3. h2_request オブジェクトをメモリ確保し、メソッド、スキーム、権限、パスを格納します。 4. apr_table_do を使用して、r->headers_in から h2_request のヘッダーテーブルへ値を転送します。この際、set_h1_header 関数が各ヘッダーの処理を担います。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_stream.c:662
  • **呼び出し先**:
  • apr_pstrdup, apr_uri_unparse, apr_pstrcat, apr_psprintf, apr_pcalloc, apr_table_make, apr_table_do, apr_time_now (APR API)
  • ap_http_scheme, ap_strchr_c, apr_uri_port_of_scheme (Apache API)
  • set_h1_header (内部関数)

関連データ構造

  • h2_request: HTTP/2 リクエストの内部表現。
  • request_rec: Apache の標準リクエスト構造体。
  • h2_hd_scratch: ヘッダー処理用のスクラッチパッド。
  • h1_ctx: ヘッダー変換時のコンテキスト保持用構造体。

失敗モードと境界条件

  • r->method、scheme、r->hostname、または path のいずれかが欠落している場合、APR_EINVAL を返します。
  • メモリ割り当て失敗時は APR のエラーコードが返されます。

保守担当者向けメモ

  • 本関数は request_rec から h2_request への変換を行うため、Apache のリクエスト処理パイプラインと HTTP/2 プロトコル層の橋渡しとして機能します。
  • 権限（authority）の構築ロジックは、HTTP/2 の仕様と Apache のサーバー設定（ポート番号の扱い）に依存しているため、変更時は注意が必要です。
