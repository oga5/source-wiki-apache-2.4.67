rewrite_perdir_conf

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:372

目的

ディレクトリ単位（.htaccess 等）のコンテキストにおける mod_rewrite の設定状態およびルールセットを保持する構造体です。

シグネチャまたは形状

typedef struct {
    int           state;              /* RewriteEngineの状態 */
    int           options;            /* RewriteOptionの状態 */
    apr_array_header_t *rewriteconds; /* RewriteCondエントリ（一時的） */
    apr_array_header_t *rewriterules; /* RewriteRuleエントリ */
    char         *directory;          /* 適用対象のディレクトリパス */
    const char   *baseurl;            /* 適用対象のベースURL */
    unsigned int state_set:1;
    unsigned int options_set:1;
    unsigned int baseurl_set:1;
} rewrite_perdir_conf;

入力と出力

本構造体は設定の保持を目的としており、関数としての入力・出力は持ちません。設定読み込み時に apr_pcalloc を介してメモリが割り当てられ、リクエスト処理時に dconf として参照されます。

前提条件と不変条件

  • rewriteconds および rewriterules は apr_array_header_t として初期化される必要があります。
  • state_set、options_set、baseurl_set の各ビットフィールドは、対応する設定値が明示的に設定されたかどうかを追跡するために使用されます。

アルゴリズムまたは振る舞い

  • ディレクトリ単位の設定マージ処理において、親ディレクトリの設定と現在の設定を統合する際のデータコンテナとして機能します。
  • mod_rewrite の hook_fixup フェーズにおいて、リクエストに対する書き換えルールを適用する際の参照先となります。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_rewrite.c 内のディレクトリ設定作成関数（create_rewrite_dir_config 等）およびマージ関数が本構造体をインスタンス化・操作します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • rewriterule_entry: rewriterules 配列内に格納される個別のルール定義。
  • rewritecond_entry: rewriteconds 配列内に格納される個別の条件定義。
  • rewrite_server_conf: サーバー全体の設定を保持する対となる構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc による割り当て失敗が発生する可能性があります。
  • 設定の不整合: baseurl や directory が適切に設定されていない場合、パスの書き換え処理で予期しない動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はリクエストごとのメモリプール（r->pool）に関連付けられて管理されることが多いため、メモリリークの懸念は少ないですが、キャッシュ関連のデータ構造との生存期間の違いに注意してください。
  • rewriteconds は一時的な保持用として定義されています。ルール適用時のライフサイクルを確認してください。
