ap_rvputs_proto_in_ascii

定義

  • 種別: function
  • 定義箇所: server/util_ebcdic.c:91
  • マクロ定義: util_ebcdic.h:83

目的

EBCDIC環境において、可変長引数で渡された文字列をASCII形式に変換し、HTTPレスポンスストリームへ出力するラッパー関数です。

シグネチャまたは形状

int ap_rvputs_proto_in_ascii(request_rec *r, ...);

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体へのポインタ。
  • ...: const char * 型の文字列引数リスト。末尾は NULL で終端する必要があります。
  • **出力**:
  • 成功時: 書き込まれた合計バイト数（int）。
  • 失敗時: -1（ap_rputs がエラーを返した場合）。

前提条件と不変条件

  • 本関数は APR_CHARSET_EBCDIC が定義されている環境でのみ有効です。
  • 引数リストの最後には必ず NULL を指定する必要があります。
  • r->pool はメモリ確保のために有効である必要があります。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. 引数を一つずつ取得し、NULL に到達するまで以下の処理を繰り返します。

  • apr_pstrmemdup を使用して、リクエストプール上に文字列のコピーを作成します。
  • ap_xlate_proto_to_ascii を呼び出し、コピーしたバッファをインプレースでEBCDICからASCIIへ変換します。
  • ap_rputs を呼び出し、変換後の文字列をレスポンスストリームへ書き出します。
  • 書き込みに失敗した場合は va_end を呼び出し、-1 を返して終了します。

    • 3. 全ての引数の処理が完了したら va_end を呼び出し、合計書き込みバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_protocol.c 内の各箇所（エラーメッセージやシグネチャの出力時など）。
  • **呼び出し先**:
  • ap_xlate_proto_to_ascii（変換処理）
  • ap_rputs（出力処理）
  • apr_pstrmemdup（メモリ確保）

関連データ構造

  • request_rec: リクエストの状態およびメモリプールを管理。

失敗モードと境界条件

  • ap_rputs が負の値を返した場合、処理を中断し -1 を返します。
  • 引数リストに NULL が含まれていない場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 非EBCDIC環境では util_ebcdic.h により空のマクロに置換されるため、呼び出し側で条件分岐を記述する必要はありません。
  • 変換はリクエストプール上のメモリに対して行われるため、呼び出し側でのメモリ解放は不要です。
