ldap_cache_lock

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:111

目的

LDAP キャッシュ操作の排他制御を行うためのグローバルミューテックスを取得する。

シグネチャまたは形状

static APR_INLINE apr_status_t ldap_cache_lock(util_ldap_state_t *st, request_rec *r)

入力と出力

  • **入力**:
  • st: LDAP モジュールの状態構造体 (util_ldap_state_t)。
  • r: 現在の HTTP リクエスト構造体 (request_rec)。
  • **出力**:
  • apr_status_t: ミューテックス取得成功時は APR_SUCCESS を返し、失敗時はエラーコードを返す。

前提条件と不変条件

  • st->util_ldap_cache_lock が初期化されている必要がある。
  • 本関数は APR_INLINE として定義されており、呼び出し側はスレッドセーフな環境下で実行されることを想定している。

アルゴリズムまたは振る舞い

1. st->util_ldap_cache_lock が存在するか確認する。 2. 存在する場合、apr_global_mutex_lock を呼び出し、キャッシュへのアクセス権を取得する。 3. ミューテックスの取得に失敗した場合、エラーログ (APLOG_CRIT) を出力し、ap_assert(0) によりプロセスを異常終了させる。 4. 成功した場合は APR_SUCCESS を返して呼び出し元へ制御を戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c 内のキャッシュ操作関数群（964, 988, 1064, 1110, 1120, 1216, 1510, 1517, 1578, 1721, 1731, 1928 行目付近）。
  • **呼び出し先**: apr_global_mutex_lock (APR ライブラリ関数)、ap_log_rerror (Apache API)。

関連データ構造

  • util_ldap_state_t: LDAP モジュールの設定と状態を保持する構造体。
  • apr_global_mutex_t: キャッシュ保護に使用されるグローバルミューテックス。

失敗モードと境界条件

  • st->util_ldap_cache_lock が未初期化の場合、ロック処理はスキップされる。
  • apr_global_mutex_lock が APR_SUCCESS 以外を返した場合、致命的なエラーとしてログ出力後にアサーション失敗によりプロセスが停止する。

保守担当者向けメモ

  • 本関数はキャッシュの整合性を維持するための重要な排他制御ポイントである。
  • 呼び出し元で ldap_cache_lock を使用した後は、必ず対となる ldap_cache_unlock を呼び出してロックを解放すること。
