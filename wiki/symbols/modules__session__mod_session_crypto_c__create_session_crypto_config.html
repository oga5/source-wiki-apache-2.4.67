create_session_crypto_config

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:615

目的

サーバー設定用の session_crypto_conf 構造体をメモリ上に確保し、初期化を行う。

シグネチャまたは形状

static void *create_session_crypto_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t プール。
  • s: 設定対象の server_rec サーバーレコード。
  • **出力**:
  • 初期化済みの session_crypto_conf 構造体へのポインタ（void * として返却）。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全メンバはゼロ初期化される。
  • APU_CRYPTO_RECOMMENDED_DRIVER が定義されている場合、library メンバにその値が設定される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、session_crypto_conf サイズ分のメモリをプール p から確保する。 2. コンパイル時に APU_CRYPTO_RECOMMENDED_DRIVER が定義されている場合、確保した構造体の library メンバにその値を代入する。 3. 初期化された構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_crypto.c:784 （サーバー設定の初期化プロセスにおいて呼び出される）。
  • **呼び出し先**: apr_pcalloc (APR ライブラリ関数)。

関連データ構造

  • session_crypto_conf: 本関数が生成・初期化するサーバー全体の設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性がある（APR のメモリ管理規約に従う）。

保守担当者向けメモ

  • 本関数は Apache モジュールの設定作成フックとして機能する。
  • library メンバのデフォルト値はコンパイル時の環境変数に依存するため、ビルド環境によって挙動が異なる可能性がある。
