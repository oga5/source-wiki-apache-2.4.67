h2_proxy_ihash_t

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.c:69

目的

整数識別子（ID）をキーとして、構造体インスタンスを効率的に管理・検索するためのハッシュテーブル構造体です。

シグネチャまたは形状

struct h2_proxy_ihash_t {
    apr_hash_t *hash;
    size_t ioff;
};

入力と出力

  • hash: apr_hash_t へのポインタ。内部的なハッシュ管理に使用されます。
  • ioff: 構造体内の整数 ID メンバーへのオフセット値。ハッシュキーを自動抽出するために使用されます。

前提条件と不変条件

  • ihash に格納される構造体内の整数 ID メンバーは、ハッシュテーブルに登録されている間は変更してはなりません。
  • ID を変更する必要がある場合は、一度 h2_proxy_ihash_remove で削除し、値を更新した後に再度 h2_proxy_ihash_add を行う必要があります。

アルゴリズムまたは振る舞い

  • apr_hash_t をラップし、構造体のメモリレイアウト上のオフセット（ioff）を利用して、キーとなる整数値を自動的に抽出します。
  • 挿入、削除、検索、およびイテレーション操作を提供します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.h にて構造体定義が参照されており、h2_proxy_session 内のストリーム管理（streams）などで利用されています。
  • **呼び出し先**: apr_hash_t 関連の APR 関数群を内部的に呼び出します。

関連データ構造

  • apr_hash_t: 実際のハッシュテーブルの実装として利用。
  • h2_proxy_session: 本構造体を保持し、ストリーム管理に使用。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数が異常終了する可能性があります。
  • 存在しない ID に対する操作や、重複した ID の追加に関する挙動は、基盤となる apr_hash_t の仕様に依存します。

保守担当者向けメモ

  • 本構造体は h2_proxy_util.c で定義されていますが、h2_proxy_session.h 等のヘッダーファイルから参照されるため、変更時は依存関係に注意してください。
  • ioff の計算には APR_OFFSETOF マクロが利用されることが想定されます。
