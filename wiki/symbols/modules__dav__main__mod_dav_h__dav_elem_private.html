dav_elem_private

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:2620

目的

ライブプロパティ（Live Property）に関連付けられたプライベートなメタデータを保持するための構造体です。

シグネチャまたは形状

typedef struct {
    int propid;                          /* live property ID */
    const dav_hooks_liveprop *provider;  /* the provider defining this prop */
} dav_elem_private;

入力と出力

本構造体は、主にプロパティ操作コンテキスト内でプロパティの識別と定義元プロバイダーを特定するために使用されます。

前提条件と不変条件

  • propid は、対応する provider 内で一意に定義されたライブプロパティの識別子である必要があります。
  • provider は、当該プロパティを定義する dav_hooks_liveprop 構造体への有効なポインタである必要があります。

アルゴリズムまたは振る舞い

ライブプロパティの読み取りや書き込み処理において、この構造体を通じてプロパティの定義元プロバイダーを特定し、適切なフック関数を呼び出すためのブリッジとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • modules/dav/fs/repos.c: リソース要素からのプライベートデータ取得。
  • modules/dav/main/props.c: プロパティ操作（dav_rw_liveprop 等）およびコンテキスト内のプロパティ情報取得。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、保持する provider フィールドを通じて dav_hooks_liveprop が提供する関数が間接的に呼び出されます。

関連データ構造

  • dav_hooks_liveprop: ライブプロパティの操作フックを定義する構造体。
  • dav_propdb: プロパティデータベースの操作コンテキスト。

失敗モードと境界条件

  • provider が NULL の場合、プロパティの操作を試みるとセグメンテーション違反が発生する可能性があります。
  • propid がプロバイダー側で無効な値として扱われる場合、プロパティの解決に失敗します。

保守担当者向けメモ

  • 本構造体は dav_elem（XML要素）のプライベートデータとして格納されることが一般的です。
  • プロパティのライフサイクル管理において、この構造体が保持するポインタが指す dav_hooks_liveprop が、リクエストの生存期間中有効であることを確認してください。
