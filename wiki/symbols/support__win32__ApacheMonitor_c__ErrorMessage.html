ErrorMessage

定義

  • 種別: function
  • 定義箇所: support/win32/ApacheMonitor.c:198

目的

ApacheMonitor アプリケーションにおけるエラー通知の集約と、致命的なエラー発生時のアプリケーション終了処理を行う。

シグネチャまたは形状

void ErrorMessage(LPCTSTR szError, BOOL bFatal)

入力と出力

  • szError (入力): 表示するエラーメッセージ文字列。NULL の場合、GetLastError() を用いてシステムエラーメッセージを自動取得する。
  • bFatal (入力): 致命的なエラーかどうかを示すフラグ。TRUE の場合、メッセージ表示後にアプリケーションを終了させる。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • g_lpMsg 配列が初期化されており、IDS_MSG_ERROR に対応するエラータイトル文字列が格納されていること。
  • g_LangID がシステムエラーメッセージ取得のために正しく設定されていること。

アルゴリズムまたは振る舞い

1. szError が指定されている場合、その文字列をメッセージボックスに表示する。 2. szError が NULL の場合、GetLastError() を呼び出し、FormatMessage を用いてシステムからエラーメッセージを取得・バッファ確保する。 3. メッセージボックスのアイコンは bFatal が TRUE なら MB_ICONERROR、FALSE なら MB_ICONEXCLAMATION を使用する。 4. FormatMessage で確保したバッファは LocalFree で解放する。 5. bFatal が TRUE の場合、PostQuitMessage(0) を呼び出し、メッセージループを終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: support/win32/ApacheMonitor.c 内の各所（サービス制御失敗時やレジストリ操作失敗時など）。
  • 呼び出し先: MessageBox (Win32 API), FormatMessage (Win32 API), LocalFree (Win32 API), PostQuitMessage (Win32 API)。

関連データ構造

  • g_lpMsg: エラーメッセージ等のリソース文字列を保持するグローバル配列。

失敗モードと境界条件

  • FormatMessage がシステムエラーメッセージの取得に失敗した場合、メッセージボックスの内容が空になる可能性がある。
  • bFatal が TRUE の場合、即座に終了処理へ移行するため、呼び出し元以降の処理は実行されない。

保守担当者向けメモ

  • 本関数はエラーハンドリングの共通窓口である。エラーメッセージの追加や変更は、ApacheMonitor.h のリソース定義と g_lpMsg の整合性を確認すること。
  • FormatMessage 使用時のメモリ確保は FORMAT_MESSAGE_ALLOCATE_BUFFER に依存しているため、LocalFree の呼び出しを忘れないこと。
