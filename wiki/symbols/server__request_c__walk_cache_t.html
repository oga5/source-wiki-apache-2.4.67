walk_cache_t

定義

  • 種別: struct
  • 定義箇所: server/request.c:441

目的

リクエスト処理におけるディレクトリやファイル単位の設定マージ（Walk）結果をキャッシュし、サブリクエストやリダイレクト発生時の再計算を回避してパフォーマンスを最適化するために使用されます。

シグネチャまたは形状

typedef struct walk_cache_t {
    const char         *cached;          /* マッチした識別子 */
    ap_conf_vector_t  **dir_conf_tested; /* マッチ対象となったセクション */
    ap_conf_vector_t   *dir_conf_merged; /* ベースとなる per_dir_config */
    ap_conf_vector_t   *per_dir_result;  /* マージ後の per_dir_config */
    apr_array_header_t *walked;          /* walk_walked_t 結果のリスト */
    struct walk_cache_t *prev;           /* 同一リクエスト内の前回のキャッシュ */
    int count;                           /* 同一リクエスト内での呼び出し回数 */
} walk_cache_t;

入力と出力

  • 本構造体自体は、prep_walk_cache 関数を介してリクエストコンテキストから生成・取得されます。
  • 各フィールドは、設定マージ処理（ap_directory_walk 等）の入力となる設定ベクトルや、その結果を保持します。

前提条件と不変条件

  • 同一リクエスト（またはサブリクエスト）内での設定マージの整合性を維持するため、prev ポインタにより過去のキャッシュ状態を追跡します。
  • walked フィールドには、個別のマージ結果である walk_walked_t 構造体のリストが格納されます。

アルゴリズムまたは振る舞い

1. 設定マージ処理の開始時に prep_walk_cache が呼び出され、既存のキャッシュを検索します。 2. キャッシュが存在する場合、dir_conf_tested を比較して設定の再利用が可能か判定します。 3. 再利用可能な場合は計算をスキップし、キャッシュされた per_dir_result を返します。 4. キャッシュがない、または無効な場合は、新たにマージを実行し、その結果を本構造体に保存します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/request.c 内の prep_walk_cache 関数（451行目）で初期化され、ap_directory_walk 等の各 Walk 関数内で参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • walk_walked_t: walked リスト内に格納される、個別のマージ結果を保持する構造体。
  • ap_conf_vector_t: Apache の設定ベクトル。
  • request_rec: リクエスト情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: apr_array_header_t の初期化や構造体の割り当てに失敗する可能性があります。
  • キャッシュ不整合: リクエスト処理中に設定が動的に変更された場合、キャッシュの有効性判定（dir_conf_tested の比較）が重要となります。

保守担当者向けメモ

  • 本構造体は server/request.c における設定マージのパフォーマンスの要です。ロジックを変更する際は、キャッシュの無効化条件が正しく機能しているか確認してください。
  • prev ポインタによる連鎖構造は、サブリクエストが多重に発生する場合のメモリ消費に影響するため、不要なキャッシュ保持が発生していないか注意が必要です。
