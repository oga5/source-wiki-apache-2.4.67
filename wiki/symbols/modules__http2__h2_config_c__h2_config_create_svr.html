h2_config_create_svr

定義

  • **種別**: function
  • **定義箇所**: modules/http2/h2_config.c:143

目的

Apache HTTP Server のサーバーレベル設定構造体（h2_config）をメモリプール上に確保し、初期化します。

シグネチャまたは形状

void *h2_config_create_svr(apr_pool_t *pool, server_rec *s)

入力と出力

  • **入力**:
  • pool: 設定構造体の割り当てに使用する apr_pool_t。
  • s: 設定対象のサーバーレコード（server_rec）。
  • **出力**:
  • 初期化済みの h2_config 構造体へのポインタ。

前提条件と不変条件

  • pool は有効なメモリプールである必要があります。
  • 構造体内の各設定値は、未設定状態を示す DEF_VAL（-1）で初期化されます。
  • priorities、push_list、early_headers などのポインタ型メンバは NULL で初期化されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して h2_config サイズのメモリをゼロクリアして確保します。 2. s->defn_name を用いて、デバッグ用の識別名 name を生成し、構造体に格納します。 3. 構造体内のすべての設定項目（h2_max_streams, h2_window_size, min_workers 等）に DEF_VAL を代入します。 4. ポインタ型のメンバを NULL に設定します。 5. 初期化済みの構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c:54 にて、サーバー設定作成用関数として登録されています。
  • **呼び出し先**: apr_pcalloc および apr_pstrcat（APR ライブラリ関数）。

関連データ構造

  • h2_config: 本関数で生成されるサーバー設定構造体。
  • server_rec: Apache のサーバー設定レコード。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させるか、プロセスを終了させる可能性があります（APR のメモリ管理規約に依存）。
  • s->defn_name が NULL の場合、apr_pstrcat の動作に依存します。

保守担当者向けメモ

  • 本関数は mod_http2 の設定フェーズで呼び出されます。
  • 新しい設定項目を h2_config 構造体に追加した場合は、必ず本関数で DEF_VAL による初期化を行ってください。
  • DEF_VAL は「未設定」を意味するマジックナンバーであり、マージ処理時に親コンテキストの設定を継承するためのフラグとして利用されます。
