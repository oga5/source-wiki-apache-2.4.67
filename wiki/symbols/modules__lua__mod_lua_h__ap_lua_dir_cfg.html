ap_lua_dir_cfg

定義

  • 種別: struct
  • 定義箇所: modules/lua/mod_lua.h:114

目的

Apache HTTP Server の mod_lua モジュールにおいて、ディレクトリ単位の設定（Lua スクリプトのパス、VM スコープ、キャッシュ設定、ハンドラマッピングなど）を保持・管理するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *package_paths;
    apr_array_header_t *package_cpaths;
    apr_array_header_t *mapped_handlers;
    apr_array_header_t *mapped_filters;
    apr_pool_t *pool;
    unsigned int vm_scope;
    unsigned int vm_min;
    unsigned int vm_max;
    apr_hash_t *hooks;
    const char *dir;
    ap_lua_inherit_t inherit;
    unsigned int codecache;
} ap_lua_dir_cfg;

入力と出力

  • **入力**: Apache の設定ファイル（httpd.conf 等）で指定されたディレクティブに基づき、mod_lua の設定読み込み関数によって値が設定されます。
  • **出力**: リクエスト処理時に ap_get_module_config を介して取得され、Lua VM の初期化やスクリプト実行の制御パラメータとして利用されます。

前提条件と不変条件

  • pool フィールドは、設定のライフサイクルを管理するための APR プールを指す必要があります。
  • vm_scope は AP_LUA_SCOPE_ONCE, REQUEST, CONN, SERVER のいずれかである必要があります。
  • codecache は AP_LUA_CACHE_NEVER, STAT, FOREVER のいずれかである必要があります。

アルゴリズムまたは振る舞い

  • ディレクトリごとの Lua 実行環境を定義します。
  • mapped_handlers および mapped_filters により、特定のパスに対する Lua スクリプトの割り当てを管理します。
  • vm_min / vm_max を用いて、当該ディレクトリで利用する Lua VM のインスタンス数を制御します。
  • inherit 設定により、サブディレクトリにおける親設定の継承順序を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_config.c: 設定のチェックやハンドラのマッピング時に参照・操作されます。
  • modules/lua/mod_lua.c: リクエスト処理の各フェーズで ap_get_module_config を通じて取得されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • apr_array_header_t: パスやハンドラリストの保持に使用。
  • apr_hash_t: hooks（wombat_hook_info）の管理に使用。
  • ap_lua_inherit_t: 設定継承の挙動を定義する列挙型。

失敗モードと境界条件

  • 設定の継承において inherit の値が不適切な場合、意図しない順序でスクリプトが実行される可能性があります。
  • vm_min が vm_max を超えるような設定値が与えられた場合の挙動は、VM 管理ロジックの実装に依存します。

保守担当者向けメモ

  • 本構造体は mod_lua の設定管理の基盤です。フィールドを追加する際は、mod_lua.c 内のコンフィグマージ関数（merge_dir_config 等）で適切に値が引き継がれるよう注意してください。
  • hooks ハッシュのキーと値の型については、mod_lua の内部的なフック管理の実装を参照してください。
