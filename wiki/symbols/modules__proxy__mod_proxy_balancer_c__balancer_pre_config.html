balancer_pre_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:48

目的

Apache HTTP Server の設定読み込みフェーズにおいて、ロードバランサ機能に必要なミューテックスの登録および外部モジュール（ヘルスチェック機能等）が提供するオプション関数の取得を行う。

シグネチャまたは形状

static int balancer_pre_config(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp)

入力と出力

  • 入力:
  • pconf: 設定用メモリプール。
  • plog: ログ用メモリプール。
  • ptemp: 一時メモリプール。
  • 出力: 成功時は OK (0)、失敗時は apr_status_t 型のエラーコードを返す。

前提条件と不変条件

  • ap_mutex_register が正常に完了し、バランサ用ミューテックスがシステムに登録される必要がある。
  • 外部モジュールから提供されるオプション関数（set_worker_hc_param 等）は、APR_RETRIEVE_OPTIONAL_FN を介して取得される。

アルゴリズムまたは振る舞い

1. ap_mutex_register を呼び出し、balancer_mutex_type を用いてミューテックスを登録する。登録に失敗した場合は即座にエラーを返す。 2. APR_RETRIEVE_OPTIONAL_FN を使用し、以下のオプション関数をグローバル変数に格納する。

  • set_worker_hc_param
  • hc_show_exprs
  • hc_select_exprs
  • hc_valid_expr

    • 3. 正常終了として OK を返す。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/mod_proxy_balancer.c:2061 にて ap_hook_pre_config に登録されている。
  • 呼び出し先:
  • ap_mutex_register (Apache API)
  • APR_RETRIEVE_OPTIONAL_FN (APR マクロ)

関連データ構造

  • apr_pool_t: メモリ管理用構造体。
  • balancer_mutex_type: ミューテックス識別子（文字列）。

失敗モードと境界条件

  • ap_mutex_register が APR_SUCCESS 以外の値を返した場合、そのエラーコードを呼び出し元に伝播させ、初期化を中断する。

保守担当者向けメモ

  • 本関数は mod_proxy_balancer の初期化の最初期段階で実行される。
  • 外部モジュール（mod_proxy_hcheck 等）の関数を動的に取得しているため、これらのモジュールがロードされていない場合でも動作する設計となっている。
