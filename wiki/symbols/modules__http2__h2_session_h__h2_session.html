h2_session

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_session.h:66

目的

HTTP/2 セッションのライフサイクル、状態管理、およびストリームの多重化を制御するためのメイン構造体です。クライアントとの単一の TCP 接続（conn_rec）を抽象化し、プロトコル層としての役割を担います。

シグネチャまたは形状

typedef struct h2_session {
    int child_num;
    apr_uint32_t id;

    request_rec *r;
    server_rec *s;
    apr_pool_t *pool;

    struct h2_workers *workers;
    struct h2_c1_io_in_ctx_t *cin;
    h2_c1_io io;
    unsigned int padding_max;
    int padding_always;
    struct nghttp2_session *ngh2;
    h2_session_state state;
    h2_session_props local;
    h2_session_props remote;
    unsigned int reprioritize : 1;
    unsigned int flush : 1;
    apr_interval_time_t wait_us;
    struct h2_push_diary *push_diary;
    struct h2_stream_monitor *monitor;
    unsigned int open_streams;
    unsigned int streams_done;
    unsigned int responses_submitted;
    unsigned int streams_reset;
    unsigned int pushes_promised;
    unsigned int pushes_submitted;
    unsigned int pushes_reset;
    unsigned int max_stream_errors;
    unsigned int stream_errors;
    apr_size_t frames_received;

入力と出力

  • **入力**: conn_rec（メイン接続）を介して受信した HTTP/2 フレーム。nghttp2 ライブラリによって解析されます。
  • **出力**: h2_mplx を経由してワーカースレッドへリクエストを振り分け、レスポンスを c1（クライアント接続）へ送出します。

前提条件と不変条件

  • ngh2 フィールドは内部的な nghttp2 セッション状態を保持しており、Apache 側の状態（h2_session_state）と同期している必要があります。
  • pool はセッションの生存期間中有効なメモリプールとして使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: h2_session_create により生成され、接続に関連付けられます。 2. **イベント処理**: h2_session_process がメインループとして機能し、nghttp2 を介してフレームを読み込み、イベントをディスパッチします。 3. **多重化**: 受信したリクエストは h2_mplx を通じてワーカースレッドへ送られ、レスポンスは h2_bucket_beam 等を介してメイン接続へ戻されます。 4. **状態管理**: h2_session_event_t を用いたイベント駆動で状態遷移を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c1.c および modules/http2/h2_c1_io.c 内の関数（h2_c1_io_init, h2_c1_read 等）から参照されています。
  • **呼び出し先**: h2_mplx, h2_workers, h2_push_diary, h2_stream_monitor 等のサブコンポーネントを保持し、これらを利用して処理を行います。

関連データ構造

  • h2_mplx: ストリーム多重化器。
  • h2_workers: リクエスト処理用ワーカースレッド管理。
  • h2_session_state: セッション状態の列挙型。
  • h2_session_props: ローカルおよびリモートのセッションプロパティ。
  • nghttp2_session: プロトコル解析用ライブラリの構造体。

失敗モードと境界条件

  • max_stream_errors を超えるストリームエラーが発生した場合、セッションの異常終了が想定されます。
  • out_c1_blocked や ready_to_process キューの状態により、I/O 待ちや処理の遅延が発生する可能性があります。

保守担当者向けメモ

  • h2_session は非常に巨大な構造体です。特定のストリームに関する情報は h2_stream.h を参照してください。
  • nghttp2 の状態と Apache 側の状態の同期が重要です。
  • デバッグ時には H2_SSSN_LOG マクロを活用してください。
