req_headers_in_table

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:327

目的

HTTP リクエストの入力ヘッダー（headers_in）を Lua テーブルに変換し、Lua スクリプトからアクセス可能にするためのブリッジ関数です。

シグネチャまたは形状

static int req_headers_in_table(lua_State *L)

入力と出力

  • **入力**: Lua スタックの第 1 引数に Apache2.Request userdata を受け取ります。
  • **出力**: 変換されたヘッダーを含む Lua テーブルをスタックにプッシュし、戻り値として 1 を返します。

前提条件と不変条件

  • スタックの第 1 引数が ap_lua_check_request_rec によって検証可能な Apache2.Request オブジェクトである必要があります。
  • request_rec の headers_in フィールドが有効な apr_table_t を指している必要があります。

アルゴリズムまたは振る舞い

1. ap_lua_check_request_rec(L, 1) を呼び出し、Lua スタックから request_rec 構造体を取得します。 2. 取得した request_rec 内の headers_in フィールドを req_aprtable2luatable 関数に渡します。 3. req_aprtable2luatable が apr_table_t を Lua テーブルへ変換し、そのテーブルをスタックにプッシュします。 4. 変換結果のテーブル数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_request.c:2833 にて、makefun を通じてディスパッチテーブルに登録されています。
  • **呼び出し先**:
  • ap_lua_check_request_rec (引数の検証と抽出)
  • req_aprtable2luatable (データ変換)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • apr_table_t: Apache のヘッダー情報を保持するテーブル構造体。
  • lua_State: Lua の実行状態。

失敗モードと境界条件

  • ap_lua_check_request_rec が失敗した場合、Lua 側でエラーが発生し、処理が中断されます。
  • headers_in が空の場合、空の Lua テーブルが返されます。

保守担当者向けメモ

  • 本関数は req_headers_out_table や req_err_headers_out_table と同様のパターンで実装されています。
  • メモリ管理は request_rec のプールに依存しているため、明示的な解放は不要です。
