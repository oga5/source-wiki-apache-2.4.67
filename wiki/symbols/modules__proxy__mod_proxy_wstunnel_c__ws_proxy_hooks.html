ws_proxy_hooks

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_wstunnel.c:496

目的

mod_proxy_wstunnel モジュールが提供する各プロキシフックを Apache HTTP Server のフックシステムに登録し、実行順序を制御する。

シグネチャまたは形状

static void ws_proxy_hooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p（フック登録に使用するメモリプール）
  • 出力: なし（void）

前提条件と不変条件

  • mod_proxy_http.c がロードされている場合、本モジュールのフックは mod_proxy_http よりも優先的に実行されるよう aszSucc で指定されている。
  • APR_HOOK_FIRST および APR_HOOK_MIDDLE を使用して、プロキシ処理パイプライン内での適切な実行タイミングが保証されている。

アルゴリズムまたは振る舞い

1. mod_proxy_http.c を後続の処理として定義する。 2. proxy_wstunnel_post_config を ap_hook_post_config に登録。 3. proxy_wstunnel_handler を proxy_hook_scheme_handler に登録（APR_HOOK_FIRST）。 4. proxy_wstunnel_check_trans を proxy_hook_check_trans に登録（APR_HOOK_MIDDLE）。 5. proxy_wstunnel_canon を proxy_hook_canon_handler に登録（APR_HOOK_FIRST）。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/mod_proxy_wstunnel.c:512（モジュールのフック登録処理内）
  • 呼び出し先: ap_hook_post_config, proxy_hook_scheme_handler, proxy_hook_check_trans, proxy_hook_canon_handler

関連データ構造

  • apr_pool_t: メモリ管理用。
  • aszSucc: 成功時の後続モジュール名（mod_proxy_http.c）を保持する静的配列。

失敗モードと境界条件

  • フック登録自体は失敗を返さない設計だが、依存する mod_proxy のフックシステムが初期化されていない環境では登録が無視される可能性がある。

保守担当者向けメモ

  • 本関数はモジュールのロード時に一度だけ実行される。
  • mod_proxy_http との競合を避けるため、フックの優先順位（APR_HOOK_FIRST 等）を変更する場合は、プロキシ全体の処理フローへの影響を十分に検証すること。
