order

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_access_compat.c:97

目的

Apache 設定ファイル内の Order ディレクティブを解析し、ディレクトリごとのアクセス制御における判定順序（allow,deny 等）を構成構造体に設定する。

シグネチャまたは形状

static const char *order(cmd_parms *cmd, void *dv, const char *arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • dv: 現在のディレクトリ設定構造体 (access_compat_dir_conf)。
  • arg: 設定ファイルから渡される順序指定文字列（allow,deny, deny,allow, mutual-failure）。
  • **出力**:
  • 成功時: NULL。
  • 失敗時: エラーメッセージ文字列 "unknown order"。

前提条件と不変条件

  • dv は access_compat_dir_conf 型としてキャスト可能であること。
  • cmd->limited に基づき、現在適用対象となっている HTTP メソッドに対してのみ設定が反映される。

アルゴリズムまたは振る舞い

1. arg を比較し、対応する内部定数（ALLOW_THEN_DENY, DENY_THEN_ALLOW, MUTUAL_FAILURE）を決定する。 2. 指定された文字列が上記いずれにも該当しない場合、エラーメッセージを返して終了する。 3. cmd->limited ビットマスクを走査し、対象となるすべての HTTP メソッドに対して、決定した順序 o を d->order[i] に格納する。 4. 正常終了時は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定解析プロセス（http_config）により、Order ディレクティブが検出された際に呼び出される。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • access_compat_dir_conf: ディレクトリごとの設定を保持する構造体。本関数はこの構造体内の order 配列を更新する。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • **失敗モード**: arg に不正な文字列が指定された場合、"unknown order" というエラー文字列を返す。
  • **境界条件**: cmd->limited が特定のメソッドに限定されている場合、そのメソッドのみが更新され、他のメソッドの設定は維持される。

保守担当者向けメモ

  • 本関数は mod_access_compat の設定ハンドラの一部である。
  • ツールによる find_callers の結果には、Apache 全体で利用される汎用的な用語としての "order" を含む参照が含まれており、本関数への直接的な呼び出し元を示すものではない。
