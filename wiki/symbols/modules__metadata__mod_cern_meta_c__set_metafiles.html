set_metafiles

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_cern_meta.c:185

目的

Apache 設定ファイル内の MetaFiles ディレクティブを処理し、ディレクトリごとの設定構造体 cern_meta_dir_config にメタファイル処理の有効/無効状態を反映する。

シグネチャまたは形状

static const char *set_metafiles(cmd_parms *parms, void *in_dconf, int arg)

入力と出力

  • **入力**:
  • parms: コマンドパラメータ構造体。
  • in_dconf: 現在のディレクトリ設定構造体 (cern_meta_dir_config)。
  • arg: MetaFiles ディレクティブで指定されたフラグ値（真偽値）。
  • **出力**:
  • 戻り値: 常に NULL を返す。

前提条件と不変条件

  • in_dconf は cern_meta_dir_config 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. 引数 in_dconf を cern_meta_dir_config 構造体へキャストする。 2. 構造体内の metafiles メンバに、引数 arg で渡された値を代入する。 3. NULL を返して正常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_cern_meta.c:196 において、AP_INIT_FLAG マクロを通じて設定解析時に呼び出される。
  • **呼び出し先**: 呼び出し先に関する情報は提供されていない。

関連データ構造

  • cern_meta_dir_config: ディレクトリごとの設定を保持する構造体。

失敗モードと境界条件

  • 特筆すべき失敗モードや境界条件はソースに含まれていない。

保守担当者向けメモ

  • 本関数は AP_INIT_FLAG を使用しているため、設定ファイルで MetaFiles On または Off が指定された際に実行される。
  • cern_meta_dir_config 構造体の metafiles メンバを直接更新する単純なセッター関数である。
