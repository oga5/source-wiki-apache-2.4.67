cache_provider

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache.h:99

目的

Apache HTTP Server のキャッシュサブシステムにおいて、ストレージバックエンド（ディスク、共有メモリ等）が実装すべき関数ポインタの集合を定義するインターフェースです。

シグネチャまたは形状

typedef struct {
    int (*remove_entity) (cache_handle_t *h);
    apr_status_t (*store_headers)(cache_handle_t *h, request_rec *r, cache_info *i);
    apr_status_t (*store_body)(cache_handle_t *h, request_rec *r, apr_bucket_brigade *in, apr_bucket_brigade *out);
    apr_status_t (*recall_headers) (cache_handle_t *h, request_rec *r);
    apr_status_t (*recall_body) (cache_handle_t *h, apr_pool_t *p, apr_bucket_brigade *bb);
    int (*create_entity) (cache_handle_t *h, request_rec *r, const char *urlkey, apr_off_t len, apr_bucket_brigade *bb);
    int (*open_entity) (cache_handle_t *h, request_rec *r, const char *urlkey);
    int (*remove_url) (cache_handle_t *h, request_rec *r);
    apr_status_t (*commit_entity)(cache_handle_t *h, request_rec *r);
    apr_status_t (*invalidate_entity)(cache_handle_t *h, request_rec *r);
} cache_provider;

入力と出力

  • **入力**: 各関数ポインタは、キャッシュハンドル (cache_handle_t) やリクエスト情報 (request_rec) を受け取り、ストレージ操作を行います。
  • **出力**: 成功時は APR_SUCCESS または OK (0)、失敗時は APR エラーコードや DECLINED 等のステータスコードを返します。

前提条件と不変条件

  • 各ストレージモジュールは、この構造体の全関数ポインタを適切に実装する必要があります。
  • cache_handle_t は、操作対象となる特定のキャッシュエントリへのアクセスを保持している必要があります。

アルゴリズムまたは振る舞い

  • キャッシュのライフサイクル（生成、検索、読み込み、保存、削除、無効化）を抽象化します。
  • mod_cache はリクエスト処理中にこの構造体を参照し、バックエンドの具体的な実装を意識することなくキャッシュ操作を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cache.c および cache_storage.c が、リクエスト処理パイプラインにおいてこの構造体の関数を呼び出します。
  • **呼び出し先**: mod_cache_disk.c (cache_disk_provider) や mod_cache_socache.c (cache_socache_provider) が、この構造体の実体を定義しています。

関連データ構造

  • cache_handle_t: キャッシュエントリへのアクセスハンドル。
  • cache_info: キャッシュされたコンテンツのメタデータ。
  • request_rec: Apache のリクエスト情報。
  • apr_bucket_brigade: データ転送用のバッファチェーン。

失敗モードと境界条件

  • ストレージへのアクセス失敗（I/O エラー、メモリ不足、キーの不一致など）が発生した場合、各関数は適切なエラーコードを返します。
  • create_entity や open_entity は、対象の URL がキャッシュに存在しない場合や、ロック競合が発生した場合に失敗する可能性があります。

保守担当者向けメモ

  • 新しいキャッシュストレージバックエンドを追加する際は、この構造体を実装し、mod_cache に登録する必要があります。
  • 各関数ポインタの戻り値の型（int vs apr_status_t）に注意して実装してください。
