authz_dbd_cr_cfg

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbd.c:58

目的

ディレクトリ単位の認可設定構造体 authz_dbd_cfg を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *authz_dbd_cr_cfg(apr_pool_t *pool, char *dummy)

入力と出力

  • **入力**:
  • pool: 設定構造体の割り当てに使用するメモリプール。
  • dummy: 使用されない引数（Apache の設定コールバックシグネチャに適合させるためのもの）。
  • **出力**:
  • 初期化された authz_dbd_cfg 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全メンバはゼロ初期化される。
  • 戻り値の redirect メンバは、デフォルト値として -1 に設定される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、authz_dbd_cfg サイズ分のメモリを pool から確保する。 2. 確保した構造体の redirect メンバに -1 を代入する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbd.c:403（Apache の設定システムにより、ディレクトリ設定作成時に呼び出される）。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ関数）。

関連データ構造

  • authz_dbd_cfg: 本関数が初期化する設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性がある（Apache の標準的なメモリ管理動作に準拠）。

保守担当者向けメモ

  • 本関数は Apache の設定作成フックとして機能する。redirect の初期値 -1 は、設定が明示的に行われていない状態を示すフラグとして利用されている。
