ap_mpm_set_max_requests

定義

  • 種別: function
  • 定義箇所: server/mpm_common.c:322

目的

Apache HTTP Server の設定ディレクティブ MaxRequestsPerChild および MaxConnectionsPerChild を処理し、子プロセスが終了するまでのリクエスト上限数（ap_max_requests_per_child）を設定します。

シグネチャまたは形状

const char *ap_mpm_set_max_requests(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: 設定値として渡される文字列（数値）。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時はエラーメッセージ（const char *）を返します。

前提条件と不変条件

  • 本関数は GLOBAL_ONLY コンテキストで呼び出される必要があります。それ以外のコンテキストで呼び出された場合、ap_check_cmd_context によりエラーが返されます。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、グローバル設定として適切か検証します。 2. 設定名が MaxRequestsPerChild である場合、非推奨である旨を APLOG_INFO レベルでログ出力します。 3. arg を atoi で整数に変換し、グローバル変数 ap_max_requests_per_child に代入します。 4. NULL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core.c: AP_INIT_TAKE1 マクロを通じて、MaxRequestsPerChild および MaxConnectionsPerChild ディレクティブのハンドラとして登録されています。
  • **呼び出し先**:
  • ap_check_cmd_context (Apache API)
  • ap_log_error (Apache API)
  • atoi (標準ライブラリ)

関連データ構造

  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。
  • ap_max_requests_per_child: 本関数によって更新されるグローバル変数。

失敗モードと境界条件

  • コンテキストがグローバルでない場合、ap_check_cmd_context がエラー文字列を返し、設定の適用が拒否されます。
  • arg に数値以外の文字列が渡された場合、atoi の仕様に従い 0 が設定されます。

保守担当者向けメモ

  • MaxRequestsPerChild は非推奨となっており、MaxConnectionsPerChild への移行が推奨されています。本関数は両方のディレクティブを同一のロジックで処理します。
