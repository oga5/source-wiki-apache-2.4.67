stream_iter_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_mplx.c:55

目的

HTTP/2 ストリームの反復処理（イテレーション）において、現在の多重化コンテキスト、対象ストリーム、時刻、および処理済みカウントを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    h2_mplx *m;
    h2_stream *stream;
    apr_time_t now;
    apr_size_t count;
} stream_iter_ctx;

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは以下の通りです。

  • m: 関連する h2_mplx インスタンスへのポインタ。
  • stream: 現在反復中の h2_stream へのポインタ。
  • now: 反復処理開始時の apr_time_t 形式の現在時刻。
  • count: 反復処理における現在のカウント値。

前提条件と不変条件

  • m は有効な h2_mplx インスタンスを指している必要があります。
  • stream は反復処理の文脈に応じて適切に設定されている必要があります。

アルゴリズムまたは振る舞い

本構造体自体は状態を保持するコンテナとして機能します。反復処理の過程で count がインクリメントされるなど、イテレータの現在の状態を追跡するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数名は、提供されたソース抜粋からは特定できません。

関連データ構造

  • h2_mplx: 多重化器のメイン構造体。
  • h2_stream: 個別の HTTP/2 ストリームを表現する構造体。

失敗モードと境界条件

  • ポインタ m や stream が NULL の状態で参照された場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は h2_mplx.c 内のストリーム反復処理専用のコンテキストです。
  • 構造体のフィールドを変更する場合は、このコンテキストを使用しているすべての反復処理ロジックに影響が及ぶため注意してください。
