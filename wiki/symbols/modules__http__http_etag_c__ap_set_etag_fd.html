ap_set_etag_fd

定義

  • 種別: function
  • 定義箇所: modules/http/http_etag.c:391

目的

指定されたファイル記述子（apr_file_t）に基づき、HTTP の ETag ヘッダーを生成してレスポンスヘッダーに設定する。

シグネチャまたは形状

void ap_set_etag_fd(request_rec *r, apr_file_t *fd)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • fd: ETag 生成の対象となるファイル記述子（apr_file_t）。
  • **出力**: なし（副作用として r->headers_out または r->notes を更新）。

前提条件と不変条件

  • r->finfo に有効なファイル情報が格納されている必要がある。
  • 生成された ETag 文字列は r->pool から割り当てられる。
  • etag_bits 設定が ETAG_NONE の場合、ETag は生成されない。

アルゴリズムまたは振る舞い

1. etag_rec 構造体を初期化し、リクエスト情報（vlist_validator, request_time, finfo）および引数の fd をセットする。 2. ap_make_etag_ex を呼び出し、設定に基づいた ETag 文字列を生成する。 3. 生成された文字列が空でない場合、apr_table_setn を使用して r->headers_out に ETag ヘッダーを追加する。 4. 生成に失敗または省略された場合、r->notes に no-etag キーで omit を設定し、後続のフィルターが ETag を付与しないよう制御する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/test/mod_dialup.c:174
  • server/core.c:4981
  • **呼び出し先**:
  • ap_make_etag_ex (ETag 文字列生成)
  • apr_table_setn (ヘッダーまたはノートへの格納)

関連データ構造

  • request_rec: リクエスト全体のコンテキスト。
  • etag_rec: ap_make_etag_ex に渡すための ETag 生成用パラメータ構造体。
  • apr_file_t: ファイル操作のための APR 抽象化ハンドル。

失敗モードと境界条件

  • ap_make_etag_ex が ETag を生成できない場合、ETag ヘッダーは設定されず、代わりに no-etag ノートが設定される。
  • ファイル情報が不完全な場合、生成される ETag の強度が低下する可能性がある。

保守担当者向けメモ

  • 本関数は ap_set_etag と異なり、ファイル記述子を直接扱うため、既に開かれているファイルに対して効率的に ETag を生成する際に使用する。
  • r->pool を使用しているため、メモリリークの懸念はない。
  • make_digest_etag が内部で呼び出される場合、ファイル内容の読み込みが発生するため、大きなファイルではパフォーマンスに影響を与える可能性がある。
