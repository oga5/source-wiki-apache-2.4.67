ap_lua_finfo

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_vmprep.h:107

目的

Lua スクリプトファイルのメタデータ（実行回数、最終更新日時、ファイルサイズ）を保持し、キャッシュの有効性判定や管理を行うための構造体です。

シグネチャまたは形状

typedef struct {
    apr_size_t runs;
    apr_time_t modified;
    apr_off_t  size;
} ap_lua_finfo;

入力と出力

  • **入力**: 構造体の各フィールドは、主にファイルシステムから取得した情報や、実行統計に基づいて設定されます。
  • **出力**: 本構造体は、Lua VM のキャッシュ管理ロジックにおいて、スクリプトの再読み込みが必要かどうかを判断するための参照情報として利用されます。

前提条件と不変条件

  • apr_pool_t を介してメモリが割り当てられることを前提としています。
  • modified は apr_time_t 型であり、APR の時間管理規約に従います。

アルゴリズムまたは振る舞い

  • mod_lua において、スクリプトの実行履歴（runs）を追跡し、ファイルシステム上の属性（modified, size）と比較することで、キャッシュされた Lua VM が最新の状態であるかを検証する際に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_vmprep.c 内の VM 管理ロジック（ap_lua_get_lua_state 等）が、キャッシュ情報の初期化や比較のために本構造体をインスタンス化・参照します。
  • **呼び出し先**: 直接的な関数呼び出しは行いません。データ保持用の構造体です。

関連データ構造

  • ap_lua_vm_spec: Lua VM の生成パラメータを保持する構造体であり、この中で ap_lua_finfo 型のポインタが保持されます。

失敗モードと境界条件

  • メモリ割り当て失敗: apr_pcalloc が失敗した場合、呼び出し元の処理が中断される可能性があります。
  • ファイル情報取得失敗: ファイルシステムから modified や size を取得できない場合、キャッシュの整合性判定が正しく機能しない可能性があります。

保守担当者向けメモ

  • 本構造体は lua_vmprep.c で頻繁に参照されるため、フィールドの追加や変更を行う際は、キャッシュ判定ロジックへの影響を考慮してください。
  • runs フィールドは、スクリプトの実行頻度に基づく最適化や統計収集に利用される可能性があります。
