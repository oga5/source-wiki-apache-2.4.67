ShowNotifyIcon

定義

  • 種別: function
  • 定義箇所: support/win32/ApacheMonitor.c:319

目的

Windows タスクトレイ上の Apache Monitor アイコンの表示状態（アイコン画像およびツールチップ）を更新または削除する。

シグネチャまたは形状

static VOID ShowNotifyIcon(HWND hWnd, DWORD dwMessage)

入力と出力

  • **入力**:
  • hWnd: アイコンに関連付けられたウィンドウハンドル。
  • dwMessage: Shell_NotifyIcon に渡すメッセージ（NIM_ADD, NIM_MODIFY, NIM_DELETE 等）。
  • **出力**: なし（VOID）。

前提条件と不変条件

  • グローバル変数 g_stServices が監視対象のサービス情報を保持していること。
  • グローバル変数 g_icoRun および g_icoStop が有効なアイコンハンドルとして初期化されていること。
  • g_lpMsg にツールチップ用のメッセージリソースが格納されていること。

アルゴリズムまたは振る舞い

1. NOTIFYICONDATA 構造体を初期化し、コールバックメッセージとして WM_TRAYMESSAGE を設定する。 2. g_stServices を走査し、実行中（dwPid != 0）のサービス数 n と全サービス数 i をカウントする。 3. dwMessage が NIM_DELETE でない場合、実行中のサービス数に応じてアイコンを切り替える（すべて実行中なら g_icoRun、停止中があれば g_icoStop）。 4. サービスの状態に基づき、ツールチップ文字列（nid.szTip）を動的に生成する。

  • すべて実行中、一部実行中、すべて停止中、サービスなしの各ケースに対応。

    • 5. Shell_NotifyIcon を呼び出し、設定を OS に反映させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/win32/ApacheMonitor.c 内の各所（NIM_ADD, NIM_MODIFY, NIM_DELETE を引数として呼び出し）。
  • **呼び出し先**:
  • Shell_NotifyIcon (Windows API)

関連データ構造

  • NOTIFYICONDATA: Windows シェル通知用構造体。
  • ST_APACHE_SERVICE: サービス状態を保持するグローバル配列 g_stServices の要素型。

失敗モードと境界条件

  • g_stServices が空の場合、ツールチップには「サービスなし」のメッセージが設定される。
  • dwMessage が NIM_DELETE の場合、アイコンは NULL に設定され、ツールチップの更新は行われない。
  • サービス数が MAX_APACHE_SERVICES を超える場合、配列の固定長制限により正しくカウントされない可能性がある。

保守担当者向けメモ

  • ツールチップの文字列生成には _sntprintf を使用しており、バッファオーバーフローを防止している。
  • サービス状態の更新は g_stServices の内容に依存するため、本関数を呼び出す前に GetApacheServicesStatus 等で最新の状態を取得しておく必要がある。
