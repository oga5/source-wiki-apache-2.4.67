md_acme_authz_state_t

定義

  • 種別: enum
  • 定義箇所: modules/md/md_acme_authz.h:38

目的

ACME プロトコルにおけるドメイン認証（Authorization）の現在のステータスを表現するために使用されます。

シグネチャまたは形状

typedef enum {
    MD_ACME_AUTHZ_S_UNKNOWN,
    MD_ACME_AUTHZ_S_PENDING,
    MD_ACME_AUTHZ_S_VALID,
    MD_ACME_AUTHZ_S_INVALID,
} md_acme_authz_state_t;

入力と出力

  • 入力: ACME サーバーから取得した認証リソースの JSON レスポンス。
  • 出力: 認証プロセスの進行状況を示す列挙値。

前提条件と不変条件

  • 証明書発行オーダーが進行するためには、認証状態が MD_ACME_AUTHZ_S_VALID である必要があります。

アルゴリズムまたは振る舞い

  • md_acme_authz_update() 関数により、サーバーから取得した最新の認証情報に基づき、この列挙値が更新されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_acme_authz.h 内の md_acme_authz_t 構造体において、state フィールドの型として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • md_acme_authz_t: 認証リソースの全体を管理する構造体。本列挙型はこの構造体の一部として保持されます。

失敗モードと境界条件

  • MD_ACME_AUTHZ_S_INVALID: 認証プロセスが失敗したことを示します。
  • MD_ACME_AUTHZ_S_UNKNOWN: サーバーからの応答が解析できない、または初期状態であることを示します。

保守担当者向けメモ

  • 本列挙型は ACME 認証のライフサイクル管理に直結しています。新しい認証状態が ACME 仕様に追加された場合は、この列挙型を拡張する必要があります。
