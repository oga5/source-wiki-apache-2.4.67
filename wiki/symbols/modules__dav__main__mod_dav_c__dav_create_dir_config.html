dav_create_dir_config

定義

  • 種別: function
  • 定義箇所: modules/dav/main/mod_dav.c:150

目的

Apache HTTP Server の WebDAV モジュールにおいて、ディレクトリごとの設定構造体（dav_dir_conf）を初期化し、メモリを割り当てる。

シグネチャまたは形状

void *dav_create_dir_config(apr_pool_t *p, char *dir)

入力と出力

  • **入力**:
  • p: 設定の割り当てに使用する APR メモリプール。
  • dir: 設定対象のディレクトリパス（NULL の場合はデフォルト設定を作成）。
  • **出力**:
  • 初期化された dav_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • dir が指定された場合、末尾のスラッシュは除去されて格納される。
  • apr_pcalloc により、構造体内の全メンバはゼロ初期化される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して dav_dir_conf サイズのメモリを確保する。 2. dir 引数が NULL でない場合、文字列を複製し、末尾にスラッシュが存在すればそれを削除（\0 で置換）して conf->dir に格納する。 3. 初期化済みの conf ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c:5229 （ディレクトリ設定作成ハンドラとして登録）。
  • **呼び出し先**: apr_pcalloc, apr_pstrmemdup, strlen （すべて APR/標準ライブラリ）。

関連データ構造

  • dav_dir_conf: WebDAV のディレクトリ設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時は apr_pcalloc が例外を発生させる可能性がある。
  • dir がルートディレクトリ（"/"）の場合、末尾のスラッシュ除去処理はスキップされる。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。
  • dir 引数の処理において、パスの正規化は末尾のスラッシュ除去のみに限定されている点に注意すること。
