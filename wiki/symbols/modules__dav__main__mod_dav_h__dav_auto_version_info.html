dav_auto_version_info

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:2199

目的

WebDAV の自動バージョン管理処理において、リソースや親リソースが自動的にバージョン管理下に入ったか、あるいはチェックアウトされたかという状態を追跡・保持するために使用されます。

シグネチャまたは形状

typedef struct {
    int resource_versioned;        /* 1 => リソースが自動バージョン管理された */
    int resource_checkedout;       /* 1 => リソースが自動チェックアウトされた */
    int parent_checkedout;         /* 1 => 親リソースが自動チェックアウトされた */
    dav_resource *parent_resource; /* 必要に応じて保持される親リソース */
} dav_auto_version_info;

入力と出力

  • 入力: 自動バージョン管理機能を持つ関数（dav_auto_checkin 等）において、現在のリソース状態を記録するために使用されます。
  • 出力: 処理後の状態を保持し、後続のコミットやエラーハンドリング時に参照されます。

前提条件と不変条件

  • 各フラグ（resource_versioned, resource_checkedout, parent_checkedout）は、論理値（0 または 1）として扱われます。
  • parent_resource は、親リソースの自動チェックアウトが必要な場合にのみ設定されます。

アルゴリズムまたは振る舞い

WebDAV メソッド（PUT や PROPPATCH 等）の実行過程で、リソースがバージョン管理下にある必要がある場合、この構造体を用いて現在の変更状態を記録します。これにより、処理の成功後に適切にチェックインを行う、あるいはエラー発生時にロールバックを行うための情報源となります。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/main/mod_dav.c 内の複数の箇所（例: 992, 1274, 2435, 2640, 2762, 3466, 4735 行目）でローカル変数として宣言・利用されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しの引数として dav_auto_checkin 等のユーティリティ関数へ渡されます。

関連データ構造

  • dav_resource: 本構造体内で親リソースの参照を保持するために使用されます。

失敗モードと境界条件

  • 初期化されていない状態で参照された場合、フラグの値が不定となる可能性があります。ソースコード上では { 0 } による初期化が行われている箇所があります。

保守担当者向けメモ

  • 本構造体は mod_dav の自動バージョン管理ロジックにおいて不可欠な状態保持コンテキストです。
  • dav_resource へのポインタを保持しているため、メモリ管理（apr_pool_t）のライフサイクルに注意してください。
