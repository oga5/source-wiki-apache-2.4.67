isapi_cmd_cachefile

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_isapi.c:121

目的

Apache 設定ファイル内の ISAPICacheFile ディレクティブを処理し、指定された ISAPI DLL を事前にロードしてキャッシュに登録する。

シグネチャまたは形状

static const char *isapi_cmd_cachefile(cmd_parms *cmd, void *dummy, const char *filename)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • filename: キャッシュ対象の DLL ファイルパス。
  • **出力**:
  • 成功時は NULL を返す。
  • エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • filename はサーバーのルートディレクトリからの相対パスとして解決される。
  • 指定されたパスは、ファイルシステム上でアクセス可能かつ「通常のファイル（regular file）」である必要がある。

アルゴリズムまたは振る舞い

1. ap_server_root_relative を使用して、filename を絶対パスに変換する。 2. apr_stat を呼び出し、ファイルが存在し、かつ通常のファイルであることを確認する。 3. isapi_lookup を呼び出し、該当 DLL をキャッシュにロードする。この際、リクエストコンテキスト（request_rec）は NULL として渡される。 4. 処理の各段階でエラーが発生した場合は、ap_log_error を通じて警告ログを出力し、処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: AP_INIT_ITERATE マクロにより、設定解析時に ISAPICacheFile ディレクティブから呼び出される（modules/arch/win32/mod_isapi.c:187）。
  • **呼び出し先**:
  • ap_server_root_relative
  • apr_stat
  • isapi_lookup
  • ap_log_error

関連データ構造

  • cmd_parms: 設定コマンドのコンテキスト情報。
  • isapi_loaded: DLL のロード状態を保持するキャッシュ構造体。

失敗モードと境界条件

  • パスが不正な場合: ap_log_error で APR_EBADPATH を記録し、NULL を返す。
  • ファイルが存在しない、または通常のファイルでない場合: 警告ログを記録し、NULL を返す。
  • isapi_lookup によるロードに失敗した場合: 警告ログを記録し、NULL を返す。

保守担当者向けメモ

  • 本関数は ISAPICacheFile ディレクティブのハンドラであり、グローバルなキャッシュテーブルへの登録を目的としている。
  • 現在の設計ではグローバルなキャッシュに登録されるが、コメントにある通り、将来的にはディレクトリごとのスコープで相対パスを解決できるような拡張が検討されている。
