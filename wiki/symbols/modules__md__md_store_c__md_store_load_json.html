md_store_load_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_store.c:106

目的

ストレージから指定されたグループおよび名称に関連付けられたJSONデータを読み込み、md_json_t 構造体として取得するためのラッパー関数です。

シグネチャまたは形状

apr_status_t md_store_load_json(md_store_t *store, md_store_group_t group, 
                                const char *name, const char *aspect, 
                                struct md_json_t **pdata, apr_pool_t *p);

入力と出力

  • **入力**:
  • store: ストレージの抽象インターフェースハンドル。
  • group: 読み込み対象のストレージグループ（例: MD_SG_ACCOUNTS）。
  • name: 対象エントリの名称。
  • aspect: 読み込むデータの側面（ファイル名や属性）。
  • p: メモリ割り当てに使用するAPRプール。
  • **出力**:
  • pdata: 読み込まれたJSONデータへのポインタを格納するアドレス。
  • 戻り値: APR_SUCCESS（成功時）、またはAPRエラーコード。

前提条件と不変条件

  • store は有効な md_store_t インスタンスである必要があります。
  • pdata は NULL でない有効なポインタである必要があります。
  • 内部で md_store_load を呼び出すため、store に設定された具体的なストレージ実装が MD_SV_JSON 形式の読み込みをサポートしている必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取ったパラメータをそのまま md_store_load 関数へ転送します。 2. その際、データ形式として MD_SV_JSON を指定し、pdata を void** にキャストして渡します。 3. md_store_load の実行結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_acct.c
  • modules/md/md_acme_order.c
  • modules/md/md_ocsp.c
  • modules/md/md_status.c
  • modules/md/md_store.c (内部の md_load 等から参照)
  • **呼び出し先**:
  • md_store_load (本モジュールのストレージ抽象化関数)

関連データ構造

  • md_store_t: ストレージの抽象化ハンドル。
  • md_json_t: JSONデータを保持する構造体。
  • md_store_group_t: ストレージグループを識別する列挙型。

失敗モードと境界条件

  • 指定された name や aspect に対応するファイルが存在しない場合、md_store_load からエラーが返されます。
  • JSONのパースに失敗した場合、またはストレージアクセス権限がない場合、APRエラーコードが返されます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のI/Oロジックは md_store_load およびその実装（md_store_fs.c 等）に依存しています。
  • 呼び出し元で pdata に NULL を渡すことで、データの存在確認のみを行うケース（md_store.c:182 等）が存在します。
