ap_cookie_read

定義

  • 種別: function
  • 定義箇所: server/util_cookies.c:245

目的

HTTPリクエストヘッダー（Cookie および Cookie2）から特定のクッキー値を検索・抽出し、必要に応じて元のヘッダーから該当クッキーを削除します。

シグネチャまたは形状

apr_status_t ap_cookie_read(request_rec *r, const char *name, const char **val, int remove);

入力と出力

  • **入力**:
  • r: リクエスト状態を保持する request_rec 構造体。
  • name: 検索対象のクッキー名。
  • remove: 真の場合、抽出したクッキーをリクエストヘッダーから削除します。
  • **出力**:
  • val: 抽出されたクッキー値が格納されるポインタ（見つからない場合は NULL）。
  • 戻り値: 成功時は APR_SUCCESS、重複検出時は APR_EGENERAL、その他エラー時は apr_status_t 型のステータスコード。

前提条件と不変条件

  • すべてのメモリ操作は r->pool を使用するため、呼び出し側での明示的な解放は不要です。
  • クッキー名に不正な文字が含まれていないことが期待されます。

アルゴリズムまたは振る舞い

1. 内部構造体 ap_cookie_do を初期化し、検索対象のクッキー名と一時的なテーブルを準備します。 2. apr_table_do を介して extract_cookie_line コールバックを呼び出し、headers_in 内の Cookie および Cookie2 ヘッダーを走査します。 3. クッキーの重複が検出された場合、エラーログを出力し APR_EGENERAL を返します。 4. remove が真の場合、元のヘッダーから Cookie/Cookie2 を削除し、抽出済みのクッキーを除外した新しいテーブルでヘッダーを再構築します。 5. 抽出した値を val に格納して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_request.c
  • modules/session/mod_session_cookie.c
  • modules/session/mod_session_dbd.c
  • **呼び出し先**:
  • extract_cookie_line (コールバック)
  • apr_table_do, apr_table_make, apr_table_unset, apr_table_overlay (APRライブラリ)
  • ap_log_rerror (Apacheログ出力)

関連データ構造

  • request_rec: リクエストの全状態を保持。
  • ap_cookie_do: 抽出処理中の状態（クッキー名、値、重複フラグ、一時テーブル）を保持する内部構造体。

失敗モードと境界条件

  • **重複クッキー**: 同一のクッキー名が複数回出現した場合、セキュリティ上の曖昧さを避けるため APR_EGENERAL を返します。
  • **メモリ不足**: APRのテーブル操作関数がメモリ不足に陥った場合、APRの規約に従ったエラーが返されます。

保守担当者向けメモ

  • 本関数は Cookie および Cookie2 ヘッダーを対象とします。
  • 重複クッキーの検出はセキュリティ要件に基づく設計であるため、変更時は注意してください。
  • remove オプション使用時は、ヘッダーテーブルの再構築が行われるため、後続のフィルタやモジュールが参照するヘッダー内容が変化することに留意してください。
