ssl_rand_feedfp

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_rand.c:137

目的

ファイルまたはコマンド出力からデータを読み込み、OpenSSL の擬似乱数生成器（PRNG）へシードとして供給する汎用ヘルパー関数です。

シグネチャまたは形状

static int ssl_rand_feedfp(apr_pool_t *p, apr_file_t *fp, int nReq)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール（未使用）。
  • fp: 読み込み対象のファイルハンドル。
  • nReq: 読み込むバイト数（0以下の場合はEOFまで読み込む）。
  • **出力**:
  • int: 実際に読み込み、RAND_seed に供給した合計バイト数。

前提条件と不変条件

  • fp は有効な apr_file_t ポインタである必要があります。
  • BUFSIZE（8192バイト）単位でデータを処理します。

アルゴリズムまたは振る舞い

1. nReq が指定されている場合、そのバイト数に達するまで、あるいは nReq が指定されていない場合は EOF に達するまでループを実行します。 2. apr_file_read を使用して caBuf（BUFSIZE）へデータを読み込みます。 3. 読み込みに成功した場合、RAND_seed(caBuf, nBuf) を呼び出し、OpenSSL の内部状態へデータを注入します。 4. 読み込んだバイト数を累積し、nReq が指定されている場合は残りの必要バイト数を減算します。 5. 読み込み失敗または要求量に達した時点でループを終了し、合計バイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_rand.c:66
  • modules/ssl/ssl_engine_rand.c:81
  • **呼び出し先**:
  • apr_file_read (APR API)
  • RAND_seed (OpenSSL API)

関連データ構造

  • apr_file_t: APR のファイルハンドル構造体。
  • BUFSIZE: 読み込み用バッファサイズ（8192バイト）。

失敗モードと境界条件

  • apr_file_read が APR_SUCCESS 以外の値を返した場合、ループを中断し、それまでに読み込んだバイト数を返します。
  • nReq が 0 の場合、ファイルが EOF に達するまで読み込みを継続します。

保守担当者向けメモ

  • 本関数は ssl_rand_seed から呼び出され、設定されたエントロピー源（ファイルやコマンド出力）を処理します。
  • 現代の OpenSSL では OS のエントロピー源が自動利用されるため、本機能の重要性は相対的に低下していますが、レガシーな環境や特定のセキュリティ要件下でのシード供給を担っています。
