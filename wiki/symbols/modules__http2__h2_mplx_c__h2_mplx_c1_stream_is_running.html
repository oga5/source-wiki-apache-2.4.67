h2_mplx_c1_stream_is_running

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_mplx.c:119

目的

指定された HTTP/2 ストリームが現在実行中であるかどうかを、多重化器（h2_mplx）の排他制御下で判定します。

シグネチャまたは形状

int h2_mplx_c1_stream_is_running(h2_mplx *m, h2_stream *stream)

入力と出力

  • **入力**:
  • m: 多重化器インスタンスへのポインタ。
  • stream: 状態を確認する対象のストリームへのポインタ。
  • **出力**:
  • int: ストリームが実行中であれば非ゼロ（真）、そうでなければ 0（偽）を返します。

前提条件と不変条件

  • m は有効な h2_mplx インスタンスである必要があります。
  • 本関数は内部で H2_MPLX_ENTER(m) および H2_MPLX_LEAVE(m) を呼び出し、ミューテックスによる排他制御を行います。

アルゴリズムまたは振る舞い

1. H2_MPLX_ENTER(m) を呼び出し、多重化器のミューテックスを取得します。 2. 内部関数 stream_is_running(stream) を呼び出し、ストリームの状態を確認します。 3. H2_MPLX_LEAVE(m) を呼び出し、ミューテックスを解放します。 4. 取得した判定結果を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:1715
  • modules/http2/h2_stream.c:1847
  • **呼び出し先**:
  • stream_is_running (同ファイル内の静的関数)

関連データ構造

  • h2_mplx: 多重化器のメイン構造体。
  • h2_stream: HTTP/2 ストリームのコンテキスト。

失敗モードと境界条件

  • m が NULL の場合、H2_MPLX_ENTER マクロの実装に依存しますが、通常はセグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は h2_mplx のロックを管理するラッパーです。ストリームの状態判定ロジック自体は stream_is_running に依存しています。
  • 排他制御が必要なコンテキストから呼び出されることを前提としています。
