ssl_rsctx_t

定義

  • 種別: enum
  • 定義箇所: modules/ssl/ssl_private.h:542

目的

乱数生成器（PRNG）のシード処理を行う際、その実行タイミング（コンテキスト）を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    SSL_RSCTX_STARTUP = 1,
    SSL_RSCTX_CONNECT = 2
} ssl_rsctx_t;

入力と出力

  • **入力**: 乱数シード処理関数（ssl_rand_seed）の引数として渡されます。
  • **出力**: 列挙値 SSL_RSCTX_STARTUP または SSL_RSCTX_CONNECT のいずれか。

前提条件と不変条件

  • SSL_RSCTX_STARTUP: サーバー起動時の初期化フェーズで使用されることを想定しています。
  • SSL_RSCTX_CONNECT: クライアント接続時の動的なシード更新フェーズで使用されることを想定しています。

アルゴリズムまたは振る舞い

この列挙型は、ssl_rand_seed 関数において、どのタイミングで乱数源からエントロピーを収集すべきかを制御するためのフラグとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_rand.c 内の ssl_rand_seed 関数がこの型を引数として受け取ります。
  • **呼び出し先**: 本シンボル自体はデータ型であり、関数を呼び出すことはありません。

関連データ構造

  • ssl_rand_seed 関数（modules/ssl/ssl_engine_rand.c:41 および modules/ssl/ssl_private.h:1092 で宣言）で使用されます。

失敗モードと境界条件

  • 定義された値（1 または 2）以外の整数値が渡された場合の挙動は、ssl_rand_seed 関数の実装に依存します。

保守担当者向けメモ

  • 本シンボルは mod_ssl の乱数生成ロジックの制御に不可欠です。新しいシードタイミングを追加する場合は、この列挙型を拡張し、ssl_rand_seed 側の条件分岐を更新する必要があります。
