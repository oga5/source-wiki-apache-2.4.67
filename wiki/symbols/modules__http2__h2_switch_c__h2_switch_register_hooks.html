h2_switch_register_hooks

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_switch.c:228

目的

Apache HTTP Server のフックシステムに対し、HTTP/2 プロトコル切り替えに必要なハンドラ関数を登録する。

シグネチャまたは形状

void h2_switch_register_hooks(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として Apache のフックテーブルを更新）

前提条件と不変条件

  • 本関数はモジュールの初期化フェーズで呼び出される必要がある。
  • 登録される各ハンドラ関数（h2_protocol_propose, h2_protocol_switch, h2_protocol_get）が同一モジュール内で定義されていること。

アルゴリズムまたは振る舞い

以下の Apache フックを APR_HOOK_MIDDLE の優先度で登録する。 1. ap_hook_protocol_propose: プロトコル提案用ハンドラ h2_protocol_propose を登録。 2. ap_hook_protocol_switch: プロトコル切り替え用ハンドラ h2_protocol_switch を登録。 3. ap_hook_protocol_get: プロトコル取得用ハンドラ h2_protocol_get を登録。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/mod_http2.c:201
  • 呼び出し先:
  • ap_hook_protocol_propose
  • ap_hook_protocol_switch
  • ap_hook_protocol_get
  • h2_protocol_propose (参照)
  • h2_protocol_switch (参照)
  • h2_protocol_get (参照)

関連データ構造

  • modules/http2/h2_switch.h (宣言)

失敗モードと境界条件

  • フック登録関数（ap_hook_*）自体は戻り値を返さないため、登録失敗時のエラーハンドリングは本関数内では行われない。

保守担当者向けメモ

  • 本関数は mod_http2 の初期化プロセスの一部である。フックの優先度を変更する必要がある場合は、APR_HOOK_MIDDLE の引数を調整すること。
