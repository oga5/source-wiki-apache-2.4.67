anon_auth_user

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authn_anon.c:59

目的

匿名認証を許可するユーザー名のリストを保持するための連結リスト構造体です。

シグネチャまたは形状

typedef struct anon_auth_user {
    const char *user;
    struct anon_auth_user *next;
} anon_auth_user;

入力と出力

  • user: 許可対象となるユーザー名（文字列）。
  • next: リスト内の次の要素へのポインタ。

前提条件と不変条件

  • next はリストの終端において NULL である必要があります。
  • user は設定ファイルから読み込まれた文字列を指します。

アルゴリズムまたは振る舞い

  • 匿名認証の許可ユーザーリストを単方向連結リストとして管理します。
  • 認証時にこのリストを走査し、入力されたユーザー名がリスト内に存在するかを判定します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/aaa/mod_authn_anon.c 内の認証ロジック（check_anonymous 等）および設定処理において、conf->users を通じて参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しは含みません。

関連データ構造

  • authn_anon_config_rec: この構造体のリストの先頭（users）を保持するディレクトリ設定構造体です。

失敗モードと境界条件

  • リストが空の場合、匿名認証は許可されません。
  • ユーザー名の比較は、モジュールの仕様に基づき大文字小文字を区別せずに行われます。

保守担当者向けメモ

  • 本構造体は mod_authn_anon.c 内でのみ使用されます。
  • リストのメモリ管理は Apache の設定プール（pool）に依存しているため、個別の解放処理は不要です。
