nt_service_ctx_t

定義

  • 種別: struct
  • 定義箇所: server/mpm/winnt/service.c:55

目的

Windowsサービスとして実行されるApache HTTP Serverのライフサイクル管理に必要な、スレッドハンドル、同期オブジェクト、およびサービス状態情報を保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct nt_service_ctx_t
{
    HANDLE mpm_thread;       /* Apacheサーバーのメインスレッドハンドル */
    HANDLE service_thread;   /* サービス/監視スレッドハンドル */
    DWORD  service_thread_id;/* サービス/監視スレッドID */
    HANDLE service_init;     /* コントローラースレッド初期化用ミューテックス */
    HANDLE service_term;     /* NTサービススレッド終了シグナル */
    SERVICE_STATUS ssStatus; /* SCMに報告するサービス状態 */
    SERVICE_STATUS_HANDLE hServiceStatus; /* SCMへの状態通知用ハンドル */
} nt_service_ctx_t;

入力と出力

本構造体は関数引数として渡され、SCM（サービスコントロールマネージャ）との通信状態やスレッドの生存確認に使用されます。直接的な入力/出力関数はありませんが、ReportStatusToSCMgr 等の関数が本構造体のメンバを参照してSCMへ情報を伝達します。

前提条件と不変条件

  • hServiceStatus は RegisterServiceCtrlHandlerEx によって取得された有効なハンドルである必要があります。
  • service_init および service_term は、スレッド間の同期および終了制御のために適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • サービス起動時に globdat（nt_service_ctx_t型の静的インスタンス）が初期化されます。
  • service_nt_ctrl や service_nt_main_fn_w などの関数内で、この構造体を通じてサービスの状態遷移（SERVICE_RUNNING 等）や終了処理が管理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/service.c 内の各サービス管理関数（service_nt_ctrl, service_nt_main_fn_w 等）が、グローバル変数 globdat を参照または引数として受け取ります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • SERVICE_STATUS: Windows APIで定義されたサービス状態構造体。
  • SERVICE_STATUS_HANDLE: SCMとの通信用ハンドル。
  • globdat: nt_service_ctx_t 型の静的インスタンス（server/mpm/winnt/service.c:66）。

失敗モードと境界条件

  • hServiceStatus が無効な場合、SCMへの状態報告が失敗します。
  • スレッドハンドルが適切にクローズされない場合、リソースリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は server/mpm/winnt/service.c 内でグローバルな状態管理に使用されています。
  • サービスのスレッド管理に関わるため、マルチスレッド環境下でのアクセスには注意が必要です。
  • SERVICE_STATUS メンバの更新は、SCMの期待するタイミングと同期している必要があります。
