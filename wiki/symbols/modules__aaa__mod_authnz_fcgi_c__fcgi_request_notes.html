fcgi_request_notes

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:57

目的

認証（Authn）フェーズで成功したプロバイダー名を記録し、その後の認可（Authz）フェーズで参照するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct {
    const char *successful_authnz_provider;
} fcgi_request_notes;

入力と出力

  • **入力**: 認証成功時にプロバイダー名（文字列）が格納されます。
  • **出力**: 認可フェーズにおいて、どのプロバイダーが認証を行ったかを特定するために読み取られます。

前提条件と不変条件

  • リクエストごとのプール（r->pool）からメモリが割り当てられることを前提としています。
  • 認証が成功した後にのみ、有効なプロバイダー名が設定されます。

アルゴリズムまたは振る舞い

1. 認証成功時、apr_palloc を用いてリクエストプール上に本構造体を確保し、successful_authnz_provider にプロバイダー名を保存します。 2. 認可フェーズにおいて、ap_get_module_config を介してリクエスト設定から本構造体を取得し、認証済みプロバイダーを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c 内の認証および認可処理関数（900, 976, 1035 行付近）から参照・生成されています。
  • **呼び出し先**:
  • 特定の関数呼び出し関係はソース抜粋からは不明です。

関連データ構造

  • request_rec (Apache リクエストオブジェクト): 本構造体をリクエスト設定として保持するために使用されます。

失敗モードと境界条件

  • 認証が成功していない場合、successful_authnz_provider は未設定または NULL となる可能性があります。
  • メモリ割り当て失敗時は apr_palloc の挙動に従い、Apache のエラー処理へ移行します。

保守担当者向けメモ

  • 本構造体は mod_authnz_fcgi モジュール内での認証・認可フェーズ間の状態共有に特化しています。
  • 構造体へのアクセスは ap_get_module_config を通じて行われるため、リクエスト設定のライフサイクルと整合していることを確認してください。
