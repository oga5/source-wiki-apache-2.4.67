md_timeperiod_contains

定義

  • 種別: function
  • 定義箇所: modules/md/md_time.c:31

目的

指定された時刻が、特定の期間（md_timeperiod_t）内に含まれているかどうかを判定します。

シグネチャまたは形状

int md_timeperiod_contains(const md_timeperiod_t *period, apr_time_t time)

入力と出力

  • **入力**:
  • period: 判定対象となる期間構造体へのポインタ。
  • time: 判定対象となる apr_time_t 形式の時刻。
  • **出力**:
  • int: 時刻が期間内に含まれる場合は非ゼロ（真）、含まれない場合は 0（偽）を返します。

前提条件と不変条件

  • period は有効な md_timeperiod_t 構造体を指している必要があります。
  • 期間の開始時刻と終了時刻の整合性については、本関数内では検証されません。

アルゴリズムまたは振る舞い

以下の論理積（AND）条件を満たす場合に真を返します。 1. md_timeperiod_has_started が真であること（時刻が期間の開始以降である）。 2. md_timeperiod_has_ended が偽であること（時刻が期間の終了に達していない）。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • md_timeperiod_has_started
  • md_timeperiod_has_ended
  • **呼び出し元**:
  • modules/md/md_time.h:34 に宣言が存在します。具体的な呼び出し元のコードベース内での利用箇所については、提供された情報からは特定できません。

関連データ構造

  • md_timeperiod_t: 期間を定義する構造体（start, end を保持）。
  • apr_time_t: APR で定義される時刻表現。

失敗モードと境界条件

  • period が NULL の場合、呼び出し元でセグメンテーション違反が発生する可能性があります。
  • 期間の境界値（開始時刻ちょうど、または終了時刻ちょうど）における判定結果は、内部で呼び出される md_timeperiod_has_started および md_timeperiod_has_ended の実装仕様に依存します。

保守担当者向けメモ

  • 本関数は md_timeperiod_has_started と md_timeperiod_has_ended のラッパーとして機能します。ロジックの変更が必要な場合は、これら依存関数の仕様を確認してください。
