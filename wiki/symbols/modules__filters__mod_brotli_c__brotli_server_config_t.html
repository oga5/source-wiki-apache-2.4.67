brotli_server_config_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_brotli.c:32

目的

Apache HTTP Server における mod_brotli モジュールのサーバー単位の設定値を保持するための構造体です。圧縮品質やウィンドウサイズなどのパラメータを管理します。

シグネチャまたは形状

typedef struct brotli_server_config_t {
    int quality;
    int lgwin;
    int lgblock;
    etag_mode_e etag_mode;
    const char *note_ratio_name;
    const char *note_input_name;
    const char *note_output_name;
} brotli_server_config_t;

入力と出力

本構造体は、httpd.conf 等の設定ファイルから読み込まれた値を保持します。

  • **入力**: set_compression_quality, set_compression_lgwin, set_compression_lgblock, set_etag_mode などの設定ハンドラを介して値が設定されます。
  • **出力**: フィルタ処理時に compress_filter 等がこの構造体を参照し、Brotli エンコーダの初期化パラメータや ETag の処理方針を決定します。

前提条件と不変条件

  • 本構造体は apr_pcalloc 等で割り当てられたメモリプール上で管理されることを前提としています。
  • etag_mode は etag_mode_e 型の列挙値に従う必要があります。

アルゴリズムまたは振る舞い

サーバー設定のコンテナとして機能します。各フィールドは以下の役割を持ちます：

  • quality, lgwin, lgblock: Brotli エンコーダの圧縮特性を制御するパラメータ。
  • etag_mode: 圧縮後のレスポンスに対する ETag の付与・変更ルールを定義。
  • note_ratio_name, note_input_name, note_output_name: 統計情報をリクエストノートに記録する際のキー名。

呼び出し元と呼び出し先

  • **呼び出し元**: create_server_config および各設定ハンドラ（set_filter_note, set_compression_quality, set_compression_lgwin, set_compression_lgblock, set_etag_mode）が本構造体のインスタンスを生成または参照します。
  • **呼び出し先**: フィルタ処理関数 compress_filter が、リクエスト処理時にこの構造体を参照して圧縮動作を決定します。

関連データ構造

  • etag_mode_e: etag_mode フィールドで使用される列挙型。
  • brotli_ctx_t: リクエスト単位のコンテキスト。サーバー設定と組み合わせて圧縮処理を実行します。

失敗モードと境界条件

  • 設定値が範囲外の場合のバリデーションは、各設定ハンドラ（set_compression_quality 等）の責務であり、本構造体自体は値を保持するのみです。
  • メモリ不足時には apr_pcalloc の呼び出し元でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はサーバー単位の設定を保持するため、リクエスト単位の brotli_ctx_t と混同しないよう注意してください。
  • 新しい設定項目を追加する場合は、対応する設定ハンドラと create_server_config での初期化処理を更新する必要があります。
