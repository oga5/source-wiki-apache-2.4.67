on_add_cert

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:234

目的

ACME サーバーから受信した HTTP レスポンスに含まれる証明書チェーンを解析し、管理対象ドメインの証明書データ構造へ追加する。

シグネチャまたは形状

static apr_status_t on_add_cert(md_acme_t *acme, const md_http_response_t *res, void *baton)

入力と出力

  • **入力**:
  • acme: ACME セッション情報（未使用）。
  • res: 証明書データを含む HTTP レスポンス。
  • baton: md_proto_driver_t へのポインタ。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • baton は md_proto_driver_t 型としてキャスト可能であること。
  • ad->cred->chain が初期化済みであること。

アルゴリズムまたは振る舞い

1. 現在の証明書チェーンの要素数を記録する。 2. add_http_certs を呼び出し、レスポンスから証明書を解析して ad->cred->chain に追加する。 3. 解析に成功した場合、追加された証明書数をデバッグログに出力する。 4. get_up_link を呼び出し、レスポンスヘッダーからアップリンク情報を抽出する。 5. 処理結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_drive.c:259 において md_acme_GET のコールバックとして指定されている。
  • **呼び出し先**:
  • add_http_certs: 証明書の解析と追加。
  • md_log_perror: ログ出力。
  • get_up_link: ヘッダーからのリンク抽出。

関連データ構造

  • md_acme_t: ACME セッション状態。
  • md_http_response_t: HTTP レスポンスデータ。
  • md_proto_driver_t: プロトコル駆動のコンテキスト。
  • md_acme_driver_t: ACME 駆動のコンテキスト。

失敗モードと境界条件

  • add_http_certs が失敗した場合、そのエラーコードをそのまま返す。
  • res が空または不正な形式の場合、add_http_certs の内部ロジックに従いエラーとなる。

保守担当者向けメモ

  • 本関数は md_acme_GET のコールバックとして設計されているため、シグネチャの変更には注意が必要。
  • acme 引数は現在使用されていないが、将来的な拡張のために保持されている。
