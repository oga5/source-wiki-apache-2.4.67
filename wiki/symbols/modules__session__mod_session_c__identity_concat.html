identity_concat

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:326

目的

セッションのキーと値を URL エンコードし、key=value 形式の文字列としてバッファに連結します。

シグネチャまたは形状

static int identity_concat(void *v, const char *key, const char *val)

入力と出力

  • **入力**:
  • v: 連結先となるバッファへのポインタ（void *）。
  • key: セッションのキー文字列。
  • val: セッションの値文字列。
  • **出力**:
  • 常に 1 を返します（apr_table_do のコールバックとして継続を指示）。
  • v が指すバッファの内容が更新されます。

前提条件と不変条件

  • v は、十分な容量を持つ書き込み可能なバッファを指している必要があります。
  • 連結される各要素は & で区切られます。

アルゴリズムまたは振る舞い

1. v が指すバッファの現在の長さを取得します。 2. バッファが空でない場合、末尾に & を追加してポインタを進めます。 3. ap_escape_urlencoded_buffer を使用して key をエンコードし、バッファに書き込みます。 4. = を追加し、同様に val をエンコードしてバッファに書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session.c:368 において apr_table_do(identity_concat, buffer, z->entries, NULL); として呼び出されます。
  • **呼び出し先**:
  • strlen (標準ライブラリ)
  • ap_escape_urlencoded_buffer (Apache HTTPD API)

関連データ構造

  • apr_table_t: セッションエントリ（z->entries）を保持するテーブル構造体。

失敗モードと境界条件

  • バッファのオーバーフローに対する保護は本関数内には存在しません。呼び出し元で十分なバッファサイズが確保されていることが前提となります。

保守担当者向けメモ

  • 本関数は apr_table_do のコールバックとして設計されています。
  • ap_escape_urlencoded_buffer の結果がバッファサイズを超える可能性がある場合、呼び出し元でのバッファ管理に注意してください。
