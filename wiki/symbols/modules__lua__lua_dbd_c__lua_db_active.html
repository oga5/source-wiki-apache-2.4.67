lua_db_active

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:125

目的

Lua スクリプトからデータベース接続の生存状態を確認します。

シグネチャまたは形状

int lua_db_active(lua_State *L);

入力と出力

  • **入力**: lua_State *L（Lua VM の状態）。内部で lua_get_db_handle(L) を呼び出し、スタック上のデータベースハンドルを取得します。
  • **出力**: Lua スタックに boolean 値をプッシュし、戻り値として 1 を返します。接続が有効な場合は true、それ以外は false を返します。

前提条件と不変条件

  • lua_get_db_handle(L) が有効な lua_db_handle 構造体を返す必要があります。
  • db->alive フラグが真であること、および apr_dbd_check_conn による接続確認が APR_SUCCESS を返すことが、接続有効とみなされる条件です。

アルゴリズムまたは振る舞い

1. lua_get_db_handle(L) を使用して、Lua VM からデータベースハンドル db を取得します。 2. db が存在し、かつ db->alive が真であることを確認します。 3. apr_dbd_check_conn を呼び出し、ドライバ、プール、ハンドルを用いて物理的な接続状態を検証します。 4. 検証結果が APR_SUCCESS であれば、Lua スタックに 1 (true) をプッシュします。 5. 上記条件を満たさない場合は、Lua スタックに 0 (false) をプッシュします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_dbd.c:702 にて、Lua の関数として登録されています。
  • **呼び出し先**:
  • lua_get_db_handle (内部関数)
  • apr_dbd_check_conn (APR DBD API)

関連データ構造

  • lua_db_handle: データベース接続状態を保持する構造体。
  • lua_State: Lua VM の状態管理構造体。

失敗モードと境界条件

  • lua_get_db_handle が NULL を返した場合、または db->alive が偽の場合、接続確認は行われず false が返されます。
  • apr_dbd_check_conn が APR_SUCCESS 以外の値を返した場合、接続は無効とみなされ false が返されます。

保守担当者向けメモ

  • 本関数は mod_dbd を介した接続の生存確認を行うため、Apache のライフサイクルや接続プールの状態に依存します。
  • 物理的な接続切断が即座に反映されない場合がある点に注意してください。
