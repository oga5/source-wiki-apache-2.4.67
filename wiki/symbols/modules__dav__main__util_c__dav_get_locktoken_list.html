dav_get_locktoken_list

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:1803

目的

HTTP リクエストの If ヘッダーを解析し、そこに含まれるロックトークンのリストを抽出して返します。

シグネチャまたは形状

dav_error *dav_get_locktoken_list(request_rec *r, dav_locktoken_list **ltl)

入力と出力

  • **入力**:
  • r: 現在の HTTP リクエスト構造体。
  • ltl: 抽出されたロックトークンリストを格納するためのポインタのポインタ。
  • **出力**:
  • 成功時: NULL を返し、ltl に構築されたリストが格納されます。
  • 失敗時: dav_error 構造体へのポインタを返します。

前提条件と不変条件

  • r->pool を使用してメモリが割り当てられます。
  • ltl は呼び出し時に NULL に初期化されます。
  • If ヘッダーが存在しない、または有効なロックトークンが含まれていない場合はエラーとなります。

アルゴリズムまたは振る舞い

1. dav_process_if_header を呼び出し、リクエストの If ヘッダーを解析して dav_if_header 構造体を取得します。 2. 取得したヘッダー構造体内の if_state 連結リストを走査します。 3. 条件が DAV_IF_COND_NORMAL かつ型が dav_if_opaquelock である状態を特定します。 4. 条件に合致する各ロックトークンを dav_locktoken_list 構造体に格納し、ltl が指すリストの先頭に順次追加します。 5. リストが空のまま終了した場合は、HTTP_BAD_REQUEST エラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c:3290
  • **呼び出し先**:
  • dav_process_if_header (内部関数)
  • apr_pcalloc (APR メモリ管理)
  • dav_new_error (エラー生成)

関連データ構造

  • dav_if_header: If ヘッダーの解析結果を保持する構造体。
  • dav_if_state_list: If ヘッダー内の状態リスト。
  • dav_locktoken_list: 抽出されたロックトークンを保持するリスト構造。

失敗モードと境界条件

  • dav_process_if_header がエラーを返した場合、そのエラーをそのまま伝播します。
  • If ヘッダー内に有効なロックトークンが一つも存在しない場合、DAV_ERR_IF_ABSENT エラー（HTTP 400）を返します。

保守担当者向けメモ

  • 本関数は If ヘッダーの解析結果に強く依存しています。RFC 4918 に基づく If ヘッダーの仕様変更や拡張が必要な場合は、dav_process_if_header のロジックと併せて確認してください。
  • メモリはリクエストプール (r->pool) から確保されるため、明示的な解放は不要です。
