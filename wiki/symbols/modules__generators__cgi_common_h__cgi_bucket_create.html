cgi_bucket_create

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:237

目的

CGI スクリプトの標準出力（stdout）および標準エラー出力（stderr）を非同期的に監視するための、カスタム CGI 型バケットを生成します。

シグネチャまたは形状

apr_bucket *cgi_bucket_create(request_rec *r,
                              apr_interval_time_t timeout,
                              apr_file_t *out, apr_file_t *err,
                              apr_bucket_alloc_t *list);

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • timeout: ポーリング時のタイムアウト値。
  • out: スクリプトの stdout に接続されたパイプ。
  • err: スクリプトの stderr に接続されたパイプ。
  • list: バケット割り当て用のアロケータ。
  • **出力**:
  • 成功時: 初期化済みの apr_bucket 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • out および err パイプの APR タイムアウト設定は、本関数内で 0（非ブロッキング）に上書きされます。
  • バケットの生存期間は request_rec のプールに依存します。
  • bucket_type_cgi 型として初期化され、長さは (apr_size_t)(-1) に設定されます。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を用いてバケットを確保し、cgi_bucket_data をリクエストプールから割り当てます。 2. apr_pollset_create を呼び出し、stdout と stderr を監視するためのポーリングセットを作成します。 3. apr_pollset_add を使用して、out を識別子 1、err を識別子 2 としてポーリングセットに登録します。 4. 成功した場合、ポーリングセットとタイムアウト値を cgi_bucket_data に格納し、バケットを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:626
  • modules/generators/mod_cgid.c:1629
  • **呼び出し先**:
  • apr_bucket_alloc, apr_file_pipe_timeout_set, APR_BUCKET_INIT, apr_bucket_free, apr_pollset_create, ap_log_rerror, apr_pollset_add, apr_palloc

関連データ構造

  • struct cgi_bucket_data: バケットのポーリング状態を保持する構造体。
  • apr_bucket: Apache のバケット構造体。
  • request_rec: リクエスト情報を保持する構造体。

失敗モードと境界条件

  • apr_pollset_create または apr_pollset_add が失敗した場合、エラーログ（APLOGNO(01217)〜01219）を出力して NULL を返します。これは主にシステムまたはユーザーのファイルディスクリプタ制限に起因します。

保守担当者向けメモ

  • 本関数は WANT_CGI_BUCKET マクロが定義されている環境でのみ有効です。
  • poll() を使用するため、呼び出し側は cgi_bucket_read を通じて適切に I/O を処理する必要があります。
  • client_data に設定された 1（stdout）と 2（stderr）の値は、後続の読み取り処理で識別子として使用されます。
