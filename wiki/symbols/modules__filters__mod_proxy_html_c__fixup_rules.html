fixup_rules

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_proxy_html.c:772

目的

リクエストごとに設定された置換ルール（urlmap）を評価し、現在のリクエストコンテキストに対して有効なルールのみを抽出・複製して適用可能な状態にする。

シグネチャまたは形状

static void fixup_rules(saxctxt *ctx)

入力と出力

  • **入力**: saxctxt *ctx（フィルタ実行時の状態保持構造体。ctx->cfg->map に全ルールが格納されている）
  • **出力**: なし（副作用として ctx->map を更新し、リクエストプール上に複製された有効なルールリストを構築する）

前提条件と不変条件

  • ctx->f->r（リクエストレコード）が有効であること。
  • ctx->cfg->map にルールリストが定義されていること。
  • ルールの条件式（p->cond）が ap_expr で評価可能であること。

アルゴリズムまたは振る舞い

1. ctx->cfg->map に格納された全ルールを順次走査する。 2. 各ルールに条件式（cond）がある場合、ap_expr_exec を実行する。評価結果が偽（0）の場合はそのルールをスキップする。 3. 条件を満たすルールを apr_pmemdup を用いてリクエストプール上に複製する。 4. 複製したルールに対し、M_INTERPOLATE_FROM または M_INTERPOLATE_TO フラグが立っている場合、interpolate_vars を呼び出して変数展開を行う。

  • from パターンが空になった場合はそのルールを破棄する。
  • M_REGEX フラグがある場合、展開後のパターンを ap_pregcomp で正規表現としてコンパイルする。

    • 5. 処理したルールを ctx->map リストに連結する。 6. 最後にリストの末尾を NULL で終端する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_proxy_html.c:865（check_filter_init 内から呼び出される）
  • **呼び出し先**:
  • ap_expr_exec（条件式評価）
  • apr_pmemdup（メモリ複製）
  • interpolate_vars（変数展開）
  • ap_pregcomp（正規表現コンパイル）
  • ap_log_rerror（エラーログ出力）

関連データ構造

  • saxctxt: フィルタの実行コンテキスト。
  • urlmap: 置換ルールの定義構造体。
  • request_rec: Apache のリクエスト情報。

失敗モードと境界条件

  • **条件式評価エラー**: ap_expr_exec がエラーを返した場合、ログに記録されるが処理は継続される。
  • **変数展開後の空パターン**: interpolate_vars の結果が空文字列となった場合、そのルールは無視される。
  • **メモリ不足**: apr_pmemdup 等のメモリ割り当て失敗時は、Apache のメモリ管理ポリシーに従い例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数はリクエストごとにルールを再構築するため、ルール数が多い場合はパフォーマンスに影響を与える可能性がある。
  • ctx->map はリクエストプールに紐付いているため、リクエスト終了時に自動的に解放される。
