proxy_fdpass_flush

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_fdpass.h:32

目的

ファイルディスクリプタ（FD）の受け渡し処理において、接続の引き継ぎ前にバッファのフラッシュを行うためのコールバック関数を保持するプラグイン構造体です。

シグネチャまたは形状

struct proxy_fdpass_flush {
    /* 内部にフラッシュ処理を行う関数ポインタを保持 */
};

※ソース excerpt には構造体の宣言のみが含まれており、メンバ変数の詳細は提供されていません。

入力と出力

  • 入力: ap_lookup_provider を通じて PROXY_FDPASS_FLUSHER 識別子により取得されます。
  • 出力: 登録されたフラッシュ処理関数が実行され、接続引き継ぎ前のデータフラッシュが行われます。

前提条件と不変条件

  • Unix 系 OS の sendmsg および SCM_RIGHTS を利用する環境でのみ動作します。
  • mod_proxy_fdpass モジュールが提供するプロバイダーとして登録されている必要があります。

アルゴリズムまたは振る舞い

1. mod_proxy_fdpass がリクエストを処理する際、ap_lookup_provider を使用して PROXY_FDPASS_FLUSHER を検索します。 2. 取得された proxy_fdpass_flush 構造体内の関数が呼び出され、接続の引き継ぎに必要なフラッシュ処理が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/mod_proxy_fdpass.c 内の処理フローから参照されています。
  • 呼び出し先: modules/proxy/mod_proxy_fdpass.c 内で builtin_flush として静的に定義された実装が参照されるケースがあります。

関連データ構造

  • PROXY_FDPASS_FLUSHER: プロバイダー検索に使用される文字列定数。
  • mod_proxy.h: プロキシモジュールの基本構造体定義。

失敗モードと境界条件

  • 非 Unix 環境では本構造体を利用したフラッシュ処理は機能しません。
  • プロバイダーが登録されていない場合、ap_lookup_provider は NULL を返します。

保守担当者向けメモ

  • 本構造体は内部 API です。FD 受け渡しのカスタマイズが必要な場合のみ参照してください。
  • 構造体の詳細なメンバ定義については、modules/proxy/mod_proxy_fdpass.h の該当行以降を確認してください。
