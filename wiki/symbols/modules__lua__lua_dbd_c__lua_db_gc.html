lua_db_gc

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:94

目的

Lua VM 上のデータベースハンドルオブジェクトがガベージコレクションされる際、関連するデータベース接続を安全にクローズし、リソースを解放する。

シグネチャまたは形状

int lua_db_gc(lua_State *L);

入力と出力

  • **入力**: Lua スタックのインデックス 1 に lua_db_handle 型のユーザーデータを受け取る。
  • **出力**: 常に 0 を返す（Lua のスタックをクリアするため）。

前提条件と不変条件

  • 入力されたユーザーデータが lua_db_handle 構造体へのポインタであること。
  • db->alive が真の場合のみ、接続のクローズ処理が実行される。
  • 処理完了後、db->alive は 0 に設定され、二重解放を防止する。

アルゴリズムまたは振る舞い

1. スタックから lua_db_handle を取得する。 2. db->alive が真であれば以下を実行する:

  • db->type が LUA_DBTYPE_APR_DBD の場合: apr_dbd_close で接続を閉じ、apr_pool_destroy でメモリプールを破棄する。
  • それ以外の場合: ap_dbd_close (オプション関数) を使用して接続を閉じる。
  • 接続ハンドル、ドライバ、プールへの参照を NULL にクリアし、alive フラグを 0 にする。

    • 3. lua_settop(L, 0) を呼び出し、スタックを空にする。

呼び出し元と呼び出し先

  • **呼び出し元**: Lua VM のガベージコレクタ（lua_pushcfunction(L, lua_db_gc) によりメタメソッドとして登録）。
  • **呼び出し先**: apr_dbd_close, apr_pool_destroy, ap_dbd_close (オプション関数)。

関連データ構造

  • lua_db_handle: データベース接続状態、ドライバ、プール、生存フラグを保持する構造体。

失敗モードと境界条件

  • db が NULL の場合、または db->alive が偽の場合は何も行わず終了する。
  • APR_RETRIEVE_OPTIONAL_FN(ap_dbd_close) が NULL を返す場合、接続クローズ処理はスキップされる。

保守担当者向けメモ

  • 本関数は Lua の __gc メタメソッドとして機能するため、明示的な lua_db_close との二重呼び出しが発生しても安全な設計となっている。
  • db->pool の破棄は、接続に関連付けられたメモリリソースを解放するために不可欠である。
