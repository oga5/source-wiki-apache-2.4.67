timeout_queue

定義

  • 種別: struct
  • 定義箇所: server/mpm/event/event.c:260

目的

Event MPM において、特定のタイムアウト期限を持つ接続をグループ化して管理するためのリングバッファ構造です。

シグネチャまたは形状

struct timeout_queue {
    struct timeout_head_t head;
    apr_interval_time_t timeout;
    apr_uint32_t count;
    apr_uint32_t *total;
    struct timeout_queue *next;
};

入力と出力

  • **入力**: TO_QUEUE_MAKE マクロにより、プール、タイムアウト値、および参照先キューを指定して初期化されます。
  • **出力**: TO_QUEUE_APPEND および TO_QUEUE_REMOVE マクロを介して、接続状態（event_conn_state_t）の追加・削除が行われます。

前提条件と不変条件

  • timeout_mutex による排他制御下での操作が前提となります。
  • next メンバにより、複数のキューが連結されたリスト構造を形成可能です。
  • total は、関連するキュー全体での接続数を追跡するための共有カウンタを指します。

アルゴリズムまたは振る舞い

  • リスナーおよびワーカーは、接続のライフサイクル（Keep-Alive 待ち、書き込み完了待ち等）に応じて、適切な timeout_queue に接続を登録します。
  • process_timeout_queue 関数により、期限切れとなった接続が定期的にスキャンされ、クローズまたは再処理へ移行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: event.c 内の waitio_q 等の静的変数として定義・利用されています。process_timeout_queue や TO_QUEUE_MAKE 等の管理関数から参照されます。
  • **呼び出し先**: 接続状態を管理する event_conn_state_t を操作対象とします。

関連データ構造

  • event_conn_state_t: キューに格納される接続状態の単位。
  • timeout_head_t: timeout_queue 内で接続を保持するためのリング構造のヘッド。

失敗モードと境界条件

  • タイムアウト値が 0 の場合や、キューが空の状態で REMOVE 操作を行った場合の挙動については、各マクロの実装に依存します。
  • メモリ不足時には TO_QUEUE_MAKE が失敗する可能性があります。

保守担当者向けメモ

  • timeout_queue は apr_ring を利用して実装されています。
  • 複数のキューが連結されている場合、total ポインタが指すカウンタの整合性に注意してください。
  • 接続の追加・削除時は必ず timeout_mutex を保持していることを確認してください。
