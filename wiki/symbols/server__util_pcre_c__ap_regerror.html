ap_regerror

定義

  • 種別: function
  • 定義箇所: server/util_pcre.c:113

目的

正規表現処理で発生したエラーコードを、人間が判読可能なエラーメッセージ文字列に変換する。

シグネチャまたは形状

apr_size_t ap_regerror(int errcode, const ap_regex_t *preg, char *errbuf, apr_size_t errbuf_size)

入力と出力

  • **入力**:
  • errcode: 変換対象のエラーコード。
  • preg: エラーに関連する ap_regex_t 構造体（オフセット情報取得用、NULL 可）。
  • errbuf: メッセージを格納するバッファ。
  • errbuf_size: errbuf のサイズ。
  • **出力**:
  • 戻り値: エラーメッセージを完全に格納するために必要なバッファサイズ（終端文字を含む）。

前提条件と不変条件

  • errcode が pstring 配列の範囲外である場合、"unknown error code" が返される。
  • preg が NULL でない場合、re_erroffset が -1 でなければエラー発生位置のオフセット情報がメッセージに付加される。

アルゴリズムまたは振る舞い

1. errcode に対応するエラーメッセージ文字列を内部配列から取得する。 2. preg が有効かつ re_erroffset が設定されている場合、" at offset [数値]" という文字列を連結するための長さを計算する。 3. errbuf_size が 0 より大きい場合:

  • バッファが十分な大きさであれば、エラーメッセージとオフセット情報を apr_snprintf で書き込む。
  • バッファが不足している場合は、エラーメッセージのみを apr_cpystrn で切り詰めてコピーする。

    • 4. 必要なバッファサイズ（メッセージ長 + オフセット情報長）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_request.c:1193 (直接の呼び出しを確認)
  • **呼び出し先**:
  • strlen (標準 C ライブラリ)
  • apr_snprintf (APR)
  • apr_cpystrn (APR)

関連データ構造

  • ap_regex_t: 正規表現のコンパイル結果およびエラー情報を保持する構造体。

失敗モードと境界条件

  • errbuf_size が 0 の場合、メッセージのコピーは行われないが、必要なサイズは正しく計算されて返される。
  • errbuf_size がメッセージ長よりも小さい場合、メッセージは切り詰められる。

保守担当者向けメモ

  • 本関数は pstring という内部的なエラー文字列配列に依存している。
  • PCRE2 への移行に伴い、エラーコード体系に変更が生じた場合は pstring の定義と本関数の参照範囲を同期させる必要がある。
