expires_dir_config

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_expires.c:168

目的

Apache モジュール mod_expires において、ディレクトリ単位（.htaccess を含む）のキャッシュ有効期限設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    int active;                     /* 有効期限設定の有効/無効フラグ */
    int wildcards;                  /* MIMEタイプにワイルドカードが含まれるかのフラグ */
    char *expiresdefault;           /* デフォルトの有効期限文字列 */
    apr_table_t *expiresbytype;     /* MIMEタイプごとの有効期限マッピングテーブル */
} expires_dir_config;

入力と出力

本構造体は、設定ファイル（httpd.conf や .htaccess）のディレクティブ解析結果を入力として受け取り、リクエスト処理時に expires_filter 等へ設定情報を提供します。

前提条件と不変条件

  • apr_pool_t を介してメモリ管理が行われ、リクエスト終了時に自動的に解放されます。
  • expiresbytype は apr_table_t として初期化されている必要があります。

アルゴリズムまたは振る舞い

  • ExpiresActive ディレクティブにより active メンバが制御されます。
  • ExpiresDefault および ExpiresByType ディレクティブにより、それぞれ expiresdefault および expiresbytype テーブルが更新されます。
  • リクエスト処理時、ap_get_module_config を通じて現在のディレクトリ設定として取得され、expires_filter がレスポンスヘッダーを生成する際の判断基準となります。

呼び出し元と呼び出し先

  • **呼び出し元（設定時）**: create_dir_expires_config（初期化）、set_expiresactive、set_expiresbytype、set_expiresdefault、merge_expires_dir_configs。
  • **呼び出し元（実行時）**: expires_insert_filter、expires_filter。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_table_t: MIME タイプと有効期限の対応付けを管理するために使用。
  • request_rec: ap_get_module_config を介して本構造体を取得する際の起点。

失敗モードと境界条件

  • check_code 関数が解析する有効期限文字列が不正な場合、期待通りのヘッダーが生成されない可能性があります。
  • ExpiresActive が ACTIVE_DONTCARE の場合、親ディレクトリの設定を継承する挙動をとります。

保守担当者向けメモ

  • expiresbytype テーブルの検索は r->content_type に依存します。
  • Expires ヘッダーが既に存在する場合、本構造体の設定値による上書きは行われません。
  • atoi() を使用した解析箇所があるため、極端に大きな数値や不正なフォーマットの入力に対する堅牢性に注意してください。
