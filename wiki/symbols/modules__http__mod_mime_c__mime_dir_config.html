mime_dir_config

定義

  • 種別: struct
  • 定義箇所: modules/http/mod_mime.c:77

目的

ディレクトリ単位での MIME タイプ、エンコーディング、言語、ハンドラ、およびフィルタの設定を保持・管理するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_hash_t *extension_mappings;
    apr_array_header_t *remove_mappings;
    char *default_language;
    int multimatch;
    int use_path_info;
} mime_dir_config;

入力と出力

  • **入力**: AddType, AddEncoding, AddLanguage, MultiviewsMatch 等のディレクティブ設定値。
  • **出力**: リクエスト処理時に find_ct 等から参照され、request_rec のメタデータ設定に利用されます。

前提条件と不変条件

  • extension_mappings は、拡張子文字列をキーとし、extension_info 構造体を値とするハッシュテーブルです。
  • remove_mappings は、特定の拡張子設定を無効化するためのリストであり、設定マージ時に走査されます。
  • use_path_info は、0（ファイル名のみ使用）、1（PATH_INFOを付加）、2（未設定/0と同等）のいずれかの状態をとります。

アルゴリズムまたは振る舞い

  • 設定の継承時、親設定を破壊しないよう apr_hash_merge や apr_pmemdup を用いた深いコピーが行われます。
  • リクエスト処理時には、ファイル名から抽出された拡張子を extension_mappings で検索し、一致するメタデータを適用します。
  • remove_mappings は、設定の優先順位に従い、特定の拡張子に対するマッピングを動的に削除または無効化するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_mime_dir_config (メモリ確保)
  • merge_mime_dir_configs (設定マージ)
  • add_extension_info / remove_extension_info (設定更新)
  • find_ct (リクエスト時のメタデータ解決)
  • **呼び出し先**:
  • apr_hash_t (マッピング管理)
  • apr_array_header_t (削除リスト管理)

関連データ構造

  • extension_info: 拡張子ごとの詳細なメタデータ（タイプ、言語、フィルタ等）を保持。
  • request_rec: 本構造体の情報が適用されるリクエストのコンテキスト。

失敗モードと境界条件

  • 拡張子が複数ある場合（例: file.tar.gz）、右から左へ順次解析されます。
  • RemoveType 等の削除系ディレクティブは、設定ファイル内での記述順序に依存して適用されます。
  • 不正なヘッダや設定値に対しては、ログ出力が行われフォールバック処理が選択されます。

保守担当者向けメモ

  • EBCDIC 環境における文字コード処理に一部課題が残っています。
  • フィルタの重複チェックは行われないため、設定次第で同一フィルタが複数回追加される可能性があります。
  • use_path_info の値が 2 の場合は 0 として扱われる点に注意してください。
