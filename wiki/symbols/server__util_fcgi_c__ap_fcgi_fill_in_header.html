ap_fcgi_fill_in_header

定義

  • 種別: function
  • 定義箇所: server/util_fcgi.c:81

目的

FastCGI プロトコル（RFC 3652）のレコードヘッダー構造体 ap_fcgi_header を、指定されたパラメータで安全に初期化します。

シグネチャまたは形状

void ap_fcgi_fill_in_header(ap_fcgi_header *header,
                            unsigned char type,
                            apr_uint16_t request_id,
                            apr_uint16_t content_len,
                            unsigned char padding_len)

入力と出力

  • **入力**:
  • header: 初期化対象の ap_fcgi_header 構造体へのポインタ。
  • type: レコードタイプ（unsigned char）。
  • request_id: リクエスト識別子（apr_uint16_t）。
  • content_len: コンテンツ長（apr_uint16_t）。
  • padding_len: パディング長（unsigned char）。
  • **出力**: なし（header が指すメモリ領域が直接更新されます）。

前提条件と不変条件

  • header は有効な ap_fcgi_header 構造体を指している必要があります。
  • FastCGI バージョンは常に AP_FCGI_VERSION_1 に設定されます。
  • reserved フィールドは常に 0 に設定されます。

アルゴリズムまたは振る舞い

1. header->version を AP_FCGI_VERSION_1 に設定します。 2. header->type を引数 type で設定します。 3. request_id を上位バイト（B1）と下位バイト（B0）に分割して格納します。 4. content_len を上位バイト（B1）と下位バイト（B0）に分割して格納します。 5. header->paddingLength を引数 padding_len で設定します。 6. header->reserved を 0 に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c
  • modules/proxy/mod_proxy_fcgi.c
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行いません。

関連データ構造

  • ap_fcgi_header: FastCGI レコードヘッダーを表現する構造体（util_fcgi.h で定義）。

失敗モードと境界条件

  • 本関数はメモリ割り当てを行わず、引数の妥当性チェックも行いません。呼び出し元が提供するポインタが有効であること、および各数値が構造体のフィールドサイズ（apr_uint16_t 等）に適合していることが前提となります。

保守担当者向けメモ

  • 本関数は FastCGI プロトコルのバイナリレイアウトを直接操作するため、構造体定義に変更があった場合は本関数のビットシフト処理も併せて確認してください。
