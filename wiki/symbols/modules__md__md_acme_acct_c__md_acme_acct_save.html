md_acme_acct_save

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_acct.c:176

目的

ACME アカウント情報（JSON）と対応する秘密鍵（PEM）を md_store を介して永続化します。

シグネチャまたは形状

apr_status_t md_acme_acct_save(md_store_t *store, apr_pool_t *p, md_acme_t *acme, 
                               const char **pid, md_acme_acct_t *acct, md_pkey_t *acct_key)

入力と出力

  • **入力**:
  • store: 永続化先となるストレージハンドル。
  • p: メモリ割り当てに使用するプール。
  • acme: ACME セッション情報。
  • pid: アカウント ID へのポインタ（既存 ID がある場合はその値、新規の場合は出力される）。
  • acct: 保存対象のアカウント構造体。
  • acct_key: 保存対象の秘密鍵。
  • **出力**:
  • rv: apr_status_t 型の処理結果（成功時は APR_SUCCESS）。
  • pid: 新規作成時に生成された一意なアカウント ID が格納されます。

前提条件と不変条件

  • アカウント ID は ACME-<servername>-<index> の形式で生成されます。
  • アカウント情報は account.json、秘密鍵は account.pem として MD_SG_ACCOUNTS グループ内に保存されます。

アルゴリズムまたは振る舞い

1. md_acme_acct_to_json を呼び出し、アカウント構造体を JSON 形式に変換します。 2. pid が指定されている場合、その ID を使用して JSON を保存します。 3. pid が指定されていない（NULL）場合、mk_acct_id を用いて一意な ID を最大 1000 回試行して生成し、衝突しない ID で保存を試みます。 4. JSON の保存が成功した場合、続いて秘密鍵を MD_FN_ACCT_KEY として保存します。 5. 最終的な処理結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c:598
  • modules/md/md_acme_acct.c:494
  • modules/md/md_acme_drive.c:1041
  • **呼び出し先**:
  • md_acme_acct_to_json
  • md_store_save
  • mk_acct_id

関連データ構造

  • md_acme_acct_t: アカウント情報。
  • md_store_t: 永続化ストレージ。
  • md_pkey_t: 秘密鍵。
  • md_json_t: シリアライズされたアカウントデータ。

失敗モードと境界条件

  • md_store_save が失敗した場合、そのステータスコードが返されます。
  • 新規 ID 生成時に 1000 回の試行で一意な ID が確保できない場合、APR_EAGAIN が返される可能性があります。

保守担当者向けメモ

  • 本関数はアカウントの JSON と秘密鍵の整合性を保つため、両方の保存が成功することを前提としています。
  • pid が NULL の場合、ループによる ID 生成を行うため、ストレージの競合状態に注意してください。
