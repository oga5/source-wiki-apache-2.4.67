dav_get_lockdb_path

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/mod_dav_fs.c:32

目的

Apache HTTP Server の設定から、WebDAV ロックデータベースのパス（DAVLockDB）を取得する。

シグネチャまたは形状

const char *dav_get_lockdb_path(const request_rec *r)

入力と出力

  • **入力**: r (現在のリクエストを保持する request_rec 構造体へのポインタ)
  • **出力**: 設定されたロックデータベースのパスを示す const char * 型の文字列。設定がない場合は NULL が返される可能性がある。

前提条件と不変条件

  • r->server が有効であり、dav_fs_module に関連付けられたサーバー設定が初期化されていること。
  • dav_fs_server_conf 構造体が ap_get_module_config を通じて正しく取得できること。

アルゴリズムまたは振る舞い

1. r->server->module_config から dav_fs_module に対応するサーバー設定構造体 dav_fs_server_conf を取得する。 2. 取得した設定構造体内の lockdb_path メンバを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/lock.c:345 (ロック操作時にパスを参照)
  • **呼び出し先**:
  • ap_get_module_config (Apache HTTP Server API)

関連データ構造

  • dav_fs_server_conf: サーバーごとの設定を保持する構造体。lockdb_path を管理する。
  • request_rec: Apache のリクエスト情報を保持する構造体。
  • dav_fs_module: 本モジュールの定義構造体。

失敗モードと境界条件

  • DAVLockDB ディレクティブが設定されていない場合、lockdb_path は NULL となり、呼び出し元で適切に処理される必要がある。
  • r が NULL の場合、あるいは r->server が不正な場合はセグメンテーション違反を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は repos.h で公開されており、ファイルシステムバックエンドのロック管理において重要な役割を果たす。
  • 設定値の継承やマージは dav_fs_merge_server_config で制御されているため、パスの整合性についてはそちらのロジックも併せて確認すること。
