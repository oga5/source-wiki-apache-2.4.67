ap_proxy_canonenc_ex

定義

  • 種別: function
  • 定義箇所: modules/proxy/proxy_util.c:220

目的

URL文字列を正規化し、指定されたエンコーディング規則（enctype）に基づいて再エンコードを行う。プロキシ経由の通信において、バックエンドへ送信するURLの安全性を確保するために使用される。

シグネチャまたは形状

char *ap_proxy_canonenc_ex(apr_pool_t *p, const char *x, int len,
                           enum enctype t, int flags,
                           int proxyreq)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • x: 対象のURL文字列。
  • len: 文字列の長さ。
  • t: エンコードタイプ（enc_path, enc_search, enc_user, enc_fpath, enc_parm）。
  • flags: 動作フラグ（PROXY_CANONENC_FORCEDEC, PROXY_CANONENC_NOENCODEDSLASHENCODING）。
  • proxyreq: プロキシリクエストの種類（PROXYREQ_REVERSE 等）。
  • **出力**:
  • 正規化・エンコードされた新しい文字列（apr_palloc で確保）。失敗時は NULL を返す。

前提条件と不変条件

  • t に応じて、エンコード対象外とする文字セット（allowed）および予約文字（reserved）が定義される。
  • reserved に指定された文字（例: enc_path における /）は、エンコード処理をスキップしてそのまま出力される。

アルゴリズムまたは振る舞い

1. t に基づき、エンコード不要な文字リスト（allowed）と予約文字（reserved）を決定する。 2. 入力文字列 x を1文字ずつ走査する。 3. reserved に含まれる文字はそのまま出力バッファへコピーする。 4. flags や proxyreq の条件に基づき、% で始まるパーセントエンコードされたシーケンスをデコードする。

  • デコード後の文字が reserved に含まれる場合は、元のエンコード状態を維持する。
  • noencslashesenc が有効な場合、特定の条件下で % を再エンコードする。

    • 5. デコード後の文字が allowed に含まれず、かつ英数字でもない場合、ap_proxy_c2hex を用いてパーセントエンコード形式に変換する。 6. 最後にヌル終端を付与して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy_http.c, mod_proxy_ajp.c, mod_proxy_ftp.c, mod_proxy_fcgi.c, mod_proxy_scgi.c, mod_proxy_uwsgi.c, mod_proxy_wstunnel.c, mod_proxy_http2.c 等の各プロトコルハンドラ。
  • **呼び出し先**: ap_proxy_hex2c, ap_proxy_c2hex（いずれも proxy_util.c 内）。

関連データ構造

  • enum enctype: エンコードの種類を定義する列挙型。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • パーセントエンコードの直後に16進数以外の文字が続く場合（% の直後に isxdigit でない文字がある場合）、NULL を返して処理を中断する。
  • 入力文字列の長さが正しく渡されない場合、バッファオーバーフローや不正なメモリ参照のリスクがある（3 * len + 1 の領域を確保）。

保守担当者向けメモ

  • reserved 文字の定義は RFC 1738 に基づくが、enc_path における ; の許可など、プロキシ特有の拡張が含まれている。変更時はプロトコルハンドラへの影響を考慮すること。
  • proxyreq が PROXYREQ_REVERSE の場合、リバースプロキシ特有のデコード制御が行われるため、ロジックの修正時は注意が必要。
