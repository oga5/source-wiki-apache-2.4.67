ssl_init_ocsp_certificates

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ocsp.c:401

目的

OCSP 検証に使用する信頼済み証明書チェーンを読み込み、SSL コンテキストの検証フラグを更新します。

シグネチャまたは形状

void ssl_init_ocsp_certificates(server_rec *s, modssl_ctx_t *mctx)

入力と出力

  • **入力**:
  • s: サーバーレコード (server_rec)。ログ出力に使用。
  • mctx: SSL コンテキスト構造体 (modssl_ctx_t)。ocsp_certs_file からファイルパスを取得し、読み込み結果を ocsp_certs に格納、検証フラグ ocsp_verify_flags を更新する。
  • **出力**: なし（副作用として mctx の状態を変更）。

前提条件と不変条件

  • mctx->ocsp_certs_file が設定されている場合にのみ処理を実行します。
  • 読み込みに失敗した場合は、エラーログを出力し ssl_die(s) を呼び出してプロセスを終了させます。

アルゴリズムまたは振る舞い

1. mctx->ocsp_certs_file が未設定であれば、何もせず終了します。 2. デバッグログを出力します。 3. modssl_read_ocsp_certificates を呼び出し、指定されたファイルから証明書を読み込みます。 4. 読み込み結果が空（NULL）の場合、エラーログを出力し、ssl_log_ssl_error を経由して ssl_die(s) でサーバーを停止させます。 5. 成功した場合、mctx->ocsp_verify_flags に OCSP_TRUSTOTHER フラグを追加設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_init.c:2249
  • **呼び出し先**:
  • modssl_read_ocsp_certificates (証明書の読み込み)
  • ap_log_error (ログ記録)
  • ssl_log_ssl_error (エラーログ記録)
  • ssl_die (致命的エラー時の終了処理)

関連データ構造

  • modssl_ctx_t: SSL 設定コンテキスト。OCSP 関連のファイルパス、証明書スタック、検証フラグを保持します。
  • server_rec: Apache サーバーの構成情報。

失敗モードと境界条件

  • **ファイル読み込み失敗**: modssl_read_ocsp_certificates が証明書を返せない場合、サーバーの起動プロセスが停止します。
  • **設定未定義**: ocsp_certs_file が設定されていない場合、本関数は何も行わず正常終了します。

保守担当者向けメモ

  • 本関数は mod_ssl の初期化フェーズで呼び出されます。
  • OCSP_TRUSTOTHER フラグは、OpenSSL の OCSP 検証において、信頼済み証明書チェーン以外の証明書を検証に使用することを許可するものです。
