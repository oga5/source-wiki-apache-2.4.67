ent

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_autoindex.c:748

目的

ディレクトリ索引生成時において、各ファイルやディレクトリのメタデータおよび表示用情報を保持するための一時的なデータ構造です。

シグネチャまたは形状

struct ent {
    char *name;
    char *icon;
    char *alt;
    char *desc;
    apr_off_t size;
    apr_time_t lm;
    struct ent *next;
    int ascending, ignore_case, version_sort;
    char key;
    int isdir;
};

入力と出力

本構造体は、index_directory 関数によるディレクトリ走査の結果を格納するコンテナとして機能します。出力として、ソート済みリストの各要素が HTML 生成関数へ渡されます。

前提条件と不変条件

  • next ポインタにより、単方向リストを構成します。
  • メモリ割り当ては、リクエストに関連付けられた apr_pool_t を介して行われることが前提です。

アルゴリズムまたは振る舞い

  • dsortf 関数において、name、size、lm（最終更新日時）、desc などのフィールドに基づいたソート処理の対象となります。
  • ascending、ignore_case、version_sort の各フラグは、ソート時の挙動を制御するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: index_directory 関数がこの構造体のインスタンスを生成・管理します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。
  • 注記: ツールが検出した他のファイル（mod_authnz_ldap.c や mod_unixd.c）内の ent は、本構造体とは無関係な同名の別シンボルです。

関連データ構造

  • autoindex_config_rec: ディレクトリごとの設定を保持し、ent の生成時に参照されます。
  • apr_finfo_t: index_directory がファイルシステムから情報を取得する際に使用する構造体であり、ent の各フィールドのソースとなります。

失敗モードと境界条件

  • メモリ不足: apr_pool_t からの割り当て失敗時に発生する可能性があります。
  • 属性欠落: ファイルシステムからメタデータが取得できない場合、size や lm がデフォルト値または未定義状態となる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_autoindex.c 内でのみ使用されるローカルなデータ構造です。
  • 他のモジュールで同名のシンボルが存在するため、コード検索時には定義箇所（modules/generators/mod_autoindex.c）を必ず確認してください。
