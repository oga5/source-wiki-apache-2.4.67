ap_regcomp_set_default_cflags

定義

  • 種別: function
  • 定義箇所: server/util_pcre.c:168

目的

正規表現コンパイル時に使用されるグローバルなデフォルトフラグ（default_cflags）を設定します。

シグネチャまたは形状

void ap_regcomp_set_default_cflags(int cflags)

入力と出力

  • **入力**: cflags (int型) - 設定する正規表現コンパイルフラグのビットマスク。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は server/util_pcre.c 内の静的変数 default_cflags を更新します。
  • 設定されたフラグは、以降の ap_regcomp 呼び出しにおいて、明示的にフラグが指定されない場合のデフォルト値として参照されます。

アルゴリズムまたは振る舞い

1. 引数として受け取った cflags を、モジュールスコープの静的変数 default_cflags に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core.c:3037
  • server/core.c:5119
  • **呼び出し先**:
  • 本関数は他の関数を呼び出しません。

関連データ構造

  • default_cflags (静的変数): 正規表現コンパイルフラグを保持する内部状態。

失敗モードと境界条件

  • 本関数自体に失敗モードはありません。
  • 不適切なビットマスクが渡された場合、後続の ap_regcomp 実行時に PCRE ライブラリ側でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は include/ap_regex.h で宣言されており、HTTPD のコア設定フェーズでデフォルトの正規表現挙動を制御するために使用されます。
  • スレッドセーフ性については、本関数が設定変更を行うタイミングがサーバー起動時の設定読み込みフェーズに限定されていることを前提としています。
