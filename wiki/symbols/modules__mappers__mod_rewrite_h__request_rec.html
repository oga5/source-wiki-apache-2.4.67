request_rec

定義

  • 種別: typedef
  • 定義箇所: modules/mappers/mod_rewrite.h:35
  • 補足: 本来の構造体定義は httpd.h:822 に存在します。

目的

Apache HTTP Server における単一の HTTP リクエストに関する全情報を保持するコア構造体です。mod_rewrite モジュールにおいては、書き換えルールやマップ関数がリクエストの属性を参照・操作するためのコンテキストとして使用されます。

シグネチャまたは形状

httpd.h で定義される構造体であり、リクエストに関連する URI、メソッド、ヘッダー、接続情報、および各モジュールが保持するリクエストごとのコンテキストデータを含みます。

入力と出力

  • 入力: HTTP リクエスト処理の各フェーズ（翻訳、修正、ハンドラ実行など）において、Apache コアから各モジュールへ渡されます。
  • 出力: 構造体内のメンバを直接変更することで、リクエストの処理状態や最終的なレスポンス内容に影響を与えます。

前提条件と不変条件

  • Apache のリクエスト処理パイプライン内で生成され、リクエストの生存期間中のみ有効です。
  • 複数のモジュール間で共有されるため、各モジュールは自身のコンテキスト領域（request_config）以外を不用意に変更してはなりません。

アルゴリズムまたは振る舞い

  • mod_rewrite においては、rewrite_mapfunc_t 型の関数ポインタの引数として渡され、書き換えロジックがリクエストの URI や環境変数を評価するために利用されます。
  • リクエスト処理の各段階で、Apache コアおよび各モジュールにより、パスの解決やコンテンツネゴシエーションの結果が順次書き込まれます。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のリクエスト処理エンジン（コア）が、各フック関数を呼び出す際に引数として渡します。
  • 呼び出し先: ap_expr_exec や ap_expr_str_exec などの式評価関数、および mod_rewrite の書き換え関数などが、リクエスト情報を取得するために本構造体を参照します。
  • 詳細は include/ap_expr.h や docs/manual/developer/mod_example_*.c 等のコードベース全体で広範に参照されています。

関連データ構造

  • rewrite_mapfunc_t: request_rec を引数に取る書き換えマップ関数型。
  • request_config: 各モジュールが request_rec 内に保持するモジュール固有のデータ領域。

失敗モードと境界条件

  • NULL ポインタの参照: リクエスト処理の初期段階や異常終了時に NULL が渡される可能性があるため、モジュール側でのチェックが必要です。
  • 構造体の変更による競合: 複数のモジュールが同一のメンバを更新しようとした場合、実行順序（フックの優先度）に依存した予期せぬ動作が発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは httpd.h で定義された構造体のエイリアス的役割を果たしています。
  • mod_rewrite.h 内での定義は、主にモジュール間インターフェースの型安全性を確保するためのものです。
  • 構造体のメンバ構成は Apache のバージョンアップに伴い変更される可能性があるため、include/ap_mmn.h（モジュールマジックナンバー）の変更履歴を確認してください。
