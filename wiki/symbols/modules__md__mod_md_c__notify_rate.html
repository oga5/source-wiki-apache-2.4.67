notify_rate

定義

  • 種別: struct
  • 定義箇所: modules/md/mod_md.c:136

目的

証明書更新等のイベント通知において、同一理由による通知が頻発することを防ぐためのレート制限設定を保持します。

シグネチャまたは形状

typedef struct {
    const char *reason;         /* 通知の理由 */
    apr_time_t min_interim;     /* 同一理由での通知間隔の最小値 */
} notify_rate;

入力と出力

本構造体はデータ保持用であり、関数ではありません。reason には通知理由を示す文字列、min_interim には apr_time_t 型の期間が格納されます。

前提条件と不変条件

  • min_interim は、通知間隔の最小閾値として使用されます。
  • reason は、通知を識別するためのキーとして機能します。

アルゴリズムまたは振る舞い

本構造体自体はデータ定義ですが、mod_md.c 内で notify_rates[] 配列の要素として使用され、通知処理の実行可否を判定するための設定値として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md.c:141 にて notify_rates 配列の定義に使用されています。
  • 呼び出し先: 提供されたソース情報からは、本構造体を参照してレート制限を適用する具体的な関数は特定されていません。

関連データ構造

  • notify_rates[]: notify_rate 型の静的配列。通知理由ごとの制限設定を管理します。

失敗モードと境界条件

  • min_interim に不適切な値（負の値や極端に短い時間）が設定された場合、レート制限が期待通りに機能しない可能性があります。

保守担当者向けメモ

  • 本構造体は mod_md における通知の抑制ロジックの基盤です。通知頻度を調整する際は、notify_rates 配列の定義箇所を確認してください。
