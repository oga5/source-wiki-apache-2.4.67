ap_get_scoreboard_worker

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:692

目的

指定されたスコアボードハンドル（ap_sb_handle_t）に対応する、特定のワーカーのスコア構造体（worker_score）へのポインタを取得します。

シグネチャまたは形状

worker_score *ap_get_scoreboard_worker(ap_sb_handle_t *sbh)

入力と出力

  • **入力**: sbh - ワーカーを特定するためのスコアボードハンドル。
  • **出力**: 成功した場合は該当する worker_score 構造体へのポインタを返し、sbh が NULL の場合は NULL を返します。

前提条件と不変条件

  • sbh が有効なハンドルである場合、内部で保持されている child_num および thread_num は、スコアボードのメモリレイアウト上の有効なインデックス範囲内である必要があります。
  • スコアボードのメモリ領域は、本関数が呼び出される前に ap_init_scoreboard によって初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 引数 sbh が NULL かどうかを確認し、NULL であれば直ちに NULL を返します。 2. sbh が有効な場合、その内部にある child_num（プロセス番号）と thread_num（スレッド番号）を抽出します。 3. ap_get_scoreboard_worker_from_indexes を呼び出し、抽出したインデックスに基づいて共有メモリ上の該当する worker_score 構造体のアドレスを計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/echo/mod_echo.c:99
  • **呼び出し先**:
  • ap_get_scoreboard_worker_from_indexes（server/scoreboard.c 内）

関連データ構造

  • ap_sb_handle_t: ワーカーを特定するためのハンドル構造体。
  • worker_score: 各ワーカーの稼働状態や統計情報を保持する構造体。

失敗モードと境界条件

  • sbh が NULL の場合、安全に NULL を返します。
  • sbh が指すインデックスがスコアボードの確保済み領域外である場合、ap_get_scoreboard_worker_from_indexes の実装に依存した動作（メモリ境界外アクセス等）が発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的なインデックス計算は ap_get_scoreboard_worker_from_indexes に委譲されています。
  • スコアボードのメモリレイアウトは ap_init_scoreboard で決定されるため、本関数が返すポインタは共有メモリ上の固定位置を指します。排他制御は行われません。
