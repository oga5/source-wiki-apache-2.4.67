h2_workers

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_workers.h:31

目的

HTTP/2 モジュールにおいて、セカンダリ接続（C2）を並列実行するための専用スレッドプールを管理するメイン構造体です。

シグネチャまたは形状

typedef struct h2_workers h2_workers;

入力と出力

  • **入力**: h2_workers_create（実装側）により、最小・最大スレッド数およびアイドル制限時間が設定されます。
  • **出力**: h2_mplx 等の上位層に対し、接続処理を並列化するためのスレッドリソースを提供します。

前提条件と不変条件

  • 本構造体は、Apache HTTP Server の MPM とは独立したスレッド管理を行います。
  • 登録される接続プロデューサー（ap_conn_producer_t）のコールバック関数は、スレッドセーフである必要があります。

アルゴリズムまたは振る舞い

  • **スレッドプール管理**: 指定されたスレッド数に基づき、ワーカーのライフサイクルを制御します。
  • **負荷適応**: アイドル状態のスレッドを一定時間後に終了させ、リソース消費を最適化します。
  • **接続処理**: h2_workers_activate によりアクティブ化されたプロデューサーから、アイドル中のワーカーが接続（conn_rec）を取得し処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/http2/h2_c1.c: プロセス単位のインスタンスとして保持。
  • modules/http2/h2_mplx.c: 多重化処理のコンテキストとして利用。
  • modules/http2/h2_session.c: セッション処理の引数として利用。
  • **呼び出し先**:
  • 構造体自体の内部実装は modules/http2/h2_workers.c に存在します。
  • 接続処理において ap_conn_producer_t のコールバック（next, done, shutdown）を呼び出します。

関連データ構造

  • ap_conn_producer_t: 接続ソースを定義し、ワーカーと紐付ける構造体。
  • conn_rec: ワーカーが処理する個別の接続オブジェクト。
  • h2_mplx: 本構造体を利用してリクエストの多重化を行う上位層。

失敗モードと境界条件

  • スレッド生成の失敗や、プロデューサーの登録・終了処理における同期不整合が発生する可能性があります。
  • idle_limit の設定値が不適切な場合、リソース消費量とスレッド生成コストのバランスが崩れる可能性があります。

保守担当者向けメモ

  • 本機能は HTTP/2 のストリーム処理効率化のために設計されています。
  • 構造体の詳細なフィールド定義は h2_workers.c を参照してください。
  • h2_workers_join を呼び出す際は、進行中の処理が完了するまで待機する仕様である点に注意してください。
