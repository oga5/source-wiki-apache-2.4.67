mva_mode_e

定義

  • 種別: enum
  • 定義箇所: modules/mappers/mod_vhost_alias.c:58

目的

mod_vhost_alias モジュールにおいて、バーチャルホストのパス生成に使用するマッピングモードを識別するために定義されています。

シグネチャまたは形状

typedef enum {
    VHOST_ALIAS_UNSET,
    VHOST_ALIAS_NONE,
    VHOST_ALIAS_NAME,
    VHOST_ALIAS_IP
} mva_mode_e;

入力と出力

  • 本シンボル自体は列挙型であり、設定構造体 mva_sconf_t のメンバーとして保持されます。
  • VHOST_ALIAS_NAME または VHOST_ALIAS_IP が選択されることで、後続のパス補間処理におけるホスト名ベースか IP ベースかの判定基準となります。

前提条件と不変条件

  • VHOST_ALIAS_UNSET は初期状態を示します。
  • 設定処理において、一度確定したモードが適切に継承または上書きされる必要があります。

アルゴリズムまたは振る舞い

  • mva_translate 関数および vhost_alias_interpolate 関数内で、現在のモードを参照し、リクエストのホスト名または IP アドレスをテンプレート文字列に適用するかどうかを決定します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • modules/mappers/mod_vhost_alias.c:68 (mva_sconf_t 内の doc_root_mode)
  • modules/mappers/mod_vhost_alias.c:69 (mva_sconf_t 内の cgi_root_mode)
  • modules/mappers/mod_vhost_alias.c:123 (設定ハンドラ内の引数)
  • modules/mappers/mod_vhost_alias.c:386 (mva_translate 内のローカル変数)
  • 呼び出し先: なし（列挙型のため）。

関連データ構造

  • mva_sconf_t: サーバー単位の設定を保持する構造体。この中で doc_root_mode および cgi_root_mode として使用されます。

失敗モードと境界条件

  • VHOST_ALIAS_UNSET のまま処理が進んだ場合、意図しないパス生成やマッピングのスキップが発生する可能性があります。

保守担当者向けメモ

  • モードの追加や変更を行う際は、vhost_alias_interpolate 内の条件分岐をすべて更新する必要があります。
