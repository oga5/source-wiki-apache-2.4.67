slot_run

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_workers.c:234

目的

HTTP/2 ワーカープールにおける個々のスレッドのメインループを実行し、接続プロデューサーからタスク（conn_rec）を取得して処理を行う。

シグネチャまたは形状

static void* APR_THREAD_FUNC slot_run(apr_thread_t *thread, void *wctx)

入力と出力

  • **入力**:
  • thread: 実行中のスレッドハンドル。
  • wctx: h2_slot 構造体へのポインタ（ワーカーのコンテキスト）。
  • **出力**: NULL（スレッド終了時に返却）。

前提条件と不変条件

  • workers->lock ミューテックスにより、リングバッファ操作および状態遷移が保護されている。
  • slot->prod が有効であること（AP_DEBUG_ASSERT で検証）。
  • c->master が存在する場合、接続 ID は (c->master->id << 8)^slot->id として計算される。

アルゴリズムまたは振る舞い

1. **初期化**: スロット状態を H2_SLOT_RUN に設定し、ループを開始する。 2. **タスク取得**: get_next(slot) を呼び出し、処理対象の接続を取得する。 3. **接続処理**:

  • c->current_thread に現在のスレッドを設定。
  • ap_process_connection または h2_c2_process を実行して接続を処理。
  • 完了後、slot->prod->fn_done を呼び出し、必要に応じて workers->prod_done をブロードキャストする。

    • 4. **アイドル待機**: タスクがない場合、workers->idle リングに移動し、more_work 条件変数で待機する。

  • idle_limit が設定されている場合、タイムアウト時にスロットを終了（H2_SLOT_ZOMBIE へ移行）させる。

    • 5. **終了処理**: ループを抜ける際、スロットを H2_SLOT_ZOMBIE に設定し、workers->zombie リングへ移動して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: activate_slot（modules/http2/h2_workers.c 内でスレッド生成時に指定）。
  • **呼び出し先**: get_next, join_zombies, ap_process_connection (または h2_c2_process), slot->prod->fn_done。

関連データ構造

  • h2_workers: プール全体の管理構造体。
  • h2_slot: ワーカーの状態と同期プリミティブを保持。
  • ap_conn_producer_t: 接続供給元。
  • conn_rec: 処理対象の接続情報。

失敗モードと境界条件

  • **接続 ID の衝突**: 256個を超えるワーカーや高負荷環境下では、接続 ID 生成ロジックにおいて衝突が発生する可能性がある。
  • **タイムアウト**: idle_limit を超えて待機した場合、スレッドは自動的に終了する。
  • **シャットダウン**: workers->aborted または slot->should_shutdown が真になると、ループを即座に終了する。

保守担当者向けメモ

  • 本関数は h2_workers のスレッドライフサイクルの中核を担う。
  • c->id の計算ロジックは mod_unique_id との互換性を維持するための暫定的な実装であり、変更には注意を要する。
  • スレッド終了時の H2_SLOT_ZOMBIE への遷移は、join_zombies によるリソース回収を確実にするために重要である。
