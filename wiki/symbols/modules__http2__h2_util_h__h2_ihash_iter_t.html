h2_ihash_iter_t

定義

  • 種別: typedef
  • 定義箇所: modules/http2/h2_util.h:43

目的

h2_ihash_t（整数IDをキーとするハッシュテーブル）内の全要素に対して反復処理を行うためのコールバック関数の型定義です。

シグネチャまたは形状

typedef int h2_ihash_iter_t(void *ctx, void *val);

入力と出力

  • **入力**:
  • void *ctx: 反復処理中に保持・更新されるユーザー定義のコンテキストデータ。
  • void *val: ハッシュテーブルに格納されている要素へのポインタ。
  • **出力**:
  • int: 反復処理の継続を制御する戻り値。通常、0以外の値を返すと反復が中断されます。

前提条件と不変条件

  • h2_ihash_iter 関数に渡される関数ポインタとして使用されます。
  • コールバック内でハッシュテーブル自体を破壊的に変更（要素の追加や削除）する場合の安全性については、h2_ihash の実装仕様に依存します。

アルゴリズムまたは振る舞い

  • h2_ihash_iter 関数がハッシュテーブルの各要素に対してこの関数を順次呼び出します。
  • 呼び出し元は、ctx を介して反復処理の状態を管理し、val に対して特定の処理（検索、集計、解放など）を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_util.c 内の h2_ihash_iter 関数が、この型の関数ポインタを引数として受け取り、内部ループで呼び出します。
  • **呼び出し先**: ユーザーが定義した具体的なコールバック関数がこの型に適合する必要があります。

関連データ構造

  • h2_ihash_t: 本イテレータが操作対象とするハッシュテーブル構造体。

失敗モードと境界条件

  • コールバック関数が 0 以外の値を返した場合、反復処理は即座に停止します。
  • val が NULL である可能性については、ハッシュテーブルへの格納ルールに依存します。

保守担当者向けメモ

  • 本シンボルは h2_ihash_iter 関数のインターフェースを規定するものです。シグネチャを変更する場合は、h2_util.c およびこれを利用するすべてのモジュールでの修正が必要です。
