ap_list_provider_groups

定義

  • 種別: function
  • 定義箇所: server/provider.c:168

目的

登録されているすべてのプロバイダーグループ名と、それに対応するバージョンの一覧を取得します。

シグネチャまたは形状

apr_array_header_t *ap_list_provider_groups(apr_pool_t *pool)

入力と出力

  • **入力**: pool (メモリ割り当てに使用する apr_pool_t へのポインタ)
  • **出力**: ap_list_provider_groups_t 構造体の配列を含む apr_array_header_t へのポインタ。

前提条件と不変条件

  • global_providers_names が初期化されている必要があります。初期化されていない場合、空の配列が返されます。
  • 戻り値の配列は、指定された pool の生存期間に依存します。

アルゴリズムまたは振る舞い

1. 指定された pool を使用して、ap_list_provider_groups_t 型の要素を格納する配列を初期化します。 2. global_providers_names ハッシュテーブルを走査します。 3. 各グループ内のハッシュテーブル（バージョン情報）をさらに走査します。 4. 各グループ名とバージョン名のペアを apr_array_push を用いて配列に追加します。 5. 構築した配列を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_info.c:717 で確認されています。
  • **呼び出し先**: apr_array_make, apr_hash_first, apr_hash_next, apr_hash_this, apr_array_push (すべて APR ライブラリ関数)。

関連データ構造

  • ap_list_provider_groups_t: グループ名とバージョン名を保持する構造体。
  • global_providers_names: プロバイダーのメタデータを管理する内部ハッシュテーブル。

失敗モードと境界条件

  • global_providers_names が NULL の場合、空の配列を返して正常終了します。
  • provider_group_hash が NULL の場合、そのエントリはスキップされます。

保守担当者向けメモ

  • 本関数は include/ap_provider.h で宣言されており、主にデバッグや情報表示用モジュール（mod_info 等）から利用されます。
  • 実行時の動的な登録・削除は想定されていないため、マルチスレッド環境での呼び出し時には注意が必要です。
