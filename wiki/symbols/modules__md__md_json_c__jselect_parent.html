jselect_parent

定義

  • 種別: function
  • 定義箇所: modules/md/md_json.c:139

目的

シグネチャまたは形状

static json_t *jselect_parent(const char **child_key, int create, md_json_t *json, va_list ap)

入力と出力

  • **入力**:
  • child_key: 最終的なキー名を格納するための出力用ポインタ。
  • create: 欠落しているノードを生成するかどうかを指定するフラグ（1: 生成する, 0: 生成しない）。
  • json: 操作対象の md_json_t 構造体。
  • ap: パスを構成するキー文字列の可変長引数リスト。
  • **出力**:
  • 戻り値: パス上の親ノードとなる json_t オブジェクト。見つからない場合は NULL。
  • child_key: 最後に到達したキー名（*child_key に格納）。

前提条件と不変条件

  • json 引数は有効な md_json_t インスタンスを指している必要があります。
  • ap は NULL で終端されたキーのリストである必要があります。

アルゴリズムまたは振る舞い

1. child_key を NULL に初期化します。 2. json->j を起点として、ap から取得したキーを順次走査します。 3. 次のキーが存在する場合（中間ノード）:

  • 現在のオブジェクトからキーを取得します。
  • 存在せず create が真であれば、新しい json_object() を作成し、現在のノードにセットします。

    • 4. 次のキーが存在しない場合（末尾のキー）:

  • 現在のキーを child_key に格納し、現在のノードを親として返します。

    • 5. 走査中にノードが NULL になった場合、処理を中断してその時点のノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_json.c 内の各種 JSON 操作関数（jselect_add 等を含む複数箇所）。
  • **呼び出し先**: jansson ライブラリの json_object_get, json_object, json_object_set_new。

関連データ構造

  • md_json_t: json_t をラップし、APR プール管理と統合された構造体。
  • json_t: Jansson ライブラリの JSON ノード型。

失敗モードと境界条件

  • json 引数が NULL の場合、ループ条件により直ちに NULL が返されます。

保守担当者向けメモ

  • 本関数は va_list を直接操作するため、呼び出し側で va_start / va_end が適切に行われていることを前提としています。
  • create モードで生成されたノードは、md_json_t が保持する APR プールの生存期間に従って管理されます。
