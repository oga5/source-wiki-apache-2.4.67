apr_byte_t

定義

  • 種別: typedef (または struct メンバーの型定義)
  • 定義箇所: modules/metadata/mod_remoteip.c:94

目的

PROXY プロトコル（v1/v2）のバイナリヘッダー構造体において、IP アドレスやシグネチャなどの固定長バイト配列を定義するために使用されます。

シグネチャまたは形状

ソースコード上では、主に unsigned char 等の 1 バイト単位のデータ型として扱われます。mod_remoteip.c 内では、以下の構造体メンバーの型として出現します。

  • src_addr[16] (IPv6 アドレス用)
  • dst_addr[16] (IPv6 アドレス用)
  • src_addr[108] (Unix ドメインソケット用)
  • dst_addr[108] (Unix ドメインソケット用)
  • sig[12] (PROXY プロトコル v2 シグネチャ用)

入力と出力

  • **入力**: ネットワークストリームから読み込まれたバイナリデータ。
  • **出力**: 構造体メンバーとして格納された生のバイト列。

前提条件と不変条件

  • PROXY プロトコル v2 の仕様に基づき、特定のフィールドは固定長である必要があります。
  • メモリレイアウトはパディングを含まない連続したバイト列として期待されます。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、特定のアルゴリズムを持ちません。mod_remoteip.c 内では、ストリームから読み取ったバイナリデータをこの型で定義された配列にコピーし、IP アドレスの抽出やプロトコルシグネチャの検証に使用します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_remoteip.c 内の PROXY プロトコル解析ロジック。
  • **呼び出し先**: 不明（本シンボルはデータ型であり、関数呼び出しを行いません）。
  • **他モジュールからの参照**: modules/filters/mod_brotli.c において、ストリーム処理用のバッファ型として apr_byte_t が参照されています。

関連データ構造

  • proxy_header: PROXY プロトコル v1/v2 の共用体。
  • proxy_v2: PROXY プロトコル v2 のヘッダー構造体。
  • proxy_v2_addr: PROXY プロトコル v2 のアドレス情報構造体。

失敗モードと境界条件

  • **境界条件**: 配列サイズ（16 バイトや 108 バイト）を超えるデータを書き込もうとした場合、バッファオーバーフローのリスクがあります。
  • **失敗モード**: ネットワークから読み取ったデータが期待されるサイズに満たない場合、解析処理が中断されます。

保守担当者向けメモ

  • apr_byte_t は Apache Portable Runtime (APR) の標準的な型定義に準拠しています。
  • mod_remoteip.c 以外のモジュール（mod_brotli 等）でも使用されているため、変更を加える際は広範な影響を考慮してください。
