sed_compile_string

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:99

目的

文字列として提供された sed スクリプトを解析し、実行可能な内部表現（sed_commands_t）へコンパイルします。

シグネチャまたは形状

apr_status_t sed_compile_string(sed_commands_t *commands, const char *s)

入力と出力

  • **入力**:
  • commands: コンパイル結果を格納する sed_commands_t 構造体へのポインタ。
  • s: コンパイル対象の sed スクリプト文字列。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • 失敗時は APR_EGENERAL を返します。

前提条件と不変条件

  • commands は sed_init_commands によって適切に初期化されている必要があります。
  • 内部的に commands->earg に文字列 s が一時的に保持され、commands->eflag がフラグとして使用されます。

アルゴリズムまたは振る舞い

1. commands->earg にスクリプト文字列 s を設定し、commands->eflag を 1 にセットして文字列モードであることを示します。 2. fcomp 関数を呼び出し、スクリプトの構文解析とコマンドチェーンの構築を行います。 3. fcomp が成功した場合、check_finalized を呼び出してコマンドチェーンの最終的な整合性を検証し、結果を commands->canbefinal に格納します。 4. 最後に commands->eflag を 0 に戻します。 5. fcomp の戻り値に基づき、成功なら APR_SUCCESS、それ以外なら APR_EGENERAL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_sed.c:223 から呼び出されることが確認されています。
  • **呼び出し先**:
  • fcomp: スクリプトの解析を実行。
  • check_finalized: コンパイル後のコマンドチェーンの検証を実行。

関連データ構造

  • sed_commands_t: コンパイルされたコマンドチェーンと状態を保持するメイン構造体。

失敗モードと境界条件

  • fcomp がエラーを返した場合、本関数は APR_EGENERAL を返します。
  • 詳細なエラーメッセージは、sed_commands_t に登録された errfn コールバックを通じて報告されます。

保守担当者向けメモ

  • 本関数は sed_compile_file と対をなすコンパイルのエントリポイントです。
  • eflag を用いて fcomp の動作を切り替えているため、並列実行環境で sed_commands_t を共有する場合は注意が必要です。
