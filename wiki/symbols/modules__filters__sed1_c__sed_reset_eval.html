sed_reset_eval

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:305

目的

sed_eval_t 構造体で管理される sed 評価コンテキストを初期化またはリセットし、新しいコマンドセットとエラーハンドラを適用する。

シグネチャまたは形状

apr_status_t sed_reset_eval(sed_eval_t *eval, sed_commands_t *commands, sed_err_fn_t *errfn, void *data)

入力と出力

  • **入力**:
  • eval: 評価状態を保持する sed_eval_t 構造体へのポインタ。
  • commands: 実行対象となるコンパイル済みコマンド構造体。
  • errfn: エラー発生時に呼び出されるコールバック関数。
  • data: エラーハンドラに渡されるユーザー定義データ。
  • **出力**:
  • 成功時は APR_SUCCESS を返す。
  • ファイルのオープンに失敗した場合は APR_EGENERAL を返す。

前提条件と不変条件

  • eval->pool が有効な apr_pool_t であること。
  • commands が sed0.c 等で適切にコンパイル済みであること。
  • バッファ（linebuf, holdbuf, genbuf）が未割り当ての場合、eval->pool から INIT_BUF_SIZE で確保される。

アルゴリズムまたは振る舞い

1. 評価コンテキストの基本フィールド（errfn, data, commands, lnum）を更新する。 2. 未割り当てのバッファ（行、ホールド、汎用）を apr_pcalloc で初期化する。 3. 内部ポインタ（lspend, hspend, lcomend）をリセットする。 4. 状態フラグ（dolflag, sflag, quitflag 等）を初期値にリセットする。 5. commands->nrep に基づき、eval->inar を再確保する。 6. commands->nfiles に含まれる各ファイルを開き、eval->fcode にファイルハンドルを格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed1.c:299 (内部的な初期化フロー)
  • **呼び出し先**:
  • apr_pcalloc (メモリ確保)
  • apr_file_open (ファイルオープン)
  • eval_errf (エラー発生時)

関連データ構造

  • sed_eval_t: 評価状態を保持するメイン構造体。
  • sed_commands_t: 実行される sed コマンドのリスト。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • **ファイルオープン失敗**: commands->fname に指定されたファイルが開けない場合、eval_errf を呼び出し APR_EGENERAL を返す。
  • **メモリ不足**: apr_pcalloc が失敗した場合、APR の規約に従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は sed_eval_t の再利用を前提としていますが、ファイルハンドルは sed_destroy_eval で適切にクローズされている必要があります。
  • eval->finalflag はデフォルトで 1 に設定されており、単一ストリームの評価を想定しています。
