token_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_include.c:88

目的

SSI（Server Side Includes）ディレクティブの解析過程において、入力ストリームから抽出された個々のトークン（型と値）を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    token_type_t  type;
    const char   *value;
#ifdef DEBUG_INCLUDE
    const char   *s;
#endif
} token_t;

入力と出力

  • **入力**: get_ptoken 関数等の解析処理において、トークン情報を格納するための出力先として渡されます。
  • **出力**: type フィールドにトークンの種類、value フィールドに該当する文字列値が格納されます。

前提条件と不変条件

  • type は token_type_t で定義された列挙値のいずれかである必要があります。
  • DEBUG_INCLUDE が定義されている場合、デバッグ用の文字列ポインタ s が構造体に含まれます。

アルゴリズムまたは振る舞い

SSI 解析エンジンにおいて、ストリームからディレクティブの構成要素（タグ名、属性名、属性値など）を切り出す際、その結果を一時的に保持するコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: get_ptoken 関数（modules/filters/mod_include.c:990）が引数として受け取り、解析結果を書き込みます。また、マクロ定義内で参照されるケースがあります。
  • **呼び出し先**: この構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • token_type_t: token_t の type フィールドに使用される列挙型。
  • include_ctx_t: 解析コンテキストを保持する構造体で、get_ptoken を通じて token_t を利用します。

失敗モードと境界条件

  • value ポインタが指すメモリ領域は、解析対象のバッファの生存期間に依存します。解析終了後に value を参照すると無効なメモリを参照するリスクがあります。

保守担当者向けメモ

  • DEBUG_INCLUDE マクロの有無により構造体のサイズが変化するため、バイナリ互換性が求められる環境では注意が必要です。
  • value は const char * であり、この構造体自体は文字列の所有権を持ちません。メモリ管理は呼び出し側のコンテキスト（dpool 等）に依存します。
