grow_hold_buffer

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:142

目的

sed_eval_t 構造体が保持するホールドバッファ（holdbuf）のメモリ領域を、指定されたサイズまで動的に拡張します。

シグネチャまたは形状

static apr_status_t grow_hold_buffer(sed_eval_t *eval, apr_size_t newsize)

入力と出力

  • **入力**:
  • eval: 評価状態を保持する sed_eval_t 構造体へのポインタ。
  • newsize: 拡張後の目標サイズ（バイト単位）。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • メモリ確保失敗時や上限超過時は、grow_buffer が返すエラーコード（APR_ENOMEM 等）を返します。

前提条件と不変条件

  • eval ポインタは有効である必要があります。
  • 内部で呼び出される grow_buffer の制約により、最大サイズは MAX_BUF_SIZE（8MB）に制限されます。
  • バッファのメモリ管理は eval->pool（apr_pool_t）に依存しており、呼び出し側での明示的な解放は不要です。

アルゴリズムまたは振る舞い

1. eval->pool を使用して、eval->holdbuf を newsize に合わせて再確保または拡張します。 2. 拡張に伴い、eval->hspend（ホールドバッファの現在位置）および eval->hsize（現在のバッファサイズ）を更新します。 3. 処理の詳細は grow_buffer ヘルパー関数に委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed1.c:233
  • **呼び出し先**:
  • grow_buffer（modules/filters/sed1.c 内の汎用バッファ拡張関数）

関連データ構造

  • sed_eval_t: libsed の実行時評価コンテキスト。ホールドバッファのポインタ、サイズ、現在位置を保持します。

失敗モードと境界条件

  • newsize が MAX_BUF_SIZE を超える場合、grow_buffer により APR_ENOMEM が返されます。
  • apr_pool_t からのメモリ割り当てに失敗した場合、エラーが返されます。

保守担当者向けメモ

  • 本関数は sed1.c 内のホールドバッファ専用のラッパーです。バッファ拡張時にポインタが再配置される可能性があるため、sed_eval_t 内のオフセット管理に注意してください。
