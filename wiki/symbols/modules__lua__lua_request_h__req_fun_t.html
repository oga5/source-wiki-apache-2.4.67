req_fun_t

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_request.h:35

目的

Lua スクリプトから呼び出される Apache 内部の C 関数へのポインタと、その関数の型情報を保持し、ディスパッチを可能にするための構造体です。

シグネチャまたは形状

typedef struct
{
    const void *fun;
    int type;
} req_fun_t;

入力と出力

  • fun: 呼び出し対象となる C 関数のポインタ。
  • type: APL_REQ_FUNTYPE_* 定数群で定義される、関数の型を識別する整数値。

前提条件と不変条件

  • fun は実行時に適切な関数ポインタへキャストして使用されることが想定されています。
  • type は Lua 側からリクエストフィールドへアクセスする際のデータ型識別子として機能します。

アルゴリズムまたは振る舞い

Lua 側でリクエストオブジェクトのメソッドが呼び出された際、本構造体を通じて対応する C 関数が特定・実行されます。lua_request.c 内の makefun 関数によって、apr_pool_t を用いてメモリが割り当てられ、初期化されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/lua/lua_request.c 内の makefun 関数が本構造体のインスタンスを生成します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すわけではありません。保持された fun ポインタが、mod_lua のディスパッチ処理において間接的に呼び出されます。

関連データ構造

  • request_rec: Apache のリクエストコンテキスト。
  • APL_REQ_FUNTYPE_*: type フィールドに格納されるデータ型識別子。

失敗モードと境界条件

  • メモリ割り当て失敗: makefun 内で apr_palloc が失敗した場合、呼び出し元に依存します。
  • 型不整合: type と fun の指す関数のシグネチャが一致していない場合、実行時に未定義の挙動を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は lua_request.c における関数ディスパッチの基盤です。新しいリクエスト処理関数を追加する際は、適切な APL_REQ_FUNTYPE_* を定義し、本構造体経由で登録してください。
