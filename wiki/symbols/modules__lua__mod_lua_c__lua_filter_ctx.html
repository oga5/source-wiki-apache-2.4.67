lua_filter_ctx

定義

  • 種別: struct
  • 定義箇所: modules/lua/mod_lua.c:75

目的

Apache HTTP Server のフィルタ処理において、Lua コルーチン（lua_State）とバケットブリゲードを保持し、ストリーム処理のコンテキストを管理する。

シグネチャまたは形状

typedef struct
{
    apr_bucket_brigade *tmpBucket;
    lua_State *L;
    ap_lua_vm_spec *spec;
    int broken;
} lua_filter_ctx;

入力と出力

  • **入力**: lua_setup_filter_ctx 関数内で apr_pcalloc を用いてメモリが割り当てられ、初期化される。
  • **出力**: フィルタ処理の各フェーズ（lua_output_filter_handle 等）で、f->ctx を通じて参照・更新される。

前提条件と不変条件

  • L は有効な Lua コルーチン状態を指す必要がある。
  • spec は VM の実行環境仕様を保持する。
  • broken フラグは、フィルタ処理中に回復不能なエラーが発生した際に非ゼロ値となる。

アルゴリズムまたは振る舞い

  • フィルタ処理中に lua_resume を呼び出し、Lua 側の coroutine.yield() と Apache のフィルタチェーン間でデータを橋渡しする。
  • tmpBucket を使用して、フィルタ処理中のバケットブリゲードを一時的に保持する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • lua_setup_filter_ctx (初期化)
  • lua_output_filter_handle (参照)
  • lua_input_filter_handle (参照)
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • apr_bucket_brigade: フィルタ処理されるデータストリームの保持に使用。
  • lua_State: Lua スクリプト実行用のコルーチン状態。
  • ap_lua_vm_spec: VM の生成およびスコープ管理のための仕様定義。

失敗モードと境界条件

  • broken メンバが非ゼロの場合、フィルタ処理は中断される。
  • メモリ割り当て失敗時は apr_pcalloc が例外を発生させる可能性がある。

保守担当者向けメモ

  • 本構造体は f->ctx に格納されるため、フィルタのライフサイクルと同期して管理する必要がある。
  • lua_resume を介したコルーチン制御を行っているため、Lua 側でのエラーハンドリングと broken フラグの整合性に注意すること。
