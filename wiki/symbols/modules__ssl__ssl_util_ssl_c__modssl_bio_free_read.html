modssl_bio_free_read

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ssl.c:188

目的

OpenSSL の BIO オブジェクトからデータを読み取り、Apache のメモリプール (apr_pool_t) にコピーした上で、元の BIO オブジェクトを解放するユーティリティ関数です。

シグネチャまたは形状

char *modssl_bio_free_read(apr_pool_t *p, BIO *bio)

入力と出力

  • **入力**:
  • p: データの割り当てに使用する apr_pool_t ポインタ。
  • bio: 読み取り対象の OpenSSL BIO オブジェクト。
  • **出力**:
  • char *: BIO から読み取ったデータを含む、apr_pool_t 上に確保されたヌル終端文字列。データがない場合は NULL を返す。

前提条件と不変条件

  • bio は有効な OpenSSL BIO オブジェクトである必要があります。
  • 呼び出し後、bio は必ず解放されるため、呼び出し側は BIO_free を再度呼び出してはなりません。
  • 返される文字列は apr_pool_t の生存期間に依存します。

アルゴリズムまたは振る舞い

1. BIO_pending(bio) を呼び出し、読み取り可能なデータ長を取得します。 2. データが存在する場合 (len > 0):

  • apr_palloc を使用して len + 1 バイトのメモリを確保します。
  • BIO_read を使用してデータを確保したメモリに読み込みます。
  • 文字列の末尾にヌル文字 (NUL) を付与します。

    • 3. BIO_free(bio) を呼び出し、BIO オブジェクトを破棄します。 4. 読み取った文字列（またはデータがない場合は NULL）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_vars.c (複数箇所)
  • modules/ssl/ssl_util_ssl.c:217
  • modules/ssl/ssl_util_ssl.c:611
  • **呼び出し先**:
  • BIO_pending (OpenSSL)
  • apr_palloc (APR)
  • BIO_read (OpenSSL)
  • BIO_free (OpenSSL)

関連データ構造

  • BIO: OpenSSL の入出力抽象化構造体。
  • apr_pool_t: Apache Portable Runtime のメモリ管理プール。

失敗モードと境界条件

  • BIO 内にデータがない場合 (len == 0)、メモリ確保は行われず NULL が返されます。
  • BIO_read が失敗した場合の戻り値チェックはソースコード上で行われていません。

保守担当者向けメモ

  • 本関数はメモリリークを防ぐための重要なヘルパーです。OpenSSL の BIO を扱う処理で、結果を Apache のメモリ管理下に置く必要がある場合は本関数を優先的に使用してください。
  • BIO_read の戻り値が len と一致しない場合の挙動については、現在の実装では考慮されていません。
