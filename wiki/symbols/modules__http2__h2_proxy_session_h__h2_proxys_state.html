h2_proxys_state

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_proxy_session.h:37

目的

バックエンドサーバーとの HTTP/2 プロキシセッションにおける現在の処理フェーズを追跡・管理するために使用されます。

シグネチャまたは形状

typedef enum {
    H2_PROXYS_ST_INIT,
    H2_PROXYS_ST_DONE,
    H2_PROXYS_ST_IDLE,
    H2_PROXYS_ST_BUSY,
    H2_PROXYS_ST_WAIT,
    H2_PROXYS_ST_LOCAL_SHUTDOWN,
    H2_PROXYS_ST_REMOTE_SHUTDOWN,
} h2_proxys_state;

入力と出力

  • 入力: セッションの状態遷移イベント（PING 応答、GOAWAY フレーム受信、タイムアウト等）。
  • 出力: 現在のセッション状態を示す列挙値。

前提条件と不変条件

  • h2_proxy_session 構造体のメンバとして保持され、セッションのライフサイクルを通じて更新されます。
  • 状態遷移は、h2_proxy_session_process() 内の処理ロジックに依存します。

アルゴリズムまたは振る舞い

各状態は以下のフェーズを表します：

  • H2_PROXYS_ST_INIT: 初期設定（SETTINGS フレームの送信など）を実行中。
  • H2_PROXYS_ST_DONE: セッション終了、接続クローズ状態。
  • H2_PROXYS_ST_IDLE: 処理すべきストリームが存在しない待機状態。
  • H2_PROXYS_ST_BUSY: ストリームの読み書きを継続的に実行中。
  • H2_PROXYS_ST_WAIT: バックエンドからのタスク報告を待機中。
  • H2_PROXYS_ST_LOCAL_SHUTDOWN: 自ノードから GOAWAY を送信済み。
  • H2_PROXYS_ST_REMOTE_SHUTDOWN: クライアント（バックエンド）から GOAWAY を受信済み。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_proxy_session.c 内のセッション処理ロジック。
  • 呼び出し先: state_name() 関数（デバッグ用文字列変換）にて参照されています。

関連データ構造

  • h2_proxy_session: この列挙型をメンバ state として保持します。

失敗モードと境界条件

  • 状態遷移の不整合が発生した場合、セッションのクローズ処理が適切に行われない可能性があります。
  • 接続がタイムアウトまたは切断された場合、H2_PROXYS_ST_DONE へ遷移する必要があります。

保守担当者向けメモ

  • 状態遷移のロジックは h2_proxy_session.c に実装されています。新しい状態を追加する場合は、state_name() 関数も併せて更新してください。
