mpm_recycle_completion_context

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/child.c:129

目的

処理済みの接続コンテキスト（winnt_conn_ctx_t）を再利用可能な状態にリセットし、リスナー・スレッドが消費する再利用キューへ戻す。

シグネチャまたは形状

static void mpm_recycle_completion_context(winnt_conn_ctx_t *context)

入力と出力

  • **入力**: winnt_conn_ctx_t *context - 再利用対象の接続コンテキスト。
  • **出力**: なし（副作用としてキューの状態を更新）。

前提条件と不変条件

  • context が NULL でないこと。
  • context->ptrans プールがクリア可能であること。
  • context->accept_socket は切断済みだが再利用可能な状態であるため、クローズしてはならない。

アルゴリズムまたは振る舞い

1. context->ptrans プールをクリアし、バケットアロケータを再作成する。 2. context->next を NULL に初期化する。 3. context->overlapped 構造体をゼロクリアするが、hEvent ハンドルは保持し、ResetEvent を呼び出してシグナル状態を解除する。 4. グローバルな再利用キューのミューテックス qlock を取得する。 5. キューの末尾（qtail）にコンテキストを連結する。 6. キューが空だった場合は qhead を更新し、qwait_event をセットしてリスナー・スレッドに通知する。 7. ミューテックスを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/child.c:744（ワーカー・スレッドの処理終了時）。
  • **呼び出し先**: apr_pool_clear, apr_bucket_alloc_create, memset, ResetEvent, apr_thread_mutex_lock, SetEvent, apr_thread_mutex_unlock。

関連データ構造

  • winnt_conn_ctx_t: 接続状態と OVERLAPPED 構造体を保持する主要な構造体。
  • qhead, qtail: 再利用待ちのコンテキストを管理するグローバルなキュー。
  • qlock: キュー操作を保護するミューテックス。
  • qwait_event: リスナー・スレッドがコンテキストの供給を待機するためのイベント。

失敗モードと境界条件

  • context が NULL の場合、何も行わずに終了する。
  • エラーパスにおいて、コンテキストの再利用が適切に行われない場合、メモリリークが発生する可能性がある（ソース内のコメントによる）。

保守担当者向けメモ

  • 本関数は winnt_conn_ctx_t のライフサイクル管理の要です。AcceptEx を使用する環境下でのメモリ断片化を防ぐための最適化として機能しています。
  • overlapped 構造体の再利用時に hEvent を保持・リセットする手順は、IOCP との整合性を保つために必須です。
