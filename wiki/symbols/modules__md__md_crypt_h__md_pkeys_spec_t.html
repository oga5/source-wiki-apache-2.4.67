md_pkeys_spec_t

定義

  • **種別**: struct
  • **定義箇所**: modules/md/md_crypt.h:73

目的

複数の秘密鍵生成仕様（md_pkey_spec_t）を保持・管理するためのコンテナ構造体です。ACME プロトコルにおける証明書要求時など、特定の要件に基づいた秘密鍵生成を支援します。

シグネチャまたは形状

typedef struct md_pkeys_spec_t {
    apr_pool_t *p;
    struct apr_array_header_t *specs;
} md_pkeys_spec_t;

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • specs: 秘密鍵の生成仕様を格納する apr_array_header_t。
  • **出力**: なし（本構造体はデータ保持用）。

前提条件と不変条件

  • p は、この構造体および内部の specs 配列の生存期間を管理する有効な apr_pool_t である必要があります。
  • specs は md_pkey_spec_t 型の要素を格納する配列として初期化されている必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、md_pkeys_spec_make() によって生成され、md_pkeys_spec_add() や md_pkeys_spec_add_default() を通じて仕様が追加されます。
  • md_pkeys_spec_contains_rsa() を使用して、保持している仕様の中に RSA 鍵生成が含まれているかを確認する用途などで利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md.h
  • modules/md/md_acme_authz.c
  • modules/md/md_acme_authz.h
  • **呼び出し先**:
  • modules/md/md_crypt.c（md_pkeys_spec_make, md_pkeys_spec_add, md_pkeys_spec_add_default, md_pkeys_spec_contains_rsa 等の関数が本構造体を操作します）

関連データ構造

  • md_pkey_spec_t: 個別の秘密鍵生成要件（RSA ビット数や EC 曲線名など）を定義する構造体。
  • apr_array_header_t: specs を格納するために使用される APR の配列構造体。
  • apr_pool_t: メモリ管理に使用される APR プール。

失敗モードと境界条件

  • apr_pool_t のメモリ不足時には、配列の初期化や追加操作が失敗する可能性があります。
  • specs が初期化されていない状態で操作を行った場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_md の暗号化抽象化レイヤーの一部であり、OpenSSL の詳細を隠蔽する役割を担います。
  • md_pkeys_spec_t を操作する関数群は md_crypt.c に実装されています。変更を加える際は、メモリ所有権が apr_pool_t に依存している点に注意してください。
