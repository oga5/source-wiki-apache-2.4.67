authz_core_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authz_core.c:83

目的

Apache HTTP Server におけるディレクトリごとの認可設定を保持するための構造体です。Require ディレクティブによって定義される認可の論理構造（ツリー）を管理します。

シグネチャまたは形状

ソースコード上では不完全型として宣言されており、実体は modules/aaa/mod_authz_core.c 内で定義されています。

typedef struct authz_core_dir_conf authz_core_dir_conf;

入力と出力

  • **入力**: 設定ファイル（httpd.conf 等）の Require ディレクティブおよび認可セクション（<RequireAll> 等）。
  • **出力**: 認可フック処理時に参照されるディレクトリ単位の構成情報。

前提条件と不変条件

  • apr_pool_t を使用してメモリが割り当てられます。
  • 設定の構築およびマージの過程で、論理的に矛盾した設定（例: 否定のみのセクション）がないか検証されます。

アルゴリズムまたは振る舞い

  • mod_authz_core の設定管理において、ディレクトリ階層に従って設定がマージされます。
  • リクエスト処理時には、この構造体が保持する論理ツリーを apply_authz_sections が再帰的に走査し、認可プロバイダーの実行結果を統合します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_authz_core_dir_config（設定の初期化）
  • merge_authz_core_dir_config（設定のマージ）
  • authz_core_check_config（設定の検証）
  • authorize_user_core（リクエスト処理時の参照）
  • **呼び出し先**:
  • 構造体内部で authz_section_conf へのポインタを保持し、認可ロジックのツリーを形成します。

関連データ構造

  • authz_section_conf: 認可の論理単位（プロバイダー、論理演算子、否定フラグを保持）。
  • authz_logic_op: 認可セクション間の論理演算（AND/OR/NOT）を定義。

失敗モードと境界条件

  • **スタック消費**: 再帰的なツリー評価を行うため、極端に深いネストを持つ設定はスタックオーバーフローのリスクがあります。
  • **メモリ不足**: apr_pcalloc によるメモリ割り当て失敗時は、Apache の標準的なエラーハンドリングに従います。

保守担当者向けメモ

  • 本構造体は mod_authz_core の中核であり、認可の「枠組み」を定義します。個別の認証方式の詳細は各サブモジュールに委譲されているため、本構造体自体はプロバイダーの具体的な実装には依存しません。
  • 構造体のメンバ定義（authz_forbidden_on_fail 等）は、APR_OFFSETOF を用いてオフセット計算に使用される箇所があるため、メンバの順序変更には注意が必要です。
