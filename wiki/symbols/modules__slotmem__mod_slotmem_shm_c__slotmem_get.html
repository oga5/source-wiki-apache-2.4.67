slotmem_get

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:573

目的

共有メモリ内の指定されたスロットからデータを読み出し、呼び出し元が提供するバッファへコピーする。

シグネチャまたは形状

apr_status_t slotmem_get(ap_slotmem_instance_t *slot, unsigned int id,
                         unsigned char *dest, apr_size_t dest_len)

入力と出力

  • **入力**:
  • slot: 共有メモリインスタンスへのポインタ。
  • id: 読み出し対象のスロットインデックス。
  • dest: データをコピーする先のメモリ領域。
  • dest_len: コピーするデータのサイズ（バイト単位）。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • slot が NULL でないこと。
  • id が slot->desc->num（スロット総数）未満であること。
  • AP_SLOTMEM_IS_PREGRAB(slot) が有効な場合、対象スロットが使用中（inuse フラグが 1）であること。

アルゴリズムまたは振る舞い

1. slot の有効性を確認し、無効な場合は APR_ENOSHMAVAIL を返す。 2. id が範囲外であれば APR_EINVAL を返す。 3. 事前確保モード（PREGRAB）かつスロットが未使用であれば APR_NOTFOUND を返す。 4. slotmem_dptr を呼び出し、指定スロットのメモリ位置を特定する。 5. 対象スロットの inuse フラグを 1 に設定する。 6. memcpy を使用して、共有メモリ上のデータから dest へ dest_len 分のデータをコピーする。 7. APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/slotmem/mod_slotmem_plain.c:309 (参照)
  • modules/slotmem/mod_slotmem_shm.c:738 (参照)
  • **呼び出し先**:
  • slotmem_dptr (内部関数)
  • memcpy (標準ライブラリ)

関連データ構造

  • ap_slotmem_instance_t: 共有メモリインスタンスの管理構造体。
  • sharedslotdesc_t: スロットの構成情報。

失敗モードと境界条件

  • slot が NULL: APR_ENOSHMAVAIL を返す。
  • id がスロット数以上: APR_EINVAL を返す。
  • PREGRAB モードでスロットが未使用: APR_NOTFOUND を返す。
  • slotmem_dptr が失敗した場合: そのエラーコードをそのまま返す。
  • dest_len に対する境界チェックはソースコード上で明示されておらず、呼び出し側の責任となる。

保守担当者向けメモ

  • 本関数は memcpy を使用しているため、dest_len がスロットの実際のサイズを超えないよう呼び出し側で保証する必要がある。
  • inuse フラグの更新は排他制御なしで行われているため、マルチスレッド環境での整合性については slotmem サブシステム全体の排他制御設計に依存する。
