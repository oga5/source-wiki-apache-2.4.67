compare_version

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_version.c:76

目的

実行中のサーバーバージョン（httpd_version）と、指定されたバージョン文字列を数値ベースで比較し、その大小関係を判定します。

シグネチャまたは形状

int compare_version(char *version_string, const char **error)

入力と出力

  • **入力**:
  • version_string: 比較対象のバージョン文字列（major[.minor[.patch]] 形式）。
  • error: エラーメッセージを格納するためのポインタのポインタ。
  • **出力**:
  • 戻り値:
  • 1: 実行中のバージョンが指定バージョンより大きい場合。
  • -1: 実行中のバージョンが指定バージョンより小さい場合。
  • 0: 両者が等しい、または入力が不正な場合。
  • error: 入力が不正な形式である場合、エラーメッセージの文字列が設定されます。正常時は NULL。

前提条件と不変条件

  • httpd_version が初期化済みである必要があります。
  • version_string は major[.minor[.patch]] の数値形式である必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列を . で分割し、メジャー・マイナー・パッチの各数値を抽出します。 2. 数値以外の文字が含まれている場合や、形式が不正な場合は 0 を返し、error にメッセージを設定します。 3. 抽出した数値と、グローバル変数 httpd_version の各フィールドをメジャー、マイナー、パッチの順に比較します。 4. 比較結果に基づき、1（実行中の方が大きい）、-1（実行中の方が小さい）、または 0（等しい）を返します。

関連データ構造

  • httpd_version: 実行中のサーバーバージョンを保持する構造体（ap_release.h に定義）。

失敗モードと境界条件

  • **不正な形式**: major 部分が数値でない場合、または数値以外の文字が混入している場合、0 を返しエラーメッセージを設定します。
  • **バージョン欠損**: minor や patch が省略された場合、内部的に 0 として扱われます。

保守担当者向けメモ

  • 本関数は設定読み込み時に実行されるため、パフォーマンスよりも正確な構文解析が優先されます。
  • version_string は破壊的に変更（\0 の挿入）されるため、呼び出し元でコピーを渡す必要があります（現状の呼び出し元は apr_pstrmemdup を使用しています）。
