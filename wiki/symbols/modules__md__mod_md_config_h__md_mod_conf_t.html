md_mod_conf_t

定義

  • 種別: struct
  • 定義箇所: modules/md/mod_md_config.h:52

目的

Apache HTTP Server の mod_md モジュールにおけるグローバル設定を保持する構造体です。ポート番号、ベースディレクトリ、OCSP 関連のデフォルト値など、モジュール全体で共有される設定情報を管理します。

シグネチャまたは形状

typedef struct md_mod_conf_t md_mod_conf_t;

※構造体の内部メンバ定義は提供されたソース excerpt に含まれていません。

入力と出力

  • 入力: Apache の設定ファイル（<MDomain> 等）から読み込まれた値。
  • 出力: モジュール内の各コンポーネント（ACME 処理、証明書管理、OCSP ステープリング等）が実行時に参照する設定値。

前提条件と不変条件

  • Apache 起動時に md_config_create_svr によって初期化される必要があります。
  • 仮想ホストごとの設定（md_srv_conf_t）から、本構造体へのポインタを通じてグローバル設定にアクセス可能です。

アルゴリズムまたは振る舞い

  • 設定のライフサイクル管理において、md_config_merge_svr を通じて仮想ホスト設定と階層的に統合されます。
  • 実行時には、md_config_get などのアクセサ関数を介して、現在のコンテキストに応じた設定値が提供されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）: modules/md/mod_md.c 内の複数の関数（setup_store, detect_supported_protocols, link_md_to_servers, link_mds_to_servers, merge_mds_with_conf, check_usage, init_cert_watch_status など）。
  • 呼び出し先: 提供されたソース excerpt には、本構造体が呼び出す関数に関する情報は含まれていません。

関連データ構造

  • md_srv_conf_t: 仮想ホスト単位の設定構造体。本構造体へのポインタを保持します。
  • md_config_var_t: 設定値を取得する際に使用する列挙型。

失敗モードと境界条件

  • 提供されたソース excerpt には、失敗モードや境界条件に関する具体的な記述はありません。

保守担当者向けメモ

  • 本構造体は mod_md の設定管理の中核です。設定値の取得には、構造体のメンバへ直接アクセスするのではなく、md_config_gets や md_config_geti などの型安全なアクセサ関数を使用することが推奨されます。
