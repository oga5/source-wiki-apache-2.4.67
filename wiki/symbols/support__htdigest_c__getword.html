getword

定義

  • 種別: function
  • 定義箇所: support/htdigest.c:79

目的

文字列（行）から指定された区切り文字までの単語を抽出し、元の文字列からその単語と区切り文字を除去して詰め直す。

シグネチャまたは形状

int getword(char *word, char *line, char stop)

入力と出力

  • **入力**:
  • word: 抽出した文字列を格納するバッファ。
  • line: 処理対象の文字列（ポインタ）。
  • stop: 区切り文字。
  • **出力**:
  • 戻り値: 成功時は 0、MAX_STRING_LEN を超えるなどエラー時は 1。
  • word: 抽出された文字列が格納される（ヌル終端）。
  • line: 抽出された単語と区切り文字が削除され、残りの文字列が前方に詰められる。

前提条件と不変条件

  • word バッファは MAX_STRING_LEN 以上のサイズを確保している必要がある。
  • line はヌル終端された文字列であること。

アルゴリズムまたは振る舞い

1. line を先頭から走査し、stop 文字またはヌル文字に到達するまで word にコピーする。 2. コピー中に MAX_STRING_LEN - 1 に達した場合、エラーとして 1 を返す。 3. 抽出した word の末尾に \0 を付与する。 4. line 内の抽出済み部分と区切り文字をスキップし、残りの文字列を line の先頭へコピー（シフト）して詰め直す。 5. 正常終了時に 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htdigest.c:271 にて、パスワードファイルの行解析時に呼び出されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • MAX_STRING_LEN: 文字列の最大長を規定する定数（support/htdigest.c 内で定義）。

失敗モードと境界条件

  • **バッファオーバーフロー**: 抽出対象の単語が MAX_STRING_LEN - 1 を超える場合、1 を返して処理を中断する。
  • **区切り文字不在**: line 内に stop が存在しない場合、行全体が word にコピーされ、line は空になる。

保守担当者向けメモ

  • 本関数は破壊的な文字列操作を行うため、line 引数に渡すバッファは変更可能である必要がある。
  • line の詰め直し処理は while ((line[y++] = line[x++])); というイディオムで行われており、line の末尾までを正しくコピーする。
