http_post_config

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:278

目的

HTTP モジュールの設定完了時（post_config）に、MPM の非同期サポート状況の確認およびマルチパート境界文字列の初期化を行う。

シグネチャまたは形状

static int http_post_config(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定用プール。
  • plog: ログ用プール。
  • ptemp: 一時作業用プール。
  • s: サーバーレコード。
  • **出力**:
  • 成功時は OK を返す。

前提条件と不変条件

  • async_mpm 変数がモジュールスコープで定義されている必要がある。
  • ap_multipart_boundary がグローバルまたはモジュールスコープで保持される必要がある。

アルゴリズムまたは振る舞い

1. ap_mpm_query を呼び出し、AP_MPMQ_IS_ASYNC フラグを用いて MPM が非同期処理をサポートしているかを確認する。 2. 取得できない場合は async_mpm を 0 に設定する。 3. ap_random_insecure_bytes を使用して 64 ビットの乱数を生成する。 4. 生成した乱数を 16 進数文字列に変換し、ap_multipart_boundary に格納する。 5. OK を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: register_hooks 関数内にて ap_hook_post_config を介して登録される。
  • **呼び出し先**:
  • ap_mpm_query
  • ap_random_insecure_bytes
  • apr_psprintf

関連データ構造

  • server_rec: サーバー設定を保持する構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • ap_mpm_query が失敗した場合、非同期モードは無効（0）として扱われる。
  • 乱数生成の成否についてはソースコード上で明示的なエラーハンドリングは行われていない。

保守担当者向けメモ

  • 本関数はサーバー起動時の初期化フェーズで一度だけ実行される。
  • ap_multipart_boundary は、マルチパートリクエストの境界線として使用されるため、プロセス起動ごとに一意性が確保される必要がある。
