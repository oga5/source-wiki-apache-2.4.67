logio_req_t

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_logio.c:57

目的

リクエストごとの TTFB (Time To First Byte) 測定値を保持するための構造体です。

シグネチャまたは形状

typedef struct logio_req_t {
    apr_time_t ttfb;
} logio_req_t;

入力と出力

  • **入力**: ttfb フィールドに apr_time_t 型の時刻情報が格納されます。
  • **出力**: mod_log_config を通じてアクセスログに出力される TTFB 値のソースとして利用されます。

前提条件と不変条件

  • 本構造体はリクエストの生存期間中、r->request_config に格納されます。
  • LogIOTrackTTFB ディレクティブが有効な場合にのみ、TTFB の計測と本構造体の利用が行われます。

アルゴリズムまたは振る舞い

1. リクエスト処理中に ap_get_module_config(r->request_config, ...) を介してインスタンスが取得されます。 2. インスタンスが存在しない場合は apr_pcalloc を用いてリクエストプール上に確保されます。 3. フィルタ処理を通じて、リクエスト開始から最初のレスポンス送信までの時間が ttfb フィールドに記録されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/loggers/mod_logio.c:126 (値の取得)
  • modules/loggers/mod_logio.c:212 (値の取得)
  • modules/loggers/mod_logio.c:215 (メモリ確保)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • request_rec: リクエストごとの設定コンテキスト (request_config) を保持する Apache コア構造体。
  • logio_config_t: 接続ごとの入出力バイト数を保持する構造体。

失敗モードと境界条件

  • メモリ確保失敗: apr_pcalloc が失敗した場合、リクエスト処理に影響を与える可能性があります。
  • 未初期化状態: LogIOTrackTTFB が無効な場合、本構造体は生成されず、アクセスされると予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はリクエスト単位のライフサイクルを持つため、接続単位の logio_config_t と混同しないよう注意してください。
  • TTFB の計測ロジックは logio_ttfb_filter に依存しています。計測精度の変更やロジックの修正を行う際は、当該フィルタの動作を確認してください。
