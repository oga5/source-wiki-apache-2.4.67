ap_expr_t

定義

  • 種別: struct (typedef)
  • 定義箇所: include/ap_expr.h:38

目的

Apache HTTP Server の式評価エンジンにおける、構文木（Expression Tree）のノードを表現するための型です。

シグネチャまたは形状

typedef struct ap_expr_node ap_expr_t;

※ ap_expr_node の具体的な構造体定義は本ソース excerpt には含まれていません。

入力と出力

  • 入力: 構文解析フェーズにおいて ap_expr_parse により生成されます。
  • 出力: 評価フェーズにおいて ap_expr_exec 等の関数へノード単位で渡され、評価結果の算出に利用されます。

前提条件と不変条件

  • 評価には apr_pool_t を含む実行コンテキスト（ap_expr_eval_ctx_t）が必要です。
  • 構文木として構築された後、評価実行中にノードの構造が変更されることは想定されていません。

アルゴリズムまたは振る舞い

  • 式の構文木を構成する再帰的なノードとして機能します。
  • server/util_expr_eval.c において、ノードの演算子（node_op）や引数（node_arg1, node_arg2）を参照し、再帰的に評価を行うために使用されます。
  • op_Concat などの演算子ノードを判定し、式ツリーを走査するロジックの基盤となります。

呼び出し元と呼び出し先

  • 呼び出し元: ap_expr_info_t 構造体（root_node として保持）、および server/util_expr_eval.c 内の評価関数群。
  • 呼び出し先: 本シンボル自体は型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • ap_expr_info_t: 式ツリーのルートを保持するメタデータ構造体。
  • ap_expr_eval_ctx_t: 評価時の実行コンテキスト。
  • ap_expr_node: ap_expr_t が指す実体となる構造体。

失敗モードと境界条件

  • 構文解析時に不正な式が与えられた場合、有効な ap_expr_t ツリーが生成されません。
  • 評価時に適切なコンテキストが提供されない場合、またはメモリ不足が発生した場合は評価が失敗します。

保守担当者向けメモ

  • ap_expr_node の内部構造（node_op や node_arg 等）は server/util_expr_private.h に定義されています。
  • 評価ロジックの変更や新しい演算子の追加を行う際は、server/util_expr_eval.c における ap_expr_t の走査ロジックを更新する必要があります。
