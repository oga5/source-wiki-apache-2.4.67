log_scripterror

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:64

目的

CGIスクリプトの標準エラー出力（stderr）をApacheのエラーログに記録し、必要に応じて設定されたログファイルへ詳細なエラー情報を追記する。

シグネチャまたは形状

static int log_scripterror(request_rec *r, cgi_server_conf *conf, int ret,
                           apr_status_t rv, const char *logno,
                           const char *error)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • conf: CGIサーバー設定構造体。
  • ret: 呼び出し元が返す予定のHTTPステータスコード。
  • rv: 発生したAPRステータスコード。
  • logno: エラーログ識別子（APLOGNO）。
  • error: 記録するエラーメッセージ文字列。
  • **出力**:
  • 戻り値: 引数 ret で渡された整数値をそのまま返す。

前提条件と不変条件

  • r->pool は有効なメモリプールである必要がある。
  • conf->logname が設定されている場合、そのファイルへの書き込み権限が必要。
  • logno が提供されない場合、ap_log_rerror は空文字列として処理する。

アルゴリズムまたは振る舞い

1. ap_log_rerror を使用し、標準のApacheエラーログへ stderr 内容を書き出す。 2. conf->logname が指定されている場合、以下の条件を順に確認する：

  • logname が未設定でないこと。
  • apr_stat でファイルサイズを取得し、conf->logbytes（最大サイズ）を超えていないこと。
  • ファイルを追記モード（APR_APPEND|APR_WRITE|APR_CREATE）で正常に開けること。

    • 3. 条件を満たせば、ファイルにタイムスタンプ、リクエストメソッド、URI、引数、プロトコル、ステータスコード、およびエラー内容をフォーマットして書き込む。 4. ファイルを閉じ、引数 ret を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/cgi_common.h:122
  • modules/generators/mod_cgi.c (複数箇所)
  • modules/generators/mod_cgid.c (複数箇所)
  • **呼び出し先**:
  • ap_log_rerror (Apache Core)
  • apr_stat, apr_file_open, apr_file_printf, apr_file_close (APR)
  • apr_ctime, apr_time_now (APR)

関連データ構造

  • request_rec: リクエスト情報。
  • cgi_server_conf: CGIモジュールの設定情報（ログファイル名やサイズ制限を含む）。
  • apr_file_t, apr_finfo_t: APRのファイル操作用構造体。

失敗モードと境界条件

  • conf->logname が指定されていても、ファイルオープンに失敗した場合やサイズ制限を超過した場合は、ログファイルへの追記をスキップし、エラーログへの出力のみを行って ret を返す。
  • apr_stat や apr_file_open の呼び出しは、高負荷時にパフォーマンスコストとなる可能性がある。

保守担当者向けメモ

  • 本関数は APLOGNO を直接持たず、呼び出し元から提供される設計となっている。
  • logname への書き込み処理はメインラインのパスとして実行されるため、頻繁なエラー発生時にはファイルI/Oがボトルネックとなる可能性がある。
