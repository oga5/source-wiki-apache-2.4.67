proxy_http2_canon

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_proxy_http2.c:104

目的

プロキシリクエストの URL を解析・正規化し、Apache の内部プロキシ形式（proxy:h2://... または proxy:h2c://...）へ変換する。

シグネチャまたは形状

static int proxy_http2_canon(request_rec *r, char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • url: 正規化対象の URL 文字列。
  • **出力**:
  • 成功時: OK を返し、r->filename に正規化された内部プロキシ URL を設定する。
  • 失敗時: DECLINED（スキーム不一致）、HTTP_BAD_REQUEST（解析エラー）、または HTTP_FORBIDDEN（不正な文字を含む場合）を返す。

前提条件と不変条件

  • URL が h2: または h2c: スキームで開始されている必要がある。
  • r->proxyreq が PROXYREQ_REVERSE または PROXYREQ_PROXY であることを前提とする。

アルゴリズムまたは振る舞い

1. **スキーム判定**: h2: (HTTPS 相当) または h2c: (HTTP 相当) を識別し、デフォルトポートを決定する。 2. **ネットワーク位置の解析**: ap_proxy_canon_netloc を使用してホスト名とポートを抽出する。 3. **パスとクエリの正規化**:

  • PROXYREQ_REVERSE の場合: proxy-nocanon や proxy-noencode ノートの設定に基づき、パスのエンコードを行う（ap_proxy_canonenc 等を使用）。
  • PROXYREQ_PROXY の場合: URL をそのまま保持する。

    • 4. **バリデーション**: パスまたはクエリ文字列に制御文字やスペースが含まれていないかチェックする。 5. **URL 再構築**: IPv6 アドレスの括弧補完やポート番号の付与を行い、r->filename に proxy:<scheme>://<host>[:port]/<path>[?<query>] の形式で格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_proxy_http2.c 内の proxy_hook_canon_handler に登録され、Apache の正規化フックとして呼び出される。
  • **呼び出し先**:
  • ap_proxy_port_of_scheme, ap_proxy_canon_netloc, ap_proxy_canonenc, ap_proxy_canonenc_ex（プロキシユーティリティ）
  • ap_log_rerror（ログ出力）
  • apr_table_get, apr_pstrcat, apr_snprintf（APR ユーティリティ）

関連データ構造

  • request_rec: Apache のリクエスト状態を保持する構造体。
  • proxy_req_t (間接的): r->proxyreq を通じてプロキシの種類を判定。

失敗モードと境界条件

  • **スキーム不一致**: h2 / h2c 以外は DECLINED を返す。
  • **解析エラー**: ap_proxy_canon_netloc が失敗した場合、HTTP_BAD_REQUEST を返す。
  • **不正文字**: パスやクエリに制御文字が含まれる場合、HTTP_FORBIDDEN を返す。
  • **エンコード失敗**: ap_proxy_canonenc が NULL を返した場合、HTTP_BAD_REQUEST を返す。

保守担当者向けメモ

  • 本関数は proxy_hook_canon_handler を通じて APR_HOOK_FIRST で登録されているため、他のプロキシモジュールより先に処理される可能性がある。
  • PROXY_CANONENC_NOENCODEDSLASHENCODING マクロの定義有無により、パスエンコードの挙動が分岐する。
