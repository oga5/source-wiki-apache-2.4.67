proxy_address

定義

  • 種別: struct
  • 定義箇所: modules/proxy/proxy_util.c:62

目的

プロキシ接続先のアドレス情報を保持し、TTL（生存期間）および参照カウントによるライフサイクル管理を行うための構造体です。

シグネチャまたは形状

typedef struct proxy_address {
    apr_sockaddr_t *addr;       /* リモートアドレス情報 */
    const char *hostname;       /* リモートホスト名 */
    apr_port_t hostport;        /* リモートホストポート */
    apr_uint32_t refcount;      /* 接続およびワーカーによる参照カウント */
    apr_uint32_t expiry;        /* 有効期限（proxy_start_time からの秒数） */
} proxy_address;

入力と出力

本構造体はデータ保持を目的としており、直接的な関数入出力は持ちませんが、proxy_address_inc や proxy_address_dec 等のユーティリティ関数を介して参照カウントの操作が行われます。

前提条件と不変条件

  • refcount は、このアドレスを使用している接続数やワーカー数と同期している必要があります。
  • expiry は、proxy_start_time を基準とした相対的なタイムスタンプとして扱われます。

アルゴリズムまたは振る舞い

  • 接続の再利用性を判定するためのメタデータとして機能します。
  • apr_atomic_xchgptr を用いたアトミックな交換操作により、proxy_worker が保持するアドレスの更新が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/proxy_util.c 内の proxy_address_inc, proxy_address_dec, worker_address_get 等の関数が本構造体を操作します。
  • **呼び出し先**: apr_sockaddr_t（APRライブラリ）を内部で保持しています。

関連データ構造

  • proxy_worker: 本構造体を保持し、接続先アドレスの管理に使用します。
  • apr_sockaddr_t: ネットワークアドレスの低レイヤー表現。

失敗モードと境界条件

  • 参照カウントが適切に管理されない場合、メモリリークや早期解放が発生する可能性があります。
  • expiry の計算において、システム時刻の不整合が影響する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_proxy.h において不透明な（opaque）TTL付きアドレスとして定義されています。
  • 共有メモリ環境下での操作が含まれるため、アトミック操作の整合性に注意してください。
