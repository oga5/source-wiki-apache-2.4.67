ap_register_input_filter

定義

  • 種別: function
  • 定義箇所: server/util_filter.c:246

目的

入力フィルタをシステムに登録し、指定された名前でフィルタを識別・利用可能にする。

シグネチャまたは形状

ap_filter_rec_t *ap_register_input_filter(
    const char *name,
    ap_in_filter_func filter_func,
    ap_init_filter_func filter_init,
    ap_filter_type ftype
)

入力と出力

  • **入力**:
  • name: フィルタを一意に識別するための名称。
  • filter_func: フィルタ処理を実行する関数ポインタ（ap_in_filter_func）。
  • filter_init: フィルタ初期化時に呼び出される関数ポインタ（任意、NULL可）。
  • ftype: フィルタの型（ap_filter_type）。チェーン内での挿入順序を決定する。
  • **出力**:
  • 登録されたフィルタの定義情報を含む ap_filter_rec_t 構造体へのポインタを返す。

前提条件と不変条件

  • name はフィルタを検索するためのトライ木構造に登録される。
  • ftype はフィルタチェーン内での相対的な位置関係を規定する。

アルゴリズムまたは振る舞い

1. 受け取った filter_func を内部の汎用フィルタ関数構造体 ap_filter_func に格納する。 2. register_filter 関数を呼び出し、指定された名前、関数、初期化関数、型を registered_input_filters トライ木に登録する。 3. 登録されたフィルタの定義情報を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/debugging/mod_dumpio.c
  • modules/examples/mod_case_filter_in.c
  • modules/filters/mod_buffer.c
  • modules/filters/mod_charset_lite.c
  • modules/filters/mod_deflate.c
  • modules/filters/mod_ext_filter.c
  • modules/filters/mod_reqtimeout.c
  • modules/filters/mod_request.c
  • **呼び出し先**:
  • register_filter (内部関数)

関連データ構造

  • ap_filter_rec_t: フィルタの定義情報を保持する構造体。
  • ap_in_filter_func: 入力フィルタの関数型。
  • registered_input_filters: 入力フィルタを管理するトライ木のルート。

失敗モードと境界条件

  • フィルタ名が重複している場合や、メモリ不足によりトライ木への登録が失敗した場合の挙動は register_filter の実装に依存する（本ソース excerpt には詳細なし）。

保守担当者向けメモ

  • 本関数は include/util_filter.h で宣言されており、モジュールが独自の入力フィルタを定義する際の標準的なインターフェースである。
  • フィルタの順序は ftype に依存するため、既存のフィルタとの競合や順序関係に注意すること。
