piped_log

定義

  • 種別: struct
  • 定義箇所: include/http_log.h:776

目的

外部プロセスへログを転送するためのパイプ処理を抽象化・管理する不透明構造体です。

シグネチャまたは形状

typedef struct piped_log piped_log;

※構造体の内部メンバは公開されておらず、不透明（opaque）な型として扱われます。

入力と出力

  • 入力: ap_open_piped_log 等の関数を通じて、メモリプール（apr_pool_t）および実行プログラム名を受け取ります。
  • 出力: ログ書き込み用のファイル記述子（apr_file_t）を保持し、ログ出力先として機能します。

前提条件と不変条件

  • ap_open_piped_log 等で作成される構造体は、生成時に渡された apr_pool_t のライフサイクルに従います。
  • 構造体のメモリ管理は Apache のプールメカニズムに依存します。

アルゴリズムまたは振る舞い

  • 外部プロセスとのパイプ接続を確立し、ログデータをそのプロセスへ転送します。
  • ap_piped_log_read_fd および ap_piped_log_write_fd を通じて、パイプの読み取り側および書き込み側のファイル記述子を提供します。
  • ap_close_piped_log により、パイプのクローズと関連リソースの解放が行われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • modules/loggers/mod_log_config.c
  • modules/loggers/mod_log_forensic.c
  • server/log.c
  • 呼び出し先:
  • ap_open_piped_log / ap_open_piped_log_ex（生成）
  • ap_close_piped_log（破棄）
  • ap_piped_log_read_fd / ap_piped_log_write_fd（アクセス）
  • piped_log_spawn（内部的なプロセス生成処理）

関連データ構造

  • apr_pool_t: ライフサイクル管理に使用。
  • apr_file_t: パイプの入出力用ファイル記述子として使用。

失敗モードと境界条件

  • 外部プロセスの起動失敗やパイプの作成失敗については、ap_open_piped_log の戻り値および server/log.c 内の内部処理で管理されます。

保守担当者向けメモ

  • 本構造体は不透明型であるため、直接メンバにアクセスせず、必ず提供されている ap_ で始まる API を介して操作してください。
  • ログ出力のフォーマット文字列には、セキュリティ上の理由から未検証の外部データを直接含めないでください。
