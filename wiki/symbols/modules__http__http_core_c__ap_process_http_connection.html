ap_process_http_connection

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:243

目的

MPM（Multi-Processing Module）の特性に基づき、HTTP 接続の処理を同期または非同期の適切な実装へ振り分けるエントリポイントとして機能します。

シグネチャまたは形状

int ap_process_http_connection(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（処理対象の接続レコード）
  • **出力**: int（処理結果のステータスコード。呼び出し先の関数が返す値をそのまま返却する）

前提条件と不変条件

  • async_mpm 変数が post_config フェーズで適切に初期化されている必要があります。
  • c は有効な接続レコードを指している必要があります。

アルゴリズムまたは振る舞い

1. async_mpm が有効であり、かつ接続 c の入力フィルタがブロッキング（clogging）状態でない場合、ap_process_http_async_connection(c) を呼び出します。 2. それ以外の場合（同期 MPM またはブロッキングフィルタが存在する場合）、ap_process_http_sync_connection(c) を呼び出します。 3. 呼び出し先の戻り値をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_core.c:293（ap_hook_process_connection への登録を通じて呼び出されます）
  • **呼び出し先**:
  • ap_process_http_async_connection
  • ap_process_http_sync_connection

関連データ構造

  • conn_rec: 接続の状態やフィルタチェーンを保持する構造体。

失敗モードと境界条件

  • 呼び出し先の関数（ap_process_http_async_connection または ap_process_http_sync_connection）が失敗した場合、そのエラーコードがそのまま伝播されます。

保守担当者向けメモ

  • 本関数は単なるディスパッチャーであり、具体的なプロトコル処理ロジックは含まれていません。
  • 非同期処理への分岐条件である !c->clogging_input_filters は、フィルタチェーンが非同期 I/O をサポートしているかを判断する重要なフラグです。
