xlate_insert_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_charset_lite.c:314

目的

リクエスト処理のフィルタチェーン構築時に、文字セット変換フィルタ（XLATEIN および XLATEOUT）を動的に挿入するかどうかを判定し、必要に応じて挿入を実行する。

シグネチャまたは形状

static void xlate_insert_filter(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）。
  • **出力**: なし（戻り値は void）。副作用として、フィルタチェーンへのフィルタ追加が行われる。

前提条件と不変条件

  • r->request_config から charset_req_t が取得可能であること。
  • r->per_dir_config から charset_dir_t が取得可能であること。
  • フィルタの挿入は、CharsetOptions ディレクティブの設定および、リクエストごとの変換コンテキスト（reqinfo->output_ctx / reqinfo->input_ctx）の存在に依存する。

アルゴリズムまたは振る舞い

1. リクエスト設定 (reqinfo) およびディレクトリ設定 (dc) を取得する。 2. dc->implicit_add が IA_NOIMPADD である場合、暗黙的なフィルタ追加を禁止して終了する。 3. reqinfo が存在する場合、以下の処理を順次行う：

  • **出力フィルタ**: reqinfo->output_ctx が存在し、かつ XLATEOUT_FILTER_NAME が未登録であれば、ap_add_output_filter を呼び出して追加する。
  • **入力フィルタ**: reqinfo->input_ctx が存在し、かつ XLATEIN_FILTER_NAME が未登録であれば、ap_add_input_filter を呼び出して追加する。

    • 4. 各ステップで、フィルタが追加されなかった場合はその理由を APLOG_TRACE3 レベルでログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_insert_filter を介して Apache のフィルタ挿入フックから呼び出される（modules/filters/mod_charset_lite.c:1125）。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_log_rerror
  • configured_on_output / configured_on_input
  • ap_add_output_filter / ap_add_input_filter

関連データ構造

  • charset_req_t: リクエストごとの変換コンテキストを保持。
  • charset_dir_t: ディレクトリごとの設定（implicit_add フラグなど）を保持。
  • request_rec: Apache のリクエスト構造体。

失敗モードと境界条件

  • reqinfo が NULL の場合、フィルタの挿入はスキップされる。
  • CharsetOptions で NoImplicitAdd が指定されている場合、フィルタは挿入されない。
  • 既に他のモジュールによって同名のフィルタが追加されている場合、重複追加は回避される。

保守担当者向けメモ

  • 本関数は APR_HOOK_REALLY_LAST で登録されており、他のフィルタがすべて挿入された後に実行される設計となっている。
  • reqinfo が NULL になる可能性があるため、dc の参照前に reqinfo の有無を確認するロジックが重要である。
