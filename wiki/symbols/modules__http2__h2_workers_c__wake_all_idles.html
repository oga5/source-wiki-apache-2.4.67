wake_all_idles

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_workers.c:361

目的

HTTP/2 ワーカープールにおいて、現在アイドル状態にあるすべてのスレッドに対して、処理すべき作業が発生したことを通知し、ウェイクアップさせる。

シグネチャまたは形状

static void wake_all_idles(h2_workers *workers)

入力と出力

  • **入力**: workers (h2_workers *) - ワーカープールを管理する構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • workers ポインタは有効であること。
  • 本関数を呼び出すスレッドは、workers->lock ミューテックスを保持している必要がある（リング操作の整合性のため）。

アルゴリズムまたは振る舞い

1. workers->idle リングバッファの先頭から走査を開始する。 2. リングが空（APR_RING_SENTINEL に到達）になるまで、各 h2_slot を順次処理する。 3. 各アイドルスロットに対して apr_thread_cond_signal(slot->more_work) を実行し、待機中のスレッドを起床させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_workers.c:385
  • modules/http2/h2_workers.c:399
  • modules/http2/h2_workers.c:557
  • **呼び出し先**:
  • apr_thread_cond_signal (APRライブラリ関数)

関連データ構造

  • h2_workers: アイドル状態のスロットを保持するリングバッファ idle を含む。
  • h2_slot: 個別のワーカー状態と、起床用条件変数 more_work を保持する。

失敗モードと境界条件

  • アイドルリストが空の場合、ループは即座に終了し、何も行われない。
  • slot->more_work が初期化されていない場合、未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は workers->lock による保護下で実行されることを前提としている。ロックなしで呼び出すと、リングバッファの走査中に競合が発生するリスクがある。
