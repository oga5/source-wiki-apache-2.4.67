cache_info

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache.h:58

目的

キャッシュされたコンテンツのメタデータ（有効期限やステータス情報など）を保持し、キャッシュの鮮度管理や検証を行うための構造体です。

シグネチャまたは形状

struct cache_info {
    /* メンバの詳細はソースコードの定義箇所を参照 */
};

入力と出力

  • 入力: キャッシュの保存時（store_headers）や、既存キャッシュの読み込み時に利用されます。
  • 出力: ap_cache_current_age 関数等へ渡され、キャッシュの有効性判定や経過時間の計算に使用されます。

前提条件と不変条件

  • cache_handle_t を介してアクセスされるキャッシュオブジェクトの一部として管理されます。
  • メモリ割り当てには apr_pcalloc 等が使用されることが一般的です。

アルゴリズムまたは振る舞い

  • キャッシュのヒット/ミス判定や、Cache-Control ヘッダーに基づく有効期限の計算ロジックにおいて、中心的なデータソースとして機能します。
  • mod_cache のポリシーエンジンと、各ストレージプロバイダ（mod_cache_disk 等）の間でメタデータを共有するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照・操作）:
  • modules/cache/cache_util.c: ap_cache_current_age 等のユーティリティ関数。
  • modules/cache/mod_cache.c: キャッシュの保存・検索ロジック。
  • modules/cache/mod_cache_disk.c: ディスクストレージへの保存・読み込み処理。
  • 呼び出し先:
  • 本構造体はデータ構造であり、関数ポインタを直接保持するものではありません。

関連データ構造

  • cache_handle_t: キャッシュエントリへのアクセスハンドル。本構造体をメンバとして保持します。
  • cache_provider: ストレージバックエンドのインターフェース。store_headers 関数等の引数として本構造体を利用します。

失敗モードと境界条件

  • 構造体が初期化されていない場合や、メモリ不足により割り当てに失敗した場合、キャッシュの検証ロジックが正しく動作しません。
  • 外部からの入力（HTTP ヘッダー等）に依存するため、不正な日付形式やメタデータが含まれる場合の境界条件については、cache_util.c 側のバリデーションロジックに依存します。

保守担当者向けメモ

  • 本構造体は mod_cache のコア契約の一部です。メンバ変数を変更する場合は、すべてのストレージプロバイダ（mod_cache_disk, mod_cache_socache 等）の実装に影響を与える可能性があるため注意してください。
