count_string

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_forensic.c:149

目的

フォレンジックログ出力時に、エスケープ処理が必要な文字を考慮した上で、文字列の出力に必要なバッファサイズを算出する。

シグネチャまたは形状

static apr_size_t count_string(const char *p)

入力と出力

  • **入力**: p (計測対象のヌル終端文字列)
  • **出力**: エスケープ後の文字数を含む、必要なバッファサイズ（apr_size_t）

前提条件と不変条件

  • 入力文字列 p はヌル終端されていること。
  • TEST_CHAR マクロおよび T_ESCAPE_FORENSIC 定数が、エスケープ対象文字の判定に利用可能であること。

アルゴリズムまたは振る舞い

1. 文字列 p を先頭から順に走査する。 2. 各文字が TEST_CHAR(c, T_ESCAPE_FORENSIC) に合致する場合、その文字は %XX 形式（3文字）に変換されるため、サイズとして3を加算する（n を1進め、さらに2を加算）。 3. 合致しない場合は、通常の1文字としてサイズを1加算する。 4. 文字列の終端に達するまで繰り返し、最終的な合計サイズを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/loggers/mod_log_forensic.c:163 (ヘッダー情報のサイズ計算)
  • modules/loggers/mod_log_forensic.c:208 (リクエストラインのサイズ計算)
  • **呼び出し先**:
  • TEST_CHAR マクロ（test_char.h 由来）

関連データ構造

  • apr_size_t: サイズ計算に使用される型。

失敗モードと境界条件

  • 空文字列が渡された場合、0を返す。
  • エスケープ対象文字が連続する場合、それぞれ個別にサイズが加算される。

保守担当者向けメモ

  • 本関数は log_escape と対になる計算ロジックである。log_escape の実装に変更を加える際は、本関数の計算ロジックも同期させる必要がある。
