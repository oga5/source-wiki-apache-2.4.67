match_address

定義

  • 種別: function
  • 定義箇所: server/listen.c:384

目的

指定されたソケットアドレス情報（sa）が、ホスト名、ポート番号、およびスコープIDの条件と一致するかを判定します。

シグネチャまたは形状

static int match_address(const apr_sockaddr_t *sa,
                         const char *hostname, apr_port_t port,
                         const char *scope_id, apr_pool_t *p)

入力と出力

  • **入力**:
  • sa: 比較対象の apr_sockaddr_t 構造体。
  • hostname: 比較対象のホスト名文字列（NULL 許容）。
  • port: 比較対象のポート番号。
  • scope_id: 比較対象のスコープID文字列（NULL 許容）。
  • p: メモリ割り当て用のプール。
  • **出力**:
  • int: 条件がすべて一致すれば非ゼロ（真）、そうでなければ 0（偽）を返します。

前提条件と不変条件

  • APR_VERSION_AT_LEAST(1,7,0) が定義されている場合、apr_sockaddr_zone_get を使用して sa からスコープIDを取得します。取得に失敗した場合、スコープIDは NULL として扱われます。

アルゴリズムまたは振る舞い

以下の3つの条件がすべて満たされた場合に真を返します。 1. port が sa->port と一致すること。 2. hostname が指定されていない場合は sa->hostname も NULL であること、または両者が指定されている場合に文字列として一致すること。 3. scope_id が指定されていない場合は取得したスコープIDも NULL であること、または両者が指定されている場合に文字列として一致すること。

呼び出し元と呼び出し先

  • **呼び出し元**: server/listen.c:420（find_listeners 関数内から呼び出されていることが確認されています）。
  • **呼び出し先**:
  • apr_sockaddr_zone_get (APR 1.7.0 以上の場合)
  • strcmp

関連データ構造

  • apr_sockaddr_t: APR のソケットアドレス構造体。
  • apr_port_t: ポート番号型。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • hostname や scope_id が NULL の場合、対応する sa 側のフィールドも NULL であることを要求します。
  • apr_sockaddr_zone_get の呼び出し失敗は無視され、スコープIDの比較は NULL との比較として継続されます。

保守担当者向けメモ

  • 本関数は server/listen.c 内のリスナー検索ロジックの一部として使用されています。
  • IPv6 のゾーンインデックス（スコープID）の扱いは APR のバージョンに依存するため、環境によって挙動が異なる可能性があります。
