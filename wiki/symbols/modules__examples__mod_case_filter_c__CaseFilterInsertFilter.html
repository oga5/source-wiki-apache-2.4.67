CaseFilterInsertFilter

定義

  • 種別: function
  • 定義箇所: modules/examples/mod_case_filter.c:44

目的

リクエスト処理の過程で、設定に基づいて出力フィルタ（CaseFilterOutFilter）をフィルタチェーンに追加するかを判断するフック関数です。

シグネチャまたは形状

static void CaseFilterInsertFilter(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）
  • **出力**: なし（戻り値は void）

前提条件と不変条件

  • r->server->module_config から取得した CaseFilterConfig が有効であること。
  • フィルタの追加は、設定構造体内の bEnabled フラグが真である場合にのみ行われます。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定から CaseFilterConfig を取得します。 2. 設定内の bEnabled を確認し、無効であれば即座に終了します。 3. 有効な場合、ap_add_output_filter を呼び出し、s_szCaseFilterName で定義されたフィルタを現在のリクエストおよび接続のフィルタチェーンに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の insert_filter フック（mod_case_filter.c:125 にて ap_hook_insert_filter により登録）。
  • **呼び出し先**:
  • ap_get_module_config（設定取得）
  • ap_add_output_filter（フィルタの登録）

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • CaseFilterConfig: モジュールの有効/無効状態を保持する設定構造体。
  • case_filter_module: モジュール定義構造体。

失敗モードと境界条件

  • 設定構造体が取得できない場合や、bEnabled が偽の場合はフィルタの追加を行わずに正常終了します。

保守担当者向けメモ

  • 本関数は insert_filter フックとして登録されています。フィルタの追加条件を変更する場合は、CaseFilterConfig の状態管理ロジックと併せて確認してください。
