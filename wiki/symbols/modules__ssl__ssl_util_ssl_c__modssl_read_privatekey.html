modssl_read_privatekey

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ssl.c:77

目的

指定されたファイルから秘密鍵を読み込み、OpenSSL の EVP_PKEY 構造体として返します。PEM 形式、Base64 エンコードされた DER 形式、およびバイナリ DER 形式を順次試行する自動判別機能を備えています。

シグネチャまたは形状

EVP_PKEY *modssl_read_privatekey(const char *filename, pem_password_cb *cb, void *s)

入力と出力

  • **入力**:
  • filename: 読み込む秘密鍵ファイルのパス。
  • cb: パスフレーズが必要な場合に呼び出されるコールバック関数。
  • s: コールバック関数に渡されるユーザーデータ。
  • **出力**:
  • 成功時: 読み込まれた秘密鍵へのポインタ (EVP_PKEY*)。
  • 失敗時: NULL。

前提条件と不変条件

  • filename で指定されたファイルは、読み取り権限がある状態で存在する必要があります。
  • 内部で BIO を使用してファイルを開くため、ファイルシステムのリソースが利用可能である必要があります。

アルゴリズムまたは振る舞い

以下の順序で読み込みを試行し、最初に成功した形式を採用します。

1. **PEM 形式**: PEM_read_bio_PrivateKey を使用して試行します。 2. **Base64 エンコードされた DER 形式**: BIO_f_base64() を適用した BIO を経由して d2i_PrivateKey_bio を使用して試行します。 3. **バイナリ DER 形式**: d2i_PrivateKey_bio を直接使用して試行します。

各ステップで BIO オブジェクトの生成と解放（BIO_free または BIO_free_all）が適切に行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_pphrase.c:227 (直接の呼び出しを確認)
  • **呼び出し先**:
  • OpenSSL API: BIO_new_file, PEM_read_bio_PrivateKey, BIO_free, BIO_new, BIO_f_base64, BIO_push, d2i_PrivateKey_bio, BIO_free_all

関連データ構造

  • EVP_PKEY: OpenSSL の秘密鍵構造体。
  • BIO: OpenSSL の I/O 抽象化レイヤー。

失敗モードと境界条件

  • ファイルが存在しない、または読み取り不可の場合、各ステップで NULL が返されます。
  • どの形式でも読み込みに失敗した場合、最終的に NULL が返されます。
  • パスフレーズが不正な場合、PEM_read_bio_PrivateKey が失敗し、後続の形式判定へ進みます。

保守担当者向けメモ

  • 本関数は OpenSSL の複数の読み込み関数をラップしており、将来的な OpenSSL の API 変更（特に d2i_PrivateKey_bio 等の挙動）に際しては、各ステップの戻り値と BIO の解放処理が正しく維持されているか確認してください。
  • BIO のスタック操作（BIO_push）を行っているため、エラー発生時のメモリリークを防ぐための BIO_free_all の呼び出し順序には注意が必要です。
