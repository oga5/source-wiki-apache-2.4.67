register_hook

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_byrequests.c:151

目的

Apache HTTP Server の mod_proxy におけるロードバランシングプロバイダーとして「byrequests」メソッドを登録し、設定読み込み後の初期化フックを構成します。

シグネチャまたは形状

static void register_hook(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（フック登録に使用するメモリプール）
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • 本関数は、モジュールのロード時に Apache のフック機構によって呼び出されることを前提としています。
  • byrequests 構造体が定義済みであり、PROXY_LBMETHOD プロバイダーとして登録可能である必要があります。

アルゴリズムまたは振る舞い

1. ap_register_provider を呼び出し、PROXY_LBMETHOD インターフェースに対して "byrequests" という名称で byrequests 構造体を登録します。 2. ap_hook_post_config を呼び出し、lbmethod_byrequests_post_config 関数を APR_HOOK_MIDDLE の優先度で登録します。これにより、サーバー設定完了後に必要な初期化処理が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール読み込みプロセス（mod_lbmethod_byrequests.c 内のフック登録箇所から間接的に呼び出されます）。
  • **呼び出し先**:
  • ap_register_provider
  • ap_hook_post_config
  • lbmethod_byrequests_post_config

関連データ構造

  • byrequests: proxy_balancer_method 型の構造体。本モジュールのロードバランシングアルゴリズムを定義します。

失敗モードと境界条件

  • メモリプール p が無効な場合、登録処理が失敗する可能性があります。
  • 他のモジュールが既に "byrequests" という名称で PROXY_LBMETHOD プロバイダーを登録している場合、登録が競合する可能性があります。

保守担当者向けメモ

  • 本関数はモジュールのエントリポイントの一部です。lbmethod_byrequests_post_config の登録順序や優先度を変更する場合は、他のロードバランサーモジュールとの整合性に注意してください。
  • ツールによる呼び出し元情報の解析では、同名の register_hook 関数が他のモジュール（mod_cgid 等）にも存在するため、混同しないよう注意が必要です。本関数は mod_lbmethod_byrequests.c 内で定義されたものに限定されます。
