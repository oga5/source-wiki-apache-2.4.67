etag_uint64_to_hex

定義

  • 種別: function
  • 定義箇所: modules/http/http_etag.c:44

目的

apr_uint64_t 型の数値を 16 進数文字列に変換し、指定されたバッファポインタの末尾に書き込みます。主にファイル属性（inode, サイズ, 更新日時）を ETag 文字列へエンコードするために使用されます。

シグネチャまたは形状

static char *etag_uint64_to_hex(char *next, apr_uint64_t u)

入力と出力

  • **入力**:
  • next: 変換後の文字列を書き込むバッファの開始位置。
  • u: 16 進数へ変換する apr_uint64_t 型の数値。
  • **出力**:
  • 書き込み後のバッファの次の位置を指すポインタを返します。

前提条件と不変条件

  • next が指すバッファは、変換結果を格納するのに十分なサイズ（最大 16 文字分）が確保されている必要があります。
  • HEX_DIGITS 配列がスコープ内で利用可能である必要があります。

アルゴリズムまたは振る舞い

1. shift を 60 から 4 ずつ減らしながら、上位ビットから順に 4 ビットずつ取り出します。 2. printing フラグを用いて、先行するゼロを抑制します（ただし、数値が 0 の場合や、一度でも非ゼロが現れた後のゼロは出力されます）。 3. 各 4 ビットの値を HEX_DIGITS テーブルで文字に変換し、next が指す位置に書き込みます。 4. 最後に、残りの下位 4 ビットを必ず書き込みます。 5. 最終的なバッファの書き込み終了位置を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_etag.c:317
  • modules/http/http_etag.c:324
  • modules/http/http_etag.c:331
  • modules/http/http_etag.c:345
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。HEX_DIGITS マクロ/配列を参照します。

関連データ構造

  • apr_uint64_t: 入力値の型。
  • HEX_DIGITS: 16 進数変換用の文字テーブル。

失敗モードと境界条件

  • 入力値 u が 0 の場合、printing フラグが立たないままループが終了し、最終行の処理によって '0' が 1 文字だけ書き込まれます。
  • バッファオーバーフローに対する保護は関数内には存在しません。呼び出し側で十分な領域を確保する必要があります。

保守担当者向けメモ

  • 本関数は http_etag.c 内の内部ヘルパーであり、ETag 生成のパフォーマンスに直結します。
  • HEX_DIGITS の定義が変更された場合、本関数の出力結果も影響を受けます。
