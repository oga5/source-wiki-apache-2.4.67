fixup_dir

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_dir.c:224

目的

ディレクトリに対するリクエストを処理し、末尾スラッシュの補完（リダイレクト）または設定されたインデックスファイル（DirectoryIndex）への内部転送を行う。

シグネチャまたは形状

static int fixup_dir(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**:
  • OK: インデックスファイルへの内部リダイレクトが成功した場合。
  • HTTP_MOVED_PERMANENTLY: スラッシュ補完のためのリダイレクトが必要な場合。
  • DECLINED: 処理対象外（path_info がある場合や、他のモジュールがハンドラを占有している場合など）。
  • HTTP_* (エラーコード): サブリクエストでエラーが発生した場合。

前提条件と不変条件

  • path_info が存在する場合、本関数は即座に DECLINED を返す。
  • DirectorySlash が無効な場合、末尾スラッシュなしのディレクトリリクエストに対しては DECLINED を返す。
  • インデックス解決の際、サブリクエストの結果が APR_REG（通常ファイル）であるか、プロキシハンドラが指定されていることを必須条件とする。

アルゴリズムまたは振る舞い

1. **初期化**: リクエストハンドラが未設定の場合、DIR_MAGIC_TYPE を割り当てる。 2. **スラッシュ補完**: URI が / で終わらない場合、DirectorySlash 設定に基づき、末尾に / を付加した URL へ 301 Moved Permanently を返す。 3. **ハンドラ競合回避**: REWRITE_REDIRECT_HANDLER_NAME や checkhandler が有効な場合、処理を中断して DECLINED を返す。 4. **インデックス探索**: DirectoryIndex で指定されたファイルリストを順次走査する。

  • ap_sub_req_lookup_uri を使用して各候補の存在を確認。
  • 候補が見つかった場合、DirectoryIndexRedirect 設定に応じて外部リダイレクトまたは ap_internal_fast_redirect による内部転送を行う。

    • 5. **エラー伝播**: サブリクエストが 404 以外のエラーを返した場合、そのエラーを保持し、全候補探索後に最終的なエラーとして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: dir_fixups（modules/mappers/mod_dir.c:395）
  • **呼び出し先**:
  • ap_sub_req_lookup_uri
  • ap_internal_fast_redirect
  • ap_destroy_sub_req
  • ap_construct_url
  • ap_escape_uri

関連データ構造

  • dir_config_rec: ディレクトリごとの設定（index_names, do_slash, redirect_index 等）を保持。
  • request_rec: Apache のリクエスト状態を管理。

失敗モードと境界条件

  • **サブリクエスト失敗**: 候補ファイルが存在しない場合、error_notfound を記録し、最終的に DECLINED または最後に発生したエラーコードを返す。
  • **リダイレクトループ**: FallbackResource 等の設定と組み合わせた際、指定先が存在しない場合に無限ループや予期せぬ挙動を招く可能性がある。
  • **非 GET リクエスト**: redirect-carefully 環境変数が設定されている場合、GET 以外のメソッドに対するスラッシュ補完リダイレクトを回避する。

保守担当者向けメモ

  • dir_config_rec は Apache の設定プール上で管理される。
  • サブリクエスト生成後は ap_destroy_sub_req を呼び出し、メモリリークを防ぐこと。
  • r->args はサブリクエストの生成時に適切に引き継ぐ必要がある。
