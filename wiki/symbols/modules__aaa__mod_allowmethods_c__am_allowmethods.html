am_allowmethods

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_allowmethods.c:106

目的

Apache 設定ファイル内の AllowMethods ディレクティブを解析し、指定された HTTP メソッドを許可リスト（ビットマスク）として設定構造体に格納します。

シグネチャまたは形状

static const char *am_allowmethods(cmd_parms *cmd, void *d, int argc, char *const argv[])

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • d: ディレクトリ設定構造体 (am_conf_t *)。
  • argc: 引数の数。
  • argv: メソッド名の配列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • d は am_conf_t 型としてキャスト可能であること。
  • argc が 0 の場合、エラーメッセージが返される。

アルゴリズムまたは振る舞い

1. 引数が reset の場合、conf->allowed を 0 にリセットし、conf->allowed_set を 1 に設定して終了する。 2. 引数がメソッド名の場合、ap_method_number_of を使用してメソッド番号を取得する。 3. 無効なメソッド名（M_INVALID）が指定された場合、エラーメッセージを返す。 4. 有効なメソッドであれば、AP_METHOD_BIT を用いて conf->allowed ビットマスクを更新する。 5. 最後に conf->allowed_set を 1 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_allowmethods.c:144 において AP_INIT_TAKE_ARGV マクロ経由で登録されている。
  • **呼び出し先**: ap_method_number_of（Apache API）、apr_pstrcat（APR API）。

関連データ構造

  • am_conf_t: ディレクトリごとの設定を保持する構造体。allowed（ビットマスク）と allowed_set（フラグ）を管理する。

失敗モードと境界条件

  • argc == 0: 引数不足としてエラーを返す。
  • ap_method_number_of が M_INVALID を返す場合: 不正なメソッド名としてエラーメッセージを生成し、終了する。

保守担当者向けメモ

  • 本関数は AllowMethods ディレクティブのハンドラとして機能します。
  • reset キーワードは、継承された設定を無効化し、許可リストをクリアするために使用されます。
