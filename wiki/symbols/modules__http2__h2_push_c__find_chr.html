find_chr

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_push.c:141

目的

指定された解析コンテキスト（link_ctx）の現在位置から、特定の文字 c を検索する。

シグネチャまたは形状

static int find_chr(link_ctx *ctx, char c, size_t *pidx)

入力と出力

  • **入力**:
  • ctx: 解析対象の文字列と現在のオフセットを保持する link_ctx 構造体へのポインタ。
  • c: 検索対象の文字。
  • pidx: 見つかった文字のインデックスを格納するための出力用ポインタ。
  • **出力**:
  • 戻り値: 文字が見つかった場合は 1、見つからなかった場合は 0 を返す。
  • pidx: 文字が見つかった場合、そのインデックスが書き込まれる。

前提条件と不変条件

  • ctx は有効な link_ctx 構造体を指している必要がある。
  • pidx は書き込み可能なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. ctx->i（現在のオフセット）から ctx->slen（文字列長）までループを回す。 2. 文字列 ctx->s の各要素を順次確認し、引数 c と一致する文字を探す。 3. 一致する文字が見つかった場合、そのインデックスを *pidx に代入し、直ちに 1 を返して終了する。 4. 文字列の末尾まで検索しても見つからない場合、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_util.c:845, 874
  • modules/http2/h2_push.c:174, 204
  • modules/md/md_util.c:1354, 1383
  • **呼び出し先**:
  • なし（標準的な比較演算のみを実行）。

関連データ構造

  • link_ctx: 文字列データとその長さを保持する構造体。

失敗モードと境界条件

  • ctx->i が ctx->slen 以上の場合、ループは実行されず 0 を返す。
  • 文字列内に目的の文字 c が存在しない場合、0 を返す。

保守担当者向けメモ

  • 本関数は単純な線形探索を行うユーティリティであり、Link ヘッダー解析の各所でクォートや区切り文字を探すために利用されている。
  • 検索範囲は ctx->i から開始されるため、呼び出し側で ctx->i が適切に更新されていることを前提としている。
