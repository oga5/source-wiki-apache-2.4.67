get_mutex_filename

定義

  • 種別: function
  • 定義箇所: server/util_mutex.c:292

目的

ミューテックスのロックメカニズムに基づき、ファイルベースのロックが必要な場合に、サーバーのルートディレクトリを基準とした絶対パスを生成する。

シグネチャまたは形状

static const char *get_mutex_filename(apr_pool_t *p, mutex_cfg_t *mxcfg,
                                      const char *type,
                                      const char *instance_id)

入力と出力

  • **入力**:
  • p: メモリ割り当て用のプール。
  • mxcfg: ミューテックスの設定情報（ディレクトリパス、メカニズム、PID省略フラグを含む）。
  • type: ミューテックスの種類を示す文字列。
  • instance_id: インスタンスを識別するID（任意）。
  • **出力**:
  • ファイルパスを示す文字列（const char *）。ファイルベースのロックが不要な場合は NULL を返す。

前提条件と不変条件

  • mxcfg は有効なポインタであること。
  • mutex_needs_file() が false を返すメカニズム（例: pthread 等）の場合、本関数はパスを生成せず NULL を返す。

アルゴリズムまたは振る舞い

1. mutex_needs_file(mxcfg->mech) を呼び出し、ファイルベースのロックが必要か判定する。不要であれば NULL を返す。 2. mxcfg->omit_pid が偽であれば、現在のプロセスID (getpid()) を含むサフィックス .PID を生成する。 3. mxcfg->dir、type、instance_id（存在する場合）、およびPIDサフィックスを結合し、ap_server_root_relative() を用いてサーバーのルートディレクトリを基準とした絶対パスに変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_mutex.c:433
  • server/util_mutex.c:480
  • **呼び出し先**:
  • mutex_needs_file
  • apr_psprintf
  • ap_server_root_relative
  • apr_pstrcat
  • getpid (システムコール)

関連データ構造

  • mutex_cfg_t: ミューテックスの設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pstrcat 等の APR 関数によりアボートまたはエラーが発生する可能性がある。
  • mxcfg->dir が未設定の場合、パス生成結果は不正なものとなる。

保守担当者向けメモ

  • 本関数は flock や fcntl 等のファイルベースのロックメカニズムを使用する際にのみ呼び出される。
  • omit_pid が指定されていない場合、PIDを付与することでプロセス間でのファイル名競合を回避する設計となっている。
