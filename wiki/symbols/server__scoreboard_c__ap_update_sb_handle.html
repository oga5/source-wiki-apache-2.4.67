ap_update_sb_handle

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:425

目的

スコアボードハンドル（ap_sb_handle_t）が指し示すワーカーの識別情報（プロセス番号およびスレッド番号）を更新します。

シグネチャまたは形状

void ap_update_sb_handle(ap_sb_handle_t *sbh, int child_num, int thread_num)

入力と出力

  • **入力**:
  • sbh: 更新対象の ap_sb_handle_t 構造体へのポインタ。
  • child_num: 設定する子プロセス番号。
  • thread_num: 設定するスレッド番号。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • sbh は有効なメモリ領域を指している必要があります。
  • 本関数は、スコアボードの各スロットが単一の所有者によって書き込まれるという設計上の不変条件に基づき、排他制御なしで実行されます。

アルゴリズムまたは振る舞い

1. 引数 sbh が指す構造体の child_num メンバに child_num を代入します。 2. 引数 sbh が指す構造体の thread_num メンバに thread_num を代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c:1072
  • server/scoreboard.c:436
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行いません。

関連データ構造

  • ap_sb_handle_t: スコアボード上の特定のワーカーを特定するためのハンドル構造体。

失敗モードと境界条件

  • sbh が NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は、スコアボードのインデックス情報を更新する軽量なユーティリティです。
  • include/ap_mmn.h によると、本関数は 2.4.30-dev (20120211.69) にて追加されました。
