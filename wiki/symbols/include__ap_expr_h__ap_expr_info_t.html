ap_expr_info_t

定義

  • 種別: struct
  • 定義箇所: include/ap_expr.h:41

目的

Apache HTTP Server の式評価エンジンにおいて、解析済みの式ツリーと、その式に関連するメタデータ（定義場所や動作フラグ）を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    ap_expr_t *root_node;
    const char *filename;
    unsigned int line_number;
    unsigned int flags;
    int module_index;
} ap_expr_info_t;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • **入力**: ap_expr_parse 等の解析関数によって各フィールドが初期化されます。
  • **出力**: ap_expr_exec や ap_expr_exec_re などの評価関数に引数として渡され、式の評価ロジックの制御に使用されます。

前提条件と不変条件

  • root_node は有効な式ツリーのルートである必要があります。
  • flags には AP_EXPR_FLAG_* で定義される定数が設定されます。
  • filename は NULL を許容します。

アルゴリズムまたは振る舞い

式評価エンジンにおいて、以下の役割を果たします。 1. **メタデータ管理**: 式が定義されたソースファイル名と行番号を保持し、エラーログ出力時に利用されます。 2. **評価制御**: flags フィールドにより、評価時の制約（RESTRICTED モードなど）や結果の型を制御します。 3. **ロギング**: module_index を用いて、評価に関連するログ出力のレベル設定を特定のモジュールに紐付けます。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_expr_parse_cmd_mi 等の解析関数が本構造体を生成します。また、http_core.h 内の condition フィールドなどで保持されます。
  • **呼び出し先**: ap_expr_exec および ap_expr_exec_re が、本構造体を引数として受け取り、内部の root_node を評価します。

関連データ構造

  • ap_expr_t: 式ツリーのノード定義。
  • ap_expr_eval_ctx_t: 評価実行時のコンテキスト（リクエスト情報やプールなど）。

失敗モードと境界条件

  • root_node が初期化されていない状態で評価関数に渡された場合、未定義の動作を引き起こす可能性があります。
  • module_index が不正な値である場合、ログ出力時に適切なモジュール設定が適用されない可能性があります。

保守担当者向けメモ

  • 本構造体は include/ap_expr.h で定義されており、式評価エンジンの公開 API の一部です。
  • 構造体のフィールドを追加・変更する場合は、既存の ap_expr_exec 系の関数シグネチャおよび ap_expr_parse_cmd_mi との整合性に注意してください。
