logio_dirconf_t

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_logio.c:53

目的

ディレクトリごとの設定情報を保持し、特に TTFB (Time To First Byte) 追跡機能の有効・無効を管理するために使用されます。

シグネチャまたは形状

typedef struct logio_dirconf_t {
    unsigned int track_ttfb:1;
} logio_dirconf_t;

入力と出力

  • **入力**: ap_get_module_config を通じてリクエストのディレクトリ設定 (r->per_dir_config) から取得されます。
  • **出力**: track_ttfb ビットフィールドにより、TTFB 追跡の実行可否を制御します。

前提条件と不変条件

  • track_ttfb は 1 ビットのフラグとして機能します。
  • Apache のディレクトリ設定コンテキスト内で管理されることを前提としています。

アルゴリズムまたは振る舞い

  • mod_logio はこの構造体を参照し、LogIOTrackTTFB ディレクティブの設定に基づいて TTFB フィルタを挿入するかどうかを判断します。
  • リクエスト処理中に ap_get_module_config を介して設定値が読み取られ、条件分岐の判定材料となります。

呼び出し元と呼び出し先

  • **呼び出し元 (参照箇所)**:
  • modules/loggers/mod_logio.c:209
  • modules/loggers/mod_logio.c:226
  • modules/loggers/mod_logio.c:235
  • **呼び出し先**:
  • apr_pcalloc (メモリ確保時)
  • 呼び出し元・呼び出し先の詳細な関数名については、提供されたソース情報からは明示されていません。

関連データ構造

  • logio_req_t: リクエストごとの TTFB 測定値を保持する構造体。
  • logio_config_t: 接続ごとの入出力統計を保持する構造体。

失敗モードと境界条件

  • メモリ確保失敗時: apr_pcalloc が失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性があります。
  • 設定未定義時: ap_get_module_config が NULL を返す可能性があるため、実装側でのポインタチェックが必要です。

保守担当者向けメモ

  • 本構造体は mod_logio のディレクトリ設定を保持する唯一の構造体です。
  • track_ttfb フラグの追加や変更を行う際は、mod_logio 内のフィルタ挿入ロジック (logio_insert_filter 等) との整合性に注意してください。
