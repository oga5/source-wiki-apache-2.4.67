attrib_info

定義

  • 種別: struct
  • 定義箇所: modules/http/mod_mime.c:54

目的

ファイル拡張子に関連付けられた属性名と、その属性が格納されている構造体内のオフセットを保持するために使用されます。

シグネチャまたは形状

typedef struct attrib_info {
    char *name;
    int   offset;
} attrib_info;

入力と出力

  • 入力: 構造体初期化時に name（属性名文字列）と offset（構造体内のメンバ位置）を受け取ります。
  • 出力: 該当する属性のメタデータへのアクセスを提供します。

前提条件と不変条件

  • offset は、対象となる構造体（extension_info 等）内での有効なメンバオフセットである必要があります。

アルゴリズムまたは振る舞い

主に mod_mime 内で、動的に拡張子ごとの設定を管理・操作する際に、特定の属性を識別するためのメタデータとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/http/mod_mime.c:177: remove->elts からのキャストによる取得。
  • modules/http/mod_mime.c:295: 変数宣言。
  • modules/http/mod_mime.c:302: apr_array_push を用いた配列への追加。
  • 呼び出し先: 情報なし。

関連データ構造

  • extension_info: この構造体で定義されたオフセットを使用して、具体的なメタデータ値が格納されます。
  • apr_array_header_t: attrib_info のインスタンスを保持する配列として利用されます。

失敗モードと境界条件

  • offset が不正な場合、メモリ上の誤った位置へのアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は、拡張子に関連する設定項目を汎用的に扱うためのメタデータ定義です。設定の追加や変更を行う際は、offset の整合性に注意してください。
