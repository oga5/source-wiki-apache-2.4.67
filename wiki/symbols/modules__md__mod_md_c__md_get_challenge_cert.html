md_get_challenge_cert

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md.c:1299

目的

TLS-ALPN-01 チャレンジに使用する証明書と秘密鍵を、指定されたサーバー名と鍵タイプに基づいて md_store から取得します。

シグネチャまたは形状

static int md_get_challenge_cert(conn_rec *c, const char *servername,
                                 md_srv_conf_t *sc,
                                 md_pkey_type_t key_type,
                                 const char **pcert_pem,
                                 const char **pkey_pem)

入力と出力

  • **入力**:
  • c: 現在の接続情報 (conn_rec)。メモリプールとして使用。
  • servername: 対象のサーバー名。
  • sc: サーバー設定構造体 (md_srv_conf_t)。
  • key_type: 検索対象の鍵タイプ (md_pkey_type_t)。
  • **出力**:
  • pcert_pem: 成功時に証明書の PEM 文字列が格納されるポインタ。
  • pkey_pem: 成功時に秘密鍵の PEM 文字列が格納されるポインタ。
  • 戻り値: 成功時は OK、失敗または該当なしの場合は DECLINED。

前提条件と不変条件

  • sc->mc->reg が有効なレジストリを保持していること。
  • c->pool がリクエストの生存期間中有効であること。

アルゴリズムまたは振る舞い

1. サーバー設定 sc に含まれる鍵仕様 (md_pkeys_spec_count) を反復処理します。 2. 指定された key_type と一致する鍵仕様を特定します。 3. tls_alpn01_fnames を使用して、該当する証明書と鍵のファイル名を生成します。 4. md_store_load を呼び出し、MD_SG_CHALLENGES ストアから証明書と鍵を順次読み込みます。 5. 読み込みに成功した場合、pcert_pem と pkey_pem に値をセットし OK を返します。 6. いずれかの読み込みで ENOENT 以外のエラーが発生した場合は、処理を中断し DECLINED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md.c:1377
  • modules/md/mod_md.c:1380
  • modules/md/mod_md.c:1383
  • **呼び出し先**:
  • md_reg_store_get
  • md_pkeys_spec_count
  • md_pkeys_spec_get
  • tls_alpn01_fnames
  • md_store_load
  • ap_log_cerror

関連データ構造

  • md_srv_conf_t: サーバー単位の MD 設定。
  • md_store_t: 証明書およびチャレンジデータの永続化ストア。
  • md_pkey_spec_t: 鍵の仕様定義。

失敗モードと境界条件

  • 指定された key_type に一致する鍵仕様が存在しない場合、ループを抜けて DECLINED を返します。
  • ストア内に該当ファイルが存在しない場合 (APR_ENOENT)、次の鍵仕様の検索へ進みます。
  • ストアからの読み込みでシステムエラーが発生した場合、cleanup ラベルへジャンプし DECLINED を返します。

保守担当者向けメモ

  • 本関数は mod_ssl の証明書コールバック等から呼び出されることを想定しています。
  • c->pool を使用してメモリを割り当てているため、接続終了時にメモリは解放されます。
