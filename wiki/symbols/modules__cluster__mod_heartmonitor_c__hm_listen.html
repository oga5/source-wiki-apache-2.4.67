hm_listen

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartmonitor.c:76

目的

ハートビート信号を受信するための UDP マルチキャストソケットを作成し、設定およびバインドを行う。

シグネチャまたは形状

apr_status_t hm_listen(hm_ctx_t *ctx)

入力と出力

  • **入力**: hm_ctx_t *ctx (モジュールの設定と実行時状態を保持する構造体。mcast_addr および p (プール) を参照)
  • **出力**: apr_status_t (成功時は APR_SUCCESS、失敗時はエラーコード)

前提条件と不変条件

  • ctx->mcast_addr が有効なマルチキャストアドレス情報で初期化されていること。
  • ctx->p がソケット作成に使用可能な有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. apr_socket_create を呼び出し、UDP ソケットを作成する。 2. apr_socket_opt_set を使用して以下のソケットオプションを有効化する。

  • APR_SO_REUSEADDR: アドレスの再利用を許可。
  • APR_SO_NONBLOCK: 非ブロッキングモードに設定。

    • 3. apr_socket_bind を呼び出し、ソケットを ctx->mcast_addr にバインドする。 4. apr_mcast_join を呼び出し、指定されたマルチキャストグループに参加する。 5. apr_mcast_loopback を呼び出し、ローカルホストからのマルチキャスト受信を許可する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cluster/mod_heartmonitor.c:611
  • **呼び出し先**: APR (Apache Portable Runtime) のソケットおよびマルチキャスト関連関数 (apr_socket_create, apr_socket_opt_set, apr_socket_bind, apr_mcast_join, apr_mcast_loopback)。

関連データ構造

  • hm_ctx_t: モジュールのコンテキスト。ソケットハンドル (ctx->sock) やアドレス情報 (ctx->mcast_addr) を保持する。

失敗モードと境界条件

  • ソケット作成、オプション設定、バインド、マルチキャストグループ参加のいずれかのステップでエラーが発生した場合、ap_log_error でエラーログを出力し、直ちにそのエラーコードを返して終了する。

保守担当者向けメモ

  • 本関数はモジュールの初期化フェーズで呼び出される。
  • ソケットの非ブロッキング設定は、後の hm_watchdog_callback における apr_poll によるイベント駆動監視を前提としている。
