ap_queue_info_set_idle

定義

  • 種別: function
  • 定義箇所: server/mpm_fdqueue.c:105

目的

ワーカーがアイドル状態になったことを通知し、必要に応じて待機中のリスナー・スレッドを起床させる。また、処理済みプールを再利用リストへ返却する。

シグネチャまたは形状

apr_status_t ap_queue_info_set_idle(fd_queue_info_t *queue_info, apr_pool_t *pool_to_recycle);

入力と出力

  • queue_info: アイドル状態を管理する fd_queue_info_t 構造体へのポインタ。
  • pool_to_recycle: 再利用のためにキューへ戻す apr_pool_t（NULL も可）。
  • 戻り値: 成功時は APR_SUCCESS。失敗時は apr_thread_mutex_lock または apr_thread_cond_signal からの apr_status_t エラーコード。

前提条件と不変条件

  • queue_info は ap_queue_info_create により初期化済みであること。
  • idlers カウンタは zero_pt を基準としたオフセット値で管理されており、負の値は待機中のスレッドが存在することを意味する。

アルゴリズムまたは振る舞い

1. ap_queue_info_push_pool を呼び出し、pool_to_recycle を再利用リストに追加する。 2. apr_atomic_inc32 を使用して queue_info->idlers をアトミックにインクリメントする。 3. インクリメント後の値が zero_pt 未満（負の値）である場合、待機中のリスナーが存在すると判断する。 4. idlers_mutex をロックし、wait_for_idler 条件変数に対して apr_thread_cond_signal を発行してリスナーを起床させる。 5. ロックを解放し、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • ap_queue_info_push_pool
  • apr_atomic_inc32
  • apr_thread_mutex_lock / apr_thread_mutex_unlock
  • apr_thread_cond_signal

関連データ構造

  • fd_queue_info_t: アイドル状態管理構造体。
  • apr_pool_t: 再利用対象のメモリプール。

失敗モードと境界条件

  • apr_thread_mutex_lock または apr_thread_cond_signal が失敗した場合、そのエラーコードを即座に返す。
  • pool_to_recycle が NULL の場合、ap_queue_info_push_pool は内部的に適切に処理する。

保守担当者向けメモ

  • 本関数はアトミック操作とミューテックスを併用している。idlers カウンタの操作はロックフリーだが、条件変数のシグナル送信には idlers_mutex による保護が必須である。
  • zero_pt は mpm_fdqueue.c 内で定義されたマジックナンバーであり、カウンタの解釈において重要な役割を果たす。
