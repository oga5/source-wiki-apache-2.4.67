skip_ws

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_push.c:131

目的

解析対象の文字列バッファにおいて、現在の読み取り位置から連続する空白文字（スペースおよびタブ）をスキップし、次の非空白文字へポインタを進める。

シグネチャまたは形状

static int skip_ws(link_ctx *ctx)

入力と出力

  • **入力**: link_ctx *ctx（解析状態を保持するコンテキスト構造体）。ctx->s は対象文字列、ctx->i は現在の読み取り位置、ctx->slen は文字列の長さを指す。
  • **出力**: int 型。スキップ処理後に読み取り位置が文字列の終端に達していない場合は非ゼロ（真）、終端に達した場合はゼロ（偽）を返す。

前提条件と不変条件

  • ctx は有効なポインタであること。
  • ctx->i は常に 0 以上 ctx->slen 以下の範囲内にあること。

アルゴリズムまたは振る舞い

1. ctx->i が ctx->slen 未満であることを確認する。 2. ctx->s[ctx->i] がスペース ' ' またはタブ '\t' である間、ctx->i をインクリメントする。 3. 処理終了後、ctx->i < ctx->slen の評価結果を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_util.c (複数箇所)
  • modules/http2/h2_push.c (複数箇所)
  • **呼び出し先**:
  • 本関数は他の関数を呼び出さない。

関連データ構造

  • link_ctx: 解析対象の文字列と現在のオフセットを保持する構造体。

失敗モードと境界条件

  • **文字列の終端**: 空白のみで文字列が終了した場合、ctx->i は ctx->slen と等しくなり、戻り値は 0 となる。
  • **空文字列**: ctx->slen が 0 の場合、ループに入らず直ちに 0 を返す。

保守担当者向けメモ

  • 本関数は Link ヘッダー解析におけるトークン間の空白除去を目的としている。
  • h2_proxy_util.c と h2_push.c の両方から参照されているため、変更時は両方のコンテキストへの影響を考慮すること。
