wake_idle_worker

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_workers.c:181

目的

HTTP/2 ワーカープールにおいて、アイドル状態のスレッドを検索してタスクを割り当てるか、必要に応じて新規スレッドを生成して処理を継続させる。

シグネチャまたは形状

static void wake_idle_worker(h2_workers *workers, ap_conn_producer_t *prod)

入力と出力

  • **入力**:
  • workers: ワーカープール全体を管理する h2_workers 構造体へのポインタ。
  • prod: 処理対象となる接続プロデューサー（ap_conn_producer_t）。
  • **出力**: なし（void）。

前提条件と不変条件

  • workers->lock による排他制御下で呼び出されることが想定されている（リング操作およびスロット状態変更のため）。
  • workers および prod は有効なポインタであること。

アルゴリズムまたは振る舞い

1. **アイドルワーカーの検索**: workers->idle リングを走査し、is_idle が真かつ should_shutdown が偽である最初の h2_slot を探す。 2. **ウェイクアップ**: 対象スロットが見つかった場合、その条件変数 more_work に対して apr_thread_cond_signal を発行し、is_idle を 0 にリセットして終了する。 3. **動的スレッド生成**: アイドルワーカーが見つからず、かつ workers->dynamic が有効で、現在の active_slots が max_slots 未満である場合、activate_slot(workers) を呼び出して新規スレッドを生成する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_workers.c:220
  • modules/http2/h2_workers.c:624
  • **呼び出し先**:
  • apr_thread_cond_signal (APR)
  • activate_slot (同ファイル内)

関連データ構造

  • h2_workers: プール管理構造体。
  • h2_slot: 個別のワーカー状態（is_idle, more_work 等を保持）。
  • ap_conn_producer_t: 接続供給元。

失敗モードと境界条件

  • アイドルワーカーが存在せず、かつ active_slots が max_slots に達している場合、新規スレッドは生成されず、本関数は何も行わずに終了する。
  • workers->shutdown が真の場合、新規スレッドの生成は抑制される。

保守担当者向けメモ

  • 本関数はスレッドのウェイクアップと生成の判断を行うクリティカルなパスである。
  • slot->is_idle のリセットは、呼び出し側がタスクをキューに投入した直後に行われる必要があるため、ロックの範囲に注意すること。
