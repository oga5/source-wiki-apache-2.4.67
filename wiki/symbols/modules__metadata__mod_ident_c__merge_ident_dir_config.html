merge_ident_dir_config

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_ident.c:271

目的

Apache HTTP Server のディレクトリ設定において、親ディレクトリの設定（old）と現在のディレクトリ設定（new）をマージし、新しい設定構造体を生成します。

シグネチャまたは形状

static void *merge_ident_dir_config(apr_pool_t *p, void *old_, void *new_)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • old_: 親ディレクトリの設定 (ident_config_rec*)。
  • new_: 現在のディレクトリの設定 (ident_config_rec*)。
  • **出力**:
  • マージ後の新しい ident_config_rec 構造体へのポインタ。

前提条件と不変条件

  • old_ および new_ は ident_config_rec 型としてキャスト可能であること。
  • apr_pcalloc により、戻り値の構造体はゼロ初期化された状態で割り当てられます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい ident_config_rec 構造体をプール p 上に確保します。 2. timeout フィールドについて、new 側の設定が未設定 (timeout_unset) であれば old の値を採用し、そうでなければ new の値を採用します。 3. do_rfc1413 フィールドについて、new 側の設定が未設定 (RFC1413_UNSET) であれば old の値を採用し、そうでなければ new の値を採用します。 4. マージされた構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_ident.c:339 にて、モジュールのコマンドテーブル（ident_cmds）に関連付けられたディレクトリマージ関数として参照されています。
  • **呼び出し先**: apr_pcalloc (APR ライブラリ関数)。

関連データ構造

  • ident_config_rec: 本関数がマージ対象とする設定構造体。timeout および do_rfc1413 フラグを保持します。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが終了する可能性があります。
  • old_ または new_ が NULL の場合の挙動については、本ソースコード内では明示的なガードは行われていません。

保守担当者向けメモ

  • 本関数は Apache の設定マージの標準的なパターンに従っています。設定項目を追加した場合は、必ず本関数内で _unset フラグを確認し、親設定を継承するロジックを追加してください。
