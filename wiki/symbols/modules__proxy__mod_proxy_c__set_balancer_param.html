set_balancer_param

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:365

目的

Apache HTTP Server のプロキシロードバランサー設定（BalancerMember 等に関連するパラメータ）を解析し、proxy_balancer 構造体に適用する。

シグネチャまたは形状

static const char *set_balancer_param(proxy_server_conf *conf,
                                      apr_pool_t *p,
                                      proxy_balancer *balancer,
                                      const char *key,
                                      const char *val)

入力と出力

  • **入力**:
  • conf: サーバー単位のプロキシ設定構造体。
  • p: 設定適用に使用するメモリプール。
  • balancer: 更新対象の proxy_balancer 構造体。
  • key: 設定項目名（例: "timeout", "lbmethod"）。
  • val: 設定値。
  • **出力**: 成功時は NULL を返し、エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • key は大文字小文字を区別せずに比較される。
  • balancer->s（共有メモリ領域）への書き込みは、適切に同期されている必要がある。
  • failonstatus の解析には apr_array_make を使用し、HTTP ステータスコードの妥当性を ap_is_HTTP_VALID_RESPONSE で検証する。

アルゴリズムまたは振る舞い

1. key に基づき、対応するパラメータを分岐処理する。 2. stickysession: セッション名とパスを balancer->s->sticky および sticky_path にコピーする。 3. lbmethod: ap_lookup_provider を使用してロードバランシングメソッドを検索し、balancer->lbmethod に設定する。 4. timeout, maxattempts, growth: 数値変換を行い、範囲チェック後に構造体に反映する。 5. failonstatus: カンマ区切りの文字列を解析し、HTTP ステータスコードの配列を生成する。 6. その他（nofailover, scolonpathdelim 等）: "on"/"off" の文字列を判定し、フラグを更新する。 7. 未知の key が渡された場合はエラーメッセージを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy.c:2131
  • modules/proxy/mod_proxy.c:2803
  • modules/proxy/mod_proxy.c:2955
  • **呼び出し先**:
  • ap_lookup_provider (ロードバランサーメソッドの検索)
  • apr_psprintf, apr_pstrdup, apr_strtok, apr_array_make, apr_array_push (メモリプール操作・文字列処理)
  • atoi (数値変換)
  • ap_is_HTTP_VALID_RESPONSE (HTTP ステータスコード検証)

関連データ構造

  • proxy_balancer: ロードバランサーの定義と状態を保持する構造体。
  • proxy_server_conf: サーバー単位のプロキシ設定。
  • proxy_balancer_method: ロードバランシングアルゴリズムのプロバイダー定義。

失敗モードと境界条件

  • stickysession の値がバッファサイズを超える場合、エラーを返す。
  • lbmethod が存在しない、または名前が長すぎる場合にエラーを返す。
  • timeout が 1 秒未満、growth が 1〜100 の範囲外の場合にエラーを返す。
  • failonstatus に不正な HTTP コードが含まれる場合にエラーを返す。

保守担当者向けメモ

  • PROXY_STRNCPY マクロを使用して共有メモリへの安全なコピーを行っている。
  • balancer->s への直接アクセスは共有メモリ領域を指すため、設定変更時の排他制御が適切に行われているか注意すること。
  • 新しいパラメータを追加する場合は、strcasecmp の分岐リストに項目を追加し、必要に応じて _set フラグを立てること。
