use_staged_acct

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:48

目的

ステージング領域から ACME アカウント情報と秘密鍵を読み込み、指定された ACME セッション構造体に適用して検証を行う。

シグネチャまたは形状

static apr_status_t use_staged_acct(md_acme_t *acme, struct md_store_t *store,
                                    const md_t *md, apr_pool_t *p)

入力と出力

  • **入力**:
  • acme: ACME セッション状態を保持する構造体。
  • store: ストレージ抽象化ハンドル。
  • md: 管理対象ドメイン情報。
  • p: 操作に使用するメモリプール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。読み込みまたは検証失敗時はエラーコードを返す。

前提条件と不変条件

  • MD_SG_STAGING グループ内に、対象ドメインに関連付けられた有効なアカウント情報が存在している必要がある。
  • acme 構造体は呼び出し元で初期化されていること。

アルゴリズムまたは振る舞い

1. md_acme_acct_load を呼び出し、MD_SG_STAGING からアカウント情報 (acct) と秘密鍵 (pkey) を読み込む。 2. 読み込みに成功した場合:

  • acme->acct_id をクリアする。
  • 読み込んだ acct と pkey を acme 構造体にセットする。
  • md_acme_acct_validate を呼び出し、アカウントの有効性を検証する。

    • 3. 処理結果（apr_status_t）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_drive.c:92 (ACME 駆動プロセス内)
  • **呼び出し先**:
  • md_acme_acct_load (アカウント情報の読み込み)
  • md_acme_acct_validate (アカウントの検証)

関連データ構造

  • md_acme_t: ACME セッション状態。
  • md_acme_acct_t: ACME アカウント情報。
  • md_pkey_t: アカウントに関連付けられた秘密鍵。
  • md_store_t: ストレージインターフェース。

失敗モードと境界条件

  • ステージング領域にアカウント情報が存在しない場合、md_acme_acct_load がエラーを返し、関数はそのまま終了する。
  • アカウントの検証 (md_acme_acct_validate) が失敗した場合、そのエラーコードが返される。

保守担当者向けメモ

  • 本関数は、証明書更新プロセスにおいて「ステージングされたアカウント」を現在のセッションに反映させるためのヘルパーである。
  • MD_SG_STAGING を使用しているため、本番環境の永続化データには直接影響を与えない。
