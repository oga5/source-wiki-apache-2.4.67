dav_set_bufsize

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:122

目的

dav_buffer 構造体のバッファサイズを確保または再設定し、現在のデータ長（cur_len）を更新します。

シグネチャまたは形状

void dav_set_bufsize(apr_pool_t *p, dav_buffer *pbuf, apr_size_t size)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t。
  • pbuf: 操作対象の dav_buffer 構造体へのポインタ。
  • size: 設定する新しいデータ長。
  • 出力: なし（pbuf の内部状態が更新されます）。

前提条件と不変条件

  • pbuf は有効な dav_buffer 構造体である必要があります。
  • 既存のバッファ内容は保持されません。
  • pbuf->alloc_len が size + DAV_BUFFER_PAD 未満の場合、新しいメモリ領域が apr_palloc により確保されます。

アルゴリズムまたは振る舞い

1. 要求された size に DAV_BUFFER_PAD を加えた値が、現在の pbuf->alloc_len を超えているか確認します。 2. 超えている場合、新しい alloc_len を計算します（size + DAV_BUFFER_PAD と DAV_BUFFER_MINSIZE の大きい方を採用）。 3. apr_palloc を用いて新しいバッファ領域を確保し、pbuf->buf を更新します。 4. 最後に pbuf->cur_len を指定された size に設定します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • modules/dav/fs/dbm.c
  • modules/dav/fs/lock.c
  • modules/dav/fs/repos.c
  • modules/dav/main/util.c
  • 呼び出し先:
  • apr_palloc (APRライブラリ)

関連データ構造

  • dav_buffer: バッファのポインタ、現在のデータ長、割り当て済みサイズを保持する構造体。

失敗モードと境界条件

  • メモリ不足時: apr_palloc が失敗した場合、Apache HTTP Server の標準的なメモリ割り当てエラー処理に従い、プロセスが終了または例外が発生します。
  • size が 0 の場合: DAV_BUFFER_MINSIZE が確保される可能性があります。

保守担当者向けメモ

  • 本関数は既存のバッファ内容を破棄するため、データを保持したい場合は dav_buffer_append 等の利用を検討してください。
  • DAV_BUFFER_PAD および DAV_BUFFER_MINSIZE は mod_dav.h で定義されています。
