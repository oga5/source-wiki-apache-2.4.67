c1_input_consumed

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_mplx.c:107

目的

h2_bucket_beam を介して転送されたデータが消費されたことを通知し、対応するストリームの入力状態を更新するコールバック関数です。

シグネチャまたは形状

static void c1_input_consumed(void *ctx, h2_bucket_beam *beam, apr_off_t length)

入力と出力

  • ctx: ストリームコンテキスト（h2_stream へのポインタ）。
  • beam: データ転送に使用された h2_bucket_beam インスタンス。
  • length: 消費されたデータのバイト数。
  • 戻り値: なし。

前提条件と不変条件

  • ctx は有効なストリームオブジェクトを指している必要があります。
  • 本関数は h2_bucket_beam の消費通知メカニズムから呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. 受け取った ctx（ストリーム）と length を引数として h2_stream_in_consumed を呼び出します。 2. これにより、ストリーム側で入力データの消費に伴う状態遷移やフロー制御の更新が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_mplx.c:885 において、h2_beam_on_consumed のコールバックとして登録されています。
  • 呼び出し先: h2_stream_in_consumed（ストリームの入力消費処理）。

関連データ構造

  • h2_bucket_beam: スレッド間データ転送パイプライン。
  • h2_stream: HTTP/2 ストリームの内部状態を保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合、h2_stream_in_consumed 内部での未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は h2_mplx 内のストリーム管理と h2_bucket_beam によるデータ転送を繋ぐブリッジです。
  • h2_stream_in_consumed の実装変更が本関数の挙動に直接影響するため、ストリームのフロー制御ロジックを変更する際は注意してください。
