dav_hooks_repository

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:270

目的

WebDAV リポジトリに対する抽象化レイヤーを提供し、具体的なストレージ実装（ファイルシステム等）と mod_dav コアロジックを分離するためのフック関数テーブルを定義します。

シグネチャまたは形状

typedef struct dav_hooks_repository dav_hooks_repository; （※構造体の詳細なメンバ構成は、本ヘッダーファイル内の定義を参照してください）

入力と出力

  • 入力: WebDAV リクエストに関連付けられたリソースやパス情報。
  • 出力: 各フック関数は、操作結果として dav_error* 型のエラー情報を返します。

前提条件と不変条件

  • dav_resource 構造体の hooks メンバを通じて参照されます。
  • プロバイダーが特定の機能を実装しない場合、対応する関数ポインタは NULL に設定されることがあります。

アルゴリズムまたは振る舞い

リポジトリ固有の操作（リソースの取得、パスの変換、プロパティの読み書き等）を抽象化します。mod_dav はリクエスト処理中にこの構造体を経由して、基盤となるストレージに対する具体的な操作を実行します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/dav/main/util.c
  • modules/dav/main/util_lock.c
  • mod_dav コアの各メソッドハンドラ
  • 呼び出し先:
  • modules/dav/fs/repos.c （dav_hooks_repository_fs として実装を提供）

関連データ構造

  • dav_resource: リポジトリフックを保持する主要な構造体。
  • dav_provider: リポジトリフックを含む、WebDAV プロバイダーの全機能を束ねる構造体。

失敗モードと境界条件

  • フック関数が NULL の場合、呼び出し側で適切なエラーハンドリングが必要です。
  • 各フックは dav_error 構造体を返すことで、Apache のレスポンス生成プロセスにエラーを伝播させます。

保守担当者向けメモ

  • 本構造体は ABI の安定性に影響するため、メンバの追加や変更には注意が必要です。
  • mod_dav_fs 以外の新しいストレージバックエンドを追加する際は、この構造体を実装し、プロバイダーとして登録する必要があります。
