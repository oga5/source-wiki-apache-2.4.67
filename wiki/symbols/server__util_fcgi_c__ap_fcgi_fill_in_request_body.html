ap_fcgi_fill_in_request_body

定義

  • **種別**: 関数
  • **定義箇所**: server/util_fcgi.c:102

目的

FastCGI プロトコルの BEGIN_REQUEST レコードのボディ構造体（ap_fcgi_begin_request_body）を、指定されたロールおよびフラグ値で初期化します。

シグネチャまたは形状

void ap_fcgi_fill_in_request_body(ap_fcgi_begin_request_body *brb,
                                  int role,
                                  unsigned char flags)

入力と出力

  • **入力**:
  • brb: 初期化対象となる ap_fcgi_begin_request_body 構造体へのポインタ。
  • role: FastCGI ロール（例: AP_FCGI_RESPONDER）。
  • flags: 接続維持等の動作を制御するフラグ。
  • **出力**: なし（brb が指すメモリ領域が直接更新されます）。

前提条件と不変条件

  • brb は有効なメモリ領域を指している必要があります。
  • 構造体内の reserved フィールドは常に 0 で初期化されます。

アルゴリズムまたは振る舞い

1. role を上位バイト（roleB1）と下位バイト（roleB0）に分割して格納します。 2. flags を構造体の flags フィールドに代入します。 3. 構造体内の 5 バイトの reserved 配列をすべて 0 に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:278
  • modules/proxy/mod_proxy_fcgi.c:334
  • **呼び出し先**: 情報なし。

関連データ構造

  • ap_fcgi_begin_request_body: FastCGI の BEGIN_REQUEST レコードのボディを定義する構造体。

失敗モードと境界条件

  • 本関数はメモリ割り当てを行わず、入力ポインタの妥当性チェックも行いません。呼び出し元が提供するバッファが ap_fcgi_begin_request_body 構造体として十分なサイズであることを前提とします。

保守担当者向けメモ

  • 本関数はメモリの所有権を管理しません。呼び出し元が確保したスタックまたはヒープ上の領域を操作します。
  • FastCGI プロトコルの仕様変更に伴い reserved 領域の扱いが変わる可能性がある場合は、本関数の初期化ロジックを確認してください。
