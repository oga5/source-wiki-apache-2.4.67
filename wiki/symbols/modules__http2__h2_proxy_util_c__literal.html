literal

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.c:475

目的

文字列とその長さをペアとして保持し、特定のヘッダー名などを静的なリストとして定義・管理するために使用されます。

シグネチャまたは形状

typedef struct {
    const char *name;
    size_t len;
} literal;

入力と出力

  • 入力: name（文字列へのポインタ）、len（文字列の長さ）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • name はヌル終端文字列である必要があります。
  • len は name が指す文字列の有効な長さと一致している必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持に特化しており、特定のロジックは持ちません。主に IgnoredRequestHeaders や IgnoredProxyRespHds といった静的配列の要素として利用され、ヘッダーのフィルタリング処理において比較対象として参照されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • modules/http2/h2_proxy_util.c:483 (IgnoredRequestHeaders の定義)
  • modules/http2/h2_proxy_util.c:491 (IgnoredProxyRespHds の定義)
  • modules/http2/h2_proxy_util.c:495 (ignore_header 関数の引数)
  • modules/http2/h2_proxy_util.c:498 (ignore_header 関数内のローカル変数)
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • IgnoredRequestHeaders: プロキシ転送時に無視すべきリクエストヘッダーのリスト。
  • IgnoredProxyRespHds: プロキシ転送時に無視すべきレスポンスヘッダーのリスト。

失敗モードと境界条件

  • len が実際の文字列長と不一致である場合、比較処理（ignore_header 等）において誤判定やメモリ読み取り範囲外へのアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は modules/http2/h2_util.c にも同名の定義が存在します。修正を行う際は、双方のファイルで一貫性が保たれているか確認してください。
  • ヘッダーの追加・削除を行う際は、name の更新と併せて len の値が正しいか注意してください。
