xml2cfg

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_xml2enc.c:89

目的

mod_xml2enc モジュールにおける、ディレクトリごとの設定情報を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const char* default_charset;
    xmlCharEncoding default_encoding;
    apr_array_header_t* skipto;
} xml2cfg;

入力と出力

  • **入力**: モジュール設定の初期化時やマージ時に、Apache の設定ファイルから読み込まれた値が格納されます。
  • **出力**: フィルタ処理（xml2enc_ffunc 等）において、エンコーディング検出のフォールバック値や、スキップ対象要素のリストとして参照されます。

前提条件と不変条件

  • apr_pcalloc を使用してメモリが確保されることが想定されています。
  • skipto は apr_array_header_t 型であり、要素のスキップ条件を保持します。

アルゴリズムまたは振る舞い

  • モジュールの設定コンテキストとして機能し、リクエスト処理中に ap_get_module_config を介して取得されます。
  • 設定のマージ処理（base と add の結合）において、新しい xml2cfg インスタンスが生成されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_xml2enc.c 内の複数の関数（ap_get_module_config を使用して取得）。
  • 設定マージ関数（apr_pcalloc を使用してインスタンスを生成）。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • xml2ctx: リクエストごとの実行状態を保持する構造体。
  • xml2enc_module: 本設定構造体を利用するモジュールのメイン構造体。

失敗モードと境界条件

  • 設定が未定義の場合、cfg ポインタが NULL になる可能性があるため、コード内では適切に初期化またはチェックが行われています。
  • apr_pcalloc によるメモリ確保が失敗した場合の挙動は、Apache のメモリ管理ポリシーに依存します。

保守担当者向けメモ

  • skipto メンバーは、コンテンツの正規化において特定の要素までをスキップする際の制御に使用されます。
  • コンパイラの警告を抑制するために、初期化時に NULL を代入する慣習が見られます。
