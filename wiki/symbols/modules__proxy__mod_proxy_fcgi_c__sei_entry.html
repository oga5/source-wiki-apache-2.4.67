sei_entry

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:24

目的

環境変数の条件付き設定（SetEnvIf）を管理するためのデータ構造です。特定の条件式に基づいて環境変数を動的に定義または置換するために使用されます。

シグネチャまたは形状

typedef struct {
    ap_expr_info_t *cond;   /* 条件式 */
    ap_expr_info_t *subst;  /* 置換式 */
    const char *envname;    /* 環境変数名 */
} sei_entry;

入力と出力

  • 本構造体は、設定解析時に ap_expr_info_t 型の式情報を保持し、実行時にこれらの式を評価して環境変数の値を決定します。

前提条件と不変条件

  • cond および subst は、Apache の式評価エンジン（ap_expr）によってコンパイルされた有効な式情報である必要があります。
  • envname は、設定された環境変数のキー名として機能します。

アルゴリズムまたは振る舞い

  • 設定フェーズにおいて、ProxyFCGISetEnvIf 等のディレクティブにより生成されたエントリが配列として保持されます。
  • リクエスト処理時に、保持された sei_entry のリストを順次走査し、cond が真と評価された場合に subst を評価して envname に値を割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_setenvif.c 内の関数群（add_envvars 等）から、設定の追加や検索、評価のために参照されています。
  • **呼び出し先**: ap_expr API を呼び出し、条件式および置換式の評価を行います。

関連データ構造

  • ap_expr_info_t: Apache の式評価エンジンで使用される構造体。
  • apr_array_header_t: sei_entry のリストを保持するために使用される配列構造体。

失敗モードと境界条件

  • 式のコンパイルに失敗した場合、設定の読み込み時にエラーが発生します。
  • 実行時に式評価が失敗した場合、そのエントリは無視されるか、デフォルトの挙動に従います。

保守担当者向けメモ

  • 本構造体は mod_proxy_fcgi.c と mod_setenvif.c の両方で定義・参照されています。修正を行う際は、両モジュール間でのバイナリ互換性に注意してください。
  • mod_proxy_fcgi.c における定義は、mod_setenvif のロジックを再利用または模倣する目的で配置されています。
