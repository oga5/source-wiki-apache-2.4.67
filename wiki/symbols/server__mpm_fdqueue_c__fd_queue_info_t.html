fd_queue_info_t

定義

  • 種別: struct
  • 定義箇所: server/mpm_fdqueue.c:31

目的

Apache HTTP Server の MPM（event や worker）において、ワーカー・スレッドのアイドル状態管理、スレッドのブロッキング制御、およびメモリプール（recycled_pools）の再利用リストを管理するための構造体です。

シグネチャまたは形状

struct fd_queue_info_t {
    apr_uint32_t volatile idlers;
    apr_thread_mutex_t *idlers_mutex;
    apr_thread_cond_t *wait_for_idler;
    int terminated;
    int max_idlers;
    int max_recycled_pools;
    apr_uint32_t recycled_pools_count;
    struct recycled_pool *volatile recycled_pools;
};

入力と出力

  • **入力**: ap_queue_info_create を通じて初期化され、各 ap_queue_info_* 関数群の第一引数として渡されます。
  • **出力**: idlers カウンタの状態や、recycled_pools リストの操作結果を保持します。

前提条件と不変条件

  • idlers フィールドは apr_atomic 操作によって更新されます。
  • idlers の値は zero_pt を基準とし、以下のように解釈されます：
  • >= zero_pt: アイドル状態のワーカー・スレッド数。
  • < zero_pt: アイドル・ワーカーを待機しているスレッド数。
  • recycled_pools はアトミックな操作を介して管理されます。

アルゴリズムまたは振る舞い

  • **アイドル管理**: ワーカーがアイドル状態になった際、ap_queue_info_set_idle を通じて idlers を更新し、待機中のリスナーを wait_for_idler 条件変数で起床させます。
  • **メモリ再利用**: ap_queue_info_push_pool および ap_queue_info_pop_pool を使用し、recycled_pools リストを介してメモリプールの再利用を行い、割り当てコストを削減します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/worker/worker.c
  • server/mpm_fdqueue.c 内の各 ap_queue_info_* 関数
  • **呼び出し先**:
  • apr_thread_mutex_t (同期)
  • apr_thread_cond_t (同期)
  • recycled_pool (データ構造)

関連データ構造

  • recycled_pool: メモリプールの再利用リストを構成する要素。
  • apr_thread_mutex_t: idlers_mutex として使用。
  • apr_thread_cond_t: wait_for_idler として使用。

失敗モードと境界条件

  • terminated フラグが設定された場合、ワーカーは終了処理へ移行します。
  • idlers カウンタが負の値になることは、待機スレッドが存在することを意味し、正常な動作範囲内です。
  • recycled_pools_count が max_recycled_pools を超える場合、メモリ再利用の制限がかかる可能性があります。

保守担当者向けメモ

  • idlers の zero_pt はマジックナンバーとして機能するため、アトミック操作時の加減算ロジックを変更する際は注意が必要です。
  • 本構造体は APR_HAS_THREADS が有効な環境でのみ使用されます。
  • ap_queue_info_pop_pool 等の関数は呼び出し側のスレッドコンテキストに依存する制約があるため、実装の変更時には server/mpm_fdqueue.c 内の同期設計を確認してください。
