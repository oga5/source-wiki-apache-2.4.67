verify

定義

  • 種別: function
  • 定義箇所: support/htpasswd.c:260

目的

提供されたパスワードと既存のハッシュ値を照合し、認証の成否を判定します。

シグネチャまたは形状

static int verify(struct passwd_ctx *ctx, const char *hash)

入力と出力

  • **入力**:
  • ctx: 認証コンテキスト（passwd_ctx 構造体）。パスワードが未設定の場合は get_password を介して取得されます。
  • hash: 照合対象となる既存のハッシュ文字列。
  • **出力**:
  • 0: 認証成功。
  • ERR_PWMISMATCH: パスワード不一致。
  • ERR_GENERAL: その他のエラー（APR 内部エラー等）。

前提条件と不変条件

  • ctx は有効なポインタである必要があります。
  • ctx->pool が初期化されている必要があります。
  • ctx->passwd が NULL の場合、get_password(ctx) が呼び出され、パスワード入力が試行されます。

アルゴリズムまたは振る舞い

1. ctx->passwd が未設定の場合、get_password(ctx) を呼び出してユーザーからパスワードを取得します。 2. apr_password_validate を使用して、入力されたパスワードと引数の hash を照合します。 3. 照合結果が APR_SUCCESS であれば 0 を返します。 4. 照合結果が APR_STATUS_IS_EMISMATCH(rv)（不一致）であれば、ctx->errstr にエラーメッセージを設定し、ERR_PWMISMATCH を返します。 5. その他のエラーが発生した場合は、apr_psprintf を用いてエラー詳細を ctx->errstr に格納し、ERR_GENERAL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコードの文脈上、main 関数内での操作フローから呼び出されることが想定されますが、本ソース excerpt 内での直接的な呼び出し元は特定されていません。
  • **呼び出し先**:
  • get_password (条件付き)
  • apr_password_validate
  • apr_psprintf

関連データ構造

  • struct passwd_ctx: 認証状態、メモリプール、エラーメッセージを保持する構造体。

失敗モードと境界条件

  • **パスワード不一致**: APR_STATUS_IS_EMISMATCH が返された場合、認証失敗として扱われます。
  • **システムエラー**: apr_password_validate が不一致以外のエラーを返した場合、ERR_GENERAL が返され、ctx->errstr に詳細なエラー情報が設定されます。
  • **入力失敗**: get_password が失敗した場合、その戻り値がそのまま伝播されます。

保守担当者向けメモ

  • 本関数は support/htpasswd.c 内の認証ロジックの一部です。
  • ctx->errstr への書き込みは、呼び出し元がエラー表示を行う際に利用されます。
  • apr_password_validate の仕様変更や、新しいハッシュアルゴリズムの追加時には本関数の挙動を確認してください。
