is_in_memory_bucket

定義

  • 種別: function
  • 定義箇所: server/core_filters.c:605

目的

指定された apr_bucket がメモリ上にデータを持つバケット型であるかを判定します。

シグネチャまたは形状

static APR_INLINE int is_in_memory_bucket(apr_bucket *b)

入力と出力

  • **入力**: apr_bucket *b (判定対象のバケット)
  • **出力**: int (真であれば非ゼロ、偽であれば 0 を返す)

前提条件と不変条件

  • 入力される b は有効な apr_bucket 構造体である必要があります。
  • この関数はインライン展開を前提としています。

アルゴリズムまたは振る舞い

以下のいずれかのバケット型に該当する場合に真を返します。

  • HEAP (ヒープメモリ)
  • POOL (プールメモリ)
  • TRANSIENT (一時的メモリ)
  • IMMORTAL (不変メモリ)

これらはデータが既にメモリ上に存在することが保証されているバケット型です。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core_filters.c:765 にて、バケットのメモリ常駐を確認するために使用されています。
  • **呼び出し先**: APR_BUCKET_IS_* マクロ群（APR ライブラリ）を呼び出します。

関連データ構造

  • apr_bucket: APR が提供するデータバケット構造体。

失敗モードと境界条件

  • b が NULL の場合の挙動は、APR のマクロ定義に依存します。

保守担当者向けメモ

  • 本関数は、メモリ上に存在しないバケット（ファイルバケットなど）をフィルタリングする際の判定ロジックとして使用されます。新しいバケット型が追加された際、メモリ常駐型であれば本関数への追加を検討してください。
