connect_to_peer

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:80

目的

FastCGI バックエンドサーバーへの TCP ソケット接続を確立します。複数のアドレスが提供された場合、接続が成功するまで順次試行します。

シグネチャまたは形状

apr_status_t connect_to_peer(apr_socket_t **newsock,
                             request_rec *r,
                             apr_sockaddr_t *backend_addrs,
                             const char *backend_name,
                             apr_interval_time_t timeout)

入力と出力

  • **入力**:
  • newsock: 生成されたソケットを格納するポインタ。
  • r: 現在のリクエストレコード。
  • backend_addrs: 接続先アドレスのリスト（apr_sockaddr_t）。
  • backend_name: ログ出力用のバックエンド識別名。
  • timeout: 接続タイムアウト値。0 の場合は r->server->timeout を使用。
  • **出力**:
  • apr_status_t: 接続成功時は APR_SUCCESS、失敗時は最後に試行したエラーコードを返します。

前提条件と不変条件

  • backend_addrs が NULL の場合、初期値 APR_EINVAL が返されます。
  • 接続試行中は APR_TCP_NODELAY が有効化されます。

アルゴリズムまたは振る舞い

1. backend_addrs リストを走査し、各アドレスに対して以下の処理を繰り返します。 2. apr_socket_create を呼び出し、ストリームソケットを作成します。 3. apr_socket_opt_set で APR_TCP_NODELAY を有効化し、apr_socket_timeout_set でタイムアウトを設定します。 4. apr_socket_connect でバックエンドへの接続を試みます。 5. 接続に失敗した場合はソケットを閉じ、次のアドレスへ進みます。 6. 接続が成功した時点でループを終了し、成功ステータスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authnz_fcgi.c:729
  • **呼び出し先**:
  • apr_socket_create, apr_socket_opt_set, apr_socket_timeout_set, apr_socket_connect, apr_socket_close (APR ネットワーク関数)
  • ap_log_rerror (Apache ログ出力関数)

関連データ構造

  • apr_socket_t: ネットワーク通信用ソケット。
  • apr_sockaddr_t: 接続先アドレス情報。
  • request_rec: Apache リクエストコンテキスト。

失敗モードと境界条件

  • **接続失敗**: 全てのアドレスへの接続が失敗した場合、最後に試行したアドレスのエラーが返されます。
  • **ログ出力**: 複数のアドレスがある場合、最後のアドレス以外への接続失敗は APLOG_DEBUG レベルで記録され、最後のアドレスでの失敗は APLOG_ERR レベルで記録されます。
  • **ソケット生成失敗**: ソケット作成に失敗した場合は、そのアドレスをスキップして次へ進みます。

保守担当者向けメモ

  • 本関数は TCP 接続のみをサポートしており、AF_UNIX ソケットには対応していません。
  • 接続試行時のログレベル制御により、冗長なエラーログを抑制する設計となっています。
