cgid_init

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgid.c:1035

目的

mod_cgid モジュールの初期化を行い、CGI デーモンプロセスを起動する。

シグネチャまたは形状

int cgid_init(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *main_server)

入力と出力

  • **入力**:
  • p: 設定用メモリプール。
  • main_server: サーバー設定構造体。
  • **出力**:
  • int: 成功時は OK、失敗時はエラーコードを返す。

前提条件と不変条件

  • ap_hook_post_config フックにより呼び出される。
  • main_server->process->pool を使用して、デーモンプロセスの状態（apr_proc_t）をプロセス間で永続化する。

アルゴリズムまたは振る舞い

1. userdata_key ("cgid_init") を使用し、プロセスプールから既存のデーモンプロセス情報を取得する。 2. データが存在しない場合、apr_proc_t 構造体を初期化してプールに保存し、初期化を終了する。 3. AP_SQ_MS_CREATE_PRE_CONFIG 状態でない場合（設定読み込み後）、以下の処理を行う:

  • Unix ドメインソケットのパスを ap_runtime_dir_relative で解決する。
  • パス長が sockaddr_un.sun_path の制限を超える場合は切り詰めを行い、警告ログを出力する。
  • sockaddr_un 構造体を構築し、cgid_start を呼び出してデーモンプロセスを起動する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_config フック（modules/generators/mod_cgid.c:1798）。
  • **呼び出し先**: cgid_start（デーモンプロセスの起動）。

関連データ構造

  • apr_proc_t: デーモンプロセスの管理構造体。
  • sockaddr_un: Unix ドメインソケットのアドレス構造体。
  • server_rec: Apache サーバー設定構造体。

失敗モードと境界条件

  • ScriptSock のパスが sockaddr_un の制限を超える場合、パスが切り詰められる。
  • cgid_start が失敗した場合、その戻り値がそのまま呼び出し元に返される。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで二度呼び出される可能性があるため、userdata を用いて二重起動を防止している。
  • cgid_start への依存が強いため、デーモンプロセスのライフサイクル変更時は本関数と cgid_start の両方を確認すること。
