slotmem_num_slots

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:218

目的

指定されたスロットメモリインスタンスが保持するスロットの総数を取得する。

シグネチャまたは形状

unsigned int slotmem_num_slots(ap_slotmem_instance_t *slot)

入力と出力

  • **入力**: slot - 情報を取得する対象の ap_slotmem_instance_t 構造体へのポインタ。
  • **出力**: インスタンス内に定義されているスロットの総数（unsigned int）。

前提条件と不変条件

  • slot 引数は有効な ap_slotmem_instance_t 構造体を指している必要がある。
  • ap_slotmem_instance_t 構造体内の num メンバは、インスタンス作成時に確定し、以降変更されないものとして扱われる。

アルゴリズムまたは振る舞い

1. 引数として渡された slot 構造体の num メンバにアクセスする。 2. その値をそのまま戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/slotmem/mod_slotmem_plain.c:311 (プロバイダー構造体への登録)
  • modules/slotmem/mod_slotmem_shm.c:671, 692, 719 (ログ出力等の用途)
  • modules/slotmem/mod_slotmem_shm.c:740 (プロバイダー構造体への登録)
  • **呼び出し先**:
  • 本関数は他の関数を呼び出さない。

関連データ構造

  • ap_slotmem_instance_t: スロットメモリの管理単位。本関数はこの構造体の num メンバを参照する。

失敗モードと境界条件

  • slot が NULL の場合、セグメンテーション違反が発生する可能性がある（ソースコード上に明示的なチェックは存在しない）。

保守担当者向けメモ

  • 本関数は ap_slotmem_provider_t インターフェースの一部として実装されている。
  • mod_slotmem_shm.c からも参照されているが、これは mod_slotmem_shm.c が mod_slotmem_plain.c と同様のプロバイダーインターフェースを定義・利用しているためである。
