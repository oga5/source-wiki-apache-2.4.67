find_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme_acct.c:264

目的

ACME アカウント検索処理において、検索条件やコンテキスト情報を保持し、コールバック関数間で受け渡すためのデータ構造です。

シグネチャまたは形状

typedef struct {
    apr_pool_t *p;
    const md_t *md;
    const char *id;
} find_ctx;

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは以下の役割を持ちます。

  • p: メモリ管理に使用する apr_pool_t。
  • md: 検索対象となる管理対象ドメイン（md_t）へのポインタ。
  • id: 検索に使用する識別子（文字列）。

前提条件と不変条件

  • p は、このコンテキストを使用する処理の生存期間をカバーする有効なプールである必要があります。
  • md および id は、検索処理中に参照されるため、処理終了まで有効である必要があります。

アルゴリズムまたは振る舞い

主に md_store_iter などのイテレータ処理において、検索対象のドメイン情報や識別子をコールバック関数へ渡す「バトン」として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_acct.c 内の検索ロジック（md_acme_find_acct_for_md 等）でインスタンス化されます。
  • **呼び出し先**: 検索処理のコールバック関数内で、baton 引数としてキャストされ参照されます。
  • **注記**: modules/md/md_util.c にも同名の構造体定義が存在しますが、本ページでは md_acme_acct.c の定義に焦点を当てています。

関連データ構造

  • md_t: 管理対象ドメインの定義。
  • apr_pool_t: APR のメモリプール。

失敗モードと境界条件

  • p が無効な場合、または md や id がヌルポインタの状態で参照された場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • modules/md/md_util.c にも同名の構造体が存在するため、コードを修正する際は対象ファイルが md_acme_acct.c であることを確認してください。
  • 本構造体は単なるデータコンテナですが、メモリ管理は apr_pool_t に依存しているため、ライフサイクル管理には注意が必要です。
