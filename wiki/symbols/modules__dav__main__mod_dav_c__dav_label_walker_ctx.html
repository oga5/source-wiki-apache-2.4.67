dav_label_walker_ctx

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.c:4110

目的

バージョン管理されたリソースに対してラベル操作（追加、設定、削除）を行う際の、走査（walk）処理のコンテキストを保持するために使用されます。

シグネチャまたは形状

typedef struct dav_label_walker_ctx
{
    dav_walk_params w;
    request_rec *r;
    const char *label;
    int label_op;
    const dav_hooks_vsn *vsn_hooks;
} dav_label_walker_ctx;

入力と出力

  • **入力**: dav_walk_params（走査パラメータ）、request_rec（元のリクエスト）、label（操作対象のラベル名）、label_op（操作の種類）、dav_hooks_vsn（バージョン管理プロバイダーのフック）。
  • **出力**: なし（構造体として状態を保持）。

前提条件と不変条件

  • label_op は以下のいずれかの定数である必要があります：
  • DAV_LABEL_ADD (1)
  • DAV_LABEL_SET (2)
  • DAV_LABEL_REMOVE (3)

アルゴリズムまたは振る舞い

この構造体は、リソースのツリー走査中にラベル操作の状態を維持します。label_op に基づき、バージョン管理プロバイダーのフック（vsn_hooks）を介して、対象リソースに対するラベルの適用や変更を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c 内のラベル操作関連関数（4187行目付近で初期化を確認）。
  • **呼び出し先**: wres->walk_ctx を通じて走査結果と連携（4134行目付近で参照を確認）。

関連データ構造

  • dav_walk_params: 走査のパラメータを定義。
  • request_rec: Apache のリクエスト情報。
  • dav_hooks_vsn: バージョン管理操作のためのプロバイダーフック。

失敗モードと境界条件

  • label_op に未定義の値が設定された場合の動作は定義されていません。
  • vsn_hooks が適切に初期化されていない場合、ラベル操作時にヌルポインタ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dav.c 内のラベル操作ロジックに特化しています。
  • label_op の定数は構造体定義内にハードコードされているため、拡張時は整合性に注意してください。
