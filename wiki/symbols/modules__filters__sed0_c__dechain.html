dechain

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:179

目的

sed スクリプト内の分岐先ラベル（sed_label_t）に関連付けられた未解決のコマンドチェーンを、解決済みのコマンドアドレス（sed_reptr_t）へ接続します。

シグネチャまたは形状

static void dechain(sed_label_t *lpt, sed_reptr_t *address)

入力と出力

  • **入力**:
  • lpt: 未解決のコマンドチェーンを保持するラベル構造体へのポインタ。
  • address: 分岐先として解決されたコマンドノードへのポインタ。
  • **出力**: なし（副作用として lpt のチェーンを解決し、lpt->chain を NULL にリセットします）。

前提条件と不変条件

  • lpt が NULL の場合、または lpt->chain が NULL の場合、あるいは address が NULL の場合は何もしません。
  • 処理完了後、lpt->chain は必ず NULL になります。

アルゴリズムまたは振る舞い

1. lpt->chain から開始し、リンクされたコマンドリストを順次辿ります。 2. 各コマンドノードの lb1 フィールドを、引数で渡された address で更新します。 3. リストの末尾まで到達し、すべてのノードの接続を完了させた後、lpt->chain を NULL に初期化して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/sed0.c:351 にて呼び出しを確認しています。
  • **呼び出し先**: 本関数は直接他の関数を呼び出しません。

関連データ構造

  • sed_label_t: 分岐先ラベルを管理し、未解決の参照チェーンを保持します。
  • sed_reptr_t: sed コマンドの内部表現。lb1 フィールドが分岐先アドレスの保持に使用されます。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、関数は安全に早期リターンします。
  • チェーンが空（lpt->chain == NULL）の場合、何も変更を行わずに終了します。

保守担当者向けメモ

  • 本関数は sed コンパイルフェーズにおけるラベル解決の最終段階で使用されます。
  • lb1 フィールドは sed_reptr_t 構造体において、分岐先アドレスを格納する重要なポインタです。このポインタの書き換えが正しく行われない場合、sed スクリプトの実行時に不正なジャンプが発生する可能性があります。
