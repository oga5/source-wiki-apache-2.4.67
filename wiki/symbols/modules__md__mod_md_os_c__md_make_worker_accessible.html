md_make_worker_accessible

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_os.c:58

目的

指定されたファイルまたはディレクトリの所有権を、Apache HTTP Server の実行ユーザー（ap_unixd_config.user_id）に変更し、ワーカプロセスからアクセス可能にします。

シグネチャまたは形状

apr_status_t md_make_worker_accessible(const char *fname, apr_pool_t *p)

入力と出力

  • **入力**:
  • fname: アクセス権を調整する対象のファイルまたはディレクトリのパス。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS を返します。OS がサポートしていない場合や権限変更に失敗した場合は、対応するエラーコードを返します。

前提条件と不変条件

  • Unix 系環境では、プロセスが適切な権限（通常は root）で実行されている必要があります。
  • ap_unixd_config.user_id が正しく初期化されていることが前提です。

アルゴリズムまたは振る舞い

1. **プラットフォーム判定**: WIN32 マクロが定義されている場合、本機能はサポート対象外として APR_ENOTIMPL を返します。 2. **権限変更**: Unix 系環境では、md_try_chown を呼び出します。

  • md_try_chown は、現在の実行ユーザー ID を取得し、対象パスの所有者をそのユーザーに変更します。
  • 実行ユーザーが root 権限を持たない場合、md_try_chown は何もしない（成功を返す）設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md.c:252
  • **呼び出し先**:
  • md_try_chown (同ファイル内)

関連データ構造

  • ap_unixd_config: Apache の Unix 固有設定を保持する構造体。user_id を参照します。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • **Windows**: 常に APR_ENOTIMPL を返します。
  • **権限不足**: root 権限で実行されていない場合、md_try_chown の内部仕様により権限変更はスキップされます。
  • **ファイル不在**: md_try_chown が内部で呼び出すシステムコールが失敗した場合、そのエラーコードが返されます。

保守担当者向けメモ

  • 本関数は mod_md が生成した証明書や鍵ファイルを、ワーカプロセスが読み書きできるようにするための重要なセキュリティ・ユーティリティです。
  • md_try_chown の挙動（root 以外では何もしない）に依存しているため、非特権環境での動作確認時には、ファイル権限が期待通りであるか個別に検証してください。
