usage

定義

  • 種別: function
  • 定義箇所: support/httxt2dbm.c:54

目的

httxt2dbm コマンドの利用方法と、ビルド時にサポートされている DBM フォーマットの有効状態を標準エラー出力へ表示する。

シグネチャまたは形状

static void usage(void)

入力と出力

  • 入力: なし（内部で errfile およびビルド時のマクロ定数 APU_HAVE_* を参照）。
  • 出力: apr_file_printf を介して、コマンドの構文、オプション説明、および各 DBM フォーマット（GDBM, SDBM, DB, NDBM）の利用可否を標準エラー出力へ書き出す。

前提条件と不変条件

  • errfile が初期化済みであること。
  • ビルド環境において APU_HAVE_SDBM, APU_HAVE_GDBM, APU_HAVE_NDBM, APU_HAVE_DB マクロが適切に定義されていること。

アルゴリズムまたは振る舞い

1. ビルド時に有効化された DBM バックエンドを判定し、それぞれ AVAIL または UNAVAIL 文字列を割り当てる。 2. apr_file_printf を使用して、プログラム名、使用方法、オプション一覧、および各 DBM フォーマットのサポート状況をフォーマット済み文字列として出力する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では明示的な呼び出し箇所は特定されていません。
  • 呼び出し先: apr_file_printf（APR ライブラリ関数）。

関連データ構造

  • なし。

失敗モードと境界条件

  • apr_file_printf が失敗した場合の挙動は APR の仕様に依存する。
  • 実行環境の権限により標準エラー出力への書き込みが制限されている場合、出力は行われない。

保守担当者向けメモ

  • 本関数は httxt2dbm のヘルプメッセージを管理する。新しい DBM フォーマットがサポートされた場合は、判定ロジックと出力フォーマットの両方を更新する必要がある。
  • ツール検索結果に表示されている他の usage シンボル（mod_proxy_html.c 等）とは無関係である。
