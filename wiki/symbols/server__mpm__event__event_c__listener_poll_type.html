listener_poll_type

定義

  • 種別: struct
  • 定義箇所: server/mpm/event/event.c:365

目的

apr_pollset に登録された各ファイル記述子（FD）に関連付けられたメタデータを保持し、イベント発生時にその種別と対象オブジェクトを識別するために使用されます。

シグネチャまたは形状

typedef struct
{
    poll_type_e type;
    void *baton;
} listener_poll_type;

入力と出力

  • **入力**: apr_pcalloc 等によるメモリ割り当て時に初期化されます。
  • **出力**: apr_pollfd_t の client_data メンバに格納され、イベントループ内で取り出されます。

前提条件と不変条件

  • type メンバは poll_type_e 列挙型に基づき、イベントの発生源（リスナーソケットや接続状態など）を特定できる必要があります。
  • baton メンバは、type に応じた適切なデータ構造（接続状態など）を指すポインタである必要があります。

アルゴリズムまたは振る舞い

1. **登録**: apr_pollset_add を呼び出す際、この構造体を apr_pollfd_t の client_data に設定します。 2. **取得**: イベント発生後、apr_pollset_poll から返された apr_pollfd_t の client_data を本構造体にキャストして参照します。 3. **識別**: type を確認し、baton が指すオブジェクトに対して適切な処理（接続の受け入れやタイムアウト処理など）をディスパッチします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c:1016: 接続初期化時にインスタンスを生成。
  • server/mpm/event/event.c:1873: イベント発生時に client_data からキャストして取得。
  • server/mpm/event/event.c:2433: 参照用変数として宣言。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • poll_type_e: type メンバの型定義。
  • apr_pollfd_t: 本構造体が格納される client_data を持つ APR の構造体。

失敗モードと境界条件

  • baton が指すメモリ領域が、イベント発生時に既に解放されている場合（ダングリングポインタ）、未定義動作を引き起こします。
  • client_data が正しく設定されていない状態でキャストを行うと、不正なメモリ参照が発生します。

保守担当者向けメモ

  • 本構造体は apr_pollset を介したイベント駆動処理の要です。type を追加・変更する際は、必ず poll_type_e の定義と、それを利用する event.c 内のスイッチ文や条件分岐の整合性を確認してください。
