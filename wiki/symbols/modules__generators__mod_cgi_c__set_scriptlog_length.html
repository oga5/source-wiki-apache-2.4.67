set_scriptlog_length

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgi.c:142

目的

CGI スクリプトのデバッグログファイルの最大サイズ（バイト単位）を設定します。

シグネチャまたは形状

static const char *set_scriptlog_length(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: ログサイズの最大値を表す文字列（数値）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • cmd->server から取得したサーバー設定構造体 cgi_server_conf が有効である必要があります。
  • arg は atol() で変換可能な数値文字列である必要があります。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定から cgi_server_conf を取得します。 2. 引数 arg を atol() で数値に変換します。 3. 変換した値を conf->logbytes に代入し、ログの最大サイズを更新します。 4. NULL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:179 (設定ディレクティブ ScriptLogLength の登録時)
  • modules/generators/mod_cgid.c:1177 (設定ディレクティブ ScriptLogLength の登録時)
  • **呼び出し先**:
  • ap_get_module_config (Apache API)
  • atol (標準ライブラリ)

関連データ構造

  • cgi_server_conf: CGI モジュールのサーバー設定を保持する構造体。logbytes メンバを更新します。

失敗モードと境界条件

  • arg が数値として解釈できない場合、atol() の仕様に従い 0 が設定されます。
  • 設定構造体が取得できない場合の挙動については、本ソースコード内では明示的なエラーハンドリングは行われていません。

保守担当者向けメモ

  • 本関数は ScriptLogLength ディレクティブのハンドラです。
  • mod_cgi と mod_cgid の両方から参照されているため、変更時は両モジュールへの影響を考慮してください。
