set_signals

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:377

目的

プロセス終了、再起動、および割り込みシグナルに対するハンドラを登録し、OS/2 環境下でのプロセスライフサイクル管理を可能にする。

シグネチャまたは形状

static void set_signals(void)

入力と出力

  • 入力: なし（グローバルなシグナルハンドラ関数 sig_term, sig_restart を参照）。
  • 出力: なし（副作用としてプロセス全体のシグナル動作を更新）。

前提条件と不変条件

  • sig_term および sig_restart 関数が同一スコープまたはリンク可能な範囲で定義されていること。
  • sigaction システムコールが利用可能であること。

アルゴリズムまたは振る舞い

1. struct sigaction 構造体を初期化し、マスクを空に設定する。 2. SIGTERM および SIGINT に対して sig_term ハンドラを割り当てる。 3. SIGHUP および SIGUSR1 に対して sig_restart ハンドラを割り当てる。 4. 各 sigaction 呼び出しが失敗した場合、ap_log_error を使用して警告ログを出力する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/mpm/mpmt_os2/mpmt_os2.c:202
  • server/mpm/mpmt_os2/mpmt_os2_child.c:109
  • server/mpm/netware/mpm_netware.c:896（ツールによる推測）
  • 呼び出し先:
  • sig_term (ハンドラ関数)
  • sig_restart (ハンドラ関数)
  • sigaction (システムコール)
  • ap_log_error (エラーログ出力)

関連データ構造

  • struct sigaction: シグナル処理の詳細を定義する標準構造体。

失敗モードと境界条件

  • sigaction がエラーを返した場合、警告ログが出力されるが、プロセスの実行は継続される。この場合、該当するシグナルに対する期待された動作（終了や再起動）が実行されない可能性がある。

保守担当者向けメモ

  • 本関数は mpmt_os2.c と mpmt_os2_child.c の両方から呼び出されており、親プロセスと子プロセスの双方でシグナルハンドリングを統一している。
  • SIGTERM/SIGINT は終了処理、SIGHUP/SIGUSR1 は再起動処理として分類されている。
