proxy_conn_pool

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:115

目的

バックエンドサーバー（ワーカー）に対する接続プールを管理するための構造体です。

シグネチャまたは形状

struct proxy_conn_pool {
    /* 内部構造は modules/proxy/mod_proxy.h:305 以降で定義 */
};

入力と出力

  • 入力: proxy_worker に永続的に関連付けられます。
  • 出力: バックエンドへの接続（proxy_conn_rec）の取得・再利用・解放を仲介します。

前提条件と不変条件

  • proxy_conn_pool は、対応する proxy_worker に対して永続的に割り当てられます。
  • apr_pool_tag により、メモリプールには "proxy_conn_pool" というタグが付けられます。

アルゴリズムまたは振る舞い

  • proxy_worker のライフサイクルに同期して生成されます。
  • apr_pcalloc を用いてメモリが確保され、接続管理の基盤として機能します。
  • 接続の再利用を管理し、必要に応じて新規接続の生成や破棄を制御します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/proxy_util.c 内のワーカー初期化処理においてインスタンス化されます。
  • 呼び出し先: proxy_conn_rec を管理し、DNS プール（dns_pool）等のリソースを保持します。

関連データ構造

  • proxy_worker: 本構造体を保持する親構造体。
  • proxy_conn_rec: 本プールによって管理される個別の接続状態。
  • apr_pool_t: メモリ管理に使用される APR プール。

失敗モードと境界条件

  • メモリ確保失敗時: apr_pcalloc が失敗した場合、プロキシ接続の初期化が中断されます。
  • 接続枯渇時: プール内の接続数が上限に達した場合、新規接続の取得は待機またはエラーとなります。

保守担当者向けメモ

  • dns_pool が追加された経緯（20120211.89）があるため、バージョンアップ時のバイナリ互換性に注意してください。
  • 構造体の詳細定義は mod_proxy.h の 305 行目以降を参照してください。
