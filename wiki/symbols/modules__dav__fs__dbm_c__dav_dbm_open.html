dav_dbm_open

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/dbm.c:183

目的

WebDAV リソースに関連付けられた DBM データベースファイルを開き、プロパティ操作のための接続状態を初期化します。

シグネチャまたは形状

static dav_error * dav_dbm_open(apr_pool_t * p, const dav_resource *resource,
                                int ro, dav_db **pdb);

入力と出力

  • p: メモリ割り当てに使用するプール。
  • resource: 対象となる WebDAV リソース。
  • ro: 読み取り専用モードで開く場合は非ゼロ、書き込みを行う場合は 0。
  • pdb: 開かれたデータベース接続状態を格納するポインタの出力先。
  • 戻り値: 成功時は NULL、失敗時は dav_error 構造体へのポインタ。

前提条件と不変条件

  • resource は有効なリソースである必要があります。
  • ro が 0（書き込みモード）の場合、リソースに対応する状態ディレクトリ（.DAV）が存在している必要があります。

アルゴリズムまたは振る舞い

1. dav_fs_dir_file_name を呼び出し、リソースに対応するディレクトリパスとファイル名を特定します。 2. ro が 0 の場合、dav_fs_ensure_state_dir を呼び出し、状態ディレクトリの存在を保証します。 3. apr_pstrcat を使用して、リソースのパスに基づいた DBM ファイルのフルパスを構築します。ディレクトリの場合は DAV_FS_STATE_FILE_FOR_DIR を使用します。 4. dav_dbm_open_direct を呼び出し、構築したパスに対してデータベースを開き、結果を pdb に格納して返します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/fs/dbm.c:433
  • 呼び出し先: dav_fs_dir_file_name, dav_fs_ensure_state_dir, dav_dbm_open_direct

関連データ構造

  • dav_resource: WebDAV リソースの抽象化。
  • dav_db: DBM データベース接続状態を保持する構造体。
  • dav_error: エラー情報を保持する構造体。

失敗モードと境界条件

  • 状態ディレクトリの作成失敗や、DBM ファイルのオープン失敗時に dav_dbm_open_direct から返されるエラーが伝播されます。
  • 読み取り専用モードで開く際、書き込み中の他プロセスとの競合に対する再試行ロジックは実装されていません。
  • dav_fs_dir_file_name の戻り値に対するテストは行われていません。

保守担当者向けメモ

  • DBM ファイルのパス構築時に umask が考慮されていません。
  • 並行書き込み時の再試行ロジックの実装が検討事項として残されています。
