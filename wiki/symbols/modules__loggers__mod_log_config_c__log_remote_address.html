log_remote_address

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_config.c:323

目的

ログフォーマット文字列内の %a 指定子に対応し、リクエストに関連付けられたリモートIPアドレスを文字列として取得する。

シグネチャまたは形状

static const char *log_remote_address(request_rec *r, char *a)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体へのポインタ。
  • a: フォーマット指定子に続く引数文字列。"c" が指定された場合は接続元のIPアドレスを要求する。
  • **出力**:
  • リクエストまたは接続に関連するIPアドレス文字列を返す。

前提条件と不変条件

  • r は有効な request_rec 構造体を指している必要がある。
  • a が "c" の場合、r->connection->client_ip が参照される。それ以外の場合は r->useragent_ip が参照される。

アルゴリズムまたは振る舞い

1. 引数 a が存在し、かつ文字列 "c" と一致するかを確認する。 2. 一致する場合、r->connection->client_ip を返す。 3. 一致しない場合、r->useragent_ip を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/loggers/mod_log_config.c:1736: log_pfn_register を介してログフォーマット項目として登録。
  • server/log.c:946: ap_register_errorlog_handler を介してエラーログハンドラとして登録。
  • **呼び出し先**:
  • strcmp (標準ライブラリ関数)

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • conn_rec (r->connection経由): 接続情報を保持する構造体。

失敗モードと境界条件

  • r が NULL の場合、未定義の動作（セグメンテーションフォールト）を引き起こす可能性がある。
  • a が NULL の場合、strcmp は呼び出されず、デフォルトの r->useragent_ip が返される。

保守担当者向けメモ

  • 本関数はログフォーマットのハンドラとして登録されるため、シグネチャの変更は log_pfn_register および ap_register_errorlog_handler の呼び出し箇所と整合させる必要がある。
  • r->useragent_ip と r->connection->client_ip の使い分けは、プロキシ経由のアクセス等におけるIPアドレスの解釈に依存する。
