add_extension_info

定義

  • 種別: function
  • 定義箇所: modules/http/mod_mime.c:245

目的

Apache HTTP Server の設定ディレクティブ（AddType, AddCharset 等）を処理し、指定されたファイル拡張子に関連付けられたメタデータ（extension_info）をディレクトリ設定構造体に登録・更新します。

シグネチャまたは形状

static const char *add_extension_info(cmd_parms *cmd, void *m_, const char *value_, const char* ext)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。cmd->info には更新対象の extension_info 構造体内のオフセットが格納されている。
  • m_: 現在のディレクトリ設定構造体 (mime_dir_config)。
  • value_: 拡張子に割り当てる値（MIMEタイプ、エンコーディング名等）。
  • ext: 対象のファイル拡張子。
  • **出力**:
  • 成功時は NULL を返す。

前提条件と不変条件

  • cmd->info には、extension_info 構造体内の有効なメンバオフセットが設定されている必要がある。
  • 拡張子の先頭に . が含まれる場合は自動的に除去される。
  • value および key（拡張子）は、処理の過程で小文字に正規化される。

アルゴリズムまたは振る舞い

1. cmd->info から更新対象のメンバオフセットを特定する。 2. ext と value_ をそれぞれメモリプールに複製し、小文字に変換する。 3. m->extension_mappings ハッシュテーブルが存在しない場合は新規作成する。 4. 指定された拡張子に対応する extension_info 構造体をハッシュテーブルから取得する。 5. 該当する extension_info が存在しない場合は、新規にメモリを確保してハッシュテーブルに追加する。 6. 特定したオフセット位置に value を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/mod_mime.c 内の AP_INIT_ITERATE2 マクロを通じて、以下のディレクティブから呼び出される:
  • AddCharset, AddEncoding, AddHandler, AddInputFilter, AddLanguage, AddOutputFilter, AddType
  • 同ファイル内の remove_extension_type 等の内部関数から直接呼び出されるケースがある（行283）。
  • **呼び出し先**:
  • apr_pstrdup, apr_pcalloc, apr_hash_make, apr_hash_get, apr_hash_set (APR ライブラリ関数)
  • ap_str_tolower (Apache ユーティリティ関数)

関連データ構造

  • mime_dir_config: ディレクトリごとの設定を保持する構造体。
  • extension_info: 拡張子ごとのメタデータ（タイプ、言語、フィルタ等）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数が異常終了（abort）する可能性がある。
  • 拡張子が空文字列の場合の挙動は、呼び出し元のディレクティブ定義に依存する。

保守担当者向けメモ

  • 本関数は cmd->info を使用して汎用的なセッターとして機能しているため、extension_info 構造体に新しいフィールドを追加した際は、対応するディレクティブの AP_INIT 定義で正しいオフセットが渡されていることを確認すること。
  • 拡張子の比較および格納はすべて小文字で行われるため、大文字小文字を区別するファイルシステム環境であっても、設定値は小文字で統一される点に注意。
