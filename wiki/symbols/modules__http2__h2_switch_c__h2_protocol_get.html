h2_protocol_get

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_switch.c:217

目的

現在の接続に関連付けられた HTTP/2 プロトコル識別子（文字列）を取得します。

シグネチャまたは形状

const char *h2_protocol_get(const conn_rec *c)

入力と出力

  • **入力**: const conn_rec *c - 問い合わせ対象の接続レコード。
  • **出力**: const char * - 接続が HTTP/2 セッションとして確立されている場合はプロトコル識別子文字列、それ以外の場合は NULL を返します。

前提条件と不変条件

  • 入力された conn_rec が子接続（スレーブ）である場合、内部で自動的にマスター接続へ参照が切り替えられます。
  • 接続コンテキスト (h2_conn_ctx_t) が存在しない場合、安全に NULL を返します。

アルゴリズムまたは振る舞い

1. 引数 c が master 接続を持つ場合、c をそのマスター接続に更新します。 2. h2_conn_ctx_get(c) を呼び出し、接続に関連付けられた HTTP/2 コンテキストを取得します。 3. コンテキストが存在すれば、その protocol メンバを返します。存在しなければ NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_switch.c:232 にて ap_hook_protocol_get に登録されています。
  • **呼び出し先**:
  • h2_conn_ctx_get (接続コンテキストの取得)
  • c->master (接続レコードの参照)

関連データ構造

  • conn_rec: Apache HTTP Server の接続状態を保持する構造体。
  • h2_conn_ctx_t: HTTP/2 接続固有のメタデータを保持する構造体。

失敗モードと境界条件

  • 接続が HTTP/2 ではない、またはコンテキストが初期化されていない場合、NULL を返して正常に終了します。

保守担当者向けメモ

  • 本関数は Apache の protocol_get フックとして機能します。
  • フィルターチェーンの操作ではなく、接続状態のメタデータ照会のみを行うため、副作用はありません。
