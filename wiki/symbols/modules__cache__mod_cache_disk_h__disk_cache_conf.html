disk_cache_conf

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache_disk.h:72

目的

mod_cache_disk モジュールにおいて、ディスクキャッシュのルートディレクトリおよびその配下のディレクトリ階層構造に関する設定情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    const char* cache_root;
    apr_size_t cache_root_len;
    int dirlevels;               /* Number of levels of subdirectories */
    int dirlength;               /* Length of subdirectory names */
} disk_cache_conf;

入力と出力

本構造体は設定保持用であり、直接的な関数呼び出しの入出力には関与しません。ただし、mod_cache_disk.c 内の各関数において、キャッシュファイルのパス生成やディレクトリ構造の作成時に参照されます。

前提条件と不変条件

  • cache_root はキャッシュを格納するベースディレクトリのパスを指します。
  • cache_root_len は cache_root 文字列の長さを保持します。
  • dirlevels および dirlength は、ファイルシステム上のファイル数制限を回避するためのディレクトリ階層設計を決定します。

アルゴリズムまたは振る舞い

本構造体は、キャッシュ対象のファイルパスを生成する際の基盤となります。mod_cache_disk.c 内の header_file や data_file 関数は、この構造体の設定値に基づき、ハッシュ値から階層化されたディレクトリパスを構築します。また、mkdir_structure 関数は、この設定に従って必要なディレクトリ構造を動的に生成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_cache_disk.c 内の ap_get_module_config を介してサーバー設定から取得されます。
  • apr_pcalloc を用いて設定初期化時にインスタンス化されます。
  • **呼び出し先**:
  • header_file, data_file, mkdir_structure, safe_file_rename, file_cache_el_final, file_cache_create 等の mod_cache_disk.c 内の関数に引数として渡され、パス生成やファイル操作のパラメータとして利用されます。

関連データ構造

  • disk_cache_object_t: キャッシュされた個々のリソースを表現する構造体。本設定構造体と組み合わせて使用されます。
  • disk_cache_file_t: ファイルI/Oを抽象化する構造体。本設定構造体を用いてファイルパスを特定します。

失敗モードと境界条件

  • dirlevels や dirlength の設定値が不適切な場合、ディレクトリ構造の生成に失敗し、キャッシュの保存や読み込みが正常に行われない可能性があります。
  • cache_root が無効なパスである場合、ファイル操作時にエラーが発生します。

保守担当者向けメモ

  • 本構造体は mod_cache_disk の設定の核となるため、変更時は mod_cache_disk.c 内のパス生成ロジックへの影響を必ず確認してください。
  • dirlevels と dirlength はファイルシステム上のファイル数制限を回避するための重要な設計要素です。これらを変更する際は、既存のキャッシュディレクトリ構造との互換性に注意してください。
