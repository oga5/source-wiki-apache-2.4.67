remove_items

定義

  • 種別: function
  • 定義箇所: modules/http/mod_mime.c:174

目的

ディレクトリ設定において、特定の拡張子に関連付けられたメタデータ（MIMEタイプ、エンコーディング等）をハッシュテーブルから削除または無効化するために使用されます。

シグネチャまたは形状

static void remove_items(apr_pool_t *p, apr_array_header_t *remove, apr_hash_t *mappings)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t。
  • remove: 削除対象の属性情報（attrib_info）を保持する配列。
  • mappings: 拡張子とメタデータ（extension_info）をマッピングするハッシュテーブル。
  • 出力: なし（mappings を直接更新）。

前提条件と不変条件

  • remove 配列内の各要素は、extension_info 構造体内の有効なオフセットを指している必要があります。
  • mappings は apr_hash_t として初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. remove 配列を走査し、各要素が指定する拡張子を mappings ハッシュから検索します。 2. 対象の extension_info が存在し、かつ指定されたオフセット位置のデータが非 NULL である場合：

  • 現在の extension_info を apr_pmemdup を用いてプール p 上に複製します。
  • 複製した構造体を mappings に再設定します（コピーオンライト的な挙動）。
  • 複製した構造体内の指定オフセット位置の値を NULL に設定し、該当するメタデータを無効化します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http/mod_mime.c:225（merge_mime_dir_configs 内で呼び出されています）。
  • 呼び出し先: apr_hash_get, apr_pmemdup, apr_hash_set（すべて APR ライブラリ関数）。

関連データ構造

  • attrib_info: 削除対象の属性名と構造体内のオフセットを保持。
  • extension_info: 拡張子に関連付けられたメタデータ（タイプ、言語、フィルタ等）を保持する構造体。
  • apr_hash_t: 拡張子をキーとして extension_info を管理するハッシュテーブル。

失敗モードと境界条件

  • mappings 内に指定された拡張子が存在しない場合、何も行われません。
  • extension_info 内の指定オフセットが既に NULL である場合、再設定は行われません。
  • メモリ不足時には apr_pmemdup が例外を発生させる可能性があります。

保守担当者向けメモ

  • 本関数は設定のマージ時に呼び出されるため、親設定のハッシュテーブルを直接変更せず、必要に応じてコピーを作成する設計になっています。
  • suffix[i].offset を使用したポインタ演算を行っているため、attrib_info の定義と extension_info 構造体のレイアウト変更には注意が必要です。
