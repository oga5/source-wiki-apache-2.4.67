set_idcheck

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_ident.c:243

目的

Apache HTTP Server の設定ディレクティブ IdentityCheck の値を解析し、モジュール固有の設定構造体 ident_config_rec に反映させる。

シグネチャまたは形状

static const char *set_idcheck(cmd_parms *cmd, void *d_, int arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • d_: 設定構造体 ident_config_rec へのポインタ（void * として渡される）。
  • arg: ディレクティブの値（IdentityCheck On なら 1、Off なら 0）。
  • **出力**:
  • 成功時は NULL を返す。

前提条件と不変条件

  • d_ は ident_config_rec 型の構造体としてキャスト可能であること。
  • arg は int 型として評価され、1 または 0 に正規化される。

アルゴリズムまたは振る舞い

1. 引数 d_ を ident_config_rec *d にキャストする。 2. arg の真偽値に基づき、d->do_rfc1413 フィールドを 1 または 0 に設定する。 3. NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_ident.c:290 において、AP_INIT_FLAG("IdentityCheck", ...) マクロを通じて設定解析時に呼び出される。
  • **呼び出し先**: なし。

関連データ構造

  • ident_config_rec: モジュール固有の設定を保持する構造体。do_rfc1413 フィールドを更新対象とする。

失敗モードと境界条件

  • 内部でエラーチェックを行わず常に NULL を返すため、設定値の妥当性は呼び出し元の Apache 設定解析フレームワークに依存する。

保守担当者向けメモ

  • 本関数は AP_INIT_FLAG を介して呼び出されるため、シグネチャは Apache のコマンドハンドラ規約に従っている。
  • do_rfc1413 の値は、後に ap_ident_lookup 関数内で IDENT プロトコルによる照会を行うかどうかの判定に使用される。
