register_hooks

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_cookie.c:268

目的

mod_session_cookie モジュールのセッション読み込みおよび保存機能を、Apache HTTP Server のセッション管理フックに登録します。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p （モジュール初期化時に使用されるメモリプール）
  • 出力: なし（戻り値は void）

前提条件と不変条件

  • 本関数は Apache のモジュール初期化プロセスにおいて呼び出される必要があります。
  • session_cookie_load および session_cookie_save が同一ファイル内で定義されている必要があります。

アルゴリズムまたは振る舞い

1. ap_hook_session_load を呼び出し、session_cookie_load 関数を APR_HOOK_MIDDLE の優先度でセッション読み込みフックに登録します。 2. ap_hook_session_save を呼び出し、session_cookie_save 関数を APR_HOOK_MIDDLE の優先度でセッション保存フックに登録します。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュール読み込みプロセス（直接的な呼び出し元はソースコード上では明示されていませんが、http_config.h の規約に基づき、モジュール構造体を通じて呼び出されます）。
  • 呼び出し先:
  • ap_hook_session_load
  • ap_hook_session_save
  • session_cookie_load （関数ポインタとして渡される）
  • session_cookie_save （関数ポインタとして渡される）

関連データ構造

  • apr_pool_t: Apache のメモリ管理用プール。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。フック登録の失敗は Apache の起動プロセス全体に影響を及ぼす可能性があります。

保守担当者向けメモ

  • 本関数は mod_session が提供するフックインターフェースに依存しています。mod_session 側のフック定義が変更された場合、本関数の呼び出しも修正が必要になる可能性があります。
  • ツールによる呼び出し元リストには他のモジュールの register_hooks が含まれていますが、これらは本シンボルとは別の関数です。本シンボルは mod_session_cookie.c 内の固有の関数です。
