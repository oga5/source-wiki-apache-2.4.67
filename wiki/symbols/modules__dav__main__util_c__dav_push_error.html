dav_push_error

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:66

目的

既存のエラーオブジェクトを新しいエラー情報でラップし、エラーのスタック（連結リスト）を構築します。

シグネチャまたは形状

dav_error* dav_push_error(apr_pool_t *p, int status, int error_id, const char *desc, dav_error *prev)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • status: HTTP ステータスコード。
  • error_id: DAV 固有のエラー識別子。
  • desc: エラー内容を説明する文字列。
  • prev: 連結する既存の dav_error オブジェクト（または NULL）。
  • **出力**:
  • 新規作成された dav_error オブジェクト。このオブジェクトの prev メンバに prev が格納されます。

前提条件と不変条件

  • p は有効なメモリプールである必要があります。
  • 戻り値の dav_error オブジェクトは、プール p から割り当てられます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、プール p から dav_error 構造体分のメモリをゼロ初期化して確保します。 2. 引数として渡された status、error_id、desc を構造体に設定します。 3. 引数 prev を構造体の prev メンバに代入し、エラーの連結リストを形成します。 4. 新しく作成したエラーオブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/lock.c
  • modules/dav/fs/repos.c
  • modules/dav/lock/locks.c
  • modules/dav/main/mod_dav.c
  • **呼び出し先**:
  • apr_pcalloc (APR ライブラリ関数)

関連データ構造

  • dav_error: エラー情報を保持する構造体。status、error_id、desc、および次のエラーを指す prev ポインタを持ちます。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが異常終了する可能性があります（APR のメモリ管理規約に依存）。
  • prev に NULL を渡すことで、エラーチェーンの末尾（または単一のエラー）として機能します。

保守担当者向けメモ

  • 本関数はエラーのスタック化を行うため、呼び出し側でエラーを追跡する際に、リストの先頭から順に prev を辿ることで発生順序の逆順（最新が先頭）でエラー情報を取得可能です。
