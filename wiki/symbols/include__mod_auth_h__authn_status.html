authn_status

定義

  • 種別: enum
  • 定義箇所: include/mod_auth.h:64

目的

認証（Authentication）プロバイダーがリクエスト処理の結果をサーバーコアへ通知するための共通ステータスコードを定義します。

シグネチャまたは形状

typedef enum {
    AUTH_DENIED,
    AUTH_GRANTED,
    AUTH_USER_FOUND,
    AUTH_USER_NOT_FOUND,
    AUTH_GENERAL_ERROR
} authn_status;

入力と出力

  • **出力**: 認証プロバイダーの関数（check_password や get_realm_hash など）が戻り値として返します。

前提条件と不変条件

  • 認証プロバイダーは、処理結果に応じて必ず定義されたいずれかの列挙値を返す必要があります。

アルゴリズムまたは振る舞い

  • AUTH_DENIED: 認証が拒否されたことを示します。
  • AUTH_GRANTED: 認証が成功したことを示します。
  • AUTH_USER_FOUND: ユーザーが存在することを確認できた状態を示します。
  • AUTH_USER_NOT_FOUND: ユーザーが見つからなかったことを示します。
  • AUTH_GENERAL_ERROR: 認証処理中に予期せぬエラーが発生したことを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_auth_basic.c, mod_auth_digest.c, mod_auth_form.c, mod_authn_anon.c, mod_authn_core.c 等の認証プロバイダー実装モジュール。
  • **呼び出し先**: 情報なし。

関連データ構造

  • authn_provider: 認証プロバイダーのインターフェースを定義する構造体。本列挙型を戻り値として使用します。

失敗モードと境界条件

  • AUTH_GENERAL_ERROR は、認証プロバイダー内部でのデータベース接続失敗や設定不備など、処理を継続できない異常系で使用されます。

保守担当者向けメモ

  • 本列挙型は認証フローの制御に直結するため、新しいステータスを追加する場合は、コアモジュール（mod_authn_core.c）側の評価ロジックも併せて更新する必要があります。
