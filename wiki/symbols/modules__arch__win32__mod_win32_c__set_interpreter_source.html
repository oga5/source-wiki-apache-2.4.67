set_interpreter_source

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_win32.c:76

目的

Apacheの設定ディレクティブ ScriptInterpreterSource の値を解析し、ディレクトリごとの設定構造体 win32_dir_conf にインタープリタ解決戦略を格納します。

シグネチャまたは形状

static const char *set_interpreter_source(cmd_parms *cmd, void *dv, char *arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • dv: ディレクトリ設定構造体 (win32_dir_conf へのポインタ)。
  • arg: 設定値として渡された文字列（registry, registry-strict, script のいずれか）。
  • **出力**:
  • 成功時: NULL を返します。
  • 失敗時: エラーメッセージを含む文字列（apr_pstrcat で生成）を返します。

前提条件と不変条件

  • dv は win32_dir_conf 型としてキャスト可能である必要があります。
  • arg は大文字小文字を区別せずに比較されます。

アルゴリズムまたは振る舞い

1. arg の値を比較し、対応する列挙型を d->script_interpreter_source に設定します。

  • "registry": INTERPRETER_SOURCE_REGISTRY
  • "registry-strict": INTERPRETER_SOURCE_REGISTRY_STRICT
  • "script": INTERPRETER_SOURCE_SHEBANG

    • 2. 上記のいずれにも一致しない場合、許可された値を列挙したエラーメッセージを生成し、cmd->temp_pool を使用して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_win32.c:546 において、AP_INIT_TAKE1 マクロを通じて設定ディレクティブ ScriptInterpreterSource のハンドラとして登録されています。
  • **呼び出し先**:
  • strcasecmp (標準Cライブラリ)
  • apr_pstrcat (APRライブラリ)

関連データ構造

  • win32_dir_conf: ディレクトリ単位の設定を保持する構造体。
  • interpreter_source_e: script_interpreter_source フィールドが保持する列挙型。

失敗モードと境界条件

  • 不正な引数が指定された場合、エラーメッセージを返して設定の読み込みを拒否します。
  • メモリ確保は cmd->temp_pool に依存しており、プールが枯渇した場合は apr_pstrcat の挙動に従います。

保守担当者向けメモ

  • 本関数は ScriptInterpreterSource ディレクティブの構文解析のみを担当します。実際のインタープリタ解決ロジックは ap_cgi_build_command 等の別関数で実装されています。
  • 新しい解決戦略を追加する場合は、列挙型 interpreter_source_e の拡張と、本関数の条件分岐への追加が必要です。
