ap_session_set

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:296

目的

セッションデータに対してキーと値のペアを設定または削除し、セッションが変更されたことを示すフラグを立てます。

シグネチャまたは形状

apr_status_t ap_session_set(request_rec *r, session_rec *z, const char *key, const char *value);

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • z: 操作対象のセッションレコード（NULL の場合は自動的にロードを試みる）。
  • key: 設定または削除するセッションキー。
  • value: 設定する値。NULL の場合は該当するキーを削除。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS を返します。セッションのロードに失敗した場合はそのエラーコードを返します。

前提条件と不変条件

  • z が NULL の場合、ap_session_load を介してセッションのロードが試行されます。
  • value が提供された場合は apr_table_set で値が更新され、NULL の場合は apr_table_unset でキーが削除されます。
  • 操作が成功すると、セッションレコードの dirty フラグが 1 に設定されます。

アルゴリズムまたは振る舞い

1. z が NULL の場合、ap_session_load(r, &z) を呼び出し、ロードに失敗すれば即座にエラーを返します。 2. z が有効な場合、value の有無を確認します。 3. value が存在すれば z->entries テーブルにキーと値を設定し、存在しなければキーを削除します。 4. 変更が適用された後、z->dirty を 1 に設定し、変更があったことを記録します。 5. APR_SUCCESS を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c にて APR_RETRIEVE_OPTIONAL_FN を通じて取得・利用されていることが確認できます。
  • **呼び出し先**:
  • ap_session_load（z が NULL の場合）
  • apr_table_set / apr_table_unset（apr_tables.h）

関連データ構造

  • session_rec: セッション情報を保持する構造体。entries（apr_table_t）および dirty フラグを含みます。

失敗モードと境界条件

  • ap_session_load が失敗した場合、そのエラーコードがそのまま返されます。
  • z が NULL であり、かつ ap_session_load も失敗した場合、セッションの更新は行われません。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN により外部モジュールから利用可能なオプション関数として登録されています。
  • dirty フラグは、後の session_output_filter において、ストレージへの書き込み（ap_run_session_save）が必要かどうかを判断するために使用されます。
