ap_rxplus_pmatch

定義

  • 種別: function
  • 定義箇所: server/util_regex.c:205

目的

正規表現のマッチング結果から、指定したキャプチャグループの内容をメモリプールから割り当てられた文字列として取得します。

シグネチャまたは形状

char* ap_rxplus_pmatch(apr_pool_t *pool, ap_rxplus_t *rx, int n)

入力と出力

  • **入力**:
  • pool: 文字列の割り当てに使用する apr_pool_t。
  • rx: マッチング結果を保持する ap_rxplus_t 構造体。
  • n: 取得対象のキャプチャグループ番号。
  • **出力**:
  • 指定されたキャプチャグループの内容を含む、pool から割り当てられた文字列（char*）。

前提条件と不変条件

  • rx は事前に ap_rxplus_exec() 等によってマッチング処理が実行済みである必要があります。
  • rx->match は ap_rxplus_exec() を呼び出すたびに更新されるため、マッチ結果を永続的に保持したい場合に本関数を使用します。

アルゴリズムまたは振る舞い

1. ap_rxplus_match() を呼び出し、対象のキャプチャグループ n に対応する文字列のポインタと長さを取得します。 2. apr_pstrndup() を使用して、取得した文字列を pool 上に複製します。 3. 複製された文字列のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • ap_rxplus_match()
  • apr_pstrndup()
  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:1237
  • modules/aaa/mod_authnz_fcgi.c:1243

関連データ構造

  • ap_rxplus_t: 正規表現のコンパイル状態およびマッチング結果を保持するメイン構造体。
  • apr_pool_t: メモリ管理を行うためのプール構造体。

失敗モードと境界条件

  • n で指定したグループが存在しない場合や、マッチングが成功していない場合の挙動は ap_rxplus_match() の仕様に依存します。
  • pool のメモリ不足時には apr_pstrndup が異常終了する可能性があります。

保守担当者向けメモ

  • 本関数は ap_rxplus_exec() が保持する一時的なマッチング結果を、呼び出し側の生存期間に合わせて安全にコピーするためのユーティリティです。
  • ap_rxplus_match が返すポインタは元の文字列を指しているため、ap_rxplus_exec の再実行による破壊を防ぐために本関数によるコピーが推奨されます。
