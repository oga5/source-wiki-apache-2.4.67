add_icon

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_autoindex.c:241

目的

Apache HTTP Server の設定ディレクティブ（AddIcon, AddIconByType, AddIconByEncoding）を解析し、ディレクトリごとの設定構造体 autoindex_config_rec にアイコン定義を追加する。

シグネチャまたは形状

static const char *add_icon(cmd_parms *cmd, void *d, const char *icon, const char *to)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。cmd->info には設定種別（BY_PATH, BY_TYPE, BY_ENCODING）が格納される。
  • d: 現在のディレクトリ設定構造体 (autoindex_config_rec)。
  • icon: アイコンのパス、または (alt,icon) 形式の文字列。
  • to: アイコンを適用する対象（ファイル拡張子、MIMEタイプ、またはエンコーディング）。
  • **出力**:
  • 成功時は NULL を返す。
  • 括弧の閉じ忘れなど、解析エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • d は autoindex_config_rec 型としてキャスト可能であること。
  • cmd->temp_pool が有効なメモリプールとして利用可能であること。

アルゴリズムまたは振る舞い

1. icon 引数が ( で始まる場合、alt（代替テキスト）とアイコンパスを分離し、add_alt を呼び出して代替テキストを登録する。 2. cmd->info が BY_PATH の場合、to が **DIRECTORY** であれば ^^DIRECTORY^^ に置換する。 3. cmd->info が BY_ENCODING の場合、to を小文字に変換する。 4. 最終的に push_item を呼び出し、autoindex_config_rec 内の icon_list にアイコン情報を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_autoindex.c 内の AP_INIT_ITERATE2 マクロ経由で、設定ファイル読み込み時に呼び出される。
  • **呼び出し先**:
  • add_alt: 代替テキストの登録に使用。
  • push_item: アイコンリストへのデータ追加に使用。
  • apr_pstrdup, ap_getword_nc, ap_str_tolower: 文字列操作ユーティリティ。

関連データ構造

  • autoindex_config_rec: ディレクトリごとの設定を保持する構造体。
  • item: icon_list を構成する個々の要素。

失敗モードと境界条件

  • icon 引数が ( で始まるにもかかわらず閉じ括弧 ) が見つからない場合、エラーメッセージを返して終了する。

保守担当者向けメモ

  • 本関数は設定解析フェーズでのみ実行される。
  • icon 引数の解析には cmd->temp_pool を使用しており、リクエスト処理時ではなく設定読み込み時のメモリ管理に従う。
