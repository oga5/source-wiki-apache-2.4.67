util_ald_cache_purge

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache_mgr.c:232

目的

LDAP キャッシュ内の期限切れエントリを走査し、メモリを解放してキャッシュの健全性を維持する。

シグネチャまたは形状

void util_ald_cache_purge(util_ald_cache_t *cache)

入力と出力

  • **入力**: cache (util_ald_cache_t *) - パージ対象のキャッシュ構造体へのポインタ。
  • **出力**: なし（副作用としてキャッシュ内のノードが削除される）。

前提条件と不変条件

  • cache が NULL の場合、関数は何もせず直ちに復帰する。
  • キャッシュ内の各ノードは add_time を保持しており、これが marktime より古いノードが削除対象となる。
  • 共有メモリ環境下では、util_ald_free を通じて apr_rmm 管理下のメモリが正しく解放される必要がある。

アルゴリズムまたは振る舞い

1. 現在時刻を取得し、cache->last_purge を更新。 2. now - cache->ttl が現在の marktime より過去であれば、marktime を更新してパージ範囲を拡大する。 3. キャッシュ内の全バケットを走査する。 4. 各バケットの連結リストを辿り、p->add_time < cache->marktime を満たすノードを特定する。 5. 該当ノードに対し、cache->free 関数でペイロードを解放し、util_ald_free でノード自体を解放する。 6. 最後に、パージ処理にかかった時間を計算し、cache->avg_purgetime を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap_cache_mgr.c 内のキャッシュ操作関数（util_ald_cache_insert 等の文脈で呼び出されることが示唆されている）。
  • ソースコード内の直接的な呼び出し箇所: modules/ldap/util_ldap_cache_mgr.c:480, 507, 535。
  • **呼び出し先**:
  • apr_time_now()
  • cache->free (関数ポインタ)
  • util_ald_free

関連データ構造

  • util_ald_cache_t: キャッシュの管理構造体。
  • util_cache_node_t: キャッシュ内の各エントリを保持する連結リストノード。

失敗モードと境界条件

  • **キャッシュ空**: cache が空の場合、安全に早期リターンする。
  • **全エントリ削除**: 全ノードが marktime より古い場合、すべてのノードが解放され、numentries は 0 になる。
  • **パフォーマンス**: O(n) の全走査を行うため、キャッシュサイズが極めて大きい場合、パージ処理中にレイテンシが発生する可能性がある。

保守担当者向けメモ

  • 本関数はキャッシュの強制パージを行うため、頻繁な呼び出しはパフォーマンスに影響を与える。
  • avg_purgetime の計算には numpurges を使用しているため、ゼロ除算が発生しないよう呼び出し順序に注意が必要（本実装では numpurges をインクリメントした後に計算している）。
