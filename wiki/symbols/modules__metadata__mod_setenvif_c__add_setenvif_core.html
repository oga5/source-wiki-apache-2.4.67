add_setenvif_core

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_setenvif.c:282

目的

SetEnvIf 系ディレクティブの解析および設定の登録を行う共通処理関数。正規表現や単純文字列パターンのコンパイル、および sei_entry 構造体への格納を担当する。

シグネチャまたは形状

static const char *add_setenvif_core(cmd_parms *cmd, void *mconfig, char *fname, const char *args);

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • mconfig: 現在のディレクトリ設定レコード。
  • fname: 対象となるヘッダ名または特殊属性名。
  • args: ディレクティブの引数（正規表現と設定する環境変数のリスト）。
  • **出力**: 成功時は NULL、失敗時はエラーメッセージを含む文字列を返す。

前提条件と不変条件

  • cmd->path が NULL の場合はサーバー全体の設定として、それ以外はディレクトリ単位の設定として処理される。
  • 同一ヘッダ名に対する連続したルールは、ポインタ比較による最適化のため、既存の sei_entry とマージされる場合がある。

アルゴリズムまたは振る舞い

1. **設定レコードの特定**: cmd->path に基づき、適用対象の sei_cfg_rec を取得する。 2. **正規表現の取得**: ap_getword_conf で正規表現パターンを抽出する。 3. **既存ルールの確認**: 同一ヘッダ名、正規表現、大文字小文字区別フラグを持つ既存ルールがあるか走査する。 4. **新規エントリの作成**: 一致するルールがない場合、apr_array_push で新しい sei_entry を追加する。

  • 単純文字列の場合は apr_strmatch_precompile で最適化する。
  • 正規表現の場合は ap_pregcomp でコンパイルする。

    • 5. **特殊属性の判定**: remote_addr 等の特殊なヘッダ名であれば special_type を設定し、それ以外は is_header_regex を用いてヘッダ名自体が正規表現か判定する。 6. **環境変数の登録**: 最後に add_envvars を呼び出し、環境変数の設定処理を完了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_setenvif.c:415
  • modules/metadata/mod_setenvif.c:469
  • **呼び出し先**:
  • ap_getword_conf, ap_get_module_config, apr_array_push, apr_strmatch_precompile, ap_pregcomp, is_header_regex, add_envvars

関連データ構造

  • sei_cfg_rec: 設定を保持するコンテナ。
  • sei_entry: 個別の条件ルールを保持する構造体。
  • cmd_parms: Apache 設定コマンドのパラメータ。

失敗モードと境界条件

  • 正規表現が指定されていない場合、エラーを返す。
  • パターンのコンパイル（apr_strmatch_precompile または ap_pregcomp）に失敗した場合、エラーを返す。
  • ヘッダ名が正規表現としてコンパイルできない場合、エラーを返す。

保守担当者向けメモ

  • 本関数は SetEnvIf および SetEnvIfNoCase の両方から呼び出される共通ロジックである。
  • cmd->info を使用して大文字小文字の区別（ICASE_MAGIC）を判定しているため、呼び出し側でのフラグ設定に依存している。
