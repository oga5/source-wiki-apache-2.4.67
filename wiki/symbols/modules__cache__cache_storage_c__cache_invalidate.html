cache_invalidate

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_storage.c:690

目的

POST、PUT、DELETE 等のメソッドに応答して、指定されたリクエスト URI および関連する Location / Content-Location ヘッダーが指すキャッシュエントリを無効化する。

シグネチャまたは形状

int cache_invalidate(cache_request_rec *cache, request_rec *r)

入力と出力

  • **入力**:
  • cache: キャッシュリクエストコンテキスト。
  • r: 現在のリクエスト構造体。r->headers_out から無効化対象の URI を特定する。
  • **出力**:
  • OK: 少なくとも1つのプロバイダーで無効化処理が成功した場合。
  • DECLINED: キャッシュ情報が不足している場合、またはキー生成に失敗した場合。

前提条件と不変条件

  • cache が NULL でないこと。
  • cache->key が未設定の場合、cache_generate_key により自動生成される。
  • Location および Content-Location ヘッダーの無効化は、ホスト名が現在のリクエストと一致する場合のみ行われる。

アルゴリズムまたは振る舞い

1. リクエスト URI のキャッシュキーを生成（未設定時）。 2. r->headers_out から Location および Content-Location を取得し、それぞれ正規化されたキャッシュキーを算出。 3. cache->providers リストを反復処理し、各プロバイダーに対して以下の順序で無効化を試行：

  • リクエスト URI に対応するエンティティを開き、無効化。
  • Location キーが存在する場合、対応するエンティティを開き、無効化。
  • Content-Location キーが存在する場合、対応するエンティティを開き、無効化。

    • 4. 処理結果を status に保持し、全プロバイダーの走査終了後に返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache.c:1702 (および同ファイル内のフィルタ処理)。
  • **呼び出し先**:
  • cache_generate_key (キー生成)
  • cache_canonicalise_key (キー正規化)
  • list->provider->open_entity (エンティティの取得)
  • list->provider->invalidate_entity (エンティティの無効化)

関連データ構造

  • cache_request_rec: キャッシュのプロバイダーリストとキーを保持。
  • cache_handle_t: キャッシュエントリへのアクセスハンドル。
  • apr_uri_t: URI 解析結果の保持に使用。

失敗モードと境界条件

  • cache が NULL の場合、エラーログを出力して DECLINED を返す。
  • cache_generate_key が失敗した場合、DECLINED を返す。
  • ホスト名が一致しない Location / Content-Location は無効化対象から除外される。
  • プロバイダーが open_entity に失敗した場合、そのエンティティの無効化はスキップされる。

保守担当者向けメモ

  • 本関数は r->pool を使用してメモリを割り当てるため、明示的な解放は不要。
  • Location 等のヘッダー解析において、apr_uri_parse および cache_canonicalise_key の結果が不完全な場合はキーを NULL と見なし、無効化をスキップする安全な設計となっている。
