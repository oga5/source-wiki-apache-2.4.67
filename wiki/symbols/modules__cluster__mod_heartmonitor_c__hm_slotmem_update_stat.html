hm_slotmem_update_stat

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartmonitor.c:195

目的

共有メモリ（slotmem）上のサーバー統計情報を、指定された hm_server_t の内容で更新または新規作成します。

シグネチャまたは形状

apr_status_t hm_slotmem_update_stat(hm_server_t *s, apr_pool_t *pool)

入力と出力

  • **入力**:
  • s: 更新対象のサーバー状態を保持する hm_server_t 構造体へのポインタ。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • 戻り値: 常に APR_SUCCESS を返します。

前提条件と不変条件

  • storage および slotmem が初期化済みであること。
  • s が指す構造体には、有効な IP アドレス、busy 数、ready 数、および最終確認時刻（seen）が格納されていること。

アルゴリズムまたは振る舞い

1. storage->doall を呼び出し、既存の共有メモリ領域を hm_update 関数で走査して、一致するサーバーエントリの更新を試みます。 2. ctx.found が真であれば、更新完了として終了します。 3. 一致するエントリが見つからなかった場合（新規サーバーの場合）:

  • hm_slot_server_t 構造体をゼロ初期化し、s の情報をコピーします。
  • storage->grab を呼び出して、共有メモリ内の空きスロットインデックス i を取得します。
  • storage->put を呼び出し、取得したインデックス i にサーバー情報を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cluster/mod_heartmonitor.c:401
  • modules/cluster/mod_heartmonitor.c:495
  • **呼び出し先**:
  • storage->doall
  • storage->grab
  • storage->put
  • hm_update (コールバックとして渡される)

関連データ構造

  • hm_server_t: 内部的なサーバー状態管理構造体。
  • hm_slot_server_ctx_t: 共有メモリ操作時のコンテキスト保持用構造体。
  • hm_slot_server_t: 共有メモリ上に保存されるサーバー情報の形式。

失敗モードと境界条件

  • 共有メモリが満杯の場合、storage->grab の挙動に依存します。
  • 本関数自体はエラー発生時も APR_SUCCESS を返す設計となっています。
  • 排他制御については、ソース内に「grab() / put() 時のロックが必要」との注釈があります。

保守担当者向けメモ

  • storage->grab および storage->put の呼び出し時に、明示的なロック機構が実装されていない点に注意してください。
  • hm_update コールバック関数との連携により、既存エントリの検索と更新が抽象化されています。
