link_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.c:736

目的

RFC 5988 に基づく Link ヘッダーのパースおよび URI 書き換え処理において、解析中の状態と対象文字列を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    apr_pool_t *pool;
    request_rec *r;
    proxy_dir_conf *conf;
    const char *s;
    int slen;
    int i;
    const char *server_uri;
    int su_len;
    const char *real_backend_uri;
    int rbu_len;
    const char *p_server_uri;
    int psu_len;
    int link_start;
    int link_end;
} link_ctx;

入力と出力

  • **入力**: パース対象の文字列 (s)、その長さ (slen)、およびリバースプロキシ環境下での URI 書き換えに必要なベース URI 情報 (server_uri, real_backend_uri, p_server_uri) を保持します。
  • **出力**: パースの進行状況 (i) や、特定されたリンクの範囲 (link_start, link_end) を更新し、後続の URI 置換処理に利用されます。

前提条件と不変条件

  • pool はメモリ管理のために有効な apr_pool_t である必要があります。
  • s はパース対象の Link ヘッダー文字列を指します。
  • i は常に 0 から slen の範囲内である必要があります。

アルゴリズムまたは振る舞い

本構造体は、再帰的でないステートマシンとして動作するパース関数群（skip_ws, read_chr, skip_qstring, read_link 等）の引数として渡されます。各関数は link_ctx 内のポインタやインデックスを更新しながら、トークン単位で文字列を走査し、必要に応じて subst_str を用いて URI の書き換えを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_proxy_link_reverse_map（ソースコード内の文脈より推測されるが、本抜粋には詳細なし）。
  • **呼び出し先**: 以下の関数が link_ctx を引数として受け取り、状態を更新します。
  • skip_ws, find_chr, read_chr, skip_qstring, skip_ptoken, read_link, skip_pname, skip_pvalue, skip_param, read_sep, subst_str

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • proxy_dir_conf: プロキシ設定情報。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • 文字列の終端 (slen) を超えたアクセスが発生しないよう、各パース関数内でインデックス i のチェックが行われる設計となっています。
  • subst_str 実行時にメモリ割り当てが失敗した場合の挙動については、本抜粋からは不明です。

保守担当者向けメモ

  • 本構造体は RFC 5988 の複雑なパースルールを実装するためのステートマシン用コンテキストです。
  • URI 書き換えロジックを変更する場合は、link_start と link_end が正しく更新されているかを確認してください。
  • 呼び出し元関数の詳細なフローについては、h2_proxy_util.c の 800 行目以降のパース関数群を参照してください。
