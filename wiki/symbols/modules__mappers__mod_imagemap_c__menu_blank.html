menu_blank

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_imagemap.c:491

目的

画像マップの代替メニュー生成において、指定されたメニュー形式（menu）に応じた空行または改行タグをレスポンスに出力します。

シグネチャまたは形状

static void menu_blank(request_rec *r, char *menu)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報を保持する request_rec 構造体へのポインタ。
  • menu: メニューの形式を指定する文字列（"formatted", "semiformatted", "unformatted"）。
  • **出力**: なし（戻り値は void）。ap_rputs を通じてレスポンスストリームへ直接出力します。

前提条件と不変条件

  • menu 引数は大文字小文字を区別せずに比較されます。
  • r は有効なリクエストオブジェクトである必要があります。

アルゴリズムまたは振る舞い

1. menu 文字列の内容を判定します。 2. menu が "formatted" または "unformatted" の場合、改行文字 \n を出力します。 3. menu が "semiformatted" の場合、HTML の改行タグ <br />\n を出力します。 4. いずれにも該当しない場合、何も出力せずに終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_imagemap.c:646
  • **呼び出し先**: Apache HTTP Server の出力関数 ap_rputs を呼び出します。

関連データ構造

  • request_rec: Apache のリクエスト処理用構造体。

失敗モードと境界条件

  • menu が上記3つのいずれにも一致しない場合、関数は何も行わず正常に終了します。
  • r が NULL の場合、ap_rputs の呼び出しによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は mod_imagemap におけるメニュー生成時のレイアウト調整用ヘルパーです。
  • ap_rputs を使用しているため、出力はリクエストのレスポンスバッファに直接書き込まれます。
