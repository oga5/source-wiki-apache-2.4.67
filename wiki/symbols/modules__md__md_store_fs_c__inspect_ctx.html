inspect_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_store_fs.c:782

目的

ファイルシステムベースのストレージにおいて、特定の条件に基づいたデータ走査や検査処理を行う際に、コンテキスト情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    md_store_fs_t *s_fs;
    md_store_group_t group;
    const char *pattern;
    const char *aspect;
    md_store_vtype_t vtype;
    md_store_inspect *inspect;
    const char *dirname;
    void *baton;
    apr_time_t ts;
} inspect_ctx;

入力と出力

本構造体は関数間でコンテキストを渡すためのコンテナです。

  • 入力: 各フィールドは、走査対象のグループ、パターン、検査用コールバック（inspect）、およびユーザーデータ（baton）などで構成されます。
  • 出力: 走査処理の結果や状態が、この構造体を通じて参照・更新されます。

前提条件と不変条件

  • s_fs は有効な md_store_fs_t インスタンスを指している必要があります。
  • baton は、呼び出し元が提供する任意のデータ構造を指すポインタとして使用されます。

アルゴリズムまたは振る舞い

ファイルシステム上のデータを走査する際、この構造体に格納された条件（pattern や group）に基づいて対象を絞り込み、inspect コールバック関数を呼び出すことで個別の検査を実行します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_store_fs.c 内の走査系関数（fs_iterate 等の内部処理）からインスタンス化および参照されます。
  • 呼び出し先: inspect フィールドに設定されたコールバック関数が、この構造体を引数（baton 経由など）として受け取り実行されます。

関連データ構造

  • md_store_fs_t: ストレージのメイン構造体。
  • md_store_group_t: 管理対象のデータグループ。
  • md_store_inspect: 検査処理を定義する関数ポインタ型。

失敗モードと境界条件

  • s_fs が NULL の状態で走査を開始した場合、未定義の動作を引き起こす可能性があります。
  • pattern や dirname が適切に初期化されていない場合、ファイルパスの生成に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は md_store_fs.c 内の走査ロジックに密結合しています。走査条件を追加する場合は、この構造体にフィールドを追加し、関連する走査関数を更新してください。
  • baton フィールドは型安全ではないため、キャスト時の型整合性に注意してください。
