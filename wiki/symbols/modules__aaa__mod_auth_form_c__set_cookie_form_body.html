set_cookie_form_body

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:245

目的

Apache HTTP Server の設定ディレクティブ AuthFormBody を処理し、認証フォームのボディ情報を設定構造体に格納します。

シグネチャまたは形状

static const char *set_cookie_form_body(cmd_parms *cmd, void *config, const char *body)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • config: 現在のディレクトリ設定構造体 (auth_form_config_rec)。
  • body: 設定ファイルから読み取られたボディ文字列。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は check_string が返すエラーメッセージ文字列を返します。

前提条件と不変条件

  • config 引数は auth_form_config_rec 型としてキャスト可能である必要があります。
  • body が設定されると、対応するフラグ body_set が 1 にセットされます。

アルゴリズムまたは振る舞い

1. config を auth_form_config_rec 構造体にキャストします。 2. 引数 body を構造体の body メンバに代入します。 3. 構造体の body_set メンバを 1 に設定します。 4. check_string を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c:367 において、AP_INIT_TAKE1 マクロを通じて AuthFormBody ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: check_string を呼び出します。

関連データ構造

  • auth_form_config_rec: ディレクトリごとの認証設定を保持する構造体。

失敗モードと境界条件

  • check_string がエラーを返した場合、そのエラーがそのまま呼び出し元に伝播します。

保守担当者向けメモ

  • 本関数は AuthFormBody ディレクティブ専用のセッターです。設定の検証ロジックは check_string に依存しています。
