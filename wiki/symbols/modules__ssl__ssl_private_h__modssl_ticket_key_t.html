modssl_ticket_key_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:758

目的

TLS セッションチケットの暗号化および復号に使用される鍵情報を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const char *file_path;
    unsigned char key_name[16];
#if OPENSSL_VERSION_NUMBER < 0x30000000L
    unsigned char hmac_secret[16];
#else
    OSSL_PARAM mac_params[3];
#endif
    unsigned char aes_key[16];
} modssl_ticket_key_t;

入力と出力

本構造体はデータ保持用であり、関数ではありません。入力として鍵ファイルパス、鍵名、HMAC シークレット（OpenSSL 1.1.x 以前）、または MAC パラメータ（OpenSSL 3.0 以降）、および AES 鍵を格納します。

前提条件と不変条件

  • key_name および aes_key は 16 バイトの固定長です。
  • OpenSSL のバージョンに応じて、HMAC 関連のメンバ（hmac_secret または mac_params）が条件付きでコンパイルされます。

アルゴリズムまたは振る舞い

TLS セッションチケットの保護に必要な暗号学的パラメータをカプセル化します。OpenSSL 3.0 以降では、OSSL_PARAM を使用して MAC 関連のパラメータを柔軟に管理する設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_init.c:1762 (初期化処理にて参照)
  • modules/ssl/ssl_engine_kernel.c:2447 (リクエスト処理にて参照)
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • modules/ssl/ssl_private.h 内の SSLSrvConfigRec 構造体（ticket_key メンバとして保持）。

失敗モードと境界条件

  • 鍵ファイルが正しく読み込めない場合や、OpenSSL バージョンと構造体定義の不整合が発生した場合に、セッションチケットの生成または復号が失敗する可能性があります。

保守担当者向けメモ

  • OpenSSL 3.0 への移行に伴い、hmac_secret から OSSL_PARAM への変更が行われています。新しい暗号化パラメータを追加する際は、バージョン分岐を適切に維持してください。
