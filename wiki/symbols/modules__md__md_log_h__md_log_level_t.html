md_log_level_t

定義

  • 種別: enum
  • 定義箇所: modules/md/md_log.h:20

目的

mod_md モジュール内におけるログ出力の重要度（レベル）を定義し、実行時のログ出力制御やフィルタリングを可能にするための列挙型です。

シグネチャまたは形状

typedef enum {
    MD_LOG_EMERG,
    MD_LOG_ALERT,
    MD_LOG_CRIT,
    MD_LOG_ERR, 
    MD_LOG_WARNING, 
    MD_LOG_NOTICE, 
    MD_LOG_INFO, 
    MD_LOG_DEBUG, 
    MD_LOG_TRACE1, 
    MD_LOG_TRACE2, 
    MD_LOG_TRACE3, 
    MD_LOG_TRACE4, 
    MD_LOG_TRACE5, 
    MD_LOG_TRACE6, 
    MD_LOG_TRACE7, 
    MD_LOG_TRACE8, 
} md_log_level_t;

入力と出力

  • **入力**: ログ出力関数（md_log_perror）や判定関数（md_log_is_level）の引数として使用されます。
  • **出力**: md_log_level_name 関数により、対応する文字列名に変換されます。

前提条件と不変条件

  • md_log_level_t の値は、ログ出力の可否を判定する md_log_is_level の内部ロジックにおいて、現在の設定レベルと比較されます。
  • 順序は重要度を示しており、EMERG が最も高く、TRACE8 が最も低い詳細度を表します。

アルゴリズムまたは振る舞い

  • ログ出力の抽象化レイヤーにおいて、特定のログメッセージが現在のログレベル設定に対して出力対象であるかを判定する基準となります。
  • md_log_perror を呼び出す際、この列挙値によってログの重要度が識別され、最終的に Apache のログシステムへ渡される際のレベルとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • modules/md/md_acme_authz.c
  • modules/md/md_result.c
  • modules/md/md_log.c
  • **呼び出し先**:
  • このシンボル自体は列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • md_log_level_cb: ログレベル判定を行うコールバック関数の型定義。
  • md_log_print_cb: ログ出力を行うコールバック関数の型定義。

失敗モードと境界条件

  • 定義外の整数値をキャストして md_log_level_t として渡した場合の挙動は未定義です。
  • md_log_is_level 等の関数において、無効なレベルが渡された場合の具体的なエラーハンドリングについては、各関数の実装に依存します。

保守担当者向けメモ

  • 新しいログレベルを追加する場合は、md_log_level_name 関数内のマッピングテーブルも併せて更新する必要があります。
  • Apache 以外の環境で利用する場合も、この列挙型は共通のインターフェースとして維持されます。
