spri_cmp

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:138

目的

HTTP/2 ストリーム間の優先順位を比較し、ストリームツリー内での相対的な順序を決定します。

シグネチャまたは形状

static int spri_cmp(int sid1, nghttp2_stream *s1, 
                    int sid2, nghttp2_stream *s2, h2_session *session)

入力と出力

  • **入力**: 比較対象のストリームID (sid1, sid2)、nghttp2_stream オブジェクト (s1, s2)、および現在の h2_session。
  • **出力**: 比較結果を示す整数。負の値は s1 が優先、正の値は s2 が優先であることを示します。

前提条件と不変条件

  • s1 および s2 は有効な nghttp2_stream オブジェクトである必要があります。
  • ストリームが共通の親を持つ場合、重み付けに基づいた比較が行われます。

アルゴリズムまたは振る舞い

1. nghttp2_stream_get_parent を使用して、各ストリームの親ノードを取得します。 2. **共通の親を持つ場合**: 各ストリームの重みを nghttp2_stream_get_weight で取得し、その差分（w2 - w1）を返します。 3. **親が異なる場合**:

  • p1 が存在しない（ルートに近い）場合は -1 を返します。
  • p2 が存在しない場合は 1 を返します。
  • 上記以外の場合、親ノードを対象として再帰的に spri_cmp を呼び出し、ツリーの上位に向かって比較を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_session.c 内の他の関数から再帰的に呼び出されます。
  • **呼び出し先**: nghttp2_stream_get_parent および nghttp2_stream_get_weight（nghttp2 ライブラリ関数）。

関連データ構造

  • nghttp2_stream: nghttp2 ライブラリが管理するストリーム構造体。
  • h2_session: HTTP/2 セッションの状態を保持する構造体。

失敗モードと境界条件

  • ストリームがルートに近い場合、再帰の終了条件として親の有無が判定されます。
  • 循環参照や無効なストリームポインタが渡された場合の挙動については、本ソースコード内では明示されていません。

保守担当者向けメモ

  • 本関数は nghttp2 のストリームツリー構造に強く依存しています。nghttp2 の API 仕様変更時には再確認が必要です。
  • 再帰呼び出しを行っているため、ストリームツリーの深さが極端に深い場合のスタック消費に留意してください。
