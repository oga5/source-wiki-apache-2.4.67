ad_setup_order

定義

  • 種別: function
  • 定義箇所: modules/md/md_acmev2_drive.c:54

目的

ACMEv2 プロトコルにおける証明書更新のための注文（Order）を準備します。既存のステージング済み注文のロード、またはサーバーへの新規注文登録を行います。

シグネチャまたは形状

static apr_status_t ad_setup_order(md_proto_driver_t *d, md_result_t *result, int *pis_new)

入力と出力

  • **入力**:
  • d: プロトコルドライバのコンテキスト。
  • result: 処理結果およびアクティビティログを記録する構造体。
  • pis_new: 新規注文が作成された場合に 1 が設定される出力用ポインタ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時はエラーコード。

前提条件と不変条件

  • d->baton が md_acme_driver_t 型としてキャスト可能であること。
  • ad->md および ad->acme が初期化されていること（assert による検証）。

アルゴリズムまたは振る舞い

1. **既存注文のロード**: md_acme_order_load を呼び出し、MD_SG_STAGING から注文の復元を試みる。成功時は即座に終了。 2. **エラー処理**: ロード失敗時、ENOENT 以外であれば既存のステージングデータをパージする。 3. **ARI (ACME Renewal Information) 確認**: 設定に基づき、更新対象の証明書から ari_cert_id を抽出する。 4. **プロファイル検証**: 設定された ACME プロファイルがサーバー側で提供されているか確認する。必須プロファイルが欠如している場合は APR_EINVAL を返す。 5. **注文登録**: md_acme_order_register を呼び出し、サーバーへ新規注文を登録する。 6. **永続化**: 登録成功後、md_acme_order_save を使用してステージング領域に注文を保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acmev2_drive.c 内の md_acmev2_drive_renew 関数（176行目および194行目）。
  • **呼び出し先**: md_acme_order_load, md_acme_order_purge, md_acme_order_register, md_acme_order_save 等の md_acme_order.c が提供する API。

関連データ構造

  • md_proto_driver_t: プロトコルドライバの基底構造体。
  • md_acme_driver_t: ACME 固有のドライバ状態。
  • md_result_t: 処理結果の追跡用。

失敗モードと境界条件

  • **ロード失敗**: ステージング領域に注文が存在しない場合、新規登録フローへ移行する。
  • **プロファイル不一致**: ad->profile_mandatory が真で、サーバーが対応プロファイルを提示しない場合、エラーとして終了する。
  • **登録失敗**: サーバーとの通信エラーやバリデーションエラーが発生した場合、そのステータスを返す。

保守担当者向けメモ

  • 本関数は ACMEv2 の注文ライフサイクルの起点です。
  • ari_cert_id の処理は ACME Renewal Information に対応しており、証明書更新の最適化に寄与します。
  • 注文の再試行ロジックは、本関数を呼び出す上位関数側で制御されます。
