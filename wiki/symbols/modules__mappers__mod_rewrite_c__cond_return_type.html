cond_return_type

定義

  • 種別: enum
  • 定義箇所: modules/mappers/mod_rewrite.c:322

目的

RewriteCond（書き換え条件）の評価結果を表現し、条件の成否およびリクエスト処理のステータス変更を呼び出し元へ伝達するために使用されます。

シグネチャまたは形状

typedef enum {
  COND_RC_NOMATCH = 0,
  COND_RC_MATCH = 1,
  COND_RC_STATUS_SET = 3
} cond_return_type;

入力と出力

  • 本シンボル自体は列挙型であり、関数 apply_rewrite_cond の戻り値として使用されます。

前提条件と不変条件

  • COND_RC_NOMATCH (0): 条件が一致しなかったことを示します。
  • COND_RC_MATCH (1): 条件が一致したことを示します。
  • COND_RC_STATUS_SET (3): 条件評価の過程で r->status が最終的な値に設定されたことを示します。

アルゴリズムまたは振る舞い

  • apply_rewrite_cond 関数において、条件式の評価結果に基づき、上記いずれかの値を返却することで、後続のルール適用処理を制御します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/mappers/mod_rewrite.c 内の apply_rewrite_cond 関数がこの型を返します。
  • 呼び出し先: 情報なし。

関連データ構造

  • rewritecond_entry: 条件式を保持する構造体。
  • rewrite_ctx: ルール適用時のリクエスト状態を保持する構造体。

失敗モードと境界条件

  • COND_RC_STATUS_SET が返された場合、リクエスト処理が中断または完了している可能性があるため、呼び出し元は追加のルール適用を停止する必要があります。

保守担当者向けメモ

  • 本列挙型は mod_rewrite の条件評価ロジックの根幹に関わります。新しい条件評価結果を追加する場合は、既存の apply_rewrite_cond における戻り値のハンドリングをすべて更新してください。
