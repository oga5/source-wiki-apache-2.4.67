extract_cookie_line

定義

  • 種別: function
  • 定義箇所: server/util_cookies.c:177

目的

HTTP ヘッダーから特定のクッキー値を抽出し、必要に応じて対象のクッキーをヘッダーから削除（フィルタリング）した新しいヘッダー文字列を構築するコールバック関数です。

シグネチャまたは形状

static int extract_cookie_line(void *varg, const char *key, const char *val)

入力と出力

  • **入力**:
  • varg: ap_cookie_do 構造体へのポインタ（検索対象のクッキー名、リクエストコンテキスト、抽出結果を保持）。
  • key: ヘッダー名（例: Cookie）。
  • val: ヘッダーの値（クッキー文字列）。
  • **出力**:
  • 常に 1 を返します。
  • 副作用として、varg が指す構造体の encoded フィールドに抽出したクッキー値を格納し、new_cookies テーブルにフィルタリング後のヘッダーを書き込みます。

前提条件と不変条件

  • varg は ap_cookie_do 型としてキャスト可能であること。
  • 文字列操作には v->r->pool（リクエストプール）を使用するため、呼び出し元でメモリ管理が保証されていること。

アルゴリズムまたは振る舞い

1. apr_strtok を使用し、カンマ（,）およびセミコロン（;）を区切り文字としてヘッダー値をトークン化します。 2. 各トークンを走査し、指定されたクッキー名と一致するか確認します。 3. 一致した場合:

  • v->encoded に値を保存します。
  • 既に値が存在し、かつ今回抽出した値と異なる場合は v->duplicated を 1 に設定します。

    • 4. 一致しない場合:

  • $ で始まる属性（$Path 等）を除外した上で、残りのクッキーを new_cookie 文字列として再構築します。

    • 5. 処理終了後、残存するクッキーが存在すれば v->new_cookies テーブルに格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_cookies.c:256 において apr_table_do の引数として呼び出されます。
  • **呼び出し先**: apr_pstrdup, apr_pstrcat, apr_strtok, apr_isspace, apr_table_addn（すべて APR ライブラリ関数）。

関連データ構造

  • ap_cookie_do: クッキー抽出処理の状態を保持する構造体。
  • apr_table_t: ヘッダーの格納および再構築に使用されるテーブル構造。

失敗モードと境界条件

  • **重複クッキー**: 同一リクエスト内に同じ名前のクッキーが異なる値で存在する場合、v->duplicated が 1 に設定されます。
  • **メモリ不足**: apr_pstrdup 等のメモリ割り当て失敗時は、APR の規約に従いプロセスが終了する可能性があります。
  • **空のヘッダー**: 入力 val が空の場合、トークン化処理はスキップされ、何も抽出されません。

保守担当者向けメモ

  • 本関数は apr_table_do からのコールバックとして設計されています。
  • クッキーのパースロジックは RFC 2109/2965 に基づいていますが、$ で始まる属性の扱いに依存しているため、仕様変更時はこのフィルタリング条件を確認してください。
