check_string

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_cookie.c:203

目的

設定値（Cookie名など）が空ではなく、かつ禁止文字（= および &）を含んでいないことを検証します。

シグネチャまたは形状

static const char *check_string(cmd_parms *cmd, const char *string)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • string: 検証対象の文字列。
  • **出力**:
  • 検証に成功した場合: NULL を返します。
  • 検証に失敗した場合: エラーメッセージを含む文字列（apr_pstrcat で生成）を返します。

前提条件と不変条件

  • cmd->pool が有効である必要があります。
  • 検証対象の文字列に = または & が含まれる場合、設定は無効とみなされます。

アルゴリズムまたは振る舞い

1. string が NULL または空文字列であるかを確認します。 2. ap_strchr_c を使用して、文字列内に = または & が含まれているかを確認します。 3. 上記のいずれかの条件に該当する場合、cmd->directive->directive を含むエラーメッセージを生成して返します。 4. いずれの条件にも該当しない場合、NULL を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_form.c (設定ディレクティブの検証)
  • modules/session/mod_session_cookie.c (set_cookie_name, set_cookie_name2 等)
  • modules/session/mod_session_dbd.c (設定ディレクティブの検証)
  • **呼び出し先**:
  • ap_strchr_c (Apache API: 文字検索)
  • apr_pstrcat (APR API: 文字列連結)

関連データ構造

  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。

失敗モードと境界条件

  • 文字列が空または NULL の場合、エラーとなります。
  • = または & を含む文字列は、Cookie名として不適切なためエラーとなります。

保守担当者向けメモ

  • 本関数は mod_session_cookie だけでなく、mod_auth_form や mod_session_dbd といった他のモジュールからも汎用的な設定検証関数として利用されています。変更時は影響範囲に注意してください。
