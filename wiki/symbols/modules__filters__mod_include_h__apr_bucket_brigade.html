apr_bucket_brigade

定義

  • 種別: typedef
  • 定義箇所: modules/filters/mod_include.h:105

目的

Apache HTTP Server のフィルタチェーンにおいて、ストリームデータを保持・転送するためのバケットの連結リスト（ブリゲード）を定義します。

シグネチャまたは形状

typedef として定義されており、mod_include.h 内の include_handler_fn_t の引数として使用されます。

typedef apr_status_t (include_handler_fn_t)(include_ctx_t *, ap_filter_t *,
                                            apr_bucket_brigade *);

入力と出力

  • **入力**: フィルタ処理の過程で、入力ストリームから取得されたバケットや、新たに生成されたバケットがこの構造体に格納されます。
  • **出力**: フィルタ処理の結果として、改変されたバケット群が後続のフィルタへ渡されます。

前提条件と不変条件

  • apr_bucket_brigade は、Apache のフィルタフレームワークにおけるデータ転送の基本単位です。
  • フィルタ処理において、バケットの挿入、削除、または分割が行われる可能性があります。

アルゴリズムまたは振る舞い

SSI フィルタの実行時、登録されたハンドラは include_ctx_t を参照して条件分岐や変数展開を行い、その結果を apr_bucket_brigade に挿入します。その後、ap_pass_brigade を通じて後続のフィルタへデータが渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: include/http_core.h や include/http_protocol.h で定義されている各種フィルタ関数（ap_core_output_filter, ap_http_header_filter 等）が、引数として apr_bucket_brigade を受け取ります。
  • **呼び出し先**: 本シンボル自体はデータ構造の定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • include_ctx_t: SSI 処理の状態を保持する構造体。
  • ap_filter_t: フィルタチェーンのコンテキスト。
  • apr_bucket: ブリゲードを構成する個々のデータ要素。

失敗モードと境界条件

  • メモリ不足時や、バケットの読み込みエラー発生時に apr_status_t 型の戻り値を通じてエラーが伝播されます。
  • ストリームの終端（EOS）に達した際の処理が適切に行われない場合、フィルタチェーンの停止やデータの欠損が発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは Apache Portable Runtime (APR) の基盤構造体であり、mod_include 固有の定義ではありません。
  • mod_include.h 内での定義は、SSI フィルタのハンドラシグネチャを規定するために使用されています。
