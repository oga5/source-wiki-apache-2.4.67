ai_desc_t

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_autoindex.c:113

目的

ディレクトリ索引生成時に、特定のファイルやパターンに関連付けられた説明文（Description）を保持するために使用されます。

シグネチャまたは形状

typedef struct ai_desc_t {
    char *pattern;      /* マッチング対象のパターン */
    char *description;  /* 表示する説明文 */
    int full_path;      /* フルパスでマッチングを行うかのフラグ */
    int wildcards;      /* ワイルドカードを使用するかのフラグ */
} ai_desc_t;

入力と出力

  • 本構造体は、autoindex_config_rec 内の desc_list（apr_array_header_t）に格納される要素として機能します。
  • 索引生成処理において、ファイル名と pattern を照合し、一致した場合に description が出力されます。

前提条件と不変条件

  • desc_list は apr_array_make によって初期化され、ai_desc_t 型の要素を保持します。
  • pattern および description は、リクエストのプール（request_rec->pool）から割り当てられたメモリ領域を指すことが想定されます。

アルゴリズムまたは振る舞い

  • AddDescription ディレクティブによって設定された情報が本構造体に変換され、リストに追加されます。
  • 索引生成時、dcfg->desc_list から本構造体の配列を取得し、各エントリの pattern とファイル名を比較して説明文を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_autoindex.c: 設定解析時（add_desc 等の内部処理）および索引生成時のループ処理で参照されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • autoindex_config_rec: 本構造体のリスト（desc_list）を保持する親構造体。
  • ent: 索引生成時に各ファイル情報を保持する構造体。

失敗モードと境界条件

  • desc_list が空の場合、説明文は表示されません。
  • pattern のマッチングロジック（full_path や wildcards の設定）が不適切な場合、意図したファイルに説明文が割り当てられない可能性があります。

保守担当者向けメモ

  • 本構造体は AddDescription ディレクティブのデータ保持に特化しています。
  • apr_array_push を使用してリストに追加されるため、メモリ確保は Apache のプール管理に依存しています。メモリリークの懸念は低いです。
