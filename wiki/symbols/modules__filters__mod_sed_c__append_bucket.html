append_bucket

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_sed.c:103

目的

sed フィルタの処理結果をバケットブリゲードに追加し、必要に応じてブリゲードのフラッシュとメモリの解放を行う。

シグネチャまたは形状

static apr_status_t append_bucket(sed_filter_ctxt* ctx, char* buf, apr_size_t sz)

入力と出力

  • **入力**:
  • ctx: フィルタの実行コンテキスト (sed_filter_ctxt)。
  • buf: 追加するデータを含むバッファ。
  • sz: バッファのサイズ。
  • **出力**:
  • apr_status_t: 操作の成功状態 (APR_SUCCESS または ap_pass_brigade の戻り値)。

前提条件と不変条件

  • ctx->bb (バケットブリゲード) が初期化されていること。
  • ctx->r->connection->bucket_alloc が有効であること。

アルゴリズムまたは振る舞い

1. **バケットの生成**:

  • ctx->tpool がリクエストプールと一致する場合、apr_bucket_pool_create を使用してプールバケットを作成する。
  • それ以外の場合、apr_bucket_transient_create を使用して一時バケットを作成し、ctx->numbuckets をインクリメントする。

    • 2. **ブリゲードへの挿入**: 生成したバケットを ctx->bb の末尾に挿入する。 3. **フラッシュ判定**: 一時バケット数が MAX_TRANSIENT_BUCKETS に達した場合:

  • フラッシュバケットを作成し、ブリゲードの末尾に挿入する。
  • ap_pass_brigade を呼び出してブリゲードを後続フィルタへ渡す。
  • apr_brigade_cleanup でブリゲードを空にし、clear_ctxpool(ctx) でコンテキストのプールを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_sed.c:142
  • modules/filters/mod_sed.c:170
  • modules/filters/mod_sed.c:176
  • ※ modules/http2/h2_bucket_beam.c:552 に同名の関数呼び出しが見られるが、本モジュール内の関数との直接的な関係は不明。
  • **呼び出し先**:
  • apr_bucket_pool_create / apr_bucket_transient_create
  • APR_BRIGADE_INSERT_TAIL
  • ap_pass_brigade
  • apr_brigade_cleanup
  • clear_ctxpool

関連データ構造

  • sed_filter_ctxt: フィルタの状態管理構造体。
  • apr_bucket_brigade: バケットの連結リスト。
  • apr_bucket: Apache のデータ転送単位。

失敗モードと境界条件

  • ap_pass_brigade がエラーを返した場合、そのステータスがそのまま呼び出し元に返される。
  • MAX_TRANSIENT_BUCKETS を超えた際のフラッシュ処理により、メモリ使用量が抑制される。

保守担当者向けメモ

  • 本関数は sed_filter_ctxt 内のプール管理と密接に関連している。clear_ctxpool を呼び出すタイミングは、メモリリークを防ぐために重要である。
  • modules/http2/h2_bucket_beam.c に存在する同名の関数は、本モジュールのものとは別個の実装である可能性が高い。
