set_cookie_form_location

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:205

目的

Apache HTTP Server の設定ディレクティブ AuthFormLocation を処理し、認証後のリダイレクト先 URL を設定構造体に格納します。

シグネチャまたは形状

static const char *set_cookie_form_location(cmd_parms *cmd, void *config, const char *location)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: 現在のディレクトリ設定構造体 (auth_form_config_rec)。
  • location: 設定ファイルから読み込まれたリダイレクト先の文字列。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は check_string が返すエラーメッセージ文字列を返します。

前提条件と不変条件

  • config 引数は auth_form_config_rec 型としてキャスト可能である必要があります。
  • conf->location_set は、設定が明示的に行われたことを示すフラグとして 1 に設定されます。

アルゴリズムまたは振る舞い

1. 引数 config を auth_form_config_rec 構造体へキャストします。 2. 構造体の location メンバに引数 location を代入します。 3. 構造体の location_set メンバを 1 に設定します。 4. check_string 関数を呼び出し、設定値の妥当性を検証してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c:360 において、AP_INIT_TAKE1 マクロを通じて AuthFormLocation ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: check_string（設定値の検証用関数）。

関連データ構造

  • auth_form_config_rec: ディレクトリごとの認証設定を保持する構造体。location および location_set フィールドを含みます。

失敗モードと境界条件

  • check_string がエラーを返した場合、そのエラーメッセージがそのまま呼び出し元に伝播されます。

保守担当者向けメモ

  • 本関数は AuthFormLocation ディレクティブ専用のセッターです。設定値の検証ロジックを変更する場合は check_string を参照してください。
