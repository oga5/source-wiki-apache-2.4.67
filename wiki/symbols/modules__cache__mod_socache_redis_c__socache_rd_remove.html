socache_rd_remove

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_redis.c:277

目的

Redis バックエンドに保存された特定のキャッシュエントリを削除します。

シグネチャまたは形状

apr_status_t socache_rd_remove(ap_socache_instance_t *ctx, server_rec *s,
                               const unsigned char *id,
                               unsigned int idlen, apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: Redis インスタンスのコンテキスト。
  • s: サーバーレコード。
  • id: 削除対象のキャッシュ ID（バイナリデータ）。
  • idlen: id の長さ。
  • p: メモリプール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返します。

前提条件と不変条件

  • socache_rd_id2key が正常にキーを生成できる必要があります。
  • ctx->rc（Redis 接続ハンドル）が有効である必要があります。

アルゴリズムまたは振る舞い

1. socache_rd_id2key を呼び出し、内部 ID を Redis 用のキー文字列（buf）に変換します。変換に失敗した場合は APR_EINVAL を返します。 2. apr_redis_delete を使用して、Redis 上の該当キーを削除します。 3. 削除処理が APR_SUCCESS 以外を返した場合、ap_log_error を用いてデバッグレベルのログを出力します。 4. 最終的なステータスコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_redis.c:395（ap_socache_provider_t 構造体への登録を通じて呼び出されます）。
  • **呼び出し先**:
  • socache_rd_id2key: キー変換。
  • apr_redis_delete: Redis サーバー上のデータ削除。
  • ap_log_error: エラーログ出力。

関連データ構造

  • ap_socache_instance_t: Redis インスタンスのコンテキストを保持。
  • server_rec: Apache のサーバー設定情報。

失敗モードと境界条件

  • socache_rd_id2key が失敗した場合、APR_EINVAL を返します。
  • Redis サーバーとの通信エラーやキーが存在しない場合、apr_redis_delete がエラーを返し、その結果が呼び出し元に伝播されます。

保守担当者向けメモ

  • 本関数は ap_socache_provider_t インターフェースの一部として実装されています。
  • apr_redis_delete の第3引数に 0 を指定しており、即時削除を意図しています。
