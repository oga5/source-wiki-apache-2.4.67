readslot_heartbeats

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_heartbeat.c:235

目的

mod_heartmonitor が管理する共有メモリから、全バックエンドサーバーのハートビート情報を読み込み、コンテキスト構造体へ反映させる。

シグネチャまたは形状

static apr_status_t readslot_heartbeats(ctx_servers_t *ctx, apr_pool_t *pool)

入力と出力

  • **入力**:
  • ctx: サーバー情報を格納するための ctx_servers_t 構造体へのポインタ。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • 戻り値: APR_SUCCESS を返す。

前提条件と不変条件

  • storage プロバイダーが初期化され、hm_serversmem（共有メモリ）が有効であること。
  • hm_read 関数が slotmem の各スロットを処理するコールバックとして適切に機能すること。

アルゴリズムまたは振る舞い

1. storage->doall を呼び出し、共有メモリ hm_serversmem 内の全スロットを走査する。 2. 各スロットに対して hm_read コールバックを実行し、情報を ctx に抽出する。 3. 処理完了後、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/balancers/mod_lbmethod_heartbeat.c:251 （read_heartbeats 関数内から呼び出される）。
  • **呼び出し先**:
  • storage->doall (共有メモリ走査プロバイダー)
  • hm_read (各スロットのデータ抽出用コールバック)

関連データ構造

  • ctx_servers_t: サーバー状態を保持するコンテキスト。
  • hm_serversmem: ハートビート情報が格納されている共有メモリ領域。

失敗モードと境界条件

  • 共有メモリがアタッチされていない場合や、storage プロバイダーが未定義の場合は実行時にエラーとなる可能性がある（ソースコード上では明示的なエラーハンドリングは記述されていない）。

保守担当者向けメモ

  • 本関数は mod_heartmonitor とのインターフェース層である。共有メモリのレイアウト変更や hm_read のロジック変更時には、本関数の呼び出し整合性に注意すること。
