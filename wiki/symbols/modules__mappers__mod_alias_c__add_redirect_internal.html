add_redirect_internal

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_alias.c:246

目的

Redirect 関連ディレクティブ（Redirect, RedirectMatch, RedirectTemp, RedirectPermanent 等）の引数を解析し、リダイレクト設定を内部構造体へ登録する。

シグネチャまたは形状

static const char *add_redirect_internal(cmd_parms *cmd,
                                         alias_dir_conf *dirconf,
                                         const char *arg1, const char *arg2,
                                         const char *arg3, int use_regex)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • dirconf: ディレクトリ単位の設定構造体。
  • arg1, arg2, arg3: ディレクティブの引数文字列。
  • use_regex: 正規表現を使用するかどうかのフラグ。
  • **出力**:
  • 成功時は NULL を返す。
  • エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • arg1 が HTTP ステータスコード（数値）またはキーワード（permanent, temp, seeother, gone）として解釈可能である必要がある。
  • cmd->path が設定されている場合、設定は dirconf に格納される。それ以外の場合は alias_server_conf に格納される。

アルゴリズムまたは振る舞い

1. **ステータス解析**: arg1 を解析し、HTTP ステータスコードを決定する。 2. **式評価**: Location コンテキスト内であれば、ap_expr_parse_cmd を使用してリダイレクト先を式として解析・保持する。 3. **引数配置**: arg3 が存在しない場合、arg1 と arg2 をそれぞれ fake（対象パス）と url（リダイレクト先）として再配置する。 4. **正規表現コンパイル**: use_regex が真の場合、ap_pregcomp で fake をコンパイルする。 5. **バリデーション**: リダイレクト先が URL として妥当か、またはパスとして適切かを検証する。 6. **登録**: 解析結果を alias_entry 構造体に格納し、対応する設定配列（dirconf->redirects または serverconf->redirects）に追加する。

関連データ構造

  • alias_entry: リダイレクト定義を保持する構造体。
  • alias_dir_conf: ディレクトリごとの設定構造体。
  • alias_server_conf: サーバー全体の設定構造体。
  • cmd_parms: コマンド実行時のコンテキスト情報。

失敗モードと境界条件

  • arg1 が不明なステータスコードである場合、エラーを返す。
  • arg3 が指定されているにもかかわらず arg1 がステータスとして解釈できない場合、エラーを返す。
  • 正規表現のコンパイルに失敗した場合、エラーを返す。
  • リダイレクト先が URL でもなく / で始まるパスでもない場合、エラーを返す。
  • ステータスがリダイレクト（3xx）ではない場合に url が指定されているとエラーを返す。

保守担当者向けメモ

  • Location コンテキスト内でのリダイレクト設定は、dirconf を介して式として処理される。
  • Redirect ディレクティブの引数順序の柔軟性を維持するため、引数の再配置ロジックが複雑になっている。変更時は arg1〜arg3 の組み合わせパターンに注意すること。
