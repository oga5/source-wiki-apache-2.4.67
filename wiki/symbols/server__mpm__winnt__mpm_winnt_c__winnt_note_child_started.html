winnt_note_child_started

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:139

目的

子プロセスの起動情報をスコアボードに記録し、子プロセスの状態変化を監視フックへ通知する。

シグネチャまたは形状

void winnt_note_child_started(int slot, pid_t pid)

入力と出力

  • **入力**:
  • slot: スコアボード内の子プロセス用スロット番号。
  • pid: 起動した子プロセスのプロセスID。
  • **出力**: なし（副作用としてスコアボードの更新とフックの実行を行う）。

前提条件と不変条件

  • ap_scoreboard_image が初期化済みであること。
  • my_generation が現在のプロセス世代を正しく保持していること。

アルゴリズムまたは振る舞い

1. 指定された slot に対応するスコアボードの parent エントリへ、引数 pid を格納する。 2. 同エントリの generation に現在の my_generation を設定する。 3. ap_run_child_status を呼び出し、MPM_CHILD_STARTED ステータスを通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/mpm_winnt.c:792
  • **呼び出し先**: ap_run_child_status（Apacheのフック関数）

関連データ構造

  • ap_scoreboard_image: プロセス間共有メモリ上のスコアボード構造体。
  • ap_server_conf: サーバー設定情報。

失敗モードと境界条件

  • slot がスコアボードの有効範囲外である場合、メモリ破壊を引き起こす可能性がある（本関数内での境界チェックは行われていない）。

保守担当者向けメモ

  • 本関数は子プロセスの生成直後に呼び出されることを想定している。
  • my_generation はグローバル変数として管理されているため、マルチスレッド環境下での整合性に注意すること。
