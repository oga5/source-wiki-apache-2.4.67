md_get_ca_name_from_url

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:427

目的

指定された認証局（CA）の URL から、対応する CA 名を解決します。

シグネチャまたは形状

const char *md_get_ca_name_from_url(apr_pool_t *p, const char *url)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • url: 解決対象の CA URL 文字列。
  • **出力**:
  • 解決された CA 名（文字列）。既知の CA に一致しない場合は、URL から抽出されたホスト名、または入力 URL そのものが返されます。

前提条件と不変条件

  • KNOWN_CAs 配列がモジュール内で定義されており、URL と名称のペアを保持している必要があります。
  • apr_pool_t は有効なメモリプールである必要があります。

アルゴリズムまたは振る舞い

1. KNOWN_CAs 配列を走査し、入力 url と一致する URL を持つ CA を検索します。一致した場合はその name を返します。 2. 一致するものがない場合、apr_uri_parse を使用して URL を解析します。 3. 解析が成功した場合、URL のホスト名部分を返します。 4. 解析に失敗した場合、入力された url を apr_pstrdup で複製して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_status.c:206
  • modules/md/mod_md_status.c:359
  • **呼び出し先**:
  • apr_cstr_casecmp (APR)
  • apr_uri_parse (APR)
  • apr_pstrdup (APR)

関連データ構造

  • KNOWN_CAs: CA の URL と名称を保持する静的配列（ソースコード内で参照）。
  • apr_uri_t: URL 解析結果を格納する APR 構造体。

失敗モードと境界条件

  • url が不正な形式の場合、apr_uri_parse が失敗し、入力文字列そのものが返されます。
  • KNOWN_CAs に含まれない URL の場合、ホスト名が CA 名として代替されます。

保守担当者向けメモ

  • 本関数は KNOWN_CAs 配列の定義に依存しています。新しい CA を追加する場合は、該当配列の更新が必要です。
  • apr_pool_t を使用しているため、戻り値の文字列はプールが破棄されるまで有効です。
