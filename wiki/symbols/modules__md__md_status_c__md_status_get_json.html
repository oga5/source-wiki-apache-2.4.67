md_status_get_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_status.c:321

目的

管理対象ドメイン（Managed Domain）のリストに基づき、全ドメインのステータス情報を集約した JSON オブジェクトを生成する。

シグネチャまたは形状

apr_status_t md_status_get_json(md_json_t **pjson, apr_array_header_t *mds, 
                                md_reg_t *reg, md_ocsp_reg_t *ocsp, apr_pool_t *p)

入力と出力

  • **入力**:
  • mds: ステータスを取得する対象ドメインの配列 (apr_array_header_t *)。
  • reg: ドメインレジストリ (md_reg_t *)。
  • ocsp: OCSP レジストリ (md_ocsp_reg_t *)。
  • p: メモリ割り当てに使用するプール (apr_pool_t *)。
  • **出力**:
  • pjson: 生成された JSON オブジェクトへのポインタ (md_json_t **)。
  • 戻り値: 常に APR_SUCCESS を返す。

前提条件と不変条件

  • mds 配列内の各要素は const md_t * 型である必要がある。
  • p プールは、生成される JSON オブジェクトの生存期間をカバーする十分な寿命を持つこと。

アルゴリズムまたは振る舞い

1. md_json_create(p) を呼び出し、ルートとなる JSON オブジェクトを作成する。 2. MOD_MD_VERSION を用いて、JSON 内にバージョン情報を設定する。 3. mds 配列を反復処理し、各ドメインに対して status_get_md_json を呼び出して個別のステータス JSON (mdj) を取得する。 4. 取得した mdj をルート JSON の MD_KEY_MDS キー配下に追加する。 5. 最終的に構築された JSON オブジェクトを *pjson に格納して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_status.c:786
  • modules/md/mod_md_status.c:971
  • **呼び出し先**:
  • status_get_md_json (内部ヘルパー)
  • md_json_create
  • md_json_sets
  • md_json_addj

関連データ構造

  • md_json_t: JSON データの保持に使用。
  • md_t: 管理対象ドメインの定義。
  • md_reg_t: ドメイン登録情報の管理。
  • md_ocsp_reg_t: OCSP ステータスの管理。

失敗モードと境界条件

  • mds が空の場合、MD_KEY_MDS キーには空のリストが設定される。
  • メモリ不足が発生した場合、md_json_create 等の内部関数が APR のエラーを返す可能性があるが、本関数自体は APR_SUCCESS を返す設計となっている。

保守担当者向けメモ

  • 本関数は mod_md_status.c を通じて HTTP レスポンスとして公開されるステータス情報の主要なエントリポイントである。
  • MD_KEY_MDS への追加ロジックは、各ドメインのステータスを階層的に管理する構造となっている。
