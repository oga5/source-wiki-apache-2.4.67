ap_request_insert_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:293

目的

リクエストの現在の状態（ボディの保持状況）に基づき、適切な入力フィルタ（kept_body_input_filter_handle または keep_body_input_filter_handle）をフィルタチェーンへ挿入します。

シグネチャまたは形状

static void ap_request_insert_filter(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト構造体）
  • **出力**: なし（副作用としてフィルタチェーンを更新）

前提条件と不変条件

  • r->per_dir_config から request_module の設定を取得可能であること。
  • kept_body_input_filter_handle および keep_body_input_filter_handle が初期化済みであること。

アルゴリズムまたは振る舞い

1. r->per_dir_config からディレクトリ設定 request_dir_conf を取得します。 2. r->kept_body が存在する場合:

  • kept_body_input_filter_handle が既に存在しないことを確認し、存在しなければ挿入します。

    • 3. r->kept_body が存在せず、かつ設定 conf->keep_body が有効な場合:

  • keep_body_input_filter_handle が既に存在しないことを確認し、存在しなければ挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_request.c 内で ap_hook_insert_filter に登録されており、Apache のフィルタ挿入フェーズで呼び出されます。
  • modules/aaa/mod_auth_form.c からオプション関数として取得・利用されている可能性があります（APR_RETRIEVE_OPTIONAL_FN）。
  • **呼び出し先**:
  • ap_get_module_config
  • request_is_filter_present
  • ap_add_input_filter_handle

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • request_dir_conf: mod_request のディレクトリ単位の設定構造体。

失敗モードと境界条件

  • フィルタが既にチェーンに存在する場合、重複挿入は行われません。
  • r->kept_body が NULL かつ conf->keep_body が無効な場合、何も行われません。

保守担当者向けメモ

  • 本関数は ap_hook_insert_filter を通じてフックされています。
  • APR_REGISTER_OPTIONAL_FN により、他のモジュールからオプション関数として利用可能です。
