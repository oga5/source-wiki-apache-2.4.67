subst_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_substitute.c:57

目的

mod_substitute モジュールにおけるディレクトリごとの設定情報を保持します。置換パターン、行長制限、および設定の継承ルールを管理します。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *patterns;
    apr_size_t max_line_length;
    int max_line_length_set;
    int inherit_before;
} subst_dir_conf;

入力と出力

本構造体は設定データとして機能します。

  • **入力**: Substitute ディレクティブ等による設定値の格納。
  • **出力**: フィルタ実行時に ap_get_module_config を介して取得され、置換処理のパラメータとして利用されます。

前提条件と不変条件

  • patterns は subst_pattern_t 型の要素を保持する配列である必要があります。
  • max_line_length はメモリ使用量を制限するための閾値として機能します。

アルゴリズムまたは振る舞い

  • patterns 配列に格納された置換ルールを順次適用します。
  • max_line_length_set は、ユーザーが明示的に SubstituteMaxLineLength を設定したかどうかを判定するために使用されます。
  • inherit_before は、親コンテキストの設定と現在の設定の適用順序を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_substitute_dcfg (メモリ確保)
  • merge_substitute_dcfg (設定の統合)
  • substitute_filter (設定の取得)
  • set_pattern (パターンの追加)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • subst_pattern_t: patterns 配列内に格納される個別の置換ルール。
  • apr_array_header_t: パターンリストを管理するための APR 配列構造。

失敗モードと境界条件

  • max_line_length を超える行が入力された場合、mod_substitute は APR_ENOMEM エラーを発生させ、メモリ保護を強制します。

保守担当者向けメモ

  • 本構造体は per_dir_config に格納されるため、設定の継承（merge_substitute_dcfg）時には inherit_before の値に基づいた適切な順序で patterns をマージする必要があります。
