session_cookie_load

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_cookie.c:109

目的

HTTPリクエストからCookieを読み取り、セッションデータ（session_rec）を復元して提供します。

シグネチャまたは形状

apr_status_t session_cookie_load(request_rec *r, session_rec **z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • z: セッションデータを格納するためのポインタのポインタ。
  • **出力**:
  • apr_status_t: 成功時は OK、Cookie名が未設定の場合は DECLINED を返します。
  • *z: 復元された session_rec 構造体へのポインタ。

前提条件と不変条件

  • session_cookie_dir_conf から取得した設定に基づき、Cookie名が定義されている必要があります。
  • 復元されたセッションは、リクエストのメインリクエスト（m->pool）のメモリプールに紐付きます。
  • 同一リクエスト内での重複解析を避けるため、解析済みのセッションは m->notes テーブルにキャッシュされます。

アルゴリズムまたは振る舞い

1. **リクエストの特定**: リクエストチェーンを遡り、メインリクエスト（m）を特定します。 2. **キャッシュ確認**: m->notes 内に該当するセッションが存在するか確認し、あればそれを返します。 3. **Cookieの読み込み**: ap_cookie_read を使用して、設定された名前のCookieからデータを抽出します。 4. **セッション生成**: session_rec 構造体をメモリ確保し、抽出したデータを encoded フィールドに格納します。 5. **キャッシュ登録**: 生成したセッションを m->notes に保存し、後続の呼び出しで再利用可能にします。 6. **ヘッダー設定**: キャッシュを防ぐため、Cache-Control: no-cache, private をレスポンスヘッダーに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_session の session_load フック経由で呼び出されます（modules/session/mod_session_cookie.c:270 にて登録）。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core)
  • ap_cookie_read (util_cookies.h)
  • apr_pcalloc, apr_table_make, apr_table_get, apr_table_setn, apr_table_addn (APR)

関連データ構造

  • session_rec: セッション情報を保持する主要構造体。
  • session_cookie_dir_conf: Cookie名や削除フラグを保持するディレクトリ単位の設定構造体。
  • request_rec: リクエストのライフサイクルとメモリプールを管理。

失敗モードと境界条件

  • **Cookie名未設定**: conf->name および conf->name2 が設定されていない場合、DECLINED を返し処理を終了します。
  • **Cookie不在**: ap_cookie_read がデータを取得できない場合、val は NULL となり、空のセッションが生成されます。
  • **リクエストチェーン**: サブクエストやリダイレクトが発生している場合でも、常にメインリクエストのプールとノートを利用して整合性を保ちます。

保守担当者向けメモ

  • 本関数はセッションの「運び屋」であり、実際のデータデシリアライズ（JSON解析等）は行いません。
  • notes テーブルへの格納には MOD_SESSION_COOKIE プレフィックスを使用しており、他モジュールとのキー衝突を回避しています。
  • メモリ管理はすべてリクエストプールに依存しているため、明示的な解放は不要です。
