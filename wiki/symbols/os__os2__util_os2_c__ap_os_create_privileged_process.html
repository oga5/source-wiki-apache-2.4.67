ap_os_create_privileged_process

定義

  • 種別: function
  • 定義箇所: os/os2/util_os2.c:31

目的

OS/2 プラットフォームにおいて、Apache HTTP Server のプロセス管理モデルと整合性を保ちつつ、特権プロセスを生成するための抽象化レイヤーを提供します。

シグネチャまたは形状

AP_DECLARE(apr_status_t) ap_os_create_privileged_process(
    const request_rec *r,
    apr_proc_t *newproc, 
    const char *progname,
    const char * const *args,
    const char * const *env,
    apr_procattr_t *attr, 
    apr_pool_t *p)

入力と出力

  • **入力**:
  • r: リクエスト情報（request_rec）
  • newproc: 生成されるプロセスを格納する構造体（apr_proc_t）
  • progname: 実行するプログラム名
  • args: 引数リスト
  • env: 環境変数リスト
  • attr: プロセス属性（apr_procattr_t）
  • p: メモリプール（apr_pool_t）
  • **出力**:
  • apr_status_t: プロセス生成の成功または失敗を示すステータスコード。

前提条件と不変条件

  • APR（Apache Portable Runtime）のプロセス管理機能が利用可能であること。
  • 引数として渡される各ポインタは有効であること。

アルゴリズムまたは振る舞い

1. 受け取った引数（newproc, progname, args, env, attr, p）をそのまま apr_proc_create 関数へ転送します。 2. apr_proc_create の実行結果（apr_status_t）をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:425
  • modules/generators/mod_cgid.c:953
  • **呼び出し先**:
  • apr_proc_create（APR ライブラリ関数）

関連データ構造

  • apr_proc_t: プロセス管理用構造体。
  • apr_procattr_t: プロセス生成時の属性設定用構造体。
  • request_rec: HTTP リクエスト情報。

失敗モードと境界条件

  • apr_proc_create が失敗した場合、そのエラーコードがそのまま返されます。
  • OS/2 特有の権限管理が必要な場合、将来的に実装が拡張される可能性がありますが、現時点では APR の標準的なエラー処理に依存しています。

保守担当者向けメモ

  • 本関数は現在 apr_proc_create の単純なラッパーです。OS/2 ネイティブな DosExecPgm 等の直接的な制御が必要になった場合は、この関数内で実装を差し替えることが想定されています。
  • OS/2 API を直接利用する場合は、INCL_DOS 等の定義が必須となります。
