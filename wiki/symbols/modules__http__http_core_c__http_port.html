http_port

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:126

目的

リクエストのサーバー設定に基づき、デフォルトのポート番号（HTTP または HTTPS）を決定して返します。

シグネチャまたは形状

apr_port_t http_port(const request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト構造体）
  • **出力**: apr_port_t（ポート番号。HTTPS の場合は DEFAULT_HTTPS_PORT、それ以外は DEFAULT_HTTP_PORT）

前提条件と不変条件

  • r および r->server が有効なポインタであることを前提とします。
  • r->server->server_scheme が設定されている場合、その文字列比較によってポートが決定されます。

アルゴリズムまたは振る舞い

1. r->server->server_scheme が "https" と一致するかを確認します。 2. 一致する場合、DEFAULT_HTTPS_PORT を返します。 3. 一致しない場合、DEFAULT_HTTP_PORT を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_core.c:298 にて ap_hook_default_port に登録されています。
  • **呼び出し先**: strcmp（標準ライブラリ関数）を呼び出します。
  • **注記**: テストコード（test/modules/http1/ および test/modules/http2/ 配下の各ファイル）において env.http_port として参照されていますが、これらはテスト環境のプロパティを指しており、本関数を直接呼び出しているかどうかの直接的な証拠はソースコードからは確認できません。

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • server_rec: r->server を介してアクセスされるサーバー設定構造体。

失敗モードと境界条件

  • r->server->server_scheme が NULL の場合、strcmp での比較は行われず、デフォルトの DEFAULT_HTTP_PORT が返されます。

保守担当者向けメモ

  • 本関数は ap_hook_default_port を通じてサーバーのデフォルトポート解決ロジックとして利用されます。
  • 新しいスキームのサポートを追加する場合は、条件分岐を拡張する必要があります。
