parse_addr

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:168

目的

ホスト名または IP アドレス文字列を解析し、ネットワークバイトオーダーの IPv4 アドレス（unsigned long）を返します。また、文字列内にポート番号が含まれる場合はそれを抽出します。

シグネチャまたは形状

unsigned long parse_addr(const char *w, unsigned short *ports)

入力と出力

  • **入力**:
  • w: 解析対象の文字列（ホスト名、IP アドレス、または host:port 形式）。
  • ports: ポート番号を格納する unsigned short へのポインタ（NULL 指定可）。
  • **出力**:
  • 成功時: ネットワークバイトオーダーの IPv4 アドレス。
  • 失敗時: 内部で exit(1) を呼び出しプロセスを終了させる。

前提条件と不変条件

  • w が * の場合、INADDR_ANY を返します。
  • ports が NULL でない場合、文字列内の : 以降の数値がポート番号として設定されます。
  • ホスト名解決が必要な場合、gethostbyname を使用します。

アルゴリズムまたは振る舞い

1. 文字列 w から : を検索し、ポート番号の指定があれば ports に格納します。 2. 文字列が * の場合は INADDR_ANY を返します。 3. apr_inet_addr を用いて IP アドレス形式の変換を試みます。成功すればその値を返します。 4. 上記で解決できない場合、gethostbyname でホスト名を解決します。 5. 解決結果が単一の IPv4 アドレスであることを確認し、そのアドレスを返します。複数のアドレスを持つホスト名が指定された場合はエラーを出力して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_nw_ssl.c:506
  • **呼び出し先**:
  • 標準ライブラリ: strchr, atoi, strcmp, htonl, fprintf, exit, gethostbyname
  • APR: apr_inet_addr

関連データ構造

  • struct hostent: ホスト名解決結果を保持。
  • struct in_addr: IPv4 アドレス構造体。

失敗モードと境界条件

  • **名前解決失敗**: gethostbyname が失敗した場合、エラーメッセージを出力して exit(1) を実行します。
  • **複数アドレス**: 指定したホスト名が複数の IP アドレスに解決される場合、曖昧さを避けるためエラーを出力して exit(1) を実行します。
  • **ポート指定**: host:* のような形式でポートが指定された場合、ポート番号は 0 として扱われます。

保守担当者向けメモ

  • 本関数は exit(1) を呼び出すため、サーバー起動時の設定フェーズでのみ使用することを想定しています。
  • gethostbyname は古い API であるため、将来的に apr_sockaddr_info_get 等の APR API への移行が推奨されます。
  • 失敗時のエラーログ出力に ap_log_error を使用する改善の余地があります。
