file_cache_xlat

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_file_cache.c:239

目的

リクエストされたファイルパスがキャッシュ対象であるかを判定し、キャッシュされている場合はそのメタデータ（a_file）をリクエストコンテキストに注入して、後続の処理（stat 等）をスキップさせるための translate_name フック関数です。

シグネチャまたは形状

static int file_cache_xlat(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）
  • **出力**: int
  • OK: キャッシュヒット時。
  • DECLINED: キャッシュ対象外、またはハッシュテーブルが空の場合。
  • ap_core_translate の戻り値: パス解決に失敗した場合など。

前提条件と不変条件

  • sconf->fileht（キャッシュハッシュテーブル）が初期化されていること。
  • ap_core_translate(r) が正常に完了し、r->filename が設定されていること。

アルゴリズムまたは振る舞い

1. サーバー設定からキャッシュハッシュテーブルを取得し、エントリが存在しない場合は DECLINED を返す。 2. ap_core_translate(r) を呼び出し、リクエストされたパスの物理的な解決を行う。解決に失敗した場合はその結果を返す。 3. 解決された r->filename をキーとしてハッシュテーブルを検索する。 4. ヒットした場合:

  • ap_set_module_config を使用して、該当する a_file 構造体を r->request_config に保存する。
  • r->finfo をキャッシュ済みの match->finfo で上書きし、後続の stat() 呼び出しを回避する。
  • OK を返す。

    • 5. ヒットしない場合は DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の translate_name フック（ap_hook_translate_name により登録）。
  • **呼び出し先**:
  • ap_get_module_config
  • apr_hash_count
  • ap_core_translate
  • apr_hash_get
  • ap_set_module_config

関連データ構造

  • a_server_config: サーバーごとのキャッシュハッシュテーブルを保持。
  • a_file: キャッシュされたファイルの属性（finfo）やハンドルを保持。
  • request_rec: リクエストのコンテキストおよびファイル情報を保持。

失敗モードと境界条件

  • **キャッシュ未登録**: ハッシュテーブルが空の場合、即座に DECLINED を返し、通常の処理フローに委ねる。
  • **パス解決失敗**: ap_core_translate が OK 以外を返した場合、そのエラー状態を伝播させる。
  • **キャッシュミス**: ハッシュテーブルに該当キーが存在しない場合、DECLINED を返し、通常のファイルシステムアクセスへ移行する。

保守担当者向けメモ

  • 本関数は ap_core_translate の後に実行されるため、Alias や RewriteRule 等によるパス変換が適用された後のパスに対してキャッシュ検索が行われます。
  • r->finfo を上書きするため、キャッシュされたファイルがディスク上で更新された場合、サーバー再起動まで古いメタデータが使用され続ける点に注意してください。
