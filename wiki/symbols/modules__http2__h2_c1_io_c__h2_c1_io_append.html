h2_c1_io_append

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:392

目的

apr_bucket_brigade を受け取り、メタデータや非バッファ対象バケットを適切に処理した上で、HTTP/2 セッションの出力キュー（io->output）へ追加します。

シグネチャまたは形状

apr_status_t h2_c1_io_append(h2_c1_io *io, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • io: 操作対象の h2_c1_io 構造体。
  • bb: 追加対象のバケットを含む apr_bucket_brigade。
  • **出力**:
  • 戻り値: 処理成功時は APR_SUCCESS、失敗時はエラーコードを返します。

前提条件と不変条件

  • bb が空でない限り、ループ内でバケットを順次処理します。
  • メタデータや MMAP バケットは、順序を維持するために既存のスクラッチバッファをフラッシュ（append_scratch）してから出力キューへ移動させます。

アルゴリズムまたは振る舞い

1. bb が空になるまで以下の処理を繰り返します。 2. **メタデータ/MMAP**: append_scratch を呼び出し、現在のスクラッチバッファを確定させた後、対象バケットを出力キューへ移動します。 3. **バッファリング有効時**:

  • assure_scratch_space で空き容量を確保します。
  • バケットサイズが空き容量を超える場合、バケットを分割し、残余分をそのまま出力キューへ追加します。
  • バケットが収まる場合は read_to_scratch でコピーし、元のバケットを削除します。

    • 4. **バッファリング無効時**: バケットをセッションのプールに setaside し、出力キューへ移動します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:566
  • modules/http2/h2_session.c:1690
  • **呼び出し先**:
  • append_scratch
  • assure_scratch_space
  • read_to_scratch
  • apr_bucket_setaside
  • APR_BUCKET_REMOVE, APR_BRIGADE_INSERT_TAIL 等の APR マクロ

関連データ構造

  • h2_c1_io: 入出力状態を保持する構造体。
  • apr_bucket_brigade: バケットの連結リスト。

失敗モードと境界条件

  • read_to_scratch の実行中にエラーが発生した場合、cleanup ラベルへジャンプし、その時点でのエラーコードを返します。

保守担当者向けメモ

  • 本関数は h2_session の出力フローにおける重要な仲介役です。
  • io->buffer_output のフラグ状態によって、データのコピー（スクラッチ）か直接転送かが分岐するため、パフォーマンスチューニング時はこの条件分岐に注意してください。
