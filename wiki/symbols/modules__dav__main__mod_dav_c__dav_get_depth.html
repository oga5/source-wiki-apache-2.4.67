dav_get_depth

定義

  • 種別: function
  • 定義箇所: modules/dav/main/mod_dav.c:711

目的

HTTP リクエストヘッダー Depth を解析し、WebDAV 操作における再帰の深さを整数値として取得します。

シグネチャまたは形状

int dav_get_depth(request_rec *r, int def_depth)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • def_depth: Depth ヘッダーが存在しない場合に返されるデフォルト値。
  • **出力**:
  • Depth ヘッダーの値（0, 1, または DAV_INFINITY）。
  • ヘッダーが未指定の場合は def_depth を返します。
  • 不正な値の場合は -1 を返します。

前提条件と不変条件

  • r->headers_in テーブルから Depth ヘッダーを読み取ります。
  • Depth の値は 0, 1, infinity (大文字小文字を区別しない) のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. apr_table_get を使用して Depth ヘッダーを取得します。 2. ヘッダーが NULL の場合、def_depth を返します。 3. 値が infinity の場合、DAV_INFINITY を返します。 4. 値が 0 または 1 の場合、対応する整数値を返します。 5. 上記以外の値が指定された場合、ap_log_rerror を使用してエラーログを出力し、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.c 内の各メソッドハンドラ（dav_method_propfind, dav_method_copymove 等）。
  • modules/dav/main/util_lock.c:195
  • **呼び出し先**:
  • apr_table_get (APR)
  • ap_cstr_casecmp (Apache)
  • strcmp (標準ライブラリ)
  • ap_log_rerror (Apache)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • **失敗モード**: Depth ヘッダーに 0, 1, infinity 以外の値が含まれている場合、-1 を返し、エラーログを記録します。呼び出し元はこの戻り値を確認し、400 Bad Request を返すことが想定されています。

保守担当者向けメモ

  • 本関数は Depth ヘッダーの妥当性検証を兼ねています。戻り値が -1 の場合、呼び出し元で適切な HTTP エラーレスポンスを生成する必要があります。
