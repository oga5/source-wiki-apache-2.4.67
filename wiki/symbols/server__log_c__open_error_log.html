open_error_log

定義

  • 種別: function
  • 定義箇所: server/log.c:367

目的

サーバー設定 (server_rec) に基づき、エラーログの出力先（パイプ、syslog、またはファイル）を初期化およびオープンする。

シグネチャまたは形状

int open_error_log(server_rec *s, int is_main, apr_pool_t *p)

入力と出力

  • **入力**:
  • s: 設定対象のサーバーレコード。s->error_fname にログ出力先パスまたはコマンドが格納されている。
  • is_main: メインサーバーか否かを示すフラグ。
  • p: ログオープンに使用するメモリプール。
  • **出力**:
  • 成功時は OK を返す。
  • 失敗時は DONE を返す。
  • 成功時、s->error_log にオープンされた apr_file_t が格納される（syslog の場合は NULL）。

前提条件と不変条件

  • s->error_fname が | で始まる場合、パイプ経由のログ出力として処理される。
  • syslog が有効な場合、syslog または syslog:facility 形式の指定を解釈する。
  • それ以外の場合はファイルパスとして扱い、ap_server_root_relative で絶対パスへ変換する。

アルゴリズムまたは振る舞い

1. **パイプログ**: | で始まる場合、log_child を呼び出し、子プロセスを生成してパイプを s->error_log に割り当てる。 2. **syslog**: syslog 指定の場合、openlog を使用してシステムログを初期化する。facility 指定がある場合は対応する値を検索し、ない場合は LOG_LOCAL7 を使用する。 3. **ファイルログ**: 上記以外の場合、apr_file_open を APR_APPEND | APR_WRITE | APR_CREATE | APR_LARGEFILE モードで実行し、ファイルを開く。 4. **エラーハンドリング**: 各ステップで失敗した場合は ap_log_error を通じてエラーメッセージを出力し、DONE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/log.c:501 (メインサーバーのログ初期化)
  • server/log.c:564 (仮想ホストのログ初期化)
  • **呼び出し先**:
  • log_child (パイプログ用)
  • ap_log_error (エラー報告用)
  • apr_file_open (ファイルオープン用)
  • openlog (syslog 初期化用)

関連データ構造

  • server_rec: ログ設定 (error_fname) と出力先ハンドル (error_log) を保持。
  • apr_file_t: オープンされたログファイルのハンドル。
  • TRANS: syslog のファシリティ名と値の対応表。

失敗モードと境界条件

  • パイプコマンドの起動失敗: log_child が APR_SUCCESS 以外を返すと DONE を返す。
  • 無効なパス: ap_server_root_relative が失敗、または apr_file_open が失敗すると DONE を返す。
  • 不正な syslog ファシリティ: 定義されていないファシリティが指定された場合、エラーを記録して DONE を返す。

保守担当者向けメモ

  • is_main フラグは、パイプログの stderr 継承制御に使用される。
  • syslog 関連の処理は #ifdef HAVE_SYSLOG で囲まれており、プラットフォーム依存性がある。
  • パイプログのコマンド型 (cmdtype) は、互換性維持のため APR_PROGRAM_ENV と APR_SHELLCMD_ENV の分岐が存在する。
