fixup_dflt

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_dir.c:171

目的

FallbackResource ディレクティブで指定されたリソースへの内部転送を処理します。ファイルが存在しないリクエストに対して、設定されたフォールバック先を解決し、成功した場合は内部リダイレクトを行います。

シグネチャまたは形状

static int fixup_dflt(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**:
  • OK: 内部リダイレクトが成功した場合。
  • DECLINED: フォールバック設定がない、または無効な場合。
  • HTTPステータスコード: サブリクエストでエラーが発生した場合。

前提条件と不変条件

  • dir_config_rec から取得した dflt 設定が NULL または "disabled" の場合は即座に DECLINED を返します。
  • サブリクエストの生成には r->pool を使用し、処理終了後に ap_destroy_sub_req で解放する必要があります。

アルゴリズムまたは振る舞い

1. 現在のリクエスト設定から dflt（フォールバック先）を取得します。 2. リクエストにクエリ文字列（r->args）がある場合、フォールバック先に付与します。 3. ap_sub_req_lookup_uri を使用して、フォールバック先リソースのサブリクエストを生成します。 4. サブリクエストの結果が HTTP_OK であり、かつハンドラが proxy-server であるか、ファイルタイプが APR_REG（通常ファイル）である場合、ap_internal_fast_redirect を実行してリクエストを転送します。 5. サブリクエストがリダイレクト（3xx）を返す場合、ヘッダーやノートを親リクエストにマージし、ステータスを保持します。 6. 最後にサブリクエストを破棄し、エラーがあればそれを返し、なければ DECLINED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_dir.c:399 （dir_fixups 関数内から呼び出されます）
  • **呼び出し先**:
  • ap_get_module_config
  • apr_pstrcat
  • ap_sub_req_lookup_uri
  • ap_internal_fast_redirect
  • ap_is_HTTP_REDIRECT
  • apr_pool_join, apr_table_overlay
  • ap_destroy_sub_req

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • dir_config_rec: mod_dir のディレクトリ設定構造体。dflt フィールドにフォールバック先パスを保持します。

失敗モードと境界条件

  • **無限再帰**: FallbackResource に指定されたリソース自体が存在しない場合、内部リダイレクトの制限回数に達するまで再帰する可能性があります。
  • **サブリクエストエラー**: サブリクエストが HTTP_OK 以外を返した場合、そのステータスコードが呼び出し元に伝播されます。

保守担当者向けメモ

  • 本関数は fixup_dir のロジックを簡略化したものです。修正を行う際は fixup_dir のコメントも併せて参照してください。
  • FallbackResource の解決先が動的コンテンツである場合、proxy-server ハンドラ等の判定が適切に行われることを確認してください。
