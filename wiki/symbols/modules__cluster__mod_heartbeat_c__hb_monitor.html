hb_monitor

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartbeat.c:45

目的

サーバーのスコアボード情報を集計し、現在の負荷状況（Ready/Busy）を UDP マルチキャストパケットとして送信する。

シグネチャまたは形状

static int hb_monitor(hb_ctx_t *ctx, apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: モジュールの設定および状態を保持する hb_ctx_t 構造体へのポインタ。
  • p: ソケット通信に使用する一時的なメモリプール。
  • **出力**:
  • 戻り値: 常に OK を返す。

前提条件と不変条件

  • ctx->server_limit および ctx->thread_limit が有効な値であること。
  • ctx->mcast_addr が初期化済みであること。
  • ap_scoreboard_image がアクセス可能な状態であること。

アルゴリズムまたは振る舞い

1. ap_mpm_query を呼び出し、現在の MPM 世代情報を取得する。 2. スコアボードを走査し、現在の世代 (mpm_generation) に一致するプロセスかつ、状態が SERVER_READY のものを ready カウント、それ以外（SERVER_DEAD 等を除く）を busy カウントとして集計する。 3. v=1&ready=X&busy=Y 形式の文字列を生成する。 4. apr_socket_create で UDP ソケットを作成し、apr_mcast_loopback を有効化する。 5. apr_socket_sendto を使用して、設定されたマルチキャストアドレスへ集計結果を送信する。 6. 送信完了後、ソケットをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cluster/mod_heartbeat.c:137 (hb_watchdog_step からの呼び出しが確認されている)
  • **呼び出し先**:
  • ap_mpm_query
  • ap_get_scoreboard_process
  • apr_snprintf
  • apr_socket_create
  • apr_mcast_loopback
  • apr_socket_sendto
  • ap_log_error
  • apr_socket_close

関連データ構造

  • hb_ctx_t: モジュールの設定（マルチキャスト先アドレス等）を保持。
  • process_score: プロセスごとのスコアボード情報。
  • worker_score: スレッドごとのスコアボード情報。

失敗モードと境界条件

  • ソケット作成、マルチキャスト設定、または送信処理が失敗した場合、ap_log_error で警告が出力されるが、関数はエラーを返さず OK を返して終了する。
  • スコアボードの走査において、現在の世代と一致しないプロセスは集計対象外となる。

保守担当者向けメモ

  • 本関数は送信ごとにソケットの生成と破棄を行うため、呼び出し頻度が高い場合はオーバーヘッドに注意が必要。
  • 送信失敗時も処理を継続する設計となっているため、ネットワーク障害が継続している場合はログが大量に出力される可能性がある。
