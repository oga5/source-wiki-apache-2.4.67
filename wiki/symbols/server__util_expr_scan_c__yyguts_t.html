yyguts_t

定義

  • 種別: struct
  • 定義箇所: server/util_expr_scan.c:636

目的

Flex によって生成された再入可能（reentrant）な字句解析器において、スキャナの全状態を保持するための内部構造体です。

シグネチャまたは形状

struct yyguts_t {
    YY_EXTRA_TYPE yyextra_r;
    FILE *yyin_r, *yyout_r;
    size_t yy_buffer_stack_top;
    size_t yy_buffer_stack_max;
    YY_BUFFER_STATE * yy_buffer_stack;
    char yy_hold_char;
    int yy_n_chars;
    int yyleng_r;
    char *yy_c_buf_p;
    int yy_init;
    int yy_start;
    int yy_did_buffer_switch_on_eof;
    int yy_start_stack_ptr;
    int yy_start_stack_depth;
    int *yy_start_stack;
    yy_state_type yy_last_accepting_state;
    char* yy_last_accepting_cpos;
    int yylineno_r;
    int yy_flex_debug_r;
    char *yytext_r;
    int yy_more_flag;
    int yy_more_len;
    YYSTYPE * yylval_r;
};

入力と出力

  • **入力**: yyextra_r（ap_expr_parse_ctx_t）を介して入力バッファやメモリプールを参照します。
  • **出力**: yylval_r を通じてパーサーへトークン値を渡します。

前提条件と不変条件

  • 再入可能なスキャナインスタンス（yyscanner）が初期化されている必要があります。
  • yyextra_r は呼び出し元が管理する解析コンテキストを指す必要があります。

アルゴリズムまたは振る舞い

Flex が生成した DFA（決定性有限オートマトン）の実行状態を保持します。ap_expr_yylex 実行中に、現在のスキャン位置（yy_c_buf_p）、バッファスタック、開始条件（yy_start）、およびマッチしたトークン情報（yytext_r, yyleng_r）を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_expr_scan.c 内の各関数（ap_expr_yylex 等）が、引数 yyscanner を struct yyguts_t * にキャストして内部状態にアクセスします。
  • **呼び出し先**: 直接的な関数呼び出しは行いませんが、スキャナの内部状態を管理する各マクロ（BEGIN, YY_INPUT 等）が本構造体のメンバを操作します。

関連データ構造

  • ap_expr_parse_ctx_t (yyextra): 解析コンテキスト。
  • YY_BUFFER_STATE: 入力バッファの状態。
  • YYSTYPE: パーサーに渡されるトークンの値型。

失敗モードと境界条件

  • バッファスタックのオーバーフローやメモリ不足が発生した場合、Flex の内部ロジックによりエラー処理が行われます。
  • yyscanner が NULL の状態でアクセスされた場合、未定義の動作となります。

保守担当者向けメモ

  • 本構造体は Flex によって自動生成されるため、手動での直接編集は避けてください。
  • 変更が必要な場合は server/util_expr_scan.l を修正し、Flex を再実行してください。
  • yyguts_t のメンバは、非再入可能スキャナにおけるグローバル変数に相当します。
