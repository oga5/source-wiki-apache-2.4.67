cfg_directory

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:137

目的

Lua スクリプトから Apache のディレクトリ設定構造体 (ap_lua_dir_cfg) にアクセスし、設定されているディレクトリパス文字列を取得する。

シグネチャまたは形状

static int cfg_directory(lua_State *L)

入力と出力

  • **入力**: Lua スタックの第1引数に ap_lua_dir_cfg 型のユーザーデータ。
  • **出力**: Lua スタック上にディレクトリパスの文字列をプッシュし、戻り値として 1 を返す。

前提条件と不変条件

  • スタックの第1引数が check_dir_config によって検証可能な有効な ap_lua_dir_cfg 構造体であること。
  • cfg->dir が有効な文字列として初期化されていること。

アルゴリズムまたは振る舞い

1. check_dir_config(L, 1) を呼び出し、Lua スタックからディレクトリ設定構造体を取得・検証する。 2. 取得した構造体内の dir メンバ（ディレクトリパス）を lua_pushstring で Lua スタックに積む。 3. スタックに積んだ要素数 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_config.c:153 において、cfg_methods テーブル内の "directory" メソッドとして登録されている。
  • **呼び出し先**: check_dir_config (Lua スタックからの型チェックとデータ取得)。

関連データ構造

  • ap_lua_dir_cfg: ディレクトリ単位の設定情報を保持する構造体。
  • lua_State: Lua インタプリタの状態を管理する構造体。

失敗モードと境界条件

  • check_dir_config が失敗した場合、Lua 側でエラーが発生し、本関数は正常終了しない可能性がある。

保守担当者向けメモ

  • 本関数は cfg_methods 配列を通じて Lua からメソッドとして呼び出されることを前提としている。
  • cfg->dir のメモリ管理は Apache のプール機構に依存しているため、本関数内での解放は不要。
