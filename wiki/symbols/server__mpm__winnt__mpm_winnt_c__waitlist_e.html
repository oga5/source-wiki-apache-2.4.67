waitlist_e

定義

  • 種別: enum
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:537

目的

Windows MPMにおけるプロセス監視用待機リストのインデックスを定義する列挙型です。

シグネチャまたは形状

enum waitlist_e {
    waitlist_ready = 0,
    waitlist_term = 1
};

入力と出力

  • 入力: なし（列挙定数）
  • 出力: waitlist_ready (0) および waitlist_term (1) の整数値。

前提条件と不変条件

  • waitlist_ready は待機リストのインデックス0に対応します。
  • waitlist_term は待機リストのインデックス1に対応します。

アルゴリズムまたは振る舞い

WaitForMultipleObjects 等のWindows API関数において、監視対象のハンドル配列（HANDLE waitlist[2]）へアクセスする際のインデックスとして使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/winnt/mpm_winnt.c:555 にて、HANDLE waitlist[2] の定義および参照に使用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • HANDLE waitlist[2]: 本列挙型をインデックスとして使用するハンドル配列。

失敗モードと境界条件

  • 配列のサイズが2であるため、定義された列挙値（0, 1）以外の値をインデックスとして使用すると範囲外アクセスとなります。

保守担当者向けメモ

  • 本列挙型は mpm_winnt.c 内のプロセス監視ロジックと密接に結合しています。ハンドル配列の構成を変更する場合は、本列挙型の定義も併せて更新する必要があります。
