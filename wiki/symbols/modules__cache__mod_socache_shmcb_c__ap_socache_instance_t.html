ap_socache_instance_t

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_shmcb.c:107
  • 別定義: include/ap_socache.h:49 にも定義が存在します。

目的

共有メモリベースのキャッシュ（SHMCB）プロバイダーにおける、モジュールインスタンスのコンテキストを保持するための構造体です。

シグネチャまたは形状

struct ap_socache_instance_t {
    apr_pool_t *pool;
    const char *data_file;
    apr_size_t shm_size;
    apr_shm_t *shm;
    SHMCBHeader *header;
};

入力と出力

  • **入力**: キャッシュプロバイダーの初期化時や操作時に、インスタンスへのポインタとして渡されます。
  • **出力**: 共有メモリセグメント（shm）やヘッダー情報（header）へのアクセスポイントを提供します。

前提条件と不変条件

  • shm は apr_shm_t を介して確保された共有メモリ領域を指す必要があります。
  • header は共有メモリの先頭に配置された SHMCBHeader 構造体を指す必要があります。

アルゴリズムまたは振る舞い

本構造体は、キャッシュのライフサイクル全体を通じて保持されるコンテキストです。socache_shmcb_init で共有メモリの確保とヘッダーの初期化が行われ、以降の store や retrieve などの操作において、共有メモリ内のデータ構造へアクセスするための基点として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cache_socache.c や mod_authn_socache.c などの上位キャッシュモジュール、および ap_socache.h で定義されたインターフェース（create, init, store, retrieve 等）を呼び出すコード。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SHMCBHeader: 共有メモリのメタデータ管理に使用。
  • apr_shm_t: 共有メモリセグメントの管理に使用。
  • apr_pool_t: インスタンスのメモリライフサイクル管理に使用。

失敗モードと境界条件

  • shm_size が非常に大きい場合、内部のサイズ計算において unsigned int のオーバーフローが発生するリスクがソース内のコメントで指摘されています。
  • 共有メモリの確保に失敗した場合、shm が NULL となる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_socache_shmcb.c 固有の実装と ap_socache.h のインターフェース定義の両方で参照されています。変更時は両方の整合性に注意してください。
  • このモジュールは AP_SOCACHE_FLAG_NOTMPSAFE を指定しており、排他制御は呼び出し側に依存します。本構造体自体にはロック機構は含まれていません。
