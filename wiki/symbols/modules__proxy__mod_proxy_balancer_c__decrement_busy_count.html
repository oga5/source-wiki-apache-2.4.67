decrement_busy_count

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:462

目的

プロキシワーカーのビジーカウントを減算し、リソースの解放を管理するクリーンアップ関数です。

シグネチャまたは形状

static apr_status_t decrement_busy_count(void *worker_)

入力と出力

  • **入力**: void *worker_ - proxy_worker 構造体へのポインタ。
  • **出力**: apr_status_t - 常に APR_SUCCESS を返します。

前提条件と不変条件

  • worker_ は proxy_worker 型にキャスト可能である必要があります。
  • worker->s が有効であり、共有メモリ上の統計情報にアクセス可能である必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った worker_ を proxy_worker 型にキャストします。 2. worker->s->busy（現在のビジーカウント）が 0 より大きいことを確認します。 3. 条件を満たす場合、worker->s->busy をデクリメントします。 4. APR_SUCCESS を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_balancer.c:613 にて、apr_pool_cleanup_register のクリーンアップハンドラとして登録されています。
  • **呼び出し先**: 外部関数呼び出しはありません。

関連データ構造

  • proxy_worker: プロキシワーカーの情報を保持する構造体。
  • proxy_worker_shared (worker->s): 共有メモリ上に配置されたワーカーの統計情報。

失敗モードと境界条件

  • worker->s->busy が 0 の場合、デクリメントは行われず、アンダーフローを防ぐ安全な実装となっています。

保守担当者向けメモ

  • 本関数は apr_pool_cleanup_register を介して呼び出されることを前提としています。プール破棄時に自動的にビジーカウントを調整する目的で使用されます。
