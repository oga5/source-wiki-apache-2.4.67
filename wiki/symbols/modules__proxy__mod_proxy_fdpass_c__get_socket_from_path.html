get_socket_from_path

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fdpass.c:51

目的

指定されたファイルシステムパスに対応する Unix ドメインソケットを作成し、接続を確立する。

シグネチャまたは形状

static apr_status_t get_socket_from_path(apr_pool_t *p,
                                         const char* path,
                                         apr_socket_t **out_sock)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • path: 接続先となる Unix ドメインソケットのファイルパス。
  • **出力**:
  • out_sock: 接続済みソケットへのポインタを格納するアドレス。
  • 戻り値: apr_status_t 型。成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • out_sock は呼び出し元で有効なメモリ領域を指している必要がある。
  • 実行環境が Unix ドメインソケット（AF_UNIX）をサポートしていること。

アルゴリズムまたは振る舞い

1. out_sock を NULL に初期化する。 2. apr_socket_create を呼び出し、AF_UNIX および SOCK_STREAM 型のソケットを作成する。 3. 作成に失敗した場合は、そのエラーコードを即座に返す。 4. ap_proxy_connect_uds を呼び出し、指定された path へソケットを接続する。 5. 接続に成功した場合、作成したソケットを out_sock に代入し、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_fdpass.c:139 (proxy_fdpass_handler 内)
  • **呼び出し先**:
  • apr_socket_create (APR API)
  • ap_proxy_connect_uds (Apache プロキシユーティリティ)

関連データ構造

  • apr_socket_t: APR のソケット抽象化構造体。
  • apr_pool_t: APR のメモリプール。

失敗モードと境界条件

  • apr_socket_create が失敗した場合、エラーコードを返して終了する。
  • ap_proxy_connect_uds が失敗した場合（パスが存在しない、権限不足、接続拒否など）、そのエラーコードを返して終了する。
  • 失敗時は out_sock は NULL のままとなる。

保守担当者向けメモ

  • 本関数は mod_proxy_fdpass における接続確立の定型処理である。
  • 接続先パスの検証やソケットのタイムアウト設定は、呼び出し先の ap_proxy_connect_uds に依存している。
