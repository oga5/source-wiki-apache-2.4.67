make_invalid_resp

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_ws.c:178

目的

WebSocket ハンドシェイクの失敗時や不正なレスポンスを検知した際に、クライアントへ返却するためのエラーレスポンスバケットを生成する。

シグネチャまたは形状

static apr_bucket *make_invalid_resp(conn_rec *c2, int status, apr_table_t *notes);

入力と出力

  • **入力**:
  • c2: 子接続（conn_rec）のポインタ。メモリ割り当てに使用。
  • status: 返却する HTTP ステータスコード（例: HTTP_BAD_GATEWAY）。
  • notes: レスポンスに付与するメモテーブル（apr_table_t）。
  • **出力**:
  • 構築された apr_bucket へのポインタ。

前提条件と不変条件

  • c2->pool が有効であること。
  • Content-Length ヘッダーは常に 0 に設定される。
  • notes が NULL の場合は新規にテーブルが作成され、存在する場合はクローンされる。

アルゴリズムまたは振る舞い

1. c2->pool を使用して、レスポンスヘッダー用のテーブルを初期化し、Content-Length: 0 を設定する。 2. notes 引数が提供されていればクローンし、そうでなければ新規作成する。 3. AP_HAS_RESPONSE_BUCKETS マクロの定義状況に応じて、以下のいずれかの方法でバケットを生成する：

  • 定義されている場合: ap_bucket_response_create を使用。
  • 定義されていない場合: h2_bucket_headers_create を使用し、h2_headers_create で生成したヘッダー構造体をラップする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_ws.c:246
  • modules/http2/h2_ws.c:258

    • (いずれも ws_handle_resp 内で、WebSocket ハンドシェイクの検証失敗時に呼び出される)

  • **呼び出し先**:
  • apr_table_make, apr_table_setn, apr_table_clone (APR)
  • ap_bucket_response_create または h2_bucket_headers_create / h2_headers_create

関連データ構造

  • conn_rec: Apache HTTPD の接続コンテキスト。
  • apr_table_t: HTTP ヘッダーやメモを保持するテーブル。
  • apr_bucket: Apache のバケットチェーンを構成するデータ単位。

失敗モードと境界条件

  • メモリ不足時には apr_table_make 等の内部関数によりアボートまたはエラーが発生する可能性がある。
  • c2 が無効な場合、セグメンテーションフォールトが発生する。

保守担当者向けメモ

  • 本関数は H2_USE_WEBSOCKETS が有効な環境でのみ使用される。
  • AP_HAS_RESPONSE_BUCKETS の有無による条件分岐は、Apache HTTPD のバージョン間での API 差異を吸収するためのものである。
