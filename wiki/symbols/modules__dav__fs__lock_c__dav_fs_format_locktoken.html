dav_fs_format_locktoken

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/lock.c:273

目的

dav_locktoken 構造体を、WebDAV プロトコルで規定される opaquelocktoken:<UUID> 形式の文字列に変換します。

シグネチャまたは形状

const char *dav_fs_format_locktoken(
    apr_pool_t *p,
    const dav_locktoken *locktoken)

入力と出力

  • **入力**:
  • p: 文字列の割り当てに使用するメモリプール。
  • locktoken: 変換対象の dav_locktoken 構造体（内部に UUID を保持）。
  • **出力**:
  • opaquelocktoken: で始まるフォーマット済みの文字列を返します。メモリは p から割り当てられます。

前提条件と不変条件

  • locktoken 引数は有効な dav_locktoken 構造体を指している必要があります。
  • p は有効な apr_pool_t である必要があります。

アルゴリズムまたは振る舞い

1. apr_uuid_format を使用して、locktoken->uuid を標準的な UUID 文字列形式（buf）に変換します。 2. apr_pstrcat を使用し、"opaquelocktoken:" というプレフィックスと変換後の UUID 文字列を連結します。 3. 連結された文字列をメモリプール p 上に生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/lock.c:269
  • modules/dav/fs/lock.c:1430
  • **呼び出し先**:
  • apr_uuid_format (APR ライブラリ)
  • apr_pstrcat (APR ライブラリ)

関連データ構造

  • dav_locktoken: ロック識別子を保持する構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • メモリプール p が枯渇している場合、apr_pstrcat の内部動作によりメモリ割り当てエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は mod_dav のロックプロバイダインターフェースの一部として、ロックトークンのシリアライズを担います。
  • UUID のフォーマット長は APR_UUID_FORMATTED_LENGTH に依存しており、APR の仕様変更に追従します。
