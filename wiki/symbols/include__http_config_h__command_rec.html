command_rec

定義

  • 種別: struct (typedef)
  • 定義箇所: include/http_config.h:203

目的

Apache HTTP Server のモジュールが提供する設定ディレクティブの定義を保持するための構造体です。

シグネチャまたは形状

typedef struct command_struct command_rec; ※詳細な構造体メンバの定義は command_struct に依存します。

入力と出力

  • 入力: 設定ファイル解析時にディレクティブ名、引数、ハンドラー関数などが格納されます。
  • 出力: ap_find_command 等の検索関数により、特定のディレクティブ定義が取得されます。

前提条件と不変条件

  • モジュールが提供するディレクティブのリスト（テーブル）として定義されます。
  • module 構造体の一部として登録され、サーバーのライフサイクルを通じて参照されます。

アルゴリズムまたは振る舞い

  • 設定ファイルの読み込み時、ap_walk_config がツリーを走査する際に、この構造体で定義されたハンドラー関数が呼び出されます。
  • ap_find_command や ap_find_command_in_modules を介して、ディレクティブ名に基づいた検索が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: include/http_config.h 内の検索関数（ap_find_command 等）や、設定解析プロセス。
  • 呼び出し先: 各モジュール（mod_access_compat.c, mod_auth_basic.c 等）で定義された静的配列として参照されます。

関連データ構造

  • command_struct: command_rec の実体。
  • module: モジュールが保持するディレクティブテーブルの型として使用されます。
  • cmd_parms: ディレクティブ解析時にハンドラーへ渡されるコンテキスト情報。

失敗モードと境界条件

  • 検索対象のディレクティブ名がどのモジュールにも存在しない場合、検索関数は NULL を返します。

保守担当者向けメモ

  • 本構造体は include/http_config.h で command_struct のエイリアスとして定義されています。
  • モジュール開発者は AP_INIT_* マクロ群を使用してこの構造体を初期化することが推奨されます。
