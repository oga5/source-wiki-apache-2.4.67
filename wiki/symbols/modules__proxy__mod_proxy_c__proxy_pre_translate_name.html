proxy_pre_translate_name

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:1049

目的

Apache HTTP Server のリクエスト処理パイプラインにおいて、translate_name フックの実行前段階でプロキシ関連の処理をトリガーするために使用されます。

シグネチャまたは形状

static int proxy_pre_translate_name(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在処理中のリクエスト構造体）
  • **出力**: int （Apache のフック戻り値。proxy_trans の結果をそのまま返す）

前提条件と不変条件

  • r は有効な request_rec 構造体である必要があります。
  • 本関数は ap_hook_pre_translate_name に登録されることを前提としています。

アルゴリズムまたは振る舞い

1. 引数として受け取った request_rec *r を proxy_trans(r, 1) に渡します。 2. proxy_trans の第2引数に 1 を指定することで、翻訳処理の初期フェーズであることを明示して実行します。 3. proxy_trans の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフック機構（ap_hook_pre_translate_name を介して登録）。
  • **呼び出し先**: proxy_trans 関数。

関連データ構造

  • request_rec: Apache のリクエスト処理における中心的なデータ構造。

失敗モードと境界条件

  • 内部で呼び出される proxy_trans の挙動に依存します。proxy_trans がエラーコードを返した場合、その値がそのまま本関数の戻り値となります。

保守担当者向けメモ

  • 本関数は proxy_trans のラッパーとして機能しています。ロジックの変更が必要な場合は、proxy_trans の実装を確認してください。
  • modules/proxy/mod_proxy.c:3363 にてフック登録が行われています。
