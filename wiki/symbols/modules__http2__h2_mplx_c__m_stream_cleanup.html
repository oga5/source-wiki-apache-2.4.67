m_stream_cleanup

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_mplx.c:149

目的

HTTP/2 ストリームの終了処理を行い、関連するリソースの解放および状態遷移（破棄待ちまたは保持）を管理します。

シグネチャまたは形状

static void m_stream_cleanup(h2_mplx *m, h2_stream *stream)

入力と出力

  • **入力**:
  • m: ストリームが所属する h2_mplx インスタンス。
  • stream: クリーンアップ対象の h2_stream オブジェクト。
  • **出力**: なし（副作用としてストリームの状態遷移とメモリ管理を実行）。

前提条件と不変条件

  • stream->state が H2_SS_CLEANUP である必要があります（ap_assert により検証）。
  • h2_mplx のミューテックスによる排他制御下で呼び出されることが想定されています。

アルゴリズムまたは振る舞い

1. **イベント購読解除**: ストリームに関連付けられた c2 接続（h2_conn_ctx_t）が存在する場合、beam_out および beam_in のコールバックをすべて NULL に設定し、イベント通知を停止します。 2. **レジストリからの削除**: h2_stream_cleanup を呼び出し、m->streams（ハッシュ）および m->q（キュー）から当該ストリームを削除します。 3. **状態に応じた後処理**:

  • c2 接続が既に終了している場合: add_for_purge を呼び出し、ストリームを破棄待ちリストへ移動します。
  • c2 接続が実行中の場合: h2_c2_abort で接続を強制終了し、ストリームを m->shold（保持リスト）へ移動します。
  • c2 接続が一度も開始されていない場合: add_for_purge を呼び出し、即座に破棄待ちリストへ移動します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_mplx.c:504
  • modules/http2/h2_mplx.c:595
  • modules/http2/h2_mplx.c:1189
  • **呼び出し先**:
  • h2_conn_ctx_get, h2_beam_on_* (イベント解除)
  • h2_stream_cleanup, h2_ihash_remove, h2_iq_remove (レジストリ操作)
  • stream_is_running, add_for_purge, h2_c2_abort, h2_ihash_add (状態管理)

関連データ構造

  • h2_mplx: 多重化器のメイン構造体。
  • h2_stream: ストリームのメタデータと状態を保持。
  • h2_conn_ctx_t: c2 接続のコンテキスト（beam_in, beam_out を含む）。

失敗モードと境界条件

  • stream->state が H2_SS_CLEANUP でない場合、アサーション失敗によりプロセスが停止します。
  • c2_ctx が NULL の場合（ストリームがワーカーに割り当てられる前にクリーンアップされた場合）、安全にスキップして破棄処理へ進みます。

保守担当者向けメモ

  • 本関数は h2_mplx のライフサイクル管理の末端を担います。spurge（破棄待ち）と shold（保持）の使い分けは、ワーカーの終了待ち状態に依存するため、状態遷移の変更時には stream_is_running の判定ロジックと整合性を確認してください。
