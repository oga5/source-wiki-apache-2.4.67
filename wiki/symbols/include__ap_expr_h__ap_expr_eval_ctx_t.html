ap_expr_eval_ctx_t

定義

  • 種別: struct
  • 定義箇所: include/ap_expr.h:105

目的

Apache HTTP Server の式評価エンジンにおいて、評価実行時のコンテキスト（リクエスト情報、メモリプール、正規表現の状態、結果格納先など）を保持・管理するために使用されます。

シグネチャまたは形状

typedef struct {
    request_rec *r;
    conn_rec *c;
    server_rec *s;
    apr_pool_t *p;
    const char **err;
    const ap_expr_info_t *info;
    ap_regmatch_t *re_pmatch;
    apr_size_t re_nmatch;
    const char **re_source;
    const char **vary_this;
    const char **result_string;
    void *data;
    int reclvl;
} ap_expr_eval_ctx_t;

入力と出力

  • **入力**: 評価対象の式情報 (info)、リクエスト/接続/サーバーの各コンテキスト、および評価に必要なメモリプール (p)。
  • **出力**: 評価結果の文字列 (result_string)、エラー発生時のメッセージ (err)、Vary ヘッダーに追加すべきヘッダー名 (vary_this)、および正規表現のマッチ結果 (re_pmatch)。

前提条件と不変条件

  • 評価には apr_pool_t が必須であり、コンテキスト作成時に割り当てられている必要があります。
  • vary_this が NULL の場合、呼び出し元は Vary ヘッダー情報の収集を要求していないものとみなされます。
  • reclvl は再帰的な評価の深さを追跡するために使用されます。

アルゴリズムまたは振る舞い

  • 評価エンジンは本構造体を参照して変数や関数の解決を行います。
  • 評価中にヘッダー名が vary_this に記録されることで、自動的に Vary ヘッダーの管理が行われます。
  • ユーザー定義のカスタム関数に対しては、data メンバーを通じて任意のコンテキストデータを渡すことが可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_expr_exec_ctx() が本構造体を引数として受け取ります。また、mod_include 等のモジュールが内部的にインスタンス化して使用します。
  • **呼び出し先**: 評価エンジン内の変数関数 (ap_expr_var_func_t)、文字列関数 (ap_expr_string_func_t)、リスト関数 (ap_expr_list_func_t)、および単項/二項演算子関数が、このコンテキストを引数として受け取ります。

関連データ構造

  • request_rec, conn_rec, server_rec: HTTP 処理の基本構造体。
  • ap_expr_info_t: 解析済み式ツリーとメタデータ。
  • ap_regmatch_t: 正規表現のバックリファレンス用マッチング情報。

失敗モードと境界条件

  • err メンバーが指す先にエラー文字列が格納されるため、呼び出し元は評価後にこの値を確認する必要があります。
  • re_pmatch のサイズ (re_nmatch) を超える正規表現バックリファレンスへのアクセスは未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • AP_EXPR_FLAG_RESTRICTED を使用する際は、機密情報へのアクセスが制限されるため、本コンテキストの data フィールド等の利用には注意が必要です。
  • 新しい機能を追加する際は、ap_expr_eval_ctx_t の構造を変更せずに済むよう、可能な限り data ポインタを活用してください。
