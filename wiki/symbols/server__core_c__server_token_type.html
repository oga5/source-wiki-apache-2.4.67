server_token_type

定義

  • 種別: enum
  • 定義箇所: server/core.c:3562

目的

HTTP レスポンスヘッダー（Server ヘッダー）に含めるサーバー情報の詳細度を制御するための列挙型です。

シグネチャまたは形状

enum server_token_type {
    SrvTk_MAJOR,
    SrvTk_MINOR,
    SrvTk_MINIMAL,
    SrvTk_OS,
    SrvTk_FULL,
    SrvTk_PRODUCT_ONLY
};

入力と出力

  • **入力**: 設定ファイルにおける ServerTokens ディレクティブの設定値。
  • **出力**: サーバーがクライアントへ送信する Server ヘッダーの文字列形式を決定するフラグ。

前提条件と不変条件

  • 各列挙子は、情報の詳細度が段階的に増加する階層構造を成しています。
  • SrvTk_FULL がデフォルト値として使用されます（server/core.c:3570）。

アルゴリズムまたは振る舞い

  • SrvTk_MAJOR: メジャーバージョンのみを表示（例: Apache/2）。
  • SrvTk_MINOR: マイナーバージョンまで表示（例: Apache/2.0）。
  • SrvTk_MINIMAL: 完全なバージョンを表示（例: Apache/2.0.41）。
  • SrvTk_OS: バージョンと OS 情報まで表示（例: Apache/2.0.41 (UNIX)）。
  • SrvTk_FULL: バージョン、OS、およびインストールされているモジュール情報まで表示（例: Apache/2.0.41 (UNIX) PHP/4.2.2 FooBar/1.2b）。
  • SrvTk_PRODUCT_ONLY: 製品名のみを表示（例: Apache）。

呼び出し元と呼び出し先

  • 呼び出し元: server/core.c 内の静的変数 ap_server_tokens がこの型を使用しています。
  • 呼び出し先: 情報は提供されていません。

関連データ構造

  • ap_server_tokens (静的変数): server_token_type 型を保持する変数。

失敗モードと境界条件

  • 情報は提供されていません。

保守担当者向けメモ

  • ServerTokens ディレクティブの挙動を変更または拡張する場合は、この列挙型に新しい値を追加し、それを利用するヘッダー生成ロジックを更新する必要があります。
