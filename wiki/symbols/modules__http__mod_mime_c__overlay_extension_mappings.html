overlay_extension_mappings

定義

  • 種別: function
  • 定義箇所: modules/http/mod_mime.c:136

目的

2つの extension_info 構造体（ベース設定とオーバーレイ設定）をマージし、新しい設定を生成する。主にディレクトリ単位の設定継承において、子設定で定義された値が親設定を上書きするために使用される。

シグネチャまたは形状

static void *overlay_extension_mappings(apr_pool_t *p,
                                        const void *key,
                                        apr_ssize_t klen,
                                        const void *overlay_val,
                                        const void *base_val,
                                        const void *data)

入力と出力

  • **入力**:
  • p: メモリ割り当て用のプール。
  • overlay_val: 上書き側の extension_info 構造体。
  • base_val: 基底となる extension_info 構造体。
  • **出力**:
  • base_val をコピーし、overlay_val に存在する非 NULL 値で更新した新しい extension_info 構造体へのポインタ。

前提条件と不変条件

  • overlay_val および base_val は extension_info 型としてキャスト可能であること。
  • p は有効な apr_pool_t であること。
  • 戻り値は apr_pmemdup によりプール p 上に確保される。

アルゴリズムまたは振る舞い

1. base_val を apr_pmemdup で複製し、新しい extension_info 構造体を作成する。 2. overlay_info の各フィールド（forced_type, encoding_type, language_type, handler, charset_type, input_filters, output_filters）を順次確認する。 3. overlay_info の該当フィールドが NULL でない場合、その値を new_info に代入して上書きする。 4. 更新済みの new_info を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/mod_mime.c:203（merge_mime_dir_configs 内のハッシュマージ処理から呼び出される）。
  • **呼び出し先**: apr_pmemdup（APRライブラリ関数）。

関連データ構造

  • extension_info: 拡張子に関連付けられた MIME タイプやフィルタ等のメタデータを保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pmemdup がアボートする可能性がある（APRの標準的な挙動）。
  • overlay_val または base_val が NULL の場合の挙動は、この関数内では明示的にガードされていない。

保守担当者向けメモ

  • 本関数は apr_hash_merge のコールバックとして設計されている。
  • フィールドの追加や削除を行う際は、extension_info 構造体の定義と本関数のコピー・上書きロジックを同期させる必要がある。
