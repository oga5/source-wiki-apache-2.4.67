walk_location_and_if

定義

  • 種別: function
  • 定義箇所: server/request.c:163

目的

リクエスト処理において、URI に基づく <Location> セクションおよび <If> 条件式の設定を順次マージし、コアモジュールのログ設定をリクエストコンテキストに適用する。

シグネチャまたは形状

static int walk_location_and_if(request_rec *r)

入力と出力

  • **入力**: request_rec *r （処理対象のリクエスト構造体）
  • **出力**: int 型のステータスコード。成功時は OK を返し、設定マージ中にエラーが発生した場合はそのステータスコードを返す。

前提条件と不変条件

  • r->per_dir_config が有効な設定コンテキストを保持していること。
  • ap_location_walk および ap_if_walk が正常に実行可能な状態であること。

アルゴリズムまたは振る舞い

1. ap_location_walk(r) を呼び出し、URI に一致する <Location> 設定をマージする。エラー時は即座にリターンする。 2. ap_if_walk(r) を呼び出し、<If> 条件式を評価して設定をマージする。エラー時は即座にリターンする。 3. ap_get_core_module_config(r->per_dir_config) を通じてコアディレクトリ設定を取得する。 4. 取得した設定内に log 設定が存在する場合、それを r->log に割り当てる。 5. 正常終了時に OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/request.c:225
  • server/request.c:274
  • server/request.c:294
  • **呼び出し先**:
  • ap_location_walk
  • ap_if_walk
  • ap_get_core_module_config

関連データ構造

  • request_rec: リクエスト情報を保持する主要構造体。
  • core_dir_config: コアモジュールのディレクトリ設定を保持する構造体。

失敗モードと境界条件

  • ap_location_walk または ap_if_walk が OK 以外の値を返した場合、その値がそのまま呼び出し元へ伝播される。

保守担当者向けメモ

  • 本関数は設定マージの後半フェーズを担う。r->log の更新はコア設定に依存するため、ap_if_walk 完了後の設定状態が反映される点に注意すること。
