terminate_headers

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_http.c:437

目的

バックエンドへの HTTP リクエストヘッダーの末尾を構成し、Connection ヘッダーの制御およびヘッダー終了を示す空行（CRLF）を header_brigade に追加します。

シグネチャまたは形状

static void terminate_headers(proxy_http_req_t *req)

入力と出力

  • **入力**: req (proxy_http_req_t*) - プロキシリクエストの状態を保持する構造体。req->header_brigade にヘッダーが蓄積されます。
  • **出力**: なし（副作用として req->header_brigade にバケットが追加されます）。

前提条件と不変条件

  • req->bucket_alloc が有効であること。
  • req->header_brigade が初期化済みであること。
  • req->p (プール) がヘッダー文字列の割り当てに利用可能であること。

アルゴリズムまたは振る舞い

1. req->force10 が偽の場合、HTTP/1.1 接続の制御を行います。

  • req->upgrade が設定されている場合: Connection: Upgrade および Upgrade ヘッダーを追加します。
  • ap_proxy_connection_reusable が真の場合: Connection: Keep-Alive を追加します。
  • それ以外の場合: Connection: close を追加します。

    • 2. 追加した各ヘッダー文字列を ap_xlate_proto_to_ascii で変換し、apr_bucket_pool_create でバケット化して header_brigade に挿入します。 3. 最後に、ヘッダー終了を示す空行（CRLF_ASCII）を apr_bucket_immortal_create で作成し、header_brigade の末尾に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_http.c:665
  • **呼び出し先**:
  • ap_xlate_proto_to_ascii
  • apr_bucket_pool_create
  • apr_bucket_immortal_create
  • APR_BRIGADE_INSERT_TAIL

関連データ構造

  • proxy_http_req_t: リクエストの状態管理に使用。
  • apr_bucket_brigade: ヘッダーの構築に使用。

失敗モードと境界条件

  • メモリ不足時: apr_pstrdup や apr_pstrcat が失敗する可能性があります。
  • req->upgrade が NULL の場合、Upgrade ヘッダーの生成はスキップされます。

保守担当者向けメモ

  • ap_xlate_proto_to_ascii を使用しているため、EBCDIC 環境への対応が考慮されています。
  • header_brigade への追加順序は HTTP プロトコル仕様に依存するため、変更時は注意してください。
