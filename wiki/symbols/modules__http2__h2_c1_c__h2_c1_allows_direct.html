h2_c1_allows_direct

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1.c:217

目的

現在の接続において、HTTP/2 の直接接続（H2Direct）が許可されているかどうかを判定する。

シグネチャまたは形状

int h2_c1_allows_direct(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（判定対象となる接続レコード）
  • **出力**: int（許可される場合は非ゼロ、それ以外は 0）

前提条件と不変条件

  • c->master が NULL であること（マスター接続ではないこと）。
  • 接続が TLS を使用しているか否かに応じて、必要なプロトコル名（h2 または h2c）が決定される。

アルゴリズムまたは振る舞い

1. 接続がマスター接続（c->master）である場合は、直ちに 0 を返す。 2. ap_ssl_conn_is_ssl(c) を使用して TLS 接続か否かを判定し、必要なプロトコル名（h2 または h2c）を決定する。 3. h2_config_cgeti(c, H2_CONF_DIRECT) を呼び出し、設定値を取得する。

  • 設定値が負の場合、TLS 接続なら 0（無効）、非 TLS 接続なら 1（有効）をデフォルト値として使用する。

    • 4. 設定値が有効であり、かつ ap_is_allowed_protocol が当該プロトコルを許可している場合に 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c:256
  • modules/http2/h2_c1.c:261
  • **呼び出し先**:
  • ap_ssl_conn_is_ssl
  • h2_config_cgeti
  • ap_is_allowed_protocol

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。

失敗モードと境界条件

  • c が NULL の場合の挙動は定義されていない（ソースコード上ではチェックなし）。
  • h2_config_cgeti が取得できない場合、プロトコル種別に基づいたデフォルト値が適用される。

保守担当者向けメモ

  • 本関数は h2_c1.h に宣言されており、HTTP/2 接続の初期化プロセスにおいて重要な判定基準となる。
  • ap_is_allowed_protocol の結果に依存するため、サーバー全体のプロトコル設定変更の影響を受ける。
