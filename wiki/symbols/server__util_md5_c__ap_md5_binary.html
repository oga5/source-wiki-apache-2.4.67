ap_md5_binary

定義

  • 種別: function
  • 定義箇所: server/util_md5.c:53

目的

バイナリデータから MD5 ハッシュを計算し、その結果を 16 進数形式の文字列として返します。

シグネチャまたは形状

char *ap_md5_binary(apr_pool_t *p, const unsigned char *buf, int length)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • buf: ハッシュ対象のバイナリデータ。
  • length: buf のバイト長。
  • **出力**:
  • 16 進数形式の MD5 ハッシュ文字列（apr_pool_t から割り当てられたメモリ領域）。

前提条件と不変条件

  • p は有効な apr_pool_t である必要があります。
  • EBCDIC 環境下では、apr_md5_set_xlate を介して適切な文字コード変換が適用されます。

アルゴリズムまたは振る舞い

1. apr_md5_init で MD5 コンテキストを初期化します。 2. EBCDIC 環境の場合、apr_md5_set_xlate を呼び出し変換設定を行います。 3. apr_md5_update で指定されたバイナリデータをコンテキストに読み込みます。 4. apr_md5_final でハッシュ値を確定させます。 5. apr_palloc を使用して、ハッシュ値（16 進数文字列）を格納するためのメモリを確保します。 6. ap_bin2hex を使用してバイナリハッシュを 16 進数文字列に変換し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_md5.c:77 (ap_md5 関数内)
  • modules/lua/lua_request.c:961
  • modules/ssl/mod_ssl.c:623
  • modules/ssl/ssl_engine_kernel.c:2393
  • **呼び出し先**:
  • apr_md5_init, apr_md5_update, apr_md5_final (APR ライブラリ)
  • apr_md5_set_xlate (APR ライブラリ、EBCDIC 環境時)
  • apr_palloc (APR ライブラリ)
  • ap_bin2hex (内部ユーティリティ)

関連データ構造

  • apr_md5_ctx_t: MD5 計算のコンテキストを保持する APR の構造体。
  • apr_pool_t: メモリ管理のためのプール構造体。

失敗モードと境界条件

  • apr_palloc がメモリ不足により失敗した場合、APR の規定に従いプロセスが終了する可能性があります。
  • length が負の値である場合の挙動は、apr_md5_update の仕様に依存します。

保守担当者向けメモ

  • 本関数は include/util_md5.h で宣言されており、公開インターフェースの一部です。
  • 戻り値の文字列は apr_pool_t に紐付いているため、呼び出し側で明示的な解放は不要です。
  • ap_bin2hex は結果文字列の末尾にヌル終端文字を付与します。
