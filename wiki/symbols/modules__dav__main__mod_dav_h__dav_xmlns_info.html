dav_xmlns_info

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:617

目的

WebDAV リクエスト処理における XML 名前空間の動的な生成と管理を行うためのコンテキスト構造体です。URI とプレフィックスの相互変換を保持します。

シグネチャまたは形状

typedef struct {
    apr_pool_t *pool;
    apr_hash_t *uri_prefix;     /* map URIs to an available prefix */
    apr_hash_t *prefix_uri;     /* map all prefixes to their URIs */
    int count;                  /* counter for "g###" prefixes */
} dav_xmlns_info;

入力と出力

  • **入力**: dav_xmlns_create 関数により、メモリ管理のための apr_pool_t を受け取って初期化されます。
  • **出力**: dav_xmlns_get_uri や dav_xmlns_get_prefix を通じて、名前空間の URI またはプレフィックスの文字列が返されます。

前提条件と不変条件

  • pool は、構造体およびハッシュテーブル内の文字列の生存期間を管理するために使用されます。
  • uri_prefix と prefix_uri は、名前空間の対応関係を保持する双方向のハッシュマップとして機能します。
  • count は、自動生成されるプレフィックス（g### 形式）の一意性を保証するために使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: dav_xmlns_create で空の構造体が作成されます。 2. **登録**: dav_xmlns_add または dav_xmlns_add_uri を使用して、URI とプレフィックスのペアをハッシュテーブルに登録します。 3. **検索**: dav_xmlns_get_uri および dav_xmlns_get_prefix を用いて、登録済みの名前空間情報を取得します。 4. **生成**: dav_xmlns_generate を呼び出すことで、XML 出力に必要な名前空間定義を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/dbm.c 内の dav_propdb_define_namespaces など、プロパティデータベースの操作を行う関数から参照されています。
  • **呼び出し先**: apr_hash 関連の APR 関数群を利用してハッシュ操作を行います。

関連データ構造

  • apr_pool_t: メモリ割り当て用プール。
  • apr_hash_t: 名前空間の対応付けを保持するハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時には apr_pool_t の割り当て失敗によりプロセスが終了する可能性があります。
  • 存在しない URI やプレフィックスを検索した場合、NULL が返される可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dav の XML 処理における名前空間管理の基盤です。
  • dav_xmlns_add 等の API は mod_dav.h で宣言されており、これらを通じて操作することが推奨されます。
  • 構造体内部のハッシュテーブルは、関連付けられた apr_pool_t の生存期間に依存するため、プールがクリアされると無効になります。
