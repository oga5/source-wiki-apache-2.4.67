create_imap_dir_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_imagemap.c:90

目的

ディレクトリ単位の画像マップ設定構造体（imap_conf_rec）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_imap_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定情報の割り当てに使用するメモリプール。
  • char *dummy: Apache 設定フレームワークの規約による未使用の引数。
  • **出力**:
  • void *: 初期化された imap_conf_rec 構造体へのポインタ。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • 戻り値の構造体内の各メンバ（imap_menu, imap_default, imap_base）は、初期状態で NULL に設定される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、リクエストプール p から imap_conf_rec サイズ分のメモリを確保する。 2. 確保した構造体の各設定項目（メニュー設定、デフォルトURL、ベースURL）を NULL で初期化する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_imagemap.c:891 にて、ディレクトリ設定作成用関数として参照されている。
  • **呼び出し先**: apr_palloc（APRメモリ管理関数）。

関連データ構造

  • imap_conf_rec: ディレクトリごとの設定（ImapMenu, ImapDefault, ImapBase）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc がアボートする可能性がある（Apache の標準的なメモリ管理挙動に準拠）。

保守担当者向けメモ

  • 本関数は Apache の設定ライフサイクル管理の一部として呼び出されます。
  • imap_conf_rec のメンバを追加した場合は、必ず本関数で NULL 初期化を行ってください。
