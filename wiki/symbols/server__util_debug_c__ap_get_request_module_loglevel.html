ap_get_request_module_loglevel

定義

  • 種別: function
  • 定義箇所: server/util_debug.c:192

目的

指定されたリクエストコンテキストにおいて、特定のモジュールに対するログレベルを解決します。モジュール固有のログレベルが設定されていない場合、階層的なフォールバックロジックを用いて適切なデフォルト値を返します。

シグネチャまたは形状

int ap_get_request_module_loglevel(const request_rec *r, int module_index)

入力と出力

  • **入力**:
  • r: リクエスト情報を保持する request_rec 構造体へのポインタ。
  • module_index: ログレベルを取得したいモジュールのインデックス。
  • **出力**:
  • 解決されたログレベル（整数値）。

前提条件と不変条件

  • r は有効な request_rec 構造体を指している必要があります。
  • module_index は、サーバーにロードされているモジュールの有効なインデックスである必要があります。

アルゴリズムまたは振る舞い

1. **ログ設定の特定**: 以下の順序で、利用可能な ap_logconf 構造体（l）を探索します。

  • r->log (リクエスト固有のログ設定)
  • r->connection->log (コネクション固有のログ設定)
  • r->server->log (サーバー全体のログ設定)

    • 2. **フォールバック判定**: 以下のいずれかの条件に該当する場合、モジュール固有のレベルではなく、コンテキストのデフォルトレベル l->level を返します。

  • module_index が負の値である。
  • l->module_levels が NULL である。
  • l->module_levels[module_index] が負の値である。

    • 3. **値の返却**: 上記の条件に該当しない場合、l->module_levels[module_index] に格納されているモジュール固有のログレベルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/log.c:1202 (直接的な呼び出しを確認)
  • include/http_log.h 内のマクロ経由で、サーバー内の複数の箇所から参照されている可能性があります。
  • **呼び出し先**:
  • 本関数は構造体メンバへのアクセスのみを行い、他の関数を呼び出しません。

関連データ構造

  • request_rec: リクエストの状態を保持する構造体。
  • ap_logconf: ログ設定を保持する構造体。module_levels 配列を介してモジュールごとのレベルを管理します。

失敗モードと境界条件

  • r が NULL の場合、構造体メンバへのアクセスによりセグメンテーションフォールトが発生します。
  • module_index が module_levels 配列の範囲外である場合、メモリ境界外アクセスが発生する可能性があります（ソースコード上ではインデックスの妥当性チェックが不十分な可能性があります）。

保守担当者向けメモ

  • 本関数は server/util_debug.c で定義されていますが、http_config.h でマクロとしても定義されており、環境に応じて切り替えられる設計になっています。
  • ログレベルの解決ロジックは ap_get_server_module_loglevel や ap_get_conn_module_loglevel と共通の設計思想に基づいています。
