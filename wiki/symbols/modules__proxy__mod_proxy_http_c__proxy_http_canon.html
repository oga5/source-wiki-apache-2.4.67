proxy_http_canon

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_http.c:81

目的

HTTP/HTTPS プロキシリクエストにおける URL の正規化および解析を行う。ホスト、ポート、パス、クエリ文字列を分解し、プロキシ転送用の内部形式（proxy:scheme://host:port/path?query）へ変換する。

シグネチャまたは形状

static int proxy_http_canon(request_rec *r, char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • url: 正規化対象の URL 文字列。
  • **出力**:
  • 成功時は OK を返し、r->filename に正規化されたプロキシ URL を格納する。
  • 失敗時は DECLINED または HTTP_BAD_REQUEST、HTTP_FORBIDDEN を返す。

前提条件と不変条件

  • get_url_scheme によりスキームが特定可能であること。
  • リバースプロキシの場合、proxy-nocanon または proxy-noencode ノートの設定により、パスのエンコード挙動が制御される。
  • IPv6 リテラルアドレスが含まれる場合、角括弧 [] で囲まれる。

アルゴリズムまたは振る舞い

1. **スキーム判定**: get_url_scheme でスキームと SSL の有無を判定。 2. **ネットワーク位置解析**: ap_proxy_canon_netloc でホストとポートを抽出。 3. **パスとクエリの処理**:

  • PROXYREQ_REVERSE の場合: proxy-nocanon があればそのまま、なければ ap_proxy_canonenc_ex でパスをエンコード。
  • PROXYREQ_PROXY の場合: パスをそのまま使用。

    • 4. **バリデーション**: パスまたはクエリ文字列に制御文字やスペースが含まれる場合は HTTP_FORBIDDEN を返す。 5. **URL 再構築**: 解析結果を連結し、r->filename に格納。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_http.c:2131 (フック登録による呼び出し)
  • modules/mappers/mod_rewrite.c:5092 (コードコメントによる言及)
  • **呼び出し先**:
  • get_url_scheme
  • ap_proxy_canon_netloc
  • ap_proxy_canonenc_ex
  • ap_scan_vchar_obstext

関連データ構造

  • request_rec: Apache のリクエスト状態を保持。
  • core_dir_config: ディレクトリごとの設定（エンコード設定等）。

失敗モードと境界条件

  • スキームが不明な場合は DECLINED。
  • URL 解析エラー時は HTTP_BAD_REQUEST。
  • パスやクエリに不正な文字（制御文字やスペース）が含まれる場合は HTTP_FORBIDDEN。
  • ap_proxy_canonenc_ex が失敗した場合は HTTP_BAD_REQUEST。

保守担当者向けメモ

  • 本関数は proxy_hook_canon_handler を通じて登録されており、プロキシ処理の初期段階で実行される。
  • r->proxyreq の値に基づき処理が分岐するため、フォワードプロキシとリバースプロキシでの挙動の違いに注意すること。
  • IPv6 アドレスの取り扱いにおいて、ポート番号との連結時に括弧の付与が適切に行われているか確認が必要。
