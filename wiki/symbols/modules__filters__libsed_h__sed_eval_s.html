sed_eval_s

定義

  • 種別: struct
  • 定義箇所: modules/filters/libsed.h:109

目的

sed ストリームエディタの実行時状態を保持するためのコンテキスト構造体です。パターンバッファ、ホールドスペース、および実行中のフラグやリソース管理を統合します。

シグネチャまたは形状

struct sed_eval_s {
    sed_err_fn_t   *errfn;
    sed_write_fn_t *writefn;
    void           *data;
    sed_commands_t *commands;
    apr_int64_t    lnum;
    void           *fout;
    apr_size_t     lsize;
    char           *linebuf;
    char           *lspend;
    apr_size_t     hsize;
    char           *holdbuf;
    char           *hspend;
    apr_size_t     gsize;
    char           *genbuf;
    char           *lcomend;
    apr_file_t    *fcode[NWFILES];
    sed_reptr_t    *abuf[SED_ABUFSIZE];
    sed_reptr_t    **aptr;
    sed_reptr_t    *pending;
    unsigned char  *inar;
    int            nrep;
    int            dolflag;
    int            sflag;
    int            jflag;
    int            delflag;
    int            lreadyflag;
    int            quitflag;
    int            finalflag;
    int            numpass;
    int            nullmatch;
    int            col;
    apr_pool_t     *pool;
};

入力と出力

  • **入力**: commands（コンパイル済みスクリプト）、errfn/writefn（コールバック関数）、および処理対象のストリームデータ。
  • **出力**: writefn を通じた処理結果の書き出し、および errfn を通じたエラー通知。

前提条件と不変条件

  • すべてのメモリリソースは pool（apr_pool_t）によって管理されます。
  • linebuf（パターンバッファ）および holdbuf（ホールドスペース）は、実行中に動的にサイズ変更される可能性があります。

アルゴリズムまたは振る舞い

1. **初期化**: sed_init_eval により構造体が初期化され、メモリプールとコマンドリストが関連付けられます。 2. **実行**: sed_eval_buffer 等の関数が本構造体を走査し、commands 内の sed_reptr_t リストを適用します。 3. **状態管理**: linebuf と holdbuf の間でデータを交換し、lnum（行番号）を更新しながらストリーム処理を行います。 4. **終了**: sed_destroy_eval により、関連するメモリプールが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/libsed.h 内で typedef struct sed_eval_s sed_eval_t; として定義されており、libsed を利用するモジュール（mod_sed.c 等）から参照されます。
  • **呼び出し先**: 実行時に errfn および writefn で指定されたコールバック関数を呼び出します。

関連データ構造

  • sed_commands_t: コンパイルされたスクリプト定義。
  • sed_reptr_t: 個々の sed コマンドノード。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • SED_NLINES, SED_DEPTH, SED_LABSIZE などの定数により、スクリプトの複雑さやネストに上限があります。
  • メモリ不足時は apr_pool_t の挙動に依存します。

保守担当者向けメモ

  • 本構造体は libsed の実行エンジンの核となるため、フィールドの追加や変更は sed1.c（実行ロジック）への影響を伴います。
  • fcode 配列は NWFILES で定義された数までファイルハンドルを保持可能です。
