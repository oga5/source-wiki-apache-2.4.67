check_code

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:214

目的

ユーザーが指定した有効期限設定文字列（例: access plus 1 month）を解析し、内部的に処理可能な形式（[AM]<seconds>）へ変換する。

シグネチャまたは形状

char *check_code(apr_pool_t *p, const char *code, char **real_code)

入力と出力

  • **入力**:
  • p: メモリ割り当て用のプール。
  • code: 解析対象の有効期限文字列。
  • real_code: 変換後の文字列を格納するポインタへのポインタ。
  • **出力**:
  • 成功時: NULL を返し、real_code に変換結果（例: A2592000）を格納。
  • 失敗時: エラーメッセージ文字列を返し、real_code は変更されない。

前提条件と不変条件

  • code が A または M で始まる場合、解析を行わずそのまま real_code に代入して終了する（後方互換性維持のため）。
  • code は ap_getword_conf を用いてトークン単位で解析される。
  • num（数値）と type（単位）のペアが順次処理される。

アルゴリズムまたは振る舞い

1. **ベース判定**: 文字列の先頭トークンを解析し、access（または now）なら A、modification なら M を基準時刻として設定する。 2. **オプションのスキップ**: plus という単語があれば読み飛ばす。 3. **数値と単位のループ**:

  • atoi で数値（num）を取得する。
  • 続く単位（years, months, weeks, days, hours, minutes, seconds）を判定し、対応する秒数（factor）を乗算する。
  • 累積値（modifier）に加算する。

    • 4. **結果生成**: 最終的な base と modifier を apr_psprintf で結合し、real_code に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_expires.c:327 (set_expiresbytype 内)
  • modules/metadata/mod_expires.c:341 (set_expiresdefault 内)
  • **呼び出し先**:
  • ap_getword_conf (Apache API: 設定文字列のトークン分割)
  • apr_pstrcat, apr_psprintf (APR API: 文字列操作)
  • strncasecmp, apr_isdigit, atoi (標準 C ライブラリ)

関連データ構造

  • expires_dir_config: 本関数で生成された real_code が格納される設定構造体。

失敗モードと境界条件

  • **不正なベース**: access や modification 以外の単語が指定された場合、エラーメッセージを返す。
  • **数値欠落**: 単位の前に数値がない場合、エラーメッセージを返す。
  • **不正な単位**: 未知の単位が指定された場合、エラーメッセージを返す。
  • **堅牢性**: atoi を使用しているため、非常に大きな数値や不正なフォーマットに対する入力チェックは限定的である。

保守担当者向けメモ

  • 本関数は古いコードベースを継承しており、Cache-Control の詳細な制御には対応していない。
  • ExpiresActive の ACTIVE_DONTCARE 状態は本関数では直接扱わず、呼び出し元で制御される。
