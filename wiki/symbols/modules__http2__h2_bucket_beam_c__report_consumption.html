report_consumption

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:113

目的

受信側スレッドで処理（消費）されたデータ量を送信側に通知し、登録されたコールバックを実行する。

シグネチャまたは形状

static int report_consumption(h2_bucket_beam *beam, int locked)

入力と出力

  • **入力**:
  • beam: 対象の h2_bucket_beam 構造体。
  • locked: 呼び出し元が既に beam->lock を保持しているかを示すフラグ（真であればロックを一時解除してコールバックを実行する）。
  • **出力**:
  • int: コールバックが実行された場合は 1、それ以外は 0 を返す。

前提条件と不変条件

  • beam->recv_bytes が beam->recv_bytes_reported 以上であることを前提とする。
  • コールバック実行中に locked が真の場合、一時的にミューテックスを解放し、終了後に再取得する。

アルゴリズムまたは振る舞い

1. 未報告の消費量 len（recv_bytes - recv_bytes_reported）を算出する。 2. len > 0 の場合:

  • コールバック関数 beam->cons_io_cb が設定されていれば、locked フラグに応じてミューテックスを制御しつつコールバックを実行する。
  • beam->recv_bytes_reported を更新し、報告済みとする。

    • 3. 処理結果（コールバックの有無）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_bucket_beam.c:389
  • modules/http2/h2_bucket_beam.c:584
  • modules/http2/h2_bucket_beam.c:858
  • **呼び出し先**:
  • beam->cons_io_cb (登録されたコールバック関数)
  • apr_thread_mutex_unlock / apr_thread_mutex_lock (同期制御)

関連データ構造

  • h2_bucket_beam: スレッド間データ転送を管理するメイン構造体。
  • h2_beam_io_callback: 消費イベントを処理するコールバック関数の型定義。

失敗モードと境界条件

  • len <= 0 の場合、何も行わず 0 を返す。
  • beam->cons_io_cb が NULL の場合、コールバックは実行されず、報告済みバイト数のみが更新される。

保守担当者向けメモ

  • 本関数は h2_bucket_beam の同期メカニズムの一部であり、受信側から送信側へのフィードバックループを担う。
  • コールバック実行中のロック解除は、デッドロックを回避しつつ、コールバック内でビームの状態変更を許可するために重要である。
