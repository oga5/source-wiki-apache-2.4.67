ap_md5

定義

  • 種別: function
  • 定義箇所: server/util_md5.c:75

目的

入力された文字列から MD5 ハッシュを計算し、16進数形式の文字列として返します。

シグネチャまたは形状

char *ap_md5(apr_pool_t *p, const unsigned char *string)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • string: ハッシュ化対象のヌル終端文字列。
  • **出力**:
  • 戻り値: 計算された MD5 ハッシュ値（16進数文字列）。メモリは p から割り当てられます。

前提条件と不変条件

  • string はヌル終端されている必要があります。
  • 戻り値の文字列は、指定されたメモリプール p の生存期間に依存します。

アルゴリズムまたは振る舞い

1. strlen を使用して入力 string の長さを取得します。 2. ap_md5_binary を呼び出し、バイナリデータとしてのハッシュ計算と 16 進数文字列への変換を委譲します。 3. 計算結果を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • ap_md5_binary (直接呼び出し)
  • strlen (標準ライブラリ)
  • **呼び出し元**:
  • modules/aaa/mod_auth_basic.c
  • modules/aaa/mod_auth_digest.c
  • modules/aaa/mod_auth_form.c
  • server/util_expr_eval.c

関連データ構造

  • apr_pool_t: メモリ管理に使用される APR プール構造体。

失敗モードと境界条件

  • string が NULL の場合、strlen の呼び出しにより未定義の動作が発生します。
  • メモリプール p が枯渇している場合、ap_md5_binary 内部でのメモリ割り当てに失敗する可能性があります。

保守担当者向けメモ

  • 本関数は ap_md5_binary のラッパーであり、文字列の長さを自動的に計算する簡易インターフェースを提供します。
  • 内部で strlen を使用しているため、バイナリデータ（ヌル文字を含む可能性のあるデータ）のハッシュ化には適していません。その場合は ap_md5_binary を直接使用してください。
