example_handler

定義

  • 種別: function
  • 定義箇所: docs/manual/developer/mod_example_2.c:15

目的

Apache HTTP Server のリクエスト処理パイプラインにおいて、example-handler として指定されたリクエストを処理し、対象ファイルの MD5 または SHA1 ハッシュ値を計算して HTML 形式で出力する。

シグネチャまたは形状

static int example_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r（Apache のリクエスト構造体）。クエリパラメータ digest（"md5" または "sha1"）を解析対象とする。
  • **出力**: int 型のステータスコード。
  • OK: 処理成功。
  • DECLINED: ハンドラ名が一致しない場合。
  • HTTP_NOT_FOUND: ファイルが存在しない場合。
  • HTTP_FORBIDDEN: ファイルへのアクセス権限がない場合。

前提条件と不変条件

  • r->handler が "example-handler" であること。
  • 対象ファイルパスは、リクエストされたファイル名から末尾の .sum を除いたものとして解釈される。
  • ファイルはディレクトリではなく、実在するファイルである必要がある。

アルゴリズムまたは振る舞い

1. **ハンドラ判定**: r->handler を確認し、一致しなければ DECLINED を返す。 2. **パス解決**: r->filename から .sum を除外したパスを生成し、apr_stat で存在確認を行う。 3. **フォーム解析**: ap_args_to_table で GET パラメータを解析し、digest の指定を取得する（デフォルトは MD5）。 4. **ハッシュ計算**:

  • ファイルを 256 バイト単位で読み込み、指定されたアルゴリズム（MD5 または SHA1）のコンテキストを更新する。
  • apr_md5_final または apr_sha1_final でダイジェストを確定させる。

    • 5. **レスポンス生成**: ap_rprintf を使用して、ファイルサイズおよび計算されたハッシュ値を HTML 形式でクライアントへ送信する。 6. **終了**: apr_file_close でファイルを閉じ、OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフックシステム（ap_hook_handler により登録）。
  • **呼び出し先**:
  • Apache API: apr_pstrdup, apr_stat, ap_args_to_table, ap_parse_form_data, ap_set_content_type, ap_rprintf, ap_rputs, apr_file_open, apr_file_read, apr_file_close
  • APR ハッシュ関数: apr_md5_init, apr_md5_update, apr_md5_final, apr_sha1_init, apr_sha1_update, apr_sha1_final

関連データ構造

  • request_rec: リクエスト情報およびメモリプール（r->pool）の保持。
  • apr_finfo_t: ファイルメタデータ。
  • apr_md5_ctx_t / apr_sha1_ctx_t: ハッシュ計算用コンテキスト。
  • apr_table_t: クエリパラメータ格納用。

失敗モードと境界条件

  • **404 Not Found**: 対象ファイルが存在しない、またはディレクトリである場合。
  • **403 Forbidden**: apr_stat が失敗した場合（アクセス権限不足等）。
  • **デフォルト動作**: digest パラメータが指定されていない場合は MD5 を選択する。

保守担当者向けメモ

  • 本関数は学習用サンプルであり、ファイル読み込みバッファサイズ（256バイト）は簡略化されている。
  • r->pool を使用してリソースを確保しているため、明示的なメモリ解放は不要（リクエスト終了時に自動解放される）。
  • mod_example_1.c および mod_example_2.c の両方で ap_hook_handler を介して本関数が登録されている。
