slotmem_fgrab

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:267

目的

指定されたスロットメモリインスタンス内の特定のインデックス（スロット）に対し、利用中フラグを強制的にセット（1 に設定）します。

シグネチャまたは形状

apr_status_t slotmem_fgrab(ap_slotmem_instance_t *slot, unsigned int id)

入力と出力

  • **入力**:
  • slot: 操作対象の ap_slotmem_instance_t 構造体へのポインタ。
  • id: 確保対象とするスロットのインデックス番号。
  • **出力**:
  • APR_SUCCESS: 成功。
  • APR_ENOSHMAVAIL: slot が NULL の場合。
  • APR_EINVAL: id がスロットの総数を超えている場合。

前提条件と不変条件

  • slot は有効な ap_slotmem_instance_t 構造体を指している必要があります。
  • id は 0 以上 slot->num 未満である必要があります。
  • 本関数は排他制御を行わないため、マルチスレッド環境で利用する場合は呼び出し側で適切な同期が必要です。

アルゴリズムまたは振る舞い

1. slot が NULL であれば APR_ENOSHMAVAIL を返して終了します。 2. id が slot->num 以上であれば APR_EINVAL を返して終了します。 3. slot->inuse 配列の id 番目の要素を特定し、その値を 1 に書き換えます。 4. APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_balancer.c (エラーログ出力時に参照)
  • modules/slotmem/mod_slotmem_plain.c (プロバイダー構造体への登録)
  • modules/slotmem/mod_slotmem_shm.c (プロバイダー構造体への登録)
  • **呼び出し先**:
  • 特筆すべき関数呼び出しはありません。

関連データ構造

  • ap_slotmem_instance_t: スロットメモリの管理構造体。inuse フラグ配列とスロット数 num を保持します。

失敗モードと境界条件

  • slot が NULL の場合、メモリ保護違反を回避し APR_ENOSHMAVAIL を返します。
  • id が範囲外の場合、メモリ破壊を回避し APR_EINVAL を返します。

保守担当者向けメモ

  • 本関数は「強制確保（Force Grab）」を行うため、既に利用中であるかどうかのチェックを行わずにフラグを上書きします。
  • mod_slotmem_shm.c でも同名の関数が定義されており、プロバイダーインターフェースとして共通化されています。
