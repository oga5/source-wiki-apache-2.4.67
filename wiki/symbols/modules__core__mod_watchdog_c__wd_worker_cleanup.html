wd_worker_cleanup

定義

  • 種別: function
  • 定義箇所: modules/core/mod_watchdog.c:74

目的

Watchdog インスタンスに関連付けられたワーカースレッドを安全に終了させ、登録されているすべてのコールバックに対して STOPPING 状態を通知します。

シグネチャまたは形状

apr_status_t wd_worker_cleanup(void *data)

入力と出力

  • **入力**: data (型: void *) - ap_watchdog_t 構造体へのポインタ。
  • **出力**: apr_status_t - スレッドの終了処理結果（apr_thread_join の戻り値）。

前提条件と不変条件

  • data は有効な ap_watchdog_t 構造体を指している必要があります。
  • スレッドが開始されていない場合（thread_started が 1 以外）、本関数は何もせず APR_SUCCESS を返します。

アルゴリズムまたは振る舞い

1. thread_started アトミック変数をチェックし、未開始であれば即座に終了します。 2. is_running が真の場合、登録されたコールバックリスト（callbacks）を走査します。 3. 各コールバックに対して AP_WATCHDOG_STATE_STOPPING 状態を引数としてコールバック関数を呼び出し、ステータスを APR_EOF に更新します。 4. is_running を 0 に設定します。 5. apr_thread_join を呼び出し、ワーカースレッドの終了を待機します。 6. スレッドの終了ステータスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_watchdog.c:285 にて、apr_pool_pre_cleanup_register を介してプール破棄時のクリーンアップハンドラとして登録されています。
  • **呼び出し先**:
  • apr_atomic_read32
  • 登録されたコールバック関数（wl->callback_fn）
  • apr_thread_join

関連データ構造

  • ap_watchdog_t: 監視インスタンスのメイン構造体。
  • watchdog_list_t: 登録されたコールバック関数のリンクリスト。

失敗モードと境界条件

  • スレッドが既に終了している、あるいは開始に失敗している場合、thread_started のチェックにより安全にスキップされます。
  • apr_thread_join が失敗した場合、そのエラーコードがそのまま呼び出し元に返されます。

保守担当者向けメモ

  • 本関数はプール破棄のフックとして登録されるため、直接呼び出すのではなく、関連するプールが破棄されるタイミングで自動的に実行されます。
  • コールバック実行中にエラーが発生した場合のハンドリングは、各コールバック関数の実装に依存します。
