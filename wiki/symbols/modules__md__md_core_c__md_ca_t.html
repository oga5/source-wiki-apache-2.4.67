md_ca_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_core.c:410

目的

認証局（CA）の名称と、それに対応する ACME サーバーの URL をペアとして保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    const char *name;
    const char *url;
} md_ca_t;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。入力および出力の概念は適用されません。

前提条件と不変条件

  • name および url は、const char * 型の文字列ポインタとして定義されます。
  • modules/md/md_core.c 内の KNOWN_CAs 配列の要素として使用されることが想定されています。

アルゴリズムまたは振る舞い

本構造体自体にアルゴリズムは含まれません。この構造体を利用する md_get_ca_url_from_name や md_get_ca_name_from_url といった関数が、KNOWN_CAs 配列を走査して名称と URL の相互解決を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_core.c:420 にて KNOWN_CAs 配列の定義の一部として参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • KNOWN_CAs: md_ca_t 型の静的配列。既知の CA 情報のマスターリストとして機能します。

失敗モードと境界条件

  • 構造体定義自体に失敗モードはありません。
  • 関連する CA 解決関数において、一致する name や url が見つからない場合の挙動は、各関数の実装に依存します。

保守担当者向けメモ

  • 本構造体は、ACME サーバーの識別子と URL のマッピングを管理する基盤です。新しい CA を追加する場合は、md_core.c 内の KNOWN_CAs 配列を更新してください。
