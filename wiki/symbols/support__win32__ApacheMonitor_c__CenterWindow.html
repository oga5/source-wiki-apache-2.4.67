CenterWindow

定義

  • 種別: function
  • 定義箇所: support/win32/ApacheMonitor.c:484

目的

指定されたウィンドウを、デスクトップの作業領域（ワークエリア）の中央に配置する。

シグネチャまたは形状

BOOL CenterWindow(HWND hwndChild)

入力と出力

  • **入力**: hwndChild (配置対象のウィンドウハンドル)
  • **出力**: BOOL (成功時に TRUE、失敗時に FALSE を返す)

前提条件と不変条件

  • hwndChild は有効なウィンドウハンドルである必要がある。
  • SystemParametersInfo による作業領域の取得が失敗した場合、システム全体の画面サイズ（プライマリモニタ）を基準として計算を行う。

アルゴリズムまたは振る舞い

1. GetWindowRect を使用して、対象ウィンドウの現在の幅と高さを取得する。 2. SystemParametersInfo (SPI_GETWORKAREA) を呼び出し、タスクバー等を除いた作業領域の矩形を取得する。取得に失敗した場合は GetSystemMetrics で画面全体を領域とする。 3. 作業領域の幅・高さからウィンドウのサイズを差し引き、中央座標 (xNew, yNew) を算出する。 4. SetWindowPos を呼び出し、ウィンドウを算出された座標へ移動させる。この際、SWP_NOSIZE フラグによりサイズ変更は行わず、SWP_SHOWWINDOW により表示状態を維持する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/win32/ApacheMonitor.c:866
  • support/win32/ApacheMonitor.c:990
  • **呼び出し先**:
  • GetWindowRect (Win32 API)
  • SystemParametersInfo (Win32 API)
  • GetSystemMetrics (Win32 API)
  • SetWindowPos (Win32 API)

関連データ構造

  • HWND: ウィンドウハンドル
  • RECT: ウィンドウおよび作業領域の座標保持に使用

失敗モードと境界条件

  • SystemParametersInfo が失敗した場合、作業領域の取得に失敗するが、GetSystemMetrics を用いたフォールバック処理により、画面全体を基準とした中央配置が試行される。
  • SetWindowPos が失敗した場合、その戻り値がそのまま関数の結果として返される。

保守担当者向けメモ

  • 本関数はダイアログボックスの表示位置を調整するために使用されている。
  • マルチモニタ環境における挙動については、SystemParametersInfo の仕様に依存する。
