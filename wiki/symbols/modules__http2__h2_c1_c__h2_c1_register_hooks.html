h2_c1_register_hooks

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1.c:338

目的

HTTP/2 プロトコルの接続処理に必要な Apache フックを登録し、接続ライフサイクルにおける本モジュールの実行タイミングを制御する。

シグネチャまたは形状

void h2_c1_register_hooks(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として Apache のフックテーブルにコールバック関数を登録）

前提条件と不変条件

  • mod_ssl の接続フィルタが利用可能であること。
  • mod_reqtimeout がロードされている場合、その後に実行される必要がある。

アルゴリズムまたは振る舞い

1. ap_hook_process_connection を呼び出し、h2_c1_hook_process_connection を登録する。

  • mod_reqtimeout の後に実行されるよう指定し、APR_HOOK_LAST を用いて HTTP/1.1 のコア処理より前に割り込めるように配置する。

    • 2. ap_hook_pre_close_connection を呼び出し、h2_c1_hook_pre_close を登録する。

  • mod_ssl の後に実行されるよう指定し、接続終了時のクリーンアップを確実に行う。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/mod_http2.c:200（モジュール初期化時）
  • 呼び出し先:
  • ap_hook_process_connection
  • ap_hook_pre_close_connection
  • h2_c1_hook_process_connection（登録対象）
  • h2_c1_hook_pre_close（登録対象）

関連データ構造

  • conn_rec (Apache HTTP Server コアの接続レコード)

失敗モードと境界条件

  • 本関数自体はエラーを返さないが、フック登録に失敗した場合、Apache の起動プロセスまたは接続処理が期待通りに動作しない可能性がある。

保守担当者向けメモ

  • reqtimeout は HTTP/1.1 特有のタイムアウト処理を行うため、HTTP/2 接続確立時に競合しないよう、本フックの実行順序（APR_HOOK_LAST）は重要である。
  • mod_ssl よりも後に実行する必要があるため、依存関係の順序変更には注意が必要。
