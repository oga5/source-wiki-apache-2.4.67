safe_file_rename

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:135

目的

一時ファイルから最終的なキャッシュファイルパスへ、ディレクトリ構造の欠落を考慮したリトライ付きのリネーム処理を行う。

シグネチャまたは形状

static apr_status_t safe_file_rename(disk_cache_conf *conf,
                                     const char *src, const char *dest,
                                     apr_pool_t *pool)

入力と出力

  • **入力**:
  • conf: ディスクキャッシュの設定情報。
  • src: リネーム元の一時ファイルパス。
  • dest: リネーム先の最終的なキャッシュファイルパス。
  • pool: メモリ割り当てに使用する APR プール。
  • **出力**:
  • apr_status_t: 操作の成功（APR_SUCCESS）または失敗を示すステータスコード。

前提条件と不変条件

  • src は既に存在し、書き込み可能な一時ファイルであること。
  • dest の親ディレクトリは、htcacheclean 等の外部プロセスによって削除されている可能性がある。

アルゴリズムまたは振る舞い

1. apr_file_rename を呼び出し、リネームを試みる。 2. 成功した場合は直ちに終了する。 3. 失敗した場合、最大2回まで以下のリトライ処理を行う。

  • 0.001秒間待機する。
  • mkdir_structure を呼び出し、dest に必要なディレクトリ構造を再作成する。
  • 再度 apr_file_rename を試みる。

    • 4. 最終的なリネーム結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache_disk.c:174（file_cache_el_final 内から呼び出される）。
  • **呼び出し先**:
  • apr_file_rename
  • apr_sleep
  • mkdir_structure

関連データ構造

  • disk_cache_conf: キャッシュ設定を保持する構造体。

失敗モードと境界条件

  • apr_file_rename が権限不足やファイルシステムエラーで失敗し続ける場合、最終的にエラーコードが返される。
  • mkdir_structure が失敗した場合、リネームは行われず、ループの次の反復へ進む。

保守担当者向けメモ

  • 本関数は、htcacheclean がディレクトリを削除する競合状態を回避するために設計されている。
  • 待機時間は固定の1ミリ秒であり、高負荷環境下での挙動については注意が必要である。
