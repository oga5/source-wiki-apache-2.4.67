request_is_filter_present

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:270

目的

指定されたフィルタが、現在のリクエストの入力フィルタチェーン内に存在するかどうかを確認する。

シグネチャまたは形状

static int request_is_filter_present(request_rec *r, ap_filter_rec_t *fn)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体へのポインタ。
  • fn: 検索対象となるフィルタのレコード定義 (ap_filter_rec_t) へのポインタ。
  • **出力**:
  • 1: 指定されたフィルタがチェーン内に存在する場合。
  • 0: 指定されたフィルタがチェーン内に存在しない場合。

前提条件と不変条件

  • r は有効な request_rec 構造体である必要がある。
  • 入力フィルタチェーン (r->input_filters) は、NULL で終了する連結リストとして構成されている。

アルゴリズムまたは振る舞い

1. r->input_filters から開始し、フィルタチェーンを順次走査する。 2. 各フィルタノードの frec メンバが、引数 fn と一致するかを比較する。 3. 一致するフィルタが見つかった時点で走査を終了し、1 を返す。 4. チェーンの末尾 (NULL) に到達するまで一致が見つからなければ、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_request.c:299
  • modules/filters/mod_request.c:305
  • **呼び出し先**:
  • 本関数は、フィルタチェーンの構造体 (ap_filter_t) を走査するのみであり、他の関数を呼び出さない。

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • ap_filter_t: フィルタチェーンの各ノードを構成する構造体。
  • ap_filter_rec_t: フィルタの定義情報を保持する構造体。

失敗モードと境界条件

  • r->input_filters が NULL（フィルタが未登録）の場合、ループは実行されず、正しく 0 を返す。
  • フィルタチェーンが空でない場合でも、対象のフィルタが登録されていなければ 0 を返す。

保守担当者向けメモ

  • 本関数は、特定のフィルタが既にチェーンに挿入されているかを判定することで、フィルタの重複登録を避けるためのガード条件として使用されている。
  • フィルタの比較は f->frec ポインタの直接比較で行われるため、フィルタ定義が正しく登録されていることが前提となる。
