ap_proxy_header_reverse_map_fn

定義

  • 種別: typedef（関数ポインタ型）
  • 定義箇所: modules/proxy/mod_proxy.h:733

目的

バックエンドサーバーから受信した特定の HTTP レスポンスヘッダー値を、プロキシ設定（ProxyPassReverse 等）に基づいてクライアント向けに書き換えるためのコールバック関数型です。

シグネチャまたは形状

typedef const char *(*ap_proxy_header_reverse_map_fn)(request_rec *r, 
                                                      char *str, 
                                                      const char *rem, 
                                                      const char *loc);

※ mod_proxy.h の定義に基づく。

入力と出力

  • **入力**:
  • request_rec *r: 現在のリクエスト情報。
  • char *str: 変換対象となるヘッダー値の文字列。
  • const char *rem: リモート（バックエンド）のパスまたはホスト情報。
  • const char *loc: ローカル（プロキシ）のパスまたはホスト情報。
  • **出力**:
  • 変換後の新しいヘッダー値（const char *）。変換が不要な場合は元の値や NULL が返される。

前提条件と不変条件

  • mod_proxy_http.c 内の transform_hdrs 配列において、特定のヘッダー名（Location, Content-Location, URI, Destination, Set-Cookie）とペアで登録されています。
  • 呼び出し側は、この関数が返す文字列がリクエストの生存期間中有効であることを期待します。

アルゴリズムまたは振る舞い

  • mod_proxy_http.c では、バックエンドからのレスポンスヘッダーを処理する際に、transform_hdrs テーブルを走査し、一致するヘッダー名が見つかった場合にこの関数ポインタを介して変換ロジック（ap_proxy_location_reverse_map や ap_proxy_cookie_reverse_map）を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_http.c 内のヘッダー処理ロジック（process_proxy_header 等）から参照されます。また、modules/http2/h2_proxy_session.c でも参照されています。
  • **呼び出し先**: 具体的な実装として ap_proxy_location_reverse_map および ap_proxy_cookie_reverse_map が割り当てられます。

関連データ構造

  • transform_hdrs: mod_proxy_http.c で定義された、ヘッダー名と本関数ポインタを保持する構造体配列。

失敗モードと境界条件

  • 変換対象のヘッダー値が NULL または空文字列の場合の挙動は、各実装関数に依存します。
  • 変換ルールに一致しない場合、関数は元の値をそのまま返すか、変換を行わない設計となっています。

保守担当者向けメモ

  • 本シンボルは mod_proxy.h で定義されたインターフェースであり、新しいヘッダー変換ルールを追加する場合は transform_hdrs 配列に新しいエントリと適切な関数を追加してください。
  • __stdcall 呼び出し規約が指定されている定義も存在するため、プラットフォーム固有のバイナリ互換性に注意してください。
