get_content_length

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ajp.c:146

目的

HTTPリクエストの Content-Length ヘッダー値を解析し、apr_off_t 型の数値として取得する。

シグネチャまたは形状

static apr_off_t get_content_length(request_rec * r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: apr_off_t 型のバイト長。
  • 値が正常に取得できた場合はその数値。
  • Content-Length が存在しない場合は 0。
  • 解析エラーが発生した場合は -1。

前提条件と不変条件

  • r->main == NULL であること（サブクエストではないメインリクエストであること）が処理の条件。
  • r->headers_in テーブル内に Content-Length ヘッダーが存在する必要がある。

アルゴリズムまたは振る舞い

1. r->main が NULL であるかを確認する。 2. r->headers_in から Content-Length ヘッダーの値を取得する。 3. ヘッダーが存在する場合、ap_parse_strict_length を使用して文字列を apr_off_t に変換する。 4. 変換に失敗した場合は、エラーを示す -1 を返す。 5. 上記以外（ヘッダー不在など）の場合は、初期値である 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ajp.c:291 にて呼び出されている。
  • **呼び出し先**: apr_table_get および ap_parse_strict_length を呼び出す。

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • apr_off_t: ファイルサイズやオフセットを表すための APR 型。

失敗モードと境界条件

  • ap_parse_strict_length が失敗した場合、戻り値は -1 となる。
  • r->main が NULL でない場合、この関数はヘッダーの解析を行わず、初期値の 0 を返す。

保守担当者向けメモ

  • 本関数は mod_proxy_ajp 内でのみ使用される静的関数である。
  • ap_parse_strict_length の仕様変更や、Content-Length の解釈に関する厳密なルール変更があった場合に影響を受ける可能性がある。
