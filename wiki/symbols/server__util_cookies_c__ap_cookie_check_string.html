ap_cookie_check_string

定義

  • 種別: function
  • 定義箇所: server/util_cookies.c:283

目的

クッキー名や値として使用される文字列が、HTTP仕様上安全であるかを検証します。

シグネチャまたは形状

apr_status_t ap_cookie_check_string(const char *string)

入力と出力

  • **入力**: string (検証対象の文字列)
  • **出力**: apr_status_t 型のステータスコード
  • APR_SUCCESS: 文字列が安全である場合。
  • APR_EGENERAL: 文字列が不正である場合。

前提条件と不変条件

  • string が NULL または空文字列である場合、不正とみなされます。

アルゴリズムまたは振る舞い

1. 入力文字列 string が NULL または空（*string == '\0'）でないかを確認します。 2. 文字列内に以下の予約文字が含まれていないか走査します。

  • = (等号)
  • & (アンパサンド)
  • ; (セミコロン)

    • 3. 上記のいずれかの条件に該当する場合、APR_EGENERAL を返します。 4. いずれにも該当しない場合、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_dbd.c:538 からの呼び出しが確認されています。
  • **呼び出し先**: ap_strchr_c を使用して文字検索を行います。

関連データ構造

  • apr_status_t (APRのステータスコード)

失敗モードと境界条件

  • string が NULL の場合、即座に APR_EGENERAL を返します。
  • 空文字列の場合、即座に APR_EGENERAL を返します。
  • 予約文字（=, &, ;）が含まれる場合、検証は失敗します。

保守担当者向けメモ

  • 本関数はクッキーの構造を破壊する可能性のある文字を排除するために使用されます。
  • 文字列のメモリ管理は呼び出し元に依存します。
