add_charset_source

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_charset_lite.c:140

目的

Apache 設定ファイルで指定された CharsetSourceEnc ディレクティブの値を、ディレクトリごとの設定構造体 (charset_dir_t) に格納します。

シグネチャまたは形状

static const char *add_charset_source(cmd_parms *cmd, void *in_dc, const char *name)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • in_dc: 設定対象のディレクトリ設定構造体 (charset_dir_t)。
  • name: 設定ファイルから読み取られた文字セット名（文字列）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • in_dc は charset_dir_t 型としてキャスト可能である必要があります。
  • name は有効な文字セット名であると想定されます。

アルゴリズムまたは振る舞い

1. 引数 in_dc を charset_dir_t *dc にキャストします。 2. dc->charset_source メンバに、引数 name で渡された文字セット名を代入します。 3. NULL を返して処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_charset_lite.c:1104 (コマンドテーブル定義内での参照を確認)。
  • **呼び出し先**: 本関数から直接呼び出している関数はありません。

関連データ構造

  • charset_dir_t: ディレクトリ単位の変換設定を保持する構造体。本関数はこの構造体の charset_source フィールドを更新します。

失敗モードと境界条件

  • 本関数内でのエラーチェックは行われておらず、常に NULL を返します。

保守担当者向けメモ

  • 本関数は Apache の設定ディレクティブハンドラとして動作します。
  • charset_dir_t 構造体の定義については、mod_charset_lite.c 内の該当構造体定義を参照してください。
