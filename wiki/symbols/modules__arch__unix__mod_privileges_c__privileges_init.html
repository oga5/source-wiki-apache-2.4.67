privileges_init

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_privileges.c:389

目的

Apache HTTP Server の起動時（pre_config フェーズ）において、現在使用されている MPM（Multi-Processing Module）がスレッドセーフであるかを確認し、本モジュールの動作要件を満たしているかを検証する。

シグネチャまたは形状

static int privileges_init(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • pconf, plog, ptemp: Apache のメモリ管理用プール。
  • **出力**:
  • OK (0): MPM が非スレッド環境であり、初期化が正常に完了した場合。
  • !OK (非ゼロ): MPM がスレッド環境であり、モジュールが使用不可と判断された場合。

前提条件と不変条件

  • 本モジュールは setppriv を使用する性質上、マルチスレッド環境（Worker/Event MPM 等）と互換性がありません。
  • ap_mpm_query を用いて MPM の特性を照会できる必要があります。

アルゴリズムまたは振る舞い

1. ap_mpm_query を呼び出し、AP_MPMQ_IS_THREADED フラグを通じて MPM がスレッド対応かどうかを確認する。 2. 照会に失敗した場合、エラーログ（APLOG_NOTICE）を出力する。 3. MPM がスレッド対応である場合、致命的エラー（APLOG_CRIT）をログ出力し、!OK を返して起動を中断する。 4. スレッド対応でないことが確認できた場合、OK を返して処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/unix/mod_privileges.c:416 にて ap_hook_pre_config に登録されており、Apache の設定読み込みフェーズで呼び出される。
  • **呼び出し先**: ap_mpm_query（Apache MPM API）、ap_log_perror（Apache ロギング API）。

関連データ構造

  • apr_pool_t: Apache のメモリ管理用プール構造体。

失敗モードと境界条件

  • **MPM 照会失敗**: ap_mpm_query が APR_SUCCESS 以外を返した場合、警告ログを出力するが、スレッド状態の判定は継続される。
  • **スレッド環境**: MPM がスレッド対応の場合、モジュールは起動を拒否し、サーバーの起動プロセスを停止させる。

保守担当者向けメモ

  • 本関数は mod_privileges の動作における「ガードレール」として機能します。将来的にマルチスレッド対応の権限分離手法が導入されない限り、このチェックは必須です。
  • ログ出力には APLOGNO(02153) および APLOGNO(02154) が割り当てられています。ログ解析時にはこれらの ID を参照してください。
