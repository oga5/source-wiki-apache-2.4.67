ssl_register_hooks

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:713

目的

Apache HTTP Server の SSL/TLS モジュールにおける各種フック関数、認証プロバイダ、およびオプションフックを登録し、リクエスト処理パイプラインに mod_ssl を統合する。

シグネチャまたは形状

static void ssl_register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p (モジュール初期化用のメモリプール)
  • **出力**: なし（副作用として Apache のフックテーブルにコールバック関数を登録）

前提条件と不変条件

  • 本関数は Apache のモジュールロード時に呼び出される。
  • mod_setenvif や mod_proxy との実行順序を制御するため、静的な依存関係配列（pre_prr, b_pc）が定義されている。

アルゴリズムまたは振る舞い

1. **I/O フィルタ登録**: ssl_io_filter_register(p) を呼び出し、SSL フィルタを登録する。 2. **フック登録**: ap_hook_* を使用し、接続処理、設定テスト、初期化、認証、リクエスト処理の各フェーズに mod_ssl の関数を割り当てる。

  • pre_connection, process_connection, post_config 等の主要フックを APR_HOOK_MIDDLE で登録。
  • post_read_request は mod_setenvif の後に実行されるよう順序を制御。

    • 3. **プロバイダ登録**: ap_register_auth_provider を使用し、ssl および ssl-verify-client 認証プロバイダを登録する。 4. **オプションフック**: APR_OPTIONAL_HOOK を介して proxy 関連のフックを登録する。 5. **変数登録**: ssl_var_register(p) を呼び出し、SSL 関連変数のルックアップ機能を有効化する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c 内のモジュール定義構造体（module）から参照される。
  • **呼び出し先**:
  • ssl_io_filter_register, ssl_var_register
  • 各種フック関数（ssl_hook_pre_connection, ssl_hook_process_connection, ssl_init_Module 等）
  • ap_hook_* 系 API および ap_register_auth_provider

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • module 構造体: 本関数を保持する Apache モジュール定義。

失敗モードと境界条件

  • 内部で呼び出される ap_hook_* や ap_register_auth_provider が失敗した場合の挙動は Apache Core のエラーハンドリングに依存する。
  • 依存するモジュール（mod_proxy.c 等）がロードされていない場合、オプションフックの登録は無視されるか、実行時に解決される。

保守担当者向けメモ

  • 本関数は mod_ssl のエントリポイントの一部であり、フックの実行順序を変更する際は APR_HOOK_* の優先度および依存関係配列（pre_prr, b_pc）を慎重に調整すること。
  • 新しい認証機能やリクエスト処理フェーズを追加する場合は、ここで適切に ap_hook_* を追加する必要がある。
