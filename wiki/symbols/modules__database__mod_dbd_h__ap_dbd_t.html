ap_dbd_t

定義

  • 種別: struct
  • 定義箇所: modules/database/mod_dbd.h:76

目的

Apache HTTP Server の mod_dbd において、データベース接続ハンドル、ドライバ情報、および準備済みステートメントを統合管理するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_dbd_t *handle;
    const apr_dbd_driver_t *driver;
    apr_hash_t *prepared;
    apr_pool_t *pool;
} ap_dbd_t;

入力と出力

  • **入力**: 構造体の各フィールドは、mod_dbd の接続プール管理ロジックによって初期化されます。
  • **出力**: データベース操作を行う他のモジュール（mod_authn_dbd 等）に対し、有効なデータベース接続とプリペア済みクエリへのアクセスを提供します。

前提条件と不変条件

  • handle は apr_dbd レイヤーの有効な接続を指している必要があります。
  • prepared ハッシュテーブルは、サーバー起動時に登録された SQL ステートメントを保持します。
  • pool は、この接続ハンドルに関連付けられたメモリ管理用プールです。

アルゴリズムまたは振る舞い

  • 接続のライフサイクル管理: ap_dbd_acquire() を通じて取得された場合、リクエストのライフサイクルに紐付き、リクエスト終了時に自動的に解放されます。
  • ステートメント管理: prepared ハッシュテーブルを介して、SQL ステートメントの事前準備と再利用を効率化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authn_dbd.c: 認証処理において接続を取得。
  • modules/aaa/mod_authz_dbd.c: 認可処理において接続を取得。
  • modules/database/mod_dbd.c: 接続プールの構築および管理ロジック内で使用。
  • **呼び出し先**:
  • apr_dbd ライブラリ（handle および driver を介した低レベル操作）。

関連データ構造

  • apr_dbd_t: apr_dbd が提供する低レベルのデータベース接続ハンドル。
  • apr_dbd_driver_t: データベースドライバ固有の関数ポインタ群。
  • dbd_cfg_t: サーバー単位のデータベース設定を保持する構造体。

失敗モードと境界条件

  • 接続の取得失敗: データベースサーバーがダウンしている場合やプールが枯渇している場合、handle が NULL になる可能性があります。
  • メモリ不足: pool の割り当てやハッシュテーブルの生成時に APR エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dbd の中核であり、API の変更は依存する認証・認可モジュールに直接影響します。
  • ap_dbd_acquire で取得した接続はリクエスト終了時に自動解放されるため、呼び出し側で明示的なクローズを行わないよう注意してください。
