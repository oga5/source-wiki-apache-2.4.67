cache_object_t

定義

  • 種別: struct (不完全型)
  • 定義箇所: modules/cache/mod_cache.h:81

目的

キャッシュシステムにおいて、個別のキャッシュエントリを表現するための抽象的なオブジェクトハンドルです。

シグネチャまたは形状

typedef struct cache_object cache_object_t;

※本ヘッダー内では不完全型として定義されており、具体的なメンバ構成は各ストレージプロバイダの実装に依存します。

入力と出力

  • 入力: apr_pcalloc 等によるメモリ割り当ての対象となります。
  • 出力: キャッシュストレージ（ディスクや共有メモリ）と連携するためのハンドルとして機能します。

前提条件と不変条件

  • mod_cache_disk や mod_cache_socache などのストレージプロバイダによって、インスタンス化および初期化が行われる必要があります。
  • 構造体内部の vobj メンバは、各ストレージ固有のオブジェクトを指すために使用されます。

アルゴリズムまたは振る舞い

  • キャッシュの検索、保存、および取得処理において、特定のキャッシュエントリを識別・保持するために使用されます。
  • next メンバを介して、複数のキャッシュオブジェクトを連結するリスト構造の一部として利用されることがあります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_cache_disk.c: キャッシュエントリの割り当ておよび初期化時に使用。
  • mod_cache_socache.c: キャッシュエントリの割り当ておよび初期化時に使用。
  • **呼び出し先**:
  • 特定の関数呼び出しはソースコード上では直接確認できませんが、各ストレージプロバイダの実装内で、vobj を通じてストレージ固有のデータ構造と相互作用します。

関連データ構造

  • cache_object: 本構造体の実体（定義元）。
  • cache_handle_t: キャッシュエントリへのアクセスを管理する上位のハンドル構造体。
  • disk_cache_object_t / cache_socache_object_t: 各ストレージプロバイダが vobj に格納する具体的なデータ構造。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc 等による割り当て失敗が発生する可能性があります。
  • 不完全型であるため、定義が完了していない状態でメンバに直接アクセスしようとするとコンパイルエラーとなります。

保守担当者向けメモ

  • 本シンボルは mod_cache のコアインターフェースの一部です。具体的なメンバ構成を確認する場合は、mod_cache_disk.c や mod_cache_socache.c などの実装ファイルを参照してください。
  • vobj メンバは、ストレージプロバイダ間で共通のインターフェースを維持しつつ、実装固有のデータを保持するための拡張ポイントとして機能しています。
