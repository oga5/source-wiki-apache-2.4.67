ShowTryServicesMenu

定義

  • 種別: function
  • 定義箇所: support/win32/ApacheMonitor.c:455

目的

Apache サービスの一覧を表示するポップアップメニューを生成し、ユーザーがサービスの状態を操作するためのインターフェースを提供します。

シグネチャまたは形状

void ShowTryServicesMenu(HWND hWnd)

入力と出力

  • **入力**: hWnd (メニューの親ウィンドウハンドル)
  • **出力**: なし（副作用としてメニューの表示と操作の実行）

前提条件と不変条件

  • グローバル配列 g_stServices が初期化されており、監視対象のサービス情報が格納されている必要があります。
  • サービス名 (szServiceName) が NULL である要素が配列の終端となります。

アルゴリズムまたは振る舞い

1. CreatePopupMenu を呼び出し、メニューハンドルを作成します。 2. g_stServices を走査し、各サービスに対して appendServiceMenu を呼び出してメニュー項目を追加します。 3. サービスが一つでも存在する場合、SetForegroundWindow で親ウィンドウを前面に出し、現在のカーソル位置 (GetCursorPos) に TrackPopupMenu を表示します。 4. メニュー操作終了後、DestroyMenu を呼び出してリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: support/win32/ApacheMonitor.c:1351
  • **呼び出し先**: CreatePopupMenu, appendServiceMenu, SetForegroundWindow, GetCursorPos, TrackPopupMenu, DestroyMenu (すべて Windows API)

関連データ構造

  • ST_APACHE_SERVICE: サービス名、表示名、PID を保持する構造体。
  • g_stServices[]: 監視対象サービスを保持するグローバル配列。

失敗モードと境界条件

  • g_stServices が空の場合、メニューは表示されません。
  • CreatePopupMenu が失敗した場合、メニューの生成と表示は行われません。
  • SetForegroundWindow が失敗した場合、フォールバックとして SetForegroundWindow(NULL) が呼び出されます。

保守担当者向けメモ

  • 本関数は g_stServices の固定長配列（最大 MAX_APACHE_SERVICES）に依存しています。
  • メニュー項目追加時の appendServiceMenu 呼び出しにおいて、dwPid != 0 を判定基準としてサービスの稼働状態をメニューに反映しています。
