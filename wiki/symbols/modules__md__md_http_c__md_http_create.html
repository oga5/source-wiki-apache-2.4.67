md_http_create

定義

  • 種別: function
  • 定義箇所: modules/md/md_http.c:61

目的

HTTP 通信セッションを管理するための md_http_t インスタンスを生成・初期化します。

シグネチャまたは形状

apr_status_t md_http_create(md_http_t **phttp, apr_pool_t *p, const char *user_agent, const char *proxy_url)

入力と出力

  • **入力**:
  • phttp: 生成されたセッションインスタンスを格納するポインタ。
  • p: メモリ割り当てに使用する APR プール。
  • user_agent: HTTP リクエストに使用する User-Agent 文字列。
  • proxy_url: プロキシサーバーの URL（任意）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、実装が未登録の場合は APR_ENOTIMPL、メモリ不足等のエラー時は対応するエラーコードを返します。

前提条件と不変条件

  • cur_impl が md_http_use_implementation() 等を通じて事前に登録されている必要があります。
  • p は有効な APR プールである必要があります。

アルゴリズムまたは振る舞い

1. **実装の確認**: グローバルな cur_impl が存在するか確認し、なければ APR_ENOTIMPL を返します。 2. **初期化**: 実装が未初期化であれば cur_impl->init() を呼び出し、成功時に cur_init_done フラグを立てます。 3. **メモリ確保**: 指定されたプール p から md_http_t 構造体分のメモリを確保します。 4. **設定のコピー**: user_agent および proxy_url をプール内に複製して保持します。 5. **リソース確保**: apr_bucket_alloc_create(p) を使用してバケットアロケータを生成します。 6. **クリーンアップ登録**: apr_pool_cleanup_register を使用し、プール破棄時に http_cleanup が実行されるよう登録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c:792
  • modules/md/md_http.c:100
  • modules/md/md_ocsp.c:916
  • **呼び出し先**:
  • cur_impl->init() (実装依存の初期化関数)
  • apr_pcalloc, apr_pstrdup, apr_bucket_alloc_create, apr_pool_cleanup_register (APR API)
  • http_cleanup (クリーンアップ用コールバック)

関連データ構造

  • md_http_t: セッション状態を保持するメイン構造体。
  • md_http_impl_t: 通信バックエンドの実装定義。

失敗モードと境界条件

  • cur_impl が未設定の場合: APR_ENOTIMPL を返します。
  • cur_impl->init() が失敗した場合: そのエラーコードを返します。
  • バケットアロケータの生成失敗時: APR_EGENERAL を返します。

保守担当者向けメモ

  • 本関数は md_http_t のライフサイクル管理の起点です。メモリ管理は渡されたプール p に依存するため、呼び出し側は適切な生存期間を持つプールを指定してください。
  • 実装の切り替えはプロセス全体で共有される cur_impl を介して行われるため、マルチスレッド環境での初期化順序には注意が必要です。
