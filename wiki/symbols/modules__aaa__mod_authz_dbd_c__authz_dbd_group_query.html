authz_dbd_group_query

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbd.c:212

目的

データベース（DBD）から特定のユーザーが所属するグループリストを取得し、認可判断のためのデータとして提供します。

シグネチャまたは形状

static int authz_dbd_group_query(request_rec *r, authz_dbd_cfg *cfg, apr_array_header_t *groups)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • cfg: データベースクエリ設定を含む authz_dbd_cfg 構造体。
  • groups: 取得したグループ名を格納するための apr_array_header_t 配列。
  • **出力**:
  • 成功時は OK を返します。
  • 失敗時は HTTP_INTERNAL_SERVER_ERROR を返します。

前提条件と不変条件

  • cfg->query が設定されている必要があります。
  • データベース接続（dbd_handle(r)）が有効である必要があります。
  • 取得したグループ名は r->pool に複製され、リクエストの生存期間中保持されます。

アルゴリズムまたは振る舞い

1. 設定されたクエリが存在するか確認し、なければエラーを返します。 2. dbd_handle(r) を介してデータベース接続を取得します。 3. cfg->query をキーとして、mod_dbd に登録済みのプリペアドステートメントを取得します。 4. apr_dbd_pvselect を実行し、現在のユーザー名（r->user）をバインドしてクエリを発行します。 5. 結果セットを反復処理し、各行の最初のカラム（グループ名）を groups 配列に追加します。 6. 処理中にエラーが発生した場合は、apr_dbd_error を用いてログを出力し、エラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: dbdgroup_check_authorization (modules/aaa/mod_authz_dbd.c:294)
  • **呼び出し先**:
  • dbd_handle (内部ヘルパー)
  • apr_hash_get
  • apr_dbd_pvselect
  • apr_dbd_get_row
  • apr_dbd_get_entry
  • apr_pstrdup
  • ap_log_rerror

関連データ構造

  • request_rec: リクエスト情報およびメモリプール管理。
  • authz_dbd_cfg: 認可設定（クエリ文字列等）。
  • ap_dbd_t: データベース接続ハンドル。
  • apr_array_header_t: 取得したグループ名のリスト。

失敗モードと境界条件

  • **クエリ未設定**: cfg->query が NULL の場合、HTTP_INTERNAL_SERVER_ERROR を返します。
  • **DB 接続失敗**: dbd_handle が NULL を返す場合、HTTP_INTERNAL_SERVER_ERROR を返します。
  • **ステートメント未登録**: プリペアドステートメントが見つからない場合、HTTP_INTERNAL_SERVER_ERROR を返します。
  • **クエリ実行失敗**: データベース側でエラーが発生した場合、エラーメッセージをログに記録し HTTP_INTERNAL_SERVER_ERROR を返します。

保守担当者向けメモ

  • 本関数は mod_dbd のプリペアドステートメント機能に強く依存しています。
  • メモリ管理はすべて r->pool に依存しているため、リクエスト終了時に自動的に解放されます。
  • apr_dbd_get_row のループ処理において、データベースドライバ固有のエラーハンドリングが組み込まれています。
