util_ald_destroy_cache

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache_mgr.c:416

目的

LDAP キャッシュ構造体（util_ald_cache_t）を破棄し、キャッシュ内に保持されているすべてのエントリおよびハッシュテーブル領域を解放する。

シグネチャまたは形状

void util_ald_destroy_cache(util_ald_cache_t *cache)

入力と出力

  • **入力**: cache - 破棄対象の util_ald_cache_t 構造体へのポインタ。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • cache が NULL の場合、関数は何もせず即座に復帰する。
  • キャッシュ内の各エントリは、キャッシュ構造体内に定義された free 関数ポインタを使用して解放される必要がある。

アルゴリズムまたは振る舞い

1. cache が NULL であれば終了する。 2. キャッシュのハッシュテーブル（nodes 配列）を全バケット分走査する。 3. 各バケット内の連結リストを辿り、以下の順序でメモリを解放する。

  • cache->free を呼び出し、ノードのペイロードを解放する。
  • util_ald_free を呼び出し、ノード構造体自体を解放する。

    • 4. 全バケットの走査完了後、ハッシュテーブル配列自体を util_ald_free で解放する。 5. 最後にキャッシュ構造体本体を util_ald_free で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap_cache.c: キャッシュノードの破棄時およびモジュール終了処理時に呼び出される。
  • modules/ldap/util_ldap_cache_mgr.c: キャッシュ生成失敗時のクリーンアップ等で呼び出される。
  • **呼び出し先**:
  • cache->free (関数ポインタ): ペイロードの解放。
  • util_ald_free: ノード、ハッシュテーブル配列、およびキャッシュ構造体本体のメモリ解放。

関連データ構造

  • util_ald_cache_t: キャッシュの管理構造体。
  • util_cache_node_t: キャッシュ内の各エントリを保持する連結リストノード。

失敗モードと境界条件

  • cache が NULL の場合は安全に無視される。
  • キャッシュが空（nodes が初期化されていない、または全バケットが NULL）の場合でも、ループ処理により正しく終了する。

保守担当者向けメモ

  • 本関数はキャッシュの全メモリを解放するため、呼び出し後に cache ポインタを参照することは不可。
  • cache->free は、キャッシュ作成時に指定された適切な解放関数を指している必要がある。
