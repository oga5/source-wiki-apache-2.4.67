serialize_request

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ocsp.c:30

目的

OCSP リクエストオブジェクトを、HTTP/1.0 POST メッセージ形式のバイナリデータ（BIO）へシリアライズします。

シグネチャまたは形状

static BIO *serialize_request(OCSP_REQUEST *req, const apr_uri_t *uri, const apr_uri_t *proxy_uri)

入力と出力

  • **入力**:
  • req: シリアライズ対象の OCSP_REQUEST オブジェクト。
  • uri: OCSP レスポンダの宛先 URI 情報。
  • proxy_uri: プロキシ経由の場合のプロキシ URI 情報（任意）。
  • **出力**:
  • 成功時: HTTP POST リクエストが書き込まれた BIO オブジェクト。
  • 失敗時: NULL。

前提条件と不変条件

  • req は有効な OCSP_REQUEST オブジェクトである必要があります。
  • uri はホスト名、ポート、パス情報を含んでいる必要があります。
  • 戻り値の BIO は呼び出し元で解放（BIO_free）する必要があります。

アルゴリズムまたは振る舞い

1. i2d_OCSP_REQUEST を用いてリクエストのデータ長を算出します。 2. メモリベースの BIO を生成します。 3. POST メソッドを含む HTTP/1.0 ヘッダーを書き込みます。プロキシ経由の場合はフル URL を使用し、それ以外はパスとクエリを連結します。 4. Host ヘッダー、Content-Type: application/ocsp-request、Connection: close、および算出した Content-Length を付与します。 5. i2d_OCSP_REQUEST_bio を使用して、OCSP リクエスト本体を BIO に書き込みます。 6. 書き込みに失敗した場合は BIO を解放し NULL を返します。成功した場合は BIO を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_util_ocsp.c:317 (直接の呼び出しを確認)
  • **呼び出し先**: OpenSSL の i2d_OCSP_REQUEST および i2d_OCSP_REQUEST_bio を使用。

関連データ構造

  • OCSP_REQUEST: OpenSSL の OCSP リクエスト構造体。
  • apr_uri_t: APR の URI 解析構造体。
  • BIO: OpenSSL の I/O 抽象化構造体。

失敗モードと境界条件

  • i2d_OCSP_REQUEST_bio が失敗した場合、関数は BIO を破棄し NULL を返します。
  • uri の各フィールドが欠落している場合、HTTP ヘッダーの生成が不完全になる可能性があります。

保守担当者向けメモ

  • 本関数は簡易的な HTTP/1.0 実装の一部です。将来的に標準的な HTTP クライアントライブラリへ移行する際は、このシリアライズ処理も置き換え対象となります。
  • proxy_uri の有無によるヘッダー生成ロジックの分岐に注意してください。
