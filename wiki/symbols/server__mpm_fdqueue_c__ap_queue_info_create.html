ap_queue_info_create

定義

  • 種別: function
  • 定義箇所: server/mpm_fdqueue.c:75

目的

ワーカー・スレッドのアイドル状態管理およびメモリ再利用リストを保持するための fd_queue_info_t 構造体を生成・初期化します。

シグネチャまたは形状

apr_status_t ap_queue_info_create(fd_queue_info_t **queue_info,
                                  apr_pool_t *pool, int max_idlers,
                                  int max_recycled_pools)

入力と出力

  • **入力**:
  • queue_info: 生成された構造体を格納するポインタのポインタ。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • max_idlers: 許容する最大アイドル・スレッド数。
  • max_recycled_pools: 再利用可能なプールの最大数。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返します。
  • *queue_info: 初期化された fd_queue_info_t 構造体へのポインタ。

前提条件と不変条件

  • pool は有効な apr_pool_t である必要があります。
  • 内部の idlers カウンタは、アトミック操作の基準点である zero_pt で初期化されます。

アルゴリズムまたは振る舞い

1. pool から fd_queue_info_t 用のメモリを確保します。 2. アイドル状態管理用のミューテックス (idlers_mutex) を作成します。 3. ワーカー起床用の条件変数 (wait_for_idler) を作成します。 4. recycled_pools を NULL に初期化し、引数で指定された制限値 (max_recycled_pools, max_idlers) を設定します。 5. apr_pool_cleanup_register を使用して、プール破棄時に queue_info_cleanup が実行されるよう登録します。 6. 成功時に *queue_info を設定して APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • apr_pcalloc
  • apr_thread_mutex_create
  • apr_thread_cond_create
  • apr_pool_cleanup_register
  • queue_info_cleanup (クリーンアップ登録時)

関連データ構造

  • fd_queue_info_t: 本関数で生成されるメインの管理構造体。

失敗モードと境界条件

  • apr_thread_mutex_create または apr_thread_cond_create が失敗した場合、そのエラーコードを即座に返します。

保守担当者向けメモ

  • 本関数はスレッド環境 (APR_HAS_THREADS) が有効な場合にのみ使用されます。
  • idlers カウンタの初期値 zero_pt は、アトミック操作による負荷判定の基点となるため、変更時は関連する ap_queue_info_* 関数群との整合性に注意してください。
