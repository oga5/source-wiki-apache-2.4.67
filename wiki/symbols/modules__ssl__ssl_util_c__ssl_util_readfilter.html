ssl_util_readfilter

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util.c:151

目的

外部コマンドを実行し、その標準出力から1行（改行コードまで）を読み取って文字列として返却します。

シグネチャまたは形状

char *ssl_util_readfilter(server_rec *s, apr_pool_t *p, const char *cmd, const char * const *argv);

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p: メモリ割り当てに使用するプール。
  • cmd: 実行するコマンドのパス。
  • argv: コマンド引数の配列。
  • **出力**:
  • 読み取った文字列を格納した静的バッファへのポインタ。失敗した場合は NULL を返します。

前提条件と不変条件

  • 戻り値のバッファは静的領域 (static char buf[MAX_STRING_LEN]) を使用するため、呼び出しごとに内容が上書きされます。
  • 読み取りは1バイトずつ行われます。

アルゴリズムまたは振る舞い

1. ssl_util_ppopen を呼び出し、指定されたコマンドをパイプ経由で実行します。 2. apr_file_read を使用して、パイプから1バイトずつ読み込みます。 3. 改行コード (\n または \r) が出現するか、MAX_STRING_LEN-1 に達するまでバッファに蓄積します。 4. 読み取り終了後、ssl_util_ppclose を呼び出してパイプを閉じます。 5. 蓄積したバッファを返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_pphrase.c:598
  • modules/ssl/ssl_engine_pphrase.c:761
  • **呼び出し先**:
  • ssl_util_ppopen
  • apr_file_read
  • ssl_util_ppclose

関連データ構造

  • server_rec: Apache サーバー設定構造体。
  • apr_pool_t: APR メモリプール。
  • apr_file_t: ファイルまたはパイプのハンドル。

失敗モードと境界条件

  • ssl_util_ppopen が失敗した場合、NULL を返します。
  • MAX_STRING_LEN を超える出力は切り詰められます。
  • 1バイトずつの読み込み処理のため、大量の出力を伴うコマンドではパフォーマンスが低下する可能性があります。

保守担当者向けメモ

  • 本関数は静的バッファを返すため、呼び出し側で結果を保持する必要がある場合は、呼び出し直後にコピーを作成してください。
