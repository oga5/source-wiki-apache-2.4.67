is_unsafe

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_headers.c:38

目的

指定されたサーバー設定において、HTTPの適合性チェックが「安全ではない（unsafe）」モードに設定されているかどうかを判定します。

シグネチャまたは形状

static int is_unsafe(server_rec *s)

入力と出力

  • **入力**: server_rec *s (Apacheサーバーの構成情報を含む構造体)
  • **出力**: int (真であれば AP_HTTP_CONFORMANCE_UNSAFE モード、偽であればそれ以外)

前提条件と不変条件

  • 引数 s は有効な server_rec ポインタである必要があります。
  • ap_get_core_module_config を介して core_server_config が正しく取得できる環境であること。

アルゴリズムまたは振る舞い

1. s->module_config からコアモジュールの設定情報を取得します。 2. 取得した設定内の http_conformance フィールドを参照します。 3. その値が AP_HTTP_CONFORMANCE_UNSAFE と一致するかを比較し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_headers.c:176 にて、リクエストのサーバー設定に基づき呼び出されています。
  • **呼び出し先**: ap_get_core_module_config (Apache Core API) を呼び出しています。
  • **注意**: h2_util.c にも同名の is_unsafe 関数が存在しますが、引数の型（h2_headers *h）が異なるため、本シンボルとは別個の実装です。

関連データ構造

  • server_rec: Apacheのサーバー設定を保持する構造体。
  • core_server_config: Apacheコアモジュールの設定構造体。

失敗モードと境界条件

  • s が NULL の場合、ap_get_core_module_config の呼び出しによりセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数は static であり、h2_headers.c 内でのみ使用されます。
  • h2_util.c に存在する同名の関数と混同しないよう注意してください。
  • HTTP適合性チェックのロジック変更に伴い、判定条件が更新される可能性があります。
