merge_auth_form_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:105

目的

Apache HTTP Server のディレクトリ設定において、親ディレクトリの設定（basev）と現在のディレクトリで追加された設定（addv）をマージし、新しい設定構造体を作成する。

シグネチャまたは形状

static void *merge_auth_form_dir_config(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する APR プール。
  • basev: 親ディレクトリの設定構造体（auth_form_config_rec）。
  • addv: 現在のディレクトリの設定構造体（auth_form_config_rec）。
  • **出力**:
  • マージされた新しい auth_form_config_rec 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は auth_form_config_rec 型としてキャスト可能であること。
  • 各設定項目には対応する _set フラグが存在し、このフラグが 0 でない場合にのみ addv の値が優先される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい auth_form_config_rec 構造体をプール p 上に確保する。 2. 各設定項目（providers, authoritative, site, username, password, location, form_size, fakebasicauth, method, mimetype, body, disable_no_store, loginsuccess, loginrequired, logout）について以下のロジックを適用する：

  • 対応する _set フラグが 0 であれば、base（親）の値を採用する。
  • _set フラグが 0 でなければ、add（現在）の値を採用する。
  • _set フラグ自体は、add または base のいずれかが設定されていれば 1 に更新する。

    • 3. マージ済みの構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c:1327 にて、ディレクトリマージ用関数として登録されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わず、メモリ確保のために apr_pcalloc を使用する。

関連データ構造

  • auth_form_config_rec: フォームベース認証のディレクトリ設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗する可能性がある（Apache の標準的なメモリ管理に従い、通常はプロセス終了を伴う）。
  • basev または addv が NULL の場合、構造体へのアクセス時にセグメンテーション違反が発生する可能性がある（Apache の設定マージの仕組み上、通常は初期化済みの構造体が渡される）。

保守担当者向けメモ

  • 本関数は設定のオーバーライドを制御する重要な箇所である。新しい設定項目を auth_form_config_rec に追加した場合は、必ず本関数にもマージロジックを追加すること。
