accept_filter_to_string

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/child.c:287

目的

accept_filter_e 列挙型の値を、対応する人間が読み取り可能な文字列（"none" または "connect"）に変換します。

シグネチャまたは形状

static const char * accept_filter_to_string(accept_filter_e accf)

入力と出力

  • **入力**: accept_filter_e 型の列挙値 accf。
  • **出力**: 変換後の文字列リテラルを指す const char *。

前提条件と不変条件

  • 入力値が ACCEPT_FILTER_NONE または ACCEPT_FILTER_CONNECT のいずれかである場合、対応する文字列を返します。
  • それ以外の値が入力された場合、空文字列 "" を返します。

アルゴリズムまたは振る舞い

1. accf の値を switch 文で評価します。 2. ACCEPT_FILTER_NONE の場合は "none" を返します。 3. ACCEPT_FILTER_CONNECT の場合は "connect" を返します。 4. 上記以外（デフォルト）の場合は "" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/child.c:432 にて、リスナー設定のログ出力時に呼び出されています。
  • **呼び出し先**: なし。

関連データ構造

  • accept_filter_e: 接続フィルタの種類を定義する列挙型。

失敗モードと境界条件

  • 未定義の列挙値が渡された場合、エラーを発生させるのではなく空文字列 "" を返す安全な設計となっています。

保守担当者向けメモ

  • 本関数は主にデバッグログや設定情報の出力用に使用されます。新しいフィルタタイプを accept_filter_e に追加した場合は、本関数の switch 文にもケースを追加してください。
