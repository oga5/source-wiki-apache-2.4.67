session_included

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:53

目的

リクエストされた URI が、設定されたセッション処理の対象パス（Include）に含まれ、かつ除外パス（Exclude）に含まれていないかを判定します。

シグネチャまたは形状

static int session_included(request_rec *r, session_dir_conf *conf)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報（request_rec）。r->uri を判定に使用。
  • conf: セッション設定（session_dir_conf）。includes および excludes リストを参照。
  • **出力**:
  • int: 判定結果。対象であれば 1、対象外であれば 0 を返します。

前提条件と不変条件

  • conf->includes が空の場合、デフォルトで全てのパスが対象（included = 1）となります。
  • includes リストが設定されている場合、いずれかのパスに前方一致（strncmp）しない限り、デフォルトで対象外となります。
  • excludes リストに一致するパスがある場合、includes の判定結果に関わらず、最終的に対象外となります。

アルゴリズムまたは振る舞い

1. included フラグを初期値 1 で開始します。 2. conf->includes に要素がある場合、included を 0 にリセットし、r->uri がいずれかの include パスで始まるかを確認します。一致すれば included を 1 に設定します。 3. conf->excludes に要素がある場合、included が 1 であることを条件に、r->uri がいずれかの exclude パスで始まるかを確認します。一致すれば included を 0 に設定します。 4. 最終的な included の値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session.c:106（ap_session_load 内でセッションロードの可否を判定するために呼び出されます）。
  • **呼び出し先**: 標準の文字列操作関数 strncmp および strlen を使用します。

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • session_dir_conf: mod_session のディレクトリ単位の設定構造体。includes および excludes（apr_array_header_t）を保持します。

失敗モードと境界条件

  • includes または excludes が空の配列である場合、ループは実行されず、初期値または直前の判定結果が維持されます。
  • r->uri が include や exclude の文字列長より短い場合、strncmp は安全に動作しますが、一致判定は行われません。

保守担当者向けメモ

  • 本関数は前方一致（strncmp）を使用しているため、パスの階層構造を考慮した厳密なマッチングが必要な場合は注意してください。
  • conf->includes が設定されている場合、明示的に指定されたパス以外はすべて除外される「ホワイトリスト」方式として動作します。
