md_http_request_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_http.h:27

目的

mod_md モジュールにおいて、個別の HTTP リクエスト（GET、HEAD、POST 等）を表現し、通信に必要なメソッド、URL、ヘッダー、ボディ、およびタイムアウト設定を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct md_http_request_t md_http_request_t;

※ 構造体の内部メンバはヘッダーファイル上で不透明（opaque）として定義されており、実装固有のデータは internals ポインタを介して管理されます。

入力と出力

  • **入力**: md_http_GET_create 等の生成関数を通じて、URL やコールバック、タイムアウト設定が入力されます。
  • **出力**: md_http_perform を通じて実行され、通信結果は関連する md_http_response_t に格納されます。

前提条件と不変条件

  • md_http_create で作成された md_http_t セッションのコンテキスト内で使用される必要があります。
  • md_http_perform の終了時に自動的に破棄されるため、呼び出し側でメモリを解放してはなりません。

アルゴリズムまたは振る舞い

  • md_http_perform が呼び出されると、登録された md_http_impl_t（バックエンド実装）がこのリクエストを処理します。
  • 非同期処理時には md_http_multi_perform を通じて並列実行されます。
  • リクエスト完了後、設定されたコールバック関数が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: md_acme.c（ACME プロトコル処理）、md_curl.c（バックエンド実装）など。
  • **呼び出し先**: md_curl.c 内の internals_setup, update_status, fire_status, md_curl_perform 等の内部関数が、リクエストの状態管理や実行のために本構造体を参照・操作します。

関連データ構造

  • md_http_t: リクエストを保持・管理するセッションインスタンス。
  • md_http_response_t: リクエストの結果（ステータス、ヘッダー、ボディ）を保持する構造体。
  • md_http_impl_t: リクエストを実行するバックエンド実装の関数テーブル。
  • apr_bucket_brigade: リクエストボディの管理に使用される APR データ構造。

失敗モードと境界条件

  • 実装が登録されていない状態で md_http_perform を呼び出すと失敗します。
  • 通信タイムアウトやストール検出が発生した場合、md_http_callbacks_t に定義されたコールバックを通じてエラーが通知されます。

保守担当者向けメモ

  • 本構造体の内部メンバは md_http.h では定義されていません。実装の詳細を確認する場合は modules/md/md_http.c および具体的なバックエンド実装（modules/md/md_curl.c 等）を参照してください。
  • メモリ管理は md_http_perform のライフサイクルに依存しているため、リクエストオブジェクトの生存期間を外部で延長しようとしないでください。
