req_aprtable2luatable_cb_len

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:177

目的

apr_table_t の内容を Lua のテーブルへ変換する際、キーと値のペアを Lua スタック上の2つのテーブル（複合値用と単純値用）に格納するコールバック関数。

シグネチャまたは形状

int req_aprtable2luatable_cb_len(void *l, const char *key, const char *value, size_t len)
  • l: Lua の状態 (lua_State *)。スタックには [table<s,t>, table<s,s>] が配置されていることを想定。
  • key: APR テーブルのキー文字列。
  • value: APR テーブルの値文字列。
  • len: 値の長さ。

入力と出力

  • 入力: Lua スタック上の2つのテーブル（インデックス -1 に単純値用テーブル、-2 に複合値用テーブル）。
  • 出力: 成功時に 1 を返す。

前提条件と不変条件

  • Lua スタックのトップに table<s,t>（複合値用）、その下に table<s,s>（単純値用）が存在すること。
  • value は len バイトの文字列として扱われる。

アルゴリズムまたは振る舞い

1. **複合値の構築**: key を用いて複合値用テーブルから値を取得する。

  • 値が未定義（nil）の場合: 新規テーブルを作成し、値を配列の1番目に格納する。
  • 値がテーブルの場合: 既存の配列の末尾に値を追加する。

    • 2. **単純値の構築**: key を用いて単純値用テーブルから値を取得する。

  • 値が未定義（nil）の場合のみ、value を文字列として格納する（重複キーの初回出現を優先）。

    • 3. 処理終了後、スタックの状態を維持して 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: modules/lua/lua_request.c:429
  • 呼び出し先: Lua C API (lua_getfield, lua_type, lua_pop, lua_newtable, lua_pushnumber, lua_pushlstring, lua_settable, lua_setfield, lua_rawlen, lua_isnoneornil)

関連データ構造

  • apr_table_t: APR のキー・値ペアを保持する構造体。
  • lua_State: Lua VM の実行状態。

失敗モードと境界条件

  • Lua スタックが想定した構造でない場合、スタック操作により Lua VM がクラッシュまたはエラーを発生させる可能性がある。
  • lua_rawlen や lua_settable 等の Lua C API 呼び出しは、メモリ不足時に Lua のエラーハンドリング機構へ制御を移す。

保守担当者向けメモ

  • 本関数は apr_table_t の反復処理中に呼び出されるコールバックである。
  • lua_pushlstring を使用しているため、value にヌル文字が含まれていても正しく処理される。
  • 単純値用テーブルは「最初の値」のみを保持し、複合値用テーブルは「すべての値」を配列として保持する設計となっている。
