h2_push_diary

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_push.h:78

目的

HTTP/2 Server Push において、既にクライアントへ送信済みのリソースを記録し、同一コネクション内での重複送信を抑制するための履歴管理構造体です。

シグネチャまたは形状

typedef struct h2_push_diary h2_push_diary;

（不完全な型として宣言されており、内部構造の詳細は modules/http2/h2_push.c に隠蔽されています。）

入力と出力

  • **入力**: h2_push オブジェクト（リソース情報）およびハッシュ値。
  • **出力**: 履歴内での存在確認結果（h2_push_diary_find）、またはキャッシュダイジェスト情報（h2_push_diary_digest_get）。

前提条件と不変条件

  • 固定サイズのメモリ領域を使用して履歴を保持するように設計されています。
  • 履歴の更新時には、古いエントリを破棄して新しいエントリを追加する循環的な管理が行われます。

アルゴリズムまたは振る舞い

1. **重複チェック**: h2_push_diary_find を使用して、指定されたハッシュ値が履歴内に存在するかを確認します。 2. **履歴更新**: h2_push_diary_append を介して新しいリソースのハッシュ値を記録します。 3. **ハッシュ計算**: 環境に応じて SHA256 または apr_hashfunc_default を用いてリソースのハッシュ値を算出します。 4. **管理操作**: 履歴の順序入れ替え（move_to_last）や、古いエントリの削除（remove_first）により、メモリ消費を一定に保ちます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_session.c（セッション単位での管理）、modules/http2/h2_push.c（内部ロジック）。
  • **呼び出し先**: calc_sha256_hash、calc_apr_hash、h2_push_diary_find、h2_push_diary_append 等の h2_push.c 内の静的関数。

関連データ構造

  • h2_push: Push 対象のリクエスト情報と優先度を保持する構造体。
  • h2_push_diary_entry: 履歴内の個別のエントリ。
  • h2_push_digest_type: ハッシュ計算アルゴリズムの種別。

失敗モードと境界条件

  • メモリ制限: 固定サイズで管理されるため、履歴の最大数を超えると古いデータから順次破棄されます。
  • ハッシュ衝突: 異なるリソースが同一のハッシュ値を持つ場合、重複とみなされ Push が抑制される可能性があります。

保守担当者向けメモ

  • 本構造体は h2_push_diary_create で初期化されます。
  • キャッシュダイジェスト機能は IETF ドラフトに基づいた実装であり、標準化から外れていますが、重複排除の効率化のために維持されています。
  • ハッシュ計算の実装は h2_push_digest_calc 型の関数ポインタを通じて切り替え可能です。
