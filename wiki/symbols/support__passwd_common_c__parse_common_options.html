parse_common_options

定義

  • 種別: function
  • 定義箇所: support/passwd_common.c:314

目的

コマンドライン引数（オプション文字と引数）を解析し、passwd_ctx 構造体の状態を更新します。

シグネチャまたは形状

int parse_common_options(struct passwd_ctx *ctx, char opt, const char *opt_arg);

入力と出力

  • **入力**:
  • ctx: 更新対象の passwd_ctx 構造体へのポインタ。
  • opt: 解析対象のオプション文字。
  • opt_arg: オプションに関連付けられた引数（C または r の場合に使用）。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時はエラーコード（ERR_ALG_NOT_SUPP または ERR_SYNTAX）を返し、ctx->errstr にエラーメッセージを格納します。

前提条件と不変条件

  • errfile グローバル変数が初期化されている必要があります。
  • opt_arg は C または r オプション以外では無視されます。

アルゴリズムまたは振る舞い

1. opt に基づき switch 文で分岐します。 2. b, i: ctx->passwd_src を設定します。 3. m, s, p, d, B, 2, 5: ctx->alg を対応するアルゴリズム定数に設定します。

  • プラットフォームでサポートされていないアルゴリズムが指定された場合、警告を出力するか、エラーコードを返します。

    • 4. C, r: opt_arg を正の整数に変換し、ctx->cost に格納します。変換失敗時は ERR_SYNTAX を返します。 5. 未定義のオプションが渡された場合は、エラーメッセージを出力し abort() を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/htdbm.c:373
  • support/htpasswd.c:201
  • **呼び出し先**:
  • strtol (標準ライブラリ)
  • apr_file_printf (APR)
  • abort (標準ライブラリ)

関連データ構造

  • struct passwd_ctx: パスワード生成のコンテキストを保持する構造体。

失敗モードと境界条件

  • **サポート外のアルゴリズム**: プラットフォームで利用できないアルゴリズム（2, 5, B, d）が指定された場合、エラーコードを返します。
  • **不正な引数**: C または r オプションに対し、正の整数以外の値が渡された場合、ERR_SYNTAX を返します。
  • **不正なオプション**: 予期しない文字が渡された場合、プログラムを強制終了（abort）します。

保守担当者向けメモ

  • 本関数は errfile グローバル変数に依存しています。
  • switch 文内の条件付きコンパイル（#ifdef）は、プラットフォームごとの crypt() 等のサポート状況を反映しています。
  • 新しいアルゴリズムを追加する際は、passwd_common.h の定義と本関数の switch 文の両方を更新してください。
