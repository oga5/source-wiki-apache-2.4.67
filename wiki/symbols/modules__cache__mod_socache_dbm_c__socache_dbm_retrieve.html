socache_dbm_retrieve

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:304

目的

DBM ファイルから指定されたキーに対応するキャッシュデータを取得し、期限切れチェックを行う。

シグネチャまたは形状

apr_status_t socache_dbm_retrieve(ap_socache_instance_t *ctx, server_rec *s,
                                  const unsigned char *id, unsigned int idlen,
                                  unsigned char *dest, unsigned int *destlen,
                                  apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンスのコンテキスト。
  • s: サーバーレコード。
  • id, idlen: 検索対象のキーとその長さ。
  • dest: 取得したデータを格納するバッファ。
  • destlen: 入力時は dest の最大サイズ、出力時は取得したデータの実際のサイズ。
  • p: メモリプール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、見つからない場合は APR_NOTFOUND、容量不足は APR_ENOSPC 等を返す。

前提条件と不変条件

  • ctx->pool は呼び出しごとにクリアされる。
  • キャッシュデータは、先頭に apr_time_t 型の期限情報が格納されている必要がある。
  • 本関数はスレッドセーフではないため、呼び出し側での排他制御が前提となる。

アルゴリズムまたは振る舞い

1. socache_dbm_expire を呼び出し、定期的な期限切れ処理を実行する。 2. apr_dbm_open2（または apr_dbm_open）で DBM ファイルを読み取り専用で開く。 3. apr_dbm_fetch でキーに対応する値を取得する。 4. 取得した値の先頭から apr_time_t（期限）を読み取り、現在時刻と比較する。 5. 期限が切れている場合は socache_dbm_remove を呼び出してエントリを削除し、APR_NOTFOUND を返す。 6. 期限内であれば、期限情報を除いたデータを dest にコピーし、destlen を更新して APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dbm.c:747 に参照あり。
  • **呼び出し先**: socache_dbm_expire, apr_dbm_open/apr_dbm_open2, apr_dbm_fetch, apr_dbm_close, socache_dbm_remove, apr_time_now 等。

関連データ構造

  • ap_socache_instance_t: キャッシュのコンテキスト（ファイルパス等を保持）。
  • apr_dbm_t: DBM ファイルのハンドル。
  • apr_datum_t: DBM のキーおよび値を保持する構造体。

失敗モードと境界条件

  • **APR_NOTFOUND**: キーが存在しない、または期限切れにより削除された場合。
  • **APR_ENOSPC**: dest バッファのサイズが取得データに対して不足している場合。
  • **APR_EGENERAL**: 取得したデータが apr_time_t のサイズ未満である場合（データ破損等）。
  • **DBM オープン失敗**: ファイルの読み込み権限がない場合やファイルが存在しない場合。

保守担当者向けメモ

  • DBM ファイルの特性上、約 1KB 程度のデータサイズ制限がある点に注意すること。
  • apr_dbm_open の呼び出しを r->pool に紐付けることでクリーンアップを簡略化できる可能性がある（ソース内の XXX コメント参照）。
