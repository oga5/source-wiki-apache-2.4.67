register_hooks

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ratelimit.c:325

目的

Apache HTTP Server のフィルタチェーンに対し、mod_ratelimit が提供する出力フィルタを登録します。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p (フック登録に使用されるメモリプール)
  • 出力: なし (void)

前提条件と不変条件

  • 本関数は Apache のモジュールロードプロセス中に呼び出されることを前提としています。
  • フィルタは AP_FTYPE_CONNECTION - 1 の位置に登録され、mod_deflate 等の他のフィルタよりも後段で動作するよう配置されます。

アルゴリズムまたは振る舞い

1. ap_register_output_filter を呼び出し、RATE_LIMIT_FILTER_NAME という名前で rate_limit_filter 関数をフィルタチェーンに登録します。 2. フィルタの実行順序を制御するため、接続レベルのフィルタ（mod_ssl など）よりは後、かつ一般的な出力フィルタよりは前となるよう AP_FTYPE_CONNECTION - 1 を指定します。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュール読み込み機構（直接的な呼び出し元コードは提供されたソースに含まれていません）。
  • 呼び出し先: ap_register_output_filter (Apache API)。

関連データ構造

  • rate_limit_filter (フィルタ本体の関数)
  • RATE_LIMIT_FILTER_NAME (フィルタ名定数)

失敗モードと境界条件

  • Apache のフィルタ登録 API が失敗した場合の挙動については、本関数内では明示的なエラーハンドリングを行っていません。

保守担当者向けメモ

  • 本関数はモジュールの初期化時に一度だけ実行されます。
  • mod_deflate 等との順序関係を維持するため、フィルタタイプ定数の変更には注意が必要です。
  • 将来的に Optional Functions への移行が検討されています。
