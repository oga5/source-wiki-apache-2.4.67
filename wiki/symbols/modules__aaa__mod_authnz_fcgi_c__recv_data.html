recv_data

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:168

目的

FastCGI バックエンドからデータを読み込み、エラー発生時にはログ出力を行う。

シグネチャまたは形状

apr_status_t recv_data(const fcgi_provider_conf *conf,
                       request_rec *r,
                       apr_socket_t *s,
                       char *buf,
                       apr_size_t *buflen)

入力と出力

  • **入力**:
  • conf: バックエンド設定情報。
  • r: 現在のリクエストレコード。
  • s: 通信に使用するソケット。
  • buf: 受信データを格納するバッファ。
  • buflen: 受信予定のバイト数（入力）、および実際に受信したバイト数（出力）。
  • **出力**:
  • apr_status_t: 操作の成功（APR_SUCCESS）またはエラーコード。

前提条件と不変条件

  • s は有効な接続済みソケットであること。
  • buf は *buflen で指定されたサイズ以上のメモリ領域を確保していること。

アルゴリズムまたは振る舞い

1. apr_socket_recv を呼び出し、ソケットからデータを読み込む。 2. 読み込みに失敗した場合、conf->backend を含むエラーログを記録し、エラーコードを返す。 3. 成功した場合、AP_MODULE_MAGIC_AT_LEAST(20130702, 2) が定義されていれば、デバッグ用に受信データをトレースログ（APLOG_TRACE5）へ出力する。 4. APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:203
  • modules/aaa/mod_authnz_fcgi.c:540
  • **呼び出し先**:
  • apr_socket_recv (APR)
  • ap_log_rerror (Apache HTTP Server)
  • ap_log_rdata (Apache HTTP Server, 条件付き)

関連データ構造

  • fcgi_provider_conf: バックエンド接続情報を保持する構造体。
  • request_rec: Apache のリクエスト状態を保持する構造体。
  • apr_socket_t: APR のソケット抽象化構造体。

失敗モードと境界条件

  • apr_socket_recv が APR_SUCCESS 以外を返した場合、エラーログが出力され、処理が中断される。
  • 読み込みバイト数が *buflen より少ない場合、apr_socket_recv の挙動に従い、部分的なデータがバッファに格納される。

保守担当者向けメモ

  • 本関数は FastCGI プロトコルのストリーム読み込みの一部として使用される。
  • ap_log_rdata はデバッグビルド時のみ有効であるため、本番環境でのパフォーマンス影響はない。
