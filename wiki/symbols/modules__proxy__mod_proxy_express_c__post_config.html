post_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_express.c:102

目的

mod_proxy モジュールがサーバーにロードされているかを確認し、その可用性を静的変数に記録します。

シグネチャまたは形状

static int post_config(apr_pool_t *p,
                       apr_pool_t *plog,
                       apr_pool_t *ptemp,
                       server_rec *s)

入力と出力

  • 入力:
  • p, plog, ptemp: APR プール。
  • s: 現在のサーバー設定レコード。
  • 出力:
  • OK: 常に正常終了として返されます。

前提条件と不変条件

  • mod_proxy.c が Apache HTTP Server のリンク済みモジュールとして存在している必要があります。

アルゴリズムまたは振る舞い

1. ap_find_linked_module("mod_proxy.c") を呼び出し、プロキシ機能の基盤となる mod_proxy がロードされているかを判定します。 2. 判定結果（真偽値）を静的変数 proxy_available に格納します。 3. OK を返して処理を終了します。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server の設定フェーズにおいて、post_config フックとして呼び出されます。
  • 呼び出し先: ap_find_linked_module (Apache Core API)。

関連データ構造

  • server_rec: Apache のサーバー設定構造体。
  • proxy_available: 本モジュール内で定義された静的変数（本関数により初期化）。

失敗モードと境界条件

  • mod_proxy.c がロードされていない場合、proxy_available は 0 (false) となります。本関数自体はエラーを返さず、常に OK を返します。

保守担当者向けメモ

  • 本関数は mod_proxy_express が動作するための前提条件を確認する役割を持ちます。
  • proxy_available 変数の宣言箇所については、提供されたソース excerpt には含まれていません。
