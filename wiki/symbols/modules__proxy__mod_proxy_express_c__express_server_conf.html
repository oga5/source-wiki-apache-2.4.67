express_server_conf

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_express.c:29

目的

mod_proxy_express モジュールにおいて、サーバー単位の設定情報を保持するための構造体です。DBM ファイルを用いた動的プロキシルーティングに必要なパラメータを管理します。

シグネチャまたは形状

typedef struct {
    const char *dbmfile; /* DBMファイルのパス */
    const char *dbmtype; /* DBMの形式 */
    int enabled;         /* モジュールの有効/無効フラグ */
} express_server_conf;

入力と出力

本構造体は直接的な関数ではありませんが、以下の設定ハンドラを通じて値が設定されます。

  • 入力: set_dbmfile, set_dbmtype, set_enabled 各関数により、設定ファイル（httpd.conf）から値が読み込まれます。
  • 出力: xlate_name フック関数が本構造体を参照し、リクエストのルーティング判断に使用します。

前提条件と不変条件

  • dbmfile および dbmtype は、apr_dbm_open2 等の APR DBM API が解釈可能な形式である必要があります。
  • enabled が真の場合、xlate_name フックは DBM ファイルを読み取り専用で開くことを試みます。

アルゴリズムまたは振る舞い

  • サーバー生成時（server_create）およびマージ時（server_merge）に、メモリプール（apr_pcalloc）を使用してインスタンス化されます。
  • リクエスト処理時、xlate_name は本構造体の enabled フラグを確認し、無効であれば処理を DECLINED します。有効な場合は dbmfile を使用してバックエンド URL を解決します。

呼び出し元と呼び出し先

  • 呼び出し元: server_create, server_merge, set_dbmfile, set_dbmtype, set_enabled, xlate_name 等のモジュール内関数が本構造体を操作・参照します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • proxy_server_conf: mod_proxy のコア設定構造体。
  • apr_dbm_t: DBM ファイル操作に使用される APR のデータ構造。

失敗モードと境界条件

  • DBM ファイルが存在しない、または読み取り権限がない場合、xlate_name によるルーティング解決が失敗します。
  • dbmtype がシステムでサポートされていない場合、DBM のオープン処理が失敗します。

保守担当者向けメモ

  • 本構造体は mod_proxy_express.c 内でのみ使用されるプライベートな設定保持用です。
  • メンバー変数の追加や変更を行う際は、server_merge 関数において、ベース設定からオーバーライド設定への継承ロジックが正しく維持されるか確認してください。
