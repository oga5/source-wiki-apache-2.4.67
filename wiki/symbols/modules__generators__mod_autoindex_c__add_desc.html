add_desc

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_autoindex.c:295

目的

Apache HTTP Server の AddDescription ディレクティブを解析し、ディレクトリごとの設定構造体（autoindex_config_rec）にファイル説明文のパターンと内容を登録する。

シグネチャまたは形状

static const char *add_desc(cmd_parms *cmd, void *d, const char *desc, const char *to)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • d: 現在のディレクトリ設定構造体（autoindex_config_rec）。
  • desc: ファイルに付与する説明文。
  • to: 説明文を適用する対象のファイル名またはパターン。
  • **出力**:
  • 成功時は NULL を返す。

前提条件と不変条件

  • 引数 d は autoindex_config_rec 型としてキャスト可能であること。
  • dcfg->desc_list は初期化済みの apr_array_header_t であること。
  • メモリ割り当ては dcfg->desc_list->pool を使用して行われる。

アルゴリズムまたは振る舞い

1. dcfg->desc_list に新しい ai_desc_t エントリを確保する。 2. to にスラッシュが含まれる場合は full_path フラグを立てる。 3. to がワイルドカードを含むか、full_path が有効な場合、パターンを */ または * で囲み、apr_fnmatch でマッチング可能な形式に変換する。 4. それ以外の場合は、to をそのままパターンとして保持する。 5. desc を ai_desc_t の description フィールドにコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_autoindex.c:591 において、AP_INIT_ITERATE2("AddDescription", ...) マクロ経由で設定解析時に呼び出される。
  • **呼び出し先**: apr_array_push、ap_strchr_c、apr_fnmatch_test、apr_pstrcat、apr_pstrdup（すべて APR/Apache API）。

関連データ構造

  • autoindex_config_rec: ディレクトリ設定を保持する構造体。
  • ai_desc_t: ファイル説明文のパターンと内容を管理する構造体。

失敗モードと境界条件

  • メモリ不足時は apr_pstrdup 等の内部で例外が発生する可能性がある。
  • to 引数が空文字列の場合の挙動は、apr_fnmatch の仕様に依存する。

保守担当者向けメモ

  • 本関数は AddDescription ディレクティブの解析専用である。
  • パターンマッチングのロジックは mod_autoindex 内の他のパターン登録関数（add_icon 等）と一貫性がある。
