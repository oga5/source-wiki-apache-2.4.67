gen_ws_accept

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_ws.c:61

目的

RFC 6455 に準拠した WebSocket ハンドシェイク用の Sec-WebSocket-Accept ヘッダー値を生成します。

シグネチャまたは形状

static const char *gen_ws_accept(conn_rec *c, const char *key_base64)

入力と出力

  • **入力**:
  • c: 接続コンテキスト (conn_rec*)。メモリ割り当てに使用。
  • key_base64: クライアントから提供された Sec-WebSocket-Key 文字列。
  • **出力**:
  • SHA-1 ハッシュを計算し、Base64 エンコードした Sec-WebSocket-Accept 用の文字列を返します。

前提条件と不変条件

  • key_base64 は有効な文字列である必要があります。
  • 内部で使用される固定 GUID 文字列 "258EAFA5-E914-47DA-95CA-C5AB0DC85B11" は RFC 6455 で規定された定数です。

アルゴリズムまたは振る舞い

1. apr_sha1_ctx_t を初期化します。 2. key_base64 を SHA-1 コンテキストに更新します。 3. 規定の WebSocket GUID を SHA-1 コンテキストに更新します。 4. SHA-1 ハッシュ値を計算し、c->pool を使用して Base64 エンコードした文字列を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_ws.c:133 にて呼び出されています。
  • **呼び出し先**:
  • apr_sha1_init, apr_sha1_update, apr_sha1_final (APR SHA-1 API)
  • apr_pencode_base64_binary (APR エンコード API)

関連データ構造

  • conn_rec: Apache HTTPD の接続構造体。メモリプール (pool) の参照元として使用。
  • apr_sha1_ctx_t: ハッシュ計算用コンテキスト。

失敗モードと境界条件

  • key_base64 が NULL の場合、strlen でクラッシュする可能性があります（ソースコード上に明示的なチェックはありません）。
  • メモリ不足時には apr_pencode_base64_binary が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は WebSocket のハンドシェイクにおけるセキュリティ検証の根幹を担います。GUID の変更は RFC 違反となるため行わないでください。
  • c->pool を使用しているため、接続のライフサイクルに合わせてメモリが自動的に解放されます。
