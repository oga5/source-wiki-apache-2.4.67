h2_rp_state_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_c2_filter.c:376

目的

HTTP/1.1 形式のレスポンスを解析する際の、現在の解析フェーズを追跡するための状態定義です。

シグネチャまたは形状

typedef enum {
    H2_RP_STATUS_LINE,  /* ステータス行の解析中 */
    H2_RP_HEADER_LINE,  /* ヘッダー行の解析中 */
    H2_RP_DONE          /* 解析完了 */
} h2_rp_state_t;

入力と出力

  • 本シンボル自体は状態遷移の列挙値であり、入力や出力は持ちません。
  • h2_response_parser 構造体内のメンバとして使用され、解析処理の制御に利用されます。

前提条件と不変条件

  • h2_response_parser が保持する状態として、H2_RP_STATUS_LINE から開始し、順次 H2_RP_HEADER_LINE を経て H2_RP_DONE へ遷移することが期待されます。

アルゴリズムまたは振る舞い

  • parse_response 関数において、ブリゲードから読み取ったデータの種類に応じて状態を更新します。
  • H2_RP_STATUS_LINE: レスポンスの最初の行を解析し、成功すれば H2_RP_HEADER_LINE へ移行します。
  • H2_RP_HEADER_LINE: 空行（ヘッダーの終了）が検出されるまでヘッダー行を解析し、終了時に H2_RP_DONE へ移行します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_c2_filter.c 内の h2_response_parser 構造体定義（385行目）および解析ロジック。
  • 呼び出し先: 該当なし（列挙型定義のため）。

関連データ構造

  • h2_response_parser: この列挙型を状態保持に使用するコンテキスト構造体。

失敗モードと境界条件

  • 状態遷移の過程で不正なフォーマットが検出された場合、解析処理はエラーを返し、状態は適切に更新されない可能性があります。

保守担当者向けメモ

  • 本列挙型は h2_c2_filter.c 内のレスポンス解析ロジックに密結合しています。解析フェーズの追加や変更を行う場合は、parse_response 関数内のスイッチ文や条件分岐の整合性を確認してください。
