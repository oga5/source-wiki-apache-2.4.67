ImportRegisterHooks

定義

  • 種別: function
  • 定義箇所: modules/test/mod_optional_fn_import.c:40

目的

モジュールロード時に、リクエスト処理およびオプション関数の解決に必要なフックを Apache HTTP Server のフックシステムへ登録します。

シグネチャまたは形状

static void ImportRegisterHooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p (フック登録に使用されるメモリプール)
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • Apache のモジュール初期化フェーズにおいて呼び出されることを前提とします。
  • APR_HOOK_MIDDLE を指定し、標準的な優先順位でフックを登録します。

アルゴリズムまたは振る舞い

1. ap_hook_log_transaction を呼び出し、ImportLogTransaction をトランザクションログ記録フェーズのハンドラとして登録します。 2. ap_hook_optional_fn_retrieve を呼び出し、ImportFnRetrieve をオプション関数の解決フェーズのハンドラとして登録します。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュールロードプロセス（モジュール定義構造体経由）。
  • 呼び出し先:
  • ap_hook_log_transaction (Apache API)
  • ap_hook_optional_fn_retrieve (Apache API)
  • ImportLogTransaction (本モジュール内のハンドラ)
  • ImportFnRetrieve (本モジュール内のハンドラ)

関連データ構造

  • apr_pool_t: Apache のメモリ管理用プール。
  • module 構造体（本関数を登録する親構造体）。

失敗モードと境界条件

  • フック登録自体はメモリ不足等の致命的なエラーを除き、通常失敗しません。
  • 登録された各ハンドラ（ImportLogTransaction, ImportFnRetrieve）の実行時における失敗モードについては、各関数の定義を参照してください。

保守担当者向けメモ

  • 本関数は mod_optional_fn_import.c におけるエントリーポイントの一つです。
  • フックの順序や優先度を変更する場合は、APR_HOOK_MIDDLE の値を調整してください。
