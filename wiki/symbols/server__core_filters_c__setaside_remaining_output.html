setaside_remaining_output

定義

  • 種別: function
  • 定義箇所: server/core_filters.c:557

目的

出力フィルタにおいて、処理されなかったバケットブリゲードを永続的なバッファへ退避させ、後続の処理に備える。

シグネチャまたは形状

static void setaside_remaining_output(ap_filter_t *f,
                                      core_output_filter_ctx_t *ctx,
                                      apr_bucket_brigade *bb,
                                      conn_rec *c)

入力と出力

  • f: フィルタインスタンス。
  • ctx: 出力フィルタのコンテキスト（バッファやプールを保持）。
  • bb: 退避対象のバケットブリゲード。
  • c: 接続レコード。
  • 出力: なし。副作用として ctx->buffered_bb へのデータ格納、または ctx->deferred_write_pool のクリアを行う。

前提条件と不変条件

  • bb 内の先頭にあるメタデータバケットまたは長さ 0 のバケットは、既に消費済みとみなされ削除される。
  • ctx->deferred_write_pool が存在しない場合、必要に応じて c->pool を親として作成される。

アルゴリズムまたは振る舞い

1. **不要バケットの削除**: bb の先頭から、メタデータまたは長さ 0 のバケットを走査し、apr_bucket_delete で削除する。 2. **状態更新**: c->data_in_output_filters を 0 にリセットする。 3. **ブリゲードの退避**: bb が空でない場合:

  • c->data_in_output_filters を 1 に設定する。
  • bb が ctx->buffered_bb と異なる場合、ap_save_brigade を使用して bb の内容を ctx->buffered_bb にコピー（退避）する。この際、ctx->deferred_write_pool を使用する。

    • 4. **クリーンアップ**: bb が空で、かつ ctx->deferred_write_pool が存在する場合は、apr_pool_clear を呼び出してプールを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: server/core_filters.c 内の ap_core_output_filter (行 549)。
  • 呼び出し先: apr_bucket_delete, apr_pool_create, apr_pool_tag, ap_save_brigade, apr_pool_clear (すべて APR/Apache API)。

関連データ構造

  • core_output_filter_ctx_t: フィルタの永続状態を保持する構造体。
  • apr_bucket_brigade: バケットのリスト構造。
  • conn_rec: 接続ごとの状態を保持する構造体。

失敗モードと境界条件

  • bb が空の場合、プール内のデータがクリアされる。
  • ap_save_brigade が失敗した場合の戻り値は本関数では無視される（戻り値がないため）。

保守担当者向けメモ

  • 本関数はパイプライン化されたリクエストのバッファリングを管理する。
  • deferred_write_pool はリクエスト終了時にクリアされるため、バッファの生存期間管理に注意すること。
