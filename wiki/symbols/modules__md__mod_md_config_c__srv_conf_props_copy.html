srv_conf_props_copy

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_config.c:190

目的

あるサーバー設定構造体（md_srv_conf_t）から別のサーバー設定構造体へ、設定プロパティをコピーします。

シグネチャまたは形状

static void srv_conf_props_copy(md_srv_conf_t *to, const md_srv_conf_t *from)

入力と出力

  • to: コピー先の md_srv_conf_t 構造体へのポインタ。
  • from: コピー元の md_srv_conf_t 構造体へのポインタ。
  • 出力: なし（戻り値は void）。

前提条件と不変条件

  • to および from は有効な md_srv_conf_t 構造体を指している必要があります。
  • 本関数は構造体のメンバを直接代入するため、ポインタ型のメンバについてはシャローコピーとなります。

アルゴリズムまたは振る舞い

from が保持する以下の設定値を、to の対応するメンバへ順次代入します。

  • 動作フラグ: transitive, require_https, must_staple, stapling, staple_others, ari_renewals, profile_mandatory
  • 数値・設定値: renew_mode, pks, warn_window, renew_window
  • 文字列・オブジェクト参照: ca_urls, ca_contact, ca_proto, ca_agreement, ca_challenges, ca_eab_kid, ca_eab_hmac, profile, dns01_cmd

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md_config.c:437 にて確認されています。
  • 呼び出し先: 本関数は代入のみを行うため、他の関数を呼び出しません。

関連データ構造

  • md_srv_conf_t: mod_md におけるバーチャルホスト単位の設定を保持する構造体。

失敗モードと境界条件

  • to または from が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は構造体のメンバが増減した際に、コピー対象のメンバを漏れなく更新する必要があります。
  • ポインタ型のメンバ（ca_urls 等）は参照コピーであるため、コピー元とコピー先でメモリの生存期間が異なる場合に注意が必要です。
