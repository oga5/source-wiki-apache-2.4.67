ap_fcgi_encode_env

定義

  • 種別: function
  • 定義箇所: server/util_fcgi.c:179

目的

apr_table_t に格納された環境変数を、FastCGI プロトコル（RFC 3652）で規定される「長さ＋データ」形式のバイトストリームへエンコードします。

シグネチャまたは形状

apr_status_t ap_fcgi_encode_env(request_rec *r,
                                apr_table_t *env,
                                void *buffer,
                                apr_size_t buflen,
                                int *starting_elem)

入力と出力

  • **入力**:
  • r: リクエスト情報（エラーログ出力に使用）。
  • env: エンコード対象の環境変数テーブル。
  • buffer: エンコード結果を書き込むバッファ。
  • buflen: buffer の利用可能なサイズ。
  • starting_elem: 処理を開始するテーブル要素のインデックス（入出力両用）。
  • **出力**:
  • 戻り値: 成功時は APR_SUCCESS、バッファ不足時は APR_ENOSPC を返します。
  • starting_elem: 次回呼び出し時に処理を再開すべきインデックスが更新されます。

前提条件と不変条件

  • 本関数はメモリ割り当てを行いません。呼び出し元が提供した buffer の範囲内で操作を行います。
  • FastCGI の長さフィールドは、値が 127 を超える場合に最上位ビットを 1 に設定した 4 バイト形式（ビッグエンディアン）を使用します。

アルゴリズムまたは振る舞い

1. starting_elem で指定された位置から env テーブルを走査します。 2. 各キーと値について、その長さを FastCGI 形式（1バイトまたは4バイト）でエンコードし、続いて実際の文字列データをバッファへコピーします。 3. バッファ容量が不足した場合は直ちに中断し、APR_ENOSPC を返します。 4. 正常に書き込めた要素については starting_elem をインクリメントします。 5. 失敗時は ap_log_rerror を介してエラーログを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c
  • modules/proxy/mod_proxy_fcgi.c
  • **呼び出し先**:
  • apr_table_elts (APR)
  • strlen (標準ライブラリ)
  • memcpy (標準ライブラリ)
  • ap_log_rerror (Apache HTTP Server)

関連データ構造

  • apr_table_t: 環境変数を保持する Apache のテーブル構造。
  • request_rec: リクエストの状態を保持する構造体。

失敗モードと境界条件

  • **バッファオーバーフロー**: 指定された buflen に収まらない場合、APR_ENOSPC を返し、その時点までの進捗を starting_elem に保持して終了します。
  • **空のキー**: キーが NULL の要素はスキップされます。

保守担当者向けメモ

  • 本関数は分割送信をサポートしており、starting_elem を介して複数回に分けて呼び出されることを前提としています。
  • 長さフィールドのエンコードロジックは FastCGI プロトコル仕様に依存しているため、変更時は RFC 3652 を参照してください。
