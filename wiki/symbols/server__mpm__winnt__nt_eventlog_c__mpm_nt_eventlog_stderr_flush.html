mpm_nt_eventlog_stderr_flush

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/nt_eventlog.c:119

目的

サービス終了時に、標準エラー出力（stderr）のキャプチャ用パイプをクローズし、ログ転送用バックグラウンドスレッドの終了を待機してリソースを解放する。

シグネチャまたは形状

void mpm_nt_eventlog_stderr_flush(void)

入力と出力

  • 入力: なし（グローバル変数 stderr_thread を参照）
  • 出力: なし（副作用としてリソースの解放を行う）

前提条件と不変条件

  • stderr_thread が有効なハンドルを保持している場合にのみクリーンアップ処理が実行される。
  • サービス終了処理の一環として呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. グローバル変数 stderr_thread をローカル変数 cleanup_thread にコピーする。 2. cleanup_thread が有効な場合：

  • GetStdHandle(STD_ERROR_HANDLE) を介して標準エラー出力のハンドルを取得する。
  • fclose(stderr) を実行し、標準エラー出力ストリームを閉じる。
  • 取得したハンドルを CloseHandle で閉じる。
  • WaitForSingleObject を使用し、最大30秒間スレッドの終了を待機する。
  • 最後に cleanup_thread ハンドル自体を CloseHandle で閉じる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/mpm/winnt/service.c:671
  • 呼び出し先:
  • GetStdHandle (Windows API)
  • fclose (標準ライブラリ)
  • CloseHandle (Windows API)
  • WaitForSingleObject (Windows API)

関連データ構造

  • stderr_thread: ログ転送用バックグラウンドスレッドのハンドルを保持するグローバル変数。

失敗モードと境界条件

  • stderr_thread が NULL の場合、関数は何も行わずに終了する。
  • 30秒以内にスレッドが終了しない場合、待機はタイムアウトするが、関数はそのまま終了する。

保守担当者向けメモ

  • 本関数は stderr をパイプへ差し替える実装に依存しているため、他のモジュールが stderr を直接操作する場合に競合が発生する可能性がある。
  • サービス終了時のクリーンアップ順序として、他のリソース解放と整合性が取れているか注意が必要である。
