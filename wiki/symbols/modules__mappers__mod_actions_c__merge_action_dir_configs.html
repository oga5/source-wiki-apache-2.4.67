merge_action_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_actions.c:72

目的

ディレクトリごとの設定構造体（action_dir_config）をマージし、親ディレクトリの設定と現在のディレクトリの設定を統合した新しい設定を作成します。

シグネチャまたは形状

static void *merge_action_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定（action_dir_config *）。
  • addv: 現在のディレクトリの設定（action_dir_config *）。
  • **出力**:
  • 統合された新しい action_dir_config 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は action_dir_config 型としてキャスト可能であること。
  • apr_palloc により、新しい設定構造体はメモリプール p 上に確保されます。

アルゴリズムまたは振る舞い

1. basev と addv を action_dir_config 型にキャストします。 2. apr_table_overlay を使用して、add->action_types（現在の設定）を base->action_types（親の設定）の上に重ね合わせます。 3. METHODS 定数に基づきループし、各メソッドに対するスクリプト設定を統合します。現在の設定（add）に値が存在する場合はそれを優先し、存在しない場合は親の設定（base）を継承します。 4. configured フラグを、親または現在の設定のいずれかが設定済みであれば true となるように更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_actions.c:227 にて、モジュール設定のディレクトリマージ用関数として登録されています。
  • **呼び出し先**: apr_palloc（メモリ確保）、apr_table_overlay（テーブル統合）。

関連データ構造

  • action_dir_config: 本関数が操作するディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性があります。
  • basev または addv が NULL の場合の挙動については、ソースコード上での明示的なチェックは含まれていません。

保守担当者向けメモ

  • 本関数は Apache の設定マージフェーズで呼び出されます。
  • action_types の統合には apr_table_overlay を使用しており、現在の設定が親の設定を上書きする順序でマージされます。
  • scripted 配列のループ処理は METHODS 定数に依存しているため、HTTP メソッドの定義変更時には注意が必要です。
