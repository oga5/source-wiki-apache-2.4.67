uri_meets_conditions

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_util.c:33

目的

設定されたフィルタ URI（filter）と、対象となるリクエスト URI（url）およびパス（path）を照合し、キャッシュの適格性を判定します。

シグネチャまたは形状

static int uri_meets_conditions(const apr_uri_t *filter, const apr_size_t pathlen,
                                const apr_uri_t *url, const char *path)

入力と出力

  • **入力**:
  • filter: 照合条件となる apr_uri_t 構造体。
  • pathlen: path 引数の長さ。
  • url: 比較対象の apr_uri_t 構造体。
  • path: 比較対象のパス文字列。
  • **出力**:
  • 1: フィルタ条件に一致した場合。
  • 0: 一致しなかった場合。

前提条件と不変条件

  • filter にスキーム（scheme）が指定されていない場合、url にスキームやホスト名が含まれていると一致しません（ローカルパス専用とみなされます）。
  • filter にスキームが指定されている場合、url もスキームを持つ必要があり、大文字小文字を区別せず一致する必要があります。
  • path が NULL の場合、filter->path が / かつ pathlen が 1 である場合にのみ一致とみなされます。

アルゴリズムまたは振る舞い

1. **スキーム判定**: filter にスキームがない場合、url にスキームやホスト名があれば即座に 0 を返します。 2. **スキーム一致**: filter にスキームがある場合、url のスキームと比較します。 3. **ホスト名照合**: filter->hostname の先頭文字に応じて以下のルールで照合します。

  • .: url->hostname の末尾が filter->hostname と一致するか確認。
  • *: url->hostname の末尾が filter->hostname（先頭の * を除く）と一致するか確認。
  • その他: url->hostname と完全一致（大文字小文字無視）を確認。

    • 4. **ポート照合**: filter のポートが指定されている場合、url のポート（未指定時はスキームのデフォルトポート）と比較します。 5. **パス照合**: 最後に filter->path と path を pathlen 分だけ比較します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/cache_util.c 内の 203 行目および 221 行目（いずれもキャッシュ設定の照合処理）。
  • **呼び出し先**: ap_cstr_casecmp, strlen, strcasecmp, apr_uri_port_of_scheme, strncmp。

関連データ構造

  • apr_uri_t: Apache Portable Runtime (APR) で定義される URI 構造体。

失敗モードと境界条件

  • url->hostname が NULL の場合、ホスト名の照合ロジック内で適切に処理されます。
  • filter->hostname が空文字列の場合、ホスト名の照合はスキップされます。
  • filter->port_str が空の場合、ポートの照合はスキップされます。

保守担当者向けメモ

  • 本関数は mod_cache の設定フィルタリングにおいて、プロキシされた URI とローカルパスを区別して処理する重要な役割を担います。
  • ホスト名のワイルドカード照合（. や *）は、url->hostname の長さを考慮して安全に実行されています。
