filter_trie_node

定義

  • 種別: struct
  • 定義箇所: server/util_filter.c:53

目的

フィルタ名を高速に検索・管理するためのトライ木（Trie）構造のノードを定義します。

シグネチャまたは形状

struct filter_trie_node {
    /* 内部構造は server/util_filter.c:64-72 に定義 */
};

※構造体の詳細なメンバ構成は、ソースコードの定義箇所（64行目以降）を参照してください。

入力と出力

  • 入力: apr_pool_t（メモリ割り当て用）、親ノード、文字データ。
  • 出力: 新規作成された filter_trie_node 構造体へのポインタ。

前提条件と不変条件

  • フィルタ名の登録および検索に使用される。
  • registered_output_filters および registered_input_filters をルートとして保持する。
  • メモリ管理は APR プール（apr_pool_t）に依存する。

アルゴリズムまたは振る舞い

  • trie_node_alloc によりノードのメモリ確保と初期化が行われる。
  • trie_node_link により、親ノードと子ノードのリンクが構築される。
  • フィルタ名の一致判定を行うための階層的なデータ保持を行う。

呼び出し元と呼び出し先

  • 呼び出し元: trie_node_alloc および trie_node_link 内で参照。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • registered_output_filters: 出力フィルタ用トライ木のルート。
  • registered_input_filters: 入力フィルタ用トライ木のルート。
  • trie_node_link: ノード間を接続する関数。
  • trie_node_alloc: ノードを生成する関数。

失敗モードと境界条件

  • メモリ不足時: apr_palloc が失敗した場合の挙動は APR の仕様に準拠する。
  • 検索失敗時: 該当するフィルタ名が存在しない場合、トライ木の探索は終了する。

保守担当者向けメモ

  • 本構造体は server/util_filter.c 内でのみ使用される内部的なデータ構造です。
  • フィルタ名の追加・検索ロジックを変更する際は、トライ木のリンク構造（trie_node_link）の整合性に注意してください。
