ssl_var_lookup_ssl_cert

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:474

目的

X.509 証明書オブジェクトから、指定された変数名に対応する属性値（DN、SAN、有効期限、シリアル番号、署名アルゴリズム等）を文字列として抽出する。

シグネチャまたは形状

char *ssl_var_lookup_ssl_cert(apr_pool_t *p, request_rec *r, X509 *xs, char *var)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • r: 現在のリクエストレコード。
  • xs: 抽出対象の OpenSSL X509 証明書構造体。
  • var: 抽出する属性を指定する文字列（例: "M_SERIAL", "S_DN", "SAN_DNS"）。
  • **出力**: 抽出された属性値を含む文字列（apr_pool_t 上に確保）。該当なしまたはエラー時は NULL を返す。

前提条件と不変条件

  • 戻り値はすべて apr_pool_t 上に確保される。
  • var に基づき、内部で ssl_var_lookup_ssl_cert_serial や ssl_var_lookup_ssl_cert_dn などのヘルパー関数を呼び出す。
  • OpenSSL のバージョン（MODSSL_USE_OPENSSL_PRE_1_1_API 等）に応じて、署名アルゴリズムや公開鍵アルゴリズムの取得方法が条件分岐される。

アルゴリズムまたは振る舞い

1. var の内容を判定し、対応する証明書属性を特定する。 2. 属性に応じて以下の処理を行う：

  • **基本情報**: バージョン、シリアル番号、有効期限（開始・終了・残り）を抽出。
  • **DN (Distinguished Name)**: S_DN (Subject) または I_DN (Issuer) を指定された形式で取得。
  • **SAN (Subject Alternative Name)**: SAN_ で始まる指定に基づき抽出。
  • **アルゴリズム**: 署名 (A_SIG) および公開鍵 (A_KEY) のアルゴリズム名を NID から取得。
  • **PEM**: CERT 指定時に PEM 形式の証明書文字列を取得。

    • 3. 抽出された文字列が一時的なものであれば apr_pstrdup を用いてプール上に複製し、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_vars.c 内の ssl_var_lookup_ssl から呼び出される（ソース内の直接的な言及あり）。
  • **呼び出し先**:
  • ssl_var_lookup_ssl_cert_serial, ssl_var_lookup_ssl_cert_valid, ssl_var_lookup_ssl_cert_remain
  • ssl_var_lookup_ssl_cert_dn_oneline, ssl_var_lookup_ssl_cert_dn
  • ssl_var_lookup_ssl_cert_san, ssl_var_lookup_ssl_cert_PEM
  • OpenSSL API (X509_get_version, X509_get_subject_name 等)

関連データ構造

  • X509: OpenSSL の証明書構造体。
  • apr_pool_t: Apache のメモリ管理用プール。
  • request_rec: Apache のリクエスト情報構造体。

失敗モードと境界条件

  • var が未知の文字列である場合、NULL を返す。
  • xs が NULL の場合、OpenSSL API 呼び出し時にクラッシュする可能性がある（ガード条件は本関数内には明示されていない）。
  • 署名や鍵のアルゴリズムが不明な場合、"UNKNOWN" という文字列を返す。

保守担当者向けメモ

  • OpenSSL 1.1.0 以降の API 変更に伴う条件分岐が含まれているため、将来的な OpenSSL バージョンアップ時には A_SIG および A_KEY の取得ロジックを確認すること。
  • resdup フラグにより、ヘルパー関数が既にプール上に確保したメモリを再利用するか、apr_pstrdup で複製するかが制御されている。メモリリークを防ぐため、このフラグの管理には注意が必要。
