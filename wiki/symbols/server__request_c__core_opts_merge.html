core_opts_merge

定義

  • 種別: function
  • 定義箇所: server/request.c:614

目的

リクエスト処理中に、特定のディレクトリ設定（core_dir_config）からオプション設定（core_opts_t）をマージし、継承および上書きのロジックを適用する。

シグネチャまたは形状

static void core_opts_merge(const ap_conf_vector_t *sec, core_opts_t *opts)

入力と出力

  • **入力**:
  • sec: マージ対象となる設定ベクトル（ap_conf_vector_t）。
  • opts: 更新対象のオプション構造体（core_opts_t）。
  • **出力**: なし（opts が直接更新される）。

前提条件と不変条件

  • sec から取得した core_dir_config が NULL の場合、関数は何もせず終了する。
  • OPT_UNSET フラグの状態に基づき、加算（add）および削除（remove）のビット演算によるマージが行われる。

アルゴリズムまたは振る舞い

1. ap_get_core_module_config(sec) を呼び出し、設定ベクトルから core_dir_config を取得する。 2. this_dir->opts に OPT_UNSET が設定されている場合:

  • opts->add および opts->remove をビット演算で更新する。
  • opts->opts を add/remove の結果に基づいて再計算する。

    • 3. OPT_UNSET が設定されていない場合:

  • opts の各フィールドを this_dir の値で直接上書きする。

    • 4. this_dir->override に OR_UNSET が設定されていない場合:

  • opts->override および opts->override_opts を更新する。

    • 5. this_dir->override_list が NULL でない場合:

  • opts->override_list を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/request.c:1033
  • server/request.c:1099
  • server/request.c:1342
  • **呼び出し先**:
  • ap_get_core_module_config（マクロまたはインライン関数と推測される）

関連データ構造

  • core_dir_config: ディレクトリ単位の設定を保持する構造体。
  • core_opts_t: マージされたオプション状態を保持する構造体。
  • ap_conf_vector_t: 設定ベクトル。

失敗モードと境界条件

  • this_dir が NULL の場合、関数は早期リターンし、opts は変更されない。
  • override_list が NULL の場合、そのフィールドは更新されない。

保守担当者向けメモ

  • 本関数は設定の階層的なマージにおいて重要な役割を果たす。ビット演算によるオプションの継承ロジックを変更する際は、OPT_UNSET の挙動に注意すること。
  • 呼び出し元が複数存在するため、変更の影響範囲は server/request.c 内の広範囲に及ぶ可能性がある。
