ap_socache_instance_t

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_dc.c:35

目的

mod_socache_dc モジュールにおいて、分散キャッシュ（distcache）接続の構成情報と実行時のクライアントコンテキストを保持するために使用されます。

シグネチャまたは形状

struct ap_socache_instance_t {
    const char *target; /* 設定された接続先サーバー */
    DC_CTX *dc;         /* distcache クライアントコンテキスト */
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは以下の通りです。

  • target: キャッシュサーバーの接続先を示す文字列（入力設定）。
  • dc: distcache ライブラリが管理するクライアントコンテキストへのポインタ（内部状態）。

前提条件と不変条件

  • dc フィールドは、socache_dc_init による初期化が成功するまで有効なコンテキストを指しません。
  • 本構造体は Apache のメモリプール（apr_pool_t）によってライフサイクルが管理されます。

アルゴリズムまたは振る舞い

本構造体は、mod_socache_dc が提供する各キャッシュ操作（Store, Retrieve, Remove 等）において、対象となる分散キャッシュサーバーへのアクセスポイントとして機能します。socache_dc_create で生成され、socache_dc_destroy で DC_CTX_free を呼び出してリソースが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cache_socache などの上位モジュールが、ap_socache_provider_t インターフェースを通じて本構造体へのポインタを操作します。
  • **呼び出し先**: distcache ライブラリの API（DC_CTX_new, DC_CTX_free, DC_CTX_add_session 等）が、本構造体内の dc フィールドを引数として使用します。

関連データ構造

  • DC_CTX: distcache ライブラリのクライアントコンテキスト構造体。
  • ap_socache_provider_t: 本構造体を利用するプロバイダーのインターフェース定義。

失敗モードと境界条件

  • target が未設定または不正な場合、socache_dc_init 呼び出し時に接続失敗が発生します。
  • distcache ライブラリのバージョン不整合がある場合、コンパイル時または実行時にエラーとなります。

保守担当者向けメモ

  • 本構造体は ap_socache.h でも定義されており、プロバイダー間で共通のインターフェースとして扱われます。
  • 現在、SESSION_CTX_FLAG_PERSISTENT は無効化されており、接続は操作ごとに確立される設計となっています。
