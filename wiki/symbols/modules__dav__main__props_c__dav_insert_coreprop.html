dav_insert_coreprop

定義

  • 種別: function
  • 定義箇所: modules/dav/main/props.c:338

目的

WebDAVのコアプロパティ（DAV: 名前空間）を解決し、その値をXML形式で出力バッファに挿入します。

シグネチャまたは形状

dav_error * dav_insert_coreprop(dav_propdb *propdb,
                                int propid, const char *name,
                                dav_prop_insert what,
                                apr_text_header *phdr,
                                dav_prop_insert *inserted)

入力と出力

  • **入力**:
  • propdb: プロパティデータベースのコンテキスト。
  • propid: 挿入対象のプロパティID。
  • name: プロパティ名。
  • what: 挿入モード（値の挿入、サポート状況の確認など）。
  • phdr: 出力用テキストヘッダー。
  • **出力**:
  • inserted: 実際に挿入されたモード（DAV_PROP_INSERT_NOTDEF 等）。
  • 戻り値: 成功時は NULL、エラー発生時は dav_error 構造体。

前提条件と不変条件

  • propdb が有効なコンテキストであること。
  • propid が DAV_PROPID_CORE_UNKNOWN の場合、処理は即座に終了する。
  • getcontenttype や getcontentlanguage の処理において、propdb->subreq が未初期化の場合は dav_do_prop_subreq が呼び出され、サブリクエストが生成される。

アルゴリズムまたは振る舞い

1. propid に基づき、lockdiscovery, supportedlock, getcontenttype, getcontentlanguage の各コアプロパティをスイッチ文で分岐処理する。 2. ロック関連プロパティの場合、propdb->lockdb を介して情報を取得し、必要に応じて dav_lock_get_activelock で値を生成する。 3. コンテンツ関連プロパティの場合、サブリクエストのヘッダー情報から値を取得する。 4. 値が取得できた場合、what パラメータに応じて以下のXML形式を phdr に追加する。

  • DAV_PROP_INSERT_SUPPORTED: <D:supported-live-property .../>
  • DAV_PROP_INSERT_VALUE: <D:name>value</D:name>
  • その他: <D:name/>

    • 5. 挿入が完了したことを inserted に記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/props.c 内の複数箇所（行466, 686, 689, 696, 703, 943）。
  • **呼び出し先**: dav_lock_query, dav_lock_get_activelock, dav_do_prop_subreq, apr_pstrdup, apr_table_get, apr_pstrcat, apr_text_append。

関連データ構造

  • dav_propdb: プロパティ操作のコンテキスト。
  • dav_error: エラー報告用構造体。
  • apr_text_header: 出力バッファ。

失敗モードと境界条件

  • ロック情報の取得に失敗した場合、dav_push_error を介してエラーが返される。
  • value が空文字列の場合、XMLタグの挿入はスキップされる（DAV_PROP_INSERT_VALUE モード時）。
  • propid がコアプロパティとして定義されていない場合、デフォルトケースにより何も挿入されず終了する。

保守担当者向けメモ

  • 本関数は DAV: 名前空間のプロパティを扱うため、将来的に新しいコアプロパティを追加する場合はスイッチ文を拡張する。
  • propdb->subreq のライフサイクル管理に注意すること（dav_close_propdb で破棄される）。
