ajp_msg_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/ajp.h:109

目的

Apache HTTP Server とサーブレットコンテナ（Tomcat 等）間で AJP (Apache Jserv Protocol) v13 通信を行う際、メッセージのメモリレイアウトと状態を保持するために使用されます。

シグネチャまたは形状

typedef struct ajp_msg ajp_msg_t;

※構造体の内部メンバは本ヘッダーファイル内では不透明（opaque）として定義されています。

入力と出力

  • **入力**: ajp_msg_create 等で初期化され、ajp_msg_append_* 関数群を通じてデータが書き込まれます。
  • **出力**: ajp_msg_get_* 関数群を通じてバッファ内のデータが抽出されます。また、ajp_ilink_send を介してソケットへ送信されます。

前提条件と不変条件

  • メモリ管理は apr_pool_t に依存しており、リクエストの生存期間に紐付きます。
  • メッセージサイズには物理的な上限（AJP_MSG_BUFFER_SZ: 8KB、AJP_MAX_BUFFER_SZ: 64KB）が存在します。
  • 内部の pos（読み取り位置）と len（メッセージ長）により、シリアライズおよびデシリアライズの状態が管理されます。

アルゴリズムまたは振る舞い

1. **構築**: ajp_msg_create によりプールからメモリが確保されます。 2. **シリアライズ**: ajp_msg_append_* 関数群により、ヘッダーやボディデータがバッファへ順次書き込まれます。 3. **デシリアライズ**: ajp_msg_get_* や ajp_msg_peek_* 関数群により、バッファからデータが読み出されます。 4. **状態管理**: ajp_msg_reset や ajp_msg_reuse により、メッセージ構造体を再利用可能な状態にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy_ajp.c および ajp_header.c 等の AJP プロトコルスタックの実装。
  • **呼び出し先**: ajp_msg_check_header, ajp_msg_reset, ajp_msg_reuse, ajp_msg_end, ajp_msg_append_*, ajp_msg_get_*, ajp_msg_peek_* などの操作関数群。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • ajp_msg (実装体): ajp_msg.c にて定義される実際の構造体。

失敗モードと境界条件

  • バッファサイズ制限（8KB または 64KB）を超過した場合、エラーが発生します。
  • EBCDIC 環境では、文字列操作時に ajp_msg_append_string_ex を通じた適切な変換が必要です。
  • 独自のエラーコード AJP_E* が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は ajp.h では前方宣言されており、具体的なメンバアクセスは ajp_msg.c 内の関数を介して行う設計となっています。
  • 構造体の詳細なメモリレイアウトを確認する場合は ajp_msg.c を参照してください。
