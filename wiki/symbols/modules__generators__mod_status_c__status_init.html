status_init

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_status.c:1003

目的

mod_status モジュールの初期化を行い、サーバーの稼働状態を示すフラグの定義と、MPM（Multi-Processing Module）から各種制限値を取得して保持する。

シグネチャまたは形状

static int status_init(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *s)

入力と出力

  • 入力:
  • p, plog, ptemp: APR プール（メモリ管理用）
  • s: サーバー設定レコード
  • 出力:
  • 戻り値: OK（常に成功を返す）

前提条件と不変条件

  • status_flags 配列がグローバルまたはファイルスコープで定義されていること。
  • thread_limit, server_limit, threads_per_child, max_servers, is_async といった変数が、MPM からの情報を格納するために利用可能であること。

アルゴリズムまたは振る舞い

1. **フラグの初期化**: status_flags 配列に対し、スコアボードの各状態（SERVER_READY 等）に対応する文字表現（., _, S, R, W 等）を割り当てる。 2. **MPM 情報の取得**: ap_mpm_query を使用して以下の情報を取得する。

  • AP_MPMQ_HARD_LIMIT_THREADS (スレッド制限)
  • AP_MPMQ_HARD_LIMIT_DAEMONS (プロセス制限)
  • AP_MPMQ_MAX_THREADS (子プロセスあたりの最大スレッド数)
  • AP_MPMQ_MAX_DAEMONS (最大サーバー数)
  • AP_MPMQ_IS_ASYNC (非同期処理の可否)

    • 3. **補正**: threads_per_child が 0 の場合、1 に補正する（MPM のバグ回避）。 4. OK を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: ap_hook_post_config により、サーバー設定のポストコンフィグフェーズで呼び出される（modules/generators/mod_status.c:1040）。
  • 呼び出し先: ap_mpm_query（Apache MPM API）。

関連データ構造

  • status_flags: スコアボードの状態を文字にマッピングする配列。
  • server_rec: Apache サーバー設定構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • threads_per_child が 0 を返す MPM の場合、明示的に 1 に補正する処理が含まれている。
  • ap_mpm_query の戻り値については本関数内ではチェックされていない。

保守担当者向けメモ

  • 本関数は post_config フックとして登録されており、サーバー起動時の初期化フェーズで一度だけ実行される。
  • status_flags のマッピングはスコアボードの定義順序に依存しない設計となっている。
