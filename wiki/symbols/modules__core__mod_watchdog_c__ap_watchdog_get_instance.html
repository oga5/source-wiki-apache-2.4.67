ap_watchdog_get_instance

定義

  • 種別: function
  • 定義箇所: modules/core/mod_watchdog.c:291

目的

指定された名前の ap_watchdog_t インスタンスを取得、または存在しない場合は新規作成してプロバイダとして登録します。

シグネチャまたは形状

apr_status_t ap_watchdog_get_instance(ap_watchdog_t **watchdog,
                                      const char *name,
                                      int parent,
                                      int singleton,
                                      apr_pool_t *p);

入力と出力

  • **入力**:
  • name: 取得対象の Watchdog インスタンス名。
  • parent: 親プロセス用インスタンスかを示すフラグ（非ゼロで真）。
  • singleton: シングルトンモードで動作させるかを示すフラグ（非ゼロで真）。
  • p: インスタンス生成に使用するメモリプール。
  • **出力**:
  • watchdog: 取得または生成された ap_watchdog_t 構造体へのポインタ。
  • 戻り値: APR_SUCCESS（成功）、APR_ENOTIMPL（親プロセス用がサポートされない場合）、またはプロバイダ登録時のエラーコード。

前提条件と不変条件

  • parent が真かつ mpm_is_forked が AP_MPMQ_NOT_SUPPORTED でない場合、処理は中断され APR_ENOTIMPL を返します。
  • singleton フラグは parent が真の場合は強制的に 0（偽）として扱われます。

アルゴリズムまたは振る舞い

1. parent フラグに基づき、プロバイダのバージョン文字列（AP_WATCHDOG_PVERSION または AP_WATCHDOG_CVERSION）を決定します。 2. ap_lookup_provider を使用し、既存のプロバイダから指定された name のインスタンスを検索します。見つかった場合はそれを返し終了します。 3. 既存のインスタンスがない場合、apr_pcalloc で ap_watchdog_t をメモリ確保します。 4. 確保した構造体に name、pool、singleton 設定を格納します。 5. ap_register_provider を呼び出し、生成したインスタンスをプロバイダとして登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_watchdog.c 内の各フック関数（wd_pre_config_hook 等）から呼び出されます。また、mod_heartmonitor.c、mod_md_drive.c、mod_md_ocsp.c、mod_proxy_hcheck.c 等の外部モジュールから APR_RETRIEVE_OPTIONAL_FN を介して参照されています。
  • **呼び出し先**: ap_lookup_provider, apr_pcalloc, ap_register_provider（いずれも Apache/APR 提供の API）。

関連データ構造

  • ap_watchdog_t: 監視インスタンスのメイン構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • parent が真で、かつフォークをサポートする MPM 環境下では APR_ENOTIMPL を返します。
  • プロバイダの登録に失敗した場合、そのエラーコードが戻り値となります。

保守担当者向けメモ

  • 本関数は APR_DECLARE_OPTIONAL_FN として公開されており、他のモジュールが動的にリンクして利用することを前提としています。
  • singleton モードの挙動は、parent フラグが優先されるため、親プロセス用として生成する場合はシングルトン設定が無効化される点に注意してください。
