authn_provider_list

定義

  • 種別: struct
  • 定義箇所: include/mod_auth.h:95

目的

認証プロバイダーを連結リスト形式で保持し、複数の認証プロバイダーを順次試行するためのデータ構造を提供します。

シグネチャまたは形状

struct authn_provider_list {
    const char *provider_name;
    const char *provider_args;
    const authn_provider *provider;
    struct authn_provider_list *next;
};

入力と出力

  • **入力**: 認証設定時に provider_name（プロバイダー名）や provider_args（引数）が設定されます。
  • **出力**: リストの next ポインタを辿ることで、登録された認証プロバイダーのシーケンスを順次参照可能です。

前提条件と不変条件

  • next ポインタはリストの終端で NULL である必要があります。
  • provider メンバは、ap_lookup_provider 等を通じて取得された有効な authn_provider 構造体を指す必要があります。

アルゴリズムまたは振る舞い

  • 設定読み込み時に apr_pcalloc を用いてメモリが確保され、リストに追加されます。
  • 認証処理時には、リストの先頭から順に provider が保持する関数ポインタ（check_password 等）を呼び出し、認証の成否を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_basic.c および modules/aaa/mod_auth_digest.c において、設定ディレクティブの解析時および認証実行時に参照・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出しません。保持している provider メンバが指す関数が実行されます。

関連データ構造

  • authn_provider: 各プロバイダーが実装すべき関数ポインタを保持する構造体。
  • request_rec: 認証処理のコンテキストとして利用されます。

失敗モードと境界条件

  • リストが空の場合、認証プロバイダーが存在しない状態として扱われます。
  • メモリ確保失敗時は apr_pcalloc の挙動に依存します。

保守担当者向けメモ

  • 本構造体は mod_auth_basic や mod_auth_digest といった認証モジュール間で共通利用されるため、変更時はこれらモジュールへの影響を考慮してください。
  • リストの走査順序は設定ファイルでの記述順に依存します。
