md_curl_internals_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_curl.c:54

目的

libcurl を使用した HTTP リクエスト処理において、リクエストごとのコンテキスト（ハンドル、ヘッダー、レスポンス状態）を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    CURL *curl;
    CURLM *curlm;
    struct curl_slist *req_hdrs;
    md_http_response_t *response;
    apr_status_t rv;
    int status_fired;
} md_curl_internals_t;

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは md_curl モジュール内のリクエスト処理フローにおいて、libcurl のハンドル管理やレスポンスの格納に使用されます。

前提条件と不変条件

  • libcurl のハンドルはスレッドセーフではないため、本構造体はリクエストごとのプール内で管理される必要があります。
  • リクエスト終了時には md_curl_req_cleanup を通じて適切にリソースを解放する必要があります。

アルゴリズムまたは振る舞い

  • curl および curlm フィールドは、libcurl の easy_handle および multi_handle を保持し、通信の実行を制御します。
  • req_hdrs は送信する HTTP ヘッダーのリストを管理します。
  • response は受信したレスポンスデータを格納します。
  • status_fired は、レスポンス処理が完了したかどうかを追跡するフラグとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_curl.c 内の複数の関数（internals_setup、md_curl_perform、md_curl_multi_perform 等）で、リクエストコンテキストとして参照されています。
  • 呼び出し先: 本構造体自体はデータ構造であり、直接的な呼び出し先はありません。

関連データ構造

  • CURL, CURLM: libcurl が提供するハンドル構造体。
  • struct curl_slist: libcurl のヘッダーリスト。
  • md_http_response_t: md_http モジュールで定義されるレスポンス保持用構造体。

失敗モードと境界条件

  • resp_limit を超えるレスポンスボディを受信した場合、resp_data_cb コールバックにより拒否され、エラーとして処理されます。
  • メモリ管理は apr_pool_t に依存しており、プールが破棄される前に適切にクリーンアップされない場合、リソースリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は md_http_request_t のプールと密接に紐付いています。
  • curl_multi の詳細なエラーハンドリングやプラットフォーム依存の SSL オプションについては、実装の深部を参照してください。
