ap_get_output_filter_handle

定義

  • 種別: function
  • 定義箇所: server/util_filter.c:193

目的

指定された名前の出力フィルタ定義（ap_filter_rec_t）を、システムに登録済みの出力フィルタ群から検索して取得する。

シグネチャまたは形状

ap_filter_rec_t *ap_get_output_filter_handle(const char *name)

入力と出力

  • **入力**: name (検索対象のフィルタ名を表す文字列)
  • **出力**: 該当する ap_filter_rec_t 構造体へのポインタ。見つからない場合は NULL を返す。

前提条件と不変条件

  • registered_output_filters が初期化済みであること。
  • フィルタ名は register_filter 等を通じて事前に登録されている必要がある。

アルゴリズムまたは振る舞い

内部関数 get_filter_handle を呼び出し、グローバル変数 registered_output_filters（フィルタ名を保持するトライ木）を探索して、一致するフィルタ定義を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_filter.c:448
  • server/util_filter.c:519
  • **呼び出し先**:
  • get_filter_handle (内部関数)

関連データ構造

  • ap_filter_rec_t: フィルタの定義情報を保持する構造体。
  • registered_output_filters: 出力フィルタを管理するトライ木構造のルート。

失敗モードと境界条件

  • 指定した name が登録されていない場合、NULL を返す。
  • name が NULL の場合の挙動は get_filter_handle の実装に依存する。

保守担当者向けメモ

  • 本関数は include/util_filter.h で公開されている API である。
  • フィルタの検索はトライ木構造で行われるため、計算量はフィルタ名の長さに依存する。
