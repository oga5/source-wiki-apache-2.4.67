exploded_time_cache_element

定義

  • 種別: struct
  • 定義箇所: server/util_time.c:34

目的

時刻変換処理を高速化するためのキャッシュ要素として、分解された時刻情報と整合性検証用のタイムスタンプを保持します。

シグネチャまたは形状

struct exploded_time_cache_element {
    apr_int64_t t;
    apr_time_exp_t xt;
    apr_int64_t t_validate;
};

入力と出力

本構造体はデータ保持用であり、直接的な入力・出力関数を持ちません。cached_explode() 関数が本構造体のインスタンスを読み書きし、時刻の分解結果を格納・取得します。

前提条件と不変条件

  • t はキャッシュされた時刻（秒単位）を保持します。
  • xt は t に対応する分解された時刻情報（apr_time_exp_t）を保持します。
  • t_validate は、マルチスレッド環境での整合性検証に使用されます。詳細は cached_explode() 内のコメントを参照してください。

アルゴリズムまたは振る舞い

本構造体はリングバッファの一部として使用されます。cached_explode() は、現在の時刻からインデックスを計算して本構造体へアクセスし、スナップショットを採取することで、ロックフリーなキャッシュ読み取りを実現します。

呼び出し元と呼び出し先

  • 呼び出し元: server/util_time.c 内の cached_explode() 関数が、キャッシュの読み書きおよび整合性検証のために本構造体を参照します。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出しません。

関連データ構造

  • apr_time_exp_t: APR が提供する分解された時刻構造体。
  • exploded_cache_localtime / exploded_cache_gmt: 本構造体の配列として定義されたリングバッファ。

失敗モードと境界条件

  • キャッシュの整合性検証（t と t_validate の比較）に失敗した場合、データは不整合とみなされ、標準の apr_time_exp_* 関数による再計算が行われます。
  • AP_TIME_RECENT_THRESHOLD（デフォルト15秒）を超える古い時刻に対しては、キャッシュは機能しません。

保守担当者向けメモ

  • 本構造体はロックフリーなキャッシュ機構の一部です。メンバの追加や変更を行う際は、cached_explode() におけるスナップショット採取と整合性検証のロジックに影響がないか確認してください。
