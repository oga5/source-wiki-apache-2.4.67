merge_isapi_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_isapi.c:96

目的

Apache の設定マージ処理において、親ディレクトリ（base）の設定と、子ディレクトリまたはリクエスト固有の追加設定（add）を統合し、新しいディレクトリ設定構造体を作成する。

シグネチャまたは形状

static void *merge_isapi_dir_configs(apr_pool_t *p, void *base_, void *add_)

入力と出力

  • **入力**:
  • p: 設定の割り当てに使用するメモリプール。
  • base_: 親ディレクトリの設定（isapi_dir_conf 型）。
  • add_: 現在のディレクトリで追加された設定（isapi_dir_conf 型）。
  • **出力**:
  • 統合された新しい isapi_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • base_ および add_ は isapi_dir_conf 構造体へのポインタとしてキャスト可能であること。
  • add 側の各設定値が ISAPI_UNDEF である場合、base 側の値が継承される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、新しい isapi_dir_conf 構造体用のメモリをプール p 上に確保する。 2. 以下の各設定項目について、add 側の値が ISAPI_UNDEF かどうかを判定する。

  • read_ahead_buflen
  • log_unsupported
  • log_to_errlog
  • log_to_query
  • fake_async

    • 3. add 側の値が ISAPI_UNDEF であれば base 側の値を採用し、そうでなければ add 側の値を採用して新しい構造体に格納する。 4. 統合された構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_isapi.c:1723 （Apache の設定マージフックとして登録されている）。
  • **呼び出し先**: 直接的な関数呼び出しは行わず、メモリ確保のために apr_palloc を使用する。

関連データ構造

  • isapi_dir_conf: ディレクトリごとの設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある。
  • base_ または add_ が NULL の場合の挙動については、本ソースコード内では明示的なチェックは行われていない。

保守担当者向けメモ

  • 本関数は ISAPI_UNDEF を用いた階層的な設定継承を実装している。新しい設定項目を isapi_dir_conf に追加した場合は、必ず本関数内でのマージロジックにも反映させること。
