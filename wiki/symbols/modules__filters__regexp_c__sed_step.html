sed_step

定義

  • 種別: function
  • 定義箇所: modules/filters/regexp.c:358

目的

入力文字列 p1 に対して、コンパイル済み正規表現バイトコード p2 を用いたパターンマッチングを実行する。

シグネチャまたは形状

int sed_step(char *p1, char *p2, int circf, step_vars_storage *vars)

入力と出力

  • **入力**:
  • p1: 検索対象の文字列（開始位置）。
  • p2: コンパイル済み正規表現バイトコード。
  • circf: 行頭マッチフラグ（非ゼロで有効）。
  • vars: マッチング状態（loc1 等）を保持する step_vars_storage 構造体。
  • **出力**:
  • int: マッチした場合は 1、マッチしなかった場合は 0 を返す。

前提条件と不変条件

  • vars は呼び出し元によって適切に初期化されている必要がある。
  • p2 は sed_compile によって生成された有効なバイトコードであること。

アルゴリズムまたは振る舞い

1. **行頭マッチング**: circf が真の場合、_advance を呼び出し、その結果を直接返す。 2. **高速検索**: p2 の先頭が CCHR（単一文字）である場合、その文字が p1 内に出現する位置までスキップし、_advance による詳細マッチングを試みる。 3. **全走査**: 上記以外の場合、p1 の各位置から _advance を呼び出し、マッチするまで繰り返す。 4. **状態更新**: マッチ成功時、vars->loc1 にマッチ開始位置を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/sed1.c（sed_step を呼び出していることが確認されている）。
  • **呼び出し先**: _advance（内部エンジン）。

関連データ構造

  • step_vars_storage: マッチングの境界（loc1）やキャプチャグループを保持する構造体。
  • regexp.h: バイトコード定数（CCHR 等）の定義。

失敗モードと境界条件

  • p1 が終端に達した場合は 0 を返す。
  • _advance がマッチに失敗した場合は、次の文字位置へ進めて再試行する。

保守担当者向けメモ

  • 本関数は _advance を呼び出すラッパーであり、マッチングの開始位置を制御する。
  • CCHR による最適化は、正規表現の先頭が固定文字である場合に有効である。
  • _advance 内の再帰的なバックトラッキングロジックに依存しているため、複雑な正規表現のデバッグ時は _advance の挙動を併せて確認すること。
