h2_push

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_push.h:30

目的

HTTP/2 Server Push における個別のプッシュ対象リソースを表現し、リクエスト情報と優先度を保持するための構造体です。

シグネチャまたは形状

typedef struct h2_push {
    const struct h2_request *req;
    h2_priority *priority;
} h2_push;

入力と出力

  • **入力**: h2_push_collect 関数によってレスポンスヘッダーから抽出されたリクエスト情報および優先度情報が本構造体に格納されます。
  • **出力**: h2_push_diary_update 関数において、重複排除のためのフィルタリング対象として参照されます。

前提条件と不変条件

  • req はプッシュ対象となるリクエストの定義を指し示します。
  • priority は当該プッシュリソースの優先度を管理します。

アルゴリズムまたは振る舞い

1. **収集**: h2_push_collect がレスポンス内の Link ヘッダー等を解析し、h2_push オブジェクトの配列を生成します。 2. **フィルタリング**: h2_push_diary_update がセッション内の h2_push_diary と照合し、既に送信済みのリソースを除外します。 3. **管理**: フィルタリングを通過したリソースのみが、後続のストリーム処理でプッシュ実行対象となります。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_push.c 内のロジックが本構造体をインスタンス化し、管理します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • **注記**: h2_config.c に存在する h2_push という名称の変数は、本構造体とは異なる設定値（HTTP/2 Server Push の有効/無効フラグ）であり、混同に注意が必要です。

関連データ構造

  • h2_request: プッシュ対象のリクエスト詳細。
  • h2_priority: プッシュリソースの優先度定義。
  • h2_push_diary: 過去のプッシュ履歴を保持し、重複送信を抑制するための構造体。

失敗モードと境界条件

  • **メモリ制約**: h2_push_diary のエントリ数制限により、履歴管理が飽和した場合は古いエントリが破棄されます。
  • **設定依存**: h2_config で Server Push が無効化されている場合、本構造体を用いたプッシュ処理はスキップされます。

保守担当者向けメモ

  • 本構造体は h2_push.c で実装されるプッシュロジックの基盤です。
  • h2_config.c に同名の変数 h2_push が存在しますが、これは設定値であり、本構造体とは別物です。コード検索時には型定義の有無を確認してください。
