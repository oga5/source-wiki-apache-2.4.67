ssl_var_lookup_ssl_cert_dn

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:586

目的

X.509 証明書の識別名（DN）から、指定された属性名に対応する値を抽出して文字列として返します。

シグネチャまたは形状

static char *ssl_var_lookup_ssl_cert_dn(apr_pool_t *p, X509_NAME *xsname, const char *var)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • xsname: 検索対象の X509_NAME 構造体。
  • var: 検索する属性名（例: O, OU, CN）。末尾に _RAW を付与することで未加工の値を指定可能。また、_N（Nは整数）を付与して同一属性のN番目のエントリを指定可能。
  • **出力**:
  • 成功時: 抽出された属性値の文字列（apr_pool_t 上に確保）。
  • 失敗時: NULL。

前提条件と不変条件

  • var に指定する属性名は、内部テーブル ssl_var_lookup_ssl_cert_dn_rec に定義されている必要があります。
  • 戻り値のメモリ管理は呼び出し元が apr_pool_t を通じて行います。

アルゴリズムまたは振る舞い

1. **修飾子解析**: var の末尾に _RAW があればフラグを立て、_N があればインデックス idx を抽出します。 2. **属性検索**: ssl_var_lookup_ssl_cert_dn_rec テーブルを走査し、var と一致する名前を持つエントリを探します。 3. **エントリ抽出**: 一致したエントリの NID（Name ID）に基づき、X509_NAME 内の全エントリを走査します。 4. **値の取得**: NID が一致し、かつ指定されたインデックス idx に該当する場合、modssl_X509_NAME_ENTRY_to_string を呼び出して文字列化し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_vars.c 内の ssl_var_lookup_ssl_cert（519行目）。
  • **呼び出し先**:
  • modssl_X509_NAME_ENTRY_to_string
  • OpenSSL API: X509_NAME_entry_count, X509_NAME_get_entry, OBJ_obj2nid, X509_NAME_ENTRY_get_object

関連データ構造

  • X509_NAME: OpenSSL の証明書識別名構造体。
  • ssl_var_lookup_ssl_cert_dn_rec: 属性名と NID のマッピングを保持する内部テーブル。

失敗モードと境界条件

  • 指定された属性名がテーブルに存在しない場合、NULL を返します。
  • 証明書内に該当する属性が存在しない、またはインデックス idx が範囲外の場合、NULL を返します。
  • var の形式が不正な場合、期待する値が取得できない可能性があります。

保守担当者向けメモ

  • 本関数は ssl_var_lookup_ssl_cert_dn_rec テーブルに依存しています。新しい属性を追加する場合は、このテーブルの更新が必要です。
  • _RAW フラグの処理は modssl_X509_NAME_ENTRY_to_string に委譲されています。文字列変換のロジックを変更する場合はそちらを確認してください。
