lua_get_result_set

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:45

目的

Lua スタック上のテーブルから、内部的なデータベース結果セット構造体（lua_db_result_set）を取得します。

シグネチャまたは形状

static lua_db_result_set *lua_get_result_set(lua_State *L)

入力と出力

  • **入力**: lua_State *L（Lua 仮想マシンの状態）。スタックのインデックス 1 に LUA_TTABLE が配置されている必要があります。
  • **出力**: lua_db_result_set *（抽出された結果セットへのポインタ）。

前提条件と不変条件

  • スタックのインデックス 1 に Lua テーブルが存在すること。
  • そのテーブルのインデックス 0 に、LUA_TUSERDATA 型として lua_db_result_set へのポインタが格納されていること。

アルゴリズムまたは振る舞い

1. luaL_checktype を使用し、スタックのインデックス 1 がテーブルであることを検証します。 2. lua_rawgeti を呼び出し、テーブルのインデックス 0 にある値をスタックトップにプッシュします。 3. プッシュされた値がユーザーデータ型であることを luaL_checktype で確認します。 4. lua_topointer を使用して、ユーザーデータから lua_db_result_set 構造体へのポインタを取り出し、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_dbd.c:235 にて、結果セットを操作する関数から呼び出されています。
  • **呼び出し先**: luaL_checktype、lua_rawgeti、lua_topointer（いずれも Lua C API）。

関連データ構造

  • lua_db_result_set: データベースクエリの結果を保持する構造体。

失敗モードと境界条件

  • インデックス 1 がテーブルでない場合、またはインデックス 0 にユーザーデータが存在しない（あるいは型が異なる）場合、luaL_checktype により Lua エラーが発生し、処理が中断されます。

保守担当者向けメモ

  • 本関数は、Lua 側で管理されているデータベース結果セットのラッパーテーブルから、C 側の実体を取り出すためのヘルパーです。
  • スタック操作に依存しているため、呼び出し前にスタックの状態が期待通りであることを確認してください。
