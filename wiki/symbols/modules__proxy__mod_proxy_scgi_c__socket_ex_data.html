socket_ex_data

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:84

目的

SOCKET_EX バケットにおいて、通信対象のソケットと、その読み込み量を追跡するための統計カウンタを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_socket_t *sock;
    apr_off_t *counter;
} socket_ex_data;

入力と出力

  • sock: 通信に使用する apr_socket_t へのポインタ。
  • counter: 読み込まれたバイト数を累積するための apr_off_t 型変数へのポインタ。

前提条件と不変条件

  • sock は有効な apr_socket_t を指している必要があります。
  • counter は、バケットの読み込み処理中に読み取ったバイト数を加算するために使用されるため、呼び出し元で適切に初期化・管理されている必要があります。

アルゴリズムまたは振る舞い

この構造体は、apr_bucket_socket をラップするカスタムバケットの実装において、ソケット I/O と統計情報の更新を紐付けるためのコンテナとして機能します。バケットの読み込み処理（bucket_socket_ex_read）の過程で、この構造体を通じてソケットからデータを取得し、同時に counter を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: bucket_socket_ex_create および bucket_socket_ex_make がこの構造体を初期化・利用します。また、scgi_handler 等の処理フロー内で sock_data として参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • bucket_type_socket_ex: この構造体を利用するカスタムバケット型。
  • apr_socket_t: APR のソケット抽象化レイヤー。
  • apr_off_t: バイト数カウントに使用される型。

失敗モードと境界条件

  • sock が NULL の場合、または counter が無効なメモリ領域を指している場合、バケット読み込み時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_proxy_scgi 内の統計追跡専用です。apr_bucket_socket の標準的な挙動に統計更新のロジックを注入する目的で設計されています。
  • 構造体のメンバーへのアクセスは、bucket_socket_ex_read 等のバケット操作関数内で行われるため、これらの関数を変更する際は整合性に注意してください。
