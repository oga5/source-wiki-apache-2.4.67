dav_fs_copymove_state

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/repos.c:475

目的

WebDAV の COPY または MOVE 操作に伴い、リソースに関連付けられた内部状態（プロパティ等）を格納するディレクトリおよびファイルを、ソースから宛先へ複製または移動する。

シグネチャまたは形状

static dav_error * dav_fs_copymove_state(
    int is_move,
    apr_pool_t * p,
    const char *src_dir, const char *src_file,
    const char *dst_dir, const char *dst_file,
    dav_buffer *pbuf)

入力と出力

  • is_move: 移動操作の場合は非ゼロ、コピー操作の場合は 0。
  • p: メモリ割り当て用プール。
  • src_dir, src_file: ソースリソースのディレクトリおよびファイル名。
  • dst_dir, dst_file: 宛先リソースのディレクトリおよびファイル名。
  • pbuf: dav_fs_copymove_file に渡されるバッファ。
  • 戻り値: 成功時は NULL、失敗時はエラー詳細を含む dav_error 構造体へのポインタ。

前提条件と不変条件

  • src_dir および dst_dir 内には DAV_FS_STATE_DIR という名前のディレクトリが存在し、そこに状態ファイルが格納されている必要がある。
  • dst_dir 内の DAV_FS_STATE_DIR はディレクトリである必要がある（ファイルであってはならない）。

アルゴリズムまたは振る舞い

1. ソースの状態ファイルパスを構築し、存在を確認する。存在しない場合は何もしない。 2. 宛先の状態ディレクトリパスを構築し、存在しない場合は作成を試みる。 3. 宛先状態ディレクトリの情報を取得し、ディレクトリであることを検証する。 4. 宛先の状態ファイルパスを構築する。 5. is_move が真の場合:

  • apr_file_rename による直接的なリネームを試みる。
  • デバイス境界を越える場合（APR_STATUS_IS_EXDEV）は dav_fs_copymove_file を呼び出す。

    • 6. is_move が偽の場合:

  • dav_fs_copymove_file を呼び出してコピーを実行する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/fs/repos.c 内の関数（行 586, 592 付近）。
  • 呼び出し先: apr_stat, apr_dir_make, apr_pstrcat, apr_file_rename, dav_fs_copymove_file, dav_new_error。

関連データ構造

  • dav_error: エラー報告用構造体。
  • apr_finfo_t: ファイル情報構造体。
  • dav_buffer: 内部バッファ。

失敗モードと境界条件

  • ソースの状態ファイルが存在しない場合、エラーを返さず正常終了する。
  • 宛先の状態ディレクトリ作成失敗時、既存ディレクトリでない場合はエラーを返す。
  • 宛先の状態ディレクトリパスに既にファイルが存在する場合、エラーを返す。
  • 移動操作で apr_file_rename が失敗した場合、エラーを返す。

保守担当者向けメモ

  • ### で示されたコメントにある通り、エラー発生時の HTTP ステータスコード（現状 500 固定）の妥当性や、ディレクトリ作成時の umask の扱いについては検討の余地がある。
  • dav_fs_copymove_file への依存が強く、ファイルシステム間の移動における一貫性に注意が必要。
