ap_varbuf

定義

  • 種別: struct
  • 定義箇所: include/util_varbuf.h:47

目的

Apache HTTP Server における可変長バッファ（Resizable Buffer）を管理するための主要なデータ構造です。メモリ管理に APR プールを利用し、文字列やバイナリデータの動的な拡張と再利用を効率的に行います。

シグネチャまたは形状

struct ap_varbuf {
    char *buf;
    apr_size_t avail;
    apr_size_t strlen;
    apr_pool_t *pool;
    struct ap_varbuf_info *info;
};

入力と出力

  • **入力**:
  • pool: メモリ割り当ておよびクリーンアップ管理に使用する apr_pool_t。
  • buf: 保持するデータ本体。
  • avail: 確保済みサイズ。
  • strlen: 現在のデータ長（または AP_VARBUF_UNKNOWN）。
  • **出力**:
  • buf を通じたデータアクセス。
  • strlen を通じた有効データ長の追跡。

前提条件と不変条件

  • avail が 0 の場合、buf は定数の \0 を指す必要があります。
  • avail が 0 でない場合、buf は pool と同じ生存期間を持つメモリ領域を指します。
  • avail の変更は必ず ap_varbuf_grow() を介して行う必要があります。
  • strlen は ap_varbuf_strmemcat() 等の操作の基準となります。

アルゴリズムまたは振る舞い

1. **初期化**: ap_varbuf_init() によりプールと構造体を関連付けます。 2. **拡張**: ap_varbuf_grow() が呼び出されると、必要に応じてメモリを指数関数的に（通常2倍）再割り当てし、既存データをコピーします。 3. **操作**: ap_varbuf_strmemcat() 等の関数が buf にデータを書き込み、strlen を更新します。 4. **解放**: 関連付けられたプールがクリアされる際に自動的にメモリが解放されます。明示的な解放には ap_varbuf_free() を使用します。

呼び出し元と呼び出し先

  • **呼び出し元**: include/util_varbuf.h 内の API（ap_varbuf_init, ap_varbuf_grow, ap_varbuf_free など）がこの構造体を操作します。
  • **呼び出し先**: 内部で apr_allocator を介したメモリ割り当てが行われます。
  • **注記**: 具体的な関数レベルの呼び出し元・呼び出し先情報は、本ソース excerpt には含まれていません。

関連データ構造

  • apr_pool_t: メモリ管理の基盤。
  • ap_varbuf_info: メモリ割り当てに関する不透明な情報。

失敗モードと境界条件

  • ap_varbuf_grow() はメモリ不足時にプールのアボート関数を呼び出すため、呼び出し側で個別のエラーチェックは不要です。
  • AP_VARBUF_UNKNOWN が設定されている場合、関数は内部で strlen を再計算するため、頻繁な呼び出しにはパフォーマンス上の注意が必要です。

保守担当者向けメモ

  • 本構造体はスタック上に配置可能ですが、内部で参照する pool が生存している必要があります。
  • avail の直接変更は禁止されています。必ず ap_varbuf_grow() を使用してください。
