eval_errf

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:77

目的

sed 実行エンジン内で発生したエラーを、設定されたエラーハンドラを通じて通知する。

シグネチャまたは形状

static void eval_errf(sed_eval_t *eval, const char *fmt, ...)

入力と出力

  • eval: 評価状態を保持する sed_eval_t 構造体へのポインタ。
  • fmt: エラーメッセージのフォーマット文字列（printf 形式）。
  • ...: フォーマット文字列に対応する可変長引数。
  • 出力: なし（戻り値は void）。

前提条件と不変条件

  • eval が有効なポインタであること。
  • eval->errfn が設定されている場合のみ、エラー通知処理が実行される。
  • eval->pool が有効な apr_pool_t であること（apr_pvsprintf によるメモリ確保のため）。

アルゴリズムまたは振る舞い

1. eval->errfn および eval->pool が存在するかを確認する。 2. apr_pvsprintf を使用し、fmt と可変長引数からエラーメッセージ文字列を生成する。 3. 生成された文字列を eval->errfn に渡して呼び出す。この際、eval->data がハンドラに渡される。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/filters/sed1.c:356
  • modules/filters/sed1.c:417
  • modules/filters/sed1.c:781
  • modules/filters/sed1.c:997
  • 呼び出し先:
  • apr_pvsprintf (APRライブラリ)
  • eval->errfn (関数ポインタ)

関連データ構造

  • sed_eval_t: 評価コンテキストを保持する構造体。エラーハンドラ関数ポインタ (errfn)、データポインタ (data)、メモリプール (pool) を含む。

失敗モードと境界条件

  • eval->errfn が NULL の場合、エラー通知は行われず、関数は何もせず終了する。
  • メモリ不足により apr_pvsprintf が失敗した場合の挙動は、APRのメモリ管理ポリシーに依存する。

保守担当者向けメモ

  • 本関数は libsed のエラー報告の抽象化レイヤーである。
  • eval->errfn に渡される eval->data は、通常 mod_sed 等の呼び出し側モジュールが提供するコンテキストである。
