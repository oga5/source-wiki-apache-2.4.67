os_forktype

定義

  • 種別: function
  • 定義箇所: os/bs2000/os.c:48

目的

現在の実行環境（デバッグモードの有無および実行ユーザーの権限）に基づき、BS2000環境で最適なプロセス生成手法（bs2_ForkType）を判定し、結果をキャッシュして返します。

シグネチャまたは形状

static bs2_ForkType os_forktype(int one_process)

入力と出力

  • **入力**: int one_process（デバッグモード等で単一プロセス実行を強制する場合は非ゼロ）
  • **出力**: bs2_ForkType 型の列挙値（bs2_noFORK, bs2_FORK, bs2_UFORK のいずれか）

前提条件と不変条件

  • 静的変数 forktype が bs2_unknown で初期化されていることを前提とします。
  • 一度判定が完了すると、OSのリリースや環境は変化しないものとして、以降の呼び出しではキャッシュされた値を返します。

アルゴリズムまたは振る舞い

1. 静的変数 forktype が未初期化（bs2_unknown）か確認します。 2. 初期化済みであれば、キャッシュされた値を即座に返します。 3. 未初期化の場合、以下の優先順位で判定を行います：

  • one_process が真の場合: bs2_noFORK を設定。
  • 実行ユーザーが非特権（getuid() != 0）の場合: bs2_FORK を設定。
  • 上記以外（特権ユーザーの場合）: bs2_UFORK を設定。

    • 4. 決定した値を forktype に格納し、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • os/bs2000/os.c 内のプロセス生成処理
  • os/unix/unixd.c 内のプロセス管理処理
  • **呼び出し先**:
  • getuid()（システムコール）

関連データ構造

  • bs2_ForkType: プロセス生成手法を定義する列挙型。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • getuid() の結果に依存するため、実行環境の権限設定が正しくない場合、意図しない bs2_ForkType が選択される可能性があります。

保守担当者向けメモ

  • 本関数は _OSD_POSIX マクロが定義された環境でのみ有効です。
  • forktype は静的変数として保持されるため、プロセス起動後の実行権限変更には追従しません。
