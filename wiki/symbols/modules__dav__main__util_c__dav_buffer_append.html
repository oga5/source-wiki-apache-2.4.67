dav_buffer_append

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:152

目的

dav_buffer 構造体で管理されるバッファの末尾に、指定された文字列を追記します。

シグネチャまたは形状

void dav_buffer_append(apr_pool_t *p, dav_buffer *pbuf, const char *str)

入力と出力

  • p: メモリ確保に使用する apr_pool_t。
  • pbuf: 追記対象の dav_buffer 構造体へのポインタ。
  • str: バッファに追加するヌル終端文字列。
  • 出力: なし（pbuf が指す構造体の内容が更新されます）。

前提条件と不変条件

  • pbuf は初期化済みである必要があります。
  • pbuf->buf は dav_check_bufsize によって必要に応じて拡張されます。
  • 追記後、pbuf->cur_len は元の長さと追加された文字列の長さの合計値となります。

アルゴリズムまたは振る舞い

1. strlen(str) で追加する文字列の長さを取得します。 2. dav_check_bufsize を呼び出し、文字列とヌル終端文字を格納するのに十分な容量を確保します。 3. memcpy を使用して、現在のバッファ末尾（pbuf->buf + pbuf->cur_len）に文字列とヌル終端文字をコピーします。 4. pbuf->cur_len を追加した文字列の長さ分だけインクリメントします。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/dav/fs/lock.c:943
  • modules/dav/main/util_lock.c:87, 95, 98, 104, 107, 110, 116, 119, 127, 130
  • 呼び出し先:
  • dav_check_bufsize (同ファイル内)
  • memcpy (標準ライブラリ)
  • strlen (標準ライブラリ)

関連データ構造

  • dav_buffer: バッファ本体 (buf)、現在の長さ (cur_len)、確保済みサイズ (alloc_len) を保持する構造体。

失敗モードと境界条件

  • apr_pool_t のメモリ不足が発生した場合、dav_check_bufsize 経由で Apache のメモリ割り当てエラーが発生する可能性があります。
  • str が NULL の場合、strlen の呼び出しによりセグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数はヌル終端を維持するように設計されています。
  • メモリ管理は apr_pool_t に依存しているため、呼び出し側で明示的な解放は不要です。
  • dav_buffer_place 等の他のバッファ操作関数と一貫した dav_buffer 構造体の更新ルールに従っています。
