ap_socache_instance_t

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_dbm.c:44 (および ap_socache.h:49)

目的

共有オブジェクトキャッシュ（socache）の各プロバイダー実装において、キャッシュインスタンスのコンテキストを保持するために使用されます。

シグネチャまたは形状

struct ap_socache_instance_t {
    const char *data_file;
    apr_pool_t *pool;
    apr_time_t last_expiry;
    apr_interval_time_t expiry_interval;
};

入力と出力

  • data_file: キャッシュデータの保存先ファイルパス（入力）。
  • pool: インスタンスに関連付けられたメモリプール（入力）。
  • last_expiry: 最後に期限切れ処理を実行した時刻（出力/状態）。
  • expiry_interval: 期限切れ処理の間隔（入力）。

前提条件と不変条件

  • pool メンバは、対応する mutex が保持されている間のみ使用可能です。
  • 本構造体を利用する mod_socache_dbm はスレッドセーフではないため、呼び出し側での排他制御が前提となります。

アルゴリズムまたは振る舞い

キャッシュインスタンスのライフサイクル管理、データパスの保持、および期限切れ処理のタイミング制御を行うためのコンテキストとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cache_socache.c や mod_authn_socache.c などの上位モジュール、および ap_socache.h で定義されたプロバイダーインターフェース（create, init, store, retrieve 等）。
  • **呼び出し先**: socache_dbm_expire などのプロバイダー固有の関数が、本構造体を引数として受け取ります。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_time_t / apr_interval_time_t: 期限切れ管理に使用。
  • ap_socache_provider_t (関連ヘッダーで定義): 本構造体を利用するプロバイダーのインターフェース定義。

失敗モードと境界条件

  • メモリプールが mutex なしでアクセスされた場合、競合による未定義動作が発生する可能性があります。
  • DBM 実装では、ファイルシステム上の制限により約 1KB 程度のデータサイズ制限を受ける場合があります。

保守担当者向けメモ

  • 本構造体は mod_socache_dbm.c だけでなく ap_socache.h にも定義が存在します。変更時は両方の整合性に注意してください。
  • pool の使用制限については、ソースコード内のコメント（Pool must only be used with the mutex held.）を厳守してください。
