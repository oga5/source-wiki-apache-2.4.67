long_res_header_for_sc

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_header.c:38

目的

AJP プロトコルにおけるレスポンスヘッダーの数値 ID を、対応する HTTP ヘッダー文字列名に変換します。

シグネチャまたは形状

static const char *long_res_header_for_sc(int sc)

入力と出力

  • **入力**: int sc (AJP レスポンスヘッダー ID)
  • **出力**: const char * (変換後のヘッダー名、または該当しない場合は NULL)

前提条件と不変条件

  • 入力値 sc は、ビットマスク 0x00FF によって下位 8 ビットのみが評価に使用されます。
  • 変換テーブル response_trans_headers がモジュール内で定義されている必要があります。

アルゴリズムまたは振る舞い

1. 入力された sc に対して sc & 0x00FF を実行し、下位 8 ビットを抽出します。 2. 抽出した値が 0 より大きく、かつ SC_RES_HEADERS_NUM 以下であるかを確認します。 3. 条件を満たす場合、response_trans_headers 配列の sc - 1 番目の要素を返します。 4. 条件を満たさない場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/ajp_header.c:615 (コード内の直接的な参照を確認)
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • response_trans_headers: ヘッダー ID と文字列名のマッピングを保持する静的配列（本ファイル内で定義）。

失敗モードと境界条件

  • sc が 0 または SC_RES_HEADERS_NUM を超える値の場合、範囲外アクセスを回避して NULL を返します。

保守担当者向けメモ

  • 本関数は static であり、ajp_header.c 内でのみ使用されます。
  • 新しい AJP レスポンスヘッダーを追加する場合は、response_trans_headers 配列の更新と SC_RES_HEADERS_NUM 定数の整合性に注意してください。
