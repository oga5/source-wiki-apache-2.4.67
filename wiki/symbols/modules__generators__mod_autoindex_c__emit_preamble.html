emit_preamble

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_autoindex.c:169

目的

ディレクトリ索引ページの HTML ドキュメントの開始部分（DOCTYPE宣言、<head>、<title>、CSSリンク、および <body> タグの開始）を生成し、クライアントへ出力する。

シグネチャまたは形状

static void emit_preamble(request_rec *r, int xhtml, const char *title)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • xhtml: XHTML モードで出力するかどうかを示すフラグ（真であれば XHTML 1.0 Transitional、偽であれば HTML 4.01）。
  • title: ページタイトルとして表示する文字列。
  • **出力**: なし（ap_rvputs および ap_rputs を通じてリクエストの出力バッファに書き込まれる）。

前提条件と不変条件

  • r->per_dir_config から autoindex_module の設定構造体を取得可能であること。
  • title は有効な文字列であること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリ設定（autoindex_config_rec）を取得する。 2. xhtml フラグに基づき、適切な DOCTYPE 宣言と <html>, <head>, <title> タグを出力する。 3. 設定構造体 d に style_sheet が定義されている場合、<link> タグを出力する（XHTML モード時は自己終了タグを使用）。 4. 設定構造体 d に head_insert が定義されている場合、その内容をそのまま出力する。 5. </head> および <body> タグを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_autoindex.c:1044
  • modules/generators/mod_autoindex.c:1082
  • modules/generators/mod_autoindex.c:1104
  • **呼び出し先**:
  • ap_get_module_config (Apache API)
  • ap_rvputs (Apache API)
  • ap_rputs (Apache API)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • autoindex_config_rec: ディレクトリごとの設定（スタイルシートやヘッダー挿入文字列など）を保持する構造体。

失敗モードと境界条件

  • 設定構造体 d 内の style_sheet や head_insert が NULL の場合、それらの出力処理はスキップされる。
  • ap_rvputs 等の出力関数が失敗した場合の戻り値は確認していない。

保守担当者向けメモ

  • 本関数は mod_autoindex の出力生成フェーズの初期段階を担う。
  • xhtml フラグの切り替えにより、HTML 4.01 と XHTML 1.0 Transitional の両方に対応している。
  • head_insert は設定ファイルで指定された任意の HTML を挿入するため、セキュリティ上の観点から内容の妥当性には注意が必要。
