proxy_detect

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:791

目的

フォワードプロキシリクエストを検出し、リクエスト処理パイプラインにおいてプロキシとして扱うべきかを判定する。

シグネチャまたは形状

static int proxy_detect(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (常に DECLINED を返す)

前提条件と不変条件

  • proxy_server_conf がサーバー設定から取得可能であること。
  • conf->req が有効（プロキシ機能が有効）であること。

アルゴリズムまたは振る舞い

1. サーバー設定からプロキシ設定を取得する。 2. **通常プロキシ**: r->parsed_uri.scheme が存在する場合、それが現在のHTTPスキームと異なり、かつ現在の仮想ホストと一致しない場合にプロキシリクエストと判定する。 3. **CONNECTメソッド**: M_CONNECT メソッドかつホスト名とポートが指定されている場合にプロキシリクエストと判定する。 4. プロキシリクエストと判定された場合、以下の変更を行う:

  • r->proxyreq を PROXYREQ_PROXY に設定。
  • r->uri を r->unparsed_uri で更新。
  • r->filename を "proxy:" を接頭辞として付与した文字列に書き換え。
  • r->handler を "proxy-server" に設定。

    • 5. 最終的に DECLINED を返し、後続のフックへ処理を継続する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_read_request に登録され、Apache のリクエスト処理の初期段階で呼び出される。
  • **呼び出し先**: ap_get_module_config, ap_http_scheme, ap_matches_request_vhost, ap_default_port, apr_pstrcat 等の Apache/APR API を利用。

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • proxy_server_conf: サーバー単位のプロキシ設定構造体。

失敗モードと境界条件

  • r->parsed_uri が不完全な場合、プロキシ判定はスキップされる。
  • conf->req が無効な場合、本関数は何も行わず DECLINED を返す。

保守担当者向けメモ

  • 本関数は post_read_request フックとして APR_HOOK_FIRST で登録されており、リクエスト処理の極めて早い段階で実行される。
  • r->filename の書き換えは、後続の翻訳フェーズに影響を与えるため注意が必要。
