uldap_simple_bind

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:551

目的

LDAP サーバーに対して単純バインド（Simple Bind）操作を実行し、認証を試みる。

シグネチャまたは形状

static int uldap_simple_bind(util_ldap_connection_t *ldc, char *binddn,
                             char* bindpw, struct timeval *timeout)

入力と出力

  • **入力**:
  • ldc: LDAP 接続情報を保持する util_ldap_connection_t 構造体。
  • binddn: バインドに使用する識別名（DN）。
  • bindpw: バインドに使用するパスワード。
  • timeout: 操作のタイムアウト時間を指定する timeval 構造体。
  • **出力**:
  • 成功時は LDAP_SUCCESS (0)、失敗時は LDAP エラーコードを返す。

前提条件と不変条件

  • ldc->ldap が有効な LDAP ハンドルとして初期化されていること。
  • ldc 構造体が適切にロックされている環境で呼び出されること（スレッドセーフな接続管理のため）。

アルゴリズムまたは振る舞い

1. ldap_simple_bind を呼び出し、非同期バインドリクエストを開始する。 2. リクエストが即座に失敗（msgid == -1）した場合は、uldap_ld_errno を経由してエラーを返す。 3. ldap_result を使用して、指定された timeout 内でバインド結果の取得を待機する。 4. 結果取得が失敗した場合はエラーを返し、タイムアウトした場合は LDAP_TIMEOUT を設定する。 5. ldap_parse_result を呼び出し、バインド操作の最終的なステータスコードを抽出する。 6. 成功時は ldc->last_backend_conn を更新し、トレースログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap.c:641
  • modules/ldap/util_ldap.c:1863
  • **呼び出し先**:
  • ldap_simple_bind (LDAP SDK)
  • ldap_result (LDAP SDK)
  • ldap_parse_result (LDAP SDK)
  • uldap_ld_errno (内部関数)
  • ap_log_rerror (Apache API)

関連データ構造

  • util_ldap_connection_t: LDAP 接続状態とリクエスト情報を保持する構造体。

失敗モードと境界条件

  • ldap_simple_bind が即座に失敗した場合、ldc->reason にエラー理由が設定される。
  • ldap_result がタイムアウトした場合、rc は LDAP_TIMEOUT となる。
  • ldap_parse_result が失敗した場合、解析エラーとして uldap_ld_errno の結果が返される。

保守担当者向けメモ

  • 本関数は ldc->reason を更新することで、呼び出し元がエラーの詳細を把握できるようにしている。
  • ldap_parse_result の第 9 引数に 1 を指定しており、内部で LDAPMessage の解放が行われる仕様である。
