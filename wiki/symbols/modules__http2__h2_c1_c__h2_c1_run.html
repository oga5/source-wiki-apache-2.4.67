h2_c1_run

定義

  • **種別**: function
  • **定義箇所**: modules/http2/h2_c1.c:121

目的

HTTP/2 セッションのメインループを実行し、MPM（Multi-Processing Module）の状態に応じて接続の読み取り待ちや書き込み完了状態への遷移を管理する。

シグネチャまたは形状

int h2_c1_run(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（Apache の接続レコード）
  • **出力**: int（常に OK を返す）

前提条件と不変条件

  • h2_conn_ctx_get(c) が有効な h2_conn_ctx_t を返すこと。
  • conn_ctx->session が初期化済みであること。

アルゴリズムまたは振る舞い

1. **セッション処理**: h2_session_process を呼び出し、HTTP/2 フレームを処理する。 2. **ループ制御**: 非同期 MPM (async_mpm) でない場合、接続が維持されている間かつ MPM が停止中でない限り、セッション処理を繰り返す。 3. **状態遷移**: セッションの状態（H2_SESSION_ST_*）に基づき、MPM の接続状態 (c->cs->state) を更新する。

  • **アクティブ時**: CONN_STATE_WRITE_COMPLETION へ遷移。
  • **待機時**: CONN_STATE_ASYNC_WAITIO（サポート時）または CONN_STATE_WRITE_COMPLETION（互換用）へ遷移し、読み取り待ち (CONN_SENSE_WANT_READ) を設定。
  • **終了時**: CONN_STATE_LINGER へ遷移。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c:309
  • modules/http2/h2_switch.c:209
  • **呼び出し先**:
  • h2_session_process
  • ap_mpm_query
  • ap_log_cerror

関連データ構造

  • conn_rec: Apache の接続状態を保持。
  • h2_conn_ctx_t: HTTP/2 接続固有のコンテキスト。
  • h2_session: HTTP/2 プロトコルロジックの管理オブジェクト。

失敗モードと境界条件

  • h2_session_process が APR_SUCCESS 以外を返した場合、接続は AP_CONN_CLOSE としてマークされる。
  • ap_mpm_query が失敗した場合、ループを即座に終了する。
  • AP_MPMQ_CAN_WAITIO が定義されていない環境では、互換性のために c->clogging_input_filters を使用してフックの再入を強制する。

保守担当者向けメモ

  • CONN_STATE_ASYNC_WAITIO への遷移は PR 63534 に関連する複雑な互換性処理を含んでおり、MPM の動作に強く依存する。
  • c->clogging_input_filters の操作は、非同期 MPM 以外の環境で読み取り待ちを正しく機能させるための回避策である。
