imap_url

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_imagemap.c:319

目的

画像マップファイル内の定義値（value）を、指定されたベースURL（base）と組み合わせて絶対URLへ変換する。

シグネチャまたは形状

static const char *imap_url(request_rec *r, const char *base, const char *value)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。メモリ割り当て（r->pool）に使用。
  • base: URL構築の基準となるパス。
  • value: マップファイルから読み取られたターゲット値。
  • **出力**:
  • 変換後の絶対URL文字列、またはエラー発生時の NULL。

前提条件と不変条件

  • r->pool を使用してメモリを確保するため、呼び出し側での明示的な解放は不要。
  • value が map または menu の場合、現在のリクエストURIを返す。
  • value が referer の場合、リクエストヘッダの Referer を参照する。
  • value が絶対URL（http: 等）である場合はそのまま返す。

アルゴリズムまたは振る舞い

1. **特殊キーワード判定**: map, menu, nocontent, error, referer を個別に処理。 2. **絶対URL判定**: value に : が含まれる場合は絶対URLとみなしてそのまま返す。 3. **ベースURLの結合**:

  • base が提供されない場合は value をそのまま返すか、ルートパスを補完する。
  • base と value が相対パス関係にある場合、base のディレクトリ構造を解析し、../ の解決（パスの遡り）を行う。

    • 4. **文字列結合**: apr_pstrcat を用いて my_base と value を連結し、正規化されたURLを生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_imagemap.c 内の imap_handler_internal（行612, 617, 700, 710, 715, 756, 776, 789, 802, 831, 840）。
  • **呼び出し先**: ap_construct_url, apr_pstrdup, apr_table_get, apr_isalpha, ap_strchr_c, ap_log_rerror, strrchr, strlen, strncmp, strcmp, apr_pstrcat。

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • **エラーログ**: base が無効な場合や、ディレクトリの遡りに失敗した場合は APLOG_ERR を記録し NULL を返す。
  • **メモリ制約**: r->pool の枯渇時は apr 関数群の挙動に依存する。
  • **入力の不整合**: value が referer であり、かつリクエストに Referer ヘッダが存在しない場合は空文字列として扱う。

保守担当者向けメモ

  • value の解析において、../ を処理するループは directory 変数を用いてパスの階層を正しく遡るよう設計されている。
  • base が指定されている場合、base の末尾のディレクトリ構造を破壊的に変更（\0 で終端）して結合を行うため、my_base としてコピーを作成してから処理している。
