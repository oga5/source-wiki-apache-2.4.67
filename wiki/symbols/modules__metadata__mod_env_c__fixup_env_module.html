fixup_env_module

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_env.c:160

目的

リクエスト処理の fixups フェーズにおいて、設定された環境変数を request_rec->subprocess_env テーブルに注入する。

シグネチャまたは形状

static int fixup_env_module(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: int （Apache のステータスコード。成功時は OK、設定がない場合は DECLINED を返す）

前提条件と不変条件

  • r->per_dir_config から env_module の設定構造体 env_dir_config_rec が取得可能であること。
  • sconf->vars は apr_table_t 型であり、設定された環境変数を保持していること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリ設定（sconf）を取得する。 2. apr_is_empty_table(sconf->vars) を確認し、設定すべき環境変数が存在しない場合は DECLINED を返して終了する。 3. 設定が存在する場合、apr_table_overlay を用いて、既存の r->subprocess_env に sconf->vars をマージする。 4. マージ後の新しいテーブルを r->subprocess_env に割り当て、OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の fixups フック（modules/metadata/mod_env.c:177 にて ap_hook_fixups に登録）。
  • **呼び出し先**:
  • ap_get_module_config
  • apr_is_empty_table
  • apr_table_overlay

関連データ構造

  • env_dir_config_rec: ディレクトリごとの環境変数設定を保持する構造体。
  • request_rec: リクエスト処理のコンテキスト。subprocess_env テーブルを保持する。

失敗モードと境界条件

  • sconf->vars が空の場合、処理は行われず DECLINED が返される。
  • メモリ割り当ての失敗については、apr_table_overlay の内部挙動に依存する（通常は APR プールによる管理）。

保守担当者向けメモ

  • 本関数は fixups フックの一部として実行されるため、他のモジュールによる環境変数の変更と競合しないよう、フックの実行順序に注意が必要である。
  • apr_table_overlay は新しいテーブルを作成するため、リクエストのライフサイクル全体でメモリ消費量が増大しないよう、r->pool の管理に依存している。
