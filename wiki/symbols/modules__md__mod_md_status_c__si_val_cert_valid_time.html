si_val_cert_valid_time

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:342

目的

管理対象ドメインの証明書情報から有効期限（valid time）を取得し、ステータスレポートへ出力するために整形・表示する。

シグネチャまたは形状

static void si_val_cert_valid_time(status_ctx *ctx, md_json_t *mdj, const status_info *info)

入力と出力

  • ctx: ステータス出力用のコンテキスト（出力先やフォーマットフラグを保持）。
  • mdj: 証明書情報を含む JSON オブジェクト。
  • info: 現在のステータス表示カラムの定義情報。
  • 出力: なし（ctx を介して出力バッファへ書き込まれる）。

前提条件と不変条件

  • mdj は有効な md_json_t オブジェクトであること。
  • info は status_info 構造体への有効なポインタであること。

アルゴリズムまたは振る舞い

1. info をコピーしてローカル変数 sub を作成する。 2. sub.key を MD_KEY_VALID に更新し、証明書の有効期限を示すキーを指定する。 3. md_json_getj を使用し、mdj から info->key に対応する証明書オブジェクトを取得する。 4. 証明書オブジェクトが存在する場合、si_val_valid_time を呼び出し、更新された sub を用いて有効期限の表示処理を行う。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md_status.c:706（status_info 配列の定義内）。
  • 呼び出し先: md_json_getj（JSON 取得）、si_val_valid_time（表示処理）。

関連データ構造

  • status_ctx: ステータス出力の制御コンテキスト。
  • md_json_t: 証明書データを含む JSON 構造体。
  • status_info: ステータス表示カラムの定義（ラベル、キー、コールバック関数）。

失敗モードと境界条件

  • md_json_getj が NULL を返した場合（指定されたキーの証明書情報が存在しない場合）、何も出力せず正常終了する。

保守担当者向けメモ

  • 本関数は status_info のキーを一時的に書き換えて si_val_valid_time に委譲するラッパーである。
  • 証明書情報の構造変更に伴い MD_KEY_VALID の定義が変更された場合は注意が必要。
