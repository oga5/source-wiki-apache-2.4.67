ap_mpm_set_coredumpdir

定義

  • 種別: function
  • 定義箇所: server/mpm_common.c:341

目的

Apache HTTP Server の設定ディレクティブ CoreDumpDirectory を処理し、コアダンプを出力するディレクトリパスを検証および設定する。

シグネチャまたは形状

const char *ap_mpm_set_coredumpdir(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • arg: 設定ファイルから渡されるディレクトリパスの文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • ap_check_cmd_context(cmd, GLOBAL_ONLY) により、本ディレクティブはグローバルコンテキストでのみ使用可能であること。
  • 指定されたパスは、サーバーのルートディレクトリからの相対パスとして解決される。
  • 指定されたパスは、ファイルシステム上に存在し、かつディレクトリである必要がある。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、グローバル設定以外での使用を拒否する。 2. ap_server_root_relative を使用して、引数 arg を絶対パスに変換する。変換に失敗した場合はエラーを返す。 3. apr_stat を使用して、パスの存在確認とファイルタイプ（ディレクトリであるか）の検証を行う。 4. 検証に合格した場合、ap_coredump_dir グローバル変数にパスをコピーし、ap_coredumpdir_configured フラグを 1 に設定する。 5. 正常終了時は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c 内の AP_INIT_TAKE1 マクロ定義により、設定ファイル内の CoreDumpDirectory ディレクティブから呼び出される。
  • **呼び出し先**:
  • ap_check_cmd_context
  • ap_server_root_relative
  • apr_stat
  • apr_pstrcat
  • apr_cpystrn

関連データ構造

  • cmd_parms: 設定コマンドのコンテキスト情報。
  • apr_finfo_t: ファイル情報を保持する構造体。
  • ap_coredump_dir: コアダンプ先パスを保持するグローバル変数。
  • ap_coredumpdir_configured: 設定済み状態を示すフラグ。

失敗モードと境界条件

  • **コンテキストエラー**: グローバル設定以外で指定された場合、エラーメッセージを返す。
  • **パス解決エラー**: ap_server_root_relative がパスを解決できない場合、エラーを返す。
  • **存在確認エラー**: 指定パスが存在しない場合、エラーを返す。
  • **型エラー**: 指定パスがディレクトリではない場合、エラーを返す。

保守担当者向けメモ

  • 本関数は server/mpm_common.c で定義されているが、server/core.c から参照されている。
  • ap_coredump_dir への書き込みは apr_cpystrn を使用しており、バッファオーバーフローを防止している。
