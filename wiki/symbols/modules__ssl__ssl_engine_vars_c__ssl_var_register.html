ssl_var_register

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:136

目的

mod_ssl モジュールの初期化時に、SSL 関連の変数解決機能および式評価（ap_expr）用フックを登録し、ライブラリのバージョン情報を初期化する。

シグネチャまたは形状

void ssl_var_register(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p - メモリ割り当てに使用するプール。
  • **出力**: なし（副作用としてグローバルなフック登録およびライブラリ情報の初期化を行う）。

前提条件と不変条件

  • モジュール初期化フェーズで呼び出されることを想定している。
  • var_library および var_library_interface が静的変数として定義されている必要がある。

アルゴリズムまたは振る舞い

1. **フック登録**:

  • ap_hook_ssl_conn_is_ssl を登録し、接続が SSL かどうかを判定する関数を有効化する。
  • ap_hook_expr_lookup を登録し、ap_expr 評価時に SSL 変数を解決できるようにする。

    • 2. **オプション関数登録**:

  • ssl_var_lookup および ssl_ext_list を APR_REGISTER_OPTIONAL_FN を介して公開する。

    • 3. **バージョン文字列の整形**:

  • MODSSL_LIBRARY_DYNTEXT からライブラリバージョン文字列を抽出し、スペースをスラッシュに置換して var_library に格納する。
  • 同様に var_library_interface の文字列を整形する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c:749
  • **呼び出し先**:
  • ap_hook_ssl_conn_is_ssl
  • APR_REGISTER_OPTIONAL_FN (マクロ)
  • ap_hook_expr_lookup
  • apr_pstrdup
  • strchr

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • var_library (静的変数): ライブラリバージョン情報を保持。

失敗モードと境界条件

  • strchr がスペースを見つけられない場合、文字列の置換処理はスキップされる。
  • メモリ不足時には apr_pstrdup がアボートする可能性がある（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数はモジュールの初期化プロセスの一部であるため、変更時は mod_ssl.c の初期化順序に注意すること。
  • var_library の文字列操作は、MODSSL_LIBRARY_DYNTEXT のフォーマットに依存している。OpenSSL のバージョン表記が変更された場合、このパースロジックが影響を受ける可能性がある。
