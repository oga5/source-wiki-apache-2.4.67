ap_proxy_canon_netloc

定義

  • 種別: function
  • 定義箇所: modules/proxy/proxy_util.c:341

目的

プロキシ対象の URL からネットワークロケーション（ホスト名、ポート番号、認証情報）を抽出し、正規化を行う。

シグネチャまたは形状

char *ap_proxy_canon_netloc(apr_pool_t *p, char **const urlp, char **userp,
                            char **passwordp, char **hostp, apr_port_t *port);

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • urlp: 解析対象の URL 文字列へのポインタ。処理後、パス部分のみに更新される。
  • userp, passwordp: 認証情報を格納するポインタ（NULL 可）。
  • hostp: ホスト名を格納するポインタ。
  • port: ポート番号を格納するポインタ。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ（char *）を返す。

前提条件と不変条件

  • urlp が指す文字列は // で始まる必要がある。
  • ホスト名に含まれる DNS 名は小文字に変換される。
  • 認証情報（ユーザー名・パスワード）が含まれる場合、ap_proxy_canonenc を用いてエンコードされる。

アルゴリズムまたは振る舞い

1. URL が // で始まらない場合、"Malformed URL" を返す。 2. ホスト部とパス部を分離する。 3. ホスト部から @ を検索し、認証情報（ユーザー名・パスワード）を抽出・正規化する。 4. apr_parse_addr_port を使用してホストとポートを分離する。 5. 解析結果が不正（addr が NULL、または scope_id が存在）な場合は "Invalid host/port" を返す。 6. ホスト名を小文字化し、urlp をパス部分へ、hostp をホスト名へ更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy_http.c, mod_proxy_ajp.c, mod_proxy_balancer.c, mod_proxy_fcgi.c, mod_proxy_ftp.c, mod_proxy_scgi.c, mod_proxy_uwsgi.c, mod_proxy_wstunnel.c, mod_proxy_http2.c 等の各プロトコルハンドラ。
  • **呼び出し先**: ap_proxy_canonenc, apr_parse_addr_port, ap_str_tolower。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_port_t: ポート番号の保持に使用。

失敗モードと境界条件

  • URL が // で始まらない: "Malformed URL"
  • パスワードのエンコード失敗: "Bad %-escape in URL (password)"
  • ユーザー名のエンコード失敗: "Bad %-escape in URL (username)"
  • ホスト/ポートの解析失敗: "Invalid host/port"

保守担当者向けメモ

  • 本関数は URL の解析と正規化を同時に行うため、呼び出し側は urlp が指すポインタが書き換えられる点に注意すること。
  • IPv6 アドレスのスコープ ID が含まれる場合はエラーとして扱われる仕様となっている。
