h2_slot_state_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_workers.c:54

目的

HTTP/2 ワーカープールにおける個々のスロット（h2_slot）の現在の実行状態を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    H2_SLOT_FREE,
    H2_SLOT_RUN,
    H2_SLOT_ZOMBIE,
} h2_slot_state_t;

入力と出力

  • 本シンボルは列挙型であり、入力や出力を持つ関数ではありません。
  • h2_slot 構造体のメンバ state として保持され、スロットの状態遷移を表現します。

前提条件と不変条件

  • h2_workers 構造体が管理するスロットの状態遷移は、workers->lock による排他制御下で行われる必要があります。

アルゴリズムまたは振る舞い

  • H2_SLOT_FREE: スロットがアイドル状態であり、新しいタスクの割り当てが可能な状態。
  • H2_SLOT_RUN: スロットが現在接続処理（slot_run）を実行中である状態。
  • H2_SLOT_ZOMBIE: スレッドが終了処理中、または終了待ちの状態。

呼び出し元と呼び出し先

  • 呼び出し元: h2_slot 構造体（modules/http2/h2_workers.c:65）で型として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • h2_slot: この列挙型を状態保持に使用する主要な構造体。
  • h2_workers: スロットを管理するプール構造体。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 状態遷移のロジックは h2_workers.c 内の各関数（activate_slot や slot_run 等）に分散しているため、状態を追加・変更する際は、これら関連関数の条件分岐をすべて確認してください。
