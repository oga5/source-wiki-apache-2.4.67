hb_watchdog_need

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartbeat.c:140

目的

mod_watchdog が提供するフックにおいて、本モジュールの処理を単一プロセス（singleton）で実行すべきかどうかを判定する。

シグネチャまたは形状

static int hb_watchdog_need(server_rec *s, const char *name, int parent, int singleton)

入力と出力

  • **入力**:
  • s: サーバー設定構造体。
  • name: ウォッチドッグの識別名。
  • parent: 親プロセスフラグ。
  • singleton: 単一プロセス実行フラグ。
  • **出力**:
  • OK: 条件を満たし、実行を許可する場合。
  • DECLINED: 条件を満たさない場合。

前提条件と不変条件

  • s->module_config から heartbeat_module の設定（hb_ctx_t）が取得可能であること。
  • ctx->active が真であり、かつ singleton が真である場合にのみ実行が許可される。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定から hb_ctx_t を取得する。 2. モジュールがアクティブ（ctx->active）であり、かつ singleton 引数が真であり、さらに name が AP_WATCHDOG_SINGLETON と一致するかを確認する。 3. すべての条件を満たす場合は OK を返し、それ以外の場合は DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_watchdog（ap_hook_watchdog_need を介して登録）。
  • **呼び出し先**: 該当なし。

関連データ構造

  • hb_ctx_t: モジュールの設定状態を保持する構造体。
  • server_rec: Apache のサーバー設定構造体。

失敗モードと境界条件

  • ctx が NULL の場合、あるいは ctx->active が偽の場合は DECLINED を返す。
  • name が AP_WATCHDOG_SINGLETON 以外の場合は DECLINED を返す。

保守担当者向けメモ

  • 本関数は mod_watchdog のフックとして登録されており、mod_heartbeat がマルチプロセス環境下で重複して動作しないよう制御する役割を持つ。
