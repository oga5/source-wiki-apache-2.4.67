htdbm_list

定義

  • 種別: function
  • 定義箇所: support/htdbm.c:211

目的

DBM 形式のパスワードデータベース内の全レコードを走査し、ユーザー名とコメントを標準エラー出力（stderr）へ一覧表示する。

シグネチャまたは形状

apr_status_t htdbm_list(htdbm_t *htdbm)

入力と出力

  • **入力**: htdbm_t *htdbm（データベース接続情報およびファイル名を含む構造体）。
  • **出力**: apr_status_t 型のステータスコード。成功時は APR_SUCCESS を返す。

前提条件と不変条件

  • htdbm->dbm は apr_dbm_open によって既にオープンされている必要がある。
  • DBM 内のデータは username をキーとし、password[:comment] を値として格納されている。
  • DBM データには \0 終端が含まれないため、apr_datum_t の dsize を用いて長さを管理する。

アルゴリズムまたは振る舞い

1. apr_dbm_firstkey を呼び出し、データベースの最初のキーを取得する。失敗した場合は APR_ENOENT を返す。 2. データベース名とヘッダーを stderr に出力する。 3. key.dptr が NULL になるまでループし、以下の処理を行う：

  • apr_dbm_fetch でキーに対応する値を取得する。
  • ユーザー名（キー）を stderr に出力する。
  • 値の中に : が含まれる場合、それ以降をコメントとして抽出し出力する。
  • apr_dbm_nextkey で次のキーへ進む。

    • 4. 最後に総レコード数を stderr に出力し、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htdbm.c:444
  • **呼び出し先**:
  • apr_dbm_firstkey, apr_dbm_fetch, apr_dbm_nextkey (APR DBM API)
  • memchr (標準 C ライブラリ)
  • fprintf (標準 C ライブラリ)

関連データ構造

  • htdbm_t: データベース接続状態を保持するメイン構造体。
  • apr_datum_t: DBM のキーおよび値を保持する APR のデータ構造。

失敗モードと境界条件

  • **データベースが空**: apr_dbm_firstkey が失敗し、APR_ENOENT を返す。
  • **データ取得失敗**: apr_dbm_fetch が失敗した場合、APR_EGENERAL を返す。
  • **次キー取得失敗**: apr_dbm_nextkey が失敗した場合、エラーメッセージを stderr に出力するが、処理は継続される。

保守担当者向けメモ

  • 本関数は stderr に直接出力を行うため、ログ出力先を変更したい場合は修正が必要である。
  • val.dptr 内のコメント抽出において、memchr で : を検索するロジックは、値の中に : が含まれないケースを正しく処理する。
