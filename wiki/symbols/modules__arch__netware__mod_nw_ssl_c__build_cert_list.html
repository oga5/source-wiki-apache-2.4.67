build_cert_list

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:148

目的

NetWare の証明書ストア（KMO）から取得した証明書リストを、システムが利用可能な Unicode 形式の配列に変換・構築します。

シグネチャまたは形状

static void build_cert_list(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（メモリ割り当てに使用する Apache プール）
  • **出力**: なし（副作用としてグローバル変数 certarray および numcerts を更新）

前提条件と不変条件

  • グローバル変数 certlist が初期化済みであること。
  • certlist は apr_array_header_t 型の構造体として期待されており、その要素は文字列（char **）であること。

アルゴリズムまたは振る舞い

1. certlist から証明書の総数を取得し、numcerts に格納します。 2. apr_palloc を使用して、証明書ポインタを保持するための certarray をメモリ確保します。 3. certlist 内の各証明書文字列に対し、以下の処理を繰り返します。

  • apr_palloc で Unicode 文字列用のバッファを確保します（元の文字列長の4倍）。
  • loc2uni 関数を呼び出し、ローカルの証明書文字列を Unicode 形式に変換して certarray に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_nw_ssl.c:834
  • **呼び出し先**:
  • apr_palloc (APR)
  • loc2uni (NetWare Unicode ライブラリ)
  • strlen (標準 C ライブラリ)

関連データ構造

  • certlist: 証明書リストを保持するグローバルな配列構造体。
  • certarray: 変換後の Unicode 証明書文字列を保持するグローバル配列。
  • numcerts: 証明書の総数を保持するグローバル整数。

失敗モードと境界条件

  • certlist が空の場合、numcerts は 0 となり、ループは実行されません。
  • メモリ不足時には apr_palloc が Apache のメモリ管理ポリシーに従い例外を発生させる可能性があります。

保守担当者向けメモ

  • 本関数はグローバル変数に依存しているため、マルチスレッド環境での実行順序には注意が必要です。
  • loc2uni の第5引数 2 は変換フラグを示しており、NetWare の Unicode 変換仕様に依存しています。
