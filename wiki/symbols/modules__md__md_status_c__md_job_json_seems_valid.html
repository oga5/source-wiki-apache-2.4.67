md_job_json_seems_valid

定義

  • 種別: function
  • 定義箇所: modules/md/md_status.c:100

目的

指定された JSON オブジェクトが、Managed Domain (MD) のジョブ状態として妥当であるかを検証します。特に、ステージング領域にある完了済みのジョブに対して、必要な成果物（証明書ファイル）が存在するかを確認します。

シグネチャまたは形状

int md_job_json_seems_valid(md_json_t *json, md_store_t *store,
                            md_store_group_t group, const char *name,
                            apr_pool_t *p)

入力と出力

  • **入力**:
  • json: 検証対象の md_json_t オブジェクト。
  • store: ストレージハンドル。
  • group: ストレージグループ（例: MD_SG_STAGING）。
  • name: ドメイン名。
  • p: メモリ割り当て用プール。
  • **出力**:
  • TRUE (1): JSON が妥当、または検証不要な状態。
  • FALSE (0): JSON が無効、または必要な成果物が欠落している。

前提条件と不変条件

  • json が NULL の場合は常に FALSE を返します。
  • ステージンググループ (MD_SG_STAGING) かつジョブが完了 (MD_KEY_FINISHED) しており、通知済み (MD_KEY_NOTIFIED_RENEWED) である場合、厳密な整合性チェックが行われます。

アルゴリズムまたは振る舞い

1. json が NULL であれば即座に FALSE を返す。 2. グループが MD_SG_STAGING で、かつジョブが完了状態であることを確認する。 3. 完了状態であれば、以下の整合性を確認する：

  • md_exists を呼び出し、該当ドメインのディレクトリが存在することを確認する。
  • md_load を呼び出し、ドメイン設定を読み込む。
  • 読み込んだドメイン内の全証明書について、md_pubcert_load を呼び出し、公開鍵仕様に基づいた証明書ファイルがストレージ上に存在することを確認する。

    • 4. 上記のいずれかのチェックで失敗した場合は FALSE を返し、すべて通過した場合は TRUE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_status.c:133
  • modules/md/md_status.c:419
  • **呼び出し先**:
  • md_exists
  • md_load
  • md_cert_count
  • md_pkeys_spec_get
  • md_pubcert_load
  • md_json_getb

関連データ構造

  • md_json_t: JSON データの保持構造。
  • md_store_t: ストレージ抽象化層。
  • md_t: 管理対象ドメインの定義構造体。

失敗モードと境界条件

  • md_json_getb でキーが見つからない場合や、md_load が APR_SUCCESS を返さない場合は FALSE と判定されます。
  • ステージング領域以外のグループが指定された場合は、詳細なファイル存在チェックをスキップし TRUE を返します。

保守担当者向けメモ

  • 本関数は、ジョブのメタデータ（JSON）と実際のファイルシステム上の状態（証明書ファイル）の同期を確認する重要なガードレールです。
  • MD_SG_STAGING 以外のグループに対する検証ロジックは現在実装されておらず、常に TRUE を返す仕様となっています。
