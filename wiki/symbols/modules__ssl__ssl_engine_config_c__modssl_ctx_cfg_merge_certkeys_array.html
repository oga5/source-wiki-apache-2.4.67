modssl_ctx_cfg_merge_certkeys_array

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_config.c:336

目的

SSL 証明書および鍵ファイルの配列を、親設定（base）から子設定（add）へマージする。特に CERTKEYS_IDX_MAX までのインデックスについては、子設定の値が親設定の値を上書き（置換）する挙動をエミュレートする。

シグネチャまたは形状

static void modssl_ctx_cfg_merge_certkeys_array(apr_pool_t *p,
                                                apr_array_header_t *base,
                                                apr_array_header_t *add,
                                                apr_array_header_t *mrg)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t。
  • base: 継承元の証明書/鍵ファイル配列。
  • add: 継承先（子）の証明書/鍵ファイル配列。
  • mrg: マージ結果を格納する配列。

前提条件と不変条件

  • mrg 配列は呼び出し元で初期化済みであること。
  • CERTKEYS_IDX_MAX は ssl_private.h 等で定義された定数であり、置換対象となるインデックスの境界を示す。

アルゴリズムまたは振る舞い

1. **置換処理**: add 配列のインデックス 0 から CERTKEYS_IDX_MAX までの要素を mrg にコピーする。これにより、対応する base の値が上書きされる。 2. **ベースの補完**: add に含まれなかった残りの base 要素を mrg に追加する。 3. **追加要素の結合**: add 配列の CERTKEYS_IDX_MAX + 1 以降の要素を mrg の末尾に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_config.c:377 (証明書ファイルのマージ)
  • modules/ssl/ssl_engine_config.c:380 (鍵ファイルのマージ)
  • **呼び出し先**:
  • APR_ARRAY_PUSH (マクロ)
  • APR_ARRAY_IDX (マクロ)

関連データ構造

  • apr_array_header_t: Apache Portable Runtime (APR) の配列構造体。

失敗モードと境界条件

  • add または base が空の場合、ループ条件により適切に処理される。
  • add の要素数が CERTKEYS_IDX_MAX を超える場合、超えた分は置換ではなく末尾への追加として扱われる。

保守担当者向けメモ

  • 本関数は、Apache HTTP Server 2.4.7 以前の cfgMergeString の挙動を維持するために実装されている。
  • CERTKEYS_IDX_MAX の定義変更は、このマージロジックの置換範囲に直接影響を与えるため注意が必要。
