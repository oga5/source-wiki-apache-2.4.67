md_acme_order_remove

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:157

目的

ACME Order オブジェクトが保持する認証 URL リストから、指定された特定の認証 URL を削除します。

シグネチャまたは形状

apr_status_t md_acme_order_remove(md_acme_order_t *order, const char *authz_url)

入力と出力

  • **入力**:
  • order: 編集対象の md_acme_order_t 構造体へのポインタ。
  • authz_url: 削除対象の認証 URL 文字列。
  • **出力**:
  • APR_SUCCESS: 正常に削除された場合。
  • APR_ENOENT: 指定された authz_url がリスト内に見つからなかった場合。

前提条件と不変条件

  • authz_url は NULL であってはなりません（assert による検証）。
  • order->authz_urls は md_array_str_remove が処理可能な文字列配列である必要があります。

アルゴリズムまたは振る舞い

1. md_array_str_index を使用して、指定された authz_url が order->authz_urls 内に存在するかを検索します。 2. URL が存在する場合、md_array_str_remove を呼び出して配列から該当要素を削除し、新しい配列を order->authz_urls に再代入します。この際、order->p（メモリプール）を使用してメモリ管理が行われます。 3. 存在しない場合は何も変更せず APR_ENOENT を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定できません。
  • **呼び出し先**:
  • md_array_str_index
  • md_array_str_remove
  • assert

関連データ構造

  • md_acme_order_t: ACME Order の状態を保持するメイン構造体。本関数はこの構造体内の authz_urls 配列を操作します。

失敗モードと境界条件

  • authz_url が NULL の場合、プログラムはアサート失敗により異常終了します。
  • 指定された URL がリスト内に存在しない場合、副作用を伴わず APR_ENOENT を返します。

保守担当者向けメモ

  • 本関数はメモリプール order->p を使用して配列の再構築を行います。メモリ管理の整合性を保つため、呼び出し側は order オブジェクトの生存期間が適切に管理されていることを確認してください。
