make_cookie

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_usertrack.c:99

目的

クライアントの行動追跡用として、一意な識別子（Cookie）を生成し、レスポンスヘッダーに Set-Cookie（または Set-Cookie2）を追加する。

シグネチャまたは形状

static void make_cookie(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）。
  • **出力**: なし（副作用として r->err_headers_out にヘッダーを追加し、r->notes に Cookie 値を格納する）。

前提条件と不変条件

  • r->server->module_config から usertrack_module の設定が取得可能であること。
  • r->per_dir_config から cookie_dir_rec が取得可能であること。
  • 生成される Cookie 値は、乱数とリクエスト時刻の組み合わせにより構成される。

アルゴリズムまたは振る舞い

1. ap_random_insecure_bytes を使用して乱数を生成し、リクエスト時刻と組み合わせて一意な Cookie 文字列を作成する。 2. 設定（cls->expires）に基づき、有効期限（expires または max-age）を Cookie 文字列に付与する。 3. ディレクトリ設定（dcfg）に基づき、domain、SameSite、Secure、HttpOnly 属性を順次追加する。 4. dcfg->style に応じて Set-Cookie または Set-Cookie2 ヘッダーを r->err_headers_out に追加する。 5. 後続のログ記録のため、生成した Cookie 値を r->notes の "cookie" キーに保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_usertrack.c:267（spot_cookie 関数内からの呼び出しが確認されている）。
  • **呼び出し先**: Apache APR API（ap_random_insecure_bytes, apr_snprintf, apr_psprintf, apr_pstrcat, apr_table_addn, apr_table_setn 等）。

関連データ構造

  • cookie_log_state: サーバー全体の設定（有効期限など）。
  • cookie_dir_rec: ディレクトリごとの設定（Cookie 名、ドメイン、属性フラグなど）。
  • request_rec: リクエスト処理のコンテキスト。

失敗モードと境界条件

  • r->request_time が設定されていない場合、apr_time_now() を使用して時刻を補完する。
  • dcfg->style が CT_COOKIE2 の場合、Set-Cookie2 ヘッダーが使用され、version=1 属性が付与される。
  • dcfg->style が CT_UNSET または CT_NETSCAPE の場合、従来の Set-Cookie 形式で処理される。

保守担当者向けメモ

  • 本関数は暗号学的に安全な乱数を使用していないため、認証目的には使用しないこと。
  • apr_random_insecure_bytes のエントロピー源は APR 実装に依存する。
  • メモリ割り当てはすべて r->pool に対して行われるため、リクエスト終了時に自動的に解放される。
