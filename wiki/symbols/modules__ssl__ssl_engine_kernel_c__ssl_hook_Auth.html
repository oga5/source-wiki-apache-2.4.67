ssl_hook_Auth

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:1291

目的

Apache HTTP Server の認証フェーズにおいて、SSL 設定に基づくアクセス制限の最終的な検証を行う。

シグネチャまたは形状

int ssl_hook_Auth(request_rec *r)

入力と出力

  • 入力: request_rec *r （現在のリクエスト情報）
  • 出力:
  • HTTP_FORBIDDEN: アクセスが拒否された場合。
  • DECLINED: 本フックでは処理を決定せず、後続の認証モジュールに委譲する場合。

前提条件と不変条件

  • r から取得可能なディレクトリ設定 (SSLDirConfigRec) が存在すること。
  • r->notes テーブルに ssl-access-forbidden キーが存在する場合、SSL_OPT_STRICTREQUIRE オプションが有効であればアクセスが厳格に拒否される。

アルゴリズムまたは振る舞い

1. myDirConfig(r) を呼び出し、現在のディレクトリ設定を取得する。 2. 設定オプションに SSL_OPT_STRICTREQUIRE が含まれ、かつリクエストの notes テーブルに ssl-access-forbidden が設定されているかを確認する。 3. 上記条件を満たす場合、HTTP_FORBIDDEN を返してアクセスを拒否する。 4. それ以外の場合は DECLINED を返し、処理を継続する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ssl/mod_ssl.c:741 にて ap_hook_check_authz に登録されている。
  • 呼び出し先: myDirConfig マクロ（内部ヘルパー）、apr_table_get（APR API）。

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • SSLDirConfigRec: mod_ssl のディレクトリ単位の設定構造体。

失敗モードと境界条件

  • SSL_OPT_STRICTREQUIRE が設定されていない場合、ssl-access-forbidden ノートの有無に関わらず本関数は DECLINED を返す。
  • r が無効な場合、myDirConfig の挙動に依存する（ソースコード上では明示的な NULL チェックなし）。

保守担当者向けメモ

  • 本関数は認証フェーズの末端で実行されるガードロジックである。
  • ssl-access-forbidden ノートは、主に ssl_hook_Access 等の先行フェーズで設定されることを想定している。
