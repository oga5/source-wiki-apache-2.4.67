util_ldap_config_t

定義

  • 種別: struct
  • 定義箇所: include/util_ldap.h:140

目的

LDAP 連携におけるディレクトリ単位（per-directory）の設定情報を保持するための構造体です。

シグネチャまたは形状

typedef struct util_ldap_config_t {
    int ChaseReferrals;
    int ReferralHopLimit;
    apr_array_header_t *client_certs;
} util_ldap_config_t;

入力と出力

本構造体は設定保持用であり、直接的な関数入出力は持ちません。各フィールドは以下の通りです。

  • ChaseReferrals (int): LDAP 参照（Referral）を追跡するかどうかのフラグ。
  • ReferralHopLimit (int): 参照を追跡する際の最大ホップ数。
  • client_certs (apr_array_header_t*): LDAP 接続に使用するクライアント証明書のリスト。

前提条件と不変条件

  • apr_array_header_t を使用しているため、APR のメモリプール管理下で初期化される必要があります。
  • modules/ldap/util_ldap.c 内で apr_pcalloc を用いてメモリ確保が行われるケースが確認されています。

アルゴリズムまたは振る舞い

本構造体は主に Apache の設定システムを通じて、リクエスト処理時に ap_get_module_config を介して取得されます。LDAP 接続の挙動（参照追跡の可否や証明書設定）を制御するためのパラメータとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c 内の各関数（ap_get_module_config を使用して設定を取得する箇所）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しは含みません。

関連データ構造

  • apr_array_header_t: 証明書リストの保持に使用。
  • util_ldap_state_t: モジュール全体のグローバルな設定を保持する構造体。

失敗モードと境界条件

  • メモリ確保失敗: apr_pcalloc が失敗した場合の挙動は APR の仕様に依存します。
  • 設定値の不整合: ReferralHopLimit が負の値や不適切な値に設定された場合の LDAP サーバー側の挙動については、本構造体定義からは判断できません。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を扱うため、mod_authnz_ldap 等のクライアントモジュールが設定を読み込む際のインターフェースとなります。
  • フィールドを追加する際は、util_ldap.c 内のメモリ確保箇所（apr_pcalloc）との整合性に注意してください。
