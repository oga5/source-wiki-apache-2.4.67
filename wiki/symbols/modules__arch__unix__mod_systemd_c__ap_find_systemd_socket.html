ap_find_systemd_socket

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_systemd.c:125

目的

systemd のソケットアクティベーション機能により引き継がれたファイルディスクリプタの中から、指定されたポート番号に一致するものを検索して返します。

シグネチャまたは形状

int ap_find_systemd_socket(process_rec *process, apr_port_t port);

入力と出力

  • **入力**:
  • process: プロセス情報を保持する process_rec 構造体（ログ出力に使用）。
  • port: 検索対象のポート番号（apr_port_t）。
  • **出力**:
  • 成功時: 指定ポートに対応するファイルディスクリプタ（整数）。
  • 失敗時: -1。

前提条件と不変条件

  • sd_listen_fds(0) が正常に実行可能であること。
  • 環境変数 LISTEN_FDS が設定されており、有効な整数値として解析可能であること。
  • SD_LISTEN_FDS_START から始まる範囲のファイルディスクリプタが、systemd によって引き継がれていること。

アルゴリズムまたは振る舞い

1. sd_listen_fds(0) を呼び出し、引き継がれたソケットの総数を取得します。エラーまたはソケットが存在しない場合はログを出力して終了します。 2. 環境変数 LISTEN_FDS を取得し、引き継がれたソケットの数（fdcount）を特定します。 3. SD_LISTEN_FDS_START から SD_LISTEN_FDS_START + fdcount - 1 までの範囲でループし、各ファイルディスクリプタに対して sd_is_socket_inet を呼び出します。 4. 指定された port と一致するソケットが見つかった場合、そのファイルディスクリプタを返します。 5. 全てのソケットを確認しても一致するものがない場合、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/listen.c 内でオプション関数として取得・利用されています。
  • **呼び出し先**:
  • sd_listen_fds (systemd API)
  • sd_is_socket_inet (systemd API)
  • ap_log_perror (Apache 内部関数)
  • getenv (標準ライブラリ)
  • atoi (標準ライブラリ)

関連データ構造

  • process_rec: プロセス管理用構造体。
  • SD_LISTEN_FDS_START: systemd ソケットの開始インデックスを示す定数。

失敗モードと境界条件

  • sd_listen_fds が負の値を返した場合、環境変数の解析エラーとしてログを出力し -1 を返します。
  • sd_listen_fds が 0 を返した場合、ソケットが一つも設定されていないと判断し、ログを出力して -1 を返します。
  • 指定ポートに一致するソケットが見つからない場合、ループ終了後に -1 を返します。

保守担当者向けメモ

  • 本関数はオプション関数として登録されており、server/listen.c から APR_RETRIEVE_OPTIONAL_FN を介して動的に解決されます。
  • sd_is_socket_inet の引数に 0, 0, -1 を渡しており、アドレスファミリーやソケットタイプを限定せずにポート番号のみで照合を行っています。
