bio_filter_in_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_engine_io.c:323

目的

Apache HTTP Server の入力フィルタと OpenSSL の BIO レイヤーを仲介するコンテキスト構造体です。復号化前のデータバッファ、フィルタ状態、および HTTPD のバケットブリゲードを保持し、SSL 入力処理のステートを管理します。

シグネチャまたは形状

typedef struct {
    SSL *ssl;
    BIO *bio_out;
    ap_filter_t *f;
    apr_status_t rc;
    ap_input_mode_t mode;
    apr_read_type_e block;
    apr_bucket_brigade *bb;
    char_buffer_t cbuf;
    apr_pool_t *pool;
    char buffer[AP_IOBUFSIZE];
    ssl_filter_ctx_t *filter_ctx;
} bio_filter_in_ctx_t;

入力と出力

  • **入力**: OpenSSL の BIO オブジェクトに BIO_set_data を介して格納されます。
  • **出力**: フィルタ処理関数（ssl_io_input_read 等）において、復号化前のデータ取得や、HTTPD フィルタスタックへのデータ受け渡しに使用されます。

前提条件と不変条件

  • bio_filter_in_ctx_t は、OpenSSL の BIO データ領域に紐付けられており、BIO の生存期間中有効である必要があります。
  • bb (バケットブリゲード) は、接続に関連付けられたメモリプールによって管理されます。

アルゴリズムまたは振る舞い

  • OpenSSL が BIO_read を実行すると、この構造体を通じて HTTPD の ap_get_brigade が呼び出され、ネットワークから暗号化データが取得されます。
  • cbuf (char_buffer_t) を使用して、読み取り時に発生した余剰データを保持し、次回の読み取り要求時に再利用します。
  • buffer フィールドは AP_IOBUFSIZE (8192バイト) の固定長バッファとして、一時的なデータ保持に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ssl_io_input_read, ssl_io_input_getline, ssl_io_filter_error 等の ssl_engine_io.c 内の関数が、BIO からこのコンテキストを取得して操作します。
  • **呼び出し先**: BIO_get_data(bio) を通じて、BIO メソッドの実装から参照されます。

関連データ構造

  • ssl_filter_ctx_t: SSL 接続のメインコンテキスト。
  • char_buffer_t: 読み取りデータのキャッシュ用構造体。
  • apr_bucket_brigade: HTTPD のデータ転送用バケットリスト。

失敗モードと境界条件

  • AP_IOBUFSIZE を超えるデータ読み取りは、複数回のフィルタ呼び出しに分割されます。
  • rc フィールドにより、直近の I/O 操作の apr_status_t が保持されます。

保守担当者向けメモ

  • 本構造体は BIO のデータ領域として格納されるため、メモリ管理は apr_pool_t のライフサイクルと同期させる必要があります。
  • buffer のサイズは AP_IOBUFSIZE に固定されているため、変更が必要な場合は HTTPD の全体設定との整合性に注意してください。
