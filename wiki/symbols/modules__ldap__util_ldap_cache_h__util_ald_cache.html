util_ald_cache

定義

  • 種別: struct
  • 定義箇所: modules/ldap/util_ldap_cache.h:43

目的

LDAP 検索、比較、DN 比較の結果を保持するキャッシュのメタデータ、統計情報、および操作インターフェースを管理するためのメイン構造体です。

シグネチャまたは形状

struct util_ald_cache {
    unsigned long size;
    unsigned long maxentries;
    unsigned long numentries;
    unsigned long fullmark;
    apr_time_t marktime;
    unsigned long ttl;
    unsigned long (*hash)(void *);
    int (*compare)(void *, void *);
    void * (*copy)(util_ald_cache_t *cache, void *);
    void (*free)(util_ald_cache_t *cache, void *);
    void (*display)(request_rec *r, util_ald_cache_t *cache, void *);
    util_cache_node_t **nodes;
    unsigned long numpurges;
    double avg_purgetime;
    apr_time_t last_purge;
    unsigned long npurged;
    unsigned long fetches;
    unsigned long hits;
    unsigned long inserts;
    unsigned long removes;
#if APR_HAS_SHARED_MEMORY
    apr_shm_t *shm_addr;
    apr_rmm_t *rmm_addr;
#endif
};

入力と出力

  • **入力**: キャッシュ操作関数（fetch, insert, remove 等）の引数として渡され、キャッシュの構成情報や統計を保持します。
  • **出力**: キャッシュ内のノード配列（nodes）や、実行時の統計情報（ヒット数、パージ回数等）を保持します。

前提条件と不変条件

  • fullmark はキャッシュ容量の 3/4 に達した時点を追跡するために使用されます。
  • APR_HAS_SHARED_MEMORY が有効な場合、shm_addr および rmm_addr を介してプロセス間で共有されるメモリ領域を管理します。
  • キャッシュ内のデータは、util_ald_alloc を使用して確保され、util_ald_free を通じて明示的に解放される必要があります。

アルゴリズムまたは振る舞い

  • **キャッシュ管理**: 登録された関数ポインタ（hash, compare, copy, free, display）を使用して、ペイロードのハッシュ計算、照合、メモリ確保、解放、表示を抽象的に実行します。
  • **パージ処理**: キャッシュが fullmark に達すると、TTL 期限切れのノードを解放するパージ処理がトリガーされます。
  • **統計更新**: フェッチ、ヒット、挿入、削除の各操作時に対応するカウンタが更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap_cache.h 内で util_ald_cache_t として typedef されています。具体的な呼び出し元については、提供されたソース範囲内では特定できません。
  • **呼び出し先**: 構造体内の関数ポインタを介して、util_ldap_cache.c で実装された各ノード型の操作関数を呼び出します。

関連データ構造

  • util_cache_node_t: キャッシュ内の個別のエントリを管理するリンクリスト構造体。
  • util_ald_cache_t: 本構造体の別名（typedef）。

失敗モードと境界条件

  • キャッシュが fullmark に達した際、パージ処理が実行されます。
  • 共有メモリ環境下では、apr_rmm を介したオフセット管理が行われるため、ポインタの直接参照は避ける必要があります。

保守担当者向けメモ

  • 本構造体はインターフェース定義に特化しています。具体的なハッシュアルゴリズムやロック制御の詳細については util_ldap_cache_mgr.c を参照してください。
