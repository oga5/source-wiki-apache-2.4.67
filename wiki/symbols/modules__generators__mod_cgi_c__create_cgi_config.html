create_cgi_config

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgi.c:100

目的

mod_cgi モジュールで使用されるサーバー設定構造体 (cgi_server_conf) を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_cgi_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t プール。
  • s: 設定対象の server_rec 構造体（本関数内では未使用）。
  • **出力**:
  • 初期化済みの cgi_server_conf 構造体へのポインタを void * として返す。

前提条件と不変条件

  • apr_pcalloc により、割り当てられたメモリ領域はゼロクリアされる。
  • 戻り値の構造体は、リクエストプール p の生存期間に依存する。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して cgi_server_conf サイズのメモリをプール p から確保する。 2. 構造体の各フィールドを以下のデフォルト値で初期化する:

  • logname: NULL
  • logbytes: DEFAULT_LOGBYTES
  • bufbytes: DEFAULT_BUFBYTES

    • 3. 初期化した構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_cgi.c:776 （サーバー設定の初期化時）
  • **呼び出し先**: apr_pcalloc (APR ライブラリ)

関連データ構造

  • cgi_server_conf: 本関数が初期化する、ログパスやバッファサイズを保持する設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボートする可能性がある（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は mod_cgi の設定フックとして機能する。
  • server_rec *s 引数は現在の実装では使用されていないが、将来的な設定の動的変更やサーバー単位の条件分岐のために保持されている。
