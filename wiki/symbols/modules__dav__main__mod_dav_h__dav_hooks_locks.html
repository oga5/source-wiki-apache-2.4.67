dav_hooks_locks

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:268

目的

WebDAV リソースに対するロック管理機能（作成、取得、削除、更新など）を抽象化し、プロバイダーが実装すべき関数ポインタの集合を定義するためのインターフェース構造体です。

シグネチャまたは形状

ソースコード上では typedef struct dav_hooks_locks dav_hooks_locks; として不透明型（opaque type）で宣言されています。具体的な構造体のメンバ構成は、このヘッダーファイル内では公開されていません。

入力と出力

  • **入力**: 各フック関数は、主に request_rec * や dav_resource *、およびロック操作に必要なパラメータを受け取ります。
  • **出力**: 各フック関数は、操作の成功または失敗を示す dav_error * 型の戻り値を返します。

前提条件と不変条件

  • プロバイダーは、この構造体のインスタンスを静的または動的に定義し、mod_dav に登録する必要があります。
  • mod_dav のメインハンドラは、DAV_GET_HOOKS_LOCKS(r) マクロを通じて、リクエストに関連付けられたロックフックを取得します。

アルゴリズムまたは振る舞い

WebDAV メソッド（LOCK, UNLOCK 等）の実行時に、mod_dav はこの構造体を通じてプロバイダー固有のロックロジックを呼び出します。これにより、ファイルシステムベースのロック（mod_dav_fs）や、汎用的なロック管理（mod_dav_lock）といった異なるバックエンドを透過的に切り替えることが可能です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.c: dav_get_lock_hooks 関数および DAV_GET_HOOKS_LOCKS マクロ経由で頻繁に参照されます。
  • **呼び出し先**:
  • modules/dav/fs/lock.c: dav_hooks_locks_fs として実装を提供。
  • modules/dav/lock/locks.c: dav_hooks_locks_generic として実装を提供。

関連データ構造

  • dav_lock: ロック情報を保持する構造体。
  • dav_provider: ロックフックを含む、WebDAV プロバイダーの全機能を束ねる構造体。

失敗モードと境界条件

  • プロバイダーがロック機能をサポートしない場合、対応するフックポインタは NULL に設定される可能性があります。
  • 呼び出し元は、フックが NULL でないことを確認してから実行する必要があります。

保守担当者向けメモ

  • 本構造体のメンバ定義は、mod_dav.h の後続セクションまたは関連する内部ヘッダーで定義されている可能性があります。
  • dav_hooks_locks の実装を追加・変更する場合は、mod_dav.c の DAV_GET_HOOKS_LOCKS マクロとの整合性を維持してください。
