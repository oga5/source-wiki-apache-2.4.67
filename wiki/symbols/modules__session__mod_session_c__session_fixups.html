session_fixups

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:518

目的

リクエスト処理の Fixups フェーズにおいて、セッションのロードおよび環境変数へのセッション情報の注入を行う。

シグネチャまたは形状

static int session_fixups(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）
  • **出力**: int（常に OK を返す）

前提条件と不変条件

  • r->per_dir_config から session_module の設定を取得できること。
  • ap_session_load が内部で呼び出され、セッションのロードを試みる。
  • 設定で環境変数への出力が有効（conf->env）な場合、ロードされたセッション情報が subprocess_env に設定される。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリ設定を取得する。 2. ap_session_load(r, &z) を呼び出し、セッションをロードする。エラー発生時やセッション未設定時は、壊れたセッションを無視して処理を続行する。 3. 設定で環境変数への出力が有効な場合:

  • セッションが存在すれば session_identity_encode でエンコードし、subprocess_env に HTTP_SESSION として格納する。
  • r->headers_in から "Session" ヘッダーを削除する。

    • 4. OK を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の Fixups フック（ap_hook_fixups(session_fixups, ...) により登録）。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_session_load
  • session_identity_encode
  • apr_table_set / apr_table_unset

関連データ構造

  • request_rec: リクエストのコンテキスト。
  • session_dir_conf: ディレクトリ単位のセッション設定。
  • session_rec: セッションデータを保持する構造体。

失敗モードと境界条件

  • ap_session_load が失敗した場合、セッションは NULL として扱われ、環境変数への注入はスキップされる。
  • conf->env が無効な場合、環境変数への注入処理は実行されない。

保守担当者向けメモ

  • 本関数はリクエスト処理の初期段階でセッションを準備する役割を持つ。
  • HTTP_SESSION 環境変数は、この関数によって subprocess_env に注入されるため、CGI 等から参照可能となる。
  • r->headers_in から "Session" ヘッダーを削除する処理は、内部的なセッション情報の漏洩を防ぐための措置である。
