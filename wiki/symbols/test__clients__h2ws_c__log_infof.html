log_infof

定義

  • 種別: function
  • 定義箇所: test/clients/h2ws.c:109

目的

グローバルな verbose フラグが有効な場合に、書式付き文字列をログ出力する。

シグネチャまたは形状

static void log_infof(const char *where, const char *msg, ...)
  • __attribute__((format(printf, 2, 3))) により、コンパイラによる書式文字列の型チェックが行われる。

入力と出力

  • where: ログの発生源を示す識別文字列。
  • msg: printf 形式の書式指定文字列。
  • ...: 書式指定に対応する可変長引数。
  • 出力: 戻り値なし。内部で vsnprintf を使用してバッファへ展開後、log_info を呼び出す。

前提条件と不変条件

  • verbose 変数が真である場合にのみ、ログ出力処理が実行される。
  • ログメッセージの最大長は 8,192 バイトに制限される。

アルゴリズムまたは振る舞い

1. verbose フラグを確認する。 2. va_list を初期化し、vsnprintf を用いて可変長引数を buffer に書き込む。 3. va_end でリストを終了する。 4. 構築した buffer を log_info(where, buffer) に渡し、最終的な出力を行う。

呼び出し元と呼び出し先

  • 呼び出し元: test/clients/h2ws.c 内の各所（フレーム送受信処理など）。
  • 呼び出し先: log_info（ログ出力用関数）。

関連データ構造

  • va_list: 可変長引数を処理するための標準ライブラリ型。

失敗モードと境界条件

  • msg に続く引数が書式指定と一致しない場合、未定義動作となる可能性がある（コンパイラの警告により検知可能）。
  • 展開後のメッセージが 8,192 バイトを超える場合、vsnprintf の仕様に従い切り詰められる。

保守担当者向けメモ

  • 本関数はテストツール h2ws.c におけるデバッグ出力の標準化を目的としている。
  • log_info の実装詳細については、当該シンボルのページを参照すること。
