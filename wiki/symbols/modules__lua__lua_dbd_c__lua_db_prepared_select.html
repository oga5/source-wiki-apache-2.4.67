lua_db_prepared_select

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:382

目的

Lua スクリプトから、事前に準備された SQL ステートメント（プリペアドステートメント）を実行し、その結果セットを Lua テーブルとして取得します。

シグネチャまたは形状

int lua_db_prepared_select(lua_State *L)

入力と出力

  • **入力**:
  • スタック第1引数: プリペアドステートメントを保持する Lua テーブル。
  • スタック第2引数以降: SQL クエリにバインドするパラメータ（可変長）。
  • **出力**:
  • 成功時: 結果セットを保持する Lua テーブル（スタックに1つプッシュ）。
  • 失敗時: nil とエラーメッセージ（スタックに2つプッシュ）、または false とエラーメッセージ。

前提条件と不変条件

  • データベース接続（st->db）が生存していること。
  • 渡されたステートメントオブジェクトが lua_db_prepared_statement 型のユーザーデータであること。
  • 必要なバインド変数の数が、ステートメント定義時の期待値以上であること。

アルゴリズムまたは振る舞い

1. スタックから lua_db_prepared_statement を取得し、バインド変数の数を検証します。 2. 指定されたパラメータを apr_pcalloc で確保したメモリにコピーします。 3. apr_dbd_pselect を呼び出し、SQL を実行します。 4. 成功した場合、結果セットを lua_db_result_set 構造体としてラップし、メタテーブル lua_apr.dbselect を設定します。このメタテーブルには __call メタメソッドとして lua_db_get_row が割り当てられ、Lua 側で結果セットをイテレート可能にします。 5. 失敗した場合は apr_dbd_error を用いてエラーメッセージを生成し、Lua に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_dbd.c 内の初期化処理（lua_pushcfunction による登録）。
  • **呼び出し先**:
  • apr_dbd_pselect (APR DBD API)
  • lua_db_get_row (メタメソッドとして登録)
  • apr_dbd_num_cols / apr_dbd_num_tuples (APR DBD API)

関連データ構造

  • lua_db_prepared_statement: プリペアドステートメントの定義と接続情報を保持。
  • lua_db_result_set: クエリ結果とドライバ情報を保持する構造体。

失敗モードと境界条件

  • **接続切断**: データベース接続が閉じている場合、false とエラーメッセージを返します。
  • **引数不足**: 必要なバインド変数が不足している場合、false と期待値・実測値を含むエラーメッセージを返します。
  • **実行エラー**: apr_dbd_pselect が失敗した場合、nil とエラーメッセージを返します。

保守担当者向けメモ

  • 本関数は apr_dbd_pselect を使用しており、バインド変数は文字列として処理されます。
  • 結果セットのメモリ管理は st->db->pool に依存しており、接続のライフサイクルと同期しています。
