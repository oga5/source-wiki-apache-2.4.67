stream_iter_aws_t

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_mplx.c:412

目的

HTTP/2 ストリームの反復処理中に、ストリームの状態（カウント、送信要求、ウィンドウ枯渇状況）を保持・集計するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    int stream_count;
    int stream_want_send;
    int stream_send_win_exhausted;
} stream_iter_aws_t;

入力と出力

本構造体は関数間でポインタとして渡され、以下のフィールドを通じて状態を更新・参照します。

  • stream_count: 処理対象のストリーム総数。
  • stream_want_send: 送信待ち状態にあるストリーム数。
  • stream_send_win_exhausted: フロー制御ウィンドウが枯渇しているストリーム数。

前提条件と不変条件

  • h2_mplx.c 内のストリーム反復処理において、初期化後に各フィールドがインクリメントまたはデクリメントされることで状態を追跡します。
  • 構造体インスタンスは、反復処理を行う関数のスコープ内でスタック上に確保されるか、コンテキストとして渡されます。

アルゴリズムまたは振る舞い

ストリームの反復処理中に、各ストリームの属性を検査し、本構造体の各フィールドを更新することで、セッション全体の送信可能状態やフロー制御の制約を評価します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_mplx.c 内のストリーム反復処理関数（420, 429, 442, 452 行付近のコードで参照を確認）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • h2_mplx: 本構造体を利用する反復処理の対象となる多重化器。
  • h2_stream: 反復処理の対象となる個別のストリーム。

失敗モードと境界条件

  • 構造体内の各 int フィールドは、ストリーム数に依存します。極端に多数のストリームが存在する場合のオーバーフローについては、h2_mplx のストリーム管理上限に依存します。

保守担当者向けメモ

  • 本構造体は h2_mplx.c 内のローカルな反復処理専用です。外部モジュールからは参照されません。
  • フィールドの追加や変更を行う際は、h2_mplx.c 内の反復処理ロジック（stream_iter_ctx 等との関連）に影響がないか確認してください。
