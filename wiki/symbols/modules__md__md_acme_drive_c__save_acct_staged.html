save_acct_staged

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:65

目的

ACME アカウント情報（アカウントデータおよび秘密鍵）を、指定された管理対象ドメイン（MD）のステージング領域（MD_SG_STAGING）に永続化する。

シグネチャまたは形状

static apr_status_t save_acct_staged(md_acme_t *acme, md_store_t *store, 
                                     const char *md_name, apr_pool_t *p)

入力と出力

  • **入力**:
  • acme: ACME セッション状態を保持する md_acme_t 構造体。
  • store: ストレージ操作を行うための md_store_t ハンドル。
  • md_name: 対象となる管理対象ドメインの名称。
  • p: メモリ割り当てに使用する APR プール。
  • **出力**:
  • apr_status_t: 操作の成功（APR_SUCCESS）または失敗を示すステータスコード。

前提条件と不変条件

  • acme->acct および acme->acct_key が有効な状態であること。
  • MD_SG_STAGING グループへの書き込み権限が存在すること。
  • 本関数による保存はステージング領域に対して行われるため、本番環境のデータには即座に影響を与えない。

アルゴリズムまたは振る舞い

1. md_acme_acct_to_json を呼び出し、acme->acct を JSON 形式に変換する。 2. md_store_save を使用して、変換した JSON データを MD_FN_ACCOUNT としてステージング領域に保存する。 3. ステップ 2 が成功した場合、acme->acct_key を MD_FN_ACCT_KEY としてステージング領域に保存する。 4. 最終的なステータスコードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_drive.c:189
  • **呼び出し先**:
  • md_acme_acct_to_json
  • md_store_save

関連データ構造

  • md_acme_t: ACME プロトコルのセッション状態を管理。
  • md_store_t: ストレージの抽象インターフェース。
  • md_json_t: ACME アカウント情報のシリアライズに使用。

失敗モードと境界条件

  • md_store_save が失敗した場合、そのステータスコードが即座に返される。
  • JSON 変換やファイル書き込み時にメモリ不足やディスク I/O エラーが発生した場合、APR_SUCCESS 以外の値が返される。

保守担当者向けメモ

  • 本関数はステージング領域への書き込みのみを担当する。本番領域への昇格には acme_preload 等の別プロセスが必要である。
  • md_store_save の第 6 引数に 0 が指定されており、デフォルトの保存挙動が適用される。
