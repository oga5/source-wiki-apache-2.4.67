proxyws_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_wstunnel.c:22

目的

mod_proxy_wstunnel モジュールにおけるディレクトリ単位の設定を保持するための構造体です。主に WebSocket トンネリング失敗時のフォールバック動作を制御します。

シグネチャまたは形状

typedef struct {
    unsigned int fallback_to_proxy_http     :1,
                 fallback_to_proxy_http_set :1;
} proxyws_dir_conf;

入力と出力

  • 入力: 設定ファイル（httpd.conf 等）のディレクティブ設定値。
  • 出力: mod_proxy_wstunnel 内の各ハンドラ（proxy_wstunnel_handler 等）が参照する設定状態。

前提条件と不変条件

  • fallback_to_proxy_http_set は、fallback_to_proxy_http が明示的に設定されたかどうかを示すフラグとして機能します。
  • メモリ割り当ては apr_pcalloc を介して行われ、リクエストプール（r->pool）のライフサイクルに依存します。

アルゴリズムまたは振る舞い

  • 本構造体は、ProxyWebsocketFallbackToProxyHttp ディレクティブの設定値を保持します。
  • リクエスト処理時、ap_get_module_config(r->per_dir_config, ...) を通じて取得され、WebSocket トンネルの確立に失敗した場合に mod_proxy_http へ処理を委譲するかどうかの判断基準として使用されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/proxy/mod_proxy_wstunnel.c 内のハンドラおよび設定マージ関数（create_proxyws_dir_config, merge_proxyws_dir_config, proxy_wstunnel_handler 等）。
  • 呼び出し先:
  • 情報なし（本構造体はデータ保持用であり、関数呼び出しを行いません）。

関連データ構造

  • proxy_dir_conf: mod_proxy 基盤側のディレクトリ設定構造体。
  • r->per_dir_config: Apache のリクエスト単位の設定コンテナ。

失敗モードと境界条件

  • 設定が明示されていない場合、fallback_to_proxy_http_set が 0 となり、デフォルトの動作が適用されます。
  • apr_pcalloc によるメモリ確保が失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本構造体はビットフィールドを使用しているため、フラグを追加する際はメモリレイアウトとパディングに注意してください。
  • mod_proxy_http へのフォールバックロジックを変更する際は、この構造体のフラグが正しく継承・マージされているかを確認してください。
