reqtimeout_eor

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reqtimeout.c:348

目的

リクエスト終了（EOR: End of Request）バケットを検知し、接続ごとのタイムアウト期限をリセットすることで、リクエスト間におけるタイムアウト状態の管理を適切に切り替える。

シグネチャまたは形状

static apr_status_t reqtimeout_eor(ap_filter_t *f, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • f: フィルタコンテキスト（reqtimeout_con_cfg を保持）。
  • bb: 処理対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: ap_pass_brigade の戻り値を返す。

前提条件と不変条件

  • フィルタコンテキスト f->ctx は reqtimeout_con_cfg 型として初期化されている必要がある。
  • bb が空でない場合、バケットの末尾を確認する。

アルゴリズムまたは振る舞い

1. bb が空でないか確認する。 2. ブリゲードの最後のバケットが AP_BUCKET_IS_EOR であるか判定する。 3. EOR バケットである場合、フィルタコンテキスト ccfg の timeout_at を 0 に設定し、タイムアウト期限を無効化する。 4. 処理したブリゲードを ap_pass_brigade を通じて後続のフィルタへ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の出力フィルタチェーン（ap_register_output_filter により登録）。
  • **呼び出し先**: ap_pass_brigade（後続のフィルタへブリゲードを渡す）。

関連データ構造

  • reqtimeout_con_cfg: 接続ごとの状態を保持する構造体。timeout_at フィールドを更新する。
  • ap_filter_t: フィルタチェーンのノード。

失敗モードと境界条件

  • bb が空の場合、timeout_at のリセットは行われない。
  • EOR バケットが含まれていない場合、タイムアウト期限の更新はスキップされる。

保守担当者向けメモ

  • 本関数は出力フィルタとして動作するため、リクエスト処理の完了タイミングを正確に捕捉する。
  • timeout_at を 0 にリセットすることで、次のリクエストが開始されるまでの間、誤ったタイムアウト判定が行われないようにしている。
