header_connection

定義

  • 種別: struct
  • 定義箇所: modules/proxy/proxy_util.c:4473

目的

HTTP ヘッダー内の Connection ヘッダー値を解析・管理するための構造体です。

シグネチャまたは形状

typedef struct header_connection {
    apr_pool_t *pool;
    apr_array_header_t *array;
    const char *first;
    unsigned int closed:1;
} header_connection;

入力と出力

  • pool: メモリ管理用のアロケータ。
  • array: Connection ヘッダーに含まれるトークンを格納する配列。
  • first: 最初のトークンへのポインタ。
  • closed: Connection: close が指定されているかを示すフラグ。

前提条件と不変条件

  • pool は、この構造体のライフサイクルを管理するために使用されます。
  • closed ビットフィールドは、接続のクローズ状態を保持します。

アルゴリズムまたは振る舞い

HTTP ヘッダーから Connection 関連の情報を抽出し、後続の処理で接続を維持するか切断するかを判断するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/proxy_util.c 内の解析ロジック（4482行目および4515行目で参照を確認）。
  • 呼び出し先: 提供されたソース excerpt には、この構造体を利用する具体的な関数呼び出しの詳細は含まれていません。

関連データ構造

  • apr_pool_t
  • apr_array_header_t

失敗モードと境界条件

  • pool が適切に初期化されていない場合、メモリ割り当てエラーが発生する可能性があります。
  • Connection ヘッダーが空、または不正な形式である場合の挙動については、本構造体定義のみでは特定できません。

保守担当者向けメモ

  • 本構造体は proxy_util.c 内のローカルな解析処理で使用されています。
  • 構造体の各フィールドがどのように更新されるかについては、proxy_util.c 内の関連する解析関数（4482行目以降の処理）を参照してください。
