md_http_next_req

定義

  • 種別: typedef
  • 定義箇所: modules/md/md_http.h:250

目的

HTTP リクエストのキューから次のリクエストを取得するためのコールバック関数型です。md_http_multi_perform 実行時に、次に処理すべきリクエストを動的に供給するために使用されます。

シグネチャまたは形状

typedef apr_status_t md_http_next_req(md_http_request_t **preq, void *baton);

※注: ソースコードの行250-251の定義および関連箇所に基づくと、この型は md_http_request_t オブジェクトへのポインタを生成し、成功時に APR_SUCCESS を返す関数を指します。

入力と出力

  • **入力**:
  • preq: 生成された md_http_request_t を格納するためのポインタのポインタ。
  • baton: コールバック実行時に渡される不透明なデータポインタ（コンテキスト）。
  • **出力**:
  • 戻り値: apr_status_t 型。リクエストの取得成功時は APR_SUCCESS、これ以上リクエストがない場合やエラー時は適切な APR エラーコードを返します。

前提条件と不変条件

  • preq は有効なメモリ領域を指している必要があります。
  • baton は、このコールバックを登録した呼び出し元によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 呼び出し元（md_http_multi_perform 等）がこのコールバックを呼び出します。 2. コールバックは、次に実行すべき HTTP リクエストを構築または取得します。 3. 成功した場合、preq にリクエストオブジェクトを格納し、APR_SUCCESS を返します。 4. 処理すべきリクエストがなくなった場合、あるいはエラーが発生した場合は、非 APR_SUCCESS を返してループを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: md_http_multi_perform (modules/md/md_http.c:284) がこのコールバックを内部で実行します。
  • **呼び出し先**: 具体的な実装は、この型を使用する各モジュール（md_curl.c 等）によって提供されます。

関連データ構造

  • md_http_request_t: このコールバックによって生成されるリクエストオブジェクト。
  • md_http_t: HTTP セッションを管理する構造体。
  • apr_pool_t: メモリ管理に使用されるプール。

失敗モードと境界条件

  • リクエストの生成に失敗した場合、APR_EGENERAL などのエラーコードを返す必要があります。
  • 供給するリクエストが枯渇した場合は、APR_EOF またはその他の終了を示すステータスを返すことで、呼び出し側のループを安全に停止させる必要があります。

保守担当者向けメモ

  • 本シンボルは関数ポインタ型であり、実装側で baton を通じて状態を保持する設計が一般的です。
  • md_http_multi_perform の引数として渡されるため、このコールバック内で実行される処理は、非同期通信の並列実行を阻害しないよう、可能な限り軽量に保つことが推奨されます。
