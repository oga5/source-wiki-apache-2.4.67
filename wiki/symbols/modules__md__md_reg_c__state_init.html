state_init

定義

  • 種別: function
  • 定義箇所: modules/md/md_reg.c:227

目的

管理対象ドメイン（md_t）に関連付けられた証明書の整合性を検証し、その状態（md_state_t）を更新する。

シグネチャまたは形状

static apr_status_t state_init(md_reg_t *reg, apr_pool_t *p, md_t *md)

入力と出力

  • **入力**:
  • reg: レジストリハンドル。
  • p: メモリプール。
  • md: 状態を更新する対象のドメイン構造体。
  • **出力**:
  • 戻り値: apr_status_t（成功時は APR_SUCCESS）。
  • 副作用: md->state および md->state_descr が更新される。

前提条件と不変条件

  • md->domains が存在する場合、そのプールは引数 p と一致する必要がある（不一致時はエラーログが出力される）。
  • md->renew_window および md->warn_window が未設定の場合、reg の値で初期化される。

アルゴリズムまたは振る舞い

1. 静的証明書ファイルが指定されている場合、更新モードを MD_RENEW_MANUAL に強制する。 2. ドメインに関連付けられた各秘密鍵仕様（md_pkeys_spec_t）に対し、md_reg_get_pubcert を呼び出して証明書をロードする。 3. 証明書が存在しない場合、状態を MD_S_INCOMPLETE に設定する。 4. 証明書が存在する場合、以下の検証を行う：

  • ドメイン名が証明書の代替名（Alt Names）に含まれているか。
  • must-staple 属性が設定と一致しているか。

    • 5. 検証失敗時は MD_S_INCOMPLETE または MD_S_ERROR を設定し、cleanup へ遷移する。 6. 最終的に md->state と md->state_descr を更新して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_reg.c 内の複数の箇所（行 315, 346, 378, 416, 437, 548, 971）から呼び出される。
  • **呼び出し先**: md_reg_get_pubcert, md_cert_count, md_pkeys_spec_get, md_is_covered_by_alt_names, md_cert_must_staple, apr_psprintf, md_log_perror。

関連データ構造

  • md_reg_t: レジストリ管理構造体。
  • md_t: 管理対象ドメインの定義。
  • md_state_t: ドメインの状態（MD_S_COMPLETE, MD_S_INCOMPLETE, MD_S_ERROR）。
  • md_pubcert_t: 公開証明書情報。

失敗モードと境界条件

  • md_reg_get_pubcert が ENOENT を返した場合、証明書欠如として MD_S_INCOMPLETE になる。
  • その他のエラー発生時は MD_S_ERROR となる。
  • 証明書がドメインをカバーしていない、または must-staple 設定が不一致の場合も MD_S_INCOMPLETE となる。

保守担当者向けメモ

  • 本関数は、ドメイン情報のロードや検索のたびに呼び出され、常に最新の状態を反映させる役割を持つ。
  • md->state_descr は apr_psprintf を使用して動的に生成されるため、メモリ消費に注意すること。
