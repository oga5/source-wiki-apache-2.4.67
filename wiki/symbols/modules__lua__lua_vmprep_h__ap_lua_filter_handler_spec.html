ap_lua_filter_handler_spec

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_vmprep.h:99

目的

Apache HTTP Server のフィルタ処理において、特定の Lua スクリプト関数をフィルタハンドラとしてマッピングするための仕様を保持する。

シグネチャまたは形状

typedef struct
{
    const char *function_name;
    const char *file_name;
    const char* filter_name;
    int         direction; /* AP_LUA_FILTER_INPUT | AP_LUA_FILTER_OUTPUT */
} ap_lua_filter_handler_spec;

入力と出力

  • function_name: 実行する Lua 関数名。
  • file_name: 実行する Lua スクリプトのファイルパス。
  • filter_name: フィルタの識別名。
  • direction: フィルタの方向（AP_LUA_FILTER_INPUT または AP_LUA_FILTER_OUTPUT）。

前提条件と不変条件

  • direction フィールドは、入力フィルタか出力フィルタかを区別するために使用される。
  • 構造体インスタンスは、通常 apr_pcalloc 等を用いてメモリプール上で確保される。

アルゴリズムまたは振る舞い

本構造体は、mod_lua におけるフィルタ設定の保持に使用される。設定読み込み時に ap_lua_filter_handler_spec が生成され、cfg->mapped_filters 配列に格納される。その後、リクエスト処理時にこの仕様を参照して Lua VM が準備され、指定された関数がフィルタとして実行される。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/mod_lua.c において、設定の解析（apr_pcalloc による確保）およびフィルタの登録処理（apr_array_push による配列への追加）が行われている。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはない。実行ロジックは mod_lua.c 内のフィルタフック処理から参照される。

関連データ構造

  • ap_lua_dir_cfg: フィルタハンドラのリスト（mapped_filters）を保持するディレクトリ設定構造体。
  • apr_array_header_t: mapped_filters の型として使用される APR 配列。

失敗モードと境界条件

  • function_name や file_name が未設定の場合、Lua VM 実行時にエラーとなる可能性がある。
  • direction に不正な値が設定された場合、フィルタチェーンへの登録が意図通りに行われない可能性がある。

保守担当者向けメモ

  • 本構造体は mod_lua.c のフィルタ関連フックと密接に結合している。変更を加える際は、mod_lua.c 内のフィルタ登録ロジックへの影響を確認すること。
