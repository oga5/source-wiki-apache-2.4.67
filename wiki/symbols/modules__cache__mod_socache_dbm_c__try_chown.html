try_chown

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:101

目的

指定されたファイル（またはその派生ファイル）の所有権を、Apache の設定で定義された実行ユーザー（ap_unixd_config.user_id）に変更する。

シグネチャまたは形状

static int try_chown(apr_pool_t *p, server_rec *s, const char *name, const char *suffix)

入力と出力

  • **入力**:
  • p: メモリ割り当て用のプール。
  • s: エラーログ出力に使用するサーバーレコード。
  • name: 対象となるファイルパスのベース名。
  • suffix: name に付加する拡張子（NULL の場合は付加しない）。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は -1 を返す。

前提条件と不変条件

  • ap_unixd_config.user_id が有効なユーザー ID であること。
  • 呼び出し元プロセスが、対象ファイルの所有権を変更する権限（通常は root 権限）を持っていること。

アルゴリズムまたは振る舞い

1. suffix が指定されている場合、apr_pstrcat を用いて name と suffix を連結する。 2. chown システムコールを呼び出し、所有者を ap_unixd_config.user_id に変更する。グループ ID は変更しない（-1 を指定）。 3. chown が失敗した場合、エラーコードが ENOENT（ファイルが存在しない）以外であれば、ap_log_error を使用してエラーログを出力する。 4. 成功または ENOENT の場合は 0 を、それ以外のエラー時は -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dbm.c 内の socache_dbm_init 関数から、DBM ファイル（.dir, .pag, .db 等）の権限調整のために複数回呼び出されている。
  • **呼び出し先**: apr_pstrcat（APR）、chown（システムコール）、ap_log_error（Apache API）。

関連データ構造

  • server_rec: サーバー設定情報。
  • ap_unixd_config: Unix 権限設定を保持するグローバル構造体。

失敗モードと境界条件

  • **ファイル不在**: 対象ファイルが存在しない場合、ENOENT エラーを無視して 0 を返す。
  • **権限不足**: プロセスに権限がない場合、chown は失敗し、エラーログが出力された上で -1 が返される。

保守担当者向けメモ

  • 本関数は DBM ファイルの作成後に実行される権限管理ユーティリティである。
  • chown の第 3 引数に -1 を渡すことで、グループ ID の変更を明示的に抑制している。
