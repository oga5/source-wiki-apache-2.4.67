ap_bucket_eoc_create

定義

  • 種別: function
  • 定義箇所: server/eoc_bucket.c:38

目的

接続終了（EOC: End of Connection）を通知するためのメタデータバケットを動的に割り当て、初期化する。

シグネチャまたは形状

apr_bucket *ap_bucket_eoc_create(apr_bucket_alloc_t *list)

入力と出力

  • **入力**: list (バケット割り当て用の apr_bucket_alloc_t 構造体へのポインタ)
  • **出力**: 初期化済みの apr_bucket 構造体へのポインタ。

前提条件と不変条件

  • 入力された list は有効なバケットアロケータであること。
  • 生成されたバケットはメタデータバケットとして振る舞い、データペイロードを持たない。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を使用して、apr_bucket 構造体分のメモリを確保する。 2. APR_BUCKET_INIT マクロでバケットの初期化を行う。 3. バケットの解放関数 (free) に apr_bucket_free を設定する。 4. バケットのリストポインタに引数の list を割り当てる。 5. ap_bucket_eoc_make を呼び出し、バケットの型を ap_bucket_type_eoc に設定して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_filters.c:1505
  • modules/proxy/mod_proxy_http.c:1153
  • server/connection.c:85
  • **呼び出し先**:
  • apr_bucket_alloc (APR)
  • APR_BUCKET_INIT (APR)
  • ap_bucket_eoc_make (server/eoc_bucket.c)

関連データ構造

  • apr_bucket: バケットの基本構造体。
  • ap_bucket_type_eoc: EOC バケットの挙動を定義する型構造体。

失敗モードと境界条件

  • メモリ不足時には apr_bucket_alloc が失敗し、呼び出し元へ制御が戻る（または異常終了する可能性がある）。

保守担当者向けメモ

  • 本関数はメタデータバケットを生成するファクトリ関数である。
  • 破棄時には apr_bucket_destroy_noop が使用されるため、バケット自体に特別なクリーンアップ処理は不要である。
