h1_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_request.c:63

目的

HTTP/1 形式のヘッダー処理および変換プロセスにおいて、一時的な状態とリソースを保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    apr_table_t *headers;
    apr_pool_t *pool;
    apr_status_t status;
    h2_hd_scratch *scratch;
} h1_ctx;

入力と出力

  • **入力**: 構造体内の各フィールドは、ヘッダーの蓄積 (headers)、メモリ管理 (pool)、処理結果のステータス (status)、およびスクラッチ領域 (scratch) を保持します。
  • **出力**: 本構造体は、HTTP/2 から HTTP/1 形式への変換処理における中間状態として利用されます。

前提条件と不変条件

  • pool は、このコンテキストに関連付けられたリクエストや処理のライフサイクルを管理するために使用されます。
  • headers は apr_table_t 型であり、HTTP ヘッダーのキーと値のペアを格納します。

アルゴリズムまたは振る舞い

本構造体は、HTTP/2 ストリームから受信したヘッダーを Apache の標準的なテーブル形式に蓄積し、変換処理の進捗やエラー状態を追跡するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_request.c および modules/http2/h2_proxy_util.c 内の関数から参照されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • apr_table_t: ヘッダーの格納に使用。
  • apr_pool_t: メモリ管理に使用。
  • h2_hd_scratch: ヘッダー変換時の作業領域として使用。

失敗モードと境界条件

  • status フィールドにより、ヘッダー処理中のエラー状態（apr_status_t）を保持します。
  • メモリ割り当ての失敗や、ヘッダーの解析エラーが発生した場合、この構造体を通じてエラーが伝播されます。

保守担当者向けメモ

  • 本構造体は h2_request.c と h2_proxy_util.c の両方で定義・利用されています。変更を加える際は、両方のファイルにおける整合性に注意してください。
  • h2_hd_scratch の詳細については、関連するヘッダー処理ロジックを参照してください。
