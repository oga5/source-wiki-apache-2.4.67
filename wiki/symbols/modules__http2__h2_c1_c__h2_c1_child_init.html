h2_c1_child_init

定義

  • **種別**: function
  • **定義箇所**: modules/http2/h2_c1.c:58

目的

HTTP/2 サブシステムにおける子プロセス起動時の初期化処理を行います。ワーカースレッドプールの生成、設定の初期化、およびログ出力用オプション関数の取得を担います。

シグネチャまたは形状

apr_status_t h2_c1_child_init(apr_pool_t *pool, server_rec *s)

入力と出力

  • **入力**:
  • pool: プロセス生存期間のメモリプール。
  • s: サーバー設定レコード。
  • **出力**:
  • apr_status_t: 初期化の成功（APR_SUCCESS）または失敗を示すステータスコード。

前提条件と不変条件

  • h2_config_init が呼び出され、HTTP/2 関連の設定が初期化されること。
  • h2_workers_create により、並列処理のためのワーカープールが生成されること。

アルゴリズムまたは振る舞い

1. **MPM 状態の確認**: ap_mpm_query を使用して、現在の MPM が非同期 I/O (AP_MPMQ_IS_ASYNC) および待機 I/O (AP_MPMQ_CAN_WAITIO) をサポートしているかを確認し、フラグを設定します。 2. **設定初期化**: h2_config_init を呼び出し、HTTP/2 の設定を適用します。 3. **ワーカー生成**: h2_get_workers_config で取得した設定に基づき、h2_workers_create でワーカープールを初期化します。 4. **オプション関数取得**: ap_logio_add_bytes_in および ap_logio_add_bytes_out を APR_RETRIEVE_OPTIONAL_FN で取得し、グローバル変数に格納します。 5. **多重化器の初期化**: 最後に h2_mplx_c1_child_init を呼び出し、多重化処理の初期化を完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c:167
  • **呼び出し先**:
  • ap_mpm_query
  • h2_config_init
  • h2_get_workers_config
  • h2_workers_create
  • APR_RETRIEVE_OPTIONAL_FN (ap_logio_add_bytes_in/out)
  • h2_mplx_c1_child_init

関連データ構造

  • apr_pool_t: メモリ管理用。
  • server_rec: サーバー設定情報。
  • h2_workers: ワーカープール管理オブジェクト。

失敗モードと境界条件

  • ap_mpm_query がサポートされていない MPM の場合、フラグはデフォルト値（0）に設定されます。
  • 依存するオプション関数（logio）がロードされていない場合、取得に失敗する可能性があります。
  • h2_workers_create や h2_mplx_c1_child_init がエラーを返した場合、そのステータスがそのまま呼び出し元へ伝播します。

保守担当者向けメモ

  • 本関数は子プロセスの起動時に一度だけ実行されることを想定しています。
  • async_mpm および mpm_can_waitio はモジュール内の静的変数として保持され、後続の接続処理で MPM の挙動を切り替えるために使用されます。
