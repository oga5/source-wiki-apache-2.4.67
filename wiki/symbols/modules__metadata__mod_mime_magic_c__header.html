header

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_mime_magic.c:191

目的

tar アーカイブフォーマットのファイルヘッダー情報を保持し、ファイル内容の解析時にアーカイブ構造を識別するために使用されます。

シグネチャまたは形状

struct header {
    char name[NAMSIZ];
    char mode[8];
    char uid[8];
    char gid[8];
    char size[12];
    char mtime[12];
    char chksum[8];
    char linkflag;
    char linkname[NAMSIZ];
    char magic[8];
    char uname[TUNMLEN];
    char gname[TGNMLEN];
    char devmajor[8];
    char devminor[8];
} header;

入力と出力

  • 入力: ファイルの先頭バイト列（tar ヘッダー構造に合致するもの）。
  • 出力: 構造体メンバへの各フィールド値の格納。

前提条件と不変条件

  • NAMSIZ、TUNMLEN、TGNMLEN などの定数が定義されている環境で利用されること。
  • ファイルの先頭から特定のオフセット位置にあるバイト列が、tar ヘッダーの仕様に従っていること。

アルゴリズムまたは振る舞い

  • ファイル解析プロセスにおいて、読み込んだバイト列をこの構造体にキャストまたはコピーすることで、アーカイブ内のファイル名、パーミッション、サイズ、タイムスタンプ等のメタデータを抽出します。
  • mod_mime_magic 内のヒューリスティックな判定処理において、特定のバイトパターンが tar アーカイブであるかを検証する際のテンプレートとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: mod_mime_magic.c 内のファイル解析関数（ascmagic 等）。
  • 呼び出し先: 情報なし。
  • 注記: ツールによる検索結果で提示された ap_expr.h 等の「header」という単語を含む参照は、HTTP ヘッダーに関するものであり、本構造体とは無関係です。

関連データ構造

  • struct magic: マジックナンバーのルールを保持する連結リスト。
  • magic_req_rec: リクエスト単位の解析結果を保持する構造体。

失敗モードと境界条件

  • ファイルサイズが sizeof(struct header) に満たない場合、構造体へのマッピングは不完全となります。
  • magic フィールドや chksum フィールドが期待される値と一致しない場合、tar アーカイブとしての識別は失敗します。

保守担当者向けメモ

  • 本構造体は UNIX の file コマンドのロジックを移植したものであり、tar フォーマットの仕様に厳密に依存しています。
  • 構造体内の各フィールドは char 配列であり、数値フィールド（size や mtime 等）を扱う際は、文字列としてのパースが必要です。
