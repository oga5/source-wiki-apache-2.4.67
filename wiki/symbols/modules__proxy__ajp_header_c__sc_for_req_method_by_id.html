sc_for_req_method_by_id

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_header.c:170

目的

Apache の内部リクエストメソッド ID を、AJP プロトコルで定義された数値コードに変換します。

シグネチャまたは形状

static int sc_for_req_method_by_id(request_rec *r)

入力と出力

  • **入力**: request_rec *r (Apache のリクエスト構造体)
  • **出力**: int (AJP メソッドコード、または変換不能な場合の UNKNOWN_METHOD)

前提条件と不変条件

  • r->method_number が有効な範囲（0 から M_INVALID まで）にあること。
  • sc_for_req_method_table がメソッド ID をインデックスとして AJP コードを保持していること。

アルゴリズムまたは振る舞い

1. r->method_number を取得し、範囲外であれば UNKNOWN_METHOD を返します。 2. r->header_only が真（HEAD リクエスト）の場合、強制的に SC_M_HEAD を返します。 3. 上記以外の場合、sc_for_req_method_table を参照し、対応する AJP メソッドコードを返します。テーブルにエントリが存在しない場合は UNKNOWN_METHOD を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/ajp_header.c:232 (ajp_marshal_into_msgb 内)
  • **呼び出し先**: 情報なし（直接的な関数呼び出しは行わず、テーブル参照と構造体メンバの評価のみを行う）

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • sc_for_req_method_table: メソッド ID から AJP コードへのマッピングを保持する静的テーブル（ファイルスコープ）。

失敗モードと境界条件

  • r->method_number が M_INVALID を超える場合、UNKNOWN_METHOD を返します。
  • sc_for_req_method_table の該当インデックスが 0（未定義）の場合、UNKNOWN_METHOD を返します。

保守担当者向けメモ

  • 新しい HTTP メソッドをサポートする場合、sc_for_req_method_table の更新が必要になる可能性があります。
  • r->header_only が優先されるロジックであるため、HEAD リクエストの挙動を変更する際は注意してください。
