authenticate_basic_fake

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_basic.c:436

目的

AuthBasicFake ディレクティブで設定された式を評価し、リクエストの Authorization ヘッダーを動的に生成・上書きすることで、バックエンドへの認証情報を偽装します。

シグネチャまたは形状

static int authenticate_basic_fake(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）。
  • **出力**:
  • OK: 認証ヘッダーの生成または更新に成功。
  • DECLINED: 偽装設定がない、または式の結果が空のため処理をスキップ。
  • HTTP_INTERNAL_SERVER_ERROR: 式の評価中にエラーが発生。

前提条件と不変条件

  • auth_basic_config_rec から取得した fakeuser が設定されている必要があります。
  • 評価対象の式（fakeuser, fakepass）は ap_expr エンジンによって処理されます。
  • 認証ヘッダーの生成には r->pool を使用し、メモリ管理は Apache のリクエストプールに依存します。

アルゴリズムまたは振る舞い

1. 設定から fakeuser の有無を確認し、なければ DECLINED を返します。 2. ap_expr_str_exec を用いてユーザー名とパスワードの式を順次評価します。 3. 評価中にエラーが発生した場合はログを出力し HTTP_INTERNAL_SERVER_ERROR を返します。 4. 評価結果が空文字列の場合、既存の Authorization ヘッダーを削除し DECLINED を返します。 5. 成功時、user:pass を Base64 エンコードし、Basic スキームを付与した文字列を Authorization ヘッダーとして r->headers_in に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の fixups フックとして登録されています（modules/aaa/mod_auth_basic.c:494）。
  • **呼び出し先**:
  • ap_get_module_config: 設定取得。
  • ap_expr_str_exec: 式の評価。
  • ap_log_rerror: エラーおよび情報ログの出力。
  • apr_table_unset / apr_table_setn: ヘッダー操作。
  • ap_pbase64encode: Base64 エンコード。

関連データ構造

  • auth_basic_config_rec: ディレクトリごとの設定を保持し、fakeuser および fakepass の式情報を含みます。
  • request_rec: リクエストのコンテキストおよびヘッダーテーブルを保持します。

失敗モードと境界条件

  • **式評価エラー**: ap_expr の構文エラー等で err が設定された場合、500 エラーを返します。
  • **空の式評価**: ユーザー名またはパスワードが空と評価された場合、認証ヘッダーを削除して処理を中断します。

保守担当者向けメモ

  • 本関数は fixups フックで実行されるため、他の認証モジュールが処理を行う前にヘッダーを上書きする可能性があります。
  • ap_expr の評価結果はリクエストプールに割り当てられるため、明示的な解放は不要です。
