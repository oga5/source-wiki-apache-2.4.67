ssl_init_ctx_tls_extensions

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_init.c:634

目的

SSL コンテキスト（SSL_CTX）に対して、SNI（Server Name Indication）、OCSP ステープリング、および TLS-SRP（Secure Remote Password）といった TLS 拡張機能の設定を適用する。

シグネチャまたは形状

apr_status_t ssl_init_ctx_tls_extensions(server_rec *s,
                                         apr_pool_t *p,
                                         apr_pool_t *ptemp,
                                         modssl_ctx_t *mctx)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p: 長期生存プール。
  • ptemp: 一時的なメモリプール。
  • mctx: 設定情報を含む modssl_ctx_t 構造体。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードまたは ssl_die() によるプロセス終了。

前提条件と不変条件

  • mctx->ssl_ctx が有効な SSL_CTX オブジェクトを保持していること。
  • OpenSSL のバージョンが、使用する拡張機能（特に SSL_CTX_set_client_hello_cb）の要件を満たしていること。

アルゴリズムまたは振る舞い

1. **SNI 設定**: ssl_callback_ServerNameIndication をコールバックとして登録し、SNI 処理を有効化する。 2. **ClientHello コールバック**: OpenSSL 1.1.1 以降（かつ LibreSSL 以外）の場合、ssl_callback_ClientHello を設定し、接続初期段階でのプロトコルバージョン制御を可能にする。 3. **OCSP ステープリング**: mctx->stapling_enabled が有効かつプロキシモードでない場合、modssl_init_stapling を呼び出して初期化する。 4. **TLS-SRP**: mctx->srp_vfile が指定されている場合、SRP データベースを読み込み、サーバーパラメータ用コールバック ssl_callback_SRPServerParams を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:1408 (推測される ssl_init_ctx 内のフロー)
  • **呼び出し先**:
  • SSL_CTX_set_tlsext_servername_callback, SSL_CTX_set_tlsext_servername_arg
  • SSL_CTX_set_client_hello_cb (条件付き)
  • modssl_init_stapling (条件付き)
  • SRP_VBASE_new, SRP_VBASE_init (条件付き)
  • ssl_die (エラー時)

関連データ構造

  • modssl_ctx_t: SSL 設定を保持する主要な構造体。
  • server_rec: Apache サーバー設定構造体。

失敗モードと境界条件

  • **SNI 設定失敗**: コールバック登録に失敗した場合、ssl_die() を呼び出しサーバー起動を停止する。
  • **SRP 初期化失敗**: ファイル読み込みや構造体生成に失敗した場合、ssl_die() を呼び出しサーバー起動を停止する。
  • **OCSP 初期化失敗**: modssl_init_stapling がエラーを返した場合、そのエラーコードを伝播させる。

保守担当者向けメモ

  • OpenSSL のバージョン依存処理（OPENSSL_VERSION_NUMBER）が含まれているため、ライブラリ更新時には挙動の変化に注意すること。
  • ssl_die() が呼び出されるとプロセスが終了するため、設定エラーは起動時に即座に検知される。
