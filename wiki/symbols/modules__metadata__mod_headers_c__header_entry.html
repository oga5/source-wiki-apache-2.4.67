header_entry

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_headers.c:127

目的

Apache HTTP Server における Header および RequestHeader ディレクティブで指定された、個別のヘッダー操作ルールを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    hdr_actions action;            /* 実行するアクション */
    const char *header;            /* 対象となるヘッダー名 */
    apr_array_header_t *ta;        /* format_tag 構造体の配列 */
    ap_regex_t *regex;             /* 正規表現（edit用） */
    const char *condition_var;     /* 条件変数名 */
    const char *subs;              /* 置換文字列 */
    ap_expr_info_t *expr;          /* 条件式（入力用） */
    ap_expr_info_t *expr_out;      /* 条件式（出力用） */
} header_entry;

入力と出力

  • 入力: Header / RequestHeader ディレクティブの解析時に生成されます。
  • 出力: do_headers_fixup 関数が本構造体を参照し、リクエストのヘッダーテーブル（headers_in, headers_out, err_headers_out）に対して操作を適用します。

前提条件と不変条件

  • ta フィールドは parse_format_string によって生成された format_tag 構造体の配列を指す必要があります。
  • 設定ファイル内の出現順序が適用順序を決定するため、構造体は headers_conf 内の配列に順次格納されます。

アルゴリズムまたは振る舞い

  • process_tags 関数により、ta 配列内の動的フォーマットタグが評価され、ヘッダー値が生成されます。
  • edit アクションが指定されている場合、regex フィールドを用いて正規表現置換が行われます。
  • expr または expr_out が設定されている場合、リクエスト処理時に式評価エンジン（ap_expr）によって適用可否が判定されます。

呼び出し元と呼び出し先

  • 呼び出し元: header_inout_cmd（ディレクティブ解析時）、do_headers_fixup（ヘッダー適用時）、process_tags（値生成時）。
  • 呼び出し先: ap_expr エンジン、正規表現ライブラリ、apr_table_t 操作関数。

関連データ構造

  • headers_conf: 本構造体を格納するディレクトリごとの設定構造体。
  • hdr_actions: 実行可能なアクション（set, add, append, merge, unset, edit 等）を定義する列挙型。
  • format_tag: ヘッダー値内の動的要素を処理するための構造体。

失敗モードと境界条件

  • 正規表現（regex）のコンパイル失敗や、式（expr）の構文エラーが発生する可能性があります。
  • edit アクションを多用した場合、リクエストごとの正規表現処理によりパフォーマンスが低下する可能性があります。

保守担当者向けメモ

  • 本構造体は request_rec->pool または cmd->pool を使用してメモリ管理されているため、リクエスト終了時に自動的に解放されます。
  • early 指定されたヘッダー処理と、通常のフィルタ処理による適用タイミングの差異に注意してください。
