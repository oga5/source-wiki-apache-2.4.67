modssl_ctx_init

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_config.c:120

目的

SSL 設定構造体 modssl_ctx_t の各メンバをデフォルト値で初期化し、メモリ領域を安全な状態にセットアップする。

シグネチャまたは形状

static void modssl_ctx_init(modssl_ctx_t *mctx, apr_pool_t *p)

入力と出力

  • mctx: 初期化対象となる modssl_ctx_t 構造体へのポインタ。
  • p: 設定の割り当てに使用する apr_pool_t プール。
  • 戻り値: なし。

前提条件と不変条件

  • mctx は有効なメモリ領域を指していること。
  • p は apr_pool_t として有効であること。
  • HAVE_SSL_CONF_CMD が定義されている環境では、mctx->ssl_ctx_config が SSL_CONF_CTX_new() で生成され、クリーンアップハンドラが登録される。

アルゴリズムまたは振る舞い

1. ポインタ型メンバ（sc, ssl_ctx, pks, pkp, ticket_key 等）を NULL に初期化する。 2. 数値型メンバ（protocol, pphrase_dialog_type, crl_check_mask, verify_depth 等）をデフォルト値または UNSET に設定する。 3. OCSP およびステープリング関連のメンバを初期化する。 4. HAVE_SSL_CONF_CMD が有効な場合、OpenSSL の SSL_CONF_CTX を構築し、フラグ（FILE, SERVER, CERTIFICATE）を設定する。また、パラメータ格納用の配列を apr_array_make で生成する。 5. 最後に、ピア検証関連のフラグを UNSET に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/ssl/ssl_engine_config.c:208
  • modules/ssl/ssl_engine_config.c:427
  • 呼び出し先:
  • SSL_CONF_CTX_new (OpenSSL)
  • apr_pool_cleanup_register
  • SSL_CONF_CTX_set_flags (OpenSSL)
  • apr_array_make

関連データ構造

  • modssl_ctx_t: 初期化対象の主要構造体。
  • apr_pool_t: メモリ管理用プール。
  • SSL_CONF_CTX: OpenSSL の設定コンテキスト（HAVE_SSL_CONF_CMD 時）。

失敗モードと境界条件

  • apr_pool_t が枯渇している場合、apr_array_make 等のメモリ確保処理で異常終了する可能性がある。
  • SSL_CONF_CTX_new が失敗した場合の戻り値チェックは本関数内には含まれていない。

保守担当者向けメモ

  • 本関数は設定の初期化のみを担当する。実際の値の適用やマージは、modssl_ctx_cfg_merge 等の関数で行われる。
  • HAVE_SSL_CONF_CMD などの条件付きコンパイルブロックが多用されているため、OpenSSL のバージョンやビルドオプションによる挙動の差異に注意すること。
