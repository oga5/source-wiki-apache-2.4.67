md_match_mode_t

定義

  • 種別: enum
  • 定義箇所: modules/md/mod_md_config.h:47

目的

Managed Domain (MD) において、DNS 名を仮想ホスト（vhost）にどのように照合するかを決定するためのマッチングモードを定義します。

シグネチャまたは形状

typedef enum {
    MD_MATCH_ALL,
    MD_MATCH_SERVERNAMES,
} md_match_mode_t;

入力と出力

  • 本列挙型は、設定構造体 md_srv_conf_t 内の match_mode メンバとして保持されます。
  • 入力値として MD_MATCH_ALL または MD_MATCH_SERVERNAMES のいずれかをとります。

前提条件と不変条件

  • 特になし。

アルゴリズムまたは振る舞い

  • MD_MATCH_ALL: すべての DNS 名を対象として照合を行います。
  • MD_MATCH_SERVERNAMES: サーバー名に基づいて照合を行います。
  • 実行時の照合ロジックにおいて、この値が参照され、ドメインと仮想ホストの関連付けの厳密さが制御されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md_config.h:87 にて md_srv_conf_t 構造体のメンバとして定義されています。
  • 呼び出し先: 提供されたソース情報からは、本シンボルを直接呼び出している関数や、本シンボルが呼び出している関数に関する情報は確認できません。

関連データ構造

  • md_srv_conf_t: 本列挙型をメンバとして保持する、仮想ホストごとの設定構造体。

失敗モードと境界条件

  • 定義された列挙値以外の値が設定された場合の挙動については、本ソース excerpt 内に記述がありません。
