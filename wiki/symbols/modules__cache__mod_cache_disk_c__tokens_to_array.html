tokens_to_array

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:321

目的

カンマ区切りの文字列リストを解析し、正規化された配列として格納します。特に Vary ヘッダー値の順序に依存しない比較を実現するために使用されます。

シグネチャまたは形状

static void tokens_to_array(apr_pool_t *p, const char *data, apr_array_header_t *arr)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • data: 解析対象のカンマ区切り文字列。
  • arr: トークンを格納する apr_array_header_t 構造体。
  • **出力**: なし（arr が更新される）。

前提条件と不変条件

  • arr は事前に初期化されている必要があります。
  • data は ap_get_list_item で処理可能な形式である必要があります。
  • 処理後、arr 内の要素は array_alphasort に基づいて昇順にソートされます。

アルゴリズムまたは振る舞い

1. ap_get_list_item を使用して、data からトークンを順次抽出します。 2. 抽出した各トークンを apr_array_push を用いて arr に追加します。 3. すべてのトークン抽出後、qsort を呼び出し、array_alphasort を用いて配列内の要素をアルファベット順にソートします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c:1034
  • modules/cache/mod_cache_socache.c:835
  • **呼び出し先**:
  • ap_get_list_item (トークン抽出)
  • apr_array_push (配列への追加)
  • qsort (標準ライブラリ関数)
  • array_alphasort (比較関数)

関連データ構造

  • apr_array_header_t: トークンを保持する動的配列構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • data が空または NULL の場合、ループは実行されず、配列は空のままとなります。
  • ap_get_list_item が解析できない形式の文字列が渡された場合の挙動は、当該関数の仕様に依存します。

保守担当者向けメモ

  • 本関数は Vary ヘッダーの正規化を目的としています。異なる順序で記述された同一のヘッダー値が、キャッシュキーとして同一視されることを保証します。
  • ソートアルゴリズムを変更する場合は、array_alphasort の実装を確認してください。
