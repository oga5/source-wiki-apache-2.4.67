scgi_canon

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:177

目的

scgi:// スキームで指定されたプロキシリクエストの URL を解析・正規化し、Apache HTTP Server のプロキシサブシステムが内部的に処理可能な形式へ変換します。

シグネチャまたは形状

static int scgi_canon(request_rec *r, char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • url: 正規化対象の URL 文字列。
  • **出力**:
  • 成功時は OK を返します。
  • スキームが一致しない場合は DECLINED を返します。
  • URL 解析エラーや正規化失敗時は HTTP_BAD_REQUEST を返します。

前提条件と不変条件

  • r->per_dir_config からコアディレクトリ設定を取得し、エンコードされたスラッシュの扱いを決定します。
  • URL が scgi:// で始まることを前提としています。
  • 動的メモリ割り当てはすべて r->pool に紐付けられ、リクエスト終了時に自動的に解放されます。

アルゴリズムまたは振る舞い

1. **スキーム検証**: URL が scgi:// で開始するか確認し、一致しない場合は処理をスキップします。 2. **ホスト・ポート解析**: ap_proxy_canon_netloc を使用してホスト名とポート番号を抽出します。ポートが指定されていない場合はデフォルトの SCGI_DEF_PORT を使用します。 3. **IPv6 対応**: ホスト名にコロンが含まれる場合（IPv6 リテラル）、ホスト名を [] で囲みます。 4. **パス正規化**: ap_proxy_canonenc_ex を使用してパス部分を正規化します。 5. **内部形式への変換**: r->filename を proxy:scgi://<host>[:port]/<path> という形式で再構築します。 6. **パス情報設定**: subprocess_env に proxy-scgi-pathinfo が設定されている場合、r->path_info を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_scgi.c:670 にて proxy_hook_canon_handler に登録されています。
  • **呼び出し先**:
  • ap_get_core_module_config
  • ap_proxy_canon_netloc
  • ap_proxy_canonenc_ex
  • apr_pstrcat
  • apr_table_get
  • ap_log_rerror

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • core_dir_config: ディレクトリ単位のコア設定。

失敗モードと境界条件

  • **URL 解析エラー**: ap_proxy_canon_netloc が失敗した場合、エラーログを出力し HTTP_BAD_REQUEST を返します。
  • **パス正規化失敗**: ap_proxy_canonenc_ex が NULL を返した場合、HTTP_BAD_REQUEST を返します。

保守担当者向けメモ

  • 本関数は mod_proxy の正規化フックとして動作します。
  • SCGI_DEF_PORT は scgi.h で定義されている定数です。
  • proxy-scgi-pathinfo 環境変数の有無によって r->path_info の挙動が変わるため、設定変更時には注意が必要です。
