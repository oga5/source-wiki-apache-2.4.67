make_secure_socket

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:246

目的

NetWare のネイティブな WinSock 拡張を利用し、SSL/TLS 通信が可能なリスニングソケットを生成・設定する。

シグネチャまたは形状

static int make_secure_socket(apr_pool_t *pconf,
                              const struct sockaddr_in *server,
                              char* key, int mutual, server_rec *sconf)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • server: 接続先アドレス情報。
  • key: 証明書識別子。
  • mutual: 相互認証の有効フラグ。
  • sconf: サーバー設定レコード。
  • **出力**:
  • 成功時: 生成されたソケットディスクリプタ（int）。
  • 失敗時: -1。

前提条件と不変条件

  • NetWare 環境の WinSock スタックが利用可能であること。
  • key は有効な証明書情報を指していること。

アルゴリズムまたは振る舞い

1. WSASocket を呼び出し、SECURITY_PROTOCOL_SSL を指定して TCP ソケットを作成する。 2. mutual が偽の場合、WSAIoctl で SO_SSL_SET_FLAGS を使用し、SSL サーバーモードを有効化する。 3. WSAIoctl で SO_SSL_SET_SERVER を使用し、提供された証明書（key）をソケットに適用する。 4. mutual が真の場合、SO_SSL_AUTH_CLIENT フラグを設定し、クライアント認証を有効化する。 5. 最後に SO_TLS_UNCLEAN_SHUTDOWN フラグを適用し、ソケットを返す。 6. 各ステップでエラーが発生した場合は、ap_log_error で詳細を記録し、-1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_nw_ssl.c:787 (直接的な呼び出しを確認)。
  • **呼び出し先**: WSASocket, WSAIoctl, ap_log_error, apr_snprintf, inet_ntoa, ntohs, htonl, strlen, memset。

関連データ構造

  • struct sockaddr_in: アドレス情報。
  • WSAPROTOCOL_INFO: WinSock プロトコル情報。
  • struct sslserveropts: SSL サーバーオプション（証明書設定など）。

失敗モードと境界条件

  • WSASocket によるソケット生成失敗。
  • WSAIoctl によるフラグ設定またはサーバーオプション適用失敗。
  • いずれの失敗も INVALID_SOCKET または戻り値のチェックにより検知され、エラーログ出力後に終了する。

保守担当者向けメモ

  • 本関数は NetWare 固有の WSAIoctl インターフェースに強く依存している。
  • SO_TLS_UNCLEAN_SHUTDOWN は無条件に適用される。
  • メモリ管理は Apache の pool に依存しているため、明示的な解放は不要。
