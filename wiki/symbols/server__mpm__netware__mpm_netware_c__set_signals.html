set_signals

定義

  • 種別: function
  • 定義箇所: server/mpm/netware/mpm_netware.c:301

目的

NetWare MPM において、サーバーの終了や異常終了を制御するためのシグナルハンドラを登録する。

シグネチャまたは形状

static void set_signals(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として APR のシグナルハンドラ設定を更新）

前提条件と不変条件

  • apr_signal が利用可能であること。
  • sig_term 関数が定義されており、シグナル受信時の適切な終了処理を実装していること。

アルゴリズムまたは振る舞い

1. apr_signal を使用して SIGTERM シグナルを sig_term 関数にバインドする。 2. apr_signal を使用して SIGABRT シグナルを sig_term 関数にバインドする。 これにより、これらのシグナルを受信した際にサーバーが安全に終了処理へ移行するように設定する。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/netware/mpm_netware.c:896
  • 呼び出し先: apr_signal (APR ライブラリ関数), sig_term (同ファイル内の静的関数)

関連データ構造

  • sig_term: シグナルハンドラとして登録される関数。

失敗モードと境界条件

  • apr_signal が内部的にエラーを返した場合の挙動については、本関数の実装内では明示的なエラーハンドリングは行われていない。

保守担当者向けメモ

  • 本関数は NetWare 環境特有のシグナル処理を抽象化している。他の MPM（mpmt_os2 など）にも同名の関数が存在するが、実装はプラットフォーム依存であるため注意すること。
