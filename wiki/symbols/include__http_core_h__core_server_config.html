core_server_config

定義

  • 種別: struct
  • 定義箇所: include/http_core.h:694

目的

Apache HTTP Server のサーバー単位（グローバル）設定を保持する構造体です。ドキュメントルート、アクセス制御、リクエスト処理の制限、およびプロトコル関連の動作設定を格納します。

シグネチャまたは形状

typedef struct {
    char *gprof_dir;
    const char *ap_document_root;
    char *access_name;
    apr_array_header_t *sec_dir;
    apr_array_header_t *sec_url;
    int redirect_limit;
    int subreq_limit;
    const char *protocol;
    apr_table_t *accf_map;
    apr_array_header_t *error_log_format;
    apr_array_header_t *error_log_conn;
    apr_array_header_t *error_log_req;
    int trace_enable;
    int merge_trailers;
    apr_array_header_t *protocols;
    int protocols_honor_order;
    char http09_enable;
    char http_conformance;
    char http_methods;
    unsigned int merge_slashes;
    apr_size_t flush_max_threshold;
    apr_int32_t flush_max_pipelined;
    unsigned int strict_host_check;
#ifdef WIN32
    apr_array_header_t *unc_list;
#endif
} core_server_config;

入力と出力

  • **入力**: 設定ファイル（httpd.conf等）のディレクティブにより値が設定されます。
  • **出力**: ap_get_module_config(s->module_config, &core_module) を通じて、サーバーのライフサイクル全体で参照されます。

前提条件と不変条件

  • trace_enable、http09_enable、http_conformance、http_methods は、それぞれ定義された定数（AP_TRACE_* 等）の範囲内で使用される必要があります。
  • sec_dir および sec_url は apr_array_header_t として初期化されている必要があります。

アルゴリズムまたは振る舞い

  • サーバーの起動時に設定が読み込まれ、リクエスト処理中に core_module を介してアクセスされます。
  • リクエスト処理の再帰制限（redirect_limit, subreq_limit）や、HTTP プロトコルの適合性チェック（http_conformance）の基準値として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_filters.c、modules/http/http_protocol.c、modules/http2/h2_c2_filter.c、modules/http2/h2_headers.c 等の各モジュールが ap_get_module_config を介して本構造体を取得しています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • core_dir_config: ディレクトリ単位の設定。
  • core_request_config: リクエスト単位の一時的な状態。
  • apr_array_header_t, apr_table_t: 設定値の保持に使用される APR データ構造。

失敗モードと境界条件

  • redirect_limit や subreq_limit を超えた場合、リクエスト処理が中断される可能性があります。
  • http_conformance が AP_HTTP_CONFORMANCE_STRICT に設定されている場合、非準拠な HTTP リクエストが拒否される可能性があります。

保守担当者向けメモ

  • 本構造体は core_module の設定として管理されています。
  • Windows 環境では unc_list が追加されるため、プラットフォーム依存のコードを扱う際は #ifdef WIN32 を考慮してください。
  • 構造体のメンバー追加は、ap_mmn.h で定義されるモジュールバイナリ互換性（MMN）に影響を与える可能性があります。
