asis_handler

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_asis.c:31

目的

HTTP レスポンスヘッダーをファイル内に直接記述したファイルを、サーバーによる自動生成ヘッダーをバイパスしてクライアントへそのまま送信（Send-As-Is）する。

シグネチャまたは形状

static int asis_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r （Apache リクエスト構造体）
  • **出力**: int （Apache ステータスコード: OK, DECLINED, HTTP_NOT_FOUND, HTTP_FORBIDDEN, HTTP_INTERNAL_SERVER_ERROR, AP_FILTER_ERROR）

前提条件と不変条件

  • リクエストのハンドラ名が ASIS_MAGIC_TYPE (httpd/send-as-is) または "send-as-is" であること。
  • リクエストメソッドが M_GET であること。
  • 対象ファイルが存在し、サーバーが読み取り権限を持っていること。
  • ファイルの先頭が HTTP ヘッダー群、その後に空行という形式であること。

アルゴリズムまたは振る舞い

1. **検証**: ハンドラ名とメソッドを確認し、不一致の場合は DECLINED を返す。 2. **ファイルオープン**: 対象ファイルを読み込み専用で開く。 3. **ヘッダー解析**: ap_scan_script_header_err_ex を呼び出し、ファイル先頭のヘッダーを解析して r->headers_out 等に反映させる。 4. **リダイレクト処理**: Location ヘッダーが存在し、かつステータスがリダイレクトまたは HTTP_OK の場合、ファイルを閉じて ap_internal_redirect_handler を実行し、OK を返す。 5. **コンテンツ送信**: header_only リクエストでない場合、ヘッダー終了位置以降のファイル本体をバケットブリゲードに追加し、ap_pass_brigade で出力フィルターチェーンへ送信する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_asis.c:116 (ap_hook_handler により登録)
  • **呼び出し先**:
  • ap_log_rerror, apr_file_open, apr_file_close, apr_file_seek
  • ap_scan_script_header_err_ex
  • apr_table_get
  • ap_internal_redirect_handler
  • apr_brigade_create, apr_brigade_insert_file, apr_bucket_eos_create, APR_BRIGADE_INSERT_TAIL
  • ap_pass_brigade

関連データ構造

  • request_rec: リクエスト情報およびレスポンスヘッダーの保持。
  • apr_file_t: 対象ファイルのファイルハンドル。
  • apr_bucket_brigade: コンテンツ送信用のバケットブリゲード。

失敗モードと境界条件

  • **ファイル不在**: APR_NOFILE の場合、HTTP_NOT_FOUND を返す。
  • **アクセス権限不足**: apr_file_open 失敗時、HTTP_FORBIDDEN を返す。
  • **ヘッダー解析後の位置特定失敗**: apr_file_seek 失敗時、HTTP_INTERNAL_SERVER_ERROR を返す。
  • **送信失敗**: ap_pass_brigade 失敗時、AP_FILTER_ERROR を返す。
  • **リダイレクト**: 内部リダイレクト時は元のメソッドに関わらず GET として再処理される。

保守担当者向けメモ

  • ファイルの先頭が空行で終わっていない場合、ヘッダー解析が正しく行われない可能性がある。
  • 内部リダイレクトが発生した場合、元のリクエストメソッドに関わらず GET メソッドとして再処理される点に注意すること。
