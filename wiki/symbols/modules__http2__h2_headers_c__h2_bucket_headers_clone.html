h2_bucket_headers_clone

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_headers.c:111

目的

既存の H2HEADERS 型バケットを複製し、新しいメモリプールとバケットアロケータを使用してディープコピーを作成します。

シグネチャまたは形状

apr_bucket *h2_bucket_headers_clone(apr_bucket *b, apr_pool_t *pool, apr_bucket_alloc_t *list);

入力と出力

  • **入力**:
  • b: 複製元の apr_bucket。
  • pool: ヘッダー情報の複製に使用する apr_pool_t。
  • list: 新しいバケットの割り当てに使用する apr_bucket_alloc_t。
  • **出力**:
  • 複製された h2_headers 構造体を含む、新しく生成された apr_bucket へのポインタ。

前提条件と不変条件

  • 入力バケット b は H2HEADERS 型のデータを含んでいる必要があります。
  • 内部の h2_headers 構造体は h2_headers_clone を介して複製されるため、メモリ所有権は指定された pool に移転します。

アルゴリズムまたは振る舞い

1. 入力バケット b から h2_bucket_headers 構造体を取り出し、保持されている h2_headers への参照を取得します。 2. h2_headers_clone を呼び出し、指定された pool 上にヘッダー情報のディープコピーを作成します。 3. h2_bucket_headers_create を呼び出し、複製されたヘッダー情報と指定された list を用いて新しいバケットを生成・返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_bucket_beam.c:656
  • **呼び出し先**:
  • h2_headers_clone
  • h2_bucket_headers_create

関連データ構造

  • apr_bucket: Apache のバケット抽象化構造体。
  • h2_headers: HTTP/2 ヘッダー情報（ステータス、テーブル、ノート）を保持する構造体。
  • h2_bucket_headers: apr_bucket のデータ領域に格納されるラッパー構造体。

失敗モードと境界条件

  • メモリ不足時、apr_pool_t または apr_bucket_alloc_t の操作によりアロケーションエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は h2_bucket_beam におけるストリーム間転送時に使用されます。
  • h2_headers_clone が指定されたプールを使用するため、生存期間の管理には注意してください。
  • AP_HAS_RESPONSE_BUCKETS が定義されている環境では、このロジックは無効化されます。
