ap_strchr_c

定義

  • 種別: function
  • 定義箇所: server/util_debug.c:53

目的

標準の strchr 関数を const char * 型に対して安全に利用するためのラッパー関数です。

シグネチャまたは形状

const char *ap_strchr_c(const char *s, int c)

入力と出力

  • **入力**:
  • s: 検索対象のヌル終端文字列（const char *）。
  • c: 検索する文字（int）。
  • **出力**:
  • 文字列 s 内で最初に見つかった文字 c へのポインタを返します。文字が見つからない場合は NULL を返します。

前提条件と不変条件

  • 引数 s は有効なヌル終端文字列である必要があります。
  • 戻り値のポインタは、入力された文字列 s の一部を指すか、NULL となります。

アルゴリズムまたは振る舞い

標準ライブラリの strchr を呼び出し、その結果をそのまま返します。const 修飾された文字列を扱う際に、型安全性を維持する目的で提供されています。

呼び出し元と呼び出し先

  • **呼び出し先**: 標準ライブラリ関数 strchr。
  • **呼び出し元**: 以下のモジュール等から参照されています（直接的な呼び出しの証拠）。
  • modules/aaa/mod_auth_form.c
  • modules/aaa/mod_authn_anon.c
  • modules/aaa/mod_authn_socache.c
  • modules/aaa/mod_authz_host.c
  • modules/arch/netware/mod_netware.c
  • modules/arch/win32/mod_win32.c
  • modules/cache/cache_storage.c
  • modules/cache/cache_util.c

関連データ構造

  • 特になし。

失敗モードと境界条件

  • s が NULL の場合の挙動は標準の strchr に依存します。
  • 文字列内に c が存在しない場合、NULL を返します。

保守担当者向けメモ
