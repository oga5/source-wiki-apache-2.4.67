socache_mc_iterate

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_memcache.c:344

目的

Memcached バックエンドにおけるキャッシュエントリの反復処理（イテレーション）を提供するためのインターフェースです。

シグネチャまたは形状

apr_status_t socache_mc_iterate(ap_socache_instance_t *instance,
                                server_rec *s, void *userctx,
                                ap_socache_iterator_t *iterator,
                                apr_pool_t *pool);

入力と出力

  • **入力**:
  • instance: キャッシュインスタンスへのポインタ。
  • s: サーバーレコード。
  • userctx: 反復処理中に渡されるユーザー定義コンテキスト。
  • iterator: 各エントリに対して実行されるコールバック関数。
  • pool: メモリ割り当て用プール。
  • **出力**:
  • apr_status_t: 常に APR_ENOTIMPL を返します。

前提条件と不変条件

  • 本関数は ap_socache_provider_t インターフェースの一部として定義されていますが、Memcached 実装では機能が提供されていません。

アルゴリズムまたは振る舞い

  • 呼び出された際、いかなる処理も行わず、即座に APR_ENOTIMPL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_memcache.c:362 において、socache_mc プロバイダー構造体のメンバとして参照されています。
  • **呼び出し先**: なし。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスの管理構造体。
  • ap_socache_iterator_t: 反復処理用のコールバック関数型。

失敗モードと境界条件

  • 本関数は常に失敗（APR_ENOTIMPL）を返します。
