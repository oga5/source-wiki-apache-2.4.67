ap_proxy_http_register_hook

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ajp.c:864

目的

AJP (Apache JServ Protocol) プロキシモジュールを Apache HTTP Server のフックシステムに登録し、ajp:// スキームの処理に必要なハンドラとオプション関数を有効化します。

シグネチャまたは形状

static void ap_proxy_http_register_hook(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p (フック登録に使用されるメモリプール)
  • **出力**: なし (戻り値なし)

前提条件と不変条件

  • 本関数は Apache のモジュール初期化フェーズで呼び出されることを前提としています。
  • proxy_ajp_handler および proxy_ajp_canon が定義済みである必要があります。

アルゴリズムまたは振る舞い

1. proxy_hook_scheme_handler を呼び出し、proxy_ajp_handler を APR_HOOK_FIRST の優先度でスキームハンドラとして登録します。 2. proxy_hook_canon_handler を呼び出し、proxy_ajp_canon を APR_HOOK_FIRST の優先度で正規化ハンドラとして登録します。 3. APR_REGISTER_OPTIONAL_FN を使用して、ajp_handle_cping_cpong を他のモジュールから利用可能なオプション関数として登録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_ajp.c:878 (モジュール内での登録処理)
  • modules/proxy/mod_proxy_http.c:2142 (ツールによる参照の可能性あり)
  • **呼び出し先**:
  • proxy_hook_scheme_handler
  • proxy_hook_canon_handler
  • APR_REGISTER_OPTIONAL_FN

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • proxy_ajp_handler: 登録されるスキームハンドラ関数。
  • proxy_ajp_canon: 登録される正規化ハンドラ関数。
  • ajp_handle_cping_cpong: 登録されるオプション関数。

失敗モードと境界条件

  • 本関数自体は戻り値を返さないため、登録失敗時のエラーハンドリングは Apache のフック登録メカニズムに依存します。

保守担当者向けメモ

  • 本関数は mod_proxy_ajp のエントリーポイントを定義する重要な役割を担っています。フックの優先度が APR_HOOK_FIRST に設定されているため、他のプロキシモジュールよりも先に評価される点に注意してください。
