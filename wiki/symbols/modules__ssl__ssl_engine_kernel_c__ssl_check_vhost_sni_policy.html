ssl_check_vhost_sni_policy

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:108

目的

SNI（Server Name Indication）によって選択されたバーチャルホスト間でのポリシー整合性を検証し、接続の継続可否を判断する。

シグネチャまたは形状

static int ssl_check_vhost_sni_policy(SSLSrvConfigRec *sc1, SSLSrvConfigRec *sc2)

入力と出力

  • **入力**:
  • sc1: 現在の接続に関連付けられたサーバー設定構造体。
  • sc2: SNI によって特定されたターゲットのサーバー設定構造体。
  • **出力**:
  • 1: ポリシー検証に合格（接続継続を許可）。
  • 0: ポリシー検証に失敗（接続継続を拒否）。

前提条件と不変条件

  • sc1 および sc2 は有効な SSLSrvConfigRec 構造体であること。
  • authonly または secure ポリシーが選択されている場合、両方の設定構造体において sni_policy_hash が初期化されていること（AP_DEBUG_ASSERT により検証）。

アルゴリズムまたは振る舞い

1. sc1 から SNI ポリシー（snivh_policy）を取得する。 2. **MODSSL_SNIVH_INSECURE**: 常に 1 を返す。 3. **MODSSL_SNIVH_STRICT**: sc1 と sc2 が同一インスタンスである場合のみ 1 を返し、それ以外は 0 を返す。 4. **その他（authonly/secure）**: 両方の設定構造体が持つ sni_policy_hash 文字列を比較し、一致すれば 1、不一致であれば 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_kernel.c:237 にて、SNI による vhost 切り替え時の検証として呼び出される。
  • **呼び出し先**: strcmp（標準ライブラリ関数）および AP_DEBUG_ASSERT マクロ。

関連データ構造

  • SSLSrvConfigRec: SSL サーバー設定を保持する構造体。sni_policy_hash および mc->snivh_policy を含む。

失敗モードと境界条件

  • sni_policy_hash が設定されていない状態で authonly や secure ポリシーが適用されている場合、デバッグビルドではアサーション失敗により停止する。
  • sc1 または sc2 が NULL の場合の挙動については、本ソースコード内では明示的なガードは存在しない。

保守担当者向けメモ

  • 本関数は TLS ハンドシェイク中の SNI 切り替えに伴うセキュリティポリシーの強制を担う。
  • sni_policy_hash は、設定の互換性を判定するための識別子として機能している。変更を加える際は、設定構造体側のハッシュ生成ロジックとの整合性に注意すること。
