modssl_smart_shutdown

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ssl.c:119

目的

SSL 接続の終了時に、OpenSSL の SSL_shutdown を安全かつ確実に実行するための再試行ロジックを提供します。不完全なシャットダウンによるプロセスハングを防ぐことを目的としています。

シグネチャまたは形状

int modssl_smart_shutdown(SSL *ssl)

入力と出力

  • **入力**: SSL *ssl - シャットダウン対象の SSL オブジェクト。
  • **出力**: int - 最終的な SSL_shutdown の戻り値（成功時は 1、失敗時は 0 または負の値）。

前提条件と不変条件

  • SSL オブジェクトは有効である必要があります。
  • SSL_shutdown が内部的にステートマシンとして動作するため、複数回の呼び出しを許容する設計となっています。

アルゴリズムまたは振る舞い

1. SSL_get_shutdown を確認し、SSL_SENT_SHUTDOWN が設定されていない場合にフラッシュフラグを立てます。 2. 最大 4 回のループ内で SSL_shutdown(ssl) を呼び出します。 3. SSL_SENT_SHUTDOWN が設定されており、かつフラッシュフラグが有効な場合、BIO_flush を実行してソケットへの書き出しを強制します。BIO_flush が失敗した場合は即座に -1 を返します。 4. SSL_shutdown が 0 以外（成功またはエラー）を返した時点でループを終了し、その値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_io.c:1144
  • **呼び出し先**:
  • SSL_shutdown (OpenSSL API)
  • SSL_get_shutdown (OpenSSL API)
  • SSL_get_wbio (OpenSSL API)
  • BIO_flush (OpenSSL API)

関連データ構造

  • SSL (OpenSSL 構造体)

失敗モードと境界条件

  • **最大試行回数**: 4 回を超えてもシャットダウンが完了しない場合は、最後に実行した SSL_shutdown の結果を返します。
  • **BIO フラッシュ失敗**: BIO_flush が 0 以下を返した場合、関数は -1 を返し、シャットダウン処理を中断します。

保守担当者向けメモ

  • 本関数は、クライアントがソケット通信において不正な挙動を示した場合のプロセスハングを回避するために、試行回数を制限しています。
  • OpenSSL のバージョンアップに伴う API 変更の影響を受ける可能性があるため、SSL_shutdown の挙動変更には注意してください。
