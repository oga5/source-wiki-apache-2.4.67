cha_type

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme_authz.c:560

目的

ACME プロトコルにおけるチャレンジタイプ（HTTP-01, TLS-ALPN-01, DNS-01 等）ごとのセットアップおよびクリーンアップ処理を定義するためのディスパッチテーブル用構造体です。

シグネチャまたは形状

typedef struct {
    const char *name;
    cha_setup *setup;
    cha_teardown *teardown;
} cha_type;

入力と出力

  • name: チャレンジタイプを識別する文字列（例: "http-01"）。
  • setup: チャレンジ解決のための環境構築を行う関数へのポインタ。
  • teardown: チャレンジ完了後またはエラー発生時にリソースを解放する関数へのポインタ。

前提条件と不変条件

  • 本構造体は、CHA_TYPES 静的配列の要素として定義されることを前提としています。
  • setup および teardown 関数は、ACME 認証プロセスにおける一連のライフサイクル管理に準拠している必要があります。

アルゴリズムまたは振る舞い

  • チャレンジの解決時に、md_acme_authz_respond が CHA_TYPES 配列を走査し、サーバーが提示したチャレンジタイプと一致する cha_type を選択します。
  • 選択された cha_type の setup 関数が呼び出され、トークンの配置や外部コマンド実行などの環境構築が行われます。
  • 処理終了後、md_acme_authz_teardown を介して対応する teardown 関数が呼び出され、クリーンアップが実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_acme_authz.c 内の CHA_TYPES 静的配列定義にて参照されています。
  • 呼び出し先: 本構造体に含まれる関数ポインタ（setup, teardown）は、各チャレンジタイプの実装関数を指します。

関連データ構造

  • CHA_TYPES: cha_type 型の静的配列。サポートされる全チャレンジタイプの定義を保持します。

失敗モードと境界条件

  • 構造体内の関数ポインタが NULL の場合、呼び出し時に未定義の動作を引き起こす可能性があります。
  • 新しいチャレンジタイプを追加する際は、本構造体の定義に従い、setup および teardown の両方を適切に実装する必要があります。

保守担当者向けメモ

  • 本構造体はディスパッチ方式を採用しており、将来的な ACME プロトコルの拡張（新しいチャレンジタイプの追加）を容易にする設計となっています。
  • setup および teardown 関数内で使用されるメモリは、呼び出し元から提供される apr_pool_t に紐付いている必要があります。
