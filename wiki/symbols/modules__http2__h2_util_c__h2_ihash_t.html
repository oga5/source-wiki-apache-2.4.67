h2_ihash_t

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.c:215

目的

整数 ID をキーとして保持するハッシュテーブル構造体です。構造体内の特定の整数メンバーをキーとして抽出し、管理するために使用されます。

シグネチャまたは形状

struct h2_ihash_t {
    apr_hash_t *hash;
    size_t ioff;
};

入力と出力

  • hash: apr_hash_t へのポインタ。実際のデータ格納に使用されます。
  • ioff: 構造体内のキーとなる整数メンバーへのオフセット値。

前提条件と不変条件

  • 本構造体はスレッドセーフではありません。呼び出し側で適切なロックを確保する必要があります。
  • ioff は、ハッシュテーブルに格納される構造体内の整数メンバーのオフセットを指している必要があります。

アルゴリズムまたは振る舞い

  • h2_ihash_create を介して初期化されます。
  • 構造体内のオフセット (offsetof) を利用して、格納されたオブジェクトから自動的にキーを抽出します。
  • h2_ihash_add, h2_ihash_get, h2_ihash_remove などの関数を通じて、整数 ID をキーとしたデータの追加・取得・削除を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_mplx.h: h2_mplx 構造体内でストリーム管理用（streams, shold）として参照されています。
  • modules/http2/h2_session.h: セッション管理に関連して参照されています。
  • **呼び出し先**:
  • apr_hash_t (APR ライブラリ): 内部的なハッシュ管理に使用されます。

関連データ構造

  • h2_mplx: 本構造体を使用してアクティブなストリームや処理中のストリームを管理します。

失敗モードと境界条件

  • メモリ不足時: h2_ihash_create 内での apr_pcalloc 失敗により、NULL が返される可能性があります。
  • キーの重複: h2_ihash_add で既存の ID を追加した場合の挙動は、apr_hash_set の仕様に依存します。

保守担当者向けメモ

  • 本構造体はメモリレイアウトに依存するため、格納対象の構造体定義を変更する際は ioff の整合性に注意してください。
  • スレッドセーフではないため、マルチスレッド環境下での利用時には外部での排他制御が必須です。
