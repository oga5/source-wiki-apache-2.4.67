reset

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_bytraffic.c:70

目的

バランサーに紐付く全ワーカーの統計情報（転送量および受信量）をゼロにリセットする。

シグネチャまたは形状

static apr_status_t reset(proxy_balancer *balancer, server_rec *s)

入力と出力

  • 入力:
  • balancer: 統計をリセットする対象の proxy_balancer 構造体。
  • s: サーバーレコード（本関数内では未使用）。
  • 出力:
  • APR_SUCCESS: 常に成功を返す。

前提条件と不変条件

  • balancer->workers が有効なワーカーリストを保持していること。
  • 本関数は、呼び出し元においてミューテックス等で保護されている前提で実装されている。

アルゴリズムまたは振る舞い

1. balancer->workers->elts からワーカー配列を取得する。 2. ワーカーの数だけループ処理を行う。 3. 各ワーカーの共有メモリ上の統計カウンタ s->transferred（転送量）および s->read（受信量）を 0 に設定する。 4. APR_SUCCESS を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内に直接的な呼び出し元は記述されていない。
  • 呼び出し先: なし。

関連データ構造

  • proxy_balancer: ロードバランサーの構成とワーカーリストを保持する構造体。
  • proxy_worker: 各バックエンドサーバーのステータス（s）を保持する構造体。

失敗モードと境界条件

  • ワーカーリストが空の場合、ループは実行されず即座に APR_SUCCESS を返す。

保守担当者向けメモ

  • 本関数は mod_proxy_balancer の管理者が統計情報をリセットする際に呼び出される。
  • 共有メモリ上の値を直接操作するため、呼び出し元での排他制御が必須である。
