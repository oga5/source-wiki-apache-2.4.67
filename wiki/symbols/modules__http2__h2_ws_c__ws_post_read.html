ws_post_read

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_ws.c:325

目的

HTTP/2 接続において、WebSocket のアップグレード要求がサーバー設定で許可されているかを検証し、許可されていない場合にリクエストを拒否する。

シグネチャまたは形状

static int ws_post_read(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエストレコード)
  • **出力**:
  • HTTP_NOT_IMPLEMENTED: WebSocket が設定で無効化されている場合。
  • DECLINED: それ以外（WebSocket が有効、または対象外の接続）。

前提条件と不変条件

  • r->connection->master が存在する場合、接続コンテキスト (h2_conn_ctx_t) が取得可能であること。
  • H2_CONF_WEBSOCKETS 設定値がサーバー設定から正しく取得できること。

アルゴリズムまたは振る舞い

1. 現在の接続がマスター接続（HTTP/2 セッション）であるかを確認する。 2. 接続コンテキストから is_upgrade フラグを確認し、WebSocket へのアップグレード要求であるかを判定する。 3. H2_CONF_WEBSOCKETS 設定をチェックし、WebSocket が無効化されている場合は HTTP_NOT_IMPLEMENTED を返す。 4. 上記条件に該当しない場合は DECLINED を返し、後続の処理へ委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_read_request により、Apache の post_read_request フックとして登録されている (modules/http2/h2_ws.c:340)。
  • **呼び出し先**:
  • h2_conn_ctx_get: 接続コンテキストの取得。
  • h2_config_sgeti: サーバー設定値の取得。

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • h2_conn_ctx_t: HTTP/2 接続ごとのメタデータ（is_upgrade フラグを保持）。

失敗モードと境界条件

  • conn_ctx が取得できない場合、または is_upgrade が偽の場合は、WebSocket 関連のチェックをスキップして DECLINED を返す。
  • H2_CONF_WEBSOCKETS が有効な場合、この関数は常に DECLINED を返す。

保守担当者向けメモ

  • 本関数は post_read_request フックの一部として実行されるため、リクエスト処理の極めて早い段階で介入します。
  • H2_USE_WEBSOCKETS マクロが定義されていない環境では、この関数は実質的に NOP として動作します。
