recycled_pool

定義

  • 種別: struct
  • 定義箇所: server/mpm_fdqueue.c:25

目的

メモリ割り当てのオーバーヘッドを削減するため、apr_pool_t を再利用可能なリストとして保持するためのノード構造体です。

シグネチャまたは形状

struct recycled_pool
{
    apr_pool_t *pool;
    struct recycled_pool *next;
};

入力と出力

  • 入力: なし（構造体定義のため）
  • 出力: なし

前提条件と不変条件

  • pool メンバは有効な apr_pool_t へのポインタ、または NULL を保持します。
  • next メンバはリスト内の次の recycled_pool 構造体、またはリストの終端を示す NULL を保持します。

アルゴリズムまたは振る舞い

  • 本構造体は、fd_queue_info_t 内の recycled_pools リストを構成する要素として使用されます。
  • ap_queue_info_push_pool および ap_queue_info_pop_pool 関数を通じて、プールのアトミックな追加・取り出し操作が行われます。
  • リストの先頭要素を操作することで、LIFO（後入れ先出し）方式でメモリプールの再利用を実現します。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm_fdqueue.c 内の ap_queue_info_push_pool および ap_queue_info_pop_pool 関数。
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • apr_pool_t: 再利用対象となるメモリプール。
  • fd_queue_info_t: recycled_pools リストのヘッドを保持する管理構造体。

失敗モードと境界条件

  • リストが空の状態で ap_queue_info_pop_pool が呼び出された場合、recycled_pool は NULL を返します。
  • メモリ不足時には、リストへの追加や新規ノードの確保が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は server/mpm_fdqueue.c 内のメモリ再利用メカニズムの基盤です。
  • recycled_pools リストの操作はアトミックに行われる必要があるため、リストのポインタを直接操作する際は apr_atomic 操作との整合性に注意してください。
