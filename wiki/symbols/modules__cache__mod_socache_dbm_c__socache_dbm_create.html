socache_dbm_create

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:79

目的

DBM ベースの共有オブジェクトキャッシュ（socache）インスタンスを初期化し、メモリを確保する。

シグネチャまたは形状

static const char *socache_dbm_create(ap_socache_instance_t **context,
                                      const char *arg,
                                      apr_pool_t *tmp, apr_pool_t *p)

入力と出力

  • **入力**:
  • context: インスタンス構造体へのポインタを格納する出力用引数。
  • arg: キャッシュファイルのパス（設定値）。
  • tmp: 一時的なメモリ割り当て用プール。
  • p: インスタンスの生存期間を管理するプール。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ（const char *）を返す。

前提条件と不変条件

  • p はインスタンスのライフサイクルを管理する有効なプールであること。
  • arg が指定されている場合、ap_server_root_relative によりサーバールートからの相対パスとして解決される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して ap_socache_instance_t 構造体を確保し、context に代入する。 2. arg が空でない場合、ap_server_root_relative を用いてキャッシュファイルの絶対パスを生成する。パスの解決に失敗した場合はエラーメッセージを返す。 3. インスタンス専用のサブプールを apr_pool_create で作成し、デバッグ用に socache_dbm_instance というタグを付与する。 4. 正常終了時は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dbm.c:743 にて参照されている（直接的な呼び出し関係の詳細はソース抜粋からは不明）。
  • **呼び出し先**: apr_pcalloc, ap_server_root_relative, apr_psprintf (エラー時), apr_pool_create, apr_pool_tag。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスのコンテキストを保持する構造体。

失敗モードと境界条件

  • ap_server_root_relative がパスを解決できない場合、Invalid cache file path というエラーメッセージを返し、インスタンス作成を中断する。

保守担当者向けメモ

  • 本関数はインスタンスのメモリ確保とパス設定のみを行う。DBM ファイルの実際のオープンや権限設定は、後続の socache_dbm_init で行われる。
  • ctx->pool はインスタンスの生存期間を管理するために作成されており、メモリ管理の境界として機能する。
