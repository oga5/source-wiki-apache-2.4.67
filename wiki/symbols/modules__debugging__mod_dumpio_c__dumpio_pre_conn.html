dumpio_pre_conn

定義

  • 種別: function
  • 定義箇所: modules/debugging/mod_dumpio.c:180

目的

接続開始時に、設定に基づいて入出力フィルタ（DUMPIO_IN および DUMPIO_OUT）をフィルタチェーンへ動的に挿入する。

シグネチャまたは形状

static int dumpio_pre_conn(conn_rec *c, void *csd)

入力と出力

  • **入力**:
  • c: 現在の接続を表す conn_rec 構造体。
  • csd: 接続ソケット記述子（本関数内では未使用）。
  • **出力**:
  • OK: フィルタの挿入が完了した、または挿入の必要がない場合。
  • DECLINED: ログレベルが APLOG_TRACE7 未満であり、処理をスキップする場合。

前提条件と不変条件

  • ログレベルが APLOG_TRACE7 以上であること。
  • c->base_server->module_config から取得した dumpio_conf_t 設定構造体が有効であること。

アルゴリズムまたは振る舞い

1. APLOGctrace7(c) を確認し、ログレベルが不十分な場合は DECLINED を返して終了する。 2. ap_get_module_config を使用して、サーバー設定から dumpio_conf_t を取得する。 3. 設定構造体の enable_input が真であれば、ap_add_input_filter を呼び出し DUMPIO_IN フィルタを登録する。 4. 設定構造体の enable_output が真であれば、ap_add_output_filter を呼び出し DUMPIO_OUT フィルタを登録する。 5. OK を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_pre_connection を介して Apache コアから呼び出される（modules/debugging/mod_dumpio.c:210）。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_add_input_filter
  • ap_add_output_filter

関連データ構造

  • conn_rec: 接続情報を保持するコア構造体。
  • dumpio_conf_t: mod_dumpio の入出力有効状態を管理する設定構造体。

失敗モードと境界条件

  • ログレベルが APLOG_TRACE7 未満の場合、フィルタは挿入されず、デバッグ出力は行われない。
  • 設定構造体が取得できない場合、または enable_input/enable_output が偽の場合は、対応するフィルタは追加されない。

保守担当者向けメモ

  • 本関数は接続確立の初期段階で実行されるため、フィルタの挿入順序は AP_FTYPE_CONNECTION + 3 に固定される。
  • フィルタの追加は dumpio_conf_t の状態に依存するため、設定変更が正しく反映されているか確認が必要な場合は dumpio_create_sconfig の初期化ロジックを参照すること。
