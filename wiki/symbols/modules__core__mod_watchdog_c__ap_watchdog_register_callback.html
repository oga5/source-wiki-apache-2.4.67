ap_watchdog_register_callback

定義

  • 種別: function
  • 定義箇所: modules/core/mod_watchdog.c:346

目的

指定された ap_watchdog_t インスタンスに対し、定期実行されるコールバック関数を登録します。

シグネチャまたは形状

apr_status_t ap_watchdog_register_callback(ap_watchdog_t *w,
                                           apr_interval_time_t interval,
                                           const void *data,
                                           ap_watchdog_callback_fn_t *callback);

入力と出力

  • **入力**:
  • w: 登録先となる ap_watchdog_t インスタンス。
  • interval: コールバックを実行する間隔（マイクロ秒単位）。
  • data: コールバック関数に渡されるコンテキストデータ。
  • callback: 実行対象の関数ポインタ。
  • **出力**:
  • APR_SUCCESS: 登録成功。
  • APR_EEXIST: 同一の data および callback の組み合わせが既に登録されている場合。

前提条件と不変条件

  • w->pool は、コールバックリストのメモリ確保に使用されます。
  • 同一の data と callback を持つコールバックの重複登録は禁止されています。

アルゴリズムまたは振る舞い

1. 既存のコールバックリストを走査し、引数 data および callback が一致するエントリが存在するか確認します。一致する場合は APR_EEXIST を返します。 2. 新規登録の場合、w->pool から watchdog_list_t 構造体分のメモリを確保します。 3. 構造体に data、callback、interval を設定し、step を 0、status を APR_EINIT に初期化します。 4. 既存のリストの先頭に新しいエントリを挿入し、w->active カウンタをインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cluster/mod_heartmonitor.c
  • modules/md/mod_md_drive.c
  • modules/md/mod_md_ocsp.c
  • modules/proxy/mod_proxy_hcheck.c
  • 上記モジュールは APR_RETRIEVE_OPTIONAL_FN を介して本関数を取得しています。
  • **呼び出し先**:
  • apr_palloc (APRライブラリ関数)

関連データ構造

  • ap_watchdog_t: 監視インスタンスを保持する構造体。
  • watchdog_list_t: コールバック関数とその実行状態を管理するリンクリスト構造体。

失敗モードと境界条件

  • APR_EEXIST: 重複登録時に発生します。
  • メモリ不足時: apr_palloc が失敗した場合、呼び出し元へ制御が戻る前に異常終了する可能性があります。

保守担当者向けメモ

  • 本関数は APR_DECLARE_OPTIONAL_FN として公開されており、他モジュールから動的に取得可能です。
  • コールバックの実行タイミングは、wd_worker 内のループ処理によって制御されます。登録後のコールバックの実行順序は、リストの先頭に追加されるため、登録順とは逆になります。
