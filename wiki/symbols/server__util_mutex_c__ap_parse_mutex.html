ap_parse_mutex

定義

  • 種別: function
  • 定義箇所: server/util_mutex.c:49

目的

文字列形式のミューテックス設定（メカニズム:ファイルパス）を解析し、APR が利用可能なロックメカニズム（apr_lockmech_e）と、必要に応じてファイルパスへ変換する。

シグネチャまたは形状

apr_status_t ap_parse_mutex(const char *arg, apr_pool_t *pool,
                            apr_lockmech_e *mutexmech,
                            const char **mutexfile)

入力と出力

  • **入力**:
  • arg: "メカニズム[:ファイルパス]" 形式の文字列。
  • pool: メモリ割り当て用プール。
  • **出力**:
  • mutexmech: 解決された apr_lockmech_e 値。
  • mutexfile: ファイルベースのロックの場合、ap_server_root_relative で解決された絶対パス。
  • 戻り値: 成功時は APR_SUCCESS。無効な指定は APR_ENOTIMPL、none 指定は APR_ENOLOCK、パス解決失敗は APR_BADARG を返す。

前提条件と不変条件

  • arg は apr_pstrdup により内部で複製されるため、呼び出し元で解放可能。
  • posixsem を除くファイルベースのロックメカニズムが指定された場合、mutexfile は ap_server_root_relative を通じてサーバーのルートディレクトリからの相対パスとして解決される。

アルゴリズムまたは振る舞い

1. arg を : で分割し、メカニズム名とファイルパス部分を分離する。 2. メカニズム名（none, default, fcntl, flock, posixsem, sysvsem, pthread 等）を大文字小文字を区別せずに判定し、対応する apr_lockmech_e を設定する。 3. posixsem の場合、ファイルパスが指定されていればそれを mutexfile に格納する。 4. その他のファイルベースのメカニズムでファイルパスが指定されている場合、ap_server_root_relative を使用して絶対パスを生成する。 5. none が指定された場合は APR_ENOLOCK を即座に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_mutex.c:189（ap_set_mutex 内で呼び出されていることが確認可能）。
  • **呼び出し先**: apr_pstrdup, strchr, strcasecmp, ap_server_root_relative 等の APR/Apache ユーティリティ関数。

関連データ構造

  • apr_lockmech_e: APR が提供するロックメカニズムの列挙型。

失敗モードと境界条件

  • arg に含まれるファイルパスが解決できない場合、APR_BADARG を返す。
  • サポートされていないメカニズムが指定された場合、APR_ENOTIMPL を返す。
  • none または no が指定された場合、APR_ENOLOCK を返す。

保守担当者向けメモ

  • posixsem および sysvsem は sem というエイリアスを共有している。
  • fcntl および flock は file というエイリアスを共有している。
  • コンパイル時の APR の機能フラグ（APR_HAS_..._SERIALIZE）に依存して利用可能なメカニズムが決定される。
