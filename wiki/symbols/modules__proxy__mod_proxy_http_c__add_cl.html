add_cl

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_http.c:261

目的

HTTPリクエストヘッダーのブリゲード（header_brigade）に対し、Content-Length ヘッダー行を追加します。

シグネチャまたは形状

static void add_cl(apr_pool_t *p,
                   apr_bucket_alloc_t *bucket_alloc,
                   apr_bucket_brigade *header_brigade,
                   const char *cl_val)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • bucket_alloc: バケット割り当て用アロケータ。
  • header_brigade: ヘッダーを格納するバケットブリゲード。
  • cl_val: Content-Length の値として設定する文字列。
  • **出力**: なし（副作用として header_brigade にバケットが挿入される）。

前提条件と不変条件

  • cl_val は有効な数値文字列である必要があります。
  • header_brigade は初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. apr_pstrcat を使用して "Content-Length: ", cl_val, CRLF を連結した文字列を生成します。 2. ap_xlate_proto_to_ascii を呼び出し、生成した文字列をプロトコル標準の ASCII 形式に変換します。 3. apr_bucket_pool_create を使用して、変換後の文字列を保持するプールバケットを作成します。 4. APR_BRIGADE_INSERT_TAIL マクロを使用して、作成したバケットを header_brigade の末尾に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_http.c:645
  • modules/proxy/mod_proxy_http.c:659
  • **呼び出し先**:
  • apr_pstrcat (APR)
  • ap_xlate_proto_to_ascii (Apache HTTPD)
  • apr_bucket_pool_create (APR)
  • APR_BRIGADE_INSERT_TAIL (APR)

関連データ構造

  • apr_bucket_brigade: HTTPヘッダーを保持するバケットのリスト。
  • apr_bucket: ヘッダー文字列を保持するデータコンテナ。

失敗モードと境界条件

  • メモリ不足が発生した場合、apr_pstrcat や apr_bucket_pool_create 内でアロケータにより例外またはエラー処理がトリガーされる可能性があります。

保守担当者向けメモ

  • 本関数は Content-Length ヘッダーの生成と挿入のみを担います。ヘッダーの重複チェックや既存ヘッダーの削除は行わないため、呼び出し側で適切なヘッダー管理を行う必要があります。
