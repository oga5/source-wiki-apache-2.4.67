proxy_connect_canon

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_connect.c:135

目的

CONNECT メソッドのリクエストに対する URL の正規化および検証を行います。

シグネチャまたは形状

static int proxy_connect_canon(request_rec *r, char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • url: 正規化対象の URL 文字列。
  • **出力**:
  • OK: リクエストが CONNECT メソッドであり、処理を継続する場合。
  • DECLINED: リクエストが CONNECT メソッドではない場合。

前提条件と不変条件

  • r->method_number を参照し、対象が M_CONNECT であることを前提とします。

アルゴリズムまたは振る舞い

1. リクエストメソッドが M_CONNECT であるかを確認します。一致しない場合は DECLINED を返して終了します。 2. CONNECT メソッドである場合、トレースログ（APLOG_TRACE1）に URL を出力します。 3. OK を返し、正規化処理が完了したことを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフックシステム経由で呼び出されます。
  • **呼び出し先**:
  • ap_log_rerror: ログ出力のために呼び出されます。
  • **補足**: modules/proxy/mod_proxy_connect.c:386 において、proxy_hook_canon_handler を通じてフックとして登録されています。

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • M_CONNECT 以外のメソッドが渡された場合、本関数は何も行わず DECLINED を返します。

保守担当者向けメモ

  • 本関数は mod_proxy の canon_handler フックとして機能します。
  • 現在の実装では URL の書き換えや詳細な検証は行わず、メソッドの確認とログ出力のみを行っています。
