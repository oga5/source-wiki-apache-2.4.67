dav_generic_alloc_lock

定義

  • 種別: function
  • 定義箇所: modules/dav/lock/locks.c:237

目的

リクエストのメモリプールを使用して、新しい dav_lock 構造体を割り当ておよび初期化します。

シグネチャまたは形状

static dav_lock *dav_generic_alloc_lock(dav_lockdb *lockdb, apr_datum_t key, const dav_locktoken *locktoken)

入力と出力

  • **入力**:
  • lockdb: ロックデータベースのコンテキスト。
  • key: DBM 内のレコードを特定するためのキー。
  • locktoken: 既存のロックトークン（新規作成時は NULL を指定可能）。
  • **出力**:
  • 初期化済みの dav_lock 構造体へのポインタ。

前提条件と不変条件

  • lockdb->info->pool が有効なメモリプールとして存在していること。
  • 戻り値の pub.rectype は常に DAV_LOCKREC_DIRECT に設定されます。
  • locktoken が NULL の場合、内部で新しい UUID が生成されます。

アルゴリズムまたは振る舞い

1. lockdb->info->pool から dav_lock_combined 構造体分のメモリを確保します。 2. 構造体の pub.rectype を DAV_LOCKREC_DIRECT に設定し、内部ポインタをリンクします。 3. 引数 key を内部構造体に保存します。 4. locktoken が提供されている場合はそれを採用し、NULL の場合は apr_uuid_get を用いて新しい UUID を生成して割り当てます。 5. 最終的に dav_lock 型の公開インターフェース部分（pub）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/lock/locks.c 内の以下の行から呼び出されています。
  • 802, 840, 856, 909, 923, 1158, 1193 行目。
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_uuid_get (APR)

関連データ構造

  • dav_lock_combined: ロックの公開情報と内部情報を保持する統合構造体。
  • dav_lock: mod_dav が使用するロックの公開インターフェース。
  • apr_datum_t: DBM のキー/値を表す APR 型。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性があります。
  • lockdb が初期化されていない場合、lockdb->info へのアクセスによりセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数は直接ロック（DAV_LOCKREC_DIRECT）の生成に特化しています。間接ロックの生成には使用されません。
  • dav_lock_combined 構造体のメモリレイアウトに依存しているため、構造体定義を変更する際は本関数の初期化ロジックも併せて確認してください。
