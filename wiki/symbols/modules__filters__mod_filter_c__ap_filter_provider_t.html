ap_filter_provider_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_filter.c:38 および util_filter.h:200

目的

Apache HTTP Server における「スマートフィルタ」のプロバイダを管理するための連結リスト構造体です。特定のフィルタ名に対して、条件式やコンテンツタイプに基づいた複数の実装（プロバイダ）を紐付け、実行時に適切なものを選択可能にします。

シグネチャまたは形状

struct ap_filter_provider_t {
    ap_expr_info_t *expr;
    const char **types;
    ap_filter_rec_t *frec;
    ap_filter_provider_t *next;
};

入力と出力

  • **入力**:
  • expr: フィルタを適用するための条件式（ap_expr_info_t）。
  • types: 適用対象となるコンテンツタイプのリスト。
  • frec: このプロバイダが実装するフィルタのレコード（ap_filter_rec_t）。
  • **出力**:
  • next: リスト内の次のプロバイダへのポインタ。

前提条件と不変条件

  • next ポインタにより、単方向連結リストを形成します。
  • プロバイダの登録は設定フェーズ（cmd->pool を使用）で行われます。

アルゴリズムまたは振る舞い

  • mod_filter のディスパッチャ（filter_lookup）は、この構造体のリストを走査し、expr の評価結果または types の一致を確認することで、リクエストに最適なプロバイダを特定します。
  • 実行時には、マッチしたプロバイダの frec が持つ関数ポインタを通じて実際のフィルタ処理が呼び出されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_filter.c 内の filter_lookup や設定関数（add_filter 等）がこの構造体を操作・参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。実際の処理は frec が指すフィルタレコード内の関数ポインタへ委譲されます。

関連データ構造

  • ap_expr_info_t: 条件判定に使用される式評価エンジン。
  • ap_filter_rec_t: フィルタの定義情報。
  • mod_filter_cfg: 本構造体を保持するモジュール設定構造体。

失敗モードと境界条件

  • リストの末尾は NULL で終端される必要があります。
  • expr が NULL の場合、条件式による評価はスキップされます。
  • 複数のプロバイダが条件に合致する場合、リストの先頭に近いものが優先される実装となっています。

保守担当者向けメモ

  • 本構造体は mod_filter の中核ですが、util_filter.h にも定義が存在するため、変更時は両方の整合性に注意してください。
  • メモリ割り当ては設定時に cmd->pool を使用して行われるため、リクエスト終了後のライフサイクル管理は自動的に行われます。
