register_hooks

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_netware.c:180

目的

NetWare プラットフォーム用モジュールの初期化プロセスにおいて、CGI 実行用のオプション関数を登録し、設定フェーズのフックを構成します。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p (メモリ割り当て用のプール)
  • **出力**: なし (void)

前提条件と不変条件

  • 本関数は Apache のモジュール読み込み時に呼び出されることを前提としています。
  • NETWARE 定義環境下でのみコンパイルおよび実行されます。

アルゴリズムまたは振る舞い

1. APR_REGISTER_OPTIONAL_FN(ap_cgi_build_command) を呼び出し、CGI コマンド構築関数を他のモジュール（mod_cgi 等）から利用可能なオプション関数として公開します。 2. ap_hook_pre_config を使用して、netware_pre_config 関数を APR_HOOK_FIRST（最優先）のタイミングで実行するように登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール読み込みメカニズム（直接の呼び出し元はソースコード上では明示されていませんが、Apache の標準的なモジュールフックシステムにより起動されます）。
  • **呼び出し先**:
  • APR_REGISTER_OPTIONAL_FN (マクロ)
  • ap_hook_pre_config (API)
  • netware_pre_config (関数)

関連データ構造

  • apr_pool_t: リソース管理用プール。

失敗モードと境界条件

  • フック登録の失敗に関する明示的なエラーハンドリングは本関数内には含まれていません。

保守担当者向けメモ

  • 本関数は mod_netware のライフサイクル管理の起点です。
  • ap_cgi_build_command の公開は、mod_cgi との連携において不可欠です。
  • APR_HOOK_FIRST を指定しているため、他の設定フックよりも先に netware_pre_config が実行される点に注意してください。
