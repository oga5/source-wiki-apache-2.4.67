find_allowdeny

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_access_compat.c:242

目的

リクエストに関連付けられたアクセス制御ルール（Allow または Deny）の配列を走査し、現在のリクエスト環境が指定された条件に一致するかを判定する。

シグネチャまたは形状

static int find_allowdeny(request_rec *r, apr_array_header_t *a, int method)

入力と出力

  • **入力**:
  • r: リクエスト構造体。
  • a: 評価対象の allowdeny 構造体配列。
  • method: 現在のリクエストメソッド（ビットマスク生成に使用）。
  • **出力**:
  • int: 条件に一致した場合は 1、一致しない場合は 0 を返す。

前提条件と不変条件

  • a は allowdeny 構造体の配列であること。
  • method は AP_METHOD_BIT を用いたビットシフト演算のインデックスとして有効であること。
  • r->useragent_addr および r->subprocess_env が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. 指定された method に対応するビットマスクを作成し、ルール配列 a を順次走査する。 2. 現在のメソッドがルールに適用されない場合はスキップする。 3. ルールの type に応じて以下の判定を行う：

  • T_ENV / T_NENV: r->subprocess_env 内の環境変数の存在有無を確認。
  • T_ALL: 無条件で 1 を返す。
  • T_IP: apr_ipsubnet_test を使用し、クライアント IP がサブネットに含まれるか確認。
  • T_HOST: ap_get_useragent_host でホスト名を取得し、in_domain を用いてドメイン一致を確認。

    • 4. いずれかのルールに一致した時点で 1 を返し、最後まで一致しなければ 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_access_compat.c 内の check_dir_access 関数（allows および denys 配列の評価時に複数回呼び出される）。
  • **呼び出し先**:
  • apr_table_get
  • apr_ipsubnet_test
  • ap_get_useragent_host
  • in_domain

関連データ構造

  • allowdeny: ルール定義を保持する構造体。
  • request_rec: Apache のリクエスト情報。
  • apr_array_header_t: ルール配列のコンテナ。

失敗モードと境界条件

  • T_HOST 判定時、DNS 逆引きが失敗または IP アドレスが返された場合、ホスト名ベースの判定はスキップされる。
  • T_FAIL 型のルールは明示的に何もしない（do nothing）。
  • ルール配列が空の場合、ループは実行されず 0 を返す。

保守担当者向けメモ

  • T_HOST 判定には REMOTE_DOUBLE_REV が使用されており、DNS 逆引きの信頼性に依存する。
  • in_domain は文字列比較ベースであるため、セキュリティ要件に応じて DNS 解決の挙動を考慮すること。
