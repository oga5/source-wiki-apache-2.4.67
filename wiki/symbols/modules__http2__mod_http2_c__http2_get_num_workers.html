http2_get_num_workers

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_http2.c:152

目的

指定されたサーバー設定に基づき、HTTP/2 処理に使用するワーカースレッドの最小数および最大数を取得します。

シグネチャまたは形状

static void http2_get_num_workers(server_rec *s, int *minw, int *maxw)

入力と出力

  • **入力**:
  • s: サーバー設定構造体 (server_rec *)。
  • minw: 最小ワーカー数を格納するメモリ領域へのポインタ (int *)。
  • maxw: 最大ワーカー数を格納するメモリ領域へのポインタ (int *)。
  • **出力**:
  • なし（引数 minw および maxw を通じて値を返却）。

前提条件と不変条件

  • s は有効な server_rec 構造体である必要があります。
  • minw および maxw は、結果を受け取るために有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 内部的なダミーの時刻変数 tdummy を宣言します。 2. h2_get_workers_config(s, minw, maxw, &tdummy) を呼び出し、設定値の取得を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/proxy_util.c: APR_RETRIEVE_OPTIONAL_FN(http2_get_num_workers) を介して取得および呼び出しが行われています。
  • **呼び出し先**:
  • h2_get_workers_config（定義箇所は本ソース excerpt に含まれていません）。
  • **登録**:
  • modules/http2/mod_http2.c:185 にて APR_REGISTER_OPTIONAL_FN によりオプション関数として登録されています。

関連データ構造

  • server_rec: Apache HTTP Server のサーバー設定を保持する構造体。

失敗モードと境界条件

  • h2_get_workers_config の内部実装に依存します。本関数自体はエラーコードを返さないため、無効なポインタが渡された場合の挙動は呼び出し元の責任となります。

保守担当者向けメモ

  • 本関数は mod_proxy 等の外部モジュールから APR のオプション関数として利用されることを想定したインターフェースです。
  • h2_get_workers_config のシグネチャ変更時には、本関数の引数および呼び出し方法も併せて確認してください。
