fs_get_modified

定義

  • 種別: function
  • 定義箇所: modules/md/md_store_fs.c:612

目的

指定されたファイルシステム上のリソース（グループ、名前、アスペクトに対応するファイル）の最終更新時刻を取得します。

シグネチャまたは形状

apr_time_t fs_get_modified(md_store_t *store, md_store_group_t group,  
                           const char *name, const char *aspect, apr_pool_t *p)

入力と出力

  • **入力**:
  • store: ストレージの抽象ハンドル。
  • group: 対象のデータグループ。
  • name: リソース名。
  • aspect: リソースの側面（ファイル種別など）。
  • p: メモリ割り当てに使用するプール。
  • **出力**:
  • 成功時: 最終更新時刻（apr_time_t）。
  • 失敗時: 0。

前提条件と不変条件

  • store は FS_STORE マクロにより md_store_fs_t 型へキャスト可能である必要があります。
  • 内部で md_util_pool_vdo を使用しており、操作は指定されたプール p のライフサイクルに依存します。

アルゴリズムまたは振る舞い

1. store から md_store_fs_t 構造体を取得します。 2. md_util_pool_vdo を介して、内部関数 pfs_get_modified を呼び出し、ファイルシステム上の最終更新時刻を mtime に格納します。 3. md_util_pool_vdo が APR_SUCCESS を返した場合、取得した mtime を戻り値として返します。 4. それ以外（エラー発生時）は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_store_fs.c:104（関数ポインタの初期化時）
  • modules/md/md_store_fs.c:312（s_fs->s.get_modified への代入）
  • **呼び出し先**:
  • pfs_get_modified（md_util_pool_vdo を介した間接呼び出し）

関連データ構造

  • md_store_t: ストレージの抽象インターフェース。
  • md_store_fs_t: ファイルシステムベースのストレージ実装構造体。
  • apr_time_t: APR で定義された時刻型。

失敗モードと境界条件

  • pfs_get_modified が失敗した場合（ファイルが存在しない、権限がない等）、APR_SUCCESS 以外のステータスが返され、本関数は 0 を返します。

保守担当者向けメモ

  • 本関数は md_store_t インターフェースの一部として実装されています。
  • pfs_get_modified の詳細な実装については、同ファイル内の当該関数の定義を参照してください。
