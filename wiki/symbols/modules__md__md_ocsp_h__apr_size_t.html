apr_size_t

定義

  • 種別: typedef
  • 定義箇所: modules/md/md_ocsp.h:49

目的

メモリ上のデータサイズやバッファ長を表現するための、プラットフォーム依存の符号なし整数型です。

シグネチャまたは形状

typedef size_t apr_size_t;

※Apache Portable Runtime (APR) によって定義される標準的なサイズ型です。

入力と出力

  • 入力: 該当なし（型定義のため）
  • 出力: 該当なし（型定義のため）

前提条件と不変条件

  • 常に非負の値をとります。
  • メモリ上のオブジェクトの最大サイズを表現できる範囲の値を保持します。

アルゴリズムまたは振る舞い

本シンボルは、md_ocsp_copy_der コールバック関数において、DER 形式でエンコードされたデータのバイト長（der_len）を指定するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_ocsp.h 内の md_ocsp_copy_der 関数ポインタ型定義において、引数として使用されています。
  • 呼び出し先: 本シンボル自体は型定義であるため、呼び出し先はありません。
  • 補足: Apache HTTP Server 全体で広く利用されており、include/ap_regex.h や include/ap_expr.h など、多数のヘッダーファイルで参照されています。

関連データ構造

  • md_ocsp_copy_der: 本型を引数に含むコールバック関数型。

失敗モードと境界条件

  • 型の最大値（APR_SIZE_MAX）を超える値を指定した場合、オーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは APR が提供する標準型です。Apache HTTP Server の他のモジュールと一貫性を保つため、サイズを扱う際は int ではなく必ず apr_size_t を使用してください。
