asn1_table_vhost_key

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:101

目的

仮想ホストごとの鍵IDを管理し、メモリリークを防ぎつつ、再起動時にも鍵情報を保持するためのハッシュテーブル操作を提供します。

シグネチャまたは形状

static const char *asn1_table_vhost_key(SSLModConfigRec *mc, apr_pool_t *p,
                                        const char *id, int i)

入力と出力

  • **入力**:
  • mc: SSL モジュールの設定構造体。
  • p: 一時的なメモリプール（再起動時等にクリアされる）。
  • id: 仮想ホストの識別子。
  • i: インデックス番号。
  • **出力**:
  • 仮想ホストの鍵を特定するための文字列（id:i 形式）。

前提条件と不変条件

  • mc->tVHostKeys ハッシュテーブルが初期化されている必要があります。
  • 引数 p は再起動時にクリアされる可能性があるため、永続的な保持が必要なデータは mc->pPool にコピーされます。

アルゴリズムまたは振る舞い

1. id と i を結合した文字列を一時プール p 上に生成します。 2. mc->tVHostKeys ハッシュテーブルから、生成した文字列をキーとして検索します。 3. 該当するエントリが存在しない場合:

  • mc->pPool（永続的なプール）上に文字列のコピーを作成します。
  • 作成したコピーをハッシュテーブルに登録します。

    • 4. 登録済み、または新規作成した文字列へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_pphrase.c:148
  • **呼び出し先**:
  • apr_psprintf
  • apr_hash_get
  • apr_pstrdup
  • apr_hash_set

関連データ構造

  • SSLModConfigRec: モジュール設定を保持する構造体。tVHostKeys（ハッシュテーブル）および pPool（永続プール）を内包します。

失敗モードと境界条件

  • メモリ不足時には apr_pstrdup 等の APR 関数がエラーを返し、処理が中断される可能性があります。
  • ハッシュテーブルのキーとして id が空の場合や、i が負の値である場合の挙動は、呼び出し側の入力値に依存します。

保守担当者向けメモ

  • 本関数は、サーバー再起動時にも鍵情報を保持し、パスフレーズ入力を最小限にする設計意図に基づいています。
  • mc->pPool を使用しているため、モジュールのライフサイクル全体でメモリが保持されます。
