xml2enc_ffunc

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_xml2enc.c:318

目的

Apache HTTP Server の出力フィルタとして、XML/HTML コンテンツの文字エンコーディングを自動検出し、UTF-8 へ変換して後続のフィルタへ渡す。

シグネチャまたは形状

apr_status_t xml2enc_ffunc(ap_filter_t* f, apr_bucket_brigade* bb)

入力と出力

  • **入力**: ap_filter_t* f (フィルタコンテキスト)、apr_bucket_brigade* bb (処理対象のバケットブリゲード)。
  • **出力**: apr_status_t (処理結果ステータス)。

前提条件と不変条件

  • f->r->content_type が存在し、かつ text/ で始まるか、application/xml 等の XML 関連メディアタイプである必要がある。
  • フィルタ初期化前は、エンコーディング検出に必要なデータ量（BUF_MIN）が蓄積されるまでバッファリングを行う。
  • 変換器（ctx->convset）が設定されている場合、データは UTF-8 に変換される。

アルゴリズムまたは振る舞い

1. **メディアタイプ検証**: 対象外のコンテンツタイプであれば、フィルタを解除してブリゲードをそのまま通過させる。 2. **バッファリングと検出**: ctx->bbsave にデータを蓄積し、BUF_MIN に達した時点で sniff_encoding を呼び出しエンコーディングを特定する。 3. **正規化**: fix_skipto により指定要素までスキップし、Content-Length ヘッダーを無効化する。 4. **変換処理**: apr_xlate_conv_buffer を使用し、バッファ内のデータを UTF-8 に変換する。 5. **ストリーム制御**: メタデータバケット（FLUSH/EOS）を適切に処理し、変換後のデータを ap_fwrite で後続フィルタへ渡す。 6. **エラー処理**: 不正なバイト列（APR_EINVAL）はスキップして処理を継続し、変換失敗時は生データのまま通過させる。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_register_output_filter_protocol によりフィルタとして登録（modules/filters/mod_xml2enc.c:591）。
  • **呼び出し先**: sniff_encoding, fix_skipto, xml2enc_run_preprocess, ap_pass_brigade, ap_fwrite, apr_xlate_conv_buffer 等。

関連データ構造

  • xml2ctx: フィルタの実行状態（バッファ、変換器、フラグ）を保持。
  • apr_bucket_brigade: 処理対象のデータストリーム。
  • xml2enc_module: モジュール設定の識別に使用。

失敗モードと境界条件

  • **データ不足**: APR_EINCOMPLETE が発生した場合、バッファを保持して次回の呼び出しを待機する。
  • **変換エラー**: APR_EINVAL の場合は該当バイトをスキップ。その他の致命的な変換エラー時は変換器を無効化し、生データのまま通過させる。
  • **メモリ不足**: apr_palloc 等のメモリ確保失敗時は ap_assert により停止する可能性がある。

保守担当者向けメモ

  • ap_assert の使用箇所は、将来的に適切なエラーハンドリングへの置き換えが推奨される。
  • libxml2 の検出ロジックに依存しており、サポート外のエンコーディングは設定されたデフォルト値へフォールバックする。
  • Content-Length を変更するため、本フィルタ実行後に再計算が必要となる。
