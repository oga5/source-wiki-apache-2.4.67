ap_expr_parse

定義

  • 種別: function
  • 定義箇所: server/util_expr_eval.c:379

目的

文字列として与えられた式を構文解析し、抽象構文木（AST）を生成して ap_expr_info_t 構造体に格納します。

シグネチャまたは形状

const char *ap_expr_parse(apr_pool_t *pool, apr_pool_t *ptemp,
                          ap_expr_info_t *info, const char *expr,
                          ap_expr_lookup_fn_t *lookup_fn)

入力と出力

  • **入力**:
  • pool: メモリ割り当て用プール。
  • ptemp: 一時的なメモリ割り当て用プール。
  • info: 解析結果（AST）を格納する構造体。
  • expr: 解析対象の式文字列。
  • lookup_fn: 変数や関数を解決するためのコールバック関数（NULL の場合は ap_expr_lookup_default が使用される）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • info->flags が解析コンテキストのフラグとして使用されます。
  • 評価中に生成される AST ノードは pool から割り当てられます。

アルゴリズムまたは振る舞い

1. ap_expr_parse_ctx_t を初期化し、入力文字列とルックアップ関数を設定します。 2. ap_expr_yylex_init および ap_expr_yyparse を呼び出し、字句解析と構文解析を実行します。 3. 解析終了後、ap_expr_yylex_destroy でスキャナを破棄します。 4. エラーが発生した場合はエラーメッセージを返し、成功した場合は info->root_node に生成された AST を設定します。 5. AP_EXPR_DEBUG が定義されている場合、解析後に AST をログ出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • include/ap_expr.h
  • modules/filters/mod_include.c
  • modules/lua/lua_request.c
  • modules/ssl/ssl_engine_config.c
  • server/util_expr_eval.c
  • **呼び出し先**:
  • ap_expr_yylex_init, ap_expr_yyparse, ap_expr_yylex_destroy (字句・構文解析)
  • ap_expr_lookup_default (デフォルトのルックアップ関数)
  • apr_psprintf (エラーメッセージ生成)
  • expr_dump_tree (デバッグ時のみ)

関連データ構造

  • ap_expr_info_t: 解析結果の AST ルートを保持する構造体。
  • ap_expr_parse_ctx_t: 解析中の状態（スキャナ、エラー情報、入力バッファ）を保持する内部コンテキスト。

失敗モードと境界条件

  • 式の構文が不正な場合、エラーメッセージが返されます。
  • ctx.error または ctx.error2 に値が設定されている場合、それらがエラー内容として返されます。
  • rc（yyparse の戻り値）が非ゼロの場合、"syntax error" が返されます。

保守担当者向けメモ

  • ctx.error は Bison が生成する汎用的なエラーメッセージであり、将来的に削除または改善が検討されています。
  • lookup_fn が提供されない場合のデフォルト動作として ap_expr_lookup_default がハードコードされています。
