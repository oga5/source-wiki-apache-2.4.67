modssl_pk_server_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:711

目的

サーバー（仮想ホスト）に関連付けられた公開鍵（Public Key）および証明書ファイルの設定情報を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *cert_files;
    apr_array_header_t *key_files;
    const char  *ca_name_path;
    const char  *ca_name_file;
    int service_unavailable;
} modssl_pk_server_t;

入力と出力

  • **入力**: サーバー設定時に証明書ファイルパス、鍵ファイルパス、および CA 名のパス/ファイルが格納されます。
  • **出力**: サーバーの初期化プロセスやリクエスト処理において、これらの設定値が参照されます。

前提条件と不変条件

  • cert_files および key_files は apr_array_header_t 型であり、APR のメモリ管理下で初期化されている必要があります。
  • service_unavailable は、TLS サービスが停止している場合に非ゼロの値をとります。

アルゴリズムまたは振る舞い

  • 本構造体は、特定の仮想ホストにおける SSL/TLS 構成の一部として保持されます。
  • ca_name_path および ca_name_file は、TLS ハンドシェイク時の CertificateRequest メッセージで送信される CA 名のリストを生成するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_config.c: サーバー設定構造体（sc->server->pks）から参照。
  • modules/ssl/ssl_engine_init.c: 初期化コンテキスト（ctx->pks）から参照。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • SSLSrvConfigRec: 本構造体をメンバとして保持する仮想ホストごとの設定構造体。
  • apr_array_header_t: 証明書および鍵のファイルリストを管理するために使用される APR の配列構造体。

失敗モードと境界条件

  • ファイルパスが正しく設定されていない場合、初期化フェーズで証明書や鍵の読み込みに失敗する可能性があります。
  • service_unavailable が設定されている場合、当該サーバーでの TLS 通信は拒否または中断される振る舞いとなります。

保守担当者向けメモ

  • 本構造体は SSLSrvConfigRec を通じてアクセスされることが一般的です。
  • cert_files や key_files を操作する際は、APR プールの生存期間と整合性が取れていることを確認してください。
