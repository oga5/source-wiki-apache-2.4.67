dav_lockdb_combined

定義

  • 種別: struct
  • 定義箇所: modules/dav/lock/locks.c:185

目的

mod_dav ロックプロバイダーにおいて、公開インターフェースである dav_lockdb と、内部的な実装詳細を保持する dav_lockdb_private を統合し、単一のメモリブロックとして管理するために使用されます。

シグネチャまたは形状

typedef struct
{
    dav_lockdb pub;
    dav_lockdb_private priv;
} dav_lockdb_combined;

入力と出力

本構造体はデータ保持を目的としており、関数ではないため直接的な入力・出力はありません。ただし、mod_dav のロック操作において、この構造体へのポインタがロックデータベースのコンテキストとして引き回されます。

前提条件と不変条件

  • pub フィールドは dav_lockdb 型として、外部インターフェースとの互換性を維持する必要があります。
  • priv フィールドは dav_lockdb_private 型として、DBM ハンドルやリクエストコンテキストを保持します。
  • 本構造体は、リクエストの生存期間中、request_rec のプールから割り当てられることを前提としています。

アルゴリズムまたは振る舞い

本構造体自体はデータコンテナであり、特定のアルゴリズムは実装していません。この構造体を通じて、ロックデータベースのオープン、読み込み、書き込みといった dav_generic_* 系関数が、公開用データと内部用データを一元的に参照します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/lock.c および modules/dav/lock/locks.c 内の関数が、ロックデータベースのコンテキストを初期化・参照する際に本構造体をインスタンス化またはキャストして使用しています。
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • dav_lockdb: 公開用のロックデータベースインターフェース。
  • dav_lockdb_private: DBM ハンドルやリクエストコンテキストを保持する内部構造体。

失敗モードと境界条件

  • メモリ割り当て失敗: request_rec プールからの割り当てが失敗した場合、本構造体は初期化されません。
  • キャストの不整合: dav_lockdb へのポインタを dav_lockdb_combined にキャストしてアクセスする際、メモリレイアウトが一致していない場合は未定義動作を引き起こします。

保守担当者向けメモ

  • 本構造体は dav_lockdb と dav_lockdb_private を密結合させています。構造体のメンバを変更する際は、これら二つの構造体との整合性に注意してください。
  • modules/dav/fs/lock.c からも参照されているため、変更時には mod_dav のファイルシステムロック実装への影響を考慮する必要があります。
