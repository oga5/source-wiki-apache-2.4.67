md_try_chown

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_os.c:35

目的

指定されたファイルまたはディレクトリの所有者（UID）およびグループ（GID）を変更します。主に、ルート権限で起動したサーバーが、ワーカプロセスがアクセス可能な権限へリソースを調整するために使用されます。

シグネチャまたは形状

apr_status_t md_try_chown(const char *fname, unsigned int uid, int gid, apr_pool_t *p)

入力と出力

  • fname: 対象となるファイルまたはディレクトリのパス。
  • uid: 設定するユーザーID。
  • gid: 設定するグループID。
  • p: メモリ割り当てに使用する APR プール。
  • 戻り値: 成功時は APR_SUCCESS。OS が機能をサポートしていない場合は APR_ENOTIMPL。chown システムコールが失敗した場合は、そのエラーコードを返します。

前提条件と不変条件

  • AP_NEED_SET_MUTEX_PERMS マクロが定義され、かつ HAVE_UNISTD_H が有効な環境でのみ動作します。
  • プロセスがルート権限（geteuid() == 0）で実行されている場合にのみ、実際の chown システムコールが発行されます。
  • ルート権限以外で実行されている場合、所有権の変更は行われず、常に APR_SUCCESS を返します。

アルゴリズムまたは振る舞い

1. コンパイル時に OS が権限変更をサポートしているか確認します。サポート外の場合は APR_ENOTIMPL を返します。 2. 現在の実行ユーザーがルートであるかを確認します。ルートでない場合は、何もせず APR_SUCCESS を返します。 3. chown(fname, uid, gid) を実行します。 4. chown が失敗した場合、そのエラーが「ファイルが存在しない（ENOENT）」以外であれば、エラーログを出力し、エラーコードを返します。 5. 成功した場合は APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/md/mod_md_os.c:63 (md_make_worker_accessible 関数内)
  • 呼び出し先:
  • chown (システムコール)
  • ap_log_perror (Apache ログ出力関数)

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • ap_unixd_config: 呼び出し元で利用される、実行ユーザー設定を保持する構造体。

失敗モードと境界条件

  • **ファイル不在**: chown が ENOENT を返した場合、エラーログを出力せずに ENOENT を返します。
  • **権限不足**: ルート権限で実行されていない場合、この関数は実質的に何もしません。
  • **OS 非対応**: AP_NEED_SET_MUTEX_PERMS が定義されていない環境（Windows 等）では、常に APR_ENOTIMPL を返します。

保守担当者向けメモ

  • 本関数は、サーバーがルート権限で起動された場合にのみ意味を持つ設計です。非特権ユーザーで実行される環境では、権限変更の失敗を無視する安全策が講じられています。
  • chown の戻り値チェックにおいて ENOENT を特別扱いしているのは、ファイルが既に削除されている等の競合状態を許容するためです。
