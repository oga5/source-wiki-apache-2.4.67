ssl_scache_remove

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_scache.c:176

目的

SSL セッションキャッシュから、指定されたセッション ID に対応するエントリを削除します。

シグネチャまたは形状

void ssl_scache_remove(server_rec *s, IDCONST UCHAR *id, int idlen, apr_pool_t *p)

入力と出力

  • s: サーバー設定を保持する server_rec 構造体へのポインタ。
  • id: 削除対象のセッション ID を指すポインタ。
  • idlen: セッション ID の長さ。
  • p: メモリ割り当てに使用する apr_pool_t プール。
  • 戻り値: なし。

前提条件と不変条件

  • myModConfig(s) を通じて取得される SSLModConfigRec が有効であること。
  • mc->sesscache プロバイダーが初期化済みであること。

アルゴリズムまたは振る舞い

1. SSLModConfigRec からキャッシュ設定を取得します。 2. キャッシュプロバイダーがスレッドセーフでない場合 (AP_SOCACHE_FLAG_NOTMPSAFE フラグが設定されている場合)、ssl_mutex_on(s) を呼び出し排他制御を開始します。 3. プロバイダーの remove メソッドを呼び出し、キャッシュから該当セッションを削除します。 4. 排他制御を開始していた場合、ssl_mutex_off(s) を呼び出しロックを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ssl/ssl_engine_kernel.c:2015
  • 呼び出し先:
  • ssl_mutex_on / ssl_mutex_off (排他制御が必要な場合)
  • mc->sesscache->remove (キャッシュプロバイダーの削除関数)

関連データ構造

  • SSLModConfigRec: モジュールの設定情報を保持する構造体。
  • server_rec: Apache HTTP Server のサーバー設定構造体。
  • ap_socache_provider_t: mc->sesscache として参照されるキャッシュプロバイダーインターフェース。

失敗モードと境界条件

  • キャッシュプロバイダーが未設定の場合、mc->sesscache へのアクセスによりクラッシュする可能性があります。
  • 排他制御が必要な環境でロック取得に失敗した場合の挙動は、ssl_mutex_on の実装に依存します。

保守担当者向けメモ

  • 本関数は ssl_private.h で宣言されており、モジュール内でのセッション削除の標準的なインターフェースとして機能します。
  • キャッシュプロバイダーの remove メソッド呼び出しは、プロバイダー側の実装に完全に委譲されています。
