service_nt_ctrl

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/service.c:341

目的

Windowsサービスコントロールマネージャ（SCM）からの制御要求を処理するコールバック関数です。

シグネチャまたは形状

static DWORD WINAPI service_nt_ctrl(DWORD dwCtrlCode, DWORD dwEventType,
                                    LPVOID lpEventData, LPVOID lpContext)

入力と出力

  • **入力**:
  • dwCtrlCode: SCMから送信された制御コード（停止、再起動、照会など）。
  • dwEventType: イベントタイプ（使用されません）。
  • lpEventData: イベントデータ（使用されません）。
  • lpContext: サービス状態を保持する nt_service_ctx_t 構造体へのポインタ。
  • **出力**:
  • 成功時は NO_ERROR を返します。
  • 未知の制御コードが渡された場合は ERROR_CALL_NOT_IMPLEMENTED を返します。

前提条件と不変条件

  • lpContext は有効な nt_service_ctx_t 構造体を指している必要があります。
  • 本関数は RegisterServiceCtrlHandlerEx を介してSCMに登録されている必要があります。

アルゴリズムまたは振る舞い

1. **停止/シャットダウン要求**: SERVICE_CONTROL_STOP または SERVICE_CONTROL_SHUTDOWN を受信した場合、ap_signal_parent(SIGNAL_PARENT_SHUTDOWN) を呼び出し、サービス状態を SERVICE_STOP_PENDING に更新して NO_ERROR を返します。 2. **再起動要求**: SERVICE_APACHE_RESTART を受信した場合、ap_signal_parent(SIGNAL_PARENT_RESTART) を呼び出し、サービス状態を SERVICE_START_PENDING に更新して NO_ERROR を返します。 3. **状態照会**: SERVICE_CONTROL_INTERROGATE を受信した場合、現在のサービス状態を ReportStatusToSCMgr を通じてSCMに通知し、NO_ERROR を返します。 4. **その他**: 上記以外の制御コードに対しては ERROR_CALL_NOT_IMPLEMENTED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Windows SCM（RegisterServiceCtrlHandlerExW / RegisterServiceCtrlHandlerExA を介して登録）。
  • **呼び出し先**:
  • ap_signal_parent
  • ReportStatusToSCMgr

関連データ構造

  • nt_service_ctx_t: サービスの状態管理に使用されるコンテキスト構造体。
  • globdat: サービス状態を保持するグローバルな静的インスタンス。

失敗モードと境界条件

  • 未定義の dwCtrlCode を受け取った場合、ERROR_CALL_NOT_IMPLEMENTED を返します。
  • サービス状態の更新（ReportStatusToSCMgr）が失敗した場合の挙動については、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数はSCMからの非同期イベントに応答するため、スレッドセーフな実装が求められます。
  • SERVICE_APACHE_RESTART はApache独自の制御コードであるため、標準的なWindows APIのコード範囲外である点に注意してください。
