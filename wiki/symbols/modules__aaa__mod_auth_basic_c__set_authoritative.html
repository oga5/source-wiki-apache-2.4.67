set_authoritative

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_basic.c:132

目的

Apache HTTP Server の設定ディレクティブ AuthBasicAuthoritative を処理し、認証失敗時に後続の認証モジュールへ処理を委譲するか否かを制御するフラグを設定します。

シグネチャまたは形状

static const char *set_authoritative(cmd_parms *cmd, void *config, int flag)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: auth_basic_config_rec 型のディレクトリ設定構造体。
  • flag: ディレクティブで指定された真偽値（1 または 0）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • config 引数は auth_basic_config_rec 構造体へのポインタとしてキャスト可能である必要があります。
  • flag が 1 の場合、認証失敗時に HTTP_UNAUTHORIZED が即座に返されます。

アルゴリズムまたは振る舞い

1. config を auth_basic_config_rec 型にキャストします。 2. flag の値を conf->authoritative に代入します。 3. conf->authoritative_set を 1 に設定し、設定が明示的に行われたことを記録します。 4. NULL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_basic.c:204: AP_INIT_FLAG マクロにより AuthBasicAuthoritative ディレクティブのハンドラとして登録。
  • modules/aaa/mod_auth_form.c:390: AuthFormAuthoritative ディレクティブのハンドラとして参照されている可能性あり（直接的な呼び出し関係はソースコードの文脈からは限定的）。
  • **呼び出し先**:
  • なし。

関連データ構造

  • auth_basic_config_rec: mod_auth_basic のディレクトリ設定を保持する構造体。authoritative および authoritative_set メンバを更新します。

失敗モードと境界条件

  • 引数 config が NULL または不正な型である場合、セグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数は設定の書き込みのみを行います。実際の認証フローにおける権限委譲の判定は、authenticate_basic_user 関数内で行われます。
  • mod_auth_form.c からの参照は、設定ハンドラの共有によるものですが、各モジュールの設定構造体のメモリレイアウトが一致していることを前提としています。
