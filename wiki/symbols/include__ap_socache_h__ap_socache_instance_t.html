ap_socache_instance_t

定義

  • 種別: struct
  • 定義箇所: include/ap_socache.h:49

目的

小規模オブジェクトキャッシュ（socache）の各バックエンド実装において、個別のキャッシュインスタンスの状態を保持するための不透明な（opaque）型です。

シグネチャまたは形状

typedef struct ap_socache_instance_t ap_socache_instance_t;

※構造体の内部定義は公開されておらず、各プロバイダー実装に委ねられています。

入力と出力

  • 本シンボル自体は型定義であるため、直接的な入出力はありません。
  • キャッシュプロバイダーのメソッド（create, init, store, retrieve 等）において、インスタンスへのポインタとして受け渡しされます。

前提条件と不変条件

  • キャッシュプロバイダーの create メソッドによって生成される必要があります。
  • init メソッドが呼び出されるまで、キャッシュ操作に使用することはできません。

アルゴリズムまたは振る舞い

  • 本型は、特定のキャッシュ実装（mod_socache_shmcb や mod_socache_dbm など）が管理するメモリ領域やファイルハンドルへのハンドルとして機能します。
  • 呼び出し側モジュールは、この型を直接操作せず、ap_socache_provider_t で定義されたインターフェースを通じて間接的に操作します。

呼び出し元と呼び出し先

  • **呼び出し元（利用側）**: mod_authn_socache.c や mod_cache_socache.c などのモジュールが、キャッシュインスタンスを保持する変数として参照しています。
  • **呼び出し先（実装側）**: ap_socache_provider_t 内の各メソッド（create, init, destroy, store, retrieve, remove, status, iterate）が、第一引数として本型のポインタを受け取ります。

関連データ構造

  • ap_socache_provider_t: 本インスタンスを操作するためのメソッド群を定義する構造体。
  • server_rec: store や retrieve 等の操作時に、関連するサーバー設定を特定するために併用されます。

失敗モードと境界条件

  • インスタンスが初期化される前にメソッドへ渡された場合、各プロバイダー実装の挙動に依存しますが、一般的には APR_EGENERAL 等のステータスコードが返されることが想定されます。

保守担当者向けメモ

  • 本型は不透明な型として定義されているため、実装の詳細を変更してもインターフェースの互換性は維持されます。
  • mod_socache_shmcb.c 等の具体的なプロバイダー実装を参照することで、内部構造の具体的な定義を確認可能です。
