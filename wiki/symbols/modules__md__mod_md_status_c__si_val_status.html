si_val_status

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:172

目的

Managed Domain (MD) の状態情報を取得し、HTML またはテキスト形式で出力バッファ（apr_bucket_brigade）に書き込む。

シグネチャまたは形状

static void si_val_status(status_ctx *ctx, md_json_t *mdj, const status_info *info)

入力と出力

  • ctx: ステータス出力用のコンテキスト（出力先 bb、フォーマットフラグ、メモリプールを含む）。
  • mdj: ドメイン状態を保持する JSON オブジェクト。
  • info: 現在処理中のカラム定義情報。
  • 出力: 戻り値なし。ctx->bb に対して状態文字列（"good", "expired", "error" 等）を書き込む。

前提条件と不変条件

  • ctx および mdj は有効なポインタであること。
  • info->key が mdj 内の有効な状態キーを参照していること。
  • HTML_STATUS(ctx) マクロにより、出力形式が HTML かテキストか判定される。

アルゴリズムまたは振る舞い

1. md_json_getl を使用して mdj から現在の状態コードを取得する。 2. 状態コードに応じた文字列を決定する:

  • MD_S_INCOMPLETE: MD_KEY_STATE_DESCR から詳細を取得し、"incomplete: [詳細]" を生成。
  • MD_S_EXPIRED_DEPRECATED / MD_S_COMPLETE: 証明書の有効期限（MD_KEY_VALID）を確認し、現在時刻と比較して "good" または "expired" を判定。
  • MD_S_ERROR: "error" を設定。
  • MD_S_MISSING_INFORMATION: "missing information" を設定。

    • 3. HTML_STATUS(ctx) が真の場合、HTML テーブルのセルとして文字列を出力。 4. 偽の場合、ctx->prefix と info->label を付与したテキスト形式で出力。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md_status.c:705 にある status_info 配列の定義内で参照されている。
  • 呼び出し先: md_json_getl, md_json_gets, md_json_get_time, apr_psprintf, HTML_STATUS マクロ, apr_brigade_puts, apr_brigade_printf。

関連データ構造

  • status_ctx: 出力処理のコンテキスト。
  • md_json_t: 状態情報を保持する JSON 構造体。
  • status_info: ステータス表示の定義（ラベルやキーを含む）。

失敗モードと境界条件

  • md_json_getl が未知の状態を返した場合、デフォルトの "unknown" が使用される。
  • 証明書の有効期限情報が取得できない場合、"expired" と判定される可能性がある。

保守担当者向けメモ

  • 本関数は status_info 配列のコールバックとして使用される。
  • 証明書の有効期限判定ロジックは apr_time_now() に依存している。
