backrefinfo

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:406

目的

正規表現の実行結果であるキャプチャグループ（バックリファレンス）の情報を保持するために使用されます。

シグネチャまたは形状

typedef struct backrefinfo {
    const char *source;
    ap_regmatch_t regmatch[AP_MAX_REG_MATCH];
} backrefinfo;

入力と出力

  • **入力**: 正規表現の照合対象となる文字列（source）および、照合結果のオフセット配列（regmatch）。
  • **出力**: なし（データ保持用の構造体）。

前提条件と不変条件

  • regmatch 配列は AP_MAX_REG_MATCH 要素を保持できるサイズである必要があります。
  • source は、対応する regmatch のオフセットが指し示す文字列の生存期間中、有効である必要があります。

アルゴリズムまたは振る舞い

  • 正規表現エンジンによるマッチング結果を格納し、後続の置換処理（$N 形式のバックリファレンス展開など）で参照可能な状態にします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_rewrite.c:428 (briRR として宣言)
  • modules/mappers/mod_rewrite.c:429 (briRC として宣言)
  • modules/mappers/mod_rewrite.c:2586 (ポインタ経由で参照)
  • **呼び出し先**: 構造体定義のため、関数呼び出しは発生しません。

関連データ構造

  • ap_regmatch_t: Apache の正規表現マッチング結果を格納する型。
  • rewrite_ctx: briRR および briRC をメンバとして保持し、リクエスト処理中のコンテキストを管理します。

失敗モードと境界条件

  • AP_MAX_REG_MATCH を超えるキャプチャグループは保持できません。
  • source が NULL の場合、またはマッチングが失敗した後の不正なオフセットを参照した場合の動作は、この構造体単体では制御されません。

保守担当者向けメモ

  • 本構造体は mod_rewrite におけるルール適用（briRR）と条件評価（briRC）の両方で利用されます。
  • source メンバは、デバッグやログ出力時にどの文字列に対してマッチングが行われたかを追跡するために利用されます。
