alloc_func

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_brotli.c:168

目的

Brotli エンコーダインスタンスがメモリを確保する際に使用する、Apache APR アロケータへのラッパー関数です。

シグネチャまたは形状

static void *alloc_func(void *opaque, size_t size)

入力と出力

  • **入力**:
  • opaque: apr_bucket_alloc_t * 型のバケットアロケータ（BrotliEncoderCreateInstance に渡されたコンテキスト）。
  • size: 確保するメモリのバイト数。
  • **出力**:
  • 確保されたメモリ領域へのポインタ。失敗した場合は NULL。

前提条件と不変条件

  • opaque は有効な apr_bucket_alloc_t 構造体へのポインタである必要があります。
  • 本関数は Brotli ライブラリのコールバックとして呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. 受け取った opaque（バケットアロケータ）と size を apr_bucket_alloc 関数に渡します。 2. apr_bucket_alloc が返したメモリ領域をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: BrotliEncoderCreateInstance（modules/filters/mod_brotli.c:197 にて、ctx->state の初期化時にコールバックとして登録）。
  • **呼び出し先**: apr_bucket_alloc（Apache APR ライブラリ）。

関連データ構造

  • apr_bucket_alloc_t: Apache のバケットアロケータ。メモリ管理の基盤として使用されます。

失敗モードと境界条件

  • apr_bucket_alloc がメモリ不足等で失敗した場合、本関数も NULL を返します。

保守担当者向けメモ

  • 本関数は free_func と対になって動作します。Brotli エンコーダのライフサイクル管理において、メモリリークを防ぐために apr_bucket_alloc を介した管理が必須です。
