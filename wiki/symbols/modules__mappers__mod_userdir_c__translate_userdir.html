translate_userdir

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_userdir.c:198

目的

~username 形式の URI を、設定されたユーザーのホームディレクトリまたは特定のディレクトリパスへマッピングし、必要に応じてリダイレクトまたはファイルパスの解決を行う。

シグネチャまたは形状

static int translate_userdir(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (Apache のステータスコード: OK, DECLINED, または HTTP_MOVED_TEMPORARILY)

前提条件と不変条件

  • URI が /~ で始まる必要がある。
  • UserDir ディレクティブが設定されている必要がある。
  • ユーザー名が空、または . や .. ではないこと。
  • ユーザーが disabled_users リストに含まれていないこと。
  • globally_disabled が有効な場合、enabled_users リストに含まれていること。

アルゴリズムまたは振る舞い

1. **URI 解析**: URI が /~ で始まるか確認し、ユーザー名を抽出する。 2. **アクセス制御**: 設定に基づき、ユーザーの有効/無効を判定する。 3. **パス変換**: UserDir 設定に従い、以下のいずれかの処理を行う。

  • **リダイレクト**: パスが URL 形式（: を含む）の場合、Location ヘッダーを設定し HTTP_MOVED_TEMPORARILY を返す。
  • **パス解決**: apr_uid_homepath_get を使用してホームディレクトリを取得し、物理パスを構築する。

    • 4. **検証と適用**: 構築したパスが存在するか（または設定の最後か）を確認し、r->filename を更新して OK を返す。 5. **終了**: 条件に合致しない場合は DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の translate_name フック (modules/mappers/mod_userdir.c:376 にて登録)。
  • **呼び出し先**:
  • ap_get_module_config, ap_getword, ap_getword_conf, ap_os_is_path_absolute, ap_set_context_info, ap_set_module_config (Apache API)
  • apr_table_get, apr_pstrcat, apr_stat, apr_uid_homepath_get (APR API)

関連データ構造

  • request_rec: リクエスト情報を保持するコア構造体。
  • userdir_config: モジュールの設定（UserDir パターン、有効/無効ユーザーリスト）を保持する構造体。

失敗モードと境界条件

  • URI が /~ で始まらない場合: DECLINED を返す。
  • ユーザー名が解決できない場合: DECLINED を返す。
  • UserDir 設定が空の場合: DECLINED を返す。
  • ユーザーがアクセス禁止リストにある場合: DECLINED を返す。
  • ファイルが存在しない場合: 最後の設定項目であれば処理を継続し、そうでなければ次の設定項目へ進む。

保守担当者向けメモ

  • dname のポインタ操作は、/~user 後のパス部分を正しく保持するための特殊な処理である。
  • HAVE_DRIVE_LETTERS 定義時とそれ以外で、リダイレクト判定のロジックが分岐する。
  • apr_uid_homepath_get は OS 依存の関数であるため、プラットフォームごとの挙動に注意すること。
  • r->finfo への代入は、システムコールを削減するための最適化として行われている。
