register_hooks

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:290

目的

Apache HTTP Server の起動プロセスにおいて、HTTP プロトコル処理に必要なサーバーフックの登録およびフィルタの登録を行う。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（設定用メモリプール）
  • **出力**: なし（副作用としてサーバーのフックテーブルおよびフィルタテーブルを更新）

前提条件と不変条件

  • 本関数はサーバー起動時の初期化フェーズで呼び出される。
  • 登録されるフィルタおよびフックは、以降の接続処理において HTTP プロトコルスタックの挙動を決定付ける。

アルゴリズムまたは振る舞い

1. **フックの登録**: 以下のサーバーライフサイクルイベントを登録する。

  • post_config: http_post_config を実行。
  • process_connection: ap_process_http_connection を APR_HOOK_REALLY_LAST で登録。
  • map_to_storage: ap_send_http_trace および http_send_options を登録。
  • http_scheme / default_port: それぞれ http_scheme と http_port を登録。
  • create_request: http_create_request を登録。

    • 2. **フィルタの登録**: 以下のフィルタを AP_FTYPE_PROTOCOL または AP_FTYPE_TRANSCODE として登録し、対応するハンドルをグローバル変数に保持する。

  • HTTP_IN (ap_http_filter)
  • HTTP_HEADER (ap_http_header_filter)
  • CHUNK (ap_http_chunk_filter)
  • HTTP_OUTERROR (ap_http_outerror_filter)
  • BYTERANGE (ap_byterange_filter)

    • 3. **メソッド初期化**: ap_method_registry_init(p) を呼び出し、HTTP メソッドのレジストリを初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュールロード機構（直接の呼び出し元はソースコード上では明示されていないが、http_config.h の register_hooks 型定義に従い、サーバー起動時に自動的に呼び出される）。
  • **呼び出し先**:
  • ap_hook_* 系関数（フック登録用）
  • ap_register_input_filter / ap_register_output_filter（フィルタ登録用）
  • ap_method_registry_init（メソッド初期化）
  • 各種コールバック関数（http_post_config, ap_process_http_connection, ap_http_filter 等）

関連データ構造

  • apr_pool_t: 設定用メモリプール。
  • ap_http_input_filter_handle 等のグローバルフィルタハンドル。

失敗モードと境界条件

  • メモリ不足により apr_pool_t 関連の操作やフィルタ登録が失敗した場合、サーバーの起動が中断される可能性がある。

保守担当者向けメモ

  • 本関数は http_core.c における設定の起点である。フィルタの追加やフックの優先順位（APR_HOOK_MIDDLE, APR_HOOK_REALLY_LAST 等）を変更する際は、他のモジュールとの競合に注意すること。
  • ツールが提示した docs/manual/... 等の参照先は、本関数を直接呼び出しているのではなく、モジュール開発のサンプルコードとして言及されているものである。
