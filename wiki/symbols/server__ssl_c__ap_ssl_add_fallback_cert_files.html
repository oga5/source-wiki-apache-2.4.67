ap_ssl_add_fallback_cert_files

定義

  • 種別: function
  • 定義箇所: server/ssl.c:231

目的

SSL サーバー設定に対して、フォールバック用の証明書ファイルおよび鍵ファイルのリストを登録します。

シグネチャまたは形状

apr_status_t ap_ssl_add_fallback_cert_files(server_rec *s, apr_pool_t *p,
                                            apr_array_header_t *cert_files,
                                            apr_array_header_t *key_files)

入力と出力

  • **入力**:
  • s: サーバー設定構造体 (server_rec)。
  • p: メモリ割り当てに使用するプール (apr_pool_t)。
  • cert_files: 証明書ファイルパスの配列 (apr_array_header_t)。
  • key_files: 鍵ファイルパスの配列 (apr_array_header_t)。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は APR_EGENERAL を返します。

前提条件と不変条件

  • ap_run_ssl_add_fallback_cert_files フックが登録されている必要があります。

アルゴリズムまたは振る舞い

1. ap_run_ssl_add_fallback_cert_files フックを実行し、登録された SSL モジュールに証明書と鍵ファイルの情報を渡します。 2. フックの戻り値 rv を評価します。 3. rv が OK または DECLINED の場合は APR_SUCCESS を返し、それ以外の場合は APR_EGENERAL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_init.c:2125 (直接の呼び出しが確認されています)
  • **呼び出し先**:
  • ap_run_ssl_add_fallback_cert_files (フック経由で実行)

関連データ構造

  • server_rec: Apache サーバー設定を保持する構造体。
  • apr_array_header_t: APR が提供する配列構造体。

失敗モードと境界条件

  • フックが OK または DECLINED 以外の値を返した場合、関数は APR_EGENERAL を返し、処理の失敗を示します。

保守担当者向けメモ

  • 本関数は include/http_ssl.h で宣言されており、API の後方互換性維持の一環として提供されています。
  • 実際の証明書処理ロジックはフック先に委譲されているため、詳細な挙動は mod_ssl 等のフック実装側に依存します。
