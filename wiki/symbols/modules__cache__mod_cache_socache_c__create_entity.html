create_entity

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_socache.c:357

目的

キャッシュ対象のレスポンスが設定された制限内に収まるかを検証し、共有オブジェクトキャッシュ（socache）用のキャッシュオブジェクト構造体を初期化する。

シグネチャまたは形状

static int create_entity(cache_handle_t *h, request_rec *r, const char *key, apr_off_t len, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • h: キャッシュハンドル。
  • r: 現在のリクエストレコード。
  • key: キャッシュの一意なキー。
  • len: レスポンスボディのサイズ。
  • bb: レスポンスデータのブリゲード。
  • **出力**:
  • 成功時: OK を返し、h->cache_obj を初期化する。
  • 失敗時: DECLINED を返し、キャッシュ処理を中断する。

前提条件と不変条件

  • conf->provider が設定されていること。
  • 部分コンテンツ（HTTP_PARTIAL_CONTENT）はキャッシュ対象外とする。
  • len が負の値である場合、サイズ不明としてキャッシュを拒否する。
  • ボディサイズおよびヘッダーサイズの合計が、ディレクトリ設定 dconf->max を超えないこと。

アルゴリズムまたは振る舞い

1. **プロバイダー確認**: socache プロバイダーが未設定であれば DECLINED を返す。 2. **バリデーション**:

  • 部分コンテンツリクエストを拒否。
  • ボディサイズ len が dconf->max を超える場合は拒否。
  • store_table を用いて、ヘッダー（headers_out, headers_in）のサイズを推定し、ボディサイズと合計して dconf->max を超えるか検証。

    • 3. **初期化**:

  • r->pool を使用して cache_object_t および cache_socache_object_t を割り当てる。
  • キャッシュキーを obj->key および sobj->key に複製して格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/cache_storage.c:97（cache_storage インターフェース経由）。
  • **呼び出し先**: ap_get_module_config, ap_log_rerror, store_table, apr_pcalloc, apr_pstrdup。

関連データ構造

  • cache_handle_t: キャッシュエントリのハンドル。
  • cache_socache_dir_conf: ディレクトリ単位のキャッシュ設定（max 等）。
  • cache_object_t: キャッシュエントリの汎用構造体。
  • cache_socache_object_t: socache 固有のキャッシュオブジェクト構造体。

失敗モードと境界条件

  • len が dconf->max を超える場合、ログを出力して DECLINED を返す。
  • ヘッダーのシリアライズサイズが制限を超える場合、store_table が失敗し DECLINED を返す。
  • メモリ割り当て失敗時は apr_pcalloc が例外を発生させる可能性がある。

保守担当者向けメモ

  • 本関数はキャッシュの「計画段階」で呼ばれるため、実際のデータサイズが確定する前に判断を行う必要がある。
  • store_table を使用したサイズ推定は、キャッシュのオーバーヘッドを考慮した保守的な計算となっている。
