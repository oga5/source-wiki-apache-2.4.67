ap_watchdog_set_callback_interval

定義

  • 種別: function
  • 定義箇所: modules/core/mod_watchdog.c:321

目的

指定された Watchdog インスタンスに登録済みのコールバック関数の実行間隔を更新します。

シグネチャまたは形状

apr_status_t ap_watchdog_set_callback_interval(ap_watchdog_t *w,
                                               apr_interval_time_t interval,
                                               const void *data,
                                               ap_watchdog_callback_fn_t *callback)

入力と出力

  • **入力**:
  • w: 対象の ap_watchdog_t インスタンス。
  • interval: 新たに設定する実行間隔（apr_interval_time_t）。
  • data: コールバックを識別するためのデータポインタ。
  • callback: コールバック関数ポインタ。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • 対象のコールバックが見つからない場合は APR_EOF を返します。

前提条件と不変条件

  • w->callbacks リスト内の要素のうち、data と callback_fn が引数と一致するものが更新対象となります。
  • 更新対象が見つかった場合、そのコールバックの step は 0 にリセットされ、status は APR_SUCCESS に初期化されます。

アルゴリズムまたは振る舞い

1. w->callbacks リストを先頭から順に走査します。 2. 現在のノードの data および callback_fn が引数と一致するか確認します。 3. 一致した場合、そのノードの interval を新しい値で上書きし、step を 0 に、status を APR_SUCCESS に設定してループを終了します。 4. リストの末尾まで一致するものが見つからなければ APR_EOF を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_drive.c および modules/md/mod_md_ocsp.c が APR_RETRIEVE_OPTIONAL_FN を介して本関数を取得し、利用していることが示唆されています。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • ap_watchdog_t: 監視インスタンスを保持する構造体。
  • watchdog_list_t: コールバック関数とその実行状態を管理するリンクリスト構造体。

失敗モードと境界条件

  • 指定された data および callback の組み合わせがリスト内に存在しない場合、APR_EOF が返されます。
  • w が NULL の場合、リスト走査時にセグメンテーション違反が発生する可能性があります（ソースコード上にガード句はありません）。

保守担当者向けメモ

  • 本関数は APR_DECLARE_OPTIONAL_FN として公開されており、他のモジュールから動的にリンクされることを前提としています。
  • コールバックの実行間隔を変更する際、step をリセットすることで、次回のループから即座に新しい間隔での計測が開始されるよう設計されています。
