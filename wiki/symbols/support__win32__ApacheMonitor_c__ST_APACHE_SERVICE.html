ST_APACHE_SERVICE

定義

  • 種別: struct
  • 定義箇所: support/win32/ApacheMonitor.c:75

目的

Windows 上で監視対象となる Apache HTTP Server サービスの属性（識別子、パス、状態）を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct _st_APACHE_SERVICE
{
    LPTSTR   szServiceName;
    LPTSTR   szDisplayName;
    LPTSTR   szDescription;
    LPTSTR   szImagePath;
    LPTSTR   szComputerName;
    DWORD    dwPid;
} ST_APACHE_SERVICE;

入力と出力

本構造体はデータ保持用であり、関数ではありません。各メンバは LPTSTR（文字列）および DWORD（プロセスID）として定義されています。

前提条件と不変条件

  • szServiceName 等の文字列メンバは、動的に確保されたメモリ（_tcsdup 等）を指すことが想定されています。
  • 終了処理時には am_ClearServicesSt 関数を介して、これらの文字列メンバのメモリを解放する必要があります。

アルゴリズムまたは振る舞い

本構造体自体にロジックは含まれませんが、g_stServices 配列の要素として、GetApacheServicesStatus によるレジストリ走査結果の格納や、ApacheManageService によるサービス制御時の参照先として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • support/win32/ApacheMonitor.c:101: グローバル配列 g_stServices の型として使用。
  • support/win32/ApacheMonitor.c:144, 1639: memset を用いた初期化処理で参照。
  • **呼び出し先**: 構造体のため、関数呼び出しは発生しません。

関連データ構造

  • ST_MONITORED_COMP: リモート監視対象のコンピュータ情報を保持する構造体。
  • g_stServices[]: 本構造体を要素とする固定長配列（最大 MAX_APACHE_SERVICES = 128）。

失敗モードと境界条件

  • **メモリ管理**: 文字列メンバの解放漏れによるメモリリークの可能性があります。
  • **配列制限**: MAX_APACHE_SERVICES（128）を超えるサービスは監視できません。

保守担当者向けメモ

  • 構造体内の文字列メンバは動的確保されるため、更新や破棄の際は必ずメモリ管理方針（am_ClearServicesSt の使用）に従ってください。
  • 大規模環境でサービス数が増加する場合、固定長配列の制限に注意が必要です。
