rotate_config

定義

  • 種別: struct
  • 定義箇所: support/rotatelogs.c:57

目的

rotatelogs ユーティリティの実行時設定（ローテーション条件、ファイルパス、動作フラグ等）を保持するための構造体です。

シグネチャまたは形状

struct rotate_config {
    unsigned int sRotation;
    int tRotation;
    int utc_offset;
    int use_localtime;
    int use_strftime;
    int force_open;
    int verbose;
    int echo;
    char *szLogRoot;
    int truncate;
    int truncate_rotated_only;
    const char *linkfile;
    const char *postrotate_prog;
#if APR_FILES_AS_SOCKETS
    int create_empty;
#endif
    int num_files;
    int create_path;
};

入力と出力

  • **入力**: コマンドライン引数から解析された値が格納されます。
  • **出力**: ログローテーションの判定ロジック（checkRotate）および実行ロジック（doRotate）の制御パラメータとして使用されます。

前提条件と不変条件

  • rotate_config_t（typedef による別名）として support/rotatelogs.c:55 で定義されています。
  • APR_FILES_AS_SOCKETS が定義されている環境では、create_empty メンバが含まれます。

アルゴリズムまたは振る舞い

  • ログファイルのローテーション閾値（サイズ sRotation または時間 tRotation）を保持し、ローテーションの実行可否を判定するための基準を提供します。
  • use_strftime や use_localtime などのフラグにより、ファイル名の生成規則や時刻の扱いを制御します。
  • postrotate_prog を通じて、ローテーション後に実行すべき外部プログラムのパスを保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数において、コマンドライン引数の解析結果が本構造体に格納されます。
  • **呼び出し先**: checkRotate および doRotate 関数が、本構造体の値を参照してローテーションの要否判定やファイル操作を行います。

関連データ構造

  • rotate_status_t: 実行時の状態（現在のファイルハンドルやプール等）を保持する対となる構造体。

失敗モードと境界条件

  • -n オプション（ファイル番号によるローテーション）と strftime 形式のファイル名は併用できません。
  • 書き込みエラー発生時は、本構造体の truncate 設定に基づき、既存ログの切り詰めやエラーメッセージの挿入が行われます。

保守担当者向けメモ

  • 本構造体は rotatelogs の設定の核となるため、新しいローテーション条件やオプションを追加する場合はこの構造体を拡張する必要があります。
  • APR_FILES_AS_SOCKETS 依存のメンバがあるため、プラットフォーム間でのバイナリ互換性やコンパイル条件に注意してください。
