request_rec

定義

  • 種別: typedef
  • 定義箇所: modules/loggers/mod_log_config.h:36
  • 補足: 本来の構造体定義は httpd.h:822 に存在します。

目的

Apache HTTP Server における個別の HTTP リクエストのライフサイクル全体を管理・追跡するための主要なデータ構造です。

シグネチャまたは形状

typedef struct request_rec request_rec; （httpd.h に準拠）

入力と出力

  • 入力: HTTP リクエスト処理の各フェーズにおいて、リクエストに関連する状態情報（URI、メソッド、ヘッダー等）を保持します。
  • 出力: ログ記録モジュール（mod_log_config 等）において、ログフォーマットハンドラ（ap_log_handler_fn_t）の引数として渡され、ログ出力に必要な情報を抽出するために使用されます。

前提条件と不変条件

  • リクエストの処理中、常に有効なポインタとして存在する必要があります。
  • ログ記録の文脈では、リクエストの終了（log_transaction フック等）まで生存していることが前提となります。

アルゴリズムまたは振る舞い

  • リクエスト処理の各段階で、サーバーの内部状態やクライアントからの入力情報がこの構造体に蓄積されます。
  • mod_log_config などのモジュールは、この構造体を参照してリクエスト固有の情報を取得し、ログ行を構築します。

呼び出し元と呼び出し先

  • 呼び出し元: Apache コアおよび各モジュール（mod_example_1.c, mod_example_2.c, ap_expr.h 等）が、リクエスト処理の引数として本構造体を使用しています。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数ではありません。

関連データ構造

  • server_rec: サーバー全体の構成情報を保持する構造体。
  • ap_log_handler_fn_t: request_rec を引数に取り、ログ文字列を生成する関数型。

失敗モードと境界条件

  • NULL ポインタの参照: 適切に初期化されていない、またはリクエスト終了後にアクセスした場合、セグメンテーション違反を引き起こす可能性があります。
  • 構造体の変更: ap_mmn.h に記載がある通り、バージョンアップに伴いメンバーの追加や削除が行われるため、バイナリ互換性に注意が必要です。

保守担当者向けメモ

  • 本シンボルは httpd.h で定義された構造体のエイリアスです。詳細なメンバー構成については httpd.h を参照してください。
  • ログ記録機能において、リクエストの特定の属性（例: 認証ユーザー、リクエスト時間）にアクセスする際の主要なエントリーポイントとなります。
