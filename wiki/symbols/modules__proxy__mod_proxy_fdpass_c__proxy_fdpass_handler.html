proxy_fdpass_handler

定義

  • **種別**: 関数
  • **定義箇所**: modules/proxy/mod_proxy_fdpass.c:123

目的

fd:// スキームのリクエストを処理し、クライアントとの接続ソケット（ファイルディスクリプタ）を Unix ドメインソケット経由で外部プロセスへ移譲（FD Passing）する。

シグネチャまたは形状

static int proxy_fdpass_handler(request_rec *r, proxy_worker *worker,
                                proxy_server_conf *conf,
                                char *url, const char *proxyname,
                                apr_port_t proxyport)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • url: fd:// で始まるターゲットパス。
  • worker: プロキシワーカー設定。
  • **出力**:
  • OK: 正常終了。
  • DECLINED: fd:// スキームではない場合。
  • HTTP_INTERNAL_SERVER_ERROR: 接続失敗、フラッシュ失敗、またはダミーソケット作成失敗時。

前提条件と不変条件

  • URL が fd:// で始まる必要がある。
  • ターゲットとなる Unix ドメインソケットへの接続が確立可能であること。
  • 外部プロセスへ FD を渡す前に、レスポンスのフラッシュが完了していること。

アルゴリズムまたは振る舞い

1. **スキーム検証**: URL が fd:// でない場合は DECLINED を返す。 2. **接続**: get_socket_from_path を呼び出し、ターゲットの Unix ドメインソケットを取得する。 3. **フラッシュ**: 設定された flusher プロバイダを呼び出し、バッファをフラッシュする。 4. **FD 送信**: send_socket を使用し、クライアントソケットの FD をターゲットソケットへ送信する。 5. **接続保護**: Apache コアが終了時にクライアントソケットをクローズしないよう、ap_set_core_module_config を用いてダミーソケットに差し替える。 6. **完了**: OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_fdpass.c 内の proxy_hook_scheme_handler に登録されている。
  • **呼び出し先**:
  • get_socket_from_path
  • ap_lookup_provider (フラッシュプロバイダ取得用)
  • flush->flusher
  • ap_get_conn_socket
  • send_socket
  • apr_socket_create
  • ap_set_core_module_config

関連データ構造

  • request_rec: リクエスト情報。
  • proxy_worker: プロキシワーカー設定。
  • proxy_fdpass_flush: レスポンスフラッシュ用プロバイダ構造体。

失敗モードと境界条件

  • fd:// 以外のスキーム: DECLINED を返す。
  • Unix ドメインソケットへの接続失敗: HTTP_INTERNAL_SERVER_ERROR を返す。
  • フラッシュプロバイダが見つからない、またはフラッシュ失敗: HTTP_INTERNAL_SERVER_ERROR を返す。
  • send_socket 失敗: HTTP_INTERNAL_SERVER_ERROR を返す。
  • ダミーソケット作成失敗: HTTP_INTERNAL_SERVER_ERROR を返す。

保守担当者向けメモ

  • 本関数は SCM_RIGHTS を利用した FD 移譲を行うため、ターゲットプロセスが FD を適切に受け取る実装になっていることが前提となる。
  • ダミーソケットへの差し替えは、Apache のクリーンアップ処理による誤ったクローズを防ぐための必須ハックである。
