ap_add_node

定義

  • 種別: function
  • 定義箇所: server/util_cfgtree.c:20

目的

設定ファイルから読み込まれたディレクティブを、メモリ上のツリー構造（ap_directive_t）に組み込みます。

シグネチャまたは形状

ap_directive_t *ap_add_node(ap_directive_t **parent, ap_directive_t *current,
                            ap_directive_t *toadd, int child)

入力と出力

  • **入力**:
  • parent: 現在の親ノードへのポインタのポインタ。
  • current: 現在の階層における直前のノード。
  • toadd: 追加対象の新しいノード。
  • child: 新しいノードが子階層を開始するかどうかを示すフラグ（真であれば子として扱う）。
  • **出力**:
  • 成功時: 追加されたノード（toadd）、または階層が切り替わった場合は NULL を返す。

前提条件と不変条件

  • toadd は呼び出し元で事前に割り当てられている必要があります。
  • parent は NULL でない場合、有効な ap_directive_t 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. **新規親の開始**: current が NULL の場合、*parent が存在すれば (*parent)->first_child に toadd を設定し、toadd->parent を更新します。 2. **兄弟の追加**: current が存在する場合、current->next に toadd を接続し、toadd->parent を *parent に設定します。 3. **階層の切り替え**: child が真の場合、*parent を toadd に更新し、以降のノードがその子として扱われるようにして NULL を返します。 4. **戻り値**: 階層切り替えが発生しない場合は、追加した toadd をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/config.c（1207行目および1242行目）から呼び出されています。
  • **呼び出し先**: 本関数内での関数呼び出しはありません。

関連データ構造

  • ap_directive_t: 設定ノードを表す構造体。first_child、parent、next ポインタを介してツリーを構成します。

失敗モードと境界条件

  • current が NULL の場合、ツリーのルートまたは新しい階層の開始点として処理されます。
  • child が真の場合、呼び出し元は *parent が更新されることを前提とする必要があります。

保守担当者向けメモ

  • ap_directive_t の last ポインタは末尾へのアクセスを高速化しますが、本関数では更新されません。
  • コンテナディレクティブの終了タグはツリーに格納されません。
