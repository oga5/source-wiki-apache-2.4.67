get_conv_set

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_ldap.c:145

目的

HTTP リクエストの Accept-Language ヘッダーに基づき、LDAP 認証・認可処理で使用する文字コード変換ハンドル（apr_xlate_t）を取得します。

シグネチャまたは形状

apr_xlate_t* get_conv_set(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: 成功時は apr_xlate_t* 型の変換ハンドル、失敗または該当なしの場合は NULL を返します。

前提条件と不変条件

  • r->headers_in に Accept-Language ヘッダーが存在する必要があります。
  • derive_codepage_from_lang 関数が、言語コードから有効な文字コード名を解決できる必要があります。
  • apr_xlate_open が指定された文字コードセットに対して成功する必要があります。

アルゴリズムまたは振る舞い

1. r->headers_in から Accept-Language ヘッダーを取得します。 2. ヘッダー値が存在する場合、最初の言語タグを抽出するために , または ; で文字列を終端させます。 3. derive_codepage_from_lang を呼び出し、言語コードを文字コード名に変換します。 4. apr_xlate_open を使用して、to_charset（グローバル変数と推測される）から変換先文字コードへのハンドルを作成します。 5. 変換ハンドルが正常に作成された場合、そのポインタを返します。それ以外の場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_ldap.c:178
  • modules/aaa/mod_authnz_ldap.c:229
  • **呼び出し先**:

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • apr_xlate_t: APR の文字コード変換ハンドル。

失敗モードと境界条件

  • Accept-Language ヘッダーが欠落している場合、NULL を返します。
  • 言語コードから文字コードへの解決に失敗した場合、NULL を返します。
  • apr_xlate_open が APR_SUCCESS を返さない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は to_charset という外部変数を参照しています。この変数の定義とスコープを確認してください。
  • r->pool を使用してメモリを割り当てているため、リクエスト終了時に自動的にクリーンアップされます。
