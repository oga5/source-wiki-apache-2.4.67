ssl_util_ppopen

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util.c:120

目的

外部コマンドをパイプ経由で実行し、その標準出力（stdout）へのファイルハンドルを取得する。

シグネチャまたは形状

apr_file_t *ssl_util_ppopen(server_rec *s, apr_pool_t *p, const char *cmd, const char * const *argv)

入力と出力

  • **入力**:
  • s: サーバーレコード（ログ出力等に使用可能）。
  • p: プロセスまたはリクエストの生存期間を管理するメモリプール。
  • cmd: 実行するコマンドのパス。
  • argv: コマンド引数の配列。
  • **出力**:
  • 成功時: 外部プロセスの標準出力に接続された apr_file_t*。
  • 失敗時: NULL。

前提条件と不変条件

  • 指定されたメモリプール p は、コマンド実行およびプロセス管理構造体の生存期間をカバーしている必要がある。
  • cmd は実行可能なパスであること。

アルゴリズムまたは振る舞い

1. apr_procattr_create を呼び出し、プロセス属性を初期化する。 2. apr_procattr_io_set により、標準入力・出力・エラーのすべてをブロックモードでパイプとして設定する。 3. apr_procattr_dir_set を使用し、コマンドの親ディレクトリを作業ディレクトリとして設定する。 4. apr_procattr_cmdtype_set で APR_PROGRAM を指定し、コマンド実行形式を定義する。 5. apr_pcalloc で apr_proc_t 構造体を確保する。 6. apr_proc_create を実行して外部プロセスを起動する。 7. 成功した場合、起動したプロセスの標準出力 (proc->out) を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_rand.c:79
  • modules/ssl/ssl_util.c:160
  • **呼び出し先**:
  • apr_procattr_create, apr_procattr_io_set, apr_procattr_dir_set, apr_procattr_cmdtype_set, ap_make_dirstr_parent, apr_pcalloc, apr_proc_create (すべて APR API)

関連データ構造

  • apr_file_t: パイプのファイルハンドル。
  • apr_proc_t: 外部プロセスの状態管理構造体。
  • apr_procattr_t: プロセス起動時の属性設定。

失敗モードと境界条件

  • APR の各関数（属性設定やプロセス生成）が APR_SUCCESS を返さない場合、関数は直ちに NULL を返す。
  • 外部コマンドの実行に失敗した場合、またはパイプの作成に失敗した場合も NULL が返される。

保守担当者向けメモ

  • 本関数は apr_proc_create を使用しており、呼び出し元は返された apr_file_t を適切にクローズする必要がある（通常は ssl_util_ppclose を使用）。
  • 実行されるコマンドのパス解決や権限については、呼び出し側で事前に検証することが推奨される。
