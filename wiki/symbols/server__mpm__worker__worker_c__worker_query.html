worker_query

定義

  • 種別: function
  • 定義箇所: server/mpm/worker/worker.c:337

目的

MPM（マルチプロセッシングモジュール）の現在の設定値や状態情報を、指定されたクエリコードに基づいて取得する。

シグネチャまたは形状

int worker_query(int query_code, int *result, apr_status_t *rv)

入力と出力

  • **入力**:
  • query_code: 取得したい情報を指定する AP_MPMQ_* 定数。
  • result: 取得した値を格納する整数型ポインタ。
  • rv: 処理結果のステータスコード（apr_status_t）を格納するポインタ。
  • **出力**:
  • 成功時は OK を返す。
  • rv には成功時 APR_SUCCESS、未対応のクエリコード指定時は APR_ENOTIMPL が設定される。

前提条件と不変条件

  • rv は呼び出し元によって有効なメモリ領域が提供されている必要がある。
  • retained 構造体が初期化済みであること。

アルゴリズムまたは振る舞い

1. rv を APR_SUCCESS で初期化する。 2. query_code に基づき switch 文で分岐する。 3. 各ケースにおいて、retained データ構造やグローバルな設定変数（server_limit, threads_per_child 等）から値を読み取り、result に代入する。 4. 未定義の query_code が渡された場合、rv に APR_ENOTIMPL を設定する。 5. 常に OK を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_mpm_query を介して登録されており、Apache HTTP Server のコア機能から呼び出される（server/mpm/worker/worker.c:2350 に登録の記述あり）。
  • **呼び出し先**: 直接的な関数呼び出しはない。

関連データ構造

  • retained: MPM の再起動を跨いで保持されるデータ構造。
  • server_limit, thread_limit, threads_per_child: MPM の動作制限を定義するグローバル変数。

失敗モードと境界条件

  • 未定義の query_code を指定した場合、result は更新されず、rv に APR_ENOTIMPL が設定される。

保守担当者向けメモ

  • 新しい MPM 状態や設定値を追加する場合は、switch 文に新しい case を追加すること。
  • retained 構造体へのアクセスは、MPM のライフサイクル管理と密接に関連しているため、変更時には整合性に注意が必要。
