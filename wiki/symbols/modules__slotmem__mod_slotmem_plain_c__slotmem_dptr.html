slotmem_dptr

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:149

目的

指定されたスロットメモリインスタンス内の特定のインデックス（id）に対応するメモリ領域への直接ポインタを取得します。

シグネチャまたは形状

static apr_status_t slotmem_dptr(ap_slotmem_instance_t *score, unsigned int id, void **mem)

入力と出力

  • **入力**:
  • score: スロットメモリインスタンスへのポインタ。
  • id: アクセス対象のスロット番号。
  • mem: 取得したポインタを格納するための出力用ポインタ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。エラー時は APR_ENOSHMAVAIL または APR_EINVAL。

前提条件と不変条件

  • score が有効なインスタンスを指している必要があります。
  • id は score->num（総スロット数）未満である必要があります。

アルゴリズムまたは振る舞い

1. score が NULL の場合、APR_ENOSHMAVAIL を返します。 2. id が score->num 以上の場合、APR_EINVAL を返します。 3. インスタンスのベースアドレス（score->base）に、スロットサイズ（score->size）と id を乗算したオフセットを加算し、対象スロットの先頭アドレスを算出します。 4. 算出されたポインタが NULL でないことを確認し、mem に格納して APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/slotmem/mod_slotmem_plain.c 内の内部関数。
  • modules/slotmem/mod_slotmem_shm.c 内の関数。
  • modules/proxy/mod_proxy_balancer.c および modules/proxy/proxy_util.c（ログ出力時のエラーハンドリング等で参照）。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • ap_slotmem_instance_t: スロットメモリの管理単位。ベースアドレスやスロットサイズ等のメタデータを保持します。

失敗モードと境界条件

  • score が NULL の場合: APR_ENOSHMAVAIL を返します。
  • id が範囲外の場合: APR_EINVAL を返します。
  • 算出されたポインタが NULL の場合: APR_ENOSHMAVAIL を返します。

保守担当者向けメモ

  • 本関数はメモリの直接ポインタを返すため、呼び出し側でメモリ保護や境界チェックを行う必要があります。
  • mod_slotmem_plain.c 内の他の関数（slotmem_get/put）とは異なり、コピーを行わずメモリを直接操作するためのインターフェースです。
