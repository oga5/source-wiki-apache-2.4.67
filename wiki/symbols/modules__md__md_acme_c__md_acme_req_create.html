md_acme_req_create

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme.c:130

目的

ACME プロトコルにおけるリクエスト処理のためのメモリプールを確保し、リクエスト状態（md_acme_req_t）を初期化する。

シグネチャまたは形状

static md_acme_req_t *md_acme_req_create(md_acme_t *acme, const char *method, const char *url)

入力と出力

  • **入力**:
  • acme: ACME 通信セッションを保持する md_acme_t 構造体へのポインタ。
  • method: HTTP メソッド（例: "POST", "GET"）。
  • url: リクエスト対象の URL。
  • **出力**:
  • 成功時: 初期化された md_acme_req_t 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • acme->p が有効な APR プールであること。
  • リクエスト処理中に確保されるメモリは、本関数で作成されたリクエスト専用プール（req->p）に紐付けられる必要がある。

アルゴリズムまたは振る舞い

1. acme->p を親として、リクエスト専用の APR プールを作成する。 2. md_acme_req_t 構造体をゼロクリアして確保する。 3. 構造体の各フィールドに引数および acme コンテキストの値を設定する。 4. md_json_create を呼び出し、リクエスト用の JSON コンテキストを初期化する。 5. md_result_make を呼び出し、初期状態（APR_SUCCESS）の結果コンテナを作成する。 6. 設定済みの req を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c:448 (POST リクエスト生成時)
  • modules/md/md_acme.c:472 (GET リクエスト生成時)
  • **呼び出し先**:
  • apr_pool_create
  • apr_pool_tag
  • apr_pcalloc
  • md_json_create
  • md_result_make
  • apr_pool_destroy (エラー発生時)

関連データ構造

  • md_acme_t: ACME 通信セッション状態。
  • md_acme_req_t: リクエストのライフサイクルと状態を管理する構造体。
  • apr_pool_t: リクエストごとのメモリ管理に使用。

失敗モードと境界条件

  • apr_pool_create が失敗した場合、NULL を返す。
  • apr_pcalloc が失敗した場合、作成したプールを破棄し NULL を返す。

保守担当者向けメモ

  • 本関数で作成されたリクエストプールは、後続の md_acme_req_done 関数で確実に破棄される設計となっている。
  • リクエストの再試行ロジックは md_acme_req_send が担うため、本関数は初期化のみに専念する。
