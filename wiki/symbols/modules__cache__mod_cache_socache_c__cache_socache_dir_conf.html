cache_socache_dir_conf

定義

  • **種別**: struct
  • **定義箇所**: modules/cache/mod_cache_socache.c:117

目的

mod_cache_socache モジュールにおいて、ディレクトリ単位でのキャッシュ動作を制御するための設定値を保持する構造体です。

シグネチャまたは形状

typedef struct cache_socache_dir_conf {
    apr_off_t max;              /* キャッシュファイルの最大サイズ */
    apr_time_t maxtime;         /* 最大有効期限 */
    apr_time_t mintime;         /* 最小有効期限 */
    apr_off_t readsize;         /* 一括キャッシュ試行時の最大データサイズ */
    apr_time_t readtime;        /* 一括キャッシュ試行時の最大時間 */
    unsigned int max_set :1;    /* max 設定フラグ */
    unsigned int maxtime_set :1;/* maxtime 設定フラグ */
    unsigned int mintime_set :1;/* mintime 設定フラグ */
    unsigned int readsize_set :1;/* readsize 設定フラグ */
    unsigned int readtime_set :1;/* readtime 設定フラグ */
} cache_socache_dir_conf;

入力と出力

  • **入力**: 設定ファイル（httpd.conf 等）で指定されたディレクティブ値。
  • **出力**: モジュール内の各処理関数（create_entity 等）が参照する設定コンテキスト。

前提条件と不変条件

  • 各設定値に対応する _set ビットフィールドは、設定が明示的に行われたかどうかを追跡するために使用されます。
  • apr_pcalloc を使用してメモリが確保されるため、初期状態ではすべてのフラグは 0（未設定）となります。

アルゴリズムまたは振る舞い

  • ディレクトリ単位の設定を保持し、リクエスト処理時に mod_cache から参照されます。
  • 設定の継承やマージの際に、base（親ディレクトリ設定）と add（現在の設定）を比較・統合するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_socache.c 内の各フック関数（行 360, 448, 786, 949, 1174, 1265 等）。
  • 設定の初期化およびマージ処理（行 1188, 1191, 1192）。
  • **呼び出し先**:
  • apr_pcalloc（メモリ確保時）。
  • 呼び出し先情報の詳細は、ソースコードの解析範囲内ではこれ以上の明示的な関数呼び出しは確認されていません。

関連データ構造

  • cache_socache_conf: サーバー単位の設定を保持する構造体。
  • cache_socache_object_t: キャッシュエントリのメモリ内表現。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc が失敗した場合、サーバーの起動またはリクエスト処理が停止する可能性があります。
  • 設定値の不整合: max や readsize に負の値が設定された場合の挙動は、本構造体単体では規定されていません。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を管理するため、mod_cache の設定マージロジックと密接に関連しています。
  • ビットフィールド（_set）の追加・変更を行う際は、マージ処理関数内でのビット演算が正しく行われているか確認してください。
