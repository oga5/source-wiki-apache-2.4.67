log_escape

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_forensic.c:122

目的

文字列内の特殊文字をエスケープし、指定されたバッファへ安全に書き込む。フォレンジックログ出力において、制御文字や特殊文字を %XX 形式に変換するために使用される。

シグネチャまたは形状

static char *log_escape(char *q, const char *e, const char *p)

入力と出力

  • q: 書き込み先のバッファポインタ。
  • e: バッファの終端ポインタ（書き込み上限）。
  • p: エスケープ対象のソース文字列。
  • 戻り値: 書き込み終了後の q の位置（ヌル終端文字の直前）。

前提条件と不変条件

  • q は e よりも小さい領域を指している必要がある。
  • TEST_CHAR マクロ（test_char.h）により、エスケープ対象文字が判定される。
  • バッファオーバーフローを防ぐため、ap_assert による厳格な境界チェックが行われる。

アルゴリズムまたは振る舞い

1. ソース文字列 p を終端まで走査する。 2. 各文字が T_ESCAPE_FORENSIC に該当する場合、% を書き込み、ap_bin2hex を用いて 2 桁の 16 進数に変換して書き込む。 3. 該当しない場合は、そのまま文字をコピーする。 4. 処理終了後、バッファの現在位置にヌル終端文字を付与する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/loggers/mod_log_forensic.c:175 (ヘッダーキーの処理)
  • modules/loggers/mod_log_forensic.c:177 (ヘッダー値の処理)
  • modules/loggers/mod_log_forensic.c:217 (リクエストラインの処理)
  • 呼び出し先:
  • ap_assert (マクロ)
  • TEST_CHAR (マクロ)
  • ap_bin2hex (Apache API)

関連データ構造

  • test_char.h: エスケープ対象文字の定義（T_ESCAPE_FORENSIC）。

失敗モードと境界条件

  • バッファ容量不足: q が e を超える場合、ap_assert によりプログラムが停止する（アサーション失敗）。
  • 空文字列: p が空の場合、何も書き込まずにヌル終端のみを行う。

保守担当者向けメモ

  • 本関数は r->pool から確保されたバッファに対して操作を行うことを前提としている。
  • ap_assert を多用しているため、呼び出し側で事前に count_string や count_headers を用いて十分なバッファサイズを確保しておくことが必須である。
