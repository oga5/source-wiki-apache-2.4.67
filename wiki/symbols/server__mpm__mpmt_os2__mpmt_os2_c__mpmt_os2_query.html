mpmt_os2_query

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:403

目的

MPM（マルチプロセッシングモジュール）の構成情報や制限値を照会するためのインターフェースを提供します。

シグネチャまたは形状

apr_status_t mpmt_os2_query(int query_code, int *result, apr_status_t *rv)

入力と出力

  • **入力**:
  • query_code: 照会する情報の種類（AP_MPMQ_* 定数）。
  • result: 照会結果を格納する整数へのポインタ。
  • rv: 処理のステータス（APR_SUCCESS または APR_ENOTIMPL）を格納する apr_status_t へのポインタ。
  • **出力**:
  • 関数自体は常に OK を返します。実際の照会結果は *result および *rv に書き込まれます。

前提条件と不変条件

  • *rv は初期値として APR_SUCCESS が設定されます。
  • 未知の query_code が渡された場合、*rv に APR_ENOTIMPL が設定されます。

アルゴリズムまたは振る舞い

query_code に応じて以下の値を *result に設定します。

  • AP_MPMQ_MAX_DAEMON_USED: 現在の ap_max_daemons_limit。
  • AP_MPMQ_IS_THREADED: AP_MPMQ_DYNAMIC（動的スレッドモデル）。
  • AP_MPMQ_IS_FORKED: AP_MPMQ_NOT_SUPPORTED（フォーク非対応）。
  • AP_MPMQ_HARD_LIMIT_DAEMONS: HARD_SERVER_LIMIT。
  • AP_MPMQ_HARD_LIMIT_THREADS: HARD_THREAD_LIMIT。
  • AP_MPMQ_MIN_SPARE_DAEMONS / MAX_SPARE_DAEMONS: 0。
  • AP_MPMQ_MAX_REQUESTS_DAEMON: ap_max_requests_per_child。
  • AP_MPMQ_GENERATION: ap_my_generation。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/mpmt_os2/mpmt_os2.c 内で ap_hook_mpm_query を介して登録されています。
  • **呼び出し先**: 直接的な呼び出し先はありません。

関連データ構造

  • HARD_SERVER_LIMIT
  • HARD_THREAD_LIMIT
  • ap_max_daemons_limit
  • ap_max_requests_per_child
  • ap_my_generation

失敗モードと境界条件

  • 未定義の query_code が指定された場合、*rv に APR_ENOTIMPL が設定され、*result は更新されません。

保守担当者向けメモ

  • 本関数は Apache の MPM クエリフック機構の一部です。新しい MPM 設定値を追加する場合は、switch 文にケースを追加してください。
