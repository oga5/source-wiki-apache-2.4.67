etag_end

定義

  • 種別: function
  • 定義箇所: modules/http/http_etag.c:79

目的

ETag 文字列の末尾を処理し、コンテンツネゴシエーション用の検証情報（VLV）の付与、または引用符による閉じ処理を行う。

シグネチャまたは形状

void etag_end(char *next, const char *vlv, apr_size_t vlv_len)

入力と出力

  • next: ETag 文字列の現在位置を指すポインタ。この位置から書き込みが行われる。
  • vlv: コンテンツネゴシエーション用の検証情報文字列（vlist_validator）。NULL の場合もある。
  • vlv_len: vlv の長さ。
  • 出力: なし（next が指すメモリ領域を直接更新する）。

前提条件と不変条件

  • next は、ETag 文字列を格納するのに十分なバッファ領域を指している必要がある。
  • vlv が存在する場合、next の位置に ; が挿入され、その後に vlv がコピーされる。
  • vlv が存在しない場合、next の位置に " が挿入され、文字列終端 \0 が付与される。

アルゴリズムまたは振る舞い

1. vlv が NULL でない場合:

  • next の位置に ; を書き込み、ポインタをインクリメントする。
  • apr_cpystrn を使用して、vlv を next 以降にコピーする。

    • 2. vlv が NULL の場合:

  • next の位置に " を書き込み、ポインタをインクリメントする。
  • next の位置に \0 を書き込み、文字列を終了させる。

関連データ構造

  • vlist_validator (VLV): コンテンツネゴシエーション時に使用される検証情報。

失敗モードと境界条件

  • next が指すバッファが不足している場合、メモリ破壊が発生する可能性がある（呼び出し元でのバッファ確保に依存）。
  • vlv が NULL の場合、必ず引用符で閉じられる。

保守担当者向けメモ

  • 本関数は ETag 生成の最終ステップを担当する。vlv が提供される場合はセミコロン区切りで情報を追加し、そうでない場合は標準的な ETag の閉じ括弧（"）を付与する。
  • apr_cpystrn を使用しているため、コピー後の文字列は必ずヌル終端される。
