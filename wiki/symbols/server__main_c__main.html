main

定義

  • 種別: function
  • 定義箇所: server/main.c:488

目的

Apache HTTP Server のメインエントリポイントとして、プロセスの初期化、コマンドライン引数の解析、設定ファイルの読み込み、およびサーバーのライフサイクル管理を行います。

シグネチャまたは形状

int main(int argc, const char * const argv[])

入力と出力

  • **入力**:
  • argc: コマンドライン引数の数。
  • argv: コマンドライン引数の文字列配列。
  • **出力**:
  • 正常終了時は OK (0)、エラー発生時は 1 などの終了ステータスを返します。

前提条件と不変条件

  • apr_app_initialize を含むプロセス初期化が完了している必要があります。
  • pconf (設定用メモリプール) は設定の再読み込み時にリセットされるため、設定関連データはここに保持される必要があります。
  • メモリ割り当て失敗時には abort_on_oom が呼び出され、プロセスが緊急停止します。

アルゴリズムまたは振る舞い

1. **初期化**: init_process を呼び出し、APR 環境とグローバルプールを構築します。 2. **引数解析**: apr_getopt を使用し、-D, -f, -t などのオプションを解析して動作モード (ap_run_mode) を決定します。 3. **設定読み込み**: ap_read_config で設定ファイルを読み込み、ap_process_config_tree で適用します。 4. **フック実行**: ap_run_pre_config や ap_run_post_config 等のフックを順次実行し、サーバーを構成します。 5. **メインループ**: do-while ループ内で MPM を呼び出し、サーバーの稼働を開始します。設定変更時にはループ内で設定を再読み込みし、MPM を再起動します。

呼び出し元と呼び出し先

  • **呼び出し元**: OS の起動プロセス（エントリポイントのため）。
  • **呼び出し先**:
  • init_process, destroy_and_exit_process
  • ap_read_config, ap_process_config_tree
  • ap_run_pre_config, ap_run_post_config, ap_run_mpm
  • apr_getopt, apr_pool_create 等の APR 関数群。
  • ※ツールによる参照情報には、本シンボルを呼び出す具体的な関数は含まれていません。

関連データ構造

  • process_rec: プロセス全体の管理構造体。
  • apr_pool_t: メモリ管理用プール (pconf, plog, ptemp 等)。
  • server_rec: サーバー設定を保持する構造体。
  • ap_conftree: 設定ファイルのツリー構造。

失敗モードと境界条件

  • 設定ファイルの読み込み失敗や、設定チェック (ap_run_check_config) の失敗時には destroy_and_exit_process を呼び出し、エラーログを出力して終了します。
  • メモリ不足時は abort_on_oom により即座にプロセスが終了します。
  • 不正なコマンドライン引数が指定された場合は usage() を呼び出し、ヘルプを表示して終了します。

保守担当者向けメモ

  • main() 内の do-while ループは設定の再読み込みをサポートしていますが、設定ファイルを二度読みするなどの冗長な処理が含まれており、将来的なリファクタリングが推奨されています。
  • プラットフォーム依存の条件付きコンパイル（WIN32, NETWARE 等）が多用されているため、修正時はクロスプラットフォームへの影響に注意してください。
