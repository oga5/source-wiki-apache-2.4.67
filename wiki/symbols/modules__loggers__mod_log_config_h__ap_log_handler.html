ap_log_handler

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_log_config.h:54

目的

ログフォーマット文字列内の特定のタグ（例: %h）に対応する値の抽出処理を保持し、ログ記録プロセスにおいて実行可能な形式で管理するために使用されます。

シグネチャまたは形状

typedef struct ap_log_handler {
    ap_log_handler_fn_t *func;
    int want_orig_default;
} ap_log_handler;

入力と出力

  • func: ログフォーマットの各要素を処理するための関数ポインタ（ap_log_handler_fn_t 型）。
  • want_orig_default: ログ記録においてデフォルトの挙動を要求するかどうかを示すフラグ。

前提条件と不変条件

  • func は、ログフォーマットの解析時に呼び出される有効な関数を指している必要があります。
  • 外部モジュールが ap_register_log_handler を通じてこの構造体を登録することで、ログ記録エンジンが利用可能になります。

アルゴリズムまたは振る舞い

1. ログ記録エンジン（mod_log_config）は、設定されたログフォーマット文字列を解析します。 2. 各タグに対応する ap_log_handler をハッシュテーブルから検索します。 3. 該当するハンドラが見つかった場合、その func を呼び出してログ出力用の文字列を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_log_config.c 内のログ処理ロジック（apr_hash_get を使用してハッシュテーブルから取得）。
  • **呼び出し先**: func メンバに格納された関数（具体的な呼び出し先は登録されたモジュールに依存します）。

関連データ構造

  • ap_log_handler_fn_t: func メンバの型定義。
  • apr_hash_t: ログハンドラを管理するために mod_log_config 内で使用されるハッシュテーブル。

失敗モードと境界条件

  • 登録されていないタグがフォーマット文字列に含まれている場合、ログ出力処理が失敗または無視される可能性があります。
  • func が NULL の場合、実行時に未定義の動作を引き起こす可能性があるため、登録時の検証が必要です。

保守担当者向けメモ

  • 本構造体は mod_log_config の拡張ポイントです。新しいログフォーマットタグを追加する場合は、この構造体を介してハンドラを登録してください。
  • 呼び出し元および呼び出し先の詳細な依存関係については、mod_log_config.c の実装を参照してください。
