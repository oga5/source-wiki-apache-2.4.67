dav_locktoken

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/lock.c:99

目的

WebDAV ロックを識別するためのトークンを保持するラッパー構造体です。

シグネチャまたは形状

struct dav_locktoken {
    apr_uuid_t uuid;
};

入力と出力

  • **入力**: apr_uuid_t 型の UUID を内部に保持します。
  • **出力**: ロックの照合、比較、および永続化の際の識別子として利用されます。

前提条件と不変条件

  • apr_uuid_t を介して一意性が保証されることが期待されます。
  • ロック操作の文脈において、この構造体へのポインタは const として扱われることが一般的です。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持に特化しており、ロックトークンの生成、解析、比較には dav_fs_parse_locktoken、dav_fs_format_locktoken、dav_fs_compare_locktoken などの専用関数が使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/lock.c 内のロック管理関数群（dav_fs_load_lock_record 等）が、ロック情報の構築や比較のために本構造体を参照します。
  • **呼び出し先**: 内部で apr_uuid_t を保持するため、APR の UUID 関連関数が間接的に利用されます。
  • *注記*: 呼び出し元・呼び出し先の詳細なコールグラフは、ソースコードの静的解析結果に基づいています。

関連データ構造

  • dav_lock_discovery: 直接ロック情報を保持する構造体で、本トークンをメンバとして含みます。
  • dav_lock_indirect: 間接ロック情報を保持する構造体で、本トークンをメンバとして含みます。

失敗モードと境界条件

  • 本構造体自体はデータコンテナであるため、メモリ割り当て失敗時を除き、構造体固有の失敗モードはありません。
  • 比較や解析の過程で、無効な UUID が渡された場合の挙動は、利用する関数（dav_fs_compare_locktoken 等）の仕様に依存します。

保守担当者向けメモ

  • 本構造体は mod_dav のロックプロバイダインターフェースにおける識別子として機能します。
  • 構造体の拡張や変更を行う際は、dav_fs_parse_locktoken などの関連関数が正しく UUID を処理できているか確認してください。
