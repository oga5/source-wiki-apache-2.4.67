run_watchdog

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_ocsp.c:157

目的

mod_watchdog を利用して、OCSP レスポンスの定期的な更新処理（md_ocsp_renew）を実行し、次回の実行タイミングをスケジュールする。

シグネチャまたは形状

static apr_status_t run_watchdog(int state, void *baton, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • state: mod_watchdog から通知される現在の状態（AP_WATCHDOG_STATE_STARTING, RUNNING, STOPPING）。
  • baton: md_ocsp_ctx_t 型のコンテキストへのポインタ。
  • ptemp: 一時的なメモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • 常に APR_SUCCESS を返す。

前提条件と不変条件

  • baton は有効な md_ocsp_ctx_t 構造体を指している必要がある。
  • mod_watchdog がロードされ、初期化されていることが前提となる。

アルゴリズムまたは振る舞い

1. **状態遷移の処理**:

  • AP_WATCHDOG_STATE_STARTING / STOPPING: ログ出力のみを行う。
  • AP_WATCHDOG_STATE_RUNNING:
  • next_run_default() を呼び出し、デフォルトの次回実行時刻を取得する。
  • md_ocsp_renew() を呼び出し、各証明書の OCSP レスポンスを更新する（next_run はこの関数内で更新される）。
  • 現在時刻と next_run の差分から wait_time を計算する。
  • wd_set_interval() を呼び出し、次回の実行をスケジュールする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_ocsp.c:275 (wd_register_callback 経由)
  • modules/md/mod_md_ocsp.c:192 (自身による再帰的スケジュール)
  • modules/md/mod_md_drive.c:297 および 406 (直接的な呼び出し元として確認)
  • **呼び出し先**:
  • md_ocsp_renew
  • next_run_default
  • wd_set_interval
  • ap_log_error

関連データ構造

  • md_ocsp_ctx_t: Watchdog の実行コンテキストを保持する構造体。

失敗モードと境界条件

  • md_ocsp_renew が失敗した場合の具体的なエラーハンドリングは本関数内には記述されていない。
  • wait_time が負の値になる場合の挙動は、wd_set_interval の実装に依存する。

保守担当者向けメモ

  • 本関数は mod_watchdog のコールバックとして動作するため、スレッドセーフティおよび mod_watchdog のライフサイクル管理に注意すること。
  • md_ocsp_renew が next_run を更新するロジックが、OCSP 更新の頻度を決定する主要な要因となる。
