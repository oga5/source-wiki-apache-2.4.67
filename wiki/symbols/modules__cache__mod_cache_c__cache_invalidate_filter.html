cache_invalidate_filter

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:1670

目的

PUT、POST、DELETE メソッドのリクエストに応じて、キャッシュされたエンティティを無効化（削除）する出力フィルタ。

シグネチャまたは形状

static apr_status_t cache_invalidate_filter(ap_filter_t *f, apr_bucket_brigade *in)

入力と出力

  • **入力**:
  • f: 現在のフィルタコンテキスト。f->ctx に cache_request_rec が格納されていることを期待する。
  • in: 処理対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: フィルタチェーンの次段 (f->next) の実行結果を返す。

前提条件と不変条件

  • f->ctx が cache_request_rec 構造体として初期化されていること。
  • フィルタ実行後、自身をフィルタチェーンから削除する。

アルゴリズムまたは振る舞い

1. f->ctx から cache_request_rec を取得する。 2. コンテキストが存在しない場合、警告をログ出力し、フィルタを削除して終了する。 3. リクエストのステータスコード (r->status) が 299 を超える場合、無効化処理を行わずに終了する。 4. ステータスコードが 299 以下の場合:

  • cache_invalidate を呼び出し、キャッシュエントリを無効化する。
  • cache_run_cache_status を呼び出し、無効化イベントを通知する。

    • 5. ap_remove_output_filter(f) を呼び出し、自身をフィルタチェーンから除去する。 6. ap_pass_brigade(f->next, in) を呼び出し、後続のフィルタへ制御を渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache.c:2692 に参照あり。
  • **呼び出し先**:
  • cache_invalidate (キャッシュエントリの削除)
  • cache_run_cache_status (ステータス通知フック)
  • ap_remove_output_filter (フィルタの除去)
  • ap_pass_brigade (後続フィルタの実行)

関連データ構造

  • cache_request_rec: キャッシュリクエストのコンテキスト情報。
  • request_rec: HTTP リクエスト情報。

失敗モードと境界条件

  • f->ctx が NULL の場合: 意図しない手動設定と判断し、無効化処理をスキップしてフィルタを削除する。
  • r->status > 299 の場合: キャッシュの無効化は行われない。

保守担当者向けメモ

  • 本フィルタは CACHE_INVALIDATE フィルタとして登録される。
  • r->pool を使用してメモリ管理が行われるため、リクエスト終了時にリソースは自動解放される。
