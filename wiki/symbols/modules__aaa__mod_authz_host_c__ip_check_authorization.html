ip_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:146

目的

クライアントの IP アドレスが、設定で許可されたサブネットリストに含まれているかを検証し、アクセス認可の可否を判定します。

シグネチャまたは形状

static authz_status ip_check_authorization(request_rec *r,
                                           const char *require_line,
                                           const void *parsed_require_line)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報を保持する request_rec 構造体。r->useragent_addr を使用してクライアント IP を取得します。
  • require_line: 設定ファイル上の文字列（未使用）。
  • parsed_require_line: ip_parse_config によって事前に解析された apr_ipsubnet_t 構造体の配列（NULL 終端）。
  • **出力**:
  • AUTHZ_GRANTED: クライアント IP がいずれかのサブネットに一致した場合。
  • AUTHZ_DENIED: 一致するサブネットが見つからなかった場合。

前提条件と不変条件

  • parsed_require_line は、apr_ipsubnet_t 構造体へのポインタの配列であり、NULL で終端されている必要があります。
  • r->useragent_addr は有効なクライアントアドレス情報を含んでいる必要があります。

アルゴリズムまたは振る舞い

1. parsed_require_line を apr_ipsubnet_t ** 型としてキャストし、サブネットリストの先頭を取得します。 2. リストの終端（NULL）に達するまで、各サブネットに対して apr_ipsubnet_test を実行します。 3. apr_ipsubnet_test が真を返した場合、直ちに AUTHZ_GRANTED を返します。 4. リスト内のすべてのサブネットを走査しても一致しない場合、AUTHZ_DENIED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_host.c:332 にて authz_ip_provider 構造体の一部として登録されています。
  • **呼び出し先**:
  • apr_ipsubnet_test (APR ライブラリ関数): IP アドレスとサブネットの照合に使用。

関連データ構造

  • request_rec: Apache のリクエスト状態を保持する構造体。
  • apr_ipsubnet_t: APR が提供する IP サブネット情報を保持する構造体。

失敗モードと境界条件

  • parsed_require_line が空（NULL）の場合、ループは実行されず、常に AUTHZ_DENIED を返します。
  • r->useragent_addr が未設定または無効な場合、apr_ipsubnet_test の挙動は APR の実装に依存します。

保守担当者向けメモ

  • apr_ipsubnet_test は本来 const な引数を受け取るべきですが、現状の API 仕様によりキャストが必要となっています。
  • 本関数は mod_authz_core から呼び出される認可プロバイダーとして実装されています。ログ出力は authz_core 側で行われるため、本関数内でのログ出力は省略されています。
