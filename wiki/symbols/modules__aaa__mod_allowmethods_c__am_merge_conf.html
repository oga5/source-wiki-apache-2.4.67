am_merge_conf

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_allowmethods.c:88

目的

Apache HTTP Server のディレクトリ設定において、親ディレクトリの設定（base）と現在のディレクトリの設定（add）をマージし、新しい設定構造体を生成する。

シグネチャまたは形状

static void *am_merge_conf(apr_pool_t *pool, void *a, void *b)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する APR プール。
  • a: 親ディレクトリの設定 (am_conf_t *)。
  • b: 現在のディレクトリの設定 (am_conf_t *)。
  • **出力**:
  • マージされた新しい am_conf_t 構造体へのポインタ。

前提条件と不変条件

  • a および b は am_conf_t 型としてキャスト可能であること。
  • pool は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、新しい am_conf_t 構造体用のメモリを確保する。 2. add->allowed_set が真（現在のディレクトリで設定が明示されている）の場合、add の設定値を優先して新しい構造体にコピーする。 3. add->allowed_set が偽の場合、base（親）の設定値を新しい構造体にコピーする。 4. マージ済みの構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_allowmethods.c:153 (モジュール構造体 allowmethods_module の定義内にて参照されている)。
  • **呼び出し先**: apr_palloc (APR ライブラリ関数)。

関連データ構造

  • am_conf_t: ディレクトリごとの許可メソッドリストと設定状態を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある。
  • a または b が NULL の場合の挙動については、ソースコード上では明示的なチェックは行われていない。

保守担当者向けメモ

  • 本関数は Apache の設定マージ機構の一部として機能する。
  • allowed_set フラグにより、設定の継承と上書きを制御している。設定をリセットする際は、このフラグの扱いが重要となる。
