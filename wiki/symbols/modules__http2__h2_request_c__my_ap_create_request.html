my_ap_create_request

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:233

目的

Apache HTTP Server のバージョンが古い環境において、HTTP/2 リクエスト処理に必要な request_rec 構造体をメモリ確保から初期化まで手動で行うためのフォールバック関数です。

シグネチャまたは形状

request_rec *my_ap_create_request(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（現在の接続情報）
  • **出力**: 初期化済みの request_rec *（Apache のリクエスト処理パイプラインで使用される構造体）

前提条件と不変条件

  • 入力される conn_rec *c は有効である必要があります。
  • 関数内で apr_pool_create を使用して新しいメモリプールを生成するため、リクエスト終了時にこのプールが適切に破棄されることが前提となります。
  • r->status は初期値として HTTP_OK が設定されます。

アルゴリズムまたは振る舞い

1. c->pool を親として新しいメモリプール p を作成し、タグ付けを行います。 2. apr_pcalloc を使用して request_rec 構造体をゼロクリアした状態で確保します。 3. AP_READ_REQUEST_ENTRY マクロを呼び出し、リクエスト開始のトレースを行います。 4. 接続情報 (connection, server) を設定し、許可メソッドリスト、各種ヘッダーテーブル (headers_in, headers_out 等)、および subprocess_env を初期化します。 5. ap_create_request_config を呼び出し、リクエスト設定を初期化します。 6. 接続のフィルタ (input_filters, output_filters) をリクエストに継承させます。 7. ap_run_create_request フックを実行し、モジュール固有の初期化をトリガーします。 8. per_dir_config をサーバーのデフォルト設定で初期化します。 9. リクエストのステータス、読み込み状態、ユーザーエージェント情報を設定して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_request.c:383（h2_create_request_rec 内から呼び出されることが確認されています）
  • **呼び出し先**:
  • apr_pool_create, apr_pool_tag, apr_pcalloc, apr_table_make
  • ap_make_method_list, ap_create_request_config, ap_run_create_request
  • AP_READ_REQUEST_ENTRY (マクロ)

関連データ構造

  • request_rec: Apache の主要なリクエスト構造体。
  • conn_rec: 接続状態を保持する構造体。
  • apr_pool_t: メモリ管理用プール。
  • apr_table_t: ヘッダーや環境変数を保持するテーブル。

失敗モードと境界条件

  • メモリ不足時には apr_pool_create や apr_pcalloc が失敗し、プロセスが終了する可能性があります。
  • ap_run_create_request フック内でエラーが発生した場合の挙動は、登録されている各モジュールの実装に依存します。

保守担当者向けメモ

  • 本関数は AP_MODULE_MAGIC_AT_LEAST 等の条件分岐を伴う環境での互換性維持を目的としています。
  • r->the_request は NULL で初期化されています。
  • r->used_path_info には AP_REQ_DEFAULT_PATH_INFO が設定されており、後続の処理で変更される可能性があります。
