clean_child_exit

定義

  • 種別: function
  • 定義箇所: server/mpm/prefork/prefork.c:219

目的

子プロセスの終了処理を安全に実行し、プロセスを終了させる。

シグネチャまたは形状

static void clean_child_exit(int code)

入力と出力

  • **入力**: code (終了ステータスコード)
  • **出力**: なし（プロセスを終了させるため戻り値はない）

前提条件と不変条件

  • retained グローバル変数が初期化されていること。
  • pchild メモリプールが有効である場合、破棄される。
  • my_bucket が現在のプロセスに関連付けられていること。

アルゴリズムまたは振る舞い

1. MPM の状態を AP_MPMQ_STOPPING に設定する。 2. SIGHUP および SIGTERM シグナルを無視するように設定する。 3. code が 0 の場合、ap_run_child_stopping フックを実行する。 4. pchild が存在する場合、メモリプールを破棄する。 5. one_process モードの場合、prefork_note_child_killed を呼び出して終了を記録する。 6. ap_mpm_pod_close を呼び出し、POD（Pipe of Death）を閉じる。 7. chdir_for_gprof を呼び出し、プロファイリング用のディレクトリ移動を行う。 8. exit(code) を呼び出し、プロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内には直接の呼び出し元は記載されていません。
  • **呼び出し先**:
  • ap_run_child_stopping
  • apr_pool_destroy
  • prefork_note_child_killed
  • ap_mpm_pod_close
  • chdir_for_gprof
  • exit
  • apr_signal

関連データ構造

  • retained: MPM の状態を保持するグローバル構造体。
  • pchild: 子プロセス固有のメモリプール。
  • my_bucket: 現在のプロセスが属するバケット（POD を含む）。

失敗モードと境界条件

  • pchild が NULL の場合、メモリプールの破棄はスキップされる。
  • one_process が真でない場合、prefork_note_child_killed は呼び出されない。

保守担当者向けメモ

  • 本関数は exit() を呼び出すため、呼び出し後は制御が戻りません。
  • event.c 等の他 MPM 実装にも同名の関数が存在しますが、本ページは prefork.c の実装に基づいています。ツールが提示した event.c の呼び出し元情報は、別ファイルの実装を参照している可能性が高いため注意してください。
