cgid_maint

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgid.c:282

目的

CGI デーモンプロセスのライフサイクル管理（監視、再起動、終了処理）を行うコールバック関数です。APR の other_child メカニズムを通じて、デーモンプロセスの状態変化に応じた保守処理を実行します。

シグネチャまたは形状

void cgid_maint(int reason, void *data, apr_wait_t status)

入力と出力

  • reason: 発生したイベントの種類（APR_OC_REASON_DEATH, APR_OC_REASON_RESTART, APR_OC_REASON_LOST, APR_OC_REASON_UNREGISTER）。
  • data: 監視対象の apr_proc_t 構造体へのポインタ。
  • status: プロセスの終了状態またはエラーコード。
  • 出力: なし（副作用としてデーモンの再起動や終了、ログ出力を行う）。

前提条件と不変条件

  • apr_proc_other_child_register によって、cgid_maint が対象プロセスと関連付けられている必要があります。
  • root_pool および root_server が有効な状態で初期化されている必要があります。

アルゴリズムまたは振る舞い

1. **APR_OC_REASON_DEATH**: プロセスが終了した場合、サーバーが停止中でなければ cgid_start を呼び出してデーモンを再起動します。起動エラー時はログを出力します。 2. **APR_OC_REASON_RESTART**: サーバーの再起動または停止に伴い、登録を解除します。 3. **APR_OC_REASON_LOST**: プロセスの状態が不明になった場合、cgid_start を呼び出してデーモンを再起動します。 4. **APR_OC_REASON_UNREGISTER**: モジュールのクリーンアップ時、デーモンに SIGHUP を送信して終了させ、Unix ドメインソケットファイルを削除します。

呼び出し元と呼び出し先

  • 呼び出し元: apr_proc_other_child_register により、APR のプロセス管理サブシステムからイベント発生時に呼び出されます。
  • 呼び出し先: cgid_start（再起動時）、kill（終了時）、unlink（ソケット削除時）、ap_log_error（ログ出力時）。

関連データ構造

  • apr_proc_t: 監視対象のプロセス情報。
  • root_pool, root_server: モジュール全体で共有されるグローバルなプールおよびサーバー設定。
  • sockname: Unix ドメインソケットのパス名。

失敗モードと境界条件

  • ap_mpm_query が失敗した場合、安全側に倒してデーモンの再起動を行わない挙動をとります。
  • unlink が失敗した場合（ENOENT 以外）、エラーログを出力します。
  • kill による終了シグナル送信後、デーモンが即座に終了しない場合の待機処理は本関数内には含まれていません。

保守担当者向けメモ

  • 本関数は apr_proc_other_child_register のコールバックとして登録されます。
  • pcgi プールのクリーンアップ時に APR_OC_REASON_UNREGISTER がトリガーされるため、デーモンの生存期間とメモリプールのライフサイクルが密接に関連しています。
  • ソケットファイルの削除は、作成時と同じ権限を維持するためにこのタイミングで行う必要があります。
