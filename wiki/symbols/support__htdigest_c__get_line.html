get_line

定義

  • 種別: function
  • 定義箇所: support/htdigest.c:100

目的

ファイルストリームから1行を読み込み、指定されたバッファに格納する。

シグネチャまたは形状

static int get_line(char *s, int n, apr_file_t *f)

入力と出力

  • **入力**:
  • s: 読み込んだ文字列を格納するバッファ。
  • n: バッファ s のサイズ。
  • f: 読み込み対象の apr_file_t ファイルポインタ。
  • **出力**:
  • 成功時（行の読み込み完了）: 0 を返す。
  • 失敗時（EOF またはエラー発生時）: 1 を返す。

前提条件と不変条件

  • バッファ s は、少なくとも n バイトの領域を確保していること。
  • n は、終端文字および改行文字を考慮して、最低限のバッファサイズ（2バイト以上）を確保していること。

アルゴリズムまたは振る舞い

1. i を 0 に初期化する。 2. i < (n - 2) かつ apr_file_getc が成功し、かつ読み込んだ文字が改行（\n）でない間、文字を s[i] に格納し i をインクリメントする。 3. 読み込んだ文字が改行（\n）であれば、それを s[i] に格納し i をインクリメントする。 4. s[i] にヌル終端文字 \0 を書き込む。 5. apr_file_getc の結果が APR_SUCCESS であれば 0 を返し、それ以外（EOF やエラー）であれば 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htdigest.c:265 にて、ファイルから行を読み込むループ内で呼び出されている。
  • **呼び出し先**: apr_file_getc (APRライブラリ関数)。
  • **注意**: ツールが報告した modules/http2/h2_c2_filter.c および modules/ssl/ssl_util_ocsp.c での呼び出しは、同名の別関数である可能性が高い（シグネチャが異なるため）。

関連データ構造

  • apr_file_t: APR のファイルハンドル構造体。

失敗モードと境界条件

  • **EOF**: ファイルの終端に達した場合、apr_file_getc が失敗し、関数は 1 を返す。
  • **バッファオーバーフロー**: n - 2 文字を超えて読み込もうとした場合、ループが終了し、それ以上の文字は読み込まれない。
  • **エラー**: ファイル読み込み中にエラーが発生した場合、1 を返す。

保守担当者向けメモ

  • 本関数は htdigest ユーティリティ専用の簡易的な行読み込み処理である。
  • 戻り値の 1 はエラーだけでなく EOF も含むため、呼び出し側で詳細な状態判別が必要な場合は注意すること。
