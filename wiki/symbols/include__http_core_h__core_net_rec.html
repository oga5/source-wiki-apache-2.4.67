core_net_rec

定義

  • 種別: struct
  • 定義箇所: include/http_core.h:795

目的

HTTP接続に関連するネットワークリソースおよびフィルタコンテキストを保持し、コアフィルタチェーン内でのデータ処理を管理する。

シグネチャまたは形状

typedef struct core_net_rec {
    apr_socket_t *client_socket;
    conn_rec *c;
    core_output_filter_ctx_t *out_ctx;
    core_ctx_t *in_ctx;
} core_net_rec;

入力と出力

  • **入力**: フィルタチェーンのコンテキスト（f->ctx）として初期化時に受け取る。
  • **出力**: 保持する client_socket や各コンテキストを通じて、HTTPの入出力フィルタ処理に寄与する。

前提条件と不変条件

  • c (conn_rec) は有効な接続レコードを指している必要がある。
  • out_ctx および in_ctx は、それぞれ出力フィルタおよび入力フィルタのライフサイクルと同期して管理される必要がある。

アルゴリズムまたは振る舞い

本構造体は、フィルタチェーンのコンテキストとして機能する。server/core_filters.c および server/protocol.c において、フィルタ関数が f->ctx から本構造体を取り出し、クライアントソケットや接続情報、および入出力用のコンテキスト（in_ctx, out_ctx）にアクセスすることで、HTTPプロトコルのフィルタ処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • server/core.c:5307
  • server/core_filters.c:95
  • server/core_filters.c:456
  • server/protocol.c:1572
  • **呼び出し先**:
  • apr_socket_t (APRライブラリ)
  • conn_rec (httpd基本構造体)
  • core_output_filter_ctx_t
  • core_ctx_t

関連データ構造

  • conn_rec: 接続全体の状態を保持する構造体。
  • core_output_filter_ctx_t: 出力フィルタ固有のコンテキスト。
  • core_ctx_t: 入力フィルタ固有のコンテキスト。

失敗モードと境界条件

  • f->ctx が初期化されていない状態でアクセスした場合、NULLポインタ参照が発生する可能性がある。
  • 接続が既に閉じられている場合、client_socket は無効な状態となる。

保守担当者向けメモ

  • 本構造体は core モジュールのフィルタ処理の要である。フィルタのコンテキスト管理に変更を加える際は、server/core_filters.c および server/protocol.c での参照整合性に注意すること。
  • in_ctx および out_ctx のメモリ管理は、関連するリクエストまたは接続のプール（pool）に依存している。
