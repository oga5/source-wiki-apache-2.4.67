ev_init

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_proxy_session.c:1230

目的

HTTP/2 プロキシセッションの初期化イベントを処理し、現在のセッション状態に基づいて後続の動作状態へ遷移させる。

シグネチャまたは形状

static void ev_init(h2_proxy_session *session, int arg, const char *msg)

入力と出力

  • session: 操作対象の h2_proxy_session 構造体へのポインタ。
  • arg: イベントに関連する整数値（本関数内では未使用）。
  • msg: イベントに関連する文字列メッセージ（本関数内では未使用）。
  • 出力: なし（副作用として session->state を更新）。

前提条件と不変条件

  • session が有効なポインタであること。
  • session->state が H2_PROXYS_ST_INIT である場合にのみ状態遷移が発生する。

アルゴリズムまたは振る舞い

1. session->state が H2_PROXYS_ST_INIT であるかを確認する。 2. session->streams（ハッシュテーブル）が空かどうかを判定する。

  • 空の場合: transit 関数を呼び出し、状態を H2_PROXYS_ST_IDLE へ遷移させる。
  • 空でない場合: transit 関数を呼び出し、状態を H2_PROXYS_ST_BUSY へ遷移させる。

    • 3. H2_PROXYS_ST_INIT 以外の状態であれば、何も行わず終了する（nop）。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_proxy_session.c:1500（直接的な呼び出しを確認）。
  • 呼び出し先: transit（状態遷移を実行する内部関数）。

関連データ構造

  • h2_proxy_session: セッションの状態およびストリーム管理を行う構造体。
  • h2_proxys_state: セッションの状態を定義する列挙型（H2_PROXYS_ST_INIT, H2_PROXYS_ST_IDLE, H2_PROXYS_ST_BUSY）。

失敗モードと境界条件

  • session->streams が初期化されていない場合、h2_proxy_ihash_empty の挙動に依存する（本ソースからは未定義）。
  • transit 関数が内部でエラーを発生させる可能性については、本ソースからは不明。

保守担当者向けメモ

  • 本関数はイベント駆動状態遷移の一部として実装されている。
  • arg および msg 引数は現在の実装では使用されていないが、将来的なデバッグログ出力や拡張のために保持されている可能性がある。
