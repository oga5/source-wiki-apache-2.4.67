server_signature_e

定義

  • 種別: enum
  • 定義箇所: include/http_core.h:517

目的

サーバーが生成するエラーページ等のフッターに付与する「サーバー署名（Server Signature）」の表示状態を制御するための列挙型です。

シグネチャまたは形状

typedef enum {
    srv_sig_unset,    /* 未設定 */
    srv_sig_off,      /* 無効 */
    srv_sig_on,       /* 有効（サーバー名とバージョンを表示） */
    srv_sig_withmail  /* 有効（サーバー名、バージョン、および管理者メールアドレスを表示） */
} server_signature_e;

入力と出力

  • 入力: ServerSignature ディレクティブの設定値。
  • 出力: サーバーが生成するレスポンスのフッターにおける署名の表示形式を決定する状態値。

前提条件と不変条件

  • 設定値は srv_sig_unset から始まり、設定ファイルによる解析を経て他のいずれかの状態へ遷移します。

アルゴリズムまたは振る舞い

  • srv_sig_off: 署名を非表示にします。
  • srv_sig_on: サーバーのバージョン情報を含む標準的な署名を表示します。
  • srv_sig_withmail: サーバー情報に加え、設定された管理者メールアドレス（ServerAdmin）を付加して表示します。

呼び出し元と呼び出し先

  • 呼び出し元: include/http_core.h:595 にて core_server_config 構造体のメンバ server_signature として使用されています。
  • 呼び出し先: 該当ソースコード内での具体的な関数呼び出し関係は提供されていません。

関連データ構造

  • core_server_config: サーバー全体の設定を保持する構造体であり、本列挙型をメンバとして保持します。

失敗モードと境界条件

  • srv_sig_unset のまま処理が進んだ場合、デフォルトの挙動（通常は非表示）が適用される必要があります。

保守担当者向けメモ

  • 本列挙型は ServerSignature ディレクティブの挙動と直結しています。新しい表示形式を追加する場合は、この列挙型を拡張し、対応する表示ロジックを modules/http/http_core.c に実装してください。
