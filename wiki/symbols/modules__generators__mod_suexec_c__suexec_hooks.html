suexec_hooks

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_suexec.c:124

目的

Apache HTTP Server の起動およびリクエスト処理フェーズにおいて、suEXEC 関連のフック関数を登録する。

シグネチャまたは形状

static void suexec_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p (リソース割り当て用のメモリプール)
  • **出力**: なし（戻り値は void）

前提条件と不変条件

  • 本関数は Apache のモジュール初期化プロセスの一部として呼び出される必要がある。
  • 登録されるフック関数（get_suexec_id_doer および suexec_post_config）が同一ファイル内に定義されていること。

アルゴリズムまたは振る舞い

1. ap_hook_get_suexec_identity を呼び出し、get_suexec_id_doer を APR_HOOK_MIDDLE の優先度で登録する。これにより、CGI 実行時に実行ユーザー ID を解決するフックが有効化される。 2. ap_hook_post_config を呼び出し、suexec_post_config を APR_HOOK_MIDDLE の優先度で登録する。これにより、サーバー起動後の設定確認フェーズで suEXEC の有効状態がログ出力される。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のモジュール読み込みプロセス（modules/generators/mod_suexec.c:138 にて登録）。
  • **呼び出し先**:
  • ap_hook_get_suexec_identity (Apache API)
  • get_suexec_id_doer (本モジュール内の関数)
  • ap_hook_post_config (Apache API)
  • suexec_post_config (本モジュール内の関数)

関連データ構造

  • apr_pool_t: Apache のメモリ管理用プール構造体。

失敗モードと境界条件

  • 本関数自体はエラーを返さないが、フック登録時にメモリ不足等の低レイヤーエラーが発生した場合、Apache の起動プロセスが中断される可能性がある。

保守担当者向けメモ

  • 本関数は mod_suexec のエントリーポイントの一つであり、フックの追加や順序変更を行う際は、APR_HOOK_MIDDLE の優先度が他のモジュールと競合しないか注意すること。
