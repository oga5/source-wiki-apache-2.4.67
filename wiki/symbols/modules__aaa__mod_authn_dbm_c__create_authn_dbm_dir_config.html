create_authn_dbm_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_dbm.c:57

目的

ディレクトリごとの認証設定を保持する authn_dbm_config_rec 構造体をメモリ上に確保し、初期値を設定する。

シグネチャまたは形状

static void *create_authn_dbm_dir_config(apr_pool_t *p, char *d)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体を割り当てるためのメモリプール。
  • char *d: 設定対象のディレクトリパス（本関数内では未使用）。
  • **出力**:
  • void *: 初期化済みの authn_dbm_config_rec 構造体へのポインタ。

前提条件と不変条件

  • apr_palloc が成功すること。
  • 返却される構造体の pwfile は NULL で初期化され、dbmtype は "default" に設定される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、authn_dbm_config_rec サイズ分のメモリをプール p から確保する。 2. 確保した構造体の pwfile メンバを NULL に設定する。 3. 確保した構造体の dbmtype メンバを "default" という文字列リテラルで初期化する。 4. 初期化した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_dbm.c:225（モジュール定義内のディレクトリ設定作成ハンドラとして参照されている）。
  • **呼び出し先**: apr_palloc（APR メモリ管理関数）。

関連データ構造

  • authn_dbm_config_rec: 本関数が初期化するディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある（Apache の標準的なメモリ管理動作に依存）。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出されるディレクトリ設定のコンストラクタです。AuthDBMUserFile や AuthDBMType ディレクティブによって後から値が上書きされることを前提としています。
