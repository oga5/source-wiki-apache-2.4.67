sig_term

定義

  • 種別: function
  • 定義箇所: os/unix/unixd.c:460

目的

MPM（Multi-Processing Module）の停止シグナル（SIGTERM または AP_SIG_GRACEFUL_STOP）を受信した際に、サーバーの停止状態を管理するシグナルハンドラとして機能します。

シグネチャまたは形状

static void sig_term(int sig)

入力と出力

  • **入力**:
  • sig: 受信したシグナル番号。
  • **出力**: なし（void）。

前提条件と不変条件

  • retained_data が初期化されている必要があります。初期化されていない場合、関数は即座に終了します。
  • retained_data は、サーバーのライフサイクル全体で生存するグローバルな状態保持構造体です。

アルゴリズムまたは振る舞い

1. retained_data が存在しない場合は終了します。 2. retained_data->mpm_state を AP_MPMQ_STOPPING に設定します。 3. すでに停止処理が保留中であり、かつ今回のシグナルが「強制停止（ungraceful）」または「グレースフル停止（AP_SIG_GRACEFUL_STOP）」の条件を満たす場合は、重複処理を避けて終了します。 4. retained_data->shutdown_pending を 1 に設定し、停止を要求します。 5. 受信したシグナルが AP_SIG_GRACEFUL_STOP でない場合、retained_data->is_ungraceful を 1 に設定し、強制停止フラグを立てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • os/unix/unixd.c 内のシグナル登録処理（apr_signal 等）。
  • server/mpm/mpmt_os2/mpmt_os2.c および mpmt_os2_child.c。
  • server/mpm/netware/mpm_netware.c。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行いませんが、retained_data 構造体のメンバ変数を更新します。

関連データ構造

  • ap_unixd_mpm_retained_data（retained_data として参照）: MPM の再起動を跨いで生存する状態保持構造体。

失敗モードと境界条件

  • retained_data が NULL の場合、何も行わずに正常終了します。
  • 既に停止処理が進行中の場合、シグナルの種類に応じて処理をスキップまたは継続します。

保守担当者向けメモ

  • 本関数はシグナルハンドラとして呼び出されるため、非同期シグナル安全（async-signal-safe）な操作のみを行う必要があります。
  • retained_data へのアクセスは、MPM のメインループが停止フラグを監視する際のトリガーとなります。
