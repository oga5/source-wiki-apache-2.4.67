core_opts_t

定義

  • 種別: struct
  • 定義箇所: server/request.c:605

目的

Apache HTTP Server におけるディレクトリ設定のオプション（AllowOverride 等）を保持およびマージするための構造体です。

シグネチャまたは形状

typedef struct core_opts_t {
    allow_options_t opts;
    allow_options_t add;
    allow_options_t remove;
    overrides_t override;
    overrides_t override_opts;
    apr_table_t *override_list;
} core_opts_t;

入力と出力

  • 入力: 本構造体は、設定マージ処理において core_opts_merge 関数等を通じて設定ベクトルから値を読み込み、集約するために使用されます。
  • 出力: マージされた設定値は、リクエスト処理中のアクセス制御やオプション検証に利用されます。

前提条件と不変条件

  • override_list は apr_table_t を指すポインタであり、メモリ管理は Apache のプール機構に依存します。
  • 各フィールドは、設定ファイル（.htaccess や <Directory> セクション）で指定されたディレクティブの状態を反映します。

アルゴリズムまたは振る舞い

  • 本構造体は、階層的な設定マージにおいて、親ディレクトリから子ディレクトリへのオプションの継承、追加、削除を管理します。
  • core_opts_merge 関数により、複数の設定ソースから本構造体へ値が統合されます。

呼び出し元と呼び出し先

  • 呼び出し元: server/request.c:840 にてローカル変数としてインスタンス化されています。
  • 呼び出し先: server/request.c:614 で定義される core_opts_merge 関数が、本構造体を引数として操作します。

関連データ構造

  • allow_options_t: オプションのビットマスク型。
  • overrides_t: オーバーライド設定の型。
  • apr_table_t: override_list を保持するための APR テーブル構造体。

失敗モードと境界条件

  • override_list が初期化されていない場合や、メモリ不足により apr_table_t の生成に失敗した場合の挙動については、本構造体定義のみでは明示されていません。

保守担当者向けメモ

  • 本構造体は server/request.c 内での設定マージロジックに特化しています。変更を加える際は、core_opts_merge 関数によるマージ順序が設定の優先順位（<Directory> vs .htaccess）と整合しているか確認してください。
