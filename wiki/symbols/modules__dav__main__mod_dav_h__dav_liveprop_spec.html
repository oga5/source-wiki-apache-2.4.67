dav_liveprop_spec

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1023

目的

WebDAV ライブプロパティ（サーバーが管理するプロパティ）の仕様を定義するために使用されます。プロバイダーがサポートするプロパティのメタデータを記述します。

シグネチャまたは形状

typedef struct {
    int ns;             /* provider-local namespace index */
    const char *name;   /* name of the property */
    int propid;         /* provider-local property ID */
    int is_writable;    /* is the property writable? */
} dav_liveprop_spec;

入力と出力

本構造体はデータ定義であり、関数ではないため直接的な入出力はありません。プロバイダー実装において、プロパティの検索や検証を行う際の参照データとして利用されます。

前提条件と不変条件

  • ns および propid は、各プロバイダー内で一意かつローカルな識別子として機能する必要があります。
  • is_writable は、プロパティがクライアントからの書き込み（PROPPATCH等）を許可するかどうかを示すフラグとして機能します。

アルゴリズムまたは振る舞い

本構造体は、ライブプロパティの静的な定義リスト（例: dav_fs_props や dav_core_props）を構成するために使用されます。mod_dav のライブプロパティ検索ロジック（liveprop.c 内）は、この構造体の配列を走査することで、リクエストされたプロパティが当該プロバイダーでサポートされているか、また書き込み可能かを判定します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/dav/fs/repos.c: ファイルシステムプロバイダーのプロパティ定義で使用。
  • modules/dav/main/std_liveprop.c: 標準的な DAV: 名前空間のプロパティ定義で使用。
  • modules/dav/main/liveprop.c: プロパティ検索および検証ロジック内で参照。
  • **呼び出し先**: なし（本構造体はデータ定義です）。

関連データ構造

  • dav_hooks_liveprop: ライブプロパティのフックテーブル。この構造体で定義された仕様を検索・操作するためのインターフェースを提供します。

失敗モードと境界条件

  • プロバイダーが定義した ns や propid が不整合な場合、プロパティの検索失敗や誤ったプロパティへのアクセスが発生する可能性があります。
  • is_writable の設定が実際のストレージの書き込み権限と一致しない場合、PROPPATCH リクエストが予期せず拒否または許可される可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dav のプロバイダーインターフェースの根幹を成すため、変更時は liveprop.c および各プロバイダー実装（fs/repos.c 等）への影響を考慮してください。
  • ns（名前空間インデックス）の管理は liveprop.c のロジックに依存しているため、新しい名前空間を追加する際は関連する登録処理を確認してください。
