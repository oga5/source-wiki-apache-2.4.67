add_charset_options

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_charset_lite.c:162

目的

CharsetOptions ディレクティブの設定値を解析し、ディレクトリごとの設定構造体 charset_dir_t に反映させる。

シグネチャまたは形状

static const char *add_charset_options(cmd_parms *cmd, void *in_dc, const char *flag)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • in_dc: 設定対象の charset_dir_t 構造体（void* として渡される）。
  • flag: 設定値を示す文字列（ImplicitAdd, NoImplicitAdd, TranslateAllMimeTypes, NoTranslateAllMimeTypes）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: 無効なオプションが指定された場合、エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • in_dc は charset_dir_t 型へのポインタとしてキャスト可能であること。
  • flag は大文字小文字を区別しない比較が行われる。

アルゴリズムまたは振る舞い

1. flag の値に基づき、以下のいずれかのフラグを dc に設定する:

  • ImplicitAdd: dc->implicit_add を IA_IMPADD に設定。
  • NoImplicitAdd: dc->implicit_add を IA_NOIMPADD に設定。
  • TranslateAllMimeTypes: dc->force_xlate を FX_FORCE に設定。
  • NoTranslateAllMimeTypes: dc->force_xlate を FX_NOFORCE に設定。

    • 2. 上記のいずれにも該当しない場合、cmd->temp_pool を使用してエラーメッセージを生成し、返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_charset_lite.c:1114（command_rec 構造体によるディレクティブ登録経由）。
  • **呼び出し先**: apr_pstrcat（エラー発生時）。

関連データ構造

  • charset_dir_t: ディレクトリごとの変換設定を保持する構造体。
  • cmd_parms: Apache の設定コマンド処理用構造体。

失敗モードと境界条件

  • 未定義の flag が渡された場合、apr_pstrcat を用いてエラー文字列を生成し、処理を中断する。

保守担当者向けメモ

  • 本関数は CharsetOptions ディレクティブのハンドラとして機能する。新しいオプションを追加する場合は、strcasecmp の条件分岐を拡張すること。
