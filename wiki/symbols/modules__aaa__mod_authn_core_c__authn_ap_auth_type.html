authn_ap_auth_type

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_core.c:334

目的

リクエストに関連付けられた認証タイプ（AuthType）を動的に評価し、取得する。

シグネチャまたは形状

const char *authn_ap_auth_type(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト構造体）
  • **出力**: 評価された認証タイプを表す文字列。評価結果が "None" である場合、または認証タイプが設定されていない場合は NULL を返す。

前提条件と不変条件

  • r->per_dir_config から authn_core_module の設定が取得可能であること。
  • AuthType が ap_expr 形式で設定されている場合、その評価が成功すること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリ設定（authn_core_dir_conf）を取得する。 2. 設定内に ap_auth_type（ap_expr_info_t）が存在するか確認する。 3. 存在する場合、ap_expr_str_exec を呼び出して式を評価する。

  • 評価中にエラーが発生した場合はログを出力し、NULL を返す。

    • 4. 評価結果が文字列 "None"（大文字小文字を区別しない）であれば NULL を返し、それ以外であれば評価された文字列を返す。 5. 設定が存在しない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c において、APR_RETRIEVE_OPTIONAL_FN を介して取得され、リクエスト処理中に呼び出される。
  • **呼び出し先**:
  • ap_get_module_config（Apache コア）
  • ap_expr_str_exec（ap_expr サブシステム）
  • ap_log_rerror（Apache コア）

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • authn_core_dir_conf: mod_authn_core のディレクトリ設定構造体。

失敗モードと境界条件

  • ap_expr_str_exec の評価失敗: エラーログを出力し、NULL を返す。
  • AuthType が未設定: NULL を返す。
  • 評価結果が "None": NULL を返す。

保守担当者向けメモ

  • 本関数はオプション関数として公開されており、server/core.c 等の外部モジュールから利用される。
  • r->ap_auth_type への直接アクセスは非推奨であり、本関数を経由することが推奨される。
