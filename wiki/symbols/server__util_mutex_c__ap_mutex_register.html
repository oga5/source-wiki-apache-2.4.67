ap_mutex_register

定義

  • 種別: function
  • 定義箇所: server/util_mutex.c:254

目的

各モジュールが自身のミューテックス型をシステムに登録し、デフォルトの動作（ディレクトリ、メカニズム、オプション）を定義するために使用されます。

シグネチャまたは形状

apr_status_t ap_mutex_register(apr_pool_t *pconf,
                               const char *type,
                               const char *default_dir,
                               apr_lockmech_e default_mech,
                               apr_int32_t options)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • type: 登録するミューテックスの識別子（文字列）。
  • default_dir: デフォルトのロックファイル配置ディレクトリ（通常は NULL）。
  • default_mech: デフォルトのロックメカニズム（apr_lockmech_e）。
  • options: AP_MUTEX_ALLOW_NONE または AP_MUTEX_DEFAULT_NONE フラグ。
  • **出力**:
  • APR_SUCCESS: 正常終了。
  • APR_EINVAL: 不正なオプションが指定された場合。

前提条件と不変条件

  • options に指定可能なフラグは AP_MUTEX_ALLOW_NONE および AP_MUTEX_DEFAULT_NONE のみに制限されます。
  • 内部ハッシュテーブル mxcfg_by_type は、本関数内で呼び出される ap_mutex_init によって初期化が保証されます。

アルゴリズムまたは振る舞い

1. pconf から mutex_cfg_t 構造体用のメモリを確保します。 2. options の妥当性を検証し、不正な値であれば APR_EINVAL を返します。 3. ap_mutex_init を呼び出し、内部ハッシュテーブルの初期化を試みます。 4. 指定された options、default_dir、default_mech を構造体に設定します。 5. AP_MUTEX_DEFAULT_NONE が指定されている場合、構造体の none フラグを立てます。 6. type をキーとして、設定済みの構造体を mxcfg_by_type ハッシュテーブルに登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: 各 Apache モジュールの初期化フェーズ（pre-config 等）。具体的な呼び出し元はソースコードの文脈からは特定されていません。
  • **呼び出し先**: apr_pcalloc (APR), ap_mutex_init (内部), apr_hash_set (APR)。

関連データ構造

  • mutex_cfg_t: ミューテックスの設定情報を保持する内部構造体。
  • mxcfg_by_type: 登録された全ミューテックス設定を保持するハッシュテーブル。

失敗モードと境界条件

  • options に許可されていないビットが設定されている場合、APR_EINVAL を返して終了します。
  • メモリ割り当て失敗時は apr_pcalloc の挙動に従い、通常はプロセスが異常終了します。

保守担当者向けメモ

  • 本関数はサーバー起動時の設定フェーズで実行されることを想定しています。
  • 登録された設定は mxcfg_lookup を通じて、後のミューテックス生成時に参照されます。
  • default_dir が NULL の場合、APR のデフォルト動作に依存します。
