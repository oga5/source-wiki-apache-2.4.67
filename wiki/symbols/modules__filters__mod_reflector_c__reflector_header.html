reflector_header

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reflector.c:202

目的

ReflectorHeader ディレクティブの設定を解析し、リクエストヘッダーをレスポンスヘッダーにマッピングするためのルールを reflector_cfg 構造体に登録します。

シグネチャまたは形状

static const char *reflector_header(cmd_parms *cmd, void *dummy, const char *in, const char *out)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: reflector_cfg 構造体へのポインタ（ディレクトリ設定）。
  • in: マッピング対象のリクエストヘッダー名。
  • out: マッピング先のレスポンスヘッダー名（省略時は in と同じ値が使用される）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • dummy は reflector_cfg 型としてキャスト可能である必要があります。
  • cfg->headers は apr_table_addn を受け入れ可能な有効なテーブルである必要があります。

アルゴリズムまたは振る舞い

1. dummy 引数を reflector_cfg 構造体へキャストします。 2. out 引数が指定されている場合はその値を、指定されていない場合は in 引数の値を、リクエストヘッダーからレスポンスヘッダーへのマッピングとして cfg->headers テーブルに追加します。 3. 処理完了後、NULL を返して正常終了を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_reflector.c:218 において、AP_INIT_TAKE12 マクロを介して Apache の設定読み込みプロセスから呼び出されます。
  • **呼び出し先**: APR の apr_table_addn 関数を呼び出し、設定テーブルを更新します。

関連データ構造

  • reflector_cfg: ディレクトリ単位の設定を保持し、ヘッダーマッピングテーブルを管理する構造体。
  • cmd_parms: Apache の設定コマンド実行時に渡されるコンテキスト情報。

失敗モードと境界条件

  • out 引数が NULL の場合、in 引数の値がそのままレスポンスヘッダー名として使用されます。
  • 設定ファイルで ReflectorHeader ディレクティブが誤った引数数で指定された場合、Apache の設定解析フェーズでエラーが発生します（AP_INIT_TAKE12 による制約）。

保守担当者向けメモ

  • 本関数は設定読み込み時にのみ実行されます。実行時のリクエスト処理（reflector_handler）には影響しません。
  • apr_table_addn を使用しているため、同一の in ヘッダーに対して複数回 ReflectorHeader を指定した場合、複数のマッピングがテーブルに蓄積されます。
