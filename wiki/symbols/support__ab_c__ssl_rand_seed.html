ssl_rand_seed

定義

  • 種別: function
  • 定義箇所: support/ab.c:631

目的

OpenSSL の乱数生成器（PRNG）に対して、現在の時刻、プロセスID、および実行時のスタック状態をシード値として提供し、初期化を行う。

シグネチャまたは形状

static void ssl_rand_seed(void)

入力と出力

  • 入力: なし（内部で time(NULL)、getpid()、スタック上のメモリ領域を参照）
  • 出力: なし（RAND_seed を通じて OpenSSL 内部状態を更新）

前提条件と不変条件

  • OpenSSL の RAND_seed 関数が利用可能であること。
  • ssl_rand_choosenum がスタック上の有効なオフセットを返すこと。

アルゴリズムまたは振る舞い

1. **時刻のシード**: time(NULL) で取得した現在時刻を RAND_seed に渡す。 2. **PIDのシード**: getpid() で取得したプロセスIDを RAND_seed に渡す。 3. **スタックのシード**: ssl_rand_choosenum を使用して決定したスタック上のランダムなオフセットから、128バイトのデータを RAND_seed に渡す。

呼び出し元と呼び出し先

  • 呼び出し元: support/ab.c:1453
  • 呼び出し先: time(), getpid(), RAND_seed(), ssl_rand_choosenum()
  • 注記: ツールにより modules/ssl/ 配下のファイルにも同名のシンボル参照が検出されていますが、これらは support/ab.c 内の当該関数とはシグネチャが異なる（引数を持つ）ため、別個の実装である可能性が高いです。

関連データ構造

  • unsigned char stackdata[256]: シード生成のためにスタック上に確保された一時バッファ。

失敗モードと境界条件

  • RAND_seed 自体の内部エラーについては本関数内では捕捉されません。
  • ssl_rand_choosenum が返すオフセットが stackdata の範囲を超えないよう、sizeof(stackdata)-128-1 を上限として計算されています。

保守担当者向けメモ

  • 本関数は ab.c 専用の静的関数です。modules/ssl/ 内の同名関数と混同しないよう注意してください。
  • スタック上のデータをシードとして利用する手法は、環境によってエントロピーの質が異なる可能性があるため、OpenSSL のバージョンアップ時には RAND_seed の推奨される利用方法を確認してください。
