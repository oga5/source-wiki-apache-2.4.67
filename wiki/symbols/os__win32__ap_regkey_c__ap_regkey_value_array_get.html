ap_regkey_value_array_get

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:443

目的

Windows レジストリの REG_MULTI_SZ 型の値を読み取り、文字列の配列（apr_array_header_t）として取得します。

シグネチャまたは形状

apr_status_t ap_regkey_value_array_get(apr_array_header_t **result,
                                       ap_regkey_t *key,
                                       const char *valuename,
                                       apr_pool_t *pool)

入力と出力

  • **入力**:
  • key: 操作対象のレジストリキー（ap_regkey_t*）。
  • valuename: 取得する値の名前。
  • pool: メモリ割り当てに使用する APR プール。
  • **出力**:
  • result: 取得した文字列配列へのポインタが格納されるアドレス。
  • 戻り値: 成功時は APR_SUCCESS、失敗時は apr_status_t エラーコード。

前提条件と不変条件

  • key は有効なレジストリキーハンドルを保持している必要があります。
  • 対象のレジストリ値は REG_MULTI_SZ 型である必要があります。
  • pool は、結果の配列および内部バッファの生存期間を管理するために使用されます。

アルゴリズムまたは振る舞い

1. ap_regkey_value_raw_get を呼び出し、レジストリから生データを取得します。 2. 取得したデータの型が REG_MULTI_SZ でない場合は APR_EINVAL を返します。 3. OS の環境（Unicode/ANSI）に応じて以下の処理を行います。

  • **Unicode**: apr_conv_ucs2_to_utf8 を使用して UTF-8 に変換し、末尾に二重の NULL 文字を保証します。
  • **ANSI**: データの末尾が二重 NULL で終わっているか確認し、不足している場合はバッファを再確保して補完します。

    • 4. バッファ内の NULL 区切り文字列を走査し、要素数をカウントします。 5. apr_array_make で配列を初期化し、各文字列へのポインタを apr_array_push で格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/service.c:619
  • **呼び出し先**:
  • ap_regkey_value_raw_get
  • apr_palloc
  • apr_conv_ucs2_to_utf8 (Unicode 環境時)
  • apr_array_make
  • apr_array_push

関連データ構造

  • ap_regkey_t: レジストリキーハンドルを保持するコンテナ。
  • apr_array_header_t: 結果を格納する APR 配列構造体。

失敗モードと境界条件

  • ap_regkey_value_raw_get が失敗した場合、そのエラーコードを返します。
  • 対象の値が REG_MULTI_SZ ではない場合、APR_EINVAL を返します。
  • Unicode 変換時にエラーが発生した場合、そのエラーコードを返します。
  • 変換後のサイズが期待値を超過する場合、APR_ENAMETOOLONG を返す可能性があります。

保守担当者向けメモ

  • 本関数は REG_MULTI_SZ の二重 NULL 終端を厳密に要求します。ANSI モードでの補完処理は、レジストリデータの不整合に対する防御的実装です。
  • Unicode 変換時のバッファ確保サイズ（size * 3 + 2）は、変換後の最大長を見越した余裕を持たせた設計となっています。
