ipaddr_chain

定義

  • 種別: struct
  • 定義箇所: server/vhost.c:67

目的

IPアドレスとポートのペアをキーとするハッシュテーブルのノードとして、仮想ホストの構成情報を保持するために使用されます。

シグネチャまたは形状

struct ipaddr_chain {
    ipaddr_chain *next;
    /* その他、IP/ポート情報および名前ベース仮想ホストリスト(names)を含む */
};

※ソースコードの構造定義行（67-69行目）および参照箇所に基づく構成です。

入力と出力

  • **入力**: find_ipaddr 関数において apr_sockaddr_t 型のソケットアドレスを受け取り、ハッシュテーブルの検索キーとして利用します。
  • **出力**: 検索結果として該当する ipaddr_chain 構造体へのポインタを返します。

前提条件と不変条件

  • 設定完了後にハッシュテーブルが構築され、実行時は読み取り専用として扱われるため、マルチスレッド環境下でも安全にアクセス可能です。
  • すべてのメモリ割り当ては apr_pool_t を介して行われます。

アルゴリズムまたは振る舞い

1. **ハッシュ検索**: find_ipaddr 関数により、接続のローカルアドレスからハッシュテーブルを走査します。 2. **リスト管理**: 同一ハッシュ値を持つエントリはリスト構造（next ポインタ）で管理されます。 3. **マッチング**: 検索対象の IP/ポートと一致するノードを特定し、そのノードが保持する names（name_chain）リストを用いて仮想ホストの解決を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_update_vhost_given_ip が接続確立時に本構造体を検索し、conn_rec に情報を格納します。
  • **呼び出し先**: new_ipaddr_chain 関数が新規ノードの生成時に本構造体をインスタンス化します。

関連データ構造

  • name_chain: 特定の IP/ポートに紐付く仮想ホストのリストを保持します。
  • iphash_table: ipaddr_chain を要素とするハッシュテーブル本体です。

失敗モードと境界条件

  • 該当する IP/ポートが見つからない場合、find_ipaddr は NULL を返します。
  • names メンバが NULL の場合、そのノードは名前ベースではなく IP ベースの仮想ホストとして扱われます。

保守担当者向けメモ

  • IPHASH_STATISTICS が有効な場合、デバッグ用の統計ダンプ機能が本構造体を利用します。
  • NameVirtualHost ディレクティブは非推奨であるため、将来的な構成変更の際は注意が必要です。
