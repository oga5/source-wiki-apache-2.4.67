abort_on_oom

定義

  • 種別: function
  • 定義箇所: support/passwd_common.c:53

目的

メモリ不足（OOM）発生時に、標準エラー出力へエラーメッセージを書き出し、プロセスを異常終了させるためのコールバック関数です。

シグネチャまたは形状

int abort_on_oom(int rc);

入力と出力

  • **入力**: rc (int型) - 呼び出し元から渡されるリターンコード（本関数内では使用されません）。
  • **出力**: なし（関数は abort() を呼び出すため、呼び出し元へは復帰しません）。

前提条件と不変条件

  • STDERR_FILENO が有効なファイル記述子としてオープンされている必要があります。
  • 本関数は apr_pool_abort_set を介して APR メモリプールの異常終了ハンドラとして登録されることを想定しています。

アルゴリズムまたは振る舞い

1. "Error: out of memory\n" という文字列を定義します。 2. write() システムコールをループ内で呼び出し、標準エラー出力（STDERR_FILENO）へ文字列を書き込みます。 3. 書き込みが中断された場合（EINTR 等）、ループを継続して残りのバイト数を書き込みます。 4. 書き込み完了後、abort() を呼び出してプロセスを強制終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_mplx.c
  • server/main.c
  • support/ab.c
  • support/htdbm.c
  • support/htpasswd.c
  • support/passwd_common.h
  • **呼び出し先**:
  • write() (システムコール)
  • abort() (標準ライブラリ)

関連データ構造

  • なし

失敗モードと境界条件

  • write() がエラーを返した場合、ループ条件により abort() が実行されます。
  • write() が部分的にしか書き込めなかった場合、残りのバッファを書き込むまでループが継続します。

保守担当者向けメモ

  • 本関数は NOTREACHED な関数として設計されています。
  • errno を参照して EINTR による中断を考慮していますが、それ以外の致命的な書き込みエラーが発生した場合でも最終的に abort() が呼び出されます。
