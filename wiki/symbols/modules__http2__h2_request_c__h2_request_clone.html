h2_request_clone

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:220

目的

既存の h2_request オブジェクトのディープコピーを作成し、指定されたメモリプール上に新しいインスタンスを生成します。

シグネチャまたは形状

h2_request *h2_request_clone(apr_pool_t *p, const h2_request *src)

入力と出力

  • **入力**:
  • p: 複製されたオブジェクトを割り当てるための apr_pool_t。
  • src: 複製元の h2_request 構造体へのポインタ。
  • **出力**: 複製された h2_request 構造体へのポインタ。

前提条件と不変条件

  • src は有効な h2_request 構造体を指している必要があります。
  • 戻り値の各文字列フィールドおよびヘッダーテーブルは、指定されたプール p のライフサイクルに依存します。

アルゴリズムまたは振る舞い

1. apr_pmemdup を使用して、src 構造体自体のメモリ領域をプール p 上に複製します。 2. method, scheme, authority, path, protocol の各文字列フィールドを apr_pstrdup で個別に複製します。 3. headers フィールド（apr_table_t）を apr_table_clone を用いて複製します。 4. 複製された新しい h2_request 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_stream.c:682
  • modules/http2/h2_ws.c:113
  • **呼び出し先**:
  • apr_pmemdup (APR)
  • apr_pstrdup (APR)
  • apr_table_clone (APR)

関連データ構造

  • h2_request: HTTP/2 リクエストを表現する構造体。
  • apr_pool_t: メモリ管理に使用される Apache Portable Runtime のプール。
  • apr_table_t: ヘッダー情報を保持するテーブル構造。

失敗モードと境界条件

  • メモリ不足時には、APR のメモリ割り当て関数がプロセスを終了させるか、例外を発生させる可能性があります。

保守担当者向けメモ

  • 本関数はディープコピーを行うため、複製元と複製先でメモリ領域が完全に分離されます。
  • h2_request 構造体のフィールド構成に変更があった場合、本関数内のコピー処理も更新する必要があります。
