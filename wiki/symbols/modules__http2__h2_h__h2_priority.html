h2_priority

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2.h:109

目的

HTTP/2 ストリームの優先度制御（依存関係および重み）を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct h2_priority {
    h2_dependency dependency;
    int           weight;
} h2_priority;

入力と出力

  • **入力**: h2_dependency 型の依存関係情報と、優先度を示す int 型の重み。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • weight は HTTP/2 仕様に基づく優先度の重みを表します。
  • dependency フィールドは、当該ストリームがどのストリームに依存しているか、あるいは排他的な依存関係にあるかを定義します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行い、ロジックは含みません。この構造体は、h2_config におけるコンテンツタイプごとの優先度設定や、h2_push におけるプッシュリクエストの優先度決定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/http2/h2_config.c: コンテンツタイプと優先度のマッピング（priorities ハッシュ）で使用。
  • modules/http2/h2_push.c: プッシュリクエスト生成時の優先度割り当てに使用。
  • modules/http2/h2_session.c: セッション内での優先度管理に使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • h2_dependency: 依存関係を定義する構造体。
  • apr_hash_t: h2_config.c 内で h2_priority レコードを管理するために使用。

失敗モードと境界条件

  • apr_pcalloc 等によるメモリ確保失敗時に、本構造体のインスタンス化が失敗する可能性があります。
  • weight の値が HTTP/2 仕様の範囲外である場合の挙動については、本構造体定義からは判断できません。

保守担当者向けメモ

  • 本構造体は h2_config.h や h2_session.h からも参照されており、モジュール全体で優先度情報を共有するための基盤となっています。
  • 変更を加える際は、h2_config のハッシュ管理および h2_push でのメモリ割り当てへの影響を確認してください。
