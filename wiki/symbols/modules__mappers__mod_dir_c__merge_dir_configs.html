merge_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_dir.c:154

目的

Apache HTTP Server の設定マージフェーズにおいて、親ディレクトリの設定（basev）と現在のディレクトリの設定（addv）を統合し、新しいディレクトリ設定構造体（dir_config_rec）を生成する。

シグネチャまたは形状

static void *merge_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: 設定メモリプール。
  • basev: 親ディレクトリの設定構造体（dir_config_rec へのポインタ）。
  • addv: 現在のディレクトリの設定構造体（dir_config_rec へのポインタ）。
  • **出力**:
  • 統合された新しい dir_config_rec 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は dir_config_rec 型としてキャスト可能であること。
  • 設定値が未設定であることを示す定数（MODDIR_UNSET, REDIRECT_UNSET）が適切に定義されていること。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい dir_config_rec 構造体をメモリプール p 上に確保する。 2. 各フィールドについて以下の優先順位で値を決定する：

  • index_names, dflt: add 側に値が存在すればそれを採用し、なければ base 側の値を採用する。
  • do_slash, checkhandler, redirect_index: add 側の値が UNSET 定数である場合のみ base 側の値を採用し、それ以外は add 側の値を採用する。

    • 3. 統合された構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_dir.c:412 にて、ディレクトリ設定のマージ用として参照されている。
  • **呼び出し先**: 内部で apr_pcalloc を呼び出す。

関連データ構造

  • dir_config_rec: ディレクトリごとの設定を保持する構造体。index_names（インデックスリスト）、do_slash（スラッシュ補完フラグ）、checkhandler、redirect_index、dflt（フォールバック先）を含む。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗する可能性がある（Apache の標準的なメモリ管理に従う）。
  • basev または addv が NULL の場合、キャスト後のアクセスで未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は dir_config_rec の各フィールドに対して「上書き」または「継承」のロジックを実装している。新しい設定項目を dir_config_rec に追加した場合は、必ず本関数でマージロジックを更新すること。
