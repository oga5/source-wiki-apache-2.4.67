server_rec

定義

  • 種別: typedef
  • 定義箇所: modules/loggers/mod_log_config.h:41
  • 補足: 本来の構造体定義は httpd.h:818 に存在します。

目的

Apache HTTP Server における仮想ホストやメインサーバーの設定、状態、および実行時情報を保持するためのコアデータ構造です。

シグネチャまたは形状

struct server_rec として定義されており、サーバーの構成情報（ホスト名、ポート、タイムアウト設定等）や、モジュールが保持するコンテキスト情報を含みます。

入力と出力

  • 入力: サーバーの初期化時やリクエスト処理時に、各フック関数や API の引数として渡されます。
  • 出力: サーバーの構成状態を読み取るための参照として利用されます。

前提条件と不変条件

  • サーバーのライフサイクル全体を通じて生存するオブジェクトです。
  • open_logs フック等の初期化フェーズにおいて、サーバー構成が確定している必要があります。

アルゴリズムまたは振る舞い

  • サーバー固有の設定値の保持。
  • 複数の仮想ホスト間での設定の継承および分離の管理。
  • モジュールがサーバー単位で保持するデータ（module_config）の格納場所としての機能。

呼び出し元と呼び出し先

  • 呼び出し元: ap_setup_listeners、ap_duplicate_listeners、mpm フック、child_status フックなど、Apache コアおよび MPM モジュールから広範に参照されます。
  • 呼び出し先: 本シンボル自体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • request_rec: リクエスト単位の情報を保持する構造体。
  • apr_pool_t: サーバーのメモリ管理に使用されるプール。

失敗モードと境界条件

  • サーバー起動時の設定解析エラーにより、不完全な状態で初期化される可能性があります。
  • 複数のスレッドから同時にアクセスされる場合、構成情報の変更には適切な同期が必要です。

保守担当者向けメモ

  • include/ap_mmn.h に記載されている通り、バージョンアップに伴いメンバーが追加されることがあります（例: server_scheme や context）。
  • ログモジュール（mod_log_config）においては、ログ出力先の設定やフォーマットの決定において、この構造体が保持するサーバー設定が参照されます。
