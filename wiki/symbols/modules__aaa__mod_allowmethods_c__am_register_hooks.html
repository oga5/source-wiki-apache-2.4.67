am_register_hooks

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_allowmethods.c:138

目的

Apache HTTP Server のフックシステムに対し、mod_allowmethods モジュールのアクセス制御関数を登録します。

シグネチャまたは形状

static void am_register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（メモリ割り当て用プール）
  • **出力**: なし（戻り値 void）

前提条件と不変条件

  • 本関数はモジュールの初期化フェーズで呼び出されることを前提としています。
  • am_check_access 関数が定義されている必要があります。

アルゴリズムまたは振る舞い

1. ap_hook_access_checker を呼び出します。 2. am_check_access をアクセスチェッカーとして登録します。 3. 優先度として APR_HOOK_REALLY_FIRST を指定し、他のアクセス制御モジュールよりも先に本モジュールの検証が実行されるように設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_allowmethods.c:157（モジュール定義構造体 module AP_MODULE_DECLARE_DATA allowmethods_module 内のフック登録エントリから間接的に呼び出されます）。
  • **呼び出し先**: ap_hook_access_checker（Apache API）。

関連データ構造

  • am_check_access: 本関数によって登録されるアクセス制御関数。

失敗モードと境界条件

  • Apache のフック登録メカニズムに依存しており、メモリ不足等のシステムエラーが発生した場合は Apache の起動プロセスが失敗します。

保守担当者向けメモ

  • 本関数は APR_HOOK_REALLY_FIRST を使用しているため、他のアクセス制御モジュールよりも先に評価されます。この優先順位を変更する場合は、他のモジュールとの競合に注意してください。
