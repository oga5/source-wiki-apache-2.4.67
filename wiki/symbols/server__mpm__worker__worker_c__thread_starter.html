thread_starter

定義

  • 種別: struct
  • 定義箇所: server/mpm/worker/worker.c:194

目的

子プロセス内でのスレッド生成および管理に必要な情報を保持するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct {
    apr_thread_t **threads;
    apr_thread_t *listener;
    int child_num_arg;
    apr_threadattr_t *threadattr;
} thread_starter;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。各フィールドは以下の役割を持ちます。

  • threads: ワーカー用スレッドの配列へのポインタ。
  • listener: リスナー用スレッドへのポインタ。
  • child_num_arg: 子プロセスの識別番号。
  • threadattr: スレッド生成時に使用する属性設定。

前提条件と不変条件

  • apr_palloc 等を用いてメモリが割り当てられている必要があります。
  • スレッド生成処理（create_listener_thread 等）に渡される前に、各フィールドが適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行います。この構造体を利用して、リスナーおよびワーカー用スレッドの生成、管理、および終了処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/worker/worker.c 内の child_main 関数などでインスタンス化されます。
  • **呼び出し先**: create_listener_thread 関数などが本構造体を引数として受け取り、スレッドの生成や制御を行います。
  • **注記**: server/mpm/event/event.c からも参照されていますが、これは類似のアーキテクチャを持つ別モジュールによる利用です。

関連データ構造

  • apr_thread_t: APR（Apache Portable Runtime）のスレッドハンドル。
  • apr_threadattr_t: スレッド属性を定義するAPR構造体。

失敗モードと境界条件

  • メモリ割り当て失敗: apr_palloc が失敗した場合、構造体へのアクセスは未定義動作となります。
  • スレッド生成失敗: threadattr や threads 配列が正しく設定されていない場合、スレッドの起動に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は worker MPM の子プロセス管理において中心的な役割を果たします。
  • event MPM と共通の構造体名を使用していますが、各モジュール内でのメモリ管理やライフサイクルに注意してください。
