hb_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/cluster/mod_heartbeat.c:32

目的

mod_heartbeat モジュールにおけるサーバー設定および実行時状態を保持するための構造体です。

シグネチャまたは形状

typedef struct hb_ctx_t
{
    int active;
    apr_sockaddr_t *mcast_addr;
    int server_limit;
    int thread_limit;
    apr_status_t status;
} hb_ctx_t;

入力と出力

  • 本構造体は mod_heartbeat の設定コンテキストとして機能します。
  • cmd_hb_address ハンドラにより、HeartbeatAddress ディレクティブから mcast_addr が設定されます。
  • hb_watchdog_init により、MPM の制限値（server_limit, thread_limit）が格納されます。

前提条件と不変条件

  • サーバープール (p) 上に割り当てられる必要があります。
  • HeartbeatAddress はグローバルコンテキストでのみ指定可能です。

アルゴリズムまたは振る舞い

  • active フラグにより、モジュールの稼働状態を管理します。
  • hb_monitor 関数に渡され、スコアボードの集計結果を送信するための宛先アドレス（mcast_addr）として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • hb_monitor (引数として参照)
  • ap_get_module_config (設定取得時)
  • apr_pcalloc (初期化時)
  • cmd_hb_address (設定格納時)
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_sockaddr_t: マルチキャスト送信先アドレスを保持するために使用されます。
  • heartbeat_module: 本構造体を設定として保持する Apache モジュール定義。

失敗モードと境界条件

  • mcast_addr が適切に初期化されていない場合、hb_monitor による送信処理が失敗する可能性があります。
  • status フィールドは、ソケット操作等の実行時ステータスを保持するために使用されます。

保守担当者向けメモ

  • 本構造体はサーバープール上に割り当てられます。
  • HEARTBEAT_INTERVAL はコンパイル時定数であり、本構造体には含まれていないため、動的な間隔変更はできません。
  • 送信処理の詳細は hb_monitor を参照してください。
