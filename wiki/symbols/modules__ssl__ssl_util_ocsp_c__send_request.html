send_request

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ocsp.c:67

目的

OCSP レスポンダ（またはプロキシ）に対して TCP ソケットを確立し、メモリ BIO に格納された OCSP リクエストデータを送信する。

シグネチャまたは形状

apr_socket_t *send_request(BIO *request, const apr_uri_t *uri,
                           apr_interval_time_t timeout,
                           conn_rec *c, apr_pool_t *p,
                           const apr_uri_t *proxy_uri)

入力と出力

  • **入力**:
  • request: 送信データが格納された OpenSSL の BIO オブジェクト。
  • uri: OCSP レスポンダの URI。
  • timeout: ソケット通信のタイムアウト時間。
  • c: ログ出力に使用する conn_rec。
  • p: メモリ割り当てに使用する apr_pool_t。
  • proxy_uri: プロキシ経由で通信する場合の URI（指定がない場合は uri を使用）。
  • **出力**:
  • 成功時: 送信済みで接続が維持された apr_socket_t へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • request は読み取り可能な状態であること。
  • uri（または proxy_uri）のホスト名解決が可能であること。
  • apr_pool_t は呼び出し元で適切に管理されていること。

アルゴリズムまたは振る舞い

1. proxy_uri の有無を確認し、接続先 URI を決定する。 2. apr_sockaddr_info_get を使用して接続先のアドレス情報を取得する。 3. アドレスリストを順に走査し、apr_socket_create および apr_socket_connect を試行する。接続成功した時点でループを終了する。 4. 接続したソケットに対して apr_socket_timeout_set でタイムアウトを設定する。 5. BIO_read を使用して request からデータを読み出し、apr_socket_send を用いてソケットへ書き出す。 6. 送信完了後、確立されたソケットを返す。送信中にエラーが発生した場合はソケットをクローズして NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_util_ocsp.c 内の modssl_dispatch_ocsp_request（直接の呼び出しを確認）。
  • **呼び出し先**: APR ネットワーク関数（apr_sockaddr_info_get, apr_socket_create, apr_socket_connect, apr_socket_send 等）および OpenSSL の BIO_read。

関連データ構造

  • BIO: OpenSSL の入出力抽象化構造体。
  • apr_uri_t: URI 解析結果を保持する APR 構造体。
  • apr_socket_t: ネットワークソケットを表現する APR 構造体。
  • conn_rec: Apache HTTP Server の接続状態を保持する構造体。

失敗モードと境界条件

  • **名前解決失敗**: apr_sockaddr_info_get が失敗した場合、エラーログを出力し NULL を返す。
  • **接続失敗**: 全てのアドレスへの接続が失敗した場合、エラーログを出力し NULL を返す。
  • **送信失敗**: apr_socket_send がエラーを返した場合、ソケットをクローズし NULL を返す。

保守担当者向けメモ

  • 本関数は簡易的な HTTP クライアントの一部であり、将来的に標準的な HTTP クライアントライブラリへ置き換えられる可能性がある。
  • HUGE_STRING_LEN サイズのバッファを使用してリクエストを読み込んでいる。
  • 接続先がプロキシかレスポンダかによってログメッセージが切り替わる仕様となっている。
