sendfile_nonblocking

定義

  • 種別: function
  • 定義箇所: server/core_filters.c:836

目的

ファイルバケットの内容を、非ブロッキングモードでソケットへ転送します。

シグネチャまたは形状

static apr_status_t sendfile_nonblocking(apr_socket_t *s,
                                         apr_bucket *bucket,
                                         core_output_filter_ctx_t *ctx,
                                         conn_rec *c)

入力と出力

  • **入力**:
  • s: 送信先ソケット。
  • bucket: 送信対象のファイルバケット。
  • ctx: 出力フィルタのコンテキスト（書き込み済みバイト数の更新に使用）。
  • c: 接続レコード（ログ出力およびトラフィック統計用）。
  • **出力**:
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS、部分書き込み時は APR_EAGAIN 等）。

前提条件と不変条件

  • bucket は apr_bucket_file 型である必要があります。
  • ctx->bytes_written は、本関数によって送信されたバイト数分だけ加算されます。

アルゴリズムまたは振る舞い

1. apr_socket_sendfile を呼び出し、ファイルの内容をソケットへ転送します。 2. ap__logio_add_bytes_out が定義されている場合、送信したバイト数を統計情報として記録します。 3. 送信完了したバイト数を ctx->bytes_written に加算します。 4. 送信結果に基づきバケットを処理します：

  • 全て送信された場合: apr_bucket_delete でバケットを削除します。
  • 一部のみ送信された場合: apr_bucket_split で送信済み分を切り離して削除し、残りのバケットを保持します。この際、戻り値を APR_EAGAIN に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core_filters.c 内の ap_core_output_filter（行678付近）。
  • **呼び出し先**:
  • apr_socket_sendfile (APR)
  • ap__logio_add_bytes_out (モジュール間連携)
  • ap_log_cerror (ログ出力)
  • apr_bucket_delete, apr_bucket_split (APR Bucket API)

関連データ構造

  • apr_socket_t: ソケット抽象化。
  • apr_bucket: データバケット。
  • core_output_filter_ctx_t: 出力フィルタの永続状態。
  • conn_rec: 接続状態。

失敗モードと境界条件

  • apr_socket_sendfile がエラーを返した場合、そのステータスがそのまま戻り値となります。
  • 部分書き込みが発生した場合、バケットの分割処理が行われ、呼び出し元に対して再試行を促す APR_EAGAIN が返されます。

保守担当者向けメモ

  • 本関数は非ブロッキング I/O を前提としています。
  • bytes_written が 0 の場合、バケットの削除や分割は行われません。
  • ap__logio_add_bytes_out はオプションのフックであるため、NULL チェックが必須です。
