add_ssi_vars

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgi.c:292

目的

SSI (Server Side Includes) 実行時に必要な環境変数を、リクエストのサブプロセス環境テーブル (subprocess_env) に追加・設定します。

シグネチャまたは形状

static void add_ssi_vars(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: なし（副作用として r->subprocess_env を更新）

前提条件と不変条件

  • r->subprocess_env が初期化されていること。
  • メモリ割り当ては r->pool を使用するため、リクエスト終了時に自動的に解放される。

アルゴリズムまたは振る舞い

1. **PATH_INFO の処理**:

  • r->path_info が存在する場合、シェルエスケープを適用して PATH_INFO を設定。
  • サブリクエスト (ap_sub_req_lookup_uri) を発行し、物理パスを解決。
  • 解決されたファイル名とパス情報を結合し、PATH_TRANSLATED を設定。
  • サブリクエストを破棄 (ap_destroy_sub_req)。

    • 2. **QUERY_STRING の処理**:

  • r->args が存在する場合、QUERY_STRING を設定。
  • 引数のコピーを作成し、URL デコード (ap_unescape_url) を実行。
  • デコード後の文字列にシェルエスケープを適用し、QUERY_STRING_UNESCAPED を設定。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:708
  • modules/generators/mod_cgid.c:1741
  • **呼び出し先**:

  • apr_pstrcat

関連データ構造

  • request_rec: リクエスト処理のコンテキストを保持。
  • apr_table_t: 環境変数を格納するテーブル。

失敗モードと境界条件

  • r->path_info や r->args が NULL または空文字列の場合、該当する環境変数の設定はスキップされます。
  • ap_sub_req_lookup_uri が失敗した場合、PATH_TRANSLATED は設定されません。

保守担当者向けメモ

  • 本関数は mod_cgi と mod_cgid の両方から参照されており、SSI 実行時の環境変数構築における共通ロジックとして機能しています。
  • ap_escape_shell_cmd を使用しているため、環境変数の値がシェルインジェクションに対して保護されます。
