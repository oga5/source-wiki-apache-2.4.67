socache_rd_iterate

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_redis.c:379

目的

キャッシュ内の全エントリを走査するためのインターフェースを提供します。

シグネチャまたは形状

apr_status_t socache_rd_iterate(ap_socache_instance_t *instance,
                                server_rec *s, void *userctx,
                                ap_socache_iterator_t *iterator,
                                apr_pool_t *pool)

入力と出力

  • 入力:
  • instance: Redis キャッシュインスタンスのコンテキスト。
  • s: サーバーレコード。
  • userctx: イテレータに渡されるユーザー定義コンテキスト。
  • iterator: 走査時に実行されるコールバック関数。
  • pool: メモリ割り当て用プール。
  • 出力:
  • APR_ENOTIMPL: 常にこの値を返し、機能が提供されていないことを示します。

前提条件と不変条件

  • 本関数は ap_socache_provider_t 構造体の一部として定義されていますが、Redis バックエンドでは実装されていません。

アルゴリズムまたは振る舞い

  • 呼び出された際、いかなる処理も行わず直ちに APR_ENOTIMPL を返します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/cache/mod_socache_redis.c:397 にて、プロバイダー構造体のメンバとして参照されています。
  • 呼び出し先: なし。

関連データ構造

  • ap_socache_instance_t: Redis インスタンスのコンテキスト。
  • ap_socache_iterator_t: 走査用コールバック関数の型定義。

失敗モードと境界条件

  • 本関数は常に失敗（APR_ENOTIMPL）を返します。

保守担当者向けメモ

  • 本関数は現在 no-op（何もしない）として実装されており、キャッシュの全走査機能は提供されていません。
