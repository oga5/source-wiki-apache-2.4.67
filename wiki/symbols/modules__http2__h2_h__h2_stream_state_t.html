h2_stream_state_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2.h:141

目的

HTTP/2 ストリームの RFC 7540 準拠の状態を表現し、セッション内でのストリームのライフサイクル管理を行うために使用されます。

シグネチャまたは形状

typedef enum h2_stream_state_t {
    H2_SS_IDLE,
    H2_SS_RSVD_R,
    H2_SS_RSVD_L,
    H2_SS_OPEN,
    H2_SS_CLOSED_R,
    H2_SS_CLOSED_L,
    H2_SS_CLOSED,
    H2_SS_CLEANUP,
    H2_SS_MAX
} h2_stream_state_t;

入力と出力

  • **入力**: 状態遷移をトリガーするフレーム種別やイベント。
  • **出力**: 現在の状態を示す列挙値。

前提条件と不変条件

  • H2_SS_MAX は列挙型の最大値であり、配列のサイズ定義等に使用されます。
  • ストリームの状態は h2_stream 構造体内で保持され、h2_stream.c 内の遷移ロジックによって更新されます。

アルゴリズムまたは振る舞い

  • h2_stream.c 内の on_frame、on_frame_send、on_frame_recv などの関数において、現在の状態に基づいた遷移の妥当性チェックや状態更新が行われます。
  • h2_stream_is_at および h2_stream_is_at_or_past 関数により、特定の状態にあるか、あるいは遷移済みであるかの判定が可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_stream.c および modules/http2/h2_stream.h 内の関数群が主に参照・操作しています。
  • **呼び出し先**: 本シンボル自体は列挙型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • h2_stream: ストリームのコンテキストを保持する構造体。h2_stream.h で定義され、本列挙型をメンバとして持ちます。

失敗モードと境界条件

  • 状態遷移の妥当性チェックにおいて、プロトコル仕様に違反する遷移が発生した場合の挙動は、h2_stream.c 内の遷移制御ロジックに依存します。
  • H2_SS_MAX を超える値が渡された場合の挙動は定義されていません。

保守担当者向けメモ

  • 本列挙型は RFC 7540 の状態遷移モデルを反映しています。状態を追加・変更する場合は、h2_stream.c 内の on_map や各フレーム処理関数における遷移ロジックの整合性を確認してください。
