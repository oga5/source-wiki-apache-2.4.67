get_time_or_size

定義

  • 種別: function
  • 定義箇所: support/rotatelogs.c:540

目的

コマンドライン引数からログローテーションの閾値（サイズまたは時間）を解析し、rotate_config_t 構造体に設定する。

シグネチャまたは形状

const char *get_time_or_size(rotate_config_t *config, const char *arg, int last);

入力と出力

  • **入力**:
  • config: 設定を格納する rotate_config_t 構造体へのポインタ。
  • arg: 解析対象の文字列（サイズ指定または時間指定）。
  • last: 最後の引数かどうかを示すフラグ（int）。
  • **出力**:
  • 成功時: NULL。
  • 失敗時: エラー内容を示す文字列（const char *）。

前提条件と不変条件

  • config->sRotation および config->tRotation が既に設定されている場合、重複設定としてエラーを返す。
  • サイズ指定（B, K, M, G）が含まれる場合、config->sRotation にバイト単位で値を格納する。
  • サイズ指定がない場合、config->tRotation に秒単位で値を格納する。

アルゴリズムまたは振る舞い

1. arg 内の文字 'B', 'K', 'M', 'G' を検索し、対応するバイト倍率（1, 1024, 1024^2, 1024^3）を決定する。 2. サイズ指定が見つかった場合:

  • 既に sRotation が設定されていればエラーを返す。
  • atoi(arg) に倍率を乗じて config->sRotation に格納する。結果が 0 ならエラーを返す。

    • 3. サイズ指定がない場合:

  • last が真かつ sRotation または tRotation が設定済みの場合、config->utc_offset に atoi(arg) * 60 を格納する（-l オプションとの併用は不可）。
  • それ以外の場合、config->tRotation に atoi(arg) を格納する。結果が 0 以下ならエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/rotatelogs.c:683（main 関数内の引数解析ループ）。
  • **呼び出し先**: atoi（標準ライブラリ）、strchr（標準ライブラリ）。

関連データ構造

  • rotate_config_t: ローテーション設定を保持する構造体。

失敗モードと境界条件

  • **重複設定**: サイズまたは時間のパラメータが複数回指定された場合にエラーメッセージを返す。
  • **無効な値**: atoi の結果が 0 または負の値になる場合にエラーを返す。
  • **オプション競合**: -l オプション使用時に UTC オフセットを指定するとエラーを返す。

保守担当者向けメモ

  • 本関数は rotatelogs の引数解析ロジックの一部であり、config 構造体の状態に強く依存する。
  • atoi を使用しているため、非常に大きな数値や不正な形式の入力に対する堅牢性は標準ライブラリの挙動に依存する。
