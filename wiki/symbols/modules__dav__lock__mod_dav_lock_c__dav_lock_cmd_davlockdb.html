dav_lock_cmd_davlockdb

定義

  • 種別: function
  • 定義箇所: modules/dav/lock/mod_dav_lock.c:65

目的

Apache 設定ファイル内の DAVGenericLockDB ディレクティブを処理し、指定されたロックデータベースのパスをディレクトリ設定構造体に格納する。

シグネチャまたは形状

static const char *dav_lock_cmd_davlockdb(cmd_parms *cmd, void *config, const char *arg1)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: 現在のディレクトリ設定構造体 (dav_lock_dir_conf)。
  • arg1: 設定ファイルで指定されたロックデータベースのパス文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • config 引数は dav_lock_dir_conf 型としてキャスト可能であること。
  • cmd->pool は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. ap_server_root_relative を使用し、引数 arg1 をサーバーのルートディレクトリからの相対パスに変換する。 2. 変換結果を conf->lockdb_path に格納する。 3. 変換に失敗した場合（パスが無効な場合）、エラーメッセージを生成して返す。 4. 成功した場合は NULL を返し、設定の適用を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定解析プロセス（AP_INIT_TAKE1 マクロを通じて DAVGenericLockDB ディレクティブが検出された際に呼び出される）。
  • **呼び出し先**: ap_server_root_relative（Apache コア関数）、apr_pstrcat（APR 関数）。

関連データ構造

  • dav_lock_dir_conf: ディレクトリごとの設定を保持する構造体。lockdb_path メンバにパスが格納される。

失敗モードと境界条件

  • 指定されたパスが絶対パスとして解決できない、またはシステム上で無効なパスである場合、ap_server_root_relative が NULL を返し、本関数はエラーメッセージを返して設定処理を中断する。

保守担当者向けメモ

  • 本関数は DAVGenericLockDB ディレクティブ専用のハンドラです。パスのバリデーションは ap_server_root_relative の挙動に依存しています。
  • データベースの書き込み権限については、本関数ではチェックを行わず、実行時の locks.c 側の処理に委ねられています。
