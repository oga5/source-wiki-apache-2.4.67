dav_if_state_list

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:852

目的

WebDAV の If ヘッダーに含まれる状態条件（State Conditions）を保持するための連結リスト構造体です。リソースに対する条件付きリクエストの検証に使用されます。

シグネチャまたは形状

typedef struct dav_if_state_list
{
    dav_if_state_type type;
    int condition;
    const char *etag;
    dav_locktoken *locktoken;
    struct dav_if_state_list *next;
} dav_if_state_list;

入力と出力

  • type: 条件の種類（dav_if_state_type）。
  • condition: 条件の論理状態。DAV_IF_COND_NORMAL (0) または DAV_IF_COND_NOT (1) を指定。
  • etag: ETag 値（文字列）。
  • locktoken: ロックトークン（dav_locktoken）。
  • next: リストの次の要素へのポインタ。

前提条件と不変条件

  • next ポインタにより、単方向連結リストを形成します。
  • condition フィールドは、DAV_IF_COND_NOT が設定されている場合、その条件が否定（"Not"）であることを示します。

アルゴリズムまたは振る舞い

WebDAV の If ヘッダーをパースする際、各条件をこの構造体に格納し、リストとして構築します。リクエスト処理時にこのリストを走査し、リソースの現在の状態が各条件を満たしているかを検証します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/main/util.c 内の If ヘッダーパース処理および検証ロジックから参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • dav_if_state_type: 条件の種類を定義する列挙型。
  • dav_locktoken: ロックトークンを保持する構造体。

失敗モードと境界条件

  • リストの末尾は NULL で終端される必要があります。
  • etag や locktoken が NULL の場合、その条件の型に応じた適切なハンドリングが必要です。

保守担当者向けメモ

  • 本構造体は modules/dav/main/util.c で頻繁に操作されます。リストのメモリ管理は、リクエストに関連付けられたプール（apr_pool_t）で行われることが前提です。
