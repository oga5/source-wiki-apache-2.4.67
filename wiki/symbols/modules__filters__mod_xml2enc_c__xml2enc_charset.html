xml2enc_charset

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_xml2enc.c:576

目的

リクエストに関連付けられた xml2ctx コンテキストから、現在検出されている文字エンコーディング情報を取得し、呼び出し元へ提供する。

シグネチャまたは形状

apr_status_t xml2enc_charset(request_rec* r, xmlCharEncoding* encp, const char** encoding)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • encp: xmlCharEncoding 型の値を格納するポインタ（出力用）。
  • encoding: エンコーディング名（文字列）を格納するポインタ（出力用）。
  • **出力**:
  • APR_SUCCESS: エンコーディングが正常に取得された場合。
  • APR_EAGAIN: コンテキストが未初期化、またはフィルタが初期化されていない場合。
  • APR_EGENERAL: エンコーディングが特定されていない場合。

前提条件と不変条件

  • r->request_config に xml2enc_module のコンテキストが設定されていること。
  • ctx->flags に ENC_INITIALISED フラグが立っていること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、リクエストから xml2ctx を取得する。 2. コンテキストが存在しない、または初期化フラグが未設定の場合は APR_EAGAIN を返す。 3. コンテキスト内の xml2enc および encoding フィールドを、引数のポインタ経由で書き出す。 4. HAVE_ENCODING マクロを用いてエンコーディングが有効か判定し、成功または失敗のステータスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_proxy_html.c が APR_RETRIEVE_OPTIONAL_FN を介して本関数を取得し、エンコーディングの確認に使用している。
  • **呼び出し先**: 内部で ap_get_module_config および HAVE_ENCODING マクロを使用。

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • xml2ctx: mod_xml2enc が保持するリクエストごとの状態構造体。
  • xmlCharEncoding: libxml2 で定義されるエンコーディング列挙型。

失敗モードと境界条件

  • コンテキストが取得できない場合や初期化前は APR_EAGAIN を返す。
  • エンコーディングが特定されていない（HAVE_ENCODING が偽）場合は APR_EGENERAL を返す。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN によりオプション関数として公開されているため、外部モジュール（mod_proxy_html 等）からの利用を前提としたインターフェースである。
  • HAVE_ENCODING マクロの定義および xml2ctx 構造体の詳細については mod_xml2enc.h を参照すること。
