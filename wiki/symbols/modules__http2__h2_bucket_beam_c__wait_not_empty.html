wait_not_empty

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:195

目的

h2_bucket_beam 内のバッファが空である場合に、データが到着するか、ビームが終了状態になるまでスレッドを待機させる。

シグネチャまたは形状

static apr_status_t wait_not_empty(h2_bucket_beam *beam, conn_rec *c, apr_read_type_e block)

入力と出力

  • **入力**:
  • beam: 対象の h2_bucket_beam 構造体。
  • c: 接続コンテキスト conn_rec。
  • block: 読み取りモード（APR_BLOCK_READ か否か）。
  • **出力**:
  • APR_SUCCESS: バッファにデータが存在する状態。
  • APR_ECONNABORTED: ビームが異常終了（aborted）している場合。
  • APR_EOF: ビームが正常終了（closed）している場合。
  • APR_EAGAIN: 非ブロッキング読み取りでデータが空の場合。
  • その他: apr_thread_cond_timedwait または apr_thread_cond_wait の戻り値。

前提条件と不変条件

  • 呼び出し側は、beam->lock ミューテックスを保持している必要がある。
  • beam は初期化済みであること。

アルゴリズムまたは振る舞い

1. buffer_is_empty(beam) が真である間、ループを継続する。 2. ビームの状態を確認し、aborted なら APR_ECONNABORTED、closed なら APR_EOF を返す。 3. block が APR_BLOCK_READ でない場合、APR_EAGAIN を返す。 4. ブロッキング読み取りの場合:

  • beam->timeout > 0 ならば apr_thread_cond_timedwait で待機。
  • それ以外は apr_thread_cond_wait で無期限待機。

    • 5. 待機解除後、再度ループ条件を評価する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_bucket_beam.c:742（h2_beam_receive 内から呼び出される）。
  • **呼び出し先**:
  • buffer_is_empty
  • apr_thread_cond_timedwait / apr_thread_cond_wait
  • H2_BEAM_LOG (マクロ)

関連データ構造

  • h2_bucket_beam: 同期対象のバケット転送構造体。
  • conn_rec: Apache HTTP Server の接続コンテキスト。
  • apr_read_type_e: ブロッキング動作を決定する列挙型。

失敗モードと境界条件

  • **タイムアウト**: beam->timeout が設定されている場合、待機時間が経過すると apr_thread_cond_timedwait の戻り値が rv に格納され、ループを抜ける可能性がある。
  • **異常終了**: beam->aborted が設定されると、データ有無に関わらず即座に APR_ECONNABORTED を返す。
  • **正常終了**: beam->closed が設定されると、バッファが空であれば APR_EOF を返す。

保守担当者向けメモ

  • 本関数は beam->lock を保持した状態で実行される前提であるため、デッドロックを避けるため呼び出し元でのロック管理に注意すること。
  • H2_BEAM_LOG マクロによるトレース出力が含まれており、デバッグ時には APLOG_TRACE2 レベルを確認すること。
