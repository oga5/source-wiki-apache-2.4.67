ap_expr_eval_var

定義

  • 種別: function
  • 定義箇所: server/util_expr_eval.c:184

目的

式評価エンジンにおいて、登録済みの変数プロバイダー関数を呼び出し、その結果（文字列）を取得します。

シグネチャまたは形状

static const char *ap_expr_eval_var(ap_expr_eval_ctx_t *ctx,
                                    ap_expr_var_func_t *func,
                                    const void *data)

入力と出力

  • **入力**:
  • ctx: 評価コンテキスト（ap_expr_eval_ctx_t）。
  • func: 実行する変数評価関数へのポインタ（ap_expr_var_func_t）。
  • data: 関数に渡される不透明なデータポインタ。
  • **出力**:
  • 評価結果として得られた文字列（const char *）。

前提条件と不変条件

  • func および data は NULL であってはなりません（AP_DEBUG_ASSERT により検証）。

アルゴリズムまたは振る舞い

1. 引数として渡された関数ポインタ func に対して、コンテキスト ctx とデータ data を引数として渡して実行します。 2. その戻り値をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_expr_eval.c:100 において、AST ノードの評価処理から呼び出されています。
  • **呼び出し先**: 引数として渡された ap_expr_var_func_t 型の関数を間接的に呼び出します。

関連データ構造

  • ap_expr_eval_ctx_t: 式評価の実行状態を保持するコンテキスト構造体。
  • ap_expr_var_func_t: 変数評価関数の型定義。

失敗モードと境界条件

  • func または data が NULL の場合、デバッグビルドではアサーションエラーが発生します。
  • 呼び出し先の関数がエラーを発生させた場合の挙動は、当該関数（プロバイダー）の実装に依存します。

保守担当者向けメモ

  • 本関数は単なるディスパッチャであり、具体的な変数解決ロジックは func に渡される関数ポインタが担います。
  • 評価中に生成される文字列は、ctx->p（リクエストプール）から割り当てられることが前提となっています。
