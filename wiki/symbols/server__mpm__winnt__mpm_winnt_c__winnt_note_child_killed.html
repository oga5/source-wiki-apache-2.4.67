winnt_note_child_killed

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:148

目的

子プロセスの終了をスコアボードに記録し、関連するステータスフックを呼び出す。

シグネチャまたは形状

static void winnt_note_child_killed(int slot)

入力と出力

  • **入力**: slot (int) - 終了した子プロセスが割り当てられていたスコアボード上のインデックス。
  • **出力**: なし。

前提条件と不変条件

  • ap_scoreboard_image が初期化済みであること。
  • 指定された slot が有効なプロセススロットを指していること。

アルゴリズムまたは振る舞い

1. ap_run_child_status を呼び出し、サーバー設定、プロセスID、世代番号、スロット番号、および終了ステータス（MPM_CHILD_EXITED）を渡して、子プロセスの終了を通知する。 2. スコアボード上の該当スロットのプロセスID（pid）を 0 にリセットし、当該スロットが空き状態であることを示す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/mpm_winnt.c:871 にて呼び出しを確認。
  • **呼び出し先**: ap_run_child_status（Apacheのフック関数）。

関連データ構造

  • ap_scoreboard_image: プロセス間での状態共有に使用されるスコアボード構造体。
  • ap_server_conf: サーバー設定情報。

失敗モードと境界条件

  • slot がスコアボードの範囲外である場合、メモリ破壊や不正なアクセスが発生する可能性がある。

保守担当者向けメモ

  • 本関数は子プロセスの終了処理におけるクリーンアップの一部を担う。スコアボードの整合性を維持するため、プロセス終了の検知直後に呼び出される必要がある。
