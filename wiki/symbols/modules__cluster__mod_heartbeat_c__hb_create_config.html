hb_create_config

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartbeat.c:159

目的

mod_heartbeat モジュールのサーバー単位の設定構造体 (hb_ctx_t) を生成し、初期化する。

シグネチャまたは形状

static void *hb_create_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定の割り当てに使用するメモリプール (apr_pool_t)。
  • s: 設定が適用されるサーバーレコード (server_rec)。
  • **出力**:
  • hb_ctx_t 構造体へのポインタを void * として返す。

前提条件と不変条件

  • apr_pcalloc を使用しているため、返される hb_ctx_t 構造体の全メンバはゼロ初期化されていることが保証される。
  • サーバープール p の生存期間中、この設定構造体は有効である。

アルゴリズムまたは振る舞い

1. 指定されたメモリプール p から hb_ctx_t 構造体のサイズ分だけメモリを確保する。 2. 確保したメモリ領域をゼロクリアする。 3. 初期化された構造体へのポインタを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール初期化プロセス（modules/cluster/mod_heartbeat.c:224 にて、サーバー設定作成用ハンドラとして登録されている）。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ関数）。

関連データ構造

  • hb_ctx_t: 本関数が生成するモジュールの設定および状態管理構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性がある（Apache の標準的なメモリ管理動作に従う）。

保守担当者向けメモ

  • 本関数は heartbeat_module 定義内でサーバー設定生成ハンドラとして指定されている。
  • hb_ctx_t のメンバに対する具体的な初期値設定は、この関数内ではなく、後続のディレクティブハンドラ（cmd_hb_address 等）やフック関数で行われる設計となっている。
