main

定義

  • 種別: function
  • 定義箇所: test/time-sem.c:504

目的

プロセス間同期（セマフォ）メカニズムのパフォーマンスを測定するためのベンチマーク実行。

シグネチャまたは形状

void main(int argc, char **argv)

入力と出力

  • **入力**:
  • argv[1]: 生成する子プロセスの数 (num_child)
  • argv[2]: 各子プロセスが実行する反復回数 (num_iter)
  • **出力**:
  • 標準出力: 計測された経過時間（秒.マイクロ秒）
  • 標準エラー出力: 引数エラーやシステムコール失敗時のメッセージ

前提条件と不変条件

  • 共有メモリが get_shared_mem により正常に確保されていること。
  • accept_mutex_init が呼び出され、同期プリミティブが初期化されていること。
  • 全子プロセスの終了後、共有カウンタの値が num_child * num_iter と一致すること。

アルゴリズムまたは振る舞い

1. **初期化**: 共有メモリを確保し、カウンタを0に初期化。 2. **同期準備**: accept_mutex_on() を呼び出し、ロックを保持した状態で子プロセスを生成する。 3. **プロセス生成**: fork() を繰り返し、子プロセスは accept_mutex_child_init() を経て、ロック獲得・カウンタインクリメント・YIELD・ロック解放を繰り返す。 4. **計測開始**: 親プロセスが gettimeofday() で開始時刻を記録し、accept_mutex_off() を呼び出して子プロセスを一斉に解放する。 5. **待機と集計**: 親プロセスは wait() で全子プロセスの終了を待ち、終了時刻を記録して経過時間を算出・表示する。 6. **終了処理**: accept_mutex_cleanup() を呼び出し、リソースを解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: OSのプロセス起動（エントリーポイント）。
  • **呼び出し先**: get_shared_mem, accept_mutex_init, accept_mutex_on, accept_mutex_child_init, accept_mutex_off, accept_mutex_cleanup, fork, wait, gettimeofday, atoi, exit, fprintf, perror, puts, printf。
  • **注記**: ツールによる索引化では、本シンボルを呼び出す特定のアプリケーションコードは特定されていません。

関連データ構造

  • struct timeval: 計測用タイムスタンプ。
  • unsigned long *shared_counter: プロセス間で共有されるカウンタ。

失敗モードと境界条件

  • **引数不足**: argc != 3 の場合、使用法を表示して exit(1)。
  • **fork失敗**: pid == -1 の場合、エラーを出力して exit(1)。
  • **整合性エラー**:
  • 子プロセス開始前にカウンタが0でない場合、エラーを出力して exit(1)。
  • 終了時にカウンタが期待値と一致しない場合、警告メッセージを表示。

保守担当者向けメモ

  • 本関数はベンチマーク専用であり、エラー発生時は即座にプロセスを終了させる設計です。
  • YIELD マクロは環境に応じて sched_yield() 等に展開されるため、移植性に関わる挙動を確認する際はマクロ定義を参照してください。
