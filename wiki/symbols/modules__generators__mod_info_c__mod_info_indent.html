mod_info_indent

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_info.c:143

目的

設定ファイル内のディレクティブ定義位置（ファイル名および行番号）を追跡し、階層構造に応じたインデントを付与して出力する。

シグネチャまたは形状

static void mod_info_indent(request_rec *r, int nest, const char *thisfn, int linenum)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体（NULL の場合はコマンドライン出力モードとして動作）。
  • nest: 現在の階層レベルを示す整数。
  • thisfn: 現在処理中のファイル名。
  • linenum: 現在のディレクティブの行番号。
  • **出力**: なし（副作用として HTML または標準出力へ書き出しを行う）。

前提条件と不変条件

  • r が NULL の場合、グローバル変数 out が有効なファイルハンドルとして初期化されている必要がある。
  • get_fn_info(r) および set_fn_info(r, ...) を通じて、直前に処理したファイル名が保持されていること。

アルゴリズムまたは振る舞い

1. **ファイル名変更の検知**: get_fn_info で取得した直前のファイル名と thisfn を比較する。異なる場合、新しいファイル名を出力し、set_fn_info で状態を更新する。 2. **行番号の出力**: r が存在する場合は HTML の <dd> 要素として、存在しない場合は # プレフィックス付きで、put_int_flush_right を用いて行番号を右寄せ出力する。 3. **インデントの適用**: nest の値に基づき、HTML の &nbsp; またはスペースを繰り返し出力して階層を視覚化する。

関連データ構造

  • request_rec: リクエスト処理用構造体。
  • out: コマンドライン出力時に使用されるファイルハンドル（ソースコード内のグローバル変数）。

失敗モードと境界条件

  • thisfn が NULL の場合、*UNKNOWN* として処理される。
  • linenum が 0 以下の場合は行番号の表示が抑制される。
  • r が NULL の場合、リクエストプールではなく out ファイルハンドルへ直接書き込まれる。

保守担当者向けメモ

  • 本関数は mod_info の表示レイアウトを制御する重要なヘルパーである。
  • r の有無によって出力先（HTTP レスポンス vs 標準出力）を切り替える設計のため、修正時は両方のパスが正しく動作することを確認すること。
