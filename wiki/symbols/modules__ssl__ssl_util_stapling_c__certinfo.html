certinfo

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_util_stapling.c:66

目的

OCSP Stapling において、特定の証明書に関連付けられた識別情報、OCSP レスポンダーの場所、およびキャッシュ検索用のインデックスを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    UCHAR idx[SHA_DIGEST_LENGTH];
    OCSP_CERTID *cid;
    char *uri;
} certinfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • **入力**: apr_pcalloc 等によりメモリが割り当てられ、各フィールドが初期化されます。
  • **出力**: 保持された情報は、OCSP レスポンスの取得、キャッシュ検索、およびレスポンダーへの問い合わせ時に参照されます。

前提条件と不変条件

  • idx は、DER エンコードされた証明書の SHA-1 ダイジェストである必要があります。
  • cid は OpenSSL の OCSP_CERTID 構造体へのポインタであり、有効な OCSP リクエスト生成に使用されます。
  • uri は OCSP レスポンダーの場所を示す文字列です。
  • メモリ管理は APR プールに依存しており、ライフサイクルは関連するリクエストやサーバーの生存期間と同期します。

アルゴリズムまたは振る舞い

本構造体自体にロジックは含まれませんが、mod_ssl の OCSP ステープリング処理において以下の役割を果たします。 1. **キャッシュ検索**: idx を用いてセッションキャッシュ内のレスポンスを特定します。 2. **レスポンス更新**: キャッシュが期限切れの場合、cid と uri を使用して OCSP レスポンダーへ再問い合わせを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: stapling_get_certinfo、ssl_stapling_init_cert 等の modules/ssl/ssl_util_stapling.c 内の関数が本構造体を生成・参照します。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • OCSP_CERTID: OpenSSL が提供する証明書識別子。
  • SHA_DIGEST_LENGTH: インデックス生成に使用されるハッシュ長。

失敗モードと境界条件

  • uri が NULL の場合、OCSP レスポンダーへの問い合わせが不可能となります。
  • cid が適切に初期化されていない場合、OCSP リクエストの生成に失敗します。
  • メモリ不足時には apr_pcalloc が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_ssl の OCSP ステープリング機能の中核となるデータ保持用構造体です。
  • 構造体内の各フィールドは apr_pool_t で管理されるため、メモリリークを防ぐために適切なプール管理が行われていることを確認してください。
  • idx の生成ロジックや cid のパース処理に変更を加える際は、ssl_engine_ocsp.c との整合性に注意してください。
