ssl_init_ssl_connection

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:577

目的

接続 (conn_rec) に対して OpenSSL の SSL オブジェクトを生成・初期化し、SSL/TLS 接続を確立するための準備を行う。

シグネチャまたは形状

int ssl_init_ssl_connection(conn_rec *c, request_rec *r)

入力と出力

  • **入力**:
  • c: 接続情報を保持する conn_rec 構造体。
  • r: リクエスト情報を保持する request_rec 構造体（存在しない場合は NULL）。
  • **出力**:
  • 成功時は APR_SUCCESS を返す。
  • 失敗時は DECLINED を返し、接続を中断 (c->aborted = 1) する。

前提条件と不変条件

  • ssl_init_connection_ctx を通じて、接続に関連付けられた SSLConnRec が取得可能であること。
  • サーバー設定 (SSLSrvConfigRec) から取得した mctx（SSL コンテキスト）が有効であること。

アルゴリズムまたは振る舞い

1. **コンテキスト取得**: ssl_init_connection_ctx を呼び出し、接続用の SSL 設定を取得。 2. **PRNG 初期化**: ssl_rand_seed を呼び出し、擬似乱数生成器をシードする。 3. **SSL オブジェクト生成**: SSL_new を使用して SSL オブジェクトを生成し、sslconn->ssl に格納。失敗時はログを出力して終了。 4. **ハンドシェイク前処理**: ssl_run_pre_handshake を実行し、フックによる事前処理を行う。 5. **セッション ID 設定**: 仮想ホストの ID から MD5 ハッシュを生成し、SSL_set_session_id_context を設定。 6. **メタデータ設定**: SSL_set_app_data 等を用いて、OpenSSL オブジェクトに接続情報 (conn_rec) を紐付ける。 7. **検証設定**: SSL_set_verify_result で検証結果を初期化。 8. **フィルタ初期化**: ssl_io_filter_init を呼び出し、SSL I/O フィルタを接続スタックに挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/mod_ssl.c:686 (接続処理の過程で呼び出される)
  • **呼び出し先**:
  • ssl_init_connection_ctx
  • ssl_rand_seed
  • SSL_new
  • ssl_run_pre_handshake
  • ap_md5_binary
  • SSL_set_session_id_context
  • SSL_set_app_data
  • SSL_set_verify_result
  • ssl_io_filter_init

関連データ構造

  • conn_rec: Apache の接続構造体。
  • SSLConnRec: mod_ssl 内部の接続用コンテキスト。
  • SSLSrvConfigRec: サーバーごとの SSL 設定。
  • SSL: OpenSSL の接続オブジェクト。

失敗モードと境界条件

  • SSL_new が失敗した場合、エラーログを出力し DECLINED を返す。
  • SSL_set_session_id_context が失敗した場合、エラーログを出力し DECLINED を返す。
  • ssl_run_pre_handshake が OK または DECLINED 以外の値を返した場合、その戻り値をそのまま返す。

保守担当者向けメモ

  • 本関数は mod_ssl の接続ライフサイクルにおける重要な初期化ポイントです。
  • XXX とコメントされている通り、エラー時の戻り値 DECLINED の妥当性については、将来的な Apache のエラーハンドリング方針に基づき再検討の余地があります。
  • modssl_set_app_data2(ssl, NULL) は、後続の処理で request_rec が設定されることを前提としています。
