http_create_request

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:253

目的

HTTP リクエスト処理の初期段階において、レスポンス生成に必要な標準的な出力フィルタをフィルタスタックへ追加します。

シグネチャまたは形状

static int http_create_request(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在処理中のリクエスト構造体）
  • **出力**: int （常に OK を返します）

前提条件と不変条件

  • r->main および r->prev が共に偽であること（サブクエストや内部リダイレクトではないメインリクエストであること）が、フィルタ追加の条件です。

アルゴリズムまたは振る舞い

1. リクエストがメインリクエストであるかを確認します。 2. 条件を満たす場合、以下の出力フィルタをフィルタスタックの末尾に追加します。

  • ap_byterange_filter_handle
  • ap_content_length_filter_handle
  • ap_http_header_filter_handle
  • ap_http_outerror_filter_handle

    • 3. 処理完了後、OK を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_create_request を介してサーバーのフックシステムから呼び出されます（modules/http/http_core.c:299）。
  • **呼び出し先**: ap_add_output_filter_handle を呼び出し、フィルタをスタックに登録します。

関連データ構造

  • request_rec: リクエストの状態を保持する構造体。
  • ap_filter_t (間接的): フィルタスタックの管理に使用されます。

失敗モードと境界条件

  • サブクエストや内部リダイレクト（r->main または r->prev が真）の場合、フィルタの追加はスキップされ、何も行わずに OK を返します。

保守担当者向けメモ

  • 本関数は APR_HOOK_REALLY_LAST として登録されており、他のモジュールがフィルタを追加した後に実行されることが意図されています。
  • フィルタの追加順序はスタックの構成に影響するため、変更時は既存のフィルタチェーンとの依存関係に注意してください。
