log_before

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_forensic.c:182

目的

リクエスト開始時に、リクエストラインやヘッダーを含む詳細情報を一意の ID と共にフォレンジックログへ記録する。

シグネチャまたは形状

static int log_before(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (Apache のステータスコード: OK または DECLINED)

前提条件と不変条件

  • ログ出力先（cfg->fd）が設定されていること。
  • 内部リダイレクト等による重複記録を防ぐため、r->prev が存在しないこと。
  • ログ出力には r->pool を使用し、リクエスト終了時に自動的にメモリが解放される。
  • log_escape は ap_assert を用いてバッファオーバーフローを厳格に防止する。

アルゴリズムまたは振る舞い

1. **初期チェック**: 設定が存在しない、またはサブクエスト（r->prev）の場合は DECLINED を返す。 2. **ID 取得/生成**: subprocess_env から UNIQUE_ID を取得。存在しない場合は PID、時刻、アトミックカウンタを用いて生成し、request_config に保存する。 3. **バッファ準備**: count_headers を用いて必要なメモリ量を算出し、apr_palloc で確保する。 4. **ログ構築**: +<id>| の形式で開始し、リクエストラインをエスケープして書き込む。続いて log_headers を用いて全ヘッダーをシリアライズする。 5. **書き込み**: 構築したバッファをファイルディスクリプタへ書き込み、r->notes に forensic-id を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_read_request に登録されており、リクエスト読み取り直後に呼び出される。
  • **呼び出し先**:
  • ap_get_module_config, ap_set_module_config
  • apr_table_get, apr_table_do, apr_table_setn
  • apr_psprintf, apr_atomic_inc32
  • count_headers, count_string, log_escape, log_headers
  • apr_file_write, ap_assert

関連データ構造

  • fcfg: サーバーごとの設定（ログファイルディスクリプタ等）を保持。
  • hlog: ログ構築用の一時的な構造体（バッファ位置やサイズを管理）。
  • request_rec: リクエストのコンテキスト情報。

失敗モードと境界条件

  • cfg->fd が無効な場合、ログ出力はスキップされる。
  • r->prev が存在する場合、同一リクエストの重複記録を避けるため処理を中断する。
  • apr_file_write が失敗した場合、ap_assert により異常終了する。
  • 高負荷環境では、ヘッダー走査とメモリ確保によるパフォーマンス低下の可能性がある。

保守担当者向けメモ

  • 本関数は r->prev をチェックすることで、内部リダイレクト時の二重ログ出力を抑制している。
  • ID 生成時のアトミックカウンタ next_id はマルチスレッド環境での一意性を保証する。
  • ログフォーマットの変更時は count_headers と log_headers の整合性に注意すること。
