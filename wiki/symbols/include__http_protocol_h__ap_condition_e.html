ap_condition_e

定義

  • 種別: enum
  • 定義箇所: include/http_protocol.h:234

目的

HTTP 条件付きリクエスト（If-Match, If-None-Match, If-Modified-Since, If-Unmodified-Since, If-Range）の評価結果を表現するために使用されます。

シグネチャまたは形状

typedef enum {
    AP_CONDITION_NONE,
    AP_CONDITION_NOMATCH,
    AP_CONDITION_WEAK,
    AP_CONDITION_STRONG
} ap_condition_e;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力は持ちません。
  • 関連する評価関数（ap_condition_if_match 等）の戻り値として使用され、条件の合致状況を伝達します。

前提条件と不変条件

  • 各列挙値は、HTTP 仕様における条件付きリクエストの検証結果に対応します。
  • 評価関数は request_rec 構造体を通じてリクエストコンテキストにアクセスします。

アルゴリズムまたは振る舞い

  • AP_CONDITION_NONE: 条件が指定されていない、または評価対象外。
  • AP_CONDITION_NOMATCH: 条件が満たされなかった（不一致）。
  • AP_CONDITION_WEAK: 条件が弱検証子（Weak Validator）として一致した。
  • AP_CONDITION_STRONG: 条件が強検証子（Strong Validator）として一致した。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_protocol.c 内の条件評価関数群が本型を返します。
  • **呼び出し先**: 本型を戻り値として利用する関数として、ap_condition_if_match, ap_condition_if_unmodified_since, ap_condition_if_none_match, ap_condition_if_modified_since, ap_condition_if_range が定義されています。

関連データ構造

  • request_rec: リクエスト処理のコンテキストを保持する構造体。

失敗モードと境界条件

  • 評価関数がリクエストの解析に失敗した場合の挙動については、各評価関数の実装に依存します。本列挙型自体にエラー状態は定義されていません。

保守担当者向けメモ

  • 本型は 2.4.5-dev (20120211.18) にて導入されました。
  • 条件付きリクエストの評価ロジックを変更する際は、modules/http/http_protocol.c 内の該当関数と併せて本列挙型の定義を確認してください。
