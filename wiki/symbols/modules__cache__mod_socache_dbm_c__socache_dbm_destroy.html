socache_dbm_destroy

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:199

目的

DBM ベースの共有オブジェクトキャッシュに関連する物理ファイルをファイルシステムから削除し、キャッシュインスタンスを破棄する。

シグネチャまたは形状

static void socache_dbm_destroy(ap_socache_instance_t *ctx, server_rec *s)

入力と出力

  • ctx: キャッシュインスタンスのコンテキスト（ap_socache_instance_t）。削除対象のファイルパス（data_file）およびメモリプールを保持。
  • s: サーバーレコード（server_rec）。本関数内では使用されていない。
  • 戻り値: なし。

前提条件と不変条件

  • ctx が有効なポインタであり、ctx->pool および ctx->data_file が初期化されていること。
  • 呼び出し元で適切な排他制御が行われていること（本モジュールはスレッドセーフではないため）。

アルゴリズムまたは振る舞い

1. ctx->pool を使用して、ctx->data_file に特定の拡張子（.dir, .pag, .db および DBM_FILE_SUFFIX_DIR, DBM_FILE_SUFFIX_PAG）を付与したパス文字列を生成する。 2. 生成された各パスに対して unlink() を実行し、ファイルを削除する。 3. 最後に ctx->data_file 自体を unlink() する。

  • プラットフォームごとの DBM 実装差異を考慮し、複数の拡張子パターンを網羅的に削除する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/cache/mod_socache_dbm.c:745（直接的な呼び出しが確認されている）。
  • 呼び出し先: apr_pstrcat（APR 文字列連結関数）、unlink（標準システムコール）。

関連データ構造

  • ap_socache_instance_t: キャッシュのコンテキストを保持する構造体。

失敗モードと境界条件

  • ファイルが存在しない場合、unlink() は失敗するが、本関数は戻り値を確認せず処理を継続する。
  • 権限不足によりファイル削除が拒否された場合、ファイルは残存する。

保守担当者向けメモ

  • DBM ファイルのレイアウトはプラットフォームに依存するため、削除対象の拡張子リストは網羅的である必要がある。
  • 本関数は unlink を連続して呼び出すのみであり、エラーハンドリングは行わない設計となっている。
