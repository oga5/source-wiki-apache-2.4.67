privileges_merge_cfg

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_privileges.c:70

目的

サーバー設定と仮想ホスト（vhost）設定をマージする際に、priv_cfg 構造体の動作モード（mode）を継承・解決するために使用されます。

シグネチャまたは形状

static void *privileges_merge_cfg(apr_pool_t *pool, void *BASE, void *ADD)

入力と出力

  • **入力**:
  • pool: メモリ割り当て用プール。
  • BASE: 親コンテキスト（サーバー設定）の priv_cfg 構造体。
  • ADD: 子コンテキスト（vhost 設定）の priv_cfg 構造体。
  • **出力**:
  • マージされた新しい priv_cfg 構造体へのポインタ。

前提条件と不変条件

  • BASE および ADD は priv_cfg 型としてキャスト可能であること。
  • apr_pmemdup により、新しい設定構造体は指定された pool 上に確保されます。

アルゴリズムまたは振る舞い

1. ADD（子設定）の内容を apr_pmemdup を用いて新しい構造体にコピーします。 2. ADD の mode が PRIV_UNSET である場合、BASE（親設定）の mode を継承します。 3. mode 以外の設定値は継承されず、ADD の値がそのまま維持されます。 4. 解決済みの新しい構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/unix/mod_privileges.c:585 にて、Apache のモジュール設定マージ処理の一部として参照されています。
  • **呼び出し先**: apr_pmemdup（Apache Portable Runtime）。

関連データ構造

  • priv_cfg: 本関数が操作する主要な設定構造体。
  • priv_mode: mode フィールドで使用される列挙型。

失敗モードと境界条件

  • メモリ不足時には apr_pmemdup が例外を発生させる可能性があります。
  • BASE または ADD が NULL の場合の挙動は、Apache の設定マージの規約に依存します。

保守担当者向けメモ

  • 本関数は mode の継承のみを扱います。他の設定値（UID/GID 等）はマージされず、子設定が優先される設計となっています。
  • priv_cfg 構造体の定義変更時には、本関数でのコピー対象範囲が適切か確認してください。
