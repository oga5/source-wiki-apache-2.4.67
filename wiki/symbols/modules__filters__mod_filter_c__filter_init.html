filter_init

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_filter.c:100

目的

mod_filter が管理するフィルタチェーン内の各プロバイダに対して初期化関数を呼び出し、実行時コンテキスト（harness_ctx）を構築します。

シグネチャまたは形状

static int filter_init(ap_filter_t *f)

入力と出力

  • **入力**: ap_filter_t *f （初期化対象のフィルタ構造体）
  • **出力**: int （成功時は OK、失敗時は HTTP_INTERNAL_SERVER_ERROR またはプロバイダが返したエラーコード）

前提条件と不変条件

  • f->r->pool を使用してメモリを割り当てます。
  • プロバイダの連鎖（Chaining）はサポートされていません。
  • プロバイダが f->ctx を設定した場合、そのコンテキストは provider_ctx 構造体に格納され、harness_ctx に保持されます。

アルゴリズムまたは振る舞い

1. harness_ctx をリクエストプールから割り当てます。 2. フィルタに登録されたプロバイダのリストを走査します。 3. プロバイダの filter_init_func が filter_init 自身である場合、連鎖と見なしてエラーを返します。 4. 各プロバイダの filter_init_func を呼び出します。 5. プロバイダが f->ctx を設定した場合、それを provider_ctx として harness_ctx のリストに追加します。 6. 最終的に f->ctx に harness_ctx を格納して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていませんが、Apache のフィルタフレームワーク（util_filter.h）により、フィルタ初期化時に呼び出されることが想定されています。
  • **呼び出し先**: 登録された各プロバイダの filter_init_func。

関連データ構造

  • harness_ctx: 実行時のコンテキストを保持する構造体。
  • ap_filter_provider_t: プロバイダのリスト。
  • provider_ctx: 個別のプロバイダごとのコンテキストを管理する連結リスト。

失敗モードと境界条件

  • **連鎖の検出**: プロバイダの初期化関数が filter_init と一致する場合、HTTP_INTERNAL_SERVER_ERROR を返します。
  • **初期化失敗**: プロバイダの filter_init_func が OK 以外の値を返した場合、そのエラーコードをそのまま伝播させます。

保守担当者向けメモ

  • 本関数は mod_filter のディスパッチャとしての役割を果たすためのものであり、個別のプロバイダがどのように ctx を操作するかは各プロバイダモジュールの実装に依存します。
  • filter_lookup 内のプロトコル調整ロジックと同様に、この初期化処理もフィルタ適用による副作用を管理するための重要な基盤です。
