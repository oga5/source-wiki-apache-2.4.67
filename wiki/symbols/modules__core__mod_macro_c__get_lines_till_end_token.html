get_lines_till_end_token

定義

  • 種別: function
  • 定義箇所: modules/core/mod_macro.c:133

目的

設定ファイルから特定の終了トークン（例: </Macro>）が出現するまでの行を読み取り、メモリ上の配列として抽出する。

シグネチャまたは形状

char *get_lines_till_end_token(apr_pool_t *pool,
                               ap_configfile_t *config_file,
                               const char *end_token,
                               const char *begin_token,
                               const char *where,
                               apr_array_header_t **plines)

入力と出力

  • **入力**:
  • pool: メモリ割り当て用プール。
  • config_file: 読み込み対象の設定ファイルストリーム。
  • end_token: 読み込みを終了させるディレクティブ名。
  • begin_token: ネストをカウントするための開始ディレクティブ名（指定時）。
  • where: エラーログ出力用のコンテキスト情報。
  • **出力**:
  • plines: 抽出された行（char *）の配列へのポインタ。
  • 戻り値: 成功時は NULL。失敗時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • config_file は有効な設定ストリームであること。
  • MAX_STRING_LEN を超える行は読み込めない。
  • macro_nesting は初期値 1 で開始し、end_token が現れるたびにデクリメントされる。

アルゴリズムまたは振る舞い

1. apr_array_make で行格納用の配列を初期化する。 2. ap_cfg_getline を用いて設定ファイルを1行ずつ読み込む。 3. # で始まるコメント行はスキップする。 4. ap_getword_conf_nc でディレクティブ名を抽出する。 5. < で始まるタグのネストを any_nesting で追跡する。 6. begin_token が指定されている場合、それが現れると macro_nesting をインクリメントする。 7. end_token が現れた場合、macro_nesting をデクリメントし、0 になれば終了する。 8. 読み込んだ行を pool を使用して plines に追加する。 9. ファイル終端に達しても end_token が見つからない場合はエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_macro.c:775（macro_section 関数内から呼び出される）。
  • **呼び出し先**: ap_cfg_getline, ap_getword_conf_nc, ap_log_error, ap_strrchr_c, warn_if_non_blank, apr_array_push, apr_psprintf。

関連データ構造

  • ap_configfile_t: 設定ファイルの読み込み状態を保持する構造体。
  • apr_array_header_t: 抽出された行を保持する動的配列。

失敗モードと境界条件

  • **終了トークン未検出**: ファイル末尾に達しても end_token が見つからない場合、エラーメッセージを返す。
  • **不正なネスト**: </ で始まるタグが過剰に存在する場合、警告ログを出力する。
  • **閉じ括弧の欠落**: end_token 行に > が含まれない場合、エラーを返す。
  • **余剰文字**: end_token の後に空白以外の文字がある場合、warn_if_non_blank により警告を発する。

保守担当者向けメモ

  • 本関数は行ベースの解析を行うため、複雑なネスト構造の解析には限界がある。
  • MAX_STRING_LEN を超える行は切り捨てられる可能性があるため注意が必要。
  • エラーメッセージ生成には cmd->temp_pool 等のライフサイクルを考慮したメモリ管理が必要。
