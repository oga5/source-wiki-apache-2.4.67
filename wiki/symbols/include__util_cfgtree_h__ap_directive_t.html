ap_directive_t

定義

  • 種別: struct
  • 定義箇所: include/util_cfgtree.h:35

目的

Apache HTTP Server の設定ファイルをメモリ上に階層構造（設定ツリー）として表現するためのノード構造体です。サーバー起動時の設定解析フェーズにおいて、各ディレクティブの保持と管理を担います。

シグネチャまたは形状

typedef struct ap_directive_t ap_directive_t;

※構造体の詳細なメンバ定義は include/util_cfgtree.h:46 以降に存在します。

入力と出力

  • **入力**: 設定ファイル解析時に ap_add_node() を通じてディレクティブ名、引数、位置情報（ファイル名・行番号）が入力されます。
  • **出力**: 構築されたツリー構造は、各モジュールが設定を適用する際の参照先として利用されます。

前提条件と不変条件

  • 設定ファイルに記述された順序を保持します。
  • 閉じタグ（例: </Directory>）はツリー内のノードとしては保存されません。
  • last メンバはツリーの末尾へのアクセスを高速化するためのショートカットですが、常に最新であるとは限りません。

アルゴリズムまたは振る舞い

設定ファイルの解析中、新しいディレクティブが検出されるたびにノードが生成され、ap_add_node() によってツリーに挿入されます。ノードが子要素を持つ場合、親ポインタが更新され、階層構造が形成されます。

呼び出し元と呼び出し先

  • **呼び出し元**: include/http_config.h 内の関数群（ap_walk_config 等）が、設定ツリーの走査や操作のために本構造体を参照します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すものではありません。

関連データ構造

  • ap_conftree: 設定ツリーのルートノードを指すグローバル変数。
  • ap_config.h: 本構造体が必要とする基本設定マクロや型定義を提供します。

失敗モードと境界条件

  • サーバー起動時の設定読み込みフェーズに特化した構造体であり、実行時の動的な設定変更には別のメカニズムが関与する場合があります。
