master_main

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:192

目的

OS/2 MPM における親プロセスのメイン処理を実行し、子プロセスの生成、監視、およびライフサイクル管理を行う。

シグネチャまたは形状

static int master_main()

入力と出力

  • 入力: グローバル変数 ap_server_conf（サーバー設定）、ap_listeners（リスナーソケットリスト）、ap_scoreboard_image（スコアボード）。
  • 出力: 終了ステータス（OK または DONE）。

前提条件と不変条件

  • ap_setup_listeners が成功し、リスナーソケットが利用可能であること。
  • 共有メモリ（parent_info_t）およびスコアボード用の共有メモリが正常に確保されていること。
  • HARD_SERVER_LIMIT（10）がプロセス数の上限として適用される。

アルゴリズムまたは振る舞い

1. **初期化**: シグナルハンドラを設定し、リスナーソケットを準備する。 2. **共有リソース確保**: リスナーソケット情報を格納する共有メモリを DosAllocSharedMem で確保し、OS/2 ミューテックス ap_mpm_accept_mutex を作成する。 3. **スコアボード初期化**: 未初期化であればスコアボード用共有メモリを確保し初期化する。 4. **メインループ**:

  • 現在の稼働プロセス数をカウントし、ap_daemons_to_start に達するまで spawn_child を呼び出して子プロセスを生成する。
  • DosWaitChild で子プロセスの終了を非ブロックで監視する。
  • 子プロセスが終了していればスコアボードを更新し、致命的エラー（APEXIT_CHILDFATAL）であればループを抜ける。
  • 終了フラグが立つまで apr_sleep で待機する。

    • 5. **終了処理**: 全子プロセスにシグナル（SIGTERM または SIGHUP）を送信し、共有メモリを解放して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: mpmt_os2_run（server/mpm/mpmt_os2/mpmt_os2.c:171）。
  • 呼び出し先: set_signals, ap_setup_listeners, DosAllocSharedMem, apr_psprintf, apr_os_sock_get, DosCreateMutexSem, ap_init_scoreboard, ap_calc_scoreboard_size, ap_mpm_child_main, spawn_child, DosWaitChild, apr_sleep, kill, DosFreeMem。
  • 注記: server/mpm/winnt/ 内のコードにも同名のシンボルへの言及があるが、本ファイル（mpmt_os2.c）の文脈とは異なる。

関連データ構造

  • parent_info_t: リスナーソケット情報とミューテックスを保持する共有メモリ構造体。
  • listen_socket_t: 個別のリスナーソケット情報を保持。
  • scoreboard: プロセス状態を管理する共有メモリ領域。

失敗モードと境界条件

  • リスナーソケットのセットアップ失敗時: !OK を返して終了。
  • 共有メモリ確保失敗時: !OK を返して終了。
  • ミューテックス作成失敗時: !OK を返して終了。
  • 子プロセスが APEXIT_CHILDFATAL で終了した場合: ループを中断しサーバーを停止する。

保守担当者向けメモ

  • 本関数は OS/2 の Dos 系 API に強く依存しており、移植性は考慮されていない。
  • HARD_SERVER_LIMIT によるプロセス数制限（10）がハードコードされている点に注意すること。
  • 共有メモリの解放（DosFreeMem）はループ終了後のクリーンアップ時にのみ行われる。
