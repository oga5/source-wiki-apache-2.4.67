prepend_rbuf

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_proxy_html.c:875

目的

コンテキスト内に保持されているバッファ (rbuf) の内容を、出力用のバケットブリゲードの先頭に挿入する。

シグネチャまたは形状

static void prepend_rbuf(saxctxt *ctxt, apr_bucket_brigade *bb)

入力と出力

  • ctxt: フィルタ実行時の状態を保持する saxctxt 構造体へのポインタ。
  • bb: データを挿入する対象の apr_bucket_brigade 構造体。
  • 出力: なし（副作用として bb を変更し、ctxt->rlen を 0 にリセットする）。

前提条件と不変条件

  • ctxt->rlen が 0 より大きい場合、ctxt->rbuf に有効なデータが格納されている必要がある。
  • bb は有効な apr_bucket_brigade であること。

アルゴリズムまたは振る舞い

1. ctxt->rlen が 0 でないかを確認する。 2. 0 であれば何もしない。 3. 0 でなければ、ctxt->rbuf の内容から apr_bucket_transient_create を用いて一時バケットを作成する。 4. 作成したバケットを APR_BRIGADE_INSERT_HEAD マクロを使用して bb の先頭に挿入する。 5. ctxt->rlen を 0 にリセットし、バッファが空であることを示す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/filters/mod_proxy_html.c:910
  • modules/filters/mod_proxy_html.c:989
  • 呼び出し先:
  • apr_bucket_transient_create (APR)
  • APR_BRIGADE_INSERT_HEAD (APR)

関連データ構造

  • saxctxt: コンテキスト構造体。rbuf（バッファ本体）と rlen（バッファ長）を保持する。
  • apr_bucket_brigade: Apache のバケット管理構造体。

失敗モードと境界条件

  • ctxt->rlen が 0 の場合は何も行われないため、安全に呼び出し可能。
  • メモリ不足等の低レイヤーエラーについては、APR のバケット作成関数に依存する。

保守担当者向けメモ

  • 本関数は、パーサが処理を中断した際に残ったバッファを、次のブリゲード処理の先頭に再挿入するために使用される。
  • apr_bucket_transient_create を使用しているため、rbuf のメモリ寿命が bb の処理期間中維持されていることを前提としている。
