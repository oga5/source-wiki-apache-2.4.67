cache_save_store

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:702

目的

バックエンドから受信したレスポンスデータをキャッシュストレージへ書き込み、フィルタチェーンを通じてクライアントへ転送する。

シグネチャまたは形状

static int cache_save_store(ap_filter_t *f, apr_bucket_brigade *in,
                            cache_server_conf *conf, cache_request_rec *cache)

入力と出力

  • **入力**:
  • f: 現在の出力フィルタコンテキスト。
  • in: キャッシュ対象となるレスポンスデータを含むバケットブリゲード。
  • conf: サーバー全体のキャッシュ設定。
  • cache: 現在のリクエストに関連付けられたキャッシュ制御構造体。
  • **出力**:
  • int: APR 準拠のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • cache->provider が有効なストレージプロバイダーを指していること。
  • cache->handle が初期化済みであること。
  • in ブリゲード内のデータは、ストレージプロバイダーの store_body を通じて処理される。

アルゴリズムまたは振る舞い

1. in ブリゲードが空になるまで、cache->provider->store_body を呼び出し、データをストレージへ渡す。 2. ストレージプロバイダーは、キャッシュ可能なバケットを cache->out ブリゲードへ移動させる。 3. cache->out 内に EOS (End of Stream) バケットが含まれている場合、cache->provider->commit_entity を呼び出し、キャッシュの保存を確定させる。 4. cache_remove_lock を呼び出し、必要に応じてキャッシュロックを解除する。 5. cache->out に格納されたデータを ap_pass_brigade でフィルタチェーンの上位へ渡す。 6. store_body が失敗した場合、フィルタを削除し、ロックを解除した上で、残りのデータをクライアントへ透過的にパススルーする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c:861
  • modules/cache/mod_cache.c:1601
  • **呼び出し先**:
  • cache->provider->store_body
  • cache->provider->commit_entity
  • cache_remove_lock
  • ap_pass_brigade
  • ap_remove_output_filter

関連データ構造

  • cache_server_conf: サーバー設定。
  • cache_request_rec: リクエストごとのキャッシュ状態。
  • apr_bucket_brigade: データ転送用バケットリスト。

失敗モードと境界条件

  • **ストレージ書き込み失敗**: store_body がエラーを返した場合、フィルタを解除し、キャッシュロックを解放してエラーを伝播させる。
  • **空のブリゲード**: store_body がデータを消費せずに空のブリゲードを返した場合、無限ループ（スピン）を防ぐためにフィルタを解除し、エラーログを出力して処理を中断する。

保守担当者向けメモ

  • 本関数は CACHE_SAVE フィルタの核心部であり、ストレージプロバイダーとのインターフェースを抽象化している。
  • cache->out ブリゲードの管理はプロバイダーの挙動に依存するため、プロバイダー実装を変更する際は store_body のバケット移動ロジックに注意すること。
