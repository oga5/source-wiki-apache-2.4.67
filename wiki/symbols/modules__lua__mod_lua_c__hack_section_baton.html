hack_section_baton

定義

  • 種別: struct
  • 定義箇所: modules/lua/mod_lua.c:962

目的

Apache HTTP Server の設定パース時において、Lua スクリプトのハンドラ仕様とフック実行タイミングを関連付け、設定ディレクティブ間で情報を保持するためのバトン（コンテキスト）として機能します。

シグネチャまたは形状

typedef struct hack_section_baton
{
    const char *name;
    ap_lua_mapped_handler_spec *spec;
    int apr_hook_when;
} hack_section_baton;

入力と出力

  • 入力: 設定パース時に apr_pcalloc を介してメモリが割り当てられ、各メンバが初期化されます。
  • 出力: 設定ディレクティブのデータとして保持され、後続のフック登録処理で参照されます。

前提条件と不変条件

  • spec は ap_lua_mapped_handler_spec 型の有効なポインタである必要があります。
  • apr_pcalloc によるメモリ確保が成功していることが前提です。

アルゴリズムまたは振る舞い

設定パース処理において、特定の Lua ディレクティブ（LuaHook* 等）が読み込まれた際、この構造体にハンドラ名、仕様、フックの実行順序（apr_hook_when）を格納します。その後、このバトンが設定データとして引き渡されることで、Apache のフック登録ロジックが実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/lua/mod_lua.c 内の設定パース処理（apr_pcalloc によるインスタンス化および directive->data への代入）。
  • 呼び出し先: 直接的な関数呼び出しは行いませんが、設定処理の過程でこの構造体のメンバが参照されます。

関連データ構造

  • ap_lua_mapped_handler_spec: Lua ハンドラの実行仕様を定義する構造体。

失敗モードと境界条件

  • メモリ不足: apr_pcalloc が失敗した場合、Apache の設定パース処理が中断されます。
  • 不正な設定値: spec が適切に初期化されていない場合、後続のフック登録時に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は「ByteCodeHack」と呼ばれる設定パース時の複雑な処理の一部として使用されています。
  • apr_hook_when は、Apache のフック登録順序（APR_HOOK_FIRST, APR_HOOK_MIDDLE, APR_HOOK_LAST 等）を制御するために使用されます。
