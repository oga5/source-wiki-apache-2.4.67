cache_object

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache.h:82

目的

キャッシュされたエンティティを表現し、そのメタデータおよび実装固有のデータへのアクセスを管理するための構造体です。

シグネチャまたは形状

struct cache_object {
    const char *key;
    cache_object_t *next;
    cache_info info;
    void *vobj;
};

入力と出力

  • **入力**: 構造体メンバとして、キャッシュキー（key）、次オブジェクトへのポインタ（next）、メタデータ（info）、および実装固有のデータ（vobj）を保持します。
  • **出力**: 本構造体はキャッシュストレージの各実装において、特定のキャッシュエントリを識別・管理するために使用されます。

前提条件と不変条件

  • cache_object_t 型（typedef struct cache_object cache_object_t;）として定義されています。
  • vobj は不透明なポインタであり、具体的なキャッシュストレージ実装（mod_cache_disk 等）がその内容を定義します。

アルゴリズムまたは振る舞い

  • キャッシュの検索や保存の過程で、特定のキーに関連付けられたオブジェクトを保持します。
  • next ポインタにより、キャッシュオブジェクトのリスト構造を形成することが可能です。
  • info メンバを通じて、キャッシュされたコンテンツの有効期限やステータスを管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache.h:81 にて cache_object_t として型定義されています。
  • **呼び出し先**: 提供されたソース excerpt 内には、本構造体を利用する具体的な関数呼び出しや、本構造体が参照する先の関数に関する情報は含まれていません。

関連データ構造

  • cache_info: キャッシュされたコンテンツのメタデータを保持する構造体。
  • cache_object_t: cache_object への型定義。

失敗モードと境界条件

  • 本構造体自体はデータ保持用であり、失敗モードは実装側のメモリ管理やストレージの読み書き処理に依存します。

保守担当者向けメモ

  • vobj は実装依存の領域であるため、特定のストレージプロバイダ（mod_cache_disk や mod_cache_socache）を調査する際は、このポインタが指す具体的な構造体定義を確認してください。
