set_polled_events

定義

  • 種別: function
  • 定義箇所: support/ab.c:518

目的

特定の接続（connection）に関連付けられたポーリングイベント（reqevents）を更新し、グローバルなポーリングセット（readbits）への登録状態を同期させる。

シグネチャまたは形状

static void set_polled_events(struct connection *c, apr_int16_t new_reqevents)

入力と出力

  • **入力**:
  • c: 状態を更新する対象の connection 構造体へのポインタ。
  • new_reqevents: 設定すべき新しいイベントフラグ（APR_POLLIN, APR_POLLOUT 等）。
  • **出力**: なし（void）。

前提条件と不変条件

  • c->pollfd が有効な apr_pollfd_t 構造体として初期化されていること。
  • readbits がグローバルな apr_pollset_t として定義されていること。

アルゴリズムまたは振る舞い

1. 現在のイベント設定（c->pollfd.reqevents）と new_reqevents を比較する。 2. 設定に変更がない場合は何もしない。 3. 現在のイベントが 0 でない場合、apr_pollset_remove を呼び出し、既存のイベントをポーリングセットから削除する。 4. 新しいイベントが 0 でない場合、c->pollfd.reqevents を更新し、apr_pollset_add を呼び出してポーリングセットに再登録する。 5. 各 APR 操作でエラーが発生した場合は apr_err を呼び出し、エラーメッセージを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: support/ab.c 内の各所（554, 825, 829, 884, 887, 910, 1610, 1613, 2102 行目）。主に接続状態の変化や I/O 待ちの切り替え時に呼び出される。
  • **呼び出し先**:
  • apr_pollset_remove
  • apr_pollset_add
  • apr_err

関連データ構造

  • struct connection: 接続状態と pollfd を保持する。
  • apr_pollfd_t: ポーリング対象のファイル記述子とイベント情報を保持する APR 構造体。
  • apr_pollset_t: 監視対象のイベントセット（readbits）。

失敗モードと境界条件

  • apr_pollset_remove または apr_pollset_add が APR_SUCCESS 以外の値を返した場合、apr_err が呼び出され、プログラムが異常終了する可能性がある。
  • new_reqevents が 0 の場合、ポーリングセットからの削除のみが行われ、追加は行われない。

保守担当者向けメモ

  • 本関数は readbits というグローバル変数に依存している。
  • 接続のライフサイクル管理において、I/O 待ちの種別を切り替える際の唯一のインターフェースとして機能している。
