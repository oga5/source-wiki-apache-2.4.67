merge_substitute_dcfg

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_substitute.c:84

目的

ディレクトリ単位の設定構造体 (subst_dir_conf) をマージし、親ディレクトリの設定と現在のディレクトリの設定を統合する。

シグネチャまたは形状

static void *merge_substitute_dcfg(apr_pool_t *p, void *basev, void *overv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • basev: 親ディレクトリの設定 (subst_dir_conf*)。
  • overv: 現在のディレクトリの設定 (subst_dir_conf*)。
  • **出力**:
  • マージされた新しい subst_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および overv は subst_dir_conf 型にキャスト可能であること。
  • p は有効な apr_pool_t であること。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して新しい subst_dir_conf 構造体をメモリ上に確保する。 2. inherit_before フラグに基づき、置換パターンの配列 (patterns) を結合する。

  • inherit_before が 1 の場合: 親のパターンを先に、現在のパターンを後に結合する。
  • それ以外の場合: 現在のパターンを先に、親のパターンを後に結合する。

    • 3. max_line_length を決定する。現在の設定で値が設定されている場合はそれを優先し、未設定の場合は親の値を継承する。 4. max_line_length_set フラグを、親と現在の設定の論理和で更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_substitute.c:761 (Apache の設定マージ機構により呼び出される)。
  • **呼び出し先**: apr_palloc, apr_array_append (いずれも APR ライブラリ関数)。

関連データ構造

  • subst_dir_conf: ディレクトリごとの設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある。
  • inherit_before が -1 の場合、親の設定値がそのまま継承される。

保守担当者向けメモ

  • 本関数は Apache の設定マージ用コールバックとして機能する。
  • SubstituteInheritBefore ディレクティブの挙動を制御するロジックが含まれており、変更時は継承順序の意図を維持すること。
