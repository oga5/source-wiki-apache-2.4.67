merge_charset_dir_conf

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_charset_lite.c:117

目的

Apache HTTP Server のディレクトリ設定において、親コンテナの設定と現在のコンテナの設定をマージし、新しい設定構造体を作成する。

シグネチャまたは形状

static void *merge_charset_dir_conf(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親コンテナの設定 (charset_dir_t へのポインタ)。
  • overridesv: 現在のコンテナの設定 (charset_dir_t へのポインタ)。
  • **出力**:
  • マージされた新しい charset_dir_t 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は charset_dir_t 型としてキャスト可能であること。
  • p は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい charset_dir_t 構造体をメモリプール p 上に確保する。 2. 各設定項目について、overridesv に値が定義されている（初期値ではない）場合はその値を採用し、そうでない場合は basev の値を継承する。

  • charset_default および charset_source: 文字列ポインタの優先選択。
  • implicit_add: IA_INIT でない場合に overridesv を採用。
  • force_xlate: FX_INIT でない場合に overridesv を採用。

    • 3. マージ済みの構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_charset_lite.c:1136 にて参照されている（Apache の設定マージ機構から呼び出される）。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ関数）。

関連データ構造

  • charset_dir_t: ディレクトリごとの変換設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが終了する可能性がある。
  • basev や overridesv が NULL の場合、キャストおよびメンバアクセス時に未定義動作が発生する可能性がある。

保守担当者向けメモ

  • 本関数は Apache の設定マージフックとして機能する。新しい設定項目を charset_dir_t に追加した場合は、必ず本関数内で適切なマージロジックを実装すること。
