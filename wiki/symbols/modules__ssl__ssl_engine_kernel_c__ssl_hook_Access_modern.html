ssl_hook_Access_modern

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:913

目的

TLS Post-Handshake Authentication (PHA) を利用し、アクセス制御フェーズにおいてクライアント証明書の検証モード変更や検証深度の再設定を動的に適用する。

シグネチャまたは形状

static int ssl_hook_Access_modern(request_rec *r, SSLSrvConfigRec *sc, SSLDirConfigRec *dc, SSLConnRec *sslconn, SSL *ssl)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • sc, dc: サーバーおよびディレクトリごとの SSL 設定。
  • sslconn: SSL 接続コンテキスト。
  • ssl: OpenSSL の SSL オブジェクト。
  • **出力**:
  • OK: 処理成功。
  • DECLINED: 設定変更不要。
  • HTTP_FORBIDDEN: 再ネゴシエーション不可（スレーブ接続時）または PHA 失敗時。

前提条件と不変条件

  • dc->nVerifyClient または sc->server->auth.verify_mode が SSL_CVERIFY_UNSET ではない場合に実行される。
  • スレーブ接続（r->connection->master が存在）では再ネゴシエーションが禁止されており、HTTP_FORBIDDEN を返す。

アルゴリズムまたは振る舞い

1. 現在の検証モード（vmode_inplace）と、設定から要求されるモード（vmode_needed）を比較。 2. 検証深度（verify_depth）の縮小、または検証モードの厳格化が必要な場合に change_vmode を真とする。 3. change_vmode が真の場合:

  • fill_reneg_buffer を呼び出し、リクエストボディをバッファリングする。
  • SSL_set_verify で新しいモードと ssl_callback_SSLVerify を設定。
  • SSL_verify_client_post_handshake を実行し、PHA を開始。
  • SSL_peek を呼び出し、ハンドシェイクをトリガーする。
  • ssl_check_post_client_verify で最終的な検証結果を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_kernel.c 内の ssl_hook_Access から呼び出される。
  • **呼び出し先**: fill_reneg_buffer, SSL_set_verify, SSL_verify_client_post_handshake, SSL_do_handshake, SSL_peek, ssl_check_post_client_verify, modssl_set_app_data2。

関連データ構造

  • SSLSrvConfigRec, SSLDirConfigRec: SSL 設定保持用。
  • SSLConnRec: 接続ごとの状態（検証深度など）を保持。
  • request_rec: Apache リクエスト情報。

失敗モードと境界条件

  • **スレーブ接続**: r->connection->master が存在する場合、再ネゴシエーションを禁止し HTTP_FORBIDDEN を返す。
  • **バッファリング失敗**: fill_reneg_buffer が失敗した場合、そのエラーコードを返す。
  • **PHA 失敗**: SSL_verify_client_post_handshake が 1 を返さない場合、HTTP_FORBIDDEN を返す。

保守担当者向けメモ

  • 本関数は TLS 1.3 以降の Post-Handshake Authentication を想定している。
  • SSL_peek によるハンドシェイクのトリガーは、OpenSSL の仕様に依存する副作用を利用した実装である。
  • sslconn->verify_depth は OpenSSL 内部ではなく、本モジュールが管理する SSLConnRec を介して追跡されている。
