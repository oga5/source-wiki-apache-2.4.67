md_timeperiod_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_time.h:25

目的

証明書の有効期限や更新間隔など、特定の開始時刻から終了時刻までの期間を表現および管理するために使用されます。

シグネチャまたは形状

不透明な構造体として定義されています。

typedef struct md_timeperiod_t md_timeperiod_t;

入力と出力

本シンボル自体は型定義ですが、関連する操作関数を通じて以下の入出力が行われます。

  • **入力**: md_json_set_timeperiod 等の関数に対し、期間情報を保持する構造体として渡されます。
  • **出力**: md_cert_get_valid や md_json_get_timeperiod 等の関数により、証明書の有効期間や JSON から読み取った期間情報が本構造体に格納されます。

前提条件と不変条件

  • 期間の開始時刻と終了時刻を保持する論理的なコンテナとして機能します。
  • 内部的な数値比較や期間の包含判定には、md_time.h で提供されるヘルパー関数（md_timeperiod_contains 等）を使用することが推奨されます。

アルゴリズムまたは振る舞い

  • 証明書の有効性判定、OCSP レスポンスの有効期間管理、および ACME プロトコルにおける時間枠の計算に使用されます。
  • JSON 形式へのシリアライズおよびデシリアライズをサポートしており、永続化された設定情報とメモリ上の状態を相互変換可能です。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/md/md_crypt.c: 証明書の有効期間取得 (md_cert_get_valid)
  • modules/md/md_json.c: JSON データの読み書き (md_json_set_timeperiod, md_json_get_timeperiod)
  • modules/md/md_ocsp.c: OCSP レスポンスの有効性確認
  • **呼び出し先**:
  • 本シンボルはデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • apr_time_t (APR): 時間の基底型として内部的に利用されます。
  • md_json_t: JSON 形式での期間情報の保存・読み込みに関連します。
  • md_cert_t: 証明書の有効期間を抽出する際の対象となります。

失敗モードと境界条件

  • 期間の開始時刻が終了時刻よりも未来に設定されている場合、論理的な不整合が生じる可能性があります。
  • JSON 変換時に期間情報が欠落している場合、md_json_get_timeperiod はエラーを返す可能性があります。

保守担当者向けメモ

  • 本構造体は md_time.h で不透明型として定義されているため、内部フィールドに直接アクセスせず、必ず提供されている API を経由して操作してください。
  • 期間の計算ロジックを変更する場合は、modules/md/md_time.c の実装と整合性を確認してください。
