md_ocsp_provide_status

定義

  • **種別**: function
  • **定義箇所**: modules/md/mod_md_ocsp.c:108

目的

mod_ssl からの要求に応じて、キャッシュされた OCSP レスポンスを供給します。

シグネチャまたは形状

int md_ocsp_provide_status(server_rec *s, conn_rec *c,
                           const char *id, apr_size_t id_len,
                           ap_ssl_ocsp_copy_resp *cb, void *userdata)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • c: 接続レコード。
  • id: OCSP レスポンスを識別するための ID。
  • id_len: id の長さ。
  • cb: レスポンスをコピーするためのコールバック関数。
  • userdata: コールバックに渡すユーザーデータ。
  • **出力**:
  • OK: ステータス供給成功時。
  • DECLINED: ステータス供給が不要、または取得失敗時。

前提条件と不変条件

  • mod_watchdog がロードされている必要があります。
  • staple_here 関数により、現在の設定で OCSP Stapling が有効であると判定される必要があります。

アルゴリズムまたは振る舞い

1. md_config_get を使用してサーバー設定を取得し、staple_here で OCSP Stapling が有効か確認します。無効な場合は DECLINED を返します。 2. ログに取得対象のドメイン名またはホスト名を出力します。 3. md_ocsp_get_status を呼び出し、ストレージから DER 形式の OCSP レスポンスを取得して cb コールバック経由で供給します。 4. md_ocsp_get_status が APR_STATUS_IS_ENOENT を返した場合は、キャッシュが存在しないものとして DECLINED を返します。 5. 正常終了時は OK を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c 内の ap_hook_ssl_ocsp_get_resp_hook に登録され、mod_ssl から呼び出されます。
  • **呼び出し先**:
  • staple_here (設定確認)
  • md_ocsp_get_status (レスポンス取得)
  • ap_log_cerror (ログ出力)

関連データ構造

  • md_srv_conf_t: サーバー設定。
  • md_t: 管理対象ドメイン情報。
  • ap_ssl_ocsp_copy_resp: mod_ssl が提供するレスポンスコピー用インターフェース。

失敗モードと境界条件

  • staple_here が偽を返す場合、OCSP Stapling はスキップされ DECLINED が返されます。
  • キャッシュファイルが存在しない場合 (ENOENT) は DECLINED が返されます。

保守担当者向けメモ

  • 本関数は mod_ssl のフックとして動作するため、戻り値の OK/DECLINED は mod_ssl の処理フローに直接影響します。
  • 実際の OCSP 通信や ASN.1 解析の詳細は md_ocsp.c に委譲されています。
