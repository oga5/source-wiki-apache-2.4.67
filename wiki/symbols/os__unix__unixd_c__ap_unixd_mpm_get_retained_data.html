ap_unixd_mpm_get_retained_data

定義

  • 種別: function
  • 定義箇所: os/unix/unixd.c:449

目的

MPM（Multi-Processing Module）が再起動を跨いで保持すべき状態（ap_unixd_mpm_retained_data）を取得または初期化します。

シグネチャまたは形状

ap_unixd_mpm_retained_data *ap_unixd_mpm_get_retained_data(void)

入力と出力

  • **入力**: なし
  • **出力**: ap_unixd_mpm_retained_data 構造体へのポインタ。

前提条件と不変条件

  • グローバル変数 retained_data が初期化されていない場合、ap_retained_data_create を用いてメモリを確保し、ap_pglobal プールにクリーンアップ関数 retained_data_cleanup を登録します。
  • 初回呼び出し時に mpm_state は AP_MPMQ_STARTING に設定されます。

アルゴリズムまたは振る舞い

1. 静的変数 retained_data が存在するか確認します。 2. 存在しない場合、ap_retained_data_create で領域を確保し、ap_pglobal プールにクリーンアップ処理を登録します。 3. 確保した領域の mpm_state を AP_MPMQ_STARTING に初期化します。 4. 最後に retained_data ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • os/unix/unixd.c:541
  • server/mpm/event/event.c:3625
  • server/mpm/prefork/prefork.c:1307
  • server/mpm/worker/worker.c:2079
  • **呼び出し先**:
  • ap_retained_data_create
  • apr_pool_pre_cleanup_register
  • retained_data_cleanup

関連データ構造

  • ap_unixd_mpm_retained_data: MPM の状態を保持する構造体。
  • ap_pglobal: サーバーのライフサイクル全体で生存するグローバルプール。

失敗モードと境界条件

  • ap_retained_data_create がメモリ不足等で失敗した場合の挙動については、本ソースコード内では明示されていません。

保守担当者向けメモ

  • 本関数は、サーバーの再起動後も生存させる必要がある MPM 固有の状態を管理するためのシングルトン的なアクセスポイントです。
  • retained_data は ap_pglobal プールに関連付けられているため、サーバーの停止時に自動的にクリーンアップされます。
