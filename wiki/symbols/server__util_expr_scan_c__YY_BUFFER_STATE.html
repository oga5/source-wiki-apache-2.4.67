YY_BUFFER_STATE

定義

  • 種別: struct へのポインタ型（typedef struct yy_buffer_state *YY_BUFFER_STATE;）
  • 定義箇所: server/util_expr_scan.c:179

目的

Flex によって生成された字句解析器において、入力ストリームの状態（バッファ）を管理するための不透明なハンドルとして機能します。

シグネチャまたは形状

struct yy_buffer_state へのポインタ型です。具体的な構造体のメンバ定義は本ソースコードの範囲外ですが、スキャナの入力バッファ、読み込み位置、および関連する状態情報を保持します。

入力と出力

  • **入力**: ap_expr_yy_create_buffer 等の生成関数に渡されるファイルポインタや文字列データ。
  • **出力**: スキャナ制御関数（ap_expr_yy_switch_to_buffer 等）の引数として使用され、スキャナの現在の入力ソースを切り替えるために利用されます。

前提条件と不変条件

  • yyscan_t 型の有効なスキャナインスタンスと組み合わせて使用する必要があります。
  • 確保されたバッファは、不要になった際に ap_expr_yy_delete_buffer を呼び出して適切に解放する必要があります。

アルゴリズムまたは振る舞い

スキャナの入力ソースを管理します。ap_expr_yy_scan_string や ap_expr_yy_scan_bytes を通じて生成され、ap_expr_yy_switch_to_buffer を介してスキャナの現在の入力ストリームとして設定されます。また、yy_buffer_stack を通じて複数のバッファをスタック管理することが可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_expr_scan.c 内の各種スキャナ制御関数（ap_expr_yy_switch_to_buffer, ap_expr_yy_create_buffer, ap_expr_yy_delete_buffer, ap_expr_yy_flush_buffer, ap_expr_yypush_buffer_state, ap_expr_yy_scan_buffer, ap_expr_yy_scan_string, ap_expr_yy_scan_bytes）。
  • **呼び出し先**: 本シンボル自体は型定義であり、関数呼び出しの対象ではありません。

関連データ構造

  • struct yy_buffer_state: 本型が指し示す実体。
  • yyscan_t: スキャナのインスタンスを保持する型。
  • yy_buffer_stack: バッファ状態を管理するスタック配列。

失敗モードと境界条件

  • メモリ不足時にバッファ生成関数が NULL を返す可能性があります。
  • 無効な YY_BUFFER_STATE をスキャナ制御関数に渡した場合の動作は未定義です。

保守担当者向けメモ

  • 本シンボルは Flex が自動生成したコードの一部です。直接的な変更は server/util_expr_scan.l に対して行う必要があります。
  • YY_CURRENT_BUFFER マクロを通じて、現在アクティブなバッファ状態を参照できます。
