h2_ngheader

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.h:379

目的

nghttp2 ライブラリが要求するヘッダー形式（nghttp2_nv 配列）への変換結果を保持し、HTTP/2 レスポンスやトレーラーの構築を支援するために使用されます。

シグネチャまたは形状

typedef struct h2_ngheader {
    nghttp2_nv *nv;
    apr_size_t nvlen;
} h2_ngheader;

入力と出力

  • **入力**: h2_res_create_ngheader や h2_res_create_ngtrailer などの関数において、メモリプール（apr_pool_t）から割り当てられ、ヘッダーデータが格納されます。
  • **出力**: 変換された nghttp2_nv 配列のポインタと、その要素数（nvlen）を保持します。

前提条件と不変条件

  • nv は nghttp2 が処理可能な形式のヘッダー配列を指す必要があります。
  • nvlen は nv 配列の有効な要素数と一致している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、HTTP/2 プロトコルスタック（nghttp2）へ渡すためのヘッダー情報のコンテナとして機能します。
  • h2_util.c 内の ngheader_create 関数等により、メモリプールから動的に確保され、初期化されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/http2/h2_push.h
  • modules/http2/h2_session.c
  • modules/http2/h2_stream.c
  • modules/http2/h2_util.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • nghttp2_nv: nghttp2 ライブラリが定義する名前・値ペアの構造体。
  • apr_pool_t: メモリ管理に使用される Apache Portable Runtime のメモリプール。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc によるメモリ確保が失敗する可能性があります。
  • 変換失敗時: h2_res_create_ngheader 等の生成関数が APR_SUCCESS 以外のステータスを返す場合があります。

保守担当者向けメモ

  • 本構造体は h2_util.c で定義される生成関数と密接に関連しています。変更を加える際は、h2_res_create_ngheader および h2_res_create_ngtrailer の実装を確認してください。
  • メモリ所有権は apr_pool_t に依存しているため、ライフサイクル管理には注意が必要です。
