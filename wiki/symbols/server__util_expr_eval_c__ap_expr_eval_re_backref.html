ap_expr_eval_re_backref

定義

  • 種別: function
  • 定義箇所: server/util_expr_eval.c:193

目的

正規表現の評価結果から、指定されたインデックスのキャプチャグループ（バックリファレンス）に対応する文字列を抽出して返します。

シグネチャまたは形状

static const char *ap_expr_eval_re_backref(ap_expr_eval_ctx_t *ctx, unsigned int n)

入力と出力

  • **入力**:
  • ctx: 式評価コンテキスト（ap_expr_eval_ctx_t）。正規表現の照合結果（re_pmatch）およびソース文字列（re_source）を保持。
  • n: 抽出対象のキャプチャグループ番号。
  • **出力**:
  • 成功時: キャプチャされた部分文字列（apr_pstrndup によりリクエストプールから割り当て）。
  • 失敗または該当なし: 空文字列 ""。

前提条件と不変条件

  • ctx->re_pmatch および ctx->re_source が有効であること。
  • 指定されたインデックス n が、実行された正規表現の照合結果数（ctx->re_nmatch）の範囲内であること。

アルゴリズムまたは振る舞い

1. コンテキスト内の正規表現照合結果（re_pmatch）およびソース文字列（re_source）の存在を確認し、無効な場合は空文字列を返す。 2. 指定されたインデックス n が re_nmatch を超えていないか検証し、超えている場合は空文字列を返す。 3. re_pmatch[n] を使用して、ソース文字列内の該当範囲（開始位置 rm_so から終了位置 rm_eo）を特定する。 4. 該当範囲の長さが 0 の場合は空文字列を返す。 5. 該当範囲の文字列を apr_pstrndup を用いてリクエストプール上に複製し、そのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_expr_eval.c:171（ap_expr_eval_var 内での参照が確認されています）。
  • **呼び出し先**: apr_pstrndup（APR ライブラリ関数）。

関連データ構造

  • ap_expr_eval_ctx_t: 式評価の実行状態を保持する構造体。re_pmatch（regmatch_t 配列）、re_source（ソース文字列へのポインタ）、re_nmatch（照合数）を含む。

失敗モードと境界条件

  • ctx->re_pmatch が NULL の場合、または re_source が空の場合、安全に空文字列を返す。
  • インデックス n が re_nmatch の範囲外である場合、空文字列を返す。
  • キャプチャされた文字列の長さが 0 の場合、空文字列を返す。

保守担当者向けメモ

  • 本関数は ap_expr エンジンにおける正規表現のバックリファレンス解決専用です。
  • 戻り値はリクエストプール（ctx->p）から割り当てられるため、呼び出し側でのメモリ解放は不要です。
