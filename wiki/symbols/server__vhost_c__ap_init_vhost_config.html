ap_init_vhost_config

定義

  • 種別: function
  • 定義箇所: server/vhost.c:133

目的

仮想ホスト構成管理のための内部データ構造を初期化し、設定チェック用のフックを登録します。

シグネチャまたは形状

void ap_init_vhost_config(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（メモリ割り当て用プール）
  • **出力**: なし（副作用としてグローバル変数の初期化とフック登録を行う）

前提条件と不変条件

  • 本関数はサーバーの初期化プロセスにおいて呼び出される必要があります。
  • 呼び出し後、iphash_table はゼロクリアされ、default_list は NULL にリセットされます。

アルゴリズムまたは振る舞い

1. iphash_table を memset によりゼロクリアします。 2. グローバル変数 default_list を NULL に初期化します。 3. ap_hook_check_config を使用して、設定チェックフェーズで vhost_check_config 関数が実行されるよう登録します（フック順序は APR_HOOK_MIDDLE）。

呼び出し元と呼び出し先

  • **呼び出し元**: server/config.c:2214
  • **呼び出し先**: vhost_check_config（フック登録経由）

関連データ構造

  • iphash_table: 仮想ホストの IP/ポート情報を保持するハッシュテーブル。
  • default_list: デフォルトの仮想ホストリストを指すポインタ。

失敗モードと境界条件

  • 本関数自体はメモリ確保を行わないため、直接的な失敗モードはありません。ただし、apr_pool_t が無効な場合は動作が保証されません。

保守担当者向けメモ

  • 本関数は server/vhost.c 内の静的変数群をリセットする役割を持ちます。
  • vhost_check_config フックは、設定ファイルの解析完了後に仮想ホストの整合性を検証するために使用されます。
