ef_filter_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_ext_filter.c:43

目的

外部の Unix スタイルフィルタプログラムを Apache HTTP Server のフィルタとして定義・管理するための設定構造体です。

シグネチャまたは形状

typedef struct ef_filter_t {
    const char *name;
    enum {INPUT_FILTER=1, OUTPUT_FILTER} mode;
    ap_filter_type ftype;
    const char *command;
    const char *enable_env;
    const char *disable_env;
    char **args;
    const char *intype;
    const char *outtype;
    int preserves_content_length;
} ef_filter_t;

入力と出力

  • **入力**: ExtFilterDefine ディレクティブによって解析されたコマンドパス、引数、適用条件（MIMEタイプ、環境変数）が格納されます。
  • **出力**: フィルタ処理の実行時に、この構造体の定義に基づいて外部プロセスが起動され、データストリームの変換が行われます。

前提条件と不変条件

  • intype が INTYPE_ALL の場合、すべての MIME タイプを処理対象とします。
  • outtype が OUTTYPE_UNCHANGED の場合、出力の MIME タイプは変更されません。
  • フィルタの実行には request_rec のプール（r->pool）が使用され、リクエスト終了時にメモリが解放されます。

アルゴリズムまたは振る舞い

  • ExtFilterDefine ディレクティブの解析時にインスタンス化されます。
  • フィルタ適用時、enable_env および disable_env に基づいて実行の可否が判定されます。
  • 外部プロセス起動時には、この構造体に保持された command と args を使用してパイプ経由でデータが受け渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_cmd（設定解析時）、find_filter_def（フィルタ検索時）、get_cfg_string（設定文字列取得時）から参照されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ef_server_t: サーバー単位の設定として、この構造体を保持するハッシュテーブルを持ちます。
  • ef_ctx_t: リクエストごとの実行コンテキストであり、この構造体で定義されたフィルタの実行状態を管理します。

失敗モードと境界条件

  • フィルタの入出力タイプは、現在の実装では単一の MIME タイプのみをサポートしています。
  • 外部プロセスがブロックした場合、EAGAIN を検知してポーリングを行う設計となっています。

保守担当者向けメモ

  • INTYPE_ALL および OUTTYPE_UNCHANGED はマクロで定義されたポインタ値であり、比較時には注意が必要です。
  • 外部プロセスは apr_pool_note_subprocess を通じてリクエストの生存期間に紐付けられており、リクエスト終了時に確実に終了するよう設計されています。
