h2_push_digest_type

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_push.h:35

目的

HTTP/2 Server Push における重複排除（Push Diary）で使用するハッシュ計算アルゴリズムの種類を識別するために定義されています。

シグネチャまたは形状

typedef enum {
    H2_PUSH_DIGEST_APR_HASH,
    H2_PUSH_DIGEST_SHA256
} h2_push_digest_type;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力を持つ関数ではありません。
  • 設定値として h2_push_diary の生成時や、h2_push 構造体のメンバとして利用されます。

前提条件と不変条件

  • H2_PUSH_DIGEST_SHA256 を選択する場合、実行環境で OpenSSL が利用可能である必要があります。
  • H2_PUSH_DIGEST_APR_HASH はフォールバックとして利用されます。

アルゴリズムまたは振る舞い

  • H2_PUSH_DIGEST_APR_HASH: apr_hashfunc_default を使用したハッシュ計算を指定します。
  • H2_PUSH_DIGEST_SHA256: SHA256 を使用したハッシュ計算を指定します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/http2/h2_push.c:549: diary_create 関数の引数として使用。
  • modules/http2/h2_push.h:89: h2_push 構造体内のメンバ dtype の型として使用。
  • 呼び出し先: 情報なし。

関連データ構造

  • h2_push_diary: この型を使用してハッシュアルゴリズムを決定します。
  • h2_push: この型をメンバとして保持します。

失敗モードと境界条件

  • 実行環境が SHA256 をサポートしていない場合に H2_PUSH_DIGEST_SHA256 を選択した際の挙動については、本ソース excerpt に記載がありません。

保守担当者向けメモ

  • 本列挙型は、mod_h2 における重複排除ロジックの柔軟性を確保するために存在します。将来的に新しいハッシュアルゴリズムを追加する場合は、この列挙型を拡張し、対応する計算ロジックを h2_push.c に実装してください。
