h2_headers_create

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_headers.c:119

目的

HTTP/2 レスポンスヘッダーおよび関連メタデータを保持するための h2_headers 構造体を初期化し、メモリプール上に割り当てる。

シグネチャまたは形状

h2_headers *h2_headers_create(int status, const apr_table_t *headers_in, 
                              const apr_table_t *notes, apr_off_t raw_bytes,
                              apr_pool_t *pool)

入力と出力

  • **入力**:
  • status: HTTP ステータスコード。
  • headers_in: ヘッダー情報のテーブル（apr_table_t）。NULL の場合は空のテーブルが作成される。
  • notes: リクエストに関連するノート情報のテーブル。NULL の場合は空のテーブルが作成される。
  • raw_bytes: ヘッダーのバイト数（本関数内では未使用）。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • 初期化済みの h2_headers 構造体へのポインタ。

前提条件と不変条件

  • 指定された pool は、生成される h2_headers 構造体およびその内部のテーブル（headers, notes）の生存期間を管理するのに十分な寿命を持つ必要がある。
  • headers_in または notes が提供された場合、それらは apr_table_clone によって指定プールへ複製される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して h2_headers 構造体分のメモリをゼロ初期化して確保する。 2. status フィールドに引数の値を設定する。 3. headers_in が存在すれば apr_table_clone で複製し、存在しなければ apr_table_make で 5 エントリ分の空テーブルを作成して headers フィールドに格納する。 4. notes についても同様に、存在すれば複製、存在しなければ空テーブルを作成して notes フィールドに格納する。 5. 構築した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2_filter.c:534
  • modules/http2/h2_headers.c:155, 187, 192
  • modules/http2/h2_stream.c:288
  • modules/http2/h2_ws.c:173, 192
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_table_clone (APR)
  • apr_table_make (APR)

関連データ構造

  • h2_headers: 本関数が生成するメイン構造体。
  • apr_table_t: ヘッダーおよびノートを保持する APR のテーブル型。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数がプロセスを終了させる可能性がある（APR の標準的な挙動）。
  • raw_bytes 引数は関数内で使用されていないため、呼び出し側で指定しても構造体には反映されない。

保守担当者向けメモ

  • 本関数は h2_headers のメモリ所有権を pool に委譲する。呼び出し側は、生成された構造体が不要になった際に pool が適切にクリアされることを確認すること。
  • raw_bytes 引数は将来的な拡張用、あるいはインターフェースの互換性のために存在している可能性がある。
