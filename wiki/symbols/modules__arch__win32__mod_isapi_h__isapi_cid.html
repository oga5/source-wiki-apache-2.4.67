isapi_cid

定義

  • 種別: struct
  • 定義箇所: modules/arch/win32/mod_isapi.h:48

目的

Apache HTTP Server 上で実行される ISAPI 拡張に対し、リクエストに関連する接続状態やコンテキスト情報を保持するために使用されます。

シグネチャまたは形状

struct isapi_cid;

※不透明な構造体として定義されています。

入力と出力

  • 入力: mod_isapi.c 内の各コールバック関数（regfnGetServerVariable, regfnReadClient, regfnWriteClient, regfnServerSupportFunction など）の第一引数として渡されます。
  • 出力: 該当なし。

前提条件と不変条件

  • mod_isapi.c 内で apr_pcalloc を用いてリクエストのライフサイクルに合わせてメモリが割り当てられます。
  • ISAPI 拡張側で永続化してはなりません。

アルゴリズムまたは振る舞い

  • HCONN 型（isapi_cid *）として、ISAPI 拡張と Apache 間の通信インターフェースにおける接続識別子として機能します。
  • mod_isapi.c において、リクエスト処理中の状態管理および Apache の内部リクエスト情報へのアクセスポイントとして利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: mod_isapi.c 内の isapi_handler 等のリクエスト処理関数。
  • 呼び出し先: regfnGetServerVariable, regfnReadClient, regfnWriteClient, regfnServerSupportFunction, send_response_header 等の mod_isapi.c 内の静的関数。

関連データ構造

  • HCONN: isapi_cid * への型定義 (modules/arch/win32/mod_isapi.h:49)。
  • EXTENSION_CONTROL_BLOCK: ISAPI 拡張が利用するリクエストコンテキストの核となる構造体。

失敗モードと境界条件

  • 構造体の内部定義が公開されていないため、直接的な境界条件は不明です。
  • メモリ割り当て失敗時は apr_pcalloc の挙動に依存します。

保守担当者向けメモ

  • 本構造体は mod_isapi.c におけるリクエスト処理のコンテキスト保持に不可欠です。構造体のメンバを変更する場合は、mod_isapi.c 内のすべてのコールバック関数との整合性を確認してください。
