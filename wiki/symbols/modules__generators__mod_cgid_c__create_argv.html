create_argv

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgid.c:234

目的

CGIプログラム実行用の引数配列（argv）を構築します。環境変数やユーザー/グループ情報、およびクエリ文字列を解析し、exec システムコールに適した形式へ変換します。

シグネチャまたは形状

char **create_argv(apr_pool_t *p, char *path, char *user, char *group, char *av0, const char *args)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • path, user, group: 実行パス、ユーザー名、グループ名（指定がある場合のみ配列に追加）。
  • av0: プログラム名。
  • args: + で区切られた引数文字列。
  • **出力**:
  • char **: NULL 終端された引数文字列の配列。

前提条件と不変条件

  • args に = が含まれる場合、引数は無視され numwords は 0 となります。
  • 引数の総数は APACHE_ARG_MAX - 5 を超えないよう切り詰められます。
  • 返される配列は、指定されたプール p から割り当てられます。

アルゴリズムまたは振る舞い

1. args 文字列内の + をカウントし、引数の数を算出します。 2. apr_pcalloc を使用して、必要な要素数分のメモリを確保します。 3. path, user, group が提供されていれば、順次配列の先頭に格納します。 4. av0 を配列にコピーします。 5. args を + で分割し、各要素に対して以下の処理を行います：

  • ap_getword_nulls で単語を抽出。
  • ap_unescape_url で URL デコード。
  • ap_escape_shell_cmd でシェルメタ文字をエスケープ。

    • 6. 配列の末尾に NULL を設定して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_cgid.c:936 にて呼び出されています。
  • **呼び出し先**:
  • apr_pcalloc, apr_pstrdup (APRメモリ管理)
  • ap_strchr_c, ap_getword_nulls, ap_unescape_url, ap_escape_shell_cmd (Apacheユーティリティ)

関連データ構造

  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • args が NULL または空、あるいは = を含む場合、引数解析はスキップされます。
  • 引数が多すぎる場合、APACHE_ARG_MAX に基づく制限により切り詰めが発生します。

保守担当者向けメモ

  • 本関数は mod_cgid におけるプロセス実行準備の重要なステップです。引数のエスケープ処理はセキュリティに直結するため、変更時は ap_escape_shell_cmd の仕様を確認してください。
