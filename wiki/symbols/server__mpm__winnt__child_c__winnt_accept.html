winnt_accept

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/child.c:350

目的

Windows 版 Apache HTTP Server におけるリスナー・スレッドの本体。リスナーソケットでクライアント接続を待ち受け、確立された接続をワーカー・スレッドへ引き渡すための準備を行う。

シグネチャまたは形状

static unsigned int __stdcall winnt_accept(void *lr_)
  • 引数 lr_: リスナー設定情報 (ap_listen_rec *)。

入力と出力

  • **入力**: ap_listen_rec 構造体（リスナーソケット情報）。
  • **出力**: スレッド終了時に 1 を返す。

前提条件と不変条件

  • mpm_winnt モジュールが初期化され、リスナーソケットが有効であること。
  • AcceptFilter 設定に基づき、AcceptEx または標準 accept() のいずれかを使用する。
  • shutdown_in_progress フラグが偽である間、ループを継続する。

アルゴリズムまたは振る舞い

1. **初期化**: get_accept_filter でフィルタ設定を確認し、AcceptEx 使用時は WSAIoctl で関数ポインタを取得する。 2. **接続待機ループ**:

  • mpm_get_completion_context を呼び出し、接続コンテキストを取得する。
  • **AcceptFilter が connect の場合**: AcceptEx を発行し、WaitForMultipleObjectsEx で接続完了またはシャットダウンイベントを待機する。成功時は SO_UPDATE_ACCEPT_CONTEXT を設定し、lpfnGetAcceptExSockaddrs でアドレス情報を取得する。
  • **AcceptFilter が none の場合**: WSAEventSelect で FD_ACCEPT を監視し、accept() を呼び出す。

    • 3. **エラー処理**: AcceptEx 失敗時に一定回数（MAX_ACCEPTEX_ERR_COUNT）を超えると、自動的に AcceptFilter none へフォールバックする。 4. **終了**: shutdown_in_progress が真になるとループを脱出する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/child.c:913 (_beginthreadex によりスレッドとして起動)。
  • **呼び出し先**:
  • mpm_get_completion_context
  • lpfnAcceptEx / lpfnGetAcceptExSockaddrs (動的取得)
  • accept
  • ap_log_error

関連データ構造

  • ap_listen_rec: リスナーソケット情報。
  • winnt_conn_ctx_t: 接続状態と OVERLAPPED 構造体を保持するコンテキスト。
  • accept_filter_e: 接続受け入れフィルタの種別。

失敗モードと境界条件

  • **リソース枯渇**: mpm_get_completion_context が取得できない場合、一時的に待機（Sleep）する。
  • **接続エラー**: WSAECONNRESET や WSAEINVAL 等のソケットエラー発生時、ソケットをクローズし再試行またはフォールバックを行う。
  • **シャットダウン**: exit_event や max_requests_per_child_event がシグナルされると、待機を中断して終了する。

保守担当者向けメモ

  • winnt_conn_ctx_t の再利用ロジックにおいて、エラーパスでメモリリークが発生する可能性がある旨の指摘がソース内に存在する。
  • AcceptEx の失敗は VPN やファイアウォールソフトとの競合が原因となることが多いため、ログ出力とフォールバックの挙動は重要である。
