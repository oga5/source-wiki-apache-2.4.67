uldap_connection_unbind

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:225

目的

LDAP 接続のリソースを解放し、接続状態を破棄する。

シグネチャまたは形状

static apr_status_t uldap_connection_unbind(void *param)

入力と出力

  • **入力**: param (LDAP 接続構造体 util_ldap_connection_t へのポインタ)
  • **出力**: APR_SUCCESS を返す。

前提条件と不変条件

  • 入力 param が NULL の場合、関数は何もしない。
  • 接続がアクティブな場合、ldc->ldap は有効な LDAP ハンドルを指している必要がある。

アルゴリズムまたは振る舞い

1. param を util_ldap_connection_t *ldc にキャストする。 2. ldc が有効な場合:

  • USE_APR_LDAP_REBIND が定義されている環境で、リファラル追跡が有効であれば ldc->rebind_pool をクリアする。
  • ldc->ldap が存在する場合、ldap_unbind_s を呼び出して LDAP サーバーとの接続を終了し、ldc->ldap を NULL に設定する。
  • 接続状態フラグ ldc->bound を 0 にリセットする。

    • 3. APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し先**: ldap_unbind_s (LDAP SDK 関数)
  • **呼び出し元**: modules/ldap/util_ldap.c 内の各所（uldap_connection_find のエラーハンドリングや接続管理フローなど、計11箇所で直接呼び出されている）。

関連データ構造

  • util_ldap_connection_t: LDAP 接続状態を保持する構造体。

失敗モードと境界条件

  • ldc が NULL の場合、関数は安全に終了する。
  • ldc->ldap が既に NULL の場合、アンバインド処理はスキップされる。

保守担当者向けメモ

  • 本関数は apr_pool_cleanup_register 等のクリーンアップハンドラとして使用されることを想定したシグネチャ（void *param）となっている。
  • ldap_unbind_s は同期的なアンバインド処理を行うため、呼び出し元のスレッドがブロックされる可能性がある点に注意すること。
