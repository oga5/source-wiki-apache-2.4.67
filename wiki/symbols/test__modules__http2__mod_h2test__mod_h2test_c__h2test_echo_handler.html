h2test_echo_handler

定義

  • 種別: function
  • 定義箇所: test/modules/http2/mod_h2test/mod_h2test.c:132

目的

HTTP/2 テストスイートにおいて、クライアントから送信されたリクエストボディをそのままレスポンスとして返すエコー機能を提供します。また、クエリパラメータによる遅延挿入や意図的なエラー発生のシミュレーションを行います。

シグネチャまたは形状

static int h2test_echo_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r (Apache リクエスト構造体)
  • **出力**: int (Apache ハンドラステータスコード: OK, DECLINED, AP_FILTER_ERROR)
  • **クエリパラメータ**:
  • chunk_delay: 各チャンク送信間の遅延時間（例: "10ms"）。
  • fail_after: 指定バイト数送信後に接続を失敗させるための閾値。

前提条件と不変条件

  • r->handler が "h2test-echo" であること。
  • リクエストメソッドが GET または POST であること。
  • メモリ管理は r->pool に依存しており、リクエスト終了時に自動解放される。

アルゴリズムまたは振る舞い

1. **初期化**: r->handler を検証し、対象外であれば DECLINED を返す。 2. **パラメータ解析**: r->args を解析し、chunk_delay や fail_after の設定を抽出する。 3. **レスポンス準備**: Content-Length を無効化し、チャンク転送エンコーディングを有効化する。 4. **ボディ転送**: ap_get_client_block でリクエストボディを読み込み、apr_brigade_write でバケットブリゲードに書き込む。

  • chunk_delay が設定されている場合、チャンクごとに apr_sleep を実行する。
  • fail_after が設定されている場合、指定バイト数に達した時点で処理を中断しエラーを発生させる。

    • 5. **終了処理**: EOS (End of Stream) バケットを挿入し、ap_pass_brigade で送信する。 6. **エラーハンドリング**: 処理中にエラーが発生した場合、cleanup ラベルへ遷移し、必要に応じてエラーバケットを生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフックシステム（ap_hook_handler により登録）。
  • **呼び出し先**:
  • duration_parse (パラメータ解析用)
  • ap_get_client_block, ap_pass_brigade (Apache プロトコル API)
  • apr_brigade_write, apr_bucket_eos_create (APR バケット API)

関連データ構造

  • request_rec: リクエスト情報の保持。
  • apr_bucket_brigade: レスポンスデータのバッファリング。
  • conn_rec: 接続状態の管理。

失敗モードと境界条件

  • **不正なパラメータ**: r->args の解析に失敗した場合、ap_die を呼び出し HTTP_BAD_REQUEST を返す。
  • **転送エラー**: ap_pass_brigade 等でエラーが発生した場合、APR_EINVAL 等のステータスに基づきエラーバケットを生成してクライアントへ通知する。
  • **接続中断**: c->aborted が真の場合、処理を中断し OK を返す。

保守担当者向けメモ

  • 本関数はテスト専用であり、意図的にエラーを発生させるロジックが含まれています。プロダクション環境での使用は厳禁です。
  • fail_after のロジックは、指定バイト数を超えた場合に APR_EINVAL を返して強制終了させる設計となっています。
