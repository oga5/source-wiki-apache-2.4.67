dav_buffer_place_mem

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:173

目的

dav_buffer 構造体の現在の末尾位置に、指定されたメモリ領域の内容をコピーして配置します。

シグネチャまたは形状

void dav_buffer_place_mem(apr_pool_t *p, dav_buffer *pbuf, const void *mem, apr_size_t amt, apr_size_t pad)

入力と出力

  • p: メモリ確保に使用する apr_pool_t。
  • pbuf: 操作対象の dav_buffer 構造体。
  • mem: コピー元のメモリ領域へのポインタ。
  • amt: コピーするバイト数。
  • pad: バッファ拡張時に考慮する追加のパディングサイズ。
  • 出力: なし（pbuf の内容が更新されます）。

前提条件と不変条件

  • pbuf は初期化済みであること。
  • pbuf->cur_len は現在の有効なデータ長を指していること。
  • dav_check_bufsize により、amt + pad 分の容量が確保されることが保証されます。

アルゴリズムまたは振る舞い

1. dav_check_bufsize を呼び出し、pbuf の容量が amt + pad 分不足していないか確認し、必要に応じて拡張します。 2. memcpy を使用し、pbuf->buf の現在の末尾（cur_len）から amt バイト分、mem の内容をコピーします。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/fs/repos.c 内の複数箇所（ファイルシステムリポジトリのパスやURI構築時）。
  • 呼び出し先: dav_check_bufsize（modules/dav/main/util.c）。

関連データ構造

  • dav_buffer: バッファ本体（buf）、現在の長さ（cur_len）、確保済みサイズ（alloc_len）を保持する構造体。

失敗モードと境界条件

  • apr_pool_t によるメモリ確保が失敗した場合、dav_check_bufsize 経由でプロセスが異常終了する可能性があります（APRのメモリ管理仕様に依存）。
  • amt が 0 の場合、コピーは行われませんが、pad 分のバッファ拡張は試行されます。

保守担当者向けメモ

  • 本関数は pbuf->cur_len を更新しません。呼び出し側で必要に応じて cur_len を調整する必要があります。
  • pad 引数は、後続の書き込みを見越したバッファ拡張を効率化するために使用されます。
