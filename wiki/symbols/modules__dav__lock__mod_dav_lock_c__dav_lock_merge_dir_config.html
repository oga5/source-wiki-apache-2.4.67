dav_lock_merge_dir_config

定義

  • 種別: function
  • 定義箇所: modules/dav/lock/mod_dav_lock.c:47

目的

ディレクトリごとの設定構造体（dav_lock_dir_conf）をマージし、子ディレクトリの設定において値が未定義の場合に親ディレクトリの設定を継承させる。

シグネチャまたは形状

static void *dav_lock_merge_dir_config(apr_pool_t *p, void *base, void *overrides)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • base: 親ディレクトリの設定（dav_lock_dir_conf *）。
  • overrides: 子ディレクトリの設定（dav_lock_dir_conf *）。
  • **出力**:
  • マージ結果を格納した新しい dav_lock_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • base および overrides は dav_lock_dir_conf 型としてキャスト可能であること。
  • p は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい設定構造体 newconf をメモリプール p 上に確保する。 2. child->lockdb_path が設定されている（NULL ではない）場合はその値を採用し、設定されていない場合は parent->lockdb_path を採用する。 3. 決定した値を newconf->lockdb_path に格納し、newconf を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定フレームワーク（modules/dav/lock/mod_dav_lock.c:99 にてモジュール構造体の一部として登録）。
  • **呼び出し先**: 直接的な関数呼び出しはなし。apr_pcalloc を利用してメモリを確保する。

関連データ構造

  • dav_lock_dir_conf: ロックデータベースのパスを保持するディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが終了する可能性がある。
  • base または overrides が NULL の場合の挙動については、本ソースコード内では明示的なチェックは行われていない。

保守担当者向けメモ

  • 本関数は、子ディレクトリの設定が親の設定を上書き（オーバーライド）する標準的なマージロジックを実装している。
  • 将来的に dav_lock_dir_conf に新しい設定項目が追加された場合は、本関数内で適切にマージ処理を追加する必要がある。
