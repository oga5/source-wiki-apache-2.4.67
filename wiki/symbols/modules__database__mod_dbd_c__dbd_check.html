dbd_check

定義

  • 種別: function
  • 定義箇所: modules/database/mod_dbd.c:769

目的

データベース接続が有効であるかを確認し、エラーが発生した場合にはログ出力を行う。

シグネチャまたは形状

static apr_status_t dbd_check(apr_pool_t *pool, server_rec *s, ap_dbd_t *rec)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • s: サーバー設定を取得するための server_rec。
  • rec: 接続状態やドライバ情報を保持する ap_dbd_t 構造体。
  • **出力**:

前提条件と不変条件

  • rec および rec->driver、rec->handle は有効なポインタである必要がある。
  • 呼び出し元は、接続の生存期間を管理するプールを提供する必要がある。

アルゴリズムまたは振る舞い

1. apr_dbd_check_conn を呼び出し、ドライバ経由で接続の生存確認を行う。 3. 上記以外の場合、apr_dbd_error を使用してドライバからエラーメッセージを取得する。 4. 取得したエラーメッセージを ap_log_error を用いてエラーログに出力する。 5. 最後に、接続チェック時のステータスコード rv を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/database/mod_dbd.c:834
  • modules/database/mod_dbd.c:844
  • **呼び出し先**:
  • apr_dbd_check_conn (APR)
  • apr_dbd_error (APR)
  • ap_get_module_config (Apache Core)
  • ap_log_error (Apache Core)

関連データ構造

  • ap_dbd_t: データベース接続ハンドル。
  • svr_cfg: サーバーごとの設定コンテナ。

失敗モードと境界条件

  • apr_dbd_check_conn がエラーを返した場合、そのエラーコードがそのまま呼び出し元に伝播される。
  • ドライバからエラーメッセージが取得できない場合、ログには (unknown) と記録される。

保守担当者向けメモ

  • 本関数は mod_dbd 内の接続再利用ロジックにおいて、接続が切断されていないかを検証する重要な役割を担う。
  • ログ出力には APLOGNO(00639) が付与されている。
