process_echo_connection

定義

  • 種別: function
  • 定義箇所: modules/echo/mod_echo.c:130

目的

Apache HTTP Server の接続処理フックとして動作し、クライアントから受信したデータをそのまま送り返すエコーサーバー機能を実行する。

シグネチャまたは形状

static int process_echo_connection(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（現在の接続情報を保持する構造体）
  • **出力**: int（処理結果を示すステータスコード。DECLINED または OK）

前提条件と不変条件

  • EchoConfig 設定において bEnabled が真である必要がある。
  • ap_get_brigade を用いて行単位（AP_MODE_GETLINE）でデータを受信することを前提とする。

アルゴリズムまたは振る舞い

1. **初期化**: 設定を確認し、無効であれば DECLINED を返す。スコアボードを SERVER_BUSY_READ に更新する。 2. **ループ処理**:

  • ap_get_brigade でクライアントからデータを読み込む。
  • 読み込み失敗時（EOF/タイムアウト以外）はエラーログを出力してループを抜ける。
  • ソケットが未取得であれば取得し、タイムアウトを設定する。
  • スコアボードを SERVER_BUSY_WRITE に更新する。
  • 受信したデータにフラッシュバケットを追加し、ap_pass_brigade でクライアントへ送り返す。
  • スコアボードを SERVER_BUSY_KEEPALIVE に更新する。

    • 3. **終了**: リソースを解放し、スコアボードを SERVER_CLOSING に更新して OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_process_connection により登録されたフック経由で呼び出される（modules/echo/mod_echo.c:206）。
  • **呼び出し先**:
  • ap_get_module_config, ap_time_process_request, update_echo_child_status, apr_brigade_create, ap_get_brigade, ap_log_error, ap_get_conn_socket, apr_socket_timeout_set, apr_bucket_flush_create, ap_pass_brigade, apr_brigade_cleanup, apr_brigade_destroy

関連データ構造

  • conn_rec: 接続状態を管理する Apache の基本構造体。
  • EchoConfig: モジュールの有効/無効状態を保持する設定構造体。
  • apr_bucket_brigade: フィルタチェーンを流れるデータストリーム。

失敗モードと境界条件

  • pConfig->bEnabled が偽の場合、即座に終了する。
  • ap_get_brigade が APR_SUCCESS 以外を返した場合、EOF またはタイムアウト以外はエラーとしてログ出力しループを中断する。
  • 受信したブリゲードが空の場合、エラーとしてループを中断する。
  • ap_pass_brigade が失敗した場合、エラーログを出力してループを中断する。

保守担当者向けメモ

  • 本関数は AP_MODE_GETLINE を使用しているため、行終端文字を含まないデータやバイナリデータの扱いには注意が必要である。
  • 独自プロトコル実装のテンプレートとして機能するため、変更時は scoreboard の更新タイミングとフィルタチェーンの整合性に留意すること。
