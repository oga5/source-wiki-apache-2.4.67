usage

定義

  • 種別: function
  • 定義箇所: support/logresolve.c:116

目的

logresolve コマンドの利用方法（ヘルプメッセージ）を標準エラー出力に表示し、プログラムを異常終了させる。

シグネチャまたは形状

static void usage(void)

入力と出力

  • 入力: なし（グローバル変数 errfile および shortname を参照）。
  • 出力: なし（標準エラー出力へメッセージを書き出し、exit(1) を呼び出す）。

前提条件と不変条件

  • errfile が有効なファイルストリームとして初期化されていること。
  • shortname にプログラム名が格納されていること。

アルゴリズムまたは振る舞い

1. apr_file_printf を使用し、以下の情報を errfile に出力する。

  • プログラムの概要。
  • 使用法（Usage: %s [-s STATFILE] [-c]）。
  • オプションの説明（-s: 統計ファイル記録、-c: ダブルルックアップ実行）。

    • 2. exit(1) を呼び出し、プロセスを終了する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では明示的な呼び出し箇所は特定されていません。
  • 呼び出し先: apr_file_printf（APRライブラリ）、exit（標準ライブラリ）。
  • 補足: ツールが検出した他のファイル内の usage というシンボルは、本関数とは無関係な別定義です。

関連データ構造

  • errfile: 出力先のファイルハンドル（apr_file_t）。
  • shortname: プログラム名を示す文字列。

失敗モードと境界条件

  • errfile がクローズされている、または書き込み不可能な場合、apr_file_printf の動作は APR の仕様に依存する。

保守担当者向けメモ

  • 本関数は logresolve のコマンドライン引数仕様と同期している必要があります。オプションを追加した際は、必ずこのメッセージを更新してください。
