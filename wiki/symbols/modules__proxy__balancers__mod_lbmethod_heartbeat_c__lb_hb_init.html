lb_hb_init

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_heartbeat.c:362

目的

mod_lbmethod_heartbeat モジュールの初期化を行い、mod_heartmonitor が提供する共有メモリ（slotmem）への接続を確立する。

シグネチャまたは形状

static int lb_hb_init(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • p, plog, ptemp: Apache のメモリプール。
  • s: 現在のサーバー設定構造体。
  • **出力**:
  • OK (0): 正常終了。

前提条件と不変条件

  • mod_heartmonitor が共有メモリを生成している必要がある。
  • ap_slotmem_shm プロバイダーがロードされている必要がある。

アルゴリズムまたは振る舞い

1. 設定フェーズの初期段階（AP_SQ_MS_CREATE_PRE_CONFIG）であれば、何もせず OK を返す。 2. ap_lookup_provider を使用して shm プロバイダーを検索する。失敗した場合はエラーログを出力して終了する。 3. storage->attach を呼び出し、mod_heartmonitor が管理する共有メモリ（hm_serversmem）への接続を試みる。 4. 接続成功時、モジュールのコンテキスト（ctx->path）を "(slotmem)" に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_config により、Apache の設定完了時フックとして登録されている。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_state_query
  • ap_lookup_provider
  • ap_log_error
  • storage->attach (関数ポインタ経由)

関連データ構造

  • lb_hb_ctx_t: モジュールの設定コンテキスト。
  • server_rec: Apache のサーバー設定。
  • hm_serversmem: mod_heartmonitor と共有されるメモリ領域（グローバル変数）。

失敗モードと境界条件

  • shm プロバイダーが見つからない場合、エラーログを出力するが、プロセス自体は停止せず OK を返す。
  • mod_heartmonitor からの共有メモリ取得に失敗した場合、エラーログを出力するが、処理は継続する。

保守担当者向けメモ

  • 本関数は post_config フックで実行されるため、共有メモリのライフサイクルは mod_heartmonitor の初期化順序に依存する。
  • 共有メモリ接続失敗時の挙動は、後続の read_heartbeats 等で適切にハンドリングされる必要がある。
