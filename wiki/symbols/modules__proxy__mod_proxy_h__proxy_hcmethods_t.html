proxy_hcmethods_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:87

目的

プロキシのヘルスチェック（Health Check）手法を定義し、各手法の識別子、名称、および実装状況を管理するための構造体です。

シグネチャまたは形状

typedef struct {
    hcmethod_t method;
    char *name;
    int implemented;
} proxy_hcmethods_t;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。各フィールドは以下の通りです。

  • method: hcmethod_t 型の列挙値。ヘルスチェック手法を識別する。
  • name: 手法を表す文字列。
  • implemented: 実装済みか否かを示すフラグ（整数）。

前提条件と不変条件

  • proxy_hcmethods[] 配列の要素として静的に定義され、システム全体で共有されます。
  • implemented フィールドは、特定のヘルスチェック手法が現在のビルドや環境で利用可能かどうかを判定するために使用されます。

アルゴリズムまたは振る舞い

本構造体自体はアルゴリズムを持ちませんが、mod_proxy_balancer.c や mod_proxy_hcheck.c などの各コンポーネントが proxy_hcmethods 配列を参照し、利用可能なヘルスチェック手法の検索や検証を行います。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/proxy/mod_proxy.c (配列の定義)
  • modules/proxy/mod_proxy_balancer.c
  • modules/proxy/mod_proxy_hcheck.c
  • modules/proxy/proxy_util.c
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • hcmethod_t: method フィールドで使用される列挙型。
  • proxy_hcmethods[]: 本構造体のインスタンスを格納するグローバル配列。

失敗モードと境界条件

  • proxy_hcmethods 配列の走査時に、一致する method が見つからない場合の挙動は、参照側の実装に依存します。
  • implemented フラグが 0 の手法を選択しようとした場合の動作は、各ハンドラの実装によって制御されます。

保守担当者向けメモ

  • 本構造体は mod_proxy のヘルスチェック機能の拡張ポイントです。新しいヘルスチェック手法を追加する場合は、hcmethod_t の定義更新と併せて proxy_hcmethods 配列へのエントリ追加が必要です。
  • PROXY_DECLARE_DATA を使用して宣言されているため、モジュール間での共有が保証されています。
