format_byte_out

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_status.c:107

目的

バイト単位の数値を、人間が読みやすい単位（B, kB, MB, GB）に変換し、HTTP レスポンスストリームへ出力する。

シグネチャまたは形状

void format_byte_out(request_rec *r, apr_off_t bytes)

入力と出力

  • **入力**:
  • r: 現在のリクエストを処理するための request_rec 構造体。
  • bytes: 変換対象のバイト数 (apr_off_t)。
  • **出力**: なし（戻り値は void）。副作用として ap_rprintf を通じてレスポンスボディに整形済み文字列を書き込む。

前提条件と不変条件

  • KBYTE, MBYTE, GBYTE 定数が定義されている環境であること。
  • r が有効なリクエストオブジェクトであり、ap_rprintf による出力が可能であること。

アルゴリズムまたは振る舞い

入力された bytes の値に応じて以下の条件分岐を行い、適切な単位でフォーマットして出力する。 1. bytes < 5 * KBYTE: そのままバイト単位（"B"）で出力。 2. bytes < MBYTE / 2: キロバイト単位（"kB"）に変換し、小数点以下1桁まで出力。 3. bytes < GBYTE / 2: メガバイト単位（"MB"）に変換し、小数点以下1桁まで出力。 4. 上記以外: ギガバイト単位（"GB"）に変換し、小数点以下1桁まで出力。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_status.c:539

関連データ構造

  • request_rec: Apache のリクエスト処理用構造体。
  • apr_off_t: APR (Apache Portable Runtime) で定義されるファイルサイズやオフセットを扱う型。

失敗モードと境界条件

  • bytes が非常に大きい場合、float へのキャストによる精度低下が発生する可能性がある。
  • ap_rprintf が失敗した場合の戻り値チェックは行われない。

保守担当者向けメモ

  • 本関数は mod_status のレポート生成において、トラフィック統計の表示に使用される。
  • 単位のしきい値（5 * KBYTE 等）はハードコードされているため、表示形式を変更する場合はこのロジックを修正する必要がある。
