arg_item_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_include.c:152

目的

SSI（Server Side Includes）ディレクティブの引数（属性名と値のペア）を保持するための連結リスト構造体です。

シグネチャまたは形状

typedef struct arg_item {
    struct arg_item  *next;
    char             *name;
    apr_size_t        name_len;
    char             *value;
    apr_size_t        value_len;
} arg_item_t;

入力と出力

  • **入力**: SSI 解析プロセスにおいて、ディレクティブから抽出された文字列が name および value に格納されます。
  • **出力**: 連結リストのノードとして、後続の引数へのポインタ（next）を保持します。

前提条件と不変条件

  • name および value は、解析コンテキストのメモリプール（dpool）から割り当てられることが想定されています。
  • name_len および value_len は、対応する文字列の長さを保持します。

アルゴリズムまたは振る舞い

SSI 解析エンジンがディレクティブを走査する際、各引数をこの構造体に格納し、リストとして構築します。これにより、handle_... 系関数がディレクティブの引数を順次参照可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_include.c 内の解析ロジック（find_argument 等）がこの構造体をインスタンス化し、リストを構築します。
  • **呼び出し先**: ssi_internal_ctx 構造体内の argv フィールドが、このリストの先頭を保持します。

関連データ構造

  • ssi_internal_ctx: 解析状態を保持する構造体であり、arg_item_t のリスト（argv）を管理します。

失敗モードと境界条件

  • メモリ不足時: dpool からのメモリ割り当てが失敗した場合の挙動は、Apache のメモリ管理ポリシーに依存します。
  • 空の引数: name や value が空の場合、len フィールドが 0 となる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_include.c 内部でのみ使用される非公開構造体です。
  • 解析中のデータは dpool を使用するため、リクエスト終了時に自動的に解放されます。メモリリークを防ぐため、この構造体のノードを永続的なプールに割り当てないよう注意してください。
