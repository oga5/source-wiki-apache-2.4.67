order_update_from_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:120

目的

ACME サーバーから取得した JSON データを用いて、md_acme_order_t 構造体の状態を更新します。

シグネチャまたは形状

static void order_update_from_json(md_acme_order_t *order, md_json_t *json, apr_pool_t *p)

入力と出力

  • order: 更新対象の md_acme_order_t 構造体（ポインタ）。
  • json: サーバーから取得した ACME Order を表す md_json_t オブジェクト。
  • p: メモリ割り当てに使用する apr_pool_t。
  • 出力: なし（order 構造体の各フィールドを直接更新）。

前提条件と不変条件

  • order および json は有効なポインタであること。
  • p はメモリ確保のために有効なプールであること。
  • order->url が未設定の場合のみ、JSON から URL がコピーされます。

アルゴリズムまたは振る舞い

1. order->url が未設定であれば、JSON の url キーから値を複製して設定します。 2. JSON の status キーを解析し、order_st_from_str を用いて order->status を更新します。 3. JSON に authorizations キーが存在する場合、その配列を order->authz_urls に複製します。 4. JSON に challenge_setups キーが存在する場合、その配列を order->challenge_setups に複製します。 5. JSON に finalize キーが存在する場合、その URL を order->finalize に複製します。 6. JSON に certificate キーが存在する場合、その URL を order->certificate に複製します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/md/md_acme_order.c:144
  • modules/md/md_acme_order.c:325
  • 呼び出し先:
  • order_st_from_str
  • md_json_has_key, md_json_dups, md_json_gets, md_json_dupsa (いずれも md_json.h 経由)

関連データ構造

  • md_acme_order_t: 更新対象のメイン構造体。
  • md_json_t: ACME サーバーとの通信用 JSON コンテナ。

失敗モードと境界条件

  • JSON 内の各キーが存在しない場合、対応する order フィールドは更新されません。
  • メモリ不足時は apr_pool_t の挙動に従い、通常はアボートまたは例外が発生します。

保守担当者向けメモ

  • 本関数は、サーバーからのレスポンスをローカルの md_acme_order_t に同期させるための主要な変換ロジックです。
  • md_json_dups および md_json_dupsa を使用しているため、更新される文字列や配列は指定されたプール p の生存期間に依存します。
