sig_restart

定義

  • 種別: function
  • 定義箇所: os/unix/unixd.c:480

目的

Apache HTTP Server の MPM における再起動シグナルを処理し、サーバーの状態遷移をトリガーする。

シグネチャまたは形状

static void sig_restart(int sig)

入力と出力

  • **入力**: int sig（受信したシグナル番号）
  • **出力**: なし（void）

前提条件と不変条件

  • retained_data が初期化されていること。初期化されていない場合、関数は即座に終了する。
  • retained_data はサーバーのライフサイクル全体で生存する共有メモリ領域である。

アルゴリズムまたは振る舞い

1. retained_data が存在しない場合、処理を中断する。 2. retained_data->mpm_state を AP_MPMQ_STOPPING に設定する。 3. すでに再起動が保留されており、かつ現在のシグナルが AP_SIG_GRACEFUL であるか、すでに非優雅（ungraceful）な再起動が要求されている場合は、重複処理を避けて終了する。 4. retained_data->restart_pending を 1 に設定する。 5. 受信したシグナルが AP_SIG_GRACEFUL でない場合、retained_data->is_ungraceful を 1 に設定し、強制的な再起動をマークする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • os/unix/unixd.c:586 (シグナルハンドラ設定)
  • os/unix/unixd.c:615 (apr_signal による SIGHUP 登録)
  • os/unix/unixd.c:618 (apr_signal による AP_SIG_GRACEFUL 登録)
  • server/mpm/mpmt_os2/mpmt_os2.c:391 (シグナルハンドラ設定)
  • **呼び出し先**:
  • 直接的な関数呼び出しはなし。retained_data 構造体のメンバ更新を行う。

関連データ構造

  • ap_unixd_mpm_retained_data (グローバル変数 retained_data): MPM の再起動状態を保持する構造体。

失敗モードと境界条件

  • retained_data が NULL の場合: メインプロセス終了時など、状態保持が不要なタイミングで呼び出された場合は安全に無視される。
  • 重複シグナル: すでに保留中の再起動がある場合、状態を上書きせずに終了する。

保守担当者向けメモ

  • 本関数はシグナルハンドラとして動作するため、非同期シグナル安全な操作のみを行う必要がある。
  • retained_data の更新は、メインループ側で MPM の再起動ロジックを制御するためのフラグとして機能する。
