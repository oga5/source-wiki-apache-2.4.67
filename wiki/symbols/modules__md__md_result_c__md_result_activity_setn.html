md_result_activity_setn

定義

  • 種別: function
  • 定義箇所: modules/md/md_result.c:74

目的

md_result_t オブジェクトの現在のアクティビティ文字列を更新し、関連するエラー情報やサブ問題をリセットした上で、状態変更の通知を行います。

シグネチャまたは形状

void md_result_activity_setn(md_result_t *result, const char *activity)

入力と出力

  • **入力**:
  • result: 更新対象の md_result_t 構造体へのポインタ。
  • activity: 現在のアクティビティを示す文字列（NULL も許容）。
  • **出力**: なし（void）。

前提条件と不変条件

  • result は有効な md_result_t オブジェクトを指している必要があります。
  • 本関数を呼び出すと、既存の problem、detail、および subproblems フィールドは強制的に NULL にクリアされます。

アルゴリズムまたは振る舞い

1. result->activity に引数 activity を代入します。 2. result->problem、result->detail、result->subproblems をすべて NULL に設定します。 3. 内部関数 on_change(result) を呼び出し、登録されている状態変更コールバックを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_drive.c (複数箇所)
  • modules/md/md_acme_order.c (複数箇所)
  • modules/md/md_acmev2_drive.c
  • modules/md/md_reg.c
  • **呼び出し先**:
  • on_change (内部関数)

関連データ構造

  • md_result_t: 処理結果を保持するメイン構造体。

失敗モードと境界条件

  • result が NULL の場合、ポインタ参照によるクラッシュが発生する可能性があります。
  • activity に NULL を渡すと、アクティビティが未設定の状態となります。

保守担当者向けメモ

  • 本関数は、エラー状態から新しいアクティビティへ遷移する際に、古いエラー情報を確実にクリアするために使用されます。
  • on_change コールバックが登録されている場合、この関数を介して外部コンポーネントへ状態変化が通知されます。
