ap_queue_info_num_idlers

定義

  • 種別: function
  • 定義箇所: server/mpm_fdqueue.c:210

目的

現在利用可能な（アイドル状態の）ワーカー・スレッド数を取得します。

シグネチャまたは形状

apr_uint32_t ap_queue_info_num_idlers(fd_queue_info_t *queue_info)

入力と出力

  • **入力**: fd_queue_info_t *queue_info - アイドル状態管理構造体へのポインタ。
  • **出力**: apr_uint32_t - 現在のアイドル・スレッド数。

前提条件と不変条件

  • queue_info は有効なポインタである必要があります。
  • 内部カウンタ idlers は、zero_pt を基準としたオフセット値として管理されています。
  • idlers の値が zero_pt 以下の場合は、待機中のスレッドが存在するか、アイドル数が 0 であるとみなされます。

アルゴリズムまたは振る舞い

1. apr_atomic_read32 を使用して、queue_info->idlers の値をアトミックに読み取ります。 2. 読み取った値が zero_pt より大きい場合、val - zero_pt を計算して返します。 3. val が zero_pt 以下の場合は、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c (行 504, 521, 1967) にて呼び出しを確認。
  • **呼び出し先**:
  • apr_atomic_read32 (APR ライブラリ関数)。

関連データ構造

  • fd_queue_info_t: アイドル状態管理構造体。
  • zero_pt: idlers カウンタの基準値となるマジックナンバー。

失敗モードと境界条件

  • queue_info が NULL の場合、未定義の動作を引き起こす可能性があります。
  • アトミック操作により、読み取り時点でのスナップショットを返しますが、呼び出し直後に値が変化する可能性があります。

保守担当者向けメモ

  • 本関数はロックフリーなアトミック操作を利用しており、Mutex を介さずに高速にアイドル数を判定する設計です。
  • idlers カウンタの設計上、負の値は「待機中のスレッド数」を表現しているため、戻り値は常に 0 以上となります。
