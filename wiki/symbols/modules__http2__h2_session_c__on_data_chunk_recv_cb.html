on_data_chunk_recv_cb

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:234

目的

nghttp2 ライブラリから通知された HTTP/2 DATA フレームのチャンクを、該当するストリームへ転送・処理する。

シグネチャまたは形状

static int on_data_chunk_recv_cb(nghttp2_session *ngh2, uint8_t flags,
                                 int32_t stream_id,
                                 const uint8_t *data, size_t len, void *userp)

入力と出力

  • **入力**:
  • ngh2: nghttp2 セッションハンドル。
  • flags: DATA フレームのフラグ。
  • stream_id: 受信したデータのストリーム ID。
  • data: 受信データバッファへのポインタ。
  • len: 受信データの長さ。
  • userp: h2_session 構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返す。
  • ストリームが見つからない場合は NGHTTP2_ERR_CALLBACK_FAILURE を返す。

前提条件と不変条件

  • userp は有効な h2_session 構造体を指している必要がある。
  • get_stream を介して stream_id に対応する h2_stream が取得可能であること。

アルゴリズムまたは振る舞い

1. get_stream を呼び出し、stream_id に対応するストリームを検索する。 2. ストリームが存在する場合:

  • トレースログを出力する。
  • h2_stream_recv_DATA を呼び出し、データをストリームへ渡す。

    • 3. ストリームが存在しない場合:

  • デバッグログを出力し、NGHTTP2_ERR_CALLBACK_FAILURE を返す。

    • 4. h2_stream_recv_DATA の処理結果が APR_SUCCESS 以外の場合:

  • nghttp2_session_consume を呼び出し、当該データを消費済みとして処理する。

呼び出し元と呼び出し先

  • **呼び出し元**: nghttp2 ライブラリ（コールバックとして登録）。
  • **呼び出し先**:
  • get_stream
  • ap_log_cerror
  • h2_stream_recv_DATA
  • nghttp2_session_consume

関連データ構造

  • h2_session: セッション状態を保持。
  • h2_stream: ストリームごとの状態とデータを管理。
  • nghttp2_session: nghttp2 ライブラリのセッションコンテキスト。

失敗モードと境界条件

  • **ストリーム不在**: stream_id がアクティブなストリームと一致しない場合、エラーを返し処理を中断する。
  • **データ処理失敗**: h2_stream_recv_DATA がエラーを返した場合、データが適切に消費されないことを防ぐため、nghttp2_session_consume を明示的に呼び出す。

保守担当者向けメモ

  • 本関数は nghttp2 のコールバックとして h2_session の初期化時に登録される。
  • h2_stream_recv_DATA の戻り値が APR_SUCCESS でない場合、データが破棄される可能性があるため、消費処理の整合性に注意すること。
