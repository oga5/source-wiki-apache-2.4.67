ssl_conn_is_ssl

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:68

目的

指定された接続（conn_rec）が SSL/TLS を使用しているかどうかを判定し、Apache のフックシステムに適したステータスコードを返します。

シグネチャまたは形状

static int ssl_conn_is_ssl(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（判定対象の接続構造体）
  • **出力**: int 型のステータスコード
  • OK: 接続が SSL/TLS を使用している場合。
  • DECLINED: 接続が SSL/TLS を使用していない、または設定が存在しない場合。

前提条件と不変条件

  • ssl_get_effective_config(c) が呼び出され、接続に関連付けられた SSL 設定が取得可能であること。
  • 戻り値は Apache のフック処理規約に従い、OK または DECLINED のいずれかとなります。

アルゴリズムまたは振る舞い

1. ssl_get_effective_config(c) を呼び出し、接続に関連付けられた SSLConnRec 構造体を取得します。 2. 取得した sslconn が存在し、かつその内部の ssl メンバ（OpenSSL の SSL オブジェクト）が有効であるかを確認します。 3. 両方の条件を満たす場合は OK を返し、それ以外の場合は DECLINED を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • ssl_get_effective_config（modules/ssl/ssl_engine_vars.c 内）
  • **呼び出し元**:
  • modules/ssl/ssl_engine_vars.c:140 にて ap_hook_ssl_conn_is_ssl を通じて登録されています。
  • server/ssl.c:261 にて AP_IMPLEMENT_HOOK_RUN_FIRST マクロにより、ssl_conn_is_ssl フックの実装として呼び出されます。

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • SSLConnRec: mod_ssl が接続ごとに保持する内部設定構造体。

失敗モードと境界条件

  • c が NULL の場合、あるいは ssl_get_effective_config が NULL を返す場合、安全に DECLINED を返します。
  • sslconn は存在するが ssl メンバが初期化されていない（NULL である）場合、SSL 接続ではないと判断され DECLINED を返します。

保守担当者向けメモ

  • 本関数は AP_DECLARE_HOOK として定義されたフック関数です。このフックを介して他のモジュールから SSL 接続かどうかの問い合わせが行われます。
  • ssl_get_effective_config の実装変更が本関数の判定ロジックに直接影響するため、設定取得ロジックを修正する際は注意してください。
