modssl_dispatch_ocsp_request

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ocsp.c:306

目的

OCSP レスポンダとの通信を仲介するメインエントリポイントです。指定された URI に対して OCSP リクエストを送信し、レスポンスを取得して OCSP_RESPONSE オブジェクトとして返却します。

シグネチャまたは形状

OCSP_RESPONSE *modssl_dispatch_ocsp_request(const apr_uri_t *uri,
                                            apr_interval_time_t timeout,
                                            OCSP_REQUEST *request,
                                            conn_rec *c, apr_pool_t *p);

入力と出力

  • **入力**:
  • uri: 接続先 OCSP レスポンダの URI。
  • timeout: 通信のタイムアウト時間。
  • request: 送信対象の OCSP_REQUEST オブジェクト。
  • c: 現在の接続情報 (conn_rec)。
  • p: メモリ割り当てに使用するプール。
  • **出力**:
  • 成功時: 受信した OCSP_RESPONSE オブジェクトへのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • conn_rec からサーバー設定を取得し、プロキシ設定 (proxy_uri) を参照します。
  • 内部で使用する BIO オブジェクトは、リクエストのシリアライズとレスポンスの読み込みの両方で再利用されます。

アルゴリズムまたは振る舞い

1. serialize_request を呼び出し、OCSP_REQUEST を HTTP POST リクエスト形式に変換して BIO に書き出します。失敗した場合はエラーログを出力して終了します。 2. send_request を呼び出し、ネットワークソケットを確立してリクエストを送信します。失敗した場合は BIO を解放して終了します。 3. BIO_reset を実行し、レスポンス読み込みのために BIO の内容をクリアします。 4. read_response を呼び出し、ソケットから HTTP レスポンスを読み取って OCSP_RESPONSE オブジェクトを生成します。 5. ソケットを閉じ、BIO を解放して、取得したレスポンスを返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_ocsp.c:157
  • modules/ssl/ssl_util_stapling.c:549
  • **呼び出し先**:
  • serialize_request
  • send_request
  • read_response
  • BIO_reset, BIO_free, apr_socket_close (OpenSSL/APR API)

関連データ構造

  • OCSP_RESPONSE, OCSP_REQUEST: OpenSSL の OCSP オブジェクト。
  • apr_uri_t: URI 解析用構造体。
  • conn_rec: Apache の接続状態管理構造体。

失敗モードと境界条件

  • リクエストのシリアライズ失敗時: エラーログを出力し NULL を返します。
  • ソケット通信失敗時: send_request 内でログが出力され、本関数は NULL を返します。
  • レスポンス読み込み失敗時: read_response の結果に従い NULL を返します。

保守担当者向けメモ

  • 本関数は簡易的な HTTP/1.0 クライアント実装の一部です。将来的に標準的な HTTP クライアントライブラリへ移行する際は、この通信フロー全体が置き換え対象となります。
  • BIO の管理（生成、リセット、解放）が本関数の責務であるため、修正時にはメモリリークに注意してください。
