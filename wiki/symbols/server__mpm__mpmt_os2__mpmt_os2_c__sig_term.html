sig_term

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:358

目的

Apache HTTP Server のシャットダウン要求を処理し、プロセス終了フラグを立てるためのシグナルハンドラです。

シグネチャまたは形状

static void sig_term(int sig)

入力と出力

  • 入力: int sig (受信したシグナル番号)
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • グローバル変数 shutdown_pending が定義されていること。
  • 本関数が SIGTERM シグナルを受信した際に呼び出されること。

アルゴリズムまたは振る舞い

1. グローバル変数 shutdown_pending に 1 を代入し、シャットダウンが保留中であることを記録します。 2. signal(SIGTERM, SIG_DFL) を呼び出し、SIGTERM シグナルのハンドラをデフォルトの動作（プロセス終了）にリセットします。

呼び出し元と呼び出し先

  • 呼び出し元: OS/2 環境下で SIGTERM シグナルが発生した際に、システムまたは set_signals 関数等によって間接的に呼び出されます。
  • 呼び出し先: signal (標準ライブラリ関数)。
  • 注記: ツールによる解析では、os/unix/unixd.c や server/mpm/netware/mpm_netware.c など他プラットフォームのファイルからも本シンボルへの参照が見られますが、これらはプラットフォーム固有の MPM 実装における同名関数の可能性があり、本ファイル（mpmt_os2.c）との直接的な呼び出し関係はソースコード上では明示されていません。

関連データ構造

  • shutdown_pending (グローバル変数)

失敗モードと境界条件

  • 本関数はシグナルハンドラとして実行されるため、非同期信号安全（async-signal-safe）でない操作は避ける必要があります。
  • SIGTERM が連続して送信された場合、2回目以降はデフォルトの動作（プロセス終了）が即座に実行されます。

保守担当者向けメモ

  • 本関数は OS/2 MPM におけるシャットダウンのトリガーです。終了処理のロジックを変更する場合は、shutdown_pending を参照しているメインループ側の処理と整合性を確認してください。
