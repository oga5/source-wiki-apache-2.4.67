ajp_send_header

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_header.c:688

目的

Apache のリクエスト情報を AJP プロトコル形式のメッセージに変換し、バックエンドサーバーへ送信する。

シグネチャまたは形状

apr_status_t ajp_send_header(apr_socket_t *sock,
                             request_rec *r,
                             apr_size_t buffsize,
                             apr_uri_t *uri,
                             const char *secret)

入力と出力

  • **入力**:
  • sock: バックエンドへの接続ソケット。
  • r: Apache のリクエスト構造体。
  • buffsize: メッセージバッファのサイズ。
  • uri: リクエスト URI 情報。
  • secret: AJP 認証用シークレット文字列。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • r->pool を使用してメッセージバッファが割り当てられるため、リクエストのライフサイクルに依存する。
  • 内部で ajp_msg_t 構造体を生成・操作する。

アルゴリズムまたは振る舞い

1. ajp_msg_create を呼び出し、指定されたサイズでメッセージバッファを確保する。 2. ajp_marshal_into_msgb を呼び出し、リクエスト情報（ヘッダー、環境変数、SSL 情報等）を AJP メッセージ形式に直列化する。 3. ajp_ilink_send を呼び出し、構築されたメッセージをソケット経由で送信する。 4. 送信後に ajp_msg_log を用いてパケットのダンプを記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ajp.c (プロキシ処理のメインループ内)。
  • **呼び出し先**:
  • ajp_msg_create
  • ajp_marshal_into_msgb
  • ajp_ilink_send
  • ajp_msg_log
  • ap_log_rerror (エラーログ出力)

関連データ構造

  • ajp_msg_t: AJP 通信用のメッセージバッファ構造体。
  • request_rec: Apache のリクエスト情報。
  • apr_socket_t: ネットワークソケット。

失敗モードと境界条件

  • ajp_msg_create が失敗した場合、エラーログを出力してそのステータスを返す。
  • ajp_marshal_into_msgb が失敗した場合、エラーログを出力してそのステータスを返す。
  • ajp_ilink_send が失敗した場合、エラーログを出力してそのステータスを返す。

保守担当者向けメモ

  • 本関数は AJP 送信フローの起点であり、メッセージの構築から送信までをカプセル化している。
  • エラー発生時は APLOGNO を含む ap_log_rerror を使用して詳細な診断情報が記録される。
