md_reg_t

定義

  • **種別**: struct
  • **定義箇所**: modules/md/md_reg.c:42

目的

管理対象ドメイン（Managed Domains: MD）のレジストリを保持するメイン構造体です。メモリプール、永続化ストア、プロトコルハンドラ、証明書キャッシュを統合し、ドメイン設定のライフサイクル管理とビジネスロジックの適用を担います。

シグネチャまたは形状

struct md_reg_t {
    apr_pool_t *p;
    struct md_store_t *store;
    struct apr_hash_t *protos;
    struct apr_hash_t *certs;
    int can_http;
    int can_https;
    const char *proxy_url;
    const char *ca_file;
    int domains_frozen;
    md_timeslice_t *renew_window;
    md_timeslice_t *warn_window;
    md_job_notify_cb *notify;
    void *notify_ctx;
    apr_time_t min_delay;
    int retry_failover;
    int use_store_locks;
    apr_time_t lock_wait_timeout;
};

入力と出力

本構造体は md_reg_create() によって初期化され、レジストリ操作のコンテキストとして各関数（md_reg_find、md_reg_get 等）に渡されます。出力として、管理下のドメイン設定（md_t）や証明書キャッシュへのアクセスを提供します。

前提条件と不変条件

  • **メモリ管理**: 生成時に渡された apr_pool_t を所有し、キャッシュや設定の生存期間を管理します。
  • **凍結状態**: domains_frozen フラグが非ゼロの場合、設定の変更や削除は禁止されます。これはサーバー起動後の安定性を維持するための保護機構です。

アルゴリズムまたは振る舞い

  • **キャッシュ管理**: certs ハッシュを用いて公開証明書をキャッシュし、検索効率を向上させます。
  • **同期**: use_store_locks が有効な場合、md_store を介したファイル操作時にロック機構を利用して整合性を保ちます。
  • **通知**: notify コールバックを通じて、ジョブの状態変化を外部へ伝達します。

呼び出し元と呼び出し先

  • **呼び出し元**: md_reg_create、md_reg_store_get、check_values、state_init、md_reg_do、md_reg_get、md_reg_find 等の modules/md/md_reg.c 内の関数群。
  • **呼び出し先**: 構造体内部で md_store_t（永続化層）や apr_hash_t（APRデータ構造）を参照します。

関連データ構造

  • md_store_t: 永続化層へのアクセスを提供。
  • md_t: 管理対象ドメインの設定と状態を保持。
  • apr_hash_t: プロトコルハンドラおよび証明書キャッシュの保持に使用。
  • md_timeslice_t: 更新および警告ウィンドウの定義に使用。

失敗モードと境界条件

  • domains_frozen が設定されている環境下での変更操作は拒否されます。
  • 証明書キャッシュの頻繁な更新はメモリ消費量に影響を与えるため、注意が必要です。

保守担当者向けメモ

  • 本構造体は「管理」に特化しており、ACMEプロトコルの詳細な通信ロジックは含まれません。
  • md_reg_get_pubcert 等のキャッシュアクセス関数は、内部で state_init を呼び出し、常に最新の証明書状態を反映させる設計になっています。
