lua_ivm_object

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_request.h:50

目的

プロセス間共有メモリや Lua VM 間でのデータ受け渡しを目的とした構造体です。

シグネチャまたは形状

typedef struct {
    int type;
    size_t size;
    size_t vb_size;
    lua_Number number;
    struct ap_varbuf vb;
} lua_ivm_object;

入力と出力

本構造体はデータ保持用であり、直接的な関数入出力は持ちません。lua_request.c 内で apr_pcalloc を用いてメモリ確保が行われます。

前提条件と不変条件

  • struct ap_varbuf を内包しており、メモリ管理には apr_pool_t のライフサイクルが関与します。
  • Lua VM 間でのデータ受け渡しに使用されるため、メモリの整合性には注意が必要です。

アルゴリズムまたは振る舞い

  • Lua スクリプトと Apache 内部データ間での数値や可変長バッファデータの保持・転送を仲介します。
  • lua_request.c において、ポインタの初期化およびメモリ確保の対象として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/lua/lua_request.c 内の関数（1992行目、2019行目、2029行目付近で参照およびメモリ確保を確認）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • struct ap_varbuf: 可変長バッファを管理するために内包されています。
  • lua_Number: Lua の数値型を保持するために使用されます。

失敗モードと境界条件

  • apr_pcalloc によるメモリ確保失敗時の挙動は、呼び出し側のメモリ管理ポリシーに依存します。
  • 共有メモリとして利用する場合、マルチスレッド環境下での競合や整合性については、本構造体単体では制御されません。

保守担当者向けメモ

  • 本構造体は通常のスクリプト実行フローとは異なり、VM 間共有に関連するため、変更時にはメモリのライフサイクルとスレッドセーフティを十分に考慮してください。
  • util_varbuf.h に依存しているため、バッファ操作の仕様変更時には影響範囲を確認してください。
