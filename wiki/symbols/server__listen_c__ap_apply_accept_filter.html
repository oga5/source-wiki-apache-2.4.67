ap_apply_accept_filter

定義

  • 種別: function
  • 定義箇所: server/listen.c:242

目的

リスナーソケットに対して、OS 固有のアクセプトフィルタ（AcceptFilter）または TCP_DEFER_ACCEPT オプションを適用し、接続待機時の挙動を最適化します。

シグネチャまたは形状

static void ap_apply_accept_filter(apr_pool_t *p, ap_listen_rec *lis, server_rec *server)

入力と出力

  • p: メモリ割り当てに使用する APR プール。
  • lis: フィルタを適用する対象のリスナー構造体 (ap_listen_rec)。
  • server: 設定情報を取得するためのサーバーレコード。
  • 出力: なし（副作用としてソケットオプションを適用）。

前提条件と不変条件

  • lis->sd に有効な apr_socket_t が格納されていること。
  • find_accf_name がプロトコル名に基づいたフィルタ名を解決できること。

アルゴリズムまたは振る舞い

1. リスナーのプロトコル名を取得します（未設定の場合は ap_get_server_protocol を使用）。 2. find_accf_name を呼び出し、該当プロトコルに対応するアクセプトフィルタ名を取得します。 3. フィルタ名が見つかった場合：

  • APR_HAS_SO_ACCEPTFILTER が定義されている環境では、apr_socket_accept_filter を使用して OS レベルのアクセプトフィルタを有効化します。
  • それ以外の環境では、apr_socket_opt_set を使用して APR_TCP_DEFER_ACCEPT を設定します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/listen.c:784
  • server/listen.c:790
  • server/listen.c:885
  • 呼び出し先:
  • ap_get_server_protocol
  • find_accf_name
  • apr_socket_accept_filter (条件付き)
  • apr_socket_opt_set (条件付き)
  • ap_log_perror

関連データ構造

  • ap_listen_rec: リスナーのソケット情報 (sd) やプロトコル名を保持。
  • server_rec: サーバー設定を保持。
  • apr_socket_t: APR のソケット抽象化オブジェクト。

失敗モードと境界条件

  • フィルタ適用が OS によってサポートされていない場合 (APR_STATUS_IS_ENOTIMPL) は、エラーとして扱わず静かに終了します。
  • フィルタ名が解決できない場合、何も適用されずに終了します。

保守担当者向けメモ

  • APR_HAS_SO_ACCEPTFILTER の有無により、使用する API が分岐します。
  • APR 1.x 系との互換性維持のため、apr_socket_accept_filter 呼び出し時に apr_pstrdup を用いて文字列のコピーを作成しています。
