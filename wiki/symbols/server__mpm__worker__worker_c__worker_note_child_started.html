worker_note_child_started

定義

  • 種別: function
  • 定義箇所: server/mpm/worker/worker.c:404

目的

子プロセスの起動をスコアボードに記録し、関連するフック関数を呼び出して状態更新を通知する。

シグネチャまたは形状

static void worker_note_child_started(int slot, pid_t pid)

入力と出力

  • **入力**:
  • slot: スコアボード内のプロセススロット番号。
  • pid: 起動した子プロセスのプロセスID。
  • **出力**: なし（副作用としてスコアボードおよびフック経由の状態更新を行う）。

前提条件と不変条件

  • retained グローバル構造体が初期化されていること。
  • ap_scoreboard_image が有効であり、指定された slot がアクセス可能であること。

アルゴリズムまたは振る舞い

1. retained->mpm->my_generation から現在の世代番号を取得する。 2. スコアボードの parent[slot] エントリに、引数で受け取った pid を設定する。 3. 同エントリに現在の世代番号を記録する。 4. ap_run_child_status フックを呼び出し、MPM_CHILD_STARTED ステータスを通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/worker/worker.c:1320
  • server/mpm/worker/worker.c:1380
  • **呼び出し先**:
  • ap_run_child_status (フック関数)

関連データ構造

  • retained: MPM の再起動を跨いで保持されるデータ構造。
  • ap_scoreboard_image: プロセスおよびスレッドの状態を管理する共有メモリ上のスコアボード。

失敗モードと境界条件

  • slot がスコアボードの範囲外である場合、メモリ破壊や不正アクセスが発生する可能性がある（本関数内での境界チェックは行われていない）。

保守担当者向けメモ

  • 本関数は子プロセスのライフサイクル管理において、起動直後の状態同期を担う。
  • ap_run_child_status フックを介して外部モジュールへ通知を行うため、このフックに登録された関数の実行時間やエラーハンドリングが全体の起動パフォーマンスに影響を与える可能性がある。
