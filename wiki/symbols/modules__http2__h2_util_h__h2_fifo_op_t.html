h2_fifo_op_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_util.h:228

目的

FIFO キューの先頭要素に対する操作結果を定義し、キューの検査関数（peek）が次にどのようなアクションを取るべきかを指示するために使用されます。

シグネチャまたは形状

typedef enum {
    H2_FIFO_OP_PULL,
    H2_FIFO_OP_REPUSH,
} h2_fifo_op_t;

入力と出力

  • **入力**: なし（列挙型定義）
  • **出力**: H2_FIFO_OP_PULL または H2_FIFO_OP_REPUSH のいずれかの値。

前提条件と不変条件

  • 本列挙型は、h2_fifo_peek_fn および h2_ififo_peek_fn の戻り値として使用されることを前提としています。

アルゴリズムまたは振る舞い

  • H2_FIFO_OP_PULL: キューから要素を取り出し、破棄（または消費）することを指示します。
  • H2_FIFO_OP_REPUSH: キューから要素を取り出した後、直ちにキューの末尾へ再挿入することを指示します。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_fifo_peek_fn および h2_ififo_peek_fn を実装するコールバック関数。
  • **呼び出し先**: 本シンボル自体は型定義であり、関数ではないため呼び出し先はありません。

関連データ構造

  • h2_fifo: スレッドセーフな FIFO キュー。
  • h2_ififo: スレッドセーフな整数型 FIFO キュー。
  • h2_fifo_peek_fn: 本型を戻り値とする汎用キューの検査関数型。
  • h2_ififo_peek_fn: 本型を戻り値とする整数キューの検査関数型。

失敗モードと境界条件

  • 列挙型であるため、定義された値以外が返された場合の挙動は、これを利用するキュー操作側の実装に依存します。

保守担当者向けメモ

  • 本シンボルは h2_util.h 内のキュー操作ロジックにおいて、要素を「取り出すか」「保持し続けるか」を制御する重要なインターフェースです。新しい操作モードを追加する場合は、関連するすべての peek 関数実装を更新する必要があります。
