ap_expr_node

定義

  • 種別: struct
  • 定義箇所: server/util_expr_private.h:76

目的

Apache HTTP Server の式評価エンジンにおいて、抽象構文木（AST）を構成する基本ノードを定義します。

シグネチャまたは形状

struct ap_expr_node {
    ap_expr_node_op_e node_op;
    const void *node_arg1;
    const void *node_arg2;
};

入力と出力

  • **入力**: 構文解析時に ap_expr_make 関数を介して、演算子型と引数が設定されます。
  • **出力**: 構築されたノードは AST の一部として保持され、評価フェーズで node_op に基づいて node_arg1 および node_arg2 が参照されます。

前提条件と不変条件

  • node_op は ap_expr_node_op_e で定義された有効な演算子である必要があります。
  • node_arg1 および node_arg2 は、node_op の種類に応じて適切なデータ型（別の ap_expr_node へのポインタや定数文字列など）を指す必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、式を木構造として表現するためのコンテナです。
  • 構文解析器（util_expr_parse.c）がトークンを読み取る際、この構造体を生成して AST を構築します。
  • 評価器（util_expr_eval.c）は、このノードを再帰的に辿り、node_op に応じた演算を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: include/ap_expr.h にて ap_expr_t として型定義（typedef）されており、式評価エンジン全体で利用されます。
  • **呼び出し先**: 構造体内部で直接関数を呼び出すことはありませんが、node_op の値が評価ロジックの分岐先を決定します。

関連データ構造

  • ap_expr_node_op_e: ノードの演算子型を定義する列挙型。
  • ap_expr_parse_ctx_t: 解析中にノードを生成・管理するためのコンテキスト構造体。
  • ap_expr_t: ap_expr_node へのエイリアス。

失敗モードと境界条件

  • 構造体自体にはメモリ管理機能がないため、生成時に割り当てられたメモリプール（ap_expr_parse_ctx_t 内のプール）のライフサイクルに依存します。
  • node_arg が指すデータが、評価時に生存している必要があります。

保守担当者向けメモ

  • 本構造体は内部実装用です。ノードの追加や変更を行う場合は、ap_expr_node_op_e の定義と、util_expr_eval.c 内の評価ロジックを同期させる必要があります。
