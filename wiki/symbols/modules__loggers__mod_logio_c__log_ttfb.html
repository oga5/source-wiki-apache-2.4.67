log_ttfb

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_logio.c:124

目的

リクエストの TTFB (Time To First Byte) を取得し、ログフォーマット用の文字列として返します。

シグネチャまたは形状

static const char *log_ttfb(request_rec *r, char *a)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • a: ログフォーマットの引数（本関数では未使用）。
  • **出力**:
  • 成功時: TTFB の値を文字列として返す。
  • 失敗時: TTFB が計測されていない場合、文字列 "-" を返す。

前提条件と不変条件

  • r->request_config から logio_module の設定が取得可能であること。
  • TTFB の計測値は logio_req_t 構造体の ttfb メンバに格納されていること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、リクエスト設定から logio_req_t 構造体を取得します。 2. 構造体が存在しない、または ttfb 値が設定されていない場合は "-" を返します。 3. 値が存在する場合、apr_psprintf を用いて apr_time_t 型の ttfb 値を文字列に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_logio.c:200 にて log_pfn_register を通じて ^FB トークンとして登録されています。
  • **呼び出し先**: ap_get_module_config (Apache API) および apr_psprintf (APR API) を呼び出します。

関連データ構造

  • logio_req_t: リクエストごとの TTFB 測定値を保持する構造体。
  • request_rec: Apache のリクエスト情報を保持するコア構造体。

失敗モードと境界条件

  • rconf が NULL の場合、または rconf->ttfb が 0 の場合は、計測不能とみなして "-" を返します。

保守担当者向けメモ

  • 本関数は mod_log_config のログフォーマットハンドラとして動作します。
  • apr_time_t のフォーマット指定子として APR_TIME_T_FMT を使用しているため、プラットフォーム間で一貫した出力が保証されます。
