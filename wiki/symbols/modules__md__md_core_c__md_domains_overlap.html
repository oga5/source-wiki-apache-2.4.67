md_domains_overlap

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:61

目的

2つのマネージドドメイン（md_t）オブジェクト間で、ドメイン名が重複しているかどうかを判定します。

シグネチャまたは形状

int md_domains_overlap(const md_t *md1, const md_t *md2)

入力と出力

  • **入力**:
  • md1: 比較対象のマネージドドメイン構造体へのポインタ。
  • md2: 比較対象のマネージドドメイン構造体へのポインタ。
  • **出力**:
  • int: 重複がある場合は非ゼロ（真）、重複がない場合は 0（偽）を返します。

前提条件と不変条件

  • md1 および md2 は有効な md_t オブジェクトを指している必要があります。
  • md_t 内のドメインリストは、md_array_str_compact 等によって正規化されていることが期待されます。

アルゴリズムまたは振る舞い

1. 内部的に md_common_name(md1, md2) を呼び出します。 2. md_common_name が共通のドメイン名を返した場合は 1 を返し、NULL を返した場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: md_common_name
  • **呼び出し元**:
  • modules/md/md.h:241 に宣言が存在します。
  • 具体的な実行時の呼び出し元については、提供されたソース範囲内では特定できません。

関連データ構造

  • md_t: マネージドドメインの情報を保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動については、本ソース範囲内では定義されていません。
  • 共通ドメインが存在しない場合、正しく 0 を返します。

保守担当者向けメモ

  • 本関数は md_common_name の結果をブール値としてラップするユーティリティです。
  • 判定ロジックの詳細は md_common_name の実装に依存します。
