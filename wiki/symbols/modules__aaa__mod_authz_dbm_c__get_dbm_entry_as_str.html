get_dbm_entry_as_str

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbm.c:48

目的

DBM ファイルから指定されたキーに対応する値を検索し、メモリプール上に複製された文字列として取得する低レベルラッパー関数です。

シグネチャまたは形状

char *get_dbm_entry_as_str(apr_pool_t *pool, apr_dbm_t *f, char *key)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • f: 検索対象のオープンされた apr_dbm_t ハンドル。
  • key: 検索に使用する文字列キー。
  • **出力**:
  • 成功時: DBM から取得した値を pool 上に複製した文字列へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • f は apr_dbm_open2 等で既にオープンされている必要があります。
  • NETSCAPE_DBM_COMPAT マクロの定義有無により、キーのサイズ計算（strlen の結果に +1 を含めるか否か）が切り替わります。

アルゴリズムまたは振る舞い

1. key を apr_datum_t 構造体 q にセットします。 2. NETSCAPE_DBM_COMPAT の定義状況に応じて q.dsize を設定します。 3. apr_dbm_fetch を呼び出し、キーに対応するデータを取得します。 4. 取得に成功し、かつデータが存在する場合、apr_pstrmemdup を使用して pool 上に値を複製して返します。 5. 取得に失敗した場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbm.c 内の以下の箇所から呼び出されています。
  • 137行目: val = get_dbm_entry_as_str(r->pool, f, key1)
  • 138行目: val = get_dbm_entry_as_str(r->pool, f, key2)
  • **呼び出し先**:
  • apr_dbm_fetch (APR)
  • apr_pstrmemdup (APR)

関連データ構造

  • apr_dbm_t: DBM ファイルハンドル。
  • apr_datum_t: DBM のキーおよび値を保持するデータ構造。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • apr_dbm_fetch が APR_SUCCESS を返さない場合、またはデータが存在しない場合は NULL を返します。
  • メモリ不足時は apr_pstrmemdup が例外を発生させる可能性があります（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は r->pool を使用してメモリを確保するため、呼び出し側で明示的な解放は不要です。
  • NETSCAPE_DBM_COMPAT の条件分岐は、特定のレガシー DBM 実装との互換性を維持するためのものです。変更時は注意してください。
