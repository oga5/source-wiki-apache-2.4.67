cache

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:386

目的

mod_rewrite における外部マップルックアップの結果を保持し、パフォーマンスを向上させるためのキャッシュ管理構造体です。

シグネチャまたは形状

typedef struct cache {
    apr_pool_t         *pool;
    apr_hash_t         *maps;
#if APR_HAS_THREADS
    apr_thread_mutex_t *lock;
#endif
} cache;

入力と出力

  • **入力**: 外部マップのルックアップ結果を格納するためのメモリプール、ハッシュテーブル、およびスレッドセーフな排他制御用ミューテックス。
  • **出力**: なし（データ構造体）。

前提条件と不変条件

  • pool はキャッシュの生存期間を管理するために使用されます。
  • APR_HAS_THREADS が有効な環境では、lock を用いてハッシュテーブルへのアクセスを排他制御する必要があります。

アルゴリズムまたは振る舞い

  • mod_rewrite の外部マップ（DBM, DBD, PRG 等）の検索結果をハッシュテーブル maps に格納します。
  • リクエストごとのメモリプールとは独立した pool を保持しており、リクエスト終了後もキャッシュが生存することを可能にします。
  • スレッド環境下では、lock を介してマップの読み書き時の整合性を保証します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、具体的な呼び出し元関数は特定できません。
  • **呼び出し先**: apr_pool_t、apr_hash_t、および apr_thread_mutex_t（APR ライブラリ）の各関数を操作します。
  • **注意**: include/ap_socache.h 等に同名の概念が存在しますが、本構造体は mod_rewrite.c 内部で定義された専用のキャッシュ構造体であり、それらとは直接的な関係はありません。

関連データ構造

  • cachedmap: キャッシュされたマップの個別のエントリを保持する構造体。
  • rewritemap_entry: 外部マップの定義情報。

失敗モードと境界条件

  • メモリ不足時、pool の割り当てやハッシュテーブルの初期化が失敗する可能性があります。
  • スレッド環境において、lock の初期化や取得に失敗した場合、キャッシュへのアクセスが安全に行えません。

保守担当者向けメモ

  • 本構造体はリクエストの生存期間を超えてキャッシュを保持するため、メモリリークには特に注意してください。
  • mod_rewrite のキャッシュは r->pool ではなく cachep->pool を使用する設計であるため、ライフサイクル管理が他のリクエスト処理と異なります。
