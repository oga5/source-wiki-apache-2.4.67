ap_set_mutex

定義

  • 種別: function
  • 定義箇所: server/util_mutex.c:164

目的

Apache HTTP Server の Mutex ディレクティブを処理し、指定されたミューテックス型に対するメカニズム、ディレクトリ、およびオプション（OmitPID）を設定します。

シグネチャまたは形状

const char *ap_set_mutex(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されません。
  • arg: Mutex ディレクティブに続く引数文字列（例: "default:flock:/tmp OmitPID"）。
  • **出力**:
  • 成功時: NULL を返します。
  • 失敗時: エラーメッセージ文字列を返します。

前提条件と不変条件

  • GLOBAL_ONLY コンテキストで呼び出される必要があります。
  • 内部ハッシュテーブル mxcfg_by_type が初期化済みである必要があります。
  • 設定対象のミューテックス型が事前に ap_mutex_register() で登録されている必要があります。

アルゴリズムまたは振る舞い

1. **コンテキスト検証**: ap_check_cmd_context を使用し、グローバル設定以外であればエラーを返します。 2. **メカニズム解析**: ap_parse_mutex を呼び出し、指定されたメカニズムとディレクトリパスを抽出します。 3. **引数解析**: arg を走査し、OmitPID オプションの有無を確認しつつ、対象となるミューテックス型のリストを構築します。引数が空の場合は "default" を対象とします。 4. **設定適用**: リスト内の各ミューテックス型について、mxcfg_by_type ハッシュから対応する mutex_cfg_t を取得し、以下の値を更新します。

  • omit_pid: 指定されたオプションを反映。
  • none: none が指定された場合、その型が AP_MUTEX_ALLOW_NONE を許可していれば有効化。
  • mech / dir: メカニズムとディレクトリを更新。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c における AP_INIT_RAW_ARGS("Mutex", ...) 定義を通じて、設定ファイル読み込み時に呼び出されます。
  • **呼び出し先**:
  • ap_check_cmd_context
  • ap_getword_conf
  • ap_parse_mutex
  • apr_array_make / apr_array_push / apr_array_pop
  • apr_hash_get

関連データ構造

  • mutex_cfg_t: ミューテックスの設定を保持する構造体。
  • mxcfg_by_type: ミューテックス型名から mutex_cfg_t を引くための内部ハッシュテーブル。

失敗モードと境界条件

  • **エラー返却**:
  • GLOBAL_ONLY 以外のコンテキストで呼び出された場合。
  • メカニズム引数が指定されていない場合。
  • ap_parse_mutex が APR_ENOTIMPL（無効なメカニズム）または APR_BADARG（無効なディレクトリ）を返した場合。
  • 存在しないミューテックス型が指定された場合。
  • none が許可されていないミューテックス型に対して none が指定された場合。

保守担当者向けメモ

  • 本関数は Mutex ディレクティブの構文解析と設定の反映のみを担当します。実際のミューテックス生成は ap_xxx_mutex_create 呼び出し時に mxcfg_lookup を通じて行われます。
  • OmitPID はリストのどこに記述されても全体に適用される仕様です。
