bucket_socket_ex_read

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:93

目的

SOCKET_EX バケットからデータを読み込み、ソケット通信の統計情報を更新しつつ、読み取ったデータをヒープバケットへ変換してバケットチェーンに反映させる。

シグネチャまたは形状

apr_status_t bucket_socket_ex_read(apr_bucket *a, const char **str,
                                   apr_size_t *len,
                                   apr_read_type_e block)

入力と出力

  • **入力**:
  • a: 読み込み対象の apr_bucket。
  • block: 読み込みモード（APR_NONBLOCK_READ 等）。
  • **出力**:
  • str: 読み取ったデータへのポインタ。
  • len: 読み取ったデータの長さ。
  • 戻り値: apr_status_t（成功時は APR_SUCCESS）。

前提条件と不変条件

  • バケット a のデータ領域 (a->data) は socket_ex_data 構造体であること。
  • data->counter は有効な統計カウンタを指していること。
  • 読み込み成功時、バケット a はヒープバケットに変換され、後続に新しい SOCKET_EX バケットが挿入される。

アルゴリズムまたは振る舞い

1. **タイムアウト制御**: APR_NONBLOCK_READ 指定時は、ソケットのタイムアウトを一時的に 0 に設定し、読み込み後に復元する。 2. **データ読み込み**: apr_socket_recv を使用してソケットからデータを取得する。 3. **統計更新**: 読み込みバイト数を data->counter に加算する。 4. **バケット変換**:

  • 読み込みデータがある場合: apr_bucket_heap_make で現在のバケットをヒープバケットへ変換し、後続に新しい SOCKET_EX バケットを生成・挿入する。
  • 読み込みデータがない場合: apr_bucket_immortal_make で空のバケットに変換する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_scgi.c:147（bucket_type_socket_ex 構造体内の関数ポインタとして参照）。
  • **呼び出し先**: apr_socket_timeout_get, apr_socket_timeout_set, apr_bucket_alloc, apr_socket_recv, apr_bucket_free, apr_bucket_heap_make, APR_BUCKET_INSERT_AFTER, bucket_socket_ex_create, apr_bucket_immortal_make。

関連データ構造

  • socket_ex_data: ソケットと統計カウンタを保持する構造体。
  • apr_bucket: Apache のバケット抽象化構造体。
  • apr_bucket_heap: ヒープ領域にデータを保持するバケット型。

失敗モードと境界条件

  • apr_socket_recv が APR_SUCCESS および APR_EOF 以外のエラーを返した場合、メモリを解放してエラーコードを返す。
  • 読み込みバイト数が 0 の場合は EOF とみなし、空のイモータルバケットを作成して終了する。

保守担当者向けメモ

  • 本関数は SOCKET_EX バケット型の read メソッドとして機能する。
  • 統計情報の更新は data->counter を介して行われるため、このカウンタが正しく初期化されていることを確認すること。
  • APR_BUCKET_BUFF_SIZE を使用してバッファを確保しているため、読み込みサイズの上限はこの定数に依存する。
