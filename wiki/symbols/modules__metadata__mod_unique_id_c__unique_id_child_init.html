unique_id_child_init

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_unique_id.c:170

目的

Apache HTTP Server の子プロセス初期化時に、リクエスト識別子（UNIQUE_ID）生成に必要な乱数シードおよびプロセス固有の初期値を設定する。

シグネチャまたは形状

static void unique_id_child_init(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: APR プール（メモリ管理用）。
  • s: サーバーレコード（サーバー設定情報）。
  • **出力**: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • APR_HAS_THREADS が定義されている場合、ap_mpm_query を通じてマルチスレッド MPM かどうかが判定される。
  • cur_unique_id.root および cur_unique_counter が、プロセス起動ごとに予測困難な値で初期化される必要がある。

アルゴリズムまたは振る舞い

1. **スレッド環境の確認**: APR_HAS_THREADS が有効な場合、ap_mpm_query を呼び出し、現在の MPM がスレッド対応かどうかを is_threaded_mpm に格納する。 2. **乱数シードの生成**: ap_random_insecure_bytes を使用し、cur_unique_id.root にプロセス固有の乱数値を設定する。 3. **カウンタの初期化**: cur_unique_counter にも同様に乱数値を設定する。これにより、再起動時の衝突リスクや、システム時計が逆行した場合の保護を強化する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_unique_id.c:323 にて ap_hook_child_init に登録されている。
  • **呼び出し先**:
  • ap_mpm_query (条件付き)
  • ap_random_insecure_bytes

関連データ構造

  • cur_unique_id: ID生成の元となるグローバルな構造体。
  • cur_unique_counter: ID生成時に使用されるグローバルなカウンタ変数。
  • is_threaded_mpm: スレッド対応状況を保持するグローバル変数。

失敗モードと境界条件

  • ap_random_insecure_bytes が失敗した場合の挙動については、本ソース excerpt 内では定義されていない。
  • ap_mpm_query の結果が is_threaded_mpm に正しく反映されない場合、後続のID生成ロジックでスレッドセーフティに関する想定外の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はプロセス初期化時に一度だけ実行される。
  • カウンタの初期値を乱数で埋めることで、再起動直後のID衝突を抑制している。この設計意図を維持すること。
