register_hooks

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_buffer.c:337

目的

Apache HTTP Server のフィルタチェーンに対し、mod_buffer が提供する出力フィルタおよび入力フィルタを登録する。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（モジュール初期化用のプール）
  • **出力**: なし（戻り値 void）

前提条件と不変条件

  • 本関数は Apache のモジュールロードプロセスにおいて、フック登録フェーズで呼び出されることを前提とする。
  • bufferFilterName が定義済みである必要がある。

アルゴリズムまたは振る舞い

1. ap_register_output_filter を呼び出し、buffer_out_filter を AP_FTYPE_CONTENT_SET 階層の出力フィルタとして登録する。 2. ap_register_input_filter を呼び出し、buffer_in_filter を AP_FTYPE_CONTENT_SET 階層の入力フィルタとして登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール読み込み機構（直接的な呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • ap_register_output_filter
  • ap_register_input_filter
  • buffer_out_filter (関数ポインタとして渡す)
  • buffer_in_filter (関数ポインタとして渡す)

関連データ構造

  • apr_pool_t: リソース管理用プール。
  • AP_FTYPE_CONTENT_SET: フィルタの挿入位置を決定するフィルタタイプ定数。

失敗モードと境界条件

  • フィルタ登録関数（ap_register_*_filter）が内部でメモリ不足等のエラーを発生させた場合、Apache の起動プロセスが中断される可能性がある。

保守担当者向けメモ

  • 本関数は mod_buffer のエントリポイントの一部であり、フィルタの優先順位や階層を変更する場合は AP_FTYPE_CONTENT_SET の値を検討すること。
  • ツールが提示した他の register_hooks の参照先は、本モジュールとは異なる各モジュール内の同名関数であり、本シンボルへの直接的な呼び出し元ではない。
