目的

Apache HTTP Server の式評価エンジン（ap_expr）における字句解析器（スキャナ）の状態を保持するための不透明なポインタ型です。

入力と出力

  • **入力**: スキャナの初期化関数（ap_expr_yylex_init）によって生成され、以降の字句解析関連関数に引数として渡されます。
  • **出力**: スキャナの内部状態（yyguts_t 構造体へのポインタ）をカプセル化し、呼び出し元に対して不透明なハンドルとして提供します。

前提条件と不変条件

  • 再入可能なスキャナのインスタンスとして機能します。
  • 呼び出し元は、このポインタを介してスキャナのライフサイクル（初期化、バッファ操作、破棄）を管理する必要があります。
  • グローバル変数に依存しないため、マルチスレッド環境下での並列解析が可能です。

アルゴリズムまたは振る舞い

Flex によって生成された再入可能スキャナの内部状態を保持します。ap_expr_yylex などの関数に渡されることで、特定の解析コンテキスト（yyextra を含む）を識別し、入力ストリームの読み込みやトークン化の状態を維持します。
