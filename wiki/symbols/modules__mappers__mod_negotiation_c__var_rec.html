var_rec

定義

  • **種別**: struct
  • **定義箇所**: modules/mappers/mod_negotiation.c:202

目的

HTTP コンテンツネゴシエーションにおいて、サーバー側で利用可能な各バリアント（リソースの代替表現）の属性を保持するために使用されます。

シグネチャまたは形状

typedef struct var_rec {
    request_rec *sub_req;
    const char *mime_type;
    const char *file_name;
    apr_off_t body;
    const char *content_encoding;
    apr_array_header_t *content_languages;
    const char *content_charset;
    const char *description;
    float lang_quality;
    float encoding_quality;
    float charset_quality;
    float mime_type_quality;
    float source_quality;
    float level;
    apr_off_t bytes;
    int lang_index;
    int is_pseudo_html;
    float level_matched;
    int mime_stars;
    int definite;
} var_rec;

入力と出力

  • **入力**: バリアントのメタデータ（MIMEタイプ、言語、エンコーディング等）および品質値（qs パラメータ等）。
  • **出力**: ネゴシエーション過程で算出された品質スコアおよびマッチング状態。

前提条件と不変条件

  • mime_type は常に小文字で格納される必要があります。
  • source_quality は set_mime_fields() を通じて、バリアントの記述または MIME タイプから設定されます。
  • lang_quality 等の品質値は、best_match() 内で各バリアントの評価時に設定されます。

アルゴリズムまたは振る舞い

  • 構造体内の大半のフィールドは「一度書き込み」のプロパティですが、level_matched、mime_stars、definite の3フィールドはネゴシエーションの進行に伴い更新されます。
  • sub_req は type-map ファイルを使用する場合など、NULL になる可能性があります。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • clean_var_rec() (初期化/クリーンアップ)
  • set_mime_fields() (フィールド設定)
  • variantsortf() (ソート処理)
  • mime_match() (マッチング判定)
  • level_cmp() (レベル比較)
  • apr_array_make() を用いた negotiation_state 内の配列確保時
  • memcpy() を用いた構造体のコピー操作時
  • **呼び出し先**: 構造体定義のため、直接的な関数呼び出し先はありません。

関連データ構造

  • request_rec: バリアントに関連付けられたサブリクエスト。
  • accept_rec: クライアント側の Accept ヘッダー情報。
  • negotiation_state: バリアントのリストを保持するコンテキスト。

失敗モードと境界条件

  • sub_req が NULL の場合、ファイル名やボディ情報に依存する処理で適切なハンドリングが必要です。
  • type-map ファイルの解析時、ファイルポインタのシークを伴う複雑なエラーパスが存在します。

保守担当者向けメモ

  • 本構造体は r->pool から割り当てられるメモリ管理下にあります。
  • is_pseudo_html は text/html または INCLUDES_MAGIC_TYPE を判定するために使用されます。
  • RFC 2296 に基づく品質値計算の詳細は、本構造体のフィールドを介して調整されます。
