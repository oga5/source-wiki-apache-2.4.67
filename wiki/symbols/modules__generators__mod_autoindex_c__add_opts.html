add_opts

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_autoindex.c:325

目的

Apache HTTP Server の IndexOptions ディレクティブを解析し、ディレクトリごとの設定構造体 autoindex_config_rec にオプション設定を反映させる。

シグネチャまたは形状

static const char *add_opts(cmd_parms *cmd, void *d, int argc, char *const argv[])

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • d: 対象ディレクトリの設定構造体 (autoindex_config_rec)。
  • argc: 引数の数。
  • argv: オプション文字列の配列（例: +FancyIndexing, NameWidth=20）。
  • **出力**: 成功時は NULL、エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • d は autoindex_config_rec 型としてキャスト可能であること。
  • None キーワードと他のオプションを混在させることはできない。
  • NameWidth や DescriptionWidth などの数値指定を伴うオプションは、指定された最小値（5 または 12）を下回ることはできない。

アルゴリズムまたは振る舞い

1. 引数リストを順次走査し、各オプションの先頭文字（+, -, またはなし）を確認してアクションを決定する。 2. オプション名（FancyIndexing, FoldersFirst 等）を照合し、対応するビットフラグを特定する。 3. IconWidth, NameWidth 等のパラメータ付きオプションについては、値を解析して d_cfg に格納する。 4. アクションに応じて、d_cfg 内の opts（現在の設定）、incremented_opts（追加分）、decremented_opts（削除分）を更新する。 5. 最後に、None と他のオプションの競合チェックを行い、結果を d_cfg に確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_autoindex.c:581 において AP_INIT_TAKE_ARGV マクロにより IndexOptions ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: apr_pstrdup（メモリ確保）、strcasecmp / strncasecmp（文字列比較）、atoi（数値変換）。

関連データ構造

  • autoindex_config_rec: ディレクトリごとの設定を保持する構造体。本関数により opts, incremented_opts, decremented_opts, name_width, desc_width 等が更新される。

失敗モードと境界条件

  • 不正なオプション名が指定された場合、「Invalid directory indexing option」を返す。
  • None と +/- を組み合わせた場合、または None と他のオプションを混在させた場合にエラーを返す。
  • NameWidth や DescriptionWidth で不正な値や範囲外の数値を指定した場合にエラーを返す。

保守担当者向けメモ

  • 本関数は IndexOptions の設定変更を累積的に管理するため、incremented_opts と decremented_opts のビット演算による状態管理が重要である。
  • 新しい IndexOptions を追加する場合は、オプション名とフラグの定義を追加し、for ループ内の if-else チェーンに条件分岐を追加する必要がある。
