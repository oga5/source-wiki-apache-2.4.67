sed_expr_config

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_sed.c:32

目的

sed 形式の正規表現置換ルールセットと、そのコンパイルまたは実行時に発生したエラー情報を保持するための設定構造体です。

シグネチャまたは形状

typedef struct sed_expr_config
{
    sed_commands_t *sed_cmds;
    const char *last_error;
} sed_expr_config;

入力と出力

  • **入力**: sed_cmds には libsed によってコンパイルされたコマンド列が格納されます。
  • **出力**: last_error には、libsed から報告されたエラーメッセージ（文字列）が保持されます。

前提条件と不変条件

  • sed_cmds は libsed の API を通じて初期化される必要があります。
  • last_error は、エラーが発生していない場合は NULL であることが期待されます。

アルゴリズムまたは振る舞い

本構造体は sed_config 内で input および output 用のインスタンスとして保持されます。compile_sed_expr 関数などを通じて libsed のコンパイル結果が sed_cmds に割り当てられ、フィルタ実行時にこのコマンド列が参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_sed.c 内の compile_sed_expr、init_context、および sed_response_filter / sed_request_filter 等のフィルタ関数から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • sed_config: ディレクトリ単位の設定を保持し、本構造体を input および output フィールドとして含みます。
  • sed_commands_t: libsed が提供するコンパイル済みコマンドの型です。

失敗モードと境界条件

  • sed_cmds が初期化されていない状態でフィルタ処理が開始された場合、予期せぬ動作を引き起こす可能性があります。
  • last_error は libsed からの通知に基づき更新されるため、メモリ管理（文字列の生存期間）に依存します。

保守担当者向けメモ

  • sed_expr_config は mod_sed の設定管理の核となる構造体です。
  • libsed のエラーハンドリング（sed_compile_errf 等）と密接に関連しているため、エラーメッセージのライフサイクル管理には注意してください。
