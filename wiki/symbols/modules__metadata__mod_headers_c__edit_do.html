edit_do

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_headers.c:145

目的

Header ディレクティブの edit アクションにおいて、リクエストヘッダーを反復処理する際のコンテキストを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    request_rec *r;
    header_entry *hdr;
    apr_table_t *t;
} edit_do;

入力と出力

  • **入力**: request_rec（リクエスト情報）、header_entry（適用するヘッダー操作ルール）、apr_table_t（操作対象のヘッダーテーブル）。
  • **出力**: なし（構造体として状態を保持）。

前提条件と不変条件

  • r は有効なリクエストレコードを指している必要があります。
  • hdr は実行対象のヘッダー操作ルールを保持している必要があります。
  • t は操作対象となる apr_table_t（headers_in 等）を指している必要があります。

アルゴリズムまたは振る舞い

Header edit アクション実行時に、対象となるヘッダーテーブルを走査するためのデータコンテナとして機能します。この構造体を通じて、正規表現による置換処理に必要なリクエスト情報とルールセットを処理関数へ受け渡します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_headers.c:825（インスタンスの生成）
  • modules/metadata/mod_headers.c:680（ポインタへのキャスト）
  • **呼び出し先**:
  • ソースコードの文脈上、特定の関数呼び出し先は明示されていません。

関連データ構造

  • request_rec: Apache のリクエスト情報を保持するコア構造体。
  • header_entry: ヘッダー操作のルールを定義する構造体。
  • apr_table_t: Apache Portable Runtime (APR) のキー・値ペアを保持するテーブル構造。

失敗モードと境界条件

  • 構造体内の各ポインタが NULL の状態で操作関数に渡された場合、セグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ
