proxy_fcgi_canon

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:58

目的

fcgi:// スキームを持つリクエストの URL を解析し、バックエンドへの転送用に正規化および設定を行う。

シグネチャまたは形状

static int proxy_fcgi_canon(request_rec *r, char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • url: 正規化対象の URL 文字列。
  • **出力**:
  • 成功時は OK を返す。
  • URL が fcgi: で始まらない場合は DECLINED を返す。
  • 解析エラー時は HTTP_BAD_REQUEST または HTTP_FORBIDDEN を返す。

前提条件と不変条件

  • r->per_dir_config から fcgi_dirconf_t 設定が取得可能であること。
  • r->request_config に fcgi_req_config_t が存在しない場合は、関数内で新規割り当てが行われる。

アルゴリズムまたは振る舞い

1. **スキーム確認**: URL が fcgi: で始まるか確認し、接頭辞を除去する。 2. **ネットワーク位置の解析**: ap_proxy_canon_netloc を使用してホスト名とポートを抽出する。IPv6 アドレスの場合は角括弧で囲む。 3. **パスの正規化**:

  • proxy-nocanon 等のフラグが設定されている場合は、制御文字のチェックのみを行い、パスをそのまま使用する。
  • それ以外の場合は ap_proxy_canonenc_ex を使用してパスを正規化する。

    • 4. **ファイル名設定**: r->filename を proxy:fcgi:// 形式の文字列で更新する。 5. **PATH_INFO の計算**: proxy-fcgi-pathinfo 環境変数の設定に基づき、r->path_info を計算・設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy のフックシステム（proxy_hook_canon_handler に登録）。
  • **呼び出し先**:
  • ap_proxy_port_of_scheme, ap_proxy_canon_netloc, ap_proxy_canonenc_ex (プロキシ共通ユーティリティ)
  • ap_get_module_config, ap_set_module_config (設定取得・保存)
  • ap_unescape_url_keep2f (URL デコード)

関連データ構造

  • fcgi_dirconf_t: ディレクトリ単位の設定（FPM モード判定等に使用）。
  • fcgi_req_config_t: リクエスト単位の設定（dirwalk_uri_path を保持）。

失敗モードと境界条件

  • fcgi: 以外のスキーム: DECLINED を返す。
  • URL 解析失敗: HTTP_BAD_REQUEST を返す。
  • 制御文字の混入: HTTP_FORBIDDEN を返す。
  • パス正規化失敗: HTTP_BAD_REQUEST を返す。

保守担当者向けメモ

  • 本関数は proxy-fcgi-pathinfo の値（full, first-dot, last-dot, unescape）に応じて PATH_INFO を動的に生成する。このロジックは FastCGI サーバー側の期待値と密接に関連しているため、変更時は注意が必要である。
  • FCGI_MAY_BE_FPM(dconf) マクロを用いて、PHP-FPM 特有の制約（? の禁止など）を適用している。
