passphrase_ui_close

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:794

目的

OpenSSL の UI_METHOD におけるクローズ処理として、パスフレーズ入力に使用されたパイプを解放します。

シグネチャまたは形状

static int passphrase_ui_close(UI *ui)

入力と出力

  • **入力**: UI *ui (OpenSSL の UI コンテキスト)
  • **出力**: 常に 1 を返します。

前提条件と不変条件

  • グローバル変数 readtty および writetty が、パイプが開かれている場合は有効なファイルハンドルを保持している必要があります。
  • パイプが閉じられた後は、readtty および writetty は NULL にリセットされます。

アルゴリズムまたは振る舞い

1. readtty が有効（非NULL）であるかを確認します。 2. 有効な場合、apr_file_close を使用して readtty および writetty を閉じます。 3. 閉じた後、両方の変数を NULL に設定し、リソースの二重解放を防ぎます。

呼び出し元と呼び出し先

  • **呼び出し元**: OpenSSL の UI_method_set_closer を介して登録され、OpenSSL の UI ライフサイクル管理から呼び出されます（modules/ssl/ssl_engine_pphrase.c:823）。
  • **呼び出し先**: apr_file_close (APR ライブラリ関数)。

関連データ構造

  • readtty, writetty: パイプ通信に使用される apr_file_t 型のグローバル変数。

失敗モードと境界条件

  • readtty が NULL の場合、何も行わずに終了します。
  • apr_file_close の戻り値はチェックされておらず、クローズ失敗時のエラーハンドリングは行われません。

保守担当者向けメモ

  • 本関数は UI_METHOD の一部として OpenSSL から呼び出されるため、直接的な呼び出しは避けてください。
  • パイプの管理は ssl_engine_pphrase.c 内の他の関数（ssl_pipe_child_create 等）と密接に関連しているため、パイプのライフサイクル変更時には注意が必要です。
