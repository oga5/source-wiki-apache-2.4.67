log_remote_host

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_config.c:310

目的

リクエストに関連付けられたリモートホスト名を取得し、ログ出力用にエスケープ処理を施した文字列を返します。

シグネチャまたは形状

static const char *log_remote_host(request_rec *r, char *a)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • a: ログフォーマットの引数文字列。"c" が指定された場合、接続レベルのホスト名解決を試みます。
  • **出力**:
  • エスケープ済みのリモートホスト名（const char *）。メモリは r->pool から割り当てられます。

前提条件と不変条件

  • r は有効な request_rec 構造体である必要があります。
  • 返される文字列は、ap_escape_logitem によってログ出力に適した形式に変換されています。

アルゴリズムまたは振る舞い

1. 引数 a が "c" であるかを確認します。 2. "c" の場合、ap_get_remote_host を呼び出し、接続情報からリモートホスト名を取得します。 3. それ以外の場合、ap_get_useragent_host を呼び出し、ユーザーエージェントに関連するホスト名を取得します。 4. 取得したホスト名を ap_escape_logitem に渡し、エスケープ処理を適用して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_log_config.c:1735 にて log_pfn_register を通じてログフォーマット項目 "h" に登録されています。
  • **呼び出し先**:
  • ap_get_remote_host (接続レベルのホスト名取得)
  • ap_get_useragent_host (ユーザーエージェントのホスト名取得)
  • ap_escape_logitem (ログ用エスケープ処理)

関連データ構造

  • request_rec: リクエストのコンテキスト情報およびメモリプール（r->pool）を提供します。

失敗モードと境界条件

  • ホスト名解決が失敗した場合、ap_get_remote_host または ap_get_useragent_host の仕様に従い、適切なデフォルト値または NULL が返される可能性があります。

保守担当者向けメモ

  • 本関数は log_pfn_register を介してログフォーマットのハンドラとして使用されます。
  • メモリ管理は request_rec のプールに依存しているため、明示的な解放は不要です。
