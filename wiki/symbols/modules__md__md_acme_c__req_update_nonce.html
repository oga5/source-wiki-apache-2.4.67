req_update_nonce

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme.c:114

目的

ACME サーバーからの HTTP レスポンスヘッダーに含まれる Replay-Nonce を抽出し、ACME 通信コンテキスト内のノンスト（Nonce）値を更新します。

シグネチャまたは形状

static void req_update_nonce(md_acme_t *acme, apr_table_t *hdrs)

入力と出力

  • **入力**:
  • acme: ACME 通信セッションを保持する md_acme_t 構造体へのポインタ。
  • hdrs: HTTP レスポンスヘッダーを格納した apr_table_t。
  • **出力**: なし（副作用として acme->nonce を更新）。

前提条件と不変条件

  • acme は有効なポインタである必要があります。
  • acme->p（メモリプール）は、新しいノンスト文字列を複製するために使用されます。

アルゴリズムまたは振る舞い

1. hdrs が NULL でない場合、apr_table_get を使用して Replay-Nonce ヘッダーの値を取得します。 2. ヘッダーが存在する場合、その値を acme->p プール上に複製し、acme->nonce に格納します。 3. ヘッダーが存在しない場合、acme->nonce は更新されません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c:126
  • modules/md/md_acme.c:282
  • **呼び出し先**:
  • apr_table_get (APR)
  • apr_pstrdup (APR)

関連データ構造

  • md_acme_t: ACME 通信状態を管理する構造体。nonce フィールドを保持します。
  • apr_table_t: HTTP ヘッダーを保持する APR のテーブル構造。

失敗モードと境界条件

  • hdrs が NULL の場合、関数は何も行わず安全に終了します。
  • Replay-Nonce ヘッダーがレスポンスに含まれていない場合、acme->nonce は変更されません。

保守担当者向けメモ

  • 本関数は md_acme_t のメモリプールを使用してノンスト文字列を保持します。リクエストごとのメモリ管理については、md_acme_req_t のライフサイクルと整合していることを確認してください。
