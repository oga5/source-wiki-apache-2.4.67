socache_mc_remove

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_memcache.c:266

目的

Memcached バックエンドから特定のキャッシュエントリを削除します。

シグネチャまたは形状

static apr_status_t socache_mc_remove(ap_socache_instance_t *ctx, server_rec *s,
                                      const unsigned char *id,
                                      unsigned int idlen, apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: Memcached インスタンスのコンテキスト。
  • s: サーバーレコード。エラーログ出力に使用。
  • id: 削除対象のキャッシュ ID（バイナリデータ）。
  • idlen: id の長さ。
  • p: メモリプール。
  • **出力**:
  • apr_status_t: 操作の結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • MC_KEY_LEN（254バイト）を超えるキーは生成できません。
  • socache_mc_id2key が成功し、有効なキー文字列が生成される必要があります。

アルゴリズムまたは振る舞い

1. socache_mc_id2key を呼び出し、バイナリ ID を Memcached 用の文字列キーに変換します。変換に失敗した場合は APR_EINVAL を返します。 2. apr_memcache_delete を呼び出し、Memcached サーバーから該当キーを削除します。 3. 削除操作が APR_SUCCESS 以外を返した場合、ap_log_error を使用してデバッグレベルのエラーログを出力します。 4. apr_memcache_delete の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_memcache.c:360（socache_mc 構造体内の関数ポインタとして参照）。
  • **呼び出し先**:
  • socache_mc_id2key: キー変換用ユーティリティ。
  • apr_memcache_delete: Memcached クライアントライブラリの削除関数。
  • ap_log_error: エラーログ出力用関数。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスのコンテキスト。
  • server_rec: Apache のサーバー設定構造体。
  • apr_memcache: apr_memcache ライブラリのハンドル。

失敗モードと境界条件

  • **キー変換失敗**: id が長すぎる場合、socache_mc_id2key が失敗し APR_EINVAL を返します。
  • **削除失敗**: Memcached サーバーとの通信エラーやキーの不在により apr_memcache_delete がエラーを返した場合、そのステータスが伝播されます。

保守担当者向けメモ

  • 本関数は ap_socache_provider_t インターフェースの一部として実装されています。
  • 削除時のエラーは APLOG_DEBUG レベルで記録されるため、運用環境で削除失敗を追跡する場合はログレベルの設定に注意してください。
