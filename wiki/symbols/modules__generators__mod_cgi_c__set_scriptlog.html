set_scriptlog

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgi.c:126

目的

Apache HTTP Server の設定ファイルにおいて ScriptLog 指令を処理し、CGI デバッグログの出力先パスをサーバー設定構造体に格納する。

シグネチャまたは形状

static const char *set_scriptlog(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • arg: 設定ファイルから渡されるログファイルのパス文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • cmd->server から取得した cgi_module の設定構造体 (cgi_server_conf) が初期化されている必要がある。
  • arg は有効なパスである必要がある。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定から cgi_server_conf を取得する。 2. ap_server_root_relative を呼び出し、arg で指定されたパスをサーバーのルートディレクトリからの相対パスとして解決する。 3. 解決されたパスを conf->logname に格納する。 4. パスの解決に失敗した場合は、エラーメッセージを生成して返す。 5. 正常終了時は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:177 (AP_INIT_TAKE1 指令)
  • modules/generators/mod_cgid.c:1175 (AP_INIT_TAKE1 指令)
  • **呼び出し先**:
  • ap_get_module_config
  • ap_server_root_relative
  • apr_pstrcat (エラー発生時)

関連データ構造

  • cgi_server_conf: CGI モジュールのサーバー設定を保持する構造体。logname フィールドにログパスが格納される。
  • cmd_parms: 設定解析中のコンテキスト情報。

失敗モードと境界条件

  • ap_server_root_relative がパスの解決に失敗した場合、conf->logname は NULL となり、関数はエラー文字列を返す。

保守担当者向けメモ

  • 本関数は ScriptLog 指令専用のハンドラである。
  • ログパスの解決には ap_server_root_relative を使用しており、サーバーのルートディレクトリ設定に依存する。
  • mod_cgi と mod_cgid の両方から参照されているため、変更時は両モジュールへの影響を考慮すること。
