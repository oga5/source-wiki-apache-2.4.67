modssl_ctx_init_server

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_config.c:201

目的

サーバー設定構造体（SSLSrvConfigRec）内の SSL コンテキスト（modssl_ctx_t）を初期化し、証明書や鍵の格納用配列およびセッションチケット用メモリを割り当てる。

シグネチャまたは形状

static void modssl_ctx_init_server(SSLSrvConfigRec *sc, apr_pool_t *p)

入力と出力

  • **入力**:
  • sc: 初期化対象のサーバー設定構造体（SSLSrvConfigRec）。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**: なし（sc->server を介して副作用として設定を適用）。

前提条件と不変条件

  • sc は有効なポインタであること。
  • p はメモリ割り当てが可能な有効なプールであること。

アルゴリズムまたは振る舞い

1. sc->server に対して modssl_ctx_t 構造体分のメモリを割り当てる。 2. modssl_ctx_init を呼び出し、コンテキストの基本初期化を行う。 3. mctx->pks（公開鍵・証明書関連構造体）を割り当てる。 4. mctx->pks->cert_files および mctx->pks->key_files 用に、初期容量 3 の配列を生成する。 5. HAVE_TLS_SESSION_TICKETS が定義されている場合、mctx->ticket_key 用のメモリを割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_config.c:239（直接的な証拠あり）。
  • **呼び出し先**: modssl_ctx_init（直接的な証拠あり）。

関連データ構造

  • SSLSrvConfigRec: サーバー設定を保持する構造体。
  • modssl_ctx_t: SSL 設定の本体。
  • apr_array_header_t: cert_files および key_files で使用される配列構造。

失敗モードと境界条件

  • apr_palloc / apr_pcalloc がメモリ不足により失敗した場合、Apache HTTP Server の標準的なメモリ割り当てエラーハンドリングに従い、プロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数はサーバー設定の初期化フェーズで呼び出される。
  • HAVE_TLS_SESSION_TICKETS の有無により、構造体のメモリレイアウトが変化するため、条件付きコンパイル環境に注意すること。
