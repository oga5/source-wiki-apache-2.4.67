proxy_ajp_handler

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ajp.c:771

目的

ajp:// スキームのリクエストを処理するメインハンドラであり、バックエンドサーバーへの接続確立から AJP プロトコルを用いたリクエスト転送およびレスポンス処理までを統括します。

シグネチャまたは形状

static int proxy_ajp_handler(request_rec *r, proxy_worker *worker,
                             proxy_server_conf *conf,
                             char *url, const char *proxyname,
                             apr_port_t proxyport)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報。
  • worker: 割り当てられたプロキシワーカー。
  • conf: プロキシサーバー設定。
  • url: 処理対象の URL 文字列。
  • proxyname, proxyport: プロキシ設定値。
  • **出力**:
  • 成功時は OK、失敗時は HTTP エラーコードまたは DECLINED を返します。

前提条件と不変条件

  • 入力 URL が ajp: で始まる必要があります。
  • r->pool を使用してメモリ管理が行われます。
  • 接続処理中、backend->close フラグにより接続の再利用可否が制御されます。

アルゴリズムまたは振る舞い

1. **URL 検証**: ajp: スキーム以外であれば DECLINED を返します。 2. **接続確保**: ap_proxy_acquire_connection を呼び出し、バックエンド接続を確保します。 3. **接続確立ループ**: 最大2回まで試行します。

  • ap_proxy_determine_connection で接続先を決定します。
  • ap_proxy_connect_backend でバックエンドへのソケット接続を確立します。
  • ajp_handle_cping_cpong が設定されている場合、疎通確認を行います。失敗時は接続を閉じ、再試行します。

    • 4. **リクエスト処理**: ap_proxy_ajp_request を呼び出し、AJP プロトコルによる通信を実行します。 5. **終了処理**: ap_proxy_release_connection を呼び出し、接続を解放（プールへ返却）します。

呼び出し元と呼び出し先

  • **呼び出し元**: proxy_hook_scheme_handler に登録され、Apache のリクエスト処理フックから呼び出されます。
  • **呼び出し先**:
  • ap_proxy_acquire_connection, ap_proxy_release_connection
  • ap_proxy_determine_connection, ap_proxy_connect_backend
  • ajp_handle_cping_cpong
  • ap_proxy_ajp_request

関連データ構造

  • request_rec: リクエストコンテキスト。
  • proxy_worker: バックエンドワーカー設定。
  • proxy_conn_rec: バックエンド接続状態。
  • apr_uri_t: URL 解析結果。

失敗モードと境界条件

  • URL が ajp: でない場合: DECLINED を返します。
  • 接続確保失敗時: ap_proxy_acquire_connection の戻り値に従いエラー終了します。
  • CPING/CPONG 失敗時: 接続をクローズし、最大1回まで再試行します。
  • バックエンド接続失敗時: HTTP_SERVICE_UNAVAILABLE を返します。

保守担当者向けメモ

  • 本関数は mod_proxy のフックシステムのエントリーポイントです。
  • エラー発生時は backend->close = 1 を設定することで、不正な接続がプールに再利用されるのを防ぐ設計になっています。
  • ap_proxy_ajp_request への処理委譲がメインの責務であり、低レベルなプロトコル詳細は ajp_*.c ファイル群に依存しています。
