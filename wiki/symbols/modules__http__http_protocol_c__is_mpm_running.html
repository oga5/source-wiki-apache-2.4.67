is_mpm_running

定義

  • 種別: function
  • 定義箇所: modules/http/http_protocol.c:201

目的

現在のマルチプロセッシングモジュール（MPM）が稼働中であるか、あるいは停止処理中ではないかを判定する。

シグネチャまたは形状

static int is_mpm_running(void)

入力と出力

  • 入力: なし
  • 出力: int 型。MPM が稼働中であれば 1 を返し、MPM が停止中または状態取得に失敗した場合は 0 を返す。

前提条件と不変条件

  • ap_mpm_query 関数が利用可能であること。
  • 戻り値は論理値として扱われ、1 は「稼働中」、0 は「非稼働または停止中」を意味する。

アルゴリズムまたは振る舞い

1. ap_mpm_query を呼び出し、AP_MPMQ_MPM_STATE を指定して現在の MPM 状態を取得する。 2. ap_mpm_query がエラーを返した場合、0 を返す。 3. 取得した mpm_state が AP_MPMQ_STOPPING（停止中）である場合、0 を返す。 4. 上記以外の場合、1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http/http_protocol.c:276
  • 呼び出し先: ap_mpm_query（Apache Core API）

関連データ構造

  • AP_MPMQ_MPM_STATE: MPM の状態を問い合わせるための定数。
  • AP_MPMQ_STOPPING: MPM が停止中であることを示す状態定数。

失敗モードと境界条件

  • ap_mpm_query が失敗した場合、関数は安全側に倒して 0（非稼働）を返す。
  • MPM が停止処理中の場合、稼働中とはみなされない。

保守担当者向けメモ

  • 本関数は static であり、http_protocol.c 内でのみ使用される。
  • MPM の状態遷移に関する詳細な仕様は ap_mpm.h を参照すること。
