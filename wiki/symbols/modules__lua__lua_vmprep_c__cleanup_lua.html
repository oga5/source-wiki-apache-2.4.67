cleanup_lua

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:240

目的

Apache のメモリプール破棄時に、関連付けられた Lua 仮想マシン（lua_State）を安全にクローズするために使用されるクリーンアップハンドラです。

シグネチャまたは形状

static apr_status_t cleanup_lua(void *l)

入力と出力

  • **入力**: void *l - lua_State へのポインタ。
  • **出力**: apr_status_t - 常に APR_SUCCESS を返します。

前提条件と不変条件

  • 引数 l は NULL であってはなりません（AP_DEBUG_ASSERT により検証されます）。
  • 引数 l は有効な lua_State オブジェクトを指している必要があります。

アルゴリズムまたは振る舞い

1. AP_DEBUG_ASSERT を使用して、入力ポインタ l が NULL でないことを確認します。 2. l を lua_State * にキャストし、lua_close() を呼び出して Lua VM を破棄します。 3. APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_vmprep.c:499 において、apr_pool_userdata_set のクリーンアップハンドラとして登録されています。
  • **呼び出し先**: lua_close (Lua C API)。

関連データ構造

  • lua_State: Lua VM の状態を保持する構造体。
  • apr_pool_t: Apache のメモリプール（この関数が登録されるライフサイクル管理単位）。

失敗モードと境界条件

  • l が NULL の場合、デバッグビルドではアサーションエラーが発生します。
  • lua_close への引数が不正なポインタである場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数は apr_pool_cleanup_register 等を介してプールに登録されることを前提としています。
  • lua_State の所有権はプール管理に委ねられているため、手動での lua_close 呼び出しと重複しないよう注意してください。
