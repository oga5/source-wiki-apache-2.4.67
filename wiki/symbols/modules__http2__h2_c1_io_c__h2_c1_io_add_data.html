h2_c1_io_add_data

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:361

目的

HTTP/2 セッションの出力バッファまたは接続フィルタチェーンに対し、指定された生データを追加します。

シグネチャまたは形状

apr_status_t h2_c1_io_add_data(h2_c1_io *io, const char *data, size_t length)

入力と出力

  • **入力**:
  • io: 操作対象の h2_c1_io 構造体へのポインタ。
  • data: 追加するデータを含むバッファ。
  • length: 追加するデータのバイト数。
  • **出力**:
  • apr_status_t: 操作の結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • io は初期化済みである必要があります。
  • io->buffer_output が真の場合、データは内部のスクラッチバッファにコピーされます。
  • io->buffer_output が偽の場合、データは直接 io->output ブリゲードへ書き込まれます。

アルゴリズムまたは振る舞い

1. io->session->c1 に対してトレースログを出力します。 2. io->buffer_output が有効な場合:

  • length が 0 になるまでループし、assure_scratch_space(io) を呼び出してスクラッチバッファの空き容量を確保します。
  • memcpy を使用してデータをスクラッチバッファへコピーし、io->slen を更新します。

    • 3. io->buffer_output が無効な場合:

  • apr_brigade_write を呼び出し、データを直接出力ブリゲードへ書き込みます。
  • io->buffered_len を書き込んだバイト数分加算します。

    • 4. 最終的なステータスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c: セッション処理においてフレームデータやヘッダーを書き込む際に呼び出されます。
  • **呼び出し先**:
  • assure_scratch_space (内部関数)
  • apr_brigade_write (APR API)
  • ap_log_cerror (Apache API)

関連データ構造

  • h2_c1_io: セッションの入出力状態を管理する構造体。
  • apr_bucket_brigade: Apache のデータ転送用バッファ構造体。

失敗モードと境界条件

  • apr_brigade_write が失敗した場合、そのエラーコードがそのまま返されます。
  • assure_scratch_space が失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は h2_session のライフサイクルに依存して動作します。
  • スクラッチバッファへのコピーは、TLS レコードの最適化（MTU 考慮）を目的としています。
  • h2_session.c からの呼び出しにおいて、フレームヘッダーやパディングの書き込みにも利用されています。
