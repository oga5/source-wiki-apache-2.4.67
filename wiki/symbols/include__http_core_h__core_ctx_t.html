core_ctx_t

定義

  • 種別: struct (typedef)
  • 定義箇所: include/http_core.h:788

目的

HTTP コアモジュールにおける入力フィルタ処理用のコンテキスト構造体です。

シグネチャまたは形状

typedef struct core_filter_ctx core_ctx_t;

※構造体の詳細なメンバ定義は本ソース抜粋には含まれていません。

入力と出力

  • 入力: server/core_filters.c 等において、ネットワーク接続構造体（net->in_ctx）から取得されます。
  • 出力: フィルタチェーン内でのリクエスト処理状態の保持に使用されます。

前提条件と不変条件

  • 接続ごとのリクエスト処理ライフサイクルにおいて、入力フィルタのコンテキストとして利用されます。

アルゴリズムまたは振る舞い

  • HTTP 入力フィルタの実行時、接続に関連付けられたコンテキストとして参照されます。
  • フィルタチェーン内でのデータ読み込みや状態管理の基盤となります。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • include/http_core.h:803: core_ctx_t *in_ctx; として定義内で参照。
  • server/core_filters.c:96: core_ctx_t *ctx = net->in_ctx; として取得。
  • 呼び出し先: 構造体の詳細定義が非公開のため、具体的な関数呼び出し先は不明です。

関連データ構造

  • core_filter_ctx: 本構造体の実体。
  • core_net_rec: ネットワーク接続情報を保持する構造体。in_ctx メンバを通じて本構造体を保持します。

失敗モードと境界条件

  • 構造体の初期化が不完全な場合や、net->in_ctx が NULL の状態でアクセスした場合の挙動については、本ソース抜粋からは判断できません。

保守担当者向けメモ

  • include/ap_mmn.h:388 によると、2.4.1-dev 以降で導入された構造体です。
  • 構造体のメンバ定義は core_filter_ctx を参照してください。
