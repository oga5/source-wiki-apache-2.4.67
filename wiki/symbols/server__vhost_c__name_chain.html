name_chain

定義

  • 種別: struct
  • 定義箇所: server/vhost.c:56

目的

特定の IP アドレスおよびポートに紐付く、名前ベースの仮想ホスト（NameVirtualHost）のリストを保持するためのデータ構造です。

シグネチャまたは形状

struct name_chain {
    name_chain *next;
    /* 内部メンバは server/vhost.c:57-58 に定義 */
};

入力と出力

  • **入力**: new_name_chain 関数により、apr_pool_t、server_rec、server_addr_rec を引数としてインスタンス化されます。
  • **出力**: 仮想ホストの検索および照合プロセスにおいて、リスト形式で後続の仮想ホスト情報を提供します。

前提条件と不変条件

  • 設定完了後は読み取り専用として扱われ、マルチスレッド環境下でも安全に参照可能です。
  • リストの各要素は apr_pool_t を介してメモリ管理されます。

アルゴリズムまたは振る舞い

  • ipaddr_chain が保持する名前ベースの仮想ホストリストとして機能します。
  • ap_update_vhost_from_headers 実行時、このリストを走査して Host ヘッダーや ServerName/ServerAlias との照合を行い、リクエストを処理する適切な server_rec を特定します。
  • initialnames として、設定解析時の初期状態を一時的に保持する用途にも使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • new_name_chain (生成)
  • ap_fini_vhost_config (ハッシュテーブル構築時)
  • ap_update_vhost_from_headers (リクエスト処理時)
  • **呼び出し先**:
  • next ポインタを介してリスト内の後続ノードを参照します。

関連データ構造

  • ipaddr_chain: IP/ポートベースのハッシュテーブルノード。name_chain をリストとして保持します。
  • server_rec: 仮想ホストの構成情報を保持する構造体。
  • server_addr_rec: サーバーのアドレス情報を保持する構造体。

失敗モードと境界条件

  • リストが空（NULL）の場合、名前ベースの仮想ホストが存在しないことを示します。
  • 照合時に一致するホスト名が見つからない場合、デフォルトの仮想ホスト設定へフォールバックします。

保守担当者向けメモ

  • NameVirtualHost ディレクティブは現在非推奨であり、設定時に警告が出力される点に留意してください。
  • IPHASH_STATISTICS が有効な場合、デバッグ目的で統計情報が収集されることがありますが、本構造体の基本動作には影響しません。
