cache_socache_conf

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache_socache.c:112

目的

mod_cache_socache モジュールにおいて、サーバー単位の設定情報を保持するための構造体です。

シグネチャまたは形状

typedef struct cache_socache_conf
{
    cache_socache_provider_conf *provider;
} cache_socache_conf;

入力と出力

  • 入力: サーバー設定の初期化時に apr_pcalloc を介してメモリが割り当てられます。
  • 出力: サーバー単位のキャッシュプロバイダー設定（provider）を保持し、モジュール内の各処理へ提供します。

前提条件と不変条件

  • ap_get_module_config を通じて r->server->module_config から取得されることが想定されています。
  • provider フィールドは、共有オブジェクトキャッシュのバックエンド設定を指し示します。

アルゴリズムまたは振る舞い

  • サーバー設定のコンテキストとして機能し、リクエスト処理中にキャッシュプロバイダーのインスタンスへアクセスするための起点となります。
  • 設定の生成（apr_pcalloc）およびマージ処理において、サーバー単位の構成情報を管理します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • ap_get_module_config を使用する各リクエスト処理関数（modules/cache/mod_cache_socache.c 内の複数箇所）。
  • 設定生成およびマージ関数（modules/cache/mod_cache_socache.c:1210 付近）。
  • 呼び出し先:
  • 保持している cache_socache_provider_conf 構造体を参照します。

関連データ構造

  • cache_socache_provider_conf: 本構造体が保持するプロバイダー設定の型。

失敗モードと境界条件

  • 設定が適切に初期化されていない場合、ap_get_module_config からの戻り値が NULL となる可能性があります。

保守担当者向けメモ

  • 本構造体はサーバー単位の設定を管理します。ディレクトリ単位の設定については cache_socache_dir_conf を参照してください。
