noproxy_entry

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:149

目的

プロキシを経由させない（直接接続する）ホスト名またはネットワークアドレスのリストを保持するためのエントリ構造体です。

シグネチャまたは形状

struct noproxy_entry {
    const char *name;
    struct apr_sockaddr_t *addr;
};

入力と出力

  • name: プロキシ除外対象のホスト名またはドメイン名（文字列）。
  • addr: name に対応する解決済みのアドレス情報（apr_sockaddr_t）。

前提条件と不変条件

  • apr_array_header_t 型の配列（noproxies）の要素として管理されることが想定されています。
  • name または addr のいずれか、あるいは両方が設定されることで、特定の宛先がプロキシ対象外であることを示します。

アルゴリズムまたは振る舞い

  • プロキシ設定において、リクエスト先のホストがこのリストに含まれている場合、プロキシ経由ではなく直接接続を行うための判定基準として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy.c: 設定の初期化およびリストへの追加処理で使用。
  • modules/proxy/proxy_util.c: リクエスト処理時にプロキシ除外リストを走査する際に参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_array_header_t: noproxy_entry のリストを格納するコンテナとして使用されます。
  • proxy_server_conf: 本構造体のリストを保持するサーバー設定構造体。

失敗モードと境界条件

  • name が解決できない場合、addr は NULL になる可能性があります。
  • リストが空の場合、プロキシ除外設定は適用されません。

保守担当者向けメモ

  • 本構造体は mod_proxy の設定フェーズで構築され、リクエスト処理フェーズで読み取られます。マルチスレッド環境下での設定変更については、Apache の設定ライフサイクルに従う必要があります。
