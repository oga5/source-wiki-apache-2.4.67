ssl_cleanup_pre_config

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:328

目的

Apache HTTP Server の終了処理フェーズにおいて、OpenSSL ライブラリが使用したリソースを解放し、クリーンアップを行う。

シグネチャまたは形状

static apr_status_t ssl_cleanup_pre_config(void *data)

入力と出力

  • **入力**: void *data (Apache のプールクリーンアップ登録時に渡されるデータ。本関数内では未使用)
  • **出力**: apr_status_t (常に APR_SUCCESS を返す)

前提条件と不変条件

  • OpenSSL 1.1.0 以降のバージョンでは、ライブラリ自体が OPENSSL_init_ssl() を通じて終了処理を自動管理するため、本関数は実質的な処理を行わない。
  • OpenSSL 1.0.x 以前のレガシー環境においてのみ、明示的なリソース解放を実行する。

アルゴリズムまたは振る舞い

1. **バージョン判定**: OpenSSL のバージョンを確認し、1.1.0 以降であれば何もせず終了する。 2. **レガシー環境のクリーンアップ**:

  • OBJ_cleanup(): オブジェクト定義の解放。
  • CONF_modules_free(): 設定モジュールの解放。
  • EVP_cleanup(): 暗号アルゴリズムの解放。
  • ENGINE_cleanup(): (定義時のみ) エンジンの解放。
  • SSL_COMP_free_compression_methods(): (定義時のみ) 圧縮メソッドの解放。
  • ERR_remove_thread_state() または ERR_remove_state(): スレッドごとのエラー状態の削除。
  • ERR_free_strings(): エラー文字列の解放。
  • CRYPTO_cleanup_all_ex_data(): 動的リンク時のみ、拡張データインデックスの解放を実行。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c:440 にて、apr_pool_cleanup_register を介して登録される。
  • **呼び出し先**: OpenSSL の各種クリーンアップ API (OBJ_cleanup, EVP_cleanup 等)。

関連データ構造

  • apr_pool_t (Apache のメモリ管理プール)

失敗モードと境界条件

  • **静的リンク時の制約**: modssl_running_statically が真の場合、CRYPTO_cleanup_all_ex_data() を呼び出すと、静的変数にキャッシュされたインデックスが削除され、OpenSSL の動作に悪影響を及ぼす可能性があるためスキップされる。
  • **バージョン依存**: OpenSSL のバージョンマクロ (OPENSSL_VERSION_NUMBER) に基づき、実行されるクリーンアップ処理が厳密に制御される。

保守担当者向けメモ

  • 本関数は mod_ssl の終了処理の要です。OpenSSL 3.0 以降への対応や、将来的な API 変更時には、条件分岐 (#if) の見直しが必要です。
