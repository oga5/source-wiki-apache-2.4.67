hc_get_backend

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_hcheck.c:597

目的

ヘルスチェック対象のバックエンドサーバーに対する接続を確立し、SSL/TLS設定の検証および接続の初期化を行います。

シグネチャまたは形状

int hc_get_backend(const char *proxy_function, proxy_conn_rec **backend,
                   proxy_worker *hc, sctx_t *ctx);

入力と出力

  • **入力**:
  • proxy_function: 呼び出し元の関数名（ログ出力用）。
  • hc: ヘルスチェック対象の proxy_worker。
  • ctx: サーバー単位のモジュール設定 sctx_t。
  • **出力**:
  • backend: 確立された接続情報 proxy_conn_rec へのポインタ。
  • 戻り値: 成功時は OK、失敗時はエラーコードを返します。

前提条件と不変条件

  • ctx->s が有効なサーバー設定を保持していること。
  • hc が有効なワーカー構造体であること。

アルゴリズムまたは振る舞い

1. ap_proxy_acquire_connection を呼び出し、バックエンドへの接続を取得します。失敗した場合は即座にそのステータスを返します。 2. スキームが https または wss の場合、ap_ssl_has_outgoing_handlers を確認します。設定されていない場合は警告をログ出力し、!OK を返します。 3. SSL接続の場合、backend の is_ssl フラグを立てます。 4. 最後に hc_determine_connection を呼び出し、接続の最終的な初期化を行ってその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_hcheck.c 内の以下の箇所から呼び出されています。
  • hc_check (HCHECK)
  • hc_check_cping (HCCPING)
  • hc_check_tcp (HCTCP)
  • hc_check_http (HCOH)
  • **呼び出し先**:
  • ap_proxy_acquire_connection
  • ap_ssl_has_outgoing_handlers
  • ap_log_error
  • hc_determine_connection

関連データ構造

  • proxy_conn_rec: バックエンド接続状態を保持。
  • proxy_worker: ヘルスチェック対象のワーカー定義。
  • sctx_t: モジュールのサーバーコンテキスト。

失敗モードと境界条件

  • ap_proxy_acquire_connection が OK 以外を返した場合、そのエラーを伝播します。
  • SSLスキームであるにもかかわらず mod_ssl が構成されていない場合、!OK を返します。

保守担当者向けメモ

  • 本関数はヘルスチェックの接続フェーズにおける共通処理です。SSLハンドラのチェックは mod_ssl の存在に依存しているため、環境構築時には注意が必要です。
