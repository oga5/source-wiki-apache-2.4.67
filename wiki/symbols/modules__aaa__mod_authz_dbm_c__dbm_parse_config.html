dbm_parse_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbm.c:298

目的

require ディレクティブで指定された文字列を Apache 式（ap_expr）として解析し、認可プロバイダーが利用可能な形式に変換します。

シグネチャまたは形状

static const char *dbm_parse_config(cmd_parms *cmd, const char *require_line,
                                     const void **parsed_require_line)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのコンテキスト情報。
  • require_line: 解析対象となる require ディレクティブの引数文字列。
  • **出力**:
  • parsed_require_line: 解析された ap_expr_info_t 構造体へのポインタが格納されます。
  • 戻り値: 解析成功時は NULL を返し、失敗時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • require_line は ap_expr_parse_cmd が処理可能な形式である必要があります。
  • 成功時、*parsed_require_line には有効な式情報が設定されます。

アルゴリズムまたは振る舞い

1. ap_expr_parse_cmd を呼び出し、require_line を AP_EXPR_FLAG_STRING_RESULT フラグ付きで解析します。 2. 解析中にエラーが発生した場合、cmd->temp_pool を使用してエラーメッセージを構築し、呼び出し元へ返します。 3. 解析が成功した場合、生成された ap_expr_info_t 構造体を *parsed_require_line に代入し、NULL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbm.c:320（authz_dbmgroup_provider または authz_dbmfilegroup_provider の登録構造体内で参照）。
  • **呼び出し先**: ap_expr_parse_cmd（Apache 式評価エンジン）。

関連データ構造

  • cmd_parms: 設定解析用コンテキスト。
  • ap_expr_info_t: 解析された式を保持する構造体。

失敗モードと境界条件

  • ap_expr_parse_cmd がエラーを検出した場合、関数は即座にエラー文字列を返します。この際、メモリ割り当てには cmd->temp_pool が使用されます。

保守担当者向けメモ

  • 本関数は require 行の動的な評価を可能にするためのラッパーです。
  • 戻り値が NULL でない場合、Apache の設定読み込みプロセスが停止するため、エラーメッセージは適切にフォーマットされています。
