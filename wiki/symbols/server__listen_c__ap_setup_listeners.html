ap_setup_listeners

定義

  • 種別: function
  • 定義箇所: server/listen.c:711

目的

サーバー起動時に、設定された全リスナーの初期化、ソケットのバインド、および各リスナーへのアクセプトフィルタの適用を行う。

シグネチャまたは形状

int ap_setup_listeners(server_rec *s)

入力と出力

  • **入力**: server_rec *s（サーバー設定構造体）
  • **出力**: int（初期化に成功したリスナーの総数。失敗時は 0 を返す場合がある）

前提条件と不変条件

  • ap_listeners グローバルリストが構築済みであること。
  • サーバーのライフサイクルにおいて、MPM がリスナーを必要とするタイミングで呼び出される。

アルゴリズムまたは振る舞い

1. **プロトコル設定**: 仮想ホスト (server_rec) にプロトコルが未設定の場合、バインド先アドレスとポートが一致する既存のリスナーからプロトコルを継承する。未定義の場合はデフォルトで "http" を設定する。 2. **リスナーのオープン**: systemd 連携が有効な場合は ap_systemd_listen_fds を介してソケットを取得し、それ以外の場合は open_listeners() を呼び出してソケットの作成・バインド・リスンを行う。 3. **フィルタ適用**: 全リスナーを走査し、各リスナーのバインド先と一致する仮想ホスト設定を検索する。一致した仮想ホスト、または一致がない場合はメインサーバー設定に基づき、ap_apply_accept_filter() を実行して OS 固有のアクセプトフィルタを適用する。

呼び出し元と呼び出し先

  • **呼び出し元**: 各種 MPM（event.c, mpmt_os2.c, mpm_netware.c, prefork.c, winnt.c, worker.c）から呼び出される。
  • **呼び出し先**:
  • ap_get_server_protocol, ap_set_server_protocol
  • apr_sockaddr_equal
  • open_listeners
  • ap_apply_accept_filter
  • APR_RETRIEVE_OPTIONAL_FN(ap_systemd_listen_fds) (条件付き)

関連データ構造

  • server_rec: サーバー設定および仮想ホスト情報。
  • ap_listen_rec: リスナーのソケット情報とプロトコルを保持する連結リスト。
  • server_addr_rec: 仮想ホストのバインド先アドレス情報。

失敗モードと境界条件

  • open_listeners() が失敗した場合、関数は 0 を返して終了する。
  • systemd 環境下で ap_systemd_listen_fds が取得できない場合、またはリスナー設定が不完全な場合、期待通りにリスナーが初期化されない可能性がある。

保守担当者向けメモ

  • systemd 関連のロジックは HAVE_SYSTEMD マクロで囲まれており、環境依存の挙動を含む。
  • ap_apply_accept_filter は、リスナーと仮想ホストの対応関係に基づいて適用されるため、設定変更時はこの対応ロジックに注意すること。
