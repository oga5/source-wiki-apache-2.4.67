set_force_priority

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_negotiation.c:110

目的

Apache 設定ディレクティブ ForceLanguagePriority の値を解析し、ディレクトリ単位の設定構造体 neg_dir_config に反映させる。

シグネチャまたは形状

static const char *set_force_priority(cmd_parms *cmd, void *n_, const char *w)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • n_: 設定対象の neg_dir_config 構造体（void* として渡される）。
  • w: 設定値（"None", "Prefer", "Fallback" のいずれか）。
  • **出力**:
  • 成功時は NULL を返す。
  • エラー時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • n_ は neg_dir_config 型としてキャスト可能であること。
  • neg_dir_config 内の forcelangpriority フィールドは、ビットマスクとして管理される。

アルゴリズムまたは振る舞い

1. 引数 w を大文字小文字を区別せずに比較する。 2. "None" が指定された場合:

  • 既に他のフラグが設定されていればエラーを返す。
  • FLP_NONE を設定する。

    • 3. "Prefer" または "Fallback" が指定された場合:

  • 既に FLP_NONE が設定されていればエラーを返す。
  • 対応するビットフラグ (FLP_PREFER または FLP_FALLBACK) を n->forcelangpriority に追加（OR演算）する。

    • 4. 上記以外の場合は、無効なオプションとしてエラーメッセージを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_negotiation.c:163 において、AP_INIT_ITERATE マクロ経由で設定ディレクティブとして登録されている。
  • **呼び出し先**: apr_pstrcat (エラー発生時のみ)。

関連データ構造

  • neg_dir_config: ディレクトリごとのネゴシエーション設定を保持する構造体。
  • cmd_parms: Apache 設定コマンドのコンテキスト。

失敗モードと境界条件

  • **競合エラー**: None と他のオプション（Prefer, Fallback）を混在させた場合、エラーメッセージを返して設定を拒否する。
  • **不正な値**: 定義されていない文字列が渡された場合、apr_pstrcat を使用してエラーメッセージを生成する。

保守担当者向けメモ

  • 本関数は ForceLanguagePriority ディレクティブの解析専用である。
  • FLP_NONE, FLP_PREFER, FLP_FALLBACK の各定数は、本モジュール内の他のネゴシエーションロジック（best_match 等）で参照されるため、ビットマスクの変更には注意が必要である。
