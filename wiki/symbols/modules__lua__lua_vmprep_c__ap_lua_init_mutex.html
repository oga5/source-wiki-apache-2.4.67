ap_lua_init_mutex

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:43

目的

マルチスレッド環境における Lua 仮想マシン（VM）の共有および排他制御のためのミューテックスを初期化します。

シグネチャまたは形状

void ap_lua_init_mutex(apr_pool_t *pool, server_rec *s)

入力と出力

  • **入力**:
  • pool: ミューテックスのライフサイクルを管理するための Apache メモリプール。
  • s: エラーログ出力に使用するサーバーレコード。
  • **出力**: なし（副作用としてグローバルおよびスレッドミューテックスを初期化）。

前提条件と不変条件

  • 本関数は、子プロセス初期化フェーズで呼び出されることを想定しています。
  • lua_ivm_mutex がグローバル変数として定義されている必要があります。

アルゴリズムまたは振る舞い

1. **グローバルミューテックスの初期化**: apr_global_mutex_child_init を使用して、子プロセス内での lua_ivm_mutex を再オープンします。 2. **スレッドミューテックスの初期化**: APR_HAS_THREADS が定義されている場合、apr_thread_mutex_create を使用して ap_lua_mutex を作成します。 3. **エラーハンドリング**: グローバルミューテックスの初期化に失敗した場合、エラーログを出力し exit(1) を実行してプロセスを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/mod_lua.c:2182（ap_hook_child_init を介して登録）。
  • **呼び出し先**:
  • apr_global_mutex_child_init
  • apr_global_mutex_lockfile
  • ap_log_error
  • apr_thread_mutex_create（APR_HAS_THREADS 定義時のみ）

関連データ構造

  • apr_pool_t: メモリ管理用。
  • server_rec: サーバー設定およびログ出力用。
  • lua_ivm_mutex: グローバルな Lua VM 用ミューテックス。
  • ap_lua_mutex: サーバープール用ミューテックス。

失敗モードと境界条件

  • apr_global_mutex_child_init が APR_SUCCESS を返さない場合、致命的エラーとしてプロセスが終了します。
  • スレッド非対応環境（APR_HAS_THREADS が未定義）では、スレッドミューテックスの初期化はスキップされます。

保守担当者向けメモ

  • 本関数は mod_lua の子プロセス初期化フックとして動作するため、マルチスレッド環境での競合を防ぐ重要な役割を担っています。
  • exit(1) を呼び出すため、初期化失敗時の影響範囲は当該プロセス全体に及びます。
