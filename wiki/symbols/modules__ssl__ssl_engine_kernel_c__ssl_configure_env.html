ssl_configure_env

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:279

目的

リクエストの環境変数 (subprocess_env) を走査し、SSL 接続のシャットダウン方式 (shutdown_type) を設定する。

シグネチャまたは形状

static void ssl_configure_env(request_rec *r, SSLConnRec *sslconn)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。subprocess_env テーブルを参照する。
  • sslconn: SSL 接続状態を保持する SSLConnRec 構造体。
  • **出力**:
  • sslconn->shutdown_type: SSL_SHUTDOWN_TYPE_UNCLEAN または SSL_SHUTDOWN_TYPE_ACCURATE に更新される（デフォルトは SSL_SHUTDOWN_TYPE_STANDARD）。

前提条件と不変条件

  • r->subprocess_env が有効なテーブルであること。
  • 設定されるシャットダウンフラグは、ssl-unclean-shutdown または ssl-accurate-shutdown というキー名で環境変数に存在する必要がある。

アルゴリズムまたは振る舞い

1. sslconn->shutdown_type を SSL_SHUTDOWN_TYPE_STANDARD で初期化する。 2. r->subprocess_env テーブルを反復処理する。 3. キーが 's' で始まるエントリを検索し、ssl- で始まるか確認する。 4. ssl-unclean が見つかれば SSL_SHUTDOWN_TYPE_UNCLEAN を、ssl-accurate が見つかれば SSL_SHUTDOWN_TYPE_ACCURATE を sslconn->shutdown_type に代入し、直ちに終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_kernel.c:268
  • **呼び出し先**: 外部関数 apr_table_elts を使用。

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • SSLConnRec: mod_ssl 内部の接続コンテキスト。
  • apr_table_entry_t: 環境変数テーブルの各要素。

失敗モードと境界条件

  • 環境変数に該当するキーが存在しない場合、デフォルトの SSL_SHUTDOWN_TYPE_STANDARD が維持される。
  • 複数の該当キーが存在する場合、最初に見つかったもののみが適用される。

保守担当者向けメモ

  • 本関数は SetEnvIf ディレクティブ等で設定された環境変数を介して、SSL シャットダウンの挙動を制御する。
  • strncmp を用いた文字列比較を行っているため、キー名の変更には注意が必要。
