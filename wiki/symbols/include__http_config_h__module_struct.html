module_struct

定義

  • 種別: struct
  • 定義箇所: include/http_config.h:345

目的

Apache HTTP Server のモジュールを表現し、サーバーのライフサイクル（初期化、設定読み込み、リクエスト処理）におけるフックや設定管理インターフェースを保持する。

シグネチャまたは形状

struct module_struct {
    int version;
    int minor_version;
    int module_index;
    const char *name;
    void *dynamic_load_handle;
    struct module_struct *next;
    unsigned long magic;
    void (*rewrite_args) (process_rec *process);
    void *(*create_dir_config) (apr_pool_t *p, char *dir);
    void *(*merge_dir_config) (apr_pool_t *p, void *base_conf, void *new_conf);
    void *(*create_server_config) (apr_pool_t *p, server_rec *s);
    void *(*merge_server_config) (apr_pool_t *p, void *base_conf, void *new_conf);
    const command_rec *cmds;
    void (*register_hooks) (apr_pool_t *p);
    int flags;
};

入力と出力

  • **入力**: サーバー起動および設定読み込み時に、モジュール定義として静的または動的にロードされる。
  • **出力**: サーバーの各フェーズ（設定解析、リクエスト処理）において、登録された関数ポインタを通じて設定データや処理ロジックが呼び出される。

前提条件と不変条件

  • module_index はサーバー起動時に割り当てられ、実行中に変更してはならない。
  • version はサーバーの API バージョンと互換性がある必要がある。
  • magic は DSO（動的共有オブジェクト）機能においてモジュール構造体を識別するために使用される。

アルゴリズムまたは振る舞い

1. **初期化**: サーバー起動時にモジュールリストが走査され、各モジュールの register_hooks が呼び出される。 2. **設定解析**: cmds テーブルに基づき、設定ファイル内のディレクティブが解析される。 3. **設定管理**: create_dir_config / create_server_config により設定構造体が生成され、merge_dir_config / merge_server_config により親コンテキストから子コンテキストへ設定が継承される。 4. **引数書き換え**: MPM（マルチプロセッシングモジュール）のみが rewrite_args を使用してコマンドライン引数を変更できる。

呼び出し元と呼び出し先

  • **呼び出し元**: サーバーのコア設定システム（server/config.c 等）が、設定ファイルの走査やライフサイクルイベントに応じて本構造体の関数ポインタを呼び出す。
  • **呼び出し先**: 各モジュールが実装した設定生成関数、マージ関数、およびフック登録関数。
  • **注記**: 具体的な呼び出し元・呼び出し先のコードパスは本ヘッダーファイル外の実装に依存する。

関連データ構造

  • command_rec: モジュールが定義するディレクティブのテーブル。
  • apr_pool_t: 設定データ等のメモリ割り当てに使用されるプール。
  • process_rec, server_rec: 設定処理のコンテキストとして渡される構造体。

失敗モードと境界条件

  • module_index の範囲外アクセスに対する保護は、デバッグビルド以外では提供されない。
  • バージョン不一致がある場合、モジュールのロードや初期化が失敗する可能性がある。

保守担当者向けメモ

  • next ポインタによりモジュールリストが構成される。
  • flags フィールドは AP_MODULE_FLAG_* を保持する。
  • dynamic_load_handle は内部使用専用である。
