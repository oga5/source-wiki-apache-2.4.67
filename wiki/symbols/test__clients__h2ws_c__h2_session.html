h2_session

定義

  • 種別: struct
  • 定義箇所: test/clients/h2ws.c:334

目的

HTTP/2 セッションの状態を保持し、nghttp2 ライブラリを用いた接続管理、ストリームのライフサイクル管理、および I/O 状態の追跡を行うための構造体です。

シグネチャまたは形状

struct h2_session {
    const char *server_name;
    const char *connect_host;
    uint16_t connect_port;
    int fd;
    nghttp2_session *ngh2;
    struct h2_stream *streams;
    int aborted;
    int want_io;
};

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。各フィールドは h2_session_open() で初期化され、h2_session_run() 内の nghttp2 コールバックや I/O 処理を通じて読み書きされます。

前提条件と不変条件

  • fd は非ブロッキングモードで設定された有効な TCP ソケットである必要があります。
  • ngh2 は nghttp2_session_client_new() 等で適切に初期化されている必要があります。
  • streams は h2_stream 構造体のリストを指し、ストリームのライフサイクル管理に使用されます。

アルゴリズムまたは振る舞い

  • **セッション管理**: nghttp2 のセッションオブジェクトを保持し、プロトコルイベントのディスパッチを仲介します。
  • **I/O 監視**: want_io フラグにより、poll() ループにおいて読み取りまたは書き込みのどちらが必要かを制御します。
  • **状態追跡**: aborted フラグにより、セッションの異常終了や中断状態を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: test/clients/h2ws.c 内の h2_session_open（初期化）、h2_session_run（イベントループ）、および各 nghttp2 コールバック関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出す主体ではありません。
  • **注意**: ツールによる索引結果には modules/http2/ 配下のファイルが含まれていますが、これらは test/clients/h2ws.c とは別のコンテキスト（Apache HTTP Server 本体のモジュール）であり、本構造体との直接的な関係性はソースコードの文脈からは確認できません。

関連データ構造

  • h2_stream: セッション内の個別のストリーム状態を保持するリスト構造。
  • nghttp2_session: nghttp2 ライブラリが提供する HTTP/2 セッション管理オブジェクト。

失敗モードと境界条件

  • fd が無効な場合や接続が切断された場合、aborted フラグが設定され、セッションループが終了します。
  • メモリ不足により ngh2 の初期化やストリームの追加に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はテストツール h2ws.c 専用であり、Apache HTTP Server 本体の h2_session とは名前が重複していますが、別個の実装です。
  • 終了時のメモリ解放処理は網羅的ではないため、テスト実行後のメモリリークには注意してください。
