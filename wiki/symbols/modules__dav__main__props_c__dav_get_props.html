dav_get_props

定義

  • 種別: function
  • 定義箇所: modules/dav/main/props.c:723

目的

PROPFIND リクエストに応答して、指定されたリソースのライブプロパティおよびデッドプロパティを検索・統合し、XML 形式の multistatus レスポンス（propstat）を生成します。

シグネチャまたは形状

dav_get_props_result dav_get_props(dav_propdb *propdb, apr_xml_doc *doc)

入力と出力

  • **入力**:
  • propdb: プロパティデータベースのコンテキスト。
  • doc: クライアントから要求されたプロパティ名を含む XML ドキュメント。
  • **出力**:
  • dav_get_props_result: 生成された propstat リストと、使用された XML 名前空間定義（xmlns）を含む構造体。

前提条件と不変条件

  • propdb は有効なリソースコンテキストを保持している必要があります。
  • doc->root には <D:prop> 要素が含まれている必要があります。
  • ライブプロパティの検索には dav_find_liveprop が使用され、結果は elem->priv にキャッシュされます。

アルゴリズムまたは振る舞い

1. **初期化**: ライブプロパティ要素の準備と、XML 出力用のバッファ（hdr_good, hdr_bad, hdr_ns）を初期化します。 2. **プロパティ走査**: 要求された各プロパティに対し、以下の順序で解決を試みます。

  • **ライブプロパティ**: dav_insert_liveprop を呼び出し、成功すれば値を hdr_good に追加し、関連する名前空間を hdr_ns に登録します。
  • **デッドプロパティ**: ライブプロパティとして解決できない場合、propdb が未オープンであれば dav_really_open_db で開き、db_hooks->output_value を介してデータベースから値を取得します。

    • 3. **エラー処理**: どちらでも見つからないプロパティは hdr_bad に追加され、最終的に 404 Not Found ステータスとしてまとめられます。 4. **統合**: hdr_good（成功分）と hdr_bad（失敗分）を連結し、最終的な XML 構造を構築して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c:2122（PROPFIND メソッド処理内）。
  • **呼び出し先**:
  • dav_get_liveprop_element, dav_find_liveprop, dav_insert_liveprop
  • dav_xmlns_create, dav_insert_xmlns, dav_xmlns_generate
  • dav_really_open_db
  • db_hooks->output_value, db_hooks->define_namespaces

関連データ構造

  • dav_propdb: プロパティ操作のメインコンテキスト。
  • dav_get_props_result: 戻り値用構造体。
  • dav_liveprop_elem: ライブプロパティのキャッシュ用構造体。
  • apr_xml_doc, apr_text_header: XML 生成用 APR ユーティリティ。

失敗モードと境界条件

  • **DB オープン失敗**: propdb->deferred が真の場合、DB オープン時にエラーが発生する可能性がありますが、現在の実装ではエラー伝播が不完全です。
  • **プロパティ未定義**: ライブ・デッド双方で見つからない場合、404 Not Found の propstat が生成されます。
  • **メモリ不足**: apr_pcalloc 等のメモリ割り当て失敗時は、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • elem->priv はリクエストプール（r->pool）を使用しており、propdb->p よりも長寿命である点に注意してください。
  • PROPPATCH 関連の複雑なエラーハンドリングにおいて、一部エラー伝播が不完全な箇所があります。
