reset_process_pconf

定義

  • 種別: function
  • 定義箇所: server/main.c:293

目的

サーバーの設定用メモリプール (pconf) を初期化またはリセットし、設定再読み込みのためのクリーンアップフックを登録します。

シグネチャまたは形状

static void reset_process_pconf(process_rec *process)

入力と出力

  • **入力**: process (process_rec *) - プロセス情報を保持する構造体。
  • **出力**: なし（副作用として process->pconf の状態を変更）。

前提条件と不変条件

  • process ポインタは有効である必要があります。
  • pconf が存在する場合、apr_pool_clear によりそのメモリプール内の全データが破棄され、ap_server_conf が NULL にリセットされます。
  • pconf が存在しない場合、process->pool を親として新規作成されます。

アルゴリズムまたは振る舞い

1. process->pconf が既に存在するか確認します。

  • 存在する場合: apr_pool_clear でプールを空にし、グローバル変数 ap_server_conf を NULL に設定します。
  • 存在しない場合: apr_pool_create で新規プールを作成し、タグ "pconf" を付与します。

    • 2. apr_pool_pre_cleanup_register を呼び出し、プール破棄時に deregister_all_hooks が実行されるよう登録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/main.c:351
  • server/main.c:773
  • **呼び出し先**:
  • apr_pool_clear / apr_pool_create (APR)
  • apr_pool_tag (APR)
  • apr_pool_pre_cleanup_register (APR)
  • deregister_all_hooks (内部関数)

関連データ構造

  • process_rec: プロセス全体のライフサイクルとメモリプールを管理する構造体。
  • apr_pool_t: APR のメモリ管理単位。

失敗モードと境界条件

  • メモリ不足が発生した場合、APR の内部挙動によりプロセスが終了する可能性があります（abort_on_oom 参照）。
  • process が NULL の場合、未定義の動作となります。

保守担当者向けメモ

  • 本関数は設定の再読み込み時に呼び出されます。pconf は設定関連データを保持する主要なプールであるため、ここでのクリア処理は設定の整合性を保つために不可欠です。
  • deregister_all_hooks の登録は、設定リセット時に古いフックが残存することを防ぐための重要なクリーンアップ処理です。
