ssl_hook_default_port

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:653

目的

現在のリクエストが SSL/TLS 接続である場合に、デフォルトのポート番号（443）を Apache のコアシステムへ通知する。

シグネチャまたは形状

apr_port_t ssl_hook_default_port(const request_rec *r)

入力と出力

  • **入力**: r (現在のリクエストを保持する request_rec 構造体へのポインタ)
  • **出力**: apr_port_t 型のポート番号。TLS 接続と判定された場合は 443、それ以外の場合は 0 を返す。

前提条件と不変条件

  • 入力される request_rec *r は有効なポインタである必要がある。
  • modssl_request_is_tls 関数が、リクエストの SSL 状態を正しく判定できる環境にあること。

アルゴリズムまたは振る舞い

1. modssl_request_is_tls(r, NULL) を呼び出し、現在のリクエストが TLS を使用しているかを確認する。 2. TLS 接続であれば 443 を返し、そうでなければ 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c:733 にて、ap_hook_default_port を通じて Apache のフックシステムに登録されている。
  • **呼び出し先**: modssl_request_is_tls を内部で呼び出す。

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • apr_port_t: APR (Apache Portable Runtime) で定義されるポート番号型。

失敗モードと境界条件

  • r が NULL の場合、modssl_request_is_tls の実装に依存する（ソースコード上では明示的なガードは確認できない）。
  • リクエストが SSL 接続でない場合、戻り値は 0 となり、デフォルトポートの指定は行われない。

保守担当者向けメモ

  • 本関数は ap_hook_default_port に登録されるフック関数である。
  • modssl_request_is_tls の判定ロジックが変更された場合、本関数の戻り値も連動して変化する。
