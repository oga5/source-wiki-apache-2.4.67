h2_proxy_stream

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_session.c:33

目的

バックエンドサーバーに対する HTTP/2 プロキシ通信において、個別のリクエスト/レスポンスのライフサイクル、状態、および入出力データを管理する内部構造体です。

シグネチャまたは形状

typedef struct h2_proxy_stream {
    int id;
    apr_pool_t *pool;
    h2_proxy_session *session;
    const char *url;
    request_rec *r;
    conn_rec *cfront;
    h2_proxy_request *req;
    const char *real_server_uri;
    const char *p_server_uri;
    int standalone;
    h2_proxy_stream_state_t state;
    unsigned int suspended : 1;
    unsigned int waiting_on_100 : 1;
    unsigned int waiting_on_ping : 1;
    unsigned int headers_ended : 1;
    uint32_t error_code;
    int proxy_status;
    apr_bucket_brigade *input;
    apr_off_t data_sent;
    apr_bucket_brigade *output;
    apr_off_t data_received;
    apr_table_t *saves;
} h2_proxy_stream;

入力と出力

  • **入力**: apr_bucket_brigade *input を通じてフロントエンドからのリクエストデータを受け取り、バックエンドへ転送します。
  • **出力**: apr_bucket_brigade *output を通じてバックエンドからのレスポンスデータを受け取り、フロントエンドへ転送します。

前提条件と不変条件

  • 各ストリームは apr_pool_t によってメモリ管理されます。
  • h2_proxy_session 構造体と紐付いており、セッションのライフサイクルに依存します。

アルゴリズムまたは振る舞い

  • **状態管理**: h2_proxy_stream_state_t を用いてストリームの状態を追跡します。
  • **フロー制御**: suspended フラグや waiting_on_100、waiting_on_ping などのビットフィールドにより、非同期的な通信待ち状態を制御します。
  • **データ転送**: data_sent および data_received により、送受信したデータ量を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c 内のストリーム操作関数群（stream_resume, submit_trailers, process_proxy_header, h2_proxy_stream_add_header_out, h2_proxy_stream_end_headers_out など）から参照されます。
  • **呼び出し先**: 直接的な関数呼び出し関係はソースコードの定義範囲内では明示されていません。

関連データ構造

  • h2_proxy_session: ストリームを保持する親セッション構造体。
  • request_rec, conn_rec: Apache HTTP Server の標準的なリクエストおよび接続コンテキスト。
  • apr_bucket_brigade: データ転送に使用される Apache のバケット管理構造体。

失敗モードと境界条件

  • error_code フィールドにより、HTTP/2 レベルのエラー状態を保持します。
  • proxy_status により、プロキシ処理時のステータスコードを管理します。

保守担当者向けメモ

  • 本構造体は modules/http2/h2_proxy_session.c の内部で使用されるプライベートな構造体です。
  • ビットフィールドによるフラグ管理が行われているため、状態遷移の追加時には既存のフラグとの競合に注意してください。
