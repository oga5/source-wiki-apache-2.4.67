mpm_common_pre_config

定義

  • 種別: function
  • 定義箇所: server/mpm_common.c:164

目的

Apache HTTP Server の起動時において、マルチプロセッシングモジュール（MPM）に関連するグローバル設定変数をデフォルト値で初期化します。

シグネチャまたは形状

void mpm_common_pre_config(apr_pool_t *pconf)

入力と出力

  • **入力**: pconf (設定フェーズで使用されるメモリプールへのポインタ)
  • **出力**: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • サーバーの起動シーケンス（pre-config フェーズ）で呼び出されることを想定しています。
  • 本関数実行後、各設定変数はデフォルト値またはシステム標準値で初期化されます。

アルゴリズムまたは振る舞い

以下のグローバル変数を初期値に設定します。 1. ap_pid_fname: DEFAULT_PIDLOG に設定。 2. ap_max_requests_per_child: 0（制限なし）に設定。 3. ap_coredump_dir: ap_server_root の値で初期化。 4. ap_coredumpdir_configured: 0（未設定）に設定。 5. ap_graceful_shutdown_timeout: 0（制限なし）に設定。 6. ap_max_mem_free: ALLOCATOR_MAX_FREE_DEFAULT に設定。 7. ap_thread_stacksize: 0（システムデフォルトを使用）に設定。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:5121
  • **呼び出し先**: apr_cpystrn（APR 文字列コピー関数）

関連データ構造

  • apr_pool_t: 設定情報のメモリ管理に使用されるプール。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • ap_coredump_dir の初期化において、ap_server_root が sizeof(ap_coredump_dir) を超える長さである場合、apr_cpystrn の仕様に従い切り詰められます。

保守担当者向けメモ

  • 本関数は MPM の共通設定を管理する基盤です。新しいグローバル設定変数を追加した場合は、この関数で適切なデフォルト値を設定してください。
  • ap_thread_stacksize 等の変数は、プラットフォームや MPM の実装によって後続のフェーズで上書きされる可能性があります。
