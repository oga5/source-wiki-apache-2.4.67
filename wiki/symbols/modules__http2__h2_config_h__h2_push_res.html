h2_push_res

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_config.h:57

目的

HTTP/2 Server Push を構成するためのリソース定義を保持する構造体です。サーバーまたはディレクトリ単位でプッシュ対象となるリソースの URI と、その優先度を管理するために使用されます。

シグネチャまたは形状

typedef struct h2_push_res {
    const char *uri_ref;
    int critical;
} h2_push_res;

入力と出力

本構造体は設定データとして利用されます。

  • uri_ref: プッシュ対象リソースの URI を示す文字列。
  • critical: プッシュの優先度を示すフラグ（非ゼロで重要/クリティカルなリソースであることを示す）。

前提条件と不変条件

  • h2_push_res のインスタンスは、通常 apr_array_header_t（APR 配列）に格納され、設定の一部として管理されます。
  • uri_ref は有効な文字列を指している必要があります。

アルゴリズムまたは振る舞い

  • h2_config.c 内の add_push 関数等を通じて、設定解析時に配列へ追加されます。
  • h2_push.c において、この構造体を参照することでプッシュ対象のリストを走査し、リソースのプッシュ処理を決定します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/http2/h2_c2.c:442: プッシュリストの走査時に使用。
  • modules/http2/h2_config.c:894: add_push 関数内で新しいプッシュリソースの追加に使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • apr_array_header_t: h2_push_res のリストを保持するために使用されます。
  • h2_config.c: 設定構造体の一部として本構造体を利用します。

失敗モードと境界条件

  • uri_ref が NULL の場合や、メモリ不足により apr_array_header_t への追加が失敗する可能性があります。
  • 呼び出し元情報の詳細は、ソースコード内の add_push や h2_c2.c のループ処理における配列のインデックス範囲に依存します。

保守担当者向けメモ

  • 本構造体は h2_config.h で定義され、h2_push.c で利用されるため、プッシュ設定の仕様変更時には両ファイルを確認してください。
  • critical フラグの解釈については h2_push.c の実装ロジックを参照してください。
