buffer_conf

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_buffer.c:39

目的

ディレクトリ単位で設定されるバッファリングの構成情報を保持します。

シグネチャまたは形状

typedef struct buffer_conf {
    apr_off_t size;   /* バッファサイズの上限 */
    int size_set;     /* サイズが明示的に設定されているかを示すフラグ */
} buffer_conf;

入力と出力

  • 入力: BufferSize ディレクティブにより設定値が与えられます。
  • 出力: 設定の初期化およびマージ処理において、この構造体のインスタンスが生成・参照されます。

前提条件と不変条件

  • size_set が 0 の場合、size は未設定とみなされます。
  • apr_pcalloc を使用してメモリが確保されるため、初期状態では各メンバはゼロ初期化されます。

アルゴリズムまたは振る舞い

  • 設定の初期化（create_buffer_config）および継承（merge_buffer_config）において、ディレクトリごとのバッファサイズを管理するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: create_buffer_config および merge_buffer_config 内でインスタンスが生成されます。
  • 呼び出し先: set_buffer_size ディレクティブハンドラにより、設定値が書き込まれます。

関連データ構造

  • buffer_ctx: リクエストごとの実行時状態を保持する構造体であり、この buffer_conf から設定値を読み取ります。

失敗モードと境界条件

  • 設定値が正しくパースされない場合や、メモリ確保（apr_pcalloc）が失敗した場合には、モジュールの動作に影響を及ぼします。

保守担当者向けメモ

  • 本構造体は mod_buffer の設定管理の基盤です。設定の継承ロジックを変更する際は、merge_buffer_config における size_set の評価ロジックに注意してください。
