ap_update_vhost_given_ip

定義

  • 種別: function
  • 定義箇所: server/vhost.c:1280

目的

接続確立時に、接続のローカルアドレス（IP/ポート）に基づいて、該当する仮想ホストの候補リストとベースサーバーを conn_rec に割り当てます。

シグネチャまたは形状

void ap_update_vhost_given_ip(conn_rec *conn)

入力と出力

  • **入力**: conn_rec *conn （接続情報。local_addr を参照して検索を行う）
  • **出力**: なし（副作用として conn->vhost_lookup_data および conn->base_server を更新）

前提条件と不変条件

  • conn->local_addr が有効なアドレス情報を含んでいること。
  • 仮想ホストのハッシュテーブル（iphash_table）が ap_fini_vhost_config によって構築済みであること。

アルゴリズムまたは振る舞い

1. **完全一致検索**: conn->local_addr をキーとしてハッシュテーブルを検索します。一致する ipaddr_chain が見つかった場合、その names（名前ベースの仮想ホストリスト）と server を conn に格納して終了します。 2. **デフォルトサーバー検索**: 完全一致がない場合、conn->local_addr->port を使用して、そのポートで待機しているデフォルトサーバー（find_default_server）を検索します。見つかった場合は同様に conn を更新します。 3. **フォールバック**: いずれも該当しない場合、conn->vhost_lookup_data を NULL に設定し、メインサーバーのみが有効な状態となります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/connection.c:212
  • server/mpm/event/event.c:1044
  • **呼び出し先**:
  • find_ipaddr (内部関数)
  • find_default_server (内部関数)

関連データ構造

  • conn_rec: 接続状態を保持する構造体。
  • ipaddr_chain: IP/ポートをキーとするハッシュテーブルのノード。
  • name_chain: 特定の IP/ポートに紐付く名前ベースの仮想ホストリスト。

失敗モードと境界条件

  • 該当する IP/ポートが見つからない場合、conn->vhost_lookup_data は NULL に設定され、名前ベースの仮想ホスト解決は行われません。
  • conn->local_addr が未初期化の場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本関数は接続確立の初期段階で呼ばれます。リクエストヘッダーに基づく最終的な仮想ホストの決定は、後続の ap_update_vhost_from_headers が担当します。
  • vhost_lookup_data に格納されるリストは、名前ベースの仮想ホスト解決時に update_server_from_aliases によって走査されます。
