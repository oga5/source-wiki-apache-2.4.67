lua_get_db_handle

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:37

目的

Lua スタック上のテーブルから、内部で保持されているデータベースハンドル（lua_db_handle）を取得します。

シグネチャまたは形状

static lua_db_handle *lua_get_db_handle(lua_State *L)

入力と出力

  • **入力**: lua_State *L（Lua 仮想マシン状態）。スタックのインデックス 1 にデータベースハンドルを保持する Lua テーブルが配置されている必要があります。
  • **出力**: 成功時は lua_db_handle 構造体へのポインタを返します。

前提条件と不変条件

  • スタックのインデックス 1 は LUA_TTABLE 型である必要があります。
  • インデックス 1 のテーブルのキー 0 には、LUA_TUSERDATA 型として lua_db_handle が格納されている必要があります。

アルゴリズムまたは振る舞い

1. luaL_checktype を使用し、スタックのインデックス 1 がテーブルであることを検証します。 2. lua_rawgeti を呼び出し、テーブルのインデックス 0 に格納されている値（ユーザーデータ）をスタックトップにプッシュします。 3. luaL_checktype を使用し、スタックトップの値がユーザーデータであることを検証します。 4. lua_topointer を使用してユーザーデータからポインタを取り出し、lua_db_handle * にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の関数から呼び出されています。
  • lua_db_close (modules/lua/lua_dbd.c:66)
  • lua_db_gc (modules/lua/lua_dbd.c:132)
  • lua_db_active (modules/lua/lua_dbd.c:162)
  • lua_db_query (modules/lua/lua_dbd.c:206)
  • lua_db_select (modules/lua/lua_dbd.c:320)
  • lua_db_prepared_select (modules/lua/lua_dbd.c:573)
  • lua_db_prepared_query (modules/lua/lua_dbd.c:628)
  • **呼び出し先**: Lua C API (luaL_checktype, lua_rawgeti, lua_topointer) を呼び出します。

関連データ構造

  • lua_db_handle: データベース接続状態を保持する構造体。

失敗モードと境界条件

  • スタックのインデックス 1 がテーブルでない場合、またはキー 0 にユーザーデータが存在しない（または型が異なる）場合、luaL_checktype により Lua エラーが発生し、処理が中断されます。

保守担当者向けメモ

  • 本関数は、Lua 側から渡されたテーブルを介して C 側のデータベースハンドルを安全に取り出すための共通ユーティリティです。型チェックを厳格に行っているため、Lua 側でテーブル構造が変更された場合に影響を受けます。
