get_line

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ocsp.c:154

目的

OCSP レスポンダから送信された HTTP レスポンスのストリームから、1行単位で文字列を読み取り、改行コードを除去してメモリプール上に返却する。

シグネチャまたは形状

char *get_line(apr_bucket_brigade *bbout, apr_bucket_brigade *bbin, conn_rec *c, apr_pool_t *p);

入力と出力

  • **入力**:
  • bbout: 読み取った行を一時的に格納するためのバケットブリゲード。
  • bbin: 読み取り元のソースとなるバケットブリゲード。
  • c: ログ出力に使用する接続レコード。
  • p: 文字列の割り当てに使用するメモリプール。
  • **出力**:
  • 成功時: 読み取った行の内容（改行コード除去済み、ヌル終端文字列）。
  • 失敗時: NULL。

前提条件と不変条件

  • bbout は呼び出しのたびに apr_brigade_cleanup によって初期化される。
  • 読み取られた行は必ず APR_ASCII_LF で終了している必要がある。
  • APR_ASCII_CR が含まれる場合は、それも除去される。

アルゴリズムまたは振る舞い

1. bbout をクリーンアップする。 2. apr_brigade_split_line を使用し、bbin から最大 8192 バイトまでを bbout に切り出す。 3. apr_brigade_pflatten を使用し、ブリゲードの内容をメモリプール p 上の文字列 line にフラット化する。 4. 読み取りサイズが 0 の場合、または行末が LF でない場合はエラーとして NULL を返す。 5. 行末の LF を \0 に置換し、直前が CR であればそれも \0 に置換する。 6. 処理された文字列 line を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_util_ocsp.c:219
  • modules/ssl/ssl_util_ocsp.c:233
  • **呼び出し先**:
  • apr_brigade_cleanup
  • apr_brigade_split_line
  • ap_log_cerror
  • apr_brigade_pflatten

*注: modules/http2/h2_c2_filter.c および support/htdigest.c に同名のシンボル参照が見られるが、本関数との直接的な呼び出し関係はソースコードの文脈からは確認できない。*

関連データ構造

  • apr_bucket_brigade: HTTP ストリームのバッファリングに使用。
  • conn_rec: 接続状態およびログ出力用。
  • apr_pool_t: メモリ管理用。

失敗モードと境界条件

  • apr_brigade_split_line が失敗した場合、エラーログを出力し NULL を返す。
  • 読み取ったデータが空（len == 0）の場合、エラーログを出力し NULL を返す。
  • 行末が LF でない場合（行が長すぎる等）、エラーログを出力し NULL を返す。

保守担当者向けメモ

  • 本関数は mod_ssl 内の簡易 HTTP クライアント実装の一部である。
  • 8192 バイトを超える行はエラーとして扱われる。
  • 将来的に標準的な HTTP クライアントライブラリへ移行する際は、本関数を含む簡易実装全体が置き換え対象となる。
