ssl_scache_store

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_scache.c:113

目的

SSL セッション情報をシリアライズし、設定された socache プロバイダーを介して永続化（キャッシュ）します。

シグネチャまたは形状

BOOL ssl_scache_store(server_rec *s, IDCONST UCHAR *id, int idlen,
                      apr_time_t expiry, SSL_SESSION *sess,
                      apr_pool_t *p)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • id: セッション識別子（キー）。
  • idlen: 識別子の長さ。
  • expiry: 有効期限。
  • sess: 保存対象の SSL_SESSION 構造体。
  • p: メモリ割り当て用プール。
  • **出力**:
  • BOOL: 保存に成功した場合は TRUE、失敗またはサイズ超過の場合は FALSE を返します。

前提条件と不変条件

  • SSL_SESSION の DER 形式へのシリアライズ結果が MODSSL_SESSION_MAX_DER を超える場合、保存は行われずエラーとなります。
  • mc->sesscache が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. i2d_SSL_SESSION を使用して、セッションを DER 形式のバイナリに変換します。 2. 変換後のサイズが MODSSL_SESSION_MAX_DER を超える場合、エラーログを出力して FALSE を返します。 3. キャッシュプロバイダーがスレッドセーフでない (AP_SOCACHE_FLAG_NOTMPSAFE) 場合、ssl_mutex_on を呼び出して排他制御を行います。 4. プロバイダーの store メソッドを呼び出し、セッションデータを保存します。 5. 排他制御を行った場合は ssl_mutex_off を呼び出して解除します。 6. プロバイダーの戻り値が APR_SUCCESS であれば TRUE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_kernel.c:1929
  • **呼び出し先**:
  • i2d_SSL_SESSION (OpenSSL API)
  • mc->sesscache->store (プロバイダー固有の実装)
  • ssl_mutex_on / ssl_mutex_off (排他制御)
  • ap_log_error (エラーログ出力)

関連データ構造

  • SSLModConfigRec: モジュールの設定情報を保持し、キャッシュプロバイダー (sesscache) へのアクセスを提供します。
  • SSL_SESSION: OpenSSL のセッション構造体。

失敗モードと境界条件

  • **サイズ超過**: セッションデータが MODSSL_SESSION_MAX_DER を超える場合、保存は拒否されます。
  • **プロバイダーエラー**: mc->sesscache->store が APR_SUCCESS 以外を返した場合、関数は FALSE を返します。

保守担当者向けメモ

  • 本関数は抽象化レイヤーであり、実際の保存先（共有メモリ、ディスク等）のロジックは socache プロバイダーに依存します。
  • 排他制御が必要なプロバイダーを使用する場合、ssl_mutex の状態が正しく管理されていることを確認してください。
