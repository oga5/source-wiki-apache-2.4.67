ap_ssl_has_outgoing_handlers

定義

  • 種別: function
  • 定義箇所: server/ssl.c:164

目的

アウトバウンド接続（プロキシ等）に対して SSL を適用可能なハンドラや設定が存在するかを判定する。

シグネチャまたは形状

int ap_ssl_has_outgoing_handlers(void)

入力と出力

  • 入力: なし
  • 出力: int 型。SSL アウトバウンドハンドラが登録されている、または設定されている場合は非ゼロ（真）、それ以外は 0（偽）を返す。

前提条件と不変条件

  • ap_hook_get_ssl_bind_outgoing() を介して取得されるフックリストが有効であること。
  • module_ssl_engine_set および module_ssl_proxy_enable は、本関数が参照する外部のグローバル変数または静的変数である。

アルゴリズムまたは振る舞い

1. ap_hook_get_ssl_bind_outgoing() を呼び出し、登録されている SSL バインド用フックのリストを取得する。 2. 以下のいずれかの条件を満たす場合に真を返す：

  • フックリストが存在し、かつ登録されたフックの数（nelts）が 0 より大きい。
  • module_ssl_engine_set が設定されている。
  • module_ssl_proxy_enable が設定されている。

    • 3. 上記のいずれも満たさない場合は偽を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/proxy/mod_proxy.c
  • modules/proxy/mod_proxy_hcheck.c
  • modules/proxy/mod_proxy_http.c
  • 呼び出し先:
  • ap_hook_get_ssl_bind_outgoing (Apache フック機構)

関連データ構造

  • apr_array_header_t: フックリストを保持するために使用される APR 配列構造体。

失敗モードと境界条件

  • フックリストが NULL の場合、フックの存在チェックはスキップされ、他の変数（module_ssl_*）の評価のみが行われる。

保守担当者向けメモ

  • 本関数は、プロキシ機能などが SSL を利用可能かどうかを判断するためのゲートキーパーとして機能する。
  • module_ssl_engine_set および module_ssl_proxy_enable の定義元については、本ソース抜粋からは直接確認できない。
