check_modules

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2.c:70

目的

現在ロードされている Apache MPM モジュールを特定し、HTTP/2 のワーカーモデル（c2）がその MPM でサポートされているかを判定するための初期化を行う。

シグネチャまたは形状

static void check_modules(int force)

入力と出力

  • **入力**:
  • force (int): 1 の場合、既に実行済みであっても強制的に再スキャンを行う。
  • **出力**: なし（副作用としてグローバル変数 mpm_module および mpm_supported を更新）。

前提条件と不変条件

  • ap_loaded_modules が有効なモジュールリストを保持していること。
  • checked 静的変数が、一度実行された後に再実行を抑制するフラグとして機能する。

アルゴリズムまたは振る舞い

1. force が真、または初回実行時のみ処理を実行する。 2. ap_loaded_modules を走査し、モジュール名（m->name）を比較する。 3. 以下の MPM を検出した場合、mpm_module に設定する。

  • event.c, motorz.c, mpm_netware.c, mpm_winnt.c, worker.c: サポート対象としてマーク。
  • prefork.c, simple_api.c: mpm_supported を 0 に設定し、HTTP/2 ワーカーモデルを無効化する。

    • 4. 走査完了後、checked を 1 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2.c 内の関数（h2_mpm_supported 等）から、引数 0 または 1 で呼び出される。
  • **呼び出し先**:
  • strcmp (標準ライブラリ)

関連データ構造

  • module (Apache HTTP Server 構造体)
  • ap_loaded_modules (ロード済みモジュール配列)

失敗モードと境界条件

  • ap_loaded_modules が空の場合、ループは即座に終了する。
  • 既知の MPM 名に一致しないモジュールは無視される。
  • prefork.c の場合、HTTP/2 は動作可能だが、単一スレッドアプリケーション（PHP等）との競合を避けるため、意図的に mpm_supported が無効化される。

保守担当者向けメモ

  • 新しい MPM が導入された場合、このリストに名前を追加する必要がある。
  • mpm_supported の判定ロジックは、HTTP/2 の多重化モデルが特定の MPM で適切に動作するかどうかに依存しているため、変更時は注意が必要。
