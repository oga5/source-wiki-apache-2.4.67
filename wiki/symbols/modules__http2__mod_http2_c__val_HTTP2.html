val_HTTP2

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_http2.c:210

目的

現在の接続が HTTP/2 セッションとして確立されているかどうかを示す文字列（"on" または "off"）を返します。

シグネチャまたは形状

static const char *val_HTTP2(apr_pool_t *p, server_rec *s,
                             conn_rec *c, request_rec *r, h2_conn_ctx_t *ctx)

入力と出力

  • **入力**:
  • p: メモリプール（未使用）。
  • s: サーバーレコード（未使用）。
  • c: 接続レコード（未使用）。
  • r: リクエストレコード（未使用）。
  • ctx: 接続に関連付けられた h2_conn_ctx_t 構造体。
  • **出力**:
  • ctx が非 NULL であれば "on" を、NULL であれば "off" を返します。

前提条件と不変条件

  • ctx は、HTTP/2 接続コンテキストが存在する場合に有効なポインタとなります。
  • 戻り値の文字列は静的領域の定数であり、呼び出し側で解放する必要はありません。

アルゴリズムまたは振る舞い

1. 引数として渡された h2_conn_ctx_t *ctx の真偽を判定します。 2. ctx が存在する場合（HTTP/2 接続として認識されている場合）、"on" を返します。 3. ctx が存在しない場合、"off" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c:300 にある h2_var_def 配列内で、環境変数 "HTTP2" の値を取得するためのハンドラとして登録されています。
  • **呼び出し先**: なし。

関連データ構造

  • h2_conn_ctx_t: HTTP/2 接続のメタデータを保持する構造体。本関数はこの構造体の有無を判定基準として使用します。

失敗モードと境界条件

  • ctx が NULL の場合でも安全に "off" を返すよう設計されており、特別なエラー処理は不要です。

保守担当者向けメモ

  • 本関数は http2_var_lookup を通じて環境変数として公開される値の生成に使用されます。
  • h2_conn_ctx_t の構造に変更があった場合、本関数の判定ロジックが適切か再確認してください。
