ap_http_chunk_filter

定義

  • 種別: function
  • 定義箇所: modules/http/chunk_filter.c:47

目的

HTTP/1.1 のチャンク転送エンコーディングを実装する出力フィルタです。レスポンスボディをチャンク形式に変換し、下位フィルタへ渡します。

シグネチャまたは形状

apr_status_t ap_http_chunk_filter(ap_filter_t *f, apr_bucket_brigade *b)

入力と出力

  • **入力**:
  • f: フィルタコンテキストを含むフィルタ構造体。
  • b: 処理対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • chunk_hdr はスタック上に確保され、transient バケットとして使用されるため、関数終了前にデータがコピーされる必要があります。
  • HTTP_BAD_GATEWAY エラーが発生した場合、f->ctx に bad_gateway_seen が設定され、チャンク終了処理が抑制されます。

アルゴリズムまたは振る舞い

1. ブリゲード内のバケットを走査し、データ長を計算します。 2. EOS（ストリーム終了）バケットを検出した場合、フィルタを解除します。 3. HTTP_BAD_GATEWAY エラーバケットを検出した場合、コンテキストに記録し、終了処理をスキップする準備をします。 4. データが存在する場合、16進数のサイズヘッダーと CRLF を付与してチャンク化します。 5. EOS 到達時、エラーが発生していなければ最終チャンクマーカー（0\r\n\r\n）を挿入します。 6. 加工済みのブリゲードを ap_pass_brigade で次のフィルタへ渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_core.c にて ap_register_output_filter("CHUNK", ...) として登録されており、フィルタチェーンを通じて呼び出されます。
  • **呼び出し先**:
  • ap_pass_brigade (次のフィルタへデータを渡す)
  • apr_bucket_read (データ長不明時の読み込み)
  • ap_remove_output_filter (EOS 到達時)

関連データ構造

  • ap_filter_t: フィルタチェーンの管理構造体。
  • apr_bucket_brigade: バケットの連結リスト。
  • bad_gateway_seen: バックエンドエラー状態を保持する静的変数。

失敗モードと境界条件

  • **バックエンドエラー**: HTTP_BAD_GATEWAY が発生すると、最終チャンクマーカーの付与をスキップし、接続を閉じる挙動をとります。
  • **データ長不明**: パイプ等でデータ長が不明な場合、apr_bucket_read を試行し、読み込み結果に応じて処理を継続します。
  • **メモリ不足**: apr_bucket_transient_create 等のバケット生成が失敗した場合、その戻り値が反映されます。

保守担当者向けメモ

  • chunk_hdr はスタック変数であるため、スコープ外への参照が発生しないよう注意してください。
  • HTTP_BAD_GATEWAY 発生時の終了処理スキップは、クライアントが壊れたレスポンスを正常終了と誤認するのを防ぐ重要なロジックです。
