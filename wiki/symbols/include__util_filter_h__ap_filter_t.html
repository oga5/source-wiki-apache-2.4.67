ap_filter_t

定義

  • 種別: struct
  • 定義箇所: include/util_filter.h:94

目的

Apache HTTP Server のフィルタチェーンにおける、インストールされたフィルタの個別のインスタンスを表現する構造体です。

シグネチャまたは形状

typedef struct ap_filter_t ap_filter_t;

※構造体の内部メンバ定義は提供されたソース抜粋には含まれていません。

入力と出力

  • 入力: フィルタチェーンを通過する apr_bucket_brigade データ。
  • 出力: 処理済み、または次段へ渡される apr_bucket_brigade データ。

    • ※具体的なメンバ構成が不明なため、詳細な入出力インターフェースは未定義です。

前提条件と不変条件

  • フィルタは ap_filter_type に基づいてチェーン内でソートされます。
  • AP_FTYPE_CONNECTION 以上のフィルタはサブリクエスト内では使用できません。

アルゴリズムまたは振る舞い

  • フィルタチェーンの一部として、ap_pass_brigade や ap_get_brigade を介してデータを順次処理します。
  • フィルタは ctx ポインタ（構造体メンバと推測される）を使用して状態を保持し、必要に応じて ap_save_brigade を用いてデータを一時保存します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • ap_core_input_filter (include/http_core.h)
  • ap_core_output_filter (include/http_core.h)
  • ap_byterange_filter, ap_http_header_filter, ap_content_length_filter, ap_old_write_filter (include/http_protocol.h)
  • ap_sub_req_output_filter (include/http_request.h)
  • 呼び出し先: 提供されたソース抜粋には、本構造体が直接呼び出す関数やデータ構造の情報は含まれていません。

関連データ構造

  • ap_filter_rec_t: フィルタの定義情報（名前、関数ポインタ、型）。
  • apr_bucket_brigade: フィルタ間で受け渡されるデータストリーム。
  • request_rec, conn_rec: フィルタが関連付けられる Apache コア構造体。

失敗モードと境界条件

  • ストリーム終了マーカー到達時に、各フィルタは内部状態をフラッシュしリソースを解放する必要があります。
  • フィルタチェーンの順序制御やエラー発生時の挙動については、server/util_filter.c の実装に依存します。

保守担当者向けメモ

  • 本構造体は不透明ポインタとして扱われることが多いため、詳細なメンバへのアクセスは include/util_filter.h の完全な定義を参照してください。
  • ap_pass_brigade に渡されるブリゲードの所有権は呼び出し元にあるため、フィルタ内でデータを保持する場合は ap_save_brigade を使用してください。
