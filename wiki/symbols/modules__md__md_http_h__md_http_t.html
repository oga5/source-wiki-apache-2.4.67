md_http_t

定義

  • **種別**: struct
  • **定義箇所**: modules/md/md_http.h:25

目的

mod_md モジュールにおける HTTP 通信のセッションを管理するための抽象化されたクライアントインスタンスです。

シグネチャまたは形状

typedef struct md_http_t md_http_t;

※不透明な構造体（opaque struct）として定義されており、詳細はヘッダーファイル内では公開されていません。

入力と出力

  • **入力**: md_http_create 等の生成関数を通じて初期化されます。
  • **出力**: 通信設定（タイムアウト、プロキシ、CA 証明書等）およびバックエンド実装へのポインタを保持します。

前提条件と不変条件

  • md_http_create を呼び出す前に、md_http_use_implementation を使用して通信バックエンド（md_http_impl_t）が登録されている必要があります。
  • 本構造体は、md_http_perform 等の実行関数を通じて通信のフロントエンドとして機能します。

アルゴリズムまたは振る舞い

  • HTTP リクエスト（md_http_request_t）の実行環境を提供します。
  • 内部的に保持する実装ポインタを介して、libcurl 等の具体的な通信ライブラリへ処理を委譲します。
  • タイムアウト設定やストール検出などの通信制御を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme.h で ACME プロトコル処理の一部として参照されています。
  • **呼び出し先**: modules/md/md_curl.c において、md_curl_multi_perform 等の関数が本構造体を引数として受け取り、内部の通信状態を操作します。

関連データ構造

  • md_http_impl_t: 通信バックエンドの関数テーブル。
  • md_http_request_t: 本セッションを使用して発行される個別の HTTP リクエスト。
  • md_http_response_t: 通信結果として返されるレスポンスデータ。
  • md_http_timeouts_t: 通信のタイムアウト設定。

失敗モードと境界条件

  • 実装が登録されていない状態でセッションを作成・使用した場合の挙動は未定義です。
  • md_http_request_t 内部の internals ポインタは実装固有のデータ用であり、本構造体自体はそれらの詳細を管理しません。

保守担当者向けメモ

  • 本構造体は md_http_impl_t を通じたプラグイン可能な設計となっているため、直接のメンバアクセスは避け、提供されている API を介して操作してください。
  • md_curl.c 内の http_spares 配列などで、複数の md_http_t インスタンスが管理されるケースがあります。
