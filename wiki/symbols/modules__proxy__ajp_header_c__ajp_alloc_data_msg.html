ajp_alloc_data_msg

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_header.c:880

目的

AJP プロトコルにおけるデータ送信用のメッセージバッファを確保し、データ書き込み用のポインタと有効長を初期化します。

シグネチャまたは形状

apr_status_t ajp_alloc_data_msg(apr_pool_t *pool, char **ptr, apr_size_t *len, ajp_msg_t **msg)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • ptr: データ書き込み開始位置を格納するポインタのポインタ。
  • len: 入力時は要求するデータ長、出力時はヘッダー分（6バイト）を差し引いた有効なデータ長。
  • msg: 生成された ajp_msg_t 構造体を格納するポインタ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は ajp_msg_create のエラーコード。

前提条件と不変条件

  • ajp_msg_create が成功すること。
  • *len は AJP メッセージのヘッダーサイズ（6バイト）以上の値であること。

アルゴリズムまたは振る舞い

1. ajp_msg_create を呼び出し、指定されたサイズでメッセージバッファを確保します。 2. ajp_msg_reset を呼び出し、メッセージバッファを初期化します。 3. *ptr に、AJP メッセージヘッダー（6バイト）の直後のバッファ位置を設定します。 4. *len からヘッダー分の6バイトを減算し、書き込み可能なデータ長に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ajp.c (263行目)
  • **呼び出し先**:
  • ajp_msg_create (内部呼び出し)
  • ajp_msg_reset (内部呼び出し)

関連データ構造

  • ajp_msg_t: AJP 通信用のメッセージバッファ構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • ajp_msg_create がメモリ不足等で失敗した場合、そのエラーコードをそのまま返します。
  • *len が6未満の場合、計算結果が負の値になる可能性があります（呼び出し側のバリデーションに依存）。

保守担当者向けメモ

  • 本関数は AJP データパケットの構築時に、ヘッダー領域をスキップしてデータ本体の書き込み位置を特定するために使用されます。
  • ajp_msg_t の内部構造（特にヘッダーサイズ6バイト）に強く依存しているため、プロトコル仕様変更時には注意が必要です。
