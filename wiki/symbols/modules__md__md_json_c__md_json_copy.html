md_json_copy

定義

  • 種別: function
  • 定義箇所: modules/md/md_json.c:111

目的

既存の md_json_t オブジェクトの深いコピー（ディープコピー）を作成し、指定された APR プールの管理下で新しい md_json_t インスタンスとして返します。

シグネチャまたは形状

md_json_t *md_json_copy(apr_pool_t *pool, const md_json_t *json);

入力と出力

  • **入力**:
  • pool: 新しい md_json_t インスタンスのメモリ管理に使用する APR プール。
  • json: コピー元の md_json_t オブジェクト。
  • **出力**:
  • コピーされた内容を持つ新しい md_json_t へのポインタ。

前提条件と不変条件

  • json 引数は有効な md_json_t 構造体である必要があります。
  • 内部の json_t オブジェクトは json_copy によって複製されます。

アルゴリズムまたは振る舞い

1. json->j（内部の json_t）に対して json_copy を実行し、JSON ツリーの深いコピーを作成します。 2. json_create を呼び出し、指定された pool と複製された JSON ツリーを紐付けて新しい md_json_t を生成します。 3. 生成されたインスタンスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_json.h:50 で宣言されています。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**:
  • json_copy (Jansson ライブラリ)
  • json_create (modules/md/md_json.c 内の内部関数)

関連データ構造

  • md_json_t: json_t をラップし、APR プールとのライフサイクル管理を行う構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • メモリ不足時には json_create または json_copy が失敗し、NULL を返す可能性があります。

保守担当者向けメモ

  • 本関数は深いコピーを行うため、元のオブジェクトと新しいオブジェクトは独立したメモリ領域を持ちます。
  • md_json_clone との使い分けに注意してください（本関数は json_copy を使用します）。
