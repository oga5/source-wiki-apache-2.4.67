socache_dbm_expire

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:443

目的

DBM ファイル内に保存されたキャッシュエントリを走査し、期限切れ（expiry）時刻を過ぎたエントリを削除してストレージを解放する。

シグネチャまたは形状

static void socache_dbm_expire(ap_socache_instance_t *ctx, server_rec *s)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンスのコンテキスト（データファイルパス、プール、期限切れ間隔を保持）。
  • s: ログ出力に使用するサーバーレコード。
  • **出力**: なし（副作用として DBM ファイルからエントリを削除）。

前提条件と不変条件

  • ctx->last_expiry と ctx->expiry_interval に基づき、前回の実行から一定時間が経過していない場合は即座に終了する。
  • DBM ライブラリの制約を考慮し、スキャンと削除を分離する2パス方式を採用している。
  • 本関数はスレッドセーフではなく、呼び出し側での排他制御が前提となる。

アルゴリズムまたは振る舞い

1. **インターバルチェック**: 現在時刻が last_expiry + expiry_interval を超えていない場合は処理を中断。 2. **パス1（スキャン）**: DBM ファイルを読み取り専用（または RW）で開き、全キーを走査。値の先頭に格納された apr_time_t を現在時刻と比較し、期限切れのキーをメモリ上の keylist に収集する。 3. **パス2（削除）**: DBM ファイルを再度開き、keylist に格納されたキーを apr_dbm_delete で削除する。 4. **ループ**: KEYMAX（1024）を超えるエントリがある場合、全エントリが処理されるまで上記パスを繰り返す。 5. **ログ**: 処理前後のエントリ数と削除数をデバッグログに出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_socache_dbm.c 内の以下の箇所から参照されていることが確認されている:
  • 73行目（プロトタイプ宣言）
  • 194行目
  • 299行目
  • 322行目
  • **呼び出し先**:
  • apr_dbm_* 系関数（apr_dbm_open, apr_dbm_firstkey, apr_dbm_fetch, apr_dbm_delete, apr_dbm_close 等）
  • apr_pool_clear, apr_palloc, apr_pmemdup（メモリ管理）
  • ap_log_error（ログ出力）

関連データ構造

  • ap_socache_instance_t: キャッシュのメタデータと設定を保持。
  • apr_dbm_t: DBM ファイルのハンドル。
  • apr_datum_t: DBM のキーおよび値を保持する構造体。

失敗モードと境界条件

  • **DBM オープン失敗**: apr_dbm_open または apr_dbm_open2 が失敗した場合、エラーログを出力して処理を中断する。
  • **メモリ不足**: apr_palloc が失敗した場合、処理を中断する。
  • **データ破損**: 値のサイズが sizeof(apr_time_t) 以下の場合、安全のため期限切れとみなして削除する。
  • **同時実行**: DBM ライブラリの制約により、スキャン中に他プロセスがファイルを変更した場合の挙動はライブラリの実装に依存する。

保守担当者向けメモ

  • 本関数は AP_SOCACHE_FLAG_NOTMPSAFE を指定しているモジュールの一部であり、呼び出し側で適切なロックが確保されていることを確認すること。
  • KEYMAX は 1024 に固定されている。大規模なキャッシュ環境で期限切れエントリが一度に大量発生する場合、この制限によるループ回数の増加がパフォーマンスに影響する可能性がある。
