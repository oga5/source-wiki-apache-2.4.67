h2_c1_pre_close

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1.c:206

目的

HTTP/2 接続の終了直前におけるクリーンアップ処理を実行し、セッションに対して終了準備（GOAWAY 送信等）を指示する。

シグネチャまたは形状

apr_status_t h2_c1_pre_close(struct h2_conn_ctx_t *ctx, conn_rec *c)

入力と出力

  • **入力**:
  • ctx: 接続に関連付けられた HTTP/2 コンテキスト構造体。
  • c: Apache の接続レコード（conn_rec）。
  • **出力**:
  • apr_status_t: 処理結果。成功時は DONE を返し、それ以外の場合はエラーコードを返す。

前提条件と不変条件

  • c から取得される h2_conn_ctx_t が有効であること。
  • セッションが存在する場合、h2_session_pre_close が呼び出される。

アルゴリズムまたは振る舞い

1. h2_conn_ctx_get(c) を使用して、接続から HTTP/2 コンテキストを取得する。 2. コンテキストおよびその内部の session が存在するかを確認する。 3. セッションが存在する場合、h2_session_pre_close(conn_ctx->session, async_mpm) を呼び出し、セッション終了処理を行う。 4. h2_session_pre_close の戻り値が APR_SUCCESS であれば DONE を返し、そうでなければそのステータスを返す。 5. セッションが存在しない場合は、そのまま DONE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c:330 (直接の呼び出しを確認)
  • **呼び出し先**:
  • h2_conn_ctx_get
  • h2_session_pre_close

関連データ構造

  • h2_conn_ctx_t: HTTP/2 接続ごとのメタデータ管理構造体。
  • conn_rec: Apache HTTP Server の接続レコード。
  • h2_session: HTTP/2 セッションの状態管理オブジェクト。

失敗モードと境界条件

  • conn_ctx が NULL の場合、あるいは conn_ctx->session が NULL の場合は、何もせず DONE を返す。
  • h2_session_pre_close がエラーを返した場合、そのステータスコードがそのまま呼び出し元に伝播する。

保守担当者向けメモ

  • 本関数は接続終了フックの一部として機能する。
  • async_mpm グローバル変数が h2_session_pre_close の引数として使用されている点に注意すること。
