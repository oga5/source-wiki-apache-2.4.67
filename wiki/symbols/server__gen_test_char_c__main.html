定義

  • 定義箇所: server/gen_test_char.c:59

目的

Apache HTTP Server の実行時に使用される文字種判定用ルックアップテーブル（test_char_table）および関連マクロ定義を生成し、標準出力へ書き出すためのビルド時ツール。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**: なし（コンパイル時に実行されるスタンドアロンプログラム）。
  • **出力**: 標準出力へ以下の内容を書き出す。
  • 各種フラグビットの #define 定義。
  • 256 要素の unsigned short 型静的配列 test_char_table の定義。
  • 文字列処理用マクロ TEST_CHAR(c, f) の定義。

前提条件と不変条件

  • 0 から 255 までの全 ASCII 文字コードに対して、各フラグの適合性を判定する。
  • 生成されたテーブルを利用する側では、インデックスアクセス時に必ず unsigned char へのキャストを行う必要がある（符号拡張による不正なインデックス参照を防ぐため）。

アルゴリズムまたは振る舞い

1. 各種エスケープ処理やトークン判定用のビットフラグ定数を標準出力に書き出す。 2. 0 から 255 までのループを実行し、各文字 c に対して以下の判定を行う。

  • apr_isalnum や strchr 等を用いて、シェルエスケープ、パスセグメント、URL エンコード、HTTP トークン、制御文字等の各フラグをビット単位で累積する。

    • 3. 判定結果を 16 進数形式で test_char_table 配列の要素として出力する。 4. 最後に、テーブルを安全に参照するための TEST_CHAR マクロ定義を出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ビルドシステム（Apache HTTP Server のビルドプロセス中に実行される）。
  • **呼び出し先**: apr_isalnum, apr_iscntrl, apr_isprint, strchr（いずれも標準ライブラリまたは APR ライブラリ）。
  • 注: ツールによる静的解析では、本関数を呼び出すソースコード上の直接的な参照関係は特定されていません。

関連データ構造

  • test_char_table: 256 要素の unsigned short 型配列。各インデックスが文字コードに対応し、ビットマスクが格納される。

失敗モードと境界条件

  • char 型が符号付きの場合、TEST_CHAR マクロで unsigned char にキャストせずに使用すると、負の値で配列外参照が発生する可能性がある。

保守担当者向けメモ

  • 新しいエスケープルールや文字種判定を追加する場合は、ループ内の判定ロジックを更新し、対応するフラグビットを定義すること。
  • NEED_ENHANCED_ESCAPES マクロの定義有無により、シェルエスケープの判定ロジックが分岐する。
