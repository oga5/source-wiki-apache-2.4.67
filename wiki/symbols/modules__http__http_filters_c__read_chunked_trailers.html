read_chunked_trailers

定義

  • 種別: function
  • 定義箇所: modules/http/http_filters.c:293

目的

HTTP チャンク転送の終了後に送信されるトレーラヘッダを読み取り、リクエストヘッダ（r->headers_in）へ統合します。

シグネチャまたは形状

static apr_status_t read_chunked_trailers(http_ctx_t *ctx, ap_filter_t *f,
                                          apr_bucket_brigade *b, int merge)

入力と出力

  • **入力**:
  • ctx: 入力フィルタの状態管理構造体。
  • f: 現在のフィルタコンテキスト。
  • b: バケットブリゲード。
  • merge: トレーラを既存のヘッダとマージするかを指定するフラグ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は APR_EINVAL を返します。
  • b: 成功時に EOS（End of Stream）バケットが挿入されます。

前提条件と不変条件

  • f->r（リクエストレコード）が有効であること。
  • r->trailers_in が初期化されていること。

アルゴリズムまたは振る舞い

1. 現在の r->headers_in を一時保存し、r->headers_in を r->trailers_in に差し替えます。 2. ap_get_mime_headers(r) を呼び出し、トレーラヘッダを解析・読み込みます。 3. 読み込み成功時、r->status を復元し、EOS バケットをブリゲード b に挿入して ctx->eos_sent を 1 に設定します。 4. 読み込み失敗時、エラーログを出力し APR_EINVAL を返します。 5. merge 引数が真の場合、保存していた元のヘッダと読み込んだトレーラを apr_table_overlay で結合し、r->headers_in に戻します。偽の場合は元のヘッダを復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_filters.c:569（ap_http_filter 内から呼び出される）。
  • **呼び出し先**:
  • ap_get_mime_headers: ヘッダ解析の実行。
  • apr_bucket_eos_create: EOS バケットの生成。
  • APR_BRIGADE_INSERT_TAIL: ブリゲードへのバケット挿入。
  • ap_log_rerror: エラーログ出力。
  • apr_table_overlay: ヘッダテーブルの結合。

関連データ構造

  • http_ctx_t: フィルタの状態管理。
  • request_rec: HTTP リクエスト情報。
  • apr_bucket_brigade: ストリームデータ保持用。

失敗モードと境界条件

  • ap_get_mime_headers が HTTP_OK 以外を返した場合、エラーとして処理され APR_EINVAL が返されます。
  • merge フラグの指定により、ヘッダの保持状態が分岐します。

保守担当者向けメモ

  • 本関数はチャンク転送の終端処理に特化しています。トレーラヘッダの解析は ap_get_mime_headers に依存しているため、解析ルールの変更はそちらを確認してください。
  • r->headers_in の差し替えと復元を行っているため、メモリ管理（r->pool）の整合性に注意してください。
