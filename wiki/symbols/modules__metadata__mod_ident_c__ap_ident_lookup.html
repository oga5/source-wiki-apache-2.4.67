ap_ident_lookup

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_ident.c:303

目的

RFC 1413 (IDENT プロトコル) を使用して、リモートホストの ident デーモンからユーザー名を照会し、接続情報 (conn_rec) に格納する。

シグネチャまたは形状

const char *ap_ident_lookup(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: 取得したユーザー名、または取得失敗・無効時の "unknown" (定数 FROM_UNKNOWN) を表す文字列ポインタ。

前提条件と不変条件

  • r->connection が有効であること。
  • ident_module の設定が r->per_dir_config に存在すること。
  • 取得されたユーザー名は conn->pool に割り当てられるため、リクエスト終了まで生存する。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリに対する ident_config_rec を取得する。 2. 設定で IDENT 照会が無効化されている場合、即座に NULL を返す。 3. rfc1413_connect を呼び出し、リモートホストの 113 番ポートへの接続を試みる。 4. 接続成功時、rfc1413_query を呼び出してクエリを送信し、レスポンスを解析して conn->remote_logname を更新する。 5. 通信完了後、ソケットを閉じる。 6. 接続またはクエリが失敗した場合、conn->remote_logname に FROM_UNKNOWN を設定する。 7. 最終的に conn->remote_logname を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core.c (オプション関数として取得・利用)
  • **呼び出し先**:
  • rfc1413_connect (接続確立)
  • rfc1413_query (クエリ送信と解析)
  • apr_socket_close (ソケット解放)
  • ap_get_module_config (設定取得)

関連データ構造

  • request_rec: リクエストのコンテキスト。
  • ident_config_rec: モジュール固有の設定（有効フラグ、タイムアウト値）。
  • conn_rec: 接続情報（remote_logname を保持）。

失敗モードと境界条件

  • **タイムアウト**: rfc1413_connect が指定時間内に応答を得られない場合、失敗として扱われる。
  • **無効な設定**: IdentityCheck がオフの場合、処理はスキップされる。
  • **通信エラー**: 接続拒否やプロトコル違反が発生した場合、remote_logname は "unknown" となる。

保守担当者向けメモ

  • 本関数はオプション関数として公開されており、APR_REGISTER_OPTIONAL_FN を介して他モジュールから利用可能である。
  • ネットワーク I/O を伴うため、高負荷環境ではパフォーマンスに悪影響を及ぼす可能性がある。
  • conn->remote_logname のメモリ管理は conn->pool に依存している。
