c1c2_stream_joined

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_mplx.c:141

目的

HTTP/2 ストリームの処理が完了した際、当該ストリームを「保持（hold）」状態から「破棄（purge）」待ち状態へ移行させるために使用されます。

シグネチャまたは形状

static void c1c2_stream_joined(h2_mplx *m, h2_stream *stream)

入力と出力

  • **入力**:
  • m: 多重化器（h2_mplx）へのポインタ。
  • stream: 処理対象のストリーム（h2_stream）へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • stream_is_running(stream) が偽であること（実行中ではないこと）を ap_assert で検証します。

アルゴリズムまたは振る舞い

1. m->shold（保持ハッシュテーブル）から、指定された stream->id を削除します。 2. add_for_purge(m, stream) を呼び出し、当該ストリームを破棄待ちリストへ追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_mplx.c:1031 からの呼び出しが確認されています。
  • **呼び出し先**:
  • stream_is_running
  • h2_ihash_remove
  • add_for_purge

関連データ構造

  • h2_mplx: 多重化器のメイン構造体。
  • h2_stream: ストリームのコンテキスト。
  • h2_ihash: ストリームIDをキーとするハッシュテーブル（m->shold）。

失敗モードと境界条件

  • stream_is_running のアサーションに失敗した場合、プログラムは異常終了します。

保守担当者向けメモ

  • 本関数は h2_mplx 内のストリーム状態遷移を管理するクリーンアップ処理の一部です。
  • m->shold からの削除と add_for_purge への追加は、ストリームのライフサイクルにおける重要な同期点です。
