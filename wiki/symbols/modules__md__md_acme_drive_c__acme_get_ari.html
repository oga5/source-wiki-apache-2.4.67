acme_get_ari

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:1097

目的

ACME サーバーから提供される「ACME Renewal Information (ARI)」を取得し、証明書の推奨更新期間を算出する。

シグネチャまたは形状

apr_status_t acme_get_ari(md_proto_driver_t *d,
                          struct md_result_t *result,
                          apr_time_t *prenew_at,
                          const char **purl)

入力と出力

  • **入力**:
  • d: 管理対象ドメインのドライバー構造体。
  • result: 処理結果を記録するコンテナ。
  • **出力**:
  • prenew_at: 算出された推奨更新開始時刻（成功時）。
  • purl: CA から提供された説明用 URL（成功時）。
  • 戻り値: APR_SUCCESS またはエラーコード。

前提条件と不変条件

  • d->md に有効なドメイン設定が存在すること。
  • 静的ファイルで構成された証明書ではないこと。
  • CA が ACMEv2 プロトコルをサポートし、かつ ARI 拡張機能を提供していること。

アルゴリズムまたは振る舞い

1. **初期チェック**: 静的ファイル設定や CA URL の欠如を確認し、該当する場合は終了。 2. **証明書収集**: レジストリから現在有効な公開証明書を収集。 3. **通信準備**: md_acme_create および md_acme_setup を呼び出し、ACME 通信セッションを確立。 4. **ARI 取得**: 各証明書に対して ari_cert_id を生成し、CA の ARI エンドポイントへ JSON を要求。 5. **期間算出**: 取得した suggestedWindow（start/end）を解析。 6. **ランダム化**: 取得した期間内でランダムな更新時刻を生成し、全証明書の中で最も早い時刻を prenew_at に設定。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_drive.c:1245（ソースコード内の直接的な参照）。
  • **呼び出し先**: md_acme_create, md_acme_setup, md_acme_get_json, md_cert_get_ari_cert_id 等の md_acme および md_crypt 関連関数。

関連データ構造

  • md_proto_driver_t: ドライバーのコンテキスト。
  • md_cert_t: 証明書情報。
  • md_json_t: CA から返される ARI JSON データ。

失敗モードと境界条件

  • **ARI 非対応**: CA が ARI をサポートしていない場合、エラーではなく APR_SUCCESS を返し、更新時刻は 0 となる。
  • **解析エラー**: JSON の形式不正や時刻フォーマットの不整合が発生した場合、その証明書はスキップされる。
  • **通信失敗**: ネットワークエラーや CA 側の応答エラー時は、ログに警告を出力し処理を継続または終了する。

保守担当者向けメモ

  • 本関数は CA 側の実装状況に強く依存する。
  • start > end となるような不正な期間が CA から返された場合は警告を出力して無視する設計となっている。
  • 算出された更新時刻は、start と end の間でランダムに分散されるよう md_rand_bytes を使用している。
