log_out

定義

  • 種別: function
  • 定義箇所: test/clients/h2ws.c:65

目的

ログメッセージを標準エラー出力（stderr）に整形して出力する。タイムスタンプ、ログレベル、発生箇所、メッセージを付与する。

シグネチャまたは形状

static void log_out(const char *level, const char *where, const char *msg)

入力と出力

  • **入力**:
  • level: ログレベルを示す文字列（例: "ERROR", "INFO", "DEBUG"）。
  • where: ログ出力元を示す文字列。
  • msg: 出力するメッセージ内容。
  • **出力**: なし（void）。stderr へフォーマット済み文字列を書き出す。

前提条件と不変条件

  • level, where, msg は有効なヌル終端文字列であること。
  • clock_gettime が CLOCK_REALTIME をサポートしている環境であること。

アルゴリズムまたは振る舞い

1. clock_gettime を使用して現在のシステム時刻を取得する。 2. localtime_r を使用して時刻をローカルタイムに変換する。 3. strftime を使用して時刻を HH:MM:SS 形式の文字列に変換し、timebuf に格納する。 4. fprintf を使用し、[HH:MM:SS.nanoseconds][level][where] message の形式で stderr に出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • test/clients/h2ws.c:79 (ERRORログ)
  • test/clients/h2ws.c:85 (INFOログ)
  • test/clients/h2ws.c:91 (DEBUGログ)
  • **呼び出し先**:
  • clock_gettime
  • localtime_r
  • strftime
  • fprintf

関連データ構造

  • struct timespec: 時刻取得に使用。
  • struct tm: 時刻の分解に使用。

失敗モードと境界条件

  • timebuf のサイズ（128バイト）を超える時刻文字列は切り詰められる可能性がある。
  • stderr への書き込みが失敗した場合の明示的なエラーハンドリングは行われない。

保守担当者向けメモ

  • 本関数は静的関数であり、h2ws.c 内のログ出力ユーティリティとして機能する。
  • strftime のバッファサイズは固定（128）であるため、将来的にフォーマットを変更する場合は注意が必要。
