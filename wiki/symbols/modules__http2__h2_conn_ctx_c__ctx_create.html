ctx_create

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_conn_ctx.c:42

目的

Apache HTTP Server の接続（conn_rec）に対して、HTTP/2 固有のコンテキスト構造体（h2_conn_ctx_t）を生成し、初期化およびアタッチを行う。

シグネチャまたは形状

static h2_conn_ctx_t *ctx_create(conn_rec *c, const char *id)

入力と出力

  • **入力**:
  • c: 接続情報を保持する conn_rec 構造体へのポインタ。
  • id: コンテキストを識別するための文字列。
  • **出力**:
  • 生成・初期化された h2_conn_ctx_t 構造体へのポインタ。

前提条件と不変条件

  • c->pool は有効であること。
  • http2_module がロードされており、ap_set_module_config が正常に動作する環境であること。

アルゴリズムまたは振る舞い

1. c->pool を使用して h2_conn_ctx_t 構造体をメモリ確保（apr_pcalloc）する。 2. 渡された id をコンテキストの id フィールドに設定する。 3. c->base_server をコンテキストの server フィールドに格納する。 4. apr_atomic_set32 を使用して started フラグを 1 に設定する。 5. apr_time_now() を使用して started_at タイムスタンプを記録する。 6. ap_set_module_config を呼び出し、http2_module の設定として conn_rec にコンテキストを紐付ける。 7. 生成したコンテキストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_conn_ctx.c:58 (h2_conn_ctx_create_for_c1 内)
  • modules/http2/h2_conn_ctx.c:93 (h2_conn_ctx_init_for_c2 内)
  • **呼び出し先**:
  • apr_pcalloc
  • apr_atomic_set32
  • apr_time_now
  • ap_set_module_config

関連データ構造

  • h2_conn_ctx_t: 本関数が生成する主要な構造体。
  • conn_rec: Apache の接続状態を保持する構造体。
  • http2_module: Apache のモジュール設定識別子。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性がある（APR の挙動に依存）。

保守担当者向けメモ

  • 本関数は static であり、h2_conn_ctx.c 内のコンテキスト初期化関数からのみ利用される。
  • started フィールドはアトミック操作されているため、マルチスレッド環境での状態追跡を意図している。
