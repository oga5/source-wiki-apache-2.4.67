copy_vm_spec

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:398

目的

既存の ap_lua_vm_spec 構造体の内容を、指定されたメモリプール上に複製して新しいインスタンスを作成する。

シグネチャまたは形状

ap_lua_vm_spec* copy_vm_spec(apr_pool_t* pool, ap_lua_vm_spec* spec)

入力と出力

  • **入力**:
  • pool: 複製先のメモリプール (apr_pool_t*)。
  • spec: 複製元の ap_lua_vm_spec 構造体へのポインタ。
  • **出力**:
  • 複製された ap_lua_vm_spec 構造体へのポインタ。

前提条件と不変条件

  • pool は有効な apr_pool_t であること。
  • spec は有効な ap_lua_vm_spec 構造体を指していること。
  • 複製された構造体の scope は強制的に AP_LUA_SCOPE_SERVER に設定される。
  • cb_arg は常に NULL で初期化される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、指定された pool 上に新しい ap_lua_vm_spec 構造体を割り当てる。 2. bytecode_len、cb、codecache を spec からコピーする。 3. bytecode および file 文字列を apr_pstrdup で複製する。 4. package_cpaths および package_paths 配列を apr_array_copy で複製する。 5. pool メンバに引数の pool を設定する。 6. scope を AP_LUA_SCOPE_SERVER に固定し、cb_arg を NULL に設定して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_vmprep.c:461 (server_vm_construct 内)
  • **呼び出し先**:
  • apr_pcalloc
  • apr_pstrdup
  • apr_array_copy

関連データ構造

  • ap_lua_vm_spec: Lua VM の設定および生成パラメータを保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数が例外を発生させる可能性がある。
  • spec 内の各フィールドが NULL の場合、apr_pstrdup や apr_array_copy の挙動は APR の実装に依存する。

保守担当者向けメモ

  • 本関数は AP_LUA_SCOPE_SERVER スコープ専用の複製を意図しているため、他のスコープで利用する際は scope の設定値に注意すること。
