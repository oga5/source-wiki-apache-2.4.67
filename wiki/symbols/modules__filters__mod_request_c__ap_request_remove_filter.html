ap_request_remove_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:315

目的

リクエストの入力フィルタチェーンから、mod_request モジュールに関連する特定のフィルタ（kept_body_filter および keep_body_filter）を検索し、存在する場合はそれらを削除します。

シグネチャまたは形状

void ap_request_remove_filter(request_rec *r)

入力と出力

  • **入力**: request_rec *r （対象となるリクエスト構造体）
  • **出力**: なし（副作用としてフィルタチェーンを直接操作します）

前提条件と不変条件

  • r->input_filters が有効なフィルタチェーンの先頭を指している必要があります。
  • フィルタチェーンの走査中、各フィルタの frec->filter_func.in_func が比較対象となります。

アルゴリズムまたは振る舞い

1. リクエスト r の入力フィルタチェーン (r->input_filters) を先頭から順に走査します。 2. 各フィルタの関数ポインタ (in_func) が kept_body_filter または keep_body_filter と一致するか確認します。 3. 一致した場合、ap_remove_input_filter(f) を呼び出して当該フィルタをチェーンから削除します。 4. チェーンの終端に到達するまでこの処理を繰り返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_form.c （APR_RETRIEVE_OPTIONAL_FN を介して取得）
  • **呼び出し先**:
  • ap_remove_input_filter （Apache HTTP Server API）

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • ap_filter_t: フィルタチェーンを構成する要素。
  • keep_body_filter / kept_body_filter: 本関数が削除対象とする mod_request 内のフィルタ関数。

失敗モードと境界条件

  • フィルタチェーンが空の場合、ループは実行されず何も行われません。
  • 対象のフィルタがチェーン内に存在しない場合、何も削除されず正常に終了します。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN を通じて外部モジュールから利用可能なオプション関数として登録されています。
  • フィルタの削除は ap_remove_input_filter に依存しているため、フィルタチェーンの整合性は Apache のフィルタフレームワークの仕様に従います。
