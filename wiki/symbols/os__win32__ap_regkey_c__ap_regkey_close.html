ap_regkey_close

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:120

目的

ap_regkey_t オブジェクトに関連付けられたレジストリキーハンドルを明示的に解放し、関連する APR プールからのクリーンアップ登録を解除します。

シグネチャまたは形状

apr_status_t ap_regkey_close(ap_regkey_t *regkey)

入力と出力

  • **入力**: regkey - 閉じる対象の ap_regkey_t 構造体へのポインタ。
  • **出力**: apr_status_t 型のステータスコード。成功した場合は APR_SUCCESS を返します。

前提条件と不変条件

  • regkey は有効な ap_regkey_t オブジェクトを指している必要があります。
  • 本関数を呼び出すと、当該オブジェクトのクリーンアップハンドラがプールから削除されるため、二重解放は防止されます。

アルゴリズムまたは振る舞い

1. regkey_cleanup 関数を呼び出し、内部の RegCloseKey を実行してレジストリハンドルを解放します。 2. regkey_cleanup が APR_SUCCESS を返した場合、apr_pool_cleanup_kill を使用して、対象の regkey に関連付けられたクリーンアップハンドラをプールから削除します。 3. 実行結果のステータスコードを返します。

呼び出し元と呼び出し先

  • **呼び出し先**: regkey_cleanup (内部関数), apr_pool_cleanup_kill (APR API)
  • **呼び出し元**:
  • modules/arch/win32/mod_win32.c
  • server/mpm/winnt/nt_eventlog.c
  • server/mpm/winnt/service.c

関連データ構造

  • ap_regkey_t: レジストリキーハンドルと関連付けられた apr_pool_t を保持するコンテナ。

失敗モードと境界条件

  • regkey が NULL の場合の挙動は、内部で呼び出される regkey_cleanup の実装に依存します。
  • regkey_cleanup が失敗した場合、apr_pool_cleanup_kill は実行されず、エラーコードがそのまま返されます。

保守担当者向けメモ

  • 本関数は明示的なクローズ用です。プールが破棄される際にも regkey_cleanup が自動的に実行されるため、リソースリークは発生しない設計となっています。
