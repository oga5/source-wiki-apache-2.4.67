dav_stream

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/repos.c:197

目的

ファイルシステムベースの WebDAV リソースに対する読み書き操作において、ストリームの状態を保持するために使用されます。

シグネチャまたは形状

struct dav_stream {
    apr_pool_t *p;
    apr_file_t *f;
    const char *pathname;
    char *temppath;
    int unlink_on_error;
};

入力と出力

本構造体は、dav_fs_open_stream 等の関数を通じて生成・管理されます。

  • p: 操作に関連付けられたメモリプール。
  • f: 現在開いているファイルハンドル。
  • pathname: 対象ファイルのパス（クローズ時に削除が必要な場合に使用）。
  • temppath: 書き込み用の一時ファイルパス。
  • unlink_on_error: エラー発生時に一時ファイルを削除するかどうかを示すフラグ。

前提条件と不変条件

  • apr_pool_t は、リクエストのライフサイクルと同期している必要があります。
  • 書き込み操作において、temppath が設定されている場合、dav_fs_close_stream でのコミット時にリネーム処理が行われます。

アルゴリズムまたは振る舞い

  • dav_fs_open_stream で初期化され、dav_fs_write_stream や dav_fs_seek_stream でファイル操作が行われます。
  • dav_fs_close_stream が呼び出されると、commit 引数に応じて一時ファイルから本来のパスへのリネーム、または unlink_on_error に基づく一時ファイルの削除が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_dav コア（mod_dav.h で定義されたフック経由）および modules/dav/fs/repos.c 内のストリーム操作関数。
  • **呼び出し先**: APR (Apache Portable Runtime) のファイル I/O 関数群。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_file_t: ファイル I/O の抽象化に使用。

失敗モードと境界条件

  • ファイルのオープン失敗や書き込みエラーが発生した場合、unlink_on_error が真であれば、クローズ処理時に一時ファイルが適切に破棄される必要があります。

保守担当者向けメモ

  • 本構造体は mod_dav_fs の内部実装であり、repos.c 内で完結する操作が主です。
  • commit 失敗時のクリーンアップ処理が dav_fs_close_stream に依存しているため、この構造体のフィールドを修正する際はクローズ時のロジックへの影響を確認してください。
