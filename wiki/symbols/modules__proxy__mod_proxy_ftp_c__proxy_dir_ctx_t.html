proxy_dir_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:436

目的

FTP ディレクトリ一覧表示フィルタ（proxy_send_dir_filter）における実行状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    apr_bucket_brigade *in;
    char buffer[MAX_STRING_LEN];
    enum {
        HEADER, BODY, FOOTER
    }    state;
}      proxy_dir_ctx_t;

入力と出力

  • **入力**: apr_bucket_brigade を介して受信した FTP サーバーからの raw データ。
  • **出力**: state に基づき、HTML 形式に変換されたディレクトリリストの断片。

前提条件と不変条件

  • buffer は MAX_STRING_LEN のサイズ制限に従う必要があります。
  • state は HEADER、BODY、FOOTER のいずれかの段階を維持し、フィルタ処理の進行状況を追跡します。

アルゴリズムまたは振る舞い

FTP サーバーから取得したディレクトリリストを逐次読み込み、HTML 形式へ変換するフィルタ処理の内部状態を管理します。in バケットブリゲードを通じてデータを受け取り、state を遷移させながら HTML タグの付与や整形を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ftp.c 内の proxy_send_dir_filter 関数が、フィルタコンテキスト（f->ctx）として本構造体を参照しています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • apr_bucket_brigade: ネットワークデータのバッファリングに使用。
  • proxy_ftp_dir_conf: ディレクトリ単位の設定情報。

失敗モードと境界条件

  • MAX_STRING_LEN を超える行データが入力された場合、バッファオーバーフローや切り捨てが発生する可能性があります。
  • フィルタの state 遷移が不正な順序で行われた場合、HTML 出力が破損する可能性があります。

保守担当者向けメモ

  • 本構造体は proxy_send_dir_filter の実行状態を保持するため、リクエストの生存期間中、適切に初期化および破棄される必要があります。
  • state の列挙型は、FTP のディレクトリリストを HTML に変換する際の現在の処理フェーズを厳密に管理するために使用されています。
