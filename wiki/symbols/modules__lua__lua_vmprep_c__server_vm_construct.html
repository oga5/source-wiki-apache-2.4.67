server_vm_construct

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:414

目的

サーバーレベルの Lua 仮想マシン（VM）を構築し、リソースリスト（apr_reslist）で管理可能な形式にラップして初期化する。

シグネチャまたは形状

static apr_status_t server_vm_construct(lua_State **resource, void *params, apr_pool_t *pool)

入力と出力

  • **入力**:
  • resource: 生成された VM 構造体（ap_lua_server_spec）を格納するためのポインタ。
  • params: VM 構築に必要なパラメータ（vm_construct へ渡される）。
  • pool: メモリ割り当てに使用する Apache プール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は APR_EGENERAL を返す。

前提条件と不変条件

  • vm_construct が正常に lua_State を生成できること。
  • pool は有効な apr_pool_t であること。
  • 成功時、*resource には ap_lua_server_spec 構造体が格納され、その内部の L には初期化済みの lua_State が保持される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して ap_lua_server_spec 構造体をメモリプール上に確保する。 2. *resource を NULL に初期化する。 3. vm_construct を呼び出し、lua_State を生成する。 4. vm_construct が成功した場合:

  • spec->finfo を初期化する。
  • spec->L に生成された lua_State を代入する。
  • *resource に spec を設定する。
  • lua_pushlightuserdata を使用して spec を Lua レジストリ（Apache2.Lua.server_spec）に登録する。
  • APR_SUCCESS を返す。

    • 5. 失敗した場合は APR_EGENERAL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_vmprep.c 内の apr_reslist_constructor（推測されるリソースリスト構築処理）。
  • **呼び出し先**: vm_construct。

関連データ構造

  • ap_lua_server_spec: サーバーレベルの VM 状態とファイル情報を保持する構造体。
  • lua_State: Lua 仮想マシンの状態。
  • apr_pool_t: Apache のメモリ管理プール。

失敗モードと境界条件

  • vm_construct が失敗した場合、本関数も APR_EGENERAL を返し、リソースは構築されない。
  • メモリ不足により apr_pcalloc が失敗した場合、Apache の標準的なメモリ割り当てエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は apr_reslist による VM プーリング機構の一部として機能する。
  • lua_setfield でレジストリに登録される Apache2.Lua.server_spec は、Lua 側からサーバー設定へアクセスするために使用される。
