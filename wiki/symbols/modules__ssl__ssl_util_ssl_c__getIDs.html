getIDs

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ssl.c:363

目的

X.509 証明書からホスト名照合に使用可能な識別子（DNS-ID および CN-ID）を抽出し、配列として収集します。

シグネチャまたは形状

static BOOL getIDs(apr_pool_t *p, X509 *x509, apr_array_header_t **ids)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • x509: 情報を抽出する対象の OpenSSL X509 構造体。
  • ids: 抽出された識別子（文字列）を格納する apr_array_header_t へのポインタ。
  • **出力**:
  • 戻り値: 識別子が一つでも抽出できた場合は TRUE、それ以外は FALSE。
  • ids: 抽出結果が格納された配列。失敗時は NULL または空の配列。

前提条件と不変条件

  • x509 が NULL の場合、関数は直ちに FALSE を返します。
  • 抽出された識別子は、指定された apr_pool_t の生存期間に依存します。

アルゴリズムまたは振る舞い

1. **DNS-ID の抽出**: modssl_X509_getSAN を呼び出し、証明書の Subject Alternative Name (SAN) 拡張から GEN_DNS 形式の項目を ids 配列に追加します。 2. **CN-ID の抽出**: 証明書の Subject DN から NID_commonName 属性を検索します。見つかったすべての Common Name を modssl_X509_NAME_ENTRY_to_string で文字列化し、ids 配列に追加します。 3. **結果判定**: 配列が空でない場合に TRUE を返し、それ以外は FALSE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_util_ssl.c:406（modssl_X509_match_name 内から呼び出されることが確認されています）。
  • **呼び出し先**:
  • modssl_X509_getSAN
  • X509_get_subject_name
  • X509_NAME_get_index_by_NID
  • X509_NAME_get_entry
  • modssl_X509_NAME_ENTRY_to_string
  • apr_is_empty_array

関連データ構造

  • X509: OpenSSL の証明書構造体。
  • apr_array_header_t: Apache Portable Runtime の動的配列。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • x509 が NULL の場合、または SAN および CN のいずれも抽出できない場合は FALSE を返します。
  • modssl_X509_getSAN が失敗し、かつ ids が初期化されていない場合、ids を NULL に設定して終了します。

保守担当者向けメモ

  • 本関数は modssl_X509_match_name の内部ヘルパーとして機能します。
  • 抽出ロジックは RFC 6125 に準拠したホスト名照合を前提としています。
