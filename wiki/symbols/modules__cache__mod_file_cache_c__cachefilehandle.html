cachefilehandle

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_file_cache.c:206

目的

cachefile ディレクティブによって指定されたファイルをキャッシュ対象として登録し、ファイルハンドルを保持する。

シグネチャまたは形状

static const char *cachefilehandle(cmd_parms *cmd, void *dummy, const char *filename)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていない引数。
  • filename: キャッシュ対象のファイルパス。
  • **出力**:
  • 成功時または警告ログ出力後: NULL を返す。

前提条件と不変条件

  • 実行環境が APR_HAS_SENDFILE をサポートしている必要がある。
  • cmd パラメータは有効なサーバー設定コンテキストを指している必要がある。

アルゴリズムまたは振る舞い

1. コンパイル時に APR_HAS_SENDFILE が有効か確認する。 2. 有効な場合、cache_the_file 関数を呼び出し、指定された filename をキャッシュ対象として処理する。 3. 無効な場合、ap_log_error を使用して、OS が sendfile をサポートしていない旨の警告ログを出力する。 4. いずれの場合も NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_file_cache.c:385 において、AP_INIT_ITERATE マクロを通じて cachefile ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: cache_the_file (条件付き)、ap_log_error (条件付き)。

関連データ構造

  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。

失敗モードと境界条件

  • **Sendfile 非対応**: OS が sendfile をサポートしていない場合、キャッシュ登録は行われず、エラーログに警告が出力される。
  • **ファイルアクセスエラー**: cache_the_file 内部でファイルが開けない、または統計情報が取得できない場合、その挙動は cache_the_file の実装に依存する。

保守担当者向けメモ

  • 本関数は cachefile ディレクティブの直接的なエントリーポイントである。
  • APR_HAS_SENDFILE の有無による条件分岐が含まれており、プラットフォーム依存の挙動を制御している。
