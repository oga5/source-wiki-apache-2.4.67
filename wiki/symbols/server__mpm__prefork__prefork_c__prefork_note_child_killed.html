prefork_note_child_killed

定義

  • 種別: function
  • 定義箇所: server/mpm/prefork/prefork.c:198

目的

子プロセスの終了をスコアボードに反映し、関連するフックを呼び出すことで、システム全体にプロセスの終了状態を通知する。

シグネチャまたは形状

static void prefork_note_child_killed(int childnum, pid_t pid, ap_generation_t gen)

入力と出力

  • **入力**:
  • childnum: スコアボード内の子プロセスインデックス。
  • pid: 終了したプロセスのプロセスID。
  • gen: プロセスの世代番号。
  • **出力**: なし（void）。

前提条件と不変条件

  • childnum は -1 であってはならない（AP_DEBUG_ASSERT により検証）。
  • スコアボードの該当インデックスが有効であることを前提としている。

アルゴリズムまたは振る舞い

1. childnum が -1 でないことをアサーションで確認する。 2. ap_run_child_status を呼び出し、サーバー設定、PID、世代、インデックス、および終了状態（MPM_CHILD_EXITED）を引数として、子プロセスの終了を通知する。 3. スコアボード内の該当インデックスの pid を 0 にリセットし、プロセススロットを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/prefork/prefork.c 内のプロセス監視ループおよび終了処理ロジックから呼び出される（行235, 1019, 1083, 1209）。
  • ap_relieve_child_processes の引数として渡されるケースがある（行1121, 1140）。
  • **呼び出し先**:
  • ap_run_child_status: 子プロセスの状態変更を通知するフック関数。

関連データ構造

  • ap_scoreboard_image: プロセスごとの状態（PID、世代など）を保持する共有メモリ上の構造体。

失敗モードと境界条件

  • childnum が -1 の場合、デバッグビルドではアサーションエラーが発生する。
  • スコアボードの更新は直接行われるため、呼び出し元が正しい childnum を保持している必要がある。

保守担当者向けメモ

  • 本関数はプロセス終了時のクリーンアップの要であり、スコアボードの整合性を維持する役割を持つ。
  • ap_run_child_status は外部モジュールがプロセス終了を監視するためのフックであるため、変更時は影響範囲に注意すること。
