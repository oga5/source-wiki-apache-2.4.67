slotmem_create

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:66

目的

指定されたサイズと個数のスロットを持つメモリ領域を確保し、管理インスタンスを作成または取得します。

シグネチャまたは形状

apr_status_t slotmem_create(ap_slotmem_instance_t **new, const char *name, apr_size_t item_size, unsigned int item_num, ap_slotmem_type_t type, apr_pool_t *pool)

入力と出力

  • **入力**:
  • name: インスタンス名（NULL の場合は "anonymous"）。
  • item_size: 各スロットのサイズ。
  • item_num: スロットの総数。
  • type: スロットメモリの型（ap_slotmem_type_t）。
  • pool: メモリ割り当てに使用するプール。
  • **出力**:
  • new: 作成または取得された ap_slotmem_instance_t 構造体へのポインタ。
  • 戻り値: APR_SUCCESS（成功時）、またはエラーコード。

前提条件と不変条件

  • gpool が初期化されていること。
  • name が指定された場合、既存のインスタンスがあればそれを再利用する。
  • res->base には item_size * item_num 分のデータ領域と、各スロットの利用状況を追跡するための item_num 分の char 領域が確保される。

アルゴリズムまたは振る舞い

1. name が指定されている場合、globallistmem を走査し、一致する名前のインスタンスがあればそれを返して終了する。 2. 該当がない場合、gpool から ap_slotmem_instance_t 構造体とデータ領域を確保する。 3. 確保した領域のベースアドレス、スロットサイズ、個数、型、および inuse フラグ領域（データ領域の末尾に配置）を初期化する。 4. インスタンスを globallistmem リストの末尾に追加する。 5. 新規作成したインスタンスを new に格納し、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cluster/mod_heartmonitor.c
  • modules/proxy/mod_proxy_balancer.c
  • modules/slotmem/mod_slotmem_plain.c（プロバイダー登録用）
  • modules/slotmem/mod_slotmem_shm.c（プロバイダー登録用）
  • **呼び出し先**:
  • ap_runtime_dir_relative
  • apr_pcalloc
  • apr_pstrdup
  • strcmp

関連データ構造

  • ap_slotmem_instance_t: スロットメモリの管理単位。
  • globallistmem: 作成された全インスタンスを保持する静的リスト。
  • gpool: インスタンス生成に使用されるグローバルプール。

失敗モードと境界条件

  • res->base の確保に失敗した場合、APR_ENOSHMAVAIL を返す。
  • name が指定されている場合、既存インスタンスの検索はリストの線形探索で行われる。

保守担当者向けメモ

  • 本関数は単一プロセス内での利用を前提としており、スレッドセーフではありません。マルチスレッド環境で利用する場合は、呼び出し側で排他制御を行ってください。
  • inuse フラグ領域はデータ領域の直後に配置されるため、メモリレイアウトの変更時には注意が必要です。
