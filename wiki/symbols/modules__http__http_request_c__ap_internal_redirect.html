ap_internal_redirect

定義

  • 種別: function
  • 定義箇所: modules/http/http_request.c:774

目的

リクエストの内部リダイレクトを実行し、新しい URI に基づくリクエストコンテキストの構築と処理フローの再実行を行う。

シグネチャまたは形状

void ap_internal_redirect(const char *new_uri, request_rec *r)

入力と出力

  • new_uri: リダイレクト先の URI 文字列。
  • r: 現在のリクエスト構造体 (request_rec)。
  • 戻り値: なし（void）。

前提条件と不変条件

  • r は有効なリクエスト構造体であること。
  • 内部リダイレクト処理において、新しいリクエストコンテキストの生成に失敗した場合、関数は即座に終了する。

アルゴリズムまたは振る舞い

1. internal_internal_redirect を呼び出し、新しい URI 用の request_rec を生成する。 2. 生成された new が NULL の場合、処理を中断する。 3. AP_INTERNAL_REDIRECT マクロを実行する。 4. ap_run_quick_handler を呼び出し、クイックハンドラによる処理を試みる。 5. クイックハンドラが DECLINED を返した場合、ap_process_request_internal で設定の適用とアクセスチェックを行い、成功すれば ap_invoke_handler でハンドラを実行する。 6. 最後に ap_die を呼び出し、リクエストの終了処理またはエラーレスポンスの生成を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/win32/mod_isapi.c
  • modules/cache/mod_cache.c
  • modules/http/http_request.c
  • modules/mappers/mod_negotiation.c
  • modules/mappers/mod_rewrite.c
  • **呼び出し先**:
  • internal_internal_redirect
  • ap_run_quick_handler
  • ap_process_request_internal
  • ap_invoke_handler
  • ap_die

関連データ構造

  • request_rec: リクエストのライフサイクルとコンテキストを保持する主要構造体。

失敗モードと境界条件

  • internal_internal_redirect が NULL を返した場合、リダイレクト処理は行われず正常終了する（エラー発生時は既に ap_die が呼び出されている前提）。

保守担当者向けメモ

  • 本関数はリダイレクト後の処理フローを再帰的に制御する。フィルタチェーンの更新については internal_internal_redirect 内部で update_r_in_filters が行われる点に注意すること。
