create_alias_dir_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_alias.c:87

目的

ディレクトリごとのエイリアス設定構造体（alias_dir_conf）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_alias_dir_config(apr_pool_t *p, char *d)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t プール。
  • d: ディレクトリパス（本関数内では未使用）。
  • **出力**:
  • 初期化済みの alias_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全メンバはゼロ初期化される。
  • redirects 配列は、初期容量 2 で alias_entry 型の要素を保持するように作成される。
  • allow_relative および alias_preserve_path は、デフォルト値（ALIAS_FLAG_DEFAULT）で設定される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、alias_dir_conf サイズのメモリをプール p から確保する。 2. apr_array_make を呼び出し、リダイレクト情報を格納するための動的配列 redirects を初期化する。 3. 設定フラグ allow_relative および alias_preserve_path にデフォルト値を代入する。 4. 構築した構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_alias.c:760（Apache のモジュール設定登録時に呼び出される）。
  • **呼び出し先**: apr_pcalloc, apr_array_make（APR API）。

関連データ構造

  • alias_dir_conf: 本関数が生成するディレクトリ設定構造体。
  • alias_entry: redirects 配列の要素として使用される構造体。

失敗モードと境界条件

  • apr_pcalloc および apr_array_make はメモリ不足時にアボートする可能性がある（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は mod_alias のディレクトリ設定の雛形を作成する。設定の継承やマージが必要な場合は、merge_alias_dir_config が別途処理を行う。
