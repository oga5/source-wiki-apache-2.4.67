remove_output_filters_below

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_switch.c:131

目的

Apache HTTP Server のフィルターチェーンにおいて、指定されたフィルター型（ftype）よりも優先度が低い（数値が小さい）出力フィルターをチェーンから順次削除します。主に HTTP/1.1 から HTTP/2 へのプロトコル切り替え時に、不要となった HTTP/1.1 固有のフィルターを解除するために使用されます。

シグネチャまたは形状

static void remove_output_filters_below(ap_filter_t *f, ap_filter_type ftype)

入力と出力

  • **入力**:
  • f: 削除対象の走査を開始する ap_filter_t 構造体へのポインタ。
  • ftype: 削除の閾値となる ap_filter_type。
  • **出力**: なし（副作用としてフィルターチェーンが変更されます）。

前提条件と不変条件

  • f が NULL の場合、何も行われません。
  • f->frec->ftype < ftype を満たすフィルターが対象となります。
  • フィルターチェーンのリンク構造を破壊しないよう、削除前に f->next を保持する実装になっています。

アルゴリズムまたは振る舞い

1. 引数 f から開始し、現在のフィルターの型（f->frec->ftype）が指定された ftype より小さい間、ループを継続します。 2. ループ内で f->next を fnext に退避します。 3. ap_remove_output_filter(f) を呼び出し、現在のフィルターをチェーンから削除します。 4. f を fnext に更新し、次のフィルターへ進みます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_switch.c:189 において、r->output_filters を引数として呼び出されています。
  • **呼び出し先**: Apache HTTP Server のコア関数である ap_remove_output_filter を呼び出します。

関連データ構造

  • ap_filter_t: Apache のフィルターチェーンを構成するノード。
  • ap_filter_type: フィルターの実行順序を決定する列挙型。

失敗モードと境界条件

  • フィルターチェーンの末尾に達した場合、ループは正常に終了します。
  • 既に空のチェーンに対して呼び出された場合、何も行われません。

保守担当者向けメモ

  • 本関数はフィルターチェーンを直接操作する強力な関数です。削除対象の ftype を誤ると、HTTP/2 動作に必要な必須フィルターまで削除してしまう可能性があるため、呼び出し時の ftype 指定には注意してください。
