authz_require_alias_section

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_core.c:248

目的

Apache 設定ファイル内の <AuthzProviderAlias> ディレクティブを解析し、既存の認可プロバイダーに対するエイリアスを登録する。

シグネチャまたは形状

static const char *authz_require_alias_section(cmd_parms *cmd, void *mconfig, const char *args)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • mconfig: ディレクトリ設定構造体。
  • args: <AuthzProviderAlias に続く引数文字列（プロバイダー名、エイリアス名、引数）。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • グローバルコンテキスト（GLOBAL_ONLY）でのみ呼び出し可能。
  • エイリアス対象となるプロバイダーは、設定ファイルが使用される時点でロードおよび登録済みである必要がある。
  • 複数の引数を渡す場合は引用符で囲む必要がある。

アルゴリズムまたは振る舞い

1. 引数からプロバイダー名、エイリアス名、および関連引数を抽出する。 2. ap_walk_config を使用して、<AuthzProviderAlias> ブロック内の子ディレクティブを再帰的に解析し、新しいディレクトリ設定を作成する。 3. ap_lookup_provider を使用して、指定されたプロバイダー名に対応するプロバイダー構造体を取得する。 4. プロバイダーが parse_require_line を持つ場合、それを呼び出して引数を解析・検証する。 5. authz_core_srv_conf のハッシュテーブル alias_rec にエイリアス情報を格納する。 6. ap_register_auth_provider を呼び出し、エイリアス名を新しい認可プロバイダーとして登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_core.c:684 (AP_INIT_RAW_ARGS マクロ経由)
  • **呼び出し先**:
  • ap_check_cmd_context
  • ap_walk_config
  • ap_lookup_provider
  • ap_register_auth_provider
  • プロバイダー固有の parse_require_line 関数

関連データ構造

  • provider_alias_rec: エイリアス設定を保持する構造体。
  • authz_core_srv_conf: サーバー設定構造体（エイリアスハッシュを保持）。
  • cmd_parms: 設定コマンドのコンテキスト。

失敗モードと境界条件

  • 閉じタグ > が見つからない場合、エラーを返す。
  • プロバイダー名やエイリアス名が不足している場合、エラーを返す。
  • 指定されたプロバイダー名が未登録の場合、エラーを返す。
  • プロバイダーの parse_require_line が失敗した場合、そのエラーを返す。
  • 複数の引数が引用符なしで渡された場合、警告をログ出力し、最初の引数のみを処理する。

保守担当者向けメモ

  • 本関数は AuthzProviderAlias の設定フェーズのみを担当する。
  • 登録されたエイリアスは、リクエスト処理時に authz_alias_provider を通じて実行される。
  • new_authz_config は cmd->pool を使用して割り当てられるため、設定のライフサイクルに依存する。
