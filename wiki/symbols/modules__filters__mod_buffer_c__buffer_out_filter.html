buffer_out_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_buffer.c:55

目的

出力フィルタチェーンにおいて、小さなバケットを BufferSize ディレクティブで指定されたサイズまで集約（collapse）し、ネットワーク I/O の効率を向上させる。

シグネチャまたは形状

apr_status_t buffer_out_filter(ap_filter_t *f, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • f: フィルタコンテキストを含むフィルタ構造体。
  • bb: 処理対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（APR_SUCCESS またはエラーコード）。

前提条件と不変条件

  • サブクエスト（f->r->main が存在する場合）ではバッファリングを行わず、フィルタを削除して透過させる。
  • コンテキスト ctx はリクエストプール（r->pool）に紐付いており、リクエスト終了時に自動解放される。
  • メタデータバケット（EOS, Flush）はバッファリングされず、即座に下流へ渡される。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時に buffer_ctx を生成し、設定を読み込む。 2. **空のブリゲード**: 入力が空の場合は即座に下流へ渡す。 3. **バケット処理**:

  • **EOS/Flush**: 蓄積済みのバッファをフラッシュし、メタデータバケットを送信する。
  • **メタデータ**: 蓄積バッファへ移動し、そのまま保持する。
  • **データ**:
  • バッファサイズが ctx->conf->size を超えた場合、蓄積分をフラッシュする。
  • ヒープバケットかつサイズが APR_BUCKET_BUFF_SIZE と一致する場合、コピーではなく移動（move）による最適化を行う。
  • それ以外は apr_brigade_write を使用してバッファへコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache フィルタチェーン（ap_register_output_filter により登録）。
  • **呼び出し先**:
  • ap_pass_brigade: 下流のフィルタへデータを渡す。
  • apr_bucket_read: バケットの内容を読み込む。
  • apr_brigade_write: バッファへデータを書き込む。
  • ap_remove_output_filter: サブクエスト時に自身を削除する。

関連データ構造

  • buffer_ctx: 実行時の状態（メインバッファ bb、設定 conf を保持）。
  • buffer_conf: BufferSize ディレクティブに基づく設定値。
  • apr_bucket_brigade: バケットの連結リスト。

失敗モードと境界条件

  • **サブクエスト**: バッファリングをスキップし、透過させる。
  • **バッファ上限超過**: len > ctx->conf->size となった時点で下流へフラッシュする。
  • **書き込み失敗**: ap_pass_brigade がエラーを返した場合、ループを継続しつつエラー状態を保持する。

保守担当者向けメモ

  • ETag 生成ロジックはコメントアウトされており、現時点では機能していない。
  • ヒープバケットの移動最適化は、バケットサイズが APR_BUCKET_BUFF_SIZE と一致する場合にのみ有効である。
  • 入力フィルタとは異なり、出力フィルタはサブクエストの判定に f->r->main を使用している。
