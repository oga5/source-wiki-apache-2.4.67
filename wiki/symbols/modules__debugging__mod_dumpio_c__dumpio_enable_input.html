dumpio_enable_input

定義

  • 種別: function
  • 定義箇所: modules/debugging/mod_dumpio.c:221

目的

Apache HTTP Server の設定ディレクティブ DumpIOInput に対応し、サーバー設定構造体内の入力ダンプ有効フラグを更新する。

シグネチャまたは形状

static const char *dumpio_enable_input(cmd_parms *cmd, void *dummy, int arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されない引数。
  • arg: ディレクティブで指定された有効/無効を示す整数値。
  • **出力**:
  • 常に NULL を返す（設定処理の成功を示す）。

前提条件と不変条件

  • cmd->server->module_config に dumpio_module に関連付けられた dumpio_conf_t 構造体が格納されている必要がある。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定から dumpio_conf_t 構造体を取得する。 2. 取得した構造体の enable_input メンバに、引数 arg の値を代入する。 3. NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/debugging/mod_dumpio.c:240 における AP_INIT_FLAG("DumpIOInput", ...) マクロ定義により、設定解析時に間接的に呼び出される。
  • **呼び出し先**: ap_get_module_config（Apache コア関数）。

関連データ構造

  • dumpio_conf_t: 入出力の有効状態を保持する設定構造体。
  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。

失敗モードと境界条件

  • ap_get_module_config が期待する設定構造体を返せない場合、セグメンテーションフォールト等の未定義動作を引き起こす可能性がある（ただし、本モジュールの初期化フローにおいて通常は保証される）。

保守担当者向けメモ

  • 本関数は AP_INIT_FLAG を介して呼び出されるため、引数 arg には DumpIOInput ディレクティブの On/Off に応じた値が渡される。
  • dumpio_conf_t の定義および dumpio_module の登録状況と整合性を保つこと。
