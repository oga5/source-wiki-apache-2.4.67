create_ident_dir_config

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_ident.c:260

目的

mod_ident モジュールで使用されるディレクトリ単位の設定構造体（ident_config_rec）をメモリ上に割り当て、初期値を設定する。

シグネチャまたは形状

static void *create_ident_dir_config(apr_pool_t *p, char *d)

入力と出力

  • **入力**:
  • p: 設定構造体の割り当てに使用するメモリプール。
  • d: 設定が適用されるディレクトリパス（本関数内では未使用）。
  • **出力**:
  • 初期化済みの ident_config_rec 構造体へのポインタ（void * 型として返却）。

前提条件と不変条件

  • apr_palloc により、指定されたプール p 上に ident_config_rec サイズのメモリが確保される。
  • conf->timeout_unset は初期状態で 1（未設定）にセットされる。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して ident_config_rec 構造体分のメモリを確保する。 2. do_rfc1413 フラグを DEFAULT_RFC1413 | RFC1413_UNSET で初期化する。 3. timeout を RFC1413_TIMEOUT 定数から変換して設定する。 4. timeout_unset フラグを 1 に設定する。 5. 確保した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の設定読み込みプロセス（modules/metadata/mod_ident.c:338 にて、ディレクトリ設定作成用関数として登録されている）。
  • **呼び出し先**: apr_palloc（メモリ確保）、apr_time_from_sec（時間変換）。

関連データ構造

  • ident_config_rec: 本関数で初期化されるモジュール固有の設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある（Apache の標準的なメモリ管理動作に依存）。

保守担当者向けメモ

  • 本関数はディレクトリごとの設定を初期化する役割を持つ。設定値の変更やデフォルト値の調整を行う際は、mod_ident.c 内の定数定義と併せて確認すること。
