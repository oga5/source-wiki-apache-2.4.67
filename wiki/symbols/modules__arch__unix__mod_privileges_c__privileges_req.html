privileges_req

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_privileges.c:173

目的

リクエスト処理時に、設定された仮想ホスト（vhost）またはディレクトリ単位の権限セットに基づき、プロセス権限の降格またはプロセス分離（fork）を実行する。

シグネチャまたは形状

static int privileges_req(request_rec *r)

入力と出力

  • 入力: request_rec *r（現在のリクエスト情報）
  • 出力: int（Apache ステータスコード: OK, DONE, DECLINED, または HTTP_INTERNAL_SERVER_ERROR）

前提条件と不変条件

  • 本関数は post_read_request および header_parser フックから呼び出される。
  • マルチスレッド MPM との互換性はなく、非スレッド環境での動作を前提とする。
  • PRIV_SECURE モードで fork された場合、子プロセスはリクエスト処理を完遂し、親プロセスは DONE を返してリクエスト処理から離脱する。

アルゴリズムまたは振る舞い

1. **設定取得**: priv_cfg を取得し、request_config 内の breadcrumb を確認して初回呼び出し（vhost）かディレクトリ単位の呼び出しを判定する。 2. **モード判定**: PRIV_SELECTIVE モードの場合、ディレクトリ設定が未確定であれば DECLINED を返し、後続の処理へ委譲する。 3. **プロセス分離**: PRIV_SECURE モードの場合、apr_proc_fork を実行する。

  • 親プロセス: 子プロセスの終了を待ち、DONE を返して終了する。
  • 子プロセス: 権限降格処理へ進む。

    • 4. **権限降格**: privileges_end_req をクリーンアップハンドラとして登録する。 5. **権限適用**: setppriv を使用し、PRIV_EFFECTIVE, PRIV_INHERITABLE, PRIV_LIMIT を設定する。UID/GID が設定されている場合は setuid/setgid を実行する。 6. **追加制限**: PRIV_SECURE モードの子プロセスでは、PRIV_PERMITTED も制限する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • ap_hook_post_read_request (登録箇所: modules/arch/unix/mod_privileges.c:410)
  • ap_hook_header_parser (登録箇所: modules/arch/unix/mod_privileges.c:412)
  • 呼び出し先:
  • apr_proc_fork, apr_proc_wait
  • setppriv, setuid, setgid
  • privileges_end_req (クリーンアップ登録)

関連データ構造

  • priv_cfg: vhost 単位の権限設定（UID, GID, 特権セット）。
  • priv_dir_cfg: ディレクトリ単位の動作モード設定。
  • request_rec: リクエストごとのコンテキスト保持に使用。

失敗モードと境界条件

  • fork 失敗時: HTTP_INTERNAL_SERVER_ERROR を返す。
  • setppriv または setuid/setgid 失敗時: ログを出力し、セキュリティリスクを避けるため HTTP_INTERNAL_SERVER_ERROR を返す。
  • PRIV_SECURE モードでの子プロセス生成時、親プロセスは r->eos_sent = 1 を設定し、リクエスト処理を終了させる。

保守担当者向けメモ

  • BIG_SECURITY_HOLE マクロで保護された機能はデフォルトで無効であり、取り扱いには注意が必要。
  • apr_pool_cleanup_register を使用しているため、リクエスト終了時には自動的に privileges_end_req が呼ばれ、権限が復元される設計となっている。
