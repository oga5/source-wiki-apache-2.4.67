shmcb_cyclic_memcmp

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_shmcb.c:234

目的

共有メモリ内の循環バッファ上に配置されたデータと、指定されたソースデータを比較する。バッファの末尾で折り返し（ラップアラウンド）が発生する場合を考慮し、メモリ内容を正しく比較する。

シグネチャまたは形状

int shmcb_cyclic_memcmp(unsigned int buf_size, unsigned char *data,
                        unsigned int dest_offset,
                        const unsigned char *src,
                        unsigned int src_len)

入力と出力

  • buf_size: 循環バッファ全体のサイズ。
  • data: 循環バッファの先頭ポインタ。
  • dest_offset: 比較を開始するバッファ内のオフセット位置。
  • src: 比較対象となるソースデータのポインタ。
  • src_len: 比較するデータの長さ。
  • 戻り値: memcmp と同様、比較結果（一致すれば 0、不一致なら非 0）。

前提条件と不変条件

  • dest_offset は buf_size 未満であること。
  • 比較対象のデータが循環バッファの境界を跨ぐ場合、バッファの末尾から先頭へ連続しているものとして処理される。

アルゴリズムまたは振る舞い

1. dest_offset + src_len が buf_size 未満の場合、バッファ内で折り返しが発生しないため、memcmp を一度実行して結果を返す。 2. 折り返しが発生する場合:

  • まず、バッファの末尾部分（buf_size - dest_offset 分）を memcmp で比較する。
  • 不一致があれば即座にその結果を返す。
  • 一致していれば、残りのデータ（src_len + dest_offset - buf_size 分）をバッファの先頭から比較し、その結果を返す。

関連データ構造

失敗モードと境界条件

  • src_len が buf_size を超えるような不正な入力に対する挙動は定義されていない。
  • dest_offset が buf_size 以上の場合、ポインタ演算がバッファ領域外を指す可能性がある。

保守担当者向けメモ

  • 本関数は循環バッファの特性を隠蔽するユーティリティである。
  • unsigned int を使用しているため、非常に大きなバッファサイズやデータ長を扱う際にはオーバーフローのリスクに注意すること。
