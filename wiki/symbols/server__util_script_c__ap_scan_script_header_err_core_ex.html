ap_scan_script_header_err_core_ex

定義

  • 種別: function
  • 定義箇所: server/util_script.c:482

目的

CGI スクリプトの出力を読み取り、HTTP ヘッダーとして解析・検証し、request_rec 構造体に反映させるコアエンジン。

シグネチャまたは形状

int ap_scan_script_header_err_core_ex(request_rec *r, char *buffer,
                                       int (*getsfunc) (char *, int, void *),
                                       void *getsfunc_data,
                                       int module_index)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • buffer: ヘッダー行を格納するバッファ（NULL の場合は内部スタック変数を使用）。
  • getsfunc: スクリプト出力を 1 行ずつ読み取るためのコールバック関数。
  • getsfunc_data: getsfunc に渡されるコンテキストデータ。
  • module_index: ログ出力に使用するモジュールインデックス。
  • **出力**:
  • 成功時は OK または ap_meets_conditions の結果を返す。
  • 失敗時は HTTP_INTERNAL_SERVER_ERROR または HTTP_GATEWAY_TIME_OUT を返す。

前提条件と不変条件

  • すべての動的メモリ割り当ては r->pool を使用する。
  • getsfunc はスクリプト出力の終端（空行）までヘッダーを供給する必要がある。
  • APR_CHARSET_EBCDIC 定義環境下では、ASCII/EBCDIC の自動変換ロジックが介入する。

アルゴリズムまたは振る舞い

1. **初期化**: ヘッダーマージ用テーブルおよび Set-Cookie 用テーブルを初期化。 2. **ループ処理**: getsfunc を使用してスクリプト出力を 1 行ずつ読み取る。

  • 読み取り失敗（rv == 0）は「ヘッダーの早期終了」、タイムアウト（rv == -1）は「ゲートウェイタイムアウト」として処理。
  • 行末の改行コード（CR/LF）を除去。

    • 3. **ヘッダー解析**:

  • 空行（\0）に到達した場合、Status ヘッダーの有無を確認し、err_headers_out をマージして終了。
  • Content-type, Status, Location, Content-Length, Content-Range, Transfer-Encoding, ETag, Last-Modified, Set-Cookie を個別に処理。
  • Last-Modified は日付の妥当性を検証し、必要に応じて更新。
  • 未知のヘッダーは merge テーブルに蓄積。

    • 4. **エラーハンドリング**: 不正なヘッダー形式（: が含まれない行）が検出された場合、エラーをログ出力し、スクリプト出力を読み切った後にエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_script.c 内の ap_scan_script_header_err 系関数群（ap_scan_script_header_err, ap_scan_script_header_err_ex, ap_scan_script_header_err_brigade など）。
  • **呼び出し先**:
  • getsfunc コールバック関数。
  • ap_meets_conditions, ap_log_rerror, apr_table_make, apr_table_do, apr_table_overlap, apr_table_overlay, apr_table_unset, apr_table_add, apr_table_set, ap_set_content_type, ap_update_mtime, ap_set_last_modified, apr_date_parse_http, apr_date_parse_rfc 等の APR/HTTPD API。

関連データ構造

  • request_rec: リクエスト状態の保持。
  • apr_table_t: ヘッダー情報の管理に使用。

失敗モードと境界条件

  • **スクリプトの早期終了**: ヘッダーが出力される前にストリームが終了した場合、HTTP_INTERNAL_SERVER_ERROR を返す。
  • **タイムアウト**: getsfunc が -1 を返した場合、HTTP_GATEWAY_TIME_OUT を返す。
  • **不正なヘッダー**: : を含まない行が検出された場合、HTTP_INTERNAL_SERVER_ERROR を返す。
  • **無効な Status**: Status ヘッダーの値が HTTP 仕様に適合しない場合、ログ出力を行う。

保守担当者向けメモ

  • 本関数はスクリプトの出力が完了するまでブロックする可能性がある。非同期 I/O が必要な場合は注意が必要。
  • Set-Cookie は他のヘッダーとマージせず、個別に保持・出力される仕様となっている。
  • Last-Modified の処理には GMT 変換や将来の日付に対するガードロジックが含まれている。
