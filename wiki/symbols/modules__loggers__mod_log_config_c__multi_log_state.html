multi_log_state

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_log_config.c:237

目的

サーバー（またはバーチャルホスト）単位のログ設定を保持するための構造体です。ログフォーマットの定義や、有効なログ出力設定のリストを管理します。

シグネチャまたは形状

typedef struct {
    const char *default_format_string;
    apr_array_header_t *default_format;
    apr_array_header_t *config_logs;
    apr_array_header_t *server_config_logs;
    apr_table_t *formats;
} multi_log_state;

入力と出力

  • **入力**: サーバー設定時に apr_palloc を介してメモリが割り当てられ、設定ディレクティブ（LogFormat, CustomLog 等）の内容で初期化されます。
  • **出力**: ログ記録処理（log_transaction フック）において、リクエストに関連付けられたサーバー設定から本構造体が取得され、ログ出力の制御に使用されます。

前提条件と不変条件

  • request_rec のプール（r->pool）から割り当てられることが想定されており、リクエスト終了時に自動的に解放されます。
  • サーバーごとの設定として ap_get_module_config を通じてアクセスされます。

アルゴリズムまたは振る舞い

  • サーバー単位のログ設定を保持し、リクエスト処理時にどのフォーマットやログ出力先を使用すべきかを決定するための基盤となります。
  • config_logs および server_config_logs は、設定されたログ出力先（config_log_state）のリストを保持します。
  • formats テーブルは、名前付きログフォーマットの定義を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_log_config.c 内の各種設定関数（add_custom_log 等）およびリクエスト処理関数（multi_log_transaction 等）から参照されます。
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数を直接呼び出すことはありません。ただし、保持する apr_array_header_t 等のデータは、ログ出力処理において config_log_state の評価に使用されます。

関連データ構造

  • config_log_state: 個別のログファイル設定を保持する構造体。multi_log_state 内のリストに含まれます。
  • log_format_item: フォーマット文字列を解析した結果の中間形式。default_format 等で参照されます。

失敗モードと境界条件

  • 設定が未定義の場合、mls が NULL になる可能性があります。
  • メモリ割り当て失敗時は apr_palloc の挙動に従います。

保守担当者向けメモ

  • 本構造体はサーバー単位の構成を管理するため、バーチャルホスト環境での設定継承やマージ処理において重要な役割を果たします。
  • 変更を加える際は、mod_log_config.c 内の ap_get_module_config を使用している箇所との整合性に注意してください。
