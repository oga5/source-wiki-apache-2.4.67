merge_core_server_configs

定義

  • server/core.c:532

目的

ベースとなるサーバー設定（basev）と仮想ホスト等の設定（virtv）をマージし、新しい設定構造体を生成する。

シグネチャまたは形状

void *merge_core_server_configs(apr_pool_t *p, void *basev, void *virtv)

入力と出力

  • **入力**:
  • p: 設定の割り当てに使用するメモリプール。
  • basev: 継承元の core_server_config 構造体。
  • virtv: 上書きまたは追加を行う core_server_config 構造体。
  • **出力**:
  • マージ結果を格納した新しい core_server_config 構造体へのポインタ。

前提条件と不変条件

  • basev および virtv は core_server_config 型としてキャスト可能であること。
  • apr_pmemdup により、ベース設定は新しいメモリ領域に複製される。
  • sec_dir および sec_url は apr_array_append を介して連結される。

アルゴリズムまたは振る舞い

1. basev を p 上に複製して初期設定を作成する。 2. virtv に値が存在する場合（UNSET ではない場合）、virtv の値を優先して conf に上書きする。 3. sec_dir および sec_url 配列を連結する。 4. protocols や flush_max_threshold などの設定項目について、virtv の値が有効であればそれを採用し、無効であれば basev の値を維持する。 5. AP_CORE_MERGE_FLAG マクロを使用して、特定のフラグ設定をマージする。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:5827（サーバー設定構造体のマージ処理として参照）。
  • **呼び出し先**: apr_pmemdup, apr_array_append, AP_CORE_MERGE_FLAG（マクロ）。

関連データ構造

  • core_server_config: サーバー単位の設定を保持する構造体。

失敗モードと境界条件

  • virtv の各フィールドが UNSET（未設定）状態の場合、basev の値がそのまま引き継がれる。
  • accf_map は仮想ホスト単位での変更が許可されていないため、マージ対象外となっている。

保守担当者向けメモ

  • sec_ ポインタの最適化（virt 側の配列が空の場合に base のポインタを再利用する）がコメントとして示唆されている。
  • AP_CORE_MERGE_FLAG はマクロであるため、変更時は http_core.h 等の定義を確認すること。
