order_st_from_str

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:61

目的

ACME サーバーから取得した文字列形式のステータスを、内部的な列挙型 md_acme_order_st に変換します。

シグネチャまたは形状

md_acme_order_st order_st_from_str(const char *s)

入力と出力

  • **入力**: s (ACME サーバーから返されるステータス文字列)
  • **出力**: 対応する md_acme_order_st 列挙値。

前提条件と不変条件

  • 入力文字列 s が NULL の場合、デフォルト値として MD_ACME_ORDER_ST_PENDING を返します。

アルゴリズムまたは振る舞い

1. 入力文字列 s が NULL でない場合、以下の文字列比較を行い、一致する列挙値を返します。

  • "valid" → MD_ACME_ORDER_ST_VALID
  • "invalid" → MD_ACME_ORDER_ST_INVALID
  • "ready" → MD_ACME_ORDER_ST_READY
  • "pending" → MD_ACME_ORDER_ST_PENDING
  • "processing" → MD_ACME_ORDER_ST_PROCESSING

    • 2. 上記のいずれにも一致しない場合、または s が NULL の場合は MD_ACME_ORDER_ST_PENDING を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_order.c:125 において、md_json_gets で取得した JSON 内のステータス値を変換するために使用されています。
  • **呼び出し先**: 標準ライブラリの strcmp を使用しています。

関連データ構造

  • md_acme_order_st: ACME Order の状態を表す列挙型。

失敗モードと境界条件

  • 未知のステータス文字列が渡された場合、エラーを返さずデフォルトの MD_ACME_ORDER_ST_PENDING を返します。

保守担当者向けメモ

  • 新しい ACME ステータスがプロトコルに追加された場合は、この関数に条件分岐を追加する必要があります。
