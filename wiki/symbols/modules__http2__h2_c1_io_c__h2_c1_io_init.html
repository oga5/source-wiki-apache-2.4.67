h2_c1_io_init

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:141

目的

HTTP/2 セッションに関連付けられた入出力管理構造体（h2_c1_io）を初期化し、TLS 接続の有無に応じたバッファリング設定および転送サイズパラメータを適用する。

シグネチャまたは形状

apr_status_t h2_c1_io_init(h2_c1_io *io, h2_session *session)

入力と出力

  • **入力**:
  • io: 初期化対象の h2_c1_io 構造体へのポインタ。
  • session: 関連付けられる h2_session 構造体へのポインタ。
  • **出力**:
  • APR_SUCCESS: 初期化成功時に返される。

前提条件と不変条件

  • session->c1 が有効な接続レコード（conn_rec）を保持していること。
  • io->output 用のブリゲードが c->pool および c->bucket_alloc を使用して生成される。

アルゴリズムまたは振る舞い

1. session->c1 から接続情報を取得し、io->session に設定する。 2. apr_brigade_create を呼び出し、出力用ブリゲードを生成する。 3. ap_ssl_conn_is_ssl を用いて TLS 接続か判定し、io->is_tls および io->buffer_output を設定する。 4. H2_CONF_STREAM_MAX_MEM 設定値に基づき io->flush_threshold を算出する。 5. TLS 接続の場合、H2_CONF_TLS_WARMUP_SIZE 等の設定値を用いて io->warmup_size および io->write_size を初期化する（非 TLS の場合は 0 に設定）。 6. APLOGctrace1 が有効な場合、初期化パラメータをログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_session.c:983 (h2_session の初期化プロセス内)
  • **呼び出し先**:
  • apr_brigade_create (APR)
  • ap_ssl_conn_is_ssl (Apache SSL API)
  • h2_config_sgeti64 / h2_config_sgeti (設定取得)
  • ap_log_cerror (ログ出力)

関連データ構造

  • h2_c1_io: 入出力状態を保持する構造体。
  • h2_session: セッション全体を管理する構造体。
  • apr_bucket_brigade: 出力データを保持するバケットブリゲード。

失敗モードと境界条件

  • apr_brigade_create がメモリ不足等で失敗した場合の戻り値については、本関数内では明示的なエラーハンドリングを行わず、APR の挙動に依存する。

保守担当者向けメモ

  • io->cooldown_usecs は現在 0 に固定されている。
  • TLS 接続時の write_size は、ウォームアップロジックの要件（Jira TS-2503 参照）に基づき、初期値 WRITE_SIZE_INITIAL または WRITE_SIZE_MAX が選択される。
