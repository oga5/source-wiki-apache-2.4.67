conn_state_e

定義

  • 種別: enum
  • 定義箇所: include/httpd.h:1262

目的

Apache HTTP Server におけるクライアント接続の現在の状態を定義し、MPM（マルチプロセッシングモジュール）および各モジュール間での状態遷移を管理する。

シグネチャまたは形状

列挙型であり、以下の状態を保持する：

  • CONN_STATE_KEEPALIVE
  • CONN_STATE_PROCESSING
  • CONN_STATE_HANDLER
  • CONN_STATE_WRITE_COMPLETION
  • CONN_STATE_SUSPENDED
  • CONN_STATE_LINGER
  • CONN_STATE_LINGER_NORMAL
  • CONN_STATE_LINGER_SHORT
  • CONN_STATE_ASYNC_WAITIO
  • CONN_STATE_NUM

入力と出力

  • 入力: 接続処理の各フェーズにおける状態遷移。
  • 出力: MPM が接続をどのように扱うか（待機、処理、終了、非同期 IO 待ちなど）を決定する指標。

前提条件と不変条件

  • CONN_STATE_ASYNC_WAITIO を返す場合、接続の c->cs->sense に CONN_SENSE_WANT_READ または CONN_SENSE_WANT_WRITE が設定されている必要がある。
  • CONN_STATE_NUM は状態の総数を示すため、エイリアス定義よりも前に配置される必要がある。

アルゴリズムまたは振る舞い

  • CONN_STATE_KEEPALIVE: MPM が KeepAliveTimeout を使用して接続を維持する。
  • CONN_STATE_PROCESSING: process_connection フックによる処理中。
  • CONN_STATE_HANDLER: モジュールのハンドラによる処理中。
  • CONN_STATE_WRITE_COMPLETION: CONN_STATE_KEEPALIVE 移行前に MPM がフラッシュを行う。
  • CONN_STATE_SUSPENDED: ap_run_resume_suspended() が呼び出されるまで MPM 内で中断される。
  • CONN_STATE_LINGER: MPM がフラッシュ後に接続を遅延クローズする。
  • CONN_STATE_ASYNC_WAITIO: 設定されたタイムアウトに基づき、読み取りまたは書き込み可能になるまで待機する。

呼び出し元と呼び出し先

  • 呼び出し元: include/httpd.h:1295 にて conn_rec 構造体内の state フィールドとして参照されている。
  • 呼び出し先: MPM 実装および接続処理フック（具体的な関数呼び出し関係はソース抜粋からは不明）。

関連データ構造

  • conn_rec: 接続情報を保持する構造体。conn_state_e 型のメンバを持つ。

失敗モードと境界条件

  • 状態遷移の不整合が発生した場合、MPM が接続を適切にクローズできず、リソースリークやタイムアウト遅延を引き起こす可能性がある。

保守担当者向けメモ

  • CONN_STATE_CHECK_REQUEST_LINE_READABLE および CONN_STATE_READ_REQUEST_LINE はレガシーなエイリアスである。
  • 新しい状態を追加する場合は、CONN_STATE_NUM の位置に注意すること。
  • ap_mmn.h に記載されている通り、特定の状態追加はモジュール API バージョン（MMN）の更新を伴う場合がある。
