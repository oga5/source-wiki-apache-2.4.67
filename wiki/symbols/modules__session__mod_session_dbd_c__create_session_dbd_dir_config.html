create_session_dbd_dir_config

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_dbd.c:493

目的

mod_session_dbd モジュールで使用されるディレクトリ単位の設定構造体（session_dbd_dir_conf）を初期化し、デフォルト値を割り当てる。

シグネチャまたは形状

static void *create_session_dbd_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • p: 設定構造体の割り当てに使用するメモリプール。
  • dummy: Apache 設定システムから渡される未使用の引数。
  • **出力**: 初期化済みの session_dbd_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体の全メンバはゼロ初期化された状態で開始される。
  • 戻り値の構造体は、Apache の設定マージプロセスにおいて後続の処理で使用される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して session_dbd_dir_conf サイズのメモリをプールから確保する。 2. remove フラグを 1（デフォルトで有効）に設定する。 3. データベース操作用のプリペアドステートメントラベルに以下のデフォルト値を割り当てる。

  • selectlabel: "selectsession"
  • insertlabel: "insertsession"
  • updatelabel: "updatesession"
  • deletelabel: "deletesession"

    • 4. 初期化した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_dbd.c:643（command_rec 構造体内のディレクトリ設定作成ハンドラとして登録）。
  • **呼び出し先**: apr_pcalloc（APR メモリ管理関数）。

関連データ構造

  • session_dbd_dir_conf: 本関数が初期化する、SQL クエリラベルやクッキー設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボートする可能性がある（APR の標準的な挙動）。
  • dummy 引数は使用されないため、呼び出し側の値は影響しない。

保守担当者向けメモ

  • 本関数で設定されるラベル名は、mod_dbd で定義された SQL クエリと一致している必要がある。
  • 設定の変更が必要な場合は、merge_session_dbd_dir_config も併せて確認すること。
