ap_watchdog_t

定義

  • 種別: struct（不透明な構造体）
  • 定義箇所: modules/core/mod_watchdog.h:79

目的

Apache HTTP Server におけるバックグラウンドタスク実行のための「ウォッチドッグ」インスタンスを保持する構造体です。プロセスやスレッドを跨いだタスク管理の抽象化レイヤーとして機能します。

シグネチャまたは形状

typedef struct ap_watchdog_t ap_watchdog_t;

※不透明な構造体として定義されており、内部メンバは公開されていません。

入力と出力

  • **入力**: ap_watchdog_get_instance を通じて取得されます。
  • **出力**: 登録されたコールバック関数に対し、AP_WATCHDOG_STATE_*（STARTING, RUNNING, STOPPING）の状態を通知します。

前提条件と不変条件

  • mod_watchdog がロードされている環境で利用可能です。
  • インスタンスの取得には ap_watchdog_get_instance を使用します。
  • シングルトンモードで使用する場合、内部でプロセス間ミューテックスが使用されるため、コールバック内での長時間のロック保持はデッドロックの原因となります。

アルゴリズムまたは振る舞い

1. **初期化**: ap_watchdog_get_instance によりインスタンスが生成・取得されます。 2. **登録**: ap_watchdog_register_callback を介して、実行間隔とコールバック関数が紐付けられます。 3. **実行**: 内部タイマーに基づき、ウォッチドッグスレッドが登録されたコールバックを適切な状態（RUNNING 等）で呼び出します。 4. **終了**: プロセス終了時に STOPPING 状態が通知され、クリーンアップが実行されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • modules/cluster/mod_heartmonitor.c
  • modules/core/mod_watchdog.c（内部実装による管理）
  • **呼び出し先**:
  • 登録されたコールバック関数（ap_watchdog_callback_fn_t 型）。
  • apr_pcalloc（インスタンス生成時）。

関連データ構造

  • ap_watchdog_callback_fn_t: 定期実行されるコールバック関数の型定義。
  • AP_WATCHDOG_STATE_*: 状態遷移を定義する列挙型。
  • apr_pool_t: インスタンスのメモリ管理に使用。

失敗モードと境界条件

  • ap_watchdog_get_instance が失敗した場合、有効なインスタンスは取得できません。
  • コールバック内での重い処理は、ウォッチドッグの定期実行タイミングに遅延を生じさせる可能性があります。

保守担当者向けメモ

  • 本構造体は不透明な型として定義されているため、直接メンバにアクセスせず、提供されている API（ap_watchdog_get_instance, ap_watchdog_register_callback 等）を介して操作してください。
  • APR_DECLARE_OPTIONAL_FN を使用しているため、モジュールのロード状態に依存しない安全な呼び出しが可能です。
