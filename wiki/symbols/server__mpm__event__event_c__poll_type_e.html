poll_type_e

定義

  • 種別: enum
  • 定義箇所: server/mpm/event/event.c:359

目的

Event MPM において、apr_pollset で監視されるイベントの発生源や種類を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
    PT_CSD,
    PT_ACCEPT
} poll_type_e;

入力と出力

  • **値**:
  • PT_CSD: 接続状態（Client Socket Descriptor）に関連するイベント。
  • PT_ACCEPT: 新規接続の受け入れに関連するイベント。

前提条件と不変条件

  • apr_pollset に登録される各オブジェクトの識別子として利用されます。

アルゴリズムまたは振る舞い

  • apr_pollset からイベントが返された際、そのイベントに関連付けられた poll_type_e を確認することで、リスナーまたはワーカーが適切な後続処理（接続の受け入れ、または既存接続のデータ読み込み等）へディスパッチするために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照として server/mpm/event/event.c:367 にて poll_type_e type; というメンバ変数定義での利用が確認されています。
  • **呼び出し先**: 提供されたソース excerpt には、具体的な関数呼び出しや代入ロジックは含まれていません。

関連データ構造

  • event_conn_state_t: 接続状態を保持する構造体であり、この中で poll_type_e 型のメンバが保持されていることが示唆されています。

失敗モードと境界条件

  • 提供されたソース excerpt には、失敗モードや境界条件に関する記述はありません。

保守担当者向けメモ
