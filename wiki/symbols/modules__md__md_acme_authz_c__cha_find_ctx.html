cha_find_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme_authz.c:573

目的

ACME 認証プロセスにおいて、サーバーから提示されたチャレンジリストの中から、特定の条件（チャレンジタイプ）に合致するものを検索・保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    apr_pool_t *p;
    const char *type;
    md_acme_authz_cha_t *accepted;
    apr_array_header_t *offered;
} cha_find_ctx;

入力と出力

  • **入力**: 検索対象のチャレンジタイプ (type) および検索に使用するメモリプール (p)。
  • **出力**: 検索結果として合致したチャレンジ (accepted)、またはサーバーから提供された全チャレンジのリスト (offered) を保持します。

前提条件と不変条件

  • p は、この構造体および保持するデータが生存する期間中有効な apr_pool_t である必要があります。
  • offered は、ACME サーバーから取得したチャレンジの配列を指すことが想定されています。

アルゴリズムまたは振る舞い

本構造体は、チャレンジ検索処理における「バトン（状態保持オブジェクト）」として機能します。検索関数（コールバック等）がこの構造体を参照し、offered 配列を走査して type に一致するチャレンジを accepted に格納するフローで使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_authz.c 内の検索処理（582, 594, 612 行目付近）にて、ローカル変数としてインスタンス化および参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • md_acme_authz_cha_t: 検索結果として格納される個別のチャレンジ情報。
  • apr_array_header_t: サーバーから提供されたチャレンジのリストを保持。

失敗モードと境界条件

  • offered が空、または type に一致するチャレンジが存在しない場合、accepted は NULL のままとなります。
  • メモリプール p が破棄された場合、本構造体および accepted が指すデータへのアクセスは無効となります。

保守担当者向けメモ

  • 本構造体は md_acme_authz.c 内の特定の検索ロジックに特化しています。将来的に検索条件が複雑化する場合、type 以外の検索基準（ステータス等）を追加する可能性があります。
