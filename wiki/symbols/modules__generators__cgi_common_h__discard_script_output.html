discard_script_output

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:47

目的

CGIスクリプトから出力されたバケットブリゲード（apr_bucket_brigade）の内容を読み取り、破棄することで、スクリプトの出力を消費・無効化します。

シグネチャまたは形状

static void discard_script_output(apr_bucket_brigade *bb)

入力と出力

  • **入力**: bb (apr_bucket_brigade *) - 破棄対象のバケットが格納されたブリゲード。
  • **出力**: なし（副作用としてブリゲード内のバケットが削除されます）。

前提条件と不変条件

  • bb は有効な apr_bucket_brigade である必要があります。
  • EOS（End of Stream）バケットに到達するか、ブリゲードが空になるまで処理を継続します。

アルゴリズムまたは振る舞い

1. ブリゲードの先頭から順にバケットを走査します。 2. EOSバケットに到達するまで、各バケットに対して apr_bucket_read を実行し、データを読み取ります。 3. 読み取ったバケットを apr_bucket_delete で削除します。 4. apr_bucket_read でエラーが発生した場合はループを中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/cgi_common.h (行487, 516)
  • modules/generators/mod_cgi.c (行215)
  • modules/generators/mod_cgid.c (行1215)
  • **呼び出し先**:
  • apr_bucket_read
  • apr_bucket_delete
  • APR_BRIGADE_FIRST / APR_BRIGADE_SENTINEL / APR_BUCKET_IS_EOS (マクロ)

関連データ構造

  • apr_bucket_brigade: バケットを保持するコンテナ。
  • apr_bucket: データの断片を保持する構造体。

失敗モードと境界条件

  • apr_bucket_read がエラーを返した場合、処理を中断して終了します。
  • ブリゲードが空の場合、ループは実行されません。

保守担当者向けメモ

  • 本関数は mod_cgi および mod_cgid の両方から利用されています。
  • apr_bucket_read に APR_BLOCK_READ を指定しているため、バケットの読み取りが完了するまでブロックする可能性があります。
