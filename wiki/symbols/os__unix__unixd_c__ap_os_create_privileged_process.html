ap_os_create_privileged_process

定義

  • 種別: function
  • 定義箇所: os/unix/unixd.c:197

目的

リクエストに関連付けられたユーザー権限情報に基づき、外部プロセスを生成する。suexec が適用可能な場合はその権限で、適用不可の場合は標準的なプロセス生成を行う。

シグネチャまたは形状

apr_status_t ap_os_create_privileged_process(
    const request_rec *r,
    apr_proc_t *newproc, 
    const char *progname,
    const char * const *args,
    const char * const *env,
    apr_procattr_t *attr, 
    apr_pool_t *p)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • newproc: 生成されるプロセスを格納する apr_proc_t 構造体へのポインタ。
  • progname: 実行するプログラムのパス。
  • args: 引数リスト。
  • env: 環境変数リスト。
  • attr: プロセス属性（apr_procattr_t）。
  • p: メモリ割り当てに使用するプール。
  • **出力**:
  • 成功時は APR_SUCCESS、失敗時は APR エラーコードを返す。

前提条件と不変条件

  • ap_run_get_suexec_identity フックが利用可能であること。
  • newproc および attr は有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. ap_run_get_suexec_identity(r) を呼び出し、対象リクエストに対する suexec 用のユーザー/グループ識別子（ugid）を取得する。 2. ugid が NULL の場合、apr_proc_create を直接呼び出し、標準的なプロセス生成を行う。 3. ugid が取得できた場合、ap_unix_create_privileged_process を呼び出し、suexec を経由した特権プロセス生成を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c
  • modules/generators/mod_cgid.c
  • **呼び出し先**:
  • ap_run_get_suexec_identity (フック)
  • apr_proc_create
  • ap_unix_create_privileged_process

関連データ構造

  • request_rec: リクエスト情報。
  • apr_proc_t: プロセス管理構造体。
  • ap_unix_identity_t: suexec 用のユーザー/グループ識別情報。

失敗モードと境界条件

  • ap_run_get_suexec_identity が NULL を返した場合、通常のプロセス生成処理にフォールバックする。
  • apr_proc_create または ap_unix_create_privileged_process が内部的に失敗した場合、そのエラーコードがそのまま返される。

保守担当者向けメモ

  • 本関数は suexec の有無を動的に判定するラッパーである。
  • ap_unix_create_privileged_process の実装詳細については os/unix/unixd.c の後続定義を参照のこと。
