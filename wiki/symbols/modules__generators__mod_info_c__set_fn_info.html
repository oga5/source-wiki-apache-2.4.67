set_fn_info

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_info.c:126

目的

リクエスト処理中または設定ダンプ時に、関数名（name）をモジュール固有のコンテキストまたはグローバル変数に保存する。

シグネチャまたは形状

static void set_fn_info(request_rec *r, const char *name)

入力と出力

  • r: request_rec 構造体へのポインタ。リクエスト処理中であることを示す。
  • name: 保存対象の関数名を示す文字列。
  • 出力: なし（副作用として状態を更新）。

前提条件と不変条件

  • r が非 NULL の場合、info_module をキーとして r->request_config に name が設定される。
  • r が NULL の場合、グローバル変数 dump_config_fn_info に name が代入される。

アルゴリズムまたは振る舞い

1. r が有効な場合、ap_set_module_config を呼び出し、現在のリクエスト設定に name を関連付ける。 2. r が NULL の場合（-DDUMP_CONFIG 時など）、静的変数 dump_config_fn_info を更新する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/generators/mod_info.c:159
  • modules/generators/mod_info.c:253
  • 呼び出し先:
  • ap_set_module_config（Apache HTTP Server API）

関連データ構造

  • request_rec: リクエストごとの設定を保持する構造体。
  • info_module: 本モジュールの識別子。
  • dump_config_fn_info: 設定ダンプ時に使用されるグローバル変数。

失敗モードと境界条件

  • r が NULL の場合、グローバル変数への書き込みが行われるため、マルチスレッド環境下での同時実行には注意が必要である（ただし、本関数は主に設定読み込みフェーズで使用される）。

保守担当者向けメモ

  • 本関数はリクエスト処理とコマンドラインからの設定ダンプの両方に対応する設計となっている。
  • r が NULL の場合の挙動は、mod_info.c 内の DUMP_CONFIG 対応ロジックと密接に関連している。
