inc_rec

定義

  • 種別: function
  • 定義箇所: server/util_expr_eval.c:76

目的

式評価エンジンにおける再帰呼び出しの深さを監視し、スタックオーバーフローを防止する。

シグネチャまたは形状

static int inc_rec(ap_expr_eval_ctx_t *ctx)

入力と出力

  • **入力**: ap_expr_eval_ctx_t *ctx（評価コンテキスト）
  • **出力**:
  • 0: 再帰制限内であり、評価を継続可能。
  • 1: 再帰制限に到達し、評価を中断すべき。

前提条件と不変条件

  • ctx->reclvl は現在の再帰深度を保持している必要がある。
  • AP_EXPR_MAX_RECURSION（定数 20）が再帰の許容上限として定義されている。

アルゴリズムまたは振る舞い

1. 現在の再帰深度 ctx->reclvl が AP_EXPR_MAX_RECURSION 未満であるかを確認する。 2. 制限内の場合、ctx->reclvl をインクリメントし、0 を返す。 3. 制限に達している場合、以下の処理を行う：

  • ctx->err にエラーメッセージ "Recursion limit reached" を格納する。
  • ctx->reclvl を INT_MAX に設定し、以降の評価をショートサーキット（強制終了）させる。
  • 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_expr_eval.c:92
  • server/util_expr_eval.c:741
  • **呼び出し先**:
  • 提供されたソース excerpt 内に呼び出し先は存在しない。

関連データ構造

  • ap_expr_eval_ctx_t: 式評価のコンテキストを保持する構造体。reclvl（現在の深度）および err（エラーメッセージ格納用）メンバを含む。

失敗モードと境界条件

  • **再帰制限到達**: ctx->reclvl が AP_EXPR_MAX_RECURSION に達すると、エラーメッセージが設定され、評価が停止する。
  • **ショートサーキット**: 制限到達時に ctx->reclvl を INT_MAX に設定することで、後続の再帰的な評価呼び出しを無効化する。

保守担当者向けメモ

  • 本関数は ap_expr エンジンの再帰的な AST 走査の直前に呼び出されるガード関数である。
  • 評価の中断を判定する唯一のロジックであるため、再帰制限を緩和または厳格化する場合は AP_EXPR_MAX_RECURSION の定義を確認すること。
