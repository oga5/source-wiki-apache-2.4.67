reg_md_iter

定義

  • 種別: function
  • 定義箇所: modules/md/md_reg.c:309

目的

管理対象ドメイン（md_t）のリストを反復処理し、各ドメインに対してコールバック関数を実行するイテレータです。

シグネチャまたは形状

static int reg_md_iter(void *baton, md_store_t *store, md_t *md, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • baton: reg_do_ctx 構造体へのポインタ。コールバック関数とレジストリコンテキストを保持。
  • store: md_store_t インスタンス（本関数内では未使用）。
  • md: 現在処理中の md_t 構造体。
  • ptemp: 一時的なメモリ割り当て用プール。
  • **出力**:
  • int: コールバック関数の戻り値、または除外対象の場合は 1。

前提条件と不変条件

  • baton は reg_do_ctx 型としてキャスト可能であること。
  • md は有効なドメイン構造体であること。

アルゴリズムまたは振る舞い

1. baton を reg_do_ctx 型のコンテキストとして取得します。 2. ctx->exclude が指定されており、かつ現在の md->name と一致する場合、そのドメインをスキップして 1 を返します。 3. 一致しない場合、state_init を呼び出して対象ドメインの証明書状態を初期化・更新します。 4. 最後に ctx->cb に格納されたコールバック関数を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_reg.c:329 において md_store_md_iter の引数として参照されています。
  • **呼び出し先**:
  • state_init: ドメインの状態初期化。
  • ctx->cb: reg_do_ctx 内に保持されたコールバック関数。

関連データ構造

  • reg_do_ctx: イテレータの実行コンテキストを保持する構造体。
  • md_t: 管理対象ドメインの定義。
  • md_store_t: 永続化層のインターフェース。

失敗モードと境界条件

  • ctx->cb が返す値に依存します。
  • md->name が NULL の場合、strcmp で予期せぬ動作をする可能性があります（ソースコード上での明示的なチェックは存在しません）。

保守担当者向けメモ

  • 本関数は md_store_md_iter から呼び出されることを前提としています。
  • store 引数は現在使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
