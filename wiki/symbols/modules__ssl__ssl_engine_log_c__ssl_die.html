ssl_die

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_log.c:66

目的

mod_ssl の初期化プロセスにおいて致命的なエラーが発生した際、ログを出力して終了処理を行うためのユーティリティ関数です。

シグネチャまたは形状

apr_status_t ssl_die(server_rec *s)

入力と出力

  • **入力**: server_rec *s (現在のサーバー設定構造体)
  • **出力**: APR_EGENERAL (常にこの汎用エラーコードを返す)

前提条件と不変条件

  • 引数 s が NULL の場合、標準的なエラーメッセージのみが出力されます。
  • s が有効な仮想ホスト (s->is_virtual) であり、かつ s->error_fname が設定されている場合、そのファイルパスがログに含まれます。

アルゴリズムまたは振る舞い

1. サーバー構造体 s の状態を確認します。 2. 仮想ホストかつエラーログファイル名が定義されている場合、ap_log_error を使用して、該当するログファイル名を含む致命的エラーメッセージを出力します。 3. それ以外の場合は、標準的な致命的エラーメッセージを出力します。 4. 最後に APR_EGENERAL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c 内の複数の初期化フェーズ（行 455, 607, 621, 659, 698, 706, 743, 811, 869, 1097, 1112, 1163）から呼び出されます。
  • **呼び出し先**: Apache HTTP Server のロギング API (ap_log_error) およびパス解決ユーティリティ (ap_server_root_relative) を呼び出します。

関連データ構造

  • server_rec: Apache のサーバー設定を保持する構造体。

失敗モードと境界条件

  • s が NULL の場合、仮想ホスト固有のログファイルパスは出力されません。
  • ログ出力自体が失敗した場合の挙動は ap_log_error の実装に依存します。

保守担当者向けメモ

  • 本関数は mod_ssl の初期化失敗時に一貫した終了ステータスを返すために使用されます。
  • ログメッセージには APLOGNO マクロによる一意な識別子（02311, 02312）が付与されています。
