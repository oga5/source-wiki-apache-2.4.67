command_struct

定義

  • 種別: struct
  • 定義箇所: include/http_config.h:204

目的

Apache HTTP Server の設定ディレクティブを定義し、設定ファイル解析時に実行されるハンドラー関数や引数の制約を保持するための構造体です。

シグネチャまたは形状

struct command_struct {
    const char *name;
    cmd_func func;
    void *cmd_data;
    int req_override;
    enum cmd_how args_how;
    const char *errmsg;
};

入力と出力

  • **入力**: 設定ファイル解析時に ap_walk_config を通じて、ディレクティブ名、引数、およびコンテキスト情報がハンドラー関数（func）に渡されます。
  • **出力**: ハンドラー関数は、設定の適用結果やエラーメッセージを返します。

前提条件と不変条件

  • name はディレクティブを一意に識別する文字列である必要があります。
  • args_how は cmd_how 列挙型に従い、引数の解析方法を正しく指定する必要があります。
  • req_override は、ディレクティブが許可されるコンテキスト（.htaccess や <Directory> 等）を制御します。

アルゴリズムまたは振る舞い

1. 設定ファイル解析中にディレクティブ名が一致すると、対応する command_struct が特定されます。 2. args_how に基づいて引数が解析されます。 3. func フィールドに登録された関数が呼び出され、設定がサーバー構造体に反映されます。 4. 構文エラーが発生した場合、errmsg がエラーメッセージとして使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: include/http_config.h:203 にて command_rec 型としてエイリアス定義されています。
  • **呼び出し先**: サーバーの起動時および設定読み込み時に、設定解析エンジンから func が呼び出されます。具体的な呼び出し元・呼び出し先の詳細は、提供されたソース範囲外のため不明です。

関連データ構造

  • cmd_rec: 本構造体の別名（typedef）。
  • cmd_func: ディレクティブ実行用の関数ポインタ型。
  • cmd_how: 引数の解析方法を定義する列挙型。

失敗モードと境界条件

  • func が NULL の場合、または args_how の指定と実際の引数が一致しない場合、設定解析時にエラーが発生します。
  • req_override の制約に違反するコンテキストでディレクティブが使用された場合、設定エラーとなります。

保守担当者向けメモ

  • 本構造体は AP_INIT_* マクロ群を使用して初期化されることが一般的です。
  • cmd_data は、複数のディレクティブを単一の関数で処理する際に、関数へ渡す追加情報を保持するために使用されます。
