ap_get_server_module_loglevel

定義

  • 種別: function
  • 定義箇所: server/util_debug.c:137

目的

サーバーコンテキストにおいて、特定のモジュールに設定されたログレベルを取得します。モジュール固有のログレベルが未設定または無効な場合に、サーバー全体のデフォルトログレベルへフォールバックするロジックを提供します。

シグネチャまたは形状

int ap_get_server_module_loglevel(const server_rec *s, int module_index)

入力と出力

  • **入力**:
  • s: サーバー設定構造体 (server_rec) へのポインタ。
  • module_index: 対象モジュールを識別するインデックス値。
  • **出力**:
  • 解決されたログレベル（整数値）。

前提条件と不変条件

  • s は有効な server_rec 構造体を指している必要があります。
  • module_index は、サーバーのモジュール設定配列に対して有効な範囲内である必要があります。

アルゴリズムまたは振る舞い

1. 以下のいずれかの条件を満たす場合、s->log.level（サーバー全体のデフォルトログレベル）を返します。

  • module_index が負の値である。
  • s->log.module_levels が NULL である。
  • s->log.module_levels[module_index] が負の値である。

    • 2. 上記条件に該当しない場合、s->log.module_levels[module_index] に格納されているモジュール固有のログレベルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/log.c:1204 からの呼び出しが確認されています。
  • include/http_log.h 内のマクロ定義から参照されています。
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、構造体のメンバアクセスのみを行います。

関連データ構造

  • server_rec: サーバー設定を保持する構造体。log メンバを通じてログレベル設定にアクセスします。

失敗モードと境界条件

  • s が NULL の場合、デリファレンスによりクラッシュします。
  • module_index が s->log.module_levels 配列の確保サイズを超えている場合、メモリの不正アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数は http_config.h でマクロとして定義される場合があり、server/util_debug.c ではそのマクロを undef してから実体を定義しています。
  • ログレベルの解決ロジックは ap_get_conn_module_loglevel や ap_get_request_module_loglevel と同様のパターンに従っています。
