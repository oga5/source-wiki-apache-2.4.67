ap_fcgi_header_to_array

定義

  • 種別: function
  • 定義箇所: server/util_fcgi.c:26

目的

FastCGI プロトコル（RFC 3652）のヘッダー構造体 ap_fcgi_header を、ネットワーク転送に適したバイト配列形式へシリアライズします。

シグネチャまたは形状

void ap_fcgi_header_to_array(ap_fcgi_header *h, unsigned char a[])

入力と出力

  • **入力**:
  • h: 変換元の ap_fcgi_header 構造体へのポインタ。
  • a: 変換結果を格納する unsigned char 配列（呼び出し元が確保したバッファ）。
  • **出力**: なし（戻り値は void）。a が指すメモリ領域が更新されます。

前提条件と不変条件

  • a は、FastCGI ヘッダーのサイズ（8バイト）以上の容量を持つ配列である必要があります。
  • h は有効な ap_fcgi_header 構造体を指している必要があります。

アルゴリズムまたは振る舞い

ap_fcgi_header 構造体の各フィールドを、定義済みのオフセット定数に従ってバイト配列 a の各インデックスへ順次コピーします。

1. version を AP_FCGI_HDR_VERSION_OFFSET へ格納。 2. type を AP_FCGI_HDR_TYPE_OFFSET へ格納。 3. requestId の上位バイト（B1）と下位バイト（B0）をそれぞれのオフセットへ格納。 4. contentLength の上位バイト（B1）と下位バイト（B0）をそれぞれのオフセットへ格納。 5. paddingLength を AP_FCGI_HDR_PADDING_LEN_OFFSET へ格納。 6. reserved を AP_FCGI_HDR_RESERVED_OFFSET へ格納。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c
  • modules/proxy/mod_proxy_fcgi.c
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、メモリへの代入操作のみを実行します。

関連データ構造

  • ap_fcgi_header: FastCGI ヘッダーの各フィールドを保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーチェックを行いません。バッファ a のサイズが不足している場合、メモリ破壊が発生する可能性があります。

保守担当者向けメモ

  • 本関数はメモリの割り当てを行いません。呼び出し元が提供したバッファに対して直接書き込みを行います。
  • FastCGI プロトコルの仕様変更や、ヘッダー構造体のフィールド構成に変更があった場合は、オフセット定数との整合性を確認してください。
