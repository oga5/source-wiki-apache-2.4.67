ap_set_etag

定義

  • 種別: function
  • 定義箇所: modules/http/http_etag.c:367

目的

リクエストされたリソースのファイル情報に基づき ETag を生成し、レスポンスヘッダーへ自動的に設定する。

シグネチャまたは形状

void ap_set_etag(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報。r->finfo, r->filename, r->vlist_validator を使用)
  • **出力**: なし（副作用として r->headers_out または r->notes を更新）

前提条件と不変条件

  • r->finfo に有効なファイル情報が格納されていること。
  • etag_bits 設定が ETAG_NONE の場合、ETag は生成されず、後続のフィルターによる付与を抑制するために r->notes に no-etag が設定される。

アルゴリズムまたは振る舞い

1. request_rec の情報から etag_rec 構造体を構築する。 2. ap_make_etag_ex を呼び出し、設定（FileETag ディレクティブ等）に基づいた ETag 文字列を生成する。 3. 生成された ETag が空でない場合、apr_table_setn を使用して r->headers_out に "ETag" ヘッダーとして格納する。 4. 生成に失敗した（または生成されなかった）場合、r->notes に no-etag キーで "omit" を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_file_cache.c
  • modules/dav/fs/repos.c
  • modules/generators/mod_autoindex.c
  • modules/mappers/mod_negotiation.c
  • **呼び出し先**:
  • ap_make_etag_ex (ETag 文字列生成のコア関数)
  • apr_table_setn (ヘッダーまたはノートへの値設定)

関連データ構造

  • request_rec: リクエスト全体の状態を保持する構造体。
  • etag_rec: ap_make_etag_ex に渡すための ETag 生成用パラメータ構造体。

失敗モードと境界条件

  • ap_make_etag_ex が ETag を生成できない場合、r->headers_out には設定されず、r->notes に no-etag が記録される。
  • ファイル情報が不完全な場合、生成される ETag の精度が低下する可能性がある。

保守担当者向けメモ

  • 本関数は r->pool を使用してメモリを割り当てるため、リクエスト終了時に自動的に解放される。
  • r->vlist_validator が設定されている場合、ETag 生成時に考慮される。
  • make_digest_etag が内部的に呼び出される場合、ファイル内容の読み込みが発生するため、大きなファイルではパフォーマンスに影響を与える可能性がある。
