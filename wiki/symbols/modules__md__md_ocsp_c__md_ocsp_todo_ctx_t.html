md_ocsp_todo_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_ocsp.c:771

目的

OCSP ステータスの更新処理（更新が必要な証明書のリストアップと並行実行）を行う際に、処理のコンテキストを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    md_ocsp_reg_t *reg;           /* OCSP レジストリへの参照 */
    apr_array_header_t *todos;    /* 更新対象のリスト */
    apr_pool_t *ptemp;            /* 一時的なメモリ割り当て用プール */
    apr_time_t time;              /* 処理基準時刻 */
    int max_parallel;             /* 並行処理の最大数 */
} md_ocsp_todo_ctx_t;

入力と出力

  • 本構造体は、OCSP 更新処理のバトン（baton）として関数間で受け渡されます。
  • 入力として、レジストリ情報、更新対象の配列、および処理に必要なリソース（プール、時刻、並行数）を保持します。

前提条件と不変条件

  • reg は有効な md_ocsp_reg_t を指している必要があります。
  • ptemp は、このコンテキストに関連する一時的なメモリ割り当てのライフサイクルを管理します。

アルゴリズムまたは振る舞い

  • OCSP 更新処理の実行中、この構造体を通じて現在の処理状態や設定値が参照されます。
  • md_ocsp_renew 等の処理フローにおいて、更新対象の管理と並行実行の制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_ocsp.c 内の更新処理関数（md_ocsp_renew 関連のロジック）から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出す側ではなく、関数内で baton として受け取られ、内部の各フィールドが処理に使用されます。

関連データ構造

  • md_ocsp_reg_t: OCSP レジストリのグローバルコンテキスト。
  • apr_array_header_t: 更新対象を格納する配列。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • reg や todos が NULL の状態で処理が開始された場合、後続の参照でクラッシュする可能性があります。
  • max_parallel が不適切な値（0以下など）に設定された場合、並行処理の制御ロジックに影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体は md_ocsp.c 内の更新処理専用のコンテキストです。
  • baton として渡されるため、型キャストの整合性に注意してください。
