dialup_callback

定義

  • 種別: function
  • 定義箇所: modules/test/mod_dialup.c:103

目的

非同期リクエスト処理の再開点として機能し、タイマーイベント発生時に帯域制限に基づいたレスポンスデータの送信を継続する。

シグネチャまたは形状

static void dialup_callback(void *baton)

入力と出力

  • **入力**: baton (型: void *) - 実際には dialup_baton_t 構造体へのポインタ。リクエスト情報や送信状態を保持する。
  • **出力**: なし（void）。

前提条件と不変条件

  • baton は有効な dialup_baton_t を指している必要がある。
  • db->r->invoke_mtx による排他制御が必須である。

アルゴリズムまたは振る舞い

1. db->r->invoke_mtx をロックし、排他制御を行う。 2. dialup_send_pulse(db) を呼び出し、制限値分のデータを送信する。 3. 送信結果（status）に基づき以下のいずれかの処理を行う：

  • SUSPENDED: 1秒後に再度 dialup_callback を呼び出すよう ap_mpm_register_timed_callback を登録する。
  • DONE: リクエストの終了処理（ap_finalize_request_protocol および ap_process_request_after_handler）を実行する。
  • その他: エラーログを出力し、ap_die を呼び出してリクエストを終了させる。

    • 4. 最後に db->r->invoke_mtx をアンロックする。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_mpm_register_timed_callback を介して MPM（Multi-Processing Module）から呼び出される。
  • **呼び出し先**:
  • dialup_send_pulse
  • ap_mpm_register_timed_callback
  • ap_finalize_request_protocol
  • ap_process_request_after_handler
  • ap_log_rerror
  • ap_die
  • apr_thread_mutex_lock / apr_thread_mutex_unlock

関連データ構造

  • dialup_baton_t: 非同期処理のコンテキストを保持する構造体。

失敗モードと境界条件

  • dialup_send_pulse が SUSPENDED でも DONE でもない値を返した場合、エラーとして ap_die が呼び出される。
  • 排他制御（invoke_mtx）の取得に失敗した場合の挙動は、Apache のスレッドモデルに依存する。

保守担当者向けメモ

  • 本関数は SUSPENDED 状態を利用した非同期ループの一部である。
  • ap_finalize_request_protocol 呼び出し後の ap_process_request_after_handler は、リクエストのライフサイクルを適切に終了させるために重要である。
  • エラーハンドリングは最小限であるため、ログ出力の拡充が必要な場合は ap_log_rerror の箇所を修正すること。
