accept_mutex_init

定義

  • 種別: function
  • 定義箇所: test/time-sem.c:85

目的

プロセス間同期のための fcntl ロック用ファイルを初期化し、ロック操作に必要な構造体を設定する。

シグネチャまたは形状

void accept_mutex_init(void);

入力と出力

  • 入力: なし
  • 出力: なし（成功時）
  • 副作用: カレントディレクトリに test-lock-thing ファイルを作成し、即座に unlink する。グローバル変数 fcntl_fd にファイル記述子を格納する。

前提条件と不変条件

  • カレントディレクトリへの書き込み権限が必要。
  • fcntl ロック用の構造体 lock_it および unlock_it が、SEEK_SET を基準としたファイル全体（l_len = 0）を対象とするよう設定される。

アルゴリズムまたは振る舞い

1. lock_it および unlock_it 構造体に、排他ロック（F_WRLCK）およびロック解除（F_UNLCK）のためのパラメータを代入する。 2. open システムコールを用いて test-lock-thing を O_CREAT | O_WRONLY | O_EXCL モードで作成する。 3. ファイルオープンに失敗した場合は perror でエラーを出力し、exit(1) でプロセスを終了する。 4. unlink を呼び出し、ファイルシステム上の名前を削除する（ファイル記述子は保持される）。

呼び出し元と呼び出し先

  • 呼び出し元: test/time-sem.c 内の各テスト実行関数（main 等から間接的に参照される）。
  • 呼び出し先: open, printf, perror, fprintf, exit, unlink（すべて標準ライブラリ）。

関連データ構造

  • struct flock（lock_it, unlock_it）: fcntl ロックの範囲とタイプを定義。
  • fcntl_fd: ロック対象のファイル記述子を保持するグローバル変数。

失敗モードと境界条件

  • ファイル作成失敗: 同名のファイルが既に存在する場合や権限不足により open が -1 を返すと、エラーメッセージを表示して即時終了する。

保守担当者向けメモ

  • 本関数はベンチマークツールの一部であり、エラー発生時に exit(1) を呼び出す設計となっている。
  • unlink 後のファイル記述子に対する fcntl 操作が、対象OSで正しく排他制御として機能することを前提としている。
