identity_count

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:318

目的

セッションデータをシリアライズする際に必要なバッファサイズを計算するため、テーブル内の各キーと値の長さを集計する。

シグネチャまたは形状

static int identity_count(void *v, const char *key, const char *val)

入力と出力

  • **入力**:
  • v: 集計結果を格納する apr_size_t 型のポインタ。
  • key: 現在処理中のキー文字列。
  • val: 現在処理中の値文字列。
  • **出力**:
  • 常に 1 を返し、apr_table_do によるテーブル走査の継続を指示する。

前提条件と不変条件

  • v は apr_size_t 型の有効なメモリ領域を指している必要がある。
  • key および val はヌル終端文字列であること。

アルゴリズムまたは振る舞い

1. v を apr_size_t * にキャストし、現在の合計値を取得する。 2. キーと値の長さそれぞれに対して3倍の領域（URLエンコード等のオーバーヘッドを想定）を確保し、区切り文字分として 2 を加算する。 3. 計算結果を *count に加算して更新する。 4. 1 を返して次のエントリの処理へ移行する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session.c:366 において apr_table_do のコールバックとして呼び出される。
  • **呼び出し先**: strlen（標準Cライブラリ）。

関連データ構造

  • apr_table_t: セッションエントリを保持するテーブル構造。

失敗モードと境界条件

  • key または val が NULL の場合、strlen によりセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は session_identity_encode がシリアライズ用バッファを割り当てる際のサイズ見積もりに使用される。
  • 計算式内の 3 という係数は、URLエンコードによる最大膨張率を考慮したものである。
