cache_the_file

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_file_cache.c:125

目的

指定されたファイルをサーバー起動時に読み込み、ファイルハンドルまたはメモリマップ（mmap）としてキャッシュに登録する。

シグネチャまたは形状

static void cache_the_file(cmd_parms *cmd, const char *filename, int mmap)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • filename: キャッシュ対象のファイルパス。
  • mmap: メモリマップを使用するかどうかを示すフラグ（0: ファイルハンドル、1: mmap）。
  • **出力**: なし（void）。内部で a_server_config のハッシュテーブルにキャッシュ情報を格納する。

前提条件と不変条件

  • ファイルは AP_MAX_SENDFILE サイズ以下であること。
  • ファイルは通常のファイル（APR_REG）であること。
  • サーバーの生存期間中、キャッシュされたリソースは保持される。

アルゴリズムまたは振る舞い

1. ap_server_root_relative で絶対パスを解決する。 2. apr_stat でファイル情報を取得し、存在確認とサイズ制限チェックを行う。 3. apr_file_open でファイルを開き、apr_file_inherit_set で継承可能にする。 4. mmap フラグが真の場合、apr_mmap_create を実行し、成功後にファイルハンドルを閉じる。 5. mmap フラグが偽の場合、ファイルハンドルを保持する。 6. apr_rfc822_date で最終更新日時を、apr_snprintf でサイズを文字列化し、a_file 構造体に格納する。 7. サーバー設定のハッシュテーブル（sconf->fileht）に a_file を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • cachefilehandle (引数 mmap=0)
  • cachefilemmap (引数 mmap=1)
  • **呼び出し先**:
  • ap_server_root_relative, ap_log_error, apr_stat, apr_file_open, apr_file_inherit_set, apr_pcalloc, apr_mmap_create, apr_file_close, apr_rfc822_date, apr_snprintf, ap_get_module_config, apr_hash_set

関連データ構造

  • a_file: キャッシュされたファイル情報（ハンドル、mmap、メタデータ文字列）を保持。
  • a_server_config: サーバーごとのキャッシュハッシュテーブルを保持。
  • cmd_parms: 設定コマンドのコンテキスト。

失敗モードと境界条件

  • パスが不正な場合、または stat に失敗した場合は警告ログを出力して終了する。
  • ファイルが通常ファイルでない場合、またはサイズが AP_MAX_SENDFILE を超える場合はスキップする。
  • ファイルのオープンや mmap の作成に失敗した場合は、エラーログを出力して終了する。

保守担当者向けメモ

  • mmap 使用時、ファイルサイズが apr_size_t の上限を超える場合の制約について注意が必要。
  • キャッシュされたファイルはサーバー再起動まで更新されないため、運用上の制限事項として明記すること。
