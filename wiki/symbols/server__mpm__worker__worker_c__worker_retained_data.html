worker_retained_data

定義

  • 種別: struct
  • 定義箇所: server/mpm/worker/worker.c:145

目的

Apache HTTP Server の worker MPM において、再起動（Graceful restart）を跨いで保持すべき状態情報を管理するための構造体です。プロセス管理の最適化や、子プロセスの生成レート制御に必要な情報を永続化します。

シグネチャまたは形状

typedef struct worker_retained_data {
    ap_unixd_mpm_retained_data *mpm;
    int first_server_limit;
    int first_thread_limit;
    int sick_child_detected;
    int maxclients_reported;
    int near_maxclients_reported;
    int max_daemons_limit;
    int *idle_spawn_rate;
    int hold_off_on_exponential_spawning;
} worker_retained_data;

入力と出力

  • 本構造体は関数ではなくデータ型です。
  • idle_spawn_rate はリスナーバケットごとに割り当てられた整数配列を指します。

前提条件と不変条件

  • max_daemons_limit は、過去の再起動を含めた全期間で割り当てられた最大の子プロセススロット数を保持します。
  • idle_spawn_rate は、メンテナンスサイクルごとに調整され、最大値 MAX_SPAWN_RATE (32) に制限されます。

アルゴリズムまたは振る舞い

  • **プロセス管理の最適化**: max_daemons_limit を使用して、スコアボード全体をスキャンするルーチンの実行範囲を限定します。
  • **指数的生成制御**: idle_spawn_rate は、アイドル状態のサーバーが不足している場合に次回のサイクルで生成する子プロセスの数を決定します。生成の必要がないサイクルが続くとリセットされます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の詳細は、提供されたソース抜粋からは特定できません。
  • server/mpm/worker/worker.c:172 にて、この構造体型の静的ポインタ変数 retained が宣言されています。

関連データ構造

  • ap_unixd_mpm_retained_data: Unix MPM 共通の保持データ。
  • scoreboard: プロセスおよびスレッドの状態を記録する共有メモリ構造。

失敗モードと境界条件

  • MaxRequestWorkers の設定変更が Graceful restart を跨いで行われる際、max_daemons_limit が適切に追跡されていない場合、スコアボードのスキャン効率が低下する可能性があります。

保守担当者向けメモ

  • 本構造体は、MPM の再読み込み時にも生存し続ける必要があるため、メモリ管理には注意が必要です。
  • idle_spawn_rate の配列サイズは、リスナーバケット数に依存します。
