ap_prep_lingering_close

定義

  • 種別: function
  • 定義箇所: server/connection.c:98

目的

接続を閉じる直前に、プロトコルハンドラに対して最終的な処理機会を提供し、スコアボード上の接続状態を「終了中（SERVER_CLOSING）」へ更新するために使用されます。

シグネチャまたは形状

int ap_prep_lingering_close(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（対象となる接続の構造体）
  • **出力**: int（常に 0 を返します）

前提条件と不変条件

  • 引数 c は有効な conn_rec 構造体を指している必要があります。
  • c->sbh が存在する場合、スコアボードの更新が行われます。

アルゴリズムまたは振る舞い

1. ap_run_pre_close_connection(c) を呼び出し、登録されているすべての pre_close_connection フックを実行します。これにより、プロトコルハンドラが終了前に必要な処理を行う機会を得ます。 2. 接続に関連付けられたスコアボードハンドル c->sbh が存在する場合、ap_update_child_status を呼び出し、当該接続の状態を SERVER_CLOSING に更新します。 3. 処理完了後、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/connection.c:127（ap_lingering_close 内での呼び出しが確認されています）
  • **呼び出し先**:
  • ap_run_pre_close_connection（フック関数）
  • ap_update_child_status（スコアボード更新関数）

関連データ構造

  • conn_rec: 接続のライフサイクルと状態を保持する構造体。
  • sbh: スコアボードハンドル。接続の現在の状態を追跡するために使用されます。

失敗モードと境界条件

  • c->sbh が NULL の場合、スコアボードの更新はスキップされますが、関数は正常に終了します。
  • フック関数 ap_run_pre_close_connection 内で発生するエラーについては、本関数側では明示的なエラーハンドリングを行わず、常に 0 を返します。

保守担当者向けメモ

  • 本関数は、ap_lingering_close による実際のソケットクローズ処理の直前に実行される準備フェーズです。
  • 接続終了時のログ出力や統計情報の確定を行いたい場合は、pre_close_connection フックを利用してください。
