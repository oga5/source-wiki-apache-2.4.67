lua_table_set

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_apr.c:38

目的

Lua スクリプトから Apache Portable Runtime (APR) のテーブル（apr_table_t）に対してキーと値を設定、または削除するためのインターフェースを提供します。

シグネチャまたは形状

static int lua_table_set(lua_State *L);

入力と出力

  • **入力**:
  • スタック第1引数: req_table_t 構造体（ap_lua_check_apr_table で検証済み）。
  • スタック第2引数: 設定対象のキー（文字列）。
  • スタック第3引数: 設定する値（文字列、または nil）。
  • **出力**:
  • 戻り値: 0（Lua スタックに値を返さない）。

前提条件と不変条件

  • 第1引数は ap_lua_check_apr_table が成功する有効な req_table_t である必要があります。
  • notes 以外のテーブルに対しては、HTTP ヘッダーインジェクションを防ぐため、値に含まれる改行コード（\n）はスペースに置換されます。

アルゴリズムまたは振る舞い

1. ap_lua_check_apr_table を呼び出し、Lua スタックから req_table_t を取得します。 2. 第3引数が nil の場合、apr_table_unset を呼び出して指定キーを削除し、終了します。 3. 第3引数が文字列の場合、テーブル名が notes でないか確認します。 4. notes 以外で値に改行が含まれる場合、apr_pstrdup で値を複製し、すべての改行をスペースに置換します。その後、警告ログを出力し、置換後の値を apr_table_set で設定します。 5. 改行が含まれない場合、または notes テーブルの場合は、そのまま apr_table_set で値を設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_apr.c:82（lua_table_methods テーブル内での set メソッド登録）。
  • **呼び出し先**:
  • ap_lua_check_apr_table
  • luaL_checkstring / luaL_optlstring
  • apr_table_unset / apr_table_set
  • apr_pstrdup
  • ap_strchr / ap_strchr_c
  • ap_log_rerror

関連データ構造

  • req_table_t: APR テーブルと関連するリクエスト情報を保持するラッパー構造体。
  • apr_table_t: APR が提供するキー・値ペアのテーブル構造。

失敗モードと境界条件

  • **値が nil**: 指定されたキーがテーブルから削除されます。
  • **改行コードの混入**: notes 以外のテーブルで値に \n が含まれる場合、セキュリティ上の理由からスペースに置換され、警告ログが記録されます。
  • **メモリ不足**: apr_pstrdup 実行時にメモリ不足が発生した場合、APR のメモリ管理ポリシーに従い異常終了する可能性があります。

保守担当者向けメモ

  • 本関数は __newindex メタメソッドとして利用されることを想定しています。
  • t->r（リクエスト構造体）は notes テーブル以外では有効である前提で動作しています。notes テーブル操作時は t->r が NULL になる可能性がありますが、ロジック内で notes のチェックを優先しているため安全です。
