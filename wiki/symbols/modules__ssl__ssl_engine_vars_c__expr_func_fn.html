expr_func_fn

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:93

目的

Apache HTTP Server の設定式（ap_expr）評価コンテキストにおいて、SSL 関連の変数や関数を解決するためのラッパー関数です。

シグネチャまたは形状

static const char *expr_func_fn(ap_expr_eval_ctx_t *ctx, const void *data, const char *arg)

入力と出力

  • **入力**:
  • ctx: ap_expr_eval_ctx_t 型の評価コンテキスト。
  • data: 使用されていない引数。
  • arg: 解決対象となる変数名または関数引数（const char *）。
  • **出力**:
  • 解決された値を示す const char * 型の文字列。arg が NULL の場合は NULL を返します。

前提条件と不変条件

  • ctx が有効な評価コンテキストであることを前提とします。
  • 戻り値は ctx->p（apr_pool_t）から割り当てられたメモリ領域を指します。

アルゴリズムまたは振る舞い

1. 引数 arg が NULL かどうかを判定します。 2. NULL であれば NULL を返します。 3. NULL でなければ、ssl_var_lookup を呼び出し、評価コンテキスト内のプール（ctx->p）、サーバー（ctx->s）、接続（ctx->c）、リクエスト（ctx->r）および変数名 arg を渡して値を解決します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_vars.c:119 にて、ap_expr の関数テーブル設定時に参照されています。
  • **呼び出し先**: ssl_var_lookup（同ファイル内）。

関連データ構造

  • ap_expr_eval_ctx_t: Apache 式評価コンテキスト。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • arg が NULL の場合、安全に NULL を返します。
  • ssl_var_lookup が特定の変数を解決できない場合、その実装に従い空文字列または NULL を返す可能性があります。

保守担当者向けメモ

  • 本関数は ap_expr インターフェースと ssl_var_lookup を繋ぐアダプターです。
  • data 引数は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
