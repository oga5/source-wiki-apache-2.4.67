dav_inherit_walker

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:566

目的

walk フック処理において、親リソースから子リソースへロックを再帰的に継承させるためのコールバック関数です。

シグネチャまたは形状

static dav_error * dav_inherit_walker(dav_walk_resource *wres, int calltype);

入力と出力

  • **入力**:
  • wres: 現在処理中のリソース情報を含む dav_walk_resource 構造体。
  • calltype: 呼び出しの種類を示す整数値。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時はエラー情報を含む dav_error 構造体へのポインタを返します。

前提条件と不変条件

  • wres->walk_ctx は dav_walker_ctx 型としてキャスト可能である必要があります。
  • ctx->w.lockdb->hooks->append_locks が有効な関数ポインタであることが前提です。

アルゴリズムまたは振る舞い

1. wres->walk_ctx からコンテキストを取得します。 2. ctx->skip_root が真かつ、現在のリソースがルートリソースと同一である場合、何もせず NULL を返します。 3. それ以外の場合、ctx->w.lockdb->hooks->append_locks を呼び出し、現在のリソース (wres->resource) に対して ctx->lock を適用します。この際、第3引数に 1 を指定し、ロックの追加を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/util_lock.c:659 にて ctx.w.func に代入され、walk フックの実行時に間接的に呼び出されます。
  • **呼び出し先**: wres->resource->hooks->is_same_resource および ctx->w.lockdb->hooks->append_locks を呼び出します。

関連データ構造

  • dav_walk_resource: ウォーク処理中のリソース状態を保持。
  • dav_walker_ctx: ロックデータベースや継承対象のロック情報を含むコンテキスト。
  • dav_lock: 継承されるロック情報。

失敗モードと境界条件

  • append_locks フックがエラーを返した場合、そのエラーをそのまま呼び出し元へ伝播させます。
  • skip_root が設定されている場合、ルートリソースへの重複適用を回避します。
