slotmem_put

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:192

目的

指定されたスロットメモリインスタンスの特定のインデックス（スロット）へ、メモリ領域からデータを書き込みます。

シグネチャまたは形状

apr_status_t slotmem_put(ap_slotmem_instance_t *slot, unsigned int id, unsigned char *src, apr_size_t src_len)

入力と出力

  • **入力**:
  • slot: 操作対象の ap_slotmem_instance_t 構造体ポインタ。
  • id: 書き込み先スロットのインデックス。
  • src: 書き込み元データのバッファポインタ。
  • src_len: 書き込むデータのサイズ。
  • **出力**:
  • 戻り値: apr_status_t 型。成功時は APR_SUCCESS を返します。

前提条件と不変条件

  • slot が NULL の場合、APR_ENOSHMAVAIL が返されます。
  • id がスロットの総数（slot->num）以上の場合、APR_EINVAL が返されます。
  • AP_SLOTMEM_IS_PREGRAB(slot) が有効な場合、対象スロットが確保済み（inuse フラグが真）でなければ APR_NOTFOUND が返されます。

アルゴリズムまたは振る舞い

1. 引数のバリデーション（slot の存在確認、id の範囲チェック）を行います。 2. AP_SLOTMEM_IS_PREGRAB フラグが設定されている場合、対象スロットが使用中であるかを確認します。 3. slotmem_dptr を呼び出し、対象スロットのメモリ先頭アドレスを取得します。 4. 取得に成功した場合、対象スロットの inuse フラグを 1 に設定します。 5. memcpy を使用して src からスロットメモリへデータをコピーします。

  • 注: memcpy 実行時の境界チェックは行われません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/slotmem/mod_slotmem_plain.c:310 (プロバイダー構造体への登録)
  • modules/slotmem/mod_slotmem_shm.c:739 (プロバイダー構造体への登録)
  • **呼び出し先**:
  • slotmem_dptr: スロットのメモリ位置特定に使用。
  • memcpy: データの転送に使用。

関連データ構造

  • ap_slotmem_instance_t: スロットメモリの管理構造体。

失敗モードと境界条件

  • APR_ENOSHMAVAIL: slot が NULL の場合に発生。
  • APR_EINVAL: id が範囲外の場合に発生。
  • APR_NOTFOUND: PREGRAB モードで未確保のスロットに書き込もうとした場合に発生。
  • memcpy の境界チェックが欠如しているため、src_len がスロットサイズを超過するとメモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は単一プロセス内での利用を想定しており、内部的な排他制御は行われません。マルチスレッド環境で使用する場合は、呼び出し側で適切な同期を行う必要があります。
  • memcpy のサイズチェックは呼び出し側の責任となります。
