send_socket

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fdpass.c:74

目的

Unix ドメインソケットを経由して、クライアントとの接続（ファイルディスクリプタ）を別のプロセスへ移譲（FD Passing）するために使用されます。

シグネチャまたは形状

static apr_status_t send_socket(apr_pool_t *p, apr_socket_t *s, apr_socket_t *outbound)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • s: 転送先となる Unix ドメインソケット。
  • outbound: 相手プロセスへ引き渡す対象のクライアントソケット。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返します。

前提条件と不変条件

  • CMSG_DATA マクロが定義されている環境（主に Unix 系）で動作することを前提とします。
  • 渡されるソケット s および outbound は、APR のソケットオブジェクトとして有効である必要があります。

アルゴリズムまたは振る舞い

1. apr_os_sock_get を使用して、APR ソケットから OS レベルのファイルディスクリプタ（rawsock および srawsock）を取得します。 2. msghdr 構造体を初期化し、送信データとしてダミーの 1 バイト（\0）を準備します。 3. cmsghdr 構造体をメモリプールから確保し、SCM_RIGHTS を設定して rawsock（引き渡す FD）を格納します。 4. sendmsg システムコールを呼び出し、制御メッセージとして FD を送信します。 5. 送信結果が -1 の場合は errno を返し、成功時は APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_fdpass.c:169（proxy_fdpass_handler 内で呼び出されます）。
  • **呼び出し先**:
  • apr_os_sock_get (APR)
  • sendmsg (システムコール)

関連データ構造

  • apr_socket_t: APR のソケット抽象化レイヤー。
  • msghdr, cmsghdr: sendmsg でファイルディスクリプタを渡すために使用される POSIX ソケット構造体。

失敗モードと境界条件

  • apr_os_sock_get が失敗した場合、そのエラーコードを返します。
  • sendmsg が -1 を返した場合、その時点の errno を返します。
  • 相手プロセスが FD を受け取る準備ができていない場合、送信処理がブロックされる可能性があります。

保守担当者向けメモ

  • 本関数は SCM_RIGHTS を用いた低レイヤーの FD 移譲を行います。引き渡し後のソケット管理（Apache 側でのクローズ防止など）は呼び出し元で行う必要があります。
  • プラットフォーム依存の CMSG_DATA マクロに依存しているため、移植時には注意してください。
