ap_mpm_child_main

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2_child.c:93

目的

OS/2 プラットフォームにおける Apache HTTP Server の子プロセスのメインエントリポイントとして、プロセス初期化、接続受け入れループ、およびワーカースレッドの管理を行う。

シグネチャまたは形状

void ap_mpm_child_main(apr_pool_t *pconf)

入力と出力

  • **入力**: pconf (親プロセスから引き継がれた設定用メモリプール)
  • **出力**: なし（プロセス終了時に clean_child_exit を呼び出すか、関数終了によりプロセスが終了する）

前提条件と不変条件

  • 親プロセスによって共有メモリ（スコアボード）が準備されていること。
  • ap_listeners が適切に設定されていること。
  • OS/2 の Dos* API が利用可能な環境であること。

アルゴリズムまたは振る舞い

1. **初期化**: シグナルハンドラの設定、子プロセス用メモリプールの作成、child_init フックの実行。 2. **リソース準備**: シャットダウン用イベントセマフォの作成、スコアボードのマップ、アクセプト用ミューテックスのオープン。 3. **プロセス特定**: スコアボードから自身の PID を検索し、スロットを初期化。 4. **通信路確立**: DosCreateQueue を用いて、接続情報や終了命令を伝達する FIFO キューを作成。 5. **スレッド起動**: 保守スレッド（server_maintenance）を起動。 6. **接続受け入れループ**:

  • apr_pollset を使用してリスニングソケットを監視。
  • 接続を受け入れた場合、worker_args_t を作成し、OS/2 キューへ投入。
  • max_requests_per_child に達した場合、または親プロセスから終了指示があった場合にループを抜ける。

    • 7. **終了処理**:

  • shutdown_event をポストし、保守スレッドの終了を待機。
  • グレースフルシャットダウン時は全ワーカーに終了命令をキューへ送り完了を待機。強制終了時は DosKillThread で全ワーカーを停止。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/mpmt_os2/mpmt_os2.c
  • **呼び出し先**:
  • ap_run_child_init, ap_init_scoreboard, ap_calc_scoreboard_size
  • Dos* 系 API (DosCreateEventSem, DosGetNamedSharedMem, DosCreateQueue, DosWriteQueue 等)
  • _beginthread (保守スレッド起動)
  • apr_pollset_poll, apr_socket_accept
  • clean_child_exit

関連データ構造

  • worker_args_t: ワーカーへ渡す接続情報。
  • HQUEUE workq: スレッド間通信用 FIFO キュー。
  • apr_pollset_t: リスニングソケット監視用ポーリングセット。

失敗モードと境界条件

  • **初期化失敗**: セマフォ作成、スコアボードアクセス、キュー作成のいずれかが失敗した場合、clean_child_exit(APEXIT_CHILDFATAL) を呼び出しプロセスを終了する。
  • **PID 未検出**: スコアボード内に自身の PID が見つからない場合、致命的エラーとして終了する。
  • **接続受け入れエラー**: apr_socket_accept が失敗し、かつ EINTR でない場合は致命的エラーとして終了する。

保守担当者向けメモ

  • 本関数は OS/2 のプロセスモデルに強く依存している。
  • HARD_THREAD_LIMIT（256）がスレッド数の上限としてハードコードされている点に注意。
  • Dos* API のエラーコードは OS/2 システムドキュメントを参照すること。
