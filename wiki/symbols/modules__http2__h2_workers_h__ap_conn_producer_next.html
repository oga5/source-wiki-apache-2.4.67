ap_conn_producer_next

定義

  • 種別: typedef
  • 定義箇所: modules/http2/h2_workers.h:80

目的

HTTP/2 ワーカープールにおいて、処理すべき次の接続（conn_rec）を取得するためのコールバック関数型を定義します。

シグネチャまたは形状

typedef conn_rec *ap_conn_producer_next(void *baton, int *pmore);

入力と出力

  • **入力**:
  • void *baton: プロデューサー固有のコンテキストデータへのポインタ。
  • int *pmore: 出力パラメータ。呼び出し元がさらに接続を取得できる可能性があるかを示すフラグを格納する。
  • **出力**:
  • 戻り値: 処理対象となる conn_rec *。取得できる接続がない場合は NULL を返す。

前提条件と不変条件

  • 本関数は h2_workers によって管理されるスレッドプール内のワーカーから呼び出されます。
  • 実装はスレッドセーフである必要があります。

アルゴリズムまたは振る舞い

1. ワーカーはアイドル状態になると、登録されたプロデューサーの next コールバックをポーリングします。 2. baton を使用して現在の接続ソースの状態を確認し、処理すべき conn_rec があればそれを返します。 3. pmore には、キューにまだ未処理の接続が残っているか、あるいは将来的に追加される可能性があるかを示す値を設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_workers.c 内のワーカー処理ループ（slot_run 等）から呼び出されます。
  • **呼び出し先**: 呼び出し元から提供された baton を介して、接続ソース固有のロジックが実行されます。

関連データ構造

  • conn_rec: Apache HTTP Server の接続構造体。
  • ap_conn_producer_t: 本関数を含むコールバック群を保持する構造体。

失敗モードと境界条件

  • 処理すべき接続が存在しない場合、NULL を返します。
  • pmore が 0 に設定された場合、呼び出し元は当該プロデューサーからの接続取得を一時停止または終了する可能性があります。

保守担当者向けメモ

  • 本関数は h2_workers_register を通じて登録されます。
  • 実装を追加・変更する際は、h2_workers.c 内のワーカーの実行サイクルとの同期に注意してください。
