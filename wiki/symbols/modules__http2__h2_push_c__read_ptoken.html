read_ptoken

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_push.c:183

目的

Link ヘッダーの解析において、トークン（ptoken）を読み取り、文字列として抽出します。

シグネチャまたは形状

static int read_ptoken(link_ctx *ctx, const char **ps)

入力と出力

  • **入力**:
  • ctx: 解析状態を保持する link_ctx 構造体へのポインタ。
  • ps: 抽出したトークン文字列を格納するためのポインタのポインタ。
  • **出力**:
  • 戻り値: トークンが正常に抽出できた場合は 1、それ以外は 0 を返します。
  • ps: 抽出に成功した場合、mk_str によって生成された文字列が設定されます。

前提条件と不変条件

  • ctx は有効な link_ctx 構造体を指している必要があります。
  • skip_ws(ctx) が呼び出され、先行する空白がスキップされることが前提となります。

アルゴリズムまたは振る舞い

1. skip_ws(ctx) を呼び出し、現在の位置から空白をスキップします。 2. ptoken_char() が真を返す文字が続く限り、インデックス i を進めます。 3. 1文字以上読み取れた場合、mk_str(ctx, i) を使用して該当範囲の文字列を生成し、ps に格納します。 4. ctx->i を更新し、成功を示す 1 を返します。 5. トークンが存在しない場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_push.c:232 (推測される箇所: read_qstring との論理和条件内)
  • **呼び出し先**:
  • skip_ws()
  • ptoken_char()
  • mk_str()

関連データ構造

  • link_ctx: 解析対象の文字列や現在のインデックスを保持する構造体。

失敗モードと境界条件

  • トークンが空（i == ctx->i）の場合、0 を返します。
  • 文字列の終端（ctx->slen）に達した場合、それまでの読み取り結果を返します。

保守担当者向けメモ

  • 本関数は Link ヘッダーの構文解析の一部です。ptoken_char の定義がトークンの許容文字セットを決定するため、仕様変更時はそちらも確認してください。
