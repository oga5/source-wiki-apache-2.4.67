val_H2_STREAM_TAG

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_http2.c:268

目的

HTTP/2 接続に関連付けられたストリームを識別するためのタグ文字列を生成します。このタグは、ログ出力や環境変数（H2_STREAM_TAG）として利用されます。

シグネチャまたは形状

static const char *val_H2_STREAM_TAG(apr_pool_t *p, server_rec *s,
                                     conn_rec *c, request_rec *r, h2_conn_ctx_t *ctx)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する APR プール。
  • c: 現在の接続情報 (conn_rec)。
  • **出力**:
  • 接続コンテキストが存在し、ストリーム ID が 0 の場合は接続 ID (conn_ctx->id) を返します。
  • ストリーム ID が 0 以外の場合は、接続 ID とストリーム ID をハイフンで連結した文字列（例: ID-STREAMID）を返します。
  • 接続情報がない、またはコンテキストが取得できない場合は空文字列 "" を返します。

前提条件と不変条件

  • c が有効な conn_rec であること。
  • h2_conn_ctx_get(c) が接続に関連付けられた h2_conn_ctx_t を正しく取得できること。

アルゴリズムまたは振る舞い

1. 引数 c を通じて h2_conn_ctx_get を呼び出し、接続コンテキストを取得します。 2. コンテキストが存在しない場合は空文字列を返します。 3. コンテキストが存在する場合、stream_id を確認します。

  • stream_id == 0 の場合、接続 ID をそのまま返します。
  • stream_id > 0 の場合、apr_psprintf を使用して 接続ID-ストリームID 形式の文字列を動的に生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/mod_http2.c:284 (環境変数生成処理)
  • modules/http2/mod_http2.c:306 (h2_var_def 配列の定義内)
  • **呼び出し先**:
  • h2_conn_ctx_get (接続コンテキストの取得)
  • apr_psprintf (文字列のフォーマットとメモリ割り当て)

関連データ構造

  • h2_conn_ctx_t: 接続ごとのメタデータを保持する構造体。id および stream_id フィールドを参照します。
  • apr_pool_t: 文字列生成時のメモリ管理に使用されます。

失敗モードと境界条件

  • c が NULL の場合、安全に空文字列を返します。
  • h2_conn_ctx_get(c) が NULL を返す場合、安全に空文字列を返します。
  • メモリ不足時には apr_psprintf が異常終了する可能性があります（APR の標準的な挙動に依存）。

保守担当者向けメモ

  • 本関数は h2_var_def 配列を通じて登録されており、http2_var_lookup 経由で環境変数として公開されます。
  • 文字列の生成には apr_pool_t を使用しているため、呼び出し元のライフサイクルに依存したメモリ管理が行われます。
