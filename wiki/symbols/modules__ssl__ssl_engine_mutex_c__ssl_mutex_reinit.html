ssl_mutex_reinit

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_mutex.c:59

目的

マルチプロセス環境において、子プロセス生成後にセッションキャッシュ保護用のグローバル Mutex を再初期化（再接続）する。

シグネチャまたは形状

int ssl_mutex_reinit(server_rec *s, apr_pool_t *p)

入力と出力

  • **入力**:
  • s: サーバー設定構造体（server_rec）。
  • p: 再初期化に使用するメモリプール（apr_pool_t）。
  • **出力**:
  • TRUE (1): 再初期化が不要、または成功した場合。
  • FALSE (0): 再初期化に失敗した場合。

前提条件と不変条件

  • myModConfig(s) を通じて取得した SSLModConfigRec が有効であること。
  • セッションキャッシュプロバイダが AP_SOCACHE_FLAG_NOTMPSAFE フラグを保持しており、かつ Mutex が設定されている場合にのみ実行される。

アルゴリズムまたは振る舞い

1. **初期チェック**: Mutex (mc->pMutex) が未設定、セッションキャッシュが未定義、またはキャッシュプロバイダがスレッドセーフ（AP_SOCACHE_FLAG_NOTMPSAFE が未設定）である場合は、何もせず TRUE を返す。 2. **ロックファイル取得**: apr_global_mutex_lockfile を呼び出し、現在の Mutex に関連付けられたロックファイルパスを取得する。 3. **再初期化**: apr_global_mutex_child_init を呼び出し、子プロセス内での Mutex を再初期化する。 4. **エラー処理**: 再初期化に失敗した場合、ap_log_error を使用してエラーまたは警告ログを出力し、FALSE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_init.c:2469
  • **呼び出し先**:
  • apr_global_mutex_lockfile (APR)
  • apr_global_mutex_child_init (APR)
  • ap_log_error (Apache Core)

関連データ構造

  • SSLModConfigRec: モジュールの設定情報を保持する構造体。pMutex（Mutex オブジェクト）および sesscache（キャッシュプロバイダ）への参照を含む。
  • server_rec: Apache HTTP Server のサーバー設定構造体。

失敗モードと境界条件

  • **失敗モード**:
  • apr_global_mutex_child_init が APR_SUCCESS 以外の値を返した場合、Mutex の再初期化に失敗し、ログ出力後に FALSE を返す。
  • **境界条件**:
  • キャッシュプロバイダがスレッドセーフである場合、この関数は実質的に何もしない（早期リターンする）。

保守担当者向けメモ

  • 本関数は子プロセス生成後の初期化フェーズで呼び出されることを前提としている。
  • SSL_CACHE_MUTEX_TYPE 定義に基づきログ出力が行われるため、Mutex 関連のトラブルシューティング時にはこの定数とログメッセージを確認すること。
