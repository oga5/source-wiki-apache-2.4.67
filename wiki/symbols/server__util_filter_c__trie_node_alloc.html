trie_node_alloc

定義

  • 種別: function
  • 定義箇所: server/util_filter.c:112

目的

フィルタ名を管理するトライ木構造において、新しいノードを割り当ておよび初期化します。

シグネチャまたは形状

static filter_trie_node *trie_node_alloc(apr_pool_t *p, filter_trie_node *parent, char c)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • parent: 親ノード（ルートノード作成時は NULL）。
  • c: このノードが表す文字。
  • **出力**:
  • 初期化済みの filter_trie_node 構造体へのポインタ。

前提条件と不変条件

  • parent が指定されている場合、指定された文字 c が既に子ノードとして存在しないことを前提とします（存在する場合はその既存ノードを返します）。
  • 割り当てられるノードの frec は NULL、nchildren は 0 に初期化されます。
  • 子ノード配列は TRIE_INITIAL_SIZE で初期化されます。

アルゴリズムまたは振る舞い

1. parent が存在する場合、既存の子ノードを走査し、文字 c が一致すればそのノードを返します。 2. 一致しない場合、または parent が NULL の場合、apr_palloc を使用して新しい filter_trie_node をメモリ確保します。 3. parent が存在する場合は trie_node_link を呼び出し、親ノードと新しいノードをリンクさせます。 4. 新しいノードの各メンバ（frec, nchildren, size, children）を初期化し、ノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_filter.c:215 (register_filter 内)
  • server/util_filter.c:223 (register_filter 内)
  • **呼び出し先**:
  • apr_palloc (APR メモリ管理)
  • trie_node_link (ノード間のリンク構築)

関連データ構造

  • filter_trie_node: トライ木の各ノードを構成する構造体。
  • filter_trie_child_ptr: 子ノードへのポインタと文字を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性があります。
  • parent が NULL の場合、ルートノードとして扱われます。

保守担当者向けメモ

  • 本関数はフィルタ登録時のトライ木構築に特化しています。
  • TRIE_INITIAL_SIZE はトライ木の初期ノード容量を決定する定数です。
  • 既存ノードの検索は parent->children を順次走査する線形探索で行われます。
