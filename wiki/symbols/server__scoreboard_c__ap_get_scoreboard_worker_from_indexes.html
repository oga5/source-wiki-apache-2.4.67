ap_get_scoreboard_worker_from_indexes

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:683

目的

共有メモリ上のスコアボードから、指定されたプロセス（子プロセス）およびスレッドのインデックスに対応する worker_score 構造体へのポインタを取得します。

シグネチャまたは形状

worker_score *ap_get_scoreboard_worker_from_indexes(int x, int y)

入力と出力

  • **入力**:
  • int x: プロセス（子プロセス）のインデックス。
  • int y: スレッドのインデックス。
  • **出力**:
  • 成功時: 指定されたインデックスに対応する worker_score 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • ap_scoreboard_image が初期化済みである必要があります。
  • x は 0 以上 server_limit 未満、y は 0 以上 thread_limit 未満である必要があります。

アルゴリズムまたは振る舞い

1. 引数 x および y が、サーバーの制限値（server_limit, thread_limit）の範囲内にあるか検証します。 2. 範囲外の場合は NULL を返します。 3. 範囲内の場合、グローバル変数 ap_scoreboard_image が保持する2次元配列 servers[x][y] のアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/scoreboard.c:697
  • server/scoreboard.c:705
  • include/scoreboard.h:213（宣言）
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行いません。

関連データ構造

  • worker_score: 各ワーカーの統計情報や状態を保持する構造体（include/scoreboard.h で定義）。
  • ap_scoreboard_image: スコアボード全体を管理するグローバル構造体。

失敗モードと境界条件

  • x または y が負の値、あるいはそれぞれの制限値（server_limit, thread_limit）以上である場合、境界外アクセスを防ぐために NULL を返します。

保守担当者向けメモ

  • 本関数は共有メモリ上の特定のオフセットを計算して返すのみであり、ロックを取得しません。これは「1スロットにつき1書き込み者」という設計上の不変条件に基づいています。
  • server_limit および thread_limit は MPM の設定に依存します。
