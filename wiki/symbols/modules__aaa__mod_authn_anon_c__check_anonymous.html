check_anonymous

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_anon.c:135

目的

Apache HTTP Server における匿名認証プロバイダーとして、指定されたユーザー名とパスワードが匿名ログインの許可条件を満たしているかを検証する。

シグネチャまたは形状

static authn_status check_anonymous(request_rec *r, const char *user, const char *sent_pw)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • user: 認証対象のユーザー名。
  • sent_pw: 認証時に提供されたパスワード（メールアドレス形式を想定）。
  • **出力**:
  • AUTH_GRANTED: 認証成功。
  • AUTH_DENIED: ユーザー名は合致したがパスワード条件を満たさない場合。
  • AUTH_USER_NOT_FOUND: ユーザー名が許可リストに存在しない場合。

前提条件と不変条件

  • authn_anon_config_rec 設定がディレクトリ単位で取得可能であること。
  • ユーザー名の比較は strcasecmp を用いて大文字小文字を区別せずに行われる。

アルゴリズムまたは振る舞い

1. **設定確認**: 設定が存在しない場合は AUTH_USER_NOT_FOUND を返す。 2. **ユーザー検証**:

  • ユーザー名が空の場合、nouserid 設定が有効であれば許可。
  • anyuserid 設定が有効であれば許可。
  • 上記以外の場合、許可リスト（conf->users）を走査し、一致するユーザー名を探す。

    • 3. **パスワード検証**:

  • mustemail が有効な場合、パスワードが空でないことを確認。
  • verifyemail が有効な場合、パスワードに @ と . が含まれていることを確認。

    • 4. **ログ出力**: 認証成功時、logemail が有効かつ初期リクエストであれば、パスワード（メールアドレス）をログに記録する。 5. **結果返却**: 全条件を満たせば AUTH_GRANTED を返し、そうでなければ AUTH_DENIED または AUTH_USER_NOT_FOUND を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_anon.c:195 にて authn_anon_provider 構造体を通じて登録・参照されている。
  • **呼び出し先**: ap_get_module_config, ap_strchr_c, ap_is_initial_req, ap_log_rerror 等の Apache API。

関連データ構造

  • authn_anon_config_rec: ディレクトリごとの設定（許可ユーザーリスト、検証フラグ等）を保持。
  • anon_auth_user: 許可ユーザー名を保持する連結リスト構造体。

失敗モードと境界条件

  • ユーザー名が空の場合、nouserid 設定の有無が判定の分かれ目となる。
  • パスワード検証（verifyemail）は、パスワードが提供されていない場合や、形式が不正な場合に AUTH_DENIED を引き起こす。
  • サブクエストによる重複ログ出力を防ぐため、ap_is_initial_req(r) によるガードが行われている。

保守担当者向けメモ

  • 本関数は認証のみを担当する。認可については mod_authz_* 系モジュールが別途処理を行う。
  • Anonymous_VerifyEmail 設定を有効にする際は、パスワードフィールドに @ と . を必須とする仕様に注意すること。
