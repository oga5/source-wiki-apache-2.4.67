h2_c1_hook_process_connection

定義

  • **種別**: function
  • **定義箇所**: modules/http2/h2_c1.c:241

目的

Apache HTTP Server の process_connection フックとして、クライアント接続が HTTP/2 プロトコルを使用すべきかを判定し、セッションの初期化および実行をトリガーします。

シグネチャまたは形状

static int h2_c1_hook_process_connection(conn_rec* c)

入力と出力

  • **入力**: conn_rec* c（処理対象の接続レコード）
  • **出力**: int（Apache のフック戻り値。OK、DECLINED、またはエラーを示す値）

前提条件と不変条件

  • c->master が設定されている場合（サブ接続である場合）、本関数は直ちに DECLINED を返します。
  • HTTP/2 セッションが確立されていない場合、AP_MODE_SPECULATIVE を用いて入力ストリームから 24 バイトを先読みし、HTTP/2 マジックトークンとの照合を行います。

アルゴリズムまたは振る舞い

1. **初期チェック**: 接続がマスター接続でないことを確認し、既存の HTTP/2 接続コンテキスト（h2_conn_ctx_t）を取得します。 2. **プロトコル検出**: コンテキストが存在せず、かつ keepalives が 0 の場合、現在のプロトコルが http/1.1 であり、かつ H2Direct が有効であれば、入力フィルタから 24 バイトを先読みします。 3. **マジックトークン検証**: 先読みしたデータが H2_MAGIC_TOKEN と一致する場合、HTTP/2 セッション用のコンテキストを生成します。 4. **セッションセットアップ**: コンテキスト内の session が未初期化であれば、h2_c1_setup を呼び出してセッションを構築します。 5. **実行**: h2_c1_run を呼び出し、HTTP/2 セッションのメインループを開始します。 6. **終了**: 検出やセットアップが不要、あるいは失敗した場合は DECLINED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の process_connection フックチェーン（ap_hook_process_connection により登録）。
  • **呼び出し先**:
  • h2_conn_ctx_get / h2_conn_ctx_create_for_c1 / h2_conn_ctx_detach
  • h2_c1_allows_direct / h2_protocol_is_acceptable_c1
  • h2_c1_setup
  • h2_c1_run
  • ap_get_brigade / apr_brigade_create / apr_brigade_destroy

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • h2_conn_ctx_t: HTTP/2 接続固有のコンテキスト情報。
  • apr_bucket_brigade: プロトコル検出のためのデータ読み込みに使用されるバケットブリゲード。

失敗モードと境界条件

  • **読み込み失敗**: ap_get_brigade が APR_SUCCESS 以外を返した場合、エラーをログ出力し DECLINED を返します。
  • **セットアップ失敗**: h2_c1_setup が失敗した場合、コンテキストをデタッチし、!OK（エラー）を返します。
  • **非対応プロトコル**: H2Direct が無効、またはマジックトークンが一致しない場合、HTTP/2 処理をスキップして DECLINED を返します。

保守担当者向けメモ

  • 本関数は HTTP/2 へのアップグレードパスと直接検出パスの両方を制御します。
  • AP_MODE_SPECULATIVE を使用した先読みは、後続のフィルタチェーンに影響を与えないよう適切にブリゲードを破棄する必要があります。
  • 接続コンテキストのライフサイクル管理（h2_conn_ctx）と h2_session の生成タイミングに依存関係があるため、変更時は h2_c1_setup の仕様を併せて確認してください。
