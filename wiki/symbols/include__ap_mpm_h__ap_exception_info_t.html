ap_exception_info_t

定義

  • 種別: struct
  • 定義箇所: include/ap_mpm.h:262

目的

致命的な例外が発生した際に、そのシグナル情報とプロセスIDを保持し、例外処理フックへ情報を伝達するために使用されます。

シグネチャまたは形状

typedef struct ap_exception_info_t {
    int sig;
    pid_t pid;
} ap_exception_info_t;

入力と出力

  • **入力**: 構造体の各メンバ（sig, pid）に値を設定して初期化します。
  • **出力**: fatal_exception フックの引数として渡され、例外発生時のコンテキスト情報として利用されます。

前提条件と不変条件

  • pid_t 型はシステム環境に依存します。
  • fatal_exception フックを介して渡される際、ポインタが有効である必要があります。

アルゴリズムまたは振る舞い

1. 例外発生時、ap_exception_info_t 構造体がインスタンス化され、発生したシグナル番号とプロセスIDが格納されます。 2. この構造体へのポインタが fatal_exception フックに渡されます。 3. フックの実装側（server/mpm_common.c 等）は、この情報を参照して例外処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm_unix.c:963 にて構造体の初期化が行われています。
  • **呼び出し先**: include/ap_mpm.h:275 で定義される fatal_exception フックの引数として利用されます。
  • **その他**: server/mpm_common.c:84 にてフックの実行時に参照されています。

関連データ構造

  • fatal_exception フック: この構造体を引数として受け取るフックインターフェース。

失敗モードと境界条件

  • 構造体の初期化が不完全な場合（例: sig や pid が未設定）、フック側で誤った情報に基づいて例外処理が行われる可能性があります。

保守担当者向けメモ

  • 本構造体は fatal_exception フック専用のデータ伝達手段です。フックのシグネチャを変更する場合は、この構造体の定義と併せて整合性を確認してください。
