store_array

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_socache.c:170

目的

apr_array_header_t に格納された文字列の配列を、CRLF（\r\n）で区切られた形式でバッファにシリアライズします。

シグネチャまたは形状

apr_status_t store_array(apr_array_header_t *arr, unsigned char *buffer,
                         apr_size_t buffer_len, apr_size_t *slider);

入力と出力

  • arr: シリアライズ対象の文字列配列。
  • buffer: 書き込み先のバッファ（NULL の場合、サイズ計算のみを行う）。
  • buffer_len: バッファの総サイズ。
  • slider: バッファ内の現在の書き込みオフセット（入出力両用）。
  • 戻り値: 成功時は APR_SUCCESS、バッファ不足時は APR_EOF を返します。

前提条件と不変条件

  • slider は、書き込み開始位置を指している必要があります。
  • 各要素の末尾には CRLF が付与されます。
  • 配列の全要素を書き込んだ後、最後に配列全体を閉じるための CRLF が追加されます。

アルゴリズムまたは振る舞い

1. 配列 arr の各要素を順次走査します。 2. 各要素の長さを取得し、バッファの残り容量（buffer_len - *slider）と比較します。 3. 容量が不足している場合は APR_EOF を返して終了します。 4. 容量が十分な場合、apr_snprintf を使用して要素と CRLF をバッファに書き込み、*slider を進めます。 5. 全要素の処理後、最後に終端として CRLF をバッファに書き込み、*slider を更新します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/cache/mod_cache_disk.c:1036
  • modules/cache/mod_cache_socache.c:837
  • 呼び出し先:
  • strlen (標準ライブラリ)
  • apr_snprintf (APR)
  • memcpy (標準ライブラリ)

関連データ構造

  • apr_array_header_t: 文字列のリストを保持する APR の配列構造体。

失敗モードと境界条件

  • buffer_len に対して書き込みデータが収まらない場合、APR_EOF を返します。
  • buffer が NULL の場合、データ書き込みは行わず、*slider の更新のみ（サイズ計算）が行われます。

保守担当者向けメモ

  • 本関数は、キャッシュエントリのシリアライズにおいてヘッダーやメタデータのパックに使用されます。
  • CRLF のサイズは sizeof(CRLF) - 1 として計算されており、終端処理と各要素の区切りで一貫しています。
