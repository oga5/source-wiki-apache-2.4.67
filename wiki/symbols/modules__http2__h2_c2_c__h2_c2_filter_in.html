h2_c2_filter_in

定義

  • **種別**: function
  • **定義箇所**: modules/http2/h2_c2.c:181

目的

HTTP/2 セカンダリ接続（c2）におけるネットワーク入力フィルタとして機能し、h2_bucket_beam を介して HTTP/2 セッションから受信したデータを Apache の apr_bucket_brigade に変換して提供します。

シグネチャまたは形状

apr_status_t h2_c2_filter_in(ap_filter_t* f,
                             apr_bucket_brigade* bb,
                             ap_input_mode_t mode,
                             apr_read_type_e block,
                             apr_off_t readbytes)

入力と出力

  • **入力**:
  • f: フィルタコンテキスト。
  • bb: データを格納するバケットブリゲード。
  • mode: 読み取りモード（AP_MODE_READBYTES, AP_MODE_GETLINE 等）。
  • block: ブロック読み取りの可否。
  • readbytes: 要求するバイト数。
  • **出力**:
  • bb: 読み取られたデータが格納される。
  • 戻り値: APR_SUCCESS、APR_EOF、APR_EAGAIN、またはエラーコード。

前提条件と不変条件

  • f->c に h2_conn_ctx_t が関連付けられていること。
  • fctx（フィルタコンテキスト）が未初期化の場合は、初回呼び出し時に f->c->pool を使用して初期化される。
  • 接続がアボートされている場合、即座に APR_ECONNABORTED を返す。

アルゴリズムまたは振る舞い

1. **初期化**: AP_MODE_INIT の場合は標準のフィルタチェーンへ委譲。 2. **コンテキスト準備**: fctx が存在しない場合、ブリゲードを作成し、beam_in が空であれば EOS バケットを挿入する。 3. **アップグレード処理**: HTTP Upgrade 時、初回読み取りで EOS を挿入し、リクエストボディの終了を通知する。 4. **データ受信**: fctx->bb が空の場合、h2_beam_receive を呼び出して beam_in からデータを取得する。必要に応じて h2_util_wait_on_pipe で待機する。 5. **モード別処理**:

  • AP_MODE_EXHAUSTIVE: 全データを連結。
  • AP_MODE_READBYTES / AP_MODE_SPECULATIVE: 指定長分を連結またはコピー。
  • AP_MODE_GETLINE: apr_brigade_split_line を使用して行単位で抽出。

    • 6. **統計**: h2_c_logio_add_bytes_in が定義されていれば、受信バイト数を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache フィルタチェーン（ap_get_brigade 等を介して呼び出される）。
  • **呼び出し先**:
  • h2_beam_receive (データ取得)
  • h2_util_wait_on_pipe (I/O 待機)
  • h2_brigade_concat_length / h2_brigade_copy_length (データ操作)
  • ap_get_brigade (初期化時)

関連データ構造

  • h2_conn_ctx_t: 接続ごとのコンテキスト。
  • h2_c2_fctx_in_t: フィルタ固有の内部状態（ブリゲードやフラグを保持）。
  • h2_bucket_beam: ストリーム間のデータ転送用バッファ。

失敗モードと境界条件

  • APR_EAGAIN: ノンブロッキング読み取りでデータがない場合に発生。
  • APR_EOF: ストリームの終了時に発生。
  • AP_MODE_EATCRLF: 未サポートのため APR_ENOTIMPL を返す。
  • 接続がアボートされている場合は APR_ECONNABORTED を返す。

保守担当者向けメモ

  • AP_MODE_GETLINE はバケット分割を伴うため注意が必要。
  • h2_c_logio_add_bytes_in はオプションのフックであり、存在する場合のみ呼び出される。
  • h2_c2_filter_in は H2_C2_NET_IN として登録されている。
