ap_duplicate_listeners

定義

  • 種別: function
  • 定義箇所: server/listen.c:797

目的

マルチコア環境における負荷分散のため、既存のリスナーリストを複製し、複数のリスナーバケットを作成する。

シグネチャまたは形状

apr_status_t ap_duplicate_listeners(apr_pool_t *p, server_rec *s,
                                    ap_listen_rec ***buckets,
                                    int *num_buckets)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • s: サーバー設定レコード。
  • buckets: 複製されたリスナーリストの配列へのポインタ。
  • num_buckets: 作成するバケット数（入力時に1未満の場合は自動算出を試みる）。
  • **出力**:
  • buckets: 複製されたリスナーの配列が格納される。
  • num_buckets: 実際に作成されたバケット数。
  • 戻り値: APR_SUCCESS またはエラーを示す apr_status_t。

前提条件と不変条件

  • ap_listeners に初期リスナーリストが構築されている必要がある。
  • ap_listencbratio が設定されている場合、SO_REUSEPORT および _SC_NPROCESSORS_ONLN のサポート状況に応じてバケット数が決定される。
  • 複製された各リスナーは、make_sock を通じてソケットの作成と設定が行われる。

アルゴリズムまたは振る舞い

1. **バケット数決定**: num_buckets が1未満の場合、ap_listencbratio に基づき CPU コア数からバケット数を算出する（SO_REUSEPORT が利用可能な場合）。 2. **メモリ確保**: buckets 配列を確保し、インデックス0に既存の ap_listeners を割り当てる。 3. **リスナー複製**: 1から num_buckets - 1 までループし、各リスナーを複製する。

  • HAVE_SYSTEMD が定義されている場合は alloc_systemd_listener を使用。
  • それ以外の場合は apr_socket_create で新規ソケットを作成し、make_sock でオプションを設定する。

    • 4. **フィルタ適用**: ap_apply_accept_filter を呼び出し、OS 固有のアクセプトフィルタを適用する。 5. **状態更新**: グローバル変数 ap_listen_buckets および ap_num_listen_buckets を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/prefork/prefork.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • make_sock
  • ap_apply_accept_filter
  • alloc_systemd_listener (条件付き)
  • apr_socket_create, apr_socket_opt_set, apr_sockaddr_info_get 等の APR 関数

関連データ構造

  • ap_listen_rec: リスナー情報を保持する構造体。
  • server_rec: サーバー設定レコード。
  • ap_listen_buckets: 複製されたリスナーバケットを保持するグローバル配列。

失敗モードと境界条件

  • apr_socket_create が失敗した場合、エラーコードを返し処理を中断する。
  • apr_socket_opt_set (ノンブロッキング設定) が失敗した場合、エラーコードを返し処理を中断する。
  • SO_REUSEPORT がサポートされていない環境で ListenCoresBucketsRatio が指定された場合、警告を出力し単一バケットで動作する。

保守担当者向けメモ

  • ap_listeners のリスト構造を維持しながら複製を行うため、リストの末尾への追加処理に注意すること。
  • AP_NONBLOCK_WHEN_MULTI_LISTEN が定義されている場合、マルチリスナー環境ではソケットをノンブロッキングモードに設定するロジックが含まれている。
