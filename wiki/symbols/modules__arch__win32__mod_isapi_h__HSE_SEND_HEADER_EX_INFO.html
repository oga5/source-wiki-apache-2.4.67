HSE_SEND_HEADER_EX_INFO

定義

  • 種別: struct
  • 定義箇所: modules/arch/win32/mod_isapi.h:101

目的

ISAPI 拡張が HTTP レスポンスヘッダーを送信する際に使用する、IIS 5.0 相当の拡張構造体です。

シグネチャまたは形状

typedef struct HSE_SEND_HEADER_EX_INFO {
    const char * pszStatus;
    const char * pszHeader;
    apr_uint32_t cchStatus;
    apr_uint32_t cchHeader;
    int          fKeepConn;
} HSE_SEND_HEADER_EX_INFO;

入力と出力

  • **入力**:
  • pszStatus: HTTP ステータス文字列（例: "200 OK"）。
  • pszHeader: HTTP ヘッダー行（例: "Content-type: text/plain\r\n"）。
  • cchStatus, cchHeader: それぞれの文字列長。
  • fKeepConn: 接続維持フラグ（Apache では無視されます）。

前提条件と不変条件

  • pszHeader には NULL 文字を含めてはなりません。含めた場合、Apache の内部処理と競合します。
  • cchStatus および cchHeader は、対応する文字列の長さを正確に指定する必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は ServerSupportFunction を通じて渡されます。
  • fKeepConn は Apache 側では無視されます。接続の維持（Keep-Alive）状態は、クライアントとの間でネゴシエーションされた HTTP コントラクトに基づいて Apache が自動的に決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_isapi.c:1326 にて、バッファから本構造体へのキャストが行われている直接的な証拠があります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しは含みません。

関連データ構造

  • EXTENSION_CONTROL_BLOCK: ISAPI 拡張のリクエストコンテキスト。
  • HSE_TF_INFO: ファイル転送用構造体。

失敗モードと境界条件

  • ヘッダー文字列内に NULL 文字が含まれると、Apache の内部処理において予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は Apache のアーキテクチャに合わせてエミュレーションされています。
  • fKeepConn が無視される仕様であるため、ISAPI 拡張側で接続維持を強制しようとしても Apache のプロトコル制御が優先される点に注意してください。
