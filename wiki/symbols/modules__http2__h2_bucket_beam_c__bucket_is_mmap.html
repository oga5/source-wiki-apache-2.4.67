bucket_is_mmap

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:92

目的

指定された APR バケットが MMAP（メモリマップ）バケットであるかどうかを判定します。

シグネチャまたは形状

static int bucket_is_mmap(apr_bucket *b)

入力と出力

  • **入力**: apr_bucket *b（判定対象のバケット）
  • **出力**: int 型。MMAP バケットであれば非ゼロ（真）、そうでなければ 0（偽）を返します。

前提条件と不変条件

  • APR_HAS_MMAP が定義されている環境では、APR の APR_BUCKET_IS_MMAP マクロの挙動に依存します。
  • APR_HAS_MMAP が定義されていない環境では、常に 0 を返します。

アルゴリズムまたは振る舞い

1. コンパイル時に APR_HAS_MMAP が有効な場合、APR_BUCKET_IS_MMAP(b) を呼び出し、その結果を返します。 2. APR_HAS_MMAP が無効な場合、常に 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の箇所で、ファイルバケットと併せて MMAP バケットの判定に使用されています。
  • modules/http2/h2_bucket_beam.c:104
  • modules/http2/h2_bucket_beam.c:143
  • modules/http2/h2_bucket_beam.c:475
  • modules/http2/h2_bucket_beam.c:498
  • **呼び出し先**: APR_BUCKET_IS_MMAP マクロ（apr_buckets.h 由来）。

関連データ構造

  • apr_bucket: APR のバケット構造体。

失敗モードと境界条件

  • 引数 b が NULL の場合の挙動は、APR_BUCKET_IS_MMAP マクロの実装に依存します。

保守担当者向けメモ

  • 本関数は h2_bucket_beam.c 内で、ファイルバケットと同様に特別な扱い（setaside 等）が必要なバケットを識別するために使用されています。
  • APR_HAS_MMAP の有無による条件分岐を隠蔽するラッパーとして機能しています。
