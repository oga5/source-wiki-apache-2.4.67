worker_main

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/child.c:785

目的

Windows 版 Apache HTTP Server におけるワーカー・スレッドのメインループを実行し、I/O Completion Port (IOCP) を介して受け取った接続の処理を行う。

シグネチャまたは形状

static DWORD __stdcall worker_main(void *thread_num_val)

入力と出力

  • **入力**: thread_num_val (スレッド番号を示す整数値)
  • **出力**: 0 (スレッド終了時に返される)

前提条件と不変条件

  • pchild プールが子プロセスの生存期間を管理していること。
  • winnt_get_connection を通じて接続コンテキストを取得するループ構造であること。
  • ap_max_requests_per_child が設定されている場合、リクエスト数が上限に達すると max_requests_per_child_event がシグナルされる。

アルゴリズムまたは振る舞い

1. **初期化**: スレッドローカルストレージの初期化（AP_HAS_THREAD_LOCAL が有効な場合）。 2. **ループ処理**:

  • スレッド状態を SERVER_READY に更新。
  • winnt_get_connection を呼び出し、接続を待機。context が取得できない場合はループを抜ける。
  • リクエスト上限チェックを行い、必要に応じて終了イベントをセット。
  • ap_run_create_connection で接続オブジェクトを作成。失敗時はソケットを閉じて継続。
  • ap_process_connection を呼び出し、HTTP 接続処理を実行。
  • ap_lingering_close で接続を終了。

    • 3. **終了処理**: スレッド状態を SERVER_DEAD に更新し、必要に応じてスレッドプールを破棄して終了。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/child.c:1023（子プロセス起動時のスレッド生成箇所）。
  • **呼び出し先**: winnt_get_connection, ap_update_child_status_from_indexes, ap_create_sb_handle, ap_run_create_connection, ap_process_connection, ap_lingering_close 等。
  • *注: 他のファイル（mpmt_os2_child.c や mpm_netware.c）にも同名の関数が存在するが、本ページは server/mpm/winnt/child.c の実装を対象としている。*

関連データ構造

  • winnt_conn_ctx_t: 接続状態を保持するコンテキスト。
  • conn_rec: HTTP 接続情報を保持する構造体。
  • ap_sb_handle_t: スコアボード管理用ハンドル。

失敗モードと境界条件

  • winnt_get_connection が NULL を返すと、スレッドはループを終了する。
  • ap_run_create_connection が失敗した場合、ソケットはクローズされ、次の接続待機へ移行する。
  • disconnected フラグが立っていない場合、accept_socket は INVALID_SOCKET に設定される。

保守担当者向けメモ

  • winnt_conn_ctx_t の再利用ロジックにおいて、エラーパスでのメモリリークの可能性があるため注意が必要。
  • AP_HAS_THREAD_LOCAL の有無により、スレッドローカルストレージの初期化と破棄の挙動が分岐する。
