proxy_balancer_method

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:116（前方宣言）、modules/proxy/mod_proxy.h:583（構造体定義）

目的

Apache HTTP Server のプロキシサブシステムにおいて、負荷分散（ロードバランシング）アルゴリズムを抽象化するためのインターフェースを定義します。

シグネチャまたは形状

struct proxy_balancer_method として定義され、負荷分散アルゴリズムを実装する各モジュール（mod_lbmethod_bybusyness 等）によって提供される関数ポインタ群を保持します。

入力と出力

具体的な入力および出力は、構造体内の各関数ポインタの定義に依存します。一般的に、バランサーの状態管理やワーカーの選択を行うためのコンテキスト情報が渡されます。

前提条件と不変条件

  • proxy_balancer 構造体と組み合わせて使用されます。
  • 負荷分散アルゴリズムを提供するモジュールは、この構造体のインスタンスを静的に定義し、プロキシサブシステムに登録する必要があります。

アルゴリズムまたは振る舞い

プロキシサブシステムは、この構造体を介して、リクエストごとに最適なバックエンドワーカーを選択するアルゴリズムを動的に呼び出します。これにより、トラフィック量、リクエスト数、あるいはサーバーの負荷状況に応じた分散戦略の切り替えが可能となります。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy.c、mod_proxy_balancer.c、および proxy_util.c 内の負荷分散処理ロジック。
  • **呼び出し先（実装提供元）**:
  • mod_lbmethod_bybusyness.c
  • mod_lbmethod_byrequests.c
  • mod_lbmethod_bytraffic.c
  • mod_lbmethod_heartbeat.c

関連データ構造

  • proxy_balancer: 負荷分散グループを管理する構造体。本構造体へのポインタを保持します。
  • proxy_worker: 負荷分散対象となるバックエンドサーバー。

失敗モードと境界条件

  • アルゴリズムがワーカーを選択できない場合、プロキシサブシステムは適切なエラーハンドリングを行う必要があります。
  • 構造体内の関数ポインタが初期化されていない場合、実行時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はプロキシの負荷分散ロジックのプラグインポイントです。新しい負荷分散アルゴリズムを追加する場合は、この構造体を実装した新しい mod_lbmethod_*.c を作成してください。
  • 構造体の定義が mod_proxy.h の 583 行目付近にあるため、メンバーの追加や変更を行う際はバイナリ互換性に注意してください。
