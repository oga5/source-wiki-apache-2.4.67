cfg_lua_map_handler

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:82

目的

Lua スクリプトから提供された設定テーブルを解析し、Apache のディレクトリ設定構造体（ap_lua_dir_cfg）にハンドラ仕様（ap_lua_mapped_handler_spec）として登録する。

シグネチャまたは形状

static int cfg_lua_map_handler(lua_State *L);

入力と出力

  • **入力**:
  • Lua スタック第1引数: ap_lua_dir_cfg 型のユーザーデータ。
  • Lua スタック第2引数: 設定情報を含む Lua テーブル（file, pattern, scope, func フィールド）。
  • **出力**: 成功時は 0 を返し、失敗時は luaL_error を通じて Lua 側に例外を伝播させる。

前提条件と不変条件

  • 第1引数は check_dir_config で検証可能な有効な ap_lua_dir_cfg であること。
  • 第2引数は Lua のテーブル型であること。
  • メモリ確保はすべて cfg->pool を使用して行われるため、呼び出し側での明示的な解放は不要。

アルゴリズムまたは振る舞い

1. check_dir_config を用いて設定構造体を取得し、ap_lua_mapped_handler_spec をメモリ確保する。 2. Lua テーブルから以下のフィールドを抽出し、構造体に格納する。

  • file: スクリプトファイル名（必須）。
  • pattern: URI マッチング用の正規表現。ap_regcomp でコンパイルし、失敗時はエラーを投げる。
  • scope: apl_toscope を用いて変換。未指定時は AP_LUA_SCOPE_ONCE を適用。
  • func: 実行する関数名。未指定時はデフォルトの "handle" を使用。

    • 3. 構築した仕様を cfg->mapped_handlers 配列に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_config.c 内の cfg_methods テーブルを通じて、Lua スクリプトから match_handler として呼び出される。
  • **呼び出し先**:
  • check_dir_config
  • apr_pcalloc / apr_pstrdup / apr_palloc / apr_array_push
  • ap_regcomp
  • luaL_checktype / lua_getfield / lua_isstring / lua_tostring / lua_pop / luaL_error
  • apl_toscope

関連データ構造

  • ap_lua_dir_cfg: ディレクトリ設定を保持する構造体。
  • ap_lua_mapped_handler_spec: ハンドラの定義（ファイル名、URI パターン、スコープ、関数名）を保持する構造体。

失敗モードと境界条件

  • pattern に無効な正規表現が指定された場合、ap_regcomp が失敗し luaL_error が発生する。
  • Lua スタックの型が期待と異なる場合、luaL_checktype 等によりエラーが発生する。

保守担当者向けメモ

  • 本関数は Lua からの動的な設定登録を担うため、メモリ確保はすべて Apache のプール管理下にある。
  • func フィールドが省略された場合のデフォルト値 "handle" は、mod_lua の慣習に基づいている。
