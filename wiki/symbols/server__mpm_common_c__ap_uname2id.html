ap_uname2id

定義

  • 種別: function
  • 定義箇所: server/mpm_common.c:231

目的

ユーザー名（文字列）をシステム上の UID（uid_t）に変換します。

シグネチャまたは形状

uid_t ap_uname2id(const char *name)

入力と出力

  • **入力**: name (ユーザー名を表す文字列)
  • **出力**: 変換された uid_t 型のユーザー ID を返します。

前提条件と不変条件

  • 入力文字列の先頭が # で始まる場合、その後の数値を UID として直接解釈します。
  • それ以外の場合は、システム標準の getpwnam() を使用してユーザー情報を検索します。

アルゴリズムまたは振る舞い

1. 入力 name の先頭が # であるか確認し、該当する場合は atoi() で数値変換して即座に返します。 2. それ以外の場合、getpwnam(name) を呼び出し、ユーザー情報を取得します。 3. ユーザー情報が見つからない場合、ap_log_error を用いてエラーログを出力し、exit(1) を呼び出してプロセスを終了します。 4. 成功した場合は、取得した passwd 構造体から pw_uid を抽出して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/unix/mod_privileges.c:423
  • modules/arch/unix/mod_unixd.c:216, 300
  • modules/generators/mod_suexec.c:74
  • **呼び出し先**:
  • getpwnam() (システムライブラリ)
  • atoi() (標準ライブラリ)
  • ap_log_error() (Apache ログ出力関数)
  • exit() (標準ライブラリ)

関連データ構造

  • struct passwd (POSIX 定義のユーザー情報構造体)

失敗モードと境界条件

  • **ユーザー名未検出**: getpwnam() が NULL を返した場合、エラーログを出力してサーバープロセスを強制終了します。
  • **数値指定**: # で始まる形式の場合、atoi() の挙動に依存します。

保守担当者向けメモ

  • 本関数はエラー発生時に exit(1) を呼び出すため、呼び出し側は設定読み込みフェーズなど、プロセス終了が許容されるタイミングで使用する必要があります。
  • getpwnam() はスレッドセーフではない可能性があるため、マルチスレッド環境での利用には注意が必要です（プラットフォーム側の実装に依存します）。
