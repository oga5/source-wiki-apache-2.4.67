create_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_connect.c:58

目的

mod_proxy_connect モジュールで使用されるサーバーごとの設定構造体（connect_conf）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定情報の割り当てに使用するメモリプール。
  • server_rec *s: 設定が適用されるサーバーレコード（本関数内では未使用）。
  • **出力**:
  • void *: 初期化された connect_conf 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、返される構造体はゼロクリアされている。
  • allowed_connect_ports は、初期容量 10 の apr_array_header_t として確保される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して connect_conf サイズのメモリをプール p から確保する。 2. apr_array_make を呼び出し、許可されたポート範囲を格納するための配列 allowed_connect_ports を初期化する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の設定フェーズにおいて、モジュールの設定作成フックとして呼び出される（modules/proxy/mod_proxy_connect.c:400 にて構造体登録を確認）。
  • **呼び出し先**: apr_pcalloc, apr_array_make（いずれも APR ライブラリ関数）。

関連データ構造

  • connect_conf: 本関数で生成されるメインの設定構造体。
  • port_range: allowed_connect_ports 配列の各要素として格納されるデータ型。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc または apr_array_make がプロセスを終了させる可能性がある（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は mod_proxy_connect のサーバー設定初期化の起点である。
  • connect_conf の構造変更や、許可ポート以外の設定項目を追加する場合は、本関数での初期化処理を更新すること。
