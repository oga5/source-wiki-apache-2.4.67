merge_alias_dir_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_alias.c:109

目的

ディレクトリごとのエイリアス設定（alias_dir_conf）をマージし、親コンテキストの設定を継承またはオーバーライドします。

シグネチャまたは形状

static void *merge_alias_dir_config(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親コンテキストの設定構造体（alias_dir_conf）。
  • overridesv: 現在のコンテキストの設定構造体（alias_dir_conf）。
  • **出力**:
  • マージされた新しい alias_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は alias_dir_conf 型としてキャスト可能であること。
  • メモリは apr_pcalloc によりゼロ初期化された状態で割り当てられます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して新しい alias_dir_conf 構造体を割り当てます。 2. redirects 配列を apr_array_append で結合します。 3. alias、alias_fake、handler、redirect、redirect_status などの各フィールドについて、overrides 側で設定（_set フラグ）が行われていない場合は base 側の値を採用し、設定がある場合は overrides 側の値を採用します。 4. alias_set および redirect_set は、両コンテキストの論理和（OR）をとります。 5. allow_relative および alias_preserve_path は、overrides 側がデフォルト値（ALIAS_FLAG_DEFAULT）でない場合にのみ overrides 側の値を採用します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server の設定マージ機構（modules/mappers/mod_alias.c:761 にて参照）。
  • **呼び出し先**: apr_pcalloc（APR）、apr_array_append（APR）。

関連データ構造

  • alias_dir_conf: ディレクトリごとのエイリアス設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が異常終了する可能性があります。
  • _set フラグが正しく管理されていない場合、意図しない設定の継承が発生する可能性があります。

保守担当者向けメモ

  • 本関数は alias_dir_conf の各フィールドに対して明示的なマージロジックを実装しています。新しい設定項目を追加した際は、本関数内でのマージ処理を忘れずに追加してください。
