format_tag

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_headers.c:117

目的

HTTP ヘッダー値に含まれる動的な要素（フォーマットタグ）を表現し、実行時に値を生成するための関数ポインタと引数を保持する。

シグネチャまたは形状

typedef struct {
    format_tag_fn *func;
    char *arg;
} format_tag;

入力と出力

  • **入力**: func は format_tag_fn 型の関数ポインタであり、arg はその関数に渡される引数文字列。
  • **出力**: process_tags 等の関数によって、この構造体が保持するロジックに基づき、リクエストコンテキストから動的な値が生成される。

前提条件と不変条件

  • format_tag の配列は、Header または RequestHeader ディレクティブの解析時に parse_format_tag によって生成される。
  • func は、リクエスト処理中に動的な値を抽出するための適切な関数を指している必要がある。

アルゴリズムまたは振る舞い

1. ヘッダー値の解析時、文字列内のタグが分解され、format_tag 構造体の配列として格納される。 2. リクエスト処理の実行時、process_tags がこの配列を走査する。 3. 各 format_tag の func が呼び出され、arg を用いて動的な値（時間、環境変数、リクエスト情報など）が生成される。 4. 生成された値が連結され、最終的なヘッダー値が構成される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • parse_format_tag: ヘッダー文字列を分解して format_tag を生成する。
  • parse_misc_string: 文字列解析の一部として format_tag を操作する。
  • process_tags (推論): hdr->ta に格納された format_tag 配列を走査する。
  • **呼び出し先**:
  • func フィールドに格納された関数ポインタ（format_tag_fn 型）。

関連データ構造

  • header_entry: format_tag の配列（ta）を保持する親構造体。
  • apr_array_header_t: format_tag 構造体を格納する動的配列。

失敗モードと境界条件

  • parse_format_tag が解析に失敗した場合、ヘッダー設定の構築が中断される可能性がある。
  • func が NULL の場合、実行時に未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • format_tag は mod_headers.c 内の動的ヘッダー生成の基盤である。
  • 新しい動的タグを追加する場合は、format_tag_fn に適合する関数を実装し、parse_format_tag で認識させる必要がある。
  • format_tag 配列のメモリは request_rec->pool または cmd->pool から割り当てられるため、明示的な解放は不要である。
