md_create

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:198

目的

指定されたドメインリストに基づき、初期化されたマネージドドメイン（md_t）オブジェクトを生成します。

シグネチャまたは形状

md_t *md_create(apr_pool_t *p, apr_array_header_t *domains)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • domains: ドメイン名のリストを含む apr_array_header_t。
  • **出力**:
  • 初期化済みの md_t 構造体へのポインタ。

前提条件と不変条件

  • domains 配列は md_array_str_compact によって正規化されます。
  • 生成された md_t の name フィールドには、domains 配列の最初の要素が設定されます。

アルゴリズムまたは振る舞い

1. md_create_empty(p) を呼び出し、空の md_t 構造体を確保・初期化します。 2. md_array_str_compact を使用して、入力された domains リストを正規化し、md->domains に格納します。 3. domains 配列のインデックス 0 にある文字列を、md->name として割り当てます。 4. 完成した md_t 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_config.c:420
  • modules/md/mod_md_config.c:490
  • **呼び出し先**:
  • md_create_empty
  • md_array_str_compact
  • APR_ARRAY_IDX

関連データ構造

  • md_t: 管理対象ドメインの設定と状態を保持するメイン構造体。
  • apr_pool_t: メモリ管理用プール。
  • apr_array_header_t: ドメイン名リストを保持する配列構造体。

失敗モードと境界条件

  • domains 配列が空または無効な場合、md_array_str_compact の挙動および APR_ARRAY_IDX によるインデックス参照に依存します。

保守担当者向けメモ

  • 本関数は md_t の生成における標準的なエントリーポイントです。
  • メモリの生存期間は引数として渡される apr_pool_t に完全に依存します。
