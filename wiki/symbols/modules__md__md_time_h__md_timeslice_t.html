md_timeslice_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_time.h:60

目的

証明書の更新や警告通知など、特定のイベントが発生するまでの時間枠（タイムスライス）を定義および管理するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_interval_time_t norm; /* if > 0, normalized base length */
    apr_interval_time_t len;  /* length of the timespan */
} md_timeslice_t;

入力と出力

  • norm: 正規化された基準長（apr_interval_time_t）。0より大きい場合、基準となる長さを表します。
  • len: 実際の時間枠の長さ（apr_interval_time_t）。

前提条件と不変条件

  • apr_interval_time_t 型を使用しているため、APR（Apache Portable Runtime）の時間単位（マイクロ秒）に依存します。
  • norm が 0 より大きい場合、それは正規化された基準値として扱われます。

アルゴリズムまたは振る舞い

本構造体は、証明書の有効期限に関連する「更新ウィンドウ（renew_window）」や「警告ウィンドウ（warn_window）」の長さを保持するために利用されます。具体的な計算ロジックは md_time.c 内の関数によって処理されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/md/md.h: renew_window および warn_window として保持。
  • modules/md/md_ocsp.c / md_ocsp.h: OCSP 更新ウィンドウの定義に使用。
  • modules/md/md_reg.c / md_reg.h: md_reg_set_renew_window_default および md_reg_set_warn_window_default 関数の引数として使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • apr_interval_time_t: APR の時間間隔型。
  • md_reg_t: md_reg_set_... 関数を通じて本構造体を設定値として保持します。

失敗モードと境界条件

  • norm または len に負の値が設定された場合の挙動については、本ヘッダー内では定義されていません。
  • apr_interval_time_t の最大値を超える値が設定された場合のオーバーフローについては、APR の仕様に依存します。

保守担当者向けメモ

  • 本構造体は md_reg や md_ocsp といった上位モジュールで設定値として頻繁に参照されます。変更を加える際は、これらのモジュールにおける設定値の整合性に注意してください。
  • norm と len の使い分け（正規化された値と実際の値の差異）については、md_time.c の実装ロジックを参照してください。
