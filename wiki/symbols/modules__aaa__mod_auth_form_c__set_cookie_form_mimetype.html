set_cookie_form_mimetype

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:237

目的

Apache HTTP Server の設定ディレクティブ AuthFormMimetype を処理し、認証フォームに関連付けられた MIME タイプを設定値として保持します。

シグネチャまたは形状

static const char *set_cookie_form_mimetype(cmd_parms *cmd, void *config, const char *mimetype)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: 現在のディレクトリ設定構造体 (auth_form_config_rec)。
  • mimetype: 設定ファイルから読み取られた MIME タイプの文字列。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は check_string の戻り値（エラーメッセージ）を返します。

前提条件と不変条件

  • config 引数は auth_form_config_rec 型にキャスト可能である必要があります。
  • conf->mimetype_set は、この関数が呼び出されることで 1 に更新されます。

アルゴリズムまたは振る舞い

1. config を auth_form_config_rec 型のポインタ conf にキャストします。 2. conf->mimetype に引数 mimetype を代入します。 3. conf->mimetype_set フラグを 1 に設定します。 4. check_string(cmd, mimetype) を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c:365 において、AP_INIT_TAKE1 マクロを通じて AuthFormMimetype ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: check_string 関数を呼び出します。

関連データ構造

  • auth_form_config_rec: ディレクトリごとの認証設定を保持する構造体。本関数はこの構造体の mimetype フィールドと mimetype_set フラグを更新します。

失敗モードと境界条件

  • check_string がエラーを返した場合、そのエラーメッセージがそのまま本関数の戻り値となります。

保守担当者向けメモ

  • 本関数は AuthFormMimetype ディレクティブ専用のセッターです。設定の検証ロジックは check_string に依存しているため、検証ルールを変更する場合は check_string を参照してください。
