h2_ws_register_hooks

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_ws.c:338

目的

HTTP/2 上での WebSocket 通信をサポートするために必要なフックおよび出力フィルタを Apache HTTP Server の処理系に登録します。

シグネチャまたは形状

void h2_ws_register_hooks(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として Apache のフックおよびフィルタチェーンに登録を行う）

前提条件と不変条件

  • 本関数はモジュールの初期化フェーズで呼び出される必要があります。
  • H2_USE_WEBSOCKETS マクロが定義されている環境でのみ有効に機能します。

アルゴリズムまたは振る舞い

1. ap_hook_post_read_request を使用して、リクエスト読み込み後のフックに ws_post_read を登録します。 2. ap_register_output_filter を使用して、H2_C2_WS_OUT という名称の出力フィルタを AP_FTYPE_NETWORK 型で登録し、その処理関数として h2_c2_ws_filter_out を指定します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/mod_http2.c:203 (モジュール初期化時)
  • 呼び出し先:
  • ap_hook_post_read_request (Apache API)
  • ap_register_output_filter (Apache API)
  • ws_post_read (本モジュール内関数)
  • h2_c2_ws_filter_out (本モジュール内関数)

関連データ構造

  • c2_ws_out_filter_handle: 登録された出力フィルタのハンドルを保持するグローバル変数。

失敗モードと境界条件

  • Apache のフィルタ登録 API が失敗した場合の挙動については、API の仕様に依存します。

保守担当者向けメモ

  • 本関数は h2_ws.h で宣言されており、mod_http2 の初期化プロセスの一部として実行されます。
  • フィルタ H2_C2_WS_OUT は、HTTP/2 ストリームのレスポンスをインターセプトし、WebSocket ハンドシェイクの検証および変換を行うために不可欠です。
