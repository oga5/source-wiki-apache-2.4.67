merge_cgi_config

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgi.c:112

目的

サーバー設定の階層構造において、親コンテキスト（basev）とオーバーライド設定（overridesv）をマージし、有効な cgi_server_conf を決定する。

シグネチャまたは形状

static void *merge_cgi_config(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当て用のプール。
  • basev: 親コンテキストの cgi_server_conf 構造体。
  • overridesv: 現在のコンテキストの cgi_server_conf 構造体。
  • **出力**:
  • マージ後の cgi_server_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は cgi_server_conf 型にキャスト可能であること。
  • 戻り値は、設定の継承ルールに従い、overrides または base のいずれか一方を指す。

アルゴリズムまたは振る舞い

1. basev と overridesv をそれぞれ cgi_server_conf 型にキャストする。 2. overrides 構造体の logname メンバが設定されている（非NULL）かを確認する。 3. logname が存在する場合は overrides を返し、存在しない場合は base を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_cgi.c:777（サーバー設定のマージ処理において呼び出される）。
  • **呼び出し先**: 直接的な関数呼び出しはなし。

関連データ構造

  • cgi_server_conf: サーバー全体の設定を保持する構造体。

失敗モードと境界条件

  • basev または overridesv が NULL の場合、キャストおよびメンバ参照時に未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は単純なポインタ選択によるマージを行っている。将来的に他の設定項目（タイムアウト値やバッファサイズ等）を個別にマージする必要が生じた場合は、新しい構造体を p から割り当てて値をコピーする実装に変更する必要がある。
