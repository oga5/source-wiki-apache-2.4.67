h2_stream

定義

  • 種別: struct
  • 定義箇所: test/clients/h2ws.c:345

目的

HTTP/2 セッション内における個別のストリームの状態を保持・管理するために使用されます。

シグネチャまたは形状

ソースコード上では struct h2_stream として定義されています。具体的なメンバ構成は提供された抜粋には含まれていませんが、ストリームID、URI、およびコールバック関数を保持する役割を担います。

入力と出力

  • 入力: ストリームのライフサイクル管理に関連するイベントやデータ。
  • 出力: ストリームの状態変化や、クローズ時のコールバック実行。

前提条件と不変条件

  • h2_session によって管理されるストリームリストの一部として存在することが想定されます。
  • ストリームのライフサイクル（作成、データ送信、クローズ）は nghttp2 のコールバックと連動して制御されます。

アルゴリズムまたは振る舞い

  • ws_stream_submit() を通じて CONNECT メソッドを用いた WebSocket ストリームを開始します。
  • nghttp2 のイベントループ（h2_session_run）内で、フレーム受信やヘッダー処理に応じてストリームの状態を更新します。
  • ストリーム終了時には、関連するコールバック関数が呼び出される設計となっています。

呼び出し元と呼び出し先

  • 呼び出し元: h2ws.c 内のストリーム管理関数（ws_stream_submit 等）から参照されます。
  • 呼び出し先: 関連するコールバック関数（h2_stream_closed_cb 等）が定義されており、ストリームの状態変化に応じて実行されます。
  • 注意: ツールによる解析では、modules/http2/ 配下のファイルからも参照されていることが示唆されていますが、これらが test/clients/h2ws.c の h2_stream と同一の定義を指しているか、あるいは同名の別構造体であるかの断定は、提供されたソースの範囲内では不可能です。

関連データ構造

  • h2_session: ストリームを保持する親構造体。
  • ws_stream: WebSocket ストリーム固有の制御構造。

失敗モードと境界条件

  • メモリ管理: テスト用ツールであるため、終了時に明示的な全メモリ解放が行われない場合があります。
  • 接続状態: 非ブロッキングソケットを使用しているため、EWOULDBLOCK 等の I/O 状態を nghttp2 のコールバック内で適切に処理する必要があります。

保守担当者向けメモ

  • 本構造体は nghttp2 のコールバック API に強く依存しています。変更を加える際は nghttp2 の仕様を確認してください。
  • test/clients/h2ws.c 内の定義と、modules/http2/ 内の同名構造体との関係性については、必要に応じて依存関係を精査してください。
