OpenSCManager

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/service.c:80

目的

Windowsサービスコントロールマネージャ（SCM）への接続ハンドルを取得するためのラッパー関数です。実行環境のOSがUnicode対応かANSI対応かに応じて、適切なWindows API（OpenSCManagerW または OpenSCManagerA）を動的に選択して呼び出します。

シグネチャまたは形状

SC_HANDLE OpenSCManager(const void *lpMachine, const void *lpDatabase, DWORD dwAccess)

入力と出力

  • **入力**:
  • lpMachine: ターゲットコンピュータ名（NULLの場合はローカルマシン）。
  • lpDatabase: サービスコントロールマネージャのデータベース名（NULLの場合は SERVICES_ACTIVE_DATABASE）。
  • dwAccess: SCMへのアクセス権限（例: SC_MANAGER_CONNECT）。
  • **出力**:
  • 成功時: SCMへのハンドル（SC_HANDLE）。
  • 失敗時: NULL。

前提条件と不変条件

  • 静的変数 pfn_OpenSCManager が初期化されていない場合、実行時に一度だけOSの文字セット判定を行い、関数ポインタを解決します。
  • APR_HAS_UNICODE_FS および APR_HAS_ANSI_FS の定義に基づき、コンパイル時に適切なAPIが選択されます。

アルゴリズムまたは振る舞い

1. 静的関数ポインタ pfn_OpenSCManager が未設定であれば、IF_WIN_OS_IS_UNICODE または ELSE_WIN_OS_IS_ANSI マクロを使用して、対応するWindows APIのアドレスを代入します。 2. 解決された関数ポインタを介して、引数 lpMachine, lpDatabase, dwAccess を渡してSCMを呼び出します。 3. 取得したハンドルを呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/service.c 内の各所（サービスインストール、アンインストール、状態確認時など）。
  • support/win32/ApacheMonitor.c（外部ツール）。
  • **呼び出し先**:
  • OpenSCManagerW または OpenSCManagerA（Windows API）。

関連データ構造

  • SC_HANDLE: Windows APIで定義されるサービスハンドル型。
  • DWORD: アクセス権限を指定する型。

失敗モードと境界条件

  • SCMへの接続権限がない場合や、指定したマシン/データベースが存在しない場合は NULL を返します。
  • 呼び出し元は、返されたハンドルが NULL でないことを確認し、不要になった時点で CloseServiceHandle を呼び出す必要があります。

保守担当者向けメモ

  • 本関数は、service.c 内で直接 OpenSCManager を呼び出す際の抽象化レイヤーとして機能しています。
  • pfn_OpenSCManager は静的変数であるため、マルチスレッド環境下での初期化競合については、呼び出し側のコンテキスト（初期化順序）に依存します。
