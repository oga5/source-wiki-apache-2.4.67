status_get_md_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_status.c:251

目的

指定された Managed Domain (MD) の現在の状態、証明書情報、および更新ジョブの進捗状況を JSON 形式で生成します。

シグネチャまたは形状

static apr_status_t status_get_md_json(md_json_t **pjson, const md_t *md, 
                                       md_reg_t *reg, md_ocsp_reg_t *ocsp, 
                                       int with_logs, apr_pool_t *p)

入力と出力

  • **入力**:
  • md: 対象の MD 構造体。
  • reg: MD レジストリハンドル。
  • ocsp: OCSP レジストリハンドル。
  • with_logs: ログ情報を含めるかどうかのフラグ。
  • p: メモリ割り当て用プール。
  • **出力**:
  • pjson: 生成された JSON オブジェクトへのポインタ。
  • 戻り値: 処理成功時は APR_SUCCESS、失敗時はエラーコード。

前提条件と不変条件

  • p (apr_pool_t) は有効であること。
  • md および reg は初期化済みであること。
  • JSON 生成には md_json ライブラリの規約に従うこと。

アルゴリズムまたは振る舞い

1. md_to_public_json を使用して MD の基本情報を JSON 化。 2. md_reg_get_pubcert を繰り返し呼び出し、MD に関連付けられた全証明書を収集して status_get_certs_json で JSON に変換・統合。 3. 更新期限 (renew_at)、ステープリング設定、監視状態を JSON に追加。 4. job_loadj を使用してステージング領域のジョブ状態を読み込み、更新が必要な場合は get_staging_certs_json を通じて更新用証明書情報を JSON にマージ。 5. 処理中にエラーが発生した場合、MD_KEY_ERROR キーにエラーコードを記録。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_status.c:318
  • modules/md/md_status.c:332
  • **呼び出し先**:
  • md_to_public_json, get_cert_count, md_reg_get_pubcert, status_get_certs_json, md_json_setj, md_reg_renew_at, md_json_set_time, md_json_setb, job_loadj, md_reg_should_renew, get_staging_certs_json, md_json_setl

関連データ構造

  • md_t: 管理対象ドメイン。
  • md_reg_t: レジストリ。
  • md_json_t: JSON 表現。
  • md_job_t: (内部的に job_loadj を介して参照) 更新ジョブ。

失敗モードと境界条件

  • job_loadj が ENOENT を返した場合、ジョブが存在しないものとして正常系として処理を継続。
  • その他のエラー発生時は leave ラベルへジャンプし、エラー情報を JSON に付与して終了。
  • 証明書取得失敗時は rv が保持され、最終的にエラーとして出力される。

保守担当者向けメモ

  • 本関数は static であり、同一ファイル内のステータス生成関数からのみ利用される。
  • MD_KEY_* 定数を使用して JSON 構造を定義しているため、スキーマ変更時は md_json.h の定義と整合性を取ること。
