ap_expr_eval_comp

定義

  • 種別: function
  • 定義箇所: server/util_expr_eval.c:234

目的

抽象構文木（AST）の比較演算ノードを評価し、真偽値（1 または 0）を算出します。

シグネチャまたは形状

static int ap_expr_eval_comp(ap_expr_eval_ctx_t *ctx, const ap_expr_t *node)

入力と出力

  • **入力**:
  • ctx: 評価コンテキスト（リクエスト情報、エラー格納用バッファ等）。
  • node: 評価対象の AST ノード。node_op に演算子、node_arg1/node_arg2 にオペランドを保持。
  • **出力**:
  • 成功時: 比較結果として 1（真）または 0（偽）を返却。
  • 失敗時: -1 を返し、ctx->err にエラーメッセージを格納。

前提条件と不変条件

  • node は比較演算子（op_EQ, op_STR_EQ, op_IN, op_REG 等）を持つノードである必要があります。
  • 評価中に生成される文字列は、ctx->p（リクエストプール）から割り当てられます。
  • op_IN 評価時、e2 が op_ListFuncCall の場合、func ポインタは非 NULL である必要があります。

アルゴリズムまたは振る舞い

1. **オペランド取得**: ap_expr_eval_word を呼び出し、node_arg1 および node_arg2 を文字列として評価します。 2. **演算子分岐**: node_op に応じて以下の処理を実行します。

  • **数値比較 (op_EQ, op_LT 等)**: intstrcmp を使用して数値を比較します。
  • **文字列比較 (op_STR_EQ 等)**: strcmp を使用して辞書順比較を行います。
  • **包含判定 (op_IN)**:
  • リスト要素 (op_ListElement) の場合、順次比較します。
  • 関数呼び出し (op_ListFuncCall) の場合、プロバイダー関数を実行し、結果の配列内に needle が含まれるか確認します。
  • **正規表現 (op_REG, op_NRE)**:
  • ap_regexec を実行します。
  • キャプチャグループが存在する場合、ctx->re_source にマッチした文字列を格納します。

    • 3. **エラー処理**: 未定義の演算子ノードが渡された場合、エラーメッセージをセットし -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_expr_eval.c:370
  • server/util_expr_eval.c:801
  • **呼び出し先**:
  • ap_expr_eval_word: オペランドの解決に使用。
  • intstrcmp, strcmp: 比較処理に使用。
  • ap_regexec: 正規表現マッチングに使用。
  • ap_array_str_contains: リスト内の包含判定に使用。

関連データ構造

  • ap_expr_eval_ctx_t: 評価コンテキスト。
  • ap_expr_t: AST ノード構造体。
  • ap_expr_list_func_t: リスト関数プロバイダーの型定義。

失敗モードと境界条件

  • **内部エラー**: 未知の演算子ノードが渡された場合、-1 を返し評価を中断します。
  • **正規表現**: ap_regexec が失敗した場合、または正規表現のコンパイル状態が不正な場合の挙動は、呼び出し元の ap_regexec 実装に依存します。
  • **再帰制限**: 本関数自体は再帰しませんが、呼び出し元の ap_expr_eval 等で AP_EXPR_MAX_RECURSION による制限を受けます。

保守担当者向けメモ

  • op_IN の op_ListFuncCall 処理において、プロバイダー関数が NULL を返した場合、0（偽）として扱います。
  • 正規表現のキャプチャグループ（$0〜$9）の更新は、regex->re_nsub > 0 の場合にのみ行われる設計です。
