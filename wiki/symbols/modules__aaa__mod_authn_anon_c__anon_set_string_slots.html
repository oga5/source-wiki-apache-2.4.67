anon_set_string_slots

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_anon.c:88

目的

Apache 設定ファイル内の Anonymous ディレクティブを解析し、匿名認証を許可するユーザー名のリストを構築します。

シグネチャまたは形状

static const char *anon_set_string_slots(cmd_parms *cmd, void *my_config, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • my_config: ディレクトリごとの設定構造体 (authn_anon_config_rec)。
  • arg: 設定ファイルから読み込まれたユーザー名文字列。
  • **出力**:
  • 成功時は NULL を返します。
  • エラー時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • my_config は authn_anon_config_rec 型としてキャスト可能である必要があります。
  • arg が空文字列の場合、エラーを返します。

アルゴリズムまたは振る舞い

1. arg が空でないことを確認し、空の場合はエラーメッセージを返します。 2. 設定構造体の anyuserid フラグが未設定であることを確認します。 3. arg が "*" の場合、anyuserid を 1 に設定し、任意のユーザー名を許可します。 4. それ以外の場合、cmd->pool からメモリを確保し、新しい anon_auth_user レコードを作成して、既存のリストの先頭に追加します。 5. 処理終了後、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_anon.c:116 において、AP_INIT_ITERATE マクロを通じて設定ディレクティブ Anonymous のハンドラとして登録されています。
  • **呼び出し先**: Apache APR のメモリ管理関数 apr_palloc を呼び出します。

関連データ構造

  • authn_anon_config_rec: 設定を保持する構造体。
  • anon_auth_user: 許可ユーザー名を保持する連結リスト構造体。
  • cmd_parms: 設定解析用の Apache 内部構造体。

失敗モードと境界条件

  • arg が空文字列の場合、設定エラーとしてメッセージを返します。
  • anyuserid が既に設定されている場合、新しいユーザー名の追加は行われません。

保守担当者向けメモ

  • 本関数は Anonymous ディレクティブの解析専用です。
  • リストへの追加は連結リストの先頭挿入で行われるため、順序は設定ファイルの記述と逆順になります。
