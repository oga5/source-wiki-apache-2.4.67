authn_dbd_merge_conf

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_dbd.c:51

目的

ディレクトリ単位の設定において、親コンテキスト（base）の設定と現在のコンテキスト（add）の設定をマージし、新しい設定構造体を生成する。

シグネチャまたは形状

static void *authn_dbd_merge_conf(apr_pool_t *pool, void *BASE, void *ADD)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する APR プール。
  • BASE: 親ディレクトリの設定（authn_dbd_conf *）。
  • ADD: 現在のディレクトリの設定（authn_dbd_conf *）。
  • **出力**:
  • マージされた新しい authn_dbd_conf 構造体へのポインタ。

前提条件と不変条件

  • BASE および ADD は authn_dbd_conf 型としてキャスト可能であること。
  • pool はリクエストまたは設定のライフサイクルに紐付いた有効なプールであること。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、新しい authn_dbd_conf 構造体用のメモリを確保する。 2. user クエリ文字列について、ADD 側が設定されていればそれを採用し、そうでなければ BASE 側の値を継承する。 3. realm クエリ文字列について、ADD 側が設定されていればそれを採用し、そうでなければ BASE 側の値を継承する。 4. マージ済みの構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_dbd.c:302（モジュール定義構造体内で参照されている）。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • authn_dbd_conf: 本モジュールが使用するディレクトリ設定構造体。user および realm のクエリ文字列を保持する。

失敗モードと境界条件

  • pool のメモリ不足時には apr_palloc がアボートする可能性がある。
  • BASE または ADD が NULL の場合、構造体メンバへのアクセスによりセグメンテーション違反が発生する可能性がある（Apache の設定マージの仕組み上、通常は適切な初期化が行われる）。

保守担当者向けメモ

  • 本関数は Apache の設定マージ・フックとして機能する。
  • 継承ロジックは単純な上書き（add->field == NULL ? base->field : add->field）であり、クエリ文字列の連結などは行わない。
