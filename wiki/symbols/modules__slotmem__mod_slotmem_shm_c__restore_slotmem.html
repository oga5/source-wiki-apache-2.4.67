restore_slotmem

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:196

目的

永続化ファイルから共有メモリ領域へデータを読み込み、MD5チェックサムと記述子（sharedslotdesc_t）を用いてデータの整合性を検証・復元する。

シグネチャまたは形状

static apr_status_t restore_slotmem(sharedslotdesc_t *desc,
                                    const char *storename, apr_size_t size,
                                    apr_pool_t *pool)

入力と出力

  • **入力**:
  • desc: 共有メモリの構成情報を含む構造体へのポインタ。
  • storename: 永続化ファイルのパス。
  • size: 共有メモリセグメントの総サイズ。
  • pool: ファイル操作に使用するメモリプール。
  • **出力**:
  • apr_status_t: 処理結果。成功時は APR_SUCCESS、不整合時は APR_EMISMATCH、読み込み不足時は APR_INCOMPLETE 等を返す。

前提条件と不変条件

  • 共有メモリ領域は [desc] [num_free] [data_slots] [inuse_array] の順でレイアウトされている必要がある。
  • AP_SLOTMEM_OFFSET 定数により、データ領域の開始位置が厳密に定義されている。

アルゴリズムまたは振る舞い

1. 指定された storename を読み書きモードで開く。 2. ファイルからデータ領域（desc の直後から size まで）を読み込む。 3. ファイルの末尾（EOF）に達していない場合、MD5チェックサムを読み込み、メモリ上のデータから計算したハッシュ値と比較する。 4. さらに記述子（desc）の整合性を確認するため、ファイル末尾の記述子データとメモリ上の desc を比較する。 5. 互換性のため、EOFに達した場合はチェックサムや記述子の検証をスキップし、成功として扱う。 6. 最後にファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/slotmem/mod_slotmem_shm.c:430（slotmem_attach 内で呼び出される）。
  • **呼び出し先**: apr_file_open, apr_file_read_full, apr_file_eof, apr_md5, memcmp, ap_log_error, apr_file_close。

関連データ構造

  • sharedslotdesc_t: スロットの構成情報（サイズ、個数、型）を保持するヘッダー。

失敗モードと境界条件

  • APR_EMISMATCH: MD5ハッシュまたは記述子の内容が一致しない場合に発生。
  • APR_INCOMPLETE: ファイルの読み込みサイズが期待値と異なる場合に発生。
  • APR_EOF: ファイルが予期せず終了した場合の処理分岐。
  • 互換性対応: 古い形式の永続化ファイルでチェックサムや記述子情報が存在しない場合、EOF判定により検証をバイパスして成功を返す。

保守担当者向けメモ

  • 本関数は mod_slotmem_shm.c の永続化ロジックの中核であり、ファイルフォーマットの変更は store_slotmem との整合性に注意すること。
  • AP_SLOTMEM_OFFSET はメモリレイアウトの境界を決定するため、構造体定義の変更時には必ず確認が必要。
