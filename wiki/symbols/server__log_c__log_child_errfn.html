log_child_errfn

定義

  • 種別: function
  • 定義箇所: server/log.c:302

目的

子プロセス生成時に発生したエラーを Apache のエラーログへ記録するためのコールバック関数です。

シグネチャまたは形状

static void log_child_errfn(apr_pool_t *pool, apr_status_t err, const char *description)

入力と出力

  • **入力**:
  • pool: エラー処理に使用されるメモリプール。
  • err: 発生した APR エラーコード。
  • description: エラー内容を記述した文字列。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • apr_procattr_child_errfn_set を介して、子プロセスのエラーハンドラとして登録されることを前提としています。
  • ap_log_error を呼び出すため、サーバーのログ記録システムが初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 受け取った err（エラーコード）と description（エラー説明）を引数として ap_log_error を呼び出します。 2. ログレベルは APLOG_ERR として記録されます。 3. ログメッセージには APLOGNO(00088) が付与されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/log.c:336
  • server/log.c:1782

    • （いずれも apr_procattr_child_errfn_set の引数として参照されています）

  • **呼び出し先**:
  • ap_log_error (Apache HTTP Server のログ出力関数)

関連データ構造

  • apr_pool_t: APR メモリプール。
  • apr_status_t: APR エラーコード型。

失敗モードと境界条件

  • ap_log_error 内部のログ出力処理に依存します。ログファイルが書き込み不可である場合や、ログシステムが未初期化の状態では、ログ出力が失敗または無視される可能性があります。

保守担当者向けメモ

  • 本関数は apr_procattr_child_errfn_set に渡すためのシグネチャに適合させています。変更の際は APR のコールバック定義との整合性に注意してください。
