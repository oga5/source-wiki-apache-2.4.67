priv_cfg_cleanup

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_privileges.c:63

目的

priv_cfg 構造体が保持する特権セット（priv_set_t）を解放し、メモリリークを防ぐためのクリーンアップ関数です。

シグネチャまたは形状

static apr_status_t priv_cfg_cleanup(void *CFG)

入力と出力

  • **入力**: void *CFG - priv_cfg 構造体へのポインタ。
  • **出力**: apr_status_t - 常に APR_SUCCESS を返します。

前提条件と不変条件

  • 入力された CFG は priv_cfg 型としてキャスト可能である必要があります。
  • cfg->priv および cfg->child_priv は、priv_allocset 等で事前に確保された有効な特権セットである必要があります。

アルゴリズムまたは振る舞い

1. 引数 CFG を priv_cfg * 型にキャストします。 2. priv_freeset() を呼び出し、cfg->priv を解放します。 3. priv_freeset() を呼び出し、cfg->child_priv を解放します。 4. APR_SUCCESS を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/unix/mod_privileges.c:107 において、apr_pool_cleanup_register の引数として登録されています。
  • **呼び出し先**: priv_freeset（Solaris 特権 API）。

関連データ構造

  • priv_cfg: 本関数が管理対象とする設定構造体。
  • priv_set_t: priv_freeset によって解放される特権セットの型。

失敗モードと境界条件

  • CFG が NULL の場合、priv_freeset 内部でセグメンテーションフォールトが発生する可能性があります。
  • 既に解放済みのポインタを渡した場合の挙動は、priv_freeset の実装（OS 依存）に準拠します。

保守担当者向けメモ

  • 本関数は Apache のプールクリーンアップ機構に登録されることを前提としています。
  • priv_cfg 構造体のライフサイクルが終了する際に自動的に呼び出されるため、手動での呼び出しは不要です。
