deref_options

定義

  • 種別: enum
  • 定義箇所: include/util_ldap.h:98

目的

LDAP 検索操作におけるエイリアス（別名）の参照（dereferencing）挙動を制御するための列挙型です。

シグネチャまたは形状

typedef enum {
    never=LDAP_DEREF_NEVER,
    searching=LDAP_DEREF_SEARCHING,
    finding=LDAP_DEREF_FINDING,
    always=LDAP_DEREF_ALWAYS
} deref_options;

入力と出力

  • 入力: LDAP 検索関数（uldap_cache_compare 等）の引数として、エイリアス解決のポリシーを指定するために使用されます。
  • 出力: 該当なし（列挙値の定義）。

前提条件と不変条件

  • 本列挙型は APR_HAS_LDAP が定義されている環境でのみ有効です。
  • 各列挙値は、基盤となる apr_ldap.h（または LDAP SDK）で定義された LDAP_DEREF_* 定数に直接マッピングされます。

アルゴリズムまたは振る舞い

LDAP サーバーに対する検索クエリ実行時、エイリアスオブジェクトに遭遇した際の挙動を以下のいずれかに決定します。

  • never: エイリアスを一切参照しません（デフォルト）。
  • searching: 検索のサブツリー内では参照しますが、検索のベースオブジェクト自体は参照しません。
  • finding: 検索のベースオブジェクトは参照しますが、サブツリー内では参照しません。
  • always: 検索のベースオブジェクトおよびサブツリー内の両方で参照します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • include/util_ldap.h: util_ldap_config_t 構造体メンバおよび関数プロトタイプ（binddn 等を含む関数群）。
  • modules/aaa/mod_authnz_ldap.c: 設定保持用構造体メンバ。
  • modules/ldap/util_ldap.c: LDAP 検索処理の実装関数。
  • 呼び出し先:
  • 本シンボル自体は定数定義であり、直接的な関数呼び出しは行いません。

関連データ構造

  • util_ldap_config_t: LDAP 設定を保持する構造体で、この型のメンバ deref を含みます。

失敗モードと境界条件

  • 無効な値が指定された場合、LDAP サーバー側の実装に依存したエラーが発生する可能性があります。
  • サーバーがエイリアスをサポートしていない場合、指定したオプションが無視されることがあります。

保守担当者向けメモ

  • 本シンボルは mod_ldap の設定インターフェースの一部です。LDAP サーバーの仕様変更や apr_ldap の更新に伴い、マッピング先の定数に変更がないか確認してください。
