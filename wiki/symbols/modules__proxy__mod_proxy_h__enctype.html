enctype

定義

  • 種別: enum
  • 定義箇所: modules/proxy/mod_proxy.h:75

目的

URL の構成要素（パス、クエリ、ユーザー情報等）に応じた適切なエンコーディング規則を指定するために使用されます。

シグネチャまたは形状

enum enctype {
    enc_path,    // パス用
    enc_search,  // クエリ文字列用
    enc_user,    // ユーザー情報用
    enc_fpath,   // ファイルパス用
    enc_parm     // パラメータ用
};

入力と出力

  • 入力: ap_proxy_canonenc および ap_proxy_canonenc_ex 関数の引数として渡されます。
  • 出力: 指定されたエンコーディング規則に基づき、URL の各部分が正規化された文字列として返されます。

前提条件と不変条件

  • ap_proxy_canonenc 等の関数において、URL のコンテキストに一致する列挙値を選択する必要があります。

アルゴリズムまたは振る舞い

  • enc_path: URL パスセグメントのエンコーディングを適用します。
  • enc_search: クエリ文字列（? 以降）のエンコーディングを適用します。
  • enc_user: ユーザー名やパスワードなどの認証情報のエンコーディングを適用します。
  • enc_fpath: ファイルシステムパスの正規化を伴うエンコーディングを適用します。
  • enc_parm: 一般的なパラメータのエンコーディングを適用します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/mod_proxy.h 内の ap_proxy_canonenc および ap_proxy_canonenc_ex がこの型を引数として受け取ります。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数を呼び出すことはありません。
  • 注意: mod_proxy_balancer.c やテストコード内の enctype という文字列は、HTML の form 属性を指しており、本シンボルとは無関係です。

関連データ構造

  • なし。

失敗モードと境界条件

  • 不適切な enctype を指定した場合、URL の正規化が正しく行われず、バックエンドサーバーへのリクエストが不正な形式になる可能性があります。

保守担当者向けメモ

  • 本シンボルはプロキシの URL 正規化ロジックの根幹です。新しいエンコーディング規則が必要な場合は、この列挙型を拡張し、対応する処理を proxy_util.c に実装する必要があります。
