server_create

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_express.c:73

目的

mod_proxy_express モジュールで使用されるサーバー単位の設定構造体 (express_server_conf) をメモリ上に割り当て、初期化する。

シグネチャまたは形状

static void *server_create(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定構造体の割り当てに使用するメモリプール (apr_pool_t)。
  • s: 設定が適用されるサーバーレコード (server_rec)。
  • **出力**:
  • 初期化済みの express_server_conf 構造体へのポインタを void * として返す。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全メンバはゼロクリアされた状態で開始される。
  • 戻り値の構造体は、Apache の設定ライフサイクルにおいてサーバー単位の設定として利用される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、メモリプール p から express_server_conf サイズ分のメモリを確保する。 2. 確保した構造体の各メンバを以下の通り初期化する:

  • dbmfile: NULL に設定。
  • dbmtype: "default" に設定。
  • enabled: 0 (無効) に設定。

    • 3. 初期化済みのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_express.c:246 (モジュールの設定フック登録時に参照される)。
  • **呼び出し先**: apr_pcalloc (APR ライブラリ関数)。

関連データ構造

  • express_server_conf: 本関数で初期化される設定構造体。
  • server_rec: Apache のサーバー設定レコード。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • apr_pcalloc はメモリ不足時にプロセスを終了させる可能性があるため、本関数内で明示的なエラーハンドリングは行われていない。

保守担当者向けメモ

  • 本関数は mod_proxy_express の設定初期化フェーズでのみ使用される。
  • dbmtype のデフォルト値 "default" は、APR DBM の実装に依存する。設定変更が必要な場合は set_dbmtype ハンドラが利用される。
