include_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_include.h:69

目的

Server Side Include (SSI) 処理における現在の状態（フラグ、メモリプール、リクエスト情報、解析コンテキスト）を保持し、フィルタチェーン全体で共有するための主要な構造体です。

シグネチャまたは形状

typedef struct {
    apr_pool_t  *pool;
    apr_pool_t  *dpool;
    int          flags;
    int          if_nesting_level;
    int          flush_now;
    unsigned     argc;
    const char  *error_str;
    const char  *time_str;
    request_rec  *r;
    struct ssi_internal_ctx *intern;
} include_ctx_t;

入力と出力

  • **入力**: request_rec を通じて現在の HTTP リクエスト情報を受け取ります。
  • **出力**: 処理結果は apr_bucket_brigade を介してフィルタチェーンへ渡されます。また、error_str や time_str などの設定値が保持されます。

前提条件と不変条件

  • pool は永続的なメモリ管理に使用し、dpool は各ディレクティブ実行後にクリアされる一時的なメモリ管理に使用します。
  • intern メンバは非公開データであり、直接操作してはなりません。

アルゴリズムまたは振る舞い

SSI フィルタのライフサイクルを通じて以下の役割を果たします。 1. **状態管理**: flags（SSI_FLAG_XXXXX）により、条件付き出力のネスト状態や実行可否を制御します。 2. **動的展開**: ap_ssi_parse_string 等の関数に渡され、変数展開やタグ解析のコンテキストとして機能します。 3. **ストリーム制御**: flush_now が真の場合、現在のバッファをフィルタチェーンの下流へ即座にフラッシュし、フラグをリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_include.c 内の各ハンドラ（handle_comment 等）やパーサー関数（parse_expr, get_ptoken 等）から参照されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • request_rec: Apache のリクエスト情報構造体。
  • ssi_internal_ctx: intern メンバが指す非公開の内部データ構造。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • parse_expr 等の解析関数でエラーが発生した場合、SSI_CREATE_ERROR_BUCKET マクロを通じてエラーメッセージが生成されます。
  • if_nesting_level は条件分岐のネスト深さを追跡し、境界条件を管理します。

保守担当者向けメモ

  • intern メンバは非公開のため、拡張が必要な場合は mod_include.c 内の内部 API を経由してください。
  • SSI_FLAG_ 定数は条件付き出力の制御に不可欠です。変更時は mod_include.c のステートマシンへの影響を考慮してください。
