lua_db_escape

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:193

目的

Lua スクリプトから渡された文字列を、データベース接続のドライバ設定に基づいて SQL インジェクション対策用にエスケープします。

シグネチャまたは形状

int lua_db_escape(lua_State *L);

入力と出力

  • **入力**:
  • L: Lua 状態。
  • スタック位置 2: request_rec オブジェクト（ap_lua_check_request_rec で検証）。
  • スタック位置 3: エスケープ対象の文字列（LUA_TSTRING である必要がある）。
  • **出力**:
  • 成功時: エスケープされた文字列を Lua スタックにプッシュし、1 を返す。
  • 失敗時（接続が無効またはリクエストが無効）: nil をプッシュし、1 を返す。
  • 引数不正時: 0 を返す。

前提条件と不変条件

  • lua_get_db_handle(L) を通じて取得可能な有効なデータベースハンドル (db->alive) が存在すること。
  • apr_dbd_init がリクエストプールに対して呼び出され、初期化されていること。

アルゴリズムまたは振る舞い

1. ap_lua_check_request_rec を呼び出し、リクエストコンテキストを検証する。 2. スタックから文字列を取得し、lua_get_db_handle でデータベースハンドルを取得する。 3. データベースハンドルが生存している場合、apr_dbd_escape を呼び出し、現在のドライバとリクエストプールを使用して文字列をエスケープする。 4. エスケープ結果が取得できた場合、その文字列を Lua スタックにプッシュする。 5. 接続が無効な場合は nil をプッシュする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_dbd.c:686 (Lua VM への関数登録時)
  • **呼び出し先**:
  • ap_lua_check_request_rec
  • lua_get_db_handle
  • apr_dbd_init
  • apr_dbd_escape
  • luaL_checktype
  • lua_tostring
  • lua_pushstring
  • lua_pushnil

関連データ構造

  • lua_db_handle: データベース接続状態およびドライバ情報を保持する構造体。
  • request_rec: Apache のリクエストコンテキスト。

失敗モードと境界条件

  • request_rec が取得できない場合、関数は 0 を返し、Lua 側へ値を返さない。
  • db ハンドルが NULL または db->alive が偽の場合、nil を返す。
  • apr_dbd_escape が失敗して NULL を返した場合、スタックには何もプッシュされず、関数は 0 を返す（実装上の挙動）。

保守担当者向けメモ

  • 本関数は apr_dbd_escape を直接ラップしているため、エスケープの具体的な仕様は使用しているデータベースドライバに依存します。
  • r->pool を使用してメモリを管理しているため、リクエスト終了時にエスケープされた文字列も自動的に解放されます。
