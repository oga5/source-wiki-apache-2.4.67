mva_translate

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_vhost_alias.c:382

目的

リクエストの translate_name フェーズにおいて、ホスト名または IP アドレスに基づき、動的バーチャルホストのテンプレートに従って物理ファイルパスを生成し、r->filename に適用する。

シグネチャまたは形状

static int mva_translate(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）。サーバー設定から mva_sconf_t を取得して使用する。
  • **出力**: int 型のステータスコード。成功時は OK、対象外または設定不備の場合は DECLINED を返す。

前提条件と不変条件

  • r->server->module_config から vhost_alias_module に関連付けられた mva_sconf_t が取得可能であること。
  • リクエスト URI が / で始まるか、設定された cgi-bin パスに合致する必要がある。

アルゴリズムまたは振る舞い

1. **設定取得**: サーバー設定から mva_sconf_t を取得する。 2. **モード判定**:

  • URI が cgi-bin/ を含む場合、CGI モードとして処理する。
  • URI が / で始まる場合、ドキュメントルートモードとして処理する。
  • いずれにも該当しない場合は DECLINED を返す。

    • 3. **識別子取得**: 設定されたモード（VHOST_ALIAS_NAME または VHOST_ALIAS_IP）に従い、サーバー名またはローカル IP アドレスを取得する。 4. **パス生成**: r->canonical_filename を初期化し、vhost_alias_interpolate を呼び出してテンプレートに基づいたパスを構築する。 5. **CGI 設定**: CGI モードの場合、r->handler を cgi-script に設定し、関連するノートおよびコンテキスト情報を更新する。 6. **終了**: OK を返して処理を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_translate_name により登録され、Apache のリクエスト処理パイプラインから呼び出される。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_get_server_name
  • vhost_alias_interpolate
  • apr_table_setn
  • ap_set_context_info

関連データ構造

  • request_rec: リクエスト情報を保持するコア構造体。
  • mva_sconf_t: モジュール固有のサーバー設定（ルートパスやモードを保持）。
  • mva_mode_e: ホスト名ベースか IP ベースかを識別する列挙型。

失敗モードと境界条件

  • 設定されたモードが不明な場合、または URI が適切な形式でない場合は DECLINED を返す。
  • cgi-bin の判定において、パスの先頭以外に cgi-bin/ が出現した場合は CGI モードとして扱われない。

保守担当者向けメモ

  • r->filename のバッファ溢れについては、内部で vhost_alias_checkspace が考慮されている前提で動作する。
