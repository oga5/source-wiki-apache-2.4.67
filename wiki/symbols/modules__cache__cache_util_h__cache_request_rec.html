cache_request_rec

定義

  • 種別: struct
  • 定義箇所: modules/cache/cache_util.h:210

目的

HTTP リクエスト処理中におけるキャッシュの状態管理およびコンテキスト保持を行うための構造体です。

シグネチャまたは形状

typedef struct {
    cache_provider_list *providers;
    const cache_provider *provider;
    const char *provider_name;
    int fresh;
    cache_handle_t *handle;
    cache_handle_t *stale_handle;
    apr_table_t *stale_headers;
    int in_checked;
    int block_response;
    apr_bucket_brigade *saved_brigade;
    apr_off_t saved_size;
    apr_time_t exp;
    apr_time_t lastmod;
    cache_info *info;
    ap_filter_t *save_filter;
    ap_filter_t *remove_url_filter;
    const char *key;
    apr_off_t size;
    apr_bucket_brigade *out;
    cache_control_t control_in;
} cache_request_rec;

入力と出力

  • 本構造体はリクエスト処理の各段階で参照・更新されます。
  • 入力: リクエストごとの設定やヘッダー情報。
  • 出力: キャッシュの鮮度判定結果（fresh）、キャッシュキー（key）、および保存対象のレスポンスデータ（saved_brigade）。

前提条件と不変条件

  • 本構造体はリクエストプール（apr_pool_t）に紐付いており、リクエストの終了とともに自動的に解放されます。
  • キャッシュの物理的な保存ロジックは各ストレージプロバイダに依存します。

アルゴリズムまたは振る舞い

  • キャッシュの検索、鮮度判定、および保存処理のコンテキストを保持します。
  • CACHE_SAVE フィルタや CACHE_OUT フィルタにおいて、レスポンスのフィルタリングやキャッシュの更新制御に使用されます。
  • cache_try_lock 等の排他制御関数において、キャッシュ更新権の管理に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照・操作）:
  • modules/cache/cache_storage.c: cache_remove_url, cache_create_entity, cache_select, cache_invalidate
  • modules/cache/cache_util.c: cache_try_lock
  • 呼び出し先:
  • 構造体内部で cache_provider や cache_handle_t などの関連型を保持しており、これらを通じてストレージプロバイダの関数を呼び出します。

関連データ構造

  • cache_provider_list: 利用可能なキャッシュプロバイダのリスト。
  • cache_handle_t: キャッシュエントリへのアクセスハンドル。
  • cache_control_t: Cache-Control ヘッダーの解析結果。
  • apr_bucket_brigade: レスポンスデータのバッファリングに使用。

失敗モードと境界条件

  • キャッシュのロック取得失敗時や、無効なキャッシュエントリへのアクセス時に、適切なフラグ（fresh 等）やフィルタの制御を通じてエラーハンドリングが行われます。
  • size フィールドが -1 の場合、コンテンツ長が不明であることを示します。

保守担当者向けメモ

  • 本構造体は mod_cache のコアロジックと各ストレージプロバイダ間の橋渡しを行います。
  • メモリ管理はリクエストプールに依存するため、手動での解放は不要です。
  • 構造体の各フィールドは、キャッシュのライフサイクル（検索、判定、保存、無効化）の各フェーズで適切に更新される必要があります。
