md_acme_authz_update

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_authz.c:114

目的

ACME サーバーからドメイン認証（Authorization）リソースの最新状態を取得し、md_acme_authz_t 構造体を更新する。

シグネチャまたは形状

apr_status_t md_acme_authz_update(md_acme_authz_t *authz, md_acme_t *acme, apr_pool_t *p)

入力と出力

  • **入力**:
  • authz: 更新対象の認証オブジェクト（url フィールドが必須）。
  • acme: ACME サーバーとの通信セッション。
  • p: メモリ割り当て用プール。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。
  • authz 構造体の内部状態（state, domain, resource, error_type 等）が更新される。

前提条件と不変条件

  • acme, acme->http, authz, authz->url は NULL であってはならない（assert による検証）。
  • authz->resource は、サーバーから取得した JSON オブジェクトへのポインタとして保持される。

アルゴリズムまたは振る舞い

1. 認証状態を MD_ACME_AUTHZ_S_UNKNOWN にリセットし、エラー情報をクリアする。 2. md_acme_get_json を呼び出し、authz->url から最新の JSON リソースを取得する。 3. JSON 内の status フィールドを解析し、状態を更新する：

  • "pending": MD_ACME_AUTHZ_S_PENDING
  • "valid": MD_ACME_AUTHZ_S_VALID
  • "invalid": MD_ACME_AUTHZ_S_INVALID を設定し、copy_challenge_error を用いてエラー詳細を抽出する。

    • 4. 取得した JSON を authz->resource に格納する。 5. 処理結果をログに出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_authz.c:90
  • **呼び出し先**:
  • md_acme_get_json
  • md_json_gets
  • md_json_itera (エラー解析時)
  • copy_challenge_error (エラー解析時)
  • md_log_perror

関連データ構造

  • md_acme_authz_t: 認証状態を保持するメイン構造体。
  • md_acme_t: 通信セッション。
  • error_ctx_t: エラー解析時に使用されるコンテキスト構造体。

失敗モードと境界条件

  • サーバーからの応答がパースできない場合、または status フィールドが含まれない場合は APR_EINVAL を返す。
  • 認証状態が不明なまま処理が終了した場合、エラーとして扱われる。
  • サーバー側でエラーが発生している場合、copy_challenge_error を介して詳細なサブ問題が収集される。

保守担当者向けメモ

  • authz->resource に格納される JSON オブジェクトのライフサイクルは、渡された apr_pool_t に依存する。
  • 状態遷移のロジックは ACME 仕様の status 値に直接依存しているため、プロトコル拡張時にはこの条件分岐の確認が必要。
