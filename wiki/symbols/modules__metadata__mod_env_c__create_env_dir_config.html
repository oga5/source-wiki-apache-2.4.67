create_env_dir_config

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_env.c:37

目的

ディレクトリ単位の環境変数設定を保持するための env_dir_config_rec 構造体を生成し、初期化します。

シグネチャまたは形状

static void *create_env_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • p: 設定構造体のメモリ割り当てに使用する apr_pool_t。
  • dummy: 使用されない引数（Apache の設定コールバックシグネチャに適合させるためのもの）。
  • **出力**:
  • 初期化された env_dir_config_rec 構造体へのポインタを void * として返します。

前提条件と不変条件

  • p は有効なメモリプールである必要があります。
  • 生成される構造体内の vars および unsetenv テーブルは、それぞれ10個の要素を保持可能な初期サイズで作成されます。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、メモリプール p から env_dir_config_rec 構造体分のメモリを確保します。 2. apr_table_make を呼び出し、環境変数を保持する vars テーブルを初期化します。 3. apr_table_make を呼び出し、削除対象の環境変数を保持する unsetenv テーブルを初期化します。 4. 確保した構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_env.c:183 において、ディレクトリ設定作成用ハンドラとして登録されています。
  • **呼び出し先**: apr_palloc および apr_table_make（いずれも APR API）。

関連データ構造

  • env_dir_config_rec: 本関数が初期化するディレクトリ設定構造体。vars と unsetenv の2つの apr_table_t を保持します。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性があります（Apache の標準的なメモリ管理動作に依存）。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出されます。
  • dummy 引数は将来的な拡張や API 互換性のために存在しますが、現在の実装では無視されます。
