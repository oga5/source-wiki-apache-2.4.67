create_win32_dir_config

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_win32.c:54

目的

Apacheのディレクトリ単位の設定構造体である win32_dir_conf を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_win32_dir_config(apr_pool_t *p, char *dir)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t プール。
  • dir: 設定が適用されるディレクトリパス（本関数内では未使用）。
  • **出力**:
  • 初期化済みの win32_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • apr_palloc により、指定されたプール p から win32_dir_conf サイズ分のメモリが確保される。
  • 初期化後、script_interpreter_source フィールドは INTERPRETER_SOURCE_UNSET に設定される。

アルゴリズムまたは振る舞い

1. apr_palloc を呼び出し、win32_dir_conf 構造体分のメモリを確保する。 2. 確保した構造体の script_interpreter_source メンバに INTERPRETER_SOURCE_UNSET を代入する。 3. 初期化した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_win32.c:555 （モジュール設定の生成プロセスにおいて呼び出される）。
  • **呼び出し先**: apr_palloc（APRライブラリのメモリ割り当て関数）。

関連データ構造

  • win32_dir_conf: 本関数が初期化対象とするディレクトリ設定構造体。
  • interpreter_source_e: script_interpreter_source が保持する列挙型。

失敗モードと境界条件

  • apr_palloc がメモリ不足により失敗した場合、APRの規約に従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は win32_dir_conf の初期化のみを担当する。設定値の変更やマージについては、それぞれ set_interpreter_source や merge_win32_dir_configs が担う。
