dav_get_liveprop_element

定義

  • 種別: function
  • 定義箇所: modules/dav/main/props.c:953

目的

指定されたリソースに関連付けられたライブプロパティ要素（dav_liveprop_elem）を、リソースのメモリプールから取得します。

シグネチャまたは形状

dav_liveprop_elem *dav_get_liveprop_element(const dav_resource *resource)

入力と出力

  • **入力**: resource (const dav_resource *) - ライブプロパティ要素を検索する対象のリソース。
  • **出力**: dav_liveprop_elem * - リソースのプールに格納されているライブプロパティ要素へのポインタ。見つからない場合は NULL が返されます。

前提条件と不変条件

  • resource および resource->pool が有効である必要があります。
  • DAV_PROP_ELEMENT キーを使用して、事前に apr_pool_userdata_set 等でデータがプールに格納されていることを前提としています。

アルゴリズムまたは振る舞い

1. resource->pool に対して apr_pool_userdata_get を呼び出します。 2. キーとして DAV_PROP_ELEMENT を使用し、プール内に保存されているユーザーデータを取得します。 3. 取得したポインタを呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/props.c:741 にて、propdb->resource を引数として呼び出されています。
  • **呼び出し先**: apr_pool_userdata_get (APR ライブラリ関数) を呼び出します。

関連データ構造

  • dav_resource: リソースの抽象表現。
  • dav_liveprop_elem: ライブプロパティの要素情報を保持する構造体。
  • apr_pool_t: リソースに関連付けられたメモリプール。

失敗モードと境界条件

  • 指定されたキー（DAV_PROP_ELEMENT）に対応するデータがプール内に存在しない場合、element は NULL となり、そのまま NULL が返されます。

保守担当者向けメモ

  • 本関数は apr_pool_userdata を介した間接的なデータ取得を行っています。このデータがいつ、どこで設定されるかについては、props.c 内の他の関数による設定ロジックを確認してください。
