add_redirect

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_alias.c:375

目的

Apache HTTP Server の設定ディレクティブ Redirect を解析し、内部的なリダイレクト設定を登録するためのラッパー関数です。

シグネチャまたは形状

static const char *add_redirect(cmd_parms *cmd, void *dirconf,
                                const char *arg1, const char *arg2,
                                const char *arg3);

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dirconf: ディレクトリごとの設定構造体。
  • arg1, arg2, arg3: Redirect ディレクティブに指定された引数（ステータスコード、URLパス、リダイレクト先URLなど）。
  • **出力**:
  • 成功時は NULL を返します。
  • エラー発生時はエラーメッセージを示す文字列を返します。

前提条件と不変条件

  • Redirect ディレクティブの解析時に呼び出されることを前提としています。
  • 内部的に add_redirect_internal を呼び出す際、ステータスコードのデフォルト値として 0 を渡します。

アルゴリズムまたは振る舞い

1. 受け取った引数 arg1〜arg3 をそのまま add_redirect_internal 関数へ転送します。 2. add_redirect_internal の第6引数に 0 を指定し、リダイレクトの内部処理を実行させます。 3. add_redirect_internal が返した結果（成功時の NULL またはエラー文字列）をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_alias.c:719 における AP_INIT_TAKE123("Redirect", ...) マクロ定義から、設定読み込み時に呼び出されます。
  • **呼び出し先**: add_redirect_internal

関連データ構造

  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。
  • alias_dir_conf: ディレクトリごとの設定を保持する構造体（dirconf として渡される）。

失敗モードと境界条件

  • add_redirect_internal の実装に依存します。引数の数が不適切な場合や、設定コンテキストが不正な場合にエラー文字列が返されます。

保守担当者向けメモ

  • 本関数は Redirect ディレクティブのインターフェースを add_redirect_internal に適合させるための単純なプロキシです。
  • ロジックの変更が必要な場合は、add_redirect_internal を修正してください。
