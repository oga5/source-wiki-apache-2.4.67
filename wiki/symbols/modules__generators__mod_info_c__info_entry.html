info_entry

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_info.c:69

目的

モジュール名とそれに関連する追加情報をペアとして保持するためのデータ構造です。mod_info モジュールにおいて、各モジュールに対する補足情報を管理するために使用されます。

シグネチャまたは形状

typedef struct {
    const char *name;           /* matching module name */
    const char *info;           /* additional info */
} info_entry;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • 入力: name（モジュール名）、info（追加情報）の各文字列ポインタ。
  • 出力: なし。

前提条件と不変条件

  • name および info は、apr_pool_t 上で確保された有効な文字列である必要があります。
  • info_svr_conf 構造体内の more_info 配列（apr_array_header_t）の要素として格納されることを前提としています。

アルゴリズムまたは振る舞い

本構造体自体はデータの格納のみを行います。mod_info モジュールは、この構造体を配列として管理し、AddModuleInfo ディレクティブによって設定された情報を保持します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • modules/generators/mod_info.c:92: conf->more_info の初期化時に使用。
  • modules/generators/mod_info.c:415: more_info 配列の要素を走査する際にキャストして使用。
  • modules/generators/mod_info.c:979: apr_array_push を用いて新しいエントリを追加する際に使用。
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • info_svr_conf: 本構造体を配列として保持するモジュール固有の設定構造体。
  • apr_array_header_t: 本構造体を格納する動的配列の型。

失敗モードと境界条件

  • メモリ不足: apr_array_make や apr_array_push が失敗した場合、APR のメモリ管理ポリシーに従いサーバーが異常終了する可能性があります。
  • 文字列の有効性: name や info に NULL を設定した場合、後続の表示処理でセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_info の設定フェーズで構築され、リクエスト処理時に読み取られます。
  • more_info 配列へのアクセスは、conf->more_info->elts を info_entry * にキャストして行われるため、型安全性を維持するためにこの構造体の定義を変更する際は注意が必要です。
