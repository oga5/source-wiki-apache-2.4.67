set_thread_limit

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:119

目的

Apache HTTP Serverの設定ディレクティブ ThreadLimit の値を解析し、内部変数 thread_limit に設定する。

シグネチャまたは形状

static const char *set_thread_limit (cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていない引数。
  • arg: 設定値として渡された文字列（数値）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • 本関数はグローバルな設定コンテキストでのみ呼び出される必要がある。
  • arg は atoi で変換可能な数値文字列であること。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、現在の設定コンテキストが GLOBAL_ONLY であるか検証する。 2. コンテキストが不適切であれば、エラーメッセージを即座に返す。 3. atoi(arg) を使用して文字列を整数に変換し、静的変数 thread_limit に代入する。 4. 正常終了として NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/mpm_winnt.c:134 (AP_INIT_TAKE1による登録)
  • server/mpm/event/event.c:4146 (他MPMからの参照)
  • server/mpm/netware/mpm_netware.c:1351 (他MPMからの参照)
  • server/mpm/worker/worker.c:2456 (他MPMからの参照)
  • **呼び出し先**:
  • ap_check_cmd_context (Apacheコア関数)
  • atoi (標準Cライブラリ)

関連データ構造

  • cmd_parms: 設定コマンドの引数を保持する構造体。
  • thread_limit: 本関数によって更新される静的変数（定義箇所はソース内に存在）。

失敗モードと境界条件

  • ap_check_cmd_context がグローバルコンテキスト以外を検出した場合、エラーメッセージを返して処理を中断する。

保守担当者向けメモ

  • 本関数は複数のMPM（winnt, event, worker, netware）から共通して参照されている。変更を加える際は、各MPMの動作に影響がないか確認すること。
  • thread_limit 変数の型や範囲チェックが必要な場合は、atoi の結果に対してバリデーションを追加すること。
