md_json_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme.h:73

目的

ACME プロトコルおよび Managed Domain (MD) サブシステムにおいて、JSON 形式のデータを保持・操作するための抽象データ構造です。

シグネチャまたは形状

ソースコード上では不完全型（opaque struct）として宣言されており、具体的なメンバ構成は md_json.h 等の基盤層で定義されています。

入力と出力

本構造体は、ACME リクエストのペイロードや、サーバーからのレスポンスボディとして受け渡されます。

  • **入力**: md_acme_req_json_cb コールバック関数の引数 jbody として渡されます。
  • **出力**: md_to_json や md_to_public_json などの関数により、ドメイン情報から生成されます。

前提条件と不変条件

  • apr_pool_t の生存期間に依存してメモリが管理されます。
  • ACME プロトコルにおける JSON メッセージのシリアライズおよびデシリアライズの基盤として使用されます。

アルゴリズムまたは振る舞い

ACME サーバーとの通信において、リクエストの初期化（acmev2_req_init）や、レスポンスの解析（md_acme_get_json）の過程で、JSON ペイロードのコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/md/md.h: md_to_json, md_from_json, md_to_public_json 等で使用。
  • modules/md/md_acme.c: acmev2_req_init, md_acme_req_body_init, md_acme_get_json 等で使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、具体的な操作は md_json.h で定義された関数群に依存します。

関連データ構造

  • md_acme_req_t: ACME リクエストコンテキスト。
  • md_t: 管理対象ドメインのメイン構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • JSON の解析失敗や、メモリ不足による割り当て失敗が発生する可能性があります。
  • 構造体が NULL の場合、または不正な JSON 形式である場合の挙動は、操作を行う関数（md_json 関連）の仕様に依存します。

保守担当者向けメモ

  • 本構造体は modules/md/md_json.h で定義された機能と密接に関連しています。詳細な操作が必要な場合は、そちらのヘッダーを参照してください。
  • メモリリークを防ぐため、関連付けられた apr_pool_t のライフサイクルを厳守してください。
