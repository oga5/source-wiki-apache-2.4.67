cmd_how

定義

  • 種別: enum
  • 定義箇所: include/http_config.h:49

目的

Apache HTTP Server の設定ディレクティブにおいて、引数の数や解析方法を定義するために使用されます。

シグネチャまたは形状

enum cmd_how {
    RAW_ARGS, TAKE1, TAKE2, ITERATE, ITERATE2, FLAG, 
    NO_ARGS, TAKE12, TAKE3, TAKE23, TAKE123, TAKE13, TAKE_ARGV
};

入力と出力

  • 入力: 設定ファイル内のディレクティブ引数。
  • 出力: 指定された列挙値に基づき、ハンドラー関数（cmd_func）が引数をどのように解釈・処理すべきかを決定します。

前提条件と不変条件

  • 各列挙値は、command_rec 構造体の args_how フィールドに設定されます。
  • FLAG は 'On' または 'Off' の値のみを許容します。
  • RAW_ARGS を指定した場合、ハンドラー関数自身がコマンドラインの解析を行う必要があります。

アルゴリズムまたは振る舞い

設定ファイルの解析時、ap_walk_config がディレクティブを走査する際、この列挙値に基づいて引数の個数や形式が検証されます。例えば TAKE1 であれば引数が1つであることを確認し、ITERATE であれば引数が複数回出現することを許容します。

呼び出し元と呼び出し先

  • 呼び出し元: include/http_config.h:214 にて command_rec 構造体のメンバー args_how として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • command_rec: ディレクティブの定義を保持する構造体。
  • cmd_parms: ディレクティブ解析時にハンドラーへ渡されるコンテキスト情報。

失敗モードと境界条件

  • 引数の数が指定された cmd_how の定義と一致しない場合、設定解析時にエラーが発生します。
  • TAKE_ARGV を使用する場合、argc と argv の適切な処理がハンドラー側に委ねられます。

保守担当者向けメモ

  • 新しい引数形式が必要な場合は、この列挙型を拡張し、server/config.c 等の解析ロジック側で対応する処理を追加する必要があります。
