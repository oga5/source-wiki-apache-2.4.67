dav_hooks_binding

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:2553 (前方宣言: modules/dav/main/mod_dav.h:272)

目的

WebDAV のバインディング機能（リソース間のリンクや名前付けに関連する操作）を抽象化し、プロバイダーが実装すべき関数ポインタを保持するためのインターフェース構造体です。

シグネチャまたは形状

struct dav_hooks_binding {
    /* 関数ポインタ群（実装により定義） */
};

※構造体の詳細なメンバ構成は mod_dav.h の定義箇所を参照してください。

入力と出力

  • 本構造体は、dav_get_binding_hooks(request_rec *r) を通じて取得されます。
  • プロバイダーの実装において、各フック関数は WebDAV リクエスト処理に関連する引数を受け取り、dav_error * 型のエラー情報を返します。

前提条件と不変条件

  • mod_dav のプロバイダーとして登録される必要があります。
  • dav_get_binding_hooks を介して取得されるポインタは、リクエストの生存期間中有効であることが期待されます。

アルゴリズムまたは振る舞い

  • mod_dav のメインハンドラは、バインディング関連の操作が必要な際、DAV_GET_HOOKS_BINDING(r) マクロを使用してこの構造体を取得し、適切なフック関数を呼び出します。
  • 具体的なバインディング操作（リソースのリンク作成や解除など）は、この構造体に登録されたプロバイダー固有の実装に委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.c: dav_get_binding_hooks 関数内および DAV_GET_HOOKS_BINDING マクロを使用する各箇所。
  • **呼び出し先**:
  • 本構造体内の関数ポインタは、プロバイダー（ストレージ実装など）によって提供される関数を指します。具体的な関数名はソースコードの定義箇所に依存します。

関連データ構造

  • request_rec: Apache のリクエスト構造体。フック取得の起点となります。
  • dav_error: フック関数が返すエラー状態を保持する構造体。

失敗モードと境界条件

  • プロバイダーがバインディング機能をサポートしていない場合、dav_get_binding_hooks は NULL を返すか、フックポインタが NULL に設定されている可能性があります。呼び出し側はこれらを適切にチェックする必要があります。

保守担当者向けメモ

  • 本構造体は mod_dav の拡張ポイントの一つです。新しいバインディング関連の操作を追加する場合は、この構造体にメンバを追加し、各プロバイダー実装を更新する必要があります。
  • modules/dav/main/mod_dav.c 内の DAV_GET_HOOKS_BINDING マクロの使用箇所が、本構造体の利用パターンの参照となります。
