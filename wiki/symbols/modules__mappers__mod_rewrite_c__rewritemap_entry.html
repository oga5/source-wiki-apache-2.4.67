rewritemap_entry

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:272

目的

RewriteMap ディレクティブによって定義される外部マップ（テキストファイル、DBM、DBD、外部プログラム等）の設定情報を保持する構造体です。

シグネチャまたは形状

typedef struct {
    const char *datafile;
    const char *dbmtype;
    const char *checkfile;
    const char *cachename;
    int   type;
    apr_file_t *fpin;
    apr_file_t *fpout;
    apr_file_t *fperr;
    char *(*func)(request_rec *, char *);
    char **argv;
    const char *dbdq;
    const char *checkfile2;
    const char *user;
    const char *group;
} rewritemap_entry;

入力と出力

  • **入力**: RewriteMap ディレクティブの設定値（ファイルパス、マップタイプ、実行ユーザー/グループ、SQLクエリ等）。
  • **出力**: 外部マップのルックアップに必要なファイルハンドル、関数ポインタ、実行引数、およびキャッシュ名。

前提条件と不変条件

  • fpin, fpout, fperr は外部プログラムマップ（PRG）の場合に使用されます。
  • func は内部マップ（Internal Map）の場合に設定されます。
  • dbdq は DBD マップの場合に SQL SELECT 文を保持します。

アルゴリズムまたは振る舞い

本構造体は、mod_rewrite の設定フェーズで生成され、リクエスト処理時にマップのルックアップを行うための参照情報として利用されます。外部プログラムマップの場合、パイプ経由でのプロセス間通信を管理し、内部マップの場合は登録された関数ポインタを介して直接処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_rewrite.c 内の RewriteMap 設定解析ロジック（例: apr_pcalloc を用いたメモリ確保箇所）。
  • **呼び出し先**: 特定の呼び出し先情報はソース抜粋からは不明です。

関連データ構造

  • rewrite_server_conf: サーバー全体のマップ定義を保持する構造体。
  • request_rec: 内部マップ関数 (func) の引数として渡されるリクエスト情報。

失敗モードと境界条件

  • checkfile および checkfile2 はマップファイルの存在確認に使用されます。これらが適切に設定されていない場合、マップの初期化やルックアップが失敗する可能性があります。
  • 外部プログラムマップにおいて、user や group の指定が不適切な場合、プロセス実行権限によるエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_rewrite のマップ機能の中核です。マップタイプ（type）に応じたフィールドの有効・無効の組み合わせが複雑であるため、変更時は各マップタイプ（txt, dbm, prg, dbd, int）の初期化ロジックを確認してください。
  • メモリ管理は cmd->pool を通じて行われており、設定のライフサイクルに依存します。
