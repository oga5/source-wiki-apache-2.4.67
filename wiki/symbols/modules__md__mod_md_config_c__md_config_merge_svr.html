md_config_merge_svr

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_config.c:293

目的

Apache HTTP Server の設定マージフェーズにおいて、サーバー単位の設定（md_srv_conf_t）をマージするためのエントリーポイントを提供します。

シグネチャまたは形状

void *md_config_merge_svr(apr_pool_t *pool, void *basev, void *addv)

入力と出力

  • pool: 設定マージに使用するメモリプール。
  • basev: 親コンテキスト（ベースサーバー）の設定構造体へのポインタ。
  • addv: 子コンテキスト（バーチャルホスト）の設定構造体へのポインタ。
  • 戻り値: マージされた新しい設定構造体へのポインタを返します。

前提条件と不変条件

  • basev および addv は md_srv_conf_t 型として扱われます。
  • Apache の設定フレームワークから呼び出されることを前提としています。

アルゴリズムまたは振る舞い

内部的に md_config_merge 関数を呼び出し、ベースサーバーの設定とバーチャルホストの設定を統合します。この処理により、階層的な設定継承が解決されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md.c:65（サーバー設定のマージ用関数として登録）。
  • 呼び出し先: md_config_merge（modules/md/mod_md_config.c 内）。

関連データ構造

  • md_srv_conf_t: バーチャルホスト単位の設定を保持する構造体。
  • apr_pool_t: Apache Portable Runtime のメモリプール。

失敗モードと境界条件

  • 呼び出し元および呼び出し先の情報に基づくと、メモリ不足時に apr_pool_t 関連の例外が発生する可能性がありますが、本関数自体は md_config_merge の結果をそのまま返す単純なラッパーです。

保守担当者向けメモ

  • 本関数は mod_md の設定マージ処理のインターフェースです。ロジックの変更が必要な場合は、実体である md_config_merge を参照してください。
