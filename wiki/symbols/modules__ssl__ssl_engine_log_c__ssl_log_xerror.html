ssl_log_xerror

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_log.c:211

目的

X.509 証明書に関連するエラー情報を、サーバーコンテキスト (server_rec) を用いて Apache のログシステムへ出力するためのラッパー関数です。

シグネチャまたは形状

void ssl_log_xerror(const char *file, int line, int level, apr_status_t rv,
                    apr_pool_t *ptemp, server_rec *s, X509 *cert,
                    const char *fmt, ...)

入力と出力

  • **入力**:
  • file, line: ログ出力元のソースファイル名と行番号。
  • level: ログレベル（APLOG_ 定数）。
  • rv: APR ステータスコード。
  • ptemp: 証明書情報の文字列化に使用する一時メモリプール。
  • s: ログ出力対象のサーバーレコード。
  • cert: ログに詳細を含める対象の X509 証明書構造体。
  • fmt, ...: ログメッセージのフォーマット文字列と可変長引数。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • s が有効な server_rec であること。
  • ptemp は証明書情報の文字列化処理に必要な生存期間を持つプールであること。
  • ログ出力は APLOG_IS_LEVEL(s, level) が真の場合にのみ実行されます。

アルゴリズムまたは振る舞い

1. APLOG_IS_LEVEL マクロにより、指定されたログレベルが現在のサーバー設定で有効か確認します。 2. 有効な場合、可変長引数リスト (va_list) を初期化します。 3. ssl_log_cert_error を呼び出し、証明書情報を文字列化してログメッセージに結合し、Apache のログシステムへ渡します。 4. va_end で引数リストを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_init.c
  • modules/ssl/ssl_util_ssl.c
  • modules/ssl/ssl_util_stapling.c
  • **呼び出し先**:
  • ssl_log_cert_error (内部関数)

関連データ構造

  • server_rec: Apache のサーバー設定構造体。
  • X509: OpenSSL の証明書構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • cert が NULL の場合、ssl_log_cert_error の仕様に従い、証明書情報なしのログ出力が行われます。
  • ログレベルが設定より低い場合、関数は何も出力せずに即座に復帰します。

保守担当者向けメモ

  • 本関数は ssl_log_cxerror や ssl_log_rxerror と同様のラッパー群の一部です。
  • 証明書情報の文字列化には BIO が内部で使用されるため、ptemp のメモリ消費量に注意してください。
