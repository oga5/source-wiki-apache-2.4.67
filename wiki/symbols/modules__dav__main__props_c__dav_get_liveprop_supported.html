dav_get_liveprop_supported

定義

  • 種別: function
  • 定義箇所: modules/dav/main/props.c:929

目的

指定されたライブプロパティがサポートされているかどうかを判定し、サポートされている場合にそのプロパティの情報を XML 形式で出力バッファ（body）に挿入します。

シグネチャまたは形状

void dav_get_liveprop_supported(dav_propdb *propdb,
                                const char *ns_uri,
                                const char *propname,
                                apr_text_header *body)

入力と出力

  • propdb: プロパティデータベースのコンテキスト。
  • ns_uri: プロパティの名前空間 URI。
  • propname: プロパティ名。
  • body: プロパティ情報を格納するための apr_text_header 構造体（出力）。

前提条件と不変条件

  • propdb は有効なポインタである必要があります。
  • dav_find_liveprop_provider を介してプロパティの存在確認が行われます。

アルゴリズムまたは振る舞い

1. dav_find_liveprop_provider を呼び出し、指定された名前空間とプロパティ名に対応するプロバイダー（hooks）とプロパティ ID（propid）を特定します。 2. プロパティが特定された場合（propid != DAV_PROPID_CORE_UNKNOWN）：

  • hooks が NULL の場合：コアプロパティとみなし、dav_insert_coreprop を使用して DAV_PROP_INSERT_SUPPORTED モードで情報を挿入します。
  • hooks が存在する場合：そのプロバイダーの insert_prop フックを呼び出し、DAV_PROP_INSERT_SUPPORTED モードで情報を挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c:1528
  • **呼び出し先**:
  • dav_find_liveprop_provider
  • dav_insert_coreprop
  • hooks->insert_prop（ライブプロパティプロバイダーのフック）

関連データ構造

  • dav_propdb: プロパティ操作のコンテキスト。
  • dav_hooks_liveprop: ライブプロパティの操作フック。
  • apr_text_header: XML 出力バッファ。

失敗モードと境界条件

  • 指定されたプロパティがライブプロパティとして見つからない場合（DAV_PROPID_CORE_UNKNOWN）、何も行われません。
  • hooks が NULL の場合、コアプロパティとして処理されます。

保守担当者向けメモ

  • 本関数は PROPFIND リクエスト等でサポートされているプロパティを列挙する際に使用されます。
  • hooks->insert_prop の実装は各ライブプロパティプロバイダーに依存するため、追加のプロパティをサポートする場合は liveprop.c や std_liveprop.c の実装を確認してください。
