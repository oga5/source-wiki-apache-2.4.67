h2_c2_destroy

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2.c:145

目的

セカンダリ接続（c2）に関連付けられたリソースを解放し、メモリプールを破棄する。

シグネチャまたは形状

void h2_c2_destroy(conn_rec *c2)

入力と出力

  • **入力**: c2 - 破棄対象のセカンダリ接続を表す conn_rec 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • c2 は有効な conn_rec 構造体であること。
  • c2->pool は、この接続専用のメモリプールとして初期化されていること。

アルゴリズムまたは振る舞い

1. h2_conn_ctx_get(c2) を呼び出し、接続に関連付けられたコンテキストを取得する。 2. 接続が中断（aborted）されておらず、かつコンテキストが存在し、送信済みバイト数（bytes_sent）が記録されている場合、h2_c2_log_io を呼び出して I/O 統計をログ出力する。 3. apr_pool_destroy(c2->pool) を呼び出し、接続に関連付けられたメモリプールを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_mplx.c:634
  • modules/http2/h2_mplx.c:954
  • **呼び出し先**:
  • h2_conn_ctx_get
  • ap_log_cerror
  • h2_c2_log_io
  • apr_pool_destroy

関連データ構造

  • conn_rec: Apache HTTP Server の接続構造体。
  • h2_conn_ctx_t: 接続ごとの HTTP/2 コンテキスト。

失敗モードと境界条件

  • c2 が NULL の場合の動作は定義されていない（ソースコード内で c2->log_id 等へのアクセスがあるため、NULLポインタ参照の可能性がある）。
  • conn_ctx が取得できない場合、I/O ログ出力はスキップされる。

保守担当者向けメモ

  • 本関数は h2_c2_create で確保されたリソースのクリーンアップを担う。メモリリークを防ぐため、接続のライフサイクル終了時に必ず呼び出される必要がある。
