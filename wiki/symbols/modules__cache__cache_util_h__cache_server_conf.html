cache_server_conf

定義

  • 種別: struct
  • 定義箇所: modules/cache/cache_util.h:120

目的

Apache HTTP Server のキャッシュサブシステム（mod_cache）において、サーバー全体に適用されるキャッシュ設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *cacheenable;
    apr_array_header_t *cachedisable;
    apr_array_header_t *ignore_headers;
    apr_array_header_t *ignore_session_id;
    const char *lockpath;
    apr_time_t lockmaxage;
    apr_uri_t *base_uri;
    unsigned int ignorecachecontrol:1;
    unsigned int ignorequerystring:1;
    unsigned int quick:1;
    unsigned int lock:1;
    unsigned int x_cache:1;
    unsigned int x_cache_detail:1;
    unsigned int ignore_headers_set:1;
    unsigned int ignore_session_id_set:1;
    unsigned int base_uri_set:1;
    unsigned int ignorecachecontrol_set:1;
    unsigned int ignorequerystring_set:1;
    unsigned int quick_set:1;
    unsigned int lock_set:1;
    unsigned int lockpath_set:1;
    unsigned int lockmaxage_set:1;
    unsigned int x_cache_set:1;
    unsigned int x_cache_detail_set:1;
} cache_server_conf;

入力と出力

  • **入力**: ap_get_module_config(r->server->module_config, ...) を通じてサーバー設定から取得されます。
  • **出力**: キャッシュの有効化/無効化リスト、無視するヘッダー、ロックパス、および各種フラグ設定を保持し、キャッシュ処理の判断基準として利用されます。

前提条件と不変条件

  • 各設定項目に対応する _set フラグ（例: lock_set）は、設定が明示的に行われたかを追跡するために使用されます。
  • 構造体内の各ポインタ（apr_array_header_t 等）は、リクエストプールまたはサーバープール上で適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • キャッシュの対象/除外URLの管理、キャッシュキー計算から除外するセッションIDの管理を行います。
  • 「サンダリング・ハード」問題を防ぐためのロック機構（lock）や、キャッシュ制御ヘッダーの無視設定（ignorecachecontrol）を制御します。
  • x_cache および x_cache_detail フラグにより、レスポンスヘッダーへのキャッシュ状態情報の付与を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/cache_storage.c および modules/cache/cache_util.c 内の関数（cache_try_lock, cache_remove_lock 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • cache_dir_conf: ディレクトリ単位のキャッシュ設定。
  • cache_request_rec: リクエストごとのキャッシュコンテキスト。
  • apr_array_header_t: 設定リストの保持に使用。
  • apr_uri_t: ベースURIの保持に使用。

失敗モードと境界条件

  • 設定が未定義の場合、_set フラグが 0 となり、デフォルトの振る舞いが適用されます。
  • lockpath が正しく設定されていない場合、ロック機構が正常に機能しない可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cache の設定フレームワークの一部であり、httpd.h の設定管理機能と密接に統合されています。
  • 構造体内のビットフィールドはメモリ効率を考慮していますが、新しい設定項目を追加する際は _set フラグの追加と整合性に注意してください。
