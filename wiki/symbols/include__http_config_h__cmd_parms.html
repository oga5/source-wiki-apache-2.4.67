cmd_parms

定義

  • 種別: struct
  • 定義箇所: include/http_config.h:74

目的

Apache HTTP Server の設定ディレクティブ解析時に、ハンドラー関数へコンテキスト情報を提供するための構造体です。

シグネチャまたは形状

typedef struct cmd_parms_struct cmd_parms;

※構造体の詳細なメンバ定義は cmd_parms_struct に依存します。

入力と出力

  • **入力**: 設定ファイル解析プロセスにおいて、ディレクティブのハンドラー関数（command_rec 内で定義）の第一引数として渡されます。
  • **出力**: ハンドラー関数は、解析結果やエラーメッセージを戻り値として返します。

前提条件と不変条件

  • 設定データは主に pconf（設定プール）に割り当てられます。
  • 解析中に使用される temp_pool は、リクエスト処理開始前に破棄されます。

アルゴリズムまたは振る舞い

1. ap_walk_config が設定ツリーを走査する際、各ディレクティブに対応するハンドラーを呼び出します。 2. その際、現在の解析状態や設定コンテキストを保持する cmd_parms がハンドラーに引き渡されます。 3. ハンドラーはこの構造体を通じて、現在の設定ファイル位置やプール情報にアクセスし、設定値をモジュール固有の構造体に反映させます。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_walk_config 等の設定解析エンジン。
  • **呼び出し先（利用例）**:
  • ap_expr_parse_cmd_mi (include/ap_expr.h)
  • ap_set_listenbacklog, ap_set_listentcpdeferaccept, ap_set_listencbratio, ap_set_listener, ap_set_send_buffer_size, ap_set_receive_buffer_size (include/ap_listen.h)
  • 各モジュールで定義された設定ハンドラー関数。

関連データ構造

  • cmd_parms_struct: 本構造体の実体定義。
  • command_rec: ディレクティブの定義とハンドラーを保持する構造体。
  • ap_conf_vector_t: モジュール固有の設定データへのアクセスに使用されるベクトル。

失敗モードと境界条件

  • 構造体内のポインタやインデックスに対する範囲外アクセス保護は、デバッグビルド以外ではマクロ実装により省略される場合があります。
  • 呼び出し元と呼び出し先の詳細なエラーハンドリングの境界条件については、本ソース excerpt には記載がありません。

保守担当者向けメモ

  • 本シンボルは cmd_parms_struct のエイリアスです。詳細なメンバ構成を確認する場合は cmd_parms_struct の定義を参照してください。
  • 設定解析のライフサイクルにおいて、この構造体は一時的なコンテキストを保持するため、ハンドラー外での保持や再利用は避けてください。
