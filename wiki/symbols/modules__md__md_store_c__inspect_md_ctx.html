inspect_md_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_store.c:348

目的

Managed Domain (MD) のストレージ走査処理において、イテレータの実行コンテキストを保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    md_store_t *store;
    md_store_group_t group;
    const char *pattern;
    const char *aspect;
    md_store_md_inspect *inspect;
    void *baton;
} inspect_md_ctx;

入力と出力

本構造体は関数への引数として渡されるコンテキストオブジェクトであり、直接的な入力・出力関数ではありません。各フィールドは以下の役割を持ちます。

  • store: 操作対象のストレージ抽象化インスタンス。
  • group: 走査対象のストレージグループ（MD_SG_DOMAINS 等）。
  • pattern: 検索対象のドメイン名パターン。
  • aspect: 検索対象のデータ側面（証明書、鍵など）。
  • inspect: 走査中に各エントリに対して実行されるコールバック関数。
  • baton: コールバック関数に渡されるユーザー定義のデータポインタ。

前提条件と不変条件

  • store は有効な md_store_t インスタンスを指している必要があります。
  • inspect コールバックが設定されている場合、走査処理中に呼び出されます。

アルゴリズムまたは振る舞い

md_store_md_iter 関数内で、ストレージ走査のイテレータ設定をカプセル化するために利用されます。走査中に見つかった各エントリに対し、このコンテキストに格納された inspect 関数が呼び出されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_store.c 内の md_store_md_iter 関数がこの構造体をインスタンス化し、走査コンテキストとして利用しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。ただし、内部の inspect フィールドに関数ポインタが格納され、走査処理中に間接的に呼び出されます。

関連データ構造

  • md_store_t: ストレージの抽象化インターフェース。
  • md_store_group_t: ストレージグループを識別する列挙型。
  • md_store_md_inspect: 走査時に実行されるコールバック関数の型定義。

失敗モードと境界条件

  • store が NULL の場合、または inspect が NULL の場合に、これらを参照する関数側で未定義の動作が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は md_store.c 内のローカルな走査ロジックに特化しています。外部モジュールから直接操作することは想定されていません。
