remoteip_proxymatch_t

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_remoteip.c:38

目的

信頼するプロキシの IP アドレス範囲と、そのプロキシが「内部」か「外部」かを識別するための設定情報を保持する。

シグネチャまたは形状

typedef struct {
    apr_ipsubnet_t *ip;
    void  *internal;
} remoteip_proxymatch_t;

入力と出力

  • **入力**: RemoteIPInternalProxy または RemoteIPTrustedProxy ディレクティブによって設定される IP サブネット情報。
  • **出力**: mod_remoteip がリクエスト処理中にプロキシの信頼性を判定するための照合データ。

前提条件と不変条件

  • ip フィールドは apr_ipsubnet_t 型であり、有効な IP サブネット範囲を指す必要がある。
  • internal フィールドはフラグとして機能し、非 NULL の場合に内部プロキシとして扱われる。

アルゴリズムまたは振る舞い

  • mod_remoteip は、この構造体の配列（config->proxymatch_ip）を保持する。
  • リクエスト処理時、接続元の IP アドレスが ip に格納されたサブネットに含まれるかを確認し、信頼できるプロキシであるかを判定する。
  • 判定結果に基づき、X-Forwarded-For ヘッダー等の解析を継続するか、あるいは最終的なクライアント IP として確定させるかを決定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_remoteip.c: 設定読み込み時（apr_array_push）にインスタンス化される。
  • modules/metadata/mod_remoteip.c: リクエスト処理時（remoteip_modify_request 等）に配列要素として参照される。
  • **呼び出し先**:
  • apr_ipsubnet_t (APR): IP アドレスのマッチング処理に使用される。

関連データ構造

  • remoteip_config_t: この構造体の配列を保持するサーバー単位の設定構造体。
  • apr_ipsubnet_t: IP アドレスの範囲検証を行うための APR 提供のデータ構造。

失敗モードと境界条件

  • 設定時に無効な IP アドレスやサブネットマスクが指定された場合、apr_ipsubnet_create 等の初期化処理でエラーが発生する可能性がある。
  • 信頼リストが空の場合、プロキシ経由の IP アドレスは信頼されず、クライアント IP の上書きは行われない。

保守担当者向けメモ

  • internal フィールドは void * 型で定義されているが、実質的にはフラグとして使用されている。型安全性の観点から、将来的なリファクタリングの対象となり得る。
  • この構造体は mod_remoteip の設定フェーズで構築され、リクエスト処理フェーズでは読み取り専用の参照として扱われる。
