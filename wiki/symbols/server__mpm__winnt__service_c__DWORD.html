DWORD

定義

  • 種別: typedef
  • 定義箇所: server/mpm/winnt/service.c:76

目的

Windows API における 32 ビット符号なし整数型を表現するための型エイリアスです。

シグネチャまたは形状

typedef unsigned long DWORD;

※Windows SDK の標準定義に準拠。

入力と出力

  • 入力: 32 ビットのデータ値。
  • 出力: 32 ビットの符号なし整数。

前提条件と不変条件

  • Windows 環境におけるデータ型であり、プラットフォーム固有のサイズ（32 ビット）を保持します。

アルゴリズムまたは振る舞い

  • Windows API（SCM 関連関数やレジストリ操作関数など）において、フラグ、サイズ、アクセス権、または戻り値の型として使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/winnt/service.c 内の関数ポインタ定義（fpt_OpenSCManager）の引数として使用。
  • 呼び出し先: os/win32/ap_regkey.c において、レジストリ値の型指定やサイズ取得、環境変数の展開処理などで広範に参照されています。

関連データ構造

  • SC_HANDLE: サービスコントロールマネージャのハンドル型。
  • RegQueryValueEx / RegQueryValueExW: レジストリ値取得時にサイズや型を保持するために使用される Windows API。

失敗モードと境界条件

  • 型の不一致: 64 ビット環境でポインタを DWORD にキャストする場合、データ切り捨てが発生する可能性があるため、ポインタ操作時には適切なキャストが必要です（ap_regkey.c では (DWORD *)&size のようにポインタ経由で扱われています）。

保守担当者向けメモ

  • 本シンボルは Windows SDK の windef.h 等で定義される標準型です。service.c 内で再定義されているわけではなく、Windows API とのインターフェースを維持するために使用されています。
  • server/mpm/winnt/service.c 内での直接的なロジック実装はなく、主に外部 API との型整合性のために存在します。
