fcgi_check_authn_provider

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:1079

目的

Apache の設定ファイルで指定された AuthnzFcgiCheckAuthnProvider ディレクティブを解析し、ディレクトリごとの認証設定（fcgi_dir_conf）を構成する。

シグネチャまたは形状

static const char *fcgi_check_authn_provider(cmd_parms *cmd, void *d, int argc, char *const argv[])

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • d: ディレクトリ設定構造体（fcgi_dir_conf）。
  • argc: 引数の数。
  • argv: 引数の配列（プロバイダー名およびオプションのキー・値ペア）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列（const char *）を返す。

前提条件と不変条件

  • 引数 d は fcgi_dir_conf 型としてキャスト可能であること。
  • プロバイダー名が必ず最初の引数として提供されること。
  • UserExpr オプションが指定された場合、ap_expr_parse_cmd による式解析が成功すること。

アルゴリズムまたは振る舞い

1. **プロバイダー名の設定**: argv[0] を dc->name に格納する。 2. **「None」の検証**: プロバイダー名が "None" の場合、追加のオプション指定がないことを確認する。 3. **オプションのループ処理**: 残りの引数をキーと値のペアとして順次処理する。

  • Authoritative: "On" または "Off" を dc->authoritative に反映。
  • DefaultUser: dc->default_user に値を設定。
  • RequireBasicAuth: "On" または "Off" を dc->require_basic_auth に反映。
  • UserExpr: ap_expr_parse_cmd を使用して式を解析し、dc->user_expr に格納。

    • 4. **エラーハンドリング**: 不明なオプションや引数不足、解析エラーが発生した場合は、cmd->pool から割り当てたエラーメッセージを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authnz_fcgi.c:1308（command_rec 構造体によるディレクティブ登録経由）。
  • **呼び出し先**:
  • apr_pstrcat, apr_psprintf: エラーメッセージ生成。
  • ap_expr_parse_cmd: UserExpr の解析。
  • strcasecmp: 引数および値の比較。

関連データ構造

  • fcgi_dir_conf: ディレクトリごとの設定を保持する構造体。
  • cmd_parms: Apache の設定コマンド実行時のコンテキスト。

失敗モードと境界条件

  • **引数不足**: プロバイダー名が指定されていない場合、またはキーに対する値が欠落している場合にエラーを返す。
  • **不正な値**: Authoritative や RequireBasicAuth に "On"/"Off" 以外の値が指定された場合、エラーを返す。
  • **式解析失敗**: UserExpr の構文が不正な場合、ap_expr_parse_cmd が返すエラーメッセージを伝播する。
  • **未知のオプション**: 定義されていないキーが指定された場合、エラーを返す。

保守担当者向けメモ

  • 本関数は設定フェーズで実行されるため、メモリ割り当てには cmd->pool を使用している。
  • UserExpr の解析には AP_EXPR_FLAG_RESTRICTED 等のフラグが使用されており、式評価のセキュリティ制約が適用されている点に注意すること。
