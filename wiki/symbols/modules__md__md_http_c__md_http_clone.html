md_http_clone

定義

  • 種別: function
  • 定義箇所: modules/md/md_http.c:95

目的

既存の md_http_t セッションの設定を継承した、新しい HTTP セッションインスタンスを作成します。

シグネチャまたは形状

apr_status_t md_http_clone(md_http_t **phttp, apr_pool_t *p, md_http_t *source_http)

入力と出力

  • **入力**:
  • phttp: 新しい md_http_t インスタンスへのポインタを格納するアドレス。
  • p: 新しいインスタンスのメモリ割り当てに使用する apr_pool_t。
  • source_http: 設定のコピー元となる既存の md_http_t インスタンス。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返します。

前提条件と不変条件

  • source_http は有効な md_http_t インスタンスである必要があります。
  • p は md_http_create が内部で使用する有効なプールである必要があります。

アルゴリズムまたは振る舞い

1. source_http の user_agent および proxy_url を使用して md_http_create を呼び出し、新しいセッションを初期化します。 2. 成功した場合、以下の設定を source_http からコピーします。

  • resp_limit (レスポンスサイズ制限)
  • timeout (タイムアウト設定)

    • 3. source_http に unix_socket_path または ca_file が設定されている場合、それらを p プール上に複製（apr_pstrdup）して新しいインスタンスに割り当てます。 4. 最終的なステータスコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_curl.c:508 (直接的な証拠)
  • **呼び出し先**:
  • md_http_create (直接的な証拠)
  • apr_pstrdup (直接的な証拠)

関連データ構造

  • md_http_t: HTTP クライアントのコンテキストを保持する構造体。

失敗モードと境界条件

  • md_http_create が失敗した場合、そのエラーコードがそのまま返されます。
  • メモリ不足により apr_pstrdup が失敗した場合、APR のメモリ管理規則に従った挙動となります。

保守担当者向けメモ

  • 本関数は、既存のセッション設定を維持しつつ、別のプールでセッションを再構築する際に使用されます。
  • unix_socket_path と ca_file は文字列として複製されるため、メモリリークを防ぐために適切なプール管理が必要です。
