dbd_load

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_dbd.c:100

目的

データベースから指定されたキーに対応するセッションデータを読み込み、呼び出し元へ提供する。

シグネチャまたは形状

apr_status_t dbd_load(apr_pool_t *p, request_rec *r, const char *key, const char **val);

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • r: 現在のリクエスト構造体。
  • key: 検索対象のセッションキー。
  • val: 読み込んだセッションデータを格納するポインタのポインタ。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。
  • *val: データベースから取得したセッション文字列（成功時）。

前提条件と不変条件

  • conf->selectlabel が設定されている必要がある。
  • mod_dbd がロードされ、データベース接続が確立可能であること。
  • val は初期状態で NULL であることが期待される。

アルゴリズムまたは振る舞い

1. ap_get_module_config を介してディレクトリ設定を取得し、selectlabel の存在を確認する。 2. dbd_init を呼び出し、データベース接続とプリペアドステートメントを準備する。 3. apr_dbd_pvbselect を実行し、現在時刻を有効期限としてクエリを投げる。 4. apr_dbd_get_row を用いて結果セットを反復処理する。 5. 最初の行が見つかった場合、その内容を apr_pstrdup でプールにコピーし、*val に格納する。 6. 全ての結果行を走査し、リソースの解放を確実に行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session_dbd.c:209
  • modules/session/mod_session_dbd.c:220
  • **呼び出し先**:
  • dbd_init
  • apr_dbd_pvbselect
  • apr_dbd_get_row
  • apr_dbd_get_entry
  • ap_log_rerror

関連データ構造

  • session_dbd_dir_conf: SQL クエリラベル等の設定を保持。
  • ap_dbd_t: データベース接続およびドライバ情報。
  • apr_dbd_prepared_t: 実行する SQL ステートメント。

失敗モードと境界条件

  • selectlabel が未定義の場合: APR_EGENERAL を返しエラーログを出力。
  • dbd_init 失敗時: そのエラーコードを伝播。
  • クエリ実行失敗時: APR_EGENERAL を返しエラーログを出力。
  • 結果取得時の行処理エラー: APR_EGENERAL を返しエラーログを出力。

保守担当者向けメモ

  • 本関数は r->pool を使用してデータベース結果を処理するため、リクエスト終了時にメモリが解放される。
