get_cookie_param

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:165

目的

HTTP リクエストの Cookie ヘッダから、指定された名前のセッションクッキー値を抽出する。

シグネチャまたは形状

char *get_cookie_param(request_rec *r, const char *name)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体へのポインタ。
  • name: 検索対象のクッキー名（文字列）。
  • **出力**:
  • 成功時: クッキーの値を含む、リクエストプールから割り当てられた文字列。
  • 失敗時: NULL。

前提条件と不変条件

  • r->headers_in に Cookie ヘッダが存在すること。
  • 抽出されたクッキー値は、リクエストのライフサイクルに紐付いたメモリプール（r->pool）上に確保される。

アルゴリズムまたは振る舞い

1. r->headers_in から Cookie ヘッダ文字列を取得する。 2. ap_strstr_c を使用して、ヘッダ内から name を検索する。 3. 見つかった箇所がクッキーの境界（行頭、;、,、または空白）にあるかを確認する。 4. クッキー名に続く = の後の値を特定する。 5. 値の末尾が ; または , であれば、そこで終端文字を挿入して値を切り出す。 6. 抽出した値を apr_pstrdup で複製して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_balancer.c:303 (find_session_route 内で呼び出されることが確認されている)
  • **呼び出し先**: apr_table_get, ap_strstr_c, strlen, isspace, apr_pstrdup, strchr

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • Cookie ヘッダが存在しない場合、NULL を返す。
  • 指定した name がヘッダ内に存在しない場合、NULL を返す。
  • クッキー値の後に = が存在しない場合や、値が空の場合は NULL を返す。
  • 複数のクッキーが含まれる場合、最初の合致箇所を優先する。

保守担当者向けメモ

  • 本関数はスティッキーセッションのルート特定に使用される。
  • strchr による ; および , の検索は、クッキー値の区切りを正しく処理するためのものである。
