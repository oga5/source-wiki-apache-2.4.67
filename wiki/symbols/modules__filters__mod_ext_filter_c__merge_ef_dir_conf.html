merge_ef_dir_conf

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ext_filter.c:105

目的

ディレクトリ単位の設定構造体 (ef_dir_t) をマージし、親設定から子設定へ設定値を継承または上書きするために使用されます。

シグネチャまたは形状

static void *merge_ef_dir_conf(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定 (ef_dir_t へのポインタ)。
  • overridesv: 現在のディレクトリで定義された設定 (ef_dir_t へのポインタ)。
  • **出力**:
  • マージ結果を格納した新しい ef_dir_t 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は ef_dir_t 型としてキャスト可能であること。
  • 設定値が -1 の場合、その項目は未設定（継承対象）とみなされます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい ef_dir_t 構造体をプール p 上に確保します。 2. log_stderr および onfail の各フィールドについて、overridesv 側が -1 でない（管理者が値を指定している）場合はその値を採用し、そうでない場合は basev の値を継承します。 3. マージされた新しい構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_ext_filter.c:949 (Apache の設定マージ機構から呼び出されます)。
  • **呼び出し先**: 直接的な関数呼び出しはありません（apr_pcalloc を除く）。

関連データ構造

  • ef_dir_t: ディレクトリ単位の設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性があります。
  • 入力ポインタが NULL の場合の挙動については、本ソースコード内では明示的なチェックは行われていません。

保守担当者向けメモ

  • 本関数は Apache の設定マージフェーズで実行されます。新しい設定項目を ef_dir_t に追加した場合は、必ず本関数内でマージロジックを更新してください。
