authn_core_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authn_core.c:55

目的

Apache HTTP Server の認証サブシステムにおいて、ディレクトリごとの認証設定（AuthType および AuthName）を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    ap_expr_info_t *ap_auth_type;
    int auth_type_set;
    ap_expr_info_t *ap_auth_name;
} authn_core_dir_conf;

入力と出力

  • **入力**: AuthType および AuthName ディレクティブにより設定されます。
  • **出力**: ap_expr_info_t 形式で保持されるため、リクエスト処理時に動的な評価結果を提供します。

前提条件と不変条件

  • ap_expr_info_t 型のフィールドは、ap_expr エンジンによって評価可能な式として格納されます。
  • auth_type_set は、AuthType が明示的に設定されたかどうかを示すフラグとして機能します。

アルゴリズムまたは振る舞い

  • 本構造体は、設定フェーズで set_authname および set_authtype 関数によって値が設定されます。
  • リクエスト処理時には、ap_get_module_config を介してディレクトリ設定として取得され、認証プロバイダーの実行やレルムの特定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authn_core.c 内の create_authn_core_dir_config（初期化）
  • merge_authn_core_dir_config（設定マージ）
  • authn_alias_check_password および authn_alias_get_realm_hash（実行時の設定参照）
  • **呼び出し先**:
  • ap_expr サブシステム（式評価時）

関連データ構造

  • ap_expr_info_t: AuthType や AuthName の動的評価ロジックを保持する構造体。
  • provider_alias_rec: エイリアスされたプロバイダー内で、本構造体をディレクトリ設定として保持します。

失敗モードと境界条件

  • ap_expr の評価が失敗した場合、認証処理に影響を及ぼす可能性があります。
  • 設定が未定義の場合、auth_type_set が適切に管理されていないと、認証タイプが正しく認識されない可能性があります。

保守担当者向けメモ

  • 本構造体は r->per_dir_config を通じてアクセスされます。
  • AuthType や AuthName への直接アクセスは非推奨であり、本モジュールが提供するオプション関数経由でのアクセスが推奨されています。
