dav_get_lock_hooks

定義

  • 種別: function
  • 定義箇所: modules/dav/main/mod_dav.c:228

目的

現在のアクティブなリクエストに関連付けられた WebDAV ロック操作用のフックセット（dav_hooks_locks）を取得します。

シグネチャまたは形状

const dav_hooks_locks *dav_get_lock_hooks(request_rec *r)

入力と出力

  • **入力**: r (Apache の request_rec 構造体へのポインタ)
  • **出力**: 現在のリクエストに対して設定されている dav_hooks_locks 構造体への定数ポインタ。

前提条件と不変条件

  • r は有効なリクエスト構造体である必要があります。
  • dav_get_provider(r) が有効なプロバイダーを返すことが前提です。

アルゴリズムまたは振る舞い

1. dav_get_provider(r) を呼び出し、現在のリクエストに対応するプロバイダー構造体を取得します。 2. 取得したプロバイダー構造体内の locks メンバ（ロック操作用フック）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.h 内で DAV_GET_HOOKS_LOCKS(r) マクロとして定義されており、WebDAV ロック機能を利用する各メソッドから間接的に呼び出されます。
  • **呼び出し先**:
  • dav_get_provider(r) (同一ファイル内のユーティリティ関数)

関連データ構造

  • dav_hooks_locks: WebDAV ロック操作（get_supportedlock, parse_locktoken, format_locktoken 等）を定義するフック構造体。
  • request_rec: Apache HTTP Server のリクエスト状態を保持する構造体。

失敗モードと境界条件

  • dav_get_provider(r) が NULL を返す場合、本関数はセグメンテーションフォールトを引き起こす可能性があります（呼び出し元でプロバイダーの存在が保証されている必要があります）。

保守担当者向けメモ

  • 本関数は mod_dav のプロバイダーアーキテクチャの抽象レイヤーの一部です。ロック機能の拡張や変更を行う際は、dav_hooks_locks 構造体の定義（mod_dav.h）と、各プロバイダー実装を確認してください。
