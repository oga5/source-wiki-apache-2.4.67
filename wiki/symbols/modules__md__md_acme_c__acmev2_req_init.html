acmev2_req_init

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme.c:228

目的

ACME v2 プロトコルにおけるリクエストペイロードの準備および JWS（JSON Web Signature）署名を行う。

シグネチャまたは形状

static apr_status_t acmev2_req_init(md_acme_req_t *req, md_json_t *jpayload)

入力と出力

  • **入力**:
  • req: ACME リクエストを保持する md_acme_req_t 構造体。
  • jpayload: 送信する JSON ペイロード（NULL の場合は空のペイロードとして扱う）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は APR_EINVAL 等のステータスコード。

前提条件と不変条件

  • req->acme->acct が初期化されていること。
  • req->p（メモリプール）が有効であること。

アルゴリズムまたは振る舞い

1. アカウント情報（req->acme->acct）の存在を確認し、存在しない場合は APR_EINVAL を返す。 2. jpayload が提供されている場合、それをコンパクトな JSON 文字列に変換する。提供されていない場合は空文字列を使用する。 3. 変換したペイロードをログに出力する。 4. md_jws_sign を呼び出し、以下の情報を用いてリクエストの署名を行う。

  • req->req_json: 署名結果を格納するターゲット。
  • req->p: メモリプール。
  • req->prot_fields: JWS プロテクトヘッダーフィールド。
  • req->acme->acct_key: アカウントの秘密鍵。
  • req->acme->acct->url: アカウント URL。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme.c 内で acme->req_init_fn に代入されており、リクエスト送信プロセスの一部として間接的に呼び出される（直接的な呼び出し箇所はソース抜粋内には記載なし）。
  • **呼び出し先**:
  • md_json_writep
  • md_log_perror
  • md_jws_sign

関連データ構造

  • md_acme_req_t: リクエストの状態とコンテキストを保持。
  • md_json_t: 送信データ。
  • md_data_t: ペイロードのデータ本体と長さを保持する一時構造体。

失敗モードと境界条件

  • req->acme->acct が NULL の場合: APR_EINVAL を返す。
  • jpayload の JSON 変換に失敗した場合: APR_EINVAL を返す。
  • md_jws_sign が失敗した場合: その戻り値をそのまま返す。

保守担当者向けメモ

  • 本関数は ACME v2 専用の初期化ロジックです。
  • md_jws_sign への依存が強いため、署名アルゴリズムやヘッダー構成に変更がある場合は md_jws.c との整合性に注意してください。
