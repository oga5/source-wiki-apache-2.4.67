ap_get_token

定義

  • 種別: function
  • 定義箇所: server/util.c:1687

目的

文字列からトークンを抽出します。主に HTTP ヘッダー値などのリスト形式の文字列を解析する際に、区切り文字や引用符を考慮して個別の要素を切り出すために使用されます。

シグネチャまたは形状

char *ap_get_token(apr_pool_t *p, const char **accept_line, int accept_white)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • accept_line: 解析対象の文字列へのポインタ。関数終了後、次のトークンの開始位置まで更新されます。
  • accept_white: 空白文字をトークンの一部として許容するかどうかを指定するフラグ。
  • **出力**:
  • 抽出されたトークン文字列（apr_pool_t から割り当てられたメモリ）。

前提条件と不変条件

  • accept_line が指す文字列はヌル終端されている必要があります。
  • コメント（() で囲まれた部分）は、この関数が呼び出される前に既に除去されていることが前提です。

アルゴリズムまたは振る舞い

1. **先頭の空白スキップ**: accept_line の先頭にある空白文字をスキップします。 2. **トークン抽出**: 以下のいずれかに到達するまでポインタを進めます。

  • 文字列の終端 (\0)
  • 区切り文字（; または ,）
  • 空白文字（accept_white が偽の場合のみ）
  • 引用符（"）で囲まれた範囲は、内部の区切り文字や空白を無視して一括して読み飛ばします。

    • 3. **メモリ確保**: 抽出したトークンを apr_pstrmemdup を用いてプールから割り当て、コピーを返します。 4. **ポインタ更新**: 抽出後の位置から再度空白をスキップし、accept_line を次のトークン開始位置へ更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_brotli.c および modules/filters/mod_deflate.c において、ヘッダー値の解析に使用されています。
  • **呼び出し先**: apr_isspace（マクロ）、apr_pstrmemdup（APR ライブラリ関数）。

関連データ構造

  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • 文字列が空の場合や、区切り文字のみで構成されている場合は、空のトークンが返される可能性があります。
  • 引用符が閉じられていない場合、文字列の終端までをトークンとして扱います。

保守担当者向けメモ

  • 本関数は HTTP ヘッダーのパースに特化しています。RFC 準拠の厳密な解析が必要な場合は、呼び出し側で結果を検証してください。
  • accept_white フラグの挙動は、トークン内に空白を含める必要があるかどうかに依存するため、呼び出し時の引数設定に注意してください。
