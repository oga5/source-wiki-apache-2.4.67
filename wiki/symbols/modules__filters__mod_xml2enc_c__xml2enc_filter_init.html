xml2enc_filter_init

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_xml2enc.c:304

目的

リクエストごとのフィルタコンテキスト（xml2ctx）を初期化し、モジュール設定に基づいて初期状態をセットアップする。

シグネチャまたは形状

static apr_status_t xml2enc_filter_init(ap_filter_t* f)

入力と出力

  • **入力**: ap_filter_t* f（フィルタインスタンス）。
  • **出力**: apr_status_t（成功時は APR_SUCCESS を返す）。

前提条件と不変条件

  • f->ctx が未初期化（NULL）である場合にのみ初期化処理が実行される。
  • f->r->per_dir_config から xml2enc_module の設定を取得できる必要がある。
  • リクエストプール（f->r->pool）を使用してメモリが確保される。

アルゴリズムまたは振る舞い

1. f->ctx が NULL かどうかを確認する。 2. NULL の場合、ap_get_module_config を呼び出し、現在のディレクトリ設定（xml2cfg）を取得する。 3. apr_pcalloc を使用して xml2ctx 構造体をリクエストプール上に確保し、f->ctx に割り当てる。 4. 確保したコンテキストの xml2enc フィールドを XML_CHAR_ENCODING_NONE に初期化する。 5. 設定内の skipto が定義されている場合、コンテキストのフラグに ENC_SKIPTO を追加する。 6. APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_xml2enc.c:592 にて参照されている。
  • **呼び出し先**:
  • ap_get_module_config (Apache API)
  • apr_pcalloc (APR API)

関連データ構造

  • ap_filter_t: フィルタチェーンの構造体。
  • xml2ctx: フィルタの実行状態を保持する構造体。
  • xml2cfg: モジュールのディレクトリごとの設定構造体。

失敗モードと境界条件

  • f->ctx が既に初期化されている場合は、何も行わず APR_SUCCESS を返す。
  • メモリ確保失敗時の挙動については、apr_pcalloc の仕様に依存する（通常はアボート）。

保守担当者向けメモ

  • 本関数はフィルタの初回実行時に一度だけ呼び出されることを想定している。
  • xml2ctx の構造変更や、初期化時に必要なフラグの追加がある場合は本関数を更新すること。
