close_logfile

定義

  • 種別: function
  • 定義箇所: support/rotatelogs.c:193

目的

ログファイルのクローズ処理および関連するメモリプールの解放を行う。

シグネチャまたは形状

static void close_logfile(rotate_config_t *config, struct logfile *logfile)

入力と出力

  • **入力**:
  • config: 実行時の設定情報（rotate_config_t 型）。verbose フラグの確認に使用。
  • logfile: クローズ対象のログファイル構造体（struct logfile 型）。
  • **出力**: なし（void）。

前提条件と不変条件

  • logfile->fd は有効なファイルハンドルであること。
  • logfile->pool は当該ログファイルに関連付けられた有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. config->verbose が真の場合、標準エラー出力にクローズ対象のファイル名を出力する。 2. apr_file_close を呼び出し、ログファイルのファイルハンドルを閉じる。 3. apr_pool_destroy を呼び出し、ログファイルに関連付けられたメモリプールを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: support/rotatelogs.c:485（status->current を引数として呼び出し）。
  • **呼び出し先**: apr_file_close（APR）、apr_pool_destroy（APR）。

関連データ構造

  • rotate_config_t: 実行設定を保持。
  • struct logfile: ログファイルのファイルハンドル (fd) とメモリプール (pool) を保持。

失敗モードと境界条件

  • logfile が NULL の場合、あるいは logfile->fd や logfile->pool が初期化されていない状態で呼び出された場合の挙動は、APR の実装に依存する。

保守担当者向けメモ

  • 本関数はログローテーションの切り替え時に呼び出され、リソースリークを防ぐために重要である。
  • apr_pool_destroy により、当該ログファイルに関連する動的メモリがすべて解放されるため、呼び出し後に logfile 構造体へのアクセスを行わないよう注意すること。
