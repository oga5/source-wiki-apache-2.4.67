req_headers_out_table

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:332

目的

HTTPレスポンスヘッダー（r->headers_out）をLuaスクリプトからアクセス可能なLuaテーブルへ変換して返す。

シグネチャまたは形状

static int req_headers_out_table(lua_State *L)

入力と出力

  • **入力**: Luaスタックの第1引数に Apache2.Request userdata を受け取る。
  • **出力**: r->headers_out の内容を反映したLuaテーブルをスタックにプッシュし、戻り値として 1 を返す。

前提条件と不変条件

  • Luaスタックの第1引数は ap_lua_check_request_rec によって検証可能な有効な request_rec ポインタである必要がある。
  • r->headers_out は apr_table_t 型の有効なポインタであること。

アルゴリズムまたは振る舞い

1. ap_lua_check_request_rec(L, 1) を呼び出し、Luaスタックから request_rec 構造体を取得する。 2. 取得した request_rec の headers_out フィールドを引数として req_aprtable2luatable を呼び出す。 3. req_aprtable2luatable が実行した結果（Luaテーブルのプッシュ）をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_request.c:2837 にて makefun を通じて登録されている。
  • **呼び出し先**:
  • ap_lua_check_request_rec
  • req_aprtable2luatable

関連データ構造

  • request_rec (Apache HTTP Server のリクエスト構造体)
  • apr_table_t (APRのテーブル構造体)
  • lua_State (Lua VMの状態)

失敗モードと境界条件

  • ap_lua_check_request_rec が失敗した場合、Lua側でエラーが発生し、本関数は正常終了しない可能性がある。
  • headers_out が空の場合、req_aprtable2luatable は空のLuaテーブルを返す。

保守担当者向けメモ

  • 本関数は req_headers_in_table や req_err_headers_out_table と同様のパターンで実装されている。
  • メモリ管理は request_rec のプールに依存しているため、明示的な解放は不要である。
