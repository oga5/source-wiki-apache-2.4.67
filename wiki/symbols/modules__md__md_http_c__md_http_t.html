md_http_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_http.c:27

目的

HTTP クライアントの通信セッションを管理するコンテキスト構造体です。APR（Apache Portable Runtime）のプールやバケットアロケータを保持し、通信設定の抽象化と実行実装（md_http_impl_t）へのディスパッチを行います。

シグネチャまたは形状

struct md_http_t {
    apr_pool_t *pool;
    apr_bucket_alloc_t *bucket_alloc;
    int next_id;
    apr_off_t resp_limit;
    md_http_impl_t *impl;
    void *impl_data;
    const char *user_agent;
    const char *proxy_url;
    const char *unix_socket_path;
    md_http_timeouts_t timeout;
    const char *ca_file;
};

入力と出力

  • **入力**: md_http_create() を通じて初期化され、各フィールドに設定値が格納されます。
  • **出力**: 保持する impl_data を通じて、具体的な HTTP 実装（md_curl.c 等）がネットワーク I/O の状態を管理します。

前提条件と不変条件

  • pool はセッションのライフサイクルを管理するため、md_http_req_destroy 等で適切に管理される必要があります。
  • impl は実行時に注入されるプラグイン可能な実装であり、プロセス全体で共有される可能性があります。

アルゴリズムまたは振る舞い

  • **セッション管理**: HTTP リクエストの構築から実行、クリーンアップまでの一連のライフサイクルを保持します。
  • **実装の委譲**: 実際の通信処理は impl メンバに登録された関数ポインタを通じて実行されます。
  • **設定の継承**: リクエスト生成時に、この構造体からタイムアウトやプロキシ設定が継承されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme.h で参照されており、ACME プロトコルの通信セッションとして利用されます。
  • **呼び出し先**: modules/md/md_curl.c が本構造体を操作し、impl_data を使用して libcurl のセッション状態を保持します。

関連データ構造

  • md_http_impl_t: 通信処理の実装インターフェース。
  • md_http_request_t: 個別のリクエスト状態を保持する構造体。
  • md_http_timeouts_t: タイムアウト設定を保持する構造体。
  • apr_pool_t: メモリ管理用プール。
  • apr_bucket_alloc_t: バケットブリゲード用アロケータ。

失敗モードと境界条件

  • impl が設定されていない状態で perform 系関数を呼び出した場合の挙動は、実装側のディスパッチロジックに依存します。
  • resp_limit を超えるレスポンスを受信した場合の挙動は、各実装の perform 実装に依存します。

保守担当者向けメモ

  • impl_data は実装側が自由に利用できる領域です。
  • cur_impl は静的変数としてプロセス全体で共有されるため、スレッドセーフな実装が求められます。
  • メモリ管理は pool に依存しているため、インスタンスの破棄時には関連するサブプールが正しく解放されることを確認してください。
