md_get_by_dns_overlap

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:177

目的

指定されたマネージドドメイン（md_t）のリストから、対象のドメインと名前が異なり、かつドメイン名が重複（オーバーラップ）している別のマネージドドメインを検索します。

シグネチャまたは形状

md_t *md_get_by_dns_overlap(struct apr_array_header_t *mds, const md_t *md)

入力と出力

  • **入力**:
  • mds: 検索対象となる md_t 構造体の配列（apr_array_header_t）。
  • md: 比較基準となる md_t 構造体。
  • **出力**:
  • 条件に合致する md_t へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • mds は有効な apr_array_header_t である必要があります。
  • md は有効な md_t 構造体である必要があります。
  • mds 内の各要素は md_t * 型として格納されていることが期待されます。

アルゴリズムまたは振る舞い

1. mds 配列内の各要素 o を順次走査します。 2. o->name と md->name を比較し、名前が異なる場合のみ次の判定へ進みます。 3. md_common_name(o, md) を呼び出し、両者のドメインリストに共通する名前が存在するかを確認します。 4. 共通するドメイン名が存在する場合、その md_t オブジェクト o を即座に返します。 5. 配列の最後まで一致が見つからなかった場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md.h:277 に宣言が存在しますが、具体的な呼び出し箇所は提供されたソースからは特定できません。
  • **呼び出し先**:
  • md_common_name（ドメインの重複判定に使用）
  • APR_ARRAY_IDX（配列要素へのアクセスに使用）

関連データ構造

  • md_t: 管理対象ドメインの情報を保持する構造体。
  • apr_array_header_t: APR が提供する動的配列構造体。

失敗モードと境界条件

  • mds が空の場合、ループは実行されず NULL を返します。
  • mds 内のすべてのドメインが md と名前が一致するか、あるいはドメインの重複がない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は、ドメイン設定の競合を検出するために使用されます。
  • md_common_name の実装に依存しているため、ドメイン名の正規化ルールに変更があった場合は影響を受けます。
