harness_ctx

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_filter.c:56

目的

mod_filter における実行時のフィルタコンテキストを保持します。選択されたプロバイダの関数ポインタと、そのプロバイダ固有の初期化コンテキストを統合し、フィルタチェーン実行時のディスパッチを可能にします。

シグネチャまたは形状

typedef struct {
    ap_out_filter_func func;
    void *fctx;
    provider_ctx *init_ctx;
} harness_ctx;

入力と出力

  • func: 実行対象となるフィルタ関数（ap_out_filter_func）。
  • fctx: フィルタ関数に渡されるコンテキストデータ。
  • init_ctx: プロバイダの初期化時に生成された provider_ctx 構造体へのポインタ。

前提条件と不変条件

  • harness_ctx はリクエストのライフサイクル（r->pool）に関連付けられて割り当てられます。
  • フィルタ実行時、f->ctx を通じて本構造体が参照されます。

アルゴリズムまたは振る舞い

1. filter_harness が呼び出されると、filter_lookup により条件判定が行われます。 2. 判定結果に基づき、マッチしたプロバイダの関数とコンテキストが本構造体に格納されます。 3. 後続のフィルタ処理において、func が fctx を引数として呼び出され、実際のデータ変換が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_filter.c:107: filter_init 内で apr_pcalloc によりインスタンス化されます。
  • modules/filters/mod_filter.c:141, 280: filter_harness および関連処理内で f->ctx からキャストして取得されます。
  • **呼び出し先**:
  • 構造体内の func フィールドは、登録された各フィルタプロバイダの関数を指します。

関連データ構造

  • ap_out_filter_func: Apache フィルタフレームワークの出力フィルタ関数型。
  • provider_ctx: プロバイダ固有の初期化状態を保持する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時: apr_pcalloc が失敗した場合、リクエスト処理は中断されます。
  • 未初期化状態: フィルタが正しく設定されていない場合、func が NULL となり、実行時にクラッシュする可能性があります。

保守担当者向けメモ

  • 本構造体は mod_filter のディスパッチャとしての役割を果たすための「実行時コンテナ」です。
  • fctx と init_ctx の役割分担については、各プロバイダモジュールの実装に依存するため、変更を加える際は filter_init および filter_harness のロジックを併せて確認してください。
