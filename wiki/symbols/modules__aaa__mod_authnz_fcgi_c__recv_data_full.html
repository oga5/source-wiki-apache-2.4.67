recv_data_full

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:191

目的

FastCGI バックエンドから指定されたバイト数（buflen）のデータを完全に読み込むまで、recv_data を繰り返し呼び出すラッパー関数です。

シグネチャまたは形状

static apr_status_t recv_data_full(const fcgi_provider_conf *conf,
                                   request_rec *r,
                                   apr_socket_t *s,
                                   char *buf,
                                   apr_size_t buflen)

入力と出力

  • **入力**:
  • conf: FastCGI プロバイダー設定。
  • r: 現在のリクエストレコード。
  • s: 通信に使用するソケット。
  • buf: 受信データを格納するバッファ。
  • buflen: 読み込むべき合計バイト数。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。読み込み中にエラーが発生した場合は、recv_data が返したエラーコードをそのまま返します。

前提条件と不変条件

  • buf は少なくとも buflen バイト以上のメモリ領域を確保している必要があります。
  • recv_data が正常に動作する環境（ソケット接続済み）であることを前提とします。

アルゴリズムまたは振る舞い

1. cumulative_len（累積読み込みバイト数）を 0 に初期化します。 2. cumulative_len が buflen に達するまで、以下のループを実行します。

  • 残りの読み込み必要量（buflen - cumulative_len）を計算します。
  • recv_data を呼び出し、データを buf のオフセット位置に読み込みます。
  • recv_data が APR_SUCCESS 以外を返した場合は、即座にその値を返して終了します。
  • 読み込んだバイト数を cumulative_len に加算します。

    • 3. 全データの読み込みが完了した時点で APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:500
  • modules/aaa/mod_authnz_fcgi.c:662
  • **呼び出し先**:
  • recv_data (同ファイル内)

関連データ構造

  • fcgi_provider_conf: FastCGI バックエンドの接続情報を保持する構造体。
  • request_rec: Apache のリクエスト状態を保持する構造体。
  • apr_socket_t: APR のソケット抽象化構造体。

失敗モードと境界条件

  • **通信エラー**: recv_data がエラーを返した場合、そのエラーコードを伝播します。
  • **部分的な読み込み**: recv_data が一度に要求された全量を読み込めない場合でも、ループによって buflen に達するまで再試行されます。

保守担当者向けメモ

  • 本関数は、FastCGI レコードのヘッダーやペイロードを確実に取得するために使用されます。
  • recv_data の挙動に依存しているため、通信タイムアウトや切断時の挙動は recv_data の実装を確認してください。
