dialup_send_pulse

定義

  • 種別: function
  • 定義箇所: modules/test/mod_dialup.c:46

目的

設定された帯域制限（bytes_per_second）に基づき、バケットブリゲードからデータを断続的に送信する。

シグネチャまたは形状

static int dialup_send_pulse(dialup_baton_t *db)

入力と出力

  • **入力**: dialup_baton_t *db（送信データ、設定、リクエスト情報を含むコンテキスト）。
  • **出力**: int 型のステータスコード。
  • DONE: 全データの送信完了。
  • SUSPENDED: 送信すべきデータが残っており、処理を一時停止。
  • HTTP_INTERNAL_SERVER_ERROR: 接続切断やブリゲード操作の失敗。
  • AP_FILTER_ERROR: ap_pass_brigade の送信失敗。

前提条件と不変条件

  • db->bb に送信対象のデータが格納されていること。
  • db->tmpbb は一時的なバケット移動用として利用可能であること。
  • リクエストプール (r->pool) を介してメモリ管理が行われていること。

アルゴリズムまたは振る舞い

1. db->bb が空になるか、設定された bytes_per_second に達するまでループする。 2. apr_brigade_partition を使用して、送信制限分を db->tmpbb に分割・移動する。 3. apr_bucket_flush_create でフラッシュバケットを作成し、db->tmpbb の末尾に追加する。 4. ap_pass_brigade を呼び出し、フィルタチェーンを通じてデータを送信する。 5. 送信後、apr_brigade_cleanup で一時ブリゲードをクリアする。 6. 全データ送信完了時は DONE を返し、未送信分がある場合は SUSPENDED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/test/mod_dialup.c:111 (dialup_handler 内)
  • modules/test/mod_dialup.c:207 (dialup_callback 内)
  • **呼び出し先**:
  • apr_brigade_partition, apr_brigade_length, apr_brigade_cleanup
  • ap_pass_brigade
  • ap_log_rerror

関連データ構造

  • dialup_baton_t: 非同期処理のコンテキストを保持。
  • apr_bucket_brigade: 送信データのキュー。

失敗モードと境界条件

  • **接続切断**: db->r->connection->aborted が真の場合、HTTP_INTERNAL_SERVER_ERROR を返す。
  • **ブリゲード操作失敗**: apr_brigade_partition が APR_SUCCESS 以外を返した場合、エラーとして終了する。
  • **送信失敗**: ap_pass_brigade が失敗した場合、ログを出力し AP_FILTER_ERROR を返す。

保守担当者向けメモ

  • SUSPENDED 状態の利用は MPM の実装に依存するため、環境によっては挙動が異なる可能性がある。
