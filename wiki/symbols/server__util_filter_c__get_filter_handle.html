get_filter_handle

定義

  • 種別: function
  • 定義箇所: server/util_filter.c:154

目的

指定された名前のフィルタに対応する ap_filter_rec_t 構造体を、フィルタ名が格納されたトライ木から検索・取得します。

シグネチャまたは形状

static ap_filter_rec_t *get_filter_handle(const char *name,
                                          const filter_trie_node *filter_set)

入力と出力

  • **入力**:
  • name: 検索対象のフィルタ名（文字列）。
  • filter_set: 検索対象となるトライ木のルートノード（filter_trie_node）。
  • **出力**:
  • 成功時: 見つかったフィルタ定義へのポインタ (ap_filter_rec_t *)。
  • 失敗時: NULL。

前提条件と不変条件

  • filter_set が NULL の場合、検索は行われず直ちに NULL を返します。
  • フィルタ名はトライ木構造に従って文字単位で走査されます。

アルゴリズムまたは振る舞い

1. filter_set が有効な場合、name の各文字に対してトライ木を降下します。 2. 各ノードにおいて、子ノードのリストを二分探索（while ループ）し、現在の文字と一致するノードを特定します。 3. 一致するノードが見つからない場合、探索を中断し NULL を返します。 4. name の全文字を走査し終えた時点で、到達したノードに frec（フィルタ定義）が保持されていればそれを返します。それ以外の場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ap_get_output_filter_handle (server/util_filter.c:195): registered_output_filters を渡して呼び出し。
  • ap_get_input_filter_handle (server/util_filter.c:200): registered_input_filters を渡して呼び出し。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • filter_trie_node: フィルタ名を保持するトライ木のノード構造体。
  • ap_filter_rec_t: フィルタの定義情報（フィルタ名やコールバック関数など）を保持する構造体。

失敗モードと境界条件

  • name がトライ木に存在しない場合、NULL を返します。
  • filter_set が空（NULL）の場合、NULL を返します。

保守担当者向けメモ

  • 本関数は server/util_filter.c 内の静的関数であり、外部からは ap_get_output_filter_handle および ap_get_input_filter_handle を介して利用されます。
  • トライ木の二分探索ロジックは、filter_trie_node の children 配列がソートされていることを前提としています。
