show_compile_settings

定義

  • 種別: function
  • 定義箇所: server/main.c:95

目的

Apache HTTP Server のビルド時設定、バージョン情報、および MPM（Multi-Processing Module）の構成情報を標準出力へ表示する。

シグネチャまたは形状

static void show_compile_settings(void)

入力と出力

  • 入力: なし（引数を受け取らない）。
  • 出力: なし（戻り値なし）。副作用として、サーバーのバージョン、ビルド日時、モジュールマジックナンバー、アーキテクチャ（ビット数）、およびコンパイル時に定義されたマクロ（-D オプション）やパス設定を stdout に出力する。

前提条件と不変条件

  • printf が利用可能な標準出力環境であること。
  • 内部で呼び出される ap_get_server_description や ap_get_server_built 等の関数が、サーバーの初期化プロセスにおいて適切な値を返せる状態であること。

アルゴリズムまたは振る舞い

1. サーバーの基本情報（バージョン、ビルド日時、マジックナンバー）を出力する。 2. APR および PCRE のバージョン情報を出力する。 3. sizeof(void *) を用いて実行環境のアーキテクチャ（ビット数）を算出・出力する。 4. show_mpm_settings() を呼び出し、MPM 固有の設定情報を出力する。 5. コンパイル時に指定された各種マクロ（BIG_SECURITY_HOLE, APR_HAS_MMAP 等）およびデフォルトパス（HTTPD_ROOT, SERVER_CONFIG_FILE 等）の定義状況を条件付きコンパイル（#ifdef）に基づいて出力する。

呼び出し元と呼び出し先

  • 呼び出し元: server/main.c 内の main 関数（コマンドライン引数解析の過程で呼び出される）。
  • 呼び出し先: show_mpm_settings()。

関連データ構造

  • なし（特定のデータ構造を操作する関数ではなく、定数やマクロの情報を表示する関数である）。

失敗モードと境界条件

  • 標準出力がクローズされている場合、出力は行われない。
  • コンパイル時のマクロ定義状況により、出力される項目は環境ごとに大きく異なる。

保守担当者向けメモ

  • 新しいコンパイルオプションや重要なパス設定を追加した際は、本関数にも出力ロジックを追加することで、デバッグや構成確認が容易になる。
  • APR_MAJOR_VERSION による分岐が存在するため、APR のバージョンアップ時にはこのロジックが適切か確認すること。
