kept_body_filter_init

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:177

目的

サブリクエストの開始時に、保持されているリクエストボディのメタデータをヘッダー情報へ反映させ、後続のフィルタが正しくデータを処理できるように初期化します。

シグネチャまたは形状

static int kept_body_filter_init(ap_filter_t *f)

入力と出力

  • **入力**: ap_filter_t *f （現在のフィルタコンテキスト。f->r を通じてリクエスト情報にアクセスする）
  • **出力**: int （常に OK を返す）

前提条件と不変条件

  • f->r->kept_body が存在する場合にのみ、ヘッダーの書き換えが行われます。
  • r->kept_body は r->pool に紐付いており、リクエストのライフサイクル内で有効であることが前提です。

アルゴリズムまたは振る舞い

1. f->r から kept_body（apr_bucket_brigade）を取得します。 2. kept_body が存在する場合：

  • r->headers_in から Transfer-Encoding ヘッダーを削除します。
  • apr_brigade_length を使用して、保持されているボディの総バイト数を計算します。
  • 計算した長さを Content-Length ヘッダーとして r->headers_in に設定します。

    • 3. OK を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_request.c:379 にてフィルタチェーンへ登録される際に参照されています。
  • **呼び出し先**:
  • apr_table_unset / apr_table_setn (Apache APR)
  • apr_brigade_length (Apache APR)
  • apr_off_t_toa (Apache APR)

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。r->kept_body および r->headers_in を参照。
  • apr_bucket_brigade: 保持されたリクエストボディのデータ構造。

失敗モードと境界条件

  • kept_body が NULL の場合、ヘッダーの更新は行われず、何もせずに OK を返します。
  • apr_brigade_length の計算結果が apr_off_t の範囲を超える場合の挙動については、ソース内に明示的なエラーハンドリングの記述はありません。

保守担当者向けメモ

  • 本関数はサブリクエストの初期化フェーズで実行されるため、ヘッダーの変更が後続のフィルタやアプリケーションに影響を与える点に注意してください。
  • Transfer-Encoding を除去し Content-Length を固定値に置き換えることで、サブリクエストがチャンク転送を意識せずにボディを読み込めるようにしています。
