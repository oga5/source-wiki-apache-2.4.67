ssl_hook_Access

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:1047

目的

Apache HTTP Server のアクセス制御フェーズにおいて、SSL/TLS 接続の検証、再ネゴシエーションの要否判断、および SSLRequire ディレクティブによる条件式評価を行う。

シグネチャまたは形状

int ssl_hook_Access(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (HTTP ステータスコードまたは DECLINED)
  • HTTP_FORBIDDEN: SSL 接続が必須であるにもかかわらず未確立の場合、または SSLRequire の条件を満たさない場合。
  • DECLINED: SSL 接続が有効でない場合、または SSL 関連のチェックを通過し、他のモジュールによるアクセス制御を許可する場合。

前提条件と不変条件

  • スレーブ接続の場合、マスター接続から SSLConnRec を取得して検証を行う。
  • サブリクエストやスレーブ接続において、並行リクエストとの競合を防ぐため再ネゴシエーションを禁止するガードロジックが適用される。

アルゴリズムまたは振る舞い

1. **接続確認**: リクエストに関連付けられた SSL 接続情報を取得。 2. **必須チェック**: SSLRequireSSL が設定されている場合、SSL 接続がなければ拒否。 3. **有効性確認**: SSL が無効な場合は DECLINED を返す。 4. **再ネゴシエーション**: プロトコルバージョンに応じて ssl_hook_Access_modern または ssl_hook_Access_classic を呼び出し、ディレクトリ単位の設定変更に伴う再ネゴシエーションを実行。 5. **ユーザー名設定**: FakeBasicAuth が無効な場合、設定されたユーザー名変数をルックアップして r->user に代入。 6. **条件評価**: SSLRequire で定義された式を ap_expr_exec で評価し、不一致の場合は HTTP_FORBIDDEN を返す。 7. **終了**: すべてのチェックを通過した場合、他のモジュールによる認証を妨げないよう DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c (Apache の ap_hook_check_access フックとして登録)
  • **呼び出し先**:
  • ssl_hook_Access_modern / ssl_hook_Access_classic
  • ssl_var_lookup
  • ap_expr_exec
  • ap_log_rerror

関連データ構造

  • SSLDirConfigRec: ディレクトリ単位の SSL 設定。
  • SSLSrvConfigRec: サーバー単位の SSL 設定。
  • SSLConnRec: 接続単位の SSL 状態。
  • request_rec: Apache のリクエスト構造体。

失敗モードと境界条件

  • **SSL 必須設定違反**: bSSLRequired が真で SSL 接続がない場合、HTTP_FORBIDDEN を返し、r->notes に ssl-access-forbidden を記録する。
  • **式評価失敗**: SSLRequire の式評価がエラーまたは不一致の場合、HTTP_FORBIDDEN を返す。

保守担当者向けメモ

  • TLS 1.3 以降では再ネゴシエーションが廃止されているため、ssl_hook_Access_modern との分岐ロジックが重要。
  • r->user の設定は FakeBasicAuth の有無によりタイミングが制御されている点に注意。
  • 戻り値として OK ではなく DECLINED を返すことで、他の認証モジュールとの協調を維持している。
