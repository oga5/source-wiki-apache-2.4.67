socache_shmcb_init

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_shmcb.c:355

目的

共有メモリベースのキャッシュ（SHMCB）プロバイダーの初期化を行い、共有メモリセグメントの確保、ヘッダーの構築、およびサブキャッシュ領域の初期化を行う。

シグネチャまたは形状

apr_status_t socache_shmcb_init(ap_socache_instance_t *ctx,
                                const char *namespace,
                                const struct ap_socache_hints *hints,
                                server_rec *s, apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンスコンテキスト。
  • namespace: キャッシュ識別用名前空間。
  • hints: キャッシュサイズ最適化のためのヒント情報。
  • s: サーバーレコード。
  • p: メモリプール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • ctx->shm_size が確保済みであること。
  • 共有メモリセグメントは 5 * ALIGNED_HEADER_SIZE 以上のサイズが必要。
  • 内部のサブキャッシュ数は 256 から開始し、インデックス数に応じて調整される。

アルゴリズムまたは振る舞い

1. **メモリ確保**: apr_shm_create を使用し、匿名共有メモリまたは名前付き共有メモリを確保する。 2. **クリーンアップ登録**: socache_shmcb_cleanup をプールクリーンアップに登録する。 3. **レイアウト計算**:

  • hints に基づき、サブキャッシュ数と各サブキャッシュ内のインデックス数を決定する。
  • SHMCBHeader を共有メモリの先頭に配置し、統計情報をリセットする。
  • サブキャッシュのデータオフセットおよびサイズを計算し、アライメントを調整する。

    • 4. **初期化**: 全サブキャッシュのインデックスおよびデータポインタを 0 に初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_shmcb.c:1059（静的解析による参照）。
  • **呼び出し先**: apr_shm_create, apr_shm_remove, apr_shm_baseaddr_get, apr_shm_size_get, apr_pool_cleanup_register, ap_log_error 等の APR/Apache API。

関連データ構造

  • SHMCBHeader: 共有メモリのメタデータ管理。
  • SHMCBSubcache: 各サブキャッシュの管理構造。
  • ap_socache_instance_t: キャッシュインスタンスのコンテキスト。

失敗モードと境界条件

  • APR_ENOSPC: 共有メモリサイズが最小要件（ヘッダー分およびインデックス5個分）を満たさない場合に発生。
  • APR_EINVAL: 匿名共有メモリが利用できず、かつ名前付きファイルパスが不正な場合に発生。
  • apr_shm_create 失敗時: 共有メモリ確保に失敗した場合、エラーを返して終了する。

保守担当者向けメモ

  • 本関数はスレッドセーフではないため、呼び出し側で排他制御を行うこと。
  • num_idx や num_subcache の計算において unsigned int を使用しているため、極端に大きなメモリサイズを指定した際のオーバーフローリスクに注意が必要。
