set_service_description

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/service.c:268

目的

Windowsサービスとして実行されているApache HTTP Serverのサービス説明文（Description）を、サーバーの現在の記述内容（ap_get_server_description()）で更新する。

シグネチャまたは形状

static void set_service_description(void)

入力と出力

  • **入力**: グローバル変数 mpm_service_name（サービス名）、mpm_service_name_w（Unicode版サービス名）、および ap_get_server_description() が返すサーバー記述文字列。
  • **出力**: なし（副作用としてWindowsサービスコントロールマネージャ経由でレジストリ設定を更新）。

前提条件と不変条件

  • 実行環境がWindowsであること。
  • mpm_service_name が設定されていること（コンソールモード時は何もしない）。
  • ChangeServiceConfig2 APIが利用可能であること。

アルゴリズムまたは振る舞い

1. サービス名が未定義（コンソール実行）であれば即座に終了する。 2. OpenSCManager を使用してサービスコントロールマネージャへの接続を試みる。 3. OpenService（または OpenServiceW）で対象サービスを SERVICE_CHANGE_CONFIG 権限で開く。 4. OSの文字コード設定（Unicode/ANSI）に応じて、ChangeServiceConfig2（または ChangeServiceConfig2W）を呼び出し、サービスの説明文を更新する。

  • Unicode環境では、apr_conv_utf8_to_ucs2 を用いて文字列を変換してから設定する。

    • 5. 開いたサービスハンドルおよびSCMハンドルを CloseServiceHandle で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/service.c:732
  • server/mpm/winnt/service.c:923
  • **呼び出し先**:
  • ap_get_server_description
  • OpenSCManager
  • OpenService / OpenServiceW
  • ChangeServiceConfig2 / ChangeServiceConfig2W
  • CloseServiceHandle
  • apr_palloc, apr_conv_utf8_to_ucs2 (Unicode時)

関連データ構造

  • SC_HANDLE: Windowsサービスハンドル。
  • mpm_service_name / mpm_service_name_w: サービス名を保持するグローバル変数。

失敗モードと境界条件

  • SCMへの接続失敗やサービスオープン失敗時は、何も更新せず静かに終了する。
  • ChangeServiceConfig2 が利用できない環境では更新処理をスキップする。
  • Unicode変換エラーが発生した場合、更新は行われない。

保守担当者向けメモ

  • ChangeServiceConfig2 の利用はWindows 2000以降を前提としている。
  • Unicode変換には pconf プールを使用しているため、呼び出し時のメモリコンテキストに注意すること。
  • サービス説明文の更新は、サーバーの再起動のたびに行われる設計となっている。
