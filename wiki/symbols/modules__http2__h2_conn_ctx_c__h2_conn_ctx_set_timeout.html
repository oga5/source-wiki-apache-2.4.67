h2_conn_ctx_set_timeout

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_conn_ctx.c:112

目的

HTTP/2 接続コンテキストに関連付けられたデータ転送用リソース（バケットビームおよびパイプ）に対して、一括でタイムアウト値を設定します。

シグネチャまたは形状

void h2_conn_ctx_set_timeout(h2_conn_ctx_t *conn_ctx, apr_interval_time_t timeout)

入力と出力

  • **入力**:
  • conn_ctx: タイムアウトを適用する対象の h2_conn_ctx_t 構造体へのポインタ。
  • timeout: 設定するタイムアウト値（apr_interval_time_t 型）。
  • **出力**: なし（void）。

前提条件と不変条件

  • conn_ctx は有効なポインタである必要があります。
  • 内部の各リソース（beam_out, beam_in, pipe_in）は、設定前に存在（NULLでないこと）が確認されます。

アルゴリズムまたは振る舞い

1. conn_ctx->beam_out が存在する場合、h2_beam_timeout_set を呼び出し、出力用バケットビームに timeout を適用します。 2. conn_ctx->beam_in が存在する場合、h2_beam_timeout_set を呼び出し、入力用バケットビームに timeout を適用します。 3. conn_ctx->pipe_in[H2_PIPE_OUT] が存在する場合、apr_file_pipe_timeout_set を呼び出し、パイプに timeout を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2.c:549
  • modules/http2/h2_c2.c:774
  • **呼び出し先**:
  • h2_beam_timeout_set
  • apr_file_pipe_timeout_set

関連データ構造

  • h2_conn_ctx_t: 接続ごとの状態を保持する構造体。
  • h2_bucket_beam: スレッド間でバケットを安全に受け渡すための同期機構。

失敗モードと境界条件

  • conn_ctx 内の各リソースが初期化されていない（NULLである）場合、そのリソースに対するタイムアウト設定はスキップされます。
  • apr_file_pipe_timeout_set が失敗した場合の戻り値は無視されます。

保守担当者向けメモ

  • 本関数は、C2（ストリーム処理用接続）のライフサイクル管理において、接続のタイムアウト設定を同期させるために使用されます。
  • h2_conn_ctx.h にプロトタイプ宣言が存在します。
