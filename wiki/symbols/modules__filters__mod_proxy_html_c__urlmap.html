urlmap

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_proxy_html.c:94

目的

HTML コンテンツ内の URL や属性を書き換えるための置換ルールを保持するデータ構造です。

シグネチャまたは形状

typedef struct urlmap {
    struct urlmap *next;
    unsigned int flags;
    unsigned int regflags;
    union {
        const char *c;
        ap_regex_t *r;
    } from;
    const char *to;
    ap_expr_info_t *cond;
} urlmap;

入力と出力

  • 入力: ProxyHTMLURLMap ディレクティブ等により設定される置換元パターン（文字列または正規表現）、置換先文字列、および適用条件式。
  • 出力: 連結リストの次要素へのポインタ (next) を含む、評価可能なルールセット。

前提条件と不変条件

  • next ポインタにより、複数のルールが連結リストとして管理されます。
  • from 共用体は、flags に基づいて文字列 (c) またはコンパイル済み正規表現 (r) のいずれかとして解釈されます。
  • cond が設定されている場合、ルール適用時に ap_expr による評価が必要です。

アルゴリズムまたは振る舞い

  • リクエスト処理時に fixup_rules を介して、現在のリクエスト環境で有効なルールのみが抽出・評価されます。
  • HTML 解析中に pstartElement 等のコールバックから参照され、属性値が from パターンに一致する場合、to で指定された値へ置換されます。

呼び出し元と呼び出し先

  • 呼び出し元: mod_proxy_html.c 内の fixup_rules や、設定反映処理、および SAX コールバック関数群。
  • 呼び出し先: ap_regex_t (正規表現エンジン)、ap_expr_info_t (条件式評価エンジン)。
  • 注: 具体的な関数呼び出しのフローについては、ソースコード内の参照箇所（ctx->map 等）から推測される範囲に留まります。

関連データ構造

  • proxy_html_conf: モジュール設定全体を保持し、urlmap のリストを管理します。
  • saxctxt: 実行時のコンテキストであり、適用対象の urlmap リストを保持します。

失敗モードと境界条件

  • cond（条件式）の評価失敗時、そのルールは適用されません。
  • 正規表現のコンパイルエラーやメモリ不足時には、apr_pmemdup 等のメモリ管理機構に依存します。

保守担当者向けメモ

  • 本構造体は apr_pool_t 上で管理されるため、リクエスト終了時に自動的に解放されます。
  • from 共用体の切り替えは flags に依存するため、フラグ操作時の整合性に注意してください。
