h2_ping_state_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_proxy_session.h:63

目的

バックエンドサーバーとの HTTP/2 接続における、PING フレームを用いた生存確認および接続状態の管理を行うための列挙型です。

シグネチャまたは形状

typedef enum {
    H2_PING_ST_NONE,
    H2_PING_ST_AWAIT_ANY,
    H2_PING_ST_AWAIT_PING,
} h2_ping_state_t;

入力と出力

  • 本シンボル自体は状態値の定義であり、入力や出力は持ちません。
  • h2_proxy_session 構造体の ping_state メンバとして保持され、セッションの状態遷移に使用されます。

前提条件と不変条件

  • H2_PING_ST_NONE は通常動作モードを示し、ProxyTimeout ルールが適用されます。
  • 状態遷移は enter_ping_state 関数を介して管理されます。

アルゴリズムまたは振る舞い

  • H2_PING_ST_NONE: 通常のプロキシ通信状態。
  • H2_PING_ST_AWAIT_ANY: バックエンドからの何らかのフレーム受信を待機している状態。
  • H2_PING_ST_AWAIT_PING: バックエンドからの PING フレーム応答を待機している状態。
  • バックエンドが PING に応答しない場合、セッションはクローズされる設計となっています。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_proxy_session.c 内の enter_ping_state 関数が、引数としてこの型を使用しています。
  • 呼び出し先: 本シンボルは列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • h2_proxy_session: この構造体内で ping_state メンバとして使用されています。

失敗モードと境界条件

  • 状態遷移の不整合に関する情報は提供されていません。
  • タイムアウト発生時の挙動については、h2_proxy_session_process の処理フローに依存します。

保守担当者向けメモ

  • 本シンボルは mod_proxy_http2 におけるバックエンド接続の健全性維持に不可欠です。
  • 状態遷移のロジックを変更する場合は、h2_proxy_session.c の enter_ping_state および関連するイベントループ処理を確認してください。
