timer_event_t

定義

  • 種別: struct
  • 定義箇所: server/mpm_fdqueue.h:65

目的

Apache HTTP Server の MPM（特に event MPM）において、タイムアウト処理が必要なイベントを管理し、スレッド間で受け渡すための構造体です。

シグネチャまたは形状

struct timer_event_t {
    APR_RING_ENTRY(timer_event_t) link;
    apr_time_t when;
    ap_mpm_callback_fn_t *cbfunc;
    void *baton;
    int canceled;
    apr_array_header_t *remove;
};

入力と出力

  • **入力**: タイムアウト発生時刻 (when)、実行すべきコールバック関数 (cbfunc)、およびその引数 (baton) を保持します。
  • **出力**: タイムアウト到達時に cbfunc が呼び出され、イベント処理が実行されます。

前提条件と不変条件

  • APR_RING 構造の一部として管理されるため、リング操作の整合性が維持される必要があります。
  • canceled フラグが非ゼロの場合、イベントは無効として扱われます。

アルゴリズムまたは振る舞い

1. **登録**: push_timer2worker 等によりキューへ投入されます。 2. **待機**: 指定された時刻 (when) に到達するまで保持されます。 3. **実行**: 時刻到達時に cbfunc が baton を引数として実行されます。 4. **解放**: 処理完了後、または明示的なキャンセル時にリングから外され、メモリが再利用（timer_free_ring 等への移動）または解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/event/event.c 内の push_timer2worker 関数などが、本構造体を生成・操作します。
  • **呼び出し先**: cbfunc に登録された関数が、タイムアウト発生時に呼び出されます。
  • **注記**: 呼び出し関係の詳細は server/mpm/event/event.c に依存しており、本ヘッダーファイル単体では網羅的な呼び出し先リストは提供されていません。

関連データ構造

  • APR_RING: イベントをリスト管理するために使用。
  • ap_mpm_callback_fn_t: タイムアウト時に実行されるコールバック関数の型。
  • apr_array_header_t: 関連する削除対象リストを保持。

失敗モードと境界条件

  • canceled フラグがセットされている場合、コールバックは実行されません。
  • メモリ割り当て失敗時（apr_skiplist_alloc 等）の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本構造体は server/mpm_fdqueue.c および server/mpm/event/event.c で頻繁に参照されます。
  • APR_RING を使用しているため、リスト操作時には適切なロックまたは排他制御がなされていることを確認してください。
