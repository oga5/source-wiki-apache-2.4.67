md_store_save_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_store.c:113

目的

Managed Domain (MD) サブシステムにおいて、JSON形式のデータを指定されたストレージグループおよびアスペクトへ永続化するためのラッパー関数です。

シグネチャまたは形状

apr_status_t md_store_save_json(md_store_t *store, apr_pool_t *p, md_store_group_t group, 
                                const char *name, const char *aspect, 
                                struct md_json_t *data, int create);

入力と出力

  • **入力**:
  • store: ストレージ抽象化インターフェースへのポインタ。
  • p: メモリ割り当てに使用するAPRプール。
  • group: 保存先のストレージグループ（例: MD_SG_DOMAINS）。
  • name: 対象ドメイン名。
  • aspect: 保存するデータの種類（ファイル名の一部）。
  • data: 保存対象のJSONデータ構造体。
  • create: 存在しない場合に新規作成するかどうかを示すフラグ。
  • **出力**:
  • apr_status_t: 操作の成功（APR_SUCCESS）または失敗を示すステータスコード。

前提条件と不変条件

  • store は有効な md_store_t インスタンスである必要があります。
  • data は md_json_t として適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

内部で md_store_save 関数を呼び出し、データ形式を MD_SV_JSON として指定することで、ストレージバックエンドへのシリアライズと保存処理を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_order.c:214
  • modules/md/md_ocsp.c:272
  • modules/md/md_status.c:432
  • modules/md/md_store.c:214
  • **呼び出し先**:
  • md_store_save (同モジュール内)

関連データ構造

  • md_store_t: ストレージの抽象化ハンドル。
  • md_json_t: JSONデータを保持する構造体。
  • md_store_group_t: ストレージグループを定義する列挙型。

失敗モードと境界条件

  • ストレージバックエンド（ファイルシステム等）への書き込み権限がない場合や、ディスク容量不足などの理由で md_store_save がエラーを返した場合、そのステータスがそのまま伝播されます。
  • create フラグが偽で対象が存在しない場合、保存は失敗します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のI/Oロジックは md_store_save およびその先の具体的な実装（md_store_fs.c 等）に依存します。
  • JSONのシリアライズ形式を変更する場合は、md_store_save の引数 MD_SV_JSON の扱いを確認してください。
