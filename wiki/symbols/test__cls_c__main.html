定義

  • 種別: function
  • 定義箇所: test/cls.c:90

目的

指定されたディレクトリ内のキャッシュファイルを走査し、その内容を解析して標準出力に整形表示するコマンドラインユーティリティのエントリポイントです。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**: argv[1] にディレクトリパスを受け取ります。
  • **出力**: 解析したキャッシュファイルの情報を標準出力（stdout）に整形して出力します。エラー発生時は標準エラー出力（stderr）にメッセージを出力します。

前提条件と不変条件

  • 引数としてディレクトリパスが1つ指定されている必要があります。
  • キャッシュファイルは、8桁の16進数で構成されたヘッダー行と、X-URL: で始まる2行目を持つフォーマットである必要があります。
  • 隠しファイル（. または # で始まるファイル名）は無視されます。

アルゴリズムまたは振る舞い

1. 引数の数を検証し、ディレクトリを opendir で開きます。 2. ディレクトリ内の各ファイルを順次走査します。 3. 各ファイルを開き、1行目を fgets で読み込み、checkmask を用いてフォーマットの妥当性を検証します。 4. hex2sec を使用して、ヘッダー行から日付、最終更新日、有効期限、バージョン、長さを抽出します。 5. 2行目を読み込み、X-URL: で始まることを確認します。 6. 抽出したタイムスタンプを gmtime と strftime で人間が読みやすい形式に変換します。 7. 抽出した情報（URL、バージョン、日付等）を printf で出力します。 8. 最後にディレクトリを closedir で閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: OSのコマンドライン実行環境。
  • **呼び出し先**: opendir, readdir, fopen, fgets, checkmask, hex2sec, fclose, strlen, strncmp, gmtime, strftime, printf, fprintf, perror, exit。
  • **注記**: ツールによる静的解析では、本シンボルを呼び出している他のソースコードは特定されていません。

関連データ構造

  • DIR, struct dirent: ディレクトリ走査に使用。
  • FILE: ファイル操作に使用。
  • struct tm: 時刻変換に使用。

失敗モードと境界条件

  • 引数が不足している場合、使用法を表示して終了します。
  • ディレクトリが開けない場合、perror を出力して終了します。
  • ファイルが開けない場合、perror を出力して次のファイルへスキップします。
  • fgets で読み込めない場合、perror を出力してファイルを閉じ、次のファイルへスキップします。
  • checkmask または strncmp による検証に失敗した場合、Bad cache file を出力してスキップします。

保守担当者向けメモ

  • 本プログラムは特定のキャッシュファイルフォーマットに強く依存しています。
  • checkmask のマスク文字（& 等）は本ファイル独自の仕様です。
  • エラーハンドリングは標準エラー出力への出力とスキップ処理で構成されており、致命的なエラー以外はプログラム全体を停止させない設計になっています。
