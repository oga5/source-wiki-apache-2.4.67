scgi_config

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:70

目的

Apache HTTP Server の mod_proxy_scgi モジュールにおいて、ディレクトリ単位の設定（ProxySCGISendfile および ProxySCGIInternalRedirect）を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    const char *sendfile;
    const char *internal_redirect;
} scgi_config;

入力と出力

  • **入力**: 設定ファイル（httpd.conf 等）で指定されたディレクティブの値。
  • **出力**: モジュール内の各処理関数が、リクエスト処理時に参照する設定値。

前提条件と不変条件

  • すべてのメンバは request_rec のプール（r->pool）または設定用プールから割り当てられたメモリ領域に存在します。
  • sendfile および internal_redirect は、設定が存在しない場合は NULL となります。

アルゴリズムまたは振る舞い

  • 本構造体は、mod_proxy の設定フックを通じてディレクトリごとの設定として生成・マージされます。
  • リクエスト処理中、scgi_request_status などの関数がこの構造体を参照し、バックエンドからのレスポンスに対して X-Sendfile や内部リダイレクトの処理を行うべきかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_scgi.c 内の設定生成関数（apr_palloc を使用してインスタンス化）。
  • リクエスト処理関数（scgi_request_status 等）が設定値の読み取りのために参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • scgi_request_config: リクエスト処理中に使用される、より動的な状態保持構造体。

失敗モードと境界条件

  • 設定が未定義の場合、メンバは NULL となるため、参照側で NULL チェックが必要です。
  • メモリ割り当て失敗時は apr_palloc が例外を発生させる可能性があります。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を保持するため、ProxySCGISendfile や ProxySCGIInternalRedirect ディレクティブの追加・変更時には本構造体へのメンバ追加を検討してください。
  • 参照箇所は modules/proxy/mod_proxy_scgi.c 内の 368, 597, 608, 621, 639 行付近に存在します。
