socache_dc_init

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dc.c:55

目的

分散キャッシュ（distcache）プロバイダーのクライアントコンテキストを初期化し、キャッシュ操作のための接続準備を行う。

シグネチャまたは形状

apr_status_t socache_dc_init(ap_socache_instance_t *ctx,
                             const char *namespace,
                             const struct ap_socache_hints *hints,
                             server_rec *s, apr_pool_t *p);

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンス構造体（target 接続先情報を含む）。
  • namespace: キャッシュの名前空間（本関数内では未使用）。
  • hints: キャッシュのヒント情報（本関数内では未使用）。
  • s: サーバーレコード（ログ出力に使用）。
  • p: メモリプール。
  • **出力**:
  • APR_SUCCESS: 初期化成功。
  • APR_EGENERAL: コンテキスト生成失敗。

前提条件と不変条件

  • ctx->target が有効な接続先文字列として設定されている必要がある。
  • distcache ライブラリが利用可能であること。

アルゴリズムまたは振る舞い

1. SESSION_CTX_FLAGS を 0 に設定し、操作ごとに一時的な接続を行うモードで動作する（fork() 後の安全性を確保するため）。 2. DC_CTX_new を呼び出し、ctx->target を用いて分散キャッシュのコンテキストを生成し、ctx->dc に格納する。 3. コンテキスト生成に失敗した場合は、エラーログを出力して APR_EGENERAL を返す。 4. 成功した場合は、情報ログを出力して APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dc.c:177（直接の呼び出し元として確認）。
  • **呼び出し先**: DC_CTX_new（distcache ライブラリ関数）、ap_log_error（Apache ロギング関数）。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスを保持する構造体。dc メンバに生成されたコンテキストが格納される。

失敗モードと境界条件

  • DC_CTX_new が NULL を返した場合、分散キャッシュのコンテキスト取得失敗として APR_EGENERAL を返す。
  • 永続接続モード（SESSION_CTX_FLAG_PERSISTENT 等）は現在無効化されており、接続のオーバーヘッドが発生する可能性がある。

保守担当者向けメモ

  • 本関数は fork() 後のファイルディスクリプタ共有問題を回避するため、一時接続モードで動作するよう設計されている。
  • 永続接続モードを有効化する場合は、SESSION_CTX_FLAG_PERSISTENT_PIDCHECK を併用し、プロセス間での通信競合を防ぐ必要がある。
