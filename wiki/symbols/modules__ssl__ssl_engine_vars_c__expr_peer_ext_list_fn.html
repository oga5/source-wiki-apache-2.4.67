expr_peer_ext_list_fn

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:78

目的

Apache 設定式（ap_expr）評価コンテキストにおいて、ピア証明書の特定の拡張フィールドをリストとして取得するためのラッパー関数です。

シグネチャまたは形状

static apr_array_header_t *expr_peer_ext_list_fn(ap_expr_eval_ctx_t *ctx,
                                                 const void *dummy,
                                                 const char *arg)

入力と出力

  • **入力**:
  • ctx: ap_expr_eval_ctx_t 構造体へのポインタ。評価コンテキストおよびメモリプール（ctx->p）、接続情報（ctx->c）を提供。
  • dummy: 未使用。
  • arg: 取得対象の拡張フィールド名を指定する文字列。
  • **出力**:
  • 成功時: 拡張フィールドの値を含む apr_array_header_t 構造体へのポインタ。
  • 失敗時: NULL または空の配列。

前提条件と不変条件

  • ctx および ctx->c が有効であること。
  • 内部で呼び出される ssl_ext_list が、メモリ確保のために ctx->p（apr_pool_t）を使用するため、呼び出し元で適切なプール生存期間が保証されていること。

アルゴリズムまたは振る舞い

1. 渡された評価コンテキスト ctx からメモリプールと接続情報を抽出します。 2. ssl_ext_list 関数を呼び出し、ピア証明書の拡張リストを取得します。この際、第3引数に 1 を指定することで、ピア証明書を対象とした検索であることを明示します。 3. ssl_ext_list から返された結果をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_vars.c:126（ssl_var_register 内で関数ポインタとして登録）。
  • **呼び出し先**: ssl_ext_list（証明書拡張抽出のメインロジック）。

関連データ構造

  • ap_expr_eval_ctx_t: Apache 式評価コンテキスト。
  • apr_array_header_t: APR の配列構造体。
  • SSLConnRec: 接続ごとの SSL 状態を保持する構造体（ctx->c を介して間接的に利用）。

失敗モードと境界条件

  • ピア証明書が存在しない場合や、指定された拡張フィールド名が証明書内に存在しない場合、ssl_ext_list の挙動に従い NULL または空の配列が返されます。
  • メモリ不足時には apr_pool_t の割り当て失敗によりプロセスが異常終了する可能性があります。

保守担当者向けメモ

  • 本関数は ap_expr インターフェースと ssl_ext_list を繋ぐアダプターです。ロジックの変更が必要な場合は、ssl_ext_list 側の実装を確認してください。
