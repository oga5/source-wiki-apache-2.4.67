rfc1413_connect

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_ident.c:87

目的

RFC 1413 (IDENT) プロトコルに基づき、リモートホストの 113 番ポートに対して接続を確立します。

シグネチャまたは形状

apr_status_t rfc1413_connect(apr_socket_t **newsock, conn_rec *conn,
                             server_rec *srv, apr_time_t timeout)

入力と出力

  • **入力**:
  • newsock: 接続用ソケットを格納するポインタのポインタ。
  • conn: 現在の接続情報 (conn_rec)。ローカルおよびリモートの IP アドレス取得に使用。
  • srv: ログ出力に使用するサーバー情報 (server_rec)。
  • timeout: 接続および通信に適用するタイムアウト時間。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時はエラーコードを返す。
  • newsock: 成功時に接続済みソケットが格納される。

前提条件と不変条件

  • conn->local_ip および conn->client_ip が有効な数値形式の IP アドレス文字列であること。
  • 接続に使用するソケットは conn->pool を使用して割り当てられるため、リクエスト終了時に自動的にクリーンアップされる。

アルゴリズムまたは振る舞い

1. apr_sockaddr_info_get を使用し、ローカルおよびリモートの IP アドレスからソケットアドレス構造体を作成する。 2. apr_socket_create でストリームソケットを作成する。 3. apr_socket_timeout_set で指定されたタイムアウトを設定する。 4. apr_socket_bind を呼び出し、ローカル側を接続元の IP アドレスにバインドする（マルチホーム環境での正しい識別のため）。 5. apr_socket_connect を呼び出し、リモートの 113 番ポートへ接続を試みる。 6. 接続失敗時はソケットをクローズしてエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_ident.c:318 (推論される直接の呼び出し箇所)
  • **呼び出し先**: APR (Apache Portable Runtime) のネットワーク I/O 関数群 (apr_sockaddr_info_get, apr_socket_create, apr_socket_timeout_set, apr_socket_bind, apr_socket_connect) およびログ出力関数 (ap_log_error)。

関連データ構造

  • conn_rec: 接続情報を保持。
  • server_rec: サーバー設定およびログ出力用。
  • apr_socket_t: 通信に使用するソケット。
  • apr_sockaddr_t: IP アドレスとポート情報を保持する構造体。

失敗モードと境界条件

  • apr_sockaddr_info_get 失敗: IP アドレスの解決や構造体作成に失敗した場合、エラーログを出力して終了する。
  • apr_socket_create 失敗: ソケット作成不可時にエラーログを出力して終了する。
  • apr_socket_connect 失敗: リモートホストが IDENT サービスを提供していない場合、エラーを返すが、この場合はログを出力せずに終了する（仕様上の挙動）。

保守担当者向けメモ

  • 本関数は、接続元の IP アドレスとポートを維持したまま IDENT サーバーへ接続するために、明示的な bind を行っています。
  • タイムアウト設定は IdentityCheckTimeout ディレクティブによって制御されます。
  • 接続失敗時のログ出力は、サービスが提供されていない可能性を考慮し、意図的に抑制されています。
