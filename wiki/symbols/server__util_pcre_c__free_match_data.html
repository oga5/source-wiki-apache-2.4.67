free_match_data

定義

  • 種別: function
  • 定義箇所: server/util_pcre.c:304

目的

正規表現マッチングに使用されたメモリ領域（PCRE2 の match_data または PCRE1 のオフセットベクトル）を解放する。

シグネチャまたは形状

void free_match_data(match_data_pt data, apr_size_t size)

入力と出力

  • data: 解放対象となるマッチングデータ構造体へのポインタ。
  • size: PCRE1 使用時にメモリ解放の判断基準となるサイズ（apr_size_t）。
  • 戻り値: なし。

前提条件と不変条件

  • HAVE_PCRE2 が定義されている場合、data は pcre2_match_data 型として扱われる。
  • HAVE_PCRE2 が定義されていない場合、size が POSIX_MALLOC_THRESHOLD を超える場合にのみ free() が実行される。

アルゴリズムまたは振る舞い

1. HAVE_PCRE2 が定義されている場合:

  • pcre2_match_data_free(data) を呼び出し、PCRE2 のマッチングデータ構造を解放する。

    • 2. HAVE_PCRE2 が定義されていない場合:

  • size が POSIX_MALLOC_THRESHOLD より大きいかを確認する。
  • 条件を満たす場合、標準の free(data) を実行してメモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: server/util_pcre.c 内の関数（行 449 および 455 での呼び出しを確認）。
  • 呼び出し先:
  • HAVE_PCRE2 定義時: pcre2_match_data_free
  • HAVE_PCRE2 未定義時: free

関連データ構造

  • match_data_pt: マッチングデータを保持するポインタ型。
  • POSIX_MALLOC_THRESHOLD: PCRE1 使用時にスタック上のバッファではなくヒープメモリを解放するかを決定する閾値。

失敗モードと境界条件

  • data が NULL の場合、PCRE2 の pcre2_match_data_free は安全に処理されるが、PCRE1 の free は呼び出し元で NULL チェックが必要となる可能性がある。

保守担当者向けメモ

  • 本関数は PCRE1 と PCRE2 の抽象化レイヤーの一部である。PCRE2 への移行に伴い、メモリ管理の責務が pcre2_match_data_free に委譲されている点に注意すること。
