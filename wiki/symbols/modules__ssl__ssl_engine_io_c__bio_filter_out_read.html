bio_filter_out_read

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_io.c:193

目的

OpenSSL の BIO_METHOD 構造体において読み取りメソッドとして登録される関数ですが、本実装では呼び出されるべきではないエラーハンドリング用のプレースホルダーとして機能します。

シグネチャまたは形状

static int bio_filter_out_read(BIO *bio, char *out, int outl)

入力と出力

  • **入力**:
  • BIO *bio: 対象の BIO オブジェクト。
  • char *out: 読み取り先バッファ（使用されません）。
  • int outl: 読み取り要求サイズ（使用されません）。
  • **出力**:
  • 常に -1 を返します。

前提条件と不変条件

  • 本関数は、出力用 BIO (bio_filter_out_method) に対して BIO_meth_set_read で登録されています。
  • 実行時に呼び出された場合、システムは異常状態とみなします。

アルゴリズムまたは振る舞い

1. BIO_get_data(bio) を介して bio_filter_out_ctx_t を取得します。 2. ap_log_cerror を使用して、本関数が呼び出されたことを示すエラーログ（TRACE1 レベル）を出力します。 3. AP_DEBUG_ASSERT(0) により、デバッグビルド環境ではアサーション失敗を発生させます。 4. -1 を返して処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上の明示的な呼び出しはありません。OpenSSL の BIO メソッドテーブルに登録されていますが、設計上呼び出されないことが前提です。
  • **呼び出し先**:
  • BIO_get_data (OpenSSL API)
  • ap_log_cerror (HTTPD API)

関連データ構造

  • bio_filter_out_ctx_t: BIO に関連付けられたコンテキスト構造体。

失敗モードと境界条件

  • 本関数自体が、呼び出された時点で「失敗」と定義されています。

保守担当者向けメモ

  • 本関数は bio_filter_out_method の初期化時に BIO_meth_set_read へ渡されますが、コメントにある通り「決して呼び出されない」ことが期待されています。
  • ログ出力やアサーションは、意図しないコードパスが実行された際のデバッグを目的としています。
