dumpio_register_hooks

定義

  • 種別: function
  • 定義箇所: modules/debugging/mod_dumpio.c:199

目的

Apache HTTP Server のフィルタチェーンに対して、入出力データの傍受用フィルタ（DUMPIO_IN, DUMPIO_OUT）を登録し、接続開始時のフックを設定する。

シグネチャまたは形状

static void dumpio_register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（フック登録に使用されるメモリプール）。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数はモジュールのロード時に呼び出されることを前提とする。
  • フィルタの挿入位置は AP_FTYPE_CONNECTION + 3 に固定されており、これは SSL フィルタ（CONNECTION + 5）よりも内側で動作するよう設計されている。

アルゴリズムまたは振る舞い

1. ap_register_output_filter を呼び出し、dumpio_output_filter を DUMPIO_OUT として登録する。 2. ap_register_input_filter を呼び出し、dumpio_input_filter を DUMPIO_IN として登録する。 3. ap_hook_pre_connection を呼び出し、接続開始時に dumpio_pre_conn が実行されるよう登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述（modules/debugging/mod_dumpio.c:254）より、モジュールのフック登録プロセスから呼び出される。
  • **呼び出し先**:
  • ap_register_output_filter
  • ap_register_input_filter
  • ap_hook_pre_connection

関連データ構造

  • dumpio_output_filter（フィルタ関数）
  • dumpio_input_filter（フィルタ関数）
  • dumpio_pre_conn（フック関数）

失敗モードと境界条件

  • フィルタ登録関数が失敗した場合の戻り値チェックは本関数内には存在しない。
  • フィルタの挿入位置（AP_FTYPE_CONNECTION + 3）はハードコードされており、他のモジュールとの競合や順序依存性に注意が必要である。

保守担当者向けメモ

  • 本関数は mod_dumpio の初期化の要である。フィルタの挿入順序を変更する場合は、AP_FTYPE_CONNECTION のオフセット値を慎重に調整すること。
