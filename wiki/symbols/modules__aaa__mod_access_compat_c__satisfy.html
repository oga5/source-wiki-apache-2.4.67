satisfy

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_access_compat.c:118

目的

Apache 設定ファイル内の Satisfy ディレクティブを解析し、ディレクトリごとのアクセス制御ポリシー（all または any）を設定構造体に格納する。

シグネチャまたは形状

static const char *satisfy(cmd_parms *cmd, void *dv, const char *arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。適用対象のメソッドビット（cmd->limited）を含む。
  • dv: 現在のディレクトリ設定構造体 (access_compat_dir_conf)。
  • arg: 設定値文字列（"all" または "any"）。
  • **出力**:
  • 成功時: NULL。
  • 失敗時: エラーメッセージ文字列（const char *）。

前提条件と不変条件

  • dv は access_compat_dir_conf 型としてキャスト可能であること。
  • cmd->limited に基づき、対象となる HTTP メソッドに対してのみ設定が適用される。

アルゴリズムまたは振る舞い

1. arg を大文字小文字を区別せずに比較し、SATISFY_ALL または SATISFY_ANY を決定する。 2. 引数が "all" でも "any" でもない場合、エラーメッセージを返して終了する。 3. cmd->limited で指定されたメソッドビットを走査する。 4. 対応するビットが立っているメソッドについて、d->satisfy[i] に決定したポリシーを代入する。 5. 正常終了時に NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定解析エンジン（AP_INIT_TAKE1 マクロ経由で Satisfy ディレクティブが検出された際に呼び出される）。
  • **呼び出し先**: 外部関数への明示的な呼び出しはなし。

関連データ構造

  • access_compat_dir_conf: 設定を保持する構造体。satisfy 配列メンバを持つ。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • **無効な引数**: "all" または "any" 以外の文字列が渡された場合、エラーメッセージを返し設定は更新されない。
  • **メソッド制限**: cmd->limited が特定のメソッドのみを対象としている場合、そのメソッドに対してのみポリシーが適用される。

保守担当者向けメモ

  • 本関数は mod_access_compat における設定フェーズのハンドラである。
  • d->satisfy 配列は METHODS 定数に基づいたサイズで確保されている必要がある（create_access_compat_dir_config で初期化済みであること）。
