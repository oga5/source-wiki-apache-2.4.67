user_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_user.c:48

目的

require user ディレクティブに基づき、認証済みユーザーがリソースへのアクセス権を持っているかを判定します。

シグネチャまたは形状

static authz_status user_check_authorization(request_rec *r,
                                             const char *require_args,
                                             const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • require_args: 未使用。
  • parsed_require_args: user_parse_config によってパースされた ap_expr_info_t 形式の式情報。
  • **出力**:
  • AUTHZ_GRANTED: ユーザーが許可されたリストに含まれる場合。
  • AUTHZ_DENIED: ユーザーが許可されていない、または式評価に失敗した場合。
  • AUTHZ_DENIED_NO_USER: リクエストにユーザー名（r->user）が設定されていない場合。

前提条件と不変条件

  • r->user が設定されていること（認証フェーズが完了していること）。
  • parsed_require_args は ap_expr_info_t 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. r->user が存在するか確認し、なければ AUTHZ_DENIED_NO_USER を返します。 2. ap_expr_str_exec を使用して parsed_require_args を評価し、許可対象のユーザー名リスト（文字列）を取得します。 3. 評価中にエラーが発生した場合はログを出力し、AUTHZ_DENIED を返します。 4. 取得した文字列を ap_getword_conf で分割し、各トークンと r->user を strcmp で比較します。 5. 一致するユーザー名が見つかれば AUTHZ_GRANTED を返し、最後まで一致しなければ AUTHZ_DENIED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_authz_core（modules/aaa/mod_authz_user.c:117 にてプロバイダー構造体経由で参照）。
  • **呼び出し先**:
  • ap_expr_str_exec: 式の評価。
  • ap_log_rerror: エラーおよびデバッグログの出力。
  • ap_getword_conf: 設定文字列のトークン分割。
  • strcmp: 文字列比較。

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • ap_expr_info_t: require 行の式評価用構造体。
  • authz_status: 認可結果の列挙型。

失敗モードと境界条件

  • **式評価失敗**: ap_expr_str_exec がエラーを返した場合、ログを出力して AUTHZ_DENIED となります。
  • **ユーザー不一致**: 許可リストに該当ユーザーが存在しない場合、デバッグログを出力して AUTHZ_DENIED となります。

保守担当者向けメモ

  • 本関数は mod_authz_core のプロバイダーとして登録されています。
  • require user の引数は式評価をサポートしているため、静的なユーザー名だけでなく、動的な変数を含む式も処理可能です。
  • 認証フェーズが完了していない場合に AUTHZ_DENIED_NO_USER を返すことで、他の認証モジュールへのフォールバックを可能にしています。
