authz_dbd_cfg

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authz_dbd.c:47

目的

mod_authz_dbd モジュールにおいて、ディレクトリ単位のデータベース認可設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const char *query;
    const char *redir_query;
    int redirect;
} authz_dbd_cfg;

入力と出力

  • **入力**: 設定ファイル（httpd.conf 等）のディレクティブから読み込まれた値。
  • **出力**: 認可プロバイダー（dbd-group, dbd-login, dbd-logout）がデータベース操作を行う際のパラメータとして利用されます。

前提条件と不変条件

  • 設定値は request_rec のプール（r->pool）を使用してメモリ管理されます。
  • query および redir_query は、mod_dbd を介してプリペアドステートメントとして登録される必要があります。

アルゴリズムまたは振る舞い

  • ディレクトリごとの設定を保持し、authz_dbd_merge_cfg 等を通じて親ディレクトリからの設定継承をサポートします。
  • redirect フラグは、ログイン成功時のリダイレクト処理の有効・無効を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • authz_dbd_cr_cfg（初期化）
  • authz_dbd_merge_cfg（マージ）
  • ap_get_module_config（リクエスト処理時の設定取得）
  • **呼び出し先**:
  • authz_dbd_login
  • authz_dbd_group_query

関連データ構造

  • request_rec: リクエストごとのコンテキスト保持に使用。
  • mod_dbd のプリペアドステートメント: query および redir_query が参照するデータベースクエリ。

失敗モードと境界条件

  • 設定が未定義の状態で認可プロバイダーが呼び出された場合、HTTP_INTERNAL_SERVER_ERROR が発生する可能性があります。
  • .htaccess 内でのクエリ定義は NOT_IN_HTACCESS 制約により許可されません。

保守担当者向けメモ

  • 本構造体の各メンバは APR_OFFSETOF を使用して設定ディレクティブとマッピングされています。変更時は mod_authz_dbd.c 内のコマンドテーブル定義との整合性に注意してください。
