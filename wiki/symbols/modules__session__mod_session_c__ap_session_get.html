ap_session_get

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:267

目的

セッションデータから指定されたキーに対応する値を取得します。

シグネチャまたは形状

apr_status_t ap_session_get(request_rec *r, session_rec *z, const char *key, const char **value)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • z: セッション構造体（NULL の場合は自動的にロードを試みる）。
  • key: 取得対象のセッションキー。
  • value: 取得した値を格納するポインタのポインタ。
  • **出力**:
  • apr_status_t: 成功時は OK、ロード失敗時は ap_session_load が返すエラーコード。
  • value: キーが見つかった場合、セッション内の値が設定される。

前提条件と不変条件

  • z が NULL の場合、ap_session_load を介してセッションのロードが行われます。
  • セッションデータは session_rec 構造体の entries（apr_table_t）に保持されている必要があります。

アルゴリズムまたは振る舞い

1. z が NULL の場合、ap_session_load(r, &z) を呼び出し、セッションのロードを試みます。ロードに失敗した場合はそのエラーを返します。 2. z が有効であり、かつ z->entries が存在する場合、apr_table_get を使用して指定された key に対応する値を検索します。 3. 検索結果を value に格納し、OK を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_form.c にて APR_RETRIEVE_OPTIONAL_FN を通じて利用されています。
  • **呼び出し先**:
  • ap_session_load（z が NULL の場合）
  • apr_table_get（APRライブラリ）

関連データ構造

  • session_rec: セッション情報を保持する構造体。
  • request_rec: Apache HTTP Server のリクエスト構造体。
  • apr_table_t: セッションのキー・値ペアを格納するテーブル。

失敗モードと境界条件

  • ap_session_load が失敗した場合、そのエラーコードがそのまま返されます。
  • 指定した key が存在しない場合、value には NULL が設定されます。
  • z が NULL であり、かつ ap_session_load も失敗した場合、値の取得は行われません。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN により公開されており、他のモジュールから動的に呼び出し可能です。
  • session_rec のメモリはリクエストプール（r->pool）で管理されているため、本関数内で明示的なメモリ解放は不要です。
