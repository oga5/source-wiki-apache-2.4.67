bucket_read

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_headers.c:49

目的

H2HEADERS 型バケットに対する読み取り操作を無効化し、メタデータ専用バケットとしての特性を維持します。

シグネチャまたは形状

apr_status_t bucket_read(apr_bucket *b, const char **str, apr_size_t *len, apr_read_type_e block)

入力と出力

  • **入力**:
  • b: 読み取り対象の apr_bucket。
  • block: 読み取りのブロッキングモード（本関数では使用されません）。
  • **出力**:
  • str: NULL が代入されます。
  • len: 0 が代入されます。
  • **戻り値**: 常に APR_SUCCESS を返します。

前提条件と不変条件

  • 本関数は H2HEADERS 型バケットの read メソッドとして登録されることを前提としています。
  • バケットの内容を直接読み取ることはできず、常に空のデータとして扱われます。

アルゴリズムまたは振る舞い

1. 引数 b および block を明示的に無視（キャストによる破棄）します。 2. 出力ポインタ str に NULL を設定します。 3. 出力サイズ len に 0 を設定します。 4. APR_SUCCESS を返し、読み取り操作が正常に「データなし」として完了したことを伝えます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_headers.c:105（バケット型定義の初期化時）
  • modules/http2/h2_bucket_eos.c:107（ツールによる直接的な参照の可能性あり）
  • **呼び出し先**: なし。

関連データ構造

  • apr_bucket: Apache Portable Runtime のバケット構造体。
  • h2_bucket_headers: 本関数が関連付けられている H2HEADERS 型バケットのラッパー構造体。

失敗モードと境界条件

  • 失敗モードはありません。常に成功を返します。
  • 呼び出し側が str や len を介してバケット内のデータにアクセスしようとした場合、NULL および 0 が返されるため、呼び出し側での適切なチェックが必要です。

保守担当者向けメモ

  • 本関数は意図的に空の実装となっています。これは H2HEADERS バケットが HTTP/2 ヘッダーメタデータを保持するための専用コンテナであり、ストリームのデータ本体（ペイロード）を保持するものではないためです。
  • 今後、バケットの仕様変更によりデータ読み取りが必要になった場合を除き、この実装を変更する必要はありません。
