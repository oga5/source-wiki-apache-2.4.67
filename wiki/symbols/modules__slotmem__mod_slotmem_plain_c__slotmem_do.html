slotmem_do

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:41

目的

指定されたスロットメモリインスタンス内の全スロットを走査し、各スロットに対してコールバック関数を適用する。

シグネチャまたは形状

apr_status_t slotmem_do(ap_slotmem_instance_t *mem, ap_slotmem_callback_fn_t *func, void *data, apr_pool_t *pool)

入力と出力

  • **入力**:
  • mem: 操作対象の ap_slotmem_instance_t 構造体へのポインタ。
  • func: 各スロットに対して実行する ap_slotmem_callback_fn_t 型のコールバック関数。
  • data: コールバック関数に渡す任意のデータポインタ。
  • pool: コールバック関数内で使用可能なメモリプール。
  • **出力**:
  • APR_SUCCESS: 正常終了。
  • APR_ENOSHMAVAIL: mem が NULL の場合。
  • その他: コールバック関数 func が返したステータスコード。

前提条件と不変条件

  • mem が有効なインスタンスを指していること。
  • func は、スロットデータへのポインタ、data、pool を引数に取る関数であること。
  • 本関数自体には排他制御が含まれないため、マルチスレッド環境では呼び出し側で同期が必要。

アルゴリズムまたは振る舞い

1. mem が NULL であれば APR_ENOSHMAVAIL を返す。 2. インスタンス内の全スロットを順次走査する。 3. AP_SLOTMEM_IS_PREGRAB(mem) が真の場合、inuse フラグが立っているスロットのみを対象とする。 4. 条件を満たす各スロットに対し、そのデータ領域へのポインタを引数として func を呼び出す。 5. func の戻り値が APR_SUCCESS 以外であれば、走査を中断しその値を返す。 6. 全スロットの走査完了後、最後に実行された func の結果（または APR_SUCCESS）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/slotmem/mod_slotmem_plain.c:305 にて、プロバイダー構造体の一部として参照されている。
  • **呼び出し先**: 引数として渡された func（コールバック関数）。

関連データ構造

  • ap_slotmem_instance_t: スロットメモリの管理構造体。
  • ap_slotmem_callback_fn_t: 走査時に実行されるコールバック関数の型定義。

失敗モードと境界条件

  • mem が NULL の場合、即座にエラーを返す。
  • コールバック関数内でエラーが発生した場合、走査は即座に停止する。
  • スロット数が 0 の場合、ループは実行されず APR_SUCCESS を返す。

保守担当者向けメモ

  • 本関数は mod_slotmem_plain.c 内のプロバイダー実装の一部であり、メモリの直接操作を行う。
  • memcpy 等の境界チェックは呼び出し側で行う必要があるため、コールバック関数内でのデータアクセスには注意すること。
