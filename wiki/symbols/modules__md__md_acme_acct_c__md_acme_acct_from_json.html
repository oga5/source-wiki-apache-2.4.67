md_acme_acct_from_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_acct.c:123

目的

JSON 形式で保存された ACME アカウントデータを解析し、md_acme_acct_t 構造体へ復元します。

シグネチャまたは形状

apr_status_t md_acme_acct_from_json(md_acme_acct_t **pacct, md_json_t *json, apr_pool_t *p)

入力と出力

  • **入力**:
  • json: 解析対象の md_json_t オブジェクト。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • pacct: 成功時に生成された md_acme_acct_t 構造体へのポインタが格納されます。
  • 戻り値: 成功時は APR_SUCCESS、失敗時はエラーコード（APR_EINVAL 等）を返します。

前提条件と不変条件

  • 入力された json オブジェクト内に、ACME アカウントを特定するための url および ca_url キーが必須として存在する必要があります。
  • 成功した場合、pacct にはメモリプール p 上で確保された有効なアカウント構造体が返されます。

アルゴリズムまたは振る舞い

1. **ステータス取得**: JSON 内の status キーからアカウント状態を読み取ります。 2. **必須フィールド検証**: url および ca_url の存在を確認し、欠落している場合はエラーとして終了します。 3. **連絡先情報の抽出**: contact キー、または registration 内の contact キーから連絡先配列を構築します。 4. **構造体生成**: acct_make を呼び出し、アカウント構造体を初期化します。 5. **属性の割り当て**: 取得した url、agreement（ToS）、orders、および EAB（External Account Binding）情報を構造体に設定します。

  • agreement は terms-of-service キーによる後方互換性チェックを含みます。
  • EAB 情報（kid, hmac）は存在する場合のみ設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_acct.c 内の複数箇所（ソースコード上の直接的な参照を確認）。
  • **呼び出し先**:
  • acct_st_from_str
  • acct_make
  • md_json_gets / md_json_has_key / md_json_getsa
  • md_log_perror

関連データ構造

  • md_acme_acct_t: アカウント情報を保持する主要構造体。
  • md_json_t: JSON データの抽象化構造体。

失敗モードと境界条件

  • url または ca_url が JSON 内に見つからない場合、APR_EINVAL を返し、ログにデバッグ情報を出力して終了します。
  • acct_make の内部処理でメモリ確保や初期化に失敗した場合、そのエラーコードが伝播されます。

保守担当者向けメモ

  • 本関数は md_store から読み込まれた JSON データをオブジェクト化する役割を担います。
  • EAB 設定の有無は kid と hmac の両方のキー存在によって判定されます。
  • 構造体のライフサイクルは引数として渡される apr_pool_t に依存するため、呼び出し元でのプール管理に注意してください。
