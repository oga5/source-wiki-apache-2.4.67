ssl_mutex_on

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_mutex.c:86

目的

セッションキャッシュへのアクセスを保護するため、グローバル Mutex をロックする。

シグネチャまたは形状

int ssl_mutex_on(server_rec *s)

入力と出力

  • **入力**: server_rec *s (現在のサーバー設定を参照するためのサーバーレコード)
  • **出力**: int 型。ロック取得に成功した場合は TRUE、失敗した場合は FALSE を返す。

前提条件と不変条件

  • myModConfig(s) を通じて取得される SSLModConfigRec 構造体内の pMutex が、ssl_mutex_init によって適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. myModConfig(s) を呼び出し、サーバー設定からモジュール設定構造体 mc を取得する。 2. apr_global_mutex_lock(mc->pMutex) を実行し、グローバル Mutex のロックを試みる。 3. ロック取得に成功した場合、TRUE を返す。 4. ロック取得に失敗した場合、ap_log_error を使用して警告ログ（APLOGNO(02026)）を出力し、FALSE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_scache.c: セッションキャッシュの読み書き処理（ssl_scache_store, ssl_scache_retrieve, ssl_scache_remove, ssl_scache_expire 等）の直前で呼び出される。
  • **呼び出し先**:
  • apr_global_mutex_lock (APR API)
  • ap_log_error (HTTPD Core API)

関連データ構造

  • SSLModConfigRec: モジュール全体の設定を保持する構造体。pMutex メンバを介してロック対象の Mutex を管理する。
  • server_rec: Apache HTTP Server のサーバー設定構造体。

失敗モードと境界条件

  • apr_global_mutex_lock が APR_SUCCESS 以外の値を返した場合、ロック取得失敗として処理される。これは主に OS のリソース制限や Mutex の状態異常に起因する。

保守担当者向けメモ

  • 本関数は ssl_mutex_off と対で使用される。ロック取得に失敗した場合は FALSE が返されるため、呼び出し側で適切にエラーハンドリングを行う必要がある。
