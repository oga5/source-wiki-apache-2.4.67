wctx_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_hcheck.c:63

目的

個別のヘルスチェック対象ワーカーに関する実行時コンテキストを保持する構造体です。ヘルスチェックのリクエスト内容や、対象となる proxy_worker への参照を管理します。

シグネチャまたは形状

typedef struct {
    const char *path;     /* 元のワーカーURLのパス */
    const char *method;   /* HTTP/AJPリクエストのメソッド */
    const char *req;      /* 事前にフォーマットされたHTTP/AJPリクエスト */
    proxy_worker *w;      /* 対象ワーカーへのポインタ */
    const char *protocol; /* HTTP 1.0 または 1.1 */
} wctx_t;

入力と出力

  • **入力**: create_hcheck_req 関数等において、設定値やワーカー情報から本構造体の各フィールドが初期化されます。
  • **出力**: ヘルスチェック実行時、本構造体に格納されたリクエスト情報（req）やプロトコル情報がバックエンドへの通信に使用されます。

前提条件と不変条件

  • w フィールドは、有効な proxy_worker を指している必要があります。
  • req フィールドは、ヘルスチェック実行前に create_hcheck_req 等によって適切にフォーマットされていることが想定されます。

アルゴリズムまたは振る舞い

  • 本構造体は、mod_proxy_hcheck がバックエンドサーバーに対して定期的に送信するヘルスチェックリクエストの「テンプレート」および「実行パラメータ」として機能します。
  • hc->context に格納されることで、監視対象ワーカーとヘルスチェック用コンテキストが紐付けられます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・生成）**:
  • modules/proxy/mod_proxy_hcheck.c:429: create_hcheck_req の引数として使用。
  • modules/proxy/mod_proxy_hcheck.c:493: apr_pcalloc によりメモリ確保。
  • modules/proxy/mod_proxy_hcheck.c:541, 854: hc->context からキャストして取得。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • proxy_worker: w フィールドが参照する、プロキシのバックエンドサーバー定義。
  • hc_template_t: ヘルスチェックのパラメータを保持する構造体。
  • baton_t: ヘルスチェック実行時のタスク管理用構造体。

失敗モードと境界条件

  • w が NULL の状態で参照された場合、セグメンテーション違反が発生する可能性があります。
  • req が適切に初期化されていない場合、バックエンドへのリクエスト送信が失敗します。

保守担当者向けメモ

  • 本構造体は mod_proxy_hcheck のライフサイクルにおいて、ワーカーのヘルスチェック設定が変更された際に再構築される可能性があります。
  • メモリ管理は apr_pcalloc 等のプールベースで行われており、チェック実行ごとのライフサイクルに注意してください。
