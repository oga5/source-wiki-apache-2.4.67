yy_symbol_value_print

目的

GNU Bison によって生成されたパーサーにおいて、デバッグ出力が有効な場合に、スタック上のシンボル値（トークンや非終端記号の値）を標準エラー出力等のストリームへ出力するために使用されます。

シグネチャまたは形状

static void yy_symbol_value_print(FILE *yyoutput, int yytype, YYSTYPE const * const yyvaluep, ap_expr_parse_ctx_t *ctx);

入力と出力

  • yyoutput: 出力先の FILE ストリーム。
  • yytype: 出力対象のシンボル型を示す整数値。
  • yyvaluep: 出力対象のシンボル値が格納された YYSTYPE 共用体へのポインタ。
  • ctx: 解析コンテキスト（ap_expr_parse_ctx_t）。
  • 出力: なし（void）。指定されたストリームへシンボルの文字列表現を書き出します。

前提条件と不変条件

  • 本関数は Bison 生成コードの一部であり、主にデバッグビルド時やトレース出力が有効な環境で呼び出されます。
  • yyvaluep は NULL であってはなりません。

アルゴリズムまたは振る舞い

1. 引数として受け取った yytype に基づき、対応するシンボルの値を yyvaluep から読み取ります。 2. yyoutput に対して、シンボルの型情報およびその値を人間が判読可能な形式でフォーマットして出力します。 3. Bison の内部的なトレースロジックに従い、解析スタックの状態を可視化する補助を行います。

呼び出し元と呼び出し先

  • 呼び出し元: server/util_expr_parse.c 内のパーサー実行ルーチン（yyparse 等）から、デバッグトレース出力の一部として呼び出されます。
  • 呼び出し先: 標準的な C ライブラリの出力関数（fprintf 等）が使用されます。

関連データ構造

  • YYSTYPE: パーサーのスタックで使用される共用体。シンボルの値を保持します。
  • ap_expr_parse_ctx_t: 解析コンテキスト。

失敗モードと境界条件

  • yyoutput が無効なストリームである場合、出力は失敗します。
  • yytype が定義されていない値である場合、予期しない出力やフォーマットエラーが発生する可能性があります。
