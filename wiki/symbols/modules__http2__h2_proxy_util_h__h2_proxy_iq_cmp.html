h2_proxy_iq_cmp

定義

  • 種別: typedef
  • 定義箇所: modules/http2/h2_proxy_util.h:85

目的

h2_proxy_iqueue（整数キュー）内の要素を比較するためのコールバック関数型を定義します。

シグネチャまたは形状

typedef int h2_proxy_iq_cmp(int i1, int i2, void *ctx);

入力と出力

  • **入力**:
  • i1, i2: 比較対象となる2つの整数値（主にストリームIDなど）。
  • ctx: 比較処理に必要なコンテキスト情報へのポインタ。
  • **出力**:
  • int: 比較結果を返します。標準的な比較関数と同様に、i1 < i2 なら負、i1 == i2 なら0、i1 > i2 なら正の値を返すことが期待されます。

前提条件と不変条件

  • h2_proxy_iqueue のソートや追加処理において、この関数ポインタが有効な比較ロジックを提供する必要があります。
  • ctx は、比較ロジック内で外部状態を参照する必要がある場合に利用されます。

アルゴリズムまたは振る舞い

  • h2_proxy_iqueue の順序を決定するための比較基準を抽象化します。
  • 具体的な比較ロジックは、この型を実装する関数側に委ねられます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • h2_proxy_iq_add (modules/http2/h2_proxy_util.c:243, h2_proxy_util.h:114)
  • h2_proxy_iq_sort (modules/http2/h2_proxy_util.c:286, h2_proxy_util.h:138)
  • modules/http2/h2_proxy_util.c 内の内部処理（217, 219, 357, 369行目付近）
  • **呼び出し先**:
  • この型自体は関数ポインタの定義であり、具体的な呼び出し先は、この型を使用して登録された関数実装に依存します。

関連データ構造

  • h2_proxy_iqueue: この比較関数を使用してソートや順序管理を行う優先度付きキュー構造体。

失敗モードと境界条件

  • 比較関数内で ctx が適切に初期化されていない場合、予期しない動作やセグメンテーション違反が発生する可能性があります。
  • 比較ロジックが推移律（a < b かつ b < c ならば a < c）を満たさない場合、キューのソート結果が不定となります。

保守担当者向けメモ
