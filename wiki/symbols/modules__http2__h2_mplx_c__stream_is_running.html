stream_is_running

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_mplx.c:112

目的

指定された HTTP/2 ストリームが現在実行中（ワーカーによって処理が開始されており、かつ完了していない状態）であるかを判定します。

シグネチャまたは形状

static int stream_is_running(h2_stream *stream)

入力と出力

  • **入力**: h2_stream *stream - 状態を確認する対象のストリーム。
  • **出力**: int - ストリームが実行中であれば非ゼロ（真）、そうでなければ 0（偽）を返します。

前提条件と不変条件

  • stream は有効な h2_stream 構造体を指している必要があります。
  • stream->c2（セカンダリ接続）が初期化されていることを前提としています。

アルゴリズムまたは振る舞い

1. h2_conn_ctx_get(stream->c2) を呼び出し、ストリームに関連付けられた接続コンテキストを取得します。 2. 以下の条件をすべて満たす場合に実行中とみなします。

  • 接続コンテキストが存在すること。
  • conn_ctx->started アトミック変数が 0 ではないこと（処理が開始されている）。
  • conn_ctx->done アトミック変数が 0 であること（処理が完了していない）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_mplx.c:124
  • modules/http2/h2_mplx.c:143
  • modules/http2/h2_mplx.c:175
  • modules/http2/h2_mplx.c:1160
  • **呼び出し先**:
  • h2_conn_ctx_get
  • apr_atomic_read32

関連データ構造

  • h2_stream: ストリームの基本構造体。
  • h2_conn_ctx_t: 接続コンテキスト。started および done フラグを保持し、スレッド間の状態同期に使用されます。

失敗モードと境界条件

  • stream->c2 が NULL の場合、h2_conn_ctx_get の挙動に依存しますが、本関数は conn_ctx が NULL であることをチェックしており、その場合は 0 を返します。

保守担当者向けメモ

  • 本関数は static であり、h2_mplx.c 内でのみ使用されます。
  • started および done フラグはアトミック操作で読み取られるため、マルチスレッド環境下でのストリーム状態確認として安全です。
