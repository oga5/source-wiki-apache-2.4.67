dump_loaded_modules

定義

  • 種別: function
  • 定義箇所: modules/core/mod_so.c:362

目的

Apache HTTP Server にロードされている静的モジュールおよび共有モジュール（DSO）の一覧を標準出力へ出力します。

シグネチャまたは形状

static void dump_loaded_modules(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t ポインタ。
  • s: 現在のサーバー設定を保持する server_rec 構造体。
  • **出力**: なし（戻り値は void）。副作用として、設定で DUMP_MODULES が定義されている場合に標準出力へモジュールリストを書き出します。

前提条件と不変条件

  • 実行環境において DUMP_MODULES が設定定義（ap_exists_config_define）されている必要があります。
  • s->module_config から so_module に関連付けられた so_server_conf が取得可能である必要があります。

アルゴリズムまたは振る舞い

1. DUMP_MODULES 定義の有無を確認し、存在しない場合は即座に終了します。 2. 標準出力ストリームをオープンします。 3. ap_prelinked_module_symbols 配列を走査し、静的モジュール名を出力します。 4. sconf->loaded_modules 配列を走査し、ロード済みの共有モジュール名を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_so.c:422 にて ap_hook_test_config に登録されています。
  • **呼び出し先**:
  • ap_exists_config_define
  • apr_file_open_stdout
  • apr_file_printf
  • ap_get_module_config

関連データ構造

  • so_server_conf: ロードされたモジュール情報を保持する構造体。
  • ap_module_symbol_t: モジュール名とシンボルを保持する構造体。
  • ap_prelinked_module_symbols: 静的にリンクされたモジュールのリスト。

失敗モードと境界条件

  • DUMP_MODULES が定義されていない場合、何も出力せず正常終了します。
  • sconf->loaded_modules が空の場合、共有モジュールの出力はスキップされます。

保守担当者向けメモ

  • 本関数は設定テストフェーズ（test_config フック）で実行されることを想定しています。
  • 出力フォーマットを変更する場合は、apr_file_printf の書式指定を確認してください。
