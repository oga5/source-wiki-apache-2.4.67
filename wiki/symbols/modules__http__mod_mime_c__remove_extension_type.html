remove_extension_type

定義

  • 種別: function
  • 定義箇所: modules/http/mod_mime.c:280

目的

Apache HTTP Server の設定ディレクティブ RemoveType を処理し、指定されたファイル拡張子に関連付けられた MIME タイプの設定を無効化（削除）するために使用されます。

シグネチャまたは形状

static const char *remove_extension_type(cmd_parms *cmd, void *m_, const char *ext);

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • m_: 現在のディレクトリ設定構造体（mime_dir_config）。
  • ext: 無効化対象のファイル拡張子文字列。
  • **出力**:
  • 成功時は NULL を返し、エラー発生時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • add_extension_info が内部で呼び出されるため、同関数が要求する引数の整合性が保たれている必要があります。
  • m_ は mime_dir_config 型としてキャスト可能なポインタである必要があります。

アルゴリズムまたは振る舞い

1. add_extension_info 関数を呼び出します。 2. その際、第3引数（MIME タイプ）に空文字列 "" を渡すことで、指定された拡張子 ext に対するマッピングを削除対象として登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/mod_mime.c:416 において、AP_INIT_ITERATE マクロにより RemoveType ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: add_extension_info を直接呼び出します。

関連データ構造

  • cmd_parms: 設定ファイル解析時のコンテキスト情報。
  • mime_dir_config: ディレクトリごとの MIME マッピングを保持する構造体。

失敗モードと境界条件

  • add_extension_info がエラーを返した場合、そのエラーメッセージをそのまま呼び出し元へ伝播します。
  • 拡張子 ext が NULL または空の場合の挙動は、内部で呼び出される add_extension_info の実装に依存します。

保守担当者向けメモ

  • 本関数は RemoveType ディレクティブのラッパーです。実質的な削除ロジックは add_extension_info 内で空文字列を渡すことで実現されています。
  • 拡張子の解析やマッピングの削除順序については、mod_mime.c 全体のマージロジック（merge_mime_dir_configs 等）と併せて確認してください。
