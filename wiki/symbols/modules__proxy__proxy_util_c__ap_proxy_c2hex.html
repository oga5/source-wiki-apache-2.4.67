ap_proxy_c2hex

定義

  • 種別: function
  • 定義箇所: modules/proxy/proxy_util.c:173

目的

文字コードをパーセントエンコーディング形式（%XX）の16進数文字列に変換します。

シグネチャまたは形状

void ap_proxy_c2hex(int ch, char *x)

入力と出力

  • **入力**:
  • ch: 変換対象の文字（整数値）。
  • x: 変換後の文字列を格納するバッファへのポインタ（少なくとも3バイト以上の領域が必要）。
  • **出力**:
  • なし（x が指すメモリ領域に結果が書き込まれます）。

前提条件と不変条件

  • x は、少なくとも3バイト（EBCDIC環境では4バイト）の書き込み可能なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. **非EBCDIC環境**:

  • x[0] に % を設定します。
  • ch の上位4ビットを16進数文字に変換し x[1] に格納します。
  • ch の下位4ビットを16進数文字に変換し x[2] に格納します。

    • 2. **EBCDIC環境**:

  • ap_xlate_proto_to_ascii を使用して文字をASCIIに変換します。
  • 変換後の値を16進数文字列として x[0] から x[2] に格納し、x[3] に終端文字 \0 を付与します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/proxy_util.c:304 での呼び出しが確認されています。
  • **呼び出し先**: EBCDIC環境下では ap_xlate_proto_to_ascii を呼び出します。

関連データ構造

  • なし。

失敗モードと境界条件

  • x に十分なバッファサイズが確保されていない場合、バッファオーバーフローが発生します。

保守担当者向けメモ

  • EBCDIC環境とそれ以外で実装が分岐しています。修正時は両方のパスを確認してください。
  • 本関数は文字列の終端文字（\0）を付与しないケース（非EBCDIC時）があるため、呼び出し側で適切に終端処理を行う必要があります。
