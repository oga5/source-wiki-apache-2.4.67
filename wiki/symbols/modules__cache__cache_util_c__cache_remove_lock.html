cache_remove_lock

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_util.c:390

目的

キャッシュ更新時に作成された排他ロックファイルを削除し、リソースを解放します。

シグネチャまたは形状

apr_status_t cache_remove_lock(cache_server_conf *conf,
                               cache_request_rec *cache,
                               request_rec *r,
                               apr_bucket_brigade *bb);

入力と出力

  • **入力**:
  • conf: キャッシュ設定情報。
  • cache: リクエストごとのキャッシュコンテキスト。
  • r: 現在のリクエストレコード。
  • bb: バケットブリゲード（オプション）。
  • **出力**:
  • apr_status_t: 操作の成功（APR_SUCCESS）またはエラーコード。

前提条件と不変条件

  • conf が設定されていない場合、またはロック設定（lock や lockpath）が欠如している場合は、何もせず APR_SUCCESS を返します。
  • bb が指定されている場合、ブリゲード内に EOS (End of Stream) バケットが存在しない限り、ロック削除は行われません。

アルゴリズムまたは振る舞い

1. **ブリゲード確認**: bb が渡された場合、EOS バケットの有無を確認し、存在しない場合は処理を中断して終了します。 2. **既存ロックのクローズ**: r->pool に CACHE_LOCKFILE_KEY で登録されたファイルハンドルが存在すれば、それをクローズします。 3. **ロックパスの特定**: CACHE_LOCKNAME_KEY からロックファイル名を取得します。 4. **ファイル名生成**: ロック名が未定義の場合、cache->key を基にハッシュ化されたファイル名を生成し、conf->lockpath 配下の2階層ディレクトリ構造に配置されるパスを構築します。 5. **削除実行**: apr_file_remove を呼び出し、ファイルシステムからロックファイルを削除します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache.c 内の複数の箇所（フィルタ処理の完了時など）から呼び出されます。
  • **呼び出し先**: apr_pool_userdata_get, apr_file_close, cache_generate_key, ap_cache_generate_name, apr_pstrcat, apr_file_remove 等の APR/Apache API。

関連データ構造

  • cache_server_conf: キャッシュ設定を保持。
  • cache_request_rec: リクエストごとのキャッシュ状態を管理。
  • request_rec: Apache のリクエストレコード。
  • apr_bucket_brigade: フィルタチェーン内のデータストリーム。

失敗モードと境界条件

  • ロックファイルが存在しない場合や、権限不足により削除できない場合は apr_file_remove の戻り値（エラーコード）が返されます。
  • conf が NULL の場合は安全に早期リターンします。

保守担当者向けメモ

  • 本関数は「サンダリング・ハード」問題を防ぐためのロック機構の一部です。
  • r->pool を使用しているため、リクエスト終了時にメモリは自動的に管理されます。
  • ロックファイルはプロセス異常終了時でも lockmaxage による期限切れチェックで保護される設計となっています。
