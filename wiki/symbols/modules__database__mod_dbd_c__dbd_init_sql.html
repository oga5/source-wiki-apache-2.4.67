dbd_init_sql

定義

  • 種別: function
  • 定義箇所: modules/database/mod_dbd.c:267

目的

設定ディレクティブ DBDInitSQL を介して指定された SQL 文を、サーバーの初期化用 SQL リストに登録する。

シグネチャまたは形状

static const char *dbd_init_sql(cmd_parms *cmd, void *dconf, const char *query)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dconf: モジュールの設定構造体（本関数内では未使用）。
  • query: 実行対象の SQL 文（文字列）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • query 引数が NULL または改行文字のみの場合はエラーとなる。

アルゴリズムまたは振る舞い

1. query が有効な文字列であるか検証する。無効な場合はエラーメッセージを返す。 2. ap_dbd_sql_init を呼び出し、cmd->server に関連付けられたサーバー設定へ query を登録する。 3. 正常終了として NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/database/mod_dbd.c:288 における AP_INIT_TAKE1("DBDInitSQL", ...) ディレクティブ定義から間接的に呼び出される。
  • **呼び出し先**: ap_dbd_sql_init（modules/database/mod_dbd.c 内で定義）。

関連データ構造

  • cmd_parms: Apache の設定コマンド用構造体。

失敗モードと境界条件

  • query が NULL または \n の場合、"You should specify SQL statement" というエラーメッセージを返す。

保守担当者向けメモ

  • 本関数は設定フェーズで実行される。登録された SQL 文は、後の接続確立時に dbd_run_post_connect を通じて実行される初期化処理で使用される。
