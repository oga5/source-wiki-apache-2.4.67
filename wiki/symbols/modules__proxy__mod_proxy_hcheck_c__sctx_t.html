sctx_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_hcheck.c:54

目的

mod_proxy_hcheck モジュールにおけるサーバー単位のヘルスチェック設定コンテキストを保持する構造体です。

シグネチャまたは形状

typedef struct {
    apr_pool_t *p;
    apr_array_header_t *templates;
    apr_table_t *conditions;
    apr_hash_t *hcworkers;
    server_rec *s;
} sctx_t;

入力と出力

本構造体は、ap_get_module_config を介してサーバー設定から取得され、ヘルスチェックの実行時や設定変更時に参照・更新されます。

前提条件と不変条件

  • p: 設定のライフサイクルを管理するメモリプール。
  • s: 関連付けられた server_rec へのポインタ。
  • hcworkers: ヘルスチェック対象のワーカーを管理するハッシュテーブル。

アルゴリズムまたは振る舞い

  • ヘルスチェックのテンプレート定義、条件式、およびワーカーのハッシュマップを保持し、モジュール全体で共有される設定状態を管理します。
  • hc_get_hcworker 等の関数を通じて、特定のワーカーに対するヘルスチェック設定の検索や構築に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • hc_create_config (初期化時)
  • hc_get_hcworker (ワーカー取得時)
  • hc_watchdog_callback 経由の各チェック関数 (実行時)
  • ap_get_module_config を呼び出す各設定ハンドラ
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • hc_template_t: templates 配列に含まれるヘルスチェックパラメータ。
  • proxy_worker: hcworkers ハッシュ内で管理されるバックエンドワーカー。
  • server_rec: Apache サーバー設定。

失敗モードと境界条件

  • 設定が未定義の場合、ap_get_module_config から NULL が返される可能性があります。
  • メモリ割り当て失敗時は apr_pcalloc が例外を発生させる可能性があります。

保守担当者向けメモ

  • sctx_t は mod_proxy_hcheck の設定状態の基点です。設定変更時には hc_get_hcworker 内で動的に再構築されるため、整合性に注意してください。
  • baton_t を介してヘルスチェック実行スレッドに渡されることが多いため、スレッドセーフなアクセスを維持してください。
