check_string

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_dbd.c:536

目的

設定値として指定された文字列が、クッキーの仕様上禁止されている文字（=, ;, &）を含んでいないかを検証する。

シグネチャまたは形状

static const char *check_string(cmd_parms *cmd, const char *string)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • string: 検証対象の文字列。
  • **出力**:
  • 検証に成功した場合: NULL を返す。
  • 検証に失敗した場合: エラーメッセージを含む文字列（char *）を返す。

前提条件と不変条件

  • ap_cookie_check_string が利用可能であること。
  • cmd->pool が有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. ap_cookie_check_string(string) を呼び出し、文字列に =, ;, & が含まれていないかを確認する。 2. APR_SUCCESS 以外が返された場合、cmd->pool を使用してエラーメッセージを生成し、それを返す。 3. 検証を通過した場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_form.c (複数箇所)
  • modules/session/mod_session_cookie.c (複数箇所)
  • modules/session/mod_session_dbd.c (585行目、605行目)
  • **呼び出し先**:
  • ap_cookie_check_string (外部関数)
  • apr_pstrcat (外部関数)

関連データ構造

  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。

失敗モードと境界条件

  • 検証対象の文字列に =, ;, & が含まれている場合、エラーメッセージが返される。
  • メモリ割り当て失敗時は apr_pstrcat の挙動に依存する。

保守担当者向けメモ

  • 本関数は mod_session 関連モジュールおよび mod_auth_form で共通のバリデーターとして利用されている。
  • クッキーの構文規則に変更があった場合は、ap_cookie_check_string の仕様を確認すること。
