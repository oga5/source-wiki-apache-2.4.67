req_err_headers_out_table

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:337

目的

Apache HTTP リクエストに関連付けられたエラー応答ヘッダー（err_headers_out）を、Lua スクリプトから操作可能な Lua テーブル形式に変換して取得する。

シグネチャまたは形状

static int req_err_headers_out_table(lua_State *L)

入力と出力

  • **入力**: Lua スタックの第1引数に Apache2.Request 型の userdata を受け取る。
  • **出力**: err_headers_out テーブルの内容を反映した Lua テーブルをスタックにプッシュし、戻り値として 1 を返す。

前提条件と不変条件

  • Lua スタックの第1引数が有効な request_rec ポインタを保持する Apache2.Request userdata であること。
  • ap_lua_check_request_rec による検証が成功すること。

アルゴリズムまたは振る舞い

1. ap_lua_check_request_rec(L, 1) を呼び出し、Lua スタックから request_rec 構造体を取得する。 2. 取得した request_rec の err_headers_out フィールド（apr_table_t）を対象とする。 3. req_aprtable2luatable を呼び出し、APR テーブルを Lua テーブルに変換してスタックに配置する。 4. 変換後のテーブル数を表す 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_request.c:2841 にて、makefun を通じてディスパッチテーブルに登録されている。
  • **呼び出し先**:
  • ap_lua_check_request_rec (検証)
  • req_aprtable2luatable (変換)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • apr_table_t: Apache のヘッダー情報を格納するテーブル構造体。
  • lua_State: Lua インタプリタの状態。

失敗モードと境界条件

  • ap_lua_check_request_rec が失敗した場合、Lua 側でエラーが発生し、本関数は正常終了しない。
  • err_headers_out が空の場合、空の Lua テーブルが返される。

保守担当者向けメモ

  • 本関数は err_headers_out（エラー発生時に送信されるヘッダー）専用である。通常のレスポンスヘッダーについては req_headers_out_table を参照すること。
  • メモリ管理は request_rec のプールに依存しているため、明示的な解放は不要である。
