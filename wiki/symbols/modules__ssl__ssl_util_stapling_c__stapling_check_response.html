stapling_check_response

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_stapling.c:386

目的

OCSP レスポンスの妥当性を検証し、TLS ハンドシェイクにおいてクライアントへステータスを返却すべきかを判断する。

シグネチャまたは形状

int stapling_check_response(server_rec *s, modssl_ctx_t *mctx, certinfo *cinf, OCSP_RESPONSE *rsp, BOOL *pok)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • mctx: SSL コンテキスト。
  • cinf: 証明書情報（certinfo）。
  • rsp: 検証対象の OCSP レスポンス。
  • pok: 検証結果を格納するフラグへのポインタ（NULL の場合、キャッシュからの検証であることを示す）。
  • **出力**:
  • 戻り値: SSL_TLSEXT_ERR_OK（レスポンスを送信）、または SSL_TLSEXT_ERR_NOACK（送信しない）。

前提条件と不変条件

  • rsp が OCSP_RESPONSE_STATUS_SUCCESSFUL でない場合、設定に応じてエラーを無視するか、送信を拒否する。
  • pok が指定されている場合、検証成功時に TRUE が設定される。
  • MAX_STAPLING_DER（10KB）を超えるレスポンスは切り捨てられる可能性がある。

アルゴリズムまたは振る舞い

1. **ステータス確認**: OCSP_response_status を確認し、成功以外の場合は設定に従い SSL_TLSEXT_ERR_OK または SSL_TLSEXT_ERR_NOACK を返す。 2. **パース**: OCSP_response_get1_basic で基本レスポンスを取得。失敗時はエラーログを出力し、デフォルトで SSL_TLSEXT_ERR_OK を返す。 3. **ID 照合**: OCSP_resp_find_status を使用し、対象証明書（cinf->cid）のステータスを検索。見つからない場合は SSL_TLSEXT_ERR_NOACK を検討する。 4. **有効期限検証**: OCSP_check_validity を呼び出し、設定されたスキューと最大有効期間内で検証。期限切れや無効な場合は SSL_TLSEXT_ERR_NOACK を返す。 5. **ログ出力**: ステータスが GOOD 以外の場合、シリアル番号と共に詳細なステータスと理由をログに記録する。 6. **クリーンアップ**: OCSP_BASICRESP_free を呼び出し、メモリを解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_util_stapling.c:570
  • modules/ssl/ssl_util_stapling.c:750
  • **呼び出し先**:
  • OpenSSL OCSP API (OCSP_response_status, OCSP_response_get1_basic, OCSP_resp_find_status, OCSP_check_validity, OCSP_BASICRESP_free, OCSP_id_get0_info, OCSP_cert_status_str, OCSP_crl_reason_str)
  • Apache ログ API (ap_log_error)

関連データ構造

  • certinfo: 証明書ハッシュや CERTID を保持。
  • modssl_ctx_t: OCSP 関連の設定（stapling_return_errors, stapling_resptime_skew, stapling_resp_maxage）を保持。
  • OCSP_RESPONSE, OCSP_BASICRESP: OpenSSL の OCSP 構造体。

失敗モードと境界条件

  • **パース失敗**: OCSP_response_get1_basic が NULL を返す場合、エラーログを出力して処理を継続する。
  • **ID 不一致**: 証明書 ID がレスポンスに含まれない場合、設定に応じて SSL_TLSEXT_ERR_NOACK を返す。
  • **期限切れ**: OCSP_check_validity が失敗した場合、キャッシュからのレスポンスであればデバッグログ、直接のレスポンスであればエラーログを出力し、送信を抑制する。

保守担当者向けメモ

  • 本関数は OpenSSL の ASN.1 パースに依存している。
  • pok 引数の有無により、キャッシュからの検証か、レスポンダーからの直接検証かを区別している。
  • メモリ管理は OpenSSL の API に準拠しており、OCSP_BASICRESP の解放漏れに注意すること。
