dav_walker_ctx

定義

  • **種別**: struct
  • **定義箇所**: modules/dav/main/mod_dav.h:1849

目的

WebDAV のリソース走査（walk）処理において、リクエストの状態、バッファ、およびプロバイダー固有のパラメータを保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct dav_walker_ctx {
    dav_walk_params w;
    apr_bucket_brigade *bb;
    apr_pool_t *scratchpool;
    request_rec *r;
    apr_xml_doc *doc;
    int propfind_type;
    apr_text *propstat_404;
    const dav_if_header *if_header;
    const dav_locktoken *locktoken;
    const dav_lock *lock;
    int skip_root;
    int flags;
    dav_buffer work_buf;
} dav_walker_ctx;

入力と出力

  • **入力**: dav_walk_params を通じて走査の基本パラメータを受け取ります。また、request_rec を介して元の HTTP リクエスト情報にアクセスします。
  • **出力**: apr_bucket_brigade を通じてレスポンスデータをフィルタへ出力し、scratchpool を使用してメモリを管理します。

前提条件と不変条件

  • scratchpool は、ストリーミングレスポンス生成時に反復的にクリアされることを前提としています。
  • propfind_type は DAV_PROPFIND_IS_ALLPROP、DAV_PROPFIND_IS_PROPNAME、または DAV_PROPFIND_IS_PROP のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. **リクエスト処理**: r を通じてリクエストを追跡し、if_header を用いてリクエストの検証を行います。 2. **データバッファリング**: bb を使用して r->output_filters へ送信するデータをバッファリングします。 3. **プロパティ操作**: PROPFIND 実行時、doc から解析された XML を参照し、propstat_404 をキャッシュとして利用して 404 エラーを管理します。 4. **ロック管理**: lock や locktoken を保持し、dav_inherit_locks() 実行時には skip_root フラグを用いてルートの扱いを制御します。 5. **メモリ管理**: scratchpool を使用して、反復的な処理におけるメモリ消費を抑制します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_dav.c および util.c、util_lock.c 内の各関数（wres->walk_ctx として参照）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、内部の各メンバは dav_validate_request() 等のユーティリティ関数によって操作されます。

関連データ構造

  • dav_walk_params: 走査の基本パラメータ。
  • dav_buffer: dav_validate_request() 用の作業バッファ。
  • request_rec: Apache HTTP Server のリクエスト構造体。
  • apr_bucket_brigade: 出力フィルタ用バッファ。
  • apr_xml_doc: PROPFIND 用の XML 解析結果。

失敗モードと境界条件

  • scratchpool のクリアタイミングが不適切な場合、メモリリークや不正なメモリ参照が発生する可能性があります。
  • propfind_type が未定義の値である場合、プロパティ取得ロジックが予期しない動作をする可能性があります。

保守担当者向けメモ

  • 本構造体は、クライアントデータ管理の肥大化を解消する過程にあります。
  • dav_walker_ctx の各メンバは、リクエストのライフサイクルと密接に関連しているため、メモリ管理には apr_pool_t の生存期間を考慮してください。
