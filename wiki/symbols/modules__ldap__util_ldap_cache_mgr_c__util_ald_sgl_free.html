util_ald_sgl_free

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache_mgr.c:184

目的

LDAP 比較サブグループ構造体（util_compare_subgroup_t）およびその内部に含まれる文字列配列を、キャッシュ専用のアロケータを使用して解放する。

シグネチャまたは形状

void util_ald_sgl_free(util_ald_cache_t *cache, util_compare_subgroup_t **sgl)

入力と出力

  • cache: キャッシュ管理構造体へのポインタ。メモリ解放に使用されるアロケータのコンテキストとして利用される。
  • sgl: 解放対象の util_compare_subgroup_t 構造体へのポインタのポインタ。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • cache は有効なキャッシュ管理構造体であること。
  • sgl が指す先の構造体は、util_ald_alloc 等のキャッシュ専用アロケータで割り当てられていること。
  • sgl が NULL または *sgl が NULL の場合、関数は何もせず即座に復帰する。

アルゴリズムまたは振る舞い

1. sgl および *sgl が NULL でないことを確認する。 2. (*sgl)->len に基づき、subgroupDNs 配列内の各文字列要素を util_ald_free で順次解放する。 3. 最後に、構造体本体である *sgl 自体を util_ald_free で解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/ldap/util_ldap.c:1630
  • modules/ldap/util_ldap_cache.c:283
  • 呼び出し先:
  • util_ald_free (キャッシュ専用のメモリ解放関数)

関連データ構造

  • util_ald_cache_t: キャッシュ管理構造体。
  • util_compare_subgroup_t: 解放対象のデータ構造。内部に len（要素数）と subgroupDNs（文字列配列）を保持する。

失敗モードと境界条件

  • sgl が NULL または *sgl が NULL の場合、安全に早期復帰する。
  • 内部の文字列配列が既に解放済みである場合や、不正なポインタである場合の挙動は未定義である。

保守担当者向けメモ

  • 本関数は util_ald_sgl_dup で複製されたメモリを解放するために使用される。
  • 共有メモリ（apr_rmm）環境下では、util_ald_free が適切なオフセット計算を行うため、必ず本関数を経由して解放を行うこと。
