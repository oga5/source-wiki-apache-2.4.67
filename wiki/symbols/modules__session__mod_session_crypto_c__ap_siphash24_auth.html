ap_siphash24_auth

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:157

目的

入力データに対して SipHash-2-4 アルゴリズムを用いて認証タグ（MAC）を生成し、指定されたバッファに格納する。

シグネチャまたは形状

static void ap_siphash24_auth(unsigned char out[AP_SIPHASH_DSIZE],
                              const void *src, apr_size_t len,
                              const unsigned char key[AP_SIPHASH_KSIZE])

入力と出力

  • out: 生成された認証タグを格納する出力バッファ（AP_SIPHASH_DSIZE バイト）。
  • src: 認証対象のデータポインタ。
  • len: src のデータ長。
  • key: 認証に使用する秘密鍵（AP_SIPHASH_KSIZE バイト）。
  • 戻り値: なし（out に結果が書き込まれる）。

前提条件と不変条件

  • out は少なくとも AP_SIPHASH_DSIZE バイトのメモリ領域を確保していること。
  • key は AP_SIPHASH_KSIZE バイトの有効な鍵データであること。

アルゴリズムまたは振る舞い

1. ap_siphash24 を呼び出し、入力データ src と鍵 key から 64 ビットのハッシュ値 h を計算する。 2. U64TO8_LE マクロを使用して、計算されたハッシュ値 h をリトルエンディアン形式で out バッファに書き込む。

呼び出し元と呼び出し先

  • 呼び出し先: ap_siphash24
  • 呼び出し元: modules/session/mod_session_crypto.c:181（compute_auth 関数内）
  • 補足: 同ファイル内で #define ap_siphash24_auth apr_siphash24_auth という定義が存在するが、本関数はソースコード上で直接実装されている。

関連データ構造

  • AP_SIPHASH_DSIZE: 認証タグのサイズ定数。
  • AP_SIPHASH_KSIZE: 鍵のサイズ定数。

失敗モードと境界条件

  • 入力データ長 len が 0 の場合でも、SipHash の仕様に基づきハッシュ計算が行われる。
  • メモリ保護違反を避けるため、out および src が指すメモリ領域が有効である必要がある。

保守担当者向けメモ

  • 本関数は compute_auth を通じてセッションデータの改ざん検知に使用される。
  • U64TO8_LE マクロによるエンディアン変換が必須であるため、移植時にはこの変換がプラットフォーム間で正しく動作することを確認すること。
