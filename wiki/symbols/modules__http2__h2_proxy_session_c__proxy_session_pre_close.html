proxy_session_pre_close

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_proxy_session.c:186

目的

HTTP/2 プロキシセッションに関連付けられたリソースを、接続のクローズ直前に安全に解放・破棄します。

シグネチャまたは形状

static apr_status_t proxy_session_pre_close(void *theconn)

入力と出力

  • **入力**: theconn (void *) - proxy_conn_rec 構造体へのポインタ。
  • **出力**: APR_SUCCESS を常に返します。

前提条件と不変条件

  • theconn は proxy_conn_rec 型としてキャスト可能である必要があります。
  • p_conn->data に h2_proxy_session 構造体が格納されていることを前提としています。

アルゴリズムまたは振る舞い

1. theconn から h2_proxy_session を取得します。 2. セッションおよび nghttp2 セッションが存在する場合にのみ処理を実行します。 3. セッションの aborted フラグを 1 に設定します。 4. H2_PROXYS_EV_PRE_CLOSE イベントをディスパッチし、状態遷移をトリガーします。 5. nghttp2_session_del を呼び出して nghttp2 セッションを破棄し、ポインタを NULL にクリアします。 6. p_conn->data を NULL に設定し、接続構造体との関連付けを解除します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c:746 にて、apr_pool_pre_cleanup_register を通じて登録されています。
  • **呼び出し先**:
  • dispatch_event (イベント通知)
  • nghttp2_session_del (ライブラリによるメモリ解放)
  • ap_log_cerror (ログ出力)
  • h2_proxy_ihash_count (ストリーム数確認)

関連データ構造

  • proxy_conn_rec: プロキシ接続の管理構造体。
  • h2_proxy_session: HTTP/2 プロキシセッションの内部状態を保持する構造体。

失敗モードと境界条件

  • session または session->ngh2 が NULL の場合、関数は何も行わず即座に APR_SUCCESS を返します。

保守担当者向けメモ

  • 本関数は apr_pool_pre_cleanup_register で登録されており、接続に関連付けられたメモリプールが破棄される直前に自動的に呼び出されます。
  • nghttp2 セッションの破棄後に session 構造体自体が解放されるため、本関数内での session へのアクセスは、nghttp2 関連のクリーンアップに限定してください。
