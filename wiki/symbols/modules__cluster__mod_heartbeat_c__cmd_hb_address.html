cmd_hb_address

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartbeat.c:166

目的

HeartbeatAddress ディレクティブのハンドラとして、マルチキャスト先のアドレスとポートを解析し、モジュールの設定構造体（hb_ctx_t）に格納する。

シグネチャまたは形状

static const char *cmd_hb_address(cmd_parms *cmd, void *dconf, const char *addr)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dconf: ディレクトリ設定（本関数では使用されない）。
  • addr: 設定ファイルから渡される host:port 形式の文字列。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ（const char *）を返す。

前提条件と不変条件

  • HeartbeatAddress はグローバルコンテキストでのみ指定可能であること。
  • モジュール設定内で一度しか呼び出せないこと（二度目の呼び出しはエラーとなる）。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、グローバルコンテキストであることを検証する。 2. hb_ctx_t の active フラグを確認し、既に設定済みであればエラーを返す。未設定であれば active を 1 にセットする。 3. apr_parse_addr_port を使用して、入力文字列からホスト名とポート番号を抽出する。 4. ホスト名またはポート番号が欠落している場合はエラーを返す。 5. apr_sockaddr_info_get を使用して、抽出した情報を ctx->mcast_addr に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cluster/mod_heartbeat.c 内の AP_INIT_TAKE1 マクロによる設定登録（215行目）。
  • **呼び出し先**:
  • ap_check_cmd_context
  • ap_get_module_config
  • apr_parse_addr_port
  • apr_sockaddr_info_get

関連データ構造

  • hb_ctx_t: モジュールの設定を保持する構造体。mcast_addr フィールドに解析結果が格納される。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • **コンテキストエラー**: グローバル以外で指定された場合。
  • **重複定義**: HeartbeatAddress が複数回指定された場合。
  • **解析エラー**: アドレス形式が不正な場合、またはホスト名・ポート番号が指定されていない場合。
  • **システムエラー**: apr_sockaddr_info_get が失敗した場合。

保守担当者向けメモ

  • 本関数は cmd->pool を使用して mcast_addr を確保するため、サーバーのライフサイクル全体で有効なメモリとして保持される。
  • HeartbeatAddress の動的な変更機能は現時点でサポートされていない。
