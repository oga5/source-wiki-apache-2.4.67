bio_filter_out_flush

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_io.c:149

目的

OpenSSL の BIO レイヤーから HTTPD のフィルタスタックへフラッシュ要求を伝播させ、バッファリングされているデータを下位フィルタへ送信する。

シグネチャまたは形状

static int bio_filter_out_flush(BIO *bio)

入力と出力

  • **入力**: BIO *bio (OpenSSL の BIO オブジェクト)
  • **出力**: int (成功時は 1、失敗時は bio_filter_out_pass の戻り値に準ずる)

前提条件と不変条件

  • BIO_get_data(bio) を通じて取得される bio_filter_out_ctx_t が有効であること。
  • フラッシュ実行時、outctx->bb (バケットブリゲード) が空であることを前提とする (AP_DEBUG_ASSERT による検証)。

アルゴリズムまたは振る舞い

1. BIO_get_data を呼び出し、コンテキスト bio_filter_out_ctx_t を取得する。 2. トレースログ (APLOG_TRACE6) にフラッシュ実行を記録する。 3. apr_bucket_flush_create を使用してフラッシュバケットを作成する。 4. 作成したフラッシュバケットを outctx->bb の末尾に挿入する。 5. bio_filter_out_pass(outctx) を呼び出し、ブリゲードを後続のフィルタへ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • bio_filter_out_ctrl (間接的な呼び出し)
  • modules/ssl/ssl_engine_io.c:279
  • modules/ssl/ssl_engine_io.c:1994
  • **呼び出し先**:
  • bio_filter_out_pass

関連データ構造

  • BIO: OpenSSL の I/O 抽象化構造体。
  • bio_filter_out_ctx_t: 出力フィルタのコンテキスト。バケットブリゲード (bb) を保持する。
  • apr_bucket: HTTPD のデータ転送単位。

失敗モードと境界条件

  • bio_filter_out_pass がエラーを返した場合、その戻り値がそのまま本関数の戻り値となる。
  • outctx->bb が空でない状態で呼び出された場合、AP_DEBUG_ASSERT によりデバッグビルド環境ではアサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は OpenSSL の BIO メソッドの一部として機能する。
  • bio_filter_out_pass への依存が強いため、データ転送ロジックの変更時は両方の関数を確認すること。
