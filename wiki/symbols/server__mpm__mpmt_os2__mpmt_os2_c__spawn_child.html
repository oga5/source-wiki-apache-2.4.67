spawn_child

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:327

目的

OS/2 環境において、指定されたスロット番号に対応する新しい子プロセスを生成し、スコアボードを更新する。

シグネチャまたは形状

static void spawn_child(int slot)

入力と出力

  • **入力**: slot (int) - 子プロセスを割り当てるスコアボード上のインデックス。
  • **出力**: なし（副作用としてプロセス生成とスコアボード更新を行う）。

前提条件と不変条件

  • ap_scoreboard_image が初期化されていること。
  • ap_my_generation が現在の世代を正しく保持していること。
  • ap_max_daemons_limit が現在の最大プロセス数制限を保持していること。

アルゴリズムまたは振る舞い

1. 指定された slot のスコアボードエントリに現在の世代 (ap_my_generation) を記録する。 2. DosGetInfoBlocks を使用して現在のプロセス情報を取得し、DosQueryModuleName で実行ファイル名を取得する。 3. DosExecPgm を呼び出し、現在のプロセスと同じプログラムを非同期で実行（EXEC_ASYNCRESULT）する。 4. DosExecPgm が失敗した場合は、ap_log_error を使用してエラーログを出力する。 5. 生成されたプロセスのスロット番号が現在の ap_max_daemons_limit を超える場合、同変数を更新する。 6. proc_rc.codeTerminate から取得したプロセスIDを、スコアボードの該当スロットに保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/mpmt_os2/mpmt_os2.c 内の master_main（行287）から呼び出される。
  • **呼び出し先**: OS/2 ネイティブ API (DosGetInfoBlocks, DosQueryModuleName, DosExecPgm) および Apache ロギング API (ap_log_error)。

関連データ構造

  • ap_scoreboard_image: プロセス間共有メモリ上のスコアボード構造体。
  • PPIB, PTIB, RESULTCODES: OS/2 API で使用されるプロセス情報および実行結果格納構造体。

失敗モードと境界条件

  • DosExecPgm の実行に失敗した場合、エラーコードがログ出力されるが、プロセス生成自体は中断される。
  • slot が HARD_SERVER_LIMIT（10）を超えて指定された場合の挙動は、本関数内では明示的にガードされていない。

保守担当者向けメモ

  • 本関数は OS/2 の DosExecPgm に強く依存している。
  • プロセス生成時の引数として ppib->pib_pchcmd を使用しており、親プロセスのコマンドライン引数を継承する設計となっている。
