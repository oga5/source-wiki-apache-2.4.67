h2_util_base64url_decode

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_util.c:133

目的

Base64URL エンコードされた文字列をデコードし、バイナリデータとして復元する。

シグネチャまたは形状

apr_size_t h2_util_base64url_decode(const char **decoded, const char *encoded, apr_pool_t *pool);

入力と出力

  • **入力**:
  • encoded: デコード対象の Base64URL 文字列。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • *decoded: デコードされたバイナリデータが格納されるバッファへのポインタ（関数内で apr_pcalloc により確保）。
  • 戻り値: デコード後のバイナリデータのバイト数。

前提条件と不変条件

  • encoded 文字列は、BASE64URL_UINT6 テーブルで定義された文字セットに従っている必要がある。
  • pool は有効なメモリプールであること。
  • *decoded には、入力文字列の長さに応じたメモリが確保される。

アルゴリズムまたは振る舞い

1. encoded 文字列を走査し、Base64URL 文字セットに含まれない文字が現れるまでの長さを特定する。 2. apr_pcalloc を使用して、デコード結果を格納するためのメモリを確保する。 3. 4文字単位のブロックを処理し、各文字を 6 ビットの値に変換して 3 バイトのデータに再構成する。 4. 残余（2文字または3文字）がある場合、それらを個別に処理して残りのバイトを復元する。 5. 最終的なデコード済みバイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:1101
  • **呼び出し先**:
  • apr_pcalloc (APR ライブラリ)
  • BASE64URL_UINT6 (マクロまたは定数配列)

関連データ構造

  • apr_pool_t: メモリ管理用構造体。

失敗モードと境界条件

  • 入力文字列が空の場合、戻り値は 0 となり、空のバッファが確保される。
  • Base64URL 文字セット外の文字が入力に含まれる場合、その文字の直前までがデコード対象となる。
  • 4 バイト単位に満たない残余データは、switch 文によって適切に処理される。

保守担当者向けメモ

  • 本関数は h2_util.h で宣言されており、HTTP/2 のヘッダー処理等で利用される。
  • BASE64URL_UINT6 配列の定義が本関数の動作の基盤となっているため、変更時は注意が必要。
