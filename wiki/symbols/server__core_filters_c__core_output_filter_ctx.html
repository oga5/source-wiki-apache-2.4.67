core_output_filter_ctx

定義

  • 種別: struct
  • 定義箇所: server/core_filters.c:81

目的

Apache HTTP Server のコア出力フィルタにおける、接続ごとの永続的な状態を保持するために使用されます。

シグネチャまたは形状

struct core_output_filter_ctx {
    apr_bucket_brigade *buffered_bb;
    apr_pool_t *deferred_write_pool;
    apr_size_t bytes_written;
    struct iovec *vec;
    apr_size_t nvec;
};

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。ただし、ap_core_output_filter がこのコンテキストを参照し、buffered_bb にデータを蓄積し、vec を用いてソケットへの書き込みを行います。

前提条件と不変条件

  • deferred_write_pool は、パイプライン化されたリクエストのバッファリングに使用され、リクエスト終了時に apr_pool_clear される必要があります。
  • 非ブロッキング I/O を前提としており、書き込み失敗時には EAGAIN を考慮した再試行ロジックが適用されます。

アルゴリズムまたは振る舞い

  • **バッファリング**: buffered_bb を通じてレスポンスデータを一時保持します。
  • **書き込み**: vec（iovec 配列）と nvec を使用して、writev 等による効率的なソケット書き込みを管理します。
  • **フロー制御**: 書き込み完了したバケットの削除や、部分的に書き込まれたバケットの分割（apr_bucket_split）を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_core_output_filter がこの構造体をコンテキストとして利用します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出しません。
  • **補足**: include/http_core.h:787 にて core_output_filter_ctx_t として型定義されています。

関連データ構造

  • apr_bucket_brigade: バッファリングされるデータの保持に使用。
  • apr_pool_t: メモリ管理（生存期間管理）に使用。
  • iovec: writev システムコールによるベクトル書き込みに使用。

失敗モードと境界条件

  • 非ブロッキング書き込み時に EAGAIN が発生した場合、残りのデータは setaside_remaining_output 等を通じて保持される必要があります。
  • メモリ不足時には deferred_write_pool の割り当てが失敗する可能性があります。

保守担当者向けメモ

  • deferred_write_pool の生存期間はリクエストのライフサイクルと密接に関連しているため、メモリリークや不正な参照に注意してください。
  • BRIGADE_NORMALIZE マクロを使用する際は、メタデータバケットを保持しつつ空のデータバケットのみを削除する挙動に留意してください。
