step_vars_storage

定義

  • 種別: struct
  • 定義箇所: modules/filters/regexp.h:57

目的

正規表現エンジン（sed）の実行時におけるマッチング状態および後方参照情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct _step_vars_storage {
    char    *loc1, *loc2, *locs;
    char    *braslist[NBRA];
    char    *braelist[NBRA];
    int    low;
    int    size;
} step_vars_storage;

入力と出力

本構造体は、sed_step や _advance などの関数に対してポインタとして渡され、マッチング処理の結果（マッチした範囲や後方参照の開始・終了位置）を格納・更新するために使用されます。

前提条件と不変条件

  • NBRA（括弧の最大数）は 9 に固定されており、これを超えるネストや後方参照はサポートされません。
  • 本構造体は、スタックまたは呼び出し元で確保されたメモリ領域を使用することを前提としています。

アルゴリズムまたは振る舞い

  • sed_step が正規表現マッチングを行う際、マッチした文字列の開始位置を loc1 に、終了位置を loc2 に記録します。
  • braslist および braelist は、正規表現内の括弧（CBRA/CKET）に対応する部分文字列の開始および終了位置を保持します。
  • getrnge 関数において、範囲指定（RNGE）のパラメータ（low, size）を保持するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/regexp.c 内の sed_step、_advance、getrnge、および modules/filters/sed1.c 内の関数群から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • sed_comp_args: コンパイル時のフラグ（行頭一致フラグ等）を管理する構造体。
  • sed_commands_t: libsed におけるコマンド構造体。

失敗モードと境界条件

  • NBRA（9）を超える括弧の使用は、本構造体の配列サイズ制限により正しく処理されません。

保守担当者向けメモ

  • 本構造体は古い AT&T sed の実装を継承しています。現代的な正規表現エンジンとは異なり、状態管理が呼び出し元に依存している点に注意してください。
  • loc1, loc2 などのポインタは、対象となる文字列バッファの生存期間中にのみ有効です。
