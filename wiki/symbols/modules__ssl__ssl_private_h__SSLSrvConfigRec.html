SSLSrvConfigRec

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:596

目的

Apache HTTP Server の mod_ssl モジュールにおいて、仮想ホスト（Virtual Host）ごとの SSL 設定を保持するための構造体です。

シグネチャまたは形状

typedef struct SSLSrvConfigRec SSLSrvConfigRec;

入力と出力

  • 入力: ssl_config_server_new 関数により、apr_pool_t を介してメモリが割り当てられます。
  • 出力: 仮想ホストの設定情報として、mod_ssl の各フック関数や初期化処理へ提供されます。

前提条件と不変条件

  • mod_ssl の設定フェーズにおいて、仮想ホストごとにインスタンス化されます。
  • mySrvConfig マクロを通じて、Apache のサーバー構造体から型安全に取得されることが想定されています。

アルゴリズムまたは振る舞い

  • サーバー起動時の設定構築（ssl_init_ConfigureServer）や、リクエスト処理時の設定参照に使用されます。
  • ssl_engine_config.c 内で、ベースとなる設定と追加設定をマージする際のデータ保持先として機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • modules/ssl/mod_ssl.c: 接続ごとの設定取得に使用。
  • modules/ssl/ssl_engine_config.c: 設定の初期化、新規作成、マージ処理に使用。
  • 呼び出し先:
  • 構造体の内部メンバへのアクセスは各ソースファイルで行われますが、本ヘッダーファイル内での直接的な関数呼び出し関係は定義されていません。

関連データ構造

  • SSLModConfigRec: プロセス全体のグローバル設定。
  • SSLDirConfigRec: ディレクトリごとの設定。
  • SSLConnRec: 接続ごとの SSL 状態。

失敗モードと境界条件

  • メモリ割り当て失敗時: apr_palloc が失敗した場合、サーバーの起動や設定適用が中断される可能性があります。

保守担当者向けメモ

  • 本構造体は mod_ssl の内部的な設定保持用であり、公開 API ではありません。
  • 仮想ホストごとの設定項目を追加・変更する場合は、ssl_engine_config.c の初期化およびマージロジックと整合性を取る必要があります。
