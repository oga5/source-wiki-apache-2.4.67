authn_ldap_request_t

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authnz_ldap.c:86

目的

LDAP 認証・認可プロセスにおける、HTTP リクエスト単位のコンテキスト情報を保持する構造体です。

シグネチャまたは形状

typedef struct {
    const char *dn;                 /* 検索成功時に保存されるDN */
    const char *user;               /* クライアントから提供されたユーザー名 */
    const char **vals;              /* DN検索中に取得された追加属性値 */
    const char *password;           /* 認証成功時に保持されるパスワード、失敗時はNULL */
} authn_ldap_request_t;

入力と出力

  • **入力**: 認証・認可フェーズにおいて、クライアントから提供された資格情報や LDAP 検索結果が格納されます。
  • **出力**: リクエスト処理中に他の認可プロバイダーや環境変数設定関数（set_request_vars 等）から参照されます。

前提条件と不変条件

  • apr_pcalloc を使用して r->pool 上に割り当てられます。
  • 認証が成功した場合にのみ password フィールドが設定されます。

アルゴリズムまたは振る舞い

  • 認証フェーズで生成され、リクエストコンテキストとして保存されます。
  • 後続の認可フェーズにおいて、ap_get_module_config を介して取得され、DN の特定や属性値の確認に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authnz_ldap.c 内の認証・認可関数（authn_ldap_check_password 等）が、apr_pcalloc でメモリを確保し、ap_get_module_config でリクエスト設定から取得しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • authn_ldap_config_t: ディレクトリごとの設定を保持する構造体。
  • request_rec: Apache のリクエスト構造体（r->pool および r->request_config を通じて関連付け）。

失敗モードと境界条件

  • メモリ割り当て失敗時は apr_pcalloc によりプロセスが終了する可能性があります。
  • 認証失敗時には password が NULL となるため、後続処理でこのフィールドを参照する際はヌルチェックが必要です。

保守担当者向けメモ

  • 本構造体はリクエストのライフサイクルに依存するため、r->pool のクリーンアップ時に自動的に解放されます。
  • 属性値 vals の取り扱いについては、util_ldap が提供するキャッシュ機能との整合性に注意してください。
