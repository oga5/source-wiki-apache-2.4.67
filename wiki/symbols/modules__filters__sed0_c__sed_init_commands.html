sed_init_commands

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:59

目的

sed_commands_t 構造体を初期化し、sed スクリプトのコンパイルに必要なメモリプール設定および初期データ構造のセットアップを行う。

シグネチャまたは形状

apr_status_t sed_init_commands(sed_commands_t *commands, sed_err_fn_t *errfn, void *data, apr_pool_t *p)

入力と出力

  • **入力**:
  • commands: 初期化対象の sed_commands_t 構造体へのポインタ。
  • errfn: コンパイルエラー発生時に呼び出されるコールバック関数。
  • data: エラーハンドラに渡されるユーザーデータ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • 成功時は APR_SUCCESS を返す。
  • メモリ割り当て失敗時は APR_EGENERAL を返す。

前提条件と不変条件

  • commands は有効なメモリ領域を指している必要がある。
  • p は有効な apr_pool_t であること。
  • 内部的に RESIZE サイズのメモリ領域が確保され、commands->respace に割り当てられる。

アルゴリズムまたは振る舞い

1. commands 構造体をゼロクリアする。 2. 引数で渡された errfn および data を構造体に保存する。 3. ラベルテーブル（labtab）およびメモリプール（pool）を設定する。 4. apr_pcalloc を使用して RESIZE 分の作業領域を確保する。失敗時は command_errf を呼び出しエラーを報告する。 5. alloc_reptr を呼び出し、最初のコマンドノードを生成して commands->rep に割り当てる。 6. 正規表現用バッファの終端（reend）およびラベルテーブルの終端（labend）を計算・設定する。 7. canbefinal フラグを 1 に設定し、初期化を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_sed.c:215 からの呼び出しが確認されている。
  • **呼び出し先**:
  • alloc_reptr (コマンドノードの割り当て)
  • command_errf (エラー発生時の報告)
  • apr_pcalloc (APR メモリ割り当て)

関連データ構造

  • sed_commands_t: sed スクリプトのコンパイル済み状態を保持するメイン構造体。
  • sed_reptr_t: コマンドチェーンを構成するノード。
  • apr_pool_t: メモリ管理に使用されるプール。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足により NULL を返した場合、APR_EGENERAL を返す。
  • alloc_reptr が失敗した場合、APR_EGENERAL を返す。

保守担当者向けメモ

  • 本関数は sed コンパイルプロセスの最初期に実行される。
  • commands->respace のサイズ RESIZE は libsed.h 等で定義された定数に依存する。
  • commands->rep に割り当てられる最初のノードは、スクリプトの開始点として機能する。
