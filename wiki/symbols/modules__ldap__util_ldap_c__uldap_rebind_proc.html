uldap_rebind_proc

定義

  • 種別: static int 関数
  • 定義箇所: modules/ldap/util_ldap.c:86

目的

LDAP サーバーへの再接続（リバインド）が必要な際に、保存された認証情報を使用して LDAP セッションを再認証するために使用されます。

シグネチャまたは形状

static int uldap_rebind_proc(LDAP *ld, const char *url, ber_tag_t request,
                             ber_int_t msgid, void *params)

入力と出力

  • **入力**:
  • ld: LDAP セッションハンドル。
  • url: 再バインド対象の LDAP URL。
  • request, msgid: LDAP 操作に関連するタグとメッセージ ID。
  • params: util_ldap_connection_t 構造体へのポインタ（void * として渡される）。
  • **出力**:
  • int: ldap_bind_s の実行結果（成功時は LDAP_SUCCESS 等）。

前提条件と不変条件

  • params 引数は、有効な util_ldap_connection_t 構造体を指している必要があります。
  • ldc->binddn および ldc->bindpw に、有効な認証情報が格納されている必要があります。

アルゴリズムまたは振る舞い

1. params を util_ldap_connection_t 型にキャストして取得します。 2. 取得した接続構造体内の binddn および bindpw を使用し、ldap_bind_s を呼び出して同期的に LDAP サーバーへの単純認証（LDAP_AUTH_SIMPLE）を実行します。 3. バインド操作の結果を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c:96 において、ldap_set_rebind_proc の引数として登録されます。
  • **呼び出し先**: ldap_bind_s（LDAP SDK 関数）。

関連データ構造

  • util_ldap_connection_t: LDAP 接続状態および認証情報を保持する構造体。

失敗モードと境界条件

  • ldap_bind_s が失敗した場合、そのエラーコードがそのまま返されます。
  • params が NULL の場合、構造体へのアクセス時に未定義の動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は LDAP SDK の再バインドコールバックとして機能します。
  • 認証情報の更新が必要な場合は、util_ldap_connection_t の内容が最新であることを確認してください。
