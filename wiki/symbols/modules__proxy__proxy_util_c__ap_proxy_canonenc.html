ap_proxy_canonenc

定義

  • 種別: function
  • 定義箇所: modules/proxy/proxy_util.c:321

目的

URL 文字列の正規化およびエンコーディング処理を行うためのラッパー関数です。

シグネチャまたは形状

char *ap_proxy_canonenc(apr_pool_t *p, const char *x, int len,
                        enum enctype t, int forcedec,
                        int proxyreq)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • x: 正規化対象の文字列。
  • len: 文字列 x の長さ。
  • t: エンコードタイプを指定する enctype 列挙型。
  • forcedec: 強制デコードを行うかを示すフラグ（非ゼロで有効）。
  • proxyreq: プロキシリクエストの種類を示すフラグ。
  • **出力**:
  • 正規化およびエンコードされた新しい文字列（char *）。

前提条件と不変条件

  • ap_proxy_canonenc_ex が利用可能であること。
  • p は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. forcedec 引数が真であれば、内部フラグ PROXY_CANONENC_FORCEDEC を設定します。 2. 設定したフラグを引数として、ap_proxy_canonenc_ex を呼び出します。 3. ap_proxy_canonenc_ex の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: ap_proxy_canonenc_ex
  • **呼び出し元**:
  • modules/http2/mod_proxy_http2.c
  • modules/proxy/mod_proxy_ftp.c
  • modules/proxy/proxy_util.c（内部の認証情報処理など）

関連データ構造

  • apr_pool_t
  • enctype (列挙型)

失敗モードと境界条件

  • メモリ不足時には apr_pool_t の割り当て失敗により異常終了する可能性があります。
  • ap_proxy_canonenc_ex の失敗モードに依存します。

保守担当者向けメモ

  • 本関数は ap_proxy_canonenc_ex への単純なパススルーラッパーです。
  • 拡張機能が必要な場合は、直接 ap_proxy_canonenc_ex を利用することを推奨します。
