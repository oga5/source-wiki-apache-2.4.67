md_status_get_md_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_status.c:315

目的

指定された Managed Domain (MD) の詳細なステータス情報を JSON 形式で生成します。

シグネチャまたは形状

apr_status_t md_status_get_md_json(md_json_t **pjson, const md_t *md, 
                                   md_reg_t *reg, md_ocsp_reg_t *ocsp, apr_pool_t *p);

入力と出力

  • **入力**:
  • pjson: 生成された JSON オブジェクトを格納するポインタのポインタ。
  • md: ステータスを取得する対象の md_t 構造体。
  • reg: MD レジストリハンドル。
  • ocsp: OCSP ステータスレジストリ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS を返し、pjson に結果が格納されます。

前提条件と不変条件

  • p は有効なメモリプールである必要があります。
  • md、reg、ocsp は初期化済みである必要があります。

アルゴリズムまたは振る舞い

内部的に status_get_md_json 関数を呼び出し、第5引数に 1 を指定して実行します。これにより、証明書の有効期限や OCSP 状態を含む詳細な JSON 構造が構築されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_status.c:92
  • modules/md/mod_md_status.c:966
  • **呼び出し先**:
  • status_get_md_json (内部ヘルパー関数)

関連データ構造

  • md_t: 管理対象ドメインの定義。
  • md_json_t: JSON データのコンテナ。
  • md_reg_t: MD レジストリ。
  • md_ocsp_reg_t: OCSP ステータス管理。

失敗モードと境界条件

  • apr_pool_t のメモリ不足により JSON 生成が失敗する可能性があります。
  • md や reg が無効な場合、status_get_md_json の内部処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は status_get_md_json のラッパーです。詳細なロジック変更が必要な場合は status_get_md_json を参照してください。
  • MD_STATUS_WITH_SCTS マクロが 0 に設定されているため、現在 SCT 情報は出力されません。
