sig_term

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2_child.c:458

目的

子プロセスに対して終了シグナル（SIGTERM）が送信された際、グレースフルではない即時終了を準備するために呼び出されます。

シグネチャまたは形状

void sig_term(int sig)

入力と出力

  • 入力: int sig（受信したシグナル番号）
  • 出力: なし（void）

前提条件と不変条件

  • グローバル変数 shutdown_pending が 1 に設定され、is_graceful が 0 に設定されます。
  • シグナルハンドラがデフォルトの動作（SIG_DFL）にリセットされます。

アルゴリズムまたは振る舞い

1. shutdown_pending フラグを 1 に設定し、終了処理を開始させます。 2. is_graceful フラグを 0 に設定し、グレースフルシャットダウンではなく即時終了のモードへ移行します。 3. signal(SIGTERM, SIG_DFL) を実行し、以降の SIGTERM シグナルに対してはシステムのデフォルト動作が行われるように設定を戻します。

呼び出し元と呼び出し先

  • 呼び出し元: シグナルハンドラとして登録される関数です。ソース内の server/mpm/mpmt_os2/mpmt_os2_child.c:481 にて sa.sa_handler に割り当てられていることが確認できます。
  • 呼び出し先: signal（標準ライブラリ関数）。

関連データ構造

  • shutdown_pending（グローバル変数）
  • is_graceful（グローバル変数）

失敗モードと境界条件

  • 本関数はシグナルハンドラとして実行されるため、非同期シグナル安全でない関数の呼び出しには注意が必要です。
  • 呼び出し元情報の欠如: os/unix/unixd.c や server/mpm/netware/mpm_netware.c 等の他ファイルにも同名のシンボル参照がありますが、本ファイル（mpmt_os2_child.c）のコンテキストにおいて、これらが同一のロジックを指しているか、あるいはプラットフォーム固有の別実装であるかの明示的な証拠は提供されていません。

保守担当者向けメモ

  • 本関数は mpmt_os2 における終了処理のトリガーです。is_graceful を 0 に設定するため、呼び出し後は即時終了シーケンスが実行されます。
  • 他の MPM モジュールにも同名の関数が存在するため、コードの共通化やリファクタリングを行う際は、プラットフォーム固有のシグナル処理の差異に注意してください。
