update_echo_child_status

定義

  • 種別: function
  • 定義箇所: modules/echo/mod_echo.c:95

目的

Apache のスコアボード（scoreboard）上のワーカーステータスを更新し、現在処理中の接続情報やエコーバックした内容を可視化するために使用されます。

シグネチャまたは形状

static int update_echo_child_status(ap_sb_handle_t *sbh,
                                    int status, conn_rec *c,
                                    apr_bucket_brigade *last_echoed)

入力と出力

  • **入力**:
  • sbh: スコアボードハンドル。
  • status: 設定するステータスコード（SERVER_BUSY_READ 等）。
  • c: 接続レコード（NULL の場合、接続情報の更新をスキップ）。
  • last_echoed: エコーバックしたデータを含むバケットブリゲード（NULL の場合、リクエスト内容の更新をスキップ）。
  • **出力**:
  • 戻り値: 更新前のステータスコードを返します。

前提条件と不変条件

  • ap_get_scoreboard_worker(sbh) が有効な worker_score 構造体を返す必要があります。
  • ap_extended_status が有効な場合のみ、クライアントホスト名や仮想ホスト名、リクエスト内容の詳細が更新されます。

アルゴリズムまたは振る舞い

1. 指定された sbh からワーカーステータスを取得し、現在のステータスを保持します。 2. ワーカーステータスを引数 status で上書きします。 3. ap_extended_status が有効な場合、apr_time_now() で最終使用時刻を更新します。 4. c が指定されている場合、クライアントのホスト名、仮想ホスト名、およびリクエスト文字列の初期値（"ECHO "）をスコアボードにコピーします。 5. last_echoed が指定されている場合、brigade_peek を呼び出し、エコーバックした内容をスコアボードのリクエストフィールドに追記します。 6. 最後に、更新前のステータスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/echo/mod_echo.c:143 (読み込み時)
  • modules/echo/mod_echo.c:174 (書き込み時)
  • modules/echo/mod_echo.c:189 (Keep-Alive時)
  • modules/echo/mod_echo.c:193 (切断時)
  • **呼び出し先**:
  • ap_get_scoreboard_worker
  • apr_time_now
  • ap_get_remote_host
  • apr_cpystrn
  • brigade_peek

関連データ構造

  • worker_score: スコアボード内のワーカーステータスを保持する構造体。
  • conn_rec: Apache の接続情報を保持する構造体。
  • apr_bucket_brigade: データストリームを管理するバケットブリゲード。

失敗モードと境界条件

  • c が NULL の場合、クライアント情報やリクエスト文字列の更新は行われません。
  • last_echoed が NULL の場合、リクエスト内容の更新は行われません。
  • apr_cpystrn を使用しているため、スコアボードの各フィールド（client64, vhost, request）のサイズを超えるデータは切り詰められます。

保守担当者向けメモ

  • 本関数は効率化のため、c が渡されるのは初期パスのみであることを想定しています。
  • brigade_peek はスコアボードの表示領域に合わせてバッファを操作するため、sizeof(ws->request) に基づく境界チェックが重要です。
