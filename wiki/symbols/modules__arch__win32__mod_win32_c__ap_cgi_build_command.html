ap_cgi_build_command

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_win32.c:369

目的

Windows環境において、CGIスクリプトや実行ファイルの起動に必要なコマンドラインと引数を構築します。レジストリ設定やファイル先頭のシバン（#!）行を解析し、適切なインタープリタを特定します。

シグネチャまたは形状

apr_status_t ap_cgi_build_command(const char **cmd, const char ***argv,
                                  request_rec *r, apr_pool_t *p,
                                  cgi_exec_info_t *e_info)

入力と出力

  • **入力**:
  • cmd: 実行対象のファイルパス（ポインタのポインタ）。
  • r: リクエスト構造体。
  • p: メモリ割り当て用プール。
  • e_info: CGI実行情報（コマンドタイプやデタッチ設定）。
  • **出力**:
  • cmd: 最終的に実行されるコマンドパスに更新されます。
  • argv: 解析された引数配列。
  • 戻り値: 成功時は APR_SUCCESS、失敗時は APR_EBADF 等のステータスコード。

前提条件と不変条件

  • すべての動的メモリ確保は apr_pool_t に依存します。
  • r->subprocess_env に含まれる環境変数は、Windowsのコードページに合わせて正規化される必要があります。
  • e_info->cmd_type が APR_SHELLCMD でない場合、インタープリタが特定されれば APR_PROGRAM_PATH に設定されます。

アルゴリズムまたは振る舞い

1. **拡張子判定**: ファイル拡張子（.exe, .com, .bat, .cmd）を確認し、標準的な実行形式であればインタープリタを空に設定します。 2. **レジストリ検索**: 設定（ScriptInterpreterSource）に基づき、レジストリからインタープリタを検索します。 3. **シバン解析**: レジストリで特定できない場合、ファイル先頭を読み込み、#! または '! で始まる行からインタープリタを抽出します。UTF-8のBOM（\xEF\xBB\xBF）はスキップされます。 4. **コマンド構築**: split_argv を呼び出し、引数テンプレートを実際のパスで展開して配列化します。 5. **環境変数調整**: prep_string を使用し、特定の環境変数（REMOTE_* 等を除く）をWindows環境向けに変換します。 6. **終了処理**: e_info->detached を1に設定し、成功を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_cgi.c が APR_RETRIEVE_OPTIONAL_FN を介して呼び出します。
  • **呼び出し先**: get_interpreter_from_win32_registry, split_argv, prep_string, apr_file_open, apr_file_read 等。

関連データ構造

  • win32_dir_conf: ディレクトリごとの設定（インタープリタ解決戦略を保持）。
  • cgi_exec_info_t: CGI実行時のプロセス制御情報。
  • request_rec: Apacheのリクエストコンテキスト。

失敗モードと境界条件

  • **ファイル読み込み失敗**: apr_file_open または read が失敗した場合、エラーログを出力して終了します。
  • **実行形式不明**: インタープリタが特定できず、シバン行も見つからない場合、APR_EBADF を返します。
  • **レジストリ検索失敗**: registry-strict 設定時に動詞が見つからない場合、エラーログを出力します。

保守担当者向けメモ

  • prep_string は現在、UTF-8からWindowsローカルコードページへの変換において限定的な実装です。
  • split_argv 内の引数解析は複雑なエスケープ処理を含むため、変更時はWindowsのコマンドライン引数ルールに注意してください。
