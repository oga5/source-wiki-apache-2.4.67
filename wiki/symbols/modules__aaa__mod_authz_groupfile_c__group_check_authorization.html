group_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_groupfile.c:136

目的

require group ディレクティブに基づき、リクエストユーザーが指定されたグループのいずれかに所属しているかを検証し、認可状態（AUTHZ_GRANTED または AUTHZ_DENIED）を決定する。

シグネチャまたは形状

static authz_status group_check_authorization(request_rec *r,
                                              const char *require_args,
                                              const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。r->user を使用してユーザー名を特定。
  • parsed_require_args: ap_expr_info_t 型の式情報。評価後にグループ名のリストとして使用。
  • **出力**:
  • authz_status: 認可結果。ユーザー不在時は AUTHZ_DENIED_NO_USER、グループファイル未設定や検証失敗時は AUTHZ_DENIED、一致時は AUTHZ_GRANTED を返す。

前提条件と不変条件

  • r->user が設定されていること。
  • conf->groupfile（AuthGroupFile ディレクティブで指定）が有効なファイルパスを保持していること。

アルゴリズムまたは振る舞い

1. ユーザー名が未設定の場合、AUTHZ_DENIED_NO_USER を返す。 2. 設定されたグループファイルが存在しない場合、エラーをログ出力し AUTHZ_DENIED を返す。 3. groups_for_user を呼び出し、当該ユーザーが所属する全グループを apr_table_t に取得する。 4. グループファイルにユーザーが存在しない場合、AUTHZ_DENIED を返す。 5. ap_expr_str_exec を用いて parsed_require_args を評価し、要求されたグループ名のリストを得る。 6. 要求された各グループ名に対し、apr_table_get を用いてユーザーの所属グループテーブルと照合する。 7. 一致するグループがあれば AUTHZ_GRANTED を返し、すべて不一致であれば AUTHZ_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_groupfile.c:295（authz_group_provider 構造体を通じて登録）。
  • **呼び出し先**:
  • groups_for_user: ユーザーの所属グループ一覧を取得。
  • ap_expr_str_exec: 認可条件式の評価。
  • ap_getword_conf: 要求されたグループ名のトークン分割。
  • ap_log_rerror: エラーおよびデバッグ情報の出力。

関連データ構造

  • authz_groupfile_config_rec: ディレクトリごとの設定（グループファイルパス等）を保持。
  • apr_table_t: ユーザーが所属するグループのキャッシュ。

失敗モードと境界条件

  • **ユーザー未認証**: r->user が NULL の場合、即座に拒否。
  • **ファイルI/Oエラー**: groups_for_user が失敗した場合、エラーログを出力して拒否。
  • **グループ不一致**: ユーザーがグループファイルに存在しない、または指定されたグループに所属していない場合、拒否。
  • **式評価エラー**: ap_expr_str_exec が失敗した場合、エラーログを出力して拒否。

保守担当者向けメモ

  • 本関数はリクエストごとにグループファイルを走査するため、大規模なグループファイルではパフォーマンスへの影響を考慮すること。
  • groups_for_user が返す apr_table_t はリクエストプール（r->pool）を使用しており、メモリ管理は適切に行われている。
