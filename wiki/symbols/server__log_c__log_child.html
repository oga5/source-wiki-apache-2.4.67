log_child

定義

  • 種別: function
  • 定義箇所: server/log.c:314

目的

ErrorLog ディレクティブで指定された外部プログラム（パイプ経由のログ出力）を子プロセスとして起動し、ログ書き込み用のパイプを確立する。

シグネチャまたは形状

static int log_child(apr_pool_t *p, const char *progname,
                     apr_file_t **fpin, apr_cmdtype_e cmdtype,
                     int dummy_stderr)

入力と出力

  • **入力**:
  • p: プロセス管理およびメモリ割り当てに使用するプール。
  • progname: 実行するコマンド文字列。
  • cmdtype: コマンド実行形式（apr_cmdtype_e）。
  • dummy_stderr: 標準エラー出力を標準出力へリダイレクトするかどうかのフラグ。
  • **出力**:
  • fpin: 成功時に子プロセスの標準入力へ接続されたパイプの書き込み用ファイル記述子。
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • progname は apr_tokenize_to_argv により解析可能な形式であること。
  • 呼び出し元は、返された fpin を通じてログデータを書き込む。
  • 子プロセスは ap_server_root をカレントディレクトリとして起動される。

アルゴリズムまたは振る舞い

1. apr_procattr_create でプロセス属性オブジェクトを作成し、ディレクトリ設定、コマンドタイプ、I/O 設定（APR_FULL_BLOCK）を適用する。 2. log_child_errfn をエラーハンドラとして登録する。 3. apr_tokenize_to_argv でコマンド文字列を引数配列に分解する。 4. dummy_stderr が真の場合、標準エラー出力を標準出力にリダイレクトする設定を追加する。 5. apr_proc_create で子プロセスを起動する。 6. 成功時、apr_pool_note_subprocess を呼び出し、親プロセスの終了時に子プロセスが適切に処理されるよう登録する。 7. 子プロセスの標準入力パイプを fpin に格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/log.c:393
  • server/log.c:1939
  • **呼び出し先**:
  • apr_procattr_* 系関数（APR プロセス管理）
  • apr_tokenize_to_argv
  • apr_pcalloc
  • apr_proc_create
  • apr_pool_note_subprocess
  • log_child_errfn（エラーハンドラとして登録）

関連データ構造

  • apr_procattr_t: プロセス起動時の属性を保持。
  • apr_proc_t: 起動された子プロセスの状態を保持。
  • apr_pool_t: メモリ管理およびサブプロセス管理のライフサイクルを制御。

失敗モードと境界条件

  • apr_procattr_* の設定や apr_proc_create が失敗した場合、そのステータスコードがそのまま返される。
  • progname が不正な場合、apr_tokenize_to_argv または apr_proc_create でエラーが発生する。
  • 子プロセスの起動に失敗した場合、fpin は更新されない。

保守担当者向けメモ

  • 本関数は ErrorLog "|..." のようなパイプ経由のログ出力に特化している。
  • fpin は書き込み専用パイプとして提供される。読み込み側のパイプは保持されないため、close_handle_in_child の呼び出しは不要である。
  • 子プロセスの異常終了時の再起動ロジックは、この関数ではなく piped_log_maintenance 等の別レイヤーで管理される。
