netware_dir_config

定義

  • 種別: struct
  • 定義箇所: modules/arch/netware/mod_netware.c:39

目的

NetWare プラットフォームにおけるディレクトリ単位の CGI 実行設定を保持するための構造体です。ファイル拡張子と CGI エンジン（NLM）の関連付け、実行モード、および環境変数を管理します。

シグネチャまたは形状

typedef struct {
    apr_table_t *file_type_handlers;    /* CGI map from file types to CGI modules */
    apr_table_t *file_handler_mode;     /* CGI module mode (spawn in same address space or not) */
    apr_table_t *extra_env_vars;        /* Environment variables to be added to the CGI environment */
} netware_dir_config;

入力と出力

  • **入力**: 設定ディレクティブ（CGIMapExtension 等）を通じて各テーブルにデータが格納されます。
  • **出力**: CGI リクエスト処理時に、ap_cgi_build_command 等の関数が本構造体を参照し、NLM の実行パスやアドレス空間設定を決定します。

前提条件と不変条件

  • 各フィールドは apr_table_t 型であり、Apache のメモリプール（apr_pool_t）によって管理される必要があります。
  • NetWare 環境下でのみ使用されることを前提としています。

アルゴリズムまたは振る舞い

1. **設定**: set_extension_map ハンドラにより、拡張子をキーとして NLM パスや実行モードがテーブルに登録されます。 2. **検索**: CGI リクエスト発生時、リクエストされたファイルの拡張子をキーにして file_type_handlers から対応する NLM を特定します。 3. **実行制御**: file_handler_mode を参照し、当該 NLM を Apache と同一のアドレス空間で実行するか、分離するかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_netware_dir_config（メモリ確保）
  • merge_netware_dir_configs（設定のマージ）
  • set_extension_map（設定値の書き込み）
  • ap_cgi_build_command（設定値の読み取り）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_table_t: 設定情報を保持するために使用される APR のテーブル構造体。

失敗モードと境界条件

  • 設定が未定義の場合、テーブル参照時に NULL が返される可能性があるため、各関数内での適切なチェックが必要です。
  • メモリ不足時には apr_palloc が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は NetWare 特有の「アドレス空間の分離」を制御する重要な役割を担っています。
  • file_handler_mode の値が、NetWare のプロセス起動ロジック（e_info->addrspace）に直接影響を与える点に注意してください。
