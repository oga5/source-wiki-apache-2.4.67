h1test_echo_handler

定義

  • 種別: function
  • 定義箇所: test/modules/http1/mod_h1test/mod_h1test.c:45

目的

HTTPリクエストボディをそのままレスポンスとして返す「エコー」機能を提供し、HTTP/1.1プロトコルスタックの動作およびトレーラーヘッダーの伝搬を検証する。

シグネチャまたは形状

static int h1test_echo_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r（Apacheリクエスト構造体）
  • **出力**: int（Apacheハンドラステータスコード: OK, DECLINED, または AP_FILTER_ERROR）

前提条件と不変条件

  • リクエストのハンドラ名が h1test-echo であること。
  • リクエストメソッドが GET または POST であること。
  • チャンク転送エンコーディング（r->chunked = 1）が強制的に有効化される。

アルゴリズムまたは振る舞い

1. **判定**: ハンドラ名とメソッドを検証し、不一致の場合は DECLINED を返す。 2. **初期化**: レスポンスステータスを 200 に設定し、コンテンツタイプを決定する。 3. **ボディ処理**: ap_get_client_block をループで呼び出し、リクエストボディを読み取る。 4. **ストリーミング**: 読み取ったデータを apr_brigade_write でバッファし、ap_pass_brigade を通じてクライアントへ即座に書き出す。 5. **トレーラー処理**: リクエストにトレーラーが含まれる場合、特定のレスポンスヘッダー（h1test-trailers-in, h1test-add-trailer）を付与する。 6. **終了**: EOS（End of Stream）バケットを挿入し、フィルタチェーンを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: Apacheのハンドラフックメカニズム（ap_hook_handler により登録）。
  • **呼び出し先**:
  • ap_get_client_block, ap_pass_brigade, ap_setup_client_block（ApacheプロトコルAPI）
  • apr_brigade_create, apr_brigade_write, apr_bucket_eos_create（APRバケットAPI）
  • ap_log_rerror（ログ出力）

関連データ構造

  • request_rec: リクエスト情報およびレスポンスヘッダー（trailers_out）の保持に使用。
  • apr_bucket_brigade: ボディデータのストリーミング用バッファ。

失敗モードと境界条件

  • ap_get_client_block や apr_brigade_write が失敗した場合、cleanup ラベルへ遷移し、エラーログを出力して AP_FILTER_ERROR を返す。

保守担当者向けメモ

  • 本関数はテスト専用であり、プロダクション環境での利用は想定されていない。
  • r->clength = -1 を設定することで、レスポンスの長さを不定とし、チャンク転送を強制している点に注意すること。
