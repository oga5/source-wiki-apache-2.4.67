get_accept_filter

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/child.c:299

目的

指定されたプロトコルに関連付けられた accept_filter 設定を解決し、対応する列挙型 accept_filter_e を返します。

シグネチャまたは形状

static accept_filter_e get_accept_filter(const char *protocol)

入力と出力

  • **入力**: protocol (const char *) - フィルタを検索する対象のプロトコル名。
  • **出力**: accept_filter_e 型の列挙値（ACCEPT_FILTER_CONNECT または ACCEPT_FILTER_NONE）。

前提条件と不変条件

  • ap_server_conf が初期化されており、module_config から core_server_config が取得可能であること。
  • core_sconf->accf_map にプロトコルとフィルタ名のマッピングが格納されていること。

アルゴリズムまたは振る舞い

1. ap_get_core_module_config を使用してコアサーバー設定を取得します。 2. apr_table_get を用いて、指定された protocol に対応するフィルタ名を accf_map から検索します。 3. フィルタ名が見つからない場合、警告ログを出力し ACCEPT_FILTER_NONE を返します。 4. フィルタ名が "data" の場合、情報ログを出力し "connect" として扱います（ACCEPT_FILTER_CONNECT を返します）。 5. フィルタ名が "connect" の場合、ACCEPT_FILTER_CONNECT を返します。 6. フィルタ名が "none" の場合、ACCEPT_FILTER_NONE を返します。 7. 上記以外（未認識のフィルタ名）の場合、警告ログを出力し ACCEPT_FILTER_NONE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/child.c:383 (リスナー・スレッドの初期化処理内)
  • **呼び出し先**:
  • ap_get_core_module_config
  • apr_table_get
  • ap_log_error
  • strcmp

関連データ構造

  • core_server_config: サーバーのコア設定を保持する構造体。
  • accept_filter_e: 接続フィルタの種類を定義する列挙型。

失敗モードと境界条件

  • プロトコル名が accf_map に存在しない場合、デフォルトで ACCEPT_FILTER_NONE が選択されます。
  • 不正なフィルタ名が設定されている場合、警告ログが記録され、安全側に倒して ACCEPT_FILTER_NONE が返されます。

保守担当者向けメモ

  • "data" フィルタは現在サポートされていないため、"connect" にフォールバックするロジックが含まれています。
  • accf_map の内容は設定ファイル（httpd.conf 等）の AcceptFilter ディレクティブによって動的に決定されます。
