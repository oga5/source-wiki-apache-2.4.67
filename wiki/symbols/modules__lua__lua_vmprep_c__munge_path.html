munge_path

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:274

目的

Lua の package.path または package.cpath を、Apache の設定に基づいたディレクトリパスおよび指定された追加パスで更新し、Lua スクリプトの検索環境を調整する。

シグネチャまたは形状

static void munge_path(lua_State *L,
                       const char *field,
                       const char *sub_pat,
                       const char *rep_pat,
                       apr_pool_t *pool,
                       apr_array_header_t *paths,
                       const char *file)

入力と出力

  • **入力**:
  • L: Lua 状態。
  • field: 操作対象のフィールド名（path または cpath）。
  • sub_pat, rep_pat: パス置換のためのパターン文字列。
  • pool: メモリ割り当て用プール。
  • paths: 追加するパスの配列。
  • file: ベースとなるファイルパス。
  • **出力**: なし（副作用として Lua 状態内の package テーブルを更新）。

前提条件と不変条件

  • L は有効な lua_State であること。
  • package テーブルが Lua のグローバル環境に存在すること。
  • 指定された field が package テーブル内に存在すること。

アルゴリズムまたは振る舞い

1. Lua の package テーブルから指定された field（パス文字列）を取得する。 2. ap_make_dirstr_parent を使用して file の親ディレクトリを取得する。 3. sub_pat を結合して置換用パターンを生成し、luaL_gsub で既存のパス文字列を置換する。 4. 置換後のパス文字列に対し、paths 配列に含まれるパスをセミコロン（;）区切りで連結する。 5. 最終的な文字列を Lua の package[field] に再設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_vmprep.c 内の vm_construct 関数（258行目、345行目、352行目付近）。
  • **呼び出し先**:
  • lua_getglobal, lua_getfield, lua_setfield, lua_tostring, lua_pushstring, lua_pop, luaL_gsub (Lua C API)
  • ap_make_dirstr_parent (Apache API)
  • apr_pstrcat, apr_array_pstrcat (APR API)

関連データ構造

  • lua_State: Lua 仮想マシン状態。
  • apr_pool_t: メモリ管理用プール。
  • apr_array_header_t: 追加パスを保持する配列。

失敗モードと境界条件

  • package テーブルや対象フィールドが存在しない場合、Lua スタックの不整合やエラーが発生する可能性がある。
  • apr_pstrcat 等のメモリ割り当てが失敗した場合、APR の規約に従いプロセスが終了する可能性がある。
  • パスの形式が期待と異なる場合、Lua のモジュールロード失敗を招く。

保守担当者向けメモ

  • 本関数は文字列操作を多用するため、パスの連結時にセミコロンの重複や欠落がないか注意が必要。
  • lua_pop によるスタック操作が厳密に行われているため、変更を加える際はスタックバランスを維持すること。
