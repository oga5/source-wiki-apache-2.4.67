ssl_mutex_off

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_mutex.c:99

目的

セッションキャッシュ保護のために使用されるグローバル Mutex を解放します。

シグネチャまたは形状

int ssl_mutex_off(server_rec *s)

入力と出力

  • **入力**: server_rec *s (現在のサーバー設定を含むサーバーレコード)
  • **出力**: int (成功時に TRUE、失敗時に FALSE を返します)

前提条件と不変条件

  • myModConfig(s) を通じて取得される SSLModConfigRec 構造体内の pMutex が、初期化済みである必要があります。
  • 本関数は、ssl_mutex_on によって取得されたロックを解放するために呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. myModConfig(s) を呼び出し、現在のサーバー設定からモジュール設定構造体 SSLModConfigRec を取得します。 2. apr_global_mutex_unlock(mc->pMutex) を実行し、グローバル Mutex のロックを解放します。 3. 解放処理が APR_SUCCESS でない場合、ap_log_error を使用して警告ログを出力し、FALSE を返します。 4. 正常に解放できた場合、TRUE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_scache.c: セッションキャッシュ操作の終了時に複数箇所から呼び出されています。
  • **呼び出し先**:
  • apr_global_mutex_unlock (APR ライブラリ)
  • ap_log_error (Apache HTTP Server API)

関連データ構造

  • SSLModConfigRec: モジュール設定を保持する構造体。pMutex メンバを介してロック対象を管理します。
  • server_rec: Apache のサーバー設定構造体。

失敗モードと境界条件

  • **ロック解放失敗**: apr_global_mutex_unlock がエラーを返した場合、ログ出力を行い FALSE を返します。これは通常、OS リソースの枯渇や Mutex の状態不整合を示唆します。

保守担当者向けメモ

  • 本関数は ssl_mutex_on と対になる操作です。キャッシュプロバイダがスレッドセーフでない場合にのみ有効に機能するよう設計されています。
  • ログ出力される APLOGNO(02027) は、ロック解放失敗時のデバッグに使用してください。
