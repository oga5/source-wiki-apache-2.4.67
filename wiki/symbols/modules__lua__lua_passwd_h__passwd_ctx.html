passwd_ctx

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_passwd.h:66

目的

パスワードハッシュ生成プロセスにおけるコンテキスト情報を保持し、入力データ、アルゴリズム選択、および出力先をカプセル化するために使用されます。

シグネチャまたは形状

struct passwd_ctx {
    apr_pool_t *pool;
    char *out;
    const char *passwd;
    int alg;
    /* その他、実装依存のフィールドを含む */
};

入力と出力

  • **入力**:
  • pool: メモリ管理用 APR プール。
  • passwd: ハッシュ化対象のパスワード文字列。
  • alg: 使用するハッシュアルゴリズムの識別子（ALG_APMD5, ALG_APSHA, ALG_BCRYPT, ALG_CRYPT 等）。
  • **出力**:
  • out: 生成されたハッシュ文字列を格納するバッファ。

前提条件と不変条件

  • passwd_ctx 内の pool は、エラーメッセージ生成（apr_psprintf 等）のために有効な APR プールである必要があります。
  • パスワードの長さは MAX_PASSWD_LEN（256）を超えてはなりません。
  • 呼び出し元は、ハッシュ生成関数を呼び出す前に構造体を適切に初期化する必要があります。

アルゴリズムまたは振る舞い

1. 呼び出し元が passwd_ctx を初期化し、メモリプールと入力パスワードを設定します。 2. mk_password_hash(passwd_ctx *ctx) が呼び出されます。 3. ctx->alg に基づき、apr_md5_encode、apr_sha1_base64、またはシステム標準の crypt() 等へ処理が分岐します。 4. 生成されたハッシュ文字列が ctx->out に書き込まれます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_passwd.c の mk_password_hash、modules/lua/lua_request.c、support/htpasswd.c 等からの参照が確認されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。ハッシュ生成ロジックは mk_password_hash 等の関数に委譲されます。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • アルゴリズム定数: ALG_APMD5, ALG_APSHA, ALG_BCRYPT, ALG_CRYPT。
  • エラーコード: ERR_PWMISMATCH, ERR_RANDOM, ERR_ALG_NOT_SUPP 等。

失敗モードと境界条件

  • **アルゴリズム未サポート**: 指定された alg が環境で利用できない場合、ERR_ALG_NOT_SUPP が発生します。
  • **メモリ不足**: APR プールからのメモリ割り当て失敗時にエラーが発生する可能性があります。
  • **プラットフォーム依存**: CRYPT_ALGO_SUPPORTED 等のフラグにより、OS によって利用可能なアルゴリズムが制限されます。

保守担当者向けメモ

  • 本構造体は lua_passwd.c と密接に関連しており、変更時は両ファイルの整合性を確認してください。
  • pool のライフサイクル管理は呼び出し元の責任です。
  • MAX_PASSWD_LEN の制約を遵守してください。
