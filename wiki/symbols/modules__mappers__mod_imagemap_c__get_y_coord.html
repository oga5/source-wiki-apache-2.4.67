get_y_coord

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_imagemap.c:249

目的

クライアントから送信されたクエリ文字列（args）から、カンマ区切りで指定された Y 座標値を抽出して double 型で返します。

シグネチャまたは形状

static double get_y_coord(const char *args)

入力と出力

  • **入力**: args (クエリ文字列。例: "100,200")
  • **出力**: 抽出された Y 座標値（double）。エラー時または座標が見つからない場合は -1 を返します。

前提条件と不変条件

  • args が NULL の場合は -1 を返します。
  • 座標値はカンマ（,）の後に続く数値として解釈されます。
  • 戻り値 -1 は、解析失敗またはカンマが存在しないことを示す境界値として使用されます。

アルゴリズムまたは振る舞い

1. ap_strchr_c を使用して、文字列内の最初のカンマ位置を特定します。 2. カンマが見つかった場合、その直後の文字から開始し、apr_isdigit を用いて最初の数字が現れるまでポインタを進めます。 3. strtod を使用して、数字の開始位置から数値を変換します。 4. 変換が成功した場合（endptr > start_of_y）、その値を返します。 5. カンマが存在しない、または数値への変換が成立しなかった場合は -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_imagemap.c:624 にて r->args を引数として呼び出されています。
  • **呼び出し先**: ap_strchr_c（Apache API）、apr_isdigit（APR）、strtod（標準ライブラリ）を呼び出します。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **カンマなし**: args にカンマが含まれない場合、-1 を返します。
  • **数値なし**: カンマの後に数字が続かない場合、strtod の変換が成立せず -1 を返します。
  • **NULL 入力**: args が NULL の場合、即座に -1 を返します。

保守担当者向けメモ

  • 本関数は get_x_coord と対になる設計です。
  • strtod の結果が -1 である場合、エラーと有効な座標値の区別が困難になる可能性があるため、利用時には注意が必要です。
