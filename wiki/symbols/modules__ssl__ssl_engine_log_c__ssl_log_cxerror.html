ssl_log_cxerror

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_log.c:224

目的

接続（conn_rec）に関連するコンテキストにおいて、X.509 証明書情報を含むエラーメッセージを Apache のログシステムへ出力する。

シグネチャまたは形状

void ssl_log_cxerror(const char *file, int line, int level, apr_status_t rv,
                     conn_rec *c, X509 *cert, const char *fmt, ...)

入力と出力

  • **入力**:
  • file, line: ログ出力元のソースファイル名と行番号。
  • level: ログレベル（APLOG_ERR 等）。
  • rv: APR ステータスコード。
  • c: 接続コンテキスト（conn_rec）。
  • cert: ログに付与する X.509 証明書オブジェクト（NULL 可）。
  • fmt, ...: フォーマット文字列と可変長引数。
  • **出力**: なし（戻り値 void）。

前提条件と不変条件

  • c->pool が有効なメモリプールとして存在していること。
  • APLOG_IS_LEVEL マクロにより、指定されたログレベルが現在のサーバー設定で有効な場合にのみ処理が実行される。

アルゴリズムまたは振る舞い

1. mySrvFromConn(c) を介して接続に関連付けられたサーバー設定を取得し、ログレベルのチェックを行う。 2. チェックを通過した場合、可変長引数リスト va_list を初期化する。 3. ssl_log_cert_error を呼び出し、以下の情報を渡してログ出力を委譲する。

  • 接続コンテキスト c およびそのメモリプール c->pool。
  • 証明書オブジェクト cert。
  • フォーマット済みのメッセージ。

    • 4. va_end で引数リストを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_kernel.c
  • modules/ssl/ssl_engine_ocsp.c
  • **呼び出し先**:
  • ssl_log_cert_error（内部関数）

関連データ構造

  • conn_rec: Apache の接続コンテキスト構造体。
  • X509: OpenSSL の証明書構造体。
  • apr_pool_t: メモリ管理に使用される APR プール。

失敗モードと境界条件

  • cert が NULL の場合、ssl_log_cert_error 内で証明書情報の出力はスキップされる。
  • ログレベルが設定値より低い場合、関数は何も出力せずに即座に終了する。

保守担当者向けメモ

  • 本関数は ssl_log_xerror や ssl_log_rxerror と同様のラッパーであり、ssl_log_cert_error へのインターフェースを提供する。
  • 証明書情報の文字列化には c->pool が使用されるため、呼び出し元でプールが解放されていないことを確認すること。
