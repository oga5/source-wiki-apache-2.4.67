ngh_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.c:1427

目的

HTTP/1 のヘッダーを HTTP/2 用の nghttp2_nv 形式へ変換する処理において、変換プロセス中の状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct ngh_ctx {
    apr_pool_t *p;
    int unsafe;
    h2_ngheader *ngh;
    apr_status_t status;
} ngh_ctx;

入力と出力

  • **入力**: 変換処理に必要なメモリプール (p)、安全でないヘッダーを許可するかどうかのフラグ (unsafe)、変換結果を格納する構造体 (ngh)。
  • **出力**: 変換処理の成否を示すステータスコード (status)。

前提条件と不変条件

  • p は、変換処理中に使用される有効な apr_pool_t である必要があります。
  • ngh は、変換されたヘッダーを格納するために初期化されている必要があります。

アルゴリズムまたは振る舞い

本構造体は、ヘッダー変換関数（add_header 等）の引数として渡され、変換の進捗と結果を追跡します。add_header はこのコンテキストを参照し、ヘッダーのバリデーションや変換を行い、その結果を ngh に追加し、エラーが発生した場合は status に値を設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_util.c 内のヘッダー変換ロジック（add_header 関数など）から参照されます。
  • **呼び出し先**: 情報なし。

関連データ構造

  • h2_ngheader: 変換後の HTTP/2 ヘッダーを保持する構造体。
  • apr_pool_t: メモリ管理に使用される Apache Portable Runtime のプール。

失敗モードと境界条件

  • status フィールドにより、ヘッダーの追加処理中に発生したエラー（メモリ不足やバリデーション失敗など）を伝播させます。

保守担当者向けメモ

  • 本構造体は h2_util.c 内のローカルな変換処理専用です。
  • unsafe フラグの具体的な判定基準については、add_header 関数の実装を参照してください。
