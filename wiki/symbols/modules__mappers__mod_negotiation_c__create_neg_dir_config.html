create_neg_dir_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_negotiation.c:69

目的

Apache HTTP Server のコンテンツネゴシエーション機能において、ディレクトリ単位の設定構造体（neg_dir_config）を新規作成し、初期化する。

シグネチャまたは形状

static void *create_neg_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体の割り当てに使用するメモリプール。
  • char *dummy: Apache の設定フック仕様に基づくダミー引数（使用されない）。
  • **出力**:
  • void *: 初期化済みの neg_dir_config 構造体へのポインタ。

前提条件と不変条件

  • apr_palloc を使用してメモリを割り当てるため、メモリ不足時は例外が発生する可能性がある。
  • 返される構造体の forcelangpriority は FLP_UNDEF に、language_priority は NULL に初期化される。

アルゴリズムまたは振る舞い

1. apr_palloc を呼び出し、neg_dir_config 構造体サイズ分のメモリをプール p から確保する。 2. 確保した構造体のメンバ forcelangpriority に FLP_UNDEF を代入する。 3. メンバ language_priority を NULL に設定する。 4. 初期化した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_negotiation.c:3217（ディレクトリ設定作成用フックとして登録されている）。
  • **呼び出し先**: apr_palloc（APR メモリ管理関数）。

関連データ構造

  • neg_dir_config: 本関数が生成・初期化するディレクトリごとのネゴシエーション設定構造体。

失敗モードと境界条件

  • メモリプール p が無効な場合、apr_palloc の挙動に依存する。

保守担当者向けメモ

  • 本関数は Apache の設定システムから呼び出されるコールバック関数である。
  • 新たな設定項目を neg_dir_config に追加した場合は、必ず本関数で初期値を設定すること。
