create_authnz_ldap_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_ldap.c:331

目的

LDAP 認証・認可モジュール用のディレクトリ単位の設定構造体 (authn_ldap_config_t) をメモリ上に確保し、デフォルト値で初期化する。

シグネチャまたは形状

void *create_authnz_ldap_dir_config(apr_pool_t *p, char *d)

入力と出力

  • **入力**:
  • p: 設定を割り当てるためのメモリプール (apr_pool_t)。
  • d: 設定対象のディレクトリパス（本関数内では未使用）。
  • **出力**:
  • 初期化済みの authn_ldap_config_t 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全メンバは初期状態でゼロクリアされる。
  • スレッド環境下では、設定更新用のミューテックス (sec->lock) が作成される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して authn_ldap_config_t 構造体をメモリプール上に確保する。 2. スレッドセーフな環境であれば、apr_thread_mutex_create でミューテックスを初期化する。 3. groupattr および subgroupclasses 用の配列を apr_array_make で初期化する。 4. LDAP 接続および検索に関連するデフォルト値を設定する（例: bind_authoritative = 1, maxNestingDepth = 10, deref = always）。 5. sgAttributes 用のメモリを確保する。 6. 初期化された構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authnz_ldap.c:2093 (ディレクトリ設定作成用フックとして登録)。
  • **呼び出し先**: apr_pcalloc, apr_thread_mutex_create, apr_array_make (すべて APR ライブラリ)。

関連データ構造

  • authn_ldap_config_t: 本関数が初期化する主要な設定構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc が失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性がある。
  • スレッド無効時: APR_HAS_THREADS が定義されていない場合、ミューテックスの初期化はスキップされる。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。
  • 新しい設定項目を追加した場合は、必ず本関数でデフォルト値を設定すること。
  • sec->lock はマルチスレッド環境での設定更新時に使用されるため、設定変更を行う際は必ずこのロックを考慮する必要がある。
