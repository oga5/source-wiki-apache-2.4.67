set_ftp_directory_charset

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:101

目的

Apache HTTP Server の設定ディレクティブ ProxyFtpDirCharset を介して、FTP ディレクトリ一覧表示時に使用する文字セットを設定する。

シグネチャまたは形状

static const char *set_ftp_directory_charset(cmd_parms *cmd, void *dconf, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dconf: ディレクトリ単位の設定構造体 (proxy_ftp_dir_conf)。
  • arg: 設定ファイルから読み込まれた文字セット名を表す文字列。
  • **出力**:
  • 成功時は NULL を返す。

前提条件と不変条件

  • dconf は proxy_ftp_dir_conf 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. 引数 dconf を proxy_ftp_dir_conf 型のポインタにキャストする。 2. dconf が保持する ftp_directory_charset メンバに、引数 arg で渡された文字列を代入する。 3. NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ftp.c:2102 において、AP_INIT_TAKE1 マクロを通じて ProxyFtpDirCharset ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: 外部関数への呼び出しは含まれない。

関連データ構造

  • proxy_ftp_dir_conf: FTP プロキシのディレクトリ設定を保持する構造体。本関数はこの構造体の ftp_directory_charset フィールドを更新する。

失敗モードと境界条件

  • 入力値の妥当性チェックは本関数内では行われない。

保守担当者向けメモ

  • 本関数は設定フェーズで呼び出される。設定値の検証が必要な場合は、本関数内または設定マージ時に追加すること。
