send_data

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:243

目的

FastCGI バックエンド接続に対して、iovec 配列で指定されたデータをソケット経由で送信し、送信バイト数を統計情報に反映させる。

シグネチャまたは形状

apr_status_t send_data(proxy_conn_rec *conn, struct iovec *vec, int nvec, apr_size_t *len)

入力と出力

  • **入力**:
  • conn: バックエンド接続情報（proxy_conn_rec）。
  • vec: 送信データを含む iovec 配列。
  • nvec: vec 配列の要素数。
  • **出力**:
  • len: 実際に送信された合計バイト数。
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • conn->sock が有効なソケットであること。
  • conn->worker->s が統計情報更新のために有効であること。

アルゴリズムまたは振る舞い

1. vec 内の全データの合計サイズ（to_write）を算出する。 2. apr_socket_sendv を呼び出し、ソケットへデータを書き込む。 3. 送信が部分的（short write）であった場合、送信済みバイト数に応じて vec のポインタと長さを更新し、残りのデータを送信するループを継続する。 4. 送信完了後、conn->worker->s->transferred に送信バイト数を加算する。 5. 最終的な送信バイト数を len に格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_fcgi.c 内の各所（行 346, 510, 525, 725, 749）。
  • **呼び出し先**: apr_socket_sendv（APR ライブラリ）。
  • 注: modules/http2/h2_session.c に同名のシンボルへの言及があるが、本関数との直接的な呼び出し関係はソースコード上確認できない。

関連データ構造

  • proxy_conn_rec: プロキシ接続状態を保持する構造体。
  • iovec: 送信データのバッファ配列。

失敗モードと境界条件

  • apr_socket_sendv が APR_SUCCESS 以外の値を返した場合、ループを中断しその時点でのエラーコードを返す。
  • nvec が 0 の場合、ループは実行されず len は 0 となる。

保守担当者向けメモ

  • 本関数はブロッキング I/O を前提としている。
  • conn->worker->s->transferred への加算は、マルチスレッド環境下でのアトミック性について proxy_util 側の実装に依存する。
