regkey_cleanup

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:46

目的

ap_regkey_t 構造体に関連付けられた Windows レジストリキーハンドルを安全に解放し、リソースリークを防止するクリーンアップハンドラです。

シグネチャまたは形状

static apr_status_t regkey_cleanup(void *key)

入力と出力

  • **入力**: void *key - ap_regkey_t 型のポインタとしてキャストされるレジストリキーコンテナ。
  • **出力**: apr_status_t - 常に APR_SUCCESS を返します。

前提条件と不変条件

  • 入力された key は ap_regkey_t 構造体へのポインタである必要があります。
  • regkey->hkey が有効なハンドルであるか、または INVALID_HANDLE_VALUE であることが前提です。

アルゴリズムまたは振る舞い

1. 引数 key を ap_regkey_t * にキャストします。 2. regkey->hkey が存在し、かつ INVALID_HANDLE_VALUE でないことを確認します。 3. Win32 API の RegCloseKey を呼び出し、レジストリハンドルを閉じます。 4. regkey->hkey を INVALID_HANDLE_VALUE に更新し、二重解放を防止します。 5. APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • os/win32/ap_regkey.c:115: apr_pool_cleanup_register の引数として登録されます。
  • os/win32/ap_regkey.c:123: ap_regkey_close 関数から直接呼び出されます。
  • **呼び出し先**:
  • RegCloseKey (Win32 API)

関連データ構造

  • ap_regkey_t: レジストリキーハンドル (HKEY) と関連する apr_pool_t を保持する構造体。

失敗モードと境界条件

  • regkey->hkey が既に INVALID_HANDLE_VALUE である場合、RegCloseKey は呼び出されず、安全に終了します。
  • 本関数自体はエラーを返さない設計ですが、RegCloseKey の内部的な失敗については考慮されていません。

保守担当者向けメモ

  • 本関数は apr_pool_cleanup_register を介して自動的に呼び出されることを想定しています。
  • ap_regkey_close から手動で呼び出す際は、その後 apr_pool_cleanup_kill を使用してプールからの登録を解除し、二重クリーンアップを防ぐ必要があります。
