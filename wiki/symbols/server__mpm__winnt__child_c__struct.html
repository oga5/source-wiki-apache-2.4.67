目的

Windows API の GetAcceptExSockaddrs 関数を呼び出すための関数ポインタ型を定義しています。

シグネチャまたは形状

typedef VOID (WINAPI *LPFN_GETACCEPTEXSOCKADDRS)(PVOID, DWORD, DWORD, DWORD,
                                                 struct sockaddr **, LPINT,
                                                 struct sockaddr **, LPINT);

入力と出力

  • **入力**:
  • PVOID: AcceptEx で取得したバッファへのポインタ。
  • DWORD: ローカルアドレスのオフセット。
  • DWORD: リモートアドレスのオフセット。
  • DWORD: 受信データ長。
  • struct sockaddr **: ローカルアドレスを受け取るポインタのポインタ。
  • LPINT: ローカルアドレスの長さを格納するポインタ。
  • struct sockaddr **: リモートアドレスを受け取るポインタのポインタ。
  • LPINT: リモートアドレスの長さを格納するポインタ。
  • **出力**: なし（VOID）。

前提条件と不変条件

  • Windows の mswsock.h で定義されている GetAcceptExSockaddrs 関数と互換性のあるシグネチャである必要があります。
  • WINAPI 呼び出し規約に従う必要があります。

アルゴリズムまたは振る舞い

AcceptEx によって取得された単一のバッファから、ローカルおよびリモートのソケットアドレス情報を解析・抽出するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt 内には具体的な呼び出し元は記載されていません。
  • 呼び出し先: Windows の mswsock.dll が提供する GetAcceptExSockaddrs 関数を指すことを意図しています。

関連データ構造

  • struct sockaddr: ソケットアドレス情報を保持する標準構造体。

失敗モードと境界条件

  • ポインタが NULL の場合、あるいはバッファサイズが不適切な場合、Windows API の仕様に従い例外やエラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは AcceptEx を使用した接続受け入れ処理において、ソケットアドレスを特定するために必須の関数ポインタです。
  • mswsock.h をインクルードして利用する環境において、実行時に WSAIoctl を通じて関数アドレスを取得して代入する運用が想定されます。
