ap_errorlog_format_item

定義

  • 種別: struct
  • 定義箇所: include/http_core.h:966

目的

エラーログのフォーマット文字列を構成する各要素を定義するための構造体です。ログ出力時に実行されるハンドラ関数、引数、フラグ、およびログレベルの閾値を保持します。

シグネチャまたは形状

typedef struct {
    ap_errorlog_handler_fn_t *func;
    const char *arg;
    unsigned int flags;
    unsigned int min_loglevel;
} ap_errorlog_format_item;

入力と出力

  • **入力**: ログ出力処理において、フォーマット文字列を解析した結果として本構造体の配列が生成されます。
  • **出力**: func に指定されたハンドラ関数が実行され、ログメッセージの一部として文字列が生成されます。

前提条件と不変条件

  • func は ap_errorlog_handler_fn_t 型の関数ポインタである必要があります。
  • flags は AP_ERRORLOG_* フラグの組み合わせである必要があります。
  • min_loglevel は、ログメッセージのレベルがこの値より高い場合にのみ項目が出力されるという条件を規定します。

アルゴリズムまたは振る舞い

1. エラーログのフォーマット文字列が解析され、ap_errorlog_format_item の配列が構築されます。 2. ログ出力時、配列内の各項目に対して min_loglevel がチェックされます。 3. 条件を満たす場合、func が呼び出され、arg が引数として渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core.c: parse_errorlog_item 関数内で構造体の生成や初期化が行われます。
  • server/log.c: ログ出力処理のループ内で配列の要素として参照されます。
  • **呼び出し先**:
  • func フィールドに格納された ap_errorlog_handler_fn_t 型の関数が実行されます。

関連データ構造

  • ap_errorlog_handler_fn_t: ログ項目を処理する関数の型定義。
  • apr_array_header_t: server/core.c および server/log.c において、本構造体の配列を保持するために使用されます。

失敗モードと境界条件

  • func が NULL の場合、実行時に未定義の動作を引き起こす可能性があります。
  • arg が NULL の場合、ハンドラ関数側で適切に処理される必要があります。

保守担当者向けメモ

  • 本構造体は include/http_core.h で定義されていますが、主に server/core.c および server/log.c で利用されます。
  • ログフォーマットの拡張を行う際は、AP_ERRORLOG_* フラグの定義と整合性を取る必要があります。
