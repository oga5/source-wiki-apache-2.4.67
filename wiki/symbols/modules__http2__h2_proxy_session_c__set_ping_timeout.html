set_ping_timeout

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_proxy_session.c:96

目的

HTTP/2 プロキシセッションにおいて、PING による生存確認を行う際、ソケットのタイムアウト設定を一時的に PING 用のタイムアウト値へ変更する。

シグネチャまたは形状

static void set_ping_timeout(h2_proxy_session *session)

入力と出力

  • **入力**: h2_proxy_session *session（セッション管理構造体へのポインタ）
  • **出力**: なし（副作用としてソケットのタイムアウト設定を変更）

前提条件と不変条件

  • session->ping_timeout が -1 でないこと。
  • session->save_timeout が -1 であること（既にタイムアウトが退避されていない状態であること）。
  • session->c から取得可能なソケットが存在すること。

アルゴリズムまたは振る舞い

1. session->ping_timeout が有効かつ session->save_timeout が未設定であることを確認する。 2. ap_get_conn_socket を使用して、セッションに関連付けられたソケットを取得する。 3. ソケットが取得できた場合、現在のソケットタイムアウト値を session->save_timeout に保存する。 4. ソケットのタイムアウト値を session->ping_timeout に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c 内の enter_ping_state 関数（行132付近）から呼び出されることが確認されている。
  • **呼び出し先**:
  • ap_get_conn_socket (Apache API)
  • apr_socket_timeout_get (APR API)
  • apr_socket_timeout_set (APR API)

関連データ構造

  • h2_proxy_session: セッション状態、PING タイムアウト値、および退避された元のタイムアウト値を保持する構造体。

失敗モードと境界条件

  • ap_get_conn_socket が NULL を返した場合、タイムアウトの保存および設定は行われない。
  • session->ping_timeout が -1 の場合、本関数は何もしない。
  • session->save_timeout が -1 以外の場合（既にタイムアウトが変更済みの場合）、二重設定を防ぐため何もしない。

保守担当者向けメモ

  • 本関数で設定したタイムアウトを元に戻すには、対となる unset_ping_timeout 関数を使用すること。
  • session->save_timeout は、タイムアウト設定が変更されていない状態を示すフラグとして -1 を使用しているため、この不変条件を維持すること。
