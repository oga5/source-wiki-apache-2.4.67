ssl_hook_Access_classic

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:361

目的

ディレクトリ単位で再設定された SSL パラメータ（暗号スイート、クライアント証明書検証設定、検証深度）を適用し、必要に応じて SSL 再ネゴシエーションを強制する。

シグネチャまたは形状

static int ssl_hook_Access_classic(request_rec *r, SSLSrvConfigRec *sc, SSLDirConfigRec *dc, SSLConnRec *sslconn, SSL *ssl)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • sc, dc: サーバーおよびディレクトリ単位の SSL 設定。
  • sslconn: SSL 接続情報。
  • ssl: OpenSSL の SSL オブジェクト。
  • **出力**:
  • OK (0): 処理成功。
  • HTTP_FORBIDDEN: 再ネゴシエーションが禁止されている場合、または設定適用に失敗した場合。
  • DECLINED: SRP 認証済みの場合など、処理をスキップする場合。

前提条件と不変条件

  • r->connection->master が真（スレーブ接続）の場合、並行リクエストとの競合を避けるため、再ネゴシエーションを伴う設定変更は禁止される。
  • SSL_OPT_OPTRENEGOTIATE オプションが有効な場合、最適化された再ネゴシエーション判定が行われる。

アルゴリズムまたは振る舞い

1. **SRP チェック**: SRP 認証済みであれば即座に DECLINED を返す。 2. **暗号スイートの適用**: 設定された SSLCipherSuite が現在と異なる場合、SSL_set_cipher_list を呼び出し、必要に応じて再ネゴシエーションフラグを立てる。 3. **検証設定の適用**: SSLVerifyClient および SSLVerifyDepth を評価し、現在の設定より厳格な要件が課される場合は再ネゴシエーションを強制する。 4. **最適化**: SSL_OPT_OPTRENEGOTIATE が有効な場合、既存の証明書が存在すれば I/O を伴うハンドシェイクをスキップする「クイック再ネゴシエーション」を試みる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_kernel.c 内の ssl_hook_Access（行 1106 付近）。
  • **呼び出し先**: SSL_get_SSL_CTX, SSL_get_srp_username, SSL_get_current_cipher, SSL_set_cipher_list, SSL_get_verify_mode, SSL_set_verify, SSL_get_peer_certificate 等の OpenSSL API。

関連データ構造

  • request_rec: Apache リクエスト情報。
  • SSLSrvConfigRec, SSLDirConfigRec: SSL 設定構造体。
  • SSLConnRec: 接続ごとの SSL 状態保持用構造体。
  • SSL: OpenSSL のセッションオブジェクト。

失敗モードと境界条件

  • **スレーブ接続**: r->connection->master が存在する場合、設定変更を伴う再ネゴシエーションは HTTP_FORBIDDEN を返す。
  • **設定エラー**: SSL_set_cipher_list が失敗した場合、警告ログを出力し HTTP_FORBIDDEN を返す。

保守担当者向けメモ

  • 本関数は TLS 1.3 以前の「クラシック」な再ネゴシエーションロジックを扱う。TLS 1.3 では再ネゴシエーションが廃止されているため、環境に応じて ssl_hook_Access_modern との使い分けに注意すること。
