dav_unlock

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:497

目的

指定されたリソースおよびその配下のロックを、提供されたロックトークンに基づいて解除します。

シグネチャまたは形状

int dav_unlock(request_rec *r, const dav_resource *resource, const dav_locktoken *locktoken)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • resource: 解除対象のリソース。
  • locktoken: 解除するロックを特定するためのトークン（NULL の場合、リソース以下の全ロックを対象とする）。
  • **出力**:
  • int: 処理結果を示すステータスコード（OK または HTTP_INTERNAL_SERVER_ERROR 等の HTTP ステータス）。

前提条件と不変条件

  • hooks（ロックプロバイダー）が NULL の場合、何もせず OK を返します。
  • locktoken が指定されている場合、dav_get_direct_resource を用いて、そのロックの起点となる「直接ロック」リソースを特定する必要があります。
  • 処理の開始時に hooks->open_lockdb を呼び出し、終了時に hooks->close_lockdb を呼び出す必要があります。

アルゴリズムまたは振る舞い

1. ロックデータベースをオープンします。 2. locktoken が提供されている場合、dav_get_direct_resource を呼び出し、再帰的な解除の起点となる直接ロックのリソースを特定します。 3. dav_walker_ctx を初期化し、dav_unlock_walker をコールバック関数として設定します。 4. repos_hooks->walk を実行し、DAV_INFINITY の深さでリソースツリーを走査してロックを解除します。 5. 走査結果に基づいてステータスを決定し、ロックデータベースをクローズして結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.c (複数箇所で呼び出し)
  • **呼び出し先**:
  • hooks->open_lockdb / hooks->close_lockdb
  • dav_get_direct_resource
  • repos_hooks->walk (コールバックとして dav_unlock_walker を使用)

関連データ構造

  • dav_resource: ロック対象のリソース表現。
  • dav_locktoken: ロックを識別するトークン。
  • dav_lockdb: ロックデータベースのハンドル。
  • dav_walker_ctx: ウォーカー処理用のコンテキスト構造体。

失敗モードと境界条件

  • ロックプロバイダーが未定義の場合、エラーにはならず OK を返します。
  • ロックデータベースのオープンに失敗した場合、HTTP_INTERNAL_SERVER_ERROR を返します。
  • dav_get_direct_resource が失敗した場合、そのエラーコードを返します。
  • repos_hooks->walk 実行時のエラーは、err->status として返されます。

保守担当者向けメモ

  • dav_get_direct_resource および walk 呼び出し時のエラーハンドリングにおいて、より詳細なログ出力や上位レベルの記述追加が推奨されます。
