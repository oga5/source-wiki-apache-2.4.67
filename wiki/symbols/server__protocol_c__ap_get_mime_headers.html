ap_get_mime_headers

定義

  • 種別: function
  • 定義箇所: server/protocol.c:1348

目的

HTTP リクエストの MIME ヘッダーを読み取り、リクエスト構造体 (request_rec) に格納するためのラッパー関数です。

シグネチャまたは形状

void ap_get_mime_headers(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報を保持する構造体)
  • **出力**: なし (戻り値は void)。副作用として r->headers_in テーブルが更新されます。

前提条件と不変条件

  • r および r->pool が有効であること。
  • r->connection->bucket_alloc が有効なバケットアロケータを指していること。

アルゴリズムまたは振る舞い

1. r->pool を使用して、一時的な apr_bucket_brigade を作成します。 2. ap_get_mime_headers_core(r, tmp_bb) を呼び出し、実際のヘッダー解析とテーブルへの格納を実行します。 3. 使用した tmp_bb を破棄し、メモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_filters.c:305
  • **呼び出し先**: ap_get_mime_headers_core

関連データ構造

  • request_rec: リクエストの状態を保持する主要構造体。
  • apr_bucket_brigade: ヘッダー読み取り用の一時的なデータバッファ。

失敗モードと境界条件

  • ap_get_mime_headers_core 内部で発生するエラー（不正なヘッダー形式など）は、当該関数に依存します。
  • メモリ不足時には apr_brigade_create が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は ap_get_mime_headers_core の単純なラッパーです。ヘッダー解析のロジックを変更する場合は ap_get_mime_headers_core を参照してください。
  • include/http_protocol.h に宣言が存在します。
