dir_fixups

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_dir.c:391

目的

ディレクトリリクエストまたはファイルが存在しないリクエストに対し、適切な処理（ディレクトリインデックスの解決またはフォールバックリソースの適用）を行うためのディスパッチャとして機能します。

シグネチャまたは形状

static int dir_fixups(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: int （Apache のステータスコード。fixup_dir または fixup_dflt の戻り値、あるいは処理対象外の場合は DECLINED）

前提条件と不変条件

  • r->finfo がリクエストされたリソースのファイル情報を保持している必要があります。
  • fixup_dir および fixup_dflt は、本関数から呼び出される前に適切な設定（dir_config_rec）がリクエストに紐付いていることを前提とします。

アルゴリズムまたは振る舞い

1. r->finfo.filetype が APR_DIR（ディレクトリ）である場合、fixup_dir(r) を呼び出し、ディレクトリインデックスの解決やスラッシュ補完処理を行います。 2. r->finfo.filetype が APR_NOFILE であり、かつ r->handler が設定されていない場合、fixup_dflt(r) を呼び出し、FallbackResource の処理を行います。 3. 上記いずれにも該当しない場合、DECLINED を返し、他のモジュールによる処理へ委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の fixups フック（modules/mappers/mod_dir.c:406 にて APR_HOOK_LAST として登録）。
  • **呼び出し先**:
  • fixup_dir(r)
  • fixup_dflt(r)

関連データ構造

  • request_rec: リクエストのメタデータおよびファイル情報を保持。
  • dir_config_rec: ディレクトリごとの設定（インデックス名、フォールバック先など）を保持。

失敗モードと境界条件

  • r->finfo が正しく取得できていない場合、意図した分岐に入らず DECLINED を返す可能性があります。
  • fixup_dir または fixup_dflt が内部でエラーを返した場合、そのステータスコードがそのまま伝播します。

保守担当者向けメモ

  • 本関数は fixups フックの最後で実行されるため、他のモジュールがリクエストを処理しなかった場合の最終的なフォールバックとして動作します。
  • 処理の分岐は r->finfo.filetype に依存しているため、ファイルシステムの状態変化に敏感です。
