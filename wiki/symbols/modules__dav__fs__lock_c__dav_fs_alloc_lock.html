dav_fs_alloc_lock

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/lock.c:216

目的

ファイルシステムベースの DAV ロック管理において、新しいロックオブジェクト（dav_lock）をメモリ上に割り当て、初期化する。

シグネチャまたは形状

static dav_lock *dav_fs_alloc_lock(dav_lockdb *lockdb, apr_datum_t key,
                                   const dav_locktoken *locktoken)

入力と出力

  • **入力**:
  • lockdb: ロックデータベースのハンドル。メモリ割り当て用プール（lockdb->info->pool）の参照元として使用。
  • key: ロック対象リソースを識別するための DBM キー。
  • locktoken: 既存のロックトークン。NULL の場合は新規 UUID が生成される。
  • **出力**:
  • 初期化済みの dav_lock 構造体へのポインタを返す。

前提条件と不変条件

  • lockdb および lockdb->info が有効であること。
  • 割り当てられるロックは、内部的に DAV_LOCKREC_DIRECT（直接ロック）としてマークされる。

アルゴリズムまたは振る舞い

1. lockdb->info->pool から dav_lock_combined 構造体分のメモリをゼロクリアして割り当てる。 2. rectype を DAV_LOCKREC_DIRECT に設定し、info ポインタを内部のプライベート領域へ向ける。 3. 指定された key をプライベート領域に格納する。 4. locktoken が NULL の場合、apr_uuid_get を使用して新しい UUID を生成し、ロックトークンとして設定する。 5. locktoken が提供されている場合は、それをそのまま使用する。 6. 構築した dav_lock 構造体の公開部分（pub）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/lock.c 内の以下の箇所から呼び出されることが確認されている。
  • 984行目, 1024行目, 1040行目, 1092行目, 1106行目, 1366行目, 1401行目
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_uuid_get (APR)

関連データ構造

  • dav_lock_combined: ロックの公開情報とプライベート情報を統合した構造体。
  • dav_lock: mod_dav が定義するロック構造体。
  • apr_datum_t: DBM キーを保持するためのデータ型。

失敗モードと境界条件

  • メモリ割り当て（apr_pcalloc）が失敗した場合、APR のメモリ管理ポリシーに従いプロセスが終了する可能性がある。
  • lockdb->info->pool が無効な場合、未定義の動作を引き起こす。

保守担当者向けメモ

  • 本関数は直接ロックの生成に特化している。間接ロックの生成には別のロジックが必要となる点に注意すること。
  • 割り当てられたメモリは request_rec に紐付くプールで管理されるため、明示的な解放は不要である。
