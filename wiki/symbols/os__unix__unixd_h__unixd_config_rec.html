unixd_config_rec

定義

  • 種別: struct
  • 定義箇所: os/unix/unixd.h:75

目的

Apache HTTP Server の Unix 系プラットフォームにおける実行環境（ユーザー、グループ、chroot 設定など）を保持するための構成構造体です。

シグネチャまたは形状

typedef struct {
    const char *user_name;
    const char *group_name;
    uid_t user_id;
    gid_t group_id;
    int suexec_enabled;
    const char *chroot_dir;
    const char *suexec_disabled_reason;
} unixd_config_rec;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。設定ファイル解析時やサーバー起動時に、実行ユーザー名やグループ名、chroot パスなどの情報が格納されます。

前提条件と不変条件

  • suexec_disabled_reason は、suexec_enabled が偽（0）である場合に、その理由を示すメッセージを保持するために使用されます。
  • 実行ユーザーおよびグループの ID（user_id, group_id）は、システム上の有効な ID と整合している必要があります。

アルゴリズムまたは振る舞い

本構造体は、サーバーの実行権限を管理するグローバル変数 ap_unixd_config の型として使用されます。MPM は起動時にこの構造体を参照し、プロセス権限の降格や chroot の適用を行います。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • os/unix/unixd.c: ap_unixd_config の定義元。
  • modules/arch/unix/mod_privileges.c: ap_unixd_config を外部参照。
  • 呼び出し先: 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • ap_unix_identity_t: suexec 等で使用されるユーザー/グループ ID の識別子。
  • ap_unixd_mpm_retained_data: MPM が再起動を跨いで保持する状態データ。

失敗モードと境界条件

  • suexec が無効な場合、suexec_disabled_reason に適切なエラーメッセージが設定されていないと、デバッグが困難になる可能性があります。
  • chroot_dir が無効なパスである場合、サーバーの起動プロセスが失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は os/unix/unixd.c で ap_unixd_config として実体化されます。
  • 構造体メンバーの追加や変更を行う際は、os/unix/unixd.c 内の初期化処理および mod_privileges 等の依存モジュールへの影響を確認してください。
