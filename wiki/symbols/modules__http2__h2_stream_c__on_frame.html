on_frame

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_stream.c:146

目的

HTTP/2 ストリームにおいて、受信または送信されたフレームの種類に基づき、現在のストリーム状態を遷移させるためのディスパッチャとして機能します。

シグネチャまたは形状

static int on_frame(h2_stream_state_t state, int frame_type, 
                    int frame_map[][H2_SS_MAX], apr_size_t maxlen)

入力と出力

  • **入力**:
  • state: 現在の h2_stream_state_t。
  • frame_type: 処理対象のフレーム種別を示す整数。
  • frame_map: フレーム種別ごとの遷移先を定義したマッピングテーブル。
  • maxlen: frame_map の有効なサイズ（apr_size_t）。
  • **出力**:
  • 遷移後の h2_stream_state_t を返します。

前提条件と不変条件

  • frame_type および state は 0 以上である必要があります（ap_assert による検証）。
  • frame_type が maxlen 以上の場合、状態遷移は行われず、現在の state がそのまま返されます。

アルゴリズムまたは振る舞い

1. frame_type と state が負の値でないことをアサートします。 2. frame_type が maxlen を超える場合、未知のフレームとして処理をスキップし、現在の状態を返します。 3. 上記以外の場合、on_map 関数を呼び出し、frame_map を参照して状態遷移を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_stream.c:159 (送信フレーム処理時)
  • modules/http2/h2_stream.c:164 (受信フレーム処理時)
  • **呼び出し先**:
  • on_map (状態遷移の実行)

関連データ構造

  • h2_stream_state_t: ストリームの状態定義。
  • trans_on_send: 送信フレーム用の遷移テーブル。
  • trans_on_recv: 受信フレーム用の遷移テーブル。

失敗モードと境界条件

  • frame_type が maxlen を超える場合、エラーにはならず NOP（何もしない）として現在の状態を維持します。
  • frame_type や state が負の値である場合、ap_assert によりプログラムが停止する可能性があります。

保守担当者向けメモ

  • 本関数は trans_on_send および trans_on_recv テーブルを介して状態遷移を抽象化しています。遷移ロジックの変更が必要な場合は、直接本関数を修正するのではなく、対応する遷移テーブルの定義を確認してください。
