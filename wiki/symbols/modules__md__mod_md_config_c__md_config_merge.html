md_config_merge

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_config.c:254

目的

Apache HTTP Server の設定階層において、ベースサーバーの設定（basev）とバーチャルホスト等の追加設定（addv）をマージし、継承された新しい設定構造体を作成する。

シグネチャまたは形状

static void *md_config_merge(apr_pool_t *pool, void *basev, void *addv)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する APR プール。
  • basev: 親となる設定構造体（md_srv_conf_t*）。
  • addv: 子となる設定構造体（md_srv_conf_t*）。
  • **出力**:
  • マージ結果を格納した新しい md_srv_conf_t 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は md_srv_conf_t 型としてキャスト可能であること。
  • pool は有効なメモリプールであること。
  • 子設定（add）で値が設定されていない項目（DEF_VAL や MD_REQUIRE_UNSET 等）については、親設定（base）の値が採用される。

アルゴリズムまたは振る舞い

1. base と add の設定から新しい md_srv_conf_t 構造体をプール上に確保する。 2. 設定名（name）を生成し、nsc->name に格納する。 3. mc（モジュール設定）や各フラグ（transitive, renew_mode 等）について、add 側がデフォルト値でない場合は add を、それ以外は base を採用する。 4. 配列型の設定（ca_urls, ca_challenges）については、add 側が存在すればそれをコピーし、なければ base 側のコピーを試みる。 5. 文字列型の設定（ca_contact, ca_proto 等）は、add 側が非 NULL であればそれを採用し、なければ base 側を採用する。 6. nsc->current は明示的に NULL に初期化される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_config.c:295
  • modules/md/mod_md_config.c:1440
  • **呼び出し先**:
  • apr_pstrcat, apr_pcalloc, apr_array_copy（APR ユーティリティ）
  • md_pkeys_spec_is_empty（md_crypt.h 関連）

関連データ構造

  • md_srv_conf_t: マージ対象および生成される設定構造体。
  • md_mod_conf_t: nsc->mc を通じて参照されるモジュール全体の設定。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等が例外（またはアボート）を発生させる可能性がある。
  • base または add が NULL の場合の挙動については、ソースコード上での明示的なガードは存在しない。

保守担当者向けメモ

  • 本関数は Apache の設定マージの仕組みに準拠している。新しい設定項目を md_srv_conf_t に追加した際は、必ず本関数内でマージロジックを更新すること。
  • 配列のコピーには apr_array_copy を使用しており、プールを介したメモリ管理が行われている点に注意。
