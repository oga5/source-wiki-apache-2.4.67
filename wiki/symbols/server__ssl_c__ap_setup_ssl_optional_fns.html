ap_setup_ssl_optional_fns

定義

  • 種別: function
  • 定義箇所: server/ssl.c:197

目的

SSL 関連のオプション関数を初期化および登録し、既存の SSL モジュールが提供する関数をキャッシュすることで、コアサーバーと SSL 実装間の互換性を維持します。

シグネチャまたは形状

void ap_setup_ssl_optional_fns(apr_pool_t *pool)

入力と出力

  • **入力**: pool (メモリ管理用の apr_pool_t ポインタ)
  • **出力**: なし（副作用としてグローバルな関数ポインタ変数への代入と、オプション関数の登録が行われます）

前提条件と不変条件

  • サーバーの起動初期段階（post_config フック）で呼び出されることを想定しています。
  • 外部モジュールによって既にインストールされている SSL 関連のオプション関数を検索し、現在の実装と競合しない場合にのみキャッシュします。

アルゴリズムまたは振る舞い

1. APR_RETRIEVE_OPTIONAL_FN を使用して、ssl_is_https および ssl_var_lookup を取得します。 2. 取得した関数が現在の実装と異なる場合、それらを module_ssl_is_https および module_ssl_var_lookup にキャッシュします。 3. APR_REGISTER_OPTIONAL_FN を呼び出し、ssl_is_https、ssl_var_lookup、ssl_proxy_enable、ssl_engine_disable、ssl_engine_set をオプション関数として登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:5138 (ap_setup_ssl_optional_fns(pconf);)
  • **呼び出し先**:
  • APR_RETRIEVE_OPTIONAL_FN (マクロ)
  • APR_REGISTER_OPTIONAL_FN (マクロ)

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • module_ssl_* 系変数: キャッシュされた外部モジュールの関数ポインタを保持するグローバル変数。

失敗モードと境界条件

  • 該当するオプション関数が未定義の場合、APR_RETRIEVE_OPTIONAL_FN は NULL を返しますが、関数は正常に終了します。
  • 既に登録済みの関数と現在の関数が一致する場合、キャッシュは行われません。

保守担当者向けメモ

  • 本関数は、SSL 関連の古い API と新しいフックベースの API を橋渡しする重要な役割を担っています。
  • module_ssl_* 変数への代入ロジックは、既存のモジュールが提供する関数を上書きしないよう慎重に設計されています。
