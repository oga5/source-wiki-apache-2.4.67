cgi_handle_exec

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:128

目的

SSI（Server Side Includes）の exec ディレクティブを処理し、指定されたコマンド（cmd）または CGI スクリプト（cgi）の実行をハンドリングする。

シグネチャまたは形状

static apr_status_t cgi_handle_exec(include_ctx_t *ctx, ap_filter_t *f, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • ctx: SSI 処理のコンテキスト（引数やフラグを含む）。
  • f: 現在のフィルタ。
  • bb: 出力バケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。
  • 副作用: 実行結果が bb に追加されるか、エラー発生時にエラーバケットが生成される。

前提条件と不変条件

  • ctx->flags に SSI_FLAG_PRINTING が設定されていない場合、処理は即座に終了する。
  • SSI_FLAG_NO_EXEC フラグが設定されている場合、実行は拒否されエラーバケットが生成される。
  • 引数がない場合、ログ出力およびエラーバケット生成が行われる。

アルゴリズムまたは振る舞い

1. 引数の有無を確認し、不足時はログ出力を行う。 2. SSI_FLAG_PRINTING フラグを確認し、出力対象外であれば終了する。 3. SSI_FLAG_NO_EXEC フラグを確認し、実行禁止であればエラーバケットを生成して終了する。 4. ループ内で cgi_pfn_gtv を使用してタグ（cmd または cgi）と値を順次取得する。

  • cmd: cgi_pfn_ps で文字列を展開後、include_cmd を呼び出す。
  • cgi: cgi_pfn_ps で文字列を展開後、include_cgi を呼び出す。

    • 5. 各実行関数が失敗した場合は、ログ出力および SSI_CREATE_ERROR_BUCKET によるエラーバケット生成を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/cgi_common.h:220 にて cgi_pfn_reg_with_ssi を介して登録される。
  • **呼び出し先**:
  • cgi_pfn_gtv, cgi_pfn_ps（SSI ユーティリティ関数）
  • include_cmd, include_cgi（実行実体）
  • ap_log_rerror（ログ出力）

関連データ構造

  • include_ctx_t: SSI 処理状態を保持。
  • ap_filter_t: フィルタチェーン情報。
  • apr_bucket_brigade: 出力データストリーム。

失敗モードと境界条件

  • 引数不足: ログ出力後、エラーバケットを生成。
  • 実行権限なし (SSI_FLAG_NO_EXEC): ログ出力後、エラーバケットを生成。
  • 不明なパラメータ: ログ出力後、エラーバケットを生成。
  • 実行失敗 (include_cmd/include_cgi の戻り値が非 APR_SUCCESS): ログ出力後、エラーバケットを生成。

保守担当者向けメモ

  • 本関数は mod_include からの呼び出しを想定している。
  • SSI_CREATE_ERROR_BUCKET マクロは、エラー発生時にクライアントへエラー情報を送出するために使用される。
  • cgi_pfn_ps の引数 SSI_EXPAND_LEAVE_NAME と SSI_EXPAND_DROP_NAME の使い分けに注意すること。
