on_event

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md.c:222

目的

md_event サブシステムから通知されるイベントを処理し、関連するジョブの通知処理（notify）へ転送するコールバック関数です。

シグネチャまたは形状

static apr_status_t on_event(const char *event, const char *mdomain, void *baton, 
                             md_job_t *job, md_result_t *result, apr_pool_t *p)

入力と出力

  • **入力**:
  • event: 発生したイベント名（文字列）。
  • mdomain: 対象のマネージドドメイン名。
  • baton: ユーザー定義のコンテキストデータ。
  • job: イベントに関連する md_job_t 構造体。
  • result: 処理結果を保持する md_result_t 構造体。
  • p: メモリ割り当て用の apr_pool_t。
  • **出力**:
  • apr_status_t: notify 関数の実行結果を返します。

前提条件と不変条件

  • mdomain 引数は関数内で使用されず、無視されます。
  • notify 関数が正常に呼び出せる状態である必要があります。

アルゴリズムまたは振る舞い

1. mdomain 引数を明示的に破棄（キャストによる無視）します。 2. 受け取った job、event、result、p、および baton をそのまま notify 関数に引き渡し、その戻り値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c:891 において md_event_subscribe に登録されています。
  • **呼び出し先**: notify 関数を呼び出します。
  • **注意**: ツールが提示した modules/http2/ 関連の on_event は同名の別シンボルであり、本関数とは無関係です。

関連データ構造

  • md_job_t: ジョブの状態管理に使用。
  • md_result_t: 処理結果の格納に使用。
  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • notify 関数が失敗した場合、そのエラーコードがそのまま本関数の戻り値となります。

保守担当者向けメモ

  • 本関数は notify への単なるパススルーとして実装されています。将来的にイベント処理のフィルタリングやログ出力の追加が必要な場合は、この関数内で実装してください。
