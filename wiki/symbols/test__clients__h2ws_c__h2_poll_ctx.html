h2_poll_ctx

定義

  • 種別: struct
  • 定義箇所: test/clients/h2ws.c:707

目的

poll() による I/O 多重化イベント発生時に、対象となる HTTP/2 セッションまたはストリームの状態を保持し、適切なイベントハンドラへディスパッチするためのコンテキスト構造体です。

シグネチャまたは形状

struct h2_poll_ctx {
    struct h2_session *session;
    struct h2_stream *stream;
    h2_poll_ev_cb *on_ev;
};

入力と出力

  • **入力**: h2_session または h2_stream へのポインタ、およびイベント発生時に実行されるコールバック関数 h2_poll_ev_cb。
  • **出力**: h2_poll_ev_cb を通じて、pollfd 構造体と連携した I/O 処理結果（整数値）が返されます。

前提条件と不変条件

  • on_ev に設定される関数は、typedef int h2_poll_ev_cb(struct h2_poll_ctx *pctx, struct pollfd *pfd); のシグネチャに従う必要があります。
  • session または stream のいずれか（あるいは両方）が有効なオブジェクトを指している必要があります。

アルゴリズムまたは振る舞い

1. poll() ループにおいて、監視対象のファイルディスクリプタに対応する h2_poll_ctx が保持されます。 2. イベントが発生すると、on_ev コールバックが呼び出され、コンテキスト内の session や stream を参照して具体的な I/O 処理（読み込みや書き込み）を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: test/clients/h2ws.c 内のイベントループ処理（poll() を含む関数群）。
  • **呼び出し先**:
  • h2_session_ev (イベントハンドラ)
  • h2_stream_ev (イベントハンドラ)

関連データ構造

  • h2_session: HTTP/2 セッション状態を保持。
  • h2_stream: 個別のストリーム状態を保持。
  • pollfd: poll() システムコールで使用されるファイルディスクリプタ監視構造体。

失敗モードと境界条件

  • on_ev が NULL の場合、イベント発生時に未定義の動作を引き起こす可能性があります。
  • session や stream が解放済みであるにもかかわらず h2_poll_ctx が参照された場合、不正なメモリ参照が発生します。

保守担当者向けメモ

  • 本構造体は h2ws.c 内の I/O 多重化の抽象化レイヤーです。新しいイベントタイプを追加する場合は、h2_poll_ev_cb に適合するハンドラを実装し、この構造体経由で登録してください。
