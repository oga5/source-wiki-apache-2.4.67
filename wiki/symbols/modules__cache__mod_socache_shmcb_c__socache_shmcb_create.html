socache_shmcb_create

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_shmcb.c:290

目的

共有メモリキャッシュ（SHMCB）プロバイダーのインスタンスコンテキストを初期化し、設定引数からキャッシュの保存先パスとサイズを解析・設定します。

シグネチャまたは形状

static const char *socache_shmcb_create(ap_socache_instance_t **context,
                                        const char *arg,
                                        apr_pool_t *tmp, apr_pool_t *p)

入力と出力

  • **入力**:
  • context: インスタンスコンテキストを格納するポインタのポインタ。
  • arg: 設定文字列（例: "/path/to/file(size)"）。
  • tmp: 一時的なメモリ割り当て用プール。
  • p: インスタンスの生存期間を管理するメモリプール。
  • **出力**:
  • 成功時は NULL を返し、*context に初期化済み構造体を設定します。
  • 失敗時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • デフォルトのキャッシュサイズは 512KB です。
  • arg にサイズ指定が含まれる場合、サイズは 8192 バイト以上かつ SHMCB_MAX_SIZE 未満である必要があります。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して ap_socache_instance_t を割り当て、プールを関連付けます。 2. arg が空の場合はデフォルト設定を適用して終了します。 3. arg に括弧 () が含まれる場合、括弧内の数値をキャッシュサイズとして解析し、括弧の外側をファイルパスとして抽出します。 4. 解析したサイズが範囲外（8192未満、またはプラットフォーム上限以上）の場合はエラーを返します。 5. 正常終了時は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_shmcb.c:1058 にて参照されています。
  • **呼び出し先**: ap_server_root_relative, strrchr, strlen, strtol, apr_psprintf 等の APR/Apache API を呼び出します。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスのコンテキストを保持する構造体。

失敗モードと境界条件

  • **引数エラー**: 括弧の不整合（閉じ括弧がない、開き括弧がない）、サイズが数値でない、サイズが 8192 バイト未満、または SHMCB_MAX_SIZE 以上の場合にエラーメッセージを返します。

保守担当者向けメモ

  • SHMCB_MAX_SIZE はプラットフォーム依存の定数です。
  • キャッシュサイズ計算に unsigned int が使用されている箇所があるため、非常に大きなサイズを指定する際のオーバーフローリスクに注意してください。
