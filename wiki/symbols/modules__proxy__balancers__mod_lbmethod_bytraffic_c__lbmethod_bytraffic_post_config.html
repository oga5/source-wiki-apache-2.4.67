lbmethod_bytraffic_post_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_bytraffic.c:98

目的

mod_lbmethod_bytraffic モジュールの初期化を行い、mod_proxy が提供するワーカー選定用のオプション関数を取得して利用可能にする。

シグネチャまたは形状

static int lbmethod_bytraffic_post_config(apr_pool_t *pconf, apr_pool_t *plog,
                                          apr_pool_t *ptemp, server_rec *s);

入力と出力

  • 入力:
  • pconf, plog, ptemp: Apache のメモリプール。
  • s: 現在のサーバー設定レコード。
  • 出力:
  • OK: 正常終了。
  • !OK: mod_proxy がロードされていない等のエラー発生時。

前提条件と不変条件

  • 本関数は起動時に2回呼び出される。
  • AP_SQ_MS_CREATE_PRE_CONFIG 状態での呼び出し時は、静的データの初期化を行わず即座に OK を返す必要がある。

アルゴリズムまたは振る舞い

1. ap_state_query(AP_SQ_MAIN_STATE) を呼び出し、現在のサーバー状態を確認する。 2. 状態が AP_SQ_MS_CREATE_PRE_CONFIG である場合、何もせず OK を返して終了する。 3. APR_RETRIEVE_OPTIONAL_FN(proxy_balancer_get_best_worker) を使用して、mod_proxy からワーカー選定関数を取得する。 4. 取得した関数ポインタが NULL の場合、エラーログを出力し !OK を返す。 5. 正常に取得できた場合、OK を返す。

呼び出し元と呼び出し先

  • 呼び出し元: ap_hook_post_config により、Apache の設定完了フェーズで呼び出される。
  • 呼び出し先:
  • ap_state_query (Apache API)
  • APR_RETRIEVE_OPTIONAL_FN (APR マクロ)
  • ap_log_error (Apache API)

関連データ構造

  • server_rec: サーバー設定情報。
  • ap_proxy_balancer_get_best_worker_fn: mod_proxy から取得されるワーカー選定用関数ポインタ。

失敗モードと境界条件

  • mod_proxy がロードされていない場合、proxy_balancer_get_best_worker の取得に失敗し、エラーログを出力して異常終了する。

保守担当者向けメモ

  • 本関数は起動時に複数回実行されるため、静的データの初期化はガード条件（AP_SQ_MS_CREATE_PRE_CONFIG）を必ず経由させること。
