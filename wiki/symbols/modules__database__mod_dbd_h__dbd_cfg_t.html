dbd_cfg_t

定義

  • 種別: struct
  • 定義箇所: modules/database/mod_dbd.h:60

目的

Apache HTTP Server の mod_dbd モジュールにおいて、サーバー単位のデータベース接続設定を保持するための構造体です。ドライバ名、接続パラメータ、および接続プールの動作設定を管理します。

シグネチャまたは形状

typedef struct {
    server_rec *server;
    const char *name;
    const char *params;
    int persist;
#if APR_HAS_THREADS
    int nmin;
    int nkeep;
    int nmax;
    int exptime;
    int set;
#endif
    apr_hash_t *queries;
    apr_array_header_t *init_queries;
} dbd_cfg_t;

入力と出力

本構造体は設定コンテナとして機能します。

  • **入力**: サーバー設定ディレクティブ（DBDriver, DBDParams 等）を通じて値が設定されます。
  • **出力**: mod_dbd の内部ロジック（接続プールの初期化やクエリの準備）において、接続パラメータやプールサイズ等の設定値として参照されます。

前提条件と不変条件

  • APR_HAS_THREADS が定義されている環境では、スレッドセーフな接続プール管理のためのパラメータ（nmin, nkeep, nmax 等）が有効になります。
  • queries および init_queries は、SQL ステートメントを管理するためのハッシュテーブルおよび配列として初期化されている必要があります。

アルゴリズムまたは振る舞い

  • サーバーの起動および設定読み込みフェーズにおいて、apr_pcalloc 等でメモリが確保され、各フィールドが初期化されます。
  • 接続取得リクエストが発生した際、本構造体に保持された params やプール設定に基づき、mod_dbd はデータベース接続の確立または再利用を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/database/mod_dbd.c 内の各種設定ハンドラおよび接続管理関数（例: dbd_prepared_init 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • server_rec: Apache のサーバー設定構造体。
  • apr_hash_t: 準備済みクエリを格納するハッシュテーブル。
  • apr_array_header_t: 初期化クエリを格納する配列。
  • ap_dbd_t: 本設定に基づいて管理される実際のデータベース接続ハンドル。

失敗モードと境界条件

  • 設定値が未定義（set フラグが 0 など）の場合、デフォルトの接続プール動作が適用される可能性があります。
  • メモリ不足時には apr_pcalloc による構造体確保が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dbd の中核となる設定保持用データ構造です。フィールドの追加や変更を行う際は、mod_dbd.c 内の dbd_post_config や設定マージ処理への影響を考慮してください。
  • スレッド環境下での接続プールパラメータ（nmin, nkeep, nmax）の整合性は、mod_dbd.c 側のロジックに依存しています。
