cache_handler

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:393

目的

HTTP リクエストに対するキャッシュの検索、保存、または無効化を制御するメインハンドラ。mod_cache のポリシーエンジンとして、リクエストのキャッシュ可能性を判定し、適切なフィルタをフィルタチェーンに挿入する。

シグネチャまたは形状

int cache_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (Apache ステータスコード: OK, DECLINED 等)

前提条件と不変条件

  • cache_server_conf の quick 設定が有効な場合、本関数は即座に DECLINED を返す。
  • キャッシュプロバイダーが取得できない場合、処理を継続せず DECLINED を返す。
  • メモリ割り当ては r->pool を使用し、リクエスト終了時に自動解放される。

アルゴリズムまたは振る舞い

1. **初期化**: リクエストごとの設定構造体 cache_request_rec を初期化。 2. **メソッド判定**:

  • PUT/POST/DELETE: cache_invalidate_filter を挿入し、キャッシュ無効化を予約して DECLINED を返す。
  • GET: キャッシュ検索へ進む。
  • その他: キャッシュ対象外として DECLINED を返す。

    • 3. **キャッシュ検索 (cache_select)**:

  • **ミス**: cache_try_lock で排他制御を試み、成功すれば cache_save_filter を挿入して DECLINED を返す。
  • **ヒット**: cache_out_filter を挿入し、既存のフィルタスタックを整理した上でキャッシュからレスポンスを送信し OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_handler により登録され、Apache のハンドラフェーズで呼び出される。
  • **呼び出し先**:
  • cache_get_providers, cache_select, cache_try_lock, cache_replace_filter, cache_get_filter, cache_run_cache_status
  • ap_add_output_filter_handle, ap_remove_output_filter, ap_pass_brigade_fchk 等の Apache API。

関連データ構造

  • cache_request_rec: リクエストごとのキャッシュ状態を保持。
  • cache_server_conf: サーバー全体の設定。
  • cache_provider_list: 利用可能なキャッシュプロバイダーのリスト。

失敗モードと境界条件

  • cache_try_lock が失敗した場合（他リクエストが既にキャッシュ更新中の場合）、キャッシュ保存を行わずバックエンドへリクエストを流す。
  • ap_meets_conditions による条件付きリクエストの検証に失敗した場合、そのステータスを返す。

保守担当者向けメモ

  • 本関数は quick ハンドラが無効な場合にのみ動作する。
  • cache_replace_filter を使用して、管理者が指定した CACHE フィルタ位置に動的に CACHE_SAVE または CACHE_OUT フィルタを挿入する仕組みを持つ。
  • サブリクエスト (r->main) の有無によって、使用するフィルタハンドルを切り替える必要がある。
