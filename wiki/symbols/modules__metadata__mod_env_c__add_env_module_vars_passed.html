add_env_module_vars_passed

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_env.c:92

目的

Apache HTTP Server の設定ディレクティブ PassEnv を処理し、OS の環境変数を現在のディレクトリ設定（env_dir_config_rec）の変数テーブルにコピーします。

シグネチャまたは形状

static const char *add_env_module_vars_passed(cmd_parms *cmd, void *sconf_, const char *arg);

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • sconf_: 現在のディレクトリ設定構造体（env_dir_config_rec）。
  • arg: PassEnv ディレクティブで指定された環境変数名。
  • **出力**:
  • 成功時および失敗時ともに NULL を返します。

前提条件と不変条件

  • sconf_ は env_dir_config_rec 型としてキャスト可能である必要があります。
  • cmd->pool は、環境変数の値を複製するためのメモリ割り当てに使用されます。

アルゴリズムまたは振る舞い

1. getenv(arg) を呼び出し、OS から指定された環境変数の値を取得します。 2. 変数が存在する場合:

  • apr_pstrdup を使用して、値を cmd->pool に複製します。
  • apr_table_setn を使用して、sconf->vars テーブルにキー（変数名）と値（複製した文字列）を登録します。

    • 3. 変数が存在しない場合:

  • ap_log_error を使用して、APLOG_WARNING レベルで警告ログを出力します。処理は中断されず、そのまま終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_env.c:151 にて AP_INIT_ITERATE("PassEnv", ...) マクロを通じて登録され、設定解析時に呼び出されます。
  • **呼び出し先**:
  • getenv() (標準ライブラリ)
  • apr_table_setn() (APR)
  • apr_pstrdup() (APR)
  • ap_log_error() (Apache API)

関連データ構造

  • env_dir_config_rec: ディレクトリごとの設定を保持する構造体。vars メンバ（apr_table_t）が環境変数の格納先となります。

失敗モードと境界条件

  • **環境変数の欠落**: 指定された環境変数が OS 上に存在しない場合、警告ログが出力されますが、プログラムはエラーを返さず正常終了します。
  • **メモリ不足**: apr_pstrdup がメモリ不足に陥った場合、APR の挙動に従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は PassEnv ディレクティブ専用です。SetEnv や UnsetEnv の処理については、それぞれ add_env_module_vars_set および add_env_module_vars_unset を参照してください。
  • apr_table_setn を使用しているため、値の文字列は永続的なメモリ領域（プール）にある必要があります。apr_pstrdup による複製はこれを保証しています。
