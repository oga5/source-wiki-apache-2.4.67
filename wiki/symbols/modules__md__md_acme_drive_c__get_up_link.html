get_up_link

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:197

目的

ACME サーバーからの HTTP レスポンスヘッダーに含まれる Link ヘッダーから、関係性（relation）が "up" であるリンク先 URL を抽出し、ドライバーのコンテキストに保存する。

シグネチャまたは形状

static void get_up_link(md_proto_driver_t *d, apr_table_t *headers)

入力と出力

  • **入力**:
  • d: プロトコルドライバーのコンテキスト (md_proto_driver_t*)。
  • headers: サーバーから受信した HTTP レスポンスヘッダーのテーブル (apr_table_t*)。
  • **出力**: なし（副作用として d->baton が指す md_acme_driver_t 構造体の chain_up_link フィールドを更新する）。

前提条件と不変条件

  • d および d->baton は有効なポインタであること。
  • d->baton は md_acme_driver_t 型としてキャスト可能であること。
  • d->p（リクエストプール）は、リンク文字列のメモリ割り当てに利用可能な状態であること。

アルゴリズムまたは振る舞い

1. d->baton から ACME ドライバー構造体 ad を取得する。 2. md_link_find_relation を呼び出し、headers から "up" 関係を持つリンク URL を検索する。 3. 取得した URL を ad->chain_up_link に格納する。 4. リンクが見つかった場合、デバッグレベルのログメッセージを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_drive.c:246
  • modules/md/md_acme_drive.c:327
  • modules/md/md_acme_drive.c:412
  • **呼び出し先**:
  • md_link_find_relation (外部ユーティリティ)
  • md_log_perror (ログ出力)

関連データ構造

  • md_proto_driver_t: プロトコル駆動の共通コンテキスト。
  • md_acme_driver_t: ACME プロトコル固有のドライバー状態。chain_up_link フィールドを保持する。
  • apr_table_t: HTTP ヘッダーを保持する APR のテーブル構造。

失敗モードと境界条件

  • headers に "up" リンクが含まれない場合、ad->chain_up_link は NULL に設定される。
  • md_link_find_relation が失敗またはメモリ不足の場合の挙動は、当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数は ACME サーバーから提供される証明書チェーンのアップリンク情報を取得するためのヘルパーです。
  • ログ出力には d->p を使用しているため、リクエストのライフサイクルと同期しています。
