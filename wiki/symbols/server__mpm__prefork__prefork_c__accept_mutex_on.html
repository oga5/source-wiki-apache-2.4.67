accept_mutex_on

定義

  • 種別: function
  • 定義箇所: server/mpm/prefork/prefork.c:243

目的

accept() 呼び出し時に発生する競合を防ぐため、プロセス間ミューテックスのロックを取得する。

シグネチャまたは形状

static apr_status_t accept_mutex_on(void)

入力と出力

  • **入力**: なし（グローバルな my_bucket を参照）。
  • **出力**: 成功時は APR_SUCCESS を返す。失敗時は内部でプロセスを終了させるため、呼び出し元へは戻らない。

前提条件と不変条件

  • my_bucket が初期化済みであること。
  • 本関数は子プロセスから呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. my_bucket->mutex に対して apr_proc_mutex_lock を実行する。 2. ロック取得に失敗した場合、現在のプロセス世代（my_generation）とスコアボード上の実行世代を比較する。

  • 世代が異なる場合: デバッグログを出力し、clean_child_exit(0) を呼び出して正常終了する。
  • 世代が一致する場合: 緊急ログを出力し、exit(APEXIT_CHILDFATAL) を呼び出して異常終了する。

    • 3. ロック取得に成功した場合は APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/prefork/prefork.c:573 にて SAFE_ACCEPT マクロ経由で呼び出される。
  • **呼び出し先**:
  • apr_proc_mutex_lock (APR)
  • ap_log_error (Apache Core)
  • clean_child_exit (本ファイル内)
  • exit (標準ライブラリ)

関連データ構造

  • prefork_child_bucket: ミューテックスを保持する構造体。
  • prefork_retained_data: 世代管理に使用される構造体。

失敗モードと境界条件

  • ミューテックスのロック取得失敗時、プロセスは終了する。
  • 世代不一致時は clean_child_exit による終了、それ以外は APEXIT_CHILDFATAL による異常終了となる。

保守担当者向けメモ

  • test/time-sem.c に同名の関数が存在するが、これらはテスト用であり本実装とは無関係である。
  • SAFE_ACCEPT マクロ経由での呼び出しが前提となっているため、直接呼び出す際は注意が必要。
