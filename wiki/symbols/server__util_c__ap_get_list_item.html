ap_get_list_item

定義

  • 種別: function
  • 定義箇所: server/util.c:1314

目的

HTTP ヘッダーなどのリスト形式の文字列から、個別の項目（トークン）を抽出し、正規化（小文字化や不要な空白の除去）して返します。

シグネチャまたは形状

char *ap_get_list_item(apr_pool_t *p, const char **field)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • field: 解析対象の文字列へのポインタ。解析が進むと、このポインタは次の項目を指すように更新されます。
  • **出力**:
  • 抽出・正規化された項目を含む新しい文字列（apr_palloc で確保）。
  • 解析対象が空または終了した場合は NULL を返します。

前提条件と不変条件

  • field が指す文字列は、カンマ区切りのリスト形式であると想定されます。
  • 引用符（"）内の文字列、コメント（()）、およびエスケープされた文字（\）は、構文として正しく維持されます。

アルゴリズムまたは振る舞い

1. ap_size_list_item を呼び出し、次の項目の開始位置と必要なバッファサイズを特定します。 2. apr_palloc でメモリを確保します。 3. 文字列を再スキャンし、以下のルールでコピーを行います。

  • 引用符内、コメント内、またはエスケープ直後の文字はそのままコピーします。
  • それ以外の場所にある英字は apr_tolower で小文字化します。
  • =、/、; の周囲の不要な空白は除去されます。
  • 連続する空白は単一のスペースにまとめられます。

    • 4. 終端文字 \0 を付与して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c
  • modules/cache/mod_cache_socache.c
  • modules/http/http_filters.c
  • modules/http2/h2_c2_filter.c
  • modules/mappers/mod_negotiation.c
  • test/test_parser.c
  • **呼び出し先**:
  • ap_size_list_item (同ファイル内)
  • apr_palloc (APR)
  • apr_tolower (APR)

関連データ構造

  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • ap_size_list_item が NULL を返した場合、本関数も NULL を返します。
  • 文字列が空の場合、またはカンマのみで構成されている場合は NULL を返します。

保守担当者向けメモ

  • 本関数は server/util.c 内の ap_size_list_item に強く依存しています。解析ロジックを変更する場合は、両関数の整合性を確認してください。
  • addspace フラグによる空白制御は、HTTP ヘッダーの正規化において重要な役割を果たしています。
