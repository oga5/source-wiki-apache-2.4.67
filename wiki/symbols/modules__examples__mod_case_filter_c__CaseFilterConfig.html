CaseFilterConfig

定義

  • 種別: struct
  • 定義箇所: modules/examples/mod_case_filter.c:30

目的

Apache HTTP Server の出力フィルタモジュールにおいて、フィルタの有効/無効状態を保持するためのサーバー設定構造体です。

シグネチャまたは形状

typedef struct
{
    int bEnabled;
} CaseFilterConfig;

入力と出力

  • **入力**: CaseFilter ディレクティブの設定値（bEnabled に格納）。
  • **出力**: サーバー設定の一部として、フィルタの挿入判断や処理実行の可否を決定するフラグとして機能。

前提条件と不変条件

  • bEnabled は、フィルタが有効な場合に非ゼロ、無効な場合にゼロとして扱われます。
  • サーバーのモジュール設定領域（module_config）に格納されることを前提としています。

アルゴリズムまたは振る舞い

  • 本構造体は、CaseFilterCreateServerConfig によってメモリが確保され、サーバー設定として保持されます。
  • CaseFilterInsertFilter フックにおいて、この構造体の bEnabled を参照することで、現在のリクエストに対してフィルタを適用すべきかどうかが判定されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • modules/examples/mod_case_filter.c:37: apr_pcalloc によるインスタンス生成時。
  • modules/examples/mod_case_filter.c:46: ap_get_module_config を介したリクエスト処理時の取得。
  • modules/examples/mod_case_filter.c:109: ap_get_module_config を介した設定コマンド実行時の取得。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • request_rec: リクエスト処理時に本構造体を取得するためのコンテキストとして使用。
  • cmd_parms: 設定コマンド実行時に本構造体を取得するためのコンテキストとして使用。

失敗モードと境界条件

  • ap_get_module_config が適切な設定を返さない場合、未初期化のメモリを参照するリスクがあります。
  • apr_pcalloc によるメモリ確保が失敗した場合、サーバーの起動または設定処理が中断される可能性があります。

保守担当者向けメモ

  • 本構造体は教育用モジュールの一部であり、拡張性よりもフィルタの有効状態を管理するという単純な責務に特化しています。
  • 構造体メンバを追加する場合は、CaseFilterCreateServerConfig での初期化処理も併せて更新してください。
