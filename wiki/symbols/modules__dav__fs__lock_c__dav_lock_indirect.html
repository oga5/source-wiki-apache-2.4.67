dav_lock_indirect

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/lock.c:139

目的

親コレクションから継承された間接ロック（Indirect Lock）の情報を保持し、ファイルシステム上のリソースに関連付けられたロック状態を管理するために使用されます。

シグネチャまたは形状

typedef struct dav_lock_indirect
{
    dav_locktoken *locktoken;
    apr_datum_t key;
    struct dav_lock_indirect *next;
    time_t timeout;
} dav_lock_indirect;

入力と出力

本構造体はデータ保持用であり、関数ではありません。dav_fs_load_lock_record 等の関数によって DBM から読み込まれたデータが本構造体に格納され、ロック解決処理（dav_fs_resolve）等で参照されます。

前提条件と不変条件

  • next ポインタは、同一リソースに関連付けられた間接ロックのリストを形成します。
  • key は、対応する直接ロック（Direct Lock）を DBM から検索するための識別子として機能します。
  • timeout は、ロックの有効期限を示します。

アルゴリズムまたは振る舞い

  • 本構造体は、dav_fs_load_lock_record によって DBM からシリアライズされたデータから構築されます。
  • dav_fs_resolve は、この構造体の key を使用して再帰的に直接ロックを特定し、間接ロックの有効性を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/lock.c 内の複数の関数（416, 524, 665, 668 行付近など）で、ロックリストの走査や生成、解決のために参照されています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • dav_locktoken: ロックを一意に識別するための UUID ラッパー。
  • dav_lock_discovery: 直接ロックの情報を保持する構造体。
  • dav_lockdb_combined: ロックデータベースのハンドルとパスを統合した構造体。

失敗モードと境界条件

  • 構造体内の key が無効な場合、dav_fs_resolve による直接ロックの解決が失敗します。
  • timeout を過ぎたロックは、dav_fs_load_lock_record の処理過程で自動的に除外される設計となっています。

保守担当者向けメモ

  • 本構造体は mod_dav のロックプロバイダインターフェースの一部として、ファイルシステム上のロック永続化に不可欠です。
  • メモリ管理は request_rec に紐付くプールで行われるため、明示的な解放は不要です。
  • 構造体定義の変更は、DBM へのシリアライズ/デシリアライズ処理（dav_fs_save_lock_record / dav_fs_load_lock_record）との整合性に注意してください。
