ap_update_child_status

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:590

目的

ワーカープロセスまたはスレッドの現在の状態をスコアボードに記録し、監視および統計情報として反映させる。

シグネチャまたは形状

int ap_update_child_status(ap_sb_handle_t *sbh, int status, request_rec *r)

入力と出力

  • **入力**:
  • sbh: 更新対象のワーカーを特定するスコアボードハンドル。
  • status: 設定する新しい状態（例: SERVER_BUSY_READ）。
  • r: 現在のリクエスト情報（request_rec）。NULL の場合、接続情報やサーバー情報は記録されない。
  • **出力**:
  • 成功時は update_child_status_internal の戻り値を返す。
  • sbh が無効（NULL または child_num < 0）な場合は -1 を返す。

前提条件と不変条件

  • sbh は有効なスコアボードハンドルであること。
  • 本関数は共有メモリ上の特定のスコアボードスロットを更新する。スロットは単一の所有者によって書き込まれる設計であるため、明示的なロックは不要である。

アルゴリズムまたは振る舞い

1. sbh が NULL または child_num が負の値であれば、更新を行わず -1 を返す。 2. r が提供されている場合、r->connection および r->server を抽出する。 3. 内部関数 update_child_status_internal を呼び出し、ワーカーのインデックス（child_num, thread_num）と状態、および関連する接続・リクエスト情報を渡してスコアボードを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_core.c
  • modules/http2/h2_c2.c
  • server/connection.c
  • server/core.c
  • server/eor_bucket.c
  • server/mpm/event/event.c
  • **呼び出し先**:
  • update_child_status_internal (同一ファイル内の静的関数)

関連データ構造

  • ap_sb_handle_t: スコアボード内の特定ワーカーを指すハンドル。
  • request_rec: リクエスト処理の状態を保持する構造体。
  • worker_score: スコアボード内のワーカーごとの状態保持構造体（include/scoreboard.h で定義）。

失敗モードと境界条件

  • sbh が NULL の場合、または初期化されていないハンドル（child_num < 0）が渡された場合、更新はスキップされ -1 が返される。
  • r が NULL の場合、リクエスト固有の情報（リクエスト文字列やクライアントIP等）はスコアボードに反映されない。

保守担当者向けメモ

  • 本関数は ap_extended_status が有効な場合に詳細な統計情報を記録する update_child_status_internal のラッパーである。
  • 高負荷環境下では、r を伴う呼び出しによる文字列コピー等のオーバーヘッドに注意すること。
