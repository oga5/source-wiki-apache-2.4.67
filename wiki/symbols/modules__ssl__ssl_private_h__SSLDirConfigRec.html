SSLDirConfigRec

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:597

目的

Apache HTTP Server の mod_ssl において、ディレクトリ単位（<Directory> や <Location> 等）で適用される SSL 関連の設定情報を保持するための構造体です。

シグネチャまたは形状

typedef struct SSLDirConfigRec SSLDirConfigRec;

入力と出力

本構造体は、Apache の設定解析プロセスを通じて生成・マージされます。

  • **入力**: mod_ssl の設定ディレクティブ（認証モード、暗号スイート設定など）から値が格納されます。
  • **出力**: リクエスト処理フェーズにおいて、現在のディレクトリに対する SSL ポリシーを決定するために参照されます。

前提条件と不変条件

  • apr_pool_t を使用したメモリ管理下で生成されます。
  • サーバーの設定マージ処理において、親ディレクトリの設定を継承し、必要に応じて上書きする形で構築されます。

アルゴリズムまたは振る舞い

  • 設定の継承とマージ: ssl_engine_config.c 内の関数群により、ベースとなる設定と追加設定を統合する際に使用されます。
  • コンテキストの初期化: プロキシ設定の初期化や、各リクエストに対する SSL コンテキストの構築時に、この構造体の値が参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_config.c 内の各種設定ハンドラおよびマージ関数（modssl_ctx_init_proxy 等）から参照・生成されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • SSLSrvConfigRec: 仮想ホスト単位の設定構造体。
  • SSLConnRec: 接続ごとの SSL 状態を保持する構造体。

失敗モードと境界条件

  • メモリ不足: apr_palloc によるメモリ確保に失敗した場合、Apache のエラーハンドリングに従いプロセスが停止またはリクエストが拒否されます。
  • 設定の不整合: ディレクトリ設定が正しくマージされない場合、意図しない SSL ポリシーが適用される可能性があります。

保守担当者向けメモ

  • 本構造体は ssl_private.h で不完全型として宣言されています。具体的なメンバ定義については、ssl_engine_config.c の実装および ssl_private.h の後続の定義を参照してください。
  • 設定の追加や変更を行う際は、ssl_engine_config.c 内のマージロジックが正しく追従していることを確認してください。
