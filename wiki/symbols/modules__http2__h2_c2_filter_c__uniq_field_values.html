uniq_field_values

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2_filter.c:172

目的

HTTP ヘッダー（主に Vary ヘッダー）の値を解析し、重複を除去して配列に格納するコールバック関数です。

シグネチャまたは形状

static int uniq_field_values(void *d, const char *key, const char *val)

入力と出力

  • **入力**:
  • d: apr_array_header_t * 型の配列へのポインタ（重複除去後の値を格納する先）。
  • key: ヘッダー名（本関数内では使用されず無視される）。
  • val: 解析対象のヘッダー値文字列（例: "Accept-Encoding, User-Agent"）。
  • **出力**:
  • 常に 1 を返します（apr_table_do の継続条件を満たすため）。

前提条件と不変条件

  • d は apr_array_header_t 構造体として初期化されている必要があります。
  • val はカンマ区切りの文字列であることを想定しています。
  • values->pool が有効なメモリプールとして設定されている必要があります。

アルゴリズムまたは振る舞い

1. val を values->pool 上に複製します。 2. 複製した文字列をカンマおよび空白を区切り文字としてトークン化します。 3. 各トークンについて、既に values 配列内に存在するかを ap_cstr_casecmp を用いて大文字小文字を区別せずに比較します。 4. 配列内に存在しない場合のみ、apr_array_push を使用して配列に追加します。 5. 文字列の終端に達するまでこの処理を繰り返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_filters.c:918
  • modules/http2/h2_c2_filter.c:234
  • いずれも apr_table_do(uniq_field_values, ...) を介して呼び出されます。
  • **呼び出し先**:
  • apr_pstrdup (APR)
  • apr_isspace (APR)
  • ap_cstr_casecmp (Apache Core)
  • apr_array_push (APR)

関連データ構造

  • apr_array_header_t: 重複除去された値のリストを保持するために使用されます。

失敗モードと境界条件

  • val が空文字列の場合、ループは即座に終了します。
  • val 内のトークンが既に配列内に存在する場合、そのトークンは無視されます。
  • メモリ割り当て失敗時は apr_pstrdup 等の APR 関数が例外を発生させる可能性があります。

保守担当者向けメモ

  • 本関数は Vary ヘッダーの正規化を目的として設計されています。
  • ap_array_strcasecmp のような専用の比較関数が存在しないため、ループによる線形探索を行っています。配列サイズが非常に大きい場合はパフォーマンスに影響する可能性があります。
