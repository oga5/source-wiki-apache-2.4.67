fd_queue_t

定義

  • 種別: struct
  • 定義箇所: server/mpm_fdqueue.h:76

目的

Apache HTTP Server の MPM（event や worker）において、リスナースレッドとワーカースレッド間でソケットおよびタイマーイベントを安全に受け渡すためのスレッドセーフなキュー構造を提供します。

シグネチャまたは形状

struct fd_queue_t {
    APR_RING_HEAD(timers_t, timer_event_t) timers;
    fd_queue_elem_t *data;
    unsigned int nelts;
    unsigned int bounds;
    unsigned int in;
    unsigned int out;
    apr_thread_mutex_t *one_big_mutex;
    apr_thread_cond_t *not_empty;
    volatile int terminated;
};

入力と出力

  • **入力**: ap_queue_push_socket や ap_queue_push_timer を介してソケットやタイマーイベントが投入されます。
  • **出力**: ap_queue_pop_something を介して、キュー内の要素がワーカースレッドへ取り出されます。

前提条件と不変条件

  • APR_HAS_THREADS が定義されている環境で利用されます。
  • キューへのアクセスは one_big_mutex によって排他制御されます。
  • nelts は現在の要素数を表し、bounds はキューの最大容量を表します。
  • in および out は、data 配列に対するリングバッファの挿入位置と取り出し位置を管理します。

アルゴリズムまたは振る舞い

  • **同期**: キューが空の状態で pop 操作が行われた場合、ワーカースレッドは not_empty 条件変数で待機します。
  • **リングバッファ**: data 配列をリングバッファとして使用し、in と out のインデックスを更新することで効率的な FIFO 処理を実現します。
  • **終了処理**: terminated フラグが設定されると、待機中のスレッドは処理を停止します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/event/event.c および server/mpm/worker/worker.c 内の worker_queue として静的に確保・利用されています。
  • **呼び出し先**: server/mpm_fdqueue.c 内の ap_queue_create, ap_queue_push_socket, ap_queue_push_timer, ap_queue_pop_something 等の関数によって操作されます。

関連データ構造

  • fd_queue_elem_t: キュー内の個々の要素（ソケット等）を保持する構造体。
  • timer_event_t: timers リングに格納されるタイマーイベント。
  • apr_thread_mutex_t, apr_thread_cond_t: 同期制御に使用される APR オブジェクト。

失敗モードと境界条件

  • **キューの満杯**: nelts が bounds に達した場合、追加操作はブロックされるかエラーを返します。
  • **キューの空**: nelts が 0 の場合、pop 操作は not_empty 条件変数で待機します。
  • **終了状態**: terminated が真の場合、キュー操作は即座に終了処理へ移行します。

保守担当者向けメモ

  • キュー操作は頻繁に行われるため、one_big_mutex の競合がパフォーマンスのボトルネックになる可能性があります。
  • server/mpm_fdqueue.c にあるユーティリティ関数（ap_queue_push_socket 等）が、この構造体の内部状態を直接操作する主要なインターフェースです。
