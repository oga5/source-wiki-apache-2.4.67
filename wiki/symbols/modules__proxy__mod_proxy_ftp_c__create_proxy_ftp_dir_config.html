create_proxy_ftp_dir_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:44

目的

FTP プロキシモジュールにおけるディレクトリ単位の設定構造体（proxy_ftp_dir_conf）を初期化し、デフォルト値を設定する。

シグネチャまたは形状

static void *create_proxy_ftp_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体のメモリ割り当てに使用する APR プール。
  • char *dummy: 未使用の引数。
  • **出力**:
  • void *: 初期化された proxy_ftp_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • 指定されたプール p が有効であること。
  • apr_pcalloc によりメモリがゼロクリアされた状態で構造体が確保される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、proxy_ftp_dir_conf サイズのメモリ領域を確保する。 2. <Location> ディレクティブ内でも設定が有効に機能するよう、以下のデフォルト値を設定する。

  • ftp_list_on_wildcard = 1
  • ftp_escape_wildcards = 1

    • 3. 確保した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ftp.c:2110 にて、ディレクトリ単位の設定構造体を作成するハンドラとして登録されている。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ関数）。

関連データ構造

  • proxy_ftp_dir_conf: FTP ディレクトリ設定を保持する構造体。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足により失敗した場合、APR の規約に従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は mod_proxy_ftp の設定フックの一部である。設定項目を追加する場合は、ここでデフォルト値を明示的に設定すること。
