am_create_conf

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_allowmethods.c:79

目的

ディレクトリ単位の設定構造体 am_conf_t を生成し、初期化を行う。

シグネチャまたは形状

static void *am_create_conf(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体の割り当てに使用するメモリプール。
  • char *dummy: Apache 設定フレームワークの規約による引数（使用されない）。
  • **出力**:
  • void *: 初期化された am_conf_t 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、メモリはゼロクリアされた状態で割り当てられる。
  • 戻り値の am_conf_t 構造体において、allowed および allowed_set フィールドは明示的に 0 に設定される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、メモリプール p から am_conf_t サイズのメモリ領域を確保する。 2. 確保した構造体の allowed メンバ（許可メソッドのビットマスク）を 0 に初期化する。 3. 確保した構造体の allowed_set メンバ（設定の有無を示すフラグ）を 0 に初期化する。 4. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_allowmethods.c:152（module 構造体の設定生成フックとして登録されている）。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ関数）。

関連データ構造

  • am_conf_t: 本関数によって生成・初期化されるディレクトリ設定構造体。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足により失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は Apache の設定生成フックとして機能する。am_conf_t 構造体のメンバに変更を加えた場合は、必ず本関数内で初期化処理を更新すること。
