ef_server_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_ext_filter.c:38

目的

Apache HTTP Server のサーバー単位における mod_ext_filter の設定情報を保持するための構造体です。

シグネチャまたは形状

typedef struct ef_server_t {
    apr_pool_t *p;
    apr_hash_t *h;
} ef_server_t;

入力と出力

  • **入力**: サーバー設定の初期化時や ExtFilterDefine ディレクティブの解析時に使用されます。
  • **出力**: 保持しているハッシュテーブル h を通じて、定義済みのフィルタ情報（ef_filter_t）を提供します。

前提条件と不変条件

  • p は、サーバーの生存期間に関連付けられた apr_pool_t である必要があります。
  • h は、フィルタ名をキーとして ef_filter_t 構造体を格納するハッシュテーブルとして初期化されている必要があります。

アルゴリズムまたは振る舞い

  • サーバー設定のコンテキストにおいて、ExtFilterDefine で定義された外部フィルタの集合を管理します。
  • apr_pcalloc を用いてメモリが割り当てられ、サーバーのライフサイクルを通じて設定情報を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_ef_server_conf (サーバー設定の生成時)
  • define_filter (ディレクティブ解析によるフィルタ登録時)
  • ef_init (初期化処理)
  • ef_output_filter / ef_input_filter (フィルタ適用時の設定取得)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ef_filter_t: ハッシュテーブル h に格納される、個別のフィルタ定義情報。
  • apr_hash_t: フィルタ名をキーとして管理するためのハッシュテーブル。
  • apr_pool_t: 設定情報のメモリ管理に使用されるプール。

失敗モードと境界条件

  • メモリ不足時に apr_pcalloc が失敗する可能性があります。
  • ハッシュテーブルの初期化に失敗した場合、フィルタ定義の登録や検索が不能となります。

保守担当者向けメモ

  • 本構造体はサーバー単位の設定を保持します。ディレクトリ単位の設定については ef_dir_t を参照してください。
  • フィルタの検索は h を介して行われるため、ハッシュキーの整合性に注意してください。
