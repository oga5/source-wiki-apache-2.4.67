find_best_hb

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_heartbeat.c:257

目的

外部から提供されるハートビート情報（サーバーの負荷状況）に基づき、負荷分散対象となる最適なバックエンドワーカーを選択する。

シグネチャまたは形状

static proxy_worker *find_best_hb(proxy_balancer *balancer, request_rec *r)

入力と出力

  • **入力**:
  • balancer: 負荷分散対象のバランサー構造体。
  • r: 現在のリクエスト情報。
  • **出力**:
  • 選択された proxy_worker へのポインタ。候補が見つからない場合は NULL を返す。

前提条件と不変条件

  • mod_proxy がロードされていること（ap_proxy_retry_worker_fn が取得可能であること）。
  • read_heartbeats により、指定されたパスから最新のサーバー負荷情報が取得できること。
  • 選択対象のワーカーは PROXY_WORKER_IS_USABLE マクロで判定可能な状態であること。
  • ハートビートの更新時刻（seen）が LBM_HEARTBEAT_MAX_LASTSEEN（10秒）以内であること。

アルゴリズムまたは振る舞い

1. **初期化**: リクエスト毎の一時プール（tpool）を作成し、ハートビート情報を格納するハッシュテーブルを初期化する。 2. **データ取得**: read_heartbeats を呼び出し、バックエンドサーバーの最新状態をハッシュに読み込む。 3. **候補選定**:

  • バランサー内の全ワーカーを走査し、ハートビート情報が存在し、かつ利用可能なワーカーを特定する。
  • 利用可能なワーカーの ready（待機中スレッド数）を合計し、openslots に加算する。

    • 4. **確率的選択**:

  • ap_random_pick(0, openslots) を使用し、各サーバーの ready 数に応じた重み付けランダム選択を行う。

    • 5. **終了処理**: 一時プールを破棄し、選択されたワーカーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_lbmethod_heartbeat.c 内の heartbeat 構造体（proxy_balancer_method）から参照される。
  • **呼び出し先**:
  • read_heartbeats: ハートビート情報の読み込み。
  • ap_proxy_retry_worker: 利用不可なワーカーの再試行。
  • ap_random_pick: 重み付けランダム選択。

関連データ構造

  • lb_hb_ctx_t: モジュール設定（ハートビート情報のパスなど）。
  • hb_server_t: 各サーバーの負荷状態（ready, seen 等）。
  • proxy_worker: mod_proxy が管理するバックエンドサーバー情報。

失敗モードと境界条件

  • **読み込み失敗**: read_heartbeats が失敗した場合、エラーログを出力し NULL を返す。
  • **候補なし**: 利用可能なサーバーが存在しない場合、または openslots が 0 の場合、NULL を返す。
  • **期限切れ**: seen が LBM_HEARTBEAT_MAX_LASTSEEN を超えているサーバーは、生存していないとみなされ選択対象から除外される。

保守担当者向けメモ

  • リクエスト毎にメモリプールを生成・破棄しているため、メモリリークの懸念は低い。
  • ready 値が 0 のサーバーは選択対象から外れる。
  • サーバーが起動直後で busy=0 かつ ready が大きい場合、過負荷を防ぐヒューリスティックな調整が read_heartbeats 側で行われている可能性があるため、挙動の変更時は注意すること。
