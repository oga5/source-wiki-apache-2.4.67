cache_create_entity

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_storage.c:72

目的

新しいキャッシュエンティティの作成を、登録されているキャッシュストレージプロバイダーに対して試行します。

シグネチャまたは形状

int cache_create_entity(cache_request_rec *cache, request_rec *r,
                        apr_off_t size, apr_bucket_brigade *in);

入力と出力

  • **入力**:
  • cache: リクエストに関連付けられたキャッシュコンテキスト。
  • r: 現在のリクエスト構造体。
  • size: 作成するエンティティのサイズ（apr_off_t）。
  • in: エンティティの初期データを含むバケットブリゲード。
  • **出力**:
  • OK: エンティティの作成に成功。
  • DECLINED: 適切なプロバイダーが見つからなかった、または作成を拒否された。
  • APR_EGENERAL: キャッシュコンテキストが不正。
  • その他: プロバイダーが返したエラーコード。

前提条件と不変条件

  • cache が NULL の場合、エラーとして APR_EGENERAL を返します。
  • cache->key が未設定の場合、内部で cache_generate_key を呼び出し、キーの生成を試みます。
  • すべてのメモリ割り当ては r->pool を使用します。

アルゴリズムまたは振る舞い

1. cache_handle_t 構造体を r->pool 上に確保します。 2. cache->key が存在しない場合、cache_generate_key を実行して生成します。 3. cache->providers リストを順次走査します。 4. 各プロバイダーの create_entity 関数を呼び出します。

  • OK が返された場合、cache->handle、cache->provider、cache->provider_name を更新して終了します。
  • DECLINED が返された場合、次のプロバイダーへ移行します。
  • それ以外の値が返された場合、その値を即座に返します。

    • 5. すべてのプロバイダーで作成できなかった場合、DECLINED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c:801
  • modules/cache/mod_cache.c:1332
  • **呼び出し先**:
  • cache_generate_key (内部関数)
  • list->provider->create_entity (各ストレージプロバイダーの実装)
  • ap_log_rerror (エラーログ出力)

関連データ構造

  • cache_request_rec: キャッシュの要求状態を保持。
  • cache_handle_t: 作成されたキャッシュエントリへのハンドル。
  • cache_provider_list: 利用可能なストレージプロバイダーのリスト。

失敗モードと境界条件

  • cache 引数が NULL の場合、ログを出力して APR_EGENERAL を返します。
  • キー生成に失敗した場合、そのエラーコードを返します。
  • 登録されたすべてのプロバイダーが DECLINED を返した場合、関数全体としても DECLINED を返します。

保守担当者向けメモ

  • 本関数はストレージの抽象化レイヤーであり、実際の I/O は各プロバイダー（mod_cache_disk 等）に委譲されます。
  • cache->handle の更新は、プロバイダーによる作成成功時にのみ行われます。
