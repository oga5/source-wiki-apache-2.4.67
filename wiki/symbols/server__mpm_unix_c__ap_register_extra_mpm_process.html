ap_register_extra_mpm_process

定義

  • 種別: function
  • 定義箇所: server/mpm_unix.c:82

目的

MPM（Multi-Processing Module）が管理するメインのプロセス群とは別に、追跡が必要な「追加のプロセス」をリストに登録します。

シグネチャまたは形状

void ap_register_extra_mpm_process(pid_t pid, ap_generation_t gen)

入力と出力

  • **入力**:
  • pid: 登録対象のプロセスID。
  • gen: プロセスが属する世代番号（ap_generation_t）。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • ap_malloc を使用してメモリを確保するため、メモリ不足時にはプロセスが終了する可能性があります。
  • 登録されたプロセスは、グローバルな静的変数 extras を先頭とする連結リストに追加されます。

アルゴリズムまたは振る舞い

1. ap_malloc を呼び出し、extra_process_t 構造体分のメモリを確保します。 2. 確保した構造体の pid および gen フィールドに引数の値を設定します。 3. 現在の extras リストの先頭ポインタを、新しく作成した構造体の next に代入します。 4. extras ポインタを更新し、新しく作成した構造体をリストの先頭に配置します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/worker/worker.c:429 からの呼び出しが確認されています。
  • **呼び出し先**: ap_malloc を呼び出します。

関連データ構造

  • extra_process_t: プロセスIDと世代番号、およびリストの次要素へのポインタを保持する構造体。
  • extras: 登録された extra_process_t のリストを保持するグローバルな静的変数。

失敗モードと境界条件

  • メモリ確保失敗時の挙動は ap_malloc の実装に依存します。
  • 連結リストの先頭に挿入する単純な実装であるため、登録順序は逆転します。

保守担当者向けメモ

  • 本関数で確保されたメモリは、対応する ap_unregister_extra_mpm_process 関数内で free される必要があります。
  • WIN32 環境では本ファイル（mpm_unix.c）の内容は無効化されるため、本関数も利用できません。
