ap_xml_parse_input

定義

  • 種別: function
  • 定義箇所: server/util_xml.c:36

目的

HTTP リクエストボディから XML データを読み込み、解析して apr_xml_doc 構造体として取得する。

シグネチャまたは形状

int ap_xml_parse_input(request_rec *r, apr_xml_doc **pdoc)

入力と出力

  • **入力**:
  • r: 現在のリクエスト状態を保持する request_rec 構造体。
  • pdoc: 解析結果の apr_xml_doc 構造体へのポインタを格納するアドレス。
  • **出力**:
  • 戻り値: 成功時は OK、失敗時は HTTP_BAD_REQUEST や HTTP_REQUEST_ENTITY_TOO_LARGE などの HTTP ステータスコード。
  • pdoc: 解析された XML ドキュメント（失敗時は NULL または未定義）。

前提条件と不変条件

  • r->input_filters を通じてリクエストボディが取得可能であること。
  • ap_get_limit_xml_body(r) により取得されるサイズ制限が適用される。
  • EBCDIC 環境では、解析後に文字コード変換が適用される。

アルゴリズムまたは振る舞い

1. apr_xml_parser_create でパーサーを初期化し、バケットブリゲードを準備する。 2. ap_get_brigade を使用してリクエストボディをストリーミングで読み込む。 3. 各バケットのサイズを累積し、LimitXMLRequestBody の制限値を超えた場合は HTTP_REQUEST_ENTITY_TOO_LARGE を返して終了する。 4. 読み込んだデータを apr_xml_parser_feed でパーサーに供給する。 5. EOS（End of Stream）到達後、apr_xml_parser_done を呼び出して解析を完了させる。 6. 解析エラー時は apr_xml_parser_geterror で詳細を取得しログ出力する。 7. 成功時は OK を返し、必要に応じて EBCDIC からの変換を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c 内の複数の箇所（WebDAV リクエスト処理時）。
  • **呼び出し先**:
  • apr_xml_parser_create, apr_xml_parser_feed, apr_xml_parser_done, apr_xml_parser_geterror (APR XML ライブラリ)
  • ap_get_brigade, ap_get_limit_xml_body, ap_log_rerror, ap_map_http_request_error (Apache HTTP Server コア)

関連データ構造

  • request_rec: リクエストのコンテキスト。
  • apr_xml_doc: 解析された XML ドキュメント構造。
  • apr_xml_parser: XML 解析用コンテキスト。
  • apr_bucket_brigade: リクエストボディのストリーミング用バッファ。

失敗モードと境界条件

  • **サイズ超過**: total_read が limit_xml_body を超えた場合、HTTP_REQUEST_ENTITY_TOO_LARGE を返す。
  • **解析エラー**: XML 形式が不正な場合、HTTP_BAD_REQUEST を返す。
  • **空のドキュメント**: total_read が 0 の場合、パーサーがエラーを返しても OK として扱う。
  • **読み込みエラー**: ap_get_brigade が失敗した場合、ap_map_http_request_error に基づくステータスを返す。

保守担当者向けメモ

  • 本関数は apr_xml のラッパーであり、HTTP リクエストのライフサイクルに特化している。
  • エラー発生時は必ず apr_xml_parser_done を呼び出し、パーサーのリソースを解放する設計となっている。
  • EBCDIC 環境での動作（APR_CHARSET_EBCDIC）が条件付きコンパイルに含まれている点に留意すること。
