create_access_compat_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_access_compat.c:78

目的

ディレクトリごとのアクセス制御設定構造体（access_compat_dir_conf）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_access_compat_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体の割り当てに使用するメモリプール。
  • char *dummy: 使用されない引数。
  • **出力**:
  • void *: 初期化された access_compat_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全フィールドはゼロ初期化される。
  • METHODS 定数に基づき、判定順序および satisfy ポリシーの配列が確保される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して access_compat_dir_conf サイズのメモリを確保する。 2. 全メソッドに対して、デフォルトの判定順序を DENY_THEN_ALLOW に設定する。 3. allows および denys 用のルール配列を apr_array_make で初期化する。 4. 全メソッドに対して、デフォルトの satisfy ポリシーを SATISFY_NOSPEC に設定する。 5. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_access_compat.c:371（ディレクトリ設定作成用ハンドラとして登録）。
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_array_make (APR)
  • apr_palloc (APR)

関連データ構造

  • access_compat_dir_conf: 本関数で初期化されるメインの設定構造体。
  • allowdeny: allows および denys 配列の要素型。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数が異常終了（abort）する可能性がある。

保守担当者向けメモ

  • 本関数は mod_access_compat の設定ライフサイクルの起点です。設定項目（Order, Allow, Deny, Satisfy）のデフォルト値変更が必要な場合は、ここを修正してください。
