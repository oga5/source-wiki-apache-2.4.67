ap_fcgi_encoded_env_len

定義

  • 種別: function
  • 定義箇所: server/util_fcgi.c:116

目的

FastCGI の PARAMS レコードとして環境変数をエンコードする際、指定されたバッファサイズ（maxlen）内に収まる環境変数の総バイト数を計算し、分割送信のための進捗インデックスを更新します。

シグネチャまたは形状

apr_size_t ap_fcgi_encoded_env_len(apr_table_t *env,
                                   apr_size_t maxlen,
                                   int *starting_elem)

入力と出力

  • **入力**:
  • env: エンコード対象の環境変数を保持する apr_table_t。
  • maxlen: 許容される最大バイト数。AP_FCGI_MAX_CONTENT_LEN を上限として制限されます。
  • starting_elem: 処理を開始する要素のインデックス（ポインタ渡し）。
  • **出力**:
  • 戻り値: バッファ内に収まる環境変数の総バイト数。
  • starting_elem: 処理が完了した要素数分だけインクリメントされます。

前提条件と不変条件

  • maxlen が AP_FCGI_MAX_CONTENT_LEN を超える場合、自動的に上限値に切り詰められます。
  • FastCGI の長さフィールド（キーおよび値の長さ）は、127以下の場合は1バイト、それ以上の場合は4バイトとして計算されます。
  • env 内のキーが NULL の要素はスキップされます。

アルゴリズムまたは振る舞い

1. starting_elem で指定された位置から apr_table_t を順次走査します。 2. 各要素のキーと値の長さを取得し、FastCGI 形式（長さフィールド＋データ本体）に基づいた必要バイト数を算出します。 3. 累積バイト数が maxlen を超える直前まで要素を処理します。 4. 収まりきらない要素に到達した時点でループを終了し、それまでに計算した総バイト数を返します。 5. 処理した要素数分だけ starting_elem を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:333
  • modules/proxy/mod_proxy_fcgi.c:475
  • **呼び出し先**:
  • apr_table_elts (APR ライブラリ関数)
  • strlen (標準 C ライブラリ関数)

関連データ構造

  • apr_table_t: 環境変数を保持する Apache Portable Runtime のテーブル構造。
  • apr_table_entry_t: テーブル内の個々のキーと値のペア。

失敗モードと境界条件

  • バッファサイズが極端に小さい場合、1つの環境変数もエンコードできず、戻り値が 0 となる可能性があります。
  • env が空の場合、ループは実行されず 0 が返されます。
  • キーが NULL の要素は無視され、インデックスのみが進められます。

保守担当者向けメモ

  • 本関数はメモリ割り当てを行いません。
  • FastCGI の可変長長さフィールド（1バイトまたは4バイト）の判定ロジックは、ap_fcgi_encode_env と一貫している必要があります。修正時は両関数を確認してください。
