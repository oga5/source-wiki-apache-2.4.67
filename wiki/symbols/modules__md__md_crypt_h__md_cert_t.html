md_cert_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_crypt.h:133

目的

X.509 証明書をラップし、mod_md モジュール内で一貫した形式で取り扱うための不透明な構造体です。

シグネチャまたは形状

typedef struct md_cert_t md_cert_t;

入力と出力

  • 本構造体は、md_cert_make() や md_cert_wrap() を通じて生成・初期化されます。
  • md_cert_get_not_before() などのアクセサ関数を通じて、証明書の属性情報（有効期限など）を出力します。

前提条件と不変条件

  • apr_pool_t に紐付いて管理されます。
  • md_cert_make() で生成された場合、メモリ所有権はプールに帰属し自動解放されます。
  • md_cert_wrap() で生成された場合、所有権は移転せず、呼び出し側が OpenSSL オブジェクトの寿命を管理する必要があります。

アルゴリズムまたは振る舞い

  • OpenSSL の X.509 構造体を内部で保持し、mod_md の各機能（証明書の検証、ドメイン整合性チェック、ACME プロトコルでの利用）に抽象化されたインターフェースを提供します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_authz.c
  • modules/md/md_acme_drive.c
  • modules/md/md_acmev2_drive.c
  • **呼び出し先**:
  • 本構造体は OpenSSL の X.509 構造体をラップしており、内部的に OpenSSL API を利用します。

関連データ構造

  • md_cert_state_t: 証明書の有効期限状態を判定するために使用されます。
  • apr_array_header_t: 証明書チェーンを保持するために md_cert_t* 型の配列として利用されます。

失敗モードと境界条件

  • 構造体自体の定義は不透明であるため、メモリ割り当て失敗時や無効な証明書データが渡された際の挙動は、生成関数（md_cert_make 等）の戻り値およびエラー処理に依存します。

保守担当者向けメモ

  • 本構造体は OpenSSL の実装詳細を隠蔽する目的で定義されています。
  • md_cert_t を扱う際は、メモリ所有権の管理戦略（make か wrap か）を常に意識してください。
  • CT（Certificate Transparency）情報の抽出など、OpenSSL のバージョン依存の処理が含まれる場合があります。
