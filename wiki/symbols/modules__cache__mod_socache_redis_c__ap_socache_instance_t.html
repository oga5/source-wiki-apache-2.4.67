ap_socache_instance_t

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_redis.c:69 (および ap_socache.h:49)

目的

Redis バックエンドを用いた共有オブジェクトキャッシュ（socache）のコンテキストを保持するための構造体です。接続先サーバー情報、Redis クライアントハンドル、および名前空間タグを管理します。

シグネチャまたは形状

struct ap_socache_instance_t {
    const char *servers;      /* 接続先サーバーリスト */
    apr_redis_t *rc;          /* Redis クライアントハンドル */
    const char *tag;          /* 名前空間タグ */
    apr_size_t taglen;        /* strlen(tag) + 1 */
};

入力と出力

  • 本構造体は、socache_rd_create で初期化され、socache_rd_init で apr_redis_t ハンドルが設定されます。
  • 各種プロバイダー関数（store, retrieve, remove 等）の引数として渡され、キャッシュ操作の対象となる Redis インスタンスを特定するために使用されます。

前提条件と不変条件

  • socache_rd_init 内において、ctx->taglen が 16 バイト以下である必要があります（AP_DEBUG_ASSERT による検証）。

アルゴリズムまたは振る舞い

  • mod_socache_redis において、Redis サーバーとの通信に必要な接続プール（apr_redis_t）と、キーの衝突を防ぐための名前空間（tag）を保持します。
  • キャッシュ操作時には、tag を用いて Redis キーを生成し、rc を介して apr_redis ライブラリの関数を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cache_socache.c や mod_authn_socache.c などのキャッシュを利用するモジュールが、ap_socache API を通じて本構造体を参照します。
  • **呼び出し先**: apr_redis ライブラリの関数（apr_redis_setex, apr_redis_getp, apr_redis_delete 等）が、本構造体内の rc を使用して呼び出されます。

関連データ構造

  • apr_redis_t: apr_redis ライブラリが提供する Redis 接続ハンドル。
  • socache_rd_svr_cfg: サーバーごとの接続設定を保持する構造体。

失敗モードと境界条件

  • taglen が 16 バイトを超過する場合、デバッグビルドではアサーションエラーが発生します。
  • rc が初期化されていない状態でキャッシュ操作関数が呼び出された場合、未定義の動作またはエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は ap_socache.h で定義された汎用インターフェースの一部であり、mod_socache_redis.c 以外にも mod_socache_dbm.c 等で異なる実装として定義される場合があります。
  • メモリ管理は Apache のリクエストプールに依存しているため、キャッシュ取得時のメモリ消費量には注意が必要です。
