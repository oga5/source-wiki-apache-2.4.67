md_store_purge

定義

  • 種別: function
  • 定義箇所: modules/md/md_store.c:93

目的

指定されたストレージグループ内の特定のドメインに関連するデータ（証明書、秘密鍵、チャレンジデータ等）をストレージから削除します。

シグネチャまたは形状

apr_status_t md_store_purge(md_store_t *store, apr_pool_t *p, md_store_group_t group, const char *name)

入力と出力

  • store: 操作対象の md_store_t 抽象化インターフェースへのポインタ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • group: 削除対象のストレージグループ（例: MD_SG_CHALLENGES, MD_SG_STAGING）。
  • name: 削除対象のドメイン名。
  • 戻り値: 操作の成功または失敗を示す apr_status_t 型のステータスコード。

前提条件と不変条件

  • store は有効な md_store_t インスタンスであり、その purge 関数ポインタが適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 引数として渡された store 構造体が保持する関数ポインタ store->purge を呼び出します。 2. 実際の削除ロジック（ファイルシステム上の削除など）は、store に関連付けられた具体的なバックエンド実装（md_store_fs.c 等）に委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_authz.c
  • modules/md/md_acme_drive.c
  • modules/md/md_reg.c
  • modules/md/mod_md_drive.c
  • **呼び出し先**:
  • store->purge 関数ポインタ（具体的なストレージ実装に依存）。

関連データ構造

  • md_store_t: ストレージ操作の抽象化インターフェース。
  • md_store_group_t: ストレージ内のデータ分類（グループ）を定義する列挙型。

失敗モードと境界条件

  • store が NULL の場合、または store->purge が初期化されていない場合は未定義の動作（クラッシュ）を引き起こす可能性があります。
  • 指定された group や name が存在しない場合、バックエンドの実装に応じてエラーコードが返されるか、あるいは正常終了（削除対象なし）として扱われます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のI/O処理は含まれていません。
  • 削除処理の挙動を調査する場合は、md_store_t を実装している md_store_fs.c 等の purge 実装を確認してください。
