reflector_cfg

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_reflector.c:29

目的

ディレクトリ単位の設定情報を保持するための構造体です。主に、リクエストヘッダーからレスポンスヘッダーへマッピングするルールを管理するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_table_t *headers;
} reflector_cfg;

入力と出力

  • **入力**: 設定ディレクティブ（ReflectorHeader）を通じて、ヘッダーマッピングのルールが headers テーブルに格納されます。
  • **出力**: リクエスト処理時に ap_get_module_config を介して取得され、ヘッダーの転送ロジックで使用されます。

前提条件と不変条件

  • headers メンバは apr_table_t 型であり、APRのメモリプール管理下で初期化される必要があります。

アルゴリズムまたは振る舞い

  • create_reflector_dir_config 関数内で apr_pcalloc を用いてメモリが割り当てられます。
  • merge_reflector_dir_config 関数内で、ベース設定と追加設定の統合に使用されます。
  • reflector_handler 内で、現在のリクエストに対するディレクトリ設定として参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_reflector_dir_config (メモリ確保)
  • merge_reflector_dir_config (設定統合)
  • reflector_handler (設定取得)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_table_t: ヘッダーのキーと値のペアを保持するために使用されます。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗する可能性があります。
  • 設定が未定義の場合、headers メンバが NULL になる可能性があるため、アクセス時には適切なチェックが必要です。

保守担当者向けメモ

  • 本構造体は mod_reflector の設定管理の基盤です。新しい設定項目を追加する場合は、create_reflector_dir_config および merge_reflector_dir_config の両方で初期化・統合処理を更新してください。
