status_pre_config

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_status.c:992

目的

mod_status モジュールが読み込まれた際に、サーバーの拡張ステータス収集設定（ExtendedStatus）をデフォルトで有効化する。

シグネチャまたは形状

static int status_pre_config(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp)

入力と出力

  • 入力:
  • p, plog, ptemp: Apache のメモリプール（APR）。
  • 出力:
  • int: 常に OK を返す。

前提条件と不変条件

  • 本関数は Apache の設定フェーズ（pre_config）で実行される。
  • ap_extended_status グローバル変数が書き込み可能であること。

アルゴリズムまたは振る舞い

1. グローバル変数 ap_extended_status に 1 を代入し、拡張ステータス収集を強制的に有効化する。 2. OK を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/generators/mod_status.c:1039 にて ap_hook_pre_config に登録されている。
  • 呼び出し先: なし。

関連データ構造

  • ap_extended_status: サーバー全体で共有される拡張ステータス収集の有効/無効フラグ。

失敗モードと境界条件

  • 失敗モード: なし（常に OK を返す）。
  • 境界条件: 他のモジュールが同様のフックで ap_extended_status を上書きした場合、最終的な設定値はフックの実行順序に依存する。

保守担当者向けメモ

  • 本関数は mod_status がロードされた際に、詳細なスコアボード情報が必要となることを前提としてデフォルト値を上書きしている。
  • MPM モジュールが独自の要件でスコアボードの形式を制御する場合、本設定と競合する可能性がある点に留意すること。
