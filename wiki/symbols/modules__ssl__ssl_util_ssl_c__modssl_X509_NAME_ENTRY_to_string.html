modssl_X509_NAME_ENTRY_to_string

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_ssl.c:223

目的

OpenSSL の X509_NAME_ENTRY 構造体から、Apache のメモリ管理下にある文字列へ変換を行う。

シグネチャまたは形状

char *modssl_X509_NAME_ENTRY_to_string(apr_pool_t *p, X509_NAME_ENTRY *xsne, int raw);

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • xsne: 変換対象の X509_NAME_ENTRY 構造体。
  • raw: 変換モードを指定するフラグ（0 または 1）。
  • **出力**:
  • 変換された文字列を指す char *。メモリは p から割り当てられる。

前提条件と不変条件

  • xsne は有効な X509_NAME_ENTRY を指している必要がある。
  • 返される文字列は apr_pool_t の生存期間に依存する。

アルゴリズムまたは振る舞い

1. X509_NAME_ENTRY_get_data(xsne) を呼び出し、ASN.1 データを取得する。 2. asn1_string_convert を呼び出し、ASN.1 データを文字列に変換する。 3. ap_xlate_proto_from_ascii を呼び出し、結果の文字列を ASCII プロトコル形式へ変換する。 4. 変換後の文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_vars.c (2箇所)
  • modules/ssl/ssl_util_ssl.c (内部呼び出し)
  • **呼び出し先**:
  • asn1_string_convert
  • X509_NAME_ENTRY_get_data
  • ap_xlate_proto_from_ascii

関連データ構造

  • X509_NAME_ENTRY: OpenSSL の証明書名エントリ構造体。
  • apr_pool_t: Apache のメモリプール。

失敗モードと境界条件

  • asn1_string_convert が失敗した場合の挙動については、当該関数の実装に依存する。
  • xsne が NULL の場合の挙動は、OpenSSL の X509_NAME_ENTRY_get_data の仕様に準拠する。

保守担当者向けメモ

  • ap_xlate_proto_from_ascii の引数 len はソースコード上で未定義のように見えるため、コンパイル環境やマクロ定義の確認が必要である。
  • 本関数は ssl_util_ssl.h で公開されている。
