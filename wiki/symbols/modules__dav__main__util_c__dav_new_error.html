dav_new_error

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:36

目的

WebDAV 処理中に発生したエラー情報を保持する dav_error 構造体を生成し、初期化します。

シグネチャまたは形状

dav_error* dav_new_error(apr_pool_t *p, int status, int error_id, apr_status_t aprerr, const char *desc);

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • status: HTTP レスポンスステータスコード。
  • error_id: DAV 固有のエラー識別子。
  • aprerr: APR 関連のエラーコード。
  • desc: エラー内容を説明する文字列。
  • **出力**:
  • 初期化済みの dav_error* 構造体へのポインタ。

前提条件と不変条件

  • p は有効なメモリプールである必要があります。
  • 生成された dav_error 構造体は、指定されたプール p の生存期間に依存します。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、プール p 上に dav_error 構造体分のメモリをゼロクリアして確保します。 2. 引数として渡された status、error_id、desc、aprerr を構造体の各メンバに代入します。 3. 初期化された構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/dbm.c (複数箇所)
  • modules/dav/fs/lock.c (複数箇所)
  • **呼び出し先**:
  • apr_pcalloc (APR ライブラリ関数)

関連データ構造

  • dav_error: エラー情報を保持するメイン構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボートする可能性があるため、本関数はメモリ確保の失敗を個別にハンドリングしません。

保守担当者向けメモ

  • 本関数はエラーオブジェクトの生成のみを担当します。エラーの連結やスタック管理が必要な場合は、dav_push_error や dav_join_error を併用してください。
  • desc 文字列はコピーされずポインタが保持されるため、呼び出し元で渡す文字列の生存期間に注意してください。
