init_store_file

定義

  • 種別: function
  • 定義箇所: modules/md/md_store_fs.c:110

目的

ファイルシステムベースのストレージにおいて、新しいストアファイル（メタデータファイル）を初期化し、初期設定とランダムなキーを生成してディスクへ書き込む。

シグネチャまたは形状

static apr_status_t init_store_file(md_store_fs_t *s_fs, const char *fname, 
                                    apr_pool_t *p, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • s_fs: ストレージ状態を保持する md_store_fs_t 構造体へのポインタ。
  • fname: 作成するファイルパス。
  • p: メモリ割り当て用のプール。
  • ptemp: 一時的な処理に使用するプール。
  • **出力**:
  • apr_status_t: 操作の成功または失敗を示すステータスコード。

前提条件と不変条件

  • s_fs は有効なポインタであること。
  • md_rand_bytes が成功し、s_fs->key に十分なエントロピーが供給されること。

アルゴリズムまたは振る舞い

1. md_json_create で新しい JSON オブジェクトを生成する。 2. JSON にストアバージョン情報（MD_STORE_VERSION）を設定する。 3. md_data_pinit で s_fs->key を初期化し、md_rand_bytes を用いてランダムなバイト列を生成する。 4. 生成したキーを Base64URL エンコードし、JSON のキー情報として格納する。 5. md_json_fcreatex を呼び出し、指定されたパス fname に JSON をファイルとして書き出す（権限は MD_FPROT_F_UONLY を使用）。 6. セキュリティのため、一時的に使用した key64 文字列をメモリ上でゼロクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_store_fs.c:289 （APR_STATUS_IS_EEXIST チェックを伴う条件分岐内で呼び出される）。
  • **呼び出し先**: md_json_create, md_json_setn, md_data_pinit, md_rand_bytes, md_util_base64url_encode, md_json_sets, md_json_fcreatex, memset, strlen。

関連データ構造

  • md_store_fs_t: ストレージのベースパスやキー情報を保持するメイン構造体。
  • md_json_t: ストアファイルのメタデータ構造。

失敗モードと境界条件

  • md_rand_bytes が失敗した場合、そのエラーコードを返して終了する。
  • md_json_fcreatex がファイル作成に失敗した場合（既存ファイルとの競合や権限不足など）、そのステータスを返す。

保守担当者向けメモ

  • 本関数はファイル作成時にのみ実行される初期化ルーチンである。
  • key64 のゼロクリアは、機密情報がメモリ上に残存する時間を最小化するために行われている。
