ssl_crlcheck_t

定義

  • 種別: enum
  • 定義箇所: modules/ssl/ssl_private.h:484

目的

SSL/TLS 接続における証明書失効リスト（CRL）の検証範囲および動作オプションを定義する列挙型です。

シグネチャまたは形状

typedef enum {
    SSL_CRLCHECK_NONE  = (0),
    SSL_CRLCHECK_LEAF  = (1 << 0),
    SSL_CRLCHECK_CHAIN = (1 << 1),
    SSL_CRLCHECK_NO_CRL_FOR_CERT_OK = (1 << 2)
} ssl_crlcheck_t;

入力と出力

  • 入力: mod_ssl の設定プロセスにおいて、CRL 検証モードを指定するために使用されます。
  • 出力: 指定されたフラグに基づき、OpenSSL の検証コンテキストに対する CRL チェックの挙動を制御します。

前提条件と不変条件

  • SSL_CRLCHECK_NONE は CRL チェックを行わないことを示します。
  • SSL_CRLCHECK_LEAF および SSL_CRLCHECK_CHAIN はビットフラグとして機能し、検証対象をリーフ証明書のみ、または証明書チェーン全体に拡張します。
  • SSL_CRLCHECK_FLAGS マクロ（~0x3）は、基本の検証範囲フラグを除外したオプションビットを識別するために使用されます。

アルゴリズムまたは振る舞い

  • SSL_CRLCHECK_NO_CRL_FOR_CERT_OK: CRL が見つからない場合に検証を失敗させず、成功とみなすオプション動作を規定します。
  • 各値はビット演算による組み合わせが可能です。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋からは特定できません。

関連データ構造

  • SSLSrvConfigRec（仮想ホスト設定）や SSLDirConfigRec（ディレクトリ設定）内で、CRL 検証設定を保持するために使用されることが想定されます。

失敗モードと境界条件

  • SSL_CRLCHECK_FLAGS マクロは、定義された基本フラグ（0x3）以外のビットをマスクするために使用されます。これ以外の値が渡された場合の挙動は、この列挙型の定義範囲外です。

保守担当者向けメモ

  • 本シンボルは OpenSSL の検証ロジックと密接に関連しています。新しいフラグを追加する場合は、既存のビットマスク（0x3）や SSL_CRLCHECK_FLAGS との競合を避ける必要があります。
