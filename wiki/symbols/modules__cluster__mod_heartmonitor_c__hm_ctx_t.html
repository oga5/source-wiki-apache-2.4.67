hm_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/cluster/mod_heartmonitor.c:55

目的

mod_heartmonitor モジュールにおける、サーバー単位の設定および実行時の状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct hm_ctx_t
{
    int active;
    const char *storage_path;
    ap_watchdog_t *watchdog;
    apr_interval_time_t interval;
    apr_sockaddr_t *mcast_addr;
    apr_status_t status;
    volatile int keep_running;
    apr_socket_t *sock;
    apr_pool_t *p;
    apr_hash_t *servers;
    server_rec *s;
} hm_ctx_t;

入力と出力

本構造体は関数引数として各処理へ渡され、状態の参照や更新に使用されます。

  • **入力**: モジュール初期化時および実行時の設定値（storage_path, interval, mcast_addr 等）。
  • **出力**: 監視ループ内でのソケット状態、サーバー情報のハッシュテーブル（servers）、および実行ステータス。

前提条件と不変条件

  • p (メモリプール) は、モジュールのライフサイクルに合わせて適切に管理される必要があります。
  • servers ハッシュテーブルは、ハートビートメッセージの受信に伴い hm_get_server 等を通じて動的に更新されます。
  • keep_running は監視タスクの継続制御に使用されるため、スレッドセーフなアクセスが求められます。

アルゴリズムまたは振る舞い

  • mod_watchdog を介したバックグラウンドタスクの実行状態を管理します。
  • UDP ソケット（sock）を介したハートビート信号の受信と、解析結果に基づく servers ハッシュテーブルの更新を行います。
  • 定期的に hm_update_stats を呼び出し、収集したサーバー情報を共有メモリ（slotmem）またはファイルへ永続化します。

呼び出し元と呼び出し先

  • **呼び出し元**: hm_listen, hm_file_update_stat, hm_update_stat, hm_file_update_stats, hm_slotmem_update_stats, hm_update_stats, hm_get_server, hm_processmsg, hm_recv 等のモジュール内関数。
  • **呼び出し先**: 構造体内部の各フィールド（watchdog, sock, servers 等）が各処理で参照・操作されます。

関連データ構造

  • hm_server_t: servers ハッシュテーブル内に格納される個々のバックエンドサーバーの状態。
  • ap_watchdog_t: バックグラウンド監視タスクの制御用。
  • apr_hash_t: サーバーリストの管理に使用。
  • server_rec: Apache HTTP Server のサーバー設定情報。

失敗モードと境界条件

  • status フィールドにより、ソケットの初期化や通信時のエラー状態を保持します。
  • storage_path が指定されていない場合や、共有メモリへの書き込み失敗時に hm_update_stats がエラーを返す可能性があります。

保守担当者向けメモ

  • hm_ctx_t は apr_pool_t を保持しており、メモリリークを防ぐため、モジュールの終了処理時に適切にプールが破棄されることを確認してください。
  • servers ハッシュテーブルの操作は、mod_watchdog のコールバック内で行われるため、他のスレッドからのアクセス競合に注意が必要です。
  • HM_UPDATE_SEC 等の定数については、将来的に実行時設定へ移行する計画があります。
