pass_response

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2_filter.c:529

目的

HTTP/1.1 形式で解析されたレスポンスデータを、HTTP/2 構造（ヘッダーバケット）へ変換し、フィルタチェーンを通じて後続のフィルタへ送信する。

シグネチャまたは形状

static apr_status_t pass_response(h2_conn_ctx_t *conn_ctx, ap_filter_t *f, h2_response_parser *parser)

入力と出力

  • **入力**:
  • conn_ctx: 現在のコネクションコンテキスト。
  • f: 現在の出力フィルタ。
  • parser: 解析済みのレスポンス状態を保持する h2_response_parser 構造体。
  • **出力**:
  • apr_status_t: フィルタチェーンの処理結果（ap_pass_brigade の戻り値）。

前提条件と不変条件

  • parser 内の http_status および hlines が、有効な HTTP/1.1 レスポンスとして解析済みであること。
  • parser->tmp ブリゲードが利用可能であること。

アルゴリズムまたは振る舞い

1. parser の情報を用いて h2_headers オブジェクトを生成する。 2. parser->tmp ブリゲードを初期化し、生成したヘッダーバケットとフラッシュバケットを挿入する。 3. ap_pass_brigade を呼び出し、後続のフィルタへデータを渡す。 4. 処理完了後、parser の状態を H2_RP_STATUS_LINE にリセットし、ヘッダー行配列をクリアして次のレスポンス解析に備える。 5. レスポンスステータスが 200 以上の場合、conn_ctx->has_final_response を 1 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c2_filter.c 内の解析ロジック（parse_response 等のフロー内）。
  • **呼び出し先**:
  • h2_headers_create, make_table, h2_bucket_headers_create, apr_brigade_cleanup, apr_bucket_flush_create, APR_BRIGADE_INSERT_TAIL, ap_pass_brigade, apr_array_clear, ap_log_cerror。
  • 注: modules/proxy/mod_proxy_scgi.c からの呼び出しが確認されていますが、これは本ファイル内の pass_response と同一名称の別関数である可能性が高く、直接的な関係は不明です。

関連データ構造

  • h2_conn_ctx_t: コネクションレベルのメタデータ管理。
  • h2_response_parser: レスポンス解析の進捗と状態を保持。
  • h2_headers: HTTP/2 レスポンスヘッダーの抽象化。

失敗モードと境界条件

  • ap_pass_brigade がエラーを返した場合、そのステータスコードをそのまま呼び出し元へ伝播させる。
  • parser->tmp の操作中にメモリ不足が発生した場合、APR の標準的なエラー挙動に従う。

保守担当者向けメモ

  • 本関数は h2_c2_filter におけるレスポンス送信の最終ステップを担う。
  • parser の状態リセットは、同一コネクション上で複数のレスポンスが連続して発生する HTTP/2 の特性をサポートするために必須である。
