http_scheme

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:113

目的

現在のリクエストに関連付けられた URI スキーム（"http" または "https"）を特定し、文字列として返します。

シグネチャまたは形状

static const char *http_scheme(const request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト構造体）
  • **出力**: const char *（スキーム名を表す文字列リテラル "http" または "https"）

前提条件と不変条件

  • r および r->server が有効なポインタであることを前提とします。
  • 戻り値は常に "http" または "https" のいずれかです。

アルゴリズムまたは振る舞い

1. r->server->server_scheme を確認します。 2. server_scheme が "https" と一致する場合、"https" を返します。 3. それ以外の場合（server_scheme が NULL または "https" 以外の場合）、デフォルト値として "http" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/protocol.c 内の AP_IMPLEMENT_HOOK_RUN_FIRST マクロによって実装されたフック経由で呼び出されます。
  • **呼び出し先**: strcmp（標準 C ライブラリ関数）を呼び出します。

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。
  • server_rec: r->server を介してアクセスされるサーバー設定情報。

失敗モードと境界条件

  • r->server->server_scheme が NULL の場合、安全に "http" を返します。
  • 本関数自体はエラーを返しません。

保守担当者向けメモ

  • 本関数は ap_hook_http_scheme を通じて登録されており、APR_HOOK_REALLY_LAST の優先順位で実行されます。
  • スキームの判定ロジックを変更する場合は、http_protocol.h で定義されているフックの仕様と整合性を保つ必要があります。
