rule_return_type

定義

  • 種別: enum
  • 定義箇所: modules/mappers/mod_rewrite.c:314

目的

RewriteRule の評価結果を表現し、後続の処理フロー（置換の実行、ステータス設定、またはスキップ）を制御するために使用されます。

シグネチャまたは形状

typedef enum {
  RULE_RC_NOMATCH = 0,
  RULE_RC_MATCH = 1,
  RULE_RC_NOSUB = 2,
  RULE_RC_STATUS_SET = 3
} rule_return_type;

入力と出力

  • **出力**: apply_rewrite_rule 関数がこの列挙型を返し、ルール適用プロセスの状態を呼び出し元に伝達します。

前提条件と不変条件

  • 各値は mod_rewrite のルールエンジンにおける特定の評価状態（不一致、置換ありの一致、置換なしの一致、エラー状態）と一対一で対応しています。

アルゴリズムまたは振る舞い

  • RULE_RC_NOMATCH: ルールが条件に一致しなかったことを示します。
  • RULE_RC_MATCH: ルールが一致し、置換処理が実行されたことを示します。
  • RULE_RC_NOSUB: ルールには一致したが、置換は行われなかったことを示します。
  • RULE_RC_STATUS_SET: ルールが一致し、r->status に HTTP エラーコードが設定されたことを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: apply_rewrite_rule 関数（modules/mappers/mod_rewrite.c:4356）がこの型を戻り値として使用しています。
  • **呼び出し先**: 呼び出し先に関する情報は提供されていません。

関連データ構造

  • rewriterule_entry: ルール定義を保持する構造体。
  • rewrite_ctx: ルール適用時のリクエスト状態を保持する構造体。

失敗モードと境界条件

  • RULE_RC_STATUS_SET は、ルール適用によってリクエストのステータスが変更された場合に発生する特殊な終了状態です。

保守担当者向けメモ

  • 本シンボルは mod_rewrite のルールエンジンにおける制御フローの基盤です。新しいルールタイプや評価ロジックを追加する際は、この列挙型の網羅性を確認してください。
