p_save

定義

  • 種別: static function
  • 定義箇所: modules/md/md_acme_order.c:197

目的

ACME Order オブジェクトを JSON 形式に変換し、md_store を介して永続ストレージへ保存するコールバック関数。

シグネチャまたは形状

static apr_status_t p_save(void *baton, apr_pool_t *p, apr_pool_t *ptemp, va_list ap)

入力と出力

  • **入力**:
  • baton: md_store_t へのポインタ。
  • p: メインのメモリプール（未使用）。
  • ptemp: 一時的なメモリプール。
  • ap: 可変長引数リスト。以下の順序で取得:

    • 1. group (md_store_group_t): 保存先のグループ。 2. md_name (const char *): 管理対象ドメイン名。 3. set (md_acme_order_t *): 保存対象の Order オブジェクト。 4. create (int): 新規作成フラグ。

  • **出力**: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • set は有効な md_acme_order_t 構造体を指していること。
  • md_acme_order_to_json が成功し、非 NULL の JSON オブジェクトを返すこと（内部で assert により検証）。

アルゴリズムまたは振る舞い

1. 可変長引数から保存に必要なコンテキスト（グループ、ドメイン名、Order オブジェクト、作成フラグ）を抽出する。 2. md_acme_order_to_json を呼び出し、ptemp を使用して md_acme_order_t を JSON 形式にシリアライズする。 3. md_store_save_json を呼び出し、指定されたグループおよびドメイン名の配下に、MD_FN_ORDER というファイル名で JSON データを書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_order.c:221 (md_util_pool_vdo を介して呼び出し)
  • modules/md/md_store.c:224 (md_util_pool_vdo を介して呼び出し)
  • **呼び出し先**:
  • md_acme_order_to_json
  • md_store_save_json

関連データ構造

  • md_acme_order_t: ACME Order の状態を保持する構造体。
  • md_store_t: 永続化ストレージのハンドル。
  • md_json_t: JSON データのコンテナ。

失敗モードと境界条件

  • md_acme_order_to_json が失敗した場合、assert によりプログラムが停止する可能性がある。
  • md_store_save_json がディスク I/O エラーや権限エラーを返した場合、そのステータスがそのまま呼び出し元に伝播される。

保守担当者向けメモ

  • 本関数は md_util_pool_vdo から呼び出されることを前提としたシグネチャを持つ。
  • p (メインプール) は使用されていないが、md_util_pool_vdo のインターフェース仕様により引数に含まれている。
  • 永続化の詳細は md_store モジュールに依存しているため、保存先のディレクトリ構造やファイル命名規則を変更する場合は md_store.h の定義を確認すること。
