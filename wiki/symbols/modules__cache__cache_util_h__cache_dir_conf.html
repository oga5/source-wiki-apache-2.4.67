cache_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/cache/cache_util.h:159

目的

ディレクトリ単位（<Directory> や <Location> 等）で適用されるキャッシュ設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_time_t minex;
    apr_time_t maxex;
    apr_time_t defex;
    double factor;
    apr_array_header_t *cacheenable;
    unsigned int disable:1;
    unsigned int x_cache:1;
    unsigned int x_cache_detail:1;
    unsigned int stale_on_error:1;
    unsigned int no_last_mod_ignore:1;
    unsigned int store_expired:1;
    unsigned int store_private:1;
    unsigned int store_nostore:1;
    /* 各設定値の有効フラグ（_set）群 */
    unsigned int minex_set:1;
    /* ... (他多数) */
} cache_dir_conf;

入力と出力

  • 本構造体は ap_get_module_config(r->per_dir_config, &cache_module) を通じてリクエスト処理中に取得されます。
  • 設定の初期化には apr_pcalloc が使用され、リクエストプールに紐付けられます。

前提条件と不変条件

  • 各設定項目には、明示的に設定されたかどうかを示す _set フラグ（例: minex_set）が存在します。
  • メモリはリクエストプール（apr_pool_t）から割り当てられるため、手動解放は不要です。

アルゴリズムまたは振る舞い

  • キャッシュの有効期限（minex, maxex, defex）や、Cache-Control ヘッダーの無視設定（store_private, store_nostore 等）を保持し、キャッシュの鮮度判定や保存可否の判断基準として利用されます。
  • x_cache および x_cache_detail フラグにより、レスポンスヘッダーへのキャッシュ状態情報の付与を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cache.c 内の各フック関数（設定の取得やマージ処理）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • cache_server_conf: サーバー全体の設定を保持する構造体。
  • cache_request_rec: リクエストごとのキャッシュコンテキスト。

失敗モードと境界条件

  • 設定が未定義の場合、_set フラグが 0 となり、デフォルト値が適用される設計となっています。
  • 構造体内のビットフィールドは、設定の有無を厳密に区別するために使用されます。

保守担当者向けメモ

  • 本構造体は mod_cache の設定マージ処理（merge_dir_config）において、親ディレクトリの設定とマージされる前提で設計されています。
  • 新しい設定項目を追加する際は、対応する _set フラグを必ず追加し、マージロジックを更新してください。
