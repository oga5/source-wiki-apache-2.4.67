compsub

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:673

目的

sed コマンドの s（置換）コマンドにおける置換文字列（RHS: Right Hand Side）を解析し、内部表現へ変換します。

シグネチャまたは形状

static char *compsub(sed_commands_t *commands, sed_comp_args *compargs, char *rhsbuf)

入力と出力

  • **入力**:
  • commands: sed コマンド全体のコンテキスト構造体。
  • compargs: 正規表現のコンパイル引数（後方参照数 nbra を含む）。
  • rhsbuf: 置換文字列を格納するバッファ。
  • **出力**:
  • 成功時: 解析後のバッファ末尾を示すポインタを返します。
  • 失敗時: NULL を返します。

前提条件と不変条件

  • commands->cp は解析対象のスクリプト文字列を指している必要があります。
  • commands->respace は置換文字列を格納するための十分な容量（RESIZE）を保持している必要があります。

アルゴリズムまたは振る舞い

1. commands->cp から文字を順次読み取り、rhsbuf にコピーします。 2. バックスラッシュ (\) が出現した場合、エスケープシーケンスとして処理します。

  • 後方参照（\1〜\9）の場合、compargs->nbra と比較して範囲内か検証します。

    • 3. commands->sseof（置換コマンドの区切り文字）が出現した時点で解析を終了し、文字列をヌル終端します。 4. 解析中にバッファサイズを超過した場合、または不正な後方参照や終端文字の欠落を検出した場合は、command_errf を呼び出してエラーを報告し、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/sed0.c:548（fcomp 内の s コマンド解析処理）。
  • **呼び出し先**: command_errf（エラー報告用）。

関連データ構造

  • sed_commands_t: sed スクリプト全体の管理構造体。
  • sed_comp_args: コンパイル時の引数管理構造体。

失敗モードと境界条件

  • SEDERR_TMMES: 置換文字列がバッファサイズ（RESIZE）を超過した場合。
  • SEDERR_DOORNG: 後方参照の番号が正規表現で定義された数を超えている場合。

保守担当者向けメモ

  • 本関数は sed の s コマンド専用のパーサーです。区切り文字（sseof）の動的な扱いに依存しているため、変更時は sed の構文仕様との整合性に注意してください。
  • エラー発生時の戻り値は NULL です。呼び出し側で必ずチェックを行ってください。
