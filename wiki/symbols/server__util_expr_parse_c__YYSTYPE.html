YYSTYPE

定義

  • 種別: union
  • 定義箇所: server/util_expr_parse.c:152

目的

Apache HTTP Server の式評価エンジン（ap_expr）における構文解析器（Bison 生成）のスタック要素として、解析中のトークン値や AST ノードを保持するために使用されます。

シグネチャまたは形状

typedef union YYSTYPE {
    char      *cpVal;
    ap_expr_t *exVal;
    int        num;
} YYSTYPE;

入力と出力

  • **入力**: 構文解析器（ap_expr_yyparse）がトークンを還元する際、字句解析器（ap_expr_yylex）から渡される値や、下位の文法規則から生成されたノードが格納されます。
  • **出力**: 最終的に構築された AST ルートノード（exVal）が解析コンテキストに渡されます。

前提条件と不変条件

  • 解析に使用されるメモリは、ap_expr_parse_ctx_t に含まれる apr_pool_t から割り当てられる必要があります。
  • YYSTYPE は Bison のスタック管理メカニズムによって自動的に確保・解放されます。

アルゴリズムまたは振る舞い

  • 構文解析の過程で、トークンの種類に応じて cpVal（文字列）、exVal（AST ノード）、num（整数）のいずれかが共用体として格納されます。
  • ap_expr_make 等のヘルパー関数によって生成された AST ノードは、この共用体を通じて上位の文法規則へ伝播されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_expr_yyparse（解析メインループ）、ap_expr_yylex（字句解析器）、および Bison が生成するデバッグ用出力関数（yy_symbol_print 等）。
  • **呼び出し先**: 直接的な関数呼び出しではなく、パーサーのスタック操作を通じて間接的に参照されます。

関連データ構造

  • ap_expr_t: AST ノードの構造体。exVal を通じて参照されます。
  • ap_expr_parse_ctx_t: 解析コンテキスト。
  • yytokentype: トークンの型定義。

失敗モードと境界条件

  • 共用体であるため、型を誤って読み出すと不正なデータにアクセスする可能性があります。
  • メモリ不足時には apr_pool_t の割り当て失敗により解析が中断されます。

保守担当者向けメモ
