get_cert_count

定義

  • 種別: function
  • 定義箇所: modules/md/md_status.c:180

目的

Managed Domain (MD) に関連付けられた証明書の数を取得します。

シグネチャまたは形状

static int get_cert_count(const md_t *md, int from_staging)

入力と出力

  • **入力**:
  • md: 情報を取得する対象の md_t 構造体へのポインタ。
  • from_staging: ステージング領域の証明書を対象とするかを示すフラグ（非ゼロでステージングを対象）。
  • **出力**:
  • 証明書の総数を示す int 型の値を返します。

前提条件と不変条件

  • md ポインタは有効である必要があります。
  • from_staging が偽（0）の場合、md->cert_files が存在し、かつその要素数が有効である場合にその値を返します。

アルゴリズムまたは振る舞い

1. from_staging が偽であり、かつ md->cert_files が存在し要素を持つ場合、その要素数（nelts）を返します。 2. 上記条件を満たさない場合、md_pkeys_spec_count(md->pks) を呼び出し、秘密鍵の仕様数に基づいてカウントを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_reg.c:1430
  • modules/md/md_status.c:210
  • modules/md/md_status.c:239
  • modules/md/md_status.c:266
  • **呼び出し先**:
  • md_pkeys_spec_count (外部関数)

関連データ構造

  • md_t: 管理対象ドメインの情報を保持する構造体。
  • apr_array_header_t (推測): md->cert_files の型として使用される APR 配列構造。

失敗モードと境界条件

  • md が NULL の場合、プログラムはクラッシュする可能性があります（ソースコード上に明示的な NULL チェックはありません）。
  • md->cert_files が NULL または空の場合、フォールバックとして md_pkeys_spec_count の結果が使用されます。

保守担当者向けメモ

  • 本関数は static であり、modules/md/md_status.c 内でのみ使用されます。
  • md_reg.c からの呼び出し（1430行目）では from_staging 引数が省略されているように見えますが、これは C 言語の仕様上、引数の不一致またはマクロによるラップの可能性があります。詳細な呼び出し規約を確認してください。
