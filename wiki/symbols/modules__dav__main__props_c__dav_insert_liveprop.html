dav_insert_liveprop

定義

  • 種別: function
  • 定義箇所: modules/dav/main/props.c:454

目的

ライブプロパティまたはコアプロパティの値を XML 形式で出力バッファに挿入します。

シグネチャまたは形状

dav_error * dav_insert_liveprop(dav_propdb *propdb,
                                const apr_xml_elem *elem,
                                dav_prop_insert what,
                                apr_text_header *phdr,
                                dav_prop_insert *inserted)

入力と出力

  • propdb: プロパティデータベースのコンテキスト。
  • elem: 挿入対象のプロパティを定義する XML 要素。priv メンバにプロバイダー情報が格納されている必要があります。
  • what: 挿入するプロパティ情報の種類（値のみ、名前のみ等）。
  • phdr: プロパティ値が追加されるテキストヘッダー。
  • inserted (出力): 挿入結果を示すステータス（DAV_PROP_INSERT_NOTDEF 等）。
  • 戻り値: 成功時は NULL、失敗時は dav_error 構造体へのポインタ。

前提条件と不変条件

  • elem->priv が初期化されており、有効なプロバイダー情報またはコアプロパティ識別子が含まれていること。
  • propdb が有効な状態であること。

アルゴリズムまたは振る舞い

1. elem->priv からプロバイダー情報を取得します。 2. プロバイダーが定義されていない場合（priv->provider == NULL）、コアプロパティとして dav_insert_coreprop を呼び出します。 3. プロバイダーが定義されている場合、そのプロバイダーの insert_prop フックを呼び出し、プロパティ値を phdr に挿入します。 4. 挿入結果を inserted に格納して返却します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/main/props.c:794 (確認済み)
  • 呼び出し先:
  • dav_insert_coreprop (プロバイダー未定義時)
  • priv->provider->insert_prop (プロバイダー定義時)

関連データ構造

  • dav_propdb: プロパティ操作のコンテキスト。
  • dav_elem_private: XML 要素に関連付けられたプロパティ情報（プロバイダーや ID を保持）。
  • apr_xml_elem: XML 要素の構造体。
  • apr_text_header: 出力用テキストバッファ。

失敗モードと境界条件

  • プロバイダーの insert_prop フックがエラーを返した場合、そのエラーが伝播されます。
  • elem->priv が正しく設定されていない場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は dav_find_liveprop によって特定されたプロバイダー情報を利用します。
  • プロバイダーの insert_prop フックの実装に依存するため、各プロバイダー側のエラーハンドリングが適切に行われているか確認してください。
