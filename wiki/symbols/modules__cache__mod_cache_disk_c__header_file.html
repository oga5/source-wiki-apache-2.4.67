header_file

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:74

目的

キャッシュ対象の URI に対応する、ディスク上のヘッダーファイルへの絶対パスを生成する。

シグネチャまたは形状

static char *header_file(apr_pool_t *p, disk_cache_conf *conf,
                         disk_cache_object_t *dobj, const char *name)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • conf: ディスクキャッシュの設定情報 (disk_cache_conf)。
  • dobj: キャッシュオブジェクトの状態を保持する disk_cache_object_t。
  • name: キャッシュキーとなる文字列。
  • **出力**:
  • 生成されたヘッダーファイルのフルパス文字列。

前提条件と不変条件

  • dobj->hashfile が未設定の場合、ap_cache_generate_name を用いてハッシュ値が生成され、dobj にキャッシュされる。
  • dobj->prefix が存在する場合はそれを優先し、存在しない場合は conf->cache_root をベースパスとして使用する。
  • ファイル名には CACHE_HEADER_SUFFIX が付与される。

アルゴリズムまたは振る舞い

1. dobj->hashfile が空であれば、conf->dirlevels と conf->dirlength に基づき ap_cache_generate_name でハッシュを生成し格納する。 2. dobj->prefix が設定されている場合、prefix + CACHE_VDIR_SUFFIX + / + hashfile + CACHE_HEADER_SUFFIX を連結して返す。 3. dobj->prefix が設定されていない場合、conf->cache_root + / + hashfile + CACHE_HEADER_SUFFIX を連結して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c:396

  • **呼び出し先**:
  • ap_cache_generate_name (キャッシュキーからのハッシュ生成)
  • apr_pstrcat (パス文字列の構築)

関連データ構造

  • disk_cache_conf: キャッシュのルートディレクトリやディレクトリ構造の設定を保持。
  • disk_cache_object_t: キャッシュ対象のハッシュ値やプレフィックスを保持。

失敗モードと境界条件

  • メモリ不足時には apr_pstrcat 経由でアボートまたはエラーが発生する可能性がある。
  • dobj->hashfile が生成される前と後で、同一の dobj に対して呼び出された場合、一貫したパスが返される。

保守担当者向けメモ

  • 本関数は data_file と対になる設計である。
  • CACHE_HEADER_SUFFIX の定義変更は、既存のキャッシュファイルへのアクセス不能を招くため注意が必要。
