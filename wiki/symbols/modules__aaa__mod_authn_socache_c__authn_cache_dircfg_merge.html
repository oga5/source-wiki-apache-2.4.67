authn_cache_dircfg_merge

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:209

目的

Apache HTTP Server のディレクトリ設定において、親ディレクトリの設定（BASE）と現在のディレクトリ設定（ADD）をマージし、新しい設定構造体を生成する。

シグネチャまたは形状

static void* authn_cache_dircfg_merge(apr_pool_t *pool, void *BASE, void *ADD)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用するプール。
  • BASE: 親ディレクトリの設定構造体（authn_cache_dircfg）。
  • ADD: 現在のディレクトリの設定構造体（authn_cache_dircfg）。
  • **出力**:
  • マージ後の新しい authn_cache_dircfg 構造体へのポインタ。

前提条件と不変条件

  • BASE および ADD は authn_cache_dircfg 型としてキャスト可能であること。
  • apr_pmemdup により、マージ結果は指定された pool の生存期間に依存する。

アルゴリズムまたは振る舞い

1. ADD の内容を pool 上に複製して ret を作成する。 2. ADD の各設定値がデフォルト値である場合、BASE の値を継承する。

  • context が directory（デフォルト）の場合、BASE の値を採用。
  • timeout が 300秒（デフォルト）の場合、BASE の値を採用。
  • providers が NULL の場合、BASE の値を採用。

    • 3. マージされた ret を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_socache.c:469（command_rec 構造体を通じて Apache 設定フレームワークから呼び出される）。
  • **呼び出し先**: apr_pmemdup（APRライブラリ関数）。

関連データ構造

  • authn_cache_dircfg: ディレクトリごとのキャッシュ設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pmemdup がアボートする可能性がある（APRの標準的な挙動）。
  • BASE または ADD が NULL の場合、構造体メンバへのアクセスでセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は Apache の設定マージフェーズで実行される。
  • デフォルト値の判定ロジック（300秒や directory 型）は、設定構造体の初期化ロジックと密接に関連しているため、変更時は整合性に注意すること。
