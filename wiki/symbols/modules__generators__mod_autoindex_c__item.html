item

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_autoindex.c:106

目的

ディレクトリ索引生成時における、ファイルやディレクトリに対するアイコン、代替テキスト、説明文などのパターンマッチング設定を保持するためのデータ構造です。

シグネチャまたは形状

struct item {
    char *type;        /* マッチング対象の型またはパターン */
    char *apply_to;    /* 適用対象のファイル名または拡張子 */
    char *apply_path;  /* 適用先のディレクトリパス */
    char *data;        /* 関連付けられるデータ（アイコンURLや説明文など） */
};

入力と出力

  • 本構造体は、AddIcon、AddAlt、AddDescription などの設定ディレクティブを解析する関数によってインスタンス化されます。
  • data フィールドには、設定されたアイコンのパスや説明文の文字列が格納されます。

前提条件と不変条件

  • 各フィールドは文字列へのポインタであり、request_rec のプールまたは設定用プールによってメモリ管理されます。
  • mod_autoindex の設定構造体（autoindex_config_rec）内でリストとして保持されます。

アルゴリズムまたは振る舞い

  • 索引生成時に、ファイル名やファイルタイプと apply_to / type を照合し、一致した場合に data を出力用データとして利用します。
  • apply_path は、特定のディレクトリ階層に対してのみ設定を適用するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: add_icon、add_alt、add_desc などの設定解析関数が本構造体を生成・登録します。
  • **呼び出し先**: 索引生成のメインループである index_directory およびその関連関数が、ファイルエントリの表示情報を決定するために本構造体を参照します。
  • **注意**: ツールが提示した include/ap_slotmem.h や include/http_core.h 内の「item」という語句は、本構造体とは異なる文脈（メモリ管理やエラーメッセージ等）を指しており、本シンボルとの直接的な関係はありません。

関連データ構造

  • autoindex_config_rec: 本構造体のリストを保持する親の設定構造体。
  • ent: 索引生成時に各ファイル情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には、設定の読み込み段階でエラーが発生する可能性があります。
  • パターンマッチングにおいて、type や apply_to が適切に設定されていない場合、意図したアイコンや説明文が適用されない可能性があります。

保守担当者向けメモ

  • 本構造体は mod_autoindex 専用です。他のモジュール（mod_cgi 等）で見られる「item」という用語と混同しないよう注意してください。
  • data フィールドの文字列は、HTML 出力時に適切にエスケープされる必要があります。
