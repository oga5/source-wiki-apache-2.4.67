groupfile_parse_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_groupfile.c:274

目的

Require ディレクティブで指定された引数（require_line）を解析し、Apache の式評価エンジン（ap_expr）で利用可能な形式に変換します。

シグネチャまたは形状

static const char *groupfile_parse_config(cmd_parms *cmd, const char *require_line,
                                          const void **parsed_require_line)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • require_line: 解析対象となる設定行の文字列。
  • **出力**:
  • parsed_require_line: 解析結果である ap_expr_info_t 構造体へのポインタが格納されます。
  • 戻り値: 成功時は NULL。解析エラーが発生した場合は、エラーメッセージを含む文字列を返します。

前提条件と不変条件

  • require_line は ap_expr_parse_cmd が処理可能な形式である必要があります。
  • cmd->temp_pool が有効である必要があります（エラーメッセージ生成に使用）。

アルゴリズムまたは振る舞い

1. ap_expr_parse_cmd を呼び出し、require_line を AP_EXPR_FLAG_STRING_RESULT フラグ付きで解析します。 2. 解析中にエラー（expr_err）が発生した場合、apr_pstrcat を用いてエラーメッセージを構築し、即座に返します。 3. 解析が成功した場合、生成された ap_expr_info_t 構造体を parsed_require_line に代入し、NULL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_groupfile.c:296（authz_groupfile_cmds テーブル内の登録処理から参照）。
  • **呼び出し先**: ap_expr_parse_cmd（Apache コアの式評価エンジン）。

関連データ構造

  • ap_expr_info_t: 解析された式情報を保持する構造体。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • 式の構文が不正な場合、ap_expr_parse_cmd がエラーを返し、本関数はエラーメッセージを返して処理を中断します。

保守担当者向けメモ

  • 本関数は ap_expr を使用しているため、動的なグループ指定が可能です。
  • エラーメッセージの生成には cmd->temp_pool を使用しており、リクエスト終了時や設定読み込み終了時に自動的に解放されます。
