h2_headers_length

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_headers.c:139

目的

h2_headers 構造体が保持するヘッダーテーブル内の全ヘッダーについて、その合計バイト数を算出します。

シグネチャまたは形状

apr_size_t h2_headers_length(h2_headers *headers)

入力と出力

  • **入力**: headers (h2_headers *) - ヘッダー情報を含む構造体へのポインタ。
  • **出力**: apr_size_t - ヘッダーの合計サイズ（バイト単位）。

前提条件と不変条件

  • headers 引数は有効な h2_headers 構造体を指している必要があります。
  • 内部で apr_table_do を使用してテーブルを走査するため、headers->headers が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 合計サイズを保持する変数 len を 0 で初期化します。 2. apr_table_do を呼び出し、headers->headers テーブル内の各要素に対してコールバック関数 add_header_lengths を適用します。 3. add_header_lengths は各ヘッダーのキーと値の長さを len に加算します。 4. 最終的な len の値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_headers.c:149 (h2_bucket_headers_headers_length 関数内)
  • **呼び出し先**:
  • apr_table_do (APR ライブラリ関数)
  • add_header_lengths (同ファイル内の静的関数)

関連データ構造

  • h2_headers: ヘッダー情報（ステータス、テーブル、ノート）を保持する構造体。

失敗モードと境界条件

  • ヘッダーテーブルが空の場合、合計サイズとして 0 を返します。
  • ヘッダー数が多い場合、テーブルの全走査が発生するためパフォーマンスへの影響が生じる可能性があります。

保守担当者向けメモ

  • 本関数は apr_table_do を介してヘッダーを走査するため、ヘッダー数に比例した計算コストがかかります。頻繁に呼び出される場合は注意してください。
  • h2_headers 構造体のメモリ所有権は生成時のプールに依存するため、生存期間の管理に留意してください。
