create_alias_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_alias.c:77

目的

Apache HTTP Server のサーバー全体の設定構造体（alias_server_conf）を初期化し、エイリアスおよびリダイレクトを保持するための配列を生成する。

シグネチャまたは形状

static void *create_alias_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • s: 設定対象のサーバーレコード（本関数内では未使用）。
  • **出力**:
  • 初期化された alias_server_conf 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内のメモリはゼロクリアされる。
  • aliases および redirects の各配列は、初期容量 20 で確保される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して alias_server_conf 構造体分のメモリを確保する。 2. apr_array_make を呼び出し、alias_entry 型の要素を格納する aliases 配列を初期化する。 3. 同様に redirects 配列を初期化する。 4. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_alias.c:762（サーバー設定の初期化時）。
  • **呼び出し先**: apr_pcalloc, apr_array_make（いずれも APR ライブラリ）。

関連データ構造

  • alias_server_conf: サーバー全体の設定を保持する構造体。
  • alias_entry: エイリアスやリダイレクトの定義を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc または apr_array_make が異常終了（abort）する可能性がある。

保守担当者向けメモ

  • 本関数はサーバーレベルの設定生成用である。ディレクトリ単位の設定については create_alias_dir_config が別途存在する。
  • 配列の初期容量（20）は、一般的な設定数に基づいているが、大規模な設定環境では再割り当てが発生する可能性がある。
