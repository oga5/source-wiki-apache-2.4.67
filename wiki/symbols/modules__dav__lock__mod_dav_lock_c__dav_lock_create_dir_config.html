dav_lock_create_dir_config

定義

  • 種別: function
  • 定義箇所: modules/dav/lock/mod_dav_lock.c:42

目的

ディレクトリごとの設定構造体 dav_lock_dir_conf をメモリ上に割り当て、初期化するために使用されます。

シグネチャまたは形状

static void *dav_lock_create_dir_config(apr_pool_t *p, char *dir)

入力と出力

  • **入力**:
  • p: 設定情報を割り当てるための APR メモリプール。
  • dir: 設定が適用されるディレクトリパス（本関数内では未使用）。
  • **出力**:
  • 割り当てられた dav_lock_dir_conf 構造体へのポインタを void * として返します。

前提条件と不変条件

  • apr_pcalloc を使用しているため、返される構造体はゼロクリアされた状態で初期化されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、dav_lock_dir_conf 構造体のサイズ分のメモリをプール p から確保します。 2. 確保したメモリ領域を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定フレームワークから、ディレクトリ設定の生成時に呼び出されます（modules/dav/lock/mod_dav_lock.c:98 にて参照）。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ関数）。

関連データ構造

  • dav_lock_dir_conf: この関数によって割り当てられる、ロックデータベースのパスを保持するためのディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させるか、プロセスを終了させる可能性があります（APR のメモリ管理規約に依存）。

保守担当者向けメモ

  • 本関数は Apache の設定マージプロセスの一部として機能します。構造体のメンバを追加した場合は、dav_lock_merge_dir_config におけるマージロジックも併せて更新してください。
