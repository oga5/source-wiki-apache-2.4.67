proxy_v1

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_remoteip.c:83

目的

PROXY プロトコル v1 のヘッダー形式を表現するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    char line[108];
} proxy_v1;

入力と出力

  • 入力: PROXY プロトコル v1 の通信ストリームから読み込まれたテキスト行。
  • 出力: 構造体内の line バッファに格納されたプロトコル文字列。

前提条件と不変条件

  • line メンバは 108 バイトの固定長です。
  • PROXY プロトコル v1 の仕様に基づき、接続情報を含むテキスト行を保持します。

アルゴリズムまたは振る舞い

  • remoteip_input_filter において、受信データのサイズを決定する際に sizeof(proxy_v1) が使用されます。
  • ストリームから読み込まれたデータがこの構造体にキャストされ、プロトコルヘッダーとして解析されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/metadata/mod_remoteip.c 内の remoteip_input_filter 関数（1137行目）にて、受信バッファの必要サイズ計算に使用。
  • 呼び出し先: 情報なし。

関連データ構造

  • proxy_header: PROXY プロトコル v1/v2 を包含する共用体。
  • remoteip_filter_context: フィルタの状態管理に使用される構造体。

失敗モードと境界条件

  • line のサイズ（108バイト）を超えるデータが入力された場合、プロトコル解析の不整合が発生する可能性があります。
  • フィルタ処理において、データが分割されて届く場合は remoteip_filter_context を介して状態が保持されます。

保守担当者向けメモ

  • 本構造体は PROXY プロトコル v1 の固定長ヘッダーを定義しています。仕様変更や拡張が必要な場合は、proxy_header 共用体との整合性に注意してください。
