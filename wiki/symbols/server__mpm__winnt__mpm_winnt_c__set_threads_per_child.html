set_threads_per_child

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:109

目的

Apache HTTP Serverの設定ディレクティブ ThreadsPerChild の値を解析し、グローバル変数 ap_threads_per_child に設定する。

シグネチャまたは形状

static const char *set_threads_per_child (cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていない引数。
  • arg: 設定ファイルから読み込まれた ThreadsPerChild の値（文字列）。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ（文字列）を返す。

前提条件と不変条件

  • 本関数は、設定コンテキストがグローバルスコープである必要がある（ap_check_cmd_context による検証）。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、設定がグローバルスコープで行われているか確認する。エラーがあればそのメッセージを返す。 2. atoi(arg) を使用して、入力された文字列を整数に変換する。 3. 変換結果をグローバル変数 ap_threads_per_child に代入する。 4. NULL を返し、正常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/mpm_winnt.c:132 (AP_INIT_TAKE1 マクロ経由)
  • server/mpm/event/event.c:4144 (AP_INIT_TAKE1 マクロ経由)
  • server/mpm/worker/worker.c:2452 (AP_INIT_TAKE1 マクロ経由)
  • **呼び出し先**:
  • ap_check_cmd_context (Apache API)
  • atoi (標準Cライブラリ)

関連データ構造

  • cmd_parms: 設定コマンドの実行時パラメータを保持する構造体。

失敗モードと境界条件

  • ap_check_cmd_context がグローバルスコープ以外での設定を検出した場合、エラーメッセージを返す。
  • arg が数値として解釈できない場合、atoi の仕様に従い 0 が設定される。

保守担当者向けメモ

  • 本関数は ThreadsPerChild ディレクティブのハンドラとして機能する。
  • server/mpm/winnt 以外のMPM（event, worker）からも参照されているが、これは各MPMが共通のディレクティブ名を使用しているためである。
