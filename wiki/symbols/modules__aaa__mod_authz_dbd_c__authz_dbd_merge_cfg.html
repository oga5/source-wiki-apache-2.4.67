authz_dbd_merge_cfg

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbd.c:65

目的

親ディレクトリの設定（BASE）と、現在のディレクトリで追加・オーバーライドされた設定（ADD）をマージし、新しい設定構造体を作成する。

シグネチャまたは形状

static void *authz_dbd_merge_cfg(apr_pool_t *pool, void *BASE, void *ADD)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • BASE: 親ディレクトリの設定構造体（authz_dbd_cfg *）。
  • ADD: 現在のディレクトリの設定構造体（authz_dbd_cfg *）。
  • **出力**:
  • マージされた新しい authz_dbd_cfg 構造体へのポインタ。

前提条件と不変条件

  • BASE および ADD は authz_dbd_cfg 型としてキャスト可能であること。
  • pool は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、新しい authz_dbd_cfg 構造体 ret をメモリプール上に確保する。 2. 各フィールドについて以下のロジックでマージを行う：

  • query: ADD で設定されていれば ADD を、そうでなければ BASE を採用する。
  • redir_query: ADD で設定されていれば ADD を、そうでなければ BASE を採用する。
  • redirect: ADD が -1（未設定）であれば BASE を、そうでなければ ADD を採用する。

    • 3. マージ済みの ret を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbd.c:404（Apache の設定マージ機構から呼び出される）。
  • **呼び出し先**: 直接的な関数呼び出しはなし（メモリ割り当て関数 apr_palloc を使用）。

関連データ構造

  • authz_dbd_cfg: 本モジュールが保持するディレクトリ単位の設定構造体。

失敗モードと境界条件

  • pool のメモリ不足時には apr_palloc が例外を発生させる可能性がある。
  • BASE または ADD が NULL の場合の挙動は、本関数内では明示的に保護されていない。

保守担当者向けメモ

  • 本関数は Apache の設定マージフェーズで実行される。
  • redirect フィールドのデフォルト値が -1 であるため、マージ時にこの値を確認して継承の可否を判断している点に注意すること。
