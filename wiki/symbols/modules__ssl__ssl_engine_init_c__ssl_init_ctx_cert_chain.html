ssl_init_ctx_cert_chain

定義

  • **種別**: 関数
  • **定義箇所**: modules/ssl/ssl_engine_init.c:1325

目的

サーバー証明書チェーンを明示的に設定します。クライアント認証用のCA設定と競合することなく、サーバー証明書チェーンのみに使用するCA証明書を構成するために使用されます。

シグネチャまたは形状

static apr_status_t ssl_init_ctx_cert_chain(server_rec *s,
                                            apr_pool_t *p,
                                            apr_pool_t *ptemp,
                                            modssl_ctx_t *mctx)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p, ptemp: メモリプール。
  • mctx: SSLコンテキスト設定構造体（mctx->cert_chain を参照）。
  • **出力**:
  • APR_SUCCESS: 正常終了。
  • ssl_die(s) の戻り値: 設定失敗時（プロセス終了）。

前提条件と不変条件

  • mctx->cert_chain が設定されている必要があります。未設定の場合は何もせず APR_SUCCESS を返します。
  • mctx->ssl_ctx が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. mctx->cert_chain が存在しない場合は直ちに終了します。 2. mctx->pks->cert_files 内のファイルリストを走査し、cert_chain と一致するファイル名が存在するか確認します。一致する場合は skip_first フラグを立てます。 3. use_certificate_chain を呼び出し、指定されたチェーンファイルを mctx->ssl_ctx に適用します。 4. 適用に失敗した場合（戻り値が負）、エラーログを出力し ssl_die(s) を呼び出してサーバーを停止します。 5. 成功した場合、設定されたCA証明書の数をデバッグログに出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:1404 （ssl_init_ctx 内から呼び出されることが確認されています）。
  • **呼び出し先**:
  • use_certificate_chain（OpenSSL APIラッパー）
  • ap_log_error
  • ssl_die

関連データ構造

  • modssl_ctx_t: SSL設定を保持する構造体。
  • server_rec: Apache サーバーレコード。

失敗モードと境界条件

  • **失敗**: use_certificate_chain が負の値を返した場合、ssl_die(s) によりサーバー起動が中断されます。
  • **境界条件**: mctx->cert_chain が NULL の場合は処理をスキップします。

保守担当者向けメモ

  • 本関数は、クライアント認証用CA設定とサーバー証明書チェーン用CA設定を分離するために存在します。
  • use_certificate_chain の第3引数 skip_first は、証明書ファイル自体がチェーンに含まれている場合に重複を避けるための制御フラグです。
