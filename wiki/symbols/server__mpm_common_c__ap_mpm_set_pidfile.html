ap_mpm_set_pidfile

定義

  • 種別: function
  • 定義箇所: server/mpm_common.c:300

目的

Apache HTTP Server の設定ディレクティブ PidFile を処理し、PID ファイルのパスをグローバル変数 ap_pid_fname に設定する。

シグネチャまたは形状

const char *ap_mpm_set_pidfile(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのコンテキスト情報。
  • dummy: 使用されない引数。
  • arg: 設定ファイルから読み込まれた PID ファイルのパス文字列。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時（コンテキストエラーや仮想ホスト内での指定）は、エラーメッセージを示す文字列を返す。

前提条件と不変条件

  • 本関数は GLOBAL_ONLY コンテキストで呼び出される必要がある。
  • 仮想ホスト（<VirtualHost>）内での設定は許可されない。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、現在の設定コンテキストがグローバルスコープであることを検証する。エラーがあればそのメッセージを返す。 2. cmd->server->is_virtual を確認し、仮想ホスト内であればエラーメッセージを返す。 3. 上記の検証を通過した場合、引数 arg をグローバル変数 ap_pid_fname に代入する。 4. NULL を返し、正常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c において、AP_INIT_TAKE1 マクロを通じて PidFile ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: ap_check_cmd_context（Apache 設定 API）。

関連データ構造

  • cmd_parms: 設定コマンドのパラメータを保持する構造体。
  • server_rec: サーバー設定を保持する構造体（cmd->server 経由で参照）。
  • ap_pid_fname: PID ファイルのパスを保持するグローバル変数。

失敗モードと境界条件

  • **コンテキスト違反**: グローバルスコープ以外で設定された場合、ap_check_cmd_context がエラーを返す。
  • **仮想ホスト内での指定**: <VirtualHost> ブロック内で指定された場合、明示的にエラーメッセージを返す。

保守担当者向けメモ

  • 本関数は server/mpm_common.c で定義されており、MPM 共通の設定管理の一部を担う。
  • ap_pid_fname はグローバル変数であるため、設定の競合や上書きには注意が必要だが、本関数は GLOBAL_ONLY 制約によりそのリスクを最小化している。
