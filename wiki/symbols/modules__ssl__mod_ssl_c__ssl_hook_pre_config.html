ssl_hook_pre_config

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:391

目的

Apache HTTP Server の起動フェーズにおいて、OpenSSL ライブラリの初期化、グローバルな設定の登録、およびモジュール終了時のクリーンアップ処理を登録する。

シグネチャまたは形状

static int ssl_hook_pre_config(apr_pool_t *pconf,
                               apr_pool_t *plog,
                               apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • plog: ログ用メモリプール。
  • ptemp: 一時的な割り当て用メモリプール。
  • **出力**:
  • int: 成功時は OK を返す。

前提条件と不変条件

  • modssl_running_statically 変数が、モジュールが静的にリンクされているかどうかに基づいて設定される。
  • OpenSSL のバージョンに応じて、適切な初期化関数（OPENSSL_init_ssl またはレガシーな SSL_library_init 等）が呼び出される。

アルゴリズムまたは振る舞い

1. **静的リンク判定**: modssl_is_prelinked() を呼び出し、静的リンク状態を記録する。 2. **OpenSSL 初期化**:

  • OpenSSL 1.1.0 以降: OPENSSL_init_ssl を使用してエンジンサポートを有効化。
  • それ以前のバージョン: CRYPTO_malloc_init、エラー文字列のロード、アルゴリズムの追加、組み込みモジュールのロードを実行。

    • 3. **オブジェクト定義**: id-on-dnsSRV (1.3.6.1.5.5.7.8.7) が未定義の場合、OBJ_create で登録する。 4. **エラー状態のクリア**: ERR_clear_error() を呼び出し、初期化中の副作用を消去する。 5. **クリーンアップ登録**: ssl_cleanup_pre_config を pconf プールのクリーンアップハンドラとして登録。 6. **機能登録**:

  • ログ変数 (ssl_var_log_config_register)。
  • ステータスページ生成 (ssl_scache_status_register)。
  • ミューテックス (ap_mutex_register) を登録。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定フェーズにおいて ap_hook_pre_config を介して登録され、サーバー起動時に呼び出される。
  • **呼び出し先**: modssl_is_prelinked, OPENSSL_init_ssl (またはレガシー関数群), OBJ_txt2nid, OBJ_create, ERR_clear_error, apr_pool_cleanup_register, ssl_cleanup_pre_config, ssl_var_log_config_register, ssl_scache_status_register, ap_mutex_register。

関連データ構造

  • apr_pool_t: Apache のメモリ管理プール。
  • SSL_CACHE_MUTEX_TYPE, SSL_STAPLING_CACHE_MUTEX_TYPE, SSL_STAPLING_REFRESH_MUTEX_TYPE: ミューテックス識別子。

失敗モードと境界条件

  • OpenSSL のバージョン判定ロジックにより、実行される初期化パスが異なる。
  • OBJ_txt2nid が NID_undef を返す場合のみ OBJ_create が実行される。
  • HAVE_OCSP_STAPLING が定義されている場合のみ、OCSP 関連のミューテックスが登録される。

保守担当者向けメモ

  • 本関数は Apache の pre_config フックとして動作するため、サーバーの初期化順序に依存する。
  • OpenSSL 1.1.0 以降とそれ以前のバージョンで初期化ロジックが大きく分岐しているため、修正時は #if マクロの条件を確認すること。
