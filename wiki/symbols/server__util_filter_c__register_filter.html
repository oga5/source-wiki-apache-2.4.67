register_filter

定義

  • 種別: function
  • 定義箇所: server/util_filter.c:203

目的

フィルタ名をトライ木構造に登録し、対応する ap_filter_rec_t 構造体を生成または取得して、フィルタの実行関数や初期化関数をシステムに紐付ける。

シグネチャまたは形状

static ap_filter_rec_t *register_filter(const char *name,
                                        ap_filter_func filter_func,
                                        ap_init_filter_func filter_init,
                                        ap_filter_type ftype,
                                        filter_trie_node **reg_filter_set)

入力と出力

  • **入力**:
  • name: フィルタ名（文字列）。
  • filter_func: フィルタの処理関数。
  • filter_init: フィルタの初期化関数。
  • ftype: フィルタの型（ap_filter_type）。
  • reg_filter_set: フィルタ名を保持するトライ木のルートノードへのポインタ。
  • **出力**:
  • 登録または取得された ap_filter_rec_t 構造体へのポインタ。

前提条件と不変条件

  • FILTER_POOL がグローバルなメモリ管理に使用可能であること。
  • reg_filter_set が指すトライ木は、フィルタ名の一意性を保証するために使用される。
  • フィルタ名は内部で小文字に正規化される。

アルゴリズムまたは振る舞い

1. reg_filter_set が空の場合、trie_node_alloc で初期ノードを生成する。 2. name を小文字に変換し、トライ木を走査する。各文字に対してノードを生成し、アルファベットの場合は大文字・小文字を区別しない検索を可能にするため trie_node_link でリンクを張る。 3. 該当するノードに frec（ap_filter_rec_t）が存在すればそれを再利用し、なければ apr_pcalloc で新規確保する。 4. frec に filter_func、filter_init_func、ftype を設定する。 5. apr_pool_cleanup_register を呼び出し、filter_cleanup をクリーンアップハンドラとして登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_filter.c:253 (ap_register_output_filter からの呼び出し)
  • server/util_filter.c:276 (ap_register_input_filter からの呼び出し)
  • **呼び出し先**:
  • trie_node_alloc
  • ap_str_tolower
  • apr_isalpha / apr_toupper
  • trie_node_link
  • apr_pcalloc
  • apr_pool_cleanup_register
  • filter_cleanup

関連データ構造

  • ap_filter_rec_t: フィルタの定義を保持する構造体。
  • filter_trie_node: フィルタ名を高速検索するためのトライ木ノード。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数が例外を発生させる可能性がある。
  • 既に登録済みのフィルタ名が渡された場合、既存の ap_filter_rec_t が上書きされる（関数ポインタと型が更新される）。

保守担当者向けメモ

  • 本関数は FILTER_POOL を使用してメモリを割り当てるため、フィルタのライフサイクルはサーバーの生存期間に依存する。
  • トライ木による名前解決は、フィルタの登録数が増加しても一定の検索性能を維持するように設計されている。
