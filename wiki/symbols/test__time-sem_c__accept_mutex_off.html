accept_mutex_off

定義

  • 種別: function
  • 定義箇所: test/time-sem.c:124

目的

fcntl を使用したプロセス間排他制御（ロック）を解放する。

シグネチャまたは形状

void accept_mutex_off(void)

入力と出力

  • 入力: なし（グローバル変数 fcntl_fd および unlock_it を使用）
  • 出力: なし（成功時）

前提条件と不変条件

  • fcntl_fd が有効なファイル記述子として初期化されていること。
  • unlock_it が fcntl の F_SETLKW 操作に適した struct flock 構造体として設定されていること。

アルゴリズムまたは振る舞い

1. fcntl システムコールを F_SETLKW（ロック設定・待機）コマンドで呼び出し、unlock_it 構造体に基づいてロックを解除する。 2. fcntl が負の値を返した場合（エラー発生時）、perror でエラーメッセージを出力し、exit(1) を呼び出してプロセスを終了する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • test/time-sem.c 内の計測ループ（accept_mutex_off() の直接呼び出し）。
  • server/mpm/prefork/prefork.c 内の SAFE_ACCEPT マクロ経由。
  • 呼び出し先:
  • fcntl (システムコール)
  • perror (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • fcntl_fd: ロック対象のファイル記述子。
  • unlock_it: struct flock 型のロック解除用設定構造体。

失敗モードと境界条件

  • fcntl がエラーを返した場合、即座にプログラムが終了する。これはベンチマークツールとしての設計によるものであり、エラーからの復旧は行わない。

保守担当者向けメモ

  • 本関数は fcntl を用いたロック方式の実装である。test/time-sem.c 内には、コンパイルオプションによって異なるロック方式（SysV sem, pthread mutex等）に対応した同名の関数が複数定義されている可能性があるため、ビルド時の構成を確認すること。
  • server/mpm/prefork/prefork.c からの呼び出しは、SAFE_ACCEPT マクロを通じて行われる。
