dav_lock_indirect

定義

  • 種別: struct
  • 定義箇所: modules/dav/lock/locks.c:134

目的

リソースに対する間接ロック（継承されたロック）の情報を保持し、ロックの追跡と解決を行うためのリスト構造体です。

シグネチャまたは形状

typedef struct dav_lock_indirect
{
    dav_locktoken *locktoken;
    apr_datum_t key;
    struct dav_lock_indirect *next;
    time_t timeout;
} dav_lock_indirect;

入力と出力

  • 本構造体はメモリ上のリストとして管理されます。
  • dav_generic_load_lock_record 関数により、DBM から読み込まれたバイナリデータから本構造体が構築されます。
  • dav_generic_save_lock_record 関数により、本構造体の内容が DBM へ書き込むためのバイナリ形式にシリアライズされます。

前提条件と不変条件

  • next ポインタにより、単方向リストを形成します。
  • timeout は time_t 型であり、期限切れの判定に使用されます。
  • すべてのインスタンスは、リクエスト処理に関連付けられたメモリプールから割り当てられます。

アルゴリズムまたは振る舞い

  • 間接ロックの解決時、dav_generic_resolve 関数がこの構造体を参照し、親リソースのロックレコードとトークンの一致を確認します。
  • ロックの読み込み処理において、timeout が現在時刻（time(NULL)）と比較され、期限切れの場合はリストから除外されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/fs/lock.c 内の複数の関数（416, 524, 665, 668 行目など）で、間接ロックの操作やリストの走査のために参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • dav_locktoken: ロックを一意に識別するために使用されます。
  • apr_datum_t: DBM 内のキー情報を保持するために使用されます。
  • dav_lock_discovery: 直接ロックを保持する対となる構造体です。

失敗モードと境界条件

  • 構造体のメモリ割り当て失敗時は、リクエスト処理が中断される可能性があります。
  • DBM からの読み込み時にデータが破損している場合、シリアライズされたバイナリ列のオフセット計算が不整合を起こす可能性があります。

保守担当者向けメモ

  • 本構造体は locks.c 内のバイナリシリアライズロジックと密結合しています。構造体のメンバを変更する場合は、dav_generic_save_lock_record および dav_generic_load_lock_record 内のオフセット計算を必ず更新してください。
  • 将来的な拡張の際は、バイナリ形式の後方互換性に注意が必要です。
