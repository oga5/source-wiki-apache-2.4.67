ap_add_file_conf

定義

  • 種別: function
  • 定義箇所: server/core.c:632

目的

core_dir_config 構造体内に保持されている <Files> や <FilesMatch> セクションの設定リストへ、新しい設定ベクトルを追加します。

シグネチャまたは形状

void ap_add_file_conf(apr_pool_t *p, core_dir_config *conf, void *url_config)

入力と出力

  • p: 設定の割り当てに使用する apr_pool_t。
  • conf: 設定を追加する対象の core_dir_config 構造体。
  • url_config: 追加する設定ベクトル（ap_conf_vector_t *）。
  • 出力: なし（conf->sec_file 配列を更新）。

前提条件と不変条件

  • conf が指す core_dir_config 構造体は有効である必要があります。
  • conf->sec_file が未初期化の場合、この関数内で apr_array_make を用いて初期化されます。

アルゴリズムまたは振る舞い

1. conf->sec_file が NULL の場合、apr_array_make を使用して初期サイズ 2 で配列を生成します。 2. apr_array_push を使用して配列の末尾に新しい要素用の領域を確保します。 3. 確保した領域に url_config を格納します。

呼び出し元と呼び出し先

  • 呼び出し元: server/core.c:2700 にて、設定解析中に呼び出されます。
  • 呼び出し先: apr_array_make および apr_array_push（APR ライブラリ関数）。

関連データ構造

  • core_dir_config: ディレクトリ単位の設定を保持する構造体。
  • ap_conf_vector_t: 設定ベクトルの型。

失敗モードと境界条件

  • メモリ不足時: apr_array_make または apr_array_push が内部的にメモリ割り当てに失敗した場合、APR の規約に従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は include/http_core.h で宣言されています。
  • ap_add_per_dir_conf や ap_add_per_url_conf と同様のパターンで実装されており、設定の階層構造を構築するために使用されます。
