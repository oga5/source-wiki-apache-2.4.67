md_acme_authz_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme_authz.h:45

目的

ACME プロトコルにおけるドメイン認証（Authorization）リソースの状態、有効期限、およびエラー詳細を保持・管理するための構造体です。

シグネチャまたは形状

typedef struct md_acme_authz_t md_acme_authz_t;

※不完全型として定義されており、詳細は実装ファイル modules/md/md_acme_authz.c に依存します。

入力と出力

  • 入力: ACME サーバーからの JSON レスポンス（md_acme_authz_update を介して反映）。
  • 出力: 認証状態（md_acme_authz_state_t）、チャレンジタイプ、および検証結果。

前提条件と不変条件

  • 認証状態が MD_ACME_AUTHZ_S_VALID に遷移するまで、証明書発行オーダーは進行できません。
  • サーバーから取得した生の JSON データは resource フィールドに保持されます。

アルゴリズムまたは振る舞い

1. **取得**: md_acme_authz_retrieve() により指定 URL からロードされます。 2. **更新**: md_acme_authz_update() を通じてサーバー上の最新状態と同期されます。 3. **応答**: md_acme_authz_respond() によりチャレンジへの応答が生成されます。 4. **終了**: md_acme_authz_teardown() により一時リソースが解放されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • md_acme_authz_create
  • md_acme_authz_update
  • setup_key_authz
  • cha_http_01_setup
  • cha_tls_alpn_01_setup
  • cha_dns_01_setup
  • 呼び出し先: 情報なし。

関連データ構造

  • md_acme_authz_state_t: 認証ステータス（Pending, Valid, Invalid 等）を定義。
  • md_acme_challenge_t: チャレンジの詳細情報を保持。

失敗モードと境界条件

  • 認証状態が Invalid になった場合、証明書発行プロセスは停止します。
  • サーバーとの通信エラーや JSON 解析失敗時は md_acme_authz_update がエラーを返します。

保守担当者向けメモ

  • 本構造体は状態管理に特化しています。実際のチャレンジ応答ロジックは md_acme_authz.c 内の実装を参照してください。
  • 構造体内の resource フィールドはデバッグ時に詳細な属性確認が可能です。
