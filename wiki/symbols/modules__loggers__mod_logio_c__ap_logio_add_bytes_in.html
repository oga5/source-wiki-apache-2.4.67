ap_logio_add_bytes_in

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_logio.c:77

目的

接続ごとの受信バイト数（bytes_in）を累積加算するために使用される関数です。

シグネチャまたは形状

void ap_logio_add_bytes_in(conn_rec *c, apr_off_t bytes)

入力と出力

  • **入力**:
  • c: 接続情報を保持する conn_rec 構造体へのポインタ。
  • bytes: 加算するバイト数（apr_off_t 型）。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • c->conn_config に logio_module に対応する logio_config_t 構造体が正しく設定されている必要があります。
  • logio_config_t は接続の生存期間中、接続プール (c->pool) に保持されます。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、指定された接続 c から logio_module の設定構造体 logio_config_t を取得します。 2. 取得した構造体内の bytes_in メンバに対し、引数 bytes を加算します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c: HTTP/2 接続処理において、受信バイト数を追跡するためにオプション関数として呼び出されます。
  • その他、APR_RETRIEVE_OPTIONAL_FN を介して外部モジュールから呼び出される可能性があります。
  • **呼び出し先**:
  • ap_get_module_config（Apache Core API）

関連データ構造

  • logio_config_t: 接続ごとの統計情報を保持する構造体。
  • conn_rec: Apache の接続状態を管理するコア構造体。

失敗モードと境界条件

  • c が NULL の場合、または c->conn_config に logio_module の設定が割り当てられていない場合、ap_get_module_config が NULL を返し、その後のメンバアクセスでセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN を通じて公開されており、他のモジュールから動的に利用可能です。
  • mod_logio の内部フィルタである logio_in_filter からも呼び出されることが想定されています。
