ap_pcre_version_string

定義

  • 種別: function
  • 定義箇所: server/util_pcre.c:93

目的

実行時にリンクされている PCRE ライブラリのバージョン情報を取得する。コンパイル時（ビルド時）のバージョンと、ロード時（実行時）のバージョンの両方を取得可能。

シグネチャまたは形状

const char *ap_pcre_version_string(int which)

入力と出力

  • **入力**:
  • which: 取得する情報の種類を指定する整数値。
  • AP_REG_PCRE_COMPILED: ビルド時に使用された PCRE のバージョン。
  • AP_REG_PCRE_LOADED: 実行時にロードされている PCRE のバージョン。
  • **出力**:
  • バージョン情報を含む文字列（const char *）。
  • 指定された which が不正な場合は "Unknown" を返す。

前提条件と不変条件

  • PCRE2 環境下では、AP_REG_PCRE_LOADED の結果を格納するために静的バッファ buf[80] が使用される。
  • PCRE1 環境下では、pcre_version() が直接呼び出される。

アルゴリズムまたは振る舞い

1. which 引数に基づき分岐する。 2. AP_REG_PCRE_COMPILED の場合、マクロ PCREn を用いてビルド時のメジャー/マイナーバージョンおよび日付を文字列として連結して返す。 3. AP_REG_PCRE_LOADED の場合:

  • PCRE2 環境では pcre2_config を呼び出し、静的バッファ buf にバージョン情報を書き込んでそのポインタを返す。
  • PCRE1 環境では pcre_version() の戻り値をそのまま返す。

    • 4. 上記以外の場合は "Unknown" を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_info.c
  • server/main.c
  • **呼び出し先**:
  • pcre2_config (PCRE2 環境時)
  • pcre_version (PCRE1 環境時)

関連データ構造

  • AP_REG_PCRE_COMPILED / AP_REG_PCRE_LOADED: include/ap_regex.h で定義された引数用定数。

失敗モードと境界条件

  • which に定義外の値を渡した場合、"Unknown" が返される。
  • PCRE2 環境において、バージョン文字列が 80 バイトを超える場合、静的バッファ buf の境界を超える可能性がある（ソースコード上の静的確保サイズに基づく）。

保守担当者向けメモ

  • 本関数は PCRE1 と PCRE2 の差異を吸収するラッパーである。PCRE2 への移行に伴い、match_data 等の管理が厳密化されているため、バージョン確認が必要な場面で利用すること。
