show_time

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_status.c:129

目的

秒単位の稼働時間（tsecs）を、「日・時間・分・秒」の人間が読みやすい形式に変換し、リクエストのレスポンスストリームへ出力します。

シグネチャまたは形状

static void show_time(request_rec *r, apr_uint32_t tsecs)

入力と出力

  • **入力**:
  • r: 現在のリクエストを処理する request_rec 構造体へのポインタ。
  • tsecs: 変換対象となる秒数（apr_uint32_t）。
  • **出力**: なし（戻り値は void）。副作用として ap_rprintf を介してリクエストの出力バッファに文字列を書き込みます。

前提条件と不変条件

  • r は有効なリクエスト構造体である必要があります。
  • tsecs は経過秒数として正の値を想定しています。

アルゴリズムまたは振る舞い

1. tsecs を基に、日・時間・分・秒を計算します。

  • 秒: tsecs % 60
  • 分: (tsecs / 60) % 60
  • 時間: (tsecs / 3600) % 24
  • 日: tsecs / 86400

    • 2. 各単位が 0 より大きい場合のみ、ap_rprintf を使用して「N unit(s)」形式の文字列を出力します。 3. 単位が 1 の場合は単数形、それ以外は複数形（末尾に 's' を付加）を選択します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_status.c:449
  • modules/generators/mod_status.c:477
  • **呼び出し先**:
  • ap_rprintf（Apache HTTP Server の出力関数）

関連データ構造

  • request_rec: Apache のリクエスト処理用構造体。

失敗モードと境界条件

  • tsecs が 0 の場合、何も出力されません。
  • 単位が 1 の場合に単数形（例: "1 day"）を正しく選択するロジックが含まれています。

保守担当者向けメモ

  • 本関数は mod_status のレポート生成において、サーバーの稼働時間（uptime）を表示するために使用されます。
  • 出力形式を変更する場合は、複数形判定のロジックと整合性を取る必要があります。
