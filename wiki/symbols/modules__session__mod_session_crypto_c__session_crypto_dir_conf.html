session_crypto_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/session/mod_session_crypto.c:45

目的

ディレクトリ単位でのセッション暗号化設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *passphrases;
    int passphrases_set;
    const char *cipher;
    int cipher_set;
} session_crypto_dir_conf;

入力と出力

本構造体は設定値のコンテナであり、直接的な入力・出力関数ではありません。設定値は Apache の設定ファイルから読み込まれ、本構造体に格納されます。

前提条件と不変条件

  • passphrases は、セッションの復号時に試行されるパスフレーズのリストを保持します。
  • cipher は、使用する暗号アルゴリズムを指定します。
  • passphrases_set および cipher_set は、各設定が明示的に設定されたかどうかを示すフラグとして機能します。

アルゴリズムまたは振る舞い

  • 本構造体は、mod_session_crypto におけるディレクトリごとの暗号化パラメータを管理します。
  • 復号処理（decrypt_string）において、passphrases リスト内のパスフレーズを順次試行することで、キーローテーションをサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_session_crypto_config（設定の初期化時）
  • session_crypto_encode / session_crypto_decode（暗号化・復号化の実行時）
  • 設定マージ関数（modules/session/mod_session_crypto.c:646-647）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • session_crypto_conf: サーバー全体の設定を保持する構造体。
  • apr_array_header_t: パスフレーズのリストを格納するために使用。

失敗モードと境界条件

  • passphrases が空の場合、暗号化または復号化処理は失敗します。
  • 設定が未定義（_set フラグが 0）の場合、デフォルト値または親コンテキストの設定が適用されます。

保守担当者向けメモ

  • 本構造体は apr_pcalloc を使用してメモリ確保されるため、リクエスト終了時に自動的に解放されます。
  • 複数のパスフレーズを保持できるため、キーローテーションを行う際は、古いパスフレーズをリストの末尾に追加する運用が推奨されます。
