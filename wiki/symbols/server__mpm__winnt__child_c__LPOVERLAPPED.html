LPOVERLAPPED

定義

  • 種別: typedef
  • 定義箇所: server/mpm/winnt/child.c:52

目的

Windows API の非同期 I/O 操作において、操作の状態を保持する OVERLAPPED 構造体へのポインタ型として使用されます。

シグネチャまたは形状

typedef struct _OVERLAPPED *LPOVERLAPPED;

※Windows SDK の標準定義に準拠。

入力と出力

  • **入力**: 非同期 I/O 関数（AcceptEx 等）へ渡すことで、操作の完了通知や結果の追跡に使用されます。
  • **出力**: I/O 操作完了後、GetQueuedCompletionStatus 等を通じて、完了した操作に関連付けられた OVERLAPPED 構造体のアドレスが返されます。

前提条件と不変条件

  • OVERLAPPED 構造体は、対応する非同期 I/O 操作が完了するまでメモリ上で有効である必要があります。
  • I/O 完了ポート（IOCP）を使用する場合、この構造体は winnt_conn_ctx_t などのコンテキスト構造体の一部として埋め込まれ、完了時にコンテキストを特定するために利用されます。

アルゴリズムまたは振る舞い

  • 非同期 I/O の開始時に、このポインタを介してシステムへ状態を渡します。
  • 操作が即座に完了しない場合、システムは操作完了時にこの構造体のアドレスを I/O 完了ポートのキューへ投入します。
  • ワーカー・スレッドは GetQueuedCompletionStatus を呼び出し、このポインタを介してどの接続に対する操作が完了したかを識別します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/child.c および server/mpm/winnt/mpm_winnt.c 内の各関数。
  • **呼び出し先**: Windows API のソケット拡張関数（AcceptEx 等）および I/O 完了ポート関連関数。
  • **補足**: mpm_winnt.c の各箇所（285, 296, 304, 319, 470, 1091行目）で (LPOVERLAPPED) NULL としてキャスト利用されている事例が確認されています。

関連データ構造

  • winnt_conn_ctx_t: 本モジュールにおいて OVERLAPPED 構造体をメンバとして保持する主要なデータ構造。
  • OVERLAPPED (Windows SDK): 本型が指し示す実体。

失敗モードと境界条件

  • **メモリ解放**: 非同期操作が保留中に構造体を解放すると、システムによる書き込み時にメモリ破壊が発生します。
  • **NULL ポインタ**: ツールによる参照箇所で確認される通り、非同期処理を必要としない同期的な呼び出し時には NULL が渡されることがあります。

保守担当者向けメモ

  • 本シンボルは Windows SDK の標準型へのエイリアスです。
  • winnt_conn_ctx_t 内の OVERLAPPED メンバと混同しないよう注意してください。
  • mpm_winnt.c での NULL キャスト利用は、非同期 I/O を行わない（または完了を待機しない）コードパスであることを示唆しています。
