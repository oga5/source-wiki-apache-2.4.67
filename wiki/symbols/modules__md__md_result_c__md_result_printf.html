md_result_printf

定義

  • 種別: function
  • 定義箇所: modules/md/md_result.c:126

目的

md_result_t オブジェクトに対して、ステータスコードとフォーマット指定された詳細メッセージを設定し、状態変更イベントを通知します。

シグネチャまたは形状

void md_result_printf(md_result_t *result, apr_status_t status, const char *fmt, ...)

入力と出力

  • result: 更新対象の md_result_t 構造体へのポインタ。
  • status: 設定する apr_status_t 型のステータスコード。
  • fmt: 詳細メッセージのフォーマット文字列（printf 形式）。
  • ...: フォーマット文字列に対応する可変長引数。
  • 出力: なし（result 構造体のフィールドを直接更新）。

前提条件と不変条件

  • result は有効な md_result_t オブジェクトを指している必要があります。
  • result が保持する apr_pool_t は、メッセージ文字列の割り当てに使用されます。
  • 呼び出し後、result->subproblems は NULL にリセットされます。

アルゴリズムまたは振る舞い

1. result->status に指定された status を代入します。 2. apr_pvsprintf を使用し、result が保持するメモリプールからフォーマット済みの文字列を生成して result->detail に格納します。 3. result->subproblems を NULL に初期化します。 4. on_change(result) を呼び出し、登録されている状態変更コールバックを実行します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/md/md_acme.c
  • modules/md/md_acme_authz.c
  • modules/md/md_acme_drive.c
  • 呼び出し先:
  • apr_pvsprintf (APR)
  • on_change (内部関数)

関連データ構造

  • md_result_t: 処理結果を保持するメイン構造体。
  • apr_status_t: APR のステータスコード型。
  • va_list: 可変長引数を処理するための標準型。

失敗モードと境界条件

  • メモリ不足: apr_pvsprintf がメモリ割り当てに失敗した場合、APR の規約に従いプロセスが終了する可能性があります。
  • result が NULL の場合: 動作は未定義です（ソースコード内に明示的なガードはありません）。

保守担当者向けメモ

  • 本関数は md_result_t の状態を上書きするため、以前に設定されていた subproblems は破棄されます。
  • 状態変更時に外部へ通知を行うため、この関数を呼び出す際は、関連するコールバックが実行されることを考慮してください。
