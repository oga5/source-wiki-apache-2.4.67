main

定義

  • 種別: function
  • 定義箇所: support/suexec.c:270

目的

Apache HTTP Server の CGI プログラムを、特定のユーザー権限で安全に実行するためのラッパープログラムのメインエントリポイントです。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。ターゲットユーザー名、グループ名、実行コマンドパスを受け取ります。
  • **出力**:
  • 成功時: execv によりプロセスイメージが置換されるため、戻り値はありません。
  • 失敗時: 終了ステータスコード（101〜126, 150, 255等）を返して終了します。

前提条件と不変条件

  • 実行ユーザーは AP_HTTPD_USER と一致している必要があります。
  • 実行対象のディレクトリおよびファイルは、グループや他者による書き込み権限（S_IWGRP, S_IWOTH）を持ってはなりません。
  • 実行対象のファイルに setuid/setgid ビットが設定されている場合は拒否されます。
  • 実行対象の UID/GID は、AP_UID_MIN / AP_GID_MIN 以上である必要があります。

アルゴリズムまたは振る舞い

1. **環境初期化**: clean_env() を呼び出し、安全な環境変数のみに制限します。 2. **権限検証**: 実行元ユーザーが許可されたユーザーか確認します。 3. **引数解析**: ターゲットユーザー、グループ、コマンドを特定します。 4. **ユーザー解決**: getpwnam / getpwuid 等を用いてターゲットの UID/GID を解決します。 5. **権限降格**: setgid()、initgroups()、setuid() を順に実行し、ターゲットユーザーへ権限を移行します。 6. **パス検証**: getcwd() と chdir() を使用して、実行対象が AP_DOC_ROOT 内に存在し、シンボリックリンク攻撃を受けていないことを確認します。 7. **実行**: execv() を呼び出し、ターゲットプログラムへ制御を移します。

呼び出し元と呼び出し先

  • **呼び出し元**: 特権（root）で動作する親プロセス（Apache HTTP Server 本体の CGI 実行ロジック）。
  • **呼び出し先**: clean_env(), log_err(), log_no_err(), getuid(), getpwuid(), getpwnam(), getgrnam(), getgrgid(), setgid(), initgroups(), setuid(), getcwd(), chdir(), lstat(), execv() 等。

関連データ構造

  • struct passwd: ユーザー情報保持用。
  • struct group: グループ情報保持用。
  • struct stat: ファイルおよびディレクトリの属性情報保持用。

失敗モードと境界条件

  • 引数が不足している場合（101）、ユーザー不一致（103）、コマンドパスの不正（104）、ユーザー/グループが無効（105, 106, 121）、権限昇格失敗（109, 110）、ドキュメントルート外へのアクセス（114）、ディレクトリのパーミッション不備（116）などでエラー終了します。
  • execv 後の失敗は exit(255) で終了します。

保守担当者向けメモ

  • 本関数はセキュリティの境界線であるため、修正には極めて慎重な検討が必要です。
  • strdup() で確保されたメモリは execv まで保持されます。
  • _OSD_POSIX 固有のロジックが含まれており、プラットフォーム依存の挙動に注意してください。
