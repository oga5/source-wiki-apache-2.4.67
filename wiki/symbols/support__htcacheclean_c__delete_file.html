delete_file

定義

  • 種別: function
  • 定義箇所: support/htcacheclean.c:323

目的

キャッシュディレクトリ内の指定されたファイルを削除し、関連する親ディレクトリのクリーンアップを試行します。

シグネチャまたは形状

static void delete_file(char *path, char *basename, apr_off_t *nodes, apr_pool_t *pool)

入力と出力

  • path: ファイルが存在するディレクトリパス。
  • basename: 削除対象のファイル名。
  • nodes: 削除成功時にデクリメントされる inode カウンタへのポインタ。
  • pool: メモリ割り当てに使用する親プール。
  • 戻り値: なし。

前提条件と不変条件

  • pool は呼び出し元で管理されている有効な apr_pool_t であること。
  • nodes は有効な apr_off_t 型のポインタであること。
  • 削除処理中、一時的なサブプールが作成・破棄されるため、メモリリークを防ぐ構造になっています。

アルゴリズムまたは振る舞い

1. pool から一時的なサブプール p を作成し、path と basename を結合して対象ファイルのフルパスを生成します。 2. dryrun フラグが有効な場合、apr_stat でファイルの存在を確認し、存在すれば nodes を減らします。 3. dryrun が無効な場合、apr_file_remove を実行してファイルを削除し、成功すれば nodes を減らします。 4. サブプール p を破棄します。 5. benice フラグが有効な場合、削除回数 delcount が DELETE_NICE に達するごとに apr_sleep を呼び出し、システム負荷を抑制します。 6. 最後に delete_parent を呼び出し、空になった親ディレクトリの削除を試みます。

呼び出し元と呼び出し先

  • 呼び出し元: support/htcacheclean.c 内の複数箇所（行 794, 818, 946）。
  • 呼び出し先: apr_pool_create, apr_pstrcat, apr_stat, apr_file_remove, apr_pool_destroy, apr_sleep, delete_parent。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_off_t: inode カウントの管理に使用。
  • dryrun, benice, delcount: グローバルな制御フラグおよびカウンタ。

失敗モードと境界条件

  • apr_file_remove が失敗した場合、nodes は減らされません。
  • dryrun モード時は実際のファイル削除は行われず、統計情報の更新のみが行われます。
  • delete_parent の呼び出しにより、ファイル削除後にディレクトリ構造が空になった場合は再帰的にディレクトリも削除されます。

保守担当者向けメモ

  • 本関数は htcacheclean のファイル削除ロジックの末端を担います。
  • benice によるスロットリングは、大量のファイル削除時にシステム負荷を抑えるために重要です。
  • delete_parent への依存があるため、ディレクトリ構造の整合性については delete_parent の仕様も併せて確認してください。
