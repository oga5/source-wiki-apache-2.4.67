send_response_header

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_isapi.c:625

目的

ISAPI 拡張から提供されたステータス行およびヘッダー文字列を解析し、Apache の request_rec に反映させることで、HTTP レスポンスヘッダーを構築します。

シグネチャまたは形状

apr_ssize_t send_response_header(isapi_cid *cid,
                                 const char *stat,
                                 const char *head,
                                 apr_size_t statlen,
                                 apr_size_t headlen)

入力と出力

  • **入力**:
  • cid: ISAPI 接続コンテキスト。
  • stat: ステータス行文字列（NULL 可）。
  • head: ヘッダー文字列（NULL 可）。
  • statlen, headlen: 各文字列の長さ。
  • **出力**:
  • 戻り値: 解析および消費したヘッダーのバイト数（apr_ssize_t）。エラー時は -1。

前提条件と不変条件

  • cid->r->pool が有効であり、メモリ割り当てに使用されます。
  • cid->ecb が初期化されており、HTTP ステータスコードの同期先として使用されます。

アルゴリズムまたは振る舞い

1. **入力正規化**: stat または head が欠落している場合、入力文字列を再配置して解析可能な形式に整えます。 2. **ステータス行の構築**: stat が数値で始まらない場合、Status: プレフィックスを付与して Apache が解釈可能な形式に変換します。 3. **ヘッダー解析**: ap_scan_script_header_err_strs_ex を呼び出し、Apache のリクエスト構造体にヘッダーを適用します。 4. **ステータスコードの同期**: 解析結果、cid->r->status、および cid->ecb->dwHttpStatusCode の間でステータスコードの整合性を図ります。 5. **完了判定**: 内部サーバーエラーが発生した場合は -1 を返し、それ以外は消費したバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/win32/mod_isapi.c 内の複数の箇所（行 826, 917, 1098, 1105, 1330）から呼び出されます。
  • **呼び出し先**:
  • ap_scan_script_header_err_strs_ex (Apache API)
  • ap_get_status_line (Apache API)
  • ap_log_rerror (Apache API)

関連データ構造

  • isapi_cid: ISAPI 拡張のリクエスト処理状態を保持する構造体。
  • EXTENSION_CONTROL_BLOCK (ECB): ISAPI 拡張とのインターフェース構造体。

失敗モードと境界条件

  • cid->r->status が HTTP_INTERNAL_SERVER_ERROR に設定された場合、関数は -1 を返して処理を中断します。
  • ヘッダーが NULL 終端されていない場合、apr_pstrndup を使用して安全なコピーを作成します。
  • ステータスコードが特定できない場合は HTTP_OK をデフォルト値として使用し、デバッグログを出力します。

保守担当者向けメモ

  • 本関数は ISAPI の HSE_REQ_SEND_RESPONSE_HEADER 相当の処理を担います。
  • fake_async 関連の処理や、IIS との挙動差異（ヘッダー末尾の \r\n の扱い等）を吸収するロジックが含まれています。変更時は ap_scan_script_header_err_strs_ex の仕様との整合性に注意してください。
