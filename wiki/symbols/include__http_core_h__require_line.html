require_line

定義

  • 種別: struct
  • 定義箇所: include/http_core.h:311

目的

Apache HTTP Server の認証・認可プロセスにおいて、設定ファイル内の Require ディレクティブによって指定された行（設定値）を保持するための構造体です。

シグネチャまたは形状

typedef struct require_line require_line;

入力と出力

  • 入力: 設定ファイルから読み込まれた Require ディレクティブの引数文字列。
  • 出力: 認証プロバイダーが解析可能な形式で保持される設定データ。

前提条件と不変条件

  • 認証プロバイダーの parse_require_line 関数によって、文字列形式の require_line が解析され、構造体として格納されることが想定されています。

アルゴリズムまたは振る舞い

  • 本構造体は、Require ディレクティブの引数を保持するコンテナとして機能します。
  • 認証モジュール（mod_authnz_fcgi や mod_authnz_ldap など）は、この構造体を通じて設定値を受け取り、アクセス制御の判定に使用します。

呼び出し元と呼び出し先

  • 呼び出し元: include/mod_auth.h で定義された parse_require_line コールバック関数が、この構造体を利用して解析結果を格納します。
  • 呼び出し先: 認証プロバイダーの実装（mod_authnz_fcgi.c, mod_authnz_ldap.c 等）が、解析済みデータとして本構造体を参照します。

関連データ構造

  • cmd_parms: 設定解析時に使用されるコマンドパラメータ構造体。
  • mod_auth.h 内の認証プロバイダー用関数ポインタ（parse_require_line）。

失敗モードと境界条件

  • 解析対象の require_line が空文字列である場合、プロバイダー側で適切に処理される必要があります（例: mod_authnz_fcgi.c での strcmp チェック）。

保守担当者向けメモ

  • 本構造体の完全なメンバ定義は include/http_core.h の後続行に存在します。
  • mod_auth.h のインターフェースと密接に関連しているため、認証関連の変更時には整合性に注意してください。
