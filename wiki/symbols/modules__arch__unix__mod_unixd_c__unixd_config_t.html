unixd_config_t

定義

  • 種別: struct
  • 定義箇所: modules/arch/unix/mod_unixd.c:62

目的

Apache HTTP Server が Unix 系 OS 上で動作する際、プロセス実行に必要なユーザー権限、グループ権限、および実行環境（chroot）の設定値を保持するために使用されます。

シグネチャまたは形状

typedef struct {
  const char *user_name;
  uid_t user_id;
  gid_t group_id;
  const char *chroot_dir;
} unixd_config_t;

入力と出力

本構造体は設定保持用であり、直接的な入力・出力関数ではありません。

  • **入力**: unixd_set_user、unixd_set_group、unixd_set_chroot_dir 等のディレクティブハンドラを通じて各フィールドが設定されます。
  • **出力**: ap_unixd_setup_child 等のプロセス管理関数が、本構造体の値を参照して setuid や chroot などのシステムコールを実行します。

前提条件と不変条件

  • user_id および group_id は、OS 上で有効なユーザー ID およびグループ ID である必要があります。
  • chroot_dir が設定されている場合、そのディレクトリはプロセス実行時に chroot システムコールが成功するパスである必要があります。
  • 権限降格操作は、プロセスが root 権限で実行されている場合にのみ有効です。

アルゴリズムまたは振る舞い

本構造体は、サーバーの起動フェーズで初期化され、設定ファイル（httpd.conf 等）のディレクティブに基づいて値が更新されます。子プロセスの生成時には、この構造体に格納された値を用いて、実行ユーザーの切り替えおよび環境の分離が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: 設定解析フェーズにおいて unixd_set_user 等のハンドラが本構造体を更新します。また、子プロセス生成時に ap_unixd_setup_child が本構造体を参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • unixd.h: 本構造体の定義を共有するためのヘッダーファイル。
  • ap_unixd_config: 本構造体の型を持つグローバルな設定インスタンス（mod_unixd.c 内で管理）。

失敗モードと境界条件

  • 設定値が不正な場合（存在しないユーザー名やグループ名など）、サーバー起動時の検証プロセスでエラーが発生します。
  • chroot や setuid の実行時に権限不足やパスの誤りがある場合、ap_unixd_setup_child 内で APLOG_ALERT レベルのログが出力され、プロセス生成が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はモジュールのライフサイクル全体で生存するグローバルな設定領域として扱われます。
  • set_group_privs 関数内では、ユーザー名から ID を解決するロジックが含まれており、設定値の整合性維持に注意が必要です。
