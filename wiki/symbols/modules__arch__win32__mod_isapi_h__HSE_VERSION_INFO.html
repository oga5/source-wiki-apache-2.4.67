HSE_VERSION_INFO

定義

  • 種別: struct
  • 定義箇所: modules/arch/win32/mod_isapi.h:36

目的

ISAPI 拡張 DLL のバージョン情報および説明文を保持し、Apache が ISAPI 拡張をロードする際にその仕様を確認するために使用されます。

シグネチャまたは形状

typedef struct HSE_VERSION_INFO {
    apr_uint32_t dwExtensionVersion;
    char         lpszExtensionDesc[256];
} HSE_VERSION_INFO;

入力と出力

  • **入力**: GetExtensionVersion 関数に引数として渡され、ISAPI 拡張側で値を設定します。
  • **出力**: 設定された構造体は、Apache 側の isapi_version フィールド（mod_isapi.c 内）に格納され、拡張機能のメタデータとして利用されます。

前提条件と不変条件

  • lpszExtensionDesc は最大 256 バイトの文字列を保持可能です。
  • dwExtensionVersion は apr_uint32_t 型であり、ISAPI のバージョン番号を格納します。

アルゴリズムまたは振る舞い

1. Apache が ISAPI DLL をロードする際、GetExtensionVersion 関数を呼び出します。 2. 拡張 DLL は、この構造体の各フィールドに自身のバージョンと説明文を書き込みます。 3. Apache はこの情報を取得し、拡張機能の識別や管理に使用します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_isapi.c 内の GetExtensionVersion 実装。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • EXTENSION_CONTROL_BLOCK: ISAPI 拡張のリクエスト処理における主要なコンテキスト構造体。
  • GetExtensionVersion: 本構造体を引数として受け取る初期化関数。

失敗モードと境界条件

  • lpszExtensionDesc のサイズ制限（256 バイト）を超えた書き込みは、バッファオーバーフローを引き起こす可能性があるため注意が必要です。

保守担当者向けメモ

  • 本構造体は Microsoft IIS の ISAPI 2.0 仕様に基づいています。
  • mod_isapi.c の 258 行目付近で apr_pcalloc を使用してメモリ確保が行われています。メモリの所有権は Apache 側にあり、ISAPI 拡張側で永続化すべきではありません。
