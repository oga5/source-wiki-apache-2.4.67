ap_ssl_conn_is_ssl

定義

  • 種別: function
  • 定義箇所: server/ssl.c:90

目的

指定された接続（conn_rec）が SSL/TLS を使用しているかどうかを判定します。

シグネチャまたは形状

int ap_ssl_conn_is_ssl(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（判定対象の接続構造体）
  • **出力**: int（SSL が有効な場合は非ゼロ、無効な場合は 0）

前提条件と不変条件

  • c は有効な conn_rec ポインタである必要があります。
  • 内部で ap_run_ssl_conn_is_ssl フックを呼び出すため、SSL 関連モジュールが適切にロードされていることが期待されます。

アルゴリズムまたは振る舞い

1. ap_run_ssl_conn_is_ssl(c) を実行し、戻り値が OK であれば SSL 接続とみなして 1 を返します。 2. フックが OK を返さなかった場合、後方互換性のために保持されている module_ssl_is_https 関数ポインタが存在すれば、それを呼び出して結果を判定します。 3. 上記のいずれにも該当しない場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • ap_run_ssl_conn_is_ssl（フック）
  • module_ssl_is_https（オプション関数）
  • **呼び出し元**:
  • modules/http2/h2_c1.c
  • modules/http2/h2_c1_io.c
  • modules/http2/h2_protocol.c
  • modules/http2/h2_stream.c
  • modules/http2/h2_switch.c
  • modules/lua/mod_lua.c
  • modules/mappers/mod_rewrite.c

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。

失敗モードと境界条件

  • SSL 関連モジュールがロードされていない、またはフックが登録されていない場合、module_ssl_is_https が NULL であれば常に 0 を返します。
  • c が NULL の場合の挙動は、渡された先のフック実装に依存します。

保守担当者向けメモ

  • 本関数は SSL 抽象化レイヤーの公開 API です。新しい SSL 実装を追加する場合は、ap_run_ssl_conn_is_ssl フックを介して実装してください。
  • module_ssl_is_https は古いモジュールとの互換性のために残されています。将来的にこのフォールバック機構が不要になった場合は削除を検討してください。
