lua_db_prepared_statement

定義

  • **種別**: struct
  • **定義箇所**: modules/lua/lua_dbd.h:47

目的

Lua スクリプトから Apache のデータベース抽象化レイヤー（apr_dbd）を介して、プリペアドステートメントを管理・実行するためのラッパー構造体です。

シグネチャまたは形状

typedef struct {
    apr_dbd_prepared_t      *statement;
    int                     variables;
    lua_db_handle           *db;
} lua_db_prepared_statement;

入力と出力

  • **入力**:
  • statement: apr_dbd_prepared_t 型のポインタ（実際のデータベースドライバ側のプリペアドステートメント）。
  • variables: ステートメントが保持する変数の数。
  • db: 関連付けられた lua_db_handle へのポインタ。
  • **出力**: なし（本構造体は状態保持用）。

前提条件と不変条件

  • db フィールドは、有効なデータベース接続（lua_db_handle）を指している必要があります。
  • statement は、apr_dbd の関数によって初期化された有効なポインタである必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、Lua のユーザーデータとしてスタック上に生成され、Lua スクリプトからデータベースクエリを実行する際のコンテキストとして機能します。
  • lua_dbd.c 内の関数によって、Lua の引数を C の型に変換し、statement を用いてデータベース操作を行う際に参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_dbd.c 内のデータベース操作関数（prepare 関連処理など）から、lua_newuserdata を介してインスタンス化されます。
  • **呼び出し先**: 構造体内部の statement フィールドが apr_dbd の関数群に渡されます。
  • *注記*: 具体的な関数呼び出しの連鎖については、提供されたソース excerpt には詳細なコールグラフが含まれていないため、これ以上の情報は不明です。

関連データ構造

  • lua_db_handle: データベース接続状態を管理する構造体。本構造体の db フィールドから参照されます。
  • apr_dbd_prepared_t: APR が提供する低レベルのプリペアドステートメント型。

失敗モードと境界条件

  • 接続が切断された状態で statement を使用しようとした場合の挙動は、apr_dbd の実装に依存します。
  • lua_newuserdata によるメモリ確保が失敗した場合、Lua VM 側でエラーが発生します。

保守担当者向けメモ

  • 本構造体は lua_dbd.c で生成・管理されます。リソースの解放が必要な場合は、対応するガベージコレクション関数（lua_db_gc 等）が適切に実装されているか確認してください。
