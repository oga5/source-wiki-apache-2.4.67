ssl_hook_http_scheme

定義

  • 種別: function
  • 定義箇所: modules/ssl/mod_ssl.c:648

目的

リクエストが SSL/TLS 経由で処理されているかを判定し、Apache のスキーム解決フェーズにおいて適切なスキーム名（https）を返す。

シグネチャまたは形状

static const char *ssl_hook_http_scheme(const request_rec *r)

入力と出力

  • **入力**: const request_rec *r （現在のリクエスト構造体）
  • **出力**:
  • TLS 接続である場合: 文字列 "https"
  • それ以外の場合: NULL

前提条件と不変条件

  • modssl_request_is_tls が現在のリクエストに対して有効な TLS 状態を判定できること。

アルゴリズムまたは振る舞い

1. modssl_request_is_tls(r, NULL) を呼び出し、現在のリクエストが TLS 接続に関連付けられているかを確認する。 2. 判定結果が真であれば "https" を返し、偽であれば NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の ap_hook_http_scheme フック（modules/ssl/mod_ssl.c:732 にて登録）。
  • **呼び出し先**: modssl_request_is_tls （TLS 判定の内部関数）。

関連データ構造

  • request_rec: Apache のリクエスト処理における主要な構造体。

失敗モードと境界条件

  • r が NULL の場合の挙動は、modssl_request_is_tls の実装に依存する。
  • 非 SSL 接続時には NULL を返すため、呼び出し側は戻り値が NULL の場合にデフォルトのスキーム（通常は http）を想定する必要がある。

保守担当者向けメモ

  • 本関数は Apache のスキーム解決フックの一部であり、URL 生成時に正しいプロトコルスキームを反映させるために重要である。
  • modssl_request_is_tls の判定ロジックに変更があった場合、本関数の戻り値にも直接影響する。
