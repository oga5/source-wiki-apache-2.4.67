remoteip_conn_config_t

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_remoteip.c:143

目的

PROXY プロトコル等を通じて解決された、接続単位のクライアント情報を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    apr_sockaddr_t *client_addr;
    char *client_ip;
} remoteip_conn_config_t;

入力と出力

  • client_addr: apr_sockaddr_t 型。解析されたクライアントの IP アドレスをネイティブ形式で保持します。
  • client_ip: char 型のポインタ。クライアント IP アドレスの文字列表現を保持します。

前提条件と不変条件

  • 本構造体は接続単位で管理されます。
  • client_addr および client_ip は、PROXY プロトコル等の解析処理が完了した後に設定されます。

アルゴリズムまたは振る舞い

  • mod_remoteip が接続を処理する際、PROXY プロトコルヘッダーの解析結果を本構造体に格納します。
  • リクエスト処理（post_read_request フック）において、この構造体が保持する情報が r->useragent_addr や r->useragent_ip の上書きに使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/metadata/mod_remoteip.c 内の複数の関数（524行目、770行目、879行目、945行目、1054行目）で、接続設定の取得や参照のために使用されています。
  • 呼び出し先: 具体的な関数呼び出しの詳細はソース抜粋からは不明です。

関連データ構造

  • apr_sockaddr_t: APR が提供するソケットアドレス構造体。
  • proxy_header: PROXY プロトコル解析時に使用される共用体。

失敗モードと境界条件

  • 構造体が初期化されていない場合、または解析に失敗した場合は、接続情報の書き換えが行われない可能性があります。

保守担当者向けメモ

  • 本構造体は mod_remoteip の接続コンテキストとして機能します。リクエスト処理の初期段階で正しい IP を後続モジュールへ伝播させるための重要なデータ保持先です。
