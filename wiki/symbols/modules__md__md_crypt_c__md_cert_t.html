md_cert_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_crypt.c:1200

目的

X.509 証明書を保持し、Apache HTTP Server の mod_md における暗号化処理の抽象化層として機能するデータ構造です。

シグネチャまたは形状

struct md_cert_t {
    apr_pool_t *pool;
    X509 *x509;
    apr_array_header_t *alt_names;
};

入力と出力

  • **入力**: md_cert_read_http や md_cert_fload 等の関数を通じて、DER/PEM 形式のデータから本構造体が生成されます。
  • **出力**: 保持する X509 オブジェクトや alt_names は、証明書の有効期限チェックや SAN（Subject Alternative Names）抽出などの検証処理に利用されます。

前提条件と不変条件

  • すべてのインスタンスは APR プールと紐付けられており、プールが破棄される際に X509_free が自動的に呼び出されるライフサイクル管理が行われます。
  • x509 フィールドは OpenSSL の X509 構造体をラップしています。

アルゴリズムまたは振る舞い

  • 本構造体は、証明書の解析結果を保持するコンテナとして機能します。
  • md_crypt.c 内の関数群（md_cert_read_chain 等）により、ファイルや HTTP レスポンスから読み込まれた証明書データが本構造体に格納されます。

呼び出し元と呼び出し先

  • **呼び出し元**: md_acme_drive.c および md_acme_authz.c において、証明書チェーンの管理や検証のために参照されています。
  • **呼び出し先**: OpenSSL の X509 関連 API を利用して証明書の内容を操作します。

関連データ構造

  • X509: OpenSSL の証明書構造体。
  • apr_pool_t: メモリ管理用プール。
  • apr_array_header_t: alt_names を格納するための APR 配列。

失敗モードと境界条件

  • メモリ割り当て失敗時や、不正な証明書データが入力された場合の挙動は、OpenSSL のエラーハンドリングおよび APR のメモリ管理ポリシーに依存します。

保守担当者向けメモ

  • OpenSSL のバージョン差異（1.0.x と 1.1.x/3.0.x）を吸収するマクロが md_crypt.c 全体で使用されているため、本構造体のフィールド操作を行う際は注意してください。
  • 手動での free は不要ですが、APR プールの生存期間と証明書オブジェクトのライフサイクルが一致していることを確認してください。
