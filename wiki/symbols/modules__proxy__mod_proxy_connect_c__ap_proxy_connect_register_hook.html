ap_proxy_connect_register_hook

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_connect.c:383

目的

mod_proxy_connect モジュールが提供するハンドラ関数を、Apache HTTP Server のフックシステムに登録します。

シグネチャまたは形状

static void ap_proxy_connect_register_hook(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（フック登録に使用されるメモリプール）
  • **出力**: なし（void）

前提条件と不変条件

  • 本関数はモジュールの初期化フェーズで呼び出されることを前提としています。
  • proxy_hook_scheme_handler および proxy_hook_canon_handler が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. proxy_hook_scheme_handler を呼び出し、proxy_connect_handler を APR_HOOK_MIDDLE の優先度で登録します。これにより、CONNECT メソッドのリクエスト処理が本モジュールに委譲されます。 2. proxy_hook_canon_handler を呼び出し、proxy_connect_canon を APR_HOOK_MIDDLE の優先度で登録します。これにより、リクエスト URL の正規化および検証処理が本モジュールに委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_connect.c:403（モジュールのフック登録処理内）
  • **呼び出し先**:
  • proxy_hook_scheme_handler
  • proxy_hook_canon_handler
  • proxy_connect_handler（登録対象）
  • proxy_connect_canon（登録対象）

関連データ構造

  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • 本関数自体は戻り値を返さないため、登録失敗時のエラーハンドリングは呼び出し側のフックシステムに依存します。

保守担当者向けメモ

  • 本関数は mod_proxy のフックシステムを利用して CONNECT メソッドの処理を統合しています。ハンドラを追加・変更する場合は、この登録順序や優先度を確認してください。
