just_die

定義

  • 種別: function
  • 定義箇所: server/mpm/prefork/prefork.c:357

目的

シグナルハンドラとして呼び出され、子プロセスを正常に終了させるためのラッパー関数です。

シグネチャまたは形状

static void just_die(int sig)

入力と出力

  • **入力**: int sig（受信したシグナル番号）
  • **出力**: なし（関数内でプロセスを終了させるため戻り値はありません）

前提条件と不変条件

  • 本関数はシグナルハンドラとして登録されることを前提としています。
  • 呼び出し後、プロセスは clean_child_exit を経由して終了するため、呼び出し元へ制御は戻りません。

アルゴリズムまたは振る舞い

1. 引数として受け取ったシグナル番号を無視し、clean_child_exit(0) を呼び出します。 2. clean_child_exit により、子プロセスのクリーンアップ処理が実行され、プロセスが終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/prefork/prefork.c 内のシグナル設定箇所（SIGHUP, SIGTERM）。
  • 他の MPM 実装（event.c, worker.c）からもシグナルハンドラとして参照されていることがツールにより示唆されています。
  • **呼び出し先**: clean_child_exit

関連データ構造

  • なし

失敗モードと境界条件

  • 本関数自体は単純なラッパーであるため、失敗モードは clean_child_exit の挙動に依存します。

保守担当者向けメモ

  • 本関数は apr_signal を介してシグナルハンドラとして登録されます。
  • just_die という名称ですが、実行されるのは clean_child_exit(0) であり、正常終了（ステータス0）として扱われます。
  • 他の MPM（event, worker）でも同様のシグナルハンドラとして参照されているため、変更時は影響範囲に注意してください。
