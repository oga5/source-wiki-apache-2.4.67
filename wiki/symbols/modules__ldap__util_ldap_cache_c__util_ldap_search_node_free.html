util_ldap_search_node_free

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache.c:200

目的

LDAP 検索結果を保持するキャッシュノード（util_search_node_t）がキャッシュから削除される際、ノード内部で動的に確保されたメモリを再帰的に解放する。

シグネチャまたは形状

void util_ldap_search_node_free(util_ald_cache_t *cache, void *n)

入力と出力

  • cache: キャッシュ管理構造体へのポインタ。util_ald_free によるメモリ解放に使用される。
  • n: 解放対象のキャッシュノード（util_search_node_t 型へのポインタ）。
  • 戻り値: なし。

前提条件と不変条件

  • n は util_search_node_t 型の有効なポインタであること。
  • ノード内の各メンバ（vals, username, dn, bindpw）は、util_ald_alloc を使用して確保された共有メモリ領域であること。
  • cache は、当該ノードが割り当てられたメモリ領域と整合していること。

アルゴリズムまたは振る舞い

1. ノード内の値配列 vals が存在する場合、各要素をループで走査し、util_ald_free を用いて個別に解放する。 2. vals 配列自体を util_ald_free で解放する。 3. ノード内の文字列メンバ username、dn、bindpw をそれぞれ util_ald_free で解放する。 4. 最後にノード構造体自体を util_ald_free で解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/ldap/util_ldap_cache.c 内のキャッシュ挿入失敗時やパージ処理（162, 169, 183, 188行目）。
  • modules/ldap/util_ldap_cache_mgr.c におけるキャッシュ管理処理（295行目）。
  • 呼び出し先:
  • util_ald_free（共有メモリ解放用関数）。

関連データ構造

  • util_ald_cache_t: キャッシュ管理構造体。
  • util_search_node_t: 解放対象となる検索ノード構造体。

失敗モードと境界条件

  • node->vals が NULL の場合、配列の解放処理はスキップされる。
  • 各文字列メンバが NULL の場合、util_ald_free は安全に処理をスキップする（util_ald_free の実装依存）。
  • 共有メモリ領域外のポインタを渡した場合、メモリ破壊やセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は標準の free() ではなく、必ず util_ald_free() を使用すること。これは共有メモリ（apr_rmm）上の領域を管理しているためである。
  • ノード構造体の定義が変更され、新しい動的メンバが追加された場合は、本関数で忘れずに解放処理を追加すること。
