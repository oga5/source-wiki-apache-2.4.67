yy_buffer_state

定義

  • **種別**: struct
  • **定義箇所**: server/util_expr_scan.c:211

目的

Flex によって生成された再入可能スキャナにおいて、入力ストリームの状態、バッファ内容、およびスキャン位置を保持するために使用されます。

シグネチャまたは形状

struct yy_buffer_state {
    FILE *yy_input_file;
    char *yy_ch_buf;
    char *yy_buf_pos;
    yy_size_t yy_buf_size;
    int yy_n_chars;
    int yy_is_our_buffer;
    int yy_is_interactive;
    int yy_at_bol;
    int yy_bs_lineno;
    int yy_bs_column;
    int yy_fill_buffer;
    int yy_buffer_status;
};

入力と出力

  • **入力**: yy_input_file を介したファイルストリーム、または yy_ch_buf に直接読み込まれた文字データ。
  • **出力**: 字句解析器がトークンを抽出するためのバッファ状態情報。

前提条件と不変条件

  • yy_ch_buf は入力データを保持し、yy_buf_pos は現在の読み取り位置を指します。
  • yy_is_our_buffer が真の場合、スキャナはバッファのメモリ管理（再確保や解放）の責任を負います。
  • yy_buffer_status は YY_BUFFER_NEW (0), YY_BUFFER_NORMAL (1), YY_BUFFER_EOF_PENDING (2) のいずれかの状態をとります。

アルゴリズムまたは振る舞い

  • **バッファ管理**: yy_fill_buffer が有効な場合、バッファ末尾に達した際に自動的に入力を補充します。
  • **対話的処理**: yy_is_interactive が設定されている場合、fread() の代わりに getc() を使用して改行ごとの入力停止を保証します。
  • **EOF 処理**: YY_BUFFER_EOF_PENDING 状態は、EOF に到達したが処理すべきトークンが残っている場合に設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_expr_yyalloc を使用してメモリを確保するスキャナ初期化関数群。
  • **呼び出し先**: 不明（本構造体はデータ保持用であり、関数を直接呼び出すものではありません）。

関連データ構造

  • YY_BUFFER_STATE: struct yy_buffer_state * への typedef。
  • yyguts_t: スキャナの全状態を保持する構造体であり、本構造体へのポインタをスタックとして管理します。

失敗モードと境界条件

  • yy_is_our_buffer が偽の場合、外部で管理されているメモリを誤って解放しないよう注意が必要です。
  • yy_buffer_status が YY_BUFFER_EOF_PENDING の間は、入力ソースからの読み込みを停止する必要があります。

保守担当者向けメモ

  • 本構造体は Flex が自動生成するコードの一部です。直接的な修正が必要な場合は util_expr_scan.l を編集してください。
  • メモリ管理については、ap_expr_yyfree を使用して適切に解放される必要があります。
