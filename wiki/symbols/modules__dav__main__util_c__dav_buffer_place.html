dav_buffer_place

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:163

目的

dav_buffer 構造体の現在のバッファ末尾に、指定された文字列をコピーして配置します。

シグネチャまたは形状

void dav_buffer_place(apr_pool_t *p, dav_buffer *pbuf, const char *str)

入力と出力

  • p: メモリ確保に使用する apr_pool_t。
  • pbuf: 操作対象の dav_buffer 構造体へのポインタ。
  • str: バッファにコピーするヌル終端文字列。
  • 出力: なし（pbuf の内容が更新されます）。

前提条件と不変条件

  • pbuf は初期化済みである必要があります。
  • dav_check_bufsize を内部で呼び出し、文字列の長さ（ヌル文字を含む）分だけバッファを自動拡張します。

アルゴリズムまたは振る舞い

1. strlen(str) で文字列の長さを取得します。 2. dav_check_bufsize を呼び出し、現在のバッファサイズが不足している場合は p を用いて拡張します。 3. memcpy を使用して、pbuf->buf の現在のオフセット位置（cur_len）に文字列とヌル終端文字をコピーします。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/fs/lock.c:759 での利用が確認されています。
  • 呼び出し先: dav_check_bufsize（modules/dav/main/util.c 内）。

関連データ構造

  • dav_buffer: バッファ本体、現在の長さ、および割り当て済みサイズを保持する構造体。

失敗モードと境界条件

  • apr_pool_t によるメモリ確保が失敗した場合、dav_check_bufsize の内部実装に従い、プロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は文字列をコピーする際にヌル終端文字も含めて書き込みます。
  • dav_buffer_append と異なり、本関数は pbuf->cur_len を更新しません（ソースコード上、cur_len の加算処理が欠落しているように見受けられます）。利用時は cur_len の整合性に注意してください。
