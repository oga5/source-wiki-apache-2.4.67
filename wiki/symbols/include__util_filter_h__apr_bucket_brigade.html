apr_bucket_brigade

定義

  • 種別: typedef
  • 定義箇所: include/util_filter.h:134

目的

Apache HTTP Server のフィルタチェーンにおいて、データ（バケット）のストリームを保持・管理するための抽象的なコンテナです。

シグネチャまたは形状

apr_bucket_brigade は apr_buckets.h で定義される不透明な構造体であり、本ヘッダーファイルではフィルタ関数の引数としてポインタ型で使用されます。

入力と出力

  • **入力**: フィルタ関数（ap_out_filter_func 等）の引数として渡され、処理対象のデータストリームを提供します。
  • **出力**: フィルタ処理の結果として、後続のフィルタへ渡されるか、あるいは呼び出し元へ返されます。

前提条件と不変条件

  • ap_pass_brigade を介してデータを渡す際、ブリゲードの所有権は呼び出し元に帰属します。
  • フィルタ内でデータを保持する必要がある場合は、ap_save_brigade を使用して所有権を管理する必要があります。

アルゴリズムまたは振る舞い

  • フィルタチェーン内を順次通過するデータストリームとして機能します。
  • フィルタはブリゲード内のバケットを順次処理し、必要に応じて変換、追加、削除を行います。
  • ストリームの終了時には、各フィルタが内部状態をフラッシュし、リソースを解放する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_core_output_filter、ap_byterange_filter、ap_http_header_filter 等のフィルタ関数が引数として受け取ります。
  • **呼び出し先**: apr_buckets.h で定義されたバケット操作関数群（バケットの作成、分割、削除など）が、ブリゲード内の要素操作のために呼び出されます。

関連データ構造

  • ap_filter_t: フィルタチェーンの各ノード。
  • apr_bucket: ブリゲード内に格納される個々のデータ単位。
  • ap_filter_func: ブリゲードを引数に取るフィルタ関数の型定義。

失敗モードと境界条件

  • フィルタチェーンの処理中にエラーが発生した場合、apr_status_t 型の戻り値を通じてエラーが伝播されます。
  • メモリ所有権の管理を誤ると、解放済みメモリへのアクセスやメモリリークが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは apr_buckets.h に依存しています。詳細な内部構造や操作関数については、APR（Apache Portable Runtime）のバケット管理ドキュメントを参照してください。
  • フィルタチェーンの実行順序は ap_filter_type に基づいて決定されます。
