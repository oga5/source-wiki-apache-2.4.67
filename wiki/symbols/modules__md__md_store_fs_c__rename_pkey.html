rename_pkey

定義

  • 種別: function
  • 定義箇所: modules/md/md_store_fs.c:132

目的

指定されたディレクトリ内のファイルを、標準的な秘密鍵ファイル名（MD_FN_PRIVKEY）へリネームします。

シグネチャまたは形状

static apr_status_t rename_pkey(void *baton, apr_pool_t *p, apr_pool_t *ptemp, 
                                const char *dir, const char *name, 
                                apr_filetype_e ftype)

入力と出力

  • **入力**:
  • baton: 使用されていません。
  • p: メインのメモリプール。
  • ptemp: 一時的なメモリプール（パス生成に使用）。
  • dir: 対象のディレクトリパス。
  • name: リネーム対象のファイル名。
  • ftype: ファイルタイプ（使用されていません）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は apr_file_rename の戻り値またはパス生成エラーコードを返します。

前提条件と不変条件

  • dir および name が有効な文字列である必要があります。
  • md_util_path_merge が成功し、from および to パスが生成可能である必要があります。

アルゴリズムまたは振る舞い

1. ptemp を使用して、現在のファイルパス (from) と、リネーム先の標準的な秘密鍵ファイルパス (to) を生成します。 2. パス生成が成功した場合、apr_file_rename を呼び出してファイルを移動（リネーム）します。 3. 処理の過程で、リネーム操作をトレースログ (MD_LOG_TRACE1) に記録します。 4. baton および ftype 引数は無視されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_store_fs.c:191 にて md_util_files_do のコールバックとして参照されています。
  • **呼び出し先**:
  • md_util_path_merge
  • md_log_perror
  • apr_file_rename

関連データ構造

  • MD_FN_PRIVKEY: 秘密鍵の標準ファイル名定義。

失敗モードと境界条件

  • md_util_path_merge が失敗した場合、リネームは行われず、初期値の APR_SUCCESS が返されます（ただし、条件式により実質的に何も実行されません）。
  • apr_file_rename が失敗した場合、そのエラーコードが呼び出し元に伝播されます。

保守担当者向けメモ

  • 本関数は md_util_files_do からの反復処理の一部として呼び出されることを想定しています。
  • baton 引数が将来的に利用される可能性があるため、シグネチャの変更には注意してください。
