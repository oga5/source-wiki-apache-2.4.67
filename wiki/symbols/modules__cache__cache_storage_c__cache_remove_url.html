cache_remove_url

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_storage.c:34

目的

指定されたリクエストに関連するキャッシュエントリを、設定されているすべてのキャッシュストレージプロバイダーから削除します。

シグネチャまたは形状

int cache_remove_url(cache_request_rec *cache, request_rec *r)

入力と出力

  • **入力**:
  • cache: キャッシュリクエストコンテキスト。
  • r: 現在のリクエスト構造体。
  • **出力**:
  • int: 常に OK を返します。

前提条件と不変条件

  • cache->stale_handle または cache->handle のいずれかが有効なキャッシュハンドルを保持している必要があります。
  • 両方のハンドルが NULL の場合、関数は何も行わず直ちに OK を返します。

アルゴリズムまたは振る舞い

1. cache->stale_handle が存在する場合はそれを、存在しない場合は cache->handle を削除対象のハンドル h として選択します。 2. h が NULL であれば処理を終了します。 3. 削除対象の URL をデバッグログに出力します。 4. cache->providers リストを走査し、リスト内の各プロバイダーに対して remove_url 関数を呼び出し、キャッシュエントリの削除を試行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c (複数箇所で呼び出しを確認)
  • **呼び出し先**:
  • list->provider->remove_url: 各ストレージプロバイダー（mod_cache_disk や mod_cache_socache 等）が実装する削除関数。
  • ap_log_rerror: ログ出力用関数。

関連データ構造

  • cache_request_rec: キャッシュ操作のコンテキストを保持。
  • cache_handle_t: 削除対象となるキャッシュエントリへのハンドル。
  • cache_provider_list: 登録されているキャッシュプロバイダーのリスト。

失敗モードと境界条件

  • ハンドルが NULL の場合、エラーにはならず OK を返します。
  • プロバイダーのリストが空の場合、ループは実行されず何も削除されません。

保守担当者向けメモ

  • 本関数はストレージ抽象化レイヤーの一部であり、実際の削除ロジックは各プロバイダーに委譲されています。
  • 削除対象のハンドル選択において stale_handle が優先されるため、古いエントリのクリーンアップが優先的に行われる設計となっています。
