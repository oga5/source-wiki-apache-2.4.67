bs2_ForkType

定義

  • 種別: enum
  • 定義箇所: os/unix/unixd.c:635

目的

BS2000（_OSD_POSIX）環境において、プロセス生成（fork）の挙動を制御・識別するために使用されます。

シグネチャまたは形状

typedef enum {
    bs2_unknown,
    bs2_noFORK,
    bs2_FORK,
    bs2_UFORK
} bs2_ForkType;

入力と出力

  • 本シンボル自体は列挙型定義です。
  • 関連する関数 os_forktype() は、引数 int one_process を受け取り、bs2_ForkType 型の値を返します。

前提条件と不変条件

  • bs2000 環境下でのみ有効なロジックを想定しています。
  • bs2_unknown は初期状態を示します。

アルゴリズムまたは振る舞い

各列挙値は以下の状態を表します：

  • bs2_unknown: 初期化前。
  • bs2_noFORK: -X フラグ指定時など、fork() を行わない状態。
  • bs2_FORK: ユーザーIDが 0（root）ではない場合に fork() を使用する状態。
  • bs2_UFORK: ユーザーIDの切り替えに ufork() を使用する通常状態。

呼び出し元と呼び出し先

  • 呼び出し元: os/bs2000/os.c および os/unix/unixd.c 内の os_forktype() 関数がこの型を参照・返却しています。
  • 呼び出し先: 情報なし。

関連データ構造

  • static bs2_ForkType forktype: 各ソースファイル内で状態保持のために定義されている静的変数。

失敗モードと境界条件

  • bs2_unknown のまま処理が進行した場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 本シンボルは _OSD_POSIX（BS2000）プラットフォーム固有のレガシーな実装です。一般的な Unix/Linux 環境では使用されません。
  • os/bs2000/os.c と os/unix/unixd.c の両方で定義・利用されているため、修正時は両方の整合性に注意してください。
