check_finalized

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:160

目的

sed スクリプトのコンパイル状態が完了可能（finalized）であるかを検証します。具体的には、ブロック構造のネストが閉じているか、および未解決のラベル参照が存在しないかを確認します。

シグネチャまたは形状

static int check_finalized(const sed_commands_t *commands)

入力と出力

  • **入力**: commands (const sed_commands_t *) - 検証対象の sed コマンド構造体。
  • **出力**: int - 完了可能であれば 1 を、そうでなければ 0 を返します。

前提条件と不変条件

  • commands は有効な sed_commands_t 構造体を指している必要があります。
  • commands->depth が 0 であることは、ブロック構造が正しく閉じていることを示す不変条件です。

アルゴリズムまたは振る舞い

2. **ラベル解決の確認**: commands->labtab + 1 から commands->lab までの範囲で、各ラベルを走査します。 3. **検証**: ラベルの address が未設定（0）であるか、または chain（未解決の参照チェーン）が存在する場合、ラベルが解決されていないと判断し 0 を返します。 4. **成功**: 上記のいずれにも該当しない場合、1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed0.c:108: commands->canbefinal の代入時に呼び出されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。構造体メンバへのアクセスのみを行います。

関連データ構造

  • sed_commands_t: sed スクリプトのコンパイル済みコマンドと状態を保持するメイン構造体。
  • sed_label_t: ラベル情報および分岐先アドレス、未解決参照チェーンを保持する構造体。

失敗モードと境界条件

  • **ラベル未解決**: lab->address == 0 または lab->chain != NULL の場合、ラベル解決失敗として 0 を返します。
  • **空の分岐チェーン**: コメントにある通り、空の分岐チェーンはエラーとはみなされません。

保守担当者向けメモ

  • 本関数はコンパイルの最終段階で呼び出される検証ロジックです。
  • commands->labtab は sed_commands_t 内のラベルテーブルの先頭を指すと想定されています。
  • 呼び出し元で canbefinal フラグとして結果が保持されるため、この関数の戻り値はコンパイル後の最終的なスクリプトの妥当性判定に使用されます。
