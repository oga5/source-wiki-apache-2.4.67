read_table

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_socache.c:195

目的

シリアライズされたバッファから HTTP ヘッダーを読み取り、apr_table_t 構造体へ復元します。

シグネチャまたは形状

apr_status_t read_table(cache_handle_t *handle, request_rec *r,
                        apr_table_t *table, unsigned char *buffer,
                        apr_size_t buffer_len, apr_size_t *slider);

入力と出力

  • **入力**:
  • handle: キャッシュハンドル。
  • r: 現在のリクエストレコード。
  • table: ヘッダーを格納する apr_table_t。
  • buffer: シリアライズされたデータを含むバッファ。
  • buffer_len: バッファの総サイズ。
  • slider: バッファ内の現在の読み取り位置（オフセット）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、終端到達時は APR_EOF、エラー時は APR_EGENERAL を返します。
  • slider: 読み取り後のオフセット位置が更新されます。
  • table: 読み取られたヘッダーが追加されます。

前提条件と不変条件

  • buffer は store_table 等で生成された、key:value\r\n 形式のシリアライズ済みデータである必要があります。
  • slider はバッファ内の有効な開始位置を指している必要があります。

アルゴリズムまたは振る舞い

1. slider を起点にバッファを走査し、: を区切り文字としてキーと値を識別します。 2. \r を行末として検出し、apr_table_addn を用いて table にヘッダーを追加します。 3. \r\n のシーケンスを適切にスキップし、次のヘッダーへ進みます。 4. \0（ヌル文字）を検出した場合は、ヘッダーブロックの終了とみなして APR_SUCCESS を返します。 5. バッファの終端に達した場合は APR_EOF を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_socache.c: read_table がキャッシュエントリの復元時に呼び出されます。
  • modules/cache/mod_cache_disk.c: read_table がディスクキャッシュのヘッダー読み込み時に呼び出されます。
  • **呼び出し先**:
  • apr_table_addn: ヘッダーの追加に使用。
  • apr_pstrmemdup: 文字列のメモリコピーに使用。
  • ap_log_rerror: エラーログ出力に使用。

関連データ構造

  • apr_table_t: HTTP ヘッダーを保持する標準的なテーブル構造。
  • cache_handle_t: キャッシュ操作のコンテキスト。

失敗モードと境界条件

  • **Premature end of cache headers**: ヘッダーの形式が不正（: が見つからない等）な場合、APR_EGENERAL を返しエラーログを出力します。
  • **バッファ終端**: ヌル文字やバッファ長を超えた場合、正常終了または APR_EOF を返します。

保守担当者向けメモ

  • 本関数は store_table と対になるデシリアライズ処理です。データフォーマットを変更する場合は、両方の関数を同期させる必要があります。
  • apr_pstrmemdup を使用しているため、リクエストプール r->pool のライフサイクルに依存します。
