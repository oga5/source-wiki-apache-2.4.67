verify_ocsp_status

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_ocsp.c:129

目的

証明書の失効状態を外部の OCSP レスポンダに問い合わせ、その結果（Good, Revoked, Unknown）を検証して OpenSSL の検証コンテキストへ反映させる。

シグネチャまたは形状

static int verify_ocsp_status(X509 *cert, X509_STORE_CTX *ctx, conn_rec *c,
                              SSLSrvConfigRec *sc, server_rec *s,
                              apr_pool_t *pool)

入力と出力

  • **入力**:
  • cert: 検証対象の X509 証明書。
  • ctx: OpenSSL の X509_STORE_CTX 検証コンテキスト。
  • c: 現在の接続情報 (conn_rec)。
  • sc: SSL サーバー設定 (SSLSrvConfigRec)。
  • s: サーバー情報 (server_rec)。
  • pool: メモリ管理用 apr_pool_t。
  • **出力**:
  • int: 検証結果ステータス（V_OCSP_CERTSTATUS_GOOD, V_OCSP_CERTSTATUS_REVOKED, V_OCSP_CERTSTATUS_UNKNOWN）。

前提条件と不変条件

  • OPENSSL_NO_OCSP が定義されていない環境で動作する。
  • determine_responder_uri が有効な URI を返さない場合、設定（SSL_OCSPCHECK_NO_OCSP_FOR_CERT_OK）に応じて GOOD または UNKNOWN を返す。
  • 通信は同期的に行われ、sc->server->ocsp_responder_timeout で指定されたタイムアウト時間が適用される。

アルゴリズムまたは振る舞い

1. **URI 取得**: determine_responder_uri を呼び出し、接続先を特定する。 2. **リクエスト生成**: create_request を使用して OCSP_REQUEST を構築する。 3. **通信**: modssl_dispatch_ocsp_request を介してレスポンダと通信し、OCSP_RESPONSE を取得する。 4. **検証**:

  • レスポンスのステータスが成功か確認。
  • OCSP_check_nonce で Nonce の整合性を確認（設定が有効な場合）。
  • OCSP_basic_verify でレスポンダの署名を検証（ocsp_noverify が無効な場合）。
  • OCSP_resp_find_status で証明書のステータスを取得。
  • OCSP_check_validity でレスポンスの有効期限を確認。

    • 5. **クリーンアップ**: 使用した OCSP 関連オブジェクトを解放し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_ocsp.c 内の modssl_verify_ocsp（直接の呼び出しを確認）。
  • **呼び出し先**:
  • determine_responder_uri
  • create_request
  • modssl_dispatch_ocsp_request
  • OpenSSL OCSP API (OCSP_response_status, OCSP_response_get1_basic, OCSP_check_nonce, OCSP_basic_verify, OCSP_resp_find_status, OCSP_check_validity)

関連データ構造

  • X509, X509_STORE_CTX: OpenSSL 証明書および検証コンテキスト。
  • OCSP_REQUEST, OCSP_RESPONSE, OCSP_BASICRESP: OCSP プロトコル用データ構造。
  • SSLSrvConfigRec: mod_ssl のサーバー設定。

失敗モードと境界条件

  • **通信失敗**: レスポンダへの接続不可やタイムアウト時は V_OCSP_CERTSTATUS_UNKNOWN を返す。
  • **検証失敗**: 署名検証エラー、Nonce 不一致、有効期限切れの場合は V_OCSP_CERTSTATUS_UNKNOWN を返す。
  • **設定依存**: ocsp_noverify が有効な場合、署名検証をスキップする。

保守担当者向けメモ

  • 本関数は同期処理であるため、ネットワーク遅延がリクエスト処理時間に直結する点に注意すること。
  • certID は request が解放される際に自動的に解放されるため、個別の解放は不要である。
