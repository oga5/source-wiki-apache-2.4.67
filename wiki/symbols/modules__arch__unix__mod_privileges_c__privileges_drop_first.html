privileges_drop_first

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_privileges.c:312

目的

サーバー起動時に、全仮想ホスト（vhost）で必要とされる特権セットを統合し、プロセスに対して初期的な権限降格（または適用）を行う。mod_unixd による setuid 実行前に特権を確定させるために使用される。

シグネチャまたは形状

static int privileges_drop_first(apr_pool_t *pool, server_rec *s)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用するプール。
  • s: サーバー設定のルート構造体。
  • **出力**:
  • 成功時は OK を返す。失敗時は PDROP_CHECK マクロによりエラーが処理される。

前提条件と不変条件

  • 本関数は APR_HOOK_FIRST としてフックされており、mod_unixd の権限降格処理よりも先に実行される必要がある。
  • Solaris 等の特権（Privileges）API が利用可能な環境であることを前提とする。

アルゴリズムまたは振る舞い

1. priv_allocset() で一時的な特権セット ppriv を確保する。 2. priv_setid（システム設定値）を ppriv にコピーする。 3. 全ての仮想ホスト（s から s->next を辿る）を走査し、各 vhost の設定（priv_cfg）に含まれる特権を priv_union を用いて ppriv に統合する。 4. setppriv を呼び出し、現在のプロセスの PRIV_PERMITTED および PRIV_EFFECTIVE セットを統合後の ppriv に設定する。 5. priv_freeset() で一時メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_drop_privileges により登録され、サーバー起動シーケンスから呼び出される（modules/arch/unix/mod_privileges.c:413）。
  • **呼び出し先**:
  • priv_allocset, priv_copyset, priv_union, priv_freeset（Solaris 特権 API）
  • ap_get_module_config（Apache 設定取得）
  • setppriv（システムコール）

関連データ構造

  • priv_cfg: 各 vhost の特権設定を保持する構造体。
  • priv_set_t: Solaris の特権セットを表現するデータ型。

失敗モードと境界条件

  • PDROP_CHECK マクロにより、setppriv が失敗した場合はエラーとして処理される。
  • 仮想ホストリストが空の場合、priv_setid のみが適用される。

保守担当者向けメモ

  • 本関数は mod_unixd の setuid 実行前に特権を確定させる必要があるため、フックの順序（APR_HOOK_FIRST）を変更してはならない。
  • マルチスレッド環境では setppriv の挙動が制限されるため、本モジュールは非スレッド環境での使用を前提としている。
