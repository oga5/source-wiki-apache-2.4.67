remote_connect_info

定義

  • 種別: struct
  • 定義箇所: modules/proxy/proxy_util.c:53

目的

バックエンドへの接続に必要な接続先ホスト、ポート、およびプロキシ認証情報を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const char   *proxy_auth;      /* Proxy authorization */
    const char   *target_host;     /* Target/origin hostname */
    apr_port_t   target_port;      /* Target/origin port */
} remote_connect_info;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。各フィールドは const char * または apr_port_t 型であり、接続先情報の読み取りに使用されます。

前提条件と不変条件

  • target_host は接続先のホスト名または IP アドレスを指す必要があります。
  • target_port は有効なポート番号である必要があります。

アルゴリズムまたは振る舞い

本構造体は、proxy_conn_rec 構造体の forward メンバを通じて保持され、バックエンド接続の確立や SSL ホスト名の特定時に参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • modules/proxy/proxy_util.c:3332: ローカル変数として初期化。
  • modules/proxy/proxy_util.c:3445: conn->forward をキャストして target_host を取得。
  • modules/proxy/proxy_util.c:3555: backend->forward をキャストして取得。
  • modules/proxy/proxy_util.c:3791: conn->forward をキャストして取得。
  • **呼び出し先**: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • proxy_conn_rec: 本構造体を forward メンバとして保持する接続管理構造体。

失敗モードと境界条件

  • forward メンバが NULL である場合、本構造体へのアクセスはセグメンテーション違反を引き起こす可能性があります。
  • target_host が未設定の場合、接続先解決に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は proxy_conn_rec の forward ポインタを介して利用されるため、型キャスト時の整合性に注意してください。
  • 接続のライフサイクル管理において、この構造体が指すメモリ領域が scpool 等の適切なプールで管理されていることを確認してください。
