util_ldap_url_node_copy

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache.c:48

目的

LDAP URL キャッシュノード（util_url_node_t）を、共有メモリ（apr_rmm）上の新しい領域へ複製します。

シグネチャまたは形状

void *util_ldap_url_node_copy(util_ald_cache_t *cache, void *c)

入力と出力

  • **入力**:
  • cache: キャッシュ管理構造体へのポインタ。
  • c: 複製対象の util_url_node_t 構造体へのポインタ。
  • **出力**:
  • 成功時: 共有メモリ上に確保・複製された新しいノードへのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • cache は有効なキャッシュ管理構造体である必要があります。
  • メモリ確保には標準の malloc ではなく、util_ald_alloc を使用する必要があります。
  • ノード内の文字列複製には util_ald_strdup を使用する必要があります。

アルゴリズムまたは振る舞い

1. util_ald_alloc を使用して、新しい util_url_node_t 用のメモリを確保します。 2. 確保に成功した場合、util_ald_strdup を使用して URL 文字列を複製します。 3. URL の複製に失敗した場合は、確保したノードを util_ald_free で解放し、NULL を返します。 4. URL の複製が成功した場合、元のノードから検索・比較・DN 比較の各キャッシュポインタをコピーし、新しいノードを返します。 5. 確保に失敗した場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap_cache.c:460（キャッシュ挿入処理の一部として利用）。
  • **呼び出し先**: util_ald_alloc, util_ald_strdup, util_ald_free（いずれも共有メモリ管理用ユーティリティ）。

関連データ構造

  • util_ald_cache_t: キャッシュ管理構造体。
  • util_url_node_t: URL キャッシュノード（url 文字列および各キャッシュへのポインタを保持）。

失敗モードと境界条件

  • 共有メモリの枯渇により util_ald_alloc が NULL を返した場合、本関数は NULL を返します。
  • URL 文字列の複製（util_ald_strdup）に失敗した場合、確保済みのノードを解放した上で NULL を返します。

保守担当者向けメモ

  • 本関数は共有メモリ上で動作するため、ポインタの直接的な代入は可能ですが、文字列データは必ず util_ald_strdup を経由して共有メモリ領域に配置する必要があります。
  • 構造体内のポインタ（search_cache 等）は、共有メモリ内のオフセットとして管理される前提で扱われています。
