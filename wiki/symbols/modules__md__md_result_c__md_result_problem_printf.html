md_result_problem_printf

定義

  • 種別: function
  • 定義箇所: modules/md/md_result.c:111

目的

md_result_t オブジェクトに対して、エラー状態、問題の種類、およびフォーマット指定された詳細メッセージを一度に設定し、状態変更イベントを通知します。

シグネチャまたは形状

void md_result_problem_printf(md_result_t *result, apr_status_t status,
                              const char *problem, const char *fmt, ...)

入力と出力

  • result: 更新対象の md_result_t 構造体へのポインタ。
  • status: 設定する apr_status_t 型のステータスコード。
  • problem: 問題の種類を示す文字列（内部でコピー・トリミングされます）。
  • fmt: 詳細メッセージを生成するための printf 形式のフォーマット文字列。
  • ...: フォーマット引数。
  • 出力: なし（result 構造体のフィールドを直接更新します）。

前提条件と不変条件

  • result は有効な md_result_t オブジェクトを指している必要があります。
  • result->p（apr_pool_t）は、文字列の割り当てに使用されます。
  • subproblems フィールドは本関数の呼び出しにより NULL にリセットされます。

アルゴリズムまたは振る舞い

1. result->status に引数の status を代入します。 2. problem 文字列を dup_trim 関数で複製・トリミングし、result->problem に格納します。 3. apr_pvsprintf を使用して、fmt と可変引数から詳細メッセージを生成し、result->detail に格納します。 4. result->subproblems を NULL に初期化します。 5. on_change(result) を呼び出し、登録されている状態変更コールバックを実行します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/md/mod_md.c:190
  • modules/md/mod_md.c:209
  • 呼び出し先:
  • dup_trim
  • apr_pvsprintf
  • on_change

関連データ構造

  • md_result_t: 処理結果を保持するメイン構造体。
  • apr_status_t: APRのステータスコード型。
  • apr_pool_t: メモリ管理用プール（result->p）。

失敗モードと境界条件

  • メモリ不足時: apr_pvsprintf 等の内部関数がメモリ割り当てに失敗した場合、APRの規約に従いアボートまたはエラーが発生する可能性があります。
  • problem が NULL の場合: dup_trim の実装に依存しますが、一般的にヌルポインタ参照の可能性があります。

保守担当者向けメモ

  • 本関数は subproblems を強制的に NULL にリセットするため、既存のサブ問題情報を保持したい場合には使用しないでください。
  • 状態変更時にコールバックが発火するため、呼び出し側は再入可能性（re-entrancy）を考慮する必要があります。
