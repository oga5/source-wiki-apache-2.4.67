vhost_alias_set

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_vhost_alias.c:120

目的

Apache HTTP Server の設定ディレクティブ（VirtualDocumentRoot, VirtualScriptAlias 等）を処理し、動的バーチャルホスト用のテンプレート文字列を検証および設定構造体に格納する。

シグネチャまたは形状

static const char *vhost_alias_set(cmd_parms *cmd, void *dummy, const char *map)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。cmd->info を介して呼び出し元のディレクティブを識別する。
  • map: 設定値として渡されたテンプレート文字列（または "none"）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • cmd->info に基づき、mva_sconf_t 構造体の適切なフィールド（doc_root または cgi_root）およびモード（VHOST_ALIAS_IP または VHOST_ALIAS_NAME）が更新される。
  • map が "none" でない場合、絶対パスである必要がある。
  • テンプレート文字列内の % 記法は、特定の構文（%、%p、%N、%N.M 等）に従う必要がある。

アルゴリズムまたは振る舞い

1. cmd->info を参照し、更新対象のサーバー設定項目を特定する。 2. map が絶対パスでない場合、"none" であれば設定を無効化（NULL）し、それ以外はエラーを返す。 3. map が絶対パスの場合、文字列を走査して % 記法を検証する。

  • %p または %% はそのまま許可。
  • % に続く数値（N）や、. に続く数値（M）の形式をチェックし、不正な場合はエラーを返す。

    • 4. 検証を通過した場合、設定構造体にテンプレート文字列とモードを保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定解析フェーズにおいて、以下のディレクティブから直接呼び出される。
  • VirtualScriptAlias
  • VirtualDocumentRoot
  • VirtualScriptAliasIP
  • VirtualDocumentRootIP
  • **呼び出し先**:
  • ap_get_module_config: 設定構造体の取得。
  • ap_os_is_path_absolute: パス形式の検証。
  • ap_cstr_casecmp: 文字列比較。
  • apr_isdigit: 数値判定。

関連データ構造

  • mva_sconf_t: サーバー単位の設定を保持する構造体。
  • mva_mode_e: モード（IPベースかホスト名ベースか）を定義する列挙型。
  • cmd_parms: Apache の設定コマンド用パラメータ構造体。

失敗モードと境界条件

  • **構文エラー**: テンプレート文字列が % 記法として不正な場合（例: % の後に数字がない）、エラーメッセージを返す。
  • **パス形式エラー**: 絶対パスではなく、かつ "none" でもない場合、エラーを返す。
  • **内部エラー**: cmd->info が予期しない値の場合、"INTERNAL ERROR" を返す。

保守担当者向けメモ

  • 本関数は、設定ファイル内のディレクティブと内部の mva_sconf_t 構造体を結びつける重要なブリッジです。
  • テンプレート文字列の構文チェックは手動のループで行われており、新しい % 記法を追加する場合はこのループのロジックを拡張する必要があります。
