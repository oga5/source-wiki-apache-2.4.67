set_scriptlog_buffer

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgi.c:153

目的

CGI スクリプトのデバッグログ出力時に使用されるバッファサイズ（バイト単位）を設定します。

シグネチャまたは形状

static const char *set_scriptlog_buffer(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: バッファサイズを指定する文字列（数値）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • cmd->server から取得したモジュール設定 cgi_server_conf が有効である必要があります。
  • arg は atoi で変換可能な数値文字列である必要があります。

アルゴリズムまたは振る舞い

1. cmd->server から現在のサーバー設定を取得します。 2. arg を atoi で整数に変換します。 3. 変換した値を cgi_server_conf 構造体の bufbytes メンバに格納します。 4. NULL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:181 (設定ディレクティブ ScriptLogBuffer の登録)
  • modules/generators/mod_cgid.c:1179 (設定ディレクティブ ScriptLogBuffer の登録)
  • **呼び出し先**:
  • ap_get_module_config (Apache API)
  • atoi (標準ライブラリ)

関連データ構造

  • cgi_server_conf: サーバー単位の CGI 設定を保持する構造体。bufbytes メンバを更新します。

失敗モードと境界条件

  • arg に数値以外の文字列が渡された場合、atoi の仕様に従い 0 が設定されます。
  • cgi_server_conf が取得できない場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は Apache の設定ディレクティブ ScriptLogBuffer を処理するためのコールバックです。
  • mod_cgi と mod_cgid の両方から参照されています。
