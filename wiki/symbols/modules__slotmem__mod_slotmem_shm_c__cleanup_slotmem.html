cleanup_slotmem

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:294

目的

共有メモリベースのスロットメモリインスタンスを破棄し、必要に応じて永続化データを保存するクリーンアップ処理を実行します。

シグネチャまたは形状

static apr_status_t cleanup_slotmem(void *param)

入力と出力

  • **入力**: void *param（使用されていません）
  • **出力**: apr_status_t（常に APR_SUCCESS を返します）

前提条件と不変条件

  • グローバル変数 globallistmem が管理する ap_slotmem_instance_t のリストが有効であること。
  • プロセス終了時に apr_pool_cleanup_register を介して呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. is_child_process() を呼び出し、現在のプロセスが子プロセスかどうかを判定します。 2. globallistmem を起点にリストを走査します。 3. 親プロセスであり、かつ AP_SLOTMEM_IS_PERSIST フラグが有効なインスタンスに対しては、store_slotmem() を呼び出して永続化ファイルへ書き出します。 4. apr_shm_destroy() で共有メモリセグメントを破棄し、apr_shm_remove() で共有メモリファイルを削除します。 5. 走査終了後、globallistmem を NULL にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/slotmem/mod_slotmem_shm.c:760（apr_pool_cleanup_register に登録）
  • **呼び出し先**:
  • is_child_process()
  • store_slotmem()
  • apr_shm_destroy()
  • apr_shm_remove()

関連データ構造

  • ap_slotmem_instance_t: 走査対象となる共有メモリ管理構造体。
  • globallistmem: 共有メモリインスタンスを保持するグローバルリスト。

失敗モードと境界条件

  • globallistmem が NULL の場合、ループは実行されず即座に終了します。
  • 子プロセスの場合、永続化処理（store_slotmem）はスキップされます。

保守担当者向けメモ

  • 本関数はプロセス終了時のクリーンアップとして登録されています。
  • globallistmem のリセットにより、二重解放を防ぐ設計となっています。
