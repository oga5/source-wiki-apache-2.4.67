create_auth_form_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:83

目的

ディレクトリ単位のフォームベース認証設定構造体（auth_form_config_rec）を生成し、デフォルト値で初期化する。

シグネチャまたは形状

static void *create_auth_form_dir_config(apr_pool_t *p, char *d)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体の割り当てに使用するメモリプール。
  • char *d: 設定が適用されるディレクトリパス。
  • **出力**:
  • void *: 初期化済みの auth_form_config_rec 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc によりメモリがゼロクリアされた状態で割り当てられる。
  • authoritative はデフォルトで 1（有効）に設定される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して auth_form_config_rec サイズのメモリを確保する。 2. 引数 d を構造体の dir メンバに格納する。 3. authoritative を 1 に設定する。 4. form_size を HUGE_STRING_LEN（8k）に設定する。 5. 以下のフォームフィールド名をデフォルト値で初期化する:

  • username: "httpd_username"
  • password: "httpd_password"
  • location: "httpd_location"
  • method: "httpd_method"
  • mimetype: "httpd_mimetype"
  • body: "httpd_body"

    • 6. 初期化された構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c:1326（Apache の設定読み込みプロセスにおいて、ディレクトリ設定作成ハンドラとして登録されている）。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ）。

関連データ構造

  • auth_form_config_rec: 本関数が初期化するディレクトリごとの認証設定を保持する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時は apr_pcalloc の挙動に従い、通常はプロセスが終了する（Apache のメモリ管理ポリシーに依存）。

保守担当者向けメモ

  • 本関数で設定されるデフォルトのフィールド名は、mod_auth_form が期待するフォームパラメータの標準値です。これらを変更する場合は、対応するディレクティブの解析ロジックと整合性を取る必要があります。
