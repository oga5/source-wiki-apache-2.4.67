netware_query

定義

  • 種別: function
  • 定義箇所: server/mpm/netware/mpm_netware.c:201

目的

NetWare MPM の構成情報や現在の状態を照会するためのクエリインターフェースを提供します。

シグネチャまたは形状

int netware_query(int query_code, int *result, apr_status_t *rv)

入力と出力

  • query_code: 照会する情報の種類を示す整数値（AP_MPMQ_* 定数）。
  • result: 照会結果を格納するポインタ。
  • rv: 処理の成功・失敗を示す apr_status_t 型のステータスコード。
  • 戻り値: 常に OK を返します。

前提条件と不変条件

  • rv は呼び出し元によって有効なメモリ領域として提供される必要があります。

アルゴリズムまたは振る舞い

1. rv を APR_SUCCESS で初期化します。 2. query_code に基づき switch 文で分岐し、対応する MPM の設定値や状態（スレッド制限、現在の状態、世代番号など）を result に書き込みます。 3. 未定義の query_code が渡された場合は、*rv に APR_ENOTIMPL を設定します。 4. 常に OK を返して終了します。

呼び出し元と呼び出し先

  • 呼び出し元: ap_hook_mpm_query を介して Apache の MPM クエリフックとして登録されています（server/mpm/netware/mpm_netware.c:1100）。
  • 呼び出し先: 直接的な関数呼び出しはありません。

関連データ構造

  • ap_threads_limit, ap_threads_min_free, ap_threads_max_free: スレッドプールの制限値。
  • mpm_state: サーバーの現在の動作状態。
  • ap_my_generation: サーバーの世代番号。
  • HARD_SERVER_LIMIT, HARD_THREAD_LIMIT: MPM のハード制限定数。

失敗モードと境界条件

  • 未サポートの query_code が渡された場合、*rv に APR_ENOTIMPL が設定されますが、関数自体は OK を返します。

保守担当者向けメモ

  • 本関数は Apache の MPM クエリフックとして機能するため、新しい MPM 関連の統計情報や設定値を追加する際は、この switch 文を拡張してください。
