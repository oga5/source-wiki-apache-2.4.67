ap_start_lingering_close

定義

  • 種別: function
  • 定義箇所: server/connection.c:121

目的

HTTP 接続終了時、TCP レベルでの正常な終了（FIN ハンドシェイク）を保証するために、残存データのフラッシュと書き込み側のシャットダウンを開始する。

シグネチャまたは形状

int ap_start_lingering_close(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（対象となる接続のレコード）
  • **出力**: int 型。成功または処理継続が必要な場合は 0、エラー発生時や即時終了が決定した場合は 1 を返す。

前提条件と不変条件

  • c に関連付けられたソケット（csd）が有効であること（ap_assert により検証）。
  • 呼び出し前に ap_prep_lingering_close が実行され、リンガリングの準備が整っていること。

アルゴリズムまたは振る舞い

1. ap_get_conn_socket を介して接続ソケットを取得する。 2. ap_prep_lingering_close(c) を呼び出し、準備が完了していれば 1 を返して終了する。 3. ap_flush_conn(c) を呼び出し、バッファ内に残っているデータをクライアントへ送信する。 4. NO_LINGCLOSE が定義されていない場合、apr_socket_shutdown(csd, APR_SHUTDOWN_WRITE) を実行し、クライアントに対して FIN パケットを送信して書き込み側を閉じる。 5. 接続が既に異常終了（c->aborted）している場合、またはシャットダウンに失敗した場合は 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/connection.c:165
  • server/mpm/event/event.c:1557
  • **呼び出し先**:
  • ap_get_conn_socket
  • ap_prep_lingering_close
  • ap_flush_conn
  • apr_socket_shutdown

関連データ構造

  • conn_rec: 接続状態を保持する構造体。aborted フラグが含まれる。
  • apr_socket_t: APR レベルのソケットハンドル。

失敗モードと境界条件

  • csd が NULL の場合、ap_assert によりプロセスが異常終了する。
  • apr_socket_shutdown が失敗した場合、その戻り値（非ゼロ）がそのまま関数の戻り値として伝播する。
  • c->aborted が真の場合、シャットダウン処理をスキップして 1 を返す。

保守担当者向けメモ

  • 本関数はリンガリング・クローズの「開始」フェーズを担当する。この後、実際にソケットを閉じるまでの待機処理は ap_lingering_close が担う。
  • NO_LINGCLOSE マクロが定義されている環境では、リンガリング処理がバイパスされる点に注意すること。
