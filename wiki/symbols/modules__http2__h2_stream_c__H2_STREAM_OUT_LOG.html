H2_STREAM_OUT_LOG

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_stream.c:187

目的

HTTP/2 ストリームの出力バッファの状態を、指定されたログレベルでデバッグ出力する。

シグネチャまたは形状

static void H2_STREAM_OUT_LOG(int lvl, h2_stream *s, const char *tag)

入力と出力

  • lvl: 出力に使用するログレベル（APLOG_TRACE2 等）。
  • s: 対象の h2_stream 構造体へのポインタ。
  • tag: ログメッセージに付与する識別タグ。
  • 出力: 戻り値なし。副作用として ap_log_cerror を介したログ出力を行う。

前提条件と不変条件

  • s および s->session が有効であること。
  • s->session->c1 が接続コンテキストとして初期化されていること。

アルゴリズムまたは振る舞い

1. APLOG_C_IS_LEVEL を使用し、現在の接続 c1 で指定された lvl が有効か確認する。 2. 有効な場合、h2_util_bb_print を呼び出し、ストリームの出力バッファ (s->out_buffer) の内容を最大 4KB のバッファに文字列として書き出す。 3. ap_log_cerror を使用し、H2_STRM_MSG マクロでストリーム情報を付与したログメッセージを出力する。バッファが空の場合は "empty" と表示する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/http2/h2_stream.c:1081 (処理開始前のトレース)
  • modules/http2/h2_stream.c:1117 (データ受信後のトレース)
  • 呼び出し先:
  • APLOG_C_IS_LEVEL
  • h2_util_bb_print
  • ap_log_cerror
  • H2_STRM_MSG

関連データ構造

  • h2_stream: ストリームの管理構造体。out_buffer フィールドを保持。
  • conn_rec: Apache HTTP Server の接続構造体。

失敗モードと境界条件

  • s->out_buffer が空の場合、ログには "empty" と記録される。
  • ログレベルが現在の設定より低い場合、関数は何も行わず即座に終了する。

保守担当者向けメモ

  • 本関数はデバッグ専用のユーティリティである。
  • h2_util_bb_print の出力バッファサイズ（4KB）は固定されているため、非常に大きなバッファ内容を出力する場合は切り詰められる可能性がある。
