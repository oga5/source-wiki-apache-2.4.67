hc_get_hcworker

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_hcheck.c:482

目的

指定されたプロキシワーカーに対応するヘルスチェック用ワーカー（hc）を取得、または存在しない場合は新規作成して返します。

シグネチャまたは形状

static proxy_worker *hc_get_hcworker(sctx_t *ctx, proxy_worker *worker, apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: サーバー単位のモジュール設定（sctx_t）。
  • worker: ヘルスチェック対象のプロキシワーカー。
  • p: メモリ割り当て用プール。
  • **出力**:
  • 対象ワーカーに関連付けられたヘルスチェック用 proxy_worker 構造体。

前提条件と不変条件

  • ctx->hcworkers ハッシュテーブルが初期化されている必要があります。
  • 返されるワーカーには PROXY_WORKER_IGNORE_ERRORS および PROXY_WORKER_GENERIC フラグが設定されます。

アルゴリズムまたは振る舞い

1. ctx->hcworkers ハッシュから、指定された worker に対応するヘルスチェック用ワーカーを検索します。 2. 存在しない場合:

  • ap_proxy_define_worker を使用して新規ワーカーを定義します。
  • 元のワーカーからホスト名、スキーム、タイムアウト設定、ポート番号等の属性をコピーします。
  • wctx_t コンテキストを初期化し、create_hcheck_req を呼び出してリクエストを構築します。
  • 新規作成したワーカーをハッシュに登録します。

    • 3. 存在する場合:

  • 現在のワーカー設定とヘルスチェック用ワーカーの設定を比較し、メソッドが変更されていれば create_hcheck_req を再実行して更新します。

    • 4. 最終的にヘルスチェック用ワーカーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_hcheck.c:641
  • **呼び出し先**:
  • apr_hash_get / apr_hash_set
  • ap_proxy_define_worker
  • ap_proxy_initialize_worker
  • create_hcheck_req
  • apr_uri_parse
  • ap_log_error

関連データ構造

  • sctx_t: サーバー設定コンテキスト。
  • proxy_worker: プロキシワーカー構造体。
  • wctx_t: ヘルスチェック用ワーカーのコンテキスト。

失敗モードと境界条件

  • apr_uri_parse が失敗した場合、パス情報は設定されませんが、処理は継続されます。
  • ワーカーのポート番号が未指定の場合、ap_proxy_port_of_scheme を用いてスキームからデフォルトポートを導出します。

保守担当者向けメモ

  • 本関数は、バランサーマネージャー等による動的な設定変更を考慮し、メソッドの不一致を検知して再構築を行うロジックを含んでいます。
  • hc->s->name および name_ex には、元のワーカーのアドレスが文字列として格納されます。
