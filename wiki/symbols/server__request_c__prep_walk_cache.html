prep_walk_cache

定義

  • 種別: function
  • 定義箇所: server/request.c:451

目的

リクエスト処理における設定マージ（Walk）結果を保持するキャッシュ構造体 walk_cache_t を取得または初期化する。サブリクエストや内部リダイレクト時に、親リクエストのキャッシュを継承・再利用することで計算コストを削減する。

シグネチャまたは形状

static walk_cache_t *prep_walk_cache(apr_size_t t, request_rec *r)

入力と出力

  • **入力**:
  • t: キャッシュの識別子（AP_NOTE_DIRECTORY_WALK 等）。
  • r: 現在のリクエスト構造体。
  • **出力**:
  • 処理対象となる walk_cache_t へのポインタ。

前提条件と不変条件

  • ap_get_request_note(r, t) が有効なメモリ領域を返すこと（AP_DEBUG_ASSERT で検証）。
  • 戻り値のキャッシュは、リクエストのプール r->pool に関連付けられる。

アルゴリズムまたは振る舞い

1. ap_get_request_note を使用して、現在のリクエストに関連付けられた既存のキャッシュを取得する。 2. サブリクエスト（r->main）または内部リダイレクト（r->prev）が存在する場合、それらのリクエストから対応するキャッシュの継承を試みる。 3. 継承可能なキャッシュが見つかった場合、その内容を r->pool に複製し、現在のリクエスト用に更新する。 4. 継承元がない場合は、新規に walk_cache_t を割り当て、walked 配列を初期化する。 5. 最終的に生成・取得したキャッシュを *note に格納し、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/request.c:708 (ap_directory_walk)
  • server/request.c:1471 (ap_location_walk)
  • server/request.c:1686 (ap_file_walk)
  • server/request.c:1878 (ap_if_walk)
  • **呼び出し先**:
  • ap_get_request_note
  • apr_pmemdup
  • apr_array_copy
  • apr_pcalloc
  • apr_array_make

関連データ構造

  • walk_cache_t: マージ結果を保持するキャッシュ構造体。
  • request_rec: リクエスト情報を保持する構造体。
  • walk_walked_t: walk_cache_t 内で保持される個別のマージ済み設定情報。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc 等の APR 関数が失敗した場合、プロセスが終了する可能性がある。
  • 継承元キャッシュが存在しない場合: 新規キャッシュが作成されるため、境界条件として適切に処理される。

保守担当者向けメモ

  • 本関数はリクエスト処理のパフォーマンスに直結する。キャッシュの継承ロジックを変更する際は、サブリクエストの階層構造と walk_cache_t->count の整合性に注意すること。
  • r->pool を使用しているため、リクエスト終了時に自動的に解放される。手動でのメモリ解放は不要。
