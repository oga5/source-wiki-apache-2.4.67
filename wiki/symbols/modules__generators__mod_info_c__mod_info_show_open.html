mod_info_show_open

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_info.c:195

目的

設定ツリーのディレクティブ開始タグ（HTML形式または標準出力）を整形して出力する。

シグネチャまたは形状

static void mod_info_show_open(request_rec * r, const ap_directive_t * dir, int nest)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体（NULLの場合、標準出力へ出力）。
  • dir: 出力対象のディレクティブ構造体。
  • nest: インデントレベルを示す整数値。
  • **出力**: なし（副作用としてストリームへの書き込みを行う）。

前提条件と不変条件

  • dir は有効な ap_directive_t 構造体を指している必要がある。
  • r が NULL の場合、グローバル変数 out が有効なファイルハンドルとして初期化されている必要がある。

アルゴリズムまたは振る舞い

1. mod_info_indent を呼び出し、指定されたネストレベルとファイル位置情報（dir->filename, dir->line_num）に基づいてインデントとメタデータを出力する。 2. r が存在する場合:

  • ap_escape_html を用いてディレクティブ名と引数をエスケープし、HTML形式でレスポンスストリームへ書き出す。

    • 3. r が NULL の場合:

  • apr_file_printf を用いて、エスケープ処理を行わずに標準出力（out）へ書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_info.c:244
  • **呼び出し先**:
  • mod_info_indent
  • ap_escape_html (r が存在する場合)
  • ap_rprintf (r が存在する場合)
  • apr_file_printf (r が NULL の場合)

関連データ構造

  • request_rec: リクエスト処理のコンテキスト。
  • ap_directive_t: 設定ツリーの各ノードを保持する構造体。

失敗モードと境界条件

  • r が NULL かつ out が初期化されていない場合、apr_file_printf の呼び出しにより実行時エラーが発生する可能性がある。
  • dir が NULL の場合、メンバアクセス時にセグメンテーションフォールトが発生する。

保守担当者向けメモ

  • 本関数は mod_info の表示ロジックにおいて、設定階層の開始部分を視覚化する役割を担う。
  • r の有無によって出力先を切り替える設計となっており、Webリクエスト経由の表示とコマンドラインからの設定ダンプの両方に対応している。
