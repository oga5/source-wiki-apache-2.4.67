ap_create_scoreboard

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:304

目的

Apache HTTP Server の実行時状態を管理するスコアボードの作成および初期化を行う。

シグネチャまたは形状

int ap_create_scoreboard(apr_pool_t *p, ap_scoreboard_e sb_type)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t ポインタ。
  • sb_type: スコアボードのタイプ（SB_SHARED 等）。
  • **出力**:
  • 成功時は OK を返す。
  • 失敗時は HTTP_INTERNAL_SERVER_ERROR を返す。

前提条件と不変条件

  • ap_scoreboard_image が既に存在する場合、既存のメモリ領域をゼロクリアし再初期化する。
  • ap_scoreboard_image が未作成の場合、ap_calc_scoreboard_size() で必要なサイズを算出した上でメモリを確保する。
  • 共有メモリを使用する場合、open_scoreboard(p) が成功する必要がある。

アルゴリズムまたは振る舞い

1. **既存チェック**: ap_scoreboard_image が存在すれば、restart_time を更新し、parent および servers 構造体をゼロクリアして ap_init_scoreboard(NULL) を呼び出す。 2. **新規作成**:

  • ap_calc_scoreboard_size() でサイズを計算。
  • sb_type が SB_SHARED の場合、open_scoreboard を経由して共有メモリを確保し、ゼロクリア後に ap_init_scoreboard を実行。
  • それ以外の場合は ap_calloc でヒープメモリを確保し、ap_init_scoreboard を実行。

    • 3. **初期化**: running_generation を 0 に設定し、restart_time を現在時刻に設定。 4. **クリーンアップ登録**: プロセス終了時に ap_cleanup_scoreboard が実行されるよう apr_pool_cleanup_register を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:5788 (ap_hook_pre_mpm を介して登録)
  • **呼び出し先**: ap_calc_scoreboard_size, open_scoreboard, ap_init_scoreboard, ap_calloc, apr_pool_cleanup_register, apr_time_now, memset

関連データ構造

  • ap_scoreboard_image (グローバル変数)
  • scoreboard_size (内部変数)
  • scoreboard_type (内部変数)
  • apr_shm_t (共有メモリハンドル)

失敗モードと境界条件

  • open_scoreboard(p) が失敗した場合、または共有メモリのベースアドレス取得に失敗した場合は HTTP_INTERNAL_SERVER_ERROR を返す。
  • ap_calloc によるメモリ確保失敗時の挙動は、ap_calloc の実装に依存する（通常はアボートまたは NULL 返却）。

保守担当者向けメモ

  • 本関数は MPM の初期化フェーズで呼び出される。
  • 共有メモリを使用する場合、apr_shm のライフサイクル管理と ap_cleanup_scoreboard の整合性に注意すること。
  • ap_scoreboard_image が既に存在する場合の再初期化処理は、プロセス再起動時の状態リセットを意図している。
