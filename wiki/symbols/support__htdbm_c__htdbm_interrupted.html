htdbm_interrupted

定義

  • 種別: function
  • 定義箇所: support/htdbm.c:98

目的

プログラム実行中に割り込み（SIGINT）が発生した際、データベース接続を安全に終了させ、エラーメッセージを出力してプロセスを終了させるためのシグナルハンドラです。

シグネチャまたは形状

static void htdbm_interrupted(void)

入力と出力

  • 入力: なし（シグナルハンドラとして呼び出される）
  • 出力: なし（プロセスを終了させるため、呼び出し元へは戻らない）

前提条件と不変条件

  • グローバル変数または静的変数 h が、有効な htdbm_t 構造体を指している必要があります。
  • htdbm_terminate が正常にリソースを解放できる状態である必要があります。

アルゴリズムまたは振る舞い

1. htdbm_terminate(h) を呼び出し、DBM データベース接続をクローズしてリソースを解放します。 2. 標準エラー出力（stderr）に "htdbm Interrupted !\n" というメッセージを出力します。 3. exit(ERR_INTERRUPTED) を実行し、割り込みを示す終了コードでプロセスを終了します。

呼び出し元と呼び出し先

  • 呼び出し元: support/htdbm.c 内の初期化処理において、apr_signal(SIGINT, (void (*)(int)) htdbm_interrupted); によりシグナルハンドラとして登録されています。
  • 呼び出し先: htdbm_terminate を呼び出します。

関連データ構造

  • htdbm_t: htdbm_terminate に渡されるデータベース接続コンテキスト。

失敗モードと境界条件

  • h が初期化されていない、または既に解放されている場合、htdbm_terminate 内で未定義の動作を引き起こす可能性があります。
  • stderr への書き込みが失敗した場合、メッセージは表示されませんが、プロセスは exit により終了します。

保守担当者向けメモ

  • 本関数はシグナルハンドラとして登録されるため、非同期シグナル安全（async-signal-safe）な関数のみを使用するように注意してください。現在の実装では fprintf を使用していますが、これは厳密には非同期シグナル安全ではない可能性があるため、将来的な改善の余地があります。
