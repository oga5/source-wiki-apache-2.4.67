encrypt_string

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:246

目的

セッション文字列を暗号化し、改ざん検知用の MAC を付与した上で Base64 エンコードされた文字列として出力する。

シグネチャまたは形状

apr_status_t encrypt_string(request_rec *r, const apr_crypto_t *f,
                            session_crypto_dir_conf *dconf, const char *in, char **out)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • f: apr_crypto コンテキスト。
  • dconf: 暗号化設定（パスフレーズ等を含む）。
  • in: 暗号化対象の文字列。
  • **出力**:
  • out: Base64 エンコードされた暗号化済み文字列（成功時）。
  • 戻り値: apr_status_t 型のステータスコード。

前提条件と不変条件

  • dconf->passphrases に少なくとも1つのパスフレーズが格納されていること。
  • r->pool が有効であり、メモリ割り当てに使用可能であること。
  • APU_HAVE_CRYPTO が有効であること。

アルゴリズムまたは振る舞い

1. apr_uuid_get でソルト（UUID）を生成する。 2. crypt_init を呼び出し、暗号アルゴリズムを特定する。 3. dconf 内の最初のパスフレーズを使用して apr_crypto_passphrase で鍵を生成する。 4. apr_crypto_block_encrypt_init で暗号化コンテキストを初期化する。 5. apr_crypto_block_encrypt および finish を用いて入力文字列を暗号化する。 6. [MAC(16バイト)][Salt(16バイト)][IV][Ciphertext] の形式でデータを連結する。 7. compute_auth を呼び出し、連結データに対して SipHash による MAC を生成・付与する。 8. apr_base64_encode を用いて最終的な文字列を生成し、out に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_crypto.c:494 (セッションエンコード処理内)
  • **呼び出し先**: crypt_init, apr_uuid_get, apr_crypto_passphrase, apr_crypto_block_encrypt_init, apr_crypto_block_encrypt, apr_crypto_block_encrypt_finish, compute_auth, apr_base64_encode, apr_palloc

関連データ構造

  • session_crypto_dir_conf: 暗号化設定を保持する構造体。
  • request_rec: リクエストごとのメモリプール (r->pool) を提供。
  • apr_crypto_t: 暗号化ライブラリのコンテキスト。

失敗モードと境界条件

  • パスフレーズからの鍵生成失敗 (APR_STATUS_IS_ENOKEY)。
  • 指定された暗号アルゴリズムのサポート外 (APR_STATUS_IS_EKEYTYPE)。
  • パディング設定の不整合 (APR_STATUS_IS_EPADDING)。
  • 暗号化処理 (encrypt_init, encrypt, finish) のいずれかが失敗した場合、エラーログを出力して即座に終了する。

保守担当者向けメモ

  • 本関数は dconf->passphrases のインデックス 0 のパスフレーズのみを使用する。
  • メモリ割り当てはすべて r->pool に依存しており、リクエスト終了時に自動的に解放される。
  • MAC の計算には compute_auth を使用しており、タイミング攻撃耐性のため ap_memeq_timingsafe が内部的に利用される。
