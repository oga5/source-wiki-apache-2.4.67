ap_scan_script_header_err_core

定義

  • 種別: function
  • 定義箇所: server/util_script.c:757

目的

スクリプト（CGI等）の出力を解析し、HTTPヘッダーとして妥当性を検証して request_rec に反映させるためのラッパー関数です。

シグネチャまたは形状

int ap_scan_script_header_err_core(request_rec *r, char *buffer,
                                   int (*getsfunc) (char *, int, void *),
                                   void *getsfunc_data)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • buffer: ヘッダー行を読み込むための作業用バッファ。
  • getsfunc: スクリプト出力を1行ずつ読み取るためのコールバック関数。
  • getsfunc_data: getsfunc に渡されるコンテキストデータ。
  • **出力**:
  • 成功時または特定のステータスコード（OK 等）を返します。失敗時はエラーコードを返します。

前提条件と不変条件

  • r->pool を使用してメモリ管理が行われます。
  • getsfunc はスクリプトの出力が完了するまでブロックする可能性があります。

アルゴリズムまたは振る舞い

内部で ap_scan_script_header_err_core_ex を呼び出し、APLOG_MODULE_INDEX を渡すことで、ログ記録のコンテキストを維持したままヘッダー解析のコアエンジンを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: include/util_script.h で宣言されており、外部モジュールから利用可能です。mod_cern_meta.c に類似の処理が存在することが指摘されています。
  • **呼び出し先**: ap_scan_script_header_err_core_ex を直接呼び出します。

関連データ構造

  • request_rec: リクエストの状態を保持する主要構造体。
  • getsfunc: 入力抽象化のための関数ポインタ型。

失敗モードと境界条件

  • スクリプト出力が不正な形式である場合、またはタイムアウトが発生した場合はエラーを返します。
  • EBCDIC 環境下では、文字コード変換ロジックが解析に介入します。

保守担当者向けメモ

  • 本関数は ap_scan_script_header_err_core_ex への単なるパススルーです。ロジックの変更が必要な場合は、コアエンジンである _ex 版を確認してください。
  • 呼び出し元情報の詳細は、現在のソースコード解析範囲では限定的です。
