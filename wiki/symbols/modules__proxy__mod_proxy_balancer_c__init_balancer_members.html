init_balancer_members

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:106

目的

ロードバランサに登録された各ワーカーの初期化状態を確認し、未初期化のワーカーに対して初期化処理を実行する。また、バランサの最大試行回数（max_attempts）のデフォルト値を設定する。

シグネチャまたは形状

static void init_balancer_members(apr_pool_t *p, server_rec *s, proxy_balancer *balancer)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • s: サーバー設定を保持する server_rec。
  • balancer: 初期化対象の proxy_balancer 構造体。
  • **出力**: なし（副作用としてワーカーの初期化およびバランサ設定の更新を行う）。

前提条件と不変条件

  • balancer->workers が有効な配列として存在していること。
  • PROXY_WORKER_IS_INITIALIZED マクロにより、ワーカーの初期化状態が判定可能であること。

アルゴリズムまたは振る舞い

1. balancer->workers 配列を反復処理する。 2. 各ワーカーに対して PROXY_WORKER_IS_INITIALIZED を実行し、未初期化であれば ap_proxy_initialize_worker を呼び出して初期化する。 3. 全ワーカーの確認後、balancer->s->max_attempts_set が未設定かつワーカー数が1より多い場合、max_attempts を「ワーカー数 - 1」に設定し、フラグを立てる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_balancer.c:2043
  • **呼び出し先**:
  • ap_proxy_worker_name
  • PROXY_WORKER_IS_INITIALIZED (マクロ)
  • ap_proxy_initialize_worker
  • ap_log_error

関連データ構造

  • proxy_balancer: ロードバランサの構成情報を保持。
  • proxy_worker: 個別のバックエンドワーカー情報。

失敗モードと境界条件

  • ワーカー数が0または1の場合、max_attempts の自動設定はスキップされる。
  • ap_proxy_initialize_worker 内部でのエラー発生時の挙動については、本関数のソース範囲内では定義されていない。

保守担当者向けメモ

  • 本関数はバランサの構成メンバーを走査する中心的な役割を果たす。
  • max_attempts のロジックは、フェイルオーバー時の試行回数を制御する重要なパラメータであるため、変更時は注意が必要。
