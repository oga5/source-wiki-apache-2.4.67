socache_mc_svr_cfg

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_memcache.c:53

目的

Memcached バックエンドを使用する socache プロバイダーにおいて、サーバー単位の設定情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_uint32_t ttl;
} socache_mc_svr_cfg;

入力と出力

  • **入力**: ttl (接続の有効期限を保持する apr_uint32_t 型のフィールド)
  • **出力**: なし（本構造体は設定値のコンテナとして機能します）

前提条件と不変条件

  • apr_pcalloc 等によりメモリが割り当てられ、初期化されることが前提です。
  • サーバーごとの設定として module_config に格納されます。

アルゴリズムまたは振る舞い

  • 本構造体は、MemcacheConnTTL ディレクティブによって設定される値を保持します。
  • socache_mc_set_ttl 関数などを通じて、この構造体内の ttl 値が管理・更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_server_config: サーバー設定の初期化時に本構造体を割り当てます。
  • ap_get_module_config: サーバー設定を取得する際に参照されます。
  • **呼び出し先**:
  • 情報なし（本構造体はデータ保持用であり、関数を呼び出す側ではありません）。

関連データ構造

  • apr_memcache: 本モジュールがラップする Memcached クライアントライブラリのメイン構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（apr_pcalloc 失敗時）の挙動は、呼び出し元のエラーハンドリングに依存します。

保守担当者向けメモ

  • 現在の定義は ttl のみですが、将来的にサーバー単位の追加設定が必要になった場合は本構造体を拡張してください。
