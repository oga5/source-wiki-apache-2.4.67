configure_checkhandler

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_dir.c:91

目的

Apache 設定ディレクティブ DirectoryCheckHandler の値を解析し、ディレクトリごとの設定構造体 dir_config_rec に反映させる。

シグネチャまたは形状

static const char *configure_checkhandler(cmd_parms *cmd, void *d_, int arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • d_: 現在のディレクトリ設定構造体 (dir_config_rec)。
  • arg: DirectoryCheckHandler に指定されたフラグ値（真偽値）。
  • **出力**:
  • 成功時は NULL を返す。

前提条件と不変条件

  • d_ は dir_config_rec 型としてキャスト可能であること。
  • arg が真（非ゼロ）の場合、checkhandler フィールドは MODDIR_ON に設定される。
  • arg が偽（ゼロ）の場合、checkhandler フィールドは MODDIR_OFF に設定される。

アルゴリズムまたは振る舞い

1. 引数 d_ を dir_config_rec 構造体ポインタにキャストする。 2. arg の値に基づき、構造体内の checkhandler メンバを MODDIR_ON または MODDIR_OFF に更新する。 3. 処理完了後、エラーメッセージなしとして NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_dir.c:135 において、AP_INIT_FLAG マクロを通じて DirectoryCheckHandler ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • dir_config_rec: ディレクトリごとの設定を保持する構造体。checkhandler メンバを更新対象とする。
  • cmd_parms: Apache の設定解析用パラメータ構造体。

失敗モードと境界条件

  • 本関数自体は単純な代入処理であるため、入力が適切であれば失敗しない。

保守担当者向けメモ

  • 本関数は DirectoryCheckHandler ディレクティブ専用のハンドラである。設定の変更が必要な場合は、dir_config_rec 構造体の定義と整合性を保つこと。
