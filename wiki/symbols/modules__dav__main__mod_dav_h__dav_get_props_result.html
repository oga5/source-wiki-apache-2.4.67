dav_get_props_result

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:490

目的

WebDAV の PROPFIND メソッド等でリソースのプロパティを取得した際、その結果（プロパティの統計情報および名前空間宣言）を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
    apr_text * propstats;       /* <propstat> element text */
    apr_text * xmlns;           /* namespace decls for <response> elem */
} dav_get_props_result;

入力と出力

  • **入力**: プロパティ取得関数（dav_get_props, dav_get_allprops など）の戻り値として生成されます。
  • **出力**: propstats には <propstat> 要素のテキスト内容が、xmlns には <response> 要素に必要な名前空間宣言が格納されます。

前提条件と不変条件

  • apr_text 型を使用しているため、関連付けられた apr_pool_t の生存期間に依存します。
  • 構造体内の各フィールドは、XML 生成時に適切な順序で連結されることを想定しています。

アルゴリズムまたは振る舞い

  • プロパティ取得処理において、取得成功したプロパティの XML 断片を propstats に蓄積し、必要な名前空間定義を xmlns に収集します。
  • 呼び出し元は、この構造体を受け取り、最終的なマルチステータスレスポンスの構築に使用します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c 内のメソッドハンドラ（dav_get_allprops の呼び出しなど）。
  • **呼び出し先**: dav_get_props や dav_get_allprops などの関数が本構造体を返却します。
  • 詳細は modules/dav/main/props.c:595 および modules/dav/main/mod_dav.h:1717 を参照してください。

関連データ構造

  • apr_text: APR が提供するテキスト連結用リスト構造体。
  • dav_propdb: プロパティデータベースのコンテキスト。

失敗モードと境界条件

  • プロパティが存在しない場合や取得に失敗した場合、各フィールドは空（NULL または初期化状態）となります。
  • メモリ不足時には apr_pool_t 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dav のプロパティ取得 API の標準的な戻り値です。変更を加える際は、mod_dav.c および props.c 内のすべてのプロパティ取得ロジックとの整合性を確認してください。
