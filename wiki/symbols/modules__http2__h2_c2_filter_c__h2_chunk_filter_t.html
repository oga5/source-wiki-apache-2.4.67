h2_chunk_filter_t

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_c2_filter.c:783

目的

HTTP/2 のセカンダリコネクション（C2）において、HTTP/1.1 形式のチャンク化転送エンコーディングをシミュレートするための状態コンテキストを保持します。

シグネチャまたは形状

struct h2_chunk_filter_t {
    const char *id;
    int eos_chunk_added;
    apr_bucket_brigade *bbchunk;
    apr_off_t chunked_total;
};

入力と出力

  • **入力**: フィルタ関数（f->ctx）を通じて、チャンク化処理の現在の状態として参照されます。
  • **出力**: チャンク化されたデータブリゲード（bbchunk）の管理および、転送済み合計バイト数（chunked_total）の追跡に使用されます。

前提条件と不変条件

  • f->ctx を介してフィルタスタックのコンテキストとして割り当てられている必要があります。
  • bbchunk は apr_bucket_brigade として初期化されている必要があります。

アルゴリズムまたは振る舞い

  • HTTP/1.1 フィルタが期待するチャンク形式を生成・管理します。
  • make_chunk 関数において、ブリゲード内のデータをチャンク形式に変換する際の作業領域として使用されます。
  • eos_chunk_added フラグにより、ストリーム終了（EOS）時のチャンク処理が完了しているかを追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c2_filter.c 内のフィルタ処理関数（f->ctx として参照）。
  • **呼び出し先**:
  • make_chunk 関数（引数として渡され、状態更新に使用される）。

関連データ構造

  • apr_bucket_brigade: チャンク化対象のデータ保持に使用。
  • h2_c2_filter 関連のフィルタスタックコンテキスト。

失敗モードと境界条件

  • メモリ割り当て失敗時（apr_bucket_brigade の生成など）の挙動は、呼び出し元のフィルタ実装に依存します。
  • チャンク化の合計サイズ（chunked_total）が apr_off_t の範囲を超える場合の挙動については、ソースコード内に明示的な記述はありません。

保守担当者向けメモ

  • 本構造体は h2_c2_filter.c 内のチャンク化フィルタ専用のコンテキストです。
  • id フィールドはデバッグやログ出力時の識別子として利用されます。
  • フィルタのライフサイクル管理とメモリプール（f->c->pool 等）の整合性に注意してください。
