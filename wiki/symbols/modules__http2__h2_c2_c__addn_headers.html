addn_headers

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2.c:420

目的

apr_table_do のコールバックとして機能し、ソーステーブルから宛先テーブルへヘッダーフィールドをコピーする。

シグネチャまたは形状

static int addn_headers(void *udata, const char *name, const char *value)

入力と出力

  • udata: 宛先となる apr_table_t * へのポインタ。
  • name: コピー対象のヘッダー名。
  • value: コピー対象のヘッダー値。
  • 戻り値: 常に 1 を返し、apr_table_do による反復処理の継続を指示する。

前提条件と不変条件

  • udata は有効な apr_table_t オブジェクトを指している必要がある。
  • apr_table_addn を使用するため、name および value のメモリ寿命は宛先テーブルのライフサイクルと整合している必要がある。

アルゴリズムまたは振る舞い

1. udata を apr_table_t * 型にキャストする。 2. apr_table_addn を呼び出し、name と value を宛先テーブルに追加する。 3. 1 を返して処理を終了する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_c2.c:450 にて apr_table_do の引数として呼び出されている。
  • 呼び出し先: apr_table_addn (APR API)。

関連データ構造

  • apr_table_t: Apache Portable Runtime のキー・値ペアを保持するテーブル構造体。

失敗モードと境界条件

  • udata が NULL の場合、apr_table_addn 内部でセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は apr_table_do の反復処理専用であるため、外部から直接呼び出すことは想定されていない。
  • apr_table_addn を使用しているため、値の重複チェックは行われない。
