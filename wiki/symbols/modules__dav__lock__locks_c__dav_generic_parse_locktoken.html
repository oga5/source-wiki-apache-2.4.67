dav_generic_parse_locktoken

定義

  • 種別: function
  • 定義箇所: modules/dav/lock/locks.c:263

目的

WebDAV の opaquelocktoken 形式の文字列を解析し、内部的な dav_locktoken 構造体に変換します。

シグネチャまたは形状

dav_error * dav_generic_parse_locktoken(apr_pool_t *p,
                                        const char *char_token,
                                        dav_locktoken **locktoken_p)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する APR プール。
  • char_token: 解析対象のロックトークン文字列。
  • **出力**:
  • locktoken_p: 解析結果の dav_locktoken 構造体へのポインタ。
  • 戻り値: 成功時は NULL、失敗時はエラー詳細を含む dav_error 構造体。

前提条件と不変条件

  • char_token は "opaquelocktoken:" で始まる必要があります。
  • char_token の後半部分は、apr_uuid_parse が解釈可能な有効な UUID 形式である必要があります。

アルゴリズムまたは振る舞い

1. char_token が "opaquelocktoken:" で始まるかを確認し、一致しない場合は HTTP_BAD_REQUEST エラーを返します。 2. 文字列ポインタを 16 文字分（"opaquelocktoken:" の長さ）進めます。 3. apr_pcalloc を使用して dav_locktoken 構造体をプールから割り当てます。 4. apr_uuid_parse を呼び出し、残りの文字列を UUID として解析します。 5. 解析に失敗した場合は HTTP_BAD_REQUEST エラーを返し、成功した場合は locktoken_p に結果を格納して NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/lock/locks.c 内のコード（行 259 および 1221 付近）から参照されています。
  • **呼び出し先**: apr_pcalloc および apr_uuid_parse（APR ライブラリ関数）、dav_new_error（mod_dav ユーティリティ）。

関連データ構造

  • dav_locktoken: ロックを一意に識別するための UUID を保持する構造体。

失敗モードと境界条件

  • **フォーマット不正**: "opaquelocktoken:" で始まらない場合、DAV_ERR_LOCK_UNK_STATE_TOKEN エラーを返します。
  • **UUID 解析失敗**: UUID 部分が不正な形式である場合、DAV_ERR_LOCK_PARSE_TOKEN エラーを返します。

保守担当者向けメモ

  • 本関数は opaquelocktoken 形式のみをサポートしています。他のトークン形式をサポートする必要がある場合は、先頭文字列のチェックロジックを拡張する必要があります。
