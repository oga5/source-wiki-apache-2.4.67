rotate_status

定義

  • 種別: struct
  • 定義箇所: support/rotatelogs.c:94

目的

rotatelogs ユーティリティにおける現在のログローテーション実行状態を保持する。

シグネチャまたは形状

struct rotate_status {
    struct logfile current;      /* 現在のログファイル */
    apr_pool_t *pool;            /* トップレベルのメモリプール */
    int rotateReason;            /* ローテーション発生理由 */
    adjusted_time_t tLogEnd;     /* ローテーション終了時刻 */
    int nMessCount;              /* メッセージカウント */
    int fileNum;                 /* ファイル番号 */
};

入力と出力

本構造体は関数引数として渡され、ローテーション処理の前後で current（ファイルハンドル等）や fileNum が更新される。

前提条件と不変条件

  • pool は APR のメモリ管理に使用される。
  • current メンバは doRotate() 関数内で新しいファイルハンドルに入れ替えられる。

アルゴリズムまたは振る舞い

  • ログローテーションの判定（checkRotate）および実行（doRotate）の過程で、現在のファイル状態を追跡する。
  • ローテーション発生時、rotateReason に理由が記録され、current が更新される。

呼び出し元と呼び出し先

  • 呼び出し元: support/rotatelogs.c 内のメインループおよびローテーション判定関数。
  • 呼び出し先: rotate_status_t として typedef されており、support/rotatelogs.c:78 で定義されている。

関連データ構造

  • logfile: current メンバとして保持され、実際のファイルハンドルを管理する。
  • rotate_config_t: 実行時の設定値を保持する対となる構造体。
  • apr_pool_t: メモリ管理に使用される。

失敗モードと境界条件

  • 書き込みエラー発生時、truncate_and_write_error() によりファイルが切り詰められ、エラー情報が記録される。
  • -n オプション（ファイル番号によるローテーション）使用時は、strftime 形式のファイル名と併用できない制約がある。

保守担当者向けメモ

  • logfile 構造体は doRotate 内で動的に入れ替わるため、ポインタの整合性に注意すること。
  • メモリ管理は APR プールで行われており、ローテーション時に古いプールが破棄される設計となっている。
