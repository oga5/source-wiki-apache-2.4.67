merge_proxyws_dir_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_wstunnel.c:455

目的

ディレクトリ単位の WebSocket プロキシ設定（proxyws_dir_conf）をマージし、親コンテキストから子コンテキストへの設定継承を管理します。

シグネチャまたは形状

static void *merge_proxyws_dir_config(apr_pool_t *p, void *vbase, void *vadd)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • vbase: 親ディレクトリの設定（proxyws_dir_conf）。
  • vadd: 現在のディレクトリで追加された設定（proxyws_dir_conf）。
  • **出力**:
  • マージされた新しい proxyws_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • vbase および vadd は proxyws_dir_conf 型としてキャスト可能である必要があります。
  • apr_pcalloc により、新しい設定構造体はゼロ初期化された状態で割り当てられます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい設定構造体 new をメモリ上に確保します。 2. fallback_to_proxy_http 設定について、vadd で明示的に設定されている場合（fallback_to_proxy_http_set が真）は vadd の値を採用し、そうでない場合は vbase の値を継承します。 3. fallback_to_proxy_http_set フラグについては、vadd または vbase のいずれかが設定されていれば真となります。 4. 構築した new を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_wstunnel.c:508（command_rec 構造体内の設定マージ関数として登録されています）。
  • **呼び出し先**: 直接的な関数呼び出しはソース内にありませんが、APR のメモリ管理関数 apr_pcalloc を利用しています。

関連データ構造

  • proxyws_dir_conf: WebSocket プロキシのディレクトリ設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、Apache HTTP Server の標準的なエラー処理（通常はプロセス終了）がトリガーされます。

保守担当者向けメモ

  • 本関数は mod_proxy_wstunnel の設定継承ロジックの要です。新しい設定項目を proxyws_dir_conf に追加した場合は、必ずこのマージ関数を更新し、値の継承と _set フラグの伝播を適切に処理してください。
