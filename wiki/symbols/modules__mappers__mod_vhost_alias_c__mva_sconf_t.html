mva_sconf_t

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_vhost_alias.c:65

目的

Apache HTTP Server における動的バーチャルホスト（Mass Virtual Hosting）の設定をサーバー単位で保持するための構造体です。

シグネチャまたは形状

typedef struct mva_sconf_t {
    const char *doc_root;
    const char *cgi_root;
    mva_mode_e doc_root_mode;
    mva_mode_e cgi_root_mode;
} mva_sconf_t;

入力と出力

  • 入力: VirtualDocumentRoot および VirtualScriptAlias ディレクティブにより設定されるテンプレート文字列と、それに対応するモード（mva_mode_e）。
  • 出力: サーバー設定として保持され、リクエスト処理時に mva_translate 等で参照されます。

前提条件と不変条件

  • doc_root および cgi_root は、テンプレート文字列として解釈されます。
  • doc_root_mode および cgi_root_mode は、ホスト名ベースか IP ベースかを決定する mva_mode_e 型の値をとります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、ロジックは持ちません。設定の初期化は mva_create_server_config、マージは mva_merge_server_config を通じて行われます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • mva_create_server_config (メモリ確保)
  • mva_merge_server_config (設定の継承)
  • vhost_alias_set (設定値の格納)
  • mva_translate (リクエスト処理時の設定参照)
  • 呼び出し先: なし（データ構造体のため）。

関連データ構造

  • mva_mode_e: doc_root_mode および cgi_root_mode の型として使用されます。

失敗モードと境界条件

  • メモリ確保失敗時: apr_pcalloc が失敗した場合、Apache の標準的なメモリ管理に従いプロセスが終了する可能性があります。
  • 設定値の不整合: テンプレート文字列が HUGE_STRING_LEN を超えるパスを生成する場合の挙動については、モジュール全体の制約に依存します。

保守担当者向けメモ

  • 本構造体はサーバー単位の設定を保持するものであり、.htaccess 等によるディレクトリ単位のオーバーライドはサポートされていません。
