md_store_remove

定義

  • 種別: function
  • 定義箇所: modules/md/md_store.c:86

目的

Managed Domain (MD) サブシステムにおいて、ストレージから特定のドメイン関連データ（証明書、秘密鍵、設定等）を削除するための抽象化インターフェースを提供します。

シグネチャまたは形状

apr_status_t md_store_remove(md_store_t *store, md_store_group_t group, 
                             const char *name, const char *aspect, 
                             apr_pool_t *p, int force);

入力と出力

  • store: ストレージ抽象化ハンドル。
  • group: 削除対象のストレージグループ（MD_SG_DOMAINS, MD_SG_ACCOUNTS 等）。
  • name: 削除対象の識別名。
  • aspect: 削除対象のデータ種別（ファイル名や属性を特定する文字列）。
  • p: メモリ割り当て用プール。
  • force: 強制削除フラグ（0以外で有効）。
  • **戻り値**: 操作結果を示す apr_status_t 型のステータスコード。

前提条件と不変条件

  • store は有効な md_store_t 構造体へのポインタであり、その内部の remove 関数ポインタが適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 引数として渡された store 構造体が保持する関数ポインタ store->remove を呼び出します。 2. 実際の削除ロジック（ファイルシステム上の削除や排他制御など）は、md_store_t の具体的な実装（md_store_fs.c 等）に委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_order.c:249
  • modules/md/md_reg.c:87, 564, 566
  • modules/md/md_store.c:238
  • **呼び出し先**:
  • store->remove（md_store_t 構造体内の関数ポインタ）

関連データ構造

  • md_store_t: ストレージ操作を抽象化する構造体。
  • md_store_group_t: ストレージグループを定義する列挙型。

失敗モードと境界条件

  • store が NULL の場合、または store->remove が未定義の場合、セグメンテーション違反が発生する可能性があります。
  • 削除対象が存在しない場合や、権限不足により削除できない場合、apr_status_t を通じてエラーが返されます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の削除処理は md_store_t の実装側に依存します。
  • 削除対象のファイル命名規則については md_store_fs.c 等の実装を確認してください。
