cmd_modem_standard

定義

  • 種別: function
  • 定義箇所: modules/test/mod_dialup.c:251

目的

Apache の設定ディレクティブ ModemStandard のコールバック関数として、指定されたモデム規格名に対応するバイトレートを検索し、ディレクトリ設定構造体 (dialup_dcfg_t) に適用する。

シグネチャまたは形状

const char *cmd_modem_standard(cmd_parms *cmd, void *dconf, const char *input)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dconf: 現在のディレクトリ設定構造体 (dialup_dcfg_t*)。
  • input: 設定ファイルで指定されたモデム規格名（文字列）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列（"mod_dialup: Unknown Modem Standard specified."）を返す。

前提条件と不変条件

  • dconf は dialup_dcfg_t 型としてキャスト可能であること。
  • modem_bitrates 配列が NULL 終端されていること。

アルゴリズムまたは振る舞い

1. dcfg->bytes_per_second を 0 に初期化する。 2. modem_bitrates テーブルを走査し、input と一致する規格名を検索する（大文字小文字を区別しない）。 3. 一致が見つかった場合、対応する bytes_per_second を dcfg に格納し、ループを終了する。 4. 検索終了後、dcfg->bytes_per_second が 0 のままであればエラーメッセージを返し、それ以外は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定解析プロセスにより、AP_INIT_TAKE1 マクロを介して ModemStandard ディレクティブが検出された際に呼び出される。
  • **呼び出し先**: strcasecmp（標準ライブラリ関数）。

関連データ構造

  • dialup_dcfg_t: 設定値を保持する構造体。
  • modem_speed_t: 規格名とバイトレートの対応を定義する構造体。

失敗モードと境界条件

  • 指定された規格名が modem_bitrates に存在しない場合、エラーメッセージを返し設定は適用されない。
  • input が NULL の場合の挙動については、ソースコード上での明示的なチェックは存在しない。

保守担当者向けメモ

  • 本関数は AP_INIT_TAKE1 を通じて登録されているため、設定ファイルで引数が正しく渡されることが前提となっている。
  • 規格を追加する場合は、modem_bitrates テーブルを更新するだけで本関数を変更する必要はない。
