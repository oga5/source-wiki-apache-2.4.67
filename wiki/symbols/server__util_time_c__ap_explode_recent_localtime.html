ap_explode_recent_localtime

定義

  • 種別: function
  • 定義箇所: server/util_time.c:146

目的

キャッシュを利用して、指定された時刻（apr_time_t）をローカルタイムの分解済み時刻構造体（apr_time_exp_t）へ高速に変換します。

シグネチャまたは形状

apr_status_t ap_explode_recent_localtime(apr_time_exp_t *tm, apr_time_t t)

入力と出力

  • **入力**:
  • tm: 分解結果を格納するための apr_time_exp_t 構造体へのポインタ。
  • t: 変換対象の時刻（apr_time_t）。
  • **出力**:
  • 戻り値: 処理の成功または失敗を示す apr_status_t 型のステータスコード。

前提条件と不変条件

  • AP_TIME_RECENT_THRESHOLD（デフォルト15秒）以内の時刻に対して最適化されています。
  • 内部で exploded_cache_localtime キャッシュを利用します。

アルゴリズムまたは振る舞い

1. 内部関数 cached_explode を呼び出します。 2. cached_explode は、exploded_cache_localtime キャッシュを参照し、現在の秒数に基づいたインデックスからキャッシュヒットを試みます。 3. キャッシュが有効（整合性が取れている）であれば、その値を tm にコピーして即座に返します。 4. キャッシュが無効または不整合な場合は、標準の apr_time_exp_lt を呼び出して時刻を分解し、その結果をキャッシュに書き戻してから返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/loggers/mod_log_config.c
  • server/log.c
  • server/util_time.c（内部参照）
  • **呼び出し先**:
  • cached_explode（内部関数）

関連データ構造

  • apr_time_exp_t: 分解された時刻情報を保持する APR 構造体。
  • exploded_time_cache_element: キャッシュ管理に使用される内部構造体。

失敗モードと境界条件

  • AP_TIME_RECENT_THRESHOLD を超える古い時刻が指定された場合、キャッシュは利用されず、標準の apr_time_exp_lt によるフォールバック処理が行われます。
  • マルチスレッド環境において、キャッシュ更新中の不整合が発生した場合は、再計算により整合性が保たれます。

保守担当者向けメモ

  • 本関数はロックフリーな設計です。キャッシュの整合性検証は cached_explode 内で行われるため、本関数自体に排他制御を実装する必要はありません。
  • ログ出力やヘッダー生成など、高頻度で呼び出される箇所でのパフォーマンス改善に寄与します。
