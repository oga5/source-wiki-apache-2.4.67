ssl_hook_ReadReq

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:132

目的

HTTP リクエスト処理の初期段階（ReadReq フェーズ）において、SSL 接続の検証、SNI ポリシーの適用、およびリクエストと SSL コンテキストの紐付けを行う。

シグネチャまたは形状

int ssl_hook_ReadReq(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在の HTTP リクエスト構造体）
  • **出力**: int （Apache のステータスコード。成功時は DECLINED、エラー時は HTTP_BAD_REQUEST、HTTP_FORBIDDEN、HTTP_SERVICE_UNAVAILABLE、HTTP_MISDIRECTED_REQUEST を返す）

前提条件と不変条件

  • sslconn（SSLConnRec）が接続またはマスター接続から取得可能であること。
  • sslconn->service_unavailable が設定されている場合、リクエストは拒否される。
  • sslconn->non_ssl_request が NON_SSL_SET_ERROR_MSG の場合、平文 HTTP を SSL ポートで受信したと判断し、エラーを返す。

アルゴリズムまたは振る舞い

1. **接続コンテキストの取得**: 現在のリクエストまたはマスター接続から SSLConnRec を特定する。 2. **状態チェック**: サービス利用不可状態や、平文 HTTP 誤送信のチェックを行い、該当すれば即座にエラーを返す。 3. **SNI 検証**: HAVE_TLSEXT が有効な場合、初期リクエストに対して以下を実行する。

  • SNI でホスト名が提供されているが HTTP リクエストにホスト名がない場合のチェック。
  • strict_sni_vhost_check が有効な場合の SNI ホスト名欠落チェック。
  • ssl_check_vhost_sni_policy による SSL 設定の互換性検証。

    • 4. **紐付け**: modssl_set_app_data2 を使用して、OpenSSL の SSL オブジェクトに現在の request_rec を登録する。 5. **環境設定**: ssl_configure_env を呼び出し、SSL 関連の環境変数を設定する。 6. **終了**: 処理を継続するため DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c の ap_hook_post_read_request に登録されている。
  • **呼び出し先**: ssl_check_vhost_sni_policy（SNI ポリシー検証）、ssl_configure_env（環境変数設定）、modssl_set_app_data2（OpenSSL へのデータ格納）。

関連データ構造

  • request_rec: HTTP リクエスト情報。
  • SSLSrvConfigRec: サーバー単位の SSL 設定。
  • SSLConnRec: 接続単位の SSL 状態管理構造体。

失敗モードと境界条件

  • **HTTP_SERVICE_UNAVAILABLE**: SSL プロパティが不完全な場合。
  • **HTTP_BAD_REQUEST**: 平文 HTTP を SSL ポートで受信した場合、または SNI ホスト名と HTTP ホスト名の不整合。
  • **HTTP_FORBIDDEN**: SNI が必須の環境で SNI が提供されなかった場合。
  • **HTTP_MISDIRECTED_REQUEST**: SNI ポリシーによる互換性チェックに失敗した場合。

保守担当者向けメモ

  • 本関数は post_read_request フックとして動作する。
  • HAVE_TLSEXT 定義の有無により SNI 関連のロジックが条件付きでコンパイルされる。
  • sslconn->non_ssl_request のリセット処理は、エラー発生後の状態遷移を制御する重要な箇所である。
