store_file_ev

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md.c:232

目的

Managed Domains (MD) のファイルストアにおいて、ディレクトリやファイルが作成・変更された際に、適切なアクセス権限を付与するためのイベントコールバック関数です。

シグネチャまたは形状

static apr_status_t store_file_ev(void *baton, struct md_store_t *store,
                                  md_store_fs_ev_t ev, unsigned int group,
                                  const char *fname, apr_filetype_e ftype,
                                  apr_pool_t *p);

入力と出力

  • **入力**:
  • baton: server_rec 構造体へのポインタ。
  • store: イベントが発生した md_store_t インスタンス。
  • ev: 発生したイベント種別 (md_store_fs_ev_t)。
  • group: ファイル/ディレクトリが属するグループ ID。
  • fname: 対象のファイルまたはディレクトリのパス。
  • ftype: ファイルタイプ (apr_filetype_e)。
  • p: メモリ割り当て用プール。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS を返します。

前提条件と不変条件

  • baton は有効な server_rec 構造体を指している必要があります。
  • ftype が APR_DIR である場合、特定のグループ（CHALLENGES, STAGING, OCSP）に対しては、ワーカープロセスがアクセス可能な権限設定が試行されます。

アルゴリズムまたは振る舞い

1. イベント発生を APLOG_TRACE3 レベルでログ出力します。 2. ftype がディレクトリ (APR_DIR) かつ、グループが MD_SG_CHALLENGES、MD_SG_STAGING、または MD_SG_OCSP のいずれかである場合、md_make_worker_accessible を呼び出してディレクトリの所有権やアクセス権を調整します。 3. md_make_worker_accessible が APR_ENOTIMPL 以外のステータスを返した場合は、その結果をそのまま返します。 4. 上記以外の場合は APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md.c:272 (直接呼び出し)
  • modules/md/mod_md.c:290 (md_store_fs_set_event_cb へのコールバック登録)
  • **呼び出し先**:
  • md_make_worker_accessible (条件合致時)
  • ap_log_error

関連データ構造

  • md_store_t: MD の永続化層を管理する構造体。
  • server_rec: Apache のサーバー設定構造体。
  • md_store_fs_ev_t: ファイルシステムイベントの列挙型。

失敗モードと境界条件

  • md_make_worker_accessible がエラーを返した場合、そのエラーコードが呼び出し元に伝播します。
  • ftype がディレクトリでない場合や、対象グループが権限調整対象外の場合は、何も変更を行わず APR_SUCCESS を返します。

保守担当者向けメモ

  • 本関数は md_store_fs のイベントコールバックとして登録されることを前提としています。
  • md_make_worker_accessible の実装状況に依存して動作が変化するため、プラットフォーム固有の権限管理の実装状況に注意してください。
