ap_lua_init

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_apr.c:88

目的

Lua VM 内に Apr.Table メタテーブルを登録し、Lua スクリプトから Apache の apr_table_t 構造体を透過的に操作するための環境を構築します。

シグネチャまたは形状

int ap_lua_init(lua_State *L, apr_pool_t *p)

入力と出力

  • **入力**:
  • L: Lua 仮想マシンの状態（lua_State）。
  • p: メモリ管理用の APR プール（apr_pool_t）。※本関数内では直接使用されていません。
  • **出力**:
  • 戻り値: 常に 0 を返します。

前提条件と不変条件

  • lua_table_methods が定義されている必要があります。
  • Lua バージョンに応じて、luaL_register または luaL_newlib が適切に選択される必要があります。

アルゴリズムまたは振る舞い

1. luaL_newmetatable を呼び出し、"Apr.Table" という名前のメタテーブルを作成します。 2. lua_table_methods を用いて、Lua のライブラリまたはレジストリにメソッドを登録します。 3. メタテーブルの __index フィールドに get メソッドを割り当てます。 4. メタテーブルの __newindex フィールドに set メソッドを割り当てます。 これにより、Lua スクリプト側で t["key"] = "val" のような構文によるテーブル操作が可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/mod_lua.c:118
  • **呼び出し先**:
  • luaL_newmetatable (Lua C API)
  • luaL_register または luaL_newlib (Lua C API)
  • lua_pushstring, lua_gettable, lua_settable (Lua C API)

関連データ構造

  • apr_table_t: Apache のテーブル構造体。
  • lua_table_methods: get および set メソッドを保持する Lua 用の関数テーブル。

失敗モードと境界条件

  • Lua VM のスタックが不足している場合、Lua C API 内部でエラーが発生する可能性があります。
  • LUA_VERSION_NUM の判定により、使用する API が切り替わります。

保守担当者向けメモ

  • 本関数は mod_lua の初期化プロセスの一部として実行されます。
  • __index および __newindex の設定において、スタックインデックス 1（メタテーブル）と 2（メソッドテーブル）を明示的に指定しているため、スタック操作の順序変更には注意が必要です。
