ajp_msg_append_string_ex

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_msg.c:331

目的

AJPメッセージバッファに文字列を追加します。オプションでEBCDICからASCIIへの文字コード変換を適用します。

シグネチャまたは形状

apr_status_t ajp_msg_append_string_ex(ajp_msg_t *msg, const char *value, int convert)

入力と出力

  • **入力**:
  • msg: 操作対象の ajp_msg_t 構造体へのポインタ。
  • value: 追加する文字列（NULLの場合は 0xFFFF が書き込まれる）。
  • convert: ゼロ以外の場合、ap_xlate_proto_to_ascii を用いて変換を実行。
  • **出力**:
  • APR_SUCCESS: 成功。
  • AJP_EOVERFLOW (内部的に ajp_log_overflow が返す): バッファ容量不足。

前提条件と不変条件

  • msg は有効な ajp_msg_t 構造体であること。
  • 文字列の長さ（len）と終端文字（\0）を含めたサイズが、msg->max_size を超えないこと。

アルゴリズムまたは振る舞い

1. value が NULL の場合、ajp_msg_append_uint16 で 0xFFFF を書き込み終了。 2. value の長さを取得し、バッファの残り容量を検証。不足時は ajp_log_overflow を呼び出しエラーを返す。 3. 文字列の長さを uint16 としてバッファに追加。 4. memcpy を使用して、文字列本体と終端文字 \0 をバッファの現在位置（msg->len）にコピー。 5. convert が真の場合、ap_xlate_proto_to_ascii を呼び出し、バッファ内の文字列を変換。 6. msg->len を更新し、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/ajp.h にて以下のマクロ経由で利用される。
  • ajp_msg_append_string(m, v) (convert=1)
  • ajp_msg_append_string_ascii(m, v) (convert=0)
  • **呼び出し先**:
  • ajp_msg_append_uint16
  • ajp_log_overflow
  • ap_xlate_proto_to_ascii (条件付き)

関連データ構造

  • ajp_msg_t: AJPメッセージを保持する構造体。

失敗モードと境界条件

  • **バッファオーバーフロー**: msg->len + len + 3 が msg->max_size を超える場合、書き込みを行わずエラーを返します。
  • **NULLポインタ**: value が NULL の場合、文字列の代わりに 0xFFFF を追加する仕様です。

保守担当者向けメモ

  • 本関数はEBCDIC環境での移植性を考慮しています。
  • memcpy 後に msg->len を更新する際、len + 1（終端文字分）を加算している点に注意してください。
