priv_cfg

定義

  • **種別**: struct
  • **定義箇所**: modules/arch/unix/mod_privileges.c:47

目的

Apache モジュール mod_privileges において、仮想ホスト（vhost）単位の権限設定および動作モードを保持するための構造体です。

シグネチャまたは形状

typedef struct {
    priv_set_t *priv;
    priv_set_t *child_priv;
    uid_t uid;
    gid_t gid;
    priv_mode mode;
} priv_cfg;

入力と出力

  • **入力**: ap_get_module_config を通じて、サーバー設定からリクエスト処理時に読み込まれます。
  • **出力**: 保持する UID, GID, 特権セットは、setppriv システムコール等の権限降格処理の引数として利用されます。

前提条件と不変条件

  • 本構造体は Solaris 等の特権 API (priv.h) に依存します。
  • 本モジュールは非スレッド環境専用であり、マルチスレッド MPM との併用は想定されていません。
  • priv および child_priv メンバは priv_allocset で確保されたメモリを指す必要があります。

アルゴリズムまたは振る舞い

  • サーバー起動時およびリクエスト処理時に、設定された UID/GID および特権セットをプロセスに適用します。
  • PRIV_SECURE モード時には、この構造体の設定に基づき fork された子プロセス内で権限の最小化が行われます。
  • リクエスト終了時には、登録されたクリーンアップ関数により権限の復元が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: privileges_merge_cfg, privileges_create_cfg, privileges_req, privileges_end_req, privileges_postconf 等のモジュール内関数が、設定の取得・マージ・適用時に参照します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • priv_set_t: Solaris の特権セットを保持する型。
  • priv_mode: 動作モードを定義する列挙型。
  • priv_dir_cfg: ディレクトリ単位の設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: apr_palloc 等によるメモリ確保に失敗する可能性があります。
  • 権限適用失敗: setppriv が失敗した場合、プロセスの権限降格が正しく行われないリスクがあります。

保守担当者向けメモ

  • priv および child_priv のメモリ管理は priv_cfg_cleanup や privileges_term で適切に行う必要があります。
  • BIG_SECURITY_HOLE マクロで囲まれた機能は非常に危険であるため、変更時は注意してください。
  • 構造体のメンバ追加時は、privileges_merge_cfg でのコピー処理が適切に行われるか確認してください。
