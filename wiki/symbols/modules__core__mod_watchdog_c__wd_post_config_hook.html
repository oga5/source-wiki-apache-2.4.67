wd_post_config_hook

定義

  • 種別: function
  • 定義箇所: modules/core/mod_watchdog.c:425

目的

Apache HTTP Server の設定完了フェーズにおいて、登録された Watchdog プロバイダを走査し、親プロセス用スレッドの生成および子プロセス用シングルトン・ミューテックスの初期化を行う。

シグネチャまたは形状

static int wd_post_config_hook(apr_pool_t *pconf, apr_pool_t *plog,
                               apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • s: サーバーレコード。
  • **出力**:
  • 成功時は OK を返す。
  • メモリ確保失敗時は APR_ENOMEM、スレッド生成やミューテックス作成失敗時は apr_status_t のエラーコードを返す。

前提条件と不変条件

  • AP_SQ_MS_CREATE_PRE_CONFIG 状態では実行されない。
  • wd_server_conf が pconf のユーザーデータとして管理されている必要がある（未存在時は新規作成される）。
  • AP_WATCHDOG_PGROUP プロバイダグループが利用可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: wd_server_conf を取得または初期化し、サーバー設定を保持する。 2. **親プロバイダの処理**: AP_WATCHDOG_PVERSION を持つプロバイダを検索し、アクティブなものに対して wd_startup を呼び出し、専用のワーカースレッドを生成する。 3. **子プロバイダの処理**: AP_WATCHDOG_CVERSION を持つプロバイダを検索する。 4. **シングルトン管理**: 子プロバイダが singleton モードの場合、ap_proc_mutex_create を用いてプロセス間排他制御用のミューテックスを作成する。 5. **状態更新**: 処理したプロバイダの active フラグを管理し、生成されたスレッド数やミューテックスの状況をログに出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_config により登録され、サーバー起動時の設定フェーズで呼び出される。
  • **呼び出し先**:
  • ap_list_provider_names, ap_lookup_provider (プロバイダ検索)
  • ap_run_watchdog_need (Watchdog の有効性確認)
  • wd_startup (親スレッド生成)
  • ap_proc_mutex_create (子プロセス用ミューテックス生成)

関連データ構造

  • wd_server_conf_t: モジュール全体のサーバー設定を保持。
  • ap_watchdog_t: 監視インスタンスの定義。
  • ap_list_provider_names_t: プロバイダ名のリスト。

失敗モードと境界条件

  • apr_pcalloc 失敗時に APR_ENOMEM を返す。
  • wd_startup が失敗した場合、そのエラーコードを返して処理を中断する。
  • ap_proc_mutex_create が失敗した場合、そのエラーコードを返して処理を中断する。

保守担当者向けメモ

  • 本関数は post_config フックとして動作するため、ここでのエラーはサーバーの起動失敗に直結する。
  • singleton モードのミューテックス作成時に使用される wd_proc_mutex_type は、外部で定義されたグローバルな排他制御タイプに依存する。
