sed_reptr_s

定義

  • 種別: struct
  • 定義箇所: modules/filters/libsed.h:39

目的

sed ストリームエディタにおいて、コンパイルされた個々のコマンド（アドレス、正規表現、置換文字列など）を保持するノード構造体です。

シグネチャまたは形状

struct sed_reptr_s {
    sed_reptr_t *next;
    char        *ad1;
    char        *ad2;
    char        *re1;
    sed_reptr_t *lb1;
    char        *rhs;
    int         findex;
    char        command;
    int         gfl;
    char        pfl;
    char        negfl;
    int         nrep;
};

入力と出力

  • **入力**: コンパイルフェーズにおいて、スクリプトの各行が解析され、本構造体の各フィールドにコマンド情報が格納されます。
  • **出力**: 実行フェーズにおいて、sed_eval_t がこの構造体の連結リストを走査し、各コマンドを現在のパターンバッファに対して適用します。

前提条件と不変条件

  • next フィールドにより、コマンドの連結リストを形成します。
  • すべてのメモリ割り当ては、Apache の apr_pool_t に依存しています。
  • sed_reptr_t 型（sed_reptr_s の typedef）として参照されることが前提です。

アルゴリズムまたは振る舞い

1. **コンパイル**: sed_compile_string 等により、スクリプトが本構造体のリストへ変換されます。 2. **走査**: 実行時、sed_eval_buffer 等がリストを先頭から順に辿ります。 3. **適用**: command フィールドに基づき、ad1/ad2（アドレス範囲）や re1（正規表現）の条件判定を行い、一致した場合に rhs（置換文字列）等の処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/sed0.c（コンパイル処理）および modules/filters/sed1.c（実行処理）。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しは含みません。
  • **注記**: 具体的な関数レベルの呼び出し関係は、提供されたソース excerpt には含まれていません。

関連データ構造

  • sed_reptr_t: sed_reptr_s の typedef。
  • sed_commands_t: コンパイルされたスクリプト全体を管理する構造体。
  • sed_eval_t: 実行時の状態（パターンバッファ等）を保持する構造体。

失敗モードと境界条件

  • SED_DEPTH などの定数により、スクリプトの複雑さやネストに上限が設けられています。
  • メモリ不足時は apr_pool_t の挙動に従い、エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は libsed のコアであり、変更はコンパイル（sed0.c）と実行（sed1.c）の両方に影響を与えます。
  • 構造体内の各フィールド（gfl, pfl, negfl 等）は sed のフラグ管理に使用されるため、追加・削除の際は実行ロジックとの整合性に注意してください。
