hc_check_cping

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_hcheck.c:658

目的

AJPプロトコルを使用するバックエンドサーバーに対してCPING/CPONGリクエストを送信し、生存確認を行う。

シグネチャまたは形状

apr_status_t hc_check_cping(baton_t *baton, apr_thread_t *thread)

入力と出力

  • **入力**:
  • baton: ヘルスチェック実行に必要なコンテキスト（ワーカー情報、サーバー設定、一時プール等）。
  • thread: 現在の実行スレッド。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS、失敗時はエラーコード）。

前提条件と不変条件

  • ajp_handle_cping_cpong 関数が利用可能であること。
  • baton 内の ptemp（一時プール）が有効であること。

アルゴリズムまたは振る舞い

1. ajp_handle_cping_cpong の存在を確認し、未定義であれば APR_ENOTIMPL を返す。 2. hc_get_backend を呼び出し、対象ワーカーのバックエンド接続を取得する。 3. ap_proxy_connect_backend を呼び出し、バックエンドへの接続を確立する。 4. create_request_rec を用いて一時的なリクエストレコードを作成し、ap_proxy_connection_create_ex で接続を関連付ける。 5. タイムアウト値を決定する（ping_timeout > conn_timeout > timeout > ソケットデフォルトの順で優先）。 6. ajp_handle_cping_cpong を呼び出し、CPINGリクエストを送信して応答を待機する。 7. backend_cleanup を呼び出し、接続の解放とステータスコードの返却を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_hcheck.c:938
  • **呼び出し先**:
  • hc_get_backend
  • ap_proxy_connect_backend
  • create_request_rec
  • ap_proxy_connection_create_ex
  • set_request_connection
  • apr_socket_timeout_get
  • ajp_handle_cping_cpong
  • backend_cleanup

関連データ構造

  • baton_t: ヘルスチェックの実行状態を保持。
  • proxy_worker: チェック対象のワーカー。
  • proxy_conn_rec: バックエンドへの接続情報。
  • request_rec: AJP通信用の一時的なリクエスト情報。

失敗モードと境界条件

  • バックエンドの取得や接続確立に失敗した場合、backend_cleanup を経由してエラーを返す。
  • タイムアウト設定が未定義の場合、ソケットのデフォルトタイムアウト値が使用される。

保守担当者向けメモ

  • 本関数はAJPプロトコルに依存しているため、AJP関連の変更時には動作確認が必要。
  • backend_cleanup は成功・失敗に関わらず呼び出され、リソースの解放を保証する設計となっている。
