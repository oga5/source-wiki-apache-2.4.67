status_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/mod_md_status.c:151

目的

Managed Domains (MD) のステータス情報を出力する際、リクエスト処理全体で共有される一時的なコンテキストを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_pool_t *p;
    const md_mod_conf_t *mc;
    apr_bucket_brigade *bb;
    int flags;
    const char *prefix;
    const char *separator;
} status_ctx;

入力と出力

本構造体は、ステータス生成処理における入力（設定情報 mc）および出力（バケットブリゲード bb）の仲介役として機能します。

前提条件と不変条件

  • p (メモリプール) は、リクエスト処理のライフサイクルに紐付いており、リクエスト終了時に自動的に解放されます。
  • bb は、生成されたコンテンツを r->output_filters へ渡すための出力バッファとして使用されます。

アルゴリズムまたは振る舞い

ステータス出力処理において、以下の役割を担います。

  • **フォーマット制御**: flags を用いて HTML またはテキスト形式の出力を切り替えます。
  • **階層出力**: prefix および separator を使用して、JSON 構造を人間が読みやすい形式へ変換する際のインデントや区切り文字を管理します。
  • **データディスパッチ**: add_status_cell 等の関数に渡され、現在の出力状態を保持しながら情報を順次書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: md_http_cert_status や md_status_handler などのハンドラから初期化されます。
  • **呼び出し先**: add_json_val, si_val_status, print_date, print_time, si_val_valid_time, si_add_header, si_val_cert_valid_time, val_url_print, si_val_ca_urls 等の表示用ヘルパー関数に引数として渡されます。

関連データ構造

  • md_mod_conf_t: モジュールの設定情報を保持します。
  • apr_bucket_brigade: Apache の出力フィルタリング用データ構造です。
  • md_json_t: ステータス情報の元となるデータ構造です。

失敗モードと境界条件

  • メモリ不足時には apr_pool_t の操作によりエラーが発生する可能性があります。
  • bb が適切に初期化されていない状態で出力関数を呼び出した場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は mod_md_status.c 内の表示ロジックに特化しています。
  • HTML_STATUS(ctx) マクロは、この構造体の flags を参照して出力形式を判定するため、フラグの追加・変更時には注意してください。
