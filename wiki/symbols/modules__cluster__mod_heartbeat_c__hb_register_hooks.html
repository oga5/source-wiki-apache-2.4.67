hb_register_hooks

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartbeat.c:151

目的

mod_heartbeat モジュールのライフサイクルを管理するため、mod_watchdog が提供する各フックポイントへコールバック関数を登録する。

シグネチャまたは形状

static void hb_register_hooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p（Apache のリソースプール）
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • 本関数は Apache のモジュール初期化フェーズで呼び出される。
  • mod_watchdog がロードされている環境で実行される必要がある。

アルゴリズムまたは振る舞い

以下の mod_watchdog フックに対して、モジュール内の関数を APR_HOOK_MIDDLE の優先度で登録する。 1. ap_hook_watchdog_need: hb_watchdog_need を登録し、単一プロセスでの実行を要求する。 2. ap_hook_watchdog_init: hb_watchdog_init を登録し、初期化処理を定義する。 3. ap_hook_watchdog_step: hb_watchdog_step を登録し、定期実行されるメインループを定義する。 4. ap_hook_watchdog_exit: hb_watchdog_exit を登録し、終了処理を定義する。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュールロードプロセス（modules/cluster/mod_heartbeat.c:227 にてモジュール定義構造体から参照される）。
  • 呼び出し先: ap_hook_watchdog_need, ap_hook_watchdog_init, ap_hook_watchdog_step, ap_hook_watchdog_exit（すべて mod_watchdog API）。

関連データ構造

  • heartbeat_module（モジュール定義構造体）

失敗モードと境界条件

  • mod_watchdog が利用できない場合、フックの登録は失敗する可能性があるが、本関数内での明示的なエラーハンドリングは記述されていない。

保守担当者向けメモ

  • 本関数はモジュールのエントリーポイントの一部であり、フックの追加や削除を行う場合はここを修正する。
  • 登録される各コールバック関数（hb_watchdog_*）の詳細は、それぞれのシンボルページを参照のこと。
