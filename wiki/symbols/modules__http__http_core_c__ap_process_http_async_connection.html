ap_process_http_async_connection

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:135

目的

非同期 MPM（Multi-Processing Module）環境において、HTTP 接続のステートマシンを駆動し、リクエストの読み込みと処理を制御します。

シグネチャまたは形状

int ap_process_http_async_connection(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（接続レコード）。c->cs を介して現在の接続状態（conn_state_t）を参照します。
  • **出力**: int 型のステータスコード（常に OK を返します）。

前提条件と不変条件

  • c->cs が NULL であってはなりません（AP_DEBUG_ASSERT により検証）。
  • 呼び出し時点で cs->state は CONN_STATE_PROCESSING である必要があります。

アルゴリズムまたは振る舞い

1. **状態更新**: 接続状態を SERVER_BUSY_READ に更新し、必要に応じて接続カウントを記録します。 2. **リクエスト読み込み**: ap_read_request(c) を呼び出し、リクエストを取得します。 3. **リクエスト処理**:

  • リクエストが正常（HTTP_OK）であれば、状態を CONN_STATE_HANDLER に遷移させ、ap_process_async_request(r) を実行してリクエストを処理します。
  • 処理後、リクエストプールが破棄される可能性があるため、r を NULL に設定します。

    • 4. **終了判定**: 処理終了後、状態が CONN_STATE_WRITE_COMPLETION または CONN_STATE_SUSPENDED でない場合、接続を CONN_STATE_LINGER（終了待機）状態へ移行させます。 5. **エラーハンドリング**: ap_read_request が失敗した場合、接続を CONN_STATE_LINGER に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_process_http_connection（modules/http/http_core.c:246）。
  • **呼び出し先**:
  • ap_update_child_status_from_conn / ap_update_child_status
  • ap_set_conn_count
  • ap_read_request
  • ap_process_async_request

関連データ構造

  • conn_rec: 接続情報を保持する構造体。
  • conn_state_t: 非同期接続のステートマシン状態を管理する構造体。
  • request_rec: HTTP リクエスト情報を保持する構造体。

失敗モードと境界条件

  • ap_read_request が失敗した場合（クライアントの切断など）、接続は CONN_STATE_LINGER へ遷移し、処理を終了します。
  • ap_process_async_request 実行後のリクエストプール破棄を考慮し、ポインタの無効化（r = NULL）を行うことで、解放済みメモリへのアクセスを防止しています。

保守担当者向けメモ

  • 本関数は非同期 MPM 専用です。同期 MPM の場合は ap_process_http_sync_connection が使用されます。
  • cs->state の遷移は、MPM が接続のライフサイクルを管理するための重要なインターフェースです。状態遷移の追加や変更を行う際は、MPM 側のステートマシンとの整合性に注意してください。
