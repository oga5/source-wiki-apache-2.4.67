dav_lock

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1394

目的

WebDAV リソースに対するロック情報を保持し、ロックの属性、有効期限、所有者、およびロックチェーンの管理を行うための構造体です。

シグネチャまたは形状

typedef struct dav_lock {
    dav_lock_rectype rectype;
    int is_locknull;
    dav_lock_scope scope;
    dav_lock_type type;
    int depth;
    time_t timeout;
    const dav_locktoken *locktoken;
    const char *owner;
    const char *auth_user;
    dav_lock_private *info;
    struct dav_lock *next;
} dav_lock;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。各フィールドはロックのメタデータを格納し、next ポインタを通じてロックのリストを構成します。

前提条件と不変条件

  • next フィールドにより、単方向リストとして管理されます。
  • info フィールドはロックデータベース（lockdb）固有のプライベートデータへのポインタです。

アルゴリズムまたは振る舞い

  • ロックの属性（スコープ、タイプ、深さ、タイムアウト）を保持します。
  • is_locknull フラグにより、そのロックが「ロックヌルリソース（locknull resource）」を確立するものかどうかを識別します。
  • auth_user にはロックを所有する認証済みユーザー名が格納されます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/fs/lock.c および modules/dav/fs/repos.c 内の関数群が本構造体のインスタンスを生成・操作しています。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数呼び出しを含みません。

関連データ構造

  • dav_locktoken: ロックを識別するトークン。
  • dav_lock_private: ロックデータベース実装が使用する非公開情報。
  • dav_lock_rectype, dav_lock_scope, dav_lock_type: ロックの性質を定義する列挙型。

失敗モードと境界条件

  • next ポインタが正しく管理されない場合、ロックリストの走査時にメモリリークや不正なメモリ参照が発生する可能性があります。
  • timeout の値が適切に設定されていない場合、ロックの期限切れ処理が正しく機能しません。

保守担当者向けメモ

  • ### put the resource in here? というコメントがソース内に存在し、将来的にリソースへの参照を構造体内に含めることが検討されています。
  • modules/dav/fs/lock.c 内で dav_lock のメモリ割り当てや管理が行われており、変更時はこの実装との整合性に注意してください。
