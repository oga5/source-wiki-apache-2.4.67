set_keep_alive_timeout

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:51

目的

Apache HTTP Server の設定ディレクティブ KeepAliveTimeout を解析し、サーバー設定構造体（server_rec）に反映させるための設定ハンドラ関数です。

シグネチャまたは形状

static const char *set_keep_alive_timeout(cmd_parms *cmd, void *dummy, const char *arg);

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: KeepAliveTimeout ディレクティブに指定された時間文字列（例: "5s"）。
  • **出力**:
  • 成功時: NULL を返します。
  • 失敗時: エラーメッセージを含む文字列（const char *）を返します。

前提条件と不変条件

  • 本関数はディレクトリコンテキスト（<Directory> 等）内では使用できません。ap_check_cmd_context により NOT_IN_DIR_CONTEXT が強制されます。
  • arg は ap_timeout_parameter_parse が解釈可能な形式である必要があります。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、サーバー設定レベルでの呼び出しであることを検証します。 2. ap_timeout_parameter_parse を使用して、入力文字列を apr_interval_time_t 型の秒数に変換します。 3. 変換した値を cmd->server->keep_alive_timeout に格納します。 4. 仮想ホスト（is_virtual）の場合、設定が明示的に行われたことを示すフラグ keep_alive_timeout_set を 1 に設定します。これにより、後続の処理で継承ロジックが適切に動作するようにします。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_core.c 内の AP_INIT_TAKE1 マクロ定義（KeepAliveTimeout ディレクティブの登録）。
  • **呼び出し先**:
  • ap_check_cmd_context（サーバーコア）
  • ap_timeout_parameter_parse（APR/Apache ユーティリティ）

関連データ構造

  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。
  • server_rec: サーバー設定を保持する構造体（cmd->server 経由でアクセス）。

失敗モードと境界条件

  • コンテキストエラー: ディレクトリコンテキスト内で使用された場合、ap_check_cmd_context がエラーメッセージを返します。
  • パースエラー: arg が不正な形式（数値以外や単位の誤りなど）の場合、"KeepAliveTimeout has wrong format" というエラーメッセージを返します。

保守担当者向けメモ

  • 本関数は mod_proxy.c の実装を参考にしています。
  • 仮想ホストにおける keep_alive_timeout_set フラグは、メインサーバーからの設定継承と、IP:Port ベースのデフォルト設定の優先順位を制御するために重要です。変更時はこの継承ロジックへの影響を考慮してください。
