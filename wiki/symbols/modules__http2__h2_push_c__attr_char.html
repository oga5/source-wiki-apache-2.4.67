attr_char

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_push.c:74

目的

HTTPヘッダーやトークン解析において、特定の文字が「属性（attribute）」の一部として許容される文字であるかを判定します。

シグネチャまたは形状

static int attr_char(char c)

入力と出力

  • **入力**: char c （判定対象の文字）
  • **出力**: int （許容される場合は 1、それ以外は 0）

前提条件と不変条件

  • apr_isalnum に依存しており、標準的な英数字は常に許容されます。
  • 記号類については、特定のリストに含まれるもののみを許容する厳格なホワイトリスト方式を採用しています。

アルゴリズムまたは振る舞い

1. 引数 c が以下のいずれかの記号であるかを確認します： !, #, $, &, +, -, ., ^, _, ` `, |, ~ 2. 上記の記号であれば 1 を返します。 3. それ以外の場合、apr_isalnum(c) を呼び出し、英数字であれば 1 を、そうでなければ 0` を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_util.c:888
  • modules/http2/h2_push.c:217
  • modules/md/md_util.c:1397
  • **呼び出し先**:
  • apr_isalnum (APRライブラリ関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 入力が char 型の範囲外である場合、apr_isalnum の挙動に依存します。
  • NULL 文字や制御文字は、apr_isalnum が偽を返すため、本関数も 0 を返します。

保守担当者向けメモ

  • 本関数はトークン解析のループ内で頻繁に呼び出されるため、変更時はパフォーマンスへの影響を考慮してください。
  • 許容する記号を追加・削除する場合は、HTTP仕様（RFC）の該当するトークン定義と整合しているか確認してください。
