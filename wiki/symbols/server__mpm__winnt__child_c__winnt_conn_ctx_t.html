winnt_conn_ctx_t

定義

  • 種別: struct
  • 定義箇所: server/mpm/winnt/child.c:89

目的

Windows 版 Apache HTTP Server において、I/O Completion Port (IOCP) を介した接続処理のコンテキストを保持および再利用するための構造体です。

シグネチャまたは形状

typedef struct winnt_conn_ctx_t_s {
    struct winnt_conn_ctx_t_s *next;
    OVERLAPPED overlapped;
    apr_socket_t *sock;
    SOCKET accept_socket;
    char buff[2*PADDED_ADDR_SIZE];
    struct sockaddr *sa_server;
    int sa_server_len;
    struct sockaddr *sa_client;
    int sa_client_len;
    apr_pool_t *ptrans;
    apr_bucket_alloc_t *ba;
    apr_bucket *data;
#if APR_HAVE_IPV6
    short socket_family;
#endif
} winnt_conn_ctx_t;

入力と出力

  • **入力**: mpm_get_completion_context を通じてプールから取得、または apr_pcalloc により子プロセス用メモリプールから割り当てられます。
  • **出力**: 処理完了後、mpm_recycle_completion_context を介して再利用キュー（qhead/qtail）へ戻されます。

前提条件と不変条件

  • OVERLAPPED 構造体は、Windows の非同期 I/O 操作において IOCP との連携に必須です。
  • 構造体内の next ポインタは、再利用キュー内での連結リスト管理に使用されます。

アルゴリズムまたは振る舞い

1. **取得**: mpm_get_completion_context が再利用キューから既存のコンテキストを検索し、存在しない場合は新規作成します。 2. **接続処理**: リスナー・スレッドが AcceptEx 等で接続を受け入れ、この構造体にソケット情報やアドレス情報を格納します。 3. **通知**: PostQueuedCompletionStatus を使用して、ワーカー・スレッドへ接続の到来を通知します。 4. **解放**: ワーカー・スレッドが GetQueuedCompletionStatus で取得し、HTTP 処理完了後に mpm_recycle_completion_context でキューへ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: mpm_get_completion_context（取得）、mpm_recycle_completion_context（返却）、winnt_get_connection（取得・変換）、child_main（初期化時の割り当て）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • OVERLAPPED: Windows API の非同期 I/O 制御用構造体。
  • qhead, qtail: winnt_conn_ctx_t を管理する再利用キューの先頭および末尾ポインタ。

失敗モードと境界条件

  • **メモリリーク**: エラーパスにおいて、コンテキストが適切に再利用キューへ戻されない場合、メモリリークが発生する可能性があります。
  • **キューの枯渇**: 接続数が急増した場合、コンテキストの新規作成コストが発生します。

保守担当者向けメモ

  • 本構造体はメモリ断片化を防ぐための最適化として再利用されています。エラーハンドリングの修正時には、必ず mpm_recycle_completion_context を経由してメモリが適切に解放または再利用されることを確認してください。
