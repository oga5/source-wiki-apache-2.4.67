proxy_walk

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:1059

目的

リクエストの URL に基づいて <Proxy> セクションで定義された設定を検索し、現在のリクエスト設定（r->per_dir_config）にマージして適用する。

シグネチャまたは形状

static int proxy_walk(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報。r->filename に proxy: 接頭辞が付与された URL が格納されていることを想定）。
  • **出力**: int（常に OK を返す）。副作用として r->per_dir_config が更新される。

前提条件と不変条件

  • r->filename は proxy: で始まる文字列である必要がある。
  • sconf->sec_proxy に格納された設定エントリが順次評価される。

アルゴリズムまたは振る舞い

1. サーバー設定から <Proxy> セクションのリストを取得する。 2. 各セクションの設定（entry_proxy）に対し、以下のいずれかの方法でマッチングを行う。

  • **正規表現マッチ**: entry_proxy->r が定義されている場合、ap_regexec を実行。マッチした場合はキャプチャグループを r->subprocess_env に設定する。
  • **文字列/fnmatch**: entry_proxy->p を使用し、p_is_fnmatch フラグに応じて apr_fnmatch または strncmp で比較する。

    • 3. マッチした設定を ap_merge_per_dir_configs を用いて現在の r->per_dir_config に累積的にマージする。 4. 処理終了後、一時的に作成した正規表現用プール（rxpool）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy.c 内の map_to_storage フック（directory_walk/file_walk の代わりとして）。
  • modules/proxy/mod_proxy.c:1145。
  • **呼び出し先**:
  • ap_get_module_config
  • apr_pool_create / apr_pool_destroy
  • ap_regexec
  • ap_merge_per_dir_configs
  • apr_fnmatch
  • apr_table_setn

関連データ構造

  • proxy_server_conf: サーバー単位のプロキシ設定。
  • proxy_dir_conf: ディレクトリ単位（<Proxy> セクション）の設定。
  • request_rec: Apache のリクエスト構造体。

失敗モードと境界条件

  • マッチングに失敗したセクションはスキップされ、次のセクションの評価へ進む。
  • 正規表現マッチング時にキャプチャグループが存在する場合、一時的なメモリプール rxpool を生成する。

保守担当者向けメモ
