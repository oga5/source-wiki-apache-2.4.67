dbdlogout_check_authorization

定義

  • **種別**: function
  • **定義箇所**: modules/aaa/mod_authz_dbd.c:332

目的

Apache HTTP Server の認可プロバイダーとして、データベース（DBD）を利用したユーザーのログアウト処理を実行し、認可ステータスを判定します。

シグネチャまたは形状

static authz_status dbdlogout_check_authorization(request_rec *r,
                                                  const char *require_args,
                                                  const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • require_args: 設定ファイルで指定された引数（未使用）。
  • parsed_require_args: 解析済みの引数（未使用）。
  • **出力**:
  • AUTHZ_DENIED_NO_USER: リクエストにユーザー情報が含まれていない場合。
  • AUTHZ_GRANTED: ログアウト処理（authz_dbd_login）が成功した場合。
  • AUTHZ_DENIED: ログアウト処理が失敗した場合。

前提条件と不変条件

  • r->per_dir_config から authz_dbd_cfg が取得可能であること。
  • データベース接続およびクエリ実行の準備が mod_dbd を通じて完了していること。

アルゴリズムまたは振る舞い

1. 現在のリクエストからディレクトリごとの設定（authz_dbd_cfg）を取得します。 2. r->user が存在するか確認し、存在しない場合は AUTHZ_DENIED_NO_USER を返します。 3. authz_dbd_login 関数を呼び出し、引数として "logout" を渡して SQL クエリを実行します。 4. authz_dbd_login の戻り値が OK であれば AUTHZ_GRANTED を、それ以外であれば AUTHZ_DENIED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbd.c:380（authz_dbd_hooks 内のプロバイダー登録時）。
  • **呼び出し先**: authz_dbd_login（modules/aaa/mod_authz_dbd.c 内で定義）。

関連データ構造

  • request_rec: リクエスト情報を保持する Apache コア構造体。
  • authz_dbd_cfg: 本モジュール固有のディレクトリ設定構造体。
  • authz_status: 認可結果を示す列挙型。

失敗モードと境界条件

  • **ユーザー未認証**: r->user が NULL の場合、処理を中断し AUTHZ_DENIED_NO_USER を返します。
  • **データベースエラー**: authz_dbd_login 内部でデータベース接続失敗やクエリ実行エラーが発生した場合、AUTHZ_DENIED が返されます。

保守担当者向けメモ

  • 本関数は authz_dbd_login を再利用してログアウト処理を行っています。authz_dbd_login の挙動変更が本関数に直接影響を与える点に注意してください。
  • require_args および parsed_require_args は現在使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
