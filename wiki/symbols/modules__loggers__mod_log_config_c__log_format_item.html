log_format_item

定義

  • **種別**: struct
  • **定義箇所**: modules/loggers/mod_log_config.c:287

目的

LogFormat ディレクティブで指定されたログフォーマット文字列を解析し、実行時に評価可能な中間形式として保持するために使用されます。

シグネチャまたは形状

typedef struct {
    ap_log_handler_fn_t *func;      /* ログ項目を生成するハンドラ関数 */
    char *arg;                      /* ハンドラに渡される引数（フォーマット指定子等） */
    int condition_sense;            /* 条件の評価結果（真偽） */
    int want_orig;                  /* オリジナルのリクエスト情報を必要とするか */
    apr_array_header_t *conditions; /* 適用条件のリスト */
} log_format_item;

入力と出力

  • **入力**: parse_log_item 関数により、フォーマット文字列から各フィールドが初期化されます。
  • **出力**: process_item 等の処理関数がこの構造体を参照し、リクエストごとのログ文字列を生成します。

前提条件と不変条件

  • func は ap_log_handler_fn_t 型の関数ポインタであり、有効なハンドラを指している必要があります。
  • apr_array_header_t 型の conditions は、リクエスト処理時に評価される条件式を保持します。

アルゴリズムまたは振る舞い

1. **解析**: parse_log_item がフォーマット文字列を走査し、各項目を log_format_item の配列として構築します。 2. **評価**: ログ記録時、各 log_format_item の func が呼び出され、arg やリクエスト情報に基づいて文字列が生成されます。 3. **条件判定**: conditions が設定されている場合、それらの評価結果に基づいてログ出力の可否や内容が制御されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • parse_log_misc_string (解析処理)
  • parse_log_item (解析処理)
  • modules/loggers/mod_log_config.c 内のログ設定構築ロジック（apr_array_make および apr_array_push を使用）
  • **呼び出し先**:
  • ap_log_handler_fn_t 型の関数（ハンドラ関数）
  • apr_array_push (配列操作)

関連データ構造

  • config_log_state: ログ設定全体を保持する構造体。
  • apr_array_header_t: log_format_item のリストを保持するために使用されます。

失敗モードと境界条件

  • フォーマット文字列の解析中に不正な指定子が見つかった場合、parse_log_item がエラーを返します。
  • メモリ不足時には apr_array_push 等の APR 関数が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_log_config.c 内で完結しており、外部モジュールからは mod_log_config.h を通じてハンドラ関数を登録することで間接的に利用されます。
  • want_orig フラグは、内部リダイレクト等が発生した際に、元のリクエスト情報を参照すべきかどうかを決定するために重要です。
