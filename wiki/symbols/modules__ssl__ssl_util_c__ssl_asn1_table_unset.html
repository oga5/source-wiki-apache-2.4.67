ssl_asn1_table_unset

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util.c:234

目的

指定されたハッシュテーブルから ASN.1 データを削除し、関連するメモリを解放します。

シグネチャまたは形状

void ssl_asn1_table_unset(apr_hash_t *table, const char *key)

入力と出力

  • **入力**:
  • table: 削除対象のデータが格納されている apr_hash_t 構造体へのポインタ。
  • key: 削除対象の項目を特定するための文字列キー。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • table は apr_hash_t として初期化済みであること。
  • key に対応するエントリが存在しない場合、関数は何もせず正常に終了する。
  • ssl_asn1_table_set によって malloc で確保されたメモリ領域を解放するため、本関数呼び出し後は当該キーに対するデータアクセスは無効となる。

アルゴリズムまたは振る舞い

1. key の長さを計算する。 2. apr_hash_get を使用して、ハッシュテーブルから ssl_asn1_t 構造体を取得する。 3. 構造体が存在しない場合は即座にリターンする。 4. 構造体内の cpData が存在する場合、free() を呼び出してメモリを解放する。 5. 構造体自体を free() で解放する。 6. apr_hash_set を呼び出し、ハッシュテーブルから当該キーのエントリを削除（NULL をセット）する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供されていません。
  • **呼び出し先**:
  • strlen (標準ライブラリ)
  • apr_hash_get (APR)
  • free (標準ライブラリ)
  • apr_hash_set (APR)

関連データ構造

  • apr_hash_t: APR のハッシュテーブル構造体。
  • ssl_asn1_t: ssl_asn1_table_set で管理される、DER 形式のデータを保持する内部構造体。

失敗モードと境界条件

  • key が NULL の場合、strlen でセグメンテーションフォールトが発生する可能性がある。
  • table が NULL の場合、apr_hash_get および apr_hash_set 内でクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は ssl_asn1_table_set と対になる関数です。ssl_asn1_table_set で確保されたメモリは APR プールではなく malloc で管理されているため、モジュールのリロードを跨いでデータを保持する設計となっています。メモリリークを防ぐため、不要になったエントリに対しては必ず本関数を呼び出してください。
