set_allowed_ports

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_connect.c:82

目的

Apache HTTP Server の設定ファイルで指定された AllowCONNECT ディレクティブの引数をパースし、許可するポート番号またはポート範囲を connect_conf 構造体に登録します。

シグネチャまたは形状

static const char *set_allowed_ports(cmd_parms *parms, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • parms: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: AllowCONNECT ディレクティブに渡された文字列（例: "443" や "80-90"）。
  • **出力**:
  • 成功時: NULL を返します。
  • 失敗時: エラーメッセージを含む文字列（const char *）を返します。

前提条件と不変条件

  • parms->server->module_config から proxy_connect_module の設定構造体（connect_conf）が取得可能である必要があります。
  • arg は数値で始まる必要があります。

アルゴリズムまたは振る舞い

1. arg の先頭が数値であることを確認し、数値でない場合はエラーを返します。 2. strtol を使用してポート番号を解析します。 3. ハイフン（-）が含まれている場合、範囲指定として開始ポートと終了ポートを解析します。ハイフンがない場合は、単一ポートとして開始・終了を同一値に設定します。 4. 解析結果が不正な形式（数値として解釈できない等）であればエラーを返します。 5. 解析したポート範囲を conf->allowed_connect_ports 配列にプッシュし、構造体に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_connect.c:391 において、AP_INIT_ITERATE マクロを通じて設定ディレクティブ AllowCONNECT のハンドラとして登録されています。
  • **呼び出し先**:
  • ap_get_module_config (Apache Core)
  • apr_isdigit (APR)
  • strtol (標準ライブラリ)
  • apr_psprintf (APR)
  • apr_array_push (APR)

関連データ構造

  • connect_conf: サーバーごとの設定を保持する構造体。allowed_connect_ports を管理します。
  • port_range: first と last のポート番号を保持する構造体。
  • cmd_parms: 設定コマンドのコンテキストを保持する構造体。

失敗モードと境界条件

  • **数値以外**: arg の先頭が数値でない場合、「AllowCONNECT: port numbers must be numeric」を返します。
  • **解析失敗**: 文字列の末尾まで正しく数値として解釈できない場合、「Cannot parse '...' as port number」を返します。

保守担当者向けメモ

  • 本関数は AllowCONNECT ディレクティブのパース専用です。ポートの検証ロジックについては、別途 allowed_port 関数を参照してください。
  • apr_array_push を使用しているため、メモリ割り当てはリクエストプールではなく、サーバー設定プール（parms->pool 経由）で行われます。
