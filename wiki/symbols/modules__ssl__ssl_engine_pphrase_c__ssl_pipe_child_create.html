ssl_pipe_child_create

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:393

目的

外部プログラム（パイプ）を子プロセスとして起動し、パスフレーズの入出力を行うための通信チャネルを確立します。

シグネチャまたは形状

apr_status_t ssl_pipe_child_create(apr_pool_t *p, const char *progname)

入力と出力

  • **入力**:
  • p: プロセスまたはリクエストの生存期間を管理する apr_pool_t。
  • progname: 実行するプログラム名（引数を含む文字列）。
  • **出力**:
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。
  • 副作用: グローバル変数（またはファイルスコープ変数）である writetty および readtty に、子プロセスの標準入力および標準出力のパイプが格納されます。

前提条件と不変条件

  • progname は apr_tokenize_to_argv で解析可能な形式である必要があります。
  • 呼び出し側は、この関数が成功した後に writetty と readtty を通じてパスフレーズの送受信を行うことを想定しています。

アルゴリズムまたは振る舞い

1. apr_procattr_create でプロセス属性を作成し、apr_procattr_io_set で標準入出力のパイプ設定（APR_FULL_BLOCK）を適用します。 2. apr_tokenize_to_argv を使用して progname を実行引数配列に変換します。 3. apr_proc_create を呼び出し、指定されたプログラムを子プロセスとして起動します。 4. 起動に成功した場合、子プロセスの入力パイプを writetty に、出力パイプを readtty に割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_pphrase.c:487
  • modules/ssl/ssl_engine_pphrase.c:639
  • **呼び出し先**:
  • apr_procattr_create, apr_procattr_io_set, apr_tokenize_to_argv, apr_pcalloc, apr_proc_create (すべて APR API)

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • apr_proc_t: 子プロセスの状態を保持する構造体。
  • apr_procattr_t: プロセス起動時の属性設定用構造体。

失敗モードと境界条件

  • apr_procattr_create や apr_proc_create が失敗した場合、そのエラーコードがそのまま返されます。
  • progname が不正な場合、子プロセスの起動に失敗し、rc にエラーが格納されます。

保守担当者向けメモ

  • 本関数は writetty および readtty という外部変数を更新します。これらがどのスコープで管理されているか、また並行実行時の競合がないか注意してください。
  • ソース内のコメントにある通り、apr_pool_note_subprocess による子プロセスの自動終了管理は現在コメントアウトされています。必要に応じて有効化を検討してください。
