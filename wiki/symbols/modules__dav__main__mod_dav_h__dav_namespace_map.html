dav_namespace_map

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1202

目的

WebDAV リクエストやプロパティ処理における XML 名前空間（Namespace）のマッピングを保持するための構造体です。

シグネチャまたは形状

typedef struct dav_namespace_map dav_namespace_map;

入力と出力

  • 入力: apr_palloc 等によるメモリ割り当て時に、プール（apr_pool_t）を介してコンテキストが初期化されます。
  • 出力: プロパティ操作や名前空間の解決が必要な関数において、ポインタ経由でマッピング情報が参照されます。

前提条件と不変条件

  • 関連するメモリ管理は、Apache の apr_pool_t に依存します。
  • modules/dav/fs/dbm.c 等のコードにおいて、db->pool からメモリを確保してインスタンス化されるケースが確認されています。

アルゴリズムまたは振る舞い

  • XML 名前空間の定義とリソース間の関連付けを管理します。
  • プロパティ処理（props.c）において、名前空間の解決やマッピングの保持に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • modules/dav/fs/dbm.c: 名前空間マッピングの生成と操作。
  • modules/dav/main/props.c: プロパティコンテキスト内での名前空間マッピング保持。
  • 呼び出し先:
  • 提供されたソース excerpt には、本構造体が内部で呼び出す関数や、本構造体を引数に取る関数の具体的な実装ロジックは含まれていません。

関連データ構造

  • dav_prop_ctx: プロパティ処理コンテキスト内で本構造体がメンバとして参照されます。
  • dav_db: DBM ベースのプロパティデータベース操作時に利用されます。

失敗モードと境界条件

  • メモリ割り当て失敗時（apr_palloc が失敗した場合）の挙動は、Apache のメモリ管理ポリシーに依存します。
  • 名前空間マッピングが未定義または NULL の状態で参照された場合の挙動については、各フック実装側のガード条件に依存します。

保守担当者向けメモ

  • 本構造体は mod_dav の XML 名前空間管理の基盤です。
  • modules/dav/main/mod_dav.h の 1264 行目および 1276 行目で関数引数として宣言されており、プロバイダーインターフェースの一部として機能しています。
  • XML 名前空間のガベージコレクションや詳細なライフサイクル管理については、mod_dav のプロバイダー実装（fs 等）のコードを併せて参照してください。
