proxy_translate_name

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:1054

目的

Apache HTTP Server のリクエスト処理パイプラインにおいて、リクエストされた URL がプロキシ対象であるかを判定し、必要に応じてリクエストをプロキシ処理へルーティングするためのフック関数です。

シグネチャまたは形状

static int proxy_translate_name(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在処理中のリクエスト情報)
  • **出力**: int (Apache のフック戻り値。OK, DECLINED, またはエラーコード)

前提条件と不変条件

  • 本関数は ap_hook_translate_name を通じて登録され、リクエストの翻訳フェーズで呼び出されます。
  • 内部で呼び出される proxy_trans が、リクエストの r->filename を proxy:<URL> 形式に書き換えることで、後続のプロキシハンドラへの引き継ぎを行います。

アルゴリズムまたは振る舞い

1. proxy_trans(r, 0) を呼び出します。 2. proxy_trans の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のリクエスト処理パイプライン（ap_hook_translate_name により登録）。
  • **呼び出し先**: proxy_trans(request_rec *r, int)。

関連データ構造

  • request_rec: Apache のリクエスト状態を保持する構造体。

失敗モードと境界条件

  • proxy_trans の挙動に依存します。リクエストがプロキシ対象でない場合は DECLINED を返し、処理を継続させます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的なロジックは proxy_trans に集約されています。
  • modules/proxy/mod_proxy.c:3366 にて ap_hook_translate_name に登録されています。
