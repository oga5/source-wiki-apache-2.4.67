md_event_raise

定義

  • 種別: function
  • 定義箇所: modules/md/md_event.c:61

目的

登録されたすべてのイベントリスナー（コールバック関数）を順次実行し、イベントを通知します。

シグネチャまたは形状

apr_status_t md_event_raise(const char *event, 
                            const char *mdomain, 
                            struct md_job_t *job, 
                            struct md_result_t *result, 
                            apr_pool_t *p)

入力と出力

  • **入力**:
  • event: イベント名を示す文字列。
  • mdomain: 対象のドメイン名。
  • job: 関連するジョブ情報 (md_job_t)。
  • result: 処理結果を保持するコンテナ (md_result_t)。
  • p: メモリ割り当てに使用するプール (apr_pool_t)。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • リスナーのいずれかがエラーを返した場合は、その apr_status_t エラーコードを返します。

前提条件と不変条件

  • EVNT 構造体が初期化されている必要があります。
  • 本関数はスレッドセーフではありません。マルチスレッド環境下での呼び出しには、呼び出し元での排他制御が必要です。

アルゴリズムまたは振る舞い

1. 静的変数 EVNT.subs から購読リストの先頭を取得します。 2. リストを順次走査し、各ノードのコールバック関数 (sub->cb) を呼び出します。 3. コールバックの戻り値が APR_SUCCESS でない場合、その時点で処理を中断し、エラーコードを返します。 4. すべてのコールバックが正常終了した場合、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_status.c:700
  • modules/md/mod_md_drive.c:177
  • **呼び出し先**:
  • 購読リスト (EVNT.subs) に登録された各コールバック関数。

関連データ構造

  • md_subscription: 購読情報を保持する構造体。
  • md_job_t: ジョブの状態管理用構造体。
  • md_result_t: 処理結果管理用構造体。
  • EVNT: モジュール内で静的に保持されるイベント管理用グローバル構造体。

失敗モードと境界条件

  • リスナーがエラーを返すと、後続のリスナーは実行されずに処理が終了します。
  • 購読解除機能がないため、プロセス生存期間中は登録されたすべてのリスナーが実行対象となります。

保守担当者向けメモ

  • 確実な通知が必要な場合は、エラーを無視して全リスナーを実行する md_event_holler の使用を検討してください。
  • 購読リストの操作は初期化時に行われる前提ですが、実行時の動的なリスト変更に対する保護は実装されていません。
