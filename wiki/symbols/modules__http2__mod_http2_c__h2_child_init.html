h2_child_init

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_http2.c:162

目的

Apache HTTP Server の子プロセス生成時に、HTTP/2 モジュールに必要な接続処理環境を初期化する。

シグネチャまたは形状

static void h2_child_init(apr_pool_t *pchild, server_rec *s)

入力と出力

  • **入力**:
  • pchild: 子プロセス用のメモリプール。
  • s: サーバー設定レコード。
  • **出力**: なし（戻り値は void）。内部でエラーが発生した場合はログ出力を行う。

前提条件と不変条件

  • 本関数は Apache の child_init フックとして呼び出される。
  • h2_c1_child_init および h2_c2_child_init が正常に初期化を完了できる環境であること。

アルゴリズムまたは振る舞い

1. h2_c1_child_init(pchild, s) を呼び出し、フロントエンド（クライアント）接続処理を初期化する。 2. 1が成功した場合に限り、h2_c2_child_init(pchild, s) を呼び出し、バックエンド接続処理を初期化する。 3. いずれかの初期化で APR_SUCCESS 以外のステータスが返された場合、ap_log_error を使用してエラーログ（APLOGNO 02949）を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフックシステム（modules/http2/mod_http2.c:195 にて ap_hook_child_init に登録）。
  • **呼び出し先**:
  • h2_c1_child_init
  • h2_c2_child_init
  • ap_log_error（エラー発生時）

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • server_rec: サーバー設定情報。

失敗モードと境界条件

  • h2_c1_child_init または h2_c2_child_init が失敗した場合、エラーコードがログに記録されるが、プロセス自体は終了しない。
  • 接続処理の初期化失敗は、HTTP/2 通信機能の利用不可につながる可能性がある。

保守担当者向けメモ

  • 本関数は h2_c1（クライアント側）と h2_c2（バックエンド側）の初期化順序を制御する重要なハブである。
  • 接続処理の安定性に直結するため、初期化ロジックを変更する際は両サブコンポーネントの依存関係に注意すること。
