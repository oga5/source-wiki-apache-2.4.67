request_rec

定義

  • 種別: typedef
  • 定義箇所: modules/metadata/mod_headers.c:111（注: httpd.h:822 に構造体定義が存在）

目的

Apache HTTP Server における単一の HTTP リクエストのライフサイクル全体を管理・保持するための主要なデータ構造です。

シグネチャまたは形状

typedef struct request_rec request_rec; （詳細は httpd.h に定義される構造体を参照）

入力と出力

本シンボル自体は型定義ですが、この型のインスタンスはリクエスト処理の各段階で入力（リクエストヘッダー等）を保持し、処理結果（レスポンスヘッダー等）を出力として書き込むためのコンテキストとして機能します。

前提条件と不変条件

  • リクエストの処理中、常に有効なメモリプール（pool）に関連付けられています。
  • リクエストの終了時に、関連付けられたメモリプールと共に自動的に解放されます。

アルゴリズムまたは振る舞い

  • mod_headers においては、do_headers_fixup 関数がこの構造体内の headers_in、headers_out、err_headers_out を直接操作し、ヘッダーの追加・削除・置換を行います。
  • process_tags 関数は、この構造体からリクエスト情報を参照して動的な値を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache コアおよび各モジュール（mod_headers, mod_example_1, mod_example_2 等）のフック関数。
  • **呼び出し先**: ap_expr_exec や ap_expr_str_exec などの式評価エンジンが、リクエストコンテキストとして本構造体を受け取ります。

関連データ構造

  • headers_in, headers_out, err_headers_out (apr_table_t): HTTP ヘッダーを保持するテーブル。
  • pool (apr_pool_t): リクエストに関連するメモリ管理用プール。
  • subprocess_env (apr_table_t): 環境変数を保持するテーブル。

失敗モードと境界条件

  • 構造体内の各テーブルやポインタが NULL である場合、アクセス時にセグメンテーションフォールトが発生する可能性があります。
  • リクエスト処理のタイミング（フックの実行順序）により、特定のフィールドがまだ初期化されていない、あるいは既に解放されている場合があります。

保守担当者向けメモ

  • mod_headers.c:111 の定義は httpd.h で定義された構造体のエイリアスです。
  • request_rec のメンバー構成は Apache のバージョンアップに伴い変更されることがあるため、include/ap_mmn.h（Module Magic Number）の変更履歴を確認してください。
  • early 処理と通常のフィルタ処理における request_rec の状態変化には注意が必要です。
