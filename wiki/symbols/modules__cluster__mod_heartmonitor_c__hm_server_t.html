hm_server_t

定義

  • 種別: struct
  • 定義箇所: modules/cluster/mod_heartmonitor.c:46

目的

個々のバックエンドサーバーの稼働状態（IPアドレス、ポート番号、負荷統計、最終確認時刻）を保持し、mod_heartmonitor による監視・集約処理のデータ単位として機能します。

シグネチャまたは形状

typedef struct hm_server_t
{
    const char *ip;
    int busy;
    int ready;
    unsigned int port;
    apr_time_t seen;
} hm_server_t;

入力と出力

本構造体はデータ保持用であり、直接的な入出力関数ではありません。各フィールドは hm_processmsg 等の解析処理によって更新され、hm_update_stat 等の永続化処理を通じて共有メモリやファイルへ書き出されます。

前提条件と不変条件

  • ip はサーバーを識別する文字列として保持されます。
  • seen は apr_time_t 型であり、最後にハートビートを受信した時刻を記録します。
  • 構造体内の各フィールドは、ハートビートメッセージの受信ごとに更新されることが想定されています。

アルゴリズムまたは振る舞い

本構造体は、mod_heartmonitor におけるサーバー状態の管理オブジェクトとして機能します。 1. **生成**: hm_get_server により、特定の IP とポートに対応するインスタンスが取得または生成されます。 2. **更新**: 受信したハートビートメッセージの内容に基づき、busy、ready、seen フィールドが更新されます。 3. **永続化**: hm_update_stat 系の関数を介して、共有メモリ（slotmem）またはファイルシステムへ状態がフラッシュされます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_heartmonitor.c 内の統計更新処理（hm_slotmem_update_stat, hm_file_update_stat, hm_update_stat）およびサーバー取得処理（hm_get_server）から参照されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • hm_ctx_t: モジュール全体の実行時状態を保持し、hm_server_t のリストやハッシュテーブルを管理します。
  • hm_slot_server_ctx_t: 共有メモリ上でサーバー状態を管理するための構造体です。

失敗モードと境界条件

  • ip が NULL または不正な形式である場合、hm_get_server によるインスタンス取得が失敗する可能性があります。
  • 共有メモリ（slotmem）への書き込み時にリソースが枯渇している場合、hm_slotmem_update_stat がエラーを返す可能性があります。

保守担当者向けメモ

  • 本構造体は mod_heartmonitor の中核となるデータモデルです。フィールドの追加や変更を行う際は、hm_update_stat 系の永続化ロジックおよび hm_processmsg のパース処理への影響を確認してください。
  • seen フィールドの更新精度は、mod_watchdog のコールバック間隔に依存します。
