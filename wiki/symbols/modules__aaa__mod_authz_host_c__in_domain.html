in_domain

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:59

目的

ホスト名（what）が指定されたドメイン（domain）のサブドメインであるか、あるいはドメインそのものであるかを判定します。

シグネチャまたは形状

static int in_domain(const char *domain, const char *what)

入力と出力

  • **入力**:
  • domain: 比較対象となるドメイン文字列。
  • what: 判定対象となるホスト名文字列。
  • **出力**:
  • 1: what が domain と一致、または domain のサブドメインである場合。
  • 0: それ以外の場合。

前提条件と不変条件

  • domain および what はヌル終端文字列である必要があります。
  • 比較はケースインセンシティブ（大文字小文字を区別しない）に行われます。

アルゴリズムまたは振る舞い

1. domain と what の長さを取得します。 2. what の長さが domain より短い場合、一致しないと判断し 0 を返します。 3. what の末尾部分と domain を strcasecmp で比較します。一致しない場合は 0 を返します。 4. 長さが完全に一致する場合（wl == dl）、1 を返します。 5. 長さが異なる場合、domain の先頭文字が . であるか、または what 内の domain の直前の文字が . であることを確認し、サブドメイン境界を検証します。条件を満たせば 1 を、そうでなければ 0 を返します。これにより、good.com が nogood.com にマッチするのを防ぎます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_access_compat.c:293
  • modules/aaa/mod_authz_host.c:217
  • **呼び出し先**:
  • 標準 C ライブラリ: strlen, strcasecmp

関連データ構造

  • 特になし。

失敗モードと境界条件

  • what が domain より短い場合、即座に 0 を返します。
  • 部分文字列が一致しても、ドメイン境界（.）が正しく配置されていない場合（例: domain="good.com", what="nogood.com"）は 0 を返します。

保守担当者向けメモ

  • 本関数は DNS 逆引き結果の検証において、ドメイン階層の包含関係を安全に判定するために使用されます。
  • strcasecmp を使用しているため、ロケール設定に依存する可能性があります。
