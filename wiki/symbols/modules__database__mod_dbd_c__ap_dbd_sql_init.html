ap_dbd_sql_init

定義

  • 種別: function
  • 定義箇所: modules/database/mod_dbd.c:140

目的

サーバー設定に対して、データベース接続確立時に実行される初期化用 SQL クエリを登録します。

シグネチャまたは形状

static void ap_dbd_sql_init(server_rec *s, const char *query)

入力と出力

  • s: 処理対象のサーバーレコード (server_rec)。
  • query: 登録する SQL クエリ文字列。
  • 出力: なし（副作用としてサーバー設定構造体 svr_cfg を更新）。

前提条件と不変条件

  • s が有効なサーバーレコードであること。
  • svr_cfg が未初期化の場合、create_dbd_config を用いて自動的に生成・設定されます。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、指定されたサーバー s から mod_dbd の設定を取得します。 2. 設定が存在しない場合、create_dbd_config を呼び出して新規に設定を作成し、ap_set_module_config でサーバーに割り当てます。 3. query が提供されている場合、そのクエリを svr->cfg->init_queries 配列に追加します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/database/mod_dbd.c:273 (設定ディレクティブハンドラ内での呼び出しが確認されています)。
  • 呼び出し先: create_dbd_config (設定が存在しない場合)、apr_array_push (クエリの追加時)。

関連データ構造

  • svr_cfg: サーバーごとの設定コンテナ。
  • dbd_cfg_t: svr_cfg 内で保持されるデータベース設定値。init_queries 配列を保持します。

失敗モードと境界条件

  • query が NULL の場合、配列への追加処理はスキップされます。
  • 設定の生成時に config_pool が不足している場合、APR のメモリ割り当てエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は主に設定フェーズで呼び出されることを想定しています。
  • 登録されたクエリは、後に dbd_run_post_connect 等の接続初期化プロセスで利用されます。
