srv_conf_props_clear

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_config.c:166

目的

md_srv_conf_t 構造体の各プロパティを初期状態（デフォルト値または NULL）にリセットします。

シグネチャまたは形状

static void srv_conf_props_clear(md_srv_conf_t *sc)

入力と出力

  • **入力**: sc - 初期化対象となる md_srv_conf_t 構造体へのポインタ。
  • **出力**: なし（副作用として sc が指す構造体のメンバが書き換えられます）。

前提条件と不変条件

  • sc は有効な md_srv_conf_t 構造体を指している必要があります。
  • 実行後、数値型メンバは DEF_VAL または MD_REQUIRE_UNSET に、ポインタ型メンバは NULL に設定されます。

アルゴリズムまたは振る舞い

以下のメンバに対して順次値を代入します。 1. 数値・列挙型: transitive, renew_mode, must_staple, profile_mandatory, stapling, staple_others, ari_renewals を DEF_VAL に設定。require_https を MD_REQUIRE_UNSET に設定。 2. ポインタ型: pks, renew_window, warn_window, ca_urls, ca_contact, ca_proto, ca_agreement, ca_challenges, ca_eab_kid, ca_eab_hmac, profile, dns01_cmd を NULL に設定。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_config.c:249
  • modules/md/mod_md_config.c:428
  • **呼び出し先**: なし（本関数は代入のみを行う末端のユーティリティです）。

関連データ構造

  • md_srv_conf_t: バーチャルホスト単位の設定を保持する構造体。

失敗モードと境界条件

  • sc が NULL の場合、プログラムはクラッシュします（防御的プログラミングは行われていません）。

保守担当者向けメモ

  • 新しい設定項目が md_srv_conf_t に追加された場合、本関数で適切に初期化されているか確認してください。
