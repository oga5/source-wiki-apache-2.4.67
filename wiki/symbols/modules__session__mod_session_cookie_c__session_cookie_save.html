session_cookie_save

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_cookie.c:57

目的

session_rec に保持されたセッションデータを、設定に基づき HTTP レスポンスヘッダー（Set-Cookie）として書き出す、または削除する。

シグネチャまたは形状

static apr_status_t session_cookie_save(request_rec *r, session_rec *z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • z: 保存対象のセッションデータ（session_rec）。
  • **出力**:
  • apr_status_t: 処理が実行された場合は OK、設定により処理対象外と判断された場合は DECLINED を返す。

前提条件と不変条件

  • r->per_dir_config から session_cookie_dir_conf が取得可能であること。
  • z->encoded にシリアライズ済みのセッション文字列が格納されていること。

アルゴリズムまたは振る舞い

1. **RFC2109 Cookie の処理**:

  • conf->name_set が有効な場合、z->encoded が存在すれば ap_cookie_write で書き込み、存在しなければ ap_cookie_remove で削除する。

    • 2. **RFC2965 Cookie2 の処理**:

  • conf->name2_set が有効な場合、z->encoded が存在すれば ap_cookie_write2 で書き込み、存在しなければ ap_cookie_remove2 で削除する。

    • 3. **終了処理**:

  • いずれかの Cookie 設定が有効であれば OK を返し、そうでなければ DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_session の session_save フックとして登録されている（modules/session/mod_session_cookie.c:271）。
  • **呼び出し先**:
  • ap_cookie_write / ap_cookie_write2（util_cookies.h）
  • ap_cookie_remove / ap_cookie_remove2（util_cookies.h）

関連データ構造

  • session_rec: セッションデータ本体。
  • session_cookie_dir_conf: Cookie 名や属性を保持するディレクトリ単位の設定構造体。

失敗モードと境界条件

  • z->encoded が空または NULL の場合、Cookie の書き込みではなく削除処理が優先される。
  • 設定で Cookie 名が指定されていない場合、その規格（RFC2109/RFC2965）の処理はスキップされる。

保守担当者向けメモ

  • 本関数は mod_session のストレージバックエンドとして動作する。
  • 実際のシリアライズ処理は本関数ではなく、呼び出し元の mod_session または mod_session_crypto 等で事前に行われる。
  • r->err_headers_out を使用してレスポンスヘッダーを操作しており、リクエスト終了時に自動的にクライアントへ送信される。
