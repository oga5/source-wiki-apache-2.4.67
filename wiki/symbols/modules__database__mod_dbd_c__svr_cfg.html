svr_cfg

定義

  • 種別: struct
  • 定義箇所: modules/database/mod_dbd.c:69

目的

Apache HTTP Server におけるサーバーごとのデータベース設定および接続グループを保持するコンテナ構造体です。

シグネチャまたは形状

typedef struct {
    dbd_cfg_t *cfg;
    dbd_group_t *group;
} svr_cfg;

入力と出力

  • 入力: サーバー設定（dbd_cfg_t）および接続管理グループ（dbd_group_t）。
  • 出力: なし（構造体定義）。

前提条件と不変条件

  • cfg はデータベース接続設定（ドライバ名、パラメータ等）を保持します。
  • group は接続プール（apr_reslist）または単一接続を管理します。
  • サーバーのライフサイクルを通じて、ap_get_module_config を介してアクセスされます。

アルゴリズムまたは振る舞い

  • create_dbd_config および merge_dbd_config を通じて、サーバーの設定フェーズで初期化・統合されます。
  • dbd_post_config などのフックにおいて、設定値に基づいた接続グループの構築に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/database/mod_dbd.c:96 (初期化)
  • modules/database/mod_dbd.c:118-120 (設定マージ)
  • modules/database/mod_dbd.c:165, 206, 242, 349, 757 (ap_get_module_config による取得)
  • 呼び出し先:
  • 構造体内部で dbd_cfg_t および dbd_group_t を参照しています。

関連データ構造

  • dbd_cfg_t: データベース接続設定値。
  • dbd_group_t: 接続プールまたは単一接続管理構造体。

失敗モードと境界条件

  • 設定が未定義の場合、cfg や group が NULL になる可能性があります。
  • ap_get_module_config が失敗した場合、ポインタのデリファレンス時にクラッシュするリスクがあります。

保守担当者向けメモ

  • 本構造体は mod_dbd の設定管理の核となるため、変更時は merge_dbd_config のロジックに影響がないか確認してください。
  • 接続プール（group）の初期化タイミングは dbd_post_config に依存しています。
