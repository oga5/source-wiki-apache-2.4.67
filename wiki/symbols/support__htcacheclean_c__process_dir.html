process_dir

定義

  • 種別: function
  • 定義箇所: support/htcacheclean.c:594

目的

キャッシュディレクトリを再帰的に走査し、ヘッダーファイルとデータファイルのペアを特定して、キャッシュエントリのリスト（ENTRY）を構築します。

シグネチャまたは形状

static int process_dir(char *path, apr_pool_t *pool, apr_off_t *nodes)

入力と出力

  • **入力**:
  • path: 走査対象のディレクトリパス。
  • pool: メモリ管理用の APR プール。
  • nodes: 走査した inode 数をカウントするためのポインタ。
  • **出力**:
  • 成功時は 0、失敗または中断時は 1 を返します。

前提条件と不変条件

  • apr_pool_t を使用してメモリを管理し、再帰呼び出し時のリークを防ぐため、関数内でサブプールを作成・破棄します。
  • interrupted フラグが真の場合、処理を即座に中断します。
  • Apache との競合を避けるため、apr_stat のリトライ処理が組み込まれています。

アルゴリズムまたは振る舞い

1. 指定されたディレクトリを apr_dir_open で開き、内容を DIRENTRY リング構造に格納します。 2. 各エントリを走査し、ファイルタイプ（ディレクトリ、ヘッダー、データ）を判定します。 3. ディレクトリの場合は再帰的に process_dir を呼び出します。 4. ファイルの場合は apr_hash_t を用いて、同一ベース名のヘッダーとデータをペアリングします。 5. ペアリング完了後、ハッシュテーブルを走査し、キャッシュフォーマット（DISK_FORMAT_VERSION 等）を検証します。 6. 有効なエントリを ENTRY リストに追加し、不正なファイルや孤立したファイルは delete_entry または delete_file で削除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/htcacheclean.c:687 (再帰呼び出し)
  • support/htcacheclean.c:1771 (メイン処理からの呼び出し)
  • **呼び出し先**:
  • delete_entry, delete_file (ファイル削除)
  • apr_dir_open, apr_dir_read, apr_stat, apr_file_open 等の APR 関数群。

関連データ構造

  • DIRENTRY: ディレクトリ走査中の一時的なファイル情報保持用。
  • ENTRY: 最終的な削除判定に使用されるキャッシュエントリリスト。
  • apr_hash_t: ヘッダーとデータのペアリングに使用。

失敗モードと境界条件

  • ディレクトリが開けない場合、または走査中に中断された場合は 1 を返します。
  • realclean モードでない場合、一時ファイル（AP_TEMPFILE_BASE で始まるもの）はスキップされます。
  • 読み取り不能なヘッダーファイルは、現在の時刻と MAXDEVIATION を比較し、古い場合にのみ削除されます。

保守担当者向けメモ

  • メモリ管理のため、関数内で作成したサブプール p が適切に破棄されているか確認してください。
  • apr_stat のリトライ回数は STAT_ATTEMPTS 定数で制御されています。
  • deldirs フラグが有効な場合、空になったディレクトリの削除が試行されます。
