register_hooks

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_groupfile.c:311

目的

Apache HTTP Server のモジュールライフサイクルにおいて、認可プロバイダーの登録および必要な外部関数の取得を行う。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（モジュール設定用のメモリプール）
  • **出力**: なし（副作用としてフックおよびプロバイダーを登録）

前提条件と不変条件

  • Apache のモジュールロードプロセスにおいて、他のモジュール初期化フックと並行して呼び出される。
  • AUTHZ_PROVIDER_GROUP 認可プロバイダーグループに対して、group および file-group プロバイダーを登録する。

アルゴリズムまたは振る舞い

1. ap_register_auth_provider を呼び出し、group プロバイダーとして authz_group_provider を登録する。 2. ap_register_auth_provider を呼び出し、file-group プロバイダーとして authz_filegroup_provider を登録する。 3. ap_hook_optional_fn_retrieve を呼び出し、authz_groupfile_getfns を介してオプション関数（authz_owner_get_file_group 等）を取得する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュールロード機構（直接の呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • ap_register_auth_provider
  • ap_hook_optional_fn_retrieve
  • authz_groupfile_getfns

関連データ構造

  • authz_group_provider
  • authz_filegroup_provider
  • AUTHZ_PROVIDER_GROUP（定数）
  • AUTHZ_PROVIDER_VERSION（定数）
  • AP_AUTH_INTERNAL_PER_CONF（定数）

失敗モードと境界条件

  • ap_register_auth_provider が失敗した場合、認可プロバイダーが利用できず、該当する Require ディレクティブの評価が失敗する可能性がある。

保守担当者向けメモ

  • 本関数はモジュール起動時に一度だけ実行される。
  • file-group 認可には mod_authz_owner が提供する関数が必要であり、ap_hook_optional_fn_retrieve を通じた取得が必須である。
