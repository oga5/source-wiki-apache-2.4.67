lua_open_callback

定義

  • 種別: function
  • 定義箇所: modules/lua/mod_lua.c:116

目的

Lua VM の初期化時に、Apache HTTP Server 関連のライブラリおよびモジュールを Lua インタプリタへロードし、実行環境をセットアップする。

シグネチャまたは形状

static void lua_open_callback(lua_State *L, apr_pool_t *p, void *ctx)

入力と出力

  • **入力**:
  • lua_State *L: 初期化対象の Lua 状態。
  • apr_pool_t *p: リソース割り当てに使用する APR プール。
  • void *ctx: 未使用（コンテキストポインタ）。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • lua_State *L は有効な Lua インタプリタインスタンスであること。
  • apr_pool_t *p は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. ap_lua_init(L, p) を呼び出し、Lua VM の基本初期化を行う。 2. ap_lua_load_apache2_lmodule(L) を呼び出し、Apache 2 関連の Lua モジュールをロードする。 3. ap_lua_load_request_lmodule(L, p) を呼び出し、リクエスト処理用の Lua モジュールをロードする。 4. ap_lua_load_config_lmodule(L) を呼び出し、設定関連の Lua モジュールをロードする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/mod_lua.c:126（直接呼び出し）
  • modules/lua/mod_lua.c:190（ap_lua_vm_spec 構造体のコールバックとして登録）
  • **呼び出し先**:
  • ap_lua_init
  • ap_lua_load_apache2_lmodule
  • ap_lua_load_request_lmodule
  • ap_lua_load_config_lmodule

関連データ構造

  • lua_State (Lua 標準)
  • apr_pool_t (APR)

失敗モードと境界条件

  • 内部で呼び出される各 ap_lua_load_* 関数が失敗した場合の挙動については、本関数内では明示的なエラーハンドリングを行っていない。

保守担当者向けメモ

  • 本関数は ap_lua_vm_spec のコールバックとして使用されるため、シグネチャを変更する場合は関連する構造体定義との整合性に注意すること。
  • 各ロード関数は Lua スタックを操作するため、ロード順序やスタックの状態に依存関係がある場合は注意が必要である。
