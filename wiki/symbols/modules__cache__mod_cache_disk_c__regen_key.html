regen_key

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:260

目的

Vary ヘッダーに基づくキャッシュキーの再生成を行います。リクエストヘッダーの値と既存のキーを結合し、特定の Vary 条件に対応する一意なキャッシュ識別子を構築します。

シグネチャまたは形状

static const char* regen_key(apr_pool_t *p, apr_table_t *headers,
                             apr_array_header_t *varray, const char *oldkey)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • headers: 参照するヘッダーテーブル。
  • varray: Vary 対象となるヘッダー名の配列。
  • oldkey: ベースとなる元のキャッシュキー。
  • **出力**:
  • 結合された新しいキー文字列を返します。

前提条件と不変条件

  • varray が指す配列内の各要素は、headers テーブル内に存在するヘッダー名である必要があります。
  • headers 内に指定されたヘッダーが存在しない場合、空文字列として扱われます。

アルゴリズムまたは振る舞い

1. varray の要素数から必要な iovec の数を計算し、メモリを確保します。 2. varray に含まれる各ヘッダー名について、headers テーブルから対応する値を取得します。 3. iovec 配列に「ヘッダー名」と「ヘッダー値」のペアを順次格納します。 4. 最後に oldkey を iovec に追加します。 5. apr_pstrcatv を使用して、すべての要素を連結した文字列を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c:477
  • modules/cache/mod_cache_disk.c:1047
  • modules/cache/mod_cache_socache.c:546
  • modules/cache/mod_cache_socache.c:876
  • **呼び出し先**:
  • apr_palloc (APR)
  • apr_table_get (APR)
  • strlen (標準ライブラリ)
  • apr_pstrcatv (APR)

関連データ構造

  • apr_table_t: ヘッダーの保持に使用。
  • apr_array_header_t: Vary ヘッダー名のリスト保持に使用。
  • struct iovec: 文字列連結のためのバッファ構造。

失敗モードと境界条件

  • headers に varray で指定されたキーが存在しない場合、空文字列として処理が継続されます。
  • メモリ不足時には apr_palloc 等の APR 関数が例外を発生させる可能性があります。
  • ヘッダー値の大文字・小文字の区別や、複数値を持つヘッダーの正規化は行われません。

保守担当者向けメモ

  • 本関数はヘッダー値の正規化を行わないため、キャッシュヒット率に影響を与える可能性があります。
  • RFC 2616 に準拠したヘッダー値の比較（大文字・小文字の区別など）の改善が検討されています。
  • mod_cache_socache.c からも呼び出されており、共通ユーティリティとしての側面を持ちます。
