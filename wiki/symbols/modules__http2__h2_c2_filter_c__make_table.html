make_table

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2_filter.c:496

目的

h2_response_parser が保持するヘッダー行の配列を解析し、Apache の apr_table_t 形式のヘッダーテーブルへ変換します。

シグネチャまたは形状

static apr_table_t *make_table(h2_response_parser *parser)

入力と出力

  • **入力**: parser (h2_response_parser *) - 解析対象のヘッダー行 (hlines) を含むパーサーコンテキスト。
  • **出力**: apr_table_t * - 生成されたヘッダーテーブル。解析失敗時は NULL を返します。

前提条件と不変条件

  • parser->pool は有効なメモリプールである必要があります。
  • parser->hlines が NULL の場合、空のテーブルが返されます。

アルゴリズムまたは振る舞い

1. parser->hlines が存在する場合、その要素数で apr_table_make を呼び出しテーブルを初期化します。 2. 各ヘッダー行に対して以下の処理を行います。

  • : を区切り文字としてヘッダー名と値に分割します。
  • : が見つからない場合はエラーログを出力し、NULL を返して処理を中断します。
  • 値の先頭にある空白文字（スペース、タブ）をスキップします。
  • h2_util_ignore_resp_header を使用して、無視すべきヘッダーでないことを確認します。
  • apr_table_merge を使用して、ヘッダー名と値をテーブルに追加します。

    • 3. parser->hlines が存在しない場合は、要素数 0 のテーブルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c2_filter.c:535 (推測される箇所)
  • **呼び出し先**:
  • apr_table_make
  • ap_strchr
  • ap_log_cerror
  • h2_util_ignore_resp_header
  • apr_table_merge

関連データ構造

  • h2_response_parser: レスポンス解析の状態を保持する構造体。
  • apr_table_t: Apache のヘッダー管理用テーブル構造。

失敗モードと境界条件

  • ヘッダー行に : が含まれていない場合、APR_EINVAL エラーとして NULL を返します。
  • h2_util_ignore_resp_header が真を返すヘッダーはテーブルに追加されません。

保守担当者向けメモ

  • 本関数は h2_c2_filter におけるレスポンス解析の最終段階で使用されます。
  • apr_table_merge を使用しているため、同じ名前のヘッダーが複数ある場合は値が結合されます。
