make_cookie_dir

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_usertrack.c:281

目的

ディレクトリごとの設定構造体 cookie_dir_rec を初期化し、デフォルトの Cookie 名に基づいた検索用正規表現を準備する。

シグネチャまたは形状

static void *make_cookie_dir(apr_pool_t *p, char *d)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定割り当て用のメモリプール。
  • char *d: ディレクトリパス（本関数内では未使用）。
  • **出力**:
  • void *: 初期化された cookie_dir_rec 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体の各フィールドはゼロ初期化される。
  • COOKIE_NAME マクロがデフォルト値として使用される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して cookie_dir_rec 構造体分のメモリを確保する。 2. cookie_name を COOKIE_NAME で初期化する。 3. style を CT_UNSET に設定する。 4. set_and_comp_regexp を呼び出し、デフォルトの COOKIE_NAME を用いて検索用の正規表現をコンパイルする。 5. 初期化済みの構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_usertrack.c:498 （ディレクトリ設定作成ハンドラとして登録）。
  • **呼び出し先**: set_and_comp_regexp。

関連データ構造

  • cookie_dir_rec: 本関数で生成されるディレクトリごとの設定保持構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボートする可能性がある（Apache の標準的なメモリ管理動作）。
  • set_and_comp_regexp が失敗した場合の挙動については、当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。
  • cookie_dir_rec の各フィールド（enabled, cookie_domain, samesite, is_secure, is_httponly）は apr_pcalloc により自動的に無効値（0/NULL）で初期化される。
