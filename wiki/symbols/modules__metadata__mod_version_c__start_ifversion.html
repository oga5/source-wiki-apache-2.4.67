start_ifversion

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_version.c:172

目的

Apache HTTP Server の設定ファイル内における <IfVersion> コンテナのディレクティブハンドラとして、実行中のサーバーバージョンに基づいた設定の有効化またはスキップを制御します。

シグネチャまたは形状

static const char *start_ifversion(cmd_parms *cmd, void *mconfig,
                                   const char *arg1, const char *arg2,
                                   const char *arg3)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • mconfig: 設定を格納するコンテナポインタ。
  • arg1, arg2, arg3: ディレクティブに渡された引数（演算子およびバージョン文字列）。
  • **出力**:
  • 成功時: NULL を返し、mconfig に設定構造体を設定。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • EXEC_ON_READ フラグにより、設定ファイルの読み込み時に即座に実行されます。
  • 初回実行時に ap_get_server_revision() を呼び出し、サーバーバージョンを httpd_version にキャッシュします。
  • 引数の末尾には > が必要であり、不足している場合はエラーを返します。

アルゴリズムまたは振る舞い

1. **引数解析**: 引数の組み合わせから演算子（=, <, >, ~）とバージョン文字列を特定します。! による条件反転をサポートします。 2. **バージョン比較**:

  • 数値比較: compare_version を使用し、メジャー・マイナー・パッチレベルを評価します。
  • 正規表現比較: match_version を使用し、バージョン文字列をパターンマッチングします。

    • 3. **コンテナ制御**:

  • 条件が真の場合: ap_build_cont_config() を呼び出し、ブロック内の設定を読み込みます。
  • 条件が偽の場合: ap_soak_end_container() を呼び出し、ブロック内の設定を無視します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_version.c:299 において AP_INIT_TAKE123 マクロにより <IfVersion> ディレクティブのハンドラとして登録されています。
  • **呼び出し先**:
  • compare_version (数値比較)
  • match_version (正規表現比較)
  • ap_get_server_revision (バージョン取得)
  • ap_build_cont_config (設定読み込み)
  • ap_soak_end_container (設定スキップ)

関連データ構造

  • cmd_parms: 設定コマンドのコンテキスト。
  • ap_directive_t: 設定ディレクティブのツリー構造。
  • httpd_version: サーバーバージョンを保持するグローバル変数。

失敗モードと境界条件

  • **引数エラー**: 演算子の欠落、閉じ括弧 > の不足、または認識できない演算子が指定された場合にエラーメッセージを返します。
  • **比較エラー**: compare_version または match_version がエラーを返した場合、そのエラーを伝播します。

保守担当者向けメモ

  • 本関数は設定読み込み時にのみ動作し、リクエスト処理時のパフォーマンスには影響しません。
  • 演算子の前に ! を付与することで条件を反転させるロジックは、reverse フラグと match 変数の論理演算によって制御されています。
