xbithack_t

定義

  • 種別: enum
  • 定義箇所: modules/filters/mod_include.c:108

目的

SSI（Server Side Includes）機能における XBitHack ディレクティブの設定状態を表現するために使用されます。

シグネチャまたは形状

typedef enum {
    XBITHACK_OFF,
    XBITHACK_ON,
    XBITHACK_FULL,
    XBITHACK_UNSET
} xbithack_t;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: XBITHACK_OFF, XBITHACK_ON, XBITHACK_FULL, XBITHACK_UNSET のいずれかの値。

前提条件と不変条件

  • include_dir_config 構造体内で、ディレクトリ単位の設定値として保持されます。

アルゴリズムまたは振る舞い

  • XBITHACK_OFF: SSI 処理を無効化します。
  • XBITHACK_ON: 実行ビットがセットされたファイルに対して SSI 処理を有効化します。
  • XBITHACK_FULL: XBITHACK_ON の機能に加え、ファイルの最終更新時刻（Last-Modified）を適切に更新します。
  • XBITHACK_UNSET: 設定が未指定であることを示します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_include.c 内の include_dir_config 構造体（119行目）でメンバ変数 xbithack の型として参照されています。
  • 呼び出し先: 提供されたソース情報からは、この列挙型自体が関数を呼び出すことはありません。

関連データ構造

  • include_dir_config: この列挙型をメンバとして保持する、ディレクトリ単位の設定構造体。

失敗モードと境界条件

  • XBITHACK_UNSET は、設定が明示的に行われていない初期状態を表すために使用されます。

保守担当者向けメモ

  • 本シンボルは mod_include の設定管理において重要な役割を果たします。設定値の追加や変更を行う際は、include_dir_config 構造体および関連する設定解析ロジックとの整合性に注意してください。
