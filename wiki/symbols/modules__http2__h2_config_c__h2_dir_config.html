h2_dir_config

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_config.c:86

目的

Apache HTTP Server のディレクトリ単位（request_rec スコープ）における HTTP/2 プロトコルの設定値を保持・管理する。

シグネチャまたは形状

typedef struct h2_dir_config {
    const char *name;
    int h2_upgrade;                  /* HTTP/1 から h2/h2c へのアップグレード許可 */
    int h2_push;                     /* HTTP/2 Server Push の有効化状態 */
    apr_array_header_t *push_list;   /* h2_push_res 設定のリスト */
    apr_table_t *early_headers;      /* 103 レスポンス用 HTTP ヘッダー */
    int early_hints;                 /* 103 ステータスコードのサポート */
    apr_interval_time_t stream_timeout;/* ストリームのタイムアウト値 */
} h2_dir_config;

入力と出力

本構造体は直接的な関数ではないが、設定の解決プロセスにおいて以下の役割を果たす。

  • **入力**: Apache の設定ディレクティブを通じて値が設定される。
  • **出力**: h2_config_rget 等のゲッター関数を介して、実行時のストリーム処理やセッション管理へ設定値を提供する。

前提条件と不変条件

  • すべてのインスタンスは Apache のメモリプール（apr_pool_t）上で確保される。
  • マージ処理（h2_config_merge_dir）において、未設定値（マジックナンバー -1）を持つフィールドは、親コンテキストの設定値で上書きされる。

アルゴリズムまたは振る舞い

  • **階層的解決**: h2_config_rget を通じてリクエストコンテキストから取得され、ディレクトリ設定が存在しない場合はサーバー設定へフォールバックする。
  • **メモリ管理**: Apache の設定フェーズで生成され、リクエストのライフサイクルに準じて破棄される。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_config_create_dir（初期化）、h2_config_merge_dir（マージ）、h2_config_rget（取得）、および各ディレクティブハンドラ。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • h2_config: サーバーレベルの設定を保持する構造体。
  • apr_array_header_t: push_list の保持に使用。
  • apr_table_t: early_headers の保持に使用。

失敗モードと境界条件

  • **未設定状態**: 各整数フィールドが -1 の場合、設定が未定義であることを示す。
  • **メモリ不足**: apr_pcalloc によるメモリ確保失敗時は、Apache の標準的なエラーハンドリングに従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本構造体は h2_config.c 内の defdconf（デフォルト値）と対で使用される。
  • フィールドを追加した際は、h2_config_merge_dir におけるマージロジックを更新し、親コンテキストからの継承が正しく行われるようにすること。
