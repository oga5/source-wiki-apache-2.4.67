ping_reuse_session

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_proxy_session.c:155

目的

既存の HTTP/2 プロキシセッションにおいて、最後にフレームを受信してからの経過時間に基づき、接続の健全性を確認するための PING 状態への遷移を判断する。

シグネチャまたは形状

static void ping_reuse_session(h2_proxy_session *session)

入力と出力

  • **入力**: session (h2_proxy_session *) - 操作対象のプロキシセッション構造体。
  • **出力**: なし（副作用としてセッションの状態を更新）。

前提条件と不変条件

  • session は有効なポインタであること。
  • session->ping_state が H2_PING_ST_NONE である場合にのみ、PING 状態への遷移が検討される。

アルゴリズムまたは振る舞い

1. セッションの現在の ping_state が H2_PING_ST_NONE であるかを確認する。 2. apr_time_now() と session->last_frame_received の差分から、前回のフレーム受信からの経過時間（age）を算出する。 3. age が 1 秒を超えている場合、enter_ping_state を呼び出し、セッションを H2_PING_ST_AWAIT_PING 状態へ遷移させる。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c:790
  • **呼び出し先**: enter_ping_state

関連データ構造

  • h2_proxy_session: セッションの生存確認状態（ping_state）および最終受信時刻（last_frame_received）を保持する構造体。

失敗モードと境界条件

  • last_frame_received が初期化されていない、あるいは極端に古い値である場合、即座に PING 状態へ遷移する可能性がある。
  • ping_state が H2_PING_ST_NONE 以外の場合、本関数は何も行わない。

保守担当者向けメモ

  • 本関数は 1 秒という固定のしきい値を使用している。環境や要件に応じてタイムアウト値を調整する必要がある場合は注意すること。
  • enter_ping_state への依存があるため、状態遷移のロジックを変更する際は両方の関数を確認すること。
