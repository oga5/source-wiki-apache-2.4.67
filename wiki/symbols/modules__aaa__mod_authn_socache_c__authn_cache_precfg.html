authn_cache_precfg

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:70

目的

モジュールの初期化フェーズにおいて、キャッシュ用ミューテックスの登録と、共有オブジェクトキャッシュ（socache）プロバイダーのバインドを行う。

シグネチャまたは形状

static int authn_cache_precfg(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptmp)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • plog: ログ用メモリプール。
  • ptmp: 一時的な作業用メモリプール。
  • **出力**:
  • 成功時は OK を返す。
  • ミューテックス登録失敗時は 500 を返す。

前提条件と不変条件

  • ap_mutex_register が正常に完了し、グローバルミューテックスがシステムに登録される必要がある。
  • AP_SOCACHE_DEFAULT_PROVIDER が利用可能であること。

アルゴリズムまたは振る舞い

1. ap_mutex_register を呼び出し、authn_cache_id に基づくグローバルミューテックスを登録する。 2. 登録に失敗した場合はエラーログを出力し、500 を返して終了する。 3. ap_lookup_provider を使用して、デフォルトの socache プロバイダーを検索し、グローバル変数 socache_provider に格納する。 4. モジュールの設定状態を示す configured フラグを 0 に初期化する。 5. OK を返して正常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_pre_config により、Apache の設定読み込みフェーズで呼び出される（modules/aaa/mod_authn_socache.c:460）。
  • **呼び出し先**:
  • ap_mutex_register
  • ap_log_perror
  • ap_lookup_provider

関連データ構造

  • socache_provider: 選択されたキャッシュバックエンドの実装を保持するグローバル変数。
  • configured: モジュールの設定状態を管理するフラグ。

失敗モードと境界条件

  • ap_mutex_register が APR_SUCCESS 以外の値を返した場合、初期化は中断され、HTTP 500 エラーとして処理される。

保守担当者向けメモ

  • 本関数はモジュールライフサイクルの初期段階で実行されるため、ここで設定されたプロバイダーやミューテックスは後続の post_config やリクエスト処理で利用される。
  • configured フラグの初期化は、設定の重複や不整合を防ぐためのガードとして機能する。
