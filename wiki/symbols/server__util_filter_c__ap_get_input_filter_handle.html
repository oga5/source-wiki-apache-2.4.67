ap_get_input_filter_handle

定義

  • 種別: function
  • 定義箇所: server/util_filter.c:198

目的

指定された名前の入力フィルタに対応するフィルタ定義（ap_filter_rec_t）を取得します。

シグネチャまたは形状

ap_filter_rec_t *ap_get_input_filter_handle(const char *name)

入力と出力

  • **入力**: name (フィルタ名を表す文字列)
  • **出力**: 該当する ap_filter_rec_t 構造体へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • 内部的に registered_input_filters（入力フィルタを管理するトライ木）が初期化されている必要があります。
  • フィルタ名は、ap_register_input_filter を通じて事前に登録されている必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った name を使用します。 2. 内部関数 get_filter_handle を呼び出し、グローバルな registered_input_filters トライ木から対象のフィルタ定義を検索します。 3. 検索結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • include/util_filter.h:431 (宣言)
  • modules/cache/mod_cache.c:527, 603
  • modules/lua/lua_request.c:1412
  • server/util_filter.c:491
  • **呼び出し先**:
  • get_filter_handle (内部関数)

関連データ構造

  • ap_filter_rec_t: フィルタの定義を保持する構造体。
  • registered_input_filters: 入力フィルタを管理するトライ木構造のルート。

失敗モードと境界条件

  • 指定された name が登録されていない場合、get_filter_handle は NULL を返し、本関数も NULL を返します。
  • name に NULL が渡された場合の挙動は、内部の get_filter_handle の実装に依存します。

保守担当者向けメモ

  • 本関数は registered_input_filters を読み取り専用で参照します。
  • フィルタ名の検索はトライ木で行われるため、フィルタ数が増加しても効率的な名前解決が維持されます。
