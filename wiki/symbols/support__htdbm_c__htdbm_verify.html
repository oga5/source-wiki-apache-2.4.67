htdbm_verify

定義

  • 種別: function
  • 定義箇所: support/htdbm.c:190

目的

DBM データベース内の既存ユーザーのパスワードを、提供された入力パスワードと比較して検証します。

シグネチャまたは形状

apr_status_t htdbm_verify(htdbm_t *htdbm)

入力と出力

  • **入力**: htdbm_t *htdbm（データベース接続情報および検証対象のユーザー名とパスワードを含む構造体）。
  • **出力**: apr_status_t 型のステータスコード。
  • APR_SUCCESS: 検証成功。
  • APR_ENOENT: ユーザーが存在しない、または取得に失敗。
  • その他: apr_password_validate が返す検証結果（パスワード不一致など）。

前提条件と不変条件

  • htdbm->dbm は既に htdbm_open 等で有効な状態に開かれている必要があります。
  • htdbm->username および htdbm->ctx.passwd が適切に設定されている必要があります。
  • メモリ割り当ては htdbm->ctx.pool を使用して行われます。

アルゴリズムまたは振る舞い

1. htdbm->username をキーとして apr_dbm_exists を呼び出し、ユーザーの存在を確認します。存在しない場合は APR_ENOENT を返します。 2. apr_dbm_fetch を使用してデータベースから値（ハッシュ化されたパスワード文字列）を取得します。 3. 取得した値からコロン（:）を探し、コメント部分を除去してパスワード文字列のみを抽出します。 4. 抽出したパスワードと htdbm->ctx.passwd を apr_password_validate に渡し、ハッシュアルゴリズムに基づいた比較を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htdbm.c:422
  • **呼び出し先**:
  • apr_dbm_exists
  • apr_dbm_fetch
  • apr_pstrndup / apr_pstrdup
  • apr_password_validate
  • strchr

関連データ構造

  • htdbm_t: データベース接続および認証コンテキストを保持するメイン構造体。
  • apr_datum_t: DBM のキーおよび値を保持する APR のデータ構造。

失敗モードと境界条件

  • ユーザー名がデータベースに存在しない場合、APR_ENOENT を返します。
  • apr_dbm_fetch が失敗した場合、APR_ENOENT を返します。
  • パスワードが一致しない場合、apr_password_validate の戻り値（エラーコード）がそのまま返されます。
  • コロンが含まれないレコード形式の場合、値全体をパスワードとして扱います。

保守担当者向けメモ

  • 本関数は passwd_common.c で定義された apr_password_validate に依存しており、ハッシュアルゴリズムの詳細はそちらで管理されています。
  • DBM から取得したデータは apr_pstrndup でプールにコピーされるため、呼び出し元での明示的なメモリ解放は不要です。
