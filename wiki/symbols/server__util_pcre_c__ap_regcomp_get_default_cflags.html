ap_regcomp_get_default_cflags

定義

  • 種別: function
  • 定義箇所: server/util_pcre.c:163

目的

正規表現コンパイル時に適用されるグローバルなデフォルトフラグ（default_cflags）を取得します。

シグネチャまたは形状

int ap_regcomp_get_default_cflags(void)

入力と出力

  • 入力: なし
  • 出力: 現在設定されているデフォルトのコンパイルフラグ（整数値）を返します。

前提条件と不変条件

  • default_cflags は、ap_regcomp_set_default_cflags によって事前に設定されている必要があります。

アルゴリズムまたは振る舞い

  • 内部変数 default_cflags の値をそのまま返却する単純なゲッター関数です。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • modules/filters/mod_substitute.c:672
  • server/core.c:3002
  • server/util_regex.c:97
  • 呼び出し先: なし（本関数は他の関数を呼び出しません）。

関連データ構造

  • default_cflags: server/util_pcre.c 内で保持される静的変数。

失敗モードと境界条件

  • 失敗モード: なし。
  • 境界条件: 設定が変更されていない場合、初期値（通常は 0）を返します。

保守担当者向けメモ

  • 本関数は include/ap_regex.h で宣言されており、HTTPD の正規表現コンパイル設定の一貫性を保つために使用されます。
  • 戻り値のフラグは AP_REG_ で始まる定数とビット演算されることが多いため、変更時は互換性に注意してください。
