ap_process_connection

定義

  • 種別: function
  • 定義箇所: server/connection.c:210

目的

HTTP 接続のライフサイクルにおけるメインエントリポイントとして、接続の初期化、バーチャルホストの特定、およびプロトコル処理の実行を管理します。

シグネチャまたは形状

void ap_process_connection(conn_rec *c, void *csd)

入力と出力

  • **入力**:
  • c: 接続状態を保持する conn_rec 構造体へのポインタ。
  • csd: 接続ソケット記述子（Client Socket Descriptor）。
  • **出力**: なし（void 型）。

前提条件と不変条件

  • c は有効な conn_rec 構造体を指している必要があります。
  • 呼び出し時点で、MPM（Multi-Processing Module）によってソケットが受け入れられている必要があります。

アルゴリズムまたは振る舞い

1. ap_update_vhost_given_ip(c) を呼び出し、接続の IP アドレスに基づいて適切なバーチャルホスト設定を特定します。 2. ap_pre_connection(c, csd) を呼び出し、接続の初期化フックを実行します。 3. 接続が中断（c->aborted）されていないことを確認します。 4. ap_run_process_connection(c) を呼び出し、登録されたプロトコルハンドラによるリクエスト処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_workers.c
  • server/mpm/mpmt_os2/mpmt_os2_child.c
  • server/mpm/netware/mpm_netware.c
  • server/mpm/prefork/prefork.c
  • server/mpm/winnt/child.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • ap_update_vhost_given_ip
  • ap_pre_connection
  • ap_run_process_connection

関連データ構造

  • conn_rec: 接続ごとの状態を管理する構造体。

失敗モードと境界条件

  • c->aborted が真の場合、ap_run_process_connection はスキップされます。これは、ap_pre_connection などの初期化フェーズでエラーが発生した場合に発生します。

保守担当者向けメモ

  • 本関数は MPM と Apache コアを繋ぐ重要なインターフェースです。接続処理のフローを変更する場合は、ap_pre_connection フックや ap_run_process_connection フックの順序に注意してください。
  • 呼び出し元は主に各 MPM 実装です。新しい MPM を追加する際は、本関数を適切に呼び出す必要があります。
