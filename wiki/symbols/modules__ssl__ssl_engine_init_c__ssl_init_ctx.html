ssl_init_ctx

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_init.c:1375

目的

SSL/TLS コンテキスト（SSL_CTX）の構築におけるメインパイプラインとして機能し、プロトコル設定から証明書チェーン、TLS 拡張に至るまでの一連の初期化処理を順次実行します。

シグネチャまたは形状

static apr_status_t ssl_init_ctx(server_rec *s,
                                 apr_pool_t *p,
                                 apr_pool_t *ptemp,
                                 modssl_ctx_t *mctx)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p: 長期生存するメモリプール。
  • ptemp: 一時的なメモリプール。
  • mctx: 初期化対象の modssl_ctx_t 構造体。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS を返し、いずれかの初期化ステップでエラーが発生した場合はそのエラーコードを返します。

前提条件と不変条件

  • mctx は呼び出し前に適切に割り当てられている必要があります。
  • mctx->pks が設定されている場合、証明書チェーンおよび TLS 拡張の初期化が試行されます。

アルゴリズムまたは振る舞い

以下の順序で初期化を実行します。いずれかのステップでエラーが発生した場合、即座にそのエラーを返して終了します。

1. **プロトコル設定**: ssl_init_ctx_protocol を呼び出し、TLS バージョン等を制御。 2. **セッションキャッシュ**: ssl_init_ctx_session_cache を実行。 3. **コールバック**: ssl_init_ctx_callbacks を実行。 4. **検証設定**: ssl_init_ctx_verify を呼び出し、CA 証明書パス等を設定。 5. **暗号スイート**: ssl_init_ctx_cipher_suite を呼び出し、暗号スイートを適用。 6. **CRL**: ssl_init_ctx_crl を実行。 7. **証明書と拡張**: mctx->pks が存在する場合、ssl_init_ctx_cert_chain を実行し、HAVE_TLSEXT が定義されていれば ssl_init_ctx_tls_extensions を実行。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_init.c:2081
  • modules/ssl/ssl_engine_init.c:2141
  • **呼び出し先**:
  • ssl_init_ctx_protocol
  • ssl_init_ctx_session_cache
  • ssl_init_ctx_callbacks
  • ssl_init_ctx_verify
  • ssl_init_ctx_cipher_suite
  • ssl_init_ctx_crl
  • ssl_init_ctx_cert_chain
  • ssl_init_ctx_tls_extensions (条件付き)

関連データ構造

  • server_rec: Apache サーバー設定構造体。
  • modssl_ctx_t: SSL モジュール固有のコンテキスト構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • 各サブ関数（ssl_init_ctx_*）が APR_SUCCESS 以外の値を返した場合、そのエラーを伝播させて即座に停止します。
  • mctx->pks が NULL の場合、証明書チェーンおよび TLS 拡張の初期化はスキップされます。

保守担当者向けメモ

  • 本関数は初期化のパイプラインを定義するものであり、個別の設定ロジックは各サブ関数に委譲されています。
  • HAVE_TLSEXT マクロによる条件付きコンパイルが含まれており、環境によって TLS 拡張の初期化有無が変化します。
  • プロキシサポートに関するコメント（XXX: proxy support?）が mctx->pks ブロック内に存在します。
