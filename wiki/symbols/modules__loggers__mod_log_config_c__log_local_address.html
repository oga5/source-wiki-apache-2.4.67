log_local_address

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_config.c:333

目的

リクエストに関連付けられたローカル（サーバー側）のIPアドレスをログ出力用に取得します。

シグネチャまたは形状

static const char *log_local_address(request_rec *r, char *a)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報を保持する request_rec 構造体へのポインタ。
  • a: 未使用の引数（ログフォーマットの引数文字列）。
  • **出力**:
  • 成功時: ローカルIPアドレスを示す文字列（const char *）。
  • 失敗時: 呼び出し元に依存しますが、r->connection->local_ip が未設定の場合は NULL または空文字列の可能性があります。

前提条件と不変条件

  • r および r->connection が有効なメモリ領域を指している必要があります。
  • 本関数は、リクエスト処理のログ記録フェーズで呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. r->connection 構造体へアクセスします。 2. 保持されている local_ip メンバの値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/loggers/mod_log_config.c:1737: log_pfn_register を介してログフォーマット項目 "A" として登録。
  • server/log.c:947: ap_register_errorlog_handler を介してエラーログハンドラ "A" として登録。
  • **呼び出し先**:
  • 外部関数呼び出しはありません。

関連データ構造

  • request_rec: Apache のリクエスト処理における中心的な構造体。
  • conn_rec (r->connection): 接続情報を保持する構造体。

失敗モードと境界条件

  • r が NULL の場合、セグメンテーション違反が発生します。
  • 接続が確立されていない、あるいは local_ip が初期化されていない状態での動作は、r->connection の状態に依存します。

保守担当者向けメモ

  • 本関数はログフォーマットのハンドラとして登録されており、フォーマット文字列中の %A に対応します。
  • r->connection->local_ip は Apache の接続処理フェーズで設定されるため、本関数自体は値を生成せず、既存の値を参照するのみです。
