bucketeer_filter_config_t

定義

  • 種別: struct
  • 定義箇所: modules/debugging/mod_bucketeer.c:37

目的

mod_bucketeer フィルタがデータストリーム内のバケット分割や制御を行う際に使用する、デリミタ（区切り文字）設定を保持するための構造体です。

シグネチャまたは形状

typedef struct bucketeer_filter_config_t
{
    char bucketdelimiter;
    char passdelimiter;
    char flushdelimiter;
} bucketeer_filter_config_t;

入力と出力

  • **入力**: 設定値として3つの char 型メンバを保持します。
  • **出力**: フィルタ処理中に各デリミタが参照され、バケットの分割、ブリゲードの送信、フラッシュのトリガー判定に使用されます。

前提条件と不変条件

  • mod_bucketeer の設定フェーズにおいて、apr_pcalloc 等でメモリ確保された後に各メンバが初期化されることが想定されています。

アルゴリズムまたは振る舞い

  • bucketdelimiter: データ内で検出された場合にバケットを分割します。
  • passdelimiter: データ内で検出された場合に現在のブリゲードを下流へ送信します。
  • flushdelimiter: データ内で検出された場合にフラッシュバケットを挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/debugging/mod_bucketeer.c:47（設定生成時）
  • modules/debugging/mod_bucketeer.c:67（フィルタ実行時の設定取得）
  • **呼び出し先**: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • bucketeer_ctx_t: フィルタ実行中の状態（ブリゲード等）を保持するコンテキスト。

失敗モードと境界条件

  • デリミタがバイナリデータと重複する場合、意図しないタイミングでバケットの分割やフラッシュが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_bucketeer の設定保持専用です。
  • フィルタの挙動を変更する際は、この構造体のメンバが各デリミタとして正しく解釈されているかを確認してください。
