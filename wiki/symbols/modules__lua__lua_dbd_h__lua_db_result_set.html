lua_db_result_set

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_dbd.h:39

目的

データベースクエリの実行結果を保持し、Lua スクリプトから結果セットの行や列データへアクセスするためのラッパー構造体です。

シグネチャまたは形状

typedef struct {
    const apr_dbd_driver_t  *driver;
    int                     rows;
    int                     cols;
    apr_dbd_results_t       *results;
    apr_pool_t              *pool;
} lua_db_result_set;

入力と出力

  • **入力**: データベースクエリ実行時に lua_newuserdata を介して Lua スタック上に生成されます。
  • **出力**: lua_get_result_set を通じて、Lua スクリプトから C 側のデータ構造として参照されます。

前提条件と不変条件

  • pool メンバは、結果セットの生存期間を管理するために使用されます。
  • results は apr_dbd_results_t を指しており、有効なクエリ結果を保持している必要があります。

アルゴリズムまたは振る舞い

1. **生成**: lua_dbd.c 内の関数（lua_db_select 等）において、lua_newuserdata を使用して Lua VM 上にメモリが確保されます。 2. **保持**: クエリ実行後の結果（apr_dbd_results_t）と、ドライバ情報、行数、列数が構造体に格納されます。 3. **解放**: Lua のガベージコレクションによって管理され、関連するリソースは pool の破棄に伴い解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_dbd.c 内の各データベース操作関数（lua_get_result_set 等）が、この構造体を生成または参照します。
  • **呼び出し先**: apr_dbd ライブラリの関数群（apr_dbd_results_t を操作する関数）が、この構造体内の results メンバを介して呼び出されます。

関連データ構造

  • apr_dbd_driver_t: データベースドライバの定義。
  • apr_dbd_results_t: APR が提供する低レベルなクエリ結果セット。
  • apr_pool_t: メモリ管理用プール。
  • lua_db_handle: データベース接続を管理する構造体。

失敗モードと境界条件

  • lua_newuserdata によるメモリ確保失敗時には、Lua VM がエラーをスローします。
  • クエリ実行時に結果が空である場合、rows や cols は 0 となり、results は無効な状態となる可能性があります。

保守担当者向けメモ

  • 本構造体は lua_dbd.c で定義される Lua ユーザーデータとして扱われます。
  • apr_pool_t を保持しているため、メモリリークを防ぐには Lua 側での適切なガベージコレクションの実行が不可欠です。
  • 構造体の各メンバは apr_dbd の仕様に強く依存しているため、バックエンドの変更時には整合性に注意してください。
