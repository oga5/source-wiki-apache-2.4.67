htdbm_usage

定義

  • 種別: function
  • 定義箇所: support/htdbm.c:276

目的

htdbm コマンドの利用方法（Usage）を標準エラー出力に表示し、プログラムをエラー終了させる。

シグネチャまたは形状

void htdbm_usage(void)

入力と出力

  • **入力**: なし（引数を取らない）。
  • **出力**: stderr に対してコマンドの構文、オプション一覧、および使用例を出力する。戻り値はない。

前提条件と不変条件

  • 本関数が呼び出されると、プログラムは必ず終了する。
  • BCRYPT_DEFAULT_COST が定義されている必要がある。

アルゴリズムまたは振る舞い

1. fprintf を使用して、htdbm のコマンドライン引数パターン（-b, -n, -v, -x, -l 等の組み合わせ）と各オプションの説明を標準エラー出力に書き出す。 2. BCRYPT_DEFAULT_COST の値をオプション説明の一部として埋め込む。 3. exit(ERR_SYNTAX) を呼び出し、構文エラーを示す終了コードでプロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htdbm.c:391（main 関数内の引数解析エラー時など）。
  • **呼び出し先**: fprintf (標準ライブラリ), exit (標準ライブラリ)。

関連データ構造

  • なし。

失敗モードと境界条件

  • 本関数自体は失敗しない。
  • stderr への書き込みが失敗した場合、プロセスは終了するが、エラーメッセージが正しく表示されない可能性がある。

保守担当者向けメモ

  • オプションの追加や変更を行った際は、本関数内のヘルプメッセージを必ず更新すること。
  • ERR_SYNTAX は htdbm.c 内で定義された終了ステータスコードである。
