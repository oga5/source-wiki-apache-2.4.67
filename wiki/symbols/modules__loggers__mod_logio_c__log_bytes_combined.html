log_bytes_combined

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_logio.c:116

目的

現在の接続における入出力バイト数の合計を算出し、ログ出力用に文字列形式で返します。

シグネチャまたは形状

static const char *log_bytes_combined(request_rec *r, char *a)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • a: ログフォーマットの引数（本関数では未使用）。
  • **出力**:
  • 接続の受信バイト数と送信バイト数の合計値を表す文字列（apr_off_t_toa による変換結果）。

前提条件と不変条件

  • r->connection->conn_config に logio_module に関連付けられた logio_config_t 構造体が格納されている必要があります。
  • cf->bytes_out および cf->bytes_in は、接続の生存期間中に蓄積された有効なバイト数である必要があります。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在の接続設定から logio_config_t 構造体を取得します。 2. 構造体内の bytes_out（送信バイト数）と bytes_in（受信バイト数）を加算します。 3. apr_off_t_toa を呼び出し、加算結果をリクエストプール (r->pool) 上の文字列に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_logio.c:199 にて log_pfn_register を通じてログフォーマットトークン "S" に登録されています。
  • **呼び出し先**:
  • ap_get_module_config (Apache API)
  • apr_off_t_toa (APR API)

関連データ構造

  • logio_config_t: 接続ごとの入出力バイト数を保持する構造体。
  • request_rec: リクエスト情報を保持する Apache コア構造体。

失敗モードと境界条件

  • conn_config が取得できない場合、あるいは logio_config_t が初期化されていない場合、NULL ポインタ参照によるクラッシュの可能性があります。
  • バイト数が 0 の場合、"0" という文字列が返されます。

保守担当者向けメモ

  • 本関数は mod_log_config のログフォーマットハンドラとして機能します。
  • 戻り値の文字列はリクエストプール (r->pool) に割り当てられるため、リクエスト終了時に自動的に解放されます。メモリリークの心配はありません。
