match_version

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_version.c:142

目的

実行中のサーバーバージョン文字列が、指定された正規表現パターンに適合するかを判定します。

シグネチャまたは形状

static int match_version(apr_pool_t *pool, char *version_string, const char **error)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する APR プール。
  • version_string: 比較対象となる正規表現パターン（文字列）。
  • error: エラー発生時にエラーメッセージを格納するポインタ。
  • **出力**:
  • 戻り値: 適合した場合は 1、適合しない場合やコンパイルエラー時は 0 を返します。

前提条件と不変条件

  • httpd_version グローバル変数が初期化されている必要があります。
  • version_string は ap_pregcomp でコンパイル可能な正規表現である必要があります。

アルゴリズムまたは振る舞い

1. ap_pregcomp を使用し、version_string を拡張正規表現としてコンパイルします。失敗した場合は error にメッセージをセットし 0 を返します。 2. 現在のサーバーバージョン（メジャー・マイナー・パッチ・追加文字列）を apr_psprintf で連結し、比較対象文字列 to_match を生成します。 3. ap_regexec を使用して、to_match が正規表現に適合するか判定します。 4. ap_pregfree を呼び出し、コンパイルした正規表現オブジェクトを解放します。 5. 適合結果（真偽値）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_version.c:241 (start_ifversion 関数内)
  • **呼び出し先**:
  • ap_pregcomp (Apache API)
  • apr_psprintf (APR API)
  • ap_regexec (Apache API)
  • ap_pregfree (Apache API)

関連データ構造

  • ap_regex_t: 正規表現コンパイル結果を保持する構造体。
  • httpd_version: サーバーのバージョン情報を保持するグローバル構造体。

失敗モードと境界条件

  • 正規表現のコンパイルに失敗した場合、error に文字列が代入され、関数は 0 を返します。
  • apr_psprintf による文字列生成がメモリ不足等で失敗した場合の挙動は、APR の仕様に依存します。

保守担当者向けメモ

  • 本関数は設定読み込み時に実行されるため、正規表現のコンパイルと解放を毎回行います。頻繁な呼び出しが発生する環境では、パフォーマンスへの影響を考慮してください。
