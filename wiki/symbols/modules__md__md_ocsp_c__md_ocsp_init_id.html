md_ocsp_init_id

定義

  • 種別: function
  • 定義箇所: modules/md/md_ocsp.c:131

目的

指定された証明書（md_cert_t）から SHA-1 ダイジェストを生成し、識別子（md_data_t）として初期化します。

シグネチャまたは形状

apr_status_t md_ocsp_init_id(md_data_t *id, apr_pool_t *p, const md_cert_t *cert)

入力と出力

  • **入力**:
  • id: 初期化対象の md_data_t 構造体へのポインタ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • cert: 識別子生成元の md_cert_t。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は APR_EGENERAL を返します。
  • id: 内部データが証明書の SHA-1 ダイジェストで更新されます。

前提条件と不変条件

  • cert は有効な md_cert_t オブジェクトである必要があります。
  • id は関数呼び出し前に md_data_null(id) によって初期化（リセット）されます。

アルゴリズムまたは振る舞い

1. md_cert_get_X509(cert) を介して OpenSSL の X509 オブジェクトを取得します。 2. X509_digest を使用し、EVP_sha1() アルゴリズムで証明書のダイジェストを計算します。 3. 計算に失敗した場合は APR_EGENERAL を返します。 4. 成功した場合、md_data_assign_pcopy を使用して、計算されたダイジェストデータを指定されたプール p 上にコピーし、id に割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_ocsp.c:333
  • modules/md/md_ocsp.c:491
  • **呼び出し先**:
  • md_cert_get_X509 (内部ヘルパー)
  • md_data_null
  • X509_digest (OpenSSL)
  • md_data_assign_pcopy

関連データ構造

  • md_data_t: 識別子データを保持する構造体。
  • md_cert_t: 証明書情報を保持する構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • X509_digest が 1 以外を返した場合、関数は APR_EGENERAL を返し、識別子の生成に失敗します。

保守担当者向けメモ

  • 本関数は SHA-1 を使用して証明書を識別しています。将来的に別のハッシュアルゴリズムへの移行が必要な場合は、EVP_sha1() の変更を検討してください。
