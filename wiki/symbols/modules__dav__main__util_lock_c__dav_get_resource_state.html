dav_get_resource_state

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:686

目的

指定されたリソースが「通常のリソース」「ロック・ヌル（lock-null）リソース」「存在しないリソース」のいずれであるかを判定します。

シグネチャまたは形状

int dav_get_resource_state(request_rec *r, const dav_resource *resource)

入力と出力

  • **入力**:
  • r: 現在の Apache リクエスト構造体。
  • resource: 状態を判定する対象の dav_resource 構造体。
  • **出力**:
  • DAV_RESOURCE_EXISTS: リソースが存在する場合。
  • DAV_RESOURCE_NULL: リソースが存在せず、ロックもされていない場合。
  • DAV_RESOURCE_LOCK_NULL: リソースが存在しないが、ロックが適用されている場合。
  • DAV_RESOURCE_ERROR: ロックデータベースのクエリ中にエラーが発生した場合。

前提条件と不変条件

  • resource->exists が真であれば、即座に DAV_RESOURCE_EXISTS を返します。
  • ロック・ヌル状態の判定には r->path_info を使用します。path_info が空でない場合、ロック・ヌルリソースではないとみなされます。

アルゴリズムまたは振る舞い

1. resource->exists を確認し、存在する場合はその旨を返します。 2. ロックフック (DAV_GET_HOOKS_LOCKS) が利用可能な場合、r->path_info をチェックしてロック・ヌルリソースの可能性を排除します。 3. ロックデータベースを open_lockdb で開き、has_locks を呼び出して対象リソースにロックが存在するかを確認します。 4. 処理終了後、必ず close_lockdb を呼び出してデータベースを閉じます。 5. ロックが存在すれば DAV_RESOURCE_LOCK_NULL を、存在しなければ DAV_RESOURCE_NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c 内の各メソッドハンドラ（dav_method_propfind, dav_method_put 等）および modules/dav/main/util.c から頻繁に呼び出されます。
  • **呼び出し先**: hooks->open_lockdb, hooks->has_locks, hooks->close_lockdb（ロックプロバイダーのフック関数）。

関連データ構造

  • dav_resource: 判定対象のリソース表現。
  • dav_hooks_locks: ロック管理機能を提供するプロバイダーのフック構造体。
  • request_rec: Apache のリクエスト情報（path_info や filename を参照）。

失敗モードと境界条件

  • **ロックDBエラー**: open_lockdb または has_locks がエラーを返した場合、エラーログを出力し DAV_RESOURCE_ERROR を返します。
  • **path_infoの制約**: ソース内のコメントによれば、path_info を用いたロック・ヌル判定はリポジトリ実装によっては正しく機能しない可能性があるという既知の懸念があります。

保守担当者向けメモ

  • 本関数はリソースの状態を判定するためにロックデータベースへのアクセスを伴う場合があり、呼び出し頻度が高い箇所ではパフォーマンスに影響を与える可能性があります。
  • ### コメントにある通り、path_info に依存したロック・ヌル判定のロジックは、将来的にリポジトリプロバイダーの API 拡張等による改善が検討されています。
