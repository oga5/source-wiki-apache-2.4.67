ssl_util_vhostid

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util.c:42

目的

サーバー名とポート番号を組み合わせ、仮想ホストを一意に識別するための文字列を生成します。

シグネチャまたは形状

char *ssl_util_vhostid(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 文字列の割り当てに使用する APR メモリプール。
  • s: 識別対象のサーバー設定構造体 (server_rec)。
  • **出力**:
  • char *: "サーバー名:ポート番号" 形式の文字列。メモリは p から割り当てられます。

前提条件と不変条件

  • s->server_hostname が有効な文字列であることを前提とします。
  • s->port が 0 の場合、mySrvConfig(s) を介して取得した設定に基づき、SSL が有効であれば DEFAULT_HTTPS_PORT、そうでなければ DEFAULT_HTTP_PORT がポート番号として使用されます。

アルゴリズムまたは振る舞い

1. サーバー構造体 s のポート番号を確認します。 2. ポート番号が 0 であれば、mySrvConfig を用いて仮想ホストの設定を取得し、SSL の有効状態に応じてデフォルトの HTTPS または HTTP ポートを割り当てます。 3. apr_psprintf を使用し、サーバー名:ポート番号 の形式で文字列を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_init.c (複数箇所)
  • modules/ssl/ssl_engine_io.c
  • modules/ssl/ssl_engine_kernel.c
  • **呼び出し先**:
  • mySrvConfig (マクロまたは関数)
  • apr_psprintf (APR ライブラリ)

関連データ構造

  • server_rec: Apache のサーバー設定構造体。
  • SSLSrvConfigRec: mod_ssl 固有の仮想ホスト設定構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • s->server_hostname が NULL の場合、apr_psprintf の動作に依存します。
  • メモリ不足時には apr_psprintf がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は ssl_private.h で宣言されており、モジュール内の広範囲で利用されています。
  • ポート番号の決定ロジックは mod_ssl の設定状態に依存するため、設定の初期化順序に変更を加える際は注意が必要です。
