charset_dir_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_charset_lite.c:72

目的

Apache HTTP Server のディレクトリ単位における文字セット変換設定を保持するための構造体です。

シグネチャまたは形状

typedef struct charset_dir_t {
    const char *charset_source;
    const char *charset_default;
    enum {IA_INIT, IA_IMPADD, IA_NOIMPADD} implicit_add;
    enum {FX_INIT, FX_FORCE, FX_NOFORCE} force_xlate;
} charset_dir_t;

入力と出力

  • **入力**: CharsetSourceEnc、CharsetDefault、CharsetOptions ディレクティブにより設定値が格納されます。
  • **出力**: モジュール内の各フック関数（find_code_page 等）が、リクエストごとの変換ロジックを決定するためにこの構造体を参照します。

前提条件と不変条件

  • apr_pcalloc を使用してメモリが確保されることが想定されています。
  • implicit_add および force_xlate は、設定の継承やマージ状態を管理するための列挙型です。

アルゴリズムまたは振る舞い

  • ディレクトリごとの設定を保持し、merge_charset_dir_conf 等を通じて親ディレクトリの設定とマージされます。
  • find_code_page フックにおいて、リクエストのコンテキストを初期化する際の判断基準として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_charset_lite.c 内の create_charset_dir_conf、merge_charset_dir_conf、find_code_page など、設定の生成・マージ・参照を行う関数群から利用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • charset_req_t: リクエストごとの設定情報として、本構造体の内容が反映されます。

失敗モードと境界条件

  • 設定が未定義（IA_INIT や FX_INIT）の状態から、どのようにマージ・解決されるかが変換の成否に影響します。

保守担当者向けメモ

  • 本構造体は mod_charset_lite.c 内でのみ使用されるプライベートな設定構造体です。
  • 設定の追加や変更を行う際は、merge_charset_dir_conf におけるマージロジックが正しく維持されていることを確認してください。
