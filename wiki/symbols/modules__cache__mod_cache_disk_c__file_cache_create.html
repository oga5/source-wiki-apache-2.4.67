file_cache_create

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:203

目的

ディスクキャッシュ用のファイル管理構造体（disk_cache_file_t）を初期化し、一時ファイルパスの設定とクリーンアップ処理の登録を行う。

シグネチャまたは形状

apr_status_t file_cache_create(disk_cache_conf *conf, disk_cache_file_t *file, apr_pool_t *pool)

入力と出力

  • **入力**:
  • conf: キャッシュ設定情報（cache_root を参照）。
  • file: 初期化対象の disk_cache_file_t 構造体。
  • pool: メモリ割り当てに使用する APR プール。
  • **出力**:
  • APR_SUCCESS: 正常終了。

前提条件と不変条件

  • file 引数は呼び出し元で確保済みであること。
  • conf->cache_root が有効なパス文字列であること。
  • pool はリクエストのライフサイクルに紐付いていること。

アルゴリズムまたは振る舞い

1. 引数 pool を file->pool に格納する。 2. conf->cache_root と AP_TEMPFILE を結合し、file->tempfile に設定する。 3. apr_pool_cleanup_register を使用し、プール破棄時に file_cache_temp_cleanup が実行されるよう登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c 内の create_entity 等の関数から、ヘッダー、Vary、データ用の各ファイル構造体に対して呼び出される（行 391-393, 518-520）。
  • **呼び出し先**:
  • apr_pstrcat (APR)
  • apr_pool_cleanup_register (APR)
  • file_cache_temp_cleanup (クリーンアップハンドラとして登録)

関連データ構造

  • disk_cache_conf: キャッシュ設定を保持。
  • disk_cache_file_t: キャッシュ対象のファイルパスやハンドルを管理する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pstrcat 等の APR 関数がエラーを返す可能性がある。

保守担当者向けメモ

  • 本関数は r->pool を利用してリソース管理を行うため、リクエスト終了時に自動的にクリーンアップが実行される設計となっている。
  • AP_TEMPFILE の定義内容がキャッシュディレクトリの構造に依存するため、変更時は注意が必要。
