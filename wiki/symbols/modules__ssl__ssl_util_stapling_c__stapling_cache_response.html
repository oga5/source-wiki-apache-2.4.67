stapling_cache_response

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_stapling.c:266

目的

OCSP レスポンスを socache（セッションキャッシュ）に保存し、後続の TLS ハンドシェイクで再利用可能にする。

シグネチャまたは形状

BOOL stapling_cache_response(server_rec *s, modssl_ctx_t *mctx,
                             OCSP_RESPONSE *rsp, certinfo *cinf,
                             BOOL ok, apr_pool_t *pool)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • mctx: SSL モジュールコンテキスト。
  • rsp: 保存対象の OCSP_RESPONSE オブジェクト。
  • cinf: 証明書情報（キャッシュキー cinf->idx を保持）。
  • ok: レスポンスが有効か否かを示すフラグ。
  • pool: メモリ割り当て用プール。
  • **出力**: 成功時に TRUE、失敗時に FALSE を返す。

前提条件と不変条件

  • MAX_STAPLING_DER（10KB）を超えるサイズのレスポンスは保存できない。
  • キャッシュバックエンドがスレッドセーフでない場合（AP_SOCACHE_FLAG_NOTMPSAFE）、呼び出し元で排他制御が必要。

アルゴリズムまたは振る舞い

1. i2d_OCSP_RESPONSE を用いて rsp を DER 形式にエンコードする。 2. エンコード後のデータ長が MAX_STAPLING_DER を超える場合はエラー終了する。 3. ok フラグに基づき、キャッシュの有効期限（stapling_cache_timeout または stapling_errcache_timeout）を計算する。 4. DER データの先頭に ok フラグ（1バイト）を付与する。 5. キャッシュバックエンドがスレッドセーフでない場合、stapling_cache_mutex_on を呼び出す。 6. mc->stapling_cache->store を実行してキャッシュに書き込む。 7. 必要に応じて stapling_cache_mutex_off を呼び出し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_util_stapling.c:583（stapling_renew_response 内での呼び出しが確認されている）。
  • **呼び出し先**:
  • i2d_OCSP_RESPONSE (OpenSSL)
  • mc->stapling_cache->store (キャッシュバックエンド)
  • stapling_cache_mutex_on/off (排他制御)
  • ap_log_error (エラーログ出力)

関連データ構造

  • OCSP_RESPONSE: OpenSSL の OCSP レスポンス構造体。
  • certinfo: 証明書とキャッシュキーを保持する構造体。
  • SSLModConfigRec: キャッシュ設定を保持するグローバル設定構造体。

失敗モードと境界条件

  • **エンコード失敗**: i2d_OCSP_RESPONSE が 0 以下の値を返した場合。
  • **サイズ超過**: resp_derlen + 1 が MAX_STAPLING_DER を超えた場合。
  • **ストア失敗**: mc->stapling_cache->store が APR_SUCCESS 以外を返した場合。

保守担当者向けメモ

  • resp_der バッファのサイズ制限（MAX_STAPLING_DER）は、OCSP レスポンスの肥大化に対するガードレールとして機能している。
  • キャッシュのタイムアウト値は modssl_ctx_t から取得されるため、設定変更が即座に反映される。
