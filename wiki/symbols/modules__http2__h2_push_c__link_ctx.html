link_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_push.c:60

目的

HTTP/2 のサーバープッシュ機能において、レスポンスヘッダーに含まれる Link ヘッダー文字列を走査・解析する際の状態を保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    const h2_request *req;          /* 関連するリクエスト情報 */
    apr_uint32_t push_policy;       /* プッシュポリシー */
    apr_pool_t *pool;               /* メモリ管理用プール */
    apr_array_header_t *pushes;     /* 抽出されたプッシュ候補の配列 */
    const char *s;                  /* 解析対象の文字列全体 */
    size_t slen;                    /* 文字列の長さ */
    size_t i;                       /* 現在の解析位置 */
    const char *link;               /* 現在解析中のリンクURI */
    apr_table_t *params;            /* リンクパラメータ */
    char b[4096];                   /* 一時作業用バッファ */
} link_ctx;

入力と出力

  • **入力**: h2_push_collect() 等の関数により、リクエスト情報や解析対象の Link ヘッダー文字列が本構造体にセットされます。
  • **出力**: 解析結果として pushes 配列にプッシュ候補が格納され、params テーブルにリンクパラメータが保持されます。

前提条件と不変条件

  • pool は apr_pool_t として適切に初期化されている必要があります。
  • b[4096] は固定長バッファであり、解析対象のトークンや文字列がこのサイズを超える場合、境界条件に注意が必要です。

アルゴリズムまたは振る舞い

Link ヘッダーの構文解析（rel=preload の抽出など）を行うためのステートフルな走査器として機能します。i を現在のオフセットとして保持し、s 内を逐次的に読み進めることで、URI やパラメータをトークン化します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_push.c 内の解析関数群（inspect_link 等）。
  • **呼び出し先**: ツールによる解析では、modules/http2/h2_proxy_util.c 内の skip_ws, find_chr, read_chr, read_link 等のユーティリティ関数が本構造体を引数として受け取り、解析処理を実行しています。

関連データ構造

  • h2_request: プッシュ対象リクエストの構築に使用。
  • apr_table_t: リンクパラメータの保持に使用。
  • apr_array_header_t: 抽出されたプッシュ候補のリスト管理に使用。

失敗モードと境界条件

  • b[4096] のバッファサイズを超える入力文字列の処理は、実装上の制約となります。
  • apr_pool_t のメモリ枯渇時には、関連するメモリ確保処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は h2_push.c で定義されていますが、h2_proxy_util.c からも参照されています。変更時は両ファイルの整合性に注意してください。
  • Link ヘッダーの解析ロジックは将来的にフックによる拡張が想定されています。
