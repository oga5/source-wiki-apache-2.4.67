set_ftp_escape_wildcards

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:91

目的

Apache HTTP Server の設定ディレクティブ ProxyFtpEscapeWildcards の値を処理し、FTP プロキシ設定構造体に反映させるための設定ハンドラ関数です。

シグネチャまたは形状

static const char *set_ftp_escape_wildcards(cmd_parms *cmd, void *dconf, int flag)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dconf: ディレクトリ単位の設定構造体 (proxy_ftp_dir_conf へのポインタ)。
  • flag: ディレクティブで指定された真偽値（int）。
  • **出力**: 常に NULL を返します。

前提条件と不変条件

  • dconf は proxy_ftp_dir_conf 型としてキャスト可能である必要があります。
  • 設定の適用後、conf->ftp_escape_wildcards_set が 1 に設定され、設定が明示的に指定されたことを示します。

アルゴリズムまたは振る舞い

1. 引数 dconf を proxy_ftp_dir_conf 構造体にキャストします。 2. flag の値を conf->ftp_escape_wildcards に代入します。 3. conf->ftp_escape_wildcards_set を 1 に設定し、設定が有効化されたことを記録します。 4. NULL を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ftp.c:2100 において、AP_INIT_FLAG マクロを通じて ProxyFtpEscapeWildcards ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • proxy_ftp_dir_conf: FTP プロキシのディレクトリ設定を保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーを返さない設計ですが、dconf が不正なポインタである場合はセグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は設定解析時にのみ呼び出されます。
  • ftp_escape_wildcards フラグは、FTP サーバーへのリクエスト時にワイルドカード文字をエスケープするかどうかを制御するために使用されます。
