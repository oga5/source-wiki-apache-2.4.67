kept_body_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:198

目的

r->kept_body に保存されたリクエストボディを、サブリクエストの入力ストリームとして再挿入する入力フィルタです。

シグネチャまたは形状

apr_status_t kept_body_filter(ap_filter_t *f, apr_bucket_brigade *b,
                              ap_input_mode_t mode,
                              apr_read_type_e block,
                              apr_off_t readbytes)

入力と出力

  • **入力**:
  • f: フィルタコンテキスト。f->ctx に kept_body_ctx_t を保持。
  • b: データを書き込むバケットブリゲード。
  • mode, block, readbytes: ap_get_brigade に準ずる読み込み制御パラメータ。
  • **出力**:
  • apr_status_t: 処理結果（APR_SUCCESS またはエラーコード）。
  • b: 保持されていたボディデータが格納されたバケットブリゲード。

前提条件と不変条件

  • r->kept_body が存在する場合にのみ、このフィルタは保持されたデータを処理します。
  • f->ctx は初回呼び出し時に r->pool から割り当てられ、以降の呼び出しで再利用されます。
  • ctx->remaining は、まだ読み出されていない保持データの残量を表します。

アルゴリズムまたは振る舞い

1. kept_body が存在しない、またはモードが AP_MODE_READBYTES / AP_MODE_GETLINE 以外の場合、後続のフィルタへ処理を委譲します。 2. 初回呼び出し時に f->ctx を初期化し、kept_body の総長を ctx->remaining に設定します。 3. ctx->remaining が 0 以下であれば、後続のフィルタへ処理を委譲します。 4. 要求された readbytes が残量を超える場合、readbytes を残量に制限します。 5. apr_brigade_partition を使用して、kept_body ブリゲードから該当範囲のバケットを特定します。 6. 特定したバケットをコピーして出力用ブリゲード b に挿入します。 7. ctx->offset と ctx->remaining を更新し、成功を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: フィルタチェーンの実行エンジン（ap_get_brigade 経由）。
  • **呼び出し先**:
  • ap_get_brigade: 保持データがない場合や、処理終了後のフォールスルーに使用。
  • apr_brigade_partition, apr_bucket_copy: データの抽出と複製に使用。
  • ap_log_rerror: エラー発生時のログ出力に使用。

関連データ構造

  • request_rec: r->kept_body を保持。
  • kept_body_ctx_t: フィルタの実行状態（offset, remaining）を管理。
  • apr_bucket_brigade: 保持されたボディデータの格納構造。

失敗モードと境界条件

  • apr_brigade_partition が失敗した場合、エラーログを出力し、そのエラーコードを返します。
  • apr_bucket_copy が失敗した場合、apr_bucket_read による強制読み込みを試みるフォールバック処理を行います。

保守担当者向けメモ

  • 本関数は mod_request のサブリクエスト再利用機能の核です。
  • r->kept_body はリクエストのライフサイクル（r->pool）に依存するため、サブリクエスト終了まで有効であることが保証されています。
