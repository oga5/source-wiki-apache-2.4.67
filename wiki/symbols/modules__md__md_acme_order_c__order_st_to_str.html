order_st_to_str

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:83

目的

ACME Order の内部状態（列挙型 md_acme_order_st）を、ACME プロトコルで規定される JSON 文字列形式に変換します。

シグネチャまたは形状

static const char *order_st_to_str(md_acme_order_st status)

入力と出力

  • **入力**: status (md_acme_order_st 型) - 変換対象の Order 状態。
  • **出力**: const char * - ACME プロトコル仕様に準拠した文字列（"pending", "ready", "processing", "valid", "invalid" のいずれか）。

前提条件と不変条件

  • 入力された status が定義済みの列挙値の範囲外である場合、デフォルトとして "invalid" を返します。

アルゴリズムまたは振る舞い

1. switch 文を用いて status の値を評価します。 2. 各列挙値に対応する文字列リテラルを返します。 3. 既知のステータス以外（default）が渡された場合は、安全策として "invalid" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_order.c:108 において、md_json_sets を介して Order 状態を JSON オブジェクトに格納する際に呼び出されています。
  • **呼び出し先**: なし（純粋な変換関数）。

関連データ構造

  • md_acme_order_st: ACME Order の状態を定義する列挙型。

失敗モードと境界条件

  • 未定義のステータス値が渡された場合、エラーを発生させるのではなく "invalid" を返すことで、呼び出し側の JSON 生成処理が継続できるように設計されています。

保守担当者向けメモ

  • 本関数は md_acme_order_to_json 等のシリアライズ処理から利用されることを想定しています。新しいステータスが md_acme_order_st に追加された場合は、必ず本関数の switch 文にもケースを追加してください。
