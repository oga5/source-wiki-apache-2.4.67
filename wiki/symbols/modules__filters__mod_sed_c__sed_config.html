sed_config

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_sed.c:38

目的

Apache HTTP Server のディレクトリ単位の設定において、入力および出力ストリームに適用する sed 形式の正規表現置換ルールを保持するために使用されます。

シグネチャまたは形状

typedef struct sed_config
{
    sed_expr_config output;
    sed_expr_config input;
} sed_config;

入力と出力

  • **入力**: sed_expr_config 型の output および input メンバ。これらはそれぞれレスポンスおよびリクエストのフィルタ設定を格納します。
  • **出力**: なし（構造体定義）。

前提条件と不変条件

  • mod_sed モジュールが有効な環境下で、ディレクトリごとの設定として ap_get_module_config を介して取得されることを前提としています。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持用であり、直接的なアルゴリズムは持ちません。
  • mod_sed 内で、リクエスト処理時に ap_get_module_config を通じて設定を取得し、内部の sed_expr_config を参照してフィルタの挙動を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_sed.c:281, 388: ap_get_module_config を通じて設定を取得。
  • modules/filters/mod_sed.c:505: apr_pcalloc によりメモリ確保。
  • modules/filters/mod_sed.c:511, 515: APR_OFFSETOF を用いてメンバのオフセットを参照。
  • **呼び出し先**: 構造体であるため、関数呼び出し先はありません。

関連データ構造

  • sed_expr_config: sed_config の各メンバの型であり、個別の sed コマンドセットとエラー情報を保持します。

失敗モードと境界条件

  • 構造体自体のメモリ確保失敗（apr_pcalloc 失敗時）が境界条件となります。

保守担当者向けメモ

  • 本構造体は mod_sed の設定管理の基盤です。sed_expr_config の定義変更や、設定項目の追加を行う際は、本構造体のメンバ構成も併せて確認してください。
