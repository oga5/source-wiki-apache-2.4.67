ap_mpm_safe_kill

定義

  • 種別: function
  • 定義箇所: server/mpm_unix.c:331

目的

指定されたプロセスID（PID）に対して、プロセスグループの整合性を検証した上で安全にシグナルを送信します。

シグネチャまたは形状

apr_status_t ap_mpm_safe_kill(pid_t pid, int sig)

入力と出力

  • **入力**:
  • pid: 送信対象のプロセスID。
  • sig: 送信するシグナル番号。
  • **出力**:
  • APR_SUCCESS: シグナルの送信に成功。
  • APR_EINVAL: PIDが不正（1未満）、対象がプロセスグループ外、またはプロセスが既に終了している場合。
  • errno: getpgid 失敗時のエラーコード。

前提条件と不変条件

  • pid は 1 以上である必要があります。
  • HAVE_GETPGID が定義されている環境では、対象プロセスが呼び出し元と同じプロセスグループに属している必要があります。

アルゴリズムまたは振る舞い

1. **PIDの妥当性検証**: pid < 1 の場合は直ちに APR_EINVAL を返します。 2. **環境に応じた検証**:

  • HAVE_GETPGID が未定義の場合: apr_proc_wait を使用してプロセスの生存を確認します。プロセスが終了済みであればステータスを記録し、エラーを返します。
  • HAVE_GETPGID が定義されている場合: getpgid(pid) を呼び出し、対象プロセスが現在のプロセスグループ (getpgrp()) と一致するか確認します。一致しない場合はログを出力し APR_EINVAL を返します。

    • 3. **シグナル送信**: kill(pid, sig) を実行し、成功すれば APR_SUCCESS を、失敗すれば errno を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/prefork/prefork.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • kill (システムコール)
  • getpgid / getpgrp (HAVE_GETPGID定義時)
  • apr_proc_wait / ap_process_child_status (HAVE_GETPGID未定義時)
  • ap_log_error

関連データ構造

  • pid_t (プロセスID)
  • apr_status_t (APRステータスコード)

失敗モードと境界条件

  • **PID不正**: pid < 1 の場合、処理を行わず APR_EINVAL を返します。
  • **プロセスグループ外**: HAVE_GETPGID 環境下で、対象PIDが現在のプロセスグループ外にある場合、セキュリティ保護のためシグナル送信を拒否します。
  • **既に終了**: プロセスが既に終了している場合、シグナル送信は行われずエラーが返されます。

保守担当者向けメモ

  • 本関数は、誤ったプロセスへのシグナル送信を防ぐための安全装置として機能します。
  • HAVE_GETPGID の有無により検証ロジックが分岐するため、プラットフォーム固有の挙動に注意してください。
