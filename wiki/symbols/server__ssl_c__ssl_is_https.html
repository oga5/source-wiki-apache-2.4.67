ssl_is_https

定義

  • 種別: function
  • 定義箇所: server/ssl.c:83

目的

古い API を使用して本関数を呼び出した呼び出し元に対し、推奨される新しい API (ap_ssl_conn_is_ssl) へリダイレクトを行うための互換性レイヤーです。

シグネチャまたは形状

static int ssl_is_https(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（現在の接続状態を表す構造体）
  • **出力**: int（接続が SSL/TLS を使用している場合は非ゼロ、それ以外は 0）

前提条件と不変条件

  • 引数 c は有効な conn_rec ポインタである必要があります。
  • 本関数は ap_ssl_conn_is_ssl を呼び出すため、その関数が提供する不変条件に依存します。

アルゴリズムまたは振る舞い

1. 呼び出し元から受け取った conn_rec ポインタをそのまま ap_ssl_conn_is_ssl(c) に渡します。 2. ap_ssl_conn_is_ssl の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: ap_ssl_conn_is_ssl
  • **呼び出し元**:
  • server/ssl.c 内で APR_REGISTER_OPTIONAL_FN(ssl_is_https) を通じて登録され、オプション関数として外部から参照される可能性があります。
  • ツールによる直接的な呼び出し元コードの特定は、本ソース抜粋範囲内では行われていません。

関連データ構造

  • conn_rec: Apache HTTP Server の接続情報を保持する構造体。

失敗モードと境界条件

  • c が NULL の場合、内部で呼び出される ap_ssl_conn_is_ssl の実装に依存します。

保守担当者向けメモ

  • 本関数は、古いモジュールとの後方互換性を維持するために提供されています。新規開発においては ap_ssl_conn_is_ssl を直接使用してください。
  • APR_DECLARE_OPTIONAL_FN を介して公開されているため、削除やシグネチャの変更を行う際は、依存する外部モジュール（mod_nw_ssl.c 等）への影響を考慮してください。
