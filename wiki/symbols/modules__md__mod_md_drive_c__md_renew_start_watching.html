md_renew_start_watching

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_drive.c:309

目的

Managed Domain (MD) の証明書更新を監視・実行するためのバックグラウンドタスクを mod_watchdog に登録する。

シグネチャまたは形状

apr_status_t md_renew_start_watching(md_mod_conf_t *mc, server_rec *s, apr_pool_t *p)

入力と出力

  • **入力**:
  • mc: モジュールの設定コンテキスト。
  • s: サーバーレコード。
  • p: 親プール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • mod_watchdog がロードされている必要がある。
  • mc->mds に監視対象の MD が含まれていること。
  • サーバー内の単一プロセスでのみ実行されることが mod_watchdog によって保証される。

アルゴリズムまたは振る舞い

1. mod_watchdog の関数ポインタ（get_instance, register_callback, set_interval）を取得する。 2. 監視専用のメモリプールを作成し、コンテキスト md_renew_ctx_t を初期化する。 3. mc->mds を走査し、watched フラグが有効な MD に対して md_job_t を作成・ロードする。 4. 前回の実行でエラーが記録されている場合、STAGING および CHALLENGES 領域をパージしてジョブをリセットする。 5. 監視対象のジョブが存在しない場合は早期終了する。 6. mod_watchdog のインスタンスを取得し、run_watchdog コールバックを登録して監視を開始する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c:1021
  • **呼び出し先**:
  • mod_watchdog API (ap_watchdog_get_instance, ap_watchdog_register_callback)
  • md_reg_job_make, md_job_load
  • md_store_purge
  • run_watchdog (コールバックとして登録)

関連データ構造

  • md_mod_conf_t: モジュール設定全体を保持。
  • md_renew_ctx_t: 更新監視プロセスの状態コンテキスト。
  • md_job_t: 個別の MD 更新ジョブの状態。

失敗モードと境界条件

  • mod_watchdog が利用できない場合、APLOG_CRIT レベルでログを出力しエラーを返す。
  • メモリプール作成失敗時はエラーを返す。
  • 監視対象の MD が一つも存在しない場合、プールを破棄して正常終了する。

保守担当者向けメモ

  • 本関数は mod_watchdog を介して非同期実行されるため、mc 内のデータは読み取り専用として扱うこと。
  • 更新処理の進捗は STAGING 領域のファイルシステムを通じて永続化される。
  • 再起動時のリカバリロジック（error_runs のチェック）が含まれているため、ジョブの失敗状態が永続化されている場合は注意が必要。
