h2_c1_io_needs_flush

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:329

目的

現在の出力バッファの状態が、フラッシュ（フィルタチェーンへの書き出し）を必要とする閾値に達しているかどうかを判定します。

シグネチャまたは形状

int h2_c1_io_needs_flush(h2_c1_io *io)

入力と出力

  • **入力**: h2_c1_io *io - 接続の入出力状態を管理する構造体へのポインタ。
  • **出力**: int - 閾値を超えている場合は非ゼロ（真）、そうでない場合は 0（偽）を返します。

前提条件と不変条件

  • 引数 io は有効な h2_c1_io 構造体を指している必要があります。

アルゴリズムまたは振る舞い

io 構造体内の現在のバッファ長 (buffered_len) と、設定されたフラッシュ閾値 (flush_threshold) を比較します。buffered_len >= flush_threshold が成立する場合にフラッシュが必要と判断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:196
  • modules/http2/h2_session.c:1364
  • modules/http2/h2_stream.c:1462
  • **呼び出し先**: 本関数は算術比較のみを行い、他の関数を呼び出しません。

関連データ構造

  • h2_c1_io: HTTP/2 クライアント接続の入出力バッファ状態を保持する構造体。buffered_len および flush_threshold メンバを使用します。

失敗モードと境界条件

  • io が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は単なる状態チェックであり、副作用はありません。
  • 閾値 flush_threshold は、接続の最適化状況に応じて動的に変更される可能性があるため、この関数は常に最新の io 状態を参照します。
