ssl_opt_t

定義

  • 種別: typedef int
  • 定義箇所: modules/ssl/ssl_private.h:421

目的

mod_ssl における SSL オプション設定のビットマスクまたはフラグ値を保持するための型定義です。

シグネチャまたは形状

typedef int ssl_opt_t;

入力と出力

  • 入力: 整数値（ビットフラグとして使用されることが想定される）。
  • 出力: なし（型定義のため）。

前提条件と不変条件

  • int 型として定義されているため、符号付き整数の範囲内でビット演算が適用されることを前提としています。

アルゴリズムまたは振る舞い

  • 設定の追加（nOptionsAdd）、削除（nOptionsDel）、および現在の設定状態（nOptions）を管理するために使用されます。
  • 複数のオプションをビット単位で組み合わせることで、SSL エンジンの挙動を制御します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/ssl/ssl_engine_config.c:1338
  • modules/ssl/ssl_private.h:877 (nOptions)
  • modules/ssl/ssl_private.h:878 (nOptionsAdd)
  • modules/ssl/ssl_private.h:879 (nOptionsDel)
  • 呼び出し先: 情報なし。

関連データ構造

  • SSLDirConfigRec: nOptions、nOptionsAdd、nOptionsDel をメンバとして保持する構造体（ssl_private.h 内で定義）。

失敗モードと境界条件

  • ビットマスクとして使用されるため、定義されたフラグ範囲外の値を代入した場合の挙動は、各フラグを処理するロジックの実装に依存します。

保守担当者向けメモ

  • 本型は int ですが、将来的にオプション数が増加しビット幅が不足する場合は、apr_uint32_t 等への変更を検討する必要があります。
  • ssl_engine_config.c での設定処理において、この型がどのようにビット演算されているかを確認してください。
