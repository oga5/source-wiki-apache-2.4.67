add_ignore

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_autoindex.c:319

目的

Apache の IndexIgnore ディレクティブで指定されたパターンを、ディレクトリごとの設定構造体（autoindex_config_rec）内の無視リストに追加する。

シグネチャまたは形状

static const char *add_ignore(cmd_parms *cmd, void *d, const char *ext)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • d: 現在のディレクトリ設定構造体（autoindex_config_rec にキャストして使用）。
  • ext: 無視対象として登録するファイル名またはパターン文字列。
  • **出力**:
  • 常に NULL を返す（設定解析の成功を示す）。

前提条件と不変条件

  • 引数 d は autoindex_config_rec 型のポインタとしてキャスト可能であること。
  • push_item 関数が内部的に呼び出され、ign_list への追加が行われる。

アルゴリズムまたは振る舞い

1. d を autoindex_config_rec 構造体にキャストする。 2. push_item を呼び出し、ign_list に ext（パターン）を登録する。この際、push_item の第2引数に 0 を渡すことで、無視リストとしてのエントリであることを指定する。 3. 処理完了後、NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_autoindex.c:585: AP_INIT_ITERATE マクロを通じて IndexIgnore ディレクティブのハンドラとして登録されている。
  • **呼び出し先**:
  • push_item: 設定構造体のリストに項目を追加するために呼び出される。

関連データ構造

  • autoindex_config_rec: ディレクトリごとの設定を保持する構造体。ign_list メンバを介して無視パターンを管理する。
  • cmd_parms: 設定解析時のコンテキスト情報。

失敗モードと境界条件

  • push_item 内部でのメモリ割り当て失敗時は、Apache のメモリ管理機構によりリクエスト処理が中断される可能性がある。
  • ext が空文字列または不正なパターンの場合、push_item の挙動に依存する。

保守担当者向けメモ

  • 本関数は IndexIgnore ディレクティブ専用のコールバックである。
  • push_item は汎用的なリスト追加関数であるため、無視リストの構造を変更する場合は push_item の実装も併せて確認すること。
