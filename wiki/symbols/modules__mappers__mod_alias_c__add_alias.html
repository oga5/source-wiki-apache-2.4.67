add_alias

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_alias.c:201

目的

Apache HTTP Server の Alias および ScriptAlias ディレクティブを解析し、設定構造体へエイリアス情報を登録します。

シグネチャまたは形状

static const char *add_alias(cmd_parms *cmd, void *dummy, const char *fake, const char *real)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 設定構造体（alias_dir_conf へのポインタ）。
  • fake: エイリアス元のパスまたは式。
  • real: エイリアス先のパス（NULL の場合、式ベースの処理へ分岐）。
  • **出力**: 成功時は NULL を返し、エラー発生時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • real が指定されている場合、内部で add_alias_internal を呼び出し、標準的なエイリアス登録を行います。
  • real が NULL の場合、ディレクトリコンテキスト内での使用は禁止されており、ap_check_cmd_context により NOT_IN_DIRECTORY および NOT_IN_FILES が検証されます。

アルゴリズムまたは振る舞い

1. real 引数の有無を確認します。 2. real が存在する場合、add_alias_internal に処理を委譲します。 3. real が存在しない場合（式ベースのエイリアス）:

  • コンテキストがグローバルであることを確認します。
  • ap_expr_parse_cmd を使用して fake を式として解析します。
  • 解析に成功した場合、dirconf 構造体に式の結果、パス、およびハンドラ情報を格納し、alias_set フラグを立てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_alias.c: AP_INIT_TAKE12 マクロにより、Alias および ScriptAlias ディレクティブのハンドラとして登録されています。
  • **呼び出し先**:
  • add_alias_internal
  • ap_check_cmd_context
  • ap_expr_parse_cmd
  • apr_pstrcat

関連データ構造

  • cmd_parms: 設定コマンドのコンテキスト。
  • alias_dir_conf: ディレクトリごとの設定を保持する構造体。

失敗モードと境界条件

  • real が NULL で、かつ cmd->path が設定されていない場合、「Alias must have two arguments when used globally」というエラーを返します。
  • 式の解析に失敗した場合、ap_expr_parse_cmd から返されたエラーメッセージを含む文字列を返します。
  • ap_check_cmd_context の制約に違反した場合、エラーメッセージを返します。

保守担当者向けメモ

  • 本関数は Alias と ScriptAlias の両方を処理します。ScriptAlias の場合、呼び出し時に cmd->info に "cgi-script" が渡されるため、これが dirconf->handler に設定されます。
  • 式ベースのエイリアス処理は、mod_alias の動的なマッピング機能の中核を担っています。
