dav_lookup_result

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:510

目的

URI のルックアップ処理の結果を保持し、サブクエスト情報とエラー状態を呼び出し元へ伝達するために使用されます。

シグネチャまたは形状

typedef struct
{
    request_rec *rnew;          /* new subrequest */
    dav_error err;              /* potential error response */
} dav_lookup_result;

入力と出力

  • **入力**: dav_lookup_uri 関数等のルックアップ処理において、結果を格納するコンテナとして利用されます。
  • **出力**: 成功時には rnew に生成されたサブクエストが格納され、失敗時には err に詳細なエラー情報が格納されます。

前提条件と不変条件

  • rnew が NULL でない場合、有効なサブクエストとして処理されることが期待されます。
  • err フィールドは dav_error 構造体であり、エラーが発生していない場合は適切に初期化（ゼロクリア等）されている必要があります。

アルゴリズムまたは振る舞い

1. ルックアップ処理（dav_lookup_uri 等）が実行される際、この構造体がインスタンス化されます。 2. 処理の結果に応じて、サブクエストのポインタまたはエラー情報が各フィールドに設定されます。 3. 呼び出し元は、返された構造体の err フィールドを確認し、エラーがなければ rnew を使用して後続の処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c 内の各メソッドハンドラ（行 2770, 3967, 4598, 4741 等）から参照されています。
  • **呼び出し先**: dav_lookup_uri 関数（modules/dav/main/util.c:190）がこの型を戻り値として生成します。

関連データ構造

  • request_rec: Apache HTTP Server のリクエスト構造体。rnew として保持されます。
  • dav_error: WebDAV 処理におけるエラー情報を保持する構造体。

失敗モードと境界条件

  • ルックアップ対象の URI が存在しない、またはアクセス権限がない場合、err フィールドにエラー情報が設定されます。
  • dav_lookup_uri 実装内（util.c:194）では、初期化時に { 0 } を使用して全フィールドをゼロクリアする慣習があります。

保守担当者向けメモ

  • 本構造体は mod_dav のリクエスト処理フローにおいて、ルックアップ結果を安全に返すための標準的なコンテナです。
  • dav_error 構造体のメモリ管理については、dav_new_error 等の関連 API の仕様に従う必要があります。
