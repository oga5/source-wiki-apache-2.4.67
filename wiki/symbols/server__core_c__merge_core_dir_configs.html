merge_core_dir_configs

定義

  • 種別: function
  • 定義箇所: server/core.c:221

目的

ディレクトリ単位の設定構造体（core_dir_config）を、親ディレクトリの設定（base）と新しい設定（new）からマージし、継承された新しい設定構造体を生成する。

シグネチャまたは形状

static void *merge_core_dir_configs(apr_pool_t *a, void *basev, void *newv)

入力と出力

  • **入力**:
  • a: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定（core_dir_config へのポインタ）。
  • newv: 現在のディレクトリで定義された設定（core_dir_config へのポインタ）。
  • **出力**:
  • マージ結果を保持する新しい core_dir_config 構造体へのポインタ。

前提条件と不変条件

  • opts_add と opts_remove の間では、常に (opts_add & opts_remove) == 0 という不変条件が維持される必要がある。
  • basev および newv は core_dir_config 型にキャスト可能であること。

アルゴリズムまたは振る舞い

1. apr_pmemdup を使用して base の内容を新しい設定構造体にコピーする。 2. ディレクトリパスや条件式などの基本情報を new から上書きする。 3. opts（オプション）の処理:

  • OPT_UNSET が設定されている場合、ビット演算により opts_add / opts_remove をマージし、不変条件を維持する。
  • Includes と IncludesNOEXEC の競合を解決する特殊なロジックを適用する。
  • 明示的な設定がある場合は new の値を優先する。

    • 4. override、override_opts、override_list を new から継承する。 5. response_code_exprs、sec_file、sec_if、cgi_var_rules などの配列やハッシュは、apr_hash_overlay や apr_array_append を用いて親と子の設定を結合する。 6. その他、hostname_lookups や limit_* などの各設定項目について、UNSET 状態でない場合に new の値を採用する。 7. 最後に AP_CORE_MERGE_FLAG マクロを用いてフラグ類をマージし、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core.c:5825（ディレクトリ単位の設定構造体マージ時）
  • server/request.c:602（リクエスト処理における設定マージ時）
  • **呼び出し先**:
  • apr_pmemdup, apr_hash_overlay, apr_array_append, ap_new_log_config, ap_merge_log_config 等の APR/Apache API。

関連データ構造

  • core_dir_config: ディレクトリ単位の設定を保持する主要な構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pmemdup 等の内部関数がエラーを発生させる可能性がある。
  • new の各フィールドが UNSET 状態である場合、base の値が保持される。

保守担当者向けメモ

  • 本関数は非常に多くの設定項目を扱うため、新しい設定ディレクティブを追加した際は、本関数内でのマージロジック（上書きか結合か）を適切に実装する必要がある。
  • sec_file や sec_if のような配列の結合は、メモリ消費量に影響を与えるため注意すること。
