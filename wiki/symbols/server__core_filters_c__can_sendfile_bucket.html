can_sendfile_bucket

定義

  • 種別: function
  • 定義箇所: server/core_filters.c:615

目的

指定されたバケットが sendfile システムコールによる転送に適しているかどうかを判定します。

シグネチャまたは形状

static APR_INLINE int can_sendfile_bucket(apr_bucket *b)

入力と出力

  • **入力**: apr_bucket *b (判定対象のバケット)
  • **出力**: int (適格であれば非ゼロ、不適格であれば 0 を返す)

前提条件と不変条件

  • バケットが APR_BUCKET_IS_FILE 型であること。
  • バケットのデータ長が AP_MIN_SENDFILE_BYTES 以上であること。
  • ファイル記述子に対して APR_SENDFILE_ENABLED フラグが設定されていること。

アルゴリズムまたは振る舞い

1. バケットがファイルバケットであり、かつサイズが AP_MIN_SENDFILE_BYTES 以上であるかを確認します。 2. 上記条件を満たす場合、バケット内のファイル記述子から apr_file_flags_get を呼び出し、APR_SENDFILE_ENABLED フラグが有効かを確認します。 3. すべての条件を満たす場合にのみ 1 を返し、それ以外（非ファイルバケット、サイズ不足、または設定による無効化）の場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core_filters.c:668 (直接的な参照を確認)
  • **呼び出し先**: apr_bucket_is_file (マクロ), apr_file_flags_get (APR API)

関連データ構造

  • apr_bucket: APR のバケット構造体。
  • apr_bucket_file: ファイルバケットの内部データ構造。

失敗モードと境界条件

  • バケットが NULL の場合、あるいはファイルバケット以外の型である場合は 0 を返します。
  • EnableSendfile off が設定されている場合、フラグがクリアされるため 0 を返します。

保守担当者向けメモ

  • 本関数は APR_INLINE で定義されており、パフォーマンスを考慮してヘッダーやフィルタ処理のホットパスで使用されることを想定しています。
  • AP_MIN_SENDFILE_BYTES の閾値は、システムごとの sendfile のオーバーヘッドと転送効率のバランスに依存します。
