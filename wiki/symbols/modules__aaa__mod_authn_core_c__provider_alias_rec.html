provider_alias_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authn_core.c:61

目的

認証プロバイダーのエイリアス（<AuthnProviderAlias>）を管理するための構造体です。元の認証プロバイダーへの参照と、そのエイリアス内で適用されるディレクトリ設定を保持します。

シグネチャまたは形状

typedef struct provider_alias_rec {
    char *provider_name;             /* プロバイダー名 */
    char *provider_alias;            /* エイリアス名 */
    ap_conf_vector_t *sec_auth;      /* エイリアス内で有効なディレクトリ設定 */
    const authn_provider *provider;  /* 元の認証プロバイダーへの参照 */
} provider_alias_rec;

入力と出力

本構造体はデータ保持用であり、関数ではありません。設定解析時（authaliassection）に生成され、実行時（authn_alias_check_password 等）にハッシュテーブルから取得されます。

前提条件と不変条件

  • AuthnProviderAlias は GLOBAL_ONLY であり、サーバー起動時に定義が確定します。
  • sec_auth は、エイリアスブロック内で定義された設定をカプセル化したものです。

アルゴリズムまたは振る舞い

認証実行時、以下の手順で利用されます： 1. authn_alias_srv_conf から該当する provider_alias_rec を検索。 2. リクエストの r->per_dir_config を一時的に sec_auth に差し替え。 3. provider が保持する関数を呼び出し。 4. 処理終了後、r->per_dir_config を復元。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authn_core.c: authaliassection（生成）、authn_alias_check_password および authn_alias_get_realm_hash（参照）。
  • modules/aaa/mod_authz_core.c: 参照の可能性あり（ツールによる直接的な呼び出し関係の詳細は不明）。
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、直接の呼び出し先はありません。保持している provider フィールドを通じて、登録された認証プロバイダーの関数が呼び出されます。

関連データ構造

  • authn_provider: 認証プロバイダーのインターフェース定義。
  • authn_alias_srv_conf: エイリアスを管理するサーバー設定構造体。
  • ap_conf_vector_t: Apache の設定ベクトル（ディレクトリ設定の保持に使用）。

失敗モードと境界条件

  • エイリアス名に対応するプロバイダーが存在しない場合、実行時のハッシュ検索で NULL が返される可能性があります。
  • メモリ割り当て失敗時は apr_pcalloc がエラーを発生させます。

保守担当者向けメモ

  • 本構造体は mod_authn_core.c と mod_authz_core.c の両方で定義されていますが、用途は各モジュールの認証・認可プロバイダーのエイリアス管理に特化しています。
  • sec_auth の差し替えは r->pool を利用して安全に行われていますが、複雑な設定マージが発生するため、変更時は ap_merge_per_dir_configs の挙動に注意してください。
