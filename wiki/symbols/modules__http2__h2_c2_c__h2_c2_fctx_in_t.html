h2_c2_fctx_in_t

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_c2.c:176

目的

セカンダリ接続（c2）における入力フィルタのコンテキストを保持するための構造体です。HTTP/2 ストリームから受信したデータを一時的に保持し、Apache のフィルタチェーンへ供給する役割を担います。

シグネチャまたは形状

typedef struct {
    apr_bucket_brigade *bb;       /* c2: data in holding area */
    unsigned did_upgrade_eos:1;   /* for Upgrade, we added an extra EOS */
} h2_c2_fctx_in_t;

入力と出力

  • **入力**: h2_c2_filter_in フィルタがこの構造体をコンテキストとして参照し、bb を介してデータを処理します。
  • **出力**: bb に格納されたバケットが、Apache の入力フィルタチェーンを通じてリクエスト処理へ渡されます。

前提条件と不変条件

  • bb は apr_bucket_brigade を指し、有効なメモリプールに関連付けられている必要があります。
  • did_upgrade_eos はビットフィールドであり、HTTP/2 へのアップグレード時に EOS（End of Stream）バケットが追加されたかどうかのフラグとして機能します。

アルゴリズムまたは振る舞い

  • h2_c2_filter_in 内で f->ctx としてキャストされ、フィルタの実行状態を管理します。
  • bb は、h2_bucket_beam から受信したデータを一時的に保持するバッファとして機能します。
  • did_upgrade_eos は、HTTP/1.1 から HTTP/2 へのアップグレードプロセスにおいて、ストリーム終了の整合性を保つために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c2.c 内の h2_c2_filter_in 関数が、フィルタコンテキストとしてこの構造体を参照しています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_bucket_brigade: Apache のバケット管理用構造体。
  • h2_c2_filter_in: この構造体をコンテキストとして利用する入力フィルタ関数。

失敗モードと境界条件

  • bb が初期化されていない、またはメモリ不足により割り当てに失敗した場合、フィルタ処理が異常終了する可能性があります。
  • did_upgrade_eos のフラグ管理が不適切な場合、ストリームの終了判定が正しく行われず、リクエスト処理がハングまたは早期終了するリスクがあります。

保守担当者向けメモ

  • 本構造体は h2_c2_filter_in の状態管理に特化しています。フィルタのロジックを変更する際は、bb の所有権とライフサイクル（特に apr_bucket_brigade のクリアと破棄）に注意してください。
  • did_upgrade_eos はアップグレード処理特有のフラグであるため、通常の HTTP/2 ストリーム処理との混同を避ける必要があります。
