get_provider

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_util.c:146

目的

指定された設定エントリに基づき、キャッシュプロバイダーを検索してプロバイダーリストに追加します。

シグネチャまたは形状

static cache_provider_list *get_provider(request_rec *r, struct cache_enable *ent, cache_provider_list *providers)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。メモリ確保（r->pool）に使用。
  • ent: プロバイダーの型情報を含む設定エントリ。
  • providers: 現在のプロバイダーリストの先頭ポインタ。
  • **出力**:
  • 更新された cache_provider_list の先頭ポインタを返します。

前提条件と不変条件

  • ap_lookup_provider を使用して CACHE_PROVIDER_GROUP からプロバイダーを検索します。
  • プロバイダーが既にリスト内に存在する場合、重複追加は行われません。

アルゴリズムまたは振る舞い

1. ap_lookup_provider を呼び出し、ent->type に対応するプロバイダーを取得します。 2. プロバイダーが見つからない場合はログ出力（実装は省略）を行います。 3. プロバイダーが見つかった場合、r->pool から cache_provider_list 用のメモリを確保し、情報を格納します。 4. リストが空であれば、新しいノードを先頭にします。 5. リストが空でない場合、末尾まで走査し、既に同じプロバイダーが登録されていないか確認します。 6. 重複がない場合、リストの末尾に新しいノードを追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/cache_util.c:214
  • modules/cache/cache_util.c:223
  • **呼び出し先**:
  • ap_lookup_provider (外部API)
  • apr_pcalloc (外部API)

関連データ構造

  • cache_provider_list: プロバイダーを保持する連結リスト構造。
  • cache_enable: プロバイダーの型情報を持つ設定構造体。
  • request_rec: Apache のリクエストコンテキスト。

失敗モードと境界条件

  • ap_lookup_provider が NULL を返した場合、プロバイダーは追加されず、元のリストがそのまま返されます。
  • リスト内に同一のプロバイダーが既に存在する場合、リストの変更は行われず、既存のリストが返されます。

保守担当者向けメモ

  • 本関数は static であり、cache_util.c 内でのみ使用されます。
  • メモリ管理は r->pool に依存しているため、リクエスト終了時に自動的に解放されます。
