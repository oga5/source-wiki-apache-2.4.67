md_http_response_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_http.h:28

目的

HTTP 通信の結果として受信したレスポンスデータを保持するための構造体です。

シグネチャまたは形状

typedef struct md_http_response_t md_http_response_t;

※不完全型として定義されており、具体的なメンバ構成はヘッダーファイル上では非公開です。

入力と出力

  • **入力**: HTTP リクエストの実行結果として、通信バックエンド（md_http_impl_t）により生成されます。
  • **出力**: ACME プロトコルの各処理（認証、オーダー取得、証明書発行など）において、ステータスコードやレスポンスボディの解析のために参照されます。

前提条件と不変条件

  • md_http_perform 等の通信処理が完了した後に有効なインスタンスとして提供されます。
  • ライフサイクルは md_http_perform の実行単位に紐付いており、リクエスト完了後に自動的に解放されます。

アルゴリズムまたは振る舞い

  • HTTP レスポンスのステータスコード、ヘッダー、およびボディ（apr_bucket_brigade）をカプセル化します。
  • 呼び出し側は、この構造体を読み取り専用（const）として受け取り、レスポンスの内容に基づいた後続処理（エラー判定やデータ抽出）を行います。

呼び出し元と呼び出し先

  • **主な呼び出し元**:
  • md_acme.c: http_update_nonce, inspect_problem, on_response, update_directory
  • md_acme_drive.c: on_add_cert, on_add_chain, csr_req
  • md_crypt.c: 証明書や鍵に関連するレスポンスの検証処理
  • **呼び出し先**: 構造体の内部実装は modules/md/md_http.c に依存しますが、本ヘッダーからは直接的な関数呼び出し関係は定義されていません。

関連データ構造

  • md_http_request_t: 対応するリクエストオブジェクト。
  • md_http_t: HTTP セッションを管理する親構造体。
  • apr_bucket_brigade: レスポンスボディのデータストリームを保持するために使用されます。

失敗モードと境界条件

  • 通信エラーが発生した場合、レスポンスが空（NULL）であるか、またはステータスコードがエラーを示す値（4xx, 5xx 等）となります。
  • 呼び出し側は、md_http_response_t を受け取るコールバック関数内で、ポインタが NULL でないことを確認する必要があります。

保守担当者向けメモ

  • 本構造体は不完全型として定義されているため、メンバへの直接アクセスは行わず、提供されるアクセサ関数（存在する場合）または md_http.c 内の実装を通じて操作してください。
  • md_http_request_t と同様に、メモリ管理は md_http サブシステムによって自動化されているため、呼び出し側で free 等を行う必要はありません。
