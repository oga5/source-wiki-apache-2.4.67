sei_cfg_rec

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_setenvif.c:118

目的

mod_setenvif モジュールにおいて、設定された条件付き環境変数ルール（sei_entry）の集合を保持するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *conditionals;
} sei_cfg_rec;

入力と出力

  • **入力**: conditionals メンバは、apr_array_header_t 型の配列であり、個別の条件ルールを格納します。
  • **出力**: 本構造体自体は設定レコードとして、サーバーまたはディレクトリ単位の構成情報として保持されます。

前提条件と不変条件

  • conditionals は apr_palloc または apr_pcalloc を使用してメモリ確保される必要があります。
  • 本構造体は、mod_setenvif の設定フェーズにおいて、ディレクティブ（SetEnvIf 等）の解析結果を蓄積するために使用されます。

アルゴリズムまたは振る舞い

  • add_setenvif_core 関数により、解析されたルールが conditionals 配列に追加されます。
  • match_headers 関数等の実行フェーズにおいて、本構造体から conditionals を取得し、格納されたルールを順次走査してリクエストの評価を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_setenvif_config (初期化)
  • merge_setenvif_config (設定結合)
  • add_setenvif_core (ルール追加)
  • match_headers (評価実行)
  • ap_get_module_config (設定取得)
  • **呼び出し先**:
  • apr_palloc / apr_pcalloc (メモリ確保)
  • apr_array_push (配列への要素追加 - 推定)

関連データ構造

  • sei_entry: conditionals 配列内に格納される個別の条件ルールを定義する構造体。
  • apr_array_header_t: APR が提供する動的配列構造体。

失敗モードと境界条件

  • メモリ不足時: apr_palloc 等のメモリ確保関数が失敗した場合、Apache のエラーハンドリングに従いプロセスが終了またはリクエスト処理が中断されます。
  • 設定未定義時: conditionals が初期化されていない場合、評価関数での参照時にヌルポインタ参照が発生する可能性があるため、各関数で適切なチェックが必要です。

保守担当者向けメモ

  • 本構造体は mod_setenvif の設定管理の核となります。設定の結合（merge_setenvif_config）を行う際は、conditionals 配列の適切なマージ処理が必須です。
  • ap_get_module_config を介して取得されるため、モジュール設定のライフサイクルとメモリプール（cmd->pool や r->pool）の管理に注意してください。
