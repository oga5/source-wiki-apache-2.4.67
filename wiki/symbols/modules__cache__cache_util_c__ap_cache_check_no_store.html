ap_cache_check_no_store

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_util.c:493

目的

リクエストに含まれる Cache-Control: no-store ヘッダーを評価し、キャッシュからのレスポンス提供やキャッシュへの保存が許可されているかを判定します。

シグネチャまたは形状

int ap_cache_check_no_store(cache_request_rec *cache, request_rec *r)

入力と出力

  • **入力**:
  • cache: キャッシュリクエストコンテキスト。
  • r: 現在のリクエスト構造体。
  • **出力**:
  • 1: キャッシュの利用または保存が許可されている（または設定により no-store が無視された）。
  • 0: no-store が指定されており、キャッシュの利用が禁止されている。

前提条件と不変条件

  • cache->control_in が未解析の場合、関数内で r->headers_in から Cache-Control および Pragma ヘッダーを読み取り、解析を実行します。
  • cache_server_conf の ignorecachecontrol 設定が有効な場合、no-store 指示は無視されます。

アルゴリズムまたは振る舞い

1. cache->control_in.parsed が偽であれば、cache_table_getm を用いてリクエストヘッダーから Cache-Control と Pragma を取得し、ap_cache_control で解析結果を cache->control_in に格納します。 2. cache->control_in.no_store が真の場合:

  • conf->ignorecachecontrol が偽であれば、キャッシュ利用不可として 0 を返します。
  • conf->ignorecachecontrol が真であれば、ログに情報を出力し、キャッシュ利用可能として 1 を返します。

    • 3. 上記以外の場合は 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c:121
  • modules/cache/mod_cache.c:431
  • **呼び出し先**:
  • ap_get_module_config
  • cache_table_getm
  • ap_cache_control
  • ap_log_rerror

関連データ構造

  • cache_request_rec: キャッシュ制御状態を保持。
  • cache_server_conf: キャッシュモジュールのサーバー設定（ignorecachecontrol を参照）。
  • request_rec: HTTP リクエスト情報。

失敗モードと境界条件

  • Cache-Control ヘッダーが存在しない、または解析不能な場合、no_store はデフォルトで無効（0）として扱われます。
  • 設定で ignorecachecontrol が有効な場合、クライアントの no-store 指示は強制的に無視されます。

保守担当者向けメモ

  • 本関数は RFC 2616 14.9.2 に基づく no-store の挙動を実装しています。
  • cache->control_in の解析結果はキャッシュされるため、同一リクエスト内で複数回呼び出されても解析処理は初回のみ実行されます。
