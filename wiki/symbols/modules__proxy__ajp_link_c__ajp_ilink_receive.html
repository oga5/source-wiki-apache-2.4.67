ajp_ilink_receive

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_link.c:74

目的

AJPプロトコルに従い、ソケットからメッセージヘッダーおよびボディを順次読み取り、ajp_msg_t 構造体へ格納する。

シグネチャまたは形状

apr_status_t ajp_ilink_receive(apr_socket_t *sock, ajp_msg_t *msg)

入力と出力

  • **入力**:
  • sock: 通信に使用する apr_socket_t へのポインタ。
  • msg: 受信データを格納する ajp_msg_t 構造体へのポインタ。
  • **出力**:
  • 成功時は APR_SUCCESS を返す。
  • 失敗時はエラーコード（APR_TIMEUP, AJP_ENO_HEADER, AJP_EBAD_HEADER, AJP_EBAD_MESSAGE 等）を返す。

前提条件と不変条件

  • msg は初期化済みであり、header_len が有効な値であること。
  • ilink_read が指定バイト数を読み取るまでブロッキングまたは待機を行うことを前提とする。

アルゴリズムまたは振る舞い

1. msg->header_len を参照し、ilink_read を用いてソケットからヘッダーを読み取る。 2. ajp_msg_check_header を呼び出し、ヘッダーの妥当性を検証し、続くボディの長さ（blen）を特定する。 3. 特定された長さ（blen）分だけ、msg->buf のヘッダー直後の位置から ilink_read を用いてボディを読み取る。 4. 読み取り完了後、デバッグログを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/ajp_header.c
  • modules/proxy/ajp_utils.c
  • **呼び出し先**:
  • ilink_read (内部ヘルパー)
  • ajp_msg_check_header
  • ap_log_error (Apache HTTP Server ログ出力関数)

関連データ構造

  • ajp_msg_t: AJPメッセージを保持する構造体。
  • apr_socket_t: APRのソケット構造体。

失敗モードと境界条件

  • **ヘッダー受信失敗**: ilink_read が APR_SUCCESS 以外を返した場合、タイムアウト時は APR_TIMEUP、それ以外は AJP_ENO_HEADER を返す。
  • **ヘッダー検証失敗**: ajp_msg_check_header がエラーを返した場合、AJP_EBAD_HEADER を返す。
  • **ボディ受信失敗**: ilink_read がボディ読み取り中にエラーを返した場合、AJP_EBAD_MESSAGE を返す。

保守担当者向けメモ

  • 本関数は低レベルなバイトストリーム受信を担当する。プロトコルの意味論的な解釈は行わない。
  • エラー発生時は APLOGNO を付与したログが出力されるため、障害調査時には該当番号を参照すること。
