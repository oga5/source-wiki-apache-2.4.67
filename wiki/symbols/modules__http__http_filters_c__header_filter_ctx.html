header_filter_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http/http_filters.c:1304

目的

HTTP レスポンスヘッダの送信状態を管理するためのコンテキスト構造体です。出力フィルタチェーンにおいて、ヘッダが既に送信されたかどうかを追跡するために使用されます。

シグネチャまたは形状

typedef struct header_filter_ctx {
    int headers_sent;
} header_filter_ctx;

入力と出力

  • **入力**: フィルタコンテキスト (f->ctx) として保持されます。
  • **出力**: headers_sent メンバが、ヘッダ送信済みであれば非ゼロ、未送信であればゼロを示します。

前提条件と不変条件

  • f->ctx を通じてアクセスされる際、apr_pcalloc 等でゼロ初期化されていることが前提となります。
  • headers_sent は、ヘッダがクライアントへ送信された直後に更新される必要があります。

アルゴリズムまたは振る舞い

1. 出力フィルタ（ap_http_header_filter 等）が呼び出された際、f->ctx から本構造体を取得します。 2. headers_sent フラグを確認し、未送信であればヘッダの構築と送信処理を行います。 3. 送信完了後、headers_sent を更新して二重送信や不正なヘッダ処理を防止します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_filters.c 内の出力フィルタ関数群（ap_http_header_filter など）。
  • **呼び出し先**: 直接的な関数呼び出しは行いません。データ保持用構造体です。

関連データ構造

  • http_ctx_t: 入力フィルタの状態管理用構造体。
  • header_struct: ヘッダ送信時に使用される補助構造体。

失敗モードと境界条件

  • **メモリ不足**: apr_pcalloc によるコンテキスト生成時に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は modules/http/http_filters.c 内の出力フィルタ専用です。
  • headers_sent は単なるフラグですが、HTTP プロトコルの整合性を保つための重要な状態変数です。変更を加える際は、ap_http_header_filter におけるフラグチェックのロジックに影響がないか確認してください。
