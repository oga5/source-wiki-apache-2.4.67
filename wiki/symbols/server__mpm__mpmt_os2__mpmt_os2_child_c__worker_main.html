worker_main

定義

  • 種別: function
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2_child.c:345

目的

OS/2 MPM の子プロセスにおいて、個別のワーカー（スレッド）として接続リクエストを処理し、リクエストの受け取りから接続の終了までを管理する。

シグネチャまたは形状

static void worker_main(void *vpArg)
  • vpArg: スレッドスロット番号（int）としてキャストして使用される。

入力と出力

  • **入力**: vpArg（スレッドスロット番号）。OS/2 キュー（/queues/httpd/work.<pid>）から読み出される worker_args_t 構造体（接続情報）。
  • **出力**: なし（戻り値 void）。副作用としてスコアボードの更新および接続の処理を行う。

前提条件と不変条件

  • DosSetExceptionHandler により、スレッド内の例外を捕捉しプロセス全体のクラッシュを防ぐ。
  • pchild プールが生存していること。
  • 接続処理中、スコアボードのステータスは SERVER_READY から SERVER_BUSY 相当の状態へ遷移し、処理完了後に SERVER_READY へ戻る。

アルゴリズムまたは振る舞い

1. **初期化**: 例外ハンドラを登録し、OS/2 キューを開く。 2. **ループ**: DosReadQueue を使用して、キューから接続情報（worker_args）が届くのを待機する。 3. **接続処理**:

  • ap_run_create_connection で接続オブジェクトを作成。
  • ap_process_connection でリクエストを処理。
  • ap_lingering_close で接続を閉じる。

    • 4. **終了**: WORKTYPE_EXIT メッセージを受信するか、キューの読み取りに失敗した時点でループを脱出し、スコアボードを SERVER_DEAD に更新して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/mpmt_os2/mpmt_os2_child.c 内の _beginthread により起動される。
  • **呼び出し先**: ap_run_create_connection, ap_process_connection, ap_lingering_close, ap_update_child_status_from_indexes, DosReadQueue, DosOpenQueue 等。
  • **注記**: server/mpm/netware/mpm_netware.c や server/mpm/winnt/child.c にも同名の関数が存在するが、これらは別実装である。

関連データ構造

  • worker_args_t: キュー経由で渡される接続情報。
  • EXCEPTIONREGISTRATIONRECORD: OS/2 の例外処理用構造体。
  • ap_sb_handle_t: スコアボードハンドル。

失敗モードと境界条件

  • **キューオープン失敗**: DosOpenQueue が失敗した場合、ログを出力し、該当スロットのスコアボードをクリアして終了する。
  • **例外発生**: thread_exception_handler が呼び出され、プロセス全体を巻き込む前にスレッドが安全に終了する。

保守担当者向けメモ

  • 本関数は OS/2 の Dos* API に強く依存している。
  • pconn プールは各接続処理のたびに作成・破棄され、メモリリークを防ぐ設計となっている。
  • HARD_THREAD_LIMIT（256）がスレッド数の上限として機能する。
