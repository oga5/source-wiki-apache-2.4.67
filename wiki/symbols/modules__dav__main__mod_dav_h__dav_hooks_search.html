dav_hooks_search

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:273（前方宣言）、modules/dav/main/mod_dav.h:2583（構造体定義）

目的

WebDAV 検索機能（DAV Search）を実装するためのプロバイダーインターフェースを定義するフック構造体です。

シグネチャまたは形状

構造体の詳細なメンバー（関数ポインタ）は modules/dav/main/mod_dav.h:2583 以降に定義されています。

入力と出力

  • 入力: dav_get_search_hooks(request_rec *r) を通じて、リクエストに関連付けられたプロバイダーのフックを取得します。
  • 出力: 検索機能を提供するプロバイダーが実装した関数ポインタのテーブルを保持します。

前提条件と不変条件

  • mod_dav のプロバイダーモデルに従い、検索機能を提供するモジュールによって適切に登録される必要があります。
  • dav_get_search_hooks を呼び出す際、リクエストプールに関連付けられたプロバイダーが有効である必要があります。

アルゴリズムまたは振る舞い

  • mod_dav は、検索リクエストの処理時に DAV_GET_HOOKS_SEARCH(r) マクロ（内部で dav_get_search_hooks を使用）を介してこの構造体を取得し、検索ロジックを実行します。
  • 検索機能がサポートされていない場合、このフックは NULL を返す可能性があります。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/dav/main/mod_dav.c:1646
  • modules/dav/main/mod_dav.c:1704
  • 呼び出し先:
  • modules/dav/main/mod_dav.c:248 (dav_get_search_hooks 実装内)
  • modules/dav/main/mod_dav.h:802 (API 宣言)

関連データ構造

  • dav_provider: 検索フックを含むプロバイダーの集合体。
  • request_rec: Apache HTTP Server のリクエスト構造体。

失敗モードと境界条件

  • プロバイダーが検索機能を実装していない場合、dav_get_search_hooks は NULL を返すか、空のフックテーブルを返します。呼び出し側は NULL チェックを行う必要があります。

保守担当者向けメモ

  • 本構造体は WebDAV 検索機能の抽象化レイヤーです。具体的な実装は mod_dav のプロバイダーとして提供される必要があります。
  • 構造体のメンバー定義については mod_dav.h の 2583 行目以降を参照してください。
