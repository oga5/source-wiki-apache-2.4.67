ap_directory_walk

定義

  • 種別: function
  • 定義箇所: server/request.c:661

目的

ファイルシステムパスに基づき、Apache の <Directory> 設定セクションをリクエストコンテキストにマージする。

シグネチャまたは形状

int ap_directory_walk(request_rec *r)

入力と出力

  • **入力**: request_rec *r （リクエスト情報。r->filename を使用してパスを特定する）
  • **出力**: int （成功時は OK、失敗時は HTTP_FORBIDDEN や HTTP_INTERNAL_SERVER_ERROR などのステータスコード）

前提条件と不変条件

  • r->filename が絶対パスとして解決可能であること。
  • r->pool を使用してメモリ管理が行われること。
  • filename_len はスコープ内において常に strlen(r->filename) と等しい。

アルゴリズムまたは振る舞い

1. **正規化**: apr_filepath_merge を使用して r->filename を絶対パスに正規化する。 2. **キャッシュ確認**: prep_walk_cache を呼び出し、既存のディレクトリウォーク結果が再利用可能か判定する。 3. **ファイル情報取得**: ap_run_dirwalk_stat を呼び出し、ファイルタイプを確認する。 4. **キャッシュヒット時**: 設定に変更がなく、パスが一致する場合はキャッシュされた per_dir_config を適用し、シンボリックリンクの制限を確認して終了する。 5. **キャッシュミス時**:

  • ルートパスから順にコンポーネントを走査する。
  • r->path_info を適切に分離・再構成する。
  • 各階層で設定をマージし、結果をキャッシュに保存する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/core.c:4818
  • modules/proxy/mod_proxy_fcgi.c:383
  • **呼び出し先**:
  • prep_walk_cache
  • ap_run_dirwalk_stat
  • resolve_symlink
  • ap_make_dirstr_parent
  • apr_filepath_merge
  • apr_filepath_root
  • ap_get_core_module_config

関連データ構造

  • request_rec: リクエストの主要な状態を保持。
  • walk_cache_t: ディレクトリウォークの結果をキャッシュし、再計算を回避する。
  • core_dir_config: ディレクトリ単位の設定（オプションやオーバーライド設定）。

失敗モードと境界条件

  • r->filename が NULL の場合: エラーログを出力し OK を返す。
  • パスが不正または非絶対パスの場合: エラーログを出力し OK を返す。
  • シンボリックリンクの解決失敗: HTTP_FORBIDDEN を返す。
  • パス情報のマージ失敗: HTTP_INTERNAL_SERVER_ERROR を返す。

保守担当者向けメモ

  • 本関数はリクエスト処理のセキュリティモデルの根幹であるため、処理順序の変更は認可バイパス等の脆弱性を招く可能性がある。
  • パフォーマンス向上のため、キャッシュ機構（walk_cache_t）のヒット率を維持することが重要である。
  • CASE_BLIND_FILESYSTEM 定義時は、大文字小文字を区別しないファイルシステム向けのパス解決ロジックが有効になる。
