substitute_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_substitute.c:365

目的

レスポンスボディに対して、設定された正規表現または文字列置換をオンザフライで適用する出力フィルタ。

シグネチャまたは形状

static apr_status_t substitute_filter(ap_filter_t *f, apr_bucket_brigade *bb)

入力と出力

  • **入力**: ap_filter_t *f (フィルタコンテキスト)、apr_bucket_brigade *bb (処理対象のデータバケット群)。
  • **出力**: apr_status_t (処理成功時は APR_SUCCESS、メモリ不足時は APR_ENOMEM 等)。

前提条件と不変条件

  • 初回呼び出し時に substitute_module_ctx が生成され、フィルタコンテキスト (f->ctx) に保持される。
  • Content-Length ヘッダーは、置換によるサイズ変動を考慮して本関数内で削除される。
  • ctx->tpool は各ループ終了時に apr_pool_clear され、メモリが再利用される。

アルゴリズムまたは振る舞い

1. **初期化**: フィルタコンテキストおよび一時的なバケットブリゲード（linebb, passbb 等）を確保。 2. **行単位のバッファリング**: 入力バケットを走査し、改行 (\n) が見つかるまで ctx->linebb に蓄積。 3. **置換実行**: 1行が完成した時点で do_pattmatch を呼び出し、置換処理を実行。

  • flatten モードまたはバケット分割モードで置換を行い、結果を ctx->passbb に格納。

    • 4. **転送**: 処理済みバケットを ap_pass_brigade で下流フィルタへ送出。 5. **安全装置**: ctx->passbb 内のバケット数が AP_MAX_BUCKETS を超えた場合、強制的にフラッシュしてメモリを解放。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache フィルタチェーン（ap_register_output_filter により登録）。

関連データ構造

  • substitute_module_ctx: フィルタ実行中の状態保持（linebb, passbb, tpool 等）。
  • subst_dir_conf: ディレクトリごとの設定（置換パターン、最大行長）。

失敗モードと境界条件

  • **メモリ制限**: 1行の長さが subst_dir_conf の max_line_length を超えた場合、APR_ENOMEM を返しエラーログを出力。
  • **断片化**: AP_MAX_BUCKETS を超えるバケット蓄積時には強制フラッシュを行い、メモリ枯渇を防止。
  • **空のブリゲード**: 入力 bb が空の場合は即座に APR_SUCCESS を返す。

保守担当者向けメモ

  • ctx->tpool は substitute_filter の各ループ終了時に apr_pool_clear されるため、このプールから確保したメモリの寿命に注意すること。
  • apr_brigade_split_line を使用せず、memchr による手動スキャンを採用しているのは、既存のバケットデータを再利用してパフォーマンスを最適化するためである。
