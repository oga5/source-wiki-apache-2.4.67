deflate_check_etag

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_deflate.c:500

目的

HTTP レスポンスの ETag ヘッダーを、コンテンツの圧縮（gzip）または伸長（gunzip）状態に合わせて更新または削除する。

シグネチャまたは形状

static void deflate_check_etag(request_rec *r, const char *transform, int etag_opt)

入力と出力

  • r: 現在のリクエスト構造体。headers_out テーブルの ETag を読み取り、更新する。
  • transform: 変換の種類を示す文字列（例: "gzip", "gunzip"）。新しい ETag の末尾に付加される。
  • etag_opt: ETag の処理方針（AP_DEFLATE_ETAG_REMOVE の場合、ヘッダーを削除する）。

前提条件と不変条件

  • ETag が存在し、かつ末尾がダブルクォーテーション（"）で終わっている場合にのみ、タグの書き換えが行われる。
  • 書き換え後の ETag は、元のタグの末尾の " を - に置き換え、transform 文字列と新たな " を付加した形式になる。

アルゴリズムまたは振る舞い

1. etag_opt が AP_DEFLATE_ETAG_REMOVE ならば、ETag を削除して終了する。 2. ETag が存在し、長さが 2 文字以上であることを確認する。 3. ETag の末尾が " である場合、元のタグの末尾の " を除いた部分に "-" + transform + " を連結した新しい文字列を生成する。 4. 生成した文字列を apr_table_setn を用いて headers_out の ETag に再設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/filters/mod_deflate.c:852 (圧縮処理時)
  • modules/filters/mod_deflate.c:1588 (伸長処理時)
  • 呼び出し先:
  • apr_table_get, apr_table_unset, apr_table_setn (Apache テーブル操作)
  • apr_palloc (メモリ割り当て)
  • strlen (文字列長取得)

関連データ構造

  • request_rec: Apache のリクエスト状態を保持する構造体。

失敗モードと境界条件

  • ETag が存在しない、または長さが 2 文字以下の場合は何も行わない。
  • ETag の末尾が " でない場合は、タグの書き換えを行わない。

保守担当者向けメモ

  • apr_palloc を使用しているため、メモリはリクエストプールから確保される。
  • ETag の形式が標準的な W/"..." や "..." であることを前提とした文字列操作を行っている。
