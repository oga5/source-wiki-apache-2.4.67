add_authn_provider

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_basic.c:87

目的

設定ファイルで指定された認証プロバイダーを検索・検証し、ディレクトリ単位の設定構造体（auth_basic_config_rec）内のプロバイダーリストに追加します。

シグネチャまたは形状

static const char *add_authn_provider(cmd_parms *cmd, void *config, const char *arg);

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: 現在のディレクトリ設定構造体（auth_basic_config_rec）。
  • arg: 設定ファイルから渡されるプロバイダー名。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列（const char *）を返す。

前提条件と不変条件

  • cmd->pool はメモリ割り当てに使用可能な有効なプールであること。
  • 指定されたプロバイダーは、呼び出し時点で ap_lookup_provider を通じてロード・登録済みであること。
  • プロバイダーは check_password 関数を実装している必要がある。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して新しい authn_provider_list ノードを割り当てる。 2. ap_lookup_provider を呼び出し、AUTHN_PROVIDER_GROUP から指定されたプロバイダーを検索・キャッシュする。 3. プロバイダーが見つからない場合、または check_password 関数をサポートしていない場合はエラーメッセージを返す。 4. 既存の conf->providers リストの末尾を走査し、新しいノードをリストの最後に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_basic.c (ディレクティブ AuthBasicProvider)
  • modules/aaa/mod_auth_digest.c (ディレクティブ AuthDigestProvider)
  • modules/aaa/mod_auth_form.c (ディレクティブ AuthFormProvider)
  • **呼び出し先**:
  • ap_lookup_provider (認証プロバイダーの取得)
  • apr_pcalloc (メモリ割り当て)
  • apr_psprintf (エラーメッセージ生成)

関連データ構造

  • auth_basic_config_rec: ディレクトリごとの認証設定を保持する構造体。
  • authn_provider_list: 認証プロバイダーの連結リスト。

失敗モードと境界条件

  • **プロバイダー未登録**: ap_lookup_provider が NULL を返した場合、「Unknown Authn provider」エラーを返す。
  • **機能不足**: プロバイダーが check_password を実装していない場合、「doesn't support Basic Authentication」エラーを返す。

保守担当者向けメモ

  • 本関数は複数のモジュールから共通して利用される設定ハンドラです。プロバイダーの追加順序が認証時の優先順位となるため、リストの末尾への追加処理は必須です。
