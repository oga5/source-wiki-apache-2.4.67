ap_queue_info_free_idle_pools

定義

  • 種別: function
  • 定義箇所: server/mpm_fdqueue.c:284

目的

fd_queue_info_t が管理する再利用可能なメモリプール（recycled_pools）をすべて解放し、関連するカウンタをリセットします。

シグネチャまたは形状

void ap_queue_info_free_idle_pools(fd_queue_info_t *queue_info)

入力と出力

  • **入力**: fd_queue_info_t *queue_info - 解放対象のキュー情報構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • queue_info は有効なポインタである必要があります。
  • 本関数実行後、queue_info->max_recycled_pools は 0 に設定され、再利用プールは空になります。

アルゴリズムまたは振る舞い

1. queue_info->max_recycled_pools を 0 に設定し、新たなプールの蓄積を停止します。 2. ap_queue_info_pop_pool をループ内で呼び出し、再利用リストからプールを取り出します。 3. 取り出したプールが NULL になるまで apr_pool_destroy(p) を実行してメモリを解放します。 4. 最後に apr_atomic_set32 を使用し、queue_info->recycled_pools_count を 0 にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c:1333
  • server/mpm/worker/worker.c:723
  • **呼び出し先**:
  • ap_queue_info_pop_pool (内部関数)
  • apr_pool_destroy (APR ライブラリ)
  • apr_atomic_set32 (APR ライブラリ)

関連データ構造

  • fd_queue_info_t: アイドル状態管理およびプール再利用リストを保持する構造体。
  • apr_pool_t: 解放対象となるメモリプール。

失敗モードと境界条件

  • queue_info が指す構造体が既に破棄されている場合や不正なポインタである場合、未定義の動作を引き起こします。
  • ap_queue_info_pop_pool がスレッドセーフな実装であるため、本関数は他のスレッドによるプール操作と競合しないよう設計されています。

保守担当者向けメモ

  • 本関数は主に MPM の終了処理やリソースクリーンアップ時に呼び出されます。
  • max_recycled_pools を 0 に設定することで、解放処理中に新たなプールがリストに追加されることを防いでいます。
