authz_req_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme_authz.c:61

目的

ACME ドメイン認証（Authorization）プロセスにおける、リクエスト処理中のコンテキスト情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_pool_t *p;
    md_acme_t *acme;
    const char *domain;
    md_acme_authz_t *authz;
    md_acme_authz_cha_t *challenge;
} authz_req_ctx;

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。authz_req_ctx_init を通じて各フィールドが初期化されます。

前提条件と不変条件

  • p は、このコンテキストが使用される期間中有効な apr_pool_t である必要があります。
  • 保持される各ポインタ（acme, authz, challenge）は、関連するライフサイクル管理に従い、有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

認証リクエストの実行中に、ドメイン名、ACME セッション、認証リソース、および現在処理中のチャレンジをひとまとめにして管理します。これにより、認証処理の各ステップ（セットアップ、応答、クリーンアップ）で必要な情報へのアクセスを簡素化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_authz.c 内の複数の関数（authz_req_ctx_init の呼び出し元、およびローカル変数として宣言している箇所）から参照されています。
  • **呼び出し先**:
  • 本構造体はデータコンテナであるため、直接的な呼び出し先はありません。

関連データ構造

  • md_acme_t: ACME サーバーとの通信セッション。
  • md_acme_authz_t: 認証リソースのメイン構造体。
  • md_acme_authz_cha_t: 個別のチャレンジ情報。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • 構造体内の各ポインタが NULL の状態で操作を行うと、セグメンテーションフォールトが発生する可能性があります。初期化関数 authz_req_ctx_init を経由して適切に設定することが求められます。

保守担当者向けメモ

  • 本構造体は modules/md/md_acme_authz.c 内で局所的に使用されるコンテキストオブジェクトです。
  • 構造体のフィールドを追加・変更する場合は、authz_req_ctx_init 関数内の初期化ロジックも併せて更新してください。
