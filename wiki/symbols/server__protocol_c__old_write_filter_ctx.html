old_write_filter_ctx

定義

  • 種別: struct
  • 定義箇所: server/protocol.c:2034

目的

出力フィルタチェーンにおいて、古い書き込みインターフェース（ap_rwrite 等）をサポートするためのバッファリング用コンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    apr_bucket_brigade *bb;
    apr_bucket_brigade *tmpbb;
} old_write_filter_ctx;

入力と出力

  • **入力**: フィルタ関数へ渡される ap_filter_t 構造体の ctx メンバとして保持されます。
  • **出力**: 保持する apr_bucket_brigade を通じて、後続のフィルタへデータが渡されます。

前提条件と不変条件

  • bb および tmpbb は、apr_bucket_brigade 型のポインタであり、有効なバケットブリゲードを指す必要があります。
  • 本構造体は、フィルタのコンテキストとして f->ctx に割り当てられて使用されます。

アルゴリズムまたは振る舞い

  • bb はメインのバッファとして機能し、書き込まれたデータを一時的に蓄積します。
  • tmpbb は、バケットの移動や一時的な処理が必要な際に補助的なブリゲードとして使用されます。
  • フィルタ処理の過程で、これらのブリゲードを介してデータのバッファリングと後続フィルタへの転送が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/protocol.c 内のフィルタ関数（ap_filter_t *f を引数に取る関数）において、f->ctx をキャストして取得されます。
  • **呼び出し先**: apr_bucket_brigade 操作関数（apr_brigade_... 系）が、この構造体内のブリゲードを操作します。

関連データ構造

  • apr_bucket_brigade: Apache HTTPD のバケットベース I/O で使用されるデータ保持構造。
  • ap_filter_t: 本構造体がコンテキストとして格納されるフィルタ構造体。

失敗モードと境界条件

  • メモリ不足: apr_bucket_brigade の生成やバケットの追加時に apr_pool_t からのメモリ確保が失敗する可能性があります。
  • フィルタチェーンの切断: フィルタが適切に初期化されていない場合、f->ctx が NULL となり、アクセス時にクラッシュする可能性があります。

保守担当者向けメモ

  • 本構造体は server/protocol.c 内の古い書き込みフィルタの実装に特化しています。
  • f->ctx を参照する際は、必ず old_write_filter_ctx 型へのキャストが正しく行われているか確認してください。
