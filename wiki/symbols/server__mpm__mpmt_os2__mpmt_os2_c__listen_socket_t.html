listen_socket_t

定義

  • 種別: struct
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:95

目的

OS/2 MPM において、親プロセスから子プロセスへリスナーソケットの情報を引き継ぐために使用されるデータ構造です。

シグネチャまたは形状

typedef struct {
    struct sockaddr_in name;
    apr_os_sock_t listen_fd;
} listen_socket_t;

入力と出力

  • **入力**: 親プロセスがソケットを生成する際に name（アドレス情報）と listen_fd（OS/2 ネイティブソケットハンドル）が設定されます。
  • **出力**: 子プロセスが共有メモリから本構造体を読み取り、接続受け入れに使用します。

前提条件と不変条件

  • 共有メモリ（parent_info_t）内に配置され、プロセス間で共有されることを前提としています。
  • listen_fd は APR のポータブルソケット型であり、OS/2 環境下で有効なハンドルである必要があります。

アルゴリズムまたは振る舞い

  • 親プロセスは、リスナーソケットの数に応じて本構造体の配列を共有メモリ上に確保します。
  • 子プロセスは、起動時にこの共有メモリをマッピングし、格納された listen_fd を使用してリクエストの待受を開始します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/mpmt_os2/mpmt_os2.c 内の親プロセス初期化処理および共有メモリ確保処理。
  • **呼び出し先**: 子プロセス内の ap_mpm_child_main（mpmt_os2_child.c 経由）が、この構造体からソケットハンドルを取得して利用します。

関連データ構造

  • parent_info_t: 本構造体を配列として保持し、共有メモリ上で管理するための親構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、子プロセスへのソケット引き継ぎが不可能となります。
  • num_listeners に基づいてメモリサイズが計算されるため（sizeof(listen_socket_t) を使用）、配列の境界外アクセスに注意が必要です。

保守担当者向けメモ

  • 本構造体は OS/2 特有のプロセス間通信に依存しています。他プラットフォームへの移植性は考慮されていません。
  • 構造体の変更は、親プロセスと子プロセス双方のメモリレイアウトに影響するため、共有メモリの確保ロジックと整合させる必要があります。
