req_aprtable2luatable

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:319

目的

Apache の apr_table_t 構造体を、Lua スクリプトから利用可能な2つの Lua テーブル（キー・値ペアのテーブルおよびキー・配列のテーブル）に変換してスタックへプッシュします。

シグネチャまたは形状

static int req_aprtable2luatable(lua_State *L, apr_table_t *t)

入力と出力

  • **入力**:
  • lua_State *L: Lua 実行状態。
  • apr_table_t *t: 変換対象となる APR テーブル。
  • **出力**:
  • 戻り値: 2（スタックにプッシュされたテーブルの数）。
  • 副作用: Lua スタック上に2つのテーブルが生成されます。

前提条件と不変条件

  • L は有効な Lua 状態である必要があります。
  • t は有効な apr_table_t ポインタである必要があります。
  • 呼び出し後、Lua スタックのトップには変換された2つのテーブルが配置されます。

アルゴリズムまたは振る舞い

1. lua_newtable(L) を呼び出し、最初のテーブル（キー・値ペア用）を作成します。 2. 再度 lua_newtable(L) を呼び出し、2番目のテーブル（キー・配列用）を作成します。 3. apr_table_do を使用して、コールバック関数 req_aprtable2luatable_cb をテーブル t の全要素に対して実行します。この際、Lua 状態 L をコールバックに渡します。 4. 最終的にスタックに2つのテーブルを残して 2 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_request.c 内の各ヘッダー/環境変数取得関数（headers_in, headers_out, err_headers_out, notes, subprocess_env の各ラッパー関数）。
  • **呼び出し先**:
  • apr_table_do (APR ライブラリ関数)
  • req_aprtable2luatable_cb (コールバック関数)

関連データ構造

  • apr_table_t: 変換元の Apache 内部テーブル構造。
  • lua_State: Lua VM の状態管理構造。

失敗モードと境界条件

  • t が NULL の場合、apr_table_do の挙動に依存します。
  • Lua VM のメモリ不足時には lua_newtable がエラーをスローする可能性があります。

保守担当者向けメモ

  • 本関数は req_aprtable2luatable_cb と密結合しています。テーブルの変換ロジックを変更する場合は、コールバック側の実装も併せて確認してください。
