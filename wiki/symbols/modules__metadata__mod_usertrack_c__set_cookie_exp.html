set_cookie_exp

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_usertrack.c:298

目的

Apache 設定ディレクティブ CookieExpires の値を解析し、Cookie の有効期限（秒単位）を計算して cookie_log_state 構造体に格納する。

シグネチャまたは形状

static const char *set_cookie_exp(cmd_parms *parms, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • parms: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていない引数。
  • arg: CookieExpires ディレクティブに指定された文字列（数値または期間指定）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列（const char *）を返す。

前提条件と不変条件

  • parms->server->module_config から usertrack_module の設定を取得可能であること。
  • 期間指定の解析には parms->temp_pool を使用する。

アルゴリズムまたは振る舞い

1. arg が数値のみで構成されている場合、その値をそのまま cls->expires に設定する。 2. 期間指定形式（例: plus 1 months 2 days）の場合、以下の手順で計算する。

  • オプションの plus をスキップする。
  • num（数値）と type（単位: years, months, weeks, days, hours, minutes, seconds）のペアを順次読み込む。
  • 各単位に対応する秒数を乗算し、累積値を cls->expires に格納する。

    • 3. 不正な形式や未定義の単位が指定された場合は、エラーメッセージを返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_usertrack.c:468 において、AP_INIT_TAKE1 マクロにより CookieExpires ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: ap_get_module_config（設定取得）、ap_getword_conf（文字列解析）、apr_isdigit、atoi、strncasecmp 等の標準/APR ユーティリティ関数。

関連データ構造

  • cookie_log_state: サーバー全体の設定を保持する構造体。expires メンバが本関数によって更新される。

失敗モードと境界条件

  • **数値のみ**: 文字列の先頭と末尾が数字でない場合、期間指定ロジックへ遷移する。
  • **解析エラー**:
  • 数値が期待される箇所に数値がない場合: "bad expires code, numeric value expected."
  • 単位が欠落している場合: "bad expires code, missing <type>"
  • 未知の単位が指定された場合: "bad expires code, unrecognized type"

保守担当者向けメモ

  • 本関数の期間計算ロジックは mod_expires.c から流用されている。
  • months は固定で 30 日として計算されるため、厳密なカレンダー上の月数とは異なる点に注意すること。
