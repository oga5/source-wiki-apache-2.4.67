tokens_to_array

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_socache.c:341

目的

カンマ区切りの文字列（例: Vary ヘッダー値）を解析し、正規化された配列として格納する。

シグネチャまたは形状

static void tokens_to_array(apr_pool_t *p, const char *data, apr_array_header_t *arr)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • data: 解析対象のカンマ区切り文字列。
  • arr: トークンを格納する apr_array_header_t 構造体。
  • **出力**: なし（arr を直接更新する）。

前提条件と不変条件

  • arr は事前に初期化されている必要がある。
  • data は ap_get_list_item で処理可能な形式であること。
  • 実行後、arr 内の要素は array_alphasort によってアルファベット順にソートされる。

アルゴリズムまたは振る舞い

1. ap_get_list_item を使用して、data からトークンを一つずつ抽出する。 2. 抽出した各トークンを apr_array_push を用いて arr に追加する。 3. 全てのトークン抽出後、qsort を呼び出し、array_alphasort を基準に配列内の要素をソートする。これにより、順序が異なるだけの同一リスト（例: "A, B" と "B, A"）を同一の配列状態に正規化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c:1034
  • modules/cache/mod_cache_socache.c:835
  • **呼び出し先**:
  • ap_get_list_item (APR/Apache ユーティリティ)
  • apr_array_push (APR)
  • qsort (標準ライブラリ)
  • array_alphasort (同ファイル内の比較関数)

関連データ構造

  • apr_array_header_t: トークンを保持する可変長配列構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • data が空または NULL の場合、ループは実行されず、arr は変更されない。
  • apr_array_push がメモリ不足等で失敗した場合の挙動は、APR の実装に依存する。

保守担当者向けメモ

  • 本関数は Vary ヘッダーの正規化を目的としており、キャッシュキー生成の一貫性を担保する重要な役割を担っている。
  • ソート順を変更する場合は array_alphasort の実装を確認すること。
