add_setup_token

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_order.c:170

目的

ACME Order オブジェクトに関連付けられたチャレンジセットアップトークンのリストを管理し、重複を防ぎつつ新しいトークンを追加します。

シグネチャまたは形状

static apr_status_t add_setup_token(md_acme_order_t *order, const char *token)

入力と出力

  • **入力**:
  • order: 操作対象の md_acme_order_t 構造体へのポインタ。
  • token: 追加するチャレンジセットアップトークン（文字列）。
  • **出力**:
  • APR_SUCCESS: 常に成功を返します。

前提条件と不変条件

  • order->challenge_setups は、apr_array_header_t として初期化されている必要があります。
  • order->p は、トークンの複製に使用される有効な apr_pool_t である必要があります。

アルゴリズムまたは振る舞い

1. md_array_str_index を使用して、指定された token が order->challenge_setups 配列内に既に存在するかを確認します。 2. 配列内に存在しない場合のみ、apr_pstrdup を用いてトークンを order->p プール上に複製し、APR_ARRAY_PUSH で配列に追加します。 3. 最後に APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_order.c:482
  • **呼び出し先**:
  • md_array_str_index (ユーティリティ関数)
  • apr_pstrdup (APR ライブラリ関数)
  • APR_ARRAY_PUSH (APR マクロ)

関連データ構造

  • md_acme_order_t: ACME Order の状態を保持する構造体。本関数はこの構造体内の challenge_setups 配列を操作します。

失敗モードと境界条件

  • order が NULL の場合、または challenge_setups が初期化されていない場合は未定義の動作（セグメンテーション違反等）を引き起こす可能性があります。
  • メモリ不足時は apr_pstrdup が失敗し、プロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は static であり、md_acme_order.c 内でのみ使用されます。
  • 重複排除のロジックは単純な文字列比較に基づいています。トークンの比較精度やパフォーマンスが問題となる場合は、md_array_str_index の仕様を確認してください。
