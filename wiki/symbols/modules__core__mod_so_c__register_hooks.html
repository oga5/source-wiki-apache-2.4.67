register_hooks

定義

  • 種別: function
  • 定義箇所: modules/core/mod_so.c:418

目的

Apache HTTP Server のフックシステムに対し、mod_so モジュールが提供する機能の登録およびオプション関数の公開を行う。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（フック登録に使用されるメモリプール）。
  • **出力**: なし（void）。

前提条件と不変条件

  • NO_DLOPEN が定義されていない環境でのみ、動的ロード関連のフックおよびオプション関数が登録される。

アルゴリズムまたは振る舞い

1. NO_DLOPEN が未定義の場合、以下の処理を実行する。

  • ap_find_loaded_module_symbol をオプション関数として登録し、他のモジュールから利用可能にする。
  • ap_hook_test_config を使用して、設定テスト時に dump_loaded_modules 関数が呼び出されるよう登録する（フックの優先順位は APR_HOOK_MIDDLE）。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール初期化プロセス（http_config.h で定義されたフック機構により呼び出される）。
  • **呼び出し先**:
  • APR_REGISTER_OPTIONAL_FN: ap_find_loaded_module_symbol を登録。
  • ap_hook_test_config: dump_loaded_modules をフックとして登録。
  • **注記**: ツールが提示した他のモジュール（mod_access_compat 等）内の register_hooks は同名の別関数であり、本シンボルの呼び出し元ではない。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • dump_loaded_modules: フックとして登録される関数。
  • ap_find_loaded_module_symbol: オプション関数として公開される関数。

失敗モードと境界条件

  • NO_DLOPEN が定義されている環境では、関数本体は空となり、何も登録されない。

保守担当者向けメモ

  • 本関数は mod_so モジュールのライフサイクルの一部として、サーバー起動時の設定フェーズで実行される。
  • ap_hook_test_config の追加や変更を行う際は、dump_loaded_modules の実行タイミングに注意すること。
