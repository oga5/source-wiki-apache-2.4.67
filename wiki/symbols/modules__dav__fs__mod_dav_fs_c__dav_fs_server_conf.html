dav_fs_server_conf

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/mod_dav_fs.c:25

目的

Apache HTTP Server の mod_dav_fs モジュールにおいて、サーバー単位の設定情報を保持するために使用される構造体です。主に WebDAV ロックデータベースのパスを管理します。

シグネチャまたは形状

typedef struct {
    const char *lockdb_path;
} dav_fs_server_conf;

入力と出力

  • **入力**: DAVLockDB ディレクティブによって指定されたパス文字列が lockdb_path に格納されます。
  • **出力**: dav_get_lockdb_path 等の関数を通じて、他のモジュールコンポーネントへロックデータベースのパスを提供します。

前提条件と不変条件

  • DAVLockDB はサーバー単位の設定であり、ディレクトリごとのオーバーライドは行われません。
  • 設定の継承は dav_fs_merge_server_config 関数によって制御されます。

アルゴリズムまたは振る舞い

  • Apache 起動時の設定読み込みフェーズにおいて、dav_fs_cmd_davlockdb ハンドラが DAVLockDB ディレクティブを処理し、その値を本構造体の lockdb_path に設定します。
  • メモリ割り当てには apr_pcalloc が使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_dav_fs.c 内の dav_fs_create_server_config（メモリ確保）
  • mod_dav_fs.c 内の dav_fs_merge_server_config（設定のマージ）
  • mod_dav_fs.c 内の dav_fs_cmd_davlockdb（設定値の書き込み）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • dav_fs_module: 本構造体を利用するモジュール定義構造体。

失敗モードと境界条件

  • lockdb_path が設定されていない場合、ロック機能の動作に影響を与える可能性があります。
  • 構造体自体のメモリ割り当て失敗については、APR のメモリ管理規約に従います。

保守担当者向けメモ

  • 本構造体は mod_dav_fs の設定管理の基盤です。設定の継承ロジックを変更する場合は dav_fs_merge_server_config を確認してください。
  • 構造体へのフィールド追加時は、dav_fs_create_server_config および dav_fs_merge_server_config での初期化・マージ処理の更新が必要です。
