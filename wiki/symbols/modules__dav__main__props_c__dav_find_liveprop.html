dav_find_liveprop

定義

  • 種別: function
  • 定義箇所: modules/dav/main/props.c:264

目的

XML 要素からライブプロパティ（liveprop）プロバイダーを特定し、その情報を要素のプライベートデータ領域にキャッシュします。

シグネチャまたは形状

static void dav_find_liveprop(dav_propdb *propdb, apr_xml_elem *elem)

入力と出力

  • **入力**:
  • propdb: プロパティデータベースのコンテキスト構造体。
  • elem: 検索対象となる XML 要素（apr_xml_elem）。elem->priv に結果が格納されます。
  • **出力**: なし（elem->priv 構造体の propid および provider フィールドを更新）。

前提条件と不変条件

  • elem->priv が dav_elem_private 型の有効なメモリ領域を指している必要があります。
  • propdb->ns_xlate が初期化されており、名前空間の変換が可能である必要があります。

アルゴリズムまたは振る舞い

1. **名前空間の解決**: elem->ns を基に、対応する URI 文字列（ns_uri）を特定します。DAV: 名前空間の場合は固定文字列を使用し、それ以外は propdb->ns_xlate を参照します。 2. **プロバイダー検索**: dav_find_liveprop_provider を呼び出し、ns_uri と要素名からプロパティ ID を取得します。 3. **キャッシュ**: 取得したプロパティ ID を priv->propid に格納します。 4. **フックの割り当て**: プロパティ ID が DAV_PROPID_CORE_UNKNOWN でない場合、対応するライブプロパティフック（hooks）を priv->provider に保存します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/props.c:789
  • modules/dav/main/props.c:978
  • **呼び出し先**:
  • dav_find_liveprop_provider

関連データ構造

  • dav_propdb: プロパティ操作のコンテキスト。
  • apr_xml_elem: 解析された XML 要素。
  • dav_elem_private: elem->priv として使用される、プロパティ解決結果を保持する構造体。

失敗モードと境界条件

  • elem->ns が APR_XML_NS_NONE の場合、ns_uri は NULL として扱われます。
  • プロバイダーが見つからない場合、priv->propid は DAV_PROPID_CORE_UNKNOWN となり、priv->provider は更新されません。

保守担当者向けメモ

  • ソースコード内のコメントにある通り、priv->propid != DAV_PROPID_CORE_UNKNOWN のチェックは冗長である可能性があります。
  • 本関数はライブプロパティの特定のみを行い、デッドプロパティの解決は別途行われます。
