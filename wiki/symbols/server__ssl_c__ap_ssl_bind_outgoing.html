ap_ssl_bind_outgoing

定義

  • 種別: function
  • 定義箇所: server/ssl.c:128

目的

アウトバウンド接続（プロキシ接続など）に対して SSL/TLS 設定を適用または解除する。

シグネチャまたは形状

int ap_ssl_bind_outgoing(conn_rec *c, struct ap_conf_vector_t *dir_conf, int enable_ssl)

入力と出力

  • **入力**:
  • c: 接続情報を保持する conn_rec 構造体。
  • dir_conf: ディレクトリ単位の設定ベクトル。
  • enable_ssl: SSL を有効にする場合は非ゼロ、無効にする場合は 0。
  • **出力**:
  • OK: 成功時。
  • DECLINED: SSL の有効化が要求されたが、フックおよびオプション関数で適用に失敗した場合。

前提条件と不変条件

  • 呼び出し時に c->outgoing が 1 に設定される。
  • SSL 関連のフックまたはオプション関数が登録されている必要がある。

アルゴリズムまたは振る舞い

1. c->outgoing を 1 に設定する。 2. ap_run_ssl_bind_outgoing フックを実行し、結果を判定する。 3. フックが成功しなかった場合、後方互換性のために古いオプション関数（module_ssl_engine_set または module_ssl_proxy_enable）を順次試行する。 4. enable_ssl が真で、かつ上記いずれの手段でも SSL が有効化できなかった場合、エラーログを出力し DECLINED を返す。 5. enable_ssl が偽の場合、既存のオプション関数を用いて SSL を無効化する。 6. 成功時は OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy.c
  • server/ssl.c (内部のラッパー関数)
  • **呼び出し先**:
  • ap_run_ssl_bind_outgoing (フック)
  • module_ssl_engine_set (オプション関数)
  • module_ssl_proxy_enable (オプション関数)
  • module_ssl_engine_disable (オプション関数)
  • ap_log_cerror (ログ出力)

関連データ構造

  • conn_rec: 接続状態を管理する構造体。
  • ap_conf_vector_t: 設定情報を保持するベクトル。

失敗モードと境界条件

  • enable_ssl が 1 で、フックおよびオプション関数がすべて未定義または失敗した場合、APLOG_ERR レベルでログが出力され、関数は DECLINED を返す。

保守担当者向けメモ

  • 本関数は SSL 実装の抽象化レイヤーであり、実際の暗号処理はフック先のモジュール（mod_ssl 等）に依存する。
  • module_ssl_* 系関数は、フックベースの API への移行期間における後方互換性を維持するためのものである。
