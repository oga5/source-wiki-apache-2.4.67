header_do

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reflector.c:33

目的

設定されたマッピングルールに基づき、リクエストヘッダーの値をレスポンスヘッダーへコピーします。

シグネチャまたは形状

static int header_do(void *dummy, const char *key, const char *value)

入力と出力

  • **入力**:
  • dummy: request_rec * 型のポインタ（void * として渡される）。
  • key: コピー対象のリクエストヘッダー名。
  • value: マッピング先のレスポンスヘッダー名。
  • **出力**:
  • 常に 1 を返します（apr_table_do のイテレーションを継続するため）。

前提条件と不変条件

  • dummy は有効な request_rec 構造体を指している必要があります。
  • key および value は apr_table_t で使用可能な文字列である必要があります。

アルゴリズムまたは振る舞い

1. dummy を request_rec *r にキャストします。 2. apr_table_get を使用し、リクエストヘッダー (r->headers_in) から key に対応する値を取得します。 3. 値が存在する場合、apr_table_setn を使用して、レスポンスヘッダー (r->headers_out) の value キーに対してその値を設定します。 4. 処理を終了し、1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_reflector.c:83 において apr_table_do(header_do, r, conf->headers, NULL); として呼び出されます。
  • **呼び出し先**:
  • apr_table_get (APR)
  • apr_table_setn (APR)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • apr_table_t: ヘッダーの格納に使用される APR のテーブル構造。

失敗モードと境界条件

  • リクエストヘッダーに key が存在しない場合、何もコピーされず、エラーにはならずに処理を続行します。

保守担当者向けメモ

  • 本関数は apr_table_do のコールバックとして設計されています。
  • apr_table_setn を使用しているため、値の文字列は永続的なメモリ領域（リクエストプール等）にあることが期待されます。
