collect_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.c:290

目的

ハッシュテーブルから要素を収集し、指定されたバッファへ格納するためのイテレータ用コンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    h2_ihash_t *ih;
    void **buffer;
    size_t max;
    size_t len;
} collect_ctx;

入力と出力

  • ih: 収集対象となる h2_ihash_t 型のハッシュテーブルへのポインタ。
  • buffer: 収集した要素を格納する void* 型の配列。
  • max: buffer が保持可能な最大要素数。
  • len: 現在までに収集された要素数。

前提条件と不変条件

  • buffer は少なくとも max 個の要素を格納できるメモリ領域を確保している必要があります。
  • len は常に 0 <= len <= max を維持します。

アルゴリズムまたは振る舞い

ハッシュテーブルの走査処理において、各要素を順次 buffer にコピーし、len をインクリメントすることで、ハッシュテーブル内のデータを配列形式へ抽出します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_util.c および modules/http2/h2_proxy_util.c 内の関数で、ハッシュテーブルの全要素を配列として取得する際に使用されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • h2_ihash_t: 本構造体が操作対象とする整数キーハッシュテーブル。

失敗モードと境界条件

  • len が max に達した場合、それ以上の要素は buffer に格納されません。
  • ih が NULL の場合、または buffer が不適切なメモリを指している場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は h2_util.c と h2_proxy_util.c の両方で定義・利用されています。変更を加える際は、両ファイル間での整合性に注意してください。
