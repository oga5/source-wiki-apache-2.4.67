proxy_balancer

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:113

目的

Apache HTTP Server のプロキシサブシステムにおいて、複数のバックエンドサーバー（proxy_worker）を束ね、負荷分散グループとして管理するための構造体です。

シグネチャまたは形状

typedef struct proxy_balancer proxy_balancer;

※構造体の内部メンバ定義は、本ヘッダーファイル内では不透明（opaque）として扱われています。

入力と出力

  • 入力: 負荷分散アルゴリズム（proxy_balancer_method）の各関数に対し、対象となる proxy_balancer インスタンスが引数として渡されます。
  • 出力: 負荷分散アルゴリズムの実行結果として、選択された proxy_worker が返されます。

前提条件と不変条件

  • 共有メモリ（ap_slotmem）を介してプロセス間で状態が共有される場合、proxy_mutex による排他制御が必要です。
  • 負荷分散グループ内の各ワーカーは、この構造体を通じて一貫した分散ポリシーに従います。

アルゴリズムまたは振る舞い

  • mod_lbmethod_bybusyness, mod_lbmethod_byrequests, mod_lbmethod_bytraffic などの各負荷分散モジュールによって、具体的な分散ロジックが適用されます。
  • 各アルゴリズムは、find_best 関数を通じて proxy_balancer を参照し、最適なワーカーを決定します。
  • reset や age といった操作を通じて、バランサーの統計情報や状態の更新が行われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • modules/proxy/balancers/mod_lbmethod_bybusyness.c
  • modules/proxy/balancers/mod_lbmethod_byrequests.c
  • modules/proxy/balancers/mod_lbmethod_bytraffic.c
  • include/ap_mmn.h (API バージョン管理用)
  • 呼び出し先:
  • 構造体内部の具体的なメンバや関数ポインタへのアクセスは、各負荷分散モジュールの実装に依存します。

関連データ構造

  • proxy_worker: バランサーが管理する個々のバックエンドサーバー。
  • proxy_balancer_method: バランサーが使用する負荷分散アルゴリズムのインターフェース。
  • proxy_server_conf: バランサーリストを保持するサーバー設定構造体。

失敗モードと境界条件

  • 共有メモリの競合: マルチプロセス環境で排他制御を欠いた場合、バランサーの状態が不整合になる可能性があります。
  • 構成の不備: バランサーにワーカーが登録されていない場合、リクエストの振り分けに失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_proxy の中心的なインターフェースの一部であり、バイナリ互換性に注意が必要です。
  • 構造体のメンバ定義は mod_proxy.h 以外で定義されている可能性があるため、詳細なフィールド構成を確認する場合は proxy_util.c や関連する mod_lbmethod_*.c を参照してください。
