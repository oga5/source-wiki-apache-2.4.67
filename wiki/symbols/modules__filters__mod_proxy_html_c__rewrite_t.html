rewrite_t

定義

  • 種別: enum
  • 定義箇所: modules/filters/mod_proxy_html.c:141

目的

HTML 要素の属性をどのように処理（書き換え）するかを分類するための列挙型です。

シグネチャまたは形状

typedef enum { ATTR_IGNORE, ATTR_URI, ATTR_EVENT } rewrite_t;

入力と出力

  • 入力: なし（列挙定数の定義）
  • 出力: ATTR_IGNORE（無視）、ATTR_URI（URIとして処理）、ATTR_EVENT（イベントハンドラとして処理）のいずれかの値。

前提条件と不変条件

  • mod_proxy_html における属性処理の判定基準として使用されます。

アルゴリズムまたは振る舞い

  • ATTR_IGNORE: 属性を書き換え対象から除外します。
  • ATTR_URI: 属性値を URI として扱い、設定されたルールに基づいて書き換えを試みます。
  • ATTR_EVENT: 属性値を JavaScript イベントハンドラとして扱い、内部の URI を抽出・書き換え対象とします。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_proxy_html.c:375 にて、is_uri という変数名でこの型が使用されていることが確認されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • tattr: 属性の定義や処理ルールを保持する構造体で使用される可能性があります。

失敗モードと境界条件

  • 列挙型であるため、定義外の値が代入された場合の動作は未定義です。

保守担当者向けメモ

  • 本シンボルは mod_proxy_html における属性処理のロジックを決定付ける重要な型です。新しい属性タイプを追加する場合は、この列挙型を拡張し、対応する処理ロジックを pstartElement 等の SAX コールバック内に実装する必要があります。
