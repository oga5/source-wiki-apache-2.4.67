non_regex_pattern

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_setenvif.c:190

目的

正規表現を含まない単純な文字列パターンを識別し、エスケープ文字を除去した文字列を返します。これにより、apr_strmatch を用いた高速な文字列検索を可能にします。

シグネチャまたは形状

static const char *non_regex_pattern(apr_pool_t *p, const char *s)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • s: 評価対象の文字列（正規表現パターン）。
  • **出力**:
  • 正規表現のメタ文字が含まれる場合: NULL。
  • 正規表現を含まない場合:
  • エスケープ文字が含まれない場合: 元の文字列 s へのポインタ。
  • エスケープ文字が含まれる場合: エスケープを除去した新しい文字列（p から割り当て）。

前提条件と不変条件

  • 入力文字列 s はヌル終端文字列である必要があります。
  • 正規表現のメタ文字（^, ., $, |, (, ), [, ], *, +, ?, {, }）がエスケープ（\）なしで出現した場合、正規表現とみなされます。

アルゴリズムまたは振る舞い

1. 文字列を走査し、正規表現メタ文字を検出します。 2. \（バックスラッシュ）が検出された場合、エスケープ状態を切り替えます。 3. エスケープ状態でないときにメタ文字が出現した場合は、即座に NULL を返します。 4. 全走査終了後、エスケープ文字が一度も出現していなければ、元の文字列をそのまま返します。 5. エスケープ文字が含まれていた場合は、apr_palloc でメモリを確保し、エスケープ文字を除去した文字列を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_setenvif.c:338 (add_setenvif_core 関数内)
  • **呼び出し先**: apr_palloc (APR ライブラリ)

関連データ構造

  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • 入力文字列が正規表現を含む場合、NULL を返して失敗（単純文字列ではないと判定）します。
  • メモリ不足時には apr_palloc が例外を発生させる可能性があります。

保守担当者向けメモ

  • 本関数は SetEnvIf ディレクティブの最適化パスで使用されます。
  • 正規表現のメタ文字セットはハードコードされているため、将来的に正規表現エンジンが拡張された場合は、このリストの更新が必要になる可能性があります。
