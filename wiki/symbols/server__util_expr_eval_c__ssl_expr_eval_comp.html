ssl_expr_eval_comp

定義

  • 種別: function
  • 定義箇所: server/util_expr_eval.c:346

目的

Apache HTTP Server の式評価エンジンにおいて、mod_ssl との互換性を維持するための文字列比較ロジックを提供します。

シグネチャまたは形状

static int ssl_expr_eval_comp(ap_expr_eval_ctx_t *ctx, const ap_expr_t *node)

入力と出力

  • **入力**:
  • ctx: 評価コンテキスト（ap_expr_eval_ctx_t）。
  • node: 評価対象の抽象構文木（AST）ノード（ap_expr_t）。
  • **出力**:
  • 比較結果に基づく真偽値（int 型の 0 または 1）。

前提条件と不変条件

  • node は有効な AST ノードである必要があります。
  • 評価中に生成される文字列は、ctx->p（リクエストプール）から割り当てられます。
  • 再帰的な評価深さは AP_EXPR_MAX_RECURSION（20）によって制限されます。

アルゴリズムまたは振る舞い

1. node から左右のオペランド（e1, e2）を取得します。 2. node->node_op に基づき、以下の比較演算子を処理します：

  • op_EQ, op_STR_EQ: 等価比較
  • op_NE, op_STR_NE: 非等価比較
  • op_LT, op_STR_LT: 小なり比較
  • op_LE, op_STR_LE: 小なりイコール比較
  • op_GT, op_STR_GT: 大なり比較
  • op_GE, op_STR_GE: 大なりイコール比較

    • 3. 比較には strcmplex 関数を使用し、オペランドの評価には ap_expr_eval_word を呼び出します。 4. 上記以外の演算子の場合は、標準の ap_expr_eval_comp に処理を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_expr_eval.c:799 で呼び出されていることが確認されています。
  • **呼び出し先**: ap_expr_eval_word、strcmplex、および ap_expr_eval_comp を呼び出します。

関連データ構造

  • ap_expr_eval_ctx_t: 評価コンテキスト。
  • ap_expr_t: AST ノード構造体。

失敗モードと境界条件

  • strcmplex の挙動に依存します。
  • 評価対象の式が複雑すぎる場合、再帰制限により評価が中断される可能性があります。

保守担当者向けメモ

  • 本関数は歴史的な mod_ssl の挙動を維持するためのものです。
  • 正規表現の実行は util_pcre.c に委譲されており、本関数では扱いません。
  • req_table_func 等の関数がオペランドに含まれる場合、Vary ヘッダーの更新という副作用が発生する可能性があるため注意してください。
