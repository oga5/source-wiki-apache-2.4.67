h2_c1_child_stopping

定義

  • **種別**: function
  • **定義箇所**: modules/http2/h2_c1.c:84

目的

Apache HTTP Server の子プロセス終了時に、HTTP/2 接続処理で使用されるワーカーリソースを解放・停止します。

シグネチャまたは形状

void h2_c1_child_stopping(apr_pool_t *pool, int graceful)

入力と出力

  • **入力**:
  • pool: プロセスに関連付けられたメモリプール。
  • graceful: 終了モードを示すフラグ（0: 即時終了、1: グレースフル終了）。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 workers が初期化されている場合にのみ、シャットダウン処理が実行されます。

アルゴリズムまたは振る舞い

1. モジュール内の workers オブジェクトが存在するか確認します。 2. h2_workers_shutdown を呼び出し、graceful フラグに従ってワーカープールのシャットダウン処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c:197 にて ap_hook_child_stopping に登録されています。
  • **呼び出し先**: h2_workers_shutdown（h2_workers.h で定義）。

関連データ構造

  • h2_workers: HTTP/2 接続処理を並列化するためのワーカースレッドプール。

失敗モードと境界条件

  • workers が NULL の場合、本関数は何の処理も行わず正常に終了します。

保守担当者向けメモ

  • 本関数は Apache の child_stopping フックとして動作します。プロセス終了時のリソースリークを防ぐため、ワーカープールの適切なクリーンアップを保証する役割を担っています。
