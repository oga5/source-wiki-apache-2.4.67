cache_socache_info_t

定義

  • 種別: struct
  • 定義箇所: modules/cache/cache_socache_common.h:36

目的

共有オブジェクトキャッシュ（socache）を利用したキャッシュ機構において、キャッシュされたレスポンスのメタデータを保持し、キャッシュの有効性検証やレスポンスの再構築を支援するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_uint32_t format;
    int status;
    apr_size_t name_len;
    apr_size_t entity_version;
    apr_time_t date;
    apr_time_t expire;
    apr_time_t request_time;
    apr_time_t response_time;
    unsigned int header_only:1;
    cache_control_t control;
} cache_socache_info_t;

入力と出力

  • **入力**: キャッシュ書き込み時に、HTTP レスポンスのメタデータ（ステータス、時刻、キャッシュ制御情報等）が本構造体に格納されます。
  • **出力**: キャッシュ読み出し時に、本構造体がデシリアライズされ、キャッシュの鮮度判定やレスポンスの再構築に利用されます。

前提条件と不変条件

  • header_only ビットフィールドは、レスポンスボディを含まないヘッダーのみのキャッシュを識別するために使用されます。
  • キャッシュフォーマットを変更する場合、CACHE_SOCACHE_DISK_FORMAT_VERSION の更新が必要となる場合があります。

アルゴリズムまたは振る舞い

mod_cache_socache は、キャッシュの書き込み時に本構造体をシリアライズしてストレージに保存し、読み出し時にバッファから memcpy を用いてデシリアライズします。これにより、キャッシュされたエンティティのメタデータを一貫した形式で管理します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**: modules/cache/mod_cache_socache.c 内の複数の箇所で、キャッシュ情報の読み書きおよびメモリレイアウトの計算に使用されています。
  • **呼び出し先**: 本構造体はデータ定義であり、関数を呼び出すことはありません。

関連データ構造

  • cache_control_t: HTTP キャッシュ制御情報を保持するために構造体内に埋め込まれています。
  • apr_time_t: 時間情報の管理に使用されます。

失敗モードと境界条件

  • バッファサイズが sizeof(cache_socache_info_t) 未満の場合、読み出し処理（mod_cache_socache.c:602）において不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cache_socache のキャッシュフォーマットの根幹を成します。フィールドの追加や変更を行う際は、既存のキャッシュとの互換性に注意してください。
  • mod_cache_socache.c において、sizeof(cache_socache_info_t) を用いてメモリ上のオフセット計算が行われているため、構造体のパディングやサイズ変更には慎重を期してください。
