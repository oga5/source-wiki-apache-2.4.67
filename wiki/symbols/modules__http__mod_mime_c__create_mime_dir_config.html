create_mime_dir_config

定義

  • 種別: function
  • 定義箇所: modules/http/mod_mime.c:118

目的

ディレクトリごとの MIME 設定構造体（mime_dir_config）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_mime_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • p: 設定の割り当てに使用する apr_pool_t。
  • dummy: 使用されない引数（Apache の設定コールバックシグネチャに適合させるためのもの）。
  • **出力**:
  • 初期化済みの mime_dir_config 構造体へのポインタ。

前提条件と不変条件

  • apr_palloc により、指定されたプール p からメモリが確保される。
  • 戻り値の構造体は、ハッシュテーブルや言語設定が NULL で初期化され、use_path_info が 2 に設定される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して mime_dir_config サイズのメモリを確保する。 2. extension_mappings および remove_mappings を NULL に設定する。 3. default_language を NULL に設定する。 4. multimatch を MULTIMATCH_UNSET に設定する。 5. use_path_info を 2 に設定する。 6. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/mod_mime.c:1031（ディレクトリごとの設定構造体作成用として登録されている）。
  • **呼び出し先**: apr_palloc（APR ライブラリ）。

関連データ構造

  • mime_dir_config: 本関数が初期化対象とするディレクトリ設定構造体。

失敗モードと境界条件

  • apr_palloc がメモリ不足により失敗した場合、APR の仕様に従い例外またはプロセス終了が発生する可能性がある。

保守担当者向けメモ

  • use_path_info のデフォルト値 2 は、このモジュールにおける特定の挙動を制御するマジックナンバーであるため、変更時は関連する find_ct 等のロジックへの影響を考慮すること。
