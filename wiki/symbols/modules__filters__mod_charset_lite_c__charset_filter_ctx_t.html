charset_filter_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_charset_lite.c:86

目的

mod_charset_lite フィルタの各インスタンスにおける実行時状態を保持するためのコンテキスト構造体です。文字セット変換処理の継続性、エラー状態、およびバッファリングを管理します。

シグネチャまたは形状

typedef struct charset_filter_ctx_t {
    apr_xlate_t *xlate;
    int is_sb;
    charset_dir_t *dc;
    ees_t ees;
    apr_size_t saved;
    char buf[FATTEST_CHAR];
    int ran;
    int noop;
    char *tmp;
    apr_bucket_brigade *bb;
    apr_bucket_brigade *tmpbb;
} charset_filter_ctx_t;

入力と出力

  • **入力**: フィルタチェーンを流れる apr_bucket_brigade を処理対象とします。
  • **出力**: 変換済みのデータを含むバケットを tmpbb を介して下流へ渡します。

前提条件と不変条件

  • buf は FATTEST_CHAR（8バイト）のサイズを持ち、バケット境界を跨ぐマルチバイト文字を保持するために使用されます。
  • フィルタインスタンスごとに個別に割り当てられ、リクエストプール (r->pool) に紐付けられます。

アルゴリズムまたは振る舞い

  • xlate を用いてバケット単位の文字コード変換を実行します。
  • マルチバイト文字がバケット境界で分断された場合、set_aside_partial_char を用いて buf に一時保存し、次回のバケットと結合して変換を試みます。
  • noop フラグが設定されている場合、変換を行わずにバケットをそのまま通過させます。
  • ran フラグにより、フィルタが既に実行済みであるかを追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: xlate_out_filter および xlate_in_filter がこの構造体をフィルタコンテキスト (f->ctx) として参照します。
  • **呼び出し先**: set_aside_partial_char および finish_partial_char が本構造体を引数として受け取り、部分的な文字の処理を行います。

関連データ構造

  • apr_xlate_t: 実際の文字セット変換ハンドル。
  • charset_dir_t: ディレクトリ単位の設定情報。
  • ees_t: 拡張エラー状態の保持。
  • apr_bucket_brigade: フィルタ間でデータを渡すためのバケットリスト。

失敗モードと境界条件

  • **バッファ制限**: 8バイト (FATTEST_CHAR) を超えるマルチバイト文字がバケットを跨ぐ場合、EES_LIMIT エラーが発生します。
  • **バケット境界**: 現在の実装では、マルチバイト文字が2つのバケットを跨ぐケースまでを処理対象としています。

保守担当者向けメモ

  • 本構造体はリクエストごとに生成されます。メモリリークを防ぐため、リクエストプールによる適切な管理が必須です。
  • Content-Length ヘッダーは変換によってサイズが変化する可能性があるため、本コンテキストを使用するフィルタでは削除される挙動に注意してください。
