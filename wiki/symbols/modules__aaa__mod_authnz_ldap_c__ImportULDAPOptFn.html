ImportULDAPOptFn

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_ldap.c:2040

目的

mod_ldap モジュールが提供するオプション関数（Optional Functions）を検索し、mod_authnz_ldap 内のグローバルな関数ポインタ変数に割り当てることで、LDAP 関連機能へのアクセスを確立します。

シグネチャまたは形状

static void ImportULDAPOptFn(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として、モジュール内の静的関数ポインタ変数を更新）

前提条件と不変条件

  • APR_RETRIEVE_OPTIONAL_FN マクロにより、指定された関数名が mod_ldap からエクスポートされている必要があります。
  • 本関数は、Apache の optional_fn_retrieve フックを通じて呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. APR_RETRIEVE_OPTIONAL_FN を使用して、以下の util_ldap 関連関数ポインタを取得します。

  • util_ldap_connection_close
  • util_ldap_connection_find
  • util_ldap_cache_comparedn
  • util_ldap_cache_compare
  • util_ldap_cache_checkuserid
  • util_ldap_cache_getuserdn
  • util_ldap_ssl_supported
  • util_ldap_cache_check_subgroups

    • 2. 取得したポインタを対応する静的変数に格納し、以降の認証・認可処理で利用可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: ap_hook_optional_fn_retrieve により登録され、Apache の初期化フェーズで呼び出されます（modules/aaa/mod_authnz_ldap.c:2087）。
  • 呼び出し先: APR_RETRIEVE_OPTIONAL_FN マクロ（Apache APR のオプション関数検索機構）。

関連データ構造

  • util_ldap が提供する関数ポインタ群（util_ldap_connection_close 等）。

失敗モードと境界条件

  • mod_ldap がロードされていない場合、または対象の関数がエクスポートされていない場合、取得される関数ポインタは NULL となります。その後の実行時にこれらのポインタを呼び出すとセグメンテーション違反が発生する可能性があるため、利用側での NULL チェックが必要です。

保守担当者向けメモ

  • 本関数は mod_authnz_ldap が mod_ldap に依存していることを示す重要な接合点です。
  • 新たな util_ldap 関数を利用する必要が生じた場合は、本関数に APR_RETRIEVE_OPTIONAL_FN を追加してください。
