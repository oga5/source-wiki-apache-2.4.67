md_reg_create

定義

  • 種別: function
  • 定義箇所: modules/md/md_reg.c:98

目的

管理対象ドメイン（Managed Domains: MD）のレジストリインスタンス md_reg_t を初期化し、永続化層である md_store_t と各種ポリシー設定をバインドする。

シグネチャまたは形状

apr_status_t md_reg_create(md_reg_t **preg, apr_pool_t *p, struct md_store_t *store,
                           const char *proxy_url, const char *ca_file,
                           apr_time_t min_delay, int retry_failover,
                           int use_store_locks, apr_time_t lock_wait_timeout)

入力と出力

  • **入力**:
  • preg: 生成されたレジストリ構造体へのポインタを格納する出力用引数。
  • p: メモリ管理に使用する apr_pool_t。
  • store: ドメイン設定の永続化を担う md_store_t インスタンス。
  • proxy_url, ca_file: プロキシ設定およびCA証明書ファイルパス（オプション）。
  • min_delay, retry_failover: 再試行ポリシー設定。
  • use_store_locks, lock_wait_timeout: ストア操作時のロック制御設定。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時はエラーコードを返し、preg は NULL に設定される。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • store は初期化済みの md_store_t であること。
  • 内部で md_acme_protos_add が呼び出され、プロトコルハンドラが登録される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して md_reg_t 構造体をメモリプール上に確保し、各フィールドを初期化する。 2. proxy_url および ca_file が指定されている場合、プール内に複製して保持する。 3. md_timeslice_create を用いて、証明書の更新および警告ウィンドウ（renew_window, warn_window）を生成する。 4. md_acme_protos_add を呼び出し、利用可能なプロトコルを登録する。 5. load_props を呼び出し、ストアから既存のプロパティを読み込む。 6. 成功時に preg にレジストリインスタンスを格納して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c:896
  • **呼び出し先**:
  • apr_pcalloc, apr_hash_make, apr_pstrdup, apr_cstr_casecmp (APR)
  • md_timeslice_create
  • md_acme_protos_add
  • load_props (内部関数)

関連データ構造

  • md_reg_t: レジストリのメイン構造体。
  • md_store_t: 永続化層の抽象インターフェース。

失敗モードと境界条件

  • メモリ確保に失敗した場合、または load_props 等の内部処理でエラーが発生した場合、APR_SUCCESS 以外のステータスが返される。
  • ca_file が "none" という文字列である場合、ca_file フィールドは NULL として扱われる。

保守担当者向けメモ

  • md_reg_t は生成時に渡された apr_pool_t を所有し、キャッシュや設定の生存期間を管理する。
  • load_props の失敗はレジストリ生成全体の失敗として扱われる。
