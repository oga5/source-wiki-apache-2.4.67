filter_trace

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_filter.c:77

目的

フィルタチェーンを通過するバケットブリゲードの内容をデバッグログに出力し、フィルタの動作を追跡する。

シグネチャまたは形状

static void filter_trace(conn_rec *c, int debug, const char *fname, apr_bucket_brigade *bb)

入力と出力

  • c: 接続情報 (conn_rec*)。ログ出力のコンテキストとして使用。
  • debug: デバッグレベル。0 の場合は何もしない。1 の場合は詳細なバケット情報をログ出力する。
  • fname: フィルタ名（文字列）。ログの識別子として使用。
  • bb: 調査対象のバケットブリゲード (apr_bucket_brigade*)。

前提条件と不変条件

  • bb は有効なバケットブリゲードであること。
  • debug が 1 の場合、bb 内の各バケットの type および length が参照される。

アルゴリズムまたは振る舞い

1. debug 値を判定する。0 なら即座にリターンする。 2. debug が 1 の場合:

  • フィルタ名 fname をデバッグログに出力する。
  • bb 内のバケットを先頭から末尾まで走査する。
  • 各バケットについて、バケットタイプ名（不明な場合は (unknown)）とデータ長をログに出力する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_filter.c:289 にて、フィルタ実行時のトレースとして呼び出されている。
  • 呼び出し先: Apache の ap_log_cerror マクロを呼び出し、ログを出力する。

関連データ構造

  • conn_rec: Apache の接続コンテキスト。
  • apr_bucket_brigade: フィルタ間でデータを渡すためのバケットの連結リスト。
  • apr_bucket: データの断片を保持する構造体。

失敗モードと境界条件

  • bb が空の場合、ループは実行されずログ出力は行われない。
  • b->type->name が NULL の場合、(unknown) という文字列で代替される。

保守担当者向けメモ

  • 本関数は mod_filter のデバッグ用ユーティリティであり、運用環境でのパフォーマンスへの影響を避けるため、debug レベルの制御が必須である。
  • ログ出力には APLOGNO が付与されており、エラーログの識別に使用される。
