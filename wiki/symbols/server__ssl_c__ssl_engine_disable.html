ssl_engine_disable

定義

  • 種別: function
  • 定義箇所: server/ssl.c:123

目的

指定された接続（conn_rec）に対して、SSL エンジンを無効化する。

シグネチャまたは形状

int ssl_engine_disable(conn_rec *c)

入力と出力

  • **入力**: c (接続情報へのポインタ conn_rec *)
  • **出力**: int 型のステータスコード。内部で呼び出される ap_ssl_bind_outgoing の戻り値をそのまま返す。

前提条件と不変条件

  • 引数 c は有効な conn_rec 構造体を指している必要がある。
  • 本関数は ap_ssl_bind_outgoing のラッパーであり、SSL 設定を無効化するために第2引数に NULL、第3引数に 0 を渡して呼び出す。

アルゴリズムまたは振る舞い

1. 引数として受け取った conn_rec *c を使用して、ap_ssl_bind_outgoing(c, NULL, 0) を呼び出す。 2. ap_ssl_bind_outgoing の実行結果を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し先**: ap_ssl_bind_outgoing (同一ファイル server/ssl.c 内)
  • **呼び出し元**:
  • modules/arch/netware/mod_nw_ssl.c (オプション関数として登録・利用)
  • modules/proxy/mod_proxy.c (オプション関数として取得・利用)
  • server/ssl.c (オプション関数として登録)

関連データ構造

  • conn_rec: Apache HTTP Server の接続状態を保持する構造体。

失敗モードと境界条件

  • ap_ssl_bind_outgoing が失敗した場合、そのエラーコードがそのまま返される。
  • c が NULL の場合の挙動は、ap_ssl_bind_outgoing の実装に依存する。

保守担当者向けメモ

  • 本関数はオプション関数（Optional Function）として公開されており、他のモジュールから動的に利用される。
  • server/ssl.c 内で APR_REGISTER_OPTIONAL_FN(ssl_engine_disable) により登録されている。
