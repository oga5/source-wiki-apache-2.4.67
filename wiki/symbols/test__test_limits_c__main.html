定義

目的

HTTPサーバーの境界値処理能力を検証するための負荷試験ツールとして、異常な長さのHTTPリクエストを生成・送信し、サーバーの堅牢性をテストします。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • コマンドライン引数: -t (テストタイプ: r, n, h, b), -a (ホスト名), -p (ポート番号), -n (反復回数)。
  • **出力**:
  • 標準出力: サーバーからの応答内容。
  • 標準エラー出力: 実行状況およびエラーメッセージ。
  • 終了コード: 成功時は 0、エラー発生時は 1。

前提条件と不変条件

  • 指定されたホストへのTCP接続が可能であること。
  • apr_ssize_t 型が利用可能な環境であること。
  • サーバー側がHTTPリクエストを解析する準備ができていること。

アルゴリズムまたは振る舞い

1. **初期化**: デフォルト値（localhost:80, 1000回）を設定し、getopt で引数を解析。 2. **接続**: gethostbyname でホストを解決し、socket および connect でTCP接続を確立。 3. **攻撃実行**: testtype に応じて以下のいずれかの異常リクエストをストリーム送信。

  • TEST_LONG_REQUEST_LINE: リクエスト行を過剰に延長。
  • TEST_LONG_REQUEST_FIELDS: ヘッダーフィールド数を過剰に増大。
  • TEST_LONG_REQUEST_FIELDSIZE: 単一ヘッダー値を過剰に延長。
  • TEST_LONG_REQUEST_BODY: ボディサイズを偽装し、過剰なデータを送信。

    • 4. **応答処理**: read を用いてサーバーからの応答を読み取り、標準出力へ転送。 5. **終了**: fclose でストリームを閉じ、プログラムを終了。

呼び出し元と呼び出し先

  • **呼び出し元**: OSのシェル（コマンドラインから直接実行）。
  • **呼び出し先**: getopt, atoi, gethostbyname, perror, exit, memset, memcpy, htons, socket, connect, fdopen, fprintf, fflush, read, write, fclose, usage。
  • **注記**: ツールによる解析では、本シンボルを呼び出している他のソースコードは特定されていません。

関連データ構造

  • struct sockaddr_in: 接続先アドレス情報の保持に使用。
  • struct hostent: ホスト名解決結果の保持に使用。
  • FILE *: サーバーとの通信用ストリーム。

失敗モードと境界条件

  • ホスト解決失敗、ソケット作成失敗、接続失敗時に perror を出力して exit(1)。
  • fprintf 中にエラーが発生した場合、ferror で検知し exit(1)。
  • num 回数のループ中にサーバーが切断した場合、ferror により異常終了。

保守担当者向けメモ

  • 本ツールは1998年当時のDoS攻撃手法を再現した歴史的なテストツールです。
  • 現代の環境で実行する場合、WAF等により遮断される可能性がある点に留意してください。
  • apr_ssize_t を使用しているため、コンパイルにはAPR環境が必要です。
