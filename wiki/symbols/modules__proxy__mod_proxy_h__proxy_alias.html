proxy_alias

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:134

目的

URLのエイリアス（別名）変換ルールを保持するための構造体です。リクエストされたパス（fake）をバックエンドのパス（real）へマッピングする際に使用されます。

シグネチャまたは形状

struct proxy_alias {
    const char  *real;
    const char  *fake;
    ap_regex_t  *regex;
    unsigned int flags;
    proxy_balancer *balancer;
};

入力と出力

  • **入力**: 設定ファイル等から読み込まれたパス情報や正規表現。
  • **出力**: ap_proxy_trans_match 等の関数を通じて、リクエストの転送先決定やパス変換に利用されます。

前提条件と不変条件

  • balancer フィールドは、リバースプロキシとして使用される場合にのみ有効です。
  • regex が設定されている場合、パスのマッチングに正規表現が使用されます。

アルゴリズムまたは振る舞い

  • mod_proxy において、リクエストパスと設定されたエイリアスを照合し、一致した場合は real パスへの書き換えや、関連付けられた balancer を用いた負荷分散処理へ移行します。
  • apr_array_make を用いて配列として管理されることが一般的です。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • modules/proxy/mod_proxy.c: ap_proxy_trans_match 関数内でのマッチング処理。
  • modules/proxy/mod_proxy.c: 設定読み込み時の配列生成および初期化処理。
  • **呼び出し先**:
  • ap_regex_t（Apache 正規表現ライブラリ）
  • proxy_balancer（負荷分散構造体）

関連データ構造

  • proxy_balancer: リバースプロキシ設定時に紐付けられる負荷分散グループ。
  • ap_regex_t: パス照合に使用される正規表現オブジェクト。

失敗モードと境界条件

  • balancer が NULL の場合、リバースプロキシ以外の用途（単純なパス変換等）として扱われます。
  • 正規表現のコンパイルに失敗した場合、または配列の初期化が不適切な場合、リクエストのルーティングが正しく行われない可能性があります。

保守担当者向けメモ

  • 本構造体は mod_proxy の設定管理において頻繁に配列として利用されます。
  • flags フィールドは ap_mmn.h (2.3.0-dev) にて追加されており、バイナリ互換性に影響を与える可能性があるため、変更時は注意してください。
