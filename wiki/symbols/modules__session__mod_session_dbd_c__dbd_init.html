dbd_init

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_dbd.c:59

目的

データベース接続を取得し、指定されたクエリラベルに対応するプリペアドステートメントを検証・取得する。

シグネチャまたは形状

apr_status_t dbd_init(request_rec *r, const char *query, ap_dbd_t **dbdp, apr_dbd_prepared_t **statementp)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • query: 実行対象の SQL クエリラベル。
  • **出力**:
  • dbdp: 取得したデータベース接続 (ap_dbd_t) へのポインタ。
  • statementp: 取得したプリペアドステートメント (apr_dbd_prepared_t) へのポインタ。
  • **戻り値**: 成功時は APR_SUCCESS、失敗時は APR_EGENERAL を返す。

前提条件と不変条件

  • mod_dbd がサーバーにロードされている必要がある。
  • session_dbd_prepare_fn および session_dbd_acquire_fn が初期化されていること。

アルゴリズムまたは振る舞い

1. オプション関数 ap_dbd_prepare および ap_dbd_acquire が未取得の場合、APR_RETRIEVE_OPTIONAL_FN を用いて取得を試みる。取得失敗時はエラーログを出力し終了する。 2. session_dbd_acquire_fn(r) を呼び出し、データベース接続を取得する。取得失敗時はエラーログを出力し終了する。 3. 取得した接続内の prepared ハッシュから、引数 query に一致するプリペアドステートメントを検索する。 4. ステートメントが見つからない場合はエラーログを出力し終了する。 5. 成功時、dbdp および statementp に結果を格納し、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • dbd_load (modules/session/mod_session_dbd.c:120)
  • dbd_save (modules/session/mod_session_dbd.c:276, 306)
  • dbd_remove (modules/session/mod_session_dbd.c:356)
  • **呼び出し先**:
  • ap_dbd_prepare (オプション関数)
  • ap_dbd_acquire (オプション関数)
  • apr_hash_get (APR ライブラリ)
  • ap_log_rerror (Apache HTTP Server API)

関連データ構造

  • request_rec: リクエストごとのコンテキスト。
  • ap_dbd_t: mod_dbd が管理するデータベース接続構造体。
  • apr_dbd_prepared_t: プリペアドステートメントを保持する APR 型。

失敗モードと境界条件

  • mod_dbd がロードされていない場合、初期化に失敗し APR_EGENERAL を返す。
  • データベース接続の取得に失敗した場合、APR_EGENERAL を返す。
  • 指定されたクエリラベルが mod_dbd に登録されていない場合、ステートメントの検索に失敗し APR_EGENERAL を返す。

保守担当者向けメモ

  • 本関数は mod_dbd のオプション関数に依存しているため、モジュールのロード順序や依存関係に注意すること。
  • プリペアドステートメントの検索は APR_HASH_KEY_STRING を使用しているため、クエリラベルの文字列一致が必須である。
