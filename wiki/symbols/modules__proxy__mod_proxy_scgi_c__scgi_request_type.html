scgi_request_type

定義

  • 種別: enum
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:55

目的

SCGI プロキシ処理において、バックエンドから要求された特殊なリクエスト処理の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    scgi_internal_redirect,
    scgi_sendfile
} scgi_request_type;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: scgi_internal_redirect または scgi_sendfile のいずれかの値。

前提条件と不変条件

  • scgi_request_config 構造体内の type フィールドとして保持されます。
  • 定義された値は、バックエンドからのレスポンスヘッダーに基づいた後続処理の分岐に使用されます。

アルゴリズムまたは振る舞い

  • scgi_internal_redirect: バックエンドが要求した内部リダイレクト処理を識別します。
  • scgi_sendfile: バックエンドが要求したファイル送信（X-Sendfile）処理を識別します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/mod_proxy_scgi.c:62 にて scgi_request_config 構造体のメンバとして参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • scgi_request_config: この列挙型をメンバとして保持する、リクエストごとの設定構造体。

失敗モードと境界条件

  • 列挙型に含まれない値が代入された場合の挙動については、ソースコード内に明示的なガード条件は確認できません。

保守担当者向けメモ

  • 本列挙型は、mod_proxy_scgi におけるレスポンス処理の拡張ポイントです。新しい特殊処理を追加する場合は、この列挙型に値を追加し、scgi_request_status フック内での処理分岐を更新してください。
