register_hooks

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_ext_filter.c:357

目的

Apache HTTP Server のフックシステムに対し、モジュールの初期化処理を登録します。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p （フック登録に使用されるメモリプール）
  • 出力: なし（void）

前提条件と不変条件

  • Apache のモジュールロードプロセスにおいて、register_hooks 関数が呼び出される必要があります。
  • ef_init 関数が定義されている必要があります。

アルゴリズムまたは振る舞い

1. ap_hook_post_config を呼び出します。 2. 設定読み込み後のフェーズ（post_config）で ef_init 関数が実行されるよう登録します。 3. 実行優先順位として APR_HOOK_MIDDLE を指定します。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュール読み込み機構（直接的な呼び出し元はソースコード上では明示されていませんが、Apache の標準的なモジュール構造に基づきます）。
  • 呼び出し先: ap_hook_post_config（Apache API）、ef_init（本モジュール内の初期化関数）。

関連データ構造

  • apr_pool_t: Apache のメモリ管理用プール。

失敗モードと境界条件

  • フック登録自体は失敗を返さない設計ですが、ef_init の実行時にエラーが発生した場合、サーバーの起動プロセスに影響を与える可能性があります。

保守担当者向けメモ

  • 本関数は mod_ext_filter のライフサイクル管理の起点です。設定の初期化順序を変更する場合は、ap_hook_post_config の優先順位（APR_HOOK_MIDDLE）を調整してください。
