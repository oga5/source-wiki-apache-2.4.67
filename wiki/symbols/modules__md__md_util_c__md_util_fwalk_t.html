md_util_fwalk_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_util.c:527

目的

ファイルシステム走査処理において、走査対象のパス、フィルタリング条件、および実行時のコンテキストを保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const char *path;
    apr_array_header_t *patterns;
    int follow_links;
    void *baton;
    md_util_fdo_cb *cb;
} md_util_fwalk_t;

入力と出力

本構造体は関数への引数として渡されるコンテキストオブジェクトです。

  • path: 走査を開始するルートディレクトリのパス。
  • patterns: 走査対象を絞り込むためのパターンリスト。
  • follow_links: シンボリックリンクを追跡するかどうかのフラグ。
  • baton: コールバック関数に渡される任意のユーザーデータ。
  • cb: 各ファイル/ディレクトリに対して実行されるコールバック関数。

前提条件と不変条件

  • cb は md_util_fdo_cb 型の関数ポインタである必要があります。
  • path は有効なファイルシステムパスを指している必要があります。

アルゴリズムまたは振る舞い

この構造体は、match_and_do などのディレクトリ走査関数において、走査の状態を維持し、各要素が見つかるたびに cb で指定されたコールバックを呼び出すために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_util.c 内の match_and_do 関数およびその他の走査関連関数（673, 767 行目付近でインスタンス化を確認）。
  • **呼び出し先**: 構造体内の cb フィールドに格納されたコールバック関数が、走査プロセス中に呼び出されます。

関連データ構造

  • apr_array_header_t: APR の配列構造体。パターンリストの保持に使用。
  • md_util_fdo_cb: 走査時に実行されるコールバック関数の型定義。

失敗モードと境界条件

  • path が無効な場合や、アクセス権限がないディレクトリを指定した場合の挙動は、走査関数（match_and_do 等）の実装に依存します。
  • 構造体内の各ポインタが NULL の場合の挙動については、各走査関数の実装を確認する必要があります。

保守担当者向けメモ

  • 本構造体は md_util.c 内の再帰的なファイル走査ロジックの基盤です。変更を加える際は、match_and_do などの走査関数への影響を考慮してください。
