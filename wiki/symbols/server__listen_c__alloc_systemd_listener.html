alloc_systemd_listener

定義

  • 種別: function
  • 定義箇所: server/listen.c:289

目的

systemd ソケットアクティベーション経由で渡されたファイル記述子（fd）から、Apache HTTP Server のリスナー構造体（ap_listen_rec）を生成および初期化します。

シグネチャまたは形状

apr_status_t alloc_systemd_listener(process_rec *process, int fd, const char *proto, ap_listen_rec **out_rec)

入力と出力

  • **入力**:
  • process: サーバーのプロセス情報（メモリプールを含む）。
  • fd: systemd から継承したソケットのファイル記述子。
  • proto: プロトコル名（例: "http"）。
  • out_rec: 生成されたリスナー構造体を格納するためのポインタ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコード。
  • *out_rec: 成功時に初期化済みの ap_listen_rec 構造体へのポインタが設定されます。

前提条件と不変条件

  • process->pool は有効である必要があります。
  • fd は有効なソケット記述子である必要があります。
  • 成功時には *out_rec が非 NULL になります。

アルゴリズムまたは振る舞い

1. getsockname を呼び出し、指定された fd のソケットアドレス情報を取得します。失敗した場合はエラーをログ出力して終了します。 2. apr_os_sock_info_t 構造体にソケット情報を設定します（SOCK_STREAM および APR_PROTO_TCP を指定）。 3. apr_pcalloc を使用して ap_listen_rec 構造体をメモリプールに割り当てます。 4. apr_os_sock_make を呼び出し、OS のソケットを APR ソケットオブジェクトに変換して rec->sd に格納します。 5. apr_socket_addr_get を呼び出し、バインドされているローカルアドレス情報を取得します。 6. proto を rec->protocol にコピーします。 7. make_sock を呼び出し、リスナーの最終的な設定（ソケットオプションの適用等）を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/listen.c:366
  • server/listen.c:850
  • **呼び出し先**:
  • getsockname (システムコール)
  • apr_pcalloc
  • apr_os_sock_make
  • apr_socket_addr_get
  • apr_pstrdup
  • make_sock

関連データ構造

  • ap_listen_rec: リスナー情報を保持する主要構造体。
  • apr_os_sock_info_t: OS 固有のソケット情報を APR に渡すための構造体。
  • process_rec: プロセス全体のライフサイクルを管理する構造体。

失敗モードと境界条件

  • getsockname が失敗した場合、エラーログを出力し、そのステータスを返します。
  • apr_os_sock_make が失敗した場合、エラーログを出力し、そのステータスを返します。
  • apr_socket_addr_get が失敗した場合、エラーログを出力し、そのステータスを返します。
  • make_sock の実行結果がエラーの場合、そのステータスを返します。

保守担当者向けメモ

  • 本関数は HAVE_SYSTEMD マクロで囲まれた環境でのみ使用されることを想定しています。
  • make_sock の呼び出しが最終ステップであり、ここでソケットオプションの適用が行われます。
  • fd の所有権は呼び出し元に依存するため、クローズ処理のタイミングに注意してください。
