dav_lock_discovery_fixed

定義

  • 種別: struct
  • 定義箇所: modules/dav/lock/locks.c:110

目的

WebDAV ロック情報を DBM ファイルへ永続化する際、メモリ上のロック構造体からバイナリ形式へシリアライズするための固定長データ部分を定義します。

シグネチャまたは形状

typedef struct dav_lock_discovery_fixed
{
    char scope;
    char type;
    int depth;
    time_t timeout;
} dav_lock_discovery_fixed;

入力と出力

本構造体は直接的な関数ではありませんが、ロックの保存（dav_generic_save_lock_record）および読み込み（dav_generic_load_lock_record）処理において、DBM レコードのバイナリバッファの一部として読み書きされます。

前提条件と不変条件

  • 本構造体は、DBM 内のバイナリ列における固定長ヘッダーとして扱われます。
  • dav_size_direct マクロを通じて、本構造体のサイズがシリアライズ時のオフセット計算に利用されています。

アルゴリズムまたは振る舞い

  • ロック情報のシリアライズ時、メモリ上のロック構造体から本構造体のフィールドへ値がコピーされます。
  • DBM からの読み込み時、バイナリデータから本構造体へキャストまたはコピーされ、ロックの有効期限（timeout）やスコープ、深さ（depth）が復元されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/lock/locks.c 内のロック保存・読み込み関数、および modules/dav/fs/lock.c 内のファイルシステムロック処理。
  • **呼び出し先**: 該当なし（データ構造定義のため）。

関連データ構造

  • dav_lock_discovery: 本構造体を含む、直接ロック情報を保持するリスト構造体。
  • dav_lockdb_private: ロックデータベースのコンテキストを保持する構造体。

失敗モードと境界条件

  • **バイナリ互換性**: 本構造体のサイズやフィールド順序を変更すると、既存の DBM ファイルとの互換性が失われます。
  • **パディング**: コンパイラによる構造体パディングが、DBM に書き込まれるバイナリ列のオフセットに影響を与える可能性があります。

保守担当者向けメモ

  • dav_size_direct マクロと本構造体の定義は密結合しています。構造体を変更する場合は、必ず dav_size_direct の計算ロジックも併せて更新してください。
  • modules/dav/fs/lock.c でも同様の定義が参照されているため、変更時はモジュールを跨いだ影響範囲を確認してください。
