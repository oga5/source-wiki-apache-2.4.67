ap_cache_current_age

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_util.c:232

目的

HTTP/1.1 仕様（RFC 2616 13.2.3）に基づき、キャッシュされたレスポンスの現在の経過時間（Age）を計算します。

シグネチャまたは形状

apr_int64_t ap_cache_current_age(cache_info *info,
                                 const apr_time_t age_value,
                                 apr_time_t now)

入力と出力

  • **入力**:
  • info: キャッシュ情報のメタデータ（response_time, date, request_time を含む）。
  • age_value: Age ヘッダーから取得した経過時間（秒）。
  • now: 現在時刻。
  • **出力**:
  • 計算された経過時間（秒単位の apr_int64_t）。

前提条件と不変条件

  • info->response_time、info->date、info->request_time が適切に設定されている必要があります。
  • 計算過程で負の値が発生した場合は、最終的に 0 に補正されます。

アルゴリズムまたは振る舞い

以下の手順で経過時間を算出します。 1. age_value をマイクロ秒に変換します。 2. apparent_age を response_time - date として算出（最小値 0）。 3. corrected_received_age を apparent_age と age_value の大きい方として算出。 4. response_delay を response_time - request_time として算出。 5. corrected_initial_age を corrected_received_age + response_delay として算出。 6. resident_time を now - response_time として算出。 7. current_age を corrected_initial_age + resident_time として算出。 8. 負の値であれば 0 に補正し、秒単位に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/cache_util.c:625 にて呼び出されていることが確認されています。
  • **呼び出し先**: apr_time_from_sec および apr_time_sec（APR ライブラリ関数）を呼び出します。

関連データ構造

  • cache_info: キャッシュのメタデータを保持する構造体。

失敗モードと境界条件

  • current_age が負になる場合、0 として扱われます。
  • apparent_age が負になる場合、0 として扱われます。

保守担当者向けメモ

  • 本関数は RFC 2616 13.2.3 に厳密に従っています。修正を行う際は、HTTP 仕様の該当セクションを参照してください。
  • cache_info 構造体の各フィールドが正しく初期化されていることを前提としています。
