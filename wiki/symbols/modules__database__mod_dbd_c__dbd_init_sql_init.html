dbd_init_sql_init

定義

  • 種別: function
  • 定義箇所: modules/database/mod_dbd.c:465

目的

データベース接続確立直後に、設定された初期化用 SQL クエリ群を順次実行する。

シグネチャまたは形状

static apr_status_t dbd_init_sql_init(apr_pool_t *pool, dbd_cfg_t *cfg, ap_dbd_t *rec)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する APR プール。
  • cfg: データベース設定構造体。実行すべき初期化クエリのリスト（init_queries）を保持。
  • rec: データベース接続ハンドル。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、クエリ実行に失敗した場合は APR_EGENERAL を返す。

前提条件と不変条件

  • cfg->init_queries は初期化済みであること。
  • rec は有効なデータベース接続ハンドルであり、ドライバおよび接続状態が確立されていること。

アルゴリズムまたは振る舞い

1. cfg->init_queries に格納されたクエリリストを反復処理する。 2. 各クエリに対して apr_dbd_query を呼び出し、データベース上で実行する。 3. いずれかのクエリ実行でエラーが発生した場合、直ちにループを中断し APR_EGENERAL を返す。 4. すべてのクエリが正常に完了した場合、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/database/mod_dbd.c:980 において、dbd モジュールの post_connect フックとして登録されている。
  • **呼び出し先**: apr_dbd_query (APR ライブラリ関数)。

関連データ構造

  • dbd_cfg_t: 初期化クエリのリストを保持する設定構造体。
  • ap_dbd_t: データベース接続ハンドル。

失敗モードと境界条件

  • 初期化クエリが空の場合、ループは実行されず APR_SUCCESS を返す。
  • apr_dbd_query がエラーを返した場合、その後のクエリは実行されず、処理は即座に終了する。

保守担当者向けメモ

  • 本関数は接続確立後のフックとして動作するため、ここで実行される SQL は接続のセッション設定（文字コード設定や一時テーブル作成など）を想定している。
  • 実行順序は cfg->init_queries に格納された順序に依存する。
