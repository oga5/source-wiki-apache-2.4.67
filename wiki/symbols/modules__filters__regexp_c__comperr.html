comperr

定義

  • 種別: function
  • 定義箇所: modules/filters/regexp.c:63

目的

正規表現のコンパイル中に発生したエラーを報告し、関連するコマンド情報とエラーメッセージを記録する。

シグネチャまたは形状

static void comperr(sed_commands_t *commands, char *msg)

入力と出力

  • **入力**:
  • commands: エラー発生時のコンテキストを保持する sed_commands_t 構造体へのポインタ。
  • msg: 報告するエラーメッセージのフォーマット文字列。
  • **出力**: なし（void）。

前提条件と不変条件

  • commands 引数は有効な sed_commands_t 構造体を指している必要がある。
  • commands->linebuf には、エラー発生時の対象行データが保持されていることが期待される。

アルゴリズムまたは振る舞い

1. 受け取った commands と msg、および commands->linebuf を引数として command_errf を呼び出す。 2. command_errf を介して、エラーメッセージと現在の行バッファの内容を統合的に処理する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/regexp.c 内の正規表現コンパイル処理（sed_compile 等の内部処理）。具体的には、範囲指定の解析エラー、括弧の不整合、無効なデリミタ、数値範囲外などの構文エラー発生時に呼び出される。
  • **呼び出し先**: command_errf（外部関数）。

関連データ構造

  • sed_commands_t: コンパイル中のコマンド状態を管理する構造体。

失敗モードと境界条件

  • 本関数自体はエラー報告を行うためのユーティリティであり、呼び出し元で構文エラーが検出された場合に実行される。
  • commands が NULL の場合、command_errf の実装に依存するが、セグメンテーション違反を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は regexp.c 内の複数のエラーチェック箇所から共通して呼び出されるラッパーである。
  • エラーメッセージの形式を変更する場合は、command_errf の仕様と照らし合わせること。
