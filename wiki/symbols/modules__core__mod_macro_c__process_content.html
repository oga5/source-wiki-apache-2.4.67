process_content

定義

  • 種別: function
  • 定義箇所: modules/core/mod_macro.c:450

目的

マクロ定義の内容を走査し、引数の置換を行って展開後の設定行を生成する。

シグネチャまたは形状

static const char *process_content(apr_pool_t * pool,
                                   const ap_macro_t * macro,
                                   const apr_array_header_t * replacements,
                                   apr_array_header_t * used,
                                   apr_array_header_t ** result)

入力と出力

  • **入力**:
  • pool: メモリ割り当て用プール。
  • macro: 処理対象のマクロ定義（ap_macro_t）。
  • replacements: 置換対象となる引数の配列。
  • used: 使用された引数を追跡するための配列。
  • result: 展開後の行を格納する配列へのポインタ（NULL指定可能）。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージを含む文字列を返す。

前提条件と不変条件

  • macro は有効な ap_macro_t 構造体を指していること。
  • MAX_STRING_LEN を超える行は切り詰められる可能性がある。

アルゴリズムまたは振る舞い

1. result が指定されている場合、macro->contents の要素数に基づいて result 配列を初期化する。 2. マクロ本体の各行に対してループ処理を行う。 3. 各行を一時バッファにコピーし、substitute_macro_args を呼び出して引数置換を実行する。 4. 置換中にエラーが発生した場合は、行番号とマクロ名を含むエラーメッセージを即座に返す。 5. result が指定されている場合、置換後の行を result 配列に追加する。 6. 全ての行の処理が完了したら NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/core/mod_macro.c:511
  • modules/core/mod_macro.c:865
  • **呼び出し先**:
  • substitute_macro_args (内部で呼び出し)
  • apr_array_make / apr_array_push / apr_pstrdup / apr_psprintf / apr_cpystrn (APR API)

関連データ構造

  • ap_macro_t: マクロの定義（名前、本体、場所など）を保持。
  • apr_array_header_t: マクロの行データや引数リストを保持する可変長配列。

失敗モードと境界条件

  • substitute_macro_args がエラーを返した場合、そのエラーを伝播させる。
  • MAX_STRING_LEN を超える行は apr_cpystrn により切り詰められる。

保守担当者向けメモ

  • 本関数は設定解析フェーズで実行されるため、メモリ管理には cmd->temp_pool 等の適切なライフサイクルを持つプールを使用すること。
  • result 引数が NULL の場合、展開後の内容を保持せず、置換の検証のみを行う動作となる。
