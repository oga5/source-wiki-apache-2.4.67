xml2ctx

定義

  • **種別**: struct
  • **定義箇所**: modules/filters/mod_xml2enc.c:77

目的

Apache HTTP Server の出力フィルタ mod_xml2enc において、リクエストごとのフィルタ実行状態（文字コード変換器、バッファ、フラグ等）を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    xmlCharEncoding xml2enc;        /* libxml2 のエンコーディング識別子 */
    char* buf;                      /* 処理用バッファ */
    apr_size_t bytes;               /* バッファ内のデータサイズ */
    apr_xlate_t* convset;           /* 文字コード変換ハンドル */
    unsigned int flags;             /* フィルタの状態フラグ */
    apr_off_t bblen;                /* バケットブリゲードの長さ */
    apr_bucket_brigade* bbnext;     /* 次のフィルタへ渡すバケットブリゲード */
    apr_bucket_brigade* bbsave;     /* 処理待ちのバケットブリゲード */
    const char* encoding;           /* 検出されたエンコーディング名 */
} xml2ctx;

入力と出力

本構造体は関数引数として渡され、フィルタの実行状態を維持します。

  • **入力**: apr_pcalloc 等によりリクエストプールからメモリが割り当てられ、sniff_encoding や fix_skipto などの関数で状態が更新されます。
  • **出力**: 変換後のデータが bbnext を通じて後続フィルタへ渡されます。

前提条件と不変条件

  • リクエストのライフサイクルに紐付いており、request_rec のプールを使用して確保されます。
  • libxml2 の検出ロジックおよび apr_xlate の変換機能に依存しています。

アルゴリズムまたは振る舞い

  • **バッファリング**: xml2enc_ffunc 内で、エンコーディング特定に必要なデータ量（BUF_MIN）が溜まるまでデータを保持します。
  • **変換**: apr_xlate_conv_buffer を用いて、バッファ内のデータを UTF-8 へ変換します。
  • **エラー処理**: 不正なバイト列に対しては APR_EINVAL を返し、該当バイトをスキップして処理を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • xml2enc_ffunc (フィルタのメインループ)
  • fix_skipto (補助関数)
  • sniff_encoding (エンコーディング特定関数)
  • xml2enc_filter_init (初期化関数)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • request_rec: リクエスト情報を保持する Apache の構造体。
  • apr_bucket_brigade: ストリーム処理に使用されるバケットの連結リスト。
  • apr_xlate_t: apr_xlate による文字コード変換ハンドル。
  • xmlCharEncoding: libxml2 で定義されるエンコーディング型。

失敗モードと境界条件

  • **エンコーディング未検出**: 設定されたデフォルト値にフォールバックします。
  • **不完全なデータ**: APR_EINCOMPLETE が発生した場合、バッファを保持して次回の ffunc 呼び出しを待ちます。
  • **変換失敗**: apr_xlate での変換が失敗した場合、デフォルト値へのフォールバックを試みます。

保守担当者向けメモ

  • メモリ管理は request_rec のプールに依存しており、リクエスト終了時に自動解放されます。
  • libxml2 の内部テーブルに依存しているため、サポート外のエンコーディングについては apr_xlate の挙動に左右されます。
  • ap_assert の使用箇所は、将来的にエラーハンドリングへの置き換えが推奨されています。
