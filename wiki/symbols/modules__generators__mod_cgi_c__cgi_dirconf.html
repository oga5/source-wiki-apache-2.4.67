cgi_dirconf

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_cgi.c:91

目的

ディレクトリ単位の CGI 設定を保持するための構造体です。主にスクリプト実行時のタイムアウト設定を管理します。

シグネチャまたは形状

typedef struct {
    apr_interval_time_t timeout;
} cgi_dirconf;

入力と出力

  • **入力**: apr_interval_time_t 型の timeout メンバ。
  • **出力**: なし（設定保持用構造体）。

前提条件と不変条件

  • timeout は apr_interval_time_t 単位（マイクロ秒）で保持されます。
  • リクエスト処理中に ap_get_module_config(r->per_dir_config, &cgi_module) を通じて取得され、CGI プロセスの実行制御に使用されます。

アルゴリズムまたは振る舞い

  • create_cgi_dirconf 関数によってメモリが割り当てられ、初期化されます。
  • set_script_timeout 指令により、設定ファイルの値がこの構造体の timeout メンバに格納されます。
  • CGI 実行時、この構造体から読み取られたタイムアウト値がプロセス制御に適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:122 (apr_pcalloc による初期化)
  • modules/generators/mod_cgi.c:166 (設定マージ処理)
  • modules/generators/mod_cgi.c:436, 526 (リクエスト処理時の設定取得)
  • **呼び出し先**: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • cgi_module: 本構造体を使用する Apache モジュール定義。
  • request_rec: リクエストごとの設定取得時に参照されるコンテキスト。

失敗モードと境界条件

  • 設定値が未定義の場合、デフォルト値が適用されます（create_cgi_dirconf の実装に依存）。
  • apr_pcalloc によるメモリ確保が失敗した場合、サーバーはリクエスト処理を継続できません。

保守担当者向けメモ

  • 本構造体は mod_cgi のディレクトリ設定を保持する唯一の場所です。新しいディレクトリ単位の設定項目を追加する場合は、この構造体にメンバを追加し、merge_cgi_config でマージロジックを更新してください。
