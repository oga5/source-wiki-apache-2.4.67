proxy_html_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_proxy_html.c:886

目的

Apache HTTP Server の出力フィルタとして、HTML コンテンツ内の URL や属性を動的に書き換えるためのメイン処理を実行する。

シグネチャまたは形状

apr_status_t proxy_html_filter(ap_filter_t *f, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • f: フィルタコンテキストを含む ap_filter_t 構造体。
  • bb: 処理対象のデータを含む apr_bucket_brigade。
  • **出力**:
  • 戻り値: apr_status_t（成功時は APR_SUCCESS）。
  • 副作用: 書き換えられた HTML データが後続のフィルタへ渡される。

前提条件と不変条件

  • check_filter_init により、実行時コンテキスト (saxctxt) が初期化されている必要がある。
  • libxml2 のパーサコンテキストは、リクエスト終了時に htmlFreeParserCtxt で解放されるようプールに登録される。

アルゴリズムまたは振る舞い

1. **初期化確認**: check_filter_init を呼び出し、コンテキストがなければデータをそのまま通過させる。 2. **バケット走査**: ブリゲード内の各バケットを順次処理する。

  • **メタデータ**: EOS (End of Stream) 受信時にパーサを終了し、残りのバッファをフラッシュする。
  • **データ**:
  • パーサ未初期化時: 最初の数バイトをバッファリングして文字コードを判定（mod_xml2enc 連携）。その後、htmlCreatePushParserCtxt でパーサを生成し、解析を開始する。
  • パーサ初期化済み: consume_buffer を通じてデータをパーサへ送り、置換処理を実行する。

    • 3. **終了処理**: 処理完了後、ブリゲードをクリーンアップする。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフィルタチェーン（ap_register_output_filter_protocol により登録）。
  • **呼び出し先**:
  • check_filter_init: コンテキストの取得。
  • htmlCreatePushParserCtxt / htmlFreeParserCtxt: libxml2 による解析制御。
  • consume_buffer: パーサへのデータ供給。
  • metafix: <meta> タグの処理。
  • ap_pass_brigade: 後続フィルタへのデータ転送。

関連データ構造

  • saxctxt: フィルタの実行状態を保持する構造体。
  • apr_bucket_brigade: 処理対象のデータストリーム。
  • htmlParserCtxt: libxml2 のパーサコンテキスト。

失敗モードと境界条件

  • **初期データ不足**: 4バイト未満のデータは解析せず、バッファに蓄積する。
  • **パーサ生成失敗**: htmlCreatePushParserCtxt が失敗した場合、フィルタを解除しデータをそのまま通過させる。
  • **読み込みエラー**: apr_bucket_read が失敗した場合、エラーログを出力する。

保守担当者向けメモ

  • libxml2 のパーサはストリーム処理を行うため、メモリ消費量は ProxyHTMLBufSize に依存する。
  • USE_OLD_LIBXML2 マクロの有無により、パーサオプションの適用方法が異なる。
  • apr_pool_cleanup_register を使用して、パーサのメモリリークを防止している。
