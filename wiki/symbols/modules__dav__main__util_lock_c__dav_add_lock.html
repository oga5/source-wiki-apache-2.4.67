dav_add_lock

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:296

目的

指定されたリソースに対してロックを追加します。対象がコレクションである場合、深さ（Depth）に応じて子リソースへ再帰的に間接ロックを適用します。

シグネチャまたは形状

dav_error *dav_add_lock(request_rec *r,
                        const dav_resource *resource,
                        dav_lockdb *lockdb, dav_lock *lock,
                        dav_response **response)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • resource: ロック対象のリソース。
  • lockdb: ロックデータベースのハンドル。
  • lock: 追加するロック情報。
  • response: マルチステータスレスポンスを格納するためのポインタ。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は dav_error 構造体へのポインタを返します。
  • response 引数には、再帰的なロック適用時にエラーが発生した場合、dav_response リストが格納されます。

前提条件と不変条件

  • リソースがコレクションでない場合、RFC 2518 9.2 に従い depth は強制的に 0 として扱われます。
  • 常にまず対象リソースに対して直接ロック（direct lock）が lockdb に追加されます。

アルゴリズムまたは振る舞い

1. リソースがコレクションでない場合、depth を 0 に設定します。 2. lockdb->hooks->append_locks を呼び出し、対象リソースに直接ロックを適用します。 3. depth > 0 の場合、resource->hooks->walk を使用してリソースツリーを走査します。 4. 走査には dav_lock_walker を使用し、各子孫リソースに対して間接ロックを適用します。 5. 走査中にエラーが発生した場合は即座にエラーを返します。 6. 走査の結果、部分的なエラーが発生した場合は HTTP_MULTI_STATUS を含む dav_error を返し、response に詳細を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c 内のロック処理ロジックから呼び出されます。
  • **呼び出し先**:
  • lockdb->hooks->append_locks: 直接ロックの追加。
  • resource->hooks->walk: 子リソースの走査。
  • dav_lock_walker: 走査時の各ノードに対するコールバック。
  • dav_new_error: エラー生成。

関連データ構造

  • dav_resource: ロック対象のリソース表現。
  • dav_lock: ロックの属性（深さ等）を保持。
  • dav_lockdb: ロックデータベースへのフックを保持。
  • dav_walker_ctx: 走査時のコンテキスト。

失敗モードと境界条件

  • append_locks が失敗した場合、そのエラーがそのまま返されます。
  • walk 処理中に致命的なエラーが発生した場合、そのエラーが返されます。
  • walk 処理中に一部のリソースでロック適用に失敗した場合、HTTP_MULTI_STATUS エラーが返され、response に詳細が設定されます。

保守担当者向けメモ

  • 本関数はロック・ヌル（lock-null）リソースの処理も包含しています。
  • depth > 0 の再帰処理におけるマルチステータスレスポンスの生成は、エラーハンドリングの改善余地がある箇所として認識されています。
