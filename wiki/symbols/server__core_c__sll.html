sll

定義

  • 種別: struct
  • 定義箇所: server/core.c:1379

目的

文字列の連結や展開処理において、複数の文字列断片を効率的に保持するための単方向連結リスト構造体です。

シグネチャまたは形状

struct sll {
    struct sll *next;
    const char *string;
    apr_size_t len;
};

入力と出力

  • 入力: apr_palloc 等によりメモリプールから割り当てられたインスタンス。
  • 出力: next ポインタによる後続ノードへの参照、および string とその長さ len。

前提条件と不変条件

  • next はリストの終端で NULL を指す必要があります。
  • len は string が指す文字列の有効な長さを保持している必要があります。

アルゴリズムまたは振る舞い

  • 文字列の断片をリスト構造として連結し、最終的に一つのバッファへ統合するためのノードとして機能します。
  • SMALL_EXPANSION 定数に基づき、スタック上に配列として確保される場合と、ヒープ（メモリプール）上に動的に確保される場合があります。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_include.c:62 にて文字列展開用途での利用が言及されています。
  • 呼び出し先: apr_palloc（メモリ割り当て時）が利用されます。

関連データ構造

  • apr_pool_t: メモリ管理に使用されます。
  • sresult: SMALL_EXPANSION サイズの sll 配列。

失敗モードと境界条件

  • メモリ不足時: apr_palloc が失敗した場合、呼び出し元で例外的な挙動が発生する可能性があります。
  • 循環参照: リスト構築時に next ポインタを誤って設定すると無限ループを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は server/core.c 内で定義されていますが、mod_include 等の外部モジュールからも参照されています。変更時は依存モジュールへの影響を考慮してください。
  • SMALL_EXPANSION の値が適切かどうかは、パフォーマンスとメモリ消費のトレードオフとなります。
