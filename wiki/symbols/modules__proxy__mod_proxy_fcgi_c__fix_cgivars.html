fix_cgivars

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:196

目的

ProxyFCGISetEnvIf ディレクティブで設定されたルールに基づき、リクエストの環境変数（subprocess_env）を動的に修正・削除する。

シグネチャまたは形状

static apr_status_t fix_cgivars(request_rec *r, fcgi_dirconf_t *dconf)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • dconf: ProxyFCGISetEnvIf 等の設定を保持するディレクトリ設定構造体。
  • **出力**:
  • 成功時は APR_SUCCESS を返す。
  • 条件評価エラー時は APR_EGENERAL を返す。

前提条件と不変条件

  • dconf->env_fixups に有効な修正ルールが格納されている必要がある。
  • r->subprocess_env が初期化済みであること。

アルゴリズムまたは振る舞い

1. dconf->env_fixups に登録された各ルールを順次走査する。 2. ap_expr_exec_re を使用して、各ルールの条件式を評価する。 3. 条件が真（rc > 0）の場合:

  • 環境変数名が ! で始まる場合、apr_table_unset で該当変数を削除する。
  • それ以外の場合、ap_expr_str_exec_re で値を評価し、apr_table_setn で環境変数を上書きまたは新規設定する。

    • 4. 条件が偽（rc == 0）の場合はスキップする。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_fcgi.c:443 (リクエスト処理フロー内)
  • **呼び出し先**:
  • ap_expr_exec_re (条件式評価)
  • ap_expr_str_exec_re (値の評価)
  • apr_table_unset / apr_table_setn (環境変数操作)
  • ap_log_rerror (ログ出力)

関連データ構造

  • fcgi_dirconf_t: ディレクトリ単位の設定構造体。env_fixups を保持。
  • sei_entry: 環境変数の修正ルールを保持する構造体。
  • request_rec: Apache のリクエスト構造体。subprocess_env を介して環境変数を管理。

失敗モードと境界条件

  • 条件式の評価に失敗した場合、APR_EGENERAL を返し処理を中断する。
  • 式の評価中にエラーが発生した場合、ログを出力し、そのルールをスキップして継続する。

保守担当者向けメモ

  • 本関数は ProxyFCGISetEnvIf の実装の中核です。
  • AP_MAX_REG_MATCH を使用して正規表現のキャプチャグループを保持しているため、条件式と置換式の間で正規表現のバックリファレンスが共有されます。
