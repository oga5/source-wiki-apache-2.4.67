set_cookie_form_password

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_form.c:221

目的

Apache HTTP Server の設定ディレクティブ AuthFormPassword を処理し、フォーム認証で使用するパスワードフィールド名を指定された値に設定します。

シグネチャまたは形状

static const char *set_cookie_form_password(cmd_parms *cmd, void *config, const char *password)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: 現在のディレクトリ設定構造体（auth_form_config_rec）。
  • password: 設定ファイルから読み取られたパスワードフィールド名（文字列）。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • config 引数は auth_form_config_rec 型としてキャスト可能である必要があります。
  • conf->password_set は、設定が適用されたことを示すフラグとして 1 に更新されます。

アルゴリズムまたは振る舞い

1. config を auth_form_config_rec 構造体にキャストします。 2. 構造体の password メンバに引数の password を代入します。 3. password_set メンバを 1 に設定します。 4. check_string 関数を呼び出し、設定値の妥当性を検証した結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_form.c:358 において、AP_INIT_TAKE1 マクロを通じて AuthFormPassword ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: check_string（引数で渡された文字列の検証用）。

関連データ構造

  • auth_form_config_rec: フォーム認証の設定を保持する構造体。

失敗モードと境界条件

  • check_string がエラーを返した場合、そのエラーメッセージがそのまま呼び出し元に伝播されます。

保守担当者向けメモ

  • 本関数は AuthFormPassword ディレクティブ専用のセッターです。設定値の検証ロジックを変更する場合は check_string を参照してください。
