wakeup_listener

定義

  • 種別: function
  • 定義箇所: server/mpm/worker/worker.c:273

目的

リスナー（listener_thread）がブロッキング状態にある場合に、それを解除して終了処理へ移行させるためのシグナル送信を行う。

シグネチャまたは形状

static void wakeup_listener(void)

入力と出力

  • 入力: なし（グローバルな状態変数および静的変数に依存）
  • 出力: なし（副作用としてリスナーの待機状態を解除）

前提条件と不変条件

  • listener_may_exit フラグが 1 に設定される。
  • worker_pollset が有効な場合、apr_pollset_wakeup が呼び出される。
  • worker_queue_info に対して ap_queue_info_term が呼び出され、ワーカー待ち状態が解除される。

アルゴリズムまたは振る舞い

1. listener_may_exit を 1 に設定し、終了を指示する。 2. worker_pollset が存在し、かつ listener_is_wakeable が真であれば、apr_pollset_wakeup を実行して poll() を解除する。 3. ap_queue_info_term を呼び出し、ワーカーを待機しているリスナーを解放する。 4. listener_os_thread が初期化済みであれば、プラットフォームに応じて以下のいずれかを実行し、スレッドへシグナルを送信する。

  • HAVE_PTHREAD_KILL が定義されている場合: pthread_kill を使用。
  • それ以外の場合: kill(ap_my_pid, LISTENER_SIGNAL) を使用。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • server/mpm/worker/worker.c:322
  • server/mpm/worker/worker.c:1061
  • 呼び出し元（可能性のある参照）:
  • server/mpm/event/event.c:636
  • server/mpm/event/event.c:2577
  • 呼び出し先:
  • apr_pollset_wakeup
  • ap_queue_info_term
  • pthread_kill または kill

関連データ構造

  • worker_pollset: リスナーが監視するソケットセット。
  • worker_queue_info: リスナーとワーカー間のキュー管理構造。
  • listener_os_thread: リスナーのスレッド識別子。

失敗モードと境界条件

  • リスナー生成直後で listener_os_thread が未設定の場合、シグナル送信処理をスキップして早期リターンする。この際、一部の終了処理が完全に行われない可能性がある旨がソースコード内に注釈されている。

保守担当者向けメモ

  • 本関数は worker MPM の終了シーケンスにおいて、リスナーを強制的に待機状態から脱出させるために不可欠である。
  • pthread_kill を使用するプラットフォームと kill を使用するプラットフォームで挙動が分かれるため、シグナルハンドリングの変更時には注意が必要。
