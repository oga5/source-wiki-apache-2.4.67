ap_create_sb_handle

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:432

目的

ワーカープロセスまたはスレッドを識別するためのスコアボードハンドル（ap_sb_handle_t）を新規作成し、初期化する。

シグネチャまたは形状

void ap_create_sb_handle(ap_sb_handle_t **new_sbh, apr_pool_t *p, int child_num, int thread_num)

入力と出力

  • new_sbh: 出力パラメータ。確保・初期化された ap_sb_handle_t 構造体へのポインタが格納される。
  • p: メモリ確保に使用する apr_pool_t。
  • child_num: スコアボード上の子プロセスインデックス。
  • thread_num: スコアボード上のスレッドインデックス。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • new_sbh は書き込み可能なポインタのアドレスであること。

アルゴリズムまたは振る舞い

1. 指定されたメモリプール p から ap_sb_handle_t 構造体サイズ分のメモリを apr_palloc で確保する。 2. 確保した領域を new_sbh に代入する。 3. ap_update_sb_handle を呼び出し、指定された child_num および thread_num を用いてハンドル内部の値を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/mpmt_os2/mpmt_os2_child.c
  • server/mpm/netware/mpm_netware.c
  • server/mpm/prefork/prefork.c
  • server/mpm/winnt/child.c
  • server/mpm/worker/worker.c
  • **呼び出し先**:
  • apr_palloc (APR)
  • ap_update_sb_handle (server/scoreboard.c)

関連データ構造

  • ap_sb_handle_t: スコアボード内の特定のワーカーを指し示すためのハンドル構造体。

失敗モードと境界条件

  • メモリ不足時: apr_palloc の挙動に従い、メモリ確保に失敗した場合はプロセスが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は MPM がワーカーのライフサイクルを開始する際に呼び出される。
  • ap_update_sb_handle への依存が強いため、ハンドル構造体の内部定義を変更する場合は両方の関数を同期させる必要がある。
