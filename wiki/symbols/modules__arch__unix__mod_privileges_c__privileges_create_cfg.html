privileges_create_cfg

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_privileges.c:79

目的

Apache モジュール mod_privileges におけるサーバー/仮想ホスト単位の設定構造体（priv_cfg）を生成し、初期化する。

シグネチャまたは形状

static void *privileges_create_cfg(apr_pool_t *pool, server_rec *s)

入力と出力

  • **入力**:
  • pool: 設定メモリを割り当てるための APR プール。
  • s: 設定対象のサーバーレコード。
  • **出力**:
  • 初期化済みの priv_cfg 構造体へのポインタ。

前提条件と不変条件

  • pool は有効なメモリプールであること。
  • priv_cfg 構造体内の priv および child_priv は、priv_str_to_set により "basic" 特権セットで初期化される。
  • uid および gid は初期値 0（未設定）として扱われる。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して priv_cfg 構造体用のメモリを確保する。 2. priv_str_to_set を呼び出し、自身用および子プロセス用の特権セットを "basic" で初期化する。 3. セキュリティ強化のため、priv_delset を用いて PRIV_FILE_LINK_ANY、PRIV_PROC_INFO、PRIV_PROC_SESSION をデフォルトの特権セットから削除する。 4. uid、gid を 0 に、mode を PRIV_UNSET に設定する。 5. apr_pool_cleanup_register を呼び出し、プール破棄時に priv_cfg_cleanup が実行されるよう登録する。 6. グローバル変数 priv_default が未設定の場合、現在の cfg->priv をデフォルトとして代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/unix/mod_privileges.c:584（モジュール定義構造体内のフックとして参照）。
  • **呼び出し先**: apr_palloc、priv_str_to_set、priv_delset、apr_pool_cleanup_register、priv_cfg_cleanup。

関連データ構造

  • priv_cfg: 本関数で生成される設定構造体。
  • priv_set_t: Solaris 特権セットを保持する型。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • priv_str_to_set が失敗した場合の挙動については、ソースコード内に明示的なエラーハンドリングの記述はない。
  • CR_CHECK マクロにより、特権削除操作が失敗した場合はログ出力とともにエラーが報告される。

保守担当者向けメモ

  • 本関数は mod_privileges の設定初期化フェーズで呼び出される。
  • PRIV_SECURE モードを前提としたデフォルト設定となっているため、特権セットの変更を行う際は、セキュリティへの影響を考慮すること。
  • priv_default への代入はトップレベルの設定を保持するためのものであり、複数仮想ホスト環境での挙動に注意が必要。
