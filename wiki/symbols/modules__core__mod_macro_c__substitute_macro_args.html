substitute_macro_args

定義

  • 種別: function
  • 定義箇所: modules/core/mod_macro.c:410

目的

マクロ定義内の引数プレースホルダーを、指定された置換値で置き換える。

シグネチャまたは形状

static const char *substitute_macro_args(
    char *buf,
    int bufsize,
    const ap_macro_t * macro,
    const apr_array_header_t * replacements,
    apr_array_header_t * used)

入力と出力

  • **入力**:
  • buf: 置換対象の文字列バッファ。
  • bufsize: バッファのサイズ。
  • macro: マクロ定義情報（引数リストを含む）。
  • replacements: 置換後の値の配列。
  • used: （オプション）各引数が使用されたかを記録する配列。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ（const char *）を返す。

前提条件と不変条件

  • used が指定されている場合、その要素数は replacements の要素数以上である必要がある（ap_assert による検証）。

アルゴリズムまたは振る舞い

1. next_substitution を呼び出し、バッファ内の次の置換対象箇所を特定する。 2. substitute 関数を呼び出し、引数（$, %, @ プレフィックス付き）を replacements の値で置換する。 3. 引数が @ で始まる場合は、エスケープ処理を伴う置換を行う。 4. 置換が成功するたびに、ポインタ ptr を置換後の文字列長分進める。 5. used が提供されている場合、該当する引数の使用フラグを立てる。 6. 全ての置換が完了するまでループを継続し、最後に NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/core/mod_macro.c:469（process_content 内で呼び出される）
  • **呼び出し先**:
  • next_substitution
  • substitute
  • ap_assert

関連データ構造

  • ap_macro_t: マクロの定義（引数名リストなど）を保持。
  • apr_array_header_t: 置換値の配列および使用フラグの管理に使用。

失敗モードと境界条件

  • substitute 関数がエラーを返した場合、そのエラーメッセージをそのまま呼び出し元へ伝播させる。
  • バッファサイズを超える置換が発生した場合の挙動は、内部で呼び出される substitute 関数の実装に依存する。

保守担当者向けメモ

  • 本関数は設定ファイルの解析フェーズで実行される。
  • used 配列の管理は、マクロ引数の未使用チェックを行うために利用されている。
  • 引数のプレフィックス（$, %, @）による置換ロジックの差異に注意すること。
