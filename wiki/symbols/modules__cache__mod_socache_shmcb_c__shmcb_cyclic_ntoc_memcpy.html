shmcb_cyclic_ntoc_memcpy

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_shmcb.c:200

目的

循環バッファ（Cyclic Buffer）の指定オフセット位置から、データをコピーする。バッファの終端に達した場合は、先頭へ折り返してコピーを行う。

シグネチャまたは形状

static void shmcb_cyclic_ntoc_memcpy(unsigned int buf_size, unsigned char *data,
                                     unsigned int dest_offset, const unsigned char *src,
                                     unsigned int src_len)

入力と出力

  • buf_size: 循環バッファの総サイズ。
  • data: コピー先の循環バッファの先頭ポインタ。
  • dest_offset: コピーを開始するバッファ内のオフセット位置。
  • src: コピー元のデータソース。
  • src_len: コピーするデータの長さ。
  • 出力: なし（data が指すメモリ領域が更新される）。

前提条件と不変条件

  • dest_offset は buf_size 未満であること。
  • dest_offset + src_len が buf_size を超える場合、循環バッファの末尾から先頭へ折り返してコピーされる。

アルゴリズムまたは振る舞い

1. dest_offset + src_len が buf_size 未満の場合：

  • memcpy を使用して、data + dest_offset から src_len 分のデータを一度にコピーする。

    • 2. dest_offset + src_len が buf_size 以上の場合（折り返し発生）：

  • 第1段階として、data + dest_offset からバッファ末尾まで（buf_size - dest_offset 分）をコピーする。
  • 第2段階として、バッファの先頭（data）から残りのデータ（src_len + dest_offset - buf_size 分）をコピーする。

呼び出し元と呼び出し先

  • 呼び出し先:

関連データ構造

  • SHMCBSubcache: 循環バッファの管理構造体。

失敗モードと境界条件

  • dest_offset や src_len が極端に大きい場合、内部の算術演算でオーバーフローが発生する可能性がある（ソースコード内のコメントによる指摘）。
  • 呼び出し側で dest_offset が buf_size を超えて渡された場合の動作は定義されていない。

保守担当者向けメモ

  • 本関数は shmcb_cyclic_cton_memcpy と対になる関数である。
  • 共有メモリの循環バッファ操作において、境界を跨ぐコピーを抽象化するために使用される。
