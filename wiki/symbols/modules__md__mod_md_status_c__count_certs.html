count_certs

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:413

目的

JSON オブジェクト内の証明書エントリをカウントするためのイテレータコールバック関数です。

シグネチャまたは形状

static int count_certs(void *baton, const char *key, md_json_t *json)

入力と出力

  • **入力**:
  • baton: カウント値を保持する整数型ポインタ (int *)。
  • key: 現在走査中の JSON キー。
  • json: 現在の JSON ノード（本関数内では未使用）。
  • **出力**:
  • 常に 1 を返し、イテレーションの継続を指示します。
  • baton が指すメモリ領域の値を更新します。

前提条件と不変条件

  • baton は有効な int 型変数へのポインタである必要があります。
  • key が MD_KEY_VALID と一致しない場合のみ、baton の値がインクリメントされます。

アルゴリズムまたは振る舞い

1. baton を int *pcount にキャストします。 2. json 引数は使用せず破棄します。 3. key が MD_KEY_VALID（有効期限情報）と等しくない場合、*pcount を 1 増加させます。 4. 常に 1 を返してイテレーションを続行します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md_status.c:463 において、md_json_iterkey の引数として呼び出されます。
  • **呼び出し先**: strcmp（標準ライブラリ関数）を呼び出します。

関連データ構造

  • md_json_t: JSON データを保持する構造体。
  • MD_KEY_VALID: 証明書の有効期限を示すキー定数。

失敗モードと境界条件

  • baton が NULL の場合、デリファレンスによりセグメンテーションフォールトが発生する可能性があります。
  • key が NULL の場合、strcmp の挙動に依存します。

保守担当者向けメモ

  • 本関数は md_json_iterkey を介して使用されることを前提としています。
  • MD_KEY_VALID 以外のキーをすべて証明書としてカウントするロジックであるため、JSON 構造の変更時にはカウント対象が意図通りか確認してください。
