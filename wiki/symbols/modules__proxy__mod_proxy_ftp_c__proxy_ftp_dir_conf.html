proxy_ftp_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:36

目的

FTP プロキシリクエストにおけるディレクトリ単位の設定情報を保持する構造体です。ProxyFtpListOnWildcard や文字セット設定など、FTP サーバーとの通信およびディレクトリ一覧表示の挙動を制御します。

シグネチャまたは形状

typedef struct {
    int ftp_list_on_wildcard;
    int ftp_list_on_wildcard_set;
    int ftp_escape_wildcards;
    int ftp_escape_wildcards_set;
    const char *ftp_directory_charset;
} proxy_ftp_dir_conf;

入力と出力

  • **入力**: 設定ファイル（httpd.conf 等）で指定されたディレクティブ値。
  • **出力**: モジュール内の各処理関数（ftp_escape_globbingchars や proxy_ftp_handler 等）が参照する設定状態。

前提条件と不変条件

  • 各 _set 接尾辞を持つメンバ（ftp_list_on_wildcard_set 等）は、対応する設定値が明示的に構成されたかどうかを示すフラグとして機能します。
  • ftp_directory_charset は、FTP サーバーから取得したディレクトリリストの文字エンコーディングを指定します。

アルゴリズムまたは振る舞い

  • 本構造体は apr_pcalloc を用いてメモリ確保され、リクエストのライフサイクル（r->per_dir_config）に関連付けられます。
  • proxy_ftp_handler 等の処理関数は、ap_get_module_config を介して本構造体を取得し、FTP コマンドの発行やディレクトリリストの変換ロジックを決定します。

呼び出し元と呼び出し先

  • **呼び出し元（参照・生成）**:
  • create_proxy_ftp_dir_config（生成）
  • merge_proxy_ftp_dir_config（マージ）
  • set_ftp_list_on_wildcard, set_ftp_escape_wildcards, set_ftp_directory_charset（設定）
  • ftp_escape_globbingchars（参照）
  • proxy_ftp_handler（参照）
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • proxy_dir_ctx_t: ディレクトリ一覧表示フィルタの実行状態を保持する構造体。
  • request_rec: Apache のリクエスト情報。per_dir_config を通じて本構造体へアクセスします。

失敗モードと境界条件

  • 設定値が未定義の場合、_set フラグが 0 であることでデフォルトの挙動が選択されます。
  • メモリ確保失敗時は apr_pcalloc の挙動に従い、Apache のエラーハンドリングへ委ねられます。

保守担当者向けメモ

  • 本構造体は mod_proxy_ftp.c 内のディレクトリ設定の基盤です。新しい FTP 関連の設定ディレクティブを追加する場合は、本構造体にメンバと対応する _set フラグを追加してください。
  • ftp_directory_charset の取り扱いには注意が必要です。サーバー側が返すエンコーディングとクライアント側の期待値が一致しない場合、ディレクトリリストの表示が文字化けする可能性があります。
