action_t

定義

  • 種別: enum (列挙型)
  • 定義箇所: server/mpm_unix.c:196 (構造体 action_table 内のメンバとして定義)

目的

子プロセスの終了処理（reap）において、親プロセスが対象プロセスに対して実行すべきアクションを識別するために使用されます。

シグネチャまたは形状

列挙型として定義されており、action_table 内で以下の値が使用されます。

  • DO_NOTHING
  • SEND_SIGTERM_NOLOG
  • SEND_SIGTERM
  • SEND_SIGKILL
  • GIVEUP

入力と出力

  • 入力: reclaim_one_pid 関数の引数として受け取られます。
  • 出力: なし（アクションの種別を決定する識別子として機能）。

前提条件と不変条件

  • action_table 内で定義された順序に従い、時間経過とともにエスカレーションされることが想定されています。
  • reclaim_one_pid 関数において、この値に基づいて適切なシグナル送信処理が分岐されます。

アルゴリズムまたは振る舞い

親プロセスが子プロセスの回収を行う際、経過時間に応じて action_table から適切な action_t を選択します。 1. DO_NOTHING: 何もせず待機。 2. SEND_SIGTERM_NOLOG: ログを出さずに SIGTERM を送信。 3. SEND_SIGTERM: SIGTERM を送信。 4. SEND_SIGKILL: SIGKILL を送信。 5. GIVEUP: 回収を諦める。

呼び出し元と呼び出し先

  • 呼び出し元: reclaim_one_pid 関数が引数として受け取ります。
  • 呼び出し先: 情報なし。

関連データ構造

  • action_table: action_t と apr_time_t を保持する構造体の配列。
  • reclaim_one_pid: action_t を引数に取り、プロセス制御を実行する関数。

失敗モードと境界条件

  • action_table の範囲外の値が渡された場合の挙動は、reclaim_one_pid 内のスイッチ文の実装に依存します。

保守担当者向けメモ

  • 本シンボルは server/mpm_unix.c 内のプロセス回収ロジックに密結合しています。新しいアクションを追加する場合は、reclaim_one_pid 内の処理も併せて更新する必要があります。
