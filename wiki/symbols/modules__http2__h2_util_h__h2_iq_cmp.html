h2_iq_cmp

定義

  • 種別: typedef
  • 定義箇所: modules/http2/h2_util.h:95

目的

h2_iqueue（整数キュー）内の要素を比較するためのコールバック関数の型定義です。

シグネチャまたは形状

typedef int h2_iq_cmp(int i1, int i2, void *ctx);

入力と出力

  • **入力**:
  • i1, i2: 比較対象となる2つの整数値（主にストリームID等）。
  • ctx: 比較ロジックで使用される任意のコンテキストデータ（ユーザー定義データ）。
  • **出力**:
  • 比較結果を示す整数値を返します。標準的な比較関数と同様に、i1 < i2 なら負の値、i1 == i2 なら 0、i1 > i2 なら正の値を返すことが期待されます。

前提条件と不変条件

  • h2_iqueue のソート操作や要素追加時に、順序を決定するための基準として使用されます。
  • ctx は呼び出し元によって提供され、比較関数内で読み取り専用として扱われることが一般的です。

アルゴリズムまたは振る舞い

  • h2_iq_sort や h2_iq_add などの関数に渡されることで、キュー内の要素の並び順を制御します。
  • 呼び出し側は、この関数ポインタを介して、特定のビジネスロジック（例：優先度順やID順）に基づいたソートを動的に適用できます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_util.c 内の h2_iq_add および h2_iq_sort がこの関数型を引数として受け取り、内部で呼び出します。
  • **呼び出し先**:
  • ユーザーが定義した比較ロジック（具体的な実装は本シンボルではなく、利用側のコードに依存します）。

関連データ構造

  • h2_iqueue: この比較関数を使用してソートや順序維持を行う整数キュー構造体。

失敗モードと境界条件

  • 比較関数内で NULL ポインタや不正な ctx を参照した場合の動作は未定義です。
  • 戻り値が期待される比較規則（負、0、正）に従わない場合、キューのソート順序が不整合になる可能性があります。

保守担当者向けメモ

  • 本シンボルは関数ポインタの型定義であり、具体的な比較ロジックは実装側に委ねられています。
  • h2_iq_add や h2_iq_sort を修正する際は、このコールバックの契約（戻り値の定義）が維持されていることを確認してください。
