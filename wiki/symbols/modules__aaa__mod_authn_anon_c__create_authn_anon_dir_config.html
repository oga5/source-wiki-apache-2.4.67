create_authn_anon_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_anon.c:73

目的

mod_authn_anon モジュールで使用されるディレクトリごとの設定構造体（authn_anon_config_rec）をメモリ上に割り当て、デフォルト値で初期化する。

シグネチャまたは形状

static void *create_authn_anon_dir_config(apr_pool_t *p, char *d)

入力と出力

  • **入力**:
  • p: 設定を割り当てるための apr_pool_t プール。
  • d: ディレクトリパス（本関数内では未使用）。
  • **出力**:
  • 初期化済みの authn_anon_config_rec 構造体へのポインタ。

前提条件と不変条件

  • apr_palloc により、指定されたプール p からメモリが割り当てられる。
  • 戻り値の構造体は、Apache の設定フェーズにおいて匿名認証の動作を制御する各フラグが初期値に設定されている。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して authn_anon_config_rec サイズのメモリを確保する。 2. 許可ユーザーリスト users を NULL に初期化する。 3. 各種フラグを以下のデフォルト値に設定する:

  • nouserid = 0
  • anyuserid = 0
  • logemail = 1
  • verifyemail = 0
  • mustemail = 1

    • 4. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_anon.c:209（ディレクトリ設定作成用フックとして登録）。
  • **呼び出し先**: apr_palloc（APR ライブラリ）。

関連データ構造

  • authn_anon_config_rec: 本関数が初期化する設定保持用構造体。

失敗モードと境界条件

  • apr_palloc がメモリ不足により失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は mod_authn_anon の設定初期化を担当する。新しい設定項目を追加した場合は、必ず本関数内でデフォルト値を定義すること。
