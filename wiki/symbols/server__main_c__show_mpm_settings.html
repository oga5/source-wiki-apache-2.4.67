show_mpm_settings

定義

  • 種別: function
  • 定義箇所: server/main.c:55

目的

現在実行中の Apache HTTP Server の MPM（Multi-Processing Module）構成情報を標準出力へ表示する。

シグネチャまたは形状

static void show_mpm_settings(void)

入力と出力

  • 入力: なし（グローバルな MPM 状態を参照）
  • 出力: 標準出力（stdout）へ MPM 名、スレッドモデル、およびフォークモデルの情報を出力する。

前提条件と不変条件

  • ap_show_mpm() および ap_mpm_query() が利用可能な状態で呼び出される必要がある。
  • 実行環境において標準出力が有効であること。

アルゴリズムまたは振る舞い

1. ap_show_mpm() を呼び出し、MPM 名を表示する。 2. ap_mpm_query() を使用して AP_MPMQ_IS_THREADED を問い合わせる。成功した場合、スレッドモデル（動的、静的、または非スレッド）を判定して表示する。 3. ap_mpm_query() を使用して AP_MPMQ_IS_FORKED を問い合わせる。成功した場合、フォークモデル（動的、静的、または非フォーク）を判定して表示する。

呼び出し元と呼び出し先

  • 呼び出し元: server/main.c:120
  • 呼び出し先:
  • ap_show_mpm()
  • ap_mpm_query()
  • printf()

関連データ構造

  • apr_status_t: ap_mpm_query の戻り値として使用されるステータスコード。

失敗モードと境界条件

  • ap_mpm_query() が APR_SUCCESS 以外の値を返した場合、該当する詳細情報（スレッドまたはフォークの状態）の表示はスキップされる。

保守担当者向けメモ

  • 本関数はサーバー起動時の診断情報出力用であり、実行時の設定変更には関与しない。
  • MPM のクエリ定数（AP_MPMQ_IS_THREADED 等）の解釈は ap_mpm.h の定義に依存している。
