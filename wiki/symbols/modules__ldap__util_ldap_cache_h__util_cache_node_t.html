util_cache_node_t

定義

  • 種別: struct
  • 定義箇所: modules/ldap/util_ldap_cache.h:35

目的

LDAP キャッシュ内の個別のエントリを管理するためのリンクリスト構造体です。キャッシュされたデータ（ペイロード）と、そのメタデータ（追加時刻）を保持します。

シグネチャまたは形状

typedef struct util_cache_node_t {
    void *payload;              /* ペイロードへのポインタ */
    apr_time_t add_time;        /* キャッシュ追加時刻 */
    struct util_cache_node_t *next; /* 次のノードへのポインタ */
} util_cache_node_t;

入力と出力

  • **入力**: util_ald_alloc を介して確保されたメモリ領域に配置されます。
  • **出力**: キャッシュ管理関数（util_ald_cache_fetch 等）を通じて、格納された payload が呼び出し元に提供されます。

前提条件と不変条件

  • payload は、キャッシュ専用のアロケータ（util_ald_alloc）によって管理されるメモリ領域を指す必要があります。
  • next ポインタは、リンクリストの終端では NULL である必要があります。
  • 共有メモリ環境下では、ポインタの直接参照ではなく apr_rmm を介したオフセット管理が前提となります。

アルゴリズムまたは振る舞い

  • キャッシュのリンクリスト構造を形成し、ハッシュ衝突が発生した際のチェイニング（連鎖）に使用されます。
  • add_time は、キャッシュのパージ（util_ald_cache_purge）時に TTL 期限切れを判定するための基準として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: util_ldap_cache_mgr.c 内のキャッシュ管理関数群（util_ald_cache_fetch, util_ald_cache_insert, util_ald_cache_remove 等）が、ノードの確保、走査、解放を行います。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • util_ald_cache_t: キャッシュ全体のメタデータとノード配列を保持するメイン構造体。
  • util_url_node_t, util_search_node_t, util_compare_node_t, util_dn_compare_node_t: 本構造体の payload として格納される具体的なデータ型。

失敗モードと境界条件

  • **メモリ不足**: util_ald_alloc が失敗した場合、ノードの生成ができません。
  • **共有メモリの制約**: 共有メモリ使用時にポインタの整合性が失われた場合、リンクリストの走査が不正なメモリアクセスを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は util_ldap_cache_mgr.c におけるメモリ管理（util_ald_alloc/free）と密接に結合しています。
  • 構造体の変更は、キャッシュのメモリレイアウトに直接影響するため、共有メモリの互換性に注意してください。
