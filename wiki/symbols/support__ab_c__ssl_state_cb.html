ssl_state_cb

定義

  • 種別: function
  • 定義箇所: support/ab.c:595

目的

OpenSSL の情報コールバックとして、SSL/TLS 通信のステータス変化、アラート、およびハンドシェイクの進捗状況を標準エラー出力（bio_err）に記録する。

シグネチャまたは形状

static void ssl_state_cb(const SSL *s, int w, int r)

入力と出力

  • s: 状態変化が発生した SSL オブジェクトへのポインタ。
  • w: イベントの種類を示すビットマスク（SSL_CB_ALERT, SSL_CB_LOOP, SSL_CB_HANDSHAKE_START, SSL_CB_HANDSHAKE_DONE 等）。
  • r: イベントの詳細情報（アラートコードなど）。
  • 出力: 戻り値なし。副作用として bio_err へのログ出力を行う。

前提条件と不変条件

  • bio_err が初期化されていること。
  • OpenSSL のコールバックとして登録されていること。

アルゴリズムまたは振る舞い

1. **アラートの処理**: w に SSL_CB_ALERT が含まれる場合、読み込み/書き込みの別とアラートの詳細をログ出力する。 2. **状態遷移の処理**: w に SSL_CB_LOOP が含まれる場合、接続初期化中か否かを含めた現在の SSL 状態をログ出力する。 3. **ハンドシェイクの処理**: w に SSL_CB_HANDSHAKE_START または SSL_CB_HANDSHAKE_DONE が含まれる場合、ハンドシェイクの開始または完了をログ出力する。

呼び出し元と呼び出し先

  • 呼び出し元: support/ab.c:2720 にて SSL_CTX_set_info_callback を介して登録される。
  • 呼び出し先: OpenSSL ライブラリ内の関数（BIO_printf, SSL_alert_type_string_long, SSL_state_string_long 等）。

関連データ構造

  • SSL (OpenSSL 構造体)
  • bio_err (エラー出力用 BIO)

失敗モードと境界条件

  • s が NULL の場合、OpenSSL 側の内部実装に依存する（本関数内では明示的なチェックなし）。
  • ログ出力先である bio_err が無効な場合、出力は失敗する。

保守担当者向けメモ

  • 本関数はデバッグおよび診断目的で提供されている。
  • OpenSSL のバージョンアップに伴い、SSL_state_string_long 等の API が非推奨となる可能性があるため、ビルド環境の OpenSSL バージョンに注意すること。
