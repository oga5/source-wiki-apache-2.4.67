socache_dc_create

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dc.c:42

目的

distcache プロバイダーのキャッシュインスタンスを初期化し、接続先ターゲット情報を保持するコンテキストを生成します。

シグネチャまたは形状

static const char *socache_dc_create(ap_socache_instance_t **context,
                                     const char *arg,
                                     apr_pool_t *tmp, apr_pool_t *p);

入力と出力

  • **入力**:
  • context: インスタンス構造体へのポインタを格納する出力用引数。
  • arg: キャッシュターゲットを指定する文字列。
  • tmp: 一時的なメモリ割り当て用プール。
  • p: インスタンスの生存期間を管理するメモリプール。
  • **出力**:
  • 成功時は NULL を返します。
  • context 引数を通じて、初期化された ap_socache_instance_t 構造体が返されます。

前提条件と不変条件

  • p は、インスタンスのライフサイクル全体で有効なメモリプールである必要があります。
  • arg は、distcache サーバーの接続先情報として有効な文字列である必要があります。

アルゴリズムまたは振る舞い

1. p プールから ap_socache_instance_t 構造体分のメモリを確保します。 2. 確保した構造体を *context に代入します。 3. arg 文字列を p プール上に複製し、構造体の target メンバに格納します。 4. NULL を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dc.c:176 にて参照されています。
  • **呼び出し先**: apr_palloc および apr_pstrdup（APRライブラリ関数）を呼び出します。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスを保持する構造体。本関数により target メンバが初期化されます。

失敗モードと境界条件

  • メモリ割り当て（apr_palloc）が失敗した場合、APRの規約に従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数はインスタンスのメモリ確保とターゲット文字列の保持のみを行います。実際の distcache クライアントコンテキスト（DC_CTX）の生成は、後続の socache_dc_init で行われます。
