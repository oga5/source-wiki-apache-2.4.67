create_proxyws_dir_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_wstunnel.c:445

目的

mod_proxy_wstunnel モジュールで使用されるディレクトリ単位の設定構造体（proxyws_dir_conf）を生成し、初期化します。

シグネチャまたは形状

static void *create_proxyws_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体のメモリ割り当てに使用するメモリプール。
  • char *dummy: Apache の設定フックのシグネチャに合わせるためのダミー引数。
  • **出力**:
  • void *: 初期化された proxyws_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体はゼロクリアされた状態で割り当てられます。
  • 戻り値の構造体は、Apache のリクエスト処理ライフサイクルに紐付いたメモリプールから確保されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して proxyws_dir_conf サイズのメモリを確保します。 2. fallback_to_proxy_http メンバを 1 に設定し、デフォルトで mod_proxy_http へのフォールバックを有効にします。 3. 初期化済みの構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_wstunnel.c:507（設定構造体の作成時）。
  • **呼び出し先**: apr_pcalloc（APRライブラリ）。

関連データ構造

  • proxyws_dir_conf: 本関数が初期化するディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がプロセスを終了させる可能性があるため、本関数内で明示的なエラーハンドリングは行われていません。

保守担当者向けメモ

  • 本関数は mod_proxy_wstunnel の設定フックの一部です。設定項目を追加する場合は、この関数でデフォルト値を設定してください。
