cookie_dir_rec

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_usertrack.c:82

目的

mod_usertrack モジュールにおいて、ディレクトリ単位（<Directory> や <Location> 等）で設定される Cookie 追跡の構成情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    int enabled;
    cookie_type_e style;
    const char *cookie_name;
    const char *cookie_domain;
    char *regexp_string;
    ap_regex_t *regexp;
    int is_secure;
    int is_httponly;
    const char *samesite;
} cookie_dir_rec;

入力と出力

本構造体は設定データとして機能します。

  • **入力**: Apache の設定ディレクティブ（CookieTracking, CookieName, CookieDomain, CookieExpires, CookieStyle, CookieSameSite 等）により値が設定されます。
  • **出力**: spot_cookie 関数等のロジックにおいて、リクエストヘッダーの解析や Set-Cookie ヘッダー生成のパラメータとして参照されます。

前提条件と不変条件

  • regexp は regexp_string を基に set_and_comp_regexp 関数によってコンパイルされた正規表現オブジェクトである必要があります。
  • regexp は Cookie ヘッダーから追跡用 Cookie を抽出するために使用されます。

アルゴリズムまたは振る舞い

  • 本構造体は、リクエスト処理中に ap_get_module_config(r->per_dir_config, ...) を通じて取得され、現在のリクエストに対する Cookie 追跡の挙動を決定します。
  • regexp_string はデバッグ目的で保持され、regexp は実行時のマッチングに使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • set_and_comp_regexp (引数として参照)
  • spot_cookie (リクエスト処理時に設定を取得)
  • 設定ハンドラ関数群 (set_cookie_name, set_cookie_domain 等)
  • make_cookie_dir (構造体のメモリ確保と初期化)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • cookie_type_e: Cookie のスタイル（フォーマット）を定義する列挙型。
  • ap_regex_t: 正規表現コンパイル結果を保持する Apache API の構造体。
  • cookie_log_state: サーバー全体の設定を保持する構造体。

失敗モードと境界条件

  • regexp が適切にコンパイルされていない場合、Cookie の抽出に失敗する可能性があります。
  • apr_pcalloc によるメモリ確保に失敗した場合、Apache のリクエスト処理が中断される可能性があります。

保守担当者向けメモ

  • 本構造体の各フィールドは、リクエスト処理のプール（request_rec）または設定プールに割り当てられており、リクエスト終了時に自動的に解放されます。
  • regexp の整合性は set_cookie_name 等の設定ハンドラで検証されます。
  • 認証目的のセキュアなトークン生成には使用しないでください。
