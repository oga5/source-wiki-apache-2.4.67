ap_fcgi_begin_request_body_to_array

定義

  • 種別: function
  • 定義箇所: server/util_fcgi.c:68

目的

FastCGI の BEGIN_REQUEST レコードのボディ構造体（ap_fcgi_begin_request_body）を、ネットワーク転送に適したバイト配列形式へシリアライズします。

シグネチャまたは形状

void ap_fcgi_begin_request_body_to_array(ap_fcgi_begin_request_body *h, unsigned char a[])

入力と出力

  • **入力**:
  • h: シリアライズ対象の ap_fcgi_begin_request_body 構造体へのポインタ。
  • a: 変換結果を格納するバイト配列（呼び出し元が確保）。
  • **出力**:
  • なし（戻り値は void）。a が指すメモリ領域に直接書き込まれます。

前提条件と不変条件

  • a は、FastCGI プロトコルで定義されたボディサイズ（8バイト）以上の容量を持つ必要があります。
  • h は有効な構造体ポインタである必要があります。

アルゴリズムまたは振る舞い

構造体の各フィールドを、定義済みのオフセット定数に基づいてバイト配列 a の各インデックスへコピーします。

1. roleB1 および roleB0 をオフセット AP_FCGI_BRB_ROLEB1_OFFSET / ROLEB0_OFFSET に配置。 2. flags を AP_FCGI_BRB_FLAGS_OFFSET に配置。 3. reserved[0] から reserved[4] までの予約領域を、対応する AP_FCGI_BRB_RESERVEDx_OFFSET に順次配置。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:281
  • modules/proxy/mod_proxy_fcgi.c:339
  • **呼び出し先**:
  • なし（本関数は直接的な関数呼び出しを行わず、メモリへの代入のみを行います）。

関連データ構造

  • ap_fcgi_begin_request_body: FastCGI の BEGIN_REQUEST レコードのボディを表現する構造体。

失敗モードと境界条件

  • 本関数自体はエラーチェックを行いません。
  • a が十分なサイズを確保していない場合、バッファオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数はメモリ割り当てを行いません。メモリの所有権は呼び出し元にあります。
  • オフセット定数（AP_FCGI_BRB_..._OFFSET）は util_fcgi.h で定義されています。プロトコル仕様の変更時にはこれらの定数と本関数のマッピングを確認してください。
