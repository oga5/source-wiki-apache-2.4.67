lua_write_body

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:280

目的

HTTP リクエスト（POST/PUT 等）のボディデータを読み取り、指定されたファイルへ書き出す。

シグネチャまたは形状

apr_status_t lua_write_body(request_rec *r, apr_file_t *file, apr_off_t *size)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体 (request_rec)。
  • file: 書き込み先の APR ファイルハンドル (apr_file_t)。
  • **出力**:
  • size: 読み取ったボディの合計サイズが格納されるポインタ。
  • 戻り値: 処理結果を示す apr_status_t（成功時は OK または DONE）。

前提条件と不変条件

  • r は有効なリクエスト構造体であること。
  • file は書き込み可能な状態でオープンされていること。
  • *size は関数開始時に 0 に初期化される。

アルゴリズムまたは振る舞い

1. ap_setup_client_block を呼び出し、リクエストボディの読み込み準備を行う。エラー時は即座に終了する。 2. ap_should_client_block でボディの存在を確認する。 3. ボディが存在する場合、ap_get_client_block を用いて HUGE_STRING_LEN バッファ単位でデータをループ読み込みする。 4. 読み込んだデータを apr_file_write_full を使用して指定ファイルへ書き出す。 5. ボディが存在しない場合は DONE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_request.c:492
  • **呼び出し先**:
  • ap_setup_client_block
  • ap_should_client_block
  • ap_get_client_block
  • apr_file_write_full

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • apr_file_t: APR のファイル操作用構造体。
  • apr_off_t: ファイルサイズやオフセットを扱う型。

失敗モードと境界条件

  • ap_setup_client_block がエラーを返した場合、そのステータスコードをそのまま返す。
  • apr_file_write_full による書き込みが APR_SUCCESS 以外となった場合、そのエラーコードを返して処理を中断する。
  • ボディサイズが r->remaining を超える場合、読み込み量を調整する安全策が講じられている。

保守担当者向けメモ

  • 本関数は POST/PUT リクエストのボディをファイルへ転送する際に使用される。
  • メモリ管理は request_rec のプールに依存している。
  • HUGE_STRING_LEN を超えるバッファサイズでの読み込みは行われない。
