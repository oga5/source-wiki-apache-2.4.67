md_data_clear

定義

  • 種別: function
  • 定義箇所: modules/md/md_util.c:112

目的

md_data_t 構造体が保持する動的メモリを解放し、構造体自体をゼロクリアすることで、安全なリセットを行う。

シグネチャまたは形状

void md_data_clear(md_data_t *d)

入力と出力

  • **入力**: md_data_t *d （クリア対象のデータ構造体へのポインタ）
  • **出力**: なし（副作用としてメモリ解放とゼロクリアを実行）

前提条件と不変条件

  • d が NULL の場合、関数は何もしない。
  • d->data が非 NULL かつ d->free_data が関数ポインタとして設定されている場合、d->free_data を呼び出してメモリを解放する。

アルゴリズムまたは振る舞い

1. 引数 d が有効か確認する。 2. d->data が存在し、かつ解放用関数 d->free_data が定義されていれば、d->data を引数として d->free_data を実行する。 3. memset を使用し、d が指すメモリ領域全体を 0 で埋める。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_ocsp.c: ostat->req_der, ostat->resp_der, der, new_der, d のクリアに使用。
  • modules/md/md_util.c: dest のクリアに使用。
  • **呼び出し先**:
  • d->free_data （構造体内に保持された関数ポインタ）

関連データ構造

  • md_data_t: データチャンクと解放用関数ポインタを保持する構造体。

失敗モードと境界条件

  • d が NULL の場合、安全に無視される。
  • d->free_data が NULL の場合、d->data が指すメモリは解放されず、構造体のみがゼロクリアされる（メモリリークの可能性があるため注意が必要）。

保守担当者向けメモ

  • 本関数は md_data_t の所有権管理において重要である。カスタムの解放ロジックが必要な場合は、free_data に適切な関数が登録されていることを確認すること。
