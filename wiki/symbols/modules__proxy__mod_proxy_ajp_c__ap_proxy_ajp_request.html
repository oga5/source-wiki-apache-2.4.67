ap_proxy_ajp_request

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ajp.c:181

目的

AJP (Apache JServ Protocol) v1.3 を使用して、Apache HTTP Server からバックエンドサーバーへリクエストを転送し、レスポンスを処理する。

シグネチャまたは形状

int ap_proxy_ajp_request(apr_pool_t *p, request_rec *r,
                         proxy_conn_rec *conn,
                         conn_rec *origin,
                         proxy_dir_conf *conf,
                         apr_uri_t *uri,
                         char *url, char *server_portstr);

入力と出力

  • **入力**: リクエスト情報 (r)、バックエンド接続情報 (conn)、URI 情報 (uri) 等。
  • **出力**: 処理結果を示すステータスコード（OK または HTTP_* エラーコード）。

前提条件と不変条件

  • conn は ap_proxy_acquire_connection 等で既に確保されている必要がある。
  • r->pool を通じたメモリ管理が行われる。
  • 冪等なリクエスト（is_idempotent が METHOD_IDEMPOTENT を返す場合）は、エラー時に再試行可能なステータスを返すことがある。

アルゴリズムまたは振る舞い

1. **ヘッダー送信**: ajp_send_header を呼び出し、AJP ヘッダーをバックエンドへ送信する。 2. **ボディ転送**: クライアントからのリクエストボディを ap_get_brigade で読み込み、ajp_send_data_msg を介してバックエンドへストリーミングする。 3. **レスポンス受信**: ajp_read_header でバックエンドからのレスポンスヘッダーを読み取り、ajp_parse_type でメッセージ種別を判定する。 4. **ループ処理**: CMD_AJP13_GET_BODY_CHUNK 等のコマンドに応じて、ボディの追加読み込みやレスポンスのフィルタチェーンへの転送を繰り返す。 5. **終了**: CMD_AJP13_END_RESPONSE を受け取り、リクエストを完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ajp.c 内の proxy_ajp_handler（直接の呼び出し箇所はソースコード行 854 に確認あり）。
  • **呼び出し先**: ajp_send_header, ajp_alloc_data_msg, ap_get_brigade, apr_brigade_flatten, ajp_send_data_msg, ajp_read_header, ajp_parse_type 等。

関連データ構造

  • proxy_conn_rec: バックエンド接続状態。
  • ajp_msg_t: AJP 通信メッセージバッファ。
  • apr_bucket_brigade: 入出力データのストリーム管理。

失敗モードと境界条件

  • **接続エラー**: バックエンドとの通信失敗時は conn->close = 1 を設定し、コネクションを破棄する。
  • **タイムアウト**: APR_STATUS_IS_TIMEUP を検知した場合、HTTP_GATEWAY_TIME_OUT や HTTP_REQUEST_TIME_OUT を返す。
  • **冪等性**: 冪等なリクエストであれば、特定の失敗時に HTTP_SERVICE_UNAVAILABLE を返し、バランサーによるリトライを許容する。

保守担当者向けメモ

  • 本関数は AJP プロトコルの高レベル制御を担当する。低レベルなパケット操作は ajp_msg.c 等に委譲されている。
  • エラー発生時は conn->close = 1 を適切に設定し、不正な状態のコネクションが再利用されないようにすること。
  • AJP には明示的なフラッシュメッセージがないため、CMD_AJP13_SEND_BODY_CHUNK 処理時にタイムアウトベースのフラッシュ制御が行われている。
