ap_input_mode_t

定義

  • 種別: enum
  • 定義箇所: include/util_filter.h:41

目的

Apache HTTP Server の入力フィルタにおける読み取り操作の動作モードを定義します。フィルタチェーンを通じてデータを要求する際、呼び出し元がどのような形式でデータを期待しているかを指定するために使用されます。

シグネチャまたは形状

typedef enum {
    AP_MODE_READBYTES,
    AP_MODE_GETLINE,
    AP_MODE_EATCRLF,
    AP_MODE_SPECULATIVE,
    AP_MODE_EXHAUSTIVE,
    AP_MODE_INIT
} ap_input_mode_t;

入力と出力

本シンボル自体は列挙型であり、入力や出力を持つ関数ではありません。フィルタチェーンの読み取り関数（ap_get_brigade 等）において、引数として渡されることで処理の制御フローを決定します。

前提条件と不変条件

  • 各モードは、フィルタが apr_bucket_brigade をどのように処理すべきかという制約を規定します。
  • AP_MODE_INIT は接続の初期化に使用されます。

アルゴリズムまたは振る舞い

  • AP_MODE_READBYTES: 指定されたバイト数以下のデータを返します。
  • AP_MODE_GETLINE: CRLF で終わる 1 行分のデータを返します（行が長すぎる場合は部分的なデータを返す可能性があります）。
  • AP_MODE_EATCRLF: フィルタが認識する CRLF ペアを暗黙的に除去します。
  • AP_MODE_SPECULATIVE: 読み取りを推測的（speculative）として扱い、取得したデータは後続の別モードでの読み取りのために保存されます。
  • AP_MODE_EXHAUSTIVE: 読み取り可能なすべてのデータを取得します（使用には極めて慎重を要します）。
  • AP_MODE_INIT: 接続の初期化（例: NNTP や FTP over SSL）を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: include/http_core.h、include/mod_core.h、include/util_filter.h 内の各関数、および mod_dumpio.c、mod_case_filter_in.c、mod_buffer.c、mod_charset_lite.c、mod_deflate.c、mod_ext_filter.c などのフィルタモジュール。
  • **呼び出し先**: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • apr_bucket_brigade: フィルタ間で受け渡されるデータストリームの単位。
  • ap_filter_t: フィルタのインスタンス構造体。

失敗モードと境界条件

  • AP_MODE_GETLINE において、CRLF が見つからない場合や行が長すぎる場合、フィルタは部分的なデータを返すことが許容されています。
  • AP_MODE_EXHAUSTIVE の使用は、リソース消費の観点から極めて慎重に行う必要があります。

保守担当者向けメモ

  • 本シンボルは include/util_filter.h で定義されており、フィルタチェーンの挙動を制御する中核的な列挙型です。
  • 新しい読み取りモードを追加する場合は、既存のフィルタ実装（mod_deflate 等）がそのモードを正しく処理できるか確認してください。
