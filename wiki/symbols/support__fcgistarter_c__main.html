main

定義

  • 種別: function
  • 定義箇所: support/fcgistarter.c:56

目的

FastCGI プロセスを起動し、指定された TCP ポートでリクエストを待ち受けるためのスタンドアロンユーティリティのエントリポイントです。

シグネチャまたは形状

int main(int argc, const char * const argv[])

入力と出力

  • **入力**: コマンドライン引数 (-c コマンド, -p ポート, -i インターフェース, -N プロセス数)
  • **出力**: 成功時は EXIT_SUCCESS、エラー時は EXIT_FAILURE または exit_error を経由した終了コードを返します。

前提条件と不変条件

  • FastCGI プロセスがファイルディスクリプタ 0 を介してソケットを受け取るため、プロセス生成後に dup2 を用いたディスクリプタの差し替えが行われます。

アルゴリズムまたは振る舞い

1. **初期化**: apr_app_initialize で環境を構築し、引数を解析します。 2. **ソケット準備**: 指定されたポートで TCP ソケットを作成、バインド、リッスン状態にします。 3. **デーモン化**: apr_proc_detach を呼び出し、制御端末から切り離します。 4. **プロセス生成**: 指定された数 (-N) だけループし、apr_proc_fork を実行します。

  • **子プロセス**: ソケットをファイルディスクリプタ 0 に複製 (dup2) し、execl でターゲットコマンドを実行します。
  • **親プロセス**: 指定数の生成完了後にソケットを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: OS の実行環境（エントリポイント）。
  • **呼び出し先**:
  • apr_* 系関数（APR ライブラリ）
  • usage（引数エラー時）
  • exit_error（APR エラー発生時）
  • execl（子プロセスでのコマンド実行）

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • apr_socket_t: 待ち受け用 TCP ソケット。
  • apr_proc_t: 生成される子プロセスの管理構造体。

失敗モードと境界条件

  • 引数不足や不正なポート指定時は usage を呼び出し終了します。
  • ソケット作成やバインド、デーモン化に失敗した場合は exit_error でエラーメッセージを出力し終了します。
  • 子プロセス生成後の dup2 や execl 実行時に問題が発生した場合、子プロセス側では exit(EXIT_FAILURE) を呼び出します。

保守担当者向けメモ

  • 本関数は Unix 系のファイルディスクリプタ操作に強く依存しています。
  • apr_proc_create を使用せず dup2 を用いる設計のため、子プロセス生成後のエラーハンドリングが制限されています。
