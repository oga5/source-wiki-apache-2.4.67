ap_create_request_config

定義

  • 種別: function
  • 定義箇所: server/config.c:356

目的

HTTPリクエスト処理で使用される、モジュールごとの設定ベクトル（ap_conf_vector_t）を初期化・生成します。

シグネチャまたは形状

ap_conf_vector_t *ap_create_request_config(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p - 設定ベクトルのメモリ割り当てに使用するメモリプール。
  • **出力**: ap_conf_vector_t * - 初期化された空の設定ベクトルへのポインタ。

前提条件と不変条件

  • 引数 p は有効な apr_pool_t である必要があります。
  • 内部的に create_empty_config を呼び出すため、同関数が要求するメモリ割り当ての制約に従います。

アルゴリズムまたは振る舞い

1. 指定されたメモリプール p を使用して、create_empty_config(p) を呼び出します。 2. create_empty_config が生成した空の設定ベクトルをそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: create_empty_config
  • **呼び出し元**:
  • modules/generators/mod_cgid.c
  • modules/http/http_request.c
  • modules/http2/h2_request.c
  • modules/proxy/mod_proxy_hcheck.c
  • modules/proxy/mod_proxy_http.c
  • modules/proxy/mod_proxy_uwsgi.c
  • server/protocol.c
  • server/request.c

関連データ構造

  • ap_conf_vector_t: モジュールごとの設定を保持する不透明なベクトル型。
  • apr_pool_t: メモリ管理のためのリソースプール。

失敗モードと境界条件

  • メモリ不足時には apr_pool_t の割り当てメカニズムにより、プロセスが終了またはエラーを報告する可能性があります。

保守担当者向けメモ

  • 本関数は create_empty_config の単なるラッパーです。将来的にリクエスト固有の初期化ロジックが必要となった場合は、この関数内で処理を追加することが想定されます。
