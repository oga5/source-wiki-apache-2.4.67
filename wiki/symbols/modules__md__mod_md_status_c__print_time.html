print_time

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:249

目的

指定されたタイムスタンプを、status_ctx の出力モード（HTML またはテキスト）に応じて人間が読みやすい形式に変換し、HTTP レスポンスバッファへ出力する。

シグネチャまたは形状

static void print_time(status_ctx *ctx, const char *label, apr_time_t t)

入力と出力

  • **入力**:
  • ctx: ステータス出力用のコンテキスト（出力先バッファやフォーマットフラグを保持）。
  • label: 出力するラベル文字列。
  • t: 出力対象の apr_time_t 型タイムスタンプ。
  • **出力**: なし（副作用として ctx->bb に文字列を書き込む）。

前提条件と不変条件

  • t が 0 の場合、本関数は何も出力せず即座に終了する（「未設定」を意味する）。
  • ctx->bb は有効な apr_bucket_brigade である必要がある。

アルゴリズムまたは振る舞い

1. t が 0 であれば終了する。 2. HTML_STATUS(ctx) が真の場合（HTML モード）:

  • t と現在時刻を比較し、過去であれば「... ago」、未来であれば「in ...」という形式で相対時間を算出する。
  • 4日以上の差がある場合は、YYYY-MM-DD 形式の日付を <span> タグで出力する。
  • それ以外の場合は、md_duration_roughly を用いて概算時間を表示する。
  • ツールチップとして RFC822 形式の完全な日時を付与する。

    • 3. それ以外の場合（テキストモード）:

  • 現在時刻からの差分（秒単位）を計算し、prefix と共にキー・値形式で出力する。

関連データ構造

  • status_ctx: 出力先バッファ (bb) や出力モードを管理する構造体。

失敗モードと境界条件

  • t が 0 の場合は何も出力されない。
  • label が空文字列または NULL の場合、ラベルなしで値のみが出力される。

保守担当者向けメモ

  • 本関数は mod_status の出力形式に強く依存している。HTML モードでの表示ロジックを変更する場合は、HTML_STATUS マクロの定義および md_duration_roughly の仕様を確認すること。
