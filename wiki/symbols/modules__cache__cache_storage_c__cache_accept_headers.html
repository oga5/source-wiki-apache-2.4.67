cache_accept_headers

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_storage.c:161

目的

キャッシュされたレスポンスヘッダーと現在のリクエストヘッダーをマージし、r->headers_out を更新して HTTP レスポンスを構築する。

シグネチャまたは形状

void cache_accept_headers(cache_handle_t *h, request_rec *r, apr_table_t *top,
                          apr_table_t *bottom, int revalidation)

入力と出力

  • h: キャッシュハンドル。
  • r: 現在のリクエスト構造体。
  • top: マージ対象のヘッダーテーブル（通常はキャッシュされたヘッダー）。
  • bottom: ベースとなるヘッダーテーブル。
  • revalidation: 再検証中か否かを示すフラグ。
  • 出力: r->headers_out が更新され、Content-Type や Last-Modified がリクエスト構造体に反映される。

前提条件と不変条件

  • すべての操作は r->pool を使用してメモリ管理される。
  • revalidation が真の場合、bottom を元に新しいテーブルが作成される。

アルゴリズムまたは振る舞い

1. **ヘッダーの初期化**: revalidation が真であれば bottom をフィルタリングして r->headers_out を作成。偽であれば bottom をコピーする。 2. **ヘッダーのマージ**: top に含まれるヘッダーを r->headers_out から削除（remove_header_do）した後、追加（add_header_do）する。 3. **Content-Type の処理**: r->headers_out から Content-Type を取得し、ap_set_content_type で設定。その後、重複を避けるため r->headers_out および r->err_headers_out から Content-Type を削除する。 4. **Last-Modified の処理**: Last-Modified ヘッダーが存在する場合、ap_update_mtime および ap_set_last_modified を呼び出し、リクエストの最終更新時刻を更新する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/cache/cache_storage.c:400
  • modules/cache/mod_cache.c:1476
  • 呼び出し先:
  • apr_table_do (ヘルパー関数 filter_header_do, remove_header_do, add_header_do を使用)
  • ap_set_content_type
  • apr_table_unset
  • ap_update_mtime
  • ap_set_last_modified

関連データ構造

  • cache_handle_t: キャッシュエントリへのアクセスハンドル。
  • request_rec: Apache のリクエスト構造体。
  • apr_table_t: ヘッダーを保持するテーブル構造。

失敗モードと境界条件

  • Content-Type や Last-Modified が存在しない場合、該当する処理はスキップされる。
  • apr_table_do 等の APR 関数が失敗した場合の挙動は、APR の仕様に依存する。

保守担当者向けメモ

  • 本関数は r->headers_out を直接操作するため、フィルタチェーン内での呼び出し順序に注意すること。
  • Content-Type の削除は、HTTP_HEADER フィルタが r->content_type を優先して上書きする仕様に基づいている。
