fcomp

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:199

目的

sed スクリプトを一行ずつ読み込み、解析して内部表現（sed_reptr_t のリスト）へ変換するコンパイルエンジン。

シグネチャまたは形状

static int fcomp(sed_commands_t *commands, apr_file_t *fin)

入力と出力

  • **入力**:
  • commands: コンパイル状態を保持する sed_commands_t 構造体。
  • fin: スクリプト読み込み元の apr_file_t。
  • **出力**:
  • 成功時は 0 を返す。
  • 構文エラーやメモリ不足などの失敗時は -1 を返す。

前提条件と不変条件

  • commands->pool が有効な apr_pool_t であること。
  • commands->linebuf が未割り当ての場合、内部で LBSIZE + 1 のサイズで確保される。
  • スクリプトの各行は rline を通じて読み込まれ、commands->cp を介して解析される。

アルゴリズムまたは振る舞い

1. **初期化**: linebuf を確保し、最初の行を読み込む。#n で始まる場合は nflag をセットする。 2. **ループ処理**: rline でスクリプトを一行ずつ読み込み、以下の手順を繰り返す。

  • 空行やコメント行をスキップする。
  • address 関数でコマンドのアドレス範囲を解析・特定する。
  • switch 文でコマンド文字（!, {, }, =, :, a, c, i, g, G, h, H, t, b, n, N, p, P, r 等）を判定する。
  • 各コマンドに応じたヘルパー関数（text 等）を呼び出し、sed_reptr_t 構造体に属性を格納する。
  • ラベル（:）や分岐（b, t）については、search や dechain を用いてラベルチェーンを解決する。

    • 3. **終了**: ファイル終端に達するか、エラー発生時に終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed0.c 内の sed_compile_string および sed_compile_file から呼び出される。
  • **呼び出し先**:
  • rline: スクリプトの読み込み。
  • address: アドレス指定の解析。
  • command_errf: エラー報告。
  • alloc_reptr: コマンドノードの割り当て。
  • text: テキストブロックの抽出。
  • search, dechain: ラベル解決。

関連データ構造

  • sed_commands_t: コンパイル全体のコンテキスト。
  • sed_reptr_t: 個々の sed コマンドを保持するノード。
  • sed_label_t: 分岐先ラベルの管理構造。

失敗モードと境界条件

  • **構文エラー**: 不明なコマンドや不正なアドレス指定、閉じ括弧の不整合などで -1 を返す。
  • **メモリ制限**: RESIZE を超えるアドレス指定や、ラベルの制限（labend）を超えた場合にエラーとなる。
  • **読み込みエラー**: rline が失敗した場合に処理を中断する。

保守担当者向けメモ

  • rline は 1 バイトずつ読み込む実装のため、大規模なスクリプトではパフォーマンスに注意が必要。
  • S5EMUL マクロの有無により、テキスト挿入時の挙動が変化する可能性がある。
  • エラー発生時は command_errf を通じて詳細が報告されるため、呼び出し側は戻り値を確認すること。
