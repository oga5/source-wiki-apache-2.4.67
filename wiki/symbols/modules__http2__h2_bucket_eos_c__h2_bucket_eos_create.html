h2_bucket_eos_create

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_eos.c:72

目的

HTTP/2 ストリームの終了（EOS: End of Stream）を通知するためのカスタム APR バケットを生成し、ストリームのクリーンアップ処理を登録します。

シグネチャまたは形状

apr_bucket *h2_bucket_eos_create(apr_bucket_alloc_t *list, h2_stream *stream)

入力と出力

  • **入力**:
  • list: バケット割り当て用のアロケータ（apr_bucket_alloc_t）。
  • stream: 紐付ける対象の h2_stream オブジェクト（NULL 許容）。
  • **出力**:
  • 初期化済みの apr_bucket 構造体へのポインタ。

前提条件と不変条件

  • list は有効なバケットアロケータである必要があります。
  • stream が提供された場合、そのプール（stream->pool）はバケットの生存期間中有効である必要があります。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を使用してバケットメモリを確保します。 2. APR_BUCKET_INIT でバケットを初期化し、解放関数を apr_bucket_free に設定します。 3. h2_bucket_eos_make を呼び出し、バケットの型を EOS 用に設定し、stream を内部データとして保持させます。 4. stream が存在する場合、stream->pool に対して bucket_cleanup をプリクリーンアップハンドラとして登録します。これにより、プール破棄時にストリームのクリーンアップが確実に行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:1688（直接的な証拠）
  • **呼び出し先**:
  • apr_bucket_alloc
  • h2_bucket_eos_make
  • apr_pool_pre_cleanup_register
  • bucket_cleanup（ハンドラとして登録）

関連データ構造

  • apr_bucket: APR の基本バケット構造。
  • h2_stream: HTTP/2 ストリームのコンテキスト。
  • h2_bucket_eos: h2_bucket_eos_make によってバケットの data メンバに格納される内部構造体。

失敗モードと境界条件

  • stream が NULL の場合、クリーンアップハンドラの登録はスキップされますが、バケット自体は正常に生成されます。
  • メモリ不足時には apr_bucket_alloc が失敗する可能性があります。

保守担当者向けメモ

  • 本関数はメタデータバケットを生成するものであり、読み取り時に実際のデータは返しません。
  • bucket_cleanup と bucket_destroy の両方でストリームの生存期間を管理する設計となっているため、変更時は両方の整合性に注意してください。
