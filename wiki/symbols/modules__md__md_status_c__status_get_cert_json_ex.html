status_get_cert_json_ex

定義

  • 種別: function
  • 定義箇所: modules/md/md_status.c:142

目的

Managed Domain (MD) に関連付けられた証明書の詳細情報を JSON 形式で生成し、OCSP ステータスや更新ジョブ情報を含めて拡張する。

シグネチャまたは形状

apr_status_t status_get_cert_json_ex(
    md_json_t **pjson,
    const md_cert_t *cert,
    const md_t *md,
    md_reg_t *reg,
    md_ocsp_reg_t *ocsp,
    int with_logs,
    apr_pool_t *p)

入力と出力

  • **入力**:
  • cert: 対象の証明書オブジェクト。
  • md: 管理対象ドメイン設定。
  • reg: MD レジストリハンドル。
  • ocsp: OCSP レジストリハンドル。
  • with_logs: ジョブログを含めるかどうかのフラグ。
  • p: メモリ割り当て用プール。
  • **出力**:
  • pjson: 生成された JSON オブジェクトへのポインタ。失敗時は NULL。
  • 戻り値: APR_SUCCESS (成功時)、またはエラーコード。

前提条件と不変条件

  • p プールは、JSON オブジェクトの生存期間をカバーする十分な寿命を持つこと。
  • md がステープリングを有効にしている場合、ocsp が有効なポインタであること。

アルゴリズムまたは振る舞い

1. status_get_cert_json を呼び出し、証明書の基本情報を JSON オブジェクトとして取得する。 2. md_cert_get_ari_cert_id を使用し、ARI (Automatic Renewal Information) 証明書 ID が存在すれば JSON に追加する。 3. md->stapling が有効かつ ocsp が提供されている場合:

  • md_ocsp_get_meta を呼び出し、OCSP ステータスと有効期間を取得する。
  • 成功すれば、OCSP ステータス名と有効期間を JSON に書き込む。
  • job_loadj を使用して OCSP 更新ジョブ情報を読み込み、存在すれば JSON の renewal フィールドにマージする。

    • 4. 最終的な JSON オブジェクトを *pjson に格納して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_status.c:214
  • **呼び出し先**:
  • status_get_cert_json
  • md_cert_get_ari_cert_id
  • md_ocsp_get_meta
  • md_ocsp_cert_stat_name
  • md_json_set_timeperiod
  • job_loadj
  • md_json_sets / md_json_setj

関連データ構造

  • md_json_t: JSON データの保持に使用。
  • md_cert_t: 証明書情報のソース。
  • md_t: ドメイン設定。
  • md_ocsp_cert_stat_t: OCSP ステータス列挙型。

失敗モードと境界条件

  • status_get_cert_json が失敗した場合、即座に終了する。
  • OCSP メタデータの取得失敗時、ENOENT（ファイルなし）以外のエラーが発生した場合は処理を中断する。
  • job_loadj の失敗は、OCSP 更新情報が取得できないことを意味するが、証明書自体の JSON 生成は継続される。

保守担当者向けメモ

  • 本関数は md_status_get_md_json の内部処理の一部として機能する。
  • MD_KEY_* 定数を使用して JSON キーが管理されているため、スキーマ変更時は md_json.h の定義と整合性を確認すること。
