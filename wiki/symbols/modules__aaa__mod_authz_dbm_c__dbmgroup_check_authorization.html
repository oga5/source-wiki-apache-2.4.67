dbmgroup_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbm.c:158

目的

Apache HTTP Server において、require dbm-group ディレクティブに基づき、DBM ファイルから取得したユーザーの所属グループ情報と、要求されたグループを照合して認可判断を行う。

シグネチャまたは形状

static authz_status dbmgroup_check_authorization(request_rec *r,
                                                 const char *require_args,
                                                 const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • require_args: 未パースの引数文字列。
  • parsed_require_args: パース済みの式情報 (ap_expr_info_t)。
  • **出力**:
  • AUTHZ_GRANTED: 認可成功。
  • AUTHZ_DENIED: 認可失敗（グループ不一致、ファイル未指定、ユーザー不在など）。
  • AUTHZ_DENIED_NO_USER: ユーザー名がリクエストに存在しない場合。
  • AUTHZ_GENERAL_ERROR: DBM ファイルの読み取りエラー。

前提条件と不変条件

  • r->user が設定されていること。
  • authz_dbm_config_rec 設定構造体において grpfile が定義されていること。
  • メモリ割り当てはすべて r->pool を使用し、リクエスト終了時に解放される。

アルゴリズムまたは振る舞い

1. r->user の存在を確認し、なければ AUTHZ_DENIED_NO_USER を返す。 2. 設定から DBM ファイルパスを取得し、未定義の場合はエラーログを出力して AUTHZ_DENIED を返す。 3. get_dbm_grp を呼び出し、user:realm をキーとして DBM ファイルからカンマ区切りのグループ文字列を取得する。 4. 取得失敗時は AUTHZ_GENERAL_ERROR を、グループ情報が空の場合は AUTHZ_DENIED を返す。 5. ap_expr_str_exec を使用して parsed_require_args を評価し、要求されたグループ名を取得する。 6. 取得したグループ文字列をカンマでパースし、要求されたグループ名と一致するものがあるか順次比較する。 7. 一致すれば AUTHZ_GRANTED を返し、最後まで一致しなければ AUTHZ_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbm.c:319 にて authz_dbmgroup_provider 構造体を通じて登録されている（直接の呼び出し元は mod_authz_core）。
  • **呼び出し先**:
  • get_dbm_grp: DBM ファイルからのグループ情報取得。
  • ap_expr_str_exec: require 式の評価。
  • ap_getword_white / ap_getword: 文字列のパース。
  • ap_log_rerror: エラーログ出力。

関連データ構造

  • authz_dbm_config_rec: DBM ファイルパスやタイプを保持するディレクトリ設定構造体。
  • request_rec: リクエスト情報およびメモリプール (r->pool) を保持。
  • ap_expr_info_t: require ディレクティブの式評価情報。

失敗モードと境界条件

  • **ユーザー不在**: r->user が NULL の場合、AUTHZ_DENIED_NO_USER を返す。
  • **設定不備**: grpfile が未設定の場合、エラーログを出力し AUTHZ_DENIED を返す。
  • **DBM 読み取り失敗**: get_dbm_grp が APR_SUCCESS 以外を返した場合、AUTHZ_GENERAL_ERROR を返す。
  • **グループ不一致**: ユーザーが DBM に存在しない、または要求されたグループに所属していない場合、AUTHZ_DENIED を返す。

保守担当者向けメモ

  • DBM ファイルはリクエストごとに開閉されるため、高負荷時にはパフォーマンスへの影響を考慮すること。
  • DBM ファイル内の値に : が含まれる場合、password:groupname の形式を想定したパースが行われる点に注意。
