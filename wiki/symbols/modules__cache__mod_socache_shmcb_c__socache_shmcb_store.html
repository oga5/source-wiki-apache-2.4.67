socache_shmcb_store

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_shmcb.c:497

目的

共有メモリベースの循環バッファ（SHMCB）キャッシュにデータを保存します。

シグネチャまたは形状

apr_status_t socache_shmcb_store(ap_socache_instance_t *ctx,
                                 server_rec *s, const unsigned char *id,
                                 unsigned int idlen, apr_time_t expiry,
                                 unsigned char *encoded,
                                 unsigned int len_encoded,
                                 apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンスのコンテキスト。
  • s: サーバーレコード。
  • id: キャッシュエントリを識別するキー。
  • idlen: id の長さ（バイト単位）。
  • expiry: エントリの有効期限。
  • encoded: 保存するデータ本体。
  • len_encoded: encoded の長さ。
  • p: メモリプール。
  • **出力**:
  • APR_SUCCESS: 正常終了。
  • APR_EINVAL: idlen が 4 未満の場合。
  • APR_ENOSPC: ストレージへの書き込みに失敗した場合。

前提条件と不変条件

  • idlen は 4 以上である必要があります。
  • 本関数は排他制御を行いません。呼び出し側で適切なロック管理が必要です。

アルゴリズムまたは振る舞い

1. SHMCB_MASK マクロを使用して、id に基づき対象のサブキャッシュを特定します。 2. idlen が 4 未満の場合はエラーを返します。 3. shmcb_subcache_remove を呼び出し、既存の同一 ID エントリを削除（置換準備）します。 4. shmcb_subcache_store を呼び出し、新しいデータをサブキャッシュに書き込みます。 5. 書き込み成功時、置換が発生したか新規作成かに応じて、header 内の統計カウンタ（stat_replaced または stat_stores）をインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_shmcb.c:1061（ap_socache_provider_t 構造体への登録経由）。
  • **呼び出し先**:
  • shmcb_subcache_remove
  • shmcb_subcache_store
  • ap_log_error

関連データ構造

  • SHMCBHeader: 共有メモリのメタデータ管理。
  • SHMCBSubcache: 個別のサブキャッシュ管理構造。
  • ap_socache_instance_t: モジュールのインスタンスコンテキスト。

失敗モードと境界条件

  • **ID長不足**: idlen < 4 の場合、APR_EINVAL を返し保存を中止します。
  • **領域不足**: shmcb_subcache_store が失敗した場合、APR_ENOSPC を返します。

保守担当者向けメモ

  • 本関数は AP_SOCACHE_FLAG_NOTMPSAFE モジュールの一部であり、スレッドセーフではありません。
  • unsigned int を使用したサイズ計算が含まれており、非常に大きなキャッシュサイズ設定時にはオーバーフローのリスクがある点に注意してください。
