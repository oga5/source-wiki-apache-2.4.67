get_port_key

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:232

目的

現在の接続情報（conn_rec）に基づき、対応するアップグレード可能なセキュアリスナーの鍵（key）を検索して返します。

シグネチャまたは形状

static char *get_port_key(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（現在の接続情報）
  • **出力**: 一致するリスナーが見つかった場合はその key（char *）、見つからない場合は NULL を返します。

前提条件と不変条件

  • グローバル変数 ap_seclistenersup が初期化されている必要があります。
  • c->local_addr および c->local_ip が有効である必要があります。

アルゴリズムまたは振る舞い

1. グローバルなアップグレード可能リスナーリスト ap_seclistenersup を走査します。 2. 以下の条件を両方満たすリスナーを探します。

  • リスナーのポート番号が接続のローカルポートと一致する。
  • リスナーのアドレスが "0.0.0.0"（全インターフェース）であるか、接続のローカルIPアドレスと一致する。

    • 3. 条件に合致した最初のリスナーの key メンバを返します。 4. リストの末尾まで一致するものがない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/netware/mod_nw_ssl.c:1207 にて、リクエストの接続情報から鍵を取得するために呼び出されています。
  • **呼び出し先**: 外部関数は呼び出していません。

関連データ構造

  • conn_rec: Apache の接続構造体。
  • seclistenup_rec: アップグレード可能なセキュアリスナーの定義を保持するリンクリスト構造体。

失敗モードと境界条件

  • ap_seclistenersup が空の場合、常に NULL を返します。
  • 接続のローカルIPやポートがリスナー設定と一致しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は ap_seclistenersup リストを線形探索するため、リスナー数が多い場合はパフォーマンスに影響を与える可能性があります。
  • 比較対象の addr は文字列比較（strcmp）で行われているため、IPアドレスの正規化状態に依存します。
