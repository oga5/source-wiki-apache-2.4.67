get_cert

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:251

目的

ACME 注文に関連付けられた証明書を、ACME サーバーから取得するために使用されます。

シグネチャまたは形状

static apr_status_t get_cert(void *baton, int attempt)

入力と出力

  • **入力**:
  • baton: md_proto_driver_t 構造体へのポインタ（md_acme_driver_t を保持）。
  • attempt: 試行回数（本関数内では使用されません）。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • baton は md_proto_driver_t 型としてキャスト可能であること。
  • d->baton が指す md_acme_driver_t 内の order->certificate が有効な URL を保持していること。

アルゴリズムまたは振る舞い

1. ad->order->certificate に格納された URL を対象に、証明書取得のログを出力します。 2. md_acme_GET を呼び出し、ACME サーバーに対して証明書の取得リクエストを発行します。 3. 取得した証明書データは、コールバック関数 on_add_cert を通じて処理されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_drive.c 内の直接呼び出し。
  • md_util_try を介した呼び出し（modules/md/md_acme_drive.c:276）。
  • ※注: test/ ディレクトリ内の Python スクリプトに同名の関数が存在しますが、これらはテスト用の別実装であり、本関数を直接呼び出しているわけではありません。
  • **呼び出し先**:
  • md_log_perror
  • md_acme_GET
  • on_add_cert (コールバック)

関連データ構造

  • md_proto_driver_t: ドライバーのコンテキスト。
  • md_acme_driver_t: ACME 固有のドライバー状態。
  • md_acme_order_t: 証明書注文情報（ad->order）。

失敗モードと境界条件

  • md_acme_GET が失敗した場合、そのエラーコードがそのまま返されます。
  • ad->order->certificate が空または無効な場合、ACME サーバーとの通信でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は md_util_try によるリトライ処理の対象となることを想定しています。
  • 証明書の取得成功後の処理は on_add_cert に依存しているため、データ保存の不整合が発生した場合は on_add_cert の実装を確認してください。
