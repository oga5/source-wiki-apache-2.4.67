make_fake_req

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_uwsgi.c:263

目的

バックエンドからのレスポンスを解析するために、独立したメモリプールを持つ一時的な request_rec 構造体を生成する。

シグネチャまたは形状

static request_rec *make_fake_req(conn_rec *c, request_rec *r)

入力と出力

  • **入力**:
  • c: バックエンドとの接続を表す conn_rec。
  • r: 元のリクエストを表す request_rec。
  • **出力**:
  • 初期化済みの request_rec 構造体へのポインタ。

前提条件と不変条件

  • 生成される request_rec は、c->pool を親とする独立したメモリプールを持つ。
  • rp->status はデフォルトで HTTP_OK に設定される。
  • headers_in, subprocess_env, headers_out, err_headers_out, notes の各テーブルが初期化される。

アルゴリズムまたは振る舞い

1. c->pool を親として新しいメモリプールを作成し、タグ付けを行う。 2. apr_pcalloc を使用して request_rec 構造体を確保。 3. 元のリクエスト r からサーバー設定、ログ、プロキシ設定、リクエスト時刻をコピー。 4. 接続 c からフィルタチェーン、クライアント IP、クライアントアドレスを割り当て。 5. ap_create_request_config でリクエスト設定を初期化。 6. proxy_run_create_req を呼び出し、プロキシ関連のフックを実行して構造体を完成させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_http.c:1088
  • modules/proxy/mod_proxy_uwsgi.c:322
  • **呼び出し先**:
  • apr_pool_create, apr_pool_tag, apr_pcalloc, apr_table_make (APR API)
  • ap_create_request_config (Apache Core)
  • proxy_run_create_req (mod_proxy フック)

関連データ構造

  • request_rec: Apache のリクエスト処理における中心的な構造体。
  • conn_rec: 接続状態を保持する構造体。
  • apr_pool_t: メモリ管理のためのプール構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pool_create や apr_pcalloc がエラーを発生させる可能性がある。
  • proxy_run_create_req フックが登録されていない場合や失敗した場合の挙動は、フックの実装に依存する。

保守担当者向けメモ

  • この関数で生成される request_rec は、バックエンドのレスポンス解析専用であり、独立したプールを持つ。リクエスト終了時に自動的に解放される。
  • proxy_run_create_req を呼び出しているため、プロキシ関連の拡張モジュールがこの一時リクエストに対して何らかの処理を行う可能性がある点に注意すること。
