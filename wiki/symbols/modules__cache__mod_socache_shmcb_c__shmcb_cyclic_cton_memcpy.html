shmcb_cyclic_cton_memcpy

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_shmcb.c:216

目的

循環バッファ（Cyclic Buffer）から線形メモリ領域へデータをコピーする。バッファの末尾でラップアラウンドが発生する場合、分割された2つのセグメントを適切に結合してコピーする。

シグネチャまたは形状

static void shmcb_cyclic_cton_memcpy(unsigned int buf_size, unsigned char *dest,
                                     const unsigned char *data, unsigned int src_offset,
                                     unsigned int src_len)

入力と出力

  • buf_size: 循環バッファ全体のサイズ。
  • dest: コピー先のメモリ領域（ポインタ）。
  • data: 循環バッファの開始位置（ベースポインタ）。
  • src_offset: 循環バッファ内での読み取り開始オフセット。
  • src_len: コピーするデータの総バイト数。
  • 出力: なし（dest が指すメモリ領域にデータが書き込まれる）。

前提条件と不変条件

  • data は循環バッファの先頭を指していること。
  • src_offset + src_len が buf_size を超える場合、バッファの末尾から先頭へ折り返してコピーが行われる。

アルゴリズムまたは振る舞い

1. src_offset + src_len < buf_size の場合、ラップアラウンドが発生しないため、data + src_offset から src_len 分を dest へ一度にコピーする。 2. それ以外の場合、以下の2段階でコピーを実行する：

  • 第1段階: data + src_offset からバッファ末尾までの残りの領域を dest へコピー。
  • 第2段階: バッファの先頭（data）から、残りの必要バイト数を dest の後続領域へコピー。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/cache/mod_socache_shmcb.c:909
  • modules/cache/mod_socache_shmcb.c:1022
  • modules/cache/mod_socache_shmcb.c:1027
  • 呼び出し先:
  • memcpy (標準ライブラリ)

関連データ構造

  • SHMCBHeader: 循環バッファのサイズ（subcache_data_size）を保持する構造体。

失敗モードと境界条件

  • src_offset や src_len がバッファサイズに対して不適切な値である場合、メモリ破壊の可能性がある。
  • ソース内のコメントによれば、サイズ計算に unsigned int を使用しているため、非常に大きなキャッシュサイズを指定した際にオーバーフローのリスクが存在する。

保守担当者向けメモ

  • 本関数は mod_socache_shmcb.c 内の循環バッファ操作における低レベルなメモリコピーユーティリティである。
  • 呼び出し側で header->subcache_data_size を buf_size として渡す設計となっている。
