status_handler

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_status.c:182

目的

Apache HTTP Server の稼働状況（サーバー状態、プロセス/スレッド統計、パフォーマンス指標）をレポートとして生成し、クライアントへ出力するメインハンドラです。

シグネチャまたは形状

static int status_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）。クエリパラメータ（refresh, notable, auto）を解析して出力形式を制御します。
  • **出力**: int（Apache のリクエスト処理結果ステータス）。成功時は OK、ハンドラ対象外の場合は DECLINED、エラー時は HTTP_INTERNAL_SERVER_ERROR を返します。

前提条件と不変条件

  • スコアボード（scoreboard.h）が初期化されている必要があります。
  • inetd モードで動作している場合、スコアボードが利用できないためエラーとなります。
  • リクエストメソッドは M_GET に限定されます。

アルゴリズムまたは振る舞い

1. **ハンドラ検証**: リクエストハンドラが server-status であるか確認します。 2. **クエリ解析**: r->args を走査し、status_options に基づいて出力形式（HTML またはプレーンテキスト）やリフレッシュ間隔を設定します。 3. **スコアボード走査**: server_limit および thread_limit に基づき、全ワーカーのスコアをループで取得します。 4. **統計集計**: 各ワーカーのステータス（SERVER_READY, SERVER_GRACEFUL 等）をカウントし、ExtendedStatus が有効な場合はアクセス数、バイト数、CPU 時間を累積します。 5. **レポート生成**: サーバーの稼働時間、負荷平均、MPM 情報、および集計した統計情報を ap_rprintf 等を用いてストリーム出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のハンドラフック機構（ap_hook_handler により登録）。
  • **呼び出し先**:
  • ap_mpm_query, ap_get_scoreboard_global, ap_get_scoreboard_process, ap_copy_scoreboard_worker（スコアボード関連）
  • ap_rprintf, ap_rvputs, ap_rputs（出力関連）
  • show_time（稼働時間の整形）
  • ap_get_loadavg（負荷情報の取得）

関連データ構造

  • request_rec: リクエスト処理のコンテキスト。
  • global_score, worker_score, process_score: スコアボード内のサーバー状態保持構造体。
  • status_options: クエリパラメータの定義テーブル。

失敗モードと境界条件

  • **スコアボード未初期化**: ap_exists_scoreboard_image() が偽の場合、HTTP_INTERNAL_SERVER_ERROR を返します。
  • **メソッド不一致**: GET 以外のメソッドが指定された場合、DECLINED を返します。
  • **環境依存**: HAVE_TIMES が定義されていない環境では、CPU 使用率の統計は出力されません。

保守担当者向けメモ

  • レポート生成にはリクエストプール (r->pool) を使用しており、メモリは自動的に解放されます。
  • 大規模なサーバーで ExtendedStatus が有効な場合、全スレッドの走査によるオーバーヘッドが発生するため注意してください。
  • 外部モジュールがレポートに情報を追加するためのフック（ap_run_status_hook）が、この関数の後半で呼び出される設計になっています。
