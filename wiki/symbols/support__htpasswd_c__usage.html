usage

定義

  • 種別: function
  • 定義箇所: support/htpasswd.c:98

目的

htpasswd コマンドの利用方法（コマンドライン引数および各オプションの意味）を標準エラー出力に表示し、プログラムを終了させる。

シグネチャまたは形状

static void usage(void)

入力と出力

  • **入力**: なし（引数を取らない）。
  • **出力**: errfile に対してコマンドの構文説明およびオプション一覧を出力する。戻り値はない。

前提条件と不変条件

  • errfile が有効なファイルストリームとして初期化されていること。
  • 本関数が呼び出されると、プログラムは ERR_SYNTAX ステータスコードで即座に終了する。

アルゴリズムまたは振る舞い

1. apr_file_printf を使用し、htpasswd の各操作モード（ファイル作成、ユーザー削除、パスワード検証等）およびオプションフラグの説明文を整形して出力する。 2. bcrypt のコスト値（BCRYPT_DEFAULT_COST）を動的に埋め込む。 3. exit(ERR_SYNTAX) を呼び出し、プロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htpasswd.c 内の引数解析処理（check_args 等）において、不正な引数が指定された際に呼び出される。
  • **呼び出し先**: apr_file_printf（APR ライブラリ関数）、exit（標準ライブラリ関数）。
  • *注記*: ツールによる静的解析では、本関数を直接呼び出している具体的な箇所は特定されていません。

関連データ構造

  • BCRYPT_DEFAULT_COST: bcrypt アルゴリズムのデフォルトコスト値。
  • ERR_SYNTAX: 終了時に返されるエラーコード。

失敗モードと境界条件

  • 本関数自体はエラーを返さず、常にプログラムを終了させる。
  • errfile への書き込みが失敗した場合の挙動は、APR の apr_file_printf の実装に依存する。

保守担当者向けメモ

  • 新しいコマンドラインオプションを追加した際は、必ず本関数の出力メッセージにも説明を追記すること。
  • BCRYPT_DEFAULT_COST などの定数変更が正しく反映されるよう、フォーマット文字列の整合性に注意すること。
