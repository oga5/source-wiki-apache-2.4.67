ap_ssl_var_lookup

定義

  • 種別: function
  • 定義箇所: server/ssl.c:186

目的

SSL/TLS 関連の環境変数（証明書情報やプロトコル情報など）を検索し、取得するための公開 API です。

シグネチャまたは形状

const char *ap_ssl_var_lookup(apr_pool_t *p, server_rec *s,
                              conn_rec *c, request_rec *r,
                              const char *name)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • s: サーバー設定構造体 server_rec。
  • c: 接続構造体 conn_rec。
  • r: リクエスト構造体 request_rec（NULL の場合あり）。
  • name: 取得したい変数の名称（文字列）。
  • **出力**:
  • 検索結果の文字列（const char *）。該当する変数が見つからない場合は NULL を返します。

前提条件と不変条件

  • 戻り値の文字列は、引数として渡された apr_pool_t *p の生存期間に依存します。
  • 内部的に ap_run_ssl_var_lookup フックを優先的に使用します。

アルゴリズムまたは振る舞い

1. ap_run_ssl_var_lookup を呼び出し、登録された SSL フックによる検索を試みます。 2. フックの結果が NULL であり、かつ古いオプション関数 module_ssl_var_lookup が定義されている場合、その関数を呼び出してフォールバックします。 3. 最終的な検索結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_deflate.c
  • modules/http2/h2_protocol.c
  • modules/lua/mod_lua.c
  • modules/mappers/mod_rewrite.c
  • modules/metadata/mod_headers.c
  • modules/proxy/mod_proxy.c
  • **呼び出し先**:
  • ap_run_ssl_var_lookup (フック)
  • module_ssl_var_lookup (オプション関数)

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • server_rec, conn_rec, request_rec: Apache HTTP Server の主要なコンテキスト構造体。

失敗モードと境界条件

  • 指定された変数名 name が存在しない場合、またはどのプロバイダも値を解決できない場合、NULL を返します。
  • r (リクエスト) が NULL の場合、リクエストに依存しない接続レベルの変数のみが解決対象となります。

保守担当者向けメモ

  • 本関数は SSL 抽象化レイヤーの一部であり、実際のロジックはフックまたはオプション関数に委譲されます。
  • module_ssl_var_lookup はレガシーな互換性のために保持されており、新規の実装ではフック機構を利用することが推奨されます。
