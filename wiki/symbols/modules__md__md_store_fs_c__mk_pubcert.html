mk_pubcert

定義

  • 種別: static function
  • 定義箇所: modules/md/md_store_fs.c:150

目的

指定されたディレクトリ内の証明書ファイルとチェーンファイルを統合し、公開証明書ファイル（pubcert.pem）を生成・保存します。

シグネチャまたは形状

static apr_status_t mk_pubcert(void *baton, apr_pool_t *p, apr_pool_t *ptemp, 
                               const char *dir, const char *name, 
                               apr_filetype_e ftype)

入力と出力

  • **入力**:
  • dir: 対象のディレクトリパス。
  • name: 証明書ファイル名。
  • ptemp: 一時的なメモリ割り当て用プール。
  • **出力**:
  • 戻り値: 成功時に APR_SUCCESS、失敗時にエラーコードを返します。
  • 副作用: dir 内に pubcert.pem が作成または更新されます。

前提条件と不変条件

  • baton、ftype、p は本関数内では使用されません。
  • dir 内に pubcert.pem が存在しない場合にのみ処理が進行します。
  • 証明書（name）およびチェーンファイル（chain.pem）の読み込みが成功する必要があります。

アルゴリズムまたは振る舞い

1. pubcert.pem が既に存在するか確認し、存在しない場合のみ処理を継続します。 2. 指定された name の証明書を読み込みます。 3. chain.pem を読み込みます。ファイルが存在しない場合は空の配列として扱います。 4. 証明書とチェーンを結合し、新しい配列を作成します。 5. 結合した証明書チェーンを MD_FPROT_F_UONLY（ユーザーのみ読み書き可）の権限で pubcert.pem として保存します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_store_fs.c 内の md_util_files_do から呼び出されます（195行目および197行目）。
  • **呼び出し先**:
  • md_util_path_merge: パス生成。
  • md_chain_fload / md_chain_fsave: 証明書チェーンの読み書き。
  • md_cert_fload: 証明書の読み込み。
  • apr_array_make / apr_array_cat / APR_ARRAY_PUSH: 配列操作。

関連データ構造

  • md_cert_t: 証明書オブジェクト。
  • apr_array_header_t: 証明書チェーンを保持する配列。

失敗モードと境界条件

  • pubcert.pem が既に存在する場合、関数は何もせず APR_SUCCESS を返します。
  • 証明書やチェーンファイルの読み込みに失敗した場合、そのエラーが返されます。
  • chain.pem が欠落している場合は、証明書単体のみでチェーンを構成します。

保守担当者向けメモ

  • 本関数は md_util_files_do を介してファイルシステムを走査する際のコールバックとして設計されています。
  • ptemp を使用しているため、メモリは呼び出し元で適切に管理されます。
  • MD_FN_PUBCERT 定数で定義されたファイル名がターゲットとなります。
