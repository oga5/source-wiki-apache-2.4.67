authz_dbm_getfns

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbm.c:329

目的

mod_authz_owner モジュールが提供するオプション関数 authz_owner_get_file_group を取得し、ローカルの静的変数に格納することで、dbm-file-group 認可機能を利用可能にする。

シグネチャまたは形状

static void authz_dbm_getfns(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな関数ポインタ変数を更新）

前提条件と不変条件

  • APR_RETRIEVE_OPTIONAL_FN マクロが authz_owner_get_file_group を解決できること。
  • 本関数は、Apache のオプション関数取得フック（ap_hook_optional_fn_retrieve）から呼び出されることを前提としている。

アルゴリズムまたは振る舞い

1. APR_RETRIEVE_OPTIONAL_FN を使用して、authz_owner_get_file_group 関数ポインタを検索する。 2. 取得した関数ポインタを、モジュール内の対応する変数に代入する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/aaa/mod_authz_dbm.c:344 において、ap_hook_optional_fn_retrieve に登録されている。
  • 呼び出し先: APR_RETRIEVE_OPTIONAL_FN マクロ（外部モジュール mod_authz_owner の関数を解決）。

関連データ構造

  • authz_owner_get_file_group: mod_authz_owner.h で定義されるオプション関数。

失敗モードと境界条件

  • mod_authz_owner がロードされていない場合、APR_RETRIEVE_OPTIONAL_FN は NULL を返す。この場合、dbm-file-group 認可機能は利用できない状態となる。

保守担当者向けメモ

  • 本関数は mod_authz_dbm が mod_authz_owner に依存する機能を初期化するためのブリッジである。
  • 依存関係の解決は Apache のフックフェーズで行われるため、モジュールのロード順序に依存しない設計となっている。
