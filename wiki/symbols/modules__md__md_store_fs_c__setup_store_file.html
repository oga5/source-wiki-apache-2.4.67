setup_store_file

定義

  • 種別: function
  • 定義箇所: modules/md/md_store_fs.c:256

目的

ファイルシステムベースのストレージにおいて、メタデータファイル（md_store.json）の存在確認、読み込み、および初期化を行う。

シグネチャまたは形状

static apr_status_t setup_store_file(void *baton, apr_pool_t *p, apr_pool_t *ptemp, va_list ap)

入力と出力

  • baton: md_store_fs_t 構造体へのポインタ。
  • p: メモリ割り当て用プール。
  • ptemp: 一時的なメモリ割り当て用プール。
  • ap: 使用されない可変長引数リスト。
  • 戻り値: APR_SUCCESS（成功時）、またはエラーを示す apr_status_t 値。

前提条件と不変条件

  • s_fs->base に有効なベースディレクトリパスが設定されていること。
  • MD_SG_DOMAINS、MD_SG_CHALLENGES、MD_SG_TMP グループの秘密鍵が暗号化対象外（平文）として設定される。

アルゴリズムまたは振る舞い

1. s_fs->plain_pkey 配列を更新し、特定のグループを暗号化対象外としてマークする。 2. md_util_path_merge を使用して、ベースディレクトリ内の md_store.json へのパスを生成する。 3. md_util_is_file でファイルが存在するか確認する。

  • 存在する場合: read_store_file を呼び出し、内容を読み込む。読み込み失敗時はエラーログを出力する。
  • 存在しない場合: init_store_file を呼び出し、新規作成を試みる。作成成功時は再度 read_store_file を実行する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_store_fs.c:353 (md_util_pool_vdo 経由)
  • 呼び出し先: read_store_file, init_store_file, md_util_path_merge, md_util_is_file, md_log_perror

関連データ構造

  • md_store_fs_t: ストレージの状態を保持するメイン構造体。

失敗モードと境界条件

  • md_store.json が存在しても JSON として解析できない場合、エラーログが出力され、読み込みに失敗する。
  • init_store_file が失敗した場合、そのエラーコードがそのまま返される。

保守担当者向けメモ

  • 本関数は md_util_pool_vdo を介して呼び出されるため、va_list 引数は無視される設計となっている。
  • md_store.json の破損や設定不整合が発生した場合、ディレクトリのクリーンアップが必要となる旨がログに記録される。
