run_watchdog

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_drive.c:254

目的

mod_watchdog のコールバックとして動作し、マネージドドメイン（MD）の証明書更新ジョブを定期的に監視・実行する。

シグネチャまたは形状

static apr_status_t run_watchdog(int state, void *baton, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • state: mod_watchdog の現在の状態（AP_WATCHDOG_STATE_STARTING, RUNNING, STOPPING）。
  • baton: md_renew_ctx_t 型のコンテキスト構造体へのポインタ。
  • ptemp: 一時的なメモリ割り当てに使用するプール。
  • **出力**:
  • apr_status_t: 常に APR_SUCCESS を返す。

前提条件と不変条件

  • baton は有効な md_renew_ctx_t を指している必要がある。
  • mod_watchdog により、サーバー内で単一スレッドとして実行されることが保証されている。

アルゴリズムまたは振る舞い

1. **状態遷移**:

  • AP_WATCHDOG_STATE_STARTING/STOPPING: ログ出力のみを行う。
  • AP_WATCHDOG_STATE_RUNNING: 以下のループを実行する。

    • 2. **ジョブ処理**:

  • next_run_default を用いて次回のデフォルト実行時刻を算出。
  • dctx->jobs に含まれる全ジョブを走査し、現在時刻が job->next_run を過ぎている場合に process_drive_job を呼び出して更新処理を実行。
  • 各ジョブの next_run を比較し、最も早い時刻を次回の実行タイミングとして保持。

    • 3. **再スケジュール**:

  • wd_set_interval を呼び出し、算出した wait_time 後に再度 run_watchdog が実行されるようスケジュールする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_watchdog（コールバックとして）
  • modules/md/mod_md_drive.c:406 (md_renew_start_watching)
  • modules/md/mod_md_ocsp.c:275 (OCSP監視用)
  • **呼び出し先**:
  • process_drive_job
  • next_run_default
  • wd_set_interval
  • ap_log_error

関連データ構造

  • md_renew_ctx_t: 更新プロセス全体のコンテキスト。
  • md_job_t: 個別の更新ジョブの状態とスケジュール情報。

失敗モードと境界条件

  • job->next_run が 0 の場合、そのジョブは通常の定期実行対象として扱われる。
  • wait_time が負の値になる可能性があるかについては、next_run の算出ロジックに依存する。

保守担当者向けメモ

  • 本関数は mod_md_drive.c と mod_md_ocsp.c の両方からコールバックとして登録されているため、修正時は両方のコンテキストへの影響を考慮すること。
  • mod_watchdog の特性上、スレッドの生存期間中は同一プロセスで繰り返し呼び出される。
