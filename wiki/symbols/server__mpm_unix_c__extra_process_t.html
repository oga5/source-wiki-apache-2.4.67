extra_process_t

定義

  • 種別: struct
  • 定義箇所: server/mpm_unix.c:74

目的

MPM（Multi-Processing Module）において、メインのプロセス管理対象外である「追加のプロセス」を追跡するためのリスト構造体です。

シグネチャまたは形状

typedef struct extra_process_t {
    struct extra_process_t *next;
    pid_t pid;
    ap_generation_t gen;
} extra_process_t;

入力と出力

  • next: リスト内の次の extra_process_t 構造体へのポインタ。
  • pid: 追跡対象となるプロセスのプロセスID。
  • gen: プロセスの世代（generation）番号。

前提条件と不変条件

  • 本構造体は ap_malloc を使用して動的に確保されます。
  • ap_unregister_extra_mpm_process を呼び出すことで、明示的にメモリを解放する必要があります。
  • WIN32 環境では本構造体は使用されません。

アルゴリズムまたは振る舞い

  • 単方向連結リストとして構成され、extras という静的変数を通じて管理されます。
  • プロセス管理のライフサイクルにおいて、特定のプロセスを追跡し、終了処理や世代管理を行うためのノードとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm_unix.c 内の ap_register_extra_mpm_process 等のプロセス管理関数が本構造体を生成・操作します。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • ap_generation_t: プロセスの世代を識別するための型。
  • extras: server/mpm_unix.c 内で定義される、本構造体のリストを保持する静的変数。

失敗モードと境界条件

  • メモリ確保失敗: ap_malloc が失敗した場合の挙動は、呼び出し側の実装に依存します。
  • リスト操作: リストの走査中に next ポインタが正しく更新されない場合、メモリリークや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は server/mpm_unix.c におけるプロセス管理の「メカニズム」の一部です。
  • 世代管理（gen）は、グレースフル再起動時などのプロセス入れ替えを制御するために重要です。
  • ap_register_extra_mpm_process で確保したメモリは、必ず対応する ap_unregister_extra_mpm_process で解放されることを確認してください。
