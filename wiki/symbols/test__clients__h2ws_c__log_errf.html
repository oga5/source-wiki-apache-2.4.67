log_errf

定義

  • 種別: function
  • 定義箇所: test/clients/h2ws.c:95

目的

フォーマット指定子付きの文字列をエラーログとして出力します。

シグネチャまたは形状

static void log_errf(const char *where, const char *msg, ...);
  • __attribute__((format(printf, 2, 3))) により、コンパイラによるフォーマット文字列の型チェックが行われます。

入力と出力

  • where: ログの発生箇所を示す識別文字列。
  • msg: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • 出力: 戻り値はありません。内部で構築した文字列を log_err へ渡します。

前提条件と不変条件

  • msg 引数は有効な printf フォーマット文字列である必要があります。
  • 内部バッファサイズは 8KB に固定されています。

アルゴリズムまたは振る舞い

1. 可変長引数リスト ap を初期化します。 2. vsnprintf を使用して、最大 8,192 バイトのバッファにフォーマット済み文字列を書き込みます。 3. va_end で引数リストを終了します。 4. 生成された文字列を log_err(where, buffer) に渡し、最終的なログ出力を行います。

呼び出し元と呼び出し先

  • 呼び出し元: h2ws.c 内の各エラー発生箇所（sock_nonblock_nodelay, h2_session_send, h2_session_recv, on header recv, stream close, terminate session, client new 等）。
  • 呼び出し先: log_err（ログ出力の実行）。

関連データ構造

  • va_list: 可変長引数を処理するための標準型。

失敗モードと境界条件

  • フォーマット後の文字列が 8,192 バイトを超える場合、vsnprintf の仕様に従い切り詰められます。

保守担当者向けメモ

  • 本関数は log_err のラッパーであり、書式化の責務を担います。ログ出力のフォーマットを変更する場合は log_err 側の実装も併せて確認してください。
