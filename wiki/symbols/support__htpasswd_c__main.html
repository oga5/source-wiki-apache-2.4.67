main

定義

  • 種別: function
  • 定義箇所: support/htpasswd.c:283

目的

Apache HTTP Server のパスワードファイル（htpasswd 形式）の管理を行うコマンドラインユーティリティのメインエントリポイントです。ユーザーの追加、更新、削除、およびパスワード検証のオーケストレーションを行います。

シグネチャまたは形状

int main(int argc, const char * const argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。
  • **出力**:
  • 戻り値: 成功時に 0、エラー時に ERR_FILEPERM, ERR_INVALID, ERR_BADUSER などのエラーコードを返却。
  • 副作用: パスワードファイルの更新、標準出力への結果表示、またはエラーメッセージの標準エラー出力への書き出し。

前提条件と不変条件

  • apr_app_initialize により APR 環境が初期化されていること。
  • すべての動的メモリは apr_pool_t によって管理され、終了時に自動解放されること。
  • ファイル操作は一時ファイル (htpasswd.tmp.XXXXXX) を経由してアトミックに行われること。
  • ユーザー名に : を含めることは不可。

アルゴリズムまたは振る舞い

1. **初期化**: APR プールの作成と passwd_ctx の初期化。 2. **引数解析**: check_args を呼び出し、操作モード（作成、削除、検証等）を決定。 3. **アクセス権限確認**: exists および accessible を使用し、対象ファイルに対する読み書き権限を検証。 4. **レコード生成**: mkrecord を呼び出し、ユーザー名とハッシュ化されたパスワードを生成。 5. **ファイル更新処理**:

  • 既存ファイルを一行ずつ読み込み、対象ユーザー以外を一時ファイルへコピー。
  • 対象ユーザーが見つかった場合、指定された操作（削除・更新・検証）を実行。
  • 最後に apr_file_copy を用いて一時ファイルを元のファイルへ上書き。

    • 6. **終了**: atexit で登録された terminate 関数によりリソースを解放。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（コマンド実行時）。
  • **呼び出し先**:
  • check_args, mkrecord, verify, exists, accessible, terminate (本ファイル内の関数)
  • apr_* 系関数 (APR ライブラリ)
  • printf, strcpy, strchr, strcmp, strcspn (標準 C ライブラリ)

関連データ構造

  • passwd_ctx: 認証処理の状態やハッシュアルゴリズムを保持する構造体。
  • apr_pool_t: メモリ管理用プール。
  • apr_file_t: ファイルハンドル。

失敗モードと境界条件

  • **権限エラー**: ファイルの読み書き権限がない場合、ERR_FILEPERM で終了。
  • **バリデーションエラー**: 不正なファイル形式や、MAX_STRING_LEN を超えるレコードは ERR_INVALID または ERR_OVERFLOW で終了。
  • **ユーザー不在**: 削除や検証時にユーザーが見つからない場合、適切なエラーメッセージを表示して終了。
  • **セキュリティ**: 本プログラムは setuid 実行を想定していないため、権限昇格の脆弱性に注意が必要。

保守担当者向けメモ

  • main 関数はプログラムのライフサイクル全体を管理しており、ファイル操作の原子性を維持するために一時ファイルを使用しています。
  • 認証ロジックの詳細は support/passwd_common.c に依存しています。
  • APR_CHARSET_EBCDIC 環境下では、ハッシュ生成前に文字コード変換処理が実行されます。
