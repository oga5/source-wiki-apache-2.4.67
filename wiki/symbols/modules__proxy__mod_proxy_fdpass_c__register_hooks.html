register_hooks

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fdpass.c:226

目的

mod_proxy_fdpass モジュールを Apache HTTP Server のフックシステムおよびプロバイダレジストリに登録する。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p (モジュールのライフサイクルを管理するメモリプール)
  • **出力**: なし (void)

前提条件と不変条件

  • Apache のモジュールロードプロセスにおいて、他のフック登録関数と同様に呼び出されることを前提とする。
  • PROXY_FDPASS_FLUSHER プロバイダおよび proxy_fdpass 関連のハンドラが正しく初期化される。

アルゴリズムまたは振る舞い

1. ap_register_provider を呼び出し、builtin_flush 関数を PROXY_FDPASS_FLUSHER プロバイダとして登録する。 2. proxy_hook_scheme_handler を呼び出し、proxy_fdpass_handler を fd:// スキームのハンドラとして登録する。優先度は APR_HOOK_FIRST。 3. proxy_hook_canon_handler を呼び出し、proxy_fdpass_canon を URL 正規化ハンドラとして登録する。優先度は APR_HOOK_FIRST。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール初期化プロセス（直接的な呼び出し元はソースコード上では明示されていないが、Apache のモジュール構造に従いサーバー起動時に実行される）。
  • **呼び出し先**:
  • ap_register_provider
  • proxy_hook_scheme_handler
  • proxy_hook_canon_handler

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • builtin_flush: 登録されるフラッシュプロバイダ関数。
  • proxy_fdpass_handler: 登録されるスキームハンドラ関数。
  • proxy_fdpass_canon: 登録される正規化ハンドラ関数。

失敗モードと境界条件

  • 本関数自体はエラーを返さないが、登録先のフックやプロバイダレジストリが利用不可である場合、モジュールの機能が正しく有効化されない可能性がある。

保守担当者向けメモ

  • 本関数は mod_proxy_fdpass のエントリポイントである。新しいハンドラやプロバイダを追加する場合は、この関数内で登録を行う必要がある。
  • APR_HOOK_FIRST を指定しているため、他のプロキシモジュールよりも優先的に処理が実行される点に注意すること。
