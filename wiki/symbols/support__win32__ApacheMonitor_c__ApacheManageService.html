ApacheManageService

定義

  • 種別: function
  • 定義箇所: support/win32/ApacheMonitor.c:540

目的

Windows Service Control Manager (SCM) を介して、指定された Apache サービスの起動、停止、または再起動を実行し、その進捗を GUI 上のリストボックスに通知する。

シグネチャまたは形状

BOOL ApacheManageService(LPCTSTR szServiceName, LPCTSTR szImagePath, LPTSTR szComputerName, DWORD dwCommand)

入力と出力

  • **入力**:
  • szServiceName: 制御対象のサービス名。
  • szImagePath: サービスの実行パス（権限昇格時の再起動に使用）。
  • szComputerName: 接続先コンピュータ名（ローカルまたはリモート）。
  • dwCommand: 実行するコマンド（SERVICE_CONTROL_STOP, SERVICE_CONTROL_CONTINUE, SERVICE_APACHE_RESTART）。
  • **出力**:
  • 成功時に TRUE、失敗時に FALSE を返す。

前提条件と不変条件

  • サービス制御には適切な権限が必要。権限不足時は am_RespawnAsUserAdmin を介して管理者権限での再起動を試みる。
  • 処理中はグローバル変数 g_bConsoleRun が TRUE に設定され、カーソルが砂時計（g_hCursorHourglass）に変更される。
  • 処理終了時には必ず CloseServiceHandle によりハンドルが解放され、カーソルが元に戻る。

アルゴリズムまたは振る舞い

1. OpenSCManager で SCM に接続し、OpenService で対象サービスを開く。 2. サービスが開けない場合、szImagePath が存在すれば am_RespawnAsUserAdmin を呼び出し、存在しなければエラーを表示して終了する。 3. dwCommand に応じて以下の処理を行う：

  • **STOP**: ControlService で停止を指示し、SERVICE_STOP_PENDING 状態の間、1秒間隔でポーリングを行う。
  • **CONTINUE**: StartService で起動を指示し、SERVICE_START_PENDING 状態の間、1秒間隔でポーリングを行う。
  • **RESTART**: ControlService で再起動を指示し、最大60秒間、状態遷移を監視する。

    • 4. 各操作の開始・終了時に、addListBoxString を使用して GUI のリストボックスへ進捗メッセージを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/win32/ApacheMonitor.c 内の複数箇所（ダイアログ操作時やコマンドライン引数処理時）。
  • **呼び出し先**:
  • Windows API: OpenSCManager, OpenService, ControlService, StartService, QueryServiceStatus, CloseServiceHandle。
  • 内部関数: ErrorMessage, am_RespawnAsUserAdmin, addListBoxString。

関連データ構造

  • SERVICE_STATUS: サービスの状態を保持する Windows 構造体。
  • g_lpMsg: メッセージ文字列を保持するグローバル配列。

失敗モードと境界条件

  • SCM への接続失敗やサービスオープン失敗時は ErrorMessage を呼び出し FALSE を返す。
  • サービスの状態遷移がタイムアウト（再起動時の60秒制限など）した場合、操作は失敗とみなされる。
  • 権限昇格が必要な状況で szImagePath が NULL の場合、再起動は行われずエラーとなる。

保守担当者向けメモ

  • サービス状態のポーリングに Sleep(1000) を使用しているため、UI スレッドで実行する場合は注意が必要。
  • MAX_PATH を超えるサービス名やパスの取り扱いには制限がある。
  • 終了処理における g_bConsoleRun のリセット漏れがないよう注意すること。
