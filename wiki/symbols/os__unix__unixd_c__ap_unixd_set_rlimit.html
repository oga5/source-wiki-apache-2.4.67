ap_unixd_set_rlimit

定義

  • 種別: function
  • 定義箇所: os/unix/unixd.c:59

目的

Apache HTTP Server の設定ファイル（httpd.conf 等）で指定されたリソース制限（rlimit）を解析し、プロセスごとの制限値として設定・保持する。

シグネチャまたは形状

void ap_unixd_set_rlimit(cmd_parms *cmd, struct rlimit **plimit,
                         const char *arg, const char *arg2, int type)

入力と出力

  • cmd: 設定コマンドのパラメータ構造体。
  • plimit: 設定値を格納する struct rlimit へのポインタのポインタ。
  • arg: 制限値（ソフトリミット）の文字列。
  • arg2: 制限値（ハードリミット）の文字列（オプション）。
  • type: 設定対象のリソース種別（RLIMIT_CPU, RLIMIT_AS 等）。
  • 出力: plimit を介してメモリが割り当てられ、設定値が格納される。

前提条件と不変条件

  • プラットフォームが getrlimit および struct rlimit をサポートしている必要がある。
  • cmd->pool からメモリが割り当てられるため、サーバーのライフサイクルに依存する。
  • 非 root ユーザーで実行されている場合、ハードリミット（rlim_max）を増やすことはできない。

アルゴリズムまたは振る舞い

1. getrlimit を呼び出し、現在のリソース制限を取得して *plimit に格納する。 2. arg を解析し、"max" が指定されていれば現在のハードリミット値を、そうでなければ数値としてソフトリミット（rlim_cur）を設定する。 3. arg2 が存在する場合は、それをハードリミット（rlim_max）として設定する。 4. 実行ユーザーが root でない場合、ハードリミットを現在値より大きく設定しようとするとエラーログを出力し、ハードリミットの更新をスキップする。 5. プラットフォームがリソース制限をサポートしていない場合、エラーログを出力して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: server/core.c 内の各設定ディレクティブハンドラ（RLIMIT_CPU, RLIMIT_AS, RLIMIT_DATA, RLIMIT_VMEM, RLIMIT_NPROC 等）。
  • 呼び出し先: apr_pcalloc, getrlimit, ap_getword_conf, ap_log_error, atol, strcasecmp, geteuid。

関連データ構造

  • struct rlimit: システムのリソース制限を保持する標準構造体。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • getrlimit が失敗した場合、エラーログを出力し *plimit を NULL に設定して終了する。
  • 引数が不正な場合、エラーログを出力して終了する。
  • 非 root ユーザーがハードリミットの引き上げを試みた場合、エラーログを出力し、その変更のみを無視する。
  • プラットフォームが rlimit をサポートしない環境では、設定試行時にエラーログが出力される。

保守担当者向けメモ

  • 本関数は os/unix/unixd.c で実装されており、unixd.h で宣言されている。
  • _OSD_POSIX 等の特殊環境や、rlimit をサポートしないプラットフォームでの挙動はプリプロセッサ条件分岐に依存している。
