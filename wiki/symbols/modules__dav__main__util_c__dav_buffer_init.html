dav_buffer_init

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:144

目的

dav_buffer 構造体を初期化し、指定された文字列の内容でバッファを充填します。

シグネチャまたは形状

void dav_buffer_init(apr_pool_t *p, dav_buffer *pbuf, const char *str)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t。
  • pbuf: 初期化対象の dav_buffer 構造体へのポインタ。
  • str: バッファにコピーするソース文字列。
  • 出力: なし（pbuf が指す構造体の内部状態が更新されます）。

前提条件と不変条件

  • str はヌル終端文字列である必要があります。
  • dav_buffer 構造体は、dav_set_bufsize によってメモリが確保され、cur_len が適切に設定されることを前提としています。

アルゴリズムまたは振る舞い

1. dav_set_bufsize を呼び出し、str の長さに応じて pbuf のバッファサイズを確保・設定します。 2. memcpy を使用して、str の内容（ヌル終端文字を含む）を pbuf->buf にコピーします。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • modules/dav/fs/lock.c:754
  • modules/dav/fs/repos.c:1824, 1849, 1854
  • 呼び出し先:
  • dav_set_bufsize (同ファイル内)
  • memcpy (標準ライブラリ)

関連データ構造

  • dav_buffer: バッファ本体 (buf)、現在の長さ (cur_len)、確保済みサイズ (alloc_len) を保持する構造体。

失敗モードと境界条件

  • apr_pool_t によるメモリ割り当てが失敗した場合、dav_set_bufsize 経由でプロセスが異常終了する可能性があります（APR のメモリ管理規約に依存）。
  • str が NULL の場合、strlen でセグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は dav_buffer の初期化専用です。追記が必要な場合は dav_buffer_append を使用してください。
  • dav_buffer のメモリ管理は apr_pool_t に依存しているため、呼び出し側で明示的な解放は不要です。
