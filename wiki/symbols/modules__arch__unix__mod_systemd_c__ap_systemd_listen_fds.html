ap_systemd_listen_fds

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_systemd.c:152

目的

systemd のソケットアクティベーション機能により引き継がれたファイルディスクリプタの数を取得するためのラッパー関数です。

シグネチャまたは形状

int ap_systemd_listen_fds(int unset_environment);

入力と出力

  • **入力**:
  • unset_environment (int): 0 以外を指定した場合、関数実行後に環境変数 LISTEN_FDS および LISTEN_PID をクリアします。
  • **出力**:
  • 成功時: 引き継がれたファイルディスクリプタの数（0 以上）。
  • 失敗時: 負の値を返します（sd_listen_fds の仕様に準拠）。

前提条件と不変条件

  • libsystemd がシステムに存在し、リンクされている必要があります。
  • 呼び出し元は、本関数がオプション関数として登録されていることを前提としています。

アルゴリズムまたは振る舞い

1. 外部ライブラリの sd_listen_fds(unset_environment) を呼び出します。 2. その戻り値をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/listen.c: APR_RETRIEVE_OPTIONAL_FN を介して本関数を取得し、ソケットの初期化プロセスで使用しています。
  • **呼び出し先**:
  • sd_listen_fds (systemd ライブラリ関数)

関連データ構造

  • なし（直接的なデータ構造の操作は行いません）。

失敗モードと境界条件

  • sd_listen_fds が返すエラーコード（負の値）がそのまま本関数の戻り値となります。
  • 環境変数が設定されていない場合や、systemd 管理下でない環境では 0 が返されます。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN を通じて公開されています。シグネチャを変更する場合は、server/listen.c 側の型定義と整合性を取る必要があります。
  • sd_listen_fds の詳細な挙動については、systemd の sd-daemon(3) マニュアルを参照してください。
