create_server_config

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_file_cache.c:117

目的

mod_file_cache モジュールが使用するサーバーごとの設定構造体（a_server_config）を生成し、初期化する。

シグネチャまたは形状

static void *create_server_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定情報の割り当てに使用するメモリプール。
  • s: 設定が適用されるサーバーレコード（本関数内では未使用）。
  • **出力**:
  • void *: 初期化された a_server_config 構造体へのポインタ。

前提条件と不変条件

  • Apache HTTP Server の設定フェーズにおいて、モジュール設定の初期化時に呼び出されることを前提とする。
  • apr_palloc によりメモリが確保され、apr_hash_make によりハッシュテーブルが生成される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、a_server_config 構造体分のメモリをプール p から確保する。 2. apr_hash_make を呼び出し、キャッシュ対象ファイルを管理するためのハッシュテーブル fileht を生成して構造体に格納する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のコア設定処理（server/config.c 等）から、モジュールの create_server_config フックとして呼び出される。
  • **呼び出し先**:
  • apr_palloc (APR)
  • apr_hash_make (APR)

関連データ構造

  • a_server_config: 本関数で生成される、キャッシュハッシュテーブルを保持するコンテナ構造体。
  • apr_hash_t: fileht フィールドとして使用されるハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時、apr_palloc または apr_hash_make が APR の規約に従いエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は mod_file_cache の設定ライフサイクルの起点となる。
  • server_rec *s 引数は現在の実装では使用されていないが、将来的にサーバー単位の固有設定が必要となった場合に備えてシグネチャに含まれている。
