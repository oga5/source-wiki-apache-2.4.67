remoteip_filter_context

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_remoteip.c:130

目的

remoteip_input_filter における PROXY プロトコル（v1/v2）の解析状態および受信バッファを保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    char header[sizeof(proxy_header)];
    apr_size_t rcvd;
    apr_size_t need;
    int version;
    ap_input_mode_t mode;
    apr_bucket_brigade *bb;
    int done;
} remoteip_filter_context;

入力と出力

  • **入力**: remoteip_input_filter が読み取ったストリームデータ（header バッファへ格納）。
  • **出力**: 解析完了後の接続情報（conn_config への反映）およびフィルタの完了状態（done）。

前提条件と不変条件

  • remoteip_input_filter の実行中、f->ctx に本構造体のインスタンスが保持されている必要があります。
  • rcvd は現在までに受信したバイト数、need は解析に必要な合計バイト数を表します。

アルゴリズムまたは振る舞い

1. **バッファリング**: ストリームからデータを読み取り、header に蓄積します。 2. **バージョン判定**: remoteip_determine_version を通じて PROXY プロトコルのバージョンを特定します。 3. **状態管理**: rcvd が need に達するまでデータを読み込み続け、完了時に done を設定してフィルタ処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_remoteip.c 内の remoteip_input_filter 関数が f->ctx として本構造体を参照しています。
  • **呼び出し先**: 直接的な関数呼び出しは含まれませんが、本構造体の各フィールドは remoteip_input_filter 内のロジックによって操作されます。

関連データ構造

  • proxy_header: header フィールドのサイズ定義に使用される共用体。
  • apr_bucket_brigade: フィルタ処理中にデータを一時保持するために使用されます。
  • remoteip_conn_config_t: 本構造体で解析された結果が最終的に格納される接続単位のコンテキスト。

失敗モードと境界条件

  • **データ不足**: ヘッダー全体が一度に届かない場合、rcvd < need となり、フィルタは APR_SUCCESS を返しつつ後続の呼び出しを待ちます。
  • **解析エラー**: 不正な PROXY プロトコルヘッダーが受信された場合、フィルタはエラーを返し処理を中断します。

保守担当者向けメモ

  • 本構造体はリクエスト単位ではなく、接続単位のフィルタコンテキストとして管理されます。
  • header フィールドは proxy_header 共用体のサイズに依存しているため、プロトコル仕様の変更時には注意が必要です。
