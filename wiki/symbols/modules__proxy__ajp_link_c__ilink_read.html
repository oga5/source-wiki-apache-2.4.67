ilink_read

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_link.c:49

目的

指定されたバイト数が完全に読み込まれるまで、ソケットからデータを継続的に受信する低レベルなヘルパー関数です。

シグネチャまたは形状

static apr_status_t ilink_read(apr_socket_t *sock, apr_byte_t *buf, apr_size_t len)

入力と出力

  • **入力**:
  • sock: 受信対象の apr_socket_t へのポインタ。
  • buf: 受信データを格納するバッファ。
  • len: 読み込むべきバイト数。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。ソケットが閉じられた場合は APR_EOF、その他のエラー発生時は対応するエラーコードを返します。

前提条件と不変条件

  • len で指定されたバイト数がすべて読み込まれるか、エラーが発生するまでループが継続されます。
  • APR_STATUS_IS_EAGAIN(status) が真となる場合、読み取りを再試行します。

アルゴリズムまたは振る舞い

1. rdlen（読み込み済みバイト数）が len に達するまでループを実行します。 2. apr_socket_recv を呼び出し、ソケットからデータを読み取ります。 3. APR_EOF が返された場合は即座に終了します。 4. EAGAIN が返された場合は、読み取りを継続するためにループの先頭へ戻ります。 5. その他のエラーが発生した場合は、そのステータスを返して終了します。 6. 正常に読み込めた場合、rdlen を更新し、残りの読み込み対象バイト数を計算してループを継続します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/ajp_link.c:82 (ajp_ilink_receive 内でヘッダー読み取りに使用)
  • modules/proxy/ajp_link.c:98 (ajp_ilink_receive 内でボディ読み取りに使用)
  • **呼び出し先**:
  • apr_socket_recv (APRライブラリ関数)

関連データ構造

  • apr_socket_t: APRのソケット構造体。
  • apr_byte_t: バイト単位のデータ型。

失敗モードと境界条件

  • **ソケット切断**: apr_socket_recv が APR_EOF を返した場合、関数は即座に APR_EOF を返します。
  • **読み取りエラー**: APR_SUCCESS および EAGAIN 以外のステータスが返された場合、そのエラーコードを呼び出し元に伝播します。

保守担当者向けメモ

  • 本関数はブロッキングおよびノンブロッキングの両方のソケット環境を想定し、EAGAIN を適切に処理する設計となっています。
  • AJPプロトコルの意味論には関与せず、純粋なバイトストリームの読み取りに特化しています。
