sed_err_fn_t

定義

  • 種別: typedef
  • 定義箇所: modules/filters/libsed.h:62

目的

libsed エンジン内で発生したエラーを通知するためのコールバック関数型です。

シグネチャまたは形状

typedef apr_status_t (sed_err_fn_t)(void *data, const char *error);

入力と出力

  • **入力**:
  • void *data: コールバック実行時に渡されるユーザー定義のコンテキストデータ。
  • const char *error: 発生したエラー内容を示す文字列。
  • **出力**:
  • apr_status_t: エラー処理の結果を示すステータスコード。

前提条件と不変条件

  • 本関数型は、sed のコンパイル環境（sed_commands_t）または実行環境（sed_eval_t）の初期化時に登録されます。
  • 登録された関数は、ライブラリ内部でエラーが発生した際に呼び出されます。

アルゴリズムまたは振る舞い

  • libsed のコンパイルフェーズ（sed_compile_string 等）や実行フェーズ（sed_eval_buffer 等）において、エラー発生時にこの関数が呼び出され、エラーメッセージが error 引数を通じて通知されます。

呼び出し元と呼び出し先

  • **呼び出し元**: libsed ライブラリ内部（sed0.c および sed1.c のエラー発生箇所）。
  • **呼び出し先**: ユーザーが提供する実装関数。
  • **参照箇所**:
  • sed_init_commands
  • sed_init_eval
  • sed_reset_eval

関連データ構造

  • sed_commands_t: コンパイル環境。
  • sed_eval_t: 実行環境。

失敗モードと境界条件

  • error 文字列が NULL である場合の挙動については、本ソース excerpt からは定義されていません。
  • コールバック関数内で発生したエラーが apr_status_t として返された場合、呼び出し元の libsed 関数はそのステータスを伝播させます。

保守担当者向けメモ

  • 本関数型は libsed のエラーハンドリングの抽象化層です。実装を追加する際は、apr_pool_t のライフサイクルと整合性が取れるように設計してください。
