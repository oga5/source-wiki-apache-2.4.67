h2_c1_hook_pre_close

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1.c:316

目的

HTTP/2 接続の終了直前に、セッションのクリーンアップ処理（GOAWAY 送信等）をトリガーするために呼び出されるフック関数です。

シグネチャまたは形状

static int h2_c1_hook_pre_close(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（Apache の接続レコード）
  • **出力**: int（Apache フックの戻り値。成功または処理対象外の場合は DECLINED、h2_c1_pre_close の結果を返す）

前提条件と不変条件

  • c->master が NULL であること（プライマリ接続であること）。

アルゴリズムまたは振る舞い

1. 渡された接続 c がセカンダリ接続（c->master が存在）である場合、何もせず DECLINED を返します。 2. h2_conn_ctx_get(c) を使用して接続コンテキストを取得します。 3. コンテキストが存在する場合、h2_c1_pre_close(ctx, c) を呼び出し、その結果を返します。 4. コンテキストが存在しない（セッションが既に終了している）場合は DECLINED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の pre_close_connection フック（modules/http2/h2_c1.c:351 にて ap_hook_pre_close_connection に登録）。
  • **呼び出し先**: h2_conn_ctx_get（コンテキスト取得）、h2_c1_pre_close（実際のクリーンアップ処理）。

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • h2_conn_ctx_t: HTTP/2 接続ごとのメタデータおよびセッション状態を保持する構造体。

失敗モードと境界条件

  • セカンダリ接続に対しては無視（DECLINED）する設計となっており、プライマリ接続のみを対象とします。
  • セッションが既にクリーンアップ済みでコンテキストが存在しない場合、安全に DECLINED を返します。

保守担当者向けメモ

  • 本関数は mod_ssl よりも後に実行されるよう APR_HOOK_LAST で登録されています。
  • セッションのライフサイクル管理において、接続終了時のリソース解放を確実に行うための重要な接点です。
