dav_dbm_fetch

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/dbm.c:217

目的

DBM データベースから指定されたキーに対応する値を取得し、エラーハンドリングを適用して呼び出し元へ返却します。

シグネチャまたは形状

dav_error * dav_dbm_fetch(dav_db *db, apr_datum_t key, apr_datum_t *pvalue)

入力と出力

  • **入力**:
  • db: データベース接続状態を保持する dav_db 構造体へのポインタ。
  • key: 検索対象のキー（apr_datum_t 型）。
  • pvalue: 取得した値を格納するための apr_datum_t 構造体へのポインタ。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は dav_error 構造体へのポインタを返します。

前提条件と不変条件

  • db は有効なデータベース接続を指している必要があります。
  • key.dptr が NULL の場合、キーの生成に失敗したとみなされ、pvalue はゼロクリアされ、エラーなし（APR_SUCCESS）として処理されます。

アルゴリズムまたは振る舞い

1. key.dptr が NULL かどうかを確認します。

  • NULL の場合、pvalue を memset でゼロクリアし、ステータスを APR_SUCCESS とします。
  • NULL でない場合、apr_dbm_fetch を呼び出し、db->file から key に対応する値を pvalue に取得します。

    • 2. 取得したステータスを dav_fs_dbm_error に渡し、必要に応じて dav_error 構造体を生成して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/dbm.c: 441行目、576行目、757行目
  • modules/dav/fs/lock.c: 552行目
  • **呼び出し先**:
  • apr_dbm_fetch (APR DBM API)
  • dav_fs_dbm_error (エラー変換用ヘルパー)

関連データ構造

  • dav_db: データベース接続状態を保持する構造体。
  • apr_datum_t: APR DBM で使用されるデータ保持用構造体。

失敗モードと境界条件

  • key.dptr が NULL の場合、エラーとは見なされず、空の結果として正常終了します。
  • apr_dbm_fetch がエラーを返した場合、dav_fs_dbm_error を通じて dav_error が生成されます。

保守担当者向けメモ

  • 本関数は apr_dbm_fetch のラッパーであり、主にエラーハンドリングの統一を目的としています。
  • dav_db 構造体は apr_pool_t に紐付いているため、メモリ管理は呼び出し元のプールライフサイクルに依存します。
