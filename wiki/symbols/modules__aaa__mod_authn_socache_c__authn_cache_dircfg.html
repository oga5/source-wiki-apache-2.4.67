authn_cache_dircfg

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authn_socache.c:37

目的

Apache HTTP Server の mod_authn_socache モジュールにおいて、ディレクトリ単位での認証キャッシュ設定を保持するための構造体です。

シグネチャまたは形状

typedef struct authn_cache_dircfg {
    apr_interval_time_t timeout;    /* キャッシュの有効期限 */
    apr_array_header_t *providers;  /* キャッシュ対象の認証プロバイダーリスト */
    const char *context;            /* キャッシュキー生成に使用するコンテキスト文字列 */
} authn_cache_dircfg;

入力と出力

本構造体は設定ファイル（httpd.conf 等）のディレクティブから値を読み込み、モジュール内の各関数へ設定状態を伝播させるために使用されます。

前提条件と不変条件

  • providers は apr_array_header_t 型であり、複数の認証プロバイダーを保持可能です。
  • timeout は apr_interval_time_t 型で定義され、キャッシュの生存期間を制御します。

アルゴリズムまたは振る舞い

  • authn_cache_dircfg_create によりメモリが割り当てられ、初期化されます。
  • authn_cache_dircfg_merge により、親ディレクトリの設定とマージされます。
  • 認証処理時、check_password 等の関数内でこの構造体を参照し、現在のリクエストに対するキャッシュの有効性や対象プロバイダーを判定します。

呼び出し元と呼び出し先

  • 呼び出し元: mod_authn_socache.c 内の authn_cache_dircfg_create、authn_cache_dircfg_merge、authn_cache_setprovider、authn_cache_timeout、および認証処理関数（check_password 等）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_array_header_t: プロバイダーリストの管理に使用。
  • apr_interval_time_t: タイムアウト値の管理に使用。

失敗モードと境界条件

  • 設定が未定義の場合、各フィールドはデフォルト値（または NULL）となります。
  • context が設定されていない場合、キャッシュキーの生成ロジックに影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体はディレクトリごとの設定を保持するため、AuthnCacheSOCache ディレクティブ等の設定変更時に適切に更新される必要があります。
  • context フィールドは APR_OFFSETOF を用いて設定コマンドから直接参照される箇所があるため、構造体のメンバ順序を変更する際は注意してください。
