ap_lookup_provider

定義

  • 種別: function
  • 定義箇所: server/provider.c:99

目的

グローバルなプロバイダーレジストリから、指定されたグループ、名前、バージョンに一致するプロバイダー（関数ポインタやデータ構造）を検索して取得する。

シグネチャまたは形状

void *ap_lookup_provider(const char *provider_group,
                         const char *provider_name,
                         const char *provider_version)

入力と出力

  • **入力**:
  • provider_group: 検索対象のプロバイダーグループ名。
  • provider_name: 検索対象のプロバイダー名。
  • provider_version: 検索対象のプロバイダーバージョン。
  • **出力**:
  • 成功時: 該当するプロバイダーへのポインタ（void *）。
  • 失敗時: NULL。

前提条件と不変条件

  • global_providers が初期化されている必要がある。
  • 検索キー（グループ、名前、バージョン）は文字列としてハッシュテーブルに格納されている必要がある。

アルゴリズムまたは振る舞い

1. global_providers が NULL の場合、直ちに NULL を返す。 2. provider_group をキーとして global_providers ハッシュテーブルから provider_group_hash を取得する。存在しない場合は NULL を返す。 3. provider_name をキーとして provider_group_hash から provider_name_hash を取得する。存在しない場合は NULL を返す。 4. provider_version をキーとして provider_name_hash からプロバイダーの実体を取得し、返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_auth_basic.c
  • modules/aaa/mod_auth_digest.c
  • modules/aaa/mod_auth_form.c
  • modules/aaa/mod_authn_core.c
  • modules/aaa/mod_authn_socache.c
  • modules/aaa/mod_authz_core.c
  • **呼び出し先**:
  • apr_hash_get (APR ライブラリ関数)

関連データ構造

  • global_providers: プロバイダーを管理する3階層（グループ → 名前 → バージョン）のハッシュテーブル。

失敗モードと境界条件

  • global_providers が未初期化の場合、NULL を返す。
  • 指定したグループ、名前、またはバージョンのいずれかが存在しない場合、NULL を返す。

保守担当者向けメモ

  • 本関数は読み取り専用の検索操作を行う。
  • 登録処理（ap_register_provider）と競合しないよう、サーバー起動時の設定読み込みフェーズでの利用が想定されている。実行時の動的な変更に対するスレッドセーフティは保証されていない。
