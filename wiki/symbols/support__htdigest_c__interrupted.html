interrupted

定義

  • 種別: function
  • 定義箇所: support/htdigest.c:179

目的

プログラム実行中にシグナル等による割り込みが発生した際、エラーメッセージを出力し、一時ファイルを適切に処理して終了するためのハンドラです。

シグネチャまたは形状

static void interrupted(void)

入力と出力

  • 入力: なし
  • 出力: なし（errfile へのメッセージ出力およびプロセス終了）

前提条件と不変条件

  • errfile が有効なファイルストリームとして初期化されていること。
  • cleanup_tempfile_and_exit が呼び出し可能であること。

アルゴリズムまたは振る舞い

1. apr_file_printf を使用し、標準エラー出力（errfile）へ "Interrupted.\n" というメッセージを出力します。 2. cleanup_tempfile_and_exit(1) を呼び出し、一時ファイルのクリーンアップと終了ステータス 1 でのプロセス終了を実行します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では明示的な呼び出し元は特定されていません。
  • 呼び出し先:
  • apr_file_printf
  • cleanup_tempfile_and_exit

関連データ構造

  • errfile: エラー出力先として使用される apr_file_t 型のファイルポインタ。

失敗モードと境界条件

  • errfile が無効な場合、apr_file_printf の呼び出しが失敗する可能性があります。
  • cleanup_tempfile_and_exit 内部でのファイル削除処理が失敗した場合、一時ファイルが残存する可能性があります。

保守担当者向けメモ

  • 本関数はシグナルハンドラとして登録されることを想定した設計ですが、シグナルハンドラ内での apr_file_printf のような非同期シグナル安全ではない関数の使用には注意が必要です。
  • ツールによる呼び出し元検索結果に表示されている他の interrupted という名称のシンボルは、本ファイル（support/htdigest.c）の関数とは無関係です。
