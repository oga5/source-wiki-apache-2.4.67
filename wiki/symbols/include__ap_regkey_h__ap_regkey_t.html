ap_regkey_t

定義

  • 種別: struct
  • 定義箇所: include/ap_regkey.h:35

目的

Windows レジストリキーを保持・管理するための不透明な構造体です。Win32 API のレジストリ操作を抽象化し、APR（Apache Portable Runtime）のメモリ管理機能と統合するために使用されます。

シグネチャまたは形状

typedef struct ap_regkey_t ap_regkey_t;

入力と出力

  • 本構造体自体は不透明（opaque）であり、内部メンバは公開されていません。
  • ap_regkey_open 等の関数を通じて、この構造体へのポインタが生成・取得されます。

前提条件と不変条件

  • **プラットフォーム**: WIN32 マクロが定義されている環境でのみ有効です。
  • **メモリ管理**: apr_pool_t に関連付けられており、プールが破棄される際に自動的にクリーンアップ（RegCloseKey の実行）が行われるよう設計されています。

アルゴリズムまたは振る舞い

  • ap_regkey_open を呼び出すことで、親キーとパスを指定してインスタンス化されます。
  • インスタンス化時に apr_pool_cleanup_register が実行され、ライフサイクルがプールによって管理されます。
  • ap_regkey_close を呼び出すことで、明示的にハンドルを解放し、クリーンアップ登録を解除することが可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: レジストリ操作を行う HTTPD モジュール群。
  • **呼び出し先**: Win32 API（RegCloseKey 等）をラップした os/win32/ap_regkey.c 内の関数群。
  • 具体的な呼び出し元コードの詳細は、提供されたソース excerpt には含まれていません。

関連データ構造

  • apr_pool_t: メモリ管理およびライフサイクル管理の基盤。
  • AP_REGKEY_LOCAL_MACHINE 等の定数: ルートキーを指定するために使用されます。

失敗モードと境界条件

  • **非再帰的削除**: ap_regkey_remove はサブキーを持つキーを削除できません。
  • **環境変数**: AP_REGKEY_EXPAND フラグ使用時、レジストリ値内の %VAR% 形式の文字列は自動的に展開されます。

保守担当者向けメモ

  • 本構造体は os/win32/ap_regkey.c で実装されています。
  • ap_regkey_t を扱う際は、必ず適切なライフサイクルを持つ apr_pool_t を使用してください。
