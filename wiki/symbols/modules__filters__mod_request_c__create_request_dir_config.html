create_request_dir_config

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:328

目的

Apache HTTP Server のディレクトリ単位の設定構造体 request_dir_conf を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_request_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • p: 設定構造体を割り当てるための apr_pool_t プール。
  • dummy: Apache の設定フレームワークから渡されるダミー引数（使用されない）。
  • **出力**:
  • 初期化済みの request_dir_conf 構造体へのポインタ（void * として返却）。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全メンバはゼロ初期化される。
  • 戻り値の構造体は、Apache のリクエスト処理ライフサイクルに紐付いたメモリプールから割り当てられる。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して request_dir_conf サイズのメモリを確保する。 2. keep_body_set を 0（未設定）に初期化する。 3. keep_body を 0（デフォルトで保持しない）に初期化する。 4. 確保した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_request.c:387（モジュールの設定構造体登録時に呼び出される）。
  • **呼び出し先**: apr_pcalloc（APRライブラリ）。

関連データ構造

  • request_dir_conf: 本関数が初期化するディレクトリ設定構造体。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足により失敗した場合、Apache の標準的なメモリ割り当てエラー処理に従い、プロセスが終了または例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は module 構造体の create_dir_config フックとして登録されることを想定している。
  • dummy 引数は Apache の設定 API の仕様上存在するが、本実装では無視される。
