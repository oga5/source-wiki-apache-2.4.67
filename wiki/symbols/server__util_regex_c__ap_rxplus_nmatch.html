ap_rxplus_nmatch

定義

  • 種別: function
  • 定義箇所: server/util_regex.c:183

目的

コンパイル済みの正規表現オブジェクトにおいて、直近のマッチング操作で検出されたキャプチャグループの総数を取得します。

シグネチャまたは形状

int ap_rxplus_nmatch(ap_rxplus_t *rx)

入力と出力

  • **入力**: rx - ap_rxplus_t 構造体へのポインタ。
  • **出力**: マッチングが成功している場合はキャプチャグループ数（int）、マッチング結果が存在しない場合は 0 を返します。

前提条件と不変条件

  • rx は有効な ap_rxplus_t オブジェクトを指している必要があります。
  • rx->match が NULL の場合、本関数は常に 0 を返します。

アルゴリズムまたは振る舞い

1. rx->match が NULL かどうかを判定します。 2. NULL であれば 0 を返します。 3. NULL でなければ、構造体内部の rx->nmatch フィールドの値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:1229
  • server/util_regex.c:196
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、構造体のメンバを参照するのみです。

関連データ構造

  • ap_rxplus_t: 正規表現のコンパイル結果、フラグ、およびマッチング状態を保持するメイン構造体。

失敗モードと境界条件

  • rx が NULL の場合、セグメンテーションフォールトが発生する可能性があります（ソースコード上でのガードは存在しません）。
  • マッチングが一度も実行されていない、または前回のマッチングが失敗している場合、rx->match が NULL となり、戻り値は 0 となります。

保守担当者向けメモ

  • 本関数は ap_regex.h:267 にてマクロとしても定義されています。
  • rx->nmatch は ap_rxplus_exec の実行結果に依存するため、マッチング処理の直後に呼び出すことが想定されています。
