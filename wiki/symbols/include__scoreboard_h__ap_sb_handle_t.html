ap_sb_handle_t

定義

  • 種別: struct
  • 定義箇所: include/scoreboard.h:167

目的

Apache HTTP Server において、特定のワーカープロセスまたはスレッドのスコアボード（実行時状態）を識別・参照するための不透明なハンドルです。

シグネチャまたは形状

typedef struct ap_sb_handle_t ap_sb_handle_t;

構造体の内部定義は公開されておらず、不透明なポインタとして扱われます。

入力と出力

  • **入力**: ap_create_sb_handle 等の関数を通じて生成されます。
  • **出力**: ap_get_scoreboard_worker 等のアクセサ関数に渡すことで、対応する worker_score 構造体へのアクセスを提供します。

前提条件と不変条件

  • 共有メモリ上に配置されるスコアボード情報を指すため、有効なライフサイクル内でのみ使用可能です。
  • ap_create_sb_handle によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • MPM（Multi-Processing Module）がワーカーの状態を更新する際、このハンドルを介して共有メモリ上の該当領域を特定します。
  • ap_update_child_status ファミリー関数群において、状態更新対象を特定するためのキーとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_echo.c (例: update_echo_child_status) および各 MPM 実装。
  • **呼び出し先**: 本シンボル自体はデータ構造であり、ap_update_child_status、ap_get_scoreboard_worker、ap_time_process_request などのスコアボード操作 API に引数として渡されます。

関連データ構造

  • worker_score: このハンドルが指し示す実体となる、ワーカーごとの統計情報構造体。

失敗モードと境界条件

  • ハンドルが初期化されていない、または無効なメモリ領域を指している場合、アクセサ関数経由で共有メモリへアクセスした際に未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は不透明な型として定義されており、直接メンバにアクセスすることは避けてください。
  • worker_score 構造体へのアクセスが必要な場合は、必ず ap_get_scoreboard_worker などの公開 API を経由してください。
