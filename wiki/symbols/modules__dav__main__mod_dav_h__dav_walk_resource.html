dav_walk_resource

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1809

目的

WebDAV リソースのツリー走査（walk）処理において、走査中のコンテキスト情報、現在のリソース状態、および生成されるレスポンスを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    void *walk_ctx;
    apr_pool_t *pool;
    const dav_resource *resource;
    dav_response *response;
} dav_walk_resource;

入力と出力

  • **入力**: 走査処理を開始する際に、クライアント定義のコンテキスト (walk_ctx) やメモリプール (pool) を設定して初期化します。
  • **出力**: 走査中に発生した結果やエラー情報は response フィールドを通じて蓄積されます。

前提条件と不変条件

  • pool は、走査中のコールバック関数内でのメモリ割り当てに使用される有効な apr_pool_t である必要があります。
  • resource は、現在処理中の有効な dav_resource を指す必要があります。

アルゴリズムまたは振る舞い

走査処理（dav_fs_copymove_walker, dav_propfind_walker 等）において、この構造体は引数として渡されます。各走査関数は、resource を参照して処理を行い、必要に応じて dav_add_response を用いて response リストに結果を追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/repos.c（dav_fs_copymove_walker, dav_fs_delete_walker）、modules/dav/main/mod_dav.c（dav_propfind_walker）、modules/dav/main/util.c（dav_validate_walker）、modules/dav/main/util_lock.c（dav_lock_walker）など。
  • **呼び出し先**: dav_add_response（modules/dav/main/mod_dav.c）がこの構造体を引数として受け取り、レスポンスの追加を行います。

関連データ構造

  • dav_resource: 走査対象のリソース表現。
  • dav_response: 走査結果として生成されるマルチステータスレスポンスのリスト。
  • dav_error: 走査関数が戻り値として返すエラー情報。

失敗モードと境界条件

  • pool が無効な場合、コールバック内でのメモリ割り当てが失敗し、セグメンテーション違反やメモリリークを引き起こす可能性があります。
  • resource が NULL の場合、走査関数内で参照解決時にエラーが発生します。

保守担当者向けメモ

  • 本構造体は、WebDAV の再帰的な操作（PROPFIND, COPY, MOVE, DELETE 等）において不可欠なコンテキスト保持用オブジェクトです。
  • walk_ctx は汎用的な void * であるため、キャスト時の型安全性には注意が必要です。
