ap_get_status_line

定義

  • 種別: function
  • 定義箇所: modules/http/http_protocol.c:1035

目的

指定された HTTP ステータスコードに対応する、標準的な HTTP ステータス行（例: "200 OK"）の文字列定数を取得します。

シグネチャまたは形状

const char *ap_get_status_line(int status)

入力と出力

  • **入力**: status (int型) - HTTP ステータスコード。
  • **出力**: const char * - ステータスコードに対応する理由句を含むステータス行文字列。

前提条件と不変条件

  • status は、ap_index_of_response が処理可能な有効なステータスコードである必要があります。
  • 返される文字列は静的配列 status_lines 内の要素を指しており、呼び出し元で解放してはなりません。

アルゴリズムまたは振る舞い

1. ap_index_of_response(status) を呼び出し、ステータスコードを内部配列のインデックスに変換します。 2. 変換されたインデックスを使用して、静的配列 status_lines から対応するステータス行文字列を取得し、そのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/win32/mod_isapi.c
  • modules/dav/main/mod_dav.c
  • modules/http/http_filters.c
  • modules/http2/h2_proxy_session.c
  • **呼び出し先**:
  • ap_index_of_response (同ファイル内)

関連データ構造

  • status_lines: modules/http/http_protocol.c 内で定義されている、全 HTTP ステータスコードと理由句を保持する静的配列。

失敗モードと境界条件

  • ap_index_of_response が未知のステータスコードに対して無効なインデックスを返した場合、配列外参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数は include/http_protocol.h で公開されています。
  • status_lines 配列は RESPONSE_CODES 定数に依存しており、新しいステータスコードを追加する際は、配列とインデックス計算ロジックの整合性に注意してください。
