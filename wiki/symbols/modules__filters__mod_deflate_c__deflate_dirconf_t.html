deflate_dirconf_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_deflate.c:76

目的

Apache HTTP Server の mod_deflate モジュールにおいて、ディレクトリ単位での圧縮・伸長処理に関する制限設定を保持するために使用されます。

シグネチャまたは形状

typedef struct deflate_dirconf_t {
    apr_off_t inflate_limit;
    int ratio_limit,
        ratio_burst;
} deflate_dirconf_t;

入力と出力

本構造体は設定値のコンテナであり、直接的な入出力関数ではありません。各フィールドは以下の値を保持します。

  • inflate_limit: 伸長処理における最大許容サイズ。
  • ratio_limit: 圧縮比の制限値。
  • ratio_burst: 圧縮比制限を一時的に許容するバースト値。

前提条件と不変条件

  • mod_deflate の設定フェーズで初期化され、リクエスト処理中にディレクトリ設定として参照されます。
  • 伸長処理時に inflate_limit を超えた場合、処理は中断されます。

アルゴリズムまたは振る舞い

本構造体は、DoS 攻撃対策としてのリソース制限を管理します。伸長後のデータサイズや圧縮比を監視し、設定された閾値を超過した場合に APR_EINVAL や APR_ENOSPC を返す判断基準として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_deflate.c 内の各フィルタ関数および設定関数（create_deflate_dirconf 等）から、ディレクトリ設定として取得・参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • deflate_filter_config: サーバー全体の設定を保持する構造体。
  • deflate_ctx: フィルタの実行状態（zlib コンテキスト等）を保持する構造体。

失敗モードと境界条件

  • inflate_limit が設定されている場合、伸長後のデータがこの値を超えると、フィルタ処理はエラーを返して停止します。
  • ratio_limit および ratio_burst は、圧縮比が異常に高い（圧縮爆弾の疑いがある）場合に処理を制限するために使用されます。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を扱うため、.htaccess や <Directory> ブロックでの設定変更が反映されます。
  • 制限値の追加や変更を行う際は、mod_deflate.c 内の関連する設定ディレクティブハンドラと、フィルタ処理中のチェックロジックの両方を更新する必要があります。
