create_userdir_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_userdir.c:103

目的

mod_userdir モジュールのサーバー設定構造体（userdir_config）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_userdir_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定を割り当てるための APR メモリプール。
  • s: 設定が適用されるサーバーレコード（本関数内では未使用）。
  • **出力**:
  • 初期化済みの userdir_config 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全フィールドはゼロ初期化される。
  • enabled_users および disabled_users は、それぞれ 4 つの要素を保持可能な空のテーブルとして作成される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、userdir_config サイズのメモリをプールから確保する。 2. globally_disabled を O_DEFAULT に設定する。 3. userdir を DEFAULT_USER_DIR に設定する。 4. apr_table_make を使用して、ユーザーの有効/無効リストを管理するテーブルを初期化する。 5. 初期化された構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_userdir.c:386（サーバー設定の作成時）。
  • **呼び出し先**: apr_pcalloc, apr_table_make（APR API）。

関連データ構造

  • userdir_config: 本関数で初期化されるモジュール固有の設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性がある（APR の挙動に依存）。

保守担当者向けメモ

  • 本関数は Apache の設定フックとして呼び出されることを想定している。
  • enabled_users および disabled_users の初期サイズ（4）は、一般的なユーザー数に対するヒューリスティックな値である。
