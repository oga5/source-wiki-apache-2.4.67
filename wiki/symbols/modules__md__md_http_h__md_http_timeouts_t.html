md_http_timeouts_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_http.h:50

目的

HTTP 通信における接続、全体的な処理時間、およびストール（通信停滞）検出のためのタイムアウト設定を保持する構造体です。

シグネチャまたは形状

typedef struct md_http_timeouts_t md_http_timeouts_t;

struct md_http_timeouts_t {
    /* 内部メンバの詳細はヘッダーの定義箇所を参照 */
};

入力と出力

本構造体は md_http_request_t の一部として設定され、HTTP リクエスト実行時の制約条件として利用されます。

前提条件と不変条件

  • md_http_create を呼び出す前に、適切なタイムアウト値が設定されている必要があります。
  • ストール検出機能を利用する場合は、md_http_set_stalling を介した設定が推奨されます。

アルゴリズムまたは振る舞い

  • HTTP クライアントの実装（md_http_impl_t）は、本構造体に格納された値に基づき、通信のタイムアウトを監視します。
  • 通信が指定された時間を超過した場合、またはストールと判定された場合にエラーを発生させ、リクエストを中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_http.c 内の HTTP リクエスト処理ロジック。
  • **呼び出し先**: HTTP 通信バックエンド（libcurl 等の実装）。
  • *注: 具体的な関数呼び出しの連鎖については、提供されたソース情報からは詳細なスタックトレースを特定できません。*

関連データ構造

  • md_http_request_t: 本構造体をメンバとして保持し、リクエストごとのタイムアウトを管理します。
  • md_http_t: HTTP クライアントセッション全体の設定を管理します。

失敗モードと境界条件

  • タイムアウト値が 0 または不適切な値に設定されている場合、通信が永久にブロックされるか、即座にエラーとなる可能性があります。
  • ネットワーク環境やバックエンド実装の制限により、指定したタイムアウト値が厳密に適用されない場合があります。

保守担当者向けメモ

  • 本構造体は md_http_request_t の一部として使用されるため、メモリ管理はリクエストのライフサイクルに依存します。
  • ストール検出のロジックを変更する場合は、md_http_impl_t の各実装（md_curl.c 等）との整合性に注意してください。
