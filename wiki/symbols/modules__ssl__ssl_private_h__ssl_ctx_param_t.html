ssl_ctx_param_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:771

目的

SSL コンテキストに関連付けられる名前と値のペアを保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    const char *name;
    const char *value;
} ssl_ctx_param_t;

入力と出力

  • name: パラメータ名を指す文字列ポインタ。
  • value: パラメータ値を指す文字列ポインタ。

前提条件と不変条件

  • name および value は、Apache のメモリプール（apr_pool_t）の生存期間に依存する文字列を指すことが想定されます。

アルゴリズムまたは振る舞い

本構造体は、主に apr_array_header_t の要素として格納され、設定値のリストを管理するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_config.c: 設定初期化時に配列の生成に使用。
  • modules/ssl/ssl_engine_init.c: 設定適用時に配列内の要素を走査・参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • apr_array_header_t: 本構造体のインスタンスを格納するコンテナとして使用されます。
  • SSLSrvConfigRec: 本構造体の配列を保持する仮想ホスト設定構造体。

失敗モードと境界条件

  • メモリ不足時: apr_array_make 等の呼び出し失敗により、システム全体の初期化が停止する可能性があります。
  • NULL ポインタ: name や value に NULL が設定された場合の挙動は、これらを参照する側の実装に依存します。

保守担当者向けメモ

  • 本構造体は、OpenSSL のコンテキスト設定を拡張する際に使用されます。新しいパラメータを追加する際は、ssl_engine_config.c での解析ロジックと、ssl_engine_init.c での適用ロジックの両方を更新してください。
