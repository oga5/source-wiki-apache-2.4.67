ajp_msg

定義

  • 種別: struct
  • 定義箇所: modules/proxy/ajp.h:112

目的

AJP (Apache Jserv Protocol) v13 通信におけるメッセージのメモリレイアウトと状態を管理するための構造体です。

シグネチャまたは形状

struct ajp_msg {
    apr_byte_t  *buf;         /* メッセージを保持するバッファ */
    apr_size_t  header_len;   /* ヘッダー長 (デフォルト: AJP_HEADER_LEN) */
    apr_size_t  len;          /* メッセージ全体の長さ */
    apr_size_t  pos;          /* 現在の読み取り位置 */
    int         server_side;  /* メッセージの起源を示すフラグ */
    apr_size_t  max_size;     /* バッファの最大サイズ */
};

入力と出力

  • **入力**: ajp_msg_create 等により初期化され、ajp_msg_append_* 関数群を通じてバッファにデータが書き込まれます。
  • **出力**: ajp_msg_get_* 関数群によりバッファからデータが抽出されるか、ajp_ilink_send を介してソケットへ送信されます。

前提条件と不変条件

  • pos は常に len 以下である必要があります。
  • len は max_size を超えてはなりません。
  • メッセージの生存期間は、関連付けられた apr_pool_t のライフサイクルに依存します。

アルゴリズムまたは振る舞い

  • **状態管理**: pos を用いてシリアライズおよびデシリアライズの進行状況を追跡します。
  • **メモリ管理**: apr_pool_t を介して動的に割り当てられたバッファを使用します。
  • **通信**: ajp_ilink_send および ajp_ilink_receive を通じて、バイナリプロトコルとしての送受信が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/ajp.h:109 にて ajp_msg_t として型定義（typedef）されており、mod_proxy_ajp モジュール全体で利用されます。
  • **呼び出し先**: 具体的な関数呼び出し関係は本ソース抜粋からは特定できません。

関連データ構造

  • ajp_msg_t: 本構造体の型定義名。
  • apr_pool_t: メモリ管理に使用されるプール構造体。

失敗モードと境界条件

  • **バッファサイズ**: AJP_MSG_BUFFER_SZ (8KB) および AJP_MAX_BUFFER_SZ (64KB) が物理的な上限となります。
  • **エラー**: AJP_E* で定義される独自のエラーコードが、バッファオーバーフローや通信エラー時に返される可能性があります。

保守担当者向けメモ

  • 本構造体は ajp_msg.c で実装される操作関数の対象となります。
  • EBCDIC 環境での文字列操作には ajp_msg_append_string_ex を使用してください。
  • ソケットのタイムアウトやバッファの再配置ロジックは、本構造体ではなく実装ファイル側に記述されています。
