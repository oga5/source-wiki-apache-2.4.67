ssl_get_effective_config

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:58

目的

指定された接続（conn_rec）に関連付けられた SSL 設定構造体（SSLConnRec）を取得します。現在の接続に SSL 設定が存在しない場合、マスター接続の設定をフォールバックとして参照します。

シグネチャまたは形状

static SSLConnRec *ssl_get_effective_config(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（対象となる接続構造体）
  • **出力**: SSLConnRec *（解決された SSL 設定構造体、または NULL）

前提条件と不変条件

  • 入力 c は有効な conn_rec ポインタである必要があります。
  • myConnConfig(c) マクロを通じて、接続に関連付けられた設定が取得可能であることを前提としています。

アルゴリズムまたは振る舞い

1. myConnConfig(c) を呼び出し、現在の接続から SSLConnRec を取得します。 2. 取得した sslconn が NULL であるか、またはその内部の ssl メンバが未定義の場合、現在の接続がマスター接続（c->master）を持っているかを確認します。 3. マスター接続が存在する場合、そのマスター接続から再度 myConnConfig を用いて設定を取得し、それを有効な設定として返します。 4. 上記のいずれにも該当しない場合は、最初に取得した（あるいは NULL の）値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_vars.c:70
  • modules/ssl/ssl_engine_vars.c:88
  • modules/ssl/ssl_engine_vars.c:273
  • modules/ssl/ssl_engine_vars.c:1049
  • modules/ssl/ssl_engine_vars.c:1193
  • **呼び出し先**:
  • myConnConfig（マクロ）

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • SSLConnRec: mod_ssl が接続ごとに保持する SSL 固有の設定および状態構造体。

失敗モードと境界条件

  • 接続に SSL 設定が紐付いておらず、かつマスター接続も存在しない場合、NULL を返します。呼び出し元はこの戻り値が NULL である可能性を考慮する必要があります。

保守担当者向けメモ

  • 本関数は static であり、ssl_engine_vars.c 内でのみ使用されます。
  • c->master を参照するロジックは、サブ接続や内部リクエストにおいて SSL コンテキストを正しく継承するために重要です。
