CRYPTO_dynlock_value

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_util.c:275

目的

OpenSSL 1.1 未満の環境において、OpenSSL が内部で使用する動的ロック（Dynamic Lock）を Apache Portable Runtime (APR) のスレッド同期プリミティブを用いて管理するためのラッパー構造体です。

シグネチャまたは形状

struct CRYPTO_dynlock_value {
    apr_pool_t *pool;
    const char* file;
    int line;
    apr_thread_mutex_t *mutex;
};

入力と出力

  • **入力**: 構造体内の pool はメモリ管理用、mutex はスレッド同期用に使用されます。file と line はデバッグおよび追跡用として保持されます。
  • **出力**: なし（本構造体自体はデータ保持用）。

前提条件と不変条件

  • MODSSL_USE_OPENSSL_PRE_1_1_API が定義されている環境でのみ使用されます。
  • mutex は apr_thread_mutex_t 型であり、APR のスレッド同期規則に従います。
  • 構造体内の各フィールドは、動的ロックの生成時に初期化される必要があります。

アルゴリズムまたは振る舞い

本構造体は、OpenSSL の動的ロックコールバック（ssl_dyn_create_function 等）によって操作されます。 1. **生成**: ssl_dyn_create_function 内で apr_palloc を用いてメモリが確保され、mutex が作成されます。 2. **ロック/アンロック**: ssl_dyn_lock_function を通じて、内部の mutex が apr_thread_mutex_lock または apr_thread_mutex_unlock を実行します。 3. **破棄**: ssl_dyn_destroy_function を通じて mutex が破棄されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ssl_dyn_create_function (modules/ssl/ssl_util.c:288)
  • ssl_dyn_lock_function (modules/ssl/ssl_util.c:326)
  • ssl_dyn_destroy_function (modules/ssl/ssl_util.c:350)
  • **呼び出し先**:
  • apr_thread_mutex_t (APR API)

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_thread_mutex_t: スレッド同期に使用。

失敗モードと境界条件

  • apr_thread_mutex_create が失敗した場合、ロックの初期化に失敗し、OpenSSL のスレッドセーフ性が損なわれる可能性があります。
  • 構造体のメモリ確保（apr_palloc）が失敗した場合、プログラムは異常終了する可能性があります。

保守担当者向けメモ

  • 本構造体は OpenSSL 1.1 以降では不要です。OpenSSL 1.1 以降の環境では、これらのコールバックおよび構造体は無視されます。
  • ssl_dyn_destroy_function において、ロックの所有権が適切に管理されていることを前提としています。
