ap_unixd_accept

定義

  • 種別: function
  • 定義箇所: os/unix/unixd.c:293

目的

Unix系プラットフォームにおいて、リスナーソケットに対する接続要求（accept）を処理し、新規接続ソケットを取得する。

シグネチャまたは形状

apr_status_t ap_unixd_accept(void **accepted, ap_listen_rec *lr, apr_pool_t *ptrans)

入力と出力

  • **入力**:
  • lr: 接続を待機するリスナーレコード（ap_listen_rec）。
  • ptrans: 接続用ソケットを割り当てるためのメモリプール。
  • **出力**:
  • accepted: 成功時に新規接続ソケット（apr_socket_t）が格納されるポインタ。
  • 戻り値: APR_SUCCESS（成功）、またはエラーを示す apr_status_t。

前提条件と不変条件

  • lr は有効なリスナーレコードであること。
  • _OSD_POSIX 環境下では、取得したファイルディスクリプタが FD_SETSIZE を超えないことが要求される。

アルゴリズムまたは振る舞い

1. apr_socket_accept を呼び出し、接続を試みる。 2. 成功時、_OSD_POSIX 環境であればファイルディスクリプタのサイズを検証し、超過していれば警告を出して APR_EINTR を返す。 3. 失敗時、EINTR（割り込み）はそのまま透過的に返す。 4. その他のエラー発生時、特定の非致命的エラー（ECONNABORTED, EAGAIN 等）は無視して呼び出し元へ制御を戻す。 5. ENETDOWN のような致命的なネットワークエラーは APR_EGENERAL を返し、サーバーの停止を促す。 6. リスナーが非アクティブ（lr->active == 0）な場合は、エラーログを出さずにステータスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/event/event.c, server/mpm/prefork/prefork.c, server/mpm/worker/worker.c の各 MPM において、lr->accept_func に代入される形で利用される。
  • **呼び出し先**: apr_socket_accept, apr_os_sock_get, apr_socket_close, ap_log_error。

関連データ構造

  • ap_listen_rec: リスナーソケットの状態を保持する構造体。
  • apr_socket_t: APR レベルのソケットオブジェクト。

失敗モードと境界条件

  • **ファイルディスクリプタ制限**: _OSD_POSIX 環境で FD_SETSIZE を超過した場合、ソケットを閉じてエラーを返す。
  • **ネットワーク停止**: ENETDOWN 発生時は致命的エラーとして処理される。
  • **非アクティブなリスナー**: サーバー停止・再起動処理中に EBADF 等が発生した場合、ログ出力を抑制して終了する。

保守担当者向けメモ

  • 本関数は歴史的な経緯から、プラットフォーム固有の挙動（HPUX の ENOBUFS や Solaris の EPROTO 等）を吸収する複雑な条件分岐を含んでいる。
  • 致命的なエラーか否かの判定は、サーバーの安定稼働に直結するため、新しいプラットフォーム対応を追加する際は慎重に行うこと。
