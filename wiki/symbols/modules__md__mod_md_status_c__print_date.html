print_date

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:220

目的

指定されたタイムスタンプを人間が読みやすい形式（YYYY-MM-DD）に変換し、status_ctx に基づいて HTML またはテキスト形式で出力する。

シグネチャまたは形状

static void print_date(status_ctx *ctx, apr_time_t timestamp, const char *title)

入力と出力

  • **入力**:
  • ctx: 出力先バッファ (bb) やフォーマットフラグを保持するコンテキスト。
  • timestamp: 変換対象の apr_time_t 型の時刻。
  • title: 表示ラベル。NULL の場合は ISO 8601 形式の文字列が自動生成される。
  • **出力**: なし（ctx->bb に直接書き込まれる）。

前提条件と不変条件

  • timestamp が 0 より大きい場合のみ出力が行われる。
  • ctx->bb は有効な apr_bucket_brigade である必要がある。

アルゴリズムまたは振る舞い

1. apr_time_exp_gmt を使用して、timestamp を GMT に変換する。 2. apr_strftime を用いて、日付を YYYY-MM-DD 形式の文字列に変換する。 3. title が指定されていない場合、YYYY-MM-DDTHH:MM:SSZ 形式の文字列を生成し、それを title として使用する。 4. HTML_STATUS(ctx) マクロにより出力形式を判定する:

  • **HTML の場合**: <span> タグで囲み、title をツールチップ（title 属性）として設定する。
  • **テキストの場合**: ctx->prefix、title、日付文字列を連結して出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_status.c:312
  • modules/md/mod_md_status.c:319
  • modules/md/mod_md_status.c:324
  • modules/md/mod_md_status.c:328
  • **呼び出し先**:
  • apr_time_exp_gmt
  • apr_strftime
  • apr_brigade_printf
  • ap_escape_html2 (HTML モード時)

関連データ構造

  • status_ctx: 出力コンテキスト。
  • apr_bucket_brigade: Apache の出力用バッファ構造体。
  • apr_time_exp_t: 時刻の分解構造体。

失敗モードと境界条件

  • timestamp <= 0: 何も出力されない。
  • title が NULL の場合: 自動生成された ISO 形式の文字列がラベルとして使用される。
  • バッファサイズ: ts および ts2 配列は 128 バイト確保されており、標準的な日付文字列の格納には十分である。

保守担当者向けメモ

  • 本関数は mod_status の出力モード（HTML/テキスト）を動的に切り替えるため、出力形式を変更する際は HTML_STATUS マクロの判定ロジックと整合性をとること。
  • ap_escape_html2 を使用しているため、HTML 出力時の XSS 対策は考慮されている。
