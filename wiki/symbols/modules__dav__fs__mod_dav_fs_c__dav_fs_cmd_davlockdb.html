dav_fs_cmd_davlockdb

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/mod_dav_fs.c:63

目的

Apache HTTP Server の DAVLockDB ディレクティブを処理し、ファイルシステムベースの WebDAV ロックデータベースのパスを設定します。

シグネチャまたは形状

static const char *dav_fs_cmd_davlockdb(cmd_parms *cmd, void *config, const char *arg1)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • config: 現在のサーバー設定（未使用）。
  • arg1: DAVLockDB ディレクティブで指定されたパス文字列。
  • **出力**:
  • 成功時: NULL を返します。
  • 失敗時: エラーメッセージを含む文字列を返します。

前提条件と不変条件

  • cmd->server->module_config から dav_fs_module に関連付けられた dav_fs_server_conf 構造体が取得可能である必要があります。
  • arg1 はロックデータベースのパスとして解釈されます。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定から dav_fs_server_conf を取得します。 2. ap_server_root_relative を呼び出し、指定された arg1 をサーバーのルートディレクトリからの絶対パスに変換して conf->lockdb_path に格納します。 3. パスの変換に失敗した場合（conf->lockdb_path が NULL の場合）、エラーメッセージを生成して返します。 4. 正常にパスが設定された場合、NULL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/mod_dav_fs.c:82 において、AP_INIT_TAKE1 マクロを通じて DAVLockDB ディレクティブのハンドラとして登録されています。
  • **呼び出し先**:
  • ap_get_module_config (Apache API)
  • ap_server_root_relative (Apache API)
  • apr_pstrcat (APR API)

関連データ構造

  • dav_fs_server_conf: ロックデータベースのパス (lockdb_path) を保持するサーバー設定構造体。
  • cmd_parms: 設定コマンドの実行コンテキスト。

失敗モードと境界条件

  • ap_server_root_relative がパスの解決に失敗した場合、関数はエラーメッセージを返し、設定処理は中断されます。

保守担当者向けメモ

  • 本関数はサーバー単位の設定のみを扱います。ディレクトリごとのオーバーライドはサポートされていません。
  • dav_fs_server_conf 構造体の定義については、関連するシンボルページを参照してください。
