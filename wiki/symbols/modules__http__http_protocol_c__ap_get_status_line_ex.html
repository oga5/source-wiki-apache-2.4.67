ap_get_status_line_ex

定義

  • 種別: function
  • 定義箇所: modules/http/http_protocol.c:1023

目的

指定されたHTTPステータスコードに対応するステータス行（Status Line）の文字列を取得します。

シグネチャまたは形状

const char *ap_get_status_line_ex(apr_pool_t *p, int status)

入力と出力

  • **入力**:
  • p: 文字列生成時に使用するメモリプール。
  • status: 取得対象のHTTPステータスコード（整数）。
  • **出力**:
  • ステータス行を表す const char * 型の文字列。

前提条件と不変条件

  • status_lines 配列が有効なステータス行を保持している必要があります。
  • index_of_response(status) が内部的なステータスインデックスを正しく計算できることを前提としています。

アルゴリズムまたは振る舞い

1. index_of_response(status) を呼び出し、ステータスコードに対応するインデックスを取得します。 2. インデックスが0以上の場合、静的配列 status_lines から該当するステータス行を返します。 3. インデックスが -2 の場合、apr_psprintf を使用して "status Status status" という形式の文字列を動的に生成して返します。 4. 上記以外（インデックスが負で -2 以外）の場合、デフォルトとして LEVEL_500 に対応するステータス行を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/protocol.c:2353 にて r->status を引数として呼び出されています。
  • **呼び出し先**:
  • index_of_response (内部関数)
  • apr_psprintf (APRライブラリ)

関連データ構造

  • status_lines: ステータスコードと理由句を保持する静的配列。
  • apr_pool_t: メモリ管理に使用されるプール構造体。

失敗モードと境界条件

  • 未知のステータスコードが渡された場合、デフォルトの LEVEL_500（内部サーバーエラー）のステータス行が返されます。
  • index_of_response が -2 を返す特殊なケースでは、メモリプール p を使用して文字列が生成されます。

保守担当者向けメモ

  • 本関数は include/ap_mmn.h に記載の通り、2.4.47-dev (20120211.96) にて追加されました。
  • status_lines 配列のインデックス計算は index_of_response に依存しているため、ステータスコードの定義を変更する際は両者の整合性に注意してください。
