conn_state_t

定義

  • 種別: struct
  • 定義箇所: include/httpd.h:824（前方宣言）、include/httpd.h:1293（構造体定義）

目的

クライアントとの接続状態を管理するための構造体です。MPM（マルチプロセッシングモジュール）が接続のライフサイクルや状態遷移を追跡するために使用します。

シグネチャまたは形状

include/httpd.h 内で定義される構造体であり、接続に関連する状態情報を保持します。

入力と出力

  • 入力: conn_rec 構造体から cs フィールドとして参照されます。
  • 出力: MPM（特に event MPM）が接続の監視やタイムアウト処理を行う際の判断材料として利用されます。

前提条件と不変条件

  • conn_rec 構造体と密接に関連しており、接続が存在する間、有効な状態を保持する必要があります。
  • MPM 固有のプライベートな情報が含まれる場合があり、モジュール開発者は直接的な内部フィールド操作に注意が必要です。

アルゴリズムまたは振る舞い

  • 接続のステータス（Keep-Alive 状態やタイムアウト待ちなど）を保持し、MPM が非同期 I/O やイベント駆動処理を行う際の制御フローを決定します。
  • event MPM 等において、接続がどの状態にあるかを追跡するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • modules/http/http_core.c
  • modules/http2/h2_c2.c
  • server/mpm/event/event.c
  • 呼び出し先: 接続状態を管理する MPM の内部ロジックに依存します。

関連データ構造

  • conn_rec: 接続の基本情報を保持する構造体。
  • conn_state_e: 接続状態の列挙型。
  • ap_conn_keepalive_e: Keep-Alive 状態の列挙型。

失敗モードと境界条件

  • 接続が切断された場合やタイムアウトが発生した場合、状態遷移が適切に行われないとメモリリークやハングアップの原因となります。
  • MPM 固有のプライベートな部分が変更された場合、バイナリ互換性に影響を与える可能性があります（ap_mmn.h に関連する記述あり）。

保守担当者向けメモ

  • include/ap_mmn.h にある通り、MPM 固有のプライベートな部分は将来的に変更される可能性があるため、モジュール開発時は可能な限り公開 API を介してアクセスしてください。
  • event MPM のソースコード（server/mpm/event/event.c）が本構造体の利用例として最も参考になります。
