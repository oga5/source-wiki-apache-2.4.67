socache_dc_status

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dc.c:151

目的

Apache の mod_status 経由で、現在使用中の分散キャッシュ（distcache）の接続先情報を出力する。

シグネチャまたは形状

static void socache_dc_status(ap_socache_instance_t *ctx, request_rec *r, int flags)

入力と出力

  • ctx: キャッシュインスタンスへのポインタ。接続先情報（target）の取得に使用。
  • r: 現在のリクエストレコード。出力先として使用。
  • flags: ステータス出力の形式を指定するフラグ（AP_STATUS_SHORT 等）。
  • 出力: r を通じて HTML またはプレーンテキスト形式でキャッシュタイプとターゲット情報を書き出す。

前提条件と不変条件

  • ctx は有効な ap_socache_instance_t 構造体を指している必要がある。
  • ctx->target には接続先の文字列が格納されていること。

アルゴリズムまたは振る舞い

1. デバッグログとして「distributed scache 'socache_dc_status'」を出力する。 2. flags に AP_STATUS_SHORT が含まれていない場合、HTML形式で cache type と target を出力する。 3. flags に AP_STATUS_SHORT が含まれている場合、プレーンテキスト形式（CacheType, CacheTarget）で出力する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内の直接的な呼び出し元は明示されていないが、mod_status 関連のフックから呼び出される設計である。
  • 呼び出し先: ap_log_rerror, ap_rprintf, ap_rputs, ap_rvputs（いずれも Apache API）。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスの保持構造体。target メンバを保持。
  • request_rec: Apache のリクエスト情報構造体。

失敗モードと境界条件

  • ctx が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • flags の指定に関わらず、出力処理は r が有効であることを前提としている。

保守担当者向けメモ

  • 本関数は mod_status の表示用であり、キャッシュの動作自体には影響を与えない。
  • AP_STATUS_SHORT の判定ロジックは、他の socache プロバイダーの実装と一貫性を保つ必要がある。
