fill_reneg_buffer

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_kernel.c:80

目的

SSL再ネゴシエーションの実行に備え、リクエストボディを一時的にバッファへ退避させる。

シグネチャまたは形状

static int fill_reneg_buffer(request_rec *r, SSLDirConfigRec *dc)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体 (request_rec)。
  • dc: SSLディレクトリ設定構造体 (SSLDirConfigRec)。
  • **出力**:
  • int: 処理結果。成功時は 0、バッファサイズが0に設定されている場合は HTTP_REQUEST_ENTITY_TOO_LARGE、その他 ssl_io_buffer_fill の戻り値。

前提条件と不変条件

  • r->expecting_100 が真であるか、リクエストボディが存在しない場合は、バッファリングを行わず直ちに 0 を返す。
  • バッファサイズは dc->nRenegBufferSize を参照し、UNSET の場合は DEFAULT_RENEG_BUFFER_SIZE を使用する。

アルゴリズムまたは振る舞い

1. リクエストがボディを持つか確認し、不要な場合は終了する。 2. 設定されたバッファサイズ（rsize）を取得する。 3. rsize が 0 より大きい場合、ssl_io_buffer_fill(r, rsize) を呼び出してリクエストボディをバッファに読み込む。 4. rsize が 0 の場合、バッファリングを拒否し HTTP_REQUEST_ENTITY_TOO_LARGE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_kernel.c:697
  • modules/ssl/ssl_engine_kernel.c:999
  • **呼び出し先**:
  • ap_request_has_body (Apache API)
  • ssl_io_buffer_fill (内部関数)

関連データ構造

  • request_rec: Apacheのリクエストコンテキスト。
  • SSLDirConfigRec: SSL関連のディレクトリ設定。

失敗モードと境界条件

  • rsize が 0 に設定されている場合、再ネゴシエーションのためのバッファリングは失敗し、HTTP_REQUEST_ENTITY_TOO_LARGE が返される。
  • ssl_io_buffer_fill が失敗した場合、その戻り値がそのまま伝播される。

保守担当者向けメモ

  • 本関数は HTTP/1.1 特有の挙動を前提としている可能性がある（ソース内のコメント参照）。
  • TLS 1.3 では再ネゴシエーションの概念が異なるため、将来的なプロトコル対応時には本関数の呼び出しパスを確認すること。
