register_hooks

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_anon.c:199

目的

Apache HTTP Server の認証フレームワークに対し、匿名認証プロバイダー（anon）を登録する。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p（リソース割り当て用のメモリプール）
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • ap_register_auth_provider が利用可能であること。
  • authn_anon_provider 構造体が定義済みであること。

アルゴリズムまたは振る舞い

1. ap_register_auth_provider を呼び出し、認証プロバイダーグループ AUTHN_PROVIDER_GROUP に登録を行う。 2. プロバイダー名を "anon" として指定する。 3. 認証プロバイダーのインターフェースとして &authn_anon_provider を渡す。 4. フラグ AP_AUTH_INTERNAL_PER_CONF を指定し、設定ごとの内部認証として登録する。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュールロードプロセス（直接的な呼び出し元関数の詳細はソースコード内に明示されていない）。
  • 呼び出し先: ap_register_auth_provider（Apache API）。

関連データ構造

  • authn_anon_provider: 認証関数へのポインタを保持するプロバイダー構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • 登録処理自体は戻り値を返さないため、失敗時の挙動は Apache のコアフレームワークの依存する。

保守担当者向けメモ

  • 本関数はモジュール初期化時に一度だけ実行される。
  • AUTHN_PROVIDER_VERSION および AUTHN_PROVIDER_GROUP は mod_auth.h で定義されている標準的な定数である。
