dumpio_conf_t

定義

  • 種別: struct
  • 定義箇所: modules/debugging/mod_dumpio.c:39

目的

mod_dumpio モジュールにおける、入出力データのダンプ機能の有効/無効状態を保持するためのサーバー設定構造体です。

シグネチャまたは形状

typedef struct dumpio_conf_t {
    int enable_input;
    int enable_output;
} dumpio_conf_t;

入力と出力

  • **入力**: enable_input および enable_output は整数値（フラグ）として保持されます。
  • **出力**: 本構造体は、サーバー設定としてモジュール内で参照され、フィルタ処理の実行可否を決定するために使用されます。

前提条件と不変条件

  • mod_dumpio の設定として ap_get_module_config を介して取得されます。
  • apr_pcalloc によりメモリが確保される際、各フラグは初期化されます。

アルゴリズムまたは振る舞い

  • dumpio_input_filter および dumpio_output_filter 内で、現在の接続に対する設定として参照されます。
  • dumpit 関数に引数として渡され、ログ出力処理の制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • dumpio_input_filter (フィルタ処理内)
  • dumpio_output_filter (フィルタ処理内)
  • dumpio_create_sconfig (設定初期化時)
  • dumpio_enable_input / dumpio_enable_output (ディレクティブ設定時)
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • ap_filter_t: フィルタコンテキスト (f->ctx) として本構造体が格納されます。
  • server_rec: module_config を通じて本構造体と関連付けられます。

失敗モードと境界条件

  • 構造体ポインタが NULL の状態でフィルタ処理が実行された場合、モジュール内の参照箇所でクラッシュする可能性があります。
  • 設定が有効であっても、ログレベルが APLOG_TRACE7 未満の場合、実際のデータダンプは行われません。

保守担当者向けメモ

  • 本構造体は mod_dumpio の設定管理の基盤です。新しい設定項目を追加する場合は、dumpio_create_sconfig での初期化処理も併せて更新してください。
