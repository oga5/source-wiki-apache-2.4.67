dav_lock_get_activelock

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:42

目的

dav_lock 構造体のリストを、WebDAV の <activelock> XML 要素を含む文字列に変換します。主に PROPFIND レスポンスやロック情報の提示に使用されます。

シグネチャまたは形状

const char *dav_lock_get_activelock(request_rec *r, dav_lock *lock, dav_buffer *pbuf)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • lock: 変換対象の dav_lock 構造体リストの先頭。
  • pbuf: 出力用バッファ（NULL の場合は内部バッファを使用）。
  • **出力**:
  • 変換された XML 文字列を格納したバッファのポインタ（pbuf->buf）。

前提条件と不変条件

  • r->pool がメモリ割り当てに使用されます。
  • lock が NULL またはロックプロバイダーが未定義の場合、空文字列を返します。
  • lock->owner が存在する場合、それは名前空間宣言や xml:lang を含む完全な <DAV:owner> 要素である必要があります。

アルゴリズムまたは振る舞い

1. ロックリストを走査し、必要なバッファサイズを概算して dav_check_bufsize で確保します。 2. 各 dav_lock 要素に対し、以下の XML 構造を構築します。

  • <D:locktype>: DAV_LOCKTYPE_WRITE に対応する要素。
  • <D:lockscope>: DAV_LOCKSCOPE_EXCLUSIVE または SHARED に対応する要素。
  • <D:depth>: DAV_INFINITY なら infinity、それ以外は 0。
  • <D:owner>: 存在すればそのまま追加。
  • <D:timeout>: DAV_TIMEOUT_INFINITE または現在時刻との差分（Second-N 形式）。
  • <D:locktoken>: フック format_locktoken を介して生成されたトークン。

    • 3. 構築した XML 文字列をバッファに順次追加し、最終的な文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.c:3348
  • modules/dav/main/props.c:382
  • **呼び出し先**:
  • hooks->format_locktoken (ロックプロバイダーのフック)
  • dav_check_bufsize
  • dav_buffer_append

関連データ構造

  • dav_lock: ロック情報を保持する構造体。
  • dav_buffer: 文字列構築用の中間バッファ。
  • dav_hooks_locks: ロック操作用フック構造体。

失敗モードと境界条件

  • lock->rectype が DAV_LOCKREC_INDIRECT_PARTIAL の場合、DAV_DEBUG が有効であれば設計エラーとして警告メッセージが埋め込まれます。
  • now >= lock->timeout の場合、タイムアウト値として Second-0 が出力されます。

保守担当者向けメモ

  • dav_buffer を使用した効率的な文字列構築を行っています。
  • lock->owner の内容は検証されずそのまま出力されるため、呼び出し側で妥当な XML であることを保証する必要があります。
