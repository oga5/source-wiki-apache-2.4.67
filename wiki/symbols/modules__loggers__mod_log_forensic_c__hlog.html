hlog

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_log_forensic.c:141

目的

フォレンジックログの生成過程において、ログ文字列の構築、バッファ管理、およびメモリ割り当てを一時的に保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct hlog {
    char *log;          /* ログ出力用バッファの先頭ポインタ */
    char *pos;          /* 現在の書き込み位置 */
    char *end;          /* バッファの終端ポインタ */
    apr_pool_t *p;      /* メモリ割り当てに使用するプール */
    apr_size_t count;   /* ログの総サイズ */
} hlog;

入力と出力

  • 入力: ログ生成時に apr_pool_t を介してメモリを確保し、log メンバにバッファを割り当てます。
  • 出力: 構築されたログ文字列は log メンバに格納され、後続のファイル書き込み処理で使用されます。

前提条件と不変条件

  • log、pos、end の各ポインタは、バッファの有効範囲内（log <= pos <= end）にある必要があります。
  • p はリクエストのライフサイクルに紐付いた apr_pool_t であり、リクエスト終了時に自動的に解放されることを前提としています。

アルゴリズムまたは振る舞い

1. ログ出力に必要なサイズを事前に計算します。 2. apr_palloc を使用して必要なメモリを確保し、hlog 構造体を初期化します。 3. pos ポインタを更新しながら、エスケープ処理された文字列をバッファへ順次コピーします。 4. 最終的に log メンバが指すバッファ全体がログファイルへ書き込まれます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • modules/loggers/mod_log_forensic.c:161: hlog *h = h_;
  • modules/loggers/mod_log_forensic.c:170: hlog *h = h_;
  • modules/loggers/mod_log_forensic.c:187: hlog h;
  • 呼び出し先:
  • 構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • fcfg: 本モジュールの設定構造体。

失敗モードと境界条件

  • メモリ不足: apr_palloc が失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了またはエラーを返します。
  • バッファオーバーフロー: pos が end を超えないよう、log_escape 等の関数内で厳格な境界チェックが行われます。

保守担当者向けメモ

  • 本構造体はリクエスト単位のログ生成に特化しています。
  • count メンバは、ログ出力時のバッファサイズ計算結果を保持するために使用されます。
  • 内部リダイレクト等で r->prev が存在する場合、ログ生成処理自体がスキップされるため、本構造体がインスタンス化されないケースがあります。
