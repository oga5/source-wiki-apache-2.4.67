dav_fs_walker_context

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/repos.c:54

目的

WebDAV のディレクトリ走査（walk）処理において、走査中の状態を保持および管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    const dav_walk_params *params;
    dav_walk_resource wres;
    dav_resource res1;
    dav_resource_private info1;
    dav_buffer path1;
    dav_buffer uri_buf;
    dav_resource res2;
    dav_resource_private info2;
    dav_buffer path2;
    dav_buffer locknull_buf;
} dav_fs_walker_context;

入力と出力

  • **入力**: 走査の初期化時に dav_walk_params が設定されます。
  • **出力**: 走査の進行に伴い、wres（現在のリソース情報）や各バッファが更新され、コールバック関数へ渡されます。

前提条件と不変条件

  • 走査を開始する前に、params が有効な dav_walk_params を指している必要があります。
  • dav_buffer 型の各フィールドは、メモリ管理のために適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • ディレクトリ走査の再帰処理中に、現在のパス、リソース情報、および URI を保持します。
  • MOVE や COPY 操作時には、ソースとデスティネーションの両方のパス（path1, path2）およびリソース情報（res1, res2）を同時に管理し、処理の整合性を維持します。
  • 走査の深さに応じて、内部の dav_resource および dav_resource_private を動的に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: dav_fs_walker 関数（repos.c:1504）がこの構造体を引数として受け取ります。また、repos.c:1798 でインスタンスが初期化されます。
  • **呼び出し先**: 走査処理の過程で、params->func に指定されたコールバック関数が本構造体を参照します。

関連データ構造

  • dav_walk_params: 走査のパラメータ。
  • dav_walk_resource: 走査中のリソース状態。
  • dav_resource / dav_resource_private: WebDAV リソースの抽象化およびファイルシステム固有のプライベートデータ。
  • dav_buffer: パスや URI を格納するための可変長バッファ。

失敗モードと境界条件

  • メモリ不足: dav_buffer の拡張時にメモリ割り当てが失敗する可能性があります。
  • パス長制限: dav_buffer が保持するパスがシステム制限を超えた場合、処理が中断される可能性があります。

保守担当者向けメモ

  • 本構造体は MOVE/COPY 操作をサポートするためにセカンダリパス（path2 等）を保持しています。単一リソースの走査のみを行う場合は、これらのフィールドが使用されない可能性があります。
  • dav_buffer の管理には apr_pool_t が使用されているため、ライフサイクルは走査リクエストの生存期間に依存します。
