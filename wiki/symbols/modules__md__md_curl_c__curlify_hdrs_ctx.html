curlify_hdrs_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_curl.c:160

目的

HTTP リクエスト処理中にヘッダーリストを構築・変換するためのコンテキストを保持します。

シグネチャまたは形状

typedef struct {
    md_http_request_t *req;
    struct curl_slist *hdrs;
    apr_status_t rv;
} curlify_hdrs_ctx;

入力と出力

  • **入力**: md_http_request_t へのポインタ（リクエスト情報）、curl_slist（libcurl 用ヘッダーリスト）。
  • **出力**: 処理結果を示す apr_status_t 型の戻り値。

前提条件と不変条件

  • req は有効な md_http_request_t 構造体を指している必要があります。
  • hdrs は curl_slist のリスト構造を保持し、リクエスト終了時に適切に解放される必要があります。

アルゴリズムまたは振る舞い

HTTP リクエストのヘッダーを libcurl が扱える形式（curl_slist）へ変換するプロセスにおいて、変換中の状態（リクエストオブジェクト、ヘッダーリスト、エラーコード）を保持・追跡するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_curl.c 内のヘッダー構築処理（curlify_headers 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • md_http_request_t: HTTP リクエストの抽象化データ構造。
  • curl_slist: libcurl が使用するヘッダーリスト構造体。

失敗モードと境界条件

  • rv フィールドにより、ヘッダー変換処理中に発生した apr_status_t エラーを記録します。
  • メモリ不足等により curl_slist の構築に失敗した場合、rv にエラーコードが格納されます。

保守担当者向けメモ

  • 本構造体はリクエストのライフサイクルに紐付いています。リクエスト終了時には md_curl_req_cleanup 等を通じて、保持している hdrs（curl_slist）がメモリリークしないよう確実に解放してください。
