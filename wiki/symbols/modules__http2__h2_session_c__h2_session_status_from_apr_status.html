h2_session_status_from_apr_status

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:67

目的

Apache Portable Runtime (APR) のステータスコードを、nghttp2 ライブラリが期待する HTTP/2 プロトコルレベルのエラーコードへ変換する。

シグネチャまたは形状

int h2_session_status_from_apr_status(apr_status_t rv)

入力と出力

  • **入力**: apr_status_t rv（APR 操作の結果コード）
  • **出力**: int（nghttp2 定義のステータスコード）

前提条件と不変条件

  • 入力値 rv は APR の有効なステータスコードであること。
  • 戻り値は NGHTTP2_NO_ERROR、NGHTTP2_ERR_WOULDBLOCK、NGHTTP2_ERR_EOF、または NGHTTP2_ERR_PROTO のいずれかである。

アルゴリズムまたは振る舞い

以下のマッピングに従って変換を行う： 1. APR_SUCCESS の場合: NGHTTP2_NO_ERROR を返す。 2. APR_STATUS_IS_EAGAIN(rv) が真の場合: NGHTTP2_ERR_WOULDBLOCK を返す。 3. APR_STATUS_IS_EOF(rv) が真の場合: NGHTTP2_ERR_EOF を返す。 4. 上記以外の場合: NGHTTP2_ERR_PROTO を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_session.c:210 にて呼び出しを確認。
  • **呼び出し先**: 外部関数呼び出しなし。

関連データ構造

  • apr_status_t (APR)
  • nghttp2 ライブラリのステータス定数

失敗モードと境界条件

  • 未知の apr_status_t 値が渡された場合、デフォルトで NGHTTP2_ERR_PROTO を返す。

保守担当者向けメモ

  • 本関数は nghttp2 コールバック内でのエラーハンドリングを簡素化するために使用される。
  • 新たな APR エラーコードの処理が必要な場合は、条件分岐の追加を検討すること。
