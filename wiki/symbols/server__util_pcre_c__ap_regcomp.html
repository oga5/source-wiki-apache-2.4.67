ap_regcomp

定義

  • 種別: function
  • 定義箇所: server/util_pcre.c:202

目的

正規表現パターンをコンパイルし、HTTPD 共通の ap_regex_t 構造体を初期化する。PCRE1 または PCRE2 ライブラリをラップし、POSIX 互換のインターフェースを提供する。

シグネチャまたは形状

int ap_regcomp(ap_regex_t *preg, const char *pattern, int cflags)

入力と出力

  • **入力**:
  • preg: 初期化対象となる ap_regex_t 構造体へのポインタ。
  • pattern: コンパイル対象の正規表現文字列。
  • cflags: コンパイルオプション（AP_REG_ICASE, AP_REG_NEWLINE 等）。
  • **出力**:
  • 成功時: 0 を返す。
  • 失敗時: AP_REG_ESPACE（メモリ不足）または AP_REG_INVARG（不正な引数/パターン）を返す。

前提条件と不変条件

  • cflags に AP_REG_NO_DEFAULT が含まれていない場合、グローバルな default_cflags が自動的に適用される。
  • コンパイル成功時、preg->re_pcre には PCRE コンパイル済みオブジェクトが格納され、preg->re_nsub にキャプチャグループ数が設定される。

アルゴリズムまたは振る舞い

1. cflags に基づき、PCRE 固有のオプションフラグ（CASELESS, MULTILINE 等）を構築する。 2. PCRE バージョンに応じて pcre2_compile または pcre_compile2 を呼び出し、パターンをコンパイルする。 3. コンパイル結果が NULL の場合、エラーコードを判定して適切な HTTPD エラー値を返す。 4. コンパイル成功時、pcre2_pattern_info または pcre_fullinfo を使用してキャプチャグループ数を取得し、preg->re_nsub に保存する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_config.c
  • modules/lua/lua_request.c
  • modules/lua/mod_lua.c
  • server/util.c
  • server/util_regex.c
  • **呼び出し先**:
  • pcre2_compile / pcre_compile2 (PCRE ライブラリ)
  • pcre2_pattern_info / pcre_fullinfo (PCRE ライブラリ)

関連データ構造

  • ap_regex_t: 正規表現のコンパイル済み状態を保持する構造体。
  • ap_regmatch_t: マッチ結果を格納するための構造体（include/ap_regex.h で定義）。

失敗モードと境界条件

  • **メモリ不足**: PCRE が内部エラーコード 21 を返した場合、AP_REG_ESPACE を返す。
  • **不正なパターン**: コンパイルに失敗した場合は AP_REG_INVARG を返す。エラー発生位置は preg->re_erroffset に格納される。

保守担当者向けメモ

  • 本関数は PCRE1 と PCRE2 の差異を吸収するラッパーである。PCRE2 使用時は pcre2_compile の第 6 引数に NULL を渡しており、コンテキストはデフォルト設定が使用される。
  • ap_regcomp で確保されたリソースは、必ず ap_regfree を呼び出して解放する必要がある。
