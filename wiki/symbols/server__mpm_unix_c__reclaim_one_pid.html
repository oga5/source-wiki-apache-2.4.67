reclaim_one_pid

定義

  • 種別: function
  • 定義箇所: server/mpm_unix.c:119

目的

指定された子プロセスの終了状態を監視し、必要に応じてシグナル（SIGTERM または SIGKILL）を送信してプロセスの回収を試みます。

シグネチャまたは形状

static int reclaim_one_pid(pid_t pid, action_t action)

入力と出力

  • **入力**:
  • pid: 回収対象のプロセスID。
  • action: プロセスが終了していない場合に実行するアクション（action_t 型）。
  • **出力**:
  • int: プロセスが既に終了しているか、アクションの結果として回収処理が完了したとみなされる場合は 1 を返します。プロセスがまだ生存しており、かつアクションが DO_NOTHING 等で即時終了を強制しない場合は 0 を返します。

前提条件と不変条件

  • pid は 1 以上である必要があります（1 未満の場合は即座に 1 を返します）。
  • apr_proc_wait を使用して非ブロッキング（APR_NOWAIT）で状態を確認します。

アルゴリズムまたは振る舞い

1. apr_proc_wait を呼び出し、対象プロセスの状態を確認します。 2. プロセスが既に終了している場合（APR_CHILD_DONE）、ap_process_child_status を呼び出して終了ステータスを処理し、1 を返します。 3. プロセスが生存している場合、引数 action に基づき以下のいずれかを行います：

  • DO_NOTHING: 何もせず 0 を返します。
  • SEND_SIGTERM / SEND_SIGTERM_NOLOG: SIGTERM を送信し、0 を返します。
  • SEND_SIGKILL: SIGKILL を送信し、0 を返します。
  • GIVEUP: ログ出力のみを行い、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm_unix.c 内のプロセス回収ループ（ap_reclaim_child_processes や ap_relieve_child_processes に関連する箇所）から呼び出されます。
  • **呼び出し先**:
  • apr_proc_wait (APR)
  • ap_process_child_status
  • ap_log_error
  • kill (システムコール)

関連データ構造

  • pid_t: プロセスID型。
  • action_t: プロセス回収時の挙動を定義する列挙型（DO_NOTHING, SEND_SIGTERM, SEND_SIGTERM_NOLOG, SEND_SIGKILL, GIVEUP）。
  • apr_proc_t: APRのプロセス管理構造体。

失敗モードと境界条件

  • pid < 1: 不正なPIDとして無視され、1 が返されます。
  • apr_proc_wait が APR_CHILD_NOTDONE 以外を返した場合、プロセスは終了済みとみなされ、それ以上のシグナル送信は行われません。
  • kill システムコールが失敗した場合の戻り値は本関数内では評価されません。

保守担当者向けメモ

  • 本関数は server/mpm_unix.c 内のプロセス回収ロジックの末端を担います。
  • action の各ケースにおいて、ログ出力の有無やシグナルの種類が厳密に定義されているため、新しいアクションを追加する際は action_t の定義と併せて確認してください。
