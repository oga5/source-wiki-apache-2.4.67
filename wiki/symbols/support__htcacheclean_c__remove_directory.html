remove_directory

定義

  • 種別: function
  • 定義箇所: support/htcacheclean.c:1087

目的

指定されたディレクトリを再帰的に走査し、内部の全ファイルおよびサブディレクトリを削除した上で、対象ディレクトリ自体を削除する。

シグネチャまたは形状

apr_status_t remove_directory(apr_pool_t *pool, const char *dir)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する APR プール。
  • dir: 削除対象となるディレクトリのパス。
  • **出力**:
  • apr_status_t: 処理結果。成功時は APR_SUCCESS を返す。

前提条件と不変条件

  • apr_pool_t が有効であること。
  • 呼び出し元は、ディレクトリの削除が失敗した場合のログ出力やエラーハンドリングを考慮する必要がある。

アルゴリズムまたは振る舞い

1. apr_dir_open でディレクトリを開く。存在しない場合は APR_SUCCESS 相当のステータスを返し終了する。 2. apr_dir_read を使用してディレクトリ内のエントリを順次読み込む。 3. エントリがディレクトリ（. および .. を除く）である場合、自身を再帰的に呼び出す。 4. エントリがファイルである場合、apr_file_remove で削除する。 5. ディレクトリ内の全エントリ処理後、apr_dir_close でディレクトリを閉じる。 6. 最後に apr_dir_remove で対象ディレクトリ自体を削除する。

  • 削除時に APR_ENOTEMPTY が発生した場合は、ディレクトリが再作成された可能性があるとみなし、成功として扱う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/htcacheclean.c:1107 (再帰呼び出し)
  • support/htcacheclean.c:1214
  • **呼び出し先**:
  • apr_dir_open, apr_dir_read, apr_dir_close, apr_dir_remove (APR ファイルシステム操作)
  • apr_file_remove (ファイル削除)
  • apr_pstrcat (パス文字列生成)
  • apr_file_printf (エラー出力)

関連データ構造

  • apr_dir_t: ディレクトリ操作用ハンドル。
  • apr_finfo_t: ファイル情報構造体。

失敗モードと境界条件

  • **ディレクトリ不在**: APR_STATUS_IS_ENOENT(rv) が真の場合、エラーとみなさず正常終了する。
  • **削除失敗**: ファイルやディレクトリの削除が権限不足等で失敗した場合、apr_file_printf でエラーを出力し、その時点の rv を返して処理を中断する。
  • **ディレクトリ非空**: apr_dir_remove 時に APR_ENOTEMPTY が返された場合、キャッシュの再作成と競合したと判断し、エラーを無視して成功を返す。

保守担当者向けメモ

  • 本関数は再帰的に動作するため、深いディレクトリ構造に対してはスタック消費量に注意が必要。
  • apr_pool_t を介したメモリ管理を行っているため、呼び出し元で適切なプール生存期間が確保されていることを確認すること。
