h2_request_end_headers

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:198

目的

HTTP/2 リクエストのヘッダー受信完了後、Host ヘッダーの整合性を検証および補完し、リクエストのバイト数を更新する。

シグネチャまたは形状

apr_status_t h2_request_end_headers(h2_request *req, apr_pool_t *pool, size_t raw_bytes)

入力と出力

  • **入力**:
  • req: 処理対象の h2_request 構造体。
  • pool: メモリ割り当てに使用する apr_pool_t。
  • raw_bytes: 受信したヘッダーのバイト数。
  • **出力**:
  • APR_SUCCESS: 正常終了。
  • APR_BADARG: Host ヘッダーが欠落しており、かつ :authority 擬似ヘッダーも存在しない場合。

前提条件と不変条件

  • req->headers テーブルが初期化されていること。
  • RFC 7540 8.1.2.3 に従い、:authority 擬似ヘッダーが存在しない場合は Host ヘッダーが必須となる。

アルゴリズムまたは振る舞い

1. req->authority が空文字列であれば NULL にリセットする。 2. req->authority が存在しない場合、req->headers から Host ヘッダーを取得する。取得できなければ APR_BADARG を返す。 3. req->authority が存在する場合、その値を req->headers の Host ヘッダーとして設定する。 4. req->raw_bytes に引数 raw_bytes を加算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_push.c:354
  • modules/http2/h2_stream.c:872
  • **呼び出し先**:
  • apr_table_get (APR)
  • apr_table_setn (APR)

関連データ構造

  • h2_request: HTTP/2 リクエスト表現を保持する構造体。
  • apr_table_t: ヘッダーを格納する APR テーブル。

失敗モードと境界条件

  • :authority 擬似ヘッダーおよび Host ヘッダーの両方が欠落している場合、APR_BADARG を返して処理を中断する。

保守担当者向けメモ

  • 本関数はヘッダーの最終的な整合性チェックを行うため、リクエスト処理パイプラインへ投入される直前の段階で呼び出される必要がある。
  • req->authority の更新は apr_table_setn を使用しており、テーブル内の既存の Host ヘッダーを上書きする点に注意すること。
