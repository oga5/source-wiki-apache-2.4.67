mk_acct_id

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_acct.c:60

目的

ACME アカウントを一意に識別するための文字列 ID を生成します。

シグネチャまたは形状

const char *mk_acct_id(apr_pool_t *p, md_acme_t *acme, int i);

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • acme: ACME サーバーとの通信セッション状態を保持する md_acme_t 構造体。
  • i: アカウントのインデックス番号。
  • **出力**:
  • 生成されたアカウント ID 文字列（ACME-<servername>-<index> 形式）。

前提条件と不変条件

  • acme->sname が有効なサーバー名文字列を保持している必要があります。
  • 生成される ID は ACME-<servername>-<04d> の形式に従い、account.json および account.pem のファイル名管理と整合します。

アルゴリズムまたは振る舞い

1. apr_psprintf を使用して、指定されたプール p 上にフォーマット済み文字列を生成します。 2. 文字列は ACME- を接頭辞とし、acme->sname（サーバー名）と i（4桁のゼロ埋め整数）を連結した形式となります。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_acct.c:191 にて呼び出されています。
  • **呼び出し先**: apr_psprintf (APR ライブラリ関数)。

関連データ構造

  • md_acme_t: サーバー名 (sname) を参照するために使用されます。
  • apr_pool_t: メモリ管理に使用されます。

失敗モードと境界条件

  • apr_psprintf の内部的なメモリ割り当て失敗時に、APR の規約に従いプロセスが終了またはエラーを返します。
  • acme->sname が NULL の場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はアカウントの永続化パスを決定する重要な役割を担います。ID フォーマットを変更する場合は、既存のストレージ上のファイル名との整合性に注意してください。
