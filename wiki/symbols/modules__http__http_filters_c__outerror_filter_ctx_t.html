outerror_filter_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/http/http_filters.c:1881

目的

出力フィルタにおいて、エラー発生時の状態を追跡し、接続の終了やエラー処理を制御するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    int seen_eoc;    /* End of Connection (EOC) バケットの検出フラグ */
    int first_error; /* 最初に発生したエラーの有無または状態 */
} outerror_filter_ctx_t;

入力と出力

  • **入力**: フィルタのコンテキスト (f->ctx) として保持されます。
  • **出力**: フィルタ処理中に seen_eoc や first_error が更新され、後続のフィルタ処理やエラーハンドリングの判断基準として利用されます。

前提条件と不変条件

  • フィルタの実行時に f->ctx が NULL の場合、apr_pcalloc を用いて本構造体が初期化されます。
  • seen_eoc は、接続終了を示すバケットが処理されたことを示すフラグとして機能します。

アルゴリズムまたは振る舞い

1. フィルタ呼び出し時に f->ctx を outerror_filter_ctx_t 型にキャストして取得します。 2. f->ctx が未初期化であれば、リクエストプール (r->pool) からメモリを確保し初期化します。 3. フィルタチェーンを流れるバケットを監視し、EOC バケットの通過やエラー発生時に、構造体内のフラグを更新して状態を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_filters.c 内の出力フィルタ関数（具体的な関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: apr_pcalloc (メモリ確保)。

関連データ構造

  • ap_filter_t: 本構造体がコンテキストとして格納されるフィルタ構造体。
  • apr_bucket: フィルタチェーン内で処理されるデータ単位。

失敗モードと境界条件

  • メモリ確保失敗時: apr_pcalloc が失敗した場合の挙動は、呼び出し元のエラーハンドリングに依存します。
  • 状態の不整合: フィルタが複数回呼び出される際、seen_eoc が適切にリセットまたは維持されない場合、誤ったエラー判定が行われる可能性があります。

保守担当者向けメモ

  • 本構造体は modules/http/http_filters.c 内の特定の出力フィルタ専用です。
  • seen_eoc や first_error の具体的な判定ロジックは、本構造体を利用するフィルタ関数の実装に依存するため、変更時は当該フィルタのロジックを併せて確認してください。
