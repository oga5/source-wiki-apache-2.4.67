ajp_msg_append_uint32

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_msg.c:260

目的

AJPメッセージのバッファ末尾に、32ビット符号なし整数をネットワークバイトオーダー（ビッグエンディアン）で追加します。

シグネチャまたは形状

apr_status_t ajp_msg_append_uint32(ajp_msg_t *msg, apr_uint32_t value)

入力と出力

  • **入力**:
  • msg: 操作対象の ajp_msg_t 構造体へのポインタ。
  • value: 追加する apr_uint32_t 型の整数値。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • バッファ容量不足時は ajp_log_overflow の戻り値（エラーコード）を返します。

前提条件と不変条件

  • msg は有効な ajp_msg_t 構造体を指している必要があります。
  • msg->len は現在のバッファ書き込み位置を示し、この関数によって 4 バイト増加します。

アルゴリズムまたは振る舞い

1. 現在のメッセージ長 msg->len が msg->max_size を超えないか検証します。 2. 超過する場合は ajp_log_overflow を呼び出し、エラーを返して終了します。 3. value を 8 ビットずつシフトし、ネットワークバイトオーダー（ビッグエンディアン）の順序で msg->buf の msg->len 以降の 4 バイトに格納します。 4. msg->len を 4 加算し、更新します。 5. APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は提供されていません。
  • **呼び出し先**:
  • ajp_log_overflow (エラー発生時)

関連データ構造

  • ajp_msg_t: AJPメッセージを保持する構造体。buf（バイト配列）、len（現在の長さ）、max_size（最大容量）を含みます。

失敗モードと境界条件

  • **バッファオーバーフロー**: msg->len + 4 が msg->max_size を超える場合、データは書き込まれず、エラーが返されます。

保守担当者向けメモ

  • 本関数はネットワークバイトオーダーへの変換を明示的に行っています。プラットフォームのエンディアンに関わらず、常にビッグエンディアンで書き込まれます。
