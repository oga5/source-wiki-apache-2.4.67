merge_auth_basic_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_basic.c:58

目的

ディレクトリ単位の Basic 認証設定（auth_basic_config_rec）をマージし、親ディレクトリの設定と現在のディレクトリの設定を統合した新しい設定構造体を生成します。

シグネチャまたは形状

static void *merge_auth_basic_dir_config(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • basev: 親ディレクトリの設定（auth_basic_config_rec）。
  • overridesv: 現在のディレクトリで上書き指定された設定（auth_basic_config_rec）。
  • **出力**:
  • 統合された新しい auth_basic_config_rec 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は auth_basic_config_rec 型としてキャスト可能であること。
  • メモリ割り当ては apr_pcalloc を使用して行われ、ゼロ初期化された領域が返されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を用いて新しい設定構造体 newconf を確保します。 2. 各設定項目について、overrides 側で設定が明示されている（_set フラグが真）場合はその値を、そうでない場合は base の値を採用します。

  • authoritative: 認証の権限委譲フラグ。
  • fakeuser / fakepass: 認証情報の偽装設定。
  • use_digest_algorithm: ダイジェストアルゴリズムの使用設定。

    • 3. providers リストについては、overrides に設定が存在すればそれを、なければ base のリストを継承します。 4. 最終的に統合された newconf を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_basic.c:503（Apache の設定マージ処理の一部として呼び出されます）。
  • **呼び出し先**: 直接的な関数呼び出しは行いません。

関連データ構造

  • auth_basic_config_rec: 本関数がマージ対象とするディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボートする可能性があります（Apache の標準的なメモリ管理動作）。
  • overrides が空（NULL）の場合の挙動については、本ソースコード内では basev を base にキャストする際に考慮されていますが、呼び出し側が適切に初期化済みであることを前提としています。

保守担当者向けメモ

  • 本関数は Apache の設定マージフェーズで実行されます。新しい設定項目を auth_basic_config_rec に追加した場合は、必ず本関数内でマージロジックを更新してください。
