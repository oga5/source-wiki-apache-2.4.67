socache_rd_id2key

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_redis.c:196

目的

バイナリ形式のキャッシュ ID を、Redis サーバーで利用可能な文字列形式のキーへ変換します。

シグネチャまたは形状

int socache_rd_id2key(ap_socache_instance_t *ctx,
                      const unsigned char *id, unsigned int idlen,
                      char *key, apr_size_t keylen)

入力と出力

  • **入力**:
  • ctx: Redis インスタンスのコンテキスト（名前空間タグ tag を保持）。
  • id: 変換対象のバイナリ ID。
  • idlen: id のバイト長。
  • key: 変換後の文字列を格納するバッファ。
  • keylen: key バッファのサイズ。
  • **出力**:
  • 成功時: 0 を返します。
  • 失敗時: 1 を返します。

前提条件と不変条件

  • key バッファは、id を 16 進数文字列に変換した長さ（idlen * 2）と、ctx->tag の長さ（ctx->taglen）の合計を格納するのに十分なサイズである必要があります。

アルゴリズムまたは振る舞い

1. idlen * 2 + ctx->taglen が keylen を超える場合、変換を中止し 1 を返します。 2. ctx->tag を key バッファの先頭にコピーします。 3. id を 16 進数文字列に変換し、key の末尾に連結します。 4. 正常終了として 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_socache_redis.c:221 (socache_rd_store)
  • modules/cache/mod_socache_redis.c:250 (socache_rd_retrieve)
  • modules/cache/mod_socache_redis.c:284 (socache_rd_remove)
  • **呼び出し先**:
  • apr_cpystrn (APR 文字列コピー関数)
  • ap_bin2hex (バイナリから 16 進数文字列への変換関数)

関連データ構造

  • ap_socache_instance_t: Redis インスタンスのコンテキストを保持し、名前空間タグ（tag）とタグ長（taglen）を提供します。

失敗モードと境界条件

  • 指定されたバッファサイズ keylen が不足している場合、関数は 1 を返し、キーの生成を行いません。

保守担当者向けメモ

  • 本関数は id を 16 進数化するため、生成されるキーの長さは元の ID の 2 倍になります。バッファサイズを計算する際は、名前空間タグの長さを含めて考慮してください。
