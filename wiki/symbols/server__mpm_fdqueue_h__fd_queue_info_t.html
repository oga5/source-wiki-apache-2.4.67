fd_queue_info_t

定義

  • 種別: struct
  • 定義箇所: server/mpm_fdqueue.h:45

目的

Apache HTTP Server の MPM（event および worker）において、ワーカースレッドのアイドル状態管理および再利用可能なメモリプール（apr_pool_t）の管理を行うための制御構造体です。

シグネチャまたは形状

不透明な構造体として宣言されています。

typedef struct fd_queue_info_t fd_queue_info_t;

※構造体の内部メンバは server/mpm_fdqueue.c 内で定義されており、ヘッダーファイル上では不透明型として扱われます。

入力と出力

本構造体は、以下の関数群の引数として渡され、状態の更新や取得に使用されます。

  • **入力**: 各種 ap_queue_info_* 関数に対し、管理対象のインスタンスとして渡されます。
  • **出力**: ap_queue_info_num_idlers 等の関数を通じて、現在のアイドルスレッド数などの状態を返します。

前提条件と不変条件

  • APR_HAS_THREADS が定義されている環境でのみ使用可能です。
  • 本構造体へのアクセスは、内部で保持するミューテックスや条件変数によってスレッドセーフに保護される必要があります。

アルゴリズムまたは振る舞い

  • **アイドル管理**: ワーカースレッドが待機状態に入る際、ap_queue_info_wait_for_idler を通じて待機し、処理可能な状態になると ap_queue_info_set_idle でプールを再利用リストへ戻します。
  • **プール管理**: ワーカースレッド間で再利用可能な apr_pool_t のスタックを管理し、メモリ割り当てのオーバーヘッドを削減します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • server/mpm/event/event.c: worker_queue_info として保持。
  • server/mpm/worker/worker.c: worker_queue_info として保持。
  • **呼び出し先（操作関数）**:
  • ap_queue_info_create (生成)
  • ap_queue_info_set_idle / ap_queue_info_try_get_idler / ap_queue_info_wait_for_idler (スレッド制御)
  • ap_queue_info_num_idlers (状態取得)
  • ap_queue_info_push_pool / ap_queue_info_pop_pool / ap_queue_info_free_idle_pools (プール管理)

関連データ構造

  • fd_queue_t: ソケットやタイマーイベントを保持するメインのキュー構造体。
  • apr_pool_t: 再利用されるメモリプール。

失敗モードと境界条件

  • キューが空の状態で ap_queue_info_wait_for_idler が呼び出された場合、条件変数によりスレッドは待機状態となります。
  • メモリ不足や同期プリミティブの初期化失敗時には、ap_queue_info_create がエラーコードを返します。

保守担当者向けメモ

  • 本構造体は server/mpm_fdqueue.c で実装されています。変更を加える際は、one_big_mutex によるロック競合がパフォーマンスに与える影響を考慮してください。
  • 構造体のメンバ定義はヘッダーではなく実装ファイル側に隠蔽されているため、詳細なフィールド構成を確認する場合は server/mpm_fdqueue.c を参照してください。
