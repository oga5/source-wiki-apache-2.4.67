log_script

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgi.c:189

目的

CGI スクリプトの実行結果（リクエストヘッダー、POSTデータ、レスポンスヘッダー、標準出力、標準エラー出力）をデバッグログファイルに記録する。

シグネチャまたは形状

static int log_script(request_rec *r, cgi_server_conf *conf, int ret,
                      char *dbuf, const char *sbuf, apr_bucket_brigade *bb,
                      apr_file_t *script_err)

入力と出力

  • **入力**:
  • r: リクエストレコード。
  • conf: サーバー設定（ログパス、サイズ制限等）。
  • ret: HTTP ステータスコード。
  • dbuf: POST/PUT データ。
  • sbuf: レスポンスヘッダー情報。
  • bb: スクリプトの標準出力を含むバケットブリゲード。
  • script_err: スクリプトの標準エラー出力ファイルハンドル。
  • **出力**:
  • 戻り値: 引数として受け取った ret をそのまま返す。

前提条件と不変条件

  • conf->logname が設定されている必要がある。
  • ログファイルへの書き込み権限およびディスク容量が確保されていること。
  • bb（バケットブリゲード）は本関数内で破棄される。

アルゴリズムまたは振る舞い

1. **ログ可否判定**: ログパスが未設定、またはログファイルがサイズ制限を超過している場合、ログ出力をスキップして終了する。 2. **ヘッダー記録**: リクエスト日時、メソッド、URI、プロトコル、ステータスコード、リクエストヘッダーをログファイルに書き出す。 3. **ボディ記録**: POST/PUT データが存在すれば書き出す。 4. **レスポンス記録**: レスポンスヘッダーを書き出す。 5. **出力記録**: bb からバケットを読み取り、標準出力の内容をログに追記する。 6. **エラー記録**: script_err から標準エラー出力を読み取り、ログに追記する。 7. **リソース解放**: bb の破棄、script_err およびログファイルのクローズを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/cgi_common.h 内のロジックから呼び出される。
  • **呼び出し先**:
  • discard_script_output (ログ出力不可時のクリーンアップ)
  • log_script_err (エラー出力の記録)
  • APR ファイル操作関数群 (apr_file_open, apr_file_printf, apr_file_puts, apr_file_write, apr_file_close 等)
  • APR バケット操作関数群 (apr_bucket_read, apr_brigade_destroy 等)

関連データ構造

  • request_rec: Apache リクエスト情報。
  • cgi_server_conf: CGI モジュールのサーバー設定。
  • apr_bucket_brigade: スクリプトの出力を保持するバケットの集合。
  • apr_file_t: ログファイルおよび標準エラー出力のハンドル。

失敗モードと境界条件

  • ログファイルが開けない、または書き込みに失敗した場合、ログ出力は行われず、discard_script_output を経由してスクリプト出力を破棄した後に元のステータスコードを返す。
  • ログファイルが設定された logbytes を超えた場合、それ以上のログ記録は行われない。

保守担当者向けメモ

  • 本関数はリクエスト処理のメインラインで実行されるため、非常に高コストである。本番環境での有効化には注意が必要。
