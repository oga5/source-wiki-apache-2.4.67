dumpit

定義

  • 種別: function
  • 定義箇所: modules/debugging/mod_dumpio.c:54

目的

Apache HTTP Server のフィルタチェーンを流れるバケットの内容を傍受し、APLOG_TRACE7 レベルでエラーログへダンプする。

シグネチャまたは形状

static void dumpit(ap_filter_t *f, apr_bucket *b, dumpio_conf_t *ptr)

入力と出力

  • **入力**:
  • f: 現在のフィルタコンテキスト (ap_filter_t)。
  • b: ダンプ対象のバケット (apr_bucket)。
  • ptr: モジュールの設定構造体 (dumpio_conf_t)。
  • **出力**: なし（戻り値は void）。副作用としてログ出力を行う。

前提条件と不変条件

  • f および b は有効なポインタであること。
  • APLOG_TRACE7 レベルのログ出力が有効であること。
  • EBCDIC 環境では apr_xlate_proto_from_ascii による変換が行われる。

アルゴリズムまたは振る舞い

1. バケットのメタデータ情報（フィルタ名、バケットタイプ、サイズ）をログ出力する。 2. バケットがメタデータでない場合、apr_bucket_read を用いてデータを読み取る。 3. 読み取り成功時、dumpio_MAX_STRING_LEN を上限としてデータを分割し、ログ出力を行う。

  • EBCDIC 環境では変換後にログ出力する。
  • それ以外の環境ではバイナリデータをそのままログ出力する。

    • 4. 読み取り失敗時は、エラーメッセージをログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/debugging/mod_dumpio.c:146 (dumpio_input_filter 内)
  • modules/debugging/mod_dumpio.c:174 (dumpio_output_filter 内)
  • **呼び出し先**:
  • ap_log_cerror (ログ出力)
  • apr_bucket_read (バケットデータ取得)
  • ap_xlate_proto_from_ascii (EBCDIC 環境のみ)

関連データ構造

  • ap_filter_t: フィルタチェーンのノード。
  • apr_bucket: フィルタチェーンを流れるデータ単位。
  • dumpio_conf_t: モジュールの設定保持用構造体。

失敗モードと境界条件

  • apr_bucket_read が失敗した場合、エラーログに error reading data と出力される。
  • バイナリデータ内に \0（ヌル文字）が含まれる場合、ログ出力時に適切にエスケープされない可能性がある。
  • dumpio_MAX_STRING_LEN を超えるデータは、ループ処理により分割して出力される。

保守担当者向けメモ

  • 本関数は APLOG_TRACE7 を使用するため、本番環境でのパフォーマンス低下に注意が必要。
  • ソース内のコメントにある通り、バイナリデータ内のヌル文字の扱いについては将来的な改善が示唆されている。
