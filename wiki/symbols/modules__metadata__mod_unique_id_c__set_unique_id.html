set_unique_id

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_unique_id.c:293

目的

HTTPリクエストに対して一意な識別子（UNIQUE_ID）を生成し、リクエストの環境変数 subprocess_env に格納する。

シグネチャまたは形状

static int set_unique_id(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のHTTPリクエスト構造体）
  • **出力**: int （常に DECLINED を返す）

前提条件と不変条件

  • 内部リダイレクトが発生している場合、既存の REDIRECT_UNIQUE_ID を優先して使用する。
  • generate_log_id() によって既に r->log_id が設定されている場合は、その値を再利用する。
  • サブクエストに対しては呼び出されない前提で設計されている。

アルゴリズムまたは振る舞い

1. リクエストが内部リダイレクト (r->prev) である場合、subprocess_env から REDIRECT_UNIQUE_ID を取得を試みる。 2. IDが未設定の場合、r->log_id が存在すればそれを採用する。 3. 上記のいずれでもIDが取得できない場合、gen_unique_id(r) を呼び出して新規IDを生成する。 4. 確定したIDを apr_table_setn を用いて subprocess_env の UNIQUE_ID キーに格納する。 5. 処理終了後、DECLINED を返して後続のモジュールへ処理を継続させる。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_read_request に登録されており、Apacheの post_read_request フックから呼び出される。
  • **呼び出し先**:
  • apr_table_get / apr_table_setn (APRテーブル操作)
  • gen_unique_id (ID生成)

関連データ構造

  • request_rec: リクエスト情報を保持する構造体。subprocess_env および log_id フィールドを使用。

失敗モードと境界条件

  • ID生成の失敗に関する明示的なエラーハンドリングは記述されていない。
  • 内部リダイレクト時以外は REDIRECT_UNIQUE_ID を参照しない。

保守担当者向けメモ

  • DECLINED を返すことで、他のモジュールが同様のフックで処理を行うことを妨げない設計となっている。
