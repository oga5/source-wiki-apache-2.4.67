passphrase_ui_write

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:777

目的

OpenSSL の UI_METHOD における書き込みコールバックとして、パスフレーズ入力用のプロンプトを適切な出力先（標準出力またはパイプ）へ転送します。

シグネチャまたは形状

static int passphrase_ui_write(UI *ui, UI_STRING *uis)

入力と出力

  • **入力**:
  • ui: OpenSSL の UI 構造体。ユーザーデータとして pphrase_cb_arg_t を保持。
  • uis: 書き込み対象の文字列情報を含む UI_STRING 構造体。
  • **出力**:
  • 戻り値: 常に 1 を返します。

前提条件と不変条件

  • UI_get0_user_data(ui) を通じて、有効な pphrase_cb_arg_t 構造体が取得可能であること。
  • writetty がグローバルまたはスコープ内で適切に初期化されていること（ソースコード上の暗黙の依存）。

アルゴリズムまたは振る舞い

1. UI_get0_user_data を呼び出し、コンテキスト情報 pphrase_cb_arg_t を取得します。 2. サーバー設定（SSLSrvConfigRec）を確認し、ダイアログタイプが SSL_PPTYPE_BUILTIN または SSL_PPTYPE_PIPE であるか判定します。 3. 条件に合致する場合、UI_get0_output_string(uis) でプロンプト文字列を取得し、apr_file_puts を用いて writetty へ出力します。 4. 処理の成否に関わらず 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: OpenSSL の UI ライブラリ（UI_method_set_writer を介して登録されたコールバックとして実行）。
  • **呼び出し先**:
  • UI_get0_user_data (OpenSSL API)
  • mySrvConfig (内部マクロ)
  • UI_get0_output_string (OpenSSL API)
  • apr_file_puts (APR API)

関連データ構造

  • pphrase_cb_arg_t: パスフレーズ収集時の状態を保持するコンテキスト。
  • SSLSrvConfigRec: サーバー設定を保持する構造体。
  • UI, UI_STRING: OpenSSL の対話型 UI 用構造体。

失敗モードと境界条件

  • sc->server->pphrase_dialog_type が上記以外の値である場合、プロンプトの出力はスキップされます。
  • writetty が無効な場合、apr_file_puts の動作は APR の仕様に依存します。

保守担当者向けメモ

  • 本関数は OpenSSL の UI_METHOD インターフェースの一部です。writetty は本ファイル内の静的変数として管理されているため、並列実行時の競合や初期化順序には注意が必要です。
  • 戻り値は常に 1 ですが、OpenSSL の UI API 仕様に準拠した実装となっています。
