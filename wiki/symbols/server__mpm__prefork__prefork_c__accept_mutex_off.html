accept_mutex_off

定義

  • 種別: function
  • 定義箇所: server/mpm/prefork/prefork.c:262

目的

accept() 呼び出し時に使用したプロセス間ミューテックスを解放し、他の子プロセスが接続を受け入れられる状態にする。

シグネチャまたは形状

apr_status_t accept_mutex_off(void)

入力と出力

  • 入力: なし（グローバルな my_bucket を参照）
  • 出力: apr_status_t 型。成功時は APR_SUCCESS を返す。

前提条件と不変条件

  • my_bucket が初期化されており、有効なミューテックスを保持していること。
  • 本関数は accept() 処理の直後に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. my_bucket->mutex に対して apr_proc_mutex_unlock を実行する。 2. 解放に失敗した場合:

  • 現在のプロセス世代（retained->mpm->my_generation）とスコアボード上の実行世代が異なる場合は、デバッグログを出力して処理を継続する（接続処理後に終了するため）。
  • 世代が一致している場合は、緊急ログを出力し、APEXIT_CHILDFATAL でプロセスを強制終了する。

    • 3. 成功時は APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/prefork/prefork.c 内の SAFE_ACCEPT マクロ経由で呼び出される（591行目、613行目、647行目）。
  • 呼び出し先: apr_proc_mutex_unlock (APRライブラリ関数)。
  • 注記: test/time-sem.c に同名の関数が存在するが、これはテスト用であり本シンボルとは別物である。

関連データ構造

  • prefork_child_bucket: ミューテックスを保持する構造体。
  • prefork_retained_data: プロセスの世代管理に使用されるデータ構造。

失敗モードと境界条件

  • apr_proc_mutex_unlock が APR_SUCCESS 以外を返した場合、プロセスが致命的エラーとして終了する可能性がある。
  • 世代交代が発生している最中の呼び出しでは、エラーログの出力レベルが DEBUG に抑制される。

保守担当者向けメモ

  • 本関数は SAFE_ACCEPT マクロを通じて呼び出されるため、直接呼び出し箇所を検索する際はマクロの展開に注意すること。
  • 世代管理の不一致による終了回避ロジックは、グレースフル再起動時の挙動に依存している。
