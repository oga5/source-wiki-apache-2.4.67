merge_magic_server_config

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_mime_magic.c:479

目的

Apache HTTP Server の設定マージ処理において、親コンテキスト（basev）と子コンテキスト（addv）の magic_server_config_rec を統合し、新しい設定構造体を生成する。

シグネチャまたは形状

static void *merge_magic_server_config(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親コンテキストの magic_server_config_rec へのポインタ。
  • addv: 子コンテキストの magic_server_config_rec へのポインタ。
  • **出力**:
  • 統合された新しい magic_server_config_rec 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は magic_server_config_rec 型としてキャスト可能であること。
  • p は有効な Apache メモリプールであること。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、新しい magic_server_config_rec 構造体用のメモリを確保する。 2. magicfile パスについて、add（子）の設定が定義されていればそれを採用し、未定義であれば base（親）の設定を継承する。 3. 新しい構造体の magic リストおよび last ポインタを NULL に初期化する（継承せず、再構築を前提とする）。 4. 統合された構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_mime_magic.c:2468（サーバー設定のマージ処理として登録されている）。
  • **呼び出し先**: 直接的な関数呼び出しは行わないが、Apache APR の apr_palloc を使用する。

関連データ構造

  • magic_server_config_rec: サーバー単位の設定を保持する構造体。magicfile パスやマジックナンバーのルールリスト（magic, last）を管理する。

失敗モードと境界条件

  • メモリ不足時: apr_palloc が失敗した場合、Apache の標準的なメモリ管理ポリシーに従いプロセスが終了する可能性がある。
  • basev または addv が NULL の場合、キャストおよびメンバアクセス時に未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は magic リストを継承せず NULL に初期化するため、マージ後に別途 apprentice() 等による再読み込みや再構築が必要となる設計である点に注意すること。
