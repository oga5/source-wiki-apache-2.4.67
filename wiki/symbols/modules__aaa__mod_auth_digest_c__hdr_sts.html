hdr_sts

定義

  • 種別: enum
  • 定義箇所: modules/aaa/mod_auth_digest.c:140

目的

HTTP Digest 認証における Authorization ヘッダーの解析状態を表現するために使用されます。

シグネチャまたは形状

enum hdr_sts { 
    NO_HEADER,    /* ヘッダーが存在しない */
    NOT_DIGEST,   /* Digest 認証スキームではない */
    INVALID,      /* 形式が不正 */
    VALID         /* 形式が妥当 */
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • Authorization ヘッダーのパース結果を分類する目的でのみ使用されます。

アルゴリズムまたは振る舞い

  • ヘッダーの解析プロセスにおいて、現在の状態を追跡するためのステータスコードとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/aaa/mod_auth_digest.c:160 にて auth_hdr_sts 変数の型として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • digest_header_rec: パースされたヘッダー内容を保持する構造体。本列挙型はこの構造体へのマッピング過程で使用されます。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 本列挙型は mod_auth_digest モジュール内でのヘッダー検証ロジックの分岐条件として利用されています。ロジック変更時は、各ステータスの意味論が RFC 2617 の要件と一致しているか確認してください。
