pass_output

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:260

目的

バッファリングされたデータを接続フィルタチェーンへ転送し、TLS 接続の最適化（ウォームアップ/クールダウン）を制御します。

シグネチャまたは形状

static apr_status_t pass_output(h2_c1_io *io, int flush)

入力と出力

  • **入力**:
  • io: h2_c1_io 構造体へのポインタ。
  • flush: 強制フラッシュを行うかを示すフラグ（非ゼロで有効）。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • io->is_passing フラグを使用して再帰呼び出しを防止します。
  • io->output ブリゲード内のデータは、呼び出し終了時に必ずクリーンアップされます。

アルゴリズムまたは振る舞い

1. **再帰チェック**: is_passing が設定されている場合、エラー（APR_EGENERAL）を返し終了します。 2. **データ準備**: append_scratch(io) を呼び出し、スクラッチバッファの内容をブリゲードへ追加します。 3. **フラッシュ処理**: flush が指定され、ブリゲードの末尾がフラッシュバケットでない場合、フラッシュバケットを追加します。 4. **転送**: ap_pass_brigade を呼び出し、フィルタチェーンへデータを送出します。 5. **最適化ロジック**:

  • 転送量（bytes_written）が warmup_size を超えると write_size を WRITE_SIZE_MAX に拡大します。
  • 一定時間（cooldown_usecs）書き込みがない場合、write_size を WRITE_SIZE_INITIAL にリセットします。

    • 6. **後処理**: ブリゲードのクリーンアップと is_passing フラグの解除を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • h2_c1_io_pass (modules/http2/h2_c1_io.c:344)
  • h2_c1_io_assure_flushed (modules/http2/h2_c1_io.c:354)
  • **呼び出し先**:
  • append_scratch
  • ap_pass_brigade
  • apr_brigade_cleanup
  • ap_log_cerror

関連データ構造

  • h2_c1_io: セッションの入出力状態を保持する主要構造体。
  • apr_bucket_brigade: 転送対象のデータバケット列。

失敗モードと境界条件

  • **再帰呼び出し**: is_passing が既にセットされている場合、出力破損を防ぐために APR_EGENERAL を返します。
  • **フィルタエラー**: ap_pass_brigade がエラーを返した場合、そのステータスを保持したままクリーンアップへ移行します。

保守担当者向けメモ

  • is_passing フラグは、H2EOS バケットの破棄など、予期せぬ再帰的呼び出しを検知するために重要です。
  • write_size の動的調整は、TLS レコードの効率的なパッキングを目的としています。
