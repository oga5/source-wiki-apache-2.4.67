argstr_to_table

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_heartbeat.c:64

目的

クエリ文字列形式（key=value&key2=value2）の文字列を解析し、その結果を apr_table_t に格納する。

シグネチャまたは形状

void argstr_to_table(apr_pool_t *p, char *str, apr_table_t *parms)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • str: 解析対象のクエリ文字列（破壊的に変更される）。
  • parms: 解析結果を格納する apr_table_t。
  • **出力**: なし（parms が更新される）。

前提条件と不変条件

  • str は apr_strtok によって内部的に分割されるため、呼び出し元で保持している文字列が変更されても問題ない状態であること。
  • value が存在しないキー（例: key&key2）の場合、値は "1" として扱われる。

アルゴリズムまたは振る舞い

1. apr_strtok を使用して & を区切り文字として文字列を走査する。 2. 各トークン内で = を検索し、キーと値に分割する。 3. = が見つからない場合は値を "1" とする。 4. ap_unescape_url を呼び出し、キーと値の URL エンコードを解除する。 5. apr_table_set を使用して、解析したキーと値を parms テーブルに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/balancers/mod_lbmethod_heartbeat.c:179 (readfile_heartbeats 関数内から呼び出される)。
  • **呼び出し先**: apr_strtok, strchr, ap_unescape_url, apr_table_set。
  • 注: server/util_script.c にも同名の関数が存在するが、本シンボルとは別定義である。

関連データ構造

  • apr_table_t: キーと値のペアを保持する Apache Portable Runtime のテーブル構造体。

失敗モードと境界条件

  • str が空の場合、ループは実行されずテーブルは更新されない。
  • URL エンコードが不正な場合、ap_unescape_url の挙動に依存する。

保守担当者向けメモ

  • 本関数は mod_heartmonitor 等の他モジュールでも類似のロジックが実装されている可能性がある。修正時は server/util_script.c 等の重複実装との整合性に注意すること。
