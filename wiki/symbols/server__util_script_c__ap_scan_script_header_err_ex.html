ap_scan_script_header_err_ex

定義

  • 種別: function
  • 定義箇所: server/util_script.c:778

目的

ファイルベースのスクリプト出力から HTTP ヘッダーを解析し、リクエスト構造体へ反映させるためのラッパー関数です。

シグネチャまたは形状

int ap_scan_script_header_err_ex(request_rec *r, apr_file_t *f, char *buffer, int module_index)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • f: 解析対象のファイルポインタ (apr_file_t)。
  • buffer: ヘッダー行の読み込みに使用するバッファ。
  • module_index: ログ出力に使用するモジュールインデックス。
  • **出力**:
  • 成功時または特定のステータスコードに応じた整数値を返します（ap_scan_script_header_err_core_ex の戻り値に依存）。

前提条件と不変条件

  • r->pool を介してメモリ管理が行われます。
  • スクリプトの出力が完了するまでブロックする可能性があります。

アルゴリズムまたは振る舞い

内部で getsfunc_FILE をリーダーとして指定し、ap_scan_script_header_err_core_ex を呼び出すことで、ファイルストリームからヘッダー行を抽出・解析します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_asis.c:59
  • **呼び出し先**: ap_scan_script_header_err_core_ex

関連データ構造

  • request_rec: リクエストの状態保持に使用。
  • apr_file_t: 入力ソースとなるファイルハンドル。

失敗モードと境界条件

  • スクリプト出力が不正な形式である場合や、読み込み中にエラーが発生した場合は、コア解析関数 (ap_scan_script_header_err_core_ex) の仕様に従いエラーを返します。

保守担当者向けメモ

  • 本関数は ap_scan_script_header_err_core_ex のファイル I/O 特化型ラッパーです。
  • EBCDIC 環境下では、ヘッダー解析時に適切な文字コード変換が介入します。
