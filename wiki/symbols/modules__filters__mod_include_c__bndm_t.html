bndm_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_include.c:168

目的

BNDM（Backward Nondeterministic Dawg Matching）アルゴリズムを用いた高速な文字列検索を行うための、コンパイル済みパターン情報を保持する構造体です。

シグネチャまたは形状

typedef struct {
    unsigned int T[256];      /* 文字ごとのビットマスクテーブル */
    unsigned int x;           /* パターンの状態ビットマスク */
    apr_size_t pattern_len;   /* パターンの長さ */
} bndm_t;

入力と出力

  • 入力: bndm_compile 関数により、検索対象のパターン文字列から本構造体が生成されます。
  • 出力: bndm 関数において、検索対象のテキストと本構造体を入力として受け取り、一致位置を返します。

前提条件と不変条件

  • T 配列は、検索対象のパターンに基づき、各文字の出現位置をビットマスクとして保持している必要があります。
  • pattern_len は、検索対象のパターン文字列の長さと一致している必要があります。

アルゴリズムまたは振る舞い

  • bndm_compile 関数により、パターンを前処理して bndm_t 構造体を構築します。
  • bndm 関数は、この構造体を使用して、テキスト内からパターンを後方から前方へ向かって効率的に検索します。

呼び出し元と呼び出し先

  • 呼び出し元: mod_include.c 内の bndm_compile および bndm 関数によって利用されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • ssi_internal_ctx: mod_include.c 内で解析状態を保持する構造体であり、この中で bndm_t 型のポインタ（start_seq_pat 等）が参照されます。

失敗モードと境界条件

  • パターン長が 0 の場合や、メモリ割り当て（apr_palloc）に失敗した場合は、検索処理やコンパイル処理が正常に動作しない可能性があります。

保守担当者向けメモ

  • 本構造体は mod_include における高速な文字列検索の基盤です。複雑な正規表現の評価には ap_expr が使用されますが、単純なタグ開始シーケンス等の検索には本アルゴリズムが優先されます。
