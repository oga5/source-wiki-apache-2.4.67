ap_find_path_info

定義

  • 種別: function
  • 定義箇所: server/util_script.c:352

目的

URI 文字列から PATH_INFO 部分が開始されるインデックスを特定します。

シグネチャまたは形状

int ap_find_path_info(const char *uri, const char *path_info)

入力と出力

  • **入力**:
  • uri: 検索対象の完全な URI 文字列。
  • path_info: URI の末尾に付与されていると想定されるパス情報文字列。
  • **出力**:
  • int: uri 内において path_info が開始される位置（インデックス）。

前提条件と不変条件

  • uri および path_info はヌル終端文字列である必要があります。
  • path_info が uri の末尾と一致しない場合、戻り値は uri の長さに依存した値となります。

アルゴリズムまたは振る舞い

1. uri と path_info の末尾から前方に向かって文字を比較します。 2. 比較中に / が出現した場合、連続する / をスキップして一致を確認します。 3. 一致が途切れた地点から、uri 内の次の / または終端文字までインデックスを進めます。 4. 最終的に、uri 内で path_info が開始される位置のインデックスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_negotiation.c:3051
  • server/request.c:2328
  • server/util_script.c:440
  • **呼び出し先**:
  • C 標準ライブラリの strlen を使用しています。その他の関数呼び出しはありません。

関連データ構造

  • 特になし（文字列操作のみ）。

失敗モードと境界条件

  • uri が path_info を含まない場合、uri の末尾を指すインデックスを返します。
  • uri が空文字列の場合、0 を返します。
  • path_info が uri より長い場合、比較ロジックにより 0 または適切なオフセットが算出されます。

保守担当者向けメモ

  • 本関数は PATH_INFO の分離位置を決定するために使用されます。戻り値を使用して uri[index] = '\0' のように文字列を切り詰める用途で利用されることが多いため、呼び出し側でのバッファ操作に注意してください。
