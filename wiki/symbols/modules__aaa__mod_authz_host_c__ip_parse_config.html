ip_parse_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:86

目的

Apache HTTP Server の Require ip ディレクティブで指定された IP アドレスまたはサブネットリストを解析し、認可時に使用可能な apr_ipsubnet_t 構造体の配列へ変換します。

シグネチャまたは形状

static const char *ip_parse_config(cmd_parms *cmd,
                                   const char *require_line,
                                   const void **parsed_require_line)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • require_line: Require ip に続く引数文字列。
  • parsed_require_line: 解析結果（apr_ipsubnet_t の配列）を格納するポインタの出力先。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • parsed_subnets が有効な場合、解析済みのサブネットはハッシュテーブルから再利用されます。
  • parsed_require_line には、cmd->pool から割り当てられた apr_ipsubnet_t の配列が格納されます。
  • require_line には少なくとも1つの IP アドレス指定が必要です。

アルゴリズムまたは振る舞い

1. require_line を空白で分割し、引数の数をカウントします。 2. 引数が存在しない場合はエラーを返します。 3. cmd->pool 上に apr_ipsubnet_t ポインタの配列を確保します。 4. 各引数について以下を繰り返します：

  • parsed_subnets ハッシュテーブルに存在する場合はそれを再利用します。
  • 存在しない場合、apr_ipsubnet_create を呼び出して IP/サブネットを生成します。
  • 生成に成功した場合、parsed_subnets があればハッシュテーブルに登録します。

    • 5. 最終的に *parsed_require_line に配列をセットし、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_host.c:333（authz_ip_provider の設定登録時）。
  • **呼び出し先**: ap_getword_conf（引数分割）、apr_pcalloc、apr_hash_get/set、apr_ipsubnet_create、ap_strchr。

関連データ構造

  • apr_ipsubnet_t: IP サブネット情報を保持する APR 構造体。
  • parsed_subnets: 設定解析中にサブネット情報を共有するためのハッシュテーブル。

失敗モードと境界条件

  • 引数が空の場合: 'require ip' requires an argument を返します。
  • IP アドレス形式が不正な場合: apr_ipsubnet_create のエラーに基づき、ip address '...' appears to be invalid を返します。

保守担当者向けメモ

  • 本関数は設定解析フェーズでのみ実行されます。
  • parsed_subnets は設定解析中のみ有効な一時的キャッシュであるため、実行時の認可判断には影響しません。
