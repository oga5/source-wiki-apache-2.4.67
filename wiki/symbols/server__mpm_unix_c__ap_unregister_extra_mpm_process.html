ap_unregister_extra_mpm_process

定義

  • 種別: function
  • 定義箇所: server/mpm_unix.c:92

目的

管理対象の「追加MPMプロセス」リストから、指定されたプロセスID（PID）を持つエントリを検索・削除し、そのプロセスが属していた世代（generation）を呼び出し元へ返却します。

シグネチャまたは形状

int ap_unregister_extra_mpm_process(pid_t pid, ap_generation_t *old_gen)

入力と出力

  • **入力**:
  • pid: 削除対象のプロセスID。
  • old_gen: 削除されたプロセスの世代情報を格納するためのポインタ。
  • **出力**:
  • int: 成功時（リスト内にPIDが存在し削除した場合）は 1 を返し、失敗時（該当PIDが見つからない場合）は 0 を返します。
  • old_gen: 成功時、削除されたエントリの gen 値が書き込まれます。

前提条件と不変条件

  • extras グローバル変数（extra_process_t 型のリスト）が、追加MPMプロセスの管理に使用されている必要があります。
  • ap_register_extra_mpm_process によって確保されたメモリ領域を解放するため、本関数は free() を呼び出します。

アルゴリズムまたは振る舞い

1. extras リストを先頭から順に走査し、指定された pid と一致するエントリを探索します。 2. 一致するエントリが見つかった場合：

  • リストのリンクを繋ぎ変えて当該エントリをリストから除外します。
  • old_gen に当該エントリの世代情報をコピーします。
  • free() を呼び出し、メモリを解放します。
  • 1 を返して終了します。

    • 3. リストの末尾まで探索しても一致するPIDが見つからない場合：

  • 0 を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/worker/worker.c:1676
  • server/mpm_unix.c:265
  • server/mpm_unix.c:315
  • include/mpm_common.h:138（ドキュメント上の言及）
  • **呼び出し先**:
  • free()（標準ライブラリ）

関連データ構造

  • extra_process_t: リストを構成する構造体。pid、gen（世代）、および次の要素へのポインタ next を保持します。

失敗モードと境界条件

  • **該当PID不在**: リスト内に指定されたPIDが存在しない場合、何も変更せず 0 を返します。
  • **リストの先頭削除**: 削除対象がリストの先頭にある場合、extras ポインタ自体を更新します。
  • **空リスト**: extras が NULL の場合、ループに入らず 0 を返します。

保守担当者向けメモ

  • 本関数は ap_register_extra_mpm_process と対になる関数です。メモリリークを防ぐため、プロセス終了時には必ず本関数を呼び出してリストから削除し、メモリを解放する必要があります。
  • WIN32 定義時は本ファイルの内容が無効化されるため、プラットフォーム依存の挙動に注意してください。
