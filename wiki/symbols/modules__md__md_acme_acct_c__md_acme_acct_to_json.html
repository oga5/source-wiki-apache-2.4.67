md_acme_acct_to_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_acct.c:89

目的

ACME アカウント情報を保持する md_acme_acct_t 構造体を、永続化や通信に適した md_json_t 形式へシリアライズします。

シグネチャまたは形状

md_json_t *md_acme_acct_to_json(md_acme_acct_t *acct, apr_pool_t *p)

入力と出力

  • **入力**:
  • acct: シリアライズ対象の md_acme_acct_t 構造体へのポインタ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • アカウント情報が格納された md_json_t 構造体へのポインタ。

前提条件と不変条件

  • acct 引数は NULL であってはなりません（assert(acct) により強制）。
  • 生成される JSON オブジェクトは、引数として渡された apr_pool_t のライフサイクルに依存します。

アルゴリズムまたは振る舞い

1. md_json_create(p) を呼び出し、空の JSON オブジェクトを初期化します。 2. アカウントの status フィールドを判定し、対応する文字列（"valid", "deactivated", "revoked"）を JSON の status キーに設定します。 3. 構造体内の各フィールド（url, ca_url, contacts, registration, agreement, orders, eab_kid, eab_hmac）が非 NULL の場合、対応するキーを使用して JSON オブジェクトに値を格納します。 4. 最後に構築された md_json_t を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_acct.c:184
  • modules/md/md_acme_drive.c:71
  • **呼び出し先**:
  • md_json_create
  • md_json_sets
  • md_json_setsa
  • md_json_setj

関連データ構造

  • md_acme_acct_t: ACME アカウント情報を保持する主要構造体。
  • md_json_t: JSON データを扱うための抽象構造体。

失敗モードと境界条件

  • acct が NULL の場合、アサーションによりプログラムが停止します。
  • 各フィールドが NULL の場合、そのフィールドは JSON に出力されません。
  • apr_pool_t のメモリ不足が発生した場合、md_json 関連関数内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はアカウントのシリアライズのみを担当します。デシリアライズには対となる md_acme_acct_from_json を使用してください。
  • EAB（External Account Binding）関連のキー（eab_kid, eab_hmac）は、JSON 内で EAB オブジェクトの配下にネストして格納されます。
