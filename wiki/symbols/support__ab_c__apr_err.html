apr_err

定義

  • 種別: function
  • 定義箇所: support/ab.c:429

目的

APR（Apache Portable Runtime）関連の操作でエラーが発生した際、エラーメッセージとステータスコードを標準エラー出力に出力し、プログラムを終了させるためのユーティリティ関数です。

シグネチャまたは形状

static void apr_err(const char *s, apr_status_t rv)

入力と出力

  • **入力**:
  • s: エラー発生箇所や状況を示す文字列。
  • rv: apr_status_t 型のエラーコード。
  • **出力**: なし（関数内で exit(rv) を呼び出しプロセスを終了する）。

前提条件と不変条件

  • apr_strerror が rv に対応するエラーメッセージを正しく取得できること。
  • プログラム終了時に done 変数がグローバルスコープで参照可能であること。

アルゴリズムまたは振る舞い

1. apr_strerror を使用して、rv に対応するエラーメッセージをローカルバッファ buf に格納する。 2. fprintf を用いて、引数 s、エラーメッセージ、および数値の rv を標準エラー出力 (stderr) に出力する。 3. グローバル変数 done が 0 以外の場合、完了したリクエスト数を標準出力 (stdout) に表示する。 4. exit(rv) を呼び出し、エラーコードを終了ステータスとしてプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: support/ab.c 内の各 APR API 呼び出し（apr_pollset_remove, apr_pollset_add, socket, bind, apr_socket_connect, apr_socket_recv, apr_pollset_create 等）の直後で、エラー判定時に呼び出される。
  • **呼び出し先**: apr_strerror (APR API), fprintf, printf, exit (標準ライブラリ)。

関連データ構造

  • apr_status_t: APR のエラーコードを保持する型。

失敗モードと境界条件

  • buf のサイズ（120バイト）を超えるエラーメッセージは切り詰められる可能性がある。
  • done が 0 の場合、完了リクエスト数の出力はスキップされる。

保守担当者向けメモ

  • 本関数はプログラムを強制終了させるため、リソースのクリーンアップが必要な箇所での呼び出しには注意すること。
  • ab.c 内で一貫したエラーハンドリングを行うために使用されている。
