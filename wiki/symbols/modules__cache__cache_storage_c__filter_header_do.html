filter_header_do

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_storage.c:116

目的

キャッシュエントリから転送用レスポンスへヘッダーをコピーする際、HTTP仕様（RFC 2616 Section 14.46）に基づき、警告コードが 1xx である Warning ヘッダーを除外するために使用されます。

シグネチャまたは形状

static int filter_header_do(void *v, const char *key, const char *val)

入力と出力

  • **入力**:
  • v: コピー先のテーブル（apr_table_t）。
  • key: 現在処理中のヘッダー名。
  • val: 現在処理中のヘッダー値。
  • **出力**:
  • 常に 1 を返します（apr_table_do の反復を継続するため）。

前提条件と不変条件

  • v は apr_table_addn が許容する有効なテーブルポインタである必要があります。
  • key および val はヌル終端文字列である必要があります。

アルゴリズムまたは振る舞い

1. key が Warning（大文字小文字を区別しない）であり、かつ val の先頭文字が '1' である場合、そのヘッダーはキャッシュエントリおよび転送レスポンスから削除されるべきものと判断し、何も行いません。 2. 上記以外の場合、apr_table_addn を使用して、指定された key と val をテーブル v に追加します。 3. 処理の継続を指示するため 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/cache_storage.c:168 において、apr_table_do のコールバック関数として呼び出されています。
  • **呼び出し先**:
  • ap_cstr_casecmp: ヘッダー名の比較に使用。
  • apr_table_addn: テーブルへの要素追加に使用。

関連データ構造

  • apr_table_t: Apache Portable Runtime のテーブル構造体。ヘッダーの保持に使用されます。

失敗モードと境界条件

  • key または val が NULL の場合、ap_cstr_casecmp や apr_table_addn の内部実装に依存した動作となります。
  • Warning ヘッダーのコードが 1xx 以外（例: 2xx）である場合は、フィルタリングされずにそのままテーブルに追加されます。

保守担当者向けメモ

  • 本関数は apr_table_do を介して呼び出されることを前提としています。
  • Warning ヘッダーのフィルタリングルールは RFC 2616 に準拠しています。将来的な仕様変更や RFC 更新時には、この条件判定ロジックの見直しが必要です。
