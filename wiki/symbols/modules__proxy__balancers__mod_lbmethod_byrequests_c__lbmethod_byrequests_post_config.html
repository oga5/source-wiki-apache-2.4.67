lbmethod_byrequests_post_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_byrequests.c:130

目的

mod_proxy が提供するワーカー選出用オプション関数を取得し、mod_lbmethod_byrequests が正常に動作するための依存関係を確立します。

シグネチャまたは形状

static int lbmethod_byrequests_post_config(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *s)

入力と出力

  • 入力:
  • pconf, plog, ptemp: APR プール。
  • s: 現在のサーバー設定レコード。
  • 出力:
  • OK: 正常終了。
  • !OK: mod_proxy がロードされていない等のエラー発生時。

前提条件と不変条件

  • 本関数はサーバー起動時に2回呼び出されます。
  • AP_SQ_MS_CREATE_PRE_CONFIG 状態での初回呼び出し時は、静的データの初期化を行わずに即座に OK を返します。

アルゴリズムまたは振る舞い

1. サーバー起動状態を確認し、プリコンフィグ段階であれば処理をスキップします。 2. APR_RETRIEVE_OPTIONAL_FN を使用して、mod_proxy が提供する proxy_balancer_get_best_worker 関数を取得し、グローバル変数 ap_proxy_balancer_get_best_worker_fn に格納します。 3. 関数取得に失敗した場合、エラーログ（APLOGNO(10152)）を出力し、エラーコードを返して終了します。 4. 成功した場合は OK を返します。

呼び出し元と呼び出し先

  • 呼び出し元: Apache の post_config フック（modules/proxy/balancers/mod_lbmethod_byrequests.c:154 にて登録）。
  • 呼び出し先: ap_state_query, APR_RETRIEVE_OPTIONAL_FN, ap_log_error。

関連データ構造

  • server_rec: サーバー設定情報。
  • ap_proxy_balancer_get_best_worker_fn: ワーカー走査のためのオプション関数ポインタ。

失敗モードと境界条件

  • mod_proxy がロードされていない場合、ap_proxy_balancer_get_best_worker_fn の取得に失敗し、緊急ログを出力して初期化に失敗します。

保守担当者向けメモ

  • 本関数は post_config フックとして登録されており、サーバー起動時の依存関係解決を担います。
  • mod_proxy との疎結合を維持するため、オプション関数として proxy_balancer_get_best_worker を取得する設計になっています。
