ws_stream

定義

  • 種別: struct
  • 定義箇所: test/clients/h2ws.c:816

目的

WebSocket ストリーム固有の制御状態を保持するためのラッパー構造体です。

シグネチャまたは形状

struct ws_stream {
  struct h2_stream s;
};

入力と出力

  • 入力: なし（構造体定義）
  • 出力: なし（構造体定義）

前提条件と不変条件

  • h2_stream 構造体をメンバとして保持しており、HTTP/2 ストリームとしての基底機能を継承する設計となっています。

アルゴリズムまたは振る舞い

  • 本構造体は、h2_stream をラップすることで、WebSocket 通信に特化したストリーム管理を可能にします。
  • ws_stream_create によりインスタンス化され、ws_stream_submit を通じて CONNECT メソッドを用いたストリームの開始処理に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • test/clients/h2ws.c:840: ws_stream_create 内での生成
  • test/clients/h2ws.c:870: h2_session_stream_get からのキャストによる取得
  • test/clients/h2ws.c:903: ws_stream_submit の引数
  • 呼び出し先:
  • 構造体メンバとして h2_stream を内包しています。

関連データ構造

  • h2_stream: 本構造体がラップしている基底ストリーム構造体。
  • h2_session: ストリームを管理するセッション構造体。
  • uri: ストリームの接続先情報を保持する構造体。

失敗モードと境界条件

  • h2_session_stream_get から取得したストリームを本構造体にキャストする際、対象のストリームが WebSocket 用でない場合に型不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は h2_stream を継承するような設計となっているため、h2_stream 側の変更が本構造体のメモリレイアウトに影響を与える点に注意してください。
