check_header_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http/http_filters.c:691

目的

HTTPヘッダの検証処理において、対象となるリクエスト情報と検証の厳格さを保持するためのコンテキスト構造体です。

シグネチャまたは形状

struct check_header_ctx {
    request_rec *r;
    int strict;
};

入力と出力

  • r: 検証対象の request_rec 構造体へのポインタ。
  • strict: 検証の厳格さを指定するフラグ（整数値）。

前提条件と不変条件

  • r は有効な request_rec を指している必要があります。
  • 本構造体は、check_header および check_headers_table 関数内での状態保持に使用されます。

アルゴリズムまたは振る舞い

本構造体は、ヘッダの検証ロジック（check_header）において、現在のリクエストコンテキストと検証ルールを共有するために利用されます。check_headers_table を通じてテーブル内の各ヘッダを走査する際、このコンテキストが参照されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/http/http_filters.c:778 (インスタンスの初期化)
  • 呼び出し先:
  • check_header (引数として利用)
  • check_headers_table (引数として利用)

関連データ構造

  • request_rec: Apache HTTP Server のリクエスト情報を保持する主要な構造体。

失敗モードと境界条件

  • r が NULL の状態で参照された場合、未定義の動作を引き起こす可能性があります。
  • strict フラグの値が検証ロジックの期待値と一致しない場合、ヘッダ検証が不適切にスキップまたは拒否される可能性があります。

保守担当者向けメモ

  • 本構造体は modules/http/http_filters.c 内の静的スコープで使用されています。
  • ヘッダ検証の厳格化や新しい検証ルールの追加を行う際は、strict フラグの定義や check_header 内のロジックと併せて本構造体の拡張を検討してください。
