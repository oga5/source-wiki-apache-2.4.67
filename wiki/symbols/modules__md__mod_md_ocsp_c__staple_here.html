staple_here

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_ocsp.c:46

目的

指定されたサーバー設定において、OCSP Stapling を有効にするべきかどうかを判定する。

シグネチャまたは形状

static int staple_here(md_srv_conf_t *sc)

入力と出力

  • **入力**: md_srv_conf_t *sc (サーバー設定構造体)
  • **出力**: int 型。有効な場合は 1、無効な場合は 0 を返す。

前提条件と不変条件

  • sc が NULL の場合、または sc->mc->ocsp が設定されていない場合は常に 0 を返す。
  • sc->assigned にドメインが1つだけ割り当てられており、かつそのドメインで stapling が有効な場合は 1 を返す。
  • 上記以外の場合、MD_CONFIG_STAPLING および MD_CONFIG_STAPLE_OTHERS の両方の設定値が有効である場合にのみ 1 を返す。

アルゴリズムまたは振る舞い

1. 引数 sc および sc->mc->ocsp の存在を確認する。 2. sc->assigned 配列を確認し、要素数が1かつ対象ドメインの stapling フラグが立っていれば 1 を返す。 3. 上記条件に合致しない場合、設定ディレクティブ MD_CONFIG_STAPLING と MD_CONFIG_STAPLE_OTHERS の両方が有効（真）であるかを評価し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_ocsp.c:67
  • modules/md/mod_md_ocsp.c:117
  • **呼び出し先**:
  • md_config_geti (設定値取得用ユーティリティ)

関連データ構造

  • md_srv_conf_t: サーバー設定を保持する構造体。
  • md_t: 管理対象ドメインの情報を保持する構造体。

失敗モードと境界条件

  • sc が NULL の場合、安全に 0 を返す。
  • sc->assigned が空、または複数のドメインが割り当てられている場合、個別のドメイン設定による判定はスキップされ、グローバルな設定値（MD_CONFIG_STAPLING 等）による判定へフォールバックする。

保守担当者向けメモ

  • 本関数が 0 を返す場合、呼び出し側では OCSP Stapling をスキップし、DECLINED を返すことが想定されている。
  • mod_watchdog がロードされていない環境では、OCSP 関連機能全体が初期化時にエラーとなるため、本関数が呼ばれる前に前提条件が満たされている必要がある。
