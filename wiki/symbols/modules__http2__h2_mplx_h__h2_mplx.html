h2_mplx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_mplx.h:56

目的

HTTP/2 セッションにおける「ストリーム多重化（Multiplexer）」のコア構造体です。クライアントとのメイン接続（c1）と、個々のリクエストを処理するセカンダリ接続（c2）の間で、データ、イベント、および処理リソースを仲介するハブとして機能します。

シグネチャまたは形状

typedef struct h2_mplx h2_mplx;

入力と出力

本構造体は、h2_mplx_c1_*（メイン接続用）、h2_mplx_c2_*（セカンダリ接続用）、および h2_mplx_worker_*（ワーカースレッド用）といった各コンテキスト向けの API を通じて操作されます。

前提条件と不変条件

  • **スレッド安全性**: 内部で apr_thread_mutex_t を使用して保護されています。複雑な状態遷移を伴うため、ロックの取得範囲を最小限に抑える設計となっています。
  • **コンテキストの分離**: 関数名のプレフィックス（c1_, c2_, worker_）は、その関数を呼び出すべきコンテキストを厳密に示しています。これらを混同するとデッドロックや競合状態を引き起こす可能性があります。

アルゴリズムまたは振る舞い

  • **リクエストのスケジューリング**: 優先度に基づいたストリーム処理の順序付けを行います。
  • **接続仲介**: c1 と c2 間のデータフローの同期と管理を行います。
  • **ライフサイクル管理**: ストリームの生成から破棄、およびワーカースレッドの割り当てを管理します。
  • **リソース制限**: 同時実行ストリーム数やメモリ使用量を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のファイルで参照されています。
  • modules/http2/h2_bucket_eos.c
  • modules/http2/h2_c1.c
  • modules/http2/h2_c2.c
  • modules/http2/h2_c2_filter.c
  • modules/http2/h2_conn_ctx.c
  • modules/http2/h2_conn_ctx.h
  • **呼び出し先**: 構造体定義のみのため、直接的な関数呼び出し先は本ソース excerpt には含まれていません。

関連データ構造

  • h2_stream: 多重化される個々のストリームの定義。
  • h2_workers: ワーカースレッド管理。
  • h2_bucket_beam: c1-c2 間のデータ転送（バケットの受け渡し）に使用される機構。
  • apr_queue: 内部キュー管理。

失敗モードと境界条件

  • ワーカースレッドの割り当て失敗や、同時実行ストリーム数の制限超過時にリクエスト処理が待機または拒否される可能性があります。
  • 適切なロック管理を怠った場合、デッドロックや競合状態が発生します。

保守担当者向けメモ

  • h2_mplx.c に実装の詳細が含まれています。
  • ログ出力時には H2_MPLX_MSG マクロを使用することで、child_num と id を含めた一貫性のあるデバッグ情報が得られます。
