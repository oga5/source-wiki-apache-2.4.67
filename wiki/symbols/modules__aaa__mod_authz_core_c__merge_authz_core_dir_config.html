merge_authz_core_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_core.c:115

目的

Apache HTTP Server のディレクトリごとの認可設定（authz_core_dir_conf）を、親ディレクトリの設定（basev）と現在のディレクトリの設定（newv）からマージする。

シグネチャまたは形状

static void *merge_authz_core_dir_config(apr_pool_t *p, void *basev, void *newv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定（authz_core_dir_conf）。
  • newv: 現在のディレクトリの設定（authz_core_dir_conf）。
  • **出力**: マージされた新しい authz_core_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および newv は authz_core_dir_conf 型としてキャストされる。
  • メモリ割り当ては p を介して行われる。
  • authz_forbidden_on_fail の値は、newv が UNSET の場合にのみ basev から継承される。

アルゴリズムまたは振る舞い

1. **設定の継承判定**: newv に認可設定（section）がなく、op が UNSET の場合は、basev の設定を複製して返す。 2. **設定の上書き判定**: newv の op が OFF または UNSET であるか、あるいは両方に設定がない場合は、newv を優先して複製する。 3. **論理ツリーのマージ**: 上記以外の場合、basev と newv の section を結合する新しいツリーを構築する。

  • 両方に section が存在する場合、新しい section を作成し、basev のセクションを first に、newv のセクションを next に配置する。
  • limited フラグはビット論理和（OR）で結合される。

    • 4. **失敗時フラグの統合**: authz_forbidden_on_fail の状態を評価し、newv が UNSET であれば basev の値を採用する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_core.c:1167（Apache の設定マージフックとして登録されている）。
  • **呼び出し先**: apr_pmemdup, apr_pcalloc（APR メモリ管理関数）。

関連データ構造

  • authz_core_dir_conf: ディレクトリごとの認可設定を保持する構造体。
  • authz_section_conf: 認可の論理単位を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等がエラーを発生させる可能性がある。
  • base->section や new->section が NULL の場合、条件分岐によって適切に処理される。

保守担当者向けメモ

  • 本関数は再帰的な設定マージを行うため、設定のネストが深い場合にスタック消費量に注意が必要である。
  • is_merged フラグが設定されることで、マージ済みセクションであることが識別可能となっている。
