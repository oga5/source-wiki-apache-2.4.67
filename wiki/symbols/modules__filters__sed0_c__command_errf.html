command_errf

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:44

目的

sed スクリプトのコンパイルおよび実行中に発生したエラーメッセージをフォーマットし、登録されたエラーハンドラ（errfn）を通じて報告する。

シグネチャまたは形状

void command_errf(sed_commands_t *commands, const char *fmt, ...)

入力と出力

  • **入力**:
  • commands: sed_commands_t 構造体へのポインタ。エラーハンドラとメモリプールを保持する。
  • fmt: apr_pvsprintf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • commands が有効なポインタであること。
  • commands->errfn が設定されている場合のみ、エラー報告が実行される。
  • commands->pool が有効なメモリプールであること（エラーメッセージの生成に使用される）。

アルゴリズムまたは振る舞い

1. commands->errfn および commands->pool が存在するか確認する。 2. va_list を使用して可変長引数を取得する。 3. apr_pvsprintf を呼び出し、commands->pool 上でフォーマット済みのエラー文字列を生成する。 4. 生成された文字列を commands->errfn に渡して実行する。 5. va_end で引数リストを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/regexp.c:65
  • modules/filters/sed0.c:73, 245, 253, 266, 278, 288, 307, 312, 323, 330
  • **呼び出し先**:
  • apr_pvsprintf (APRライブラリ)
  • commands->errfn (構造体内に保持された関数ポインタ)

関連データ構造

  • sed_commands_t: sed コンパイル状態を管理する構造体。エラーハンドラ (errfn)、データポインタ (data)、メモリプール (pool) を保持する。

失敗モードと境界条件

  • commands->errfn が NULL の場合、エラーメッセージは生成されるが報告は行われない。
  • commands->pool が無効な場合、apr_pvsprintf の呼び出しにより未定義の動作が発生する可能性がある。

保守担当者向けメモ

  • 本関数は libsed のエラー報告の集約点です。エラーメッセージの出力先を変更したい場合は、sed_commands_t を初期化する際に適切な errfn を設定してください。
  • apr_pvsprintf を使用しているため、メモリ割り当ては commands->pool のライフサイクルに依存します。
