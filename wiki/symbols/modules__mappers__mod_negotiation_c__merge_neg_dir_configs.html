merge_neg_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_negotiation.c:79

目的

Apache HTTP Server のディレクトリ単位の設定において、親ディレクトリの設定とサブディレクトリの設定をマージし、ネゴシエーション設定（neg_dir_config）の新しいインスタンスを作成する。

シグネチャまたは形状

static void *merge_neg_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定 (neg_dir_config *)。
  • addv: サブディレクトリの設定 (neg_dir_config *)。
  • **出力**:
  • マージされた新しい neg_dir_config 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は neg_dir_config 型としてキャスト可能であること。
  • p は有効なメモリプールであること。
  • サブディレクトリ側の設定（addv）が優先される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して、新しい neg_dir_config 構造体用のメモリを確保する。 2. forcelangpriority フィールドについて、addv が FLP_UNDEF でない場合は addv の値を採用し、それ以外は basev の値を採用する。 3. language_priority フィールドについて、addv が設定されていれば addv を採用し、それ以外は basev を採用する。 4. マージされた構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_negotiation.c:3218 （Apache の設定マージ機構により呼び出される）。
  • **呼び出し先**: apr_palloc（APR メモリ管理関数）。

関連データ構造

  • neg_dir_config: ディレクトリごとのネゴシエーション設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある。
  • basev または addv が NULL の場合の挙動については、本ソースコード内では明示的なチェックは行われていない。

保守担当者向けメモ

  • 本関数は Apache の設定マージ用コールバックとして機能する。
  • forcelangpriority の判定において FLP_UNDEF を使用しており、設定が未定義であることを明示的に区別している点に注意すること。
