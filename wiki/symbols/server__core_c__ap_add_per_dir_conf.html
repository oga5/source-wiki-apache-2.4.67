ap_add_per_dir_conf

定義

  • 種別: function
  • 定義箇所: server/core.c:616

目的

指定されたサーバー設定構造体（server_rec）に対し、ディレクトリ単位の設定構造体（dir_config）を登録・追加します。

シグネチャまたは形状

void ap_add_per_dir_conf(server_rec *s, void *dir_config)

入力と出力

  • **入力**:
  • s: 設定を追加する対象の server_rec 構造体。
  • dir_config: 追加するディレクトリ設定へのポインタ。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • s->module_config から取得される core_server_config が有効である必要があります。
  • sconf->sec_dir は apr_array_header_t 型の配列であり、apr_array_push が成功する状態である必要があります。

アルゴリズムまたは振る舞い

1. ap_get_core_module_config を使用して、サーバー設定からコアモジュール固有の設定構造体（core_server_config）を取得します。 2. 取得した設定内の sec_dir 配列に対して apr_array_push を呼び出し、新しい要素用のメモリ領域を確保します。 3. 確保された領域に、引数として渡された dir_config ポインタを格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:2541（設定解析時に新しいディレクトリ設定を登録する際に使用）。
  • **呼び出し先**:
  • ap_get_core_module_config（コアモジュール設定の取得）
  • apr_array_push（APR 配列への要素追加）

関連データ構造

  • server_rec: サーバーの構成情報を保持する構造体。
  • core_server_config: コアモジュールが保持するサーバー単位の設定構造体。sec_dir メンバにディレクトリ設定のリストを保持します。

失敗モードと境界条件

  • apr_array_push は内部的にメモリ割り当てを行うため、メモリ不足時には apr_pool_t の制約によりプロセスが異常終了する可能性があります。

保守担当者向けメモ

  • 本関数は設定の構築フェーズで使用されます。sec_dir 配列の順序は、後の設定マージや適用順序に影響を与える可能性があるため、呼び出しタイミングに注意してください。
