dav_register_liveprop_namespace

定義

  • 種別: function
  • 定義箇所: modules/dav/main/liveprop.c:36

目的

WebDAV ライブプロパティの名前空間（URI）をグローバルなインデックスに登録し、XML 出力時などに使用される一意な数値 ID を割り当てます。

シグネチャまたは形状

static void dav_register_liveprop_namespace(apr_pool_t *p, const char *uri)

入力と出力

  • p: 名前空間ハッシュの初期化に使用されるメモリプール。
  • uri: 登録対象となる名前空間の URI 文字列。
  • 出力: なし（副作用としてグローバルなハッシュテーブル dav_liveprop_uris に登録）。

前提条件と不変条件

  • dav_liveprop_uris が未初期化の場合、提供されたプール p を使用してハッシュが作成され、クリーンアップ関数 dav_cleanup_liveprops が登録されます。
  • 名前空間は 1 から始まる連番で管理されます。
  • 同一 URI が既に登録されている場合、再登録は行われず処理を終了します。

アルゴリズムまたは振る舞い

1. dav_liveprop_uris が NULL の場合、apr_hash_make でハッシュを生成し、プール破棄時のクリーンアップを登録します。 2. 指定された uri が既にハッシュ内に存在するか確認します。存在する場合は即座にリターンします。 3. 存在しない場合、グローバルカウンタ dav_liveprop_count をインクリメントし、その値をキー uri に対応する値としてハッシュに格納します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/main/liveprop.c:138 (dav_register_liveprop_group 内から呼び出されることが確認されています)。
  • 呼び出し先: apr_hash_make, apr_pool_cleanup_register, apr_hash_get, apr_hash_set (APR ライブラリ関数)。

関連データ構造

  • dav_liveprop_uris: 登録された URI とインデックスを保持する apr_hash_t。
  • dav_liveprop_count: 現在の最大インデックスを保持する静的変数。

失敗モードと境界条件

  • メモリ不足時: apr_hash_make 等の APR 関数が失敗した場合の挙動は APR の仕様に依存します。
  • 境界条件: dav_liveprop_count がオーバーフローするような極端な数の名前空間登録は想定されていません。

保守担当者向けメモ

  • 本関数はプロセス生存期間中に有効な静的ハッシュを操作します。
  • インデックスは XML 出力時の lpN プレフィックス生成に直結するため、登録順序がインデックス値に影響します。
