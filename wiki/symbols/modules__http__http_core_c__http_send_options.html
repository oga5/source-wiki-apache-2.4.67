http_send_options

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:269

目的

HTTP OPTIONS メソッドに対するサーバーの応答を制御します。特に、URI が *（サーバー全体への問い合わせ）である場合に、Allow ヘッダーを含まない応答を即座に返す役割を担います。

シグネチャまたは形状

static int http_send_options(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: int 型のステータスコード
  • DONE: リクエスト処理が完了したことを示します。
  • DECLINED: この関数では処理を行わず、他のハンドラに委譲することを示します。

前提条件と不変条件

  • r->method_number が M_OPTIONS であることが前提です。
  • URI が * であるかどうかの判定は、r->uri が存在し、かつ文字列が "*" であるかを確認することで行われます。

アルゴリズムまたは振る舞い

1. リクエストメソッドが OPTIONS かつ URI が * であるかを確認します。 2. 条件に合致する場合、DONE を返して処理を終了します（HTTP pong の送信）。 3. 条件に合致しない場合、DECLINED を返して後続の処理へ回します。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_map_to_storage を介してフックとして登録されています (modules/http/http_core.c:296)。
  • **呼び出し先**: 直接的な関数呼び出しはソース内に見当たりません。

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。method_number および uri メンバーを参照します。

失敗モードと境界条件

  • r->uri が NULL の場合、安全に判定が行われるようガードされています。
  • URI が * 以外の場合、本関数は何も行わず DECLINED を返します。

保守担当者向けメモ

  • 本関数は map_to_storage フックに登録されています。このフックの実行順序や、他のモジュールによる OPTIONS 処理との競合に注意してください。
