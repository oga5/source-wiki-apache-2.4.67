stream_reqbody

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_http.c:320

目的

クライアントから受信したリクエストボディを、バックエンドサーバーへストリーミング転送する。

シグネチャまたは形状

static int stream_reqbody(proxy_http_req_t *req)

入力と出力

  • **入力**: proxy_http_req_t *req（リクエスト状態、入力ブリゲード、バックエンド接続情報を含む）
  • **出力**: int（成功時は OK、失敗時は HTTP_INTERNAL_SERVER_ERROR またはその他のエラーコード）

前提条件と不変条件

  • req->input_brigade および req->header_brigade が適切に初期化されていること。
  • rb_method に基づき、転送方式（チャンク転送または固定長）が決定済みであること。

アルゴリズムまたは振る舞い

1. **ループ処理**: input_brigade が空の場合、ap_proxy_read_input を呼び出しデータを読み込む。 2. **EOS 処理**: 入力ブリゲードの末尾に EOS (End of Stream) がある場合、それを削除し終了フラグを立てる。 3. **転送方式の適用**:

  • RB_STREAM_CHUNKED: チャンクサイズを先頭に付与し、末尾に CRLF を追加。EOS 到達時に 0 サイズのチャンクを送信。
  • RB_STREAM_CL: 転送バイト数が req->cl_val と一致するか検証し、不一致の場合は HTTP_INTERNAL_SERVER_ERROR を返す。

    • 4. **ヘッダー結合**: 初回ループ時に header_brigade を input_brigade の先頭に結合する。 5. **送信**: ap_proxy_pass_brigade を呼び出し、バックエンドへデータを送信する。 6. **終了**: EOS が確認されるまでループを継続する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_http.c:685
  • modules/proxy/mod_proxy_http.c:994
  • **呼び出し先**:
  • ap_proxy_read_input
  • ap_proxy_pass_brigade
  • ap_log_rerror

関連データ構造

  • proxy_http_req_t: リクエスト処理中の状態を保持。
  • rb_methods: リクエストボディの転送方式を定義する列挙型。
  • apr_bucket_brigade: データのバッファリングに使用。

失敗モードと境界条件

  • **Content-Length 不一致**: RB_STREAM_CL モードで、ストリームされたバイト数が期待値と異なる場合、HTTP リクエスト分割攻撃を防ぐためエラーを返す。
  • **読み込みエラー**: ap_proxy_read_input が OK 以外を返した場合、そのエラーコードを伝播する。
  • **送信エラー**: ap_proxy_pass_brigade が OK 以外を返した場合、そのエラーコードを伝播する。

保守担当者向けメモ

  • chunk_hdr は transient バケットとして使用されるため、ループ内で適切に管理されている。
  • proxy-sendextracrlf 環境変数が設定されている場合、EOS 時に追加の CRLF を送信するロジックが含まれている。
