split_argv

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_win32.c:236

目的

Windows環境において、CGI実行用のインタープリタ引数文字列を解析し、Apacheのプロセス生成に適した引数配列へ変換する。

シグネチャまたは形状

apr_array_header_t *split_argv(apr_pool_t *p, const char *interp,
                               const char *cgiprg, const char *cgiargs)

入力と出力

  • **入力**:
  • p: メモリ確保用プール。
  • interp: インタープリタのコマンドラインテンプレート（$1, %1, $*, %* を含む可能性がある）。
  • cgiprg: 実行対象のCGIプログラムパス。
  • cgiargs: URLエンコードされたCGI引数文字列。
  • **出力**:
  • apr_array_header_t*: 解析・展開された引数文字列の配列。末尾は NULL で終端される。

前提条件と不変条件

  • すべての動的メモリ確保は apr_pool_t に依存する。
  • cgiargs は + で区切られた引数を含むと想定される。
  • prep_string 関数による文字列の正規化が適用される。

アルゴリズムまたは振る舞い

1. interp 文字列を走査し、以下のプレースホルダを置換・展開する。

  • $* / %*: cgiargs を + で分割し、各要素をアンエスケープ、正規化、シェルエスケープして配列に追加。
  • $1 / %1: cgiprg をパスとして配列に追加（%1 の場合は / を \ に置換）。

    • 2. プレースホルダ以外の部分は、引用符（"）やバックスラッシュ（\）のエスケープルールに従ってトークン化し、配列に追加。 3. prgtaken（プログラムパスが挿入されたか）が偽の場合、cgiprg を配列に追加。 4. argtaken（引数が挿入されたか）が偽の場合、cgiargs を解析して配列に追加。 5. 配列の末尾に NULL を追加して返却。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_win32.c:514（ap_cgi_build_command 内）。
  • **呼び出し先**: apr_array_make, apr_palloc, ap_getword_nulls, ap_unescape_url, prep_string, ap_escape_shell_cmd, apr_array_push, apr_pstrdup, strchr。

関連データ構造

  • apr_array_header_t: 引数リストを保持するAPR配列。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • interp が空の場合、引数なしの配列が生成される。
  • cgiargs が空の場合、引数展開はスキップされる。
  • 引用符の不整合や複雑なエスケープシーケンスが含まれる場合、Windowsのコマンドライン解析ルールに基づき処理されるが、エスケープ処理には注意が必要。

保守担当者向けメモ

  • prep_string は現在、UTF-8からWindowsローカルコードページへの変換において限定的な実装である。
  • $1 プレースホルダの処理において、短いファイル名（Short Name）への変換が将来的な課題としてコメントされている。
  • 引数解析ロジックはWindowsのコマンドラインルールを模倣しているため、変更時はエスケープ処理の挙動を慎重に検証すること。
