h2_c2_filter_out

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2.c:368

目的

セカンダリ接続（c2）から出力されるデータを、h2_bucket_beam を介して HTTP/2 セッションへ送出するネットワーク出力フィルタ。

シグネチャまたは形状

apr_status_t h2_c2_filter_out(ap_filter_t* f, apr_bucket_brigade* bb)

入力と出力

  • **入力**:
  • f: フィルタコンテキスト（conn_rec を含む）。
  • bb: 送出対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • f->c に関連付けられた h2_conn_ctx_t が存在すること（ap_assert により検証）。
  • bb が NULL の場合、フィルタは即座に成功を返し、必要に応じて data_in_output_filters フラグをリセットする。

アルゴリズムまたは振る舞い

1. bb が NULL の場合、早期リターンする。 2. AP_HAS_RESPONSE_BUCKETS が有効な場合、ブリゲード内のバケットを走査し、ステータスコード 200 以上のレスポンスバケットが見つかれば conn_ctx->has_final_response を 1 に設定する。 3. beam_out を呼び出し、バケットブリゲードを HTTP/2 セッションへ転送する。 4. 転送結果がエラーの場合、h2_c2_abort を呼び出して接続を異常終了させる。 5. 処理終了時にトレースログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache フィルタチェーン（ap_register_output_filter により "H2_C2_NET_OUT" として登録）。
  • **呼び出し先**:
  • h2_conn_ctx_get: コンテキスト取得。
  • beam_out: データ転送の実行。
  • h2_c2_abort: エラー発生時の接続終了処理。
  • ap_log_cerror: ログ出力。

関連データ構造

  • ap_filter_t: フィルタチェーンの構造体。
  • apr_bucket_brigade: 送出データ。
  • h2_conn_ctx_t: 接続ごとの HTTP/2 コンテキスト。

失敗モードと境界条件

  • beam_out がエラーを返した場合、h2_c2_abort が実行され、接続が強制終了される。
  • bb が空の場合、フィルタは何もせず成功を返す。

保守担当者向けメモ

  • 本フィルタは mod_http2 のワーカー側（c2）の出力パイプラインの末端に位置する。
  • AP_HAS_RESPONSE_BUCKETS の有無により、レスポンス状態の追跡ロジックが条件付きでコンパイルされる。
  • 呼び出し元情報の詳細は、modules/http2/h2_c2.c 内の ap_register_output_filter 呼び出し箇所（632行目および943行目）を参照のこと。
