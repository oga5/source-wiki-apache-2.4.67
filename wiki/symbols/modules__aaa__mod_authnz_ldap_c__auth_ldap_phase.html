auth_ldap_phase

定義

  • 種別: enum
  • 定義箇所: modules/aaa/mod_authnz_ldap.c:93

目的

LDAP 処理における現在のフェーズ（認証または認可）を識別し、後続の処理や環境変数設定の挙動を切り替えるために使用されます。

シグネチャまたは形状

enum auth_ldap_phase {
    LDAP_AUTHN, 
    LDAP_AUTHZ
};

入力と出力

  • 入力: 本列挙型は、set_request_vars 関数等の引数として渡されます。
  • 出力: なし（状態識別子として機能）。

前提条件と不変条件

  • LDAP_AUTHN は認証フェーズ、LDAP_AUTHZ は認可フェーズを指します。
  • 実行時にこのいずれかの値が指定されることを前提としています。

アルゴリズムまたは振る舞い

  • set_request_vars 関数において、現在のフェーズが認証か認可かに応じて、LDAP 属性をリクエスト環境変数（subprocess_env）へマッピングする際のロジックを制御します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/aaa/mod_authnz_ldap.c:376 の set_request_vars 関数にて引数として利用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • set_request_vars: 本列挙型を引数に取る関数。

失敗モードと境界条件

  • 定義外の値を渡した場合の挙動については、ソースコード上に明示的なガード条件は確認できません。

保守担当者向けメモ

  • 本列挙型は mod_authnz_ldap.c 内の内部的な状態管理に使用されています。
  • 新たな LDAP フェーズが追加される場合は、この列挙型を拡張する必要があります。
