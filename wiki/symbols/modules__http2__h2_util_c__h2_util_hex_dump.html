h2_util_hex_dump

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_util.c:64

目的

バイナリデータをデバッグ出力用に 16 進数文字列へ変換し、指定されたバッファに格納します。

シグネチャまたは形状

size_t h2_util_hex_dump(char *buffer, size_t maxlen, const char *data, size_t datalen)

入力と出力

  • **入力**:
  • buffer: 変換後の文字列を格納するバッファ。
  • maxlen: buffer の最大サイズ。
  • data: 16 進数化するバイナリデータ。
  • datalen: data のバイト長。
  • **出力**:
  • 戻り値: buffer に書き込まれた文字列の長さ（strlen(buffer)）。

前提条件と不変条件

  • maxlen は 4 以上である必要があります（内部で maxlen - 4 を計算するため）。
  • buffer は、少なくとも maxlen バイトの書き込み可能な領域を確保している必要があります。

アルゴリズムまたは振る舞い

1. maxlen - 4 を上限として、data の各バイトを 16 進数（%2x）で buffer に書き込みます。 2. 16 バイトごとに改行文字 \n を挿入し、それ以外はスペースで区切ります。 3. datalen が maxlen の制限を超えた場合、文字列の末尾に ... を付加します。 4. 最後に strlen(buffer) を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:1105
  • **呼び出し先**:
  • apr_snprintf (APR ライブラリ)
  • strcpy (標準 C ライブラリ)
  • strlen (標準 C ライブラリ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • maxlen が 4 未満の場合、maxoffset が負の値となり、予期せぬ動作を引き起こす可能性があります。
  • data が NULL の場合、未定義の動作となります。

保守担当者向けメモ

  • 本関数はデバッグ用途に特化しています。ログ出力のフォーマットを統一する目的で使用されています。
  • apr_snprintf を使用しているため、バッファオーバーフローの防止は考慮されていますが、呼び出し側で maxlen を適切に設定してください。
