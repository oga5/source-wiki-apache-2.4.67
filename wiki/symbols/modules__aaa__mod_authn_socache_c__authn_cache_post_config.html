authn_cache_post_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:86

目的

Apache HTTP Server の設定完了フェーズにおいて、認証キャッシュ用の共有オブジェクトキャッシュ（socache）インスタンスの初期化およびグローバルミューテックスの作成を行う。

シグネチャまたは形状

static int authn_cache_post_config(apr_pool_t *pconf, apr_pool_t *plog,
                                   apr_pool_t *ptmp, server_rec *s)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • plog: ログ出力用メモリプール。
  • ptmp: 一時メモリプール。
  • s: サーバーレコード。
  • **出力**:
  • 成功時は OK を返す。
  • 失敗時は 500（内部エラー）を返す。

前提条件と不変条件

  • configured フラグが真であること。
  • socache_provider が設定済みであること（未設定の場合はエラー）。
  • socache_instance が未作成の場合、socache_provider->create を介して初期化される。

アルゴリズムまたは振る舞い

1. **初期化チェック**: configured が偽であれば即座に OK を返す。 2. **プロバイダー検証**: socache_provider が存在しない場合、エラーログを出力し 500 を返す。 3. **インスタンス作成**: socache_instance が未初期化の場合、socache_provider->create を呼び出す。失敗時は 500 を返す。 4. **ミューテックス作成**: ap_global_mutex_create を使用して authn_cache_mutex を生成。失敗時は 500 を返す。 5. **クリーンアップ登録**: remove_lock をプールクリーンアップに登録。 6. **キャッシュ初期化**: socache_provider->init を実行し、authn_cache_hints を適用。失敗時は 500 を返す。 7. **終了処理**: destroy_cache をプールクリーンアップに登録し、OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_config により、サーバー起動時の設定完了フェーズで呼び出される。
  • **呼び出し先**:
  • socache_provider->create
  • socache_provider->init
  • ap_global_mutex_create
  • apr_pool_cleanup_register
  • ap_log_perror

関連データ構造

  • ap_socache_hints: キャッシュの初期化パラメータ（サイズ、エントリ数等）。
  • authn_cache_mutex: キャッシュアクセスを保護するグローバルミューテックス。
  • socache_provider: キャッシュバックエンドの関数テーブル。
  • socache_instance: キャッシュのインスタンス状態。

失敗モードと境界条件

  • socache_provider が未設定の場合、初期化に失敗し 500 を返す。
  • socache_provider->create または init が失敗した場合、エラーログを出力し 500 を返す。
  • ap_global_mutex_create が失敗した場合、エラーログを出力し 500 を返す。

保守担当者向けメモ

  • 本関数は post_config フックとして登録されており、サーバー起動時に一度だけ実行される。
  • socache_instance の作成には ptmp（一時プール）が使用されるが、ミューテックスやキャッシュの初期化には pconf（設定プール）が使用される点に注意。
  • remove_lock および destroy_cache が pconf のクリーンアップに登録されており、サーバー停止時のリソース解放を保証している。
