bio_filter_out_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_engine_io.c:113

目的

OpenSSL の BIO（Basic I/O）抽象化レイヤーと Apache HTTP Server のフィルタスタックを接続するための出力用コンテキスト構造体です。暗号化前のデータを保持し、HTTPD のフィルタチェーンへ転送する役割を担います。

シグネチャまたは形状

typedef struct {
    ssl_filter_ctx_t *filter_ctx;
    conn_rec *c;
    apr_bucket_brigade *bb;    /* Brigade used as a buffer. */
    apr_status_t rc;
} bio_filter_out_ctx_t;

入力と出力

  • **入力**: ssl_filter_ctx_t（SSL 接続コンテキスト）および conn_rec（接続レコード）を保持します。
  • **出力**: apr_bucket_brigade を介して、暗号化前のデータを HTTPD の出力フィルタへ渡します。rc フィールドには直近の操作ステータスが格納されます。

前提条件と不変条件

  • BIO_set_data を使用して OpenSSL の BIO オブジェクトに関連付けられる必要があります。
  • bb（バケットブリゲード）は、接続のプール（c->pool）に関連付けられ、フィルタのライフサイクルと同期して管理されます。

アルゴリズムまたは振る舞い

OpenSSL が SSL_write を実行すると、このコンテキストを介してデータが apr_bucket_brigade に蓄積されます。その後、bio_filter_out_pass 等の関数を通じて、蓄積されたデータが ap_pass_brigade により下位の HTTPD フィルタへ渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: bio_filter_out_ctx_new によりインスタンス化されます。また、bio_filter_out_pass、bio_filter_out_write、bio_filter_out_flush、bio_filter_out_read などの BIO メソッド実装内で BIO_get_data を通じて参照されます。
  • **呼び出し先**: apr_palloc（メモリ確保）、ap_pass_brigade（HTTPD フィルタへのデータ転送）がこの構造体を利用します。

関連データ構造

  • ssl_filter_ctx_t: SSL 接続のメインコンテキスト。
  • conn_rec: HTTPD の接続状態を保持する構造体。
  • apr_bucket_brigade: データのバッファリングに使用される APR のデータ構造。

失敗モードと境界条件

  • rc フィールドにより、バケットの転送や書き込み操作における apr_status_t エラーを追跡します。
  • メモリ不足時には apr_palloc が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は BIO_set_data で BIO に格納されるため、BIO の破棄時に適切にクリーンアップされる必要があります。
  • bb は暗号化前のデータを一時的に保持するバッファとして機能するため、メモリリークを防ぐためにブリゲードの適切なクリアと破棄が重要です。
