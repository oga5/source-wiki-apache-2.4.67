ap_bucket_eor_create

定義

  • 種別: function
  • 定義箇所: server/eor_bucket.c:68

目的

リクエスト処理の終了（End of Request: EOR）を通知するためのメタデータバケットを生成し、リクエストプールに関連付けられたクリーンアップ処理を登録します。

シグネチャまたは形状

apr_bucket *ap_bucket_eor_create(apr_bucket_alloc_t *list, request_rec *r)

入力と出力

  • **入力**:
  • list: バケット割り当て用のメモリプール（apr_bucket_alloc_t）。
  • r: リクエストの状態を保持する request_rec 構造体。
  • **出力**:
  • 初期化済みの apr_bucket 構造体へのポインタ。

前提条件と不変条件

  • r が NULL でない場合、リクエストプール（r->pool）に対して eor_bucket_cleanup が apr_pool_pre_cleanup_register を介して登録されます。
  • バケット自体はデータを含まず、イベント通知のみを目的とします。

アルゴリズムまたは振る舞い

1. apr_bucket_alloc を使用してバケット構造体を割り当てます。 2. APR_BUCKET_INIT でバケットを初期化し、解放関数を apr_bucket_free に設定します。 3. ap_bucket_eor_make を呼び出し、バケットの型を EOR バケットとして構成します。 4. r が存在する場合、apr_pool_pre_cleanup_register を使用して eor_bucket_cleanup をリクエストプールに登録します。

  • **理由**: EOR バケットがリクエストプールとは異なるプール（親プールなど）から割り当てられる可能性があるため、リクエストプールが先に破棄された場合でも安全にクリーンアップを実行するためです。また、log_transaction フックでサブプールが必要になる可能性があるため、pre_cleanup を使用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_request.c
  • modules/http2/h2_c2_filter.c
  • modules/http2/h2_request.c
  • server/protocol.c
  • **呼び出し先**:
  • apr_bucket_alloc
  • APR_BUCKET_INIT
  • ap_bucket_eor_make
  • apr_pool_pre_cleanup_register
  • eor_bucket_cleanup (登録先として)

関連データ構造

  • apr_bucket: バケットの基本構造体。
  • request_rec: リクエストのライフサイクルを管理する構造体。
  • ap_bucket_eor: EOR バケット固有のデータ構造。

失敗モードと境界条件

  • r が NULL の場合、クリーンアップ処理の登録はスキップされます。
  • メモリ不足時には apr_bucket_alloc が失敗する可能性があります。

保守担当者向けメモ

  • eor_bucket_cleanup 内で *rp = NULL とすることで、バケット破棄とプール破棄の順序が前後しても二重解放が発生しないよう設計されています。
  • このバケットはデータ転送用ではなく、フィルタチェーンを通じたイベント伝播用であることに注意してください。
