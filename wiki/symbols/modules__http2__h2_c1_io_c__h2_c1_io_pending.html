h2_c1_io_pending

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:334

目的

HTTP/2 クライアント接続（C1）において、送信待ちのデータがバッファ内に存在するかどうかを判定します。

シグネチャまたは形状

int h2_c1_io_pending(h2_c1_io *io)

入力と出力

  • **入力**: h2_c1_io *io - 接続の入出力状態を管理する構造体へのポインタ。
  • **出力**: int - 送信待ちデータがある場合は非ゼロ（真）、ない場合は 0（偽）を返します。

前提条件と不変条件

  • 引数 io は有効な h2_c1_io 構造体を指している必要があります。
  • io->output（ブリゲード）および io->scratch（スクラッチバッファ）は、セッションのライフサイクル内で適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

以下のいずれかの条件を満たす場合に、送信保留中とみなします。 1. io->output ブリゲードが空ではない場合。 2. io->scratch バッファが割り当てられており、かつ io->slen > 0（スクラッチ領域にデータが存在する）場合。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1_io.c:343
  • modules/http2/h2_c1_io.c:353
  • modules/http2/h2_session.c:1343
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わず、構造体のメンバ参照のみを行います。

関連データ構造

  • h2_c1_io: 接続の入出力状態を保持する構造体。
  • apr_bucket_brigade: Apache のデータ転送用バケットコンテナ。

失敗モードと境界条件

  • io が NULL の場合の挙動は定義されていません（セグメンテーション違反の可能性があります）。
  • io->output が初期化されていない場合、APR_BRIGADE_EMPTY マクロの挙動は未定義です。

保守担当者向けメモ

  • 本関数は、接続のフラッシュ制御やセッションのアイドル判定において頻繁に使用されます。
  • io->scratch の状態と io->output の状態を統合的に判断する唯一のインターフェースであるため、バッファリングロジックを変更する際は本関数の条件式を再確認してください。
