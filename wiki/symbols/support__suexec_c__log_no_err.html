log_no_err

定義

  • 種別: function
  • 定義箇所: support/suexec.c:212

目的

suexec の実行ログを記録するためのラッパー関数です。エラー以外の通常のトランザクション情報を記録するために使用されます。

シグネチャまたは形状

static void log_no_err(const char *fmt, ...)

入力と出力

  • **入力**:
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（void）。

前提条件と不変条件

  • コンパイル時に AP_LOG_EXEC マクロが定義されている場合にのみ、ログ出力処理が有効化されます。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. err_output(0, fmt, ap) を呼び出し、ログ出力の実体へ処理を委譲します。第1引数に 0 を渡すことで、エラーではない（!is_error）ことを明示します。 3. va_end を使用して引数リストを終了します。 4. AP_LOG_EXEC が定義されていない場合、何も行わずに制御を戻します。

呼び出し元と呼び出し先

  • **呼び出し元**: support/suexec.c:475（実行時の詳細ログ出力）。
  • **呼び出し先**: err_output（support/suexec.c 内で定義）。

関連データ構造

  • va_list: 可変長引数を処理するための標準ライブラリ型。

失敗モードと境界条件

  • AP_LOG_EXEC が定義されていない環境では、この関数は実質的に何もしない（No-op）関数として振る舞います。
  • ログ出力先（syslog またはファイル）への書き込み失敗時の挙動は err_output の実装に依存します。

保守担当者向けメモ

  • 本関数は log_err と対になる関数です。ログの記録方式を変更する場合は err_output を修正してください。
  • AP_LOG_EXEC の定義状況により挙動が大きく変わるため、デバッグ時にはビルド設定を確認してください。
