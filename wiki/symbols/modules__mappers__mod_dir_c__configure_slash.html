configure_slash

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_dir.c:84

目的

Apache HTTP Server の設定ディレクティブ DirectorySlash を解析し、ディレクトリリクエストの末尾スラッシュ補完機能を有効または無効に設定します。

シグネチャまたは形状

static const char *configure_slash(cmd_parms *cmd, void *d_, int arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • d_: 現在のディレクトリ設定構造体 (dir_config_rec)。
  • arg: DirectorySlash ディレクティブの値（On であれば非ゼロ、Off であれば 0）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • d_ は dir_config_rec 型のポインタとしてキャスト可能である必要があります。
  • arg に基づき、設定構造体の do_slash フィールドが MODDIR_ON または MODDIR_OFF のいずれかに更新されます。

アルゴリズムまたは振る舞い

1. 引数 d_ を dir_config_rec *d にキャストします。 2. arg が真（On）であれば d->do_slash を MODDIR_ON に設定します。 3. arg が偽（Off）であれば d->do_slash を MODDIR_OFF に設定します。 4. NULL を返して処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_dir.c:133 において、AP_INIT_FLAG マクロを通じて DirectorySlash ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: 外部関数は呼び出していません。

関連データ構造

  • dir_config_rec: ディレクトリごとの設定を保持する構造体。do_slash フィールドを介して本関数と連携します。

失敗モードと境界条件

  • 本関数自体はエラーチェックを行わず、常に NULL を返します。
  • d_ が NULL の場合、デリファレンスによりクラッシュする可能性があります（Apache の設定フレームワークにより、通常は有効なポインタが渡されます）。

保守担当者向けメモ

  • 本関数は DirectorySlash ディレクティブ専用のセッターです。設定の変更は dir_config_rec 構造体を通じて行われ、後の fixup_dir フック処理時に参照されます。
