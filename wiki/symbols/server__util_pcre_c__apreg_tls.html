apreg_tls

定義

  • 種別: struct
  • 定義箇所: server/util_pcre.c:318

目的

スレッドローカルストレージ（TLS）において、正規表現のマッチングに使用するワークスペース（マッチデータ）と、そのバッファサイズを保持するために使用されます。

シグネチャまたは形状

struct apreg_tls {
    match_data_pt data;
    apr_size_t size;
};

入力と出力

  • **入力**: 構造体内の data フィールドはマッチング結果を格納する領域を指し、size は確保されたバッファの容量を示します。
  • **出力**: なし（本構造体は状態保持のためのコンテナです）。

前提条件と不変条件

  • data フィールドは、PCRE のマッチング処理に必要なワークスペース（PCRE2 の match_data や PCRE1 の ovector）を指す必要があります。
  • size は、現在割り当てられている data の有効なサイズと一致している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、正規表現マッチング時に get_match_data 関数を通じて TLS から取得されます。
  • マッチング処理において、既存の size が要求されるバッファサイズを満たしている場合、data を再利用することで malloc/free のオーバーヘッドを削減します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_pcre.c 内の TLS を利用するマッチング関数群（get_match_data 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • match_data_pt: マッチング用ワークスペースの型定義。
  • apr_size_t: APR で定義されたサイズ型。

失敗モードと境界条件

  • メモリ不足時: data の再確保に失敗した場合、マッチング処理がエラーを返す可能性があります。
  • スレッド環境: AP_HAS_THREAD_LOCAL が定義されていない環境では、本構造体による TLS 最適化は機能しません。

保守担当者向けメモ

  • 本構造体は server/util_pcre.c 内でのみ使用される内部的なデータ構造です。
  • PCRE1 から PCRE2 への移行に伴い、match_data_pt の実体が変更される可能性があるため、構造体の変更時は get_match_data および free_match_data との整合性に注意してください。
