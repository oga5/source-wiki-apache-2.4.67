alloc_reptr

定義

  • 種別: function
  • 定義箇所: modules/filters/sed0.c:1002

目的

sed コマンドを保持するための新しい sed_reptr_t 構造体をメモリプールから割り当て、コンパイル中のコマンドリストの末尾に連結します。

シグネチャまたは形状

static sed_reptr_t *alloc_reptr(sed_commands_t *commands)

入力と出力

  • **入力**: commands (sed_commands_t *) - 現在のコンパイル状態を保持する構造体。
  • **出力**: 成功時は新しく割り当てられた sed_reptr_t * を返し、失敗時は 0 を返します。

前提条件と不変条件

  • commands->pool が有効な apr_pool_t であること。
  • commands->nrep は、割り当てられたコマンドの通し番号としてインクリメントされます。
  • commands->ptrspace がリストの先頭、commands->ptrend がリストの末尾を指す不変条件を維持します。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して sed_reptr_t サイズのメモリをゼロ初期化して確保します。 2. メモリ確保に失敗した場合は command_errf を呼び出し、エラーメッセージを出力して 0 を返します。 3. 確保した構造体に nrep を設定し、findex を -1 に初期化します。 4. commands->nrep をインクリメントします。 5. リストが空であれば ptrspace に、空でなければ現在の ptrend->next に新しいノードを接続します。 6. ptrend を新しいノードに更新し、labtab->address を新しいノードに設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/sed0.c 内の各コマンド解析関数（行番号 77, 299, 656 など）。
  • **呼び出し先**: apr_pcalloc (APR), command_errf (エラー報告)。

関連データ構造

  • sed_commands_t: コンパイル状態とコマンドリストの管理。
  • sed_reptr_t: 個別の sed コマンドノード。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足で NULL を返した場合、command_errf を経由してエラーが報告されます。

保守担当者向けメモ

  • 本関数は sed コマンドの連結リストを構築する中心的な役割を担います。
  • labtab->address への代入は、ラベル解決の過程で参照される可能性があるため、変更時はラベル管理ロジックへの影響を確認してください。
