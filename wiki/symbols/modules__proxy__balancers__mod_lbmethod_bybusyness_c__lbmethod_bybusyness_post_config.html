lbmethod_bybusyness_post_config

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_bybusyness.c:89

目的

mod_lbmethod_bybusyness モジュールの初期化を行い、mod_proxy が提供するワーカー選定用のオプション関数を取得・登録します。

シグネチャまたは形状

static int lbmethod_bybusyness_post_config(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • pconf, plog, ptemp: Apache のメモリプール。
  • s: サーバーレコード。
  • **出力**:
  • OK (0): 正常終了。
  • !OK (非0): mod_proxy がロードされていない場合の異常終了。

前提条件と不変条件

  • 本関数はサーバー起動時に2回呼び出されます。
  • AP_SQ_MS_CREATE_PRE_CONFIG 状態での初回呼び出し時は、静的データの初期化を行わずに即座に OK を返します。

アルゴリズムまたは振る舞い

1. サーバーの状態を ap_state_query で確認し、初回起動フェーズであれば処理をスキップします。 2. APR_RETRIEVE_OPTIONAL_FN(proxy_balancer_get_best_worker) を呼び出し、mod_proxy からワーカー反復処理用の関数ポインタを取得します。 3. 関数ポインタの取得に失敗した場合、エラーログ（APLOGNO(10151)）を出力し、異常終了します。 4. 正常に取得できた場合、OK を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の post_config フック（modules/proxy/balancers/mod_lbmethod_bybusyness.c:113 にて登録）。
  • **呼び出し先**:
  • ap_state_query
  • APR_RETRIEVE_OPTIONAL_FN (マクロ)
  • ap_log_error

関連データ構造

  • server_rec: サーバー設定情報。
  • ap_proxy_balancer_get_best_worker_fn: mod_proxy から取得したワーカー走査用関数を保持するグローバル変数。

失敗モードと境界条件

  • mod_proxy がロードされていない場合、ap_proxy_balancer_get_best_worker_fn が NULL となり、緊急エラー（APLOG_EMERG）として記録され、初期化に失敗します。

保守担当者向けメモ

  • 本関数は mod_proxy への依存関係を解決するためのガードとして機能します。mod_proxy のロード順序が本モジュールより後になる構成では初期化に失敗するため、設定順序に注意が必要です。
