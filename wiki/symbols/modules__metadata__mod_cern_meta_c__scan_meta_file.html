scan_meta_file

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_cern_meta.c:208

目的

指定されたメタファイルの内容を読み込み、解析して HTTP レスポンスヘッダー（r->headers_out）に反映させる。

シグネチャまたは形状

static int scan_meta_file(request_rec *r, apr_file_t *f)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • f: 読み込み対象のメタファイルハンドル。
  • **出力**:
  • 戻り値: 成功時は OK、解析エラー時は HTTP_INTERNAL_SERVER_ERROR を返す。
  • 副作用: r->headers_out へのヘッダー追加、r->status および r->status_line の更新、ap_set_content_type による MIME タイプの変更。

前提条件と不変条件

  • r->pool が有効であり、メモリ割り当てに使用される。
  • メタファイルは行単位で構成され、各行は ヘッダー名: 値 の形式である必要がある。
  • ファイル内の空行は処理の終了条件として扱われる。

アルゴリズムまたは振る舞い

1. 一時的なテーブル tmp_headers を作成する。 2. ファイルから行単位で読み込み、末尾の改行コード（CRLF または LF）を除去する。 3. 空行を検出した場合は即座に OK を返して終了する。 4. : を区切り文字としてヘッダー名と値を分離する。分離できない場合はエラーログを出力し、HTTP_INTERNAL_SERVER_ERROR を返す。 5. ヘッダー名が Content-type の場合、値を小文字化し ap_set_content_type を呼び出す。 6. ヘッダー名が Status の場合、値を数値として r->status に、文字列として r->status_line に設定する。 7. それ以外のヘッダーは tmp_headers に格納する。 8. ループ終了後、apr_table_overlap を使用して tmp_headers を r->headers_out にマージする。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_cern_meta.c:351 (関数 add_cern_meta_data 内)
  • **呼び出し先**:
  • apr_table_make, apr_table_set, apr_table_overlap (APR テーブル操作)
  • apr_file_gets (ファイル読み込み)
  • ap_log_rerror (エラーログ出力)
  • ap_set_content_type, ap_content_type_tolower (HTTP 応答設定)
  • ap_cstr_casecmp (文字列比較)

関連データ構造

  • request_rec: リクエスト情報およびレスポンスヘッダーを保持。
  • apr_file_t: ファイルハンドル。
  • apr_table_t: ヘッダー情報を一時的に保持するテーブル。

失敗モードと境界条件

  • **形式不正**: 行内に : が含まれない場合、HTTP_INTERNAL_SERVER_ERROR を返し処理を中断する。
  • **空行**: ファイル内の空行は解析の終了点として機能する。
  • **メモリ**: apr_pstrdup 等のメモリ割り当ては r->pool に依存しており、リクエスト終了時に解放される。

保守担当者向けメモ

  • 本関数の解析ロジックは ap_scan_script_header_err_core と類似しているが、コードの重複と経年劣化（bit rot）の可能性がある。
  • Content-type や Status 以外のヘッダーは apr_table_overlap を用いて一括でマージされるため、既存のヘッダーとの上書き優先順位に注意すること。
