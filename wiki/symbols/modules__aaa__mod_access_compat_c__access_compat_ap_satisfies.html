access_compat_ap_satisfies

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_access_compat.c:307

目的

mod_access_compat モジュールで設定された Satisfy ディレクティブの値を、他のモジュールからオプション関数として参照可能にするために提供されます。

シグネチャまたは形状

int access_compat_ap_satisfies(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: int （リクエストのメソッド番号に対応する Satisfy 設定値）

前提条件と不変条件

  • r->per_dir_config に access_compat_module の設定構造体 access_compat_dir_conf が格納されている必要があります。
  • r->method_number は、設定された satisfy 配列のインデックスとして有効な範囲内である必要があります。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、リクエストに関連付けられた access_compat_dir_conf を取得します。 2. 取得した設定構造体内の satisfy 配列から、現在のリクエストメソッド (r->method_number) に対応する値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c において、APR_RETRIEVE_OPTIONAL_FN を介して取得され、コアのアクセス制御ロジックから呼び出されます。
  • **呼び出し先**: ap_get_module_config （Apache HTTP Server API）を呼び出します。

関連データ構造

  • access_compat_dir_conf: ディレクトリごとの設定を保持する構造体。satisfy 配列を含みます。
  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • r->per_dir_config が適切に初期化されていない場合、ap_get_module_config の戻り値が NULL となり、その後のメンバアクセスでセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN を通じて公開されています。シグネチャを変更する場合は、include/http_core.h の宣言と、呼び出し側である server/core.c の実装との整合性に注意してください。
