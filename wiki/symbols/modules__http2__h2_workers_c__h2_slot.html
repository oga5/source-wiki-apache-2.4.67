h2_slot

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_workers.c:60

目的

HTTP/2 サブシステムにおいて、個々のワーカースレッドの状態を保持および管理するための構造体です。

シグネチャまたは形状

struct h2_slot {
    APR_RING_ENTRY(h2_slot) link;
    /* 内部メンバ: スレッドハンドル、条件変数、現在処理中のプロデューサーへの参照等 */
};

※ソースコード上の構造体定義の詳細は、h2_workers.c の該当箇所を参照してください。

入力と出力

  • 入力: h2_workers 管理下でのスレッド生成およびタスク割り当て時に使用されます。
  • 出力: 接続処理の完了後、状態遷移（free, idle, busy, zombie）を経てリングバッファへ戻されます。

前提条件と不変条件

  • h2_workers が保持する workers->lock によって、リングバッファ操作および状態遷移は保護されています。
  • 各スロットは h2_workers のサブプールによってメモリ管理されます。

アルゴリズムまたは振る舞い

  • **状態管理**: APR_RING を用いて、free（空き）、idle（待機中）、busy（実行中）、zombie（終了待ち）の各リスト間を移動します。
  • **タスク実行**: slot_run 関数がこの構造体を介してワーカーのメインループを制御し、プロデューサーから接続を取得して処理を行います。
  • **接続識別**: slot->id は、接続 ID 生成（c->id = (c->master->id << 8)^slot->id）に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: h2_workers 構造体（slots 配列として保持）、activate_slot（新規生成時）、wake_idle_worker（タスク割り当て時）。
  • 呼び出し先: slot_run（ワーカーの実行ロジック）、get_next（タスク取得）。

関連データ構造

  • h2_workers: プール全体の管理構造体。
  • h2_slot_state_t: スロットの状態定義。
  • apr_ring: スロットを管理するための双方向リスト構造。

失敗モードと境界条件

  • **ID 衝突**: slot->id を用いた接続 ID 生成において、極めて高い並列度では衝突の可能性があります。
  • **シャットダウン**: workers_pool_cleanup 実行時に全スロットの終了待機が行われます。

保守担当者向けメモ

  • 本構造体は h2_workers.c 内で定義されており、h2_workers.h で公開されるインターフェースと密接に関連しています。
  • スレッドセーフな操作が必要なため、リング操作を行う際は必ず workers->lock を保持していることを確認してください。
