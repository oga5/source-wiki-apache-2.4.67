CommandLineInterpreter

定義

  • 種別: function
  • 定義箇所: server/mpm/netware/mpm_netware.c:1179

目的

NetWare コンソールから入力されたコマンドを解析し、Apache HTTP Server に対する管理操作（再起動、終了、状態表示など）を実行する。

シグネチャまたは形状

static int CommandLineInterpreter(scr_t screenID, const char *commandLine)

入力と出力

  • **入力**:
  • screenID: コマンドが発行されたコンソール画面の識別子。
  • commandLine: コンソールから入力されたコマンド文字列。
  • **出力**:
  • HANDLEDCOMMAND: コマンドがこのモジュールによって処理された場合。
  • NOTMYCOMMAND: コマンドが対象外であるか、処理対象外のインスタンスである場合。

前提条件と不変条件

  • コマンド文字列は "APACHE2 " で開始する必要がある。
  • 保護メモリ空間で動作している場合、-p オプションで指定されたインスタンスIDが現在のサーバーインスタンスと一致しなければならない。

アルゴリズムまたは振る舞い

1. 入力文字列が "APACHE2 " で始まるか検証し、一致しない場合は NOTMYCOMMAND を返す。 2. インスタンスID（-p）の検証を行い、自インスタンス宛のコマンドかを確認する。 3. コマンド引数に応じて以下の処理を実行する:

  • RESTART: restart() を呼び出しサーバーを再起動。
  • VERSION: サーバーのバージョンとビルド情報を表示。
  • MODULES: ロードされているモジュール一覧を表示。
  • DIRECTIVES: 設定ディレクティブ一覧を表示。
  • SHUTDOWN: shutdown_pending フラグを立てて終了を予約。
  • SETTINGS: スレッド状態の表示切り替えを行う。

    • 4. 不明なコマンドの場合は使用方法（Usage）を表示する。 5. 処理完了後、HANDLEDCOMMAND を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: NetWare コンソールサブシステム（NX_WRAP_INTERFACE を介して登録）。
  • **呼び出し先**: restart(), ap_get_server_description(), ap_get_server_built(), ap_show_modules(), ap_show_directives(), show_server_data(), display_settings()。

関連データ構造

  • shutdown_pending: サーバー終了を制御するグローバルフラグ。
  • ap_my_addrspace: 現在のインスタンスを識別するアドレス空間文字列。
  • show_settings: 設定表示状態を保持するフラグ。

失敗モードと境界条件

  • commandLine が NULL の場合、NOTMYCOMMAND を返す。
  • コマンド文字列が "APACHE2 " より短い場合、NOTMYCOMMAND を返す。
  • 適切なインスタンスIDが指定されていない、または一致しない場合、コマンドを無視して NOTMYCOMMAND を返す。

保守担当者向けメモ

  • 本関数は NX_WRAP_INTERFACE を通じてコンソールハンドラとして登録されている。
  • NetWare のスレッドコンテキストで動作するため、標準出力（printf）はコンソール画面へ出力される。
  • コマンドの追加や変更を行う際は、szCommand のプレフィックスルールと strnicmp による比較ロジックを維持すること。
