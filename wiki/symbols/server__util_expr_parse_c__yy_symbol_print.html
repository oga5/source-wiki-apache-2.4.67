yy_symbol_print

定義

  • 種別: function
  • 定義箇所: server/util_expr_parse.c:826

目的

Bison によって生成されたパーサーにおいて、デバッグ出力用にシンボルの内容を FILE ストリームへ出力します。

シグネチャまたは形状

static void yy_symbol_print (FILE *yyoutput, int yytype, YYSTYPE const * const yyvaluep, ap_expr_parse_ctx_t *ctx);

入力と出力

  • yyoutput: 出力先の FILE ポインタ（通常は stderr）。
  • yytype: 出力対象のシンボル型（トークンまたは非終端記号）。
  • yyvaluep: 出力対象のシンボル値（YYSTYPE 共用体へのポインタ）。
  • ctx: 解析コンテキスト（ap_expr_parse_ctx_t）。
  • 戻り値: なし（void）。

前提条件と不変条件

  • yyoutput は有効な書き込み可能なファイルストリームである必要があります。
  • yyvaluep は yytype に対応する有効な値である必要があります。

アルゴリズムまたは振る舞い

1. 指定された yytype に基づき、シンボルの型名を特定します。 2. yyvaluep が保持する値（文字列、AST ノード、数値など）を、型に応じて yyoutput に出力します。 3. 内部的に yy_symbol_value_print を呼び出し、値の詳細な表示を行います。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/util_expr_parse.c:775
  • server/util_expr_parse.c:900
  • 呼び出し先:
  • yy_symbol_value_print（シンボル値の出力用）

関連データ構造

  • YYSTYPE: パーサーのスタック値（共用体）。
  • ap_expr_parse_ctx_t: 解析コンテキスト。
  • FILE: 標準 C ライブラリのファイルストリーム。

失敗モードと境界条件

  • yyoutput が NULL の場合、未定義の動作を引き起こす可能性があります。
  • yytype が未知の型である場合、シンボル名として unknown 等の汎用的な出力が行われる可能性があります。

保守担当者向けメモ

  • 本関数は Bison が生成するデバッグ用コードの一部です。直接編集せず、util_expr_parse.y の文法定義や Bison の設定を介して制御してください。
  • 構文解析のトレースが必要な場合にのみ使用されます。
