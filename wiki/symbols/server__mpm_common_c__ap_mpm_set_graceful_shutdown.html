ap_mpm_set_graceful_shutdown

定義

  • 種別: function
  • 定義箇所: server/mpm_common.c:369

目的

Apache HTTP Server の設定ディレクティブ GracefulShutdownTimeout を処理し、グローバルな猶予期間（タイムアウト値）を設定します。

シグネチャまたは形状

const char *ap_mpm_set_graceful_shutdown(cmd_parms *cmd, void *dummy, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのコンテキスト情報。
  • dummy: 使用されていません。
  • arg: 設定ファイルから渡されるタイムアウト値（文字列）。
  • **出力**:
  • 成功時: NULL を返します。
  • 失敗時: エラーメッセージ（文字列）を返します。

前提条件と不変条件

  • 本関数は、設定コンテキストが GLOBAL_ONLY であることを要求します。これに違反する場合、ap_check_cmd_context によりエラーが返されます。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、現在の設定コンテキストがグローバルスコープであることを検証します。 2. 検証に失敗した場合は、エラーメッセージを即座に返します。 3. 検証に成功した場合、引数 arg を atoi で整数に変換し、グローバル変数 ap_graceful_shutdown_timeout に代入します。 4. NULL を返し、正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • include/mpm_common.h 内の AP_INIT_TAKE1 マクロにより、設定ファイル内の GracefulShutdownTimeout ディレクティブから呼び出されます。
  • **呼び出し先**:
  • ap_check_cmd_context (Apache HTTP Server コア API)
  • atoi (標準 C ライブラリ)

関連データ構造

  • cmd_parms: 設定コマンドのパラメータを保持する構造体。
  • ap_graceful_shutdown_timeout: 本関数によって更新されるグローバル変数。

失敗モードと境界条件

  • **コンテキストエラー**: グローバルスコープ以外でディレクティブが使用された場合、ap_check_cmd_context がエラーを返します。
  • **入力値**: arg が数値として解釈できない場合、atoi の挙動に従い 0 が設定されます。

保守担当者向けメモ

  • 本関数は GracefulShutdownTimeout ディレクティブのハンドラです。
  • ap_graceful_shutdown_timeout 変数の利用箇所については、本ファイルおよび関連する MPM 実装を確認してください。
