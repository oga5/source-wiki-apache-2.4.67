apl_toscope

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:39

目的

Lua スクリプトで指定されたスコープ文字列を、Apache Lua モジュール内部で使用される整数定数へ変換します。

シグネチャまたは形状

static int apl_toscope(const char *name)

入力と出力

  • **入力**: name (文字列) - スコープ名（"once", "request", "connection", "conn", "thread"）。
  • **出力**: int - 対応する AP_LUA_SCOPE_* 定数。

前提条件と不変条件

  • 入力文字列が既知のスコープ名と一致しない場合、デフォルト値として AP_LUA_SCOPE_ONCE を返します。

アルゴリズムまたは振る舞い

1. 引数 name を既知のスコープ文字列と比較します。 2. 一致するものがあれば、対応する定数を返します。

  • "once" → AP_LUA_SCOPE_ONCE
  • "request" → AP_LUA_SCOPE_REQUEST
  • "connection" または "conn" → AP_LUA_SCOPE_CONN
  • "thread" → AP_LUA_SCOPE_THREAD

    • 3. いずれにも該当しない場合、AP_LUA_SCOPE_ONCE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_config.c:73
  • modules/lua/lua_config.c:114
  • **呼び出し先**:
  • strcmp (標準 C ライブラリ)

関連データ構造

  • AP_LUA_SCOPE_* 定数（mod_lua.h 等で定義されるスコープ識別子）

失敗モードと境界条件

  • name が NULL の場合、strcmp の挙動に依存します（セグメンテーション違反の可能性があります）。
  • 未定義の文字列が渡された場合、エラーにはならずデフォルトの AP_LUA_SCOPE_ONCE が返されます。

保守担当者向けメモ

  • 新しいスコープタイプを追加する場合は、本関数の条件分岐にケースを追加してください。
  • AP_LUA_SCOPE_CONN に対して "connection" と "conn" の両方が割り当てられている点に留意してください。
