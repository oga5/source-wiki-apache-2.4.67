ap_cache_check_no_cache

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_util.c:450

目的

RFC 2616 14.9.4 に基づき、クライアントからのリクエストにキャッシュの利用を禁止する指示（Cache-Control: no-cache または Pragma: no-cache）が含まれているかを判定します。

シグネチャまたは形状

int ap_cache_check_no_cache(cache_request_rec *cache, request_rec *r);

入力と出力

  • **入力**:
  • cache: リクエストに関連付けられたキャッシュコンテキスト。
  • r: 現在の HTTP リクエスト構造体。
  • **出力**:
  • 0: キャッシュを利用してはならない（no-cache が指定されており、かつ無視設定が無効）。
  • 1: キャッシュを利用可能（no-cache が指定されていない、または無視設定が有効）。

前提条件と不変条件

  • cache->control_in が未解析の場合、リクエストヘッダーから Cache-Control および Pragma を取得し、解析を実行します。
  • cache_server_conf の ignorecachecontrol 設定が有効な場合、クライアントの no-cache 指示を無視してキャッシュを返却する挙動をとります。

アルゴリズムまたは振る舞い

1. リクエストヘッダーから Cache-Control および Pragma を取得し、ap_cache_control を用いて cache->control_in に格納します。 2. cache->control_in.no_cache が真の場合:

  • ignorecachecontrol 設定が有効であれば、ログを出力し 1 を返します。
  • 設定が無効であれば、キャッシュ利用不可として 0 を返します。

    • 3. 上記以外の場合は 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/cache_storage.c:231
  • **呼び出し先**:
  • cache_table_getm (ヘッダー取得)
  • ap_cache_control (ヘッダー解析)
  • ap_get_module_config (設定取得)
  • ap_log_rerror (ログ出力)

関連データ構造

  • cache_request_rec: キャッシュ制御状態を保持。
  • cache_server_conf: キャッシュモジュールのサーバー設定（ignorecachecontrol を保持）。
  • request_rec: HTTP リクエスト情報。

失敗モードと境界条件

  • ignorecachecontrol が有効な場合、RFC 準拠の no-cache 指示が無視されるため、クライアントが期待する「最新のレスポンス」ではなく「キャッシュされた古いレスポンス」が返される可能性があります。

保守担当者向けメモ

  • 本関数は RFC 2616 14.9.4 に準拠したエンドツーエンドの再検証制御を実装しています。
  • cache->control_in.parsed フラグにより、リクエスト処理中に複数回呼び出されてもヘッダー解析が重複しないよう最適化されています。
