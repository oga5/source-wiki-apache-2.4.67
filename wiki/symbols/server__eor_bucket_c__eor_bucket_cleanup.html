eor_bucket_cleanup

定義

  • 種別: function
  • 定義箇所: server/eor_bucket.c:27

目的

リクエスト終了時（End of Request）に、リクエストに関連する統計情報の更新およびログ記録を実行し、リソースのクリーンアップを完了させるために使用されます。

シグネチャまたは形状

static apr_status_t eor_bucket_cleanup(void *data)

入力と出力

  • **入力**: void *data。実際には request_rec ** 型のポインタを受け取ります。
  • **出力**: apr_status_t 型のステータスコード（常に APR_SUCCESS を返します）。

前提条件と不変条件

  • data は request_rec 構造体へのポインタを保持するポインタである必要があります。
  • *rp が NULL の場合、本関数は何も行わず即座に終了します。

アルゴリズムまたは振る舞い

1. 引数 data を request_rec **rp にキャストします。 2. *rp が有効な場合、以下の処理を実行します：

  • *rp を NULL に設定し、後続の eor_bucket_destroy による二重のクリーンアップを防止します。
  • ap_update_child_status を呼び出し、スコアボード上の子プロセスの状態を SERVER_BUSY_LOG に更新します。
  • ap_run_log_transaction を呼び出し、リクエストのログ記録を実行します。
  • ap_extended_status が有効な場合、ap_increment_counts を呼び出して統計情報を更新します。

    • 3. APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: apr_pool_pre_cleanup_register を介して、リクエストプール（r->pool）の破棄時に自動的に呼び出されます（server/eor_bucket.c:88）。
  • **呼び出し先**:
  • ap_update_child_status
  • ap_run_log_transaction
  • ap_increment_counts

関連データ構造

  • request_rec: リクエストの状態を保持する構造体。
  • apr_pool_t: リクエストのライフサイクルを管理するメモリプール。

失敗モードと境界条件

  • *rp が NULL の場合、処理はスキップされます。これは eor_bucket_destroy が先に実行された場合を想定した安全策です。

保守担当者向けメモ

  • 本関数は apr_pool_pre_cleanup_register を使用して登録されています。これは、リクエスト終了時にモジュールがサブプールを必要とする可能性があるためです。
  • *rp = NULL とすることで、バケット破棄とプール破棄の順序が前後しても安全に動作するよう設計されています。
