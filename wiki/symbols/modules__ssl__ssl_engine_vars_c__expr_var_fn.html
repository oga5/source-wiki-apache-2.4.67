expr_var_fn

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:85

目的

Apache HTTP Server の設定式（ap_expr）評価エンジンに対し、SSL 関連の変数値を解決して提供するコールバック関数です。

シグネチャまたは形状

static const char *expr_var_fn(ap_expr_eval_ctx_t *ctx, const void *data)

入力と出力

  • **入力**:
  • ctx: ap_expr_eval_ctx_t 構造体へのポインタ。評価コンテキスト（リクエスト r や接続 c を含む）。
  • data: 解決対象の変数名を示す文字列（char *）。
  • **出力**:
  • 解決された変数値を表す文字列（const char *）。解決できない場合や SSL 接続が存在しない場合は NULL を返す。

前提条件と不変条件

  • ctx->c（接続情報）が有効であること。
  • ssl_get_effective_config が接続から SSLConnRec を取得できること。
  • 戻り値の文字列は、ctx->p（リクエストプール）のメモリ領域で確保されることが期待される（ssl_var_lookup_ssl の実装に依存）。

アルゴリズムまたは振る舞い

1. ctx->c を引数として ssl_get_effective_config を呼び出し、現在の接続に関連付けられた SSLConnRec を取得する。 2. sslconn が存在する場合、ssl_var_lookup_ssl を呼び出し、リクエストコンテキスト ctx->r と変数名 data を用いて値を解決する。 3. sslconn が存在しない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_vars.c:110（ssl_var_register 内で関数ポインタとして登録される）。
  • **呼び出し先**:
  • ssl_get_effective_config
  • ssl_var_lookup_ssl

関連データ構造

  • ap_expr_eval_ctx_t: 式評価のコンテキスト。
  • SSLConnRec: SSL 接続固有の情報を保持する構造体。

失敗モードと境界条件

  • 接続が SSL を使用していない場合、ssl_get_effective_config が NULL を返し、結果として NULL が返される。
  • ssl_var_lookup_ssl が指定された変数名を解決できない場合、その実装に従い NULL または空文字列が返される。

保守担当者向けメモ

  • 本関数は ap_expr の変数解決フックとして登録されるため、シグネチャの変更は ap_expr の仕様と整合させる必要があります。
  • ssl_var_lookup_ssl への委譲を行っているため、変数解決ロジックの詳細は ssl_var_lookup_ssl を参照してください。
