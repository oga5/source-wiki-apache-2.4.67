event_check_config

定義

  • 種別: function
  • 定義箇所: server/mpm/event/event.c:3776

目的

Apache HTTP Server の起動および再起動時に、Event MPM 関連の設定値（ServerLimit, ThreadLimit, ThreadsPerChild, MaxRequestWorkers, StartServers, MinSpareThreads）が妥当な範囲内にあるか検証し、必要に応じて自動補正を行う。

シグネチャまたは形状

static int event_check_config(apr_pool_t *p, apr_pool_t *plog,
                              apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • p, plog, ptemp: APR プール。
  • s: 現在のサーバー設定レコード。
  • **出力**:
  • 常に OK を返す。設定値の補正はグローバル変数への直接代入によって行われる。

前提条件と不変条件

  • retained グローバル構造体が初期化されていること。
  • ServerLimit および ThreadLimit は、再起動時に変更できない（初回起動時の値が保持される）。
  • MaxRequestWorkers は ThreadsPerChild の整数倍である必要がある。

アルゴリズムまたは振る舞い

1. **起動判定**: retained->mpm->module_loads を確認し、初回起動時のみ startup フラグを立てる（ログ出力の形式を制御するため）。 2. **制限値の検証**:

  • ServerLimit, ThreadLimit がコンパイル時の最大値を超えていれば補正し、1未満であれば1に補正する。
  • 再起動時に ServerLimit や ThreadLimit が変更されていた場合、警告を出して元の値に強制的に戻す。

    • 3. **スレッド・プロセス数の整合性チェック**:

  • ThreadsPerChild が ThreadLimit を超えないよう調整。
  • MaxRequestWorkers が ThreadsPerChild 未満であれば引き上げる。
  • MaxRequestWorkers を ThreadsPerChild の倍数に切り下げる。
  • active_daemons_limit（必要サーバー数）が ServerLimit を超えないよう調整。

    • 4. **最小値の保証**: StartServers および MinSpareThreads が 1 未満の場合、1 に補正する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_check_config を介して Apache 設定フェーズから呼び出される（server/mpm/event/event.c:4006）。
  • **呼び出し先**: ap_log_error（エラーログ出力用）。

関連データ構造

  • server_rec: サーバー設定情報。
  • retained: MPM のライフサイクルをまたいで保持されるデータ構造。

失敗モードと境界条件

  • 設定値が許容範囲外の場合、エラー終了はせず、警告ログを出力した上で安全な値へ自動補正する。
  • 再起動時の設定変更は無視され、初回起動時の値が優先される。

保守担当者向けメモ

  • 本関数は設定の整合性を保つためのガードレールとして機能する。新しい MPM 設定パラメータを追加する際は、ここで同様の境界チェックを行うこと。
  • startup フラグはコンソールへの出力有無を制御するため、ログメッセージの重要度に応じて適切に使い分けること。
