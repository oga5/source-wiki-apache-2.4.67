win32_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/arch/win32/mod_win32.c:49

目的

Windows環境におけるディレクトリ単位のCGI実行設定を保持するための構造体です。特に、スクリプトを実行するためのインタープリタをどのように特定するかという戦略を管理します。

シグネチャまたは形状

typedef struct {
    interpreter_source_e script_interpreter_source;
} win32_dir_conf;

入力と出力

  • **入力**: interpreter_source_e 型の列挙値（レジストリ優先、シバン優先など）。
  • **出力**: なし（設定保持用のデータ構造）。

前提条件と不変条件

  • apr_pool_t を用いてメモリが確保されることを前提としています。
  • script_interpreter_source は、ScriptInterpreterSource ディレクティブによって設定される値（registry, registry-strict, script）を保持します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、ロジックは持ちません。この構造体は、create_win32_dir_config で初期化され、merge_win32_dir_configs でディレクトリ間の設定継承に使用されます。また、ap_cgi_build_command が実行される際、リクエストのコンテキストから本構造体を取得し、インタープリタ解決戦略を決定するために参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_win32_dir_config (メモリ確保)
  • merge_win32_dir_configs (設定のマージ)
  • set_interpreter_source (設定値の更新)
  • ap_cgi_build_command (設定の読み取り)
  • **呼び出し先**: 情報なし。

関連データ構造

  • interpreter_source_e: インタープリタ解決戦略を定義する列挙型。

失敗モードと境界条件

  • 本構造体自体に失敗モードはありませんが、ap_get_module_config を通じて取得する際に、ディレクトリ設定が未定義である場合はNULLポインタを参照する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_win32 の設定管理の基盤です。新しい設定項目を追加する場合は、merge_win32_dir_configs におけるマージロジックの更新を忘れないでください。
  • すべてのインスタンスは apr_pool_t のライフサイクルに依存するため、手動でのメモリ解放は不要です。
