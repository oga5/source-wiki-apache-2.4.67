decl_die

定義

  • 種別: function
  • 定義箇所: server/request.c:112

目的

リクエスト処理の各フェーズにおいて、モジュールが DECLINED（処理拒否）を返した際のエラーログ出力およびステータスコードの変換を行う。

シグネチャまたは形状

static int decl_die(int status, const char *phase, request_rec *r)

入力と出力

  • **入力**:
  • status: モジュールから返されたステータスコード。
  • phase: 実行中だった処理フェーズ名（文字列）。
  • r: 現在のリクエスト構造体 (request_rec)。
  • **出力**:
  • int: 最終的な HTTP ステータスコード。

前提条件と不変条件

  • status が DECLINED の場合、設定エラーとみなして HTTP_INTERNAL_SERVER_ERROR を返す。
  • status が DECLINED 以外の場合、その値をそのまま返す。

アルゴリズムまたは振る舞い

1. status が DECLINED と等しいか判定する。 2. DECLINED の場合:

  • APLOG_CRIT レベルで設定エラーログを出力する。
  • HTTP_INTERNAL_SERVER_ERROR を戻り値として返す。

    • 3. DECLINED 以外の場合:

  • APLOG_TRACE3 レベルで認証フェーズのステータスをログ出力する。
  • 入力された status をそのまま戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/request.c 内の各リクエスト処理フェーズ（translate, check user, check authorization, check access, find types 等）。
  • **呼び出し先**: ap_log_rerror（Apache のログ出力関数）。

関連データ構造

  • request_rec: リクエストのコンテキスト情報を保持する構造体。

失敗モードと境界条件

  • status が DECLINED の場合、ログ出力が失敗しても関数は HTTP_INTERNAL_SERVER_ERROR を返す。
  • status が DECLINED 以外の場合、ログ出力の有無に関わらず status がそのまま伝播される。

保守担当者向けメモ

  • 本関数は、リクエスト処理の各フェーズでモジュールが期待通りに動作しなかった場合の標準的なエラーハンドリングを提供します。
  • ログレベルの変更やエラーメッセージの形式変更を行う際は、server/request.c 内の呼び出し箇所全体に影響するため注意してください。
