interpolate_vars

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_proxy_html.c:728

目的

文字列内の ${var} または ${var|default} 形式の変数を、Apache のリクエスト環境変数 (r->subprocess_env) の値で置換する。

シグネチャまたは形状

static const char *interpolate_vars(request_rec *r, const char *str)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • str: 変数展開対象の文字列。
  • **出力**:
  • 展開後の新しい文字列。展開が行われない場合は元の文字列が返される。

前提条件と不変条件

  • r->pool を使用してメモリを割り当てるため、リクエストのライフサイクルに依存する。
  • 変数名が ${ で始まり } で終わる形式であること。
  • | を含む場合、それは ${...} の内部に存在する必要がある。

アルゴリズムまたは振る舞い

1. 文字列内から ${ を検索し、対応する } を探す。 2. | が存在する場合、それを区切り文字として変数名とデフォルト値に分割する。 3. apr_table_get を使用して r->subprocess_env から変数名に対応する値を取得する。 4. 値が見つからない場合、| 以降の文字列をデフォルト値として使用する（| がない場合は空文字列）。 5. apr_pstrcat を用いて、展開前後の文字列と置換値を結合する。 6. 文字列内に展開可能な箇所がなくなるまで、上記処理を繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_proxy_html.c:795
  • modules/filters/mod_proxy_html.c:804
  • **呼び出し先**:
  • ap_strstr_c, ap_strchr_c (文字列検索)
  • apr_pstrmemdup, apr_pstrcat (メモリ管理・文字列結合)
  • apr_table_get (環境変数取得)
  • ap_log_rerror (ログ出力)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • apr_table_t: r->subprocess_env として使用されるテーブル構造。

失敗モードと境界条件

  • ${ や } が正しくペアになっていない場合、その箇所は展開されずに無視される。
  • | が } より後ろにある場合、区切り文字として認識されない。
  • 変数が未定義かつデフォルト値も指定されていない場合、空文字列に置換される。

保守担当者向けメモ

  • 本関数は apr_pool_t を使用して動的に文字列を生成するため、頻繁な呼び出しはメモリ使用量に影響を与える可能性がある。
  • libxml2 の解析処理中に呼び出されるため、パフォーマンス上のボトルネックにならないか注意が必要である。
