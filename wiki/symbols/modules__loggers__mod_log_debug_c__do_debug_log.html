do_debug_log

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_debug.c:56

目的

Apache HTTP Server のリクエスト処理の各フェーズにおいて、設定された条件式（ap_expr）を評価し、その結果が真である場合にデバッグメッセージをエラーログへ出力します。

シグネチャまたは形状

static void do_debug_log(request_rec *r, const char *hookname)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • hookname: 現在実行中のフック名を示す文字列。
  • **出力**: なし（戻り値は void）。副作用としてエラーログへの書き込みを行います。

前提条件と不変条件

  • r->per_dir_config から取得される log_debug_dirconf が有効であること。
  • dconf->entries が初期化されていること（NULL の場合は即座にリターンします）。

アルゴリズムまたは振る舞い

1. リクエストのディレクトリ設定から log_debug_dirconf を取得します。 2. 設定された msg_entry のリストを順次走査します。 3. 現在の hookname がエントリの対象フックと一致するか確認します（allhooks または一致する名前のみ処理）。 4. entry->condition が定義されている場合、ap_expr_exec で評価します。評価失敗時はエラーログを出力し、成功かつ結果が偽の場合はスキップします。 5. ap_expr_str_exec を用いてメッセージ文字列を生成します。 6. 生成されたメッセージを ap_log_rerror を介して出力します。デバッグレベルが有効な場合は、フック名やソースコードの場所情報も付加されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_log_debug.c 内の各フック関数（log_debug_quick_handler, log_debug_translate_name, log_debug_fixups 等、計12箇所）。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_expr_exec
  • ap_expr_str_exec
  • ap_log_rerror
  • APLOGrdebug (マクロ)

関連データ構造

  • log_debug_dirconf: ディレクトリ単位の設定を保持する構造体。
  • msg_entry: 個別のログ設定（条件式、メッセージ式、対象フック）を保持する構造体。
  • request_rec: Apache のリクエストコンテキスト。

失敗モードと境界条件

  • **式評価エラー**: ap_expr の評価中にエラーが発生した場合、ap_log_rerror を使用してエラーログに詳細を出力し、そのエントリの処理をスキップします。
  • **設定未定義**: dconf->entries が NULL の場合、処理を行わずに終了します。

保守担当者向けメモ

  • 本関数はリクエスト処理の各フェーズで頻繁に呼び出されるため、複雑な式を多用するとパフォーマンスに影響を与える可能性があります。
  • APLOGrdebug が有効な場合、ログ出力にソースコードのファイル名と行番号が含まれるため、デバッグ時には有用ですが、本番環境でのログ量に注意してください。
