h2_conn_ctx_detach

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_conn_ctx.c:37

目的

指定された接続（conn_rec）から HTTP/2 関連のコンテキストを切り離し、設定をクリアします。

シグネチャまたは形状

void h2_conn_ctx_detach(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c - コンテキストを切り離す対象の接続構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 c が有効な conn_rec ポインタであること。
  • http2_module が Apache の設定システムに登録されていること。

アルゴリズムまたは振る舞い

1. ap_set_module_config を呼び出します。 2. 接続 c の conn_config 内にある http2_module に対応する設定値を NULL に設定します。これにより、当該接続と HTTP/2 コンテキストの紐付けが解除されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c:305
  • modules/http2/h2_session.c:880
  • modules/http2/h2_switch.c:205
  • **呼び出し先**:
  • ap_set_module_config (Apache HTTP Server API)

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • http2_module: mod_http2 モジュールの設定識別子。

失敗モードと境界条件

  • c が NULL の場合、ap_set_module_config の内部実装に依存します（通常はセグメンテーションフォールトの可能性があります）。

保守担当者向けメモ

  • 本関数は接続のクリーンアップ時に呼び出されます。コンテキストが保持していたメモリリソース（h2_conn_ctx_t 等）の解放は、この関数を呼び出す前後の処理で行われる必要があります。本関数自体は設定ポインタを NULL にするのみです。
