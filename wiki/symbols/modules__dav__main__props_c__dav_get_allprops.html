dav_get_allprops

定義

  • 種別: function
  • 定義箇所: modules/dav/main/props.c:595

目的

PROPFIND リクエストに応答するため、指定されたリソースに関連するすべてのプロパティ（ライブプロパティおよびデッドプロパティ）を収集し、XML 形式の propstat 要素として生成する。

シグネチャまたは形状

dav_get_props_result dav_get_allprops(dav_propdb *propdb, dav_prop_insert what);

入力と出力

  • **入力**:
  • propdb: プロパティデータベースのコンテキスト。
  • what: プロパティの挿入モード（値を含めるか、名前のみか、サポート対象のみか）。
  • **出力**:
  • dav_get_props_result: 生成された XML テキストの断片（propstats）と名前空間定義（xmlns）を保持する構造体。

前提条件と不変条件

  • propdb は有効なポインタであること。
  • propdb->deferred が真の場合、内部で dav_really_open_db が呼び出され、データベースが読み取り専用で開かれる。
  • what が DAV_PROP_INSERT_SUPPORTED 以外の場合、デッドプロパティの走査とライブプロパティの挿入が行われる。

アルゴリズムまたは振る舞い

1. **デッドプロパティの走査**: what が DAV_PROP_INSERT_SUPPORTED でない場合、データベース内の全プロパティを走査する。

  • 名前空間を定義し、各プロパティの値を XML 出力バッファに追加する。
  • DAV:getcontenttype および DAV:getcontentlanguage の存在を確認し、フラグを立てる。

    • 2. **ライブプロパティの挿入**: dav_run_insert_all_liveprops を呼び出し、登録されているすべてのライブプロパティプロバイダーにプロパティの挿入を依頼する。 3. **コアプロパティの挿入**: supportedlock および lockdiscovery を強制的に挿入する。 4. **不足プロパティの補完**: デッドプロパティとして見つからなかった場合、getcontenttype および getcontentlanguage をコアプロパティとして挿入する。 5. **終了処理**: what が DAV_PROP_INSERT_SUPPORTED でない場合、XML の閉じタグとステータスコード（200 OK）を付与する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/mod_dav.c: PROPFIND 処理に関連する箇所で呼び出されている。
  • **呼び出し先**:
  • dav_really_open_db, dav_xmlns_create, dav_xmlns_generate, dav_add_all_liveprop_xmlns, dav_run_insert_all_liveprops, dav_insert_coreprop, dav_output_prop_name
  • db_hooks（define_namespaces, first_name, next_name, output_value）

関連データ構造

  • dav_propdb: プロパティ操作のコンテキスト。
  • dav_get_props_result: 結果格納用構造体。
  • apr_text_header: XML 出力生成用バッファ。
  • dav_xmlns_info: XML 名前空間管理用構造体。

失敗モードと境界条件

  • データベースのオープン失敗時、エラーハンドリングが不完全である旨のコメントがソース内に存在する。
  • db_hooks->output_value がエラーを返した場合、そのキーの処理をスキップする。
  • dav_insert_coreprop の戻り値（エラー）は無視されている。

保守担当者向けメモ

  • PROPPATCH のエラーハンドリングと同様に、プロパティ取得時のエラー伝播が不完全である。
  • DAV_PROP_INSERT_SUPPORTED モード時は、XML の開始/終了タグやステータス行が付与されない点に注意すること。
