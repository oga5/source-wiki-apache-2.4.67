h2_post_config

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_http2.c:88

目的

Apache HTTP Server の起動時（post_config フック）に、mod_http2 モジュールの初期化と環境の検証を行う。

シグネチャまたは形状

static int h2_post_config(apr_pool_t *p, apr_pool_t *plog,
                          apr_pool_t *ptemp, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定用メモリプール。
  • plog: ログ用メモリプール（使用せず）。
  • ptemp: 一時メモリプール（使用せず）。
  • s: サーバーレコード。
  • **出力**:
  • int: 成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • Apache の post_config フックは設定確認のための「ドライラン」と「本番」の2回呼ばれる可能性がある。
  • apr_pool_userdata_get を使用して、初回呼び出し（ドライラン）を検出し、重複初期化を防止している。

アルゴリズムまたは振る舞い

1. **機能フラグの設定**: コンパイル時の定義に基づき、myfeats 構造体に nghttp2 のサポート機能（優先度制御、SHA256、無効ヘッダー処理、動的ウィンドウサイズ）を記録する。 2. **ドライラン判定**: mod_http2_init_counter キーを使用して、初回実行時はログを出力して即座に APR_SUCCESS を返す。 3. **初期化ログ**: 2回目以降の実行時、モジュールバージョン、有効な機能フラグ、および nghttp2 のバージョンをログ出力する。 4. **MPM 互換性チェック**: h2_mpm_supported() を呼び出し、現在の MPM が HTTP/2 に対応していない場合は警告ログを出力する。 5. **プロトコル初期化**: h2_protocol_init(p, s) および h2_switch_init(p, s) を順次呼び出し、プロトコルスタックと切り替えロジックを準備する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフックシステム（ap_hook_post_config により登録）。
  • **呼び出し先**:
  • h2_mpm_supported()
  • h2_conn_mpm_name()
  • h2_protocol_init()
  • h2_switch_init()
  • nghttp2_version()

関連データ構造

  • server_rec: サーバー設定情報。
  • nghttp2_info: nghttp2 ライブラリのバージョン情報。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • h2_protocol_init または h2_switch_init がエラーを返した場合、そのステータスコードをそのまま呼び出し元へ伝播させる。
  • MPM が非対応の場合でも、警告ログを出力するのみで処理は継続される。

保守担当者向けメモ

  • 本関数は Apache のフック仕様に従い、設定の二重適用を避けるための userdata 管理を行っている。
  • h2_c1（クライアント側）と h2_c2（バックエンド側）の初期化順序は、接続処理の安定性に直結するため、変更時は注意が必要である。
