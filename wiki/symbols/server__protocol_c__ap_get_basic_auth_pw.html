ap_get_basic_auth_pw

定義

  • 種別: function
  • 定義箇所: server/protocol.c:1758

目的

HTTP Basic 認証ヘッダーからユーザー名とパスワードを抽出し、デコードします。

シグネチャまたは形状

int ap_get_basic_auth_pw(request_rec *r, const char **pw)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体 (request_rec)。
  • pw: パスワード文字列を格納するためのポインタへのポインタ。
  • **出力**:
  • int: 処理結果を示すステータスコード（OK, DECLINED, HTTP_UNAUTHORIZED, HTTP_INTERNAL_SERVER_ERROR）。
  • *pw: 成功時、デコードされたパスワード文字列が設定されます。

前提条件と不変条件

  • r->headers_in に適切な認証ヘッダー（Authorization または Proxy-Authorization）が存在する必要があります。
  • 認証スキームが "Basic" である必要があります。

アルゴリズムまたは振る舞い

1. リクエストのプロキシ設定に応じて Authorization または Proxy-Authorization ヘッダーを取得します。 2. 認証タイプが "Basic" でない場合、DECLINED を返します。 3. AuthName が設定されていない場合、エラーログを出力し HTTP_INTERNAL_SERVER_ERROR を返します。 4. ヘッダーが存在しない場合、ap_note_auth_failure を呼び出し HTTP_UNAUTHORIZED を返します。 5. ヘッダー値のスキームが "Basic" でない場合、エラーログを出力し HTTP_UNAUTHORIZED を返します。 6. Base64 エンコードされた認証情報をデコードし、ユーザー名を r->user に格納します。 7. r->notes に AP_GET_BASIC_AUTH_PW_NOTE を設定し、r->ap_auth_type を "Basic" に更新します。 8. デコードされたパスワードを *pw に格納し、OK を返します。

  • **呼び出し先**:

関連データ構造

  • request_rec: リクエストの状態とヘッダー情報を保持します。

失敗モードと境界条件

  • AuthName 未設定: HTTP_INTERNAL_SERVER_ERROR を返します。
  • 認証ヘッダー欠落: HTTP_UNAUTHORIZED を返します。
  • スキーム不一致: HTTP_UNAUTHORIZED を返します。

保守担当者向けメモ

  • 本関数は古い API と見なされており、修正は推奨されません。
  • r->pool を使用してメモリを確保しているため、リクエスト終了時に自動的に解放されます。
