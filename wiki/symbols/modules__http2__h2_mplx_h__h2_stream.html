h2_stream

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_mplx.h:170

目的

HTTP/2 セッション内における個別のリクエストおよびレスポンスのライフサイクルと状態を管理するためのコンテキスト構造体です。

シグネチャまたは形状

ソースコード上では不完全型（opaque struct）として宣言されており、具体的なメンバ構成は modules/http2/h2_stream.h に定義されています。

入力と出力

  • 入力: h2_bucket_eos_make 関数等において、ストリームの状態を保持する引数として受け渡されます。
  • 出力: ストリームに関連付けられたデータや状態が、h2_bucket_beam を介してメイン接続（c1）とワーカースレッド（c2）間で転送されます。

前提条件と不変条件

  • h2_mplx によって管理される多重化対象のリソースです。
  • ワーカースレッド（c2）とメイン接続（c1）の間で共有されるため、アクセスには適切な同期が必要です。

アルゴリズムまたは振る舞い

  • h2_mplx を介してリクエストのスケジューリング、処理、およびクリーンアップが行われます。
  • ストリームの処理完了時には、h2_mplx_c1_stream_cleanup を経て破棄キューへ移動し、リソースが解放されます。

呼び出し元と呼び出し先

  • 呼び出し元: h2_bucket_eos.c や h2_c1.c など、ストリームの状態を操作する各モジュールから参照されます。
  • 呼び出し先: 本構造体自体はデータ構造であり、具体的な関数呼び出しの依存関係は h2_stream.h および h2_stream.c の実装に依存します。

関連データ構造

  • h2_mplx: ストリームを多重化・管理するハブ。
  • h2_bucket_beam: ストリーム内のデータをスレッド間で受け渡すための機構。
  • h2_session: ストリームを包含するセッション。

失敗モードと境界条件

  • 存在しないストリーム ID に対する操作は、h2_stream_get_fn 等の関数を通じてセッションレベルで制御されます。
  • ワーカースレッドとメイン接続間での不適切なアクセスは、デッドロックや競合状態を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体の定義は h2_mplx.h にありますが、詳細な実装は h2_stream.h および h2_stream.c を参照してください。
  • c1（メイン接続）と c2（ワーカースレッド）のコンテキストを跨ぐため、メンバへの直接アクセスは避け、提供されている API を介して操作してください。
