ap_unixd_set_global_mutex_perms

定義

  • 種別: function
  • 定義箇所: os/unix/unixd.c:281

目的

グローバルミューテックスのパーミッションを、サーバーの実行ユーザー設定（ap_unixd_config）に合わせて調整する。

シグネチャまたは形状

apr_status_t ap_unixd_set_global_mutex_perms(apr_global_mutex_t *gmutex)

入力と出力

  • **入力**: gmutex (調整対象の apr_global_mutex_t へのポインタ)
  • **出力**: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）

前提条件と不変条件

  • サーバーが root 権限で起動されていること。
  • ap_unixd_config が適切に初期化され、ターゲットとなるユーザー/グループ情報が保持されていること。

アルゴリズムまたは振る舞い

1. プラットフォームの APR_PROC_MUTEX_IS_GLOBAL 定義に基づき分岐する。 2. グローバルミューテックスとプロセスミューテックスが異なる場合（!APR_PROC_MUTEX_IS_GLOBAL）、apr_os_global_mutex_get を使用して内部のプロセスミューテックスハンドルを取得する。 3. 取得したハンドル（または同一の場合は gmutex そのもの）を引数として ap_unixd_set_proc_mutex_perms を呼び出し、パーミッション設定を適用する。 4. ap_unixd_set_proc_mutex_perms の戻り値をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_mutex.c:445
  • **呼び出し先**:
  • apr_os_global_mutex_get (APR API)
  • ap_unixd_set_proc_mutex_perms (同一ファイル内)

関連データ構造

  • apr_global_mutex_t: APR が提供するグローバルミューテックス構造体。
  • apr_os_global_mutex_t: OS 固有のグローバルミューテックス表現。

失敗モードと境界条件

  • apr_os_global_mutex_get または ap_unixd_set_proc_mutex_perms がエラーを返した場合、そのエラーコードが伝播される。
  • gmutex が NULL の場合の挙動は、呼び出し先の ap_unixd_set_proc_mutex_perms の実装に依存する。

保守担当者向けメモ

  • 本関数は unixd.h で宣言されており、MPM が権限降格を行う際の標準的なインターフェースとして機能する。
  • APR_PROC_MUTEX_IS_GLOBAL の定義により、プラットフォームごとに内部的なミューテックスの扱いが異なるため、変更時はこの条件分岐を維持すること。
