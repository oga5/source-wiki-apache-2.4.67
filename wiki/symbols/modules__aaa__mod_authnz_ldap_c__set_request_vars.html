set_request_vars

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_ldap.c:376

目的

LDAP 検索結果の属性値を Apache のリクエスト環境変数（subprocess_env）に展開し、必要に応じてリクエストのユーザー名（r->user）を設定する。

シグネチャまたは形状

int set_request_vars(request_rec *r, enum auth_ldap_phase phase, const char **vals)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • phase: 認証フェーズ（LDAP_AUTHN）または認可フェーズ（LDAP_AUTHZ）。
  • vals: LDAP 属性値の配列。
  • **出力**:
  • int: remote_user_attribute が設定された場合は 1、それ以外は 0 を返す。

前提条件と不変条件

  • r->per_dir_config から authnz_ldap_module の設定構造体（authn_ldap_config_t）が取得可能であること。
  • sec->attributes が定義されている場合、vals はそれに対応する値の配列であること。

アルゴリズムまたは振る舞い

1. phase に応じて環境変数のプレフィックス（AUTHN_PREFIX または sec->authz_prefix）を決定する。 2. sec->attributes に定義された各属性について、以下の処理を行う：

  • プレフィックスと属性名を結合し、すべて大文字に変換した文字列を環境変数名として生成する。
  • apr_table_setn を使用し、対応する vals の値を環境変数に格納する。

    • 3. 認証フェーズ（LDAP_AUTHN）かつ sec->remote_user_attribute が現在の属性と一致する場合：

  • r->user に当該属性値をコピーして設定し、戻り値として 1 をマークする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_ldap.c 内の複数箇所（認証フェーズ: 622行目、認可フェーズ: 753, 775, 967, 1006, 1133, 1267, 1422, 1508行目）。
  • **呼び出し先**:
  • ap_get_module_config
  • apr_pstrcat
  • apr_toupper
  • apr_table_setn
  • apr_pstrdup

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • authn_ldap_config_t: モジュールのディレクトリ設定（属性リストやプレフィックス設定を保持）。
  • auth_ldap_phase: 認証/認可フェーズを識別する列挙型。

失敗モードと境界条件

  • sec->attributes または vals が NULL の場合、環境変数の設定はスキップされる。
  • vals[i] が NULL の場合、空文字列 "" が環境変数に設定される。

保守担当者向けメモ

  • 本関数は r->pool を使用してメモリを割り当てるため、リクエスト終了時に自動的に解放される。
  • 環境変数名の大文字変換は、プレフィックス部分を含めて行われる点に注意すること。
