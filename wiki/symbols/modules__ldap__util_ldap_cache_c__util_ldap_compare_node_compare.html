util_ldap_compare_node_compare

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache.c:246

目的

LDAP 比較キャッシュにおいて、2つの util_compare_node_t ノードが同一の比較条件（DN、属性、値）を持つかどうかを判定する。

シグネチャまたは形状

int util_ldap_compare_node_compare(void *a, void *b)

入力と出力

  • **入力**:
  • a, b: 比較対象となる util_compare_node_t 構造体へのポインタ（void * として渡される）。
  • **出力**:
  • 1 (真): 3つのフィールド（dn, attrib, value）がすべて一致する場合。
  • 0 (偽): いずれかのフィールドが一致しない場合。

前提条件と不変条件

  • 入力ポインタ a および b は、util_compare_node_t 型としてキャスト可能であること。
  • 各ノード内の dn, attrib, value は、strcmp による比較が可能な有効な文字列ポインタであること。

アルゴリズムまたは振る舞い

1. 引数 a と b を util_compare_node_t * 型にキャストする。 2. strcmp を用いて以下の3項目を順次比較する。

  • dn (識別名)
  • attrib (属性名)
  • value (属性値)

    • 3. すべての比較結果が 0（一致）である場合にのみ 1 を返し、それ以外は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap_cache_mgr.c:301 にて、キャッシュ管理用の関数ポインタテーブルの一部として参照されている。
  • **呼び出し先**:
  • 標準 C ライブラリの strcmp を呼び出す。

関連データ構造

  • util_compare_node_t: 本関数が比較対象とするキャッシュノード構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、strcmp 呼び出しによりセグメンテーション違反が発生する可能性がある。
  • 文字列フィールドが NULL である場合も同様に未定義動作となる。

保守担当者向けメモ

  • 本関数はキャッシュの等価性判定に使用されるため、比較ロジックを変更する場合は util_ldap_compare_node_hash と整合性が取れていることを確認すること。
  • 共有メモリ上で動作するキャッシュノードを扱うため、メモリレイアウトやポインタの扱いに注意が必要である。
