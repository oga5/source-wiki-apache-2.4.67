rewrite_server_conf

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:361

目的

Apache HTTP Server における mod_rewrite のサーバー単位の設定情報を保持するための構造体です。エンジン全体の有効状態、ルールセット、および外部マップ定義を管理します。

シグネチャまたは形状

typedef struct {
    int           state;              /* RewriteEngineの状態 */
    int           options;            /* RewriteOptionの状態 */
    apr_hash_t         *rewritemaps;  /* RewriteMapエントリ */
    apr_array_header_t *rewriteconds; /* RewriteCondエントリ（一時的） */
    apr_array_header_t *rewriterules; /* RewriteRuleエントリ */
    server_rec   *server;             /* 対応するサーバー指標 */
    unsigned int state_set:1;         /* state設定済みフラグ */
    unsigned int options_set:1;       /* options設定済みフラグ */
} rewrite_server_conf;

入力と出力

  • 本構造体は、設定読み込み時に apr_pcalloc を介してメモリ確保されます。
  • サーバー設定の初期化およびマージ処理において、server_rec や他の設定値が入力として利用されます。

前提条件と不変条件

  • rewritemaps、rewriteconds、rewriterules は apr_pool_t を介して管理されるメモリ領域に格納されます。
  • state_set および options_set ビットフィールドは、設定が明示的に適用されたかどうかを追跡するために使用されます。

アルゴリズムまたは振る舞い

  • サーバー全体の RewriteEngine のオン/オフ状態を保持し、リクエスト処理時にルールを適用すべきかを判断する基準となります。
  • rewriterules 配列に格納されたルールは、リクエスト処理フェーズにおいて順次評価されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_rewrite.c 内のサーバー設定作成関数（create_rewrite_server_config 等）および設定マージ関数から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。リクエスト処理中に rewrite_ctx 等のコンテキストを通じて、ルール適用ロジックから参照されます。

関連データ構造

  • rewritemap_entry: rewritemaps ハッシュ内に格納される外部マップ定義。
  • rewriterule_entry: rewriterules 配列内に格納される解析済みルール。
  • rewritecond_entry: rewriteconds 配列内に格納される条件式。
  • server_rec: Apache HTTP Server のサーバー設定構造体。

失敗モードと境界条件

  • メモリ不足時、apr_pcalloc による割り当てが失敗する可能性があります。
  • 設定の不整合（例: state_set が未設定のままルールが評価される等）が発生した場合の挙動は、モジュール内のルール適用ロジックに依存します。

保守担当者向けメモ

  • 本構造体はサーバーコンテキスト用です。ディレクトリ単位の設定には rewrite_perdir_conf が別途存在するため、混同しないよう注意してください。
  • rewriteconds は一時的な保持用として定義されています。ルール適用時のライフサイクル管理に留意してください。
