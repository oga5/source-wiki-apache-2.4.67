seclistenup_rec

定義

  • 種別: struct
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:105

目的

RFC 2817 に基づく TLS アップグレードが可能なリスナー情報を保持するための構造体です。

シグネチャまたは形状

struct seclistenup_rec {
    seclistenup_rec *next;
    /* その他、アップグレード対象ポートや設定情報を含むフィールド */
};

※ソース excerpt には構造体の前方宣言のみが含まれており、詳細なメンバ構成は mod_nw_ssl.c の定義行（119行目以降）に依存します。

入力と出力

  • 入力: apr_pcalloc を介してメモリが割り当てられ、設定値が格納されます。
  • 出力: リンクリストのノードとして ap_seclistenersup に保持されます。

前提条件と不変条件

  • ap_seclistenersup を起点とするリンクリストの一部として管理されます。
  • メモリは Apache の pool に紐付けられており、明示的な解放は不要です。

アルゴリズムまたは振る舞い

  • mod_nw_ssl.c 内で、アップグレード可能なリスナーを管理するリンクリストの要素として機能します。
  • SecureListen 等の設定ディレクティブにより、新しいインスタンスが生成・追加されます。

呼び出し元と呼び出し先

  • 呼び出し元: mod_nw_ssl.c 内のリスナー設定処理（apr_pcalloc による初期化など）。
  • 呼び出し先: 情報なし。

関連データ構造

  • seclisten_rec: セキュアリスナーの定義を保持する類似の構造体。
  • ap_seclistenersup: seclistenup_rec 型のリンクリストのヘッドポインタ。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc が失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが異常終了する可能性があります。

保守担当者向けメモ

  • 本構造体は NetWare 固有の SSL アップグレード機能に特化しています。
  • メンバ変数の詳細については、mod_nw_ssl.c の119行目以降の定義を参照してください。
