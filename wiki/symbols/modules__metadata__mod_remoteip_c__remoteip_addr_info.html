remoteip_addr_info

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_remoteip.c:45

目的

プロキシの IP アドレス情報と、その情報源であるサーバー設定を連結リスト形式で保持するために使用されます。

シグネチャまたは形状

typedef struct remoteip_addr_info {
    struct remoteip_addr_info *next;
    apr_sockaddr_t *addr;
    server_rec *source;
} remoteip_addr_info;

入力と出力

  • 入力: apr_palloc を用いてメモリ確保され、addr（IP アドレス情報）および source（サーバー設定）が格納されます。
  • 出力: リストのノードとして、後続の remoteip_addr_info 構造体へのポインタを返します。

前提条件と不変条件

  • next ポインタは、リストの終端で NULL である必要があります。
  • addr は apr_sockaddr_t 型であり、有効な IP アドレス情報を含んでいる必要があります。

アルゴリズムまたは振る舞い

  • 連結リストのノードとして機能し、remoteip_addr_in_list 関数によって特定の IP アドレスがリスト内に存在するかどうかの照合に使用されます。
  • remoteip_warn_enable_conflict 関数において、設定の競合を警告するために参照されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照）:
  • remoteip_addr_in_list (引数として使用)
  • remoteip_warn_enable_conflict (引数として使用)
  • remoteip_config_t 構造体内の proxy_protocol_enabled および proxy_protocol_disabled フィールド
  • 呼び出し先:
  • apr_palloc (メモリ確保時)
  • apr_sockaddr_t (データ保持)
  • server_rec (データ保持)

関連データ構造

  • remoteip_config_t: 本構造体をリストとして保持するメインの設定構造体。
  • apr_sockaddr_t: IP アドレスのネットワーク情報を保持する APR の構造体。
  • server_rec: サーバー設定情報を保持する Apache コアの構造体。

失敗モードと境界条件

  • メモリ不足時: apr_palloc が失敗した場合、プロセスが終了する可能性があります。
  • リストの走査: リストが空（NULL）の場合、remoteip_addr_in_list は適切に処理を行う必要があります。

保守担当者向けメモ

  • 本構造体は mod_remoteip における信頼できるプロキシリストの管理に不可欠です。リストの操作を行う際は、メモリリークを防ぐため conf->pool のライフサイクルを意識してください。
