h2_proxy_ngheader

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.h:164

目的

Apache HTTP Server の内部ヘッダー形式（apr_table_t）を、nghttp2 ライブラリが要求する形式（nghttp2_nv 配列）へ変換・保持するためのラッパー構造体です。

シグネチャまたは形状

typedef struct h2_proxy_ngheader {
    nghttp2_nv *nv;    /* nghttp2用ヘッダー配列へのポインタ */
    apr_size_t nvlen;  /* 配列の要素数 */
} h2_proxy_ngheader;

入力と出力

  • **入力**: h2_proxy_util_nghd_make 等の関数において、apr_pool_t からメモリを割り当てられ、apr_table_t 形式のヘッダー情報から変換されたデータが格納されます。
  • **出力**: 変換済みの nghttp2_nv 配列およびその長さが、HTTP/2 プロキシ転送処理のために提供されます。

前提条件と不変条件

  • nv は apr_pcalloc 等で確保されたメモリ領域を指す必要があります。
  • nvlen は nv が指す配列の有効な要素数と一致している必要があります。

アルゴリズムまたは振る舞い

1. h2_proxy_util_nghd_make または h2_proxy_util_nghd_make_req が呼び出されると、メモリプールから本構造体と必要な nghttp2_nv 配列の領域が確保されます。 2. Apache のヘッダーテーブルを走査し、nghttp2 形式へ変換して nv に格納します。 3. 変換後のヘッダー数（nvlen）が構造体に記録されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c が本構造体を利用してヘッダーを管理しています。
  • **呼び出し先**: modules/http2/h2_proxy_util.c 内の h2_proxy_util_nghd_make および h2_proxy_util_nghd_make_req が本構造体の生成と初期化を行います。

関連データ構造

  • nghttp2_nv: nghttp2 ライブラリが定義するヘッダー名と値のペア構造体。
  • apr_table_t: Apache HTTP Server が提供するヘッダー管理用のテーブル構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc が失敗した場合、呼び出し元で適切なエラーハンドリングが必要です。
  • ヘッダー変換時: 不正なヘッダー名や値が含まれる場合、nghttp2 側の仕様に従い変換処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_proxy_http2 におけるヘッダー変換の中間表現です。
  • h2_proxy_util.c での変換ロジックを変更する際は、nvlen が正確に更新されていることを確認してください。
