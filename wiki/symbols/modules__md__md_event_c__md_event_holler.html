md_event_holler

定義

  • 種別: function
  • 定義箇所: modules/md/md_event.c:78

目的

登録されたすべてのイベントリスナー（コールバック関数）を順次実行します。各リスナーの戻り値に関わらず、リストの最後まで処理を継続することが目的です。

シグネチャまたは形状

void md_event_holler(const char *event, 
                     const char *mdomain, 
                     struct md_job_t *job, 
                     struct md_result_t *result, 
                     apr_pool_t *p)

入力と出力

  • **入力**:
  • event: イベント名を示す文字列。
  • mdomain: 対象となるドメイン名。
  • job: 関連するジョブ情報（md_job_t）。
  • result: 処理結果やエラー詳細（md_result_t）。
  • p: メモリ割り当てに使用する APR プール。
  • **出力**: なし（void）。

前提条件と不変条件

  • EVNT グローバル構造体が初期化されている必要があります。
  • 本関数はスレッドセーフではありません。マルチスレッド環境下での呼び出しには、呼び出し元での排他制御が必要です。
  • 購読解除機能は存在しないため、登録されたコールバックはプロセスの生存期間中有効です。

アルゴリズムまたは振る舞い

1. 静的変数 EVNT が保持する購読リストの先頭（EVNT.subs）を取得します。 2. リストが空になるまで、各ノードのコールバック関数 sub->cb を順次呼び出します。 3. コールバックには、引数として受け取った event, mdomain, job, result, p および各購読時に設定された baton を渡します。 4. いずれかのコールバックがエラーを返した場合でも、中断することなく次のリスナーを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_ocsp.c: OCSP エラーおよび更新イベントの通知。
  • modules/md/md_reg.c: 証明書インストールイベントの通知。
  • modules/md/md_status.c: ステータス更新イベントの通知。
  • modules/md/mod_md_drive.c: ジョブエラーイベントの通知。
  • **呼び出し先**:
  • 登録された各 md_subscription 内のコールバック関数（sub->cb）。

関連データ構造

  • md_subscription: イベント購読情報を保持するリスト構造体。
  • md_job_t: ジョブの状態を保持する構造体。
  • md_result_t: 処理結果を保持する構造体。
  • EVNT: モジュール内で静的に保持されるイベント管理用グローバル構造体。

失敗モードと境界条件

  • 購読リストが空の場合、何も実行せずに正常終了します。
  • コールバック関数内で発生したエラーは本関数では捕捉されず、通知の継続性に影響を与えません。

保守担当者向けメモ

  • md_event_raise と異なり、エラー発生時も処理を中断しないため、通知の確実性が求められる処理に適しています。
  • 購読リストの走査中にコールバックがリストを変更するような操作を行うと、予期せぬ動作を引き起こす可能性があるため注意してください。
