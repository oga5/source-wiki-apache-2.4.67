next_run_default

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_drive.c:245

目的

マネージドドメイン（MD）の更新処理における、次回の実行タイミングをランダムなジッター（ゆらぎ）を含めて算出する。

シグネチャまたは形状

apr_time_t next_run_default(md_renew_ctx_t *dctx)

入力と出力

  • **入力**: md_renew_ctx_t *dctx（更新プロセスのコンテキスト構造体。設定されたチェック間隔 mc->check_interval を参照するために使用）。
  • **出力**: apr_time_t（現在時刻にチェック間隔とランダムなジッターを加算した、次回実行予定時刻）。

前提条件と不変条件

  • dctx が有効なポインタであること。
  • dctx->mc->check_interval が正しく初期化されていること。
  • md_rand_bytes が乱数生成のために利用可能であること。

アルゴリズムまたは振る舞い

1. dctx から設定されたチェック間隔（delay）を取得する。 2. md_rand_bytes を使用して 1 バイトの乱数 c を生成する。 3. c を 0〜255 の範囲で正規化し、delay の -50% から +50% の範囲でジッターを計算する。 4. 現在時刻 + delay + ジッター を計算し、次回実行時刻として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_drive.c:279
  • modules/md/mod_md_ocsp.c:182（注: ソースコード上の記述と実際の引数整合性については、mod_md_ocsp.c 側の定義を確認する必要がある）
  • **呼び出し先**:
  • md_rand_bytes（乱数生成）
  • apr_time_now（現在時刻取得）

関連データ構造

  • md_renew_ctx_t: 更新プロセスの状態と設定を保持する構造体。

失敗モードと境界条件

  • md_rand_bytes が失敗した場合の挙動は本関数内では定義されていない。
  • dctx が NULL の場合、デリファレンスによるクラッシュの可能性がある。

保守担当者向けメモ

  • 本関数は、複数のプロセスやサーバーが同時に更新処理を開始して負荷が集中する「サンダリング・ハード」問題を回避するためにジッターを導入している。
  • mod_md_ocsp.c からの呼び出しについては、引数なしで呼び出されているように見える箇所があるため、シグネチャの整合性を確認すること。
