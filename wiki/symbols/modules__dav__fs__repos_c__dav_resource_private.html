dav_resource_private

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/repos.c:46

目的

mod_dav_fs において、WebDAV リソースに関連付けられたファイルシステム固有のプライベート情報を保持するための構造体です。

シグネチャまたは形状

struct dav_resource_private {
    apr_pool_t *pool;        /* リクエストに関連付けられたメモリプール */
    const char *pathname;    /* リソースのフルパス */
    apr_finfo_t finfo;       /* ファイルシステム情報 */
    request_rec *r;          /* Apache リクエストレコード */
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。dav_resource 構造体の info メンバを通じてアクセスされ、リソース操作に必要なコンテキスト情報を提供します。

前提条件と不変条件

  • pool は、リクエストの生存期間と同期した apr_pool_t である必要があります。
  • r は、有効な Apache request_rec を指している必要があります。
  • pathname は、ファイルシステム上の絶対パスまたは相対パスを保持します。

アルゴリズムまたは振る舞い

本構造体自体は状態を保持するコンテナです。mod_dav_fs の各関数（dav_fs_get_resource 等）において、リソースの初期化時に生成され、以降のファイル操作、プロパティ管理、ディレクトリ走査の各処理で参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/repos.c 内の多数の関数（例: dav_fs_get_resource 相当の処理、dav_fs_copymove_file 等）が、resource->info をキャストして本構造体を取得しています。
  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な呼び出し先はありません。保持している r や pool を通じて、Apache HTTP Server や APR の API が呼び出されます。

関連データ構造

  • dav_resource: 本構造体を info メンバとして保持する上位構造体。
  • request_rec: Apache のリクエスト情報を保持する構造体。
  • apr_finfo_t: ファイルの属性情報（サイズ、権限、時刻等）を保持する APR 構造体。

失敗モードと境界条件

  • resource->info が NULL の状態でアクセスした場合、セグメンテーション違反が発生します。
  • pool が破棄された後に本構造体を参照すると、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は mod_dav_fs の内部実装詳細であり、repos.c 内でのみ使用されることを想定しています。
  • pathname の文字列は pool から割り当てられているため、明示的な解放は不要ですが、リクエスト終了時のプール破棄に伴い無効化される点に注意してください。
