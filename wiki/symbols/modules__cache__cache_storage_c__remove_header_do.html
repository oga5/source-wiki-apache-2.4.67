remove_header_do

定義

  • 種別: function
  • 定義箇所: modules/cache/cache_storage.c:130

目的

HTTPレスポンスヘッダーのテーブルから、特定のヘッダー（Warning以外）を削除するために使用されるコールバック関数です。

シグネチャまたは形状

static int remove_header_do(void *v, const char *key, const char *val)

入力と出力

  • **入力**:
  • v: 操作対象となる apr_table_t へのポインタ。
  • key: 削除対象のヘッダー名。
  • val: ヘッダーの値（本関数内では未使用）。
  • **出力**:
  • 常に 1 を返し、apr_table_do による反復処理を継続させます。

前提条件と不変条件

  • v は有効な apr_table_t である必要があります。
  • Warning ヘッダーについては、warn-code 2xx を保持する必要があるため、削除対象から除外されます。

アルゴリズムまたは振る舞い

1. key が Warning（大文字小文字を区別しない）であるかを確認します。 2. Warning ヘッダーの場合は何もしません。 3. それ以外のヘッダーの場合は、apr_table_unset を呼び出し、テーブルから該当するヘッダーを削除します。 4. 処理継続のため 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/cache_storage.c:173 において apr_table_do の引数として呼び出されています。
  • **呼び出し先**: apr_table_unset (APRライブラリ関数)。

関連データ構造

  • apr_table_t: APRのテーブル構造体。ヘッダーの保持に使用されます。

失敗モードと境界条件

  • key が NULL の場合の挙動は、apr_table_do の仕様に依存します。
  • Warning ヘッダー以外のすべてのヘッダーが削除対象となるため、呼び出し側でテーブルの内容を完全にクリアしたい場合に適しています。

保守担当者向けメモ

  • 本関数は apr_table_do からのコールバックとして設計されています。
  • Warning ヘッダーの保持ロジックは、HTTP仕様（RFC）に基づく要件であるため、変更の際は関連するキャッシュ仕様を確認してください。
