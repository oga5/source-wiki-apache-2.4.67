spot_cookie

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_usertrack.c:222

目的

HTTP リクエストから追跡用 Cookie を検索し、存在しない場合に新規 Cookie を生成してクライアントに付与する。

シグネチャまたは形状

static int spot_cookie(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (Apache フックの戻り値: OK, DECLINED, または HTTP_INTERNAL_SERVER_ERROR)

前提条件と不変条件

  • dcfg->enabled が真であること。
  • サブリクエスト（r->main）ではないこと。
  • NUM_SUBS (3) は、正規表現内のキャプチャグループ数と一致していること。

アルゴリズムまたは振る舞い

1. 設定構造体 dcfg を取得し、無効な場合やサブリクエストの場合は DECLINED を返す。 2. r->headers_in から "Cookie" ヘッダーを取得する。 3. 取得したヘッダーに対し、dcfg->regexp を用いて正規表現マッチングを行う。 4. マッチした場合:

  • キャプチャグループ $1 または $2 から Cookie 値を抽出する。
  • 抽出した値を r->notes に "cookie" というキーで格納し、DECLINED を返す。

    • 5. マッチしない場合:

  • make_cookie(r) を呼び出し、新規 Cookie を生成・付与して OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_fixups により fixups フックとして登録されている (modules/metadata/mod_usertrack.c:493)。
  • **呼び出し先**: make_cookie (新規 Cookie 生成時)、ap_pregsub (値抽出時)、ap_log_rerror (エラー時)。

関連データ構造

  • cookie_dir_rec: ディレクトリごとの設定（正規表現や有効状態を保持）。
  • request_rec: リクエストのコンテキストおよびヘッダー情報。

失敗モードと境界条件

  • **メモリ不足**: ap_pregsub が失敗した場合、HTTP_INTERNAL_SERVER_ERROR を返し、エラーログを出力する。
  • **サブリクエスト**: r->main が存在する場合、処理を行わず DECLINED を返す。
  • **Cookie 未設定**: ヘッダーが存在しない、または正規表現にマッチしない場合は make_cookie へ遷移する。

保守担当者向けメモ

  • 本関数は APR_HOOK_REALLY_FIRST で登録されており、リクエスト処理の初期段階で実行される。
  • 正規表現のキャプチャグループ（$1, $2）の扱いは、set_cookie_name での整合性チェックに依存している。
  • 抽出した Cookie 値は r->pool に割り当てられるため、リクエスト終了時に自動的に解放される。
