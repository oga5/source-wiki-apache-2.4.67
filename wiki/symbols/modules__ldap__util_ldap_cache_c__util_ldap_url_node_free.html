util_ldap_url_node_free

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache.c:68

目的

LDAP URL キャッシュノードが保持する動的メモリおよびネストされたキャッシュ構造を解放し、ノード自体を破棄する。

シグネチャまたは形状

void util_ldap_url_node_free(util_ald_cache_t *cache, void *n)

入力と出力

  • cache: キャッシュ管理構造体へのポインタ。メモリ解放に使用される。
  • n: 解放対象の util_url_node_t 構造体へのポインタ（void * として渡される）。
  • 戻り値: なし。

前提条件と不変条件

  • n は util_url_node_t 型の有効なポインタであること。
  • cache は util_ald_alloc 等でメモリを管理している有効なキャッシュインスタンスであること。
  • 本関数は共有メモリ（apr_rmm）上の領域を操作するため、標準の free ではなく util_ald_free を使用する必要がある。

アルゴリズムまたは振る舞い

1. 引数 n を util_url_node_t 型にキャストする。 2. node->url を util_ald_free で解放する。 3. node->search_cache、node->compare_cache、node->dn_compare_cache の各キャッシュを util_ald_destroy_cache で再帰的に破棄する。 4. 最後に node 自体を util_ald_free で解放する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ldap/util_ldap_cache.c:461（キャッシュパージ処理等から参照されている）。
  • 呼び出し先: util_ald_free および util_ald_destroy_cache。

関連データ構造

  • util_url_node_t: 本関数が解放対象とするノード構造体。
  • util_ald_cache_t: キャッシュ管理構造体。

失敗モードと境界条件

  • n が NULL の場合、util_ald_free 内部の挙動に依存する（通常はセグメンテーション違反の可能性があるため、呼び出し側でガードが必要）。
  • 共有メモリが破損している場合、util_ald_free または util_ald_destroy_cache の実行時にメモリ破壊が発生する。

保守担当者向けメモ

  • 本関数は util_url_node_t が保持するサブキャッシュを再帰的に破棄するため、ノード構造体に変更を加えた場合は、必ず本関数内で対応する util_ald_destroy_cache を呼び出すこと。
  • 共有メモリ環境下では、ポインタの絶対アドレスではなくオフセット管理が前提となるため、util_ald_ 系関数以外のメモリ解放関数は絶対に使用しないこと。
