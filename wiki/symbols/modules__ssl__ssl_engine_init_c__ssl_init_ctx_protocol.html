ssl_init_ctx_protocol

定義

  • **種別**: function
  • **定義箇所**: modules/ssl/ssl_engine_init.c:718

目的

Apache HTTP Server の起動・再起動時に、指定された設定に基づき OpenSSL の SSL_CTX を生成し、サポートするプロトコルバージョンや暗号化オプションを初期化する。

シグネチャまたは形状

apr_status_t ssl_init_ctx_protocol(server_rec *s,
                                   apr_pool_t *p,
                                   apr_pool_t *ptemp,
                                   modssl_ctx_t *mctx)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p: メモリプール。
  • ptemp: 一時メモリプール。
  • mctx: SSL コンテキスト設定情報（modssl_ctx_t）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は ssl_die() を経由してエラー終了。

前提条件と不変条件

  • mctx->protocol が SSL_PROTOCOL_NONE であってはならない。
  • OpenSSL のバージョン（OPENSSL_VERSION_NUMBER）に応じて、利用可能なメソッドやプロトコル制御 API が条件分岐される。

アルゴリズムまたは振る舞い

1. **プロトコル検証**: 設定されたプロトコルが有効か確認し、無効な場合は ssl_die() を呼び出す。 2. **メソッド選択**: OpenSSL バージョンに応じ、クライアント（プロキシ）またはサーバー用の SSL_METHOD を決定する。 3. **コンテキスト生成**: SSL_CTX_new() でコンテキストを作成し、mctx->ssl_ctx に格納する。 4. **プロトコル制限**:

  • OpenSSL 1.1.0 未満: SSL_CTX_set_options を使用して、許可されないプロトコルを明示的に無効化する。
  • OpenSSL 1.1.0 以上: SSL_CTX_set_min_proto_version および SSL_CTX_set_max_proto_version を使用して範囲を指定する。

    • 5. **オプション設定**: 暗号スイートの優先順位、圧縮の無効化、セッションチケット、再ネゴシエーション設定などを SSL_CTX_set_options で適用する。 6. **環境設定**: SSL_CTX_set_app_data でサーバーレコードを紐付け、メモリ解放モードやキーログコールバックを設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:1382
  • **呼び出し先**:
  • OpenSSL API: SSL_CTX_new, SSL_CTX_set_options, SSL_CTX_set_max_proto_version, SSL_CTX_set_min_proto_version, SSL_CTX_set_app_data, SSL_CTX_set_mode, SSL_CTX_set_keylog_callback 等。
  • Apache API: ap_log_error, ssl_die, apr_pstrcat。

関連データ構造

  • SSL_CTX: OpenSSL の SSL コンテキスト構造体。
  • modssl_ctx_t: mod_ssl 内部の SSL 設定管理構造体。
  • SSLSrvConfigRec: サーバーごとの SSL 設定レコード。

失敗モードと境界条件

  • **プロトコル未指定**: SSL_PROTOCOL_NONE の場合、ssl_die() によりプロセスが終了する。
  • **コンテキスト生成失敗**: SSL_CTX_new() が NULL を返した場合、ssl_log_ssl_error でエラーを記録し ssl_die() で終了する。
  • **バージョン不整合**: OpenSSL のバージョンにより利用可能なプロトコル（TLS 1.3 等）が制限される場合がある。

保守担当者向けメモ

  • 本関数は OpenSSL のバージョン差異を吸収する重要な箇所である。OPENSSL_VERSION_NUMBER による条件分岐が多用されているため、OpenSSL のアップデート時にはプロトコル設定ロジックの整合性を確認すること。
  • mctx->pkp フラグにより、サーバーモードとプロキシモードの挙動が分岐する点に注意。
