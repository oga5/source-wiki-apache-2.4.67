h2_config_var_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_config.h:26

目的

mod_http2 モジュールにおける設定項目を識別するための列挙型です。サーバー全体およびディレクトリ単位の設定値にアクセスする際のキーとして使用されます。

シグネチャまたは形状

typedef enum {
    H2_CONF_MAX_STREAMS,
    H2_CONF_WIN_SIZE,
    H2_CONF_MIN_WORKERS,
    H2_CONF_MAX_WORKERS,
    H2_CONF_MAX_WORKER_IDLE_LIMIT,
    H2_CONF_STREAM_MAX_MEM,
    H2_CONF_DIRECT,
    H2_CONF_MODERN_TLS_ONLY,
    H2_CONF_UPGRADE,
    H2_CONF_TLS_WARMUP_SIZE,
    H2_CONF_TLS_COOLDOWN_SECS,
    H2_CONF_PUSH,
    H2_CONF_PUSH_DIARY_SIZE,
    H2_CONF_COPY_FILES,
    H2_CONF_EARLY_HINTS,
    H2_CONF_PADDING_BITS,
    H2_CONF_PADDING_ALWAYS,
    H2_CONF_OUTPUT_BUFFER,
    H2_CONF_STREAM_TIMEOUT,
    H2_CONF_MAX_DATA_FRAME_LEN,
    H2_CONF_PROXY_REQUESTS,
    H2_CONF_WEBSOCKETS,
    H2_CONF_MAX_HEADER_BLOCK_LEN,
    H2_CONF_MAX_STREAM_ERRORS,
} h2_config_var_t;

入力と出力

本シンボル自体は列挙値の定義であり、入力や出力は持ちません。これを利用するアクセサ関数（h2_config_geti 等）が、この列挙値を引数として受け取り、対応する設定値を返します。

前提条件と不変条件

  • h2_config_var_t に新しい項目を追加した場合は、h2_config.c 内のアクセサ関数（h2_config_geti, h2_config_geti64 等）にも対応するロジックを追加する必要があります。

アルゴリズムまたは振る舞い

本シンボルは、設定値の取得・設定を行うための識別子として機能します。実行時には、h2_config_cgeti（接続単位）や h2_config_rgeti（リクエスト単位）などの関数を通じて、現在のコンテキストに適用される設定値の解決に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_config.c 内の以下の関数群がこの列挙型を引数として利用しています。
  • h2_srv_config_geti64 / h2_srv_config_seti / h2_srv_config_seti64
  • h2_dir_config_geti64 / h2_config_seti / h2_config_seti64
  • h2_config_cgeti / h2_config_cgeti64
  • h2_config_sgeti / h2_config_sgeti64
  • h2_config_geti / h2_config_geti64
  • **呼び出し先**: 本シンボルは列挙型であるため、呼び出し先はありません。

関連データ構造

  • h2_config: サーバーレベルの設定を保持する構造体。
  • h2_dir_config: ディレクトリ単位の設定を保持する構造体。

失敗モードと境界条件

  • 未定義の列挙値がアクセサ関数に渡された場合の挙動は、各アクセサ関数の実装に依存します。

保守担当者向けメモ

  • 設定値のデフォルト値や具体的な取得ロジックは h2_config.c に記述されています。
  • 設定値を取得する際は、直接構造体にアクセスせず、必ず rgeti や cgeti などの提供されたアクセサ関数を使用してください。これにより、サーバー全体の設定とディレクトリ単位の上書きが正しくマージされます。
