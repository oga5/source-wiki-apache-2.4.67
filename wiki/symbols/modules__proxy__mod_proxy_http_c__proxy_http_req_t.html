proxy_http_req_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_http.c:294

目的

HTTP プロキシ処理におけるリクエスト単位の状態管理を行うための中心的な構造体です。バックエンドへの接続情報、転送用バッファ、およびリクエストの処理状態を保持します。

シグネチャまたは形状

typedef struct {
    apr_pool_t *p;
    request_rec *r;
    const char *proto;
    proxy_worker *worker;
    proxy_server_conf *sconf;
    char server_portstr[32];
    proxy_conn_rec *backend;
    conn_rec *origin;
    apr_bucket_alloc_t *bucket_alloc;
    apr_bucket_brigade *header_brigade;
    apr_bucket_brigade *input_brigade;
    char *old_cl_val, *old_te_val;
    apr_off_t cl_val;
    rb_methods rb_method;
    const char *upgrade;
    unsigned int do_100_continue        :1,
                 prefetch_nonblocking   :1,
                 force10                :1;
} proxy_http_req_t;

入力と出力

本構造体は、proxy_http_handler 等の関数において、リクエスト処理のコンテキストとして引数で受け渡されます。直接的な入出力関数を持つわけではなく、各処理関数がこの構造体のメンバを読み書きすることで状態を共有します。

前提条件と不変条件

  • r->pool を使用してメモリ管理が行われます。
  • header_brigade および input_brigade は、バケット操作のライフサイクルに合わせて適切にクリーンアップされる必要があります。

アルゴリズムまたは振る舞い

リクエスト処理の各フェーズ（プリフェッチ、ヘッダー送信、ボディ転送、レスポンス処理）において、この構造体を通じて以下の制御が行われます。

  • rb_method: リクエストボディの転送方式（ストリーム、チャンク、スプール）を決定します。
  • do_100_continue 等のビットフィールド: HTTP プロトコル固有の挙動を制御します。

呼び出し元と呼び出し先

以下の関数で引数として参照されています。

  • stream_reqbody
  • terminate_headers
  • ap_proxy_http_prefetch
  • ap_proxy_http_request
  • send_continue_body
  • ap_proxy_http_process_response

関連データ構造

  • proxy_worker: バックエンドワーカー設定。
  • proxy_conn_rec: バックエンドへの接続情報。
  • apr_bucket_brigade: 送受信データのバッファリングに使用。
  • rb_methods: リクエストボディ転送方式の列挙型。

失敗モードと境界条件

  • apr_brigade_cleanup の呼び出し漏れはメモリリークを引き起こします。
  • バックエンドとの接続切断時には、backend->close フラグ等の制御と併せて本構造体の状態を整合させる必要があります。

保守担当者向けメモ

  • 本構造体は mod_proxy_http.c 内の HTTP 固有ロジックに特化しています。
  • 100-Continue や Upgrade ヘッダーの処理は複雑であり、変更を加える際は process_proxy_header 等の関連関数への影響を考慮してください。
