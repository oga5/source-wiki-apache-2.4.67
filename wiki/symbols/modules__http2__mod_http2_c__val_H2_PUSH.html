val_H2_PUSH

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_http2.c:216

目的

HTTP/2 のサーバープッシュ機能が現在有効であるかどうかを判定し、環境変数として利用可能な文字列（"on" または "off"）を返します。

シグネチャまたは形状

static const char *val_H2_PUSH(apr_pool_t *p, server_rec *s,
                               conn_rec *c, request_rec *r,
                               h2_conn_ctx_t *conn_ctx)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • s: サーバー設定レコード。
  • c: 接続レコード。
  • r: リクエストレコード。
  • conn_ctx: HTTP/2 接続コンテキスト。
  • **出力**:
  • "on": プッシュが有効な場合。
  • "off": プッシュが無効な場合。

前提条件と不変条件

  • conn_ctx が提供されている場合、その内部状態（ストリームIDやセッション）に基づいて判定されます。
  • conn_ctx が提供されていない場合、サーバー設定（s）の H2_CONF_PUSH 設定値が参照されます。

アルゴリズムまたは振る舞い

1. conn_ctx が存在する場合:

  • r (リクエスト) が存在し、かつ conn_ctx->stream_id が有効であれば、該当ストリームのプッシュポリシーを確認します。ポリシーが H2_PUSH_NONE でなければ "on" を返します。
  • r が存在しない場合、c (接続) に対して h2_session_push_enabled を呼び出し、セッションレベルでプッシュが許可されていれば "on" を返します。

    • 2. conn_ctx が存在しない場合:

  • s (サーバー) の設定から H2_CONF_PUSH を取得し、有効であれば "on" を返します。

    • 3. 上記のいずれにも該当しない場合、デフォルトとして "off" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c 内の h2_var_def 配列（H2PUSH および H2_PUSH キーの定義箇所）。
  • **呼び出し先**:
  • h2_mplx_c2_stream_get
  • h2_session_push_enabled
  • h2_config_geti

関連データ構造

  • h2_conn_ctx_t: HTTP/2 接続のメタデータ管理構造体。
  • h2_stream: 個別のストリーム状態を保持する構造体。
  • server_rec: Apache サーバー設定構造体。

失敗モードと境界条件

  • conn_ctx が NULL の場合、サーバー設定のみに依存した判定が行われます。
  • r が NULL かつ conn_ctx が存在する場合、接続レベルのセッション設定が優先されます。

保守担当者向けメモ

  • 本関数は http2_var_lookup を通じて環境変数として公開されるため、判定ロジックを変更する際は、HTTP/2 仕様におけるプッシュの有効範囲（セッション単位 vs ストリーム単位）との整合性に注意してください。
