remoteip_req_t

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_remoteip.c:71

目的

リクエスト処理中に解析されたクライアント IP アドレスおよびプロキシ関連の情報を一時的に保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    apr_sockaddr_t *useragent_addr;
    char *useragent_ip;
    const char *proxy_ips;
    const char *proxied_remote;
} remoteip_req_t;

入力と出力

  • **入力**: リクエスト処理（post_read_request フック等）において、HTTP ヘッダーや PROXY プロトコルから抽出された情報が本構造体の各フィールドに格納されます。
  • **出力**: 最終的に特定されたクライアント IP 情報が r->useragent_addr および r->useragent_ip へ反映される際のソースとして利用されます。

前提条件と不変条件

  • r->pool からメモリが割り当てられることを想定しています。
  • 信頼できるプロキシ経由の接続であることを前提に、IP アドレスの検証および抽出が行われます。

アルゴリズムまたは振る舞い

1. リクエスト処理の初期段階で、apr_palloc を用いて r->pool 上にインスタンス化されます。 2. HTTP ヘッダー（X-Forwarded-For 等）の解析結果に基づき、信頼できるプロキシ IP リストと、信頼できないリモート IP リストを各フィールドに記録します。 3. 記録された情報を基に、最終的なクライアントの接続情報を特定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_remoteip.c 内の remoteip_modify_request 関数において、リクエスト処理中にインスタンス化および参照が行われます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_sockaddr_t: IP アドレスとポート情報を保持する APR の標準構造体。
  • request_rec: Apache のリクエスト情報を保持する構造体。本構造体は request_rec の処理を補助します。

失敗モードと境界条件

  • メモリ割り当て失敗時: apr_palloc が失敗した場合の挙動は Apache の標準的なメモリ管理に依存します。
  • 解析対象のヘッダーが存在しない場合や、信頼リストに合致しない IP が含まれる場合、各フィールドは適切に初期化またはスキップされます。

保守担当者向けメモ

  • 本構造体はリクエスト単位のライフサイクルを持つため、r->pool の生存期間を超えて保持しないでください。
  • proxy_ips および proxied_remote は文字列として保持されるため、解析後の検証ロジックと整合性を保つ必要があります。
