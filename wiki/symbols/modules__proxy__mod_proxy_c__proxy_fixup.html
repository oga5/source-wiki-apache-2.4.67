proxy_fixup

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:1159

目的

リクエストの修正フェーズにおいて、プロキシリクエストのURLを正規化（canonicalization）します。

シグネチャまたは形状

static int proxy_fixup(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: int （Apache のステータスコード。成功時は ap_proxy_canon_url の戻り値、対象外の場合は DECLINED）

前提条件と不変条件

  • r->proxyreq が真であること。
  • r->filename が存在し、かつ "proxy:" で始まる文字列であること。

アルゴリズムまたは振る舞い

1. リクエストがプロキシリクエストでない、または r->filename が "proxy:" で始まらない場合は、即座に DECLINED を返して処理を終了します。 2. 上記条件を満たす場合、ap_proxy_canon_url(r) を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の fixups フックとして登録されています（modules/proxy/mod_proxy.c:3371 にて ap_hook_fixups(proxy_fixup, ...) を実行）。
  • **呼び出し先**: ap_proxy_canon_url を呼び出します。
  • **備考**: proxy_util.c:2620 のコメントにおいて、本関数が URL から UDS（Unix Domain Socket）パスを削除済みであることが言及されています。

関連データ構造

  • request_rec: Apache のリクエスト構造体。

失敗モードと境界条件

  • r->proxyreq が偽、または r->filename が NULL もしくは "proxy:" で始まらない場合、本関数は何も行わず DECLINED を返します。
