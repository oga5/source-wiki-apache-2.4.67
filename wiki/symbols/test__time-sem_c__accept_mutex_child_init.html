accept_mutex_child_init

定義

  • 種別: function
  • 定義箇所: test/time-sem.c:158

目的

ベンチマーク実行時、子プロセスごとに排他制御用のファイル記述子を初期化する。

シグネチャまたは形状

void accept_mutex_child_init(void)

入力と出力

  • 入力: なし（グローバル定数 FNAME を使用）
  • 出力: なし（成功時）
  • 副作用: flock_fd グローバル変数にファイル記述子を代入する。

前提条件と不変条件

  • FNAME で指定されたファイルパスがアクセス可能であること。
  • 本関数は子プロセス生成後に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. FNAME を O_WRONLY モードでオープンする。 2. オープンに失敗した場合、perror でエラーを出力し、exit(1) でプロセスを即時終了する。 3. 成功した場合、取得したファイル記述子を flock_fd に格納する。

呼び出し元と呼び出し先

  • 呼び出し元: test/time-sem.c:538 に直接の呼び出しを確認。また、同ファイル内の複数の箇所でマクロとして定義されている。
  • 呼び出し先: open, perror, exit（標準ライブラリ）。

関連データ構造

  • flock_fd: ロック操作に使用されるファイル記述子を保持するグローバル変数。

失敗モードと境界条件

  • ファイルオープン失敗時: perror を出力して exit(1) を実行する。

保守担当者向けメモ

  • 本関数はベンチマークツール専用であり、エラー発生時は即座にプロセスを終了させる設計となっている。
  • マクロ定義による抽象化が行われているため、呼び出し箇所によって挙動が異なる可能性がある点に注意すること。
