md_acme_problem_is_input_related

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme.c:92

目的

ACME サーバーから返されたエラー（Problem）が、ユーザー入力に起因する問題であるかどうかを判定します。

シグネチャまたは形状

int md_acme_problem_is_input_related(const char *problem);

入力と出力

  • **入力**: problem (ACME エラータイプを示す文字列)
  • **出力**: ユーザー入力に関連するエラーであれば非ゼロ（真）、それ以外または不明な場合は 0（偽）を返します。

前提条件と不変条件

  • problem が NULL の場合は 0 を返します。
  • ACME プロトコルの標準的なプレフィックスである urn:ietf:params: または urn: が含まれている場合、それらをスキップして比較を行います。

アルゴリズムまたは振る舞い

1. problem 文字列が urn:ietf:params: で始まる場合、その長さをスキップします。 2. 上記に該当しない場合で、urn: で始まる場合はその長さをスキップします。 3. 内部で保持している Problems 配列を走査し、指定された problem 文字列と一致するエントリを探します。 4. 一致したエントリの input_related フラグの値を返します。 5. 一致するものが見つからない場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_status.c:638: 最終エラーがユーザー入力に関連しているかを確認するために呼び出されています。
  • **呼び出し先**:
  • apr_cstr_casecmp (APR ライブラリ関数)

関連データ構造

  • Problems: modules/md/md_acme.c 内で定義されている、ACME エラータイプと input_related フラグを保持する静的配列（詳細な構造定義は本関数のソース範囲外）。

失敗モードと境界条件

  • problem が空文字列の場合、ループは実行されず 0 を返します。
  • Problems 配列に存在しない未知のエラータイプが渡された場合、デフォルトで 0 を返します。

保守担当者向けメモ

  • 本関数は Problems 配列の定義に依存しています。新しい ACME エラータイプを追加する際は、Problems 配列の更新が必要です。
  • 比較には apr_cstr_casecmp を使用しているため、大文字小文字を区別しない比較が行われます。
