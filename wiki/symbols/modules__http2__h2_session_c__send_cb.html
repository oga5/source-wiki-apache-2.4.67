send_cb

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:187

目的

nghttp2 ライブラリが生成した HTTP/2 フレームデータを、セッションの I/O バッファ（h2_c1_io）へ転送するコールバック関数です。

シグネチャまたは形状

ssize_t send_cb(nghttp2_session *ngh2, const uint8_t *data, size_t length, int flags, void *userp)

入力と出力

  • **入力**:
  • ngh2: nghttp2 セッションポインタ（未使用）。
  • data: 送信対象のデータバッファ。
  • length: 送信データのバイト長。
  • flags: 送信フラグ（未使用）。
  • userp: h2_session 構造体へのポインタ。
  • **出力**:
  • 成功時: 送信したバイト数（length）。
  • ブロック時: NGHTTP2_ERR_WOULDBLOCK。
  • エラー時: h2_session_status_from_apr_status(rv) により変換されたエラーコード。

前提条件と不変条件

  • userp は有効な h2_session 構造体を指している必要があります。
  • session->io が初期化されていることが前提です。

アルゴリズムまたは振る舞い

1. h2_c1_io_needs_flush を呼び出し、I/O バッファがフラッシュを必要としている場合は NGHTTP2_ERR_WOULDBLOCK を返して処理を中断します。 2. h2_c1_io_add_data を呼び出し、データをセッションの I/O バッファへ追加します。 3. 追加が成功した場合は length を返します。 4. APR_STATUS_IS_EAGAIN(rv) が真の場合は NGHTTP2_ERR_WOULDBLOCK を返します。 5. その他のエラーが発生した場合は、ログを出力し、APR ステータスを nghttp2 用のエラーコードに変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: nghttp2 ライブラリ内部から呼び出されます（h2_session.c:724 にて NGH2_SET_CALLBACK を通じて登録）。
  • **呼び出し先**:
  • h2_c1_io_needs_flush
  • h2_c1_io_add_data
  • ap_log_cerror
  • h2_session_status_from_apr_status

関連データ構造

  • h2_session: セッション状態と I/O コンテキストを保持。
  • nghttp2_session: nghttp2 のセッション制御構造体。

失敗モードと境界条件

  • I/O バッファが満杯またはフラッシュ待ちの場合、NGHTTP2_ERR_WOULDBLOCK を返してフロー制御を維持します。
  • h2_c1_io_add_data が APR エラーを返した場合、デバッグログを出力し、セッション状態に応じたエラーコードを返します。

保守担当者向けメモ

  • 本関数は nghttp2 のコールバックインターフェースに準拠しています。
  • ngh2 および flags 引数は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
