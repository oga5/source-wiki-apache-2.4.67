util_ald_free

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache_mgr.c:79

目的

LDAP キャッシュ管理用のアロケータによって割り当てられたメモリ領域を解放します。共有メモリ（SHM）環境と標準ヒープ環境の両方に対応した抽象化レイヤーを提供します。

シグネチャまたは形状

void util_ald_free(util_ald_cache_t *cache, const void *ptr)

入力と出力

  • cache: キャッシュ管理構造体へのポインタ。共有メモリの管理情報（rmm_addr）を参照するために使用されます。
  • ptr: 解放対象のメモリ領域へのポインタ。NULL の場合は何もしません。
  • 戻り値: なし。

前提条件と不変条件

  • cache が有効な構造体であること。
  • ptr が util_ald_alloc または util_ald_strdup 等のキャッシュ専用アロケータによって割り当てられた領域であること。
  • 共有メモリが有効な場合、cache->rmm_addr が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. APR_HAS_SHARED_MEMORY が有効な場合:

  • cache->rmm_addr が存在すれば、apr_rmm_offset_get を用いてポインタをオフセットに変換し、apr_rmm_free で共有メモリ領域から解放します。
  • cache->rmm_addr が存在しない（共有メモリ未使用）場合は、標準の free() を呼び出します。

    • 2. APR_HAS_SHARED_MEMORY が無効な場合:

  • 標準の free() を呼び出します。

    • 3. いずれのケースでも、ptr が NULL の場合は解放処理を行いません。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ldap/util_ldap_cache.c 内の各キャッシュノード破棄処理（util_ald_free(cache, node) 等）から頻繁に呼び出されます。
  • 呼び出し先: apr_rmm_free（APRライブラリ）または標準ライブラリの free()。

関連データ構造

  • util_ald_cache_t: キャッシュのメモリ管理状態を保持する構造体。
  • apr_rmm_t: 共有メモリ管理用のアロケータ（APR）。

失敗モードと境界条件

  • ptr が NULL の場合、安全に無視されます。
  • 共有メモリ環境で cache->rmm_addr が正しく設定されていない状態で ptr を渡すと、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は util_ald_alloc と対になる関数です。メモリ割り当て戦略を変更する場合は、必ず本関数とセットで修正してください。
  • 共有メモリ使用時はポインタの直接解放ではなく、必ず apr_rmm を介した解放を行う必要があります。
