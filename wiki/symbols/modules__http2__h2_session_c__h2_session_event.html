h2_session_event

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:86

目的

HTTP/2 セッションに対して外部からイベントを投入し、状態遷移をトリガーするためのインターフェースを提供します。

シグネチャまたは形状

void h2_session_event(h2_session *session, h2_session_event_t ev, int err, const char *msg);

入力と出力

  • session: 操作対象の h2_session 構造体へのポインタ。
  • ev: 発生したイベントの種類（h2_session_event_t）。
  • err: イベントに関連するエラーコード。
  • msg: イベントに関する詳細な説明メッセージ。
  • 戻り値: なし（void）。

前提条件と不変条件

  • session ポインタは有効である必要があります。
  • 内部で呼び出される h2_session_dispatch_event が、セッションの状態遷移を安全に処理できる状態である必要があります。

アルゴリズムまたは振る舞い

受け取ったイベント ev、エラーコード err、メッセージ msg をそのまま h2_session_dispatch_event 関数へ転送し、セッションの状態に応じたルーティングを実行します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_c1_io.c:455（プロトコルエラー発生時など）。
  • 呼び出し先: h2_session_dispatch_event。

関連データ構造

  • h2_session: HTTP/2 セッションの管理構造体。
  • h2_session_event_t: セッションイベントの種類を定義する列挙型。

失敗モードと境界条件

  • session が NULL の場合、呼び出し元で適切なガードが行われていないとクラッシュする可能性があります。
  • 転送先の h2_session_dispatch_event がエラーを処理できない状態にある場合、セッションの異常終了や不正な状態遷移を招く可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーです。ロジックの変更が必要な場合は、実処理を行っている h2_session_dispatch_event を参照してください。
