clean_warning_headers

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_http.c:182

目的

HTTPレスポンスヘッダーに含まれる Warning ヘッダーを解析し、指定された時刻条件に合致するもののみを抽出して新しいテーブルへ格納する。

シグネチャまたは形状

static int clean_warning_headers(void *data, const char *key, const char *val)

入力と出力

  • **入力**:
  • data: header_dptr 構造体へのポインタ（解析対象のテーブル、メモリプール、比較用時刻を保持）。
  • key: ヘッダー名（通常は "Warning"）。
  • val: ヘッダーの値（warn-code SP warn-agent SP warn-text [SP warn-date] 形式）。
  • **出力**:
  • 常に 1 を返す（apr_table_do のコールバックとして動作）。
  • 副作用として、data 内のテーブルにフィルタリング済みのヘッダーが追加される。

前提条件と不変条件

  • data は header_dptr 型としてキャスト可能であること。
  • グローバル変数 warn_rx が正規表現として初期化されていること。

アルゴリズムまたは振る舞い

1. data からテーブルとプールを取得し、テーブルが未初期化であれば作成する。 2. warn_rx 正規表現を用いて val を走査する。 3. 各 Warning 値を解析し、warn-date が存在する場合は apr_date_parse_http で時刻に変換する。 4. 以下のいずれかの場合に、その Warning ヘッダーを新しいテーブルへ追加する。

  • warn-date が解析できない場合。
  • warn-date が data 内の基準時刻と一致する場合。

    • 5. val 内のすべてのマッチ箇所に対して処理を繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_http.c:236 において apr_table_do から呼び出される。
  • **呼び出し先**: apr_table_make, apr_pstrndup, ap_regexec, apr_date_parse_http, apr_table_addn。

関連データ構造

  • header_dptr: 解析中の状態（テーブル、プール、時刻）を保持する構造体。

失敗モードと境界条件

  • headers が NULL の場合、内部で apr_table_make を呼び出し初期化を行う。
  • 正規表現にマッチしないヘッダー値は無視される。
  • warn-date の解析に失敗した場合、そのヘッダーは無条件で保持される。

保守担当者向けメモ

  • Warning ヘッダーの複雑な構文解析に正規表現を使用しているため、RFC 7234 等の仕様変更に伴う構文の厳密化が必要な場合は warn_rx の定義を確認すること。
  • apr_table_do のコールバックとして設計されているため、戻り値はループ制御用であり、エラー発生時も 1 を返す仕様となっている。
