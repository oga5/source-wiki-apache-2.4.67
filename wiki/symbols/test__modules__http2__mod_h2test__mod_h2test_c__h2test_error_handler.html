h2test_error_handler

定義

  • 種別: function
  • 定義箇所: test/modules/http2/mod_h2test/mod_h2test.c:429

目的

HTTP/2 プロトコルのテスト環境において、意図的な HTTP ステータスコードの返却やエラー発生、レスポンスの遅延送信をシミュレートするリクエストハンドラです。

シグネチャまたは形状

static int h2test_error_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r（Apache リクエスト構造体）。r->args を通じて以下のパラメータを受け付けます。
  • status: HTTP ステータスコード
  • error / body_error: シミュレートするエラー種別（文字列）
  • error_bucket: エラーバケットの有効/無効フラグ
  • delay / body_delay: 送信前の遅延時間（例: "10s"）
  • **出力**: int（Apache ハンドラ戻り値: OK, DECLINED, AP_FILTER_ERROR）

前提条件と不変条件

  • r->handler が "h2test-error" であること。
  • リクエストメソッドが GET または POST であること。
  • メモリ管理は r->pool に依存しており、リクエスト終了時に自動解放されます。

アルゴリズムまたは振る舞い

1. **パラメータ解析**: r->args を解析し、遅延時間やエラー条件を抽出します。不明な引数は ap_die で HTTP_BAD_REQUEST を返します。 2. **環境設定**: レスポンスヘッダーから Content-Length や圧縮設定（Brotli/Gzip）を無効化し、application/octet-stream を設定します。 3. **初期遅延とエラー**: delay 指定があれば待機し、error 指定があれば ap_map_http_request_error を経由して即座にエラーを返します。 4. **レスポンス送信**: apr_bucket_brigade を使用してフラッシュバケットを送信後、ループ内でダミーデータ（'X'）を分割送信します。 6. **終了処理**: 正常終了時は EOS バケットを送信し OK を返します。エラー時は error_bucket 設定に基づきエラーバケットを挿入し AP_FILTER_ERROR を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフックシステム（h2test_hooks にて ap_hook_handler で登録）。
  • **呼び出し先**: status_from_str, duration_parse, ap_map_http_request_error, ap_pass_brigade, ap_bucket_error_create 等の Apache/APR API。

関連データ構造

  • request_rec: リクエスト情報の保持。
  • apr_bucket_brigade: レスポンスデータの構築とフィルタチェーンへの受け渡し。

失敗モードと境界条件

  • **引数解析失敗**: 不正な引数指定時は ap_die により HTTP_BAD_REQUEST を返します。
  • **送信失敗**: ap_pass_brigade が失敗した場合、cleanup ラベルへジャンプし、リソース解放後にエラーバケットを生成します。
  • **エラーバケット無効化**: error_bucket=0 が指定された場合、エラー発生時にバケットを挿入せず終了します。

保守担当者向けメモ

  • 本関数はテスト専用です。意図的にエラーを発生させるため、プロダクション環境での有効化は厳禁です。
  • goto cleanup パターンによるリソース管理が行われています。修正時は apr_brigade のクリーンアップ漏れに注意してください。
