alias_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_alias.c:59

目的

ディレクトリ単位（<Directory> や <Location> 等）で適用されるエイリアスおよびリダイレクト設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    unsigned int alias_set:1;
    unsigned int redirect_set:1;
    apr_array_header_t *redirects;
    const ap_expr_info_t *alias;
    const char *alias_fake;
    char *handler;
    const ap_expr_info_t *redirect;
    int redirect_status;
    int allow_relative;
    int alias_preserve_path;
} alias_dir_conf;

入力と出力

  • **入力**: 設定ファイル内のディレクティブ（Alias、Redirect 等）から生成されます。
  • **出力**: リクエスト処理時に ap_get_module_config を通じて取得され、try_alias や try_redirect 等の関数でリクエストの翻訳やリダイレクト判定に使用されます。

前提条件と不変条件

  • alias_set および redirect_set は、設定が明示的に定義されたかを示すフラグとして機能します。
  • alias および redirect フィールドは ap_expr_info_t を使用しており、式評価エンジンによる動的な解決を前提としています。

アルゴリズムまたは振る舞い

  • create_alias_dir_config により初期化され、merge_alias_dir_config により親コンテキストの設定とマージされます。
  • リクエスト処理の fixups フックにおいて、この構造体内の設定に基づきリダイレクト先が決定され、HTTP レスポンスヘッダの構築が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_alias.c 内の create_alias_dir_config、merge_alias_dir_config、try_alias、fixup_redir 等から参照されています。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • alias_entry: 個別のエイリアス定義を保持する構造体。
  • ap_expr_info_t: 式評価のための構造体。
  • apr_array_header_t: リダイレクト設定のリストを保持するために使用されます。

失敗モードと境界条件

  • redirect_status には 301, 302, 303, 410 などの HTTP ステータスコードが格納されます。
  • Alias ディレクティブはディレクトリコンテキスト内では使用できない制約があり、設定解析時にチェックされます。

保守担当者向けメモ

  • alias_preserve_path や allow_relative は、パス変換の挙動を微調整するためのフラグです。
  • apr_pcalloc を使用してメモリ確保が行われており、リクエスト終了時に自動的に解放されるライフサイクルを前提としています。
