include_server_config

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_include.c:125

目的

SSI（Server Side Includes）処理において、サーバー単位の設定情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    const char *default_start_tag;
    const char *default_end_tag;
} include_server_config;

入力と出力

  • 入力: サーバー設定の初期化時や、SSIディレクティブの解析時に参照される。
  • 出力: SSIタグの開始および終了シーケンスを定義する文字列（default_start_tag, default_end_tag）。

前提条件と不変条件

  • Apache HTTP Server のモジュール設定コンテキストの一部として管理される。
  • apr_palloc 等のメモリプールから割り当てられることが想定されている。

アルゴリズムまたは振る舞い

  • サーバー単位でSSIタグの形式をカスタマイズするための設定保持を行う。
  • ap_get_module_config を通じて、リクエスト処理中にサーバー設定から取得される。

呼び出し元と呼び出し先

  • 呼び出し元: mod_include.c 内の各関数（ap_get_module_config を使用して設定を取得する箇所）。
  • 呼び出し先: 明示的な関数呼び出しではなく、設定値の保持および参照を行うデータ構造である。

関連データ構造

  • include_dir_config: ディレクトリ単位の設定を保持する構造体。
  • ssi_internal_ctx: 解析状態を保持する内部構造体。

失敗モードと境界条件

  • 設定が初期化されていない状態でアクセスした場合、NULLポインタ参照の可能性がある。
  • メモリ割り当て失敗時は apr_palloc が例外を発生させる可能性がある。

保守担当者向けメモ

  • 本構造体はサーバー単位の設定を管理する。ディレクトリ単位の設定が必要な場合は include_dir_config を参照すること。
  • タグの開始・終了シーケンスを変更する際は、本構造体の値を適切に更新する必要がある。
