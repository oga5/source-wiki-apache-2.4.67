ap_expr_node_op_e

定義

  • 種別: enum
  • 定義箇所: server/util_expr_private.h:48

目的

Apache HTTP Server の式評価エンジン（ap_expr）において、抽象構文木（AST）の各ノードが表す演算子やオペランドの種類を識別するために使用されます。

シグネチャまたは形状

列挙型であり、以下のカテゴリの値を保持します。

  • **論理・比較演算子**: op_Not, op_Or, op_And, op_EQ, op_NE, op_LT, op_LE, op_GT, op_GE, op_IN 等
  • **文字列・正規表現演算子**: op_STR_EQ, op_REG, op_NRE 等
  • **リテラル・変数**: op_Digit, op_String, op_Var 等
  • **関数呼び出し関連**: op_UnaryOpCall, op_BinaryOpCall, op_StringFuncCall, op_ListFuncCall 等（およびそれらに付随する Info や Args ノード）

入力と出力

  • 入力: 構文解析器（util_expr_parse.c）がトークンを解釈する際に、ノードの型として割り当てます。
  • 出力: ap_expr_make 関数に引数として渡され、AST ノードの生成に使用されます。

前提条件と不変条件

  • ap_expr_make 関数において、ノード生成時の型識別子として必須です。
  • 関数呼び出し（Call）系の演算子を使用する場合、関連する Info ノードや Args ノードと組み合わせて AST を構成する必要があります。

アルゴリズムまたは振る舞い

式評価エンジンは、この列挙値に基づいて AST ノードを走査し、対応する評価ロジック（util_expr_eval.c）を分岐させます。特に Call 系の演算子は、関数ポインタや引数情報を保持するノードとリンクして動作します。

呼び出し元と呼び出し先

  • 呼び出し元: server/util_expr_parse.c（パーサー）および server/util_expr_eval.c 内の ap_expr_make 関数。
  • 呼び出し先: 本シンボル自体は列挙型定義であり、直接的な呼び出し先はありません。

関連データ構造

  • ap_expr_node: この列挙値を node_op フィールドとして保持する AST ノード構造体。
  • ap_expr_parse_ctx_t: 解析コンテキスト内で AST 構築の際に参照されます。

失敗モードと境界条件

  • 未定義の演算子型を ap_expr_make に渡した場合の挙動は、パーサーの実装に依存します。
  • Call 系の演算子を使用する際、対応する Info ノードや Args ノードのリンクが不完全な場合、評価時に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しい演算子を追加する場合は、本列挙型に値を追加し、util_expr_eval.c の評価ロジックを更新する必要があります。
  • Call 系のノード構造（Info や Args とのリンク関係）は、複雑な式を扱う際のメモリレイアウトに直結するため、変更時には注意が必要です。
