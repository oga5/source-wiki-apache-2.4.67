dav_fs_copymove_walk_ctx

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/repos.c:75

目的

WebDAV の COPY および MOVE 操作におけるディレクトリ走査中に、状態を保持および伝播するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    int is_move;                /* MOVE操作か否か */
    dav_buffer work_buf;        /* ファイル操作用バッファ */
    const dav_resource *res_dst; /* 特殊管理される二次リソース */
    const dav_resource *root;    /* 走査のルートリソース */
    apr_pool_t *pool;            /* 走査に使用するメモリプール */
} dav_fs_copymove_walk_ctx;

入力と出力

  • **入力**: 走査開始時に is_move フラグ、ルートリソース、およびメモリプールが設定されます。
  • **出力**: 走査プロセスを通じて、work_buf を介したファイル操作や、res_dst を通じたリソース状態の管理が行われます。

前提条件と不変条件

  • root および pool は、走査処理全体を通じて不変（invariant）です。
  • pool は、dav_walk_params から引き継がれたリクエスト単位のメモリ管理領域です。

アルゴリズムまたは振る舞い

  • COPY/MOVE 実行時に、走査対象の各リソースに対してこのコンテキストが渡されます。
  • is_move フラグに基づき、ファイル移動またはコピーのロジックを切り替えます。
  • work_buf は、dav_fs_copymove_file() 内での一時的なデータ処理に再利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/repos.c:1800 にて初期化されます。
  • modules/dav/fs/repos.c:1166 にて、走査中のコールバック関数内で wres->walk_ctx から取得されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、dav_fs_copymove_file() 等の関数に引数として渡され、操作の制御に使用されます。

関連データ構造

  • dav_resource: 走査対象および宛先リソースの定義に使用されます。
  • dav_buffer: ファイル操作用のバッファとして内部保持されます。
  • apr_pool_t: メモリ管理に使用されます。

失敗モードと境界条件

  • 構造体内の各ポインタが NULL である場合、走査中のアクセスでクラッシュする可能性があります。
  • is_move フラグの不整合は、ファイルシステム上のコピーと移動の挙動を混同させるリスクがあります。

保守担当者向けメモ

  • 本構造体は dav_walk_params の一部として走査中に引き回されます。
  • res_dst は特定の条件下で特殊なリソース管理を行うためのフィールドであり、通常の走査とは異なるパスを処理する際に使用されます。
