pattern_type

定義

  • 種別: enum
  • 定義箇所: modules/mappers/mod_rewrite.c:292

目的

RewriteCond ディレクティブにおいて、評価対象となるパターンや条件式の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    CONDPAT_REGEX, CONDPAT_FILE_EXISTS, CONDPAT_FILE_SIZE,
    CONDPAT_FILE_LINK, CONDPAT_FILE_DIR, CONDPAT_FILE_XBIT,
    CONDPAT_LU_URL, CONDPAT_LU_FILE,
    CONDPAT_STR_LT, CONDPAT_STR_LE, CONDPAT_STR_EQ, CONDPAT_STR_GT, CONDPAT_STR_GE,
    CONDPAT_INT_LT, CONDPAT_INT_LE, CONDPAT_INT_EQ, CONDPAT_INT_GT, CONDPAT_INT_GE,
    CONDPAT_AP_EXPR
} pattern_type;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • mod_rewrite のルール評価エンジンにおいて、条件式の解析結果としてこの列挙値が保持されます。

アルゴリズムまたは振る舞い

  • 正規表現 (CONDPAT_REGEX)、ファイルシステム属性チェック (CONDPAT_FILE_*)、マップルックアップ (CONDPAT_LU_*)、文字列比較 (CONDPAT_STR_*)、数値比較 (CONDPAT_INT_*)、および ap_expr 式 (CONDPAT_AP_EXPR) の各評価ロジックを分岐させるための識別子として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/mappers/mod_rewrite.c:334 にて rewritecond_entry 構造体のメンバ ptype として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • rewritecond_entry: この列挙型を保持する構造体。

失敗モードと境界条件

  • 列挙型であるため、未定義の値を代入した際の動作は未定義です。

保守担当者向けメモ

  • 新しい条件式タイプを追加する場合は、この列挙型に値を追加し、対応する評価ロジックを apply_rewrite_cond 内に実装する必要があります。
