util_ldap_strdup

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:134

目的

既存の文字列ポインタが指すメモリを解放し、新しい文字列を複製して代入する安全な文字列更新処理を提供します。

シグネチャまたは形状

static void util_ldap_strdup (char **str, const char *newstr)

入力と出力

  • str: 更新対象の文字列ポインタへのポインタ（char **）。
  • newstr: 新しくコピーするソース文字列（const char *）。
  • 出力: なし（str が指すポインタを直接更新）。

前提条件と不変条件

  • str が指すメモリ領域は、free() で解放可能なヒープメモリである必要があります。
  • newstr が NULL の場合、str は NULL に設定されます。

アルゴリズムまたは振る舞い

1. *str が非 NULL の場合、free(*str) を呼び出して既存のメモリを解放し、*str を NULL に初期化します。 2. newstr が非 NULL の場合、strdup(newstr) を呼び出して新しいメモリ領域を確保し、そのアドレスを *str に代入します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/ldap/util_ldap.c:835
  • modules/ldap/util_ldap.c:836
  • modules/ldap/util_ldap.c:888
  • modules/ldap/util_ldap.c:889
  • 呼び出し先:
  • free()

関連データ構造

  • 特になし。

失敗モードと境界条件

  • strdup() がメモリ不足により NULL を返した場合、*str は NULL になります。
  • newstr が NULL の場合、既存のメモリは解放されますが、新しいメモリは確保されません。

保守担当者向けメモ

  • 本関数は static であり、util_ldap.c 内でのみ使用されます。
  • strdup を使用しているため、呼び出し元は apr_pool_t によるメモリ管理ではなく、標準の free() による解放が適切に行われることを前提としています。
