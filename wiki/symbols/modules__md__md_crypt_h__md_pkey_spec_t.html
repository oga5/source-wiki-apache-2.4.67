md_pkey_spec_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_crypt.h:63

目的

秘密鍵の生成要件（アルゴリズムの種類およびそのパラメータ）を定義するための構造体です。

シグネチャまたは形状

typedef struct md_pkey_spec_t {
    md_pkey_type_t type;
    union {
        md_pkey_rsa_params_t rsa;
        md_pkey_ec_params_t ec;
    } params;
} md_pkey_spec_t;

入力と出力

本構造体は、md_pkey_gen() などの鍵生成関数に対する入力パラメータとして使用されます。

前提条件と不変条件

  • type フィールドの値に応じて、params 共用体の適切なメンバ（rsa または ec）が初期化されている必要があります。
  • 構造体内の各パラメータは、OpenSSL がサポートする有効な鍵生成要件を満たしている必要があります。

アルゴリズムまたは振る舞い

本構造体は、特定の暗号アルゴリズム（RSA または EC）に基づいた鍵生成の仕様を保持します。md_pkeys_spec_t を介してリスト管理されることがあり、md_crypt.c 内の関数によって配列として操作される場合があります。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • modules/md/md_acme_acct.c: アカウント鍵の仕様定義に使用。
  • modules/md/md_acme_authz.c: tls_alpn01_fnames 関数の引数として使用。
  • modules/md/md_acme_drive.c: ドメイン鍵の仕様管理に使用。
  • modules/md/md_acmev2_drive.c: md_pkeys_spec_get を介した取得。
  • modules/md/md_crypt.c: md_pkeys_spec_add による配列への追加。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • md_pkey_type_t: 鍵の種類（RSA/EC）を識別する列挙型。
  • md_pkey_rsa_params_t: RSA 鍵生成パラメータ。
  • md_pkey_ec_params_t: EC 鍵生成パラメータ。
  • md_pkeys_spec_t: 複数の md_pkey_spec_t を保持するコンテナ構造体。

失敗モードと境界条件

  • 不正な type が指定された場合、または params が未初期化の状態で鍵生成関数に渡された場合、OpenSSL 側の処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_md の暗号化抽象化レイヤーの一部です。新しい鍵タイプを追加する場合は、params 共用体に新しいメンバを追加し、関連する生成ロジックを md_crypt.c に実装する必要があります。
  • md_crypt.c 内で md_pkey_spec_t* の配列として扱われている箇所があるため、メモリ管理には注意してください。
