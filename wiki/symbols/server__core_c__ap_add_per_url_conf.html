ap_add_per_url_conf

定義

  • 種別: function
  • 定義箇所: server/core.c:624

目的

指定されたサーバー設定構造体（server_rec）に対し、URL単位の設定（url_config）を登録します。

シグネチャまたは形状

void ap_add_per_url_conf(server_rec *s, void *url_config)

入力と出力

  • **入力**:
  • s: 設定を追加する対象の server_rec 構造体。
  • url_config: 登録対象となる URL 設定へのポインタ。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • s->module_config から取得される core_server_config が有効である必要があります。
  • sconf->sec_url は apr_array_header_t 型の配列であり、apr_array_push が成功するメモリ状態である必要があります。

アルゴリズムまたは振る舞い

1. ap_get_core_module_config を使用して、サーバー設定からコアモジュール設定（core_server_config）を取得します。 2. sconf->sec_url 配列に新しい要素を確保するために apr_array_push を呼び出します。 3. 確保されたメモリ領域に url_config ポインタを格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:2613（設定解析プロセスの一部として呼び出されます）。
  • **呼び出し先**: ap_get_core_module_config および apr_array_push（APR ライブラリ関数）。

関連データ構造

  • server_rec: サーバー設定を保持するメイン構造体。
  • core_server_config: server/core.c で定義される、コアモジュール固有のサーバー設定構造体。sec_url メンバを保持します。

失敗モードと境界条件

  • apr_array_push は内部でメモリ割り当てを行うため、メモリ不足時にはアボートまたはエラーが発生する可能性があります。
  • s が NULL の場合、ap_get_core_module_config の呼び出しで未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は server/core.c 内の core_server_config 構造体にある sec_url 配列を操作します。設定の追加順序が重要となるケースがあるため、変更時は sec_url の走査順序への影響を考慮してください。
