content_length_ctx

定義

  • 種別: struct
  • 定義箇所: server/protocol.c:1849

目的

HTTP コンテンツ長計算フィルタ（ap_content_length_filter）における処理状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

struct content_length_ctx {
    int data_sent;
    apr_bucket_brigade *tmpbb;
};

入力と出力

  • data_sent (int): フィルタが既に後続の出力フィルタへ少なくとも1つのバケットを送信済みであるかを示すフラグ。
  • tmpbb (apr_bucket_brigade*): 一時的なバケットブリゲード。

前提条件と不変条件

  • data_sent は、フィルタがストリームの処理を開始した後に状態を追跡するために使用されます。

アルゴリズムまたは振る舞い

  • 本構造体は、ap_content_length_filter がレスポンスボディの長さを計算し、必要に応じて Content-Length ヘッダーを挿入する際の内部状態管理に使用されます。
  • data_sent が真の場合、フィルタは既にデータの一部を後続に渡していることを示します。

呼び出し元と呼び出し先

  • 呼び出し元: server/protocol.c 内の ap_content_length_filter 関数（直接的な参照が server/protocol.c:1866 に存在）。
  • 呼び出し先: 情報なし。

関連データ構造

  • apr_bucket_brigade: Apache HTTPD のバケットベース I/O 操作で使用されるデータ構造。

失敗モードと境界条件

  • 特筆すべき失敗モードや境界条件の記述はソースに含まれていません。

保守担当者向けメモ

  • 本構造体は server/protocol.c 内のコンテンツ長計算フィルタ専用のプライベートなコンテキストです。
  • tmpbb のメモリ管理については、リクエストのライフサイクル（request_rec のプール）に依存している可能性があります。
