dav_core_insert_prop

定義

  • 種別: function
  • 定義箇所: modules/dav/main/std_liveprop.c:59

目的

WebDAV の標準ライブプロパティ（DAV: 名前空間）の値を XML 形式で構築し、レスポンスヘッダーへ挿入します。

シグネチャまたは形状

static dav_prop_insert dav_core_insert_prop(const dav_resource *resource,
                                            int propid, dav_prop_insert what,
                                            apr_text_header *phdr);

入力と出力

  • **入力**:
  • resource: 対象となる DAV リソース。
  • propid: 挿入対象のプロパティ ID。
  • what: 挿入モード（値の挿入、またはサポート状況の確認）。
  • phdr: XML テキストを蓄積するヘッダー構造体。
  • **出力**:
  • 戻り値: 挿入結果を示す dav_prop_insert 型の値。

前提条件と不変条件

  • propid が DAV_PROPID_resourcetype の場合、リソースの型に応じた XML 文字列が構築されます。
  • DAV_PROPID_resourcetype 以外のプロパティが指定された場合、本関数は DAV_PROP_INSERT_NOTDEF を返し、処理を終了します。
  • 成功時には phdr に XML 文字列が追加されます。

アルゴリズムまたは振る舞い

1. **プロパティ判定**: propid が DAV_PROPID_resourcetype かどうかを判定します。 2. **拡張情報の収集**: resourcetype の場合、DAV_RESOURCE_TYPE_GROUP に登録された外部プロバイダから拡張情報を取得し、XML 文字列に連結します。 3. **リソース型の判定**: リソースの type（REGULAR, VERSION, COLLECTION 等）に基づき、対応する DAV XML タグ（<D:collection/>, <D:version-history/> 等）を value に追加します。 4. **XML 生成**: what モードに従い、dav_get_liveprop_info で取得した名前空間情報を用いて、<lpN:name>...</lpN:name> 形式またはサポート状況を示す XML を構築します。 5. **出力**: apr_text_append を使用して構築した文字列を phdr に書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/std_liveprop.c 内の dav_core_insert_all_liveprops 等から呼び出されます（ソース内の直接的な言及に基づく）。
  • **呼び出し先**: ap_list_provider_names, dav_get_resource_type_providers, dav_get_liveprop_info, apr_pstrcat, apr_psprintf, apr_text_append 等。

関連データ構造

  • dav_resource: リソースの属性を保持する構造体。
  • dav_liveprop_spec: ライブプロパティの仕様定義。
  • apr_text_header: XML 生成用のテキストバッファ。

失敗モードと境界条件

  • propid がサポート対象外の場合: DAV_PROP_INSERT_NOTDEF を返します。
  • resourcetype の判定で未知の型に遭遇した場合: DAV_PROP_INSERT_NOTDEF を返します。
  • value が空の場合: XML タグは空要素として出力されます。

保守担当者向けメモ

  • 本関数は DAV: 名前空間のプロパティのみを処理します。他の名前空間のプロパティは別のプロバイダによって処理されるため、拡張時は注意してください。
  • resourcetype の生成ロジックはリソースモデルに強く依存しています。新しいリソース型を追加する場合は、switch 文の拡張が必要です。
