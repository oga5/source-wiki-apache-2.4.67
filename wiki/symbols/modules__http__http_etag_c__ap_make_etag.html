ap_make_etag

定義

  • 種別: function
  • 定義箇所: modules/http/http_etag.c:353

目的

request_rec からリソースの ETag 文字列を生成するための簡易ラッパー関数です。

シグネチャまたは形状

char *ap_make_etag(request_rec *r, int force_weak)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報を保持する request_rec 構造体。
  • force_weak: ETag を弱識別子（W/ プレフィックス付き）として強制生成するかどうかを示すフラグ。
  • **出力**:
  • 生成された ETag 文字列（r->pool から割り当て）。

前提条件と不変条件

  • r->finfo に有効なファイル情報が格納されている必要があります。
  • 生成される文字列は r->pool に紐付いているため、リクエスト終了時に自動的に解放されます。

アルゴリズムまたは振る舞い

1. ローカルの etag_rec 構造体を初期化し、request_rec からファイルパス、ファイル情報、リクエスト時刻、および force_weak フラグをコピーします。 2. vlist_validator を NULL に設定します。 3. 構築した etag_rec を引数として ap_make_etag_ex を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_request.c
  • modules/mappers/mod_negotiation.c
  • **呼び出し先**:
  • ap_make_etag_ex (同一ファイル内)

関連データ構造

  • request_rec: リクエストのコンテキスト情報。
  • etag_rec: ap_make_etag_ex に渡すための、ETag 生成に必要な属性をまとめた内部構造体。

失敗モードと境界条件

  • r->finfo が適切に設定されていない場合、生成される ETag が不正確になる可能性があります。
  • FileETag ディレクティブの設定により、生成される ETag の内容がフィルタリングされる場合があります。

保守担当者向けメモ

  • 本関数は ap_make_etag_ex の簡易インターフェースです。詳細な制御が必要な場合は ap_make_etag_ex を直接使用してください。
  • r->pool を使用しているため、メモリリークの懸念はありません。
