ap_socache_hints

定義

  • 種別: struct
  • 定義箇所: include/ap_socache.h:53

目的

キャッシュプロバイダーの初期化時（init メソッド）に、バックエンド実装がメモリ割り当てや最適化の判断材料として利用するためのヒント情報を提供します。

シグネチャまたは形状

struct ap_socache_hints {
    apr_size_t avg_id_len;              /* IDの平均長 */
    apr_size_t avg_obj_size;            /* オブジェクトの平均サイズ */
    apr_interval_time_t expiry_interval; /* 有効期限のクリーンアップ間隔 */
};

入力と出力

  • **入力**: キャッシュを利用するモジュールが、自身の特性に合わせて各フィールドに値を設定します。
  • **出力**: キャッシュプロバイダー（mod_socache_shmcb, mod_socache_dbm 等）が、この構造体を参照して内部バッファのサイズ決定やタイマー設定を行います。

前提条件と不変条件

  • 各フィールドの値は、キャッシュの運用開始前に設定される必要があります。
  • expiry_interval は apr_interval_time_t 型であり、時間単位（マイクロ秒）で指定されます。

アルゴリズムまたは振る舞い

キャッシュプロバイダーは、init メソッドの引数として受け取ったこの構造体に基づき、以下の最適化を行います。 1. **メモリ確保**: avg_id_len と avg_obj_size を用いて、キャッシュエントリの平均的なメモリ消費量を予測します。 2. **メンテナンス**: expiry_interval を使用して、期限切れデータの削除処理を実行するバックグラウンドタスクの頻度を調整します。

  • modules/ssl/ssl_scache.c

関連データ構造

  • ap_socache_provider_t: この構造体を受け取る init メソッドを定義するプロバイダー構造体。

失敗モードと境界条件

  • **値の不適切設定**: avg_id_len や avg_obj_size に 0 を指定した場合の挙動は、各プロバイダーの実装に依存します（デフォルト値が使用されるか、エラーとなる可能性があります）。
  • **型制限**: apr_interval_time_t の範囲を超える値を expiry_interval に設定した場合の動作は未定義です。

保守担当者向けメモ

  • 本構造体は、キャッシュ実装のパフォーマンスを左右する重要なパラメータです。新しいキャッシュプロバイダーを追加する際は、これらのヒントを適切に活用しているか確認してください。
  • retrieve メソッドの datalen と同様、ヒントの値が実際のデータ特性と乖離している場合、キャッシュ効率が低下する可能性がある点に留意してください。
