action_dir_config

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_actions.c:52

目的

Apache HTTP Server において、ディレクトリごとの設定として MIME タイプおよび HTTP メソッドに関連付けられたスクリプト実行情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    apr_table_t *action_types;       /* Added with Action... */
    const char *scripted[METHODS];   /* Added with Script... */
    int configured;                  /* True if Action or Script has been called */
} action_dir_config;

入力と出力

  • 本構造体は、設定ファイル内の Action および Script ディレクティブによって構築・更新されます。
  • action_types は MIME タイプとスクリプトパスのペアを格納するテーブルです。
  • scripted は HTTP メソッド（METHODS 定数で定義された数）に対応するスクリプトパスを保持する配列です。

前提条件と不変条件

  • configured フィールドは、Action または Script ディレクティブが少なくとも一度呼び出された場合に真（非ゼロ）となります。
  • scripted 配列のインデックスは、Apache のメソッド定数に対応しています。

アルゴリズムまたは振る舞い

  • 本構造体は、リクエスト処理時に action_handler によって参照されます。
  • MIME タイプベースのアクション（Action）は、メソッドベースのスクリプト（Script）よりも優先的に評価されます。
  • 内部リダイレクトループを避けるため、リクエスト処理時には r->prev のチェックが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_actions.c 内の create_action_dir_config（初期化）、merge_action_dir_configs（設定マージ）、add_action / set_script（設定登録）、および action_handler（リクエスト処理）から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_table_t: action_types の格納に使用される APR のテーブル構造体。
  • METHODS: scripted 配列のサイズを決定する定数。

失敗モードと境界条件

  • Action ディレクティブで virtual オプションが指定されていない場合、スクリプトの実在チェックが行われます。
  • 設定が未構成（configured が偽）の場合、本モジュールはリクエスト処理をスキップします。

保守担当者向けメモ

  • Action による MIME タイプベースの処理は Script によるメソッドベースの処理を上書きする仕様であるため、マージや設定の優先順位を扱う際は注意してください。
  • action_handler におけるリクエスト転送には ap_internal_redirect_handler が利用されます。
