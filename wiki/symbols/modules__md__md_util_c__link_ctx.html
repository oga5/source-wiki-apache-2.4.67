link_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_util.c:1240

目的

HTTP Link ヘッダーの解析処理において、解析対象の文字列と現在の走査位置、および抽出されたリンク要素やパラメータのオフセット情報を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    const char *s;          /* 解析対象の文字列 */
    apr_size_t slen;        /* 文字列の長さ */
    apr_size_t i;           /* 現在の走査位置 */
    apr_size_t link_start;  /* リンクの開始位置 */
    apr_size_t link_len;    /* リンクの長さ */
    apr_size_t pn_start;    /* パラメータ名の開始位置 */
    apr_size_t pn_len;      /* パラメータ名の長さ */
    apr_size_t pv_start;    /* パラメータ値の開始位置 */
    apr_size_t pv_len;      /* パラメータ値の長さ */
} link_ctx;

入力と出力

本構造体は、HTTP Link ヘッダーを解析する関数群（read_link 等）の引数として渡され、解析状態の更新および抽出結果の保持に使用されます。

前提条件と不変条件

  • s は解析対象の有効な文字列を指す必要があります。
  • i は常に 0 から slen の範囲内である必要があります。
  • 各 _start および _len フィールドは、s 内の有効な部分文字列を指すように更新されることが期待されます。

アルゴリズムまたは振る舞い

この構造体は、Link ヘッダーの構文解析におけるステートマシン的な役割を果たします。解析関数は link_ctx を参照して、空白のスキップ、文字の検索、パラメータの特定を行い、その結果を構造体内のオフセット情報に書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_util.c 内の複数の静的関数（skip_ws, find_chr, read_chr, skip_qstring, skip_ptoken, read_link, skip_pname, skip_pvalue, skip_param, read_sep, subst_str）が、この構造体を引数として受け取り、解析状態を操作しています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • apr_size_t: APR（Apache Portable Runtime）で定義されたサイズ型。

失敗モードと境界条件

  • slen を超えるアクセスが発生しないよう、各解析関数は i の値を適切にチェックする必要があります。
  • 文字列が空の場合や、期待されるフォーマット（Link ヘッダー形式）に適合しない場合、解析関数はエラーを返すことが想定されます。

保守担当者向けメモ

  • 本構造体は modules/md/md_util.c で定義されていますが、modules/http2/h2_proxy_util.c からも参照されています。変更を加える際は、両方のファイルにおける解析ロジックへの影響を確認してください。
