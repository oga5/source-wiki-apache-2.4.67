mxcfg_lookup

定義

  • 種別: function
  • 定義箇所: server/util_mutex.c:319

目的

指定されたミューテックス型に対し、設定ディレクティブ、デフォルト値、および登録時の設定を優先順位に従って解決し、有効な mutex_cfg_t 構造体を返します。

シグネチャまたは形状

static mutex_cfg_t *mxcfg_lookup(apr_pool_t *p, const char *type)

入力と出力

  • **入力**:
  • p: 設定の複製に使用するメモリプール。
  • type: 検索対象のミューテックス型名。
  • **出力**:
  • 解決された mutex_cfg_t 構造体へのポインタ。型が見つからない場合は NULL を返します。

前提条件と不変条件

  • mxcfg_by_type ハッシュテーブルが初期化済みであること。
  • 検索対象の type が ap_mutex_register() を通じて登録済みであること。

アルゴリズムまたは振る舞い

1. mxcfg_by_type から "default" 設定と指定された type の設定を取得します。 2. 以下の優先順位で設定を選択します：

  • 1. 指定された type の Mutex ディレクティブ設定。
  • 2. "default" の Mutex ディレクティブ設定。
  • 3. ap_mutex_register() 時の type 固有デフォルト。
  • 4. グローバルデフォルト。

    • 3. ファイルベースのロック機構（flock 等）を使用する場合で、かつディレクトリが未設定であれば、優先順位に従ってディレクトリパスを補完します。この際、メモリプール p を使用して新しい設定構造体を複製します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/util_mutex.c:416
  • server/util_mutex.c:463
  • server/util_mutex.c:513
  • **呼び出し先**:
  • apr_hash_get: ハッシュテーブルからの設定取得。
  • apr_pmemdup: ディレクトリパス補完時の構造体複製。
  • mutex_needs_file: ファイルパスが必要なメカニズムかどうかの判定。

関連データ構造

  • mutex_cfg_t: ミューテックスのメカニズム、ディレクトリ、オプションを保持する構造体。
  • mxcfg_by_type: 登録済みミューテックス設定を保持する内部ハッシュテーブル。

失敗モードと境界条件

  • 指定された type がハッシュテーブルに存在しない場合、NULL を返します。
  • ファイルベースのロックでディレクトリがどこにも定義されていない場合、ディレクトリは設定されません。

保守担当者向けメモ

  • 本関数は設定の解決のみを行い、実際のロック生成は ap_global_mutex_create 等の呼び出し元で行われます。
  • newcfg の複製は、ディレクトリパスの補完が必要な場合にのみ発生するため、メモリ効率に留意してください。
