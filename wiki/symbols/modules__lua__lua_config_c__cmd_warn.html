cmd_warn

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:192

目的

Lua スクリプトから Apache のログシステムに対し、APLOG_WARNING レベルでメッセージを出力するためのインターフェースを提供します。

シグネチャまたは形状

static int cmd_warn(lua_State *L)

入力と出力

  • **入力**: lua_State *L（Lua の実行状態）。スタックにはログ出力対象のメッセージ文字列が含まれていることが想定されます。
  • **出力**: int（Lua スタックにプッシュされた戻り値の数）。内部で呼び出される cmd_log_at の戻り値をそのまま返します。

前提条件と不変条件

  • L は有効な lua_State である必要があります。
  • 呼び出し元は、Lua スクリプトから cmd_warn メソッドが実行されるコンテキストである必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った lua_State *L を cmd_log_at 関数に渡します。 2. その際、ログレベルとして APLOG_WARNING を指定します。 3. cmd_log_at が実行した結果（通常はスタック操作の結果）を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_config.c:253 において、cmd_methods テーブルの "warn" キーに関連付けられています。
  • **呼び出し先**: cmd_log_at（同ファイル内）。

関連データ構造

  • cmd_methods: CommandParameters 用のメソッドテーブル。cmd_warn はこのテーブルの一部として登録されています。

失敗モードと境界条件

  • cmd_log_at の内部実装に依存します。Lua スタックが不正な場合や、ログ出力先が確保できない場合には cmd_log_at 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は cmd_log_at のラッパーです。ログレベルを変更する場合を除き、ロジックの修正は cmd_log_at に対して行ってください。
  • APLOGNO マクロは意図的に省略されています。
