ap_update_child_status_from_conn

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:603

目的

接続情報（conn_rec）に基づいて、スコアボード上の特定のワーカープロセスの状態を更新します。

シグネチャまたは形状

int ap_update_child_status_from_conn(ap_sb_handle_t *sbh, int status, conn_rec *c)

入力と出力

  • **入力**:
  • sbh: 更新対象のワーカーを特定するスコアボードハンドル。
  • status: 設定する新しい状態（例: SERVER_BUSY_READ）。
  • c: 接続情報を保持する conn_rec 構造体。
  • **出力**:
  • 成功時は update_child_status_internal の戻り値を返します。
  • sbh が無効（NULL または child_num < 0）な場合は -1 を返します。

前提条件と不変条件

  • sbh は有効なスコアボードハンドルである必要があります。
  • 本関数は、スコアボードの各スロットが単一の所有者によってのみ書き込まれるという設計上の不変条件に依存しており、排他制御なしで実行されます。

アルゴリズムまたは振る舞い

1. sbh が NULL であるか、child_num が負の値であるかを検証し、無効な場合は即座に -1 を返します。 2. 有効な場合、sbh から抽出した child_num および thread_num を使用して、内部関数 update_child_status_internal を呼び出します。 3. update_child_status_internal には、接続情報 c を渡し、リクエストやその他の詳細情報は NULL として引き渡します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_core.c (直接の呼び出しを確認)
  • **呼び出し先**:
  • update_child_status_internal (内部関数)

関連データ構造

  • ap_sb_handle_t: スコアボード上のワーカーを特定するためのハンドル。
  • conn_rec: 接続ごとの状態を保持する構造体。

失敗モードと境界条件

  • sbh が NULL の場合、または child_num が初期化されていない（負の）場合は、更新を行わず -1 を返します。

保守担当者向けメモ

  • 本関数は ap_update_child_status ファミリーの一部であり、接続コンテキストからの更新に特化しています。
  • update_child_status_internal への引数としてリクエスト関連のパラメータが NULL で渡されているため、この関数経由ではリクエスト文字列等の詳細な統計更新は行われません。
