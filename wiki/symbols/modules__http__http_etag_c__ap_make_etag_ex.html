ap_make_etag_ex

定義

  • 種別: function
  • 定義箇所: modules/http/http_etag.c:214

目的

リクエストされたリソースに対して、設定（FileETag ディレクティブ）およびファイル属性に基づいた一意な ETag 文字列を生成します。

シグネチャまたは形状

char *ap_make_etag_ex(request_rec *r, etag_rec *er)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。r->per_dir_config から設定を取得し、r->vlist_validator があれば検証情報を統合します。
  • er: ETag 生成に必要なファイル情報（finfo）や強制弱タグフラグ（force_weak）を含む構造体。
  • **出力**: 生成された ETag 文字列（char *）。メモリは r->pool から割り当てられます。

前提条件と不変条件

  • etag_bits が ETAG_NONE の場合、空文字列 "" を返します。
  • 生成される文字列は r->pool の生存期間に依存します。
  • er->finfo が有効である必要があります。

アルゴリズムまたは振る舞い

1. **設定評価**: core_dir_config から etag_bits を取得し、適用すべき属性を決定します。 2. **弱タグ判定**: er->force_weak が真の場合、または r->vlist_validator が存在しそれが弱タグである場合、W/ プレフィックスを付与します。 3. **強力な ETag の生成**: AP_REQUEST_IS_STRONG_ETAG(r) が真、または ETAG_DIGEST が設定されている場合、make_digest_etag を呼び出し SHA1 ハッシュを生成します。 4. **属性ベースの生成**: 通常時は inode, size, mtime を連結した文字列を生成します。

  • ファイルの最終更新時刻がリクエスト時刻から 1 秒以内の場合、安全のため弱タグとして扱います。
  • etag_bits に応じて inode, size, mtime を選択的に含めます。

    • 5. **VLV 統合**: r->vlist_validator が存在する場合、生成した ETag の末尾に結合します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/repos.c:1903
  • modules/http/http_etag.c:364 (ラッパー ap_make_etag)
  • modules/http/http_etag.c:380 (ラッパー ap_set_etag)
  • modules/http/http_etag.c:404 (ラッパー ap_set_etag_fd)
  • **呼び出し先**:
  • make_digest_etag (条件付き)
  • etag_start, etag_end
  • etag_uint64_to_hex

関連データ構造

  • request_rec: リクエストコンテキスト。
  • etag_rec: ETag 生成の入力パラメータを保持。
  • core_dir_config: FileETag ディレクティブの設定値。

失敗モードと境界条件

  • etag_bits が ETAG_NONE の場合、ETag 生成をスキップし空文字列を返します。
  • ファイル情報が取得できない場合（APR_NOFILE）、mtime のみを使用して ETag を生成します。
  • r->vlist_validator が存在する場合、その内容に応じて ETag 全体が弱タグ化されることがあります。

保守担当者向けメモ

  • 本関数は r->pool を使用するため、メモリリークの心配はありません。
  • make_digest_etag はファイル読み込みを伴うため、パフォーマンスへの影響を考慮してください。
  • ETAG_BACKWARD は etag_bits が未設定（ETAG_UNSET）の場合のデフォルト動作として機能します。
