md_rand_bytes

定義

  • 種別: function
  • 定義箇所: modules/md/md_crypt.c:170

目的

OpenSSL の乱数生成機能を利用して、指定されたサイズの暗号学的に安全な乱数を生成し、提供されたバッファに格納する。

シグネチャまたは形状

apr_status_t md_rand_bytes(unsigned char *buf, apr_size_t len, apr_pool_t *p)

入力と出力

  • **入力**:
  • buf: 乱数を格納するメモリ領域へのポインタ。
  • len: 生成する乱数のバイト数。
  • p: メモリ管理および初期化チェックに使用する APR プール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS を返し、失敗時はエラーコードを返す。
  • buf: 乱数データで上書きされる。

前提条件と不変条件

  • len が INT_MAX を超える場合、処理は行われず APR_ENOTIMPL が返される。
  • 内部で md_crypt_init(p) を呼び出し、OpenSSL の初期化と乱数シードのセットアップが完了していることを保証する。

アルゴリズムまたは振る舞い

1. len が INT_MAX を超えていないか検証する。 2. md_crypt_init(p) を呼び出し、OpenSSL の初期化状態を確認・実行する。 3. 初期化が成功した場合、OpenSSL の RAND_bytes を呼び出し、buf に len バイトの乱数を書き込む。 4. 最終的なステータスコードを呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_drive.c:1225
  • modules/md/md_crypt.c:1965
  • modules/md/md_status.c:666
  • modules/md/md_store_fs.c:120
  • modules/md/mod_md_drive.c:250
  • **呼び出し先**:
  • md_crypt_init (OpenSSL 初期化)
  • RAND_bytes (OpenSSL API)

関連データ構造

  • apr_pool_t: メモリ管理および初期化状態の追跡に使用。

失敗モードと境界条件

  • len > INT_MAX: APR_ENOTIMPL を返す。
  • md_crypt_init の失敗: md_crypt_init が返すエラーコードをそのまま伝播する。

保守担当者向けメモ

  • 本関数は OpenSSL の RAND_bytes をラップしている。
  • md_crypt_init が内部で呼ばれるため、スレッドセーフな初期化が保証されている。
  • len の型が apr_size_t であるのに対し、OpenSSL の RAND_bytes は int を引数にとるため、INT_MAX による境界チェックが必須となっている。
