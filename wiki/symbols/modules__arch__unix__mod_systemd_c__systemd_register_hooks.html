systemd_register_hooks

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_systemd.c:156

目的

Apache HTTP Server のライフサイクルイベントおよびオプション機能に対するフックを登録し、systemd との統合を有効化する。

シグネチャまたは形状

static void systemd_register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p (フック登録に使用されるメモリプール)
  • **出力**: なし (void)

前提条件と不変条件

  • 本関数はモジュール読み込み時に呼び出される。
  • 登録される各フック関数（systemd_pre_config 等）は、Apache のフック機構を通じて適切なタイミングで実行される。

アルゴリズムまたは振る舞い

1. ap_systemd_listen_fds および ap_find_systemd_socket をオプション関数として登録する。 2. ap_hook_pre_config を APR_HOOK_LAST で登録し、設定読み込み開始時に systemd_pre_config を実行する。 3. ap_hook_post_config を APR_HOOK_REALLY_LAST で登録し、設定完了後に systemd_post_config を実行する。 4. ap_hook_pre_mpm を APR_HOOK_LAST で登録し、MPM 起動直前に systemd_pre_mpm を実行する。 5. ap_hook_monitor を APR_HOOK_MIDDLE で登録し、定期的な監視処理として systemd_monitor を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/unix/mod_systemd.c:178 (モジュール定義内のフック登録箇所)
  • **呼び出し先**:
  • APR_REGISTER_OPTIONAL_FN (マクロ)
  • ap_hook_pre_config, ap_hook_post_config, ap_hook_pre_mpm, ap_hook_monitor (Apache フック登録関数)

関連データ構造

  • apr_pool_t: フック登録のライフサイクルを管理するメモリプール。

失敗モードと境界条件

  • フック登録自体はメモリ確保に依存するが、本関数内での明示的なエラーハンドリングは定義されていない。

保守担当者向けメモ

  • 本関数は mod_systemd のエントリーポイントの一部であり、フックの実行順序（LAST, REALLY_LAST 等）は他のモジュールとの競合を避けるために重要である。変更時は Apache のフック優先順位に注意すること。
