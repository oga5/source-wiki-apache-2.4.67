slotmem_attach

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:120

目的

既存の ap_slotmem_instance_t インスタンスを名前で検索し、プロセス内で再利用可能にする。

シグネチャまたは形状

apr_status_t slotmem_attach(ap_slotmem_instance_t **new, const char *name, apr_size_t *item_size, unsigned int *item_num, apr_pool_t *pool)

入力と出力

  • **入力**:
  • name: 検索対象のインスタンス名。
  • pool: パス解決に使用するメモリプール。
  • **出力**:
  • new: 成功時に見つかったインスタンスへのポインタが格納される。
  • item_size: 成功時にインスタンスのスロットサイズが格納される。
  • item_num: 成功時にインスタンスのスロット数が格納される。
  • **戻り値**: 成功時は APR_SUCCESS、失敗時は APR_ENOSHMAVAIL。

前提条件と不変条件

  • name が NULL の場合、直ちに APR_ENOSHMAVAIL を返す。
  • name が : で始まる場合はそのまま使用し、それ以外の場合は ap_runtime_dir_relative を用いてランタイムディレクトリからの相対パスとして解決する。

アルゴリズムまたは振る舞い

1. name を基に検索用のファイルパス fname を決定する。 2. 静的変数 globallistmem を先頭とするリストを走査する。 3. リスト内の各インスタンスの name と fname を strcmp で比較する。 4. 一致するインスタンスが見つかった場合、そのインスタンス情報を new, item_size, item_num に書き込み APR_SUCCESS を返す。 5. リストの末尾まで一致するものがない場合、APR_ENOSHMAVAIL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_balancer.c:2028 (エラーログ出力)
  • modules/proxy/proxy_util.c:1358 (エラーログ出力)
  • modules/slotmem/mod_slotmem_plain.c:307 (プロバイダー構造体への登録)
  • modules/slotmem/mod_slotmem_shm.c:736 (プロバイダー構造体への登録)
  • **呼び出し先**:
  • ap_runtime_dir_relative (パス解決)
  • strcmp (文字列比較)

関連データ構造

  • ap_slotmem_instance_t: 管理対象となるスロットメモリのインスタンス。
  • globallistmem: インスタンスを保持する静的リストの先頭ポインタ。

失敗モードと境界条件

  • name が NULL の場合: APR_ENOSHMAVAIL を返す。
  • 指定された名前のインスタンスがリスト内に存在しない場合: APR_ENOSHMAVAIL を返す。

保守担当者向けメモ

  • 本関数は単一プロセス内での検索を行うため、ロック機構は存在しない。
  • name の比較は strcmp による完全一致で行われる。
  • item_size および item_num は、インスタンスが見つかった場合にのみ更新される。
