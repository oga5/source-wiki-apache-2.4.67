h2_c1_io_assure_flushed

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1_io.c:349

目的

HTTP/2 クライアント接続（C1）において、保留中の出力データが存在する場合に、それらを強制的にフィルタチェーンへフラッシュする。

シグネチャまたは形状

apr_status_t h2_c1_io_assure_flushed(h2_c1_io *io)

入力と出力

  • **入力**: h2_c1_io *io - 操作対象のクライアント接続 IO 構造体。
  • **出力**: apr_status_t - 操作の結果。成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • io は有効な h2_c1_io 構造体を指している必要がある。
  • 内部的に pass_output を呼び出すため、呼び出し元は再帰的な呼び出しが発生しない状態であることを保証する必要がある。

アルゴリズムまたは振る舞い

1. h2_c1_io_pending(io) により保留中のデータがあるか、または io->unflushed フラグがセットされているかを確認する。 2. 上記のいずれかが真である場合、pass_output(io, 1) を呼び出し、データをフィルタチェーンへ書き出す。 3. pass_output がエラーを返した場合は、そのステータスを保持して終了する。 4. 成功した場合は APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c (複数箇所: 781, 830, 1366, 1934, 2015 行目)
  • **呼び出し先**:
  • h2_c1_io_pending (判定用)
  • pass_output (データ転送用)

関連データ構造

  • h2_c1_io: クライアント接続の入出力状態を管理する構造体。unflushed フラグや出力ブリゲードを保持する。

失敗モードと境界条件

  • pass_output が失敗した場合、そのエラーコードがそのまま戻り値となる。
  • 保留データがない場合は何も行わず APR_SUCCESS を返す。

保守担当者向けメモ

  • 本関数は h2_session の各イベント処理の終了時や、接続のクリーンアップ時にフラッシュを保証するために使用される。
  • pass_output の第2引数に 1 を渡すことで、強制的なフラッシュを指示している。
