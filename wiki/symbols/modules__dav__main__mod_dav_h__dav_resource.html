dav_resource

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:387

目的

WebDAV リソースの抽象表現。リポジトリ固有の情報と、そのリソースを操作するためのフックおよびメモリ管理用プールを保持する。

シグネチャまたは形状

typedef struct dav_resource {
    dav_resource_type type;
    int exists;
    int collection;
    int versioned;
    int baselined;
    int working;
    const char *uri;
    dav_resource_private *info;
    const dav_hooks_repository *hooks;
    apr_pool_t *pool;
} dav_resource;

入力と出力

  • 入力: リクエスト処理時にプロバイダーの get_resource フックを通じて生成される。
  • 出力: 各種 WebDAV メソッド（PROPFIND, PUT, LOCK 等）の処理において、リソースの状態判定やバックエンド操作の起点として利用される。

前提条件と不変条件

  • pool: この構造体に関連するアイテムの割り当てに使用され、その生存期間は dav_resource 構造体以上である必要がある。
  • baselined が 1 の場合、versioned も 1 である必要がある。
  • uri: 現在の ABI では、エンコード済みか否かが混在する可能性があるため注意が必要。

アルゴリズムまたは振る舞い

リソースの属性（exists, collection, versioned, baselined, working）に基づき、WebDAV プロトコル上のリソース種別を特定する。プロバイダーは info フィールドにリポジトリ固有のデータを格納し、hooks フィールドにそのリソースを操作するためのリポジトリフックを保持させる。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/fs/dbm.c および modules/dav/fs/lock.c 内の各関数が、ファイルシステム実装のコンテキストで本構造体を参照している。
  • 呼び出し先: hooks フィールドに格納された dav_hooks_repository 構造体内の関数ポインタを介して、リポジトリ固有の操作が実行される。

関連データ構造

  • dav_resource_type: リソースの型定義。
  • dav_resource_private: プロバイダー固有のプライベート情報。
  • dav_hooks_repository: リソース操作のためのフックテーブル。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • uri フィールドのエンコード状態が不確定であるため、文字列操作時には注意が必要。
  • exists が 0 の場合、そのリソースは「null リソース」として扱われる。

保守担当者向けメモ

  • 本構造体は mod_dav のコア API であり、変更は ABI に影響を与える可能性がある。
  • uri フィールドのエンコードに関する ABI の不整合は既知の課題である。
  • プロバイダーが提供する info は dav_resource_private 型として抽象化されており、具体的な実装は各プロバイダー（mod_dav_fs 等）に依存する。
