send_handles_to_child

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:349

目的

親プロセスから子プロセスへ、プロセス間通信および同期に必要なハンドル（イベント、ミューテックス、共有メモリ）をパイプ経由で受け渡す。

シグネチャまたは形状

int send_handles_to_child(apr_pool_t *p,
                          HANDLE child_ready_event,
                          HANDLE child_exit_event,
                          apr_proc_mutex_t *child_start_mutex,
                          apr_shm_t *scoreboard_shm,
                          HANDLE hProcess,
                          apr_file_t *child_in);

入力と出力

  • **入力**:
  • child_ready_event, child_exit_event: 子プロセスに渡す同期用イベントハンドル。
  • child_start_mutex: 子プロセス起動同期用ミューテックス。
  • scoreboard_shm: プロセス間で共有するスコアボード用共有メモリ。
  • hProcess: 子プロセスのプロセスハンドル。
  • child_in: 子プロセスとの通信用パイプ（書き込み用）。
  • **出力**:
  • 成功時: 0
  • 失敗時: -1

前提条件と不変条件

  • hProcess は有効な子プロセスのハンドルであること。
  • child_in は子プロセスが読み取り可能なパイプとして開かれていること。
  • my_generation（グローバル変数）が現在の世代番号として保持されていること。

アルゴリズムまたは振る舞い

1. 現在の世代番号 (my_generation) をパイプに書き込む。 2. DuplicateHandle を使用し、各リソース（イベント、ミューテックス、共有メモリ）を子プロセスのコンテキストで有効なハンドルに複製する。 3. 複製したハンドルを順次パイプへ書き込む。 4. 各ステップでエラーが発生した場合、ap_log_error でログを出力し、即座に -1 を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/mpm_winnt.c:654（子プロセス生成プロセスの一部として呼び出される）。
  • **呼び出し先**:
  • DuplicateHandle (Windows API)
  • apr_file_write_full (APR)
  • apr_os_proc_mutex_get (APR)
  • apr_os_shm_get (APR)
  • ap_log_error (Apache Core)

関連データ構造

  • apr_proc_mutex_t: プロセス間ミューテックス。
  • apr_shm_t: 共有メモリ。
  • HANDLE: Windows API のハンドル型。

失敗モードと境界条件

  • DuplicateHandle が失敗した場合、リソースの複製ができずエラーを返す。
  • パイプへの書き込み (apr_file_write_full) が失敗した場合、子プロセスとの同期が確立できないためエラーを返す。
  • 各エラー発生時に APLOGNO を含む詳細なエラーログが記録される。

保守担当者向けメモ

  • 本関数はWindows特有のハンドル継承メカニズムに依存している。
  • 複製するハンドルのアクセス権限（EVENT_MODIFY_STATE 等）は、子プロセス側での利用目的に厳密に合わせているため、変更時は子プロセス側の get_handles_from_parent との整合性に注意すること。
