assign_headers

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:319

目的

HTTP/2 リクエスト (h2_request) から Apache のネイティブなリクエスト構造体 (request_rec) へヘッダー情報を転送し、HTTP/2 特有の擬似ヘッダー（:authority）を標準的な Host ヘッダーへ変換・統合する。

シグネチャまたは形状

static void assign_headers(request_rec *r, const h2_request *req,
                           int no_body, int is_connect)

入力と出力

  • **入力**:
  • r: 構築対象の Apache リクエスト構造体。
  • req: 変換元の HTTP/2 リクエスト表現。
  • no_body: ボディが存在しないことを示すフラグ。
  • is_connect: CONNECT メソッドであるかを示すフラグ。
  • **出力**: なし（r->headers_in および r->subprocess_env を直接更新）。

前提条件と不変条件

  • r->pool が有効であること。
  • req->headers が apr_table_t として初期化されていること。
  • is_connect が真の場合、Host ヘッダーの補完処理はスキップされる。

アルゴリズムまたは振る舞い

1. **ヘッダーの複製**: req->headers を r->pool 上に複製し、r->headers_in に割り当てる。 2. **:authority の処理**:

  • is_connect が偽の場合、:authority を Host ヘッダーとして設定する。
  • 既存の Host ヘッダーと :authority が異なる場合、元の Host を H2_ORIGINAL_HOST 環境変数に退避させ、:authority の値で上書きする。

    • 3. **コンテンツ長ガード**:

  • no_body が真かつ Content-Type が存在する場合、Content-Length を "0" に設定する。
  • is_connect が偽かつ Content-Length が未指定の場合、Transfer-Encoding: chunked を追加する（AP_HAS_RESPONSE_BUCKETS が定義されていない環境のみ）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_request.c:387
  • modules/http2/h2_request.c:484
  • **呼び出し先**:
  • apr_table_clone, apr_table_get, apr_table_setn, apr_table_mergen (APR テーブル操作)
  • ap_log_rerror (ログ出力)
  • strcmp (文字列比較)

関連データ構造

  • request_rec: Apache のリクエスト処理用構造体。
  • h2_request: HTTP/2 プロトコル層のリクエスト表現。
  • apr_table_t: ヘッダー保持に使用される APR のキーバリューテーブル。

失敗モードと境界条件

  • :authority と Host の不一致時はログ出力のみ行い、処理を継続する（RFC 9113 に基づく厳格な拒否は行わない）。
  • Content-Length が欠落している場合、HTTP/1.1 互換のために chunked エンコーディングを強制する設計となっている。

保守担当者向けメモ

  • 本関数は h2_create_request_rec の一部として呼び出され、HTTP/2 から Apache へのブリッジングの要となる。
  • AP_HAS_RESPONSE_BUCKETS の有無によって Transfer-Encoding の付与ロジックが分岐するため、ビルド環境の差異に注意すること。
