ftp_check_string

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:181

目的

FTP プロトコルで送信される文字列（ユーザー名、パスワード、パスなど）が、制御文字や不正なバイナリデータを含んでいないかを検証する。

シグネチャまたは形状

static int ftp_check_string(const char *x)

入力と出力

  • **入力**: const char *x （検証対象の文字列）
  • **出力**: int （有効な場合は 1、不正な文字が含まれる場合は 0 を返す）

前提条件と不変条件

  • 入力文字列 x はヌル終端されている必要がある。
  • APR_CHARSET_EBCDIC の定義有無により、文字コード変換の挙動が異なる。

アルゴリズムまたは振る舞い

1. 文字列を順次走査する。 2. % で始まる 3 文字のシーケンス（例: %20）を検出した場合、ap_proxy_hex2c を使用してデコードし、その値を検証対象とする。 3. 非 EBCDIC 環境では、CR (\015)、LF (\012)、または 8 ビット目が立っている文字（ch & 0x80）が含まれている場合に 0 を返す。 4. EBCDIC 環境では、CR (\r)、LF (\n) をチェックし、さらに ap_xlate_proto_to_ascii で ASCII に変換した後の文字が 8 ビット目を持っている場合に 0 を返す。 5. 全ての文字が条件を通過すれば 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_ftp.c:310 (ユーザー名の検証)
  • modules/proxy/mod_proxy_ftp.c:312 (パスワードの検証)
  • modules/proxy/mod_proxy_ftp.c:336 (パスの検証)
  • modules/proxy/mod_proxy_ftp.c:1088 (パスワードの検証)
  • **呼び出し先**:
  • ap_proxy_hex2c (URL エンコードのデコード)
  • apr_isxdigit (16 進数判定)
  • ap_xlate_proto_to_ascii (EBCDIC 環境時のみ)

関連データ構造

  • なし

失敗モードと境界条件

  • 文字列内に CR (\r) や LF (\n) が含まれる場合、FTP プロトコルインジェクションを防ぐため 0 を返す。
  • 8 ビット目がセットされた文字（非 ASCII 文字）が含まれる場合、0 を返す。
  • URL エンコードされたシーケンスが不完全な場合（% の後に 16 進数以外の文字が続く場合）、デコードは行われず、そのままの文字として検証される。

保守担当者向けメモ

  • 本関数は FTP コマンドの安全性を確保するための重要なバリデーターである。
  • ftp_string_read と同様、歴史的な経緯により独自の文字チェックロジックを含んでいるため、変更時はプロトコル仕様への影響を考慮すること。
