ap_lua_server_spec

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_vmprep.h:113

目的

サーバー単位で管理される Lua 仮想マシン（VM）の状態と、関連するファイル情報を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    lua_State* L;
    ap_lua_finfo* finfo;
} ap_lua_server_spec;

入力と出力

  • L: 実行中の Lua 仮想マシンの状態（lua_State*）。
  • finfo: スクリプトのファイル情報（ap_lua_finfo*）。

前提条件と不変条件

  • 本構造体は mod_lua におけるサーバーレベルの Lua 実行環境を表現します。
  • L は有効な lua_State を指している必要があります。

アルゴリズムまたは振る舞い

  • mod_lua の初期化時やリクエスト処理時に、サーバーコンテキストに関連付けられた Lua VM を保持するために使用されます。
  • lua_touserdata を介して Lua スタックから取得されるケースがあります。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • modules/lua/lua_vmprep.c: resource からのキャスト、および apr_pcalloc によるメモリ確保。
  • modules/lua/mod_lua.c: lua_touserdata を用いた Lua スタックからの取得。
  • 呼び出し先:
  • 情報なし。

関連データ構造

  • lua_State: Lua 言語のコア構造体。
  • ap_lua_finfo: スクリプトのファイル属性を保持する構造体。

失敗モードと境界条件

  • apr_pcalloc によるメモリ確保失敗時は、呼び出し元で適切なエラーハンドリングが必要です。
  • lua_touserdata で取得する際、スタック上のデータが本構造体と一致しない場合、不正なポインタ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は lua_vmprep.c および mod_lua.c で利用されています。
  • サーバー単位のスコープで VM を管理する際の基盤となるため、変更時は ap_lua_get_lua_state 等の VM 管理ロジックへの影響を確認してください。
