connection

定義

  • 種別: struct
  • 定義箇所: support/ab.c:262

目的

ApacheBench (ab) において、個別の HTTP 接続の状態、ソケット情報、SSL コンテキスト、および統計計測用のタイムスタンプを保持するために使用される構造体です。

シグネチャまたは形状

struct connection {
    apr_pool_t *ctx;
    apr_socket_t *aprsock;
    apr_pollfd_t pollfd;
    int state;
    apr_size_t read;
    apr_size_t bread;
    apr_size_t rwrite, rwrote;
    apr_size_t length;
    char cbuff[CBUFFSIZE];
    int cbx;
    int keepalive;
    int gotheader;
    apr_time_t start, connect, endwrite, beginread, done;
    int socknum;
#ifdef USE_SSL
    SSL *ssl;
#endif
};

入力と出力

  • **入力**: ab の実行時に指定された同時接続数に基づき、test() 関数内で動的に配列として確保されます。
  • **出力**: 接続のライフサイクルを通じて、ソケットの読み書き状態や統計データが更新されます。

前提条件と不変条件

  • ctx (APR プール) は、接続のライフサイクル全体でメモリ管理に使用されます。
  • cbuff はサーバーからのレスポンスヘッダーを格納する固定長バッファです。
  • USE_SSL が定義されている場合、ssl メンバが OpenSSL のセッション管理に使用されます。

アルゴリズムまたは振る舞い

  • 非同期 I/O モデルにおいて、apr_pollset_t を介してイベント駆動で処理されます。
  • state メンバにより、接続が UNCONNECTED から READ までのどの段階にあるかを追跡します。
  • rwrite および rwrote メンバは、EAGAIN が発生した際のリクエスト送信の進捗を保持します。
  • 各タイムスタンプ（start, connect, endwrite, beginread, done）は、接続の各フェーズの計測に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: support/ab.c 内の test()、start_connect()、write_request()、read_connection() 等から参照されます。
  • **呼び出し先**: APR (Apache Portable Runtime) のソケットおよびポーリング API を呼び出します。
  • **注意**: include/ 配下のヘッダーファイルに見られる connection 構造体への言及は、Apache HTTP Server 本体の接続管理に関するものであり、本構造体とは別物です。

関連データ構造

  • connect_state_e: 接続のライフサイクル状態を定義する列挙型。
  • data: 個別のリクエストごとの時間計測データを保持する構造体。

失敗モードと境界条件

  • CBUFFSIZE を超えるヘッダー受信時の挙動は、バッファオーバーフローを防ぐための実装に依存します。
  • apr_socket_t の操作において、ネットワークエラーやタイムアウトが発生した場合、state の遷移と close_connection() によるクリーンアップが必要です。

保守担当者向けメモ

  • 本構造体は ab 専用であり、サーバー本体の接続管理構造体と混同しないよう注意してください。
  • メモリは ctx プールに紐付いているため、接続終了時には適切にプールを管理する必要があります。
  • SSL 関連のコードは OpenSSL のバージョン依存が強いため、変更時はビルド環境の OpenSSL バージョンを確認してください。
