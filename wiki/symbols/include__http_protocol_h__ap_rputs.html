ap_rputs

定義

  • 種別: function
  • 定義箇所: include/http_protocol.h:487

目的

クライアントへのレスポンスとして、指定されたヌル終端文字列をフィルタスタック経由で送信します。

シグネチャまたは形状

static APR_INLINE int ap_rputs(const char *str, request_rec *r)

入力と出力

  • **入力**:
  • str: 送信するヌル終端文字列。
  • r: 現在のリクエストを保持する request_rec 構造体。
  • **出力**:
  • 成功時は送信されたバイト数または 0 以上のステータスコードを返します。
  • 失敗時は負の値を返します。

前提条件と不変条件

  • str は有効なヌル終端文字列である必要があります。
  • r は有効なリクエストコンテキストを指している必要があります。
  • 本関数はインライン関数として定義されており、マクロとして実装される可能性もあります。

アルゴリズムまたは振る舞い

1. strlen(str) を用いて文字列の長さを取得します。 2. 文字列の長さが INT_MAX 以下の場合、ap_rwrite を呼び出して全データを送信します。 3. 文字列の長さが INT_MAX を超える場合、ループ内で INT_MAX ずつ分割して ap_rwrite を繰り返し呼び出します。 4. ap_rwrite の呼び出しで負の値（エラー）が返された場合、即座にその値を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し先**: ap_rwrite
  • **呼び出し元**:
  • docs/manual/developer/mod_example_2.c
  • modules/cache/mod_cache_socache.c
  • その他、Apache モジュール開発においてレスポンスボディを生成する各モジュールから利用されます。

関連データ構造

  • request_rec: リクエスト処理のライフサイクルを管理する主要構造体。

失敗モードと境界条件

  • ap_rwrite が失敗した場合、そのエラーコードを伝播します。
  • INT_MAX を超える文字列を渡した場合、内部ループにより分割送信されます。

保守担当者向けメモ

  • 本関数は APR_INLINE で定義されており、パフォーマンスを考慮して設計されています。
  • フィルタチェーンを通じたデータ出力の標準的な手段です。
  • 文字列の長さが INT_MAX を超えるケースを考慮した実装となっています。
