dbdlogin_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbd.c:318

目的

Apache HTTP Server の認可プロバイダーとして、データベース（DBD）を用いたログイン処理を実行し、アクセス権限の判定を行う。

シグネチャまたは形状

static authz_status dbdlogin_check_authorization(request_rec *r,
                                                 const char *require_args,
                                                 const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • require_args: 設定ファイルで指定された引数（未使用）。
  • parsed_require_args: 解析済みの引数（未使用）。
  • **出力**:
  • AUTHZ_GRANTED: ログイン処理が正常に完了した場合。
  • AUTHZ_DENIED: ログイン処理が失敗した場合。
  • AUTHZ_DENIED_NO_USER: リクエストにユーザー名が含まれていない場合。

前提条件と不変条件

  • r->user が設定されている必要がある。
  • mod_dbd が有効であり、データベース接続が利用可能であること。
  • authz_dbd_module の設定がリクエストのディレクトリコンテキストに存在すること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリに対する authz_dbd_cfg を取得する。 2. r->user が存在するか確認し、存在しない場合は AUTHZ_DENIED_NO_USER を返す。 3. authz_dbd_login 関数を呼び出し、引数 "login" を渡して SQL ログインクエリを実行する。 4. authz_dbd_login の戻り値が OK であれば AUTHZ_GRANTED を返し、それ以外であれば AUTHZ_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbd.c:374 にて、認可プロバイダーの登録構造体から参照されている。
  • **呼び出し先**: authz_dbd_login（共通のログインロジックを実行）。

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • authz_dbd_cfg: mod_authz_dbd のディレクトリ単位の設定構造体。
  • authz_status: 認可結果を示す列挙型。

失敗モードと境界条件

  • ユーザー名が未設定の場合、即座に AUTHZ_DENIED_NO_USER を返す。
  • データベース接続エラーや SQL クエリの実行失敗は authz_dbd_login 内部で処理され、結果として AUTHZ_DENIED が返される。

保守担当者向けメモ

  • 本関数は Require dbd-login ディレクティブに対応するプロバイダー関数である。
  • ログイン成功時に client_login フックがトリガーされるため、セッション管理を行う他モジュールとの連携に依存している点に注意すること。
