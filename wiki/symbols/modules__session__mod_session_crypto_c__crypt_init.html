crypt_init

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:189

目的

指定された暗号化ドライバがサポートする暗号アルゴリズム（cipher）のリストから、設定されたアルゴリズムを検索し、検証する。

シグネチャまたは形状

static apr_status_t crypt_init(request_rec *r,
        const apr_crypto_t *f, apr_crypto_block_key_type_e **cipher,
        session_crypto_dir_conf * dconf)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • f: 初期化済みの apr_crypto_t コンテキスト。
  • dconf: 設定情報（使用する暗号アルゴリズム名 dconf->cipher を保持）。
  • **出力**:
  • cipher: 検索に成功した場合、対応する apr_crypto_block_key_type_e 構造体へのポインタが格納される。
  • 戻り値: 成功時は APR_SUCCESS、失敗時はエラーコード（APR_EGENERAL 等）。

前提条件と不変条件

  • f は有効な apr_crypto_t コンテキストであること。
  • dconf->cipher には、ドライバがサポートするアルゴリズム名が指定されていること。

アルゴリズムまたは振る舞い

1. apr_crypto_get_block_key_types を呼び出し、ドライバがサポートする全アルゴリズムのハッシュマップを取得する。 2. dconf->cipher をキーとしてハッシュマップからアルゴリズムを検索する。 3. 見つかった場合は *cipher に設定し、APR_SUCCESS を返す。 4. 見つからない場合は、サポートされている全アルゴリズム名をカンマ区切りの文字列として生成し、エラーログに詳細を出力した上で APR_EGENERAL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session_crypto.c:266
  • modules/session/mod_session_crypto.c:372
  • **呼び出し先**:
  • apr_crypto_get_block_key_types
  • apr_hash_get
  • ap_log_rerror
  • apr_hash_first / apr_hash_next / apr_hash_this
  • apr_palloc

関連データ構造

  • request_rec: リクエスト処理用構造体。
  • apr_crypto_t: APR 暗号化コンテキスト。
  • session_crypto_dir_conf: ディレクトリ単位の暗号化設定。
  • apr_crypto_block_key_type_e: 暗号化アルゴリズムの型定義。

失敗モードと境界条件

  • **ドライバの取得失敗**: apr_crypto_get_block_key_types が失敗した場合、エラーログを出力して即座に終了する。
  • **アルゴリズム未対応**: 指定された dconf->cipher がドライバでサポートされていない場合、利用可能なアルゴリズム一覧をログに含めてエラーを返す。

保守担当者向けメモ

  • 本関数は apr_pool_t を使用してメモリを割り当てるため、リクエスト終了時に自動的に解放される。
  • エラーログ出力時には APLOGNO マクロを使用しており、ログの追跡性を確保している。
