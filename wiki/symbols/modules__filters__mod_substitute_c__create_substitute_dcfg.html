create_substitute_dcfg

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_substitute.c:72

目的

mod_substitute モジュールのディレクトリ単位の設定構造体 (subst_dir_conf) を初期化し、メモリプール上に割り当てる。

シグネチャまたは形状

static void *create_substitute_dcfg(apr_pool_t *p, char *d)

入力と出力

  • **入力**:
  • p: 設定値を割り当てるための apr_pool_t。
  • d: ディレクトリパス（本関数内では使用されない）。
  • **出力**:
  • 初期化済みの subst_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • apr_palloc により、指定されたプール p の生存期間に依存するメモリが確保される。
  • patterns 配列は初期容量 10 で作成される。
  • max_line_length はデフォルト値 AP_SUBST_MAX_LINE_LENGTH で初期化される。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して subst_dir_conf サイズのメモリを確保する。 2. apr_array_make を呼び出し、置換パターンを格納する配列を初期化する。 3. max_line_length をデフォルト値に設定し、max_line_length_set フラグを 0（未設定）に初期化する。 4. inherit_before を -1 に設定する。 5. 初期化された構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_substitute.c:760（Apache の設定読み込みプロセスから呼び出されるディレクトリ設定作成関数として参照されている）。
  • **呼び出し先**: apr_palloc, apr_array_make（APR ライブラリ関数）。

関連データ構造

  • subst_dir_conf: 本関数が初期化する設定構造体。
  • subst_pattern_t: patterns 配列の要素として格納される型。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。設定のデフォルト値を変更する場合は、ここを修正すること。
