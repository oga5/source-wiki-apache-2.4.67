collect_ctx

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.c:144

目的

ハッシュテーブルから要素を収集し、指定されたバッファへ格納するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    h2_proxy_ihash_t *ih;
    void **buffer;
    size_t max;
    size_t len;
} collect_ctx;

入力と出力

  • **入力**:
  • ih: 収集対象のハッシュテーブル (h2_proxy_ihash_t)。
  • buffer: 収集した要素を格納するポインタ配列。
  • max: buffer が保持可能な最大要素数。
  • **出力**:
  • len: 実際に収集された要素数。

前提条件と不変条件

  • buffer は少なくとも max 個の void* を格納できる領域を確保している必要があります。
  • ih は初期化済みの有効なハッシュテーブルである必要があります。

アルゴリズムまたは振る舞い

ハッシュテーブル内の要素を走査し、buffer に順次格納します。len は収集された要素のカウントを保持します。この構造体は、ハッシュテーブルの内容を配列形式へ変換する際のイテレータ的な役割を果たします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_util.c 内の関数（直接の関数名はソース抜粋に記載なし）。
  • modules/http2/h2_util.c 内の関数（直接の関数名はソース抜粋に記載なし）。
  • **呼び出し先**:
  • ソース抜粋からは特定の関数呼び出しは確認できません。

関連データ構造

  • h2_proxy_ihash_t: 本構造体が操作対象とするハッシュテーブル型。

失敗モードと境界条件

  • max を超える要素がハッシュテーブルに存在する場合、バッファオーバーフローを防ぐための境界チェックが必要です（実装側のロジックに依存）。

保守担当者向けメモ

  • 本構造体は h2_proxy_util.c と h2_util.c の両方で定義・利用されています。変更を加える際は、両方のファイル間での整合性に注意してください。
