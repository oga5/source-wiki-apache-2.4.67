xml2enc_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_xml2enc.c:102

目的

XML/HTML コンテンツの文字エンコーディング変換を行うフィルタを初期化し、Apache の出力フィルタチェーンに登録する。

シグネチャまたは形状

apr_status_t xml2enc_filter(request_rec* r, const char* enc, unsigned int mode)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • enc: 変換対象または変換先の文字エンコーディング名。
  • mode: 処理モード（ENCIO_OUTPUT, ENCIO_INPUT, ENCIO_INPUT_CHECKS のいずれか）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • mode が ENCIO_OUTPUT または ENCIO_INPUT の場合、apr_xlate_open による変換器の初期化が必須。
  • mode が ENCIO_INPUT_CHECKS の場合、変換器は初期化されず、後続の「スニッフィング（自動検出）」処理で初期化される。
  • 変換器のメモリ確保およびフィルタコンテキストの割り当てには、r->pool が使用される。

アルゴリズムまたは振る舞い

1. **モード判定**: mode に基づき、apr_xlate_open を用いて UTF-8 と enc 間の変換器を準備する。 2. **エラーハンドリング**: 不正なモードが指定された場合、または変換器のオープンに失敗した場合はエラーログを出力し、処理を中断する。 3. **コンテキスト生成**: xml2ctx 構造体を確保し、フラグと変換器を設定する。 4. **フィルタ登録**: ap_add_output_filter を呼び出し、xml2enc フィルタを現在の接続の出力チェーンに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_proxy_html.c が APR_RETRIEVE_OPTIONAL_FN を介して本関数を呼び出す。
  • **呼び出し先**:
  • apr_xlate_open: 文字コード変換器の作成。
  • apr_pcalloc / apr_palloc: メモリ確保。
  • ap_log_rerror: エラーログ出力。
  • ap_add_output_filter: フィルタの登録。

関連データ構造

  • xml2ctx: フィルタの実行状態（変換器、バッファ、フラグ）を保持する構造体。
  • request_rec: リクエストごとのメモリプール（r->pool）を提供。

失敗モードと境界条件

  • **サポート外のエンコーディング**: apr_xlate_open が失敗した場合、APR_EINVAL 等が返され、フィルタは登録されない。
  • **不正なモード**: 定義されていない mode が渡された場合、APR_EGENERAL を返しログを出力する。

保守担当者向けメモ

  • 本関数は APR_DECLARE_OPTIONAL_FN により公開されており、他のモジュール（mod_proxy_html 等）から動的に利用される。
  • libxml2 のエンコーディング検出ロジックに依存しているため、変換器の初期化タイミングがモードによって異なる点に注意すること。
