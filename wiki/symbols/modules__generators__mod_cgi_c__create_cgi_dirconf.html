create_cgi_dirconf

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgi.c:120

目的

ディレクトリ単位のCGI設定構造体 (cgi_dirconf) をメモリ上に割り当て、初期化するために使用されます。

シグネチャまたは形状

static void *create_cgi_dirconf(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • dummy: 使用されない引数（設定ディレクティブのコンテキスト用）。
  • **出力**:
  • ゼロクリアされた cgi_dirconf 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc を使用しているため、返される構造体はすべてのメンバがゼロ（またはNULL）で初期化されています。
  • p はリクエストプールまたは設定プールであり、この関数が返すメモリはプールの生存期間に依存します。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、cgi_dirconf 構造体のサイズ分のメモリをプール p から確保します。 2. 確保したメモリ領域をゼロクリアした状態で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_cgi.c:774 （ディレクトリ設定作成用ハンドラとして登録）。
  • **呼び出し先**: apr_pcalloc (APRライブラリ)。

関連データ構造

  • cgi_dirconf: ディレクトリ単位の設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc の挙動に従い、Apache のメモリ管理ポリシーに基づいてプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される標準的なコンストラクタです。
  • dummy 引数は Apache の設定 API のシグネチャに合わせるために存在しており、本関数内では使用されていません。
