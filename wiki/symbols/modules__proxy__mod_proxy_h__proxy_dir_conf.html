proxy_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:214

目的

Apache HTTP Server のプロキシサブシステムにおいて、ディレクトリ単位（<Location> 等）のプロキシ設定を保持するための構造体です。パス変換ルールやリクエスト処理の挙動を制御します。

シグネチャまたは形状

typedef struct {
    const char *p;
    ap_regex_t  *r;
    apr_array_header_t *raliases;
    apr_array_header_t* cookie_paths;
    apr_array_header_t* cookie_domains;
    signed char p_is_fnmatch;
    signed char interpolate_env;
    struct proxy_alias *alias;
    unsigned int error_override:1;
    unsigned int preserve_host:1;
    unsigned int preserve_host_set:1;
    unsigned int error_override_set:1;
    unsigned int alias_set:1;
    unsigned int add_forwarded_headers:1;
    unsigned int add_forwarded_headers_set:1;
    apr_array_header_t *refs;
    unsigned int forward_100_continue:1;
    unsigned int forward_100_continue_set:1;
    apr_array_header_t *error_override_codes;
} proxy_dir_conf;

入力と出力

  • 入力: 設定ファイル（httpd.conf 等）で指定されたプロキシ関連ディレクティブ。
  • 出力: プロキシモジュールがリクエストを処理する際、当該ディレクトリ設定に基づく挙動（エラーページのオーバーライドやヘッダーの転送制御など）を決定するために参照されます。

前提条件と不変条件

  • 設定構造体は Apache の設定プール（pool）によってライフサイクルが管理されます。
  • error_override 等のビットフィールドは、対応する _set フラグによって設定の有無が管理されます。

アルゴリズムまたは振る舞い

  • error_override が有効な場合、バックエンドから返されたエラーページをマスクし、メインサーバー側で生成したエラーページをクライアントに返します。
  • preserve_host が有効な場合、バックエンドへのリクエスト時に元の Host ヘッダーを保持します。
  • forward_100_continue は、100-continue リクエストのバックエンドへの転送を制御します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_proxy_session.c および modules/http2/h2_proxy_util.c 等で、設定取得のために参照されていることが確認されています。
  • 呼び出し先: 構造体内部で apr_array_header_t や ap_regex_t などの APR/Apache Core のデータ構造を利用しています。

関連データ構造

  • proxy_alias: パス変換ルールを保持。
  • apr_array_header_t: エイリアス、クッキーパス、ドメイン、エラーコードのリスト管理に使用。

失敗モードと境界条件

  • ProxyPassReverse 等のディレクティブが複数の <Location> で定義された場合、サーバー設定（server_conf）からは適切に参照できないという既知の制約がソース内のコメントで指摘されています。

保守担当者向けメモ

  • 本構造体はバイナリ互換性に影響を与えるため、メンバーの追加や変更時には ap_mmn.h で定義されるモジュールマジックナンバーの更新を検討してください。
  • ftp_directory_charset 等の過去のメンバーについては、ap_mmn.h の履歴を参照してください。
