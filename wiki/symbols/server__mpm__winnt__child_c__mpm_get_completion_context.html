mpm_get_completion_context

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/child.c:162

目的

Windows 版 Apache HTTP Server において、接続処理に使用する winnt_conn_ctx_t（接続コンテキスト）を再利用キューから取得、または新規作成して提供します。

シグネチャまたは形状

winnt_conn_ctx_t *mpm_get_completion_context(int *timeout)

入力と出力

  • **入力**: int *timeout（タイムアウト発生時に 1 が設定されるフラグへのポインタ）
  • **出力**: 成功時は winnt_conn_ctx_t 構造体へのポインタ、失敗時は NULL を返却。

前提条件と不変条件

  • qlock ミューテックスによる再利用キューの排他制御が行われます。
  • num_completion_contexts が max_num_completion_contexts を超えない範囲で新規割り当てが行われます。
  • 新規作成時は pchild プールからメモリが確保されます。

アルゴリズムまたは振る舞い

1. **キューの確認**: qlock を取得し、再利用キュー (qhead) からコンテキストの取り出しを試みます。 2. **待機処理**: キューが空で、かつ最大コンテキスト数に達している場合、qwait_event を用いてワーカーがコンテキストを解放するまで最大 1 秒間待機します。 3. **新規作成**: 上限に達していない場合、pchild プールからメモリを確保し、OVERLAPPED 用のイベントオブジェクト作成、トランザクション用メモリプールの生成、バケットアロケータの初期化を行います。 4. **終了**: 取得したコンテキストを呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/child.c:438 (worker_main 内)
  • **呼び出し先**: apr_thread_mutex_lock/unlock, ResetEvent, WaitForSingleObject, apr_pcalloc, CreateEvent, apr_pool_create_ex, apr_bucket_alloc_create, apr_atomic_inc32 等の APR/Windows API。

関連データ構造

  • winnt_conn_ctx_t: 接続状態を保持する主要構造体。
  • qhead, qtail: 再利用コンテキストのキュー。
  • qwait_event: コンテキストの空きを待機するためのイベントハンドル。

失敗モードと境界条件

  • **リソース枯渇**: max_num_completion_contexts に達し、かつ 1 秒以内にコンテキストが解放されない場合、timeout を 1 に設定して NULL を返します。
  • **システムエラー**: CreateEvent や apr_pool_create_ex が失敗した場合、エラーログを出力し NULL を返します。
  • **メモリリーク**: ソース内のコメントによれば、エラーパスにおいて一部のメモリが pchild プールからリークする可能性があります。

保守担当者向けメモ

  • 本関数は winnt_conn_ctx_t のライフサイクル管理の起点です。
  • num_completion_contexts のアトミック操作によりスレッド間の整合性を保っています。
  • エラーパスにおけるメモリリークは、現在の実装では許容されています。
