h2_c1_can_upgrade

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c1.c:232

目的

現在のリクエスト接続において、HTTP/1.1 から HTTP/2 へのプロトコルアップグレードが許可されているかを判定します。

シグネチャまたは形状

int h2_c1_can_upgrade(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエストレコード)
  • **出力**: int (1 の場合アップグレード可能、0 の場合不可)

前提条件と不変条件

  • r->connection が有効である必要があります。
  • マスター接続（r->connection->master）ではない接続に対してのみ判定が行われます。

アルゴリズムまたは振る舞い

1. リクエストの接続がマスター接続である場合、直ちに 0 を返します。 2. マスター接続ではない場合、設定値 H2_CONF_UPGRADE を取得します。

  • 設定値が正（> 0）であれば、アップグレードを許可（1）します。
  • 設定値が負（< 0）かつ SSL 接続ではない場合に限り、アップグレードを許可（1）します。
  • それ以外の場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_switch.c:86 にて呼び出されていることが確認されています。
  • **呼び出し先**:
  • h2_config_rgeti (設定取得)
  • ap_ssl_conn_is_ssl (SSL 接続判定)

関連データ構造

  • request_rec: Apache のリクエスト情報構造体。
  • conn_rec: 接続情報構造体。

失敗モードと境界条件

  • SSL 接続下では、設定値が負（デフォルト値等）の場合、アップグレードは許可されません。
  • マスター接続（HTTP/2 のサブストリーム等）では、常にアップグレード不可となります。

保守担当者向けメモ

  • 本関数は h2_c1.h で宣言されており、プロトコル切り替えの判断ロジックとして h2_switch.c から参照されています。
  • SSL 接続時の挙動は ap_ssl_conn_is_ssl の結果に依存するため、SSL モジュールのロード状態に注意してください。
