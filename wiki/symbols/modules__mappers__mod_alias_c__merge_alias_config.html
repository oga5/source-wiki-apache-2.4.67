merge_alias_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_alias.c:97

目的

サーバー設定レベルにおける alias_server_conf 構造体のマージを行います。親コンテキストの設定と現在の設定を統合し、新しい設定構造体を生成します。

シグネチャまたは形状

static void *merge_alias_config(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親コンテキストの設定（alias_server_conf 型）。
  • overridesv: 現在のコンテキストの設定（alias_server_conf 型）。
  • **出力**:
  • マージされた新しい alias_server_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は alias_server_conf 型としてキャスト可能である必要があります。
  • apr_pcalloc により、マージ結果を格納するためのメモリが p プールから確保されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい alias_server_conf 構造体をメモリ上に確保します。 2. basev と overridesv をそれぞれ alias_server_conf 型にキャストします。 3. apr_array_append を使用して、overrides（優先）と base（継承）の aliases 配列を結合します。 4. 同様に、redirects 配列についても overrides と base を結合します。 5. 結合された新しい構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_alias.c:763（サーバー設定のマージ処理において呼び出されます）。
  • **呼び出し先**: apr_pcalloc および apr_array_append（APR ライブラリ関数）。

関連データ構造

  • alias_server_conf: サーバー全体の設定を保持する構造体。aliases および redirects の配列を含みます。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が異常終了する可能性があります（APR の標準的な挙動）。
  • basev または overridesv が NULL の場合、キャスト後のアクセスでセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数はサーバーレベルの設定マージ専用です。ディレクトリ単位の設定マージには merge_alias_dir_config が別途存在します。
  • apr_array_append は新しい配列を生成するため、設定の数が多い場合、メモリ消費量に注意してください。
