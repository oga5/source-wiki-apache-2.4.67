dav_get_search_hooks

定義

  • 種別: function
  • 定義箇所: modules/dav/main/mod_dav.c:248

目的

現在のアクティブな WebDAV プロバイダーから、検索（Search）機能に関連するフックセットを取得します。

シグネチャまたは形状

const dav_hooks_search *dav_get_search_hooks(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在の Apache リクエスト構造体）
  • **出力**: const dav_hooks_search * （検索機能を提供するフック構造体へのポインタ）

前提条件と不変条件

  • dav_get_provider(r) が有効なプロバイダーを返す必要があります。
  • 返されるフック構造体は、プロバイダーが検索機能をサポートしている場合に有効です。

アルゴリズムまたは振る舞い

1. dav_get_provider(r) を呼び出し、現在のリクエストに関連付けられたプロバイダーを取得します。 2. 取得したプロバイダー構造体内の search メンバを返します。

呼び出し元と呼び出し先

  • **呼び出し先**: dav_get_provider
  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供されていませんが、mod_dav.h にて DAV_GET_HOOKS_SEARCH(r) マクロとして定義されており、検索機能を利用する各メソッドから参照されることが想定されます。

関連データ構造

  • dav_hooks_search: 検索機能のフックを定義する構造体。
  • dav_provider: プロバイダーの機能を保持する構造体。

失敗モードと境界条件

  • dav_get_provider(r) が NULL を返す場合、本関数はセグメンテーション違反を引き起こす可能性があります。
  • プロバイダーが検索機能を実装していない場合、search メンバは NULL となる可能性があります。

保守担当者向けメモ

  • 本関数は mod_dav.h で定義された DAV_GET_HOOKS_SEARCH マクロ経由での利用が推奨されます。
  • 検索機能の拡張や変更を行う際は、dav_hooks_search 構造体の定義と整合性を取る必要があります。
