ssl_util_vhost_matches

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util.c:61

目的

指定されたサーバー名が、対象の仮想ホスト設定（server_rec）における ServerName、ServerAlias、またはワイルドカード付きの ServerAlias のいずれかと一致するかを判定します。

シグネチャまたは形状

BOOL ssl_util_vhost_matches(const char *servername, server_rec *s)

入力と出力

  • **入力**:
  • servername: 照合対象のサーバー名（文字列）。
  • s: 照合先の仮想ホスト設定を保持する server_rec 構造体。
  • **出力**:
  • 一致する場合は TRUE、一致しない場合は FALSE を返します。

前提条件と不変条件

  • s は有効な server_rec 構造体へのポインタである必要があります。
  • servername は NULL であってはなりません。

アルゴリズムまたは振る舞い

1. **ServerName の照合**: servername と s->server_hostname を大文字小文字を区別せずに比較します。一致すれば即座に TRUE を返します。 2. **ServerAlias の照合**: s->names に格納されたエイリアスリストを走査し、各要素と servername を大文字小文字を区別せずに比較します。一致すれば TRUE を返します。 3. **ワイルドカード付き ServerAlias の照合**: s->wild_names に格納されたエイリアスリストを走査し、ap_strcasecmp_match を用いてワイルドカードパターンとの照合を行います。一致すれば TRUE を返します。 4. 上記のいずれにも該当しない場合、FALSE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_kernel.c:2343
  • **呼び出し先**:
  • strcasecmp (標準ライブラリ)
  • ap_strcasecmp_match (Apache Core API)

関連データ構造

  • server_rec: Apache の仮想ホスト設定を保持する構造体。names（エイリアス）および wild_names（ワイルドカード付きエイリアス）のメンバを参照します。

失敗モードと境界条件

  • s->names または s->wild_names が NULL の場合、そのリストの走査はスキップされます。
  • リスト内の各要素が NULL の場合はスキップされます。
  • servername が空文字列の場合、比較ロジックに従い一致しないと判定されます。

保守担当者向けメモ

  • 本関数は ssl_private.h で宣言されており、モジュール内で広く利用可能です。
  • ワイルドカード照合には ap_strcasecmp_match を使用しているため、Apache の標準的なパターンマッチング規則に従います。
