dav_lock_parse_lockinfo

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:170

目的

クライアントから送信された LOCK リクエストの XML ボディを解析し、dav_lock 構造体を生成・初期化します。

シグネチャまたは形状

dav_error *dav_lock_parse_lockinfo(request_rec *r,
                                   const dav_resource *resource,
                                   dav_lockdb *lockdb,
                                   const apr_xml_doc *doc,
                                   dav_lock **lock_request)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • resource: ロック対象のリソース。
  • lockdb: ロックデータベースのハンドル。
  • doc: 解析済みの XML ドキュメント。
  • **出力**:
  • lock_request: 生成された dav_lock 構造体へのポインタ。
  • 戻り値: 成功時は NULL、失敗時はエラー詳細を含む dav_error 構造体。

前提条件と不変条件

  • doc のルート要素が lockinfo である必要があります。
  • lockdb->hooks->create_lock が正常にロック構造体を割り当て可能である必要があります。
  • Depth ヘッダーが有効な値（0 または infinity）である必要があります。

アルゴリズムまたは振る舞い

1. dav_validate_root を使用して XML ルートが lockinfo であることを確認します。 2. lockdb->hooks->create_lock を呼び出し、初期化された dav_lock 構造体を取得します。 3. dav_get_depth および dav_get_timeout を呼び出し、リクエストヘッダーから深さとタイムアウト情報を取得・設定します。 4. XML ボディ内の子要素を走査します:

  • locktype: write であることを確認し設定します。
  • lockscope: exclusive または shared を判定し設定します。
  • owner: XML 要素をテキスト形式に変換して lock->owner に格納します。

    • 5. 未知の XML 要素が含まれている場合は HTTP_PRECONDITION_FAILED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/mod_dav.c:3244
  • **呼び出し先**:
  • lockdb->hooks->create_lock (ロックデータベースプロバイダー)
  • dav_validate_root, dav_get_depth, dav_get_timeout (DAV ユーティリティ)
  • apr_xml_quote_elem, apr_xml_to_text (APR XML ユーティリティ)

関連データ構造

  • dav_lock: ロック情報を保持する主要構造体。
  • dav_lockdb: ロックデータベースへのインターフェース。
  • apr_xml_doc: 解析された XML ドキュメント構造。

失敗モードと境界条件

  • XML ルートが lockinfo でない場合: HTTP_BAD_REQUEST。
  • ロック構造体の作成失敗: dav_push_error による内部エラー。
  • Depth ヘッダーが不正な場合: HTTP_BAD_REQUEST。
  • 未知の XML 要素が含まれる場合: HTTP_PRECONDITION_FAILED。

保守担当者向けメモ

  • owner 要素の解析において APR_XML_X2T_FULL_NS_LANG を使用しており、名前空間定義を保持する設計となっています。
  • 本関数はリクエストプール (r->pool) を使用してメモリを割り当てるため、リクエスト終了時に自動的に解放されます。
