merge_authn_alias_svr_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_core.c:184

目的

認証プロバイダーエイリアスのサーバー設定をマージします。

シグネチャまたは形状

static void *merge_authn_alias_svr_config(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • p: メモリ割り当てに使用するプール。
  • basev: 親コンテキスト（ベース）のサーバー設定。
  • overridesv: 現在のコンテキストで上書きされるサーバー設定。
  • 戻り値: マージされた設定（本関数では basev をそのまま返却）。

前提条件と不変条件

  • AuthnProviderAlias は GLOBAL_ONLY であり、サーバー起動時に設定が確定するため、サーバー設定のマージにおいてはベース設定を優先する実装となっています。

アルゴリズムまたは振る舞い

  • 引数として受け取った basev をそのまま返却します。設定の統合処理は行わず、親の設定を継承する挙動をとります。

呼び出し元と呼び出し先

  • 呼び出し元: modules/aaa/mod_authn_core.c:423 （サーバー設定のマージ処理として登録）
  • 呼び出し先: なし

関連データ構造

  • authn_alias_srv_conf: サーバー全体の設定を管理する構造体。

失敗モードと境界条件

  • 特になし。常に basev を返すため、入力が NULL であれば NULL を返します。

保守担当者向けメモ

  • 本関数は単にベース設定を返すのみであり、エイリアス設定の複雑なマージロジックは含まれていません。将来的にエイリアスの継承ルールに変更が必要な場合は、この関数を拡張してください。
