h2_c1_io

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_c1_io.h:29

目的

HTTP/2 セッションにおけるクライアント接続（C1）の入出力処理を抽象化し、Apache HTTP Server の接続フィルタスタックとプロトコル層間のデータ転送を管理します。

シグネチャまたは形状

typedef struct {
    struct h2_session *session;
    apr_bucket_brigade *output;
    int is_tls;
    int unflushed;
    apr_time_t cooldown_usecs;
    apr_int64_t warmup_size;
    apr_size_t write_size;
    apr_time_t last_write;
    apr_int64_t bytes_read;
    apr_int64_t bytes_written;
    int buffer_output;
    apr_off_t buffered_len;
    apr_off_t flush_threshold;
    unsigned int is_flushed : 1;
    unsigned int is_passing : 1;
    char *scratch;
    apr_size_t ssize;
    apr_size_t slen;
} h2_c1_io;

入力と出力

  • **入力**: h2_session へのポインタ、および apr_bucket_brigade を介したデータ。
  • **出力**: 接続フィルタスタックへのデータ送出（ap_pass_brigade 経由）、および内部バッファ（scratch）の状態管理。

前提条件と不変条件

  • h2_session と対になって動作します。
  • output メンバは apr_bucket_brigade として初期化されている必要があります。
  • write_size 等の定数は、TLS レコードサイズや MTU と整合している必要があります。

アルゴリズムまたは振る舞い

  • **出力バッファリング**: output ブリゲードにデータを蓄積し、flush_threshold に基づいてネットワークへの書き込みを最適化します。
  • **フラッシュ制御**: is_flushed および is_passing ビットフラグにより、再帰的なフラッシュ呼び出しを防止し、適切なタイミングで ap_pass_brigade を実行します。
  • **スクラッチ領域**: scratch バッファを使用して、効率的なデータ読み込みと一時的なデータ保持を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_c1_io.c 内の h2_c1_io_init 等の関数群から参照されています。
  • **呼び出し先**: apr_bucket_brigade API および Apache HTTP Server のフィルタ API（ap_pass_brigade 等）を呼び出します。
  • ※詳細なコールグラフは提供されたソース excerpt には含まれていません。

関連データ構造

  • h2_session: 本構造体を所有し、セッションライフサイクルを管理します。
  • apr_bucket_brigade: データ転送の基本単位として使用されます。

失敗モードと境界条件

  • is_passing フラグによる再帰呼び出しの検知。
  • flush_threshold を超えた際のフラッシュ処理の強制。
  • scratch 領域のメモリ不足や、ブリゲード操作時の APR エラーコード。

保守担当者向けメモ

  • 本構造体は書き込みの集約を担うため、パフォーマンスチューニング時は write_size や flush_threshold の設定値に注意してください。
  • デバッグ時には h2_c1_io_bb_log を使用して、ブリゲード内のバケット状態（EOS, FLUSH 等）を追跡することが推奨されます。
