dav_prop_name

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1205

目的

WebDAV プロパティを識別するための名前空間（namespace）と名前（name）のペアを保持する構造体です。

シグネチャまたは形状

typedef struct {
    const char *ns;     /* "" signals "no namespace" */
    const char *name;
} dav_prop_name;

入力と出力

  • ns: プロパティの名前空間を示す文字列。空文字列 "" は名前空間が指定されていないことを示します。
  • name: プロパティの名称を示す文字列。

前提条件と不変条件

  • ns および name は、プロパティの識別子として使用されるため、有効な文字列ポインタである必要があります。
  • 名前空間が不要な場合は ns に空文字列を指定する規約となっています。

アルゴリズムまたは振る舞い

WebDAV プロパティのデータベース操作において、特定のプロパティを検索、保存、削除するためのキーとして利用されます。プロバイダーの実装において、この構造体を用いてプロパティのユニークな識別を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/dbm.c: dav_build_key, dav_propdb_store, dav_propdb_remove, dav_propdb_exists, dav_set_name, dav_propdb_next_name, dav_propdb_first_name 等で引数として使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数呼び出しを含みません。

関連データ構造

  • dav_db: プロパティデータベースの操作コンテキスト。
  • dav_hooks_propdb: プロパティデータベースのフックテーブル（output_value や store メソッドの引数として本構造体を使用）。

失敗モードと境界条件

  • ns または name に NULL を渡した場合の挙動は、各プロバイダーの実装に依存します。
  • 空文字列 "" の扱いについては、プロバイダー側で名前空間なしのプロパティとして正しく解釈される必要があります。

保守担当者向けメモ

  • 本構造体は mod_dav のプロパティ管理における基盤となる型です。変更を行う際は、modules/dav/fs/dbm.c などのプロバイダー実装におけるキー生成ロジックへの影響を考慮してください。
