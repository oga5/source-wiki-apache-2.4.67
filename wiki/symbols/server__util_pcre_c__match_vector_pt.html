match_vector_pt

定義

  • 種別: typedef
  • 定義箇所: server/util_pcre.c:278

目的

正規表現のマッチング結果（オフセット）を格納するための配列型を定義します。PCRE ライブラリがマッチング位置を記録するために使用するメモリ領域を指すポインタ型です。

シグネチャまたは形状

typedef size_t* match_vector_pt;

入力と出力

  • **入力**: 正規表現のマッチング処理において、マッチ結果を書き込むためのメモリ領域を指すポインタとして使用されます。
  • **出力**: マッチング成功時、このポインタが指す配列には、マッチした文字列の開始位置と終了位置のオフセットが格納されます。

前提条件と不変条件

  • size_t 型の配列を指す必要があります。
  • マッチング関数（ap_regexec 等）に渡される際、十分な要素数が確保されている必要があります。

アルゴリズムまたは振る舞い

  • PCRE1/PCRE2 のマッチング結果（ovector）を保持するための抽象化として機能します。
  • 内部的には、マッチング処理のワークスペースとして、スレッドローカルストレージ（TLS）から取得されたバッファや、スタック上の配列を指すために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_pcre.c 内の ap_regexec_len や、マッチング用ワークスペースを管理する内部関数から参照されます。
  • **呼び出し先**: 直接的な関数呼び出しではなく、PCRE ライブラリの実行関数（pcre_exec や pcre2_match）へマッチング結果の格納先として渡されます。

関連データ構造

  • ap_regex_t: 正規表現のコンパイル済みオブジェクト。
  • match_data_pt: PCRE2 におけるマッチングデータ構造体へのポインタ。

失敗モードと境界条件

  • **メモリ不足**: 適切なサイズの配列が割り当てられていない場合、マッチング実行時にバッファオーバーフローやセグメンテーション違反が発生する可能性があります。
  • **不整合**: size_t 型のサイズが環境によって異なる場合、PCRE ライブラリの期待するオフセット型と不整合が生じる可能性があります。

保守担当者向けメモ

  • 本シンボルは server/util_pcre.c 内で定義されており、PCRE1 と PCRE2 の差異を吸収するレイヤーの一部です。
  • ソースコード内の記述によれば、int* 型として定義されている箇所も存在するため、型定義の整合性には注意が必要です（server/util_pcre.c:281 参照）。
