ap_proxy_header_reverse_map_fn

定義

  • 種別: typedef (関数ポインタ型)
  • 定義箇所: modules/proxy/mod_proxy.h:733

目的

プロキシされたレスポンスヘッダー（Location や Set-Cookie など）に含まれるバックエンドの URL やパスを、クライアントから見た適切な形式に書き換えるためのコールバック関数を定義します。

シグネチャまたは形状

typedef const char *(*ap_proxy_header_reverse_map_fn)(request_rec *r, 
                                                      char *str, 
                                                      const char *url);

※ mod_proxy.h に基づく標準的なシグネチャです。

入力と出力

  • **入力**:
  • request_rec *r: 現在のリクエスト情報。
  • char *str: 書き換え対象のヘッダー値。
  • const char *url: マッピングの基準となるバックエンドの URL。
  • **出力**:
  • 書き換え後の新しいヘッダー値（const char *）。書き換えが不要な場合は元の値や NULL が返される可能性があります。

前提条件と不変条件

  • h2_proxy_session.c 内の transform_hdrs 配列において、特定のヘッダー名とこの関数型を実装する関数（ap_proxy_location_reverse_map 等）がペアで管理されています。

アルゴリズムまたは振る舞い

h2_proxy_session.c においては、バックエンドから受け取った特定のレスポンスヘッダーを走査し、transform_hdrs テーブルに定義されたマッピング関数を適用することで、プロキシ透過性を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c 内のヘッダー処理ロジック（直接的な呼び出し箇所はソース抜粋外）。
  • **呼び出し先**: ap_proxy_location_reverse_map および ap_proxy_cookie_reverse_map がこの型に適合する実装として参照されています。

関連データ構造

  • transform_hdrs: h2_proxy_session.c 内で定義された、ヘッダー名と本関数ポインタを保持する静的構造体配列。

失敗モードと境界条件

  • 該当するヘッダーが存在しない場合や、マッピング関数が NULL を返す場合の挙動は、各実装関数に依存します。

保守担当者向けメモ

  • 本シンボルは mod_proxy のコア機能であり、httpd 全体で共有されています。HTTP/2 プロキシ特有の処理を追加する際は、transform_hdrs 配列に新しいヘッダー名と適切なマッピング関数を追加してください。
