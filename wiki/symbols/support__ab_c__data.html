data

定義

  • 種別: struct
  • 定義箇所: support/ab.c:289

目的

ApacheBench (ab) において、個別の HTTP リクエスト実行時に計測された時間メトリクスを保持するために使用されます。

シグネチャまたは形状

struct data {
    apr_time_t starttime;         /* 接続開始時刻 */
    apr_interval_time_t waittime; /* リクエスト送信からレスポンス受信開始までの待機時間 */
    apr_interval_time_t ctime;    /* 接続確立にかかった時間 */
    apr_interval_time_t time;     /* 接続全体の処理時間 */
};

入力と出力

  • **入力**: 接続処理の各フェーズ（接続開始、リクエスト送信、レスポンス受信）で取得された apr_time_t および apr_interval_time_t 型の値。
  • **出力**: 収集されたデータは、最終的に output_results() 関数によって統計処理（平均、標準偏差、パーセンタイル計算）の対象となります。

前提条件と不変条件

  • ab の実行中に確保される data 型の配列は、同時接続数（concurrency）に基づいたサイズで動的に確保されます。
  • 各フィールドには APR (Apache Portable Runtime) の時間計測関数によって得られた値が格納されます。

アルゴリズムまたは振る舞い

1. 接続のライフサイクルに応じて、各タイムスタンプが記録されます。 2. 全リクエスト終了後、これらのデータが格納された配列がソートされ、統計情報の算出に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: support/ab.c 内の統計収集処理。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • **注記**: ツールによる検索結果に含まれる include/ap_expr.h 等の data は、本構造体とは無関係な別名シンボルです。

関連データ構造

  • struct connection: 接続の状態やソケットを管理する構造体。本構造体と対になって接続ごとの詳細を保持します。

失敗モードと境界条件

  • 統計計算時にデータが不完全な場合（接続エラー等）、計測値が初期値のままとなる可能性があります。

保守担当者向けメモ

  • 本構造体は ab の統計解析の根幹をなすため、計測精度の変更や新しいメトリクスの追加を行う場合は、output_results() での集計ロジックと整合性を取る必要があります。
