opt_retr

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_file.c:173

目的

認証キャッシュ機能を提供するオプション関数 ap_authn_cache_store を取得し、モジュール内のグローバル変数 authn_cache_store に格納する。

シグネチャまたは形状

static void opt_retr(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として authn_cache_store 変数を更新）

前提条件と不変条件

  • APR_RETRIEVE_OPTIONAL_FN マクロが利用可能であること。
  • 本関数が ap_hook_optional_fn_retrieve を介して呼び出されること。

アルゴリズムまたは振る舞い

1. APR_RETRIEVE_OPTIONAL_FN(ap_authn_cache_store) を実行し、オプション関数ポインタを解決する。 2. 取得した関数ポインタを静的変数 authn_cache_store に代入する。これにより、後続の認証処理においてキャッシュ機能が利用可能になる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/aaa/mod_authn_file.c:182 (フック登録時)
  • modules/aaa/mod_authn_dbd.c:295 (同名の関数がフックされている)
  • modules/aaa/mod_authn_dbm.c:219 (同名の関数がフックされている)
  • 呼び出し先: ap_authn_cache_store (オプション関数として取得)

関連データ構造

  • authn_cache_store: 取得したキャッシュ格納関数を保持する静的変数。

失敗モードと境界条件

  • ap_authn_cache_store が提供されていない場合、authn_cache_store には NULL が格納される。この場合、認証結果のキャッシュ処理はスキップされる。

保守担当者向けメモ

  • 本関数は mod_authn_file.c だけでなく、他の認証モジュール（mod_authn_dbd.c, mod_authn_dbm.c）でも同名の関数が定義され、同様の目的で使用されている。
  • APR_RETRIEVE_OPTIONAL_FN は実行時に解決されるため、依存モジュールのロード順序に依存しない設計となっている。
