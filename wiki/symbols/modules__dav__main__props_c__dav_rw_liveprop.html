dav_rw_liveprop

定義

  • 種別: function
  • 定義箇所: modules/dav/main/props.c:288

目的

指定されたプロパティが書き込み可能かどうかを判定します。

シグネチャまたは形状

int dav_rw_liveprop(dav_propdb *propdb, dav_elem_private *priv)

入力と出力

  • **入力**:
  • propdb: プロパティ操作のコンテキスト。
  • priv: 判定対象のプロパティ情報（propid およびプロバイダー情報を含む）。
  • **出力**:
  • 1: 書き込み可能。
  • 0: 読み取り専用。

前提条件と不変条件

  • priv->provider が設定されている場合、そのプロバイダーの is_writable フックの結果に従います。
  • DAV_DISABLE_WRITABLE_PROPS が定義されている場合、一部のコアプロパティ（getcontenttype, getcontentlanguage）は読み取り専用として扱われます。

アルゴリズムまたは振る舞い

1. **プロバイダー確認**: priv->provider が存在する場合、そのプロバイダーの is_writable 関数を呼び出し、その結果を返します。 2. **コアプロパティの判定**:

  • lockdiscovery, supportedlock は常に読み取り専用（0）を返します。
  • DAV_DISABLE_WRITABLE_PROPS が有効な場合、getcontenttype および getcontentlanguage も読み取り専用（0）となります。

    • 3. **書き込み可能プロパティの判定**: getcontenttype, getcontentlanguage, DAV_PROPID_CORE_UNKNOWN は書き込み可能（1）と判定します。 4. **デフォルト**: 上記のいずれにも該当しない（認識されない）プロパティは、デッドプロパティと見なし、書き込み可能（1）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/props.c:987 にて呼び出しを確認。
  • **呼び出し先**: priv->provider->is_writable（プロバイダーが定義されている場合）。

関連データ構造

  • dav_propdb: プロパティ操作のコンテキスト。
  • dav_elem_private: プロパティの識別子（propid）およびプロバイダーを保持する構造体。

失敗モードと境界条件

  • 未知のプロパティ（DAV_PROPID_CORE_UNKNOWN）はデフォルトで書き込み可能と判定されます。
  • DAV_DISABLE_WRITABLE_PROPS の定義有無により、特定のコアプロパティの書き込み可否が変化します。

保守担当者向けメモ

  • 本関数は PROPPATCH 等の処理において、プロパティの更新可否を決定する重要な役割を担います。
  • 新しいコアプロパティを追加する際は、本関数の判定ロジックに含める必要があります。
