remove_input_filters_below

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_switch.c:142

目的

指定されたフィルター型よりも低い（優先度の低い）入力フィルターを、フィルターチェーンから順次削除します。

シグネチャまたは形状

static void remove_input_filters_below(ap_filter_t *f, ap_filter_type ftype)
  • f: 削除を開始する起点となる入力フィルター。
  • ftype: 削除対象を判定するためのフィルター型（ap_filter_type）。

入力と出力

  • **入力**:
  • ftype: 削除の閾値となるフィルター型。
  • **出力**: なし（void）。副作用として、条件に合致するフィルターがチェーンから取り除かれます。

前提条件と不変条件

  • f が NULL の場合、何も行われません。
  • f->frec->ftype < ftype を満たすフィルターが削除対象となります。

アルゴリズムまたは振る舞い

1. 引数 f から開始し、現在のフィルターの型 (f->frec->ftype) が指定された ftype 未満である限りループを継続します。 2. ループ内で、現在のフィルターの次にあるフィルター (f->next) を一時保存します。 3. ap_remove_input_filter(f) を呼び出し、現在のフィルターをチェーンから削除します。 4. 保存しておいた次のフィルターを現在のフィルターとして更新し、判定を繰り返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_switch.c:188 にて、r->input_filters を引数として呼び出されています。
  • **呼び出し先**:
  • ap_remove_input_filter (Apache HTTP Server API)

関連データ構造

  • ap_filter_t: Apache のフィルターチェーンを構成する構造体。
  • ap_filter_type: フィルターの順序を決定する型定義。

失敗モードと境界条件

  • f が NULL の場合、安全に終了します。
  • フィルターチェーンの末尾に達した場合、ループは終了します。
  • 削除対象のフィルターが存在しない場合、チェーンに変更は加えられません。

保守担当者向けメモ

  • 本関数は HTTP/1.1 のフィルターを破棄し、HTTP/2 用のスタックへ差し替える際に使用されます。
  • ap_remove_input_filter を呼び出すため、フィルターの再構成中にチェーンの整合性が保たれるよう注意してください。
