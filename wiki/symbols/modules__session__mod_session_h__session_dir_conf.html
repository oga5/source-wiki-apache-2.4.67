session_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/session/mod_session.h:103

目的

Apache HTTP Server の各ディレクトリ（または場所）における mod_session の動作設定を保持する構造体です。

シグネチャまたは形状

typedef struct {
    int enabled;
    int enabled_set;
    long maxage;
    int maxage_set;
    const char *header;
    int header_set;
    int env;
    int env_set;
    apr_array_header_t *includes;
    apr_array_header_t *excludes;
    apr_time_t expiry_update_time;
    int expiry_update_set;
} session_dir_conf;

入力と出力

  • **入力**: Apache の設定ファイル（httpd.conf 等）から読み込まれたディレクティブ値。
  • **出力**: mod_session の各フック関数が参照する設定値。

前提条件と不変条件

  • *_set 接尾辞を持つ各メンバは、対応する設定値が明示的に設定されたかどうかを示すフラグとして機能します。
  • includes および excludes は apr_array_header_t 型であり、URL プレフィックスのリストを保持します。

アルゴリズムまたは振る舞い

  • ディレクトリ単位の設定として ap_get_module_config を通じて取得されます。
  • session_included 関数において、リクエストパスが includes に含まれ、かつ excludes に含まれていないかを判定するために使用されます。
  • expiry_update_time は、セッションの有効期限を更新せずに保持し続けるための閾値として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session.c 内の各関数（session_included、およびリクエスト処理中の各種ハンドラ）。
  • **呼び出し先**: apr_pcalloc（メモリ確保時）、ap_get_module_config（設定取得時）。

関連データ構造

  • session_rec: セッションの実行時状態を保持する構造体。
  • apr_array_header_t: パス制限（include/exclude）のリスト管理に使用。

失敗モードと境界条件

  • includes が空の場合、すべての URL が対象となります。
  • excludes が空の場合、除外される URL はありません。
  • 設定が未定義の場合、*_set フラグによりデフォルト値との区別が行われます。

保守担当者向けメモ

  • 本構造体は mod_session の設定基盤です。新しい設定項目を追加する場合は、対応する _set フラグを必ず定義し、マージ関数（mod_session.c 内）で適切に処理してください。
