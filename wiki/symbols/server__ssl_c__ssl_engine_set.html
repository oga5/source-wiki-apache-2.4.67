ssl_engine_set

定義

  • 種別: function
  • 定義箇所: server/ssl.c:99

目的

接続（conn_rec）に対して SSL エンジンの有効化または無効化を制御し、プロキシ接続か否かに応じて適切な SSL 処理関数へルーティングする。

シグネチャまたは形状

static int ssl_engine_set(conn_rec *c,
                          ap_conf_vector_t *per_dir_config,
                          int proxy, int enable)

入力と出力

  • **入力**:
  • c: 処理対象の接続構造体。
  • per_dir_config: ディレクトリごとの設定ベクトル。
  • proxy: プロキシ接続の場合は非ゼロ、それ以外は 0。
  • enable: SSL を有効にする場合は非ゼロ、無効にする場合は 0。
  • **出力**:
  • 成功時または処理実行時に 1（または OK）、失敗時や該当する処理がない場合に 0 を返す。

前提条件と不変条件

  • module_ssl_engine_set 等の外部モジュール由来のオプション関数が ap_setup_ssl_optional_fns 等を通じて適切に取得されている必要がある。
  • proxy が真の場合、ap_ssl_bind_outgoing が呼び出される。

アルゴリズムまたは振る舞い

1. proxy が真の場合: ap_ssl_bind_outgoing を呼び出し、その戻り値が OK であるかを判定して結果を返す。 2. proxy が偽の場合:

  • module_ssl_engine_set が定義されていれば、それを呼び出す。
  • enable が真で module_ssl_proxy_enable が定義されていれば、それを呼び出す。
  • enable が偽で module_ssl_engine_disable が定義されていれば、それを呼び出す。

    • 3. 上記のいずれにも該当しない場合、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy.c（APR_OPTIONAL_FN_TYPE(ssl_engine_set) として利用）
  • modules/ssl/mod_ssl.c（sslconn->dc のリセット時に使用）
  • **呼び出し先**:
  • ap_ssl_bind_outgoing
  • module_ssl_engine_set（オプション関数）
  • module_ssl_proxy_enable（オプション関数）
  • module_ssl_engine_disable（オプション関数）

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • ap_conf_vector_t: 設定情報を保持するベクトル。

失敗モードと境界条件

  • 外部モジュール（mod_ssl 等）がロードされていない場合、オプション関数ポインタが NULL となり、条件分岐の結果として 0 が返される。
  • ap_ssl_bind_outgoing が OK 以外の値を返した場合、ssl_engine_set は 0 を返す。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN(ssl_engine_set) によりオプション関数として登録されており、他モジュールから動的に呼び出される。
  • module_ssl_* 系変数は server/ssl.c 内で APR_RETRIEVE_OPTIONAL_FN を用いて初期化されるため、モジュールのロード順序に依存する可能性がある。
