h2_proxy_iqueue

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_proxy_util.h:66

目的

HTTP/2 プロキシ処理において、ストリームIDなどの整数値を管理し、優先度付きでソート可能なキューを提供します。

シグネチャまたは形状

typedef struct h2_proxy_iqueue {
    int *elts;           /* 整数要素の配列 */
    int head;            /* 先頭インデックス */
    int nelts;           /* 現在の要素数 */
    int nalloc;          /* 確保済みの容量 */
    apr_pool_t *pool;    /* メモリ管理用プール */
} h2_proxy_iqueue;

入力と出力

  • **入力**: h2_proxy_iq_create を介して apr_pool_t と初期容量を指定して生成されます。
  • **出力**: 内部的に int 型の配列を保持し、キュー操作の結果としてストリームID等の整数値を返します。

前提条件と不変条件

  • apr_pool_t によるメモリ管理下にあります。
  • nelts は常に nalloc 以下である必要があります。
  • elts 配列は iq_grow 関数によって動的に拡張される可能性があります。

アルゴリズムまたは振る舞い

  • **ソート**: 優先度付きキューとして機能し、iq_bubble_up および iq_bubble_down を用いてヒープ構造を維持します。
  • **動的拡張**: 要素数が容量を超えると iq_grow が呼び出され、メモリが再確保されます。
  • **操作**: h2_proxy_iq_add による追加、h2_proxy_iq_remove による削除、および h2_proxy_iq_empty / h2_proxy_iq_count による状態確認が可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.h にて構造体が参照されており、h2_proxy_session の一部として利用されます。
  • **呼び出し先**: modules/http2/h2_proxy_util.c 内の iq_grow, iq_swap, iq_bubble_up, iq_bubble_down などの内部関数が、キューの操作ロジックを実装しています。

関連データ構造

  • apr_pool_t: メモリ割り当てに使用。
  • h2_proxy_iq_cmp: キュー内の要素を比較するための関数ポインタ型。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc 等の APR 関数が失敗した場合の挙動は APR の仕様に依存します。
  • 空のキュー: h2_proxy_iq_remove を空のキューに対して実行した場合の挙動については、ソースコード上の明示的なガード条件を確認する必要があります。

保守担当者向けメモ

  • 本構造体は h2_proxy_session において suspended ストリームの管理などに利用されています。
  • 比較ロジックを変更する場合は h2_proxy_iq_cmp の実装を確認してください。
  • iq_swap 等の内部関数は static で定義されており、h2_proxy_util.c 内でのみ利用可能です。
