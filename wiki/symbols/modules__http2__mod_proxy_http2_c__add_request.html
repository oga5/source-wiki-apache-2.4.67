add_request

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_proxy_http2.c:213

目的

HTTP/2 プロキシセッションに対して、新しいリクエストをサブミット（投入）します。

シグネチャまたは形状

static apr_status_t add_request(h2_proxy_session *session, request_rec *r)

入力と出力

  • **入力**:
  • session: 操作対象の h2_proxy_session 構造体。
  • r: 処理対象の request_rec 構造体。
  • **出力**:
  • apr_status_t: 操作の成否（成功時は APR_SUCCESS）。

前提条件と不変条件

  • session->user_data に h2_proxy_ctx が格納されている必要があります。
  • r->notes に H2_PROXY_REQ_URL_NOTE キーで対象 URL が設定されている必要があります。

アルゴリズムまたは振る舞い

1. session->user_data からコンテキスト ctx を取得します。 2. r->notes からリクエスト URL を取得します。 3. バックエンド接続のローカルポート情報を取得し、r->notes の proxy-source-port に設定します。 4. h2_proxy_session_submit を呼び出し、セッションへリクエストを投入します。 5. 投入に失敗した場合、エラーログを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_proxy_http2.c:285
  • **呼び出し先**:
  • apr_table_get / apr_table_setn (APR テーブル操作)
  • apr_psprintf (文字列フォーマット)
  • h2_proxy_session_submit (HTTP/2 セッションへのリクエスト投入)
  • ap_log_cerror (エラーログ出力)

関連データ構造

  • h2_proxy_session: HTTP/2 プロキシセッションの状態管理。
  • h2_proxy_ctx: リクエスト処理中の接続情報や状態を保持する構造体。
  • request_rec: Apache のリクエスト情報。

失敗モードと境界条件

  • h2_proxy_session_submit が APR_SUCCESS 以外の値を返した場合、エラーログが出力され、そのステータスコードが呼び出し元へ戻されます。

保守担当者向けメモ

  • 本関数は h2_proxy_session_submit のラッパーとして機能しており、主にプロキシ特有のメタデータ（ソースポート等）の付与とエラーハンドリングを担当しています。
