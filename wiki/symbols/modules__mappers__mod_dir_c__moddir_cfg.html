moddir_cfg

定義

  • 種別: enum
  • 定義箇所: modules/mappers/mod_dir.c:36

目的

mod_dir モジュールにおける、ディレクトリ関連の設定値（DirectorySlash や DirectoryCheckHandler など）の有効・無効状態を表現するための列挙型です。

シグネチャまたは形状

typedef enum {
    MODDIR_OFF = 0,
    MODDIR_ON,
    MODDIR_UNSET
} moddir_cfg;

入力と出力

  • **入力**: 設定ファイル内のディレクティブ（DirectorySlash 等）の値。
  • **出力**: 設定構造体 dir_config_rec 内のメンバとして保持される状態値。

前提条件と不変条件

  • MODDIR_UNSET は、設定が明示的に指定されていない初期状態を表します。
  • MODDIR_OFF (0) および MODDIR_ON (1) は、論理的なオン・オフを制御します。

アルゴリズムまたは振る舞い

  • 設定の継承やマージの際、MODDIR_UNSET である場合は親設定から値を引き継ぐための判定基準として使用されます。
  • 実際の処理（fixup_dir 等）において、この値が MODDIR_ON である場合にのみ、対応する機能（スラッシュ補完やハンドラチェック）が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_dir.c:47 (dir_config_rec 構造体メンバ do_slash)
  • modules/mappers/mod_dir.c:48 (dir_config_rec 構造体メンバ checkhandler)
  • **呼び出し先**: 情報なし。

関連データ構造

  • dir_config_rec: 本列挙型をメンバとして保持し、ディレクトリごとの設定を管理する構造体。

失敗モードと境界条件

  • 設定値が MODDIR_UNSET のまま処理フェーズに到達した場合、モジュールはデフォルトの挙動（通常はオフまたは継承値）を選択する必要があります。

保守担当者向けメモ

  • 本列挙型は mod_dir モジュール専用です。新しい設定項目を追加する際は、この型を利用することで一貫した状態管理が可能です。
