ssl_config_server_new

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_config.c:220

目的

Apache HTTP Server の mod_ssl におけるサーバー設定構造体（SSLSrvConfigRec）を新規作成し、初期値で埋める。

シグネチャまたは形状

static SSLSrvConfigRec *ssl_config_server_new(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（設定構造体のメモリ割り当てに使用するメモリプール）
  • **出力**: 初期化済みの SSLSrvConfigRec 構造体へのポインタ

前提条件と不変条件

  • 引数 p は有効なメモリプールであること。
  • 戻り値の構造体は、modssl_ctx_init_server によって内部コンテキストが初期化された状態であること。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して SSLSrvConfigRec 構造体分のメモリを確保する。 2. 構造体の各フィールドを以下の通り初期化する:

  • mc を NULL に設定。
  • enabled を SSL_ENABLED_UNSET に設定。
  • vhost_id を NULL、vhost_id_len を 0 に設定。
  • session_cache_timeout, cipher_server_pref, insecure_reneg, session_tickets を UNSET に設定。
  • コンパイル環境に応じて strict_sni_vhost_check（HAVE_TLSEXT 定義時）および compression（OPENSSL_NO_COMP 未定義時）を初期化。

    • 3. modssl_ctx_init_server(sc, p) を呼び出し、サーバー固有の SSL コンテキストを初期化する。 4. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_config.c:249 (ssl_config_server_create 内)
  • modules/ssl/ssl_engine_config.c:396 (ssl_config_server_merge 内)
  • **呼び出し先**:
  • apr_palloc (APR ライブラリ)
  • modssl_ctx_init_server (modules/ssl/ssl_engine_config.c)

関連データ構造

  • SSLSrvConfigRec: 本関数が生成するサーバー設定構造体。
  • apr_pool_t: メモリ管理に使用されるプール。

失敗モードと境界条件

  • メモリ不足時には apr_palloc がアボートする可能性がある（APR の標準的な挙動）。
  • コンパイル時のマクロ定義（HAVE_TLSEXT, OPENSSL_NO_COMP）により、構造体の初期化対象フィールドが変動する。

保守担当者向けメモ

  • 本関数は設定の新規作成およびマージ時のベース作成に使用される。
  • 新たな SSL 設定項目を追加した場合は、本関数での初期化漏れがないか確認すること。
  • vhost_id 等のフィールドは、この時点では NULL であり、後のモジュール初期化フェーズで設定される前提となっている。
