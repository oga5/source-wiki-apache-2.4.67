merge_userdir_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_userdir.c:115

目的

Apache HTTP Server の設定マージフェーズにおいて、親コンテキスト（base）の設定と現在のコンテキスト（overrides）の設定を統合し、新しい userdir_config 構造体を生成する。

シグネチャまたは形状

static void *merge_userdir_config(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親コンテキストの設定（userdir_config 型）。
  • overridesv: 現在のコンテキストで上書きされる設定（userdir_config 型）。
  • **出力**:
  • 統合された新しい userdir_config 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は userdir_config 型としてキャスト可能であること。
  • apr_pcalloc によりメモリがゼロ初期化された状態で構造体が作成される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して新しい userdir_config 構造体をメモリ上に確保する。 2. globally_disabled 設定を統合する。overrides がデフォルト値（O_DEFAULT）でない場合は overrides の値を、それ以外は base の値を採用する。 3. userdir 設定を統合する。overrides がデフォルト値（DEFAULT_USER_DIR）でない場合は overrides の値を、それ以外は base の値を採用する。 4. enabled_users および disabled_users リストについてはマージを行わず、overrides の値をそのまま新しい構造体にコピーする。 5. 統合された構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_userdir.c:387（サーバー設定のマージ時に呼び出される）。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ）。

関連データ構造

  • userdir_config: モジュールの設定を保持する構造体。globally_disabled、userdir、enabled_users、disabled_users を含む。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが終了する可能性がある。
  • enabled_users および disabled_users はマージされないため、子コンテキストでこれらの設定を定義すると親の設定は完全に隠蔽される。

保守担当者向けメモ

  • 本関数は設定の階層的な継承を管理する。userdir パターンやグローバルな無効化フラグは継承されるが、ユーザーリストは継承されない仕様となっている点に注意すること。
