md_mod_conf_get

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_config.c:144

目的

mod_md モジュール全体で共有されるグローバル設定構造体 md_mod_conf_t を取得、または必要に応じて初期化します。

シグネチャまたは形状

static md_mod_conf_t *md_mod_conf_get(apr_pool_t *pool, int create)

入力と出力

  • **入力**:
  • pool: 設定構造体のメモリ割り当てに使用する APR プール。
  • create: 構造体が未初期化の場合に新規作成するかどうかを指定するフラグ（非ゼロで作成）。
  • **出力**:
  • 成功時: md_mod_conf_t 構造体へのポインタ。
  • 失敗時: 構造体が未初期化かつ create が 0 の場合は NULL。

前提条件と不変条件

  • mod_md_config はモジュール内で共有される静的変数です。
  • 一度初期化されると、そのプールが存続する限り同じインスタンスが再利用されます。
  • 初期化時にはデフォルト設定 defmc の内容がコピーされます。

アルゴリズムまたは振る舞い

1. 静的変数 mod_md_config が既に存在する場合は、それをそのまま返します。 2. create が真かつ mod_md_config が未初期化の場合:

  • apr_pcalloc を使用してメモリを確保します。
  • defmc から初期値をコピーします。
  • 内部配列（mds, unused_names）、テーブル（env）、ハッシュ（init_errors）を初期化します。
  • cleanup_mod_config をプールクリーンアップ関数として登録します。

    • 3. 構築済みの mod_md_config を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md_config.c:247
  • modules/md/mod_md_config.c:1443
  • **呼び出し先**:
  • apr_pcalloc, apr_array_make, apr_table_make, apr_hash_make, apr_pool_cleanup_register (APR API)
  • cleanup_mod_config (クリーンアップ関数)

関連データ構造

  • md_mod_conf_t: モジュール全体の設定を保持する主要構造体。
  • defmc: 初期化時に使用されるデフォルト設定値。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数がプロセスを終了させる可能性があります。
  • create が 0 の状態で未初期化のまま呼び出された場合、NULL を返します。

保守担当者向けメモ

  • 本関数は mod_md の設定ライフサイクルにおいて、グローバル設定のシングルトン的なアクセスポイントとして機能します。
  • cleanup_mod_config が登録されているため、プール破棄時に設定構造体も適切に解放されます。
