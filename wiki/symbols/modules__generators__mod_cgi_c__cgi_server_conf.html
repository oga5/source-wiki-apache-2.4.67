cgi_server_conf

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_cgi.c:85

目的

Apache HTTP Server における CGI モジュールのサーバー全体の設定情報を保持する構造体です。主にデバッグログの出力先や制限値を管理します。

シグネチャまたは形状

typedef struct {
    const char *logname;   /* スクリプトログのファイルパス */
    long        logbytes;  /* ログに記録する最大バイト数 */
    apr_size_t  bufbytes;  /* ログ記録用バッファサイズ */
} cgi_server_conf;

入力と出力

  • **入力**: ap_get_module_config を介してサーバー設定から取得されます。
  • **出力**: log_script や log_scripterror などの関数へ引数として渡され、CGI 実行時のデバッグ情報の制御に使用されます。

前提条件と不変条件

  • request_rec のプール (r->pool) を通じてメモリが割り当てられることが一般的です。
  • サーバー起動時に設定が読み込まれ、リクエスト処理中に参照されます。

アルゴリズムまたは振る舞い

  • mod_cgi.c 内の create_cgi_config 関数により、apr_pcalloc を用いてメモリが確保されます。
  • merge_cgi_config 関数により、ベース設定とオーバーライド設定の統合が行われます。
  • log_script などの関数内で、logname が設定されている場合にのみログ出力処理が実行されるという条件分岐の基準となります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c: 設定の生成 (create_cgi_config)、マージ (merge_cgi_config)、および設定値の取得 (set_scriptlog 等) で使用。
  • modules/generators/cgi_common.h: log_script や log_scripterror などの共通関数が引数として受け取ります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • cgi_dirconf: ディレクトリ単位の設定構造体。
  • request_rec: リクエストごとのコンテキスト情報。

失敗モードと境界条件

  • logname が NULL の場合、ログ出力は無効化されます。
  • logbytes や bufbytes の設定値が不適切な場合、ログ記録処理がスキップされるか、期待通りのサイズ制限が適用されない可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cgi と mod_cgid の両方で利用される cgi_common.h の関数群と密接に関係しています。
  • log_script は高コストな処理であるため、本番環境での設定変更時にはパフォーマンスへの影響を考慮してください。
