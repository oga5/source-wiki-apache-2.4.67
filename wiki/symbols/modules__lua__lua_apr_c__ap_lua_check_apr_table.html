ap_lua_check_apr_table

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_apr.c:22

目的

Lua スタック上の値が Apr.Table 型であることを検証し、対応する C 言語側の req_table_t 構造体ポインタを取得します。

シグネチャまたは形状

req_table_t *ap_lua_check_apr_table(lua_State *L, int index)

入力と出力

  • **入力**:
  • L: Lua 仮想マシンの状態 (lua_State)。
  • index: スタック上の検証対象となるインデックス。
  • **出力**:
  • 成功時: req_table_t 構造体へのポインタ。
  • 失敗時: luaL_checkudata により Lua エラーがスローされ、呼び出し元へは戻りません。

前提条件と不変条件

  • 指定されたスタックインデックス index に、Apr.Table メタテーブルを持つユーザーデータが存在する必要があります。

アルゴリズムまたは振る舞い

1. luaL_checkudata を呼び出し、スタック上の指定インデックスの値が Apr.Table 型であることを確認します。型が一致しない場合、Lua エラーを発生させます。 2. lua_unboxpointer を使用して、ユーザーデータ内に格納されている req_table_t へのポインタを取り出します。 3. 取得したポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_apr.c:40
  • modules/lua/lua_apr.c:74
  • **呼び出し先**:
  • luaL_checkudata (Lua C API)
  • lua_unboxpointer (Lua C API)

関連データ構造

  • req_table_t: apr_table_t をラップし、Lua から操作可能にするための構造体。
  • Apr.Table: Lua 側で req_table_t を識別するためのメタテーブル名。

失敗モードと境界条件

  • 指定されたインデックスに Apr.Table 以外の型が存在する場合、luaL_checkudata が Lua エラーをスローし、処理が中断されます。

保守担当者向けメモ

  • 本関数は lua_apr.h にて公開されており、mod_lua 内で Lua から渡されたテーブル操作リクエストを C の内部構造体に変換する際の標準的な検証手段として利用されています。
