md_http_impl_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_http.h:253

目的

mod_md モジュールにおける HTTP 通信機能の抽象化レイヤーを定義する関数テーブル（バックエンド実装）の型です。特定の HTTP クライアントライブラリ（libcurl 等）に依存しないプラグイン構造を提供します。

シグネチャまたは形状

typedef struct md_http_impl_t md_http_impl_t;

struct md_http_impl_t {
    /* 関数ポインタ等の実装詳細が続く */
};

※ソース excerpt には構造体のメンバ定義は含まれていません。

入力と出力

  • 入力: md_http_use_implementation() 関数を通じて、システムに登録されます。
  • 出力: md_http_t インスタンスが保持する通信バックエンドとして機能し、HTTP リクエストの実行を代行します。

前提条件と不変条件

  • md_http_create() を呼び出す前に、有効な実装が md_http_use_implementation() を介して登録されている必要があります。
  • md_http モジュールの内部静的変数 cur_impl を通じて管理されます。

アルゴリズムまたは振る舞い

  • 本構造体は、init、perform、multi_perform といった通信処理を抽象化するインターフェースを定義します。
  • 呼び出し側が md_http_perform() を実行すると、登録された md_http_impl_t の実体が実際のネットワーク通信を処理します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/md/md_http.c: cur_impl として保持および使用。
  • modules/md/md_curl.c: md_curl_get_impl() を通じて実装を提供。
  • 呼び出し先:
  • ソース excerpt には具体的な関数ポインタの呼び出し先は記載されていません。

関連データ構造

  • md_http_t: この実装を使用する HTTP セッションオブジェクト。
  • md_http_request_t: 通信対象となるリクエストデータ。
  • md_http_response_t: 通信結果として返されるレスポンスデータ。

失敗モードと境界条件

  • 実装が登録されていない状態で通信関数を呼び出した場合の挙動については、本ヘッダーの excerpt には記載がありません。

保守担当者向けメモ

  • md_http_request_t 内の internals ポインタは、この md_http_impl_t の実装側が自由に利用できる領域です。
  • 新しい通信バックエンドを追加する場合は、md_http_impl_t の全関数ポインタを実装し、md_http_use_implementation() で登録してください。
