authz_provider

定義

  • 種別: struct
  • 定義箇所: include/mod_auth.h:103

目的

Apache HTTP Server における認可（Authorization）プロバイダーのインターフェースを定義します。各認可モジュールはこの構造体を実装し、サーバーに登録することで、Require ディレクティブに対するアクセス制御ロジックを提供します。

シグネチャまたは形状

typedef struct {
    authz_status (*check_authorization)(request_rec *r,
                                        const char *require_line,
                                        const void *parsed_require_line);
    const char *(*parse_require_line)(cmd_parms *cmd, const char *require_line,
                                      const void **parsed_require_line);
} authz_provider;

入力と出力

  • check_authorization:
  • 入力: リクエスト情報 (request_rec *r)、設定行 (require_line)、事前解析済みデータ (parsed_require_line)。
  • 出力: 認可状態を示す authz_status（AUTHZ_GRANTED 等）。
  • parse_require_line:
  • 入力: 設定ディレクティブ (cmd_parms *cmd)、設定行 (require_line)。
  • 出力: 成功時は NULL、失敗時はエラーメッセージ文字列。parsed_require_line を通じて解析結果を格納。

前提条件と不変条件

  • parse_require_line は設定読み込み時に一度だけ実行されます。
  • check_authorization はリクエスト処理時に呼び出されます。
  • parse_require_line は NULL を許容します（その場合、解析処理はスキップされます）。

アルゴリズムまたは振る舞い

1. **設定解析**: サーバー起動時または設定再読み込み時に parse_require_line が呼び出され、設定行の構文チェックと最適化が行われます。 2. **認可判定**: リクエスト受信時、check_authorization が呼び出され、ユーザーのアクセス可否が判定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_authz_core.c が主にこの構造体を利用してプロバイダーを管理・実行します。
  • **呼び出し先（実装例）**: 以下のモジュールがこの構造体を実装しています。
  • mod_authnz_fcgi.c
  • mod_authnz_ldap.c
  • mod_authz_core.c (alias, env プロバイダー等)

関連データ構造

  • authz_status: 認可結果を保持する列挙型。
  • request_rec: HTTP リクエスト情報を保持するコア構造体。
  • cmd_parms: 設定ディレクティブのパラメータを保持する構造体。

失敗モードと境界条件

  • parse_require_line がエラーメッセージを返した場合、サーバーの設定読み込みが失敗します。
  • check_authorization が AUTHZ_DENIED やエラーを返した場合、認可プロセスは失敗として扱われます。

保守担当者向けメモ

  • parse_require_line で解析した結果を parsed_require_line に格納することで、リクエストごとの処理負荷を軽減できます。パフォーマンス最適化のために活用してください。
  • 本構造体はインターフェース定義のみであり、具体的なロジックは各モジュール側に実装されます。
