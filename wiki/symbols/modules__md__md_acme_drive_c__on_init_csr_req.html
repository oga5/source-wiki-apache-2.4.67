on_init_csr_req

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_drive.c:286

目的

ACME 証明書発行リクエスト（CSR）送信時のリクエストボディを初期化し、Base64 エンコードされた CSR データを JSON ペイロードに設定する。

シグネチャまたは形状

static apr_status_t on_init_csr_req(md_acme_req_t *req, void *baton)

入力と出力

  • **入力**:
  • req: ACME リクエストを保持する md_acme_req_t 構造体。
  • baton: md_proto_driver_t 型のドライバーコンテキスト。
  • **出力**:
  • 戻り値: apr_status_t 型。成功時は APR_SUCCESS。

前提条件と不変条件

  • baton は md_proto_driver_t 型としてキャスト可能であること。
  • md_proto_driver_t 内の baton が md_acme_driver_t を指しており、その csr_der_64 フィールドに有効な Base64 エンコード済み CSR が格納されていること。

アルゴリズムまたは振る舞い

1. req->p（リクエストプール）を使用して新しい JSON オブジェクトを生成する。 2. ad->csr_der_64 に保持されている CSR データを、キー MD_KEY_CSR を用いて JSON オブジェクトに格納する。 3. md_acme_req_body_init を呼び出し、構築した JSON をリクエストボディとして設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_drive.c:385 において、md_acme_POST 関数のコールバック引数として渡されている。
  • **呼び出し先**:
  • md_json_create
  • md_json_sets
  • md_acme_req_body_init

関連データ構造

  • md_acme_req_t: ACME 通信リクエストのコンテナ。
  • md_proto_driver_t: プロトコル駆動のドライバー状態。
  • md_acme_driver_t: ACME 固有のドライバー状態。

失敗モードと境界条件

  • JSON オブジェクトの生成失敗や、メモリ不足により md_json_create が失敗する場合がある。
  • md_acme_req_body_init が内部的なエラーを返す可能性がある。

保守担当者向けメモ

  • 本関数は md_acme_POST の実行フェーズで呼び出されるコールバックである。
  • MD_KEY_CSR は ACME プロトコルで定義された標準的なキー名であり、変更時にはプロトコル仕様との整合性に注意すること。
