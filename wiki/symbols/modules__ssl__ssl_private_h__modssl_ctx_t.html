modssl_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_private.h:777

目的

mod_ssl における SSL コンテキスト設定を保持するための構造体です。サーバーや仮想ホストごとの SSL 設定、証明書、OCSP、SRP、および OpenSSL の SSL_CTX へのパラメータを統合的に管理します。

シグネチャまたは形状

typedef struct {
    SSLSrvConfigRec *sc;
    SSL_CTX *ssl_ctx;
    modssl_pk_server_t *pks;
    modssl_pk_proxy_t  *pkp;
    /* ... (TLSチケット、プロトコル、パスフレーズ、CRL、OCSP、SRP、認証設定、SSL_CONF_CTX等) */
    BOOL ssl_check_peer_cn;
    BOOL ssl_check_peer_name;
    BOOL ssl_check_peer_expire;
} modssl_ctx_t;

入力と出力

  • **入力**: サーバー設定 (SSLSrvConfigRec)、証明書関連パス、OCSP 設定、および OpenSSL 固有の構成パラメータ。
  • **出力**: 保持された設定に基づく SSL_CTX オブジェクトの生成および管理。

前提条件と不変条件

  • sc フィールドは、このコンテキストを所有するサーバー設定への逆参照を保持します。
  • pks (サーバー用) または pkp (プロキシ用) のいずれかが使用されます。
  • HAVE_TLS_SESSION_TICKETS、HAVE_OCSP_STAPLING、HAVE_SRP、HAVE_SSL_CONF_CMD などのコンパイル時フラグにより、構造体のメモリレイアウトが変化します。

アルゴリズムまたは振る舞い

  • サーバー起動時および設定読み込み時に、modssl_ctx_init 等を通じて初期化されます。
  • SSL_CTX を中心に、証明書チェーン、CRL、OCSP レスポンダ設定、およびピア検証オプションをカプセル化します。
  • OpenSSL 3.0 等で導入された SSL_CONF_CTX を使用する場合、ssl_ctx_param に格納されたパラメータが SSL_CTX に適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/mod_ssl.c および modules/ssl/ssl_engine_config.c において、設定の初期化、マージ、および管理のために参照されています。
  • **呼び出し先**: OpenSSL ライブラリの SSL_CTX、SRP_VBASE、X509 関連のデータ構造を保持・操作します。

関連データ構造

  • SSLSrvConfigRec: 仮想ホストごとの設定。
  • modssl_pk_server_t / modssl_pk_proxy_t: 公開鍵・秘密鍵の構成。
  • modssl_auth_ctx_t: 認証コンテキスト。
  • SSL_CTX (OpenSSL): SSL/TLS セッションの基盤となるコンテキスト。

失敗モードと境界条件

  • 必要な証明書や鍵ファイルが存在しない場合、初期化フェーズでエラーが発生します。
  • OCSP や SRP などの機能がコンパイル時に無効化されている場合、対応するフィールドへのアクセスは未定義となります。
  • ssl_ctx_param の設定が不正な場合、OpenSSL 側の設定適用時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_ssl の設定管理の要です。フィールドを追加する際は、ssl_engine_config.c 内の modssl_ctx_init およびマージ関数（modssl_ctx_cfg_merge 等）の更新を忘れないでください。
  • OpenSSL のバージョン依存コードが含まれるため、条件付きコンパイル (#ifdef) の範囲を厳密に管理してください。
