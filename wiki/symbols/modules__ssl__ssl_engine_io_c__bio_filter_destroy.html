bio_filter_destroy

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_io.c:181

目的

OpenSSL の BIO オブジェクト破棄時に呼び出されるコールバック関数として機能し、カスタム BIO のライフサイクル管理を行います。

シグネチャまたは形状

static int bio_filter_destroy(BIO *bio)

入力と出力

  • **入力**: BIO *bio（破棄対象の BIO オブジェクト）
  • **出力**: int（成功時は 1、bio が NULL の場合は 0 を返す）

前提条件と不変条件

  • bio が NULL でない場合、この関数は常に成功（1）を返します。
  • 本関数内でメモリ解放処理は行われません。

アルゴリズムまたは振る舞い

1. bio が NULL かどうかを確認し、NULL であれば 0 を返して終了します。 2. NULL でない場合、Apache HTTP Server 側で apr_bucket_brigade が適切に破棄されるため、本関数内では明示的なメモリ解放を行わず、1 を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: OpenSSL の BIO メソッド破棄処理から間接的に呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありません。
  • **参照箇所**:
  • modules/ssl/ssl_engine_io.c:635（bio_filter_out_method の破棄関数として登録）
  • modules/ssl/ssl_engine_io.c:644（bio_filter_in_method の破棄関数として登録）
  • modules/ssl/ssl_engine_io.c:604, 617（ソースコード内で言及あり）

関連データ構造

  • BIO (OpenSSL)

失敗モードと境界条件

  • bio 引数が NULL の場合、0 を返して安全に終了します。

保守担当者向けメモ

  • 本関数は OpenSSL の BIO_METHOD に登録されるコールバックです。Apache のバケットブリゲードのライフサイクル管理と統合されているため、この関数内でメモリを解放しようとすると二重解放等の問題が発生する可能性があります。変更を加える際は注意してください。
