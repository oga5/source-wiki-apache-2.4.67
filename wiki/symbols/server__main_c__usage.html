usage

定義

  • 種別: function
  • 定義箇所: server/main.c:376

目的

Apache HTTP Server のコマンドライン引数のヘルプメッセージを標準エラー出力（APLOG_STARTUP）に表示し、プロセスを終了させる。

シグネチャまたは形状

static void usage(process_rec *process)

入力と出力

  • **入力**: process_rec *process（現在のプロセス情報。実行バイナリ名 argv[0] を取得するために使用）。
  • **出力**: なし（関数内で destroy_and_exit_process を呼び出し、プロセスを終了させるため戻り値はない）。

前提条件と不変条件

  • process 引数は有効な process_rec 構造体を指している必要がある。
  • 本関数は呼び出し後に制御を戻さない。

アルゴリズムまたは振る舞い

1. process->argv[0] から実行バイナリ名を取得し、ヘルプメッセージの整形に使用する。 2. ap_log_error を繰り返し呼び出し、利用可能なコマンドラインオプション（-D, -f, -k, -v 等）の一覧を標準エラー出力へ出力する。 3. プラットフォーム固有のオプション（WIN32 のサービス管理オプションや NETWARE のスクリーン名設定など）を条件付きコンパイルで含める。 4. 最後に destroy_and_exit_process(process, 1) を呼び出し、終了ステータス 1 でプロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: server/main.c 内の main 関数において、コマンドライン引数の解析中に不正なオプションが指定された場合や、-h オプションが指定された場合に呼び出される。
  • **呼び出し先**:
  • ap_log_error: メッセージの出力に使用。
  • destroy_and_exit_process: プロセスのクリーンアップと終了に使用。
  • strlen: バイナリ名の長さを計算するために使用。

関連データ構造

  • process_rec: プロセス情報を保持する構造体。argv 配列を通じて実行ファイル名を参照する。

失敗モードと境界条件

  • process が NULL の場合、process->argv[0] へのアクセスでセグメンテーション違反が発生する可能性がある。
  • ログ出力先がクローズされている場合、ap_log_error の挙動は基盤となるログシステムに依存する。

保守担当者向けメモ

  • 本関数内のヘルプメッセージは、新しいコマンドラインオプションが追加されるたびに手動で更新する必要がある。
  • WIN32 や NETWARE 固有のオプションが混在しているため、プラットフォームごとのビルド確認が必須である。
  • ap_log_error の APLOG_STARTUP フラグにより、サーバー起動時の初期段階でも適切にログが出力されるよう設計されている。
