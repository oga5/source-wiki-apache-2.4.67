log_is_level

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md.c:78

目的

指定されたログレベルが現在の設定で有効かどうかを判定します。

シグネチャまたは形状

static int log_is_level(void *baton, apr_pool_t *p, md_log_level_t level)

入力と出力

  • **入力**:
  • baton: 使用されていません。
  • p: 使用されていません。
  • level: 判定対象の md_log_level_t 型のログレベル。
  • **出力**:
  • 判定結果を int 型で返します（真であれば非ゼロ、偽であれば 0）。

前提条件と不変条件

  • 静的変数 log_server が初期化されている場合、Apache の APLOG_IS_LEVEL マクロを使用して判定を行います。
  • log_server が未設定の場合、level が MD_LOG_INFO 以下であれば真を返します。

アルゴリズムまたは振る舞い

1. 引数 baton および p を破棄（無視）します。 2. log_server が存在する場合、APLOG_IS_LEVEL を用いて Apache のログ設定に基づいた判定結果を返します。 3. log_server が存在しない場合、level <= MD_LOG_INFO という条件式の結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md.c:93: log_print 関数内で条件判定に使用。
  • modules/md/mod_md.c:883: md_log_set 関数にコールバックとして渡されています。
  • **呼び出し先**:
  • APLOG_IS_LEVEL (Apache HTTP Server マクロ)

関連データ構造

  • md_log_level_t: ログレベルを定義する列挙型または型定義。
  • log_server: モジュール内で保持されるサーバー設定構造体（server_rec）。

失敗モードと境界条件

  • log_server が NULL の場合、ハードコードされた MD_LOG_INFO を基準としたフォールバック判定が行われます。

保守担当者向けメモ

  • 本関数は md_log_set を介してログ出力のフィルタリングに使用されます。
  • baton および p は将来的な拡張のためにシグネチャに含まれていますが、現在の実装では使用されていません。
