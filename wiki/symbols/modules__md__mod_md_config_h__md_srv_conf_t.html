md_srv_conf_t

定義

  • 種別: struct
  • 定義箇所: modules/md/mod_md_config.h:90

目的

Apache HTTP Server の各仮想ホスト（サーバー）単位における mod_md の設定状態を保持する構造体です。ACME プロトコルによる証明書管理、更新ポリシー、および関連する設定値を格納します。

シグネチャまたは形状

typedef struct md_srv_conf_t {
    const char *name;
    const server_rec *s;
    md_mod_conf_t *mc;
    int transitive;
    md_require_t require_https;
    int renew_mode;
    int must_staple;
    struct md_pkeys_spec_t *pks;
    md_timeslice_t *renew_window;
    md_timeslice_t *warn_window;
    struct apr_array_header_t *ca_urls;
    const char *ca_contact;
    const char *ca_proto;
    const char *ca_agreement;
    struct apr_array_header_t *ca_challenges;
    const char *ca_eab_kid;
    const char *ca_eab_hmac;
    const char *profile;
    int profile_mandatory;
    int stapling;
    int staple_others;
    int ari_renewals;
    const char *dns01_cmd;
    md_t *current;
    struct apr_array_header_t *assigned;
    int is_ssl;
} md_srv_conf_t;

入力と出力

  • **入力**: Apache の設定ファイル（<MDomain> ディレクティブ等）から読み込まれた値。
  • **出力**: 実行時に md_config_get() 等のアクセサを通じて、他のモジュールコンポーネントへ設定値を提供します。

前提条件と不変条件

  • mc ポインタを通じて、常にグローバル設定（md_mod_conf_t）への参照を保持します。
  • post_config フェーズ以降、assigned 配列には当該サーバーに適用される md_t オブジェクトが格納されます。

アルゴリズムまたは振る舞い

1. **初期化**: Apache 起動時にデフォルト値で生成されます。 2. **マージ**: md_config_merge_svr により、グローバル設定と仮想ホスト固有の設定が階層的に統合されます。 3. **管理**: 設定ディレクティブの解析を通じて、ACME プロトコル設定、証明書更新ウィンドウ、OCSP ステープリング設定などが更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_md.c 内の merge_srv_config や、サーバー設定の生成・参照を行う各関数から利用されます。
  • **呼び出し先**: 構造体内部で md_t、md_mod_conf_t、md_timeslice_t などのモジュール内共通型を参照しています。

関連データ構造

  • md_mod_conf_t: モジュール全体のグローバル設定。
  • md_t: 管理対象ドメインの定義。
  • md_timeslice_t: 更新や警告のタイミングを定義する時間枠。
  • server_rec: Apache HTTP Server のサーバー構造体。

失敗モードと境界条件

  • 設定値が未定義の場合、デフォルト値が適用されます（mod_md_config.c に定義）。
  • ca_eab_kid や ca_eab_hmac が設定されていない場合、外部アカウントバインディングは無効となります。

保守担当者向けメモ

  • 直接メンバにアクセスせず、md_config_gets や md_config_geti などの型安全なアクセサ関数を使用することが推奨されます。
  • transitive フラグは、VirtualHost 名やエイリアスが自動的に追加されるかどうかを制御します。
