get_suexec_id_doer

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_userdir.c:347

目的

suEXEC が有効な環境において、リクエストに関連付けられたユーザーの UID および GID を解決し、ap_unix_identity_t 構造体として提供する。

シグネチャまたは形状

ap_unix_identity_t *get_suexec_id_doer(const request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: 成功時はユーザーの UID/GID を保持する ap_unix_identity_t 構造体へのポインタ。失敗時またはユーザー情報が取得できない場合は NULL。

前提条件と不変条件

  • APR_HAS_USER が定義されている環境でのみ動作する。
  • リクエストの request_config に userdir_module キーでユーザー名が格納されている必要がある。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、リクエストから対象のユーザー名を取得する。 2. ユーザー名が NULL の場合は NULL を返す。 3. apr_palloc を使用して ap_unix_identity_t 構造体用のメモリをリクエストプールから確保する。 4. apr_uid_get を呼び出し、システムからユーザー名に対応する UID と GID を取得する。 5. 取得に成功した場合、ugid->userdir フラグを 1 に設定し、構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_userdir.c:378 (ap_hook_get_suexec_identity に登録)
  • modules/generators/mod_suexec.c:126 (同フックへの登録が確認される)
  • **呼び出し先**:
  • ap_get_module_config
  • apr_palloc
  • apr_uid_get

関連データ構造

  • ap_unix_identity_t: UID, GID, および userdir フラグを保持する構造体。
  • request_rec: Apache のリクエスト処理における中心的な構造体。

失敗モードと境界条件

  • APR_HAS_USER が未定義の場合、常に NULL を返す。
  • リクエスト内にユーザー名が設定されていない場合、NULL を返す。
  • メモリ確保 (apr_palloc) に失敗した場合、NULL を返す。
  • apr_uid_get が APR_SUCCESS 以外の値を返した場合（ユーザーが存在しない等）、NULL を返す。

保守担当者向けメモ

  • 本関数は mod_userdir の suEXEC 連携の要です。OS 依存のユーザー情報取得 (apr_uid_get) を行っているため、プラットフォーム固有のユーザー管理設定の影響を受ける点に注意してください。
