ap_strrchr

定義

  • 種別: function
  • 定義箇所: server/util_debug.c:63

目的

標準ライブラリの strrchr をラップし、Apache HTTP Server のビルド環境において一貫した文字列検索インターフェースを提供します。

シグネチャまたは形状

char *ap_strrchr(char *s, int c)

入力と出力

  • **入力**:
  • s: 検索対象のヌル終端文字列へのポインタ。
  • c: 検索する文字（int 型）。
  • **出力**:
  • 文字列 s 内で最後に出現した文字 c へのポインタを返します。文字が見つからない場合は NULL を返します。

前提条件と不変条件

  • s は有効なヌル終端文字列である必要があります。
  • 本関数はメモリの所有権を保持せず、渡されたポインタのライフサイクルには関与しません。

アルゴリズムまたは振る舞い

標準ライブラリ関数 strrchr(s, c) を直接呼び出すラッパーとして動作します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_deflate.c
  • modules/mappers/mod_rewrite.c
  • modules/proxy/mod_proxy_fcgi.c
  • server/core.c
  • **呼び出し先**:
  • 標準ライブラリ関数 strrchr

関連データ構造

  • 特になし（標準的な文字列操作関数）。

失敗モードと境界条件

  • s が NULL の場合、標準ライブラリの strrchr の挙動に従い、未定義動作（通常はセグメンテーション違反）を引き起こす可能性があります。
  • 文字列内に c が存在しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は httpd.h 内で strrchr マクロとして定義されることがあり、server/util_debug.c では競合を避けるために一度 undef してから再定義されています。
  • 類似の型安全なバリアントとして ap_strrchr_c（const char * を扱う）が存在します。
