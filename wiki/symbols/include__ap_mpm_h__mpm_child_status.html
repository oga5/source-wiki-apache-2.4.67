mpm_child_status

定義

  • 種別: enum
  • 定義箇所: include/ap_mpm.h:208

目的

Apache HTTP Server のマルチプロセッシングモジュール（MPM）において、子プロセスのライフサイクル状態を表現するために使用されます。

シグネチャまたは形状

typedef enum mpm_child_status {
    MPM_CHILD_STARTED,
    MPM_CHILD_EXITED,
    MPM_CHILD_LOST_SLOT
} mpm_child_status;

入力と出力

  • 本シンボルは列挙型であり、関数等の引数として状態を渡すために使用されます。
  • child_status フックの引数として、現在のプロセスの状態を通知する役割を持ちます。

前提条件と不変条件

  • MPM_CHILD_STARTED: プロセスが正常に起動したことを示します。
  • MPM_CHILD_EXITED: プロセスが終了したことを示します。
  • MPM_CHILD_LOST_SLOT: プロセスがスコアボード上のスロットを喪失したことを示します。

アルゴリズムまたは振る舞い

  • MPM は子プロセスの状態変化が発生するたびに、この列挙値を用いて child_status フックを呼び出し、他のモジュールへ通知を行います。
  • モジュール側は、受け取った mpm_child_status の値に基づいて、ログ出力や統計情報の更新などの処理を分岐させることが期待されます。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm_common.c 内のプロセス管理ロジックが、状態変化時にこの列挙値を引数として渡します。
  • **呼び出し先**: child_status フックを実装している各モジュールが、この値を受け取って処理を行います。
  • ※具体的な呼び出し関係の詳細は、各 MPM 実装（worker, event, prefork 等）のソースコードに依存します。

関連データ構造

  • scoreboard.h: プロセス/スレッドの状態を共有メモリ上で追跡するために使用されます。
  • ap_mpm.h: child_status フックの定義が含まれています。

失敗モードと境界条件

  • モジュール側で mpm_child_status の値が適切に処理されない場合、サーバーの統計情報やプロセス監視に不整合が生じる可能性があります。
  • include/ap_mpm.h:230 のコメントにある通り、モジュールは未知のステータス値に対しては無視するような実装が推奨されます。

保守担当者向けメモ

  • 新しい状態を追加する場合は、既存の child_status フックを実装している全モジュールへの影響を考慮してください。
  • server/mpm_common.c での利用状況を追跡することで、状態遷移のトリガーとなる具体的なコードパスを特定可能です。
