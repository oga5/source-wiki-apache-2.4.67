file_cache_recall_mydata

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:218

目的

ディスクキャッシュファイルからメタデータ（disk_cache_info_t）およびキャッシュされた URL 文字列を読み込み、整合性を検証する。

シグネチャまたは形状

static int file_cache_recall_mydata(apr_file_t *fd, cache_info *info,
                                    disk_cache_object_t *dobj, request_rec *r)

入力と出力

  • **入力**:
  • fd: 読み込み対象のキャッシュファイルハンドル。
  • info: 読み込んだメタデータを格納する cache_info 構造体。
  • dobj: ディスクキャッシュオブジェクト（disk_info への書き込みおよび name との比較に使用）。
  • r: メモリ割り当てに使用するリクエストレコード。
  • **出力**:
  • 成功時は APR_SUCCESS を返す。
  • 失敗時は apr_file_read_full のエラーコード、または URL 不一致による APR_EGENERAL を返す。

前提条件と不変条件

  • fd は読み込み可能な状態で開かれている必要がある。
  • dobj->disk_info は disk_cache_info_t 構造体として適切に初期化されていること。
  • r->pool が有効であること。

アルゴリズムまたは振る舞い

1. fd から disk_cache_info_t サイズ分のデータを読み込み、dobj->disk_info に格納する。 2. 読み込んだメタデータから info 構造体の各フィールド（status, date, expire, request_time, response_time, control）を更新する。 3. dobj->disk_info.name_len に基づき、r->pool からメモリを確保し、キャッシュファイルから URL 文字列を読み込む。 4. 読み込んだ URL 文字列と dobj->name を strcmp で比較し、一致しない場合はエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache_disk.c:525
  • **呼び出し先**:
  • apr_file_read_full (APR)
  • apr_palloc (APR)
  • memcpy (標準ライブラリ)
  • strcmp (標準ライブラリ)

関連データ構造

  • disk_cache_info_t: キャッシュのメタデータ（有効期限、ステータス等）。
  • disk_cache_object_t: ディスクキャッシュの管理用構造体。
  • cache_info: mod_cache 側で利用されるキャッシュ情報構造体。

失敗モードと境界条件

  • apr_file_read_full が失敗した場合、そのエラーコードをそのまま返す。
  • キャッシュファイルから読み取った URL と dobj->name が一致しない場合、APR_EGENERAL を返す。

保守担当者向けメモ

  • URL の比較に strcmp を使用しているが、将来的に strncmp 等の検討が必要かどうかのコメントがソース内に存在する。
  • dobj->disk_info.name_len に基づくメモリ確保を行っているため、不正なキャッシュファイルによるメモリ枯渇リスクがないか注意が必要。
