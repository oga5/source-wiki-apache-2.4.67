set_pattern

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_substitute.c:609

目的

Substitute ディレクティブで指定された置換ルール（s/from/to/flags）を解析し、コンパイル済みのパターンとして設定構造体に登録する。

シグネチャまたは形状

static const char *set_pattern(cmd_parms *cmd, void *cfg, const char *line)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • cfg: subst_dir_conf 構造体へのポインタ。
  • line: Substitute ディレクティブに続く文字列（例: s/foo/bar/i）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • line は s で始まる必要がある。
  • s/// 形式の区切り文字が正しく指定されていること。
  • from（検索対象）および to（置換後）の文字列が空でないこと。

アルゴリズムまたは振る舞い

1. **構文解析**: line から区切り文字を特定し、from、to、flags を抽出する。 2. **フラグ処理**: flags を走査し、以下の動作を決定する。

  • i: 大文字小文字を区別しない（ignore_case）。
  • n: 正規表現ではなく固定文字列検索とする（is_pattern）。
  • f: flatten モード（バッファ結合）を有効化。
  • q: flatten モードを無効化。

    • 3. **コンパイル**:

  • is_pattern が偽の場合、ap_pregcomp を用いて正規表現をコンパイルする。
  • is_pattern が真の場合、apr_strmatch_precompile を用いて高速文字列検索パターンを生成する。

    • 4. **登録**: cfg->patterns 配列に新しい subst_pattern_t エントリを追加し、コンパイル結果と置換文字列を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_substitute.c:748（AP_INIT_TAKE1 マクロ経由で Substitute ディレクティブのハンドラとして登録）。
  • **呼び出し先**: apr_pstrdup, apr_tolower, ap_pregcomp, apr_array_push, apr_strmatch_precompile, strlen。

関連データ構造

  • subst_dir_conf: 設定を保持する構造体。
  • subst_pattern_t: 個別の置換ルールを保持する構造体。

失敗モードと境界条件

  • s で始まらない場合や、s/// 形式が不完全な場合はエラーメッセージを返す。
  • サポート外のフラグが指定された場合はエラーを返す。
  • 正規表現のコンパイルに失敗した場合はエラーを返す。

保守担当者向けメモ

  • flatten モードのデフォルトは 1（有効）である。
  • is_pattern が真のときは apr_strmatch が使用され、偽のときは ap_pregcomp（正規表現）が使用されるという排他的な設計になっている。
