read_quoted

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_imagemap.c:290

目的

文字列からダブルクォーテーションで囲まれた部分を抽出し、元の文字列ポインタをその直後まで進める。

シグネチャまたは形状

static void read_quoted(char **string, char **quoted_part)

入力と出力

  • string: 解析対象文字列へのポインタ。処理後、解析済みの末尾位置まで更新される。
  • quoted_part: 抽出された引用部分の開始位置を格納する出力用ポインタ。引用がない場合は NULL が設定される。

前提条件と不変条件

  • string が指す文字列は、関数内で \0 を挿入して破壊的に変更される可能性がある。
  • 引用符で囲まれていない場合、quoted_part は NULL となる。

アルゴリズムまたは振る舞い

1. string が指す先から空白文字をスキップする。 2. 次の文字が " であれば、その直後を quoted_part として記録する。 3. 次の " が現れるまでポインタを進め、その位置に \0 を挿入して文字列を終端する。 4. 最後に string のポインタを、閉じた " の次の文字位置まで進める。

呼び出し元と呼び出し先

  • 呼び出し元: modules/mappers/mod_imagemap.c 内の以下の箇所から呼び出されている。
  • 707行目
  • 753行目
  • 呼び出し先: apr_isspace (APR ライブラリ関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 文字列内に閉じの " が存在しない場合、文字列の末尾までが quoted_part として扱われる。
  • 空白のみの文字列や、引用符で始まらない文字列に対しては、quoted_part に NULL を設定して正常終了する。

保守担当者向けメモ

  • 本関数は入力文字列を直接書き換えるため、呼び出し元で渡す文字列バッファが変更可能であることを確認すること。
  • apr_isspace を使用しているため、ロケール設定に依存する可能性がある点に注意。
