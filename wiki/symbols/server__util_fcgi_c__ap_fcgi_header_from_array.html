ap_fcgi_header_from_array

定義

  • 種別: function
  • 定義箇所: server/util_fcgi.c:39

目的

FastCGI プロトコルのバイナリ形式であるバイト配列から、Apache 内部で利用される ap_fcgi_header 構造体へデータを展開します。

シグネチャまたは形状

void ap_fcgi_header_from_array(ap_fcgi_header *h, unsigned char a[])

入力と出力

  • **入力**:
  • h: 展開先の ap_fcgi_header 構造体へのポインタ。
  • a: FastCGI ヘッダーデータを含むバイト配列。
  • **出力**: なし（h が指す構造体の各フィールドが更新されます）。

前提条件と不変条件

  • a は、FastCGI ヘッダーの各フィールドを保持するのに十分なサイズ（少なくとも AP_FCGI_HEADER_LEN 分）のバイト配列である必要があります。
  • メモリの割り当ては行われません。呼び出し元が提供したバッファに対して操作を行います。

アルゴリズムまたは振る舞い

提供されたバイト配列 a の各オフセット位置から値を読み取り、ap_fcgi_header 構造体の対応するメンバに代入します。 1. version, type を代入。 2. requestId の上位バイト（B1）と下位バイト（B0）を代入。 3. contentLength の上位バイト（B1）と下位バイト（B0）を代入。 4. paddingLength および reserved を代入。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は、提供された情報からは特定できません。
  • **呼び出し先**: 本関数は構造体への代入のみを行うため、他の関数を呼び出しません。
  • **宣言**: include/util_fcgi.h:159 にて宣言されています。

関連データ構造

  • ap_fcgi_header: FastCGI プロトコルのヘッダー情報を保持する構造体。

失敗モードと境界条件

  • 本関数は入力値の検証を行いません。配列 a が不適切なサイズである場合、メモリ境界外アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数はメモリ所有権を持たず、純粋なデータ変換のみを担います。
  • FastCGI プロトコルの仕様変更に伴い、ヘッダー構造体のフィールド定義が変更された場合は、本関数のオフセット参照も更新する必要があります。
