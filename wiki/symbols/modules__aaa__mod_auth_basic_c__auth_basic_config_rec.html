auth_basic_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_auth_basic.c:36

目的

Apache HTTP Server の mod_auth_basic モジュールにおいて、ディレクトリ単位の認証設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    authn_provider_list *providers;
    char *dir; /* 未使用 */
    int authoritative;
    ap_expr_info_t *fakeuser;
    ap_expr_info_t *fakepass;
    const char *use_digest_algorithm;
    unsigned int fake_set:1,
                 use_digest_algorithm_set:1,
                 authoritative_set:1;
} auth_basic_config_rec;

入力と出力

本構造体は設定ファイル（httpd.conf 等）のディレクティブから生成されます。

  • **入力**: AuthBasicProvider, AuthBasicAuthoritative, AuthBasicFake, AuthBasicUseDigestAlgorithm 各ディレクティブの設定値。
  • **出力**: モジュール内の認証フック（authenticate_basic_user 等）が参照する設定情報。

前提条件と不変条件

  • メモリは request_rec のプールまたは設定プールから割り当てられます。
  • authoritative が 1 の場合、認証失敗時は後続の認証モジュールへ処理を委譲せず、即座に HTTP_UNAUTHORIZED を返します。

アルゴリズムまたは振る舞い

  • ディレクトリごとの設定を保持し、マージ処理を通じて階層的な設定を管理します。
  • fake_set などのビットフィールドは、各ディレクティブが明示的に設定されたかどうかを追跡し、マージ時の優先順位決定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_auth_basic.c 内の create_auth_basic_dir_config（初期化）、merge_auth_basic_dir_config（マージ）、および各ディレクティブハンドラ（add_authn_provider, set_authoritative, add_basic_fake, set_use_digest_algorithm）。
  • **呼び出し先**: authenticate_basic_user および authenticate_basic_fake が本構造体を参照して認証フローを制御します。

関連データ構造

  • authn_provider_list: 認証プロバイダーのリスト。
  • ap_expr_info_t: AuthBasicFake で使用される式評価エンジン用の構造体。

失敗モードと境界条件

  • AuthBasicFake で指定された式が空の場合、認証ヘッダーが削除され認証がスキップされる可能性があります。
  • authoritative が 0 の場合、認証失敗は DECLINED として扱われ、他の認証モジュールに処理が引き継がれます。

保守担当者向けメモ

  • char *dir は現在使用されていません。
  • ビットフィールド（fake_set 等）は、設定のオーバーライドを正しく処理するために不可欠です。変更時はマージロジックへの影響を確認してください。
