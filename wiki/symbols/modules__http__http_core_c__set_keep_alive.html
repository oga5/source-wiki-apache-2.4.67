set_keep_alive

定義

  • 種別: function
  • 定義箇所: modules/http/http_core.c:78

目的

Apache HTTP Server の設定ディレクティブ KeepAlive を解析し、サーバー設定構造体へ反映させるための設定ハンドラ関数です。

シグネチャまたは形状

static const char *set_keep_alive(cmd_parms *cmd, void *dummy, int arg);

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていません。
  • arg: KeepAlive ディレクティブの値（真偽値）。
  • **出力**:
  • 成功時は NULL を返します。
  • エラー時はエラーメッセージを示す文字列を返します。

前提条件と不変条件

  • 本関数は、ディレクトリコンテキスト（.htaccess 等）内では使用できません。ap_check_cmd_context により NOT_IN_DIR_CONTEXT が強制されます。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、現在の設定コンテキストが許可されているか検証します。エラーがあればそのメッセージを返して終了します。 2. 検証が成功した場合、cmd->server->keep_alive に引数 arg の値を代入します。 3. NULL を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_core.c:108 にて、AP_INIT_FLAG マクロを通じて KeepAlive ディレクティブのハンドラとして登録されています。
  • **呼び出し先**: ap_check_cmd_context（Apache Core API）。

関連データ構造

  • cmd_parms: 設定コマンドのコンテキスト情報を保持する構造体。
  • server_rec (cmd->server): サーバー設定を保持する構造体。keep_alive フィールドを更新します。

失敗モードと境界条件

  • ディレクトリコンテキストで KeepAlive が指定された場合、ap_check_cmd_context がエラー文字列を返し、設定の読み込みが失敗します。

保守担当者向けメモ

  • 本関数は KeepAlive ディレクティブ専用のフラグハンドラです。設定の変更は server_rec を直接更新するため、サーバー起動時または設定再読み込み時の構成に影響を与えます。
