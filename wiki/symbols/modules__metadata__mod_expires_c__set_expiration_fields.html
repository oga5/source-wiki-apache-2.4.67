set_expiration_fields

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:390

目的

リクエストの有効期限を算出し、HTTPレスポンスヘッダー（Cache-Control および Expires）を設定する。

シグネチャまたは形状

static int set_expiration_fields(request_rec *r, const char *code, apr_table_t *t)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • code: 有効期限の基準とオフセットを示す文字列（例: A123, M456）。
  • t: ヘッダーを格納する apr_table_t。
  • **出力**:
  • 成功時は OK を返す。
  • 失敗時は DECLINED または HTTP_INTERNAL_SERVER_ERROR を返す。

前提条件と不変条件

  • code の先頭文字は 'M'（最終更新時刻基準）または 'A'（アクセス時刻基準）である必要がある。
  • code の残りの部分は数値として解釈可能であること。

アルゴリズムまたは振る舞い

1. **基準時刻の決定**:

  • code[0] == 'M': r->finfo.mtime を基準とする。ファイルが存在しない場合は DECLINED を返す。
  • code[0] == 'A': r->request_time を基準とする。

    • 2. **有効期限の算出**:

  • code の数値部分を秒数に変換し、基準時刻に加算する。
  • 算出された expires が r->request_time より過去の場合、r->request_time に補正する。

    • 3. **ヘッダー設定**:

  • Cache-Control: max-age=<seconds> を t に追加（max-age は現在時刻からの差分）。
  • Expires ヘッダーを RFC 822 形式の文字列で t に設定。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_expires.c:519（expires_filter 関数内）。
  • **呼び出し先**:
  • apr_time_from_sec, apr_time_sec, apr_psprintf, apr_palloc, apr_rfc822_date, apr_table_mergen, apr_table_setn (APR API)
  • ap_log_rerror (Apache Core API)

関連データ構造

  • request_rec: リクエスト情報およびファイル情報（finfo）を保持。
  • apr_table_t: HTTP ヘッダーの格納用テーブル。

失敗モードと境界条件

  • code が 'M' でファイルが存在しない場合: DECLINED を返し、ヘッダー操作を行わない。
  • code が 'M'/'A' 以外の場合: HTTP_INTERNAL_SERVER_ERROR を返し、エラーログを出力する。
  • expires が現在時刻より過去の場合: r->request_time に切り上げられる。

保守担当者向けメモ

  • atoi() を使用しているため、非常に大きな数値や不正なフォーマットに対する堅牢性は限定的である。
  • 本関数は Expires ヘッダーの有無をチェックせず、呼び出し元（expires_filter）が既にヘッダーが存在しないことを確認している前提で動作する。
