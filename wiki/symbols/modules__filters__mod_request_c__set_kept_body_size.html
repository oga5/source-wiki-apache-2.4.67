set_kept_body_size

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_request.c:351

目的

Apache HTTP Server の設定ディレクティブ KeptBodySize の値を解析し、ディレクトリごとの設定構造体 request_dir_conf に反映させる。

シグネチャまたは形状

static const char *set_kept_body_size(cmd_parms *cmd, void *dconf, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dconf: 設定対象の request_dir_conf 構造体へのポインタ。
  • arg: KeptBodySize ディレクティブに指定された文字列値。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを示す文字列（const char *）を返す。

前提条件と不変条件

  • dconf は request_dir_conf 型としてキャスト可能であること。
  • arg は数値として解釈可能な文字列であること。

アルゴリズムまたは振る舞い

1. apr_strtoff を使用して、arg を long 型の数値に変換する。 2. 変換結果が APR_SUCCESS であり、かつ値が 0 以上であることを検証する。 3. 文字列の末尾に不正な文字が含まれていないか（*end が終端文字か）を確認する。 4. 検証に成功した場合、conf->keep_body に値を格納し、conf->keep_body_set を 1 に設定する。 5. 検証に失敗した場合は、エラーメッセージを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_request.c:367 において、AP_INIT_TAKE1 マクロを通じて設定登録時に呼び出される。
  • **呼び出し先**: apr_strtoff（APR ライブラリ関数）。

関連データ構造

  • request_dir_conf: mod_request モジュールが使用するディレクトリごとの設定構造体。
  • cmd_parms: Apache の設定解析用パラメータ構造体。

失敗モードと境界条件

  • **負の値**: conf->keep_body < 0 の場合、エラーメッセージを返す。
  • **不正な形式**: 数値として解析できない文字列や、数値の後にゴミデータが含まれる場合、エラーメッセージを返す。
  • **解析失敗**: apr_strtoff が APR_SUCCESS を返さない場合、エラーメッセージを返す。

保守担当者向けメモ

  • 本関数は KeptBodySize ディレクティブのバリデーションを担う。設定値の変更は request_dir_conf 構造体の keep_body フィールドに直接反映される。
