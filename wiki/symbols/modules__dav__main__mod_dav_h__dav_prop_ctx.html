dav_prop_ctx

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1767

目的

WebDAV プロパティの変更操作（検証、実行、コミット、ロールバック）を管理するための 3 フェーズ処理用コンテキスト構造体です。

シグネチャまたは形状

typedef struct dav_prop_ctx {
    dav_propdb *propdb;
    apr_xml_elem *prop;
    int operation;
    int is_liveprop;
    void *liveprop_ctx;
    struct dav_rollback_item *rollback;
    dav_error *err;
    request_rec *r;
} dav_prop_ctx;

入力と出力

  • **入力**: propdb（プロパティデータベース）、prop（操作対象の XML 要素）、operation（DAV_PROP_OP_SET または DAV_PROP_OP_DELETE）、および request_rec（リクエスト情報）。
  • **出力**: err フィールドに操作の結果として発生した dav_error が格納されます。

前提条件と不変条件

  • operation フィールドは DAV_PROP_OP_SET (1) または DAV_PROP_OP_DELETE (2) のいずれかである必要があります。
  • r（request_rec）は mod_dav.c 内部での処理に使用されます。

アルゴリズムまたは振る舞い

  • 本構造体は、プロパティ変更のライフサイクル全体を通じて状態を保持します。
  • dav_prop_validate、dav_prop_exec、dav_prop_commit の各関数に引数として渡され、トランザクション的な更新処理を制御します。
  • is_liveprop および liveprop_ctx は、ライブプロパティ固有の処理を識別・管理するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_dav.c 内のプロパティ処理ロジック（apr_array_make でリストを生成し、apr_array_push で追加）。
  • **呼び出し先**: dav_prop_validate、dav_prop_exec、dav_prop_commit（いずれも mod_dav.h で宣言）。
  • **補足**: dav_prop_log_errors や dav_process_ctx_list などの内部関数からも参照されます。

関連データ構造

  • dav_propdb: プロパティデータベースのハンドル。
  • apr_xml_elem: 操作対象の XML 要素。
  • dav_error: エラー情報を保持するスタック構造体。
  • dav_rollback_item: ロールバック情報を保持する構造体。
  • request_rec: Apache のリクエスト情報。

失敗モードと境界条件

  • err フィールドが NULL でない場合、操作中にエラーが発生したことを示します。
  • operation に未定義の値が設定された場合の挙動は未定義です。

保守担当者向けメモ

  • operation 定義に GET 操作を追加する可能性がコメントとして示唆されています。
  • mod_dav.c 内部の ctx_list を介して複数のプロパティ操作をバッチ処理する設計となっています。
