hm_slot_server_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/cluster/mod_heartmonitor.c:70

目的

共有メモリ（slotmem）上のサーバー状態を操作・検索する際のコンテキスト情報を保持するために使用されます。

シグネチャまたは形状

typedef struct hm_slot_server_ctx_t {
  hm_server_t *s;         /* サーバー状態へのポインタ */
  int found;              /* 検索結果フラグ */
  unsigned int item_id;   /* 共有メモリ内のアイテムID */
} hm_slot_server_ctx_t;

入力と出力

本構造体は、slotmem のコールバック関数や検索処理において、外部から渡されたデータ（data）をキャストして利用されます。

  • 入力: 共有メモリの走査時に、対象サーバーの特定や状態更新のために使用されます。
  • 出力: 検索処理の結果（found）や、更新対象のサーバー情報（s）を保持します。

前提条件と不変条件

  • s は有効な hm_server_t 構造体を指している必要があります。
  • item_id は slotmem プロバイダーが管理する有効なインデックスと整合している必要があります。

アルゴリズムまたは振る舞い

主に slotmem のイテレータや検索関数内で、特定のサーバー情報を特定するための「検索用コンテキスト」として機能します。 1. 検索対象の条件を保持した状態で slotmem の走査を開始します。 2. 走査中に一致するサーバーが見つかると、found フラグを立て、対象の s を更新します。

関連データ構造

  • hm_server_t: 個々のバックエンドサーバーの稼働状態を保持する構造体。本構造体から参照されます。

失敗モードと境界条件

  • s が NULL の状態でアクセスされた場合、セグメンテーション違反が発生する可能性があります。
  • item_id が slotmem の確保済み領域外を指している場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は mod_heartmonitor.c 内の slotmem 操作における一時的な状態保持用です。
  • 構造体の各フィールドが slotmem のコールバック関数内でどのように更新されるかについては、各参照箇所のロジックを確認してください。
