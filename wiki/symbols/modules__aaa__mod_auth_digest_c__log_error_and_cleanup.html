log_error_and_cleanup

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_digest.c:232

目的

共有メモリやロックの初期化・設定中にエラーが発生した際、エラーログを出力し、Digest 認証における nonce カウントチェックおよびワンタイム nonce 機能を無効化するためにリソースを解放する。

シグネチャまたは形状

static void log_error_and_cleanup(char *msg, apr_status_t sts, server_rec *s)

入力と出力

  • msg (char *): ログに記録するエラーメッセージ。
  • sts (apr_status_t): 発生したエラーのステータスコード。
  • s (server_rec *): エラーログ出力先となるサーバーレコード。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • cleanup_tables(NULL) が呼び出されることで、共有メモリ上のテーブルがクリーンアップされる。
  • 本関数が実行された後、Digest 認証の nonce 関連機能は無効化された状態となる。

アルゴリズムまたは振る舞い

1. ap_log_error を使用し、指定されたメッセージとステータスコードをエラーログ（APLOG_ERR）に記録する。ログには「nonce-count checking and one-time nonces disabled」という旨が追記される。 2. cleanup_tables(NULL) を呼び出し、共有メモリ上のテーブルを解放・初期化する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/aaa/mod_auth_digest.c 内の初期化処理（initialize_module や initialize_child に相当する箇所）において、共有メモリ（SHM）、メモリ管理（RMM）、またはミューテックス（client_lock, opaque_lock）の生成・アタッチに失敗した際に呼び出される。
  • 呼び出し先:
  • ap_log_error (Apache HTTP Server API)
  • cleanup_tables (同ファイル内)

関連データ構造

  • server_rec: サーバー設定情報を保持する構造体。
  • apr_status_t: APR（Apache Portable Runtime）のステータスコード。

失敗モードと境界条件

  • msg が NULL の場合、ap_log_error の動作に依存する。
  • cleanup_tables(NULL) は内部で共有メモリの解放を行うため、既に解放済みであるか、あるいは初期化が不完全な状態で呼び出された場合の挙動は cleanup_tables の実装に依存する。

保守担当者向けメモ

  • 本関数は、モジュールの初期化フェーズで致命的なリソース確保失敗が発生した際の安全な終了処理として機能する。
  • 呼び出し元で渡される sts には、APR のエラーコードだけでなく、メモリ確保失敗時などに -1 が渡されるケースがあるため注意が必要。
