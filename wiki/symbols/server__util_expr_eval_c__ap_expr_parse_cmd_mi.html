ap_expr_parse_cmd_mi

定義

  • 種別: function
  • 定義箇所: server/util_expr_eval.c:428

目的

設定コマンドのコンテキスト内で式を解析し、評価に必要な情報を保持する ap_expr_info_t 構造体を生成する。

シグネチャまたは形状

ap_expr_info_t* ap_expr_parse_cmd_mi(const cmd_parms *cmd,
                                     const char *expr,
                                     unsigned int flags,
                                     const char **err,
                                     ap_expr_lookup_fn_t *lookup_fn,
                                     int module_index)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • expr: 解析対象の式文字列。
  • flags: 式の動作を制御するフラグ。
  • lookup_fn: 変数や関数を解決するためのコールバック関数。
  • module_index: モジュール識別子。
  • **出力**:
  • 戻り値: 解析済みの ap_expr_info_t 構造体へのポインタ。失敗時は NULL。
  • err: 解析エラーが発生した場合、エラーメッセージが格納される。

前提条件と不変条件

  • cmd->pool からメモリが割り当てられる。
  • err は呼び出し元で提供されたポインタであること。

アルゴリズムまたは振る舞い

1. cmd->pool を使用して ap_expr_info_t 構造体を確保する。 2. cmd から取得したファイル名と行番号、および引数で渡された flags と module_index を構造体に設定する。 3. ap_expr_parse を呼び出し、式を解析して抽象構文木（AST）を構築する。 4. ap_expr_parse がエラーを返した場合（*err が非 NULL）、NULL を返す。 5. 正常終了時は、構築された info 構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: include/ap_expr.h 内のラッパー関数（APLOG_MODULE_INDEX を渡すもの）から参照されている。
  • **呼び出し先**: ap_expr_parse（式解析の実行）。

関連データ構造

  • ap_expr_info_t: 式の評価に必要なメタデータと AST を保持する構造体。
  • cmd_parms: 設定処理中のコマンドパラメータ。

失敗モードと境界条件

  • ap_expr_parse が構文エラーを検出した場合、*err にエラー文字列が設定され、関数は NULL を返す。

保守担当者向けメモ

  • 本関数は ap_expr_parse の設定フェーズにおけるラッパーであり、モジュールごとのログ記録用インデックス（module_index）を明示的に管理するために使用される。
