dav_provider

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:678

目的

WebDAV の各機能（リポジトリ、プロパティDB、ロック、バージョン管理、バインディング、検索）を実装するプロバイダーのフック関数群を束ねるための構造体です。

シグネチャまたは形状

typedef struct {
    const dav_hooks_repository *repos;
    const dav_hooks_propdb *propdb;
    const dav_hooks_locks *locks;
    const dav_hooks_vsn *vsn;
    const dav_hooks_binding *binding;
    const dav_hooks_search *search;
    void *ctx;
} dav_provider;

入力と出力

  • 入力: 各機能に対応する dav_hooks_* 構造体へのポインタおよび、プロバイダー固有のコンテキストデータ (ctx)。
  • 出力: なし（構造体定義）。

前提条件と不変条件

  • 各フック構造体は const として定義され、プロバイダーの生存期間中不変であることが期待されます。
  • ctx フィールドは、プロバイダーが実行時に必要とする状態を保持するために使用されます。

アルゴリズムまたは振る舞い

  • mod_dav は、リクエスト処理中に dav_get_provider を通じてこの構造体を取得し、各メソッドの実行に必要なフック関数を呼び出します。
  • ライブプロパティやレポート拡張は、本構造体には含まれません。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/dav/fs/repos.c: dav_fs_provider の定義に使用。
  • modules/dav/main/mod_dav.c: dav_get_provider で取得。
  • modules/dav/main/providers.c: プロバイダーの登録・検索処理で使用。
  • 呼び出し先:
  • 構造体内に保持する各 dav_hooks_* 構造体（repository, propdb, locks, vsn, binding, search）。

関連データ構造

  • dav_hooks_repository, dav_hooks_propdb, dav_hooks_locks, dav_hooks_vsn, dav_hooks_binding, dav_hooks_search

失敗モードと境界条件

  • 必須ではないフック（バージョン管理など）が実装されていない場合、対応するポインタは NULL に設定されます。呼び出し側はこれを確認する必要があります。

保守担当者向けメモ

  • 本構造体は mod_dav のコア API の中心です。新しいフックカテゴリを追加する場合は、本構造体へのフィールド追加と、関連する登録・検索ロジックの更新が必要です。
  • ctx フィールドは、プロバイダーがリクエストをまたいで状態を保持する必要がある場合に活用されます。
