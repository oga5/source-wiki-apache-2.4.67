ap_register_provider

定義

  • 種別: function
  • 定義箇所: server/provider.c:35

目的

指定されたグループ、名前、バージョンをキーとして、任意のプロバイダー（関数ポインタやデータ構造）をグローバルなレジストリに登録します。

シグネチャまたは形状

apr_status_t ap_register_provider(apr_pool_t *pool,
                                  const char *provider_group,
                                  const char *provider_name,
                                  const char *provider_version,
                                  const void *provider)

入力と出力

  • **入力**:
  • pool: 登録に使用するメモリプール。
  • provider_group: プロバイダーのグループ名。
  • provider_name: プロバイダーの名称。
  • provider_version: プロバイダーのバージョン文字列。
  • provider: 登録するプロバイダーの実体（関数ポインタ等）。
  • **出力**:
  • APR_SUCCESS: 登録成功。

前提条件と不変条件

  • global_providers および global_providers_names が未初期化の場合、渡された pool を使用して初期化され、プロセス終了時のクリーンアップ処理が登録されます。
  • 登録は3階層（グループ → 名前 → バージョン）のハッシュ構造として管理されます。
  • 同一キーで登録が行われた場合、既存のプロバイダーは上書きされます。

アルゴリズムまたは振る舞い

1. グローバルなハッシュテーブルが未初期化であれば、apr_hash_make で作成し、クリーンアップフックを登録します。 2. global_providers ハッシュに対し、グループ、名前、バージョンの順で階層を辿り、存在しない場合は apr_hash_make で新規作成して格納します。 3. 最終的な階層に provider ポインタを格納します。 4. 検索効率化のため、global_providers_names ハッシュに対しても同様の階層構造を作成し、プロバイダー名を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c
  • modules/cache/mod_cache_socache.c
  • modules/cache/mod_socache_dbm.c
  • modules/cache/mod_socache_dc.c
  • modules/cache/mod_socache_memcache.c
  • modules/cache/mod_socache_redis.c
  • modules/cache/mod_socache_shmcb.c
  • modules/core/mod_watchdog.c
  • modules/dav/lock/mod_dav_lock.c
  • modules/dav/main/providers.c
  • **呼び出し先**:
  • apr_hash_make, apr_hash_get, apr_hash_set (APRライブラリ)
  • apr_pool_cleanup_register (APRライブラリ)
  • cleanup_global_providers (内部関数)

関連データ構造

  • global_providers: プロバイダーの実体を保持する階層型ハッシュテーブル。
  • global_providers_names: プロバイダーのメタデータを逆引きするためのハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時には apr_hash_make 等の内部関数がエラーを返す可能性があります。
  • 実行時の動的な登録・削除は想定されておらず、マルチスレッド環境での同時実行には注意が必要です。

保守担当者向けメモ

  • 本関数は Apache の起動時（設定読み込みフェーズ）に実行されることを前提としています。
  • 登録されたプロバイダーのライフサイクルは、渡された pool の生存期間に依存します。
