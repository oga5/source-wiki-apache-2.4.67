ident_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_ident.c:81

目的

mod_ident モジュールにおけるディレクトリ単位の設定情報を保持するための構造体です。IdentityCheck および IdentityCheckTimeout ディレクティブによって設定される値を格納します。

シグネチャまたは形状

typedef struct {
    int do_rfc1413;
    int timeout_unset;
    apr_time_t timeout;
} ident_config_rec;

入力と出力

本構造体は設定の保持を目的としており、直接的な関数入力や出力は持ちません。設定の読み込み時（set_idcheck, set_timeout）に値が書き込まれ、ap_ident_lookup 等の処理実行時に参照されます。

前提条件と不変条件

  • timeout_unset は、タイムアウト値が明示的に設定されていない状態を識別するために使用されます。
  • timeout は apr_time_t 型であり、ネットワーク通信時の待機時間として利用されます。

アルゴリズムまたは振る舞い

本構造体は、Apache の設定マージプロセスを通じて、ディレクトリ階層間で設定を継承・統合するために使用されます。create_ident_dir_config で初期化され、merge_ident_dir_config で親ディレクトリの設定とマージされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_ident_dir_config (初期化)
  • merge_ident_dir_config (マージ)
  • set_idcheck / set_timeout (設定値の更新)
  • ap_ident_lookup (設定の参照)
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • request_rec: Apache のリクエストコンテキスト。本構造体は ap_get_module_config を介してリクエストに関連付けられた設定として取得されます。

失敗モードと境界条件

  • 設定が未定義の場合、timeout_unset フラグによりデフォルトの動作が選択されます。
  • メモリ割り当て失敗時は apr_palloc / apr_pcalloc の挙動に従い、Apache のメモリ管理機構によって処理されます。

保守担当者向けメモ

  • 本構造体は mod_ident の設定管理の中核です。新しい設定項目を追加する場合は、create および merge 関数での初期化・コピー処理を忘れずに追加してください。
