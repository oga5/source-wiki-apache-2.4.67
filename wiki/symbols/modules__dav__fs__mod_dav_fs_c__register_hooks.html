register_hooks

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/mod_dav_fs.c:88

目的

Apache HTTP Server のモジュール初期化フェーズにおいて、ファイルシステムベースの WebDAV バックエンドに必要なフックを登録し、mod_dav へプロバイダーを登録する。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p (モジュールのライフサイクルに関連付けられたメモリプール)
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • Apache のモジュールロードプロセスにより、サーバー起動時に呼び出される。
  • mod_dav が提供するフック登録関数が利用可能であること。

アルゴリズムまたは振る舞い

1. dav_hook_gather_propsets を呼び出し、dav_fs_gather_propsets をプロパティセット収集フックとして登録する。 2. dav_hook_find_liveprop を呼び出し、dav_fs_find_liveprop をライブプロパティ検索フックとして登録する。 3. dav_hook_insert_all_liveprops を呼び出し、dav_fs_insert_all_liveprops を全ライブプロパティ挿入フックとして登録する。 4. dav_fs_register(p) を呼び出し、ファイルシステムバックエンドを mod_dav に登録する。

  • すべてのフック登録において APR_HOOK_MIDDLE が指定され、標準的な実行順序で処理される。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュールロード機構（直接の呼び出し元はソースコード上では明示されていないが、http_config.h の定義に基づきモジュール構造体から参照される）。
  • 呼び出し先:
  • dav_hook_gather_propsets
  • dav_hook_find_liveprop
  • dav_hook_insert_all_liveprops
  • dav_fs_register

関連データ構造

  • apr_pool_t: リソース管理用メモリプール。

失敗モードと境界条件

  • フック登録関数が失敗した場合の挙動は mod_dav の実装に依存する。本関数自体はエラーコードを返さない。

保守担当者向けメモ

  • 本関数は mod_dav_fs モジュールのエントリーポイントの一部である。フックの追加やバックエンド登録の変更が必要な場合は、この関数を更新すること。
  • 呼び出し先である dav_fs_gather_propsets 等の各関数は、mod_dav_fs.c 内で定義されている。
