enter_ping_state

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_proxy_session.c:122

目的

HTTP/2 プロキシセッションにおける PING 状態の遷移を管理し、タイムアウト設定および PING フレームの送信を制御します。

シグネチャまたは形状

static void enter_ping_state(h2_proxy_session *session, h2_ping_state_t state)

入力と出力

  • session: 操作対象の h2_proxy_session 構造体へのポインタ。
  • state: 遷移先の h2_ping_state_t 型の状態値。
  • 戻り値: なし（void）。

前提条件と不変条件

  • session->ping_state が既に state と一致している場合、関数は即座に終了します。
  • H2_PING_ST_NONE 以外の状態間での直接的な遷移は行われません。

アルゴリズムまたは振る舞い

1. **状態チェック**: 現在の session->ping_state が state と同じであれば何もせず終了します。 2. **NONE からの遷移**:

  • 遷移先が H2_PING_ST_AWAIT_PING の場合、既存のタイムアウトを解除し、nghttp2_submit_ping を用いて PING フレームを送信します。
  • タイムアウトを設定し、session->ping_state を更新します。

    • 3. **NONE への遷移**:

  • 既存のタイムアウトを解除し、ping_arrived(session) を呼び出して PING 到着処理を実行した後、状態を H2_PING_ST_NONE に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_session.c 内の各所（H2_PING_ST_AWAIT_ANY、H2_PING_ST_AWAIT_PING、H2_PING_ST_NONE への遷移時）。
  • **呼び出し先**:
  • unset_ping_timeout
  • set_ping_timeout
  • nghttp2_submit_ping
  • ping_arrived

関連データ構造

  • h2_proxy_session: セッション状態を保持するメイン構造体。
  • h2_ping_state_t: PING 状態を定義する列挙型。

失敗モードと境界条件

  • H2_PING_ST_NONE 以外の状態から別の非 NONE 状態への遷移は無視されます。

保守担当者向けメモ

  • 本関数は nghttp2 の PING フレーム送信と Apache 側のタイムアウト管理を同期させる役割を持ちます。
  • ping_arrived が呼び出されるのは、状態が H2_PING_ST_NONE に戻る際のみです。
