ssl_config_server_create

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_config.c:247

目的

Apache HTTP Server の仮想ホスト（Server）スコープにおける SSL 設定構造体（SSLSrvConfigRec）を生成および初期化します。

シグネチャまたは形状

void *ssl_config_server_create(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定構造体の割り当てに使用する apr_pool_t。
  • s: 設定対象となるサーバーレコード（server_rec）。
  • **出力**:
  • void *: 生成・初期化された SSLSrvConfigRec 構造体へのポインタ。

前提条件と不変条件

  • apr_pool_t が有効であること。
  • server_rec が有効であること。
  • 内部で呼び出される ssl_config_server_new および ssl_config_global_create が正常にメモリを割り当て、初期化できること。

アルゴリズムまたは振る舞い

1. ssl_config_server_new(p) を呼び出し、新しいサーバー設定構造体 SSLSrvConfigRec を生成します。 2. ssl_config_global_create(s) を呼び出し、グローバル設定構造体（SSLModConfigRec）を取得または生成して、サーバー設定内の mc メンバに割り当てます。 3. 初期化済みの SSLSrvConfigRec 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/mod_ssl.c:767（仮想ホストごとの設定構造体作成用ハンドラとして登録）
  • **呼び出し先**:
  • ssl_config_server_new
  • ssl_config_global_create

関連データ構造

  • SSLSrvConfigRec: サーバー単位の SSL 設定を保持する構造体。
  • SSLModConfigRec: サーバープロセス全体で共有されるグローバルな SSL 設定構造体。
  • server_rec: Apache HTTP Server のサーバー設定レコード。

失敗モードと境界条件

  • メモリ不足時: apr_pool_t の割り当て失敗により、Apache HTTP Server のメモリ管理ポリシーに従ってプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は mod_ssl の設定フェーズで呼び出される初期化ロジックの一部です。
  • sc->mc へのグローバル設定の割り当ては、サーバーの生存期間を通じて共有される設定への参照を確立するために不可欠です。
