ap_dir_match_t

定義

  • 種別: struct
  • 定義箇所: include/httpd.h:2652

目的

ディレクトリ内のファイルやサブディレクトリを走査し、指定された条件に基づいてコールバック関数を実行するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct ap_dir_match_t ap_dir_match_t;

struct ap_dir_match_t {
    const char *(*cb)(ap_dir_match_t *w, const char *fname);
    /* 他の内部フィールドを含む */
};

入力と出力

  • **入力**: ap_dir_cfgmatch 関数を介して初期化されます。コールバック関数 cb は、走査対象のファイル名（fname）を引数として受け取ります。
  • **出力**: コールバック関数 cb は、処理結果を示す const char * を返します。

前提条件と不変条件

  • ap_dir_match_t 構造体は、ap_dir_cfgmatch を使用して cmd_parms から適切に初期化されている必要があります。
  • 走査対象のパスがファイルであるかディレクトリであるかによって、コールバックの実行条件や再帰処理の挙動が異なります。

アルゴリズムまたは振る舞い

1. ap_dir_fnmatch または ap_dir_nofnmatch 関数に本構造体を渡すことで、ディレクトリ走査を開始します。 2. 走査中、対象がファイルであればコールバック cb が呼び出されます。 3. 対象がディレクトリであり、かつ構造体内の再帰フラグが有効な場合、ディレクトリ内を再帰的に走査します。 4. 走査の過程で、オプションフラグの設定に応じて特定の条件で処理が中断または継続されます。

呼び出し元と呼び出し先

  • **呼び出し元**: include/http_config.h 内の ap_dir_cfgmatch 関数によって生成されます。
  • **呼び出し先**: ap_dir_fnmatch および ap_dir_nofnmatch 関数が、本構造体を引数として受け取り、内部のコールバック cb を実行します。

関連データ構造

  • cmd_parms: ap_dir_cfgmatch を通じて本構造体を生成する際に使用されます。

失敗モードと境界条件

  • コールバック関数がエラーを示す値を返した場合、走査処理は停止または特定の挙動を示します。
  • ap_dir_match_t 内のフラグ設定（再帰、オプション等）が不適切な場合、意図しない範囲のファイル走査が行われる可能性があります。

保守担当者向けメモ

  • 本構造体は include/httpd.h で定義されていますが、操作関数群は include/http_config.h に存在します。変更時は両ファイルの整合性に注意してください。
  • コールバック関数 cb のシグネチャを変更する場合は、ap_dir_fnmatch および ap_dir_nofnmatch の実装への影響を考慮してください。
