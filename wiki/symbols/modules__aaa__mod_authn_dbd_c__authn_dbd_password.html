authn_dbd_password

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_dbd.c:95

目的

Apache HTTP Server の AuthBasic 認証において、データベース（DBD）をバックエンドとして利用し、ユーザーのパスワードを検証する。

シグネチャまたは形状

static authn_status authn_dbd_password(request_rec *r, const char *user, const char *password)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • user: 認証対象のユーザー名。
  • password: クライアントから提供された平文パスワード。
  • **出力**:
  • authn_status: 認証結果（AUTH_GRANTED, AUTH_DENIED, AUTH_USER_NOT_FOUND, AUTH_GENERAL_ERROR のいずれか）。

前提条件と不変条件

  • mod_dbd が有効であり、データベース接続が確立可能であること。
  • 設定ファイルで AuthDBDUserPWQuery が定義されていること。
  • クエリ結果の最初のカラム（インデックス0）に、検証対象のパスワードまたはハッシュが含まれていること。
  • すべてのメモリ割り当ては r->pool から行われ、リクエスト終了時に自動的に解放される。

アルゴリズムまたは振る舞い

1. **接続取得**: authn_dbd_acquire_fn を呼び出し、データベース接続を取得する。失敗時は AUTH_GENERAL_ERROR を返す。 2. **クエリ実行**: 設定された AuthDBDUserPWQuery を apr_dbd_pvselect で実行する。 3. **結果処理**:

  • 最初のカラムをパスワードとして取得する。
  • 2番目以降のカラムが存在する場合、AUTHENTICATE_ プレフィックスを付与して環境変数（r->subprocess_env）にセットする。カラム名は英数字に正規化される。

    • 4. **検証**: apr_password_validate を使用して、提供されたパスワードとデータベースから取得したパスワードを比較する。 5. **キャッシュ**: 成功時、AUTHN_CACHE_STORE マクロを介して結果をキャッシュに保存する。 6. **戻り値**: 検証結果に基づき AUTH_GRANTED または AUTH_DENIED を返す。ユーザーが見つからない場合は AUTH_USER_NOT_FOUND を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_dbd.c:288 にて、認証プロバイダーとして登録されている（直接の呼び出し元は mod_authn_core 等のフレームワーク）。
  • **呼び出し先**:
  • ap_get_module_config: 設定取得。
  • authn_dbd_acquire_fn: データベース接続取得。
  • apr_dbd_pvselect, apr_dbd_get_row, apr_dbd_get_name, apr_dbd_get_entry: データベース操作。
  • apr_password_validate: パスワード比較。
  • AUTHN_CACHE_STORE: キャッシュ保存。

関連データ構造

  • request_rec: リクエスト情報およびメモリプール。
  • authn_dbd_conf: ディレクトリ単位の設定（SQL クエリ等）。
  • ap_dbd_t: データベース接続ハンドル。

失敗モードと境界条件

  • **データベース接続失敗**: AUTH_GENERAL_ERROR を返し、認証を中断する。
  • **クエリ未定義**: AuthDBDUserPWQuery が設定されていない場合、AUTH_GENERAL_ERROR を返す。
  • **ユーザー不在**: クエリ結果が空の場合、AUTH_USER_NOT_FOUND を返す。
  • **クエリ実行エラー**: データベースエラーが発生した場合、AUTH_GENERAL_ERROR を返す。

保守担当者向けメモ

  • SQL インジェクション対策として、クエリのパラメータバインディングは mod_dbd に依存している。
  • クエリ結果のループ処理において、apr_dbd_get_row の呼び出しを適切に行い、リソースリークを防ぐ必要がある。
  • 環境変数へのセット処理では、カラム名が AUTHENTICATE_ プレフィックス付きで大文字に変換される点に注意すること。
