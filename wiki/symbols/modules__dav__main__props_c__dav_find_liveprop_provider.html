dav_find_liveprop_provider

定義

  • 種別: function
  • 定義箇所: modules/dav/main/props.c:229

目的

指定された名前空間 URI とプロパティ名に基づき、対応するライブプロパティプロバイダーを特定し、そのプロパティ ID を返します。

シグネチャまたは形状

int dav_find_liveprop_provider(dav_propdb *propdb,
                               const char *ns_uri,
                               const char *propname,
                               const dav_hooks_liveprop **provider);

入力と出力

  • **入力**:
  • propdb: プロパティデータベースのコンテキスト。
  • ns_uri: プロパティの名前空間 URI。
  • propname: プロパティ名。
  • provider: プロバイダー構造体へのポインタを格納する出力用引数。
  • **出力**:
  • 成功時: プロパティ ID（正の整数）。
  • 失敗時: DAV_PROPID_CORE_UNKNOWN（プロバイダーが見つからない場合）。

前提条件と不変条件

  • ns_uri が NULL の場合、ライブプロパティプロバイダーは定義できないため、即座に DAV_PROPID_CORE_UNKNOWN を返します。
  • provider 引数は、関数開始時に必ず NULL に初期化されます。

アルゴリズムまたは振る舞い

1. ns_uri が NULL であればエラーを返します。 2. dav_run_find_liveprop を呼び出し、登録済みのライブプロパティプロバイダーを検索します。プロバイダーが見つかり、プロパティ ID が取得できた場合はその値を返します。 3. ライブプロパティで見つからない場合、ns_uri が "DAV:" 名前空間であれば、dav_core_props 配列を走査してコアプロパティとの一致を確認します。 4. いずれにも該当しない場合は DAV_PROPID_CORE_UNKNOWN を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/props.c:278
  • modules/dav/main/props.c:937
  • **呼び出し先**:
  • dav_run_find_liveprop (ライブプロパティ検索フック)
  • strcmp (標準 C ライブラリ)

関連データ構造

  • dav_propdb: プロパティ操作のコンテキスト。
  • dav_hooks_liveprop: ライブプロパティの機能を提供するフック構造体。
  • dav_core_props: DAV: 名前空間に属するコアプロパティ名のリスト。

失敗モードと境界条件

  • ns_uri が NULL の場合、検索を行わずに終了します。
  • 該当するプロパティがライブプロパティにもコアプロパティにも存在しない場合、DAV_PROPID_CORE_UNKNOWN が返されます。

保守担当者向けメモ

  • 本関数はライブプロパティを優先的に検索し、その後にコアプロパティをチェックする順序になっています。
  • dav_core_props の定義については props.c 内の静的定義を参照してください。
