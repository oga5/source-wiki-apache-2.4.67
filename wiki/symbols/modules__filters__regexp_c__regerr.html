regerr

定義

  • 種別: function
  • 定義箇所: modules/filters/regexp.c:70

目的

正規表現のコンパイル時に発生したエラーコードを、人間が読める形式のメッセージに変換し、comperr を介して報告する。

シグネチャまたは形状

static int regerr(sed_commands_t *commands, int err)

入力と出力

  • **入力**:
  • commands: エラー報告先となる sed_commands_t 構造体へのポインタ。
  • err: コンパイルエラーを示す整数コード。
  • **出力**:
  • 戻り値: 常に int 型を返すが、現在の実装では戻り値の具体的な意味は定義されていない。

前提条件と不変条件

  • commands は有効な sed_commands_t 構造体を指している必要がある。
  • err は regexp.c 内で定義された既知のエラーコード（11, 16, 25, 36, 41-46, 49, 50）のいずれか、またはそれ以外（デフォルト）として処理される。

アルゴリズムまたは振る舞い

1. err の値に基づき switch 文で分岐する。 2. 各エラーコードに対応するエラーメッセージ文字列を選択する。 3. comperr(commands, message) を呼び出し、エラーを記録する。 4. err が 0 の場合は何も行わず終了する。 5. 定義されていないエラーコードが渡された場合は、「Unknown regexp error code」という汎用メッセージを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/regexp.c:350 (直接的な呼び出しを確認)
  • **呼び出し先**: comperr (エラーメッセージの記録に使用)

関連データ構造

  • sed_commands_t: エラー報告のコンテキストを保持する構造体。

失敗モードと境界条件

  • err に 0 が渡された場合は正常系として処理され、エラー報告は行われない。
  • 未定義の err 値が渡された場合、default ケースにより「Unknown regexp error code」として処理される。

保守担当者向けメモ

  • 本関数は libsed のエラーハンドリングの一部であり、エラーコードの追加やメッセージの変更は regexp.c 内の switch 文を更新することで行う。
  • comperr への引数として渡される文字列には、%s が含まれている場合があるため、呼び出し元で適切なコンテキスト情報が補完されることを前提としている。
