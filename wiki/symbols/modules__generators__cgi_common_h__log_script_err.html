log_script_err

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:103

目的

CGIスクリプトの標準エラー出力（stderr）をパイプから行単位で読み取り、Apacheのエラーログへ転送する。

シグネチャまたは形状

static apr_status_t log_script_err(request_rec *r, apr_file_t *script_err)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • script_err: スクリプトの標準エラー出力に接続された apr_file_t 型のパイプ。
  • **出力**:
  • 戻り値: apr_status_t 型。読み取り完了時またはエラー発生時のステータスを返す。

前提条件と不変条件

  • r および script_err は有効なポインタであること。
  • cgi_module がサーバー設定に存在し、ap_get_module_config で取得可能であること。

アルゴリズムまたは振る舞い

1. cgi_module の設定情報を取得する。 2. apr_file_gets を使用して、script_err から HUGE_STRING_LEN を上限として行単位でデータを読み込む。 3. 読み込んだ行末の改行コード（\n）を検索し、CRLF（\r\n）が含まれる場合は適切に除去して終端文字（\0）に置換する。 4. 処理した行を log_scripterror 関数へ渡し、Apacheのエラーログへ書き出す。 5. apr_file_gets が APR_SUCCESS を返さなくなるまで（EOFまたはエラーまで）ループを継続する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/cgi_common.h: 395行目、492行目、546行目
  • modules/generators/mod_cgi.c: 216行目
  • **呼び出し先**:
  • ap_get_module_config
  • apr_file_gets
  • strchr
  • log_scripterror

関連データ構造

  • request_rec: Apacheのリクエスト処理用構造体。
  • apr_file_t: APRのファイル/パイプ操作用構造体。
  • cgi_server_conf: CGIモジュールの設定を保持する構造体。

失敗モードと境界条件

  • apr_file_gets が APR_SUCCESS 以外のステータス（EOFや読み取りエラー）を返した場合、ループを終了しそのステータスを返す。
  • 行の長さが HUGE_STRING_LEN を超える場合、apr_file_gets の仕様に従い分割して読み込まれる。

保守担当者向けメモ

  • log_scripterror 内で apr_stat が呼び出される場合があり、高負荷時のパフォーマンスに影響を与える可能性がある。
  • 本関数は static で定義されており、cgi_common.h をインクルードするモジュール内でのみ利用可能である。
