get_staging_certs_json

定義

  • 種別: function
  • 定義箇所: modules/md/md_status.c:229

目的

Managed Domain (MD) のステージング領域に存在する証明書情報を取得し、JSON 形式のレポートを生成する。

シグネチャまたは形状

apr_status_t get_staging_certs_json(md_json_t **pjson, const md_t *md, 
                                    md_reg_t *reg, apr_pool_t *p);

入力と出力

  • **入力**:
  • md: 対象の Managed Domain 構造体。
  • reg: MD レジストリハンドル。
  • p: メモリ割り当て用プール。
  • **出力**:
  • pjson: 生成された JSON オブジェクトへのポインタ。
  • 戻り値: apr_status_t 型のステータスコード（成功時は APR_SUCCESS）。

前提条件と不変条件

  • md および reg は有効なポインタであること。
  • p はメモリ割り当てのために有効なプールであること。

アルゴリズムまたは振る舞い

1. apr_array_make を使用して、証明書ポインタを格納する配列を初期化する。 2. get_cert_count(md, 1) で取得した数だけループし、各鍵仕様（md_pkey_spec_t）に対応する証明書を md_pubcert_load でステージング領域（MD_SG_STAGING）から読み込む。 3. 読み込みに成功した場合、証明書チェーンの先頭要素を配列に格納する。 4. 最後に status_get_certs_json を呼び出し、収集した証明書リストを JSON 構造に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_status.c:300
  • **呼び出し先**:
  • get_cert_count
  • md_pkeys_spec_get
  • md_pubcert_load
  • md_reg_store_get
  • APR_ARRAY_IDX
  • APR_ARRAY_PUSH
  • status_get_certs_json

関連データ構造

  • md_t: 管理対象ドメイン情報。
  • md_reg_t: レジストリ情報。
  • md_cert_t: 証明書データ構造。
  • md_json_t: JSON 出力用コンテナ。

失敗モードと境界条件

  • md_pubcert_load が失敗した場合、その特定の証明書は NULL として配列に格納される。
  • status_get_certs_json の戻り値が関数の最終的なステータスとなる。

保守担当者向けメモ

  • 本関数はステージング領域の証明書のみを対象とする。
  • get_cert_count の第2引数に 1 を渡しており、ステージング状態のカウントを明示的に指定している。
