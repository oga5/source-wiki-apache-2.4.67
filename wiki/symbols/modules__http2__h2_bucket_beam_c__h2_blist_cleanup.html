h2_blist_cleanup

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:250

目的

指定された h2_blist 構造体内に保持されているすべての apr_bucket を削除し、リストを空にするために使用されます。

シグネチャまたは形状

static void h2_blist_cleanup(h2_blist *bl)

入力と出力

  • **入力**: bl - クリーンアップ対象の h2_blist へのポインタ。
  • **出力**: なし（副作用としてリスト内の全バケットが破棄されます）。

前提条件と不変条件

  • bl は有効な h2_blist 構造体を指している必要があります。
  • リスト内の各バケットは apr_bucket_delete を呼び出すことで安全に破棄可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. H2_BLIST_EMPTY(bl) を用いてリストが空になるまでループを実行します。 2. ループ内で H2_BLIST_FIRST(bl) を呼び出し、リストの先頭にあるバケットを取得します。 3. apr_bucket_delete(e) を呼び出し、バケットを破棄します。 4. リストが空になった時点で処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_bucket_beam.c:277 にて &beam->buckets_to_send を引数として呼び出されています。
  • **呼び出し先**: apr_bucket_delete（APR API）。

関連データ構造

  • h2_blist: APR_RING をラップした軽量なリスト構造。
  • apr_bucket: APR が提供するデータバケット構造体。

失敗モードと境界条件

  • リストが既に空である場合、ループは実行されず何も行われません。
  • apr_bucket_delete の内部実装に依存し、バケットの参照カウントが正しく管理されていない場合、メモリリークや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数は h2_bucket_beam 内の内部的なクリーンアップ処理です。
  • h2_blist は動的メモリ確保を行わない構造であるため、リスト自体のメモリ解放は行わず、保持しているバケット要素のみを解放します。
