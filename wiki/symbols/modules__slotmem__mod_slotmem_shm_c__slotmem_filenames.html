slotmem_filenames

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:87

目的

共有メモリおよび永続化ファイルに使用するファイルパスを生成します。

シグネチャまたは形状

static int slotmem_filenames(apr_pool_t *pool,
                             const char *slotname,
                             const char **filename,
                             const char **persistname)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用するプール。
  • slotname: スロットの名前（識別子）。
  • filename: 生成された共有メモリ用ファイルパスを格納するポインタのポインタ。
  • persistname: 生成された永続化用ファイルパスを格納するポインタのポインタ（NULL可）。
  • **出力**:
  • 戻り値: int 型。ファイル名が生成された場合は 1、それ以外は 0 を返します。
  • *filename および *persistname に生成されたパス文字列が設定されます。

前提条件と不変条件

  • slotname が "none" 以外である必要があります。
  • 相対パスが指定された場合、AP_MPMQ_GENERATION を用いてMPM世代ごとに一意なファイル名が生成されます。
  • 絶対パスが指定された場合、パスの加工は行われず、そのまま使用されます。

アルゴリズムまたは振る舞い

1. slotname が有効か確認し、無効な場合は NULL を返します。 2. 相対パスの場合、DEFAULT_SLOTMEM_PREFIX、slotname、MPM世代番号、DEFAULT_SLOTMEM_SUFFIX を結合し、ap_runtime_dir_relative で実行時ディレクトリ配下のパスに変換します。 3. 絶対パスの場合、入力された slotname をそのまま使用します。 4. persistname が指定されている場合、同様のルールで永続化用ファイルパス（DEFAULT_SLOTMEM_PERSIST_SUFFIX を付与）を生成します。 5. 最後に生成されたパスを引数経由で返し、成功時に 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/slotmem/mod_slotmem_shm.c:361 (slotmem_create 内)
  • modules/slotmem/mod_slotmem_shm.c:491 (slotmem_attach 内)
  • **呼び出し先**:
  • ap_os_is_path_absolute
  • ap_mpm_query
  • apr_psprintf
  • ap_runtime_dir_relative
  • apr_pstrcat

関連データ構造

  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • slotname が NULL、空文字列、または "none" の場合、ファイルパスは生成されず 0 が返されます。
  • persistname が NULL の場合、永続化用パスの生成処理はスキップされます。

保守担当者向けメモ

  • 本関数はMPM世代管理に依存しており、サーバー再起動時のファイル競合を防ぐ設計となっています。
  • 絶対パス指定時は一意性の保証が呼び出し側に委ねられるため、利用時には注意が必要です。
