socache_rd_create

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_redis.c:76

目的

Redis バックエンドを使用する共有オブジェクトキャッシュ（socache）インスタンスの初期化と、接続先サーバーリストの構成を行う。

シグネチャまたは形状

static const char *socache_rd_create(ap_socache_instance_t **context,
                                     const char *arg,
                                     apr_pool_t *tmp, apr_pool_t *p)

入力と出力

  • **入力**:
  • context: インスタンス構造体を格納するためのポインタ。
  • arg: Redis サーバーリストを含む設定文字列。
  • tmp: 一時的なメモリ割り当て用プール。
  • p: インスタンスの生存期間を管理するメモリプール。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ（const char *）を返す。
  • *context: 初期化された ap_socache_instance_t 構造体へのポインタが設定される。

前提条件と不変条件

  • p プールは、インスタンスの生存期間中有効であること。
  • arg には有効な Redis サーバーリストの文字列が含まれている必要がある。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して ap_socache_instance_t 構造体をメモリプール p 上に確保する。 2. arg が NULL または空文字列であるかを確認し、該当する場合はエラーメッセージを返して終了する。 3. arg に指定されたサーバーリスト文字列を apr_pstrdup を用いてインスタンス構造体の servers メンバにコピーする。 4. 正常終了として NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_redis.c:390 にて参照されている（直接的な呼び出し関係の詳細はソース抜粋からは不明）。
  • **呼び出し先**: apr_pcalloc, apr_pstrdup（APR ライブラリ関数）。

関連データ構造

  • ap_socache_instance_t: Redis インスタンスのコンテキストを保持する構造体。本関数で servers メンバが初期化される。

失敗モードと境界条件

  • arg が指定されていない、または空文字列の場合、エラーメッセージを返し処理を中断する。
  • メモリ不足時は apr_pcalloc または apr_pstrdup が例外（アボート）を発生させる可能性がある。

保守担当者向けメモ

  • 本関数はインスタンスのメモリ確保とサーバーリストの保存のみを行う。実際の Redis 接続やサーバーリストの解析は、後続の socache_rd_init で行われる。
