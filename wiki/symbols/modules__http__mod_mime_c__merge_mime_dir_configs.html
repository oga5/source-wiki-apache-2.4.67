merge_mime_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/http/mod_mime.c:194

目的

親ディレクトリと子ディレクトリの mime_dir_config 設定をマージし、新しい設定構造体を生成します。

シグネチャまたは形状

static void *merge_mime_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • basev: 親ディレクトリの設定（mime_dir_config *）。
  • addv: 子ディレクトリの設定（mime_dir_config *）。
  • **出力**: マージされた新しい mime_dir_config 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は mime_dir_config 型としてキャスト可能であること。
  • apr_pool_t は有効なメモリプールであること。
  • マージ後、new->remove_mappings は常に NULL にリセットされます。

アルゴリズムまたは振る舞い

1. **拡張子マッピングのマージ**:

  • 両方にマッピングが存在する場合、apr_hash_merge を使用して overlay_extension_mappings 関数を介して結合します。
  • 片方のみ存在する場合はそのポインタを継承します。ただし、add 側に削除対象（remove_mappings）がある場合は、安全のため apr_hash_copy で複製を作成します。

    • 2. **削除処理**: add->remove_mappings が存在する場合、remove_items を呼び出してマッピングから該当項目を除去します。 3. **設定値の継承**:

  • default_language および multimatch は、add 側の値が設定されていればそれを優先し、未設定なら base 側の値を採用します。
  • use_path_info は、add 側のフラグ（ビット2）が立っていない場合にのみ add の値を採用し、それ以外は base の値を継承します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/mod_mime.c:1032（モジュール設定の登録時）。
  • **呼び出し先**:
  • apr_palloc, apr_hash_merge, apr_hash_copy
  • overlay_extension_mappings
  • remove_items

関連データ構造

  • mime_dir_config: ディレクトリごとの設定を保持する構造体。
  • extension_info: 拡張子ごとのメタデータ。

失敗モードと境界条件

  • メモリ不足時には apr_palloc 等がエラーを発生させる可能性があります。
  • base または add のハッシュテーブルが NULL の場合、適切にフォールバックしてマージを継続します。

保守担当者向けメモ

  • use_path_info のマージロジックはビット演算に依存しているため、フラグ定義の変更時には注意が必要です。
  • remove_mappings はマージプロセス終了後にクリアされるため、この関数を複数回呼び出すと意図しない挙動になる可能性があります。
