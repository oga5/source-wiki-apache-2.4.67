fs_get_fname

定義

  • 種別: function
  • 定義箇所: modules/md/md_store_fs.c:405

目的

ファイルシステム上の特定のデータ要素（証明書、鍵、メタデータ等）を指す絶対パスを生成します。

シグネチャまたは形状

static apr_status_t fs_get_fname(const char **pfname, 
                                 md_store_t *store, md_store_group_t group, 
                                 const char *name, const char *aspect, 
                                 apr_pool_t *p)

入力と出力

  • pfname: 生成されたパス文字列を格納するポインタへのポインタ（出力）。
  • store: ファイルシステム実装の基底となる md_store_t 構造体。
  • group: ファイルが属するグループ（md_store_group_t）。
  • name: 対象となるオブジェクト名。
  • aspect: ファイルの側面（拡張子や役割）。
  • p: パス文字列の割り当てに使用する apr_pool_t。
  • 戻り値: APR_SUCCESS またはエラーコード。

前提条件と不変条件

  • store は FS_STORE() マクロを通じて md_store_fs_t へキャスト可能であること。
  • p は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. store からファイルシステムのベースパスを取得します。 2. group が MD_SG_NONE の場合、ベースパスと aspect を結合したパスを生成します。 3. それ以外の場合、ベースパス、グループ名、name、aspect を結合した階層パスを生成します。 4. パスの結合には md_util_path_merge を使用します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/md/md_store_fs.c 内の md_store_fs_init（関数ポインタの代入）
  • modules/md/md_store_fs.c 内のファイル操作関数（fs_load, fs_save, fs_move 等の内部処理）
  • 呼び出し先:
  • md_util_path_merge
  • md_store_group_name

関連データ構造

  • md_store_t: ストレージの抽象インターフェース。
  • md_store_fs_t: ファイルシステム実装の内部状態を保持する構造体。

失敗モードと境界条件

  • md_util_path_merge が失敗した場合、そのエラーコードを返します。
  • group が MD_SG_NONE の場合とそれ以外で、パス生成の階層構造が異なります。

保守担当者向けメモ

  • 本関数は md_store_fs_t の base パスに依存しています。
  • パス生成は apr_pool_t を使用するため、呼び出し側で適切なライフサイクルのプールを渡す必要があります。
