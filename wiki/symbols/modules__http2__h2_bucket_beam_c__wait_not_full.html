wait_not_full

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:221

目的

h2_bucket_beam のバッファに空き容量ができるまで、送信側スレッドを待機させるための同期関数です。

シグネチャまたは形状

static apr_status_t wait_not_full(h2_bucket_beam *beam, conn_rec *c,
                                  apr_read_type_e block,
                                  apr_size_t *pspace_left)

入力と出力

  • **入力**:
  • beam: 対象の h2_bucket_beam 構造体。
  • c: 接続コンテキスト (conn_rec)。ログ出力に使用。
  • block: 読み込みモード (APR_BLOCK_READ 等)。
  • pspace_left: 空き容量を格納するポインタ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。異常終了時は APR_ECONNABORTED、非ブロックモードで満杯の場合は APR_EAGAIN を返す。
  • *pspace_left: 待機終了時点での残り空き容量。

前提条件と不変条件

  • beam->lock ミューテックスが呼び出し元で保持されている必要があります。
  • calc_space_left(beam) が 0 を返す限り、ループ内で条件変数による待機が行われます。

アルゴリズムまたは振る舞い

1. calc_space_left を呼び出し、バッファに空きがあるか確認する。 2. 空きがない場合:

  • beam->aborted が真であれば APR_ECONNABORTED を返す。
  • block が APR_BLOCK_READ でなければ APR_EAGAIN を返す。
  • それ以外の場合、beam->timeout の値に応じて apr_thread_cond_timedwait または apr_thread_cond_wait を実行し、条件変数のシグナルを待つ。

    • 3. 空きができるか、エラーが発生するまでループを継続する。 4. 最終的な空き容量を *pspace_left に書き込み、ステータスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_bucket_beam.c:567 (直接の呼び出しを確認)
  • **呼び出し先**:
  • calc_space_left
  • apr_thread_cond_timedwait / apr_thread_cond_wait
  • H2_BEAM_LOG (マクロ)

関連データ構造

  • h2_bucket_beam: 同期対象のバッファ構造体。
  • conn_rec: Apache HTTP Server の接続コンテキスト。
  • apr_thread_cond_t: スレッド間同期に使用する条件変数。

失敗モードと境界条件

  • **タイムアウト**: beam->timeout が設定されている場合、待機時間が経過すると apr_thread_cond_timedwait がエラーを返す。
  • **異常終了**: beam->aborted が設定されている場合、待機を中断して APR_ECONNABORTED を返す。
  • **非ブロック**: block が APR_BLOCK_READ でない場合、即座に APR_EAGAIN を返す。

保守担当者向けメモ

  • 本関数は beam->lock を保持した状態で呼び出されることを前提としています。デッドロックを避けるため、呼び出し元でのロック管理を徹底してください。
  • H2_BEAM_LOG マクロによるトレース出力が含まれており、デバッグ時には APLOG_TRACE2 レベルを確認してください。
