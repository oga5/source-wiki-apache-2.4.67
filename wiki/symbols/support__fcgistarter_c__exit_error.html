exit_error

定義

  • 種別: function
  • 定義箇所: support/fcgistarter.c:44

目的

APR（Apache Portable Runtime）関連の関数呼び出しでエラーが発生した際、エラーメッセージを標準エラー出力に出力し、プログラムを異常終了させるためのユーティリティ関数です。

シグネチャまたは形状

static void exit_error(apr_status_t rv, const char *func)

入力と出力

  • **入力**:
  • rv: apr_status_t 型のエラーコード。
  • func: エラーが発生した関数名を示す文字列（const char *）。
  • **出力**: なし（関数は exit(EXIT_FAILURE) を呼び出し、呼び出し元へは戻りません）。

前提条件と不変条件

  • apr_strerror が rv に対応するエラーメッセージを正しく解決できること。
  • stderr が書き込み可能であること。

アルゴリズムまたは振る舞い

1. apr_strerror を使用して、エラーコード rv に対応する人間が読める形式のエラーメッセージをローカルバッファ（1024バイト）に取得します。 2. fprintf を使用し、func（関数名）と取得したエラーメッセージを stderr に出力します。 3. exit(EXIT_FAILURE) を呼び出し、プロセスを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: support/fcgistarter.c 内の以下の箇所から呼び出されています。
  • apr_sockaddr_info_get 失敗時 (128行目)
  • apr_socket_create 失敗時 (133行目)
  • apr_socket_opt_set 失敗時 (138行目)
  • apr_socket_bind 失敗時 (143行目)
  • apr_socket_listen 失敗時 (148行目)
  • apr_proc_detach 失敗時 (153行目)
  • apr_proc_fork 失敗時 (213行目)
  • **呼び出し先**:
  • apr_strerror (APRライブラリ)
  • fprintf (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • apr_status_t: APR関数の戻り値として使用されるステータスコード型。

失敗モードと境界条件

  • apr_strerror がエラーコードを解決できない場合、標準的なエラーメッセージが返されます。
  • バッファサイズ（1024バイト）を超えるエラーメッセージは切り詰められます。

保守担当者向けメモ

  • 本関数は fcgistarter のエラーハンドリングを一元化するために使用されています。
  • 子プロセス生成後のエラーハンドリングは設計上困難であるため、本関数は主に親プロセス側での初期化フェーズでの失敗を想定しています。
