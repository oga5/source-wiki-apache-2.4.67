ap_fcgi_header

定義

  • 種別: struct
  • 定義箇所: include/util_fcgi.h:40

目的

FastCGI プロトコル（RFC 3652）における、8バイトの固定長レコードヘッダーを表現するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    unsigned char version;
    unsigned char type;
    unsigned char requestIdB1;
    unsigned char requestIdB0;
    unsigned char contentLengthB1;
    unsigned char contentLengthB0;
    unsigned char paddingLength;
    unsigned char reserved;
} ap_fcgi_header;

入力と出力

本構造体はデータ保持用であり、直接的な入力や出力は持ちません。ネットワークバイトオーダーとの変換には、ap_fcgi_header_to_array や ap_fcgi_header_from_array などのユーティリティ関数を介してバイト配列と相互変換します。

前提条件と不変条件

  • FastCGI 仕様に基づき、常に 8 バイトの固定長です。
  • 各フィールドはネットワークバイトオーダー（ビッグエンディアン）を想定しています。
  • コンパイラのパディングによる影響を避けるため、直接的なメモリコピーではなく、提供されている変換関数経由での操作が推奨されます。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行います。通信時には、ap_fcgi_fill_in_header を用いて各フィールドを初期化し、ap_fcgi_header_to_array でバイトストリームへ変換して送信します。受信時には、バイトストリームから ap_fcgi_header_from_array を用いて本構造体にアンパックし、type や contentLength を判定します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な使用例）**:
  • modules/aaa/mod_authnz_fcgi.c (ローカル変数として宣言)
  • **呼び出し先**:
  • 本構造体を操作する関数として ap_fcgi_header_to_array, ap_fcgi_header_from_array, ap_fcgi_fill_in_header が定義されています。

関連データ構造

  • ap_fcgi_begin_request_body: BEGIN_REQUEST レコードのボディを定義する構造体。

失敗モードと境界条件

  • 本構造体自体に失敗モードはありません。
  • 変換関数使用時、contentLength が AP_FCGI_MAX_CONTENT_LEN (65535バイト) を超える場合や、不正なバイト配列をアンパックした場合の挙動は、変換関数の実装に依存します。

保守担当者向けメモ

  • 本構造体はメモリ割り当てを行いません。利用側でスタックまたはヒープ上のバッファを管理してください。
  • フィールドの並び順はプロトコル仕様に固定されているため、構造体のメンバ順序を変更しないでください。
