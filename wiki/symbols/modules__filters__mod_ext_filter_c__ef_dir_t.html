ef_dir_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_ext_filter.c:58

目的

Apache HTTP Server の mod_ext_filter において、ディレクトリ単位の設定情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct ef_dir_t {
    int log_stderr;
    int onfail;
} ef_dir_t;

入力と出力

  • log_stderr (int): 外部フィルタプロセスの標準エラー出力をログに記録するかどうかを制御するフラグ。
  • onfail (int): 外部フィルタプロセスが失敗した場合の挙動を定義する設定値。

前提条件と不変条件

  • 本構造体は apr_pcalloc を用いてリクエストプール（r->pool）上に割り当てられることが一般的です。
  • 設定の生成およびマージは、create_ef_dir_conf および merge_ef_dir_conf 関数によって管理されます。

アルゴリズムまたは振る舞い

  • ディレクトリごとの設定として、フィルタの実行時エラーハンドリングやログ出力の要否を決定するパラメータを提供します。
  • get_cfg_string 関数などで、現在の設定状態を文字列として取得するために参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: mod_ext_filter.c 内のコンフィグ生成関数（create_ef_dir_conf）、マージ関数（merge_ef_dir_conf）、およびフィルタ実行コンテキスト（ef_ctx_t）の初期化処理から参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ef_ctx_t: リクエストごとの実行コンテキスト。ef_dir_t へのポインタを保持し、フィルタ処理中に設定を参照します。
  • ef_filter_t: フィルタの定義情報。

失敗モードと境界条件

  • 設定が未初期化の状態で参照された場合、apr_pcalloc によるゼロ初期化が前提となります。
  • onfail の具体的な値域やエラー時の動作定義については、mod_ext_filter.c 内のディレクティブ解析ロジックに依存します。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を保持するため、merge_ef_dir_conf における設定の継承ロジックが正しく実装されていることを確認してください。
  • 構造体メンバを追加する場合は、設定の継承およびマージ処理への影響を考慮してください。
