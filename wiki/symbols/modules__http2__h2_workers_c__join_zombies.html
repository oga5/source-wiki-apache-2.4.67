join_zombies

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_workers.c:156

目的

終了済み（H2_SLOT_ZOMBIE 状態）のワーカースレッドを回収し、リソースを解放してスロットを再利用可能な状態（H2_SLOT_FREE）に戻す。

シグネチャまたは形状

static void join_zombies(h2_workers *workers)

入力と出力

  • **入力**: workers (h2_workers *) - 管理対象のワーカープール構造体。
  • **出力**: なし（副作用として workers->zombie リング内のスロットが処理される）。

前提条件と不変条件

  • workers->lock ミューテックスが呼び出し元によって保持されていること（関数内で一時的に解除・再取得される）。
  • workers->zombie リング内の各 h2_slot は H2_SLOT_ZOMBIE 状態であり、有効な thread ハンドルを持つこと。

アルゴリズムまたは振る舞い

1. workers->zombie リングが空になるまで以下の処理を繰り返す。 2. リングから最初の h2_slot を取り出す。 3. workers->lock を一時的に解除し、apr_thread_join を呼び出してスレッドの終了を待機する。 4. workers->lock を再取得する。 5. スレッドハンドルをクリアし、スロット状態を H2_SLOT_FREE に変更する。 6. スロットに割り当てられていたサブプール (slot->pool) を破棄する。 7. スロットを workers->free リングの末尾に挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_workers.c:321
  • modules/http2/h2_workers.c:419
  • **呼び出し先**:
  • apr_thread_join
  • apr_thread_mutex_lock / unlock
  • apr_pool_destroy
  • APR_RING マクロ群

関連データ構造

  • h2_workers: ワーカープール全体を管理する構造体。
  • h2_slot: 個別のワーカー状態（thread, pool, state）を保持する構造体。

失敗モードと境界条件

  • workers->zombie リングが空の場合、何も行わず即座に復帰する。
  • apr_thread_join の実行中、他のスレッドが workers->lock を介してプール状態にアクセスする可能性があるため、ミューテックスの解除と再取得が必須となっている。

保守担当者向けメモ

  • 本関数は workers->lock を一時的に解放するため、呼び出し元でこの関数が実行されている間に他のスレッドが workers 構造体の状態を変更する可能性を考慮する必要がある。
  • スロットの再利用時にサブプールを破棄・再生成することで、スレッドごとのメモリリークを防いでいる。
