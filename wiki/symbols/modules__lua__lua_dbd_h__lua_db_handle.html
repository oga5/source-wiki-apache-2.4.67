lua_db_handle

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_dbd.h:28

目的

Apache HTTP Server の mod_lua において、データベース接続状態を保持し、Lua スクリプトから apr_dbd または mod_dbd を介したデータベース操作を可能にするための管理構造体です。

シグネチャまたは形状

typedef struct {
    apr_dbd_t               *handle;
    const apr_dbd_driver_t  *driver;
    int                     alive;
    apr_pool_t              *pool;
    char                    type;
    ap_dbd_t *              dbdhandle;
    server_rec              *server;
} lua_db_handle;

入力と出力

本構造体は直接的な関数ではありませんが、Lua 側のデータベース操作関数（select, query 等）において、データベース接続のコンテキストとして使用されます。

前提条件と不変条件

  • alive フラグにより接続の生存状態が管理されます。
  • pool メンバにより、関連するリソースのメモリ管理が行われます。
  • type メンバは、使用するバックエンド（LUA_DBTYPE_APR_DBD または LUA_DBTYPE_MOD_DBD）を識別するために使用されます。

アルゴリズムまたは振る舞い

  • Lua スクリプトからのデータベース操作要求時、lua_get_db_handle を介して取得されます。
  • 接続の取得は lua_db_acquire で行われ、Lua のユーザーデータとしてスタックに保持されます。
  • リソースの解放は lua_db_close または Lua のガベージコレクタ（lua_db_gc）によって自動的に実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_dbd.c 内の各データベース操作関数（lua_get_db_handle 等）から参照されます。
  • **呼び出し先**: 構造体内部の handle（apr_dbd_t）および dbdhandle（ap_dbd_t）を通じて、apr_dbd および mod_dbd の各関数が呼び出されます。

関連データ構造

  • apr_dbd_t: APR レベルのデータベース接続ハンドル。
  • ap_dbd_t: Apache モジュールレベルのデータベース接続ハンドル。
  • lua_db_result_set: クエリ結果を保持する構造体。
  • lua_db_prepared_statement: プリペアドステートメントを保持する構造体。

失敗モードと境界条件

  • alive フラグが偽の場合、接続が切断されていることを示します。
  • マルチスレッド環境下での接続再利用は、バックエンドである mod_dbd の仕様に依存します。

保守担当者向けメモ

  • 本構造体は mod_lua とデータベースバックエンドの仲介役です。
  • lua_db_gc によるリソース解放が正しく行われるよう、構造体の各メンバの初期化と破棄には注意が必要です。
  • 接続の生存確認には alive フラグを適切に更新・参照してください。
