require_expr_info

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authz_core.c:1058

目的

Require ディレクティブにおいて、式（expression）ベースの認可条件を保持するための構造体です。

シグネチャまたは形状

struct require_expr_info {
    ap_expr_info_t *expr;
    int want_user;
};

入力と出力

  • **入力**: ap_expr_info_t 型の式情報と、ユーザー認証を必要とするかを示すフラグ。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • expr は ap_expr.h で定義された式評価エンジンによって解析された有効な式である必要があります。
  • want_user は、認可評価時に認証済みユーザー情報が必要かどうかを制御するフラグとして機能します。

アルゴリズムまたは振る舞い

本構造体は、mod_authz_core における expr プロバイダーの設定情報を保持します。設定解析時に apr_pcalloc を用いてメモリが割り当てられ、解析された式情報が格納されます。その後、認可実行時にこの構造体を参照して式を評価し、アクセス可否を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_core.c 内の Require ディレクティブ解析処理（1067, 1079 行目付近）および、認可実行時のプロバイダー評価処理（1111 行目付近）。
  • **呼び出し先**: ap_expr（式評価エンジン）。

関連データ構造

  • ap_expr_info_t: ap_expr.h で定義される、評価対象の式を保持する構造体。
  • authz_provider: 本構造体を利用する認可プロバイダーのインターフェース。

失敗モードと境界条件

  • expr が NULL の場合、式評価時に予期せぬ動作を引き起こす可能性があります。
  • 設定ファイルで不正な式が指定された場合、解析フェーズでエラーが発生します。

保守担当者向けメモ

  • 本構造体は mod_authz_core の expr プロバイダー専用です。
  • want_user フラグの変更は、認可プロセス全体における認証要件に影響を与えるため、変更時は authorize_user_core の挙動と照らし合わせて確認してください。
