ap_xlate_proto_to_ascii

定義

  • 種別: function
  • 定義箇所: server/util_ebcdic.c:73

目的

EBCDIC環境において、プロトコルレベルのデータ（ヘッダー等）をローカルの文字セットからASCII形式へインプレース変換します。

シグネチャまたは形状

void ap_xlate_proto_to_ascii(char *buffer, apr_size_t len)

入力と出力

  • **入力**:
  • buffer: 変換対象の文字列データが格納されたバッファ。
  • len: 変換対象データの長さ（バイト単位）。
  • **出力**:
  • なし（戻り値は void）。変換結果は buffer に直接上書きされます。

前提条件と不変条件

  • 本関数は APR_CHARSET_EBCDIC が定義されている環境でのみ有効です。
  • グローバル変数 ap_hdrs_to_ascii が ap_init_ebcdic によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. inbytes_left および outbytes_left に len を設定します。 2. apr_xlate_conv_buffer を呼び出し、ap_hdrs_to_ascii ハンドルを使用して buffer 内のデータを変換します。 3. 変換はインプレースで行われ、元のバッファがASCII形式に書き換えられます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/byterange_filter.c
  • modules/http/chunk_filter.c
  • modules/http/http_filters.c
  • modules/metadata/mod_ident.c
  • modules/proxy/ajp_msg.c
  • **呼び出し先**:
  • apr_xlate_conv_buffer (APRライブラリ関数)

関連データ構造

  • ap_hdrs_to_ascii: server/util_ebcdic.c 内で静的に管理される apr_xlate_t 型の変換ハンドル。

失敗モードと境界条件

  • apr_xlate_conv_buffer の内部的な変換エラーについては、本関数内で明示的なエラーハンドリングは行われていません。
  • 非EBCDIC環境では、util_ebcdic.h により空のマクロとして定義されるため、変換処理は実行されません。

保守担当者向けメモ

  • 本関数は APR_CHARSET_EBCDIC 定義環境下でのみ動作します。移植性を維持するため、呼び出し側で条件分岐を記述する必要はありません。
  • 変換ハンドル ap_hdrs_to_ascii の初期化順序に依存するため、起動時の初期化フロー（ap_init_ebcdic）の変更には注意が必要です。
