socache_dc_iterate

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dc.c:165

目的

キャッシュ内の全要素を走査（イテレート）するためのインターフェースを提供します。

シグネチャまたは形状

apr_status_t socache_dc_iterate(ap_socache_instance_t *instance,
                                server_rec *s, void *userctx,
                                ap_socache_iterator_t *iterator,
                                apr_pool_t *pool);

入力と出力

  • **入力**:
  • instance: キャッシュインスタンスへのポインタ。
  • s: サーバーレコード。
  • userctx: ユーザー定義のコンテキスト。
  • iterator: 走査時に実行されるコールバック関数へのポインタ。
  • pool: メモリ割り当て用のプール。
  • **出力**:
  • apr_status_t: 処理結果。常に APR_ENOTIMPL を返します。

前提条件と不変条件

  • 本関数は distcache プロバイダーにおいて、走査機能が実装されていないことを前提としています。

アルゴリズムまたは振る舞い

  • 呼び出された際、いかなる走査処理も行わず、即座に APR_ENOTIMPL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dc.c:183 にて、ap_socache_provider_t 構造体のメンバとして登録されています。
  • **呼び出し先**: なし。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスを保持する構造体。
  • ap_socache_iterator_t: 走査用コールバック関数の型定義。

失敗モードと境界条件

  • 本関数は常に APR_ENOTIMPL を返すため、機能が利用できないことを示すエラーとして扱われます。
