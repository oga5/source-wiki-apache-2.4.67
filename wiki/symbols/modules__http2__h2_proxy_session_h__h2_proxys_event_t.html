h2_proxys_event_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_proxy_session.h:47

目的

バックエンドサーバーとの HTTP/2 プロキシセッションにおける状態遷移をトリガーするためのイベント種別を定義します。

シグネチャまたは形状

typedef enum {
    H2_PROXYS_EV_INIT,
    H2_PROXYS_EV_LOCAL_GOAWAY,
    H2_PROXYS_EV_REMOTE_GOAWAY,
    H2_PROXYS_EV_CONN_ERROR,
    H2_PROXYS_EV_PROTO_ERROR,
    H2_PROXYS_EV_CONN_TIMEOUT,
    H2_PROXYS_EV_NO_IO,
    H2_PROXYS_EV_STREAM_SUBMITTED,
    H2_PROXYS_EV_STREAM_DONE,
    H2_PROXYS_EV_STREAM_RESUMED,
    H2_PROXYS_EV_DATA_READ,
    H2_PROXYS_EV_NGH2_DONE,
    H2_PROXYS_EV_PRE_CLOSE,
} h2_proxys_event_t;

入力と出力

本シンボル自体は列挙型であり、入力や出力は持ちません。この列挙値は、セッションの状態管理を行う関数へ引数として渡されます。

前提条件と不変条件

  • h2_proxy_session のライフサイクル内で発生する特定の事象（エラー、ストリームの完了、I/O 待機など）に対応して使用されます。
  • 状態遷移の整合性を保つため、適切なタイミングで dispatch_event 関数に渡される必要があります。

アルゴリズムまたは振る舞い

  • セッションの現在の状態（h2_proxys_state）と、このイベントを組み合わせて、セッションの次の状態を決定します。
  • H2_PROXYS_EV_NGH2_DONE は nghttp2 が読み書きすべきデータがないことを示し、H2_PROXYS_EV_NO_IO は実際の I/O が発生しなかったことを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c 内のセッション処理ロジック。
  • **呼び出し先**: dispatch_event 関数（modules/http2/h2_proxy_session.c:63, 1495）がこの列挙型を引数として受け取り、内部的な状態遷移を実行します。

関連データ構造

  • h2_proxy_session: このイベントが適用される対象のメイン構造体。
  • h2_proxys_state: このイベントによって遷移する先の状態定義。

失敗モードと境界条件

  • 未定義のイベント値を dispatch_event に渡した場合の挙動は定義されていません。
  • 接続エラー（H2_PROXYS_EV_CONN_ERROR）やプロトコルエラー（H2_PROXYS_EV_PROTO_ERROR）が発生した際は、セッションのクローズ処理へ遷移することが想定されます。

保守担当者向けメモ

  • 新しいイベントを追加する場合は、dispatch_event 内の switch 文または状態遷移テーブルを更新し、すべての状態においてそのイベントが適切に処理されるか確認してください。
