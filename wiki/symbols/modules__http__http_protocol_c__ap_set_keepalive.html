ap_set_keepalive

定義

  • 種別: function
  • 定義箇所: modules/http/http_protocol.c:217

目的

HTTP 接続の永続化（Keep-Alive）の可否を判定し、レスポンスヘッダーの調整および接続状態の更新を行う。

シグネチャまたは形状

int ap_set_keepalive(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: int （Keep-Alive が有効な場合は 1、無効な場合は 0 を返す）

前提条件と不変条件

  • r->connection->keepalive が AP_CONN_CLOSE でないこと。
  • サーバー設定で Keep-Alive が有効であり、かつタイムアウト設定が正であること。
  • クライアントが Connection: close を要求していないこと。
  • MPM プロセスが終了処理中でないこと。

アルゴリズムまたは振る舞い

1. **判定**: 複数の条件（リクエストバージョン、メソッド、コンテンツ長、サーバー設定、クライアントの意図など）を評価し、接続を維持可能か判断する。 2. **有効時**:

  • r->connection->keepalive を AP_CONN_KEEPALIVE に設定し、カウンタをインクリメントする。
  • クライアントが Keep-Alive トークンを送信している場合、Keep-Alive ヘッダーをレスポンスに追加し、Connection ヘッダーを Keep-Alive に設定する。

    • 3. **無効時**:

  • Connection: close ヘッダーをレスポンスに追加する。
  • r->connection->keepalive を AP_CONN_CLOSE に設定する。
  • 接続上限に達している場合は、接続カウンタをインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_filters.c
  • modules/http2/h2_c2_filter.c
  • modules/lua/lua_request.c
  • modules/proxy/mod_proxy_http.c
  • **呼び出し先**:
  • is_mpm_running()
  • ap_find_token()
  • ap_is_chunked()
  • ap_status_drops_connection()
  • apr_table_get, apr_table_setn, apr_table_mergen
  • apr_psprintf, apr_time_sec

関連データ構造

  • request_rec: リクエストおよび接続状態（r->connection）を保持する主要構造体。
  • server_rec: サーバーの Keep-Alive 設定（keep_alive, keep_alive_max, keep_alive_timeout）を保持。

失敗モードと境界条件

  • r->server->keep_alive_max が 0 の場合、リクエスト数制限なしとして扱われる。
  • クライアントが HTTP/1.0 で Keep-Alive を明示しない場合、接続は維持されない。
  • r->chunked の副作用として、HTTP/1.1 でチャンク転送が強制的に有効化される場合がある。

保守担当者向けメモ

  • 条件判定ロジックは非常に複雑であり、評価順序が重要である。修正時はソース内のコメントにある条件リストを厳密に参照すること。
  • r->chunked = 1 という副作用が含まれているため、この関数を呼び出すタイミングには注意が必要である。
