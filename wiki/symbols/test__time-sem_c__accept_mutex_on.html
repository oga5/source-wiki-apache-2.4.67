accept_mutex_on

定義

  • 種別: function
  • 定義箇所: test/time-sem.c:111

目的

プロセス間同期のための排他ロック（fcntl ベース）を獲得する。

シグネチャまたは形状

void accept_mutex_on(void)

入力と出力

  • 入力: なし（グローバル変数 fcntl_fd および lock_it を使用）
  • 出力: なし（成功時はロックを獲得して復帰、失敗時はプロセスを終了）

前提条件と不変条件

  • fcntl_fd が有効なファイル記述子として初期化されていること。
  • lock_it 構造体が適切なロック設定（F_SETLKW 用）で初期化されていること。

アルゴリズムまたは振る舞い

1. fcntl システムコールを F_SETLKW（ロック待ち）コマンドで呼び出し、ロックの獲得を試みる。 2. EINTR（システムコール割り込み）が発生した場合は、ループを継続して再試行する。 3. ロック獲得に失敗（ret < 0）した場合は、perror でエラーを出力し、exit(1) でプロセスを終了する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/mpm/prefork/prefork.c:573
  • test/time-sem.c 内の複数箇所（532行目、542行目など）
  • 呼び出し先:
  • fcntl (システムコール)
  • perror (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • fcntl_fd: ロック対象のファイル記述子。
  • lock_it: struct flock 型のロック設定構造体。

失敗モードと境界条件

  • fcntl が EINTR 以外のエラーを返した場合、即座にプログラムが終了する。
  • 本関数はブロッキング動作を行うため、デッドロックが発生している環境では呼び出し元が永久に待機状態となる。

保守担当者向けメモ

  • 本関数は test/time-sem.c における抽象化レイヤーの一部であり、他のロック方式（SysVセマフォ等）とインターフェースが共通化されている。
  • エラーハンドリングは exit(1) による即時終了が前提となっているため、本番環境のコードへ組み込む際は適切なエラー復帰処理への書き換えが必要である。
