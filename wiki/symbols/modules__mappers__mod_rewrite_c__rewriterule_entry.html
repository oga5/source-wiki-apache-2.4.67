rewriterule_entry

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:344

目的

mod_rewrite における RewriteRule ディレクティブの解析済み設定を保持する構造体です。正規表現パターン、置換文字列、およびルール適用時の動作を制御するフラグやメタデータを格納します。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *rewriteconds;
    char      *pattern;
    ap_regex_t *regexp;
    char      *output;
    int        flags;
    char      *forced_mimetype;
    char      *forced_handler;
    int        forced_responsecode;
    data_item *env;
    data_item *cookie;
    int        skip;
    int        maxrounds;
    const char *escapes;
    const char *noescapes;
} rewriterule_entry;

入力と出力

  • **入力**: 設定ファイル（httpd.conf や .htaccess）から読み込まれたルール定義。
  • **出力**: apply_rewrite_rule 関数等で参照され、リクエストの書き換え処理（URIの置換、環境変数の設定、MIMEタイプやハンドラの強制変更など）に利用されます。

前提条件と不変条件

  • regexp は pattern をコンパイルした結果である必要があります。
  • rewriteconds は、このルールに先行する RewriteCond のリストを保持します。
  • メモリ管理はリクエストプール（r->pool）に依存します。

アルゴリズムまたは振る舞い

  • apply_rewrite_rule 関数において、リクエスト URI が regexp にマッチした場合、output を用いて置換を実行します。
  • flags に基づき、リダイレクト、プロキシ転送、または後続ルールのスキップ（skip）などの制御を行います。
  • [N] フラグ使用時には maxrounds を参照し、無限ループを防止します。

呼び出し元と呼び出し先

  • **呼び出し元**: apply_rewrite_rule、do_expand、force_type_handler、およびルールセットを初期化する設定読み込み処理から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • rewritecond_entry: このルールに付随する条件式。
  • data_item: 環境変数やクッキーの定義。
  • rewrite_perdir_conf / rewrite_server_conf: ルールを保持する親の設定構造体。

失敗モードと境界条件

  • maxrounds を超えるループが発生した場合、書き換え処理は停止します。
  • 正規表現のコンパイル失敗や、メモリ不足による割り当て失敗が発生する可能性があります。

保守担当者向けメモ

  • [PT] フラグの扱いや、バックリファレンスのエスケープ処理（escapes, noescapes）は複雑なため、変更時は mod_rewrite.c の適用ロジックを十分に確認してください。
  • ルールセットの反復処理において skip フィールドが正しく機能しているか注意が必要です。
