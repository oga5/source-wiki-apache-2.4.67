register_hooks

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbm.c:334

目的

Apache HTTP Server の起動プロセスにおいて、DBM ベースの認可プロバイダーを登録し、必要なオプション関数を取得するために使用されます。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（モジュール初期化用のメモリプール）
  • **出力**: なし（戻り値は void）

前提条件と不変条件

  • 本関数は Apache のモジュールロードプロセスの一部として呼び出されます。
  • authz_dbmgroup_provider および authz_dbmfilegroup_provider が定義されている必要があります。

アルゴリズムまたは振る舞い

1. ap_register_auth_provider を呼び出し、dbm-group プロバイダーとして authz_dbmgroup_provider を登録します。 2. ap_register_auth_provider を呼び出し、dbm-file-group プロバイダーとして authz_dbmfilegroup_provider を登録します。 3. ap_hook_optional_fn_retrieve を呼び出し、authz_dbm_getfns をフックしてオプション関数の取得を試みます。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュールロード機構（直接的な呼び出し元はソースコード上では明示されていませんが、Apache の標準的なモジュールライフサイクルに従います）。
  • **呼び出し先**:
  • ap_register_auth_provider
  • ap_hook_optional_fn_retrieve
  • authz_dbm_getfns（コールバックとして登録）

関連データ構造

  • authz_dbmgroup_provider
  • authz_dbmfilegroup_provider
  • apr_pool_t

失敗モードと境界条件

  • プロバイダー登録時にメモリ不足が発生した場合、Apache のエラーログに記録され、サーバーの起動が失敗する可能性があります。
  • ap_hook_optional_fn_retrieve が失敗した場合、オプション関数が正しく取得できず、関連する認可機能が正常に動作しない可能性があります。

保守担当者向けメモ

  • 本関数は mod_authz_dbm のエントリーポイントの一つです。新しい認可プロバイダーを追加する場合は、ここに登録処理を追加してください。
  • AP_AUTH_INTERNAL_PER_CONF フラグが使用されており、設定ごとの内部認可処理として登録されています。
