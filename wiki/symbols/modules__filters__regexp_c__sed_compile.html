sed_compile

定義

  • 種別: function
  • 定義箇所: modules/filters/regexp.c:132

目的

文字列形式の正規表現を、sed_step が解釈可能なバイトコード形式へ変換する。

シグネチャまたは形状

char *sed_compile(sed_commands_t *commands, sed_comp_args *compargs,
                  char *ep, char *endbuf, int seof);

入力と出力

  • **入力**:
  • commands: cp（現在の解析位置）を含むコマンド構造体。
  • compargs: コンパイル結果を格納する引数構造体。
  • ep: バイトコード書き込み先バッファの開始ポインタ。
  • endbuf: バッファの終了ポインタ（境界チェック用）。
  • seof: 正規表現の終了文字。
  • **出力**:
  • 成功時: バイトコード書き込み後のバッファ末尾ポインタ。
  • 失敗時: NULL（エラー発生時、regerr を経由して報告）。

前提条件と不変条件

  • NBRA（キャプチャグループ数）は 9 に制限されている。
  • ep は呼び出し元が確保した十分なサイズのバッファを指している必要がある。
  • コンパイル中、bracket スタックにより \( と \) のネストが管理される。

アルゴリズムまたは振る舞い

1. GETC() マクロを用いて入力文字列を順次読み込み、ep に対応するバイトコード（CDOT, CCL, CBRA, CKET, CCHR 等）を書き込む。 2. * や {n,m} などの修飾子を検出した場合、直前のバイトコードにフラグ（STAR, RNGE）を付与する。 3. [ から始まる文字クラスはビットマップ形式（CCL）または拡張形式（CXCL）に変換する。 4. \( \) のネストを bracket スタックで追跡し、対応が取れていない場合はエラーを発生させる。 5. 終了文字 seof に到達した時点で CCEOF を書き込み終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/sed0.c:992 での呼び出しが確認されている。
  • **呼び出し先**: regerr（エラー報告用）。

関連データ構造

  • sed_commands_t: コンパイル対象のコマンド状態を保持。
  • sed_comp_args: コンパイル結果（circf, nbra 等）を格納。
  • regexp.h: バイトコード定数（CBRA, CCHR, CCL 等）を定義。

失敗モードと境界条件

  • **バッファオーバーフロー**: ep >= endbuf となった場合、SEDCOMPILE_ERROR(50) を発行。
  • **構文エラー**: 括弧の不一致（エラー42）、範囲指定の不正（エラー16, 44, 45, 46）、キャプチャグループの制限超過（エラー43）などで regerr を呼び出し NULL を返す。

保守担当者向けメモ

  • 本関数は非常に古い AT&T sed の実装を継承しており、goto を多用した制御フローを持つ。
  • メモリ管理は呼び出し元に依存するため、endbuf による境界チェックが必須である。
  • _advance 内のバックトラッキングロジックと密接に関連しているため、バイトコードの変更時は regexp.h の定義と併せて確認すること。
