md_json_fmt_t

定義

  • 種別: enum
  • 定義箇所: modules/md/md_json.h:42

目的

JSON データのシリアライズ（書き出し）時におけるフォーマット形式を指定するために使用されます。

シグネチャまたは形状

typedef enum {
    MD_JSON_FMT_COMPACT,
    MD_JSON_FMT_INDENT,
} md_json_fmt_t;

入力と出力

  • **入力**: JSON 書き出し関数（md_json_writeb, md_json_writep, md_json_writef 等）の引数として渡されます。
  • **出力**: 内部的に fmt_to_flags 関数へ渡され、jansson ライブラリが解釈可能なフラグへ変換されます。

前提条件と不変条件

  • MD_JSON_FMT_COMPACT は、空白や改行を含まない最小限の形式を意味します。
  • MD_JSON_FMT_INDENT は、人間が読みやすいようにインデントされた形式を意味します。

アルゴリズムまたは振る舞い

md_json.c 内の fmt_to_flags 関数において、この列挙値に基づき jansson のダンプフラグが決定されます。このフラグは、ファイルシステムへの書き出しやバケットへのシリアライズ処理の挙動を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • md_json_writeb
  • md_json_writep
  • md_json_writef
  • md_json_fcreatex
  • md_json_freplace
  • **呼び出し先**:
  • fmt_to_flags (内部関数)

関連データ構造

  • md_json_t: 本フォーマットを用いてシリアライズされる対象の JSON オブジェクト。

失敗モードと境界条件

  • 定義された列挙値以外の値が渡された場合の挙動は、fmt_to_flags の実装に依存します（ソースコード上では明示的なエラーハンドリングの詳細は未確認です）。

保守担当者向けメモ

  • 新たな JSON 出力形式が必要になった場合は、この列挙型に値を追加し、modules/md/md_json.c 内の fmt_to_flags 関数を更新してください。
