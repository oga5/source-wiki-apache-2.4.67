rl_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_ratelimit.c:33

目的

Apache HTTP Server の出力フィルタ mod_ratelimit において、レスポンスの転送速度制限およびバースト制御の状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct rl_ctx_t
{
    int speed;                      /* 制限速度 (KB/s) */
    int chunk_size;                 /* チャンク単位のサイズ */
    int burst;                      /* 初期バースト量 */
    int do_sleep;                   /* スリープ処理のフラグ */
    rl_state_e state;               /* 現在の動作モード (RATE_LIMIT/RATE_FULLSPEED) */
    apr_bucket_brigade *tmpbb;      /* 一時処理用ブリゲード */
    apr_bucket_brigade *holdingbb;  /* 次回持ち越し用ブリゲード */
} rl_ctx_t;

入力と出力

  • 本構造体は rate_limit_filter 内で f->ctx として保持されます。
  • f->r->pool からメモリが割り当てられ、リクエスト終了時に自動的に解放されます。

前提条件と不変条件

  • state は rl_state_e 型で定義され、RATE_LIMIT または RATE_FULLSPEED のいずれかの状態をとります。
  • tmpbb および holdingbb は apr_bucket_brigade 型であり、Apache のバケット管理フレームワークに準拠します。

アルゴリズムまたは振る舞い

  • RATE_LIMIT モード時: chunk_size に基づいてデータを分割し、apr_sleep を用いて転送速度を制御します。
  • RATE_FULLSPEED モード時: RL_END バケットが検出されるまで、データを制限なく通過させます。
  • 処理しきれなかったデータは holdingbb に保存され、次回のフィルタ呼び出し時に再利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: rate_limit_filter (modules/filters/mod_ratelimit.c:63, 108) がコンテキストの取得および初期化を行います。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • rl_state_e: フィルタの動作モードを定義する列挙型。
  • apr_bucket_brigade: Apache のバケット管理用データ構造。

失敗モードと境界条件

  • 環境変数の設定が無効な場合、フィルタは解除されます。
  • RATE_INTERVAL_MS (200ms) ごとの apr_sleep により、高負荷時にはスレッドがブロッキングされる可能性があります。
  • サブリクエストに対しては自動的にフィルタが解除されるため、本コンテキストは使用されません。

保守担当者向けメモ

  • 本構造体は f->r->pool に紐付いているため、明示的なメモリ解放は不要です。
  • 将来的に Optional Functions への移行が検討されています。
