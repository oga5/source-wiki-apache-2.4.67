pphrase_cb_arg_t

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_engine_pphrase.c:38

目的

暗号化された秘密鍵を復号するためのパスフレーズ収集プロセスにおいて、サーバー設定、メモリ管理、および現在の試行状態を保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    server_rec         *s;                    /* サーバー設定 */
    apr_pool_t         *p;                    /* メモリプール */
    apr_array_header_t *aPassPhrase;          /* パスフレーズの配列 */
    int                 nPassPhraseCur;       /* 現在のパスフレーズインデックス */
    char               *cpPassPhraseCur;      /* 現在のパスフレーズ文字列 */
    int                 nPassPhraseDialog;    /* ダイアログの種類 */
    int                 nPassPhraseDialogCur; /* 現在のダイアログ試行回数 */
    BOOL                bPassPhraseDialogOnce;/* ダイアログを一度のみ表示するフラグ */
    const char         *key_id;               /* 鍵の識別子 */
    const char         *pkey_file;            /* 鍵ファイルのパス */
} pphrase_cb_arg_t;

入力と出力

  • **入力**: サーバー設定 (server_rec) およびメモリプール (apr_pool_t) を初期化時に受け取ります。
  • **出力**: 収集されたパスフレーズ (cpPassPhraseCur) を OpenSSL の復号処理へ提供します。

前提条件と不変条件

  • p (メモリプール) は、パスフレーズのライフサイクルを管理するために使用されます。
  • 収集された機密データ（パスフレーズ）は、使用後に OPENSSL_cleanse によってメモリから消去される必要があります。
  • 構造体内の各フィールドは、ssl_load_encrypted_pkey 等の初期化関数によって適切に設定される必要があります。

アルゴリズムまたは振る舞い

1. **状態保持**: サーバー起動時の鍵読み込み処理において、現在の試行回数やダイアログの進捗を追跡します。 2. **コンテキスト提供**: OpenSSL の UI_METHOD コールバック（passphrase_ui_read 等）において、UI_get0_user_data を介して本構造体を取得し、パスフレーズの供給源を特定します。 3. **キャッシュ連携**: サーバー再起動時に、key_id を用いて既存の鍵キャッシュとの照合を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: ssl_load_encrypted_pkey および modssl_load_engine_keypair 等の初期化関数。
  • **呼び出し先**: OpenSSL の UI_METHOD コールバック関数群（passphrase_ui_open, read, write, close）が、本構造体をユーザーデータとして参照します。

関連データ構造

  • server_rec: Apache のサーバー設定構造体。
  • apr_pool_t: Apache Portable Runtime のメモリプール。
  • apr_array_header_t: パスフレーズのリストを保持する配列構造体。

失敗モードと境界条件

  • **メモリ不足**: apr_pool_t への割り当て失敗時に処理が中断される可能性があります。
  • **入力失敗**: nPassPhraseDialogCur が上限に達した場合、パスフレーズの取得に失敗し、鍵の読み込みが停止します。
  • **Win32 の制約**: Windows 環境では builtin ダイアログがサポートされないため、本構造体を用いたパイプ経由の入力が必須となります。

保守担当者向けメモ

  • 本構造体内のパスフレーズ文字列は機密情報です。デバッグ出力等で誤ってログに記録しないよう注意してください。
  • asn1_table_vhost_key との連携により、再起動時のパスフレーズ入力を省略する設計となっています。変更時はキャッシュの整合性に注意してください。
