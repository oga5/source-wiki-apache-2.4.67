post_rotate

定義

  • 種別: function
  • 定義箇所: support/rotatelogs.c:274

目的

ログファイルのローテーション完了後に、オプションで指定されたハードリンクの作成や、外部プログラム（postrotate_prog）の実行を行う。

シグネチャまたは形状

static void post_rotate(apr_pool_t *pool, struct logfile *newlog,
                        rotate_config_t *config, rotate_status_t *status)

入力と出力

  • **入力**:
  • pool: メモリ管理用 APR プール。
  • newlog: 新しく作成されたログファイルの情報。
  • config: ローテーション設定（リンク先パス、外部プログラムパス等）。
  • status: 現在の実行状態。
  • **出力**: なし（void）。エラー発生時は stderr にメッセージを出力し、リンク失敗時は exit(2) で終了する。

前提条件と不変条件

  • config->linkfile が設定されている場合、既存のリンクファイルを削除してから新しいハードリンクを作成する。
  • 外部プログラムの実行は、config->postrotate_prog が設定されている場合にのみ行われる。

アルゴリズムまたは振る舞い

1. **ハードリンク作成**: config->linkfile が存在する場合、既存のリンクを削除し、newlog->name から config->linkfile へハードリンクを作成する。 2. **ゾンビプロセス回収**: 外部プログラム実行前に apr_proc_wait_all_procs を呼び出し、完了済みのプロセスを回収する。 3. **プロセス属性設定**: apr_procattr_create を使用してプロセス属性を初期化し、環境変数付きのコマンド実行モード (APR_PROGRAM_ENV) を設定する。 4. **プログラム実行**: 外部プログラムを起動する。引数として newlog->name を渡し、現在のログファイル名 (status->current.name) が存在する場合はそれも渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/rotatelogs.c:480
  • **呼び出し先**:
  • apr_file_remove, apr_file_link
  • apr_proc_wait_all_procs
  • apr_procattr_create, apr_procattr_error_check_set, apr_procattr_cmdtype_set
  • apr_proc_create

関連データ構造

  • rotate_config_t: ローテーション設定を保持。
  • rotate_status_t: 実行状態を保持。
  • struct logfile: ログファイル名やプールを保持。

失敗モードと境界条件

  • **リンク作成失敗**: apr_file_link が失敗した場合、エラーメッセージを出力して exit(2) を呼び出す。
  • **プロセス属性設定失敗**: apr_procattr_create や属性設定が失敗した場合、エラーメッセージを出力して処理を中断する。
  • **プログラム起動失敗**: apr_proc_create が失敗した場合、エラーメッセージを出力して処理を中断する。

保守担当者向けメモ

  • 外部プログラムの実行は非同期で行われる。
  • ログファイル名が status->current.name に存在しない場合、外部プログラムには引数が2つ（プログラム名と新しいログファイル名）のみ渡される。
