set_cookie_domain

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_usertrack.c:389

目的

Apache 設定ディレクティブ CookieDomain の値を解析・検証し、ディレクトリごとの設定構造体に格納する。

シグネチャまたは形状

static const char *set_cookie_domain(cmd_parms *cmd, void *mconfig, const char *name)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • mconfig: 現在のディレクトリ設定構造体 (cookie_dir_rec)。
  • name: 設定ファイルから読み取られたドメイン文字列。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列（const char *）を返す。

前提条件と不変条件

  • mconfig は cookie_dir_rec 型としてキャスト可能であること。
  • name は空文字列であってはならない。
  • name はドット（.）で開始する必要がある。
  • name は開始ドットを除いて、少なくとも1つのドットを含まなければならない。

アルゴリズムまたは振る舞い

1. mconfig を cookie_dir_rec 型にキャストする。 2. name が空でないか確認し、空の場合はエラーを返す。 3. name の先頭文字が . であるか確認し、そうでない場合はエラーを返す。 4. name の2文字目以降に . が含まれているか ap_strchr_c で確認し、含まれない場合はエラーを返す。 5. 検証を通過した場合、dcfg->cookie_domain に name を代入し、NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_usertrack.c:470 において、AP_INIT_TAKE1 マクロを通じて CookieDomain ディレクティブのハンドラとして登録されている。
  • **呼び出し先**: ap_strchr_c（Apache API）を呼び出してドメイン文字列の検証を行う。

関連データ構造

  • cookie_dir_rec: ディレクトリごとの設定を保持する構造体。cookie_domain フィールドにドメイン文字列が格納される。

失敗モードと境界条件

  • **空文字列**: CookieDomain が空の場合、「CookieDomain values may not be null」を返す。
  • **ドットで始まらない**: 「CookieDomain values must begin with a dot」を返す。
  • **埋め込みドットの欠如**: ドメイン名として不適切な形式（例: .local）の場合、「CookieDomain values must contain at least one embedded dot」を返す。

保守担当者向けメモ

  • 本関数は設定読み込み時にのみ実行される。
  • name の検証ロジックは RFC 6265 等の Cookie 仕様に基づく制約を反映している。
  • dcfg->cookie_domain への代入は、リクエスト処理時の Set-Cookie ヘッダー生成時に参照される。
