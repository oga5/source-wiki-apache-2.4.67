modssl_ctx_cfg_merge

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_config.c:266

目的

SSL コンテキスト設定（modssl_ctx_t）において、親スコープの設定（base）と子スコープの設定（add）を統合し、結果を mrg に格納する。

シグネチャまたは形状

static void modssl_ctx_cfg_merge(apr_pool_t *p,
                                 modssl_ctx_t *base,
                                 modssl_ctx_t *add,
                                 modssl_ctx_t *mrg)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t。
  • base: 継承元の設定構造体。
  • add: 継承先（上書き側）の設定構造体。
  • mrg: マージ結果を書き込む出力用構造体。

前提条件と不変条件

  • add の protocol_set が真の場合、mrg には add の値が優先される。
  • cfgMerge 系マクロにより、add 側が未設定（UNSET）またはデフォルト値の場合、base 側の値が継承される。

アルゴリズムまたは振る舞い

1. **プロトコル設定**: add->protocol_set を確認し、設定済みであれば add の値を、未設定であれば base の値を mrg にコピーする。 2. **各フィールドのマージ**: cfgMerge、cfgMergeString、cfgMergeInt、cfgMergeBool マクロを使用し、パスフレーズ、証明書チェーン、CRL、認証設定、OCSP 設定、SRP 設定、およびピア検証オプションを順次統合する。 3. **条件付きコンパイル**: HAVE_OCSP_STAPLING、HAVE_SRP、HAVE_SSL_CONF_CMD が定義されている場合のみ、対応するフィールドがマージされる。

関連データ構造

  • modssl_ctx_t: SSL 設定を保持する主要な構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • add が UNSET 状態のフィールドを持つ場合、base の値がそのまま引き継がれる。
  • コンパイル環境（HAVE_* マクロ）によって、マージ対象となるフィールドの範囲が変化する。

保守担当者向けメモ

  • cfgMerge マクロは単純な継承ロジックであるため、複雑なデータ構造（配列など）を扱う場合は、専用の _merge 関数（例: modssl_ctx_cfg_merge_certkeys_array）が別途必要となる。
