ctx_run

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_proxy_http2.c:260

目的

HTTP/2 プロキシセッションのセットアップ、リクエストの投入、およびセッション終了までのイベントループ実行を行う。

シグネチャまたは形状

apr_status_t ctx_run(h2_proxy_ctx *ctx)

入力と出力

  • **入力**: h2_proxy_ctx *ctx（リクエスト処理中の状態を保持するコンテキスト構造体）
  • **出力**: apr_status_t（処理結果。成功時は APR_SUCCESS、失敗時はエラーコードまたは HTTP_SERVICE_UNAVAILABLE）

前提条件と不変条件

  • ctx は有効な h2_proxy_ctx 構造体を指している必要がある。
  • セッション実行中、ctx->r_done が完了フラグとして機能し、ループの終了条件を制御する。
  • セッションの再利用可能性は、関数終了時に ctx->has_reusable_session に格納される。

アルゴリズムまたは振る舞い

1. h2_proxy_session_setup を呼び出し、HTTP/2 セッションを初期化する。セッション生成に失敗した場合は HTTP_SERVICE_UNAVAILABLE を返す。 2. add_request を呼び出し、現在のリクエストをセッションに投入する。 3. ctx->cfront（フロントエンド接続）が切断されておらず、かつ ctx->r_done が偽である間、h2_proxy_session_process を呼び出してセッションの I/O を駆動する。 4. セッション処理中にエラーが発生した場合、h2_proxy_session_cleanup を実行してリソースを解放し、ループを抜ける。 5. ループ終了後、ctx->cfront が切断されていれば全リクエストをキャンセルし、セッションの再利用可否を判定して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_proxy_http2.c:435（proxy_http2_handler 内）
  • **呼び出し先**:
  • h2_proxy_session_setup
  • add_request
  • h2_proxy_session_process
  • h2_proxy_session_cleanup
  • h2_proxy_session_cancel_all
  • h2_proxy_session_is_reusable

関連データ構造

  • h2_proxy_ctx: リクエストの接続情報や状態を管理するメイン構造体。
  • h2_proxy_session: HTTP/2 プロキシセッションの内部状態を保持する構造体。

失敗モードと境界条件

  • **セッション生成失敗**: h2_proxy_session_setup が NULL を返した場合、HTTP_SERVICE_UNAVAILABLE を返す。
  • **セッション処理エラー**: h2_proxy_session_process が APR_SUCCESS 以外を返した場合、セッションのクリーンアップを行いエラーを伝播させる。
  • **マスター接続の切断**: ctx->cfront->aborted が真となった場合、進行中の全リクエストをキャンセルする。

保守担当者向けメモ

  • 本関数は h2_proxy_session のライフサイクルを管理する中心的なループである。
  • session_req_done コールバックがセッションの各リクエスト完了時に呼び出される前提で設計されている。
  • セッションの再利用判定は、関数終了直前に h2_proxy_session_is_reusable を介して行われるため、セッションのクリーンな終了状態が重要となる。
