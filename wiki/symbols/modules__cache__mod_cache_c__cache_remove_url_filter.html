cache_remove_url_filter

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:1625

目的

HTTP レスポンスフィルタチェーンにおいて、特定のキャッシュエントリをキャッシュストレージから削除するために使用されます。

シグネチャまたは形状

apr_status_t cache_remove_url_filter(ap_filter_t *f, apr_bucket_brigade *in)

入力と出力

  • **入力**:
  • f: 現在のフィルタコンテキストを含む ap_filter_t 構造体。
  • in: 処理対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS または ap_pass_brigade の戻り値）。

前提条件と不変条件

  • f->ctx に cache_request_rec 構造体が格納されていることが期待されます。
  • フィルタは実行後に自身をフィルタチェーンから削除します。

アルゴリズムまたは振る舞い

1. f->ctx から cache_request_rec を取得します。 2. コンテキストが存在しない場合（手動設定等による予期せぬ呼び出し）:

  • デバッグログを出力します。
  • 自身をフィルタチェーンから削除します。
  • ap_pass_brigade を呼び出し、後続のフィルタへ処理を渡して終了します。

    • 3. コンテキストが存在する場合:

  • cache_remove_url(cache, r) を呼び出し、キャッシュから該当 URL を削除します。
  • 自身をフィルタチェーンから削除します。
  • ap_pass_brigade を呼び出し、後続のフィルタへ処理を渡して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_cache.c:2687（ソースコード内の直接的な参照として確認）。
  • **呼び出し先**:
  • cache_remove_url (キャッシュ削除処理)
  • ap_remove_output_filter (フィルタの自己削除)
  • ap_pass_brigade (後続フィルタへのデータ転送)
  • ap_log_rerror (エラー/デバッグログ出力)

関連データ構造

  • ap_filter_t: Apache フィルタチェーンのノード。
  • cache_request_rec: キャッシュリクエスト固有の情報を保持する構造体。
  • apr_bucket_brigade: フィルタ間でデータを渡すためのバケットリスト。

失敗モードと境界条件

  • f->ctx が NULL の場合、キャッシュ削除処理はスキップされ、警告ログが出力された上で後続のフィルタへ処理が透過的に渡されます。

保守担当者向けメモ

  • 本関数は CACHE_REMOVE_URL フィルタとして機能します。管理者が意図せず手動でこのフィルタを有効にした場合でも、システムがクラッシュしないよう安全策が講じられています。
