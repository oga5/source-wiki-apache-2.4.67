socache_dc_remove

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dc.c:136

目的

分散キャッシュ（distcache）から、指定された識別子（ID）に対応するセッションデータを削除します。

シグネチャまたは形状

apr_status_t socache_dc_remove(ap_socache_instance_t *ctx,
                               server_rec *s, const unsigned char *id,
                               unsigned int idlen, apr_pool_t *p);

入力と出力

  • **入力**:
  • ctx: 分散キャッシュのコンテキストを含む ap_socache_instance_t 構造体。
  • s: ログ出力に使用する server_rec 構造体。
  • id: 削除対象のセッションを特定するバイト列。
  • idlen: id の長さ。
  • p: メモリプール（本関数内では未使用）。
  • **出力**:
  • APR_SUCCESS: 削除成功時。
  • APR_NOTFOUND: 削除対象が見つからなかった場合。

前提条件と不変条件

  • ctx および ctx->dc が初期化済みである必要があります。
  • id は有効なセッション識別子である必要があります。

アルゴリズムまたは振る舞い

1. DC_CTX_remove_session を呼び出し、distcache サーバーに対してセッション削除を要求します。 2. 削除が成功した場合（戻り値が真）、APLOG_ERR レベルで "distributed scache 'remove' HIT" というログを出力し、APR_SUCCESS を返します。 3. 削除が失敗した場合（戻り値が偽）、APLOG_ERR レベルで "distributed scache 'remove' MISS" というログを出力し、APR_NOTFOUND を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_dc.c:181 にて、ap_socache_provider_t 構造体の一部として参照されています。
  • **呼び出し先**: distcache ライブラリの DC_CTX_remove_session を呼び出します。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスを保持し、内部で DC_CTX（distcache コンテキスト）を管理します。

失敗モードと境界条件

  • distcache サーバーとの通信失敗や、指定した ID が存在しない場合は APR_NOTFOUND を返します。
  • ログ出力は APLOG_ERR レベルで固定されており、削除の成否に関わらずエラーログとして記録されます。

保守担当者向けメモ

  • 本関数は distcache API の戻り値を直接評価しています。
  • ログレベルが APLOG_ERR に設定されていますが、キャッシュミス（APR_NOTFOUND）が通常の運用フローで発生しうる場合、ログレベルの妥当性を再検討してください。
