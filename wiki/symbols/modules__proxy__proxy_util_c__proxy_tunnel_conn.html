proxy_tunnel_conn

定義

  • 種別: struct
  • 定義箇所: modules/proxy/proxy_util.c:5369

目的

プロキシトンネルにおける双方向通信の片側（クライアントまたはバックエンド）の接続状態を管理する構造体です。

シグネチャまたは形状

struct proxy_tunnel_conn {
    struct proxy_tunnel_conn *other; // トンネルの反対側の端点
    conn_rec *c;                     // 接続情報
    const char *name;                // 識別名
    apr_pollfd_t *pfd;               // ポーリング用ファイル記述子
    apr_bucket_brigade *bb;          // データ転送用バケットブリゲード
    unsigned int down_in:1,          // 入力側クローズフラグ
                 down_out:1;         // 出力側クローズフラグ
};

入力と出力

  • **入力**: apr_pcalloc 等によりメモリが割り当てられ、conn_rec や apr_pollfd_t が設定されます。
  • **出力**: トンネル内のデータ転送処理において、other メンバを通じて対向接続へデータが渡されます。

前提条件と不変条件

  • other メンバは常にトンネルの対向端点を指す必要があります。
  • c (conn_rec) は有効な接続状態である必要があります。

アルゴリズムまたは振る舞い

  • トンネルの各端点として機能し、apr_pollfd_t を介した非同期 I/O 監視をサポートします。
  • down_in および down_out ビットフィールドにより、接続の各方向の終了状態を個別に追跡します。
  • proxy_util.c 内のトンネル処理ロジックにおいて、in->other を参照することで対向接続へのデータ転送を実現します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/proxy_util.c 内のトンネル初期化および転送処理（ap_proxy_tunnel_run 等に関連する箇所）。
  • **呼び出し先**: conn_rec 構造体（Apache HTTP Server のコア接続管理）および apr_pollfd_t（APR のポーリング機能）。

関連データ構造

  • conn_rec: Apache HTTP Server の接続コンテキスト。
  • apr_pollfd_t: APR のポーリング用構造体。
  • apr_bucket_brigade: データ転送用バッファ。

失敗モードと境界条件

  • メモリ割り当て失敗時（apr_pcalloc の失敗）。
  • 接続が予期せず切断された場合、down_in / down_out フラグが適切に更新されないと、トンネルループが終了せずリソースリークを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は mod_proxy.h で不透明（opaque）なポインタとして宣言されているため、外部モジュールからは直接メンバにアクセスせず、proxy_util.c が提供するインターフェースを介して操作してください。
  • ビットフィールドの操作はスレッドセーフではないため、呼び出し元の同期制御に依存します。
