ap_die

定義

  • 種別: function
  • 定義箇所: modules/http/http_request.c:232

目的

リクエスト処理中に発生したエラーを処理し、適切なエラーレスポンスを生成するための汎用ハンドラです。

シグネチャまたは形状

void ap_die(int type, request_rec *r)

入力と出力

  • type: 発生したエラーのステータスコード（例: HTTP_INTERNAL_SERVER_ERROR）。
  • r: 現在のリクエストを保持する request_rec 構造体。
  • 戻り値: なし（void）。

前提条件と不変条件

  • r は有効なリクエストレコードである必要があります。
  • 内部で r->status を参照するため、呼び出し前に r->status が適切に設定されていることが期待されます。

アルゴリズムまたは振る舞い

1. 受け取った type とリクエストレコード r を引数として、内部関数 ap_die_r を呼び出します。 2. その際、第3引数として r->status を渡します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http/http_filters.c (複数箇所)
  • modules/http/http_request.c (複数箇所)
  • modules/cache/mod_cache.c (参照のみ)
  • **呼び出し先**:
  • ap_die_r (直接の呼び出し先)

関連データ構造

  • request_rec: リクエストのコンテキストと状態を保持する主要な構造体。

失敗モードと境界条件

  • エラー処理が再帰的に呼び出される可能性があるため、呼び出し元では ap_die 自体の再帰的な呼び出しを避ける制御が必要となる場合があります。

保守担当者向けメモ

  • 本関数は ap_die_r への単なるラッパーです。エラー処理のロジックを変更する場合は ap_die_r を参照してください。
  • http_filters.c 等のコメントによれば、エラー発生時にフィルタチェーンの状態が変更される可能性があるため、呼び出し後のフローには注意が必要です。
