remove_extension_info

定義

  • 種別: function
  • 定義箇所: modules/http/mod_mime.c:291

目的

指定されたファイル拡張子に関連付けられたメタデータ（Charset, Encoding, Handler, Filter等）を、ディレクトリ設定から削除するための予約エントリを作成します。

シグネチャまたは形状

static const char *remove_extension_info(cmd_parms *cmd, void *m_, const char *ext)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。cmd->info には削除対象の属性種別を示すオフセット値が格納されています。
  • m_: 現在のディレクトリ設定構造体 (mime_dir_config)。
  • ext: 削除対象のファイル拡張子文字列。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • ext に先頭のドットが含まれている場合、自動的に除去されます。
  • 拡張子文字列は小文字に正規化されます。
  • m->remove_mappings 配列が未初期化の場合は、cmd->pool を使用して新規作成されます。

アルゴリズムまたは振る舞い

1. ext の先頭が . であればポインタを進めます。 2. m->remove_mappings 配列が存在しない場合、apr_array_make で初期化します。 3. apr_array_push を使用して、remove_mappings 配列に新しい attrib_info 構造体を追加します。 4. 拡張子文字列を cmd->pool に複製し、ap_str_tolower で小文字化して suffix->name に格納します。 5. cmd->info から取得したオフセット値を suffix->offset に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のディレクティブのハンドラとして登録されています。
  • RemoveCharset
  • RemoveEncoding
  • RemoveHandler
  • RemoveInputFilter
  • RemoveLanguage
  • RemoveOutputFilter
  • **呼び出し先**:
  • apr_array_make / apr_array_push (APR配列操作)
  • apr_pstrdup (メモリ複製)
  • ap_str_tolower (文字列正規化)

関連データ構造

  • mime_dir_config: ディレクトリごとの設定を保持する構造体。
  • attrib_info: 拡張子と属性オフセットを保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pool の割り当て失敗によりプロセスが終了する可能性があります。
  • cmd->info が正しく設定されていない場合、誤った属性オフセットが記録されます。

保守担当者向けメモ

  • 本関数は設定ファイル読み込み時に実行されます。
  • 削除処理は即時実行されるのではなく、remove_mappings 配列に記録された後、後のマージフェーズで適用されます。
  • cmd->info の値は、各ディレクティブの登録時に AP_INIT_ITERATE の引数として渡される定数に依存します。
