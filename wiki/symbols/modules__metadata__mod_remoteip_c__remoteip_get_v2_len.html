remoteip_get_v2_len

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_remoteip.c:1018

目的

PROXY プロトコル v2 ヘッダーから、後続データの長さを取得します。

シグネチャまたは形状

apr_size_t remoteip_get_v2_len(proxy_header *hdr)

入力と出力

  • **入力**: proxy_header *hdr - PROXY プロトコルヘッダーを保持する共用体へのポインタ。
  • **出力**: apr_size_t - ネットワークバイトオーダーからホストバイトオーダーに変換された、v2 ヘッダー内のデータ長。

前提条件と不変条件

  • 引数 hdr は有効な proxy_header 構造体を指している必要があります。
  • hdr->v2 メンバが初期化されていることが前提です。

アルゴリズムまたは振る舞い

1. hdr->v2.len に格納されている値を参照します。 2. ntohs() を使用して、ネットワークバイトオーダーからホストバイトオーダーへ変換します。 3. 変換後の値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_remoteip.c:1152 にて、ctx->header をキャストした値が渡されています。
  • **呼び出し先**: ntohs() (システム標準のネットワークバイトオーダー変換関数)。

関連データ構造

  • proxy_header: PROXY プロトコル v1 および v2 のヘッダー情報を保持する共用体。
  • proxy_v2: proxy_header 内の v2 固有構造体。

失敗モードと境界条件

  • hdr が NULL の場合、未定義の動作を引き起こします。
  • hdr->v2.len の値が、実際のバッファサイズと整合しない場合、後続の処理でバッファオーバーランや読み取りエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なるアクセサであり、値の検証は行いません。呼び出し側で hdr が v2 プロトコルであることを確認してから使用してください。
