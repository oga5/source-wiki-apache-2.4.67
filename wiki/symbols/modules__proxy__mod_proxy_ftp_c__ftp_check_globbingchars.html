ftp_check_globbingchars

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:166

目的

指定されたパス文字列内に、FTP サーバー側でワイルドカードとして解釈される可能性のある文字（globbing characters）が含まれているかを判定します。

シグネチャまたは形状

static int ftp_check_globbingchars(const char *path)

入力と出力

  • **入力**: path (検査対象の文字列へのポインタ)
  • **出力**: TRUE (ワイルドカード文字が見つかった場合)、または FALSE (見つからなかった場合)

前提条件と不変条件

  • path はヌル終端文字列である必要があります。
  • バックスラッシュ (\) はエスケープ文字として扱われ、直後の文字はワイルドカード判定の対象から除外されます。

アルゴリズムまたは振る舞い

1. 文字列を先頭から順に走査します。 2. バックスラッシュ (\) に遭遇した場合、ポインタを一つ進め、次の文字をスキップします。 3. 現在の文字が FTP_GLOBBING_CHARS マクロで定義されたいずれかの文字と一致するかを strchr で確認します。 4. 一致した場合は直ちに TRUE を返し、文字列の終端まで一致が見つからなければ FALSE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_ftp.c:519: 相対パスのディレクトリ判定に使用。
  • modules/proxy/mod_proxy_ftp.c:1654: 設定値 ftp_list_on_wildcard と組み合わせて、ディレクトリ一覧表示の可否判定に使用。
  • **呼び出し先**:
  • 標準ライブラリ関数 strchr を使用。

関連データ構造

  • FTP_GLOBBING_CHARS: 判定対象となる文字セットを定義するマクロ（本ソース抜粋外で定義）。

失敗モードと境界条件

  • 空文字列が渡された場合、ループは実行されず FALSE を返します。
  • 文字列の末尾がバックスラッシュで終わる場合、ループ内で *path が \0 となり、その後の strchr 判定は行われずに終了します。

保守担当者向けメモ

  • 本関数は ftp_escape_globbingchars と対になる処理です。ワイルドカード文字の定義を変更する場合は、両方の関数で整合性が保たれているか確認してください。
