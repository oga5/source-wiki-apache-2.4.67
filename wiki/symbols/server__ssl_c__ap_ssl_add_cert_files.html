ap_ssl_add_cert_files

定義

  • 種別: function
  • 定義箇所: server/ssl.c:223

目的

SSL 証明書および秘密鍵ファイルをサーバー設定に追加するためのインターフェースを提供します。

シグネチャまたは形状

apr_status_t ap_ssl_add_cert_files(server_rec *s, apr_pool_t *p,
                                   apr_array_header_t *cert_files,
                                   apr_array_header_t *key_files)

入力と出力

  • **入力**:
  • s: サーバー設定構造体 (server_rec)。
  • p: メモリ割り当てに使用するプール (apr_pool_t)。
  • cert_files: 証明書ファイルパスの配列 (apr_array_header_t)。
  • key_files: 鍵ファイルパスの配列 (apr_array_header_t)。
  • **出力**:
  • 成功時は APR_SUCCESS を返し、失敗時は APR_EGENERAL を返します。

前提条件と不変条件

  • ap_run_ssl_add_cert_files フックが登録されている必要があります。

アルゴリズムまたは振る舞い

1. ap_run_ssl_add_cert_files フックを呼び出し、登録されている SSL モジュールに証明書と鍵の情報を渡します。 2. フックの戻り値が OK または DECLINED の場合は APR_SUCCESS を返します。 3. それ以外（エラー等）の場合は APR_EGENERAL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:2120 からの呼び出しが確認されています。
  • **呼び出し先**: ap_run_ssl_add_cert_files（フック関数）。

関連データ構造

  • server_rec: サーバー設定を保持する構造体。
  • apr_array_header_t: ファイルパスを格納する APR 配列。

失敗モードと境界条件

  • フックが OK または DECLINED 以外の値を返した場合、関数は APR_EGENERAL を返します。

保守担当者向けメモ

  • 本関数は SSL 抽象化レイヤーの一部であり、実際の処理はフック先のモジュール（mod_ssl 等）に委譲されます。
