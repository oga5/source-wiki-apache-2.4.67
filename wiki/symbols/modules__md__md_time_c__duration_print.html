duration_print

定義

  • 種別: function
  • 定義箇所: modules/md/md_time.c:64

目的

apr_interval_time_t 型の期間を、人間が読みやすい文字列（例: "3 days 2 hours"）に変換する。

シグネチャまたは形状

static const char *duration_print(apr_pool_t *p, int roughly, apr_interval_time_t duration)

入力と出力

  • **入力**:
  • p: メモリ確保に使用する apr_pool_t。
  • duration: 変換対象の期間（apr_interval_time_t）。
  • **出力**:
  • 変換された文字列を指す const char *。メモリは p から確保される。

前提条件と不変条件

  • duration が 0 の場合、"0 seconds" を返す。
  • duration が 0 以外で日数が 0 の場合、ミリ秒単位（ms）で出力される。
  • roughly が真の場合、日、時、分、秒のいずれかの単位で値が見つかった時点で、それ以降の計算をスキップして文字列を返す。

アルゴリズムまたは振る舞い

1. duration を日、時、分、秒に分解する。 2. roughly が真であれば、文字列の先頭に ~ を付与する。 3. 日数が 0 より大きい場合、"X days" を文字列に追加する。 4. 残りの時間（時、分、秒）を順次計算し、値が 0 より大きい場合にのみ、対応する単位（"X hours", "X minutes", "X seconds"）を文字列に追加する。 5. 日数が 0 の場合、ミリ秒単位での出力を試みる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_time.c:110 (md_duration_print から呼び出し)
  • modules/md/md_time.c:115 (md_duration_roughly から呼び出し)
  • **呼び出し先**:
  • apr_psprintf (文字列の動的生成)
  • apr_time_sec / apr_time_msec (期間の分解)

関連データ構造

  • apr_interval_time_t: APR で定義された時間間隔を表す型。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • duration が 0 の場合、"0 seconds" を返す。
  • duration が 0 以外で日数が 0 の場合、ミリ秒単位の出力にフォールバックする。
  • roughly フラグの指定により、出力される情報の粒度が変化する。

保守担当者向けメモ

  • 本関数は static であり、同一ファイル内の md_duration_print および md_duration_roughly からのみ利用される。
  • 文字列生成には apr_psprintf を使用しているため、呼び出し側で適切なライフサイクルの apr_pool_t を提供する必要がある。
