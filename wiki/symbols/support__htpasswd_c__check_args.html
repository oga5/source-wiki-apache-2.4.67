check_args

定義

  • 種別: function
  • 定義箇所: support/htpasswd.c:168

目的

コマンドライン引数を解析し、操作モード（フラグ）の決定、ファイル名およびユーザー名の検証、およびパスワードソースの構成を行う。

シグネチャまたは形状

static void check_args(int argc, const char *const argv[],
                       struct passwd_ctx *ctx, unsigned *mask, char **user,
                       char **pwfilename)

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。
  • ctx: 認証コンテキスト（passwd_ctx）。
  • **出力**:
  • mask: 実行する操作フラグ（APHTP_NEWFILE, APHTP_NOFILE, APHTP_DELUSER, APHTP_VERIFY）。
  • user: 解析されたユーザー名。
  • pwfilename: 解析されたパスワードファイル名。

前提条件と不変条件

  • mask に設定されるフラグは、排他的（c, n, v, D のうち最大1つ）である必要がある。
  • ユーザー名に : を含めることはできない。
  • ファイル名およびユーザー名は、それぞれ APR_PATH_MAX および MAX_STRING_LEN を超えてはならない。

アルゴリズムまたは振る舞い

1. apr_getopt を使用してオプション（c, n, m, s, p, d, B, b, D, i, 2, 5, C, r, v）を解析する。 2. 不明なオプションは parse_common_options に委譲する。 3. 複数の操作フラグが指定されていないかビット演算で検証する。 4. APHTP_VERIFY モードかつパスワード入力がプロンプト経由の場合、ctx->passwd_src を PW_PROMPT_VERIFY に更新する。 5. 残りの引数（ファイル名、ユーザー名、必要に応じてパスワード）を解析し、メモリプール（ctx->pool）へコピーする。 6. 引数の数が期待値と一致しない場合は usage() を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htpasswd.c:329 (main 関数)
  • **呼び出し先**: apr_getopt_init, apr_getopt, parse_common_options, usage, apr_file_printf, apr_pstrdup, strchr

関連データ構造

  • passwd_ctx: 認証処理の状態を保持する構造体。
  • apr_getopt_t: 引数解析の状態を保持する APR 構造体。

失敗モードと境界条件

  • **ERR_SYNTAX**: 引数の数が不正、または複数の操作フラグが指定された場合に発生。
  • **ERR_OVERFLOW**: ファイル名やユーザー名がバッファサイズ制限を超えた場合に発生。
  • **ERR_BADUSER**: ユーザー名に不正な文字（:）が含まれている場合に発生。

保守担当者向けメモ

  • 本関数は apr_pool_t を使用してメモリを割り当てるため、呼び出し元の main 関数で管理されるライフサイクルに依存する。
  • APHTP_NOFILE フラグが指定されている場合、ファイル名の解析はスキップされる。
