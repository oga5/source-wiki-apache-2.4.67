acmev2_new_nonce

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme.c:158

目的

ACME v2 サーバーから新しいノンスト（Nonce）を取得し、通信セッションを更新するために使用されます。

シグネチャまたは形状

static apr_status_t acmev2_new_nonce(md_acme_t *acme)

入力と出力

  • **入力**: acme (md_acme_t *) - ACME 通信セッションの状態を保持する構造体。
  • **出力**: apr_status_t - HTTP リクエストの実行結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • acme 引数は有効なポインタである必要があります。
  • acme->api.v2.new_nonce に、サーバーの「newNonce」エンドポイント URL が格納されている必要があります。
  • acme->http が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. acme->api.v2.new_nonce に指定された URL に対して、HEAD リクエストを実行します。 2. md_http_HEAD_perform を呼び出し、レスポンスヘッダーからノンストを抽出・更新するためのコールバック関数 http_update_nonce を指定します。 3. 取得したノンストは、acme セッション内の内部状態として保持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme.c:742 にて、acme->new_nonce_fn 関数ポインタに代入されています。
  • **呼び出し先**:
  • md_http_HEAD_perform (外部ライブラリ関数)
  • http_update_nonce (コールバックとして指定)

関連データ構造

  • md_acme_t: ACME 通信のコンテキストを保持する主要構造体。
  • md_http_t: HTTP 通信エンジン。

失敗モードと境界条件

  • サーバーが到達不能な場合や、HTTP ステータスコードがエラーを示す場合は APR_EGENERAL 等の非ゼロのステータスが返されます。
  • http_update_nonce コールバック内でのノンスト抽出に失敗した場合、そのエラーが伝播されます。

保守担当者向けメモ

  • 本関数は ACME v2 プロトコル専用です。
  • md_acme_t 構造体の api.v2.new_nonce フィールドが正しく設定されていることを前提としています。
  • サーバーから返される Replay-Nonce ヘッダーの解析は http_update_nonce に依存しているため、ヘッダー形式の変更時にはそちらを確認してください。
