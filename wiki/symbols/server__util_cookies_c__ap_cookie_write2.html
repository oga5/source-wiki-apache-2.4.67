ap_cookie_write2

定義

  • 種別: function
  • 定義箇所: server/util_cookies.c:82

目的

RFC 2965 準拠のクッキー文字列を生成し、指定された複数のヘッダーテーブルに Set-Cookie2 ヘッダーとして追加します。

シグネチャまたは形状

apr_status_t ap_cookie_write2(request_rec *r, const char *name2, const char *val, 
                              const char *attrs2, long maxage, ...);

入力と出力

  • **入力**:
  • r: リクエストレコード（メモリプール r->pool を使用）。
  • name2: クッキー名。
  • val: クッキー値。
  • attrs2: クッキー属性（NULL または空文字の場合は DEFAULT_ATTRS が使用される）。
  • maxage: 有効期限（秒単位）。0 の場合は Max-Age 属性を省略。
  • ...: 可変長引数として、クッキーを追加する apr_table_t 型のテーブルポインタを渡す（末尾は NULL で終端）。
  • **出力**:
  • 戻り値: 常に APR_SUCCESS を返します。

前提条件と不変条件

  • すべての文字列操作は r->pool 上で行われるため、呼び出し側でのメモリ解放は不要です。
  • 可変長引数で渡されるテーブルは、apr_table_t 型である必要があります。

アルゴリズムまたは振る舞い

1. maxage が指定されている場合、Max-Age=<maxage>; 文字列を生成します。 2. name2、val、および attrs2（またはデフォルト値）を結合し、RFC 2965 準拠のクッキー文字列を構築します。 3. 生成された文字列をデバッグログに出力します。 4. 可変長引数で渡された各 apr_table_t に対して、apr_table_addn を使用して Set-Cookie2 キーでクッキー文字列を挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session_cookie.c:79
  • modules/session/mod_session_dbd.c:449
  • **呼び出し先**:
  • apr_pstrcat (メモリ確保と文字列結合)
  • apr_ltoa (数値の文字列変換)
  • ap_log_rerror (ログ出力)
  • apr_table_addn (テーブルへの追加)

関連データ構造

  • request_rec: リクエスト状態およびメモリプールの管理に使用。
  • apr_table_t: ヘッダーを保持するテーブル構造。

失敗モードと境界条件

  • attrs2 が NULL または空文字の場合、内部定義の DEFAULT_ATTRS が適用されます。
  • 可変長引数の終端が NULL でない場合、va_arg の呼び出しにより未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は RFC 2965 に特化しています。RFC 2109 準拠のクッキーを扱う場合は ap_cookie_write を使用してください。
  • Set-Cookie2 ヘッダーの追加には apr_table_addn を使用しているため、同一キーで複数のクッキーが設定される可能性があります。
