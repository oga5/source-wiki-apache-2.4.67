create_session_cookie_dir_config

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_cookie.c:171

目的

mod_session_cookie モジュールで使用されるディレクトリ単位の設定構造体（session_cookie_dir_conf）をメモリ上に確保し、初期化する。

シグネチャまたは形状

static void *create_session_cookie_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体を割り当てるためのメモリプール。
  • char *dummy: Apacheの設定フレームワークから渡される引数（未使用）。
  • **出力**:
  • void *: 確保された session_cookie_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • Apacheのメモリプール p が有効であること。
  • apr_pcalloc により、構造体内の全メンバはゼロ初期化される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、session_cookie_dir_conf サイズ分のメモリをプール p から確保する。 2. 確保したメモリ領域を session_cookie_dir_conf 型としてキャストし、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_cookie.c:277（Apacheの設定ハンドラとして登録されている）。
  • **呼び出し先**: apr_pcalloc（APRライブラリのメモリ確保関数）。

関連データ構造

  • session_cookie_dir_conf: 本関数が生成する設定構造体。Cookie名、属性、削除フラグを保持する。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボート（プロセス終了）する可能性がある（APRの標準的な挙動）。

保守担当者向けメモ

  • 本関数はApacheのディレクトリ設定生成フックとして機能する。
  • 構造体のメンバ追加や初期値の変更が必要な場合は、apr_pcalloc の後の処理として記述すること。
