ef_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_ext_filter.c:63

目的

外部フィルタプログラムを Apache HTTP Server のフィルタとして実行する際、リクエストごとの実行状態とプロセス管理情報を保持するために使用されます。

シグネチャまたは形状

typedef struct ef_ctx_t {
    apr_pool_t *p;
    apr_proc_t *proc;
    apr_procattr_t *procattr;
    ef_dir_t *dc;
    ef_filter_t *filter;
    int noop, hit_eos;
#if APR_FILES_AS_SOCKETS
    apr_pollset_t *pollset;
#endif
} ef_ctx_t;

入力と出力

本構造体は関数引数として渡されるのではなく、f->ctx（フィルタコンテキスト）として request_rec に紐付いたプールから apr_pcalloc で割り当てられ、フィルタ処理の各段階で参照・更新されます。

前提条件と不変条件

  • すべてのメンバはリクエストの生存期間に紐付いたプール（r->pool）によって管理されます。
  • filter メンバは、適用されるフィルタの定義情報を指す必要があります。

アルゴリズムまたは振る舞い

  • 外部プロセスの起動時に proc および procattr が初期化されます。
  • noop フラグが真の場合、フィルタ処理はバイパスされます。
  • hit_eos はストリームの終端（EOS）に到達したかを追跡し、プロセスの終了処理を制御します。
  • APR_FILES_AS_SOCKETS が有効な環境では、pollset を用いて非同期的な入出力監視を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: ef_output_filter や ef_input_filter などのフィルタ処理関数が、f->ctx を通じて本構造体を取得・利用します。
  • **呼び出し先**: 外部プロセスの起動（init_ext_filter_process）や、データ転送・読み取り処理（ef_unified_filter 等）において、本構造体のメンバが引数として渡されます。

関連データ構造

  • ef_dir_t: ディレクトリ単位の設定情報。
  • ef_filter_t: フィルタの定義情報（コマンドパス等）。
  • apr_proc_t: 外部プロセスハンドル。
  • apr_pollset_t: 非同期 I/O 用のポーリングセット（条件付き）。

失敗モードと境界条件

  • プロセス起動失敗時やパイプの読み書きエラー発生時、ef_ctx_t 内のフラグやハンドルを通じてエラーハンドリングが行われます。
  • APR_FILES_AS_SOCKETS が定義されていない環境では、ポーリングの代わりに apr_sleep を用いた待機が発生し、パフォーマンスに影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体はリクエスト終了時に自動的に解放されるため、手動でのメモリ解放は不要です。
  • プロセスの生存期間は apr_pool_note_subprocess によって管理されており、リクエスト終了時に確実に終了するよう設計されています。
