ap_meets_conditions

定義

  • 種別: function
  • 定義箇所: modules/http/http_protocol.c:546

目的

HTTP リクエストに含まれる条件付きヘッダー（If-Match, If-Modified-Since 等）を評価し、リソースが条件を満たしているか、あるいは 304 Not Modified や 412 Precondition Failed を返す必要があるかを判定します。

シグネチャまたは形状

int ap_meets_conditions(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）
  • **出力**: int 型のステータスコード。
  • OK: 条件を満たしている（または条件なし）。
  • HTTP_NOT_MODIFIED (304): 条件によりリソースの更新が不要。
  • HTTP_PRECONDITION_FAILED (412): 前提条件が不一致。

前提条件と不変条件

  • リクエストのステータスが成功（ap_is_HTTP_SUCCESS）であり、かつサブクエストや ErrorDocument の処理中でない場合にのみ評価が行われます。
  • r->no_local_copy が真の場合、条件評価をスキップして OK を返します。

アルゴリズムまたは振る舞い

以下の順序で条件を評価し、最終的なステータスを決定します。 1. **If-Match**: 一致しない場合、412 を即時返却。 2. **If-Unmodified-Since**: 条件を満たさない場合、412 を返却。 3. **If-None-Match**: 一致する場合、GET/HEAD なら 304、それ以外なら 412 を返却。 4. **If-Modified-Since**: GET/HEAD リクエストにおいて、リソースが更新されていない場合に 304 を設定。 5. **If-Range**: Range ヘッダーと併用されている場合、条件に関わらず OK を返却。 6. 最終的に not_modified フラグが立っていれば 304 を返し、それ以外は OK を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c
  • modules/cache/mod_file_cache.c
  • modules/dav/main/mod_dav.c
  • modules/mappers/mod_negotiation.c
  • **呼び出し先**:
  • ap_condition_if_match
  • ap_condition_if_unmodified_since
  • ap_condition_if_none_match
  • ap_condition_if_modified_since
  • ap_condition_if_range

関連データ構造

  • request_rec: リクエストのメタデータおよびヘッダー情報（headers_out）を保持。
  • ap_condition_e: 評価結果を示す列挙型（AP_CONDITION_NOMATCH, AP_CONDITION_WEAK 等）。

失敗モードと境界条件

  • If-Match の値が * の場合、リソースの存在確認は行われません。
  • If-Modified-Since で指定された日付が現在時刻より未来の場合は無効とみなされます。
  • GET または HEAD 以外のメソッドでは、If-None-Match 失敗時に 304 ではなく 412 が優先されます。

保守担当者向けメモ

  • 評価順序は RFC に基づいており、ETag 評価が時刻比較よりも優先されます。
  • not_modified 変数は、複数の条件評価を経て最終的なレスポンスコードを決定するための内部状態として使用されています。
