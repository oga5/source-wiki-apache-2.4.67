forward_dns_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:227

目的

クライアントの IP アドレスが、指定されたホスト名の正引き結果（IP アドレスリスト）のいずれかと一致するかを検証し、アクセス許可（AUTHZ_GRANTED）または拒否（AUTHZ_DENIED）を判定する。

シグネチャまたは形状

static authz_status forward_dns_check_authorization(request_rec *r, const char *require_line, const void *parsed_require_line)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • require_line: 設定ファイル上の Require ディレクティブの引数文字列。
  • parsed_require_line: コンパイル済みの ap_expr_info_t 構造体。
  • **出力**:
  • AUTHZ_GRANTED: クライアント IP が指定ホスト名のいずれかの IP と一致した場合。
  • AUTHZ_DENIED: 一致しない場合、または評価エラーが発生した場合。

前提条件と不変条件

  • parsed_require_line は ap_expr_info_t 型としてキャスト可能であること。
  • リクエストの r->useragent_addr が有効なソケットアドレスとして保持されていること。

アルゴリズムまたは振る舞い

1. ap_expr_str_exec を呼び出し、parsed_require_line を評価してホスト名のリストを取得する。 2. 取得したリストを ap_getword_conf でトークン化し、各ホスト名を順次処理する。 3. apr_sockaddr_info_get を使用して、各ホスト名を IP アドレスに正引きする。 4. 正引きで得られたアドレスリスト（sa）を走査し、apr_sockaddr_equal を用いて r->useragent_addr と比較する。 5. 一致するものが見つかった時点で AUTHZ_GRANTED を返し、リストの最後まで一致がなければ AUTHZ_DENIED を返す。 6. 設定行内の # 以降はコメントとして扱い、処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_authz_core が提供する認可プロバイダーインターフェース経由で呼び出される（ソースコード内では authz_forward_dns_provider 構造体で参照）。
  • **呼び出し先**:
  • ap_expr_str_exec: 式の評価。
  • ap_getword_conf: トークンの切り出し。
  • apr_sockaddr_info_get: ホスト名の正引き。
  • apr_sockaddr_equal: IP アドレスの比較。
  • ap_log_rerror: ログ出力。

関連データ構造

  • request_rec: リクエスト情報およびクライアント IP (r->useragent_addr) を保持。
  • ap_expr_info_t: Require ディレクティブの評価式。
  • apr_sockaddr_t: 正引きされた IP アドレスのリスト構造体。

失敗モードと境界条件

  • **式評価失敗**: ap_expr_str_exec がエラーを返した場合、ログを出力し AUTHZ_DENIED を返す。
  • **DNS 解決失敗**: apr_sockaddr_info_get が失敗した場合、警告ログを出力し、そのホスト名についてはスキップして次のトークンへ進む。
  • **コメント処理**: # がトークンの先頭にある場合は即座に処理を終了する。

保守担当者向けメモ

  • 本関数は DNS 正引きに依存するため、DNS サーバーの応答速度がリクエスト処理時間に直接影響する。
  • forward-dns は正引き結果のリストを走査するため、マルチホーム環境やラウンドロビン DNS 等で複数の IP が返される場合、そのすべてが検証対象となる。
