ap_regkey_remove

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:130

目的

指定されたレジストリキーを削除します。

シグネチャまたは形状

apr_status_t ap_regkey_remove(const ap_regkey_t *parent,
                              const char *keyname,
                              apr_pool_t *pool)

入力と出力

  • parent: 削除対象のキーを含む親レジストリキーへのポインタ。
  • keyname: 削除するキーの名前。
  • pool: メモリ割り当てに使用する APR プール。
  • 戻り値: 成功時は APR_SUCCESS、失敗時は apr_status_t 型のエラーコードを返します。

前提条件と不変条件

  • 本関数は非再帰的です。
  • Windows の仕様により、削除対象のキーにサブキーが存在する場合、削除は失敗します。

アルゴリズムまたは振る舞い

1. **プラットフォーム判定**: 実行環境に応じて Unicode (RegDeleteKeyW) または ANSI (RegDeleteKey) API を選択します。 2. **Unicode 環境**: apr_conv_utf8_to_ucs2 を使用してキー名を変換し、RegDeleteKeyW を呼び出します。 3. **ANSI 環境**:

  • RegOpenKeyEx で対象キーを読み取り専用で開きます。
  • RegQueryInfoKey を使用してサブキーの有無を確認します。
  • サブキーが存在する場合、ERROR_ACCESS_DENIED を返します。
  • サブキーが存在しない場合、RegDeleteKey を呼び出します。

    • 4. **終了処理**: OS から返されたエラーコードを APR_FROM_OS_ERROR を通じて APR 形式に変換して返します。

呼び出し元と呼び出し先

  • 呼び出し元: include/ap_regkey.h に宣言が存在しますが、具体的な呼び出し箇所は提供されたソース内にはありません。
  • 呼び出し先: RegDeleteKeyW, RegDeleteKey, RegOpenKeyEx, RegQueryInfoKey, RegCloseKey (Win32 API)、および apr_conv_utf8_to_ucs2 (APR)。

関連データ構造

  • ap_regkey_t: レジストリキーハンドル (hkey) を保持する構造体。

失敗モードと境界条件

  • サブキーが存在する場合: ERROR_ACCESS_DENIED に相当するエラーを返します。
  • キーが存在しない場合や権限不足の場合: OS から返されたエラーコードを apr_status_t に変換して返します。
  • Unicode 変換失敗時: apr_conv_utf8_to_ucs2 の戻り値がエラーを返します。

保守担当者向けメモ

  • 本関数は非再帰的であるため、サブキーを含むキーを削除する機能は提供していません。
  • ANSI 環境でのサブキー確認処理は、Win9x 系と NT 系で一貫した動作を保証するために実装されています。
