gset_encoder

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_push.c:703

目的

Golomb Coded Set (GCS) を用いて、プッシュ日記（h2_push_diary）の内容をバイナリ形式のビットストリームへ圧縮・エンコードするための状態管理構造体です。

シグネチャまたは形状

typedef struct {
    h2_push_diary *diary;
    unsigned char log2p;
    int mask_bits;
    int delta_bits;
    int fixed_bits;
    apr_uint64_t fixed_mask;
    apr_pool_t *pool;
    unsigned char *data;
    apr_size_t datalen;
    apr_size_t offset;
    unsigned int bit;
    apr_uint64_t last;
} gset_encoder;

入力と出力

  • **入力**: h2_push_diary 構造体へのポインタを保持し、エンコード対象のハッシュ値を受け取ります。
  • **出力**: data バッファにビット単位でエンコードされた GCS 形式のバイナリデータを出力します。

前提条件と不変条件

  • pool は apr_pool_t によるメモリ管理に使用されます。
  • last フィールドは、デルタエンコードのために直前に処理した値を保持します。
  • bit は現在のバイト内でのビット位置（0-7）を追跡します。

アルゴリズムまたは振る舞い

1. h2_push_diary_digest_get() から呼び出され、日記内のハッシュ値をソートして処理します。 2. gset_encode_next() を介して各ハッシュ値を順次処理し、Golomb 符号化を適用します。 3. gset_encode_bit() を使用して、計算されたビットを data バッファへ書き込みます。 4. デルタ符号化により、ソートされたハッシュ値間の差分を圧縮します。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_push_diary_digest_get（modules/http2/h2_push.c）
  • **呼び出し先**:
  • gset_encode_next（modules/http2/h2_push.c:761）
  • gset_encode_bit（modules/http2/h2_push.c:739）

関連データ構造

  • h2_push_diary: エンコード対象となるプッシュ済みリソースのハッシュ履歴を保持する構造体。
  • apr_pool_t: メモリ割り当てに使用される Apache Portable Runtime のプール。

失敗モードと境界条件

  • data バッファの容量不足が発生した場合、エンコード処理が失敗する可能性があります。
  • last 値の更新時にオーバーフローが発生しないことが前提となります。

保守担当者向けメモ

  • 本構造体は、IETF で廃案となった Cache-Digest ドラフトの概念に基づいたサーバー側のプッシュ履歴管理に使用されています。
  • log2p や fixed_bits などのパラメータは、GCS の圧縮効率を決定するため、変更時は Golomb 符号化の仕様と整合性を取る必要があります。
