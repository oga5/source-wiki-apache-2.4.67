ap_rxplus_exec

定義

  • 種別: function
  • 定義箇所: server/util_regex.c:141

目的

コンパイル済みの正規表現オブジェクト (ap_rxplus_t) を使用して、対象文字列に対するマッチングおよび置換処理を実行します。

シグネチャまたは形状

int ap_rxplus_exec(apr_pool_t *pool, ap_rxplus_t *rx, const char *pattern, char **newpattern)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する APR プール。
  • rx: コンパイル済みの正規表現構造体。
  • pattern: マッチング対象の文字列。
  • newpattern: 置換結果を格納するためのポインタ（置換が不要な場合は NULL 指定可）。
  • **出力**:
  • 戻り値: マッチした回数（マッチしない場合は 0）。
  • newpattern: 置換が行われた場合、置換後の文字列がプールから割り当てられて格納されます。

前提条件と不変条件

  • rx は事前に ap_rxplus_compile 等で初期化されている必要があります。
  • rx->match は ap_rxplus_exec を呼び出すたびに更新されるため、マッチ結果を保持したい場合は ap_rxplus_pmatch を使用してコピーを作成する必要があります。

アルゴリズムまたは振る舞い

1. ap_regexec を呼び出し、pattern に対してマッチングを行います。マッチしない場合は 0 を返します。 2. マッチした場合、rx->match に pattern を設定します。 3. 置換文字列 (rx->subs) が定義されている場合:

  • ap_pregsub を用いて置換後の文字列を生成します。
  • AP_REG_MULTI フラグが設定されている場合、残りの文字列に対して再帰的に ap_rxplus_exec を呼び出し、マッチングを継続します。
  • 最終的に、元の文字列と置換部分を結合した新しい文字列を newpattern に格納します。

    • 4. マッチした回数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:1228
  • modules/filters/mod_proxy_html.c:1131
  • server/util_regex.c:167 (再帰呼び出し)
  • **呼び出し先**:
  • ap_regexec (低レイヤーの正規表現マッチング)
  • ap_pregsub (置換文字列の生成)
  • apr_palloc (メモリ割り当て)
  • ap_rxplus_exec (自身を再帰呼び出し)

関連データ構造

  • ap_rxplus_t: 正規表現、置換文字列、フラグ、マッチング結果を保持するメイン構造体。

失敗モードと境界条件

  • ap_regexec がマッチに失敗した場合、0 を返します。
  • ap_pregsub による置換文字列の生成に失敗した場合、0 を返します。
  • AP_REG_MULTI フラグ指定時は、文字列全体に対して再帰的な置換が行われます。

保守担当者向けメモ

  • 本関数は ap_regex.h で定義された ap_rxplus_t を利用します。
  • rx->match は exec を呼び出すたびに内容が更新されるため、呼び出し元でのライフサイクル管理に注意してください。
