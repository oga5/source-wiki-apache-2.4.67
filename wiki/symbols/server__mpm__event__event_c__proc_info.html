proc_info

定義

  • 種別: struct
  • 定義箇所: server/mpm/event/event.c:342

目的

プロセスおよびスレッドの識別情報を保持し、スレッド起動時に必要なコンテキストを渡すために使用されます。

シグネチャまたは形状

typedef struct
{
    int pslot;  /* process slot */
    int tslot;  /* worker slot of the thread */
} proc_info;

入力と出力

  • **入力**: スレッド生成時に ap_malloc を介してメモリが割り当てられ、pslot（プロセススロット）および tslot（ワーカー用スレッドスロット）が設定されます。
  • **出力**: スレッド内で void *dummy 引数からキャストされ、スレッド固有の識別情報として利用されます。

前提条件と不変条件

  • pslot および tslot は、Apache MPM のプロセス管理およびスレッド管理におけるインデックスとして有効な値である必要があります。

アルゴリズムまたは振る舞い

  • スレッド起動時に、この構造体のインスタンスがヒープ上に確保され、スレッド関数へ渡されます。
  • スレッド関数内では、受け取った void * ポインタを proc_info * にキャストして、自身の所属するプロセスおよびスレッドの識別子を参照します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/event/event.c 内の thread_starter 等、スレッドを生成する関数群が ap_malloc を使用して本構造体を生成しています。
  • **呼び出し先**: スレッド関数（dummy 引数として受け取る関数）が、この構造体からスロット情報を読み取ります。
  • **注記**: server/mpm/worker/worker.c にも同名の構造体が存在しますが、本ページは event.c の定義に焦点を当てています。

関連データ構造

  • event_conn_state_t: 接続状態を管理する構造体。
  • event_child_bucket: プロセス管理に関連する構造体。

失敗モードと境界条件

  • ap_malloc によるメモリ確保が失敗した場合、スレッド起動プロセスが中断される可能性があります。

保守担当者向けメモ

  • 本構造体は event.c と worker.c の両方で定義されています。変更を加える際は、両 MPM への影響を考慮してください。
  • スレッド間でのデータ受け渡しに使用されるため、ライフサイクル管理（確保と解放のタイミング）に注意してください。
