dhparam

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_engine_init.c:123

目的

Diffie-Hellman (DH) 鍵交換で使用される素数生成関数、生成された DH パラメータの保持、およびそのパラメータが適用される最小鍵長を管理するための構造体定義です。

シグネチャまたは形状

struct dhparam {
    BIGNUM *(*const prime)(BIGNUM *);
    DH *dh;
    const unsigned int min;
};

入力と出力

  • **入力**: prime フィールドは BIGNUM を引数に取り BIGNUM を返す関数ポインタを保持します。
  • **出力**: dh フィールドは生成された DH 構造体へのポインタを保持します。

前提条件と不変条件

  • dhparams[] 配列として定義されており、各要素は RFC 3526 または RFC 2409 で規定された素数生成関数と、それに対応する最小鍵長（min）を保持します。
  • dh フィールドは初期状態では NULL です。

アルゴリズムまたは振る舞い

  • dhparams 配列には、8192ビットから1024ビットまでの複数の DH パラメータ設定が降順で格納されています。
  • 各要素は、特定の鍵長以上の要求に対して適切な DH パラメータを選択・生成するためのテーブルとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース抜粋内には、この構造体を利用する関数や呼び出し元に関する情報はありません。
  • **呼び出し先**: prime フィールドに格納された関数（BN_get_rfc3526_prime_8192 等）を呼び出す可能性がありますが、本ソース抜粋内での直接的な呼び出しは確認できません。

関連データ構造

  • BIGNUM: OpenSSL の多倍長整数構造体。
  • DH: OpenSSL の Diffie-Hellman パラメータ構造体。

失敗モードと境界条件

  • dh フィールドが NULL の場合、パラメータがまだ生成されていないことを示します。
  • min 値は鍵長の閾値であり、これより小さい鍵長に対しては当該パラメータが適用されない境界条件を持ちます。

保守担当者向けメモ

  • 本構造体は OpenSSL 1.1.0 未満の API をサポートするための互換性維持に関連しています。
  • dh フィールドのメモリ管理（確保および解放）を行う関数が、本ファイル内の他の箇所に存在するか確認してください。
