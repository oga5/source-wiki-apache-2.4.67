CaseFilterInConfig

定義

  • 種別: struct
  • 定義箇所: modules/examples/mod_case_filter_in.c:35

目的

mod_case_filter_in モジュールにおける、入力フィルタの有効/無効状態を保持するためのサーバー設定構造体です。

シグネチャまたは形状

typedef struct
{
    int bEnabled;
} CaseFilterInConfig;

入力と出力

  • **入力**: bEnabled メンバは、設定ファイル内の CaseFilterIn ディレクティブによって制御されます。
  • **出力**: フィルタの挿入可否を判断するフラグとして機能します。

前提条件と不変条件

  • Apache の設定フェーズにおいて、apr_pcalloc を用いてメモリが確保されることを前提としています。
  • bEnabled は論理値として扱われ、フィルタの実行可否を決定します。

アルゴリズムまたは振る舞い

  • 本構造体は、リクエスト処理時に ap_get_module_config を通じてサーバー設定から取得されます。
  • CaseFilterInInsertFilter 関数内で bEnabled の値が参照され、フィルタをリクエストチェーンに挿入するかどうかの判定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/examples/mod_case_filter_in.c:47 (設定生成時)
  • modules/examples/mod_case_filter_in.c:56 (フィルタ挿入時の設定取得)
  • modules/examples/mod_case_filter_in.c:127 (フィルタ処理時の参照)
  • **呼び出し先**: 構造体メンバへの直接アクセスのみであり、関数呼び出しは含まれません。

関連データ構造

  • CaseFilterInContext: フィルタ実行中のバケットブリゲードなどの状態を保持する構造体。

失敗モードと境界条件

  • 設定が適切に初期化されていない場合、bEnabled が不定値となる可能性があります（apr_pcalloc によるゼロ初期化が推奨されます）。

保守担当者向けメモ

  • 本構造体は mod_case_filter_in.c 内でのみ使用されるプライベートな設定保持用構造体です。
  • フィルタの有効範囲を拡張する場合、本構造体にメンバを追加することで対応可能です。
