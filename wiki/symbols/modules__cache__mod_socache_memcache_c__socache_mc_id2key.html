socache_mc_id2key

定義

  • **種別**: 関数
  • **定義箇所**: modules/cache/mod_socache_memcache.c:183

目的

内部的なバイナリIDを、Memcached で使用可能な文字列形式のキャッシュキーへ変換します。

シグネチャまたは形状

int socache_mc_id2key(ap_socache_instance_t *ctx,
                      const unsigned char *id, unsigned int idlen,
                      char *key, apr_size_t keylen);

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンス（名前空間タグ tag とその長さ taglen を保持）。
  • id: 変換対象のバイナリID。
  • idlen: バイナリIDの長さ。
  • key: 変換後の文字列を格納するバッファ。
  • keylen: key バッファのサイズ。
  • **出力**:
  • 成功時: 0 を返し、key に tag + id の16進数表現が書き込まれる。
  • 失敗時: 1 を返す。

前提条件と不変条件

  • key バッファは、id の2倍の長さと ctx->taglen を合わせたサイズを格納するのに十分な大きさである必要があります。
  • MC_KEY_LEN（254バイト）が実質的なキー長の制限として機能します。

アルゴリズムまたは振る舞い

1. idlen * 2 + ctx->taglen が keylen を超えるか判定し、超える場合はエラー（1）を返します。 2. apr_cpystrn を使用して、ctx->tag を key バッファの先頭にコピーします。 3. ap_bin2hex を使用して、バイナリIDを16進数文字列に変換し、tag の直後に連結します。 4. 成功した場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • socache_mc_store (modules/cache/mod_socache_memcache.c:207)
  • socache_mc_retrieve (modules/cache/mod_socache_memcache.c:239)
  • socache_mc_remove (modules/cache/mod_socache_memcache.c:273)
  • **呼び出し先**:
  • apr_cpystrn (APR)
  • ap_bin2hex (Apache HTTP Server API)

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスのコンテキスト。名前空間タグの管理に使用。

失敗モードと境界条件

  • **バッファオーバーフロー**: 指定された keylen に対して生成されるキーが長すぎる場合、処理を中断し 1 を返します。

保守担当者向けメモ

  • 本関数は、Memcached のキー空間において一意性を確保するために、名前空間タグとバイナリIDを結合する重要な役割を担っています。
  • idlen の2倍のサイズを消費するため、IDの長さと keylen の関係には注意が必要です。
