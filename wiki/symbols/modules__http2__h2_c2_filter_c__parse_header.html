parse_header

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2_filter.c:394

目的

HTTP/1.1 形式のレスポンスヘッダー行を解析し、h2_response_parser コンテキスト内のヘッダー配列へ格納する。

シグネチャまたは形状

static apr_status_t parse_header(h2_response_parser *parser, char *line)

入力と出力

  • **入力**:
  • parser: 解析状態を保持する h2_response_parser 構造体へのポインタ。
  • line: 解析対象となる HTTP/1.1 ヘッダー行（ヌル終端文字列）。
  • **出力**:
  • 戻り値: 成功時は APR_SUCCESS、形式が不正な場合は APR_EINVAL を返す。

前提条件と不変条件

  • parser->hlines は apr_array_header_t として初期化されている必要がある。
  • parser->pool はヘッダー文字列のメモリ割り当てに使用される有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. **継続行の判定**: line の先頭がスペースまたはタブで始まる場合、ヘッダーの継続行とみなす。

  • 直前のヘッダー行を parser->hlines から取得し、現在の行と連結する。
  • 取得できない（配列が空）場合は APR_EINVAL を返す。

    • 2. **新規ヘッダーの判定**: 先頭がスペース/タブ以外の場合、新しいヘッダー行として扱う。 3. **格納**: 処理されたヘッダー行を parser->hlines 配列の末尾に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c2_filter.c:618 (推測される箇所)
  • **呼び出し先**:
  • apr_array_pop (APR)
  • apr_psprintf (APR)
  • apr_pstrdup (APR)
  • APR_ARRAY_PUSH (APR)

関連データ構造

  • h2_response_parser: レスポンス解析の状態を保持する構造体。hlines（ヘッダー行の配列）と pool を含む。

失敗モードと境界条件

  • **不正な継続行**: ヘッダーの先頭行が継続行（スペース/タブ開始）である場合、apr_array_pop が NULL を返し、APR_EINVAL で終了する。

保守担当者向けメモ

  • 本関数は h2_c2_filter.c 内のレスポンス解析ロジックの一部であり、HTTP/1.1 のレガシーなヘッダー形式を HTTP/2 の構造へ変換する役割を担う。
  • 連結されたヘッダー行は parser->pool からメモリが割り当てられるため、メモリリークの懸念はないが、巨大なヘッダー行が繰り返される場合のメモリ消費量に注意が必要。
