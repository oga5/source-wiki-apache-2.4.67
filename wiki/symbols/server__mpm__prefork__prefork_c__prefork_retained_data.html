prefork_retained_data

定義

  • 種別: struct
  • 定義箇所: server/mpm/prefork/prefork.c:101

目的

Apache HTTP Server の再起動（AP_SIG_GRACEFUL 等）を跨いで、Prefork MPM のプロセス管理状態やスケーリング情報を保持するために使用されます。

シグネチャまたは形状

typedef struct prefork_retained_data {
    ap_unixd_mpm_retained_data *mpm;
    int first_server_limit;
    int maxclients_reported;
    int max_daemons_limit;
    int idle_spawn_rate;
    int hold_off_on_exponential_spawning;
} prefork_retained_data;

入力と出力

  • 本構造体はデータ保持用であり、関数ではないため直接的な入力・出力はありません。
  • 内部メンバは、プロセス生成数やスケーリングの調整ロジック（perform_idle_server_maintenance 等）によって読み書きされます。

前提条件と不変条件

  • max_daemons_limit は、MaxRequestWorkers の変更を伴う再起動時においても、スコアボード全体をスキャンするルーチンの最適化のために保持されます。
  • idle_spawn_rate は、アイドルサーバーが不足している際の次回のメンテナンスサイクルで生成する子プロセス数を示します。

アルゴリズムまたは振る舞い

  • **指数的生成制御**: idle_spawn_rate は、必要に応じて最大 MAX_SPAWN_RATE（デフォルト 32）まで倍増します。生成の必要がないサイクルが経過した際にリセットされます。
  • **世代管理**: 再起動を跨いで情報を保持することで、設定変更に対する一貫したプロセス管理を実現します。

呼び出し元と呼び出し先

  • 呼び出し元: prefork_pre_config フックにて初期化されます。
  • 呼び出し先: モジュール内のプロセス管理関数（perform_idle_server_maintenance 等）が、この構造体のインスタンスである retained を参照します。
  • 注: 具体的な関数呼び出しの網羅的なリストは、提供されたソース excerpt には含まれていません。

関連データ構造

  • ap_unixd_mpm_retained_data: Unix 固有の MPM 保持データ。
  • retained: prefork.c 内で定義される prefork_retained_data 型の静的インスタンス。

失敗モードと境界条件

  • MAX_SPAWN_RATE が定義されていない場合、デフォルトで 32 が使用されます。
  • ServerLimit の変更は再起動を跨いで反映されない制約があり、本構造体はその制限下での整合性を維持する役割を担います。

保守担当者向けメモ

  • 本構造体は再起動を跨ぐため、メンバの追加や変更を行う際は、共有メモリレイアウトや再起動時の互換性に注意してください。
  • idle_spawn_rate の指数的増加ロジックは、急激な負荷変動に対する応答性とシステム負荷のバランスを制御しています。
