md_result_problem_set

定義

  • 種別: function
  • 定義箇所: modules/md/md_result.c:100

目的

md_result_t オブジェクトにエラー状態、問題の詳細、および関連するサブ問題情報を設定し、状態変更イベントを通知します。

シグネチャまたは形状

void md_result_problem_set(md_result_t *result, apr_status_t status,
                           const char *problem, const char *detail, 
                           const md_json_t *subproblems)

入力と出力

  • result: 更新対象の md_result_t 構造体へのポインタ。
  • status: 設定する apr_status_t 型のステータスコード。
  • problem: エラーの種類を示す文字列（dup_trim によりトリミングされて格納）。
  • detail: エラーの詳細説明文字列。
  • subproblems: 関連する詳細な問題情報を含む md_json_t 構造体（NULL 指定可能）。

前提条件と不変条件

  • result は有効な md_result_t 構造体を指している必要があります。
  • result が保持する apr_pool_t を介してメモリが割り当てられます。
  • subproblems が提供された場合、md_json_clone を用いて result のプール内に複製されます。

アルゴリズムまたは振る舞い

1. result->status に status を代入します。 2. problem 文字列を dup_trim で処理し、result->problem に格納します。 3. detail 文字列を apr_pstrdup で複製し、result->detail に格納します。 4. subproblems が存在する場合、md_json_clone を使用して複製し result->subproblems に格納します。存在しない場合は NULL を設定します。 5. 最後に on_change(result) を呼び出し、登録されたコールバックを通じて状態変更を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c
  • modules/md/md_acme_acct.c
  • modules/md/md_acme_order.c
  • **呼び出し先**:
  • dup_trim (内部ユーティリティ)
  • apr_pstrdup (APR)
  • md_json_clone (JSON操作)
  • on_change (内部コールバック通知)

関連データ構造

  • md_result_t: 処理結果を保持するメイン構造体。
  • md_json_t: サブ問題情報を保持するためのJSON構造体。
  • apr_pool_t: メモリ管理に使用されるAPRプール。

失敗モードと境界条件

  • problem や detail が NULL の場合、apr_pstrdup や dup_trim の挙動に依存します。
  • メモリ不足時には、APRのメモリ割り当て関数がプロセスを終了させる可能性があります。

保守担当者向けメモ

  • 本関数は md_result_t のフィールドを直接更新するため、状態変更の副作用（コールバック実行）が必ず発生します。
  • subproblems は深いコピー（Deep Copy）が行われるため、呼び出し元で渡した元のJSONオブジェクトの生存期間とは独立して管理されます。
