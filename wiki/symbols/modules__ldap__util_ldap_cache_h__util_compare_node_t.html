util_compare_node_t

定義

  • 種別: struct
  • 定義箇所: modules/ldap/util_ldap_cache.h:127

目的

LDAP サーバーに対する比較操作（DN、属性、値の組み合わせ）の結果をキャッシュし、同一クエリに対する再検索を回避するために使用されます。

シグネチャまたは形状

typedef struct util_compare_node_t {
    const char *dn;
    const char *attrib;
    const char *value;
    apr_time_t lastcompare;
    int result;
    int sgl_processed;
    struct util_compare_subgroup_t *subgroupList;
} util_compare_node_t;

入力と出力

  • **入力（キー）**: dn, attrib, value の3要素が組み合わさることで、キャッシュ内のユニークなキーとして機能します。
  • **出力（保持データ）**: result（比較結果）、lastcompare（最終比較時刻）、subgroupList（サブグループ情報）を保持します。

前提条件と不変条件

  • キャッシュ内のデータは util_ald_alloc を使用して確保されたメモリ領域に配置される必要があります。
  • sgl_processed は、サブグループリスト（SGL）の処理状態を示すフラグであり、0 は未処理、1 は処理済み（NULL の場合を含む）を意味します。

アルゴリズムまたは振る舞い

  • util_ldap_cache.c 内の関数において、キャッシュノードの比較、コピー、およびメモリ確保の対象として利用されます。
  • sgl_processed フラグにより、リーフノードにおける重複したサブグループ処理を抑制します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c および modules/ldap/util_ldap_cache.c 内の各関数から、キャッシュノードの操作や参照のために利用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、subgroupList を通じて util_compare_subgroup_t を参照します。

関連データ構造

  • util_compare_subgroup_t: subgroupList メンバにより参照される関連構造体。
  • util_url_node_t: 本ノードを保持する上位のキャッシュ階層。

失敗モードと境界条件

  • メモリ不足時: util_ald_alloc によるメモリ確保が失敗する可能性があります。
  • 共有メモリ環境下: apr_rmm を介したオフセット管理が必要であり、ポインタの直接参照は無効となる場合があります。

保守担当者向けメモ

  • 本構造体は util_ldap_cache.c の実装と密接に結合しています。ノードの追加や削除を行う際は、必ず util_ald_cache_fetch/insert 等のキャッシュ管理インターフェースを介してください。
  • sgl_processed フラグの更新ロジックを変更する場合は、リーフノードの処理効率に影響を与えるため注意してください。
