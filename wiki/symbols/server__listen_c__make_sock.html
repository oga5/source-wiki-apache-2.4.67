make_sock

定義

  • 種別: function
  • 定義箇所: server/listen.c:69

目的

ネットワークソケットの作成、各種ソケットオプションの適用、およびバインドとリスン処理を行う低レベル関数です。

シグネチャまたは形状

static apr_status_t make_sock(apr_pool_t *p, ap_listen_rec *server, int do_bind_listen)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • server: ソケット情報およびバインド先アドレスを保持する ap_listen_rec 構造体。
  • do_bind_listen: バインドおよびリスン処理を実行するかどうかを示すフラグ。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返します。

前提条件と不変条件

  • server->sd には既に初期化済みの apr_socket_t が格納されている必要があります。
  • do_bind_listen が真の場合、server->bind_addr が有効である必要があります。
  • 成功時には server->sd が更新され、server->active が 1 に設定されます。

アルゴリズムまたは振る舞い

1. **オプション設定**: SO_REUSEADDR（Windows以外）、SO_KEEPALIVE を適用します。 2. **バッファ設定**: SendBufferSize および ReceiveBufferSize が設定されている場合、SO_SNDBUF / SO_RCVBUF を適用します。 3. **TCP設定**: プラットフォームがサポートしていれば ap_sock_disable_nagle を呼び出します。 4. **マルチコア最適化**: SO_REUSEPORT が利用可能かつ設定されている場合、ソケットに適用します。 5. **バインド・リスン**: do_bind_listen が真の場合、IPv6 の IPV6_V6ONLY 設定（該当時）、apr_socket_bind、apr_socket_listen を順次実行します。 6. **Windows固有処理**: Windows環境では、バインド後に再度 SO_REUSEADDR を適用します。 7. **状態更新**: 成功時に server->active を 1 にし、server->accept_func を NULL に初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/listen.c:338
  • server/listen.c:633
  • **呼び出し先**:
  • APR ネットワーク関数 (apr_socket_opt_set, apr_socket_bind, apr_socket_listen, apr_socket_close 等)
  • ap_log_perror (ログ出力)
  • ap_sock_disable_nagle (TCP_NODELAY設定)
  • setsockopt (OSネイティブのソケットオプション設定)

関連データ構造

  • ap_listen_rec: リスナーのソケット、アドレス、状態を管理する構造体。
  • apr_socket_t: APR のソケット抽象化オブジェクト。

失敗モードと境界条件

  • ソケットオプション設定、バインド、またはリスン処理が失敗した場合、apr_socket_close を呼び出してソケットを閉じ、エラーコードを返します。
  • SO_SNDBUF / SO_RCVBUF の設定失敗は致命的なエラーとは見なされず、警告ログを出力して処理を継続します。

保守担当者向けメモ

  • 本関数は APR の抽象化が不十分な箇所があり、OS 固有のソケットオプション設定が直接記述されています。
  • SO_REUSEPORT の挙動はプラットフォーム依存であるため、変更時は注意が必要です。
  • systemd 関連のロジックは、ソース内のマクロ定義により環境依存の挙動が含まれる可能性があります。
