h2_session_event_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_session.h:50

目的

HTTP/2 セッションの状態遷移をトリガーし、セッション管理のロジックへイベントを伝達するために使用されます。

シグネチャまたは形状

typedef enum {
    H2_SESSION_EV_INIT,
    H2_SESSION_EV_INPUT_PENDING,
    H2_SESSION_EV_INPUT_EXHAUSTED,
    H2_SESSION_EV_LOCAL_GOAWAY,
    H2_SESSION_EV_REMOTE_GOAWAY,
    H2_SESSION_EV_CONN_ERROR,
    H2_SESSION_EV_PROTO_ERROR,
    H2_SESSION_EV_CONN_TIMEOUT,
    H2_SESSION_EV_NGH2_DONE,
    H2_SESSION_EV_MPM_STOPPING,
    H2_SESSION_EV_PRE_CLOSE,
    H2_SESSION_EV_NO_MORE_STREAMS,
    H2_SESSION_EV_BAD_CLIENT,
} h2_session_event_t;

入力と出力

  • **入力**: セッションのライフサイクルにおける特定の状況（初期化、エラー、タイムアウト、プロトコル状態の変化など）を表現する列挙値。
  • **出力**: なし（本シンボルは状態を識別するための識別子です）。

前提条件と不変条件

  • h2_session 構造体の状態管理において、このイベントが h2_session_dispatch_event に渡されることで、セッションの内部状態が遷移します。

アルゴリズムまたは振る舞い

  • セッションのメインループにおいて、発生した事象を h2_session_event または h2_session_dispatch_event に渡すことで、セッションの挙動を制御します。
  • 例えば、H2_SESSION_EV_LOCAL_GOAWAY はローカル側からの GOAWAY フレーム送信を、H2_SESSION_EV_CONN_ERROR は接続エラーの発生をそれぞれ通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_session.c 内のセッション処理ロジック。
  • **呼び出し先**: h2_session_event および h2_session_dispatch_event 関数が、この列挙型を引数として受け取ります。

関連データ構造

  • h2_session: セッションの全状態を保持するメイン構造体。

失敗モードと境界条件

  • 不適切なタイミングでイベントがディスパッチされた場合、セッションの状態遷移が予期しない動作を引き起こす可能性があります。

保守担当者向けメモ

  • nghttp2 ライブラリの状態と Apache 側のセッション状態を同期させるための重要なトリガーです。
  • 新しいイベントを追加する場合は、h2_session_dispatch_event 内のスイッチ文で適切に処理されることを確認してください。
