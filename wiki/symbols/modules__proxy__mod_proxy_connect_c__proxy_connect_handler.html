proxy_connect_handler

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_connect.c:147

目的

HTTP CONNECT メソッドを処理し、クライアントとリモートサーバー間のSSL/TLSトンネリングを確立・仲介する。

シグネチャまたは形状

static int proxy_connect_handler(request_rec *r, proxy_worker *worker,
                                 proxy_server_conf *conf,
                                 char *url, const char *proxyname,
                                 apr_port_t proxyport)

入力と出力

  • **入力**:
  • r: リクエストレコード。
  • worker: プロキシワーカー設定。
  • conf: プロキシサーバー設定。
  • url: 接続先URL。
  • proxyname, proxyport: 上位プロキシを使用する場合の接続先情報。
  • **出力**:
  • int: 処理結果ステータス（OK, DECLINED, HTTP_BAD_REQUEST, HTTP_FORBIDDEN, HTTP_SERVICE_UNAVAILABLE, HTTP_INTERNAL_SERVER_ERROR）。

前提条件と不変条件

  • リクエストメソッドが M_CONNECT であること。
  • AllowCONNECT ディレクティブにより、対象ポートが許可されていること。
  • ap_proxy_checkproxyblock2 によるアクセス制限チェックを通過すること。

アルゴリズムまたは振る舞い

1. **検証**: リクエストメソッドが CONNECT か確認し、それ以外は DECLINED を返す。 2. **宛先解決**: URLを解析し、DNSルックアップを実行して接続先アドレスを取得。 3. **接続**: ap_proxy_connect_to_backend を呼び出し、バックエンドへのソケット接続を確立。 4. **接続設定**: ap_run_create_connection でバックエンド接続を生成し、SSLエンジン等の初期化を行う。 5. **応答**:

  • 上位プロキシ経由の場合: CONNECT リクエストを転送。
  • 直接接続の場合: クライアントへ 200 Connection Established を送信。

    • 6. **トンネル**: ap_proxy_tunnel_create および ap_proxy_tunnel_run を使用し、ソケット間の双方向データ転送を実行。 7. **終了**: ap_lingering_close を用いて接続を適切にクローズ。

呼び出し元と呼び出し先

  • **呼び出し元**: proxy_hook_scheme_handler に登録され、プロキシサブシステムから呼び出される。
  • **呼び出し先**:
  • apr_uri_parse_hostinfo, apr_sockaddr_info_get (URL/DNS処理)
  • ap_proxy_checkproxyblock2, allowed_port (検証)
  • ap_proxy_connect_to_backend (接続)
  • ap_run_create_connection, ap_run_pre_connection (接続生成)
  • ap_proxy_tunnel_create, ap_proxy_tunnel_run (トンネル処理)
  • ap_lingering_close (終了)

関連データ構造

  • connect_conf: 許可ポート範囲を管理する設定構造体。
  • proxy_tunnel_rec: トンネル状態を保持する構造体。
  • request_rec, conn_rec: Apache HTTP Server のリクエストおよび接続管理構造体。

失敗モードと境界条件

  • **DNS解決失敗**: HTTP_BAD_GATEWAY を返す。
  • **アクセス拒否**: AllowCONNECT 違反や ProxyBlock 制限により HTTP_FORBIDDEN を返す。
  • **接続失敗**: バックエンドへの接続が確立できない場合、HTTP_SERVICE_UNAVAILABLE または DECLINED を返す。
  • **トンネルエラー**: ap_proxy_tunnel_run がエラーを返した場合、状況に応じてエラーコードを伝播する。

保守担当者向けメモ

  • 本関数は透過的なトンネルを目的としており、HTTPヘッダー解析やログ出力は最小限に留められている。
  • トンネル作成時に割り当てられるリソースは r->pool を使用しており、リクエスト終了時に自動的に解放される。
