mkrecord

定義

  • 種別: function
  • 定義箇所: support/htpasswd.c:74

目的

ユーザー名とハッシュ化されたパスワードを結合し、htpasswd ファイル形式（user:hash）のレコード文字列を生成する。

シグネチャまたは形状

static int mkrecord(struct passwd_ctx *ctx, char *user)

入力と出力

  • **入力**:
  • ctx: 認証コンテキスト（passwd_ctx 構造体）。ハッシュ生成に必要な設定とメモリプールを含む。
  • user: レコードに含めるユーザー名文字列。
  • **出力**:
  • 戻り値: 成功時は 0、失敗時はエラーコード（ERR_OVERFLOW 等）。
  • 副作用: ctx->out に生成されたレコード文字列が格納され、ctx->out_len にその長さが設定される。

前提条件と不変条件

  • ctx->pool が有効なメモリプールとして初期化されていること。
  • mkhash(ctx) が呼び出され、ctx 内にハッシュ値が準備されていること。
  • ユーザー名に : を含めてはならない（ファイル形式の区切り文字と競合するため）。

アルゴリズムまたは振る舞い

1. 一時バッファ hash_str を確保し、ctx->out を一時的にこのバッファへ向ける。 2. mkhash(ctx) を呼び出し、ハッシュ値を生成する。失敗した場合は即座にエラーを返す。 3. apr_pstrcat を使用し、user、:、hash_str、および改行コードを結合した文字列を ctx->pool 上に生成する。 4. 生成された文字列の長さを確認し、MAX_STRING_LEN を超える場合は ERR_OVERFLOW を返す。 5. 成功時は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htpasswd.c:385（main 関数内から呼び出される）。
  • **呼び出し先**: mkhash（passwd_common.c で定義）、apr_pstrcat、strlen。

関連データ構造

  • passwd_ctx: 認証処理の状態を保持する構造体。passwd_common.h で定義。

失敗モードと境界条件

  • mkhash が失敗した場合、そのエラーコードを伝播する。
  • 生成されたレコード長が MAX_STRING_LEN 以上の場合、ctx->errstr にエラーメッセージをセットし ERR_OVERFLOW を返す。

保守担当者向けメモ

  • 本関数は apr_pstrcat を使用してメモリを割り当てるため、呼び出し元の apr_pool_t のライフサイクルに依存する。
  • ユーザー名に : が含まれる場合のバリデーションは本関数の外部で行う必要がある。
  • MAX_STRING_LEN を超えるレコードは拒否されるため、非常に長いパスワードやユーザー名には注意が必要。
