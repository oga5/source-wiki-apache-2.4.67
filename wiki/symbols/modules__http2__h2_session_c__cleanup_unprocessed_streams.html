cleanup_unprocessed_streams

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:109

目的

HTTP/2 セッション終了時または異常発生時に、まだ処理が完了していないストリームを特定し、それらをリセット（RST_STREAM）してリソースを解放する。

シグネチャまたは形状

static void cleanup_unprocessed_streams(h2_session *session)

入力と出力

  • **入力**: session (h2_session *) - 対象となる HTTP/2 セッション構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • session ポインタは有効であること。
  • session->mplx が初期化されており、多重化器として機能していること。

アルゴリズムまたは振る舞い

1. h2_mplx_c1_streams_do 関数を呼び出し、セッション内の多重化器が管理する全ストリームを走査する。 2. 各ストリームに対して rst_unprocessed_stream コールバックを適用する。 3. これにより、未処理のストリームに対してリセット処理が実行される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:1528
  • modules/http2/h2_session.c:1538
  • **呼び出し先**:
  • h2_mplx_c1_streams_do
  • rst_unprocessed_stream（コールバックとして渡される）

関連データ構造

  • h2_session: HTTP/2 セッションの状態を保持するメイン構造体。
  • h2_mplx: ストリームの多重化とワーカー管理を行う構造体。

失敗モードと境界条件

  • session->mplx が NULL の場合、h2_mplx_c1_streams_do 内部での挙動に依存する（通常はセッション初期化不備によるクラッシュの可能性がある）。

保守担当者向けメモ

  • 本関数はセッション終了処理の一部として呼び出される。ストリームのリセットロジックを変更する場合は、コールバック関数である rst_unprocessed_stream を参照すること。
