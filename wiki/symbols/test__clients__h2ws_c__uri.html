uri

定義

  • 種別: struct
  • 定義箇所: test/clients/h2ws.c:206

目的

WebSocket 接続先 URI の各構成要素を保持し、解析結果を格納するために使用されます。

シグネチャまたは形状

struct uri {
  const char *scheme;
  const char *host;
  const char *authority;
  const char *path;
  uint16_t port;
  int ipv6;
};

入力と出力

  • 本構造体は parse_uri() 関数によって入力文字列から各フィールドが抽出・設定されます。
  • 出力として、h2_session の初期化や接続先情報の特定に使用されます。

前提条件と不変条件

  • scheme は ws または wss を想定しています。
  • port は uint16_t 型であり、有効なポート番号が格納されます。
  • ipv6 はフラグとして機能し、ホストが IPv6 アドレスである場合に非ゼロとなります。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータの保持のみを行うコンテナです。
  • h2ws.c 内の parse_uri() 関数が、入力された URI 文字列を分解し、本構造体の各メンバにポインタまたは値を割り当てます。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数内の parse_uri() 呼び出しにより初期化されます。
  • 呼び出し先: 本構造体は h2_session_open() 等のセッション管理関数へ渡され、接続先情報の参照元として利用されます。
  • 注記: ツールが提示した include/ 配下の ap_parse_uri 等との関連性は、名前の類似によるものであり、本構造体との直接的な呼び出し関係はソースコード上確認できません。

関連データ構造

  • h2_session: 接続先情報を保持する uri 構造体を利用してセッションを構築します。

失敗モードと境界条件

  • parse_uri() が解析に失敗した場合、本構造体の各メンバは未初期化または NULL となる可能性があります。
  • wss:// (TLS) が指定された場合、本構造体はそれを認識しますが、本ツール自体は TLS ハンドシェイクを実装していないため、接続処理において境界条件となります。

保守担当者向けメモ

  • 本構造体はテスト用クライアント h2ws.c 専用の定義です。Apache HTTP Server 本体の request_rec 等で使用される URI 関連構造体とは別物である点に注意してください。
  • メモリ所有権については、parse_uri() が入力文字列をどのように分割・保持しているか（静的メモリか動的確保か）を h2ws.c の実装詳細で確認してください。
