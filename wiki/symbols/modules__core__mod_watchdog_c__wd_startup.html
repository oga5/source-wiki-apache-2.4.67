wd_startup

定義

  • 種別: function
  • 定義箇所: modules/core/mod_watchdog.c:268

目的

Watchdog インスタンスの実行スレッドを生成し、ライフサイクル管理のためのクリーンアップハンドラを登録する。

シグネチャまたは形状

apr_status_t wd_startup(ap_watchdog_t *w, apr_pool_t *p)

入力と出力

  • **入力**:
  • w: 起動対象の ap_watchdog_t 構造体へのポインタ。
  • p: スレッドおよびクリーンアップ処理に使用するメモリプール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • w->thread_started はアトミックに 0 にリセットされる。
  • w->singleton が真の場合、w->mutex が初期化済みである必要がある。

アルゴリズムまたは振る舞い

1. w->thread_started を 0 に設定する。 2. w->singleton が有効な場合、apr_proc_mutex_child_init を呼び出し、子プロセス内での mutex を初期化する。失敗した場合は即座にエラーを返す。 3. ap_thread_create を呼び出し、wd_worker をエントリポイントとしてスレッドを生成する。 4. スレッド生成に成功した場合、apr_pool_pre_cleanup_register を使用して、プール破棄時に wd_worker_cleanup が実行されるよう登録する。 5. スレッド生成の結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/core/mod_watchdog.c:479
  • modules/core/mod_watchdog.c:576
  • **呼び出し先**:
  • apr_atomic_set32
  • apr_proc_mutex_child_init
  • apr_proc_mutex_lockfile
  • ap_thread_create
  • wd_worker (スレッドエントリ)
  • apr_pool_pre_cleanup_register
  • wd_worker_cleanup (クリーンアップハンドラ)

関連データ構造

  • ap_watchdog_t: 監視インスタンスの状態を保持する構造体。
  • apr_pool_t: リソース管理用プール。

失敗モードと境界条件

  • apr_proc_mutex_child_init が失敗した場合、そのエラーコードを返して終了する。
  • ap_thread_create が失敗した場合、スレッドは生成されず、クリーンアップハンドラも登録されない。

保守担当者向けメモ

  • 本関数はスレッドの生成とクリーンアップの紐付けを行うため、呼び出し側は p のライフサイクルがスレッドの生存期間を包含していることを保証する必要がある。
