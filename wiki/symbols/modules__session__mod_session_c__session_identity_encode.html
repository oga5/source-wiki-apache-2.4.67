session_identity_encode

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:357

目的

session_rec 内のセッションデータを、URLエンコード形式の文字列にシリアライズし、z->encoded に格納する。

シグネチャまたは形状

static apr_status_t session_identity_encode(request_rec *r, session_rec *z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • z: シリアライズ対象のセッション構造体。z->entries（テーブル）および z->expiry を参照する。
  • **出力**:
  • 戻り値: OK (成功時)。
  • 副作用: z->encoded にシリアライズされた文字列が割り当てられる。

前提条件と不変条件

  • z->entries は apr_table_t として初期化されている必要がある。
  • z->pool および r->pool は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. **有効期限の処理**: z->expiry が設定されている場合、文字列に変換し z->entries テーブルへ SESSION_EXPIRY キーで追加する。 2. **サイズ計算**: identity_count 関数を apr_table_do で呼び出し、全エントリをシリアライズした際の合計長を算出する。 3. **メモリ確保**: r->pool から計算した長さ分（+1）のバッファを確保する。 4. **シリアライズ**: identity_concat 関数を apr_table_do で呼び出し、バッファへデータを書き込む。 5. **格納**: 生成されたバッファを z->encoded に代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session.c:533 (直接呼び出し)
  • modules/session/mod_session.c:692 (フック登録時)
  • **呼び出し先**:
  • identity_count (テーブル走査によるサイズ計算)
  • identity_concat (テーブル走査による文字列連結)
  • apr_psprintf, apr_table_setn, apr_table_do, apr_pcalloc (APR API)

関連データ構造

  • session_rec: セッション情報を保持する構造体。
  • request_rec: Apache リクエスト構造体。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足で失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性がある。
  • z->entries が空の場合、空の文字列が生成される。

保守担当者向けメモ

  • 本関数はデフォルトのシリアライズ形式を定義する。カスタム形式が必要な場合は、別のフック実装を検討すること。
  • identity_count および identity_concat と密結合しているため、変更時はこれらも併せて確認すること。
