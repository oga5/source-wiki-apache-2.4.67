find_route_worker

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:201

目的

指定されたルート文字列に一致するバックエンドワーカーをバランサ内から検索し、必要に応じてリトライやリダイレクト先への再帰的な探索を行う。

シグネチャまたは形状

static proxy_worker *find_route_worker(proxy_balancer *balancer,
                                       const char *route, request_rec *r,
                                       int recursion)

入力と出力

  • **入力**:
  • balancer: 検索対象のバランサ構造体。
  • route: 検索するルート識別子。
  • r: 現在のリクエストレコード。
  • recursion: 再帰回数（リダイレクトループ防止用）。
  • **出力**:
  • 条件に合致し、かつ利用可能な proxy_worker 構造体へのポインタ。見つからない場合は NULL。

前提条件と不変条件

  • balancer->workers に有効なワーカーリストが存在すること。
  • recursion はバランサのワーカー数を超えない範囲で指定されること（再帰呼び出し時にインクリメントされる）。

アルゴリズムまたは振る舞い

1. **二段階検索**: まず非スタンバイワーカーを検索し、見つからなければスタンバイワーカーを検索する。 2. **ルート照合**: 各ワーカーの worker->s->route と引数の route を比較する。 3. **利用可能性の確認**:

  • ワーカーが利用可能（PROXY_WORKER_IS_USABLE）であればそのワーカーを返す。
  • 利用不可の場合、ap_proxy_retry_worker_fn を呼び出して復旧を試みる。復旧成功時はそのワーカーを返す。

    • 4. **リダイレクト処理**:

  • ワーカーが利用不可で、かつ redirect 設定が存在する場合、再帰的に find_route_worker を呼び出し、リダイレクト先のワーカーを探索する。
  • 再帰回数がワーカー数に達した場合はループを避けるため探索を停止する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_balancer.c 内の find_session_route（推測される文脈）および本関数自身の再帰呼び出し。
  • **呼び出し先**:
  • ap_proxy_retry_worker_fn: ワーカーの再試行処理。
  • find_route_worker: 自身を再帰的に呼び出す。

関連データ構造

  • proxy_balancer: バランサの状態とワーカーリストを保持。
  • proxy_worker: 個別のバックエンドワーカー情報。
  • proxy_worker_shared: ワーカーの共有状態（route や redirect 設定を含む）。

失敗モードと境界条件

  • **一致なし**: 指定されたルートを持つワーカーが存在しない場合、NULL を返す。
  • **リダイレクトループ**: recursion が balancer->workers->nelts に達すると、それ以上の再帰探索を打ち切る。
  • **全ワーカー利用不可**: 復旧を試みても利用可能なワーカーが見つからない場合、NULL を返す。

保守担当者向けメモ

  • 本関数はスティッキーセッションのルーティングにおいて中心的な役割を果たす。
  • ap_proxy_retry_worker_fn による復旧処理は、共有メモリ上の状態を更新する副作用を持つ。
  • リダイレクト先探索時の再帰制限は、設定ミスによる無限ループを防止するために重要である。
