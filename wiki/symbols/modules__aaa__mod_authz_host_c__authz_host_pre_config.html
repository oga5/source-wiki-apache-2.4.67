authz_host_pre_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:355

目的

設定解析フェーズで使用するサブネット共有用ハッシュテーブル parsed_subnets を初期化し、ローカルホスト（IPv4/IPv6）のサブネット情報を事前登録します。

シグネチャまたは形状

static int authz_host_pre_config(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • p: 設定用プール。
  • plog: ログ用プール。
  • ptemp: 一時的な設定解析用プール。
  • **出力**:
  • 戻り値: OK（成功時）。

前提条件と不変条件

  • parsed_subnets は設定解析フェーズでのみ使用されるため、ptemp プールを使用して初期化されます。
  • APR_HAVE_IPV6 が定義されている場合のみ、IPv6 のローカルホスト設定が行われます。

アルゴリズムまたは振る舞い

1. apr_hash_make(ptemp) を呼び出し、parsed_subnets を初期化します。 2. apr_ipsubnet_create を使用して IPv4 のローカルサブネット（127.0.0.0/8）を作成し、parsed_subnets に登録します。 3. IPv6 が利用可能な環境では、同様に ::1 を作成し、parsed_subnets に登録します。 4. 最後に OK を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_host.c:383 にて ap_hook_pre_config に登録されています。
  • **呼び出し先**:
  • apr_hash_make
  • apr_ipsubnet_create
  • apr_hash_set

関連データ構造

  • parsed_subnets: サブネット情報を保持するハッシュテーブル。
  • localhost_v4, localhost_v6: ローカルホスト判定用の apr_ipsubnet_t 構造体。

失敗モードと境界条件

  • メモリ不足により apr_hash_make や apr_ipsubnet_create が失敗した場合の挙動は、APR の標準的なエラー処理に依存します。

保守担当者向けメモ

  • 本関数で初期化された parsed_subnets は、設定解析フェーズ終了後に破棄されることが想定されています（post_config 等でのクリーンアップを確認してください）。
  • 実行時の認可判断には影響しないため、設定の最適化のみを目的としています。
