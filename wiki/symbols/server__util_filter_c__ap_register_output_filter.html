ap_register_output_filter

定義

  • 種別: function
  • 定義箇所: server/util_filter.c:257

目的

出力フィルタをシステムに登録し、フィルタチェーンへの挿入を可能にするためのインターフェースを提供します。

シグネチャまたは形状

ap_filter_rec_t *ap_register_output_filter(const char *name,
                                           ap_out_filter_func filter_func,
                                           ap_init_filter_func filter_init,
                                           ap_filter_type ftype)

入力と出力

  • **入力**:
  • name: フィルタを一意に識別する名称。
  • filter_func: フィルタ処理を実行するコールバック関数。
  • filter_init: フィルタ初期化時に呼び出されるコールバック関数（NULL 可）。
  • ftype: フィルタの優先度を決定する型（ap_filter_type）。
  • **出力**:
  • 登録されたフィルタの定義情報を含む ap_filter_rec_t 構造体へのポインタを返します。

前提条件と不変条件

  • 内部で ap_register_output_filter_protocol を呼び出しており、プロトコルフラグはデフォルトの 0 が設定されます。
  • フィルタ名はシステム内で一意である必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った name, filter_func, filter_init, ftype をそのまま ap_register_output_filter_protocol へ転送します。 2. プロトコルフラグに 0 を指定して、フィルタの登録処理を完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/netware/mod_nw_ssl.c
  • modules/cache/mod_cache.c
  • その他、フィルタを定義する各モジュール。
  • **呼び出し先**:
  • ap_register_output_filter_protocol（server/util_filter.c 内の内部関数）。

関連データ構造

  • ap_filter_rec_t: フィルタの定義を保持する構造体。
  • ap_out_filter_func: 出力フィルタの関数型。
  • ap_init_filter_func: フィルタ初期化関数の型。

失敗モードと境界条件

  • 登録処理の失敗に関する具体的なエラーハンドリングの詳細は、本関数単体では定義されていません（ap_register_output_filter_protocol の実装に依存します）。

保守担当者向けメモ

  • 本関数は ap_register_output_filter_protocol のラッパーです。プロトコル関連の拡張が必要な場合は、直接 ap_register_output_filter_protocol を使用することを検討してください。
