ap_get_scoreboard_process

定義

  • 種別: function
  • 定義箇所: server/scoreboard.c:718

目的

指定されたインデックスに対応するプロセススコア構造体（process_score）へのポインタを取得します。

シグネチャまたは形状

process_score *ap_get_scoreboard_process(int x)

入力と出力

  • **入力**: int x（取得対象のプロセススロット番号）
  • **出力**: 成功時は process_score 構造体へのポインタ、失敗時は NULL を返します。

前提条件と不変条件

  • ap_scoreboard_image が初期化済みである必要があります。
  • x は 0 以上 server_limit 未満である必要があります。

アルゴリズムまたは振る舞い

1. 引数 x が 0 未満、または server_limit 以上であるかを確認します。 2. 範囲外の場合は NULL を返します。 3. 範囲内の場合、グローバルなスコアボードイメージ ap_scoreboard_image 内の parent 配列から、指定されたインデックス x の要素のアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cluster/mod_heartbeat.c
  • modules/generators/mod_status.c
  • modules/lua/lua_request.c
  • server/mpm/event/event.c
  • server/mpm/prefork/prefork.c
  • server/mpm/worker/worker.c
  • server/mpm_unix.c
  • **呼び出し先**: 外部関数呼び出しはありません。

関連データ構造

  • process_score: プロセスごとの状態を保持する構造体（include/scoreboard.h で定義）。
  • ap_scoreboard_image: スコアボード全体を管理するグローバル構造体。

失敗モードと境界条件

  • x が負の値、または server_limit 以上の場合、境界外アクセスを防ぐために NULL を返します。

保守担当者向けメモ

  • 本関数は共有メモリ上の parent 配列へ直接アクセスします。server_limit は MPM の設定に依存するため、実行時の構成変更には注意が必要です。
