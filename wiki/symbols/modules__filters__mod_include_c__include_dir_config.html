include_dir_config

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_include.c:115

目的

SSI（Server Side Includes）機能におけるディレクトリ単位の設定情報を保持する構造体です。

シグネチャまたは形状

typedef struct {
    const char *default_error_msg;
    const char *default_time_fmt;
    const char *undefined_echo;
    xbithack_t  xbithack;
    signed char lastmodified;
    signed char etag;
    signed char legacy_expr;
} include_dir_config;

入力と出力

  • **入力**: Apache の設定ファイルから読み込まれたディレクトリごとの設定値。
  • **出力**: SSI フィルタ実行時に、現在のリクエストに対する動作（エラーメッセージの形式、時刻フォーマット、XBitHack の有効化状態など）を決定するためのパラメータとして使用されます。

前提条件と不変条件

  • ap_get_module_config を介して per_dir_config から取得されることを前提としています。
  • apr_pcalloc 等でメモリ確保された後、設定値が割り当てられます。

アルゴリズムまたは振る舞い

  • SSI フィルタ（includes_filter）がリクエストを処理する際、この構造体を参照して SSI ディレクティブの挙動を制御します。
  • 設定の結合（マージ）処理において、ベースとなる設定とオーバーライドする設定を組み合わせて新しいインスタンスを生成する際に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_include.c 内のフィルタ処理関数（includes_filter 等）および設定マージ関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • xbithack_t: xbithack フィールドの型として使用される列挙型または整数型。

失敗モードと境界条件

  • 設定が正しく初期化されていない場合、default_error_msg や default_time_fmt が未定義（NULL）の状態で参照される可能性があります。
  • メモリ確保失敗時は apr_palloc 等がエラーを返しますが、本構造体自体に固有の境界条件はありません。

保守担当者向けメモ

  • 本構造体は mod_include.c の内部設定管理用です。
  • 設定値の追加や変更を行う際は、設定マージ関数（merge_include_dir_configs 相当の処理）での初期化漏れに注意してください。
