open_log

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_forensic.c:71

目的

サーバー設定に基づき、フォレンジックログの出力先（ファイルまたはパイプ）を初期化し、ファイルディスクリプタを確保する。

シグネチャまたは形状

static int open_log(server_rec *s, apr_pool_t *p)

入力と出力

  • **入力**:
  • s: サーバー設定を保持する server_rec 構造体。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**:
  • 成功時は 1 を返し、cfg->fd に有効なファイルディスクリプタが格納される。
  • 失敗時は 0 を返す。

前提条件と不変条件

  • fcfg 構造体から取得した cfg->logname が設定されていること。
  • 既に cfg->fd が設定されている場合、処理はスキップされる。
  • パイプ出力の場合、logname の先頭文字が | である必要がある。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のサーバー設定 fcfg を取得する。 2. logname が未設定、または既に fd が開かれている場合は即座に 1 を返す。 3. logname が | で始まる場合:

  • ap_open_piped_log を呼び出し、パイプ経由のログ出力を試みる。
  • 失敗した場合はエラーログを出力し 0 を返す。
  • 成功した場合は ap_piped_log_write_fd で取得したディスクリプタを cfg->fd に保存する。

    • 4. それ以外（通常ファイル）の場合:

  • apr_file_open を使用し、APR_WRITE | APR_APPEND | APR_CREATE モードでファイルを開く。
  • 失敗した場合はエラーログを出力し 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/loggers/mod_log_forensic.c:110 (log_init 関数内から呼び出されることが確認されている)
  • **呼び出し先**:
  • ap_get_module_config
  • ap_server_root_relative
  • ap_open_piped_log
  • ap_log_error
  • ap_piped_log_write_fd
  • apr_file_open

関連データ構造

  • fcfg: ログファイル名とファイルディスクリプタを保持する設定構造体。
  • server_rec: Apache のサーバー設定を保持する構造体。
  • piped_log: パイプ経由のログ出力を管理する構造体。

失敗モードと境界条件

  • パイプの生成失敗（ap_open_piped_log が NULL を返す場合）。
  • ファイルのオープン失敗（権限不足やパスの誤りなど、apr_file_open が APR_SUCCESS 以外を返す場合）。
  • いずれの失敗時も ap_log_error を通じてエラーが記録される。

保守担当者向けメモ

  • 本関数はサーバー起動時の初期化フェーズで使用される。
  • cfg->fd の管理は fcfg 構造体を通じて行われるため、マルチスレッド環境での整合性に注意すること。
  • パイプ出力とファイル出力の分岐ロジックは、mod_log_forensic の設定ディレクティブの仕様に依存している。
