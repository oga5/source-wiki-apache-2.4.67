ap_filter_type

定義

  • 種別: enum
  • 定義箇所: include/util_filter.h:158

目的

Apache HTTP Server のフィルタチェーンにおけるフィルタの優先順位および分類を定義します。フィルタの実行順序を決定するための指標として使用されます。

シグネチャまたは形状

typedef enum {
    AP_FTYPE_RESOURCE    = 10,
    AP_FTYPE_CONTENT_SET = 20,
    AP_FTYPE_PROTOCOL    = 30,
    AP_FTYPE_TRANSCODE   = 40,
    AP_FTYPE_CONNECTION  = 50,
    AP_FTYPE_NETWORK     = 60
} ap_filter_type;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: フィルタの分類を示す整数値（10〜60）

前提条件と不変条件

  • AP_FTYPE_CONNECTION 以上のフィルタは、サブリクエスト内では使用できません。
  • フィルタは本列挙型の値に基づいてソートされ、実行順序が決定されます。同じ型の場合は FIFO で実行されます。

アルゴリズムまたは振る舞い

  • AP_FTYPE_RESOURCE: コンテンツ変換用（SSI, PHP等）。
  • AP_FTYPE_CONTENT_SET: コンテンツ全体への変更用（deflate等）。RESOURCE フィルタの後に実行されます。
  • AP_FTYPE_PROTOCOL: サーバー・クライアント間のプロトコル処理（HTTP, POP等）。
  • AP_FTYPE_TRANSCODE: トランスポートエンコーディング（chunking等）。
  • AP_FTYPE_CONNECTION: 接続に関連する処理（複数リクエストの分割やバッファリング等）。
  • AP_FTYPE_NETWORK: ネットワーク送受信（コンテンツ変更は行わない）。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • include/util_filter.h: ap_filter_rec_t 構造体メンバ ftype 等。
  • modules/filters/mod_ext_filter.c: フィルタ定義。
  • modules/http2/h2_switch.c: remove_output_filters_below / remove_input_filters_below 関数。
  • server/util_filter.c: フィルタ登録およびチェーン管理ロジック。
  • 呼び出し先: なし（列挙型のため）。

関連データ構造

  • ap_filter_rec_t: フィルタ定義情報。
  • ap_filter_t: インストールされたフィルタインスタンス。

失敗モードと境界条件

  • サブリクエスト内で AP_FTYPE_CONNECTION 以上のフィルタを適用しようとした場合、意図した動作にならない可能性があります。

保守担当者向けメモ

  • フィルタの実行順序を変更する際は、本列挙型の値の大小関係がチェーンの構成に直結するため注意してください。
  • 新しいフィルタ型を追加する場合は、既存の数値間隔（10刻み）を考慮して挿入してください。
