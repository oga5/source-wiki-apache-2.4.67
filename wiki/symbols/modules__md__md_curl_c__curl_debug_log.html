curl_debug_log

定義

  • 種別: function
  • 定義箇所: modules/md/md_curl.c:195

目的

libcurl の通信イベント（テキスト情報、ヘッダー、データ転送）をキャプチャし、md_log を介してデバッグログとして出力する。

シグネチャまたは形状

static int curl_debug_log(CURL *curl, curl_infotype type, char *data, size_t size, void *baton)

入力と出力

  • **入力**:
  • curl: libcurl のハンドル（本関数内では未使用）。
  • type: curl_infotype 型のログ種別。
  • data: ログ対象のデータバッファ。
  • size: データのサイズ。
  • baton: md_http_request_t 構造体へのポインタ。
  • **出力**:
  • 常に 0 を返す。

前提条件と不変条件

  • baton は有効な md_http_request_t 構造体を指している必要がある。
  • req->pool はログ出力のためのメモリ割り当てに使用される。

アルゴリズムまたは振る舞い

1. type に基づき、以下のいずれかの処理を行う：

  • CURLINFO_TEXT: テキスト情報を MD_LOG_TRACE4 で出力。
  • CURLINFO_HEADER_OUT / CURLINFO_HEADER_IN: 送受信ヘッダーを MD_LOG_TRACE4 で出力。
  • CURLINFO_DATA_OUT / CURLINFO_DATA_IN: 送受信データサイズを MD_LOG_TRACE4 で出力。

    • 2. データ転送時（DATA_OUT/DATA_IN）において、ログレベルが MD_LOG_TRACE5 に設定されている場合、データを 16 進数形式に変換して追加出力する。 3. apr_pstrndup を使用して、ログ出力に必要な文字列をリクエストのプールから確保する。

呼び出し元と呼び出し先

  • **呼び出し元**: libcurl 内部（CURLOPT_DEBUGFUNCTION に設定されたコールバックとして呼び出される）。
  • **呼び出し先**:
  • md_log_perror (ログ出力)
  • md_log_is_level (ログレベル判定)
  • md_data_to_hex (16進数変換)
  • apr_pstrndup (メモリ確保)

関連データ構造

  • md_http_request_t: リクエストの状態とプールを保持するコンテキスト。
  • curl_infotype: libcurl が定義するログ情報の種類。

失敗モードと境界条件

  • data が NULL または size が 0 の場合、apr_pstrndup の挙動に依存する。
  • ログレベルが低い場合、一部の詳細なログ（16進数ダンプ）はスキップされる。

保守担当者向けメモ

  • 本関数は curl_easy_setopt(curl, CURLOPT_DEBUGFUNCTION, curl_debug_log) を通じて登録される。
  • req->id をログに含めることで、並行実行されるリクエストの識別を容易にしている。
  • curl 引数は未使用であるため、コンパイラの警告を避けるために (void)curl; で明示的に無視している。
