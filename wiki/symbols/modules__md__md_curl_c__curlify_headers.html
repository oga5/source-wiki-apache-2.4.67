curlify_headers

定義

  • 種別: function
  • 定義箇所: modules/md/md_curl.c:166

目的

apr_table_t に格納された HTTP ヘッダーを、libcurl が利用可能な curl_slist 形式のリストへ変換して追加します。

シグネチャまたは形状

static int curlify_headers(void *baton, const char *key, const char *value)

入力と出力

  • **入力**:
  • baton: curlify_hdrs_ctx 構造体へのポインタ（コンテキスト）。
  • key: ヘッダー名。
  • value: ヘッダー値。
  • **出力**:
  • 成功時は 1 を返し、処理を継続します。
  • ヘッダーに不正な文字（\r または \n）が含まれる場合は 0 を返し、処理を中断します。

前提条件と不変条件

  • baton は curlify_hdrs_ctx 型としてキャスト可能である必要があります。
  • ctx->req->pool が有効な apr_pool_t である必要があります。

アルゴリズムまたは振る舞い

1. key または value にキャリッジリターン (\r) または改行 (\n) が含まれているかを確認します。 2. 不正な文字が含まれている場合、ctx->rv に APR_EINVAL を設定し、0 を返して反復処理を終了します。 3. 文字列が正常な場合、apr_psprintf を使用して "key: value" 形式の文字列を生成します。 4. 生成した文字列を curl_slist_append を用いて ctx->hdrs リストに追加します。 5. 1 を返して次のヘッダー処理へ進みます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_curl.c:338 において apr_table_do(curlify_headers, &ctx, req->headers, NULL) として呼び出されます。
  • **呼び出し先**:
  • strchr (標準ライブラリ)
  • apr_psprintf (APR)
  • curl_slist_append (libcurl)

関連データ構造

  • curlify_hdrs_ctx: ヘッダー変換のコンテキストを保持する構造体。
  • curl_slist: libcurl が使用するヘッダーリスト構造体。

失敗モードと境界条件

  • **入力バリデーション**: ヘッダー名または値に改行コードが含まれる場合、APR_EINVAL エラーとして処理が停止します。
  • **メモリ不足**: apr_psprintf や curl_slist_append が失敗した場合の挙動は、基盤ライブラリのメモリ確保ポリシーに依存します。

保守担当者向けメモ

  • 本関数は apr_table_do のコールバックとして設計されています。
  • ctx->hdrs のメモリ管理は libcurl の curl_slist_free_all 等で適切に行われる必要があります。
