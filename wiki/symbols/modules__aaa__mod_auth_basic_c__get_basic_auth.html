get_basic_auth

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_basic.c:248

目的

HTTPリクエストから Authorization（またはプロキシ経由の場合は Proxy-Authorization）ヘッダーを取得し、Basic認証情報をデコードしてユーザー名とパスワードを抽出する。

シグネチャまたは形状

int get_basic_auth(request_rec *r, const char **user, const char **pw)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • user: 抽出したユーザー名を格納するポインタのポインタ。
  • pw: 抽出したパスワードを格納するポインタのポインタ。
  • **出力**:
  • 成功時: OK を返し、user と pw に値を設定。r->user も更新される。
  • 失敗時: HTTP_UNAUTHORIZED を返し、note_basic_auth_failure を呼び出して認証失敗を通知。

前提条件と不変条件

  • すべてのメモリ割り当ては r->pool を使用する。
  • 認証ヘッダーが存在しない、またはスキームが "Basic" でない場合は認証失敗とみなされる。

アルゴリズムまたは振る舞い

1. リクエストヘッダーから認証情報を取得する。 2. スキームが "Basic" であることを検証する（大文字小文字を区別しない）。 3. Base64エンコードされた文字列をデコードする。 4. デコードされた文字列を : で分割し、前半をユーザー名、後半をパスワードとして抽出する。 5. 抽出したユーザー名を r->user に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_basic.c:317 (authenticate_basic_user 関数内)
  • **呼び出し先**:
  • apr_table_get
  • note_basic_auth_failure
  • ap_cstr_casecmp
  • ap_getword
  • ap_log_rerror
  • ap_pbase64decode
  • ap_getword_nulls

関連データ構造

  • request_rec: リクエスト情報を保持するApacheコア構造体。

失敗モードと境界条件

  • **ヘッダー欠損**: Authorization ヘッダーがない場合、HTTP_UNAUTHORIZED を返す。
  • **スキーム不一致**: "Basic" 以外のスキームが指定された場合、エラーログを出力し HTTP_UNAUTHORIZED を返す。
  • **デコード失敗**: Base64デコードが失敗した場合の挙動は ap_pbase64decode の実装に依存する。

保守担当者向けメモ

  • 本関数は認証の「仲介」ロジックの一部であり、実際のパスワード照合は行わない。
  • r->user への代入は、認証が完了する前であっても行われる点に注意すること。
