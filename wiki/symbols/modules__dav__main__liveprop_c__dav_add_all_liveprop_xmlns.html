dav_add_all_liveprop_xmlns

定義

  • 種別: function
  • 定義箇所: modules/dav/main/liveprop.c:66

目的

登録済みのすべてのライブプロパティ名前空間を XML 名前空間宣言（xmlns:lpN="URI"）の形式で出力バッファに追加します。

シグネチャまたは形状

void dav_add_all_liveprop_xmlns(apr_pool_t *p, apr_text_header *phdr)

入力と出力

  • p: 文字列生成およびメモリ確保に使用する APR メモリプール。
  • phdr: 名前空間宣言文字列が追加される apr_text_header 構造体へのポインタ。
  • 出力: 戻り値なし。phdr が指す構造体に、登録済み名前空間の文字列が追記されます。

前提条件と不変条件

  • dav_liveprop_uris ハッシュテーブルが初期化済みであること。
  • phdr が有効な apr_text_header 構造体であること。
  • 各名前空間のインデックス（val）は 1 から始まる連番として保持されていること。

アルゴリズムまたは振る舞い

1. グローバル変数 dav_liveprop_uris を走査するためのハッシュイテレータを初期化します。 2. ハッシュ内の各要素（キー: 名前空間 URI、値: インデックス ID）に対してループ処理を行います。 3. apr_psprintf を使用し、xmlns:lp<ID>="<URI>" という形式の文字列を生成します。 4. 生成した文字列を apr_text_append を用いて phdr に追加します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/main/props.c:678 (dav_add_all_liveprop_xmlns(propdb->p, &hdr_ns);)
  • 呼び出し先:
  • apr_hash_first, apr_hash_next, apr_hash_this (APR ハッシュ操作)
  • apr_psprintf (文字列フォーマット)
  • apr_text_append (テキストヘッダーへの追加)

関連データ構造

  • dav_liveprop_uris: 登録された URI をキー、インデックスを値とする apr_hash_t。
  • apr_text_header: XML 出力用のテキストバッファ構造体。

失敗モードと境界条件

  • dav_liveprop_uris が空の場合、ループは実行されず何も追加されません。
  • メモリ不足時には apr_psprintf や apr_text_append 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は XML レスポンス生成時に lpN プレフィックスを自動生成するために使用されます。
  • インデックス lpN の N は dav_liveprop_count に基づく連番であり、名前空間の登録順序に依存します。
