prefork_child_bucket

定義

  • 種別: struct
  • 定義箇所: server/mpm/prefork/prefork.c:126

目的

Prefork MPM において、子プロセスが接続を受け入れるために必要なリソース（リスナー、終了通知用パイプ、排他制御用ミューテックス）をひとまとめに管理する構造体です。

シグネチャまたは形状

typedef struct prefork_child_bucket {
    ap_pod_t *pod;                /* Pipe of Death (終了通知用パイプ) */
    ap_listen_rec *listeners;     /* リスニングソケットのリスト */
    apr_proc_mutex_t *mutex;      /* accept() 排他制御用ミューテックス */
} prefork_child_bucket;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。各フィールドは APR（Apache Portable Runtime）および Apache HTTP Server のコアモジュールによって提供されるリソースへのポインタです。

前提条件と不変条件

  • pod は子プロセスへの終了シグナル伝達に使用されます。
  • mutex は、複数のプロセスが同時に accept() を実行して競合することを防ぐために使用されます。
  • 構造体内の各ポインタは、プロセス起動時の初期化フェーズで適切に割り当てられている必要があります。

アルゴリズムまたは振る舞い

本構造体自体は状態を保持するコンテナです。子プロセスは、この構造体内の mutex を用いて accept() を保護し、pod を監視することで親プロセスからの終了要求を検知します。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/prefork/prefork.c 内の初期化処理およびプロセス管理ロジック。
  • 呼び出し先: 構造体内の各フィールドは、APR のミューテックス操作関数や、Apache のリスナー管理関数によって参照されます。
  • 補足: 具体的な関数呼び出しの連鎖については、提供されたソース範囲内では詳細な情報が不足しています。

関連データ構造

  • ap_pod_t: 終了通知用パイプ（Pipe of Death）の管理構造体。
  • ap_listen_rec: リスニングソケットの情報を保持する構造体。
  • apr_proc_mutex_t: プロセス間排他制御を行うための APR ミューテックス。

失敗モードと境界条件

  • mutex の初期化やロックに失敗した場合、接続の受け入れ処理が正常に機能せず、デッドロックやリクエストの取りこぼしが発生する可能性があります。
  • pod が正しく設定されていない場合、親プロセスからの終了指示が子プロセスに伝わらず、ゾンビプロセスが発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は all_buckets（prefork.c:131）として静的に定義された配列等で管理されます。
  • メモリ割り当ては sizeof(prefork_child_bucket) を用いて行われており（prefork.c:1259）、構造体のサイズ変更時には注意が必要です。
