SHMCBHeader

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_socache_shmcb.c:58

目的

共有メモリベースのキャッシュ（SHMCB）の先頭に配置されるメタデータおよび統計情報を保持する構造体です。

シグネチャまたは形状

typedef struct {
    unsigned long stat_stores;
    unsigned long stat_replaced;
    unsigned long stat_expiries;
    unsigned long stat_scrolled;
    unsigned long stat_retrieves_hit;
    unsigned long stat_retrieves_miss;
    unsigned long stat_removes_hit;
    unsigned long stat_removes_miss;
    unsigned int subcache_num;
    unsigned int index_num;
    unsigned int subcache_size;
    unsigned int subcache_data_offset;
    unsigned int subcache_data_size;
} SHMCBHeader;

入力と出力

本構造体は関数への引数として渡され、共有メモリ内のレイアウト情報や統計の更新に使用されます。直接的な入力・出力関数はありません。

前提条件と不変条件

  • 共有メモリの先頭に配置される必要があります。
  • subcache_num 個のサブキャッシュが本構造体の直後に配置されるメモリレイアウトを前提としています。
  • subcache_data_offset および subcache_data_size は、各サブキャッシュ内のデータ領域を特定するための最適化値として保持されます。

アルゴリズムまたは振る舞い

  • キャッシュ操作（ストア、取得、削除、期限切れ処理）の統計情報を保持します。
  • サブキャッシュの構造（数、インデックス数、サイズ、データ領域のオフセット）を定義し、メモリ内のポインタ演算の基準となります。

呼び出し元と呼び出し先

  • **呼び出し元**: socache_shmcb_init（初期化時）、shmcb_subcache_store、shmcb_subcache_retrieve、shmcb_subcache_expire、shmcb_subcache_remove 等のサブキャッシュ操作関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • SHMCBSubcache: 本ヘッダーの直後に続くサブキャッシュ管理構造体。
  • SHMCBIndex: サブキャッシュ内で使用されるインデックス構造体。

失敗モードと境界条件

  • unsigned int を使用しているため、非常に大きなキャッシュサイズを指定した場合、計算時にオーバーフローが発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は共有メモリの先頭に固定配置されるため、フィールドの追加や変更を行う際は、共有メモリのレイアウト全体に影響を与える点に注意してください。
  • ALIGNED_HEADER_SIZE マクロ（mod_socache_shmcb.c:51）により、メモリ配置の整合性が保たれています。
