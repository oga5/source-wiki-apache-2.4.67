util_ldap_search_node_copy

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache.c:149

目的

LDAP 検索結果ノード（util_search_node_t）をキャッシュ専用メモリ領域へ複製し、キャッシュ挿入用の新しいノードインスタンスを作成する。

シグネチャまたは形状

void *util_ldap_search_node_copy(util_ald_cache_t *cache, void *c)

入力と出力

  • **入力**:
  • cache: キャッシュ管理構造体へのポインタ。
  • c: 複製対象の util_search_node_t 構造体へのポインタ。
  • **出力**:
  • 成功時: 複製された新しいノードへのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • cache は有効なキャッシュ管理構造体であること。
  • メモリ確保には標準の malloc ではなく、util_ald_alloc および util_ald_strdup を使用すること（共有メモリ管理のため）。
  • 複製に失敗した場合は、確保済みのメモリを解放するために util_ldap_search_node_free が呼び出される。

アルゴリズムまたは振る舞い

1. util_ald_alloc を使用して新しいノード構造体 newnode のメモリを確保する。 2. node->vals（検索値の配列）が存在する場合、その数分だけメモリを確保し、各文字列を util_ald_strdup で複製する。 3. username および dn を util_ald_strdup で複製する。 4. bindpw が存在する場合、同様に util_ald_strdup で複製する。 5. lastbind タイムスタンプをコピーする。 6. いずれかのメモリ確保に失敗した場合は、util_ldap_search_node_free を呼び出して部分的に確保されたメモリを解放し、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap_cache_mgr.c: キャッシュ挿入処理の関数ポインタとして参照されている。
  • **呼び出し先**:
  • util_ald_alloc / util_ald_strdup: キャッシュ用メモリ確保。
  • util_ldap_search_node_free: 失敗時のクリーンアップ。

関連データ構造

  • util_search_node_t: 検索結果を保持する構造体。
  • util_ald_cache_t: キャッシュ管理用構造体。

失敗モードと境界条件

  • メモリ不足: util_ald_alloc または util_ald_strdup が NULL を返した場合、関数は即座に失敗し、それまでに確保したメモリを解放して NULL を返す。
  • vals が空の場合: newnode->vals は NULL として適切に処理される。

保守担当者向けメモ

  • 本関数は共有メモリ（apr_rmm）を操作するため、必ず util_ald_ 系関数を使用すること。標準の free や strdup を使用するとメモリ破壊を引き起こす。
  • vals 配列のコピーにおいて、numvals に基づくループ処理が正しく行われていることを確認すること。
