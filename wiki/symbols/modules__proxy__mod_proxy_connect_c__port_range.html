port_range

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_connect.c:53

目的

AllowCONNECT ディレクティブで指定された、プロキシ接続を許可するポート番号の範囲を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    int first;
    int last;
} port_range;

入力と出力

  • 入力: first（範囲の開始ポート番号）、last（範囲の終了ポート番号）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • first は last 以下である必要があります（論理的な範囲として）。
  • connect_conf 構造体内の allowed_connect_ports 配列に格納される要素として利用されます。

アルゴリズムまたは振る舞い

  • set_allowed_ports 関数において、設定から読み取られたポート範囲を格納するためにインスタンス化されます。
  • proxy_connect_handler 内で、接続先ポートが許可範囲内にあるかを検証する際の比較対象として参照されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/proxy/mod_proxy_connect.c:61: apr_array_make による配列初期化時に型情報として使用。
  • modules/proxy/mod_proxy_connect.c:89: set_allowed_ports 内で一時変数として使用。
  • modules/proxy/mod_proxy_connect.c:120: proxy_connect_handler 内で許可リストの走査時に使用。
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • connect_conf: allowed_connect_ports メンバを通じて本構造体の配列を管理します。

失敗モードと境界条件

  • ポート範囲が正しく設定されていない場合（first > last など）、接続許可判定が意図通りに機能しない可能性があります。

保守担当者向けメモ

  • 本構造体は mod_proxy_connect.c 内でのみ使用されるプライベートな定義です。
  • 許可ポートのリストは apr_array_header_t として管理されているため、走査時には elts メンバを port_range * にキャストしてアクセスします。
