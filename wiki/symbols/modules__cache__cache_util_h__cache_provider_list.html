cache_provider_list

定義

  • 種別: struct
  • 定義箇所: modules/cache/cache_util.h:201

目的

キャッシュストレージプロバイダーのリストを保持するためのデータ構造です。複数のキャッシュプロバイダーを連結リストとして管理し、リクエスト処理中に利用可能なプロバイダーを順次走査するために使用されます。

シグネチャまたは形状

typedef struct cache_provider_list cache_provider_list;

※構造体の内部メンバは公開ヘッダー上では不透明（opaque）として扱われていますが、cache_util.c 等の実装ではリストのノードとして利用されます。

入力と出力

  • 入力: cache_get_providers 等の関数において、リクエストコンテキスト（request_rec）に基づき生成・初期化されます。
  • 出力: キャッシュ検索や保存処理を行う各ストレージプロバイダーへの参照を提供します。

前提条件と不変条件

  • 本構造体は、リクエストプール（r->pool）上で割り当てられることが想定されています。
  • リクエストの終了とともに、関連するメモリは自動的に解放されます。

アルゴリズムまたは振る舞い

  • cache_util.c 内の get_provider 関数により、新しいプロバイダーノードがリストに追加されます。
  • apr_pcalloc を使用してメモリが確保され、リストの末尾への追加や走査が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/cache_storage.c: キャッシュの検索や操作を行う際にリストを参照します。
  • modules/cache/cache_util.c: プロバイダーリストの構築および取得処理（cache_get_providers）で利用されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • cache_request_rec: リクエストごとのキャッシュコンテキストであり、このリストを保持します。
  • cache_enable: キャッシュ設定エントリ。
  • request_rec: Apache のリクエスト構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc が失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性があります。
  • リストが空の場合: プロバイダーが設定されていない場合、リストは NULL となり、キャッシュ処理はスキップされます。

保守担当者向けメモ

  • 本構造体は cache_request_rec と共にプライベートなデータ構造として扱われるべきものです（ap_mmn.h の記述に基づく）。
  • 構造体の内部メンバ定義は cache_util.c 側の実装に依存しているため、メンバを追加・変更する場合は関連する cache_storage.c および cache_util.c の走査ロジックを必ず確認してください。
