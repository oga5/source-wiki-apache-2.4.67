so_sconf_create

定義

  • 種別: function
  • 定義箇所: modules/core/mod_so.c:111

目的

Apache HTTP Server の DSO（動的共有オブジェクト）管理用サーバー設定構造体（so_server_conf）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *so_sconf_create(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定用のメモリプール（apr_pool_t）。
  • s: サーバーレコード（server_rec）。本関数内では未使用。
  • **出力**:
  • 初期化済みの so_server_conf 構造体へのポインタ（void *）。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • DYNAMIC_MODULE_LIMIT が定義されていること。
  • 戻り値の soc->loaded_modules は、DYNAMIC_MODULE_LIMIT 個の ap_module_symbol_t を保持可能な配列として初期化される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、メモリプール p 上に so_server_conf 構造体分のメモリをゼロクリアして確保する。 2. apr_array_make を使用して、ロード済みモジュールを追跡するための配列 loaded_modules を作成する。 3. 作成した so_server_conf 構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/core/mod_so.c:438（サーバー設定の初期化フェーズ）。
  • **呼び出し先**:
  • apr_pcalloc（APRメモリ管理）
  • apr_array_make（APR配列管理）

関連データ構造

  • so_server_conf: 本関数が初期化するメインの設定構造体。
  • ap_module_symbol_t: loaded_modules 配列の各要素の型。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc および apr_array_make は、APRの規約に従い、メモリ確保に失敗した場合はプロセスを終了させる（通常はエラーを返さない）。

保守担当者向けメモ

  • 本関数は mod_so の設定生成ハンドラとして機能する。
  • server_rec *s 引数は現在の実装では使用されていないが、将来的なサーバーごとの設定分離を見越してシグネチャに含まれている。
