ap_dlltoken_e

定義

  • 種別: enum
  • 定義箇所: os/win32/os.h:87

目的

Windows プラットフォームにおいて、実行時に動的ロード（遅延ロード）を行う対象のシステム DLL を識別するための列挙型です。

シグネチャまたは形状

typedef enum {
    AP_DLL_WINBASEAPI = 0,    /* kernel32 */
    AP_DLL_WINADVAPI = 1,     /* advapi32 */
    AP_DLL_WINSOCKAPI = 2,    /* mswsock  */
    AP_DLL_WINSOCK2API = 3,   /* ws2_32   */
    AP_DLL_defined = 4        /* 終端インデックス */
} ap_dlltoken_e;

入力と出力

  • 入力: 該当なし（列挙定数）。
  • 出力: ap_load_dll_func 関数等の引数として渡され、ロード対象の DLL を指定するために使用されます。

前提条件と不変条件

  • AP_DLL_defined は、列挙型の最後のインデックスとして常に定義されている必要があります。
  • 各列挙値は、util_win32.c 内の DLL ロード処理において、対応するシステム DLL を一意に特定するために使用されます。

アルゴリズムまたは振る舞い

本シンボル自体は定数定義ですが、AP_DECLARE_LATE_DLL_FUNC マクロを通じて、特定の Windows API を呼び出す際に「どの DLL から関数をロードするか」を決定する識別子として機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • os/win32/os.h: ap_load_dll_func 関数の宣言引数として使用。
  • os/win32/util_win32.c: ap_load_dll_func 関数の実装引数として使用。
  • 呼び出し先: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • FARPROC: ap_load_dll_func が返す関数ポインタ型。

失敗モードと境界条件

  • AP_DLL_defined 以外の範囲外の値を ap_load_dll_func に渡した場合の挙動は、実装側の ap_load_dll_func に依存します。

保守担当者向けメモ

  • 新しいシステム DLL を動的ロード対象に追加する必要がある場合は、この列挙型に新しいメンバを追加し、AP_DLL_defined を更新してください。
  • util_win32.c 側のロードロジックと同期させる必要があります。
