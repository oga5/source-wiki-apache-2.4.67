socache_mc_create

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_memcache.c:64

目的

Memcached をバックエンドとする共有オブジェクトキャッシュ（socache）のインスタンスを生成し、接続先サーバーリストを初期化します。

シグネチャまたは形状

static const char *socache_mc_create(ap_socache_instance_t **context,
                                     const char *arg,
                                     apr_pool_t *tmp, apr_pool_t *p);

入力と出力

  • **入力**:
  • context: 生成されたインスタンスを格納するポインタのポインタ。
  • arg: Memcached サーバーリストを含む設定文字列。
  • tmp: 一時的な割り当てに使用するプール。
  • p: インスタンスの生存期間を管理するプール。
  • **出力**:
  • 成功時: NULL を返し、*context に初期化済みインスタンスを格納。
  • 失敗時: エラーメッセージ（const char *）を返す。

前提条件と不変条件

  • arg には有効な Memcached サーバーリストの文字列が指定されている必要があります。
  • p はインスタンスのライフサイクルを保持するのに十分な生存期間を持つ必要があります。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して ap_socache_instance_t 構造体をプール p から割り当て、*context に設定します。 2. arg が空または NULL の場合、エラーメッセージを返して終了します。 3. arg が指定されている場合、apr_pstrdup を使用してサーバーリスト文字列をプール p にコピーし、インスタンスの servers メンバに格納します。 4. 正常終了として NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_memcache.c:355（socache_mc プロバイダー構造体の初期化プロセスに関連）。
  • **呼び出し先**: apr_palloc (APR), apr_pstrdup (APR)。

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスのコンテキストを保持する構造体。本関数内で servers メンバが設定されます。

失敗モードと境界条件

  • arg が NULL または空文字列の場合、サーバーリストが不足している旨のエラーメッセージを返します。
  • メモリ割り当て失敗時は apr_palloc の挙動に従い、プロセスが異常終了する可能性があります。

保守担当者向けメモ

  • 本関数はインスタンスの構造体作成とサーバーリストの保持のみを行います。実際の Memcached サーバーへの接続確立は、後続の socache_mc_init で行われます。
