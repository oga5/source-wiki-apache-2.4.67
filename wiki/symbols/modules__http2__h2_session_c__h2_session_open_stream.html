h2_session_open_stream

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_session.c:114

目的

HTTP/2 セッション内での新しいストリームの生成、メモリプールの割り当て、および nghttp2 セッションへのストリーム情報の登録を行います。

シグネチャまたは形状

h2_stream *h2_session_open_stream(h2_session *session, int stream_id, int initiated_on)

入力と出力

  • **入力**:
  • session: 対象となる h2_session 構造体へのポインタ。
  • stream_id: 新規作成するストリームの識別子。
  • initiated_on: ストリームの開始元（親ストリームID等）。
  • **出力**:
  • 成功時: 生成された h2_stream 構造体へのポインタ。
  • 失敗時: NULL（h2_stream_create が失敗した場合）。

前提条件と不変条件

  • session は有効な h2_session オブジェクトである必要があります。
  • ストリーム専用のメモリプール（stream_pool）が session->pool を親として作成されます。

アルゴリズムまたは振る舞い

1. session->pool を親として、ストリーム用のサブプール stream_pool を作成し、タグ付けを行います。 2. h2_stream_create を呼び出し、指定された ID とプールを用いてストリームオブジェクトを初期化します。 3. ストリームの生成に成功した場合、nghttp2_session_set_stream_user_data を使用して、nghttp2 セッション内の該当 stream_id に生成した h2_stream オブジェクトを紐付けます。 4. 生成したストリームオブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c:299
  • modules/http2/h2_session.c:1121
  • modules/http2/h2_session.c:1215
  • **呼び出し先**:
  • apr_pool_create
  • apr_pool_tag
  • h2_stream_create
  • nghttp2_session_set_stream_user_data

関連データ構造

  • h2_session: セッション状態を保持する構造体。
  • h2_stream: ストリームのライフサイクルと状態を管理する構造体。
  • apr_pool_t: メモリ管理に使用される Apache Portable Runtime プール。
  • nghttp2_session: nghttp2 ライブラリ側のセッション管理構造体。

失敗モードと境界条件

  • apr_pool_create が失敗した場合、プロセスは異常終了する可能性があります（APR の標準的な挙動）。
  • h2_stream_create が NULL を返した場合、nghttp2 への登録はスキップされ、関数は NULL を返します。

保守担当者向けメモ

  • 本関数はストリームのメモリ所有権を session->pool のサブプールに委譲しています。セッション終了時にストリームのリソースも一括解放される設計です。
  • nghttp2 側のストリームデータと h2_stream オブジェクトの同期を維持するため、ストリーム作成時には必ず nghttp2_session_set_stream_user_data を呼び出す必要があります。
