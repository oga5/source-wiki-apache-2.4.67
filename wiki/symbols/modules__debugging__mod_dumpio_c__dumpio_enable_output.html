dumpio_enable_output

定義

  • 種別: function
  • 定義箇所: modules/debugging/mod_dumpio.c:230

目的

Apache HTTP Server の設定ディレクティブ DumpIOOutput に対応し、サーバー設定構造体内の出力ダンプ有効フラグを更新する。

シグネチャまたは形状

static const char *dumpio_enable_output(cmd_parms *cmd, void *dummy, int arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていない引数。
  • arg: ディレクティブで指定された有効/無効を示す整数値。
  • **出力**:
  • 常に NULL を返す（設定成功を示す）。

前提条件と不変条件

  • cmd->server->module_config に dumpio_module 用の設定構造体が割り当てられている必要がある。
  • dumpio_conf_t 型の構造体が ap_get_module_config によって取得可能であること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を呼び出し、現在のサーバー設定から dumpio_conf_t 構造体を取得する。 2. 取得した構造体の enable_output メンバに、引数 arg を代入する。 3. NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/debugging/mod_dumpio.c 内の dumpio_cmds 配列において、AP_INIT_FLAG("DumpIOOutput", ...) マクロ経由で呼び出される。
  • **呼び出し先**: ap_get_module_config（Apache コア関数）。

関連データ構造

  • dumpio_conf_t: mod_dumpio の設定を保持する構造体。enable_output メンバが本関数によって操作される。

失敗モードと境界条件

  • ap_get_module_config が無効なポインタを返した場合、その後のメンバアクセスでセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は AP_INIT_FLAG を介して呼び出される設定ハンドラである。
  • DumpIOOutput ディレクティブの挙動を変更する場合は、本関数および dumpio_conf_t 構造体の定義を確認すること。
