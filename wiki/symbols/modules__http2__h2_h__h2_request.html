h2_request

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2.h:168

目的

HTTP/2 ストリームを Apache HTTP Server 内部の HTTP/1.1 形式へ変換するための中間構造体です。request_rec が生成される前のリクエストメタデータを保持します。

シグネチャまたは形状

typedef struct h2_request h2_request;

※構造体の詳細なメンバー定義は modules/http2/h2.h の169行目以降に記述されています。

入力と出力

  • **入力**: HTTP/2 フレームから抽出されたヘッダー情報およびリクエストメタデータ。
  • **出力**: mod_http2 のフィルタチェーンを通じて、Apache 内部の request_rec 構造体へ変換されます。

前提条件と不変条件

  • h2_request 構造体に含まれる apr_table_t 等のデータは、通常 request_pool に紐付けられて管理されます。

アルゴリズムまたは振る舞い

本構造体は、HTTP/2 プロトコルスタックと Apache のリクエスト処理パイプラインを仲介する役割を担います。HTTP/2 ストリームとして受信したリクエストを、Apache が処理可能な形式に正規化・保持するデータコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: h2_c2.c や h2_c2_filter.c 等の接続処理コンポーネントが、処理対象のリクエストとして本構造体を参照しています。
  • **呼び出し先**: h2_headers.c 等のヘッダー処理関数が、本構造体を引数として受け取り、ヘッダーの解析や変換を行います。

関連データ構造

  • request_rec: 本構造体から変換される Apache の標準的なリクエスト表現。
  • h2_conn_ctx: 接続ごとのコンテキスト内で、処理すべきリクエストとして h2_request を保持します。

失敗モードと境界条件

  • 構造体内のメモリ管理は request_pool に依存しているため、プールが破棄された後のアクセスは不正となります。

保守担当者向けメモ

  • 本構造体は h2_request.h に定義が分割されている場合があるため、実装の詳細については同ヘッダーファイルも併せて参照してください。
  • h2_c2_filter.h のコメントにある通り、理想的にはヘッダーの取り扱いをより効率化する余地があります。
