ajp_parse_header

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_header.c:781

目的

AJP プロトコルにおけるレスポンスパケットの型を検証し、SEND_HEADERS パケットであればその内容を解析して Apache の request_rec 構造体に反映させる。

シグネチャまたは形状

apr_status_t ajp_parse_header(request_rec *r, proxy_dir_conf *conf, ajp_msg_t *msg)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体。
  • conf: プロキシのディレクトリ設定。
  • msg: 受信した AJP メッセージバッファ。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS、パケット型が不一致の場合は AJP_EBAD_HEADER、メッセージ読み込み失敗時はエラーコードを返す。

前提条件と不変条件

  • msg は有効な ajp_msg_t 構造体であり、ネットワークから読み込まれたパケットデータを含んでいること。
  • パケットの先頭バイトが AJP メッセージの型を示す識別子であること。

アルゴリズムまたは振る舞い

1. ajp_msg_get_uint8 を呼び出し、メッセージの型識別子を取得する。 2. 取得に失敗した場合はエラーログを出力し、そのエラーコードを返す。 3. 取得した型が CMD_AJP13_SEND_HEADERS (0x03) と一致するか検証する。 4. 型が一致しない場合はエラーログを出力し、AJP_EBAD_HEADER を返す。 5. 型が一致する場合、ajp_unmarshal_response を呼び出し、ヘッダー情報の解析と request_rec への反映を行い、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ajp.c:490 (直接的な証拠あり)
  • **呼び出し先**:
  • ajp_msg_get_uint8 (メッセージ型取得)
  • ajp_type_str (ログ出力用文字列変換)
  • ap_log_rerror (エラーログ出力)
  • ajp_unmarshal_response (ヘッダー解析)

関連データ構造

  • ajp_msg_t: AJP 通信用のメッセージバッファ。
  • request_rec: Apache のリクエスト情報構造体。
  • proxy_dir_conf: プロキシのディレクトリ設定構造体。

失敗モードと境界条件

  • **メッセージ読み込み失敗**: ajp_msg_get_uint8 が APR_SUCCESS 以外を返した場合、そのエラーを伝播する。
  • **パケット型不一致**: 期待される CMD_AJP13_SEND_HEADERS 以外の型が受信された場合、AJP_EBAD_HEADER を返し処理を中断する。

保守担当者向けメモ

  • 本関数は mod_proxy_ajp のレスポンス処理フローにおけるゲートキーパーとして機能する。
  • 新しい AJP パケット型をサポートする場合、型チェックのロジックを拡張する必要がある。
  • ajp_type_str はデバッグログ出力にのみ使用されており、パケットの型判定には CMD_AJP13_SEND_HEADERS 定数が直接使用されている。
