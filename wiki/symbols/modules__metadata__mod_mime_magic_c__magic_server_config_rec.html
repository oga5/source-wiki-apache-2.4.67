magic_server_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_mime_magic.c:455

目的

Apache HTTP Server の mod_mime_magic モジュールにおいて、サーバー単位の設定情報を保持するために使用される構造体です。マジックナンバー定義ファイルのパスと、解析ルールを格納する連結リストの先頭および末尾ポインタを管理します。

シグネチャまたは形状

typedef struct {
    const char *magicfile;    /* マジックファイルへのパス */
    struct magic *magic;      /* マジック設定リストの先頭 */
    struct magic *last;       /* マジック設定リストの末尾 */
} magic_server_config_rec;

入力と出力

  • **入力**: MimeMagicFile ディレクティブ等により設定された値が、create_magic_server_config 等を通じて本構造体に格納されます。
  • **出力**: モジュール内の各処理関数（magic_process 等）が、本構造体を参照してマジックファイルパスやルールリストを取得します。

前提条件と不変条件

  • magic および last メンバは、apprentice() 関数によって構築された struct magic 型の連結リストを指します。
  • メモリ割り当ては Apache の apr_pool_t を介して行われることが前提です。

アルゴリズムまたは振る舞い

  • 本構造体は、サーバー起動時または設定読み込み時に初期化され、mod_mime_magic が MIME タイプを判定する際の参照元となります。
  • magicfile に基づき apprentice() がマジックファイルをパースし、その結果を magic リストとして保持します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • create_magic_server_config (メモリ確保)
  • merge_magic_server_config (設定マージ)
  • magic_init (初期化)
  • magic_find_ct (リクエスト処理時の参照)
  • その他、mod_mime_magic.c 内の各設定取得関数。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • struct magic: マジックナンバーのルールを保持する単方向連結リストのノード。
  • magic_req_rec: リクエスト単位の解析結果を保持する構造体。

失敗モードと境界条件

  • 設定ファイルが存在しない、またはパースに失敗した場合、magic リストが空（NULL）のままとなる可能性があります。
  • メモリ不足時には apr_pcalloc 等の呼び出しが失敗し、サーバーの起動やリクエスト処理が中断される可能性があります。

保守担当者向けメモ

  • 本構造体はスレッドセーフな環境での利用を想定して設計されていますが、元の file コマンド由来のコードベースであるため、グローバルな状態を持たないよう注意して管理してください。
  • magic リストの構築は apprentice() に依存しているため、ルール追加や変更時は同関数のロジックを確認してください。
