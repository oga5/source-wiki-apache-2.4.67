dav_lock_discovery

定義

  • 種別: struct
  • 定義箇所: modules/dav/lock/locks.c:118

目的

WebDAV リソースに対する「直接ロック（Direct Lock）」の情報をメモリ上で保持するためのリスト構造体です。

シグネチャまたは形状

typedef struct dav_lock_discovery
{
    struct dav_lock_discovery_fixed f;
    dav_locktoken *locktoken;
    const char *owner;
    const char *auth_user;
    struct dav_lock_discovery *next;
} dav_lock_discovery;

入力と出力

  • **入力**: dav_generic_load_lock_record 関数により、DBM から読み込まれたバイナリデータが本構造体にデシリアライズされます。
  • **出力**: dav_generic_save_lock_record 関数により、本構造体のリストがバイナリ形式にシリアライズされ、DBM へ永続化されます。

前提条件と不変条件

  • すべてのインスタンスは、関連する request_rec のメモリプールから割り当てられます。
  • next ポインタにより、単方向リストを形成します。
  • 構造体内の各フィールドは、DBM への書き込み時に dav_size_direct 等の計算ロジックと整合している必要があります。

アルゴリズムまたは振る舞い

  • **読み込み時**: dav_generic_load_lock_record が DBM からレコードを読み込む際、現在時刻と比較して期限切れのロックを排除し、有効なロックのみを本構造体のリストとして構築します。
  • **書き込み時**: dav_generic_save_lock_record がリストを走査し、各要素をバイナリバッファにシリアライズして DBM に保存します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/lock.c 内の複数の関数（dav_fs_get_locks 等）から、ロック情報の保持や操作のために参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • dav_lock_discovery_fixed: 本構造体の先頭に埋め込まれる固定長データ構造。
  • dav_locktoken: ロックを一意に識別するための UUID ラッパー。
  • dav_lock_indirect: 間接ロック（継承ロック）を管理する構造体。

失敗モードと境界条件

  • **メモリ不足**: リクエストプールからの割り当て失敗時に処理が中断されます。
  • **シリアライズ不整合**: locks.c 内のオフセット計算と構造体の定義が一致しない場合、DBM からの読み込み時にデータ破損が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は modules/dav/fs/lock.c と密接に関連しています。変更を行う際は、locks.c 内のシリアライズロジックと fs/lock.c 側の参照ロジックの両方に影響がないか確認してください。
  • 将来的なデータ構造の拡張時には、バイナリ形式の後方互換性に注意が必要です。
