ap_condition_if_range

定義

  • 種別: function
  • 定義箇所: modules/http/http_protocol.c:488

目的

HTTP リクエストに含まれる If-Range ヘッダーを評価し、リクエストされた範囲（Range）が現在のリソースと一致するかを判定する。

シグネチャまたは形状

ap_condition_e ap_condition_if_range(request_rec *r, apr_table_t *headers)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。r->headers_in から If-Range および Range ヘッダーを取得する。
  • headers: 比較対象となるレスポンスヘッダー（ETag, Last-Modified, Date を含む）。
  • **出力**:
  • AP_CONDITION_STRONG: 条件が一致した（範囲リクエストを継続可能）。
  • AP_CONDITION_NOMATCH: 条件が一致しなかった。
  • AP_CONDITION_NONE: If-Range ヘッダーが存在しない、または Range ヘッダーがないため評価対象外。

前提条件と不変条件

  • If-Range と Range の両方がリクエストに含まれている場合にのみ評価が行われる。
  • HTTP 仕様に基づき、比較は秒単位の解像度で行われる。

アルゴリズムまたは振る舞い

1. If-Range ヘッダーが " で始まる場合（ETag 形式）:

  • headers 内の ETag と比較し、一致すれば AP_CONDITION_STRONG を返す。不一致なら AP_CONDITION_NOMATCH を返す。

    • 2. If-Range ヘッダーが日付形式の場合:

  • headers から Last-Modified を取得し、なければ r->mtime を使用する。
  • headers から Date を取得し、なければ r->request_time を使用する。
  • If-Range の日付と Last-Modified を比較する。
  • 一致する場合、Date が Last-Modified から 60 秒以内であれば AP_CONDITION_NOMATCH（弱い一致は不可）、それ以外は AP_CONDITION_STRONG を返す。
  • 日付が一致しない場合は AP_CONDITION_NOMATCH を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/cache_storage.c
  • modules/http/byterange_filter.c
  • modules/http/http_protocol.c
  • **呼び出し先**:
  • apr_table_get (APR)
  • apr_date_parse_http (APR)
  • apr_time_sec (APR)
  • strcmp (標準ライブラリ)

関連データ構造

  • request_rec: リクエスト情報および入力ヘッダーの保持。
  • apr_table_t: ヘッダー情報の格納に使用。
  • ap_condition_e: 戻り値の列挙型。

失敗モードと境界条件

  • If-Range または Range ヘッダーが欠落している場合は AP_CONDITION_NONE を返す。
  • Last-Modified が解析不能な場合、r->mtime または現在時刻で代替する。
  • Date が解析不能な場合、r->request_time で代替する。

保守担当者向けメモ

  • 本関数は Range リクエストの妥当性を検証する重要なロジックである。
  • 比較ロジックは RFC に準拠しているが、時刻の解像度や弱い一致の扱いに依存しているため、仕様変更時には注意が必要である。
