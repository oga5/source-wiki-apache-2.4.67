file_cache_handler

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_file_cache.c:309

目的

キャッシュされた静的ファイルのリソース（ファイルハンドルまたはメモリマップ）を使用して、HTTP レスポンスを生成・送信する。

シグネチャまたは形状

int file_cache_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）。
  • **出力**: int（Apache HTTP Server のステータスコード。OK, DECLINED, またはエラーコード）。

前提条件と不変条件

  • file_cache_xlat フックが事前に実行され、r->request_config に a_file 構造体が格納されている必要がある。
  • GET メソッドのみを処理対象とする。
  • r->handler がデフォルト値または */* に一致する必要がある。

アルゴリズムまたは振る舞い

1. **検証**: リクエストメソッドが GET でない場合、またはハンドラが不適切な場合は DECLINED を返す。 2. **コンテキスト取得**: ap_get_module_config を使用して、file_cache_module に関連付けられた a_file 構造体を取得。見つからない場合は DECLINED を返す。 3. **リクエスト処理**:

  • リクエストボディを破棄（ap_discard_request_body）。
  • Last-Modified ヘッダーを事前計算値（match->mtimestr）で最適化して設定。
  • Content-Length ヘッダーを事前計算値（match->sizestr）で設定。
  • ap_meets_conditions で条件付きリクエスト（If-Modified-Since 等）を評価。

    • 4. **レスポンス送信**: r->header_only が偽の場合、is_mmapped フラグに応じて mmap_handler または sendfile_handler を呼び出し、データを送信する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のハンドラフック機構（ap_hook_handler により登録）。
  • **呼び出し先**:
  • ap_discard_request_body, ap_update_mtime, ap_rationalize_mtime, apr_rfc822_date, apr_table_setn, ap_set_etag, ap_meets_conditions
  • mmap_handler (条件付き)
  • sendfile_handler (条件付き)

関連データ構造

  • a_file: キャッシュされたファイルのメタデータ（finfo, mtimestr, sizestr, is_mmapped）を保持。
  • request_rec: Apache のリクエスト状態を保持。

失敗モードと境界条件

  • match が NULL の場合: キャッシュ対象外として DECLINED を返す。
  • GET 以外のメソッド: DECLINED を返す。
  • ap_discard_request_body が失敗した場合: そのエラーコードをそのまま返す。
  • ap_meets_conditions が OK 以外を返した場合: そのステータスを返す。

保守担当者向けメモ

  • 本関数は Last-Modified や Content-Length の計算をスキップして事前計算値を使用することで性能を最適化している。
  • mmap_handler と sendfile_handler の呼び出しは、match->is_mmapped の値に依存する。
  • リクエストボディの読み込みと破棄は、将来的な拡張を見越して実装されている。
