push_item

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_autoindex.c:198

目的

autoindex_config_rec 内のリスト（アイコン、代替テキスト、無視リストなど）に対して、新しい設定項目（struct item）を追加および初期化します。

シグネチャまたは形状

static void push_item(apr_array_header_t *arr, char *type, const char *to,
                      const char *path, const char *data)

入力と出力

  • **入力**:
  • arr: 追加先の apr_array_header_t 配列。
  • type: 項目の種別（BY_PATH 等）。
  • to: 適用対象のパターン。
  • path: 適用先のディレクトリパス。
  • data: 項目に関連付けられたデータ（アイコン名や説明文など）。
  • **出力**: なし（arr を直接更新）。

前提条件と不変条件

  • arr は apr_array_header_t として有効であること。
  • arr->pool を使用してメモリが割り当てられるため、呼び出し元は適切なプールを保持していること。
  • to または path が NULL の場合、空文字列として扱われます。

アルゴリズムまたは振る舞い

1. apr_array_push を使用して、配列 arr に新しい struct item 領域を確保します。 2. to および path が NULL の場合は空文字列に置換します。 3. data を arr->pool に複製して p->data に格納します。 4. path に * を付加して p->apply_path に格納します。 5. type が BY_PATH であり、かつ to がワイルドカード式（ap_is_matchexp）でない場合、to の先頭に * を付加して p->apply_to に格納します。それ以外の場合は to をそのまま複製して格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_autoindex.c:236 (add_alt 経由)
  • modules/generators/mod_autoindex.c:268 (add_icon 経由)
  • modules/generators/mod_autoindex.c:321 (add_ignore 経由)
  • **呼び出し先**:
  • apr_array_push
  • apr_pstrdup
  • apr_pstrcat
  • ap_is_matchexp

関連データ構造

  • struct item: 本関数で初期化されるデータ構造。
  • apr_array_header_t: 項目を保持する動的配列。

失敗モードと境界条件

  • メモリ不足時には apr_pstrdup や apr_pstrcat が例外（Apache のメモリ管理ポリシーに従う）を発生させる可能性があります。
  • to が NULL の場合、空文字列として処理されます。

保守担当者向けメモ

  • 本関数は mod_autoindex の設定ディレクティブ（AddIcon, AddAlt, IndexIgnore など）の解析時に共通して使用されます。
  • p->apply_to の生成ロジックにおいて、BY_PATH の場合にのみワイルドカードの自動付与が行われる点に注意してください。
