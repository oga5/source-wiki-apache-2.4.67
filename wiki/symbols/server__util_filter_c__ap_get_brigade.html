ap_get_brigade

定義

  • 種別: function
  • 定義箇所: server/util_filter.c:546

目的

入力フィルタチェーンからデータを読み込み、バケットブリゲード（apr_bucket_brigade）に格納する。

シグネチャまたは形状

apr_status_t ap_get_brigade(ap_filter_t *next,
                            apr_bucket_brigade *bb,
                            ap_input_mode_t mode,
                            apr_read_type_e block,
                            apr_off_t readbytes)

入力と出力

  • **入力**:
  • next: 読み込みを開始するフィルタノード。
  • bb: 読み込んだデータを格納するバケットブリゲード。
  • mode: 読み込みモード（ap_input_mode_t）。
  • block: ブロック読み込みか非ブロック読み込みか（apr_read_type_e）。
  • readbytes: 読み込むバイト数。
  • **出力**:
  • apr_status_t: 処理結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • bb は呼び出し前に空である必要がある（include/util_filter.h による規定）。
  • next が NULL の場合、データ読み込みは行われない。

アルゴリズムまたは振る舞い

1. next が存在する場合、そのフィルタの in_func（入力フィルタ関数）を呼び出し、引数をそのまま渡す。 2. next が NULL の場合、AP_NOBODY_READ を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_heartmonitor.c, mod_dav.c, mod_dumpio.c, mod_echo.c, mod_case_filter_in.c, mod_buffer.c, mod_charset_lite.c 等の各モジュール。
  • **呼び出し先**:
  • next->frec->filter_func.in_func: フィルタチェーン内の次のフィルタ関数。

関連データ構造

  • ap_filter_t: フィルタチェーンのノード。
  • apr_bucket_brigade: データのバケットを保持するコンテナ。
  • ap_filter_rec_t: フィルタの定義情報。

失敗モードと境界条件

  • next が NULL の場合: AP_NOBODY_READ を返す。
  • フィルタ関数がエラーを返した場合: そのステータスコードをそのまま呼び出し元に返す。

保守担当者向けメモ

  • 本関数はフィルタチェーンのトラバースを抽象化するラッパーである。
  • フィルタチェーンの順序や in_func の実装詳細については、各フィルタモジュールのロジックに依存する。
