merge_info_config

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_info.c:96

目的

サーバー設定構造体 info_svr_conf のマージを行います。親設定（base）と子設定（overrides）の情報を統合し、新しい設定インスタンスを生成します。

シグネチャまたは形状

static void *merge_info_config(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親設定（info_svr_conf 型へのポインタ）。
  • overridesv: 子設定（info_svr_conf 型へのポインタ）。
  • **出力**:
  • 統合された info_svr_conf 構造体へのポインタを void * として返します。

前提条件と不変条件

  • basev および overridesv は info_svr_conf 型としてキャスト可能である必要があります。
  • p は有効なメモリプールである必要があります。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい info_svr_conf 構造体をメモリプール p 上に確保します。 2. overrides（子）の more_info 配列と base（親）の more_info 配列を、apr_array_append を用いて結合します。 3. 結合された新しい配列を new->more_info に格納し、新しい構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_info.c:1015（サーバー設定のマージ処理において呼び出されます）。
  • **呼び出し先**: apr_pcalloc（APRライブラリ）、apr_array_append（APRライブラリ）。

関連データ構造

  • info_svr_conf: モジュール固有のサーバー設定構造体。more_info 配列を保持します。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、サーバーが異常終了する可能性があります。
  • basev や overridesv が NULL の場合、構造体へのキャストおよびメンバアクセス時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は Apache の設定マージライフサイクルの一部です。more_info 配列の順序は apr_array_append の仕様（overrides が先、base が後）に従います。
