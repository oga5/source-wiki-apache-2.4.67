proxy_v2_addr

定義

  • 種別: union
  • 定義箇所: modules/metadata/mod_remoteip.c:87

目的

PROXY プロトコル v2 におけるアドレス情報（送信元/宛先 IP アドレスおよびポート番号）を格納するための共用体です。

シグネチャまたは形状

typedef union {
    struct { apr_uint32_t src_addr; apr_uint32_t dst_addr; apr_uint16_t src_port; apr_uint16_t dst_port; } ip4;
    struct { apr_byte_t src_addr[16]; apr_byte_t dst_addr[16]; apr_uint16_t src_port; apr_uint16_t dst_port; } ip6;
    struct { apr_byte_t src_addr[108]; apr_byte_t dst_addr[108]; } unx;
} proxy_v2_addr;

入力と出力

  • **入力**: PROXY プロトコル v2 のバイナリデータストリームから抽出されたアドレス情報。
  • **出力**: 構造体内の各フィールド（ip4, ip6, unx）を通じて、接続の送信元および宛先情報を提供。

前提条件と不変条件

  • 使用する構造体メンバは、PROXY プロトコル v2 のヘッダーで指定されたアドレスファミリー（IPv4, IPv6, AF_UNIX）と一致している必要があります。
  • 各メンバのサイズはプロトコル仕様に基づき固定されています（ip4: 12バイト, ip6: 36バイト, unx: 216バイト）。

アルゴリズムまたは振る舞い

  • 接続のプロトコルタイプに応じて、共用体の適切なメンバを選択してアクセスします。
  • ip4 および ip6 メンバは、送信元と宛先の IP アドレスおよびポート番号を保持します。
  • unx メンバは、UNIX ドメインソケットのパス情報を保持します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/metadata/mod_remoteip.c 内の PROXY プロトコル解析ロジック（proxy_v2_addr addr; として宣言）。
  • 呼び出し先: 情報はなし。

関連データ構造

  • proxy_header: PROXY プロトコルヘッダー全体を定義する共用体。
  • proxy_v2: PROXY プロトコル v2 のヘッダー構造体。

失敗モードと境界条件

  • 不適切なアドレスファミリーを指定してメンバにアクセスした場合、メモリの誤解釈が発生します。
  • プロトコルヘッダーの長さと実際のデータ長が一致しない場合、不正なアドレス情報が格納される可能性があります。

保守担当者向けメモ

  • 本構造体は PROXY プロトコル v2 のバイナリレイアウトに直接マッピングされています。仕様変更や新しいアドレスファミリーのサポートを追加する際は、各構造体のメンバサイズとアライメントに注意してください。
