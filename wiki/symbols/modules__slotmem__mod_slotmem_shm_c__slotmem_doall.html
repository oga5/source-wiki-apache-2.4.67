slotmem_doall

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_shm.c:312

目的

共有メモリインスタンス内の全スロットを走査し、各スロットに対して指定されたコールバック関数を実行する。

シグネチャまたは形状

static apr_status_t slotmem_doall(ap_slotmem_instance_t *mem,
                                  ap_slotmem_callback_fn_t *func,
                                  void *data, apr_pool_t *pool)

入力と出力

  • **入力**:
  • mem: 操作対象の ap_slotmem_instance_t 構造体へのポインタ。
  • func: 各スロットに対して実行する ap_slotmem_callback_fn_t 型のコールバック関数。
  • data: コールバック関数に渡す任意のユーザーデータ。
  • pool: コールバック処理で使用するメモリプール。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS。コールバック関数がエラーを返した場合はそのステータスコードを返し、走査を中断する。

前提条件と不変条件

  • mem が NULL の場合、APR_ENOSHMAVAIL を返す。
  • スロットの走査は mem->desc->num で定義された個数分行われる。
  • ptr（データポインタ）は mem->base から始まり、各反復で mem->desc->size ずつ加算される。

アルゴリズムまたは振る舞い

1. mem が有効か確認し、無効ならエラーを返す。 2. スロットのデータ開始位置 ptr と利用状況フラグ inuse を初期化する。 3. 全スロットに対してループ処理を行う:

  • AP_SLOTMEM_IS_PREGRAB(mem) が偽であるか、または *inuse が真（スロットが使用中）である場合に限り、コールバック func を呼び出す。
  • コールバックの戻り値が APR_SUCCESS 以外であれば、即座にループを終了する。
  • 次のスロットへ移動するため、ptr を size 分進め、inuse ポインタをインクリメントする。

    • 4. 最終的なステータスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/slotmem/mod_slotmem_shm.c:734 にて storage 構造体の一部として参照されている。
  • **呼び出し先**: 引数として渡された func（コールバック関数）。

関連データ構造

  • ap_slotmem_instance_t: スロットメモリの管理構造体。ベースアドレスや利用状況フラグ配列を保持。
  • sharedslotdesc_t: スロットのサイズや個数などの構成情報を保持。

失敗モードと境界条件

  • mem が NULL の場合: APR_ENOSHMAVAIL を返す。
  • コールバック関数がエラーを返した場合: 走査を中断し、そのエラーコードを呼び出し元に伝播させる。

保守担当者向けメモ

  • 本関数は mod_slotmem_shm のプロバイダーインターフェースの一部であり、外部モジュールから ap_slotmem_provider_t を介して利用されることを想定している。
  • AP_SLOTMEM_IS_PREGRAB マクロの定義により、走査対象のフィルタリング条件が変化するため、変更時は注意が必要。
