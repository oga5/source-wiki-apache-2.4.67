ssl_stapling_init_cert

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util_stapling.c:131

目的

サーバー証明書に関連付けられた OCSP ステープリング情報を初期化し、キャッシュ管理用の certinfo 構造体を生成・登録します。

シグネチャまたは形状

int ssl_stapling_init_cert(server_rec *s, apr_pool_t *p, apr_pool_t *ptemp,
                           modssl_ctx_t *mctx, X509 *x)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p: 長期生存するメモリプール。
  • ptemp: 一時的なメモリプール。
  • mctx: SSL コンテキスト。
  • x: 初期化対象の証明書（X509）。
  • **出力**:
  • 成功時は 1、失敗時は 0 を返します。

前提条件と不変条件

  • x が NULL の場合は直ちに 0 を返します。
  • 証明書の発行者（Issuer）証明書が取得可能である必要があります。
  • OCSP URI が証明書内に存在するか、SSLStaplingForceURL が設定されている必要があります。

アルゴリズムまたは振る舞い

1. stapling_get_issuer を呼び出し、証明書の発行者を特定します。 2. 証明書の SHA-1 ダイジェストを計算し、一意な識別子（idx）を生成します。 3. ap_ssl_ocsp_prime または ssl_run_init_stapling_status フックにより、他モジュールによる処理の有無を確認します。処理された場合、必要に応じて stapling_cb を登録して終了します。 4. 既にキャッシュ済みの certinfo が存在するか確認し、存在する場合は設定の整合性を検証します。 5. OCSP_cert_to_id で OCSP_CERTID を生成し、証明書から OCSP URI を抽出します。 6. certinfo 構造体を割り当て、stapling_certinfo ハッシュテーブルに格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c（サーバー初期化フェーズ）。
  • **呼び出し先**:
  • stapling_get_issuer, modssl_cert_get_pem, ap_ssl_ocsp_prime, ssl_run_init_stapling_status, OCSP_cert_to_id, X509_get1_ocsp, ssl_stapling_certid_free 等。

関連データ構造

  • certinfo: OCSP ステープリング情報を保持する内部構造体。
  • stapling_certinfo: 証明書ハッシュをキーとするグローバルなハッシュテーブル。
  • modssl_ctx_t: SSL 設定コンテキスト。

失敗モードと境界条件

  • 発行者証明書が取得できない場合、stapling_enabled が有効であればエラーログを出力し 0 を返します。
  • OCSP URI が取得できず、かつ強制 URL 設定もない場合、エラーログを出力し 0 を返します。
  • CertID の生成に失敗した場合、エラーログを出力し 0 を返します。

保守担当者向けメモ

  • 本関数は mod_ssl の初期化プロセスの一部です。
  • certinfo のメモリ管理には apr_pool_t を使用しており、cid（CertID）の解放はプールクリーンアップ時に ssl_stapling_certid_free が実行されるよう登録されています。
  • stapling_certinfo ハッシュテーブルへのアクセスは、マルチスレッド環境下での競合に注意してください。
