env_dir_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/metadata/mod_env.c:30

目的

Apache HTTP Server の mod_env モジュールにおいて、ディレクトリ単位で設定された環境変数の状態を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_table_t *vars;
    apr_table_t *unsetenv;
} env_dir_config_rec;

入力と出力

  • vars: 設定された環境変数のキーと値のペアを保持する apr_table_t。
  • unsetenv: 削除対象として指定された環境変数のキーを保持する apr_table_t。

前提条件と不変条件

  • 各フィールドは Apache のメモリプール（apr_pool_t）によって管理されます。
  • merge_env_dir_configs 処理において、unsetenv に含まれる変数は、vars に含まれる変数よりも優先的に処理（削除）される不変条件を持ちます。

アルゴリズムまたは振る舞い

  • SetEnv ディレクティブによって指定された変数は vars に格納されます。
  • UnsetEnv ディレクティブによって指定された変数は unsetenv に格納されます。
  • リクエスト処理の fixups フェーズにおいて、この構造体の内容が request_rec->subprocess_env にマージされます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • create_env_dir_config (初期化)
  • merge_env_dir_configs (設定マージ)
  • add_env_module_vars_passed, add_env_module_vars_set, add_env_module_vars_unset (ディレクティブ処理)
  • fixup_env_module (リクエスト時の適用)
  • 呼び出し先:
  • apr_palloc (メモリ確保)
  • ap_get_module_config (設定取得)

関連データ構造

  • apr_table_t: 環境変数のキー・値ペアを格納する APR のテーブル構造体。
  • request_rec: リクエスト情報を保持する構造体（subprocess_env フィールドを通じて本構造体と連携）。

失敗モードと境界条件

  • SetEnv で変数名に = が含まれる場合、警告がログ出力されますが、構造体への格納は継続されます。
  • PassEnv で指定した環境変数が OS 上に存在しない場合、警告ログが出力されます。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を保持するため、階層構造を持つ設定のマージロジック（merge_env_dir_configs）の正確性が重要です。
  • UnsetEnv を先に処理してから SetEnv を適用する順序は、設定の競合を解決するための仕様です。
