slotmem_grab

定義

  • 種別: function
  • 定義箇所: modules/slotmem/mod_slotmem_plain.c:243

目的

指定されたスロットメモリインスタンスにおいて、利用可能な最初の空きスロットを検索し、そのスロットを「使用中」として確保する。

シグネチャまたは形状

apr_status_t slotmem_grab(ap_slotmem_instance_t *slot, unsigned int *id)

入力と出力

  • **入力**:
  • slot: 操作対象の ap_slotmem_instance_t 構造体へのポインタ。
  • id: 確保されたスロットのインデックス番号を格納するための出力用ポインタ。
  • **出力**:
  • 成功時: APR_SUCCESS を返し、id に確保したインデックスを格納。
  • 失敗時: APR_ENOSHMAVAIL（slot が無効な場合）または APR_EINVAL（空きスロットがない場合）を返す。

前提条件と不変条件

  • slot 引数が NULL でないこと。
  • 本関数は内部的なロック機構を持たないため、マルチスレッド環境下での呼び出しには、呼び出し側での排他制御が必要である。

アルゴリズムまたは振る舞い

1. slot が NULL の場合、APR_ENOSHMAVAIL を返して終了する。 2. インスタンス内の inuse フラグ配列を先頭から順に走査する。 3. inuse フラグが 0（未使用）である最初のスロットを特定する。 4. 全スロットを走査しても空きが見つからない場合、APR_EINVAL を返して終了する。 5. 空きスロットを発見した場合、該当する inuse フラグを 1 に設定し、そのインデックスを id に書き込んで APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_balancer.c: バランサーおよびワーカーの管理において本関数を呼び出し、失敗時にエラーログを出力している。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。
  • **登録**:
  • modules/slotmem/mod_slotmem_plain.c および modules/slotmem/mod_slotmem_shm.c 内のプロバイダー構造体にて、関数ポインタとして登録されている。

関連データ構造

  • ap_slotmem_instance_t: スロットメモリの管理構造体。本関数は、この構造体内の inuse 配列と num（スロット総数）を参照する。

失敗モードと境界条件

  • slot が NULL の場合: APR_ENOSHMAVAIL を返す。
  • 空きスロットが存在しない場合: APR_EINVAL を返す。
  • 境界条件: i が slot->num に達した時点で走査を終了し、確保失敗とみなす。

保守担当者向けメモ

  • 本関数は mod_slotmem_plain.c の実装であるが、mod_slotmem_shm.c からも参照されている。
  • メモリの排他制御は行われないため、呼び出し側で適切に同期されていることを確認すること。
