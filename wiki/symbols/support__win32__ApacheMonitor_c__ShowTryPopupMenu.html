ShowTryPopupMenu

定義

  • 種別: function
  • 定義箇所: support/win32/ApacheMonitor.c:425

目的

ApacheMonitor のタスクトレイアイコンに対するコンテキストメニュー（ポップアップメニュー）を生成し、ユーザーの操作を受け付けるためのインターフェースを表示します。

シグネチャまたは形状

void ShowTryPopupMenu(HWND hWnd)

入力と出力

  • **入力**: hWnd (HWND) - メニューのオーナーとなるウィンドウハンドル。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • g_lpMsg が初期化されており、メニュー項目用の文字列リソースが参照可能であること。
  • hWnd は有効なウィンドウハンドルであること。

アルゴリズムまたは振る舞い

1. CreatePopupMenu() で新しいメニューを作成します。 2. appendMenuItem を使用し、「Restore」「Services Manager」「Exit」および区切り線を追加します。 3. SetForegroundWindow(hWnd) を呼び出し、メニューを表示するウィンドウをフォアグラウンドに設定します（失敗時は NULL を指定）。 4. GetCursorPos(&pt) で現在のマウスカーソル位置を取得します。 5. TrackPopupMenu を使用し、マウス位置にメニューを表示します。 6. ユーザーの選択を待機した後、DestroyMenu(hMenu) を呼び出し、作成したメニューリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: support/win32/ApacheMonitor.c:1355（確認済み）。
  • **呼び出し先**: CreatePopupMenu, appendMenuItem, SetForegroundWindow, GetCursorPos, TrackPopupMenu, DestroyMenu（すべて Windows API または内部補助関数）。

関連データ構造

  • g_lpMsg: メニュー項目に表示する文字列を保持するグローバル配列。

失敗モードと境界条件

  • CreatePopupMenu() が失敗した場合、メニューは表示されず、関数は何も行わずに終了します。
  • SetForegroundWindow が失敗した場合、フォアグラウンド設定の試行はスキップされますが、メニュー表示処理は継続されます。

保守担当者向けメモ

  • メニュー項目の追加や変更を行う場合は、appendMenuItem の引数と g_lpMsg のインデックス（IDS_MSG_...）の整合性に注意してください。
  • メニューリソースの解放漏れを防ぐため、DestroyMenu は必ず呼び出されるパスに配置されています。
