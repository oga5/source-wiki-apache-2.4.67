socache_rd_store

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_redis.c:211

目的

Redis バックエンドを使用して、共有オブジェクトキャッシュ（socache）にデータを保存します。

シグネチャまたは形状

apr_status_t socache_rd_store(ap_socache_instance_t *ctx, server_rec *s,
                              const unsigned char *id, unsigned int idlen,
                              apr_time_t expiry,
                              unsigned char *ucaData, unsigned int nData,
                              apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: Redis インスタンスのコンテキスト。
  • s: サーバーレコード。
  • id, idlen: キャッシュ識別子とその長さ。
  • expiry: キャッシュの有効期限（絶対時刻）。
  • ucaData, nData: 保存するデータ本体とそのサイズ。
  • p: メモリプール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返します。

前提条件と不変条件

  • socache_rd_id2key によるキー変換が成功すること。
  • expiry から現在時刻を引いた値（TTL）が正の数であること。

アルゴリズムまたは振る舞い

1. socache_rd_id2key を呼び出し、内部 ID を Redis 用の文字列キーに変換します。 2. expiry と現在時刻から TTL（秒単位）を算出します。TTL が 0 以下の場合は APR_EINVAL を返します。 3. apr_redis_setex を使用して、算出された TTL を伴うデータを Redis に保存します。 4. 保存処理が失敗した場合は、エラーログを出力し、そのステータスコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_socache_redis.c:393（直接的な参照が確認されています）。
  • **呼び出し先**:
  • socache_rd_id2key: キー変換用。
  • apr_time_sec, apr_time_now: 時間計算用。
  • apr_redis_setex: Redis へのデータ書き込み用。
  • ap_log_error: エラーログ出力用。

関連データ構造

  • ap_socache_instance_t: Redis 接続ハンドル（rc）を保持するコンテキスト構造体。
  • server_rec: サーバー設定情報。

失敗モードと境界条件

  • socache_rd_id2key が失敗した場合: APR_EINVAL を返します。
  • TTL が 0 以下の場合: APR_EINVAL を返します。
  • apr_redis_setex が失敗した場合: APR_SUCCESS 以外のステータスを返し、エラーログを記録します。

保守担当者向けメモ

  • 本関数は ap_socache_provider_t の store プロバイダー関数として登録されています。
  • apr_redis_setex の第 6 引数（フラグ）は 0 で固定されています。
