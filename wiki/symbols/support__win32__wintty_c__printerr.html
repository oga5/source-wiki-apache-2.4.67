printerr

定義

  • 種別: function
  • 定義箇所: support/win32/wintty.c:64

目的

デバッグ用の詳細なエラーメッセージを、Windowsのデバッグ出力（OutputDebugString）へ送信する。

シグネチャまたは形状

void printerr(char *fmt, ...)

入力と出力

  • 入力: fmt（書式指定文字列）、および可変長引数。
  • 出力: なし（戻り値なし）。内部で OutputDebugString を呼び出し、システムデバッガへ文字列を渡す。

前提条件と不変条件

  • グローバル変数 verbose が真（非ゼロ）である場合にのみ実行される。
  • 内部バッファ str は1024バイト固定長であり、これを超える文字列は切り詰められる可能性がある。

アルゴリズムまたは振る舞い

1. verbose フラグを確認し、偽であれば即座に終了する。 2. va_start を使用して可変長引数を初期化する。 3. wvsprintf を使用して、書式指定文字列と引数をバッファ str に展開する。 4. OutputDebugString を呼び出し、生成された文字列をデバッグ出力へ送出する。

呼び出し元と呼び出し先

  • 呼び出し元: support/win32/wintty.c 内の各エラーハンドリング箇所（例: GetProcessWindowStation, GetStdHandle, DupHandle, FreeConsole, GetThreadDesktop, OpenWinSta の失敗時）。
  • 呼び出し先: wvsprintf (Win32 API), OutputDebugString (Win32 API)。

関連データ構造

  • va_list: 可変長引数リストを扱うための標準型。

失敗モードと境界条件

  • verbose が設定されていない場合、呼び出しは無視される。
  • wvsprintf による文字列生成が1024バイトを超える場合、バッファオーバーフローは発生しないが、出力が切り詰められる可能性がある。

保守担当者向けメモ

  • 本関数はデバッグ目的でのみ使用される。運用環境で詳細なログが必要な場合は、verbose フラグの制御を確認すること。
  • wvsprintf は Windows API の関数であり、標準ライブラリの vsnprintf 等とは挙動が異なる点に注意。
