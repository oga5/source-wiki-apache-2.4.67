cache_replace_filter

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache.c:350

目的

Apache HTTP Server の出力フィルタチェーンにおいて、特定のフィルタ（from）を別のフィルタ（to）の設定で置き換える、または不要なフィルタを削除するために使用されます。

シグネチャまたは形状

static int cache_replace_filter(ap_filter_t *next, ap_filter_rec_t *from,
                                ap_filter_rec_t *to, ap_filter_rec_t *stop)

入力と出力

  • **入力**:
  • next: 走査を開始するフィルタチェーンの先頭ポインタ。
  • from: 置き換え対象となるフィルタのレコード定義。
  • to: 置き換え元となる（新しい設定を持つ）フィルタのレコード定義。
  • stop: 走査を終了するフィルタのレコード定義。
  • **出力**:
  • int: 置き換えに成功した場合は 1、from フィルタが見つかり削除されたが置き換えが行われなかった場合は 0 を返します。

前提条件と不変条件

  • next から stop に到達するまでのフィルタチェーン内に from フィルタが存在する必要があります。
  • フィルタの置き換えは、from と to の両方がチェーン内に存在する場合にのみ実行されます。

アルゴリズムまたは振る舞い

1. next から stop に達するまでフィルタチェーンを走査し、from フィルタと to フィルタのインスタンスを特定します。 2. from と to の両方が見つかった場合:

  • from フィルタのレコード定義とコンテキストを to フィルタの内容で上書きします。
  • to フィルタをチェーンから削除（ap_remove_output_filter）します。
  • 1 を返します。

    • 3. from のみが存在する場合:

  • from フィルタをチェーンから削除します。
  • 0 を返します。

    • 4. いずれも見つからない場合は何もせず 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache.c:525
  • modules/cache/mod_cache.c:602
  • **呼び出し先**:
  • ap_remove_output_filter (Apache HTTP Server API)

関連データ構造

  • ap_filter_t: フィルタチェーンの各ノード。
  • ap_filter_rec_t: フィルタの型定義。

失敗モードと境界条件

  • フィルタチェーン内に from フィルタが存在しない場合、何も変更されず 0 が返されます。
  • to フィルタがチェーン内に存在しない場合、from フィルタは単に削除されます。

保守担当者向けメモ

  • 本関数は、管理者が CACHE フィルタを明示的に配置した場合などに、動的にフィルタを差し替える目的で利用されます。
  • フィルタの削除には ap_remove_output_filter を使用しているため、Apache のフィルタ管理ルールに従う必要があります。
