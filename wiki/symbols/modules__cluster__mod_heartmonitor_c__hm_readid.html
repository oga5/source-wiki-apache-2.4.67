hm_readid

定義

  • 種別: function
  • 定義箇所: modules/cluster/mod_heartmonitor.c:183

目的

共有メモリ（slotmem）内のサーバーエントリを走査し、特定のサーバー（IPアドレス）が既に存在するかを確認・特定する。

シグネチャまたは形状

static apr_status_t hm_readid(void* mem, void *data, apr_pool_t *p)

入力と出力

  • **入力**:
  • mem: 共有メモリ上のサーバーエントリ（hm_slot_server_t*）。
  • data: 検索対象のサーバー情報と検索結果を保持するコンテキスト（hm_slot_server_ctx_t*）。
  • p: メモリプール（未使用）。
  • **出力**:
  • 常に APR_SUCCESS を返す。
  • data 引数経由で、一致した場合は found フラグを 1 に設定し、item_id に共有メモリ上の ID を格納する。

前提条件と不変条件

  • mem は hm_slot_server_t 構造体としてキャスト可能であること。
  • data は hm_slot_server_ctx_t 構造体としてキャスト可能であること。
  • data->s には比較対象となる hm_server_t が格納されていること。

アルゴリズムまたは振る舞い

1. mem から共有メモリ上のサーバー IP を取得する。 2. data->s から検索対象のサーバー IP を取得する。 3. 両者の IP を strcmp で比較する。 4. 一致した場合、data->found を 1 に設定し、data->item_id に mem の ID を代入する。 5. 処理結果に関わらず APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cluster/mod_heartmonitor.c:222 において、storage->doall のコールバック関数として呼び出される。
  • **呼び出し先**: strcmp（標準ライブラリ）。

関連データ構造

  • hm_slot_server_t: 共有メモリ上のサーバー状態定義。
  • hm_slot_server_ctx_t: 検索処理のコンテキストおよび結果格納用構造体。
  • hm_server_t: サーバーの稼働状態を保持する構造体。

失敗モードと境界条件

  • IP アドレスが一致しない場合、found フラグは更新されず、次のエントリの走査へ進む。
  • 共有メモリが空の場合や、該当する IP が存在しない場合は found は 0 のままとなる。

保守担当者向けメモ

  • 本関数は slotmem の doall メソッドから呼び出されるイテレータである。
  • 比較対象の IP アドレスが一致した際、共有メモリ上の ID を取得する役割を持つため、この ID は後続の更新処理（hm_slotmem_update_stat 等）で利用される。
