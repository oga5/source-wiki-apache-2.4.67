add_for_purge

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_mplx.c:129

目的

指定されたストリームを、後続の破棄処理（purge）対象リストに追加する。

シグネチャまたは形状

static int add_for_purge(h2_mplx *m, h2_stream *stream)

入力と出力

  • **入力**:
  • m: 多重化インスタンス（h2_mplx）へのポインタ。
  • stream: 破棄対象とするストリーム（h2_stream）へのポインタ。
  • **出力**:
  • int: ストリームがリストに新規追加された場合は TRUE、既にリストに存在していた場合は FALSE を返す。

前提条件と不変条件

  • m->spurge は apr_array_header_t 型の配列である必要がある。
  • 本関数は、h2_mplx インスタンスのミューテックスによって保護されたコンテキストで呼び出されることが想定されている（直接的な排他制御は関数内で行われていない）。

アルゴリズムまたは振る舞い

1. m->spurge 配列を走査し、指定された stream が既に存在するか確認する。 2. 既に存在する場合、何もせず FALSE を返す。 3. 存在しない場合、APR_ARRAY_PUSH を使用して stream を配列の末尾に追加し、TRUE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_mplx.c:146
  • modules/http2/h2_mplx.c:179
  • modules/http2/h2_mplx.c:193
  • **呼び出し先**:
  • APR_ARRAY_IDX (マクロ)
  • APR_ARRAY_PUSH (マクロ)

関連データ構造

  • h2_mplx: 多重化器のメイン構造体。spurge メンバ（破棄待ちストリームの配列）を保持する。
  • h2_stream: HTTP/2 ストリームのコンテキスト。

失敗モードと境界条件

  • m->spurge が初期化されていない場合、またはメモリ不足により APR_ARRAY_PUSH が失敗した場合の挙動は、APR ライブラリの仕様に依存する。

保守担当者向けメモ

  • 本関数はストリームの重複登録を防ぐための単純な線形探索を行っている。spurge 配列のサイズが非常に大きくなる場合、パフォーマンスへの影響を考慮する必要がある。
