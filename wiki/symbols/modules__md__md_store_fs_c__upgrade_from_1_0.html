upgrade_from_1_0

定義

  • 種別: function
  • 定義箇所: modules/md/md_store_fs.c:183

目的

ファイルシステム上のストレージ構造をバージョン 1.0 から現行バージョンへ移行します。具体的には、秘密鍵のファイル名変更と、証明書チェーンの統合（fullcert.pem の生成）を行います。

シグネチャまたは形状

static apr_status_t upgrade_from_1_0(md_store_fs_t *s_fs, apr_pool_t *p, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • s_fs: ストレージの状態を保持する md_store_fs_t 構造体。
  • p: メモリ割り当てに使用するプール。
  • ptemp: 一時的なメモリ割り当てに使用するプール（本関数内では未使用）。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS を返し、失敗時はエラーコードを返します。

前提条件と不変条件

  • s_fs は有効な md_store_fs_t 構造体を指している必要があります。
  • 呼び出し元は、ストアのバージョンが 1.0 であることを確認済みである必要があります。

アルゴリズムまたは振る舞い

1. **秘密鍵の移行**: 全てのストレージグループに対し、rename_pkey 関数を用いて pkey.pem を privkey.pem にリネームします。 2. **証明書の統合**: MD_SG_DOMAINS および MD_SG_ARCHIVE グループ内の各ディレクトリにおいて、mk_pubcert 関数を呼び出し、既存の cert.pem と chain.pem から fullcert.pem を生成します。 3. 各ステップでエラーが発生した場合、その時点で処理を中断し、エラーコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_store_fs.c:239
  • **呼び出し先**:
  • md_util_files_do: ファイルシステム上のファイルを走査し、指定された関数を適用します。
  • rename_pkey: 秘密鍵のリネームを行うコールバック関数。
  • mk_pubcert: 証明書統合を行うコールバック関数。
  • md_store_group_name: グループ列挙型から名前文字列を取得します。

関連データ構造

  • md_store_fs_t: ファイルシステムストレージのコンテキスト。
  • md_store_group_t: ストレージ内のデータグループ（ドメイン、アーカイブ等）。

失敗モードと境界条件

  • ファイルシステム上の権限不足や、リネーム対象のファイルが存在しない場合、md_util_files_do がエラーを返し、本関数もそのエラーを伝播します。
  • 移行対象のファイルが既に存在しない場合や、既に移行済みである場合の挙動については、各コールバック関数（rename_pkey, mk_pubcert）の実装に依存します。

保守担当者向けメモ

  • 本関数は md_store_fs.c 内の内部的な移行処理です。
  • ptemp 引数は現在使用されていませんが、将来的なファイル操作の最適化のために保持されています。
  • mk_pubcert による fullcert.pem の生成は、既存の証明書ファイルが存在する場合にのみ実行されます。
