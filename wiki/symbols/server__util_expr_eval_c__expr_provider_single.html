expr_provider_single

定義

  • 種別: struct
  • 定義箇所: server/util_expr_eval.c:1664

目的

Apache HTTP Server の式評価エンジン（ap_expr）において、単一の引数または特定の機能を持つ変数、関数、演算子プロバイダーを定義するための構造体です。

シグネチャまたは形状

struct expr_provider_single {
    const void *func;
    const char *name;
    ap_expr_lookup_fn_t *arg_parsing_func;
    int restricted;
};

入力と出力

  • func: 実行時に呼び出される関数ポインタ。
  • name: プロバイダーを識別するための名称。
  • arg_parsing_func: 引数の構文解析を行う関数へのポインタ。
  • restricted: セキュリティ制限（ファイルシステム操作の可否など）を示すフラグ。

前提条件と不変条件

  • name は一意である必要があります。
  • func は、対応する評価コンテキストで期待されるシグネチャと一致している必要があります。

アルゴリズムまたは振る舞い

この構造体は、式評価エンジンが利用可能な変数、関数、演算子のテーブルを構築するために使用されます。評価エンジンは、式に含まれる識別子をこのテーブルから検索し、一致するプロバイダーの func を実行します。

呼び出し元と呼び出し先

  • 呼び出し元（定義の参照）:
  • string_func_providers（server/util_expr_eval.c:1684）
  • unary_op_providers（server/util_expr_eval.c:1710）
  • binary_op_providers（server/util_expr_eval.c:1728）
  • 呼び出し先:
  • 評価エンジン内部の検索処理（server/util_expr_eval.c:1758）にて、プロバイダーの検索時に参照されます。

関連データ構造

  • ap_expr_lookup_fn_t: 引数解析関数の型定義。
  • expr_provider_multi: 複数のプロバイダーを扱うための関連構造体。

失敗モードと境界条件

  • 検索時に該当する name が見つからない場合、評価エンジンはエラーを返します。
  • restricted フラグが設定されている場合、非許可のコンテキストからの呼び出しは拒否されます。

保守担当者向けメモ

  • 本構造体は util_expr_eval.c 内の静的テーブル定義の基盤です。新しい演算子や関数を追加する際は、この構造体を用いて各テーブルを拡張してください。
  • func ポインタの型は汎用的な void * であるため、キャスト時の型安全性には十分注意してください。
