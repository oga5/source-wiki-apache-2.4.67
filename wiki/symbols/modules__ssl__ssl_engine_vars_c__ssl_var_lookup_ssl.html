ssl_var_lookup_ssl

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:346

目的

SSLConnRec 構造体から SSL 接続に関連する情報を抽出し、指定された変数名に対応する値を文字列として返します。

シグネチャまたは形状

static char *ssl_var_lookup_ssl(apr_pool_t *p, SSLConnRec *sslconn, 
                                request_rec *r, char *var)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • sslconn: SSL 接続状態を保持する SSLConnRec 構造体。
  • r: 現在のリクエスト情報（証明書抽出時に使用）。
  • var: 取得対象の変数名（例: PROTOCOL, SESSION_ID, CLIENT_CERT_CHAIN_... 等）。
  • **出力**:
  • 成功時: 解決された値を含む文字列（apr_pool_t 上に確保）。
  • 失敗時: NULL。

前提条件と不変条件

  • sslconn が NULL の場合、VERSION_ で始まる変数以外は解決されません。
  • 戻り値はすべて apr_pool_t 上に確保されるため、呼び出し元がプールを管理する必要があります。
  • SSL_get_peer_certificate を呼び出した場合は、必ず X509_free を実行して参照カウントを解放します。

アルゴリズムまたは振る舞い

1. var のプレフィックスや名称に基づき、条件分岐で適切な抽出処理を選択します。 2. VERSION_ 系は ssl_var_lookup_ssl_version を呼び出します。 3. PROTOCOL, SESSION_ID, SESSION_RESUMED などの基本情報は SSL オブジェクトから直接取得します。 4. CIPHER, CLIENT_CERT_CHAIN_, CLIENT_VERIFY などの複雑な情報は、専用のヘルパー関数（ssl_var_lookup_ssl_cipher 等）へ委譲します。 5. CLIENT_ または SERVER_ で始まる変数の場合、証明書（X509）を取得し、ssl_var_lookup_ssl_cert を介して属性を抽出します。 6. TLS_SNI, SECURE_RENEG, SRP_ 関連は、コンパイルオプション（HAVE_TLSEXT, HAVE_SRP）に応じて処理します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_vars.c 内の ssl_var_lookup および他の内部関数。
  • modules/arch/netware/mod_nw_ssl.c（直接の呼び出しを確認）。
  • **呼び出し先**:
  • ssl_var_lookup_ssl_version, ssl_var_lookup_ssl_cipher, ssl_var_lookup_ssl_cert_chain, ssl_var_lookup_ssl_cert_rfc4523_cea, ssl_var_lookup_ssl_cert_verify, ssl_var_lookup_ssl_cert, ssl_var_lookup_ssl_compress_meth 等のヘルパー関数。
  • OpenSSL API (SSL_get_version, SSL_get_session, SSL_get_peer_certificate 等)。

関連データ構造

  • SSLConnRec: SSL 接続のコンテキスト。
  • SSL: OpenSSL の接続オブジェクト。
  • X509: OpenSSL の証明書構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • sslconn->ssl が NULL の場合、多くの変数は解決できず NULL を返します。
  • SSL_get_peer_certificate が NULL を返す場合（クライアント証明書がない場合など）、CLIENT_ 系の変数は解決されません。
  • var が未知の文字列である場合、何も処理されず NULL が返されます。

保守担当者向けメモ

  • SSL_get_certificate と SSL_get_peer_certificate では参照カウントの扱いが異なるため、X509_free の有無に注意してください。
  • OpenSSL のバージョン依存処理（OPENSSL_NO_SSL_INTERN 等）が含まれているため、OpenSSL アップグレード時には挙動の確認が必要です。
