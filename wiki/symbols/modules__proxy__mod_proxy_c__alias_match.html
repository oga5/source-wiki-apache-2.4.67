alias_match

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:532

目的

リクエストURIとプロキシ設定上のエイリアス（alias_fakename）を比較し、パスの先頭部分が一致するかを判定する。

シグネチャまたは形状

int alias_match(const char *uri, const char *alias_fakename)

入力と出力

  • **入力**:
  • uri: 比較対象のリクエストURI文字列。
  • alias_fakename: マッチングの基準となるエイリアス文字列。
  • **出力**:
  • int: マッチしたURIの文字数。一致しない場合は 0 を返す。

前提条件と不変条件

  • スラッシュ（/）は、連続していても単一の / と同様に扱われる。
  • エイリアス側で指定されたパス構成要素が、URI側で完全に一致する必要がある。

アルゴリズムまたは振る舞い

1. uri と alias_fakename を先頭から順に比較する。 2. スラッシュが現れた場合、両方の文字列で連続するスラッシュをスキップし、少なくとも1つのスラッシュが両方に存在することを確認する。 3. スラッシュ以外の文字は逐次比較を行う。 4. URIの末尾に達した際、エイリアスがまだ残っている場合は不一致（0）とする。 5. 最後に、エイリアスの末尾がスラッシュでない場合、URI側の次の文字が終端かスラッシュであることを確認する（パス構成要素の境界チェック）。 6. 成功時は、マッチしたURIの長さを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy.c:904
  • modules/proxy/mod_proxy.c:914
  • **呼び出し先**:
  • 標準ライブラリ関数: strlen

関連データ構造

  • 特になし。

失敗モードと境界条件

  • URIがエイリアスより短い場合、またはパスの境界で不一致が生じた場合は 0 を返す。
  • URIがエイリアスと完全に一致するか、エイリアスのパス構成要素をすべて含んでいる必要がある。

保守担当者向けメモ

  • 本関数は、ProxyPass 等の設定におけるパスのプレフィックスマッチングに使用される。
  • 戻り値はマッチした文字数であり、単なる真偽値ではない点に注意すること。
