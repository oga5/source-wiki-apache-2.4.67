add_alias_internal

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_alias.c:139

目的

Apache HTTP Server の Alias および AliasMatch ディレクティブの設定を内部構造体 alias_entry に登録し、サーバー設定へ追加する。

シグネチャまたは形状

static const char *add_alias_internal(cmd_parms *cmd, void *dummy,
                                      const char *fake, const char *real,
                                      int use_regex)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • dummy: 使用されていない引数。
  • fake: マッピング元の URI パスまたは正規表現。
  • real: マッピング先のファイルシステムパス。
  • use_regex: 正規表現を使用するかどうかを示すフラグ。
  • **出力**: 成功時は NULL を返し、エラー発生時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • Alias ディレクティブはディレクトリコンテキスト（<Directory> 等）内では使用できない（ap_check_cmd_context により NOT_IN_DIR_CONTEXT が強制される）。
  • use_regex が真の場合、fake は ap_pregcomp でコンパイル可能な正規表現である必要がある。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、ディレクトリコンテキスト外であることを確認する。 2. alias_server_conf の aliases 配列に新しい alias_entry を確保する。 3. use_regex が真の場合、fake を正規表現としてコンパイルし、失敗した場合はエラーを返す。 4. fake、real、および cmd->info（ハンドラ情報）を alias_entry に格納する。 5. use_regex が偽の場合、既存のエイリアス定義との重複をチェックし、重複が検出された場合は ap_log_error で警告を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_alias.c:206 (非正規表現用)
  • modules/mappers/mod_alias.c:243 (正規表現用)
  • **呼び出し先**:
  • ap_get_module_config
  • apr_array_push
  • ap_check_cmd_context
  • ap_pregcomp (正規表現時)
  • alias_matches (重複チェック時)
  • ap_regexec (正規表現の重複チェック時)
  • ap_log_error

関連データ構造

  • alias_entry: エイリアスの定義を保持する構造体。
  • alias_server_conf: サーバー全体の設定を保持する構造体。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • ディレクトリコンテキスト内で呼び出された場合、エラーメッセージを返す。
  • 正規表現のコンパイルに失敗した場合、エラーメッセージを返す。
  • 既存のエイリアスと重複する定義が追加された場合、警告ログを出力する（処理は継続される）。

保守担当者向けメモ

  • real パスが DocumentRoot に対して相対的ではないという既知の互換性バグが存在する。
  • use_regex が偽の場合の最適化余地がコメントとして残されているが、古い ap_canonical 関連の整理が完了するまで保留されている。
