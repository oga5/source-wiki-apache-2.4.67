ap_regkey_t

定義

  • 種別: struct
  • 定義箇所: os/win32/ap_regkey.c:24

目的

Windows レジストリキーのハンドル (HKEY) と、そのキーに関連付けられた APR プール (apr_pool_t) を保持し、レジストリ操作のライフサイクルを管理するためのコンテナです。

シグネチャまたは形状

struct ap_regkey_t {
    apr_pool_t *pool;
    HKEY        hkey;
};

入力と出力

  • pool: レジストリキーの生存期間を管理する APR プールへのポインタ。
  • hkey: Win32 API で使用されるレジストリキーハンドル。

前提条件と不変条件

  • hkey は、関連付けられた pool が破棄されるか、ap_regkey_close() が呼び出されるまで有効である必要があります。
  • pool のクリーンアップハンドラとして regkey_cleanup が登録されており、プール破棄時に自動的に RegCloseKey が実行される不変条件を持ちます。

アルゴリズムまたは振る舞い

  • 本構造体は、レジストリ操作の抽象化レイヤーにおいて、リソース（ハンドル）とメモリ管理（プール）を紐付ける役割を果たします。
  • ap_regkey_open() 等の関数によって生成され、ap_regkey_close() によって明示的に解放されるか、プール破棄時に自動的にクリーンアップされます。

呼び出し元と呼び出し先

  • 呼び出し元: ap_regkey_open, ap_regkey_close, ap_regkey_remove, ap_regkey_value_get/set 等の ap_regkey_* 系関数が本構造体を引数として受け取ります。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_pool_t: メモリ管理およびクリーンアップハンドラの登録に使用。
  • HKEY: Windows API のレジストリハンドル。

失敗モードと境界条件

  • pool が破棄された後に本構造体を参照した場合、hkey は無効なハンドルとなり、後続のレジストリ操作は失敗します。

保守担当者向けメモ

  • 本構造体は include/ap_regkey.h:35 にも定義が存在します。
  • レジストリの列挙機能は本モジュールには含まれていないため、拡張が必要な場合は注意してください。
  • ap_regkey_remove は非再帰的であり、サブキーが存在する場合には削除が失敗する仕様です。
