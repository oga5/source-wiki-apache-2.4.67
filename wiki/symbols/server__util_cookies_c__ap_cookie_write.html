ap_cookie_write

定義

  • 種別: function
  • 定義箇所: server/util_cookies.c:40

目的

RFC 2109 準拠のクッキー文字列を生成し、指定された一つ以上のヘッダーテーブル（apr_table_t）に追加します。

シグネチャまたは形状

apr_status_t ap_cookie_write(request_rec *r, const char *name, const char *val,
                             const char *attrs, long maxage, ...);

入力と出力

  • **入力**:
  • r: リクエストレコード（メモリ割り当て用プール r->pool を含む）。
  • name: クッキー名。
  • val: クッキー値。
  • attrs: クッキー属性（指定がない場合は DEFAULT_ATTRS を使用）。
  • maxage: 有効期限（秒数）。0 の場合は Max-Age 属性を省略。
  • ...: 可変長引数として渡される apr_table_t * のリスト（NULL 終端）。
  • **出力**:
  • 戻り値: APR_SUCCESS を返します。

前提条件と不変条件

  • すべての文字列操作は r->pool を使用して行われるため、呼び出し側での明示的なメモリ解放は不要です。
  • 可変長引数リストは NULL で終了している必要があります。

アルゴリズムまたは振る舞い

1. maxage が指定されている場合、Max-Age 属性文字列を生成します。 2. name、val、attrs（またはデフォルト値）、および maxage 文字列を連結し、RFC 2109 形式のクッキー文字列を作成します。 3. デバッグログに生成されたクッキー文字列を出力します。 4. 可変長引数で渡された各 apr_table_t に対して、apr_table_addn を用いて Set-Cookie ヘッダーとしてクッキー文字列を追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session_cookie.c
  • modules/session/mod_session_dbd.c
  • **呼び出し先**:
  • apr_pstrcat (メモリ確保と文字列連結)
  • apr_ltoa (数値の文字列変換)
  • ap_log_rerror (ログ出力)
  • apr_table_addn (ヘッダーテーブルへの追加)

関連データ構造

  • request_rec: リクエスト状態とメモリプールを保持。
  • apr_table_t: HTTP ヘッダーを格納するテーブル構造。

失敗モードと境界条件

  • attrs が NULL または空文字列の場合、デフォルトの属性が適用されます。
  • maxage が 0 の場合、Max-Age 属性はヘッダーに含まれません。
  • 可変長引数リストが正しく NULL 終端されていない場合、メモリ不正アクセスのリスクがあります。

保守担当者向けメモ

  • 本関数は Set-Cookie ヘッダーのみを生成します。Set-Cookie2 を扱う場合は ap_cookie_write2 を検討してください。
  • クッキー名のバリデーションが必要な場合は、事前に ap_cookie_check_string を使用してください。
