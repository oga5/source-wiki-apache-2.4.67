authn_dbd_prepare

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_dbd.c:61

目的

Apache の設定ファイルで指定された SQL クエリを解析し、mod_dbd の準備済みステートメントとして登録することで、認証フェーズでのクエリ実行を可能にする。

シグネチャまたは形状

static const char *authn_dbd_prepare(cmd_parms *cmd, void *cfg, const char *query)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • cfg: ディレクトリ設定構造体。
  • query: 設定ファイルから読み込まれた SQL クエリ文字列。
  • **出力**:
  • 成功時は NULL。
  • 失敗時はエラーメッセージ文字列を返す。

前提条件と不変条件

  • .htaccess ファイル内での使用は許可されない（ap_check_cmd_context による制限）。
  • mod_dbd がロードされている必要がある。ロードされていない場合、ap_dbd_prepare 関数の取得に失敗しエラーを返す。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、コンテキストが適切か検証する。 2. authn_dbd_prepare_fn が未初期化の場合、APR_RETRIEVE_OPTIONAL_FN を使用して mod_dbd の ap_dbd_prepare および ap_dbd_acquire 関数を取得する。 3. 内部カウンタ label_num をインクリメントし、一意なラベル名（authn_dbd_N）を生成する。 4. authn_dbd_prepare_fn を呼び出し、クエリとラベルを mod_dbd に登録する。 5. ap_set_string_slot を使用して、生成したラベルを cfg に保存する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authn_dbd.c 内の AP_INIT_TAKE1 マクロ（AuthDBDUserPWQuery および AuthDBDUserRealmQuery ディレクティブのハンドラとして登録）。
  • **呼び出し先**:
  • ap_check_cmd_context
  • APR_RETRIEVE_OPTIONAL_FN (ap_dbd_prepare, ap_dbd_acquire)
  • apr_psprintf
  • ap_set_string_slot

関連データ構造

  • cmd_parms: 設定コマンドの実行コンテキスト。
  • authn_dbd_conf: cfg として渡されるディレクトリ設定構造体。

失敗モードと境界条件

  • mod_dbd がロードされていない場合、認証機能の有効化を拒否する。
  • ap_check_cmd_context がエラーを返した場合、そのエラーメッセージをそのまま伝播させる。

保守担当者向けメモ

  • 本関数は設定読み込み時にのみ実行される。
  • label_num は静的変数であり、モジュール内で一意なラベルを生成するために使用される。
  • mod_dbd への依存はオプション関数経由で行われており、疎結合が維持されている。
