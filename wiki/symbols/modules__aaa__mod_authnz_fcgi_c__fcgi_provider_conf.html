fcgi_provider_conf

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:36

目的

FastCGI プロトコルを用いた外部認証（Authn）および認可（Authz）バックエンドへの接続情報と、その役割を保持するための設定構造体です。

シグネチャまたは形状

typedef struct {
    const char *name;             /* プロバイダー名 */
    const char *backend;          /* 設定されたバックエンドアドレス */
    const char *host;             /* ホスト名 */
    apr_port_t port;              /* ポート番号 */
    apr_sockaddr_t *backend_addrs; /* バックエンドのソケットアドレス */
    int is_authn;                 /* 認証プロバイダーとして使用するか */
    int is_authz;                 /* 認可プロバイダーとして使用するか */
} fcgi_provider_conf;

入力と出力

本構造体は設定データとして保持され、モジュール内の各関数へ引数として渡されます。

  • **入力**: 設定ファイル（AuthnzFcgiDefineProvider）から読み込まれた値。
  • **出力**: ネットワーク通信関数（recv_data, sendv_data 等）およびログ出力関数（log_provider_info）への参照として利用されます。

前提条件と不変条件

  • backend_addrs は、バックエンドへの接続確立後に有効な apr_sockaddr_t を指す必要があります。
  • is_authn および is_authz は、プロバイダーの役割を示すフラグとして機能します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用ですが、以下の処理で参照されます。

  • **通信**: req_rsp 関数において、FastCGI リクエストの送信およびレスポンス受信の制御に使用されます。
  • **接続**: connect_to_peer（直接の言及なし）等の処理において、接続先ホストやポートの特定に利用されます。
  • **解析**: handle_response 関数において、バックエンドからのレスポンスを処理する際のコンテキストとして使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_authnz_fcgi.c 内の各処理関数（log_provider_info, recv_data, recv_data_full, sendv_data, handle_response, req_rsp）が本構造体を引数として受け取ります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • apr_sockaddr_t: APR ライブラリのソケットアドレス構造体。
  • request_rec: Apache のリクエスト処理用構造体（req_rsp 等で併用）。

失敗モードと境界条件

  • backend_addrs が初期化されていない状態で通信関数が呼び出された場合、接続エラーが発生します。
  • is_authn および is_authz が共に 0 の場合、プロバイダーとして機能せず、認証・認可フェーズで無視される可能性があります。

保守担当者向けメモ

  • 本構造体は TCP 接続のみを想定しており、AF_UNIX ソケットには対応していません。
  • req_rsp 関数内では、リクエスト処理ごとの一時的なメモリプール（temp_pool）と組み合わせて使用されることが多いため、メモリ管理に注意してください。
