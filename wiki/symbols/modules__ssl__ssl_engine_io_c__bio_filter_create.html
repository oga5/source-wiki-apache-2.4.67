bio_filter_create

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_io.c:165

目的

OpenSSL のカスタム BIO オブジェクトの初期化を行い、HTTPD フィルタスタックとの統合準備を行う。

シグネチャまたは形状

static int bio_filter_create(BIO *bio)

入力と出力

  • **入力**: BIO *bio (初期化対象の OpenSSL BIO オブジェクト)
  • **出力**: int (成功時は 1 を返す)

前提条件と不変条件

  • 引数 bio は有効な BIO 構造体ポインタであること。
  • OpenSSL 1.1.0 未満の環境では、内部の num フィールドが -1 に設定される。

アルゴリズムまたは振る舞い

1. BIO_set_shutdown(bio, 1) を呼び出し、BIO のシャットダウンフラグを有効にする。 2. BIO_set_init(bio, 1) を呼び出し、BIO の初期化済みフラグを立てる。 3. OpenSSL 1.1.0 未満の API を使用している場合、bio->num を -1 に設定する。 4. BIO_set_data(bio, NULL) を呼び出し、BIO に関連付けられたデータポインタを初期化（クリア）する。 5. 成功として 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_io.c:634 (BIO_meth_set_create 経由で bio_filter_out_method に登録)
  • modules/ssl/ssl_engine_io.c:643 (BIO_meth_set_create 経由で bio_filter_in_method に登録)
  • **呼び出し先**:
  • BIO_set_shutdown
  • BIO_set_init
  • BIO_set_data

関連データ構造

  • BIO (OpenSSL 構造体)

失敗モードと境界条件

  • 本関数は常に 1 を返すため、関数内部での失敗モードは定義されていない。

保守担当者向けメモ

  • 本関数は BIO_METHOD の作成時にコールバックとして登録される。
  • BIO_set_data で NULL を設定しているため、この BIO を使用する際は別途コンテキスト（bio_filter_in_ctx_t や bio_filter_out_ctx_t）を BIO_set_data で紐付ける必要がある。
