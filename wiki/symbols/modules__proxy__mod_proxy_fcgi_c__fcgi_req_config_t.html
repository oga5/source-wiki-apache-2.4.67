fcgi_req_config_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:30

目的

FastCGI プロキシリクエストにおいて、リクエスト単位で保持すべきパス情報を管理するための構造体です。

シグネチャまたは形状

typedef struct {
    char *dirwalk_uri_path;
} fcgi_req_config_t;

入力と出力

  • **入力**: dirwalk_uri_path は、リクエスト処理中に PATH_INFO の計算やパスの正規化に使用される文字列を保持します。
  • **出力**: なし（本構造体は設定の保持を目的としています）。

前提条件と不変条件

  • apr_pcalloc を使用してリクエストプール（r->pool）上に割り当てられることが想定されています。
  • ap_get_module_config を介してリクエストコンテキストから取得されます。

アルゴリズムまたは振る舞い

  • リクエスト処理の初期段階でメモリが割り当てられ、その後、リクエストのライフサイクルを通じて PATH_INFO の決定などのパス関連処理において参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_fcgi.c:141: apr_pcalloc によるメモリ割り当て。
  • modules/proxy/mod_proxy_fcgi.c:362: ap_get_module_config による設定の取得。
  • **呼び出し先**: 情報なし。

関連データ構造

  • fcgi_dirconf_t: ディレクトリ単位の設定を保持する構造体。
  • request_rec: Apache のリクエスト構造体（r->request_config を通じて本構造体を保持）。

失敗モードと境界条件

  • メモリ割り当て失敗時（apr_pcalloc）の挙動は Apache の標準的なメモリ管理に依存します。
  • dirwalk_uri_path が初期化されていない状態で参照された場合の動作については、ソースコードの断片からは判断できません。

保守担当者向けメモ

  • 本構造体は PATH_INFO の計算に関連しています。将来的にパス処理のロジックを変更する場合は、この構造体が保持するパス情報の整合性に注意してください。
