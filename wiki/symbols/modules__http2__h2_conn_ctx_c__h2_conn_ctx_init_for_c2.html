h2_conn_ctx_init_for_c2

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_conn_ctx.c:77

目的

ストリーム処理用接続（C2）のコンテキストを初期化し、親接続（C1）から必要な情報を継承・設定する。

シグネチャまたは形状

apr_status_t h2_conn_ctx_init_for_c2(h2_conn_ctx_t **pctx, conn_rec *c2,
                                     struct h2_mplx *mplx, struct h2_stream *stream,
                                     struct h2_c2_transit *transit)

入力と出力

  • **入力**:
  • c2: ストリーム処理用の接続レコード（master 接続を持つこと）。
  • mplx: 接続に関連付けられるマルチプレクサ。
  • stream: 処理対象の HTTP/2 ストリーム。
  • transit: C2 転送用コンテキスト。
  • **出力**:
  • pctx: 初期化された h2_conn_ctx_t 構造体へのポインタ。
  • 戻り値: 成功時は APR_SUCCESS。

前提条件と不変条件

  • c2->master が存在すること（ap_assert により検証）。
  • c2->master に紐付く c1_ctx およびその session が存在すること。

アルゴリズムまたは振る舞い

1. c2 に既存のコンテキストがない場合、親接続（c2->master）から c1_ctx を取得し、ctx_create を用いて新規コンテキストを生成・設定する。 2. 取得したコンテキストに対し、以下の値を設定する：

  • mplx, transit, stream_id。
  • c2->pool を親として req_pool を作成し、タグ付けを行う。
  • stream->request をコンテキストに保持。

    • 3. アトミック操作により started を 1 に設定し、started_at に現在時刻を記録する。 4. done フラグおよび done_at を初期化（0）する。 5. 初期化済みコンテキストを *pctx に格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_mplx.c:867
  • **呼び出し先**: h2_conn_ctx_get, ap_assert, ctx_create, apr_pool_create, apr_pool_tag, apr_atomic_set32, apr_time_now

関連データ構造

  • h2_conn_ctx_t: 接続ごとの状態を保持する主要構造体。
  • conn_rec: Apache の接続レコード。
  • h2_mplx, h2_stream, h2_c2_transit: HTTP/2 処理に関連するサブシステム構造体。

失敗モードと境界条件

  • c2->master が NULL の場合、ap_assert により異常終了する。
  • c1_ctx または c1_ctx->session が NULL の場合、ap_assert により異常終了する。

保守担当者向けメモ

  • 本関数は C2 接続のライフサイクル管理の一部であり、req_pool の作成を通じてリクエスト単位のメモリ管理を行う。
  • apr_atomic_set32 を使用しているため、マルチスレッド環境での状態追跡を意図している。
