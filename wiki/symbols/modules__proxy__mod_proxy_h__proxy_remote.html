proxy_remote

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy.h:119

目的

フォワードプロキシ設定において、特定のスキームやホストに対する転送先プロキシサーバーの情報を保持するために使用されます。

シグネチャまたは形状

struct proxy_remote {
    const char *scheme;     /* 処理対象のスキーム、または '*' */
    const char *protocol;   /* プロキシとの通信に使用するスキーム */
    const char *hostname;   /* プロキシのホスト名 */
    ap_regex_t *regexp;     /* 正規表現（使用時） */
    const char *creds;      /* プロキシ認証用クレデンシャル */
    int use_regex;          /* 正規表現パターンを使用する場合に真 */
    apr_port_t  port;       /* プロキシのポート番号 */
};

入力と出力

本構造体は設定データとして保持されます。入力は主に ProxyRemote ディレクティブの設定解析時に生成され、出力（参照）はリクエスト処理時のプロキシ転送先決定に使用されます。

前提条件と不変条件

  • use_regex が真の場合、regexp フィールドにコンパイル済みの正規表現が格納されている必要があります。
  • scheme には特定のプロトコル名、またはワイルドカード '*' が指定可能です。

アルゴリズムまたは振る舞い

リクエストされた URL のスキームやホスト名に基づき、設定された proxy_remote のリストを走査して一致する転送先を特定します。正規表現が指定されている場合は、それを用いてマッチングを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy.c 内の ProxyRemote ディレクティブ処理および設定反映ロジック（mod_proxy.c:1221, 1562, 1867 付近）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出し先は定義されていません。

関連データ構造

  • ap_regex_t: 正規表現マッチングに使用される Apache の構造体。
  • apr_port_t: ポート番号を保持する APR 型。

失敗モードと境界条件

  • use_regex が真であるにもかかわらず regexp が適切に初期化されていない場合、マッチング処理で予期せぬ動作を引き起こす可能性があります。
  • ポート番号が指定されていない場合、デフォルト値の扱いがプロトコルハンドラ側に依存します。

保守担当者向けメモ

  • 本構造体は apr_array_header_t 等の配列として管理されることが一般的です。
  • 認証情報（creds）の取り扱いにはセキュリティ上の注意が必要です。
  • 呼び出し元・呼び出し先の詳細なフローについては、mod_proxy.c の設定処理およびリクエスト処理ロジックを参照してください。
