isapi_load

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_isapi.c:254

目的

指定された ISAPI 拡張（DLL）を動的にロードし、必要なエントリポイントを解決して初期化を行う。

シグネチャまたは形状

apr_status_t isapi_load(apr_pool_t *p, server_rec *s, isapi_loaded *isa)

入力と出力

  • **入力**:
  • p: リクエストまたはモジュールスコープのメモリプール。
  • s: サーバーレコード。
  • isa: ロード対象の DLL 情報およびエントリポイントを保持する isapi_loaded 構造体（filename が設定済みであること）。
  • **出力**:
  • 戻り値: 成功時は APR_SUCCESS、失敗時は apr_status_t エラーコード。
  • isa 構造体: handle、GetExtensionVersion、HttpExtensionProc、TerminateExtension、isapi_version が更新される。

前提条件と不変条件

  • isa->filename にロードすべき DLL のパスが格納されている必要がある。
  • isa->isapi_version は関数内で apr_pcalloc により割り当てられる。

アルゴリズムまたは振る舞い

1. isa->isapi_version 用のメモリを確保し、バージョン情報（0x500）とタイムアウト値を設定する。 2. apr_dso_load を使用して DLL をロードする。 3. apr_dso_sym を使用して以下の関数ポインタを解決する。

  • GetExtensionVersion（必須）
  • HttpExtensionProc（必須）
  • TerminateExtension（任意）

    • 4. GetExtensionVersion を呼び出し、拡張の初期化を行う。失敗した場合は DLL をアンロードする。 5. apr_pool_cleanup_register を使用し、プール破棄時に cleanup_isapi が実行されるよう登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/win32/mod_isapi.c:394
  • modules/arch/win32/mod_isapi.c:446
  • **呼び出し先**:
  • apr_dso_load, apr_dso_sym, apr_dso_unload (APR)
  • ap_log_error (Apache)
  • GetExtensionVersion (ロードした DLL 内の関数)
  • cleanup_isapi (クリーンアップハンドラ)

関連データ構造

  • isapi_loaded: DLL のハンドル、関数ポインタ、ファイル名、バージョン情報を保持する構造体。
  • HSE_VERSION_INFO: ISAPI 拡張のバージョン情報を保持する構造体。

失敗モードと境界条件

  • DLL のロード失敗 (apr_dso_load): エラーログを出力し、エラーコードを返す。
  • 必須シンボルの解決失敗 (GetExtensionVersion, HttpExtensionProc): エラーログを出力し、DLL をアンロードした上でエラーコードを返す。
  • GetExtensionVersion の実行失敗: エラーログを出力し、DLL をアンロードした上でエラーコードを返す。

保守担当者向けメモ

  • TerminateExtension はオプションとして扱われており、解決に失敗しても処理は中断されない。
  • isa->timeout は現在使用されていない。
  • DLL のロードおよびシンボル解決は apr_dso API に依存している。
