ocsp_resp

定義

  • 種別: struct
  • 定義箇所: modules/ssl/ssl_util_stapling.c:780

目的

OCSP レスポンスのバイナリデータとその長さを保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    unsigned char *data;
    apr_size_t len;
} ocsp_resp;

入力と出力

  • data: OCSP レスポンスの DER エンコードされたバイト列へのポインタ。
  • len: data が指すバッファのサイズ（バイト単位）。

前提条件と不変条件

  • data は有効なメモリ領域を指している必要があります。
  • len は格納されているデータの実際のサイズと一致している必要があります。

アルゴリズムまたは振る舞い

本構造体は、OCSP レスポンスの生データをカプセル化し、mod_ssl のステープリング処理においてキャッシュへの保存や取得、あるいは OpenSSL API への受け渡しを行う際のコンテナとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/ssl/ssl_util_stapling.c 内の関数（例: stapling_cb 等）で、レスポンスの保持や受け渡しに使用されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • 補足: modules/md/md_ocsp.c において同名の変数が宣言されていますが、これが本構造体と同一の定義を参照しているかは、提供された情報からは断定できません。

関連データ構造

  • certinfo: OCSP レスポンスに関連付けられる証明書情報を保持する構造体。

失敗モードと境界条件

  • data が NULL の場合や len が不適切な値である場合、後続の OpenSSL 関数（d2i_OCSP_RESPONSE 等）によるパース処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は modules/ssl/ssl_util_stapling.c 内で定義されています。
  • modules/md/md_ocsp.c に存在する同名のシンボルとの関連性については、ソースコードの依存関係を詳細に調査する必要があります。
