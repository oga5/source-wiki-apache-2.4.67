ap_realloc

定義

  • 種別: function
  • 定義箇所: server/util.c:3168

目的

標準の realloc をラップし、メモリ割り当て失敗時に Apache HTTP Server のプロセスを安全に終了させるためのラッパー関数です。

シグネチャまたは形状

void *ap_realloc(void *ptr, size_t size)

入力と出力

  • **入力**:
  • ptr: 再割り当て対象のメモリ領域へのポインタ（NULL の場合、malloc と同様に動作）。
  • size: 要求する新しいメモリサイズ（バイト単位）。
  • **出力**:
  • 成功時: 再割り当てされたメモリ領域へのポインタ。
  • 失敗時: 戻り値なし（プロセスが終了するため）。

前提条件と不変条件

  • size が 0 以外で、かつ realloc が NULL を返した場合、本関数は ap_abort_on_oom() を呼び出し、プロセスを強制終了します。
  • size が 0 の場合、realloc の仕様に従い、メモリの解放が行われる可能性があります。

アルゴリズムまたは振る舞い

1. 標準ライブラリの realloc(ptr, size) を実行します。 2. 戻り値が NULL かつ size が 0 でない場合、メモリ不足（OOM）と判断し、ap_abort_on_oom() を呼び出して異常終了します。 3. 正常に割り当てられた場合、そのポインタを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_util.c:210 などで利用されていることが確認されています。
  • **呼び出し先**: 標準ライブラリの realloc および ap_abort_on_oom を呼び出します。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • size が 0 の場合: realloc の挙動に依存します。
  • メモリ不足時: ap_abort_on_oom() によりプロセスが終了するため、呼び出し元へ制御は戻りません。

保守担当者向けメモ

  • 本関数はマクロとして実装される可能性がある旨が include/httpd.h に記載されています。
  • メモリ割り当て失敗時の挙動を統一するため、直接 realloc を使用せず、本関数を使用することが推奨されます。
