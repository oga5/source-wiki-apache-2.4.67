proxy_html_conf

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_proxy_html.c:105

目的

mod_proxy_html モジュールの設定情報を保持する構造体です。リバースプロキシ環境において、HTML コンテンツ内の URL 置換ルール、Doctype、バッファサイズ、および各種有効化フラグを一元管理します。

シグネチャまたは形状

typedef struct {
    urlmap *map;
    const char *doctype;
    const char *etag;
    unsigned int flags;
    int bufsz;
    apr_hash_t *links;
    apr_array_header_t *events;
    const char *charset_out;
    int extfix;
    int metafix;
    int strip_comments;
    int interp;
    int enabled;
} proxy_html_conf;

入力と出力

  • 本構造体は、Apache の設定フェーズで生成・マージされ、リクエスト処理時にフィルタコンテキストへ渡されます。
  • apr_pcalloc や apr_palloc を用いてメモリプール (apr_pool_t) 上に割り当てられます。

前提条件と不変条件

  • apr_pool_t に紐付いており、リクエスト終了時に自動的に解放されます。
  • urlmap などのポインタメンバは、設定のロードおよびマージ処理によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • 設定の継承（マージ）: BASE 設定と ADD 設定を組み合わせて新しい設定を作成する際に利用されます。
  • フィルタ実行時: proxy_html_filter 内で、現在のリクエストに対する設定 (cfg) として参照され、置換ルールや動作フラグの判定に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: mod_proxy_html.c 内の複数の箇所で、設定の初期化、マージ、およびフィルタ実行時の参照として使用されています（例: 1028, 1039, 1040, 1041, 1151 行など）。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • urlmap: 置換ルールの定義を保持する構造体。
  • apr_hash_t: リンク定義の保持に使用。
  • apr_array_header_t: イベント定義の保持に使用。

失敗モードと境界条件

  • メモリ不足: apr_pcalloc 等のメモリ割り当て失敗時に Apache のエラー処理フローへ依存します。
  • 設定の不整合: urlmap が未定義または不正な状態でフィルタが実行された場合、置換処理が正しく行われない可能性があります。

保守担当者向けメモ

  • 本構造体は mod_proxy_html の設定管理の核です。新しい設定ディレクティブを追加する場合は、この構造体にメンバを追加し、マージ処理を更新する必要があります。
  • apr_pool_t を使用しているため、手動での free は不要ですが、複雑なデータ構造をメンバに追加する場合はクリーンアップハンドラの登録を検討してください。
