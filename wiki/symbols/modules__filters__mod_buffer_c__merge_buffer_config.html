merge_buffer_config

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_buffer.c:308

目的

ディレクトリ単位の設定構造体（buffer_conf）をマージし、親設定から子設定への継承ロジックを提供します。

シグネチャまたは形状

static void *merge_buffer_config(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: 設定割り当て用のメモリプール。
  • basev: 親ディレクトリの設定（buffer_conf）。
  • addv: 現在のディレクトリの設定（buffer_conf）。
  • **出力**:
  • マージされた新しい buffer_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は buffer_conf 型としてキャスト可能であること。
  • apr_pcalloc により、戻り値の構造体はゼロ初期化された状態で確保されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい buffer_conf 構造体をメモリプール p 上に確保します。 2. size フィールドのマージ: add->size_set が設定されている（!= 0）場合は add->size を採用し、そうでない場合は base->size を継承します。 3. size_set フィールドのマージ: 親または子のいずれかで設定が行われていれば、新しい設定でも設定済み（true）として扱います。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_buffer.c:348 にて、ディレクトリ設定のマージ用として参照されています。
  • **呼び出し先**: 直接的な関数呼び出しは含まれていません。

関連データ構造

  • buffer_conf: バッファサイズ（size）と設定有無フラグ（size_set）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、Apache の標準的なエラー処理（プロセス終了等）に依存します。

保守担当者向けメモ

  • 本関数は Apache の設定マージ機構の一部です。size_set フラグの論理和をとることで、設定の継承階層において意図した値が正しく伝播するように設計されています。
