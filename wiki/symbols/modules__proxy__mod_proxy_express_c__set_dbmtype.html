set_dbmtype

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_express.c:49

目的

Apache HTTP Server の設定ディレクティブ ProxyExpressDBMType を処理し、DBM ファイルの形式を指定する。

シグネチャまたは形状

static const char *set_dbmtype(cmd_parms *cmd, void *dconf, const char *arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • dconf: ディレクトリ設定（本関数では未使用）。
  • arg: 設定ファイルから渡される DBM タイプ文字列。
  • **出力**:
  • 成功時は NULL を返す。

前提条件と不変条件

  • ap_get_module_config を介して、現在のサーバー設定 (express_server_conf) が取得可能であること。

アルゴリズムまたは振る舞い

1. cmd->server->module_config から proxy_express_module に関連付けられた express_server_conf 構造体を取得する。 2. 取得した構造体の dbmtype メンバに、引数 arg を代入する。 3. NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_express.c:228 における AP_INIT_TAKE1 マクロ（設定解析時）。
  • **呼び出し先**: ap_get_module_config（Apache Core API）。

関連データ構造

  • express_server_conf: サーバーごとの設定を保持する構造体。dbmtype メンバを更新する。

失敗モードと境界条件

  • cmd->server が無効な場合、ap_get_module_config の呼び出しで異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は単なる設定値の代入を行うハンドラである。
  • dbmtype の値は、後続の apr_dbm_open2 等の呼び出し時に使用される。
