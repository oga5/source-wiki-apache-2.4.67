watchdog_list_t

定義

  • 種別: struct
  • 定義箇所: modules/core/mod_watchdog.c:33

目的

mod_watchdog において、登録されたコールバック関数とその実行状態を管理するためのリンクリスト構造体です。

シグネチャまたは形状

ソースコード上では不完全型として宣言されており、内部構造の詳細は modules/core/mod_watchdog.c 内で定義されます。リンクリストとして機能するため、自身へのポインタ（next）を保持する形状です。

入力と出力

  • **入力**: ap_watchdog_register_callback を通じて登録されるコールバック関数および実行間隔。
  • **出力**: wd_worker スレッドのループ内でのコールバック実行。

前提条件と不変条件

  • ap_watchdog_t 構造体の callbacks メンバを起点とするリストの一部として管理されます。
  • リスト内の各要素は、登録時に指定されたプール（w->pool）のライフサイクルに従ってメモリが確保されます。

アルゴリズムまたは振る舞い

  • wd_worker スレッドが定期的にリストを走査します。
  • 各要素に設定された実行間隔（interval）に基づき、経過時間が閾値を超えた場合に登録されたコールバック関数を呼び出します。
  • リストの末尾は NULL で終端されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_watchdog_register_callback（新規登録時）、wd_worker（リスト走査およびコールバック実行時）。
  • **呼び出し先**: 登録された各コールバック関数。
  • 詳細は modules/core/mod_watchdog.c 内の各関数（wd_worker 等）がこの構造体を走査・参照しています。

関連データ構造

  • ap_watchdog_t: このリストを保持する監視インスタンスのメイン構造体。
  • apr_pool_t: メモリ管理に使用されるプール。

失敗モードと境界条件

  • リスト走査中にコールバック関数内で長時間ブロックが発生すると、同一インスタンス内の後続コールバックの実行が遅延します。
  • apr_palloc によるメモリ確保失敗時は、システム全体の動作に影響を及ぼす可能性があります。

保守担当者向けメモ

  • 本構造体は mod_watchdog.c 内で定義されており、外部モジュールからは直接操作せず、提供される API を介して利用してください。
  • リストの走査は wd_worker のメインループ内で行われるため、コールバックの追加・削除時にはスレッドセーフな操作に留意してください。
