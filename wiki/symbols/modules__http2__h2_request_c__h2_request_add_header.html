h2_request_add_header

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:137

目的

HTTP/2 ヘッダーフレームの各行を解析し、擬似ヘッダーを h2_request 構造体のフィールドへ格納、または標準ヘッダーを req->headers テーブルへ追加します。

シグネチャまたは形状

apr_status_t h2_request_add_header(h2_request *req, apr_pool_t *pool,
                                   const char *name, size_t nlen,
                                   const char *value, size_t vlen,
                                   struct h2_hd_scratch *scratch,
                                   int *pwas_added)

入力と出力

  • **入力**:
  • req: 操作対象の h2_request オブジェクト。
  • pool: メモリ割り当て用プール。
  • name, value: ヘッダー名および値の文字列（長さ nlen, vlen）。
  • scratch: ヘッダー処理用スクラッチ領域。
  • **出力**:
  • pwas_added: ヘッダーがテーブルに追加された場合に 1 が設定される（擬似ヘッダーの場合は 0）。
  • 戻り値: 成功時は APR_SUCCESS、エラー時は APR_EGENERAL 等のステータスコード。

前提条件と不変条件

  • 擬似ヘッダー（: で始まる名前）は、リクエストのヘッダー処理の先頭で受信される必要があります。
  • req->headers が空でない状態で擬似ヘッダーが受信された場合、不正な順序としてエラーを返します。

アルゴリズムまたは振る舞い

1. nlen が 0 以下であれば即座に終了します。 2. name[0] == ':' の場合、擬似ヘッダーとして以下のいずれかにマッピングします。

  • :method, :scheme, :path, :authority, :protocol
  • 一致するものは req の各フィールドに apr_pstrndup でコピーされます。
  • 未知の擬似ヘッダーは警告ログを出力し無視します。

    • 3. 擬似ヘッダー以外の場合、h2_req_add_header を呼び出し、req->headers テーブルへ追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_stream.c:787
  • **呼び出し先**: h2_req_add_header（内部ヘルパー関数）

関連データ構造

  • h2_request: HTTP/2 リクエスト表現。
  • apr_table_t: req->headers として保持されるヘッダーテーブル。

失敗モードと境界条件

  • **境界条件**: 未知の擬似ヘッダーを受信した場合、先頭 31 文字をログに記録し、処理を継続します。

保守担当者向けメモ

  • 本関数は h2_request.h で宣言されています。
  • 擬似ヘッダーの順序制約は RFC 7540 8.1.2.3 に準拠しています。
  • メモリ管理は apr_pool_t に依存しているため、呼び出し側で適切なライフサイクルのプールを渡す必要があります。
