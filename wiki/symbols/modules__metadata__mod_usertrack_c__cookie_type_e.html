cookie_type_e

定義

  • 種別: enum
  • 定義箇所: modules/metadata/mod_usertrack.c:75

目的

HTTPレスポンスで送信するCookieの形式（仕様）を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    CT_UNSET,
    CT_NETSCAPE,
    CT_COOKIE,
    CT_COOKIE2
} cookie_type_e;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: CT_UNSET（未設定）、CT_NETSCAPE（Netscape形式）、CT_COOKIE（RFC 2109形式）、CT_COOKIE2（RFC 2965形式）のいずれかの値を保持します。

前提条件と不変条件

  • mod_usertrack モジュール内でのCookie生成および設定管理において、Cookieの出力形式を決定する際の型として利用されます。

アルゴリズムまたは振る舞い

  • 設定ディレクティブ等を通じて選択されたCookieの仕様を保持し、後続のCookie生成処理においてどの形式で Set-Cookie ヘッダーを構築するかを決定するフラグとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/metadata/mod_usertrack.c 内の cookie_dir_rec 構造体（84行目）でメンバ変数 style として参照されています。
  • 呼び出し先: 該当なし。

関連データ構造

  • cookie_dir_rec: ディレクトリごとの設定を保持する構造体であり、本列挙型をメンバとして含みます。

失敗モードと境界条件

  • CT_UNSET は、設定が明示的に行われていない初期状態を示します。

保守担当者向けメモ

  • 本列挙型は mod_usertrack におけるCookieの出力スタイルを規定する重要な型です。新しいCookie仕様に対応する際は、この列挙型に値を追加し、それを利用する生成ロジックを更新する必要があります。
