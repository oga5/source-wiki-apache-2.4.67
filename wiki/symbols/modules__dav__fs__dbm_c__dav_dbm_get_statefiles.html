dav_dbm_get_statefiles

定義

  • 種別: function
  • 定義箇所: modules/dav/fs/dbm.c:78

目的

指定されたファイル名に基づき、WebDAV プロパティを格納する DBM データベースの物理ファイル名（状態ファイル）を取得します。

シグネチャまたは形状

void dav_dbm_get_statefiles(apr_pool_t *p, const char *fname,
                            const char **state1, const char **state2)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • fname: データベースのベースとなるファイル名（NULL の場合はディレクトリ用のデフォルト値が使用される）。
  • **出力**:
  • state1, state2: DBM 実装が使用する実際のファイルパスを格納するためのポインタ。

前提条件と不変条件

  • fname が NULL の場合、内部定数 DAV_FS_STATE_FILE_FOR_DIR が使用されます。
  • 出力される state1 および state2 は、apr_dbm_get_usednames によって生成されるパスを指します。

アルゴリズムまたは振る舞い

1. fname が NULL かどうかを判定し、NULL であればディレクトリ用のデフォルトファイル名に置き換えます。 2. apr_dbm_get_usednames を呼び出し、プラットフォーム固有の DBM 実装が使用するファイル名（通常は .dir や .pag 拡張子を持つファイル）を特定し、引数 state1 および state2 に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/fs/repos.c: 複数の箇所でリソースのプロパティデータベースパスを特定するために呼び出されています。
  • **呼び出し先**:
  • apr_dbm_get_usednames (APR ライブラリ関数)

関連データ構造

  • apr_pool_t: メモリ管理に使用。

失敗モードと境界条件

  • fname が NULL の場合、デフォルトのディレクトリ用ファイル名が適用されるため、エラーにはなりません。
  • apr_dbm_get_usednames の内部的な失敗については、本関数自体はエラーコードを返さないため、呼び出し側で結果の妥当性を確認する必要があります。

保守担当者向けメモ

  • 本関数は repos.c におけるリポジトリ操作の基盤です。
  • repos.c:581 にある通り、本関数の利用に関連する設計上のエラーメッセージが存在するため、呼び出し側の実装には注意が必要です。
