util_ald_cache_insert

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap_cache_mgr.c:467

目的

LDAP キャッシュに新しいエントリを挿入し、必要に応じてキャッシュのパージ（削除）処理を行う。

シグネチャまたは形状

void *util_ald_cache_insert(util_ald_cache_t *cache, void *payload)

入力と出力

  • **入力**:
  • cache: 操作対象のキャッシュ構造体へのポインタ。
  • payload: キャッシュに格納するデータ本体。
  • **出力**:
  • 成功時: キャッシュに格納されたデータ（payload のコピー）へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • cache および payload は NULL であってはならない。
  • キャッシュが満杯の場合、内部で util_ald_cache_purge が呼び出される。
  • 挿入されるデータは、キャッシュ構造体内の copy 関数ポインタを通じてキャッシュ専用メモリ領域に複製される。

アルゴリズムまたは振る舞い

1. **容量確認**: numentries が maxentries に達している場合、util_ald_cache_purge を実行する。パージ後も上限に達している場合はエラーを返す。 2. **メモリ確保**: util_ald_alloc を使用して util_cache_node_t を確保する。確保失敗時はパージを試行し、再確保を試みる。 3. **データ複製**: cache->copy を呼び出し、ペイロードをキャッシュ管理下のメモリへ複製する。 4. **挿入**: ハッシュ値を計算し、該当するバケットの連結リストの先頭にノードを挿入する。 5. **状態更新**: numentries をインクリメントし、fullmark に達した場合は marktime を現在時刻に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ldap/util_ldap.c (複数箇所: dn_compare_cache, compare_cache, search_cache への挿入)
  • modules/ldap/util_ldap_cache_mgr.c:325
  • **呼び出し先**:
  • util_ald_cache_purge
  • util_ald_alloc
  • util_ald_free
  • cache->copy (関数ポインタ)
  • cache->hash (関数ポインタ)
  • ap_log_error

関連データ構造

  • util_ald_cache_t: キャッシュ管理構造体。
  • util_cache_node_t: キャッシュ内の各エントリを保持する連結リストノード。

失敗モードと境界条件

  • **メモリ不足**: util_ald_alloc または cache->copy が失敗した場合、エラーログを出力し NULL を返す。
  • **キャッシュ溢れ**: パージ処理後も numentries >= maxentries の状態が続く場合、挿入を中断する。
  • **共有メモリ制約**: 共有メモリ使用時は、ポインタの直接代入ではなく apr_rmm 経由の管理が必要（実装上の注意点）。

保守担当者向けメモ

  • 本関数はキャッシュサイズ制限に達した際、強制的なパージを行う設計となっている。将来的に、より効率的なキャッシュ置換アルゴリズムへのリファクタリングが推奨される。
  • LDAPSharedCacheSize が小さい場合に警告ログが出力されるが、キャッシュ管理ロジック自体の改善が望まれる。
