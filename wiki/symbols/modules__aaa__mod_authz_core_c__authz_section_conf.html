authz_section_conf

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authz_core.c:66

目的

Apache HTTP Server の認可サブシステムにおいて、Require ディレクティブによって構成される認可ロジックの単位（セクション）を保持する。

シグネチャまたは形状

ソースコード上では不完全型として宣言されており、構造体の詳細は modules/aaa/mod_authz_core.c 内で定義されています。

入力と出力

  • 入力: 設定ファイル（Require ディレクティブ）および apr_pool_t によるメモリ割り当て。
  • 出力: 認可ツリーのノードとして、論理演算子やプロバイダー情報を含む構造体インスタンス。

前提条件と不変条件

  • apr_pool_t を使用してメモリが割り当てられる。
  • 認可ツリーのノードとして、子セクション（first）や次の兄弟セクション（next）へのポインタを保持する再帰的な構造を持つ。

アルゴリズムまたは振る舞い

  • 認可ツリーの構成要素として、apply_authz_sections 関数による再帰的な走査対象となる。
  • 設定ファイル解析時に create_default_section 等を通じてインスタンス化され、論理演算（AND/OR/NOT）の階層構造を形成する。

呼び出し元と呼び出し先

  • 呼び出し元: create_default_section（初期化）、設定解析関数（authz_require_alias_section 等）。
  • 呼び出し先: 構造体内部で保持する子セクション（first）および兄弟セクション（next）へのポインタを通じて、ツリー構造を辿る。

関連データ構造

  • authz_core_dir_conf: 本構造体を保持するディレクトリごとの設定構造体。
  • authz_logic_op: 認可セクションが保持する論理演算子。

失敗モードと境界条件

  • 認可ツリーのネストが深い場合、再帰的な走査（apply_authz_sections）によりスタック消費が発生する可能性がある。
  • 否定のみのセクションなど、論理的に矛盾した設定は authz_core_check_config による検証フェーズで検出される。

保守担当者向けメモ

  • 本構造体は mod_authz_core の認可ロジックの中核であるため、変更時は apply_authz_sections の再帰処理への影響を考慮すること。
  • メモリ管理は apr_pool_t に依存しているため、リクエスト処理中のライフサイクルに注意が必要。
