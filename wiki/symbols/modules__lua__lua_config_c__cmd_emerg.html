cmd_emerg

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:176

目的

Lua スクリプトから Apache のログシステムに対し、APLOG_EMERG レベルでメッセージを出力するためのインターフェースを提供します。

シグネチャまたは形状

static int cmd_emerg(lua_State *L)

入力と出力

  • **入力**: lua_State *L (Lua 仮想マシンの状態)。Lua スタックにはログ出力対象のメッセージやコンテキスト情報が含まれることが想定されます。
  • **出力**: int (Lua への戻り値の数)。内部で呼び出す cmd_log_at の戻り値をそのまま返します。

前提条件と不変条件

  • cmd_log_at が正常に動作する環境であること。
  • L は有効な Lua 状態であること。

アルゴリズムまたは振る舞い

1. 引数として受け取った lua_State *L を cmd_log_at 関数に渡します。 2. 第2引数として APLOG_EMERG 定数を指定し、緊急レベルでのログ出力を実行します。 3. cmd_log_at の実行結果を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_config.c:257 にある cmd_methods テーブル内で、Lua からのメソッド呼び出し用として参照されています。
  • **呼び出し先**: cmd_log_at (同ファイル内)。

関連データ構造

  • cmd_methods: CommandParameters 用のメソッドテーブルの一部として定義されています。

失敗モードと境界条件

  • cmd_log_at の内部実装に依存します。Lua スタックからソースファイル名や行番号が取得できない場合、ログ出力の内容が制限される可能性があります。

保守担当者向けメモ

  • 本関数は cmd_alert, cmd_crit 等と同様のラッパー関数です。ログレベルの変更が必要な場合を除き、ロジックは cmd_log_at に集約されています。
