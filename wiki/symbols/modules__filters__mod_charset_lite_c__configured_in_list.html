configured_in_list

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_charset_lite.c:287

目的

指定されたフィルタ名が、現在のフィルタチェーン（ap_filter_t リスト）内に既に存在するかを確認する。

シグネチャまたは形状

static int configured_in_list(request_rec *r, const char *filter_name,
                              struct ap_filter_t *filter_list)

入力と出力

  • **入力**:
  • r: リクエストレコード（未使用）。
  • filter_name: 検索対象のフィルタ名（文字列）。
  • filter_list: 検索を開始するフィルタチェーンの先頭ポインタ。
  • **出力**:
  • 1: 指定された名前のフィルタがリスト内に存在する場合。
  • 0: 指定された名前のフィルタがリスト内に存在しない場合。

前提条件と不変条件

  • filter_list は NULL であるか、有効な ap_filter_t 構造体の連結リストを指している必要がある。
  • filter_name は比較可能な有効な文字列であること。

アルゴリズムまたは振る舞い

1. filter_list を先頭から順に走査する。 2. 各フィルタの frec->name と filter_name を strcasecmp で大文字小文字を区別せずに比較する。 3. 一致するフィルタが見つかった時点で 1 を返し終了する。 4. リストの末尾（NULL）に到達した場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_charset_lite.c:303 (入力フィルタの確認)
  • modules/filters/mod_charset_lite.c:308 (出力フィルタの確認)
  • **呼び出し先**:
  • strcasecmp (標準Cライブラリ)

関連データ構造

  • ap_filter_t: フィルタチェーンを構成する構造体。
  • filter_rec_t (filter->frec): フィルタの定義情報（名前を含む）。

失敗モードと境界条件

  • filter_list が NULL の場合、ループに入らず即座に 0 を返す。
  • filter_name が NULL の場合、strcasecmp の挙動に依存する（セグメンテーション違反の可能性がある）。

保守担当者向けメモ

  • 本関数は mod_charset_lite 内でのフィルタ重複登録を防ぐためのユーティリティとして機能している。
  • request_rec *r 引数は現在の実装では使用されていないが、将来的な拡張やログ出力のために保持されている可能性がある。
