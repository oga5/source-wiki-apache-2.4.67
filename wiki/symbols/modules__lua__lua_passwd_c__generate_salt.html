generate_salt

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_passwd.c:47

目的

Base64エンコードされたランダムなソルト文字列を生成し、指定されたバッファに格納します。

シグネチャまたは形状

static int generate_salt(char *s, size_t size, const char **errstr, apr_pool_t *pool)

入力と出力

  • **入力**:
  • s: 生成されたソルトを格納する文字配列バッファ。
  • size: 生成するソルトの長さ（文字数）。
  • pool: メモリ割り当てに使用するAPRメモリプール。
  • **出力**:
  • s: ヌル終端されたソルト文字列が書き込まれる。
  • errstr: 失敗時にエラーメッセージが格納される。
  • 戻り値: 成功時は 0、失敗時は ERR_RANDOM を返す。

前提条件と不変条件

  • size に基づいて計算される必要なバイト数が、内部バッファ rnd[32] のサイズを超えてはなりません。
  • apr_generate_random_bytes が正常に乱数を取得できる必要があります。

アルゴリズムまたは振る舞い

1. size から必要なバイト数 n を計算します。 2. n が 32 バイトを超える場合、エラーを返します。 3. apr_generate_random_bytes を呼び出し、乱数を生成します。 4. 生成されたバイト列を itoa64 テーブル（./0-9A-Za-z）を用いて6ビットずつ抽出し、Base64形式の文字列に変換します。 5. 変換後の文字列を s に書き込み、最後にヌル文字を付与します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_passwd.c 内の関数（mk_password_hash 等）から呼び出されます。
  • support/passwd_common.c からも参照されています。
  • **呼び出し先**:
  • apr_generate_random_bytes (APRライブラリ)
  • apr_psprintf (APRライブラリ)

関連データ構造

  • passwd_ctx: 本関数が生成したソルトを利用するコンテキスト構造体。

失敗モードと境界条件

  • **バッファ不足**: size が大きすぎて内部バッファ（32バイト）に収まらない場合、ERR_RANDOM を返しエラーメッセージを設定します。
  • **乱数生成失敗**: apr_generate_random_bytes が失敗した場合、ERR_RANDOM を返しエラーメッセージを設定します。

保守担当者向けメモ

  • 本関数は static であり、lua_passwd.c および passwd_common.c で利用される内部ユーティリティです。
  • itoa64 テーブルは、APRのハッシュ生成関数と互換性のある形式で定義されています。変更時はハッシュ生成アルゴリズムとの整合性に注意してください。
