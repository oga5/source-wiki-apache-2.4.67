response_is_html

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_autoindex.c:156

目的

指定されたリクエストのコンテンツタイプが HTML 形式（text/html または application/xhtml+xml）であるかどうかを判定します。

シグネチャまたは形状

static APR_INLINE int response_is_html(request_rec *r)

入力と出力

  • **入力**: request_rec *r （判定対象のリクエスト構造体）
  • **出力**: int （HTML 形式であれば非ゼロ、それ以外であれば 0 を返す）

前提条件と不変条件

  • r->content_type が有効な文字列ポインタであることを前提とします。
  • ap_field_noparam を使用してコンテンツタイプからパラメータ（charset 等）を除去した上で比較を行います。

アルゴリズムまたは振る舞い

1. r->content_type から ap_field_noparam を用いて MIME タイプ部分のみを抽出します。 2. 抽出された文字列を ap_cstr_casecmp により大文字小文字を区別せずに "text/html" および "application/xhtml+xml" と比較します。 3. いずれかと一致する場合に真（非ゼロ）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_autoindex.c:1037
  • modules/generators/mod_autoindex.c:1155
  • modules/generators/mod_autoindex.c:1205
  • **呼び出し先**:
  • ap_field_noparam (Apache API)
  • ap_cstr_casecmp (Apache API)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • r->content_type が NULL の場合、ap_field_noparam の挙動に依存します。
  • コンテンツタイプにパラメータが含まれている場合でも、ap_field_noparam により正しく無視されます。

保守担当者向けメモ

  • 本関数は APR_INLINE として定義されており、ヘッダー解析のループ内など頻繁に呼び出される箇所でのオーバーヘッドを最小限に抑える設計となっています。
