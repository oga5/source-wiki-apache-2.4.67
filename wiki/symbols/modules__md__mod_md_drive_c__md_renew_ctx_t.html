md_renew_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/md/mod_md_drive.c:63

目的

Apache HTTP Server の mod_md における、マネージドドメイン（MD）の証明書更新プロセス全体の状態を保持および管理するためのコンテキスト構造体です。

シグネチャまたは形状

struct md_renew_ctx_t {
    apr_pool_t *p;
    server_rec *s;
    md_mod_conf_t *mc;
    ap_watchdog_t *watchdog;
    apr_array_header_t *jobs;
};

入力と出力

本構造体は関数間で受け渡されるコンテキストとして機能します。

  • **入力**: 初期化時に apr_pool_t や server_rec 等の実行環境情報が格納されます。
  • **出力**: jobs メンバを通じて、現在スケジュールされている更新ジョブのリストを保持します。

前提条件と不変条件

  • mod_watchdog を介した非同期実行環境下で使用されることを前提としています。
  • 保持する apr_pool_t は、コンテキストの生存期間中有効である必要があります。

アルゴリズムまたは振る舞い

  • mod_watchdog のコールバック関数（run_watchdog）において、現在の更新ジョブの状態を追跡するために使用されます。
  • process_drive_job 関数に渡され、個別の MD 更新処理における設定情報や実行環境へのアクセスポイントとして機能します。
  • next_run_default 関数において、次回の更新実行タイミングを計算するための基準データとして参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_md_drive.c 内の run_watchdog および関連するジョブ処理関数から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありませんが、process_drive_job や next_run_default 等の関数に引数として渡されます。

関連データ構造

  • md_job_t: jobs 配列内に格納される個別の更新ジョブ情報。
  • md_mod_conf_t: モジュール全体の設定情報。
  • ap_watchdog_t: 非同期タスク実行のための監視ハンドル。

失敗モードと境界条件

  • 構造体内の各ポインタが NULL である場合、参照先の関数でセグメンテーションフォールトが発生する可能性があります。
  • jobs 配列が空の場合、更新処理はスキップされます。

保守担当者向けメモ

  • 本構造体は mod_md_drive.c 内のプライベートな状態管理用です。
  • 構造体のメンバ追加や変更を行う際は、run_watchdog におけるライフサイクル管理と整合性を取る必要があります。
