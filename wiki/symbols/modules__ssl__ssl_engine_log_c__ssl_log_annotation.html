ssl_log_annotation

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_log.c:55

目的

OpenSSL のエラー文字列に基づき、原因究明を補助するための補足説明（アノテーション）を静的配列から検索して返します。

シグネチャまたは形状

static const char *ssl_log_annotation(const char *error)

入力と出力

  • **入力**: error (const char *) - 検索対象となる OpenSSL のエラー文字列。
  • **出力**: const char * - 一致したパターンに対応する補足説明文字列。一致するものがない場合は、静的配列の終端にあるデフォルト値（通常は NULL）を返します。

前提条件と不変条件

  • ssl_log_annotate という名前の静的配列が定義されており、各要素が cpPattern（検索パターン）と cpAnnotation（説明文）を持つ構造体である必要があります。
  • ssl_log_annotate 配列は、cpPattern が NULL である要素を終端として保持している必要があります。

アルゴリズムまたは振る舞い

1. インデックス i を 0 に初期化します。 2. ssl_log_annotate[i].cpPattern が NULL でなく、かつ ap_strcmp_match 関数を用いて error が現在のパターンと一致しない間、インデックス i をインクリメントします。 3. ループ終了後、ssl_log_annotate[i].cpAnnotation を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_log.c:109 (ssl_log_ssl_error 関数内)
  • **呼び出し先**: ap_strcmp_match (Apache HTTP Server の文字列マッチングユーティリティ)

関連データ構造

  • ssl_log_annotate: エラー文字列のパターンとアノテーションのペアを保持する静的配列。

失敗モードと境界条件

  • 一致するパターンが見つからない場合、配列の終端要素が返されます。
  • error 引数が NULL の場合、ap_strcmp_match の挙動に依存します。

保守担当者向けメモ

  • 本関数は ssl_log_ssl_error から呼び出され、OpenSSL エラーキューの処理中に使用されます。
  • 新しいエラーパターンを追加する場合は、ssl_log_annotate 配列を更新してください。
  • ap_strcmp_match はワイルドカードマッチングをサポートしているため、パターン定義時にはこの特性を考慮してください。
