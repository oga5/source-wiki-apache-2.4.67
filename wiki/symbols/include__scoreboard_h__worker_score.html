worker_score

定義

  • 種別: struct
  • 定義箇所: include/scoreboard.h:89

目的

個別のスレッドまたはワーカープロセスの稼働状態および統計情報を保持するための構造体です。共有メモリ上に配置され、サーバーの健全性監視や MPM によるプロセス管理に使用されます。

シグネチャまたは形状

struct worker_score {
    /* 内部フィールドは実装依存（pid, generation, アクセス統計、リクエスト情報等を含む） */
};

※詳細は include/scoreboard.h の定義を参照してください。

入力と出力

  • **入力**: MPM やプロトコルモジュールが ap_update_child_status 等の関数を介して、本構造体の各フィールドを更新します。
  • **出力**: mod_status 等の監視モジュールが ap_get_scoreboard_worker 等を介して、共有メモリ上の本構造体を参照し、統計情報を出力します。

前提条件と不変条件

  • 共有メモリ（apr_shm）上に配置されるため、アクセスには適切な同期制御が必要です。
  • client や request フィールドはサイズが固定されており、オーバーフローを防ぐ設計が求められます。
  • 一部のフィールド（client, request）は現在非推奨（DEPRECATED）扱いとなっています。

アルゴリズムまたは振る舞い

  • サーバーのライフサイクルを通じて、ワーカーのステータス（SERVER_READY, SERVER_BUSY_READ 等）やリクエスト処理時間、転送量などを追跡します。
  • MPM がプロセスを生成・管理する際、この構造体を通じて状態のレジストリとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元（参照・更新）**:
  • server/scoreboard.c (実装および管理)
  • modules/generators/mod_status.c (情報取得)
  • 各 MPM 実装 (状態更新)
  • **呼び出し先**:
  • 本シンボルはデータ構造であり、関数ではないため「呼び出し先」は存在しません。

関連データ構造

  • scoreboard: 本構造体を保持するメインの管理構造体。
  • ap_sb_handle_t: ワーカーを識別するためのハンドル。
  • process_score: プロセス単位の管理情報。

失敗モードと境界条件

  • 共有メモリの競合: 頻繁な更新が発生するため、ロックの競合が発生する可能性があります。
  • メモリ制限: 文字列フィールドの固定サイズを超過するデータは切り詰められる可能性があります。

保守担当者向けメモ

  • 本構造体は include/ap_mmn.h で定義されるモジュールマジックナンバー（MMN）に関連する変更履歴を持ちます。構造を変更する際は MMN の更新を検討してください。
  • client や request フィールドは将来的に変更される可能性があるため、利用には注意が必要です。
