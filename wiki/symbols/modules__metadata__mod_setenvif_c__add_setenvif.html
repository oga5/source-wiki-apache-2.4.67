add_setenvif

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_setenvif.c:404

目的

Apache 設定ファイル内の SetEnvIf および SetEnvIfNoCase ディレクティブを解析し、指定されたヘッダ名と条件式を抽出して、内部的な設定登録処理へ引き渡す。

シグネチャまたは形状

static const char *add_setenvif(cmd_parms *cmd, void *mconfig, const char *args)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • mconfig: 現在のディレクトリまたはサーバーの設定レコード。
  • args: ディレクティブに続く引数文字列（ヘッダ名および条件式）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • args から抽出されるヘッダ名が空である場合、エラーメッセージを返す。
  • cmd->pool を使用してメモリ割り当てを行う。

アルゴリズムまたは振る舞い

1. ap_getword_conf を使用して、args から最初のトークン（ヘッダ名）を抽出する。 2. 抽出したヘッダ名が空であれば、エラーメッセージを生成して終了する。 3. 抽出したヘッダ名と残りの args を引数として add_setenvif_core を呼び出し、その戻り値をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_setenvif.c:474 (AP_INIT_RAW_ARGS による "SetEnvIf" の登録)
  • modules/metadata/mod_setenvif.c:476 (AP_INIT_RAW_ARGS による "SetEnvIfNoCase" の登録)
  • **呼び出し先**:
  • add_setenvif_core (設定のコンパイルと登録)

関連データ構造

  • cmd_parms: 設定解析用の Apache 構造体。
  • sei_cfg_rec: add_setenvif_core を通じて更新される設定コンテナ。

失敗モードと境界条件

  • ヘッダ名が指定されていない場合、apr_pstrcat を用いてエラーメッセージを生成し、処理を中断する。

保守担当者向けメモ

  • 本関数はディレクティブの解析層であり、実際の正規表現コンパイルやデータ構造への格納は add_setenvif_core に委譲されている。
  • SetEnvIfNoCase の場合、cmd->info を通じて渡されるフラグが add_setenvif_core 内で処理される点に注意すること。
