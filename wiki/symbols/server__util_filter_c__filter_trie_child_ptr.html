filter_trie_child_ptr

定義

  • 種別: struct
  • 定義箇所: server/util_filter.c:55

目的

フィルタ名を管理するトライ木において、各ノードから子ノードへのリンクを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    int c;
    filter_trie_child_ptr *child;
} filter_trie_child_ptr;

入力と出力

  • c: 子ノードへ続く文字（文字コード）。
  • child: 対応する子ノード（filter_trie_node 型）へのポインタ。

前提条件と不変条件

  • 本構造体は、トライ木のノードが保持する子ノード配列の要素として使用されます。
  • 配列のメモリ確保は apr_palloc を介して行われます。

アルゴリズムまたは振る舞い

  • フィルタ名の検索時、現在のノードから特定の文字 c に一致する filter_trie_child_ptr を走査し、見つかった場合に child ポインタを辿ることでトライ木を降下します。

呼び出し元と呼び出し先

  • 呼び出し元: server/util_filter.c 内のトライ木操作関数群（register_filter 等）が、ノードの children 配列を操作する際に本構造体を使用します。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • filter_trie_node: トライ木のノード本体。本構造体の child メンバが指し示す先です。

失敗モードと境界条件

  • メモリ不足時: apr_palloc が失敗した場合、Apache HTTP Server の標準的なエラー処理に従いプロセスが終了する可能性があります。
  • 検索失敗: 指定された文字 c に対応する子ノードが存在しない場合、検索処理は終了します。

保守担当者向けメモ

  • 本構造体は filter_trie_node と相互再帰的な定義関係にあります。
  • フィルタ名のトライ木は registered_output_filters および registered_input_filters を起点として構築されます。
