dav_new_error_tag

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util.c:51

目的

WebDAV エラー情報を保持する dav_error 構造体を生成し、XML 名前空間とタグ名による詳細なエラー識別情報を付与します。

シグネチャまたは形状

dav_error* dav_new_error_tag(apr_pool_t *p, int status, int error_id, 
                             apr_status_t aprerr, const char *desc, 
                             const char *namespace, const char *tagname)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • status: HTTP ステータスコード。
  • error_id: DAV 固有のエラー識別子。
  • aprerr: APR エラーコード。
  • desc: エラーの説明文字列。
  • namespace: エラータグに関連付けられた XML 名前空間。
  • tagname: エラー詳細を示す XML タグ名。
  • **出力**:
  • 生成された dav_error 構造体へのポインタ。

前提条件と不変条件

  • p は有効な apr_pool_t である必要があります。
  • 生成された dav_error 構造体のメモリは p から割り当てられます。

アルゴリズムまたは振る舞い

1. dav_new_error を呼び出し、基本となる dav_error 構造体を生成します。 2. 引数として受け取った namespace および tagname を、生成された構造体の対応するメンバに代入します。 3. 構築済みの dav_error 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: dav_new_error (同モジュール内)
  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供されていませんが、modules/dav/main/mod_dav.h:150 に宣言が存在します。

関連データ構造

  • dav_error: WebDAV エラー情報を格納する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pool_t の割り当てメカニズムによりアボートが発生する可能性があります。
  • namespace や tagname に NULL を渡した場合の挙動は、構造体メンバへの代入に依存します。

保守担当者向けメモ

  • 本関数は、クライアントに対してより詳細なエラー情報（XML 形式の error 要素など）を返す必要がある場合に利用されます。
  • dav_new_error のラッパーであるため、基本エラー生成ロジックに変更がある場合は dav_new_error を確認してください。
