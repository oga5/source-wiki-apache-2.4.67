rotate_config_t

定義

  • 種別: struct (typedef)
  • 定義箇所: support/rotatelogs.c:55

目的

rotatelogs ユーティリティにおける実行時の設定情報を保持するための構造体です。コマンドライン引数から解析されたローテーションの閾値、ファイルパス、および動作フラグを格納します。

シグネチャまたは形状

typedef struct rotate_config rotate_config_t;

※構造体の内部メンバ定義は提供されたソース抜粋には含まれていません。

入力と出力

  • **入力**: main 関数において apr_getopt を通じて解析されたコマンドライン引数が、この構造体に格納されます。
  • **出力**: ログローテーションの判定ロジック（checkRotate）や、ファイル操作ロジック（doRotate）において、設定値として参照されます。

前提条件と不変条件

  • rotatelogs の実行期間中、設定値は基本的に不変として扱われます。
  • 構造体へのポインタは、ログローテーションに関連する各関数へ引数として渡されます。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、ロジックは持ちません。この構造体を参照する関数群（checkRotate, doRotate, get_now 等）が、保持された設定値に基づいてログファイルの切り替えや時刻計算を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数にて静的変数として宣言・初期化されます。
  • **呼び出し先**: 以下の関数が本構造体を引数として受け取ります。
  • get_now
  • close_logfile
  • dumpConfig
  • checkRotate
  • doRotate
  • get_time_or_size

関連データ構造

  • rotate_status_t: 実行時の動的な状態（現在のファイルハンドル、プール、ローテーション理由など）を保持する対となる構造体です。

失敗モードと境界条件

  • 構造体へのポインタが NULL の状態で参照された場合の挙動は、各関数の実装に依存します（ソース抜粋からは詳細不明）。

保守担当者向けメモ

  • 本構造体のメンバ定義は support/rotatelogs.c 内の非公開領域にあるため、構造を変更する場合は同ファイル内の関連関数（dumpConfig 等）への影響を確認してください。
  • rotate_config_t は rotate_config 構造体のエイリアスです。
