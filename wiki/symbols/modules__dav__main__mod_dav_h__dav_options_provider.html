dav_options_provider

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:2631

目的

WebDAV の OPTIONS メソッドに対する拡張機能を提供するためのプロバイダーインターフェースを定義します。

シグネチャまたは形状

typedef struct dav_options_provider
{
    dav_error* (*dav_header)(request_rec *r,
                             const dav_resource *resource,
                             apr_text_header *phdr);

    dav_error* (*dav_header)(request_rec *r,
                             const dav_resource *resource,
                             apr_text_header *phdr);

    void *ctx;
} dav_options_provider;

入力と出力

  • dav_header: OPTIONS レスポンスのヘッダー情報を生成します。apr_text_header にテキストを追加することで、レスポンスヘッダーを構築します。
  • dav_method: OPTIONS メソッドの処理ロジックを実行します。
  • 戻り値: いずれの関数も成功時に NULL を返し、失敗時には dav_error 構造体へのポインタを返します。
  • ctx: プロバイダー固有のコンテキストデータへのポインタ。

前提条件と不変条件

  • 各関数ポインタは NULL である可能性があります。呼び出し側は実行前にチェックを行う必要があります。
  • ctx はプロバイダーの実装に応じて適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. OPTIONS リクエスト受信時、登録されたプロバイダーの dav_header が呼び出され、レスポンスヘッダー（Allow や DAV ヘッダー等）が構築されます。 2. 続いて dav_method が呼び出され、メソッド固有の処理が実行されます。 3. 各処理は dav_error を通じてエラーを伝播させます。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/main/mod_dav.c 内の OPTIONS メソッドハンドラから参照されています。
  • 呼び出し先: プロバイダーの実装に依存します（具体的な実装コードは本抜粋には含まれていません）。
  • 登録・取得: dav_get_options_providers 関数（modules/dav/main/providers.c）を介して取得されます。

関連データ構造

  • dav_resource: WebDAV リソースの抽象表現。
  • request_rec: Apache HTTP Server のリクエスト構造体。
  • apr_text_header: レスポンスヘッダー構築用の APR テキストリスト。
  • dav_error: エラー状態を保持するスタック構造体。

失敗モードと境界条件

  • 各フック関数が NULL 以外の dav_error を返した場合、呼び出し側はそれをエラーとして処理し、適切な HTTP ステータスコードを生成します。
  • プロバイダーが未登録または関数ポインタが NULL の場合、呼び出し側は該当する処理をスキップします。

保守担当者向けメモ

  • 本構造体は OPTIONS メソッドの拡張性を確保するためのものです。新しい OPTIONS 関連の機能を追加する場合は、この構造体にフックを追加し、providers.c での登録処理を更新してください。
  • ctx フィールドは、プロバイダーが状態を保持するために使用されます。メモリ管理は関連付けられたリクエストプール（r->pool）のライフサイクルに従うことが推奨されます。
