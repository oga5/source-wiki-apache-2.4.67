register_hooks

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:381

目的

Apache HTTP Server の認可フェーズにおいて、ホストおよびIPベースのアクセス制御プロバイダーを登録し、設定フェーズのフックを構成する。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（モジュール設定用のメモリプール）
  • **出力**: なし（副作用としてフックおよびプロバイダーを登録）

前提条件と不変条件

  • 本関数は Apache モジュールのロード時に呼び出されることを前提とする。
  • 登録される各プロバイダーは AUTHZ_PROVIDER_GROUP に属し、AP_AUTH_INTERNAL_PER_CONF フラグにより設定ごとの内部認可として扱われる。

アルゴリズムまたは振る舞い

1. ap_hook_pre_config を使用して authz_host_pre_config を登録し、設定開始前の初期化を行う。 2. ap_hook_post_config を使用して authz_host_post_config を登録し、設定完了後のクリーンアップを行う。 3. ap_register_auth_provider を呼び出し、以下の4つの認可プロバイダーを登録する:

  • ip: authz_ip_provider を使用。
  • host: authz_host_provider を使用。
  • forward-dns: authz_forward_dns_position を使用。
  • local: authz_local_provider を使用。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール読み込みプロセス（直接の呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • ap_hook_pre_config, ap_hook_post_config
  • ap_register_auth_provider
  • authz_host_pre_config, authz_host_post_config
  • 各プロバイダー構造体 (authz_ip_provider, authz_host_provider, authz_forward_dns_provider, authz_local_provider)

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • authz_provider: 認可プロバイダーのインターフェース定義。

失敗モードと境界条件

  • プロバイダー登録時にメモリ不足が発生した場合、Apache の起動プロセスが中断される可能性がある。

保守担当者向けメモ

  • 本関数は mod_authz_host のエントリーポイントである。プロバイダーの追加や認可ロジックの変更を行う際は、ここで登録されるプロバイダー構造体を確認すること。
  • ツールによる呼び出し元情報（mod_example_1.c 等）は、本モジュール固有の呼び出しではなく、Apache のモジュール開発における一般的なフック登録パターンの例示である。
