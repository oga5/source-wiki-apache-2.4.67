cmd_parms_struct

定義

  • 種別: struct
  • 定義箇所: include/http_config.h:288

目的

Apache HTTP Server の設定ディレクティブ解析時に、ハンドラー関数へコンテキスト情報と実行環境を渡すための構造体です。

シグネチャまたは形状

struct cmd_parms_struct {
    void *info;
    int override;
    int override_opts;
    apr_table_t *override_list;
    apr_int64_t limited;
    apr_array_header_t *limited_xmethods;
    ap_method_list_t *xlimited;
    ap_configfile_t *config_file;
    ap_directive_t *directive;
    apr_pool_t *pool;
    apr_pool_t *temp_pool;
    server_rec *server;
    char *path;
    const command_rec *cmd;
    struct ap_conf_vector_t *context;
    const ap_directive_t *err_directive;
};

入力と出力

  • **入力**: 設定ファイル解析プロセスから、ディレクティブの定義 (command_rec) および現在の解析状態（プール、サーバーコンテキスト、パス情報）が格納されます。
  • **出力**: ハンドラー関数は、この構造体を通じて設定値を読み取り、必要に応じて err_directive にエラー情報を設定して呼び出し元へ通知します。

前提条件と不変条件

  • temp_pool は設定解析中のみ有効であり、最初のリクエストが処理される前に破棄されます。
  • path フィールドは、サーバーコンテキスト（NULL）かディレクトリコンテキスト（非NULL）かを判別するために使用されます。

アルゴリズムまたは振る舞い

設定ファイルの走査中、ap_walk_config はツリーを辿り、各ディレクティブに対応するハンドラーを呼び出します。その際、本構造体に現在の解析コンテキストを詰め込み、引数として渡します。ハンドラーはこれを用いて、設定の継承や検証を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: include/http_config.h:74 にて cmd_parms 型としてエイリアス定義されています。
  • **呼び出し先**: 具体的なハンドラー関数（command_rec 内で定義された関数）に渡されます。
  • **補足**: 呼び出し元の詳細な関数名や、具体的な呼び出し先関数のリストは提供されたソースからは特定できません。

関連データ構造

  • command_rec: ディレクティブの定義。
  • ap_directive_t: 設定ツリーのノード。
  • ap_conf_vector_t: モジュール固有の設定データを保持するベクトル。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • err_directive が設定された場合、解析エラーとして扱われます。
  • path の値は、<Files> や <Location> 等の複雑なマッチングが存在するため、単なるディレクトリパスとして解釈すべきではありません。

保守担当者向けメモ

  • override_list メンバーは include/ap_mmn.h:346 にて追加されました。
  • 本構造体のメンバーは、設定解析のライフサイクル全体で一貫性を保つ必要があります。特に temp_pool の生存期間には注意してください。
