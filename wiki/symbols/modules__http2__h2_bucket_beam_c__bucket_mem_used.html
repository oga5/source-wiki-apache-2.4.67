bucket_mem_used

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:102

目的

指定された apr_bucket が占有するメモリサイズを計算します。HTTP/2 通信におけるバッファ制限の監視に使用されます。

シグネチャまたは形状

apr_off_t bucket_mem_used(apr_bucket *b)

入力と出力

  • **入力**: apr_bucket *b (評価対象のバケット)
  • **出力**: apr_off_t (バケットのデータ長。ファイルまたは MMAP バケットの場合は 0)

前提条件と不変条件

  • 入力されるバケット b は有効なポインタである必要があります。
  • ファイルバケットおよび MMAP バケットは、メモリ使用量の計算対象外（0）として扱われます。

アルゴリズムまたは振る舞い

1. バケットがファイルバケット (APR_BUCKET_IS_FILE) または MMAP バケット (bucket_is_mmap) であるかを確認します。 2. 上記に該当する場合、メモリ使用量として 0 を返します。 3. それ以外の場合、バケットの length フィールドの値を apr_off_t にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_bucket_beam.c:837 にて、バッファサイズの集計処理の一部として呼び出されています。
  • **呼び出し先**: bucket_is_mmap を内部で呼び出します。

関連データ構造

  • apr_bucket: APR が提供するデータバケット構造体。
  • apr_off_t: ファイルサイズやデータ長を表すための型。

失敗モードと境界条件

  • バケットの length が未定義（APR_BUCKET_UNK_LENGTH）である場合、その値がそのまま返される可能性があります。

保守担当者向けメモ

  • 本関数は、メモリ使用量の計算において「ファイルや MMAP はメモリを消費しない（または計算対象外）」という前提に基づいています。この前提が将来的に変更される場合は、本関数のロジックも更新が必要です。
