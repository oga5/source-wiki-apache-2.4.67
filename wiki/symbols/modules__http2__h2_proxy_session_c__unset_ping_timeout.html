unset_ping_timeout

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_proxy_session.c:109

目的

バックエンド接続のソケットタイムアウト設定を、PING 送信前の元の値に復元する。

シグネチャまたは形状

static void unset_ping_timeout(h2_proxy_session *session)

入力と出力

  • **入力**: h2_proxy_session *session （セッション管理構造体へのポインタ）
  • **出力**: なし（副作用としてソケットのタイムアウト設定を更新）

前提条件と不変条件

  • session->save_timeout が -1 でない場合、以前に設定されたタイムアウト値が保存されているとみなす。
  • session->c は有効な接続レコードである必要がある。

アルゴリズムまたは振る舞い

1. session->save_timeout が -1 でないか確認する。 2. ap_get_conn_socket(session->c) を呼び出し、現在の接続ソケットを取得する。 3. ソケットが取得できた場合、apr_socket_timeout_set を使用して、保存されていた session->save_timeout の値をソケットに再設定する。 4. 復元完了後、session->save_timeout を -1 にリセットし、復元済みであることを示す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_session.c:129
  • modules/http2/h2_proxy_session.c:139
  • **呼び出し先**:
  • ap_get_conn_socket (Apache API)
  • apr_socket_timeout_set (APR API)

関連データ構造

  • h2_proxy_session: セッション状態およびタイムアウト値を保持する構造体。

失敗モードと境界条件

  • ap_get_conn_socket が NULL を返した場合、タイムアウトの復元は行われない。
  • session->save_timeout が -1 の場合、本関数は何も行わない（二重復元の防止）。

保守担当者向けメモ

  • 本関数は enter_ping_state で一時的に変更されたソケットタイムアウトを元に戻すために使用される。
  • タイムアウト値の管理は h2_proxy_session 構造体の save_timeout フィールドに依存しているため、このフィールドを直接操作する他の関数との競合に注意すること。
