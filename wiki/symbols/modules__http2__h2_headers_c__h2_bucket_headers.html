h2_bucket_headers

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_headers.c:44

目的

Apache HTTP Server のバケットチェーン内で、HTTP/2 レスポンスヘッダー（h2_headers）を保持・伝搬させるためのカスタムバケットデータ構造です。

シグネチャまたは形状

typedef struct {
    apr_bucket_refcount refcount;
    h2_headers *headers;
} h2_bucket_headers;

入力と出力

  • **入力**: h2_headers 構造体へのポインタを保持します。
  • **出力**: バケットチェーンを通じて、後続のフィルタやモジュールに対してヘッダー情報を提供します。

前提条件と不変条件

  • refcount フィールドにより、バケットの生存期間が管理されます。
  • h2_headers 構造体は、生成時に指定された apr_pool_t の生存期間に依存します。
  • AP_HAS_RESPONSE_BUCKETS が定義されている環境では、本構造体は使用されません。

アルゴリズムまたは振る舞い

  • **カプセル化**: HTTP/2 特有のヘッダー情報を apr_bucket のデータ領域に格納します。
  • **ライフサイクル管理**: bucket_destroy 等の操作により、参照カウントがゼロになった際に h2_headers とバケットメモリが解放されます。
  • **読み取り**: bucket_read は意図的に空（str = NULL）として実装されており、メタデータとしての保持に特化しています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • h2_bucket_headers_create / h2_bucket_headers_make (生成時)
  • h2_bucket_headers_get (アクセサ)
  • h2_bucket_headers_clone (複製時)
  • h2_bucket_headers_headers_length (サイズ計算時)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • h2_headers: ヘッダー情報（ステータス、テーブル、ノート）を保持するメイン構造体。
  • apr_bucket_refcount: APR バケットの参照カウント管理用構造体。

失敗モードと境界条件

  • メモリプールが破棄された後に本構造体へアクセスすると、未定義の動作を引き起こします。
  • 参照カウントの不整合が発生した場合、メモリリークまたは二重解放の原因となります。

保守担当者向けメモ

  • 本構造体は H2HEADERS 型バケットの内部表現です。
  • h2_headers_clone を使用する際は、メモリリークを防ぐために適切なプールを指定してください。
  • h2_headers_length はテーブル内の全ヘッダーを走査するため、ヘッダー数が多い場合はパフォーマンスへの影響を考慮してください。
