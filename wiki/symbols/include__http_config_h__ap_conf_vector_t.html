ap_conf_vector_t

定義

  • 種別: struct
  • 定義箇所: include/http_config.h:509

目的

Apache HTTP Server において、各モジュール固有の設定データを保持・管理するための不透明な（opaque）ベクトル型です。

シグネチャまたは形状

typedef struct ap_conf_vector_t ap_conf_vector_t;

構造体の内部定義は公開されておらず、ポインタを介して操作されます。

入力と出力

  • **入力**: ap_create_request_config 等の生成関数に apr_pool_t が渡され、メモリが割り当てられます。
  • **出力**: ap_get_module_config を通じて、特定のモジュールに関連付けられた設定データ（void *）が返されます。

前提条件と不変条件

  • 設定データは主に pconf（設定プール）に割り当てられます。
  • ap_get_module_config はインデックスの範囲外アクセスに対する保護を提供しません。

アルゴリズムまたは振る舞い

  • **設定の保持**: サーバー、接続、リクエストごとの設定コンテキストを保持します。
  • **アクセス**: ap_get_module_config および ap_set_module_config を使用して、モジュール単位でデータへの読み書きを行います。
  • **マージ**: ap_merge_per_dir_configs により、親コンテキストから子コンテキストへ設定を継承・統合します。

呼び出し元と呼び出し先

  • **呼び出し元（利用箇所）**:
  • ap_get_module_config / ap_set_module_config
  • ap_create_request_config / ap_create_per_dir_config / ap_create_conn_config
  • ap_merge_per_dir_configs
  • ap_parse_htaccess
  • **呼び出し先**:
  • 提供されたソース情報には、本構造体が内部で呼び出す具体的な関数や、本構造体を呼び出す具体的なモジュール実装の詳細は含まれていません。

関連データ構造

  • module: 設定ベクトルとモジュールを紐付けるために使用されます。
  • apr_pool_t: 設定ベクトルのメモリ管理に使用されます。

失敗モードと境界条件

  • インデックスの範囲外アクセス（ap_get_module_config 使用時）は未定義の動作を引き起こします。
  • メモリ確保失敗時は、apr_pool_t の動作に従いエラーが発生します。

保守担当者向けメモ

  • 本構造体は不透明型として設計されており、直接のメンバ操作は避けてください。
  • 設定の継承やマージのロジックは server/config.c 等の実装側に依存しています。
