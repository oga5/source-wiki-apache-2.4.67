seclisten_rec

定義

  • 種別: struct
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:104

目的

NetWare 環境において、SecureListen ディレクティブで指定されたセキュアなリスニングポートの定義を保持するための構造体です。

シグネチャまたは形状

ソースコード上では不完全型として宣言されており、リンクリストのノードとして機能します。

  • next: 次の seclisten_rec 構造体へのポインタ。

入力と出力

  • 入力: SecureListen ディレクティブの設定値に基づき、apr_pcalloc を介してメモリが割り当てられます。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • Apache のプロセスプール（cmd->server->process->pool）上で管理される必要があります。
  • リンクリストの末尾は NULL で終端されます。

アルゴリズムまたは振る舞い

  • ap_seclisteners 静的変数によって、システム内の全セキュアリスナーのリストが管理されます。
  • find_secure_listener 関数等でリストを走査し、特定のリスナー設定を検索するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: mod_nw_ssl.c 内のリスナー設定処理および初期化処理。
  • 呼び出し先: なし（データ構造体のため）。
  • 注: 具体的な関数呼び出し関係の詳細は、提供されたソース excerpt の範囲外です。

関連データ構造

  • seclistenup_rec: アップグレード可能なリスナーを保持する類似の構造体。
  • NWSSLSrvConfigRec: サーバーごとの SSL 設定を保持する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時: apr_pcalloc が失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性があります。
  • リストの不整合: 複数のリスナーが登録される際、リストのリンク操作が正しく行われない場合、リスナーの検索や初期化が失敗します。
