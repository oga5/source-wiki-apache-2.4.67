ap_process_child_status

定義

  • 種別: function
  • 定義箇所: server/mpm_unix.c:386

目的

子プロセスの終了ステータスを解析し、致命的なエラーや異常終了の判定、およびログ出力を行う。

シグネチャまたは形状

int ap_process_child_status(apr_proc_t *pid, apr_exit_why_e why, int status)

入力と出力

  • pid: 終了したプロセスの apr_proc_t 構造体へのポインタ。
  • why: プロセス終了の理由を示す apr_exit_why_e 列挙型。
  • status: 終了ステータスまたはシグナル番号。
  • 戻り値:
  • APEXIT_CHILDSICK: 子プロセスがリソース不足等で「病気」状態。
  • APEXIT_CHILDFATAL: 致命的なエラーが発生し、サーバー終了が必要。
  • 0: 正常終了、またはログ出力済みのシグナル終了。

前提条件と不変条件

  • why は apr_proc_wait 等から得られる有効な終了理由であること。
  • status は why に応じて終了コードまたはシグナル番号として解釈される。

アルゴリズムまたは振る舞い

1. **終了理由の判定**: APR_PROC_CHECK_EXIT(why) が真の場合、終了コードを評価する。

  • APEXIT_CHILDSICK の場合はそのまま値を返す。
  • APEXIT_CHILDFATAL の場合はエラーログを出力し、値を返す。
  • それ以外は 0 を返す。

    • 2. **シグナル終了の処理**: APR_PROC_CHECK_SIGNALED(why) が真の場合、シグナル番号を解析する。

  • SIGTERM, SIGHUP, AP_SIG_GRACEFUL, SIGKILL の場合はログを出力せず無視する。
  • 上記以外のシグナルの場合、コアダンプの有無を確認し、プロセスIDとシグナル名を含む通知ログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/event/event.c
  • server/mpm/prefork/prefork.c
  • server/mpm/worker/worker.c
  • server/mpm_unix.c (内部のプロセス回収処理)
  • **呼び出し先**:
  • apr_signal_description_get (シグナル名取得)
  • ap_log_error (ログ出力)

関連データ構造

  • apr_proc_t: プロセス情報を保持するAPR構造体。
  • apr_exit_why_e: プロセス終了理由の列挙型。

失敗モードと境界条件

  • APEXIT_CHILDFATAL が返された場合、呼び出し側はサーバーのシャットダウン処理を行う必要がある。
  • 不明なシグナルやコアダンプ発生時は、ap_coredump_dir を参照してログに詳細を出力する。

保守担当者向けメモ

  • 本関数はMPM共通の終了処理ロジックである。特定のMPMで終了時の挙動をカスタマイズしたい場合は、本関数の戻り値に基づいた制御フローを各MPM側で実装すること。
