get_path_param

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:139

目的

URL 文字列から指定された名前のパスパラメータ値を抽出する。主にスティッキーセッションのルート情報を取得するために使用される。

シグネチャまたは形状

static char *get_path_param(apr_pool_t *pool, char *url, const char *name, int scolon_sep)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する APR プール。
  • url: 検索対象の URL 文字列。
  • name: 検索するパラメータ名。
  • scolon_sep: セミコロン（;）を区切り文字として扱うかどうかのフラグ。
  • **出力**:
  • 抽出されたパラメータ値（文字列）。見つからない場合は NULL を返す。

前提条件と不変条件

  • url および name は有効な文字列であること。
  • 抽出された値は pool から割り当てられたメモリ領域にコピーされる。

アルゴリズムまたは振る舞い

1. scolon_sep が真の場合、区切り文字として ";?&" を使用し、偽の場合は "?&" を使用する。 2. strstr を使用して url 内から name を検索する。 3. name の直後に = が続く箇所を特定する。 4. = の後の値を抽出し、apr_strtok を用いて区切り文字で分割して最初のトークンを返す。 5. 一致しない場合は次の出現箇所を検索し、最後まで見つからなければ NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_balancer.c:295 において、balancer->s->sticky_path をキーとしてルート情報を取得するために呼び出されている。
  • **呼び出し先**: strstr, strlen, apr_pstrdup, apr_strtok（APR ライブラリ関数）。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • proxy_balancer: 呼び出し元で sticky_path や scolonsep の設定値を提供するために使用されている。

失敗モードと境界条件

  • name が URL 内に存在しない場合、NULL を返す。
  • = が直後に続かない場合、その箇所は無視して検索を継続する。
  • url が空または name が空の場合の挙動は標準的な文字列検索関数に依存する。

保守担当者向けメモ

  • 本関数は mod_proxy_balancer におけるセッション維持のための重要なユーティリティである。
  • apr_strtok を使用しているため、戻り値の文字列は呼び出し元のプール生存期間に依存する。
