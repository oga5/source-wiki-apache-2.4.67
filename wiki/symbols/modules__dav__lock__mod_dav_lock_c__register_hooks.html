register_hooks

定義

  • 種別: function
  • 定義箇所: modules/dav/lock/mod_dav_lock.c:89

目的

Apache HTTP Server のモジュール初期化プロセスにおいて、mod_dav が利用する dav-lock プロバイダーを登録します。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p (モジュール初期化に使用されるメモリプール)
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • ap_register_provider が利用可能であること。
  • dav_hooks_locks_generic が定義済みであり、有効なロック操作フックを保持していること。

アルゴリズムまたは振る舞い

1. ap_register_provider を呼び出します。 2. dav-lock という名前のプロバイダーグループに対し、generic という名称で dav_hooks_locks_generic 構造体を登録します。 3. バージョン番号として "0" を指定します。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュール読み込みプロセス（直接の呼び出し元はソースコード上では明示されていませんが、Apache のモジュール構造に従い、サーバー起動時に自動的に呼び出されます）。
  • 呼び出し先: ap_register_provider (Apache Core API)。

関連データ構造

  • dav_hooks_locks_generic: ロック操作のフック関数群を保持する構造体。

失敗モードと境界条件

  • ap_register_provider が失敗した場合の挙動については、本関数内では明示的なエラーハンドリングを行っていません。

保守担当者向けメモ

  • 本関数は mod_dav_lock モジュールのエントリポイントの一部です。プロバイダーのインターフェースを変更する場合は、locks.c で定義されている dav_hooks_locks_generic の内容と整合性を取る必要があります。
