bio_filter_out_pass

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_io.c:134

目的

bio_filter_out_ctx_t 内のバケットブリゲード (bb) に保持されたデータを、HTTPD のフィルタチェーンを通じて下位フィルタへ送信する。

シグネチャまたは形状

static int bio_filter_out_pass(bio_filter_out_ctx_t *outctx)

入力と出力

  • **入力**: outctx (bio_filter_out_ctx_t*) - 送信対象のバケットブリゲードとフィルタコンテキストを保持する構造体。
  • **出力**: int - 成功時に 1、失敗時に -1 を返す。

前提条件と不変条件

  • outctx->bb（バケットブリゲード）が空であってはならない（AP_DEBUG_ASSERT により検証）。

アルゴリズムまたは振る舞い

1. outctx->bb に格納されたデータを ap_pass_brigade を用いて、現在のフィルタの次段 (outctx->filter_ctx->pOutputFilter->next) へ渡す。 2. ap_pass_brigade の戻り値を outctx->rc に保存する。 3. ap_pass_brigade が成功 (APR_SUCCESS) した場合でも、接続状態 (outctx->c->aborted) を確認し、接続が中断されている場合は APR_ECONNRESET をエラーとして扱う。 4. 最終的なステータスが APR_SUCCESS であれば 1 を、それ以外であれば -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_io.c:162
  • modules/ssl/ssl_engine_io.c:247
  • **呼び出し先**:
  • ap_pass_brigade (HTTPD API)

関連データ構造

  • bio_filter_out_ctx_t: 出力フィルタのコンテキスト。バケットブリゲード (bb) および接続情報 (c) を保持。

失敗モードと境界条件

  • ap_pass_brigade がエラーを返した場合、戻り値は -1 となる。

保守担当者向けメモ

  • 本関数は OpenSSL の BIO から HTTPD のフィルタスタックへのデータ転送の末端を担う。
  • outctx->bb の所有権やライフサイクルについては、呼び出し元が適切に管理していることを前提としている。
