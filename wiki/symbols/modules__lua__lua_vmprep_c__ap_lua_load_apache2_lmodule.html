ap_lua_load_apache2_lmodule

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:145

目的

Lua 仮想マシン（VM）内に apache2 テーブルを生成し、Apache HTTP Server の定数およびサーバー情報を Lua スクリプトから利用可能にするための初期化を行う。

シグネチャまたは形状

void ap_lua_load_apache2_lmodule(lua_State *L)

入力と出力

  • **入力**: lua_State *L（初期化対象の Lua VM 状態）
  • **出力**: なし（副作用として Lua VM のグローバル環境および package.loaded テーブルを更新）

前提条件と不変条件

  • 引数 L は有効な lua_State であること。
  • 実行後、Lua のグローバル環境に apache2 テーブルが定義され、必要な定数が格納される。

アルゴリズムまたは振る舞い

1. package.loaded テーブルに apache2 エントリを作成し、グローバル変数 apache2 に登録する。 2. apache2.version フィールドに ap_get_server_banner() の戻り値を設定する。 3. makeintegerfield マクロを介して、以下の定数を apache2 テーブルに注入する。

  • 処理結果: OK, DECLINED, DONE
  • プロキシ関連: PROXYREQ_NONE, PROXYREQ_PROXY, PROXYREQ_REVERSE, PROXYREQ_RESPONSE
  • 認証関連: AUTHZ_DENIED, AUTHZ_GRANTED, AUTHZ_NEUTRAL, AUTHZ_GENERAL_ERROR, AUTHZ_DENIED_NO_USER
  • その他: HTTP_MOVED_TEMPORARILY

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/mod_lua.c:119
  • **呼び出し先**: ap_get_server_banner() (Apache API), makeintegerfield (マクロ)

関連データ構造

  • lua_State (Lua C API)

失敗モードと境界条件

  • Lua スタックが不足している場合、Lua VM のパニックを引き起こす可能性がある。
  • makeintegerfield で定義されている定数の一部はソースコード内でコメントアウトされており、これらは Lua 側から参照できない。

保守担当者向けメモ

  • 本関数は apache2 モジュールの初期化を担う。HTTP ステータスコード等の定数を追加する場合は、コメントアウトされているリストを参照し、makeintegerfield を追加すること。
  • package.loaded への登録は、Lua の require 機構との整合性を保つために行われている。
