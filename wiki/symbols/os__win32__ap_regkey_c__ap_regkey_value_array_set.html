ap_regkey_value_array_set

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:525

目的

文字列の配列を Windows レジストリの REG_MULTI_SZ 型の値として設定します。

シグネチャまたは形状

apr_status_t ap_regkey_value_array_set(ap_regkey_t *key,
                                       const char *valuename,
                                       int nelts,
                                       const char * const * elts,
                                       apr_pool_t *pool)

入力と出力

  • key: 操作対象のレジストリキー（ap_regkey_t）。
  • valuename: 設定する値の名前。
  • nelts: 配列内の要素数。
  • elts: 文字列配列へのポインタ。
  • pool: メモリ割り当てに使用する APR プール。
  • 戻り値: 成功時は APR_SUCCESS、失敗時はエラーコード（apr_status_t）を返します。

前提条件と不変条件

  • key は有効なレジストリキーハンドルを保持している必要があります。
  • pool はメモリ割り当てのために有効である必要があります。
  • REG_MULTI_SZ の仕様に従い、データは各文字列がヌル終端され、全体が二重のヌル終端で終わる形式に変換されます。

アルゴリズムまたは振る舞い

1. **バッファサイズ計算**: 配列内の全文字列の長さと、各要素および末尾のヌル文字に必要なサイズを計算します。 2. **メモリ確保**: apr_palloc を使用して必要なバッファを確保します。 3. **データ変換**:

  • APR_HAS_UNICODE_FS が有効な場合: apr_conv_utf8_to_ucs2 を使用して UTF-8 から UCS-2 へ変換します。
  • APR_HAS_ANSI_FS が有効な場合: memcpy を使用してバイト列をコピーします。

    • 4. **終端処理**: データの末尾に二重のヌル終端（\0\0）を付与します。 5. **レジストリ書き込み**: ap_regkey_value_raw_set を呼び出し、REG_MULTI_SZ 型としてレジストリへ書き込みます。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/winnt/service.c:931 で確認されています。
  • 呼び出し先: ap_regkey_value_raw_set を呼び出します。また、プラットフォーム設定に応じて apr_conv_utf8_to_ucs2 を呼び出す場合があります。

関連データ構造

  • ap_regkey_t: レジストリキーハンドルとプールを保持するコンテナ。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • apr_conv_utf8_to_ucs2 が失敗した場合、そのエラーコードを返します。
  • nelts が 0 の場合、空の配列として二重のヌル終端のみが書き込まれます。
  • ap_regkey_value_raw_set が失敗した場合、そのエラーコードを返します。

保守担当者向けメモ

  • 本関数は REG_MULTI_SZ 型の書き込みに特化しています。
  • Unicode/ANSI の切り替えはコンパイル時のマクロ（APR_HAS_UNICODE_FS, APR_HAS_ANSI_FS）に依存します。
  • 変換後のバッファは pool から割り当てられるため、呼び出し側で個別に解放する必要はありません。
