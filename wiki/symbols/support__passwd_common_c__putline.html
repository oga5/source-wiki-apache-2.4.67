putline

定義

  • 種別: function
  • 定義箇所: support/passwd_common.c:107

目的

指定されたファイルポインタに対して文字列を書き込み、書き込み失敗時にエラーメッセージを出力してプロセスを終了させるためのユーティリティ関数です。

シグネチャまたは形状

void putline(apr_file_t *f, const char *l)

入力と出力

  • **入力**:
  • f: 書き込み先の apr_file_t 構造体へのポインタ。
  • l: 書き込む対象の文字列（const char *）。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • errfile グローバル変数が apr_file_open_stderr() 等で適切に初期化されている必要があります。
  • f が NULL の場合、関数は何もせず即座に復帰します。

アルゴリズムまたは振る舞い

1. f が NULL であれば処理を終了します。 2. apr_file_puts を使用して文字列 l をファイル f に書き込みます。 3. 書き込み結果（rv）が APR_SUCCESS でない場合:

  • errfile に対してエラーメッセージを出力します。
  • ファイル f をクローズします。
  • ERR_FILEPERM ステータスコードでプロセスを終了（exit）させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/htdigest.c (267行目, 277行目)
  • support/htpasswd.c (436行目, 459行目, 491行目, 508行目)
  • **呼び出し先**:
  • apr_file_puts
  • apr_file_printf
  • apr_file_close
  • exit

関連データ構造

  • apr_file_t: APR（Apache Portable Runtime）のファイルハンドル構造体。

失敗モードと境界条件

  • f が NULL の場合は安全に無視されます。
  • ファイルへの書き込みが失敗した場合（ディスクフルや権限エラー等）、エラーメッセージを出力してプログラムが強制終了します。

保守担当者向けメモ

  • 本関数は errfile グローバル変数に依存しています。使用するツール側でこの変数が正しく設定されていることを確認してください。
  • 書き込み失敗時に exit を呼び出すため、呼び出し側でリソースのクリーンアップが必要な場合は注意が必要です。
