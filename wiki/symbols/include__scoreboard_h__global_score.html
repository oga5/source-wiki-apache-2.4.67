global_score

定義

  • 種別: struct
  • 定義箇所: include/scoreboard.h:123

目的

Apache HTTP Server 全体の稼働統計およびプロセス管理情報を保持する共有メモリ上の構造体です。

シグネチャまたは形状

typedef struct {
    int             server_limit;
    int             thread_limit;
    ap_generation_t running_generation;
    apr_time_t      restart_time;
#ifdef HAVE_TIMES
    struct tms      times;
#endif
} global_score;

入力と出力

  • 本構造体は共有メモリ上に配置され、ap_get_scoreboard_global() を通じてポインタとして取得されます。
  • 直接的な関数引数としての入出力ではなく、サーバーのライフサイクルを通じて状態が更新・参照されます。

前提条件と不変条件

  • 共有メモリ（apr_shm）上に割り当てられる必要があります。
  • server/scoreboard.c 内の SIZE_OF_global_score マクロにより、アライメント調整されたサイズでメモリ確保が行われることが前提です。

アルゴリズムまたは振る舞い

  • サーバーの再起動時刻、スレッド制限、および現在稼働中の世代（running_generation）を追跡します。
  • HAVE_TIMES が定義されている環境では、プロセス統計（struct tms）を保持します。
  • MPM（Multi-Processing Module）がプロセスを管理する際のグローバルな状態レジストリとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元（参照・更新）**:
  • server/scoreboard.c: 共有メモリの初期化および ap_get_scoreboard_global の実装。
  • modules/generators/mod_status.c: サーバー統計情報の表示時に参照。
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • scoreboard: 本構造体を含むメインのスコアボード管理構造体。
  • ap_generation_t: 世代管理に使用される型。
  • struct tms: プロセス時間を記録するためのシステム構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、本構造体へのアクセスはヌルポインタ参照を引き起こす可能性があります。
  • worker_score 等の他のスコアボード構造体と異なり、サーバー全体で単一のインスタンスとして扱われるため、競合制御は scoreboard.c の実装に依存します。

保守担当者向けメモ

  • ap_mmn.h の記録によると、過去に sb_type フィールドが削除されるなどの変更履歴があります。
  • client や request フィールドを持つ worker_score とは異なり、本構造体はサーバー全体のメタデータを管理します。
  • 構造体のサイズ変更は共有メモリレイアウトに影響するため、バイナリ互換性に注意してください。
