sp_reason

定義

  • 種別: enum
  • 定義箇所: modules/mappers/mod_speling.c:118

目的

URLのスペルミスや大文字小文字の不一致を判定する際の、修正品質（類似度のレベル）を定義する。

シグネチャまたは形状

typedef enum {
    SP_IDENTICAL = 0,
    SP_MISCAPITALIZED = 1,
    SP_TRANSPOSITION = 2,
    SP_MISSINGCHAR = 3,
    SP_EXTRACHAR = 4,
    SP_SIMPLETYPO = 5,
    SP_VERYDIFFERENT = 6
} sp_reason;

入力と出力

  • 本シンボル自体は列挙型定数であり、入力や出力を持たない。
  • spdist() 関数の戻り値として使用され、比較対象の文字列間の差異の性質を示す。

前提条件と不変条件

  • 値は 0 から 6 の範囲で定義される。
  • SP_IDENTICAL が最も高い一致度（0）を示し、値が大きくなるほど差異が大きくなる。

アルゴリズムまたは振る舞い

  • spdist() 関数において、2つの文字列を比較した結果に基づき、この列挙値のいずれかが返される。
  • check_speling() 関数内で、この値を用いて候補のソートや、リダイレクトの可否を判断する。

呼び出し元と呼び出し先

  • 呼び出し元: spdist() 関数がこの型を返し、check_speling() 関数がその結果を評価する。
  • 呼び出し先: なし（列挙型定義のため）。
  • 関連する構造体: misspelled_file 構造体の quality メンバとして使用される。

関連データ構造

  • misspelled_file: 候補ファイル名と、その品質（sp_reason）を保持する構造体。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • spdist() は「1文字の誤り」までを許容するロジックであるため、この列挙値の各要素は、その範囲内での差異の分類を意図している。
  • 新たな判定基準を追加する場合は、spdist() のロジックと check_speling() での評価順序を併せて更新する必要がある。
