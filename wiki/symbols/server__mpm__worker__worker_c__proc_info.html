proc_info

定義

  • 種別: struct
  • 定義箇所: server/mpm/worker/worker.c:185

目的

プロセスおよびスレッドに関連する識別情報とソケット記述子を保持し、スレッド生成時やワーカーのコンテキスト管理において情報を引き渡すために使用されます。

シグネチャまたは形状

typedef struct {
    int pid;
    int tid;
    int sd;
} proc_info;

入力と出力

  • **入力**: 構造体の各メンバ（pid, tid, sd）は整数型であり、プロセスID、スレッドID、およびソケット記述子を格納します。
  • **出力**: 本構造体はデータ保持用であり、関数としての戻り値はありません。

前提条件と不変条件

  • ap_malloc を用いて動的にメモリ確保されるケース（worker.c:852 等）があり、その場合は呼び出し側でメモリ管理を行う必要があります。

アルゴリズムまたは振る舞い

  • スレッド生成時やワーカーの初期化時に、プロセスID、スレッドID、ソケット記述子をパッケージ化して保持します。
  • dummy 引数として渡されたポインタを proc_info * にキャストして内部情報にアクセスするパターンが確認されています（worker.c:554, worker.c:744）。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • server/mpm/worker/worker.c: 554, 744, 849, 852 行目にて、スレッドの初期化やコンテキストの受け渡しに使用されています。
  • server/mpm/event/event.c: 1719, 2159, 2299, 2302, 2468, 2491 行目にて、同様の目的で参照されています。
  • **呼び出し先**: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • worker_thread: 本構造体を利用してスレッドのコンテキストを管理する可能性があります。

失敗モードと境界条件

  • メモリ確保失敗: ap_malloc を使用している箇所では、メモリ不足時にプログラムが終了する可能性があります。
  • キャストの安全性: dummy ポインタからキャストして使用する際、ポインタが正しく初期化されていない場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • server/mpm/event/event.c からも参照されているため、変更を加える際は event MPM への影響を考慮してください。
  • pid, tid, sd の各フィールドが、実行環境の OS における型定義（pid_t 等）と整合しているか確認が必要です。
