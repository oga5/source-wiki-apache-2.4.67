dbd_remove

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_dbd.c:339

目的

データベースから特定のセッションデータを削除します。

シグネチャまたは形状

apr_status_t dbd_remove(request_rec *r, const char *key)

入力と出力

  • **入力**:
  • r: 現在のリクエストを指す request_rec 構造体。
  • key: 削除対象のセッションを特定するためのキー文字列。
  • **出力**:
  • apr_status_t: 処理の成功（APR_SUCCESS）または失敗を示すステータスコード。

前提条件と不変条件

  • r->per_dir_config から session_dbd_module の設定が取得可能であること。
  • 設定において deletelabel（削除用SQLクエリのラベル）が定義されていること。
  • dbd_init が成功し、有効なデータベース接続とプリペアドステートメントが取得できること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリ設定 session_dbd_dir_conf を取得します。 2. 設定内に deletelabel が存在するか確認し、未定義の場合は APR_EGENERAL を返してエラーログを出力します。 3. dbd_init を呼び出し、データベース接続と削除用プリペアドステートメントを準備します。 4. apr_dbd_pvbquery を実行し、指定された key をバインドしてデータベースから該当レコードを削除します。 5. クエリ実行結果を返し、成功した場合は APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/session/mod_session_dbd.c:435
  • **呼び出し先**:
  • ap_get_module_config
  • ap_log_rerror
  • dbd_init
  • apr_dbd_pvbquery

関連データ構造

  • request_rec: リクエストのコンテキストとメモリプールを保持。
  • session_dbd_dir_conf: 削除用クエリラベル（deletelabel）を保持する設定構造体。
  • ap_dbd_t: mod_dbd が提供するデータベース接続ハンドル。

失敗モードと境界条件

  • deletelabel が設定されていない場合: APR_EGENERAL を返し、エラーログを出力します。
  • dbd_init が失敗した場合: その戻り値をそのまま返します（エラーログは dbd_init 内で出力済み）。
  • データベースクエリの実行に失敗した場合: apr_dbd_pvbquery のエラーをログ出力し、そのステータスを返します。

保守担当者向けメモ

  • 本関数は mod_dbd のプリペアドステートメント機能に強く依存しています。
  • 削除対象のキーが存在しない場合でも、SQLの実行自体が成功すれば APR_SUCCESS が返されます。
