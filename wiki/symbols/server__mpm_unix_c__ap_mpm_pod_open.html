ap_mpm_pod_open

定義

  • 種別: function
  • 定義箇所: server/mpm_unix.c:447

目的

子プロセスとの通信に使用する「Pipe of Death (POD)」を初期化し、親プロセスから子プロセスへ終了や再起動の合図を送るためのパイプペアを作成します。

シグネチャまたは形状

apr_status_t ap_mpm_pod_open(apr_pool_t *p, ap_pod_t **pod)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t プール。
  • pod: ap_pod_t 構造体へのポインタを格納する出力用引数。
  • **出力**:
  • 戻り値: APR_SUCCESS（成功時）、または apr_file_pipe_create_ex が返すエラーコード。
  • *pod: 初期化された ap_pod_t 構造体。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • pod は書き込み可能なメモリ領域を指していること。
  • 作成されたパイプの読み込み側 (pod_in) は、apr_file_inherit_unset により子プロセスへ継承されない設定となります。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して ap_pod_t 構造体分のメモリを確保します。 2. apr_file_pipe_create_ex を呼び出し、書き込みブロックモードでパイプを作成します。 3. パイプの読み込み側 (pod_in) のタイムアウトを 0 に設定し、非ブロッキング読み込みを可能にします。 4. apr_file_inherit_unset を呼び出し、パイプのファイル記述子が exec 時に子プロセスへ継承されないように設定します。 5. 成功時に APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/prefork/prefork.c (確認済み)
  • **呼び出し先**:
  • apr_palloc
  • apr_file_pipe_create_ex
  • apr_file_pipe_timeout_set
  • apr_file_inherit_unset

関連データ構造

  • ap_pod_t: PODのパイプ（pod_in, pod_out）と関連プールを保持する構造体。

失敗モードと境界条件

  • apr_file_pipe_create_ex が失敗した場合、そのエラーコードをそのまま返します。
  • メモリ不足時には apr_palloc がアボートする可能性があります（APRの挙動に依存）。

保守担当者向けメモ

  • 本関数は Unix 系 MPM におけるプロセス間通信の基盤です。
  • パイプの継承フラグを解除しているため、子プロセス生成時に意図せずパイプが引き継がれることを防いでいます。
