match_data_pt

定義

  • 種別: typedef
  • 定義箇所: server/util_pcre.c:277

目的

PCRE2 ライブラリにおけるマッチング結果を保持するデータ構造 pcre2_match_data へのポインタ型を定義し、HTTPD 内での正規表現マッチング処理を抽象化する。

シグネチャまたは形状

typedef pcre2_match_data* match_data_pt;

入力と出力

  • **入力**: なし（型定義）。
  • **出力**: pcre2_match_data 構造体へのポインタ。

前提条件と不変条件

  • 本型は PCRE2 環境下で使用される。
  • alloc_match_data によって確保されたメモリは、対応する free_match_data によって適切に解放される必要がある。

アルゴリズムまたは振る舞い

  • PCRE2 のネイティブ型 pcre2_match_data* のエイリアスとして機能する。
  • get_match_data 関数において、マッチング用ワークスペースとしてこの型が利用され、スレッドローカルストレージ（TLS）を介した再利用や新規確保が行われる。

呼び出し元と呼び出し先

  • **呼び出し元（利用箇所）**:
  • alloc_match_data (戻り値型)
  • free_match_data (引数型)
  • get_match_data (戻り値型および内部変数)
  • ap_regexec_len (内部変数として利用)
  • **呼び出し先**:
  • pcre2_match_data (PCRE2 ライブラリの構造体)

関連データ構造

  • pcre2_match_data: PCRE2 がマッチング結果を格納するために使用する構造体。
  • ap_regex_t: HTTPD が正規表現を管理するために使用する構造体。

失敗モードと境界条件

  • alloc_match_data がメモリ確保に失敗した場合、本型は NULL を保持する可能性がある。
  • PCRE1 環境下では、ソースコード内の条件分岐により int* として定義されるケースがある（server/util_pcre.c:280）。

保守担当者向けメモ

  • 本型は PCRE2 への依存を隠蔽するための抽象化レイヤーである。PCRE1 と PCRE2 の両方をサポートするビルド構成において、条件コンパイルによる型の差異に注意すること。
  • get_match_data で取得したインスタンスの所有権（解放責任）が呼び出し元にあるか、TLS 管理下にあるかを明確に区別して扱うこと。
