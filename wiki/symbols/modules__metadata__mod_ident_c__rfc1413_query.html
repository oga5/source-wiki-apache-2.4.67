rfc1413_query

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_ident.c:158

目的

リモートホストの IDENT デーモン（RFC 1413）に対してクエリを送信し、応答からユーザー名を取得して conn_rec に格納する。

シグネチャまたは形状

apr_status_t rfc1413_query(apr_socket_t *sock, conn_rec *conn, server_rec *srv)

入力と出力

  • **入力**:
  • sock: 接続済みの apr_socket_t。
  • conn: 接続情報を保持する conn_rec。
  • srv: エラーログ出力に使用する server_rec。
  • **出力**:
  • 戻り値: 成功時は APR_SUCCESS、失敗時はエラーコード（APR_EINVAL 等）。
  • 副作用: 成功時、conn->remote_logname が取得したユーザー名で更新される。

前提条件と不変条件

  • sock はリモートホストの 113 番ポートに対して既に接続が確立されていること。
  • conn->local_addr および conn->client_addr が有効なポート情報を保持していること。
  • 応答は \r\n で終端されている必要がある。

アルゴリズムまたは振る舞い

1. **クエリ送信**: conn から取得したローカルポートとリモートポートを "%hu,%hu\r\n" 形式でバッファに書き込み、ASCII 変換後にソケットへ送信する。送信時は部分的な書き込み（short write）を考慮してループ処理を行う。 2. **応答受信**: ソケットから \012（改行）が検出されるまでデータを読み込む。 3. **パース**: 受信したデータを ASCII 変換し、sscanf を用いて "%hu , %hu : USERID :%*[^:]:%512s" 形式で解析する。 4. **検証**: パース結果が 3 項目であること、およびポート番号が送信時と一致することを確認する。 5. **格納**: ユーザー名末尾の \r を除去し、conn->pool を使用して conn->remote_logname にコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_ident.c 内の関数（ソースコード上の直接的な呼び出し箇所: modules/metadata/mod_ident.c:320）。
  • **呼び出し先**:
  • apr_socket_send, apr_socket_recv (APR ネットワーク I/O)
  • ap_xlate_proto_to_ascii, ap_xlate_proto_from_ascii (プロトコル変換)
  • ap_log_error (エラーログ出力)
  • apr_pstrdup (メモリ割り当て)
  • strchr, memset, strlen, sscanf (標準 C ライブラリ)

関連データ構造

  • conn_rec: 接続情報および remote_logname の格納先。
  • apr_socket_t: 通信に使用するソケット。
  • server_rec: エラーログ出力用。

失敗モードと境界条件

  • **送信失敗**: apr_socket_send がエラーを返した場合、エラーログを出力して終了する。
  • **受信失敗**: apr_socket_recv がエラーを返した場合、または改行が見つかる前にデータが枯渇した場合は APR_EINVAL を返す。
  • **パース失敗**: sscanf のフォーマットに一致しない、またはポート番号が一致しない場合は APR_EINVAL を返す。
  • **バッファ制限**: RFC1413_MAXDATA を超える応答や、ユーザー名が RFC1413_USERLEN (512バイト) を超える場合はパースに失敗する。

保守担当者向けメモ

  • sscanf の %512s は RFC1413_USERLEN に依存している。定数を変更する場合はこのサイズ指定も更新が必要。
  • strchr で \012 を検索しているのは、ASCII/EBCDIC 両環境で改行コードを正しく扱うための設計である。
  • conn->remote_logname はリクエストプール (conn->pool) に割り当てられるため、明示的な解放は不要である。
