session_crypto_encode

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session_crypto.c:483

目的

mod_session のセッションデータを暗号化し、永続化可能な形式に変換します。

シグネチャまたは形状

apr_status_t session_crypto_encode(request_rec *r, session_rec *z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報。
  • z: 暗号化対象のセッションデータ (session_rec)。
  • **出力**:
  • 戻り値: 成功時は OK、失敗時は apr_status_t エラーコード。
  • 副作用: 暗号化に成功した場合、z->encoded が暗号化および Base64 エンコードされた文字列で更新されます。

前提条件と不変条件

  • dconf->passphrases_set が真であり、かつ z->encoded にデータが存在する場合のみ暗号化処理が実行されます。
  • 暗号化には r->server->process->pconf から取得される CRYPTO_KEY コンテキストが使用されます。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、現在のディレクトリ設定 (session_crypto_dir_conf) を取得します。 2. apr_pool_userdata_get により、サーバープロセスから暗号化コンテキストを取得します。 3. encrypt_string を呼び出し、セッション文字列の暗号化、MAC（認証タグ）の付与、および Base64 エンコードを実行します。 4. 暗号化に失敗した場合はエラーログを出力し、そのステータスを返します。 5. 成功した場合は、変換後の文字列を z->encoded に格納し、OK を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_session の session_encode フックとして登録されています (modules/session/mod_session_crypto.c:774)。
  • **呼び出し先**:
  • ap_get_module_config
  • apr_pool_userdata_get
  • encrypt_string
  • ap_log_rerror

関連データ構造

  • session_rec: セッションデータを保持する構造体。
  • session_crypto_dir_conf: ディレクトリ単位の暗号化設定（パスフレーズ等）。
  • request_rec: リクエストごとのコンテキスト。

失敗モードと境界条件

  • encrypt_string がエラーを返した場合、そのステータスがそのまま呼び出し元に伝播されます。
  • z->encoded が空または未設定の場合、暗号化処理はスキップされます。

保守担当者向けメモ

  • 本関数は mod_session のフックとして動作するため、暗号化の失敗はセッションの保存失敗に直結します。
  • メモリ管理はリクエストプール (r->pool) に依存しており、encrypt_string 内での割り当てを含め、リクエスト終了時に解放されます。
