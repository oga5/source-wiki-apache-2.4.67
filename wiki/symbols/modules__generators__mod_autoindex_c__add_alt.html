add_alt

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_autoindex.c:222

目的

Apache HTTP Server の設定ディレクティブ（AddAlt, AddAltByType, AddAltByEncoding）を処理し、ディレクトリ索引におけるファイル名やタイプに対応する代替テキスト（Altテキスト）を構成情報に登録します。

シグネチャまたは形状

static const char *add_alt(cmd_parms *cmd, void *d, const char *alt, const char *to)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。cmd->info には登録タイプ（BY_PATH, BY_TYPE, BY_ENCODING）が格納される。
  • d: 現在のディレクトリ設定構造体 (autoindex_config_rec)。
  • alt: 設定される代替テキスト。
  • to: マッチング対象のパターン（ファイル名、MIMEタイプ、またはエンコーディング）。
  • **出力**: 成功時は NULL を返し、設定エラー時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • 引数 d は autoindex_config_rec 型としてキャスト可能である必要があります。
  • cmd->info の値に応じて、to パラメータに対して特定の正規化処理が行われます。

アルゴリズムまたは振る舞い

1. **条件付き正規化**:

  • cmd->info が BY_PATH の場合、to が **DIRECTORY** であれば ^^DIRECTORY^^ に置換します。
  • cmd->info が BY_ENCODING の場合、to を小文字に変換します。

    • 2. **登録**: push_item 関数を呼び出し、autoindex_config_rec 内の alt_list に、指定されたパターンと代替テキストのペアを格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • mod_autoindex.c 内の AP_INIT_ITERATE2 マクロ経由で、設定ファイル解析時に呼び出されます。
  • mod_autoindex.c:255 にて、内部的に呼び出される箇所があります。
  • **呼び出し先**:
  • push_item (データ構造への追加)
  • apr_pstrdup, ap_str_tolower (文字列操作)

関連データ構造

  • autoindex_config_rec: ディレクトリごとの設定を保持する構造体。
  • item: alt_list を構成する要素。

失敗モードと境界条件

  • cmd->info が未知の値である場合、push_item に渡される引数が意図しない挙動をとる可能性があります。
  • メモリ不足時には apr_pstrdup 等がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は設定フェーズでのみ実行されます。
  • to パラメータの正規化ロジックは、mod_autoindex の他の設定関数（add_icon 等）と整合性を保つ必要があります。
