ap_lua_request_cfg

定義

  • 種別: struct
  • 定義箇所: modules/lua/mod_lua.h:162

目的

HTTP リクエスト単位で保持される、Lua 仮想マシン（VM）および関連するリクエスト詳細情報を管理するための構造体です。

シグネチャまたは形状

typedef struct
{
    mapped_request_details *mapped_request_details;
    apr_hash_t *request_scoped_vms;
} ap_lua_request_cfg;

入力と出力

  • **入力**: リクエスト処理中に apr_palloc を介してメモリが割り当てられ、リクエストコンテキストの一部として初期化されます。
  • **出力**: リクエストのライフサイクル内で、Lua VM の取得やリクエスト詳細の参照先として利用されます。

前提条件と不変条件

  • request_scoped_vms は apr_hash_t 型であり、リクエストスコープで管理される Lua VM を保持するために使用されます。
  • 本構造体はリクエストの生存期間（r->pool）に依存してメモリ管理されます。

アルゴリズムまたは振る舞い

  • リクエスト処理の過程で、Lua スクリプトの実行に必要な VM を request_scoped_vms から検索または生成するために使用されます。
  • mapped_request_details を通じて、現在実行中のリクエストに関連付けられた Lua スクリプトのメタデータや設定情報にアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/mod_lua.c:1995 において、apr_palloc を用いて本構造体のインスタンスが生成されています。
  • **呼び出し先**: 構造体内部のフィールド（mapped_request_details, request_scoped_vms）は、Lua VM の準備やリクエスト処理を行う lua_vmprep.c や lua_request.c 等の関数群から参照されることが想定されますが、本ソース excerpt 内では具体的な呼び出し先関数は特定されていません。

関連データ構造

  • mapped_request_details: リクエストにマッピングされた Lua スクリプトの詳細情報。
  • apr_hash_t: VM を管理するためのハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が失敗する可能性があります。
  • request_scoped_vms が初期化されていない場合や、ハッシュテーブルからの VM 取得に失敗した場合の挙動は、本構造体の定義のみからは判断できません。

保守担当者向けメモ

  • 本構造体はリクエストごとの状態を保持するため、マルチスレッド環境下での request_scoped_vms へのアクセス競合がないか、各フック処理の実装を確認してください。
  • mapped_request_details のライフサイクルと本構造体のライフサイクルが一致していることを前提としています。
