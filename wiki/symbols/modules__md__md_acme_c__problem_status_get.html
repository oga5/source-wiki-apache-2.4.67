problem_status_get

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme.c:74

目的

ACME プロトコルのエラー型文字列（type）を、Apache HTTP Server 内部で使用される apr_status_t 型のステータスコードへ変換します。

シグネチャまたは形状

static apr_status_t problem_status_get(const char *type)

入力と出力

  • **入力**: type (const char *) - ACME サーバーから返されたエラータイプ文字列。
  • **出力**: apr_status_t - 対応する Apache 内部ステータスコード。一致するものがない場合は APR_EGENERAL を返します。

前提条件と不変条件

  • type 引数は NULL であってはなりません。
  • Problems という名前の静的配列（acme_problem_status_t 型の構造体配列と推測される）が、比較対象として定義されている必要があります。

アルゴリズムまたは振る舞い

1. **プレフィックスの除去**: type 文字列が urn:ietf:params: または urn: で始まる場合、その接頭辞をスキップして比較対象の文字列を短縮します。 2. **検索**: Problems 配列を反復処理し、apr_cstr_casecmp を用いて大文字小文字を区別せずに type と一致するエントリを探索します。 3. **返却**: 一致したエントリが見つかった場合、そのエントリに関連付けられた rv（apr_status_t）を返します。最後まで一致しない場合は APR_EGENERAL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme.c:188 にて、ACME レスポンスの解析処理中に呼び出されています。
  • **呼び出し先**: strstr, strlen, apr_cstr_casecmp などの標準ライブラリおよび APR 文字列操作関数を呼び出しています。

関連データ構造

  • Problems: ACME エラータイプと apr_status_t をマッピングする静的配列（定義詳細は modules/md/md_acme.c 内の acme_problem_status_t 構造体を参照）。

失敗モードと境界条件

  • **一致なし**: 既知の ACME エラータイプに該当しない文字列が渡された場合、デフォルト値として APR_EGENERAL を返します。
  • **NULL ポインタ**: type が NULL の場合、strstr 呼び出し時にセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • Problems 配列の定義が変更された場合、この関数による検索ロジックも影響を受ける可能性があります。
  • 現在のプレフィックス除去ロジックは、ACME 仕様の変更に伴い、将来的に対応すべき URI スキームが増える可能性があります。
