ap_regkey_value_raw_get

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:341

目的

Windows レジストリから指定されたキーの値を、型を問わずバイナリデータとして取得します。

シグネチャまたは形状

apr_status_t ap_regkey_value_raw_get(void **result,
                                     apr_size_t *resultsize,
                                     apr_int32_t *resulttype,
                                     ap_regkey_t *key,
                                     const char *valuename,
                                     apr_pool_t *pool)

入力と出力

  • **入力**:
  • key: 操作対象の ap_regkey_t 構造体。
  • valuename: 取得する値の名前（文字列）。
  • pool: メモリ確保に使用する apr_pool_t。
  • **出力**:
  • result: 取得したデータが格納されるバッファへのポインタ。
  • resultsize: 取得したデータのサイズ（バイト単位）。
  • resulttype: 取得したレジストリ値の型（REG_SZ 等）。
  • 戻り値: 成功時は APR_SUCCESS、失敗時は apr_status_t エラーコード。

前提条件と不変条件

  • key は有効な ap_regkey_t ハンドルであること。
  • pool はメモリ確保のために有効であること。
  • OS 環境に応じて Unicode (RegQueryValueExW) または ANSI (RegQueryValueEx) API が選択されます。

アルゴリズムまたは振る舞い

1. 指定された valuename を使用し、まず RegQueryValueEx を NULL バッファで呼び出して、必要なデータサイズと型を特定します。 2. 取得に失敗した場合は、Win32 エラーを apr_status_t に変換して即時終了します。 3. 特定されたサイズに基づき、apr_palloc を使用して pool からメモリを確保します。 4. 再度 RegQueryValueEx を呼び出し、確保したバッファに実際のデータを読み込みます。 5. 成功時に APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: os/win32/ap_regkey.c:458 (直接的な証拠あり)。
  • **呼び出し先**: RegQueryValueExW または RegQueryValueEx (Win32 API)、apr_palloc、apr_conv_utf8_to_ucs2 (Unicode モード時)。

関連データ構造

  • ap_regkey_t: レジストリキーハンドル (HKEY) を保持するコンテナ。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • レジストリ値が存在しない場合、Win32 エラーコードが返されます。
  • Unicode 変換に失敗した場合、または名前が長すぎる場合はエラーが返されます。
  • メモリ確保失敗時は apr_palloc の挙動に従います。

保守担当者向けメモ

  • 本関数は REG_EXPAND_SZ 型の環境変数展開は行いません（ソース内のコメントは誤解を招く可能性がありますが、実装上は生のバイナリ取得に徹しています）。
  • Unicode/ANSI の切り替えはコンパイル時の APR_HAS_UNICODE_FS / APR_HAS_ANSI_FS マクロに依存します。
