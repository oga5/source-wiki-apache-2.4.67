magic_init

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_mime_magic.c:2359

目的

Apache HTTP Server の起動時（post_config フック）に、各サーバー設定に対してマジックナンバー定義ファイルを読み込み、解析ルールをメモリ上に構築する。

シグネチャまたは形状

static int magic_init(apr_pool_t *p, apr_pool_t *plog, apr_pool_t *ptemp, server_rec *main_server)

入力と出力

  • **入力**:
  • p, plog, ptemp: Apache のメモリプール。
  • main_server: サーバー設定のルートとなる server_rec 構造体。
  • **出力**:
  • 成功時は OK を返す。
  • apprentice 関数が失敗した場合は OK を返す（エラー発生時も処理を継続する設計）。

前提条件と不変条件

  • main_server から mime_magic_module の設定が取得可能であること。
  • magicfile が設定されている場合、apprentice 関数がそのファイルを解析し、magic_server_config_rec 内の magic リストを構築する。

アルゴリズムまたは振る舞い

1. main_server からメインの設定を取得する。 2. サーバーリストを走査し、各サーバーの設定を確認する。 3. magicfile が未設定のサーバーは、親サーバーの設定を継承する。 4. magicfile が設定されている場合、apprentice を呼び出してマジック定義ファイルをパースし、ルールリストを生成する。 5. MIME_MAGIC_DEBUG が有効な場合、構築されたマジックリストの整合性チェック（ポインタ破損の検証）を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_post_config により、サーバー起動時の post_config フェーズで呼び出される。
  • **呼び出し先**:
  • ap_get_module_config: 設定取得。
  • apprentice: マジックファイルのパースとリスト構築。
  • ap_log_error: デバッグログ出力（MIME_MAGIC_DEBUG 有効時）。

関連データ構造

  • magic_server_config_rec: サーバーごとの設定（マジックファイルパス、ルールリスト）を保持。
  • server_rec: Apache のサーバー設定構造体。
  • struct magic: マジックナンバーのルールを保持する単方向連結リスト。

失敗モードと境界条件

  • apprentice が -1 を返した場合、そのサーバーの初期化は中断されるが、関数自体は OK を返して終了する。
  • MIME_MAGIC_DEBUG が有効な場合、マジックリストのポインタが印字可能文字として解釈されるような破損状態にあると、エラーログを出力してループを中断する。

保守担当者向けメモ

  • 本関数は post_config フックとして登録されており、サーバー起動時の設定反映において重要な役割を果たす。
  • MIME_MAGIC_DEBUG は開発時のデバッグ用であり、本番環境では無効化されることが想定されている。
  • apprentice の戻り値が -1 の場合に OK を返す設計となっているため、設定ファイルの構文エラーが無視される可能性がある点に注意が必要。
