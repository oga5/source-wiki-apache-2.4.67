ssl_rand_choosenum

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_rand.c:165

目的

指定された範囲 [l, h] 内の整数を擬似乱数として生成します。

シグネチャまたは形状

static int ssl_rand_choosenum(int l, int h)

入力と出力

  • **入力**:
  • l (int): 範囲の下限値。
  • h (int): 範囲の上限値。
  • **出力**:
  • int: l 以上 h 以下の範囲で生成された整数。

前提条件と不変条件

  • rand() 関数が呼び出し前に初期化されていることを前提としています。
  • 戻り値は必ず l <= result <= h の範囲内に収まるよう、計算後に補正されます。

アルゴリズムまたは振る舞い

1. rand() を使用して 0 から RAND_MAX の範囲の乱数を生成し、それを RAND_MAX で割ることで 0.0 から 1.0 の間の浮動小数点数を得ます。 2. その値に (h - l) を乗算し、apr_snprintf を用いて文字列としてフォーマットすることで整数部を抽出します。 3. atoi で整数に変換し、1 を加算します。 4. 最終的な値が l 未満であれば l に、h を超えていれば h に調整して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_rand.c:118
  • support/ab.c:655
  • **呼び出し先**:
  • rand() (標準ライブラリ)
  • apr_snprintf() (APR)
  • atoi() (標準ライブラリ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • l > h の場合、計算結果の補正ロジックにより、常に h が返される挙動となります。
  • rand() の品質に依存するため、暗号学的に強力な乱数が必要な用途には適していません。

保守担当者向けメモ

  • 本関数は ssl_rand_seed 内でスタックメモリのオフセットを決定する際などに使用されています。
  • apr_snprintf を経由した浮動小数点数から整数への変換は、パフォーマンス上のオーバーヘッドがあるため、より効率的な整数演算への置き換えが検討可能です。
