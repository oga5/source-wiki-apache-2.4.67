alias_match_servlet

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:588

目的

リクエストURIとエイリアス（ProxyPass等で指定されたパス）を照合し、パスパラメータ（;foo=bar）やパスセグメント（/./, /../）を正規化しつつ、エイリアスに一致する部分を特定して残りのパスを抽出します。

シグネチャまたは形状

static int alias_match_servlet(apr_pool_t *p, const char **urip, const char *alias);

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • urip: 照合対象のURI文字列へのポインタ。
  • alias: 比較対象のエイリアス文字列。
  • **出力**:
  • 戻り値: 一致した場合は、URI内のエイリアス以降の開始位置（整数）。一致しない場合は 0。
  • urip: 正規化されたURI文字列で更新される。

前提条件と不変条件

  • uri と alias は共に / で始まる必要がある。
  • パスパラメータ ; は無視される。
  • /./ は削除され、/../ は上位ディレクトリへの移動として処理される。
  • 処理中にルートディレクトリより上位へ移動しようとした場合は 0 を返す。

アルゴリズムまたは振る舞い

1. stack を初期化し、URIを正規化しながら map バッファに構築する。 2. URI内のパスパラメータ ; をスキップする。 3. /./ や /../ を検出し、map 上のセグメントを巻き戻すことでパスを正規化する。 4. 各セグメントごとに alias との照合を行い、一致状況を stack に記録する。 5. 全ての alias セグメントが一致したかを確認する。 6. 一致した場合、エイリアス以降のURI位置を計算して返す。末尾のパラメータ保持やスラッシュの有無を調整する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy.c:900 にて呼び出されていることが確認されている。
  • **呼び出し先**: apr_array_make, apr_palloc, APR_ARRAY_PUSH, APR_ARRAY_IDX, apr_array_pop などの APR 配列操作関数を使用。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_array_header_t: パスセグメントのスタック管理に使用。

失敗モードと境界条件

  • uri または alias が / で始まらない場合: 0 を返す。
  • .. によりルートより上位へ移動しようとした場合: 0 を返す。
  • alias が uri よりも長い場合: 0 を返す。
  • alias の各セグメントが uri と一致しない場合: 0 を返す。

保守担当者向けメモ

  • 本関数は mod_jk 等の挙動を模倣しており、複雑なパス操作を含みます。
  • alias_depth と stack を用いたセグメント単位の照合ロジックは、パスパラメータの保持と正規化が密接に関わっているため、変更時は注意が必要です。
