ap_cgi_build_command

定義

  • 種別: function
  • 定義箇所: modules/arch/netware/mod_netware.c:97

目的

NetWare 環境において、リクエストされたファイルと設定されたハンドラを組み合わせて、CGI 実行用のコマンドラインを構築します。

シグネチャまたは形状

apr_status_t ap_cgi_build_command(const char **cmd, const char ***argv,
                                  request_rec *r, apr_pool_t *p,
                                  cgi_exec_info_t *e_info)

入力と出力

  • **入力**:
  • r: リクエストレコード。
  • p: コマンド構築に使用するメモリプール。
  • e_info: CGI 実行情報（プロセスフラグ、アドレス空間設定など）。
  • cmd: 実行対象のコマンドパス（入出力兼用）。
  • **出力**:
  • cmd: 構築された実行コマンド文字列。
  • argv: トークン化された引数配列。
  • 戻り値: 成功時は APR_SUCCESS、失敗時は APR_EBADF 等のステータスコード。

前提条件と不変条件

  • netware_dir_config がリクエストのディレクトリ設定から取得可能であること。
  • e_info->process_cgi が真の場合、r->filename をベースにコマンドが初期化される。
  • 拡張子に基づいたハンドラが d->file_type_handlers に登録されていること。

アルゴリズムまたは振る舞い

1. **初期化**: リクエスト設定から netware_dir_config を取得。 2. **コマンド解析**: r->filename からファイル拡張子を抽出。 3. **ハンドラ検索**: 抽出した拡張子をキーとして d->file_type_handlers から実行コマンド（NLM）を検索。 4. **コマンド構築**:

  • ハンドラが見つからない場合はエラーをログ出力し APR_EBADF を返す。
  • ハンドラが "OS" 以外の場合、[ハンドラ] [ファイル名] の形式でコマンド文字列を生成。
  • d->file_handler_mode に設定があれば e_info->addrspace を有効化。

    • 5. **トークン化**: 生成されたコマンド文字列を apr_tokenize_to_argv で引数配列に変換。 6. **パス解決**: ap_server_root_relative を使用して実行ファイルの絶対パスを確定。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_cgi.c が APR_RETRIEVE_OPTIONAL_FN を介して呼び出す。
  • **呼び出し先**: ap_get_module_config, apr_pstrdup, apr_filepath_name_get, apr_table_get, ap_log_rerror, stricmp, apr_pstrcat, apr_tokenize_to_argv, ap_server_root_relative。

関連データ構造

  • netware_dir_config: 拡張子マッピング (file_type_handlers) および実行モード (file_handler_mode) を保持。
  • cgi_exec_info_t: CGI 実行時のプロセス制御情報。
  • request_rec: リクエスト固有情報。

失敗モードと境界条件

  • **ハンドラ未登録**: 拡張子に対応するコマンドが file_type_handlers に存在しない場合、APR_EBADF を返し処理を中断する。
  • **拡張子なし**: 拡張子がない場合は空文字列として扱い、ハンドラ検索を行う。

保守担当者向けメモ

  • 本関数は APR_REGISTER_OPTIONAL_FN を介して公開されており、mod_cgi 等の外部モジュールから動的に利用される。
  • Windows 版のロジックと類似しているが、NetWare 特有の「アドレス空間の分離」制御（e_info->addrspace）が含まれている点に留意すること。
