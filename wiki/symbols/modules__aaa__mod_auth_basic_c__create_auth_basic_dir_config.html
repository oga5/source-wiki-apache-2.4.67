create_auth_basic_dir_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_basic.c:48

目的

HTTP Basic 認証モジュール用のディレクトリ単位の設定構造体（auth_basic_config_rec）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_auth_basic_dir_config(apr_pool_t *p, char *d)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t プール。
  • d: ディレクトリパス（本関数内では未使用）。
  • **出力**:
  • void *: 初期化された auth_basic_config_rec 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全フィールドはゼロ初期化される。
  • 認証の権限委譲フラグ authoritative はデフォルトで 1（有効）に設定される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、auth_basic_config_rec サイズ分のメモリをリクエストプールから確保する。 2. 確保した構造体の authoritative メンバに 1 を代入する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_basic.c:502（モジュール定義におけるディレクトリ設定作成ハンドラとして登録）。
  • **呼び出し先**: apr_pcalloc（APRライブラリ）。

関連データ構造

  • auth_basic_config_rec: 本関数で生成されるディレクトリ設定構造体。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足により失敗した場合、Apache の標準的なメモリ割り当てエラー処理に従い、プロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出される。
  • authoritative を 1 に設定することで、認証失敗時に後続の認証モジュールへ処理を渡さない（即座に HTTP_UNAUTHORIZED を返す）というデフォルトの不変条件を確立している。
