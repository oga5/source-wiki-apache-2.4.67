ldap_cache_unlock

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:122

目的

LDAP キャッシュ保護用のグローバルミューテックスを解放し、共有メモリへのアクセスを終了します。

シグネチャまたは形状

static APR_INLINE apr_status_t ldap_cache_unlock(util_ldap_state_t *st, request_rec *r)

入力と出力

  • **入力**:
  • st: LDAP モジュールの状態構造体（util_ldap_state_t）。キャッシュロック（util_ldap_cache_lock）を保持。
  • r: 現在のリクエストレコード（request_rec）。ログ出力に使用。
  • **出力**:
  • apr_status_t: ミューテックス解放の結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • st->util_ldap_cache_lock が初期化されている必要があります。
  • 本関数は、対応する ldap_cache_lock によってロックが取得されているコンテキストで呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. st->util_ldap_cache_lock が存在するか確認します。 2. apr_global_mutex_unlock を呼び出し、ミューテックスを解放します。 3. 解放に失敗した場合、APLOG_CRIT レベルでエラーログを出力し、ap_assert(0) によりプロセスを異常終了させます。 4. 成功した場合は APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c 内のキャッシュ操作関数群（971, 996, 1002, 1075, 1116, 1159, 1168, 1245, 1513, 1559, 1643, 1728 行目付近）。
  • **呼び出し先**: apr_global_mutex_unlock (APR ライブラリ関数)。

関連データ構造

  • util_ldap_state_t: LDAP モジュールの設定と共有リソース（ミューテックス等）を保持する構造体。

失敗モードと境界条件

  • apr_global_mutex_unlock が APR_SUCCESS 以外を返した場合、致命的なエラーとしてログ出力後にアサーション失敗（プロセス停止）が発生します。

保守担当者向けメモ

  • 本関数は APR_INLINE で定義されており、パフォーマンスを考慮してヘッダーまたは同一ファイル内でのインライン展開が意図されています。
  • ミューテックスの解放失敗は共有メモリの整合性に重大な影響を及ぼす可能性があるため、アサーションによる停止が選択されています。
