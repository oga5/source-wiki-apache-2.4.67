h2_get_workers_config

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_config.c:1047

目的

HTTP/2 ワーカースレッドの動作パラメータ（最小・最大スレッド数およびアイドル制限時間）を、サーバー設定および MPM の状態に基づいて解決・取得する。

シグネチャまたは形状

void h2_get_workers_config(server_rec *s, int *pminw, int *pmaxw, apr_time_t *pidle_limit)

入力と出力

  • **入力**:
  • s: サーバー設定構造体 (server_rec*)。
  • **出力**:
  • pminw: 最小ワーカースレッド数へのポインタ。
  • pmaxw: 最大ワーカースレッド数へのポインタ。
  • pidle_limit: アイドル制限時間（apr_time_t）へのポインタ。

前提条件と不変条件

  • s は有効な server_rec であること。
  • 設定値が未設定（<= 0）の場合、MPM のスレッド設定や計算式に基づくデフォルト値で補完される。

アルゴリズムまたは振る舞い

1. h2_config_sgeti を使用し、サーバー設定から H2_CONF_MIN_WORKERS および H2_CONF_MAX_WORKERS を取得する。 2. ap_mpm_query(AP_MPMQ_MAX_THREADS, ...) を呼び出し、現在の MPM のスレッド制限値を取得する。 3. 最小ワーカースレッド数が未設定（<= 0）の場合、MPM のスレッド制限値を適用する。 4. 最大ワーカースレッド数が未設定（<= 0）の場合、4 と (3 * 最小スレッド数 / 2) の大きい方を適用する。 5. h2_config_sgeti64 を使用し、H2_CONF_MAX_WORKER_IDLE_LIMIT を取得して pidle_limit に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_c1.c:75
  • modules/http2/mod_http2.c:156
  • **呼び出し先**:
  • h2_config_sgeti (設定値取得)
  • h2_config_sgeti64 (設定値取得)
  • ap_mpm_query (MPM 情報取得)

関連データ構造

  • server_rec: Apache のサーバー設定コンテキスト。
  • apr_time_t: APR の時間表現型。

失敗モードと境界条件

  • 設定値が 0 または負の値である場合、上記アルゴリズムに従い自動的にデフォルト値が算出される。
  • ap_mpm_query が失敗した場合の挙動については、本関数内での明示的なエラーハンドリングは行われていない。

保守担当者向けメモ

  • 本関数は mod_http2 のワーカースレッドプールの初期化や調整時に使用される。
  • 計算式 3 * (*pminw) / 2 は、最小スレッド数に対して 1.5 倍のバッファを持たせる設計意図がある。
