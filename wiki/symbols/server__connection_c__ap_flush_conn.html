ap_flush_conn

定義

  • 種別: function
  • 定義箇所: server/connection.c:93

目的

接続に対してフラッシュ処理を行い、出力フィルタをクリーンアップして接続を安全に閉じるための準備を行います。

シグネチャまたは形状

void ap_flush_conn(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（対象となる接続の構造体）
  • **出力**: なし（void）

前提条件と不変条件

  • 引数 c は有効な conn_rec 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. 内部的に ap_shutdown_conn(c, 1) を呼び出します。 2. ap_shutdown_conn を通じて、接続に対して EOC（End Of Connection）バケットの挿入とフラッシュ処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/connection.c:137
  • modules/proxy/mod_proxy_ftp.c:2019, 2055, 2076
  • **呼び出し先**:
  • ap_shutdown_conn

関連データ構造

  • conn_rec: Apache の接続状態を保持する主要な構造体。

失敗モードと境界条件

  • ap_shutdown_conn の内部動作に依存します。引数 c が NULL の場合の挙動については、本ソースコード範囲内では定義されていません。

保守担当者向けメモ

  • 本関数は ap_shutdown_conn のラッパーです。接続終了時のフィルタチェーンのクリーンアップが必要な箇所で使用されます。
  • ap_shutdown_conn の第2引数に 1 を渡しているため、強制的なフラッシュと終了処理が行われます。
