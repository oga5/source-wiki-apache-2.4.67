create_dir_config

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_dir.c:143

目的

ディレクトリごとの設定を保持する dir_config_rec 構造体をメモリ上に割り当て、初期値を設定する。

シグネチャまたは形状

static void *create_dir_config(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定を割り当てるためのメモリプール。
  • char *dummy: 未使用の引数（ディレクティブのコンテキスト用）。
  • **出力**:
  • void *: 初期化された dir_config_rec 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc を使用してメモリを確保するため、構造体内の全フィールドは初期状態でゼロクリアされる。
  • 戻り値は dir_config_rec 型としてキャストして利用されることが前提となっている。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、dir_config_rec サイズのメモリをプールから確保する。 2. index_names を NULL に設定する。 3. do_slash、checkhandler、redirect_index の各設定フィールドを、それぞれ MODDIR_UNSET または REDIRECT_UNSET で初期化する。 4. 確保した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定フレームワーク（server/config.c 等）が、モジュール設定の初期化時に呼び出す。
  • **呼び出し先**: apr_pcalloc（APR メモリ管理関数）。

関連データ構造

  • dir_config_rec: 本関数が初期化するディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が例外を発生させる可能性がある（Apache の標準的なメモリ管理動作に依存）。

保守担当者向けメモ

  • 本関数は mod_dir の設定ライフサイクルにおける最初のステップである。設定値が追加された場合は、必ず本関数で適切な初期値（UNSET 状態など）を割り当てること。
