workers_pool_cleanup

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_workers.c:372

目的

HTTP/2 ワーカープールを安全にシャットダウンし、全ワーカースレッドの終了待機およびリソース解放を行うためのクリーンアップ関数です。

シグネチャまたは形状

static apr_status_t workers_pool_cleanup(void *data)

入力と出力

  • **入力**: data (ワーカープール管理構造体 h2_workers* へのポインタ)
  • **出力**: apr_status_t (常に APR_SUCCESS を返す)

前提条件と不変条件

  • 本関数は MPM（Multi-Processing Module）のシャットダウン時に呼び出されることを想定しています。
  • workers->lock ミューテックスにより、スレッド状態の変更および同期が保護されています。

アルゴリズムまたは振る舞い

1. **シャットダウン開始**: workers->shutdown および workers->aborted フラグを立て、wake_all_idles を呼び出して全アイドルスレッドを起床させます。 2. **終了待機**: 最大5秒間、active_slots が0になるまで apr_thread_cond_timedwait を用いて待機します。待機中も定期的に wake_all_idles を呼び出し、スレッドの終了を促します。 3. **タイムアウト判定**: 5秒経過してもスレッドが残存している場合は警告ログを出力します。 4. **ゾンビ回収**: 最後に join_zombies を呼び出し、終了したスレッドのリソースを回収します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_workers.c:535 (apr_pool_pre_cleanup_register を介して登録)
  • **呼び出し先**: wake_all_idles, join_zombies, apr_thread_mutex_lock/unlock, apr_thread_cond_timedwait

関連データ構造

  • h2_workers: プール全体の管理構造体。
  • apr_thread_mutex_t: スレッド同期用ミューテックス。
  • apr_thread_cond_t: スレッド待機用条件変数。

失敗モードと境界条件

  • **タイムアウト**: 5秒以内に全ワーカーが終了しない場合、警告ログを出力しつつ処理を続行します。
  • **強制終了**: MPM のシャットダウンプロセスの一環として実行されるため、接続処理が完了しているかどうかにかかわらず終了処理が進行します。

保守担当者向けメモ

  • 本関数は apr_pool_pre_cleanup_register により登録されており、メモリプール破棄の直前に実行されます。
  • 接続処理が長引いている場合でも、最大待機時間（5秒）を超えると強制的にゾンビ回収へ移行するため、デバッグ時にはログの APLOGNO(10291) に注目してください。
