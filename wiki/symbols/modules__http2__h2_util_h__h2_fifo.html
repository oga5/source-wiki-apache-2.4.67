h2_fifo

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.h:195

目的

スレッドセーフな FIFO（先入れ先出し）キューを提供し、HTTP/2 モジュール内での非同期なデータ受け渡しやタスク管理を支援します。

シグネチャまたは形状

typedef struct h2_fifo h2_fifo;

（不透明な構造体として定義されており、内部実装は h2_util.c に隠蔽されています。）

入力と出力

  • **入力**: h2_fifo_create 等の生成関数において、apr_pool_t（メモリ管理用）および capacity（キューの最大容量）を受け取ります。
  • **出力**: h2_fifo_push や h2_fifo_pull 等の操作関数を通じて、キュー内の要素（void *）の挿入および取り出しを行います。

前提条件と不変条件

  • スレッドセーフな操作が保証されています。
  • h2_fifo_peek を使用する場合、キューのロックを保持したままコールバック関数を実行することで、競合を防ぎつつ先頭要素を安全に検査できます。
  • メモリ管理は apr_pool_t のライフサイクルに依存します。

アルゴリズムまたは振る舞い

  • **ブロッキング操作**: h2_fifo_push および h2_fifo_pull は、必要に応じてブロッキングを行い、キューの状態に応じた同期処理を提供します。
  • **非破壊的検査**: h2_fifo_peek により、キューから要素を取り出すことなく先頭要素を検査可能です。
  • **状態管理**: h2_fifo_count により、現在のキュー内の要素数を取得できます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_util.c 内の複数の関数（h2_fifo_create, h2_fifo_push, h2_fifo_term, h2_fifo_count 等）から参照されています。
  • **呼び出し先**: 外部の具体的な呼び出し先については、提供されたソース excerpt に情報が含まれていないため不明です。

関連データ構造

  • h2_fifo_op_t: FIFO 操作に関連する型。
  • h2_fifo_peek_fn: h2_fifo_peek で使用されるコールバック関数の型定義。
  • h2_ififo: 整数専用の FIFO キュー。

失敗モードと境界条件

  • キューが満杯の場合の push 動作や、空の場合の pull 動作は、apr_status_t 型の戻り値を通じてエラーや状態が報告されます。
  • 境界条件として、キューの容量制限（capacity）に達した際の挙動が定義されています。

保守担当者向けメモ

  • 本構造体は h2_util.c で実装されており、詳細な内部構造やロックの仕組みを確認する場合は同ファイルを参照してください。
  • メモリ所有権については、コンテナが管理する場合と呼び出し元が管理する場合があるため、各関数のドキュメントを確認してください。
