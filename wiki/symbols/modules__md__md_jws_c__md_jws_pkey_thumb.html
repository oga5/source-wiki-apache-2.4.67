md_jws_pkey_thumb

定義

  • 種別: function
  • 定義箇所: modules/md/md_jws.c:93

目的

RSA公開鍵から、RFC 8555に準拠したJWK（JSON Web Key）形式の指紋（Thumbprint）を計算する。

シグネチャまたは形状

apr_status_t md_jws_pkey_thumb(const char **pthumb, apr_pool_t *p, struct md_pkey_t *pkey)

入力と出力

  • **入力**:
  • pkey: 指紋計算対象のRSA公開鍵構造体。
  • p: メモリ割り当てに使用するAPRプール。
  • **出力**:
  • pthumb: 計算されたSHA-256指紋（Base64URLエンコード済み文字列）へのポインタ。
  • 戻り値: 成功時に APR_SUCCESS、失敗時にエラーコードを返す。

前提条件と不変条件

  • pkey はRSA鍵を含んでいる必要がある。
  • 生成されるJSON文字列は、キー順序（e, kty, n）および空白の有無が厳密に固定されている必要がある（指紋計算の整合性のため）。

アルゴリズムまたは振る舞い

1. md_pkey_get_rsa_e64 および md_pkey_get_rsa_n64 を使用して、公開鍵の指数（e）と係数（n）をBase64URLエンコード文字列として取得する。 2. 取得した値を用いて、{"e":"...","kty":"RSA","n":"..."} という形式のJSON文字列を構築する。 3. 構築した文字列に対して md_crypt_sha256_digest64 を実行し、SHA-256ハッシュを計算してBase64URLエンコードする。 4. 結果を pthumb に格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_authz.c:227
  • **呼び出し先**:
  • md_pkey_get_rsa_e64 (ソース内)
  • md_pkey_get_rsa_n64 (ソース内)
  • md_data_init_str (ソース内)
  • md_crypt_sha256_digest64 (ソース内)

関連データ構造

  • md_pkey_t: 公開鍵情報を保持する構造体。
  • md_data_t: ハッシュ計算対象のデータを保持する構造体。

失敗モードと境界条件

  • e64 または n64 の取得に失敗した場合（RSA鍵ではない場合など）、APR_EINVAL を返す。
  • 内部のハッシュ計算関数 md_crypt_sha256_digest64 が失敗した場合、そのエラーコードを伝播する。

保守担当者向けメモ

  • 本関数が生成するJSON文字列のフォーマットはRFC 8555の要件に直結しているため、文字列構築時の順序やフォーマットを変更してはならない。
  • メモリはすべて引数 p（apr_pool_t）から割り当てられるため、呼び出し側でプールのライフサイクルを管理すること。
