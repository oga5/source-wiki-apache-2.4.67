mod_filter_ctx

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_filter.c:72

目的

リクエスト処理中に mod_filter が使用するリクエスト単位のコンテキストを保持します。

シグネチャまたは形状

typedef struct {
    const char* range;
} mod_filter_ctx;

入力と出力

  • **入力**: ap_get_module_config を介してリクエスト設定から取得されます。
  • **出力**: apr_pcalloc を用いてリクエストプール (r->pool) 上に割り当てられます。

前提条件と不変条件

  • リクエストごとのライフサイクルを持ち、r->pool に紐付いて管理されます。

アルゴリズムまたは振る舞い

  • mod_filter の実行時に、リクエストに関連付けられたコンテキストとしてインスタンス化されます。
  • メンバ変数 range は、フィルタ処理における範囲指定や条件判定に関連する文字列情報を保持するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_filter.c:145: ap_get_module_config を通じてリクエスト設定から取得。
  • modules/filters/mod_filter.c:645: apr_pcalloc を通じてメモリ確保。
  • **呼び出し先**:
  • 呼び出し先に関する情報は提供されていません。

関連データ構造

  • mod_filter_cfg: モジュール全体の設定を保持する構造体。
  • harness_ctx: 実行時のフィルタプロバイダ選択状態を保持する構造体。

失敗モードと境界条件

  • メモリ確保失敗時、apr_pcalloc の挙動に依存します。
  • range メンバが初期化されていない場合や、不正な文字列が設定された場合の挙動については、本ソース抜粋からは判断できません。

保守担当者向けメモ

  • 本構造体は mod_filter のディスパッチ処理における状態保持用です。
  • range メンバの具体的な利用用途については、mod_filter.c 内の他の関数による参照箇所を確認してください。
