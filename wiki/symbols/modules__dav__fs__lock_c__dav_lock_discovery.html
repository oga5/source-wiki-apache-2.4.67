dav_lock_discovery

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/lock.c:123

目的

ファイルシステムベースの WebDAV ロック管理において、リソースに対する「直接ロック（Direct Lock）」の情報を保持するために使用されます。

シグネチャまたは形状

typedef struct dav_lock_discovery
{
    struct dav_lock_discovery_fixed f;
    dav_locktoken *locktoken;
    const char *owner;
    const char *auth_user;
    struct dav_lock_discovery *next;
} dav_lock_discovery;

入力と出力

本構造体はデータ保持用であり、関数ではありません。主に dav_fs_load_lock_record によって DBM から読み込まれ、リンクドリストとして構築されます。また、dav_fs_save_lock_record を通じて DBM へシリアライズされます。

前提条件と不変条件

  • next ポインタにより、同一リソースに対する複数のロックをリンクドリストとして管理します。
  • f メンバ（dav_lock_discovery_fixed）には、ロックの有効期限やスコープなどの固定長データが格納されます。
  • メモリ割り当ては、関連する request_rec のプール（r->pool）で行われることが想定されています。

アルゴリズムまたは振る舞い

  • ロック情報の永続化において、この構造体のリストを走査し、各要素をバイナリ形式に変換して DBM に保存します。
  • ロックの読み込み時には、期限切れのロックを除外しながらリストを再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/lock.c 内のロック操作関数（dav_fs_save_lock_record, dav_fs_load_lock_record 等）がこの構造体を操作します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な呼び出し先はありません。

関連データ構造

  • dav_lock_discovery_fixed: 構造体内の固定長データ部分。
  • dav_locktoken: ロックを一意に識別するための UUID ラッパー。
  • dav_lock_indirect: 間接ロック（継承ロック）を管理する構造体。

失敗モードと境界条件

  • DBM が読み取り専用で開かれている場合、この構造体を用いた保存操作は失敗します。
  • データベースの破損や整合性エラーについては、外部要因に依存しており、本構造体自体には保護機能はありません。

保守担当者向けメモ

  • 本構造体は dav_lock_discovery_fixed と分離して管理されています。
  • メモリ管理は Apache のプール機構に依存しているため、明示的な free は不要ですが、長期間実行されるプロセスではメモリ消費量に注意してください。
