rename_original_env

定義

  • 種別: function
  • 定義箇所: modules/http/http_request.c:511

目的

環境変数テーブル（apr_table_t）内の各キーに対し、プレフィックス REDIRECT_ を付与して新しいテーブルを作成する。主に内部リダイレクト発生時に、元の環境変数を保持しつつリダイレクト先へ引き継ぐために使用される。

シグネチャまたは形状

static apr_table_t *rename_original_env(apr_pool_t *p, apr_table_t *t)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • t: 変換対象の環境変数テーブル。
  • **出力**:
  • 変換後の新しい apr_table_t。各キーは REDIRECT_ + 元のキー に書き換えられている。

前提条件と不変条件

  • 入力テーブル t が有効であること。
  • p は新しいテーブルの生存期間を管理するのに十分な寿命を持つプールであること。
  • キーが NULL のエントリはスキップされる。

アルゴリズムまたは振る舞い

1. 入力テーブル t の全要素を走査する。 2. 各要素のキーに対し、apr_pstrcat を用いて REDIRECT_ を先頭に連結する。 3. 連結後の新しいキーと元の値を、apr_table_make で作成した新しいテーブルに apr_table_setn で格納する。 4. 変換後のテーブルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http/http_request.c:608
  • **呼び出し先**:
  • apr_table_elts
  • apr_table_make
  • apr_pstrcat
  • apr_table_setn

関連データ構造

  • apr_table_t: Apache のキー・値ペアを保持するテーブル構造。
  • apr_table_entry_t: テーブル内の個々のエントリ。

失敗モードと境界条件

  • メモリ不足時: apr_table_make や apr_pstrcat がメモリ割り当てに失敗した場合、APR の規約に従いプロセスが異常終了する可能性がある。
  • 空のテーブル: 入力テーブルが空の場合、空の新しいテーブルが返される。

保守担当者向けメモ

  • 本関数は内部リダイレクト時の環境変数引き継ぎ専用である。
  • apr_table_setn を使用しているため、値の文字列はコピーされずポインタが共有される。元のテーブル t が破棄される場合、値の生存期間に注意が必要である。
