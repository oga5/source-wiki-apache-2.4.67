create_request

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_ocsp.c:105

目的

OpenSSL の OCSP_REQUEST オブジェクトを生成し、指定された証明書の ID を追加して、必要に応じて Nonce を付与する。

シグネチャまたは形状

static OCSP_REQUEST *create_request(X509_STORE_CTX *ctx, X509 *cert,
                                    OCSP_CERTID **certid,
                                    server_rec *s, apr_pool_t *p,
                                    SSLSrvConfigRec *sc)

入力と出力

  • **入力**:
  • ctx: 証明書検証コンテキスト (X509_STORE_CTX)。
  • cert: 検証対象の証明書 (X509)。
  • certid: 生成された OCSP_CERTID を格納するためのポインタ。
  • s: サーバーレコード (server_rec)。
  • p: メモリプール (apr_pool_t)。
  • sc: SSL サーバー設定 (SSLSrvConfigRec)。
  • **出力**:
  • 成功時: 初期化済みの OCSP_REQUEST オブジェクト。
  • 失敗時: NULL。

前提条件と不変条件

  • ctx から発行者証明書が取得可能であること。
  • sc->server->ocsp_use_request_nonce が FALSE でない場合、リクエストに Nonce が追加される。

アルゴリズムまたは振る舞い

1. OCSP_REQUEST_new() で新しいリクエストオブジェクトを生成する。 2. OCSP_cert_to_id() を使用して、対象証明書と発行者から OCSP_CERTID を生成する。 3. 生成した certid をリクエストに追加する。失敗した場合はエラーログを出力して NULL を返す。 4. 設定に基づき、OCSP_request_add1_nonce() を呼び出してリクエストに Nonce を追加する。 5. 完成した OCSP_REQUEST オブジェクトを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_ocsp.c:152 (verify_ocsp_status 関数内から呼び出される)。
  • **呼び出し先**: OpenSSL API (OCSP_REQUEST_new, OCSP_cert_to_id, X509_STORE_CTX_get0_current_issuer, OCSP_request_add0_id, OCSP_request_add1_nonce) および Apache ログ関数 (ap_log_error, ssl_log_ssl_error)。
  • 注意: include/http_request.h 等に存在する create_request フックとの関連性は、名前の衝突によるものであり、本関数とは別物である。

関連データ構造

  • OCSP_REQUEST: OpenSSL の OCSP リクエスト構造体。
  • OCSP_CERTID: 証明書識別子。
  • SSLSrvConfigRec: mod_ssl のサーバー設定構造体。

失敗モードと境界条件

  • OCSP_cert_to_id が失敗した場合、または OCSP_request_add0_id が失敗した場合は、エラーログを記録して NULL を返す。
  • メモリ不足等の理由で OCSP_REQUEST_new が失敗した場合は NULL が返る。

保守担当者向けメモ

  • 本関数は mod_ssl の OCSP 検証フローの一部であり、OpenSSL の OCSP API に依存している。
  • ocsp_use_request_nonce 設定の有無によってリクエストの構造が変化するため、レスポンス検証側との整合性に注意すること。
