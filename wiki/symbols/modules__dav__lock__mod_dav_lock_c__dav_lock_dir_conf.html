dav_lock_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/dav/lock/mod_dav_lock.c:26

目的

Apache HTTP Server の mod_dav において、ディレクトリ単位で設定されたロックデータベースのパスを保持するための設定構造体です。

シグネチャまたは形状

typedef struct {
    const char *lockdb_path;
} dav_lock_dir_conf;

入力と出力

  • **入力**: DAVGenericLockDB ディレクティブにより設定されたパス文字列。
  • **出力**: lockdb_path メンバを通じて、ロックデータベースの物理的な場所を後続のロック操作処理へ提供します。

前提条件と不変条件

  • lockdb_path は、サーバーの実行ユーザーが書き込み可能なファイルシステム上のパスである必要があります。
  • 設定は mod_dav のディレクトリ設定マージロジックに従い、子ディレクトリは親ディレクトリの設定を継承可能です。

アルゴリズムまたは振る舞い

  • 設定読み込み時、dav_lock_cmd_davlockdb ハンドラによってパスが格納されます。
  • dav_lock_merge_dir_config 関数により、ディレクトリ階層間での設定の継承が行われます。
  • ロック操作が必要な際、この構造体からパスが読み出され、locks.c で実装されたデータベース操作に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • dav_lock_create_dir_config: 構造体のメモリ確保時に参照。
  • dav_lock_merge_dir_config: 設定のマージ処理時に参照。
  • dav_lock_cmd_davlockdb: 設定値の書き込み時に参照。
  • dav_generic_get_lockdb_path (推測): ロックデータベースパスの取得時に参照。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • dav_lockdb_private: locks.c で使用される、DBM ハンドルとリクエストコンテキストを保持する構造体。

失敗モードと境界条件

  • lockdb_path が未設定の場合、ロック機能は正しく動作しません。
  • パスが相対パスの場合、サーバーのルートディレクトリからの相対パスとして扱われます。

保守担当者向けメモ

  • 本構造体は mod_dav_lock.c 内のディレクトリ設定管理の基盤です。構造体にフィールドを追加する場合は、dav_lock_merge_dir_config でのマージロジックも併せて更新してください。
