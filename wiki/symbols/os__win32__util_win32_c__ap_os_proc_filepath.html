ap_os_proc_filepath

定義

  • 種別: function
  • 定義箇所: os/win32/util_win32.c:30

目的

現在実行中のプロセス（Apache）のフルパスを取得し、UTF-8文字列として呼び出し元へ提供します。

シグネチャまたは形状

apr_status_t ap_os_proc_filepath(char **binpath, apr_pool_t *p)

入力と出力

  • **入力**:
  • binpath: 取得したパスを格納する文字列ポインタへのポインタ。
  • p: メモリ確保に使用する apr_pool_t。
  • **出力**:
  • 成功時: APR_SUCCESS を返し、*binpath にパスが格納される。
  • 失敗時: apr_status_t 型のエラーコードを返す。

前提条件と不変条件

  • p は有効なAPRプールである必要があります。
  • Windows環境において、OSがUnicodeをサポートしているか（APR_HAS_UNICODE_FS）、ANSIのみか（APR_HAS_ANSI_FS）に応じて処理が分岐します。

アルゴリズムまたは振る舞い

1. **Unicode環境 (APR_HAS_UNICODE_FS)**:

  • GetModuleFileNameW を呼び出し、ワイド文字でパスを取得します。
  • 取得したパスを apr_conv_ucs2_to_utf8 を用いてUTF-8へ変換します。
  • 変換後の文字列は apr_palloc でプールから確保されます。

    • 2. **ANSI環境 (APR_HAS_ANSI_FS)**:

  • GetModuleFileName を呼び出し、ANSI文字列としてパスを取得します。
  • apr_pstrdup を用いてプールからメモリを確保し、パスをコピーします。

    • 3. **共通**:

  • Windows APIの呼び出しに失敗した場合、apr_get_os_error() を通じてエラーを伝播させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/winnt/mpm_winnt.c:581
  • server/mpm/winnt/mpm_winnt.c:1128
  • **呼び出し先**:
  • Windows API: GetModuleFileNameW または GetModuleFileName
  • APR: apr_get_os_error, apr_palloc, apr_conv_ucs2_to_utf8, apr_pstrdup

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_wchar_t: Unicodeパス格納用の一時バッファ。

失敗モードと境界条件

  • GetModuleFileName 系APIが失敗した場合、OSのエラーコードが返されます。
  • Unicode変換時に apr_conv_ucs2_to_utf8 が失敗した場合、そのエラーコードが返されます。
  • 変換後の文字列長が想定を超えた場合、APR_ENAMETOOLONG が返される可能性があります。

保守担当者向けメモ

  • 本関数は APR_HAS_UNICODE_FS および APR_HAS_ANSI_FS マクロに強く依存しています。
  • Unicode変換ロジックは、Windows環境における文字コードの差異を吸収するための重要な箇所です。
  • 将来的にAPR側の機能と重複する可能性があるため、APRの更新状況に応じて統合を検討してください。
