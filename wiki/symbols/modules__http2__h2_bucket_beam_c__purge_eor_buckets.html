purge_eor_buckets

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:170

目的

h2_bucket_beam 内の buckets_eor リストに保持されているすべてのバケットを削除し、リソースを解放する。

シグネチャまたは形状

static void purge_eor_buckets(h2_bucket_beam *beam)

入力と出力

  • **入力**: beam (h2_bucket_beam *) - 操作対象のバケットビーム構造体。
  • **出力**: なし（副作用として beam->buckets_eor が空になる）。

前提条件と不変条件

  • 本関数は**送信側スレッドからのみ**呼び出される必要がある。
  • beam->buckets_eor は H2_BLIST 型のリスト構造である。

アルゴリズムまたは振る舞い

1. H2_BLIST_EMPTY マクロを使用して、beam->buckets_eor リストが空でないかを確認する。 2. リストが空になるまでループを繰り返す。 3. ループ内で H2_BLIST_FIRST を使用してリストの先頭バケットを取得する。 4. apr_bucket_delete を呼び出し、当該バケットを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_bucket_beam.c:285 にて呼び出しを確認。
  • **呼び出し先**: apr_bucket_delete (APR API)。

関連データ構造

  • h2_bucket_beam: バケット転送を管理するメイン構造体。
  • buckets_eor: h2_bucket_beam 内の h2_blist 型メンバ。EOR (End of Request) バケットを保持する。
  • apr_bucket: APR が提供するデータバケット構造体。

失敗モードと境界条件

  • リストが既に空である場合、ループは実行されず何も行われない。
  • apr_bucket_delete の内部実装に依存し、バケットの参照カウントが 0 になることでメモリが解放される。

保守担当者向けメモ

  • 本関数は送信側スレッドのコンテキストで実行されることを前提としているため、マルチスレッド環境での競合を避けるため、呼び出し元の同期状態を維持すること。
