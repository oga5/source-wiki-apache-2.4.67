log_init

定義

  • 種別: function
  • 定義箇所: modules/loggers/mod_log_forensic.c:106

目的

サーバー起動時に、設定されたすべてのサーバーインスタンスに対してフォレンジックログファイルを開き、初期化を行う。

シグネチャまたは形状

static int log_init(apr_pool_t *pc, apr_pool_t *p, apr_pool_t *pt, server_rec *s)

入力と出力

  • **入力**:
  • pc, p, pt: Apache のメモリプール。
  • s: サーバー設定のリスト（server_rec）。
  • **出力**:
  • 成功時は OK を返す。
  • ログファイルのオープンに失敗したサーバーが存在する場合、HTTP_INTERNAL_SERVER_ERROR を返す。

前提条件と不変条件

  • s が指すサーバーリストを順次走査する。
  • 各サーバーに対して open_log が呼び出され、ログファイルディスクリプタの初期化が試行される。

アルゴリズムまたは振る舞い

1. 渡されたサーバーリスト s をループで走査する。 2. 各サーバーに対して open_log(s, p) を実行する。 3. open_log が失敗（0を返却）した場合、直ちに HTTP_INTERNAL_SERVER_ERROR を返して処理を中断する。 4. 全てのサーバーで正常にログが開かれた場合、OK を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_open_logs に登録されており、Apache のログオープンフェーズで呼び出される（modules/loggers/mod_log_forensic.c:275）。
  • **呼び出し先**: open_log

関連データ構造

  • server_rec: サーバー設定を保持する構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • ログファイルのパスが無効、または書き込み権限がない等の理由で open_log が失敗した場合、サーバーの起動プロセスがエラーで停止する。

保守担当者向けメモ

  • 本関数は open_logs フックとして動作するため、設定の変更やログファイルのローテーションに関連する初期化処理の起点となる。
  • s が NULL の場合はループが実行されず、即座に OK が返される。
