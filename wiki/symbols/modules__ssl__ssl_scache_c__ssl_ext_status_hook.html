ssl_ext_status_hook

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_scache.c:197

目的

Apache HTTP Server の mod_status 拡張として、SSL セッションキャッシュの現在の利用状況をサーバー管理画面（ステータスページ）に出力します。

シグネチャまたは形状

static int ssl_ext_status_hook(request_rec *r, int flags)

入力と出力

  • **入力**:
  • r: 現在のリクエストを処理する request_rec 構造体。
  • flags: ステータス表示の形式を指定するフラグ（AP_STATUS_SHORT 等）。
  • **出力**:
  • int: 常に OK を返します。

前提条件と不変条件

  • SSLModConfigRec がサーバー設定から取得可能であること。
  • セッションキャッシュ (mc->sesscache) が初期化されていること。これらが存在しない場合、処理は即座に終了します。

アルゴリズムまたは振る舞い

1. **表示形式の判定**: flags に AP_STATUS_SHORT が含まれていない場合、HTML テーブル形式でヘッダーを出力します。含まれている場合は簡潔なテキスト形式で出力します。 2. **排他制御**: キャッシュプロバイダーがスレッドセーフでない (AP_SOCACHE_FLAG_NOTMPSAFE) と判断された場合、ssl_mutex_on を呼び出して排他制御を開始します。 3. **ステータス取得**: プロバイダー固有の status メソッドを呼び出し、キャッシュの統計情報を出力します。 4. **排他制御の解除**: 排他制御を開始した場合は ssl_mutex_off を呼び出します。 5. **フッター出力**: HTML 形式の場合、テーブルを閉じるタグを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_status のフックとして登録されています（modules/ssl/ssl_scache.c:236 にて APR_OPTIONAL_HOOK を使用）。
  • **呼び出し先**:
  • ssl_mutex_on / ssl_mutex_off (排他制御用)
  • mc->sesscache->status (プロバイダー固有のステータス出力関数)

関連データ構造

  • SSLModConfigRec: SSL モジュールの設定情報を保持する構造体。
  • request_rec: Apache のリクエスト処理用構造体。
  • ap_socache_provider_t: mc->sesscache を通じてアクセスされるキャッシュプロバイダーのインターフェース。

失敗モードと境界条件

  • mc (モジュール設定) または mc->sesscache が NULL の場合、何も出力せず OK を返します。
  • プロバイダーの status メソッド実行中にエラーが発生した場合の挙動は、各プロバイダーの実装に依存します。

保守担当者向けメモ

  • 本関数は mod_status のフックとして動作するため、出力内容は mod_status の表示仕様に準拠する必要があります。
  • 排他制御のフラグチェックは、プロバイダーの特性に応じて適切に行われています。
