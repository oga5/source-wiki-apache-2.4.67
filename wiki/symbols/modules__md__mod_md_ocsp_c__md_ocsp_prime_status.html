md_ocsp_prime_status

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_ocsp.c:56

目的

証明書チェーンを解析し、OCSP Stapling のためのステータス取得準備を行う。

シグネチャまたは形状

int md_ocsp_prime_status(server_rec *s, apr_pool_t *p,
                         const char *id, apr_size_t id_len, const char *pem)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p: メモリプール。
  • id, id_len: OCSP ステータス識別子。
  • pem: 証明書チェーンを含む PEM 形式の文字列。
  • **出力**:
  • int: 成功時は OK、失敗または対象外の場合は DECLINED を返す。

前提条件と不変条件

  • staple_here(sc) が真であること（設定により OCSP Stapling が有効であること）。
  • 入力された pem データから少なくとも2つの証明書が読み取れること。

アルゴリズムまたは振る舞い

1. staple_here を呼び出し、現在のサーバー設定で OCSP Stapling が適用可能か確認する。適用外の場合は DECLINED を返す。 2. md_cert_read_chain を使用して、pem 文字列から証明書チェーンを解析する。 3. チェーン内の証明書数が2未満の場合はエラーとして DECLINED を返す。 4. md_ocsp_prime を呼び出し、チェーンの先頭2つの証明書（対象証明書と発行者証明書）を用いて OCSP ステータスの取得準備を行う。 5. 処理結果に基づき OK または DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c 内の ap_hook_ssl_ocsp_prime_hook に登録されている。
  • **呼び出し先**: staple_here, md_cert_read_chain, md_ocsp_prime。

関連データ構造

  • server_rec: Apache サーバー設定。
  • md_srv_conf_t: mod_md のサーバー設定。
  • md_t: 管理対象ドメイン情報。
  • md_cert_t: 証明書情報。

失敗モードと境界条件

  • staple_here が偽を返す場合、処理はスキップされる。
  • md_cert_read_chain による PEM 解析失敗時はエラーログを出力し DECLINED を返す。
  • 証明書チェーンが2枚未満の場合、APR_EINVAL 相当の状態で DECLINED を返す。

保守担当者向けメモ

  • 本関数は mod_ssl のフックから呼び出される。
  • 実際の OCSP 通信プロトコルや ASN.1 解析の詳細は md_ocsp.c の md_ocsp_prime に委譲されている。
  • md_ocsp_prime の戻り値が APR_SUCCESS の場合のみ OK を返す。
