winnt_check_config

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/mpm_winnt.c:1423

目的

MPM固有の設定ディレクティブ（ThreadLimit および ThreadsPerChild）の値を検証し、コンパイル時の制限値や実行時の制約に基づいて調整・修正を行う。

シグネチャまたは形状

static int winnt_check_config(apr_pool_t *pconf, apr_pool_t *plog,
                              apr_pool_t *ptemp, server_rec* s)

入力と出力

  • **入力**:
  • pconf, plog, ptemp: 設定およびログ用メモリプール。
  • s: 現在のサーバー設定レコード。
  • **出力**:
  • 戻り値: OK (常に成功を返す)。
  • 副作用: グローバル変数 thread_limit および ap_threads_per_child の値を、検証ルールに従って更新する。

前提条件と不変条件

  • thread_limit は MAX_THREAD_LIMIT を超えてはならない。
  • thread_limit は 1 未満であってはならない。
  • thread_limit は再起動をまたいで変更することはできない（初回設定値 first_thread_limit が優先される）。
  • ap_threads_per_child は thread_limit を超えてはならない。
  • ap_threads_per_child は 1 未満であってはならない。

アルゴリズムまたは振る舞い

1. 現在のプロセスが親プロセスかつ初期設定フェーズであるかを判定し、ログ出力の冗長性を制御する。 2. thread_limit が MAX_THREAD_LIMIT を超える場合、制限値まで減算する。1 未満の場合は 1 に補正する。 3. first_thread_limit が未設定であれば現在の thread_limit を記録し、設定済みであれば現在の値が初回値と一致するか確認する。不一致の場合は初回値に強制的に戻す。 4. ap_threads_per_child が thread_limit を超える場合、thread_limit の値に合わせる。1 未満の場合は 1 に補正する。 5. 補正が必要な場合、startup（起動時）か否かに応じて適切なログレベルで警告を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/mpm_winnt.c 内の winnt_hooks 関数にて ap_hook_check_config に登録されている。
  • **呼び出し先**: ログ出力のために ap_log_error を呼び出す。

関連データ構造

  • server_rec: サーバー設定情報。
  • thread_limit (グローバル変数): 設定可能な最大スレッド数。
  • ap_threads_per_child (グローバル変数): 子プロセスあたりのスレッド数。
  • first_thread_limit (グローバル変数): 再起動時の整合性維持のための初回設定値。

失敗モードと境界条件

  • 設定値が範囲外の場合、エラーを返さずに警告ログを出力し、許容範囲内の値へ自動的に修正（クランプ）する。
  • 再起動時に ThreadLimit を変更しようとした場合、変更は無視され、警告ログが出力される。

保守担当者向けメモ

  • 本関数は設定の検証と修正を行うため、設定読み込みフェーズの後半で実行されることを前提としている。
  • startup フラグの判定には ap_state_query(AP_SQ_MAIN_STATE) を使用しており、Apacheのライフサイクル状態に依存している。
