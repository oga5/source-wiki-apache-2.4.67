md_acme_order_st

定義

  • 種別: enum
  • 定義箇所: modules/md/md_acme_order.h:24

目的

ACME プロトコルにおける「Order（注文）」オブジェクトの現在の状態を表現し、証明書発行プロセスのライフサイクルを管理するために使用されます。

シグネチャまたは形状

typedef enum {
    MD_ACME_ORDER_ST_PENDING,
    MD_ACME_ORDER_ST_READY,
    MD_ACME_ORDER_ST_PROCESSING,
    MD_ACME_ORDER_ST_VALID,
    MD_ACME_ORDER_ST_INVALID,
} md_acme_order_st;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: md_acme_order_t 構造体の status フィールドとして保持されます。

前提条件と不変条件

  • ACMEv2 プロトコルの仕様に基づいた状態遷移（pending -> ready -> valid 等）を前提としています。
  • md_acme_order_update() 関数により、サーバーから取得した最新情報とローカルのこの状態値が同期されます。

アルゴリズムまたは振る舞い

  • 証明書発行の各ステップ（認証、処理中、完了）を識別します。
  • order_st_to_str() を介して文字列へ変換され、JSON 形式での永続化やログ出力に利用されます。
  • order_st_from_str() を介して、保存された JSON データから状態を復元します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/md/md_acme_order.c: order_st_from_str および order_st_to_str 関数内で参照。
  • modules/md/md_acme_order.h: md_acme_order_t 構造体のメンバとして定義。
  • 呼び出し先: なし（列挙型のため）

関連データ構造

  • md_acme_order_t: この列挙型を保持するメインの注文管理構造体。

失敗モードと境界条件

  • 未定義の文字列が order_st_from_str に渡された場合の挙動については、ソースコードの該当関数実装に依存します。

保守担当者向けメモ

  • 状態遷移のロジックを追跡する際は、modules/md/md_acme_order.c 内の md_acme_order_update 実装を参照してください。
  • ACME サーバーから返される JSON レスポンスと密接に結びついているため、プロトコル仕様の変更時にはこの列挙値の整合性を確認してください。
