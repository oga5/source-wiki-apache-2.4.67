set_systemd_listener

定義

  • 種別: function
  • 定義箇所: server/listen.c:341

目的

systemd ソケットアクティベーション経由で渡されたファイル記述子（fd）を取得し、Apache のリスナーリスト (ap_listeners) に追加する。

シグネチャまたは形状

static const char *set_systemd_listener(process_rec *process, apr_port_t port, const char *proto);

入力と出力

  • **入力**:
  • process: 現在のプロセスレコード。
  • port: 検索対象のポート番号。
  • proto: プロトコル名（例: "http"）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラー内容を示す文字列（const char *）を返す。

前提条件と不変条件

  • mod_systemd がロードされており、ap_find_systemd_socket 関数が利用可能であること。
  • ap_listeners はリスナーの連結リストであり、本関数は新しいリスナーをその末尾に追加する。

アルゴリズムまたは振る舞い

1. APR_RETRIEVE_OPTIONAL_FN を使用して ap_find_systemd_socket を取得する。取得できない場合はエラーを返す。 2. find_systemd_socket を呼び出し、指定されたポートに対応するファイル記述子 fd を取得する。fd < 0 の場合はエラーを返す。 3. ap_listeners リストの末尾まで走査する。 4. alloc_systemd_listener を呼び出し、取得した fd と proto を用いて新しいリスナー構造体 new を生成する。 5. リストが空であれば ap_listeners を new に設定し、空でなければリスト末尾の next ポインタを new に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/listen.c:1053
  • **呼び出し先**:
  • ap_find_systemd_socket (オプション関数)
  • alloc_systemd_listener

関連データ構造

  • ap_listen_rec: リスナー情報を保持する構造体。
  • process_rec: プロセス情報を保持する構造体。

失敗モードと境界条件

  • mod_systemd がロードされていない場合、エラー文字列を返す。
  • 指定したポートが systemd 側で設定されていない場合、エラー文字列を返す。
  • alloc_systemd_listener によるリスナー生成に失敗した場合、エラー文字列を返す。

保守担当者向けメモ

  • 本関数は HAVE_SYSTEMD マクロ環境下でのみ有効なロジックを含みます。
  • リスナーリストの末尾への追加処理は、リストが空の場合とそうでない場合で分岐しています。
