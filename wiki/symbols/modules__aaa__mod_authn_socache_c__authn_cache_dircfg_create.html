authn_cache_dircfg_create

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:199

目的

ディレクトリ単位の認証キャッシュ設定構造体（authn_cache_dircfg）をメモリ上に割り当て、デフォルト値で初期化する。

シグネチャまたは形状

static void* authn_cache_dircfg_create(apr_pool_t *pool, char *s)

入力と出力

  • **入力**:
  • pool: 設定構造体の割り当てに使用するメモリプール。
  • s: ディレクトリパス（本関数内では使用されていない）。
  • **出力**:
  • 初期化済みの authn_cache_dircfg 構造体へのポインタ。

前提条件と不変条件

  • pool は有効な apr_pool_t であること。
  • 割り当てられた構造体の timeout はデフォルトで300秒に設定される。
  • providers は初期状態で NULL となる。

アルゴリズムまたは振る舞い

1. apr_palloc を使用して authn_cache_dircfg サイズのメモリを確保する。 2. ret->timeout に apr_time_from_sec(300) を代入する。 3. ret->providers を NULL に初期化する。 4. ret->context に directory を設定する。 5. 構築した構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_socache.c:468（Apache の設定読み込みプロセスから呼び出される）。
  • **呼び出し先**: apr_palloc, apr_time_from_sec（APR ライブラリ関数）。

関連データ構造

  • authn_cache_dircfg: 本関数が初期化するディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_palloc が例外を発生させる可能性がある。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズでディレクトリ設定が生成される際に呼び出されます。
  • ret->context に代入されている directory は、ソースコード上では列挙型または定数として定義されているものと推測されます。
