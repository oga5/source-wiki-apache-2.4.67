queue_info_cleanup

定義

  • 種別: function
  • 定義箇所: server/mpm_fdqueue.c:54

目的

fd_queue_info_t 構造体が保持する同期プリミティブの破棄および、再利用リスト（recycled_pools）に蓄積されたメモリプールの解放を行うクリーンアップ処理。

シグネチャまたは形状

apr_status_t queue_info_cleanup(void *data_)

入力と出力

  • **入力**: data_ (クリーンアップ対象の fd_queue_info_t 構造体へのポインタ)
  • **出力**: APR_SUCCESS を返す。

前提条件と不変条件

  • data_ は有効な fd_queue_info_t 構造体を指している必要がある。
  • 本関数は、apr_pool_cleanup_register を介してプール破棄時に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. qi->wait_for_idler (条件変数) を破棄する。 2. qi->idlers_mutex (ミューテックス) を破棄する。 3. qi->recycled_pools リストを走査し、各要素をアトミックな比較交換（apr_atomic_casptr）を用いてリストから取り出し、対応する apr_pool_destroy を実行してメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm_fdqueue.c:97 において、ap_queue_info_create 内でプールクリーンアップハンドラとして登録されている。
  • **呼び出し先**: apr_thread_cond_destroy, apr_thread_mutex_destroy, apr_atomic_casptr, apr_pool_destroy (すべて APR 提供の関数)。

関連データ構造

  • fd_queue_info_t: 本関数が管理するリソースを保持する構造体。
  • recycled_pool: qi->recycled_pools リストを構成するノード。

失敗モードと境界条件

  • recycled_pools が空の場合、ループは即座に終了する。
  • apr_atomic_casptr による競合が発生した場合、リストの走査を継続する設計となっている。

保守担当者向けメモ

  • 本関数は fd_queue_info_t のライフサイクル終了時に呼び出される。
  • recycled_pools の解放においてアトミック操作を使用しているため、他のスレッドが同時にプールをプッシュしている状況でも安全にクリーンアップが可能である。
