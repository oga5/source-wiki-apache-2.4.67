example_handler

定義

  • 種別: function
  • 定義箇所: docs/manual/developer/mod_example_1.c:11

目的

Apache HTTP Server のリクエスト処理パイプラインにおいて、特定のハンドラ名（example-handler）に対するリクエストを捕捉し、クライアントへ "Hello, world!" を応答する。

シグネチャまたは形状

static int example_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r （Apache のリクエスト情報を保持する構造体）
  • **出力**: int 型のステータスコード
  • DECLINED: リクエストが対象外である場合。
  • OK: リクエストの処理が正常に完了した場合。

前提条件と不変条件

  • r->handler が "example-handler" と一致する場合のみ、コンテンツの生成を行う。
  • コンテンツ生成前に ap_set_content_type を呼び出し、MIME タイプを text/html に設定する必要がある。

アルゴリズムまたは振る舞い

1. r->handler を確認し、"example-handler" でない場合は直ちに DECLINED を返して処理を委譲する。 2. ap_set_content_type(r, "text/html") を呼び出し、レスポンスのコンテンツタイプを設定する。 3. ap_rprintf(r, "Hello, world!") を呼び出し、クライアントへレスポンスボディを書き出す。 4. 処理完了を示す OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のハンドラフック機構。
  • docs/manual/developer/mod_example_1.c:34 にて ap_hook_handler に登録されている。
  • docs/manual/developer/mod_example_2.c:127 にも同名の関数が登録されているが、本シンボルとの直接的な呼び出し関係は不明。
  • **呼び出し先**:
  • ap_set_content_type (Apache API)
  • ap_rprintf (Apache API)

関連データ構造

  • request_rec: リクエストのコンテキストおよびハンドラ名を保持するために使用。

失敗モードと境界条件

  • r->handler が NULL または "example-handler" 以外の場合、本関数は何も行わず DECLINED を返す。

保守担当者向けメモ

  • 本関数は学習用サンプルです。実運用環境では、セキュリティ上の観点から適切なアクセス制御や入力バリデーションの追加を検討してください。
  • 戻り値の DECLINED と OK は Apache のパイプライン制御において重要な意味を持つため、変更時は注意が必要です。
