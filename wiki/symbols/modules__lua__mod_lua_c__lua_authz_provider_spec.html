lua_authz_provider_spec

定義

  • 種別: struct
  • 定義箇所: modules/lua/mod_lua.c:61

目的

Apache HTTP Server の認証プロバイダとして登録された Lua 関数のメタデータを保持し、実行に必要なスクリプト情報と VM 仕様を管理する。

シグネチャまたは形状

typedef struct {
    const char *name;          /* プロバイダ名 */
    const char *file_name;     /* 実行する Lua スクリプトのファイルパス */
    const char *function_name; /* 呼び出す Lua 関数名 */
    ap_lua_vm_spec *spec;      /* VM の実行環境仕様 */
} lua_authz_provider_spec;

入力と出力

  • 入力: 認証プロバイダ登録時に設定値（名前、ファイル、関数、VM 仕様）が本構造体に格納される。
  • 出力: 認証フェーズにおいて、本構造体を参照して特定の Lua 関数が呼び出される。

前提条件と不変条件

  • spec フィールドは ap_lua_vm_spec 型へのポインタであり、有効な VM ライフサイクル管理が保証されている必要がある。
  • 認証プロバイダとして機能するためには、Apache の認証フックと適切に紐付けられている必要がある。

アルゴリズムまたは振る舞い

1. 設定フェーズで、LuaAuthzProvider ディレクティブ等により本構造体が初期化される。 2. 認証リクエスト発生時、登録されたプロバイダの spec を通じて Lua VM を取得する。 3. file_name で指定されたスクリプトを読み込み、function_name で指定された Lua 関数を実行する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/lua/mod_lua.c 内の認証フック登録およびプロバイダ処理ロジック（1725, 1759, 1835 行付近で参照を確認）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、実行時には ap_lua_get_lua_state 等の VM 管理関数が呼び出される。

関連データ構造

  • ap_lua_vm_spec: VM のスコープやプール管理を定義する構造体。

失敗モードと境界条件

  • file_name が無効な場合、または function_name がスクリプト内に存在しない場合、認証処理は失敗し、HTTP 500 エラーが返される可能性がある。
  • VM の取得に失敗した場合、認証プロバイダは適切に動作しない。

保守担当者向けメモ

  • 本構造体は mod_lua.c 内で認証プロバイダのメタデータを保持する中心的な役割を果たす。
  • 認証プロバイダの追加や変更を行う際は、spec が指す ap_lua_vm_spec のライフサイクル（メモリ管理）に注意すること。
