passwd_ctx

定義

  • 種別: struct
  • 定義箇所: support/passwd_common.h:86

目的

パスワード生成および検証プロセス全体の状態を保持・管理するためのコンテキスト構造体です。メモリプール、入力元、アルゴリズム設定、および処理結果を統合的に管理します。

シグネチャまたは形状

struct passwd_ctx {
    apr_pool_t      *pool;
    const char      *errstr;
    char            *out;
    apr_size_t      out_len;
    char            *passwd;
    int             alg;
    int             cost;
    enum {
        PW_PROMPT = 0,
        PW_ARG,
        PW_STDIN,
        PW_PROMPT_VERIFY,
    } passwd_src;
};

入力と出力

  • **入力**: passwd_src に基づき、標準入力、コマンドライン引数、またはプロンプトからパスワードを取得します。alg および cost はハッシュ生成時のアルゴリズムと計算コストを指定します。
  • **出力**: 生成されたハッシュ値は out に格納され、処理結果やエラーメッセージは errstr に保持されます。

前提条件と不変条件

  • pool は apr_pool_t によって管理され、メモリ確保の基盤となります。
  • abort_on_oom() が登録されている場合、メモリ確保失敗時にはプロセスが即座に終了します。
  • 処理前に errfile が初期化されている必要があります（未初期化の場合、putline 等でセグメンテーション違反が発生する可能性があります）。

アルゴリズムまたは振る舞い

1. get_password() を通じて passwd_src に応じたパスワード入力を passwd メンバに格納します。 2. mkhash() が ctx 内の alg および cost を参照し、ハッシュを計算して out に書き込みます。 3. 成功時は 0 を返し、失敗時は ERR_* コードを返すと同時に errstr にエラー詳細をセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htpasswd.c (mkrecord 関数等)、support/htdbm.c、および modules/lua/ や modules/md/ 内の各関数から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。操作は passwd_common.c で定義されたユーティリティ関数（get_password, mkhash 等）を介して行われます。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • passwd_src (enum): パスワードの入力ソースを定義。

失敗モードと境界条件

  • メモリ不足時: abort_on_oom によりプロセスが強制終了します。
  • パスワード不一致や構文エラー: ERR_PWMISMATCH や ERR_SYNTAX 等の共通エラーコードが返され、errstr に詳細が記録されます。
  • 初期化不備: errfile が未初期化の状態で putline 等を呼び出すとクラッシュします。

保守担当者向けメモ

  • 本構造体は support モジュール内の複数のツールで共有されています。変更を加える際は htpasswd や htdbm への影響を考慮してください。
  • cost メンバは bcrypt のコスト値や SHA-2 のラウンド数として利用されます。
  • mkhash の内部ロジックは passwd_common.c に隠蔽されています。
