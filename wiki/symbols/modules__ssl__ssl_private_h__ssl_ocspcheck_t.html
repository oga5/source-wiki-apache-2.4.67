ssl_ocspcheck_t

定義

  • 種別: enum
  • 定義箇所: modules/ssl/ssl_private.h:496

目的

SSL/TLS 通信における OCSP（Online Certificate Status Protocol）検証の適用範囲および動作モードを定義する列挙型です。

シグネチャまたは形状

typedef enum {
    SSL_OCSPCHECK_NONE  = (0),
    SSL_OCSPCHECK_LEAF  = (1 << 0),
    SSL_OCSPCHECK_CHAIN = (1 << 1),
    SSL_OCSPCHECK_NO_OCSP_FOR_CERT_OK = (1 << 2)
} ssl_ocspcheck_t;

入力と出力

  • 本シンボルは設定値として使用されます。
  • SSL_OCSPCHECK_NONE: OCSP 検証を行いません。
  • SSL_OCSPCHECK_LEAF: リーフ証明書のみを検証対象とします。
  • SSL_OCSPCHECK_CHAIN: 証明書チェーン全体を検証対象とします。
  • SSL_OCSPCHECK_NO_OCSP_FOR_CERT_OK: OCSP 検証が利用できない場合でも、証明書自体が有効であれば許可するフラグです。

前提条件と不変条件

  • 各値はビットフラグとして定義されており、組み合わせが可能です。
  • SSL_OCSPCHECK_NONE は他のフラグと併用されないことが想定されます。

アルゴリズムまたは振る舞い

  • mod_ssl の設定処理において、ユーザーが指定した OCSP 検証ポリシーを保持するために使用されます。
  • 実際の検証ロジックは、ssl_engine_ocsp.c や ssl_util_ocsp.c 等のコンポーネントで、この列挙値に基づいて実行されます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数関係については、提供されたソース情報からは特定できません。
  • ssl_engine_ocsp.c がこの列挙型を参照している可能性が高いですが、直接的な呼び出し関係の証拠はありません。

関連データ構造

  • SSLSrvConfigRec（仮想ホストごとの設定構造体）内で、OCSP 検証設定を保持するために使用されることが想定されます。

失敗モードと境界条件

  • 無効なビット組み合わせが設定された場合の挙動は、本定義のみでは規定されていません。
  • SSL_OCSPCHECK_NO_OCSP_FOR_CERT_OK を使用する場合、OCSP サーバーへの接続失敗が「検証成功」と見なされるため、セキュリティポリシーとの整合性に注意が必要です。

保守担当者向けメモ

  • 本シンボルは ssl_private.h で定義されており、mod_ssl 内部のプライベートな設定管理に使用されます。
  • OCSP 関連の機能拡張を行う際は、ssl_engine_ocsp.c の実装と併せて、この列挙型に新しいフラグが必要か検討してください。
