ap_lua_check_request_rec

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_request.c:118

目的

Lua スタック上の userdata が Apache2.Request 型であることを検証し、対応する Apache の内部構造体 request_rec へのポインタを抽出する。

シグネチャまたは形状

request_rec *ap_lua_check_request_rec(lua_State *L, int index)

入力と出力

  • **入力**:
  • L: Lua 状態（lua_State*）。
  • index: Lua スタック上の対象 userdata の位置。
  • **出力**:
  • 成功時: request_rec 構造体へのポインタを返す。
  • 失敗時: luaL_checkudata により Lua エラーが発生し、呼び出し元へは戻らない。

前提条件と不変条件

  • 指定されたスタック位置 index に Apache2.Request メタテーブルを持つ userdata が存在すること。
  • lua_unboxpointer を介して、有効な request_rec ポインタが格納されていること。

アルゴリズムまたは振る舞い

1. luaL_checkudata(L, index, "Apache2.Request") を呼び出し、スタック上の値が期待される型であることを検証する。型が一致しない場合、Lua エラーをスローする。 2. lua_unboxpointer(L, index) を呼び出し、userdata に埋め込まれた request_rec ポインタを取得する。 3. 取得した request_rec ポインタを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_dbd.c: データベース操作関数（r の取得に使用）。
  • modules/lua/lua_request.c: リクエスト属性操作関数（ヘッダー、ボディ操作等）。
  • **呼び出し先**:
  • luaL_checkudata (Lua API)
  • lua_unboxpointer (Lua API)

関連データ構造

  • request_rec: Apache HTTP Server のリクエスト情報を保持する主要構造体。
  • Apache2.Request: Lua 側でリクエストを表現するためのメタテーブル名。

失敗モードと境界条件

  • スタック位置 index が無効、または型が Apache2.Request ではない場合、Lua VM がエラーを発生させ、処理が中断される。

保守担当者向けメモ

  • 本関数は mod_lua における型安全性の要である。Lua 側から渡される userdata が不正な場合、即座に Lua エラーとして処理されるため、呼び出し側で NULL チェックを行う必要はない。
