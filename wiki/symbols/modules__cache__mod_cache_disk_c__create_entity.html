create_entity

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_disk.c:338

目的

ディスクキャッシュにおける新しいキャッシュエントリの初期化と、メタデータ構造の割り当てを行う。

シグネチャまたは形状

static int create_entity(cache_handle_t *h, request_rec *r, const char *key, apr_off_t len, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • h: キャッシュハンドル。
  • r: 現在のリクエストレコード。
  • key: キャッシュキー（URI）。
  • len: コンテンツ長（不明な場合は -1）。
  • bb: バケットブリゲード（本関数内では未使用）。
  • **出力**:
  • OK: 成功。
  • DECLINED: キャッシュ不可と判断された場合。

前提条件と不変条件

  • conf->cache_root が設定されている必要がある。
  • リクエストが HTTP_PARTIAL_CONTENT（Range リクエスト）でないこと。
  • コンテンツ長 len が設定されている場合、dconf->minfs 以上かつ dconf->maxfs 以下であること。

アルゴリズムまたは振る舞い

1. 設定値に基づき、キャッシュルートの存在確認およびサイズ制限チェックを行う。 2. r->pool を使用して cache_object_t および disk_cache_object_t を割り当てる。 3. キャッシュキー、キャッシュルート情報を構造体に格納する。 4. r->pool の子プールを作成し、キャッシュのライフサイクル管理用とする。 5. file_cache_create を呼び出し、ヘッダー、Vary 情報、データ用のファイル管理構造を初期化する。 6. header_file および data_file を使用して、各ファイルのパスを生成・設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/cache_storage.c の cache_storage_create_entity を経由して呼び出される。
  • **呼び出し先**:
  • ap_get_module_config: 設定取得。
  • apr_pcalloc, apr_pstrdup, apr_pstrmemdup: メモリ割り当て。
  • apr_pool_create, apr_pool_tag: プール管理。
  • file_cache_create: ファイル管理構造の初期化。
  • header_file, data_file: パス生成。
  • ap_log_rerror: ログ出力。

関連データ構造

  • cache_handle_t: キャッシュ操作のハンドル。
  • disk_cache_object_t: ディスクキャッシュ固有のオブジェクト状態。
  • disk_cache_conf: サーバー全体の設定。
  • disk_cache_dir_conf: ディレクトリ単位の設定。

失敗モードと境界条件

  • cache_root が未設定の場合、DECLINED を返す。
  • Range リクエストの場合、キャッシュ対象外として DECLINED を返す。
  • コンテンツサイズが設定範囲外の場合、DECLINED を返す。

保守担当者向けメモ

  • 本関数は r->pool に依存しており、リクエスト終了時に自動的にクリーンアップされる。
  • len が -1 の場合はサイズチェックがスキップされるため、後続の処理で整合性を担保する必要がある。
  • Range リクエストのキャッシュは現在サポートされていない。
