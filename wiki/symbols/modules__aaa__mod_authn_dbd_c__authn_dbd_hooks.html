authn_dbd_hooks

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_dbd.c:285

目的

mod_authn_dbd モジュールを Apache HTTP Server の認証プロバイダーとして登録し、キャッシュ連携に必要なオプション関数を解決するためのフックを設定する。

シグネチャまたは形状

static void authn_dbd_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（モジュール初期化に使用されるメモリプール）
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • authn_dbd_password および authn_dbd_realm 関数が定義済みであること。
  • opt_retr 関数がキャッシュモジュールとの連携のために利用可能であること。

アルゴリズムまたは振る舞い

1. authn_provider 構造体（authn_dbd_password および authn_dbd_realm を保持）を静的に定義する。 2. ap_register_auth_provider を呼び出し、AUTHN_PROVIDER_GROUP に対して「dbd」という名前でプロバイダーを登録する。 3. ap_hook_optional_fn_retrieve を呼び出し、opt_retr をフックとして登録することで、実行時にキャッシュ用関数の遅延解決を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は modules/aaa/mod_authn_dbd.c:306 である（モジュールの登録プロセスの一部）。
  • **呼び出し先**:
  • ap_register_auth_provider
  • ap_hook_optional_fn_retrieve

関連データ構造

  • authn_provider: 認証プロバイダーのインターフェースを定義する構造体。

失敗モードと境界条件

  • 登録処理の失敗に関する明示的なエラーハンドリングは本関数内には含まれていない。

保守担当者向けメモ

  • 本関数はモジュールのロード時に一度だけ実行される。
  • AP_AUTH_INTERNAL_PER_CONF フラグにより、認証プロバイダーのスコープがディレクトリ設定単位に制限されている点に注意すること。
