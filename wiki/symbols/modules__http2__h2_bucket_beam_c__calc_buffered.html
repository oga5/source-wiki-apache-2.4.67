calc_buffered

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_bucket_beam.c:133

目的

h2_bucket_beam 内の送信待ちバケットリスト（buckets_to_send）に含まれる、メモリ上に実体を持つデータ量を計算します。

シグネチャまたは形状

static apr_size_t calc_buffered(h2_bucket_beam *beam)

入力と出力

  • **入力**: beam (h2_bucket_beam *) - 処理対象のバケットビーム構造体。
  • **出力**: apr_size_t - メモリ消費量としてカウントされた合計バイト数。

前提条件と不変条件

  • beam は有効なポインタである必要があります。
  • beam->buckets_to_send リストは H2_BLIST 構造に従って初期化されている必要があります。

アルゴリズムまたは振る舞い

1. beam->buckets_to_send リストを先頭から末尾まで走査します。 2. 各バケットの属性に基づき、以下のルールで加算を行います。

  • バケット長が (apr_size_t)-1 の場合はカウントしません。
  • ファイルバケット（APR_BUCKET_IS_FILE）または MMAP バケット（bucket_is_mmap）は、メモリフットプリントを持たないものとして無視します。
  • 上記以外（ヒープバケット等）の場合、そのバケット長を合計値 len に加算します。

    • 3. 最終的な合計値 len を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_bucket_beam.c:184 にて呼び出しを確認。
  • **呼び出し先**:
  • H2_BLIST_FIRST, H2_BLIST_SENTINEL, APR_BUCKET_NEXT (マクロ)
  • bucket_is_mmap (関数)

関連データ構造

  • h2_bucket_beam: バケット転送を管理するメイン構造体。
  • apr_bucket: APR のバケット構造体。
  • h2_blist: buckets_to_send を保持するリスト構造。

失敗モードと境界条件

  • リストが空の場合、合計値として 0 を返します。
  • バケット長が不明なバケットが含まれる場合、そのバケットは計算から除外されます。

保守担当者向けメモ

  • 本関数は、ファイルや MMAP バケットを「メモリ消費なし」と見なす設計です。これらが将来的にメモリへ読み込まれるような変更を行う場合は、本関数のロジックを更新する必要があります。
