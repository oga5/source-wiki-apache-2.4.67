md_get_by_domain

定義

  • 種別: function
  • 定義箇所: modules/md/md_core.c:165

目的

指定されたマネージドドメイン（md_t）のリストから、特定のドメイン名を含む最初の md_t オブジェクトを検索して返します。

シグネチャまたは形状

md_t *md_get_by_domain(struct apr_array_header_t *mds, const char *domain)

入力と出力

  • **入力**:
  • mds: 検索対象となる md_t ポインタを格納した apr_array_header_t 配列。
  • domain: 検索対象のドメイン名（文字列）。
  • **出力**:
  • 指定ドメインを含む md_t へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • mds は有効な apr_array_header_t 構造体である必要があります。
  • md_t のドメインリストは、md_array_str_compact 等によって正規化されていることが期待されます。

アルゴリズムまたは振る舞い

1. mds 配列内の各要素を順次走査します。 2. 各要素（md_t）に対して md_contains 関数を呼び出し、指定された domain が含まれているか判定します。 3. md_contains が真を返した時点で、その md_t ポインタを即座に返します。 4. 配列の最後まで一致が見つからなかった場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/mod_md.c:1418
  • modules/md/mod_md_status.c:80
  • modules/md/mod_md_status.c:962
  • **呼び出し先**:
  • md_contains（modules/md/md_core.c 内の関数）

関連データ構造

  • md_t: 管理対象ドメインの情報を保持する構造体。
  • apr_array_header_t: APR が提供する動的配列構造体。

失敗モードと境界条件

  • mds が空（nelts == 0）の場合、ループは実行されず NULL が返されます。
  • domain がどの md_t にも含まれない場合、NULL が返されます。

保守担当者向けメモ

  • 本関数は線形探索を行うため、mds 配列の要素数が増大した場合にはパフォーマンスへの影響を考慮する必要があります。
  • md_contains の内部実装（ワイルドカードマッチング等）に依存しているため、ドメイン照合の仕様変更時は md_contains の挙動を確認してください。
