header_request_duration

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_headers.c:170

目的

HTTP リクエストの処理に要した時間を計算し、D=マイクロ秒単位の経過時間 という形式の文字列を生成します。

シグネチャまたは形状

static const char *header_request_duration(request_rec *r, char *a)

入力と出力

  • **入力**:
  • r: 現在のリクエストを保持する request_rec 構造体へのポインタ。
  • a: 未使用の引数（char *）。
  • **出力**:
  • リクエスト開始時刻から現在時刻までの経過時間をマイクロ秒単位で表した文字列を返します。メモリは r->pool から割り当てられます。

前提条件と不変条件

  • r は有効な request_rec 構造体を指している必要があります。
  • r->request_time には、リクエストが開始された時刻が格納されている必要があります。

アルゴリズムまたは振る舞い

1. apr_time_now() を呼び出し、現在のシステム時刻を取得します。 2. 現在時刻から r->request_time を減算し、リクエスト処理の経過時間を算出します。 3. apr_psprintf を使用し、"D=%ld"（APR_TIME_T_FMT）の形式で文字列を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_headers.c:979 にて、register_format_tag_handler("D", header_request_duration) を通じてフォーマットタグハンドラとして登録されています。
  • **呼び出し先**:
  • apr_time_now() (APR)
  • apr_psprintf() (APR)

関連データ構造

  • request_rec: リクエストのライフサイクルと時刻情報を保持する Apache コア構造体。

失敗モードと境界条件

  • apr_psprintf によるメモリ割り当てが失敗した場合、Apache のメモリ管理ポリシーに従いプロセスが終了する可能性があります。
  • r->request_time が正しく設定されていない場合、経過時間は不正確な値となります。

保守担当者向けメモ

  • 本関数は mod_headers における %D フォーマットタグの処理ロジックです。
  • 経過時間はマイクロ秒単位で出力されます。単位の変更やフォーマットの変更を行う場合は、apr_psprintf の書式指定を確認してください。
