register_hook

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_proxy_http2.c:481

目的

Apache HTTP Server のプロキシモジュールとして、HTTP/2 関連のフック関数を登録する。

シグネチャまたは形状

static void register_hook(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p (モジュール初期化に使用されるメモリプール)
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • 本関数は Apache のモジュールロードプロセス中に呼び出される必要がある。
  • 登録される各ハンドラ（h2_proxy_post_config, proxy_http2_handler, proxy_http2_canon）が定義済みであること。

アルゴリズムまたは振る舞い

1. ap_hook_post_config を使用して、h2_proxy_post_config を設定完了後のフックとして登録する。 2. proxy_hook_scheme_handler を使用して、proxy_http2_handler をスキームハンドラとして登録する（APR_HOOK_FIRST）。 3. proxy_hook_canon_handler を使用して、proxy_http2_canon を URL 正規化ハンドラとして登録する（APR_HOOK_FIRST）。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュールロード機構（直接の呼び出し元はソースコード上では明示されていない）。
  • 呼び出し先:
  • ap_hook_post_config
  • proxy_hook_scheme_handler
  • proxy_hook_canon_handler
  • h2_proxy_post_config
  • proxy_http2_handler
  • proxy_http2_canon

関連データ構造

  • apr_pool_t: Apache Portable Runtime のメモリプール。

失敗モードと境界条件

  • 登録処理自体は戻り値を持たないため、フック登録の失敗は Apache の起動プロセス全体のエラーとして扱われる。

保守担当者向けメモ

  • APR_HOOK_FIRST が指定されているため、他のプロキシモジュールよりも優先して実行される点に注意すること。
  • ツールが提示した他のファイル（mod_cgid.c 等）内の register_hook は同名の別関数であり、本シンボルとは無関係である。
