user_parse_config

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_user.c:96

目的

設定ファイル内の require ディレクティブで指定された引数を解析し、実行時に評価可能な式（ap_expr_info_t）へ変換します。

シグネチャまたは形状

static const char *user_parse_config(cmd_parms *cmd, const char *require_line,
                                     const void **parsed_require_line)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのコンテキスト情報。
  • require_line: 解析対象となる require ディレクティブの引数文字列。
  • **出力**:
  • parsed_require_line: 解析結果の ap_expr_info_t 構造体へのポインタが格納されます。
  • 戻り値: 解析成功時は NULL を返し、失敗時はエラーメッセージ文字列を返します。

前提条件と不変条件

  • require_line は ap_expr_parse_cmd が処理可能な形式である必要があります。
  • 解析成功時、*parsed_require_line には有効な ap_expr_info_t 構造体のアドレスが設定されます。

アルゴリズムまたは振る舞い

1. ap_expr_parse_cmd を呼び出し、require_line を文字列結果を返す式として解析します。 2. 解析中にエラーが発生した場合（expr_err が非 NULL）、cmd->temp_pool を使用してエラーメッセージを構築し、呼び出し元へ返します。 3. 解析が正常に完了した場合、生成された式オブジェクトを *parsed_require_line に代入し、NULL を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_user.c:118（authz_user_provider 構造体の初期化時に参照されています）。
  • **呼び出し先**: ap_expr_parse_cmd（Apache の式評価エンジン）。

関連データ構造

  • cmd_parms: 設定コマンドのパラメータを保持する構造体。
  • ap_expr_info_t: 解析された式を保持する構造体。

失敗モードと境界条件

  • ap_expr_parse_cmd が構文エラーを検出した場合、関数はエラーメッセージを返し、認可プロバイダーの設定は失敗します。

保守担当者向けメモ

  • 本関数は mod_authz_core が提供するプロバイダーインターフェースの一部として実装されています。
  • ap_expr を使用しているため、require user で動的なユーザー指定が可能です。
  • エラーメッセージの生成には cmd->temp_pool を使用しており、メモリリークの懸念はありません。
