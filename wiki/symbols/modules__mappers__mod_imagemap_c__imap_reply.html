imap_reply

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_imagemap.c:458

目的

画像マップの判定結果に基づき、HTTP レスポンスコードを決定するか、または指定された URL へのリダイレクトを指示する。

シグネチャまたは形状

static int imap_reply(request_rec *r, const char *redirect)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報を保持する request_rec 構造体。
  • redirect: マップファイルから読み取られたリダイレクト先の文字列、または制御コマンド（"error", "nocontent"）。
  • **出力**:
  • int: HTTP ステータスコード（HTTP_INTERNAL_SERVER_ERROR, HTTP_NO_CONTENT, HTTP_MOVED_TEMPORARILY のいずれか）。

前提条件と不変条件

  • redirect 引数が NULL または空文字列の場合、デフォルトで HTTP_INTERNAL_SERVER_ERROR を返す。
  • r->headers_out は、リダイレクト時に Location ヘッダーを設定するために使用される。

アルゴリズムまたは振る舞い

1. redirect 文字列が "error" と一致する場合、HTTP_INTERNAL_SERVER_ERROR を返す。 2. redirect 文字列が "nocontent" と一致する場合、HTTP_NO_CONTENT を返す。 3. redirect が有効な文字列である場合、r->headers_out に Location ヘッダーとして設定し、HTTP_MOVED_TEMPORARILY を返す。 4. 上記のいずれにも該当しない場合、HTTP_INTERNAL_SERVER_ERROR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/mappers/mod_imagemap.c 内の以下の行から呼び出される。
  • 780行目, 793行目, 806行目, 835行目, 844行目
  • **呼び出し先**:
  • apr_table_setn (Apache APR ライブラリ関数)

関連データ構造

  • request_rec: Apache のリクエスト処理用構造体。

失敗モードと境界条件

  • redirect が空文字列または NULL の場合、エラーとして扱われる。
  • 大文字小文字を区別しない比較 (strcasecmp) を行うため、"ERROR" や "NoContent" といった表記もそれぞれ対応する動作として処理される。

保守担当者向けメモ

  • 本関数はリダイレクト先の URL を検証せず、そのまま Location ヘッダーに設定する。URL の妥当性は呼び出し側で担保する必要がある。
  • HTTP_MOVED_TEMPORARILY (302) を返しているため、永続的なリダイレクトが必要な場合は別の実装を検討すること。
