abort_on_oom

定義

  • 種別: function
  • 定義箇所: support/ab.c:510

目的

メモリ割り当て失敗時にエラーメッセージを出力し、プロセスを強制終了させるためのコールバック関数です。

シグネチャまたは形状

static int abort_on_oom(int retcode)

入力と出力

  • **入力**: retcode (int型) - APRのメモリ割り当て失敗時に渡されるエラーコード。
  • **出力**: なし（関数内で exit(1) を呼び出すため、呼び出し元へは戻りません）。

前提条件と不変条件

  • APR（Apache Portable Runtime）のメモリプールにおいて、メモリ不足が発生した際に呼び出されることを前提としています。
  • 本関数が実行された場合、プロセスは即座に終了するため、呼び出し元での正常な処理継続は不可能です。

アルゴリズムまたは振る舞い

1. 標準エラー出力 (stderr) に "Could not allocate memory" というメッセージを出力します。 2. exit(1) を呼び出し、プロセスを異常終了させます。 3. return retcode; は到達不能コードであり、実行されません。

呼び出し元と呼び出し先

  • **呼び出し元**: apr_pool_abort_set を通じて、APRのメモリプール管理機能から呼び出されます。
  • support/ab.c:2367 (本ファイル内)
  • modules/http2/h2_mplx.c:227
  • server/main.c:338, server/main.c:518
  • support/htdbm.c:113
  • support/htpasswd.c:306
  • **呼び出し先**: 標準ライブラリの fprintf および exit を呼び出します。

関連データ構造

  • apr_pool_t: APRのメモリプール構造体。本関数は、このプールに対するメモリ割り当て失敗時のハンドラとして登録されます。

失敗モードと境界条件

  • メモリ不足が発生した時点で本関数がトリガーされます。
  • 呼び出し元で apr_pool_abort_set を使用して適切に登録されていない場合、本関数は機能しません。
