h2_push_diary_entry

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_push.c:477

目的

HTTP/2 サーバープッシュ機能において、過去にプッシュしたリソースを識別するためのハッシュ値を保持するデータ構造です。

シグネチャまたは形状

typedef struct h2_push_diary_entry {
    apr_uint64_t hash;
} h2_push_diary_entry;

入力と出力

  • **入力**: apr_uint64_t 型のハッシュ値。
  • **出力**: なし（構造体として値を保持）。

前提条件と不変条件

  • h2_push_diary 構造体内の entries 配列（apr_array_header_t）の要素として使用されます。
  • 格納される hash は、プッシュ対象リソースの識別子として機能します。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持のみを行います。
  • h2_push_diary 内で、重複プッシュの判定や、LRU（Least Recently Used）的な順序管理のために、memmove を用いた配列操作の対象となります。
  • h2_push_diary_append 関数により、新しいエントリが配列の末尾に追加されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_push.c: h2_push_diary の初期化時および更新時に参照されます。
  • APR_ARRAY_IDX や APR_ARRAY_PUSH マクロを介して、配列操作の単位として利用されます。
  • **呼び出し先**:
  • 情報なし。

関連データ構造

  • h2_push_diary: 本構造体を要素として保持する、プッシュ履歴管理用のメイン構造体。

失敗モードと境界条件

  • 配列操作時（memmove）に、インデックス範囲外へのアクセスが発生しないよう、呼び出し側の h2_push_diary 実装における境界チェックに依存します。

保守担当者向けメモ

  • apr_uint64_t 型のハッシュ値は、calc_sha256_hash または calc_apr_hash によって生成された値が格納されます。
  • 配列のサイズ変更や要素の移動は apr_array_header_t の API を通じて行われるため、メモリ管理は apr_pool_t に依存します。
