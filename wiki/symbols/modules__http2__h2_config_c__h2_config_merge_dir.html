h2_config_merge_dir

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_config.c:251

目的

Apache HTTP Server のディレクトリ設定において、親コンテキスト（basev）と子コンテキスト（addv）の設定値をマージし、新しい設定構造体を生成する。

シグネチャまたは形状

void *h2_config_merge_dir(apr_pool_t *pool, void *basev, void *addv)

入力と出力

  • **入力**:
  • pool: 設定構造体の割り当てに使用するメモリプール。
  • basev: 親ディレクトリの設定構造体（h2_dir_config）。
  • addv: 子ディレクトリの設定構造体（h2_dir_config）。
  • **出力**:
  • マージ結果を保持する新しい h2_dir_config 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は h2_dir_config 型にキャスト可能であること。
  • マージ処理は元の構造体を変更せず、常に新しいメモリ領域を確保して返す。
  • H2_CONFIG_GET マクロにより、子設定が未設定（DEF_VAL）の場合に親設定が優先される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して新しい h2_dir_config 構造体を確保する。 2. name フィールドにマージ元の名称を連結して格納する。 3. h2_upgrade, h2_push, early_hints, stream_timeout については H2_CONFIG_GET を用いて階層的に解決する。 4. push_list は両方に存在する場合は apr_array_append で結合し、片方のみの場合は存在する方を採用する。 5. early_headers は両方に存在する場合は apr_table_overlay で結合し、片方のみの場合は存在する方を採用する。 6. 生成した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c:53（Apache の設定マージフックとして登録）。
  • **呼び出し先**:
  • apr_pcalloc, apr_pstrcat, apr_array_append, apr_table_overlay (APR API)
  • H2_CONFIG_GET (マクロ)

関連データ構造

  • h2_dir_config: ディレクトリ単位の設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc 等の APR 関数がプロセスを終了させる可能性がある。
  • basev または addv が NULL の場合の挙動については、ソースコード上での明示的なガードは確認できない。

保守担当者向けメモ

  • 本関数は Apache の設定フェーズで呼び出されるため、パフォーマンスよりも設定の正確な継承が優先される。
  • push_list や early_headers のマージには APR のテーブル/配列操作関数を使用しており、これらはメモリプールに依存している。
