timeout_sec

定義

  • 種別: function
  • 定義箇所: modules/md/md_curl.c:189

目的

apr_time_t 形式のタイムアウト値を、libcurl が要求する秒単位（long）の整数値に変換する。

シグネチャまたは形状

static long timeout_sec(apr_time_t timeout)

入力と出力

  • **入力**: apr_time_t timeout（マイクロ秒単位のタイムアウト値）
  • **出力**: long（秒単位のタイムアウト値）

前提条件と不変条件

  • 入力値が 0 の場合、出力も 0 となる。
  • 入力値が 0 より大きい場合、apr_time_sec による変換結果が 0 であっても、最小値として 1 が返される。

アルゴリズムまたは振る舞い

1. apr_time_sec(timeout) を呼び出し、マイクロ秒を秒に変換する。 2. 変換結果が 0 以外であれば、その値をそのまま返す。 3. 変換結果が 0 であり、かつ元の timeout が 0 でない場合（1マイクロ秒以上の場合）、切り上げとして 1 を返す。 4. 元の timeout が 0 であれば、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_curl.c:305 において、curl_easy_setopt の CURLOPT_LOW_SPEED_TIME オプション設定時に使用されている。
  • **呼び出し先**: apr_time_sec（APR ライブラリ関数）。

関連データ構造

  • apr_time_t: APR で定義される時間表現（マイクロ秒単位）。

失敗モードと境界条件

  • 入力が非常に小さい値（1マイクロ秒以上）の場合、0 ではなく 1 秒として扱われる。
  • long 型の範囲を超える大きな値が入力された場合の挙動は、apr_time_sec の実装および環境の long 型サイズに依存する。

保守担当者向けメモ

  • 本関数は libcurl のオプション設定における「0 は無効（または無限）」という仕様と、APR の時間計算の差異を吸収するために設計されている。
  • CURLOPT_LOW_SPEED_TIME 等の秒単位指定が必要な libcurl オプション専用のユーティリティであるため、他の用途への転用には注意すること。
