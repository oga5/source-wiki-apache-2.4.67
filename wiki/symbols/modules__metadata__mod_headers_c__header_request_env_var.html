header_request_env_var

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_headers.c:197

目的

HTTP リクエストに関連付けられた環境変数から値を取得し、ヘッダー値として利用可能な形式に変換して返します。

シグネチャまたは形状

static const char *header_request_env_var(request_rec *r, char *a)
  • r: 現在のリクエスト情報を保持する request_rec 構造体へのポインタ。
  • a: 取得対象となる環境変数名。
  • 戻り値: 環境変数の値（変換済み）、または環境変数が存在しない場合は "(null)"。

入力と出力

  • **入力**: r->subprocess_env テーブルから、引数 a をキーとして値を検索します。
  • **出力**:
  • 環境変数が存在する場合: unwrap_header を介して処理された文字列を返します。
  • 環境変数が存在しない場合: 文字列 "(null)" を返します。

前提条件と不変条件

  • r は有効な request_rec 構造体を指している必要があります。
  • unwrap_header は、メモリ割り当てに r->pool を使用します。

アルゴリズムまたは振る舞い

1. apr_table_get を使用し、r->subprocess_env から環境変数 a の値を取得します。 2. 値が見つかった場合、unwrap_header 関数を呼び出し、その結果を返します。 3. 値が見つからなかった場合、静的文字列 "(null)" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_headers.c:981 にて、register_format_tag_handler("e", header_request_env_var) を通じてフォーマットタグハンドラとして登録されています。
  • **呼び出し先**:
  • apr_table_get (APR)
  • unwrap_header (同ファイル内)

関連データ構造

  • request_rec: リクエスト処理のコンテキストを保持。
  • subprocess_env: request_rec 内の環境変数を格納する apr_table_t。

失敗モードと境界条件

  • 環境変数が未定義の場合、エラーではなく "(null)" を返します。
  • r->subprocess_env が初期化されていない場合、apr_table_get の挙動に依存します。

保守担当者向けメモ

  • 本関数は register_format_tag_handler を介して動的に呼び出されるため、直接的な関数呼び出しの連鎖はソースコード上には現れません。
  • unwrap_header の詳細については、同ファイルの該当シンボル定義を参照してください。
