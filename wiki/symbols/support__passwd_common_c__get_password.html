get_password

定義

  • 種別: function
  • 定義箇所: support/passwd_common.c:120

目的

ユーザーからパスワードを取得し、設定されたソース（標準入力、プロンプト、または検証付きプロンプト）に応じて検証およびメモリへの格納を行う。

シグネチャまたは形状

int get_password(struct passwd_ctx *ctx);

入力と出力

  • **入力**: ctx（passwd_ctx 構造体へのポインタ）。ctx->passwd_src に基づき入力元を決定する。
  • **出力**: 成功時は 0 を返す。失敗時はエラーコード（ERR_GENERAL, ERR_OVERFLOW, ERR_PWMISMATCH）を返す。
  • **副作用**: 取得したパスワード文字列を ctx->passwd に格納（apr_pstrdup を使用）。処理終了後、一時バッファを memset でゼロクリアする。

前提条件と不変条件

  • ctx->pool が有効な APR メモリプールとして初期化されていること。
  • ctx->passwd_src が PW_STDIN, PW_PROMPT_VERIFY, またはそれ以外（標準プロンプト）のいずれかに設定されていること。

アルゴリズムまたは振る舞い

1. ctx->passwd_src の値に応じて分岐する。

  • PW_STDIN: 標準入力からパスワードを読み込み、末尾の改行コード（CR/LF）を除去する。
  • PW_PROMPT_VERIFY: apr_password_get を使用してプロンプトから入力を取得する。
  • その他: apr_password_get でパスワードを2回入力させ、一致を確認する。不一致の場合はエラーを返し、入力バッファをゼロクリアする。

    • 2. 取得したパスワードを ctx->passwd にコピーする。 3. 一時バッファ buf を memset でゼロクリアして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/htdbm.c:411
  • support/htpasswd.c:265
  • support/passwd_common.c:200
  • **呼び出し先**:
  • apr_file_open_stdin, apr_file_read_full, apr_file_close (APR ファイル操作)
  • apr_password_get (APR パスワード取得)
  • apr_pstrdup (APR 文字列複製)
  • apr_psprintf (エラーメッセージ生成)
  • memset, strcmp, strlen (標準 C ライブラリ)

関連データ構造

  • struct passwd_ctx: パスワード取得のコンテキスト（入力元、メモリプール、エラー文字列、パスワード格納先）を保持する。

失敗モードと境界条件

  • **ERR_GENERAL**: 標準入力のオープンに失敗した場合。
  • **ERR_OVERFLOW**: 入力パスワードが MAX_STRING_LEN を超える場合。
  • **ERR_PWMISMATCH**: パスワードの再入力確認で値が一致しない場合。

保守担当者向けメモ

  • 本関数は機密情報を扱うため、エラー発生時や関数終了時に必ず memset によるゼロクリアが行われていることを確認すること。
  • errfile グローバル変数への依存があるため、呼び出し前に apr_file_open_stderr() が適切に初期化されている必要がある。
