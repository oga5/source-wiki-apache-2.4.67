ssl_rand_seed

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_rand.c:41

目的

OpenSSL の擬似乱数生成器（PRNG）に対し、設定された複数のソースからエントロピーを収集・供給し、初期化および強化を行う。

シグネチャまたは形状

int ssl_rand_seed(server_rec *s, apr_pool_t *p, ssl_rsctx_t nCtx, char *prefix)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p: メモリ割り当て用のプール。
  • nCtx: 実行コンテキスト（起動時または接続時）。
  • prefix: ログ出力用の識別文字列。
  • **出力**:
  • int: 供給されたエントロピーの総バイト数。

前提条件と不変条件

  • SSLModConfigRec に乱数シードソースのリストが設定されていること。
  • RAND_seed() が OpenSSL の内部状態を更新可能であること。
  • 処理完了後、RAND_status() によりエントロピーの充足が検証される。

アルゴリズムまたは振る舞い

1. 設定された乱数ソースリストを順次走査し、指定されたコンテキスト（nCtx）に合致するものを処理する。 2. **ソース別処理**:

  • SSL_RSSRC_FILE: 指定ファイルを読み込み ssl_rand_feedfp で供給。
  • SSL_RSSRC_EXEC: 指定コマンドを実行し、その標準出力を ssl_rand_feedfp で供給。
  • SSL_RSSRC_EGD: RAND_egd() を呼び出し、EGD からエントロピーを取得。
  • SSL_RSSRC_BUILTIN: 現在時刻、PID、および実行時のスタックメモリの一部を RAND_seed() に渡す。

    • 3. 供給した総バイト数をログ出力する。 4. RAND_status() が 0（エントロピー不足）を返す場合、警告ログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/mod_ssl.c:597
  • modules/ssl/ssl_engine_init.c:447
  • support/ab.c:1453
  • **呼び出し先**:
  • ssl_rand_feedfp
  • ssl_util_ppopen / ssl_util_ppclose
  • RAND_seed (OpenSSL)
  • RAND_egd (OpenSSL)
  • RAND_status (OpenSSL)
  • ssl_rand_choosenum

関連データ構造

  • SSLModConfigRec: 乱数シード設定を保持するモジュール設定構造体。
  • ssl_randseed_t: 個別のシードソース定義。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • ファイルオープン失敗やコマンド実行失敗時は、該当ソースをスキップして処理を継続する。
  • RAND_egd が -1 を返した場合、そのソースからの供給は行われない。
  • RAND_status() が 0 の場合、エントロピー不足として警告が記録されるが、関数自体は供給したバイト数を返して正常終了する。

保守担当者向けメモ

  • SSL_RSSRC_BUILTIN によるスタックからのデータ取得は補助的な手法であり、これのみに依存すべきではない。
  • 外部コマンド実行（SSL_RSSRC_EXEC）はサーバー起動時間に影響を与える可能性がある。
  • 現代の OpenSSL 環境では OS のエントロピー源が自動利用されるため、本機能の重要性は以前より低下している。
