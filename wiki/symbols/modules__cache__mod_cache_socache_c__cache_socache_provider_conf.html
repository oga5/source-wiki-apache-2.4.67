cache_socache_provider_conf

定義

  • 種別: struct
  • 定義箇所: modules/cache/mod_cache_socache.c:105

目的

mod_cache_socache モジュールにおいて、特定の共有オブジェクトキャッシュ（socache）プロバイダーとそのインスタンス設定を保持するために使用されます。

シグネチャまたは形状

typedef struct cache_socache_provider_conf
{
    const char *args;
    ap_socache_provider_t *socache_provider;
    ap_socache_instance_t *socache_instance;
} cache_socache_provider_conf;

入力と出力

本構造体は設定データとして機能します。

  • args: プロバイダーに渡される設定引数文字列。
  • socache_provider: ロードされた ap_socache_provider_t へのポインタ。
  • socache_instance: 初期化された ap_socache_instance_t へのポインタ。

前提条件と不変条件

  • socache_provider および socache_instance は、サーバーの構成フェーズにおいて適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、mod_cache_socache の設定コンテキスト内で、使用するバックエンドストレージ（shmcb, redis 等）のインターフェースとインスタンスを紐付ける役割を果たします。

呼び出し元と呼び出し先

  • 呼び出し元: modules/cache/mod_cache_socache.c 内の構成解析関数（apr_pcalloc を用いてメモリ確保が行われる箇所など）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ap_socache_provider_t: ap_socache.h で定義されるストレージ抽象化インターフェース。
  • ap_socache_instance_t: ap_socache.h で定義されるプロバイダーの実行時インスタンス。
  • cache_socache_conf: サーバー単位の設定を保持する構造体。

失敗モードと境界条件

  • 構成時にプロバイダーが見つからない、またはインスタンスの初期化に失敗した場合、本構造体のポインタメンバが NULL となる可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cache_socache の設定管理において中心的な役割を果たします。プロバイダーの追加や変更を行う際は、この構造体の初期化フロー（mod_cache_socache.c 内の構成関数）を確認してください。
