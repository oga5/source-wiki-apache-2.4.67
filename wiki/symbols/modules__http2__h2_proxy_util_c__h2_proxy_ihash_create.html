h2_proxy_ihash_create

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_proxy_util.c:79

目的

整数識別子（ID）をキーとして構造体を管理するためのハッシュテーブル（h2_proxy_ihash_t）を生成・初期化します。

シグネチャまたは形状

h2_proxy_ihash_t *h2_proxy_ihash_create(apr_pool_t *pool, size_t offset_of_int)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • offset_of_int: ハッシュのキーとして使用する整数メンバーの、構造体先頭からのオフセット値（offsetof マクロで取得）。
  • **出力**:
  • 初期化済みの h2_proxy_ihash_t 構造体へのポインタ。

前提条件と不変条件

  • pool は有効なメモリプールである必要があります。
  • ハッシュテーブルに格納される構造体は、指定された offset_of_int の位置に整数型のキーを持つ必要があります。
  • ihash という名前のカスタムハッシュ関数（内部定義）が、キーの抽出に使用されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して h2_proxy_ihash_t 構造体分のメモリをゼロ初期化して確保します。 2. apr_hash_make_custom を呼び出し、カスタムハッシュ関数 ihash を指定して内部のハッシュテーブルを生成します。 3. 引数で渡された offset_of_int を構造体の ioff メンバーに保存します。 4. 初期化されたハッシュテーブル構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_session.c:758: ストリーム管理用のハッシュテーブル生成時に呼び出されています。
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_hash_make_custom (APR)

関連データ構造

  • h2_proxy_ihash_t: 本関数で生成されるハッシュ管理用構造体。
  • apr_hash_t: 内部で使用される APR のハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がプロセスを終了させる可能性があります（APR の標準的な挙動）。
  • offset_of_int に誤った値（構造体内の整数メンバー以外の位置）を指定した場合、ハッシュ操作時に不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数は h2_proxy_ihash_t のライフサイクルを pool に依存させます。明示的な解放は不要です。
  • ハッシュのキーとなる整数値は、ハッシュテーブルに追加された後に変更してはいけません。変更が必要な場合は、一度 remove してから add する必要があります。
