ssl_init_ctx_crl

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_init.c:1194

目的

SSL/TLS コンテキスト（SSL_CTX）に対して、証明書失効リスト（CRL）の読み込み場所を設定し、検証モード（リーフ証明書のみ、またはチェーン全体）を適用する。

シグネチャまたは形状

static apr_status_t ssl_init_ctx_crl(server_rec *s,
                                     apr_pool_t *p,
                                     apr_pool_t *ptemp,
                                     modssl_ctx_t *mctx)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p, ptemp: メモリプール。
  • mctx: SSL コンテキスト設定構造体（crl_file, crl_path, crl_check_mask 等を含む）。
  • **出力**:
  • APR_SUCCESS: 正常終了。
  • ssl_die(s) の戻り値: 設定エラー発生時（プロセス終了を伴う）。

前提条件と不変条件

  • mctx->ssl_ctx が有効な SSL_CTX オブジェクトを保持していること。
  • mctx->crl_check_mask が UNSET の場合、内部で SSL_CRLCHECK_NONE に初期化される。

アルゴリズムまたは振る舞い

1. **初期化**: crl_check_mask が未設定であれば SSL_CRLCHECK_NONE を割り当てる。 2. **検証モードの確認**: CRL チェックが有効（LEAF または CHAIN）であるにもかかわらず、crl_file および crl_path が未指定の場合はエラーとして ssl_die を呼び出す。 3. **ストアのロード**: modssl_X509_STORE_load_locations を使用し、指定されたファイルまたはパスから CRL を OpenSSL の X509_STORE に読み込む。失敗時は ssl_die を呼び出す。 4. **フラグ設定**: crl_check_mode に応じて OpenSSL の検証フラグを設定する。

  • SSL_CRLCHECK_LEAF: X509_V_FLAG_CRL_CHECK
  • SSL_CRLCHECK_CHAIN: X509_V_FLAG_CRL_CHECK | X509_V_FLAG_CRL_CHECK_ALL

    • 5. **警告**: CRL の場所が指定されているがチェックモードが有効でない場合、警告ログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:1398（ssl_init_ctx 内から呼び出される）。
  • **呼び出し先**:
  • SSL_CTX_get_cert_store (OpenSSL)
  • modssl_X509_STORE_load_locations (OpenSSL ラッパー)
  • X509_STORE_set_flags (OpenSSL)
  • ap_log_error (Apache API)
  • ssl_die (Apache/mod_ssl)

関連データ構造

  • modssl_ctx_t: SSL 設定を保持する主要構造体。
  • X509_STORE: OpenSSL の証明書・CRL 管理用構造体。

失敗モードと境界条件

  • **設定不整合**: CRL チェックが有効で場所が未指定の場合、または CRL のロードに失敗した場合は ssl_die によりサーバー起動が停止する。
  • **未設定**: CRL チェックが有効でない場合、関数は何もせず APR_SUCCESS を返す。

保守担当者向けメモ

  • 本関数は ssl_init_ctx パイプラインの一部として実行される。
  • crl_check_mask のビット演算により、OpenSSL 側の検証フラグを制御している。
  • OpenSSL のバージョン差異については modssl_X509_STORE_load_locations 等のラッパー関数が吸収している。
