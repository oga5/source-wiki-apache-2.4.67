original_uri

定義

  • 種別: function
  • 定義箇所: server/util_script.c:376

目的

HTTP リクエスト行から元の URI 文字列を抽出します。

シグネチャまたは形状

char *original_uri(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト構造体)
  • **出力**: char * (リクエスト行から抽出された URI 文字列。メモリは r->pool から割り当てられる)

前提条件と不変条件

  • r->the_request が NULL の場合、空の文字列（\0）を返します。
  • r->the_request が存在する場合、HTTP リクエスト行の形式（METHOD URI VERSION）に従って、メソッド後の最初の空白から次の空白までの間を URI とみなして抽出します。

アルゴリズムまたは振る舞い

1. r->the_request が NULL であれば、apr_pcalloc で確保した空文字列を返して終了します。 2. r->the_request を走査し、最初の空白文字までスキップしてメソッド部分を読み飛ばします。 3. 続く空白文字をスキップし、URI の開始位置を特定します。 4. 次の空白文字（または文字列終端）までを URI の範囲として特定します。 5. apr_pstrmemdup を使用し、特定した範囲の文字列を r->pool 上に複製して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_script.c:422 (直接的な参照として確認)
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_isspace (APR)
  • apr_pstrmemdup (APR)

関連データ構造

  • request_rec: リクエストの状態を保持する構造体。the_request メンバおよびメモリプール pool を使用します。

失敗モードと境界条件

  • r->the_request が NULL の場合、安全に空文字列を返します。
  • リクエスト行に空白が含まれない場合、メソッド以降の全文字列を URI とみなして抽出します。

保守担当者向けメモ

  • 本関数は r->the_request の構造に依存しています。HTTP プロトコル解析の仕様変更時には、このパースロジックが影響を受ける可能性があります。
  • メモリ管理は r->pool に依存しているため、呼び出し側で個別の解放は不要です。
