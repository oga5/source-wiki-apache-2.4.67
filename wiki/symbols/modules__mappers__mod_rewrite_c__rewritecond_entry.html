rewritecond_entry

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_rewrite.c:328

目的

RewriteCond ディレクティブによって定義された条件式を保持し、リクエスト処理時にルール適用可否を判定するための設定構造体です。

シグネチャまたは形状

typedef struct {
    char           *input;   /* Input string of RewriteCond   */
    char           *pattern; /* the RegExp pattern string     */
    ap_regex_t     *regexp;  /* the precompiled regexp        */
    ap_expr_info_t *expr;    /* the compiled ap_expr          */
    int             flags;   /* Flags which control the match */
    pattern_type    ptype;   /* pattern type                  */
    int             pskip;   /* back-index to display pattern */
} rewritecond_entry;

入力と出力

  • **入力**: RewriteCond ディレクティブで指定されたテスト文字列（input）および比較対象のパターン（pattern）。
  • **出力**: 構造体内の regexp（正規表現）または expr（ap_expr 式）としてコンパイルされた形式で保持され、評価結果は apply_rewrite_cond 関数を通じて判定に使用されます。

前提条件と不変条件

  • input および pattern は、設定読み込み時に解析され、必要に応じて regexp または expr にコンパイルされます。
  • ptype は、パターンが正規表現か、あるいは文字列比較や数値比較などのどのタイプであるかを識別するために使用されます。

アルゴリズムまたは振る舞い

1. 設定読み込み時に rewritecond_entry の配列が生成されます。 2. リクエスト処理時、apply_rewrite_cond 関数がこの構造体を受け取ります。 3. input 文字列に対して変数展開が行われた後、ptype に応じて regexp または expr を用いたマッチング評価が実行されます。 4. 評価結果に基づき、条件が満たされたかどうかが判定されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_rewrite.c 内のルール適用ロジック（apply_rewrite_list 等）から、条件評価のために参照されます。
  • 設定の初期化および構築時に apr_array_make を介して配列として管理されます。
  • **呼び出し先**:
  • apply_rewrite_cond 関数が本構造体を引数として受け取り、評価を実行します。

関連データ構造

  • rewriterule_entry: 本構造体は RewriteRule に付随する条件として管理されます。
  • rewrite_ctx: 評価時のコンテキスト情報（リクエスト状態やバックリファレンス）を提供します。
  • pattern_type: パターンの種類を定義する列挙型。

失敗モードと境界条件

  • input の変数展開が失敗した場合や、regexp のコンパイルが不正な場合は、条件評価が意図通りに機能しない可能性があります。
  • [N] フラグ等による無限ループの制限（REWRITE_MAX_ROUNDS）が適用される境界条件が存在します。

保守担当者向けメモ

  • 本構造体は mod_rewrite の条件評価の核となるため、フィールドの追加や変更を行う際は apply_rewrite_cond での評価ロジックとの整合性に注意してください。
  • ap_expr への移行が進んでいるため、古い正規表現ベースの評価と新しい式評価の混在に留意してください。
