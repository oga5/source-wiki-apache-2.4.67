append_to_linebuf

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:187

目的

文字列を sed_eval_t の行バッファ（linebuf）に追加し、必要に応じてバッファを再配置した上で、関連する正規表現マッチング用ポインタを同期させる。

シグネチャまたは形状

apr_status_t append_to_linebuf(sed_eval_t *eval, const char* sz, step_vars_storage *step_vars)

入力と出力

  • **入力**:
  • eval: 評価状態を保持する sed_eval_t 構造体へのポインタ。
  • sz: 追加するヌル終端文字列。
  • step_vars: 正規表現マッチングのオフセットを保持する step_vars_storage 構造体へのポインタ（NULL可）。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、メモリ拡張失敗時は APR_ENOMEM 等のステータスコード。

前提条件と不変条件

  • eval->linebuf は apr_pool_t によって管理されており、本関数内で再確保された場合、古いポインタは無効となる。
  • step_vars が提供されている場合、バッファの再確保に伴うアドレスの移動を追跡し、loc1, loc2, locs の各ポインタを新しいメモリ領域へオフセット計算により再同期させる必要がある。

アルゴリズムまたは振る舞い

1. strlen(sz) を用いて追加文字列の長さを取得する。 2. appendmem_to_linebuf を呼び出し、ヌル文字を含む文字列全体を linebuf に追加する。 3. 成功時、eval->lspend をデクリメントし、ヌル文字を指すように調整する。 4. step_vars が指定されており、かつ appendmem_to_linebuf によって eval->linebuf のアドレスが変更された場合、loc1, loc2, locs の各ポインタに対して (新アドレス - 旧アドレス) の差分を適用して更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed1.c:221
  • modules/filters/sed1.c:503
  • modules/filters/sed1.c:828
  • modules/filters/sed1.c:830
  • modules/filters/sed1.c:965
  • **呼び出し先**:

関連データ構造

  • sed_eval_t: 評価中の行バッファや状態を保持するメイン構造体。
  • step_vars_storage: 正規表現マッチングの現在位置を保持する構造体。

失敗モードと境界条件

  • appendmem_to_linebuf がメモリ不足等で失敗した場合、そのエラーコードをそのまま呼び出し元に返す。
  • step_vars が NULL の場合、ポインタの同期処理はスキップされる。

保守担当者向けメモ

  • appendmem_to_linebuf によるバッファ再確保が発生すると、linebuf のベースアドレスが変わるため、step_vars 内のポインタ更新は必須である。
  • 本関数は lspend をヌル文字位置に調整するため、呼び出し後の linebuf は常に正しい文字列として扱える状態が維持される。
