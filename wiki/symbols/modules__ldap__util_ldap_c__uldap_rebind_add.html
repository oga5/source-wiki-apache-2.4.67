uldap_rebind_add

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:94

目的

LDAP 接続に対して再バインド（rebind）処理用のコールバック関数を登録し、LDAP サーバーとのセッション維持を可能にする。

シグネチャまたは形状

static apr_status_t uldap_rebind_add(util_ldap_connection_t *ldc)

入力と出力

  • **入力**: ldc (util_ldap_connection_t *) - 再バインド設定を適用する LDAP 接続構造体。
  • **出力**: APR_SUCCESS を常に返す。

前提条件と不変条件

  • ldc が有効な util_ldap_connection_t 構造体を指していること。
  • ldc->ldap に有効な LDAP ハンドルが格納されていること。

アルゴリズムまたは振る舞い

1. ldap_set_rebind_proc を呼び出し、LDAP ハンドル (ldc->ldap) に対して再バインド用プロシージャ uldap_rebind_proc を関連付ける。 2. その際、コンテキストデータとして ldc 構造体自身を渡す。 3. 処理完了後、APR_SUCCESS を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c:380 にて直接呼び出されている。
  • **呼び出し先**:
  • ldap_set_rebind_proc (LDAP SDK 関数)
  • uldap_rebind_proc (コールバック関数)

関連データ構造

  • util_ldap_connection_t: LDAP 接続状態および再バインドに必要な情報を保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーを返さない設計となっている。
  • 内部で呼び出される ldap_set_rebind_proc が失敗した場合の挙動は、使用している LDAP SDK の実装に依存する。

保守担当者向けメモ

  • modules/ldap/util_ldap.c:106 にて apr_ldap_rebind_add をラップするマクロ定義が存在するが、本関数 uldap_rebind_add は静的関数として定義されており、モジュール内部での再バインド設定の標準化を担っている。
