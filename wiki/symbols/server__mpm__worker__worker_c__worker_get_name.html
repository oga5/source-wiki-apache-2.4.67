worker_get_name

定義

  • 種別: function
  • 定義箇所: server/mpm/worker/worker.c:433

目的

現在のマルチプロセッシングモジュール（MPM）の名称を識別子として返す。

シグネチャまたは形状

static const char *worker_get_name(void)

入力と出力

  • 入力: なし
  • 出力: 文字列リテラル "worker" へのポインタを返す。

前提条件と不変条件

  • 常に静的な文字列 "worker" を返す。
  • 呼び出しによって状態が変化することはない。

アルゴリズムまたは振る舞い

1. 関数が呼び出される。 2. 文字列 "worker" を即座にリターンする。

呼び出し元と呼び出し先

  • 呼び出し元: server/mpm/worker/worker.c:2351 にて、ap_hook_mpm_get_name の引数として登録されている。
  • 呼び出し先: なし。

関連データ構造

  • なし。

失敗モードと境界条件

  • 失敗モード: なし。常に成功する。
  • 境界条件: なし。

保守担当者向けメモ

  • 本関数は ap_hook_mpm_get_name フックを介して Apache HTTP Server のコアに MPM 名を通知するために使用される。
  • MPM 名を変更する必要がある場合は、この戻り値を更新すること。
