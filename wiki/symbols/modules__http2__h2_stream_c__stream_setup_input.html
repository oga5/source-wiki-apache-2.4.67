stream_setup_input

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_stream.c:200

目的

HTTP/2 ストリームにおける入力データ転送用の h2_bucket_beam を初期化します。

シグネチャまたは形状

static void stream_setup_input(h2_stream *stream)

入力と出力

  • **入力**: h2_stream *stream - 初期化対象のストリーム構造体へのポインタ。
  • **出力**: なし（副作用として stream->input が設定されます）。

前提条件と不変条件

  • stream は有効なポインタである必要があります。
  • stream->input_closed が真（true）であることは許容されません（ap_assert により検証）。

アルゴリズムまたは振る舞い

1. **重複チェック**: stream->input が既に存在する場合は、何もせず即座に終了します。 2. **状態検証**: stream->input_closed が偽であることを確認します。 3. **ログ出力**: APLOG_TRACE2 レベルで「setup input beam」というメッセージを記録します。 4. **ビーム生成**: h2_beam_create を呼び出し、以下のパラメータで入力用ビームを生成します。

  • stream->input: 生成されたビームを格納。
  • stream->session->c1: 接続コンテキスト。
  • stream->pool: ストリームのメモリプール。
  • stream->id: ストリーム ID。
  • "input": ビームの識別名。
  • 0: フラグ。
  • stream->session->s->timeout: タイムアウト値。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_stream.c:216
  • modules/http2/h2_stream.c:250
  • modules/http2/h2_stream.c:260
  • **呼び出し先**:
  • ap_assert
  • ap_log_cerror
  • h2_beam_create

関連データ構造

  • h2_stream: ストリームのコンテキストを保持する構造体。
  • h2_bucket_beam: ストリーム間でのデータ受け渡しに使用される同期機構。

失敗モードと境界条件

  • stream->input_closed が真の場合、ap_assert によりプログラムが異常終了します。
  • メモリ不足等の理由で h2_beam_create が失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は、ストリームがデータを受信する準備が整った際に呼び出されるべき初期化ルーチンです。
  • h2_beam_create に渡されるタイムアウト値は、セッションのサーバー設定から取得されています。
