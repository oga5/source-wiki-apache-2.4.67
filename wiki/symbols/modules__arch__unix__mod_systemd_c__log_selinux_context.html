log_selinux_context

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_systemd.c:53

目的

SELinux が有効な環境において、現在のプロセスが実行されているセキュリティコンテキストを Apache のエラーログに通知します。

シグネチャまたは形状

static void log_selinux_context(void)

入力と出力

  • 入力: なし（内部で is_selinux_enabled および getcon を使用）
  • 出力: なし（副作用としてログ出力およびメモリ解放）

前提条件と不変条件

  • HAVE_SELINUX が定義されている環境でコンパイルされていること。
  • getcon によって取得されたコンテキスト文字列は、使用後に必ず freecon で解放される必要があります。

アルゴリズムまたは振る舞い

1. is_selinux_enabled() を呼び出し、システムで SELinux が有効か確認します。 2. getcon(&con) を呼び出し、現在のプロセスのセキュリティコンテキストを取得します。 3. 取得に成功した場合、ap_log_error を使用して APLOG_NOTICE レベルでコンテキスト文字列をログに出力します。 4. freecon(con) を呼び出し、取得したコンテキスト文字列のメモリを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/arch/unix/mod_systemd.c:80 (直接の呼び出しを確認)
  • 呼び出し先:
  • is_selinux_enabled() (SELinux ライブラリ)
  • getcon() (SELinux ライブラリ)
  • ap_log_error() (Apache HTTP Server API)
  • freecon() (SELinux ライブラリ)

関連データ構造

  • なし

失敗モードと境界条件

  • is_selinux_enabled() が偽を返す場合、ログ出力は行われません。
  • getcon() が失敗した場合（戻り値が 0 以外）、ログ出力および freecon() はスキップされます。

保守担当者向けメモ

  • 本関数は mod_systemd.c 内の内部ヘルパーです。
  • con ポインタは getcon で割り当てられるため、freecon による解放が必須です。メモリリークを防ぐため、このパターンを維持してください。
