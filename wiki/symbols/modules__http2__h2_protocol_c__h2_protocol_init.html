h2_protocol_init

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_protocol.c:423

目的

HTTP/2 プロトコル関連の初期化処理を実行します。主に、TLS 暗号スイートのブラックリスト構築をトリガーする役割を担います。

シグネチャまたは形状

apr_status_t h2_protocol_init(apr_pool_t *pool, server_rec *s)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • s: サーバー設定を保持する server_rec。
  • **出力**:
  • APR_SUCCESS: 初期化が正常に完了した場合に返されます。

前提条件と不変条件

  • cipher_init(pool) が呼び出され、内部的な暗号スイートのブラックリストがハッシュテーブルとして構築されます。
  • pool は cipher_init 内でのメモリ確保に利用されます。

アルゴリズムまたは振る舞い

1. pool 引数を受け取りますが、関数内では使用されません（void キャスト）。 2. APLOG_TRACE1 レベルでログを出力し、初期化の開始を記録します。 3. cipher_init(pool) を呼び出し、RFC 7540 で禁止されている脆弱な暗号スイートのブラックリストを初期化します。 4. APR_SUCCESS を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/mod_http2.c:140: モジュールの初期化プロセスにおいて呼び出されます。
  • **呼び出し先**:
  • cipher_init(pool): ブラックリスト構築のために呼び出されます。

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • server_rec: Apache HTTP Server のサーバー設定構造体。

失敗モードと境界条件

  • 現在のソースコード上では、cipher_init の戻り値に関わらず常に APR_SUCCESS を返す実装となっています。

保守担当者向けメモ

  • 本関数は mod_http2 の初期化時に一度だけ実行されることを想定しています。
  • 新しい脆弱な暗号スイートが発見された場合は、cipher_init 側のブラックリスト定義を更新してください。
