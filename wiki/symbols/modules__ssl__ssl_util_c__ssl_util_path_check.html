ssl_util_path_check

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util.c:175

目的

指定されたファイルパスに対して、属性（存在、種類、サイズ）の検証を行う。

シグネチャまたは形状

BOOL ssl_util_path_check(ssl_pathcheck_t pcm, const char *path, apr_pool_t *p)

入力と出力

  • **入力**:
  • pcm: 検証フラグ（ssl_pathcheck_t 型のビットマスク）。
  • path: 検証対象のファイルパス文字列。
  • p: apr_pool_t 型のメモリプール。
  • **出力**:
  • TRUE: すべての指定された条件を満たした場合。
  • FALSE: パスがNULL、存在しない、または指定された属性条件を満たさない場合。

前提条件と不変条件

  • path が NULL の場合は常に FALSE を返す。
  • SSL_PCM_EXISTS フラグが指定されている場合、apr_stat によるファイル情報の取得が成功しなければならない。
  • SSL_PCM_EXISTS が指定されていない場合、SSL_PCM_ISREG、SSL_PCM_ISDIR、SSL_PCM_ISNONZERO のいずれのフラグも指定してはならない（AP_DEBUG_ASSERT による検証）。

アルゴリズムまたは振る舞い

1. path が NULL なら即座に FALSE を返す。 2. SSL_PCM_EXISTS が指定されている場合、apr_stat を呼び出し、ファイル情報を取得する。失敗すれば FALSE を返す。 3. SSL_PCM_ISREG が指定されている場合、finfo.filetype が APR_REG であるかを確認する。 4. SSL_PCM_ISDIR が指定されている場合、finfo.filetype が APR_DIR であるかを確認する。 5. SSL_PCM_ISNONZERO が指定されている場合、finfo.size が 0 より大きいかを確認する。 6. 上記の条件をすべて通過した場合に TRUE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/ssl_engine_config.c (複数箇所で設定値の検証に使用)
  • **呼び出し先**:
  • apr_stat (APRライブラリ)

関連データ構造

  • apr_finfo_t: ファイル情報を格納するAPR構造体。
  • ssl_pathcheck_t: 検証フラグを保持するビットマスク型。

失敗モードと境界条件

  • apr_stat が失敗した場合（ファイルが存在しない、アクセス権がない等）、SSL_PCM_EXISTS が指定されていれば FALSE を返す。
  • SSL_PCM_ISNONZERO 指定時にファイルサイズが 0 の場合は FALSE を返す。
  • AP_DEBUG_ASSERT により、論理的に矛盾するフラグの組み合わせ（存在チェックなしでの属性チェック）はデバッグビルド時に停止する。

保守担当者向けメモ

  • 本関数は apr_stat を使用しているため、ファイルシステムへのI/Oが発生する。頻繁に呼び出されるパスでの使用には注意が必要。
  • ssl_private.h にてプロトタイプ宣言されている。
