get_suexec_id_doer

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_suexec.c:82

目的

リクエスト処理時に、設定されたディレクトリごとの suEXEC 実行ユーザーおよびグループ ID を取得し、呼び出し元へ提供する。

シグネチャまたは形状

static ap_unix_identity_t *get_suexec_id_doer(const request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト情報）
  • **出力**: ap_unix_identity_t * （成功時は設定された UID/GID 構造体へのポインタ、無効または未設定時は NULL）

前提条件と不変条件

  • r->per_dir_config から suexec_module に関連付けられた設定が取得可能であること。
  • suexec_config_t 構造体の active フラグが、設定の有効性を判定する不変条件として機能する。

アルゴリズムまたは振る舞い

1. リクエストのディレクトリ設定から suexec_config_t 構造体を取得する。 2. cfg->active が真であれば、設定済みの cfg->ugid（ap_unix_identity_t 型）のアドレスを返す。 3. cfg->active が偽であれば、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ap_hook_get_suexec_identity を通じて登録されており、Apache のフック機構により呼び出される。
  • modules/generators/mod_suexec.c:126 にてフック登録。
  • modules/mappers/mod_userdir.c:378 にてフック登録。
  • **呼び出し先**:
  • ap_get_module_config （Apache API）を呼び出し、設定情報を取得する。

関連データ構造

  • suexec_config_t: mod_suexec が保持するディレクトリごとの設定構造体。
  • ap_unix_identity_t: UNIX ユーザーおよびグループ ID を保持する構造体。
  • request_rec: Apache のリクエスト情報構造体。

失敗モードと境界条件

  • cfg->active が偽の場合、実行ユーザー情報が提供されないため、呼び出し元は suEXEC を適用しない判断を行う必要がある。
  • 設定が未初期化の場合の挙動については、本関数単体では明示的なエラーハンドリングを行わず、NULL を返すことで対応する。

保守担当者向けメモ

  • 本関数は ap_hook_get_suexec_identity フックのハンドラとして機能する。
  • 実際の権限切り替え処理は本関数ではなく、外部の suexec バイナリが担当する。
  • 戻り値が NULL の場合の挙動は、呼び出し側のモジュール（mod_cgi 等）の実装に依存する。
