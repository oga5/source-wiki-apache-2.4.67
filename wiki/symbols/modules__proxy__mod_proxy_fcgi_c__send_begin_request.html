send_begin_request

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:321

目的

FastCGI プロトコルにおける BEGIN_REQUEST レコードをバックエンドへ送信し、新しいリクエストセッションを開始します。

シグネチャまたは形状

static apr_status_t send_begin_request(proxy_conn_rec *conn, apr_uint16_t request_id)

入力と出力

  • **入力**:
  • conn: バックエンドへの接続状態を保持する proxy_conn_rec 構造体。
  • request_id: FastCGI リクエストを一意に識別するための ID。
  • **出力**:
  • apr_status_t: 送信処理の結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • conn は有効な接続状態である必要があります。
  • FastCGI プロトコルの仕様に基づき、ヘッダとボディの各データ構造が正しく初期化されている必要があります。

アルゴリズムまたは振る舞い

1. ap_fcgi_fill_in_header を使用して、AP_FCGI_BEGIN_REQUEST 型の FastCGI ヘッダを構築します。 2. ap_proxy_connection_reusable を確認し、接続が再利用可能であれば AP_FCGI_KEEP_CONN フラグを付与して ap_fcgi_fill_in_request_body を呼び出します。 3. ヘッダとボディの構造体をそれぞれ ap_fcgi_header_to_array および ap_fcgi_begin_request_body_to_array でバイト配列に変換します。 4. iovec を使用してヘッダとボディを連結し、send_data 関数を介してバックエンドへ送信します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_fcgi.c:1044
  • modules/aaa/mod_authnz_fcgi.c:734
  • **呼び出し先**:
  • ap_fcgi_fill_in_header
  • ap_fcgi_fill_in_request_body
  • ap_proxy_connection_reusable
  • ap_fcgi_header_to_array
  • ap_fcgi_begin_request_body_to_array
  • send_data

関連データ構造

  • proxy_conn_rec: プロキシ接続管理用構造体。
  • ap_fcgi_header: FastCGI プロトコルヘッダ。
  • ap_fcgi_begin_request_body: BEGIN_REQUEST レコードのボディ部。

失敗モードと境界条件

  • send_data が失敗した場合、そのエラーコードがそのまま返されます。
  • request_id が不正な場合、バックエンド側でプロトコルエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は FastCGI の多重化をサポートしておらず、単一の接続に対してリクエストを開始する設計です。
  • send_data はブロッキング I/O を含む可能性があるため、呼び出し時のコンテキストに注意してください。
