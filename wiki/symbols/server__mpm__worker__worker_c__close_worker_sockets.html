close_worker_sockets

定義

  • 種別: function
  • 定義箇所: server/mpm/worker/worker.c:262

目的

ワーカープロセスが保持するすべてのソケットを強制的にクローズし、リソースを解放する。

シグネチャまたは形状

static void close_worker_sockets(void)

入力と出力

  • 入力: なし（グローバル変数 threads_per_child および worker_sockets 配列を参照）
  • 出力: なし（副作用としてソケットのクローズと配列のクリアを行う）

前提条件と不変条件

  • threads_per_child が正しく初期化されていること。
  • worker_sockets 配列が、各ワーカーのソケット状態を保持していること。

アルゴリズムまたは振る舞い

1. 0 から threads_per_child - 1 までのインデックスでループを実行する。 2. 各インデックス i について、worker_sockets[i] が有効（NULL ではない）かを確認する。 3. 有効な場合、apr_socket_close() を呼び出してソケットを閉じる。 4. クローズ後、worker_sockets[i] を NULL に設定し、二重解放や不正な参照を防ぐ。

呼び出し元と呼び出し先

  • 呼び出し元:
  • server/mpm/worker/worker.c:331 (強制終了処理時)
  • server/mpm/event/event.c:645 (強制終了処理時)
  • 呼び出し先:
  • apr_socket_close (APR ライブラリ関数)

関連データ構造

  • worker_sockets: 各ワーカーが使用するソケットを格納する配列。
  • threads_per_child: 子プロセスあたりのスレッド数（ループの終了条件として使用）。

失敗モードと境界条件

  • worker_sockets が初期化されていない場合や、既に閉じられたソケットに対しては、NULL チェックにより安全にスキップされる。
  • apr_socket_close 自体の失敗については、本関数内ではエラーハンドリングを行っていない。

保守担当者向けメモ

  • 本関数は強制終了時に呼び出されることを想定している。
  • event MPM からも参照されているが、これは worker とのアーキテクチャ上の類似性によるものである。
