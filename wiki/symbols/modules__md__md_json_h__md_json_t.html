md_json_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_json.h:29

目的

mod_md モジュールにおいて、JSON データの操作を抽象化・簡素化するための不透明な構造体です。内部で jansson ライブラリを利用し、APR（Apache Portable Runtime）プールと統合された JSON オブジェクトのライフサイクル管理を提供します。

シグネチャまたは形状

typedef struct md_json_t md_json_t;

入力と出力

  • **入力**: 本構造体は、md_json_create 等の生成関数を通じて APR プールと紐付けられます。
  • **出力**: JSON の各ノードへのアクセスや、ファイル・ネットワークバケットへのシリアライズ結果を提供します。

前提条件と不変条件

  • **メモリ管理**: 生成時に渡された APR プールによって管理されます。プールがクリアされると自動的にクリーンアップされます。
  • **不透明性**: 構造体の内部メンバは外部から直接操作せず、提供される API（md_json_get*, md_json_set* 等）を介してアクセスする必要があります。

アルゴリズムまたは振る舞い

  • **ライフサイクル**: md_json_create で生成され、明示的な破棄または関連付けられた APR プールの破棄により解放されます。
  • **データアクセス**: 可変長引数を用いたパス指定により、ネストされた JSON 構造の走査を行います。
  • **自動生成**: md_json_set* 系関数は、指定されたパスが存在しない場合に中間ノードを自動的に作成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md.h: md_to_json, md_from_json, md_to_public_json
  • modules/md/md_acme.c: acmev2_req_init, md_acme_req_body_init, md_acme_get_json, collect_profiles
  • **呼び出し先**:
  • 内部実装として jansson ライブラリを呼び出します。

関連データ構造

  • apr_pool_t: メモリ管理の基盤となる APR プール。
  • md_json_type_t, md_json_fmt_t: JSON の型やフォーマット定義。

失敗モードと境界条件

  • **エラーハンドリング**: 多くの操作関数は apr_status_t を返し、成功時は APR_SUCCESS を返します。
  • **境界条件**: JSON パスの不正や型不一致が発生した場合、エラーが返されます。

保守担当者向けメモ

  • 本構造体は jansson の複雑なメモリ管理を隠蔽するラッパーです。直接的なメンバ操作は避け、md_json.c で定義された API を通じて拡張してください。
