util_ldap_handler

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:158

目的

Apache HTTP Server のハンドラーとして、LDAP キャッシュの統計情報を HTML 形式で出力する。

シグネチャまたは形状

static int util_ldap_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (Apache のステータスコード: OK, DECLINED)

前提条件と不変条件

  • リクエストメソッドが GET であること。
  • リクエストのハンドラー名が ldap-status であること。
  • これらを満たさない場合、本関数は DECLINED を返し、処理を後続のハンドラーへ委譲する。

アルゴリズムまたは振る舞い

1. リクエストメソッドが GET か確認し、異なれば DECLINED を返す。 2. ハンドラー名が ldap-status か確認し、異なれば DECLINED を返す。 3. ldap_module からサーバー設定 util_ldap_state_t を取得する。 4. コンテンツタイプを text/html; charset=ISO-8859-1 に設定する。 5. r->header_only が真であれば、ヘッダーのみ送信して OK を返す。 6. HTML のヘッダーおよびタイトルを出力する。 7. util_ald_cache_display を呼び出し、キャッシュの内容を表示する。 8. OK を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ldap/util_ldap.c 内の util_ldap_register_hooks において、ap_hook_handler に登録されている。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_set_content_type_ex
  • ap_rputs
  • util_ald_cache_display

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • util_ldap_state_t: LDAP モジュールのサーバー設定構造体。

失敗モードと境界条件

  • GET 以外のメソッドや ldap-status 以外のハンドラー名が指定された場合、処理を拒否（DECLINED）する。
  • header_only リクエストの場合、キャッシュ表示処理をスキップする。

保守担当者向けメモ

  • 本関数は ldap-status ハンドラーの表示ロジックを担う。キャッシュ表示の詳細は util_ald_cache_display に依存している。
