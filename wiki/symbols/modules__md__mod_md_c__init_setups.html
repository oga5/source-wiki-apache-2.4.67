init_setups

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md.c:127

目的

モジュール全体のログ出力先となるサーバーレコードを初期化し、プール破棄時のクリーンアップ処理を登録します。

シグネチャまたは形状

static void init_setups(apr_pool_t *p, server_rec *base_server)

入力と出力

  • p: クリーンアップ処理を登録するための apr_pool_t。
  • base_server: モジュール内でログ出力に使用される server_rec 構造体。
  • 出力: なし（副作用としてグローバル変数 log_server への代入およびプールへのクリーンアップ登録を行う）。

前提条件と不変条件

  • base_server は有効なサーバーレコードである必要があります。
  • cleanup_setups 関数が定義されている必要があります。

アルゴリズムまたは振る舞い

1. 引数 base_server をモジュール内の静的変数 log_server に格納します。 2. apr_pool_cleanup_register を呼び出し、プール p が破棄される際に cleanup_setups 関数が実行されるよう登録します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/mod_md.c:882
  • 呼び出し先: cleanup_setups（プール破棄時に実行）

関連データ構造

  • server_rec: Apache HTTP Server のサーバー設定構造体。
  • apr_pool_t: APR のメモリプール。

失敗モードと境界条件

  • apr_pool_cleanup_register がメモリ不足等で失敗した場合の挙動については、APR の仕様に依存します。

保守担当者向けメモ

  • 本関数はモジュールの初期化フェーズで一度だけ呼び出されることを想定しています。
  • log_server はモジュール内のログ出力用グローバル変数であるため、マルチスレッド環境での初期化タイミングに注意してください。
