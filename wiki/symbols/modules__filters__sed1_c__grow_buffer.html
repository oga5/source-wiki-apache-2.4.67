grow_buffer

定義

  • 種別: function
  • 定義箇所: modules/filters/sed1.c:95

目的

バッファの動的拡張を行い、メモリを再確保して既存データをコピーする。

シグネチャまたは形状

apr_status_t grow_buffer(apr_pool_t *pool, char **buffer, char **spend, apr_size_t *cursize, apr_size_t newsize)

入力と出力

  • **入力**:
  • pool: メモリ確保に使用する APR プール。
  • buffer: 拡張対象のバッファへのポインタ。
  • spend: バッファ内の現在位置を示すポインタへのポインタ。
  • cursize: 現在のバッファサイズ（ポインタ経由）。
  • newsize: 要求された最小サイズ。
  • **出力**:
  • 成功時は APR_SUCCESS を返し、buffer、cursize、および spend を更新する。
  • 失敗時は APR_ENOMEM を返す。

前提条件と不変条件

  • newsize が現在の cursize 以下の場合、何もせず APR_SUCCESS を返す。
  • 拡張後のサイズは 4KB 境界にアライメントされる。
  • MAX_BUF_SIZE（8MB）を超える拡張は許可されない。

アルゴリズムまたは振る舞い

1. newsize が現在のサイズの 2 倍未満であれば、効率化のため 2 倍のサイズを確保する。 2. 4KB 境界にアライメント調整を行う。 3. MAX_BUF_SIZE を超える場合は APR_ENOMEM を返す。 4. apr_pcalloc で新しいバッファを確保する。 5. 既存のデータ（*cursize 分）を新しいバッファへコピーする。 6. *buffer を新しいバッファで更新し、*cursize を新しいサイズに更新する。 7. spend が buffer と異なる場合、相対オフセットを維持して *spend を再計算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/sed1.c 内の grow_line_buffer、grow_hold_buffer、grow_gen_buffer から呼び出される。
  • **呼び出し先**:
  • apr_pcalloc (APR ライブラリ)
  • memcpy (標準 C ライブラリ)

関連データ構造

  • sed_eval_t: 評価コンテキスト内で linebuf、holdbuf、genbuf の管理に使用される。

失敗モードと境界条件

  • newsize > MAX_BUF_SIZE: APR_ENOMEM を返す。
  • メモリ確保失敗: apr_pcalloc の挙動に依存（通常はアボートまたはエラー）。

保守担当者向けメモ

  • spend ポインタの更新ロジックは、バッファ再確保後のポインタのズレを補正するために必須である。
  • MAX_BUF_SIZE は libsed の設計上の制約であり、これを超える行データは処理できない。
