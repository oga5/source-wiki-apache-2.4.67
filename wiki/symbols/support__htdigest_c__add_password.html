add_password

定義

  • 種別: function
  • 定義箇所: support/htdigest.c:132

目的

ユーザー名とレルムに対応するパスワードを対話的に取得し、MD5 ハッシュを生成して指定されたファイルストリームに user:realm:hash 形式で書き込む。

シグネチャまたは形状

static void add_password(const char *user, const char *realm, apr_file_t *f)

入力と出力

  • **入力**:
  • user: ユーザー名（文字列）
  • realm: レルム名（文字列）
  • f: 書き込み対象の apr_file_t ストリーム
  • **出力**: なし（副作用としてファイルストリームへの書き込みおよび標準エラー出力へのメッセージ出力を行う）

前提条件と不変条件

  • MAX_STRING_LEN (256) を超えるパスワードは入力できない。
  • 入力されたパスワードと再入力されたパスワードが一致する必要がある。
  • APR 環境が初期化されていること。

アルゴリズムまたは振る舞い

1. apr_password_get を使用してパスワードを標準入力から取得する。 2. パスワードの再入力を求め、初回入力値と一致するか検証する。不一致の場合はエラーメッセージを表示し、cleanup_tempfile_and_exit(1) を呼び出して終了する。 3. user:realm:password という形式の文字列を構築する。 4. APR の MD5 API (apr_md5_init, apr_md5_update, apr_md5_final) を使用して、構築した文字列の MD5 ハッシュを計算する。 5. user:realm: に続けて、計算した 16 バイトのハッシュ値を 16 進数文字列としてファイルストリームに書き込む。

関連データ構造

  • apr_file_t: ファイル操作用ハンドル
  • apr_md5_ctx_t: MD5 ハッシュ計算用コンテキスト

失敗モードと境界条件

  • **パスワード長超過**: apr_password_get が APR_SUCCESS を返さない場合、エラーメッセージを出力し終了コード 5 で終了する。
  • **パスワード不一致**: 入力値が一致しない場合、エラーメッセージを出力し終了コード 1 で終了する。
  • **文字列制限**: MAX_LINE_LEN (768) および MAX_STRING_LEN (256) による物理的な長さ制限がある。

保守担当者向けメモ

  • EBCDIC 環境対応のコードが含まれているが、現代の環境では影響しない。
  • パスワードのハッシュ化には MD5 が使用されている。
  • ファイルへの書き込みはアトミックではなく、呼び出し元で管理される一時ファイル経由の処理に依存している。
