proxy_ajp_canon

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ajp.c:30

目的

ajp:// スキームの URL を解析・正規化し、Apache の内部的なプロキシリクエスト形式（proxy:ajp://...）へ変換します。

シグネチャまたは形状

static int proxy_ajp_canon(request_rec *r, char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • url: 正規化対象の URL 文字列。
  • **出力**:
  • OK: 成功時。r->filename に正規化された内部 URL が格納されます。
  • DECLINED: ajp: スキームではない場合。
  • HTTP_BAD_REQUEST: URL 解析やエンコード処理に失敗した場合。
  • HTTP_FORBIDDEN: パスやクエリ文字列に不正な制御文字が含まれる場合。

前提条件と不変条件

  • url は ajp: で始まる必要があります。
  • r->pool を使用してメモリを割り当てます。
  • proxy-nocanon または proxy-noencode ノートの設定により、パスのエンコード処理がスキップされる場合があります。

アルゴリズムまたは振る舞い

1. **スキーム確認**: ajp: 接頭辞を確認し、以降の文字列を処理対象とします。 2. **ネットワーク位置の解析**: ap_proxy_canon_netloc を使用してホスト名とポートを抽出します。 3. **パスとクエリの処理**:

  • proxy-nocanon 設定時は生のパスを使用します。
  • proxy-noencode 設定時はエンコード済みのパスと r->args を使用します。
  • 通常時は ap_proxy_canonenc_ex を使用してパスをエンコードします。

    • 4. **バリデーション**: パスおよびクエリ文字列に制御文字やスペースが含まれていないか検証します。 5. **URL 再構築**: ホスト（IPv6 の場合はブラケットで囲む）、ポート、パス、クエリを結合し、proxy:ajp:// 形式の文字列を r->filename に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: proxy_hook_canon_handler に登録されており、Apache のリクエスト処理フェーズで呼び出されます。
  • **呼び出し先**: ap_proxy_port_of_scheme, ap_proxy_canon_netloc, ap_proxy_canonenc_ex, ap_scan_vchar_obstext 等の mod_proxy ユーティリティ関数。

関連データ構造

  • request_rec: リクエスト情報およびメモリプール (r->pool) を保持。
  • core_dir_config: ディレクトリ単位の設定（エンコードフラグの判定に使用）。

失敗モードと境界条件

  • **解析エラー**: ap_proxy_canon_netloc が失敗した場合、HTTP_BAD_REQUEST を返します。
  • **不正文字**: パスやクエリに制御文字が含まれる場合、HTTP_FORBIDDEN を返します。
  • **エンコード失敗**: ap_proxy_canonenc_ex が NULL を返した場合、HTTP_BAD_REQUEST を返します。

保守担当者向けメモ

  • 本関数は mod_proxy のカノニカライザとして機能します。
  • IPv6 アドレスのホスト名指定時にブラケットを付与する処理が含まれています。
  • proxy-nocanon 等のノート設定による挙動の変化に注意してください。
