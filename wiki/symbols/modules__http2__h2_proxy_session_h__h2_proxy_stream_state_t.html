h2_proxy_stream_state_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2_proxy_session.h:27

目的

バックエンドサーバーとの HTTP/2 通信における、個別のストリームのライフサイクル状態を追跡するために使用されます。

シグネチャまたは形状

typedef enum {
    H2_STREAM_ST_IDLE,
    H2_STREAM_ST_OPEN,
    H2_STREAM_ST_RESV_LOCAL,
    H2_STREAM_ST_RESV_REMOTE,
    H2_STREAM_ST_CLOSED_INPUT,
    H2_STREAM_ST_CLOSED_OUTPUT,
    H2_STREAM_ST_CLOSED,
} h2_proxy_stream_state_t;

入力と出力

  • 入力: 該当なし（列挙型定義）。
  • 出力: 該当なし。

前提条件と不変条件

  • HTTP/2 プロトコル（RFC 7540）で定義されるストリーム状態遷移に準拠することが期待されます。
  • 状態は h2_proxy_session 内のストリーム管理において、現在の通信フェーズを特定するために参照されます。

アルゴリズムまたは振る舞い

  • ストリームの生成から終了までの各段階（アイドル、オープン、予約、入力/出力のクローズ、完全クローズ）を識別します。
  • プロキシセッションの処理ループにおいて、特定のストリームがどのフェーズにあるかを判定するフラグとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_proxy_session.c において、ストリーム状態を保持する変数として参照されています。
  • 呼び出し先: 該当なし。

関連データ構造

  • h2_proxy_session: 本列挙型を使用してストリームの状態を管理するメイン構造体。

失敗モードと境界条件

  • 状態遷移の整合性: プロトコル仕様に反する状態遷移（例: CLOSED から OPEN への直接遷移など）が発生した場合の挙動については、本定義のみでは規定されていません。

保守担当者向けメモ

  • 本列挙型は nghttp2 のストリーム状態と密接に関連しています。状態遷移ロジックを修正する際は、modules/http2/h2_proxy_session.c 内のストリーム処理ロジックと併せて確認してください。
