am_conf_t

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_allowmethods.c:46

目的

ディレクトリ単位での HTTP メソッド制限設定を保持するための構造体です。AllowMethods ディレクティブによって指定された許可メソッドのリストをビットマスクとして管理します。

シグネチャまたは形状

typedef struct am_conf_t {
    int allowed_set;
    apr_int64_t allowed;
} am_conf_t;

入力と出力

  • allowed_set: 設定が有効化されているかを示すフラグ（入力）。
  • allowed: 許可されたメソッドを表現するビットマスク（入力および出力）。

前提条件と不変条件

  • allowed フィールドは、Apache のメソッド番号に対応するビットマスクとして使用されます。
  • allowed_set が設定されていない場合、本モジュールはリクエストの検証を行わず DECLINED を返します。

アルゴリズムまたは振る舞い

1. **設定生成**: am_create_conf および am_merge_conf により、ディレクトリごとの設定としてメモリが割り当てられ、初期化されます。 2. **メソッド管理**: am_allowmethods ディレクティブハンドラが、設定ファイルから読み取ったメソッド名をビットマスクに変換し、allowed フィールドに格納します。 3. **アクセス検証**: am_check_access がリクエスト受信時に本構造体を参照し、現在のメソッドが allowed ビットマスクに含まれているかを検証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_allowmethods.c 内の各関数（am_check_access, am_create_conf, am_merge_conf, am_allowmethods）が、設定の取得・生成・マージのために本構造体を利用しています。
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • per_dir_config: Apache のリクエスト処理において、ディレクトリごとの設定を取得するために ap_get_module_config を介して本構造体が格納されます。

失敗モードと境界条件

  • AllowMethods reset が指定された場合、制限が無効化され、すべてのメソッドが許可される状態となります。
  • 許可リストにないメソッドがリクエストされた場合、am_check_access は HTTP_METHOD_NOT_ALLOWED を返し、リクエストを拒否します。

保守担当者向けメモ

  • 本構造体は mod_allowmethods.c 内でのみ使用されます。
  • apr_int64_t を使用しているため、最大 64 種類のメソッドまで管理可能です。
  • 設定の継承（マージ）ロジックは am_merge_conf に依存しているため、設定の優先順位を変更する際は当該関数を確認してください。
