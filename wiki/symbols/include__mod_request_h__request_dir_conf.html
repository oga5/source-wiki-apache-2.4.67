request_dir_conf

定義

  • 種別: struct
  • 定義箇所: include/mod_request.h:50

目的

mod_request モジュールにおいて、ディレクトリ単位でリクエストボディの保持（Keep）に関する設定を管理するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_off_t keep_body;
    int keep_body_set;
} request_dir_conf;

入力と出力

  • **入力**: KeepBody ディレクティブを通じて設定値が読み込まれます。
  • **出力**: mod_request モジュールのフィルタ（KEEP_BODY_FILTER 等）が、リクエストボディを保持する際の最大サイズ制限として本構造体の値を利用します。

前提条件と不変条件

  • keep_body は保持するボディの最大サイズ（バイト単位）を保持します。
  • keep_body_set は、設定が明示的に行われたかどうかを示すフラグとして機能します。

アルゴリズムまたは振る舞い

1. 設定フェーズで KeepBody ディレクティブが解析される際、本構造体に値が格納されます。 2. リクエスト処理中、ap_get_module_config を介して per_dir_config から本構造体が取得されます。 3. フィルタチェーンにおいて、保持すべきボディサイズが keep_body を超えないよう制御されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_request.c 内の各関数が、設定の取得や初期化のために本構造体を参照・生成しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • request_rec: Apache のリクエスト構造体。per_dir_config を介して本構造体と関連付けられます。
  • apr_off_t: ファイルサイズやオフセットを扱うための APR 型。

失敗モードと境界条件

  • keep_body に負の値や不適切なサイズが設定された場合の挙動は、mod_request.c 側の実装に依存します。
  • メモリ割り当て失敗時、apr_pcalloc を使用している箇所でプロセスが停止する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_request の内部設定用です。変更を加える際は、modules/filters/mod_request.c における設定マージ処理（merge_request_dir_conf 等）への影響を確認してください。
