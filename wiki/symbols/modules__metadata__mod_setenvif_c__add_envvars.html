add_envvars

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_setenvif.c:249

目的

設定ファイルから環境変数の定義式を解析し、sei_entry 構造体の features テーブルに格納する。

シグネチャまたは形状

static const char *add_envvars(cmd_parms *cmd, const char *args, sei_entry *new)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • args: 解析対象の文字列（環境変数定義のリスト）。
  • new: 環境変数を格納する対象の sei_entry 構造体。
  • **出力**:
  • 成功時は NULL を返す。
  • 失敗時はエラーメッセージ（const char *）を返す。

前提条件と不変条件

  • new->features は初期化済みの apr_table_t であること。
  • args は ap_getword_conf で分割可能な形式であること。

アルゴリズムまたは振る舞い

1. args を空白区切りで順次読み込み、各トークンを処理する。 2. 各トークンを = で分割し、変数名と値に分離する。 3. 値が存在する場合、apr_table_setn を使用してテーブルに登録する。 4. 変数名が ! で始まる場合、その変数を削除対象（値として "!" を設定）として登録する。 5. 値が指定されていない場合、デフォルト値として "1" を設定する。 6. トークンが一つも読み込まれなかった場合はエラーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/metadata/mod_setenvif.c:401
  • modules/metadata/mod_setenvif.c:459
  • **呼び出し先**:
  • ap_getword_conf (Apache API)
  • ap_getword (Apache API)
  • apr_table_setn (APR API)
  • apr_pstrcat (APR API)

関連データ構造

  • sei_entry: 環境変数設定ルールを保持する構造体。
  • cmd_parms: 設定コマンドのコンテキスト。

失敗モードと境界条件

  • 環境変数の式が指定されていない場合、"Missing envariable expression for ..." というエラーメッセージを返す。
  • メモリ割り当ては cmd->pool を使用するため、設定読み込みフェーズのメモリ管理に従う。

保守担当者向けメモ

  • 本関数は SetEnvIf ディレクティブの引数解析を担う。
  • apr_table_setn を使用しているため、キーと値の文字列は設定期間中生存している必要がある（cmd->pool 由来であるため問題ない）。
