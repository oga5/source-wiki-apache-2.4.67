ssl_init_ctx_cipher_suite

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_init.c:1136

目的

SSL コンテキスト（SSL_CTX）に対して、許可される暗号スイートを設定します。

シグネチャまたは形状

static apr_status_t ssl_init_ctx_cipher_suite(server_rec *s,
                                              apr_pool_t *p,
                                              apr_pool_t *ptemp,
                                              modssl_ctx_t *mctx)

入力と出力

  • **入力**:
  • s: サーバーレコード。
  • p: 長期生存プール。
  • ptemp: 一時プール。
  • mctx: SSL コンテキスト設定情報を含む modssl_ctx_t 構造体。
  • **出力**:
  • 成功時は APR_SUCCESS を返します。
  • 失敗時は ssl_die(s) を呼び出し、プロセスを終了させます。

前提条件と不変条件

  • mctx->ssl_ctx が有効な SSL_CTX オブジェクトを保持している必要があります。
  • mctx->auth.cipher_suite が設定されていない場合、SSL_DEFAULT_CIPHER_LIST に !aNULL:!eNULL:!EXP を付加したデフォルト値が使用されます。

アルゴリズムまたは振る舞い

1. 暗号スイート文字列を決定します（設定値がない場合はデフォルト値を使用）。 2. SSL_CTX_set_cipher_list を呼び出し、OpenSSL コンテキストに暗号スイートを適用します。失敗した場合はエラーログを出力し、ssl_die で終了します。 3. SSL_HAVE_PROTOCOL_TLSV1_3 が定義されている場合、mctx->auth.tls13_ciphers が存在すれば SSL_CTX_set_ciphersuites を呼び出し、TLS 1.3 用の暗号スイートを設定します。失敗した場合は同様に終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:1394 （ssl_init_ctx 内から呼び出されます）。
  • **呼び出し先**:
  • SSL_CTX_set_cipher_list (OpenSSL API)
  • SSL_CTX_set_ciphersuites (OpenSSL API, 条件付き)
  • ap_log_error (Apache API)
  • ssl_die (Apache/mod_ssl API)

関連データ構造

  • modssl_ctx_t: SSL 設定を保持する内部構造体。
  • SSL_CTX: OpenSSL の SSL コンテキスト構造体。

失敗モードと境界条件

  • 暗号スイートの文字列が OpenSSL によって無効と判断された場合、ssl_die が呼び出され、サーバーの起動が停止します。
  • TLS 1.3 の暗号スイート設定が失敗した場合も同様にサーバーの起動が停止します。

保守担当者向けメモ

  • 本関数は OpenSSL のバージョン差異を吸収するため、SSL_HAVE_PROTOCOL_TLSV1_3 マクロを使用して TLS 1.3 の設定を条件分岐させています。
  • !aNULL:!eNULL:!EXP という制限は、セキュリティ強化のためにデフォルトで強制的に適用されます。
