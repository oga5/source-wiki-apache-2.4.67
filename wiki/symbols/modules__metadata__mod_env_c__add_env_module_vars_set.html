add_env_module_vars_set

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_env.c:111

目的

Apache HTTP Server の SetEnv ディレクティブを処理し、ディレクトリごとの設定構造体に環境変数を登録する。

シグネチャまたは形状

static const char *add_env_module_vars_set(cmd_parms *cmd, void *sconf_,
                                           const char *name, const char *value)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • sconf_: env_dir_config_rec 型のディレクトリ設定構造体。
  • name: 設定する環境変数名。
  • value: 環境変数の値（省略時は空文字列として扱われる）。
  • **出力**:
  • 成功時は NULL を返す。

前提条件と不変条件

  • sconf_ は env_dir_config_rec 構造体へのポインタであること。
  • name は必須であり、value は任意である。

アルゴリズムまたは振る舞い

1. name に = が含まれているか確認する。含まれている場合、警告ログを出力し、正しい使用法を提示する。 2. value が指定されていない場合は、空文字列 "" をデフォルト値として使用する。 3. apr_table_setn を使用し、sconf->vars テーブルに環境変数名と値を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_env.c:153 の AP_INIT_TAKE12 マクロにより、SetEnv ディレクティブのハンドラとして登録されている。
  • **呼び出し先**:
  • ap_strchr_c (文字列検索)
  • apr_strtok, apr_pstrdup (文字列操作)
  • ap_log_error (ログ出力)
  • apr_table_setn (テーブル操作)

関連データ構造

  • env_dir_config_rec: 環境変数を保持するテーブル vars を管理する構造体。
  • cmd_parms: 設定コマンドのコンテキスト情報。

失敗モードと境界条件

  • name に = が含まれる場合、処理は中断されず警告ログが出力されるのみで、そのままテーブル登録が継続される。
  • value が NULL の場合、空文字列として登録される。

保守担当者向けメモ

  • 本関数は SetEnv ディレクティブの解析時にのみ実行される。
  • apr_table_setn を使用しているため、キーと値の寿命は呼び出し元が保証する必要がある（Apache の設定プールにより管理される）。
