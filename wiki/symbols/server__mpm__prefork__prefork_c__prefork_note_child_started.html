prefork_note_child_started

定義

  • 種別: function
  • 定義箇所: server/mpm/prefork/prefork.c:209

目的

子プロセスの起動をスコアボードに記録し、関連するフックを呼び出して状態更新を通知する。

シグネチャまたは形状

static void prefork_note_child_started(int slot, pid_t pid)

入力と出力

  • **入力**:
  • slot: スコアボード内のプロセススロット番号（int）。
  • pid: 起動した子プロセスのプロセスID（pid_t）。
  • **出力**: なし。

前提条件と不変条件

  • retained グローバル変数が初期化されていること。
  • ap_scoreboard_image が有効な共有メモリを指していること。

アルゴリズムまたは振る舞い

1. retained->mpm->my_generation から現在の世代番号を取得する。 2. スコアボードの指定された slot に、プロセスID (pid) と世代番号 (gen) を書き込む。 3. ap_run_child_status を呼び出し、MPM_CHILD_STARTED ステータスを通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/prefork/prefork.c:704
  • server/mpm/prefork/prefork.c:776
  • **呼び出し先**:
  • ap_run_child_status (Apache MPM フック関数)

関連データ構造

  • retained: プロセス間および再起動を跨いで保持される prefork_retained_data 構造体。
  • ap_scoreboard_image: プロセス状態を管理する共有メモリ上のスコアボード構造体。

失敗モードと境界条件

  • slot がスコアボードの範囲外である場合、メモリ破壊を引き起こす可能性がある（境界チェックは本関数内には存在しない）。

保守担当者向けメモ

  • 本関数は子プロセスのライフサイクル管理において、プロセスが正常に起動したことをシステム全体に伝播させる重要な役割を担う。
  • ap_run_child_status は外部モジュールがフックする可能性があるため、本関数内での処理順序や引数の整合性は厳密に維持する必要がある。
