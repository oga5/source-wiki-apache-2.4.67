fcgi_check_password

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:935

目的

FastCGI プロトコルを使用して外部サーバーに対してユーザーのパスワード認証を行い、その結果を Apache の認証フレームワークに反映させる。

シグネチャまたは形状

static authn_status fcgi_check_password(request_rec *r, const char *user, const char *password)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報。r->notes からプロバイダー名を取得する。
  • user: 認証対象のユーザー名。
  • password: 認証に使用するパスワード。
  • **出力**:
  • authn_status: 認証結果（AUTH_GRANTED, AUTH_DENIED, AUTH_GENERAL_ERROR のいずれか）。

前提条件と不変条件

  • r->notes に AUTHN_PROVIDER_NAME_NOTE キーでプロバイダー名が設定されている必要がある。
  • fcgi_authn_providers ハッシュテーブルに、指定されたプロバイダー名に対応する設定構造体が存在すること。

アルゴリズムまたは振る舞い

1. r->notes からプロバイダー名を取得し、存在しない場合は AUTH_GENERAL_ERROR を返す。 2. fcgi_authn_providers ハッシュテーブルからプロバイダーの設定（fcgi_provider_conf）を取得し、存在しない場合は AUTH_GENERAL_ERROR を返す。 3. req_rsp 関数を呼び出し、FastCGI バックエンドと通信を行う。この際、認可も兼ねる場合は適切なロール文字列を渡す。 4. 通信後の r->status を評価する:

  • HTTP_OK: 認証成功。認可も兼ねる場合は fcgi_request_notes を生成して設定し、AUTH_GRANTED を返す。
  • HTTP_INTERNAL_SERVER_ERROR: サーバーエラーとして AUTH_GENERAL_ERROR を返す。
  • その他: 認証失敗として AUTH_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていないが、fcgi_authn_provider 構造体の初期化時に参照されている（modules/aaa/mod_authnz_fcgi.c:992）。
  • **呼び出し先**:
  • apr_table_get, apr_hash_get, apr_palloc (APR API)
  • ap_log_rerror, APLOGrdebug (Apache Logging)
  • log_provider_info (モジュール内関数)
  • req_rsp (モジュール内関数)
  • ap_set_module_config (Apache API)

関連データ構造

  • request_rec: Apache のリクエスト状態を保持。
  • fcgi_provider_conf: FastCGI プロバイダーの設定情報。
  • fcgi_request_notes: 認証・認可成功時にリクエスト設定へ格納される構造体。

失敗モードと境界条件

  • プロバイダー名が r->notes にない場合: AUTH_GENERAL_ERROR を返し、ログに記録。
  • プロバイダー設定が見つからない場合: AUTH_GENERAL_ERROR を返し、ログに記録。
  • バックエンドとの通信エラー等で HTTP_INTERNAL_SERVER_ERROR が発生した場合: AUTH_GENERAL_ERROR を返す。
  • 認証拒否またはその他のステータス: AUTH_DENIED を返す。

保守担当者向けメモ

  • 本関数は req_rsp を介して外部プロセスと通信するため、バックエンドの応答速度や可用性が認証処理のパフォーマンスに直結する。
  • 認可を兼ねる場合（conf->is_authz が真）の処理フローが含まれているため、認証と認可の分離・統合の変更時には注意が必要。
