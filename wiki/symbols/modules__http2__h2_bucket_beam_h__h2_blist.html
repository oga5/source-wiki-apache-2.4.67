h2_blist

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_bucket_beam.h:40

目的

apr_bucket（データチャンク）のリストを保持するための軽量なリング構造体です。apr_bucket_brigade と同様のバケット管理機能を提供します。

シグネチャまたは形状

typedef struct {
    APR_RING_HEAD(h2_bucket_list, apr_bucket) list;
} h2_blist;

入力と出力

  • 入力: apr_bucket 型の要素をリスト構造として保持します。
  • 出力: リスト内のバケットに対する反復処理や、カウント、クリーンアップ操作を可能にします。

前提条件と不変条件

  • APR_RING マクロを利用したリング構造として実装されています。
  • h2_bucket_beam 構造体の一部として使用される場合、その構造体が保持するロック等の同期メカニズムと組み合わせて利用されることが前提となります。

アルゴリズムまたは振る舞い

  • apr_bucket のリストを管理し、バケットの追加や削除、計数、クリーンアップをサポートします。
  • h2_bucket_beam 内で、送信待ち（buckets_to_send）、消費済み（buckets_consumed）、EOR（buckets_eor）の各状態を管理するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • modules/http2/h2_bucket_beam.c: h2_blist_count および h2_blist_cleanup 関数内で操作されています。
  • modules/http2/h2_bucket_beam.h: h2_bucket_beam 構造体のメンバとして定義されています。
  • 呼び出し先:
  • APR_RING マクロ群（APRライブラリ）を内部的に利用しています。

関連データ構造

  • apr_bucket: リストに格納される個々のデータチャンク。
  • h2_bucket_beam: 本構造体をメンバとして保持し、スレッド間のデータ転送を管理する上位構造体。

失敗モードと境界条件

  • リストが空の状態での操作や、メモリ不足時のバケット割り当て失敗については、APRのバケット管理機能の挙動に依存します。

保守担当者向けメモ

  • 本構造体は h2_bucket_beam の内部状態を管理するための補助的な型です。直接的な外部APIとしての利用よりも、h2_bucket_beam の実装内での利用が主となります。
