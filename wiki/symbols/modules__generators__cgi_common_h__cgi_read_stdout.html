cgi_read_stdout

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:310

目的

CGI スクリプトの標準出力（stdout）をファイルから読み取り、バケット（apr_bucket）へ変換してバケットブリゲードに組み込むための処理を行います。

シグネチャまたは形状

static apr_status_t cgi_read_stdout(apr_bucket *a, apr_file_t *out,
                                    const char **str, apr_size_t *len)

入力と出力

  • **入力**:
  • a: 読み取り対象のバケット。
  • out: スクリプトの標準出力が接続されたファイルハンドル。
  • **出力**:
  • str: 読み取ったデータへのポインタ（成功時）。
  • len: 読み取ったデータのバイト数。
  • 戻り値: apr_status_t 型のステータスコード（APR_SUCCESS, APR_EOF, またはエラーコード）。

前提条件と不変条件

  • a->list を使用してメモリ割り当てが行われます。
  • 読み取り成功時、バケット a はヒープバケットに変換され、そのデータ領域が更新されます。
  • 読み取りデータが存在する場合、後続のバケットとして cgi_bucket_dup が生成され、ブリゲードに挿入されます。

アルゴリズムまたは振る舞い

1. APR_BUCKET_BUFF_SIZE 分のメモリを確保します。 2. apr_file_read を呼び出し、ファイルハンドル out からデータを読み取ります。 3. 読み取りエラーが発生した場合、確保したメモリを解放してエラーを返します。 4. データが読み取れた場合（*len > 0）:

  • apr_bucket_heap_make を使用して、現在のバケット a をヒープバケットに変換します。
  • 確保したバケットの alloc_len を APR_BUCKET_BUFF_SIZE に設定します。
  • APR_BUCKET_INSERT_AFTER を使用し、次の処理のためのバケットを挿入します。

    • 5. データが読み取れなかった場合（*len == 0）:

  • 確保したメモリを解放し、バケットを空のイモータルバケット（apr_bucket_immortal_make）に変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/cgi_common.h:388（cgi_bucket_read 内から呼び出されることが確認されています）。
  • **呼び出し先**: apr_bucket_alloc, apr_file_read, apr_bucket_free, apr_bucket_heap_make, cgi_bucket_dup, APR_BUCKET_INSERT_AFTER, apr_bucket_immortal_make。

関連データ構造

  • apr_bucket: データの断片を保持する構造体。
  • cgi_bucket_data: バケットに関連付けられた CGI 固有のメタデータ。
  • apr_bucket_heap: ヒープ領域に割り当てられたバケットデータ。

失敗モードと境界条件

  • apr_file_read が APR_SUCCESS および APR_EOF 以外のステータスを返した場合、関数は即座にエラーを返します。
  • メモリ割り当て失敗時のチェックはソースコード上で XXX とコメントされており、明示的なエラーハンドリングは省略されています。

保守担当者向けメモ

  • 本関数は cgi_bucket_read の一部として機能します。バケットのライフサイクル管理が apr_bucket_heap_make に依存しているため、バケットの生存期間とメモリ管理には注意が必要です。
  • APR_BUCKET_BUFF_SIZE のサイズ変更や、バケットの挿入ロジックを変更する際は、ブリゲード全体の整合性に影響を与える可能性があります。
