jselect_add

定義

  • 種別: function
  • 定義箇所: modules/md/md_json.c:165

目的

JSON オブジェクト内の指定されたパスにある配列に対し、新しい要素を追加します。

シグネチャまたは形状

static apr_status_t jselect_add(json_t *val, md_json_t *json, va_list ap)

入力と出力

  • **入力**:
  • val: 追加する json_t オブジェクト。
  • json: 操作対象の md_json_t コンテナ。
  • ap: パスを指定する可変長引数リスト。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は APR_EINVAL を返します。

前提条件と不変条件

  • ap で指定されるパスは、jselect_parent が解決可能な形式である必要があります。
  • 指定されたパスの末尾にある要素は、既に配列であるか、あるいは存在しない場合に新しく配列として作成可能である必要があります。

アルゴリズムまたは振る舞い

1. jselect_parent を呼び出し、パスの親ノードを取得します。 2. 親ノードが存在しない、またはオブジェクト型でない場合は APR_EINVAL を返します。 3. 指定されたキーに対応する値を取得します。 4. 値が存在しない場合は、新しい JSON 配列を作成し、親オブジェクトにセットします。 5. 取得または作成した値が配列型でない場合は APR_EINVAL を返します。 6. json_array_append を使用して、val を配列の末尾に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_json.c:561 (直接的な参照を確認)
  • **呼び出し先**:
  • jselect_parent (パス解決)
  • json_is_object, json_is_array (型チェック)
  • json_object_get, json_object_set_new (オブジェクト操作)
  • json_array, json_array_append (配列操作)

関連データ構造

  • md_json_t: JSON ツリーと APR プールを管理するラッパー構造体。
  • json_t: Jansson ライブラリの JSON ノード型。

失敗モードと境界条件

  • パスが不正または存在しない場合: APR_EINVAL を返します。
  • パス上のノードがオブジェクト型でない場合: APR_EINVAL を返します。
  • 指定されたキーの値が配列型でない場合: APR_EINVAL を返します。

保守担当者向けメモ

  • 本関数は jselect_parent に依存しており、パス解決のロジックはそちらに集約されています。
  • json_object_set_new を使用しているため、新しく作成された配列の所有権は JSON ツリー側に移転されます。
