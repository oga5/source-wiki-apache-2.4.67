ap_pod_t

定義

  • 種別: struct
  • 定義箇所: include/mpm_common.h:246

目的

「Pipe-of-Death (POD)」を管理するための構造体です。Apache HTTP Server のマルチプロセッシングモジュール（MPM）において、親プロセスから子プロセスへ終了通知を伝達するメカニズムを提供します。

シグネチャまたは形状

typedef struct ap_pod_t ap_pod_t;

※構造体の内部メンバは公開ヘッダー上では不透明（opaque）として扱われます。

入力と出力

  • **入力**: ap_mpm_pod_open 等の関数を通じて、メモリプール（apr_pool_t）から生成・初期化されます。
  • **出力**: 終了通知のシグナル送信や、子プロセス側での終了チェックに使用されます。

前提条件と不変条件

  • 使用前に ap_mpm_pod_open または ap_mpm_podx_open による初期化が必要です。
  • 使用終了後は ap_mpm_pod_close または ap_mpm_podx_close を呼び出し、リソースを解放する必要があります。

アルゴリズムまたは振る舞い

  • **終了通知**: 親プロセスが ap_mpm_pod_signal を呼び出すと、Pipe-of-Death を介して子プロセスに終了を指示します。
  • **監視**: 子プロセスは ap_mpm_pod_check を定期的に実行し、終了通知が届いているかを確認します。
  • **拡張機能**: ap_mpm_podx_* 系関数を用いることで、より詳細な再起動や終了の種別を伴う制御が可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/event/event.c 等の各 MPM 実装が、プロセス管理のために本構造体を参照しています。
  • **呼び出し先**: 本構造体を引数として受け取る以下の関数群が定義されています。
  • ap_mpm_pod_open / ap_mpm_pod_check / ap_mpm_pod_close / ap_mpm_pod_signal / ap_mpm_pod_killpg
  • ap_mpm_podx_open / ap_mpm_podx_check / ap_mpm_podx_close / ap_mpm_podx_signal / ap_mpm_podx_killpg

関連データ構造

  • apr_pool_t: 生成時に使用されるメモリ管理単位。
  • ap_mpm_podx_restart_t: 拡張 POD で使用される再起動/終了種別の列挙型。

失敗モードと境界条件

  • 初期化失敗時、ap_mpm_pod_open は apr_status_t 型のエラーコードを返します。
  • 適切なクローズ処理を行わない場合、ファイルディスクリプタのリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は Unix 系環境でのプロセス間通信を主眼としています。
  • 内部実装は server/mpm_common.c に存在します。
  • ap_mpm_pod_* と ap_mpm_podx_* の使い分けについては、MPM の要件に応じて選択してください。
