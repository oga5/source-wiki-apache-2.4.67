uwsgi_send

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_uwsgi.c:120

目的

uWSGI バックエンドへのデータ送信をラップし、ソケットへの書き込み処理と統計情報の更新を一元管理する。

シグネチャまたは形状

int uwsgi_send(proxy_conn_rec *conn, const char *buf, apr_size_t length, request_rec *r)

入力と出力

  • **入力**:
  • conn: バックエンド接続情報 (proxy_conn_rec)。
  • buf: 送信するデータバッファ。
  • length: 送信するデータのバイト長。
  • r: 現在のリクエスト情報 (request_rec)。
  • **出力**:
  • 成功時は OK を返す。
  • 失敗時は HTTP_SERVICE_UNAVAILABLE を返す。

前提条件と不変条件

  • conn->sock は有効なソケットである必要がある。
  • length が 0 より大きい間、ループ処理により全データの送信を試みる。

アルゴリズムまたは振る舞い

1. length が 0 になるまでループを実行する。 2. apr_socket_send を呼び出し、ソケットへデータを書き込む。 3. 送信に失敗した場合、エラーログを出力し HTTP_SERVICE_UNAVAILABLE を返して終了する。 4. 送信成功時、conn->worker->s->transferred に送信済みバイト数を加算し、統計情報を更新する。 5. 送信済みバイト数分だけバッファポインタを進め、残りの長さを減算してループを継続する。 6. 全データの送信完了後、OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_uwsgi.c:234
  • modules/proxy/mod_proxy_uwsgi.c:247
  • **呼び出し先**:
  • apr_socket_send (APR API)
  • ap_log_rerror (Apache Core API)

関連データ構造

  • proxy_conn_rec: バックエンド接続状態を保持。
  • request_rec: Apache のリクエスト処理コンテキスト。
  • proxy_worker: バックエンドワーカーの統計情報 (s) を保持。

失敗モードと境界条件

  • **ソケットエラー**: apr_socket_send が APR_SUCCESS 以外を返した場合、送信を中断しエラーを報告する。
  • **部分送信**: apr_socket_send が要求された length より少ないバイト数を書き込んだ場合でも、ループにより残りのデータを送信する。

保守担当者向けメモ

  • 本関数はブロッキング I/O を前提としている。
  • 統計情報の更新 (transferred) はワーカー単位で行われるため、マルチスレッド環境下での競合については proxy_worker 側の実装に依存する。
