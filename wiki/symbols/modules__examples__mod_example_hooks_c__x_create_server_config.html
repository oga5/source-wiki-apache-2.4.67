x_create_server_config

定義

  • 種別: function
  • 定義箇所: modules/examples/mod_example_hooks.c:636

目的

Apache HTTP Server のサーバー単位設定構造体（x_cfg）を初期化し、メモリを割り当てる。また、サーバー起動時のトレースログに本関数の実行を記録する。

シグネチャまたは形状

static void *x_create_server_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定データ割り当て用のメモリプール (apr_pool_t*)。
  • s: 設定対象のサーバーレコード (server_rec*)。
  • **出力**:
  • 初期化された x_cfg 構造体へのポインタ（void* として返却）。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • s は有効なサーバーレコードであること。
  • 生成される x_cfg の cmode は常に CONFIG_MODE_SERVER に設定される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、メモリプール p 上に x_cfg 構造体分のメモリをゼロ初期化して確保する。 2. cfg->local および cfg->congenital を 0 に初期化する。 3. cfg->cmode を CONFIG_MODE_SERVER に設定する。 4. s->server_hostname を取得し、SVR(hostname) 形式の文字列を生成して cfg->loc に格納する（ホスト名が NULL の場合は空文字を扱う）。 5. trace_startup を呼び出し、起動トレースに本関数の実行を記録する。 6. 初期化した cfg を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定読み込みプロセス（module 構造体の create_server_config フック経由で呼び出される）。
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_pstrcat (APR)
  • trace_startup (mod_example_hooks.c 内のユーティリティ)

関連データ構造

  • x_cfg: モジュール固有の設定構造体。
  • server_rec: Apache のサーバー設定を保持するコア構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • apr_pcalloc がメモリ不足により失敗した場合、Apache の標準的なメモリ割り当てエラー処理に従い、プロセスが終了する可能性がある。
  • s->server_hostname が NULL の場合、cfg->loc は SVR() となる。

保守担当者向けメモ

  • 本関数は教育用モジュールの一部であり、実運用環境での使用は想定されていない。
  • cfg->loc の生成には apr_pstrcat を使用しており、メモリはリクエストプールではなくサーバーのライフサイクルに関連付けられたプールから割り当てられる点に注意すること。
