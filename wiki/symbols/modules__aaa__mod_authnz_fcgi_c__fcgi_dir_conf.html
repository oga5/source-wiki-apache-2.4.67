fcgi_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authnz_fcgi.c:46

目的

ディレクトリごとの設定情報を保持し、FastCGI 認証・認可プロバイダーの動作（ユーザー名の決定、権限の強制、認証要件など）を制御するために使用されます。

シグネチャまたは形状

typedef struct {
    const char *name;              /* プロバイダー名 */
    const char *default_user;      /* ユーザー式が空の場合のデフォルトユーザー */
    ap_expr_info_t *user_expr;     /* r->user を設定するための式評価情報 */
    char authoritative;            /* ユーザーが拒否された場合にリクエストを失敗させるか */
    char require_basic_auth;       /* クライアントが認証情報を送信しなかった場合に失敗させるか */
} fcgi_dir_conf;

入力と出力

  • 本構造体は、Apache の設定フェーズでディレクトリ単位に割り当てられ、リクエスト処理時に ap_get_module_config を通じて取得されます。
  • 出力として、認証・認可の判定ロジック（fcgi_check_authn 等）に対して、現在のディレクトリにおけるポリシー設定を提供します。

前提条件と不変条件

  • user_expr は ap_expr_info_t 型であり、実行時に評価可能な式である必要があります。
  • authoritative および require_basic_auth はフラグとして機能し、認証・認可の成否判定の厳格さを決定します。

アルゴリズムまたは振る舞い

  • リクエスト処理中、fcgi_dir_conf に格納された設定に基づき、外部 FastCGI サーバーからのレスポンスをどのように解釈し、Apache のリクエスト状態（r->user など）に反映させるかを制御します。
  • default_user は、FastCGI サーバーが成功を返したものの、user_expr の評価結果が空文字列である場合に適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_fcgi.c:809 (ap_get_module_config による取得)
  • modules/aaa/mod_authnz_fcgi.c:1338, 1346, 1347, 1348 (設定の初期化およびマージ処理)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。ただし、保持している user_expr は ap_expr エンジンによって評価されます。

関連データ構造

  • ap_expr_info_t: user_expr フィールドで使用される式評価用構造体。
  • fcgi_provider_conf: バックエンドの接続情報を保持する対となる設定構造体。

失敗モードと境界条件

  • user_expr が正しく設定されていない場合、ユーザー名の抽出に失敗する可能性があります。
  • authoritative が有効な場合、バックエンドがユーザーを拒否すると、他の認証プロバイダーへのフォールスルーを行わずにリクエストが即座に失敗します。

保守担当者向けメモ

  • 本構造体は apr_pcalloc を使用してメモリ確保されるため、リクエストまたはディレクトリのライフサイクルに従って管理されます。
  • 設定のマージ処理（1346-1348 行付近）において、ベース設定とオーバーライド設定のフィールドが正しく引き継がれるよう注意してください。
