check_dbm_pw

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_dbm.c:138

目的

DBMファイルから取得したパスワードハッシュを用いて、提供されたユーザーの認証を行う。

シグネチャまたは形状

static authn_status check_dbm_pw(request_rec *r, const char *user, const char *password)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • user: 認証対象のユーザー名。
  • password: 検証対象のプレーンテキストパスワード。
  • **出力**:
  • authn_status: 認証結果（AUTH_GRANTED, AUTH_DENIED, AUTH_USER_NOT_FOUND, AUTH_GENERAL_ERROR のいずれか）。

前提条件と不変条件

  • r->per_dir_config から authn_dbm_module の設定が取得可能であること。
  • fetch_dbm_value が正常に動作し、DBMファイルへのアクセスが可能であること。

アルゴリズムまたは振る舞い

1. ap_get_module_config を使用して、ディレクトリごとの設定（authn_dbm_config_rec）を取得する。 2. fetch_dbm_value を呼び出し、DBMファイルからユーザーに対応するパスワード文字列を検索する。 3. 検索結果がエラーの場合は AUTH_GENERAL_ERROR を、ユーザーが見つからない場合は AUTH_USER_NOT_FOUND を返す。 4. 取得したパスワード文字列に : が含まれる場合、それ以降を切り捨ててハッシュ部分のみを抽出する。 5. AUTHN_CACHE_STORE マクロを呼び出し、認証結果をキャッシュに保存する。 6. apr_password_validate を使用して、提供されたパスワードとDBM内のハッシュを比較する。 7. 検証結果が成功であれば AUTH_GRANTED を、失敗であれば AUTH_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_dbm.c:206（authn_dbm_provider 構造体を通じて登録）。
  • **呼び出し先**:
  • ap_get_module_config
  • fetch_dbm_value
  • ap_strchr
  • AUTHN_CACHE_STORE
  • apr_password_validate

関連データ構造

  • request_rec: Apacheのリクエスト情報。
  • authn_dbm_config_rec: DBMファイルパスやタイプを保持する設定構造体。
  • authn_status: 認証結果の列挙型。

失敗モードと境界条件

  • DBMファイルが開けない、または読み取りエラーが発生した場合は AUTH_GENERAL_ERROR を返す。
  • ユーザー名がDBMファイル内に存在しない場合は AUTH_USER_NOT_FOUND を返す。
  • パスワードが一致しない場合は AUTH_DENIED を返す。

保守担当者向けメモ

  • 本関数は mod_authn_dbm の認証プロバイダーインターフェースの一部です。
  • AUTHN_CACHE_STORE はキャッシュモジュールが有効な場合にのみ機能します。
  • パスワード文字列の : 以降を切り捨てる処理は、DBM内のデータ形式（user:hash）に依存しています。
