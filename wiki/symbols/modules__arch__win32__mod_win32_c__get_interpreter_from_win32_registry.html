get_interpreter_from_win32_registry

定義

  • 種別: function
  • 定義箇所: modules/arch/win32/mod_win32.c:141

目的

Windowsレジストリを走査し、指定されたファイル拡張子に関連付けられたスクリプトインタープリタの実行パスを取得します。

シグネチャまたは形状

static char* get_interpreter_from_win32_registry(apr_pool_t *p, const char* ext, int strict);

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • ext: 検索対象のファイル拡張子（例: .pl）。
  • strict: 真の場合、ExecCGI 動詞のみを検索対象とします。偽の場合は Open 動詞へのフォールバックを許可します。
  • **出力**:
  • 成功時: インタープリタの実行パスを含む文字列（apr_pool_t 上に割り当て）。
  • 失敗時: NULL。

前提条件と不変条件

  • ext が NULL の場合は直ちに NULL を返します。
  • 検索結果の文字列は、リクエスト終了時に自動解放されるメモリプール上に確保されます。

アルゴリズムまたは振る舞い

1. AP_REGKEY_CLASSES_ROOT を起点に、拡張子に対応するキーを開きます。 2. 拡張子に関連付けられたファイルタイプ名を取得します。 3. 以下の順序でレジストリキーを探索し、コマンドパスを取得します：

  • 1) ファイルタイプ名キーの SHELL\EXECCGI\COMMAND
  • 2) 拡張子キーの SHELL\EXECCGI\COMMAND
  • 3) strict が偽の場合、ファイルタイプ名キーの SHELL\OPEN\COMMAND
  • 4) strict が偽の場合、拡張子キーの SHELL\OPEN\COMMAND

    • 4. 取得したパスが空でないことを確認し、成功すればその文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_win32.c:419（ap_cgi_build_command 内から呼び出されることを確認）。
  • **呼び出し先**: ap_regkey_open, ap_regkey_value_get, ap_regkey_close（APRのレジストリ操作関数群）。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • ap_regkey_t: レジストリキーのハンドル。

失敗モードと境界条件

  • レジストリキーが存在しない、またはアクセス権限がない場合は NULL を返します。
  • strict モードで ExecCGI が見つからない場合、あるいは strict が偽で ExecCGI と Open の両方が見つからない場合は NULL を返します。
  • 取得したレジストリ値が空文字列である場合は失敗とみなされます。

保守担当者向けメモ

  • レジストリ検索はコストが高いため、将来的な最適化として、検索結果を拡張子ハッシュにキャッシュする実装が推奨されています。
  • 現在のコードでは、name_key および type_key のクローズ処理が条件分岐ごとに適切に行われるよう設計されています。変更時はリソースリークに注意してください。
