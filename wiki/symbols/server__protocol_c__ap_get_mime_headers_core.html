ap_get_mime_headers_core

定義

  • 種別: function
  • 定義箇所: server/protocol.c:1089

目的

HTTP リクエストの MIME ヘッダーを読み取り、解析し、request_rec 構造体の headers_in テーブルへ格納する。

シグネチャまたは形状

void ap_get_mime_headers_core(request_rec *r, apr_bucket_brigade *bb)

入力と出力

  • **入力**:
  • r: リクエスト情報を保持する request_rec 構造体。
  • bb: 入力データを供給する apr_bucket_brigade。
  • **出力**:
  • なし（r->headers_in を更新し、エラー発生時は r->status を設定する）。

前提条件と不変条件

  • r->server の設定（limit_req_fieldsize 等）に基づき、ヘッダーサイズや個数の制限が適用される。
  • http_conformance 設定により、RFC 7230 準拠の厳格な解析またはレガシーな解析が選択される。
  • 処理終了後、headers_in は apr_table_compress により重複フィールドがマージされる。

アルゴリズムまたは振る舞い

1. ap_rgetline を用いてヘッダー行を順次読み取る。 2. 行頭が空白（タブまたはスペース）で始まる場合、前の行の継続（obs-fold）とみなし、バッファを拡張して連結する。 3. 通常のヘッダー行の場合、: を区切り文字としてフィールド名と値に分離する。 4. 厳格モードでは ap_scan_http_token 等で構文を検証し、非厳格モードではレガシーな解析を行う。 5. 解析したヘッダーを r->headers_in に追加する。 6. 空行（ヘッダー終了）を検出するまでループし、最後に重複ヘッダーの圧縮とサイズ制限の再チェックを行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/protocol.c:1352
  • server/protocol.c:1478
  • **呼び出し先**:
  • ap_rgetline (ヘッダー行の取得)
  • ap_scan_http_token / ap_scan_http_field_content (厳格モード時の構文解析)
  • apr_table_addn / apr_table_compress (ヘッダーテーブル操作)
  • ap_log_rerror (エラーログ出力)

関連データ構造

  • request_rec: リクエスト状態とヘッダーテーブル (headers_in) を保持。
  • apr_bucket_brigade: 入力ストリームのバッファ。
  • core_server_config: HTTP 準拠設定や制限値の保持。

失敗モードと境界条件

  • **サイズ超過**: APR_ENOSPC または limit_req_fieldsize 超過時に HTTP_BAD_REQUEST を設定。
  • **ヘッダー数超過**: limit_req_fields を超えた場合に HTTP_BAD_REQUEST を設定。
  • **構文エラー**: 不正な文字や : の欠落、空のフィールド名などが検出された場合に HTTP_BAD_REQUEST を設定。
  • **タイムアウト**: APR_STATUS_IS_TIMEUP 時に HTTP_REQUEST_TIME_OUT を設定。

保守担当者向けメモ

  • 本関数は RFC 7230 への準拠を重視している。修正を行う際は、strict モードとレガシーモードの両方の挙動に注意すること。
  • メモリ割り当てには r->pool を使用しており、リクエスト終了時に自動解放される。
  • obs-fold（行の折り返し）の処理は、メモリ効率を考慮して二乗ベースのバッファ拡張を行っている。
