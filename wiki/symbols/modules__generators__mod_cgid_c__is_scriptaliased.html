is_scriptaliased

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_cgid.c:125

目的

リクエストが ScriptAlias によって強制的に CGI スクリプトとして扱われているかどうかを判定します。

シグネチャまたは形状

static int is_scriptaliased(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在の HTTP リクエスト構造体)
  • **出力**: int 型。条件を満たす場合は非ゼロ（真）、それ以外は 0（偽）を返します。

前提条件と不変条件

  • r->notes テーブル内に "alias-forced-type" というキーが存在し、その値が "cgi-script" である場合にのみ真を返します。
  • 大文字小文字を区別しない比較 (strcasecmp) が行われます。

アルゴリズムまたは振る舞い

1. r->notes テーブルから "alias-forced-type" に対応する値を取得します。 2. 値が存在し、かつその文字列が "cgi-script" と一致するかを判定します。 3. 判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:541
  • modules/generators/mod_cgid.c:1507
  • **呼び出し先**:
  • apr_table_get (APR ライブラリ)
  • strcasecmp (標準 C ライブラリ)

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する主要な構造体。notes フィールドを介してリクエストごとのメタデータにアクセスします。

失敗モードと境界条件

  • r->notes が NULL の場合や、指定されたキーが存在しない場合は 0 を返します。
  • apr_table_get が NULL を返した場合、strcasecmp は実行されず安全に 0 が返されます。

保守担当者向けメモ

  • 本関数は mod_cgi と mod_cgid の両方で同様のロジックとして利用されています。
  • mod_vhost_alias.c のコメントにおいて、本関数が mod_cgi の実装を参照している旨が言及されています。
