systemd_pre_config

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_systemd.c:42

目的

Apache HTTP Server の設定読み込み開始を systemd に通知し、サーバーの稼働状況監視を有効化します。

シグネチャまたは形状

static int systemd_pre_config(apr_pool_t *pconf, apr_pool_t *plog, apr_pool_t *ptemp)

入力と出力

  • **入力**:
  • pconf: 設定用メモリプール。
  • plog: ログ用メモリプール。
  • ptemp: 一時的な割り当て用メモリプール。
  • **出力**:
  • 戻り値: OK (常に成功を返す)。

前提条件と不変条件

  • 本関数は ap_hook_pre_config を介して呼び出されることを前提としています。
  • sd_notify を通じて systemd に通知を行うため、実行環境で systemd が稼働している必要があります。

アルゴリズムまたは振る舞い

1. sd_notify を呼び出し、systemd に対して以下の通知を行います。

  • RELOADING=1: 設定の再読み込み中であることを伝達。
  • STATUS=Reading configuration...: 現在のステータスを「設定読み込み中」に更新。

    • 2. グローバル変数 ap_extended_status を 1 に設定し、サーバーの拡張ステータス収集を強制的に有効化します。 3. OK を返して処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache の設定読み込みフェーズにおいて、ap_hook_pre_config により登録されたフックとして実行されます（modules/arch/unix/mod_systemd.c:162）。
  • **呼び出し先**: sd_notify（外部ライブラリ libsystemd の関数）。

関連データ構造

  • apr_pool_t: Apache のメモリ管理用プール。
  • ap_extended_status: サーバーの拡張統計情報の有効/無効を制御するグローバルフラグ。

失敗モードと境界条件

  • sd_notify の戻り値は確認されておらず、通知が失敗した場合でも処理は中断されず OK を返します。

保守担当者向けメモ

  • 本関数は設定読み込みの初期段階で実行されます。ap_extended_status を強制的に有効化するため、他のモジュールや設定による意図しない動作変更がないか注意してください。
  • sd_notify に渡される文字列は、systemd のプロトコル仕様に従った改行区切りの形式です。
