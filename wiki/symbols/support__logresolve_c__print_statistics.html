print_statistics

定義

  • 種別: function
  • 定義箇所: support/logresolve.c:92

目的

ログ処理の実行結果に関する統計情報を、指定されたファイルストリームへ出力する。

シグネチャまたは形状

static void print_statistics(apr_file_t *output)

入力と出力

  • **入力**: output (apr_file_t *) - 統計情報を書き出す対象のファイルハンドル。
  • **出力**: なし（戻り値は void）。指定されたファイルへ統計テキストを書き出す。

前提条件と不変条件

  • output は有効な apr_file_t ポインタであること。
  • グローバル変数 entries, withname, resolves, noreverse, doublefailed, cachehits, cachesize がプログラムの実行過程で適切に更新されていること。

アルゴリズムまたは振る舞い

1. logresolve Statistics: というヘッダーを出力する。 2. 処理されたエントリ数、名前解決成功数、解決試行数を順次出力する。 3. noreverse が真の場合、逆引き無効化のカウントを出力する。 4. doublefailed が真の場合、ダブルルックアップ失敗のカウントを出力する。 5. 最後にキャッシュヒット数と現在のキャッシュサイズを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: support/logresolve.c:324 (main 関数内)
  • **呼び出し先**: apr_file_printf (APR ライブラリ関数)

関連データ構造

  • apr_file_t: APR のファイル I/O 抽象化構造体。
  • グローバル変数群: 統計情報を保持する整数型変数（entries, withname, resolves, noreverse, doublefailed, cachehits, cachesize）。

失敗モードと境界条件

  • output が無効な場合、apr_file_printf の内部でエラーが発生する可能性がある。
  • 統計値が 0 の場合でも、条件分岐（noreverse, doublefailed）に該当しない限り、その項目は出力されない。

保守担当者向けメモ

  • 本関数は logresolve の終了処理の一部として呼び出される。
  • 新たな統計項目を追加する場合は、グローバル変数の定義と本関数の出力ロジックの両方を更新する必要がある。
