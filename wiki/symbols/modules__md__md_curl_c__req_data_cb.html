req_data_cb

定義

  • 種別: function
  • 定義箇所: modules/md/md_curl.c:63

目的

libcurl の CURLOPT_READFUNCTION コールバックとして機能し、apr_bucket_brigade に保持された送信データを読み取って libcurl のバッファへ転送します。

シグネチャまたは形状

size_t req_data_cb(void *data, size_t len, size_t nmemb, void *baton)
  • data: libcurl から提供される書き込み先バッファ。
  • len, nmemb: 書き込み可能な最大サイズ（len * nmemb）。
  • baton: apr_bucket_brigade へのポインタ。

入力と出力

  • **入力**: baton を通じて渡される apr_bucket_brigade 内のデータ。
  • **出力**: 実際に書き込まれたバイト数。エラー発生時は CURL_READFUNC_ABORT を返します。

前提条件と不変条件

  • baton は有効な apr_bucket_brigade を指している必要があります。
  • libcurl のリクエスト処理中に呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. baton から apr_bucket_brigade を取得し、バッファが空になるか、要求されたサイズを満たすまでループします。 2. バケットがメタデータ（EOSなど）の場合はスキップまたは終了処理を行います。 3. データバケットの場合、apr_bucket_read を使用してデータを読み取ります。 4. 読み取ったデータが max_len を超える場合は apr_bucket_split で分割し、memcpy で data バッファへコピーします。 5. 処理済みのバケットは apr_bucket_delete で削除します。 6. EOF 以外の読み取りエラーが発生した場合は CURL_READFUNC_ABORT を返して転送を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: libcurl（CURLOPT_READFUNCTION に設定されたタイミングで内部的に呼び出されます）。
  • **呼び出し先**: apr_bucket_read, apr_bucket_split, apr_bucket_delete（APR Bucket API）。
  • **参照元**: modules/md/md_curl.c:266 にて curl_easy_setopt を通じて登録されています。

関連データ構造

  • apr_bucket_brigade: 送信データのストリーム管理に使用。
  • apr_bucket: データの断片を保持する単位。

失敗モードと境界条件

  • apr_bucket_read が APR_SUCCESS 以外を返した場合、EOF でなければ CURL_READFUNC_ABORT を返し、通信を異常終了させます。
  • バケットが空の場合、ループは直ちに終了し 0 を返します。

保守担当者向けメモ

  • 本関数は libcurl のスレッド非依存なコールバックとして動作するため、apr_bucket_brigade のメモリ管理（プール）がリクエストのライフサイクルと正しく同期していることを確認してください。
