ping_ev_frame_received

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_proxy_session.c:165

目的

バックエンドとの HTTP/2 セッションにおいて、フレーム受信時に PING 関連の状態を更新し、接続の健全性確認プロセスを管理する。

シグネチャまたは形状

static void ping_ev_frame_received(h2_proxy_session *session, const nghttp2_frame *frame)

入力と出力

  • **入力**:
  • session: 状態更新対象となる h2_proxy_session 構造体へのポインタ。
  • frame: 受信した nghttp2_frame 構造体へのポインタ。
  • **出力**: なし（副作用として session の状態を更新）。

前提条件と不変条件

  • session は有効な h2_proxy_session 構造体を指している必要がある。
  • frame は nghttp2 から渡された有効なフレームであること。

アルゴリズムまたは振る舞い

1. session->last_frame_received を現在の時刻（apr_time_now()）で更新する。 2. session->ping_state に基づき以下の処理を行う：

  • H2_PING_ST_NONE: 何もしない。
  • H2_PING_ST_AWAIT_ANY: 任意のフレームを受信したため、enter_ping_state を呼び出し状態を H2_PING_ST_NONE に遷移させる。
  • H2_PING_ST_AWAIT_PING: 受信したフレームが NGHTTP2_PING 型である場合のみ、enter_ping_state を呼び出し状態を H2_PING_ST_NONE に遷移させる。それ以外のフレームは無視する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_proxy_session.c:275 (直接的な呼び出しを確認)
  • **呼び出し先**: enter_ping_state (状態遷移用関数)

関連データ構造

  • h2_proxy_session: セッションの状態（ping_state）および最終受信時刻を保持する構造体。
  • nghttp2_frame: 受信した HTTP/2 フレームの型情報（hd.type）を保持。

失敗モードと境界条件

  • H2_PING_ST_AWAIT_PING 状態において、PING 以外のフレームを受信した場合は状態遷移が発生せず、待機状態が維持される。これはバックエンドからの応答待ち中に他の通信が発生する可能性を考慮した仕様である。

保守担当者向けメモ

  • 本関数は nghttp2 のフレーム受信イベントハンドラの一部として機能する。
  • session->last_frame_received の更新は、PING 状態に関わらず全てのフレーム受信時に行われる。
