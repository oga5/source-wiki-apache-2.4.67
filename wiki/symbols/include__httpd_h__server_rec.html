server_rec

定義

  • 種別: struct
  • 定義箇所: include/httpd.h:818

目的

Apache HTTP Server において、仮想ホストごとの設定情報およびサーバーの状態を保持するための中心的なデータ構造です。

シグネチャまたは形状

typedef struct server_rec server_rec;

入力と出力

  • 本構造体は、サーバーのライフサイクル全体およびリクエスト処理の各フェーズにおいて、設定コンテキストとして参照されます。
  • 構造体内部には、タイムアウト設定、サーバー名、スキーム、コンテキスト情報などが含まれます（詳細は ap_mmn.h の変更履歴を参照）。

前提条件と不変条件

  • サーバーの起動時に初期化され、設定ファイル（httpd.conf 等）の内容に基づいて構築されます。
  • 複数の仮想ホストが存在する場合、各ホストに対応する server_rec インスタンスが生成されます。

アルゴリズムまたは振る舞い

  • サーバーの構成管理およびリクエスト処理の過程で、モジュールが特定の仮想ホストの設定にアクセスするための基点となります。
  • MPM（マルチプロセッシングモジュール）のフック関数において、サーバー設定の共有や状態通知のために引数として渡されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • include/ap_expr.h: 式評価コンテキスト内での参照。
  • include/ap_listen.h: リスナー設定および複製処理。
  • include/ap_mpm.h: MPM フック（mpm, child_status 等）の引数。
  • **呼び出し先**:
  • 構造体自体はデータ保持を目的としており、直接的な関数呼び出しは行いません。ただし、本構造体を引数に取るフック関数や API 関数が多数存在します。

関連データ構造

  • request_rec: リクエストごとの状態を保持する構造体。
  • conn_rec: クライアントとの接続状態を保持する構造体。
  • process_rec: サーバープロセス全体の情報を保持する構造体。

失敗モードと境界条件

  • 構造体内の各フィールド（タイムアウト値やスキーム等）が適切に設定されていない場合、リクエスト処理や接続管理において予期せぬ動作を引き起こす可能性があります。
  • 構造体は非常に巨大であるため、モジュール開発者は必要なフィールドのみにアクセスし、不必要な変更を避けることが推奨されます。

保守担当者向けメモ

  • 本構造体は Apache HTTP Server のコア部分であり、変更は ap_mmn.h（Module Magic Number）の更新を伴う可能性があります。
  • 構造体の定義は include/httpd.h にありますが、詳細なフィールド構成はサーバーのバージョンやビルド設定に依存します。最新のフィールドについてはソースコードの定義箇所を参照してください。
