setup_store

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md.c:277

目的

Managed Domains (MD) サブシステムで使用するファイルシステムベースの証明書ストアを初期化し、必要なディレクトリ構造を準備する。

シグネチャまたは形状

static apr_status_t setup_store(md_store_t **pstore, md_mod_conf_t *mc, apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • pstore: ストアハンドルを格納するポインタのポインタ。
  • mc: モジュールの設定構造体（ベースディレクトリ情報を含む）。
  • p: メモリ割り当てに使用するプール。
  • s: サーバーレコード。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時はエラーコードを返す。
  • *pstore: 初期化された md_store_t 構造体。

前提条件と不変条件

  • mc->base_dir が有効なパスとして設定されていること。
  • md_store_fs_init がファイルシステムへのアクセス権限を持っていること。

アルゴリズムまたは振る舞い

1. ap_server_root_relative を使用して、設定されたベースディレクトリを絶対パスに変換する。 2. md_store_fs_init を呼び出し、ファイルシステムストアを初期化する。 3. md_store_fs_set_event_cb を呼び出し、ストアのイベントコールバックとして store_file_ev を登録する。 4. check_group_dir を使用して、以下のディレクトリがストア内に存在するか確認（または作成）する：

  • MD_SG_CHALLENGES (チャレンジ用)
  • MD_SG_STAGING (ステージング用)
  • MD_SG_ACCOUNTS (アカウント用)
  • MD_SG_OCSP (OCSP用)

    • 5. いずれかのステップでエラーが発生した場合、エラーログを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c:893
  • **呼び出し先**:
  • ap_server_root_relative
  • md_store_fs_init
  • md_store_fs_set_event_cb
  • check_group_dir
  • ap_log_error

関連データ構造

  • md_store_t: 証明書や設定を永続化するための抽象ストア構造体。
  • md_mod_conf_t: モジュールの設定を保持する構造体。
  • server_rec: Apache のサーバー設定レコード。

失敗モードと境界条件

  • md_store_fs_init が失敗した場合、エラーログ 10046 が出力される。
  • check_group_dir によるディレクトリ確認・作成が失敗した場合、エラーログ 10047 が出力される。
  • いずれの失敗も rv を保持したまま leave ラベルへ遷移し、呼び出し元へエラーを伝播させる。

保守担当者向けメモ

  • 本関数は mod_md の初期化プロセスの一部であり、ファイルシステムへの書き込み権限に依存する。
  • check_group_dir で指定されているグループ（CHALLENGES, STAGING, ACCOUNTS, OCSP）は、md_store の仕様変更時に整合性を確認すること。
