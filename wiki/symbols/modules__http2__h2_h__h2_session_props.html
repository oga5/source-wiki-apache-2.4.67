h2_session_props

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2.h:130

目的

HTTP/2 セッションにおける統計情報、ストリーム ID の追跡、およびセッションのエラー状態を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct h2_session_props {
    int accepted_max;
    int completed_max;
    int emitted_count;
    int emitted_max;
    int error;
    const char *error_msg;
    unsigned int accepting : 1;
    unsigned int shutdown : 1;
} h2_session_props;

入力と出力

本構造体は関数ではなくデータ型であるため、直接的な入力・出力はありません。各フィールドはセッションの進行に伴い、h2_session の管理下で更新されます。

前提条件と不変条件

  • accepting および shutdown はビットフィールドとして定義されており、それぞれ 0 または 1 の値をとります。
  • error_msg はエラー発生時に設定される文字列へのポインタであり、有効なセッション期間中は参照可能である必要があります。

アルゴリズムまたは振る舞い

  • **ストリーム管理**: accepted_max（受信した最大ストリーム ID）や emitted_max（送信した最大ストリーム ID）を追跡し、セッション内のストリームライフサイクルを管理します。
  • **エラー状態**: error フィールドに HTTP/2 エラーコードを保持し、error_msg に詳細なメッセージを格納することで、セッション終了時の診断情報を提供します。
  • **セッション制御**: accepting フラグにより新規ストリームの受け入れ可否を、shutdown フラグにより GOAWAY フレーム送信済みかどうかを判定します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/h2_session.h 内で h2_session 構造体のメンバとして利用されています。
  • h2_session_props local: ローカルセッションのプロパティ
  • h2_session_props remote: リモートセッションのプロパティ
  • 呼び出し先: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • h2_session: 本構造体を保持し、セッション全体のライフサイクルを管理する主要構造体。

失敗モードと境界条件

  • error フィールドが 0 以外の場合、セッションは異常終了状態にあるとみなされます。
  • accepting が 0 の場合、新しいストリームの生成要求は拒否されるべき境界条件となります。

保守担当者向けメモ

  • 本構造体は h2_session.h でローカルおよびリモートの双方のプロパティとしてインスタンス化されています。状態更新ロジックを変更する際は、両者の整合性に注意してください。
  • ビットフィールドを使用しているため、メモリレイアウトに依存する処理を行う場合は注意が必要です。
