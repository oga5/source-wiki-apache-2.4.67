dav_inherit_locks

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:588

目的

親リソースまたは指定されたリソースからロック情報を取得し、対象リソースおよびその配下へ再帰的にロックを伝播（継承）させます。

シグネチャまたは形状

static dav_error * dav_inherit_locks(request_rec *r, dav_lockdb *lockdb,
                                     const dav_resource *resource,
                                     int use_parent)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • lockdb: ロックデータベースのハンドル。
  • resource: ロックを適用する対象リソース。
  • use_parent: 真の場合、親リソースからロックを取得して継承します。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は dav_error 構造体へのポインタを返します。

前提条件と不変条件

  • resource->hooks が有効なリポジトリフックを保持している必要があります。
  • use_parent が真の場合、repos_hooks->get_parent_resource が正常に親リソースを特定できる必要があります。
  • 継承対象のロックリストにおいて、DAV_LOCKREC_DIRECT かつ深さが DAV_INFINITY ではないロックは、継承プロセスから除外されます。

アルゴリズムまたは振る舞い

1. use_parent が真であれば親リソースを取得し、偽であれば対象リソース自身を起点とします。 2. lockdb->hooks->get_locks を呼び出し、起点リソースのロックを取得します。ロックが存在しない場合は即座に終了します。 3. 取得したロックリストを走査し、直接ロック（DAV_LOCKREC_DIRECT）かつ深さが無限（DAV_INFINITY）ではないものをリストから除外します。 4. dav_walker_ctx を初期化し、コールバック関数に dav_inherit_walker を設定します。 5. repos_hooks->walk を呼び出し、DAV_INFINITY の深さで再帰的にロックを伝播させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/dav/main/util_lock.c 内の複数箇所（dav_notify_created 等の文脈で利用）。
  • **呼び出し先**:
  • repos_hooks->get_parent_resource
  • lockdb->hooks->get_locks
  • repos_hooks->walk（コールバックとして dav_inherit_walker を使用）

関連データ構造

  • dav_resource: 操作対象のリソース抽象。
  • dav_lock: ロック情報のリスト構造。
  • dav_walker_ctx: 再帰的なウォーク操作のためのコンテキスト。

失敗モードと境界条件

  • 親リソースの取得に失敗した場合、HTTP_INTERNAL_SERVER_ERROR を含むエラーを返します。
  • ロックデータベースからの取得に失敗した場合、そのエラーを伝播します。
  • 継承すべきロックが存在しない場合、処理は正常終了します。

保守担当者向けメモ

  • 本関数は dav_inherit_walker を介して再帰的にロックを適用します。
  • ### コメントで示されている通り、エラー発生時の詳細な説明（higher-level desc）の追加や、multi_status の適切なハンドリングが今後の課題です。
  • use_parent フラグの制御により、リソース作成時や移動時のロック継承ロジックを共通化しています。
