HSE_URL_MAPEX_INFO

定義

  • 種別: struct
  • 定義箇所: modules/arch/win32/mod_isapi.h:171

目的

ISAPI 拡張において、URL を物理パスにマッピングした結果や、そのマッピングに関する詳細情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct HSE_URL_MAPEX_INFO {
    char         lpszPath[260];
    apr_uint32_t dwFlags;
    apr_uint32_t cchMatchingPath;
    apr_uint32_t cchMatchingURL;
    apr_uint32_t dwReserved1;
    apr_uint32_t dwReserved2;
} HSE_URL_MAPEX_INFO;

入力と出力

本構造体は、ServerSupportFunction を通じてサーバーから ISAPI 拡張へ情報を渡す際、またはその逆のデータ受け渡しに使用されます。

  • lpszPath: マッピングされた物理パス（入力/出力）。
  • dwFlags, cchMatchingPath, cchMatchingURL: マッピングの属性や一致した長さに関する情報。

前提条件と不変条件

  • lpszPath は最大 260 文字（MAX_PATH 相当）のバッファとして確保されています。
  • dwReserved1 および dwReserved2 は将来の拡張用であり、現在の実装では使用されません。

アルゴリズムまたは振る舞い

ISAPI 拡張が ServerSupportFunction を呼び出す際、この構造体へのポインタを介して、URL から物理パスへの変換結果や、パスの照合情報を取得または設定します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/arch/win32/mod_isapi.c 内の ServerSupportFunction 実装において、data_type から本構造体へのキャストが行われています（1240行目）。
  • 呼び出し先: 情報なし。

関連データ構造

  • EXTENSION_CONTROL_BLOCK: ISAPI 拡張のメインコンテキスト。
  • ServerSupportFunction: 本構造体を利用する主要なインターフェース関数。

失敗モードと境界条件

  • lpszPath のバッファサイズ（260文字）を超えるパスは切り詰められるか、エラーの原因となる可能性があります。
  • dwFlags に不正な値が設定された場合の挙動は、呼び出し側の ISAPI 拡張の実装に依存します。

保守担当者向けメモ

  • 本構造体は IIS 5.0 互換の ISAPI 仕様に基づいています。Apache 側の実装において、Windows のパス長制限（MAX_PATH）と整合していることを確認してください。
