NWSSLSrvConfigRec

定義

  • 種別: struct
  • 定義箇所: modules/arch/netware/mod_nw_ssl.c:103

目的

NetWare プラットフォームにおける Apache HTTP Server のサーバーごとの SSL 設定を保持するための構造体です。リスナーテーブルやアップグレード可能ポートの定義など、SSL/TLS 処理に必要な構成情報を管理します。

シグネチャまたは形状

ソースコード上では不完全型として typedef struct NWSSLSrvConfigRec NWSSLSrvConfigRec; と宣言されています。具体的なメンバ構成については、提供されたソース excerpt には含まれていません。

入力と出力

本構造体は、ap_get_module_config を介してサーバー設定から取得され、主に設定値の読み取りや、サーバー間での設定マージ処理に使用されます。

前提条件と不変条件

  • Apache の pool メモリ管理下で割り当てられることが前提です。
  • サーバー設定のライフサイクルに依存しており、モジュールの初期化プロセス中に適切に初期化される必要があります。

アルゴリズムまたは振る舞い

  • get_nwssl_cfg マクロを通じて、nwssl_module の設定コンテキストから本構造体へのポインタが取得されます。
  • 設定の生成時（apr_palloc）や、サーバー設定のマージ処理において、メモリの確保とコピーが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/netware/mod_nw_ssl.c 内の各設定ハンドラおよびフック関数（get_nwssl_cfg を経由）。
  • 設定のマージ処理を行う関数群。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • seclisten_rec: セキュアリスナーの定義を保持するリンクリスト。
  • seclistenup_rec: アップグレード可能リスナーの定義を保持するリンクリスト。
  • nwssl_module: 本構造体を保持する Apache モジュール構造体。

失敗モードと境界条件

  • ap_get_module_config が適切なコンテキストを返さない場合、NULL ポインタ参照が発生する可能性があります。
  • メモリ不足時には apr_palloc が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体のメンバ定義は modules/arch/netware/mod_nw_ssl.c の 126 行目以降に存在します。詳細なフィールド構成を確認する場合は該当箇所を参照してください。
  • NetWare の SSL スタックに強く依存しているため、メンバの追加や変更を行う際は、WinSock IOCTL への影響を考慮する必要があります。
