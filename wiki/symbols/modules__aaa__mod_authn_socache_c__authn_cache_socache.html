authn_cache_socache

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:153

目的

Apache HTTP Server の設定ディレクティブ AuthnCacheSOCache を処理し、認証情報キャッシュ用の共有オブジェクトキャッシュ（socache）プロバイダーを初期化および設定する。

シグネチャまたは形状

static const char *authn_cache_socache(cmd_parms *cmd, void *CFG, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • CFG: ディレクトリ設定構造体（本関数内では未使用）。
  • arg: ディレクティブに指定された引数（形式: name または name:args）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージを含む文字列を返す。

前提条件と不変条件

  • 本関数はグローバルコンテキスト（GLOBAL_ONLY）でのみ呼び出し可能であること。
  • 指定された socache プロバイダーがロード済みであること。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、グローバル設定以外での使用を拒否する。 2. arg を解析し、プロバイダー名（name）とオプション引数（args）に分離する。 3. ap_lookup_provider を使用して、指定された名前の socache プロバイダーを検索する。 4. プロバイダーが見つからない場合はエラーを返す。 5. プロバイダーが見つかった場合、その create 関数を呼び出し、socache_instance を初期化する。 6. 処理中に発生したエラーは、接頭辞 AuthnCacheSOCache: を付与して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_socache.c:251 の AP_INIT_TAKE1 マクロによるディレクティブ登録経由。
  • **呼び出し先**:
  • ap_check_cmd_context
  • ap_strchr_c
  • apr_pstrmemdup
  • ap_lookup_provider
  • socache_provider->create

関連データ構造

  • socache_provider: 選択されたキャッシュバックエンドの実装。
  • socache_instance: 初期化されたキャッシュインスタンス。
  • cmd_parms: 設定処理用のコンテキスト。

失敗モードと境界条件

  • 指定されたプロバイダー名が不明な場合、適切なモジュールがロードされていない旨のエラーを返す。
  • socache_provider->create が失敗した場合、そのエラーを伝播する。
  • グローバル設定以外でディレクティブが使用された場合、エラーを返す。

保守担当者向けメモ

  • 本関数は socache_provider および socache_instance というグローバルな状態変数を更新する。
  • socache_instance の生成には cmd->temp_pool および cmd->pool が使用されるため、メモリライフサイクルに注意すること。
