cern_meta_dir_config

定義

  • **種別**: struct
  • **定義箇所**: modules/metadata/mod_cern_meta.c:137

目的

Apache HTTP Server において、ディレクトリ単位で CERN HTTPD 形式のメタファイル処理を制御するための設定値を保持する構造体です。

シグネチャまたは形状

typedef struct {
    const char *metadir;    /* メタファイルが格納されているディレクトリ名 */
    const char *metasuffix; /* メタファイルのファイルサフィックス */
    int metafiles;          /* メタファイル処理の有効フラグ */
} cern_meta_dir_config;

入力と出力

  • **入力**: 設定ファイル（httpd.conf 等）のディレクティブ（MetaDir, MetaSuffix, MetaFiles）から値が設定されます。
  • **出力**: モジュール内のリクエスト処理関数（add_cern_meta_data 等）が、この構造体を参照してメタファイルのパス構築や処理の要否を判断します。

前提条件と不変条件

  • request_rec のプール (r->pool) を介してメモリが割り当てられることが前提です。
  • 設定値は、create_cern_meta_dir_config による初期化および merge_cern_meta_dir_configs による親ディレクトリからの継承を経て構築されます。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持用であり、ロジックは含みません。
  • add_cern_meta_data 関数がリクエスト処理時に本構造体を参照し、metafiles が有効な場合に限り、metadir と metasuffix を用いて対象メタファイルのパスを生成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_cern_meta_dir_config (初期化)
  • merge_cern_meta_dir_configs (マージ)
  • set_metadir, set_metasuffix, set_metasuffix (設定値のセット)
  • add_cern_meta_data (リクエスト処理時の参照)
  • **呼び出し先**: 構造体であるため、関数呼び出しは発生しません。

関連データ構造

  • request_rec: リクエストごとのコンテキストを保持し、本構造体へのアクセス起点となります。
  • apr_table_t: メタファイルから読み込まれたヘッダー情報を格納するために使用されます。

失敗モードと境界条件

  • 設定が未定義の場合、各フィールドは NULL または 0 となります。
  • metadir や metasuffix が適切に設定されていない場合、メタファイルの検索パス構築に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_cern_meta.c 内でのみ使用されます。
  • 設定の継承ロジック（merge_cern_meta_dir_configs）を変更する際は、各フィールドのデフォルト値の扱い（NULL チェック等）に注意してください。
