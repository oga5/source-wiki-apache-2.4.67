proxyws_fallback_to_proxy_http

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_wstunnel.c:469

目的

ProxyWebsocketFallbackToProxyHttp ディレクティブの設定値を処理し、WebSocket トンネリングが失敗した際に mod_proxy_http へフォールバックするかどうかをディレクトリ設定構造体に反映させる。

シグネチャまたは形状

static const char * proxyws_fallback_to_proxy_http(cmd_parms *cmd, void *conf, int arg)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • conf: proxyws_dir_conf 型のディレクトリ設定構造体。
  • arg: ディレクティブで指定された整数値（真偽値）。
  • **出力**:
  • 成功時は NULL を返す。

前提条件と不変条件

  • conf は proxyws_dir_conf 型としてキャスト可能であること。
  • dconf->fallback_to_proxy_http_set が設定済みであることを示すフラグとして使用される。

アルゴリズムまたは振る舞い

1. 引数 conf を proxyws_dir_conf 型にキャストする。 2. arg を真偽値に変換し、dconf->fallback_to_proxy_http に格納する。 3. dconf->fallback_to_proxy_http_set を 1 に設定し、設定が明示的に行われたことを記録する。 4. NULL を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_wstunnel.c:489 において、Apache のコマンドテーブル定義（command_rec）から間接的に呼び出される。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • proxyws_dir_conf: WebSocket プロキシのディレクトリ単位の設定を保持する構造体。

失敗モードと境界条件

  • conf が NULL の場合、キャストおよびメンバアクセス時にクラッシュする可能性がある（Apache のコマンドハンドラとして呼び出される限り、通常は有効なポインタが渡される）。

保守担当者向けメモ

  • 本関数は ProxyWebsocketFallbackToProxyHttp ディレクティブのハンドラである。
  • fallback_to_proxy_http_set フラグは、設定の継承やマージロジックにおいて、デフォルト値と明示的な設定を区別するために使用される。
