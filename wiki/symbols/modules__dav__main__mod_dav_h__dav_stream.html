dav_stream

定義

  • 種別: struct
  • 定義箇所: modules/dav/main/mod_dav.h:1906

目的

WebDAV リソースに対するストリーム操作（読み込み、書き込み、シーク、クローズ）を抽象化するための不透明な構造体です。

シグネチャまたは形状

typedef struct dav_stream dav_stream;

入力と出力

  • 入力: 本構造体は、リポジトリプロバイダーが提供するストリーム操作関数の引数として渡されます。
  • 出力: ストリーム操作関数（close_stream, write_stream, seek_stream）は、操作の結果として dav_error* を返します。

前提条件と不変条件

  • 本構造体は、特定のプロバイダー（例: mod_dav_fs）によって実体が定義されます。
  • ストリーム操作を行う関数は、本構造体のポインタを介してバックエンドのデータにアクセスします。

アルゴリズムまたは振る舞い

  • mod_dav のコアロジックは、本構造体を直接操作せず、プロバイダーが提供する関数ポインタ（close_stream, write_stream, seek_stream）を介して間接的にストリームを制御します。
  • ストリームのライフサイクルは、プロバイダーの実装に依存します。

呼び出し元と呼び出し先

  • 呼び出し元: mod_dav.c および modules/dav/fs/repos.c 内のストリーム操作関数。
  • 呼び出し先: modules/dav/fs/repos.c 内の dav_fs_close_stream, dav_fs_write_stream, dav_fs_seek_stream 等が本構造体を引数として受け取ります。

関連データ構造

  • dav_hooks_repository: ストリーム操作関数を保持するフック構造体。
  • dav_error: ストリーム操作の結果として返されるエラー構造体。

失敗モードと境界条件

  • ストリーム操作関数が NULL を返す場合、操作は成功とみなされます。
  • ストリーム操作関数が dav_error を返す場合、そのエラーは dav_handle_err() を通じて HTTP レスポンスに変換されます。

保守担当者向けメモ

  • 本構造体は mod_dav.h では不透明な型として宣言されています。具体的なメンバ変数は各プロバイダー（modules/dav/fs/repos.c など）の実装を確認してください。
  • ストリームのクローズ時に commit フラグを渡すことで、変更の確定または破棄を制御する設計となっています。
