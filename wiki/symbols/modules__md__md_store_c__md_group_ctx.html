md_group_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_store.c:174

目的

ストレージ操作のコンテキストを保持し、特定のストレージグループに対する操作をカプセル化するために使用されます。

シグネチャまたは形状

typedef struct {
    md_store_t *store;
    md_store_group_t group;
} md_group_ctx;

入力と出力

  • **入力**: store（md_store_t へのポインタ）、group（md_store_group_t 列挙型）。
  • **出力**: なし（構造体定義）。

前提条件と不変条件

  • store は有効な md_store_t インスタンスを指している必要があります。
  • group は md_store がサポートする有効なストレージグループ（MD_SG_DOMAINS, MD_SG_ACCOUNTS 等）である必要があります。

アルゴリズムまたは振る舞い

本構造体は、ストレージ操作のコールバック関数やイテレータにおいて、操作対象のストレージインスタンスとグループを引数として渡すための「バトン（baton）」として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_store.c 内の関数（md_store_iter 等のイテレータ処理や、グループ指定が必要な操作）において、ローカル変数としてインスタンス化されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • md_store_t: ストレージの抽象インターフェース。
  • md_store_group_t: ストレージグループを定義する列挙型。

失敗モードと境界条件

  • store が NULL の状態で操作に使用された場合、後続の処理でデリファレンスエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は md_store.c 内の内部的なコンテキスト受け渡し用です。外部モジュールから直接操作されることは想定されていません。
  • 構造体のフィールド追加や変更を行う際は、md_store.c 内のすべてのバトン受け渡し箇所（baton 引数を使用する関数）との整合性を確認してください。
