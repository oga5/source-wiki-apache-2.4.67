create_authz_core_svr_config

定義

  • **種別**: function
  • **定義箇所**: modules/aaa/mod_authz_core.c:178

目的

Apache HTTP Server の認可コアモジュールにおける、サーバー単位の設定構造体（authz_core_srv_conf）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_authz_core_svr_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: 設定情報の割り当てに使用する apr_pool_t プール。
  • s: 設定対象となる server_rec 構造体（本関数内では未使用）。
  • **出力**:
  • 初期化された authz_core_srv_conf 構造体へのポインタを void * として返す。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • 戻り値の構造体は apr_pcalloc によりゼロクリアされている。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して authz_core_srv_conf サイズのメモリをプール p から確保する。 2. 認可プロバイダーのエイリアスを管理するためのハッシュテーブルを apr_hash_make(p) で作成し、構造体の alias_rec メンバに格納する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_core.c:1168（サーバー設定の作成時）。
  • **呼び出し先**: apr_pcalloc, apr_hash_make（Apache Portable Runtime）。

関連データ構造

  • authz_core_srv_conf: 本関数が初期化するサーバー設定構造体。
  • apr_hash_t: alias_rec メンバとして使用されるハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がプロセスを終了させる可能性がある（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数はサーバー単位の設定を扱う。ディレクトリ単位の設定については create_authz_core_dir_config が担当する。
  • server_rec *s は現在の実装では使用されていないが、将来的な拡張のためにシグネチャに含まれている。
