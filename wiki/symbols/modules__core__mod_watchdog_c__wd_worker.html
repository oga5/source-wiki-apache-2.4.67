wd_worker

定義

  • 種別: function
  • 定義箇所: modules/core/mod_watchdog.c:107

目的

Watchdog インスタンス専用のワーカースレッドのメインループとして機能し、登録されたコールバック関数の定期実行、状態遷移（STARTING, RUNNING, STOPPING）の管理、およびプロセス間排他制御（シングルトンモード時）を行います。

シグネチャまたは形状

static void* APR_THREAD_FUNC wd_worker(apr_thread_t *thread, void *data)
  • thread: 実行中のスレッドハンドル。
  • data: ap_watchdog_t 構造体へのポインタ。

入力と出力

  • 入力: ap_watchdog_t 構造体（監視対象のコールバックリスト、mutex、状態フラグを含む）。
  • 出力: なし（スレッド終了時に apr_thread_exit を呼び出す）。

前提条件と不変条件

  • w->pool はスレッド生成時に有効であること。
  • シングルトンモードの場合、w->mutex を用いたプロセス間排他制御が成功するまでループが継続されること。
  • MPM の状態が AP_MPMQ_STOPPING になった場合、速やかにループを終了すること。

アルゴリズムまたは振る舞い

1. **初期化**: スレッド開始フラグを立て、シングルトンモードであれば apr_proc_mutex_trylock でロックを獲得するまで待機する。 2. **開始処理**: AP_WATCHDOG_STATE_STARTING 状態で登録済みコールバックを順次実行する。 3. **メインループ**: AP_WD_TM_SLICE（100ms）間隔でループし、以下の処理を行う。

  • MPM の停止状態を監視。
  • 各コールバックの経過時間を計算し、interval を超過していれば AP_WATCHDOG_STATE_RUNNING 状態で実行。
  • コールバック未登録の場合は ap_run_watchdog_step フックを実行。

    • 4. **終了処理**: ループ終了後、AP_WATCHDOG_STATE_STOPPING 状態でコールバックを実行し、ロックを解放してスレッドを終了する。

呼び出し元と呼び出し先

  • 呼び出し元: modules/core/mod_watchdog.c 内の ap_thread_create（wd_startup 等から間接的に起動）。
  • 呼び出し先:
  • ap_mpm_query（MPM 状態確認）
  • apr_proc_mutex_trylock / apr_proc_mutex_unlock（排他制御）
  • wl->callback_fn（登録されたコールバック関数の実行）
  • ap_run_watchdog_init / ap_run_watchdog_step / ap_run_watchdog_exit（フックの実行）

関連データ構造

  • ap_watchdog_t: 監視インスタンスのメイン構造体。
  • watchdog_list_t: コールバック関数と実行間隔を保持するリスト。

失敗モードと境界条件

  • ap_mpm_query が失敗した場合、スレッドは安全に終了する。
  • シングルトンモードでロック獲得に失敗し続ける場合、プロセス終了まで待機ループを継続する。
  • コールバック関数内で長時間ブロックが発生すると、当該 Watchdog インスタンス全体の実行間隔に遅延が生じる。

保守担当者向けメモ

  • 本関数は apr_pool_t を使用してメモリを管理しており、ループ内で temp_pool をクリアすることでメモリ消費を抑制している。
  • AP_WD_TM_SLICE はスレッドの応答性と負荷のトレードオフを決定する定数である。
  • シングルトンモードのロック取得後の待機処理は、シャットダウン時の競合を避けるためのヒューリスティックな実装が含まれている。
