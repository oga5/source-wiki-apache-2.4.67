list_urls

定義

  • 種別: function
  • 定義箇所: support/htcacheclean.c:407

目的

指定されたディレクトリパスを再帰的に走査し、キャッシュされた URL の一覧を表示する。

シグネチャまたは形状

static int list_urls(char *path, apr_pool_t *pool, apr_off_t round)

入力と出力

  • **入力**:
  • path: 走査対象のディレクトリパス。
  • pool: メモリ管理用プール。
  • round: サイズ計算時に使用する端数切り上げ単位。
  • **出力**:
  • 成功時は 0 を返し、エラーまたは中断時は 1 を返す。
  • listextended フラグが有効な場合、URL に加えてサイズや時間情報などを標準出力（outfile）へ書き出す。

前提条件と不変条件

  • path は有効なディレクトリである必要がある。
  • キャッシュファイルは DISK_FORMAT_VERSION に準拠している必要がある。
  • 処理中に interrupted フラグが立つと、走査を中断して終了する。

アルゴリズムまたは振る舞い

1. 指定されたディレクトリを apr_dir_open で開き、内容を順次読み込む。 2. ディレクトリの場合は再帰的に list_urls を呼び出す。 3. ファイルの場合は CACHE_HEADER_SUFFIX を持つヘッダーファイルを特定し、以下の処理を行う。

  • ファイルから disk_cache_info_t を読み込み、URL を抽出する。
  • listextended が有効な場合、対応するデータファイル（CACHE_DATA_SUFFIX）の存在確認と整合性チェック（デバイスID/inodeの一致）を行い、詳細情報を出力する。
  • 通常モードでは URL のみを出力する。

    • 4. 処理終了後、benice フラグが有効であれば NICE_DELAY 分だけ待機する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • support/htcacheclean.c:432 (再帰呼び出し)
  • support/htcacheclean.c:1696 (メイン処理からの呼び出し)
  • **呼び出し先**:
  • apr_dir_open, apr_dir_read, apr_dir_close
  • apr_file_open, apr_file_read_full, apr_file_close, apr_file_printf
  • apr_stat, apr_file_info_get
  • round_up (関連シンボル)

関連データ構造

  • disk_cache_info_t: キャッシュエントリのメタデータ（URL、サイズ、時間情報など）。
  • apr_finfo_t: ファイル情報構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • ディレクトリのオープン失敗時は 1 を返す。
  • 読み込み中にシグナル等で interrupted が真になった場合、即座に処理を中断し 1 を返す。
  • ヘッダーファイルやデータファイルの整合性（デバイスID/inodeの不一致）が確認できない場合、そのファイルは無視される。

保守担当者向けメモ

  • apr_pool_create でサブプールを作成し、ディレクトリ走査ごとのメモリ消費を抑制している。
  • listextended モードでは apr_stat を用いて詳細なファイル整合性チェックを行っているため、ファイルシステムへの負荷が高くなる可能性がある。
  • benice モードによるスロットリングが実装されており、システム負荷を考慮した設計となっている。
