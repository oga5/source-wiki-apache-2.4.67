register_hooks

定義

  • 種別: function
  • 定義箇所: modules/proxy/balancers/mod_lbmethod_heartbeat.c:399

目的

Apache HTTP Serverのモジュール初期化プロセスにおいて、本モジュールを mod_proxy の負荷分散メソッドプロバイダーとして登録し、設定読み込み後の初期化フックを定義します。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • **入力**: apr_pool_t *p（モジュールライフサイクルに関連付けられたメモリプール）
  • **出力**: なし（副作用としてプロバイダー登録とフック登録を実行）

前提条件と不変条件

  • mod_heartmonitor.c が存在し、設定の順序において先行している必要があります。
  • heartbeat という名称で PROXY_LBMETHOD プロバイダーが登録されます。

アルゴリズムまたは振る舞い

1. ap_register_provider を呼び出し、heartbeat メソッドを PROXY_LBMETHOD として登録します。この際、heartbeat 構造体をプロバイダーとして提供します。 2. ap_hook_post_config を呼び出し、lb_hb_init 関数を APR_HOOK_MIDDLE の優先度で登録します。 3. aszPred により、mod_heartmonitor.c が先に処理されるよう依存関係を明示しています。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache HTTP Server のモジュール読み込みプロセス（直接的な呼び出し元コードは本ソース内にはありません）。
  • **呼び出し先**:
  • ap_register_provider (Apache API)
  • ap_hook_post_config (Apache API)
  • lb_hb_init (本モジュール内の初期化関数)

関連データ構造

  • heartbeat: PROXY_LBMETHOD として登録される負荷分散メソッド構造体。

失敗モードと境界条件

  • Apache API の登録関数が失敗した場合の戻り値チェックは本関数内には含まれていません。

保守担当者向けメモ

  • 本関数は Apache モジュールの標準的なフック登録パターンに従っています。
  • mod_heartmonitor との依存関係（aszPred）は、共有メモリの初期化順序を保証するために重要です。変更時は注意してください。
