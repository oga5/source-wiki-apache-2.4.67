merge_reflector_dir_config

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_reflector.c:190

目的

Apache HTTP Server のディレクトリ設定において、親ディレクトリの設定（basev）と現在のディレクトリで追加された設定（addv）をマージし、新しい設定構造体を作成します。

シグネチャまたは形状

static void *merge_reflector_dir_config(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: 設定の割り当てに使用するメモリプール。
  • basev: 親ディレクトリの設定（reflector_cfg 型へのキャストが必要）。
  • addv: 現在のディレクトリで定義された設定（reflector_cfg 型へのキャストが必要）。
  • **出力**:
  • マージ済みの新しい reflector_cfg 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は reflector_cfg 構造体として解釈可能である必要があります。
  • apr_pcalloc により、新しい設定構造体はゼロ初期化された状態で割り当てられます。

アルゴリズムまたは振る舞い

1. メモリプール p から新しい reflector_cfg 構造体を割り当てます。 2. apr_table_overlay を使用して、add->headers（現在の設定）を base->headers（親の設定）の上に重ね合わせます。これにより、設定の優先順位が維持されたヘッダーテーブルが生成されます。 3. マージされたテーブルを新しい設定構造体に格納し、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_reflector.c:226（reflector_hooks 内の command_rec 構造体等から参照される設定マージ関数として登録されていると推測されます）。
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_table_overlay (APR)

関連データ構造

  • reflector_cfg: ディレクトリ単位の設定を保持する構造体。headers メンバ（apr_table_t）を管理します。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが終了する可能性があります（APR の標準的な挙動）。
  • basev または addv が NULL の場合、apr_table_overlay の挙動に依存します。

保守担当者向けメモ

  • 本関数は Apache の設定マージの定石に従っています。apr_table_overlay は add のテーブルを base のテーブルの前に配置するため、設定のオーバーライド順序が意図通りであることを確認してください。
