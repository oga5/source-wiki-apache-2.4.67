set_group_privs

定義

  • 種別: function
  • 定義箇所: modules/arch/unix/mod_unixd.c:87

目的

プロセス実行時のグループ権限を確定させ、補助グループを初期化することで、サーバーの特権降格プロセスを完遂させる。

シグネチャまたは形状

static int set_group_privs(void)

入力と出力

  • **入力**: グローバル変数 ap_unixd_config（ユーザー名、グループID、ユーザーIDを含む）。
  • **出力**: 成功時は 0、失敗時は -1 を返す。

前提条件と不変条件

  • 呼び出し元プロセスが root 権限（実効 UID が 0）で実行されている必要がある。
  • OS2 環境ではグループ設定はサポートされていないため、本関数は何も行わない。

アルゴリズムまたは振る舞い

1. **ユーザー名の解決**: ap_unixd_config.user_name が # で始まる場合、文字列を UID として解釈し、getpwuid を用いてユーザー名を取得する。解決できない場合はエラーを返す。 2. **グループ ID の設定**: setgid を呼び出し、プロセスグループ ID を ap_unixd_config.group_id に設定する。 3. **補助グループの初期化**: initgroups を呼び出し、指定されたユーザー名とグループ ID に基づいて補助グループリストを更新する。 4. **エラーハンドリング**: setgid または initgroups が失敗した場合、APLOG_ALERT レベルでログを出力し、-1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/arch/unix/mod_unixd.c:141
  • modules/arch/unix/mod_unixd.c:331
  • **呼び出し先**:
  • getpwuid (libc)
  • setgid (libc)
  • initgroups (libc)
  • ap_log_error (Apache API)

関連データ構造

  • ap_unixd_config: ユーザー名やグループ ID を保持するグローバルな設定構造体。

失敗モードと境界条件

  • **UID 解決失敗**: # 指定された UID がシステムに存在しない場合、エラーログを出力して終了する。
  • **権限不足**: root 権限以外で実行された場合、geteuid() チェックにより処理がスキップされる。
  • **システムコール失敗**: setgid や initgroups が OS 側で拒否された場合、プロセスは特権降格に失敗し、エラーを返す。

保守担当者向けメモ

  • setgid は initgroups の前に実行する必要がある（一部プラットフォームで initgroups がグループリストを上書きする挙動を回避するため）。
  • #uid 形式のサポートは、設定ファイルとの互換性維持のために不可欠である。
