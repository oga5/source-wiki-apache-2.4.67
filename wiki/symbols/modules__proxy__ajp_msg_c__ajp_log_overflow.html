ajp_log_overflow

定義

  • 種別: function
  • 定義箇所: modules/proxy/ajp_msg.c:244

目的

AJPメッセージ操作中に発生したバッファオーバーフローを記録し、エラーコードを返却する。

シグネチャまたは形状

static APR_INLINE int ajp_log_overflow(ajp_msg_t *msg, const char *context)

入力と出力

  • **入力**:
  • msg: 操作対象の ajp_msg_t 構造体へのポインタ。
  • context: エラーが発生した関数名を示す文字列。
  • **出力**:
  • 戻り値: AJP_EOVERFLOW（固定）。

前提条件と不変条件

  • msg は有効な ajp_msg_t 構造体を指している必要がある。
  • 本関数は、メッセージの現在の書き込み/読み出し位置（msg->pos）がバッファの最大長（msg->len）を超過した際に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. ap_log_error を使用し、エラーレベル APLOG_ERR でログを出力する。 2. ログには context、現在のカーソル位置 msg->pos、およびバッファの全長 msg->len が含まれる。 3. 常に AJP_EOVERFLOW を返して呼び出し元にエラーを通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/ajp_msg.c 内のデータ操作関数群。
  • ajp_msg_append_uint32, ajp_msg_append_uint16, ajp_msg_append_uint8
  • ajp_msg_append_cvt_string, ajp_msg_append_bytes
  • ajp_msg_get_uint32, ajp_msg_get_uint16, ajp_msg_get_uint8
  • ajp_msg_peek_uint16, ajp_msg_peek_uint8
  • ajp_msg_get_string
  • **呼び出し先**:
  • ap_log_error (Apache HTTP Server のログ出力関数)

関連データ構造

  • ajp_msg_t: AJPメッセージのバッファ、カーソル位置、および長さを保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーハンドリングを行うためのものであり、内部で失敗することはない。
  • ログ出力先が設定されていない場合や、msg が NULL の場合は未定義の動作となる可能性がある。

保守担当者向けメモ

  • 本関数は APR_INLINE で定義されており、ヘッダーのオーバーフローチェックを行う各関数から直接呼び出される。
  • ログ出力のフォーマットを変更する場合は、APLOGNO(03229) の識別子を維持すること。
