cgi_exec_info_t

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_cgi.h:33

目的

CGI プロセス生成時に必要なパイプ設定、実行タイプ、フィルタコンテキスト、および実行環境パラメータを集約し、OS 固有のコマンド構築ロジックへ受け渡すためのデータ構造です。

シグネチャまたは形状

typedef struct {
    apr_int32_t          in_pipe;
    apr_int32_t          out_pipe;
    apr_int32_t          err_pipe;
    int                  process_cgi;
    apr_cmdtype_e        cmd_type;
    apr_int32_t          detached;
    prog_types           prog_type;
    apr_bucket_brigade **bb;
    include_ctx_t       *ctx;
    ap_filter_t         *next;
    apr_int32_t          addrspace;
} cgi_exec_info_t;

入力と出力

本構造体は、CGI 実行モジュール（mod_cgi や mod_cgid）から OS 固有の実行関数へ渡される入力パラメータとして機能します。実行結果として、プロセス生成に必要なパイプラインやフィルタチェーンの状態が保持されます。

前提条件と不変条件

  • ap_cgi_build_command を介して利用される場合、オプション関数として登録されている必要があります。
  • bb（バケットブリゲード）や ctx（SSI コンテキスト）は、実行コンテキストに応じて適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. CGI 実行モジュールがプロセス生成パラメータを本構造体に格納します。 2. OS 固有の調整が必要な場合、ap_cgi_build_command が本構造体を参照してコマンドや引数を調整します。 3. 調整後、apr_proc_create 等の実行フェーズへ情報が引き継がれ、プロセスが生成されます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_cgi.c（mod_cgi）、mod_cgid.c（mod_cgid）、mod_netware.c、mod_win32.c で使用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。この構造体を受け取る関数（ap_cgi_build_command 等）が、この構造体の内容に基づいてプロセス生成を行います。

関連データ構造

  • prog_types: スクリプトの実行モード（SSI か CGI か）を識別する列挙型。
  • include_ctx_t: SSI コンテキスト（mod_include.h で定義）。
  • ap_filter_t: フィルタチェーンの管理に使用。
  • apr_bucket_brigade: I/O データのバッファリングに使用。

失敗モードと境界条件

  • ap_cgi_build_command が登録されていない場合、デフォルトの実行ロジックが適用されます。
  • パイプ設定（in_pipe, out_pipe, err_pipe）が不適切な場合、プロセス生成時や I/O 処理時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は CGI プロセスの入出力状態を保持するため、デバッグ時にプロセスの入出力が停滞している場合は、各パイプの状態を確認してください。
  • addrspace フィールドはプラットフォーム固有のメモリ空間制御に使用されるため、移植性を考慮する場合は注意が必要です。
