ap_lua_push_apr_table

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_apr.c:31

目的

C 言語側の req_table_t 構造体を Lua のユーザーデータとしてスタックにプッシュし、Lua スクリプトから Apr.Table として操作可能にする。

シグネチャまたは形状

void ap_lua_push_apr_table(lua_State *L, req_table_t *t)

入力と出力

  • **入力**:
  • L: Lua 状態（lua_State）へのポインタ。
  • t: プッシュ対象の req_table_t 構造体へのポインタ。
  • **出力**: なし（Lua スタックのトップにユーザーデータが配置される）。

前提条件と不変条件

  • Lua VM 内に "Apr.Table" という名称のメタテーブルが登録済みであること。
  • t は有効な req_table_t 構造体を指していること。

アルゴリズムまたは振る舞い

1. lua_boxpointer を使用して、ポインタ t を Lua のユーザーデータとしてスタックにプッシュする。 2. luaL_getmetatable を呼び出し、"Apr.Table" メタテーブルを取得してスタックに積む。 3. lua_setmetatable を呼び出し、スタックトップのユーザーデータにメタテーブルを関連付ける。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_request.c:1891
  • modules/lua/lua_request.c:3025
  • **呼び出し先**:
  • lua_boxpointer (Lua C API)
  • luaL_getmetatable (Lua C API)
  • lua_setmetatable (Lua C API)

関連データ構造

  • req_table_t: apr_table_t をラップした構造体。
  • Apr.Table: Lua 側で定義されたメタテーブル。__index および __newindex が設定されており、テーブル操作をフックする。

失敗モードと境界条件

  • Lua スタックのメモリ不足が発生した場合、Lua のエラーハンドリング機構により例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は lua_apr.c 内の他の関数（ap_lua_check_apr_table など）と連携し、C と Lua 間のデータ変換層を構成している。
  • メタテーブル名 "Apr.Table" を変更する場合は、ap_lua_init 内の登録処理と整合性を取る必要がある。
