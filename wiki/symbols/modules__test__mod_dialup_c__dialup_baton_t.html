dialup_baton_t

定義

  • 種別: struct
  • 定義箇所: modules/test/mod_dialup.c:38

目的

非同期リクエスト処理におけるコンテキスト保持用構造体です。Apache HTTP Server の SUSPENDED 状態を利用した帯域制限付きデータ送信において、送信状態を維持するために使用されます。

シグネチャまたは形状

typedef struct dialup_baton_t {
    apr_size_t bytes_per_second;    /* 1秒あたりの送信バイト数 */
    request_rec *r;                 /* リクエスト情報 */
    apr_file_t *fd;                 /* 送信対象ファイル */
    apr_bucket_brigade *bb;         /* 全データを含むバケットブリゲード */
    apr_bucket_brigade *tmpbb;      /* 送信分を一時保持するバケットブリゲード */
} dialup_baton_t;

入力と出力

  • **入力**: dialup_handler にてリクエスト処理開始時に初期化され、r->pool からメモリが割り当てられます。
  • **出力**: 構造体自体はリクエストプールに紐付いているため、リクエスト終了時に自動的に解放されます。

前提条件と不変条件

  • 本構造体は r->pool 上に割り当てられる必要があります。
  • bb および tmpbb は、dialup_send_pulse によるバケット移動操作の対象となります。
  • マルチスレッド環境下では、r->invoke_mtx による排他制御下で本構造体のメンバへアクセスする必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: dialup_handler がリクエストを捕捉した際、本構造体を生成し、ファイル記述子とバケットブリゲードをセットアップします。 2. **データ送信**: dialup_send_pulse が本構造体を参照し、bytes_per_second に基づいて bb から tmpbb へデータを移動させ、ネットワークへ送信します。 3. **非同期ループ**: 送信すべきデータが残っている場合、ap_mpm_register_timed_callback を用いて次回の dialup_callback 実行を予約します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • dialup_handler (初期化)
  • dialup_send_pulse (引数として参照)
  • dialup_callback (バトンとしてキャストして参照)
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • apr_bucket_brigade: 送信データのバッファリングに使用。
  • dialup_dcfg_t: 帯域制限値（bytes_per_second）の元となる設定構造体。

失敗モードと境界条件

  • ap_pass_brigade が失敗した場合の挙動は最小限の実装となっており、エラー時の詳細なログ出力は省略されています。
  • SUSPENDED 状態の利用は MPM の実装に依存するため、環境によっては期待通りに動作しない可能性があります。

保守担当者向けメモ

  • 本構造体は実験的なモジュールの一部です。エラーハンドリングの強化が必要な場合は、dialup_send_pulse 内のコメントにある通り、ログ出力の追加を検討してください。
  • メモリ管理はリクエストプールに依存しているため、手動での free は不要です。
