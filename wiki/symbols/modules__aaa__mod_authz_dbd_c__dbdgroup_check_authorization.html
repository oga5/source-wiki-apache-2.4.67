dbdgroup_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbd.c:273

目的

Require dbd-group ディレクティブに基づき、データベースから取得したユーザーの所属グループ情報と、設定された許可グループリストを照合して認可判定を行う。

シグネチャまたは形状

static authz_status dbdgroup_check_authorization(request_rec *r,
                                                 const char *require_args,
                                                 const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • require_args: 未使用。
  • parsed_require_args: ap_expr_info_t 型の式情報。評価後に許可グループリストとして使用される。
  • **出力**:
  • AUTHZ_GRANTED: ユーザーが指定グループのいずれかに所属している場合。
  • AUTHZ_DENIED: ユーザーが所属していない、または式評価に失敗した場合。
  • AUTHZ_DENIED_NO_USER: r->user が設定されていない場合。
  • AUTHZ_GENERAL_ERROR: データベースクエリ実行に失敗した場合。

前提条件と不変条件

  • r->user が認証済みであること。
  • authz_dbd_module の設定が r->per_dir_config に存在すること。
  • データベース接続およびクエリ実行が mod_dbd を介して利用可能であること。

アルゴリズムまたは振る舞い

1. r->user の存在を確認し、未認証なら AUTHZ_DENIED_NO_USER を返す。 2. authz_dbd_group_query を呼び出し、現在のユーザーが所属する全グループを groups 配列に取得する。クエリ失敗時は AUTHZ_GENERAL_ERROR を返す。 3. parsed_require_args を ap_expr_str_exec で評価し、許可グループの文字列リストを取得する。評価失敗時はログを出力し AUTHZ_DENIED を返す。 4. 取得した許可グループリストを空白区切りで反復し、各グループが groups 配列に含まれているかを確認する。 5. 一致するグループが見つかれば AUTHZ_GRANTED を返し、最後まで見つからなければ AUTHZ_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbd.c:368 にて authz_dbd_hooks 内で登録されている（直接の呼び出し箇所は Apache の認可フレームワーク）。
  • **呼び出し先**:
  • authz_dbd_group_query: ユーザーのグループリスト取得。
  • ap_expr_str_exec: 許可グループ式の評価。
  • ap_getword_white: グループ名のトークン分割。
  • ap_array_str_contains: グループ名の照合。

関連データ構造

  • authz_dbd_cfg: ディレクトリ単位の設定情報。
  • request_rec: リクエストごとの状態管理およびメモリプール (r->pool)。
  • apr_array_header_t: データベースから取得したグループリストの保持に使用。

失敗モードと境界条件

  • データベース接続エラーやクエリ実行失敗時は AUTHZ_GENERAL_ERROR を返す。
  • ユーザーが認証されていない場合、認可処理は即座に拒否される。
  • 式評価 (ap_expr_str_exec) が失敗した場合、エラーログを出力して AUTHZ_DENIED となる。

保守担当者向けメモ

  • すべての動的メモリ割り当ては r->pool を使用しており、リクエスト終了時に自動的に解放される。
  • dbdgroup_check_authorization は mod_authz_core の認可プロバイダーとして登録されるため、直接呼び出すのではなく、Apache の設定ディレクティブを通じて制御される。
