hc_condition_t

定義

  • 種別: struct
  • 定義箇所: modules/proxy/mod_proxy_hcheck.c:49

目的

ヘルスチェックの判定条件を保持するための構造体です。ProxyHCExpr ディレクティブで定義された式を解析済み状態で保持し、バックエンドの応答評価に使用します。

シグネチャまたは形状

typedef struct {
    char *expr;                  /* 式の文字列 */
    ap_expr_info_t *pexpr;       /* 解析済みの式情報 */
} hc_condition_t;

入力と出力

  • **入力**: 設定ファイルから読み込まれた式文字列（expr）。
  • **出力**: ap_expr によって解析された ap_expr_info_t 構造体（pexpr）。

前提条件と不変条件

  • pexpr は ap_expr_parse 等の関数によって事前に解析済みである必要があります。
  • expr は apr_pool_t 上で管理されるメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 設定読み込み時に apr_palloc でメモリを確保し、式を解析して pexpr に格納します。 2. **評価**: ヘルスチェック実行時、apr_table_get で取得した hc_condition_t インスタンスの pexpr を ap_expr_exec に渡すことで、バックエンドの応答に対する真偽判定を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_hcheck.c 内の各関数（設定解析時およびヘルスチェック実行時の条件検索時）。
  • **呼び出し先**:
  • ap_expr API（ap_expr_info_t を介した式の評価）。

関連データ構造

  • ap_expr_info_t: ap_expr モジュールが提供する、解析済みの式を保持する構造体。
  • sctx_t: conditions テーブルを保持しており、そこから hc_condition_t が検索されます。

失敗モードと境界条件

  • **解析失敗**: pexpr が NULL の場合、式評価は実行できません。
  • **メモリ不足**: apr_palloc が失敗した場合、設定の読み込みが中断されます。

保守担当者向けメモ

  • 本構造体は mod_proxy_hcheck における動的なヘルスチェック判定の核となります。
  • ap_expr の仕様変更や、新しいヘルスチェック変数の追加を行う際は、pexpr の評価コンテキスト（ap_expr_eval_ctx_t）の構築ロジックと併せて確認してください。
