duration_parse

定義

  • 種別: function
  • 定義箇所: test/modules/http2/mod_h2test/mod_h2test.c:49

目的

文字列形式で指定された期間（例: "10s", "50ms"）を解析し、Apache Portable Runtime (APR) で利用可能な apr_interval_time_t 形式の数値に変換する。

シグネチャまたは形状

apr_status_t duration_parse(apr_interval_time_t *ptimeout, const char *value, const char *def_unit)

入力と出力

  • **入力**:
  • ptimeout: 変換結果を格納する apr_interval_time_t へのポインタ。
  • value: 解析対象の文字列（数値と単位）。
  • def_unit: 単位が省略された場合に適用されるデフォルトの単位文字列。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS。失敗時は APR_EINVAL、APR_EGENERAL、または apr_strtoi64 が返すエラーコード。

前提条件と不変条件

  • ptimeout は有効なメモリ領域を指している必要がある。
  • value が数値で始まらない場合、APR_EINVAL が返される。
  • 単位の指定がサポートされていない形式の場合、APR_EGENERAL が返される。

アルゴリズムまたは振る舞い

1. apr_strtoi64 を使用して文字列の先頭から数値を抽出する。 2. 単位が指定されていない場合、def_unit を使用する。 3. 単位文字（'d', 's', 'h', 'm' など）に基づき、apr_time_from_sec 等を用いて apr_interval_time_t に変換する。

  • 'd': 日単位
  • 's': 秒単位
  • 'h': 時間単位
  • 'ms': ミリ秒単位
  • 'mi': 分単位

    • 4. 変換結果を *ptimeout に書き込み、APR_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • test/modules/http2/mod_h2test/mod_h2test.c 内の各ハンドラ（h2test_echo_handler, h2test_delay_handler, h2test_trailer_handler 等）から、リクエスト引数の解析時に呼び出される。
  • **呼び出し先**:
  • apr_strtoi64
  • apr_time_from_sec

関連データ構造

  • apr_interval_time_t: 時間間隔を表す型。
  • apr_status_t: APR のステータスコード。

失敗モードと境界条件

  • errno が設定されている場合（数値オーバーフロー等）、その値をそのまま返す。
  • 単位指定が不明な場合や、'm' の後に 's' または 'i' 以外が続く場合は APR_EGENERAL を返す。
  • 数値部分が空の場合は APR_EINVAL を返す。

保守担当者向けメモ

  • 本関数はテストモジュール専用のユーティリティである。
  • 単位の追加や変更が必要な場合は、switch 文のケースを追加することで対応可能。
