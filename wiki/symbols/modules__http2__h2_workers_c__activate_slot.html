activate_slot

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_workers.c:108

目的

HTTP/2 ワーカープールにおいて、利用可能なスロット（h2_slot）を確保し、新しいワーカースレッドを生成して処理を開始します。

シグネチャまたは形状

static apr_status_t activate_slot(h2_workers *workers)

入力と出力

  • **入力**: workers (h2_workers *) - ワーカープール全体の管理構造体。
  • **出力**: apr_status_t - 成功時は APR_SUCCESS、スロットが不足している場合は APR_EAGAIN、スレッド生成失敗時はエラーコードを返します。

前提条件と不変条件

  • workers->free リングが空でないこと（空の場合は APR_EAGAIN を返します）。
  • 呼び出し側で workers->lock による排他制御が行われていることが前提です。
  • 選択された slot の状態は H2_SLOT_FREE である必要があります。

アルゴリズムまたは振る舞い

1. workers->free リングから最初の空きスロットを取得し、リングから取り除きます。 2. スロットの状態を H2_SLOT_RUN に更新し、active_slots カウンタをインクリメントします。 3. スロット専用のサブプール（apr_pool_create）を生成します。 4. slot_run をエントリポイントとして ap_thread_create（または apr_thread_create）を呼び出し、新しいスレッドを起動します。 5. スレッド生成に失敗した場合は、サブプールを破棄し、スロットを H2_SLOT_FREE に戻して workers->free リングの末尾に再挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_workers.c:197
  • modules/http2/h2_workers.c:521
  • **呼び出し先**:
  • slot_run (スレッドのエントリポイントとして指定)
  • apr_pool_create, apr_pool_destroy, apr_pool_tag
  • ap_thread_create / apr_thread_create
  • ap_log_error

関連データ構造

  • h2_workers: ワーカープールの管理構造体。
  • h2_slot: 個々のワーカーの状態を保持する構造体。

失敗モードと境界条件

  • **スロット不足**: workers->free が空の場合、APR_EAGAIN を返します。
  • **プール生成失敗**: apr_pool_create が失敗した場合、エラーを返し終了します。
  • **スレッド生成失敗**: apr_thread_create が失敗した場合、AP_DEBUG_ASSERT をトリガーし、スロットの状態を復元してエラーを返します。

保守担当者向けメモ

  • 本関数は workers->lock による保護下で実行されることを想定しています。
  • slot->pool はスレッド終了時にクリーンアップされるため、スレッド生成失敗時のメモリリーク防止処理が cleanup ラベル配下に実装されています。
  • AP_HAS_THREAD_LOCAL の有無によりスレッド生成関数が分岐しますが、いずれも slot_run を実行します。
