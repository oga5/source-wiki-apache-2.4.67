deflate_ctx

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_deflate.c:392

目的

Apache HTTP Server の mod_deflate フィルタにおいて、圧縮・伸長処理のライフサイクル全体を通じて状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct deflate_ctx_t
{
    z_stream stream;
    unsigned char *buffer;
    unsigned long crc;
    apr_bucket_brigade *bb, *proc_bb;
    int (*libz_end_func)(z_streamp);
    unsigned char *validation_buffer;
    apr_size_t validation_buffer_length;
    char header[10];
    apr_size_t header_len;
    int zlib_flags;
    int ratio_hits;
    apr_off_t inflate_total;
    unsigned int consume_pos, consume_len;
    unsigned int filter_init:1;
    unsigned int done:1;
} deflate_ctx;

入力と出力

  • **入力**: フィルタチェーンを流れる apr_bucket_brigade データ。
  • **出力**: zlib を介して処理された圧縮/伸長済みデータ、および EOS (End of Stream) 到達時のフッター情報。

前提条件と不変条件

  • リクエストプールに紐付いており、リクエスト終了時に apr_pool_cleanup_register を通じて zlib リソースが解放される必要があります。
  • buffer は設定された bufferSize に基づいて動的に確保されます。

アルゴリズムまたは振る舞い

1. **初期化**: フィルタ初回呼び出し時に filter_init フラグを用いて zlib を初期化します。 2. **データ処理**: stream を使用してバケット内のデータを圧縮・伸長し、proc_bb を経由して後続のフィルタへ渡します。 3. **終了処理**: done フラグが立つまで処理を継続し、EOS 検知時に libz_end_func を呼び出してリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: deflate_out_filter, inflate_out_filter, deflate_in_filter 等のフィルタ関数。
  • **呼び出し先**: consume_buffer, flush_libz_buffer, check_ratio, consume_zlib_flags 等のモジュール内ユーティリティ関数。

関連データ構造

  • z_stream: zlib のコア状態管理構造体。
  • apr_bucket_brigade: Apache のデータストリーム管理構造体。
  • deflate_filter_config: サーバー全体の設定情報。

失敗モードと境界条件

  • **制限超過**: ratio_limit や inflate_limit を超えた場合、APR_EINVAL や APR_ENOSPC を返して処理を中断します。
  • **スキップ条件**: HEAD リクエスト、204 No Content、または既に圧縮済みのコンテンツは処理対象外となります。

保守担当者向けメモ

  • 本構造体は mod_deflate の状態管理の要です。zlib の内部的なハフマン符号化や LZ77 圧縮アルゴリズムの詳細は zlib ライブラリ側に依存するため、本ファイルでは管理していません。
