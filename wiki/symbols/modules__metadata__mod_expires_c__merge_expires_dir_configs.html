merge_expires_dir_configs

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:360

目的

ディレクトリ単位で設定された expires_dir_config 構造体をマージし、親ディレクトリの設定と現在の設定を統合する。

シグネチャまたは形状

static void *merge_expires_dir_configs(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • basev: 親ディレクトリの設定（expires_dir_config）。
  • addv: 現在のディレクトリの設定（expires_dir_config）。
  • **出力**:
  • 統合された新しい expires_dir_config 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は expires_dir_config 型としてキャスト可能であること。
  • apr_pcalloc により、新しい設定構造体はゼロ初期化された状態で確保される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して新しい expires_dir_config 構造体をメモリ上に確保する。 2. active 設定の統合: add->active が ACTIVE_DONTCARE の場合は base->active を継承し、それ以外の場合は add->active を優先する。 3. expiresdefault 設定の統合: add->expiresdefault が設定されていればそれを採用し、そうでなければ base->expiresdefault を継承する。 4. wildcards 設定の統合: add->wildcards をそのまま新しい構造体にコピーする。 5. expiresbytype テーブルの統合: apr_table_overlay を使用し、add のテーブルを優先して base のテーブルと結合する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_expires.c:566（Apache の設定マージ機構から呼び出される）。
  • **呼び出し先**: apr_pcalloc (APR), apr_table_overlay (APR)。

関連データ構造

  • expires_dir_config: ディレクトリごとの設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗する可能性がある（Apache の標準的な挙動に従い、通常はプロセス終了等の致命的エラーとなる）。
  • add->expiresbytype または base->expiresbytype が NULL の場合、apr_table_overlay の仕様に従って処理される。

保守担当者向けメモ

  • 本関数は Apache の設定マージフェーズで呼び出される。
  • expiresbytype のマージには apr_table_overlay を使用しており、後から追加された設定が優先される順序となっている。
