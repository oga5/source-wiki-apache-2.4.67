md_result_set

定義

  • 種別: function
  • 定義箇所: modules/md/md_result.c:91

目的

md_result_t オブジェクトの状態をリセットし、新しいステータスコードと詳細メッセージを設定して、変更通知をトリガーします。

シグネチャまたは形状

void md_result_set(md_result_t *result, apr_status_t status, const char *detail)

入力と出力

  • **入力**:
  • result: 更新対象の md_result_t 構造体へのポインタ。
  • status: 設定する apr_status_t 型のステータスコード。
  • detail: 処理の詳細を示す文字列（NULL許容）。
  • **出力**: なし（副作用として result の内容を更新）。

前提条件と不変条件

  • result は有効な md_result_t オブジェクトを指している必要があります。
  • result が保持する apr_pool_t は、文字列の複製に使用されます。

アルゴリズムまたは振る舞い

1. result->status に status を代入します。 2. result->problem を NULL にクリアします。 3. detail が提供されている場合、result->p プールからメモリを割り当てて文字列を複製し、result->detail に格納します。提供されていない場合は NULL を設定します。 4. result->subproblems を NULL にクリアします。 5. 最後に on_change(result) を呼び出し、登録された変更通知コールバックを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme.c
  • modules/md/md_acme_drive.c
  • **呼び出し先**:
  • apr_pstrdup (APRライブラリ)
  • on_change (同一ファイル内の静的関数)

関連データ構造

  • md_result_t: 処理結果を保持するメイン構造体。
  • apr_status_t: APRのステータスコード型。

失敗モードと境界条件

  • detail が NULL の場合、result->detail は安全に NULL として処理されます。
  • メモリ不足時には apr_pstrdup が例外（アボート）を発生させる可能性があります。

保守担当者向けメモ

  • 本関数は problem や subproblems をクリアするため、エラーの詳細情報を保持したい場合は md_result_problem_set 等の他のセッター関数を検討してください。
  • 直接フィールドを操作せず、必ず本関数を経由することで on_change コールバックによる状態同期が保証されます。
