ap_get_exec_line

定義

  • 種別: function
  • 定義箇所: server/util.c:3386

目的

外部コマンドを実行し、その標準出力から最初の1行を読み取って文字列として返します。

シグネチャまたは形状

char *ap_get_exec_line(apr_pool_t *p, const char *cmd, const char * const * argv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t。
  • cmd: 実行するコマンドのパス。
  • argv: コマンドに渡す引数リスト。
  • **出力**:
  • 成功時: コマンドの出力の最初の1行（改行文字は含まない）。
  • 失敗時: NULL。

前提条件と不変条件

  • p は有効なメモリプールである必要があります。
  • コマンドの出力は MAX_STRING_LEN バイト未満である必要があります。

アルゴリズムまたは振る舞い

1. apr_procattr_t を作成し、入出力設定（APR_FULL_BLOCK）および実行ディレクトリを設定します。 2. apr_proc_create を呼び出し、指定されたコマンドを子プロセスとして実行します。 3. 子プロセスの標準出力（proc->out）から1バイトずつ読み込みます。 4. 改行文字（\n または \r）に到達するか、バッファ上限に達するまで読み取りを継続します。 5. 読み取った内容を apr_pstrndup でプール上に複製して返します。 6. 最後にファイルハンドルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authnz_ldap.c
  • modules/session/mod_session_crypto.c
  • **呼び出し先**:
  • APR プロセス管理関数群 (apr_procattr_create, apr_proc_create 等)
  • APR ファイル操作関数群 (apr_file_read, apr_file_close)
  • ap_make_dirstr_parent

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_proc_t: 子プロセス管理に使用。
  • apr_procattr_t: プロセス属性設定に使用。

失敗モードと境界条件

  • プロセス属性の作成やコマンド実行に失敗した場合、NULL を返します。
  • 出力が MAX_STRING_LEN を超える場合、バッファ上限までの内容が返されます。
  • fp が NULL の場合、NULL を返します。

保守担当者向けメモ

  • 現在、標準出力を1バイトずつ読み取る実装となっており、パフォーマンス上の懸念があります。
  • MAX_STRING_LEN に依存しているため、非常に長い出力行には対応していません。
