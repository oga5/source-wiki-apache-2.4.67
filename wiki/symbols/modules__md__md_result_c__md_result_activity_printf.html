md_result_activity_printf

定義

  • 種別: function
  • 定義箇所: modules/md/md_result.c:82

目的

md_result_t オブジェクトに、現在の処理アクティビティを示すメッセージをフォーマット付きで設定する。

シグネチャまたは形状

void md_result_activity_printf(md_result_t *result, const char *fmt, ...)

入力と出力

  • result: 更新対象の md_result_t 構造体へのポインタ。
  • fmt: printf 形式の書式指定文字列。
  • ...: 書式指定に対応する可変長引数。
  • 出力: なし（result 構造体内部のフィールドを更新）。

前提条件と不変条件

  • result は有効な md_result_t 構造体を指している必要がある。
  • result->p（APRプール）がメモリ割り当てのために使用される。
  • 内部で md_result_activity_setn を呼び出すため、状態変更時のコールバック（on_change）がトリガーされる。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化する。 2. apr_pvsprintf を呼び出し、result->p プールからメモリを確保してフォーマット済み文字列を生成する。 3. 生成された文字列を md_result_activity_setn に渡し、アクティビティとして設定する。 4. va_end で引数リストを終了する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • modules/md/md_acme_authz.c: チャレンジ設定時のログ記録。
  • modules/md/md_acme_drive.c: アカウント作成、CSR提出、証明書取得等の各フェーズ。
  • modules/md/md_acme_order.c: チャレンジ監視および開始時。
  • 呼び出し先:
  • apr_pvsprintf (APR)
  • md_result_activity_setn (同一モジュール内)

関連データ構造

  • md_result_t: 処理結果を保持するメイン構造体。
  • apr_pool_t: 文字列のメモリ管理に使用されるAPRプール。

失敗モードと境界条件

  • result が NULL の場合、プログラムはクラッシュする可能性がある（内部で result->p にアクセスするため）。
  • メモリ不足時には apr_pvsprintf が例外を発生させる可能性がある。

保守担当者向けメモ

  • 本関数は md_result_activity_setn のラッパーであり、文字列構築の利便性を提供します。
  • ログ出力やUIへの進捗表示を目的としており、頻繁に呼び出される可能性があるため、フォーマット文字列の構築コストに留意してください。
