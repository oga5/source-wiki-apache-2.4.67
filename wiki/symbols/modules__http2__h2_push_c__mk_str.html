mk_str

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_push.c:162

目的

link_ctx が保持する文字列バッファから、指定された範囲の文字列を新しいメモリ領域に複製して抽出します。

シグネチャまたは形状

static char *mk_str(link_ctx *ctx, size_t end)

入力と出力

  • **入力**:
  • ctx: 解析状態を保持する link_ctx 構造体へのポインタ。現在の読み込み位置（ctx->i）と文字列バッファ（ctx->s）、メモリプール（ctx->pool）を参照します。
  • end: 抽出対象となる文字列の終了インデックス。
  • **出力**:
  • 成功時: ctx->pool から割り当てられた、ctx->s の ctx->i から end までの部分文字列。
  • 失敗または範囲外時: 空文字列 ""。

前提条件と不変条件

  • ctx->pool は有効な apr_pool_t である必要があります。
  • ctx->s は少なくとも end までのインデックスを含む有効な文字列バッファである必要があります。

アルゴリズムまたは振る舞い

1. 現在の読み込み位置 ctx->i が end より小さいかを確認します。 2. 条件を満たす場合、apr_pstrndup を使用して ctx->pool 上に部分文字列を複製し、そのポインタを返します。 3. 条件を満たさない（ctx->i >= end）場合、静的な空文字列 "" を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:

関連データ構造

  • link_ctx: Link ヘッダーの解析状態を管理する構造体。

失敗モードと境界条件

  • ctx->i >= end の場合、メモリ割り当てを行わず空文字列を返します。
  • apr_pstrndup がメモリ不足に陥った場合、APRの仕様に従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は Link ヘッダー解析中のトークン抽出に使用されます。
  • 戻り値は apr_pool_t に依存するため、呼び出し側でプールが破棄されるタイミングに注意してください。
