access_compat_dir_conf

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_access_compat.c:69

目的

Apache HTTP Server 2.2 以前のアクセス制御ディレクティブ（Order, Allow, Deny, Satisfy）をディレクトリ単位で保持・管理するための設定構造体です。

シグネチャまたは形状

typedef struct {
    int order[METHODS];
    apr_array_header_t *allows;
    apr_array_header_t *denys;
    int *satisfy;
} access_compat_dir_conf;

入力と出力

  • **入力**: 設定ファイル内の Order, Allow, Deny, Satisfy ディレクティブ。
  • **出力**: check_dir_access 等のアクセス判定ロジックが参照する設定値。

前提条件と不変条件

  • apr_pool_t を介してメモリが割り当てられ、リクエスト終了時に自動的に解放されます。
  • satisfy 配列はメソッドごとに存在し、各メソッドのアクセスポリシーを保持します。

アルゴリズムまたは振る舞い

  • Order ディレクティブに基づき、allows および denys 配列に格納されたルールを評価する順序を決定します。
  • find_allowdeny 関数が本構造体の allows および denys を走査し、リクエストの IP アドレスやホスト名との照合を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: create_access_compat_dir_config（メモリ確保）、allow_cmd（設定値の追加）、check_dir_access（アクセス判定）、find_allowdeny（ルール評価）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_array_header_t: allows および denys ルールのリストを保持するために使用。
  • METHODS: order 配列のサイズを決定する定数。

失敗モードと境界条件

  • 設定が未定義の場合、各フィールドは初期化状態（NULL またはデフォルト値）となります。
  • in_domain を用いたホスト名検証を行う際、DNS 逆引きの結果に依存するため、ネットワーク環境や設定によっては判定が遅延または失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_access_compat の中核であり、mod_authz_host への移行を検討する際の参照先となります。
  • メモリ管理は Apache のプール機構に依存しているため、手動での解放は不要です。
