EXTENSION_CONTROL_BLOCK

定義

  • 種別: struct
  • 定義箇所: modules/arch/win32/mod_isapi.h:77

目的

ISAPI (Internet Server Application) 拡張に対して、Apache HTTP Server 上のリクエストコンテキストと、サーバー機能へのアクセス手段（コールバック関数）を提供するための主要なデータ構造です。

シグネチャまたは形状

typedef struct EXTENSION_CONTROL_BLOCK {
    apr_uint32_t   cbSize;
    apr_uint32_t   dwVersion;
    HCONN          ConnID;
    apr_uint32_t   dwHttpStatusCode;
    char           lpszLogData[80];
    char          *lpszMethod;
    char          *lpszQueryString;
    char          *lpszPathInfo;
    char          *lpszPathTranslated;
    apr_uint32_t   cbTotalBytes;
    apr_uint32_t   cbAvailable;
    unsigned char *lpbData;
    char          *lpszContentType;

    PFN_GETSERVERVARIABLE     GetServerVariable;
    PFN_WRITECLIENT           WriteClient;
    PFN_READCLIENT            ReadClient;
    PFN_SERVERSUPPORTFUNCTION ServerSupportFunction;
} EXTENSION_CONTROL_BLOCK;

入力と出力

  • **入力**: Apache がリクエスト処理時に apr_pcalloc を使用してメモリを確保し、リクエスト情報を設定します。
  • **出力**: ISAPI 拡張側がこの構造体を通じてサーバーへ情報を書き戻す（WriteClient 等）か、サーバーが拡張側の処理結果（dwHttpStatusCode 等）を読み取ります。

前提条件と不変条件

  • Apache がリクエストのライフサイクルに合わせてメモリを管理します。ISAPI 拡張側でこの構造体を永続化してはなりません。
  • cbSize には sizeof(EXTENSION_CONTROL_BLOCK) が設定されている必要があります。

アルゴリズムまたは振る舞い

  • Apache は HttpExtensionProc を呼び出す際、この構造体を引数として渡します。
  • 構造体内の関数ポインタ（GetServerVariable 等）を通じて、ISAPI 拡張は Apache の内部機能と通信します。
  • 非同期処理が要求された場合、Apache は HSE_STATUS_PENDING を受け取り、完了通知があるまでセッションを維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_isapi.c 内のハンドラが、リクエスト処理の初期化時にこの構造体をインスタンス化し、HttpExtensionProc へ渡します。
  • **呼び出し先**: 構造体内の関数ポインタを通じて、mod_isapi.c で実装されたサーバーサポート関数が呼び出されます。

関連データ構造

  • HCONN: isapi_cid への不透明なポインタ。
  • PFN_*: 各種コールバック関数型。

失敗モードと境界条件

  • HSE_SEND_HEADER_EX_INFO を使用する場合、ヘッダー文字列に NULL 文字を含めると Apache の内部処理と競合する可能性があります。
  • 一部の IIS 固有機能（HSE_REQ_GET_SSPI_INFO 等）は Apache のアーキテクチャと整合しないため、サポートされていません。

保守担当者向けメモ

  • 本構造体はマルチスレッド環境（APR_THREAD_FUNC）で利用されるため、スレッドセーフな実装を維持してください。
  • mod_isapi.c における cid->ecb の初期化フローが、本構造体のライフサイクル管理の基準となります。
