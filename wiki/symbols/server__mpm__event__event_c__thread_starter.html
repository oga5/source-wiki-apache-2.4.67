thread_starter

定義

  • 種別: struct
  • 定義箇所: server/mpm/event/event.c:351

目的

Event MPM において、子プロセス内のスレッド（リスナーおよびワーカー）を生成・管理するための情報を保持する構造体です。

シグネチャまたは形状

typedef struct {
    apr_thread_t **threads;
    apr_thread_t *listener;
    int child_num_arg;
    apr_threadattr_t *threadattr;
} thread_starter;

入力と出力

  • **入力**: スレッド生成時に必要な属性（threadattr）やスレッド管理用配列（threads）を保持します。
  • **出力**: 構造体内の各フィールドを通じて、生成されたスレッドのハンドルやプロセス固有の識別子（child_num_arg）を共有します。

前提条件と不変条件

  • threads は、生成されるワーカー数に応じたメモリ領域が確保されている必要があります。
  • threadattr は、APR のスレッド属性として適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

本構造体は、子プロセス起動時にスレッドプールを初期化する際のコンテキストとして使用されます。create_listener_thread 等の関数に渡され、リスナーおよびワーカーの生成プロセスを制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/event/event.c 内のプロセス管理ロジック（create_listener_thread 等の引数として利用）。
  • **呼び出し先**: APR のスレッド生成関数（apr_thread_create 等）へ渡される情報源となります。
  • **注記**: server/mpm/worker/worker.c にも同名の構造体が存在しますが、本ページは event.c の定義を対象としています。

関連データ構造

  • apr_thread_t: APR のスレッドハンドル。
  • apr_threadattr_t: スレッドの属性設定。

失敗モードと境界条件

  • スレッド生成時にメモリ不足やリソース制限が発生した場合、本構造体を用いた初期化プロセスが失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は worker MPM と共通の設計思想を持っていますが、event MPM 特有の接続管理スレッドのライフサイクルと密接に関係しています。変更時は event.c 内のスレッド生成フローへの影響を確認してください。
