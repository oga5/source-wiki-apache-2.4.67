initialize_tables

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_auth_digest.c:263

目的

HTTP Digest 認証に必要な共有メモリセグメント、ハッシュテーブル、およびプロセス間同期用ミューテックスを初期化する。

シグネチャまたは形状

static int initialize_tables(server_rec *s, apr_pool_t *ctx)

入力と出力

  • **入力**:
  • s: サーバー設定構造体。
  • ctx: メモリ割り当てに使用するプール。
  • **出力**: 成功時は OK、失敗時は HTTP_INTERNAL_SERVER_ERROR または !OK を返す。

前提条件と不変条件

  • shmem_size および num_buckets が事前に定義されている必要がある。
  • 共有メモリの初期化に失敗した場合、log_error_and_cleanup を経由してエラーログが出力される。

アルゴリズムまたは振る舞い

1. **共有メモリ作成**: apr_shm_create を使用し、匿名共有メモリを試行。失敗時はファイルベース（authdigest_shm）で再試行する。 2. **メモリ管理初期化**: apr_rmm_init を用いて共有メモリ領域を管理下に置く。 3. **ハッシュテーブル構築**: rmm_malloc で共有メモリ上に client_list を確保し、バケットを初期化する。 4. **同期機構**: ap_global_mutex_create を使用し、クライアントリスト用（client_lock）および opaque 値用（opaque_lock）のミューテックスを作成する。 5. **カウンター初期化**: opaque_cntr および otn_counter を共有メモリ上に確保し、初期値を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_auth_digest.c 内の初期化プロセス（ソース内の直接的な呼び出し元は 421 行目付近）。
  • **呼び出し先**: apr_shm_create, apr_rmm_init, rmm_malloc, ap_global_mutex_create, log_error_and_cleanup 等の APR/Apache API。

関連データ構造

  • client_shm, client_rmm: 共有メモリおよびメモリ管理ハンドル。
  • client_list: クライアント追跡用ハッシュテーブル。
  • client_lock, opaque_lock: プロセス間同期用ミューテックス。
  • opaque_cntr, otn_counter: 共有メモリ上のカウンター。

失敗モードと境界条件

  • 共有メモリの確保失敗、またはミューテックスの作成失敗時にエラーを返し、プロセスを停止させる可能性がある。
  • ファイルベースの共有メモリを使用する場合、以前の異常終了時に残存するファイルを apr_shm_remove で削除する処理が含まれる。

保守担当者向けメモ

  • 本関数はマルチプロセス環境での共有状態を管理するため、ミューテックスのロック順序や共有メモリの生存期間に注意すること。
  • otn_counter にはロックが適用されていない点に留意が必要。
