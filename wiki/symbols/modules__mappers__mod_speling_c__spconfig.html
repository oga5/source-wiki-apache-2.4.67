spconfig

定義

  • 種別: struct
  • 定義箇所: modules/mappers/mod_speling.c:58

目的

mod_speling モジュールの動作設定を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    int enabled;
    int check_case_only;
    int check_basename_match;
} spconfig;

入力と出力

本構造体は設定値を保持するコンテナであり、直接的な関数入出力は持ちません。各メンバは CheckSpelling などのディレクティブを通じて設定されます。

前提条件と不変条件

  • enabled: モジュールのスペルチェック機能を有効にするか否か。
  • check_case_only: 大文字小文字の不一致のみを修正対象とするか。
  • check_basename_match: ファイルのベース名のみをマッチング対象とするか。

アルゴリズムまたは振る舞い

本構造体は、mod_speling のリクエスト処理（check_speling）において、現在の設定状態を判断するための参照データとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/mappers/mod_speling.c:75 (メモリ確保時)
  • modules/mappers/mod_speling.c:107, 110, 113 (ディレクティブ定義時)
  • modules/mappers/mod_speling.c:199 (設定取得時)
  • **呼び出し先**: なし（データ構造のため）。

関連データ構造

  • command_rec (設定ディレクティブ定義に使用)
  • misspelled_file (スペル修正候補の保持に使用)

失敗モードと境界条件

  • 構造体の初期化に失敗した場合、apr_pcalloc の挙動に依存します。
  • 各メンバは int 型であるため、設定値が論理値として正しく解釈されることを前提としています。

保守担当者向けメモ

  • 本構造体のメンバを追加・変更した場合は、mod_speling.c 内の speling_cmds 配列における APR_OFFSETOF 指定を必ず更新してください。
