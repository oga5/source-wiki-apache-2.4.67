dav_deadprop_rollback

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/dbm.c:312

目的

WebDAV プロパティ（Dead Properties）の更新処理において、操作が失敗した場合やトランザクション的なロールバックが必要な際に、元のプロパティ値を保持・復元するためのデータ構造です。

シグネチャまたは形状

struct dav_deadprop_rollback {
    apr_datum_t key;
    apr_datum_t value;
};

入力と出力

  • **入力**: プロパティ更新前の DBM キーと値が apr_datum_t 型として格納されます。
  • **出力**: 該当なし（本構造体は状態保持のためのコンテナです）。

前提条件と不変条件

  • 本構造体はプロバイダー固有（provider-private）の構造体として定義されています。
  • apr_datum_t 型を使用しているため、メモリ管理は apr_pool_t に依存します。

アルゴリズムまたは振る舞い

  • プロパティの更新処理中に、変更前の値をこの構造体に退避させます。
  • 更新が失敗した場合、保持していた key と value を使用して DBM 上のデータを元の状態に戻すために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/fs/dbm.c 内のプロパティ更新ロジックから、ロールバック情報の生成および利用が行われます。
  • **呼び出し先**: modules/dav/main/props.c 等の上位レイヤーから、プロバイダーインターフェースを通じて間接的に参照されます。
  • **補足**: 具体的な関数呼び出しの連鎖については、提供されたソース情報からは断定できません。

関連データ構造

  • apr_datum_t: APR DBM で使用されるデータ保持用構造体。
  • dav_db: プロパティデータベースの接続状態を保持する構造体。

失敗モードと境界条件

  • メモリ不足: apr_pcalloc によるメモリ確保に失敗した場合、ロールバック情報の生成ができません。
  • DBM 整合性: ロールバック実行時に DBM 自体が破損している場合、復元操作は失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は mod_dav のプロバイダー実装においてプライベートな役割を担います。
  • modules/dav/main/mod_dav.h にも定義が存在するため、変更時はインターフェースの整合性に注意してください。
