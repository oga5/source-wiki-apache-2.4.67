backref_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_include.c:160

目的

SSI（Server Side Includes）処理における正規表現マッチングの結果および関連情報を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    const char *source;
    const char *rexp;
    apr_size_t  nsub;
    ap_regmatch_t match[AP_MAX_REG_MATCH];
    int have_match;
} backref_t;

入力と出力

  • **入力**: 正規表現のソース文字列、パターン文字列、およびマッチング結果の格納領域。
  • **出力**: マッチングの成否（have_match）およびキャプチャされたサブグループの範囲情報。

前提条件と不変条件

  • match 配列は AP_MAX_REG_MATCH サイズで固定されています。
  • have_match はマッチングが成功したか否かを示すフラグとして機能します。

アルゴリズムまたは振る舞い

SSI ディレクティブ実行時に正規表現による置換や条件判定が行われる際、この構造体を通じてマッチング状態が管理されます。マッチングが実行されると、source（対象文字列）と rexp（パターン）に基づき、match 配列に結果が格納されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_include.c 内の各ハンドラ関数（194, 655, 967, 1595 行付近）から、ctx->intern->re を介して参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • ssi_internal_ctx: この構造体を内部メンバとして保持し、SSI 解析の実行状態を管理します。
  • ap_regmatch_t: Apache HTTP Server の正規表現マッチング結果を格納する標準的な型です。

失敗モードと境界条件

  • AP_MAX_REG_MATCH を超えるサブグループを持つ正規表現は、この構造体では正しく扱えません。
  • re ポインタが NULL の場合、正規表現マッチングがまだ実行されていないことを示します。

保守担当者向けメモ

  • 本構造体は mod_include.c の内部的な状態保持用であり、外部モジュールへの公開は意図されていません。
  • nsub はマッチングされたサブグループの数を表しますが、使用時には have_match の値と併せて整合性を確認してください。
