send_environment

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:349

目的

HTTP リクエストに関連付けられた環境変数を FastCGI プロトコルの PARAMS レコードとしてシリアライズし、バックエンドへ送信する。

シグネチャまたは形状

apr_status_t send_environment(proxy_conn_rec *conn, request_rec *r,
                              apr_pool_t *temp_pool,
                              apr_uint16_t request_id)

入力と出力

  • **入力**:
  • conn: バックエンドへの接続情報。
  • r: 現在のリクエスト構造体。
  • temp_pool: 一時的なメモリ割り当て用プール。
  • request_id: FastCGI リクエスト識別子。
  • **出力**:
  • apr_status_t: 処理成功時は APR_SUCCESS、失敗時はエラーコードを返す。

前提条件と不変条件

  • r->subprocess_env に環境変数が格納されていること。
  • temp_pool は呼び出し元で管理され、本関数内で apr_pool_clear により適宜解放される。
  • FastCGI の多重化はサポートしておらず、request_id は単一のセッションを前提とする。

アルゴリズムまたは振る舞い

1. **環境変数の準備**: ap_add_common_vars および ap_add_cgi_vars を呼び出し、CGI 関連の環境変数を構築する。 2. **動的調整**: fix_cgivars を呼び出し、設定に基づいた環境変数の書き換えを行う。 3. **シリアライズ**: r->subprocess_env から環境変数を取得し、最大 16KB 単位の PARAMS レコードに分割してエンコードする。 4. **送信**: send_data を使用して、ヘッダとボディ（環境変数）をバックエンドへ送信する。 5. **終了通知**: 全ての環境変数の送信後、データ長 0 の PARAMS レコードを送信して終了を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_fcgi.c:1056
  • modules/aaa/mod_authnz_fcgi.c:742
  • **呼び出し先**:
  • fix_cgivars, send_data, ap_fcgi_encode_env, ap_fcgi_fill_in_header 等のユーティリティ関数。

関連データ構造

  • proxy_conn_rec: 接続管理用。
  • request_rec: リクエスト情報および環境変数テーブル。
  • fcgi_req_config_t, fcgi_dirconf_t: リクエストおよびディレクトリ単位の設定。

失敗モードと境界条件

  • 環境変数のエンコードに失敗した場合、警告ログを出力して該当変数をスキップする。
  • send_data がエラーを返した場合、即座にそのエラーを呼び出し元へ伝播する。
  • 環境変数が 16KB を超える場合、複数の PARAMS レコードに分割して送信される。

保守担当者向けメモ

  • 本関数は temp_pool を使用してメモリ効率を最適化している。
  • r->filename の一時的な書き換えと復元を行っているため、修正時はパス処理の整合性に注意すること。
