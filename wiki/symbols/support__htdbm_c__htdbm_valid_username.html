htdbm_valid_username

定義

  • 種別: function
  • 定義箇所: support/htdbm.c:263

目的

htdbm_t 構造体に保持されたユーザー名が、DBM データベースのキーとして許容される形式であるかを検証します。

シグネチャまたは形状

apr_status_t htdbm_valid_username(htdbm_t *htdbm)

入力と出力

  • **入力**: htdbm_t *htdbm （ユーザー名を含むコンテキスト構造体）
  • **出力**:
  • APR_SUCCESS: ユーザー名が有効な場合。
  • APR_EINVAL: ユーザー名が空、長すぎる、または不正な文字を含む場合。

前提条件と不変条件

  • htdbm->username は NULL でないか、または NULL の場合に APR_EINVAL を返します。
  • ユーザー名の長さは 1 文字以上 64 文字以下である必要があります。

アルゴリズムまたは振る舞い

1. htdbm->username が NULL であるか、文字列長が 1 未満または 64 を超える場合、標準エラー出力に "Invalid username length" を出力し、APR_EINVAL を返します。 2. ユーザー名にコロン (:) が含まれている場合、標準エラー出力に "Username contains invalid characters" を出力し、APR_EINVAL を返します。 3. 上記のいずれにも該当しない場合、APR_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: support/htdbm.c:404
  • **呼び出し先**:
  • strlen (標準 C ライブラリ)
  • strchr (標準 C ライブラリ)
  • fprintf (標準 C ライブラリ)

関連データ構造

  • htdbm_t: ユーザー名を含むメインのコンテキスト構造体。

失敗モードと境界条件

  • **長さの境界**: 0 文字および 65 文字以上のユーザー名は拒否されます。
  • **文字の制限**: コロン (:) は区切り文字として使用されるため、ユーザー名には含められません。

保守担当者向けメモ

  • 本関数は htdbm ユーティリティにおける入力バリデーションの要所です。将来的にユーザー名の文字制限を緩和または厳格化する場合は、この関数を更新してください。
