set_h1_header

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_request.c:70

目的

apr_table_t に格納されたヘッダーを、HTTP/2 リクエスト処理用の内部構造体へ転送・追加するためのコールバック関数です。

シグネチャまたは形状

static int set_h1_header(void *ctx, const char *key, const char *value)

入力と出力

  • **入力**:
  • ctx: h1_ctx 構造体へのポインタ（ヘッダー格納先とメモリプールを保持）。
  • key: ヘッダー名。
  • value: ヘッダー値。
  • **出力**:
  • 常に 1 を返します（apr_table_do のイテレーションを継続するため）。

前提条件と不変条件

  • ctx は h1_ctx 型としてキャスト可能である必要があります。
  • h1_ctx 内の headers および pool が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. ctx を h1_ctx 型にキャストします。 2. h2_req_add_header を呼び出し、key と value を h1_ctx 内のヘッダーテーブルへ追加します。 3. この際、key と value の長さを strlen で算出し、h1_ctx 内のスクラッチ領域を使用して処理を行います。 4. 処理完了後、1 を返して呼び出し元へ制御を戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_proxy_util.c:640 (apr_table_do 経由)
  • modules/http2/h2_request.c:131 (apr_table_do 経由)
  • **呼び出し先**:
  • h2_req_add_header (HTTP/2 リクエストへのヘッダー追加処理)

関連データ構造

  • h1_ctx: ヘッダー追加処理に必要なコンテキスト（ヘッダーテーブル、メモリプール、スクラッチ領域を保持）。

失敗モードと境界条件

  • h2_req_add_header 内部でのメモリ割り当て失敗や、不正なヘッダー形式に対する挙動は h2_req_add_header の実装に依存します。本関数自体はエラー発生時も 1 を返し、イテレーションを中断しません。

保守担当者向けメモ

  • 本関数は apr_table_do のコールバックとして設計されています。シグネチャを変更する場合は、apr_table_do の仕様に適合させる必要があります。
