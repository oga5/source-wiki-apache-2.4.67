md_http_cert_status

定義

  • 種別: function
  • 定義箇所: modules/md/mod_md_status.c:60

目的

Managed Domain (MD) の証明書ステータス情報を JSON 形式でクライアントに提供する HTTP ハンドラ。

シグネチャまたは形状

int md_http_cert_status(request_rec *r);

入力と出力

  • **入力**: request_rec *r (Apache HTTP リクエストコンテキスト)
  • **出力**:
  • 成功時: DONE (HTTP 200 OK、Content-Type: application/json)
  • 失敗時: DECLINED (リクエストが対象外の場合)、HTTP_NOT_IMPLEMENTED (GET 以外)、HTTP_INTERNAL_SERVER_ERROR (データ取得失敗)

前提条件と不変条件

  • r->parsed_uri.path が MD_STATUS_RESOURCE と一致する必要がある。
  • サーバー設定で certificate_status_enabled が有効である必要がある。
  • リクエストされたホスト名が管理対象ドメイン (md_t) として登録されている必要がある。
  • HTTP メソッドは GET のみに限定される。

アルゴリズムまたは振る舞い

1. リクエストパスと設定を確認し、対象外であれば DECLINED を返す。 2. md_status_get_md_json を呼び出し、ドメインのステータス情報を JSON オブジェクトとして取得する。 3. 取得した JSON から、証明書の有効性、シリアル番号、SHA256 フィンガープリント、および更新情報を抽出・再構築する。 4. 構築した JSON を apr_bucket_brigade を介してレスポンスボディとして書き出し、DONE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/mod_md.c において ap_hook_post_read_request に登録されている。
  • **呼び出し先**:
  • md_get_by_domain (ドメイン検索)
  • md_status_get_md_json (ステータスデータ生成)
  • md_json_* 系関数 (JSON 操作)
  • ap_pass_brigade (レスポンス出力)

関連データ構造

  • request_rec: Apache リクエスト情報。
  • md_t: 管理対象ドメイン情報。
  • md_json_t: ステータス情報の保持に使用される JSON コンテナ。
  • md_srv_conf_t: モジュールのサーバー設定。

失敗モードと境界条件

  • GET 以外のメソッドが指定された場合、HTTP_NOT_IMPLEMENTED を返す。
  • ステータス情報の取得 (md_status_get_md_json) が失敗した場合、HTTP_INTERNAL_SERVER_ERROR を返す。
  • 設定が無効な場合やドメインが見つからない場合は DECLINED を返し、他のハンドラに処理を委譲する。

保守担当者向けメモ

  • 本関数は r->pool を使用してメモリを管理しているため、明示的な解放は不要。
  • JSON の出力フォーマットは MD_JSON_FMT_INDENT を使用しており、人間が読みやすい形式で出力される。
  • 証明書の更新ロジックそのものは含まれておらず、あくまでステータスの公開に特化している。
