beam_out

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_c2.c:356

目的

セカンダリ接続（c2）から生成されたレスポンスデータを、h2_bucket_beam を介して HTTP/2 セッション側へ送出する。

シグネチャまたは形状

static apr_status_t beam_out(conn_rec *c2, h2_conn_ctx_t *conn_ctx, apr_bucket_brigade* bb)

入力と出力

  • **入力**:
  • c2: セカンダリ接続の接続構造体。
  • conn_ctx: 接続に関連付けられた HTTP/2 コンテキスト。
  • bb: 送出対象のバケットブリゲード。
  • **出力**:
  • apr_status_t: 操作の結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • conn_ctx->beam_out が初期化済みであること。
  • bb は h2_beam_send に渡すために有効な状態であること。

アルゴリズムまたは振る舞い

1. h2_beam_send を呼び出し、conn_ctx->beam_out を使用して bb 内のデータを転送する。 2. 転送時に APR_BLOCK_READ を指定し、同期的な書き込みを試みる。 3. h2_beam_send が APR_EAGAIN を返した場合、これを APR_SUCCESS に変換して呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/h2_c2.c:409 (h2_c2_filter_out 内から呼び出される)。
  • **呼び出し先**: h2_beam_send (HTTP/2 セッションへのデータ転送用 API)。

関連データ構造

  • conn_rec: Apache HTTP Server の接続構造体。
  • h2_conn_ctx_t: HTTP/2 接続コンテキスト。beam_out メンバを保持する。
  • apr_bucket_brigade: データの断片（バケット）を保持するリスト構造。

失敗モードと境界条件

  • h2_beam_send が APR_EAGAIN 以外のエラーを返した場合、そのステータスがそのまま呼び出し元へ伝播する。
  • conn_ctx が無効な場合、セグメンテーションフォールトが発生する可能性がある（呼び出し元でのガード状況は本関数内では未定義）。

保守担当者向けメモ

  • 本関数は h2_c2_filter_out のラッパーとして機能しており、フィルタチェーンと h2_bucket_beam 間のインターフェースを抽象化している。
  • APR_EAGAIN の抑制は、HTTP/2 セッション側でのフロー制御や一時的なバッファフル状態を透過的に扱うための設計である。
