host_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_host.c:163

目的

クライアントのホスト名に基づき、Require host ディレクティブで指定された条件と照合してアクセス認可を判定する。

シグネチャまたは形状

authz_status host_check_authorization(request_rec *r,
                                      const char *require_line,
                                      const void *parsed_require_line)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • require_line: 設定ファイルに記述された Require 行の文字列。
  • parsed_require_line: コンパイル済みの ap_expr_info_t 構造体。
  • **出力**:
  • AUTHZ_GRANTED: ホスト名が許可リストのいずれかに一致した場合。
  • AUTHZ_DENIED: ホスト名の取得失敗、式評価エラー、または一致するホスト名がない場合。

前提条件と不変条件

  • ap_get_useragent_host を使用してクライアントのホスト名を取得する。
  • REMOTE_DOUBLE_REV フラグにより、DNS 逆引きが必須となる。
  • parsed_require_line は ap_expr_info_t 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. ap_get_useragent_host を呼び出し、クライアントのホスト名を取得する。取得できない場合や IP アドレスとして解決された場合は拒否する。 2. ap_expr_str_exec を使用して parsed_require_line を評価し、対象となるホスト名のリスト（文字列）を取得する。 3. リスト内に # が含まれる場合、コメントとして扱い、それ以降の文字列を無視する。 4. ap_getword_conf を用いてリストを単語単位に分割し、各単語に対して in_domain 関数でクライアントのホスト名との照合を行う。 5. 一つでも一致すれば AUTHZ_GRANTED を返し、すべて不一致であれば AUTHZ_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_host.c 内で authz_provider 構造体に登録され、mod_authz_core から呼び出される（直接の呼び出し箇所として modules/aaa/mod_authz_host.c:338 に参照あり）。
  • **呼び出し先**:
  • ap_get_useragent_host
  • ap_log_rerror
  • ap_expr_str_exec
  • ap_strchr_c
  • apr_pstrmemdup
  • ap_getword_conf
  • in_domain

関連データ構造

  • request_rec: リクエスト情報。
  • ap_expr_info_t: Require 行の評価式。
  • authz_status: 認可結果の列挙型。

失敗モードと境界条件

  • **ホスト名解決失敗**: remotehost が NULL または IP アドレスである場合、エラーログを出力し AUTHZ_DENIED を返す。
  • **式評価エラー**: ap_expr_str_exec がエラーを返した場合、ログを出力し AUTHZ_DENIED を返す。
  • **不正なコメント**: Require host # のように # のみ、または # で始まる記述がある場合、警告またはエラーログを出力し AUTHZ_DENIED を返す。

保守担当者向けメモ

  • 本関数は DNS 逆引きに依存しているため、DNS サーバーの応答遅延がリクエスト処理時間に直結する。
  • in_domain 関数によるドメインマッチングは、サブドメインの包含関係を判定する。
  • parsed_require_line の評価結果に含まれる # の処理は、設定の誤用を防ぐための安全策である。
