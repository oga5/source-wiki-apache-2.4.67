authn_cache_setprovider

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authn_socache.c:227

目的

Apache 設定ディレクティブ AuthnCacheProvideFor を介して指定された認証プロバイダー名を、ディレクトリごとの設定構造体（authn_cache_dircfg）に登録します。

シグネチャまたは形状

static const char *authn_cache_setprovider(cmd_parms *cmd, void *CFG, const char *arg)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • CFG: 現在のディレクトリ設定構造体（authn_cache_dircfg）。
  • arg: 設定ファイルから渡される認証プロバイダー名（文字列）。
  • **出力**:
  • 成功時は NULL を返します。

前提条件と不変条件

  • CFG は authn_cache_dircfg 型としてキャスト可能である必要があります。
  • cmd->pool は、プロバイダーリストを保持する配列の割り当てに使用されます。

アルゴリズムまたは振る舞い

1. cfg->providers が未初期化の場合、cmd->pool を使用して 4 要素分の配列を新規作成します。 2. arg で指定されたプロバイダー名を配列の末尾に追加します。 3. グローバル変数 configured を 1 に設定し、モジュールが設定済みであることを示します。 4. NULL を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authn_socache.c:256 にある AP_INIT_ITERATE マクロ（AuthnCacheProvideFor ディレクティブの定義）から呼び出されます。
  • **呼び出し先**:
  • apr_array_make（APR 配列の初期化）
  • APR_ARRAY_PUSH（配列への要素追加）

関連データ構造

  • authn_cache_dircfg: ディレクトリごとの設定を保持する構造体。providers メンバ（apr_array_header_t*）を更新します。

失敗モードと境界条件

  • メモリ不足時には apr_array_make がエラーを発生させる可能性があります。
  • AuthnCacheProvideFor が複数回呼び出された場合、プロバイダー名は順次配列に追加されます。

保守担当者向けメモ

  • 本関数は AuthnCacheProvideFor ディレクティブ専用のハンドラです。
  • configured フラグはグローバル変数であり、設定の有効化状態を追跡するために使用されています。
