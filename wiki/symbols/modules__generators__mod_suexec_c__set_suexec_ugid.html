set_suexec_ugid

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_suexec.c:58

目的

Apache 設定ファイル内の SuexecUserGroup ディレクティブを処理し、指定されたユーザー名とグループ名を UID/GID に変換して設定構造体に格納する。

シグネチャまたは形状

static const char *set_suexec_ugid(cmd_parms *cmd, void *mconfig,
                                   const char *uid, const char *gid)

入力と出力

  • **入力**:
  • cmd: 設定コマンドのパラメータ構造体。
  • mconfig: 現在のディレクトリ設定構造体 (suexec_config_t)。
  • uid: 設定ファイルで指定されたユーザー名。
  • gid: 設定ファイルで指定されたグループ名。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • SuexecUserGroup はディレクトリコンテキスト (<Directory> 等) 内では使用できない（NOT_IN_DIR_CONTEXT）。
  • システム全体で suEXEC が有効 (ap_unixd_config.suexec_enabled) である必要がある。

アルゴリズムまたは振る舞い

1. ap_check_cmd_context を呼び出し、ディレクトリコンテキスト内での使用を拒否する。 2. ap_unixd_config.suexec_enabled を確認し、無効な場合はエラーメッセージを返す。 3. ap_uname2id(uid) および ap_gname2id(gid) を使用して、名前を UID/GID に変換する。 4. 変換結果を mconfig (suexec_config_t) に格納する。 5. userdir フラグを 0 に設定し、active フラグを 1 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_suexec.c:119 において、AP_INIT_TAKE2 マクロを通じて SuexecUserGroup ディレクティブのハンドラとして登録されている。
  • **呼び出し先**:
  • ap_check_cmd_context
  • apr_pstrcat
  • ap_uname2id
  • ap_gname2id

関連データ構造

  • suexec_config_t: 設定情報を保持する構造体（mod_suexec.h で定義）。
  • cmd_parms: Apache 設定コマンドのパラメータ構造体。

失敗モードと境界条件

  • ディレクトリコンテキスト内で設定された場合、エラーを返す。
  • suEXEC が無効な環境で設定された場合、ap_unixd_config.suexec_disabled_reason を含むエラーメッセージを返す。

保守担当者向けメモ

  • ap_uname2id および ap_gname2id はスレッドセーフではない可能性があるため、.htaccess での利用は制限されている。
  • 本関数は設定の格納のみを行い、実際のプロセス切り替えは外部の suexec バイナリが担当する。
