cmd_alert

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:180

目的

Lua スクリプトから Apache のログシステムに対し、APLOG_ALERT レベルでメッセージを出力するためのインターフェースを提供します。

シグネチャまたは形状

static int cmd_alert(lua_State *L)

入力と出力

  • **入力**: lua_State *L（Lua の実行スタック）。スタック上にはログ出力対象のメッセージやコンテキスト情報が含まれることが期待されます。
  • **出力**: int（Lua のスタックに積まれた戻り値の数）。内部で呼び出される cmd_log_at の戻り値をそのまま返します。

前提条件と不変条件

  • lua_State *L は有効な Lua VM の状態を指している必要があります。
  • 内部で cmd_log_at を呼び出すため、ログ出力に必要なソースファイル名や行番号などの情報が Lua のスタックから正しく取得できる状態である必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った lua_State *L を cmd_log_at 関数に渡します。 2. その際、ログレベルとして APLOG_ALERT を指定します。 3. cmd_log_at の実行結果（通常はスタック操作の結果）を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_config.c:256 にある cmd_methods テーブル内で、Lua からのメソッド呼び出し（alert）に対応する関数として登録されています。
  • **呼び出し先**: cmd_log_at（同ファイル内）。

関連データ構造

  • cmd_methods: CommandParameters 用のメソッドテーブル。本関数はこのテーブルの一部として定義されています。

失敗モードと境界条件

  • Lua のスタックが不正な場合や、cmd_log_at 内部でログ出力のコンテキスト（ファイル名や行番号）が特定できない場合、cmd_log_at の振る舞いに依存します。

保守担当者向けメモ

  • 本関数は cmd_log_at のラッパーです。ログレベルを変更する必要がある場合は、APLOG_ALERT 定数を変更するか、別の cmd_* 関数を参照してください。
  • APLOGNO マクロは意図的に省略されています。
