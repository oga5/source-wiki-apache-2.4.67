merge_proxy_ftp_dir_config

定義

  • **種別**: 関数
  • **定義箇所**: modules/proxy/mod_proxy_ftp.c:56

目的

FTP プロキシのディレクトリ単位の設定構造体（proxy_ftp_dir_conf）をマージし、親コンテキスト（base）と子コンテキスト（add）の設定を統合して新しい設定を生成する。

シグネチャまたは形状

static void *merge_proxy_ftp_dir_config(apr_pool_t *p, void *basev, void *addv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する APR プール。
  • basev: 親ディレクトリの設定構造体（proxy_ftp_dir_conf）。
  • addv: 現在のディレクトリ（または <Location> 等）の設定構造体。
  • **出力**:
  • マージ後の新しい proxy_ftp_dir_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および addv は proxy_ftp_dir_conf 型としてキャスト可能であること。
  • p は有効なメモリプールであること。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい proxy_ftp_dir_conf 構造体をプール上に確保する。 2. ftp_list_on_wildcard および ftp_escape_wildcards について、子設定側で値が設定されている（_set フラグが真）場合はその値を採用し、そうでない場合は親設定の値を継承する。 3. ftp_directory_charset については、子設定側で値が指定されていればそれを採用し、なければ親設定の値を継承する。 4. 最終的にマージされた構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ftp.c:2111（モジュール定義内の設定マージ処理として参照されている）。
  • **呼び出し先**: 直接的な関数呼び出しはなし（APR のメモリ割り当て関数 apr_pcalloc を使用）。

関連データ構造

  • proxy_ftp_dir_conf: FTP ディレクトリ設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗し、プロセスが終了する可能性がある。
  • _set フラグが正しく管理されていない場合、意図しないデフォルト値が継承される可能性がある。

保守担当者向けメモ

  • 本関数は <Location> 等のネストされた設定を正しく継承するために使用される。新しい設定項目を proxy_ftp_dir_conf に追加した場合は、必ず本関数でマージロジックを更新すること。
