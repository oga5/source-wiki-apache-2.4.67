md_store_save

定義

  • 種別: function
  • 定義箇所: modules/md/md_store.c:78

目的

Managed Domain (MD) サブシステムにおいて、指定されたデータ（証明書、秘密鍵、JSON設定など）を永続化ストレージへ保存するための抽象化インターフェースを提供します。

シグネチャまたは形状

apr_status_t md_store_save(md_store_t *store, apr_pool_t *p, md_store_group_t group, 
                           const char *name, const char *aspect, 
                           md_store_vtype_t vtype, void *data, 
                           int create);

入力と出力

  • **入力**:
  • store: ストレージの抽象ハンドル。
  • p: メモリ割り当て用プール。
  • group: 保存先グループ（MD_SG_DOMAINS, MD_SG_ACCOUNTS 等）。
  • name: 対象ドメイン名または識別子。
  • aspect: 保存するデータの種類（ファイル名の一部となる）。
  • vtype: データの型（MD_SV_JSON, MD_SV_PKEY, MD_SV_CHAIN 等）。
  • data: 保存するデータへのポインタ。
  • create: 新規作成を許可するかどうかのフラグ。
  • **出力**:
  • apr_status_t: 操作の成功（APR_SUCCESS）または失敗を示すステータスコード。

前提条件と不変条件

  • store 引数が有効であり、その内部の関数ポインタ save が適切に初期化されている必要があります。
  • p プールは、呼び出し元によって適切に管理されている必要があります。

アルゴリズムまたは振る舞い

本関数は、md_store_t 構造体が保持する関数ポインタ store->save を呼び出すラッパーです。具体的なファイルI/Oや排他制御のロジックは、store に設定された具体的な実装（md_store_fs.c など）に委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • md_acme_acct.c: アカウント情報の保存。
  • md_acme_authz.c: チャレンジデータの保存。
  • md_acme_drive.c: ステージングデータの保存。
  • md_reg.c: HTTPD設定の保存。
  • md_store.c: md_store_save_json、md_pkey_save、md_pubcert_save 等のヘルパー関数。
  • **呼び出し先**:
  • store->save: md_store_t 構造体内の関数ポインタ（具体的なストレージ実装）。

関連データ構造

  • md_store_t: ストレージの抽象インターフェース。
  • md_store_group_t: ストレージグループの列挙型。
  • md_store_vtype_t: 保存データの型定義。

失敗モードと境界条件

  • store が NULL の場合、または store->save が未定義の場合、セグメンテーションフォールトが発生する可能性があります。
  • ストレージバックエンド（ファイルシステム等）の権限不足やディスクフルにより、APR エラーコードが返されます。
  • create フラグが 0 の場合、対象が存在しないと失敗する可能性があります。

保守担当者向けメモ

  • 本関数は単なる委譲レイヤーです。実際の保存ロジックを修正する場合は、md_store_t の実装側（md_store_fs.c 等）を確認してください。
  • メモリ管理は apr_pool_t に依存しているため、呼び出し元でのプール生存期間に注意してください。
