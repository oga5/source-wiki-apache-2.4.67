h2_bucket_headers_headers_length

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_headers.c:146

目的

H2HEADERS 型のカスタムバケット内に保持されているヘッダー情報の合計バイト数を算出します。

シグネチャまたは形状

apr_size_t h2_bucket_headers_headers_length(apr_bucket *b)

入力と出力

  • **入力**: apr_bucket *b - ヘッダー情報を含むバケットへのポインタ。
  • **出力**: apr_size_t - ヘッダーの合計バイト数。バケットが空または無効な場合は 0 を返します。

前提条件と不変条件

  • 入力バケット b は H2HEADERS 型のデータを含んでいる必要があります。
  • 内部で呼び出される h2_bucket_headers_get(b) により、バケットから h2_headers 構造体が正しく取得できることが前提です。

アルゴリズムまたは振る舞い

1. h2_bucket_headers_get(b) を呼び出し、バケットから h2_headers 構造体を取得します。 2. 取得した構造体が NULL でない場合、h2_headers_length(h) を呼び出してヘッダーの合計サイズを計算し、その値を返します。 3. 構造体が NULL の場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • h2_bucket_headers_get
  • h2_headers_length
  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供されていませんが、modules/http2/h2_headers.h:100 にプロトタイプ宣言が存在します。

関連データ構造

  • apr_bucket: Apache Portable Runtime のバケット構造体。
  • h2_headers: ヘッダー情報（ステータス、テーブル、ノート）を保持する構造体。

失敗モードと境界条件

  • バケット b が NULL または H2HEADERS 型ではない場合、h2_bucket_headers_get の挙動に依存します。
  • ヘッダーテーブルが空の場合、h2_headers_length は 0 を返します。

保守担当者向けメモ

  • 本関数はテーブル内の全ヘッダーを走査するため、ヘッダー数が多い場合にはパフォーマンスへの影響を考慮してください。
  • AP_HAS_RESPONSE_BUCKETS が定義されている環境では、このモジュールのロジックは無効化される点に注意してください。
