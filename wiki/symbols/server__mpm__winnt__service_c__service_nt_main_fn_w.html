service_nt_main_fn_w

定義

  • 種別: function
  • 定義箇所: server/mpm/winnt/service.c:376

目的

Windowsサービスコントロールマネージャ（SCM）から呼び出されるサービスのエントリポイントとして、サービスの初期化、状態通知、およびコマンドライン引数のマージを実行します。

シグネチャまたは形状

void __stdcall service_nt_main_fn_w(DWORD argc, LPWSTR *argv)

入力と出力

  • **入力**:
  • argc: SCMから渡された引数の数。
  • argv: SCMから渡された引数の配列（Wide文字列）。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • globdat（nt_service_ctx_t 型）がグローバルなサービスコンテキストとして初期化されていること。
  • mpm_new_argv が有効な引数リスト構造体として存在していること。
  • RegisterServiceCtrlHandlerExW が成功し、サービス制御ハンドラが登録されていること。

アルゴリズムまたは振る舞い

1. **サービス名の設定**: argv[0] をUTF-8に変換し、mpm_service_set_name を呼び出してサービス名を登録します。 2. **状態初期化**: globdat の ssStatus を SERVICE_START_PENDING に設定し、RegisterServiceCtrlHandlerExW で service_nt_ctrl をハンドラとして登録します。 3. **SCMへの通知**: ReportStatusToSCMgr を呼び出し、初期化中であることをSCMに通知します。 4. **引数のマージ**: argc > 1 の場合、SCMから渡された引数を mpm_new_argv に追加・統合します。 5. **同期**: ctx->service_init イベントをセットしてメインスレッドの処理を再開させ、その後 ctx->service_term イベントがシグナルされるまで待機（WaitForSingleObject）します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/mpm/winnt/service.c:516（SERVICE_TABLE_ENTRYW 構造体内のエントリとして参照）。
  • **呼び出し先**:
  • apr_conv_ucs2_to_utf8
  • mpm_service_set_name
  • RegisterServiceCtrlHandlerExW
  • ap_log_error
  • ReportStatusToSCMgr
  • SetEvent
  • WaitForSingleObject

関連データ構造

  • nt_service_ctx_t (globdat): サービスの状態や同期イベントを保持するグローバル構造体。
  • SERVICE_STATUS: SCMに報告するサービス状態構造体。

失敗モードと境界条件

  • **ハンドラ登録失敗**: RegisterServiceCtrlHandlerExW が失敗した場合、エラーログを出力して即座に終了します。
  • **メモリ不足**: malloc によるメモリ確保が失敗した場合の明示的なハンドリングはソース内に記述されていません。

保守担当者向けメモ

  • 本関数は __stdcall 呼び出し規約を使用しており、SCMの仕様に準拠しています。
  • mpm_new_argv のメモリ管理は pconf プールに依存しているため、引数マージ時のメモリ確保と解放のライフサイクルには注意が必要です。
  • argc が1より大きい場合、引数リストの再構築が行われます。この際、mpm_new_argv->elts のポインタが更新されるため、他のスレッドがこのリストを参照している場合は競合に注意してください。
