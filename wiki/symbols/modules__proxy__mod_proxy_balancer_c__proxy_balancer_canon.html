proxy_balancer_canon

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_balancer.c:71

目的

balancer:// スキームで指定されたプロキシリクエストの URL を正規化し、内部的なプロキシパス形式へ変換する。

シグネチャまたは形状

int proxy_balancer_canon(request_rec *r, char *url)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • url: 正規化対象の URL 文字列。
  • **出力**:
  • OK: 正常終了。r->filename が更新される。
  • DECLINED: balancer: スキームではない場合。
  • HTTP_BAD_REQUEST: URL の解析に失敗した場合。

前提条件と不変条件

  • 入力された url が balancer: で始まる必要がある。
  • ap_proxy_canon_netloc を使用してホスト名とポートの構文チェックを行う。

アルゴリズムまたは振る舞い

1. url が balancer: で始まるか確認し、一致しない場合は DECLINED を返す。 2. ap_proxy_canon_netloc を呼び出し、URL をホスト、ポート、パスに分解する。解析失敗時は HTTP_BAD_REQUEST を返す。 3. r->filename を proxy:balancer:<host>/<url> という形式の文字列で構築する。 4. OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: proxy_hook_canon_handler に登録されており、Apache のプロキシハンドラフックから呼び出される。
  • **呼び出し先**:
  • ap_log_rerror: ログ出力。
  • ap_proxy_canon_netloc: URL のネットワーク部分の解析。
  • apr_pstrcat: r->filename の構築。

関連データ構造

  • request_rec: Apache のリクエスト情報構造体。

失敗モードと境界条件

  • balancer: スキーム以外が渡された場合、処理を行わず DECLINED を返す。
  • URL の構文が不正な場合、ap_proxy_canon_netloc がエラーを返し、関数は HTTP_BAD_REQUEST で終了する。

保守担当者向けメモ

  • 本関数は balancer_fixup() で行われる BalancerMember 単位の正規化の前段階として、URL の基本構造を整える役割を持つ。
  • BALANCER_PREFIX マクロが定義されている前提で動作する。
