buffer_in_filter

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_buffer.c:189

目的

入力フィルタチェーンにおいて、上流から読み込んだデータをバッファリングし、呼び出し元が要求するサイズ分を効率的に提供する。

シグネチャまたは形状

apr_status_t buffer_in_filter(ap_filter_t *f, apr_bucket_brigade *bb,
                              ap_input_mode_t mode, apr_read_type_e block,
                              apr_off_t readbytes)

入力と出力

  • **入力**: フィルタ構造体 f、バケットブリゲード bb、読み込みモード mode、ブロック設定 block、要求バイト数 readbytes。
  • **出力**: apr_status_t 型のステータスコード。成功時は APR_SUCCESS を返し、bb に要求されたデータが格納される。

前提条件と不変条件

  • AP_MODE_READBYTES モード時のみバッファリングを行う。
  • 初期リクエスト（ap_is_initial_req）に対してのみ有効。
  • フィルタコンテキスト f->ctx はリクエストプールに紐付いており、リクエスト終了時に自動解放される。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時に buffer_ctx を作成し、設定を読み込む。 2. **透過処理**: 初期リクエストでない場合、または AP_MODE_READBYTES 以外のモードでは、下流のフィルタへ処理を委譲する。 3. **バッファリング**: バッファが空の場合、設定された BufferSize に達するか、EOS/Flush バケットに遭遇するまで上流からデータを読み込み、内部の ctx->bb に蓄積する。 4. **データ提供**: apr_brigade_partition を使用して、蓄積されたデータから要求された readbytes 分を切り出し、呼び出し元の bb へ移動させる。 5. **終了処理**: EOS バケットを読み込んだ場合、フィルタをチェーンから削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache フィルタチェーン（ap_get_brigade 経由）。
  • **呼び出し先**: ap_get_brigade（上流フィルタ）、ap_remove_input_filter。
  • 補足: modules/filters/mod_buffer.c:341 にて ap_register_input_filter により登録されている。

関連データ構造

  • buffer_ctx: 実行時の状態（メインバッファ bb、一時バッファ tmp、残り容量 remaining 等）を保持。
  • buffer_conf: ディレクトリ単位の設定（バッファサイズ上限）。

失敗モードと境界条件

  • **エラー発生**: 上流の ap_get_brigade がエラーを返した場合、そのステータスをそのまま返す。
  • **データ不足**: 上流からデータが取得できない場合、空のブリゲードを返す。
  • **サブクエスト**: 初期リクエストでない場合はバッファリングを行わず、フィルタを即座に削除する。

保守担当者向けメモ

  • 入力フィルタは AP_MODE_READBYTES 以外のモード（GETLINE 等）ではバッファリングを行わず透過する仕様である。
  • メモリ管理はリクエストプールに依存しているため、明示的な解放は不要。
