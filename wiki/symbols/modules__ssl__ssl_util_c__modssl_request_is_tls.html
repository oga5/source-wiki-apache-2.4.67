modssl_request_is_tls

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_util.c:103

目的

現在処理中のリクエストが TLS 接続経由であるかを判定し、必要に応じて接続コンテキスト（SSLConnRec）を取得する。

シグネチャまたは形状

int modssl_request_is_tls(const request_rec *r, SSLConnRec **scout)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体 (request_rec)。
  • scout: 接続コンテキストを格納するためのポインタ（NULL 指定可能）。
  • **出力**:
  • 戻り値: TLS 接続であれば 1、それ以外は 0。
  • scout: 戻り値が 1 の場合、引数で渡されたポインタに現在の接続コンテキストが格納される。

前提条件と不変条件

  • r および r->connection は有効であること。
  • サーバー設定 (SSLSrvConfigRec) が取得可能であること。

アルゴリズムまたは振る舞い

1. myConnConfig を用いて現在の接続から SSLConnRec を取得する。 2. 接続が SSL を保持していない場合、r->connection->master を参照してマスター接続から再度 SSLConnRec の取得を試みる。 3. サーバー設定 (sc) を確認し、SSL が無効化されている場合、または接続コンテキストが存在しない場合は 0 を返す。 4. scout が NULL でない場合、取得した sslconn を代入する。 5. 上記を通過した場合、1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/ssl/mod_ssl.c: リクエストのスキーム判定やポート番号の解決に使用。
  • modules/ssl/ssl_engine_kernel.c: リクエスト処理中の SSL 状態検証に使用。
  • **呼び出し先**:
  • myConnConfig (マクロ)
  • mySrvConfig (マクロ)

関連データ構造

  • request_rec: Apache のリクエスト構造体。
  • SSLConnRec: 接続ごとの SSL 状態を保持するモジュール内部構造体。
  • SSLSrvConfigRec: サーバーごとの SSL 設定構造体。

失敗モードと境界条件

  • r->connection が SSL 接続でない場合、またはマスター接続も存在しない場合は 0 を返す。
  • サーバー設定で SSL が明示的に無効 (SSL_ENABLED_FALSE) に設定されている場合、TLS 接続であっても 0 を返す。

保守担当者向けメモ

  • 本関数は ssl_private.h で宣言されており、モジュール内の広範囲で使用される。
  • r->connection->master へのフォールバック処理は、HTTP/2 等のマルチプレクス接続において、サブストリームから元の SSL 接続情報を辿るために不可欠である。
