dbmfilegroup_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_dbm.c:239

目的

DBM ファイルに記録されたユーザーの所属グループと、対象ファイルの所有グループを照合し、アクセス認可を判定する。

シグネチャまたは形状

static authz_status dbmfilegroup_check_authorization(request_rec *r,
                                                     const char *require_args,
                                                     const void *parsed_require_args)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • require_args: 未使用。
  • parsed_require_args: 未使用。
  • **出力**:
  • AUTHZ_GRANTED: ユーザーがファイルの所有グループに所属している場合。
  • AUTHZ_DENIED: ユーザーが未認証、グループファイル未設定、DBM 検索失敗、またはグループ不一致の場合。
  • AUTHZ_DENIED_NO_USER: ユーザー名がリクエストに存在しない場合。

前提条件と不変条件

  • r->user が設定されていること。
  • authz_dbm_config_rec 設定構造体において grpfile が定義されていること。
  • authz_owner_get_file_group 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. ユーザー名が空であれば AUTHZ_DENIED_NO_USER を返す。 2. 設定された DBM グループファイルが存在しない場合はエラーログを出力し AUTHZ_DENIED を返す。 3. get_dbm_grp を呼び出し、user:realm をキーとして DBM からユーザーの所属グループリストを取得する。 4. DBM にユーザーが存在しない場合はエラーログを出力し AUTHZ_DENIED を返す。 5. authz_owner_get_file_group(r) を呼び出し、対象ファイルの所有グループを取得する。 6. 取得した所属グループリスト（カンマ区切り）をパースし、ファイルの所有グループと一致するものが存在すれば AUTHZ_GRANTED を返す。 7. 一致するものがない場合はエラーログを出力し AUTHZ_DENIED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/aaa/mod_authz_dbm.c 内の authz_dbmfilegroup_provider 構造体を通じて参照される（直接の関数呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • ap_get_module_config
  • ap_auth_name
  • ap_log_rerror
  • get_dbm_grp
  • apr_pstrcat
  • authz_owner_get_file_group
  • ap_getword

関連データ構造

  • request_rec: リクエスト情報およびプール管理。
  • authz_dbm_config_rec: DBM ファイルパスやタイプを保持するディレクトリ設定。
  • authz_status: 認可結果の列挙型。

失敗モードと境界条件

  • **ユーザー未認証**: r->user が NULL の場合に AUTHZ_DENIED_NO_USER を返す。
  • **設定不備**: grpfile が未設定の場合、ログ出力後に AUTHZ_DENIED を返す。
  • **DBM 検索失敗**: get_dbm_grp が APR_SUCCESS 以外を返した場合、AUTHZ_DENIED を返す。
  • **グループ不一致**: ユーザーが DBM に存在しても、ファイルの所有グループと一致しない場合は AUTHZ_DENIED を返す。

保守担当者向けメモ

  • 本関数は mod_authz_owner モジュールが提供する authz_owner_get_file_group に強く依存している。
  • メモリ割り当てはすべて r->pool を使用しているため、明示的な解放は不要。
  • DBM ファイルの検索キーとして user:realm を使用している点に注意。
