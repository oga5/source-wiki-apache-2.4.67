features

定義

  • 種別: struct
  • 定義箇所: modules/http2/mod_http2.c:65

目的

HTTP/2 モジュールにおける特定の機能フラグを保持するための構造体です。

シグネチャまたは形状

typedef struct {
    unsigned int change_prio : 1;
    unsigned int sha256 : 1;
    unsigned int inv_headers : 1;
    unsigned int dyn_windows : 1;
} features;

入力と出力

  • **入力**: 構造体の各ビットフィールドは、unsigned int 型のフラグとして機能します。
  • **出力**: なし（本構造体は状態保持のためのデータコンテナです）。

前提条件と不変条件

  • 各フィールドはビットフィールド（1ビット）として定義されており、真偽値（0または1）のみを保持します。

アルゴリズムまたは振る舞い

  • change_prio: 優先度の変更機能の有効/無効を管理します。
  • sha256: SHA-256 ハッシュ機能の有効/無効を管理します。
  • inv_headers: 無効なヘッダーの扱いに関するフラグです。
  • dyn_windows: 動的ウィンドウ制御の有効/無効を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c:72 にて static features myfeats; として静的変数定義に使用されています。
  • **呼び出し先**: 提供されたソース情報からは、本構造体を利用している具体的な関数呼び出し関係は確認できません。

関連データ構造

  • myfeats（modules/http2/mod_http2.c 内の静的変数）

失敗モードと境界条件

  • ビットフィールドのサイズ制限により、各フラグは2値の状態しか保持できません。

保守担当者向けメモ

  • 本構造体は mod_http2 の機能フラグを管理する目的で使用されています。
  • 他のモジュール（mod_setenvif.c 等）にも同名の features という名前の変数や構造体が存在しますが、それらとは無関係です。
  • 構造体の各フィールドが具体的にどのタイミングで評価・更新されるかについては、h2_post_config 等の初期化ロジックのさらなる解析が必要です。
