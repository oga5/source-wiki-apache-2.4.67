dav_lock_walker

定義

  • 種別: function
  • 定義箇所: modules/dav/main/util_lock.c:260

目的

WebDAV の再帰的なロック操作において、対象リソースの子孫リソースに対して間接的なロック（indirect lock）を記録するためのコールバック関数です。

シグネチャまたは形状

static dav_error * dav_lock_walker(dav_walk_resource *wres, int calltype)

入力と出力

  • **入力**:
  • wres: 現在処理中のリソース情報を含む dav_walk_resource 構造体。
  • calltype: 呼び出しの種類（本関数内では未使用）。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時は dav_error 構造体へのポインタを返します。

前提条件と不変条件

  • wres->walk_ctx には、ロックデータベースのフックや適用対象のロック情報を含む dav_walker_ctx が格納されている必要があります。
  • wres->resource は有効なリソースオブジェクトである必要があります。

アルゴリズムまたは振る舞い

1. **ルート判定**: wres->resource がロック操作の起点（ctx->w.root）と同一である場合、間接ロックを設定せずに即座に NULL を返します。 2. **ロックの追加**: 上記以外の場合、ctx->w.lockdb->hooks->append_locks を呼び出し、現在のリソースに対して ctx->lock を間接ロックとして追加します。 3. **エラー処理**:

  • サーバーエラー（500 Internal Server Error 等）が発生した場合は、そのエラーを即座に返します。
  • それ以外のエラーが発生した場合は、dav_add_response を呼び出してマルチステータスレスポンスにエラー情報を追加し、処理を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/dav/main/util_lock.c 内の dav_add_lock 関数から、dav_walker_ctx の func メンバとして参照されます。
  • **呼び出し先**:
  • wres->resource->hooks->is_same_resource
  • ctx->w.lockdb->hooks->append_locks
  • dav_add_response

関連データ構造

  • dav_walk_resource: ウォーカー処理中のリソース状態を保持。
  • dav_walker_ctx: ロック操作のコンテキスト（ロックデータベースフックや適用するロック情報）。
  • dav_error: 操作失敗時のエラー詳細。

失敗モードと境界条件

  • **リソース不一致**: ルートリソースに対してはロックを適用しないガード節が存在します。
  • **ロック追加失敗**: append_locks が失敗した場合、サーバーエラーであれば処理が中断されます。クライアントエラー等の場合はマルチステータスレスポンスに記録されますが、全体のロック処理が不完全な状態になる可能性がある旨がソース内のコメントで指摘されています。

保守担当者向けメモ

  • 本関数は walk フック経由で呼び出されるため、再帰的なロック適用における各ノードの処理単位となります。
  • エラー発生時の挙動（マルチステータスへの追加 vs 処理の中断）については、将来的な改善の余地がある旨のコメントがソース内に残されています。
