dump_content

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_proxy_html.c:223

目的

HTML コンテンツ内のテキストノードに対し、設定された置換ルール（urlmap）を適用し、URL や文字列の書き換えを行う。

シグネチャまたは形状

static void dump_content(saxctxt *ctx)

入力と出力

  • **入力**: saxctxt *ctx（フィルタ実行時のコンテキスト）。ctx->buf に保持されたテキストデータと、ctx->map に格納された置換ルールセットを使用する。
  • **出力**: なし（戻り値は void）。処理結果は AP_fwrite を通じて出力バケットへ書き出される。

前提条件と不変条件

  • ctx->buf はヌル終端されている必要がある（関数冒頭で pappend により保証される）。
  • ctx->map には適用可能な置換ルールが格納されていること。

アルゴリズムまたは振る舞い

1. ctx->buf の末尾にヌルバイトを追加し、文字列として扱えるようにする。 2. ctx->map に含まれる各ルールを順次走査する。

  • M_CDATA フラグが立っていないルールはスキップする。
  • **正規表現置換 (M_REGEX)**: ap_regexec を用いてマッチ箇所を検索し、ap_pregsub で置換文字列を生成する。置換後のサイズが元のサイズと異なる場合は preserve でバッファを拡張し、memmove でメモリをシフトして挿入する。
  • **文字列置換**: strstr を用いてマッチ箇所を検索する。M_ATSTART や M_ATEND フラグに従い、マッチ位置の制約を適用する。同様にバッファのシフトとコピーを行う。

    • 3. 最後に、置換済みの ctx->buf を AP_fwrite で出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/filters/mod_proxy_html.c:335
  • modules/filters/mod_proxy_html.c:360
  • **呼び出し先**:
  • pappend (バッファへのヌルバイト追加)
  • ap_regexec, ap_pregsub (正規表現処理)
  • preserve (バッファ拡張)
  • strstr, strlen, memmove, memcpy (標準文字列・メモリ操作)
  • AP_fwrite (出力処理)
  • ap_log_rerror (ログ出力)

関連データ構造

  • saxctxt: フィルタの実行状態とバッファを保持する構造体。
  • urlmap: 置換ルール（正規表現または固定文字列、フラグ）を保持する構造体。

失敗モードと境界条件

  • s_to > s_from の場合、バッファの再確保（preserve）が発生する。
  • M_ATSTART や M_ATEND フラグにより、マッチしても置換されないケースがある。
  • ctx->buf が空の場合、ループは実行されずそのまま出力される。

保守担当者向けメモ

  • 本関数は libxml2 の SAX コールバックではなく、バッファリングされたテキストの最終処理として呼び出される。
  • memmove を多用するため、大規模なバッファに対する置換処理はパフォーマンスに影響を与える可能性がある。
  • GO_FASTER マクロが定義されていない場合、APLOGrtrace1 を用いた詳細なトレースログが出力される。
