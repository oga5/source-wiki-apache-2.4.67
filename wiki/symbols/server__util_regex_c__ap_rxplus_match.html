ap_rxplus_match

定義

  • 種別: function
  • 定義箇所: server/util_regex.c:193

目的

コンパイル済み正規表現オブジェクト ap_rxplus_t のマッチング結果から、指定したキャプチャグループに対応する部分文字列とその長さを取得します。

シグネチャまたは形状

void ap_rxplus_match(ap_rxplus_t *rx, int n, int *len, const char **match)

入力と出力

  • rx: マッチング結果を保持する ap_rxplus_t 構造体へのポインタ。
  • n: 取得対象のキャプチャグループ番号（0は全体）。
  • len: マッチした部分文字列の長さを格納する出力用ポインタ。
  • match: マッチした部分文字列の開始位置を指すポインタを格納する出力用ポインタ。

前提条件と不変条件

  • rx は ap_rxplus_exec によってマッチング処理が実行済みである必要があります。
  • n が有効なキャプチャグループ範囲内（0 以上かつ ap_rxplus_nmatch(rx) 未満）である必要があります。

アルゴリズムまたは振る舞い

1. n が有効な範囲内かを確認します。 2. 有効な場合、rx->match（元の文字列）の開始位置に pmatch[n].rm_so（開始オフセット）を加算し、match に設定します。 3. pmatch[n].rm_eo（終了オフセット）から rm_so を減算し、長さを len に設定します。 4. 範囲外の場合は、len に -1 を、match に NULL を設定します。

呼び出し元と呼び出し先

  • 呼び出し元: server/util_regex.c:209（内部参照）
  • 呼び出し先: ap_rxplus_nmatch（インライン関数）

関連データ構造

  • ap_rxplus_t: 正規表現のコンパイル結果とマッチング状態を保持する構造体。

失敗モードと境界条件

  • n が範囲外の場合、len に -1 が代入され、match は NULL になります。
  • ap_rxplus_exec が呼び出されていない、またはマッチしなかった場合の結果は、構造体内の pmatch の状態に依存します。

保守担当者向けメモ

  • 本関数はマッチした文字列のコピーを作成しません。match が指すポインタは rx->match（元の文字列）の一部であるため、ap_rxplus_exec を再実行すると内容が更新される点に注意してください。永続的な保持が必要な場合は、呼び出し元でコピーを作成する必要があります。
