ap_get_module_flags

定義

  • 種別: function
  • 定義箇所: server/util_debug.c:110

目的

指定されたモジュール構造体のバージョンが、現在の Apache HTTP Server が要求する最小バージョンを満たしているかを確認し、適合する場合にそのモジュールのフラグを返します。

シグネチャまたは形状

int ap_get_module_flags(const module *m)

入力と出力

  • **入力**: const module *m - 検査対象のモジュール構造体へのポインタ。
  • **出力**: int - バージョンが適合する場合はモジュールの flags メンバを返し、適合しない場合は 0 を返します。

前提条件と不変条件

  • 入力ポインタ m は有効な module 構造体を指している必要があります。
  • バージョンチェックには AP_MODULE_FLAGS_MMN_MAJOR および AP_MODULE_FLAGS_MMN_MINOR 定数が使用されます。

アルゴリズムまたは振る舞い

1. モジュールのメジャーバージョン (m->version) が AP_MODULE_FLAGS_MMN_MAJOR より小さい場合、またはメジャーバージョンが等しくマイナーバージョン (m->minor_version) が AP_MODULE_FLAGS_MMN_MINOR より小さい場合、バージョン不適合と判断し 0 を返します。 2. 上記の条件に該当しない（バージョンが適合する）場合、モジュール構造体に保持されている flags を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/config.c:341: モジュールのロードまたは設定プロセスにおいて、フラグの評価に使用されています。
  • **呼び出し先**:
  • 本関数は標準ライブラリや他の外部関数を直接呼び出しません。

関連データ構造

  • module: http_config.h で定義される、モジュールのメタデータ（バージョン、フラグ等）を保持する構造体。

失敗モードと境界条件

  • バージョンチェックに失敗した場合は、エラーを通知するのではなく 0 を返すことで、フラグが設定されていない状態として扱われます。

保守担当者向けメモ

  • 本関数は、モジュールのバイナリ互換性（MMN: Module Magic Number）を動的に検証するためのガードとして機能します。
  • include/http_config.h でプロトタイプ宣言されています。
