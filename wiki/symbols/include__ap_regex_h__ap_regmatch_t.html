ap_regmatch_t

定義

  • 種別: struct
  • 定義箇所: include/ap_regex.h:115

目的

正規表現のマッチング結果において、対象文字列内のマッチした部分の開始位置と終了位置を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    int rm_so;
    int rm_eo;
} ap_regmatch_t;

入力と出力

  • rm_so: マッチした部分文字列の開始オフセット（Start Offset）。
  • rm_eo: マッチした部分文字列の終了オフセット（End Offset）。

前提条件と不変条件

  • rm_so および rm_eo は、ap_regexec 等の関数によって書き込まれます。
  • マッチが成功した場合、rm_so はマッチ開始位置を、rm_eo はマッチ終了位置（マッチした文字列の直後のインデックス）を指します。

アルゴリズムまたは振る舞い

ap_regexec などの実行関数に配列として渡され、正規表現エンジンによってマッチング結果が各フィールドに格納されます。POSIX 互換の regmatch_t と同等の役割を果たします。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • include/ap_expr.h
  • include/ap_regex.h
  • include/httpd.h
  • include/util_varbuf.h
  • modules/aaa/mod_authnz_ldap.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数を呼び出す側ではありません。

関連データ構造

  • ap_regex_t: コンパイル済み正規表現を保持する構造体。マッチング実行時に本構造体と組み合わせて使用されます。

失敗モードと境界条件

  • マッチしなかった場合、またはサブパターンがマッチしなかった場合の rm_so および rm_eo の値は、使用する正規表現エンジンの実装（PCRE）に依存します。

保守担当者向けメモ

  • include/ap_mmn.h に示唆されている通り、POSIX の regmatch_t との互換性を意識して設計されています。
  • AP_REG_* フラグの設定により、マッチングの挙動が変化するため、利用時は ap_regexec のドキュメントを併せて参照してください。
