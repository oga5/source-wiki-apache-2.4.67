ssl_init_ctx_callbacks

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_init.c:1008

目的

SSL コンテキスト（SSL_CTX）に対して、DH パラメータ、情報コールバック、および ALPN（Application-Layer Protocol Negotiation）選択コールバックを設定する。

シグネチャまたは形状

static void ssl_init_ctx_callbacks(server_rec *s,
                                   apr_pool_t *p,
                                   apr_pool_t *ptemp,
                                   modssl_ctx_t *mctx)

入力と出力

  • **入力**:
  • s: サーバー設定レコード。
  • p: 長期生存するメモリプール。
  • ptemp: 一時的なメモリプール。
  • mctx: 設定情報を含む modssl_ctx_t 構造体。
  • **出力**: なし（mctx->ssl_ctx を直接変更する）。

前提条件と不変条件

  • mctx->ssl_ctx が有効な SSL_CTX オブジェクトを指していること。

アルゴリズムまたは振る舞い

1. **DH パラメータ設定**: OpenSSL 1.1 未満の場合、ssl_callback_TmpDH を一時 DH パラメータ用コールバックとして登録する。 2. **情報コールバック設定**: ログレベルがデバッグ設定であるか、または再ネゴシエーションがブロックされている場合、ssl_callback_Info を情報コールバックとして登録する。 3. **ALPN 設定**: HAVE_TLS_ALPN が定義されている場合、ssl_callback_alpn_select を ALPN 選択コールバックとして登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_init.c:1388
  • **呼び出し先**:
  • ssl_callback_TmpDH (OpenSSL 1.1 未満)
  • ssl_callback_Info
  • ssl_callback_alpn_select (ALPN 有効時)

関連データ構造

  • modssl_ctx_t: SSL コンテキストの管理構造体。
  • SSL_CTX: OpenSSL の SSL コンテキスト構造体。

失敗モードと境界条件

  • 本関数自体はエラーを返さない。
  • OpenSSL のバージョンやビルド時の機能フラグ（HAVE_TLS_ALPN 等）により、設定されるコールバックが動的に変化する。

保守担当者向けメモ

  • OpenSSL 1.1 以降では DH パラメータの自動選択が推奨されるため、MODSSL_USE_OPENSSL_PRE_1_1_API による条件分岐が設けられている。
  • 再ネゴシエーションの制御とデバッグログの出力が同一のコールバック（ssl_callback_Info）に依存している点に注意すること。
