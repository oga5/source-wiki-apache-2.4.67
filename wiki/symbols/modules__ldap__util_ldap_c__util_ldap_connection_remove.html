util_ldap_connection_remove

定義

  • 種別: function
  • 定義箇所: modules/ldap/util_ldap.c:259

目的

LDAP 接続に関連付けられたリソース（メモリ、バインド情報、接続プール）を解放し、接続リストから当該接続を削除する。

シグネチャまたは形状

apr_status_t util_ldap_connection_remove(void *param)

入力と出力

  • **入力**: void *param（util_ldap_connection_t 構造体へのポインタ）
  • **出力**: apr_status_t（常に APR_SUCCESS を返す）

前提条件と不変条件

  • 入力 param が NULL の場合、何もせず APR_SUCCESS を返す。
  • 接続リストの操作およびミューテックスのロックには st->mutex が使用される。

アルゴリズムまたは振る舞い

1. uldap_connection_unbind を呼び出し、LDAP サーバーとのバインドを解除する。 2. スレッド環境下では st->mutex をロックし、st->connections リストから対象の接続を検索・削除する。 3. 接続構造体内の bindpw および binddn が存在する場合は free を実行する。 4. スレッド環境下では ldc->lock および st->mutex をアンロックする。 5. 最後に apr_pool_destroy を呼び出し、接続に関連付けられたメモリプールを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内のコメント（253行目）に、LDAP 関連のストレージを解放する目的で使用される旨の記述がある。具体的な関数呼び出しの証拠は提供されていない。
  • **呼び出し先**: uldap_connection_unbind、apr_thread_mutex_lock、apr_thread_mutex_unlock、free、apr_pool_destroy。

関連データ構造

  • util_ldap_connection_t: 接続状態を保持する構造体。
  • util_ldap_state_t: モジュールの状態および接続リストを保持する構造体。

失敗モードと境界条件

  • param が NULL の場合は早期リターンする。
  • 接続リスト内に該当する接続が見つからない場合、リストの操作はスキップされる。

保守担当者向けメモ

  • 本関数は接続のライフサイクルの終端を担う。メモリリークを防ぐため、接続に関連する動的確保されたメンバ（bindpw, binddn）の解放順序と、プール破棄のタイミングに注意すること。
