ap_session_save

定義

  • 種別: function
  • 定義箇所: modules/session/mod_session.c:180

目的

セッションデータ（session_rec）の変更状態を確認し、必要に応じてシリアライズおよびストレージへの永続化を行う。

シグネチャまたは形状

apr_status_t ap_session_save(request_rec *r, session_rec *z)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • z: 保存対象のセッション構造体。
  • **出力**:
  • APR_SUCCESS: 保存成功、または保存不要と判断された場合。
  • APR_EGENERAL: 保存失敗（二重保存の試行、期限切れ、バックエンド未設定、またはエンコード/保存エラー）。

前提条件と不変条件

  • z が NULL の場合は何もしない。
  • z->written が真の場合、二重保存とみなされエラーを返す。
  • セッションが期限切れ（z->expiry < now）の場合、保存は行われない。
  • dirty フラグが立っていない場合、かつ有効期限の更新が微小であるか、maxage が設定されていない場合は保存をスキップする。

アルゴリズムまたは振る舞い

1. **検証**: 二重保存の試行や期限切れセッションの保存試行をチェックし、該当する場合はエラーを返す。 2. **有効期限の更新**: 設定（dconf->maxage）に基づき、セッションの有効期限を再計算する。 3. **保存判定**: dirty フラグや更新間隔の閾値を確認し、変更がない場合は処理を終了する。 4. **エンコード**: ap_run_session_encode を呼び出し、セッションデータをシリアライズする。 5. **永続化**: ap_run_session_save を呼び出し、バックエンド（Cookie/DBD等）へ書き出す。 6. **完了**: 成功時に z->written を 1 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/session/mod_session.c:486（session_output_filter 内）
  • **呼び出し先**:
  • ap_run_session_encode（フック）
  • ap_run_session_save（フック）
  • ap_log_rerror（ログ出力）

関連データ構造

  • session_rec: セッション内容、dirty フラグ、有効期限を保持。
  • session_dir_conf: ディレクトリ単位のセッション設定（maxage 等）。

失敗モードと境界条件

  • **二重保存**: z->written が設定されている場合に APR_EGENERAL を返す。
  • **期限切れ**: 既に期限が切れているセッションを保存しようとすると APR_EGENERAL を返す。
  • **バックエンド未設定**: ap_run_session_save が DECLINED を返した場合、保存先がないものとして APR_EGENERAL を返す。
  • **処理エラー**: エンコードまたは保存フックが OK 以外を返した場合、そのエラーコードを返す。

保守担当者向けメモ

  • 本関数は session_output_filter から呼び出されることを想定している。
  • dirty フラグによる最適化が行われているため、変更がない場合はフックが実行されない。
  • 内部リダイレクトが発生した場合、フィルタ経由で複数回呼び出される可能性があるため、written フラグによるガードが重要である。
