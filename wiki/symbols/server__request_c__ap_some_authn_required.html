ap_some_authn_required

定義

  • 種別: function
  • 定義箇所: server/request.c:127

目的

リクエストに対して認証（Authentication）が必要かどうかを判定します。

シグネチャまたは形状

int ap_some_authn_required(request_rec *r)

入力と出力

  • **入力**: request_rec *r （現在のリクエスト構造体）
  • **出力**: int （認証が必要な場合は TRUE、不要な場合は FALSE を返す）

前提条件と不変条件

  • r->user の値は、関数実行前後で変更されないよう保護されます（ローカル変数に退避し、終了時に復元）。
  • ap_satisfies(r) の結果に基づいて処理が分岐します。

アルゴリズムまたは振る舞い

1. ap_satisfies(r) を呼び出し、設定された認証要件（SATISFY_ALL / SATISFY_ANY / SATISFY_NOSPEC）を確認します。 2. **SATISFY_ALL または SATISFY_NOSPEC の場合**:

  • ap_run_access_checker(r) を実行し、結果が OK でなければ終了します。
  • ap_run_access_checker_ex(r) を実行し、結果が DECLINED であれば認証が必要と判断し TRUE を返します。

    • 3. **SATISFY_ANY の場合**:

  • ap_run_access_checker(r) が OK であれば認証不要と判断し終了します。
  • ap_run_access_checker_ex(r) を実行し、結果が DECLINED であれば認証が必要と判断し TRUE を返します。

    • 4. 最後に r->user を復元して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述はありません。
  • **呼び出し先**:
  • ap_satisfies(r)
  • ap_run_access_checker(r)
  • ap_run_access_checker_ex(r)

関連データ構造

  • request_rec: リクエストのコンテキストを保持する主要な構造体。

失敗モードと境界条件

  • ap_run_access_checker や ap_run_access_checker_ex が OK 以外の値を返した場合、認証の必要性は FALSE（初期値）のまま維持されます。
  • 認証プロバイダが設定されていない場合、DECLINED が返されることで認証が必要と判定される挙動となります。

保守担当者向けメモ

  • 本関数は include/http_request.h で宣言されています。
  • 2.4.13-dev で導入された関数であり、古い認証チェックロジックの代替として利用されます。
