passwd_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_crypt.c:183

目的

パスフレーズとその長さを保持し、暗号化操作やパスワード認証処理においてコンテキストとして受け渡すためのデータ構造です。

シグネチャまたは形状

typedef struct {
    const char *pass_phrase;
    int pass_len;
} passwd_ctx;

入力と出力

  • pass_phrase: パスフレーズ文字列へのポインタ。
  • pass_len: パスフレーズの長さ（バイト数）。

前提条件と不変条件

  • pass_phrase は有効な文字列を指している必要があります。
  • pass_len は pass_phrase の実際の長さを正確に反映している必要があります。

アルゴリズムまたは振る舞い

本構造体は、OpenSSL のコールバック関数やパスワード入力を必要とする暗号化ルーチンにおいて、必要な認証情報を一括して保持するコンテナとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/md/md_crypt.c 内の各関数（passwd_ctx を初期化して使用）、modules/lua/lua_request.c、support/htdbm.c、support/htpasswd.c など。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • modules/lua/lua_passwd.h にも同名の構造体定義が存在します。

失敗モードと境界条件

  • pass_phrase が NULL の場合や、pass_len が負の値である場合に、これを利用する関数側で未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • modules/md/md_crypt.c と modules/lua/lua_passwd.h の両方で定義されているため、変更時には両方の定義に整合性があるか確認してください。
