HSE_TF_INFO

定義

  • 種別: struct
  • 定義箇所: modules/arch/win32/mod_isapi.h:150

目的

IIS 5.0 相当の TransmitFile 機能をエミュレートするための構造体です。Apache 上で ISAPI 拡張が効率的にファイル転送を行うためのパラメータを保持します。

シグネチャまたは形状

typedef struct HSE_TF_INFO {
    PFN_HSE_IO_COMPLETION pfnHseIO;
    void                 *pContext;
    apr_os_file_t         hFile;
    const char           *pszStatusCode;
    apr_uint32_t          BytesToWrite;
    apr_uint32_t          Offset;
    void                 *pHead;
    apr_uint32_t          HeadLength;
    void                 *pTail;
    apr_uint32_t          TailLength;
    apr_uint32_t          dwFlags;
} HSE_TF_INFO;

入力と出力

  • **入力**: ISAPI 拡張が ServerSupportFunction を通じて本構造体を渡します。
  • **出力**: hFile で指定されたファイルの内容、および pHead/pTail で指定されたバッファの内容がクライアントへ送信されます。

前提条件と不変条件

  • pszStatusCode は HSE_IO_SEND_HEADERS フラグが設定されていない限り無視されます。
  • BytesToWrite に 0 を指定した場合、ファイル全体が送信されます。
  • pHead および pTail は、ファイル転送の前後に付与されるデータとして扱われます。

アルゴリズムまたは振る舞い

1. pfnHseIO が指定されている場合、デフォルトの HSE_REQ_IO_COMPLETION 設定を上書きします。 2. dwFlags に基づき、ヘッダーの送信やファイル転送の挙動を制御します。 3. pszStatusCode には、HTTP ステータスコードとヘッダー行（例: "200 OK\r\nContent-type: text/plain\r\n"）を含める必要があります。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/arch/win32/mod_isapi.c 内の ServerSupportFunction 実装において、HSE_REQ_TRANSMIT_FILE リクエストを処理する際に本構造体へキャストされます。
  • **呼び出し先**: 構造体内の pfnHseIO が定義されている場合、I/O 完了時にコールバックとして呼び出されます。

関連データ構造

  • EXTENSION_CONTROL_BLOCK: ISAPI 拡張のメインコンテキスト。
  • PFN_HSE_IO_COMPLETION: I/O 完了通知用の関数ポインタ型。

失敗モードと境界条件

  • pszStatusCode に NULL 文字が含まれる場合、Apache の内部処理と競合する可能性があるため注意が必要です。
  • HSE_IO_SEND_HEADERS フラグが未設定の場合、pszStatusCode の内容は無視されます。

保守担当者向けメモ

  • 本構造体は IIS 5.0 互換機能の一部です。Apache のアーキテクチャと整合しない機能が含まれる可能性があるため、実装変更時は mod_isapi.c の ServerSupportFunction 処理フローを確認してください。
