register_hooks

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_userdir.c:371

目的

Apache HTTP Server のフック機構に対し、mod_userdir が提供する翻訳処理および suEXEC 連携機能を登録する。

シグネチャまたは形状

static void register_hooks(apr_pool_t *p)

入力と出力

  • 入力: apr_pool_t *p（フック登録に使用されるメモリプール）
  • 出力: なし（void）

前提条件と不変条件

  • mod_alias.c が翻訳フェーズの先行モジュールとして定義されている。
  • mod_vhost_alias.c が翻訳フェーズの後続モジュールとして定義されている。

アルゴリズムまたは振る舞い

1. translate_userdir 関数を ap_hook_translate_name に登録する。この際、mod_alias.c の後、mod_vhost_alias.c の前に実行されるよう順序が制御される。 2. HAVE_UNIX_SUEXEC が定義されている環境では、get_suexec_id_doer 関数を ap_hook_get_suexec_identity に登録する（実行順序は APR_HOOK_FIRST）。

呼び出し元と呼び出し先

  • 呼び出し元: Apache HTTP Server のモジュールロードプロセス（直接の呼び出し元はソースコード上では明示されていません）。
  • 呼び出し先:
  • ap_hook_translate_name
  • ap_hook_get_suexec_identity (条件付き)
  • translate_userdir (関数ポインタとして渡される)
  • get_suexec_id_doer (関数ポインタとして渡される、条件付き)

関連データ構造

  • apr_pool_t: フック登録のライフサイクル管理に使用。

失敗モードと境界条件

  • HAVE_UNIX_SUEXEC が未定義の場合、suEXEC 関連のフック登録はスキップされる。

保守担当者向けメモ

  • 本関数は Apache の標準的なモジュールフック登録パターンに従っている。
  • 実行順序の変更が必要な場合は、aszPre および aszSucc 配列の内容を調整すること。
