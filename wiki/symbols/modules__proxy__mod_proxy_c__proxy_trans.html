proxy_trans

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy.c:979

目的

リクエストの URL を解析し、設定された ProxyPass ルールに基づいてリクエストがプロキシ対象であるかを判定し、必要に応じてリクエストをプロキシ処理へルーティングする。

シグネチャまたは形状

static int proxy_trans(request_rec *r, int pre_trans)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体 (request_rec)。
  • pre_trans: フックの実行フェーズを示すフラグ（1: pre_translate_name フック、0: translate_name フック）。
  • **出力**:
  • OK: プロキシ対象として処理が完了。
  • DECLINED: プロキシ対象外。
  • DONE: 処理済み（pre_trans が真の場合）。
  • その他: ap_proxy_trans_match が返す HTTP ステータスコード。

前提条件と不変条件

  • r->proxyreq が既に設定されている場合、本関数は即座に終了する。
  • r->uri が / で始まらない場合や、no-proxy 環境変数が設定されている場合はプロキシ対象外となる。
  • pre_trans フラグと PROXYPASS_MAP_ENCODED フラグの整合性が一致する場合のみ、マッチング処理が実行される。

アルゴリズムまたは振る舞い

1. **初期チェック**: r->proxyreq の確認および URI の妥当性チェックを行う。 2. **環境変数確認**: no-proxy 環境変数が設定されていれば DECLINED を返す。 3. **ディレクトリ設定の確認**: dconf->alias が存在する場合、ap_proxy_trans_match を呼び出し、マッチすればその結果を返す。 4. **サーバー設定の走査**: conf->aliases リストを順次走査し、各エイリアスに対して ap_proxy_trans_match を試行する。 5. **終了**: マッチするルールが見つからない場合は DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • proxy_pre_translate_name (modules/proxy/mod_proxy.c:1051)
  • proxy_translate_name (modules/proxy/mod_proxy.c:1056)
  • **呼び出し先**:
  • ap_proxy_trans_match (マッチング実行)
  • ap_get_module_config (設定取得)
  • apr_table_get (環境変数取得)

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • proxy_dir_conf: ディレクトリ単位のプロキシ設定。
  • proxy_server_conf: サーバー単位のプロキシ設定。
  • proxy_alias: プロキシパスの定義構造体。

失敗モードと境界条件

  • r->uri が NULL または / で始まらない場合、即座に DECLINED となる。
  • pre_trans とエンコードフラグの不一致により、意図せずマッチングがスキップされる場合がある。

保守担当者向けメモ

  • 本関数は pre_translate_name と translate_name の両フックから呼び出されるため、pre_trans フラグによる条件分岐が重要である。
  • ap_proxy_trans_match の戻り値が HTTP_CONTINUE 以外の場合、その値がそのまま関数の戻り値となる。
