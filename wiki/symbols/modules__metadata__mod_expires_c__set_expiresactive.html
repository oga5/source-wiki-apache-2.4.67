set_expiresactive

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:196

目的

Apache の ExpiresActive ディレクティブを処理し、ディレクトリごとの設定構造体における有効期限制御の有効・無効状態を更新します。

シグネチャまたは形状

static const char *set_expiresactive(cmd_parms *cmd, void *in_dir_config, int arg)

入力と出力

  • cmd: コマンドパラメータ構造体。
  • in_dir_config: 設定対象となる expires_dir_config 構造体へのポインタ。
  • arg: ディレクティブの値（On の場合は 1、Off の場合は 0）。
  • 戻り値: 常に NULL を返します。

前提条件と不変条件

  • in_dir_config は expires_dir_config 型としてキャスト可能である必要があります。
  • 本関数が呼び出された時点で、対象ディレクトリに対して ExpiresActive が明示的に設定されたとみなされます。

アルゴリズムまたは振る舞い

1. 引数 in_dir_config を expires_dir_config 構造体にキャストします。 2. arg が 0（Off）以外の場合、dir_config->active を ACTIVE_ON に設定します。 3. arg が 0 の場合、dir_config->active を ACTIVE_OFF に設定します。 4. 処理完了後、NULL を返します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/metadata/mod_expires.c:351 の AP_INIT_FLAG マクロ定義により、Apache 設定解析プロセスから呼び出されます。
  • 呼び出し先: 本関数内から明示的に呼び出している関数はありません。

関連データ構造

  • expires_dir_config: mod_expires のディレクトリ設定を保持する構造体。active メンバを更新対象とします。

失敗モードと境界条件

  • 本関数は常に成功し、エラーメッセージを返しません。
  • arg に 0 以外の値が渡された場合はすべて ACTIVE_ON として扱われます。

保守担当者向けメモ

  • ExpiresActive の ACTIVE_DONTCARE 状態は、本関数経由ではなく、設定の初期化やマージ処理においてのみ使用される内部状態です。本関数は明示的な設定変更のみを扱います。
