md_store_iter

定義

  • 種別: function
  • 定義箇所: modules/md/md_store.c:99

目的

ストレージ内の特定グループおよびパターンに一致するエントリを走査し、各エントリに対してユーザー定義の検査関数（コールバック）を実行します。

シグネチャまたは形状

apr_status_t md_store_iter(md_store_inspect *inspect, void *baton, md_store_t *store, 
                           apr_pool_t *p, md_store_group_t group, const char *pattern, 
                           const char *aspect, md_store_vtype_t vtype);

入力と出力

  • **入力**:
  • inspect: 各エントリに対して呼び出されるコールバック関数。
  • baton: コールバックに渡されるユーザー定義のコンテキストデータ。
  • store: 操作対象の md_store_t インスタンス。
  • p: メモリ割り当てに使用するプール。
  • group: 走査対象のストレージグループ（MD_SG_DOMAINS 等）。
  • pattern: ファイル名やエントリ名のフィルタパターン。
  • aspect: 走査対象の側面（MD_FN_MD 等）。
  • vtype: 期待されるデータ型（MD_SV_JSON 等）。
  • **出力**:
  • apr_status_t: 操作の成功または失敗を示すステータスコード。

前提条件と不変条件

  • store は有効な md_store_t 構造体であり、その iterate 関数ポインタが適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った store 構造体の iterate 関数ポインタを呼び出します。 2. 実際の走査ロジックは store の具体的な実装（md_store_fs.c 等）に完全に委譲されます。 3. 走査中に見つかった各エントリに対し、inspect コールバックが実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_acct.c (複数箇所)
  • modules/md/md_store.c:380 (md_store_md_iter 内)
  • **呼び出し先**:
  • store->iterate (引数として渡された store インスタンス内の関数ポインタ)

関連データ構造

  • md_store_t: ストレージの抽象インターフェース。
  • md_store_group_t: ストレージグループの列挙型。
  • md_store_inspect: エントリ検査用の関数ポインタ型。

失敗モードと境界条件

  • store が NULL である場合、または store->iterate が初期化されていない場合は未定義の動作（クラッシュ）を引き起こす可能性があります。
  • 戻り値は store->iterate の実装に依存し、ファイルシステムエラーや権限不足などが含まれます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のI/Oや走査ロジックは含まれていません。
  • 動作の詳細は、実行時に store に割り当てられている具体的な実装（md_store_fs など）を確認してください。
