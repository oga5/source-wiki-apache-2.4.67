cgid_dirconf

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_cgid.c:94

目的

mod_cgid モジュールにおいて、ディレクトリ単位で適用される設定情報を保持するための構造体です。

シグネチャまたは形状

typedef struct { 
    apr_interval_time_t timeout;
} cgid_dirconf;

入力と出力

  • **入力**: timeout フィールドは apr_interval_time_t 型であり、CGI スクリプトの実行タイムアウト値をマイクロ秒単位で保持します。
  • **出力**: なし（本構造体は設定保持専用のデータコンテナです）。

前提条件と不変条件

  • apr_pcalloc 等を用いてメモリが割り当てられることを前提としています。
  • timeout は mod_cgid の処理において、CGI デーモンとの通信やスクリプト実行の待機時間として参照されます。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行います。実行時には ap_get_module_config(r->per_dir_config, &cgid_module) を通じてリクエストごとのディレクトリ設定として取得され、CGI 実行時のタイムアウト制御に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • modules/generators/mod_cgid.c:1105: 設定の初期化時に apr_pcalloc で割り当て。
  • modules/generators/mod_cgid.c:1165: 設定の結合処理等で参照。
  • modules/generators/mod_cgid.c:1483: 内部処理での変数宣言。
  • modules/generators/mod_cgid.c:1736: cgid_handler 内でリクエストに関連付けられた設定を取得。
  • **呼び出し先**: 構造体であるため、関数呼び出しは発生しません。

関連データ構造

  • cgid_module: 本構造体を利用するモジュール定義。
  • cgid_server_conf: サーバー全体の設定を保持する構造体。

失敗モードと境界条件

  • 設定が未定義の場合、ap_get_module_config が NULL を返す可能性があるため、呼び出し側でのポインタチェックが必要です。
  • timeout 値が適切に設定されていない場合、CGI デーモンとの通信が意図しないタイミングでタイムアウトする可能性があります。

保守担当者向けメモ

  • 本構造体はディレクトリ単位の設定を管理するため、mod_cgid の設定ディレクティブ（ScriptLog 等）が追加される際は、この構造体にフィールドが追加される可能性があります。
  • 現在のフィールドは timeout のみですが、将来的にディレクトリごとの制限事項が増えた場合はここが拡張ポイントとなります。
