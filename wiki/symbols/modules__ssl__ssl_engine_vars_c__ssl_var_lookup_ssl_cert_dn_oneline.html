ssl_var_lookup_ssl_cert_dn_oneline

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_vars.c:446

目的

X.509 証明書の識別名（DN）を、設定に応じてレガシー形式または RFC 2253 準拠の文字列に変換し、Apache のメモリプール上に確保して返します。

シグネチャまたは形状

char *ssl_var_lookup_ssl_cert_dn_oneline(apr_pool_t *p, request_rec *r, X509_NAME *xsname)

入力と出力

  • **入力**:
  • p: メモリ確保に使用する apr_pool_t。
  • r: リクエスト情報（SSL_OPT_LEGACYDNFORMAT オプションの確認に使用）。
  • xsname: 変換対象の X509_NAME 構造体。
  • **出力**:
  • 変換された DN 文字列（apr_pool_t 上に確保）。失敗した場合は NULL。

前提条件と不変条件

  • r が NULL の場合、レガシー形式は無効（legacy_format = 0）として扱われます。
  • 戻り値は apr_pool_t に依存するため、呼び出し側でプールが破棄されるまで有効です。
  • X509_NAME_oneline を使用するレガシーパスでは、内部で一時的に確保されたメモリを OPENSSL_free で解放します。

アルゴリズムまたは振る舞い

1. r からディレクトリ設定を取得し、SSL_OPT_LEGACYDNFORMAT が有効か判定します。 2. **レガシー形式の場合**: X509_NAME_oneline を呼び出し、結果を apr_pstrdup でプールにコピーして返します。 3. **標準形式（RFC 2253）の場合**:

  • BIO_s_mem() を使用してメモリ BIO を作成します。
  • X509_NAME_print_ex を呼び出し、XN_FLAG_RFC2253 フラグ（MSB エスケープを除く）で BIO に書き出します。
  • modssl_bio_free_read を介して BIO から文字列を読み取り、プールに確保して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_vars.c:509（ssl_var_lookup_ssl_cert_dn 関数内）
  • **呼び出し先**:
  • X509_NAME_oneline (OpenSSL)
  • apr_pstrdup (APR)
  • OPENSSL_free (OpenSSL)
  • BIO_new / BIO_s_mem (OpenSSL)
  • X509_NAME_print_ex (OpenSSL)
  • modssl_bio_free_read (mod_ssl ユーティリティ)

関連データ構造

  • X509_NAME: OpenSSL の証明書識別名構造体。
  • SSLDirConfigRec: ディレクトリごとの SSL 設定を保持する構造体。

失敗モードと境界条件

  • BIO_new が失敗した場合、NULL を返します。
  • xsname が NULL の場合、OpenSSL の関数仕様に依存した挙動となります。

保守担当者向けメモ

  • 本関数は mod_ssl の変数解決エンジンの一部であり、ログ出力や式評価で利用されます。
  • RFC 2253 形式への変換時に ASN1_STRFLGS_ESC_MSB を除外しているため、非 ASCII 文字の扱いに注意してください。
  • modssl_bio_free_read は mod_ssl 内部のユーティリティであり、BIO の解放と文字列のコピーを同時に行います。
