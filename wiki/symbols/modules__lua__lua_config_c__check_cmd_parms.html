check_cmd_parms

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_config.c:31

目的

Lua スタック上のユーザーデータから Apache の cmd_parms 構造体へのポインタを取得し、型検証を行う。

シグネチャまたは形状

static cmd_parms *check_cmd_parms(lua_State *L, int index)

入力と出力

  • **入力**:
  • L: Lua 状態へのポインタ。
  • index: cmd_parms 構造体が格納されている Lua スタック上のインデックス。
  • **出力**:
  • 成功時: cmd_parms 構造体へのポインタを返却。
  • 失敗時: 型が一致しない場合、luaL_checkudata により Lua 側でエラーが発生し、制御が戻らない。

前提条件と不変条件

  • 指定された index に Apache2.CommandParameters メタテーブルを持つユーザーデータが存在すること。

アルゴリズムまたは振る舞い

1. luaL_checkudata を呼び出し、スタック上の指定インデックスにあるデータが Apache2.CommandParameters 型であることを検証する。 2. lua_unboxpointer を使用して、ユーザーデータ内にラップされている cmd_parms 構造体へのポインタを取り出す。 3. 取り出したポインタを呼び出し元へ返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_config.c:162 (確認済み)
  • **呼び出し先**:
  • luaL_checkudata (Lua API)
  • lua_unboxpointer (Lua API)

関連データ構造

  • cmd_parms: Apache の設定コマンドパラメータを保持する構造体。
  • Apache2.CommandParameters: Lua 側で cmd_parms を表現するためのメタテーブル名。

失敗モードと境界条件

  • 指定されたインデックスにデータが存在しない、または型が Apache2.CommandParameters ではない場合、luaL_checkudata が Lua エラーをスローし、関数の実行が中断される。

保守担当者向けメモ

  • 本関数は mod_lua の設定バインディング層の一部であり、Lua スクリプトから Apache の設定コマンドパラメータを操作する際に使用される。
  • lua_unboxpointer は lua_config.c 内で定義されたマクロまたはヘルパー関数であり、ポインタのアンラップを行う。
