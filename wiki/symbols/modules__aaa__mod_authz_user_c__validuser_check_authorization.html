validuser_check_authorization

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_authz_user.c:85

目的

require valid-user ディレクティブに対する認可判定を行い、リクエストが認証済みユーザーによるものかを確認する。

シグネチャまたは形状

static authz_status validuser_check_authorization(request_rec *r,
                                                  const char *require_line,
                                                  const void *parsed_require_line)

入力と出力

  • **入力**:
  • r: 現在のリクエスト構造体。
  • require_line: 設定ファイル内の require 行の文字列（本関数では未使用）。
  • parsed_require_line: パース済みの設定データ（本関数では未使用）。
  • **出力**:
  • AUTHZ_GRANTED: ユーザーが認証済みの場合。
  • AUTHZ_DENIED_NO_USER: ユーザーが未認証（r->user が NULL）の場合。

前提条件と不変条件

  • r->user が設定されていることが、アクセス許可の唯一の条件である。
  • 本関数は状態を持たない。

アルゴリズムまたは振る舞い

1. リクエスト構造体 r の user フィールドを確認する。 2. r->user が NULL の場合、認証が完了していないと判断し AUTHZ_DENIED_NO_USER を返す。 3. r->user が存在する場合、認証済みユーザーとみなし AUTHZ_GRANTED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_authz_core（modules/aaa/mod_authz_user.c:122 にてプロバイダー構造体経由で参照されている）。
  • **呼び出し先**: なし。

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。
  • authz_status: 認可判定結果を示す列挙型。

失敗モードと境界条件

保守担当者向けメモ

  • 本関数は require valid-user のための非常に単純な判定ロジックである。
  • require_line および parsed_require_line は引数として受け取るが、ロジック上使用していない。将来的な仕様変更がない限り、このまま維持される。
