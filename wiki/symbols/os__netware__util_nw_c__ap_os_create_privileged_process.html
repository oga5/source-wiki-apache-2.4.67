ap_os_create_privileged_process

定義

  • 種別: function
  • 定義箇所: os/netware/util_nw.c:34

目的

NetWare 環境において、Apache HTTP Server の標準的なプロセス生成インターフェースを、プラットフォーム固有のプロセス生成機能へマッピングするラッパー関数です。

シグネチャまたは形状

AP_DECLARE(apr_status_t) ap_os_create_privileged_process(
    const request_rec *r,
    apr_proc_t *newproc, 
    const char *progname,
    const char * const *args,
    const char * const *env,
    apr_procattr_t *attr, 
    apr_pool_t *p)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • newproc: 生成されるプロセス情報を格納する apr_proc_t 構造体へのポインタ。
  • progname: 実行するプログラムのパス。
  • args: 引数リスト。
  • env: 環境変数リスト。
  • attr: プロセス属性（apr_procattr_t）。
  • p: メモリ割り当てに使用するプール。
  • **出力**:
  • apr_status_t: プロセス生成の結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • NetWare OS 上で動作していること。
  • apr_proc_create が正常に動作する環境であること。

アルゴリズムまたは振る舞い

1. 受け取った引数をそのまま apr_proc_create に転送します。 2. apr_proc_create の戻り値を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/generators/mod_cgi.c:425
  • modules/generators/mod_cgid.c:953
  • **呼び出し先**:
  • apr_proc_create (APR ライブラリ関数)

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • apr_proc_t: APR のプロセス管理構造体。
  • apr_procattr_t: APR のプロセス属性構造体。
  • apr_pool_t: APR のメモリプール。

失敗モードと境界条件

  • apr_proc_create が失敗した場合、そのエラーコードがそのまま返されます。
  • 引数 progname や attr が無効な場合、apr_proc_create の仕様に従いエラーが発生します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、NetWare 特有の追加処理は現在の実装には含まれていません。
  • include/ap_mpm.h:108 に宣言が存在します。
