check_dir_access

定義

  • 種別: function
  • 定義箇所: modules/aaa/mod_access_compat.c:315

目的

Apache HTTP Server のリクエスト処理において、レガシーなアクセス制御ディレクティブ（Order, Allow, Deny）に基づき、リクエストのアクセス可否を判定する。

シグネチャまたは形状

static int check_dir_access(request_rec *r)

入力と出力

  • **入力**: request_rec *r（現在のリクエスト情報）
  • **出力**: int（判定結果。許可の場合は OK、拒否の場合は HTTP_FORBIDDEN）

前提条件と不変条件

  • r->per_dir_config から access_compat_dir_conf 構造体が取得可能であること。
  • r->method_number が access_compat_dir_conf の order 配列のインデックスとして有効であること。

アルゴリズムまたは振る舞い

1. リクエストのメソッド番号に基づき、設定された order（ALLOW_THEN_DENY, DENY_THEN_ALLOW, またはデフォルト）を取得する。 2. order に応じて find_allowdeny を呼び出し、allows および denys 配列を走査してアクセス可否を評価する。

  • ALLOW_THEN_DENY: allow に一致すれば OK、その後 deny に一致すれば HTTP_FORBIDDEN で上書きする。
  • DENY_THEN_ALLOW: deny に一致すれば HTTP_FORBIDDEN、その後 allow に一致すれば OK で上書きする。
  • デフォルト: allow に一致し、かつ deny に一致しない場合にのみ OK とする。

    • 3. 最終的な判定結果が HTTP_FORBIDDEN の場合、ap_log_rerror を使用してエラーログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_check_access に登録されており、Apache のリクエスト処理フックから呼び出される。
  • **呼び出し先**:
  • ap_get_module_config: 設定構造体の取得。
  • find_allowdeny: ルールの評価。
  • ap_log_rerror: 拒否時のログ出力。

関連データ構造

  • access_compat_dir_conf: ディレクトリごとのアクセス制御設定を保持する構造体。
  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • ret の初期値は OK である。
  • find_allowdeny が一致しない場合、order の設定に応じたデフォルトの挙動（許可または拒否）が適用される。
  • r->filename が存在しない場合は URI をログに出力する。

保守担当者向けメモ

  • 本関数は mod_access_compat のメインエントリポイントである。
  • find_allowdeny の戻り値は真偽値として扱われるため、評価順序がアクセス制御のロジックに直結する。
  • ログ出力には APLOGNO(01797) が割り当てられている。
