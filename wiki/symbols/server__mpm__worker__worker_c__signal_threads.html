signal_threads

定義

  • 種別: function
  • 定義箇所: server/mpm/worker/worker.c:311

目的

Apache HTTP Server の worker MPM において、子プロセス内のスレッド群に対して終了モード（Graceful または Ungraceful）に応じた通知を行い、停止処理を開始する。

シグネチャまたは形状

static void signal_threads(int mode)

入力と出力

  • **入力**:
  • mode: 終了モードを指定する整数値（ST_GRACEFUL または ST_UNGRACEFUL）。
  • **出力**: なし（void）。

前提条件と不変条件

  • terminate_mode が既に指定された mode と一致する場合、関数は即座に終了する。
  • 呼び出しにより、グローバルな terminate_mode が更新され、retained->mpm->mpm_state が AP_MPMQ_STOPPING に設定される。

アルゴリズムまたは振る舞い

1. terminate_mode を更新し、サーバー状態を AP_MPMQ_STOPPING に遷移させる。 2. wakeup_listener() を呼び出し、リスナースレッドがブロッキング状態であれば解除する。 3. mode が ST_UNGRACEFUL の場合:

  • workers_may_exit フラグを 1 に設定する。
  • ap_queue_interrupt_all(worker_queue) を呼び出し、全ワーカーをキューから解放する。
  • close_worker_sockets() を呼び出し、現在処理中の全接続ソケットを強制的に閉じる。

    • 4. 最後に ap_run_child_stopping(pchild, ...) を呼び出し、子プロセスの停止処理フックを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には worker.c 内の直接的な呼び出し元は記載されていない。ツールによるヒューリスティックな検索では server/mpm/event/event.c からの呼び出しが多数検出されているが、これは worker MPM とのコード共有による参照の可能性がある。
  • **呼び出し先**:
  • wakeup_listener()
  • ap_queue_interrupt_all()
  • close_worker_sockets()
  • ap_run_child_stopping()

関連データ構造

  • retained: MPM の再起動時にも保持される状態管理構造体。
  • worker_queue: ワーカー間で共有される接続キュー。
  • pchild: 子プロセスの生存期間を管理するメモリプール。

失敗モードと境界条件

  • mode が既に設定済みの terminate_mode と一致する場合、処理はスキップされる。
  • ST_UNGRACEFUL モードでは、接続のクリーンアップを待たずにソケットが強制的に閉じられるため、進行中のリクエストは中断される。

保守担当者向けメモ

  • 本関数は worker.c 内で定義されているが、event.c からの参照が検出されている。モジュール間でのコード共有範囲に注意すること。
  • ap_run_child_stopping の第2引数は、mode == ST_GRACEFUL の場合に真となる。
