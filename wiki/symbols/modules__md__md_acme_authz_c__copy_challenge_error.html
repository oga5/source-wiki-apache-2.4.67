copy_challenge_error

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_authz.c:101

目的

ACME チャレンジの JSON オブジェクトからエラー情報を抽出し、認証コンテキスト（md_acme_authz_t）へコピーします。

シグネチャまたは形状

static int copy_challenge_error(void *baton, size_t index, md_json_t *json)

入力と出力

  • **入力**:
  • baton: error_ctx_t 構造体へのポインタ。認証コンテキストとメモリプールを保持。
  • index: 使用されないインデックス値。
  • json: 解析対象のチャレンジ JSON オブジェクト。
  • **出力**:
  • 戻り値: 常に 1 を返します。

前提条件と不変条件

  • baton は error_ctx_t 型としてキャスト可能である必要があります。
  • ctx->authz が有効な md_acme_authz_t 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. baton を error_ctx_t 型にキャストします。 2. md_json_has_key を使用し、対象の json に MD_KEY_ERROR キーが存在するか確認します。 3. キーが存在する場合、以下の情報を ctx->p（メモリプール）を使用して複製し、ctx->authz の各フィールドに格納します。

  • error_type: エラータイプ
  • error_detail: エラー詳細
  • error_subproblems: サブ問題の JSON 配列

    • 4. 処理完了後、反復処理を継続するために 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_acme_authz.c:153 において、md_json_itera のコールバック関数として呼び出されます。
  • **呼び出し先**: md_json_has_key, md_json_dups, md_json_dupj（いずれも md_json ライブラリ関数）。

関連データ構造

  • error_ctx_t: 認証コンテキストとメモリプールを保持する内部構造体。
  • md_acme_authz_t: 認証状態およびエラー情報を保持する構造体。

失敗モードと境界条件

  • json 内に MD_KEY_ERROR が存在しない場合、エラー情報の更新は行われません。
  • メモリ不足時は md_json_dups 等の内部関数が失敗する可能性があります（本関数内での明示的なエラーハンドリングは行われていません）。

保守担当者向けメモ

  • 本関数は md_json_itera による反復処理専用のコールバックです。
  • index 引数はインターフェース仕様上存在しますが、本実装では使用されていません。
