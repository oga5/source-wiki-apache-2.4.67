log_debug_dirconf

定義

  • 種別: struct
  • 定義箇所: modules/loggers/mod_log_debug.c:34

目的

ディレクトリ単位で設定された LogMessage ディレクティブの情報を保持し、リクエスト処理中に参照可能な状態にするためのコンテナ構造体です。

シグネチャまたは形状

typedef struct {
    apr_array_header_t *entries;
} log_debug_dirconf;

入力と出力

  • **入力**: LogMessage ディレクティブにより設定された複数の msg_entry（entries 配列に格納）。
  • **出力**: なし（本構造体は設定の保持のみを行う）。

前提条件と不変条件

  • entries は apr_array_header_t 型であり、初期化時に apr_pcalloc 等でメモリが割り当てられている必要があります。
  • 設定マージ時、親ディレクトリと子ディレクトリの entries は apr_array_append を用いて結合されます。

アルゴリズムまたは振る舞い

  • 本モジュールの設定管理において、ディレクトリごとの LogMessage エントリをリストとして管理します。
  • リクエスト処理中、ap_get_module_config を介して現在のリクエストコンテキストから取得され、do_debug_log 関数によって各エントリが走査されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • log_debug_create_dconf: コンフィグ領域の初期化時に使用。
  • log_debug_merge_dconf: 設定マージ時に使用。
  • do_debug_log: リクエスト処理中のログ出力判定時に使用。
  • cmd_log_message: ディレクティブ解析時に使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • msg_entry: entries 配列内に格納される個別の設定エントリ。
  • apr_array_header_t: 設定エントリを保持するための APR 配列構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pcalloc 等のメモリ割り当て失敗により、Apache のエラーログ出力やプロセス終了が発生する可能性があります。
  • 設定なし: entries が空（NULL または要素数 0）の場合、do_debug_log 内の走査処理はスキップされます。

保守担当者向けメモ

  • 本構造体は RSRC_CONF および ACCESS_CONF でのみ有効です。
  • apr_array_append を使用したマージ処理が行われるため、深いディレクトリ階層で多数の LogMessage を定義するとメモリ消費量が増大する点に注意してください。
