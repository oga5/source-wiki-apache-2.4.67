authn_ldap_config_t

定義

  • **種別**: struct
  • **定義箇所**: modules/aaa/mod_authnz_ldap.c:43

目的

Apache HTTP Server の LDAP 認証・認可モジュールにおいて、ディレクトリごとの設定（URL、ホスト、DN、フィルタ、グループ属性など）を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    apr_pool_t *pool;
#if APR_HAS_THREADS
    apr_thread_mutex_t *lock;
#endif
    char *url;
    char *host;
    int port;
    char *basedn;
    char *attribute;
    char **attributes;
    int scope;
    char *filter;
    deref_options deref;
    char *binddn;
    char *bindpw;
    int bind_authoritative;
    int user_is_dn;
    char *remote_user_attribute;
    int compare_dn_on_server;
    int have_ldap_url;
    apr_array_header_t *groupattr;
    int group_attrib_is_dn;
    char **sgAttributes;
    apr_array_header_t *subgroupclasses;
    int maxNestingDepth;
    int secure;
    char *authz_prefix;
    int initial_bind_as_user;
    ap_regex_t *bind_regex;
    const char *bind_subst;
    int search_as_user;
    int compare_as_user;
} authn_ldap_config_t;

入力と出力

  • **入力**: AuthLDAPURL 等のディレクティブ設定値に基づき初期化されます。
  • **出力**: モジュール内の各認証・認可関数が、LDAP サーバーへの接続や検索条件を決定するために本構造体を参照します。

前提条件と不変条件

  • pool メンバは、設定が割り当てられたメモリプールを指します。
  • lock メンバは、マルチスレッド環境において設定更新時の排他制御に使用されます（APR_HAS_THREADS が有効な場合）。
  • mod_ldap がロードされていることが前提となります。

アルゴリズムまたは振る舞い

  • ディレクトリ単位の設定を保持し、リクエスト処理時に ap_get_module_config を通じて取得されます。
  • 認証・認可フェーズにおいて、LDAP サーバーへのバインド、検索フィルタの構築、属性比較などの挙動を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_authnz_ldap.c 内の各関数（設定生成時やリクエスト処理時）がこの構造体を参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • authn_ldap_request_t: リクエスト単位のコンテキストを保持する構造体。
  • util_ldap が提供する接続管理機能と連携します。

失敗モードと境界条件

  • bindpw や binddn が NULL の場合、匿名バインドまたは設定に応じた挙動をとります。
  • maxNestingDepth はグループの再帰的処理における深さ制限を規定します。
  • initial_bind_as_user が有効な場合、ユーザーのパスワードが LDAP バインドに使用されるため、セキュリティ上の境界条件に注意が必要です。

保守担当者向けメモ

  • 本構造体は dir_config プールで管理されます。
  • sgAttributes は post-config フェーズで構築される配列です。
  • メンバの追加や変更を行う際は、マルチスレッド環境での lock の整合性に留意してください。
