server_merge

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_express.c:86

目的

サーバー単位の設定構造体（express_server_conf）をマージし、親サーバーの設定を継承した新しい設定構造体を生成します。

シグネチャまたは形状

static void *server_merge(apr_pool_t *p, void *basev, void *overridesv)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • basev: 継承元となるベース設定（express_server_conf）。
  • overridesv: 現在のサーバーで上書き設定された設定（express_server_conf）。
  • **出力**:
  • マージ後の新しい express_server_conf 構造体へのポインタ。

前提条件と不変条件

  • basev および overridesv は express_server_conf 型にキャスト可能である必要があります。
  • apr_pcalloc によりメモリがゼロ初期化された状態で割り当てられます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、新しい express_server_conf 構造体をメモリ上に確保します。 2. dbmfile、dbmtype、enabled の各フィールドについて、overrides 側に値が設定されていればそれを採用し、設定されていなければ base 側の値を継承します。 3. マージされた構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_express.c:247（サーバー設定の統合プロセスにおいて呼び出されます）。
  • **呼び出し先**: apr_pcalloc（APRライブラリ）。

関連データ構造

  • express_server_conf: サーバーごとの設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は Apache HTTP Server の設定マージ機構の一部です。新しい設定項目を express_server_conf に追加した場合は、必ず本関数内でマージロジックを更新してください。
