modssl_ctx_cfg_merge_server

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_config.c:364

目的

サーバーおよび仮想ホストレベルの SSL 設定コンテキストをマージし、継承関係にある設定を統合する。

シグネチャまたは形状

static void modssl_ctx_cfg_merge_server(apr_pool_t *p,
                                        modssl_ctx_t *base,
                                        modssl_ctx_t *add,
                                        modssl_ctx_t *mrg)

入力と出力

  • p: メモリ割り当てに使用する apr_pool_t。
  • base: 継承元（親）の SSL 設定コンテキスト。
  • add: 継承先（子）の SSL 設定コンテキスト。
  • mrg: マージ結果を格納する出力用コンテキスト。

前提条件と不変条件

  • base, add, mrg は有効な modssl_ctx_t 構造体を指している必要がある。
  • cfgMerge マクロおよび modssl_ctx_cfg_merge_certkeys_array 関数が、未設定項目を適切に処理できる状態であること。

アルゴリズムまたは振る舞い

1. modssl_ctx_cfg_merge を呼び出し、共通の SSL 設定項目をマージする。 2. modssl_ctx_cfg_merge_certkeys_array を使用して、証明書ファイルおよび鍵ファイルの配列をマージする（後方互換性維持のため）。 3. cfgMergeString マクロを用いて、ca_name_path および ca_name_file をマージする。 4. HAVE_TLS_SESSION_TICKETS が定義されている場合、ticket_key->file_path をマージする。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_config.c:411（ssl_config_server_merge 内から呼び出される）。
  • **呼び出し先**:
  • modssl_ctx_cfg_merge
  • modssl_ctx_cfg_merge_certkeys_array

関連データ構造

  • modssl_ctx_t: SSL 設定を保持する主要構造体。
  • apr_pool_t: メモリ管理用プール。

失敗モードと境界条件

  • base や add が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • apr_pool_t のメモリ不足時には、アロケータによるエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、2.4.7 以前のリリースとの後方互換性を維持するため、証明書と鍵の配列マージに特別な処理を行っている。変更を加える際は PR 56306 および 56353 を参照すること。
  • cfgMergeString マクロは、add 側の値が設定されていない場合に base 側の値を採用する。
