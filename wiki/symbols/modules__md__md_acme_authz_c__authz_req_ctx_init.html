authz_req_ctx_init

定義

  • 種別: function
  • 定義箇所: modules/md/md_acme_authz.c:69

目的

ACME 認証リクエスト処理用コンテキスト構造体 authz_req_ctx を初期化し、各フィールドへ値を割り当てる。

シグネチャまたは形状

static void authz_req_ctx_init(authz_req_ctx *ctx, md_acme_t *acme, 
                               const char *domain, md_acme_authz_t *authz, apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: 初期化対象の authz_req_ctx 構造体へのポインタ。
  • acme: ACME 通信セッションを保持する md_acme_t へのポインタ。
  • domain: 対象ドメイン名（文字列）。
  • authz: 認証リソースを保持する md_acme_authz_t へのポインタ。
  • p: メモリ割り当てに使用する apr_pool_t。
  • **出力**: なし（ctx が指すメモリ領域が更新される）。

前提条件と不変条件

  • ctx は有効なメモリ領域を指していること。
  • memset により ctx の全フィールドがゼロクリアされる。

アルゴリズムまたは振る舞い

1. memset を使用して ctx 構造体全体を 0 で初期化する。 2. 引数として渡された p、acme、domain、authz を ctx の各メンバに代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/md/md_acme_authz.c:287
  • modules/md/md_acme_authz.c:408
  • modules/md/md_acme_authz.c:490
  • **呼び出し先**:
  • memset (標準ライブラリ)

関連データ構造

  • authz_req_ctx: 本関数で初期化されるコンテキスト構造体。
  • md_acme_t: ACME サーバーとの通信状態を管理。
  • md_acme_authz_t: 認証リソースの状態を保持。

失敗モードと境界条件

  • ctx が NULL の場合、memset 呼び出し時にセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は静的関数であり、md_acme_authz.c 内の認証処理フローにおけるコンテキスト準備の定型処理として使用されている。
  • authz_req_ctx 構造体の定義が変更された場合、本関数の初期化ロジックも追従する必要がある。
