create_deflate_dirconf

定義

  • 種別: function
  • 定義箇所: modules/filters/mod_deflate.c:221

目的

Apache HTTP Server のディレクトリ単位の設定構造体（deflate_dirconf_t）を初期化し、デフォルト値を設定する。

シグネチャまたは形状

static void *create_deflate_dirconf(apr_pool_t *p, char *dummy)

入力と出力

  • **入力**:
  • apr_pool_t *p: 設定構造体の割り当てに使用するメモリプール。
  • char *dummy: Apache の設定コールバック規約に基づく未使用の引数。
  • **出力**:
  • void *: 初期化された deflate_dirconf_t 構造体へのポインタ。

前提条件と不変条件

  • apr_pcalloc により、構造体内の全フィールドはゼロ初期化される。
  • 戻り値の構造体は、AP_INFLATE_RATIO_LIMIT および AP_INFLATE_RATIO_BURST のデフォルト値で構成される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、リクエストプール p 上に deflate_dirconf_t サイズのメモリを確保する。 2. 確保した構造体の ratio_limit フィールドに AP_INFLATE_RATIO_LIMIT を代入する。 3. 確保した構造体の ratio_burst フィールドに AP_INFLATE_RATIO_BURST を代入する。 4. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/filters/mod_deflate.c:1947 （ディレクトリ設定作成用ハンドラとして登録）。
  • **呼び出し先**: apr_pcalloc（APR ライブラリ関数）。

関連データ構造

  • deflate_dirconf_t: 本関数によって生成されるディレクトリ設定構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボートする可能性がある（APR のメモリ管理規約に準拠）。

保守担当者向けメモ

  • 本関数は Apache の設定システムから呼び出されるコールバックである。
  • ratio_limit および ratio_burst のデフォルト値は、DoS 攻撃対策として伸長時の圧縮比を制限するために使用される。
