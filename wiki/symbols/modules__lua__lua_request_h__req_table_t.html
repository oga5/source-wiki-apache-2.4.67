req_table_t

定義

  • 種別: struct
  • 定義箇所: modules/lua/lua_request.h:43

目的

Apache の apr_table_t（ヘッダーや環境変数など）を Lua 環境から操作可能にするためのラッパー構造体です。

シグネチャまたは形状

typedef struct
{
    request_rec *r; /* Request_rec */
    apr_table_t *t; /* apr_table_t* */
    const char  *n; /* name of table */
} req_table_t;

入力と出力

  • 入力: request_rec へのポインタ、操作対象の apr_table_t へのポインタ、およびテーブル名を識別する文字列。
  • 出力: なし（構造体定義）。

前提条件と不変条件

  • r は有効な Apache リクエストコンテキストを指している必要があります。
  • t は apr_table_t 型の有効なテーブルである必要があります。

アルゴリズムまたは振る舞い

本構造体は、Lua スクリプトから Apache 内部のテーブルデータへアクセスする際の仲介役として機能します。mod_lua はこの構造体を介して、Lua のテーブルと Apache の apr_table_t 間でのデータ同期を行います。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • modules/lua/lua_apr.c: ap_lua_check_apr_table, ap_lua_push_apr_table 等で使用。
  • modules/lua/lua_request.c: req_headers_in, req_headers_out 等でインスタンス化。
  • modules/lua/lua_apr.h: 関数シグネチャ内で使用。
  • 呼び出し先: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • request_rec: Apache のリクエストコンテキスト。
  • apr_table_t: Apache の汎用テーブル構造体。

失敗モードと境界条件

  • apr_palloc によるメモリ割り当て失敗時、Apache のメモリ管理機構によりプロセスが終了する可能性があります。
  • 無効な request_rec や apr_table_t を保持した場合、Lua 側からのアクセス時にセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は apr_palloc を使用してリクエストプールからメモリを確保する設計となっています。ライフサイクルはリクエストの生存期間に依存します。
  • Lua 側への公開時には、ap_lua_push_apr_table を通じて userdata としてスタックへプッシュされます。
