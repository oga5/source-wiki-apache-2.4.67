ap_proxy_strncpy

定義

  • 種別: function
  • 定義箇所: modules/proxy/proxy_util.c:94

目的

ソース文字列を宛先バッファへ安全にコピーし、バッファサイズを超過した場合はエラーを報告する。

シグネチャまたは形状

apr_status_t ap_proxy_strncpy(char *dst, const char *src, apr_size_t dlen)

入力と出力

  • dst: コピー先の文字配列。
  • src: コピー元のヌル終端文字列（NULL の場合、dst をヌル終端して APR_SUCCESS を返す）。
  • dlen: 宛先バッファのサイズ。
  • 戻り値: コピーが成功した場合は APR_SUCCESS、バッファサイズ不足または dlen が 0 の場合は APR_EGENERAL を返す。

前提条件と不変条件

  • dlen が 0 の場合、コピーは行われず APR_EGENERAL が返される。
  • src が NULL の場合、dst の先頭に \0 が書き込まれ、コピーは行われない。

アルゴリズムまたは振る舞い

1. dlen が 0 であれば APR_EGENERAL を返す。 2. src が NULL であれば、dst をヌル終端し APR_SUCCESS を返す。 3. apr_cpystrn を使用して、最大 dlen バイトまで src を dst にコピーする。 4. コピーされた文字列の長さを計算し、src の該当位置がヌル終端文字であるかを確認する。 5. src が完全にコピーされていれば APR_SUCCESS を、そうでなければ（切り詰めが発生した場合は） APR_EGENERAL を返す。

呼び出し元と呼び出し先

  • 呼び出し元: modules/proxy/mod_proxy.h にて PROXY_STRNCPY マクロとして定義されており、プロキシモジュール全体で利用されている可能性がある。
  • 呼び出し先: apr_cpystrn (APR ライブラリ関数)。

関連データ構造

  • なし。

失敗モードと境界条件

  • dlen == 0: APR_EGENERAL を返す。
  • src が dlen バイトに収まらない場合: APR_EGENERAL を返す。
  • src が NULL: APR_SUCCESS を返す。

保守担当者向けメモ

  • 本関数は、単なるコピーではなく「バッファに収まりきったか」を厳密に検証する用途で使用される。
  • APR_EGENERAL ではなく APR_ENOSPACE を使用すべきという旨のコード内コメントが存在する。
