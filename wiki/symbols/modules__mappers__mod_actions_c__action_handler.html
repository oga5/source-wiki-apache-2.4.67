action_handler

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_actions.c:149

目的

リクエストされた MIME タイプまたは HTTP メソッドに基づき、設定された CGI スクリプトへリクエストを内部リダイレクトするハンドラ。

シグネチャまたは形状

static int action_handler(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (Apache のステータスコード: OK, DECLINED, または HTTP_NOT_FOUND)

前提条件と不変条件

  • conf->configured が真であること（設定が存在しない場合は DECLINED を返す）。
  • 無限リダイレクトループを防止するため、r->prev が二重に存在する場合は DECLINED を返す。
  • Action ディレクティブによる MIME タイプベースの処理は、Script ディレクティブによるメソッドベースの処理を上書きする。

アルゴリズムまたは振る舞い

1. **メソッドベースのスクリプト確認**: conf->scripted 配列から現在のメソッドに対応するスクリプトを特定する。 2. **アクション（MIMEタイプ）の確認**: r->handler または r->content_type をキーとして conf->action_types テーブルを検索する。 3. **スクリプトの実行**:

  • スクリプトが特定された場合、apr_table_setn で環境変数 HANDLER を設定する。
  • virtual オプションが指定されていない場合、ファイルの実在チェックを行い、存在しなければ HTTP_NOT_FOUND を返す。
  • ap_internal_redirect_handler を呼び出し、スクリプトパスに URI とクエリ文字列を付与してリクエストを転送する。

    • 4. **終了**: スクリプトが特定できない場合は DECLINED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Apache のフック機構 (ap_hook_handler により APR_HOOK_LAST で登録)。
  • **呼び出し先**: ap_internal_redirect_handler (リクエスト転送用)。

関連データ構造

  • action_dir_config: ディレクトリごとの設定（スクリプトパスやアクションの紐付け）を保持する構造体。
  • request_rec: Apache のリクエスト情報構造体。

失敗モードと境界条件

  • **ファイル不在**: virtual オプションがオフの状態で対象ファイルが存在しない場合、HTTP_NOT_FOUND を返す。
  • **ループ検知**: r->prev->prev が存在する場合、リダイレクトループとみなして DECLINED を返す。
  • **未設定**: モジュール設定が空の場合、即座に DECLINED を返す。

保守担当者向けメモ

  • 本関数はすべてのリクエストに対して実行されるため、処理の優先順位（メソッド vs MIMEタイプ）が厳密に管理されています。
  • virtual スクリプトの判定ロジックは、テーブルから取得した文字列の先頭文字 ('0' または '1') に依存しているため、設定構造体の構築ロジックと整合性を保つ必要があります。
