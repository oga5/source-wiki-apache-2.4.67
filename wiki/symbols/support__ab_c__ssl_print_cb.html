ssl_print_cb

定義

  • 種別: function
  • 定義箇所: support/ab.c:568

目的

OpenSSL の BIO オブジェクトに対するデバッグ用コールバック関数であり、SSL 通信における読み込み（read）および書き込み（write）操作のログを BIO 出力先にダンプする。

シグネチャまたは形状

static long ssl_print_cb(BIO *bio, int cmd, const char *argp, int argi, long argl, long ret)

入力と出力

  • **入力**:
  • bio: 対象の BIO オブジェクト。
  • cmd: 実行された操作の種類（BIO_CB_READ や BIO_CB_WRITE 等）。
  • argp, argi, argl: 操作に関連するバッファやサイズ情報。
  • ret: 操作の戻り値。
  • **出力**:
  • long: 操作の戻り値（ret をそのまま返す）。

前提条件と不変条件

  • BIO_get_callback_arg(bio) を介して、ログ出力先の BIO オブジェクトが設定されている必要がある。
  • out（出力先）が NULL の場合、ログ出力は行われず、即座に ret を返す。

アルゴリズムまたは振る舞い

1. BIO_get_callback_arg を使用して、コールバック引数から出力先 BIO を取得する。 2. cmd が BIO_CB_READ|BIO_CB_RETURN の場合、読み込み操作の情報を out に出力し、BIO_dump を用いて受信データをダンプする。 3. cmd が BIO_CB_WRITE|BIO_CB_RETURN の場合、書き込み操作の情報を out に出力し、BIO_dump を用いて送信データをダンプする。 4. いずれの場合も、受け取った ret をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: support/ab.c 内の BIO_set_callback_ex または BIO_set_callback を介して登録される。
  • **呼び出し先**: BIO_get_callback_arg、BIO_printf、BIO_dump（いずれも OpenSSL API）。

関連データ構造

  • BIO: OpenSSL の I/O 抽象化構造体。

失敗モードと境界条件

  • BIO_get_callback_arg(bio) が NULL を返す場合、ログ出力を行わずに正常終了する。
  • cmd が読み込みまたは書き込みの完了イベント以外である場合、何も出力せずに ret を返す。

保守担当者向けメモ

  • 本関数は OpenSSL のバージョンによって BIO_set_callback_ex と BIO_set_callback のいずれかで登録される。
  • OPENSSL_VERSION_NUMBER >= 0x30000000L の条件分岐が含まれており、OpenSSL 3.0 以降の環境での動作を考慮している。
