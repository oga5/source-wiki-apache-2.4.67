cgi_optfns_retrieve

定義

  • 種別: function
  • 定義箇所: modules/generators/cgi_common.h:208

目的

mod_include が提供するオプション関数を検索し、CGI モジュールが SSI の exec ディレクティブを処理するためのハンドラを登録します。

シグネチャまたは形状

static void cgi_optfns_retrieve(void)

入力と出力

  • 入力: なし（内部で APR_RETRIEVE_OPTIONAL_FN を使用してグローバルなオプション関数を検索）
  • 出力: なし（副作用として ap_register_include_handler を呼び出し、SSI ハンドラを登録）

前提条件と不変条件

  • mod_include が提供する ap_register_include_handler、ap_ssi_get_tag_and_value、ap_ssi_parse_string のすべてが取得可能である必要があります。
  • 取得に成功した場合のみ、cgi_handle_exec が exec ディレクティブのハンドラとして登録されます。

アルゴリズムまたは振る舞い

1. APR_RETRIEVE_OPTIONAL_FN を使用して、SSI 処理に必要な3つのオプション関数（登録用、タグ取得用、文字列解析用）を取得します。 2. 取得した関数ポインタがすべて有効（NULL でない）であることを確認します。 3. 条件を満たす場合、cgi_pfn_reg_with_ssi を呼び出し、exec ディレクティブに対して cgi_handle_exec をハンドラとして登録します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • modules/generators/mod_cgi.c:768 (ap_hook_optional_fn_retrieve 経由)
  • modules/generators/mod_cgid.c:1800 (ap_hook_optional_fn_retrieve 経由)
  • 呼び出し先:
  • ap_register_include_handler (オプション関数)
  • ap_ssi_get_tag_and_value (オプション関数)
  • ap_ssi_parse_string (オプション関数)
  • cgi_handle_exec (登録対象のハンドラ)

関連データ構造

  • cgi_pfn_reg_with_ssi: ap_register_include_handler 型の関数ポインタ。
  • cgi_pfn_gtv: ap_ssi_get_tag_and_value 型の関数ポインタ。
  • cgi_pfn_ps: ap_ssi_parse_string 型の関数ポインタ。

失敗モードと境界条件

  • 必要なオプション関数のいずれかが取得できない場合、ハンドラの登録はスキップされます。この場合、SSI の exec ディレクティブは本モジュールによって処理されません。

保守担当者向けメモ

  • 本関数は ap_hook_optional_fn_retrieve フックを通じて呼び出されるため、モジュールの初期化フェーズで実行されます。
  • cgi_pfn_reg_with_ssi 等の静的変数は、本ヘッダ内で定義されている他の関数からも参照される可能性があります。
