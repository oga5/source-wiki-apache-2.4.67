set_expiresbytype

定義

  • 種別: function
  • 定義箇所: modules/metadata/mod_expires.c:312

目的

ExpiresByType ディレクティブの設定を解析し、指定された MIME タイプに対応する有効期限設定をディレクトリ設定構造体に登録する。

シグネチャまたは形状

static const char *set_expiresbytype(cmd_parms *cmd, void *in_dir_config,
                                     const char *mime, const char *code)

入力と出力

  • **入力**:
  • cmd: コマンドパラメータ構造体。
  • in_dir_config: 現在のディレクトリ設定構造体 (expires_dir_config)。
  • mime: 設定対象の MIME タイプ文字列。
  • code: 有効期限を示す設定文字列（例: access plus 1 month）。
  • **出力**:
  • 成功時: NULL を返す。
  • 失敗時: エラーメッセージ文字列を返す。

前提条件と不変条件

  • mime 引数にはスラッシュ (/) が含まれている必要がある。
  • code 引数は check_code 関数による検証を通過する必要がある。

アルゴリズムまたは振る舞い

1. mime 文字列に / が含まれているか確認し、なければエラーを返す。 2. MIME タイプがワイルドカード（例: image/*）であるか判定し、該当する場合は dir_config->wildcards フラグを 1 に設定する。 3. check_code を呼び出し、code を内部形式（real_code）に変換する。 4. 変換に成功した場合、dir_config->expiresbytype テーブルに mime をキーとして real_code を登録する。 5. 変換に失敗した場合は、エラーメッセージを構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/metadata/mod_expires.c:353 (Apache の AP_INIT_TAKE2 マクロ経由でディレクティブとして登録)。
  • **呼び出し先**: check_code (有効期限文字列の解析用)。

関連データ構造

  • expires_dir_config: ディレクトリごとの設定を保持する構造体。expiresbytype テーブルおよび wildcards フラグを含む。

失敗モードと境界条件

  • mime に / が含まれない場合: "Invalid mimetype: should contain a slash" を返す。
  • check_code が解析に失敗した場合: check_code が生成したエラーメッセージを返す。

保守担当者向けメモ

  • 本関数は AP_INIT_TAKE2 を介して設定ファイルから呼び出される。
  • expiresbytype テーブルへの登録には apr_table_setn が使用されており、メモリ管理は cmd->pool に依存する。
  • check_code の実装に依存しているため、将来的な有効期限構文の拡張は check_code 側で行う必要がある。
