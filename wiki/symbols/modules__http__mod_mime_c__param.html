param

定義

  • 種別: struct
  • 定義箇所: modules/http/mod_mime.c:96

目的

Content-Type ヘッダ等の MIME パラメータ（属性と値のペア）を保持するための連結リスト構造体です。

シグネチャまたは形状

typedef struct param_s {
    char *attr;
    char *val;
    struct param_s *next;
} param;

入力と出力

  • attr: パラメータ名（例: charset）。
  • val: パラメータ値（例: UTF-8）。
  • next: リスト内の次のパラメータへのポインタ。

前提条件と不変条件

  • next はリストの終端で NULL を指す必要があります。
  • 各フィールドは apr_pool_t 上で割り当てられた文字列を指すことが想定されます。

アルゴリズムまたは振る舞い

analyze_ct 関数において、Content-Type 文字列を解析する際にパラメータを抽出・格納するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: analyze_ct（modules/http/mod_mime.c 内）での使用が確認されています。
  • 呼び出し先: 情報なし。
  • 注意: ツールが検出した include/ap_expr.h 内の @param は、Doxygen 等のドキュメントコメントであり、本シンボルとは無関係です。

関連データ構造

  • extension_info: mod_mime におけるメタデータ管理構造体。

失敗モードと境界条件

  • メモリ不足時: apr_pool_t からの割り当て失敗によりプロセスが終了する可能性があります。
  • 不正なフォーマット: analyze_ct がヘッダを解析する際、パラメータの構文が不正な場合は無視またはログ出力されます。

保守担当者向けメモ

  • 本構造体は mod_mime.c 内のローカルなデータ構造です。
  • analyze_ct による解析ロジックを変更する際は、この連結リストのメモリ管理（プール）が適切に行われているか確認してください。
