h2_config_merge_svr

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_config.c:232

目的

サーバーレベルの HTTP/2 設定構造体をマージし、親コンテキストから子コンテキストへ設定値を継承させる。

シグネチャまたは形状

void *h2_config_merge_svr(apr_pool_t *pool, void *basev, void *addv)

入力と出力

  • **入力**:
  • pool: メモリ割り当てに使用する apr_pool_t。
  • basev: 親（ベース）となるサーバー設定構造体へのポインタ。
  • addv: 子（追加）となるサーバー設定構造体へのポインタ。
  • **出力**:
  • マージ済みの新しいサーバー設定構造体へのポインタを void * として返す。

前提条件と不変条件

  • basev および addv は h2_config 構造体として解釈可能であること。
  • マージ処理は既存の構造体を変更せず、常に新しい構造体を生成して返す。

アルゴリズムまたは振る舞い

内部的に h2_config_merge 関数を呼び出し、階層的な設定マージを実行する。子設定が未設定（DEF_VAL）である場合に親設定の値が適用されるロジックを包含する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/mod_http2.c:55: モジュールのサーバー設定マージ用フックとして登録されている。
  • **呼び出し先**:
  • h2_config_merge (同ファイル内)

関連データ構造

  • h2_config: サーバー全体の設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pool_t の挙動に従い、例外またはエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は h2_config_merge のラッパーとして機能している。設定の継承ロジックを変更する場合は h2_config_merge を参照すること。
