piped_log

定義

  • 種別: struct
  • 定義箇所: server/log.c:186 (および http_log.h:776)

目的

サーバーと外部ログ処理プロセス間のパイプ通信を管理し、ログの転送および子プロセスのライフサイクルを保持するための構造体です。

シグネチャまたは形状

struct piped_log {
    apr_pool_t *p;
    apr_file_t *read_fd, *write_fd;
#ifdef AP_HAVE_RELIABLE_PIPED_LOGS
    char *program;
    apr_proc_t *pid;
    apr_cmdtype_e cmdtype;
#endif
};

入力と出力

  • **入力**: ap_open_piped_log 等の関数を通じて、メモリプール (apr_pool_t) および実行プログラム名が提供されます。
  • **出力**: ログ書き込み用のファイル記述子 (write_fd) および読み取り用記述子 (read_fd) を提供します。

前提条件と不変条件

  • p は、この構造体の生存期間を管理する有効な apr_pool_t である必要があります。
  • AP_HAVE_RELIABLE_PIPED_LOGS が定義されている環境では、子プロセスの異常終了時に再起動を行うためのプロセス情報 (pid) が保持されます。

アルゴリズムまたは振る舞い

  • サーバーからログをパイプ経由で外部プロセスへ転送します。
  • AP_HAVE_RELIABLE_PIPED_LOGS が有効な場合、piped_log_maintenance 関数が apr_proc_other_child_register を介して子プロセスの死活監視を行い、必要に応じて再起動を試みます。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_log_config.c や mod_log_forensic.c 等のログ関連モジュールがこの構造体を利用します。
  • **呼び出し先**: ap_piped_log_read_fd および ap_piped_log_write_fd が、構造体内のファイル記述子を返すために使用されます。また、piped_log_spawn が内部的にこの構造体を参照してプロセスを生成します。

関連データ構造

  • apr_pool_t: メモリ管理に使用。
  • apr_file_t: パイプの読み書き用ファイル記述子。
  • apr_proc_t: (条件付き) 外部ログプロセスのプロセス情報。

失敗モードと境界条件

  • パイプの作成失敗や、外部プログラムの起動失敗が発生する可能性があります。
  • AP_HAVE_RELIABLE_PIPED_LOGS が未定義の環境では、子プロセスの自動再起動機能は提供されません。

保守担当者向けメモ

  • 本構造体は server/log.c と include/http_log.h の両方で定義されています。変更時は両方の整合性に注意してください。
  • ログ出力は I/O ボトルネックになり得るため、高負荷環境でのパイプ処理の挙動には留意が必要です。
