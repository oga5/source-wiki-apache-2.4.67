vm_construct

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_vmprep.c:333

目的

ap_lua_vm_spec 構造体に基づき、新しい Lua 仮想マシン（lua_State）を構築・初期化し、Apache のライフサイクル管理下に置く。

シグネチャまたは形状

apr_status_t vm_construct(lua_State **vm, void *params, apr_pool_t *lifecycle_pool)

入力と出力

  • **入力**:
  • vm: 生成された lua_State を格納するポインタのポインタ。
  • params: ap_lua_vm_spec 構造体へのポインタ（VM の設定情報）。
  • lifecycle_pool: VM の生存期間を管理する Apache メモリプール。
  • **出力**:
  • apr_status_t: 成功時は APR_SUCCESS、失敗時は APR_EBADF 等のステータスコードを返す。

前提条件と不変条件

  • params は ap_lua_vm_spec 型としてキャスト可能であること。
  • lifecycle_pool は有効なメモリプールであり、VM 破棄時のクリーンアップに使用される。
  • Lua のレジストリには、lifecycle_pool が "Apache2.Wombat.pool" というキーで格納される。

アルゴリズムまたは振る舞い

1. luaL_newstate() で VM を生成し、標準ライブラリをロードする（AP_ENABLE_LUAJIT 定義時は JIT も有効化）。 2. spec->package_paths および spec->package_cpaths が指定されていれば、munge_path を呼び出して Lua の検索パスを調整する。 3. spec->cb が存在する場合、ユーザー定義の初期化コールバックを実行する。 4. spec->bytecode があればそれをロードし、なければ spec->file を読み込む。 5. lua_pcall を実行してスクリプトを初期化する。 6. AP_ENABLE_LUAJIT 定義時は loadjitmodule を実行する。 7. lifecycle_pool を Lua レジストリに保存し、生成した lua_State を *vm に代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/lua/lua_vmprep.c 内の関数（ap_lua_get_lua_state 等から呼び出される）。
  • **呼び出し先**:
  • luaL_newstate, luaL_openlibs, luaL_loadbuffer, luaL_loadfile, lua_pcall (Lua C API)
  • munge_path, loadjitmodule (内部関数)
  • spec->cb (コールバック関数)

関連データ構造

  • lua_State: Lua 仮想マシンの状態。
  • ap_lua_vm_spec: VM の構築パラメータ（パス、バイトコード、ファイルパス、コールバック等）。
  • apr_pool_t: メモリ管理およびクリーンアップ用プール。

失敗モードと境界条件

  • luaL_loadfile または lua_pcall が失敗した場合、エラーログを出力し APR_EBADF を返す。
  • メモリ割り当てエラー（LUA_ERRMEM）が発生した場合も同様にエラーとして扱われる。

保守担当者向けメモ

  • 本関数は ap_lua_get_lua_state 等から呼び出される低レベルな構築関数である。
  • lifecycle_pool は Lua VM の生存期間と密接に関連しており、プールが破棄されると VM も無効化される前提で設計されている。
  • munge_path によるパス操作は文字列置換に依存しているため、パス設定の形式には注意が必要である。
