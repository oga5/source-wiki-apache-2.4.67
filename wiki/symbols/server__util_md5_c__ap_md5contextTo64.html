ap_md5contextTo64

定義

  • 種別: function
  • 定義箇所: server/util_md5.c:125

目的

MD5 ハッシュコンテキストを確定させ、その結果を Base64 形式の文字列に変換して返します。

シグネチャまたは形状

char *ap_md5contextTo64(apr_pool_t *a, apr_md5_ctx_t *context)

入力と出力

  • **入力**:
  • a: メモリ割り当てに使用する apr_pool_t。
  • context: 処理対象の apr_md5_ctx_t（MD5 ハッシュ計算状態）。
  • **出力**:
  • 変換された Base64 文字列（apr_pool_t から割り当てられたメモリ領域）。

前提条件と不変条件

  • context は apr_md5_init() および apr_md5_update() によって適切に初期化・更新されている必要があります。
  • 内部で basis_64 テーブルを参照するため、当該テーブルがスコープ内で利用可能である必要があります。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、結果格納用の 25 バイトのメモリを確保します。 2. apr_md5_final を呼び出し、ハッシュ計算を完了させます。 3. ダイジェストの末尾 2 バイトを 0 で埋めます。 4. basis_64 テーブルを用い、3 バイト単位でビットシフトを行いながら Base64 エンコードを行います。 5. 文字列の末尾に == パディングを付与し、ヌル終端文字列として返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • apr_pcalloc
  • apr_md5_final
  • **呼び出し元**:
  • server/util_md5.c 内で利用されています。

関連データ構造

  • apr_pool_t: メモリ管理用プール。
  • apr_md5_ctx_t: MD5 計算コンテキスト。
  • basis_64: Base64 エンコード用の変換テーブル（ソースファイル内で定義）。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc が異常終了する可能性があります（APR のメモリ管理ポリシーに依存）。
  • 入力 context が無効な場合、apr_md5_final の動作は未定義です。

保守担当者向けメモ

  • 本関数は標準的な Base64 ライブラリを使用せず、レガシーな互換性のために独自実装されています。
  • 変換ロジックは 3 バイト単位の処理を前提としているため、ダイジェストのサイズ変更時には注意が必要です。
