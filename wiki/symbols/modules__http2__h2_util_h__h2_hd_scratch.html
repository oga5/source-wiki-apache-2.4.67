h2_hd_scratch

定義

  • 種別: struct
  • 定義箇所: modules/http2/h2_util.h:400

目的

HTTP/2 ヘッダーフィールドの処理中に、ヘッダー名と値を一時的に保持するためのスクラッチパッド（作業用メモリ領域）を提供します。

シグネチャまたは形状

typedef struct h2_hd_scratch {
    size_t max_len; /* ヘッダー名 + ': ' + 値 の最大長 */
    char *name;     /* max_len + 1 バイトのバッファ */
    char *value;    /* max_len + 1 バイトのバッファ */
} h2_hd_scratch;

入力と出力

本構造体は関数引数として渡され、主にヘッダー変換や構築処理の過程で、文字列操作のバッファとして利用されます。

前提条件と不変条件

  • name および value メンバは、少なくとも max_len + 1 バイトのメモリ領域を指している必要があります。
  • 構造体のライフサイクルは、呼び出し元のコンテキスト（セッションやストリーム）に依存します。

アルゴリズムまたは振る舞い

HTTP/2 ヘッダーの変換や検証を行う際、動的なメモリ割り当てを繰り返すオーバーヘッドを避けるために使用されます。既存のバッファを再利用することで、ヘッダー処理のパフォーマンスを最適化します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • modules/http2/h2_request.c: リクエスト処理時に使用。
  • modules/http2/h2_session.h: セッション構造体の一部として保持。
  • modules/http2/h2_stream.c: ストリーム処理時に使用。
  • modules/http2/h2_util.c: ヘッダー変換ユーティリティ関数内で使用。
  • **呼び出し先**:
  • 構造体であるため、関数としての呼び出し先はありません。

関連データ構造

  • nghttp2_nv: nghttp2 ライブラリが使用するヘッダー形式。本構造体は、この形式への変換や検証の過程で併用されます。

失敗モードと境界条件

  • max_len で指定されたサイズを超えるヘッダーを処理しようとした場合、バッファオーバーフローが発生する可能性があるため、呼び出し側での適切なサイズ管理が必須です。

保守担当者向けメモ

  • 本構造体は h2_session や h2_stream の一部として埋め込まれることが多いため、メモリレイアウトや生存期間の変更には注意が必要です。
  • h2_util.c 内のヘッダー処理関数と密接に結合しているため、変更時は関連する h2_util 関数群の動作確認を行ってください。
