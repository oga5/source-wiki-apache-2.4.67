bs2_ForkType

定義

  • 種別: enum
  • 定義箇所: os/bs2000/os.c:35

目的

BS2000（OSD/POSIX）環境において、Apache HTTP Server がプロセスを生成する際に使用する手法を識別・管理するために定義された列挙型です。

シグネチャまたは形状

typedef enum {
    bs2_unknown,
    bs2_noFORK,
    bs2_FORK,
    bs2_UFORK
} bs2_ForkType;

入力と出力

  • 本シンボル自体は列挙型定義であり、入力や出力は持ちません。
  • この型を使用する関数（os_forktype 等）は、実行環境のフラグやユーザー権限に基づき、この列挙値のいずれかを返します。

前提条件と不変条件

  • _OSD_POSIX マクロが定義されている環境でのみ有効です。
  • bs2_unknown は初期化前の状態を示します。

アルゴリズムまたは振る舞い

プロセス生成手法の決定に使用されます：

  • bs2_noFORK: -X フラグ指定時など、フォークを行わない場合。
  • bs2_FORK: uid が 0 以外の場合など、標準の fork() を使用する場合。
  • bs2_UFORK: ユーザー ID の切り替えが必要な場合に、BS2000 特有の ufork() を使用する場合。

呼び出し元と呼び出し先

  • 呼び出し元: os_forktype()（os/bs2000/os.c:48）および os_fork()（os/bs2000/os.c:78）にて、プロセス生成戦略の判定および実行に使用されます。
  • 呼び出し先: 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • static bs2_ForkType forktype: os/bs2000/os.c 内で、判定結果をキャッシュするために使用される静的変数。

失敗モードと境界条件

  • 初期化されていない状態（bs2_unknown）で参照された場合、後続のプロセス生成ロジックが意図しない挙動をとる可能性があります。

保守担当者向けメモ

  • os/unix/unixd.c にも同名の定義が存在しますが、本ページは os/bs2000/os.c の文脈に基づいています。
  • ufork() はユーザー名の大文字変換などの OS 固有制約を伴うため、この列挙型を用いた条件分岐を変更する際は、BS2000 のジョブ環境設定への影響を考慮してください。
