core_request_config

定義

  • 種別: struct
  • 定義箇所: include/http_core.h:394

目的

HTTP リクエストのライフサイクル全体を通じて保持される、リクエストごとの一時的な状態を管理するための構造体です。バケットブリゲード、作業用データ、カスタムレスポンス情報、およびコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct {
    struct apr_bucket_brigade *bb;
    void **notes;
    char **response_code_strings;
    const char *document_root;
    const char *context_document_root;
    const char *context_prefix;
    int deliver_script;
    int suppress_charset;
} core_request_config;

入力と出力

  • **入力**: ap_get_core_module_config(r->request_config) を介してリクエスト構造体から取得されます。
  • **出力**: フィルタチェーン（ap_core_input_filter / ap_core_output_filter）やハンドラが、この構造体内のメンバ（bb や document_root 等）を参照・更新します。

前提条件と不変条件

  • 本構造体はリクエストプール（r->pool）から割り当てられます。
  • notes 配列は、初期化時に ap_register_request_note() を使用して要素を登録し、ap_get_request_note() でアクセスする必要があります。

アルゴリズムまたは振る舞い

  • **バケット管理**: bb は getline による先読みや、リクエストボディの残りを保持するために使用されます。
  • **コンテキスト管理**: context_document_root および context_prefix は、mod_alias や mod_userdir などのモジュールによって設定され、リソースの物理パスと URI プレフィックスを管理します。
  • **スクリプト処理**: deliver_script フラグが有効な場合、default_handler は GET 以外のメソッドであってもスクリプトファイルとして処理を継続します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/generators/mod_cgid.c
  • modules/http/http_filters.c
  • modules/http/http_protocol.c
  • server/core.c
  • **呼び出し先**:
  • ap_get_core_module_config() (設定取得)
  • ap_custom_response() (カスタムレスポンス文字列の登録)
  • ap_get_request_note() / ap_register_request_note() (作業用データの管理)

関連データ構造

  • apr_bucket_brigade: フィルタチェーン間でのデータ転送用。
  • request_rec: リクエストの基本構造体。
  • core_dir_config: ディレクトリ単位の設定（本構造体で見つからない場合のフォールバック先）。

失敗モードと境界条件

  • response_code_strings が NULL の場合、core_dir_config を参照する必要があります。
  • notes 配列へのアクセス時に ID が正しく登録されていない場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はリクエスト処理の基盤であり、フィルタチェーンの挙動に直結します。
  • document_root メンバは、mass vhosting モジュールとの互換性のために存在しますが、可能な限り context_* 情報を優先して使用してください。
