dav_lock_discovery_fixed

定義

  • 種別: struct
  • 定義箇所: modules/dav/fs/lock.c:115

目的

WebDAV ロック情報の永続化において、直接ロック（direct lock）の固定長属性を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct dav_lock_discovery_fixed
{
    char scope;
    char type;
    int depth;
    time_t timeout;
} dav_lock_discovery_fixed;

入力と出力

本構造体は、ロック情報をバイナリ形式で DBM データベースへ保存・復元する際のデータレイアウトとして機能します。直接的な関数呼び出しの入力/出力ではなく、メモリ上のデータ表現として使用されます。

前提条件と不変条件

  • dav_size_direct マクロにより、本構造体のサイズに 1 バイトを加えた値が、シリアライズ時のデータ長として計算に使用されます。
  • ロックの有効期限（timeout）は time_t 型で管理されます。

アルゴリズムまたは振る舞い

ロック情報のシリアライズおよびデシリアライズ処理において、構造体のメモリレイアウトをそのままバイナリデータとして扱うことで、効率的な永続化を実現します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/dav/fs/lock.c および modules/dav/lock/locks.c 内のロック管理ロジック。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • dav_lock_discovery: 直接ロックの情報を保持する構造体であり、本構造体はその一部（固定長部分）を構成します。

失敗モードと境界条件

  • 本構造体は固定長であるため、フィールドの型定義に依存します。プラットフォーム間での time_t や int のサイズ差異が、DBM ファイルの互換性に影響を与える可能性があります。

保守担当者向けメモ

  • modules/dav/lock/locks.c にも同名の構造体定義が存在します。変更を加える際は、ファイルシステムバックエンドとコアロジックの両方で整合性が保たれているか確認してください。
  • dav_size_direct マクロの定義と本構造体のサイズが常に同期していることを確認してください。
