rb_methods

定義

  • 種別: enum
  • 定義箇所: modules/proxy/mod_proxy_http.c:287

目的

HTTP プロキシ処理における、バックエンドへのリクエストボディ転送方式を定義する列挙型です。

シグネチャまたは形状

typedef enum {
    RB_INIT = 0,
    RB_STREAM_CL,
    RB_STREAM_CHUNKED,
    RB_SPOOL_CL
} rb_methods;

入力と出力

  • 入力: なし（列挙型の定義）
  • 出力: RB_INIT, RB_STREAM_CL, RB_STREAM_CHUNKED, RB_SPOOL_CL のいずれかの値。

前提条件と不変条件

  • RB_INIT は初期状態を示します。
  • 転送方式は、リクエストのプリフェッチ処理後に決定され、proxy_http_req_t 構造体の rb_method メンバに保持されます。

アルゴリズムまたは振る舞い

  • RB_STREAM_CL: Content-Length を用いたストリーミング転送。
  • RB_STREAM_CHUNKED: チャンク転送エンコーディングを用いたストリーミング転送。
  • RB_SPOOL_CL: 一時ファイル等にスプールした後の Content-Length 転送。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • modules/proxy/mod_proxy_http.c:311: proxy_http_req_t 構造体内のメンバ定義。
  • modules/proxy/mod_proxy_http.c:331: req->rb_method からの代入処理。
  • 呼び出し先: 情報なし。

関連データ構造

  • proxy_http_req_t: リクエスト状態を管理する構造体であり、本列挙型を保持します。

失敗モードと境界条件

  • 転送方式が RB_INIT のまま処理が進んだ場合、適切な転送ロジックが選択されない可能性があります。

保守担当者向けメモ

  • 新たな転送方式を追加する場合は、stream_reqbody 関数内での条件分岐を更新する必要があります。
