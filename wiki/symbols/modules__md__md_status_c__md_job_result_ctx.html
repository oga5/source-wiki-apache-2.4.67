md_job_result_ctx

定義

  • 種別: struct
  • 定義箇所: modules/md/md_status.c:542

目的

証明書更新ジョブ（md_job_t）の実行結果を監視し、その状態を永続化ストレージと同期させるためのコンテキストを保持します。

シグネチャまたは形状

typedef struct {
    apr_pool_t *p;
    md_job_t *job;
    md_store_t *store;
    md_result_t *last;
    apr_time_t last_save;
} md_job_result_ctx;

入力と出力

本構造体は、ジョブの監視プロセスにおいて、メモリプール、ジョブハンドル、ストレージハンドル、および最新の処理結果を保持するコンテナとして機能します。

前提条件と不変条件

  • p は、このコンテキストの生存期間を管理する apr_pool_t である必要があります。
  • job および store は、監視対象のジョブおよびストレージへの有効なポインタである必要があります。

アルゴリズムまたは振る舞い

本構造体は、md_result_t の変更を監視するコールバック関数（job_result_update 等）内で使用されます。ジョブの結果が更新されるたびに、このコンテキストを通じてログへの追記やストレージへの保存処理が行われます。last_save フィールドは、最後にストレージへ保存した時刻を記録し、過度な書き込みを抑制するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/md/md_status.c 内のジョブ監視関連関数（job_observation_start 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • md_job_t: 監視対象のジョブインスタンス。
  • md_store_t: ジョブ状態を永続化するためのストレージインターフェース。
  • md_result_t: ジョブの実行結果やエラー詳細を保持するコンテナ。

失敗モードと境界条件

  • 構造体内の各ポインタが NULL の場合、参照時にクラッシュする可能性があります。
  • apr_pool_t が破棄された場合、本構造体も無効となります。

保守担当者向けメモ

  • 本構造体は md_status.c 内のジョブ監視ロジックに特化しています。
  • last_save を用いた保存間隔の制御ロジックについては、md_job_end_run 等の関連関数での実装を確認してください。
