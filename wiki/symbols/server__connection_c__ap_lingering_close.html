ap_lingering_close

定義

  • 種別: function
  • 定義箇所: server/connection.c:149

目的

HTTP 接続終了時に、クライアントからの残存データを読み捨てることで、TCP の正常終了（FIN/ACK 交換）を保証し、RST パケットによるデータ欠損を防ぐ。

シグネチャまたは形状

void ap_lingering_close(conn_rec *c)

入力と出力

  • **入力**: conn_rec *c（接続状態を保持する構造体）
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • ap_get_conn_socket(c) により有効なソケットが取得できること。
  • ソケットが取得できない場合、関数は c->aborted = 1 を設定して即座に終了する。

アルゴリズムまたは振る舞い

1. **初期化**: ap_start_lingering_close(c) を呼び出し、書き込み側のシャットダウンを行う。失敗した場合は即座にソケットを閉じる。 2. **設定**: ソケットのタイムアウトを SECONDS_TO_LINGER（2秒）に設定し、APR_INCOMPLETE_READ を有効化する。 3. **読み込みループ**:

  • apr_socket_recv を使用して残存データを読み捨てる。
  • 初回読み込み時に、c->notes に short-lingering-close が存在すれば2秒、それ以外は MAX_SECS_TO_LINGER（30秒）を終了期限（timeup）として設定する。
  • 読み込みデータが0バイトになるか、現在時刻が timeup を超えるまでループを継続する。

    • 4. **終了**: 最後に apr_socket_close(csd) を呼び出し、ソケットを解放する。

関連データ構造

  • conn_rec: 接続情報を保持する構造体。
  • apr_socket_t: APR レベルのソケットハンドル。

失敗モードと境界条件

  • **ソケット不在**: ap_get_conn_socket が NULL を返す場合、安全に終了する。
  • **読み込み失敗**: apr_socket_recv がエラーを返した場合、ループを中断してソケットを閉じる。
  • **タイムアウト**: 期限内にデータが読み込めない場合、ループを終了してソケットを閉じる。

保守担当者向けメモ

  • short-lingering-close ノートは DoS 攻撃緩和のために使用される。
  • タイムアウト値（2秒および30秒）は Apache 1.3 からの継承値であり、変更時はネットワーク負荷への影響を考慮すること。
