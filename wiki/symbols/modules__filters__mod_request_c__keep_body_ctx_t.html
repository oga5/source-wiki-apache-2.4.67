keep_body_ctx_t

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_request.c:56

目的

リクエストボディをメモリ上に保持する際、現在の保持状況と制限値を管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct keep_body_filter_ctx {
    apr_off_t remaining;
    apr_off_t keep_body;
} keep_body_ctx_t;

入力と出力

  • remaining: 現在保持可能な残りバイト数。
  • keep_body: 設定された最大保持サイズ（KeptBodySize ディレクティブに基づく）。

前提条件と不変条件

  • keep_body_filter 実行時に、フィルタのコンテキスト（f->ctx）として割り当てられます。
  • remaining は、リクエストボディの読み込みが進むにつれて減少します。

アルゴリズムまたは振る舞い

  • keep_body_filter がリクエストボディを読み込む際、この構造体を参照して保持サイズ制限を超過していないかを確認します。
  • 制限を超過した場合、bail_out_on_error を呼び出し、エラーレスポンスを生成して処理を中断します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/filters/mod_request.c 内の keep_body_filter 関数が、f->ctx をこの型にキャストして使用しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • request_rec: HTTP リクエスト情報を保持する構造体。
  • apr_bucket_brigade: 実際に保持されるリクエストボディのデータ本体。

失敗モードと境界条件

  • remaining が 0 に達した状態でさらにデータが入力された場合、HTTP_REQUEST_ENTITY_TOO_LARGE エラーとして処理されます。

保守担当者向けメモ

  • 本構造体は keep_body_filter 専用のステート管理用です。サブリクエストへの再挿入に使用される kept_body_ctx_t と混同しないよう注意してください。
