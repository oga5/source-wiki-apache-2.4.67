send_headers

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_scgi.c:258

目的

Apache のリクエスト環境変数を SCGI プロトコル仕様に従った「ネットストリング（netstring）」形式にシリアライズし、バックエンドへ送信する。

シグネチャまたは形状

static int send_headers(request_rec *r, proxy_conn_rec *conn)

入力と出力

  • **入力**:
  • r: 現在のリクエスト情報（subprocess_env を含む）。
  • conn: バックエンドへの接続情報。
  • **出力**:
  • int: 成功時は OK、失敗時は sendall が返すエラーコード。

前提条件と不変条件

  • r->subprocess_env に CGI 環境変数が設定されていること。
  • CONTENT_LENGTH は常にヘッダーの先頭に配置される。
  • GATEWAY_INTERFACE、CONTENT_LENGTH、SCGI_MAGIC はループ処理から除外される。
  • すべての動的メモリ割り当ては r->pool を使用する。

アルゴリズムまたは振る舞い

1. ap_add_common_vars(r) および ap_add_cgi_vars(r) を呼び出し、環境変数を準備する。 2. r->subprocess_env を走査し、ヘッダー全体のサイズを計算する。 3. r->remaining から取得したボディ長をネットストリングの先頭変数として設定する。 4. バッファを確保し、以下の順序でデータをコピーする:

  • ネットストリングの長さ（ns_len）
  • CONTENT_LENGTH 変数
  • SCGI_MAGIC および SCGI_PROTOCOL_VERSION
  • 残りの環境変数（キーと値をヌル文字で区切る）

    • 5. 最後にカンマ（,）を付与し、sendall を介してバックエンドへ送信する。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_scgi.c:580（scgi_handler 内）。
  • **呼び出し先**: sendall（バックエンドへのデータ送信）。

関連データ構造

  • request_rec: リクエストのコンテキストと環境変数を保持。
  • proxy_conn_rec: バックエンド接続の管理。
  • apr_table_entry_t: 環境変数のキー・値ペアの格納に使用。

失敗モードと境界条件

  • sendall が失敗した場合、その戻り値がそのまま返される。
  • 環境変数の数が 0 の場合でも、必須のヘッダー（CONTENT_LENGTH 等）は送信される。

保守担当者向けメモ

  • 本関数は SCGI プロトコルのネットストリング形式を構築する責務を持つ。
  • GATEWAY_INTERFACE 等の特定の変数をスキップするロジックは、SCGI 仕様に基づくハードコードである。
  • sendall の挙動については、mod_proxy の共通基盤を参照すること。
