deflate_filter_config

定義

  • 種別: struct
  • 定義箇所: modules/filters/mod_deflate.c:64

目的

Apache HTTP Server の mod_deflate モジュールにおいて、サーバー全体の設定（圧縮レベル、バッファサイズ、ログ用ノート名等）を保持するための構造体です。

シグネチャまたは形状

typedef struct deflate_filter_config_t
{
    int windowSize;
    int memlevel;
    int compressionlevel;
    int bufferSize;
    const char *note_ratio_name;
    const char *note_input_name;
    const char *note_output_name;
    int etag_opt;
} deflate_filter_config;

入力と出力

  • **入力**: サーバー設定読み込み時に ap_get_module_config を介して取得され、各設定ディレクティブの値で初期化されます。
  • **出力**: フィルタ処理関数（consume_buffer, flush_libz_buffer 等）へ引数として渡され、圧縮動作のパラメータとして参照されます。

前提条件と不変条件

  • apr_pcalloc によってメモリが確保され、初期化される必要があります。
  • フィルタ処理中に参照される際、設定値は変更されず読み取り専用として扱われます。

アルゴリズムまたは振る舞い

  • サーバー単位の設定値を一元管理し、mod_deflate の各フィルタが zlib を呼び出す際のパラメータ（ウィンドウサイズ、メモリレベル、圧縮レベル）を提供します。
  • 圧縮統計情報を記録するためのノート名（note_ratio_name 等）を保持し、リクエスト処理中にこれらの名前を使用して情報を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • create_deflate_server_config（設定の初期化）
  • 各種設定ディレクティブ処理関数（ap_get_module_config を介した取得）
  • deflate_out_filter 等のフィルタ処理関数
  • **呼び出し先**:
  • consume_buffer
  • flush_libz_buffer

関連データ構造

  • deflate_ctx: フィルタの実行時状態を保持する構造体。本構造体と組み合わせてフィルタ処理に使用されます。

失敗モードと境界条件

  • 設定値が未定義または不正な値である場合、zlib の初期化関数（deflateInit2 等）がエラーを返す可能性があります。
  • 構造体自体はメモリ確保に失敗しない限り、境界条件による直接的なクラッシュは想定されていません。

保守担当者向けメモ

  • 本構造体はサーバー全体の設定を保持します。ディレクトリ単位の設定が必要な場合は deflate_dirconf_t が使用されるため、混同しないよう注意してください。
  • etag_opt は ETag の処理方針を決定するために使用されます。
