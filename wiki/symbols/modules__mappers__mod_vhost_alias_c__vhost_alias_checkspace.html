vhost_alias_checkspace

定義

  • 種別: function
  • 定義箇所: modules/mappers/mod_vhost_alias.c:237

目的

動的なバーチャルホストパス生成時に、一時バッファ buf が HUGE_STRING_LEN を超過するのを防ぎ、生成されたパスを r->filename に安全に蓄積する。

シグネチャまたは形状

static APR_INLINE void vhost_alias_checkspace(request_rec *r, char *buf, char **pdest, int size)

入力と出力

  • r: リクエストレコード。r->filename の更新およびメモリ割り当て（r->pool）に使用。
  • buf: パス構築用の一時バッファ。
  • pdest: buf 内の現在の書き込み位置を指すポインタへのポインタ。
  • size: 次に追加しようとしているデータのサイズ。
  • 出力: なし（副作用として r->filename を更新し、pdest を buf の先頭にリセットする）。

前提条件と不変条件

  • buf は HUGE_STRING_LEN のサイズを持つことが前提となっている。
  • *pdest は常に buf 内の有効な書き込み位置を指している必要がある。

アルゴリズムまたは振る舞い

1. 現在の書き込み位置 *pdest に size を加えた値が buf の終端（HUGE_STRING_LEN）を超えるか判定する。 2. 超過する場合:

  • 現在の buf の内容を終端文字 \0 で閉じる。
  • r->filename が既に存在する場合は apr_pstrcat で連結し、存在しない場合は apr_pstrdup でコピーして r->filename を更新する。
  • *pdest を buf の先頭にリセットし、以降の書き込みをバッファの先頭から再開させる。

    • 3. 超過しない場合は何もしない。

呼び出し元と呼び出し先

  • 呼び出し元: modules/mappers/mod_vhost_alias.c 内の vhost_alias_interpolate 関数（285, 294, 302, 362行目）。
  • 呼び出し先: apr_pstrcat, apr_pstrdup（Apache Portable Runtime API）。

関連データ構造

  • request_rec: Apache のリクエスト情報を保持する構造体。

失敗モードと境界条件

  • size > HUGE_STRING_LEN: ソースコード内のコメントにて「size が HUGE_STRING_LEN を超える場合」の挙動が未定義（XXX）として指摘されている。
  • メモリ不足: apr_pstrcat や apr_pstrdup が失敗した場合、APR のメモリ管理ポリシーに従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は APR_INLINE で定義されており、パフォーマンスを考慮してインライン展開される。
  • r->filename の更新はリクエストプール（r->pool）を使用しているため、リクエスト終了時に自動的に解放される。
  • size が HUGE_STRING_LEN を超えるケースについては、将来的な堅牢性向上のために検討が必要。
