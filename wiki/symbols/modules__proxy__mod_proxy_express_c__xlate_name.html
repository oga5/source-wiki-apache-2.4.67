xlate_name

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_express.c:112

目的

リクエストのホスト名をキーとしてDBMファイルを検索し、バックエンドのプロキシ先URLを動的に解決してリクエストを書き換える。

シグネチャまたは形状

static int xlate_name(request_rec *r)

入力と出力

  • **入力**: request_rec *r (現在のリクエスト情報)
  • **出力**: int (Apacheのステータスコード: OK または DECLINED)

前提条件と不変条件

  • proxy_express_module が有効であること。
  • DBMファイルが APR_DBM_READONLY モードで読み取り可能であること。
  • リクエストのホスト名がDBM内のキーとして存在すること。

アルゴリズムまたは振る舞い

1. sconf->enabled を確認し、無効な場合は DECLINED を返す。 2. r->filename が既に proxy: で始まる場合は、内部プロキシリクエストとみなし DECLINED を返す。 3. apr_dbm_open2 (または apr_dbm_open_ex) を使用してDBMファイルを開く。 4. ap_get_server_name(r) で取得したホスト名をキーに apr_dbm_fetch を実行。 5. 解決成功時、r->filename を proxy:<backend><uri> に書き換え、r->handler を proxy-server に設定し、r->proxyreq を PROXYREQ_REVERSE に設定する。 6. 現在のディレクトリ設定 (dconf) 内の raliases を走査し、該当バックエンドの ProxyPassReverse エントリが存在しない場合は動的に追加する。 7. 処理終了時に apr_dbm_close を呼び出しリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ap_hook_translate_name により APR_HOOK_FIRST で登録され、Apacheの翻訳フェーズで呼び出される。
  • **呼び出し先**:
  • ap_get_module_config, ap_log_rerror, ap_get_server_name (Apache API)
  • apr_dbm_get_driver, apr_dbm_open2, apr_dbm_open_ex, apr_dbm_fetch, apr_dbm_close (APR DBM API)
  • apr_pstrmemdup, apr_pstrcat, apr_pstrdup, apr_array_push (APR メモリ管理/配列操作)

関連データ構造

  • express_server_conf: サーバーごとの設定（DBMパス、タイプ、有効フラグ）。
  • proxy_dir_conf: ディレクトリごとの設定（raliases を保持）。
  • proxy_alias: ProxyPassReverse エントリを管理する構造体。

失敗モードと境界条件

  • DBMファイルが開けない場合や、ホスト名に対応する値が見つからない場合は DECLINED を返す。
  • ProxyPassReverse の動的追加はプロセス/スレッド固有の dconf に対して行われるため、設定の反映範囲がプロセス境界に依存する可能性がある。

保守担当者向けメモ

  • DBM操作にはリクエストプール (r->pool) を使用しており、リクエスト終了時に自動的にメモリが解放される。
  • APU_MAJOR_VERSION 等の条件分岐により、APRのバージョン差異を吸収している。
  • dconf への ProxyPassReverse 追加は、現在のプロセスまたはスレッドがバックエンドを処理している間のみ有効である点に注意すること。
