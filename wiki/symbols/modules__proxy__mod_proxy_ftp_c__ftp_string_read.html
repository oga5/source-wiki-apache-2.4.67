ftp_string_read

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_ftp.c:215

目的

ネットワークバケットから FTP サーバーのレスポンスを 1 行ずつ読み込み、指定されたバッファに格納するヘルパー関数。

シグネチャまたは形状

apr_status_t ftp_string_read(conn_rec *c, apr_bucket_brigade *bb, char *buff, apr_size_t bufflen, int *eos, apr_size_t *outlen)

入力と出力

  • **入力**:
  • c: 接続情報 (conn_rec)
  • bb: バケットブリゲード (apr_bucket_brigade)
  • buff: 読み込み先バッファ
  • bufflen: バッファの最大長
  • **出力**:
  • buff: 読み込まれた文字列（NULL 終端）
  • eos: ストリーム終了フラグ（終了時に 1）
  • outlen: 実際に読み込まれたバイト数
  • 戻り値: apr_status_t 型のステータスコード

前提条件と不変条件

  • buff は bufflen 以上のサイズを確保していること。
  • bb は ap_get_brigade を通じてネットワークからデータを取得するために使用される。
  • 読み込みは APR_BLOCK_READ モードで行われる。

アルゴリズムまたは振る舞い

1. buff を初期化し、eos と outlen を 0 に設定する。 2. ap_get_brigade を呼び出し、AP_MODE_GETLINE モードでネットワークからデータを取得する。 3. 取得したバケットを順次走査し、以下の処理を行う：

  • EOS バケットを検出した場合は eos を 1 に設定する。
  • データバケットから内容を読み込み、LF (ASCII 10) が含まれているか確認する。
  • LF が見つかった時点で読み込み完了 (found = 1) とみなす。
  • バッファの残り容量に応じてデータを buff にコピーし、outlen を更新する。
  • 処理済みのバケットを削除する。

    • 4. LF が見つかるまで、または接続が切断されるまでループを繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/proxy/mod_proxy_ftp.c 内の 392 行目および 417 行目から呼び出されている。
  • **呼び出し先**: ap_get_brigade、apr_bucket_read、apr_bucket_delete、memchr、memcpy。

関連データ構造

  • conn_rec: Apache の接続状態を保持する構造体。
  • apr_bucket_brigade: ネットワーク I/O のためのバケット管理構造体。

失敗モードと境界条件

  • **接続異常**: ap_get_brigade がエラーを返した場合、そのステータスをそのまま返す。
  • **接続切断**: バケットが空、または EOS に到達したにもかかわらず LF が見つからない場合は APR_ECONNABORTED を返す。
  • **バッファオーバーフロー**: bufflen を超えるデータは切り捨てられ、バッファに収まる分のみが格納される。

保守担当者向けメモ

  • 本関数は ap_rgetline と類似の機能を持つが、歴史的経緯により独自実装されている。
  • ASCII 以外の文字コードに対する挙動は保証されていない。
  • LF の検出効率を向上させるため、バッファの末尾を直接確認する最適化の余地がある（ソース内のコメント参照）。
