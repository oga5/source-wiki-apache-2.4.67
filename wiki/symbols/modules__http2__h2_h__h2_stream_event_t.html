h2_stream_event_t

定義

  • 種別: enum
  • 定義箇所: modules/http2/h2.h:153

目的

HTTP/2 ストリームの状態遷移をトリガーするためのイベント種別を定義します。

シグネチャまたは形状

typedef enum {
    H2_SEV_CLOSED_L,
    H2_SEV_CLOSED_R,
    H2_SEV_CANCELLED,
    H2_SEV_EOS_SENT,
    H2_SEV_IN_ERROR,
    H2_SEV_IN_DATA_PENDING,
    H2_SEV_OUT_C1_BLOCK,
} h2_stream_event_t;

入力と出力

本シンボルは列挙型であり、関数への引数として渡されることで、ストリームの状態遷移ロジックに対する入力を構成します。戻り値はありません。

前提条件と不変条件

  • h2_stream_dispatch 関数等を通じて、特定の h2_stream インスタンスに対してイベントが発行される必要があります。
  • 各イベントは、ストリームのライフサイクルにおける特定の状態変化（終了、キャンセル、エラー、データ保留など）に対応しています。

アルゴリズムまたは振る舞い

  • h2_stream_dispatch や on_state_event などの関数において、渡されたイベント値に基づき、ストリームの状態マシンを遷移させるための識別子として機能します。
  • 実行時のロジックは、この列挙値を用いて、ストリームが現在どのようなイベントに直面しているかを判定し、適切な後続処理（クリーンアップやエラーハンドリング等）を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_session.c: on_stream_state_event, on_stream_event
  • modules/http2/h2_stream.c: on_event, on_state_event, h2_stream_dispatch
  • **呼び出し先**:
  • 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • h2_stream: ストリームの管理構造体。本イベントはこの構造体の状態遷移に使用されます。

失敗モードと境界条件

  • 未定義のイベント値が渡された場合の挙動は、呼び出し側の実装（switch 文の default 節など）に依存します。

保守担当者向けメモ

  • 新しいストリームイベントを追加する場合は、本列挙型に値を追加するとともに、h2_stream.c 内の状態遷移ロジックを更新する必要があります。
