modssl_ERR_peek_error_data

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_log.c:81

目的

OpenSSL のバージョン差異（1.1.x 系と 3.0+ 系）を吸収し、エラーキューからエラー情報を取得するための互換レイヤーとして機能します。

シグネチャまたは形状

unsigned long modssl_ERR_peek_error_data(const char **data, int *flags)

入力と出力

  • **入力**:
  • data: エラーに関連するデータ文字列を受け取るためのポインタのポインタ。
  • flags: エラーデータのフラグを受け取るための整数ポインタ。
  • **出力**:
  • 戻り値: OpenSSL のエラーコード（unsigned long）。エラーがない場合は 0 を返します。

前提条件と不変条件

  • OpenSSL ライブラリがリンクされている環境で動作します。
  • 呼び出し元は、data および flags に有効なメモリ領域のアドレスを渡す必要があります。

アルゴリズムまたは振る舞い

1. コンパイル時の OPENSSL_VERSION_NUMBER を判定します。 2. OpenSSL 3.0 未満の場合: ERR_peek_error_line_data(NULL, NULL, data, flags) を呼び出し、ファイル名や行番号情報を除外してエラーデータとフラグを取得します。 3. OpenSSL 3.0 以上の場合: ERR_peek_error_data(data, flags) を直接呼び出します。 4. 取得したエラーコードを呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_log.c 内の ssl_log_ssl_error 関数が、エラーキューを走査するループ内で本関数を呼び出しています。
  • **呼び出し先**:
  • ERR_peek_error_line_data (OpenSSL 3.0 未満)
  • ERR_peek_error_data (OpenSSL 3.0 以上)

関連データ構造

  • 特になし（OpenSSL の標準的なエラー型を使用）。

失敗モードと境界条件

  • エラーキューが空の場合、戻り値として 0 を返します。
  • OpenSSL の API 仕様に基づき、data や flags に渡されたポインタが指す値は、エラーが存在しない場合には更新されない可能性があります。

保守担当者向けメモ

  • 本関数は OpenSSL 3.0 への移行に伴う API 変更を隠蔽するための重要な互換レイヤーです。将来的に OpenSSL のバージョン要件が変更された場合、条件分岐の整理が必要です。
