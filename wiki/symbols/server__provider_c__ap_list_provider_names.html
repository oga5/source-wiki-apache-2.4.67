ap_list_provider_names

定義

  • 種別: function
  • 定義箇所: server/provider.c:127

目的

指定されたプロバイダーグループおよびバージョンに登録されているプロバイダー名のリストを取得します。

シグネチャまたは形状

apr_array_header_t *ap_list_provider_names(apr_pool_t *pool,
                                           const char *provider_group,
                                           const char *provider_version);

入力と出力

  • **入力**:
  • pool: リストの割り当てに使用するメモリプール。
  • provider_group: 検索対象のプロバイダーグループ名。
  • provider_version: 検索対象のプロバイダーバージョン。
  • **出力**:
  • 成功時: ap_list_provider_names_t 構造体の配列（apr_array_header_t）。
  • 失敗時: 空の配列（要素数 0 または 1 の初期化済み配列）。

前提条件と不変条件

  • global_providers_names ハッシュテーブルが初期化されている必要があります。
  • 戻り値の配列は、指定された pool の寿命に従います。

アルゴリズムまたは振る舞い

1. グローバルなプロバイダー名ハッシュテーブル (global_providers_names) から、provider_group に対応するハッシュを取得します。 2. 取得したグループハッシュから、さらに provider_version に対応するハッシュ（プロバイダー名の集合）を取得します。 3. 該当するハッシュが存在しない場合は、空の配列を返します。 4. 該当するハッシュが存在する場合、その内容をイテレートし、各プロバイダー名を apr_pstrdup で複製して apr_array_header_t に格納します。 5. 最終的に、プロバイダー名を含む配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/core/mod_watchdog.c
  • modules/dav/main/mod_dav.c
  • modules/dav/main/std_liveprop.c
  • modules/generators/mod_info.c
  • modules/proxy/mod_proxy_balancer.c
  • modules/ssl/ssl_engine_config.c
  • server/request.c
  • **呼び出し先**:
  • apr_hash_get, apr_hash_count, apr_hash_first, apr_hash_next, apr_hash_this (APR ハッシュ操作)
  • apr_array_make, apr_array_push (APR 配列操作)
  • apr_pstrdup (メモリ割り当てと文字列コピー)

関連データ構造

  • global_providers_names: プロバイダーのメタデータを保持する内部ハッシュテーブル。
  • ap_list_provider_names_t: プロバイダー名を保持する構造体。
  • apr_array_header_t: 結果を格納する APR 配列型。

失敗モードと境界条件

  • global_providers_names が未初期化の場合、または指定されたグループやバージョンが見つからない場合は、空の配列を返して終了します。
  • メモリ不足時は apr_array_make や apr_pstrdup が例外（またはアボート）を発生させる可能性があります。

保守担当者向けメモ

  • 本関数は読み取り専用の操作であり、登録済みのプロバイダー一覧を動的に取得するために使用されます。
  • 実行時のプロバイダー登録変更は想定されていないため、マルチスレッド環境での競合には注意が必要です。
