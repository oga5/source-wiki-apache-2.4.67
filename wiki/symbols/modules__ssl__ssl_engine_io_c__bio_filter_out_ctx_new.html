bio_filter_out_ctx_new

定義

  • 種別: function
  • 定義箇所: modules/ssl/ssl_engine_io.c:120

目的

SSL出力フィルタ用のコンテキスト構造体 (bio_filter_out_ctx_t) を新規作成し、初期化する。

シグネチャまたは形状

static bio_filter_out_ctx_t *bio_filter_out_ctx_new(ssl_filter_ctx_t *filter_ctx, conn_rec *c);

入力と出力

  • **入力**:
  • filter_ctx: SSL接続のメインコンテキスト (ssl_filter_ctx_t) へのポインタ。
  • c: 現在の接続レコード (conn_rec) へのポインタ。
  • **出力**:
  • 初期化済みの bio_filter_out_ctx_t 構造体へのポインタ。

前提条件と不変条件

  • c->pool および c->bucket_alloc が有効であること。
  • 戻り値のコンテキストは、接続の生存期間中有効なメモリプールから割り当てられる。

アルゴリズムまたは振る舞い

1. c->pool を使用して bio_filter_out_ctx_t 構造体分のメモリを割り当てる。 2. 引数で受け取った filter_ctx および c を構造体のメンバに格納する。 3. c->pool と c->bucket_alloc を使用して、暗号化前のデータを保持するための空の apr_bucket_brigade を作成し、outctx->bb に割り当てる。 4. 初期化されたコンテキスト構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/ssl/ssl_engine_io.c:2271 にて、BIO_set_data の引数として呼び出されている。
  • **呼び出し先**:
  • apr_palloc (APRメモリ割り当て)
  • apr_brigade_create (APRバケットブリゲード作成)

関連データ構造

  • bio_filter_out_ctx_t: 本関数が生成する出力フィルタ用コンテキスト。
  • ssl_filter_ctx_t: SSL接続のメインコンテキスト。
  • conn_rec: HTTPDの接続状態を保持する構造体。
  • apr_bucket_brigade: データ転送に使用されるバケットのリスト。

失敗モードと境界条件

  • メモリ割り当て (apr_palloc) またはブリゲード作成 (apr_brigade_create) が失敗した場合、APRの規約に従い、通常はプロセスが終了するか例外が発生する。

保守担当者向けメモ

  • 本関数で作成されたコンテキストは、OpenSSLのBIOオブジェクトに BIO_set_data で関連付けられ、後続の出力フィルタ処理で参照される。
  • apr_bucket_brigade のライフサイクルは conn_rec のプールと同期しているため、明示的な破棄は接続終了時に行われる必要がある。
