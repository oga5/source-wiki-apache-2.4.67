priv_mode

定義

  • 種別: enum
  • 定義箇所: modules/arch/unix/mod_privileges.c:45

目的

mod_privileges モジュールにおいて、リクエスト処理時の権限分離およびプロセス実行モードを定義するために使用されます。

シグネチャまたは形状

typedef enum { 
    PRIV_UNSET, 
    PRIV_FAST, 
    PRIV_SECURE, 
    PRIV_SELECTIVE 
} priv_mode;

入力と出力

  • 入力: 設定ファイル（PrivilegesMode ディレクティブ）から読み取られた値。
  • 出力: priv_cfg や priv_dir_cfg 構造体の一部として保持され、リクエスト処理時の権限降格ロジックの分岐条件として使用されます。

前提条件と不変条件

  • 本モジュールは Solaris 等の特権機能を利用するため、OS が対応している必要があります。
  • PRIV_SECURE モードが選択された場合、リクエストは別プロセスで実行されます。

アルゴリズムまたは振る舞い

  • PRIV_UNSET: モードが未設定の状態。
  • PRIV_FAST: 権限を動的に変更するが、プロセス分離は行わないモード。
  • PRIV_SECURE: リクエストを fork し、子プロセス側で権限を最小化して実行するモード。
  • PRIV_SELECTIVE: 特定の権限セットを選択的に適用するモード。

呼び出し元と呼び出し先

  • 呼び出し元: modules/arch/unix/mod_privileges.c 内の設定構造体（priv_cfg 等）およびリクエスト処理関数。
  • 呼び出し先: 直接的な関数呼び出しではなく、条件分岐の判定値として使用されます。

関連データ構造

  • priv_cfg: サーバー/vhost レベルの設定構造体。
  • priv_dir_cfg: ディレクトリ単位の設定構造体。

失敗モードと境界条件

  • 設定値が PRIV_UNSET のままリクエスト処理に到達した場合の挙動は、モジュール内のデフォルト設定に依存します。

保守担当者向けメモ

  • 本列挙型は mod_privileges の動作モードを決定する重要な指標です。新しいモードを追加する場合は、privileges_req 関数内の switch 文や条件分岐をすべて更新する必要があります。
