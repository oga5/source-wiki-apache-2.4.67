authn_anon_config_rec

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authn_anon.c:64

目的

Apache HTTP Server の mod_authn_anon モジュールにおいて、ディレクトリごとの匿名認証設定を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    anon_auth_user *users;
    int nouserid;
    int logemail;
    int verifyemail;
    int mustemail;
    int anyuserid;
} authn_anon_config_rec;

入力と出力

  • **入力**: Anonymous ディレクティブ等の設定値が、anon_set_string_slots 等の関数を通じて本構造体の各フィールドに格納されます。
  • **出力**: 認証処理時、check_anonymous 関数が本構造体を参照し、認証の可否を決定するためのフラグとして利用します。

前提条件と不変条件

  • apr_palloc 等でメモリ確保された後に使用されます。
  • 各 int フィールドは、認証ポリシー（メールアドレス形式の検証要否など）を示すフラグとして機能します。

アルゴリズムまたは振る舞い

  • 許可されたユーザー名のリスト（users）と、認証時の挙動を制御するフラグ群を保持します。
  • check_anonymous 関数において、リクエストごとの設定として読み込まれ、ユーザー名やパスワードの検証ロジックの分岐条件として使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authn_anon.c 内の各種設定関数および check_anonymous 関数が、設定の読み込みや参照のために使用しています。
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • anon_auth_user: 許可ユーザーリストを構成するデータ構造。
  • per_dir_config: Apache のリクエスト処理において、本構造体が格納されるディレクトリ設定コンテキスト。

失敗モードと境界条件

  • 設定が適切に初期化されていない場合、認証ロジックが意図しない挙動（認証の拒否または誤った許可）を引き起こす可能性があります。

保守担当者向けメモ

  • APR_OFFSETOF を使用して設定スロットのオフセットが計算されているため、構造体のメンバ順序を変更する際は注意が必要です。
  • 認証ポリシーの追加や変更を行う場合は、本構造体にフィールドを追加し、設定解析ロジックを更新してください。
