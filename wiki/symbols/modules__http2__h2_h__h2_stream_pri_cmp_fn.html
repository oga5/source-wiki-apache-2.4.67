h2_stream_pri_cmp_fn

定義

  • 種別: typedef
  • 定義箇所: modules/http2/h2.h:194

目的

HTTP/2 ストリーム間の優先度を比較するためのコールバック関数型です。mod_http2 におけるストリームのスケジューリングや再優先順位付けの際に使用されます。

シグネチャまたは形状

typedef int h2_stream_pri_cmp_fn(int stream_id1, int stream_id2, void *session);

入力と出力

  • **入力**:
  • stream_id1: 比較対象のストリーム ID (int)
  • stream_id2: 比較対象のストリーム ID (int)
  • session: セッションコンテキストへのポインタ (void*)
  • **出力**:
  • 比較結果を示す整数値。一般的に、stream_id1 が stream_id2 より優先される場合は負の値、逆の場合は正の値、同等の場合は 0 を返します。

前提条件と不変条件

  • session 引数は、呼び出し元が管理するセッションコンテキストを指している必要があります。
  • 比較関数は、与えられた session 内でストリームの優先度を決定するために必要な情報を参照可能である必要があります。

アルゴリズムまたは振る舞い

  • 本シンボルは関数ポインタの型定義であり、具体的な比較ロジックは実装側に委ねられます。
  • 呼び出し元は、この関数を使用してストリームの優先順位を評価し、リソースの割り当てや処理順序を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/http2/h2_mplx.c: h2_mplx_c1_reprioritize 関数などで、ストリームの再優先順位付けを行う際にこの関数ポインタを受け取ります。
  • modules/http2/h2_mplx.h: h2_mplx_c1_reprioritize の宣言において引数として使用されています。
  • **呼び出し先**:
  • この型に適合する具体的な比較関数が、モジュール内の各所で実装・提供されることが想定されています。

関連データ構造

  • h2_mplx: ストリームの多重化と優先度管理を行う構造体。
  • h2_session: セッション状態を保持する構造体（session 引数として渡される）。

失敗モードと境界条件

  • stream_id が無効な場合や、session が NULL の場合の挙動は、実装側の比較関数に依存します。
  • 比較関数内でセッション状態へのアクセスが競合しないよう、呼び出し元での適切な同期が必要です。

保守担当者向けメモ

  • 本型定義は h2_mplx 層でのストリーム優先度制御の抽象化を目的としています。
  • 新たな優先度アルゴリズムを導入する際は、このシグネチャに従った関数を実装し、h2_mplx_c1_reprioritize 等に渡すことで統合可能です。
