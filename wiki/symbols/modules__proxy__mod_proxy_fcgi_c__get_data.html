get_data

定義

  • 種別: function
  • 定義箇所: modules/proxy/mod_proxy_fcgi.c:288

目的

バックエンドのFastCGIサーバーからデータを読み込み、接続に関連付けられたワーカの統計情報を更新する。

シグネチャまたは形状

static apr_status_t get_data(proxy_conn_rec *conn, char *buffer, apr_size_t *buflen)

入力と出力

  • **入力**:
  • conn: バックエンド接続情報を保持する proxy_conn_rec 構造体。
  • buffer: 受信データを格納するバッファ。
  • buflen: 読み込むバイト数（入力時）および実際に読み込まれたバイト数（出力時）。
  • **出力**:
  • apr_status_t: 操作の結果（成功時は APR_SUCCESS）。

前提条件と不変条件

  • conn および conn->sock が有効であること。
  • buflen が指す値が、buffer の容量を超えないこと。

アルゴリズムまたは振る舞い

1. apr_socket_recv を呼び出し、conn->sock からデータを読み込む。 2. 読み込みが成功した場合（APR_SUCCESS）、conn->worker->s->read に読み込んだバイト数を加算し、統計情報を更新する。 3. 読み込み結果のステータスコードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/proxy/mod_proxy_fcgi.c:311
  • modules/proxy/mod_proxy_fcgi.c:805
  • **呼び出し先**:
  • apr_socket_recv (APRライブラリ)

関連データ構造

  • proxy_conn_rec: バックエンド接続状態を管理する構造体。
  • proxy_worker: バックエンドのワーカ情報を保持する構造体。

失敗モードと境界条件

  • apr_socket_recv がエラーを返した場合、統計情報の更新は行われず、そのエラーコードがそのまま返される。

保守担当者向けメモ

  • 本関数はブロッキングI/Oを前提としています。
  • 統計情報の更新は conn->worker->s を介して行われるため、マルチスレッド環境下での整合性は proxy_worker の実装に依存します。
