sendfile_handler

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_file_cache.c:291

目的

キャッシュされたファイルハンドルを使用して、OSの sendfile 機能により効率的にレスポンスを送信する。

シグネチャまたは形状

static int sendfile_handler(request_rec *r, a_file *file)

入力と出力

  • **入力**:
  • r: 現在のリクエストレコード。
  • file: キャッシュされたファイル情報を含む a_file 構造体。
  • **出力**:
  • OK: 正常終了。
  • AP_FILTER_ERROR: ap_pass_brigade による送信失敗時。

前提条件と不変条件

  • APR_HAS_SENDFILE が定義されている環境でのみ有効。
  • file 引数は有効なファイルハンドルおよびファイル属性（finfo.size）を保持している必要がある。

アルゴリズムまたは振る舞い

1. リクエストのプールから apr_bucket_brigade を作成する。 2. apr_brigade_insert_file を使用し、キャッシュされたファイル全体をバケットとしてブリゲードに挿入する。 3. ストリームの終了を示す EOS (End of Stream) バケットを作成し、ブリゲードの末尾に挿入する。 4. ap_pass_brigade を呼び出し、出力フィルタチェーンを通じてデータを送信する。 5. 送信結果が APR_SUCCESS でない場合は AP_FILTER_ERROR を返し、それ以外は OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/cache/mod_file_cache.c:377 (file_cache_handler 内)
  • **呼び出し先**:
  • apr_brigade_create
  • apr_brigade_insert_file
  • apr_bucket_eos_create
  • APR_BRIGADE_INSERT_TAIL
  • ap_pass_brigade

関連データ構造

  • request_rec: Apache のリクエスト情報。
  • a_file: mod_file_cache 固有のキャッシュファイル管理構造体。
  • apr_bucket_brigade: 送信データのバッファリング用構造体。

失敗モードと境界条件

  • APR_HAS_SENDFILE が未定義の場合、この関数は何も行わず OK を返す（実質的な無操作）。
  • ap_pass_brigade がエラーを返した場合、AP_FILTER_ERROR を返して処理を中断する。

保守担当者向けメモ

  • 本関数は APR_HAS_SENDFILE マクロに依存しているため、プラットフォームごとの sendfile サポート状況に挙動が左右される。
  • file->file が指すファイルハンドルがクローズされていないことを前提としている。
