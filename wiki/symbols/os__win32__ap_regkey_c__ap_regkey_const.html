ap_regkey_const

定義

  • 種別: function
  • 定義箇所: os/win32/ap_regkey.c:30

目的

Windows の定義済みルートレジストリキー（HKEY）を、APR プール管理に対応した ap_regkey_t 構造体として取得します。

シグネチャまたは形状

const ap_regkey_t *ap_regkey_const(int i)

入力と出力

  • **入力**: int i - 取得するルートキーを識別するインデックス（0〜6）。
  • **出力**: const ap_regkey_t * - 指定されたインデックスに対応する ap_regkey_t 構造体へのポインタ。

前提条件と不変条件

  • 関数内部で定義された静的配列 ap_regkey_consts の要素数は 7 です。
  • 引数 i は 0 から 6 の範囲である必要があります。範囲外の値を指定した場合の動作は定義されていません。
  • 返される ap_regkey_t 構造体内の pool メンバは NULL です。これは、これらのキーが永続的なシステムハンドルであり、APR プールによるクリーンアップの対象外であることを示します。

アルゴリズムまたは振る舞い

1. 関数スコープ内の静的配列 ap_regkey_consts を初期化します。これには以下の順序で HKEY が格納されます：

  • 0: HKEY_CLASSES_ROOT
  • 1: HKEY_CURRENT_CONFIG
  • 2: HKEY_CURRENT_USER
  • 3: HKEY_LOCAL_MACHINE
  • 4: HKEY_USERS
  • 5: HKEY_PERFORMANCE_DATA
  • 6: HKEY_DYN_DATA

    • 2. 引数 i をオフセットとして、配列内の該当する要素のアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: include/ap_regkey.h 内の以下のマクロから参照されます。
  • AP_REGKEY_CLASSES_ROOT (i=0)
  • AP_REGKEY_CURRENT_CONFIG (i=1)
  • AP_REGKEY_CURRENT_USER (i=2)
  • AP_REGKEY_LOCAL_MACHINE (i=3)
  • AP_REGKEY_USERS (i=4)
  • AP_REGKEY_PERFORMANCE_DATA (i=5)
  • AP_REGKEY_DYN_DATA (i=6)
  • **呼び出し先**: 外部関数への依存はありません。

関連データ構造

  • ap_regkey_t: レジストリキーハンドル (HKEY) と apr_pool_t を保持するコンテナ構造体。

失敗モードと境界条件

  • 引数 i が 0〜6 の範囲外である場合、配列外参照が発生します。

保守担当者向けメモ

  • 本関数は静的配列を返すため、スレッドセーフな読み取り専用アクセスが可能です。
  • 新しいルートキー定数を追加する場合は、配列サイズとインデックスの整合性に注意してください。
