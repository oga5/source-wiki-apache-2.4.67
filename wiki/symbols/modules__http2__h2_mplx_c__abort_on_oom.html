abort_on_oom

定義

  • 種別: function
  • 定義箇所: modules/http2/h2_mplx.c:78

目的

メモリ割り当て失敗（OOM）発生時にプロセスを強制終了させるためのコールバック関数です。

シグネチャまたは形状

int abort_on_oom(int retcode)

入力と出力

  • **入力**: retcode (int) - 呼び出し元が保持する戻り値。
  • **出力**: 本関数は ap_abort_on_oom() を呼び出した後、呼び出し元へ retcode を返そうと試みますが、実際にはプロセスが終了するため到達不能です。

前提条件と不変条件

  • ap_abort_on_oom() が正常に動作し、プロセスを終了できる環境であること。

アルゴリズムまたは振る舞い

1. ap_abort_on_oom() を呼び出し、メモリ不足による致命的なエラーとしてプロセスを終了させます。 2. 引数として受け取った retcode を戻り値として返します。ただし、このコードパスは到達不能であることが想定されています。

呼び出し元と呼び出し先

  • **呼び出し先**: ap_abort_on_oom()
  • **呼び出し元**: modules/http2/h2_mplx.c:227 にて apr_pool_abort_set の引数として登録されています。また、server/main.c や support/ 配下の各ツールからも同様に apr_pool_abort_set を介して参照されています。

関連データ構造

  • apr_pool_t: メモリプール管理構造体（apr_pool_abort_set を通じて本関数が関連付けられます）。

失敗モードと境界条件

  • 本関数自体が失敗するケースは想定されていません。ap_abort_on_oom() がプロセスを終了させられない環境下では、呼び出し元に制御が戻る可能性があります。

保守担当者向けメモ

  • 本関数は apr_pool_abort_set に登録して使用することを前提としています。
  • support/passwd_common.h など、他のソースファイルでも同名の関数が定義されている可能性があるため、リンク時の重複定義やスコープに注意してください。
