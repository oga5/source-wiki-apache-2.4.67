authn_alias_srv_conf

定義

  • 種別: struct
  • 定義箇所: modules/aaa/mod_authn_core.c:68

目的

Apache HTTP Server の認証プロバイダーエイリアス（<AuthnProviderAlias>）をサーバー全体で管理するための設定構造体です。

シグネチャまたは形状

typedef struct authn_alias_srv_conf {
    apr_hash_t *alias_rec;
} authn_alias_srv_conf;

入力と出力

  • **入力**: サーバー設定時に apr_hash_t 型のハッシュテーブルとして初期化されます。
  • **出力**: 認証実行時に、このハッシュテーブルからエイリアス名に対応する provider_alias_rec 構造体が検索されます。

前提条件と不変条件

  • alias_rec は apr_hash_t であり、エイリアス名をキーとして provider_alias_rec を格納します。
  • 本構造体はサーバーレベルの設定（server_rec->module_config）として保持されます。

アルゴリズムまたは振る舞い

1. **設定時**: AuthnProviderAlias ディレクティブの解析時に、この構造体のハッシュテーブルへ新しいエイリアス定義が登録されます。 2. **実行時**: 認証が必要なリクエストが発生すると、ap_get_module_config を介して本構造体を取得し、リクエストに関連付けられたプロバイダー名をキーにして alias_rec から該当するエイリアス設定を特定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/aaa/mod_authn_core.c 内の認証実行関数（authn_alias_check_password, authn_alias_get_realm_hash）が、リクエスト処理中に本構造体を取得します。
  • authaliassection（ディレクティブ解析関数）が、エイリアス登録のために本構造体を参照します。
  • create_authn_alias_svr_config が本構造体のメモリ確保と初期化を行います。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • provider_alias_rec: alias_rec ハッシュテーブルの各エントリに格納される、エイリアス化されたプロバイダーの定義構造体。
  • apr_hash_t: エイリアス名を管理するためのハッシュテーブル。

失敗モードと境界条件

  • サーバー設定時に alias_rec が初期化されていない場合、認証実行時の検索で NULL ポインタ参照が発生する可能性があります。
  • 存在しないエイリアス名で認証が要求された場合、ハッシュ検索は失敗します。

保守担当者向けメモ

  • 本構造体はサーバー起動時に確定する GLOBAL_ONLY な設定を保持します。
  • r->server->module_config から取得する際は、適切なモジュールポインタを指定してキャストする必要があります。
