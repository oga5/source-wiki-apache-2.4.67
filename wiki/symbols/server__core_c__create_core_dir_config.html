create_core_dir_config

定義

  • 種別: function
  • 定義箇所: server/core.c:153

目的

Apache HTTP Server のディレクトリ単位の設定構造体（core_dir_config）をメモリ上に確保し、デフォルト値で初期化します。

シグネチャまたは形状

static void *create_core_dir_config(apr_pool_t *a, char *dir)

入力と出力

  • **入力**:
  • a: 設定構造体を割り当てるための apr_pool_t プール。
  • dir: 設定対象のディレクトリパス（文字列）。
  • **出力**:
  • 初期化済みの core_dir_config 構造体へのポインタ（void * として返却）。

前提条件と不変条件

  • apr_pcalloc を使用してメモリを確保するため、構造体のメンバは明示的に初期化されない限り NULL または 0 で初期化されます。
  • dir が指定されている場合、conf->opts は OPT_UNSET に設定されます。指定がない場合は OPT_UNSET|OPT_SYM_LINKS が適用されます。

アルゴリズムまたは振る舞い

1. apr_pcalloc を呼び出し、core_dir_config サイズのメモリをゼロクリアして確保します。 2. ディレクトリ設定に関連する各種フラグ（opts, override, content_md5, limit_req_body 等）を、未設定状態を示す定数（_UNSET 系）またはデフォルト値で初期化します。 3. 文字セットのデフォルト値として DEFAULT_ADD_DEFAULT_CHARSET_NAME を設定します。 4. 初期化された構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: server/core.c:5824 にて、ディレクトリ単位の設定構造体を作成する際に呼び出されます。
  • **呼び出し先**: apr_pcalloc（APRライブラリ関数）。

関連データ構造

  • core_dir_config: 本関数が初期化する主要な構造体。
  • apr_pool_t: メモリ管理に使用されるプール構造体。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc がアボートする可能性があります（APRのメモリ管理仕様に依存）。
  • dir 引数が NULL の場合、opts の初期値が OPT_SYM_LINKS を含む特殊な状態になります。

保守担当者向けメモ

  • 本関数で初期化される core_dir_config の各メンバは、後続の merge_core_dir_configs で親ディレクトリの設定とマージされる前提で設計されています。
  • 新しい設定項目を core_dir_config に追加した場合は、必ず本関数で適切な _UNSET 値またはデフォルト値による初期化を行ってください。
