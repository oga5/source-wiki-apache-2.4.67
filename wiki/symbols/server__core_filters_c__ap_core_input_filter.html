ap_core_input_filter

定義

  • 種別: function
  • 定義箇所: server/core_filters.c:90

目的

HTTP リクエストの読み込みにおいて、OS のソケット層からデータを取得し、フィルタチェーン上の上位モジュールへ供給する最下位の入力フィルタとして機能します。

シグネチャまたは形状

apr_status_t ap_core_input_filter(ap_filter_t *f, apr_bucket_brigade *b,
                                  ap_input_mode_t mode, apr_read_type_e block,
                                  apr_off_t readbytes)

入力と出力

  • **入力**:
  • f: フィルタコンテキストを含むフィルタ構造体。
  • b: 読み込んだデータを格納するバケットブリゲード。
  • mode: 読み込みモード（AP_MODE_GETLINE, AP_MODE_READBYTES 等）。
  • block: ブロッキング読み込みの可否。
  • readbytes: 読み込み要求バイト数。
  • **出力**:
  • apr_status_t: 処理結果（APR_SUCCESS, APR_EOF, またはエラーコード）。

前提条件と不変条件

  • f->ctx に core_net_rec が保持されている必要があります。
  • 初回呼び出し時に core_ctx_t が初期化され、ソケットバケットがブリゲードに挿入されます。
  • 内部状態 ctx->b は常に正規化（BRIGADE_NORMALIZE）された状態を維持することが期待されます。

アルゴリズムまたは振る舞い

1. **初期化**: AP_MODE_INIT の場合は何もしません。ctx が未初期化であれば、ソケットからデータを読み込むためのブリゲードを生成し、ネットワークバケットをシードします。 2. **正規化**: BRIGADE_NORMALIZE を実行し、空のバケットを削除します。 3. **モード別処理**:

  • AP_MODE_GETLINE: apr_brigade_split_line を使用して行単位でデータを抽出します。
  • AP_MODE_EATCRLF: ソケット内の CRLF/LF を消費し、空行をスキップします。
  • AP_MODE_EXHAUSTIVE: ソケット内の全データをブリゲードに連結し、EOS バケットを付与します。
  • AP_MODE_READBYTES / AP_MODE_SPECULATIVE: 要求バイト数分を読み込みます。SPECULATIVE の場合はデータをコピーしてブリゲードに挿入し、元のブリゲードを維持します。

    • 4. **終了**: 読み込み結果を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: フィルタチェーンの上位モジュール（ap_get_brigade を経由）。
  • **呼び出し先**: ap_run_insert_network_bucket（初期化時）、apr_bucket_read（データ読み込み時）、apr_brigade_split_line（行読み込み時）。

関連データ構造

  • ap_filter_t: フィルタチェーンのノード。
  • core_net_rec: ネットワーク接続のコア情報。
  • core_ctx_t: 入力フィルタの内部状態（b: 読み込み用ブリゲード, tmpbb: 一時作業用ブリゲード）。
  • apr_bucket_brigade: データのバケット列。

失敗モードと境界条件

  • **EOF**: ソケットからの読み込みが終了した場合、APR_EOF を返します。
  • **EAGAIN**: 非ブロッキング読み込み時にデータが未到着の場合、モードに応じて APR_SUCCESS または APR_EAGAIN を返します。
  • **エラー**: apr_bucket_read 等の APR 関数がエラーを返した場合、そのステータスを伝播します。

保守担当者向けメモ

  • AP_MODE_PEEK（AP_MODE_EATCRLF）の名称と挙動に懸念が示されています。
  • BRIGADE_NORMALIZE 後の空ブリゲードチェックは、一部の API が APR_EOF ではなくエラーコードを期待する仕様に対応するための暫定的な実装です。
  • AP_MODE_EXHAUSTIVE は非常に危険なモードであるため、使用には細心の注意が必要です。
