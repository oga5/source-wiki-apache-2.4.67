val_H2_PUSHED_ON

定義

  • 種別: function
  • 定義箇所: modules/http2/mod_http2.c:253

目的

HTTP/2 リクエストがどのストリームによってプッシュ（Server Push）されたかを示すストリームIDを、文字列として取得します。

シグネチャまたは形状

static const char *val_H2_PUSHED_ON(apr_pool_t *p, server_rec *s,
                                    conn_rec *c, request_rec *r,
                                    h2_conn_ctx_t *conn_ctx)

入力と出力

  • **入力**:
  • p: メモリ割り当て用プール。
  • conn_ctx: 現在の接続コンテキスト。
  • **出力**:
  • プッシュされたストリームである場合、その親ストリームIDの文字列表現を返します。
  • 条件を満たさない場合、空文字列 "" を返します。

前提条件と不変条件

  • conn_ctx が有効である必要があります。
  • conn_ctx->stream_id が設定されており、かつそのストリームがクライアントによって開始されたものではない（サーバープッシュである）場合にのみ値が計算されます。

アルゴリズムまたは振る舞い

1. conn_ctx が存在するか確認します。 2. conn_ctx->stream_id が存在し、かつ H2_STREAM_CLIENT_INITIATED マクロが偽（サーバープッシュ）であることを確認します。 3. h2_mplx_c2_stream_get を使用して、現在の多重化器から該当ストリームを取得します。 4. ストリームが存在する場合、その initiated_on メンバ（プッシュ元のストリームID）を apr_itoa で文字列に変換して返します。 5. 上記の条件のいずれかが満たされない場合は、空文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/http2/mod_http2.c:304 の h2_var_def 配列内で参照されています。
  • **呼び出し先**:
  • h2_mplx_c2_stream_get (ストリーム取得)
  • apr_itoa (数値の文字列変換)

関連データ構造

  • h2_conn_ctx_t: 接続ごとのHTTP/2コンテキスト。
  • h2_stream: 個別のストリーム状態を保持する構造体。

失敗モードと境界条件

  • conn_ctx が NULL の場合、空文字列を返します。
  • ストリームがクライアントによって開始されたものである場合、プッシュ元IDは取得されず空文字列を返します。
  • h2_mplx_c2_stream_get がストリームを見つけられない場合、空文字列を返します。

保守担当者向けメモ

  • 本関数は環境変数 H2_PUSHED_ON の値を解決するために使用されます。
  • H2_STREAM_CLIENT_INITIATED マクロの定義と整合性を保つ必要があります。
