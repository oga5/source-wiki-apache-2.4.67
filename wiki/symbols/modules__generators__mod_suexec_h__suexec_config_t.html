suexec_config_t

定義

  • 種別: struct
  • 定義箇所: modules/generators/mod_suexec.h:28

目的

Apache HTTP Server において、CGI プログラム等を実行する際のユーザー権限（UID/GID）および suEXEC 機能の有効状態を保持するための設定構造体です。

シグネチャまたは形状

typedef struct {
    ap_unix_identity_t ugid;
    int active;
} suexec_config_t;

入力と出力

  • **入力**: mod_suexec.c 内の設定ディレクティブ処理を通じて、ユーザーおよびグループの識別情報が ugid に格納されます。
  • **出力**: CGI 実行系モジュールがリクエストコンテキストから本構造体を取得し、プロセス生成時の権限設定として利用します。

前提条件と不変条件

  • unixd.h で定義される ap_unix_identity_t 型に依存します。
  • suEXEC 機能の利用にあたっては、本構造体の値だけでなく、サーバー全体の設定である ap_unixd_config.suexec_enabled の確認が必須です。

アルゴリズムまたは振る舞い

  • mod_suexec.c 内の set_suexec_ugid 関数によって設定値が更新されます。
  • get_suexec_id_doer 関数がリクエストコンテキストから本構造体を参照し、CGI プロセス実行時の権限を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: mod_suexec.c 内の各関数（設定ディレクティブのハンドラや、CGI 実行時のフック関数）が本構造体をメモリ確保または参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ap_unix_identity_t: ugid フィールドの型。Unix 系のユーザー/グループ識別情報を保持します。

失敗モードと境界条件

  • 構造体内の active フラグが適切に設定されていない場合、suEXEC による権限分離が意図通りに機能しない可能性があります。
  • メモリ確保失敗時（apr_palloc 等）の挙動については、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本構造体は mod_suexec モジュールのコントラクトです。直接操作せず、必ず mod_suexec.c が提供するアクセサやフック経由でアクセスしてください。
  • セキュリティに関わる重要なデータ構造であるため、変更時は mod_suexec.c および unixd.h への影響を十分に考慮してください。
