parent_info_t

定義

  • 種別: struct
  • 定義箇所: server/mpm/mpmt_os2/mpmt_os2.c:100

目的

OS/2 プラットフォームにおいて、親プロセスから子プロセスへリスナーソケット情報と同期用ミューテックスを共有メモリ経由で受け渡すためのデータ構造です。

シグネチャまたは形状

typedef struct {
    HMTX accept_mutex;
    listen_socket_t listeners[1];
} parent_info_t;

入力と出力

  • **入力**: 親プロセスが共有メモリを確保する際、リスナー数に応じて動的にサイズを計算し、本構造体を配置します。
  • **出力**: 子プロセスが共有メモリから本構造体を読み取り、リスナーソケットの取得およびミューテックスによる排他制御を行います。

前提条件と不変条件

  • 親プロセスによって共有メモリ上に確保される必要があります。
  • listeners 配列は、実際のリスナー数に応じてメモリ領域が拡張されることを前提としています（sizeof(parent_info_t) + num_listeners * sizeof(listen_socket_t) で確保）。

アルゴリズムまたは振る舞い

1. 親プロセスが DosAllocSharedMem を使用して本構造体をメモリ上に確保します。 2. 親プロセスがリスナーソケット情報を listeners 配列に格納します。 3. 子プロセスが DosGetNamedSharedMem を通じてこのメモリ領域にアクセスし、共有されたソケットとミューテックスを取得します。 4. 子プロセス終了後、共有メモリは解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • server/mpm/mpmt_os2/mpmt_os2.c:114 (変数宣言)
  • server/mpm/mpmt_os2/mpmt_os2.c:196 (変数宣言)
  • server/mpm/mpmt_os2/mpmt_os2.c:217 (メモリ確保時のサイズ計算)
  • **呼び出し先**: 構造体内部の listen_socket_t および OS/2 API の HMTX 型（ミューテックスハンドル）。

関連データ構造

  • listen_socket_t: 本構造体に含まれる、リスナーソケット情報を保持する構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、プロセス間でのソケット受け渡しが不可能となり、子プロセスの起動が失敗します。
  • listeners[1] は固定長ですが、実際にはメモリ確保時に後続の領域を含めてアクセスされるため、確保サイズと実際のリスナー数の整合性が取れていない場合、メモリ破壊が発生します。

保守担当者向けメモ

  • 本構造体は OS/2 の共有メモリモデルに強く依存しています。
  • listeners 配列のサイズは動的であるため、構造体定義の [1] はあくまでプレースホルダーとして扱われています。メモリ確保ロジックを変更する際は、sizeof 計算箇所との整合性に注意してください。
