regen_key

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_cache_socache.c:279

目的

Vary ヘッダーに基づき、リクエストヘッダーの内容を組み込むことで、キャッシュの識別に使用する一意なキーを再生成する。

シグネチャまたは形状

const char* regen_key(apr_pool_t *p, apr_table_t *headers,
                      apr_array_header_t *varray, const char *oldkey,
                      apr_size_t *newkeylen);

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用するプール。
  • headers: リクエストヘッダーのテーブル。
  • varray: Vary ヘッダーに含まれるヘッダー名の配列。
  • oldkey: 元となるベースキー。
  • newkeylen: 生成されたキーの長さを格納するポインタ。
  • **出力**:
  • 生成された新しいキー文字列を返す。

前提条件と不変条件

  • varray に含まれる各ヘッダー名に対応する値が headers テーブルから取得される。
  • ヘッダーが存在しない場合は空文字列として扱われる。

アルゴリズムまたは振る舞い

1. varray の要素数から必要な iovec の数を計算し、プールからメモリを確保する。 2. varray に列挙された各ヘッダー名と、そのヘッダー値を headers テーブルから検索し、ペアとして iovec に格納する。 3. 最後に oldkey を iovec に追加する。 4. apr_pstrcatv を使用して、すべての要素を連結した新しいキー文字列を生成し、その長さを newkeylen に設定して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • modules/cache/mod_cache_disk.c (行477, 1047)
  • modules/cache/mod_cache_socache.c (行546, 876)
  • **呼び出し先**:
  • apr_palloc (APR)
  • apr_table_get (APR)
  • strlen (標準ライブラリ)
  • apr_pstrcatv (APR)

関連データ構造

  • apr_table_t: ヘッダーの保持に使用。
  • apr_array_header_t: Vary ヘッダーのリスト保持に使用。
  • struct iovec: 連結対象の文字列データと長さを管理するために使用。

失敗モードと境界条件

  • headers 内に varray で指定されたヘッダーが存在しない場合、空文字列として処理される。
  • Vary ヘッダー値の比較において、大文字小文字の区別や複数値を持つヘッダーの扱いに関する制限がソースコード内のコメントで指摘されている。

保守担当者向けメモ

  • Vary ヘッダーの仕様（RFC 2616）に基づき、特定のヘッダーを大文字小文字区別あり/なしで扱うかの判断が必要な場合がある。
