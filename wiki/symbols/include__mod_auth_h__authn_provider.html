authn_provider

定義

  • 種別: struct
  • 定義箇所: include/mod_auth.h:80

目的

Apache HTTP Server における認証プロバイダーモジュールが実装すべきインターフェースを定義し、コアサーバーと認証モジュール間の疎結合な連携を実現する。

シグネチャまたは形状

typedef struct {
    authn_status (*check_password)(request_rec *r, const char *user, const char *password);
    authn_status (*get_realm_hash)(request_rec *r, const char *user, const char *realm, char **rethash);
} authn_provider;

入力と出力

  • check_password:
  • 入力: request_rec *r (リクエスト情報), user (ユーザー名), password (パスワード)
  • 出力: authn_status (認証結果)
  • get_realm_hash:
  • 入力: request_rec *r, user, realm (レルム名)
  • 出力: authn_status (結果), char **rethash (ハッシュ値の格納先)

前提条件と不変条件

  • 各関数ポインタは、認証の成否に応じて authn_status 型の値を返す必要がある。
  • get_realm_hash は、成功時に user:realm:password の MD5 ハッシュを rethash に格納することが期待される。

アルゴリズムまたは振る舞い

1. サーバーはリクエスト処理中に、登録されたプロバイダーの check_password を呼び出し、パスワード検証を行う。 2. ダイジェスト認証等のレルムベースの認証では、get_realm_hash を呼び出し、保存されたハッシュ値を取得して検証を行う。

呼び出し元と呼び出し先

  • 呼び出し元: mod_authn_core.c などの認証コアモジュールが、登録されたプロバイダーを介して本構造体の関数を呼び出す。
  • 呼び出し先: mod_auth_basic.c, mod_auth_digest.c, mod_auth_form.c, mod_authn_anon.c 等の各認証プロバイダーモジュールが、この構造体のインスタンスを定義・提供する。

関連データ構造

  • authn_status: 認証処理の結果を示す列挙型。
  • request_rec: HTTP リクエスト情報を保持するコア構造体。

失敗モードと境界条件

  • 各関数ポインタが NULL の場合、呼び出し側で適切にハンドリングする必要がある。
  • get_realm_hash においてハッシュの生成や取得に失敗した場合、適切な authn_status を返す必要がある。

保守担当者向けメモ

  • 本構造体は include/mod_auth.h で定義されており、新しい認証方式を追加する際は、この構造体を実装したプロバイダーを ap_register_provider で登録する。
  • 過去に has_realm_hash が存在したが削除された経緯があるため、API の変更履歴（ap_mmn.h）に注意すること。
