http2_output_consumed

定義

  • 種別: typedef
  • 定義箇所: modules/http2/mod_http2.h:73

目的

HTTP/2 接続において、出力データが消費されたことを通知するためのコールバック関数の型定義です。

シグネチャまたは形状

typedef void http2_output_consumed(void *ctx, conn_rec *c, apr_off_t consumed);

入力と出力

  • void *ctx: コールバック実行時に使用されるコンテキストポインタ。
  • conn_rec *c: 関連する Apache 接続構造体。
  • apr_off_t consumed: 消費されたデータのバイト数。
  • 戻り値: なし（void）。

前提条件と不変条件

  • 本シンボルは mod_http2 のオプション関数インターフェースの一部として定義されています。
  • http2_req_engine_init 関数において、この型のポインタへのポインタ（http2_output_consumed **pconsumed）が引数として渡されます。

アルゴリズムまたは振る舞い

  • データの消費（送信完了やバッファからの解放など）が発生した際に、登録された関数が呼び出されることを想定したインターフェースです。
  • 具体的な実行ロジックは、この関数ポインタを保持するモジュール側の実装に依存します。

呼び出し元と呼び出し先

  • 呼び出し元: modules/http2/mod_http2.h 内の http2_req_engine_init 関数定義において、引数として参照されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な呼び出し先（実装側）の特定はできません。

関連データ構造

  • conn_rec: Apache HTTP Server の接続状態を保持する構造体。
  • apr_off_t: APR（Apache Portable Runtime）で定義されるオフセット型。
  • http2_req_engine_init: 本型を引数として受け取る初期化関数。

失敗モードと境界条件

  • オプション関数として提供されるため、mod_http2 がロードされていない場合、関連する関数ポインタが NULL になる可能性があります。利用時には必ずチェックが必要です。

保守担当者向けメモ

  • 本シンボルは mod_http2 のパブリック API の一部です。変更を行う際は、他のモジュールとの互換性に注意してください。
  • 実装の詳細は modules/http2/mod_http2.c を参照してください。
