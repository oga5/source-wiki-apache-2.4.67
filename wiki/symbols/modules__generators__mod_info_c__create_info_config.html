create_info_config

定義

  • 種別: function
  • 定義箇所: modules/generators/mod_info.c:87

目的

mod_info モジュールで使用するサーバー設定構造体（info_svr_conf）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void *create_info_config(apr_pool_t *p, server_rec *s)

入力と出力

  • **入力**:
  • p: メモリ割り当てに使用する apr_pool_t プール。
  • s: 設定対象の server_rec 構造体（本関数内では未使用）。
  • **出力**:
  • 初期化された info_svr_conf 構造体へのポインタ。

前提条件と不変条件

  • p は有効なメモリプールであること。
  • 割り当てられた info_svr_conf の more_info メンバは、初期容量 20 の info_entry 配列として確保される。

アルゴリズムまたは振る舞い

1. apr_pcalloc を使用して、info_svr_conf サイズのメモリ領域をゼロクリアして確保する。 2. apr_array_make を呼び出し、more_info 配列を初期化する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/generators/mod_info.c:1014（サーバー設定の初期化プロセスにおいて呼び出される）。
  • **呼び出し先**:
  • apr_pcalloc (APR)
  • apr_array_make (APR)

関連データ構造

  • info_svr_conf: 本関数が初期化するモジュール固有の設定構造体。
  • info_entry: more_info 配列の要素として格納されるデータ型。

失敗モードと境界条件

  • メモリ不足時には apr_pcalloc または apr_array_make がアボートする可能性がある（APR の標準的な挙動）。

保守担当者向けメモ

  • 本関数は mod_info の設定ライフサイクルの一部であり、リクエスト処理とは独立してサーバー起動時に実行される。
  • more_info 配列の初期容量（20）は、AddModuleInfo ディレクティブの一般的な使用量に基づいている。
