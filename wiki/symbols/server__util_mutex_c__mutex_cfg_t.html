mutex_cfg_t

定義

  • 種別: struct
  • 定義箇所: server/util_mutex.c:129

目的

Apache HTTP Server におけるミューテックス（排他制御）の動作設定（メカニズム、ディレクトリ、オプション）を保持し、管理するための内部構造体です。

シグネチャまたは形状

typedef struct {
    apr_int32_t options;    /* ミューテックスのオプションフラグ */
    int set;                /* 設定が明示的に指定されたかを示すフラグ */
    int none;               /* ミューテックスを無効化するかを示すフラグ */
    int omit_pid;           /* ファイル名にPIDを含めないかを示すフラグ */
    apr_lockmech_e mech;    /* 使用するロックメカニズム (APR定義) */
    const char *dir;        /* ロックファイルを作成するディレクトリパス */
} mutex_cfg_t;

入力と出力

本構造体は直接的な関数ではありませんが、ap_set_mutex 等のディレクティブハンドラによって設定値が入力され、mxcfg_lookup 等の関数を通じて解決された設定が出力（参照）されます。

前提条件と不変条件

  • mxcfg_by_type ハッシュテーブル内で管理され、サーバーの起動期間中（pconf プール）有効です。
  • mxcfg_lookup により、特定のミューテックス設定 > デフォルト設定 > 登録時のデフォルト値 の優先順位で解決されます。

アルゴリズムまたは振る舞い

  • flock や fcntl を使用する場合、get_mutex_filename が本構造体の dir と omit_pid を参照してロックファイルのパスを生成します。
  • omit_pid が偽の場合、ファイル名に PID が付与され、プロセスごとの競合を回避する設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元**: server/util_mutex.c 内の ap_set_mutex（設定更新）、mxcfg_lookup（設定取得）、get_mutex_filename（パス生成）、ap_dump_mutexes（デバッグ出力）などから参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • apr_lockmech_e: APR が提供するロックメカニズムの列挙型。
  • mxcfg_by_type: server/util_mutex.c 内で定義される、型名と mutex_cfg_t を紐付けるハッシュテーブル。

失敗モードと境界条件

  • mxcfg_lookup で該当する型が見つからない場合、デフォルト設定が適用されます。
  • dir が指定されていない場合、APR のデフォルトのロックディレクトリが使用されます。

保守担当者向けメモ

  • 本構造体はミューテックスの「設定」を保持するものであり、実際のロック取得・解放ロジックは APR (apr_proc_mutex_lock 等) に委譲されています。
  • メモリ管理は pconf プールに依存しているため、設定のライフサイクルはサーバーの起動期間と一致します。
