isapi_global_conf

定義

  • 種別: struct
  • 定義箇所: modules/arch/win32/mod_isapi.c:199

目的

Apache HTTP Server 上でロードされた ISAPI モジュール（DLL）のキャッシュ管理および排他制御を行うためのグローバルな設定構造体です。

シグネチャまたは形状

struct isapi_global_conf {
    apr_pool_t         *pool;
    apr_thread_mutex_t *lock;
    apr_hash_t         *hash;
};

入力と出力

  • 入力: 本構造体はグローバル変数 loaded として静的に確保されます。
  • 出力: 保持するハッシュテーブルを通じて、ロード済み DLL のハンドルや関数ポインタへのアクセスを提供します。

前提条件と不変条件

  • pool: モジュールのライフサイクル全体を管理するメモリプール。
  • lock: 複数スレッドからの同時アクセスによるハッシュテーブルの競合を防ぐためのミューテックス。
  • hash: ロードされた ISAPI モジュールを管理するためのハッシュテーブル。

アルゴリズムまたは振る舞い

  • isapi_pre_config 時に初期化され、グローバルなキャッシュプールとハッシュテーブルを構築します。
  • isapi_lookup および isapi_load 処理において、この構造体の lock を用いて排他制御を行いながら、DLL の検索やロード状態の管理を行います。

呼び出し元と呼び出し先

  • 呼び出し元: isapi_pre_config（初期化時）、isapi_lookup（DLL 検索時）、isapi_load（DLL ロード時）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • apr_pool_t: メモリ管理用。
  • apr_thread_mutex_t: スレッドセーフなアクセス制御用。
  • apr_hash_t: isapi_loaded 構造体を格納するためのハッシュテーブル。

失敗モードと境界条件

  • ミューテックスの初期化失敗やメモリ確保失敗が発生した場合、モジュールの初期化プロセスが停止する可能性があります。
  • ハッシュテーブルへのアクセス時にロックが適切に取得されない場合、マルチスレッド環境下でデータ破損のリスクがあります。

保守担当者向けメモ

  • 本構造体は loaded という名前で静的に定義されています。
  • isapi_loaded 構造体と組み合わせて使用され、DLL のロード状態を追跡します。
  • 非同期エミュレーション (fake_async) を使用する場合、このグローバルな管理構造体を通じたリクエスト処理の同期が重要となります。
