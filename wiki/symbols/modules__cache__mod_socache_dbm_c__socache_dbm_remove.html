socache_dbm_remove

定義

  • 種別: function
  • 定義箇所: modules/cache/mod_socache_dbm.c:392

目的

DBM ファイルから特定のキーに対応するキャッシュエントリを削除します。

シグネチャまたは形状

static apr_status_t socache_dbm_remove(ap_socache_instance_t *ctx,
                                       server_rec *s, const unsigned char *id,
                                       unsigned int idlen, apr_pool_t *p)

入力と出力

  • **入力**:
  • ctx: キャッシュインスタンスのコンテキスト（DBM ファイルパス等を含む）。
  • s: サーバーレコード（エラーログ出力用）。
  • id: 削除対象のキーデータ。
  • idlen: キーデータの長さ。
  • p: メモリプール。
  • **出力**:
  • apr_status_t: 操作の成功（APR_SUCCESS）または失敗時のエラーコード。

前提条件と不変条件

  • ctx->pool は、操作のたびに apr_pool_clear によってリセットされます。
  • DBM ファイルは APR_DBM_RWCREATE モードでオープン可能である必要があります。
  • 呼び出し側で適切な排他制御が行われていることが前提です（本モジュールはスレッドセーフではありません）。

アルゴリズムまたは振る舞い

1. 指定された id と idlen を apr_datum_t 構造体に格納し、DBM キーを作成します。 2. ctx->pool をクリアします。 3. APR のバージョンに応じて apr_dbm_get_driver または apr_dbm_open を使用し、対象の DBM ファイルを書き込みモードで開きます。 4. apr_dbm_delete を呼び出し、該当するキーを削除します。 5. apr_dbm_close で DBM ファイルを閉じ、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • 同一ファイル内の socache_dbm_retrieve（期限切れエントリの削除時）。
  • 同一ファイル内の socache_dbm_expire（期限切れエントリの削除時）。
  • ap_socache_provider_t 構造体を通じて、mod_cache_socache 等の外部モジュールから参照される可能性があります。
  • **呼び出し先**:
  • apr_dbm_get_driver / apr_dbm_open / apr_dbm_open2
  • apr_dbm_delete
  • apr_dbm_close
  • ap_log_error

関連データ構造

  • ap_socache_instance_t: キャッシュインスタンスの管理構造体。
  • apr_dbm_t: DBM ファイルのハンドル。
  • apr_datum_t: DBM のキーおよび値を保持するデータ構造。

失敗モードと境界条件

  • **ファイルオープン失敗**: DBM ファイルが開けない場合、エラーログを出力し、その時点でのエラーコードを返します。
  • **キー不在**: apr_dbm_delete はキーが存在しない場合でもエラーを返さない仕様（APR の DBM 実装に依存）ですが、関数全体としては成功を返します。

保守担当者向けメモ

  • 本関数は AP_SOCACHE_FLAG_NOTMPSAFE を指定しているモジュールの一部であるため、呼び出し側での排他制御が必須です。
  • APR 1.7 以上の環境では apr_dbm_get_driver を使用するパスが優先されます。
