lua_db_close

定義

  • 種別: function
  • 定義箇所: modules/lua/lua_dbd.c:59

目的

Lua スクリプトから利用されているデータベース接続を明示的に終了し、関連するリソースを解放します。

シグネチャまたは形状

int lua_db_close(lua_State *L)

入力と出力

  • **入力**: Lua ステート L（スタックから lua_db_handle を取得）。
  • **出力**: スタックのトップに apr_status_t 型の戻り値（成功時は 0、失敗時はエラーコード）を数値としてプッシュし、1 を返します。

前提条件と不変条件

  • lua_get_db_handle(L) により、有効な lua_db_handle が取得できる必要があります。
  • 接続が既に閉じられている場合（db->alive が 0）、関数は何も行わず 0 を返します。

アルゴリズムまたは振る舞い

1. lua_get_db_handle を呼び出し、現在のデータベースハンドルを取得します。 2. 接続が生存している場合（db->alive が真）:

  • db->type が LUA_DBTYPE_APR_DBD の場合、apr_dbd_close を実行し、関連する apr_pool_t を破棄します。
  • それ以外の場合、オプション関数 ap_dbd_close を取得して実行します。

    • 3. 接続状態をリセットします（driver, handle, pool を NULL に設定し、alive を 0 に変更）。 4. スタックをクリアし、終了ステータスをプッシュして呼び出し元へ戻ります。

呼び出し元と呼び出し先

  • **呼び出し元**: modules/lua/lua_dbd.c:690 にて lua_pushcfunction を介して登録されています。
  • **呼び出し先**: lua_get_db_handle, apr_dbd_close, apr_pool_destroy, APR_RETRIEVE_OPTIONAL_FN(ap_dbd_close)。

関連データ構造

  • lua_db_handle: データベース接続状態、ドライバ、ハンドル、メモリプールを保持する構造体。

失敗モードと境界条件

  • db が NULL の場合、または db->alive が偽の場合は何も実行されません。
  • APR_RETRIEVE_OPTIONAL_FN が NULL を返す場合、外部 DBD クローズ処理はスキップされます。

保守担当者向けメモ

  • 本関数は lua_db_gc と同様のクリーンアップロジックを共有していますが、明示的なクローズを目的としています。
  • apr_pool_destroy を呼び出すため、この関数実行後に当該ハンドルを再利用することはできません。
