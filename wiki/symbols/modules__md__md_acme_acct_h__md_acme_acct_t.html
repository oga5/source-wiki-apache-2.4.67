md_acme_acct_t

定義

  • 種別: struct
  • 定義箇所: modules/md/md_acme_acct.h:29

目的

ACME プロトコルにおけるアカウント情報を保持するためのデータ構造です。サーバーとの通信に必要な認証情報、連絡先、ステータス、および外部アカウントバインディング（EAB）情報を管理します。

シグネチャまたは形状

typedef struct md_acme_acct_t md_acme_acct_t;

※不完全型として定義されており、具体的なメンバ構成はソース抜粋からは確認できません。

入力と出力

  • **入力**: md_acme_acct_from_json を介して JSON 形式からインスタンス化されます。
  • **出力**: md_acme_acct_to_json を介して JSON 形式にシリアライズされます。

前提条件と不変条件

  • アカウントに関連付けられる秘密鍵は、通常 RSA 3072 ビットが推奨されます。
  • 永続化には md_store.h で定義されるストレージインターフェースが利用されます。

アルゴリズムまたは振る舞い

  • **ライフサイクル管理**: md_acme_acct_register による新規登録、md_acme_acct_validate によるサーバーとの状態同期、および md_acme_acct_load によるストレージからの読み込みをサポートします。
  • **同意管理**: md_acme_check_agreement を通じて、サーバーが要求する利用規約（ToS）への同意状況を検証します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • modules/md/md_acme.c: ACME セッション内でのアカウント保持。
  • modules/md/md_acme.h: ACME 制御構造体 md_acme_t 内での参照。
  • modules/md/md_acme_acct.c: アカウントの生成、JSON 変換、読み込み、URL 照合処理。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、具体的な関数呼び出し先は提供されたソース抜粋には含まれていません。

関連データ構造

  • md_acme_acct_st: アカウントの有効状態（valid, deactivated, revoked 等）を示す列挙型。
  • md_acme_t: ACME プロトコル全体の制御構造体。
  • md_pkey_t: アカウントに関連付けられる秘密鍵。

失敗モードと境界条件

  • アカウント情報のパース時、サーバー側の仕様変更により JSON 構造が予期しない形式である場合に失敗する可能性があります。
  • EAB を使用する場合、eab_kid や eab_hmac が正しく設定されていないと、サーバーとの認証に失敗します。

保守担当者向けメモ

  • 本構造体は md_acme_acct.c で実装される各関数によって操作されます。
  • 秘密鍵は account.pem としてファイルシステム上に保存されるため、ファイルパーミッションの管理に注意してください。
