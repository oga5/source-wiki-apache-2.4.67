support/htcacheclean.c

概要

Apache HTTP Server のディスクキャッシュ（mod_cache_disk）を管理・整理するためのスタンドアロンユーティリティです。キャッシュサイズの制限（バイト数および inode 数）に基づき、古いエントリや期限切れのエントリを削除し、ディスク領域を解放します。

モジュール内での役割

support モジュールの一部として、サーバー実行中または停止中にキャッシュディレクトリをクリーンアップする運用ツールを提供します。mod_cache_disk が生成するファイル構造（ヘッダーファイルとデータファイルのペア）を理解し、整合性を保ちながら削除を行います。

責務

  • **キャッシュの監視と削除**: 指定された制限値を超えたキャッシュエントリの特定と削除。
  • **ディレクトリ管理**: キャッシュディレクトリの再帰的な走査と、不要になった空ディレクトリの削除。
  • **デーモンモード**: 指定間隔での定期的なクリーンアップ実行。
  • **URL 個別削除**: コマンドライン引数で指定された特定の URL キャッシュの削除。
  • **統計情報の出力**: クリーンアップ結果のサマリー表示。

重要なシンボル

データ構造

  • DIRENTRY: ディレクトリ走査中に一時的に保持されるファイル情報（ヘッダー/データファイルのペア管理用）。
  • ENTRY: 最終的に削除対象を決定するために使用されるキャッシュエントリのリスト構造。
  • stats: クリーンアップ処理中の統計情報（サイズ、inode 数、削除数など）を保持。

制御フラグ・状態

  • interrupted: シグナル（SIGINT/SIGTERM）による中断フラグ。
  • realclean: Apache が停止していると仮定して徹底的にクリーンアップを行うフラグ。
  • dryrun: 削除を行わず、削除対象のシミュレーションのみを行うフラグ。

主要関数

  • process_dir(): キャッシュディレクトリを再帰的に走査し、ENTRY リストを構築する。
  • purge(): ENTRY リストに基づき、制限値に収まるよう古いエントリから順に削除を実行する。
  • delete_entry(): ヘッダーファイル、データファイル、および関連するディレクトリを削除する。
  • list_urls(): キャッシュ内の URL 一覧を表示する。

制御フローとデータフロー

1. **初期化**: 引数を解析し、動作モード（デーモン、単発、URL 削除）を決定。 2. **走査**: process_dir() がキャッシュディレクトリを再帰的に探索し、apr_hash_t を用いてヘッダーとデータのペアを特定。 3. **リスト構築**: 有効なキャッシュエントリを ENTRY リング構造に格納。 4. **削除判定**: purge() が合計サイズと inode 数を計算し、制限を超過している場合に「期限切れ」「古い順」の優先度で delete_entry() を呼び出す。 5. **クリーンアップ**: delete_entry() はファイル削除後、必要に応じて delete_parent() を呼び出し、空になったディレクトリを再帰的に削除する。

読者向けメモ

  • **メモリ管理**: apr_pool_t を多用しています。特に process_dir や delete_file 内では、メモリリークを防ぐために一時的なサブプールを作成・破棄するパターンが徹底されています。
  • **競合回避**: Apache との同時実行を考慮し、apr_stat のリトライ処理や、ファイルが存在しない場合のハンドリングが随所に含まれています。
  • **制約**: NICE_DELAY や DELETE_NICE 定数により、システム負荷を抑えるためのスロットリング機能が実装されています。
  • **省略箇所**: 本ファイルは非常に長大ですが、主に「ファイルシステム操作の抽象化」と「キャッシュフォーマットのパース」に注力しています。エラーハンドリングの詳細は、APR のステータスコードチェックに依存しています。
