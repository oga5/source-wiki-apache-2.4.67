support/suexec.h

概要

suexec.h は、Apache HTTP Server の suEXEC ラッパープログラムの動作を規定する設定定数を定義するヘッダーファイルです。このファイルは、セキュリティ上の境界条件をコンパイル時に決定するために使用されます。

モジュール内での役割

support モジュールにおいて、suexec.c が実行時に参照するセキュリティポリシーの「設定ファイル」として機能します。Apache のビルドプロセスにおいて、システム環境に応じた安全なデフォルト値を定義する役割を担います。

責務

  • suEXEC が実行を許可する Apache ユーザー（AP_HTTPD_USER）の定義。
  • 実行対象となるユーザーおよびグループの UID/GID 下限値の制限。
  • CGI スクリプトの実行を許可するディレクトリ構造（DOC_ROOT、USERDIR_SUFFIX）の制約。
  • 実行環境に渡す安全な環境変数パス（PATH）の定義。

重要なシンボル

  • AP_HTTPD_USER: suEXEC を呼び出し可能な唯一のユーザー名。
  • AP_UID_MIN / AP_GID_MIN: 権限昇格を許可する最小の UID/GID。これより小さい値はシステムアカウントとみなされ、保護されます。
  • AP_DOC_ROOT: suEXEC が CGI 実行を許可するルートディレクトリ。
  • AP_SAFE_PATH: CGI プログラムに渡される安全な PATH 環境変数。

制御フローとデータフロー

本ファイルは純粋な定数定義ヘッダーであるため、直接的な制御フローは持ちません。suexec.c がコンパイル時にこれらをインクルードすることで、実行バイナリにハードコードされたセキュリティチェックロジックが生成されます。

直接依存

  • ap_config_layout.h: Apache のインストールディレクトリレイアウト（DEFAULT_EXP_HTDOCSDIR 等）を取得するために依存しています。

関連コード

  • support/suexec.c: 本ヘッダーで定義された定数を使用して、実際の権限チェックやパス検証を行う実装ファイルです。

読者向けメモ

  • **セキュリティ上の注意**: このファイルの設定を変更することは、サーバーのセキュリティモデルに直接影響します。特に AP_UID_MIN や AP_DOC_ROOT の設定を誤ると、意図しない権限昇格や不正なファイル実行を許すリスクがあります。
  • **カスタマイズ**: 多くの値は configure スクリプトのオプションを通じて上書き可能です。直接このファイルを編集する前に、ビルド時の設定オプションを確認することを強く推奨します。
