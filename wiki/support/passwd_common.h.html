support/passwd_common.h

概要

support/passwd_common.h は、Apache HTTP Server のパスワード管理ユーティリティ（htpasswd, htdbm 等）で共通して利用されるデータ構造、定数、およびユーティリティ関数の宣言を提供します。パスワードのハッシュ化、入力取得、エラーハンドリングのインターフェースを標準化し、各ツール間での一貫性を確保します。

モジュール内での役割

support モジュール内のパスワード関連ツールにおける「共有ライブラリ的役割」を担います。個別のツールが直接実装すべきパスワード処理ロジックを抽象化し、passwd_common.c と対になって実装を提供します。

責務

  • **定数定義**: サポートするハッシュアルゴリズム（MD5, SHA, BCRYPT等）およびエラーコードの定義。
  • **コンテキスト管理**: passwd_ctx 構造体を通じた、パスワード生成に必要な設定（アルゴリズム、コスト、入力元）の保持。
  • **共通ユーティリティ**: メモリ不足時のハンドリング、ファイル書き込み、パスワード入力取得の抽象化。

重要なシンボル

データ構造

  • struct passwd_ctx: パスワード生成プロセス全体の状態を保持するコンテキスト。メモリプール、入力元（passwd_src）、アルゴリズム設定、エラーメッセージを管理します。

アルゴリズム定数

  • ALG_PLAIN, ALG_CRYPT, ALG_APMD5, ALG_APSHA, ALG_BCRYPT, ALG_CRYPT_SHA256, ALG_CRYPT_SHA512: サポートされるハッシュアルゴリズムの識別子。

エラーコード

  • ERR_*: ERR_FILEPERM, ERR_SYNTAX, ERR_PWMISMATCH 等、パスワード処理中に発生しうる共通エラーの列挙。

ユーティリティ関数

  • abort_on_oom(): メモリ確保失敗時の強制終了ハンドラ。
  • putline(): ファイルへの書き込みとエラー時の終了処理をラップ。
  • get_password(): passwd_ctx に基づき、標準入力やプロンプトからパスワードを取得。
  • mkhash(): 指定されたアルゴリズムでパスワードハッシュを生成。

制御フローとデータフロー

1. **初期化**: ツール起動時に errfile を apr_file_open_stderr() で初期化する必要があります。 2. **コンテキスト設定**: passwd_ctx を初期化し、parse_common_options() でアルゴリズムやコストを設定します。 3. **入力取得**: get_password() が呼び出され、passwd_src に応じてユーザー入力を ctx->passwd に格納します。 4. **ハッシュ生成**: mkhash() が ctx 内の情報を元にハッシュを計算し、ctx->out に結果を格納します。 5. **エラー処理**: 各関数は成功時に 0 を返し、失敗時には ERR_* コードを返すと同時に ctx->errstr に詳細をセットします。

直接依存

  • apr.h, apr_lib.h, apr_strings.h: APR (Apache Portable Runtime) の基本機能。
  • apr_file_io.h: ファイル操作および標準入出力の抽象化。
  • apr_version.h, apu_version.h: バージョン依存の機能切り替え（特にBCRYPTのサポート判定）。
  • ap_config_auto.h
  • apr_errno.h
  • apr_general.h

関連コード

  • support/passwd_common.c: 上記ヘッダーで宣言された関数の実装。
  • support/htpasswd.c: 本ヘッダーを利用する主要なクライアントツール。

読者向けメモ

  • **メモリ管理**: passwd_ctx 内のメモリは apr_pool_t に依存します。abort_on_oom が登録されているため、メモリ不足時はプロセスが即座に終了します。
  • **プラットフォーム依存**: CRYPT_ALGO_SUPPORTED 等のフラグにより、OSごとの機能差分を吸収しています。
  • **省略された詳細**: mkhash の内部的なハッシュ計算ロジック（apr_md5 や crypt の呼び出し）は passwd_common.c に隠蔽されており、本ヘッダーではインターフェースのみが定義されています。
  • **注意**: errfile はグローバル変数として定義されていますが、使用前に必ず初期化が必要です。初期化を忘れると putline 等でセグメンテーション違反が発生します。
