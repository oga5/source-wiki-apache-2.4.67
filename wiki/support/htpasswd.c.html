support/htpasswd.c

概要

Apache HTTP Server のパスワードファイル（htpasswd 形式）を管理するためのコマンドラインユーティリティです。ユーザーの追加、更新、削除、およびパスワードの検証を行います。

モジュール内での役割

support モジュールにおいて、認証用フラットファイルデータベースの操作を担当します。passwd_common.c で定義された共通のハッシュ生成ロジックやコンテキスト構造体を利用し、ファイルシステム上のパスワードファイルを安全に更新する役割を担います。

責務

  • **コマンドライン解析**: ユーザー入力の検証と操作モード（作成、削除、検証、表示）の決定。
  • **ファイル操作**: 一時ファイルを用いたアトミックなファイル更新処理。
  • **認証ロジック**: passwd_common.h を介したパスワードハッシュの生成と検証。
  • **エラーハンドリング**: ファイル権限、不正なユーザー名、バッファオーバーフロー等の検出と終了コードの返却。

重要なシンボル

操作フラグ (Masks)

  • APHTP_NEWFILE: 新規ファイル作成モード。
  • APHTP_NOFILE: ファイル更新を行わず、標準出力へ結果を表示。
  • APHTP_DELUSER: 指定ユーザーの削除。
  • APHTP_VERIFY: 指定ユーザーのパスワード検証。

主要関数

  • main(): プログラムのライフサイクル管理、引数解析、ファイル操作のオーケストレーション。
  • mkrecord(): ユーザー名とハッシュ化されたパスワードを結合し、ファイル書き込み用のレコードを生成。
  • check_args(): apr_getopt を用いた引数解析とバリデーション。
  • verify(): apr_password_validate を呼び出し、既存ハッシュと入力パスワードを照合。
  • accessible() / exists(): ファイルの存在確認およびアクセス権限チェック。

制御フローとデータフロー

1. **初期化**: apr_pool_t を作成し、passwd_ctx を初期化。 2. **引数解析**: check_args により操作モードを決定し、必要な引数（ファイル名、ユーザー名）を検証。 3. **ファイルチェック**: exists と accessible を用いて、対象ファイルの読み書き権限を確認。 4. **レコード生成**: mkrecord がハッシュを生成し、ctx->out に格納。 5. **ファイル更新**:

  • 一時ファイルを作成。
  • 既存ファイルを一行ずつ読み込み、対象ユーザー以外を一時ファイルへコピー。
  • 対象ユーザーが見つかった場合、操作（削除・更新・検証）を実行。
  • 最後に apr_file_copy で一時ファイルを元のファイルへ上書き。

直接依存

  • passwd_common.h: 共通のハッシュ関数やコンテキスト構造体。
  • apr_signal.h, apr_getopt.h: APR によるシグナル処理と引数解析。
  • apr_md5.h, apr_sha1.h: ハッシュアルゴリズムの実装。
  • 標準ライブラリ: stdio.h, stdlib.h, string.h, unistd.h 等。
  • conio.h

関連コード

  • support/passwd_common.c: mkhash や get_password など、本ファイルの核心となる認証ロジックが実装されています。
  • support/htdbm.c: DBM 形式のパスワードファイルを扱う類似のツール。

読者向けメモ

  • **メモリ管理**: すべての動的メモリは apr_pool_t によって管理されており、main 終了時に自動的に解放されます。
  • **アトミック性**: 一時ファイル (htpasswd.tmp.XXXXXX) を作成し、最後に apr_file_copy を行うことで、書き込み中のクラッシュによるファイル破損を防いでいます。
  • **セキュリティ警告**: ソースコード内のコメントにある通り、本プログラムは **setuid 実行を想定していません**。権限昇格の脆弱性につながるため、setuid ビットを立てないでください。
  • **制限事項**: mkrecord 内で MAX_STRING_LEN を超えるレコードは ERR_OVERFLOW として拒否されます。また、ユーザー名に : を含めることはできません。
