support/htdigest.c

概要

Apache HTTP Server 用のダイジェスト認証パスワードファイル（htdigest ファイル）を管理するためのコマンドラインユーティリティです。既存のユーザーのパスワード更新、新規ユーザーの追加、および新規パスワードファイルの作成機能を提供します。

モジュール内での役割

support モジュールの一部として、サーバー管理者が認証情報をオフラインで管理するためのツールを提供します。htpasswd と同様の目的を持ちますが、MD5 を用いたダイジェスト認証形式（user:realm:hash）に特化しています。

責務

  • **ファイル操作**: パスワードファイルの読み込み、一時ファイルへの書き出し、および最終的なファイル置換によるアトミックな更新。
  • **認証情報管理**: ユーザー名、レルム、パスワードの入力を受け取り、MD5 ハッシュを生成してファイルに保存。
  • **バリデーション**: 入力パスワードの一致確認および行の長さ制限の強制。

重要なシンボル

ライフサイクル・初期化

  • main: プログラムのエントリポイント。APR の初期化、シグナルハンドリング、引数解析、およびメインの処理ループを実行。
  • terminate: atexit で登録される終了処理。apr_terminate を呼び出しリソースを解放。
  • cleanup_tempfile_and_exit: エラー発生時に一時ファイルをクローズして終了するユーティリティ。

認証・ハッシュ処理

  • add_password: ユーザーからパスワードを読み取り、user:realm:password を MD5 でハッシュ化し、指定されたファイルストリームに書き込む。

ファイル・文字列操作

  • get_line: ファイルから 1 行を読み込む。
  • getword: 行から区切り文字（:）までの文字列を抽出する。
  • putline: ファイルストリームへ文字列を書き出す。

制御フローとデータフロー

1. **初期化**: apr_app_initialize で APR 環境を構築し、一時ディレクトリを特定。 2. **ファイル処理**:

  • 新規作成モード（-c）の場合、指定ファイルをオープンして add_password を実行。
  • 更新モードの場合、既存ファイルを読み込み、一時ファイル（apr_file_mktemp）へコピー。
  • 対象ユーザーが見つかれば一時ファイルに新しいハッシュを書き込み、見つからなければ末尾に追加。

    • 3. **完了**: apr_file_copy を使用して一時ファイルを元のファイル名に上書きし、更新を確定させる。

直接依存

  • **APR (Apache Portable Runtime)**: ファイル I/O (apr_file_io.h)、MD5 計算 (apr_md5.h)、メモリプール (apr_general.h)、文字列操作 (apr_strings.h) 等の基盤機能。
  • **標準ライブラリ**: stdlib.h (exit 等)、sys/types.h。
  • apr.h
  • apr_lib.h
  • apr_signal.h
  • apr_want.h
  • conio.h

関連コード

  • support/htpasswd.c: 基本認証用のパスワード管理ツール。設計思想が共通。
  • support/passwd_common.c: パスワード管理ツール間で共有される共通ロジック（将来的なリファクタリング対象）。

読者向けメモ

  • **メモリ管理**: すべてのメモリ割り当ては cntxt（APR プール）を介して行われます。明示的な free は不要ですが、プログラム終了時にプールが破棄されます。
  • **安全性**: ソースコード内の警告にある通り、**setuid 実行は非推奨**です。権限昇格の脆弱性を防ぐため、適切なファイルパーミッションで運用してください。
  • **制限事項**: MAX_STRING_LEN (256) および MAX_LINE_LEN (768) により、ユーザー名やレルムの長さに物理的な制限があります。
  • **未実装・浅い部分**: EBCDIC 環境への対応コードが含まれていますが、現代の一般的な環境では無視されます。また、ファイル更新時の排他制御（ロック）は明示的な apr_file_lock を使用しておらず、一時ファイル経由のコピーによる簡易的な実装となっています。
