support/httxt2dbm.c

概要

httxt2dbm.c は、Apache HTTP Server の RewriteMap 機能で使用されるテキスト形式のマップファイルを、高速な検索が可能な DBM（Database Manager）形式に変換するためのコマンドラインユーティリティです。

モジュール内での役割

support モジュール内のツール群の一つであり、サーバーの実行時ではなく、管理者が設定ファイルを準備する段階で使用されるオフラインツールです。

責務

  • テキストファイル（キーと値のペア）を読み込み、指定された DBM フォーマットでデータベースを生成する。
  • APR（Apache Portable Runtime）の DBM 抽象化レイヤーを利用し、環境に応じた複数の DBM バックエンド（SDBM, GDBM, DB, NDBM）をサポートする。
  • 標準入力からの読み込みをサポートし、パイプライン処理に適した設計を提供する。

重要なシンボル

  • to_dbm(): テキストファイルを一行ずつ解析し、apr_dbm_store を用いて DBM ファイルへ書き込むコアロジック。
  • main(): 引数の解析、DBM ファイルのオープン、および処理のライフサイクル管理を行う。
  • REWRITE_MAX_TXT_MAP_LINE: 1行あたりの最大文字数（デフォルト 1024）。

制御フローとデータフロー

1. **初期化**: apr_app_initialize で環境を整え、apr_getopt でコマンドライン引数（入力元、出力先、フォーマット）を解析する。 2. **ファイルオープン**: 入力ファイルを apr_file_open（または標準入力）、出力 DBM を apr_dbm_open_ex で開く。 3. **変換ループ**: to_dbm 関数がテキストファイルを走査し、空白文字でキーと値を分割。apr_pstrmemdup でメモリを確保しつつ、apr_dbm_store で DBM に格納する。 4. **終了処理**: apr_dbm_close でデータベースを閉じ、atexit に登録された apr_terminate によりリソースを解放する。

直接依存

  • apr.h / apr_dbm.h: DBM 操作およびポータブルなファイル操作の基盤。
  • apr_getopt.h: コマンドライン引数の解析。
  • apr_pools.h: メモリ管理（プールベース）。
  • stdlib.h: atexit による終了処理。

関連コード

  • mod_rewrite.c: 本ツールで生成された DBM ファイルを実際に読み込んで書き換えルールを適用する Apache モジュール。
  • apr_dbm.h: 本ツールが依存する DBM 抽象化インターフェースの定義。

読者向けメモ

  • **メモリ管理**: to_dbm 内では apr_pool_clear を使用して、ループごとのメモリ消費を抑制しています。大量のデータを変換する場合でもメモリリークの心配は少ない設計です。
  • **フォーマット指定**: -f オプションで指定可能なフォーマットは、ビルド時に APR-util がサポートしているものに依存します。利用可能なフォーマットは usage() 関数内のチェックロジックで確認可能です。
  • **エラーハンドリング**: DBM のオープン失敗時や変換エラー時には、APR のエラーコードを人間が読みやすい形式（%pm）で出力するようになっています。
