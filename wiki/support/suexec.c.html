support/suexec.c

概要

suexec.c は、Apache HTTP Server において CGI プログラムを特定のユーザー権限で安全に実行するためのラッパープログラムです。特権（root）で動作する親プロセスから呼び出され、環境変数のクリーンアップ、ユーザー/グループの検証、ファイルシステムのパーミッションチェックを経て、最終的に execv でターゲットプログラムへ制御を移します。

モジュール内での役割

support モジュールにおけるセキュリティの要であり、Web サーバーの実行ユーザー（AP_HTTPD_USER）と、CGI を実行するターゲットユーザーの間の仲介役を果たします。

責務

  • **環境の隔離**: clean_env() を通じて、安全な環境変数のみを保持し、攻撃の踏み台となる変数を排除する。
  • **権限の検証**: 実行元ユーザーが許可されたユーザーであるか、ターゲットユーザー/グループがシステム上で有効かを確認する。
  • **セキュリティポリシーの強制**: 実行対象のファイルおよびディレクトリが、他者による書き込み権限を持たないこと、および所有者がターゲットユーザーと一致することを厳格に検証する。
  • **監査ログ**: 実行成功および失敗の履歴を syslog または指定されたログファイルに記録する。

重要なシンボル

環境管理

  • clean_env(): environ をリセットし、safe_env_lst に定義されたホワイトリストに基づき環境変数を再構築する。
  • safe_env_lst: 許可される環境変数の接頭辞および完全一致リスト。

ログ出力

  • log_err() / log_no_err(): エラーおよび通常のトランザクションを記録するラッパー。
  • err_output(): syslog またはファイルへの書き込みを行う実体。

メイン処理

  • main(): 引数の解析、ユーザー権限の切り替え（setuid/setgid）、ファイルシステムの検証、execv によるプロセス置換を行う。

制御フローとデータフロー

1. **初期化**: clean_env() で環境変数を最小化。 2. **検証**: getuid() で実行元を確認し、getpwnam() / getgrnam() でターゲットの UID/GID を解決。 3. **権限降格**: setgid() / initgroups() / setuid() を順に呼び出し、ターゲットユーザーへ権限を移行。 4. **パス検証**: getcwd() と chdir() を駆使して、実行対象が AP_DOC_ROOT 内に存在し、かつシンボリックリンク攻撃を受けていないかを lstat() で確認。 5. **実行**: execv() を呼び出し、現在のプロセスイメージをターゲットプログラムで置換。

  • suexec.h: コンパイル時の定数（AP_HTTPD_USER, AP_UID_MIN 等）を定義。

読者向けメモ

  • **メモリ所有権**: strdup() で確保された文字列は、プログラム終了（execv）まで保持されます。calloc された環境変数バッファは environ に代入されます。
  • **セキュリティ不変条件**: 実行対象のディレクトリおよびファイルは、グループや他者による書き込み権限（S_IWGRP, S_IWOTH）があってはなりません。また、setuid/setgid ビットが立っているファイルは拒否されます。
  • **エッジケース**: ~userdir リクエスト時は、AP_USERDIR_SUFFIX を用いてホームディレクトリ配下を探索します。
  • **省略された詳細**: _OSD_POSIX（BS2000/OSD）固有の環境構築ロジックは、一般的な Unix 環境では使用されないため、詳細な解析からは除外しています。
  • **警告**: このコードはセキュリティの境界線であるため、修正には極めて慎重な検討が必要です。execv 後の失敗は exit(255) で終了します。
