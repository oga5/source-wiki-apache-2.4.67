support/passwd_common.c

概要

Apache HTTP Server のサポートツール（htpasswd 等）で共通して利用される、パスワードハッシュ生成および管理のためのユーティリティ関数群を提供します。

モジュール内での役割

support モジュール内のパスワード関連ツールにおいて、アルゴリズムの抽象化と入出力の標準化を担います。各ツールが個別に実装すべきパスワードのハッシュ化ロジックや、ユーザーからの入力取得処理を共通化し、一貫した動作を保証します。

責務

  • **パスワード入力管理**: 標準入力、引数、または対話型プロンプトからのパスワード取得と検証。
  • **ハッシュ生成**: MD5, SHA1, Crypt, BCRYPT, SHA-2 等の多様なアルゴリズムを用いたハッシュ値の生成。
  • **エラーハンドリング**: 共通のエラーコード体系に基づいたエラーメッセージの生成と管理。
  • **プラットフォーム抽象化**: OS 依存の crypt() サポート状況に応じたフォールバックや警告の出力。

重要なシンボル

  • struct passwd_ctx: パスワード生成のコンテキスト（アルゴリズム、コスト値、入力元、出力バッファ等）を保持する構造体。
  • mkhash(): passwd_ctx に基づき、指定されたアルゴリズムでパスワードをハッシュ化するメイン関数。
  • get_password(): ユーザーからパスワードを安全に取得・検証する関数。
  • parse_common_options(): コマンドライン引数を解析し、コンテキストを更新する関数。

制御フローとデータフロー

1. **初期化**: ツール側で passwd_ctx を初期化し、parse_common_options でアルゴリズムやコストを設定します。 2. **入力**: get_password が呼び出され、設定された passwd_src に応じてパスワードをメモリに読み込みます。 3. **ハッシュ化**: mkhash が呼び出され、選択されたアルゴリズム（switch 文）に従ってソルト生成とハッシュ計算を行い、結果を ctx->out に書き込みます。 4. **終了処理**: 処理完了後、機密情報を含むバッファを memset でゼロクリアし、メモリリークや情報漏洩を防ぎます。

直接依存

  • passwd_common.h: 共通定義と構造体宣言。
  • apr_md5.h, apr_sha1.h: APR 提供のハッシュ関数。
  • crypt.h (プラットフォーム依存): 標準的な crypt() 関数。
  • apr_strings.h, apr_errno.h: 文字列操作および APR エラーコード。
  • io.h
  • stdio.h
  • stdlib.h
  • string.h
  • time.h
  • unistd.h

関連コード

  • support/htpasswd.c: 本モジュールを直接利用する代表的なツール。
  • support/htdbm.c: データベース形式のパスワード管理ツール。
  • apr_password_get(): APR が提供する、エコーバックなしでパスワードを取得する標準関数。

読者向けメモ

  • 本ファイルは errfile グローバル変数に依存しています。使用前に必ず apr_file_open_stderr() で初期化してください。
  • mkhash 内の switch 文は、プラットフォームのサポート状況に応じて条件付きコンパイル（#if）されています。特定のアルゴリズムが利用できない環境では、警告やエラーが適切に返される設計になっています。
  • 機密情報（パスワード）を扱うため、エラー発生時や処理終了時には必ず memset によるゼロクリアが行われている点に注目してください。
