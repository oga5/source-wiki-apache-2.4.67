support/logresolve.c

概要

Apache HTTP Server のアクセスログに含まれる IP アドレスをホスト名に変換するコマンドラインユーティリティです。ログ処理のパフォーマンスを向上させるため、内部キャッシュを備えています。

モジュール内での役割

support モジュールの一部として、ログ解析の前処理を担います。HTTPD 自体で DNS 逆引きを行う負荷を回避し、オフラインでログを整形するために使用されます。

責務

  • 標準入力からログを読み込み、各行の先頭にある IP アドレスを抽出する。
  • apr_hash_t を用いたキャッシュ機構により、同一 IP の重複解決を抑制する。
  • オプションによる「ダブルルックアップ（正引き確認）」の実行。
  • 処理結果の統計情報の出力（オプション指定時）。

重要なシンボル

  • cache: IP アドレスとホスト名のマッピングを保持するハッシュテーブル。
  • main: プログラムのメインループ。ログの読み込み、解析、解決、書き出しを制御。
  • print_statistics: 処理終了後にキャッシュヒット率や解決失敗数などの統計を表示。
  • doublelookups: -c オプションにより有効化される、DNS の整合性チェックフラグ。

制御フローとデータフロー

1. **初期化**: apr_app_initialize で APR 環境を構築し、コマンドライン引数を解析。 2. **バッファ設定**: apr_file_buffer_set を使用し、入出力の効率化を図る。 3. **メインループ**:

  • 行単位で読み込み、先頭の IP アドレスを特定。
  • apr_hash_get でキャッシュを検索。ヒットすれば即座に置換して出力。
  • キャッシュミス時は apr_getnameinfo で逆引きを実行。
  • -c 指定時は apr_sockaddr_info_get で正引きを行い、IP の一致を確認。
  • 結果をキャッシュに保存し、ログ行を書き出す。

    • 4. **終了処理**: 統計ファイルが指定されていれば書き出し、リソースを解放。

直接依存

  • apr.h, apr_lib.h: APR の基本型とユーティリティ。
  • apr_hash.h: キャッシュ用ハッシュテーブル。
  • apr_getopt.h: コマンドライン引数解析。
  • apr_file_io.h, apr_network_io.h: ファイルおよびネットワーク（DNS）操作。
  • stdlib.h: 標準ライブラリ。
  • apr_strings.h

関連コード

  • support/rotatelogs.c: ログ管理に関連する他のユーティリティ。
  • apr_hash_t (APR): 本プログラムのパフォーマンスを支えるハッシュ実装。

読者向けメモ

  • 本プログラムは「ログの最初のフィールドが IP アドレスである」ことを前提としています。
  • pline プールは、ループ内で DNS 解決に使用される一時的なメモリを管理しており、apr_pool_clear で定期的に解放することでメモリ消費を抑えています。
  • 解決に失敗した場合も、その IP をキャッシュに登録することで、再度の無駄な DNS 問い合わせを防ぐ設計になっています。
