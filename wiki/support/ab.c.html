support/ab.c

概要

ab.c は Apache HTTP Server プロジェクトに含まれる標準的な HTTP ベンチマークツール「ApacheBench (ab)」のソースコードです。非同期 I/O と apr_pollset_t を活用し、単一プロセスで多数の同時接続をシミュレートしてサーバーの性能を測定します。

モジュール内での役割

support モジュールにおける主要なユーティリティの一つです。他のサポートツールが管理や設定に特化しているのに対し、本ファイルはサーバーの負荷試験と統計解析という動的な役割を担います。

責務

  • **負荷生成**: 指定された同時接続数（concurrency）に基づき、非同期に HTTP リクエストを送信する。
  • **統計計測**: 接続時間、処理時間、待機時間、転送量などのメトリクスを収集・計算する。
  • **SSL/TLS 対応**: OpenSSL を用いた HTTPS 通信のサポート。
  • **レポート出力**: テキスト形式または HTML 形式での結果出力、および GNUplot/CSV 用のデータエクスポート。

重要なシンボル

接続管理

  • struct connection: 各接続の状態、ソケット、SSL コンテキスト、タイムスタンプを保持する構造体。
  • connect_state_e: 接続のライフサイクル（UNCONNECTED, CONNECTING, CONNECTED, READ）を定義。
  • start_connect(): 非同期接続を開始し、apr_pollset_t に登録する。
  • close_connection(): 接続を終了し、統計を保存して次のリクエストへ遷移させる。

処理・通信

  • write_request(): HTTP リクエストをソケットへ書き込む。
  • read_connection(): サーバーからのレスポンスを読み込み、ヘッダー解析とボディの受信を行う。
  • ssl_proceed_handshake(): SSL/TLS ハンドシェイクを非同期に進行させる。

統計・出力

  • struct data: 個別のリクエストごとの時間計測データを保持。
  • output_results(): 収集した統計の集計、平均、標準偏差、パーセンタイル計算、および標準出力への表示。

制御フローとデータフロー

1. **初期化**: main() で引数を解析し、APR プールと apr_pollset_t を作成。 2. **接続開始**: test() が最初の接続を start_connect() で開始。 3. **イベントループ**: apr_pollset_poll() を使用して、ソケットの読み込み/書き込み可能イベントを待機。 4. **状態遷移**: イベントに応じて write_request()（送信）または read_connection()（受信）を呼び出し、connect_state_e を更新。 5. **完了処理**: 全リクエスト終了後、output_results() が stats 配列をソートし、統計情報を計算して出力。

読者向けメモ

  • **メモリ所有権**: apr_pool_t を使用してメモリを管理しており、cntxt プールが破棄される際に一括解放されます。
  • **レイアウト**: struct connection 配列は concurrency 分だけ動的に確保されます。
  • **既知の制限**: HTTP/1.x の解析は完全な仕様準拠ではなく、パフォーマンスを優先した簡易的な実装です。
  • **注意点**: ab は非常に古いコードベースを含んでおり、一部のバッファ操作に strcpy 等の古い関数が残っています。また、SSL 関連のコードは OpenSSL のバージョンによって条件分岐が複雑化しているため、ビルド環境の OpenSSL バージョンに注意してください。
