support/win32/ApacheMonitor.c

概要

ApacheMonitor.c は、Windows 上で Apache HTTP Server サービスを監視・管理するためのタスクトレイ常駐型 GUI アプリケーションです。サービスの状態（起動・停止・再起動）の制御、リモートサーバーへの接続、およびステータス表示を担います。

モジュール内での役割

support/win32 モジュールにおけるフロントエンド層として機能します。サービス管理の基盤となる Windows API（SCM: Service Control Manager）をラップし、ユーザーフレンドリーな対話インターフェースを提供します。

責務

  • **サービス監視**: レジストリをスキャンして Apache サービスを特定し、定期的に状態をポーリングする。
  • **サービス制御**: SCM を介したサービスの起動、停止、再起動の実行。
  • **UI 管理**: タスクトレイアイコンの表示、サービス管理ダイアログの描画、およびリモート接続管理。
  • **プロセス間通信**: 権限昇格が必要な操作のためのプロセス再起動（runas）および既存インスタンスの終了処理。

重要なシンボル

サービス管理構造体

  • ST_APACHE_SERVICE: サービス名、表示名、パス、PID 等を保持する。
  • ST_MONITORED_COMP: リモート監視対象のコンピュータ名とレジストリハンドルを保持する。

グローバル状態

  • g_stServices[], g_stComputers[]: 監視対象のサービスおよびコンピュータの配列。
  • g_stcSection: サービス状態更新時の競合を防ぐためのクリティカルセクション。

コア関数

  • ApacheManageService(): SCM を使用してサービスを制御するメインロジック。
  • GetApacheServicesStatus(): レジストリを走査し、Apache サービスを列挙・更新する。
  • am_RespawnAsUserAdmin(): 権限不足時に管理者権限で自身を再起動する。
  • WndProc() / ServiceDlgProc(): メインウィンドウおよびサービス管理ダイアログのメッセージループ処理。

制御フローとデータフロー

1. **初期化**: WinMain が OS バージョンを確認し、レジストリから監視対象をロードしてメインウィンドウを作成。 2. **監視ループ**: WM_TIMER メッセージにより定期的に FindRunningServices が呼び出され、サービス状態を更新。変更があれば ShowNotifyIcon でトレイアイコンを更新。 3. **ユーザー操作**: ユーザーがダイアログで操作を行うと WM_MANAGEMESSAGE が発行され、ApacheManageService が SCM を通じてサービスを制御。 4. **終了**: KillAllMonitors または WM_QUIT により、リソース（ハンドル、クリティカルセクション）を解放して終了。

読者向けメモ

  • **メモリ管理**: am_ClearServicesSt および am_ClearComputersSt で動的に確保された文字列（_tcsdup）を解放します。終了処理でのメモリリークには注意が必要です。
  • **権限**: サービス制御には管理者権限が必要です。権限がない場合、am_RespawnAsUserAdmin が ShellExecute で runas を呼び出し、昇格を試みます。
  • **制約**: MAX_APACHE_SERVICES (128) および MAX_APACHE_COMPUTERS (32) の固定長配列を使用しているため、大規模な環境では制限を受ける可能性があります。
  • **未実装/浅い部分**: ネットワークブラウズ機能は SHBrowseForFolder に依存しており、環境によっては動作が不安定になる可能性があります。また、エラーハンドリングは ErrorMessage 関数に集約されていますが、一部のレジストリ操作は簡略化されています。
