support/win32/wintty.c

概要

wintty.c は、Windows サービス（Apache HTTP Server など）のコンテキストで実行されているプロセスに対し、標準入出力（stdin/stdout）を対話的なコンソールウィンドウへリダイレクトするためのユーティリティです。

モジュール内での役割

support/win32 モジュールにおいて、サービスプロセスとユーザー間の対話を実現する補助ツールとして機能します。サービスとして実行中のプロセスが本来持たない「コンソール」を動的に割り当て、ユーザー入力を受け付けたり、出力を表示したりすることを可能にします。

責務

  • **コンソール分離と再割り当て**: サービスセッションからコンソールを切り離し、新しいコンソールウィンドウを生成する。
  • **ハンドル管理**: サービス環境下で標準入出力ハンドルを適切に複製・継承し、プロセス間通信を維持する。
  • **双方向ストリームの仲介**: main スレッドで stdin からの入力をコンソールへ送り、feedback スレッドでコンソールからの入力を stdout へ返す。

重要なシンボル

  • main: プログラムの起点。引数解析、サービス検出、コンソール生成、スレッド起動を統括。
  • feedback: 別スレッドで動作し、コンソール入力（CONIN$）を監視して stdout へ転送する。
  • feedback_args_t: feedback スレッドに渡される入出力ハンドルの構造体。
  • isservice: 現在のプロセスがサービスセッションで動作しているかを判定するフラグ。

制御フローとデータフロー

1. **初期化**: コマンドライン引数を解析し、現在のプロセスがサービスセッション内にあるかを確認。 2. **ハンドル複製**: DuplicateHandle を使用して、現在の標準入出力ハンドルをサービス環境下でも有効な状態に保つ。 3. **コンソール生成**:

  • サービス環境であれば、CreateProcess を用いて WinSta0\Default デスクトップ上で新しいコンソールプロセスを起動。
  • 通常環境であれば、AllocConsole でコンソールを割り当て。

    • 4. **データ転送**:

  • メインスレッド: ReadFile(stdin) → WriteFile(CONOUT$)
  • フィードバックスレッド: ReadFile(CONIN$) → WriteFile(stdout)

    • 5. **終了処理**: EOF を検知するとスレッドを待機し、コンソールを解放して終了。

直接依存

  • windows.h: Windows API（プロセス、スレッド、コンソール制御）。
  • stdlib.h, stdio.h: 標準入出力および文字列操作。

関連コード

  • support/win32/ApacheMonitor.c: サービスの状態監視を行う関連ツール。
  • Win32 API (Console Functions): 本ファイルが多用している AllocConsole, SetConsoleMode, GetStdHandle 等のドキュメントを参照すること。

読者向けメモ

  • 本コードは「実験的」な実装であり、エラーハンドリングが限定的です。
  • isservice の検出ロジックは、WindowStation 名が "Service-" で始まるかを確認する独自の手法をとっています。
  • サービス環境下でのコンソール操作はセキュリティ上の制約が厳しいため、CreateProcess による別プロセス起動が重要な役割を果たしています。デバッグ時は verbose フラグを有効にすると OutputDebugString で詳細が出力されます。
