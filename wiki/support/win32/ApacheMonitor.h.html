support/win32/ApacheMonitor.h

概要

ApacheMonitor.h は、Windows 用 Apache サービス監視ツール（ApacheMonitor.exe）の GUI リソースおよび識別子を定義するヘッダーファイルです。ApacheMonitor.rc（リソース定義）と ApacheMonitor.c（実装）の間で共有される定数群を提供します。

モジュール内での役割

support/win32 モジュールにおいて、ユーザーインターフェースの構成要素を特定するための「名前空間」として機能します。このファイルにより、コードとリソースファイル間で ID の整合性が保たれます。

責務

  • **GUI 識別子の定義**: ダイアログ、メニュー、アイコン、ビットマップ、コントロールの ID を一元管理する。
  • **文字列リソースの定義**: アプリケーションのメッセージやタイトルに使用される文字列リソースのインデックスを定義する。
  • **コマンド ID の定義**: サービス操作（開始・停止・再起動）に関連する動的なメニューコマンド ID のベース値を定義する。

重要なシンボル

ダイアログおよびコントロール ID

  • IDD_DLGSERVICES, IDD_DLGCONNECT: メインサービス管理画面および接続ダイアログの ID。
  • IDC_SSTART, IDC_SSTOP, IDC_SRESTART: サービス操作用ボタンのコントロール ID。
  • IDC_STATBAR, IDL_SERVICES: ステータスバーおよびサービス一覧リストボックスの ID。

リソース識別子

  • IDI_APSRVMON, IDI_ICOSTOP, IDI_ICORUN: アプリケーションアイコンおよびサービス状態（停止/実行中）を示すアイコン。
  • IDB_BMPSTOP, IDB_BMPRUN, IDB_BMPHEADER: サービス状態表示用ビットマップ。

メッセージ文字列範囲

  • IDS_MSG_FIRST ～ IDS_MSG_LAST: アプリケーションが使用するローカライズ可能なメッセージ文字列のインデックス範囲。

コマンド ID ベース

  • IDM_SM_SERVICE, IDM_SM_START, IDM_SM_STOP, IDM_SM_RESTART: サービス操作メニューのベース ID。これらは実行時にオフセットを加えて個別のサービスを識別するために使用されます。

制御フローとデータフロー

本ファイル自体は定数定義のみを行うため、直接的な制御フローは持ちません。

  • **データフロー**: ApacheMonitor.rc がこれらの ID を参照してリソースを構築し、ApacheMonitor.c が WM_COMMAND メッセージの wParam を通じてこれらの ID を受信し、対応する処理（サービス制御 API の呼び出し）を分岐させます。

直接依存

  • なし（純粋な定数定義ヘッダー）。

関連コード

  • support/win32/ApacheMonitor.c: 本ヘッダーで定義された ID を使用して GUI イベントを処理する実装。
  • ApacheMonitor.rc (非表示): 本ヘッダーの ID を使用してダイアログレイアウトやアイコンを定義するリソーススクリプト。

読者向けメモ

  • **定数の性質**: すべてのシンボルはプリプロセッサマクロとして定義されています。
  • **拡張性**: IDM_SM_ 系定数は、複数のサービスを動的に管理するためにビットシフトや加算を前提とした設計になっています。
  • **省略された詳細**: 本ファイルはリソース定義に特化しており、サービス管理のロジック（ST_APACHE_SERVICE 構造体など）は ApacheMonitor.c 側に定義されています。
  • **注意点**: IDC_STATIC が -1 に定義されており、リソースエディタで自動生成される静的コントロールと競合しないよう配慮されています。
