support/fcgistarter.c

概要

fcgistarter.c は、FastCGI プログラムを起動し、指定されたポートでリクエストを待ち受けるためのスタンドアロンのユーティリティです。主に Unix 系プラットフォームにおいて、FastCGI プロセスが標準入力（ファイルディスクリプタ 0）経由でソケットを受け取れるように環境を整える役割を担います。

モジュール内での役割

support モジュール内の他のツールが設定ファイルやログ管理を行うのに対し、本ファイルはプロセス管理とネットワークソケットのバインドに特化しています。Apache HTTP Server の外部で FastCGI プロセスを先行起動させるための補助ツールとして機能します。

責務

  • コマンドライン引数による起動コマンド、ポート番号、インターフェース、プロセス数の指定。
  • 指定されたポートでの TCP ソケットの作成、バインド、およびリッスン。
  • デーモン化（apr_proc_detach）の実行。
  • 子プロセスへのソケットの継承と、FastCGI が期待するファイルディスクリプタ 0 へのリダイレクト。

重要なシンボル

  • main: プログラムの全ライフサイクルを管理するエントリポイント。
  • usage: 引数エラー時にヘルプを表示して終了する関数。
  • exit_error: APR のエラーコードを解釈し、メッセージを出力して終了するユーティリティ。

制御フローとデータフロー

1. **初期化**: apr_app_initialize で APR 環境を構築し、引数を解析します。 2. **ソケット準備**: apr_sockaddr_info_get と apr_socket_create を経由し、指定ポートでリッスン状態にします。 3. **デーモン化**: apr_proc_detach を呼び出し、制御端末から切り離します。 4. **プロセス生成**: apr_proc_fork をループ内で実行します。

  • **子プロセス**: apr_os_sock_get でソケットを取得し、apr_file_dup2 を用いてファイルディスクリプタ 0 に複製した後、execl でターゲットコマンドを実行します。
  • **親プロセス**: 指定された数のプロセスを生成するまで待機し、完了後にソケットを閉じます。

直接依存

  • apr.h, apr_pools.h: APR の基本構造とメモリ管理。
  • apr_network_io.h: ソケット通信用。
  • apr_thread_proc.h: プロセス生成および管理用。
  • apr_getopt.h: コマンドライン引数解析用。
  • apr_portable.h: プラットフォーム固有のファイルディスクリプタ操作用。
  • stdlib.h
  • unistd.h

関連コード

  • support/suexec.c: プロセス実行権限を管理する別のサポートツール。
  • apr_file_dup2: 本ファイルで最も重要な、ファイルディスクリプタを差し替えるための APR 関数。

読者向けメモ

  • **プラットフォーム依存性**: 本ファイルは Unix 系のファイルディスクリプタ操作に強く依存しており、Windows 等の非 Unix 環境では実装が未完了（#error）です。
  • **実装の制約**: FastCGI プロセスが fd 0 を必要とするため、APR の標準的な apr_proc_create を使用せず、dup2 を用いた低レイヤーなハックを行っています。このため、エラーハンドリングが子プロセス側で困難であるという設計上の制約があります。
