httpd-2.4.67

このシステムは何か

httpd-2.4.67 は、Apache HTTP Server のソースコードベースです。モジュール式アーキテクチャを採用しており、コアエンジン、マルチプロセッシングモジュール（MPM）、および認証・キャッシュ・プロキシ・HTTP/2 などの多様な機能モジュールで構成されています。

アーキテクチャ概観

本システムは、リクエスト処理のライフサイクルを「フック（Hook）」と「フィルタ（Filter）」で制御するパイプライン構造を核としています。

  • **コア**: プロセス/スレッド管理（MPM）、設定解析、リクエストディスパッチを統括。
  • **MPM**: OS 固有の並行処理モデル（Event, Worker, Prefork 等）を抽象化。
  • **モジュール**: 認証（AAA）、キャッシュ、プロキシ、HTTP/2 等の機能をフック経由でパイプラインに挿入。
  • **ストレージ抽象化**: socache や slotmem などの共通 API を通じて、共有メモリや外部キャッシュを透過的に利用。

主要モジュール

  • **server**: サーバーの起動、設定管理、MPM の基盤。
  • **modules/http / modules/http2**: HTTP プロトコル処理および多重化ストリーム管理。
  • **modules/proxy**: ロードバランサおよびバックエンド転送機能。
  • **modules/aaa**: 認証・認可のプロバイダフレームワーク。
  • **modules/cache**: コンテンツおよび共有オブジェクトキャッシュ。
  • **modules/md**: ACME プロトコルによる証明書自動管理。

実行時ライフサイクル

1. **起動**: main 関数が設定を読み込み、MPM を初期化。 2. **リスナー**: listener_thread がソケットを監視し、接続を受け付け。 3. **ディスパッチ**: process_socket を経由して、リクエストがワーカースレッドへ割り当て。 4. **処理**: ap_process_request がフック（translate_name, fixups, handler 等）を順次実行。 5. **フィルタリング**: Bucket Brigade を介してデータがフィルタチェーンを通過。 6. **終了**: clean_child_exit 等によりリソースを解放し、プロセス/スレッドを終了。

中核データ構造

  • **request_rec**: リクエストの全コンテキスト（URI、ヘッダー、プール等）を保持。
  • **conn_rec**: 接続ごとの状態（ソケット、フィルタチェーン等）を保持。
  • **ap_filter_t**: フィルタチェーンの各ノード。
  • **apr_bucket_brigade**: フィルタ間を流れるデータストリームのコンテナ。
  • **module**: モジュールのエントリポイントとフックを定義する構造体。

横断的トピック

  • **メモリ管理**: apr_pool_t を用いた階層的なメモリ管理（リクエスト終了時に一括解放）。
  • **同期・排他制御**: apr_proc_mutex や slotmem によるプロセス間通信・排他制御。
  • **ロギング**: ap_log_error 等の API を通じたモジュールごとのログレベル制御。
  • **プロトコル抽象化**: socache や slotmem によるストレージ/メモリの抽象化。

推奨読書順

1. include/httpd.h (基本構造の理解) 2. server/config.c (設定解析とモジュールロード) 3. server/mpm/event/event.c (並行処理モデルの理解) 4. modules/http/http_request.c (リクエスト処理フロー) 5. include/util_filter.h (フィルタアーキテクチャ) 6. modules/http2/h2_session.c (高度なプロトコル実装例)
