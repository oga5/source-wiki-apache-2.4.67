httpd-2.4.67

このシステムは何か

httpd-2.4.67 は、世界で最も広く利用されているオープンソースの HTTP サーバーソフトウェアです。モジュール式のアーキテクチャを採用しており、動的な機能拡張、多様なマルチプロセッシングモデル（MPM）、高度な認証・認可、キャッシュ、プロキシ、および HTTP/2 や WebDAV といった最新のプロトコルをサポートしています。

アーキテクチャ概観

本システムは、コアサーバー（server）を基盤とし、その上にリクエスト処理パイプライン（フィルタチェーン）とフックシステムを構築しています。

  • **コア**: プロセス/スレッド管理（MPM）、設定解析、リクエストライフサイクル管理。
  • **フィルタチェーン**: apr_bucket_brigade を介してデータをストリーム処理するパイプライン。
  • **モジュール**: フックシステムを通じてコアと疎結合に連携し、認証、キャッシュ、プロキシ、コンテンツ生成などの機能を提供。
  • **プラットフォーム抽象化**: os および APR により、Unix、Windows、NetWare 等の OS 差異を吸収。

主要モジュール

  • **コア**: server, server/mpm (event, worker, prefork, winnt 等)
  • **プロトコル/フィルタ**: modules/http, modules/http2, modules/filters
  • **プロキシ/クラスタ**: modules/proxy, modules/cluster
  • **認証/認可**: modules/aaa
  • **WebDAV**: modules/dav
  • **キャッシュ**: modules/cache
  • **SSL/TLS**: modules/ssl
  • **MD (Managed Domains)**: modules/md (ACME 自動証明書管理)

実行時ライフサイクル

1. **起動**: main 関数が設定を読み込み、MPM がリスナーソケットを準備。 2. **プロセス/スレッド生成**: make_child や start_threads によりリクエスト処理主体を生成。 3. **リクエスト受付**: リスナースレッドが接続を待ち受け、process_socket を経由してワーカースレッドへ引き渡し。 4. **リクエスト処理**: request_rec を中心に、翻訳、認証、ハンドラ実行、フィルタリングの各フェーズをフック経由で実行。 5. **終了**: just_die や clean_child_exit を通じて、シグナルに応じた安全なシャットダウンを実行。

中核データ構造

  • request_rec: リクエストごとの状態、URI、ヘッダー、プールを保持。
  • conn_rec: 接続ごとの状態、ソケット、フィルタチェーンを保持。
  • server_rec: 仮想ホスト設定を含むサーバー単位のコンテキスト。
  • apr_pool_t: リソースのライフサイクルを管理するメモリプール。
  • apr_bucket_brigade: フィルタチェーンを流れるデータストリームの連結リスト。
  • module: モジュールのエントリポイントとフックを定義する構造体。

横断的トピック

  • **メモリ管理**: apr_pool_t による階層的なリソース管理と自動解放。
  • **フィルタチェーン**: apr_bucket_brigade を用いたストリーム処理パイプライン。
  • **フックシステム**: モジュール間連携のための疎結合なコールバック機構。
  • **MPM**: プロセス/スレッドモデルの抽象化と並行処理制御。
  • **設定管理**: ap_conf_vector_t を用いた階層的な設定マージ。

推奨読書順

1. include/httpd.h (基本構造の理解) 2. server/core.c (サーバーの全体像と設定構造) 3. server/request.c (リクエスト処理のライフサイクル) 4. include/util_filter.h (フィルタチェーンの仕組み) 5. server/mpm/event/event.c (並行処理モデルの理解) 6. modules/http/http_protocol.c (プロトコルの基礎) 7. docs/manual/developer/mod_example_1.c (モジュール開発の基本)
