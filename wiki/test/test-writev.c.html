test/test-writev.c

概要

test-writev.c は、OS の writev() システムコールが複数のバッファを単一の TCP パケットに効率的に結合して送信できるかを検証するための診断ツールです。

モジュール内での役割

test モジュール内のネットワーク診断ユーティリティの一つです。特定のプラットフォームにおける writev() の実装が、断片的なパケット送信を行わないかを確認するために使用されます。

責務

  • 指定されたリモートホストおよびポートへの TCP 接続の確立。
  • TCP_NODELAY オプションの有効化による Nagle アルゴリズムの無効化。
  • 3 つのバッファ（各 100 バイト）を writev() で送信し、パケット結合の挙動を観測可能にする。

重要なシンボル

  • main: プログラムの唯一のエントリポイント。引数解析、ソケット接続、writev() の実行を順次行う。
  • struct iovec: writev() で使用される、複数のバッファを指し示すための構造体配列。
  • writev(): 本テストの対象となるシステムコール。

制御フローとデータフロー

1. **初期化**: コマンドライン引数から接続先 IP とポートを取得し、sockaddr_in を構築。 2. **接続**: socket() で TCP ソケットを作成し、connect() でリモートホストへ接続。 3. **設定**: setsockopt() を呼び出し、TCP_NODELAY を設定して即時送信を強制。 4. **実行**: 3 つの iovec 要素に同一バッファを割り当て、writev() を実行。 5. **出力**: 送信バイト数と errno を標準出力に表示し、終了。

直接依存

  • stdio.h, stdlib.h, unistd.h: 標準入出力およびプロセス制御。
  • sys/socket.h, netinet/in.h, arpa/inet.h: ソケット通信およびアドレス変換。
  • sys/uio.h: writev() および iovec 構造体の定義。
  • errno.h: エラーコードの参照。
  • sys/types.h

関連コード

  • test/test_select.c: 同様にネットワーク関連の挙動を検証するテストコード。
  • test/time-sem.c: システムの同期プリミティブを検証するテストコード。

読者向けメモ

  • 本ツールは単体で動作する診断用プログラムです。
  • 実行結果を評価するには、tcpdump や Wireshark 等のパケットキャプチャツールを併用し、実際にネットワーク上に流れるパケット数を確認する必要があります。
  • 意図した挙動（300 バイトが 1 パケットで送信されること）が確認できない場合、OS のカーネルレベルでの writev() 実装に問題がある可能性があります。
