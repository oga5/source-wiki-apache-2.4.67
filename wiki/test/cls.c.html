test/cls.c

概要

test/cls.c は、指定されたディレクトリ内のキャッシュファイルを走査し、その内容を解析して標準出力に整形表示するコマンドラインユーティリティです。主にキャッシュデータの検証やデバッグ用途に使用されます。

モジュール内での役割

test モジュール内の他のテストコードが機能テストを目的とするのに対し、本ファイルは特定のファイルフォーマット（キャッシュファイル）を読み取り、人間が判読可能な形式に変換するツールとしての役割を担います。

責務

  • ディレクトリ内のファイル一覧を取得し、隠しファイル（. や # で始まるもの）を除外する。
  • キャッシュファイルのヘッダーを checkmask 関数で検証する。
  • 16進数文字列で記録されたタイムスタンプやメタデータを hex2sec 関数で数値に変換する。
  • 解析結果を X-URL ヘッダー情報と共に整形して出力する。

重要なシンボル

  • checkmask: 文字列が特定のパターン（英字、数字、16進数など）に一致するかを検証する。
  • hex2sec: 8桁の16進数文字列を int 型の秒数（タイムスタンプ）に変換する。
  • main: ディレクトリ走査、ファイルオープン、解析ループ、出力の制御を行うメインエントリポイント。

制御フローとデータフロー

1. **初期化**: 引数からディレクトリパスを取得し、opendir で開く。 2. **ループ処理**: readdir でファイルを順次取得し、各ファイルに対して以下を実行する。

  • fopen でファイルを開き、最初の行を fgets で読み込む。
  • checkmask でフォーマットの妥当性を確認する。
  • hex2sec を用いて、行内の特定オフセットから日付、最終更新日、有効期限、バージョン、長さを抽出する。
  • 2行目から X-URL を読み取り、検証する。

    • 3. **変換と出力**: gmtime と strftime を使用して、抽出した数値を人間が読みやすい日付形式に変換し、結果を printf する。

直接依存

  • ctype.h: 文字種判定（isdigit, isupper 等）に使用。
  • dirent.h: ディレクトリ操作（opendir, readdir）に使用。
  • stdio.h: 入出力操作（printf, fopen, fgets 等）に使用。
  • string.h: 文字列操作（strlen, strncmp 等）に使用。
  • time.h: 時刻変換（gmtime, strftime）に使用。

関連コード

  • test/test_parser.c: 本ファイルと同様に、特定のデータ形式の解析を行うテストコード。
  • test/time-sem.c: 時刻関連の処理を扱うため、時間操作のロジックを比較する際に参照される。

読者向けメモ

  • 本プログラムは、特定のキャッシュファイルフォーマット（8桁の16進数で構成されたヘッダーを持つもの）を前提としています。
  • checkmask のマスク文字（@, #, & など）は、このファイル独自の仕様です。
  • エラーハンドリングは perror を使用して標準エラー出力に行われますが、キャッシュファイルの形式が不正な場合は Bad cache file と出力してスキップする設計になっています。
