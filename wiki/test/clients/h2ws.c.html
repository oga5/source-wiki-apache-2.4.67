test/clients/h2ws.c

概要

h2ws.c は、HTTP/2 を介した WebSocket (RFC 8441) の接続テストを行うためのコマンドラインクライアントです。nghttp2 ライブラリを利用して HTTP/2 セッションを管理し、特定のプロトコル違反や正常系シナリオをシミュレートしてサーバーの挙動を検証します。

モジュール内での役割

test/clients モジュールにおける唯一のツールであり、HTTP/2 サーバーの WebSocket 対応機能をテストするための統合テスト用クライアントとして機能します。

責務

  • **URI 解析**: WebSocket URI の分解と正規化。
  • **セッション管理**: nghttp2 を用いた TCP ソケット接続、セッションの初期化、およびコールバックのディスパッチ。
  • **ストリーム制御**: HTTP/2 ストリームのライフサイクル管理（作成、データ送信、クローズ処理）。
  • **シナリオ実行**: 指定されたテストケースに基づき、意図的に不完全なヘッダーセットを送信するなどのプロトコルテスト。
  • **I/O 多重化**: poll() を使用した非同期的な読み書きの制御。

重要なシンボル

セッション管理

  • struct h2_session: HTTP/2 セッションの状態（ソケット、nghttp2 セッションオブジェクト、ストリームリスト）を保持。
  • h2_session_open(): 接続の確立、非ブロッキング設定、nghttp2 コールバックの登録を行う。
  • h2_session_run(): poll() ループを回し、I/O イベントを処理するメインループ。

ストリーム管理

  • struct h2_stream: 個別の HTTP/2 ストリームの状態（ID、URI、コールバック関数）を保持。
  • ws_stream_submit(): CONNECT メソッドを用いた WebSocket ストリームの開始。
  • ws_stream_read_req_body(): ストリームのデータ送信（stdin 等からの読み込み）を処理する nghttp2 コールバック。

ユーティリティ

  • parse_uri(): ws:// または wss:// URI を解析し、ホスト・ポート・パスを抽出。
  • MAKE_NV, MAKE_NV_CS: nghttp2 用のヘッダーフィールド（Name-Value）を生成するマクロ。

制御フローとデータフロー

1. **初期化**: main が parse_uri で引数を解析し、h2_session_open で TCP 接続と nghttp2 セッションを構築。 2. **ストリーム生成**: シナリオに応じて ws_stream_submit が呼ばれ、nghttp2_submit_request で CONNECT フレームを送信。 3. **イベントループ**: h2_session_run が poll() を呼び出し、ソケットの読み書き可能状態を監視。 4. **コールバック**: nghttp2 からのイベント（on_frame_recv, on_header 等）が h2_session の各関数をトリガーし、状態を更新。 5. **終了**: 全ストリームがクローズされるとループを抜け、セッションを終了。

読者向けメモ

  • **メモリ所有権**: uri 構造体内の文字列や h2_stream は動的に確保されることがありますが、本ツールはテスト用であり、終了時に明示的な全メモリ解放を行わない箇所があります。
  • **非ブロッキング**: ソケットは sock_nonblock_nodelay により非ブロッキングモードで動作します。EWOULDBLOCK 等の処理は nghttp2 のコールバック内で適切にハンドリングされています。
  • **制限事項**: 本ファイルは「完全な URI パーサー」ではなく、テストに必要な最小限の機能のみを実装しています。また、wss:// (TLS) の実装は parse_uri で認識されますが、実際の TLS ハンドシェイク処理は本ソース内には含まれておらず、平文の ws:// 接続を主眼としています。
