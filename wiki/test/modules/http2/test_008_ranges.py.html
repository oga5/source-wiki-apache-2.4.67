test/modules/http2/test_008_ranges.py

概要

本ファイルは、Apache HTTP Server の mod_http2 における HTTP Range リクエストの処理能力と、それに伴うログ記録・統計情報の正確性を検証するテストスイートです。

モジュール内での役割

test/modules/http2 モジュールの一部として、HTTP/2 プロトコル特有のデータ転送制御（Range）が、HTTP/1.1 と同等に機能するか、およびサーバーの内部状態（ログやステータス）に正しく反映されるかを検証します。

責務

  • **Range リクエストの検証**: 部分コンテンツ（206 Partial Content）のレスポンスが、HTTP/2 経由で正しく返されることの確認。
  • **ログ記録の整合性**: CustomLog に出力されるバイト数（送信・受信・レスポンス）が、Range リクエストの仕様と一致しているかの検証。
  • **サーバー統計の追跡**: server-status モジュールが、Range リクエストを含むトラフィックを正しく集計しているかの確認。
  • **異常系テスト**: 転送途中で切断されたリクエストに対するサーバーの挙動確認。

重要なシンボル

  • TestRanges: テストクラス。クラススコープで環境構築（データファイル生成、Apache 設定）を行います。
  • test_h2_008_01: HTTP/2 と HTTP/1.1 の Range リクエスト結果を比較し、ログのバイト数を確認するメインテスト。
  • test_h2_008_04: server-status を解析し、Range リクエストによる統計情報の増分が期待通りか検証する。
  • get_server_status: /status?auto エンドポイントを叩き、レスポンスを辞書形式に変換するヘルパー。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィーチャが 100MB のダミーデータを作成し、CustomLog を有効にした Apache を起動します。 2. **リクエスト実行**: curl を使用して、特定の Range ヘッダーを付与したリクエストを送信します。 3. **検証**:

  • レスポンスコード（206）とボディサイズを直接検証します。
  • Apache を再起動してログバッファをフラッシュし、生成されたログファイルを JSON として読み込みます。
  • ログ内の bytes_resp_B（レスポンスバイト数）や bytes_tx_O（送信バイト数）が、期待される範囲内にあるかをアサートします。

直接依存

  • pytest: テストフレームワーク。
  • env.H2Conf, env.H2TestEnv: テスト環境の構成と Apache の制御に使用。
  • json, os, re: ログ解析およびファイル操作に使用。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。H2Conf や H2TestEnv の定義元。
  • test/modules/http2/test_003_get.py: 基本的な GET リクエストのテスト。比較対象として有用。

読者向けメモ

  • **ログのフラッシュ**: Apache のログは即座に書き込まれない場合があるため、テストの検証フェーズ前に env.apache_restart() を呼び出してログをフラッシュしています。
  • **バイト数計算**: ログの bytes_tx_O はフレームオーバーヘッドを含むため、厳密な値ではなく「期待値以上かつ一定の許容範囲内」というアサーションになっています。
  • **バグ追跡**: test_h2_008_04 は Apache Bugzilla #66801 に関連しており、サーバー統計の正確性に関する回帰テストの側面を持ちます。
