test/modules/http2/test_200_header_invalid.py

概要

本ファイルは、Apache HTTP Server の mod_http2 における HTTP/2 ヘッダー処理の堅牢性を検証するテストスイートです。不正な文字、過大なヘッダーサイズ、およびヘッダー数制限に対するサーバーの挙動（エラーレスポンスや接続切断）を網羅的にテストします。

モジュール内での役割

test/modules/http2 モジュールの一部として、プロトコル仕様（RFC）への準拠およびセキュリティ境界の検証を担当します。特に、HTTP/2 のストリーム制御と Apache のリクエスト制限設定（LimitRequest* ディレクティブ）の相互作用を検証する役割を担います。

責務

  • **不正文字の検証**: ヘッダー名や値に含まれる制御文字（0x01-0x1F, 0x7F）に対するサーバーの拒否動作の確認。
  • **リクエスト制限の検証**: LimitRequestLine, LimitRequestFieldSize, LimitRequestFields ディレクティブの設定値が、HTTP/2 通信において正しく適用されるかの検証。
  • **エラーハンドリングの確認**: 制限超過時に適切な HTTP ステータスコード（400, 414, 431）が返されるか、あるいはプロトコルレベルの RST_STREAM が発生するかの検証。

重要なシンボル

  • TestInvalidHeaders: テストクラス。pytest のクラススコープフィクスチャを使用して、テストごとに Apache を再起動し、CGI 環境を構築します。
  • env.curl_post_data / env.curl_get: curl を用いて不正なヘッダーやデータを送信するためのヘルパーメソッド。
  • env.nghttp: nghttp クライアントを使用して、より低レイヤーな HTTP/2 フレーム制御が必要なテストを実行します。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが H2Conf を介して Apache 設定を書き換え、apache_restart() で設定を反映します。 2. **実行**: 各テストメソッドが curl や nghttp を呼び出し、意図的に不正なリクエスト（長大なヘッダー、制御文字など）を送信します。 3. **検証**:

  • 期待される HTTP ステータスコード（4xx系）の確認。
  • 接続が切断された場合（r.response is None）の挙動確認。
  • env.httpd_error_log を使用し、サーバー内部で期待通りのエラーログ（AH02429, AH02430 等）が出力されているかを検証します。

直接依存

  • pytest: テストフレームワーク。
  • re: ログ解析のための正規表現処理。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の構築と制御を抽象化します。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。設定のインストールやプロセス管理のロジックが含まれます。
  • test/modules/http2/test_201_header_conditional.py: ヘッダー関連のテストの続きとして、条件付きリクエストの検証を行います。

読者向けメモ

  • 本テストは Apache のバージョン（特に 2.5.0 以降）によって挙動が異なるケースがあるため、env.httpd_is_at_least('2.5.0') による条件分岐が含まれています。
  • LimitRequestFields 0 のような境界値設定のテストが含まれており、設定値の解釈が正しく行われているかを確認する際の参考になります。
  • ログの無視（ignore_recent）やスキャン（scan_recent）は、テスト実行時に発生する意図的なエラーログを適切に処理するために不可欠です。
