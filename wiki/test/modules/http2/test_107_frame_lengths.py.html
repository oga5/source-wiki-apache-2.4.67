test/modules/http2/test_107_frame_lengths.py

概要

HTTP/2 通信におけるデータフレームの最大長（H2MaxDataFrameLen）設定が、サーバーのレスポンス生成に正しく適用されるかを検証するテストスイートです。

モジュール内での役割

test/modules/http2 テストスイートの一部として、HTTP/2 プロトコルスタックの境界条件（フレームサイズ制限）を検証します。サーバー設定がプロトコルレベルの制約を遵守しているかを確認する役割を担います。

責務

  • テスト用ファイルの生成（指定サイズのテキストファイル作成）。
  • H2MaxDataFrameLen 設定を動的に変更した Apache サーバーの構成と再起動。
  • nghttp クライアントを用いたリクエスト実行と、受信したデータフレーム長の検証。

重要なシンボル

  • mk_text_file: 指定された行数でファイルを作成するヘルパー関数。1行あたり128バイトの固定長データを出力します。
  • TestFrameLengths: テストクラス。pytest のフィクスチャを使用してテスト環境をセットアップします。
  • H2MaxDataFrameLen: Apache の mod_http2 設定ディレクティブ。テスト対象の主要なパラメータです。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャがテスト用ディレクトリに異なるサイズのテキストファイルを生成します。 2. **構成**: test_h2_107_01 がパラメータ化された data_frame_len を受け取り、H2Conf を用いて Apache 設定を書き換えます。 3. **実行**: env.apache_restart() で設定を反映後、nghttp を介して各ファイルにリクエストを送信します。 4. **検証**: レスポンスに含まれる data_lengths（受信した各データフレームのサイズ）を検査し、設定値を超過するフレームが存在しないことをアサーションで確認します。

直接依存

  • os: ファイルシステム操作に使用。
  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の抽象化を行います。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。H2Conf や nghttp のラッパーが含まれており、本テストの実行基盤です。
  • test/modules/http2/test_005_files.py: ファイル転送に関連するテストの参考になります。

読者向けメモ

  • 本テストは nghttp の出力結果（r.results["data_lengths"]）に強く依存しています。この値は nghttp が受信した HTTP/2 フレームのペイロードサイズを反映しています。
  • H2MaxDataFrameLen の値が小さい場合、大きなファイル転送が複数のフレームに分割されることを確認する仕組みになっています。
  • テスト実行には mod_http2 が有効な Apache 環境が必要です。
