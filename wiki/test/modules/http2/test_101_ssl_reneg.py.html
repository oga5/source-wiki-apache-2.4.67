test/modules/http2/test_101_ssl_reneg.py

概要

HTTP/2 接続における SSL 再ネゴシエーション（Renegotiation）の挙動を検証するテストスイートです。HTTP/2 仕様では、接続確立後の再ネゴシエーションが禁止されているため、Apache HTTP Server が適切に HTTP_1_1_REQUIRED エラーを返すかを検証します。

モジュール内での役割

test/modules/http2 モジュールの一部として、セキュリティおよびプロトコル制約の境界条件をテストします。特に、SSL 設定（SSLVerifyClient や SSLCipherSuite）が HTTP/2 接続に与える影響を確認する役割を担います。

責務

  • SSL 再ネゴシエーションをトリガーする設定（ディレクトリやロケーション単位の認証要求）の検証。
  • 再ネゴシエーション試行時に、サーバーが HTTP/2 接続を拒否し HTTP_1_1_REQUIRED を通知することの確認。
  • SSLRequireSSL および Require ssl 指令が HTTP/2 接続で正しく機能することの確認。
  • 期待されるエラーログ（AH02261 など）が生成されることの検証。

重要なシンボル

  • TestSslRenegotiation: テストクラス。mod_ssl が有効な環境でのみ実行されます。
  • _class_scope: クラススコープのフィクスチャ。テスト用の Apache 設定（VHost、ディレクトリ設定）を動的に生成・適用します。
  • test_h2_101_01 〜 test_h2_101_11: 各種再ネゴシエーションシナリオを網羅するテストメソッド。

制御フローとデータフロー

1. **セットアップ**: _class_scope が H2Conf を使用して、再ネゴシエーションを強制する Apache 設定を生成し、サーバーを再起動します。 2. **実行**: 各テストメソッドが curl を呼び出し、再ネゴシエーションが必要なリソースへアクセスします。 3. **検証**:

  • サーバーが HTTP/2 接続を維持したまま再ネゴシエーションを許可していないかを確認。
  • curl の stderr から HTTP_1_1_REQUIRED エラーコードを検出。
  • env.httpd_error_log.ignore_recent を使用して、意図的に発生させたエラーログを無視しつつ、ログの整合性を確認。

直接依存

  • pytest: テストフレームワーク。
  • re: エラーログや stderr のパターンマッチングに使用。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の構築と制御を行います。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。curl のバージョン判定やログ監視機能を提供します。
  • test/modules/http2/conftest.py: テスト実行に必要な共有フィクスチャを定義しています。

読者向けメモ

  • 本テストは mod_ssl が必須です。環境にない場合はスキップされます。
  • curl のバージョン（8.1.x 系）によっては挙動が異なるため、バージョンチェックによるスキップ処理が含まれています。
  • test_h2_101_05 は、再ネゴシエーションが発生する状況下で、h2load を用いて複数リクエストがどのように処理されるか（成功しないこと）を検証する負荷テスト的な側面を持っています。
