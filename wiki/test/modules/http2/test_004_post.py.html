test/modules/http2/test_004_post.py

概要

HTTP/2 プロトコルにおける POST メソッドの挙動を検証するための統合テストスイートです。Apache HTTP Server の mod_http2 が、様々なデータサイズや転送エンコーディング、クライアント（curl, nghttp）に対して正しくリクエストを処理できるかを検証します。

モジュール内での役割

test/modules/http2 モジュールの一部として、プロトコル機能の検証を担当します。特に、GET 以外のデータ送信を伴うリクエスト処理の堅牢性を確認する役割を担っています。

責務

  • **データ転送の整合性検証**: 送信したファイルデータとサーバーから返却されたデータが一致することを確認する。
  • **クライアント互換性**: curl および nghttp を使用し、HTTP/1.1 と HTTP/2 の両方で同一の挙動を示すかを確認する。
  • **エッジケースのテスト**: Expect: 100-continue、チャンク転送、コンテンツ長指定の有無など、HTTP 仕様の境界条件をテストする。
  • **環境変数の検証**: CGI 環境で HTTP/2 関連の環境変数が正しく設定されているかを確認する。

重要なシンボル

  • TestPost: テストクラス。クラススコープのフィクスチャで Apache の設定（CGI vhost）を管理します。
  • curl_upload_and_verify: curl を用いたアップロードと結果の検証を行うヘルパーメソッド。
  • nghttp_post_and_verify: nghttp を用いた POST 送信とエコーバックの検証を行うヘルパーメソッド。
  • test_h2_004_01 ～ test_h2_004_41: 個別のテストケース。データサイズやオプションを切り替えて実行されます。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが H2Conf を使用して Apache を再起動し、CGI 実行環境を構築します。 2. **実行**: 各テストメソッドが env.curl_upload や env.nghttp().upload を呼び出し、生成されたテストデータ（data-1k 等）をサーバーへ送信します。 3. **検証**: サーバーからのレスポンス（ステータスコード、ボディ、ヘッダー）をキャプチャし、ローカルのソースファイルと比較して整合性をアサートします。 4. **クリーンアップ**: テスト終了後、Apache の状態は維持されますが、各テストは独立したリクエストとして実行されます。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の構築と Apache 操作を抽象化します。
  • email.parser: マルチパート形式のレスポンスボディを解析するために使用されます。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。H2TestEnv クラスが Apache の制御や URL 生成を担います。
  • test/modules/http2/test_003_get.py: GET リクエストの検証用。対照的なテストとして参照されます。

読者向けメモ

  • 本ファイルは mod_http2 の機能検証に特化しており、Apache の CGI 設定（RewriteRule 等）を動的に生成しています。
  • nghttp を使用したテストは、より低レイヤーな HTTP/2 フレームレベルの挙動を確認する際に有用です。
  • test_h2_004_41 は特定のバグ（PR66597）の回帰テストとして実装されており、リダイレクト時のチャンクエンコーディングの挙動を検証しています。
