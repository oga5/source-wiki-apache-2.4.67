test/modules/http2/htdocs/cgi/upload.py

概要

HTTP/2 テストスイートにおける CGI ベースのファイルアップロードおよび削除処理を実装したスクリプトです。クライアントからの multipart/form-data リクエストを解析し、サーバー上の指定ディレクトリへの保存や削除を行います。

モジュール内での役割

test/modules/http2/htdocs/cgi ディレクトリ内で、動的なファイル操作を伴うテストケースを担当します。主に HTTP/2 サーバーのアップロード機能の検証に使用されます。

責務

  • **ファイルアップロード**: multipart/form-data 形式の入力を受け取り、安全なファイル名に変換して保存する。
  • **ファイル削除**: 指定されたファイル名の削除処理を行い、結果を HTTP レスポンスとして返す。
  • **UI 提供**: アップロード用フォームの生成（リクエストパラメータがない場合）。

重要なシンボル

  • get_request_params(): requestparser.py からインポート。リクエストボディを解析し、フォームデータとファイルオブジェクトを分離する。
  • fitem.save_to(fpath): アップロードされたファイルを指定パスに保存するメソッド。
  • os.path.basename(): パストラバーサル攻撃を防ぐため、ファイル名からディレクトリパスを除去する。

制御フローとデータフロー

1. **初期化**: get_request_params() を呼び出し、リクエスト内容を forms と files に展開。 2. **条件分岐**:

  • files に 'file' キーが存在する場合: ファイルを保存し、201 Created を返す。
  • forms に 'remove' キーが存在する場合: 指定ファイルを削除し、成功または失敗（404）を返す。
  • いずれでもない場合: アップロード用 HTML フォームを出力する。

    • 3. **レスポンス**: 標準出力へ HTTP ヘッダーと HTML ボディを書き出す。

直接依存

  • os: ファイル操作および環境変数（DOCUMENT_ROOT 等）の取得に使用。
  • sys: 標準ライブラリ。
  • requestparser: 同ディレクトリ内のモジュール。リクエスト解析の抽象化を提供。

関連コード

  • test/modules/http2/htdocs/cgi/requestparser.py: 本スクリプトが依存するリクエスト解析ロジックの実装。
  • test/modules/http2/htdocs/cgi/: 本スクリプトが配置されている CGI テスト群のディレクトリ。

読者向けメモ

  • **セキュリティ**: os.path.basename() を使用してパストラバーサルを防止していますが、本スクリプトはあくまでテスト環境用です。本番環境での利用は想定されていません。
  • **環境変数**: DOCUMENT_ROOT や REQUEST_SCHEME など、Web サーバーから渡される環境変数に強く依存しています。テスト実行時にはこれらの環境変数が適切に設定されている必要があります。
  • **保存先**: ファイルは ./files/ ディレクトリ（または DOCUMENT_ROOT 配下）に保存されるため、実行環境の書き込み権限に注意してください。
