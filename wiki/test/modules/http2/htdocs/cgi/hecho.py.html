test/modules/http2/htdocs/cgi/hecho.py

概要

HTTP/2 テストスイートにおける CGI 動作確認用の Python スクリプトです。クライアントから送信されたリクエストパラメータ（フォーム値または環境変数）を読み取り、指定されたヘッダー名とその値を出力する「ヘッダーエコー」機能を提供します。

モジュール内での役割

test/modules/http2/htdocs/cgi ディレクトリ内に配置された CGI スクリプト群の一つです。主に HTTP/2 のストリーム処理やヘッダー圧縮、リクエストのパースが正しく行われているかを検証するためのバックエンドとして機能します。

責務

  • **リクエスト解析**: requestparser を利用して、POST フォームデータまたは環境変数を取得する。
  • **ヘッダーの動的生成**: クライアントが指定した name を HTTP レスポンスヘッダーのキーとして使用し、value をその値として出力する。
  • **エラーハンドリング**: パラメータが不足している場合や、リクエストが空の場合に適切な HTTP ステータスコード（400 または 200）と HTML フォームを返す。

重要なシンボル

  • get_request_params(): requestparser.py からインポートされる関数。リクエストボディを解析し、フォームデータとファイルデータを抽出します。
  • forms: リクエストパラメータを保持する辞書ライクなオブジェクト。
  • os.environ: name に対応する環境変数が存在しない場合のフォールバックとして利用されます。

制御フローとデータフロー

1. **初期化**: get_request_params() を呼び出し、リクエスト内容を forms に格納します。 2. **パラメータ抽出**:

  • forms['name'] をキーとして取得を試みます。
  • forms['value'] があればそれを値とし、なければ HTTP_{name} という形式の環境変数を検索します。

    • 3. **条件分岐**:

  • name が存在する場合: Status: 200 と共に、動的に生成したヘッダーを出力します。
  • name が欠落している場合: 400 Bad Request を返し、エラーメッセージを表示します。
  • KeyError（パラメータ未指定）が発生した場合: 入力用 HTML フォームを表示して終了します。

直接依存

  • os, sys: 標準ライブラリ。環境変数の取得やシステム制御に使用。
  • requestparser: 同一ディレクトリ内のモジュール。リクエストのパースを抽象化するために依存しています。

関連コード

  • test/modules/http2/htdocs/cgi/requestparser.py: 本スクリプトが依存するリクエスト解析ロジックの実装。
  • test/modules/http2/htdocs/cgi/echo.py: 類似したエコー機能を持つスクリプト。比較対象として有用です。

読者向けメモ

  • 本スクリプトは CGI プロトコルに従い、標準出力へ直接 HTTP レスポンスを書き出します。
  • print 文の直後に \n を含めることで、HTTP ヘッダーとボディの区切り（空行）を生成しています。
  • HTTP/2 のテストにおいて、特定のヘッダーが正しく伝搬・処理されるかを確認する際に利用してください。
