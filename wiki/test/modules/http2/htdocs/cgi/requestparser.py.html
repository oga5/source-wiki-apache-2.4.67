test/modules/http2/htdocs/cgi/requestparser.py

概要

HTTP/2 テストスイートの CGI 環境において、クライアントから送信されたリクエストパラメータ（クエリ文字列、フォームデータ、マルチパートファイルアップロード）を解析するためのユーティリティスクリプトです。

モジュール内での役割

test/modules/http2/htdocs/cgi ディレクトリ内の各 CGI スクリプトが、共通してリクエスト入力を処理するためのヘルパーライブラリとして機能します。特に upload.py 等のファイルアップロードを伴うテストケースで中心的な役割を果たします。

責務

  • **環境変数の解釈**: REQUEST_URI や CONTENT_TYPE を読み取り、適切な解析戦略を選択する。
  • **フォームデータのパース**: application/x-www-form-urlencoded 形式の入力を辞書形式に変換する。
  • **マルチパート解析**: multipart ライブラリを利用し、フォームフィールドとアップロードされたファイルを分離・抽出する。
  • **バイナリ互換性の確保**: Windows 環境での標準入出力のバイナリモード設定を自動化する。

重要なシンボル

  • FileItem: マルチパート解析で得られたファイルオブジェクトをラップし、ファイル名の抽出や保存（save_to）を簡略化するクラス。
  • get_request_params(): リクエストを解析し、(フォーム辞書, ファイル辞書) のタプルを返すメイン関数。

制御フローとデータフロー

1. **初期化**: スクリプト実行時に msvcrt を確認し、Windows 環境であれば標準入出力をバイナリモードに切り替える。 2. **クエリ解析**: os.environ["REQUEST_URI"] からクエリ文字列を抽出し、urllib.parse で辞書化する。 3. **ボディ解析**: CONTENT_TYPE に応じて分岐する。

  • application/x-www-form-urlencoded: sys.stdin から読み込み、parse_qs で処理。
  • multipart/*: multipart.parse_form を呼び出し、コールバック関数 on_field と on_file を通じてデータを収集する。

    • 4. **結果返却**: 解析済みのフォームデータと FileItem オブジェクトの辞書を呼び出し元へ返す。

直接依存

  • os, sys: 環境変数および標準入出力の制御。
  • urllib.parse: URL およびクエリ文字列の解析。
  • multipart: マルチパート形式のストリーム解析（外部ライブラリ）。
  • shutil: ファイルコピー操作。

関連コード

  • test/modules/http2/htdocs/cgi/upload.py: 本スクリプトを利用してファイルアップロードのテストを行う代表的な CGI スクリプト。

読者向けメモ

  • 本スクリプトはスタンドアロンの CGI としてではなく、他のスクリプトから import されることを前提としています。
  • multipart ライブラリ（python3-multipart）が実行環境にインストールされている必要があります。
  • FileItem.save_to メソッドは、内部で fsrc.seek(0) を呼び出しているため、ストリームの読み込み位置に注意してください。
