test/modules/http2/htdocs/cgi/env.py

概要

HTTP/2 モジュールのテスト環境において、サーバーの環境変数（os.environ）の内容を動的に取得・表示するための CGI スクリプトです。

モジュール内での役割

test/modules/http2/htdocs/cgi ディレクトリ内に配置されたテスト用ユーティリティの一つです。主に HTTP リクエストを通じてサーバー側の環境変数が正しく伝播・取得できるかを確認する検証用エンドポイントとして機能します。

責務

  • クライアントから送信された name パラメータを解析する。
  • 指定された名前の環境変数を os.environ から検索し、その値をレスポンスとして返す。
  • パラメータが未指定の場合は、入力フォームを表示する UI を提供する。

重要なシンボル

  • get_request_params(): requestparser.py からインポートされる関数。POST/GET リクエストのパラメータを解析し、辞書形式で取得します。
  • os.environ: Python の標準ライブラリ。サーバープロセスの環境変数にアクセスするために使用されます。

制御フローとデータフロー

1. **初期化**: get_request_params() を呼び出し、リクエストから forms データを抽出します。 2. **パラメータ検証**: forms['name'] を取得し、環境変数名として有効かを確認します。 3. **条件分岐**:

  • name が存在する場合: os.environ から該当する値を読み取り、text/plain 形式で出力します。
  • name が欠落している場合: HTTP 400 エラーを返し、エラーメッセージを表示します。
  • KeyError（パラメータ自体がない場合）: 環境変数を入力するための HTML フォームをレンダリングします。

直接依存

  • os: 環境変数へのアクセスに使用。
  • sys: 標準ライブラリ。
  • requestparser: 同一ディレクトリ内のモジュール。リクエスト解析のロジックを依存しています。

関連コード

  • test/modules/http2/htdocs/cgi/requestparser.py: 本スクリプトが依存するリクエスト解析ロジックが定義されています。
  • test/modules/http2/htdocs/cgi/echo.py: 同様の CGI テストスクリプト群であり、リクエストの echo や検証を行う際の比較対象となります。

読者向けメモ

  • 本スクリプトは CGI として動作するため、標準出力（print）がそのまま HTTP レスポンスボディとなります。
  • Status: ヘッダーを明示的に出力している点に注意してください。
  • テスト環境専用のスクリプトであるため、セキュリティ上の入力バリデーションは最小限です。本番環境への流用は避けてください。
