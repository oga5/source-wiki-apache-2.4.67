test/modules/http2/htdocs/cgi/mnot164.py

概要

HTTP/2 テストスイートの一部として、特定のデータ量を出力する CGI スクリプトです。主にストリーム制御やフロー制御の挙動を検証するために、指定された回数だけ文字列を繰り返し出力する負荷生成の役割を担います。

モジュール内での役割

test/modules/http2/htdocs/cgi ディレクトリ内に配置されたテスト用 CGI の一つです。他の CGI スクリプトがリクエストの echo や環境変数の確認を行うのに対し、本ファイルは「大量のデータ転送」という特定の条件下でのサーバーの応答性をテストするために使用されます。

責務

  • HTTP リクエストパラメータ（text, count）の解析。
  • 指定された文字列を、指定された回数だけ標準出力へ書き出す。
  • CGI プロトコルに従い、HTTP ステータスと Content-Type ヘッダーを正しく送出する。

重要なシンボル

  • get_request_params(): requestparser.py からインポートされる関数。リクエストのフォームデータとファイルを解析し、辞書形式で取得します。
  • sys.stdout.write(): バッファリングを考慮しつつ、指定回数の文字列をストリームに書き出すために使用されます。

制御フローとデータフロー

1. **初期化**: requestparser.get_request_params() を呼び出し、クエリパラメータまたは POST データから text と count を抽出します。 2. **デフォルト値**: パラメータが欠落している場合、text は "a"、count は 77784 に設定されます。 3. **ヘッダー送出**: Status: 200 OK および Content-Type: text/html を出力し、sys.stdout.flush() でヘッダーを即座に確定させます。 4. **データ生成**: ループ処理により、text を count 回数分、標準出力へ連続的に書き出します。

直接依存

  • os, sys: 標準ライブラリ。プロセス制御および標準出力の操作に使用。
  • requestparser: 同一ディレクトリ内のモジュール。リクエスト解析ロジックを共通化しています。

関連コード

  • test/modules/http2/htdocs/cgi/requestparser.py: 本スクリプトが依存するリクエスト解析ロジックの実装。
  • test/modules/http2/htdocs/cgi/echo.py: リクエスト内容をそのまま返す基本的な CGI。比較対象として有用です。

読者向けメモ

  • 本スクリプトは、HTTP/2 のフロー制御（WINDOW_UPDATE）のテストを意図している可能性が高いです。count を調整することで、サーバーが一度に送信するデータ量や、クライアント側の受信バッファの挙動を検証できます。
  • sys.stdout.flush() の呼び出し位置に注意してください。ヘッダー送信後にフラッシュすることで、クライアントがヘッダーを確実に受信してからボディのストリームを受け取れるように設計されています。
