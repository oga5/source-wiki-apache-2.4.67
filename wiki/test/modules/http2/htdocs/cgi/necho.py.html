test/modules/http2/htdocs/cgi/necho.py

概要

HTTP/2 テストスイートにおける CGI スクリプトの一つ。クライアントから送信されたパラメータに基づき、指定された回数だけテキストを繰り返し出力する「N回エコー」機能を提供します。

モジュール内での役割

test/modules/http2/htdocs/cgi ディレクトリ内で、動的なレスポンス生成と遅延（スリープ）を伴うテストケースを担当します。主に HTTP/2 のストリーム制御やタイムアウト、レスポンスの断片化を検証するために使用されます。

責務

  • **パラメータ解析**: requestparser を利用して POST/GET パラメータを取得。
  • **動的レスポンス生成**: 指定された回数（count）だけテキスト（text）を標準出力へ書き出す。
  • **タイミング制御**: レスポンス生成の前後およびループ中に、指定された秒数（wait1, wait2, wait3）の待機処理を挿入する。
  • **エラーハンドリング**: 必須パラメータが欠落している場合に、デバッグ情報を含む HTML を返す。

重要なシンボル

  • get_request_params(): リクエストからフォームデータとアップロードファイルを抽出するヘルパー関数。
  • forms: クライアントから送信されたパラメータを格納する辞書。
  • wait1, wait2, wait3: レスポンスの各フェーズで挿入される待機時間（秒）。

制御フローとデータフロー

1. **初期化**: get_request_params() を呼び出し、リクエスト内容を解析。 2. **待機1**: wait1 が指定されていれば time.sleep() を実行。 3. **メインループ**: count が有効な場合、Status: 200 を出力し、wait2 待機後に指定回数 text を出力。最後に wait3 待機を行う。 4. **例外処理**: パラメータ不足（KeyError）が発生した場合、現在の環境変数とフォームの状態を HTML 形式で出力し、デバッグを支援する。

直接依存

  • requestparser: リクエストパラメータの解析に使用。
  • time: time.sleep() による遅延処理に使用。
  • os, sys: 環境変数（REQUEST_URI 等）の取得に使用。

関連コード

  • test/modules/http2/htdocs/cgi/requestparser.py: 本スクリプトが依存するリクエスト解析ロジックの定義。
  • test/modules/http2/htdocs/cgi/echo.py: より単純なエコー機能を提供するスクリプト。比較対象として有用。

読者向けメモ

  • 本スクリプトは CGI として動作するため、標準出力への書き込みがそのまま HTTP レスポンスボディとなります。
  • wait パラメータを調整することで、HTTP/2 のストリームが長時間開かれた状態での挙動や、サーバー側のタイムアウト設定のテストが可能です。
  • KeyError 発生時の出力には os.environ が含まれており、サーバー側の環境変数をデバッグする際のヒントになります。
