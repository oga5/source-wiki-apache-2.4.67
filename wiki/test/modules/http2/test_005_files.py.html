test/modules/http2/test_005_files.py

概要

本ファイルは、HTTP/2 プロトコルにおけるファイル転送の正確性を検証するためのテストスイートです。Apache HTTP Server の mod_http2 環境下で、異なるサイズの静的ファイルを正しく取得できるかをテストします。

モジュール内での役割

test/modules/http2 テストスイートの一部として、基本的なリクエスト処理（test_003_get.py など）の後に位置し、ファイルシステム上の実データを用いた転送テストを担当します。

責務

  • **テスト用データの生成**: 指定されたサイズ（1k, 10k, 100k, 10000k）のテキストファイルを自動生成し、サーバーのドキュメントルートに配置する。
  • **環境構築**: テスト実行前に Apache サーバーを再起動し、生成したファイルにアクセス可能な仮想ホスト設定を適用する。
  • **転送検証**: curl を用いて生成したファイルへアクセスし、HTTP 200 OK が返却されることを確認する。

重要なシンボル

  • mk_text_file(fpath, lines): 指定された行数のテキストファイルを生成するユーティリティ関数。1行あたり128バイトの固定パターンを書き込む。
  • TestFiles: テストクラス。pytest のクラススコープフィクスチャを利用して、テスト実行前に環境を準備する。
  • _class_scope(self, env): テストクラス全体で一度だけ実行されるフィクスチャ。ファイル生成と Apache の設定更新・再起動を行う。

制御フローとデータフロー

1. **初期化**: _class_scope が呼び出され、env.server_docs_dir 配下にテスト用ディレクトリを作成。 2. **データ生成**: mk_text_file をループで呼び出し、多様なサイズのファイルを生成して URI_PATHS リストにパスを格納。 3. **設定適用**: H2Conf を使用して、生成したファイルを公開するための仮想ホスト設定を Apache に反映し、サーバーを再起動。 4. **検証**: 各テストメソッド（test_h2_005_01 等）が env.curl_get を実行し、サーバーからのレスポンスを検証する。

直接依存

  • os: ファイルシステム操作に使用。
  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv をインポートし、テスト環境の制御を行う。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤クラス。サーバーの再起動や curl ラッパーを提供。
  • test/modules/http2/test_003_get.py: 基本的な GET リクエストのテスト。本ファイルよりもシンプルな検証を行う。

読者向けメモ

  • 本テストは、ファイルサイズが異なる複数のファイルを生成するため、ディスク I/O が発生します。
  • mk_text_file は固定パターンのテキストを生成するため、コンテンツの整合性チェックが必要な場合、この生成ロジックを拡張する必要があります。
  • pytest の scope='class' を使用しているため、テストクラス内の全テストケースで同一のファイルセットが再利用されます。
