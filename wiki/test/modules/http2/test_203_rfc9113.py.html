test/modules/http2/test_203_rfc9113.py

概要

本ファイルは、HTTP/2 プロトコル仕様（RFC 9113）に関連するヘッダーフィールドのパースおよび処理ルールを検証するためのテストスイートです。特に、HTTPヘッダー値に含まれる空白文字（スペース、タブ）の扱いに関する挙動をテストします。

モジュール内での役割

test/modules/http2 モジュールの一部として、特定のプロトコル仕様への準拠性を確認する回帰テストを提供します。他のテストファイルが接続やストリーム制御を扱うのに対し、本ファイルはメッセージヘッダーの厳密な処理に焦点を当てています。

責務

  • リクエストヘッダーに含まれる末尾の空白文字が適切に無視されることの検証。
  • レスポンスヘッダーにおいて、先行・末尾の空白文字が適切にトリミングされることの検証。
  • 不正な制御文字が含まれるヘッダーに対するサーバーのエラーハンドリング（500エラーやRST_STREAM）の確認。

重要なシンボル

  • TestRfc9113: テストクラス。pytest を使用し、クラススコープで Apache の再起動を管理します。
  • test_h2_203_01_ws_ignore: リクエストヘッダーの空白文字許容をテストするメソッド。
  • test_h2_203_02: パラメータ化されたテストメソッド。レスポンスヘッダーのトリミングルールを網羅的に検証します。
  • H2Conf: テスト用の Apache 設定を動的に生成・インストールするためのヘルパークラス。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャにより、テスト対象の仮想ホスト設定がインストールされ、Apache が再起動されます。 2. **リクエストテスト**: curl を使用して、空白文字を含むヘッダーを送信し、サーバーが正常に応答（200 OK）することを確認します。 3. **レスポンステスト**: test_h2_203_02 が実行されるたびに、特定のヘッダー値を持つ設定が動的に再生成されます。curl で取得したレスポンスヘッダーを解析し、期待されるトリミング結果と一致するかを比較します。 4. **ログ検証**: 不正な文字を含むケースでは、サーバーログに特定の警告（AH02430 等）が出力されることを確認し、テスト終了時に無視リストへ追加します。

直接依存

  • pytest: テストフレームワーク。
  • pyhttpd.env.HttpdTestEnv: Apache テスト環境の基底クラス。
  • .env.H2Conf, .env.H2TestEnv: HTTP/2 特有の設定および環境管理クラス。

関連コード

  • test/modules/http2/env.py: H2Conf や H2TestEnv の定義が含まれており、設定生成のロジックを理解するために参照が必要です。
  • test/modules/http2/conftest.py: テスト環境全体の初期化ロジックが定義されています。

読者向けメモ

  • 本テストは、Apache のヘッダー処理ロジックが RFC 9113 に準拠しているかを検証するものです。
  • test_h2_203_02 は動的に設定を書き換えるため、テスト実行順序や並列実行には注意が必要です（通常は pytest のデフォルト設定で安全に動作します）。
  • 制御文字のテストにおいて、Apache のバージョンによってエラーの返し方（500エラー vs RST_STREAM）が異なる可能性があるため、テストコード内で条件分岐による許容が行われています。
