test/modules/http2/test_100_conn_reuse.py

概要

本ファイルは、Apache HTTP Server の mod_http2 における HTTP/2 コネクション再利用（Connection Reuse）および SNI（Server Name Indication）に関連する挙動を検証するテストスイートです。

モジュール内での役割

test/modules/http2 テストスイートの一部として、HTTP/2 の仕様である「同一コネクション上での複数ホストへのリクエスト」が、SSL 設定の互換性に基づいて正しく制御されているかを確認します。

責務

  • プロトコル選択（h2 vs h1.1）の妥当性検証。
  • SNI で指定したホスト名と、HTTP リクエストヘッダーの Host が異なる場合の挙動確認。
  • SSL 設定が異なるホストへのリクエスト時に、期待通り 421 Misdirected Request が返されることの保証。

重要なシンボル

  • TestConnReuse: テストクラス。pytest を使用し、クラススコープで Apache の設定を適用します。
  • H2Conf: テスト用の Apache 設定を生成・適用するヘルパー。
  • env.curl_protocol_version: 指定した URL に対して特定のオプションでリクエストし、使用されたプロトコルバージョンを判定します。
  • env.curl_get: HTTP リクエストを実行し、レスポンスオブジェクトを返します。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャにより、noh2（HTTP/2 無効）、test1、cgi の各 vhost を設定し、Apache を再起動します。 2. **テスト実行**: 各テストメソッドは env.mkurl でベース URL を取得し、curl を介してリクエストを送信します。 3. **検証**:

  • プロトコルネゴシエーションの確認（test_h2_100_01）。
  • SNI と Host ヘッダーの不一致による 421 エラーの発生確認（test_h2_100_04, test_h2_100_05）。

    • 4. **ログ管理**: 意図的なエラー（421）発生時に、Apache のエラーログを ignore_recent で無視し、テスト失敗を防ぎます。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv クラスを提供。

関連コード

  • test/modules/http2/env.py: テスト環境の構築と Apache 操作の基盤。
  • test/modules/http2/test_003_get.py: 基本的な GET リクエストのテスト。
  • test/modules/http2/test_103_upgrade.py: プロトコルアップグレードに関連するテスト。

読者向けメモ

  • 本テストは、HTTP/2 のコネクション再利用ルール（RFC 9113）が Apache で正しく実装されているかを検証するものです。
  • 421 Misdirected Request は、SSL 証明書が共有できない場合に発生する正常な挙動です。テスト内で AH02032 エラーを無視しているのは、これが期待されたエラーであるためです。
  • テスト実行には curl がインストールされている必要があります。
