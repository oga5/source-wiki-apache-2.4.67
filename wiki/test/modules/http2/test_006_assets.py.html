test/modules/http2/test_006_assets.py

概要

本ファイルは、HTTP/2 プロトコルにおけるアセット（画像、CSS、JS等）の取得およびリソースの依存関係解決が正しく行われるかを検証するテストスイートです。

モジュール内での役割

test/modules/http2 テストスイートの一部として、サーバーがクライアントからのリクエストに対して、HTML 内で参照される複数のリソースを適切に配信できるかをテストします。

責務

  • HTML ページおよびそれに付随する静的アセットの取得テスト。
  • 大量のリソース（多数の画像ファイル等）を伴うリクエストの処理検証。
  • HTTP/2 のウィンドウサイズ設定などのオプションを伴うリクエストの挙動確認。

重要なシンボル

  • TestAssets: テストクラス。クラススコープで Apache の設定（vhost）を適用します。
  • env.nghttp().assets(): nghttp クライアントを使用して、指定 URL から HTML とその依存アセットを再帰的に取得するヘルパーメソッド。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィーチャが H2Conf を使用して Apache を再起動し、テスト用仮想ホストを有効化します。 2. **テスト実行**: 各テストメソッドが env.mkurl で対象 URL を生成し、nghttp().assets() を呼び出します。 3. **動的生成**: test_h2_006_03 では、テスト実行時に一時的な画像ファイルをサーバーディレクトリに生成し、大量リソースの取得をシミュレートします。 4. **検証**: 取得されたアセットのリスト（ステータスコード、サイズ、パス）が期待値と一致するかを assert で確認します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の構築と制御を行います。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。nghttp クライアントのラッパーやサーバー設定の管理ロジックが含まれます。
  • test/modules/http2/test_005_files.py: 静的ファイルの配信に関する基本的なテストが含まれており、本ファイルの前提知識となります。

読者向けメモ

  • 本テストは nghttp コマンドラインツールに強く依存しています。環境に nghttp がインストールされていない場合、テストは失敗またはスキップされます。
  • test_h2_006_03 はファイルシステムへの書き込みを行うため、テスト実行環境の権限に注意してください。
  • すべてのテストで -Haccept-encoding: none オプションが指定されており、圧縮による影響を排除した純粋なリソース取得の検証を意図しています。
