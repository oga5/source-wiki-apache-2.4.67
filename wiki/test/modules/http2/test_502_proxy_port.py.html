test/modules/http2/test_502_proxy_port.py

概要

本ファイルは、Apache HTTP Server の mod_http2 におけるプロキシ機能のテストスイートの一部です。特に、異なるポート間での HTTP/2 クリアテキスト（h2c）プロキシ転送が正しく動作するかを検証します。

モジュール内での役割

test/modules/http2 モジュールにおけるプロキシ関連テスト群（test_500系）の一つです。プロキシ経由の通信において、ポート番号の解決や転送が期待通りに行われるかを検証する役割を担います。

責務

  • mod_http2 のプロキシ設定（ProxyPassMatch 等）を用いた h2c 通信の検証。
  • 特定のポート（env.proxy_port）でリッスンする仮想ホストの設定。
  • リクエストがプロキシ経由で転送された際、元のポート情報が保持・反映されているかの確認（PR 65881 関連の回帰テスト）。

重要なシンボル

  • TestProxyPort: テストクラス。クラススコープで Apache の設定を構築します。
  • _class_scope: pytest フィクスチャ。テスト実行前に H2Conf を用いて Apache の設定ファイルを生成し、サーバーを再起動します。
  • test_h2_502_01: メインのテストメソッド。curl を使用してプロキシ経由のリクエストを送信し、レスポンスの整合性を検証します。

制御フローとデータフロー

1. **セットアップ**: _class_scope が H2Conf を使用して、Listen ディレクティブと ProxyPassMatch を含む設定を生成します。 2. **サーバー起動**: env.apache_restart() により、設定を反映した Apache を起動します。 3. **リクエスト実行**: test_h2_502_01 が curl を呼び出し、プロキシポート経由でターゲット URL へリクエストを送信します。 4. **検証**: レスポンスのステータスコードが 200 であること、およびレスポンス内のポート情報が期待値と一致することを確認します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf（設定生成用）および H2TestEnv（環境情報・ユーティリティ用）を提供。

関連コード

  • test/modules/http2/test_500_proxy.py: 基本的なプロキシ機能のテスト。
  • test/modules/http2/test_503_proxy_fwd.py: プロキシ転送のより詳細な検証。
  • test/modules/http2/env.py: テスト環境の基盤となるクラス定義。

読者向けメモ

  • 本テストは PR 65881 の修正を確認するためのものです。
  • h2c（HTTP/2 Cleartext）プロキシの挙動に焦点を当てているため、SSL/TLS 関連のテストとは設定の前提が異なります。
  • env.proxy_port はテスト環境ごとに動的に割り当てられる可能性があるため、ハードコードせず env オブジェクト経由で参照してください。
