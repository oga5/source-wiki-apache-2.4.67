test/modules/http2/test_102_require.py

概要

Apache HTTP Server の mod_http2 における Require ディレクティブの動作検証を行うテストスイートです。特に、HTTP/2 プロトコルの有効・無効状態に基づいたアクセス制御が正しく機能するかを確認します。

モジュール内での役割

test/modules/http2 テストスイートの一部として、サーバー設定の条件分岐（expr）が HTTP/2 固有の環境変数（%{HTTP2}）を正しく評価できるかを検証する統合テストです。

責務

  • HTTP/2 の有効状態（on/off）に応じたアクセス制限のテスト。
  • Require expr ディレクティブを用いた条件付きアクセス制御の検証。
  • テスト実行時の Apache 設定の動的生成と再起動。

重要なシンボル

  • TestRequire: テストクラス。クラススコープで Apache 設定を生成します。
  • H2Conf: テスト用の Apache 設定を構築するためのヘルパークラス。
  • %{HTTP2}: Apache の環境変数。HTTP/2 接続であれば on、それ以外は off となります。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが H2Conf を使用して、特定のパス（/h2only.html, /noh2.html）に対して Require expr を含む仮想ホスト設定を生成・インストールします。 2. **実行**:

  • test_h2_102_01: HTTP/2 接続で /h2only.html にアクセスし、成功（または設定に応じた応答）を検証します。
  • test_h2_102_02: HTTP/1.1 接続（または意図的な条件不一致）で /noh2.html にアクセスし、403 Forbidden が返されることを確認します。

    • 3. **クリーンアップ**: env.httpd_error_log.ignore_recent を呼び出し、期待されるアクセス拒否エラー（AH01630）をログから除外します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供する環境設定モジュール。

関連コード

  • test/modules/http2/env.py: テスト環境の構築や Apache の制御を行う基盤コード。
  • test/modules/http2/conftest.py: テスト全体で共有されるフィクスチャ定義。

読者向けメモ

  • 本テストは、Apache の mod_authz_core および mod_http2 が連携して動作することを前提としています。
  • AH01630 エラーは Require ディレクティブによる意図的な拒否であるため、テスト実行後のログチェックで無視するように設計されています。
  • curl コマンドの挙動に依存しているため、環境内の curl が HTTP/2 をサポートしている必要があります。
