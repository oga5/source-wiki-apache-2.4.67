test/modules/http2/test_001_httpd_alive.py

概要

HTTP/2 モジュールのテストスイートにおける最初のテストケースであり、Apache HTTP Server が HTTP/2 プロトコルを有効にした状態で正常に起動し、リクエストに応答できるかを確認するスモークテストです。

モジュール内での役割

test/modules/http2 ディレクトリ内のテスト群において、環境の健全性を検証するゲートキーパーの役割を果たします。他の複雑なテストを実行する前に、基本的な接続性と設定の適用が成功していることを保証します。

責務

  • HTTP/2 環境の初期化と検証。
  • テスト用バーチャルホスト（test1）の構成と Apache の再起動。
  • /alive.json エンドポイントへのリクエストを通じた、サーバーの生存確認とレスポンス内容の検証。

重要なシンボル

  • TestBasicAlive: テストクラス。pytest を使用して環境の生存確認をカプセル化します。
  • H2Conf: HTTP/2 用のサーバー設定を生成・適用するためのユーティリティクラス。
  • H2TestEnv: テスト環境のメタデータや状態（サポート状況など）を管理するクラス。
  • env.curl_get: curl コマンドをラップし、HTTP/2 リクエストを送信してレスポンスを解析するヘルパーメソッド。

制御フローとデータフロー

1. **初期化**: _class_scope フィクスチャが実行され、H2Conf を用いてバーチャルホスト設定を生成・インストールし、Apache を再起動します。 2. **実行**: test_h2_001_01 が env.mkurl で対象 URL を生成し、curl_get を呼び出します。 3. **検証**: サーバーからのレスポンスコードが 0 であること、および返却された JSON オブジェクトの alive フィールドが True であることをアサーションで確認します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供する環境管理モジュール。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤となるクラス定義が含まれており、本ファイルの動作の根幹を支えています。
  • test/modules/http2/conftest.py: env フィクスチャの定義など、テスト実行に必要な共有設定が記述されています。

読者向けメモ

  • 本ファイルは、HTTP/2 モジュールがビルド・ロードされているかを確認する最小限のテストです。
  • pytest.mark.skipif により、環境が HTTP/2 をサポートしていない場合は自動的にスキップされます。
  • サーバーの再起動を伴うため、クラススコープのフィクスチャを使用しており、テスト実行順序において最初の方に配置されることが想定されています。
