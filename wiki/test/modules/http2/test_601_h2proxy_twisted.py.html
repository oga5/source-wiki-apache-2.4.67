test/modules/http2/test_601_h2proxy_twisted.py

概要

本ファイルは、Apache HTTP Server の mod_http2 における HTTP/2 プロキシ機能（h2proxy）の堅牢性を検証するテストスイートです。特に、バックエンドとの通信におけるアップロード処理、遅延、およびエラーハンドリングの挙動をテストします。

モジュール内での役割

test/modules/http2 モジュールの一部として、プロキシ機能の境界条件をテストします。test_600_h2proxy.py が基本的なプロキシ動作を扱うのに対し、本ファイルはより複雑なストリーム制御やエラーケースに焦点を当てています。

責務

  • HTTP/2 プロキシ経由でのファイルアップロードの正常性確認。
  • バックエンド応答の遅延（chunk_delay）に対するプロキシの追従性テスト。
  • バックエンド側での意図的なエラー発生時（fail_after）における、プロキシのストリーム終了処理の検証。
  • 大量リクエスト時の並列処理における安定性の確認。

重要なシンボル

  • TestH2ProxyTwisted: テストクラス。H2TestEnv を使用して環境を構築し、テストを実行します。
  • test_h2_601_01_echo_uploads: ファイルサイズを変えてアップロードとエコーバックを検証する基本テスト。
  • test_h2_601_03_echo_fail_early / 04_echo_fail_late: バックエンドが途中で失敗した場合のプロキシの挙動（HTTP 502 または curl エラーコード 92/18）を検証。
  • test_h2_601_05_echo_fail_many: curl --parallel を使用した高負荷時のエラーハンドリングテスト。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが H2Conf を用いて CGI を含むプロキシ設定を生成し、Apache を再起動します。 2. **実行**: 各テストメソッドは env.curl_upload または env.run を呼び出し、指定された URL に対してファイルを POST します。 3. **検証**:

  • 正常系: レスポンスステータスコードが 200 番台であることを確認。
  • 異常系: バックエンドが途中で切断するシナリオにおいて、curl が適切なエラーコード（92: HTTP/2 ストリームエラー、18: 部分転送エラー）を返すか、プロキシが 502 を返すかを検証します。

直接依存

  • pytest: テストフレームワーク。
  • env.H2Conf, env.H2TestEnv: テスト環境の設定および Apache の制御用クラス。
  • os, json, logging: 標準ライブラリ。

関連コード

  • test/modules/http2/test_600_h2proxy.py: プロキシ機能の基本テスト。
  • test/modules/http2/env.py: テスト環境の基盤。H2TestEnv の実装詳細が含まれます。

読者向けメモ

  • 本テストの多くは httpd 2.4.58 以上を要求します。環境が古い場合はスキップされます。
  • test_h2_601_05_echo_fail_many は curl v8.0.0 以上を必要とします。
  • エラーコード 92（CURLE_HTTP2_STREAM）は、HTTP/2 ストリームが異常終了した際に curl が返す一般的なコードです。プロキシテストにおいて、バックエンドの意図的な切断を正しく検知できているかの指標となります。
