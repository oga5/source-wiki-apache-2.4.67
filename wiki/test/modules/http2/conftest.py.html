test/modules/http2/conftest.py

概要

HTTP/2 モジュール (mod_h2) のテストスイート全体で使用される pytest の設定および共有フィクスチャを定義するファイルです。テスト実行環境の初期化とライフサイクル管理を担います。

モジュール内での役割

test/modules/http2 ディレクトリ内の全テストケースに対する基盤を提供します。個別のテストファイルが実行される前に、Apache HTTP Server の環境構築やログ設定を自動的に行うためのエントリーポイントとして機能します。

責務

  • **環境のセットアップ**: H2TestEnv を用いた Apache HTTP Server の起動と設定。
  • **ログ管理**: テスト実行時のログ出力設定および、無視すべき既知の警告ログ（AH10400 等）の登録。
  • **ライフサイクル管理**: パッケージスコープでのテスト環境の初期化と、全テスト終了後の Apache プロセス停止。
  • **レポート出力**: pytest 実行時のヘッダー情報に Apache のバージョンや MPM モジュール情報を付与。

重要なシンボル

  • env: パッケージスコープのフィクスチャ。テスト環境 (H2TestEnv) を提供し、Apache のセットアップとログのクリアを行います。
  • _h2_package_scope: autouse=True のフィクスチャ。テストスイート全体の終了時に Apache を安全に停止させる役割を持ちます。
  • pytest_report_header: pytest の出力に mod_h2 の環境情報を追加するフック関数。

制御フローとデータフロー

1. **初期化**: pytest が起動すると env フィクスチャが呼び出され、H2TestEnv がインスタンス化されます。 2. **セットアップ**: env.setup_httpd() により Apache が起動し、アクセスログとエラーログがクリアされます。 3. **テスト実行**: 各テストファイルが env フィクスチャを注入し、設定済みの Apache に対してリクエストを送信します。 4. **終了処理**: 全テスト終了後、_h2_package_scope の yield 以降が実行され、env.apache_stop() が呼び出されてプロセスが終了します。

直接依存

  • pytest: テストフレームワーク。
  • logging: テスト実行時のログ出力制御。
  • os, sys: パス解決のための標準ライブラリ。
  • .env.H2TestEnv: HTTP/2 テスト環境の抽象化クラス。

関連コード

  • test/modules/http2/env.py: H2TestEnv クラスの定義。Apache の制御ロジックやパス管理の詳細が記述されています。
  • test/modules/http2/test_*.py: 本ファイルで定義されたフィクスチャを利用する各テストケース。

読者向けメモ

  • 本ファイルは scope="package" を使用しているため、このディレクトリ内のテストは単一の Apache インスタンスを共有します。
  • AH10400 などの無視リストは、テスト環境特有の構成で発生する無害な警告を抑制するために重要です。環境の挙動を調査する際は、まず env.py を参照してください。
