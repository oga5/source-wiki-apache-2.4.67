test/modules/http2/ws_server.py

概要

本ファイルは、HTTP/2 テストスイート内で使用されるスタンドアロンの WebSocket エコーサーバーです。主に WebSocket プロトコルの挙動検証や、サーバー側の応答テストを目的として実装されています。

モジュール内での役割

test/modules/http2 ディレクトリにおいて、WebSocket 関連のテストケース（例: test_800_websockets.py）が通信相手として利用するサーバープロセスを提供します。

責務

  • 指定されたポートで WebSocket サーバーを起動し、接続を待機する。
  • リクエストパスに基づいたルーティング（echo, text, file）の実行。
  • ファイル転送時のバッファサイズ制御や遅延挿入など、テスト用の特殊な応答パターンの生成。
  • 接続終了時の適切なステータスコード返却。

重要なシンボル

  • on_async_conn: 接続ごとのハンドラ。パスを解析し、エンドポイントに応じた処理を振り分ける中心的な関数。
  • run_server: websockets.server.serve を用いてサーバーを非同期実行する。
  • echo: /echo エンドポイント専用のループ処理。受信したメッセージをそのまま送信する。

制御フローとデータフロー

1. **起動**: main() が引数からポート番号を受け取り、run_server() を呼び出す。 2. **接続**: on_async_conn が新しい WebSocket 接続を捕捉し、パス（rpath）を分割して処理を分岐する。 3. **処理**:

  • echo: async for ループでメッセージを反復処理し、即座に返送。
  • file: 指定されたファイルをバイナリモードで読み込み、設定されたバッファサイズと遅延時間に従ってチャンク単位で送信。

    • 4. **終了**: 処理完了後、conn.close() を呼び出し接続をクローズする。

直接依存

  • asyncio: 非同期処理の基盤。
  • websockets: WebSocket プロトコル実装ライブラリ。
  • argparse, logging, os, sys: 標準ライブラリによる環境設定およびロギング。

関連コード

  • test/modules/http2/test_800_websockets.py: 本サーバーをテスト対象として呼び出し、クライアント側の挙動を検証するテストコード。

読者向けメモ

  • 本サーバーはテスト専用であり、セキュリティや堅牢性は考慮されていません。
  • file エンドポイントでは、パスの先頭に ../ を付与してファイルシステムにアクセスするため、実行ディレクトリには注意が必要です。
  • time.sleep が使用されているため、大量のデータ転送や遅延設定時にはイベントループがブロックされる可能性がある点に留意してください。
