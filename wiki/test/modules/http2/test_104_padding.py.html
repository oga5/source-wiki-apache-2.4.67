test/modules/http2/test_104_padding.py

概要

HTTP/2 通信におけるパディング（Padding）機能の検証を行うテストスイートです。Apache HTTP Server の H2Padding 設定が、レスポンスフレームに期待通りのパディングバイトを付与しているかを検証します。

モジュール内での役割

test/modules/http2 モジュールの一部として、HTTP/2 プロトコル仕様におけるフレームパディングの挙動をテストします。特定のドメインに対して異なる H2Padding 設定を適用し、クライアント（nghttp）側で受信したパディングサイズを検証する役割を担います。

責務

  • 複数の仮想ホストを構築し、それぞれ異なる H2Padding 値（0, 1, 2, 3, 8 ビット）を設定する。
  • h2test-echo ハンドラを使用して、送信データに対するレスポンスのパディング量を測定する。
  • パディングサイズが指定されたビット数に基づいた期待値の範囲内にあることをアサートする。

重要なシンボル

  • frame_padding(payload, padbits): 指定されたペイロードサイズとパディングビット数から、期待されるパディングバイト数を計算するユーティリティ関数。
  • TestPadding: テストクラス。クラススコープのフィクスチャで Apache の設定（H2Padding）を動的に生成・適用します。
  • test_h2_104_01 ～ 06: 各パディング設定に対応するテストケース。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが実行され、H2Conf を用いて異なる H2Padding 設定を持つ複数の仮想ホストを定義し、Apache を再起動します。 2. **リクエスト実行**: 各テストメソッドが env.nghttp().post_data() を呼び出し、エコーサーバーへデータを送信します。 3. **検証**: レスポンスに含まれる paddings フィールドを抽出し、frame_padding で算出した理論値または許容範囲内であるかを検証します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf（設定生成）および H2TestEnv（環境管理）を提供。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。nghttp クライアントのラッパーや Apache 設定の管理ロジックが含まれます。
  • test/modules/http2/test_004_post.py: POST リクエストの基本的な挙動を確認するテスト。

読者向けメモ

  • 本テストは mod_http2 が有効な環境でのみ実行されます。
  • H2Padding は、HTTP/2 フレームのサイズを隠蔽するために使用される機能です。テストでは、パディングが 0 ビットの場合は固定値、それ以外の場合は指定ビット数に応じた範囲（$2^n$）に収まることを確認しています。
  • nghttp の実行結果から paddings を取得する仕組みに依存しているため、テスト環境の nghttp がパディング情報を正しくパースできる必要があります。
