test/modules/http2/test_007_ssi.py

概要

Apache HTTP Server の mod_http2 環境下における、Server Side Includes (SSI) 機能の動作検証を行うテストスイートです。

モジュール内での役割

test/modules/http2 テストスイートの一部として、HTTP/2 通信経由で SSI が正しく処理され、期待通りの HTML 出力が得られるかを検証します。

責務

  • mod_include を有効化したテスト環境の構築。
  • AddOutputFilter INCLUDES および Options +Includes を用いた SSI 設定の適用。
  • HTTP/2 プロトコル経由でリクエストを送信し、SSI 展開後のレスポンスボディが正確であることを確認する。

重要なシンボル

  • TestSSI: SSI 関連のテストケースを保持するクラス。
  • _class_scope: クラススコープのフィクスチャ。Apache の設定（H2Conf）を生成し、SSI 用のディレクトリ設定を注入してサーバーを再起動します。
  • test_h2_007_01: 実際に curl を使用して SSI ページを取得し、期待値と比較するテストメソッド。

制御フローとデータフロー

1. **環境構築**: _class_scope が実行され、H2Conf を通じて Apache 設定ファイルに SSI 関連ディレクティブを書き込みます。 2. **サーバー再起動**: env.apache_restart() により、新しい設定でサーバーをリロードします。 3. **リクエスト実行**: test_h2_007_01 が env.curl_get() を呼び出し、HTTP/2 で対象の .html ファイルにアクセスします。 4. **検証**: サーバーから返されたレスポンスボディ（r.stdout）と、ハードコードされた期待値文字列を比較し、SSI が正しく展開されているかを確認します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の抽象化と設定管理を行います。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤クラス。設定の注入やサーバー制御のロジックが含まれます。
  • Apache Bugzilla 66483: 本テストの背景となったバグ報告。

読者向けメモ

  • 本テストは、SSI が HTTP/2 のストリーム処理において正しくフィルタリングされるかを検証するものです。
  • test_h2_007_01 内の期待値文字列は、SSI の <!--#include ... --> がサーバー側で展開された後の最終的な HTML です。
  • mod_http2 がサポートされていない環境では、@pytest.mark.skipif により自動的にスキップされます。
