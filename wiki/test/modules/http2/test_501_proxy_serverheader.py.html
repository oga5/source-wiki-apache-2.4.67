test/modules/http2/test_501_proxy_serverheader.py

概要

HTTP/2 プロキシ環境下における Server レスポンスヘッダーの挙動を検証するテストスイートです。特に、バックエンド（CGI）で設定されたヘッダーが、プロキシ経由でクライアントに正しく反映されるかを確認します。

モジュール内での役割

test/modules/http2 モジュールの一部として、プロキシ機能の境界条件テストを担当します。test_500_proxy.py などの基本プロキシテストを補完し、ヘッダー操作の整合性に焦点を当てています。

責務

  • Apache HTTP Server のプロキシ設定において、Server ヘッダーの書き換えが意図通りに行われるかの検証。
  • CGI 経由のレスポンスにおいて、特定のヘッダー値が保持されることの保証。

重要なシンボル

  • TestProxyServerHeader: テストクラス。クラススコープのフィクスチャで Apache の設定を制御します。
  • _class_scope: Apache の設定（H2Conf）を生成し、Server ヘッダーを cgi に固定する設定を適用して再起動します。
  • test_h2_501_01: 実際に curl を使用してリクエストを送信し、レスポンスヘッダーの Server 値が期待通りかを確認するテストケース。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが H2Conf を用いて Header unset Server および Header always set Server cgi を含む設定を生成し、Apache を再起動します。 2. **実行**: test_h2_501_01 が env.mkurl で生成した URL に対して curl_get を実行します。 3. **検証**: レスポンスの JSON ボディに含まれるプロトコル情報と、レスポンスヘッダーの Server フィールドが期待値（"cgi"）と一致するかをアサートします。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の構築と Apache の制御を担います。

関連コード

  • test/modules/http2/test_500_proxy.py: プロキシ機能の基本テスト。
  • test/modules/http2/env.py: テスト環境の基盤クラス。設定生成や Apache のライフサイクル管理の詳細はここにあります。

読者向けメモ

  • 本テストは mod_http2 がサポートされている環境でのみ実行されます。
  • Header always set を使用しているため、バックエンドからのレスポンスがプロキシによって上書き・付与されるプロセスを検証しています。
  • テスト実行時に curl がインストールされている必要があります。
