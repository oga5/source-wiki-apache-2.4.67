test/modules/http2/test_710_load_post_static.py

概要

本ファイルは、mod_http2 における静的ファイルへの POST リクエストに対する負荷テストを実行するテストスイートです。h2load ツールを使用して、異なるデータサイズや並列数での POST リクエストの処理能力と安定性を検証します。

モジュール内での役割

test/modules/http2 テストスイートの一部として、HTTP/2 プロトコルにおける高負荷時の挙動を検証する「負荷テスト（Load Test）」カテゴリに属します。特に、サーバー側でデータが読み込まれる（slurped）静的ファイルへの POST 処理に焦点を当てています。

責務

  • h2load を用いた HTTP/2 POST リクエストの自動実行。
  • 異なるデータサイズ（1k, 10k, 100k）およびリクエスト数・並列数での負荷シナリオの提供。
  • 実行結果の検証（ステータスコード 2xx の確認およびエラー発生の有無のチェック）。

重要なシンボル

  • TestLoadPostStatic: テストクラス。クラススコープで Apache の設定と再起動を管理します。
  • check_h2load_ok: h2load の実行結果を解析し、全リクエストが成功したことをアサートするヘルパーメソッド。
  • test_h2_710_xx: 各テストケース。リクエスト数（n）、並列数（m）、接続数（conn）、および送信データファイルを定義して負荷をかけます。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャにより、テスト対象の仮想ホスト設定がインストールされ、Apache が再起動されます。 2. **実行**: 各テストメソッドが env.run(args) を呼び出し、外部ツール h2load を起動します。この際、env.gen_dir に配置されたテスト用データファイルが POST ボディとして指定されます。 3. **検証**: h2load の終了後、check_h2load_ok が結果オブジェクトを検証し、期待される成功数とステータスコード分布を照合します。

直接依存

  • pytest: テストフレームワーク。
  • os: ファイルパス操作に使用。
  • .env: H2Conf（設定生成）および H2TestEnv（環境管理）クラスを提供。

関連コード

  • test/modules/http2/test_700_load_get.py: GET リクエストに対する同様の負荷テスト。
  • test/modules/http2/test_711_load_post_cgi.py: 静的ファイルではなく CGI を対象とした POST 負荷テスト。
  • test/modules/http2/env.py: テスト環境の基盤となるクラス定義。

読者向けメモ

  • 本テストは外部ツール h2load に強く依存しています。環境に h2load がインストールされていない場合、テストは失敗またはスキップされます。
  • test_h2_710_00 から 02 は、データサイズと並列度を変えることで、サーバーのメモリ消費やスループットの境界条件をテストするように設計されています。
  • check_h2load_ok 内のアサーションは厳格であり、1件でも 4xx や 5xx が発生するとテストは失敗します。
