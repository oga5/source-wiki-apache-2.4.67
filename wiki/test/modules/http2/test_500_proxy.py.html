test/modules/http2/test_500_proxy.py

概要

本ファイルは、Apache HTTP Server の mod_http2 におけるプロキシ機能（mod_proxy 経由の HTTP/2 通信）の統合テストスイートです。クライアント（curl および nghttp）からプロキシ経由でバックエンドの CGI スクリプトを呼び出し、リクエストの転送、データアップロード、およびエラーハンドリングが正しく機能するかを検証します。

モジュール内での役割

test/modules/http2 モジュール群の一部として、HTTP/2 プロキシ構成の堅牢性を確認する役割を担います。特に、HTTP/2 で受け取ったリクエストをバックエンドへ転送する際のプロトコル変換や、ストリーム処理の整合性をテストします。

責務

  • **プロキシ経由の通信検証**: curl や nghttp を使用した GET/POST リクエストの疎通確認。
  • **データ転送の整合性**: アップロードしたファイルとサーバーから取得したデータがバイナリレベルで一致することの保証。
  • **エラーケースのテスト**: バックエンドからの 500 エラーやタイムアウト発生時の挙動、およびエラーログの確認。
  • **環境設定**: クラススコープのフィクスチャにより、プロキシ設定を含む Apache を再起動し、テスト環境を構築。

重要なシンボル

  • TestProxy: プロキシ機能テストをまとめるメインクラス。
  • curl_upload_and_verify: curl を用いてアップロードとダウンロードを行い、内容を比較するヘルパー。
  • nghttp_post_and_verify: nghttp を用いて POST データの送受信と整合性を検証するヘルパー。
  • test_h2_500_30〜32: プロキシ経由の意図的なエラー発生と、それに対するサーバーの応答・ログ出力を検証するテストケース。

制御フローとデータフロー

1. **初期化**: _class_scope フィクスチャが H2Conf を使用して Apache の設定を書き換え、プロキシを有効化して再起動します。 2. **リクエスト実行**: 各テストメソッドは env.mkurl で生成した URL に対し、curl や nghttp を呼び出します。 3. **検証**:

  • 正常系では、レスポンスコードの確認および env.gen_dir にあるソースファイルとレスポンスボディのバイナリ比較を行います。
  • 異常系では、env.httpd_error_log.ignore_recent を使用して、期待されるエラーログ（AH01110 等）を無視しつつ、クライアント側の終了コードを検証します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf（設定生成）および H2TestEnv（環境管理）を提供。
  • inspect, os, re: ファイルパス操作や正規表現による URL 変換に使用。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。設定生成や Apache の制御ロジックが含まれます。
  • test/modules/http2/test_600_h2proxy.py: プロキシ関連のテストとして、より高度な H2 プロキシ構成を扱う関連ファイル。

読者向けメモ

  • 本テストは mod_proxy が有効な環境を前提としています。mod_http2 がサポートされていない環境では pytest.mark.skipif によりスキップされます。
  • nghttp を使用したテストでは、--no-content-length オプションなどを駆使して、HTTP/2 のストリーム制御やチャンク転送の境界条件を網羅的にテストしています。
  • テスト実行時に生成される一時ファイルは env.gen_dir に配置されます。デバッグ時はこのディレクトリを確認してください。
