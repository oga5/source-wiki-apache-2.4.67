test/modules/http2/test_503_proxy_fwd.py

概要

本ファイルは、Apache HTTP Server のフォワードプロキシ機能における HTTP/2 サポート（H2ProxyRequests）の動作検証を行うためのテストスイートです。

モジュール内での役割

test/modules/http2 モジュールの一部として、プロキシ設定に関連するテストケース群（test_500系）を構成します。特に、クライアントがプロキシ経由でリクエストを送信する際の、HTTP/1.1 と HTTP/2 の切り替え挙動を検証します。

責務

  • Apache をフォワードプロキシとして設定し、HTTP/2 の有効/無効を切り替えた環境の構築。
  • curl を使用したプロキシ経由のリクエスト送信と、期待されるレスポンス（200 OK または 404）の検証。
  • H2ProxyRequests ディレクティブが正しく機能し、プロキシ接続のプロトコルネゴシエーションが意図通りに行われることの確認。

重要なシンボル

  • TestProxyFwd: テストクラス。クラススコープで Apache のプロキシ設定を制御します。
  • config_fwd_proxy: Apache の設定を動的に生成・適用するメソッド。H2ProxyRequests のオン/オフを引数で制御します。
  • test_h2_503_01_proxy_fwd_h1: HTTP/1.1 プロキシ接続の正常性確認。
  • test_h2_503_03_proxy_fwd_h2_on: HTTP/2 プロキシ接続の正常性確認。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィーチャにより、クラス実行前に config_fwd_proxy が呼び出され、Apache がフォワードプロキシとして起動します。 2. **テスト実行**: 各テストメソッドは env.curl_get を使用し、--proxy オプションを付与してリクエストを送信します。 3. **検証**:

  • test_h2_503_01: HTTP/1.1 でのプロキシ経由アクセスが成功することを確認。
  • test_h2_503_02: HTTP/2 が無効なプロキシに対して HTTP/2 で接続を試みた場合、適切に失敗（404）することを確認。
  • test_h2_503_03: 設定を HTTP/2 有効に再構成し、HTTP/2 でのプロキシ経由アクセスが成功することを確認。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf（設定生成）および H2TestEnv（環境情報・実行ヘルパー）を提供。

関連コード

  • test/modules/http2/test_500_proxy.py: プロキシ関連の基本テスト。
  • test/modules/http2/env.py: テスト環境の基盤。curl のラッパーや Apache の制御ロジックが含まれます。

読者向けメモ

  • 本テストは curl バージョン 8.1.0 以上を要求します。古い環境では一部のテストがスキップされます。
  • H2ProxyRequests は Apache のフォワードプロキシ機能における重要な設定項目であり、このテストはプロトコルネゴシエーションの境界条件を理解するのに適しています。
  • config_fwd_proxy 内で apache_restart() を呼び出しているため、テスト実行ごとに Apache の再起動が発生します。
