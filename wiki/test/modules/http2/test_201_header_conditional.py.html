test/modules/http2/test_201_header_conditional.py

概要

本ファイルは、HTTP/2 通信における条件付きリクエストヘッダー（If-Modified-Since, If-None-Match）および接続管理ヘッダーの挙動を検証するテストスイートです。

モジュール内での役割

test/modules/http2 テストスイートの一部として、HTTP/2 プロトコル実装における標準的なキャッシュ制御およびヘッダー処理の正確性を検証します。

責務

  • 条件付きリクエスト（304 Not Modified）が正しく処理されることの確認。
  • HTTP/2 において禁止されている、あるいは不要なヘッダー（Keep-Alive 等）が適切に除去または無視されることの確認。
  • Apache HTTP Server の mod_http2 設定が、期待通りにキャッシュ関連ヘッダーを解釈しているかの検証。

重要なシンボル

  • TestConditionalHeaders: テストクラス。クラススコープで Apache の設定を初期化します。
  • test_h2_201_01: If-Modified-Since ヘッダーの検証。
  • test_h2_201_02: If-None-Match (ETag) ヘッダーの検証。
  • test_h2_201_04: HTTP/2 における Keep-Alive ヘッダーの除去確認。

制御フローとデータフロー

1. **環境構築**: _class_scope フィーチャにより、H2Conf を用いて Apache の KeepAlive 設定を有効化し、サーバーを再起動します。 2. **リクエスト実行**: env.curl_get を使用して、特定のヘッダーを付与した HTTP/2 リクエストを送信します。 3. **検証**: レスポンスのステータスコード（304 または 200）およびレスポンスヘッダーの内容をアサーションで確認します。 4. **比較**: 必要に応じて --http1.1 オプションを用いたリクエストと比較し、プロトコル間でのヘッダー処理の差異を検証します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の構築と Apache の制御を行います。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。curl のラッパーやサーバー設定の管理ロジックが含まれます。
  • test/modules/http2/test_200_header_invalid.py: ヘッダー関連のテストとして、不正なヘッダー処理を扱う対照的なテストファイルです。

読者向けメモ

  • test_h2_201_03 は現在スキップされています。これは Vary ヘッダーの挙動に関する既知の制限事項に基づいています。
  • 本テストは mod_http2 が有効な環境でのみ実行されます。
  • Keep-Alive ヘッダーのテスト（test_h2_201_04）は、HTTP/1.1 と HTTP/2 の仕様上の違い（HTTP/2 では接続管理ヘッダーが禁止されている）を理解する上で重要なサンプルです。
