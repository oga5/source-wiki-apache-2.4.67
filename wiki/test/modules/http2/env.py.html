test/modules/http2/env.py

概要

HTTP/2 モジュールのテスト環境を構築・構成するための基盤クラス群を提供します。Apache HTTP Server (httpd) のテストスイートにおいて、H2 固有のモジュールロード、テスト用バイナリのコンパイル、および仮想ホスト設定のテンプレートを定義します。

モジュール内での役割

test/modules/http2 ディレクトリ内の各テストケースが共通して利用する「テスト環境の定義」を担います。pyhttpd フレームワークを拡張し、H2 特有のプロトコル設定や証明書要件を注入する役割を持ちます。

責務

  • **環境構築 (H2TestSetup)**: テスト用補助モジュール mod_h2test のコンパイルとロード、およびテスト用データファイル（1k〜1m）の生成。
  • **環境定義 (H2TestEnv)**: H2 関連のモジュール（http2, proxy_http2 等）の有効化、およびテスト用ドメインに対する証明書仕様の定義。
  • **設定生成 (H2Conf)**: H2/H2C プロトコルを有効化した仮想ホスト設定の生成と、テスト用ハンドラ（echo, delay, error）の登録。

重要なシンボル

  • H2TestSetup: テスト実行前のセットアップ処理をカプセル化。make() メソッドで環境を準備します。
  • H2TestEnv: テスト環境のベースクラス。setup_httpd をオーバーライドし、H2 向けの設定を注入します。
  • H2Conf: HttpdConf を継承し、H2 固有の Protocols 指令やハンドラ設定を容易にするためのヘルパー。

制御フローとデータフロー

1. **初期化**: H2TestEnv がインスタンス化され、必要なモジュールと証明書が登録されます。 2. **セットアップ**: setup_httpd が呼ばれると H2TestSetup が起動し、mod_h2test.c を apxs でコンパイルして modules.conf に追記します。 3. **データ生成**: _setup_data_1k_1m が実行され、テスト用ドキュメントルートにサイズ別のダミーファイルが配置されます。 4. **構成**: テストケース側で H2Conf を使用し、start_vhost を通じて特定のドメインに H2/H2C プロトコルを割り当てます。

直接依存

  • pyhttpd.certs.CertificateSpec: 証明書設定用。
  • pyhttpd.conf.HttpdConf: 設定ファイル生成用。
  • pyhttpd.env.HttpdTestEnv, HttpdTestSetup: テスト環境の基底クラス。
  • subprocess, inspect, os: モジュールのコンパイルとパス解決に使用。

関連コード

  • test/modules/http2/mod_h2test/: H2TestSetup がコンパイルするテスト用 Apache モジュールのソースディレクトリ。
  • test/modules/http2/conftest.py: この環境クラスを pytest のフィクスチャとして提供するエントリーポイント。

読者向けメモ

  • mpm_prefork は H2 をサポートしていないため、is_unsupported プロパティで明示的に除外されています。
  • mod_h2test はインストール済みのモジュールではなく、テスト実行時に動的にコンパイルされるため、環境に apxs がインストールされている必要があります。
  • 仮想ホストの設定で Protocols を動的に切り替えるロジックが含まれているため、特定のテストケースで H2 を無効化したい場合は add_vhost_noh2 を活用してください。
