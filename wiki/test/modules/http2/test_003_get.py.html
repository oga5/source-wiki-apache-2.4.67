test/modules/http2/test_003_get.py

概要

HTTP/2 プロトコルにおける基本的な GET リクエストの挙動を検証するテストスイートです。Apache HTTP Server の mod_http2 モジュールが、標準的なリクエスト、条件付きリクエスト、およびエラーハンドリングを正しく処理できるかを網羅的に確認します。

モジュール内での役割

test/modules/http2 ディレクトリ内で、HTTP/2 の基本機能（GET メソッド）を検証する中心的なテストファイルです。test_001 や test_002 での接続確認を経て、より具体的なプロトコル仕様の適合性をテストするフェーズに位置します。

責務

  • **プロトコル検証**: HTTP/2 経由でのリクエストが正しく処理され、適切なレスポンスが返るかの確認。
  • **機能テスト**: 条件付きリクエスト（If-Modified-Since, If-None-Match）や Range リクエストの検証。
  • **エッジケースの確認**: 不正なスキーム指定や、レスポンス生成中のタイムアウト・リセット等のエラー処理の検証。
  • **環境整合性**: CGI スクリプトを通じた環境変数（SSL/TLS 情報等）の正確な伝達の確認。

重要なシンボル

  • TestGet: テストクラス。pytest を使用し、クラススコープで Apache の設定（vhost）をインストールします。
  • env: テスト環境を管理するオブジェクト。URL 生成や curl コマンドの実行を抽象化します。
  • H2Conf: Apache の設定ファイルを生成・適用するためのヘルパークラス。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが H2Conf を用いて CGI およびテスト用 vhost を設定し、Apache を再起動します。 2. **リクエスト実行**: 各テストメソッド内で env.curl_get() を呼び出し、サーバーへリクエストを送信します。 3. **検証**: レスポンスオブジェクト r.response に格納されたステータスコード、プロトコルバージョン、ヘッダー、ボディの内容をアサーションで検証します。 4. **反復テスト**: pytest.mark.parametrize を活用し、異なるパスやデータサイズに対して同一のロジックを繰り返し実行します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv クラス（テスト環境の基盤）。
  • re: ヘッダーのクリーニングやレスポンスボディの正規化に使用。

関連コード

  • test/modules/http2/test_004_post.py: POST メソッドの検証を行う後続のテスト。
  • test/modules/http2/ws_server.py: WebSocket 等のより複雑な通信テスト用モジュール。

読者向けメモ

  • 本ファイルは curl を多用しています。curl のオプション（-I, -H 等）が HTTP/2 の挙動をどう制御しているかに注目してください。
  • test_h2_003_40 等のループテストは、レスポンスボディのサイズがプロトコルスタックのバッファリングに与える影響を検証しています。
  • clean_header メソッドは、動的に変化するヘッダー（Date, ETag 等）を除外して比較するための重要なユーティリティです。
