test/modules/http2/test_002_curl_basics.py

概要

本ファイルは、mod_http2 の基本的な動作検証を行うための統合テストスイートです。curl コマンドをクライアントとして使用し、HTTP/1.1 と HTTP/2 のプロトコルネゴシエーションや、仮想ホストごとの挙動が期待通りであるかを検証します。

モジュール内での役割

test/modules/http2 テストスイートにおける初期段階の検証フェーズを担います。test_001_httpd_alive.py でサーバーの起動を確認した後、プロトコルレベルの通信が正しく行えるかを保証する役割を持ちます。

責務

  • **環境構築**: テストクラス単位で H2Conf を用いて仮想ホスト（test1, test2）を設定し、Apache を再起動する。
  • **プロトコル検証**: curl のオプション（--http1.1, --http2, --no-alpn）を駆使し、サーバーが適切なプロトコルを選択・応答することを確認する。
  • **ホスト識別**: 異なる仮想ホストに対してリクエストを送り、正しいホスト名が返されるかを確認する。

重要なシンボル

  • TestCurlBasics: テストケースを保持するメインクラス。
  • _class_scope: クラス単位で実行されるフィクスチャ。Apache の設定と再起動を一度だけ行い、テストのオーバーヘッドを削減します。
  • env.curl_get: curl を実行し、レスポンス（ステータス、プロトコル、JSONボディ、ヘッダー）を解析して返すヘルパーメソッド。

制御フローとデータフロー

1. **セットアップ**: _class_scope が H2Conf を通じて httpd.conf を生成し、Apache を再起動します。 2. **リクエスト実行**: 各テストメソッドが env.mkurl で URL を生成し、env.curl_get を呼び出します。 3. **検証**: curl_get が返す r.response オブジェクトの各フィールド（protocol, status, json）を pytest の assert 文で検証します。 4. **終了**: 各テストケースは独立して実行され、結果が成功または失敗として報告されます。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf（設定生成）および H2TestEnv（環境情報）を提供。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。curl のラッパーや設定生成ロジックの詳細が定義されています。
  • test/modules/http2/test_001_httpd_alive.py: サーバーの生存確認を行う先行テスト。

読者向けメモ

  • 本ファイルは、HTTP/2 のネゴシエーション（ALPN）の挙動を理解するための最適なサンプルです。
  • test_h2_002_05 のように、--no-alpn を指定して HTTP/1.1 にフォールバックする挙動は、プロトコルネゴシエーションの理解に役立ちます。
  • テストが失敗する場合、Apache のログ（env が管理するパス）を確認することが推奨されます。
