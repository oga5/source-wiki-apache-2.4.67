test/modules/http2/test_600_h2proxy.py

概要

本ファイルは、Apache HTTP Server の mod_proxy_http2 モジュール（h2proxy）の機能検証を行うための統合テストスイートです。HTTP/2 プロキシとしての基本的な動作、接続の再利用、ヘッダー転送、およびエラー処理の妥当性を検証します。

モジュール内での役割

test/modules/http2 サブシステムの一部として、プロキシ関連のテストケースを網羅します。他のテストファイルが HTTP/2 の基本機能やプロトコル仕様に焦点を当てるのに対し、本ファイルは「バックエンドへのプロキシ転送」という特定のユースケースに特化しています。

責務

  • **プロキシ設定の検証**: ProxyPass や ProxyPreserveHost を用いた多様なプロキシ構成のテスト。
  • **接続再利用の確認**: enablereuse 設定が HTTP/2 ストリームの再利用に与える影響の検証。
  • **ヘッダー転送の整合性**: X-Forwarded-* ヘッダーが HTTP/1.1 プロキシと HTTP/2 プロキシ間で一貫しているかの確認。
  • **エラーハンドリング**: バックエンドからの 5xx エラーやタイムアウト発生時のプロキシ側の挙動（RST または 502/503 レスポンス）の検証。

重要なシンボル

  • TestH2Proxy: テストケースを保持するメインクラス。
  • H2Conf: テスト用の Apache 設定を動的に生成・インストールするためのユーティリティクラス。
  • env.curl_get / env.curl_raw: curl を用いてプロキシ経由のリクエストを実行し、レスポンスを解析するヘルパー。

制御フローとデータフロー

1. **セットアップ**: 各テストメソッド内で H2Conf を使用し、特定の ProxyPass ルールを含む Apache 設定を生成します。 2. **デプロイ**: env.apache_restart() を呼び出し、生成した設定を反映させます。 3. **実行**: curl を使用してプロキシ経由でバックエンドの CGI スクリプトを叩きます。 4. **検証**: レスポンスに含まれる JSON データ（プロトコルバージョン、ホスト名、ストリーム ID、ヘッダー情報）をアサーションで確認します。 5. **接続再利用の追跡**: h2_stream_id を監視し、同一接続上で複数のリクエストが処理されているかを判定します。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の構築と制御を担います。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。H2Conf の詳細な実装が含まれます。
  • test/modules/http2/test_500_proxy.py: HTTP/1.1 プロキシのテスト。本ファイルとの比較対象として有用です。

読者向けメモ

  • **バージョン依存**: 一部のテスト（test_h2_600_04 など）には、Apache HTTP Server のバージョン（2.4.60 以上か否か）による挙動の分岐が含まれています。これは、HTTP/2 接続再利用の仕様変更を考慮したものです。
  • **デバッグ**: テストが失敗した場合、env.apache_restart() の戻り値や、生成された設定ファイル（通常は gen/ ディレクトリ配下）を確認してください。
  • **CGI 依存**: バックエンドの検証には htdocs/cgi にあるスクリプトが利用されています。これらはリクエスト情報を JSON で返すため、プロキシの挙動を詳細にトレース可能です。
