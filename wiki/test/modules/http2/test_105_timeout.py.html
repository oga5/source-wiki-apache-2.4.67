test/modules/http2/test_105_timeout.py

概要

本ファイルは、Apache HTTP Server の HTTP/2 モジュールにおけるタイムアウト制御の挙動を検証するテストスイートです。サーバーのグローバルな Timeout 設定や mod_reqtimeout、および HTTP/2 特有の H2StreamTimeout が、接続やストリームのライフサイクルに正しく適用されるかをテストします。

モジュール内での役割

test/modules/http2 サブシステムの一部として、HTTP/2 通信における異常系および境界条件のテストを担当します。特に、低速なクライアントやタイムアウト発生時のサーバーの切断挙動を検証し、RFC 準拠と安定性を保証します。

責務

  • **接続タイムアウトの検証**: SSL ハンドシェイク中や接続確立直後のタイムアウト設定の有効性を確認する。
  • **リクエストタイムアウトの検証**: mod_reqtimeout がハンドシェイク完了後の接続に干渉しないことを確認する。
  • **ストリームタイムアウトの検証**: 接続全体のタイムアウトと個別のストリームタイムアウトが独立して機能し、適切なエラー（408 Request Timeout）を返すことを確認する。

重要なシンボル

  • TestTimeout: テストケースを保持するメインクラス。
  • H2Conf: テスト用の Apache 設定を動的に生成・適用するヘルパークラス。
  • CurlPiper: curl をバックグラウンドで実行し、ストリームの入出力を制御するユーティリティ。

制御フローとデータフロー

1. **設定生成**: H2Conf を使用して、テスト対象の Timeout や H2StreamTimeout を含む設定ファイルを生成し、Apache を再起動します。 2. **通信実行**:

  • socket を用いた低レイヤーの通信（ハンドシェイクのタイムアウト確認）。
  • CurlPiper を用いた高レイヤーのストリーム通信（POST 送信や遅延応答の確認）。

    • 3. **検証**: サーバーからの応答コード（408）や、ソケットが期待通りに閉じられたかを確認します。 4. **クリーンアップ**: env.httpd_error_log.ignore_recent を呼び出し、意図的なタイムアウトで発生したエラーログを無視してテストの成功を確定させます。

直接依存

  • socket: 低レイヤーの接続テストに使用。
  • time: タイムアウト待ち合わせやログフラッシュの待機に使用。
  • pytest: テストフレームワーク。
  • env.H2Conf: テスト環境の設定管理。
  • pyhttpd.curl.CurlPiper: HTTP/2 ストリームの制御。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤設定。
  • test/modules/http2/test_009_timing.py: HTTP/2 におけるタイミング関連のテスト。
  • test/modules/http2/test_106_shutdown.py: 接続終了処理に関連するテスト。

読者向けメモ

  • 本テストは、OS やネットワーク環境の微妙な差異（接続の検知タイミングなど）に影響を受ける可能性があるため、一部のテストで time.sleep や try-except による柔軟な判定が行われています。
  • test_h2_105_12 は curl v8.0.0 以上および Apache httpd 2.5.0 以上を要求するため、環境によってはスキップされることがあります。
  • タイムアウト発生時に出力される AH10373 などのエラーログは、テストの期待動作であるため ignore_recent で適切に処理されています。
