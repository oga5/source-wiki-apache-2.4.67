test/modules/http2/test_103_upgrade.py

概要

HTTP/1.1 から HTTP/2 へのプロトコルアップグレード（h2 および h2c）の挙動を検証するテストスイートです。サーバー側の設定（Protocols ディレクティブや H2Upgrade）に基づき、期待通りにアップグレードが許可・拒否されるかを網羅的に確認します。

モジュール内での役割

test/modules/http2 モジュールの一部として、HTTP/2 の接続確立プロセスにおけるネゴシエーションとアップグレードメカニズムの正当性を保証します。

責務

  • **アップグレード広告の検証**: サーバーが Upgrade ヘッダーを適切に送信するか（あるいは抑制するか）の確認。
  • **設定依存の検証**: H2Upgrade ディレクティブの ON/OFF および Location 単位の制御が正しく機能するかの確認。
  • **プロトコル優先順位の検証**: Protocols 設定における優先順位がアップグレードの可否に与える影響の確認。
  • **クライアント要求の処理**: nghttp を用いた明示的なアップグレード要求に対するサーバーの応答検証。

重要なシンボル

  • TestUpgrade: テストクラス。pytest を使用して環境構築とテストケースを実行します。
  • _class_scope: クラス単位のフィクスチャ。H2Conf を用いて、テストケースごとに異なる Apache 設定（vhost）を動的に生成・適用します。
  • env: テスト環境オブジェクト。URL 生成や curl/nghttp の実行インターフェースを提供します。

制御フローとデータフロー

1. **セットアップ**: _class_scope が H2Conf を介して複数の vhost を定義し、apache_restart() を呼び出して設定を反映させます。 2. **テスト実行**: 各 test_h2_103_xx メソッドが env.curl_get または env.nghttp() を呼び出し、特定の URL に対してリクエストを送信します。 3. **検証**: レスポンスヘッダー内の Upgrade フィールドの有無や、ステータスコードをアサーションで確認します。 4. **終了**: 各テストケース終了後、次のテストのために環境が維持されます。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv クラス（テスト環境の抽象化）。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。H2Conf の詳細な実装が含まれます。
  • test/modules/http2/conftest.py: テスト実行に必要な共有フィクスチャの定義。

読者向けメモ

  • 本ファイルは、Apache の mod_http2 が提供するアップグレード関連のディレクティブの挙動を理解するためのリファレンスとしても有用です。
  • test_h2_103_24 は、H2Upgrade がグローバルで無効であっても、特定の Location で有効化できるという設定の柔軟性をテストしています。
  • テスト実行には curl および nghttp がシステムにインストールされている必要があります。
