test/modules/http2/test_400_push.py

概要

HTTP/2 Server Push 機能の動作検証を行うテストスイートです。Apache HTTP Server の mod_http2 におけるサーバープッシュのトリガー条件、設定の優先順位、およびクライアントからの制御ポリシーを網羅的にテストします。

モジュール内での役割

test/modules/http2 モジュールの一部として、HTTP/2 特有の機能である「サーバープッシュ」の仕様適合性を検証します。他のテストファイルが基本的な通信やプロキシ動作を扱うのに対し、本ファイルはサーバーから能動的にリソースを送信する挙動に特化しています。

責務

  • Link ヘッダーによるプッシュ指定の解析と実行の検証。
  • H2PushResource ディレクティブによるサーバー側設定の検証。
  • H2Push off によるプッシュ無効化の検証。
  • クライアントからの accept-push-policy ヘッダーによるプッシュ制御の検証。
  • GET/POST リクエスト時におけるプッシュ挙動の差異確認。

重要なシンボル

  • TestPush: テストクラス。pytest を使用し、クラススコープで Apache の仮想ホスト設定を構築します。
  • H2Conf: テスト用の Apache 設定を生成するヘルパークラス。
  • nghttp: HTTP/2 クライアントツール。プッシュされたリソース（promises）の取得と検証に使用します。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィーチャが H2Conf を用いて、各テストケースに対応する Location ディレクティブを含む仮想ホスト設定を生成・適用します。 2. **実行**: 各 test_h2_400_xx メソッドが nghttp を介して特定の URL にリクエストを送信します。 3. **検証**: レスポンスに含まれる promises（サーバーからプッシュされたストリーム）の数と内容をチェックし、期待されるリソースがプッシュされたか、あるいは意図通りに抑制されたかをアサーションで確認します。

直接依存

  • os: 一時的なテスト用ファイル作成に使用。
  • pytest: テストフレームワーク。
  • .env: H2Conf および H2TestEnv を提供し、テスト環境の構築と制御を担います。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。H2TestEnv や H2Conf の定義が含まれており、すべてのテストで共通して利用されます。
  • test/modules/http2/test_006_assets.py: プッシュ対象となる静的アセットの配置や基本的な取得テストに関連します。

読者向けメモ

  • 本テストは nghttp コマンドがシステムにインストールされていることを前提としています。
  • Link ヘッダーの解析ロジックは非常に繊細です。nopush 属性や rel パラメータの組み合わせによる挙動の違いは、RFC 準拠のプッシュ実装を理解する上で非常に参考になります。
  • プッシュの検証には r.results["streams"][r.response["id"]]["promises"] という構造を使用します。これは nghttp の出力をパースしたもので、プッシュされたストリームを特定するために不可欠です。
