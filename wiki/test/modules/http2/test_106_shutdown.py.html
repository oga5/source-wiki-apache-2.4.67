test/modules/http2/test_106_shutdown.py

概要

本ファイルは、Apache HTTP Server の mod_http2 におけるシャットダウンおよび再起動時の挙動を検証するテストスイートです。特に、長時間実行されるリクエストの処理中におけるサーバーの再読み込み（Graceful Restart）や、プロセス制限下での接続終了処理の堅牢性を確認します。

モジュール内での役割

test/modules/http2 テストスイートの一部として、HTTP/2 プロトコル特有の接続管理とライフサイクルイベントを検証します。他のテストファイルが基本的なリクエストやプロキシ機能を扱うのに対し、本ファイルはサーバーの運用状態の変化が通信に与える影響をテストします。

責務

  • **Graceful Restart の検証**: リクエスト処理中に apache_reload() を実行し、接続が切断されずに完了するかを確認する。
  • **プロセス制限下の挙動確認**: MaxRequestsPerChild 等の制限に達した際の GOAWAY フレーム処理や、接続終了時のクライアント側の挙動（curl の終了コード）を検証する。

重要なシンボル

  • TestShutdown: テストクラス。pytest を使用してテストケースをグループ化。
  • test_h2_106_01: 長時間リクエスト中にサーバーをリロードし、レスポンスが正しく返るかを検証するテスト。
  • test_h2_106_02: MaxRequestsPerChild を低く設定し、プロセス終了に伴う接続の挙動を検証するテスト。

制御フローとデータフロー

1. **test_h2_106_01**:

  • 別スレッドで curl を実行し、サーバー側で wait2 を含む長時間リクエストを開始。
  • メインスレッドで time.sleep(0.5) 後に apache_reload() を実行。
  • スレッドの終了を待ち、curl の実行結果（ExecResult）を検証。

    • 2. **test_h2_106_02**:

  • H2Conf を用いて ServerLimit と MaxRequestsPerChild を制限した設定を適用。
  • ループ内で複数回リクエストを送り、接続が切断された場合（curl エラーコード 16/55）と正常終了した場合の双方を許容しつつ、プロトコルが HTTP/2 であることを確認。

直接依存

  • pytest: テストフレームワーク。
  • threading.Thread: 非同期リクエスト実行用。
  • .env.H2Conf, .env.H2TestEnv: テスト環境設定および Apache 操作用ユーティリティ。
  • pyhttpd.result.ExecResult: curl コマンドの実行結果を保持するデータクラス。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。apache_restart() や apache_reload() の実装が含まれる。
  • test/modules/http2/test_105_timeout.py: タイムアウト関連のテスト。本ファイルと同様に接続の安定性を検証する。

読者向けメモ

  • test_h2_106_02 で発生する curl のエラーコード（16: HTTP/2 stream error, 55: Failed to send/recv）は、サーバーのプロセス終了に伴う正常な挙動として扱われています。
  • apache_reload() は Graceful Restart をトリガーします。このテストは、HTTP/2 のストリームが再起動後も適切に維持されるかを確認する重要な指標となります。
