test/modules/http2/test_009_timing.py

概要

本ファイルは、Apache HTTP Server の mod_http2 におけるリクエスト処理時間の計測（time_taken）が正しく行われているかを検証するテストスイートです。特に、Keep-Alive 接続下での計測精度や、ログ出力のタイミングに関するエッジケースをカバーしています。

モジュール内での役割

test/modules/http2 テストスイートの一部として、HTTP/2 プロトコル特有のパフォーマンス計測機能の妥当性を検証します。他のテストファイルが機能の正当性を確認するのに対し、本ファイルはサーバーが記録するログの正確性に焦点を当てています。

責務

  • 単一リクエストにおける time_taken の正の値を検証する。
  • Keep-Alive 接続において、後続のリクエストを待たずに前段のリクエストの処理時間が適切に記録されるかを検証する（Issue #253 関連）。
  • h2load を用いた負荷テストスクリプトの実行と、生成されたログファイルの解析。

重要なシンボル

  • TestTiming: テストクラス。クラススコープのフィクスチャで Apache の設定と再起動を管理します。
  • test_h2_009_01: 単純な GET リクエストに対する時間計測の基本テスト。
  • test_h2_009_02: タイミングスクリプトを用いた、連続するリクエスト間での時間計測の正確性テスト。
  • LOGFILE: テスト結果を検証するために解析される Apache のカスタムログファイルパス。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが CustomLog を含む Apache 設定を生成し、サーバーを再起動します。 2. **実行**: h2load を使用して HTTP/2 リクエストを送信します。test_h2_009_02 では --timing-script-file を使用してリクエスト間隔を制御します。 3. **ログフラッシュ**: apache_restart() を呼び出すことで、バッファされたログをディスクに強制的に書き出します。 4. **検証**: 生成された JSON 形式のログファイルを読み込み、対象リクエストの time_taken フィールドが期待値の範囲内にあるかを確認します。

直接依存

  • pytest: テストフレームワーク。
  • env.H2Conf, env.H2TestEnv: テスト環境の構築および Apache 設定の生成に使用。
  • h2load: HTTP/2 負荷テストツール。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤クラス。H2TestEnv や H2Conf の定義が含まれます。
  • test/modules/http2/test_105_timeout.py: タイムアウト関連のテストであり、時間計測という観点で関連性が高いです。

読者向けメモ

  • 本テストは Apache のログ出力に依存しているため、テスト終了時に apache_restart() を呼び出してログをフラッシュする手順が必須となっています。
  • h2load のバージョンが古い場合（1.41.0 未満）、--connect-to オプションが使用できないためテストがスキップされます。
  • ログ解析には json.loads を使用しているため、Apache のログフォーマットが JSON 形式であることを前提としています。
