test/modules/http2/test_700_load_get.py

概要

本ファイルは、h2load を使用した HTTP/2 プロトコルの負荷試験およびストリーム制御の検証を行うためのテストスイートです。主に CGI スクリプトへの大量リクエストや、HTTP/2 のフロー制御ウィンドウサイズがパフォーマンスに与える影響を測定します。

モジュール内での役割

test/modules/http2 テストスイートの一部として、高負荷環境下でのサーバーの安定性と、HTTP/2 特有のウィンドウサイズ設定が正しく機能するかを確認する統合テストの役割を担います。

責務

  • **負荷生成**: h2load を用いて、指定された接続数や並列数で CGI スクリプトへリクエストを送信する。
  • **結果検証**: h2load の実行結果を解析し、すべてのリクエストが 2xx ステータスで正常終了したことを保証する。
  • **フロー制御の検証**: HTTP/2 のコネクションおよびストリームのウィンドウサイズを動的に変更し、サーバーの応答性をテストする。

重要なシンボル

  • TestLoadGet: テストクラス。pytest を使用して負荷試験シナリオを定義。
  • check_h2load_ok: h2load の実行結果（r）を検証するヘルパーメソッド。リクエスト数やステータスコードの整合性を確認する。
  • test_h2_700_10 / test_h2_700_11: CGI スクリプトに対する負荷試験。接続数やリクエスト数を変えて実行。
  • test_h2_700_20: ウィンドウサイズ（connbits, streambits）をパラメータ化して実行する負荷試験。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャにより、テスト対象の Apache サーバーに CGI およびテスト用 vhost を設定し、再起動を行う。 2. **実行**: 各テストメソッドが h2load コマンドの引数を構築し、env.run() を通じて外部プロセスとして実行する。 3. **検証**: check_h2load_ok が h2load の出力結果をパースし、期待されるリクエスト数と成功ステータス（2xx）が一致するかをアサートする。 4. **パラメータ化**: pytest.mark.parametrize を活用し、接続数やウィンドウサイズを変化させて同一ロジックを繰り返し実行する。

直接依存

  • pytest: テストフレームワーク。
  • .env: H2Conf（設定生成用）および H2TestEnv（環境情報・実行用）クラス。

関連コード

  • test/modules/http2/env.py: テスト環境の基盤。h2load のパスや実行メソッドが定義されている。
  • test/modules/http2/test_004_post.py: 同様に CGI を使用するテストだが、こちらは POST リクエストに焦点を当てている。

読者向けメモ

  • test_h2_700_20 は現在 skip 状態であり、将来的な mpm_event の改善を待つ必要がある点に注意してください。
  • 本テストは h2load がインストールされており、かつ --connect-to オプションをサポートしているバージョン（1.41.0 以上）であることを前提としています。
  • 負荷試験のため、実行環境の CPU やメモリリソースが不足すると、テストが不安定になる可能性があります。
