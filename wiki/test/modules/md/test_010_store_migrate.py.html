test/modules/md/test_010_store_migrate.py

概要

本ファイルは、mod_md のストレージ形式が旧バージョン（1.0）から新バージョン（1.1以降）へ正しく移行されることを検証する回帰テストです。

モジュール内での役割

test/modules/md テストスイートの一部として、ストレージの互換性と自動移行機能の堅牢性を保証します。

責務

  • 旧形式のストレージデータをテスト環境へ展開する。
  • a2md コマンドを実行し、ストレージの自動移行をトリガーする。
  • 移行前後でファイル名（pkey.pem → privkey.pem 等）が正しく更新されているかを検証する。

重要なシンボル

  • TestStoreMigrate: 移行テストをカプセル化するテストクラス。
  • test_md_010_000: 移行プロセスを実行するメインのテストメソッド。
  • env.replace_store(): テスト用の旧ストレージデータを環境に配置するヘルパー。
  • env.a2md(): mod_md の管理ツールを呼び出し、ストレージの読み込みと移行を誘発する。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャにより、テストクラス実行前に MDConf を用いて Apache を初期化。 2. **データ配置**: replace_store で data/store_migrate/1.0/ 配下の旧ファイルをストレージディレクトリにコピー。 3. **事前検証**: 移行前のファイル名（pkey.pem 等）が存在することを確認。 4. **移行実行**: a2md list コマンドを実行。この際、mod_md はストレージを読み込み、必要に応じて自動的にファイル名をリネームする。 5. **事後検証**: 移行後のファイル名（privkey.pem 等）が存在し、旧ファイル名が削除されていることを確認。

直接依存

  • pytest: テストフレームワーク。
  • os: ファイルシステムのパス操作および存在確認。
  • .md_conf.MDConf: テスト用 Apache 設定の管理。
  • .md_env.MDTestEnv: テスト環境の抽象化と a2md 実行インターフェース。

関連コード

  • test/modules/md/test_001_store.py: ストレージの基本操作に関するテスト。
  • test/modules/md/data/store_migrate/: 移行テストに使用される旧バージョンのストレージデータセット。

読者向けメモ

  • 本テストは a2md バイナリの存在に依存しており、環境にない場合はスキップされます。
  • ファイル名の変更は mod_md のストレージ管理ロジックの重要な変更点です。このテストが失敗する場合、ストレージの互換性が損なわれている可能性があります。
  • a2md の list コマンドが単なる情報取得だけでなく、内部的な移行処理のトリガーとして機能している点に注意してください。
