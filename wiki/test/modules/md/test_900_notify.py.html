test/modules/md/test_900_notify.py

概要

Apache HTTP Server の mod_md における MDNotifyCmd ディレクティブの動作を検証する統合テストスイートです。証明書の更新や管理イベント発生時に外部コマンドが正しく呼び出されるかをテストします。

モジュール内での役割

test/modules/md モジュール群の一部として、mod_md の通知機能（証明書更新完了後のフック処理）の信頼性を保証します。

責務

  • **異常系テスト**: 存在しないコマンドや、実行時に失敗するコマンドが指定された際の挙動（エラーログの生成とステータス確認）を検証する。
  • **正常系テスト**: 正当なコマンドが期待通りの引数（ドメイン名や追加引数）で呼び出されることを検証する。
  • **マルチドメイン対応**: 複数のマネージドドメイン（MD）に対して、それぞれ通知が正しく発行されるかを検証する。

重要なシンボル

  • TestNotify: テストクラス。pytest を使用し、クラススコープで ACME サーバーの起動と環境の初期化を行います。
  • configure_httpd(): テストケースごとに MDNotifyCmd を含む Apache 設定を生成し、サーバーを再起動するヘルパーメソッド。
  • test_md_900_xxx: 各テストケース。通知コマンドの成否、引数の受け渡し、複数ドメインへの通知発行を個別に確認します。

制御フローとデータフロー

1. **セットアップ**: _class_scope で ACME サーバーを起動し、_method_scope で通知ログの初期化とドメイン名の取得を行います。 2. **設定**: configure_httpd を通じて MDNotifyCmd を含む設定ファイルを生成し、Apache を再起動します。 3. **実行と待機**: env.await_completion を使用して、証明書管理プロセスが完了するまで待機します。 4. **検証**:

  • 異常系: env.get_md_status でエラーコード（AH10108 等）を確認します。
  • 正常系: 指定されたログファイルに書き込まれた引数リストを読み込み、期待値と一致するかをアサートします。

直接依存

  • pytest: テストフレームワーク。
  • os, time: ファイル操作および待機処理に使用。
  • .md_conf.MDConf: Apache 設定生成用クラス。
  • .md_env.MDTestEnv: テスト環境の管理および ACME サーバーとの通信用クラス。

関連コード

  • test/modules/md/notify.py: テスト用の通知スクリプト（通知内容をログに記録する）。
  • test/modules/md/notifail.py: 意図的に失敗する通知スクリプト。
  • test/modules/md/md_conf.py: mod_md の設定生成ロジック。

読者向けメモ

  • 本テストは MDTestEnv.has_acme_server() が真の場合のみ実行されます。
  • time.sleep(1) は非同期の通知処理がログファイルに書き込まれるのを待つための暫定的な待機です。
  • AH10108 などのエラーコードは、mod_md が通知コマンドの実行失敗を検知した際に発行する標準的なログ識別子です。
