test/modules/md/test_702_auto.py

概要

本ファイルは、Apache HTTP Server の mod_md モジュールにおける「自動駆動（Auto-drive）」機能の統合テストスイートです。ACME プロトコルを用いた証明書の自動取得、更新、および設定変更時の挙動を検証します。

モジュール内での役割

test/modules/md サブシステムの一部として、mod_md の主要な運用シナリオ（自動更新、チャレンジタイプ、プロキシ設定、ドメイン構成変更など）を網羅するテスト群を提供します。

責務

  • **自動駆動の検証**: MDDriveMode auto 設定下での証明書ライフサイクル管理。
  • **チャレンジ検証**: http-01 および tls-alpn-01 チャレンジの成功・失敗条件の確認。
  • **設定変更の追従**: ドメイン追加・削除、VirtualHost 構成変更時の mod_md の整合性維持。
  • **エラーハンドリング**: 不正な設定や通信エラー発生時の挙動（503 応答など）の確認。

重要なシンボル

テストクラス

  • TestAutov2: ACME サーバー環境下で実行されるテストクラス。pytest フィクスチャを用いて Apache の設定と再起動を自動化。

フィクスチャ（ライフサイクル管理）

  • _class_scope: クラス単位で Apache 設定の初期化、ACME サーバーの起動、ストアのクリアを行う。
  • _method_scope: 各テストメソッド実行前にストアをクリアし、テスト対象ドメインを準備する。

設定・操作ユーティリティ

  • MDConf: Apache 設定ファイル（httpd.conf）を動的に生成・インストールするヘルパー。
  • env: HttpdTestEnv インスタンス。Apache の再起動、ログ監視、証明書ストアの操作、HTTP リクエストの実行を担当。
  • MDCertUtil: 証明書の解析と検証を行うユーティリティ。

制御フローとデータフロー

1. **セットアップ**: MDConf を使用して httpd.conf を生成し、env.apache_restart() でサーバーを起動。 2. **駆動**: mod_md が設定を読み込み、ACME サーバーと通信して証明書を取得。 3. **待機**: env.await_completion() を呼び出し、証明書取得完了までポーリングを行う。 4. **検証**: env.get_cert() や env.get_md_status() を通じて、期待される証明書がストアに存在し、有効であることを確認。 5. **クリーンアップ**: env.clear_store() により、各テストケースの独立性を保証。

直接依存

  • pytest: テストフレームワーク。
  • pyhttpd.certs, pyhttpd.env: Apache テスト環境の基盤ライブラリ。
  • test.modules.md.md_conf, test.modules.md.md_env: mod_md 専用のテストヘルパー。

関連コード

  • test/modules/md/md_acme.py: ACME サーバーの制御ロジック。
  • test/modules/md/md_certs.py: 証明書管理のテストロジック。
  • test/modules/md/test_740_acme_errors.py: エラーケースのより詳細な検証。

読者向けメモ

  • **メモリとライフサイクル**: 各テストは Apache プロセスを再起動するため、実行時間は長めです。env.clear_store() はファイルシステム上の状態をリセットするため、テスト間の副作用を避けるために必須です。
  • **エラーログ**: env.httpd_error_log.ignore_recent() は、意図的に発生させたエラー（チャレンジ失敗など）をテスト結果として無視するために使用されています。
  • **制限事項**: 一部のテストは mod_ssl 固有の挙動に依存しており、skipif で制御されています。また、DNS 名の長さ制限など、ACME サーバー（Boulder/Pebble）の仕様に依存するエッジケースが含まれています。
