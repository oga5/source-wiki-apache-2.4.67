test/modules/md/test_710_profiles.py

概要

本ファイルは、Apache HTTP Server の mod_md における「MDProfile」設定の動作検証を行うテストスイートです。ACME プロトコルを通じて取得される証明書のプロファイル（有効期間や制約）が、設定通りに適用されるかを確認します。

モジュール内での役割

test/modules/md テストスイートの一部として、証明書管理の構成オプション（MDProfile）の機能テストを担当します。他のテストファイルがストレージや登録処理を検証するのに対し、本ファイルはポリシー適用に焦点を当てています。

責務

  • default プロファイルによる標準的な証明書発行の検証。
  • shortlived プロファイルによる短期有効期限証明書の発行検証。
  • 不正なプロファイル指定時の挙動（MDProfileMandatory が有効な場合の失敗処理）の検証。

重要なシンボル

  • TestProfiles: テストクラス。ACME サーバーが設定されている環境でのみ実行されます。
  • test_md_710_001〜004: 各プロファイル設定のシナリオを網羅するテストメソッド。
  • MDConf: テスト用の Apache 設定を動的に生成・インストールするためのヘルパークラス。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャにより、ACME サーバーの起動と Apache の初期設定が行われます。 2. **構成生成**: 各テストメソッド内で MDConf を使用し、特定の MDProfile を含む設定ファイルを生成します。 3. **実行と検証**:

  • env.apache_restart() で設定を反映。
  • env.await_completion() で証明書発行を待機。
  • env.get_md_status() で取得した証明書のメタデータ（有効期限やプロファイル名）を検証。

    • 4. **エラーハンドリング**: test_md_710_004 では、不正なプロファイル指定時に await_error を呼び出し、エラーログの発生と証明書発行の失敗をアサートします。

直接依存

  • pytest: テストフレームワーク。
  • pyhttpd.certs.CertificateSpec: 証明書仕様の定義。
  • pyhttpd.env.HttpdTestEnv: HTTPD テスト環境の基盤。
  • test.modules.md.md_conf.MDConf: mod_md 設定生成用ユーティリティ。
  • test.modules.md.md_env.MDTestEnv: mod_md 専用のテスト環境設定。

関連コード

  • test/modules/md/md_conf.py: 設定生成ロジックの詳細はここで定義されています。
  • test/modules/md/md_env.py: mod_md のテスト環境における前提条件やヘルパーメソッドが定義されています。

読者向けメモ

  • 本テストは外部の ACME サーバーへの接続を前提としています。環境変数や設定で ACME サーバーが利用できない場合、テストはスキップされます。
  • test_md_710_004 では、意図的にエラーを発生させているため、env.httpd_error_log.ignore_recent を使用してログのクリーンアップを行っています。デバッグ時にログを確認する際は注意してください。
