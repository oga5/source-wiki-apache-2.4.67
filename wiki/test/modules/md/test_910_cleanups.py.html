test/modules/md/test_910_cleanups.py

概要

本ファイルは、Apache mod_md におけるマネージドドメイン（MD）関連のストレージおよびチャレンジディレクトリの自動クリーンアップ機能を検証するテストスイートです。

モジュール内での役割

test/modules/md テストスイートの一部として、mod_md が不要なディレクトリや古いチャレンジデータを適切に削除（ガーベジコレクション）できるかを保証する回帰テストの役割を担います。

責務

  • **環境の初期化**: テスト実行ごとに MDTestEnv を使用してストレージをクリーンな状態にリセットする。
  • **クリーンアップ検証**: Apache の再起動や mod_md の動作に伴い、管理対象外のディレクトリが期待通りに削除されることを確認する。
  • **設定の分離**: MDConf を用いて、テストケースごとに独立した仮想ホスト設定を生成する。

重要なシンボル

  • TestCleanups: 本テストクラス。ACME サーバーが利用可能な環境でのみ実行される。
  • _class_scope: クラス単位のフィクスチャ。ACME サーバーの起動と初期設定を行う。
  • _method_scope: 関数単位のフィクスチャ。各テスト実行前にストレージをクリアし、ドメインを準備する。
  • test_md_910_01: メインのテストメソッド。チャレンジディレクトリ内に意図的に作成した「不要なディレクトリ」が、Apache 再起動後に削除されているかを検証する。

制御フローとデータフロー

1. **セットアップ**: _class_scope が Apache を起動し、_method_scope が env.clear_store() を呼び出してストレージを初期化する。 2. **テスト実行**: test_md_910_01 が MDConf を使用して設定をインストールする。 3. **汚染**: テスト対象のディレクトリ（challenges_dir）内に、mod_md が管理しないダミーディレクトリ（aaa, bbb, zzz）を作成する。 4. **検証**: env.apache_restart() を実行し、mod_md のクリーンアップ処理をトリガーする。その後、ダミーディレクトリが削除され、管理対象のドメインディレクトリのみが残っていることをアサーションで確認する。

直接依存

  • os: ファイルシステム操作（ディレクトリ作成・存在確認）に使用。
  • pytest: テストフレームワーク。
  • .md_conf.MDConf: テスト用 Apache 設定の生成と管理。
  • .md_env.MDTestEnv: テスト環境の抽象化とストレージ管理。

関連コード

  • test/modules/md/md_env.py: テスト環境の基盤。clear_store() や store_challenges() の実装が含まれる。
  • test/modules/md/md_conf.py: 設定生成ロジック。本テストでの設定構築の基礎。

読者向けメモ

  • 本テストは mod_md がファイルシステム上のディレクトリを「所有」し、管理対象外の項目を自動的に削除する挙動をテストしています。
  • env.clear_store() は非常に強力なメソッドであるため、テスト環境の汚染を防ぐために各メソッドの開始時に必ず呼び出されています。
  • pytest.mark.skipif により、ACME サーバーが設定されていない環境ではテストがスキップされます。ローカルで実行する際は環境変数の設定に注意してください。
