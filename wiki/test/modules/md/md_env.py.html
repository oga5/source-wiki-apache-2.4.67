test/modules/md/md_env.py

概要

md_env.py は、Apache mod_md モジュールの統合テスト環境を定義する基盤モジュールです。pyhttpd フレームワークを拡張し、ACME プロトコル（Pebble/Boulder）を用いた証明書管理、ローカルストレージの操作、および HTTPD のライフサイクル管理を抽象化します。

モジュール内での役割

test/modules/md ディレクトリ内の全テストケースにおける共通のテストフィクスチャおよびユーティリティを提供します。テスト実行時の環境構築、ACME サーバーとの通信設定、および mod_md が管理する証明書ストアの検証を担います。

責務

  • **環境構築**: MDTestSetup を通じた mod_md 関連モジュールのロードと Pebble 用の設定ファイル生成。
  • **ACME サーバー制御**: ACME サーバー（Pebble/Boulder）の接続状態管理と、証明書発行プロセスの待機（ポーリング）。
  • **ストア管理**: mod_md のローカルストレージ（md_store.json, domains/, accounts/ 等）の初期化、パージ、および権限検証。
  • **検証ユーティリティ**: 証明書の内容、OCSP ステータス、JSON レスポンス、およびファイルシステム上の状態を検証するヘルパーメソッドの提供。

重要なシンボル

環境定義

  • MDTestEnv: テスト環境のメインクラス。HttpdTestEnv を継承し、ACME 関連のパスや設定を保持。
  • MDTestSetup: HttpdTestSetup を継承。テスト開始時に mod_md 関連の Apache モジュールを有効化し、Pebble 用のテンプレートを配置。

ストア操作

  • clear_store() / purge_store(): テスト実行前後のクリーンアップ。
  • store_domain_file() / path_account(): ストア内の特定のファイルパスを生成するアクセサ群。
  • check_md_complete(): 指定ドメインの証明書発行が完了し、鍵と証明書が存在することを検証。

検証・待機

  • await_completion() / await_renewal(): ACME プロセスが完了するまで md-status をポーリング。
  • check_md_credentials(): 証明書の SAN、有効期限、秘密鍵との整合性を検証。
  • get_ocsp_status(): openssl s_client を用いて OCSP レスポンスを解析。

制御フローとデータフロー

1. **初期化**: MDTestEnv がインスタンス化される際、環境変数 ACME に基づいてサーバー URL を決定し、ストアディレクトリを初期化します。 2. **セットアップ**: setup_httpd() が呼ばれると、MDTestSetup が mod_md 関連の Apache 設定を生成し、サーバーを起動します。 3. **コマンド実行**: a2md() メソッドが a2md バイナリを呼び出し、ACME サーバーと対話します。引数は a2md_stdargs で事前設定されます。 4. **検証フロー**: テストは get_md_status() で JSON を取得し、check_md() や check_md_complete() で期待される状態（MD_S_COMPLETE 等）と比較します。

直接依存

  • pyhttpd.env: 基盤となるテスト環境クラス。
  • pyhttpd.certs: 証明書生成および検証ユーティリティ。
  • md_cert_util.py: 証明書のパースや検証を行うヘルパー。

関連コード

  • test/modules/md/md_cert_util.py: 証明書検証ロジックの分離先。
  • test/modules/md/test_*.py: 本モジュールを利用する各テストケース。

読者向けメモ

  • **メモリとライフサイクル**: MDTestEnv は pytest のセッションまたはモジュールスコープで生存することを想定しています。
  • **副作用**: clear_store() は物理的にディレクトリを削除するため、テスト実行順序や並列実行には注意が必要です。
  • **未実装・浅い部分**: get_apxs_var 等の外部コマンド実行は、実行環境の apxs パスに依存します。また、ACME サーバーが none の場合の挙動は限定的です。
  • **制約**: Pebble v2.5.0 以前の EAB サポートに関する制限がコード内にコメントとして残されており、特定の環境ではテストがスキップされる可能性があります。
