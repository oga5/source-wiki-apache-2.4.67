test/modules/md/test_001_store.py

概要

test_001_store.py は、mod_md のストレージ管理機能に対する統合テストスイートです。a2md バイナリを直接呼び出し、マネージドドメイン（MD）の追加、削除、更新、および一覧表示といった基本的なライフサイクル操作が正しく機能することを検証します。

モジュール内での役割

test/modules/md サブシステムにおける「ストレージ層」の検証を担当します。他のテストファイルが ACME プロトコルや証明書発行の複雑なフローを扱うのに対し、本ファイルはストレージの整合性とコマンドラインインターフェース（CLI）の基本動作を保証する基盤的な役割を担います。

責務

  • **ストレージ操作の検証**: a2md store コマンド群（add, remove, list, update）の正常系および異常系の動作確認。
  • **JSON 出力の検証**: a2md が返す JSON レスポンスの構造と内容が期待通りであることの保証。
  • **環境のクリーンアップ**: 各テストケース実行前に env.purge_store() を呼び出し、テスト間の状態汚染を防止。

重要なシンボル

  • TestStore: テストケースを保持するメインクラス。a2md が利用可能な環境でのみ実行されるよう pytest.mark.skipif が設定されています。
  • env.a2md(): a2md バイナリを呼び出し、結果を解析して返すヘルパーメソッド。
  • env.check_json_contains(): 期待する JSON スキーマと実際の出力を比較するアサーションメソッド。

制御フローとデータフロー

1. **セットアップ**: _method_scope フィクスチャが実行され、テスト環境のストレージディレクトリを初期化します。 2. **実行**: 各 test_md_001_xxx メソッドが env.a2md() を通じて CLI コマンドを発行します。 3. **検証**: コマンドの終了コード (exit_code) および標準出力の JSON を取得し、check_json_contains を用いて期待値と比較します。 4. **終了**: テスト終了時に環境がリセットされ、次のテストに備えます。

直接依存

  • pytest: テストフレームワーク。
  • re: バージョン文字列の解析やパスの検証に使用。
  • .md_env.MDTestEnv: テスト環境の構築、バイナリパスの管理、ストレージ操作の抽象化を行う基盤クラス。

関連コード

  • test/modules/md/test_010_store_migrate.py: ストレージのマイグレーションに関するテスト。本ファイルで検証する基本操作の発展形です。
  • test/modules/md/md_env.py: MDTestEnv クラスの定義。テスト実行に必要な環境変数やヘルパーメソッドの全容が記述されています。

読者向けメモ

  • 本ファイルは a2md バイナリの CLI インターフェースに対する「ブラックボックステスト」です。内部実装の詳細よりも、ユーザー（またはシステム）が期待するコマンドの振る舞いに焦点を当てています。
  • テストが失敗する場合、a2md の JSON 出力フォーマットが変更されたか、ストレージのファイルシステム操作に競合が発生している可能性が高いです。
  • env.a2md() が返す exit_code の確認は、エラーハンドリングのテストにおいて重要です。
