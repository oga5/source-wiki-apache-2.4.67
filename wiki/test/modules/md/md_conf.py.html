test/modules/md/md_conf.py

概要

md_conf.py は、mod_md のテスト環境における Apache HTTP Server の設定ファイル生成を抽象化するクラスを提供します。テストスイートが動的に設定を構築し、検証を行うための基盤となります。

モジュール内での役割

test/modules/md モジュールにおいて、テストケースが Apache の設定（httpd.conf）をプログラム的に生成するためのインターフェースとして機能します。MDTestEnv と連携し、テスト環境固有のパスやポート情報を設定に注入します。

責務

  • Apache 設定のテンプレート生成（MDCertificateAuthority 等の mod_md 固有ディレクティブの管理）。
  • テスト用仮想ホスト（vhost）やプロキシ設定の定型的な追加。
  • MDomain や MDomainSet などの設定ブロックを構築するためのヘルパーメソッドの提供。

重要なシンボル

  • MDConf: HttpdConf を継承したメインクラス。設定の追加と管理を担います。
  • __init__: テスト環境（env）に基づき、デフォルトの管理用メールアドレスや mod_md の基本設定を自動適用します。
  • add_md / start_md / end_md: MDomain ディレクティブを生成するためのメソッド群。

制御フローとデータフロー

1. **初期化**: MDTestEnv を受け取り、super().__init__ で基底クラスを初期化します。 2. **デフォルト設定**: local_ca や std_ports フラグに基づき、ACME サーバーの URL やポートマッピングを self.add() で設定バッファに追加します。 3. **動的拡張**: テストケース側から add_md などのメソッドを呼び出し、設定文字列をバッファに追記します。 4. **出力**: 最終的に HttpdConf の機能を通じて、これらの設定が実際の httpd.conf ファイルとして書き出されます。

直接依存

  • test.modules.md.md_env.MDTestEnv: テスト環境のパスやポート情報を提供。
  • pyhttpd.conf.HttpdConf: Apache 設定生成の基底クラス。

関連コード

  • test/modules/md/md_env.py: MDConf が依存する環境変数の定義元。
  • test/modules/md/test_300_conf_validate.py: MDConf を利用して設定の妥当性を検証するテストケース。

読者向けメモ

  • このクラスは mod_md のテスト専用です。本番環境の設定生成には使用しないでください。
  • MDRetryDelay や MDRenewViaARI などの設定は、テストの実行速度と安定性を優先するためにハードコードされています。
  • start_md と end_md は、<MDomain ...> ブロックを生成する際に使用します。引用符の有無など、mod_md の構文要件に注意してください。
