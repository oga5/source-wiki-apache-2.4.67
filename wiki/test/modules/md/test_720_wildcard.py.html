test/modules/md/test_720_wildcard.py

概要

Apache mod_md におけるワイルドカード証明書の取得および管理機能の統合テストスイートです。ACME プロトコルの dns-01 チャレンジを用いた証明書発行の成功・失敗シナリオを網羅しています。

モジュール内での役割

test/modules/md サブシステムの一部として、特定の機能（ワイルドカード）に焦点を当てた回帰テストを提供します。mod_md が設定ファイルや外部コマンドとどのように連携し、証明書ストアを更新するかを検証します。

責務

  • ワイルドカードドメインを含む Managed Domain (MD) の設定と検証。
  • dns-01 チャレンジの実行環境（スクリプトのパス、引数、実行結果）のテスト。
  • 設定不備（コマンド不在、実行失敗）時のエラーハンドリングの確認。
  • 証明書発行後の Apache 構成への適用と、正常な SSL 通信の確認。

重要なシンボル

  • TestWildcard: テストクラス。MDTestEnv を通じて ACME サーバーとの接続を管理します。
  • MDConf: Apache 設定ファイルを動的に生成・インストールするためのヘルパークラス。
  • env.await_completion(): 非同期の証明書発行プロセスが完了するまで待機するユーティリティ。
  • env.await_error(): 発行失敗が予期されるケースで、エラー状態を検証するためのメソッド。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィーチャが Apache を初期化し、_method_scope が各テストごとに証明書ストアをクリーンアップします。 2. **設定**: MDConf を使用して、対象ドメインとワイルドカード設定を含む httpd.conf を生成します。 3. **実行**: Apache を再起動し、mod_md が自動的に ACME チャレンジを開始するのを待ちます。 4. **検証**:

  • 成功ケース: env.await_completion() で発行完了を確認し、証明書の SAN リストを検証します。
  • 失敗ケース: env.await_error() で期待されるエラーコード（challenge-mismatch や challenge-setup-failure）が発生しているかを確認します。

直接依存

  • pytest: テストフレームワーク。
  • os: ファイルパス操作に使用。
  • .md_conf.MDConf: 設定生成用。
  • .md_env.MDTestEnv: テスト環境の制御および検証用。

関連コード

  • test/modules/md/dns01.py: テストで使用されるモックの dns-01 チャレンジ実行スクリプト。
  • test/modules/md/conftest.py: env や acme フィクスチャの定義元。

読者向けメモ

  • 本ファイルは dns-01 チャレンジの挙動を深く理解するための良いサンプルです。
  • test_md_720_008 は、dns-01 が設定されている場合に http-01 チャレンジが適切にバイパス（無視）されることを確認する重要なテストケースです。
  • テスト実行には ACME サーバー環境が必須であり、環境がない場合は pytest.mark.skipif によりスキップされます。
