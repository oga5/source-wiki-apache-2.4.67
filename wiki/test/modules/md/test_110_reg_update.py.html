test/modules/md/test_110_reg_update.py

概要

本ファイルは、mod_md における管理対象ドメイン（Managed Domain）の構成情報を更新する機能の統合テストです。ACME プロトコルに関連する設定項目（ドメインリスト、CA URL、アカウントID、連絡先、利用規約同意）の変更操作が正しく処理されるかを検証します。

モジュール内での役割

test/modules/md テストスイートの一部として、a2md コマンドラインツールを介した設定更新のバリデーションを担当します。主に test_100_reg_add.py で追加されたドメインに対し、後続の更新操作が期待通りに反映されるかを確認する役割を持ちます。

責務

  • a2md update コマンドの各サブコマンド（domains, ca, account, contacts, agreement）の正常系および異常系のテスト。
  • 入力値のバリデーション（不正なドメイン名、URL、メールアドレスの拒否）。
  • 更新後の状態（JSON 出力）が期待されるスキーマと一致することの検証。

重要なシンボル

  • TestRegUpdate: テストケースを保持するメインクラス。pytest のフィクスチャを利用して環境を初期化します。
  • env.a2md(): mod_md の管理ツールを呼び出し、結果を JSON として取得・検証するための主要なヘルパーメソッド。
  • env.check_json_contains(): 更新されたドメイン設定が期待する JSON 構造を部分的に含んでいるかを検証するユーティリティ。

制御フローとデータフロー

1. **セットアップ**: _method_scope フィクスチャが実行され、env.clear_store() で環境をリセット後、テスト対象のドメインを a2md add で登録します。 2. **実行**: 各テストメソッドが env.a2md(["update", ...]) を呼び出し、コマンドライン引数として更新内容を渡します。 3. **検証**:

  • 正常系: a2md の戻り値（JSON）を解析し、期待値と一致するかを assert で確認します。
  • 異常系: 不正な引数に対して exit_code == 1 が返されることを確認します。

    • 4. **クリーンアップ**: teardown_method が各テスト後に実行されます。

直接依存

  • pytest: テストフレームワーク。
  • .md_env.MDTestEnv: テスト環境の抽象化クラス。a2md コマンドの実行環境や ACME サーバーの接続情報を提供します。

関連コード

  • test/modules/md/test_100_reg_add.py: ドメイン登録の基本操作を扱う先行テスト。
  • test/modules/md/test_120_reg_list.py: 更新後の状態確認に関連するリスト操作のテスト。
  • test/modules/md/md_env.py: テスト環境のセットアップと a2md コマンドのラッパー定義。

読者向けメモ

  • 本テストは a2md コマンドのインターフェース仕様書としても機能します。特定の更新コマンドがどのような JSON 出力を生成するかを確認する際の参照先として適しています。
  • pytest.mark.skipif により、ACME サーバーや a2md バイナリが存在しない環境ではテストがスキップされる設計になっています。
  • 不正なメールアドレスや URL のテストケースは、RFC 準拠の厳密なバリデーションというよりは、mod_md 側の基本的な入力チェックの堅牢性を確認する目的で記述されています。
