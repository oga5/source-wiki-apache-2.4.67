test/modules/md/test_752_zerossl.py

概要

ZeroSSL ACME サービスとの統合および外部アカウントバインディング（EAB）機能の動作を検証する統合テストスイートです。

モジュール内での役割

test/modules/md モジュールの一部として、特定の CA（ZeroSSL）に対する mod_md の挙動をテストします。特に、EAB が必須となる環境での証明書発行プロセスとエラーハンドリングに焦点を当てています。

責務

  • ZeroSSL の EAB 認証情報の取得と、それを用いた証明書発行の正常系テスト。
  • EAB が欠落している場合や、不正な認証情報が提供された場合の ACME エラーハンドリングの検証。
  • 証明書更新時のドメイン追加とアカウント再利用の確認。
  • OCSP ステープリングが有効な環境での証明書検証。

重要なシンボル

  • TestZeroSSL: ZeroSSL 関連のテストケースを保持するクラス。環境変数 ZEROSSL_TLD が設定されている場合のみ実行されます。
  • get_new_eab(env): ZeroSSL の API を叩き、テストに必要な kid と hmac キーを取得するヘルパー関数。
  • MDConf: mod_md の設定を生成・適用するためのユーティリティクラス。

制御フローとデータフロー

1. **環境準備**: pytest のクラススコープフィクスチャにより、Apache の再起動と設定の初期化が行われます。 2. **EAB 取得**: 各テストケースで get_new_eab を呼び出し、ZeroSSL 用の認証情報を取得します。 3. **設定適用**: MDConf を使用して、MDExternalAccountBinding を含む Apache 設定を生成・インストールします。 4. **実行と検証**:

  • env.await_completion() で証明書発行を待機。
  • curl を使用して HTTPS 接続を試行し、証明書の有効性を確認。
  • 失敗系テストでは env.await_error() を使用し、期待される ACME エラーコード（externalAccountRequired 等）が発生するかを確認します。

直接依存

  • os, time: 環境変数の取得および待機処理に使用。
  • pytest: テストフレームワーク。
  • .md_conf.MDConf: テスト対象の Apache 設定生成用モジュール。

関連コード

  • test/modules/md/md_env.py: テスト環境の管理、Apache の制御、証明書ステータスの取得を行う基盤。
  • test/modules/md/test_751_sectigo.py: 他の CA（Sectigo）との統合テスト。比較対象として有用です。

読者向けメモ

  • **実行条件**: このテストは環境変数 ZEROSSL_TLD が設定されていない場合、自動的にスキップされます。
  • **デバッグ**: テストが失敗した場合、env.apachectl_stderr を確認することで Apache の設定エラーや起動失敗の詳細を追跡できます。
  • **注意**: ZeroSSL のデモ用エンドポイントを使用しているため、ネットワーク環境や API の可用性に依存します。
