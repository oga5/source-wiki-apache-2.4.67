test/modules/md/test_800_must_staple.py

概要

本ファイルは、Apache HTTP Server の mod_md における「OCSP Must-Staple」機能の動作検証を行うためのテストスイートです。証明書に Must-Staple 拡張が含まれるべきケースと、そうでないケースの挙動を網羅的にテストします。

モジュール内での役割

test/modules/md テストスイートの一部として、mod_md が管理する証明書の属性（Must-Staple）が、設定ファイル（MDMustStaple ディレクティブ）の変更に応じて正しく反映・更新されるかを検証します。

責務

  • MDMustStaple ディレクティブの on/off 切り替えによる証明書属性の制御。
  • 生成された証明書が期待通りの Must-Staple 拡張を持っているかの検証。
  • mod_ssl の OCSP Stapling 設定と連携した、実際の OCSP レスポンスの検証。

重要なシンボル

  • TestMustStaple: テストクラス。クラススコープで ACME 環境を初期化します。
  • configure_httpd(): 指定された設定行を注入し、Apache を再起動して環境を構築するヘルパーメソッド。
  • MDCertUtil: 証明書ファイルを解析し、Must-Staple 属性の有無を確認するためのユーティリティ。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィーチャにより ACME サーバーを起動し、テスト用ドメインを準備します。 2. **設定注入**: configure_httpd を通じて MDMustStaple 設定を Apache 構成に反映します。 3. **証明書検証**: env.await_completion で証明書発行を待ち、MDCertUtil を用いて証明書内のフラグをアサートします。 4. **OCSP 確認**: env.get_ocsp_status を呼び出し、サーバーが実際に OCSP レスポンスを返しているか（または拒否しているか）を検証します。

直接依存

  • pytest: テストフレームワーク。
  • MDConf: Apache 設定ファイルの生成と管理。
  • MDCertUtil: 証明書解析。
  • MDTestEnv: テスト環境の制御（ACME サーバーの有無、OCSP 対応状況の確認）。

関連コード

  • test/modules/md/test_801_stapling.py: OCSP Stapling 自体の詳細なテストを行う関連ファイル。
  • test/modules/md/md_cert_util.py: 証明書解析ロジックの定義元。

読者向けメモ

  • 本テストは mod_ssl の OCSP Stapling 機能に強く依存しています。環境によっては skipif によりテストがスキップされるため、実行環境の OCSP 対応状況を確認してください。
  • test_md_800_003 では、on から off への設定変更が証明書に正しく反映されるかという「更新」のライフサイクルをテストしています。これは mod_md の設定変更追従性を確認する重要なケースです。
