test/modules/md/test_202_acmev2_regs.py

概要

本ファイルは、mod_md における ACMEv2 アカウント登録および管理機能の統合テストスイートです。Apache HTTP Server の a2md 管理ツールを介して、ACME サーバーとの対話（登録、検証、削除）が正しく行われるかを検証します。

モジュール内での役割

test/modules/md モジュール群の一部として、ACME アカウントライフサイクルの基本操作をテストします。他のテストファイルが証明書発行や設定検証に焦点を当てるのに対し、本ファイルはアカウント管理（Registration）に特化しています。

責務

  • ACME アカウントの新規登録（利用規約同意の有無を含む）の検証。
  • コンタクト情報のフォーマット（mailto: プレフィックス等）の妥当性チェック。
  • アカウントの検証（validate）および削除（delreg）操作のテスト。
  • 不正なアカウント操作に対するエラーハンドリングの確認。

重要なシンボル

  • TestAcmeAcc: 本テストクラス。pytest を使用し、クラススコープで Apache の起動と環境設定を行います。
  • env.a2md(): mod_md の管理ツール a2md を実行するヘルパーメソッド。テストの主軸となります。
  • _prepare_account(): テストの前提条件としてアカウントを作成し、その ID を返すユーティリティ。
  • _check_account(): 指定されたアカウントの JSON 設定ファイルを読み込み、登録内容を検証するヘルパー。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが ACME テストサーバーを起動し、Apache を設定して再起動します。 2. **実行**: 各テストメソッドが env.a2md() を呼び出し、コマンドライン引数として ACME 操作を送信します。 3. **検証**:

  • stdout を正規表現で解析し、登録されたアカウント ID を取得します。
  • env.path_account() で生成された JSON ファイルを直接読み込み、期待値と比較します。
  • exit_code を確認し、成功・失敗の期待値と一致するかを判定します。

直接依存

  • pytest: テストフレームワーク。
  • re: アカウント ID の抽出やエラーメッセージの照合に使用。
  • json: アカウント設定ファイルの読み込みに使用。
  • .md_conf.MDConf: テスト用 Apache 設定のインストールに使用。
  • .md_env.MDTestEnv: テスト環境の構築、a2md バイナリの実行、パス管理に使用。

関連コード

  • test/modules/md/md_env.py: テスト環境の基盤。a2md の実行環境やパス解決の定義が含まれます。
  • test/modules/md/md_conf.py: Apache 設定のテンプレート管理。
  • test/modules/md/test_100_reg_add.py: 類似した登録テスト。

読者向けメモ

  • 本テストは a2md が利用可能であり、かつ ACME テストサーバー（Pebble 等）が設定されている環境でのみ実行されます。
  • test_md_202_000b のように、サーバーのレスポンス（Pebble かどうか）によってエラーメッセージの正規表現を切り替える工夫がなされています。
  • アカウントの永続化（store）が適切に行われているかを確認するため、ファイルシステム上の JSON ファイルを直接操作・検証する手法が多用されています。
