test/modules/md/dns01_v2.py

概要

ACMEプロトコルのDNS-01チャレンジをテスト環境でシミュレートするためのユーティリティスクリプトです。外部のテスト用DNSサーバー（challtestsrv）に対して、レコードの登録および削除を指示します。

モジュール内での役割

test/modules/md モジュールにおいて、ACME認証フローの検証に必要なDNSレコード操作を抽象化します。主に統合テストのセットアップおよびクリーンアップフェーズで使用されます。

責務

  • **DNSレコードのセットアップ**: 指定されたドメインに対するACMEチャレンジ用TXTレコードを challtestsrv に登録する。
  • **環境のクリーンアップ**: テスト終了後に不要なDNSレコードを削除し、環境を初期状態に戻す。
  • **CLIインターフェースの提供**: コマンドライン引数から setup または teardown を実行可能にする。

重要なシンボル

  • setup(domain, challenge): 指定ドメインのDNSレコードを構成し、チャレンジ値を設定する。
  • teardown(domain): 指定ドメインに関連するDNSレコードをクリアする。
  • run(args): subprocess を用いて curl コマンドを実行し、結果を標準出力/エラーに出力するラッパー。
  • challtestsrv: テスト用DNSサーバーの接続先（デフォルト: localhost:8055）。

制御フローとデータフロー

1. **CLI実行**: main 関数が引数を受け取り、setup または teardown を呼び出す。 2. **コマンド発行**: run 関数が curl を呼び出し、JSON形式のペイロードを challtestsrv のエンドポイント（/set-txt や /clear-txt）へ送信する。 3. **状態遷移**:

  • setup: 既存レコードを teardown で消去した後、新しいレコードを登録する。
  • teardown: 指定されたホスト名のレコードを削除する。

    • 4. **終了処理**: subprocess の戻り値に基づき、正常終了（0）または異常終了（非0）を sys.exit で呼び出し元に返す。

直接依存

  • subprocess: 外部コマンド（curl）の実行に使用。
  • sys: 引数の取得およびプロセスの終了コード制御に使用。

関連コード

  • test/modules/md/dns01.py: 本ファイルと類似の目的を持つ、あるいは前身となる実装。
  • test/modules/md/test_502_acmev2_drive.py: ACMEv2のフローテストにおいて、本スクリプトがどのように活用されているかを確認できる。

読者向けメモ

  • 本スクリプトは curl コマンドが実行環境にインストールされていることを前提としています。
  • challtestsrv は、ACMEテストスイート（Boulder等）で一般的に使用されるモックサーバーです。
  • 実行時に sys.stderr へ詳細なログが出力されるため、テスト失敗時のデバッグに役立ちます。
