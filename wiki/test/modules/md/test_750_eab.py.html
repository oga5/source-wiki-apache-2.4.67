test/modules/md/test_750_eab.py

概要

ACME プロトコルにおける「外部アカウントバインディング (EAB: External Account Binding)」機能の Apache mod_md 実装を検証するためのテストスイートです。

モジュール内での役割

test/modules/md サブシステムの一部として、ACME サーバーが要求する EAB 設定の正当性、エラーハンドリング、および設定変更時の挙動を検証します。

責務

  • EAB 設定の有無による ACME 登録の成功・失敗の検証。
  • 不正な KID (Key ID) や HMAC キー、無効な JSON 設定ファイルに対する Apache のエラー応答の確認。
  • 設定変更（EAB の追加・削除・更新）に伴う証明書更新プロセスの追跡。
  • Apache 設定ファイル (MDExternalAccountBinding) のパースおよび検証ロジックのテスト。

重要なシンボル

  • TestEab: EAB 関連テストをまとめたクラス。MDTestEnv.has_acme_eab() が真の場合のみ実行されます。
  • MDConf: テスト用 Apache 設定を生成・インストールするためのユーティリティクラス。
  • MDExternalAccountBinding: テスト対象となる Apache 設定ディレクティブ。

制御フローとデータフロー

1. **セットアップ**: 各テストメソッド実行前に env.clear_store() で環境を初期化し、MDConf を用いて Apache 設定を生成します。 2. **実行**: env.apache_restart() を呼び出し、設定を反映させます。 3. **検証**:

  • 成功ケース: env.await_completion() で証明書取得の完了を待機します。
  • 失敗ケース: env.await_error() で ACME サーバーからのエラー応答（externalAccountRequired, unauthorized 等）をキャプチャし、ログの AH10056 エラーコードと照合します。

    • 4. **クリーンアップ**: env.httpd_error_log.ignore_recent() を使用して、意図したエラーログを無視し、テストの失敗を防ぎます。

直接依存

  • pytest: テストフレームワーク。
  • MDConf: 設定管理用ヘルパー。
  • MDTestEnv: テスト環境の制御および ACME サーバーの状態確認用。
  • json, os, re: 設定ファイルの生成およびログの正規表現マッチングに使用。

関連コード

  • test/modules/md/md_conf.py: MDConf クラスの定義。
  • test/modules/md/md_env.py: MDTestEnv クラスの定義。
  • conf/pebble-eab.json: テストで使用される有効な EAB 資格情報のソース（Pebble サーバー用）。

読者向けメモ

  • 本テストは、ACME サーバーが EAB をサポートしている環境（Pebble 等）でのみ動作します。
  • test_md_750_018 から 022 は、ファイルベースの EAB 設定（JSON）の読み込みとバリデーションを網羅しています。
  • ログの AH10056 は mod_md が EAB 関連の失敗を記録する際の標準的なエラーコードです。テストコード内でこのログを明示的に無視している点に注意してください。
