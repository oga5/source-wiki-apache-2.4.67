test/modules/md/test_310_conf_store.py

概要

本ファイルは、Apache HTTP Server の mod_md モジュールにおける設定（MDomain 等）と、永続化ストア（a2md 管理下のディレクトリ）との同期および整合性を検証する統合テストスイートです。

モジュール内での役割

test/modules/md サブシステムにおいて、設定変更がどのようにストアの状態へ反映されるか、あるいは設定削除時にストアがどのように振る舞うか（保持されるか、デフォルト値へフォールバックするか）を網羅的にテストします。

責務

  • **設定の適用と検証**: MDConf を用いて Apache 設定を生成・インストールし、a2md コマンド経由でストアの状態を検証する。
  • **ライフサイクル管理**: 設定の追加、更新、削除、および再起動時の整合性確認。
  • **デフォルト値の検証**: 設定項目を削除した際に、システムが適切にデフォルト値へ復帰するかを確認する。
  • **エッジケースのテスト**: 不正なファイルがストアに混入した場合の耐性や、ストアディレクトリの切り替えテスト。

重要なシンボル

  • **TestConf クラス**: テストの基底クラス。pytest フィクスチャを用いて環境の初期化（acme.start）とクリーンアップを行う。
  • **MDConf**: テスト用の Apache 設定を構築・インストールするヘルパー。
  • **env (MDTestEnv)**: テスト環境の抽象化。a2md コマンドの実行、Apache の再起動、ストア状態の取得（check_md）を担当。
  • **a2md**: mod_md の管理ツール。ストアのリスト取得、更新、ドライブ操作を行う。

制御フローとデータフロー

1. **設定生成**: MDConf(env, text="...") で設定文字列を定義し、.install() で Apache 設定ファイルへ書き出す。 2. **適用**: env.apache_restart() を呼び出し、mod_md に設定を再読み込みさせる。 3. **検証**: env.a2md(["list"]) または env.check_md(...) を呼び出し、JSON 出力を解析して期待される状態（renew-mode, ca, privkey 等）と照合する。 4. **状態遷移**: テストケース内で設定を段階的に変更し、ストア内のデータが期待通りに更新または保持されるかを確認する。

直接依存

  • pytest: テストフレームワーク。
  • MDConf: 設定生成用ユーティリティ。
  • MDTestEnv: テスト環境制御用ユーティリティ。

関連コード

  • test/modules/md/md_conf.py: MDConf クラスの定義。
  • test/modules/md/md_env.py: MDTestEnv クラスの定義。
  • test/modules/md/test_001_store.py: ストアの基本的な操作に関するテスト。

読者向けメモ

  • **テストの前提**: a2md バイナリと ACME テストサーバーが利用可能であることが必須条件です。
  • **メモリと永続化**: このテストは実際のファイルシステム上のストアディレクトリを操作します。各テストメソッドの開始時に env.clear_store() が呼ばれ、クリーンな状態が保証されます。
  • **エラーパス**: env.apache_restart() の戻り値と env.apachectl_stderr をチェックすることで、設定エラー時の挙動を検証しています。
  • **省略箇所**: 本ファイルは設定の「ストアへの反映」に特化しており、ACME プロトコルの詳細なハンドシェイクや証明書の暗号学的な検証は他のテストファイル（test_502_acmev2_drive.py 等）に委譲されています。
