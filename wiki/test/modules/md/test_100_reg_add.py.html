test/modules/md/test_100_reg_add.py

概要

本ファイルは、mod_md（Apache HTTP Server のマネージド・ドメインモジュール）におけるドメイン登録機能（add コマンド）の統合テストスイートです。ACME プロトコルを用いたドメイン管理設定の妥当性を検証します。

モジュール内での役割

test/modules/md テストスイートの一部として、a2md 管理ツールに対するコマンドライン引数のインターフェースをテストします。設定の保存、重複チェック、バリデーションロジックが正しく動作することを保証します。

責務

  • a2md add コマンドによるマネージド・ドメインの新規登録の検証。
  • 不正なドメイン名、不正な ACME URL、重複ドメインに対するエラーハンドリングの確認。
  • 登録後の状態（state）やドメインリストの整合性チェック。

重要なシンボル

  • TestRegAdd: テストケースを保持するメインクラス。pytest のフィクスチャを利用して環境を制御します。
  • env.a2md(): a2md バイナリを呼び出し、結果を JSON として取得するヘルパーメソッド。
  • env.MD_S_INCOMPLETE: 登録直後の初期状態を示す定数。

制御フローとデータフロー

1. **セットアップ**: 各テスト関数実行前に env.purge_store() が呼び出され、テスト環境がクリーンな状態にリセットされます。 2. **実行**: env.a2md(["add", ...]) を通じて、CLI 引数が a2md ツールに渡されます。 3. **検証**:

  • 正常系: 返却された JSON 構造が期待されるドメインリストや CA 設定と一致するかを env.check_json_contains() で検証します。
  • 異常系: exit_code が 1（エラー）であることを確認します。

    • 4. **状態確認**: a2md list を実行し、内部ストアに正しく反映されているかを確認します。

直接依存

  • pytest: テストフレームワーク。
  • .md_env.MDTestEnv: テスト環境の抽象化クラス。a2md バイナリのパスや ACME サーバーの接続設定を提供します。

関連コード

  • test/modules/md/md_env.py: テスト環境の基盤。a2md の実行やストアの管理ロジックが定義されています。
  • test/modules/md/test_110_reg_update.py: 登録後の更新処理をテストする関連ファイル。

読者向けメモ

  • 本テストは a2md バイナリが利用可能であり、かつ ACME テストサーバーが設定されている環境でのみ実行されます（@pytest.mark.skipif）。
  • テストケース test_md_100_005 や test_md_100_006 はパラメータ化されており、境界値テストのパターンとして参考になります。
  • a2md の出力は JSON 形式であるため、検証には check_json_contains を用いるのが定石です。
