test/modules/md/dns01.py

概要

ACMEプロトコルのDNS-01チャレンジをテスト環境でシミュレートするためのユーティリティスクリプトです。外部のテスト用DNSサーバー（challtestsrv）に対して、レコードの登録および削除を指示します。

モジュール内での役割

test/modules/md サブシステムにおいて、ACME証明書発行フローの検証に必要なDNS環境を構築・破棄する補助ツールです。主に統合テストのセットアップおよびクリーンアップフェーズで使用されます。

責務

  • **DNSレコードの管理**: challtestsrv APIを呼び出し、指定されたドメインに対するTXTレコードの追加・削除を行う。
  • **テスト環境の同期**: チャレンジ開始前の環境リセット（teardown）と、チャレンジに必要なレコードの注入（setup）を自動化する。
  • **CLIインターフェースの提供**: テストスクリプトから直接呼び出せるよう、コマンドライン引数による操作をサポートする。

重要なシンボル

  • setup(domain, challenge): 指定ドメインの既存レコードをクリアし、ACMEチャレンジ用のTXTレコードを登録する。
  • teardown(domain): テスト終了時にDNSレコードをクリーンアップする。
  • run(args): curl コマンドをラップし、外部の challtestsrv と通信する汎用実行関数。
  • challtestsrv: テスト用DNSサーバーのエンドポイント（デフォルト: localhost:8055）。

制御フローとデータフロー

1. **CLI実行**: main 関数が引数（setup または teardown）を受け取り、対応する関数を呼び出す。 2. **通信**: run 関数が curl を使用して JSON ペイロードを challtestsrv に送信する。 3. **状態変更**:

  • setup: clear-txt で既存レコードを削除後、set-txt でドメインのAレコードとチャレンジ用のTXTレコードを登録する。
  • teardown: clear-txt を呼び出し、DNS環境を初期状態に戻す。

    • 4. **終了**: subprocess の終了コードを sys.exit に渡し、テストの成否を呼び出し元に伝える。

直接依存

  • subprocess: 外部コマンド（curl）の実行に使用。
  • sys: コマンドライン引数の取得および終了コードの制御に使用。

関連コード

  • test/modules/md/dns01_v2.py: DNS-01チャレンジの別バージョンや拡張実装が含まれる可能性があり、比較対象として参照されることがあります。
  • test/modules/md/md_acme.py: 本スクリプトがサポートするACME関連のテストロジックが定義されています。

読者向けメモ

  • 本スクリプトは curl コマンドが実行環境にインストールされていることを前提としています。
  • challtestsrv が起動していない場合、通信エラーが発生します。
  • 実行時に sys.stderr へ詳細なログが出力されるため、テスト失敗時のデバッグに役立ちます。
