test/modules/md/test_120_reg_list.py

概要

本ファイルは、mod_md における管理対象ドメイン（Managed Domains）のリスト表示機能および状態管理の検証を行うテストスイートです。a2md コマンドラインツールを介して、ストア内のドメイン情報が正しく取得・解析されるかをテストします。

モジュール内での役割

test/modules/md テストスイートの一部として、ドメイン登録後の状態確認（list コマンド）を担当します。test_100_reg_add.py で追加されたドメインが、期待通りのメタデータとステータスを保持しているかを検証する役割を担います。

責務

  • **ストアの整合性確認**: 空のストアおよび複数のドメインが登録されたストアのリスト表示の検証。
  • **状態遷移の検証**: ドメインの証明書や秘密鍵の有無、有効期限に基づき、INCOMPLETE / COMPLETE / EXPIRED といった状態が正しく反映されるかの確認。
  • **エラーハンドリング**: 不正な証明書ファイルが存在する場合のステータス変化の検証。

重要なシンボル

  • TestRegAdd: テストクラス。a2md および ACME サーバーの利用可否をチェックし、テスト実行の前提条件を制御します。
  • env.a2md(): mod_md の管理ツールを呼び出し、結果を JSON 形式で取得する主要なインターフェース。
  • env.MD_S_INCOMPLETE / env.MD_S_COMPLETE: ドメインの状態を判定するための定数。

制御フローとデータフロー

1. **初期化**: _method_scope フィクスチャにより、各テストケース実行前に env.clear_store() が呼び出され、クリーンな環境が保証されます。 2. **操作**: a2md(["add", ...]) や a2md(["update", ...]) を使用してストアの状態を構築します。 3. **検証**: a2md(["list", ...]) を実行し、返却された JSON オブジェクトを env.check_json_contains 等で検証します。 4. **状態操作**: ファイルシステム上の証明書ファイルを直接操作（copyfile や上書き）することで、a2md が外部要因による状態変化を正しく検知できるかを確認します。

直接依存

  • pytest: テストフレームワーク。
  • shutil.copyfile: 証明書ファイルのモック作成に使用。
  • .md_env.MDTestEnv: テスト環境の抽象化クラス。a2md の実行やパス管理を提供。

関連コード

  • test/modules/md/test_100_reg_add.py: ドメイン追加機能のテスト。本ファイルの前提となる操作を扱います。
  • test/modules/md/md_env.py: テスト環境の基盤。a2md のラッパーや定数定義が含まれます。

読者向けメモ

  • 本テストは a2md コマンドの出力（JSON）に強く依存しています。テストが失敗する場合、a2md の出力フォーマットが変更されていないか確認してください。
  • 証明書ファイルの直接操作を行っているため、テスト実行環境のファイルシステム権限やパス設定に注意が必要です。
  • MDTestEnv.has_a2md() が偽の場合、このファイル内の全テストはスキップされます。環境構築が正しく行われているか確認してください。
