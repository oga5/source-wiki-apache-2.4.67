test/modules/md/test_920_status.py

概要

Apache HTTP Server の mod_md におけるステータスレポート機能（JSON API および server-status 統合）を検証する統合テストスイートです。

モジュール内での役割

test/modules/md モジュール群の中で、管理対象ドメイン（Managed Domains）の証明書状態、更新プロセス、および設定の可視化機能が正しく動作しているかを保証します。

責務

  • /.httpd/certificate-status エンドポイントが返す JSON 構造の妥当性検証。
  • md-status ハンドラおよび server-status 経由でのステータス情報の取得と解析。
  • 証明書の更新中（staging）および完了後の状態遷移の追跡。
  • MDCertificateStatus 設定ディレクティブの有効/無効切り替えによる挙動確認。
  • 複数鍵タイプ（RSA/ECC）や静的証明書設定時のステータス表示の正確性検証。

重要なシンボル

  • TestStatus: テストクラス。pytest のフィクスチャを用いて各テストメソッド実行前に環境をクリーンアップします。
  • env.get_certificate_status(domain): /.httpd/certificate-status からステータスを取得するヘルパー。
  • env.get_md_status(domain): md-status ハンドラから詳細な管理情報を取得するヘルパー。
  • MDConf: テスト用の Apache 設定を構築するためのユーティリティクラス。

制御フローとデータフロー

1. **環境構築**: _class_scope フィクスチャが ACME サーバーを起動し、_method_scope がテストごとのクリーンな状態を保証します。 2. **設定適用**: MDConf を使用して、対象ドメインの mod_md 設定を生成し、Apache を再起動します。 3. **状態遷移**: env.await_completion により ACME 証明書発行プロセスを待機します。 4. **検証**:

  • 発行前後のステータス JSON を取得し、sha256-fingerprint や renewal などのキーの有無をアサーションで確認します。
  • 必要に応じて copyfile でステージング領域を操作し、外部からの証明書差し替え時の挙動をシミュレートします。

直接依存

  • pytest: テストフレームワーク。
  • pyhttpd.certs.CertificateSpec: 証明書仕様の定義。
  • test.modules.md.md_conf.MDConf: 設定生成用。
  • test.modules.md.md_env.MDTestEnv: テスト環境の制御と検証用。

関連コード

  • test/modules/md/md_env.py: テスト環境の基盤。get_certificate_status などのメソッド定義元。
  • test/modules/md/md_conf.py: mod_md の設定ディレクティブを扱うためのヘルパー。

読者向けメモ

  • 本ファイルは、mod_md が提供する監視用エンドポイントの「仕様書」としても機能します。JSON の期待されるスキーマを確認したい場合は、各テストケース内の assert 文を参照してください。
  • test_md_920_010 は、server-status の ?auto クエリによるテキスト形式のステータス出力のパース方法を示しており、監視ツール連携の実装例として参考になります。
  • 一部のテストでは、意図的にエラーログを無視する設定（httpd_error_log.ignore_recent）が含まれています。これは、証明書の検証過程で発生する既知の警告を抑制するためのものです。
