test/modules/md/test_801_stapling.py

概要

本ファイルは、Apache HTTP Server の mod_md における OCSP Stapling（証明書ステータス確認の高速化）機能の統合テストスイートです。ACME プロトコルを通じて取得した証明書や、静的に設定された証明書に対して、OCSP レスポンスの取得・キャッシュ・更新が正しく行われるかを検証します。

モジュール内での役割

test/modules/md モジュール群の一部として、mod_md の設定ディレクティブ（MDStapling 等）が期待通りに動作し、mod_ssl や他の証明書管理機能と競合・連携せずに機能することを保証します。

責務

  • MDStapling の有効/無効切り替えによる OCSP レスポンスの挙動確認。
  • MDStaplingKeepResponse や MDStaplingRenewWindow 等の更新制御ロジックの検証。
  • 複数の MDomain 設定や、仮想ホストとの組み合わせにおける整合性チェック。
  • OCSP レスポンダーが存在しない環境や、証明書に不備がある場合のフォールバック挙動の確認。

重要なシンボル

  • TestStapling: テストクラス。pytest を使用し、クラススコープで Apache のセットアップとクリーンアップを管理します。
  • configure_httpd(): テストケースごとに Apache の設定ファイルを生成するヘルパーメソッド。MDStapling 関連のディレクティブを柔軟に注入します。
  • env.get_ocsp_status() / env.await_ocsp_status(): OCSP レスポンスの状態を検証するためのテスト環境ユーティリティ。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィーチャにより、ACME サーバーの起動とテスト用ドメインの初期化が行われます。 2. **設定注入**: 各テストメソッドが configure_httpd を呼び出し、httpd.conf に MDStapling 関連の設定を書き込みます。 3. **実行と検証**:

  • apache_restart() で設定を反映。
  • env.await_ocsp_status() で OCSP レスポンスが取得されるまで待機。
  • env.get_md_status() を通じて、内部的なステータス（stapling フラグや OCSP キャッシュの状態）をアサーションします。

    • 4. **クリーンアップ**: 各テスト終了時に apache_stop() を呼び出し、環境をリセットします。

直接依存

  • pytest: テストフレームワーク。
  • pyhttpd.certs: 証明書仕様の定義。
  • .md_conf: Apache 設定生成用クラス。
  • .md_env: テスト環境の管理と状態チェック用クラス。

関連コード

  • test/modules/md/test_800_must_staple.py: 関連する Stapling 関連のテスト。
  • test/modules/md/md_conf.py: 設定生成ロジックの基盤。
  • test/modules/md/md_env.py: テスト環境の共通基盤。

読者向けメモ

  • 本テストは mod_ssl の OCSP Stapling と mod_md の Stapling が混在する環境を想定しています。
  • test_md_801_011 は、curl_multi_perform を使用した並列処理におけるバグ（Bugzilla 65567）の回帰テストとして重要です。
  • OCSP レスポンダーが利用できない環境では、多くのテストがスキップされるため、ローカル環境で実行する場合は OCSP サーバーの構成を確認してください。
