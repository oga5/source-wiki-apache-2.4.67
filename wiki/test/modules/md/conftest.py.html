test/modules/md/conftest.py

概要

test/modules/md/conftest.py は、mod_md モジュールのテストスイート全体で使用される pytest の設定および共有フィクスチャを定義するファイルです。テスト環境の構築、ACME サーバーのライフサイクル管理、およびログ設定を一元化します。

モジュール内での役割

本モジュールにおけるテスト実行の基盤です。各テストファイルが個別に環境構築を行う必要がないよう、パッケージスコープのフィクスチャを提供し、Apache HTTP Server と ACME サーバー（Pebble/Boulder）の連携を抽象化します。

責務

  • **テスト環境の初期化**: MDTestEnv を用いた Apache HTTP Server のセットアップとクリーンアップ。
  • **ACME サーバーの制御**: テスト設定に基づいた ACME サーバー（Pebble または Boulder）の起動と停止。
  • **ログの標準化**: テスト実行時のログ出力設定および、無視すべき既知の Apache エラーログ（AH10085 等）の登録。
  • **レポート出力**: pytest 実行時のヘッダー情報に、使用中の Apache バージョンや ACME サーバー情報を付与。

重要なシンボル

  • env: パッケージスコープのフィクスチャ。Apache の起動・停止を管理し、テスト環境のメタデータを提供。
  • acme: ACME サーバーのインスタンスを管理するフィクスチャ。MDPebbleRunner または MDBoulderRunner をラップ。
  • pytest_report_header: テスト実行時に環境情報を出力するためのフック関数。

制御フローとデータフロー

1. **初期化**: pytest 起動時に env フィクスチャが呼び出され、MDTestEnv が Apache をセットアップします。 2. **ログ設定**: env フィクスチャ内でロギング設定が適用され、_md_package_scope が無視するログパターンを登録します。 3. **サーバー起動**: acme フィクスチャが env.acme_server の値を確認し、対応するランナー（Pebble/Boulder）を起動します。 4. **テスト実行**: 各テストケースが env および acme フィクスチャを注入して利用します。 5. **終了処理**: テスト終了時に env が Apache を停止し、acme がサーバープロセスを終了させます。

直接依存

  • pytest: テストフレームワーク。
  • MDTestEnv (.md_env): テスト環境の構成管理クラス。
  • MDPebbleRunner, MDBoulderRunner (.md_acme): ACME サーバーの操作クラス。

関連コード

  • test/modules/md/md_env.py: テスト環境のディレクトリ構造や Apache 設定の詳細はここで定義されています。
  • test/modules/md/md_acme.py: ACME サーバーの具体的な起動・停止ロジックが実装されています。

読者向けメモ

  • 本ファイルは scope="package" を多用しており、テストスイート全体で Apache と ACME サーバーが共有されます。個別のテストでサーバーを再起動させたい場合は注意が必要です。
  • sys.path.append により、親ディレクトリをインポートパスに追加しています。これはテスト実行環境の構成に依存するため、実行時にパスが正しく解決されていることを確認してください。
