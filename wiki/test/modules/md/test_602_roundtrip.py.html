test/modules/md/test_602_roundtrip.py

概要

Apache HTTP Server の mod_md モジュールにおける、設定から証明書取得、Vhost への適用までの一連のライフサイクル（ラウンドトリップ）を検証する統合テストスイートです。

モジュール内での役割

test/modules/md テストスイートの一部として、mod_md が管理するマネージドドメイン（MD）が、設定変更や再起動を跨いで正しく状態を維持し、証明書を発行・適用できるかを保証します。

責務

  • **設定のライフサイクル検証**: MDConf を用いた設定生成、Apache の再起動、a2md コマンドによる証明書取得（drive）のシーケンスをテストする。
  • **マルチドメイン・マルチVhostの整合性**: 複数のドメインや、単一の MD が複数の Vhost をカバーする構成における証明書の正当性を検証する。
  • **ファイルシステムと権限の確認**: 生成された証明書やドキュメントルートのパーミッションが適切であるかをチェックする。

重要なシンボル

  • TestRoundtripv2: 本ファイルのメインテストクラス。a2md と ACME サーバーが利用可能な環境でのみ実行されます。
  • MDConf: テスト用の Apache 設定を構築・インストールするためのヘルパークラス。
  • env.a2md(): mod_md の管理コマンドを呼び出し、証明書取得プロセスをトリガーする主要なインターフェース。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィーチャにより、ACME サーバーの起動とテスト用設定の初期化が行われます。 2. **設定生成**: MDConf を使用して ManagedDomain を定義し、Apache 設定をインストールします。 3. **状態遷移**: apache_restart() で設定を反映させ、a2md drive コマンドで証明書の発行を試行します。 4. **検証**: 再度 apache_restart() を経て、get_cert() や get_content() を用いて、SSL 証明書が正しく適用され、HTTPS アクセスが可能であることを確認します。

直接依存

  • pytest: テストフレームワーク。
  • os: ファイルシステム操作用。
  • .md_conf.MDConf: 設定管理用クラス。
  • .md_env.MDTestEnv: テスト環境の抽象化とユーティリティ。

読者向けメモ

  • 本テストは a2md バイナリと外部 ACME サーバーの存在に強く依存しています。環境が整っていない場合、テストは自動的にスキップされます。
  • test_md_602_002 は、単一の証明書が複数の Vhost をカバーする複雑な構成を検証しており、mod_md の共有証明書機能の理解に役立ちます。
  • テスト実行時に発生する Apache のエラーログは、env.apachectl_stderr を通じてアサーション失敗時に確認可能です。
