test/modules/md/test_730_static.py

概要

本ファイルは、Apacheの mod_md における「静的証明書（Static Certificate）」設定の挙動を検証するテストスイートです。外部のACMEサーバーに依存せず、手動で用意した証明書ファイルを MDCertificateFile および MDCertificateKeyFile ディレクティブで指定した場合の動作をテストします。

モジュール内での役割

test/modules/md サブシステムの一部として、mod_md が自動管理（ACME）以外の証明書をどのように扱い、設定ミスに対してどう反応するかを検証する統合テストを提供します。

責務

  • 静的証明書設定時のApache起動および正常動作の確認。
  • MDRenewMode が always に設定された際の、静的証明書から自動管理への移行（更新）プロセスの検証。
  • 不完全な設定（証明書のみ、または鍵のみの指定）に対するApacheの起動失敗（エラーハンドリング）の検証。

重要なシンボル

  • TestStatic: テストクラス。MDTestEnv を使用して環境を構築し、各テストメソッドで異なる設定パターンを試行します。
  • test_md_730_001: 静的証明書を指定し、自動更新が行われない（renew が True だが renewal 項目がない）ことを確認するテスト。
  • test_md_730_002: MDRenewMode always を併用し、静的証明書からACMEによる自動更新へ強制的に切り替わるフローを検証するテスト。
  • test_md_730_003: 設定不備（ファイル欠損）時にApacheが正しく起動失敗することを確認するネガティブテスト。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィーチャがApache環境を初期化し、_method_scope がテストごとにストアをクリーンアップします。 2. **証明書生成**: env.create_self_signed_cert を使用して、期限切れ間近の自己署名証明書を一時ディレクトリに作成します。 3. **設定適用**: MDConf クラスを介して、Apacheの設定ファイルに MDCertificateFile 等を動的に書き込みます。 4. **検証**: env.apache_restart() または env.apache_fail() で期待される結果を確認し、env.get_md_status() で内部状態をアサーションします。

直接依存

  • pytest: テストフレームワーク。
  • pyhttpd.certs.CertificateSpec: テスト用証明書の仕様定義。
  • .md_conf.MDConf: Apache設定生成用ヘルパー。
  • .md_env.MDTestEnv: テスト環境の管理および状態取得用クラス。

読者向けメモ

  • 本テストは MDTestEnv.has_acme_server() が真の場合のみ実行されます。
  • AH10170 や AH10171 といった特定のApacheエラーログを ignore_recent で除外している箇所は、意図的な設定ミスに対する正常な挙動確認の一部です。
  • 静的証明書を使用する場合でも、mod_md の管理下にあるため、設定次第で自動更新プロセスがトリガーされる点に注意してください。
