test/modules/proxy/env.py

概要

本ファイルは、test/modules/proxy モジュールにおけるテスト環境の定義とセットアップを担う基盤クラスを提供します。Apache HTTP Server のプロキシ機能（mod_proxy 等）をテストするための環境設定をカプセル化しています。

モジュール内での役割

test/modules/proxy におけるテスト実行時の「環境構築」の役割を担います。test_01_http.py や test_02_unix.py などの各テストケースが、共通して利用するプロキシ設定やドメイン定義、証明書要件を管理する中心的な役割を果たします。

責務

  • **環境構築の定義**: HttpdTestEnv を継承し、プロキシ関連のモジュール（proxy, proxy_http, proxy_balancer 等）をロードする設定を提供。
  • **ドメイン管理**: テストで使用するフォワードプロキシ、リバースプロキシ、混合環境用のドメイン名を生成・保持。
  • **証明書管理**: プロキシテストに必要な SSL 証明書のスペック（CertificateSpec）を定義し、テスト環境へ登録。

重要なシンボル

  • ProxyTestEnv: テスト環境のメインクラス。HTTPD の設定、ログモジュールの追加、ドメイン名の管理を行う。
  • ProxyTestSetup: HttpdTestSetup を継承したセットアップクラス。テストに必要な Apache モジュールのロードを制御する。
  • d_forward, d_reverse, d_mixed: プロキシテストで利用される各ドメイン名へのアクセス用プロパティ。

制御フローとデータフロー

1. **初期化**: ProxyTestEnv がインスタンス化される際、基底クラスの初期化に加え、プロキシ関連のログモジュールと証明書スペックが登録されます。 2. **セットアップ**: テスト実行時に setup_httpd が呼び出されると、ProxyTestSetup がインスタンス化されます。 3. **モジュールロード**: ProxyTestSetup の初期化時に add_modules が実行され、Apache 側で必要なプロキシ関連モジュールが有効化されます。 4. **ドメイン解決**: テストケースは ProxyTestEnv のプロパティを通じて、動的に生成されたテスト用ドメイン名を取得します。

直接依存

  • pyhttpd.certs.CertificateSpec: SSL 証明書定義用。
  • pyhttpd.conf.HttpdConf: HTTPD 設定管理用。
  • pyhttpd.env.HttpdTestEnv, HttpdTestSetup: テスト環境構築の基底クラス。

関連コード

  • test/modules/proxy/test_01_http.py: 本ファイルで定義された環境を利用する HTTP プロキシテスト。
  • test/modules/proxy/test_02_unix.py: 本ファイルで定義された環境を利用する Unix ドメインソケットプロキシテスト。
  • test/modules/proxy/conftest.py: pytest のフィクスチャとして ProxyTestEnv を提供する設定。

読者向けメモ

  • 本ファイルは pyhttpd フレームワークの拡張実装です。プロキシ関連のテストを追加する際は、まず ProxyTestEnv のドメイン定義やモジュールリストを確認してください。
  • add_modules に指定されているモジュールは、Apache の実行時にロードされる必須コンポーネントです。テストが失敗する場合、ここに必要なモジュールが漏れていないか確認してください。
