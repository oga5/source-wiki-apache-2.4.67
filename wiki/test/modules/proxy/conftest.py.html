test/modules/proxy/conftest.py

概要

本ファイルは test/modules/proxy ディレクトリ配下のテストスイート全体で共有される pytest の設定およびフィクスチャを定義する構成ファイルです。

モジュール内での役割

プロキシ機能のテスト実行環境（Apache HTTP Server）のライフサイクル管理と、テスト実行時のメタデータ提供を担います。各テストファイルが共通して利用する env フィクスチャの起点となります。

責務

  • **環境の初期化とクリーンアップ**: ProxyTestEnv を用いた Apache のセットアップ、ログの初期化、およびテスト終了時のプロセス停止。
  • **テストレポートの拡張**: pytest_report_header を介した、実行環境の Apache バージョンおよびパス情報の出力。
  • **ロギング設定**: テスト実行時の標準出力に対するロギングレベルとフォーマットの適用。

重要なシンボル

  • env: package スコープで定義されたフィクスチャ。テスト環境のセットアップ済みインスタンスを提供します。
  • pytest_report_header: テスト実行開始時に環境情報を出力するためのフック関数。
  • _stop_package_scope: autouse=True で設定されたフィクスチャ。パッケージ内の全テスト終了後に Apache を停止させます。

制御フローとデータフロー

1. **初期化**: pytest 実行時、env フィクスチャが呼び出されると ProxyTestEnv がインスタンス化され、Apache の起動とログのクリアが行われます。 2. **実行**: 各テストケースは env を引数として受け取り、設定済みの Apache インスタンスに対してリクエストを送信します。 3. **終了**: パッケージ内の全テストが完了すると、_stop_package_scope がトリガーされ、env.apache_stop() が実行されてプロセスが終了します。

直接依存

  • pytest: テストフレームワークの基盤。
  • .env.ProxyTestEnv: テスト対象の Apache 環境を抽象化したクラス。
  • logging, os, sys: 標準ライブラリ。パス操作およびログ出力に使用。

関連コード

  • test/modules/proxy/env.py: ProxyTestEnv クラスの定義元。環境構築の詳細ロジックが含まれます。
  • test/modules/proxy/test_01_http.py / test_02_unix.py: 本ファイルで定義された env フィクスチャを利用する具体的なテストケース。

読者向けメモ

  • 本ファイルは package スコープのフィクスチャを使用しているため、このディレクトリ内のテストは Apache プロセスを共有して実行されます。
  • sys.path.append により、テスト実行時にルートディレクトリがパスに追加されます。環境構築時にパス解決で問題が発生した場合は、この設定箇所を確認してください。
  • ログ設定が logging.getLogger('').addHandler でグローバルに行われているため、テスト実行時のログ出力が冗長になる可能性があります。デバッグ時はこの設定を調整してください。
