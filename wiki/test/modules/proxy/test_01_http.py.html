test/modules/proxy/test_01_http.py

概要

HTTP プロキシ機能（リバースプロキシ、フォワードプロキシ、およびロードバランサ）の動作を検証するための統合テストスイートです。Apache HTTP Server の mod_proxy モジュールの設定とリクエスト処理の整合性を確認します。

モジュール内での役割

test/modules/proxy モジュールにおける主要なテストファイルの一つです。env.py で定義されたテスト環境を利用し、実際の HTTP リクエストを通じてプロキシの挙動を検証します。

責務

  • **プロキシ設定の構築**: HttpdConf を用いて、リバース、フォワード、および混在（mixed）プロキシの仮想ホスト設定を生成・適用する。
  • **リクエスト検証**: curl を介してプロキシ経由のリクエストを実行し、期待されるレスポンス（ホスト名やドキュメントルートの整合性）をアサートする。
  • **ロードバランサのテスト**: balancer:// スキームを用いたプロキシ設定の動作確認。

重要なシンボル

  • TestProxyHttp: テストクラス。クラススコープのフィクスチャで Apache の設定を一度だけ行い、テスト効率を向上させています。
  • _class_scope: pytest.fixture。テスト開始時に HttpdConf を使用して、リバース/フォワード/混在の各プロキシ設定を Apache に適用します。
  • test_proxy_01_001 / 002 / 003: 個別のテストケース。それぞれ異なるプロキシ構成とリクエストパターンを検証します。

制御フローとデータフロー

1. **セットアップ**: _class_scope が実行され、env オブジェクトからポートやドメイン情報を取得して HttpdConf を構築し、Apache を再起動します。 2. **テスト実行**: 各テストメソッドが env.curl_get を呼び出し、プロキシ経由でターゲットへリクエストを送信します。 3. **検証**: レスポンスのステータスコードおよび JSON レスポンス内の host フィールドをチェックし、プロキシが正しくルーティングを行っているかを確認します。 4. **動的設定**: test_proxy_01_003 では、テストメソッド内で動的に HttpdConf を再生成し、ロードバランサ設定を追加して再起動するフローをとります。

直接依存

  • pytest: テストフレームワーク。
  • pyhttpd.conf.HttpdConf: Apache 設定生成用ユーティリティ。
  • env (fixture): テスト環境の構成情報（ポート、ドメイン、証明書パスなど）を提供。

関連コード

  • test/modules/proxy/env.py: テスト環境の基盤。env フィクスチャの定義元。
  • test/modules/proxy/test_02_unix.py: Unix ドメインソケットを用いたプロキシテストの別実装。

読者向けメモ

  • test_proxy_01_002 は curl v8.0.0 以上を要求します。環境のバージョンが古い場合はスキップされる仕様です。
  • ProxyPreserveHost や ProxyRequests といった Apache のプロキシディレクティブが、どのように HttpdConf を通じて適用されているかに注目してください。
  • テスト実行時に Apache の再起動が発生するため、テストケースの追加時には設定の競合に注意が必要です。
