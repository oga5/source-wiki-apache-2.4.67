test/modules/http1/test_007_strict.py

概要

Apache HTTP Server の HTTP/1.1 プロトコルにおける「厳格な構文解析（Strict Parsing）」およびヘッダー処理の挙動を検証するテストスイートです。

モジュール内での役割

test/modules/http1 モジュールの一部として、プロトコル準拠性の境界条件をテストします。特に、不正なリクエスト形式やヘッダー値の正規化ルールが、期待通りにエラー（400 Bad Request等）を返すか、あるいは適切に処理されるかを検証します。

責務

  • **プロトコル厳格性の検証**: HttpProtocolOptions Strict 設定下での不正なリクエストラインやヘッダーの拒否確認。
  • **ヘッダー値の正規化テスト**: Header add ディレクティブにおける空白文字（スペース、タブ）のトリミングや、不正な文字が含まれる場合の挙動確認。
  • **環境の再構成**: テストケースごとに Apache の設定を動的に書き換え、再起動を行うことで隔離されたテスト環境を提供。

重要なシンボル

  • TestRequestStrict: 本ファイルのメインテストクラス。
  • _class_scope: pytest.fixture。クラス内の全テスト実行前に HttpProtocolOptions Strict を適用し、Apache を再起動します。
  • test_h1_007_01: 生のソケット通信を用いて、不正な HTTP リクエストに対するサーバーの応答（ステータスコードや接続切断）を検証します。
  • test_h1_007_02 / test_h1_007_03: curl を利用し、ヘッダー値の正規化ルール（先頭/末尾の空白除去など）を検証します。

制御フローとデータフロー

1. **セットアップ**: H1Conf を使用して Apache の設定ファイルを生成し、env.apache_restart() でサーバーを適用状態にします。 2. **リクエスト送信**:

  • test_h1_007_01 は socket を直接操作し、不正なバイト列をサーバーに送りつけます。
  • test_h1_007_02/03 は env.curl_get を経由して、設定されたヘッダー値が期待通りに正規化されているかを確認します。

    • 3. **検証**: 応答のステータスコードやヘッダー内容を assert 文で比較し、期待値と一致するかを判定します。

直接依存

  • pytest: テストフレームワークおよびパラメータ化機能。
  • socket: 低レイヤーの HTTP リクエスト送信に使用。
  • re: 応答メッセージの解析に使用。
  • .env.H1Conf: テスト環境の設定管理クラス。

関連コード

  • test/modules/http1/env.py: テスト環境の基盤。H1Conf や env フィクスチャの定義元。
  • test/modules/http1/conftest.py: テスト実行に必要な共有フィクスチャの定義。

読者向けメモ

  • 本ファイルは、Apache の mod_http における厳格な構文解析ルールを網羅する目的で作成されています。
  • test_h1_007_01 の status パラメータが None の場合、サーバーが応答を返さずに接続を閉じることを期待しています。
  • ヘッダー値のテストでは、%{unescape:...} などの式評価が含まれるケースがあり、エラーログの監視（ignore_recent）が必要な場合があります。
