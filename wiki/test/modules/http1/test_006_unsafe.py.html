test/modules/http1/test_006_unsafe.py

概要

Apache HTTP Server の HttpProtocolOptions Unsafe 設定下における、HTTP リクエストのパース挙動を検証するテストスイートです。

モジュール内での役割

test/modules/http1 モジュールの一部として、HTTP プロトコルの厳密性（Strictness）が緩和された環境でのサーバーの堅牢性を確認します。

責務

  • HttpProtocolOptions Unsafe を有効化した Apache 環境の構築。
  • 多種多様な不正・境界値リクエスト（不正な文字、異常なヘッダー、無効な URI 形式など）に対するサーバーの応答検証。
  • 期待される HTTP ステータスコードや、接続切断の挙動の網羅的なテスト。

重要なシンボル

  • TestRequestUnsafe: テストクラス。クラススコープで HttpProtocolOptions Unsafe を適用します。
  • test_h1_006_01: パラメータ化されたテストメソッド。intext（リクエスト文字列）、status（期待値）、lognos（無視するエラーログ）を引数に取ります。
  • H1Conf: env.py 由来のヘルパークラス。Apache の設定変更と再起動を管理します。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが H1Conf を使用して Apache を Unsafe モードで再起動します。 2. **リクエスト送信**: test_h1_006_01 が socket を介して intext をサーバーに送信し、即座に送信側をシャットダウンします。 3. **応答検証**:

  • status が None の場合: サーバーが接続を閉じる（応答が空である）ことを確認します。
  • status が 1 の場合: 成功（200番台以上）を期待します。
  • status が数値の場合: 応答の HTTP ステータスコードが期待値と一致するか検証します。

    • 4. **ログ処理**: lognos が指定されている場合、検証後に該当するエラーログを無視リストに追加します。

直接依存

  • pytest: テストフレームワークおよびパラメータ化機能。
  • socket: 低レイヤーの HTTP 通信に使用。
  • re: 応答行のステータスコード抽出に使用。
  • .env.H1Conf: テスト環境の設定管理。

関連コード

  • test/modules/http1/env.py: テスト環境の基盤。env フィクスチャの定義元。
  • test/modules/http1/test_007_strict.py: 本テストの対となる、厳密なパース設定下でのテスト。

読者向けメモ

  • 本ファイルは、HTTP 仕様の境界線や、過去の脆弱性に関連するエッジケースを網羅しています。
  • status パラメータの定義（1 は成功、0 は任意の HTTP エラー、None は接続切断）を理解することが、テストケース追加の鍵となります。
  • socket.sendall を使用した直接的なバイト列送信を行っているため、プロトコルレベルの挙動を正確に再現しています。
