test/modules/http1/test_005_trailers.py

概要

HTTP/1.1 プロトコルにおける「トレーラー（Trailers）」フィールドの送受信を検証するテストスイートです。Apache HTTP Server がトレーラーを正しく処理し、クライアントへ伝播できるかをテストします。

モジュール内での役割

test/modules/http1 モジュールの一部として、特定のプロトコル機能（トレーラー）の回帰テストを担当します。他のテストファイルが基本的な GET/POST を扱うのに対し、本ファイルはより高度なヘッダー制御に焦点を当てています。

責務

  • Apache HTTP Server のトレーラーサポート機能の検証。
  • 直接的なリクエストと、プロキシ経由のリクエストの両方でトレーラーが保持されることの確認。
  • サーバーバージョン（2.5.0以上）の要件チェック。

重要なシンボル

  • TestTrailers: テストクラス。クラススコープで Apache の設定（vhost_cgi）を初期化します。
  • test_h1_005_01: 直接アクセス時のトレーラー受信テスト。
  • test_h1_005_02: プロキシ経由でのトレーラー受信テスト。
  • H1Conf: テスト環境の構成を管理するヘルパークラス。

制御フローとデータフロー

1. **セットアップ**: _class_scope フィクスチャが H1Conf を使用して CGI を有効化し、Apache を再起動します。 2. **実行**: 各テストメソッドは env.curl_upload を呼び出し、Add-Trailer ヘッダーを付与してリクエストを送信します。 3. **検証**: レスポンスオブジェクト r.response["trailer"] を解析し、期待されるトレーラー値が含まれているかをアサーションで確認します。

直接依存

  • os: ファイルパス操作に使用。
  • pytest: テストフレームワーク。
  • .env.H1Conf: テスト環境設定用ユーティリティ。

関連コード

  • test/modules/http1/env.py: env フィクスチャの定義元。テスト環境の共通設定や curl ラッパーが含まれます。
  • test/modules/http1/conftest.py: テスト実行に必要な共有フィクスチャの定義。

読者向けメモ

  • 本テストは Apache 2.5.0 以降の機能を前提としています。それ以前のバージョンでは pytest.skip によりスキップされます。
  • トレーラーの検証には、トレーラーを適切に扱えるクライアント（curl 等）が必須です。
  • Add-Trailer ヘッダーは、テスト対象の CGI スクリプトに対してトレーラーを生成させるためのトリガーとして機能しています。
