test/modules/http1/test_001_alive.py

概要

HTTP/1.1 モジュールの基本的な疎通確認を行うためのテストスイートです。サーバーが正常に起動し、リクエストに対して期待通りの JSON レスポンスを返せるかを検証します。

モジュール内での役割

test/modules/http1 テストスイートの最初の検証ステップです。他の複雑な HTTP 機能（POST や Trailers 等）をテストする前に、サーバー環境が正しく構築され、リクエストを受け付けられる状態にあることを保証する「スモークテスト」の役割を担います。

責務

  • テスト用仮想ホスト（vhost）の構成とサーバーの再起動。
  • 指定されたエンドポイント（/alive.json）に対する HTTP GET リクエストの実行。
  • レスポンスのステータスコードおよび JSON コンテンツの整合性検証。

重要なシンボル

  • TestBasicAlive: テストクラス。クラススコープで環境構築を行います。
  • _class_scope: pytest.fixture。テスト実行前に H1Conf を使用して仮想ホストをインストールし、Apache を再起動します。
  • test_h1_001_01: 疎通確認を行うメインのテストメソッド。

制御フローとデータフロー

1. **セットアップ**: _class_scope が実行され、env オブジェクトを介して H1Conf が仮想ホストを設定し、サーバーを再起動します。 2. **リクエスト生成**: env.mkurl を使用してテスト対象の URL を構築します。 3. **実行**: env.curl_get を呼び出し、外部プロセスとして curl を実行してレスポンスを取得します。 4. **検証**: 取得した r.response["json"] を解析し、alive フラグと host 名が期待値と一致するかをアサートします。

直接依存

  • pytest: テストフレームワーク。
  • .env.H1Conf: HTTP/1.1 テスト用の環境設定クラス。

関連コード

  • test/modules/http1/env.py: env オブジェクトの定義元。サーバー制御やリクエスト実行の基盤機能が実装されています。
  • test/modules/http1/conftest.py: テスト全体で共有される env フィクスチャの定義が含まれています。

読者向けメモ

  • 本ファイルは、HTTP/1.1 モジュールのテスト環境が正しくセットアップされているかを確認する最もシンプルなケースです。
  • env.curl_get は内部で curl コマンドを呼び出すため、実行環境に curl がインストールされていることが前提となります。
  • テストが失敗する場合、まずは env.apache_restart() の戻り値や、サーバーのログを確認してください。
