test/modules/http1/htdocs/cgi/upload.py

概要

HTTP/1.1 テストスイートにおけるファイルアップロードおよび削除機能を実装した CGI スクリプトです。標準的な HTTP POST リクエストを処理し、サーバー上の指定ディレクトリに対するファイルの保存・削除を行います。

モジュール内での役割

test/modules/http1/htdocs/cgi モジュールの一部として、サーバーのファイル操作能力を検証するためのエンドポイントを提供します。主に requestparser.py が解析したリクエストデータを受け取り、動的なレスポンスを生成する役割を担います。

責務

  • **ファイルアップロード処理**: multipart/form-data 形式のリクエストからファイルを取り出し、安全なパスへ保存する。
  • **ファイル削除処理**: 指定されたファイル名を検証し、サーバー上のファイルを削除する。
  • **CGI レスポンス生成**: HTTP ステータスコード（200, 201, 404）および HTML コンテンツを標準出力へ書き出す。

重要なシンボル

  • get_request_params(): requestparser.py からインポートされる関数。リクエストのフォームデータとアップロードファイルを分離して取得します。
  • fitem.save_to(fpath): アップロードされたファイルオブジェクトのメソッド。指定したパスへデータを書き込みます。
  • os.environ: 環境変数（DOCUMENT_ROOT, REQUEST_SCHEME, HTTP_HOST）を参照し、動的なパスや URL を構築します。

制御フローとデータフロー

1. **初期化**: get_request_params() を呼び出し、リクエストの解析結果を forms と files に格納します。 2. **条件分岐**:

  • files に 'file' キーが存在する場合: ファイルを保存し、201 Created を返します。
  • forms に 'remove' キーが存在する場合: 指定されたファイルを削除し、結果に応じたステータスを返します。
  • いずれでもない場合: アップロード用の HTML フォームを表示します。

    • 3. **出力**: 各処理の結果を HTTP ヘッダーと HTML 本文として標準出力へ送出します。

直接依存

  • os: ファイルシステム操作（os.remove）および環境変数取得に使用。
  • sys: 標準ライブラリ。
  • requestparser: 同一ディレクトリ内のモジュール。リクエスト解析の抽象化層。

関連コード

  • test/modules/http1/htdocs/cgi/requestparser.py: 本スクリプトが依存するリクエスト解析ロジックの実装。
  • test/modules/http1/htdocs/cgi/hello.py: 同一ディレクトリ内のシンプルな CGI サンプル。

読者向けメモ

  • **セキュリティ**: os.path.basename を使用してディレクトリトラバーサル攻撃を防止していますが、本スクリプトはテスト環境用です。本番環境での利用には適していません。
  • **環境依存**: DOCUMENT_ROOT などの環境変数が CGI 実行環境で正しく設定されていることを前提としています。
  • **デバッグ**: 動作確認時は、スクリプトが実行されるディレクトリに files/ ディレクトリが存在し、書き込み権限があることを確認してください。
