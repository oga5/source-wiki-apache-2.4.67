test/modules/http1/test_004_post.py

概要

HTTP/1.1 プロトコルにおける POST メソッドの動作検証を行うテストスイートです。特に、クライアントからサーバーへのファイルアップロード機能と、その整合性確認に焦点を当てています。

モジュール内での役割

test/modules/http1 モジュールの一部として、HTTP POST リクエストの正常系および特定のヘッダー（Expect: 100-continue）を伴う通信の信頼性を検証します。

責務

  • Apache サーバー環境への CGI アップロード用 vhost の設定とデプロイ。
  • 様々なサイズのデータ（1k〜1M）を用いたアップロードテストの実行。
  • アップロード後のリソース取得（GET）による、送信データと受信データの完全一致の検証。

重要なシンボル

  • TestPost: テストクラス。クラススコープで環境設定を一度だけ行います。
  • curl_upload_and_verify: アップロードと検証を統合したヘルパーメソッド。curl を使用してアップロードを行い、レスポンスヘッダーから取得した URL を通じて内容を照合します。
  • _class_scope: pytest フィクスチャ。H1Conf を用いて CGI 環境を構築し、サーバーを再起動します。

制御フローとデータフロー

1. **初期化**: _class_scope が実行され、H1Conf を通じて CGI アップロード用の仮想ホストが構成されます。 2. **実行**: 各 test_h1_004_xx メソッドが curl_upload_and_verify を呼び出します。 3. **アップロード**: curl コマンドで指定ファイルを送信し、サーバーからのレスポンス（Location ヘッダー）を待ちます。 4. **検証**: レスポンスの Location URL に対して GET リクエストを発行し、ローカルのソースファイルとダウンロードしたボディをバイナリ比較します。

直接依存

  • pytest: テストフレームワーク。
  • env.H1Conf: HTTP/1.1 テスト環境の構成管理クラス。
  • os, inspect: ファイルパスの解決とディレクトリ操作に使用。

関連コード

  • test/modules/http1/env.py: テスト環境の基盤となる env オブジェクトの定義。
  • test/modules/http1/conftest.py: env フィクスチャの提供元。

読者向けメモ

  • 本テストは curl コマンドの実行結果に強く依存しています。テストが失敗する場合は、環境内の curl の挙動や、CGI スクリプト（upload.py）の配置を確認してください。
  • test_h1_004_05 では Expect: 100-continue ヘッダーを明示的に付与しており、HTTP/1.1 の期待値制御の挙動を確認する重要なケースとなっています。
