test/modules/http1/env.py

概要

HTTP/1.1 プロトコルテストのためのテスト環境構築および設定管理を行うクラス群を定義します。Apache HTTP Server (httpd) のテストフレームワークを拡張し、HTTP/1.1 特有のテストケースに必要なモジュールやデータセットを準備します。

モジュール内での役割

test/modules/http1 モジュールにおける「環境構築の基盤」です。テスト実行前に必要な Apache モジュールのコンパイル、設定ファイルの生成、およびテスト用データの作成を担います。

責務

  • **テスト用モジュールのビルド**: mod_h1test を動的にコンパイルし、httpd 設定にロードする。
  • **テストデータの生成**: HTTP リクエスト/レスポンスの検証に使用する各種サイズ（1k〜1m）のダミーファイルを生成する。
  • **環境設定の拡張**: HttpdTestEnv を継承し、HTTP/1.1 テストに必要なログレベルや CGI 設定を適用する。

重要なシンボル

  • H1TestSetup: テスト環境のセットアップクラス。make() メソッドでビルドとデータ生成を順次実行する。
  • H1TestEnv: テスト実行環境の定義。setup_httpd をオーバーライドし、H1TestSetup を利用するように構成する。
  • H1Conf: テスト用の Apache 設定を生成するクラス。LogLevel や SetHandler などのテスト固有設定を注入する。

制御フローとデータフロー

1. **初期化**: H1TestEnv がインスタンス化され、setup_httpd が呼び出される。 2. **ビルド**: H1TestSetup.make() が apxs を使用して mod_h1test.c をコンパイルし、modules.conf に LoadModule 行を追記する。 3. **データ準備**: _setup_data_1k_1m() が env.gen_dir 配下にテスト用ファイルを生成する。 4. **設定適用**: H1Conf が HttpdConf を継承し、base 設定や cgi 関連のディレクティブをマージして最終的な設定ファイルを構築する。

直接依存

  • pyhttpd.env: HttpdTestEnv, HttpdTestSetup（基底クラス）
  • pyhttpd.conf: HttpdConf（設定管理の基底）
  • subprocess: テスト用モジュールのコンパイル実行に使用。

関連コード

  • mod_h1test/: H1TestSetup がコンパイル対象とする C ソースコードディレクトリ。
  • test/modules/http1/conftest.py: 本ファイルで定義された H1TestEnv を利用して pytest のフィクスチャを提供する。

読者向けメモ

  • 本ファイルはテスト実行時の「環境構築」に特化しています。テストロジックそのものは test_*.py ファイル群に記述されています。
  • mod_h1test のコンパイルには apxs がシステムにインストールされている必要があります。
  • _setup_data_1k_1m で生成されるファイルは、HTTP のチャンク転送や大きなペイロードのテストに使用されます。
