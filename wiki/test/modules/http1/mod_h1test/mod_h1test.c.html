test/modules/http1/mod_h1test/mod_h1test.c

概要

HTTP/1.1 プロトコルのテストを目的とした Apache HTTP Server 用の最小限のテストモジュールです。主にリクエストボディをそのままレスポンスとして返す「エコー」機能を提供し、HTTP リクエスト処理の検証に使用されます。

モジュール内での役割

Apache のハンドラメカニズムを利用して、特定のハンドラ名（h1test-echo）に対するリクエストをインターセプトします。HTTP プロトコルスタックの動作確認や、トレーラーヘッダーの伝搬テストなど、プロトコルレベルの挙動を検証するための補助的な役割を担います。

責務

  • **リクエストのインターセプト**: h1test-echo ハンドラ名を持つリクエストを検出し、処理を実行する。
  • **ボディのエコーバック**: クライアントから送信されたリクエストボディを読み取り、そのままレスポンスとしてストリーミングする。
  • **トレーラーヘッダーの検証**: 入力されたトレーラーヘッダーを検出し、特定のレスポンスヘッダーを付与することで、トレーラーの処理能力をテストする。

重要なシンボル

  • h1test: モジュール定義構造体。Apache のモジュールロード時に参照されます。
  • h1test_echo_handler: リクエストを処理するメインのハンドラ関数。
  • h1test_hooks: モジュールのフック登録関数。ap_hook_handler を使用してハンドラを登録します。

制御フローとデータフロー

1. **登録**: h1test_hooks が起動時に h1test_echo_handler をハンドラとして登録します。 2. **判定**: h1test_echo_handler がリクエストを受け取り、ハンドラ名が h1test-echo かつメソッドが GET/POST であるかを確認します。 3. **ストリーミング**: ap_get_client_block をループで呼び出し、リクエストボディをバッファ経由で apr_brigade_write に渡します。 4. **出力**: ap_pass_brigade を使用して、蓄積されたブリゲードをフィルタチェーンへ流し込みます。 5. **トレーラー処理**: リクエストにトレーラーが含まれている場合、特定のレスポンスヘッダーを r->trailers_out にセットします。

直接依存

  • **APR (Apache Portable Runtime)**: apr_optional.h, apr_strings.h, apr_bucket_brigade 等のメモリ管理および I/O 抽象化レイヤー。
  • **Apache HTTP Server Core**: httpd.h, http_protocol.h, http_request.h 等、リクエスト処理とフィルタリングに必要なコア API。
  • apr_cstr.h
  • apr_optional_hooks.h
  • apr_time.h
  • apr_want.h
  • http_log.h

関連コード

  • http_protocol.h: ap_get_client_block や ap_pass_brigade などのリクエスト処理 API の定義元。
  • mod_http: HTTP/1.1 プロトコルを実装する標準モジュール。本モジュールはこれら標準機能のテスト用として機能します。

読者向けメモ

  • 本モジュールはプロダクション環境用ではなく、あくまでテスト用です。
  • r->clength = -1 および r->chunked = 1 を設定することで、チャンク転送エンコーディングを強制しています。
  • エラーハンドリングは最小限であり、主に apr_brigade の操作とフィルタチェーンへの受け渡しに焦点を当てています。
