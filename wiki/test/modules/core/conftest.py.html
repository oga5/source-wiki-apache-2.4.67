test/modules/core/conftest.py

概要

test/modules/core モジュールにおけるテスト実行環境のセットアップとライフサイクル管理を行う pytest 設定ファイルです。

モジュール内での役割

本モジュール内の各テストケースが共有するテスト環境（CoreTestEnv）の初期化と、Apache HTTP Server のプロセス管理を担う基盤です。

責務

  • **環境の初期化**: CoreTestEnv をインスタンス化し、Apache のセットアップおよびログの初期化を行う。
  • **ライフサイクル管理**: パッケージスコープで Apache プロセスを起動し、テスト終了時に確実に停止させる。
  • **診断情報の提供**: pytest 実行時のレポートヘッダーに、使用する Apache のバージョンや MPM モジュール情報を出力する。

重要なシンボル

  • env(pytestconfig): テスト全体で共有される CoreTestEnv インスタンスを提供するパッケージスコープのフィクスチャ。
  • _stop_package_scope(env): autouse フィクスチャ。テスト終了時に Apache を停止させるクリーンアップ処理を担当。
  • pytest_report_header: pytest 実行開始時に環境情報を表示するためのフック関数。

制御フローとデータフロー

1. **初期化**: pytest 実行時、env フィクスチャが呼び出されると CoreTestEnv が生成され、setup_httpd() により Apache が起動されます。 2. **ログ準備**: env フィクスチャ内でロギング設定が行われ、テスト実行中のログ出力が標準出力に流れるよう構成されます。 3. **実行**: 各テストケースは env フィクスチャを介して Apache の状態や設定にアクセスします。 4. **終了**: 全テスト終了後、_stop_package_scope が yield 後の処理を実行し、apache_stop() を呼び出してプロセスを終了させます。

直接依存

  • pytest: テストフレームワークの基盤。
  • .env.CoreTestEnv: 本モジュール固有のテスト環境定義。
  • pyhttpd.env.HttpdTestEnv: Apache HTTP Server の制御を行う基底環境クラス。
  • logging, os, sys: 標準ライブラリ。パス操作およびログ出力に使用。

関連コード

  • test/modules/core/env.py: CoreTestEnv クラスの定義。環境固有のロジックの詳細はこちらを参照。
  • test/modules/core/test_*.py: 本ファイルで定義された env フィクスチャを利用する具体的なテストケース群。

読者向けメモ

  • 本ファイルは scope="package" を使用しているため、このディレクトリ内のテストは単一の Apache プロセスを共有します。
  • sys.path.append により、テスト実行時にプロジェクトルートがパスに追加されます。環境構築時にパス解決で問題が発生した場合は、この箇所を確認してください。
  • ログ設定が env フィクスチャ内でグローバルに行われるため、テスト実行時のログ出力レベルはここで制御されています。
