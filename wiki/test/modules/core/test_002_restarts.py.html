test/modules/core/test_002_restarts.py

概要

本ファイルは、Apache HTTP Server のプロセス管理（MPM）における再起動およびリロード時の子プロセス生成・終了挙動を検証するテストスイートです。h2load を用いた負荷環境下で、サーバーの動的なスケーリングと世代交代が正しく行われるかをログ解析を通じて確認します。

モジュール内での役割

test/modules/core モジュールの一部として、サーバーのコア機能である「プロセスライフサイクル管理」の堅牢性を検証します。他のテストがエンコーディング等の静的な検証を行うのに対し、本ファイルは時間経過を伴う動的な状態遷移を扱います。

責務

  • **負荷生成**: Loader クラスによる並列リクエストの実行。
  • **ログ解析**: ChildDynamics クラスによる error_log からのプロセス生成・終了イベントの抽出と時系列解析。
  • **ライフサイクル検証**: 設定変更やリロードに伴う子プロセスの世代交代（Generation）が期待通りに行われるかの検証。

重要なシンボル

  • Loader: h2load をラップし、バックグラウンドで負荷をかけ続けるためのヘルパークラス。
  • ChildDynamics: ログファイルをパースし、正規表現を用いて子プロセスの PID、世代、アクション（started/stopped）を抽出する解析器。
  • TestRestarts: pytest クラス。STRESS_TEST 環境変数と h2load のバージョンチェックを前提条件として実行されます。

制御フローとデータフロー

1. **セットアップ**: HttpdConf を用いて、少ないスレッド数でプロセスが頻繁に生成されるようなタイトな設定を投入します。 2. **負荷開始**: Loader が別スレッドで起動し、CGI スクリプトに対してリクエストを送信します。 3. **動的監視**: ChildDynamics がログを読み込み、初期状態（世代0）のプロセス生成を検知します。 4. **リロード検証**: env.apache_reload() を実行後、一定時間待機し、世代0のプロセスが停止し、新しい世代のプロセスが生成されているかをログから照合します。 5. **終了**: loader.join() で負荷を終了させ、最終的なプロセス停止を確認します。

直接依存

  • os, re, time, threading: 標準ライブラリ。
  • pytest: テストフレームワーク。
  • .env.CoreTestEnv: テスト環境の抽象化。
  • pyhttpd.conf.HttpdConf: Apache 設定生成用ユーティリティ。

関連コード

  • test/modules/core/env.py: テスト環境の基盤設定。
  • test/modules/core/conftest.py: テスト実行に必要なフィクスチャ定義。
  • pyhttpd パッケージ: Apache 制御用の基盤ライブラリ。

読者向けメモ

  • **タイミング依存**: 本テストは time.sleep() を使用しており、環境負荷によってはテストが不安定になる可能性があります。
  • **ログ解析の前提**: ChildDynamics は mpm_event のログフォーマットに強く依存しています。ログ出力形式が変更された場合、正規表現の修正が必要です。
  • **実行条件**: STRESS_TEST 環境変数が設定されていない場合、テストはスキップされます。CI環境で実行する際は注意してください。
