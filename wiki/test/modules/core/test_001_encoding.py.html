test/modules/core/test_001_encoding.py

概要

Apache HTTP Server の URL エンコーディングおよびパス正規化機能の堅牢性を検証するテストスイートです。特に、パストラバーサル攻撃や不正なエンコーディングに対するサーバーの応答を網羅的にテストします。

モジュール内での役割

test/modules/core モジュールの一部として、HTTP サーバーのコア機能である「リクエストパスの解釈」の正当性を保証します。他のテストが機能の正常系を扱うのに対し、本ファイルは境界条件やセキュリティに関連するエッジケースに焦点を当てています。

責務

  • URL エンコードされたパス（%2e, %2f 等）の正規化処理の検証。
  • パストラバーサル（..）を含むリクエストに対する適切な拒否（400エラー）の確認。
  • AllowEncodedSlashes 設定の有効・無効による挙動の差異の検証。
  • 期待されるエラーログ（AH10244 等）の発生確認と、テスト実行時のログ無視設定。

重要なシンボル

  • TestEncoding: テストクラス。pytest フィクスチャを使用して Apache の設定をクラススコープで管理します。
  • _class_scope: Apache の設定（HttpdConf）を生成し、テスト対象の仮想ホスト（test1, test2）を構築してサーバーを再起動するフィクスチャ。
  • test_core_001_01 ～ test_core_001_20: 各種エンコーディングパターンを検証するテストメソッド。

制御フローとデータフロー

1. **セットアップ**: _class_scope が実行され、env を通じて Apache の設定ファイルが生成されます。ここで test2 ホストに対して AllowEncodedSlashes on が設定されます。 2. **テスト実行**: 各テストメソッドが pytest.mark.parametrize を使用して、多様なパス文字列を env.curl_get に渡します。 3. **検証**: サーバーからのレスポンスステータスコードを検証し、必要に応じて env.httpd_error_log.ignore_recent を呼び出して、意図したエラーログをテストの失敗として扱わないように制御します。

直接依存

  • pytest: テストフレームワーク。
  • pyhttpd.conf.HttpdConf: Apache 設定生成用クラス。
  • env (フィクスチャ): テスト環境の抽象化（URL 生成、Apache 操作、ログ管理）。

関連コード

  • test/modules/core/env.py: テスト環境の基盤。env フィクスチャの定義元。
  • test/modules/core/conftest.py: テスト全体で共有される設定やフィクスチャ。

読者向けメモ

  • 本テストは、Apache のセキュリティ設定（特にパスの正規化）が意図通りに機能しているかを検証するものです。
  • test_core_001_20 は、AllowEncodedSlashes 設定がホストごとに異なる場合に、エンコードされたスラッシュ（%2f）がどのように扱われるかを比較する重要なテストケースです。
  • ログの無視（ignore_recent）は、意図的にエラーを発生させるテストケースにおいて、テストスイート全体がエラーログによって「失敗」と判定されるのを防ぐために必須です。
