test/modules/core/env.py

概要

test/modules/core/env.py は、test/modules/core モジュールにおけるテスト環境の構成と初期化を定義するファイルです。HTTPサーバーのテスト実行に必要な環境変数やモジュールのロード設定を管理します。

モジュール内での役割

本モジュールにおける「テスト実行基盤」の役割を担います。pytest 実行時に読み込まれる環境設定のカスタマイズポイントであり、他のテストケースが共通して利用する HTTP サーバーの構成を規定します。

責務

  • **環境のカスタマイズ**: HttpdTestEnv を継承し、core および http モジュールのログ出力を有効化する。
  • **サーバー構成の定義**: CoreTestSetup を通じて、テストに必要なソースディレクトリの追加や cgid モジュールのロードを自動化する。
  • **セットアップの統合**: setup_httpd メソッドをオーバーライドし、テスト環境構築時に自動的に CoreTestSetup が適用されるように制御する。

重要なシンボル

  • CoreTestEnv: テスト環境のメインクラス。pytest の設定を受け取り、サーバーのログレベルやモジュール構成を初期化する。
  • CoreTestSetup: サーバーのセットアップクラス。テスト対象のディレクトリ構造や、依存する Apache モジュール（cgid）を登録する。

制御フローとデータフロー

1. **初期化**: CoreTestEnv がインスタンス化される際、親クラスの初期化とともに http および core モジュールのログ設定が追加されます。 2. **セットアップ**: テスト実行プロセスが setup_httpd を呼び出すと、CoreTestSetup がインスタンス化されます。 3. **構成適用**: CoreTestSetup の初期化時に inspect モジュールを使用して自身のソースディレクトリを特定し、add_source_dir でテスト対象として登録します。その後、add_modules(["cgid"]) により必要なサーバーモジュールがロードリストに追加されます。

直接依存

  • pyhttpd.env: 基底クラスである HttpdTestEnv および HttpdTestSetup を提供。
  • inspect, logging, os: 標準ライブラリ。パスの解決やログ出力に使用。

関連コード

  • test/modules/core/conftest.py: 本ファイルで定義された CoreTestEnv を利用して、pytest のフィクスチャとして環境を注入する役割を持つ。
  • pyhttpd パッケージ: 本ファイルが継承している基底クラス群の定義元。

読者向けメモ

  • 本ファイルは pyhttpd フレームワークの拡張パターンに従っています。新しいテストモジュールを追加する際は、このファイルをテンプレートとして環境設定を継承・拡張するのが定石です。
  • inspect.getfile を使用してディレクトリを動的に取得しているため、ファイル構成を変更する際は注意が必要です。
