test/pyhttpd/env.py

概要

test/pyhttpd/env.py は、Apache HTTP Server のテスト環境を構築・管理するための中心的なモジュールです。テスト実行に必要なディレクトリ構造の生成、設定ファイルのテンプレート展開、モジュールのロード設定、およびテスト用クライアント（curl, nghttp, h2load）の実行制御を担います。

モジュール内での役割

test/pyhttpd サブシステムにおける「司令塔」です。他のモジュール（certs.py, nghttp.py, result.py など）を統合し、テストケースが Apache インスタンスと対話するための統一的なインターフェースを提供します。

責務

  • **環境構築**: テスト用ディレクトリ（gen/）の作成、設定ファイルの生成、証明書の自動発行。
  • **Apache ライフサイクル管理**: apachectl を介したサーバーの起動、停止、再起動、および生存確認。
  • **設定管理**: config.ini の読み込みと、実行時の動的な設定（modules.conf の生成など）の適用。
  • **クライアント実行**: curl 等のコマンド実行結果の解析、ヘッダーの抽出、およびテスト結果の構造化。

重要なシンボル

環境管理

  • HttpdTestEnv: テスト環境のグローバル状態を保持するクラス。ポート番号、パス、モジュールリスト、証明書仕様を管理。
  • HttpdTestSetup: 環境構築ロジックを分離したクラス。make() メソッドでディレクトリ、設定、モジュール、ドキュメントルートを初期化。

実行・制御

  • HttpdTestEnv.run(): subprocess をラップし、コマンド実行結果を ExecResult として返す汎用メソッド。
  • HttpdTestEnv.apache_stop() / apache_restart(): サーバーのプロセス制御。
  • HttpdTestEnv.curl_raw(): curl を実行し、ヘッダーファイルを解析してレスポンスを構造化する主要な通信メソッド。

ユーティリティ

  • HttpdTestEnv.is_live() / is_dead(): サーバーの稼働状態をポーリングで確認する監視関数。

制御フローとデータフロー

1. **初期化**: HttpdTestEnv が config.ini を読み込み、パスやポートを確定。 2. **構築**: setup_httpd() が HttpdTestSetup を呼び出し、conf/ テンプレートを gen/ 下に展開。 3. **設定**: install_test_conf() により、テスト固有の設定を test.conf に書き込み、Apache を再起動。 4. **通信**: curl_raw() が curl_complete_args() で引数を生成し、curl_parse_headerfile() で一時ファイルからレスポンスヘッダーを解析して ExecResult に格納。

直接依存

  • configparser: 設定ファイルの読み込み。
  • subprocess: 外部コマンド（httpd, curl, nghttp）の実行。
  • string.Template: 設定ファイルのテンプレート展開。
  • test.pyhttpd 内の各モジュール (certs, log, nghttp, result)。

関連コード

  • test/pyhttpd/certs.py: 証明書発行ロジック。
  • test/pyhttpd/result.py: コマンド実行結果のデータ構造。
  • test/pyhttpd/nghttp.py: HTTP/2 関連のテスト補助。

読者向けメモ

  • **メモリとライフサイクル**: HttpdTestEnv はテストスイート全体で共有されることを想定しています。gen/ ディレクトリはテスト実行ごとにクリーンアップされる前提で設計されています。
  • **エラーパス**: apachectl の失敗は ExecResult を通じて通知されますが、設定ファイルの生成失敗などは Exception をスローします。
  • **制約**: curl のヘッダー解析は一時ファイル（curl.headers.*）に依存しています。並列実行時には _curl_headerfiles_n の競合に注意が必要です。
  • **省略箇所**: Dummy クラスはパス解決のためのプレースホルダーであり、機能的な意味はありません。また、h2load の統計解析ロジックは正規表現ベースの簡易的な実装です。
