test/pyhttpd/result.py

概要

test/pyhttpd/result.py は、テスト実行時のコマンド実行結果を保持・管理するためのデータクラス ExecResult を提供します。外部プロセス（curl や nghttp など）の実行結果を構造化し、後続のテスト検証で利用しやすい形式に変換します。

モジュール内での役割

test/pyhttpd サブシステムにおいて、各テストツール（curl.py, nghttp.py 等）が実行したコマンドの出力をラップする共通のコンテナとして機能します。

責務

  • **結果の正規化**: プロセスの終了コード、標準出力、標準エラー出力、実行時間を保持する。
  • **JSON 解析**: 標準出力が JSON 形式である場合、自動的にパースして json プロパティ経由でアクセス可能にする。
  • **メタデータの集約**: テスト実行に関連するレスポンス情報、追加の検証結果、アセット情報を動的に保持する。

重要なシンボル

  • ExecResult: 本ファイルの核となるクラス。実行結果の保持とアクセサを提供。
  • ExecResult.json: 標準出力を JSON としてパースした結果を返すプロパティ。パース失敗時は None を返す。
  • ExecResult.add_results(): テスト実行後の追加情報を統合するためのメソッド。

制御フローとデータフロー

1. **初期化**: ExecResult インスタンス生成時に、コマンド引数、終了コード、出力ストリームが渡されます。 2. **パース処理**: コンストラクタ内で標準出力（stdout）のデコードと JSON パースを試行し、成功すれば _json_out に格納します。 3. **データ拡張**: テスト実行後、add_results や add_assets を通じて、解析済みのレスポンスや関連アセットがインスタンスに注入されます。 4. **アクセス**: テストコードはプロパティ経由で、生のバイト列、デコード済み文字列、またはパース済み JSON オブジェクトにアクセスします。

直接依存

  • json: 標準出力の JSON パースに使用。
  • datetime.timedelta: コマンド実行時間の計測に使用。
  • typing: 型ヒントの定義に使用。

関連コード

  • test/pyhttpd/curl.py: ExecResult を生成する主要なクライアントの一つ。
  • test/pyhttpd/nghttp.py: 同様にコマンド実行結果を ExecResult としてラップするクライアント。

読者向けメモ

  • 本クラスは、コマンド実行結果の「入れ物」として設計されています。
  • __repr__ メソッドが実装されており、デバッグ時に print(result) を実行すると、終了コードと標準出力・エラー出力が整形されて表示されます。
  • JSON パースは try-except で囲まれており、パース失敗時も例外を投げずに None を返す安全な設計になっています。
