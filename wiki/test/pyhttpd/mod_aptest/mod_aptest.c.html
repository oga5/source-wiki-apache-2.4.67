test/pyhttpd/mod_aptest/mod_aptest.c

概要

mod_aptest.c は、Apache HTTP Server のテスト自動化を支援するための最小限の診断用モジュールです。HTTP リクエストのヘッダーを監視し、特定のテスト用ヘッダーが存在する場合にそのリクエスト内容をログ出力します。

モジュール内での役割

test/pyhttpd サブシステムの一部として、Python ベースのテストフレームワークから Apache の内部動作を検証するための「観測点」を提供します。

責務

  • **リクエスト監視**: post_read_request フックを利用し、リクエスト処理の初期段階で特定のヘッダーを検査する。
  • **診断ログの生成**: AP-Test-Name ヘッダーが含まれるリクエストを検出し、リクエスト行の内容をログに記録する。
  • **透過的な動作**: 自身の処理後は DECLINED を返し、後続のモジュールや本来の処理を阻害しない。

重要なシンボル

  • aptest: モジュール定義構造体。Apache のモジュールロード時に参照される。
  • aptest_hooks: モジュールのフック登録関数。post_read_request を登録する。
  • aptest_post_read_request: リクエスト処理のフック関数。ヘッダーの抽出とログ出力を行う。

制御フローとデータフロー

1. **初期化**: Apache 起動時、aptest_hooks が呼び出され、ap_hook_post_read_request に aptest_post_read_request が登録される。 2. **リクエスト受信**: HTTP リクエストが到着すると、post_read_request フェーズで aptest_post_read_request が実行される。 3. **ヘッダー検査**: r->headers_in から AP-Test-Name を取得し、存在すれば ap_log_rerror でログを記録する。 4. **制御の返却**: 常に DECLINED を返し、リクエスト処理を次のハンドラへ引き継ぐ。

直接依存

  • **APR (Apache Portable Runtime)**: apr_optional.h, apr_strings.h 等。文字列操作やオプション機能の利用。
  • **Apache Core**: httpd.h, http_protocol.h, http_request.h。リクエスト構造体 (request_rec) およびフック機構へのアクセス。
  • apr_cstr.h
  • apr_optional_hooks.h
  • apr_want.h
  • http_log.h

関連コード

  • http_request.h: post_read_request フックの定義と request_rec 構造体の詳細。
  • http_log.h: ログ出力関数 (ap_log_rerror) の仕様。

読者向けメモ

  • 本モジュールは実運用環境での使用を想定していません。テスト環境において、特定のテストケースが正しくリクエストを送信しているかを確認するためのデバッグツールです。
  • DECLINED を返しているため、このモジュールが有効であっても通常の HTTP 処理には影響を与えません。
  • ログを確認する際は、Apache のログレベル設定が info 以上（または trace1）になっていることを確認してください。
