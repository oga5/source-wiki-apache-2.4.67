test/pyhttpd/conf.py

概要

Apache HTTP Server (httpd) のテスト用設定ファイルをプログラムから動的に生成するためのユーティリティクラスを提供します。

モジュール内での役割

test/pyhttpd モジュールにおいて、テスト環境（env.py）と連携し、テスト実行時に必要な httpd.conf を構築する役割を担います。

責務

  • **設定の抽象化**: VirtualHost や Proxy などの複雑な設定ディレクティブをメソッド呼び出しで生成。
  • **TLS/SSL 変換**: mod_tls (rustls) 向けの設定を、mod_ssl や mod_gnutls 向けに自動変換する互換レイヤーを提供。
  • **設定のインストール**: 生成した設定行を HttpdTestEnv を通じてファイルシステムへ書き出す。

重要なシンボル

  • HttpdConf: 設定生成のメインクラス。設定行の保持、インデント管理、変換ロジックをカプセル化。
  • add(): 設定行を追加する汎用メソッド。mod_tls 以外の環境では自動的にディレクティブを変換。
  • start_vhost() / end_vhost(): VirtualHost ブロックの開始と終了を管理。
  • replaceinstr(): mod_tls 固有のディレクティブを mod_ssl 等の標準的な形式へ変換する内部ロジック。

制御フローとデータフロー

1. **初期化**: HttpdTestEnv を受け取り、extras で指定された追加設定を読み込む。 2. **構築**: add() や add_vhost() を呼び出し、内部リスト _lines に設定行を蓄積。この際、mod_tls が無効な場合は replaceinstr() が呼び出され、ディレクティブが書き換えられる。 3. **インストール**: install() を呼び出すと、蓄積された _lines が env.install_test_conf() に渡され、実際のテスト用設定ファイルとして出力される。

直接依存

  • typing: 型ヒント用。
  • pyhttpd.env.HttpdTestEnv: テスト環境情報およびモジュール構成の判定に使用。

関連コード

  • test/pyhttpd/env.py: HttpdConf が依存する環境設定の定義。
  • test/modules/tls/: mod_tls 関連のテストケース。HttpdConf の変換ロジックがどのように利用されるかの実例。

読者向けメモ

  • **自動変換の注意**: add() メソッドは HttpdTestEnv.has_shared_module("tls") をチェックし、TLS モジュールがない場合にのみ replaceinstr() を実行します。この挙動により、テスト環境の構成に応じて自動的に設定が最適化されます。
  • **インデント**: _indents プロパティにより、VirtualHost 内部のディレクティブに対して自動的にインデントが付与されます。
  • **拡張性**: merge_extras 静的メソッドを使用することで、複数の設定辞書を安全に結合可能です。
