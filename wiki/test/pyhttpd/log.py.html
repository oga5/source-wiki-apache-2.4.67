test/pyhttpd/log.py

概要

Apache HTTP Server のエラーログファイルを監視・解析するためのユーティリティクラス HttpdErrorLog を提供します。テスト実行中に発生した予期せぬエラーや警告を検出し、特定のログ出力を無視（除外）する機能を提供します。

モジュール内での役割

test/pyhttpd モジュールにおいて、テストの成否判定を補完する役割を担います。テストケース実行前後のログ状態を比較し、意図しないエラーが発生していないかを検証するために使用されます。

責務

  • **ログの追跡**: 指定されたログファイルの読み込み開始位置を保持し、差分のみを解析する。
  • **フィルタリング**: 特定の正規表現や AHxxxxx 形式のログ番号（APLOGNO）に基づき、既知の警告やエラーを無視する。
  • **検証**: テスト中に発生した「見逃された（missed）」エラーや警告を抽出する。
  • **待機**: 特定のパターンがログに出現するまで一定時間ポーリングを行う。

重要なシンボル

  • HttpdErrorLog: ログファイルを管理するメインクラス。
  • ignore_recent(): 意図的に発生させたエラーを「捕捉済み」としてマークし、検証対象から除外する。
  • get_missed(): 最後にログをスキャンして以降、捕捉されていないエラーと警告のリストを返す。
  • scan_recent(): 指定したパターンがログに出現するまで待機する（タイムアウト付き）。

制御フローとデータフロー

1. **初期化**: インスタンス生成時にログファイルの末尾位置を _start_pos として記録し、以降の読み込み基準とします。 2. **無視リストの更新**: add_ignored_matches 等で除外条件を設定します。 3. **捕捉処理**: ignore_recent を呼び出すと、現在のファイル位置から末尾までを走査し、条件に合致する行を _caught_ 系のセットに格納します。 4. **検証処理**: get_missed を呼び出すと、_start_pos から現在位置までを再走査し、_caught_ セットに含まれない行をエラー/警告として抽出します。 5. **状態更新**: 走査完了後、_start_pos を最新のファイル末尾に更新し、捕捉セットをリセットします。

直接依存

  • os, re, time, datetime, io (標準ライブラリのみ)

関連コード

  • test/pyhttpd/env.py: テスト環境の構成情報を提供しており、ログファイルのパス解決に利用されることが想定されます。
  • test/pyhttpd/conf.py: HTTPD の設定を管理しており、ログ出力レベルの制御と密接に関連します。

読者向けメモ

  • 本クラスは、ログファイルが追記型であることを前提としています。clear_log() を呼び出すとファイルが切り詰められるため、テストのセットアップ時に注意が必要です。
  • scan_recent はポーリングベースの同期処理であるため、タイムアウト設定はテストの実行環境に合わせて調整してください。
  • AHxxxxx 形式のログ番号によるフィルタリングは、Apache の標準的なログフォーマットに依存しています。
