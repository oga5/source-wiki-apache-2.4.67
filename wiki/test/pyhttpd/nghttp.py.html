test/pyhttpd/nghttp.py

概要

nghttp コマンドラインツールを Python から制御し、HTTP/2 通信のテストを実行するためのラッパーモジュールです。

モジュール内での役割

test/pyhttpd サブシステムにおいて、HTTP/2 クライアントとしての振る舞いをシミュレートする役割を担います。主に curl モジュールと並び、サーバーのレスポンス検証やストリーム制御のテストに使用されます。

責務

  • nghttp コマンドの引数生成と実行の抽象化。
  • nghttp -v 出力の解析による、HTTP/2 フレーム（HEADERS, DATA, PUSH_PROMISE）の構造化。
  • ファイルアップロードや POST リクエストのための、一時ファイルを用いたデータ準備。
  • 実行結果の ExecResult オブジェクトへの集約。

重要なシンボル

  • Nghttp: メインクラス。テスト対象の URL やオプションを管理し、実行を制御します。
  • parse_output(btext): nghttp の冗長な出力を解析し、ストリームごとのヘッダーやボディを辞書形式で抽出する中核メソッドです。
  • get_stream(streams, sid): ストリーム ID に基づき、解析中のストリーム状態を初期化または取得します。
  • _run(args): subprocess.run を呼び出し、実行時間や結果をラップする低レベル実行メソッドです。

制御フローとデータフロー

1. **初期化**: Nghttp インスタンスに実行パスや接続先を設定。 2. **実行**: get, post_data, upload 等のメソッドが _raw を呼び出し、_baserun を経由して nghttp を起動。 3. **解析**: _raw は parse_output を呼び出し、nghttp の標準出力（バイト列）を正規表現で走査。 4. **構造化**: parse_output 内でストリームごとのヘッダーやデータ長を streams 辞書に蓄積し、最終的に ExecResult に付与して返却。

直接依存

  • re: 出力解析のための正規表現処理。
  • subprocess: nghttp コマンドの外部実行。
  • urllib.parse: URL の分解と再構築。
  • .result.ExecResult: 実行結果の格納用データクラス。

関連コード

  • test/pyhttpd/result.py: 実行結果を保持する ExecResult クラスが定義されています。
  • test/pyhttpd/curl.py: HTTP/1.1 や一般的な HTTP テストを行うための並行ツールです。

読者向けメモ

  • **解析の限界**: parse_output は正規表現ベースの簡易的なパーサーです。nghttp の出力形式が変更されると壊れやすいため、複雑なプロトコル検証には注意が必要です。
  • **一時ファイル**: post_data や upload_file は TMP_DIR に一時ファイルを生成します。テスト実行環境の権限とクリーンアップに留意してください。
  • **ストリームの識別**: HTTP/2 のストリーム ID を追跡しており、特に奇数 ID のメインストリームのレスポンスを優先的に抽出するロジックが含まれています。
