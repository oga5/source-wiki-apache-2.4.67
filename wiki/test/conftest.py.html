test/conftest.py

概要

test/conftest.py は、Apache HTTP Server のテストスイートにおける pytest の設定および拡張機能を提供する構成ファイルです。テスト実行環境の初期化、カスタムコマンドライン引数の定義、およびテスト実行時のメタデータ管理を行います。

モジュール内での役割

本モジュールは、Python ベースのテスト環境と C 言語で記述された HTTP Server のテスト対象を橋渡しする役割を担います。他の C 言語ソースファイルが個別のテストケースを実装するのに対し、本ファイルはテスト実行基盤としての共通設定を提供します。

責務

  • **環境情報の提供**: テスト実行時に HTTP Server のバージョンや MPM（Multi-Processing Module）情報をレポートヘッダーに付与する。
  • **テスト実行制御**: --repeat オプションによるテストの反復実行機能の提供。
  • **テストコンテキストの管理**: 各テストケースの実行前後に環境変数（テスト名）を自動的に設定・解除するフックの提供。

重要なシンボル

  • pytest_report_header: テスト実行開始時に環境情報を出力するためのフック。
  • pytest_addoption: テスト実行時に使用可能なカスタム引数を定義するフック。
  • pytest_generate_tests: テストのパラメータ化を動的に制御し、反復実行を実現する。
  • _function_scope: 全てのテスト関数に自動適用されるフィクスチャ。テスト実行中の環境状態を管理する。

制御フローとデータフロー

1. **初期化**: sys.path を操作し、pyhttpd モジュールをインポート可能にする。 2. **設定**: pytest_addoption により、ユーザーが指定した --repeat 値を pytest の設定オブジェクトに格納する。 3. **パラメータ化**: pytest_generate_tests がテスト関数に repeat フィクスチャが存在するかを確認し、指定回数分だけテストをパラメータ化して実行キューに追加する。 4. **実行時フック**: _function_scope フィクスチャが各テストの開始時に env.set_current_test_name を呼び出し、終了時にクリアする。

直接依存

  • sys, os: パス操作および環境設定用。
  • pytest: テストフレームワーク本体。
  • pyhttpd.env.HttpdTestEnv: HTTP Server の環境情報（バージョン、MPM 等）を取得するためのクラス。

関連コード

  • test/ ディレクトリ内の各 test_*.c ファイル: 本ファイルで定義された環境設定を利用して実行されるテストケース群。
  • pyhttpd/env.py (推測): HttpdTestEnv が定義されているモジュール。テスト環境の抽象化層。

読者向けメモ

  • 本ファイルは pytest の標準的な conftest.py の仕組みを利用しています。テスト実行時に環境情報が正しく表示されない場合は、HttpdTestEnv の初期化処理を確認してください。
  • _function_scope は autouse=True で設定されているため、全てのテスト関数で暗黙的に実行されます。デバッグ時には、このフィクスチャがテストのセットアップ・ティアダウンに影響を与えている点に留意してください。
