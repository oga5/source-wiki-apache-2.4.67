os/bs2000/os.c

概要

本ファイルは、BS2000（OSD/POSIX）環境における Apache HTTP Server のプロセス生成およびユーザー権限管理を抽象化する OS 固有の実装を提供します。

モジュール内での役割

os/bs2000 モジュールの一部として、標準的な Unix の fork() では不十分な BS2000 特有のジョブ環境切り替え（ufork()）をラップし、サーバーが適切な権限で動作するためのブリッジとして機能します。

責務

  • **プロセス生成の抽象化**: 実行モード（デバッグか通常か）や現在の権限に基づき、適切なプロセス生成手法（fork() または ufork()）を選択する。
  • **ジョブ環境の初期化**: CGI スクリプトなどが BS2000 ネイティブコマンドを実行する際に必要な、ユーザー ID に基づくジョブ環境の切り替え準備を行う。
  • **プラットフォーム固有の制約管理**: 特権ユーザー（uid=0）での実行時と非特権ユーザーでの実行時で、プロセス生成の挙動を制御する。

重要なシンボル

  • bs2_ForkType: プロセス生成手法を決定するための列挙型（noFORK, FORK, UFORK）。
  • os_forktype(): 現在の実行環境に基づき、最適な bs2_ForkType を判定・キャッシュする関数。
  • os_fork(): os_forktype の結果に基づき、実際のプロセス生成を行うラッパー関数。
  • os_init_job_environment(): ユーザー切り替え時のジョブ環境設定を管理する関数。

制御フローとデータフロー

1. **初期化**: os_forktype() が最初に呼び出された際、one_process フラグと getuid() の結果から forktype を静的変数にキャッシュします。 2. **プロセス生成**: os_fork() が呼び出されると、キャッシュされた forktype を参照します。

  • bs2_FORK: 標準の fork() を実行。
  • bs2_UFORK: ユーザー名を大文字に変換し、BS2000 特有の ufork() を実行してユーザー ID を切り替えます。

    • 3. **エラーハンドリング**: ufork() 失敗時（EPERM 等）にはログを出力し、サーバーを緊急停止させます。

直接依存

  • os.h: プラットフォーム定義およびプロトタイプ宣言。
  • httpd.h, http_config.h, http_log.h: Apache のコア機能およびログ出力用。
  • apr_lib.h: 文字列操作ユーティリティ（apr_cpystrn 等）。

関連コード

  • os/bs2000/os.h: 本ファイルのプロトタイプ宣言およびプラットフォーム定義。
  • os/bs2000/ebcdic.c: BS2000 環境特有の文字コード変換処理。
  • os/unix/os.h: Unix 共通の OS 定義（本ファイルが依存するベース）。

読者向けメモ

  • 本ファイルは _OSD_POSIX マクロが定義されている環境でのみ有効です。それ以外の環境では bs2000_os_is_not_here() という空の関数のみが定義されます。
  • ufork() は BS2000 におけるユーザー ID 切り替えを伴う特殊なプロセス生成関数です。ユーザー名の大文字変換など、OS 固有の制約が実装に含まれている点に注意してください。
  • デバッグモード（one_process）での実行時は、特権ユーザーでの起動を避けるためのチェックが行われます。
