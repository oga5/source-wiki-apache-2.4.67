os/bs2000/ebcdic.c

概要

BS2000 メインフレーム環境における EBCDIC と ASCII 間の文字コード変換機能を提供します。Apache HTTP Server がネイティブの EBCDIC 環境で動作する際、ネットワーク通信やプロトコル処理に必要な ASCII 変換をサポートします。

モジュール内での役割

os/bs2000 サブシステムの一部として、文字セットの不一致を吸収する低レベルな変換レイヤーを担います。ebcdic.h で宣言された変換テーブルと関数を実装し、他のモジュールが文字コードを意識せずにデータ処理を行えるようにします。

責務

  • **変換テーブルの保持**: EBCDIC から ASCII への変換（通常版および厳密版）、および ASCII から EBCDIC への変換テーブルを静的データとして提供。
  • **メモリブロック変換**: 指定されたバイト数（count）のメモリ領域を、変換テーブルを用いて一括変換する関数の提供。
  • **プロトコル互換性の維持**: 改行コード（\n）など、プロトコル上で特別な意味を持つ文字の変換ルールを制御。

重要なシンボル

  • os_toascii: EBCDIC から ASCII への変換テーブル。0x0A（LF）を ASCII の 0x0A にマッピングするプロトコル向け。
  • os_toascii_strictly: 厳密な EBCDIC-ASCII 変換テーブル。
  • os_toebcdic: ASCII から EBCDIC への変換テーブル。
  • ebcdic2ascii() / ascii2ebcdic(): メモリブロックを変換する主要なユーティリティ関数。

制御フローとデータフロー

本ファイルは主にデータ変換のルックアップテーブルとして機能します。 1. 呼び出し元が変換対象のバッファとサイズを関数に渡します。 2. 関数は while ループ内で、入力バッファの各バイトをインデックスとして変換テーブルを参照します。 3. 変換後の値を宛先バッファに書き込みます。 4. os_toascii はプロトコル文字列の変換を考慮し、特定の制御文字を適切にマッピングするよう設計されています。

直接依存

  • ap_config.h: Apache の設定および環境定義。
  • ebcdic.h: 本ファイルのヘッダーであり、変換関数のプロトタイプとテーブルの外部参照を定義。

関連コード

  • os/bs2000/os.c: BS2000 特有の OS 抽象化レイヤー。
  • APR_CHARSET_EBCDIC: Apache Portable Runtime (APR) 側で定義される、EBCDIC 環境を有効化するためのコンパイル時フラグ。

読者向けメモ

  • **非連続性**: EBCDIC では英字（'a'..'z'）が連続していないため、単純なオフセット計算ではなく、本ファイルで定義された変換テーブルによるルックアップが必須です。
  • **厳密性の違い**: os_toascii と os_toascii_strictly の違いは、0x0A（LF）の扱いのみです。プロトコル文字列を扱う場合は、改行コードが正しく変換される os_toascii を使用してください。
  • **オーバーラップ**: 変換関数は dest と srce が重なるメモリ領域を想定していません。使用時は注意が必要です。
