os/netware/pre_nw.h

概要

pre_nw.h は、Apache HTTP Server を NetWare 環境でコンパイルするためのプラットフォーム固有のプリプロセッサ定義および環境設定を行うヘッダーファイルです。

モジュール内での役割

os/netware サブシステムにおける最上位の構成ファイルです。他のソースファイルがコンパイルされる前に読み込まれることで、NetWare 特有のコンパイラ挙動や SDK の差異を吸収し、コードベース全体で一貫した環境を提供します。

責務

  • **プラットフォーム識別**: NetWare 環境であることを示すマクロ（NETWARE, N_PLAT_NLM 等）の定義。
  • **コンパイラ互換性の確保**: FAR や cdecl といった古い環境向けのキーワードを無効化し、32bit 環境での競合を防ぐ。
  • **型定義の調整**: wchar_t や __int64 の定義を調整し、Novell SDK や MSL C++ ライブラリとの型衝突を回避する。
  • **ビルド制約の定義**: インクルードの深さ制限（AP_MAX_INCLUDE_DEPTH）などのビルド時パラメータの設定。

重要なシンボル

  • NETWARE / __NETWARE__: NetWare プラットフォームであることを示す主要なフラグ。
  • FAR / far: 32bit 環境では不要な修飾子を空定義として無効化。
  • _WCHAR_T: C++ コンパイル時に wchar_t の二重定義を防ぐためのガード。
  • AP_MAX_INCLUDE_DEPTH: インクルードのネスト制限を 48 に設定。

制御フローとデータフロー

本ファイルは実行時のフローを持たず、プリプロセッサによる静的な置換を行います。 1. コンパイル開始時にインクルードされる。 2. __pre_nw__ ガードにより多重読み込みを防止。 3. コンパイラ（主に CodeWarrior を想定）のオプションや環境に応じて、マクロ定義を注入する。 4. 以降のソースコードでこれらのマクロが参照され、条件付きコンパイルが実行される。

直接依存

  • stdint.h: 標準的な整数型定義のためにインクルード。

関連コード

  • os/netware/os.h: 本ファイルで定義された環境設定を前提として、OS 固有の機能や型を定義する次のステップのファイル。
  • os/netware/util_nw.c: NetWare 固有のユーティリティ関数群。本ファイルで定義されたマクロを利用して実装される。

読者向けメモ

  • 本ファイルは主に CodeWarrior コンパイラ環境を強く意識した設計になっています。
  • FAR や far の無効化は、古い NetWare SDK との互換性を維持しつつ、モダンな 32bit 環境でコンパイルを通すための典型的な手法です。
  • 新規に NetWare 向けコードを追加する際は、プラットフォーム固有の分岐が必要な場合にこのファイルで定義されたマクロを活用してください。
