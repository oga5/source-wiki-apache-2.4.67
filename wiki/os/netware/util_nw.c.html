os/netware/util_nw.c

概要

NetWare 環境における Apache HTTP Server のライフサイクル管理およびプロセス制御をサポートするユーティリティファイルです。主に NLM (NetWare Loadable Module) としての起動・終了処理と、OS レベルのイベント通知機構とのブリッジを提供します。

モジュール内での役割

os/netware サブシステムの一部として、Apache のコア機能と NetWare OS 特有の API を接続します。特に、サーバーのシャットダウンイベントの捕捉や、特権プロセス生成の抽象化を担います。

責務

  • **NLM ライフサイクル管理**: _NonAppStart および _NonAppCheckUnload を通じて、モジュールのロード・アンロード時のフックを提供します。
  • **イベント通知**: NetWare の EVENT_DOWN_SERVER を監視し、サーバー停止時に適切なクリーンアップ処理を実行します。
  • **プロセス抽象化**: ap_os_create_privileged_process を提供し、プラットフォーム固有のプロセス生成を Apache の標準的な apr_proc_create にマッピングします。

重要なシンボル

  • _NonAppStart: NLM 起動時に呼び出されるエントリーポイント。イベント通知の登録と終了ハンドラのセットアップを行います。
  • ap_down_server_cb: サーバー停止イベント発生時に実行されるコールバック関数。
  • ap_cb_destroy: モジュール終了時にリソース（イベント通知やインターフェース）を解放するクリーンアップ関数。
  • ap_os_create_privileged_process: 特権プロセス生成のためのラッパー関数。

制御フローとデータフロー

1. **起動時**: _NonAppStart が OS から呼び出され、AllocateResourceTag でリソースを確保します。RegisterForEventNotification を使用して EVENT_DOWN_SERVER を購読し、終了ハンドラを NXVmRegisterExitHandler で登録します。 2. **実行中**: サーバーが停止シグナルを受け取ると、ap_down_server_cb がトリガーされ、nlmUnloadSignaled を呼び出して終了処理を開始します。 3. **終了時**: ap_cb_destroy が呼び出され、UnRegisterEventNotification でイベント購読を解除し、NX_UNWRAP_INTERFACE でインターフェース参照を解放します。

直接依存

  • httpd.h, http_log.h, ap_mpm.h: Apache HTTP Server のコア定義。
  • netware.h, nks/netware.h, nks/vm.h: NetWare Kernel Services (NKS) および NetWare OS 特有のシステム API。

関連コード

  • os/netware/os.h: NetWare 用の OS 抽象化定義。
  • os/netware/modules.c: NetWare 上でのモジュール構成管理。
  • apr_proc_create (APR): ap_os_create_privileged_process の内部で呼び出されるプロセス生成関数。

読者向けメモ

  • 本ファイルは NetWare 特有の NLM 構造に強く依存しています。#pragma unused を多用しているのは、NLM の標準的なエントリーポイントシグネチャに合わせるためです。
  • ref や dum といった変数は、NetWare のインターフェースラッパー（NX_WRAP_INTERFACE）で使用される一時的な参照保持用です。
  • NetWare 環境以外ではコンパイルされないことを前提としています。
