os/netware/modules.c

概要

NetWare プラットフォーム向け Apache HTTP Server において、静的にリンクされる（組み込み）モジュールのリストを定義する構成ファイルです。動的ロード（DSO）ではなく、サーバーバイナリに直接含める必要がある主要モジュールの登録を管理します。

モジュール内での役割

Apache のモジュール登録システム（http_config）に対し、サーバー起動時にロードされるべきモジュールの静的配列を提供します。これにより、NetWare 環境下でのサーバーの基本機能セットを確定させます。

責務

  • **モジュール登録の集約**: サーバーコア、MPM、認証、コンテンツ処理などの必須モジュールをリスト化する。
  • **シンボル解決の提供**: モジュール名（文字列）とモジュール構造体（module）の対応付けを行い、設定解析や動的検索をサポートする。
  • **プラットフォーム固有の条件分岐**: USE_WINSOCK 定義に基づき、NetWare 特有の SSL モジュール（nwssl_module）の有無を制御する。

重要なシンボル

モジュール構造体（外部参照）

  • core_module, mpm_netware_module, http_module, so_module 等: サーバーの基本動作を定義する各モジュールの実体。これらは他のソースファイルで定義されており、extern で宣言されています。

モジュール管理配列

  • ap_prelinked_modules: サーバー起動時に初期化されるモジュールのポインタ配列。core_module が先頭にあることが必須条件です。
  • ap_preloaded_modules: ap_prelinked_modules と同様の構成を持つ、プリロード用モジュール配列。
  • ap_prelinked_module_symbols: モジュール名と構造体アドレスを保持する ap_module_symbol_t 型の配列。設定ファイルからのモジュール検索に使用されます。

制御フローとデータフロー

本ファイルは実行コードを含まず、データ定義のみで構成されます。 1. **データフロー**: 外部で定義された module 型のシンボルを extern でインポートし、配列構造体に格納します。 2. **制御フロー**: Apache の起動プロセスにおいて、http_config サブシステムがこれらの配列を走査し、各モジュールの register_hooks や create_dir_config などの関数を呼び出すことでサーバーの機能が構築されます。

直接依存

  • httpd.h: Apache の基本データ構造（module 型など）の定義。
  • http_config.h: モジュール設定および登録用 API の定義。

関連コード

  • server/config.c: ここで定義された ap_prelinked_modules 等を読み込み、サーバーの初期化を行う中心的なコード。
  • include/http_config.h: module 構造体および ap_module_symbol_t の定義元。

読者向けメモ

  • **メモリとライフサイクル**: 本ファイルで定義される配列は静的データセグメントに配置され、サーバーの生存期間中永続します。
  • **制約**: core_module は必ず配列の先頭に配置する必要があります。これは Apache の設定解析順序においてコア機能が最初に初期化される必要があるためです。
  • **省略された詳細**: 各モジュールの内部実装（フック関数や設定構造体）については、各モジュールのソースファイルに依存するため、本ファイルでは言及していません。
  • **注意点**: 新しいモジュールを静的に追加する場合は、extern 宣言、ap_prelinked_modules、ap_prelinked_module_symbols、ap_preloaded_modules の 4 箇所すべてに追記する必要があります。
