os/unix/unixd.c

概要

os/unix/unixd.c は、Apache HTTP Server の Unix 系プラットフォームにおける MPM（Multi-Processing Module）共通の低レイヤー機能を提供します。プロセス権限の管理、リソース制限（rlimit）の設定、Unix 特有のシグナルハンドリング、および suexec を介した特権プロセス生成の抽象化を担います。

モジュール内での役割

os/unix モジュール群の中核であり、unixd.h で定義されたデータ構造の実装を提供します。MPM が OS レベルの制約やセキュリティ要件（root 権限からの降格など）を扱う際のユーティリティとして機能します。

責務

  • **プロセス権限管理**: root 権限で起動した際のプロセス ID 降格に伴う、ミューテックス（SysV セマフォ等）の所有権・パーミッション調整。
  • **リソース制限**: setrlimit を用いたプロセスごとのリソース上限設定。
  • **特権プロセス生成**: suexec を利用した、指定ユーザー権限での外部プロセス実行。
  • **シグナル管理**: MPM のライフサイクル（停止・再起動）を制御するシグナルハンドラの登録と、retained_data を介した状態保持。
  • **プラットフォーム固有対応**: _OSD_POSIX（BS2000）環境における特殊な fork 処理の抽象化。

重要なシンボル

設定・状態管理

  • ap_unixd_config: サーバーの実行ユーザー/グループ情報や suexec 設定を保持するグローバル構造体。
  • retained_data: MPM 再起動時にも生存させる必要がある状態（停止フラグ、世代管理など）を保持する内部静的変数。

プロセス・権限操作

  • ap_unixd_set_rlimit(): rlimit を設定するコマンドハンドラ。
  • ap_os_create_privileged_process(): suexec を経由して特権プロセスを生成するエントリポイント。
  • ap_unixd_set_proc_mutex_perms(): プロセスミューテックスの所有権を ap_unixd_config のユーザーに合わせる。

シグナル・ライフサイクル

  • ap_unixd_mpm_set_signals(): MPM の停止・再起動シグナルを登録する。
  • sig_term, sig_restart: シグナルハンドラ。retained_data を更新して MPM の状態遷移をトリガーする。

制御フローとデータフロー

1. **シグナルフロー**: ap_unixd_mpm_set_signals がシグナルハンドラを登録。シグナル受信時、ハンドラは retained_data のフラグを更新。メインループはこれらを監視し、適切なタイミングで停止・再起動を実行する。 2. **権限降格フロー**: サーバー起動時に root で実行されている場合、ap_unixd_set_proc_mutex_perms が呼び出され、ミューテックスのパーミッションを ap_unixd_config で指定された非特権ユーザーに変更する。 3. **特権プロセス生成**: ap_os_create_privileged_process は get_suexec_identity フックを呼び出し、対象ユーザー情報を取得。suexec バイナリを介してプロセスを起動する。

直接依存

  • **APR**: apr_proc_mutex.h, apr_signal.h, apr_thread_proc.h 等によるプラットフォーム抽象化。
  • **OS ヘッダー**: pwd.h, grp.h, sys/resource.h, sys/sem.h 等の Unix システムコール用ヘッダー。
  • **Apache Core**: httpd.h, http_config.h, mpm_common.h 等のサーバーコア定義。

関連コード

  • os/unix/unixd.h: 本ファイルのデータ構造と関数宣言。
  • server/mpm_common.c: MPM 共通のシグナル処理やリソース管理の補完。
  • modules/arch/unix/mod_suexec.c: suexec の具体的な実装ロジック。

読者向けメモ

  • **メモリ所有権**: retained_data は ap_pglobal プールに関連付けられており、サーバーのライフサイクル全体で生存します。
  • **エッジケース**: ap_unixd_accept は、EMFILE や ENETDOWN などのソケットエラーを適切にハンドリングし、必要に応じてサーバーを停止させるロジックを含みます。
  • **注意点**: _OSD_POSIX 関連のコードは特定のレガシー環境向けであり、一般的な Unix/Linux 環境では無効化されます。
  • **省略箇所**: ap_unixd_accept 内のプラットフォーム固有のエラーハンドリング（HPUX 等）は、歴史的な経緯による複雑な条件分岐が含まれていますが、本質的には「致命的なエラーか否かの判定」に集約されます。
