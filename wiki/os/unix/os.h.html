os/unix/os.h

概要

os/unix/os.h は、Apache HTTP Server における Unix 系プラットフォーム向けの OS 固有定義およびインターフェースを規定するヘッダーファイルです。全ソースコードからインクルードされることを前提としており、プラットフォーム依存の機能フラグや抽象化レイヤーを提供します。

モジュール内での役割

os/unix モジュールにおける最上位の定義ファイルです。このファイルは、Apache が実行される環境が Unix であることを宣言し、プラットフォーム固有の動作（引数の書き換えやミューテックスのパーミッション管理など）をコアモジュールに対して通知する役割を担います。

責務

  • **プラットフォーム識別**: PLATFORM マクロによる環境の特定。
  • **機能フラグの提供**: AP_NEED_SET_MUTEX_PERMS 等の定義を通じた、モジュール開発者へのプラットフォーム固有の制約の通知。
  • **OS 固有インターフェースの宣言**: _OSD_POSIX 環境下における os_fork 等の特殊なプロセス生成関数のプロトタイプ宣言。

重要なシンボル

  • PLATFORM: 現在のプラットフォーム名（デフォルトは "Unix"）。
  • AP_NEED_SET_MUTEX_PERMS: 親プロセスで作成されたミューテックスに対して、パーミッション設定が必要であることを示すフラグ。
  • AP_PLATFORM_REWRITE_ARGS_HOOK: コマンドライン引数の書き換え処理を行うフック関数名。
  • os_fork(): _OSD_POSIX 環境において、ユーザー権限を切り替えてプロセスを生成するための関数。

制御フローとデータフロー

本ファイルは主にコンパイル時の定義を提供します。

  • **データフロー**: AP_PLATFORM_REWRITE_ARGS_HOOK を通じて、コアモジュールが起動時にプラットフォーム固有の引数処理関数（ap_mpm_rewrite_args）を呼び出せるようにします。
  • **制御フロー**: _OSD_POSIX が定義されている場合のみ os_fork が宣言され、プロセス生成時の権限管理ロジックが有効化されます。

直接依存

  • apr.h: Apache Portable Runtime の基本定義。
  • ap_config.h: Apache の設定およびプラットフォーム固有の構成定義。

関連コード

  • os/bs2000/os.c: _OSD_POSIX 定義時に os_fork の実体が実装されている対となるファイル。
  • os/unix/unixd.c: Unix 系システムにおけるプロセス権限管理やミューテックス制御の実装。
  • include/http_config.h: フック機構の定義元。

読者向けメモ

  • 本ファイルは非常に軽量ですが、全ソースからインクルードされるため、ここへの変更は全モジュールの再コンパイルを誘発します。
  • AP_NEED_SET_MUTEX_PERMS が定義されている場合、モジュール開発者はミューテックス作成後に適切なパーミッション設定関数を呼び出す義務がある点に注意してください。
  • _OSD_POSIX 以外の環境では os_fork は宣言されません。プラットフォーム固有の拡張を行う際は、必ず条件付きコンパイルを適切に管理してください。
