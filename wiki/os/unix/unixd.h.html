os/unix/unixd.h

概要

os/unix/unixd.h は、Apache HTTP Server の Unix 系プラットフォーム向け MPM（Multi-Processing Module）が共通して利用するデータ構造、定数、およびユーティリティ関数を定義するヘッダーファイルです。

モジュール内での役割

os/unix モジュールの中核を成すヘッダーであり、Unix 特有のプロセス管理、権限設定、リソース制限（rlimit）、およびシグナル処理のインターフェースを提供します。MPM がプラットフォーム依存の差異を吸収するための共通基盤として機能します。

責務

  • **ID 管理**: プロセス実行ユーザー/グループの識別と suexec 関連の ID 解決。
  • **リソース制御**: rlimit の設定およびプロセス権限の管理。
  • **MPM 状態保持**: 再起動やシグナル処理を跨いで保持すべき MPM 固有データの定義。
  • **プラットフォーム抽象化**: killpg のような Unix 固有機能のポータビリティ確保。

重要なシンボル

  • unixd_config_rec: 実行ユーザー、グループ、chroot 設定など、サーバーの Unix 実行環境を保持する構造体。
  • ap_unix_identity_t: suexec 等で使用されるユーザー/グループ ID の識別子。
  • ap_unixd_mpm_retained_data: MPM が再起動時にも保持すべき状態（シグナルフラグや世代管理など）を格納する構造体。
  • ap_unixd_set_proc_mutex_perms: プロセス間ミューテックスの権限を適切に設定する関数。

制御フローとデータフロー

本ファイルは定義のみを提供します。 1. MPM は起動時に ap_unixd_config を参照して権限をドロップします。 2. ap_unixd_set_rlimit が設定ファイル解析時に呼び出され、プロセスリソース制限を適用します。 3. シグナルハンドラが ap_unixd_mpm_retained_data 内のフラグ（shutdown_pending 等）を更新し、メインループがそれを監視することで制御フローを分岐させます。

直接依存

  • **APR**: apr_thread_proc.h, apr_proc_mutex.h, apr_global_mutex.h 等によるプロセス・スレッド・同期プリミティブ。
  • **システムヘッダー**: <pwd.h>, <grp.h>, <sys/resource.h> 等の Unix 標準 API。
  • **Apache Core**: httpd.h, http_config.h, scoreboard.h 等のサーバーコア定義。

関連コード

  • os/unix/unixd.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • server/mpm/: 各種 MPM（worker, event, prefork 等）が本ヘッダーをインクルードして Unix 固有の処理を実装します。

読者向けメモ

  • 本ファイル内の ap_unixd_mpm_retained_data は、再起動（graceful restart）時に MPM が状態を維持するために重要です。volatile 修飾子が使われているのは、シグナルハンドラから非同期に更新されるためです。
  • DEFAULT_USER / DEFAULT_GROUP の # プレフィックスは、数値 ID を直接指定する場合の慣習です。
  • ap_unixd_set_proc_mutex_perms は、root 権限で起動した後に非特権ユーザーへ移行する際、ミューテックスの所有権を適切に引き継ぐために必須の処理です。
