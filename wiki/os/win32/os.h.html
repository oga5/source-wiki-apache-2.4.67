os/win32/os.h

概要

Apache HTTP Server の Windows プラットフォーム向け OS 抽象化レイヤーの基底ヘッダーです。すべてのソースファイルからインクルードされることを想定しており、Windows 特有の機能フラグ、型定義、および実行時の動的リンクを支援するマクロを提供します。

モジュール内での役割

os/win32 モジュールにおける最上位の定義ファイルです。プラットフォーム固有の挙動（ファイルシステムの大文字小文字区別やパス形式など）を定義し、Apache Core が Windows 上で動作するために必要な OS 固有のインターフェースを公開します。

責務

  • **プラットフォーム機能の宣言**: HAVE_DRIVE_LETTERS 等のフラグにより、Windows 特有のファイルシステム特性を Apache に伝達する。
  • **exit 関数のラップ**: exit() を再定義し、ApacheCore.dll 内での終了ステータスを捕捉・管理する。
  • **動的リンクの抽象化**: AP_DECLARE_LATE_DLL_FUNC マクロを通じて、実行時に Windows API を遅延ロードするためのメカニズムを提供する。

重要なシンボル

  • AP_PLATFORM_REWRITE_ARGS_HOOK: コマンドライン引数の書き換えフック（現在は NULL）。
  • ap_real_exit_code: exit() 呼び出し時の終了コードを保持するグローバル変数。
  • ap_dlltoken_e: ロード対象の DLL（kernel32, advapi32 等）を識別する列挙型。
  • AP_DECLARE_LATE_DLL_FUNC: 指定した Windows API を実行時に動的ロードし、関数ポインタをキャッシュするラッパーを生成するマクロ。

制御フローとデータフロー

本ファイルは主にコンパイル時の定義を提供します。 1. **初期化**: インクルード時に WIN32 マクロが定義されているか確認し、プラットフォーム固有の定数を設定します。 2. **遅延ロード**: AP_DECLARE_LATE_DLL_FUNC を使用した関数（例: CancelIo）が初めて呼び出されると、内部で ap_load_dll_func が実行され、DLL から関数アドレスが解決されます。 3. **終了処理**: exit() マクロが呼び出されると、ap_real_exit_code にステータスが記録され、標準の exit() へと制御が移ります。

直接依存

  • apr_pools.h: APR プール機能。
  • io.h, fcntl.h: Windows 低レベル I/O 定義。
  • stddef.h, stdlib.h: 標準ライブラリ（exit 等）。

関連コード

  • os/win32/util_win32.c: ap_load_dll_func の実装が含まれており、本ヘッダーで宣言された遅延ロード関数の実体を提供します。
  • os/win32/ap_regkey.c: Windows レジストリ操作を行うための関連モジュール。

読者向けメモ

  • **警告**: exit() の再定義は、ApacheCore.dll にリンクされたモジュール内でのみ有効です。外部モジュールからこのヘッダーをインクルードして exit() を呼ぶと予期せぬ動作を引き起こす可能性があるため注意してください。
  • **設計方針**: Windows のバージョン依存（Win9x/NT/2K）を吸収するため、直接 API を呼ぶのではなく、AP_DECLARE_LATE_DLL_FUNC を介した遅延ロードが推奨されています。これにより、特定の OS バージョンでしか存在しない API を安全に扱うことができます。
