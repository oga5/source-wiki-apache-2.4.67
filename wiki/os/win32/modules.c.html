os/win32/modules.c

概要

本ファイルは、Apache HTTP Server (Win32版) において、静的にリンクされる主要なモジュール群を定義および登録するための構成ファイルです。実行バイナリに組み込まれるべき必須モジュールのリストを管理します。

モジュール内での役割

os/win32 サブシステムにおいて、サーバー起動時にロードされるべきコアモジュールとプラットフォーム固有モジュールの「静的リスト」を保持する役割を担います。動的ロード（DSO）ではなく、バイナリに直結されたモジュールのエントリポイントを提供します。

責務

  • サーバーの基本機能に必要なモジュール（core, win32, mpm_winnt 等）の外部参照を宣言する。
  • サーバー起動時に参照されるモジュール配列（ap_prelinked_modules 等）を構築し、初期化順序を規定する。
  • モジュール名とシンボルアドレスの対応付けを行い、設定やデバッグ時に利用可能なシンボルテーブルを提供する。

重要なシンボル

  • ap_prelinked_modules: サーバー起動時に自動的にリンク・初期化されるモジュールの配列。core_module が先頭にあることが必須です。
  • ap_prelinked_module_symbols: モジュール名と構造体へのポインタを保持するテーブル。動的なモジュール検索や検証に使用されます。
  • ap_preloaded_modules: プリロードされるモジュールのリスト。通常は ap_prelinked_modules と同等の構成をとります。

制御フローとデータフロー

本ファイルは実行ロジックを持たず、データ定義のみを行います。 1. **宣言**: extern キーワードにより、各モジュールの構造体定義を外部からインポートします。 2. **静的初期化**: コンパイル時に配列が構築され、サーバーの起動プロセスにおいて ap_init_modules 等の関数がこれらの配列を走査することで、各モジュールのフックや設定が有効化されます。

直接依存

  • httpd.h: Apache のコアデータ構造（module 型など）の定義。
  • http_config.h: モジュール設定および AP_DECLARE_DATA マクロの定義。

関連コード

  • server/modules.c: Unix系プラットフォームにおける同等の役割を果たすファイル。
  • include/http_config.h: module 構造体の定義元であり、モジュール登録の仕組みを理解するために不可欠です。
  • os/win32/os.h: Win32固有のOS抽象化レイヤー。

読者向けメモ

  • **順序の重要性**: ap_prelinked_modules の先頭に core_module を配置することは、サーバーの初期化順序において厳格なルールです。変更を加える際は注意してください。
  • **静的 vs 動的**: ここにリストされているモジュールは静的リンク用です。DSO（Dynamic Shared Object）としてロードされるモジュールは、このリストには含まれません。
  • **Win32特有**: win32_module や mpm_winnt_module が含まれている点が、Unix版の modules.c との主な違いです。
