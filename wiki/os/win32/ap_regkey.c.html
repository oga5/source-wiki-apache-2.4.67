os/win32/ap_regkey.c

概要

Windows レジストリ操作を抽象化し、APR（Apache Portable Runtime）スタイルのインターフェースを提供するモジュールです。Win32 API（RegOpenKeyEx, RegQueryValueEx 等）をラップし、メモリ管理を APR プールに統合することで、リソースリークを防ぎつつ安全なレジストリ操作を実現します。

モジュール内での役割

os/win32 サブシステムの一部として、Windows 特有のレジストリ設定を読み書きする機能を提供します。ap_regkey.h で定義された抽象データ型 ap_regkey_t を実装し、HTTP サーバーの設定や環境情報の取得・永続化をサポートします。

責務

  • **ライフサイクル管理**: レジストリキーのオープン・クローズを管理し、APR プールクリーンアップハンドラを通じて自動的にハンドルを解放する。
  • **プラットフォーム抽象化**: Unicode（W 系 API）と ANSI（A 系 API）の双方をサポートし、実行時の OS 環境に応じて適切な API を選択する。
  • **データ変換**: レジストリの型（REG_SZ, REG_EXPAND_SZ, REG_MULTI_SZ）と C 文字列/配列間の変換および環境変数展開を行う。
  • **エラーハンドリング**: Win32 エラーコードを apr_status_t に変換し、一貫したエラー報告を行う。

重要なシンボル

データ構造

  • struct ap_regkey_t: レジストリキーのハンドル (HKEY) と、そのキーに関連付けられた apr_pool_t を保持するコンテナ。

ライフサイクル管理

  • ap_regkey_const(): 定義済みのルートキー（HKEY_LOCAL_MACHINE 等）を ap_regkey_t 型として取得する。
  • ap_regkey_open(): レジストリキーを開く/作成し、プールにクリーンアップを登録する。
  • ap_regkey_close(): 明示的にキーを閉じ、プールからクリーンアップ登録を解除する。

値の操作

  • ap_regkey_value_get/set(): 文字列値の読み書き（REG_SZ / REG_EXPAND_SZ）。
  • ap_regkey_value_raw_get/set(): 型を問わないバイナリデータの読み書き。
  • ap_regkey_value_array_get/set(): REG_MULTI_SZ 型の配列データの操作。
  • ap_regkey_remove(): レジストリキーの削除（非再帰的）。

制御フローとデータフロー

1. **初期化**: ap_regkey_open が呼び出されると、OS の RegCreateKeyEx または RegOpenKeyEx を実行し、成功時に apr_pool_cleanup_register を使用して regkey_cleanup を登録します。 2. **データアクセス**: get 系関数は、まず RegQueryValueEx を呼び出して必要なバッファサイズを特定し、apr_palloc でメモリを確保してから実際のデータを取得します。 3. **環境変数展開**: REG_EXPAND_SZ 型の場合、ExpandEnvironmentStrings を呼び出して %SystemRoot% 等を解決します。 4. **終了**: プールの破棄時、または ap_regkey_close 呼び出し時に regkey_cleanup が実行され、RegCloseKey でハンドルが解放されます。

読者向けメモ

  • **メモリ所有権**: ap_regkey_t が保持するハンドルは、関連付けられた apr_pool_t の生存期間に依存します。プールが破棄されると自動的にクローズされます。
  • **Unicode/ANSI**: APR_HAS_UNICODE_FS および APR_HAS_ANSI_FS マクロにより、コンパイル時にどちらの API を使用するかが決定されます。
  • **制約**: ap_regkey_remove は非再帰的です。サブキーが存在する場合、Windows の仕様に従い削除は失敗します。
  • **省略箇所**: レジストリの列挙（RegEnumKeyEx 等）機能は本ファイルには含まれていません。これは主に設定の読み書きに特化した設計であるためです。
