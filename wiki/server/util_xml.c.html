server/util_xml.c

概要

Apache HTTP Server における XML リクエストボディの読み込みと解析を抽象化するユーティリティモジュールです。主に WebDAV などの XML を利用するモジュールが、リクエストボディを安全かつ効率的に取得するために使用します。

モジュール内での役割

server モジュール群の一部として、apr_xml（Apache Portable Runtime の XML ライブラリ）を HTTP リクエスト処理の文脈に統合する役割を担います。

責務

  • **リクエストボディのストリーミング読み込み**: ap_get_brigade を使用し、バケットブリゲード経由でリクエストボディを段階的に取得します。
  • **サイズ制限の適用**: LimitXMLRequestBody ディレクティブで設定された制限値を遵守し、超過した場合はエラーを返します。
  • **XML 解析の実行**: apr_xml パーサーへデータを供給し、最終的なドキュメント構造（apr_xml_doc）を生成します。
  • **文字コード変換**: EBCDIC 環境において、必要に応じて文字コード変換を適用します。

重要なシンボル

  • ap_xml_parse_input: 本ファイルの主要な公開関数。リクエストボディを読み込み、解析済みの XML ドキュメントを返します。
  • READ_BLOCKSIZE: 読み込み時に使用するバッファサイズ（2048バイト）。
  • apr_xml_parser: apr_xml が提供する解析用コンテキスト。

制御フローとデータフロー

1. **初期化**: apr_xml_parser_create でパーサーを生成し、apr_brigade_create で読み込み用ブリゲードを準備します。 2. **ループ処理**: ap_get_brigade を呼び出し、EOS（End of Stream）に達するまでデータを読み込みます。 3. **検証と供給**: 読み込んだ各バケットのサイズを累積し、制限値を超えていないか確認した後、apr_xml_parser_feed でパーサーにデータを渡します。 4. **終了処理**: EOS 到達後、apr_xml_parser_done を呼び出して解析を完了させ、結果を pdoc に格納します。 5. **エラーハンドリング**: 読み込みエラーや解析エラーが発生した場合は、ログを出力し適切な HTTP ステータスコード（HTTP_BAD_REQUEST や HTTP_REQUEST_ENTITY_TOO_LARGE）を返します。

直接依存

  • apr_xml.h: APR の XML 解析機能。
  • httpd.h, http_protocol.h, http_log.h, http_core.h: Apache HTTP Server のコア機能およびログ出力機能。
  • util_charset.h: 文字コード変換用ユーティリティ。
  • util_xml.h: 本ファイルのヘッダー。

関連コード

  • include/util_xml.h: 本関数の宣言とインターフェース定義。
  • server/core.c: ap_get_limit_xml_body など、設定値の取得に関連する関数が定義されています。

読者向けメモ

  • 本ファイルは apr_xml のラッパーであり、HTTP リクエストのライフサイクル（request_rec）に特化した実装です。
  • 解析エラーが発生した場合、apr_xml_parser_geterror を使用して詳細なエラーメッセージを取得し、ログに記録する設計になっています。
  • EBCDIC 環境での動作を考慮した条件付きコンパイルが含まれている点に注意してください。
