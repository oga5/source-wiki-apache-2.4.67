server/mpm_fdqueue.c

概要

server/mpm_fdqueue.c は、Apache HTTP Server のマルチプロセスモジュール（MPM）、特に event や worker において、リスナー・スレッドとワーカー・スレッド間でソケットやタイマーイベントを安全に受け渡すためのスレッドセーフなキュー構造を提供します。

モジュール内での役割

server モジュール内のスレッド管理インフラの一部であり、mpm_common.c 等と連携してリクエスト処理のパイプラインを構築します。本ファイルは、スレッド間通信の同期プリミティブ（Mutex/Condition Variable）と、メモリ効率化のためのプール再利用メカニズムをカプセル化しています。

責務

  • **スレッド間通信**: ソケット（apr_socket_t）およびタイマーイベント（timer_event_t）のキューイングとデキューイング。
  • **ワーカー管理**: fd_queue_info_t を通じたワーカーのアイドル状態管理と、スレッドのブロッキング/ウェイクアップ制御。
  • **メモリ最適化**: apr_pool_t の再利用リストを管理し、頻繁なメモリ割り当て・解放コストを削減。
  • **ライフサイクル管理**: キューおよび関連リソースの初期化、破棄、および終了処理。

重要なシンボル

キュー管理 (fd_queue_t)

  • ap_queue_create: キューの初期化と Mutex/CondVar の生成。
  • ap_queue_push_socket / ap_queue_push_timer: キューへの要素追加と待機スレッドへのシグナル送信。
  • ap_queue_pop_something: キューからの要素取得。空の場合は not_empty 条件変数で待機。

ワーカー状態管理 (fd_queue_info_t)

  • ap_queue_info_create: アイドル状態管理構造体の生成。
  • ap_queue_info_wait_for_idler: ワーカーが利用可能になるまで待機。
  • ap_queue_info_set_idle: ワーカーがアイドル状態になったことを通知し、待機中のリスナーを起床。

プール再利用

  • ap_queue_info_push_pool / ap_queue_info_pop_pool: recycled_pools リストに対するアトミックな操作。

制御フローとデータフロー

  • **ソケットの受け渡し**: リスナーが ap_queue_push_socket を呼び出し、one_big_mutex を取得してデータを格納後、not_empty 条件変数でワーカーを起床させます。
  • **ワーカーの待機**: ワーカーは ap_queue_pop_something を呼び出し、キューが空であれば not_empty で待機します。
  • **アトミックな状態管理**: idlers カウンタは apr_atomic 操作を用いて、Mutex を介さずに現在の負荷状況を高速に判定します。zero_pt を基準としたオフセット値により、負の値で「待機中のスレッド数」を表現する巧妙な設計がなされています。

直接依存

  • mpm_fdqueue.h: インターフェース定義。
  • apr_atomic.h: アトミック操作によるロックフリーなカウンタ管理。
  • apr_thread_mutex.h / apr_thread_cond.h: スレッド同期。
  • apr_ring.h: タイマーイベントの管理に使用。

関連コード

  • server/mpm_common.c: MPM の共通ロジック。
  • server/mpm_unix.c: Unix 系 OS での MPM 実装。
  • server/event.c (MPM): 本キューを実際に利用する主要なモジュール。

読者向けメモ

  • **メモリ所有権**: ap_queue_info_push_pool で渡されたプールは、キューの管理下に入り、再利用または破棄の対象となります。
  • **スレッド安全性**: fd_queue_t は one_big_mutex によって保護されていますが、ap_queue_info 関連の関数はアトミック操作を多用しており、呼び出し側のスレッドコンテキストに依存する制約（例: pop_pool はリスナー専用など）があるため注意が必要です。
  • **設計上の注意**: idlers カウンタの zero_pt は、アトミックな加減算で「待機中」と「アイドル中」を区別するためのマジックナンバーです。
  • **省略箇所**: 本ファイルはスレッド環境 (APR_HAS_THREADS) が有効な場合のみコンパイルされます。非スレッド環境での動作は考慮されていません。
