server/util_mutex.c

概要

Apache HTTP Server におけるミューテックス（排他制御）設定の管理および生成を抽象化するライブラリです。各モジュールが独自のミューテックスを登録し、管理者が Mutex ディレクティブを通じてメカニズム（flock, pthread 等）やディレクトリを一元的に制御できるようにします。

モジュール内での役割

server モジュールの一部として、http_config.c 等で処理される設定ディレクティブと、実際にミューテックスを使用する各モジュール（MPM やプロキシ等）の橋渡しを行います。

責務

  • **設定管理**: Mutex ディレクティブによるグローバルな設定の解析と保持。
  • **登録と解決**: 各モジュールによるミューテックス型の登録と、実行時の設定解決（継承とデフォルト値の適用）。
  • **インスタンス生成**: apr_global_mutex_t および apr_proc_mutex_t の生成と、OS 権限設定の適用。
  • **デバッグ支援**: 現在のミューテックス設定のダンプ機能。

重要なシンボル

設定管理・登録

  • mutex_cfg_t: ミューテックスのメカニズム、ディレクトリ、オプションを保持する内部構造体。
  • ap_mutex_register(): モジュールが自身のミューテックス型を登録するための API。
  • ap_set_mutex(): Mutex ディレクティブのハンドラ。設定をハッシュテーブルに反映。
  • ap_mutex_init(): 内部ハッシュテーブル mxcfg_by_type の初期化。

インスタンス生成

  • ap_global_mutex_create(): グローバルミューテックスの生成と権限設定。
  • ap_proc_mutex_create(): プロセス間ミューテックスの生成と権限設定。

ユーティリティ

  • ap_parse_mutex(): 文字列形式のミューテックス設定を apr_lockmech_e に変換。
  • mxcfg_lookup(): 登録済み設定、デフォルト設定、グローバル設定を優先順位に従って解決。

制御フローとデータフロー

1. **初期化**: ap_mutex_init が呼ばれ、mxcfg_by_type ハッシュが作成される。 2. **登録**: 各モジュールが ap_mutex_register で型を登録。 3. **設定**: ap_set_mutex が設定ファイルを読み込み、ハッシュ内の mutex_cfg_t を更新。 4. **生成**: モジュールが ap_xxx_mutex_create を呼び出すと、mxcfg_lookup が現在のコンテキストに最適な設定を決定し、get_mutex_filename でパスを生成して APR のミューテックス生成関数を呼び出す。

読者向けメモ

  • **メモリ管理**: mxcfg_by_type は pconf プールで管理され、設定のライフサイクルはサーバーの起動期間と一致します。
  • **優先順位**: mxcfg_lookup 内のロジックにより、特定のミューテックス設定 > デフォルト設定 > 登録時のデフォルト値 の順で解決されます。
  • **ファイルベースの制約**: flock や fcntl を使用する場合、get_mutex_filename がパスを生成します。omit_pid が指定されていない場合、ファイル名に PID が付与され、プロセスごとの競合を避ける設計になっています。
  • **省略箇所**: 本ファイルはミューテックスの「設定と生成」に特化しており、実際のロック取得・解放のロジックは APR (apr_proc_mutex_lock 等) に委譲されています。
