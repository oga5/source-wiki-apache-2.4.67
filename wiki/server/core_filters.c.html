server/core_filters.c

概要

Apache HTTP Server のコアネットワークフィルタを実装するファイルです。クライアントとのソケット通信における入力（リクエスト読み込み）および出力（レスポンス送信）の低レイヤー処理を担います。

モジュール内での役割

server モジュールの一部として、util_filter.c が提供するフィルタフレームワーク上で動作する「コアフィルタ」を提供します。HTTP プロトコル層と OS のソケット層を仲介する最下位のフィルタとして機能します。

責務

  • **入力フィルタ (ap_core_input_filter)**: ソケットからのバイトストリームを apr_bucket_brigade に変換し、HTTP リクエストの読み込みモード（行単位、バイト単位、スペック読み込み等）に応じたデータ供給を行う。
  • **出力フィルタ (ap_core_output_filter)**: バッファリングされたレスポンスデータをソケットへ書き出す。writev や sendfile を活用した効率的なデータ転送と、フロー制御（閾値によるフラッシュ）を管理する。
  • **リソース管理**: 接続ごとのコンテキスト (core_output_filter_ctx_t) の保持、バッファの生存期間管理、およびソケットの非ブロッキング制御。

重要なシンボル

フィルタコンテキスト

  • core_output_filter_ctx: 出力フィルタの永続状態。バッファリングされたブリゲード、書き込み用 iovec 配列、および遅延書き込み用プールを保持。

入力フィルタ処理

  • ap_core_input_filter: 入力フィルタのメインエントリポイント。AP_MODE_GETLINE や AP_MODE_READBYTES 等のモードに応じたデータ抽出を行う。

出力フィルタ処理

  • ap_core_output_filter: 出力フィルタのメインエントリポイント。ブリゲードをソケットへ書き出す。
  • should_send_brigade: 書き込みのフラッシュタイミングを決定するヒューリスティック関数。
  • send_brigade_nonblocking: ブリゲード内のバケットを走査し、writev または sendfile を呼び出す。
  • writev_nonblocking: apr_socket_sendv を使用した非ブロッキング書き込み。

ユーティリティ

  • BRIGADE_NORMALIZE: ブリゲード内の長さ 0 のバケットを削除し、処理を簡略化するマクロ。

制御フローとデータフロー

1. **入力**: ap_core_input_filter は ctx->b にソケットから読み込んだデータを蓄積し、要求されたモードに応じて apr_brigade_split_line や apr_brigade_partition を用いてデータを切り出し、呼び出し元へ渡す。 2. **出力**: ap_core_output_filter は should_send_brigade でフラッシュの必要性を判断。必要であれば send_brigade_nonblocking を呼び出し、バケットの種類に応じて writev または sendfile を選択してソケットへ転送する。 3. **フロー制御**: writev_nonblocking は書き込みが完了したバケットを順次削除し、部分的に書き込まれたバケットは apr_bucket_split で分割して残りを保持する。

読者向けメモ

  • **メモリ管理**: core_output_filter_ctx 内の deferred_write_pool は、パイプライン化されたリクエストのバッファリングに使用されます。リクエスト終了時に apr_pool_clear されるため、生存期間に注意が必要です。
  • **非ブロッキング**: このフィルタは基本的に非ブロッキング I/O を前提としています。EAGAIN が発生した場合の再試行ロジックが ap_core_output_filter 内に実装されています。
  • **省略された詳細**: sendfile のプラットフォーム依存の最適化や、EBCDIC 変換に関する詳細は、コードの可読性を優先し、主要なフローに焦点を当てて説明を省略しています。
  • **注意点**: BRIGADE_NORMALIZE はメタデータバケットを保持しつつ、空のデータバケットのみを削除する点に注意してください。
