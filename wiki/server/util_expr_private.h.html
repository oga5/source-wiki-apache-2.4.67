server/util_expr_private.h

概要

Apache HTTP Server の式評価エンジン（ap_expr）における、内部的なデータ構造とパーサー・スキャナー間の共有定義を保持するヘッダーファイルです。

モジュール内での役割

server モジュール内の式評価サブシステムにおいて、構文解析（util_expr_parse.c）と字句解析（util_expr_scan.c）を繋ぐインターフェースとして機能します。パース処理中に生成される抽象構文木（AST）のノード定義と、解析コンテキストを定義します。

責務

  • 式評価エンジンで使用される AST ノードの演算子型（ap_expr_node_op_e）の定義。
  • パーサーとスキャナー間で共有される解析コンテキスト（ap_expr_parse_ctx_t）の管理。
  • Bison/Flex で生成されたコードに対する互換性マクロの提供。
  • AST ノード生成用ヘルパー関数の宣言。

重要なシンボル

  • ap_expr_node_op_e: 式の演算子やオペランドの種類を網羅する列挙型。
  • ap_expr_node: AST を構成する基本ノード構造体。
  • ap_expr_parse_ctx_t: 解析中の入力バッファ、メモリプール、エラー情報、生成された AST を保持するコンテキスト構造体。
  • ap_expr_make: AST ノードを動的に生成するためのファクトリ関数。

制御フローとデータフロー

本ファイルは実行コードを含みませんが、以下のデータフローを規定します。 1. **初期化**: ap_expr_parse_ctx_t が初期化され、入力文字列がセットされます。 2. **解析**: ap_expr_yyparse が呼び出され、スキャナーがトークンを読み取り、ap_expr_make 等を通じて AST を構築します。 3. **格納**: 構築された AST は ap_expr_parse_ctx_t 内の expr フィールドに格納され、評価フェーズへと引き渡されます。

直接依存

  • httpd.h: Apache の基本型定義。
  • apr_strings.h, apr_tables.h: APR の文字列・テーブル操作ユーティリティ。
  • ap_expr.h: 公開されている式評価エンジンの API 定義。

関連コード

  • server/util_expr_parse.c: 本ヘッダーで定義された構造体を利用して構文解析を行う Bison 生成コード。
  • server/util_expr_scan.c: 本ヘッダーのコンテキストを利用して字句解析を行う Flex 生成コード。
  • server/util_expr_eval.c: 構築された AST を実際に評価するロジック。

読者向けメモ

  • 本ファイルは内部実装用であり、モジュール外からの直接参照は推奨されません。
  • YYMALLOC や YYFREE などのマクロは、Bison 生成コードの環境依存性を吸収するために定義されています。
  • ap_expr_parse_ctx_t は解析のライフサイクル全体を管理するため、メモリ管理（pool, ptemp）の境界を理解する上で重要です。
