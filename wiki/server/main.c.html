server/main.c

概要

Apache HTTP Server のエントリポイントであり、プロセス初期化、コマンドライン引数の解析、設定ファイルの読み込み、および MPM (Multi-Processing Module) のライフサイクル管理を担う中心的なファイルです。

モジュール内での役割

server モジュールにおける最上位の制御層です。他のソースファイルが提供する設定解析やリクエスト処理のロジックを統合し、サーバーの起動から終了までの状態遷移を管理します。

責務

  • **プロセス管理**: apr_app_initialize を用いた環境の初期化と、終了時のリソース解放。
  • **設定のライフサイクル**: 設定ファイルの読み込み、構文チェック、および設定の再読み込み（graceful restart 等）の制御。
  • **コマンドライン解析**: サーバー起動オプション（-D, -f, -t 等）の解釈と実行モードの決定。
  • **MPM 統合**: サーバーのメインループにおいて MPM を呼び出し、リクエスト処理を開始する。

重要なシンボル

プロセス・ライフサイクル管理

  • main(): サーバーのメインエントリポイント。
  • init_process(): apr の初期化、メモリプール作成、プロセス構造体の構築を行う。
  • destroy_and_exit_process(): サーバー終了時のクリーンアップとプロセス終了処理。
  • reset_process_pconf(): 設定用メモリプール (pconf) のリセットとフックの再登録。

設定・診断

  • show_compile_settings() / show_mpm_settings(): コンパイル時のオプションや MPM の構成情報を出力する。
  • usage(): コマンドライン引数のヘルプを表示し、プロセスを終了する。

内部状態・フック

  • deregister_all_hooks(): 設定リセット時に既存のフックを解除する。
  • abort_on_oom(): メモリ割り当て失敗時の緊急停止ハンドラ。

制御フローとデータフロー

1. **初期化**: init_process で APR を初期化し、グローバルプールを作成。 2. **引数解析**: apr_getopt を使用し、-D や -f などの引数を解析。ap_run_mode を設定して動作モード（通常起動、設定テスト、ダンプ等）を決定。 3. **設定読み込み**: ap_read_config で設定ファイルをツリー構造に読み込み、ap_process_config_tree で適用。 4. **フックの実行**: ap_run_pre_config, ap_run_post_config 等のフックを順次実行し、サーバーを構成。 5. **メインループ**: do-while ループ内で ap_run_mpm を呼び出し、サーバーの稼働を開始。設定変更（シグナル受信）が発生すると、ループ内で設定を再読み込みし、MPM を再起動する。

読者向けメモ

  • **メモリ管理**: pconf (設定用プール) は設定の再読み込み時にクリアされるため、設定関連のデータはここに保持する必要があります。
  • **エラー処理**: abort_on_oom が示す通り、メモリ不足は致命的なエラーとして扱われます。
  • **設計上の注意**: main() 内の do-while ループは、設定の再読み込みをサポートするための設計ですが、現状では設定ファイルを二度読みするなどの冗長な部分があり、将来的なリファクタリングの対象とされています。
  • **プラットフォーム**: WIN32 や NETWARE 向けの条件付きコンパイルが多用されており、クロスプラットフォーム対応のための複雑な分岐が含まれています。
