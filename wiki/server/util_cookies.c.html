server/util_cookies.c

概要

HTTP クッキー（RFC 2109 および RFC 2965）の生成、削除、および解析を行うためのユーティリティ関数群を提供します。Apache HTTP Server のリクエスト処理において、クッキーの読み書きを抽象化し、一貫したヘッダー操作を可能にします。

モジュール内での役割

server モジュールの一部として、HTTP プロトコルレベルでのクッキー管理を担います。主に mod_cookie などのモジュールや、クッキーベースの認証・セッション管理を行うコンポーネントから利用される下位レイヤーのライブラリです。

責務

  • **クッキーの生成と書き込み**: Set-Cookie および Set-Cookie2 ヘッダーを生成し、指定されたテーブルに追加します。
  • **クッキーの削除**: Max-Age=0 を設定することで、クライアント側でのクッキー破棄を促すヘッダーを生成します。
  • **クッキーの読み取りと抽出**: Cookie および Cookie2 ヘッダーから特定のクッキー値を抽出し、必要に応じてヘッダーから削除（秘匿）します。
  • **バリデーション**: クッキー名に不正な文字（=, ;, &）が含まれていないかを確認します。

重要なシンボル

  • ap_cookie_write / ap_cookie_write2: RFC 2109/2965 準拠のクッキーをヘッダーテーブルに追加します。
  • ap_cookie_remove / ap_cookie_remove2: クッキーを無効化するヘッダーを生成します。
  • ap_cookie_read: リクエストヘッダーから特定のクッキー値を検索・抽出します。
  • ap_cookie_check_string: クッキー名が仕様上安全か検証します。

制御フローとデータフロー

1. **書き込み/削除**: 可変長引数（...）で渡された apr_table_t のリストを va_list で走査し、各テーブルに対して apr_table_addn を用いてクッキー文字列を挿入します。 2. **読み取り**: ap_cookie_read が apr_table_do を介して extract_cookie_line コールバックを呼び出します。 3. **抽出処理**: extract_cookie_line は apr_strtok を使用してヘッダー文字列をトークン化し、対象のクッキーを特定します。重複がある場合はフラグを立て、不要なクッキーをフィルタリングした新しいテーブルを構築します。

直接依存

  • util_cookies.h: 公開インターフェース定義。
  • apr_lib.h, apr_strings.h: 文字列操作およびメモリ管理（apr_pool_t）。
  • http_config.h, http_core.h, http_log.h: Apache の設定、コア機能、およびログ出力機能。

関連コード

  • include/util_cookies.h: 本ファイルのヘッダーファイル。
  • server/util.c: Apache の汎用ユーティリティ関数群。
  • modules/session/: クッキーを利用したセッション管理モジュールの実装例。

読者向けメモ

  • **メモリ管理**: すべての文字列操作は request_rec のプール（r->pool）を使用しているため、明示的な解放は不要です。
  • **重複クッキー**: ap_cookie_read はクッキーの重複を検出すると APR_EGENERAL を返します。これはセキュリティ上の曖昧さを避けるための設計です。
  • **ヘッダーの書き換え**: ap_cookie_read で remove 引数を真にすると、元のヘッダーからクッキーが削除され、バックエンドに渡る情報が制御されます。
