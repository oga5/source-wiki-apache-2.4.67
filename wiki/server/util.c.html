server/util.c

概要

server/util.c は、Apache HTTP Server のコア機能で頻繁に使用される低レベルな文字列操作、パス正規化、設定ファイル解析、およびユーティリティ関数群を保持する汎用ライブラリです。HTTP プロトコル処理や設定読み込みに必要な「泥臭い」処理を抽象化し、一貫したインターフェースを提供します。

モジュール内での役割

module:server の基盤として、他の多くのソースファイル（config.c, protocol.c, request.c など）から呼び出されるユーティリティ層です。特に、文字列のパターンマッチング、URL のエスケープ処理、設定ファイルの行単位読み込みといった、HTTPD の動作に不可欠な共通処理を担います。

責務

  • **文字列操作と変換**: 大文字小文字を無視した比較、プレフィックス除去、トークン抽出、HTML/ログ用エスケープ。
  • **パスとURIの処理**: パスの正規化（.. の解決）、絶対パス判定、URL のアンエスケープ、ディレクトリ操作。
  • **設定ファイル解析**: ap_configfile_t を介した設定ファイルの読み込み、行継続の処理、トークン化。
  • **HTTP ヘッダー解析**: リスト形式のヘッダー値のパース、トークンリストの抽出、ETag 比較。
  • **プロセス・スレッド管理**: スレッドローカルストレージの管理、プロセス間通信の補助。
  • **セキュリティ**: タイミング攻撃耐性のある文字列比較、メモリ割り当て失敗時の強制終了処理。

重要なシンボル

文字列・パターンマッチング

  • ap_strcmp_match, ap_strcasecmp_match: ワイルドカード（*, ?）を含む文字列比較。
  • ap_strcasestr: 大文字小文字を無視した部分文字列検索。
  • ap_getword, ap_getword_conf: 区切り文字や引用符を考慮したトークン抽出。

パス・URI 処理

  • ap_normalize_path: パスの正規化（.. や // の解決）。
  • ap_unescape_url: URL のデコード。
  • ap_escape_path_segment, ap_escape_urlencoded: URL エンコード。

設定ファイル解析

  • ap_pcfg_openfile: 設定ファイルを開き、ap_configfile_t を初期化。
  • ap_cfg_getline: 設定ファイルから1行読み込み、行継続を処理。

セキュリティ・ユーティリティ

  • ap_memeq_timingsafe, ap_streq_timingsafe: タイミング攻撃を防ぐための定数時間比較関数。
  • ap_abort_on_oom: メモリ割り当て失敗時にログを出力してプロセスを終了させる。

制御フローとデータフロー

  • **設定読み込み**: ap_pcfg_openfile でファイルハンドルをラップし、ap_cfg_getline が内部で cfg_getstr を呼び出してバッファを埋めます。行継続（\）やコメント除去は ap_cfg_getline_core で行われます。
  • **URL デコード**: unescape_url がメインのロジックであり、x2c（16進数から文字への変換）を呼び出してインプレースで変換を行います。
  • **フォームデータ解析**: ap_parse_form_data は、入力フィルターからバケットブリゲードを読み込み、状態マシン（ap_form_type_t）を用いて application/x-www-form-urlencoded データをパースします。

読者向けメモ

  • **メモリ管理**: 多くの関数が apr_pool_t を引数に取り、メモリのライフサイクルをプールに依存させています。ap_pregcomp のように、クリーンアップハンドラを登録して自動解放を行う設計が随所に見られます。
  • **パフォーマンス**: ucharmap テーブルを用いた大文字小文字変換や、test_char.h による文字種判定など、頻繁に呼ばれる処理は最適化されています。
  • **セキュリティ**: ap_normalize_path はセキュリティ上の脆弱性（ディレクトリトラバーサル）を防ぐための重要な関数です。変更時は RFC 3986 等の仕様を十分に確認してください。
  • **省略範囲**: ap_varbuf 関連の動的バッファ管理は、大規模な文字列構築を効率化するためのものであり、本ファイル内でも重要な役割を果たしていますが、実装の詳細は util_varbuf.c に委譲されている部分もあります。
