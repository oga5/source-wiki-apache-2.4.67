server/util_script.c

概要

server/util_script.c は、Apache HTTP Server における CGI およびスクリプト実行に関連するユーティリティ関数群を提供します。主に、リクエスト環境変数の構築、CGI ヘッダーの解析、およびスクリプト出力の正規化を担います。

モジュール内での役割

server モジュールの一部として、CGI ゲートウェイやスクリプト実行エンジン（mod_cgi, mod_cgid など）が共通して利用するロジックをカプセル化しています。これにより、各モジュール間でのコード重複を避け、CGI 仕様への準拠を維持します。

責務

  • **環境変数管理**: HTTP リクエストヘッダーから CGI 環境変数への変換、およびシステム固有の環境変数の注入。
  • **スクリプトヘッダー解析**: スクリプトからの出力を読み取り、HTTP ヘッダーとして妥当性を検証し、request_rec に反映させる。
  • **URI/パス変換**: PATH_INFO や SCRIPT_NAME の計算、およびスクリプト実行に必要なパス情報の正規化。
  • **データ変換**: クエリ文字列の解析や、スクリプト出力のバッファリング・ストリーム処理。

重要なシンボル

環境変数生成

  • ap_create_environment: apr_table_t から exec 形式の環境変数配列を作成。
  • ap_add_common_vars: リクエストヘッダーやサーバー情報を基に、標準的な CGI 環境変数を r->subprocess_env に追加。
  • ap_add_cgi_vars: CGI/1.1 仕様に基づいたリクエスト固有の変数を設定。

ヘッダー解析 (Dispatch Family)

  • ap_scan_script_header_err_core_ex: スクリプト出力解析のコアエンジン。getsfunc コールバックを用いて入力を抽象化。
  • ap_scan_script_header_err: ファイルベースの解析用ラッパー。
  • ap_scan_script_header_err_brigade: apr_bucket_brigade ベースの解析用ラッパー。

内部ヘルパー

  • http2env: ヘッダー名を HTTP_ プレフィックス付きの環境変数名に変換。
  • getsfunc_FILE / getsfunc_BRIGADE / getsfunc_STRING: それぞれファイル、バケットブリゲード、文字列からヘッダー行を抽出するリーダー。

制御フローとデータフロー

1. **環境変数構築**: ap_add_common_vars が呼び出されると、r->headers_in を走査し、http2env を経由して環境変数テーブルを構築します。その後、OS 固有の環境変数を env2env で追加します。 2. **ヘッダー解析**: ap_scan_script_header_err_core_ex がメインループとなり、getsfunc を通じてスクリプト出力を 1 行ずつ読み取ります。

  • 空行（\0）に到達するまでヘッダーを解析し、Status や Location などの特殊ヘッダーを処理します。
  • 不正なヘッダーやタイムアウト時はエラーを返し、正常時は r->err_headers_out にマージします。

読者向けメモ

  • **メモリ管理**: すべての動的割り当ては r->pool を使用しており、リクエスト終了時に自動解放されます。
  • **セキュリティ**: Authorization ヘッダーの取り扱いなど、CGI 経由の認証情報漏洩を防ぐためのガードロジックが含まれています。
  • **EBCDIC**: APR_CHARSET_EBCDIC 定義時は、ASCII との変換ロジックがヘッダー解析に介入します。
  • **制約**: ap_scan_script_header_err 系関数は、スクリプトの出力が完了するまでブロックする可能性があるため、非同期 I/O が必要な場合は注意が必要です。
