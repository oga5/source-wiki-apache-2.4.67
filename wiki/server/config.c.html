server/config.c

概要

Apache HTTP Server の設定管理およびモジュールライフサイクル管理の中核を担うファイルです。設定ファイルのパース、モジュールの動的ロード、設定ベクトルのマージ、およびサーバーの初期化プロセスを統括します。

モジュール内での役割

server モジュールにおける「設定のエンジン」です。http_config.h で定義されたインターフェースを実装し、他のモジュールが提供する設定コマンドを統合して、サーバー全体の構成を構築します。

責務

  • **モジュール管理**: モジュールのロード、登録、削除、およびフックの管理。
  • **設定パース**: 設定ファイル（httpd.conf や .htaccess）の読み込みとツリー構造への変換。
  • **設定ベクトル管理**: モジュールごとの設定データ（ディレクトリ、サーバー、リクエスト単位）の作成、マージ、保持。
  • **コマンドディスパッチ**: 設定ファイル内のディレクティブを対応するモジュールのハンドラへ振り分け。

重要なシンボル

モジュール・フック管理

  • ap_top_module: ロード済みモジュールリストの先頭。
  • ap_add_module() / ap_remove_module(): モジュールの動的追加・削除。
  • ap_register_hooks(): モジュールのフック関数を登録。

設定ベクトル・マージ

  • ap_merge_per_dir_configs(): ディレクトリ設定のマージ（継承）。
  • create_default_per_dir_config(): デフォルトのディレクトリ設定を作成。
  • merger_func_cache: マージ関数をキャッシュし、高速なアクセスを実現。

設定パース・実行

  • ap_build_config(): 設定ファイルを読み込み、ap_directive_t ツリーを構築。
  • ap_walk_config(): 構築されたツリーを走査し、各ディレクティブを実行。
  • invoke_cmd(): 特定のディレクティブに対応するモジュールのコマンド関数を呼び出す。

制御フローとデータフロー

1. **初期化**: ap_setup_prelinked_modules() が静的モジュールを登録し、設定ベクトル領域を確保。 2. **パース**: ap_process_resource_config() がファイルを開き、ap_build_config() が行単位でディレクティブを解析。 3. **ツリー構築**: ap_build_config_sub() がディレクティブを ap_directive_t ノードとしてツリーに追加。 4. **実行**: ap_walk_config() がツリーを再帰的に走査し、invoke_cmd() を介して各モジュールの設定関数を起動。 5. **マージ**: リクエスト処理時やサーバー起動時に、ap_merge_per_dir_configs() が階層的な設定をマージして最終的な設定を決定。

読者向けメモ

  • **メモリ管理**: 設定データは主に apr_pool_t を使用して管理されます。ライフサイクルはサーバーの起動・再起動サイクルに依存します。
  • **パフォーマンス**: merger_func_cache は、モジュールリストを毎回辿るコストを避けるための最適化です。
  • **制約**: conf_vector_length は動的モジュールのロード数に影響されます。DYNAMIC_MODULE_LIMIT を超える場合は再コンパイルが必要です。
  • **省略**: 本ファイル内の ap_hook_* マクロ展開や、詳細なエラーハンドリングのロジックは、コードの可読性を優先して詳細な解説を省略しています。
