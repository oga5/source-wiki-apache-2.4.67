server/ssl.c

概要

server/ssl.c は、Apache HTTP Server における SSL/TLS 関連機能の抽象化レイヤーです。特定の SSL 実装（mod_ssl など）に依存せず、コアサーバーが SSL 機能を透過的に利用するためのインターフェースを提供します。

モジュール内での役割

本ファイルは、SSL 機能を実装するモジュールと、それを利用するコアサーバーや他のモジュールとの間の「仲介役」です。Apache のフック機構（Hooks）およびオプション関数（Optional Functions）を利用して、SSL 機能を動的に統合します。

責務

  • **SSL 抽象化**: SSL 関連の操作（接続の確認、変数検索、証明書管理、OCSP 応答取得）をフックとして定義し、実行する。
  • **後方互換性の維持**: 以前のオプション関数ベースの API と、新しいフックベースの API を統合し、古いモジュールとの互換性を保つ。
  • **SSL ライフサイクル管理**: ap_setup_ssl_optional_fns を通じて、SSL 関連のオプション関数を初期化・登録する。

重要なシンボル

フック定義（SSL 抽象化）

  • ssl_conn_is_ssl: 接続が SSL を使用しているか判定。
  • ssl_var_lookup: SSL 関連の環境変数（証明書情報など）を検索。
  • ssl_add_cert_files / ssl_add_fallback_cert_files: 証明書と鍵ファイルの登録。
  • ssl_answer_challenge: ACME 等のチャレンジ応答処理。
  • ssl_ocsp_prime_hook / ssl_ocsp_get_resp_hook: OCSP ステープリング関連の処理。
  • ssl_bind_outgoing: アウトバウンド接続（プロキシ等）への SSL 適用。

API エントリポイント

  • ap_ssl_conn_is_ssl: 接続が SSL かを判定する公開 API。
  • ap_ssl_var_lookup: SSL 変数取得の公開 API。
  • ap_ssl_bind_outgoing: アウトバウンド接続の SSL 設定を適用する公開 API。

制御フローとデータフロー

1. **初期化**: ap_setup_ssl_optional_fns が起動時に呼ばれ、既存の SSL モジュールが提供するオプション関数を検索・キャッシュします。 2. **フック実行**: ap_run_ssl_* 系関数が呼ばれると、登録されたフックが実行されます。 3. **フォールバック**: フックが DECLINED を返した場合、またはフックが未定義の場合、キャッシュされた古いオプション関数（module_ssl_*）を呼び出して互換性を維持します。 4. **エラーハンドリング**: ap_ssl_bind_outgoing などでは、フックの戻り値に基づき、SSL が必須であるにもかかわらず有効化に失敗した場合に APLOG_ERR を出力します。

読者向けメモ

  • **メモリ管理**: apr_pool_t を介したメモリ管理が行われています。特に ssl_var_lookup 等で返される文字列は、渡されたプールに依存するため注意が必要です。
  • **設計思想**: 本ファイルは「SSL 実装」ではなく「SSL インターフェース」です。ロジックの大部分はフックを通じて外部モジュールに委譲されます。
  • **省略された詳細**: OCSP や証明書管理の具体的な暗号学的処理は、本ファイルには含まれず、すべてフック先のモジュールに委譲されています。
  • **注意点**: APLOG_MODULE_INDEX が AP_CORE_MODULE_INDEX に再定義されており、コアモジュールとしてログ出力が行われます。
