server/util_time.c

概要

Apache HTTP Server における、高頻度な時刻変換およびフォーマット処理を最適化するためのユーティリティモジュールです。

モジュール内での役割

server モジュールの一部として、ログ出力やヘッダー生成など、サーバー内で頻繁に発生する「現在時刻の文字列化」処理を高速化します。標準的な APR（Apache Portable Runtime）の時刻関数をラップし、キャッシュ層を提供することでシステムコール呼び出しを削減します。

責務

  • **時刻キャッシュの管理**: 直近数秒間の時刻情報を保持するリングバッファを維持し、重複する時刻変換処理を回避する。
  • **高速な時刻分解**: apr_time_exp_t 構造体への変換を、キャッシュヒット時はメモリコピーのみで完結させる。
  • **フォーマット変換**: ctime 形式や RFC822 形式への文字列変換を、キャッシュされた分解済み時刻を用いて効率的に行う。

重要なシンボル

  • exploded_time_cache_element: 時刻分解結果を保持するキャッシュ構造体。整合性検証用のタイムスタンプを含む。
  • cached_explode(): キャッシュの読み書きと、競合発生時のフォールバック処理を行う内部関数。
  • ap_explode_recent_localtime() / ap_explode_recent_gmt(): キャッシュを利用した時刻分解の公開インターフェース。
  • ap_recent_ctime_ex(): 拡張オプション（マイクロ秒、ISO 8601 形式、タイムゾーンオフセット）をサポートするフォーマット関数。

制御フローとデータフロー

1. **キャッシュ探索**: cached_explode が呼び出されると、現在の秒数からリングバッファのインデックスを計算します。 2. **整合性チェック**: キャッシュ内のタイムスタンプが現在の秒と一致するか確認します。 3. **競合回避**: マルチスレッド環境での書き込み競合を許容しつつ、読み取り時に「スナップショット」を採取して整合性を検証します。データが不整合であれば、標準の apr_time_exp_* 関数を直接呼び出します。 4. **フォーマット**: ap_recent_ctime_ex 等は、分解された時刻構造体から文字列バッファへ直接数値を書き込みます。

直接依存

  • util_time.h: 公開インターフェースおよび定数定義。
  • apr_time.h: APR の時刻処理プリミティブ。

関連コード

  • server/log.c: ログ出力時に本モジュールの ap_recent_ctime 等が頻繁に利用されます。
  • server/protocol.c: HTTP レスポンスヘッダーの Date フィールド生成に利用されます。

読者向けメモ

  • **スレッド安全性**: 本モジュールはロックフリーな設計を採用しています。キャッシュ更新中の不整合を検知して再計算するアプローチをとっているため、高負荷時でもロックによるボトルネックが発生しません。
  • **キャッシュの制約**: AP_TIME_RECENT_THRESHOLD（デフォルト15秒）以内の時刻に対してのみ有効です。それより古い時刻を扱う場合は、標準の APR 関数を直接使用してください。
  • **拡張性**: ap_recent_ctime_ex は、将来的なフォーマット変更にも対応できるよう設計されています。新しい形式が必要な場合は、この関数を拡張することを検討してください。
