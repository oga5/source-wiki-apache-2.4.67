server/eor_bucket.c

概要

Apache HTTP Server における「EOR (End of Request)」バケットを定義するソースファイルです。このバケットは、リクエスト処理の終了を通知し、リクエストに関連するリソースのクリーンアップやログ記録をトリガーするために使用されます。

モジュール内での役割

server モジュールの一部として、フィルタチェーンを通じてリクエストの完了を伝播させる役割を担います。データそのものを運ぶのではなく、リクエストのライフサイクル管理を行うためのメタデータバケットとして機能します。

責務

  • **リクエスト終了のシグナル**: フィルタチェーンの末尾に到達したことを示す EOR バケットの生成。
  • **リソースの解放**: リクエストに関連付けられたプール（request_pool）の破棄。
  • **事後処理の実行**: リクエスト終了時のログ記録（log_transaction）や、スコアボードの統計更新のトリガー。

重要なシンボル

  • ap_bucket_type_eor: EOR バケットの型定義。バケットの破棄や読み込み時の挙動を規定します。
  • ap_bucket_eor_create: EOR バケットを生成し、リクエストプールに対するクリーンアップ処理を登録する公開関数。
  • eor_bucket_cleanup: プール破棄時に呼び出される関数。ログ記録や統計更新を実行します。

制御フローとデータフロー

1. **生成**: ap_bucket_eor_create が呼び出されると、request_rec を保持するバケットが作成され、apr_pool_pre_cleanup_register を通じて eor_bucket_cleanup がリクエストプールに登録されます。 2. **伝播**: EOR バケットがフィルタチェーンを通過します。 3. **破棄**: バケットが破棄される際、eor_bucket_destroy が実行されます。ここでリクエストプールが破棄され、その過程で登録済みの eor_bucket_cleanup が発火します。 4. **終了処理**: eor_bucket_cleanup 内で ap_run_log_transaction が実行され、最終的なログ出力と統計情報の更新が行われます。

直接依存

  • httpd.h: Apache HTTP Server のコア定義。
  • http_request.h: request_rec 構造体およびリクエスト処理関連。
  • http_protocol.h: プロトコル処理およびログフック。
  • scoreboard.h: サーバーの稼働状況管理。

関連コード

  • server/eoc_bucket.c: 「End of Connection」バケット。接続終了時の処理を担当する類似のメタデータバケット。
  • server/protocol.c: リクエスト処理のメインフロー。ここで EOR バケットが生成されることが多い。
  • server/log.c: ap_run_log_transaction の実装元。

読者向けメモ

  • **クリーンアップの順序**: apr_pool_pre_cleanup_register を使用しているのは、リクエスト処理の終了時にモジュールがまだサブプールを必要としている可能性があるためです。
  • **メタデータバケット**: このバケットは eor_bucket_read が常に空を返すように設計されており、データ転送ではなく「イベント通知」のみを目的としています。
  • **二重解放の防止**: eor_bucket_cleanup 内で *rp = NULL とすることで、バケット破棄とプール破棄の順序が前後しても安全に動作するよう設計されています。
