server/util_expr_scan.c

概要

本ファイルは、Apache HTTP Server の式評価エンジン（ap_expr）で使用される字句解析器（Lexer）のソースコードです。Flex によって生成された再入可能（reentrant）なスキャナであり、入力文字列をトークン化してパーサー（util_expr_parse.c）に供給します。

モジュール内での役割

server モジュール内の式評価サブシステムにおいて、文字列や設定ファイル内の式を解析するためのフロントエンドを担います。util_expr_parse.c（パーサー）と密接に連携し、util_expr_private.h で定義されたコンテキスト構造体を使用して状態を保持します。

責務

  • **字句解析**: 入力ストリームを解析し、演算子、識別子、文字列、正規表現などのトークンを抽出する。
  • **状態管理**: 文字列内、変数展開内、正規表現内といった複数のスキャン状態（Start Conditions）を管理する。
  • **バッファ管理**: 入力データの読み込み、バッファの動的拡張、および再入可能なスキャナ状態の保持。
  • **エラー報告**: 不正なエスケープシーケンスや未終了の文字列などの構文エラーを検出し、コンテキスト経由で通知する。

重要なシンボル

スキャナ制御・ライフサイクル

  • ap_expr_yylex: メインの字句解析関数。
  • ap_expr_yylex_init / ap_expr_yylex_destroy: スキャナインスタンスの生成と破棄。
  • yyscan_t: スキャナの状態を保持する不透明なポインタ型。

状態管理（Start Conditions）

  • INITIAL: デフォルトの解析状態。
  • str: 文字列リテラル解析中。
  • var: 変数展開（%{...}）解析中。
  • regex: 正規表現解析中。

コンテキスト・データ構造

  • yyguts_t: 再入可能なスキャナの全状態を保持する内部構造体。
  • ap_expr_parse_ctx_t (via yyextra): 解析コンテキスト。入力バッファ、メモリプール、エラーメッセージなどを保持。

制御フローとデータフロー

1. **初期化**: ap_expr_yylex_init で yyguts_t 構造体が確保され、初期状態が設定されます。 2. **入力供給**: YY_INPUT マクロが yyextra->inputptr からデータを読み込みます。 3. **トークン抽出**: ap_expr_yylex が呼び出されると、Flex が生成した DFA（決定性有限オートマトン）テーブルに基づき、入力文字からトークンをマッチングします。 4. **状態遷移**: BEGIN() マクロや yy_push_state() / yy_pop_state() を使用して、文字列や変数展開などのサブ状態へ遷移します。 5. **トークン返却**: マッチしたルールに基づき、yylval に値を格納し、トークン種別（T_ID, T_STRING 等）を返します。

直接依存

  • util_expr_private.h: 解析コンテキスト定義。
  • util_expr_parse.h: トークン定義（YYSTYPE 等）。
  • stdio.h, string.h, stdlib.h: 標準 C ライブラリ。
  • errno.h
  • inttypes.h
  • unistd.h

関連コード

  • util_expr_parse.y (生成元): 文法定義。
  • util_expr_eval.c: 解析された式ツリーの評価ロジック。

読者向けメモ

  • **メモリ所有権**: yyextra（ap_expr_parse_ctx_t）は呼び出し元が管理するメモリプール（apr_pool_t）を使用します。スキャナ自体が確保するバッファは ap_expr_yyfree で適切に解放される必要があります。
  • **再入可能性**: 本スキャナは yyscan_t を引数に取る再入可能な設計です。グローバル変数に依存しないため、マルチスレッド環境下での安全な並列解析が可能です。
  • **実装の性質**: 本ファイルは Flex によって自動生成されたコードをベースにしているため、手動編集は推奨されません。変更が必要な場合は util_expr_scan.l を修正してください。
  • **省略された詳細**: Flex 内部の DFA テーブル（yy_accept, yy_base 等）は非常に巨大であるため、本ページでは構造的な役割に焦点を当て、テーブルの内容は省略しています。
