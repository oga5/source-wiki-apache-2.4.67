server/mpm_common.c

概要

server/mpm_common.c は、Apache HTTP Server のマルチプロセッシングモジュール（MPM）間で共有される共通機能、設定ディレクティブ、およびライフサイクル管理ロジックを実装するファイルです。特定のプラットフォームに依存せず、マルチプロセスサーバー全般で必要とされるユーティリティを提供します。

モジュール内での役割

server モジュール内において、各 MPM（worker, event, prefork 等）が共通して利用する「基盤」として機能します。MPM 固有のロジックを各実装から分離し、設定の標準化とコードの重複排除を担います。

責務

  • **MPM 共通設定の管理**: PidFile, MaxConnectionsPerChild, CoreDumpDirectory 等のグローバル設定のパースと保持。
  • **プロセスライフサイクル管理**: 子プロセスの終了監視、世代（Generation）管理、および終了時のフック実行。
  • **OS 抽象化ユーティリティ**: ユーザー名/グループ名の ID 変換、ソケットオプション（TCP_NODELAY）の制御。
  • **MPM フックの定義**: MPM 関連の拡張ポイント（monitor, child_status, drop_privileges 等）の定義と実装。

重要なシンボル

設定変数 (AP_DECLARE_DATA)

  • ap_pid_fname: PID ファイルのパス。
  • ap_max_requests_per_child: 子プロセスが終了するまでのリクエスト上限数。
  • ap_coredump_dir: コアダンプ出力先ディレクトリ。
  • ap_thread_stacksize: スレッドスタックサイズ。

プロセス管理・フック

  • ap_wait_or_timeout(): 子プロセスの終了を待機し、定期的に monitor フックを呼び出す。
  • ap_core_child_status(): 子プロセスの状態変化を追跡し、世代ごとの終了処理を管理する。
  • ap_mpm_register_timed_callback(): タイマーベースのコールバック登録用ラッパー。

ユーティリティ

  • ap_uname2id(), ap_gname2id(): 名前から UID/GID への変換。
  • ap_sock_disable_nagle(): TCP_NODELAY を設定し、Nagle アルゴリズムを無効化。

制御フローとデータフロー

  • **設定フロー**: mpm_common_pre_config() が起動時に呼ばれ、各設定変数をデフォルト値で初期化します。その後、ap_mpm_set_* 系関数が設定ファイルを読み込む際に値を更新します。
  • **プロセス監視フロー**: ap_wait_or_timeout() はメインループ内で呼び出され、apr_proc_wait_all_procs を非ブロックで実行します。一定回数ごとに ap_run_monitor() を実行し、サーバーの健全性を維持します。
  • **世代管理フロー**: mpm_gen_info_t 構造体と APR_RING を使用して、プロセス世代ごとのアクティブ数を追跡します。ap_core_child_status() が MPM_CHILD_EXITED を検知すると、アクティブ数が 0 になった時点で end_gen() を呼び出し、世代終了フックを発火させます。

読者向けメモ

  • **メモリ所有権**: geninfo などの管理構造体は s->process->pool（プロセス生存期間）に割り当てられており、サーバー終了まで生存します。
  • **制約事項**: gen_head_init 変数による初期化判定など、一部に古い設計の名残が見られます。
  • **注意点**: ap_wait_or_timeout 内の apr_sleep は、子プロセスが終了していない場合に 1 秒間ブロッキングします。高負荷環境での挙動に影響を与える可能性があるため注意してください。
  • **省略範囲**: 本ファイルは MPM の「共通基盤」であるため、特定の MPM（event 等）に深く依存する複雑なキューイングやスレッドプール管理ロジックは含まれていません。これらは各 MPM 実装側に委譲されています。
