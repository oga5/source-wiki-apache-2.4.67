server/util_fcgi.c

概要

FastCGI プロトコル（RFC 3652）のバイナリ構造を Apache HTTP Server 内で扱うためのユーティリティ群を提供します。主に FastCGI レコードのヘッダーおよびボディのシリアライズ・デシリアライズ、ならびに環境変数（apr_table_t）の FastCGI 形式へのエンコードを担います。

モジュール内での役割

server モジュールの一部として、FastCGI を利用するバックエンド（mod_proxy_fcgi 等）と Apache の内部データ構造の橋渡しを行います。プロトコルの低レイヤーなバイト操作を抽象化し、一貫したインターフェースを提供します。

責務

  • **プロトコル構造の変換**: ap_fcgi_header および ap_fcgi_begin_request_body 構造体と、ネットワーク転送用のバイト配列間の相互変換。
  • **環境変数のエンコード**: FastCGI の PARAMS レコードで要求される「長さ＋データ」形式への環境変数の変換。
  • **バッファ管理**: エンコード時のバッファオーバーフロー防止と、分割送信のためのイテレータ管理。

重要なシンボル

ヘッダー・ボディ操作

  • ap_fcgi_header_to_array / from_array: ヘッダー構造体とバイト配列の変換。
  • ap_fcgi_header_fields_from_array: ヘッダーの各フィールドを個別の変数に展開。
  • ap_fcgi_fill_in_header / fill_in_request_body: 構造体の各フィールドを安全に初期化。

環境変数エンコード

  • ap_fcgi_encoded_env_len: 指定されたバッファサイズ内に収まる環境変数の総バイト数を計算し、処理すべき要素のインデックスを更新。
  • ap_fcgi_encode_env: apr_table_t から FastCGI 形式のバイトストリームへ変換。

制御フローとデータフロー

1. **ヘッダー処理**: ap_fcgi_fill_in_header でヘッダーを構築し、ap_fcgi_header_to_array でバイト列に変換して送信バッファへコピーします。 2. **環境変数エンコード**:

  • ap_fcgi_encoded_env_len を呼び出し、現在のバッファ容量でエンコード可能な要素数とバイト数を算出します。
  • ap_fcgi_encode_env がその範囲内で apr_table_t を走査し、キーと値の長さを FastCGI 形式（1バイトまたは4バイトの長さフィールド）でエンコードしてバッファに書き込みます。
  • starting_elem ポインタを介して、複数回にわたる分割送信の進捗を保持します。

直接依存

  • httpd.h: Apache のコアデータ構造（request_rec 等）。
  • http_core.h: コアモジュールの定義。
  • http_log.h: エラーログ出力用。
  • util_fcgi.h: プロトコル定義と構造体宣言。

関連コード

  • modules/proxy/mod_proxy_fcgi.c: 本ユーティリティの主要な利用者。FastCGI 通信のロジックが実装されています。

読者向けメモ

  • **メモリ所有権**: 本ファイル内の関数はメモリの割り当てを行いません。呼び出し元が提供したバッファ（unsigned char a[] や void *buffer）に対して操作を行います。
  • **境界条件**: ap_fcgi_encode_env は APR_ENOSPC を返し、バッファ不足を明示的に通知します。
  • **レイアウト**: FastCGI の長さフィールドは、値が 127 を超える場合に 4 バイト（最上位ビットが 1）を使用する可変長エンコーディングを採用しており、ap_fcgi_encode_env 内でこのビット演算が行われています。
  • **省略された領域**: FastCGI のパケット受信時のパディング処理や、ソケット I/O 自体は本ファイルには含まれていません。これらはプロトコル定義のサポートに特化しているためです。
