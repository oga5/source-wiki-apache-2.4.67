server/mpm/winnt/mpm_winnt.h

概要

mpm_winnt.h は、Windows 版 Apache HTTP Server におけるマルチプロセッシングモジュール（MPM）のインターフェース定義ファイルです。Windows 特有のサービス管理、イベントログ出力、および親プロセスと子プロセス間の通信に必要な定数や関数プロトタイプを定義しています。

モジュール内での役割

server/mpm/winnt ディレクトリ内の各ソースファイル（service.c, nt_eventlog.c, mpm_winnt.c, child.c）が共有する共通ヘッダーです。各コンポーネントが互いの関数を呼び出し、グローバルな状態を共有するための「契約」を定義しています。

責務

  • **Windows サービス管理**: サービスのインストール、起動、停止、および設定情報の取得に関するインターフェースの提供。
  • **プロセス間通信の抽象化**: 親プロセスへのシグナル送信（シャットダウンや再起動）や、子プロセスのライフサイクル管理に必要な定数の定義。
  • **プラットフォーム固有機能の公開**: Windows イベントログへのエラー出力や、コンソールハンドラの制御。

重要なシンボル

  • ap_signal_parent_e: 親プロセスに対して送るシグナルの種類（シャットダウン、再起動、グレースフル再起動）を定義する列挙型。
  • mpm_service_*: Windows サービスとしての Apache を制御するための関数群。
  • ap_threads_per_child: 子プロセス内で生成されるスレッド数を保持するグローバル変数。
  • winnt_mpm_state: 現在の MPM の状態（起動中、実行中など）を追跡する変数。

制御フローとデータフロー

本ファイル自体は宣言のみを行うため、直接的なフローは持ちません。しかし、ここで定義された ap_signal_parent 関数は、子プロセスから親プロセスへイベントを通知する際の中心的な役割を果たします。また、mpm_service_ 系関数は、service.c で実装されたロジックを介して Windows Service Control Manager (SCM) との対話フローを構築します。

直接依存

  • apr_proc_mutex.h: プロセス間での排他制御に使用。
  • ap_listen.h: リスニングソケットの管理に関連する定義。

関連コード

  • mpm_winnt.c: MPM のメインロジック。本ヘッダーで宣言された変数の実体や、MPM の状態管理が含まれます。
  • service.c: Windows サービスとしての登録・制御ロジックの実装。
  • child.c: 子プロセスのメインループおよびスレッド生成処理。
  • nt_eventlog.c: Windows イベントログへの出力処理。

読者向けメモ

  • 本ファイルは Windows 環境専用です。#ifdef WIN32 で保護されたコードベースにおいて、プラットフォーム固有の挙動を理解するための起点となります。
  • ap_signal_parent は、Windows におけるプロセス間連携の要です。親プロセスがどのように子プロセスの終了や再起動を検知しているかを追う際は、この関数の実装（mpm_winnt.c 内）を参照してください。
  • SERVICE_APACHE_RESTART などの定数は、Windows SCM からの制御コードと密接に関係しています。
