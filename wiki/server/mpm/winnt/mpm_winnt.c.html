server/mpm/winnt/mpm_winnt.c

概要

Windows環境におけるApache HTTP Serverのマルチプロセッシングモジュール（MPM）のメイン実装です。Windowsのプロセスモデル（親プロセスが子プロセスを管理し、子プロセスが複数のスレッドでリクエストを処理する）を制御します。

モジュール内での役割

mpm_winnt モジュールの中心であり、親プロセスのライフサイクル管理、子プロセスの生成、およびプロセス間通信（IPC）の調整を行います。Windowsサービスとしての動作や、コマンドラインからの制御（-k オプション）を統合する役割を担います。

責務

  • **プロセス管理**: 親プロセスによる子プロセスの生成、監視、および終了処理。
  • **IPCと同期**: 名前付きイベント（Event）やミューテックスを用いた、親・子プロセス間のシグナル伝達とリソース共有。
  • **設定管理**: ThreadsPerChild や ThreadLimit などのMPM固有ディレクティブの解析と検証。
  • **サービス統合**: Windowsサービスコントロールマネージャ（SCM）との連携およびサービスライフサイクルの管理。

重要なシンボル

プロセス・ライフサイクル管理

  • master_main(): 親プロセスのメインループ。子プロセスの監視と、シャットダウン/再起動イベントの待機を行う。
  • create_process(): 子プロセスを生成し、必要なハンドル（パイプ、イベント、ミューテックス）を継承させる。
  • winnt_run(): MPMの実行エントリポイント。親と子の役割分担を決定する。

IPC・同期

  • ap_signal_parent(): 親プロセスに対してシャットダウンや再起動のシグナルを送る。
  • send_handles_to_child() / get_handles_from_parent(): 子プロセス起動時に、共有メモリや同期オブジェクトのハンドルをパイプ経由で受け渡す。
  • start_mutex: 子プロセスがリスニングソケットを安全に受け入れるための同期用ミューテックス。

設定・フック

  • winnt_rewrite_args(): 起動引数を解析し、サービスモードやコマンドラインオプションを処理する。
  • winnt_hooks(): Apacheの各フェーズ（pre_config, post_config, child_init等）に処理を登録する。

制御フローとデータフロー

1. **初期化**: winnt_rewrite_args が引数を解析し、サービスモードか否かを判定。 2. **起動**: winnt_run が親プロセスとして master_main を呼び出し、create_process で子プロセスを生成。 3. **ハンドル継承**: create_process はパイプを作成し、DuplicateHandle を用いて子プロセスにスコアボードやイベントのハンドルを渡す。 4. **監視**: master_main は WaitForMultipleObjects を使用し、子プロセスの終了、シャットダウンイベント、再起動イベントのいずれかを待機する。 5. **終了/再起動**: イベントが発生すると、対応するフラグを立ててプロセスを再生成または終了させる。

読者向けメモ

  • **メモリ所有権**: pconf（設定用プール）は親プロセスの生存期間中保持されます。子プロセスへ渡されるハンドルは、DuplicateHandle によって子プロセスのコンテキストで有効化されます。
  • **制約**: AP_ENABLE_V4_MAPPED はサポートされていません。
  • **設計の注意**: 本ファイルはWindows特有のAPI（CreateEvent, WaitForMultipleObjects, WSADuplicateSocket）を多用しており、移植性はありません。
  • **省略箇所**: リクエスト処理の詳細なスレッドプールロジックは child.c に委譲されており、本ファイルはあくまで「プロセスのオーケストレーション」に焦点を当てています。
