server/mpm/winnt/nt_eventlog.c

概要

Windows環境において、Apacheの標準エラー出力（stderr）をWindowsイベントログへリダイレクトするためのユーティリティです。サービスとして実行される際、コンソールを持たないプロセスから出力されるエラーメッセージを、Windowsのイベントビューアで確認可能にします。

モジュール内での役割

server/mpm/winnt モジュールの一部として、主にサービス起動時のエラーハンドリングを担います。標準的なログファイルへの書き出しが困難な初期段階や、サービス特有のログ出力要件を満たすためのブリッジとして機能します。

責務

  • **パイプの構築**: stderr をキャプチャするための匿名パイプを作成し、プロセス標準の stderr をそのパイプへリダイレクトする。
  • **イベントログへの転送**: 専用スレッドでパイプを監視し、受信した文字列を ReportEvent API を介してWindowsイベントログへ書き出す。
  • **レジストリ設定**: イベントログのソースとしてApacheを登録し、メッセージ定義（netmsg.dll）を適切に設定する。

重要なシンボル

  • mpm_nt_eventlog_stderr_open: サービス起動時に呼び出され、パイプと監視スレッドを初期化するエントリポイント。
  • service_stderr_thread: パイプを読み取り、イベントログへ変換するバックグラウンドスレッド。
  • mpm_nt_eventlog_stderr_flush: サービス終了時にパイプをクローズし、スレッドの終了を待機するクリーンアップ関数。

制御フローとデータフロー

1. **初期化**: mpm_nt_eventlog_stderr_open がパイプを作成し、service_stderr_thread を起動。stderr をパイプの書き込み端へ dup2 する。 2. **監視**: service_stderr_thread は ReadFile でパイプをブロック読み取りし、改行コードを区切りとしてメッセージをバッファリングする。 3. **転送**: バッファが埋まるか改行を検知すると、ReportEvent を呼び出し、Windowsイベントログへメッセージを送信する。 4. **終了**: mpm_nt_eventlog_stderr_flush が呼ばれると、パイプが閉じられ、スレッドが終了処理を行う。

直接依存

  • httpd.h, http_log.h: Apacheの基本定義とログ出力基盤。
  • mpm_winnt.h: MPM WinNT固有の定義。
  • ap_regkey.h: Windowsレジストリ操作用API。
  • apr_portable.h, apr_strings.h: APRのポータブルなファイル操作および文字列処理。
  • apr_lib.h

関連コード

  • service.c: Windowsサービスとしてのライフサイクル管理。本ファイルはこのサービス起動プロセスから呼び出される。
  • mpm_winnt.c: MPMのメインループ。ログ出力の初期化タイミングを制御する。

読者向けメモ

  • **スレッド同期**: stderr_ready イベントを使用して、スレッドが完全に起動しイベントソースの登録が完了するまでメインスレッドを待機させています。
  • **メッセージDLL**: イベントログのメッセージ定義には netmsg.dll を流用しています。これはApache専用のメッセージファイルを作成するコストを避けるための実装です。
  • **注意点**: この実装は stderr をパイプへ差し替えるため、他のモジュールが stderr を直接操作する場合に競合や予期せぬ挙動を引き起こす可能性があります。必ずサービス起動の初期段階で呼び出される必要があります。
