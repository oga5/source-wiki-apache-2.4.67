server/mpm/winnt/mpm_default.h

概要

Windows 版 Apache HTTP Server (WinNT MPM) における、スレッドおよびプロセス管理のデフォルト設定値を定義するヘッダーファイルです。

モジュール内での役割

server/mpm/winnt モジュール全体で使用される、コンパイル時の定数定義を提供します。設定ファイル（httpd.conf）で明示的な指定がない場合に適用されるフォールバック値や、安全な動作を保証するための上限値を規定します。

責務

  • **スレッド制限の定義**: 子プロセスあたりのスレッド数に関するデフォルト値および上限値の提供。
  • **プロセス制限の定義**: WinNT MPM における単一プロセスモデルを強制するためのハードリミットの定義。
  • **メモリ消費の抑制**: スコアボード（共有メモリ）のサイズを決定するパラメータの基準値設定。

重要なシンボル

  • DEFAULT_THREAD_LIMIT: 設定がない場合の最大スレッド数のデフォルト値（1920）。
  • MAX_THREAD_LIMIT: 設定可能なスレッド数の絶対上限（15000）。誤設定を防ぐためのコンパイル時ガード。
  • DEFAULT_THREADS_PER_CHILD: 子プロセス起動時に生成されるデフォルトのスレッド数（64）。
  • HARD_SERVER_LIMIT: WinNT MPM におけるプロセス数の上限（1）。Windows 版のアーキテクチャ上、単一プロセスで動作することを明示します。

制御フローとデータフロー

本ファイルは純粋な定数定義ヘッダーであるため、実行時の制御フローは存在しません。プリプロセッサがこれらの値を参照し、mpm_winnt.c 等のソースコード内でスレッドプールやスコアボードのメモリ割り当てサイズを決定する際に使用されます。

直接依存

本ファイルは他のヘッダーファイルをインクルードしておらず、純粋なマクロ定義のみで構成されています。

関連コード

  • server/mpm/winnt/mpm_winnt.c: 本ファイルで定義された定数を使用して、実際のプロセス・スレッドの生成や管理を行うメインロジック。
  • server/mpm/winnt/mpm_winnt.h: MPM 固有の構造体や型定義が記述されており、本ファイルと併せて MPM の構成を理解する上で重要です。

読者向けメモ

  • **変更時の注意**: MAX_THREAD_LIMIT を変更する場合、スコアボードのメモリ割り当てに直接影響するため、システム全体のメモリ消費量に注意してください。
  • **アーキテクチャの制約**: HARD_SERVER_LIMIT が 1 に固定されている点は、WinNT MPM がマルチプロセスではなく、マルチスレッドによる並列処理を主軸に設計されていることを示しています。
  • **設定の優先順位**: ここで定義されている値は、httpd.conf で ThreadLimit や ThreadsPerChild ディレクティブが指定された場合、それらの値によって上書きされます。
