server/mpm/winnt/child.c

概要

Windows 版 Apache HTTP Server におけるマルチプロセッシングモジュール（MPM）の「子プロセス」のライフサイクルと、接続処理の基盤を実装するファイルです。Windows の I/O Completion Port (IOCP) を活用した非同期接続モデルを管理します。

モジュール内での役割

mpm_winnt モジュールの一部として、子プロセスの起動、リスナー（Accept）スレッドの管理、およびワーカー（Worker）スレッドへの接続受け渡しを担います。mpm_winnt.c がプロセス全体の制御を行うのに対し、本ファイルは子プロセス内部の実行環境を完結させます。

責務

  • **接続コンテキスト管理**: winnt_conn_ctx_t を用いた接続情報のプールと再利用。
  • **リスナー・スレッド**: AcceptEx または標準 accept() を用いた接続受け入れと IOCP への通知。
  • **ワーカー・スレッド**: IOCP を監視し、接続を順次処理するメインループの実行。
  • **プロセス制御**: 子プロセスの正常終了、強制終了、および親プロセス監視のイベントループ。

重要なシンボル

接続コンテキスト

  • winnt_conn_ctx_t: 接続状態を保持する構造体。OVERLAPPED 構造体を含み、IOCP との連携に使用。
  • mpm_get_completion_context(): プールからコンテキストを取得、または新規作成。
  • mpm_recycle_completion_context(): 処理済みコンテキストを再利用キューへ戻す。

スレッド・メイン関数

  • winnt_accept(): リスナー・スレッドの本体。接続を待ち受け、PostQueuedCompletionStatus でワーカーへ通知。
  • worker_main(): ワーカー・スレッドの本体。GetQueuedCompletionStatus で接続を待機し、HTTP 接続処理を実行。
  • child_main(): 子プロセスのメイン制御スレッド。スレッド生成、イベント監視、シャットダウン処理を統括。

状態管理

  • ThreadDispatchIOCP: 接続受け渡しに使用される I/O Completion Port ハンドル。
  • qhead, qtail: 接続コンテキストの再利用キュー。

制御フローとデータフロー

1. **初期化**: child_main が ThreadDispatchIOCP を作成し、worker_main スレッド群を起動。 2. **接続受け入れ**: winnt_accept がリスナーソケットで待機。接続確立後、PostQueuedCompletionStatus を介して ThreadDispatchIOCP に IOCP_CONNECTION_ACCEPTED を送信。 3. **ワーカー処理**: worker_main は GetQueuedCompletionStatus でブロックし、通知を受け取ると winnt_conn_ctx_t を取得して HTTP 接続処理（ap_process_connection）を開始。 4. **終了処理**: child_main が exit_event を検知すると、リスナーを停止し、ワーカーに IOCP_SHUTDOWN を送信して全スレッドの終了を待機。

読者向けメモ

  • **メモリ管理**: pchild プールが子プロセスの生存期間を管理します。winnt_conn_ctx_t の再利用はメモリ断片化を防ぐための最適化ですが、エラーパスで一部リークする可能性がある旨がソース内にコメントされています。
  • **AcceptEx**: Windows 固有の高性能な接続受け入れ API です。AcceptFilter 設定により none（標準 accept）と connect（AcceptEx）を切り替え可能です。
  • **シャットダウン**: g_blocked_threads を用いて、終了時に全ワーカーが確実に IOCP から抜けるよう調整しています。
  • **省略箇所**: apr_proc_other_child_refresh_all 等の外部プロセス監視ロジックは、ビルド環境（APR_HAS_OTHER_CHILD）に依存するため、環境によって挙動が異なります。
