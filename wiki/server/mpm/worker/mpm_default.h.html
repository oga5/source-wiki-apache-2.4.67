server/mpm/worker/mpm_default.h

概要

Apache HTTP Server の Worker MPM（Multi-Processing Module）における、プロセスおよびスレッド管理のデフォルト値を定義するヘッダーファイルです。

モジュール内での役割

Worker MPM の動作パラメータの基準を提供します。設定ファイル（httpd.conf）で明示的な指定がない場合に適用されるフォールバック値として機能します。

責務

  • プロセス生成数、スレッド数、およびアイドル状態のプロセス管理に関するデフォルト定数の定義。
  • 設定値が未定義の場合に備えた、安全なデフォルト値の提供。

重要なシンボル

  • DEFAULT_START_DAEMON: 起動時に生成される初期プロセス数。
  • DEFAULT_MAX_FREE_DAEMON: 待機状態（アイドル）として許容される最大プロセス数。これを超えるとプロセスは終了されます。
  • DEFAULT_MIN_FREE_DAEMON: 待機状態として維持すべき最小プロセス数。これ未満になると新規プロセスが生成されます。
  • DEFAULT_THREADS_PER_CHILD: 各子プロセスが生成する固定スレッド数。

制御フローとデータフロー

本ファイルは純粋な定数定義ヘッダーであるため、実行時の制御フローやデータフローは存在しません。コンパイル時に worker.c 等のソースコードへ定数として埋め込まれます。

直接依存

  • なし（標準的なプリプロセッサマクロのみで構成されています）。

関連コード

  • server/mpm/worker/worker.c: 本ファイルで定義された定数を参照し、実際のプロセス管理ロジック（スレッド生成やプロセス監視）を実装しています。

読者向けメモ

  • **設定の優先順位**: ここで定義されている値は、Apache の設定ファイルでディレクティブ（StartServers, MaxSpareServers 等）が指定されていない場合にのみ有効です。
  • **変更の注意**: これらの値を変更して再コンパイルすることでデフォルト値を変更可能ですが、運用環境では設定ファイルでの制御が推奨されます。
  • **設計意図**: Worker MPM は「プロセス数 × スレッド数」でリクエストを処理するため、これらの定数はメモリ消費量と同時接続数のバランスを決定する重要なパラメータとなります。
