server/mpm/mpmt_os2/mpmt_os2_child.c

概要

OS/2 プラットフォームにおける Apache HTTP Server のマルチプロセス・マルチスレッド MPM（MPMT_OS2）の子プロセス実行環境を定義します。本ファイルは、子プロセスのライフサイクル管理、OS/2 ネイティブのキューを用いたスレッド間通信、および接続の受け入れループを担います。

モジュール内での役割

mpmt_os2.c が親プロセスの制御を行うのに対し、本ファイルは子プロセス生成後の実行コンテキストを完全に制御します。OS/2 特有の API（Dos* 系）を駆使し、プロセス内のスレッドプール管理とリクエスト処理のディスパッチを行います。

責務

  • **プロセス初期化**: スコアボードへの接続、シグナルハンドラの登録、OS/2 キューの作成。
  • **接続管理**: apr_pollset を用いたリスニングソケットの監視と、受け入れた接続のキューイング。
  • **スレッドプール管理**: server_maintenance による動的なスレッド生成・破棄（add_worker）。
  • **例外処理**: thread_exception_handler によるワーカー内のクラッシュ捕捉とプロセス全体の健全性維持。
  • **終了処理**: グレースフルシャットダウンおよび強制終了の制御。

重要なシンボル

プロセス・スレッド管理

  • ap_mpm_child_main: 子プロセスのメインエントリポイント。
  • worker_main: 各ワーカーの実行ループ。接続を受け取り、ap_process_connection を呼び出す。
  • server_maintenance: アイドル状態を監視し、スレッド数を調整する管理スレッド。
  • add_worker: 新規ワーカーを生成し、スコアボードを更新する。

OS/2 固有の同期・通信

  • shutdown_event: 子プロセス終了時に全スレッドへ通知するためのイベントセマフォ。
  • workq: DosCreateQueue で作成される FIFO キュー。接続情報（WORKTYPE_CONN）や終了命令（WORKTYPE_EXIT）を伝達する。
  • thread_exception_handler: OS/2 の例外登録レコード。ワーカーのクラッシュ時にプロセスを安全に終了させる。

制御フローとデータフロー

1. **初期化**: ap_mpm_child_main が起動し、共有メモリからスコアボードをマップし、OS/2 キューを作成します。 2. **接続受け入れ**: メインループで apr_pollset を使用して接続を待機。接続が発生すると worker_args_t を作成し、DosWriteQueue でキューに投入します。 3. **ワーカー処理**: worker_main が DosReadQueue で接続を取り出し、ap_run_create_connection を経由してリクエストを処理します。 4. **保守**: server_maintenance が定期的にアイドルスレッド数を確認し、不足していれば add_worker を呼び出し、過剰であれば終了命令をキューに送ります。

読者向けメモ

  • **メモリ所有権**: pchild プールが子プロセスの生存期間を管理します。各接続は pconn プールで個別に管理され、処理終了後に破棄されます。
  • **例外安全性**: OS/2 の DosSetExceptionHandler を使用してワーカーのクラッシュを捕捉していますが、重大なエラー時はプロセス全体を終了させる設計です。
  • **制約**: HARD_THREAD_LIMIT（デフォルト 256）がスレッド数の上限です。OS/2 のキューはプロセスローカルな通信として使用されており、外部プロセスからは直接参照できません。
  • **省略箇所**: 本ファイルは OS/2 特有の API に強く依存しており、POSIX 互換層の詳細は省略されています。また、Dos* API の詳細なエラーコード定義は OS/2 のシステムドキュメントを参照してください。
