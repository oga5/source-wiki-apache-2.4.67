server/mpm/mpmt_os2/mpm_default.h

概要

本ファイルは、Apache HTTP Server の OS/2 用マルチプロセッシングモジュール（MPM）である mpmt_os2 における、プロセスおよびスレッド管理のデフォルト値を定義するヘッダーファイルです。

モジュール内での役割

mpmt_os2 モジュール全体で使用される設定定数のソースとして機能します。実行時の設定ファイル（httpd.conf）で明示的な指定がない場合に適用されるフォールバック値を保持しています。

責務

  • プロセス生成数、スレッドのスペア数、およびスコアボードの更新間隔のデフォルト値を定義する。
  • 各定数に対して #ifndef ガードを適用し、ビルド時や他のヘッダーによる上書きを可能にする。

重要なシンボル

  • DEFAULT_START_DAEMON: 起動時に生成される初期サーバープロセス数。
  • DEFAULT_MAX_SPARE_THREAD: 待機状態の最大スレッド数。これを超えるとスレッドが終了されます。
  • DEFAULT_MIN_SPARE_THREAD: 待機状態の最小スレッド数。これを下回ると新しいスレッドが生成されます。
  • SCOREBOARD_MAINTENANCE_INTERVAL: スコアボード（プロセス/スレッドの稼働状況を記録する共有メモリ）をメンテナンスする間隔（マイクロ秒単位）。

制御フローとデータフロー

本ファイルは純粋な定数定義のみで構成されており、実行時の制御フローは持ちません。コンパイル時にプリプロセッサによって、これらの値が mpmt_os2.c などのソースコード内に埋め込まれます。

直接依存

  • 特筆すべき外部ヘッダーへの直接依存はありません。純粋なマクロ定義ファイルです。

関連コード

  • server/mpm/mpmt_os2/mpmt_os2.c: 本ファイルで定義された定数を実際に参照し、プロセス管理ロジックを実装しているメインモジュールです。
  • include/scoreboard.h: SCOREBOARD_MAINTENANCE_INTERVAL が影響を与えるスコアボード構造の定義元です。

読者向けメモ

  • 本ファイルで定義されている値は、OS/2 環境におけるリソース消費とパフォーマンスのバランスを決定する重要なチューニングパラメータです。
  • 運用環境に合わせて調整が必要な場合は、まず httpd.conf のディレクティブを確認してください。ソースコードの変更は、それらのディレクティブが提供されていない場合の最終手段となります。
  • SCOREBOARD_MAINTENANCE_INTERVAL は、サーバーの負荷状況を監視する頻度に直結するため、変更時には CPU 使用率への影響を考慮してください。
