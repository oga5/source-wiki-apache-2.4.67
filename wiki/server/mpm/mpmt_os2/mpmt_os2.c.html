server/mpm/mpmt_os2/mpmt_os2.c

概要

OS/2 プラットフォーム向けのマルチプロセス・マルチスレッド（MPM）実装のメインエントリーポイントです。親プロセスによる子プロセスの管理と、子プロセス内でのスレッドプールによるリクエスト処理の橋渡しを行います。

モジュール内での役割

Apache HTTP Server の MPM サブシステムの一部として、OS/2 特有のプロセス間通信（共有メモリ、ミューテックス）を利用したプロセス管理層を提供します。

責務

  • **プロセス管理**: 親プロセスによる子プロセスの生成（DosExecPgm）、監視（DosWaitChild）、および終了処理。
  • **リソース共有**: 共有メモリを用いたリスナーソケットとスコアボードのプロセス間受け渡し。
  • **設定管理**: StartServers や MinSpareThreads などの MPM 固有ディレクティブの解析と検証。
  • **シグナル処理**: OS/2 環境下でのシャットダウンおよび再起動シグナルのハンドリング。

重要なシンボル

プロセス管理・状態

  • mpmt_os2_run: MPM のメインループ。親プロセスと子プロセスの分岐点。
  • master_main: 親プロセスのメイン処理。子プロセスの監視と管理を行う。
  • spawn_child: 新しい子プロセスを生成し、スコアボードを更新する。
  • ap_mpm_child_main: 子プロセスが実行するメイン処理（mpmt_os2_child.c で定義）。

データ構造

  • listen_socket_t: 子プロセスへ渡すリスナーソケットの情報を保持。
  • parent_info_t: 共有メモリに配置される構造体。リスナー配列と同期用ミューテックスを保持。

グローバル変数

  • ap_mpm_accept_mutex: 複数の子プロセスがリスナーソケットを競合しないように制御する OS/2 ミューテックス。
  • shutdown_pending / restart_pending: プロセス終了・再起動のフラグ。

制御フローとデータフロー

1. **初期化**: mpmt_os2_hooks によりフックが登録され、mpmt_os2_run が呼び出されます。 2. **分岐**: DosGetNamedSharedMem を試行し、成功すれば子プロセスとして ap_mpm_child_main へ移行、失敗すれば親プロセスとして master_main を実行します。 3. **親プロセス**: 共有メモリを確保し、リスナーソケットを格納。DosWaitChild で子プロセスの終了を監視し、必要に応じて spawn_child で補充します。 4. **子プロセス**: 親から渡された共有メモリからソケットを取得し、ap_mpm_child_main でスレッドプールを起動してリクエストを処理します。

読者向けメモ

  • **メモリ所有権**: 共有メモリ（parent_info_t）は親プロセスで確保され、子プロセスで読み取った後に DosFreeMem で解放されます。
  • **制約**: HARD_SERVER_LIMIT により最大プロセス数が 10 に制限されています。
  • **省略箇所**: ap_mpm_child_main の内部詳細（スレッドプールの動的調整ロジック）は本ファイルには含まれず、mpmt_os2_child.c に委譲されています。
  • **注意**: OS/2 特有の Dos 系 API を多用しているため、他プラットフォームへの移植性は考慮されていません。
