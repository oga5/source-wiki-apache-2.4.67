server/mpm/prefork/mpm_default.h

概要

mpm_default.h は、Apache HTTP Server の Prefork MPM（Multi-Processing Module）におけるプロセス管理のデフォルト値を定義するヘッダーファイルです。

モジュール内での役割

Prefork MPM が起動時や実行時に参照する、プロセス生成・破棄の閾値を規定します。設定ファイル（httpd.conf）で明示的な指定がない場合に適用される、システムの安全な動作基準を提供します。

責務

  • プロセス管理に関する定数の定義。
  • 各定数に対して ifndef によるガードを設け、コンパイル時や環境変数による上書きを可能にする。

重要なシンボル

  • DEFAULT_START_DAEMON: 起動時に生成される子プロセスの初期数。
  • DEFAULT_MAX_FREE_DAEMON: 待機状態（アイドル）として許容される子プロセスの最大数。これを超えると余剰プロセスが終了されます。
  • DEFAULT_MIN_FREE_DAEMON: 待機状態として維持すべき子プロセスの最小数。これより少なくなると、新規プロセスが生成されます。

制御フローとデータフロー

本ファイルは純粋な定数定義ヘッダーであるため、実行時の制御フローは持ちません。prefork.c がコンパイルされる際にこれらの値がプリプロセッサによって展開され、プロセス管理ロジックの初期値として組み込まれます。

直接依存

  • なし（標準的な C プリプロセッサの機能のみを使用）。

関連コード

  • server/mpm/prefork/prefork.c: 本ファイルで定義された定数を実際に利用し、プロセス生成の監視や調整を行うメインロジックが含まれています。

読者向けメモ

  • 本ファイルで定義されている値は、あくまで「デフォルト」です。実際の運用環境では、httpd.conf の StartServers, MaxSpareServers, MinSpareServers ディレクティブによってこれらの値は上書きされます。
  • コードを読み解く際は、これらの定数が prefork.c 内のどのプロセス監視ループで使用されているかに注目してください。
