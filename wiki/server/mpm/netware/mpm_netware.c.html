server/mpm/netware/mpm_netware.c

概要

NetWare OS 上で動作する Apache HTTP Server のマルチプロセッシングモジュール (MPM) です。NetWare のスレッドモデル（NKS: NetWare Kernel Services）を活用し、単一プロセス内で複数のワーカースレッドを管理するスレッドベースの MPM を実装しています。

モジュール内での役割

Apache の MPM インターフェースを実装し、NetWare 特有のコンソール操作、スレッド管理、およびネットワークリスニングのライフサイクルを制御します。

責務

  • **スレッド管理**: worker_main を用いたワーカースレッドの生成、監視、およびスレッドプールの動的調整（perform_idle_server_maintenance）。
  • **コンソール統合**: NetWare コンソールコマンド（RESTART, SHUTDOWN, SETTINGS 等）の登録と処理。
  • **ライフサイクル管理**: netware_run を中心としたサーバーの起動、リスニングソケットの監視、および正常終了/再起動の制御。
  • **設定管理**: StartThreads, MinSpareThreads, MaxSpareThreads, MaxThreads などの MPM 固有ディレクティブの解析と適用。

重要なシンボル

ワーカースレッド制御

  • worker_main: 各ワーカースレッドのメインループ。select() でリスニングソケットを監視し、接続を処理。
  • make_child: NXThreadCreate を用いて新しいワーカースレッドを生成。
  • clean_child_exit: スレッド終了時のクリーンアップとスコアボード更新。

サーバー状態管理

  • mpm_state: サーバーの現在の状態（AP_MPMQ_STARTING, RUNNING, STOPPING）。
  • ap_threads_limit, ap_threads_min_free: スレッドプールの制限値。
  • shutdown_pending, restart_pending: シグナルやコンソール操作による状態遷移フラグ。

コンソール統合

  • CommandLineInterpreter: NetWare コンソールからの入力を解析・実行。
  • InstallConsoleHandler: RegisterConsoleCommand を用いたコマンドハンドラの登録。

制御フローとデータフロー

1. **初期化**: netware_mpm_hooks がフックを登録。netware_rewrite_args でコマンドライン引数を処理し、コンソールハンドラをインストール。 2. **起動**: netware_run がリスニングソケットを準備し、startup_workers で初期スレッド群を生成。 3. **メインループ**: netware_run 内で perform_idle_server_maintenance を定期実行し、スレッドの過不足を調整。 4. **リクエスト処理**: 各 worker_main が select() で待機し、接続を受け取ると ap_process_connection を呼び出す。 5. **終了**: shutdown_pending がセットされると、全スレッドの終了を待機し、mpm_main_cleanup でリソースを解放。

読者向けメモ

  • **メモリ管理**: pmain プールがサーバーのライフサイクルを管理し、ptrans プールがリクエストごとのトランザクションメモリを管理します。
  • **スレッドモデル**: NetWare は単一プロセスモデルであるため、fork() は使用されず、すべてスレッドとして実装されています。
  • **制約**: HARD_SERVER_LIMIT は NetWare の特性上 1 に固定されています。
  • **未実装**: netware_post_config などの一部のフックはコメントアウトされており、必要最小限の機能に絞られています。
  • **注意**: コンソールコマンドの処理は NX_WRAP_INTERFACE を介して行われており、NetWare のスレッドコンテキストを意識した設計になっています。
