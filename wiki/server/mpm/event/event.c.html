server/mpm/event/event.c

概要

event.c は Apache HTTP Server の「Event MPM (Multi-Processing Module)」の中核を担う実装です。従来の Worker MPM が抱えていた「Keep-Alive 接続がスレッドを占有し続ける」という問題を解決するため、リスナー専用スレッドと非同期 I/O（epoll/kqueue）を活用した接続管理を行います。

モジュール内での役割

Event MPM は、接続のライフサイクルを「処理中」「書き込み完了待ち」「Keep-Alive 待ち」「遅延クローズ」の各状態に分離し、スレッドを効率的に再利用します。本ファイルは、プロセス管理、スレッドプール管理、および接続状態遷移のロジックを統合するメインエントリーポイントです。

責務

  • **プロセス・スレッド管理**: 子プロセスの生成、スレッドの起動、およびシグナルによるライフサイクル制御。
  • **接続状態管理**: event_conn_state_t を用いた接続状態の追跡と、タイムアウトキューによる期限管理。
  • **非同期 I/O 統合**: リスナーおよび待機中の接続を apr_pollset に登録し、イベント駆動で処理をディスパッチ。
  • **リソース調整**: AsyncRequestWorkerFactor に基づく動的な接続受け入れ制御と、負荷に応じたスレッドの割り当て。

重要なシンボル

接続状態とキュー

  • event_conn_state_t: 接続ごとの状態（状態遷移、タイムアウト、プール、ソケット情報）を保持する構造体。
  • timeout_queue: タイムアウト期限ごとに接続を管理するリングバッファ構造。waitio_q, write_completion_q, keepalive_q 等が存在。

スレッド・プロセス制御

  • listener_thread(): リスナーソケットの監視と、タイムアウトした接続の再キューイングを行うメインループ。
  • worker_thread(): キューから接続を取り出し、process_socket() を呼び出してリクエストを処理するスレッド。
  • event_run(): MPM のメインループ。子プロセスの生成と監視を行う。

グローバル設定・状態

  • event_pollset: 非同期 I/O を実現するための apr_pollset インスタンス。
  • worker_queue: 接続をリスナーからワーカーへ受け渡すためのスレッドセーフなキュー。

制御フローとデータフロー

1. **リスナー**: listener_thread が apr_pollset_poll でソケットの読み込みを待機。 2. **受け入れ**: 新規接続は ap_queue_push_socket を経由してワーカーへ渡される。 3. **処理**: ワーカーは process_socket でリクエストを処理。処理終了後、Keep-Alive 等の状態に応じて timeout_queue に接続を登録し、apr_pollset_add で監視を再開。 4. **タイムアウト**: リスナーは定期的に process_timeout_queue を呼び出し、期限切れの接続をクローズまたは再処理へ回す。

読者向けメモ

  • **メモリ所有権**: ptrans (トランザクションプール) は接続のライフサイクルを管理します。event_conn_state_t はこのプールから割り当てられ、接続終了時にクリーンアップされます。
  • **スレッド安全性**: timeout_mutex はキューへの追加・削除を保護します。apr_pollset は APR_POLLSET_THREADSAFE を前提としています。
  • **制約**: この MPM は epoll または kqueue をサポートするプラットフォームでのみ真価を発揮します。
  • **省略**: 本ファイルは非常に巨大であるため、各フック関数の詳細な内部ロジックや、プラットフォーム固有の最適化（HAVE_BINDPROCESSOR 等）については詳細な記述を省略しています。
