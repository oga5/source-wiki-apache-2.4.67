server/mpm/event/mpm_default.h

概要

mpm_default.h は、Apache HTTP Server の Event MPM（Multi-Processing Module）におけるプロセスおよびスレッド管理のデフォルト値を定義するヘッダーファイルです。

モジュール内での役割

Event MPM の動作基盤となる設定値の定数定義を提供します。event.c が実行時にプロセスやスレッドの生成・破棄を判断する際の基準値として利用されます。

責務

  • MPM のスケーリングアルゴリズム（プロセス数やスレッド数の増減）に使用されるデフォルト値の提供。
  • 設定ファイル（httpd.conf 等）で明示的な指定がない場合に適用されるフォールバック値の定義。

重要なシンボル

  • DEFAULT_START_DAEMON: 起動時に生成される子プロセスの初期数。
  • DEFAULT_MAX_FREE_DAEMON: 待機状態（アイドル）を許容する子プロセスの最大数。これを超えるとプロセスは終了されます。
  • DEFAULT_MIN_FREE_DAEMON: 待機状態を維持すべき子プロセスの最小数。これ未満になると新しいプロセスが生成されます。
  • DEFAULT_THREADS_PER_CHILD: 各子プロセス内で生成されるワーカースレッドの数。

制御フローとデータフロー

本ファイルは純粋な定数定義ヘッダーであるため、実行時の制御フローは持ちません。event.c 内でこれらのマクロが参照され、ap_mpm_query や設定パース処理を通じて MPM の動作パラメータとして組み込まれます。

直接依存

  • なし（標準的な C プリプロセッサの #ifndef ガードのみを使用）。

関連コード

  • server/mpm/event/event.c: 本ファイルで定義された定数を実際に利用し、プロセス管理ロジックを実装しているメインモジュール。
  • include/mpm_common.h: MPM 共通のインターフェースや構造体が定義されており、本ファイルの設定値がどのように MPM 全体で扱われるかを理解する助けとなります。

読者向けメモ

  • **設定の優先順位**: ここで定義されている値はあくまでデフォルトです。httpd.conf で StartServers や ThreadsPerChild などのディレクティブが指定されている場合、それらが優先されます。
  • **調整の指針**: パフォーマンスチューニングを行う際は、これらの定数を直接書き換えるのではなく、設定ファイルを通じて調整することが推奨されます。
  • **設計意図**: Event MPM はスレッドベースのモデルであるため、DEFAULT_THREADS_PER_CHILD はメモリ消費量と同時接続数のバランスを決定する重要なパラメータとなります。
