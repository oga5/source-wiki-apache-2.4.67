server/util_cfgtree.c

概要

Apache HTTP Server の設定ファイルをメモリ上にツリー構造として保持するためのユーティリティです。設定ファイル内のディレクティブ（Listen や <Directory> など）をノードとして連結し、階層構造を構築します。

モジュール内での役割

server モジュールにおいて、設定の解析フェーズで生成される中間表現を管理します。このツリーは、サーバー起動時に設定ファイルを読み込み、モジュールが設定値を解釈する際の基盤となります。

責務

  • 設定ディレクティブのノード（ap_directive_t）をツリー構造に組み込む。
  • 親子関係（コンテナディレクティブ）および兄弟関係（同一階層のディレクティブ）のポインタを適切に更新する。
  • 設定解析中の現在位置（親ノード）の追跡を支援する。

重要なシンボル

  • ap_directive_t: 設定ノードを表す構造体。ディレクティブ名、引数、子ノード、親ノード、次ノードへのポインタを保持します。
  • ap_add_node(): 新しいディレクティブノードをツリーに追加する主要関数。

制御フローとデータフロー

1. **ノード追加**: ap_add_node が呼び出されると、current ノードの有無を確認します。 2. **階層構築**:

  • current が NULL の場合、新しい親ノードの first_child として設定されます。
  • current が存在する場合、current->next に新しいノードを接続します。

    • 3. **親の更新**: child フラグが真の場合、parent ポインタが新しく追加されたノードへと更新され、以降のディレクティブがその子として扱われるようになります。

直接依存

  • util_cfgtree.h: ap_directive_t 構造体および ap_add_node の宣言。
  • stdlib.h: 標準ライブラリ（本ファイルでは直接的な利用は最小限）。

関連コード

  • server/config.c: このツリー構造を実際に利用して設定ファイルをパースし、ツリーを構築するメインロジックが含まれます。
  • include/http_config.h: 設定関連のデータ構造や API 定義の全体像を把握するために参照してください。

読者向けメモ

  • 本ファイルは非常に軽量ですが、Apache の設定システムにおける「ツリー構築」の根幹を担っています。
  • ap_directive_t の last ポインタは、ツリーの末尾へのアクセスを高速化するためのショートカットですが、常に最新であるとは限らない点に注意してください。
  • コンテナディレクティブ（<Directory> 等）の終了タグ（</Directory>）は、このツリーには格納されません。ツリーの構造自体が階層を表現しているためです。
