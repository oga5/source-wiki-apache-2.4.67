server/core.c

概要

Apache HTTP Server の「コア」モジュールであり、サーバーの基本動作、設定の解析、およびリクエスト処理の基盤を提供します。他のモジュールが依存する設定構造の定義や、HTTP プロトコルの基本ハンドラ、ディレクトリ/ファイル単位のアクセス制御、フィルタの登録などを担う、サーバーの「官僚機構」とも呼べる中心的なファイルです。

モジュール内での役割

server モジュール群の心臓部です。他のモジュール（mod_proxy, mod_so など）が利用する設定の保存場所や、リクエスト処理の最終的なフォールバック（default_handler）を提供します。また、設定ファイルのディレクティブ（<Directory>, <VirtualHost> など）を解釈し、サーバーの実行時状態を構築します。

責務

  • **設定管理**: ディレクトリおよびサーバー単位の設定構造（core_dir_config, core_server_config）の作成、マージ、および保持。
  • **ディレクティブ処理**: 設定ファイル内の主要なディレクティブ（DocumentRoot, Options, AllowOverride 等）の解析と適用。
  • **リクエスト処理**: リクエストの翻訳（ap_core_translate）、ストレージへのマッピング、およびデフォルトのコンテンツ配信ハンドラ。
  • **フィルタ管理**: 入出力フィルタ（CORE_IN, CORE）の登録と管理。
  • **環境抽象化**: OS 固有の制限（RLIMIT）やパス操作の抽象化。

重要なシンボル

設定構造体とライフサイクル

  • create_core_dir_config / merge_core_dir_configs: ディレクトリ単位の設定の初期化とマージ。
  • create_core_server_config / merge_core_server_configs: サーバー（仮想ホスト含む）単位の設定の初期化とマージ。
  • core_cmds: サーバー全体で利用可能な全ディレクティブの定義テーブル。

リクエスト処理ハンドラ

  • ap_core_translate: URI をファイルシステムパスに変換する。
  • default_handler: 最終的なコンテンツ配信（ファイル読み込みと送信）。
  • core_map_to_storage: ディレクトリおよびファイルのウォーク処理。

フィルタ・フック

  • ap_core_input_filter_handle / ap_core_output_filter_handle: コアネットワークフィルタのハンドル。
  • register_hooks: サーバー起動時に呼び出され、フックを登録する。

制御フローとデータフロー

1. **設定フェーズ**: core_cmds に定義された関数が設定ファイルを読み込み、apr_pool_t を使用して設定構造体を構築します。merge_core_dir_configs 等により、親ディレクトリから子ディレクトリへの設定継承が行われます。 2. **リクエストフェーズ**:

  • ap_core_translate が URI を r->filename に変換します。
  • core_map_to_storage が ap_directory_walk を呼び出し、該当パスの設定を適用します。
  • default_handler がファイルを開き、apr_bucket_brigade を使用してネットワークフィルタへデータを渡します。

    • 3. **エラー処理**: ap_custom_response や ErrorDocument ディレクティブにより、エラー発生時の挙動をカスタマイズ可能です。

読者向けメモ

  • **メモリ管理**: すべての設定構造体は apr_pool_t に紐付いており、サーバーの再起動や設定の再読み込み時に適切に解放される必要があります。
  • **不変条件**: core_dir_config 内の opts_add と opts_remove は、常に (opts_add & opts_remove) == 0 を維持する必要があります。
  • **注意点**: このファイルは非常に巨大です。特定のディレクティブの挙動を追う場合は、core_cmds テーブルから該当関数を探すのが最も効率的です。
  • **省略範囲**: 複雑な正規表現の評価や、OS 固有の細かい最適化（sendfile 等）については、詳細な実装ロジックを省略しています。これらは mod_core ではなく、この core.c が直接所有する責任範囲です。
