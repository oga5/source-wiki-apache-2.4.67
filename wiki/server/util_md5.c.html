server/util_md5.c

概要

Apache HTTP Server における MD5 ハッシュ計算およびエンコーディング機能を提供するユーティリティモジュールです。主にデータの整合性チェックやダイジェスト生成に使用されます。

モジュール内での役割

server モジュール群の一部として、HTTP プロトコルや設定処理で必要となる MD5 計算の抽象化レイヤーを提供します。APR (Apache Portable Runtime) の MD5 実装をラップし、HTTPD 固有のメモリ管理（プール）や文字コード変換（EBCDIC 対応）を統合しています。

責務

  • 文字列およびバイナリデータからの MD5 ハッシュ生成。
  • ファイル内容に基づいた MD5 ダイジェストの計算。
  • MD5 コンテキストから Base64 形式への変換。
  • EBCDIC 環境下での適切な文字コード変換の適用。

重要なシンボル

  • ap_md5(): 文字列から MD5 ハッシュ（16進数文字列）を生成する簡易関数。
  • ap_md5_binary(): バイナリデータから MD5 ハッシュを生成するコア関数。
  • ap_md5contextTo64(): MD5 コンテキストを Base64 形式の文字列に変換する。
  • ap_md5digest(): 指定されたファイルの内容を読み込み、MD5 ダイジェストを生成する。

制御フローとデータフロー

1. **初期化**: apr_md5_init() を呼び出し、コンテキストを準備します。 2. **更新**: apr_md5_update() を使用して、データ（メモリバッファまたはファイル）を順次コンテキストに読み込ませます。 3. **終了**: apr_md5_final() でハッシュ値を確定させます。 4. **変換/出力**:

  • ap_md5_binary は ap_bin2hex を経由して 16 進数文字列を返します。
  • ap_md5contextTo64 は basis_64 テーブルを用いて、手動でビットシフトを行い Base64 エンコードを行います。

    • 5. **メモリ管理**: すべての関数は apr_pool_t を引数に取り、結果の文字列をプールから割り当てます。

関連コード

  • server/util_ebcdic.c: EBCDIC 変換の詳細実装。

読者向けメモ

  • 本ファイルは歴史的な経緯から、NCSA HTTPd や Carnegie Mellon University 等の古いコードベースを継承しています。
  • ap_md5contextTo64 は標準的な Base64 ライブラリを使用せず、独自の実装を行っています。これは特定のレガシーな互換性を維持するためのものです。
  • ファイル読み込みを行う ap_md5digest は、処理後に apr_file_seek を使用してファイルポインタを先頭に戻すため、呼び出し側はファイルの状態が維持されることを期待できます。
