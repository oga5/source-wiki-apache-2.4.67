server/provider.c

概要

Apache HTTP Server における「プロバイダー」メカニズムの中核を担うソースファイルです。モジュール間で機能（プロバイダー）を動的に登録・検索するための汎用的なレジストリ機能を提供します。

モジュール内での役割

server モジュールの一部として、特定の機能実装を分離し、疎結合なプラグインアーキテクチャを実現するための基盤を提供します。認証、認可、フィルタなど、複数の実装が競合または共存する機能の管理に利用されます。

責務

  • **プロバイダーの登録**: group, name, version をキーとして、任意の関数ポインタやデータ構造をグローバルなハッシュテーブルに格納します。
  • **プロバイダーの検索**: 指定されたキーに基づき、登録済みのプロバイダーを高速に取得します。
  • **メタデータの管理**: 登録されているプロバイダーのグループ名やバージョンの一覧を動的に提供します。
  • **リソース管理**: プロセス終了時にハッシュテーブルをクリーンアップするためのフックを管理します。

重要なシンボル

  • global_providers: プロバイダーの実体（関数ポインタ等）を保持する階層型ハッシュテーブル。
  • global_providers_names: プロバイダーのメタデータ（グループ名やバージョン）を逆引きするためのハッシュテーブル。
  • ap_register_provider: プロバイダーを登録する公開 API。
  • ap_lookup_provider: 特定のプロバイダーを取得する公開 API。

制御フローとデータフロー

1. **初期化**: 初めて ap_register_provider が呼ばれた際、apr_pool_cleanup_register を使用してグローバルなハッシュテーブルの初期化とクリーンアップ設定が行われます。 2. **登録**: 3階層のハッシュ（グループ → 名前 → バージョン）にプロバイダーを格納します。同時に、検索効率化のために global_providers_names にも情報を複製します。 3. **検索**: ap_lookup_provider は、提供されたキーをハッシュテーブルで辿り、該当するポインタを返します。 4. **一覧取得**: ap_list_provider_names や ap_list_provider_groups は、ハッシュテーブルをイテレートし、結果を apr_array_header_t に詰め替えて返却します。

直接依存

  • apr_pools.h: メモリ管理（プール）用。
  • apr_hash.h: プロバイダー管理用ハッシュテーブル。
  • apr_tables.h: 一覧取得時の配列管理。
  • apr_strings.h: 文字列操作。
  • ap_provider.h: プロバイダー関連の型定義とインターフェース。

関連コード

  • include/ap_provider.h: このファイルで実装されている関数のプロトタイプ宣言と、プロバイダーのデータ構造定義が含まれています。
  • 各モジュール（modules/ 配下）: 実際に ap_register_provider を呼び出して機能を登録する側です。

読者向けメモ

  • **スレッドセーフティ**: この実装は Apache の起動時（設定読み込みフェーズ）に登録が行われることを前提としています。実行時の動的な登録・削除は想定されていないため、マルチスレッド環境での実行時変更には注意が必要です。
  • **ハッシュ構造**: 3階層のハッシュ構造になっているため、検索時は group → name → version の順でキーを特定する必要があります。
  • **メモリ管理**: 登録に使用するプールは、通常はサーバーのライフサイクルに紐付いたものが渡されます。
