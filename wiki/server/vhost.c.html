server/vhost.c

概要

HTTP サーバーにおける仮想ホスト（Virtual Host）の構成管理および実行時のルーティングを担当するモジュールです。IP ベースおよび名前ベース（NameVirtualHost）の仮想ホストを効率的に解決するためのハッシュテーブル管理と、リクエストヘッダーに基づいたサーバーの動的切り替え機能を提供します。

モジュール内での役割

server モジュール群の一部として、http_vhost.h で定義されたインターフェースを実装します。接続確立時（IP ベース）とリクエストヘッダー受信時（名前ベース）の二段階で request_rec や conn_rec のサーバー情報を更新し、適切な server_rec を割り当てる役割を担います。

責務

  • **構成管理**: 設定ファイル解析時に仮想ホストの IP/ポート情報をハッシュテーブル（iphash_table）に構築する。
  • **実行時ルーティング**: 接続のローカルアドレスに基づき、候補となる仮想ホストリストを特定する。
  • **リクエスト解決**: Host ヘッダーやリクエスト URI を検証・正規化し、適切な仮想ホストへリクエストを紐付ける。
  • **統計・デバッグ**: 仮想ホスト構成のダンプおよびハッシュテーブルの統計情報の出力。

重要なシンボル

データ構造

  • ipaddr_chain: IP アドレスとポートのペアをキーとするハッシュテーブルのノード。名前ベースの仮想ホストリスト（names）を保持する。
  • name_chain: 特定の IP/ポートに紐付く仮想ホストのリスト。

設定・初期化

  • ap_init_vhost_config: 仮想ホスト設定の初期化。
  • ap_parse_vhost_addrs: <VirtualHost> ディレクティブの引数を解析し、アドレスリストを構築する。
  • ap_fini_vhost_config: 設定完了時にハッシュテーブルを構築し、デフォルトサーバーを確定させる。

実行時マッチング

  • ap_update_vhost_given_ip: 接続確立時に IP/ポートから conn_rec に候補リストをセットする。
  • ap_update_vhost_from_headers: リクエストヘッダー受信後に Host ヘッダー等を解析し、最終的な server_rec を決定する。
  • ap_matches_request_vhost: 特定のホスト名とポートが仮想ホストに適合するかを判定する。

制御フローとデータフロー

1. **初期化**: ap_fini_vhost_config が設定終了時に呼ばれ、iphash_table を構築します。IP アドレスは hash_addr でハッシュ化され、衝突時はリスト構造で管理されます。 2. **接続時**: ap_update_vhost_given_ip が接続の local_addr を使い、ハッシュテーブルから ipaddr_chain を検索。見つかった names（name_chain）を conn->vhost_lookup_data に格納します。 3. **リクエスト時**: ap_update_vhost_from_headers が呼ばれます。

  • fix_hostname で Host ヘッダーを正規化。
  • update_server_from_aliases が vhost_lookup_data を走査し、ServerName や ServerAlias と照合して r->server を更新します。

読者向けメモ

  • **メモリ管理**: すべてのデータ構造は apr_pool_t を介して割り当てられます。設定時のメモリは process_pool に依存し、実行時の正規化されたホスト名は request_pool に依存します。
  • **スレッド安全性**: iphash_table は設定完了後に構築され、実行時は読み取り専用となるため、マルチスレッド環境でも安全です。
  • **制限事項**: NameVirtualHost ディレクティブは現在非推奨であり、設定時に警告が出力されます。
  • **省略箇所**: IPHASH_STATISTICS が有効な場合の統計ダンプ機能は、デバッグ目的の補助的な実装であるため、詳細なアルゴリズム解説は省略しています。
