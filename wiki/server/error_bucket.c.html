server/error_bucket.c

概要

Apache HTTP Server における「エラーバケット」の定義と実装を行うソースファイルです。このバケットは、レスポンス処理中に発生したエラー情報をフィルタチェーンを通じて下流へ伝播させるために使用されます。

モジュール内での役割

server モジュールの一部として、HTTP プロトコル処理におけるメタデータバケットの一種を提供します。通常のデータバケットとは異なり、コンテンツ本体ではなく、HTTP ステータスコードやエラーメッセージを保持する特殊なバケット型を定義します。

責務

  • **エラーバケットの生成**: ap_bucket_error_create を通じて、エラー情報（ステータスコードとメッセージ）を保持するバケットを初期化する。
  • **バケット型の定義**: ap_bucket_type_error を定義し、バケットのライフサイクル管理（破棄、読み込み等）の振る舞いを規定する。
  • **メモリ管理**: バケットに関連付けられたエラーデータのメモリ確保と解放を管理する。

重要なシンボル

  • ap_bucket_type_error: エラーバケットの型定義。バケットの振る舞い（メソッド）を保持する構造体。
  • ap_bucket_error_create: エラーバケットを新規作成する公開関数。
  • ap_bucket_error_make: 既存のバケット構造体をエラーバケットとして初期化する関数。

制御フローとデータフロー

1. **作成**: ap_bucket_error_create が呼び出されると、apr_bucket 構造体が確保され、ap_bucket_error_make へ渡されます。 2. **初期化**: ap_bucket_error_make は、エラーコードとメッセージを ap_bucket_error 構造体に格納し、バケットの型を ap_bucket_type_error に設定します。 3. **読み込み**: error_bucket_read が呼び出されると、データは存在しない（メタデータである）ため、即座に APR_SUCCESS を返し、長さ 0 を通知します。 4. **破棄**: error_bucket_destroy が呼び出されると、共有参照カウントを確認し、ゼロであればメモリを解放します。

直接依存

  • http_protocol.h: HTTP プロトコル関連の定義（ap_is_HTTP_VALID_RESPONSE 等）。
  • apr_buckets.h: APR バケットフレームワークの基盤。
  • apr_strings.h: 文字列操作ユーティリティ。
  • strings.h

関連コード

  • server/core_filters.c: エラーバケットを実際に処理・解釈するフィルタの実装が含まれる可能性があります。
  • server/eoc_bucket.c / server/eor_bucket.c: 他の特殊なメタデータバケットの実装例。

読者向けメモ

  • 本バケットは「メタデータ」として定義されており、error_bucket_read は意図的に空のデータを返すよう設計されています。
  • エラーコードが不正な場合は、自動的に HTTP_INTERNAL_SERVER_ERROR にフォールバックする安全策が講じられています。
  • バケットのコピーや分割はサポートされておらず、apr_bucket_split_notimpl 等が割り当てられています。
