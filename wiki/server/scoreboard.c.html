server/scoreboard.c

概要

Apache HTTP Server の実行時状態を管理するスコアボード機能の実装です。マルチプロセス/マルチスレッド環境において、各ワーカーの現在の状態（接続数、処理中のリクエスト、稼働時間など）を共有メモリまたはプロセス内メモリに保持し、監視や管理を可能にします。

モジュール内での役割

server モジュールの基盤コンポーネントであり、MPM（Multi-Processing Module）と密接に連携します。MPM が子プロセスやスレッドを生成・管理する際、そのライフサイクルと状態を記録するための共有データ構造を提供します。

責務

  • **スコアボードの初期化とメモリ管理**: 共有メモリ（SHM）またはヒープメモリの確保、および構造体のレイアウト構築。
  • **状態更新のディスパッチ**: ワーカーのステータス変化（SERVER_READY から SERVER_BUSY_READ 等）を記録するインターフェースの提供。
  • **統計情報の集計**: リクエスト数、転送バイト数、処理時間などのメトリクス更新。
  • **設定管理**: ExtendedStatus などの設定値に基づいた詳細な状態追跡の制御。

重要なシンボル

グローバル状態

  • ap_scoreboard_image: スコアボード全体を指すポインタ。global（全体）、parent（プロセス）、servers（ワーカー）の各構造体へのアクセス起点。
  • ap_scoreboard_fname: 共有メモリファイル名。
  • ap_extended_status: 詳細な統計情報を記録するかどうかのフラグ。

初期化・ライフサイクル

  • ap_init_scoreboard(): 確保されたメモリ領域を scoreboard 構造体としてマッピングする。
  • ap_create_scoreboard(): スコアボードの作成と初期化。MPM から呼び出される。
  • ap_cleanup_scoreboard(): プロセス終了時のメモリ解放。

状態更新（Mutators）

  • ap_update_child_status_...() ファミリー: ワーカーの状態を更新する一連の関数。request_rec や conn_rec から情報を抽出し、スコアボードに書き込む。
  • ap_increment_counts(): リクエスト処理完了時の統計（アクセス数、バイト数）の加算。
  • ap_time_process_request(): リクエスト処理時間の計測。

アクセサ

  • ap_get_scoreboard_worker() / ap_get_scoreboard_process(): 特定のワーカーやプロセスのスコア構造体を取得する。

制御フローとデータフロー

1. **初期化**: MPM が ap_create_scoreboard を呼び出し、ap_calc_scoreboard_size で必要なサイズを計算。共有メモリが確保されると、ap_init_scoreboard がポインタ演算を用いて ap_scoreboard_image をメモリ上に構築します。 2. **更新**: リクエスト処理の各フェーズで ap_update_child_status が呼ばれます。内部の update_child_status_internal が、ap_extended_status が有効な場合に限り、リクエスト文字列のコピーやクライアント IP の記録を行います。 3. **データフロー**: ap_sb_handle_t（ハンドル）がワーカーを特定し、共有メモリ上の該当オフセットにある worker_score 構造体を直接更新します。ロックは不要と設計されています（各スロットは単一の所有者によってのみ書き込まれるため）。

読者向けメモ

  • **メモリレイアウト**: ap_init_scoreboard は、単一のメモリブロックを global、parent 配列、servers（ワーカーの2次元配列）の順に分割して配置します。このため、メモリ確保後の再配置は不可能です。
  • **ロックの不在**: 本実装は「1スロットにつき1書き込み者」という不変条件に依存しています。このため、排他制御なしで高速な更新が可能です。
  • **制限事項**: ap_cleanup_shared_mem の TODO にある通り、IPC 手法が共有メモリに限定されている点は将来的な拡張の余地があります。
  • **詳細な統計**: ap_extended_status が有効な場合、リクエスト文字列のコピーなどが発生するため、高負荷時にはパフォーマンスへの影響を考慮する必要があります。
