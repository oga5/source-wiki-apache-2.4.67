server/util_pcre.c

概要

本ファイルは、Apache HTTP Server における正規表現処理の抽象化レイヤーです。PCRE (Perl Compatible Regular Expressions) ライブラリ（PCRE1 または PCRE2）をラップし、POSIX 互換のインターフェースを提供します。HTTPD 内部で正規表現をコンパイル、実行、管理するための標準的な API を定義しています。

モジュール内での役割

server モジュール内の正規表現ユーティリティとして、設定解析やリクエスト処理におけるパターンマッチングの基盤を提供します。PCRE のバージョン差異（PCRE1/PCRE2）を吸収し、HTTPD の他のコンポーネントがライブラリの仕様変更を意識せずに正規表現を利用できるようにします。

責務

  • **正規表現のライフサイクル管理**: コンパイル (ap_regcomp) から破棄 (ap_regfree) までの管理。
  • **POSIX 互換ラッパー**: PCRE 特有の API を HTTPD の ap_regex_t 構造体へマッピング。
  • **スレッドセーフなマッチング**: スレッドローカルストレージ (TLS) を活用したマッチング用バッファの再利用によるメモリ効率化。
  • **エラーハンドリング**: PCRE のエラーコードを HTTPD 共通のエラー定義へ変換。

重要なシンボル

コンパイル・管理

  • ap_regcomp: 正規表現をコンパイルし、ap_regex_t を初期化する。
  • ap_regfree: コンパイル済みの正規表現リソースを解放する。
  • ap_regcomp_set_default_cflags: グローバルなデフォルトコンパイルフラグを設定する。

マッチング

  • ap_regexec / ap_regexec_len: 文字列に対して正規表現を適用する。_len 版はバイナリセーフなマッチングを行う。
  • get_match_data: マッチングに必要なワークスペース（PCRE2 の match_data や PCRE1 の ovector）を TLS から取得または確保する。

ユーティリティ

  • ap_pcre_version_string: 実行時にリンクされている PCRE ライブラリのバージョン情報を取得する。
  • ap_regerror: エラーコードを人間が読める文字列に変換する。

制御フローとデータフロー

1. **コンパイル**: ap_regcomp が呼び出されると、フラグを PCRE 形式に変換し、pcre_compile2 (PCRE1) または pcre2_compile (PCRE2) を実行。結果を ap_regex_t に格納する。 2. **マッチング**: ap_regexec_len が呼び出される際、get_match_data が TLS を確認。

  • TLS に十分なサイズのバッファがあれば再利用し、なければ確保する。
  • PCRE2 の場合は pcre2_match、PCRE1 の場合は pcre_exec を実行。

    • 3. **結果変換**: PCRE のマッチ結果（オフセット配列）を pmatch 構造体にコピーし、呼び出し元へ返す。 4. **解放**: to_free フラグが立っている場合（TLS を使用しない場合など）、free_match_data でメモリを解放する。

  • apr_*.h: APR (Apache Portable Runtime) のメモリプール、スレッド、文字列操作関数。
  • pcre.h / pcre2.h: 実際の正規表現エンジン。
  • apr_strings.h

読者向けメモ

  • **メモリ所有権**: ap_regcomp で生成された ap_regex_t は、ap_regfree を呼び出すまでメモリを保持します。
  • **TLS の最適化**: AP_HAS_THREAD_LOCAL が有効な場合、マッチング用バッファはスレッドごとにキャッシュされます。これにより、高負荷時の malloc/free のオーバーヘッドを大幅に削減しています。
  • **PCRE2 への対応**: #ifdef HAVE_PCRE2 により、PCRE1 と PCRE2 の両方をサポートしています。PCRE2 では match_data オブジェクトが必須となるため、PCRE1 よりもメモリ管理が厳密になっています。
  • **制限事項**: 本ファイルは POSIX 互換のラッパーであるため、PCRE の高度な機能（再帰的パターンや複雑なコールアウトなど）を直接利用したい場合は、PCRE のネイティブ API を直接使用する必要があります。
