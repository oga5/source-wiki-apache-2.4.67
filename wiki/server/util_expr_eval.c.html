server/util_expr_eval.c

概要

Apache HTTP Server の式評価エンジン（ap_expr）の実行時評価ロジックを担うソースファイルです。構文解析された抽象構文木（AST）を走査し、リクエストコンテキストに基づいて文字列や真偽値を算出します。

モジュール内での役割

server モジュールの一部として、設定ファイルやリクエスト処理中に動的に評価される条件式（例: <If> ディレクティブ、RewriteCond 等）のバックエンドを提供します。util_expr_parse.c が生成した AST を受け取り、実行時の値を解決します。

責務

  • **AST 評価**: 演算子（論理、比較、正規表現）およびオペランド（変数、関数呼び出し）の再帰的評価。
  • **コンテキスト管理**: ap_expr_eval_ctx_t を通じたリクエスト、接続、サーバー情報へのアクセスと、評価結果のキャッシュ。
  • **プロバイダー・ディスパッチ**: 変数や関数名から実際の C 関数へのマッピングと実行。
  • **セキュリティ制限**: ファイルシステム操作など、制限されたコンテキストでの関数実行の制御。

重要なシンボル

評価エンジン（コア）

  • ap_expr_exec_ctx: 式評価のメインエントリーポイント。コンテキストを初期化し、結果を算出する。
  • ap_expr_eval: AST を再帰的に走査し、論理演算（AND/OR/NOT）を解決する。
  • ap_expr_eval_word: 文字列や変数、関数呼び出しの結果を解決する。

プロバイダー・ディスパッチ

  • core_expr_lookup: 登録された変数や関数を検索するフック関数。
  • var_providers / string_func_providers: 利用可能な変数や関数の定義テーブル。

ユーティリティ

  • inc_rec: 再帰呼び出しの深さを監視し、スタックオーバーフローを防止する。
  • strcmplex: SSL 互換性のための文字列比較関数。

制御フローとデータフロー

1. **初期化**: ap_expr_exec_ctx が呼び出され、ap_expr_eval_ctx_t にリクエスト情報がセットされる。 2. **再帰的評価**: ap_expr_eval が AST のルートノードから開始。論理演算子は inc_rec で深さをチェックしつつ、子ノードを評価。 3. **値の解決**: オペランド（op_Var, op_StringFuncCall 等）に到達すると、対応する関数（request_var_fn 等）が呼び出され、request_rec から値を取得。 4. **結果の返却**: 最終的な真偽値または文字列が呼び出し元に返される。エラー発生時は ctx->err にメッセージが格納される。

読者向けメモ

  • **メモリ管理**: 評価中に生成される文字列は、すべて ctx->p（リクエストプール）から割り当てられます。
  • **再帰制限**: AP_EXPR_MAX_RECURSION（20）により、複雑すぎる式は評価を中断します。
  • **SSL 互換性**: ssl_expr_eval_comp など、歴史的な mod_ssl の挙動を維持するためのコードが含まれています。
  • **省略された詳細**: 正規表現のコンパイル処理は util_pcre.c に委譲されており、本ファイルは実行（ap_regexec）のみを扱います。
  • **注意点**: req_table_func 等の関数は、リクエストヘッダーの Vary ヘッダーを自動的に更新する副作用を持つため、評価タイミングに注意が必要です。
