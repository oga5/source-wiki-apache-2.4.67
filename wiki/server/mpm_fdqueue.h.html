server/mpm_fdqueue.h

概要

server/mpm_fdqueue.h は、Apache HTTP Server のマルチプロセッシングモジュール（MPM）、特に event や worker において、スレッド間でソケットやタイマーイベントを安全に受け渡すためのキュー構造を定義しています。

モジュール内での役割

MPM におけるリスナースレッドとワーカースレッド間の通信を仲介するデータ構造を提供します。リスナーが受け取った接続（ソケット）をワーカースレッドに引き継ぎ、効率的に処理を分散させるための基盤です。

責務

  • **スレッドセーフなキュー管理**: ソケットとタイマーイベントのプッシュ・ポップ操作の同期。
  • **ワーカースレッドのアイドル管理**: fd_queue_info_t を通じたアイドルスレッドの追跡と待機制御。
  • **メモリ管理の最適化**: ワーカースレッド間で再利用可能な apr_pool_t のプール管理。

重要なシンボル

  • fd_queue_t: ソケットとタイマーイベントを保持するメインのキュー構造体。
  • fd_queue_info_t: アイドルスレッド数や再利用プールを管理する制御構造体。
  • timer_event_t: タイムアウト処理のためのイベント構造体。
  • ap_queue_push_socket: キューに新しいソケットを追加する主要関数。
  • ap_queue_pop_something: キューからソケットまたはタイマーイベントを取り出す汎用関数。

制御フローとデータフロー

1. **リスナースレッド**: ap_queue_push_socket を呼び出し、接続済みソケットをキューに投入します。 2. **ワーカースレッド**: ap_queue_pop_something を呼び出し、キューが空の場合は条件変数（not_empty）で待機します。 3. **アイドル管理**: ワーカースレッドは ap_queue_info_wait_for_idler を通じて、処理可能な状態になるまで待機し、完了後に ap_queue_info_set_idle でプールを再利用リストへ戻します。

直接依存

  • apr.h: APR（Apache Portable Runtime）の基本定義。
  • apr_ring.h: キュー内の要素管理に使用されるリング構造。
  • apr_thread_mutex.h / apr_thread_cond.h: スレッド間の同期と排他制御。
  • apr_network_io.h: ソケット操作の型定義。
  • ap_mpm.h
  • apr_pools.h

関連コード

  • server/mpm_fdqueue.c: 本ヘッダーで宣言された関数の実装。
  • server/mpm_common.c: MPM 共通のユーティリティ関数群。
  • server/event.c (MPM): 本キューを実際に利用する主要な MPM 実装。

読者向けメモ

  • 本ファイルは APR_HAS_THREADS が定義されている環境でのみ有効です。
  • ap_queue_pop_socket は ap_queue_pop_something をラップしたマクロであり、ソケット取得専用の簡便なインターフェースとして提供されています。
  • キューの操作は非常に頻繁に行われるため、ロックの競合を最小限にする設計になっています。デバッグ時には one_big_mutex の競合状況に注意してください。
