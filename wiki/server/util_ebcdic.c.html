server/util_ebcdic.c

概要

EBCDIC（拡張二進化十進コード）環境下で動作するApache HTTP Serverにおいて、プロトコルデータとASCII間の文字コード変換を管理するユーティリティです。

モジュール内での役割

server モジュールの一部として、プラットフォーム固有の文字コード差異を吸収します。非EBCDIC環境ではコンパイル時に除外されるか、空の処理（NOOP）に置換されるため、移植性を維持しつつ特定のメインフレーム環境等での動作を保証します。

責務

  • **変換ハンドラの初期化**: ap_init_ebcdic を通じて、変換に必要な apr_xlate ハンドルを生成し、MD5やSHA1等のライブラリにEBCDIC変換設定を適用する。
  • **バッファ変換**: プロトコルレベルのデータ（ヘッダー等）を、実装環境の文字セットとASCII間で相互変換する。
  • **ストリーム出力の変換**: ASCII形式で送信すべきデータを、変換後にストリームへ書き出す。

重要なシンボル

  • ap_init_ebcdic: 起動時に呼び出され、変換テーブルと関連ライブラリを初期化する。
  • ap_xlate_proto_to_ascii: バッファ内のデータをローカル文字セットからASCIIへ変換する。
  • ap_xlate_proto_from_ascii: バッファ内のデータをASCIIからローカル文字セットへ変換する。
  • ap_rvputs_proto_in_ascii: 可変長引数を受け取り、変換後に ap_rputs で出力するラッパー。

制御フローとデータフロー

1. **初期化**: ap_init_ebcdic が apr_xlate_open を呼び出し、変換ハンドルをグローバル変数に保持します。同時に、MD5/SHA1/Base64の各処理系に対して変換ハンドルを登録します。 2. **変換処理**: ap_xlate_proto_to_ascii 等は、apr_xlate_conv_buffer を利用し、インプレース（同一バッファ内）で文字コード変換を行います。 3. **出力フロー**: ap_rvputs_proto_in_ascii は、引数の文字列をプールから確保したメモリにコピーし、変換を適用してから ap_rputs へ渡すことで、ネットワーク送信データが常にASCIIであることを保証します。

直接依存

  • apr_xlate: 文字コード変換の基盤。
  • httpd.h: request_rec 等のコア構造体定義。
  • util_ebcdic.h: 公開インターフェース定義。
  • http_log.h: エラーログ出力用。
  • ap_config.h
  • apr_strings.h
  • http_core.h

関連コード

  • include/util_charset.h: 文字セット定義に関連するヘッダー。
  • server/util.c: ap_rputs 等の出力系関数の実装元。
  • apr (Apache Portable Runtime): apr_xlate や apr_MD5InitEBCDIC などの低レイヤーAPIを提供。

読者向けメモ

  • 本ファイルは APR_CHARSET_EBCDIC が定義されている環境でのみ有効です。
  • 非EBCDIC環境では、ヘッダーファイル util_ebcdic.h 内で関数が空のマクロに置換されるため、呼び出し側で条件分岐を意識する必要はありません。
  • ap_hdrs_to_ascii 等のグローバル変数は、このファイル内で静的に管理されており、初期化順序には注意が必要です。
