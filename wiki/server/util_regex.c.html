server/util_regex.c

概要

Perl互換の正規表現（PCRE）をラップし、Apache HTTP Server内でより高レベルな操作（コンパイル、置換、マッチング）を簡素化するためのユーティリティ群を提供します。

モジュール内での役割

server モジュールの一部として、設定解析やリクエスト処理において正規表現を扱うための抽象化レイヤーを提供します。低レイヤーの ap_regex.h（PCREラッパー）を直接操作する負担を軽減します。

責務

  • **正規表現のコンパイル**: Perl形式の構文（s/rx/subs/ や /rx/）を解析し、実行可能な ap_rxplus_t 構造体を生成する。
  • **置換処理**: 正規表現マッチングに基づいた文字列置換を実行する。
  • **メモリ管理**: apr_pool_t を利用し、正規表現オブジェクトのライフサイクルを管理する。
  • **マッチング情報の抽出**: マッチした部分文字列の取得や、キャプチャグループの管理を行う。

重要なシンボル

  • ap_rxplus_t: 正規表現、置換文字列、フラグ、マッチング結果を保持するメイン構造体。
  • ap_rxplus_compile(): パターン文字列から ap_rxplus_t を生成するコンストラクタ。
  • ap_rxplus_exec(): コンパイル済みオブジェクトを用いてマッチングおよび置換を実行する。
  • ap_rxplus_pmatch(): 特定のキャプチャグループの内容をプールから割り当てられた文字列として取得する。

制御フローとデータフロー

1. **コンパイル**: ap_rxplus_compile が入力文字列を解析し、区切り文字やフラグ（i, m, g 等）を抽出。ap_regcomp を呼び出し、結果をプールに登録されたクリーンアップ関数と共に格納する。 2. **実行**: ap_rxplus_exec が ap_regexec を呼び出す。置換（s///）が指定されている場合、ap_pregsub を用いて置換文字列を生成し、必要に応じて再帰的にマッチングを継続（g フラグ）して文字列を構築する。 3. **取得**: ap_rxplus_match や ap_rxplus_pmatch を通じて、呼び出し元がマッチした部分文字列にアクセスする。

直接依存

  • apr.h, apr_pools.h, apr_strings.h: APRのメモリ管理および文字列操作。
  • ap_regex.h: Apache標準の正規表現API。
  • httpd.h: Apacheのコア定義。
  • ap_config.h
  • apr_lib.h

関連コード

  • server/util_pcre.c: PCREライブラリとのより直接的な統合。
  • include/ap_regex.h: 本ファイルがラップしている低レイヤーAPIの定義。

読者向けメモ

  • **Perl形式のサポート**: 現状のパーサーは単純化されており、複雑なエスケープシーケンスや入れ子構造には制限があります。
  • **メモリの安全性**: rx->match は ap_rxplus_exec に渡された元の文字列へのポインタです。exec を呼び出すたびに内容が更新されるため、マッチ結果を保持したい場合は ap_rxplus_pmatch を使用してコピーを作成してください。
