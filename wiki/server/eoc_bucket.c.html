server/eoc_bucket.c

概要

Apache HTTP Server における「EOC (End of Connection)」バケットの定義と生成ロジックを実装するソースファイルです。

モジュール内での役割

server モジュールの一部として、接続の終了を通知するためのメタデータバケットを提供します。フィルタチェーンを通じて、接続が完全に終了したことを後続のフィルタに伝えるための特殊なマーカーとして機能します。

責務

  • EOC バケット型の定義 (ap_bucket_type_eoc)。
  • バケットの動的生成 (ap_bucket_eoc_create) および初期化 (ap_bucket_eoc_make)。
  • 読み取り操作 (eoc_bucket_read) の実装（メタデータバケットのため、データは保持せず空を返す）。

重要なシンボル

  • ap_bucket_type_eoc: EOC バケットの挙動を規定する構造体。APR_BUCKET_METADATA として定義されています。
  • ap_bucket_eoc_create: 新しい EOC バケットを割り当てて初期化する公開関数。
  • eoc_bucket_read: バケット読み取り時のコールバック。メタデータであるため、常に成功し、長さ 0 を返します。

制御フローとデータフロー

1. **生成**: ap_bucket_eoc_create が呼び出されると、apr_bucket_alloc でメモリが確保され、ap_bucket_eoc_make で型情報が ap_bucket_type_eoc に設定されます。 2. **処理**: フィルタチェーンを流れる際、このバケットはデータを含まないため、eoc_bucket_read は単に空のポインタと長さを返します。 3. **終了**: 接続終了処理を行うフィルタがこのバケットを検知し、接続のクリーンアップやログ記録などの最終処理を実行します。

直接依存

  • httpd.h: Apache HTTP Server のコア定義。
  • http_connection.h: 接続管理に関連する定義。

関連コード

  • server/eor_bucket.c: EOR (End of Request) バケット。リクエスト終了を通知する類似のメタデータバケット。
  • server/error_bucket.c: エラー発生を通知するメタデータバケット。
  • apr_buckets.h (APR): バケットフレームワークの基盤。

読者向けメモ

  • EOC バケットは「メタデータ」バケットです。実際のデータペイロードは持たず、フィルタチェーン上の「イベント通知」としてのみ存在します。
  • このバケットがフィルタチェーンに挿入された場合、それは接続が終了することを意味するため、後続のフィルタは接続に関連するリソースの解放準備を行う必要があります。
  • apr_bucket_destroy_noop が使用されている通り、このバケット自体は特別な破棄処理を必要としません。
