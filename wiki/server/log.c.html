server/log.c

概要

Apache HTTP Server のエラーログ記録およびパイプ経由のログ出力（Piped Logs）を管理する中核モジュールです。ログレベルの判定、フォーマット処理、子プロセスへのログ転送、および PID ファイルの管理を担当します。

モジュール内での役割

server モジュール群において、ログ出力の抽象化レイヤーを提供します。http_log.h で定義されたインターフェースを実装し、他のモジュールが ap_log_error 等を通じてログを出力する際のバックエンドとして機能します。

責務

  • **ログ出力制御**: ログレベルに応じたフィルタリングと、標準エラー出力またはファイルへの書き込み。
  • **パイプログ管理**: 外部プロセスへのログ転送（ログローテーション等）と、子プロセスのライフサイクル管理。
  • **フォーマット処理**: ErrorLogFormat ディレクティブに基づくログ行の構築。
  • **プロセス管理**: PID ファイルの生成・削除およびサーバー起動時のコマンドライン記録。
  • **フック提供**: ログ生成時の ID 生成や、ログ出力後のカスタム処理のためのフックを提供。

重要なシンボル

ログ管理・構造体

  • struct piped_log: パイプ経由のログ出力用構造体。プロセス管理とファイル記述子を保持。
  • ap_default_loglevel: サーバー全体のデフォルトログレベル。
  • stderr_log: 標準エラー出力にリダイレクトされたログファイル記述子。

ログ出力・フォーマット

  • ap_log_error_core: ログ出力のメインエントリポイント。フォーマットと書き込みを統括。
  • do_errorlog_format: ErrorLogFormat に基づく動的なログ行構築。
  • do_errorlog_default: フォーマット未定義時のデフォルトログ出力。
  • write_logline: 最終的なファイル書き込みまたは syslog への転送。

フック・登録

  • ap_register_log_hooks: ログフォーマット用ハンドラ（%t, %L 等）の登録。
  • ap_run_error_log: ログ出力後に実行されるフック。
  • ap_run_generate_log_id: ログ ID 生成フック。

制御フローとデータフロー

1. **エントリ**: ap_log_error 等の関数が log_error_core を呼び出し、ログレベルとコンテキスト（server_rec, conn_rec, request_rec）を渡します。 2. **フィルタリング**: log_error_core 内で、現在のモジュールレベルとログレベルを比較し、出力の可否を決定します。 3. **フォーマット**: do_errorlog_format が apr_array_header_t に格納されたフォーマット定義を走査し、登録されたハンドラ関数を呼び出してバッファを構築します。 4. **出力**: write_logline がバッファをファイルまたは syslog に書き込みます。 5. **パイプ管理**: piped_log_maintenance が apr_proc_other_child_register を通じて子プロセスの死活監視を行い、必要に応じて再起動します。

読者向けメモ

  • **メモリ管理**: ログ出力は apr_pool_t を介して行われます。stderr_pool はサーバーの再起動サイクルを跨ぐために特別に管理されています。
  • **スレッド安全性**: ログ出力は APR のファイルロックやアトミックな操作に依存していますが、高負荷時のログ書き込みは I/O ボトルネックになり得ます。
  • **パイプログの注意点**: AP_HAVE_RELIABLE_PIPED_LOGS が定義されている環境では、子プロセスの異常終了を検知して自動再起動する堅牢な仕組みが働きます。
  • **省略された詳細**: syslog 関連の複雑なファシリティマッピングや、プラットフォーム固有の __FILE__ パス処理は、可読性を優先して詳細な分岐を省略しています。
