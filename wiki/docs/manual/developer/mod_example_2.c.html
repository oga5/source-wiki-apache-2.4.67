概要

mod_example_2.c は、Apache HTTP Server 2.4 向けのサンプルモジュールです。指定されたファイルの MD5 または SHA1 ハッシュ値を計算し、その結果を HTML 形式でクライアントに返却する機能を提供します。

モジュール内での役割

Apache のリクエスト処理パイプラインにおいて、特定のハンドラ（example-handler）をフックし、ファイルシステム上のファイルに対するメタデータおよびダイジェスト計算を行う実演用コンポーネントです。

責務

  • **ハンドラの実装**: example-handler が指定されたリクエストを捕捉し、処理の可否を判定する。
  • **ファイル検証**: リクエストされたパスから拡張子を除去し、対象ファイルが実在し、かつディレクトリではないことを確認する。
  • **ダイジェスト計算**: APR (Apache Portable Runtime) のユーティリティを用いて、ファイルのバイナリデータから MD5 または SHA1 ハッシュを生成する。
  • **レスポンス生成**: 計算結果を HTML として整形し、クライアントへ出力する。

重要なシンボル

  • example_handler: リクエスト処理のメインロジック。ハンドラの判定からハッシュ計算、レスポンス出力までを担う。
  • register_hooks: Apache のフックシステムに example_handler を登録する関数。
  • example: モジュール定義構造体。Apache がモジュールをロードする際の起点となる。

制御フローとデータフロー

1. **フック**: register_hooks により、リクエスト処理の最終段階で example_handler が呼び出されるよう設定されます。 2. **判定**: r->handler が example-handler であるかを確認し、一致しない場合は DECLINED を返して他のモジュールへ処理を委譲します。 3. **検証**: apr_stat を使用して対象ファイルの存在を確認し、エラー時は適切な HTTP ステータス（404/403）を返します。 4. **解析**: クエリパラメータ（digest）から計算対象のアルゴリズムを特定します。 5. **計算と出力**: ファイルをストリームとして読み込み、apr_md5 または apr_sha1 コンテキストで逐次計算を行い、結果を ap_rprintf でレスポンスボディに書き込みます。

直接依存

  • httpd.h, http_config.h, http_protocol.h: Apache コア API。
  • apr_md5.h, apr_sha1.h: ハッシュ計算用 APR ユーティリティ。
  • apr_file_io.h: ファイルシステム操作用 API。
  • apr_tables.h: クエリパラメータ解析用テーブル構造。

関連コード

  • mod_example_1.c: 同ディレクトリ内の対となるサンプルモジュール。
  • ap_args_to_table: クエリ文字列を解析するための Apache API。
  • apr_pool_t: Apache のメモリ管理機構。本ファイル内の全リソース確保に使用。

読者向けメモ

  • 本モジュールは学習用であり、実運用環境での使用は想定されていません。
  • ファイル読み込み時に 256 バイトの固定バッファを使用していますが、これはサンプルとしての簡略化です。
  • apr_stat の結果やファイルポインタの管理には、Apache のメモリプール（r->pool）が活用されており、リクエスト終了時に自動的にクリーンアップされる設計になっています。
