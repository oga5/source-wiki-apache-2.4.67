docs/manual/developer/mod_example_1.c

概要

Apache HTTP Server 2.4 向けの最小構成のモジュール実装例です。特定のハンドラ名（example-handler）に対するリクエストを捕捉し、クライアントへ "Hello, world!" を返却する基本的な動作を提供します。

モジュール内での役割

Apache モジュール開発における「ハンドラ」の登録と実行のライフサイクルを実演するテンプレートです。開発者が Apache のフック機構とリクエスト処理の基本フローを理解するための導入部として機能します。

責務

  • **ハンドラの登録**: register_hooks を通じて、Apache のリクエスト処理パイプラインに example_handler を組み込む。
  • **リクエストの選別**: r->handler を検証し、対象外のリクエストを DECLINED で透過させる。
  • **レスポンス生成**: コンテンツタイプの設定と、ap_rprintf を用いたレスポンスボディの書き出し。

重要なシンボル

  • example_handler: リクエストを処理するメイン関数。request_rec を引数に取り、処理の成否を返します。
  • register_hooks: モジュール読み込み時に実行され、フックを登録する関数。
  • example: AP_DECLARE_MODULE マクロで定義されるモジュール構造体。Apache がモジュールを認識するためのエントリポイントです。

制御フローとデータフロー

1. **初期化**: Apache 起動時に register_hooks が呼ばれ、ap_hook_handler により example_handler がハンドラチェーンの末尾（APR_HOOK_LAST）に登録されます。 2. **リクエスト受信**: HTTP リクエストが発生すると、登録されたハンドラが順次実行されます。 3. **判定**: example_handler 内で r->handler が "example-handler" かを確認します。一致しない場合は即座に DECLINED を返し、他のモジュールへ処理を委譲します。 4. **応答**: 一致した場合、ap_set_content_type で MIME タイプを指定し、ap_rprintf でデータを送信後、OK を返して処理を完了します。

直接依存

  • httpd.h: Apache のコアデータ構造（request_rec 等）の定義。
  • http_config.h: モジュール定義および設定関連の API。
  • http_protocol.h: リクエスト処理およびレスポンス送信用の API。
  • ap_config.h: Apache の設定用マクロ群。

読者向けメモ

  • 本ファイルは学習用です。実運用環境では、r->handler のチェックだけでなく、セキュリティ上の観点から適切なアクセス制御やエラーハンドリングを検討してください。
  • DECLINED と OK の戻り値の違いは、Apache のリクエスト処理パイプラインにおいて極めて重要です。DECLINED は「自分は処理しない（他を試せ）」、OK は「処理完了」を意味します。
