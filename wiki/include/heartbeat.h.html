include/heartbeat.h

概要

heartbeat.h は、Apache HTTP Server におけるサーバーの稼働状況監視（ハートビート）機能のための共通データ構造を定義するヘッダーファイルです。主に mod_heartmonitor と mod_lbmethod_heartbeat 間で共有されるデータ形式を規定します。

モジュール内での役割

本ファイルは、サーバーの負荷情報や生存確認情報を共有メモリやファイル経由で受け渡すための「契約」として機能します。ロードバランサーモジュールがバックエンドサーバーの健全性を判断するための共通言語を提供します。

責務

  • ハートビート通信におけるサーバー情報のデータ構造（hm_slot_server_t）の定義。
  • IPアドレスの最大長やデフォルトのデータ保存ファイル名など、モジュール間で一貫性が必要な定数の管理。

重要なシンボル

  • hm_slot_server_t: 各サーバーの稼働状況（IP、ビジー状態、待機状態、最終確認時刻）を保持する構造体。
  • MAXIPSIZE: IPv4-mapped IPv6 アドレスを格納するために必要なバッファサイズ（46バイト）。
  • DEFAULT_HEARTBEAT_STORAGE: 共有メモリが利用できない場合にフォールバックとして使用されるデータファイル名。

制御フローとデータフロー

本ファイル自体はデータ構造の定義のみを行うため、直接的な制御フローは持ちません。

  • **データフロー**: mod_heartmonitor が収集したサーバーの統計情報が hm_slot_server_t 構造体に書き込まれ、mod_lbmethod_heartbeat がその構造体を読み取ることで、負荷分散先の選定ロジックに反映させます。

直接依存

  • apr.h: Apache Portable Runtime の基本定義。
  • apr_time.h: 最終確認時刻（apr_time_t）を扱うための時間関連定義。

関連コード

  • mod_heartmonitor.c: ハートビート情報を収集し、共有メモリやファイルへ書き込むモジュール。
  • mod_lbmethod_heartbeat.c: 本ヘッダーの構造体を利用して、動的な負荷分散を行うモジュール。
  • mod_slotmem_shm.c: 共有メモリ管理を行うモジュール（本ヘッダーのデータが格納される場所）。

読者向けメモ

  • 本ファイルは非常に軽量であり、主にデータ構造のシリアライズ形式を定義しています。
  • MAXIPSIZE は IPv6 アドレスを考慮した設計になっています。将来的なネットワークアドレスの拡張が必要な場合は、この定数と構造体の整合性に注意してください。
  • 共有メモリ（slotmem）を使用する設計であるため、構造体のパディングやアライメントがプラットフォーム間で一致していることを前提としています。
