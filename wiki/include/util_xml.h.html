include/util_xml.h

概要

Apache HTTP Server における XML 解析ユーティリティのインターフェースを定義するヘッダーファイルです。主にリクエストボディとして送信された XML データの読み込みと解析を抽象化します。

モジュール内での役割

APACHE_CORE モジュールの一部として、HTTP リクエストの処理中に XML 形式のペイロードを扱うための標準的な手段を提供します。主に WebDAV や XML-RPC などの XML ベースのプロトコルを処理するモジュールから利用されます。

責務

  • XML 解析機能の公開インターフェースの提供。
  • apr_xml（APR の XML ライブラリ）を Apache のリクエスト処理パイプライン（request_rec）に統合する。

重要なシンボル

  • ap_xml_parse_input: リクエストボディから XML を読み込み、解析して apr_xml_doc 構造体として返すメイン関数。

制御フローとデータフロー

本ヘッダーで宣言された ap_xml_parse_input は、server/util_xml.c で実装されています。 1. **入力取得**: ap_get_brigade を使用してリクエストボディをバケット単位で読み込みます。 2. **制限チェック**: ap_get_limit_xml_body を参照し、設定されたサイズ制限を超えていないか監視します。 3. **解析**: apr_xml_parser_feed を介してデータをパーサーに供給し、最後に apr_xml_parser_done でドキュメントツリーを生成します。 4. **変換**: EBCDIC 環境下では、必要に応じて文字コード変換が適用されます。

直接依存

  • apr_xml.h: APR が提供する基本的な XML 解析機能。
  • httpd.h: Apache のコア構造体（request_rec など）の定義。

関連コード

  • server/util_xml.c: ap_xml_parse_input の実装本体。
  • include/http_core.h: XML ボディサイズの制限設定（LimitXMLRequestBody）を取得するために参照されます。

読者向けメモ

  • 本ファイルは非常にシンプルですが、背後では apr_xml（通常は expat ライブラリ）が動作しています。
  • 解析エラーが発生した場合、ap_xml_parse_input は適切な HTTP ステータスコード（HTTP_BAD_REQUEST や HTTP_REQUEST_ENTITY_TOO_LARGE）を返します。
  • 大規模な XML を扱う際は、メモリ消費量と LimitXMLRequestBody の設定に注意してください。
