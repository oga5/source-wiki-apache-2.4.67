include/util_time.h

概要

Apache HTTP Server における日付・時刻処理の最適化モジュールです。主に、頻繁に発生する時刻変換処理を高速化するためのキャッシュ機構を提供します。

モジュール内での役割

apr_time.h（Apache Portable Runtime）のラッパーとして機能します。標準的な時刻変換関数を直接呼び出す代わりに、直近の時刻をキャッシュすることで、ログ出力やヘッダー生成など、同一秒内に複数回発生する時刻変換のオーバーヘッドを削減します。

責務

  • **高速な時刻変換**: 直近 AP_TIME_RECENT_THRESHOLD 秒以内の時刻に対する apr_time_exp_t への変換（explode）の提供。
  • **フォーマット変換**: ctime 形式や RFC822 形式への高速な文字列変換。
  • **拡張フォーマットのサポート**: マイクロ秒やタイムゾーンオフセットを含む柔軟な時刻文字列生成の定義。

重要なシンボル

  • AP_TIME_RECENT_THRESHOLD: キャッシュが有効な過去の秒数（デフォルト 15）。
  • ap_explode_recent_localtime / ap_explode_recent_gmt: キャッシュを利用した時刻分解関数。
  • ap_recent_ctime_ex: 拡張オプション（AP_CTIME_OPTION_*）を指定可能なフォーマット変換関数。

制御フローとデータフロー

1. **キャッシュ参照**: ap_explode_recent_* が呼び出されると、時刻 t を秒単位でハッシュ化し、リングバッファ（exploded_cache_localtime 等）を検索します。 2. **ヒット時**: キャッシュ内のデータが要求された時刻と一致すれば、計算をスキップして即座に返却します。 3. **ミス時**: apr_time_exp_* を実行して時刻を分解し、結果をキャッシュに格納してから返却します。 4. **データフロー**: apr_time_t（内部表現）から apr_time_exp_t（人間が読める構造体）への変換が主軸となります。

直接依存

  • apr.h: APR の基本定義。
  • apr_time.h: 時刻処理の基本ライブラリ。
  • httpd.h: Apache のコア定義（AP_DECLARE 等）。

関連コード

  • server/util_time.c: 本ヘッダーで宣言された関数の実装およびキャッシュの実体（リングバッファ）が含まれます。

読者向けメモ

  • **スレッド安全性**: 実装（util_time.c）には意図的なレースコンディションが含まれています。これは、キャッシュ更新中に別のスレッドが古い値を読み取っても、システム全体の整合性に影響を与えないという設計判断に基づいています。
  • **使用上の注意**: ap_explode_recent_* 系関数は、引数 t が AP_TIME_RECENT_THRESHOLD 秒以内であることを前提としています。これより古い時刻を渡すとキャッシュミスが発生し、パフォーマンス上の利点が失われます。
  • **拡張性**: 複雑なフォーマットが必要な場合は、ap_recent_ctime_ex を使用してください。AP_CTIME_OPTION_USEC や AP_CTIME_OPTION_GMTOFF を組み合わせることで、ログの精度向上に役立ちます。
