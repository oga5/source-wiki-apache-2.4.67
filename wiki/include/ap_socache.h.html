include/ap_socache.h

概要

ap_socache.h は、Apache HTTP Server における「小規模オブジェクトキャッシュ（Small Object Cache）」の抽象化レイヤーを定義するヘッダーファイルです。メモリ、共有メモリ、ディスクなど、バックエンドの実装を問わず、一貫したインターフェースでキャッシュ機能を提供するためのプロバイダー・アーキテクチャを規定します。

モジュール内での役割

Apache のモジュール（mod_ssl のセッションキャッシュなど）が、特定のキャッシュ実装に依存せずにキャッシュ機能を利用できるようにする仲介役です。ap_provider.h を利用して、実行時にキャッシュプロバイダーを動的にロード・選択する仕組みを支えています。

責務

  • キャッシュプロバイダーが実装すべきメソッド（create, init, store, retrieve, remove 等）の定義。
  • キャッシュの初期化や運用時に必要なヒント（ap_socache_hints）の提供。
  • スレッドセーフではないプロバイダーを識別するためのフラグ（AP_SOCACHE_FLAG_NOTMPSAFE）の管理。
  • キャッシュ内容を走査するためのイテレータ・インターフェースの提供。

重要なシンボル

  • ap_socache_provider_t: プロバイダーの機能を定義する構造体。各キャッシュ実装はこの構造体を埋めて登録します。
  • ap_socache_instance_t: キャッシュインスタンスを保持する不透明な型。
  • AP_SOCACHE_PROVIDER_GROUP: ap_provider でキャッシュプロバイダーを検索する際のグループ名。
  • AP_SOCACHE_FLAG_NOTMPSAFE: マルチプロセス/スレッド環境で外部ロックが必要であることを示すフラグ。

制御フローとデータフロー

1. **登録**: 各キャッシュモジュールは ap_register_provider を使用し、AP_SOCACHE_PROVIDER_GROUP に自身の ap_socache_provider_t を登録します。 2. **生成**: 利用側モジュールは ap_lookup_provider でプロバイダーを取得し、create メソッドを呼び出してインスタンスを生成します。 3. **初期化**: init メソッドでキャッシュの識別名（cname）とヒントを渡し、利用可能な状態にします。 4. **操作**: store, retrieve, remove を通じて、バイナリデータ（IDとデータ本体）の読み書きを行います。 5. **破棄**: destroy メソッドにより、リソースが解放されます。

直接依存

  • httpd.h: Apache の基本構造体（server_rec, request_rec 等）の定義。
  • ap_provider.h: プロバイダー登録・検索機能。
  • apr_pools.h: メモリ管理（プール）。
  • apr_time.h: 有効期限管理のための時間型。

関連コード

  • mod_socache_shmcb.c 等: ap_socache インターフェースを実装する具体的なキャッシュモジュール群。
  • mod_ssl.c: このインターフェースの主要な利用者の一つ（SSLセッションキャッシュ）。
  • ap_provider.h: プロバイダーの仕組みそのものを理解するために参照が必要。

読者向けメモ

  • 本ファイルはインターフェース定義のみであり、実装は含まれません。
  • retrieve メソッドの datalen は入出力両用である点に注意してください（呼び出し時にバッファサイズを渡し、戻り値として書き込まれたサイズを受け取ります）。
  • status メソッドは mod_status 向けですが、内部で ap_r* インターフェースを使用しているため、将来的なリファクタリングの対象となる可能性があります。
  • スレッドセーフでないプロバイダーを使用する場合、呼び出し側で明示的な排他制御を行う必要があります。
