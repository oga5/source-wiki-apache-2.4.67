include/util_mutex.h

概要

Apache HTTP Server におけるミューテックス（排他制御）設定および管理のための共通ライブラリです。各モジュールが独自のミューテックスを登録し、ユーザーが Mutex ディレクティブを通じてメカニズム（flock, pthread 等）を選択・制御するためのインターフェースを提供します。

モジュール内での役割

Apache のコア機能の一部として、プラットフォーム依存の排他制御メカニズムを抽象化します。各モジュールが直接 APR のミューテックスを扱うのではなく、本ヘッダーを介して登録・作成を行うことで、サーバー全体での一貫した設定管理を実現します。

責務

  • **設定のパース**: Mutex ディレクティブの文字列を解析し、適切なロックメカニズムとファイルパスへ変換する。
  • **ミューテックスの登録**: モジュールが使用するミューテックスの種類をコアに事前登録する。
  • **インスタンス生成**: 登録済みの定義に基づき、apr_global_mutex_t や apr_proc_mutex_t を生成・初期化する。
  • **設定の優先順位管理**: 個別設定、グローバル設定、モジュールデフォルト、組み込みデフォルトの順でメカニズムを決定する。

重要なシンボル

  • ap_mutex_register(): モジュールが起動時にミューテックス型を登録するための必須関数。
  • ap_global_mutex_create() / ap_proc_mutex_create(): 登録済み型に基づき、実際にロックオブジェクトを生成する関数。
  • ap_parse_mutex(): 設定文字列を解析する内部ユーティリティ。
  • AP_MUTEX_ALLOW_NONE: 「none」設定を許可するためのフラグ。

制御フローとデータフロー

1. **登録フェーズ**: モジュールは pre_config フック内で ap_mutex_register() を呼び出し、ミューテックスの型名とデフォルト値をコアに通知します。 2. **設定フェーズ**: ユーザーが httpd.conf で Mutex ディレクティブを指定すると、コアは設定を保持します。 3. **生成フェーズ**: 実行時にモジュールが ap_global_mutex_create() を呼び出すと、コアは保持している設定（優先順位に従い解決済み）を参照し、適切な APR ミューテックスを生成して返します。

直接依存

  • httpd.h: サーバーの基本構造体（server_rec 等）の定義。
  • http_config.h: 設定ディレクティブ処理のための定義。
  • apr_global_mutex.h: APR レベルのグローバルミューテックス機能。

関連コード

  • server/util_mutex.c: 本ヘッダーで宣言された関数の実装。
  • include/http_core.h: Mutex ディレクティブの定義やコア設定構造体が含まれる。

読者向けメモ

  • **登録の重要性**: ap_mutex_register() を呼び出さないミューテックス型は、設定ファイルで指定しても拒否されます。必ず pre_config フックで行ってください。
  • **マルチインスタンス**: 同一のミューテックス型で複数のインスタンスが必要な場合（例: 複数の共有メモリ領域）、instance_id を使用して一意なロックファイルを生成できます。
  • **プラットフォーム依存**: 利用可能なメカニズム（flock, fcntl, pthread 等）はコンパイル時の APR の機能に依存します。AP_AVAILABLE_MUTEXES_STRING マクロを使用すると、現在の環境で利用可能な選択肢をログ等で提示できます。
