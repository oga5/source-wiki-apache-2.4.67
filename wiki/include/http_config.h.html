include/http_config.h

概要

Apache HTTP Server の設定システムの中核を定義するヘッダーファイルです。モジュールが設定ディレクティブを定義し、サーバーのライフサイクル（初期化、設定読み込み、リクエスト処理）にフックするためのインターフェースを提供します。

モジュール内での役割

APACHE_CORE モジュールの一部として、サーバー全体の構成管理を担います。他のモジュールは本ヘッダーを介して自身の構成データ構造を登録し、設定ファイル解析時のコールバックを定義します。

責務

  • **ディレクティブ定義**: モジュールが提供する設定コマンドの構文とハンドラーの登録。
  • **設定ライフサイクル管理**: 設定ファイルの読み込み、ツリー構築、サーバー構造への反映。
  • **モジュール管理**: 動的/静的モジュールのロード、アンロード、およびフックの登録。
  • **設定ベクトル管理**: リクエスト、接続、サーバーごとの設定データへのアクセス提供。

重要なシンボル

設定定義とディレクティブ

  • command_rec: モジュールが実装するディレクティブの定義構造体。
  • cmd_parms: ディレクティブ解析時にハンドラーへ渡されるコンテキスト情報。
  • cmd_how: 引数の解析方法（TAKE1, FLAG, RAW_ARGS 等）。
  • AP_INIT_*: command_rec を初期化するためのマクロ群。

モジュール構造

  • module: サーバーの各モジュールを表現するメイン構造体。フック登録や設定マージ関数を保持。
  • AP_DECLARE_MODULE: モジュール構造体を宣言し、ログレベルインデックスを初期化する便利マクロ。
  • STANDARD20_MODULE_STUFF: モジュール構造体の初期化用定数。

設定アクセス

  • ap_get_module_config / ap_set_module_config: 設定ベクトルからモジュール固有のデータを取得・設定するアクセサ。
  • ap_conf_vector_t: 設定データを保持する不透明なベクトル型。

制御フローとデータフロー

1. **初期化**: ap_read_config が設定ファイルを解析し、ap_directive_t ツリーを構築します。 2. **解析**: ap_walk_config がツリーを走査し、各ディレクティブに対応する command_rec の関数を呼び出します。 3. **マージ**: モジュールの merge_dir_config や merge_server_config が呼ばれ、親コンテキストから子コンテキストへ設定が継承されます。 4. **実行**: リクエスト処理中、ap_get_module_config を通じて各モジュールが自身のコンテキストを取得し、処理を行います。

直接依存

  • util_cfgtree.h: 設定ツリー構造の定義。
  • ap_config.h: サーバーの基本設定とプラットフォーム依存の定義。
  • apr_tables.h: 設定値やリストを保持するための APR テーブル構造。

関連コード

  • http_core.c: 設定システムの主要な実装（<Directory> 等の処理）。
  • util_cfgtree.c: 設定ツリーの操作ロジック。
  • mod_so.c: 動的モジュールのロードと module 構造体の登録。

読者向けメモ

  • **メモリ管理**: 設定データは主に pconf (設定プール) に割り当てられます。temp_pool は解析中のスクラッチ領域として使用され、リクエスト処理開始前に破棄されます。
  • **不変条件**: module 構造体の module_index は、サーバー起動時に割り当てられ、実行中に変更してはなりません。
  • **設計の注意**: ap_get_module_config はデバッグビルド以外ではマクロとして実装されており、非常に高速ですが、インデックスの範囲外アクセスに対する保護はありません。
  • **省略箇所**: 本ファイルはインターフェース定義に特化しており、具体的な解析ロジックやメモリ確保の詳細は server/config.c 等の実装側に委ねられています。
