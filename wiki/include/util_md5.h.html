include/util_md5.h

概要

util_md5.h は、Apache HTTP Server における MD5 ハッシュ計算のためのユーティリティインターフェースを定義するヘッダーファイルです。主に文字列やバイナリデータ、ファイルから MD5 ダイジェストを生成するための抽象化レイヤーを提供します。

モジュール内での役割

Apache のコア機能の一部として、認証、キャッシュキーの生成、または Content-MD5 ヘッダーの計算など、データ整合性や識別子が必要な場面で利用されます。APR (Apache Portable Runtime) の MD5 実装をラップし、Apache のメモリ管理（プール）と統合した使いやすい API を提供します。

責務

  • APR の MD5 コンテキストを用いたハッシュ計算の簡略化。
  • メモリプール (apr_pool_t) を活用した結果文字列の自動管理。
  • バイナリデータ、文字列、ファイルストリームに対する一貫したダイジェスト生成インターフェースの提供。
  • 必要に応じた Base64 エンコーディングへの変換サポート。

重要なシンボル

  • ap_md5(): 文字列から MD5 ハッシュ（16進数文字列）を生成する簡易関数。
  • ap_md5_binary(): 任意の長さのバイナリデータから MD5 ハッシュを生成するコア関数。
  • ap_md5contextTo64(): MD5 コンテキストから Base64 エンコードされたダイジェストを生成する。
  • ap_md5digest(): 指定されたファイルの内容から MD5 ダイジェストを生成する。

制御フローとデータフロー

1. **初期化**: 呼び出し元が apr_pool_t を提供し、関数を呼び出します。 2. **計算**: ap_md5_binary 等の関数内で apr_md5_ctx_t が初期化され、データが更新 (apr_md5_update) されます。 3. **ファイナライズ**: apr_md5_final によりバイナリ形式のハッシュ値が取得されます。 4. **変換と返却**: 取得したバイナリハッシュを 16 進数文字列または Base64 文字列に変換し、提供されたプールからメモリを割り当てて呼び出し元に返却します。

直接依存

  • apr_md5.h: APR が提供する低レベル MD5 実装およびコンテキスト構造体。

関連コード

  • server/util_md5.c: 本ヘッダーで宣言された関数の実装が含まれています。EBCDIC 環境への対応や、バイナリから 16 進数への変換ロジックが記述されています。

読者向けメモ

  • 本モジュールは APR の MD5 実装をラップしているため、高度な制御（コンテキストの直接操作など）が必要な場合は、直接 apr_md5.h の API を使用することを検討してください。
  • 返却される文字列はすべて apr_pool_t から割り当てられるため、呼び出し元で free() を呼ぶ必要はありません。プールの生存期間に依存することに注意してください。
