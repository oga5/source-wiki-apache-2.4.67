include/mod_request.h

概要

mod_request.h は、Apache HTTP Server の mod_request モジュールが提供する機能のプライベートインターフェースを定義するヘッダーファイルです。主に、リクエストボディを一時的に保持（Keep）し、後続の処理で再利用可能にするための設定構造体とフィルタ識別子を提供します。

モジュール内での役割

mod_request モジュールは、入力フィルタチェーンを操作してリクエストボディをバッファリングする役割を担います。本ヘッダーは、このモジュールが提供する設定項目やフィルタ名を他のモジュールやコア機能から参照可能にするためのブリッジとして機能します。

責務

  • **設定定義**: リクエストボディの保持サイズを制御する request_dir_conf 構造体の提供。
  • **フィルタ識別**: KEEP_BODY および KEPT_BODY フィルタの定数定義。
  • **オプション機能の公開**: ap_request_insert_filter 等のオプション関数を介した、フィルタの動的な挿入・削除インターフェースの提供。

重要なシンボル

  • request_dir_conf: ディレクトリ単位の設定構造体。keep_body メンバで保持するボディの最大サイズを管理します。
  • KEEP_BODY_FILTER: リクエストボディを読み込み、保持するための入力フィルタ名。
  • KEPT_BODY_FILTER: 既に保持されたボディを後続のフィルタへ提供するための入力フィルタ名。
  • request_module: モジュール本体のグローバル変数。

制御フローとデータフロー

本ファイル自体は宣言のみですが、mod_request.c と連携して以下のフローを支えます。 1. **設定**: request_dir_conf を通じて KeepBody ディレクティブが読み込まれます。 2. **フィルタ挿入**: ap_request_insert_filter が呼ばれると、KEEP_BODY_FILTER が入力フィルタチェーンに挿入されます。 3. **ボディ保持**: keep_body_filter がリクエストボディを読み込み、必要に応じてメモリまたは一時ファイルに退避させます。 4. **再利用**: 別のモジュールがボディを要求した際、KEPT_BODY_FILTER が保持されたデータを供給します。

直接依存

  • apr.h, apr_buckets.h: APRの基本型およびバケット管理機能。
  • apr_optional.h: オプション関数（APR_DECLARE_OPTIONAL_FN）の利用。
  • httpd.h: Apacheの基本リクエスト構造体（request_rec）の定義。
  • util_filter.h: フィルタチェーン操作のための基盤。

関連コード

  • modules/filters/mod_request.c: 本ヘッダーで定義された構造体やフィルタの実装本体。
  • include/http_request.h: リクエスト処理のライフサイクルに関する定義。
  • include/util_filter.h: フィルタチェーンの仕組みを理解するための必須ヘッダー。

読者向けメモ

  • このヘッダーは mod_request の内部的な設定共有を主目的としています。
  • APR_DECLARE_OPTIONAL_FN を使用しているため、mod_request がロードされていない環境でもコンパイルエラーにはなりませんが、実行時に機能が有効かどうかを確認する必要があります。
  • リクエストボディの保持はメモリ消費を伴うため、keep_body の設定値には注意が必要です。
