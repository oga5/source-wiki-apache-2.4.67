include/util_cookies.h

概要

HTTP Cookie（RFC 2109 および RFC 2965）の読み取り、書き込み、および削除を行うための Apache HTTP Server 用のユーティリティライブラリです。

モジュール内での役割

Apache のコア機能の一部として、リクエスト処理中にヘッダーテーブルを操作し、セッション管理や状態保持に必要な Cookie 情報を抽象化して提供します。

責務

  • **Cookie の生成と書き込み**: 指定された属性（Max-Age 等）を付与した Set-Cookie / Set-Cookie2 ヘッダーの構築。
  • **Cookie の削除**: 有効期限を過去に設定することで、クライアント側での Cookie 破棄を促すヘッダーの生成。
  • **Cookie の読み取り**: リクエストヘッダーから特定の名前の Cookie 値を抽出。
  • **バリデーション**: Cookie 名が不正な文字を含まないか確認する簡易チェック。

重要なシンボル

  • ap_cookie_write / ap_cookie_write2: RFC 2109/2965 準拠の Cookie をヘッダーテーブルに追加する関数。可変長引数で複数のテーブル（r->headers_out 等）を対象にできる。
  • ap_cookie_read: リクエストから Cookie 値を取得する。remove フラグにより、バックエンドへ渡す前にヘッダーから削除可能。
  • DEFAULT_ATTRS: デフォルトの Cookie 属性（HttpOnly;Secure;Version=1）。
  • ap_cookie_do: Cookie 処理の内部状態を保持するための構造体。

制御フローとデータフロー

1. **書き込み**: 呼び出し元が request_rec と対象のテーブル（apr_table_t）を渡します。関数内で apr_pstrcat を使用して Cookie 文字列を構築し、apr_table_addn を通じてヘッダーテーブルに挿入します。 2. **読み取り**: ap_cookie_read がリクエストヘッダーをスキャンします。重複がある場合は APR_EGENERAL を返し、remove が指定されていればテーブルから該当エントリを削除します。

直接依存

  • apr_errno.h: APR エラーコード定義。
  • httpd.h: Apache の基本構造体（request_rec 等）およびマクロ定義。

関連コード

  • server/util_cookies.c: 上記ヘッダーで宣言された関数の実装。apr_strings.h や http_log.h を利用して文字列操作とログ出力を行っています。

読者向けメモ

  • 本ライブラリは可変長引数（...）を使用して複数の apr_table_t を一度に更新できる設計になっています。呼び出し時には必ず最後に NULL を渡す必要があります。
  • ap_cookie_write 系の関数は、属性が指定されない場合に DEFAULT_ATTRS を自動的に適用するため、セキュリティ要件（HttpOnly 等）がデフォルトで考慮されています。
