include/httpd.h

概要

httpd.h は Apache HTTP Server の中心的なヘッダーファイルです。サーバーの基本設定、HTTP プロトコル定数、リクエスト処理の主要なデータ構造（request_rec, conn_rec, server_rec）、およびモジュール開発者が利用する広範なユーティリティ関数を定義しています。

モジュール内での役割

Apache のコアおよび全モジュールが依存する最上位のインターフェースです。他の http_*.h や util_*.h ファイルが提供する機能の基盤となる型定義と定数を提供し、サーバーのライフサイクル全体にわたるデータ共有の規約を定めています。

責務

  • **グローバル設定定数**: サーバーのルートディレクトリ、デフォルトのタイムアウト、リクエスト制限などの定義。
  • **HTTP プロトコル定義**: ステータスコード、メソッド、MIME タイプ、HTML Doctype の定義。
  • **リクエスト処理のデータ構造**: プロセス、サーバー、コネクション、リクエストの各コンテキストの定義。
  • **ユーティリティ関数**: 文字列操作、パス正規化、URL エンコード/デコード、正規表現、メモリ管理ラッパーの提供。

重要なシンボル

データ構造

  • request_rec: リクエストごとの状態（ヘッダー、URI、メソッド、フィルタチェーン等）を保持する中心的な構造体。
  • conn_rec: クライアントとの接続状態（IP、ソケット、フィルタ等）を保持。
  • server_rec: 仮想ホストごとの設定と状態を保持。
  • process_rec: サーバープロセス全体の情報を保持。

HTTP 定数

  • HTTP_*: 100番台から500番台までの標準的な HTTP ステータスコード。
  • M_*: M_GET, M_POST 等の HTTP メソッド定数。
  • APEXIT_*: MPM（マルチプロセッシングモジュール）の子プロセス終了ステータス。

ユーティリティ関数群

  • **文字列/パス操作**: ap_getword_*, ap_escape_*, ap_normalize_path, ap_make_full_path。
  • **メモリ管理**: ap_malloc, ap_calloc, ap_realloc（OOM 時に ap_abort_on_oom を呼び出す安全なラッパー）。
  • **正規表現**: ap_pregcomp, ap_pregsub。
  • **セキュリティ**: ap_memeq_timingsafe 等のタイミング攻撃対策関数。

制御フローとデータフロー

  • **リクエスト処理**: request_rec が各フェーズ（翻訳、認証、ハンドラ実行）で受け渡され、モジュールが per_dir_config や notes を通じて情報を蓄積・共有します。
  • **メモリ管理**: ほとんどの動的メモリは apr_pool_t に紐付けられ、リクエスト終了時に自動解放されます。
  • **フィルタチェーン**: input_filters / output_filters を通じてデータがストリームとして処理されます。

直接依存

  • apr.h 系: APR (Apache Portable Runtime) の基本機能。
  • ap_config.h, ap_mmn.h: ビルド設定とモジュール API バージョン。
  • ap_regex.h: 正規表現エンジン。
  • apr_uri.h: URI 解析機能。

  • util_filter.c: フィルタチェーンの管理。

読者向けメモ

  • **メモリ所有権**: request_rec 内の多くの文字列はリクエストプールに割り当てられており、手動解放は不要です。
  • **レイアウト**: request_rec は非常に巨大な構造体です。プロトコル処理以外のモジュールは、必要なフィールドのみにアクセスし、不必要な変更を避けるべきです。
  • **注意点**: AP_DECLARE マクロはプラットフォームごとのエクスポート属性を制御します。モジュール開発時はこのマクロを介して API を利用してください。
  • **省略箇所**: 本ファイルは非常に広範なため、個別の MPM 特有のロジックや、特定のモジュール（mod_auth等）の詳細な内部構造については、各モジュールのヘッダーを参照してください。
