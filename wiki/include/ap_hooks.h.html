include/ap_hooks.h

概要

Apache HTTP Server のフック機構を定義するヘッダーファイルです。Apache 独自のフックシステムを構築するためのマクロ群を提供し、モジュール間での疎結合な連携を可能にします。

モジュール内での役割

Apache のコア機能と各モジュールを繋ぐ「イベント駆動の基盤」です。APR（Apache Portable Runtime）のフック機能をラップし、Apache 固有の命名規則（ap_ プレフィックス）を適用することで、サーバー全体の拡張性を担保しています。

責務

  • フックの宣言および実装用マクロの提供。
  • オプションフック（動的にロードされるモジュール間での連携）のサポート。
  • フック実行時のプロファイリング（ap_hook_probes.h）の有効化制御。

重要なシンボル

  • AP_DECLARE_HOOK: 新しいフックを宣言するためのマクロ。
  • AP_IMPLEMENT_HOOK_RUN_ALL: 登録された全関数を順次実行するフックの実装。
  • AP_IMPLEMENT_HOOK_RUN_FIRST: 最初の関数が DECLINED 以外を返すまで実行するフックの実装。
  • AP_IMPLEMENT_HOOK_VOID: 戻り値を無視して全関数を実行するフックの実装。
  • AP_OPTIONAL_HOOK: 必須ではない（オプションの）フックを登録するためのマクロ。

制御フローとデータフロー

本ファイルはマクロ定義のみで構成されており、実行時のロジックは持ちません。 1. **宣言**: AP_DECLARE_HOOK を用いてフック名とシグネチャを定義。 2. **実装**: AP_IMPLEMENT_HOOK_* マクロが ap_run_<name> という関数を生成。 3. **実行**: サーバーのメインループやリクエスト処理中に ap_run_<name>(...) が呼び出されると、APR のフック管理機構を通じて登録済みのモジュール関数が順次実行されます。

直接依存

  • apr.h: 基本的なプラットフォーム定義。
  • apr_hooks.h: APR レベルのフック基盤。
  • apr_optional_hooks.h: オプションフックのサポート。
  • ap_hook_probes.h: （条件付き）フック実行のトレース用。

関連コード

  • apr_hooks.c: 本ファイルで定義されたマクロが内部的に呼び出す、実際のフック管理ロジックの実装。
  • http_config.h: モジュールがフックを登録する際に使用する ap_hook_* 関数群が定義されています。

読者向けメモ

  • 本ファイルは直接的な関数実装を含まず、APR のフック機能を Apache 用に抽象化するインターフェースです。
  • AP_IMPLEMENT_HOOK_* マクロを使用すると、自動的に ap_run_<name> という関数が生成されます。モジュール開発者はこの命名規則に従ってフックを呼び出します。
  • フックの実行順序（pre, succ）や優先度については、APR のドキュメントを参照してください。
