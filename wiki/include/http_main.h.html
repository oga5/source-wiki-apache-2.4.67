include/http_main.h

概要

http_main.h は、Apache HTTP Server の起動プロセスおよびコマンドライン引数の解析に関連するグローバル変数と定数を定義するヘッダーファイルです。サーバーの初期化フェーズで必要となる設定情報や実行状態を保持します。

モジュール内での役割

Apache コア（APACHE_CORE）の一部として、サーバーのメインプロセス（http_main.c）と他のモジュール間で共有されるグローバルな設定状態を定義します。主に起動時の引数処理や、サーバーのライフサイクル管理の基盤となります。

責務

  • コマンドライン引数（AP_SERVER_BASEARGS）の定義と共有。
  • サーバーの実行モード（ap_run_mode）や状態（ap_main_state）の管理。
  • 起動時に読み込まれる設定ディレクティブ（-C, -c, -D）の保持。
  • サーバーの基本パス（ServerRoot, RuntimeDir）およびグローバルプール（ap_pglobal）の公開。

重要なシンボル

  • AP_SERVER_BASEARGS: apr_getopt 形式で定義された、Apache が標準で受け付けるコマンドライン引数の文字列。
  • ap_server_conf: サーバー全体の構成を保持する server_rec 構造体へのポインタ。
  • ap_server_pre_read_config / ap_server_post_read_config: 設定ファイル読み込み前後に処理されるディレクティブを格納する配列。
  • ap_signal_server: -k オプション（シグナル送信）を処理するためのオプション関数。

制御フローとデータフロー

本ファイルは実行ロジックを含みませんが、サーバー起動時に以下のフローを支えます。 1. **引数解析**: http_main.c が AP_SERVER_BASEARGS を使用してコマンドラインを解析。 2. **設定蓄積**: 解析された -C や -D の値が、対応する ap_server_... 配列に格納される。 3. **状態共有**: サーバーの起動状態や実行モードがグローバル変数として更新され、各モジュールが ap_main_state 等を参照して自身の動作を決定する。

直接依存

  • httpd.h: サーバーの基本構造体（server_rec 等）の定義。
  • apr_optional.h: オプション関数（ap_signal_server）の登録と取得のための APR 機能。

関連コード

  • server/main.c (または http_main.c): 本ヘッダーで定義された変数を実際に操作・初期化するメインエントリーポイント。
  • include/http_config.h: 設定ディレクティブの処理に関連する定義。
  • include/ap_mpm.h: マルチプロセッシングモジュール（MPM）がコマンドライン引数を再利用する際のインターフェース。

読者向けメモ

  • 本ファイルで定義されている変数は AP_DECLARE_DATA で修飾されており、Windows 等のプラットフォームで DLL 境界を越えてエクスポートされることを想定しています。
  • ap_pglobal はサーバーの生存期間中ずっと有効なメモリプールです。初期化の非常に早い段階で使用されます。
  • サーバーの実行モード（テスト、ダンプ、通常起動）を判定する際は、ap_run_mode を参照してください。
