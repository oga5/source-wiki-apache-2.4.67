include/util_charset.h

概要

util_charset.h は、Apache HTTP Server における文字セット変換、特に EBCDIC 環境と ASCII 環境間のプロトコルヘッダー変換を抽象化するためのヘッダーファイルです。

モジュール内での役割

Apache のコア機能の一部として、プラットフォーム固有の文字コード差異を吸収する役割を担います。主にネットワーク経由で送受信されるプロトコルヘッダーの変換ハンドルを定義し、システムが EBCDIC ベースか否かに応じて動的に動作を切り替えます。

責務

  • プロトコルヘッダー変換用の apr_xlate_t ハンドルの宣言。
  • EBCDIC 環境における変換ハンドルのグローバルな提供。
  • ASCII 環境における不要なシンボル定義の抑制（コンパイル時の条件分岐）。

重要なシンボル

  • APACHE_UTIL_CHARSET_H: 多重インクルード防止用ガード。
  • ap_hdrs_to_ascii: ローカル形式からネットワーク用 ASCII への変換ハンドル（EBCDIC 環境のみ）。
  • ap_hdrs_from_ascii: ネットワーク用 ASCII からローカル形式への変換ハンドル（EBCDIC 環境のみ）。
  • APR_CHARSET_EBCDIC: プラットフォームが EBCDIC かどうかを判定する APR 定数。

制御フローとデータフロー

このファイル自体はデータ構造の宣言のみを行います。実際のデータフローは以下の通りです： 1. **初期化**: サーバー起動時に ap_init_ebcdic() が呼び出され、ap_hdrs_to_ascii および ap_hdrs_from_ascii が初期化されます。 2. **変換**: プロトコルヘッダーの送受信時に、これらのハンドルを使用して apr_xlate_conv_buffer() 等が実行され、メモリ上のデータが変換されます。 3. **条件分岐**: ASCII システムではこれらの変数は存在しないため、コード内で使用する際は必ず #if APR_CHARSET_EBCDIC で囲む必要があります。

直接依存

  • apr.h: APR の基本定義。
  • apr_xlate.h: APR の文字セット変換機能（EBCDIC 環境時のみ）。

関連コード

  • server/util_charset.c: 宣言されたハンドルの実体定義。
  • ap_init_ebcdic(): 変換ハンドルの初期化を行う関数（http_core.c 等で呼び出される）。

読者向けメモ

  • **移植性の注意**: このヘッダーで定義される変数は、ASCII システムではリンクエラーの原因となります。必ず #if APR_CHARSET_EBCDIC によるガードを徹底してください。
  • **目的**: 本質的には EBCDIC 環境（メインフレーム等）での互換性維持のための仕組みであり、一般的な Linux/Windows 環境（ASCII/UTF-8）では実質的に何も行われません。
