include/util_script.h

概要

util_script.h は、Apache HTTP Server における CGI スクリプトや外部プログラムとの連携を支援するためのユーティリティ関数群を定義するヘッダーファイルです。

モジュール内での役割

Apache のコアモジュールの一部として、CGI プロトコルに準拠した環境変数の構築や、スクリプトから出力されるヘッダーの解析・検証を担います。server/util_script.c と対になっており、モジュール間の依存関係を最小化しつつ、スクリプト実行に関連する共通処理を一元化しています。

責務

  • **環境変数の管理**: request_rec から CGI 規格に準拠した環境変数を生成・設定する。
  • **スクリプト出力の解析**: スクリプトが生成したヘッダーを読み取り、HTTP レスポンスヘッダーとして適切か検証・統合する。
  • **クエリ文字列の処理**: リクエストのクエリ引数を解析し、操作可能なテーブル形式へ変換する。

重要なシンボル

  • ap_create_environment: テーブルから実行用環境変数配列を生成する。
  • ap_add_common_vars: CGI 必須の環境変数をリクエストに追加する。
  • ap_scan_script_header_err_...: スクリプト出力を解析する一連の関数群。_brigade 版はバケットブリゲード経由での読み取りに対応。
  • AP_TRUST_CGILIKE_CL_ENVVAR: CGI 互換の Content-Length 処理を制御するための環境変数名。

制御フローとデータフロー

1. **環境構築**: ap_add_common_vars 等がリクエストの subprocess_env テーブルを更新し、ap_create_environment がそれを OS の exec 呼び出しに適した形式へ変換します。 2. **ヘッダー解析**: スクリプトの標準出力（ファイルまたはバケットブリゲード）から行単位で読み取り、ap_scan_script_header_err_core がヘッダーの妥当性を検証します。 3. **エラーハンドリング**: 解析中に不正なフォーマットが検出された場合、buffer 引数にエラー原因が格納され、HTTP_INTERNAL_SERVER_ERROR が返されます。

直接依存

  • apr_buckets.h: バケットブリゲードによるストリーム処理に使用。
  • ap_config.h: Apache の基本設定および AP_DECLARE マクロ定義。

関連コード

  • server/util_script.c: 本ヘッダーで宣言された関数の実装。
  • include/http_protocol.h: HTTP プロトコルレベルのヘッダー操作に関連。
  • include/http_request.h: request_rec 構造体の定義元。

読者向けメモ

  • _ex サフィックスが付いた関数は、ログ出力時に使用する module_index を明示的に指定可能です。モジュール開発時には、ログの追跡性を高めるために可能な限り _ex 版の使用が推奨されます。
  • ap_scan_script_header_err_brigade は、近年の Apache のストリーム処理（フィルタチェーン）において、スクリプト出力を効率的に処理するための主要なインターフェースです。
