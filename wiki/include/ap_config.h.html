include/ap_config.h

概要

ap_config.h は、Apache HTTP Server のビルドおよびプラットフォーム抽象化レイヤーの基盤となるヘッダーファイルです。主にシンボルのエクスポート制御（DLL/DSO 関連）、コンパイラ機能の検出、およびプラットフォーム固有の構成定義を担います。

モジュール内での役割

Apache の全モジュールおよびコアコンポーネントが依存する最下位層の構成ヘッダーです。プラットフォーム間の差異（特に Windows の __declspec 制御）を吸収し、一貫した API インターフェースを提供します。

責務

  • **シンボル可視性の制御**: AP_DECLARE 等のマクロを通じ、静的リンクと動的リンク（DSO）の切り替えを抽象化。
  • **プラットフォーム抽象化**: Windows, NetWare, Unix 間の呼び出し規約や属性の差異を吸収。
  • **コンパイラ機能の検出**: C99 準拠状況や、GCC/Clang 固有の属性（sentinel, warn_unused_result 等）のサポート判定。
  • **DTrace 統合**: システムプローブの有効化判定とヘッダーの切り替え。

重要なシンボル

シンボルエクスポート制御

  • AP_DECLARE(type): コア関数の宣言に使用。プラットフォームに応じて __stdcall や __declspec を付与。
  • AP_DECLARE_NONSTD(type): 可変長引数関数用。C 呼び出し規約を強制。
  • AP_DECLARE_DATA: グローバル変数やモジュール構造体のエクスポート用。
  • AP_MODULE_DECLARE(...): モジュール固有のシンボル定義用。

コンパイラ属性マクロ

  • AP_FN_ATTR_SENTINEL: 可変長引数関数の末尾 NULL チェック用。
  • AP_FN_ATTR_WARN_UNUSED_RESULT: 戻り値の無視を警告する属性。
  • AP_FN_ATTR_ALLOC_SIZE(x): メモリ割り当て関数のサイズ引数指定用。

環境構成

  • DEFAULT_PIDLOG: プロセス ID ログファイルのデフォルトパス。
  • AP_HAVE_C99: C99 標準サポートの有無。
  • AP_HAVE_DESIGNATED_INITIALIZER: 指定初期化子のサポート有無。

制御フローとデータフロー

本ファイルは実行時の制御フローを持たず、プリプロセッサによる条件分岐を通じてコンパイル時の型定義や属性付与を決定します。

  • **フロー**: WIN32 の定義有無 → AP_DECLARE_STATIC / AP_DECLARE_EXPORT の定義有無 → 最終的な __declspec 属性の決定。
  • **データ**: apache_probes.h または apache_noprobes.h のいずれか一方が、_DTRACE_VERSION の有無に基づいてインクルードされます。

直接依存

  • ap_hooks.h: フックシステムの基盤。
  • os.h: OS 固有の定義。
  • ap_config_auto.h: configure スクリプトで生成された自動構成。
  • ap_config_layout.h: インストールディレクトリレイアウト。
  • sys/sdt.h: DTrace 用システムヘッダー。
  • apache_noprobes.h
  • apache_probes.h

関連コード

  • include/ap_hooks.h: フック関数の定義。
  • include/httpd.h: Apache の主要なデータ構造と定数。
  • apache_probes.h: DTrace プローブの定義。

読者向けメモ

  • **メモリ所有権**: 本ファイルはマクロ定義のみであり、メモリ管理には関与しません。
  • **レイアウト**: Windows 環境でビルドする場合、AP_DECLARE_STATIC を適切に定義しないとリンクエラーが発生します。
  • **注意点**: AP_DECLARE マクロは、将来的な ABI 変更を容易にするための抽象化レイヤーです。直接 __declspec をコードに記述せず、必ずこれらのマクロを使用してください。
  • **省略箇所**: ap_config_auto.h の内容は環境依存のため本ファイルには含まれていません。また、DTrace の詳細なプローブ定義は apache_probes.h に委譲されています。
