include/http_ssl.h

概要

http_ssl.h は、Apache HTTP Server における SSL/TLS プロトコル処理の抽象化レイヤーを定義するヘッダーファイルです。特定の SSL 実装（OpenSSL 等）に依存せず、モジュール間で SSL 関連の機能や情報を共有するための共通インターフェースを提供します。

モジュール内での役割

Apache のコア機能と、SSL を提供するモジュール（mod_ssl など）を繋ぐブリッジとして機能します。このファイルで定義されたフックや関数を通じて、コアは SSL の状態確認、証明書の管理、OCSP ステープリングなどの高度な機能を、具体的な実装を意識することなく呼び出すことが可能です。

責務

  • **SSL 状態の可視化**: 接続が SSL で保護されているかの判定。
  • **証明書管理の抽象化**: サーバー証明書や秘密鍵の登録・収集プロセスの定義。
  • **変数ルックアップ**: SSL セッションに関連する変数（証明書情報など）の取得。
  • **OCSP ステープリング**: 証明書の有効性確認のための OCSP レスポンスの提供と取得。
  • **チャレンジ応答**: TLS-ALPN-01 等のドメイン検証用証明書の動的提供。

重要なシンボル

  • ssl_conn_is_ssl: 接続が SSL を使用しているかを確認するフック。
  • ap_ssl_var_lookup: SSL 関連の環境変数や情報を取得する汎用関数。
  • ssl_add_cert_files: サーバー証明書と秘密鍵を登録するためのフック。
  • ap_ssl_ocsp_get_resp: OCSP ステープリング用のレスポンスを取得する関数。
  • ap_ssl_answer_challenge: TLS ハンドシェイク中に特定の証明書を動的に提供するフック。

制御フローとデータフロー

本ファイルはインターフェース定義であるため、実際の処理はフックの登録と呼び出しによって行われます。 1. **初期化**: ap_setup_ssl_optional_fns が呼び出され、古いスタイルの SSL モジュール関数との互換性を確保します。 2. **設定時**: ssl_add_cert_files や ssl_ocsp_prime_hook を通じて、各モジュールが証明書や OCSP 情報を登録します。 3. **実行時**: リクエスト処理中に ap_ssl_var_lookup が呼ばれると、登録されたフックが順次実行され、適切な値が返されます。 4. **ハンドシェイク時**: ap_ssl_answer_challenge が呼ばれると、特定の条件（SNI 等）に基づき、動的に証明書が差し替えられます。

直接依存

  • httpd.h: Apache の基本構造体（conn_rec, server_rec 等）の定義。
  • apr_portable.h: ポータブルな型定義。
  • apr_mmap.h: メモリマップドファイル操作用。

関連コード

  • mod_ssl: 本ヘッダーで定義されたフックの主要な実装元。
  • http_core.h: サーバー設定や接続管理の基盤。
  • util_filter.h: SSL フィルタリング処理の接続点。

読者向けメモ

  • 本ファイルは「SSL モジュールが提供すべき機能」の契約書です。新しい SSL 関連モジュールを作成する場合、これらのフックを実装することで Apache のエコシステムに統合できます。
  • AP_DECLARE_HOOK で定義されたフックは、複数のモジュールが登録可能です。戻り値の扱いに注意し、必要に応じて DECLINED を適切に返す設計が求められます。
  • OCSP 関連の関数は、メモリの生存期間が呼び出し中のみである点に注意してください。データを保持する必要がある場合は、必ず呼び出し側でコピーを作成してください。
