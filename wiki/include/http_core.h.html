include/http_core.h

概要

Apache HTTP Server の「コア」モジュールを定義するヘッダーファイルです。HTTP デーモンの基本機能、設定構造体、およびモジュール間での設定共有のためのインターフェースを提供します。

モジュール内での役割

Apache の設定システムにおいて、core はすべてのリクエスト処理の基盤となります。他のモジュールが利用する設定の保持、リクエストのライフサイクル管理、およびフィルタチェーンの定義を担います。

責務

  • **設定管理**: ディレクトリ単位およびサーバー単位の設定構造体（core_dir_config, core_server_config）の定義。
  • **アクセス制御**: AllowOverride や Options などのディレクティブの解釈と状態保持。
  • **リクエスト処理の基盤**: リクエストの再帰制限、認証状態の保持、およびカスタムレスポンスの管理。
  • **フィルタ管理**: HTTP 入出力フィルタ、チャンクフィルタ、バイトレンジフィルタのハンドル定義。
  • **ログと統計**: エラーログのフォーマット定義と、モジュールごとの管理統計情報のフック提供。

重要なシンボル

設定構造体

  • core_dir_config: ディレクトリごとの設定（Options, AllowOverride, フィルタ設定など）。
  • core_server_config: サーバー全体の設定（ドキュメントルート、ログフォーマット、プロトコル設定など）。
  • core_request_config: リクエストごとの一時的な状態（バケットブリゲード、メモ、カスタムレスポンス）。

アクセサとユーティリティ

  • ap_allow_options(), ap_allow_overrides(): 設定値の取得。
  • ap_get_core_module_config(), ap_set_core_module_config(): core_module 固有の設定データへの高速アクセス。
  • ap_get_remote_host(): クライアントの DNS/IP 解決。

フィルタハンドル

  • ap_http_input_filter_handle, ap_http_header_filter_handle, ap_chunk_filter_handle 等。

定数

  • OPT_*: Options ディレクティブのビットマスク。
  • SATISFY_*: Satisfy ディレクティブの条件指定。
  • AP_MIN_BYTES_TO_WRITE: 最小書き込みバッファサイズ（8000バイト）。

制御フローとデータフロー

  • **設定の継承**: AP_CORE_MERGE_FLAG マクロ等を用いて、親ディレクトリから子ディレクトリへ設定がマージされます。
  • **リクエスト処理**: core_request_config がリクエストのライフサイクルを通じて保持され、フィルタチェーン（ap_core_input_filter / ap_core_output_filter）がこの構造体を通じてデータをやり取りします。
  • **フック**: get_mgmt_items や error_log などのフックを通じて、外部モジュールがコアの統計情報やログ処理に介入します。

直接依存

  • apr.h, apr_hash.h, apr_tables.h: APR 基盤データ構造。
  • http_config.h: モジュール設定フレームワーク。
  • util_filter.h: フィルタチェーンの定義。
  • ap_expr.h: 式評価エンジン。
  • sys/resource.h: リソース制限（RLIMIT）用。
  • apr_optional.h
  • apr_poll.h
  • sys/time.h

関連コード

  • modules/http/http_core.c: 本ヘッダーで宣言された構造体の実装と、ディレクティブの処理ロジック。
  • include/httpd.h: request_rec や conn_rec などの基本構造体。
  • include/http_config.h: モジュール設定の登録と取得メカニズム。

読者向けメモ

  • **メモリ管理**: 本ファイルで定義される構造体は、主にリクエストプール（r->pool）または設定プールから割り当てられます。エラーログハンドラなど、一部のコールバックではメモリ割り当てに制限があるため注意が必要です。
  • **不変条件**: core_dir_config 内の opts 関連ビットマスクは、opts_add と opts_remove が排他的であることを前提としています。
  • **省略された詳細**: MPM（Multi-Processing Module）固有の複雑な非同期処理や、特定のプラットフォーム（Windows等）固有の最適化については、実装ファイル（.c）側で詳細化されるため、本ヘッダーでは抽象的な定義に留まっています。
