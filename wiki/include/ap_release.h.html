include/ap_release.h

概要

Apache HTTP Server のバージョン情報および製品識別子を定義するヘッダーファイルです。サーバーのバージョン文字列や著作権表示など、ビルドおよび実行時に参照される静的なメタデータを一元管理します。

モジュール内での役割

include モジュールにおいて、サーバーのアイデンティティを定義する最下層の構成要素です。他のヘッダーファイルやソースコードがサーバーのバージョンを判定・表示する際の唯一のソース（Single Source of Truth）として機能します。

責務

  • **バージョン管理**: メジャー、マイナー、パッチレベルの数値定義。
  • **製品識別**: Server ヘッダー等で使用される製品名、プロジェクト名、ベンダー名の定義。
  • **文字列変換**: 数値定義をプリプロセッサで文字列化し、連結してバージョン文字列を生成する。
  • **ビルド状態の反映**: 開発ビルドか否かのフラグに基づき、バージョンサフィックス（-dev 等）を制御する。

重要なシンボル

バージョン数値定義

  • AP_SERVER_MAJORVERSION_NUMBER: メジャーバージョン。
  • AP_SERVER_MINORVERSION_NUMBER: マイナーバージョン。
  • AP_SERVER_PATCHLEVEL_NUMBER: パッチレベル。
  • AP_SERVER_DEVBUILD_BOOLEAN: 開発ビルドフラグ（0: リリース, 1: 開発）。

文字列生成マクロ

  • APR_STRINGIFY(n): 数値を文字列リテラルに変換する汎用マクロ。
  • AP_SERVER_VERSION: 最終的なバージョン文字列（例: "Apache/2.4.67"）。
  • AP_SERVER_BASEVERSION: 製品名とバージョン番号を組み合わせた基本文字列。

製品識別子

  • AP_SERVER_BASEPRODUCT: "Apache"
  • AP_SERVER_BASEPROJECT: "Apache HTTP Server"
  • AP_SERVER_BASEVENDOR: "Apache Software Foundation"

プラットフォーム用定義

  • AP_SERVER_PATCHLEVEL_CSV: Windows リソースファイル（.rc）で使用されるカンマ区切りの数値形式。

制御フローとデータフロー

本ファイルは実行時のロジックを含まず、プリプロセッサによる静的なデータ構築を行います。 1. 数値定義（_NUMBER）がベースとなり、AP_SERVER_ADD_STRING が開発フラグに応じて決定されます。 2. APR_STRINGIFY を経由して数値が文字列化されます。 3. 文字列連結により AP_SERVER_MINORREVISION から AP_SERVER_VERSION までが段階的に構築されます。

直接依存

なし（純粋なマクロ定義ファイル）。

関連コード

  • docs/manual/style/version.ent: 本ファイルの数値定義と同期させる必要があるドキュメント用定義ファイル。
  • ap_add_version_component(): 本ファイルで定義された製品トークンを拡張するための API（http_core.c 等で利用）。

読者向けメモ

  • **メモリ・ライフタイム**: すべてプリプロセッサ定数であるため、メモリ消費やライフタイムの概念はありません。
  • **不変条件**: AP_SERVER_DEVBUILD_BOOLEAN を変更した場合は、関連するドキュメント（version.ent）との整合性を手動で確認する必要があります。
  • **注意点**: APR_STRINGIFY は apr_general.h と重複定義を避けるガードが施されていますが、本ファイル単体で完結するように設計されています。
  • **省略範囲**: 本ファイルは宣言のみで構成されているため、実行時のロジックや動的なメモリ管理に関する記述は存在しません。
