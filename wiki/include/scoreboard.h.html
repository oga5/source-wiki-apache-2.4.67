include/scoreboard.h

概要

Apache HTTP Server の実行時状態を追跡するための共有メモリ管理ライブラリです。各ワーカープロセスおよびスレッドの稼働状況、リクエスト統計、接続状態を保持し、mod_status 等の監視モジュールや MPM (Multi-Processing Module) がプロセス間通信（IPC）を介してサーバーの健全性を把握するために使用します。

モジュール内での役割

include ディレクトリに位置し、サーバーのライフサイクル全体で共有される状態管理の基盤を提供します。MPM がプロセスを生成・管理する際の「状態のレジストリ」として機能し、server/scoreboard.c によって実装されます。

責務

  • **状態の集約**: 全ワーカーの稼働状態、接続数、リクエスト処理時間、統計情報を構造化して保持。
  • **IPC 管理**: 共有メモリ（apr_shm）の確保、初期化、およびプロセス間での同期的なアクセス提供。
  • **ライフサイクル管理**: サーバーの起動・再起動に伴う世代（Generation）管理と、プロセスの終了に伴う状態のクリーンアップ。
  • **抽象化**: MPM の実装詳細を隠蔽し、共通の API を通じて状態更新を可能にする。

重要なシンボル

データ構造

  • scoreboard: 共有メモリ上の global_score, process_score, worker_score を指すポインタを保持するメイン構造体。
  • worker_score: 個別のスレッド/ワーカーの統計（アクセス数、転送量、現在のリクエスト内容、ステータス）。
  • process_score: プロセス単位の管理情報（PID、世代、非同期接続のカウント）。
  • global_score: サーバー全体の稼働統計（再起動時刻、スレッド制限など）。

ステータス定数

  • SERVER_READY, SERVER_BUSY_READ, SERVER_BUSY_WRITE 等: ワーカーの現在の状態を示す列挙値。

API (アクセサ・ミューテータ)

  • ap_update_child_status_...: ワーカーの状態を更新するファミリー関数群。
  • ap_get_scoreboard_...: 共有メモリから特定のスコアボード情報を取得する関数群。
  • ap_create_sb_handle: スレッド/プロセスを識別するためのハンドルを作成。

制御フローとデータフロー

1. **初期化**: MPM が ap_create_scoreboard を呼び出し、共有メモリを確保。ap_init_scoreboard で各構造体を初期化。 2. **更新**: リクエスト処理中、MPM やプロトコルモジュールが ap_sb_handle_t を用いて ap_update_child_status を呼び出し、共有メモリ上の worker_score を直接更新。 3. **参照**: mod_status 等が ap_get_scoreboard_worker 等を介して共有メモリを読み取り、HTML/JSON 形式で出力。 4. **終了**: プロセス終了時に ap_cleanup_scoreboard が呼ばれ、共有メモリの整合性を維持。

読者向けメモ

  • **メモリ所有権**: スコアボードは共有メモリ上に配置されるため、ポインタの扱いに注意が必要です。特に worker_score 構造体内の文字列フィールド（client, request 等）はサイズが固定されており、オーバーフローを防ぐための設計がなされています。
  • **非同期 MPM**: process_score にある connections や write_completion 等のフィールドは、非同期 MPM（event など）で重要となります。
  • **制限事項**: worker_score 内の client や request フィールドは現在非推奨（DEPRECATED）の扱いとなっており、将来的な変更の可能性があります。
  • **パフォーマンス**: 頻繁な更新が発生するため、ロックの競合を最小限にする設計が求められます。本ヘッダーはインターフェース定義に徹しており、具体的な同期制御は実装側（scoreboard.c）に依存します。
