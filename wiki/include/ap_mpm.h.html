include/ap_mpm.h

概要

ap_mpm.h は、Apache HTTP Server のマルチプロセッシングモジュール（MPM）の抽象化レイヤーを定義するヘッダーファイルです。OS とのインターフェースを抽象化し、接続の分配、プロセス/スレッドの管理、およびサーバーのライフサイクル制御を担います。

モジュール内での役割

Apache コア（APACHE_CORE）の一部として、http_main と各 MPM 実装（worker, event, prefork 等）を繋ぐブリッジの役割を果たします。MPM はこのヘッダーで定義されたフックやクエリインターフェースを通じて、サーバーの動作状態をコアに通知します。

責務

  • **ライフサイクル管理**: サーバーの起動、停止、再起動の制御。
  • **接続分配の抽象化**: OS レベルのプロセス/スレッドモデルを隠蔽し、ap_process_connection を介した接続処理の提供。
  • **状態監視**: MPM の動作状態（スレッド/フォークの可否、現在の状態）の照会機能。
  • **イベント通知**: 子プロセスの状態変化や、世代交代（generation）の追跡。
  • **特権プロセス管理**: 特定の権限でプロセスを生成するユーティリティの提供。

重要なシンボル

ライフサイクル・フック

  • AP_DECLARE_HOOK(int, mpm, ...): MPM のメインループを実行するフック。
  • AP_DECLARE_HOOK(void, child_status, ...): 子プロセスの状態変化（開始、終了、スロット喪失）を通知。
  • AP_DECLARE_HOOK(void, end_generation, ...): 特定の世代の全プロセス終了を通知。

MPM クエリインターフェース

  • ap_mpm_query(int query_code, int *result): MPM の能力（スレッド/フォークの有無）や現在の設定値（最大スレッド数、状態など）を取得。
  • AP_MPMQ_*: クエリ用の定数群。AP_MPMQ_IS_THREADED, AP_MPMQ_MPM_STATE など。

プロセス管理

  • ap_os_create_privileged_process(...): 権限を伴うプロセス生成。
  • ap_mpm_register_timed_callback(...): 非同期タイマーコールバックの登録（Event MPM 等で利用）。

制御フローとデータフロー

1. **初期化**: http_main が mpm フックを呼び出し、MPM が制御権を握ります。 2. **接続処理**: 接続が到着すると、MPM は ap_process_connection を呼び出します。将来的に非同期処理が導入された場合、MPM は接続を保持し、スレッドを跨いで処理を継続する可能性があります。 3. **状態照会**: 他のモジュールは ap_mpm_query を通じて MPM の現在の設定や状態を動的に取得します。 4. **終了処理**: 停止要求時、MPM はプロセスを終了させ、apr_pool_t に登録されたクリーンアップハンドラを呼び出します。

直接依存

  • apr_thread_proc.h: プロセスおよびスレッド制御のプリミティブ。
  • httpd.h: Apache の基本データ構造（server_rec, request_rec 等）。
  • scoreboard.h: プロセス/スレッドの状態を共有メモリ上で追跡するための構造。

関連コード

  • mpm_common.h: 複数の MPM で共通して利用される補助関数群。
  • scoreboard.h: MPM がプロセス状態を記録するために使用する共有メモリ構造の定義。
  • http_main.c: MPM を起動し、サーバーのメインループを管理する実装。

読者向けメモ

  • **メモリ所有権**: MPM は pchild プールを管理しており、非正常終了時にはこのプールが破棄され、登録されたクリーンアップが実行されます。
  • **シグナル**: MPM は SIGUSR1 等のシグナルを予約する場合があります。ユーザーコードはシグナルハンドラの設定に注意が必要です。
  • **非同期性**: 将来的な非同期接続処理を見据えた設計となっており、ap_process_connection が必ずしも同期的に完了するとは限らない点に注意してください。
  • **省略された詳細**: 本ファイルはインターフェース定義に特化しており、具体的な MPM 実装（server/mpm/ 配下の各ディレクトリ）のロジックは含まれていません。
