include/http_log.h

概要

Apache HTTP Server のコアロギングライブラリを定義するヘッダーファイルです。サーバーの起動からリクエスト処理、接続管理に至るまで、システム全体の診断メッセージやエラーログを出力するためのインターフェースを提供します。

モジュール内での役割

APACHE_CORE モジュールの一部として、他のすべてのモジュールが利用する標準的なログ出力手段を提供します。httpd.h や http_config.h と密接に連携し、モジュールごとのログレベル制御や、リクエスト/接続コンテキストに基づいたログのフィルタリングを実現します。

責務

  • **ログレベルの定義**: syslog 互換の優先度および Apache 独自のトレースレベル（TRACE1-8）の管理。
  • **ログ出力インターフェース**: server_rec, conn_rec, request_rec に紐づくログ出力関数の提供。
  • **バイナリデータロギング**: バッファの内容を 16 進数形式で出力する機能。
  • **パイプ付きログ管理**: 外部プロセスへログを転送するためのパイプ処理の抽象化。
  • **プロセス管理**: PID ファイルの読み書きおよび管理。

重要なシンボル

ログレベルと定数

  • APLOG_EMERG ～ APLOG_TRACE8: ログの重大度。APLOG_LEVELMASK でマスク可能。
  • APLOGNO(n): ログメッセージに一意のタグ（AHxxxxx）を付与するためのマクロ。
  • APLOG_MARK: 呼び出し元のファイル名、行番号、モジュールインデックスを自動挿入する便利マクロ。

ログ出力関数（マクロ経由）

  • ap_log_error(...): サーバー全体（server_rec）に関連するログ。
  • ap_log_rerror(...): 特定のリクエスト（request_rec）に関連するログ。
  • ap_log_cerror(...): 特定の接続（conn_rec）に関連するログ。
  • ap_log_perror(...): 特定のコンテキストを持たないログ（プールベース）。

ログレベル判定

  • APLOG_MODULE_IS_LEVEL(...): 指定されたモジュールとコンテキストでログ出力が必要か判定するマクロ。

パイプ付きログ

  • piped_log: 外部プロセスへのパイプを管理する不透明構造体。
  • ap_open_piped_log, ap_close_piped_log: パイプのライフサイクル管理。

制御フローとデータフロー

1. **マクロ展開**: ap_log_* 関数はマクロとして定義されており、呼び出し時に APLOG_MARK を通じてコンテキスト情報が渡されます。 2. **レベル判定**: APLOG_MODULE_IS_LEVEL 等のマクロが、現在の設定（server_rec や conn_rec に保持されたログレベル）と出力しようとしているレベルを比較します。 3. **フィルタリング**: 判定が偽であれば、関数呼び出し自体がスキップされ、パフォーマンスへの影響を最小限に抑えます。 4. **出力**: 判定が真の場合、内部の ap_log_*_ 関数が呼び出され、フォーマット文字列と可変長引数が処理されます。

直接依存

  • apr_thread_proc.h: プロセス管理およびパイプ処理用。
  • http_config.h: server_rec やモジュールインデックスの定義用。
  • syslog.h: HAVE_SYSLOG 定義時に使用。

関連コード

  • server/log.c: 本ヘッダーで宣言された関数の実体。
  • include/httpd.h: server_rec, conn_rec, request_rec の定義元。
  • include/http_config.h: モジュール設定とログレベルの関連付け。

読者向けメモ

  • **メモリ所有権**: ap_open_piped_log 等で作成される構造体は、渡された apr_pool_t のライフサイクルに従います。
  • **セキュリティ警告**: ap_log_* 関数のフォーマット文字列に、ネットワークからの未検証データ（URI やヘッダー）を直接含めないでください。DoS 攻撃の脆弱性となります。必ず "%s" を介して出力してください。
  • **C++ 互換性**: C++ モジュールを使用する場合、APLOG_USE_MODULE マクロを適切に呼び出さないと、APLOG_MODULE_INDEX が正しく解決されません。
  • **省略された詳細**: ログ出力の内部的なフォーマット変換ロジックや、syslog への具体的な転送処理は server/log.c に隠蔽されており、本ヘッダーではインターフェース定義に集中しています。
