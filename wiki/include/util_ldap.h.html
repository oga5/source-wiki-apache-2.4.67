include/util_ldap.h

概要

util_ldap.h は、Apache HTTP Server における LDAP 連携機能の基盤となるヘッダーファイルです。LDAP サーバーへの接続管理、キャッシュ機構、および認証・認可のための共通インターフェースを定義しています。

モジュール内での役割

本ファイルは mod_ldap の中核であり、Apache の他のモジュール（mod_authnz_ldap など）が LDAP 機能を利用するための API を提供します。APR-util の LDAP サポートをラップし、Apache のリクエスト処理ライフサイクルに最適化された抽象化層として機能します。

責務

  • **接続管理**: LDAP 接続のオープン、クローズ、および接続プールからの検索。
  • **キャッシュ管理**: 検索結果や比較結果を共有メモリ上に保持し、LDAP サーバーへの負荷を軽減する。
  • **認証・認可支援**: ユーザー DN の取得、パスワード検証（バインド）、グループメンバーシップの確認（ネストされたグループを含む）。
  • **プラットフォーム抽象化**: Windows 等の環境差異を吸収するためのマクロ定義。

重要なシンボル

  • util_ldap_connection_t: LDAP 接続状態を保持する構造体。接続プールや SSL 設定、バインド情報を管理します。
  • util_ldap_state_t: モジュール全体のグローバルな設定とキャッシュ状態を保持する構造体。
  • APR_DECLARE_OPTIONAL_FN: 他のモジュールから動的に呼び出し可能な LDAP 関数群を定義するマクロ。
  • AP_LDAP_IS_SERVER_DOWN(s): LDAP サーバーがダウンしているかを判定するマクロ。

制御フローとデータフロー

1. **接続取得**: uldap_connection_find を呼び出し、既存の接続を再利用するか、新規接続を作成します。 2. **操作実行**: uldap_cache_compare や uldap_cache_checkuserid などの関数が、キャッシュの確認と LDAP サーバーへのクエリを仲介します。 3. **キャッシュ更新**: 成功したクエリ結果は util_ldap_state_t 内の共有メモリキャッシュに格納されます。 4. **解放**: リクエスト終了時に uldap_connection_close を呼び出し、接続をプールへ返却します。

読者向けメモ

  • 本ヘッダーは APR_HAS_LDAP が定義されている場合にのみ有効です。ビルド時に APR-util が LDAP サポート付きでコンパイルされている必要があります。
  • util_ldap_connection_t の管理はスレッドセーフである必要があります。マルチスレッド環境下での接続のロック/アンロック処理には特に注意してください。
  • キャッシュの TTL やサイズは util_ldap_state_t を通じて設定されます。パフォーマンスチューニングを行う際は、この構造体のパラメータを確認してください。
