include/ap_regkey.h

概要

ap_regkey.h は、Apache HTTP Server における Windows レジストリ操作のための抽象化レイヤーを定義するヘッダーファイルです。APR（Apache Portable Runtime）の設計思想に基づき、Win32 API を直接扱う際の複雑さを隠蔽し、メモリ管理を apr_pool_t に統合します。

モジュール内での役割

本ファイルは include モジュールの一部であり、主に Windows 環境下での設定管理やシステム情報の取得を担います。os/win32/ap_regkey.c と対になっており、HTTPD の他のモジュールがプラットフォーム固有のレジストリ操作を安全かつ一貫した方法で行うためのインターフェースを提供します。

責務

  • **レジストリハンドルの抽象化**: ap_regkey_t 構造体によるレジストリキーの管理。
  • **メモリ管理の統合**: apr_pool_t を用いた自動的なリソース解放（クリーンアップ）。
  • **Win32 API のラップ**: レジストリのオープン、クローズ、値の読み書き（文字列、バイナリ、配列）の提供。
  • **環境差異の吸収**: Unicode/ANSI 対応や、レジストリ定数の標準化。

重要なシンボル

  • ap_regkey_t: レジストリキーを保持する不透明な構造体。
  • ap_regkey_open(): レジストリキーを開くための主要なエントリポイント。
  • AP_REGKEY_LOCAL_MACHINE 等: Windows のルートキーを指す定数群。
  • AP_REGKEY_EXPAND: レジストリ値内の環境変数を展開するためのフラグ。

制御フローとデータフロー

1. **初期化**: ap_regkey_open を呼び出し、親キーとパスを指定して ap_regkey_t を取得します。この際、apr_pool_cleanup_register により、プール破棄時に自動的に RegCloseKey が呼ばれるよう登録されます。 2. **操作**: ap_regkey_value_get や ap_regkey_value_set 等を使用して、キー配下の値を操作します。 3. **終了**: ap_regkey_close を呼び出すことで、明示的にハンドルを解放し、プール内のクリーンアップ登録を解除します。

直接依存

  • apr.h: プラットフォーム固有の型定義。
  • apr_pools.h: メモリ管理機能。
  • ap_config.h: AP_DECLARE マクロ（関数エクスポート用）。

関連コード

  • os/win32/ap_regkey.c: 本ヘッダーで宣言された関数の実装。
  • apr_pools.h: メモリ管理のライフサイクルを理解するために重要。
  • http_config.h: 設定読み込み時にレジストリ操作が必要となるケースが多い。

読者向けメモ

  • **Win32 限定**: 本機能は WIN32 マクロが定義されている環境でのみ有効です。
  • **非再帰的削除**: ap_regkey_remove はサブキーを持つキーを削除できません。
  • **メモリ管理**: ap_regkey_t はプールに関連付けられているため、呼び出し元は適切なライフサイクルを持つプールを渡す必要があります。
  • **環境変数**: AP_REGKEY_EXPAND を使用する場合、レジストリ内の %VAR% 形式の文字列は自動的に展開されます。
