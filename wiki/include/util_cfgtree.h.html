include/util_cfgtree.h

概要

Apache HTTP Server の設定ファイルをメモリ上に表現するための「設定ツリー（Config Tree）」構造を定義するヘッダーファイルです。設定ファイル内のディレクティブを階層構造として保持し、サーバー起動時の設定解析プロセスをサポートします。

モジュール内での役割

APACHE_CORE_CONFIG サブシステムの一部として、設定解析フェーズで生成される中間表現を提供します。このツリーは、サーバーが設定を読み込み、各モジュールが自身のディレクティブを処理するための基盤となります。

責務

  • 設定ディレクティブを表現する ap_directive_t 構造体の定義。
  • 設定ツリーを構築するためのノード追加インターフェースの提供。
  • 親子関係（コンテナディレクティブなど）および兄弟関係（同一階層のディレクティブ）の管理。

重要なシンボル

  • ap_directive_t: 設定ツリーの各ノードを表す構造体。ディレクティブ名、引数、親・子・次ノードへのポインタ、およびソースファイル上の位置情報（ファイル名・行番号）を保持します。
  • ap_conftree: 設定ツリーのルートノードを指すグローバル変数。
  • ap_add_node(): ツリーに新しいノードを挿入する関数。ノードが子要素を持つ場合、親ポインタを更新する役割も担います。

制御フローとデータフロー

1. 設定ファイルの解析中、新しいディレクティブが見つかるたびに ap_add_node が呼び出されます。 2. current ノードが NULL の場合、新しい階層の開始とみなされ、parent が更新されます。 3. child フラグが真の場合、toadd ノードが parent の first_child として設定され、以降のノードはその配下に連なります。 4. 最終的に、設定ファイル全体がメモリ上の木構造として構築され、サーバーの各モジュールがこのツリーを走査して設定を適用します。

直接依存

  • ap_config.h: Apache の基本設定マクロや型定義を提供します。

関連コード

  • server/util_cfgtree.c: ap_add_node 関数の実装が含まれています。
  • include/http_config.h: 設定ツリーを利用してディレクティブを処理する上位のロジックが定義されています。

読者向けメモ

  • このツリーは「設定ファイルに記述された順序」を保持しますが、閉じタグ（例: </Directory>）自体はツリー内にノードとして保存されません。
  • ap_directive_t 内の last ポインタは、ツリーの末尾へのアクセスを高速化するためのショートカットですが、常に最新であるとは限らない点に注意してください。
  • この構造体は、サーバー起動時の設定読み込みフェーズでのみ主に利用されます。実行時の設定変更には別のメカニズムが関与する場合があります。
