include/ap_regex.h

概要

ap_regex.h は、Apache HTTP Server における正規表現処理の抽象化レイヤーを定義するヘッダーファイルです。PCRE (Perl Compatible Regular Expressions) ライブラリをラップし、POSIX 互換のインターフェースを提供することで、サーバー全体で一貫した正規表現のコンパイルおよび実行機能を提供します。

モジュール内での役割

include モジュール群において、テキスト処理や設定解析の基盤を担います。他のモジュール（mod_rewrite や mod_setenvif など）が正規表現を利用する際の標準的な窓口として機能し、PCRE の詳細を隠蔽します。

責務

  • **POSIX 互換ラッパー**: ap_regcomp や ap_regexec を通じて、PCRE の機能を POSIX スタイルの API で提供する。
  • **高レベル抽象化 (ap_rxplus)**: Perl 形式の構文（s/match/replace/flags など）をサポートする高レベルな正規表現操作機能の提供。
  • **設定管理**: 正規表現のコンパイルオプション（大文字小文字の区別、改行の扱い等）の定義と管理。

重要なシンボル

  • ap_regex_t: コンパイル済み正規表現を保持する構造体。
  • ap_regmatch_t: マッチした文字列のオフセット情報を格納する構造体。
  • ap_regcomp() / ap_regexec(): 正規表現のコンパイルおよび実行を行う主要関数。
  • ap_rxplus_t: 高レベルな正規表現操作（置換等）のための構造体。
  • AP_REG_*: コンパイルおよび実行時の挙動を制御するビットフラグ群。

制御フローとデータフロー

1. **コンパイル**: ap_regcomp に正規表現文字列とフラグを渡し、ap_regex_t を初期化します。 2. **実行**: ap_regexec または ap_regexec_len に ap_regex_t と対象文字列を渡し、マッチ結果を ap_regmatch_t 配列に書き込みます。 3. **解放**: 不要になった ap_regex_t は ap_regfree でメモリを解放します。 4. **高レベル操作**: ap_rxplus_compile でパターンを解析し、ap_rxplus_exec で文字列に対するマッチングや置換を実行します。

直接依存

  • apr.h: Apache Portable Runtime の基本定義。

関連コード

  • **PCRE ライブラリ**: 本ファイルがラップしている実体。
  • mod_rewrite 等の正規表現を利用する各モジュール。
  • util_pcre.c (実装側): 実際の PCRE 呼び出しを行うソースコード。

読者向けメモ

  • 本ファイルは PCRE のラッパーであるため、挙動の詳細は PCRE の仕様に依存します。
  • AP_REG_EXTENDED や AP_REG_NOSUB は現在未使用であり、互換性のために存在しています。
  • ap_rxplus 系関数は、Perl ライクな置換処理を簡略化したい場合に有用ですが、メモリ管理には apr_pool_t を使用するため、呼び出し元のプール生存期間に注意してください。
