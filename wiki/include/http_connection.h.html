include/http_connection.h

概要

Apache HTTP Server におけるコネクション処理のライフサイクルを定義するヘッダーファイルです。接続の確立からプロトコル処理、そして終了（リンガリングクローズ）に至るまでの一連のインターフェースを提供します。

モジュール内での役割

APACHE_CORE サブシステムの一部として、コアの接続管理機能を提供します。MPM（Multi-Processing Module）が受け取ったソケットを、HTTP プロトコルレベルの処理へ橋渡しする役割を担います。

責務

  • **接続のライフサイクル管理**: 接続の開始、処理、終了の各フェーズをフックとして公開。
  • **プロトコルドライバー**: ap_process_connection を通じて、登録されたプロトコルモジュールへ処理を委譲。
  • **クローズ処理の最適化**: TCP RST を防ぐための「リンガリングクローズ（遅延クローズ）」の実装。
  • **EOC（End Of Connection）管理**: 接続終了をフィルタチェーンに通知するためのバケット型の定義。

重要なシンボル

  • ap_process_connection: 接続処理のメインエントリポイント。
  • ap_lingering_close: 接続終了時にデータを完全に送受信し切るための安全なクローズ処理。
  • AP_BUCKET_IS_EOC: フィルタチェーン内で「接続終了」バケットを判定するマクロ。
  • create_connection, pre_connection, process_connection: 接続の各段階でモジュールが介入するための主要フック。

制御フローとデータフロー

1. **接続開始**: MPM がソケットを受け取り、create_connection フックが conn_rec を生成します。 2. **準備**: pre_connection フックが呼ばれ、フィルタのセットアップやプロトコル固有の初期化が行われます。 3. **処理**: ap_process_connection が呼び出され、登録されたプロトコルハンドラがリクエストを処理します。 4. **終了**: 処理完了後、ap_lingering_close が呼ばれ、クライアントからの残存データ読み込みや TCP 接続の正常な終了を試みます。この際、EOC バケットがフィルタチェーンに投入され、終了が伝播されます。

直接依存

  • apr_network_io.h: ソケット等のネットワーク抽象化レイヤー。
  • apr_buckets.h: フィルタチェーンでデータを扱うためのバケット管理。

関連コード

  • httpd.h: conn_rec 構造体定義など、接続管理の基盤となるデータ構造。
  • util_filter.h: 接続に関連付けられたフィルタチェーンの操作。
  • mpm_common.h: MPM 側での接続受け入れ処理との連携。

読者向けメモ

  • 本ファイルは「プロトコルモジュール」を開発する際の最重要インターフェースです。
  • ap_process_connection は、単一の接続に対して複数のプロトコルが順次処理を行う可能性がある（プロトコル切り替えなど）ことを考慮して設計されています。
  • リンガリングクローズは、HTTP 1.1 のパイプライン処理や Keep-Alive 接続において、クライアントとの整合性を保つために不可欠な処理です。実装時にはこの挙動を意識してください。
