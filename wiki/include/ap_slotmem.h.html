include/ap_slotmem.h

概要

ap_slotmem.h は、Apache HTTP Server における共有メモリ管理のための抽象インターフェースを定義するヘッダーファイルです。固定サイズのメモリブロック（スロット）を管理し、プロセス間でのデータ共有や状態保持を可能にするためのプロバイダーモデルを提供します。

モジュール内での役割

Apache のモジュールシステムにおいて、slotmem プロバイダーとして機能します。このヘッダーは、共有メモリの確保、アタッチ、読み書き、およびスロットの割り当て管理を行うための標準的な API を定義しており、具体的な実装（mod_slotmem_shm など）とそれを利用するモジュール（mod_proxy_balancer 等）の仲介役を果たします。

責務

  • 共有メモリを均等なサイズの「スロット」に分割して管理するインターフェースの提供。
  • プロセス間でのメモリ共有を可能にするための create および attach 操作の定義。
  • スロットの動的な確保（grab）と解放（release）、および直接アクセス（dptr）の制御。
  • スロットの状態（使用中/空き）を追跡し、一貫性を保つための抽象化。

重要なシンボル

  • ap_slotmem_provider_t: スロットメモリ操作の関数ポインタ群を保持する構造体。プロバイダーの実装はこの構造体を埋める必要があります。
  • ap_slotmem_instance_t: 共有メモリ領域を指す不透明なハンドル。
  • ap_slotmem_type_t: メモリの特性（永続化の有無、スレッドセーフティ、事前確保など）を指定するフラグ群。
  • AP_SLOTMEM_PROVIDER_GROUP: ap_provider を介してこの機能を探すための識別子。

制御フローとデータフロー

1. **初期化**: モジュールは create を呼び出し、名前付きの共有メモリ領域を確保します。既存の領域がある場合は attach を使用します。 2. **アクセス**:

  • dptr を使用してメモリの直接ポインタを取得し、高速な読み書きを行います。
  • get / put を使用して、特定のデータ長を指定した安全な読み書きを行います。

    • 3. **管理**: grab / release を通じて、スロットのライフサイクルを管理します。doall を使用すると、全スロットに対して一括でコールバック関数を実行できます。

  • apr_pools.h: メモリ管理のためのプール機構。

関連コード

  • mod_slotmem_shm.c: 本ヘッダーで定義されたインターフェースの主要な実装。
  • mod_proxy_balancer.c: slotmem を利用してロードバランサーのワーカー状態を共有する代表的なモジュール。
  • ap_provider.h: プロバイダーの登録と検索メカニズムの詳細。

読者向けメモ

  • 本ファイルはインターフェース定義のみであり、実際のメモリ確保や同期処理は apr_shm や apr_global_mutex に依存しています。
  • AP_SLOTMEM_TYPE_NOTMPSAFE フラグが指定されていない場合、実装側で適切な排他制御を行うことが期待されます。
  • スロットのサイズや数は create 時に固定されるため、動的な拡張が必要な場合は設計上の注意が必要です。
