include/ap_compat.h

概要

ap_compat.h は、Apache HTTP Server 1.3 系の古いシンボル名と、2.0 以降の新しい API シンボル名との互換性を維持するためのヘッダーファイルです。

モジュール内での役割

include ディレクトリ内に配置され、古いモジュールやレガシーコードが新しい Apache API 環境下でもコンパイル可能にするための「橋渡し」の役割を担います。

責務

  • 古いシンボル名を新しいマクロや関数へ再定義（リダイレクト）する。
  • 廃止された関数を空の操作として定義し、コンパイルエラーを回避する。

重要なシンボル

  • MODULE_VAR_EXPORT: AP_MODULE_DECLARE_DATA に再定義されます。モジュールのグローバル変数をエクスポートする際に使用されます。
  • ap_send_http_header(r): 空のステートメントに再定義されます。Apache 2.0 以降ではヘッダー送信が自動化されたため、古いコードでの呼び出しを無効化します。

制御フローとデータフロー

本ファイルはプリプロセッサによる置換のみを行います。実行時のデータフローは存在せず、コンパイル時に古いシンボルが新しい定義に置き換わることで、ビルドプロセスを正常に完了させます。

直接依存

本ファイルは他のヘッダーファイルを直接インクルードしていません。純粋なマクロ定義の集合体です。

関連コード

  • httpd.h: Apache のコア定義が含まれており、本ファイルで定義される新しいシンボル（AP_MODULE_DECLARE_DATA など）の本来の定義元です。

読者向けメモ

  • このファイルは、Apache 1.3 からの移行期に作成された互換性レイヤーです。
  • 新規に開発するモジュールでは、本ファイルで定義されている古いシンボルを使用せず、最新の API を直接使用することが強く推奨されます。
  • ap_send_http_header のように空定義されているものは、コードの移植時に削除を検討すべき技術的負債の指標となります。
