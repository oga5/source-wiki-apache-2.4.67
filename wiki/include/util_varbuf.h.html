include/util_varbuf.h

概要

util_varbuf.h は、Apache HTTP Server における可変長バッファ（Resizable Buffer）を管理するためのライブラリです。主に文字列操作を想定していますが、バイナリデータの保持にも対応しています。メモリ管理に APR（Apache Portable Runtime）のプール機構を利用し、効率的なバッファの拡張と再利用を提供します。

モジュール内での役割

include ディレクトリ内のユーティリティ群の一つとして、メモリの断片化を抑えつつ、動的にサイズが変化する文字列処理を安全に行うための基盤を提供します。特に、設定ファイルの読み込みや正規表現の置換処理など、バッファサイズが事前に予測しにくい処理で多用されます。

責務

  • **動的メモリ管理**: ap_varbuf 構造体を通じたバッファの自動拡張と、プールによるライフサイクル管理。
  • **文字列操作の抽象化**: 文字列の連結、複製、および正規表現置換結果のバッファへの直接書き込み。
  • **効率的なリソース解放**: プールがクリアされる前でも、必要に応じてバッファメモリを明示的に解放する手段の提供。

重要なシンボル

  • struct ap_varbuf: バッファ本体、現在のサイズ、有効長、および関連するプールを保持する主要構造体。
  • ap_varbuf_init(): バッファの初期化。
  • ap_varbuf_grow(): バッファの拡張。必要に応じてメモリを再割り当てし、既存データをコピーします。
  • ap_varbuf_strmemcat(): バッファへのデータ連結を行う低レベル関数。
  • ap_varbuf_strcat(): 文字列連結のための便利なマクロ。
  • AP_VARBUF_UNKNOWN: バッファ内の文字列長が不明であることを示す定数。

制御フローとデータフロー

1. **初期化**: ap_varbuf_init で構造体をプールに関連付けます。 2. **拡張**: 連結や置換の際、ap_varbuf_grow が呼び出されます。この関数は、現在の avail サイズが不足している場合に、指数関数的にバッファを拡大（通常は2倍）し、必要に応じて apr_allocator を介してメモリを確保します。 3. **操作**: ap_varbuf_strmemcat 等の関数が vb->buf に対してデータを書き込み、vb->strlen を更新します。 4. **終了**: プールのクリア時に自動的にメモリが解放されます。明示的な解放が必要な場合は ap_varbuf_free を使用します。

直接依存

  • apr.h: 基本的な型定義。
  • apr_allocator.h: メモリ割り当ての制御。
  • httpd.h: Apache サーバーのコア定義。

関連コード

  • apr_pools.h: メモリ管理の基盤となるプール機構。
  • ap_regex.h: ap_varbuf_regsub で利用される正規表現エンジン。
  • util_cfgtree.h: ap_varbuf_cfg_getline が読み込み対象とする設定ファイル構造。

読者向けメモ

  • ap_varbuf はスタック上に配置しても安全ですが、内部で参照するプールは生存している必要があります。
  • ap_varbuf_grow はメモリ不足時にプールのアボート関数を呼び出すため、呼び出し側でメモリ確保の失敗を個別にチェックする必要はありません。
  • AP_VARBUF_UNKNOWN を使用している場合、関数は内部で strlen を実行して長さを再計算するため、頻繁な呼び出しではパフォーマンスに注意してください。
