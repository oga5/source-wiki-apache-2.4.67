include/mpm_common.h

概要

mpm_common.h は、Apache HTTP Server のマルチプロセッシングモジュール（MPM）間で共有される共通機能、定数、およびフックを定義するヘッダーファイルです。プラットフォーム固有の差異を吸収しつつ、複数の MPM 実装（prefork, worker, event 等）で再利用可能なロジックを提供します。

モジュール内での役割

include ディレクトリにおいて、MPM 関連のインターフェースを統合する中心的な役割を果たします。ap_mpm.h が MPM の抽象的な定義を扱うのに対し、本ファイルはプロセス管理、シグナル処理、および設定の共通化といった「実装の詳細」を共有するための基盤となります。

責務

  • **プロセス管理**: 子プロセスの生存監視、終了処理（reclaim/relieve）、およびプロセス間通信（Pipe-of-Death）の定義。
  • **設定の共通化**: MaxRequestsPerChild や PidFile など、多くの MPM が共通して持つ設定項目の管理。
  • **フックの提供**: MPM のライフサイクルや状態変化（子プロセスの停止、接続のサスペンド/レジューム等）を外部モジュールが監視・介入するためのフック定義。
  • **OS 抽象化**: ユーザー名/グループ名の ID 変換や、TCP_NODELAY のようなソケットオプションの共通操作。

重要なシンボル

  • ap_pod_t: 「Pipe-of-Death」を管理する構造体。子プロセスに終了を通知するために使用されます。
  • ap_reclaim_child_processes(): 親プロセスが終了時に子プロセスを確実に回収するための関数。
  • AP_SIG_GRACEFUL: サーバーのグレースフルリスタートを指示するシグナル（通常 SIGUSR1）。
  • AP_ACCEPT_MUTEX_TYPE: 複数の MPM が受け入れ用ミューテックスを識別するための共通文字列。
  • ap_mpm_podx_restart_t: 拡張 Pipe-of-Death で使用される再起動/終了の種別を定義する列挙型。

制御フローとデータフロー

本ファイルは主に宣言を提供しますが、MPM の制御フローは以下のフックを通じて行われます。 1. **初期化**: mpm_common_pre_config を通じて MPM 共通の設定が初期化されます。 2. **監視**: ap_wait_or_timeout を通じて、親プロセスは子プロセスの終了を監視し、必要に応じて monitor フックを呼び出します。 3. **終了処理**: ap_mpm_pod_signal 等により Pipe-of-Death にデータが書き込まれると、子プロセスはそれを検知して終了処理（child_stopping フック）へ移行します。

直接依存

  • ap_config.h: Apache の基本設定マクロ。
  • ap_mpm.h: MPM の基本インターフェース。
  • scoreboard.h: プロセス状態を共有するためのスコアボード構造。
  • apr_proc_mutex.h: プロセス間ミューテックスの抽象化。
  • netinet/tcp.h: TCP_NODELAY などのソケットオプション用。

関連コード

  • server/mpm_common.c: 本ヘッダーで宣言された関数の実装が含まれる対となるソースファイル。
  • include/ap_mpm.h: MPM の抽象化レイヤー。
  • include/scoreboard.h: プロセス管理のデータ構造の詳細。

読者向けメモ

  • 本ファイル内の関数やマクロの多くは、WIN32 や NETWARE 以外の環境（主に Unix 系）を想定しています。
  • AP_DECLARE_HOOK で定義されているフックは、MPM の挙動をカスタマイズするモジュール開発者にとって非常に重要です。
  • Pipe-of-Death (ap_pod_t) は、Apache が子プロセスを安全に終了させるための重要なメカニズムであるため、MPM の終了処理を追う際はここから読み解くのが近道です。
