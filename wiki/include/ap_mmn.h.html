include/ap_mmn.h

概要

Apache HTTP Server の「モジュールマジックナンバー (MMN)」を定義するヘッダーファイルです。サーバーコアとロード可能なモジュール間のバイナリ互換性を保証するためのバージョン管理メカニズムを提供します。

モジュール内での役割

include ディレクトリにおいて、サーバーの API バージョン管理の基盤として機能します。モジュールがロードされる際、サーバーは自身の MMN とモジュールがコンパイルされた MMN を比較し、互換性がない場合にロードを拒否することで、構造体の不整合によるクラッシュを防ぎます。

責務

  • **バージョン定義**: メジャーおよびマイナーバージョン番号の保持。
  • **互換性チェック**: モジュールが特定の API 機能を利用可能か判定するためのマクロ提供。
  • **変更履歴の記録**: 2001 年以降の API 変更履歴をコメントとして保持し、開発者が過去の変更点を確認できるようにする。

重要なシンボル

バージョン管理定数

  • MODULE_MAGIC_COOKIE: モジュールが Apache 2.4 系であることを識別するためのシグネチャ (0x41503234UL = "AP24")。
  • MODULE_MAGIC_NUMBER_MAJOR: API の破壊的変更（構造体サイズの変更など）が発生した際に更新されるメジャーバージョン。
  • MODULE_MAGIC_NUMBER_MINOR: バイナリ互換性を維持したままの機能追加時に更新されるマイナーバージョン。

判定マクロ

  • AP_MODULE_MAGIC_AT_LEAST(major, minor): 現在のサーバーの MMN が指定したバージョン以上であるかを判定する。条件付きコンパイルに必須。

制御フローとデータフロー

本ファイルは実行時の制御フローを持たず、コンパイル時のプリプロセッサによる判定に特化しています。

  • **データフロー**: モジュール開発者が AP_MODULE_MAGIC_AT_LEAST を使用することで、特定の API が存在するかをコンパイル時に決定し、利用可能なコードパスのみをコンパイル対象とします。

直接依存

  • 依存するヘッダーファイルはありません。純粋な定数定義ファイルです。

関連コード

  • http_config.h: モジュール構造体 (module) の定義が含まれており、ここで MMN が実際にチェックされます。
  • ap_release.h: サーバーのリリースバージョン情報を保持します。

読者向けメモ

  • **バイナリ互換性**: メジャーバージョンが異なる場合、バイナリ互換性は一切保証されません。
  • **変更履歴の重要性**: 本ファイル内の巨大なコメントブロックは、単なる履歴ではなく、特定の API がいつ導入・変更されたかを知るための公式なリファレンスです。
  • **設計上の注意**: MODULE_MAGIC_NUMBER は非推奨です。必ず AP_MODULE_MAGIC_AT_LEAST を使用してください。
  • **省略された領域**: 本ファイルは定数定義のみであるため、ロジックの深掘りは不要です。ただし、モジュール開発時には、このファイルで定義されたバージョンと、実際に使用したい API が導入されたバージョンを照らし合わせる必要があります。
