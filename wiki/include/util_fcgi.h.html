include/util_fcgi.h

概要

FastCGI プロトコル（RFC 3652）を Apache HTTP Server 内で扱うためのデータ構造定義およびユーティリティ関数群を提供します。FastCGI レコードのバイナリ形式と、環境変数テーブルのシリアライズをサポートします。

モジュール内での役割

server/util_fcgi.c と対をなし、mod_proxy_fcgi 等の FastCGI を利用するモジュールに対して、プロトコルレベルのデータ変換層を提供します。

責務

  • **プロトコル定義**: FastCGI レコードヘッダーおよび BEGIN_REQUEST ボディの構造体定義。
  • **バイナリ変換**: ネットワークバイトオーダーを意識した、構造体とバイト配列間のパッキング/アンパッキング。
  • **環境変数シリアライズ**: apr_table_t 形式の環境変数を FastCGI の PARAMS レコード形式に変換するための計算およびエンコード支援。

重要なシンボル

データ構造

  • ap_fcgi_header: FastCGI レコードの固定長ヘッダー（8バイト）。
  • ap_fcgi_begin_request_body: BEGIN_REQUEST レコードのボディ構造体。

変換・操作関数

  • **パッキング/アンパッキング**: ap_fcgi_header_to_array, ap_fcgi_header_from_array, ap_fcgi_header_fields_from_array。
  • **ボディ構築**: ap_fcgi_begin_request_body_to_array, ap_fcgi_fill_in_header, ap_fcgi_fill_in_request_body。
  • **環境変数エンコード**: ap_fcgi_encoded_env_len（サイズ計算）、ap_fcgi_encode_env（シリアライズ実行）。

定数

  • AP_FCGI_HEADER_LEN: ヘッダーサイズ（8バイト）。
  • AP_FCGI_MAX_CONTENT_LEN: コンテンツ部の最大長（65535バイト）。
  • AP_FCGI_VERSION_1: プロトコルバージョン。
  • 各種 AP_FCGI_* マクロ: レコードタイプ（PARAMS, STDIN 等）およびロール定義。

制御フローとデータフロー

1. **ヘッダー処理**: 通信路から読み取ったバイト配列を ap_fcgi_header_from_array で構造体に変換し、type や contentLength を判定します。 2. **環境変数転送**:

  • ap_fcgi_encoded_env_len を呼び出し、現在のバッファ制限内でエンコード可能な環境変数の範囲を特定します。
  • ap_fcgi_encode_env を呼び出し、指定されたバッファに NAME-VALUE ペアを書き込みます。
  • starting_elem ポインタを更新しながら、全変数が転送されるまでこのプロセスを繰り返します。

直接依存

  • httpd.h: Apache の基本型および request_rec 定義。

関連コード

  • server/util_fcgi.c: 本ヘッダーで宣言された関数の実装。
  • modules/proxy/mod_proxy_fcgi.c: 本ヘッダーを利用して FastCGI バックエンドと通信する主要モジュール。

読者向けメモ

  • **メモリ管理**: 本ヘッダー内の関数はメモリ割り当てを行いません。呼び出し側がバッファ（unsigned char a[] や void *buffer）のライフサイクルとサイズを管理する必要があります。
  • **レイアウト**: ap_fcgi_header 等の構造体は、ネットワークプロトコルに準拠したフィールド順序になっていますが、コンパイラのパディングに依存しないよう、バイト配列への変換関数を介して操作することが推奨されます。
  • **制約**: ap_fcgi_encode_env は apr_table_t を処理しますが、FastCGI の仕様上、環境変数の名前と値の長さが 127 バイトを超える場合のエンコーディングルール（長さフィールドのビットフラグ）が実装に含まれている点に注意してください。
