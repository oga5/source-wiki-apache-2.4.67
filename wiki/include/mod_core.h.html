include/mod_core.h

概要

mod_core.h は、Apache HTTP Server のコアモジュール（mod_core）が提供する内部機能およびフィルタインターフェースを定義するプライベートヘッダーファイルです。HTTP プロトコルの基本処理や、コアフィルタの制御に必要なシンボルを公開します。

モジュール内での役割

Apache のフィルタチェーンにおいて、HTTP プロトコルの処理（チャンク転送、バイトレンジ、エラー処理など）を担うコアフィルタのハンドルと、それらを操作するための関数群を提供します。

責務

  • **コアフィルタの管理**: HTTP 入出力フィルタやチャンクフィルタなどの主要なフィルタハンドルを外部公開する。
  • **HTTP プロトコル処理の補助**: TRACE や OPTIONS メソッドへの応答生成、HTTP ヘッダーの基本構築機能を提供する。
  • **乱数生成の初期化**: プロセス起動時およびフォーク後の乱数生成器（RNG）の管理。

重要なシンボル

  • ap_http_input_filter_handle / ap_http_header_filter_handle: HTTP プロトコル処理の基盤となるフィルタハンドル。
  • ap_http_filter: HTTP リクエストの入力処理を行うメインフィルタ関数。
  • ap_basic_http_header: HTTP レスポンスヘッダーの最小構成を生成する関数。
  • ap_send_http_trace / ap_send_http_options: 特定の HTTP メソッドに対する標準的な応答処理。

制御フローとデータフロー

本ファイルは主に宣言のみを提供します。実際のデータフローは、Apache のフィルタチェーンを通じて apr_bucket_brigade が各フィルタ関数（ap_http_filter 等）に渡されることで発生します。モジュールはこれらのハンドルを使用して、自身のフィルタをチェーン内の適切な位置に挿入・制御します。

直接依存

  • apr.h: Apache Portable Runtime の基本定義。
  • apr_buckets.h: データのストリーム処理（バケットブリゲード）の定義。
  • httpd.h: Apache の主要なデータ構造（request_rec 等）の定義。
  • util_filter.h: フィルタチェーンの管理構造。

関連コード

  • modules/http/mod_core.c: 本ヘッダーで宣言された関数の実装が含まれる。
  • include/util_filter.h: フィルタチェーンの仕組みそのものを理解するために参照が必要。
  • include/http_protocol.h: HTTP プロトコルレベルのより広範な処理定義。

読者向けメモ

  • 本ファイルは mod_core の内部インターフェースであり、一般的なモジュール開発者が直接利用する機会は限られます。
  • ap_basic_http_header の使用には注意が必要です。HTTP/1.1 の仕様準拠は完全なヘッダー生成に依存しているため、この関数を不適切に使用するとプロトコル違反を招く可能性があります。
  • フィルタハンドルは AP_DECLARE_DATA で宣言されており、モジュール間で共有されるグローバルな参照として機能します。
