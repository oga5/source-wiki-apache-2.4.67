include/util_filter.h

概要

Apache HTTP Server のフィルタチェーン機構を定義するヘッダーファイルです。データ（バケットブリゲード）を処理するフィルタの登録、チェーンへの追加・削除、およびフィルタ間でのデータ受け渡しを行うためのインターフェースを提供します。

モジュール内での役割

HTTP リクエストおよびレスポンスの処理パイプラインにおいて、コンテンツ生成からネットワーク送信までのデータ変換層を担います。server/util_filter.c と対になり、フィルタの実行順序制御や状態管理を抽象化します。

責務

  • **フィルタ定義と登録**: フィルタのコールバック関数、型、プロトコル制約の定義。
  • **チェーン管理**: フィルタの動的な追加・削除および実行順序の決定。
  • **データ受け渡し**: apr_bucket_brigade を用いたストリーム処理の抽象化。
  • **ユーティリティ提供**: フィルタ内でのバッファリング、フラッシュ、フォーマット出力の簡略化。

重要なシンボル

フィルタ定義・登録

  • ap_filter_t: インストールされたフィルタのインスタンス構造体。
  • ap_filter_rec_t: フィルタの定義情報（名前、関数ポインタ、型）。
  • ap_register_input_filter / ap_register_output_filter_protocol: フィルタをシステムに登録する関数。

フィルタ実行・制御

  • ap_get_brigade / ap_pass_brigade: チェーンの次段へデータを渡すための主要インターフェース。
  • ap_input_mode_t: 入力フィルタの読み取りモード（READBYTES, GETLINE 等）。
  • ap_filter_type: フィルタの優先順位を決定する分類（RESOURCE から NETWORK まで）。

ユーティリティ・マクロ

  • ap_fwrite, ap_fputs, ap_fprintf: フィルタ内でのバッファリング書き込み用ラッパー。
  • AP_FILTER_PROTO_*: フィルタのプロトコル制約フラグ（キャッシュ不可、バイトレンジ不可など）。

制御フローとデータフロー

1. **登録**: モジュール初期化時に ap_register_*_filter を呼び出し、フィルタ名をコールバック関数と関連付けます。 2. **接続**: リクエスト処理中に ap_add_*_filter が呼ばれ、フィルタチェーンにノードが挿入されます。 3. **データ処理**:

  • 出力の場合、ap_pass_brigade を通じてブリゲードがチェーンを順次通過します。
  • 各フィルタは ctx ポインタを使用して状態を保持し、必要に応じて ap_save_brigade でデータを一時保存します。

    • 4. **終了**: ストリーム終了マーカーが到達すると、各フィルタは内部状態をフラッシュし、リソースを解放します。

直接依存

  • apr.h, apr_buckets.h: Apache Portable Runtime のバケット管理機能。
  • httpd.h: Apache のコア構造体（request_rec, conn_rec 等）。
  • stdarg.h

関連コード

  • server/util_filter.c: 本ヘッダーで宣言された関数の実装およびフィルタチェーンの内部管理ロジック。
  • include/http_protocol.h: フィルタチェーンを介してやり取りされる HTTP プロトコル処理。

読者向けメモ

  • **メモリ所有権**: ap_pass_brigade に渡されるブリゲードの所有権は呼び出し元にあります。フィルタ内でデータを保持したい場合は ap_save_brigade を使用してください。
  • **実行順序**: フィルタは ap_filter_type に基づいてソートされます。同じ型の場合は FIFO で実行されます。
  • **制約**: AP_FTYPE_CONNECTION 以上のフィルタはサブリクエスト内では使用できません。
  • **省略事項**: 本ファイルはインターフェース定義に特化しており、フィルタの具体的な実装（mod_deflate 等）や、内部的なトライ木構造によるフィルタ検索の実装詳細は server/util_filter.c に委譲されています。
