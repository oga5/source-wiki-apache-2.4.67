include/mod_auth.h

概要

Apache HTTP Server における認証（Authentication）および認可（Authorization）フレームワークの基盤となるヘッダーファイルです。モジュールが認証・認可プロバイダーとして機能するためのインターフェースと、共通のデータ構造を定義しています。

モジュール内での役割

Apache の認証・認可サブシステム（mod_authn_*, mod_authz_*）における「共通言語」としての役割を果たします。各プロバイダーモジュールは、このファイルで定義された構造体や列挙型に従うことで、コアサーバーと疎結合に連携できます。

責務

  • 認証（authn）および認可（authz）プロバイダーのインターフェース定義。
  • プロバイダー間での結果共有や状態管理のための定数・列挙型の提供。
  • 認証結果のキャッシュや設定解析のためのフック用データ構造の提供。

重要なシンボル

  • authn_provider / authz_provider: 各プロバイダーが実装すべき関数ポインタを保持する構造体。
  • authn_status / authz_status: 処理結果（成功、拒否、エラー等）を示す列挙型。
  • AUTHN_PROVIDER_GROUP / AUTHZ_PROVIDER_GROUP: プロバイダー登録時に使用するグループ識別子。
  • SATISFY_ALL / SATISFY_ANY: 複数の認証・認可条件を評価する際の論理条件を指定する定数。

制御フローとデータフロー

1. **プロバイダー登録**: 各モジュールは ap_register_provider を使用し、authn_provider または authz_provider 構造体を登録します。 2. **認証フロー**: サーバーはリクエスト処理中に登録された check_password や get_realm_hash を呼び出し、authn_status を受け取ります。 3. **認可フロー**: parse_require_line で設定を事前解析し、リクエスト時に check_authorization を呼び出してアクセス可否を判定します。 4. **キャッシュ連携**: ap_authn_cache_store（オプション関数）を通じて、認証結果を外部キャッシュモジュールと共有可能です。

直接依存

  • apr_pools.h, apr_hash.h: メモリ管理およびハッシュテーブル用。
  • apr_optional.h: オプション関数の動的リンク用。
  • httpd.h: request_rec 等のコアデータ構造。
  • http_config.h: 設定ディレクティブ処理用。

関連コード

  • mod_authn_core.c / mod_authz_core.c: 本ヘッダーで定義されたプロバイダーを管理・実行するコアモジュール。
  • mod_authn_file.c 等: 本ヘッダーを利用した具体的な認証プロバイダーの実装例。

読者向けメモ

  • 本ファイルはインターフェース定義のみであり、ロジックは含みません。
  • authz_provider の parse_require_line は、設定読み込み時に一度だけ実行されるため、パフォーマンス最適化のために積極的に活用してください。
  • SATISFY 定数は、Require ディレクティブと認証設定の組み合わせにおいて、アクセス制御の論理を決定する重要なフラグです。
