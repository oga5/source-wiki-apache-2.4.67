include/http_vhost.h

概要

Apache HTTP Server におけるバーチャルホスト（vhost）機能のインターフェースを定義するヘッダーファイルです。IPベースおよび名前ベースのバーチャルホストの解決、設定、およびリクエスト処理時のサーバー選択ロジックを提供します。

モジュール内での役割

APACHE_CORE モジュールの一部として、サーバーの起動からリクエスト処理に至るまでのライフサイクル全体で、リクエストがどの server_rec（バーチャルホスト設定）に属するかを決定する中核的な役割を担います。

責務

  • **設定の初期化と構築**: サーバー起動時の設定読み込み前後の初期化およびコンパイル。
  • **ディレクティブの解析**: <VirtualHost> や NameVirtualHost などの設定ディレクティブの処理。
  • **リクエストのルーティング**: 接続情報やリクエストヘッダー（Hostヘッダー等）に基づき、適切なバーチャルホストを特定し r->server を更新する。
  • **イテレーション**: 特定の接続に関連付けられたバーチャルホストを列挙するためのコールバック機構の提供。

重要なシンボル

  • ap_init_vhost_config / ap_fini_vhost_config: 設定フェーズにおけるvhostテーブルの構築関数。
  • ap_update_vhost_given_ip: 接続確立時にIPアドレスから候補となるvhostを特定する。
  • ap_update_vhost_from_headers: リクエストヘッダーを解析し、最終的なvhostを決定する。
  • ap_vhost_iterate_conn_cb: 接続に関連するvhostを走査するためのコールバック型。

制御フローとデータフロー

1. **起動時**: ap_init_vhost_config で初期化され、設定読み込み後に ap_fini_vhost_config で実行時の高速検索用テーブルが構築されます。 2. **接続時**: ap_update_vhost_given_ip が呼ばれ、接続のIP/ポートからベースとなるvhostが特定されます。 3. **リクエスト処理時**: ap_update_vhost_from_headers が実行され、HTTPの Host ヘッダーに基づき、より詳細な名前ベースのvhostへ r->server が切り替えられます。

直接依存

本ファイルは直接的な #include を持たず、主に server_rec や conn_rec、request_rec といった Apache のコア構造体定義に依存しています。

関連コード

  • server/vhost.c: 本ヘッダーで宣言された関数の実装が含まれており、実際の検索ロジックやテーブル管理が行われています。
  • include/httpd.h: server_rec や conn_rec などの主要なデータ構造が定義されています。
  • include/http_config.h: 設定コマンドの処理に必要な cmd_parms が定義されています。

読者向けメモ

  • 本ファイルは「バーチャルホストの解決」という複雑な処理の入り口です。実装の詳細（ハッシュテーブルの構造やマッチングアルゴリズム）を追う際は、server/vhost.c を参照してください。
  • ap_update_vhost_from_headers はリクエスト処理の初期段階で必ず呼び出される重要な関数であり、ここでの変更はサーバー全体のルーティング挙動に直結します。
  • require_match フラグを持つ _ex 版の関数は、厳密なホスト名マッチングが必要な場合に利用されます。
