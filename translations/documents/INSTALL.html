INSTALL

APACHE インストールの概要

クイックスタート - Unix
------------------

完全なインストール手順については、[ht]docs/manual/install.html または
http://httpd.apache.org/docs/2.4/install.html を参照してください。

     $ ./configure --prefix=PREFIX
     $ make
     $ make install
     $ PREFIX/bin/apachectl start

     注意: * PREFIX は Apache をインストールするファイルシステム上のパスに置き換えてください。
              一般的なインストールでは、PREFIX に "/usr/local/apache2" (引用符なし) を使用します。

            * 以前にインストールされた APR および APR-Util (多くの OS に付属しているものなど) を使用するか、
              apr.apache.org プロジェクトから提供されている APR および APR-Util を使用するかを検討してください。
              後者の場合は、最新バージョンをダウンロードして ./srclib/apr および ./srclib/apr-util に展開し
              (ディレクトリ名にバージョン番号は含めないでください)、./configure の --with-included-apr オプションを使用します。
              これは、システム APR がビルドされたコンパイラが手元にない場合に必要です。
              また、Apache とコードをリンクしたり、デバッガを使用してサーバーコードをステップ実行したりする開発者の場合、
              APR および APR-Util コードとのバージョンやコンパイルオプションの不一致を排除できるため、
              この方法が有利になることがあります。利便性のため、APR および APR-Util のパッケージ化されたソースコードバンドルが、
              httpd-2.X.X-deps.tar.gz として提供されることもあります。

            * Subversion から直接 Apache をビルドする開発者は、configure を実行する前に ./buildconf を実行する必要があります。
              このスクリプトはビルド環境をブートストラップするもので、Python、GNU autoconf、libtool が必要です。
              リリース tarball から Apache をビルドする場合は、buildconf を実行する必要はありません。

            * FreeBSD でスレッド化された MPM (worker など) をビルドする場合、FreeBSD 5.4-RELEASE より前のバージョンでは
              Apache でスレッドがうまく動作しないことに注意してください。それより前のバージョンの FreeBSD で
              スレッド化された Apache を試したい場合は、./configure の --with-mpm パラメータに加えて
              --enable-threads パラメータを使用してください。

            * Mac OS X (Darwin) 上で Subversion から直接ビルドする場合は、GNU Libtool 1.4.2 以降を使用するようにしてください。
              このプラットフォームの最近の開発ツールには、十分なバージョンの GNU Libtool (glibtool という名前ですが、
              buildconf はその場所を認識しています) が含まれています。

どのような設定が可能かを示す簡単な例として、インストールツリー /sw/pkg/apache に対して、
特定のコンパイラとフラグを指定し、さらに DSO メカニズムを通じて後から読み込むための
2 つの追加モジュール mod_rewrite と mod_speling を有効にして Apache を設定する例を挙げます。

     $ CC="pgcc" CFLAGS="-O2" \
     ./configure --prefix=/sw/pkg/apache \
     --enable-rewrite=shared \
     --enable-speling=shared 

Apache 2.4 のすべての設定フラグを確認する最も簡単な方法は、./configure --help を実行することです。
