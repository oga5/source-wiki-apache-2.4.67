modules/examples/README

Apache 2.0 Example Module用README
[1997年4月、2000年5月更新]

Apache配布ディレクトリツリー下のsrc/modules/examplesディレクトリにあるファイルは、Apache APIを使用したモジュールを作成したい方へのサンプルとして提供されています。

メインファイルはmod_example_hooks.cであり、すべての異なるコールバックメカニズムと呼び出し構文を説明しています。アドオンモジュールがすべてのコールバックのルーチンを含める必要があるというわけでは決してありません。むしろその逆です！

このサンプルモジュールは実際に動作するモジュールです。サーバーにリンクし、特定の場所に対して "example-hooks-handler" ハンドラを有効にしてその場所にブラウズすると、様々なコールバックが行われた際にサンプルモジュールが実行したトレースの一部が表示されます。

サンプルモジュールをサーバーに含めるには、httpdソースツリーから実行される他の ./configure 引数に --enable-example-hooks を追加してください。その後、'make' を実行します。

独自のモジュールを追加するには：

    A. cp modules/examples/mod_example_hooks.c modules/examples/mod_myexample.c
    B. ファイルを修正する
    C. modules/examples/config.m4 にエントリを追加する（例：
         APACHE_MODULE(myexample, my new module, , , no)
       最後の引数は、モジュールがデフォルトでビルドされるかどうかを指定します）
    D. --enable-myexample を指定してサーバーをビルドする

Windowsの場合、手順は少し異なります：

    A. copy modules\examples\mod_example_hooks.c modules\examples\mod_myexample.c
    B. copy modules\examples\mod_example_hooks.dsp modules\examples\mod_myexample.dsp
    C. 'example_hooks' という文字列を自分のモジュール名に置換する。
    D. 新しい .dsp を Apache.dsw ワークスペースに追加し、libapr、libaprutil、libhttpd プロジェクトへの依存関係を設定する。新しい Developer Studio 2002 から 2005 では、新しい .dsp ファイルを追加すると .vcproj ファイルに変換されます。

サンプルモジュールを有効にするには、httpd.conf ファイルに以下のようなブロックを含めてください：

    <Location /example-info>
	SetHandler example-hooks-handler
    </Location>

別の方法として、.htaccess ファイルに以下を記述し、その場所から "test.example" というファイルをリクエストすることもできます：

    AddHandler example-hooks-handler .example

サーバーをリロード/再起動した後、その場所にブラウズすると、前述の簡単な表示を確認できるはずです。
