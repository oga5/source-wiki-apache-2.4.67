modules/ssl/README

概要

本Apacheモジュールは、Eric A. YoungおよびTim J. HudsonによるSSLeayをベースとしたSSL/TLS実装ライブラリであるOpenSSLの助けを借りて、Secure Sockets Layer (SSL v2/v3) および Transport Layer Security (TLS v1) プロトコルを介し、Apache 2 Webサーバーに強力な暗号化機能を提供します。

mod_sslパッケージは1998年4月にRalf S. Engelschallによって作成され、元々はApache-SSL HTTPサーバープロジェクトで使用するためにBen Laurieが開発したソフトウェアから派生したものです。Apache 1.3向けのmod_ssl実装は、引き続きmodsslプロジェクト <http://www.modssl.org/> によってサポートされています。

ソース

ファイルの説明については、トップレベルのLAYOUTファイルを参照してください。

ソースファイルはクリーンなANSI Cで記述されており、``gcc -O -g -ggdb3 -Wall -Wshadow -Wpointer-arith -Wcast-align -Wmissing-prototypes -Wmissing-declarations -Wnested-externs -Winline'' コンパイラテスト（`gcc` がGCC 2.95.2以降であることを前提）を警告なしで通過します。変更や追加を行う際は、ソースが引き続きこのコンパイラテストを通過することを確認してください。

関数

ソースコード内では、接頭辞によって識別可能な以下の種類の関数に遭遇します。

   ap_xxxx() ............... Apache API関数
   ssl_xxxx() .............. mod_ssl関数
   SSL_xxxx() .............. OpenSSL関数 (SSLライブラリ)
   OpenSSL_xxxx() .......... OpenSSL関数 (SSLライブラリ)
   X509_xxxx() ............. OpenSSL関数 (Cryptoライブラリ)
   PEM_xxxx() .............. OpenSSL関数 (Cryptoライブラリ)
   EVP_xxxx() .............. OpenSSL関数 (Cryptoライブラリ)
   RSA_xxxx() .............. OpenSSL関数 (Cryptoライブラリ)

データ構造

ソースコード内では、以下のデータ構造に遭遇します。

   server_rec .............. Apache (バーチャル) サーバー
   conn_rec ................ Apache 接続
   request_rec ............. Apache リクエスト
   SSLModConfig ............ mod_ssl (グローバル) モジュール設定
   SSLSrvConfig ............ mod_ssl (バーチャル) サーバー設定
   SSLDirConfig ............ mod_ssl ディレクトリ設定
   SSLConnConfig ........... mod_ssl 接続設定
   SSLFilterRec ............ mod_ssl フィルタコンテキスト
   SSL_CTX ................. OpenSSL コンテキスト
   SSL_METHOD .............. OpenSSL プロトコルメソッド
   SSL_CIPHER .............. OpenSSL 暗号スイート
   SSL_SESSION ............. OpenSSL セッション
   SSL ..................... OpenSSL 接続
   BIO ..................... OpenSSL 接続バッファ

これらがどのように関連し、連鎖しているかの概要については、README.dsov.{fig,ps} のページを参照してください。そこには、これらのデータ構造の概要図が含まれています。DIN A4用紙サイズ用に設計されていますが、XFigのExportパネルで倍率を指定することで、より小さなバージョンを簡単に生成できます。

非互換性

Apache 1.3のmod_ssl 2.xと、このApache 2用mod_sslバージョンとの間には、以下の意図的な非互換性が存在します。

 o EAPIベースのSSL_VENDOR関連機能はすべて削除されました。
 o EAPIベースのSSL_COMPAT関連機能はすべて削除されました。
 o <IfDefine> 変数 MOD_SSL は自動的に提供されなくなりました。

主な変更点

Apache 2 mod_sslの変更履歴の完全なリストについては、トップレベルディレクトリのCHANGESファイルを参照してください。以下は、Apache 1.3のmod_ssl 2.xからこのApache 2用mod_sslバージョンへ行われた主な変更点の要約です。

 o DBMベースのセッションキャッシュは、APRのDBM APIのみに基づくようになりました。
 o 共有メモリベースのセッションキャッシュは、APRのAPIに基づくようになりました。
 o SSL I/Oは、BUFFではなくフィルタとして実装されるようになりました。
 o ap_global_ctxを排除しました。永続的な情報は process_rec->pool->user_data に格納されます。ssl_pphrase_Handle_CB() および ssl_config_global_* () 関数には、SSLModConfigRecを取得するために使用される追加のパラメータ "server_rec *" が追加されました。
 o 再起動を適切にサポートし、既に実行中のサーバーにmod_sslを追加したり、再起動時にサーバーの証明書/鍵を変更したりできるようになりました。
 o さまざまなパフォーマンスの向上。
 o プロキシサポートは「拡張機能」ではなくなりました。mod_sslコアの大部分（ssl_engine_{init,kernel,config}.c）は、プロキシモードで再利用できるように汎用化されました。
   - mod_proxyがプロキシサポートを有効にするためのオプション関数 ssl_proxy_enable が提供されています。
   - プロキシサポートには 'SSLProxyEngine on' の設定が必要です。
   - プロキシは、元のSSLProxy* ディレクティブに加え、SSLProxyCARevocation{Path,File} をサポートするようになりました。
 o ディレクトリごとの SSLCACertificate{File,Path} はスレッドセーフになりましたが、OpenSSLへの SSL_set_cert_store パッチが必要です。
 o ssl_engine_{ds,ext}.c ソースファイルは廃止され、存在しません。
