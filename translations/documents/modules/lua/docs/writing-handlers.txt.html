mod_luaは、CGIスタイルのようにスクリプト本体を評価するのではなく、常にハンドラー用の関数を呼び出そうとします。ハンドラー関数は次のような形式になります。

    -- example.lua --
    require "string"

    function handle_something(r)
        r.content_type = "text/plain"
        r:puts("Hello Lua World!\n")
    
        if r.method == 'GET' then
            for k, v in pairs( r:parseargs() ) do
                r:puts( string.format("%s: %s", k, v) )
            end
        elseif r.method == 'POST' then
            for k, v in pairs( r:parsebody() ) do
                r:puts( string.format("%s: %s", k, v) )
            end
        else
            r:puts("unknown HTTP method " .. r.method)
        end 
    end

このハンドラー関数は、URIまたはフォームエンコードされた引数をプレーンテキストページに出力するだけです。

これは、同じスクリプト内に複数のハンドラー（またはフック、フィルター）を持てることを意味し（実際、それが推奨されています）、それを可能にします。

データ構造:
    request_rec:
        request_recはユーザーデータとしてマッピングされます。これには、便利な操作を可能にするメタテーブルが用意されています。大部分において、request_rec構造体と同じフィールドを持ちます（より詳細なドキュメントが用意されるまではhttpd.hを参照してください）。その多くは読み取りだけでなく書き込みも可能であり、（少なくとも）以下のメソッドを持っています。
        
        r:puts("hello", " world", "!") -- レスポンスボディに出力する
        
        -- ログ出力関数
        r:debug("This is a debug log message")
        r:info("This is an info log message")
        r:notice("This is an notice log message")
        r:warn("This is an warn log message")
        r:err("This is an err log message")
        r:alert("This is an alert log message")
        r:crit("This is an crit log message")
        r:emerg("This is an emerg log message")
