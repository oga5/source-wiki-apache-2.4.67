Lua 5.1 をインストール
    http://www.lua.org/download.html
    
    Lua はコンパイルに autoconf を使用しません。つまり、./configure は使用しません。ただし、優れたビルド手順が用意されているため、問題なく進むはずです。
    
    私は Lua のインストール先ディレクトリを変更するのが好みです。これを行うには、Lua の設定用 Makefile で LUA_TOP を設定する必要があります。この手順では、LUA_TOP を /Users/brianm/.opt/lua-5.1.2 に設定しています。このディレクトリについては後述します。
    
    
Apache HTTPD 2.2 をインストール
    http://httpd.apache.org/download.cgi
    
    mod_wombat を読み込めるように動的モジュール読み込み (--enable-so) を有効にする限り、Apache は基本的に好きな方法でビルドできます。
    
    スレッド対応 MPM を使用することをお勧めします。mod_wombat はそれらと問題なく動作します。

    私は以下のフラグでビルドしています：
        
    ./configure --prefix=/Users/brianm/.opt/httpd-2.2.4-worker-wombat \
                --with-mpm=worker  \
                --enable-so
    
    
libapreq2 をインストール
    http://httpd.apache.org/apreq/download.cgi
        ダウンロードリンクはページ本文内にあります。左側の列にある「Download!」リンクの下ではありません。
    
    現在、mod_wombat はエンティティボディの解析に libapreq2 を必要とします。この依存関係は近い将来オプションになる可能性がありますが、現時点では必要です。
    
    私は以下のフラグでビルドしています：
    
    ./configure --prefix=/Users/brianm/.opt/libapreq2-2.0.8 \
      --with-apache2-apxs=/Users/brianm/.opt/httpd-2.2.4-worker-wombat/bin/apxs
    
    
Subversion から mod_wombat をインストール
    http://svn.apache.org/repos/asf/httpd/mod_wombat/trunk
    
    Subversion からビルドする場合の最初のステップは、autoconf をブートストラップすることです。これを行うには、bootstrap スクリプトを実行します：
    
    ./bootstrap
    
    bootstrap スクリプトが libtoolize または glibtoolize が見つからないというエラーを報告する場合がありますが、両方とも見つからないという報告でなければ問題ありません。このスクリプトは単に autoconf の魔法をセットアップするだけです。
    
    その後は、通常の configure とビルドを行います：
    
    ./configure  --with-lua=/Users/brianm/.opt/lua-5.1.2/ \
      --with-apxs=/Users/brianm/.opt/httpd-2.2.4-worker-wombat/bin/apxs \
      --with-apreq2=/Users/brianm/.opt/libapreq2-2.0.8/
      
    コンパイル (make) 中に libapreq2 ヘッダーファイルが見つからないというエラーが報告された場合は、私 ( brianm@apache.org ) までお知らせください。これは一部の設定で発生しますが、libapreq2 のインストールに関する奇妙な挙動をまだ完全に把握できていないためです。--prefix 設定オプションを使用して libapreq2 をビルドすれば、常にうまくいくようです。
  
      
以上です。mod_wombat の設定については、basic-configuration.txt ドキュメントを参照してください。
