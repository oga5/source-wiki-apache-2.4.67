Apache HTTP Server

  これは何か？
  -----------

  Apache HTTP Server は、強力で柔軟な HTTP/1.1 準拠の Web サーバーです。元々は NCSA HTTP Server の代替として設計されましたが、現在ではインターネット上で最も人気のある Web サーバーへと成長しました。Apache Software Foundation のプロジェクトとして、開発者たちは、自由に利用可能なソースコードを備えた、堅牢で商用グレードの標準ベースのサーバーを共同で開発・維持することを目指しています。

  最新バージョン
  ------------------

  最新バージョンの詳細は、Apache HTTP Server プロジェクトページ（https://httpd.apache.org/）で確認できます。

  本リリースの時点で利用可能なドキュメントは、HTML 形式で docs/manual/ ディレクトリに含まれています。最新のドキュメントは https://httpd.apache.org/docs/2.4/ で確認できます。

  暗号ソフトウェアに関する通知
  -----------------------------

  この配布物には、暗号ソフトウェアと共に使用するように設計されたソフトウェアが含まれている場合があります。現在居住している国によっては、暗号ソフトウェアの輸入、所持、使用、および/または他国への再輸出に制限がある場合があります。暗号ソフトウェアを使用する前に、その使用が許可されているかどうか、暗号ソフトウェアの輸入、所持、使用、および再輸出に関する自国の法律、規制、方針を確認してください。詳細については <https://www.wassenaar.org/> を参照してください。

  米国商務省産業安全保障局 (BIS) は、本ソフトウェアを輸出管理品目番号 (ECCN) 5D002.C.1 に分類しています。これには、非対称アルゴリズムを用いた暗号機能を使用または実行する情報セキュリティソフトウェアが含まれます。この Apache Software Foundation の配布物の形式および方法は、オブジェクトコードとソースコードの両方について、ライセンス例外 ENC Technology Software Unrestricted (TSU) に基づく輸出の対象となります（BIS 輸出管理規則、セクション 740.13 を参照）。

  以下に、暗号ソフトウェアの輸出管理の対象となる可能性のある含まれるファイルの詳細を記載します。

    Apache httpd 2.0 以降のバージョンには、以下のディレクトリに mod_ssl モジュールが含まれています。
       modules/ssl/
    これは、OpenSSL やオペレーティングシステムのプラットフォーム固有の SSL 機能といった汎用暗号ライブラリへの呼び出しを実行することで、SSL 暗号化ネットワークソケット経由の接続を構成およびリッスンするためのものです。

    さらに、apr-util の一部のバージョンでは、OpenSSL、NSS、またはオペレーティングシステムのプラットフォーム固有の機能といった汎用暗号ライブラリを利用する対称暗号機能のための抽象インターフェースを提供しています。このインターフェースは apr_crypto インターフェースとして知られており、/crypto ディレクトリ配下に実装されています。apr_crypto インターフェースは、以下のディレクトリで利用可能な mod_session_crypto モジュールによって使用されます。
      modules/session
    これはセッション情報のオプションの暗号化のためのものです。

    パッケージ名に "crypto" という単語が含まれる Apache httpd の一部のオブジェクトコード配布物には、<https://www.openssl.org/source/> からオープンソース形式で配布されている OpenSSL 暗号ライブラリのオブジェクトコードが含まれている場合があります。

  上記のファイルはオプションであり、暗号機能が不要な場合や再配布から除外する必要がある場合は削除できます。パッケージ名に "nossl" という単語が含まれる Apache httpd の配布パッケージは、上記のファイルを含まずに作成されているため、この通知の対象外となります。

  連絡先
  --------

     o 新しいコードのリリース、バグ修正、セキュリティ修正、Apache サーバーに関する一般的なニュースや情報について知りたい場合は、<https://httpd.apache.org/lists.html#http-announce> に記載されている通り、apache-announce メーリングリストに登録してください。

     o Apache の運用に関する無料のサポートが必要な場合は、<https://httpd.apache.org/support.html> のリソースを参照してください。

     o Apache に関する具体的なバグ報告がある場合は、<https://httpd.apache.org/bug_report.html> にあるバグ報告の手順を参照してください。

     o Apache の積極的な開発に参加したい場合は、<https://httpd.apache.org/lists.html#http-dev> に記載されている通り、`dev@httpd.apache.org' メーリングリストに登録してください。
